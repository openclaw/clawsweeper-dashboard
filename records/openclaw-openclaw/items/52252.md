## Summary

Keep open. Current main still only stabilizes Homebrew Cellar Node paths, and this PR addresses the real Volta daemon pinning problem from #52184, but the submitted implementation underfixes supported custom Volta homes and Windows Volta layouts, so it should not merge as-is.

## What This Changes

This PR adds Volta tools/image path rewriting to `resolveStableNodePath` plus tests for accessible-shim and missing-shim fallback behavior.

## Best Possible Solution

Land a narrow daemon runtime-path fix that recognizes Volta image paths by Volta home root instead of hard-coding `~/.volta`, preserves the existing missing-shim fallback and Homebrew behavior, and adds explicit Unix custom-root plus Windows layout tests.

## Reproduction Assessment

Yes. A focused unit check can call `resolveStableNodePath` with a Volta image path and an accessible shim; current main returns the image path unchanged, and the submitted PR still fails for `/opt/volta/...` custom roots and Windows Volta layouts.

## Solution Assessment

No, not as submitted. The approach is the right seam, but the implementation should be layout-aware across default Unix, custom `VOLTA_HOME`, and Windows Volta paths instead of matching only `/.volta/.../bin/node[.exe]`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Match custom Volta homes when rewriting image paths:** `src/infra/stable-node-path.ts:19`
  - body: The new regex requires a `/.volta/tools/image/...` segment. OpenClaw already preserves `VOLTA_HOME/bin` in service PATH, so a supported custom root like `/opt/volta/tools/image/node/<ver>/bin/node` keeps returning the version-pinned binary and can still break after Volta changes versions.
  - confidence: 0.94
- **[P2] Use Volta's Windows image and shim layout:** `src/infra/stable-node-path.ts:19-25`
  - body: The Windows branch matches `.../bin/node.exe` and returns `bin/node.exe`, but Volta stores Windows Node images directly under the version directory and creates shims under `bin` as command/Git-Bash shims. Windows Volta installs therefore will not be rewritten to a stable shim.
  - confidence: 0.89
