## Summary

Keep open: current main still lacks Volta image-path canonicalization, and this PR is directionally useful but not mergeable because the matcher only covers the default Unix `.volta` layout and misses supported custom Volta homes and Windows image layouts.

## What This Changes

The PR adds Volta tools/image path rewriting to `resolveStableNodePath` plus unit tests for accessible and missing default Unix shims.

## Best Possible Solution

Land a narrow current-main repair that derives the Volta home from the matched image root, covers Unix default/custom and Windows default/custom layouts, preserves missing-shim fallback, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main returns Volta `tools/image` paths unchanged, and the PR still returns custom `VOLTA_HOME` or Windows Volta image paths unchanged despite an accessible stable shim.

## Solution Assessment

No, not as submitted. Rewriting Volta image paths to stable shims is the right fix direction, but the implementation should parse `<VOLTA_HOME>/tools/image/node/<version>/...` generically and cover platform-specific layouts before merge.
