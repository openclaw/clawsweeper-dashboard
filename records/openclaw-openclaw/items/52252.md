## Best Possible Solution

## Solution Assessment

No, not as submitted. Resolver-level canonicalization is the right boundary, but the matcher should follow Volta's default/custom Unix and native Windows layouts before merge.

Full review comments:

- **[P1] Match custom Volta homes:** `src/infra/stable-node-path.ts:19`
  - body: The matcher only recognizes `<home>/.volta/tools/image/...`, but daemon service PATH handling already supports `${VOLTA_HOME}/bin`.   - confidence: 0.94
- **[P2] Handle native Windows Volta layout:** `src/infra/stable-node-path.ts:19-25`
  - body: The Windows handling assumes a Unix-shaped `.volta/.../bin/node.exe` image and a `node.exe` shim. Volta's Windows layout puts Node images at the version root and shims at `node.cmd`, so native Windows Volta paths fall through unchanged or construct a nonexistent stable shim.
  - confidence: 0.88
