## Summary

Keep open. Current main and the latest release still default the Windows Startup-folder fallback to a minimized cmd.exe launcher; the existing hidden VBS launcher is only used when an internal environment flag is set.

## What This Changes

The issue asks for the Windows Startup-folder gateway fallback to launch without a visible cmd.exe flash at login.

## Best Possible Solution

Make the existing hidden launcher path the canonical default for the Startup-folder fallback, preserve legacy `.cmd` detection and cleanup, and prove the Windows login behavior after the change.

## Reproduction Assessment

Yes from source: current main and `v2026.6.1` still render the default Startup-folder fallback as `start "" /min cmd.exe /d /c ...`, with the VBS launcher only behind `OPENCLAW_WINDOWS_TASK_HIDDEN_LAUNCHER`. I did not run a live Windows login repro in this read-only review.

## Solution Assessment

Yes: reusing the existing hidden launcher as the default appears to be the narrowest maintainable fix. The repair should keep status detection, uninstall cleanup, and legacy `.cmd` handling intact rather than adding another launcher mode.
