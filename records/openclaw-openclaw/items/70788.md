## Summary

Keep open. Current main and latest release still use the minimized cmd.exe Startup-folder launcher for the ordinary Windows gateway fallback, while the hidden VBS launcher exists only behind `OPENCLAW_WINDOWS_TASK_HIDDEN_LAUNCHER`; the related closed update-restart issue does not cover this login-item path.

## What This Changes

The issue requests a silent Windows Startup-folder launcher so gateway startup at login does not briefly show a cmd.exe window.

## Best Possible Solution

Make the hidden launcher the canonical Windows gateway Startup-folder fallback, keep uninstall/status tolerant of legacy cmd entries, update focused tests, and coordinate with the open scheduled-task visible-window tracker if the same launcher helper should cover both paths.

## Reproduction Assessment

Yes from source inspection: current main still writes the reported `start "" /min cmd.exe /d /c` launcher by default, and tests explicitly assert that default. I did not run a live Windows login-session visual reproduction in this read-only review.

## Solution Assessment

Yes, the right fix is likely to promote the existing hidden VBS launcher path to the default for this Windows gateway fallback while preserving legacy cleanup. The exact PR should still include Windows after-fix proof because the user-visible claim is no visible window flash.
