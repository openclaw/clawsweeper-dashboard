## Summary

Keep open. Current main and the latest release still leave the OpenClaw Gateway Startup-folder fallback on the visible `.cmd` path unless the invocation environment explicitly opts into the hidden launcher, and https://github.com/openclaw/openclaw/pull/91249 is the open linked fix candidate rather than a merged implementation.

## What This Changes

The issue asks OpenClaw to suppress the Windows Startup-folder fallback console flash by replacing the default minimized `cmd.exe` launcher with a hidden `wscript.exe`/VBScript launcher.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/91249, or a narrow replacement, after proving the native Windows login item is hidden and resolving the VBS compatibility/docs concerns.

## Reproduction Assessment

Yes, at source level: the current gateway Startup fallback still defaults to a `.cmd` entry that runs `start "" /min cmd.exe /d /c`, while the hidden VBS path is opt-in. I did not run a native Windows login-session visual repro in this read-only review.

## Solution Assessment
