## Summary

Keep open. Current main and v2026.5.18 still default the Windows gateway Startup-folder fallback to a minimized cmd.exe launcher; the merged hidden-launcher work is present but currently opt-in and node-focused, not a gateway fallback default fix.

## What This Changes

The issue asks to hide the Windows gateway Startup-folder login item by launching through a wscript/VBS hidden launcher instead of minimized cmd.exe, with matching install and uninstall cleanup.

## Best Possible Solution

Make the gateway Startup-folder fallback launch hidden by default, preserve status/stop/restart detection and uninstall cleanup, and leave broader Scheduled Task policy changes to their own review.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: force the Scheduled Task access-denied/hung fallback and inspect the generated Startup-folder script. Current tests assert the default script still contains the minimized cmd.exe launcher; no live Windows login run was performed in this review.

## Solution Assessment

Yes on the fix boundary: a narrow hidden Startup-folder launcher is the maintainable path. The exact .cmd-plus-sidecar versus direct .vbs entry shape should be tested against gateway detection, cleanup, and Windows login behavior.
