## Summary

Keep open. Current main and the latest release still generate the Windows Startup-folder fallback as a minimized `cmd.exe` launcher, and focused tests still assert that launcher shape, so the reported login-window flash remains an actionable Windows startup bug.

## What This Changes

The issue requests replacing the Windows Startup-folder fallback's minimized `cmd.exe` launcher with a hidden launcher while preserving the existing `OpenClaw Gateway.cmd` detector path and uninstall cleanup.

## Best Possible Solution

Keep the fix scoped to the Windows Startup-folder fallback: preserve the existing `.cmd` path used for detection, launch the gateway without a visible console frame, remove any generated sidecar on uninstall, and validate on Windows 11 login startup.
