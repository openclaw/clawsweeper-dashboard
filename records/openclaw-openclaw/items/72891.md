## Summary

Keep open. Current main still has no Discord voice realtime-provider mode: the schema, docs, UI metadata, tests, and runtime all describe the existing segmented STT -> text agent -> TTS pipeline, while related #73699 remains open for the session/transcript/tool-routing decision.

## What This Changes

Request adding Discord voice configuration and runtime support to select a realtime audio provider such as Gemini Live instead of only using the current text-mediated voice pipeline.

## Best Possible Solution

Define an additive Discord voice realtime mode that reuses the shared realtime voice provider bridge, keeps `channels.discord.voice.model` as the text-pipeline LLM override, and lands schema, runtime, docs, and tests after the Discord session/transcript/tool model is settled.

## Reproduction Assessment

Not applicable as a feature request. The current-main verification path is high-confidence: inspect the strict Discord voice schema, docs, tests, and segment runtime to confirm the requested realtime provider mode is absent.

## Solution Assessment

Unclear. A separate Discord voice realtime config branch is plausible, but the maintainable fix depends on #73699 deciding whether Discord voice should act as I/O for the text-channel session or remain an isolated realtime session.
