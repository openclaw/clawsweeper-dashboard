## Summary

Keep open: the regression fix direction is valid, but the PR appears to inherit allow-list-only plugin repair in the automatic package-update path, which conflicts with its stated scope and broadens automatic installs.

## What This Changes

The PR wires a 2026.4.x to 2026.5.x package-update migration that installs missing configured official plugins before restart/doctor and updates upgrade-survivor planning, docs, changelog, and tests.

## Best Possible Solution

Keep the package-update migration narrowly limited to actively configured channel/provider/plugin usage, leave doctor’s broader allow-only repair intact, and land after current PR-head validation.

## Reproduction Assessment

Yes, from source and the linked report, but I did not run a live package update because this review is read-only. Current main only repairs configured missing plugins in the later doctor path, while the linked issue gives concrete 2026.4.27 to 2026.5.2 channel disappearance steps.

## Solution Assessment

No as currently written. The migration should use a package-update-specific collector or filter so allow-list-only official plugins are not auto-installed during update while the existing doctor repair behavior remains available.
