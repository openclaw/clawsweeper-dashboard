## Summary

## What This Changes

The PR changes doctor plugin dependency cleanup to prune stale version-scoped runtime-deps children while preserving the managed root/local entries, expands tests, adds a changelog entry, and includes minor formatting-only edits.

## Best Possible Solution

Keep the managed-root-preserving direction, but prune OpenClaw-owned marker-bearing non-versioned runtime-deps children alongside stale versioned children, preserve unmarked local data and current-version roots, and align docs/update-migration expectations.

Yes. Source inspection of the PR head shows the collector only targets versioned `openclaw-<version>-...` children, while current upgrade-survivor coverage seeds and probes marker-bearing non-versioned runtime-deps debris that would survive doctor cleanup.

## Solution Assessment

No. Preserving the managed root is the right direction, but the patch is incomplete until marker-bearing non-versioned OpenClaw-owned children are removed and the docs/e2e cleanup contract is updated.
