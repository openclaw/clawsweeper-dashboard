## Summary

Keep open: current main still has the unsafe broad doctor cleanup, but this PR now leaves marker-bearing legacy `plugin-runtime-deps/<plugin>` directories behind despite the existing docs and upgrade-survivor contract expecting doctor to clean them up.

## What This Changes

The PR changes doctor plugin dependency cleanup to prune stale version-scoped `plugin-runtime-deps` children while preserving the managed root, current-version roots, local entries, and adds regression tests plus a changelog entry.

## Best Possible Solution

Keep the root-preserving direction, but prune OpenClaw-owned marker-bearing legacy children and stale versioned children while preserving current-version roots and unmarked local entries, then align docs and the upgrade-survivor expectations.

## Reproduction Assessment

Yes. Source inspection shows current main deletes the whole managed runtime root, while the PR head preserves a `plugin-runtime-deps/discord` child even when it has the legacy runtime-deps marker shape seeded by upgrade-survivor.

## Solution Assessment

No. The PR is on the right path by preserving the managed root and current versioned children, but the maintainable fix also needs marker-based cleanup for non-versioned legacy plugin children plus docs and CI follow-through.
