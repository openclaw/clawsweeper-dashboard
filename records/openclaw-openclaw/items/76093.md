## Summary

Keep open. The PR is useful, but the latest head narrows runtime-deps cleanup in a way that conflicts with the existing published upgrade-survivor cleanup contract for non-versioned OpenClaw-owned runtime-deps children.

## What This Changes

This PR changes doctor cleanup to prune stale version-scoped `plugin-runtime-deps` child directories while preserving the runtime-deps root, current-version entries, local entries, and adds regression tests plus a changelog entry.

## Best Possible Solution

Land the conservative root-preserving cleanup after it also handles OpenClaw-owned non-versioned legacy children safely, or explicitly updates the upgrade-survivor/docs contract if preserving those children is the intended product behavior.

## Reproduction Assessment

Yes. The PR regression can be reproduced from existing code by following the `plugin-deps-cleanup` upgrade-survivor seed path: it creates non-versioned children under `plugin-runtime-deps`, while the PR filter only removes versioned `openclaw-<version>-...` children.

## Solution Assessment

No. The versioned matcher is a good direction for stale cache roots, but the proposed patch is not the best merge shape until the existing non-versioned legacy cleanup contract is reconciled.
