## Summary

Keep open: this PR addresses a real current-main doctor cleanup bug by narrowing stale plugin runtime-deps deletion, but it is not implemented on main and the latest PR head still has unresolved CI failures plus a small docs mismatch.

## What This Changes

The PR changes doctor plugin dependency cleanup to delete stale version-scoped `plugin-runtime-deps/openclaw-<version>-...` children while preserving the runtime root, current-version children, non-versioned entries, files, symlinks, and local data.

## Best Possible Solution

Land the conservative child-pruning implementation after CI is green and the dependency-resolution docs describe root preservation versus stale child pruning, leaving the broader cache inventory/prune CLI design to #75301.

## Reproduction Assessment

Yes. Source inspection of current main shows `doctor --fix` targets the whole plugin-runtime-deps root, and the current test asserts that deletion path without needing live filesystem mutation.

## Solution Assessment

Yes for the code direction: pruning only OpenClaw-owned stale version-scoped children is narrower and safer than deleting the managed runtime root. The PR still needs the docs mismatch and failing checks resolved before merge.
