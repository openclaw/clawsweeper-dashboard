## Summary

Keep open. The PR is aimed at a real doctor cleanup gap from #75301, but the proposed matcher is too narrow for existing legacy version-scoped runtime roots, so it would leave roots covered by the current upgrade-survivor cleanup contract.

## What This Changes

The PR changes doctor cleanup to remove stale versioned `plugin-runtime-deps/openclaw-<version>-<hash>` children while preserving the runtime-deps root, current-version children, non-versioned entries, files, symlinks, and explicit stage roots named `plugin-runtime-deps`.

## Best Possible Solution

Keep the conservative child-pruning direction, but collect every OpenClaw-owned legacy `openclaw-<version>-*` runtime root that is not for the current package version, including existing non-hex and beta-version forms, without restoring broad deletion of the whole runtime-deps root.

## Reproduction Assessment

Yes. A high-confidence reproduction path is the current `versioned-runtime-deps` upgrade-survivor fixture, or a focused unit fixture with `plugin-runtime-deps/openclaw-2026.4.24-discord`; the PR regex would not select that stale root.

## Solution Assessment

No. The proposed approach is safer than deleting the whole runtime root, but the hex-only suffix matcher is not the narrowest maintainable fix because current product coverage treats any `openclaw-<version>-*` legacy root as stale unless it belongs to the current version.
