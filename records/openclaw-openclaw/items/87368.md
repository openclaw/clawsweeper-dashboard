## Summary

Keep open: current main and v2026.6.9 still suppress configured room-event ack reactions under `messages.ackReactionScope: "all"`, and https://github.com/openclaw/openclaw/pull/87433 is the active linked fix path.

## What This Changes

The issue reports that `messages.ackReactionScope: "all"` does not send configured ack reactions for inbound room-event messages because channel-level room-event gates short-circuit the shared ack-scope decision.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/87433 or a maintainer-owned replacement after policy acceptance, matching runtime/tests/docs, and real transport proof; otherwise explicitly document that room events remain reaction-silent.

## Reproduction Assessment

Yes. Source inspection shows current main gates Discord and Telegram room-event ack reactions before the shared `all` scope can apply, and current Discord/Slack/Telegram tests encode room-event reaction silence under `all`.

## Solution Assessment

Unclear. Honoring `all` for room events is mechanically plausible and already proposed, but it changes the quiet ambient-room contract across sibling chat plugins, so the linked PR needs maintainer policy acceptance and proof before this issue closes.
