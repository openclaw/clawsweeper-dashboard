## Summary

Keep open: current main and latest release still suppress configured ack reactions for room-event inbounds despite `ackReactionScope: "all"`, and the open linked PR is the active implementation path for resolving the cross-channel policy and proof.

## What This Changes

The issue reports that `messages.ackReactionScope: "all"` is blocked by room-event gates, so configured initial ack reactions are not sent for ambient group/thread messages.

## Best Possible Solution

Use the linked PR or a maintainer-owned replacement to settle the cross-channel policy, then land matching runtime changes, tests, and public docs or explicitly document that room events stay reaction-silent.

## Reproduction Assessment

Yes, source-reproducible: current main gates Discord and Telegram room-event ack reactions before the shared `all` scope can apply, and sibling tests expect room-event reaction silence under `all`. I did not run live channel transport because this review is read-only.

## Solution Assessment

Unclear: honoring `all` for room events is mechanically plausible and already proposed, but it reverses the current quiet-room-event contract across sibling chat plugins. Maintainers should decide the policy through the linked PR before merging or closing this issue.
