## Summary

Keep open: the PR removes a misleading alias phrase, but the replacement overstates `/new` behavior for the core slash-command docs because only some surfaces create a separately selectable session.

## What This Changes

The PR modifies `docs/tools/slash-commands.md` to replace the `/reset` alias wording with separate descriptions of `/new` and `/reset`.

## Best Possible Solution

Revise the docs to remove the alias claim while scoping session-preservation language to the surfaces that actually create a separately selectable session, and describe Gateway `/reset` as an in-place current-session reset.

## Reproduction Assessment

Yes. Current main has the misleading alias sentence, and source inspection shows the replacement must account for Gateway reset-trigger behavior plus Control UI and TUI surface-specific behavior.

## Solution Assessment

No. Removing the alias wording is the right direction, but the proposed wording is not the narrowest maintainable fix because it promises accessible previous sessions too broadly.
