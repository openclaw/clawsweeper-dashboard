## What This Changes

This PR edits `docs/tools/slash-commands.md` to describe `/new`, Control UI `/new` handling, and Gateway `/reset` handling separately instead of calling `/reset` a `/new` alias.

## Best Possible Solution

Land the narrow docs clarification after normal maintainer review, and track any broader command rename, warning, or command-picker copy audit separately if maintainers want it.

## Reproduction Assessment

Yes. Current main reproduces the misleading docs sentence at `docs/tools/slash-commands.md:127`, and source plus tests show `/new` and `/reset` take different paths on Control UI, TUI, and Gateway surfaces.

## Solution Assessment

Yes. Removing the alias wording and documenting the Control UI/Gateway split is the narrow maintainable fix for the docs error; renaming `/reset` or adding stronger destructive-action policy would be a separate product decision.
