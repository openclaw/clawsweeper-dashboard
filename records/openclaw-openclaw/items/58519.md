## Summary

Keep open. Current main and v2026.5.18 still keep Slack Socket Mode in the gateway process and await the same inbound turn path; the new v2026.5.18 live report confirms the silent inbound gap is still observed, so no conservative close reason applies.

## What This Changes

The issue asks OpenClaw to prevent Slack Socket Mode pong timeouts and silent message loss when heavy agent work starves the gateway event loop.

## Best Possible Solution

Design and ship a Slack-owned transport or scheduling fix that keeps Socket Mode heartbeat, event acknowledgement, and inbound dispatch responsive under heavy gateway work, while retaining timeout and reconnect tuning as mitigations.

## Reproduction Assessment

No controlled current-main reproduction was run in this read-only review. The v2026.5.18 live report plus current source and Slack SDK source give a credible source-reproducible path for same-process heartbeat starvation and silent Slack event gaps.

## Solution Assessment

Unclear. Worker-thread transport isolation may be the right direction, but maintainers should choose between transport isolation, fast ack/dispatch scheduling, and liveness instrumentation with real Slack starvation proof.
