## Summary

This PR should stay open for maintainer handling, but it should not merge as-is: it changes the public TTS directive contract, removes currently supported inline/compact forms, and bundles unrelated Slack/config validation work after maintainer feedback asked for a split and API/docs direction.

## What This Changes

The PR refactors TTS directive parsing to a standalone-line scanner with new contract tests, and also changes Slack socket-mode Bolt options plus config plugin validation.

## Best Possible Solution

Keep the current markdown-code escape behavior, decide any standalone-only TTS directive contract deliberately, and land any approved work as focused TTS-only, Slack-only, and config-validation-only changes with docs and compatibility coverage.

## Reproduction Assessment

Yes for the behavior delta by source inspection: current main docs and tests support inline directives and compact text blocks, while the PR scanner only recognizes standalone lines. No high-confidence real-world evidence shows the current documented behavior is broken.

## Solution Assessment

No. Standalone-only semantics may be acceptable after API/product direction, but this mixed stale branch is not the narrow maintainable way to make that contract change.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Preserve supported inline TTS directives:** `src/tts/directives.ts:72-84`
  - body: The scanner only records directive bodies when `[[tts:...]]` occupies a full trimmed line. Current docs and contract tests still support inline provider overrides and compact text blocks, so tagged replies using those supported forms would stop triggering TTS or applying overrides unless the contract is deliberately migrated.
  - confidence: 0.95
- **[P2] Scope explicit-provider tokens to the declared provider:** `src/tts/directives.ts:166-173`
  - body: After `provider=...` is parsed, later directive tokens are still offered to every provider in registry order without selected-provider context. That can apply overlapping keys such as `model` or `speed` to the wrong provider, or lose unsupported-key warnings, compared with the current explicit-provider routing contract.
  - confidence: 0.91
- **[P2] Keep Slack self filtering when disabling ignoreSelf:** `extensions/slack/src/monitor/provider.ts:283`
  - body: Setting Bolt `ignoreSelf: false` in socket mode disables Bolt's self-event guard, but this hunk does not add the current OpenClaw self-event middleware. In `allowBots` configurations, bot-authored messages can be processed again, so this should be split or rebased onto the current `createSlackBoltApp` path.
  - confidence: 0.84
- **[P2] Rebase provider metadata validation onto current main:** `src/config/validation.ts:4`
  - body: This import targets an older bundled-capability metadata path and would bypass the current contract/test-only inventory boundary. The Google-provider validation fix should be a separate change based on the current plugin metadata snapshot surface.
  - confidence: 0.86
