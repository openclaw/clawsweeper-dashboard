Keep open: the branch is not safe to merge as-is because it changes the public TTS directive contract, bundles unrelated Slack/config changes, lacks real behavior proof, and conflicts with newer current-main work and maintainer review guidance.

## What This Changes

The PR replaces TTS directive regex parsing with a standalone-line scanner, updates TTS contract coverage, and also changes Slack Bolt socket-mode options plus config plugin validation.

## Best Possible Solution

Decide the TTS directive contract first; if standalone-only is desired, land a focused TTS-only PR with docs, compatibility tests, and changelog coverage, while rebasing or splitting Slack/config fixes separately.

Yes for the behavior delta by source inspection: current main docs/tests accept inline directives and compact blocks, while the PR scanner only activates full-line markers. No high-confidence proof shows current documented behavior is broken.

## Solution Assessment

No. Standalone-only semantics may be a valid product/API decision, but this branch is not the narrow maintainable vehicle because it changes public behavior without migration docs and bundles unrelated stale changes.

  - body: The new scanner only collects directive bodies when `[[tts:...]]` occupies a full trimmed line.   - confidence: 0.83
