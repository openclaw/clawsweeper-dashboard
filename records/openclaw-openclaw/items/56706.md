Full review comments:

- **[P1] Preserve supported inline TTS directives:** `src/tts/directives.ts:80-84`
  - body: Current main and the TTS docs still support inline provider directives and compact `[[tts:text]]...[[/tts:text]]` blocks, but this scanner only consumes directive wrappers when the whole trimmed line matches. Existing tagged-mode replies such as the current contract test input would stop triggering audio/provider overrides and leave directive text visible unless maintainers explicitly migrate the contract.
  - confidence: 0.92
- **[P2] Keep directive tokens scoped to the declared provider:** `src/tts/directives.ts:166-173`
  - body: Current main first resolves `provider=...` and then sends the rest of that directive only to the declared provider, warning on unsupported keys. This version loops over every provider for every token, so provider-specific keys can be consumed by the wrong provider or fail silently when a directive declares a provider.
  - confidence: 0.84
- **[P2] Keep Slack self-event filtering with ignoreSelf disabled:** `extensions/slack/src/monitor/provider.ts:283`
  - body: Current main disables Bolt `ignoreSelf` inside `createSlackBoltApp` and immediately installs OpenClaw's explicit self-event middleware. This stale provider-level hunk only flips the Bolt option at the old construction site, so rebasing it mechanically risks letting bot-authored events reach message handling in `allowBots` configurations or dropping the current self-filter path.
  - confidence: 0.78
