## Summary

Keep open: current main still supports documented inline and same-line TTS directive forms, while this PR changes that public contract to standalone-only and also bundles unrelated Slack/config changes. A maintainer already asked for product/API review, docs, and splitting before merge, and the external PR still lacks after-fix real behavior proof.

## What This Changes

This PR replaces TTS directive parsing with a standalone-line scanner and updates TTS contract tests, while also adding Slack Bolt `ignoreSelf: false` and bundled-provider config validation changes.

## Best Possible Solution

Decide the TTS directive contract first; if standalone-only is desired, land a focused parser, docs, tests, and changelog migration, with Slack and config-validation fixes split into separate rebased PRs.

## Reproduction Assessment

Yes for the behavior delta: source inspection shows current main accepts inline provider directives and same-line text blocks, while the PR scanner only activates full-line markers. No high-confidence proof shows current main is broken under the current documented contract.

## Solution Assessment

No. Standalone-only parsing may be a valid future contract, but this branch is not the best vehicle because it changes public semantics without the migration/docs surface and bundles unrelated stale changes.
