## What This Changes

The PR replaces regex-based TTS directive parsing with standalone-line parsing, updates TTS contract tests, and also changes Slack Bolt socket-mode options plus config plugin validation.

## Best Possible Solution

Decide the TTS directive contract first; if standalone-only is desired, land a focused TTS-only PR with docs, compatibility tests, and changelog coverage, while splitting or rebasing Slack/config fixes separately.

  - body: The new scanner only collects directive bodies when [[tts:...]] occupies a full trimmed line.   - confidence: 0.95

  - body: After this refactor, provider=... only sets overrides.provider; later tokens are still offered to every provider in registry order with no selected-provider context. That drops the current explicit-provider routing contract, so overlapping keys such as model or speed can be applied to the wrong provider or lose unsupported-key warnings.
  - confidence: 0.91
- **[P2] Keep Slack self filtering when disabling ignoreSelf:** `extensions/slack/src/monitor/provider.ts:283`
  - body: This hunk disables Bolt's self-event filter in the older direct App construction path without adding the current OpenClaw self-event middleware. Bot-authored messages can be processed when allowBots is enabled, so the Slack change should be split or rebased onto the current createSlackBoltApp path.
  - confidence: 0.83

  - body: The added validation import targets a bundled-capability metadata module path that no longer exists on current main, and it bypasses the current metadata-snapshot/control-plane boundary. Rebase this fix onto the current plugin metadata surface before merging.
