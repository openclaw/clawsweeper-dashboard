## Summary

Keep open: current main still documents and tests permissive inline/same-line TTS directives, and maintainer review says standalone-only parsing is a product/API decision while the unrelated Slack/config changes should be split.

## What This Changes

This PR replaces TTS directive parsing with a standalone-line scanner and tests, while also changing Slack Bolt socket-mode options and bundled provider plugin validation.

## Best Possible Solution

Decide the TTS directive contract first, then land a focused parser/docs/tests/changelog change; keep Slack and plugin-validation changes in separate PRs if still needed.

## Reproduction Assessment

Yes. Source, docs, and contract tests show current main accepts inline provider directives and same-line text blocks, while the PR scanner only activates standalone directive lines.

## Solution Assessment

No. Standalone-only parsing may be valid after a product/API decision, but this PR is not the best solution as-is because it changes the public contract without the full migration surface and bundles unrelated changes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve documented inline TTS directives:** `src/tts/directives.ts:70-78`
  - body: The scanner only recognizes [[tts:...]] and [[tts:text]] when they occupy full trimmed lines, but current docs and contract tests support inline provider directives and same-line hidden text blocks. Tagged-mode inputs like [[tts:text]]Hello world[[/tts:text]] would stop triggering synthesis, and inline overrides would remain visible unless the public contract, docs, tests, and changelog are migrated in the same focused change.
  - confidence: 0.94
- **[P2] Guard self messages when disabling Bolt ignoreSelf:** `extensions/slack/src/monitor/provider.ts:281`
  - body: This disables Bolt's self filtering without adding an equivalent guard in the same Slack path. On the PR's base, bot_message events without a user can pass when allowBots is enabled and cause self-processing; current main solves this through createSlackBoltApp with explicit self filtering, so this hunk should be split or rebased rather than merged as-is.
  - confidence: 0.82
