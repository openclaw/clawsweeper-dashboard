## Summary

Keep open: current main still documents and tests permissive inline/compact TTS directive forms, while this PR changes that public contract to standalone-only and also carries unrelated Slack and config-validation changes. The discussion includes a maintainer review saying the branch should not merge as-is and needs product/API direction plus splitting.

## What This Changes

The PR rewrites TTS directive parsing to require standalone directive lines, updates TTS contract tests, and also changes Slack socket-mode App options plus plugin config validation.

## Best Possible Solution

Decide the TTS directive contract first; if standalone-only is desired, land a focused TTS-only PR with docs, compatibility tests, and changelog coverage, while splitting Slack and config-validation fixes into separate rebased PRs.

Full review comments:

- **[P2] Preserve supported inline TTS directives:** `src/tts/directives.ts:72-84`
  - body: The new scanner only collects directive bodies when `[[tts:...]]` occupies a full trimmed line. Current docs and contract tests still support inline provider overrides and compact `[[tts:text]]` blocks, so tagged-mode replies using those forms would stop triggering TTS or applying overrides unless the contract is deliberately migrated.
  - confidence: 0.95
- **[P2] Scope explicit-provider tokens to the declared provider:** `src/tts/directives.ts:166-173`
  - body: After this refactor, `provider=...` only sets `overrides.provider`; later tokens are still offered to every provider in registry order with no selected-provider context. That drops the current explicit-provider routing contract, so overlapping keys such as `model` or `speed` can be applied to the wrong provider or lose unsupported-key warnings.
  - confidence: 0.91
- **[P2] Keep Slack self filtering when disabling ignoreSelf:** `extensions/slack/src/monitor/provider.ts:283`
  - body: This hunk disables Bolt's self-event filter in the older direct App construction path without adding the current OpenClaw self-event middleware. Bot-authored messages can be processed when `allowBots` is enabled, so the Slack change should be split or rebased onto the current `createSlackBoltApp` path.
  - confidence: 0.83
- **[P2] Rebase provider metadata validation onto current main:** `src/config/validation.ts:4`
  - body: The added validation import targets a bundled-capability metadata module path that no longer matches current main, and it bypasses the current metadata-snapshot/control-plane boundary. Rebase this fix onto the current plugin metadata surface before merging.
  - confidence: 0.86
