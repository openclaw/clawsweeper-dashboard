## Summary

Keep open: current main and the latest release still generate a reload handoff that bootouts the LaunchAgent and immediately bootstraps it with no fallback, so the reported deregistration/offline failure remains plausible and narrowly actionable.

## What This Changes

The issue reports that macOS gateway restart reload mode can deregister the LaunchAgent after a bootout to bootstrap race, leaving the gateway down until manual bootstrap.

## Best Possible Solution

Patch the macOS launchd reload handoff so plist rewrites reload without leaving the service unregistered, with focused regression tests and macOS launchd proof.

## Reproduction Assessment

No live macOS reproduction was run in this review. Source inspection does reproduce the risky path: current main emits a reload script that bootouts and immediately bootstraps with no recovery, and the test suite asserts there is no kickstart fallback.

## Solution Assessment

Yes: the right fix boundary is the macOS launchd reload handoff, not a new config option or plugin surface. The implementation should preserve plist reload semantics while adding unload confirmation or recovery for bootstrap failures.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/84630. Current main's detached launchd restart handoff `mode: "reload"` in `src/daemon/launchd-restart-handoff.ts` calls enable, bootout, then immediate bootstrap with no fallback; `restartLaunchAgent` schedules this mode when a plist rewrite is needed inside the current LaunchAgent. Implement a resilient reload path without changing plugin/provider/channel behavior: preserve plist reload semantics, wait or probe for bootout completion before bootstrap or otherwise recover from already-registered/EIO bootstrap failures, and avoid leaving the service deregistered. Do not replace reload with kickstart-only unless the implementation proves plist changes are applied. Add focused tests in `src/daemon/launchd-restart-handoff.test.ts` and, if needed, `src/daemon/launchd.test.ts`. Coordinate with open PR https://github.com/openclaw/openclaw/pull/81728, which handles `bootstrapLaunchAgentOrThrow` but not the detached shell reload path. Add a concise CHANGELOG.md fix entry if the PR is prepared for landing.
