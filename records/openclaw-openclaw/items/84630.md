## Best Possible Solution

Review and land the linked launchd reload handoff fix once live macOS proof shows plist-rewrite reload leaves the LaunchAgent registered and running.

## Reproduction Assessment

No live reproduction was run here. Source inspection gives a high-value reproduction path: current main emits a reload script that bootouts and immediately bootstraps with no fallback, matching the reporter's log and failure mode.

## Solution Assessment

Yes, the best fix boundary is the macOS launchd reload handoff. The fix should add unload confirmation and/or a fallback in that helper with focused tests and live launchd proof, not add a new user-facing mode or config option.
