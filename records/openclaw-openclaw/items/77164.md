## What This Changes

The issue requests Telegram support or explicit documentation for `sessions_spawn({ runtime: "subagent", mode: "session", thread: true })` when invoked from Telegram conversations.

## Best Possible Solution

Keep #42986 as the canonical tracker and review, land, revise, or replace #42707 with a Telegram plugin-boundary fix that covers DMs, DM topics, forum topics, delivery targets, and cleanup.

## Reproduction Assessment

Yes. Source inspection on current main shows Telegram `thread:true` subagent spawns reach the core `subagent_spawning` hook gate, while the Telegram entry does not register that hook even though Telegram binding infrastructure exists.

Yes. The best path is not another parallel issue; it is to preserve this direct-chat evidence on the canonical Telegram thread-binding tracker and complete the Telegram plugin-boundary hook implementation in #42707 or a narrow equivalent.
