## Summary

Keep open. Current main still has no docs-focused bug intake path, the general bug form still requires runtime/model/provider fields, and an open linked PR is already the implementation candidate for this issue.

## What This Changes

Add a docs-specific bug report issue form or equivalent split so documentation defects do not require runtime-only bug-report fields.

## Best Possible Solution

Keep the general runtime bug form strict and add a small docs bug template with docs path, expected/actual content, impact, evidence, and bug/docs routing if maintainers accept that triage shape.

## Reproduction Assessment

## Solution Assessment

Yes: a dedicated docs bug form is the narrowest maintainable direction because it avoids weakening runtime bug reports while giving docs defects correct bug/docs intake. The linked PR follows that direction but is still open and unmerged.
