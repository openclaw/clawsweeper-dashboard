## Summary

Keep open. Current main still lacks a docs-focused bug intake path, and the linked implementation PR is open and unmerged, so this issue should remain open until that PR lands or maintainers choose a different intake policy.

## What This Changes

Request to add a documentation-only bug intake path so docs defects do not require runtime, model, provider, OS, or install placeholders.

## Best Possible Solution

Adopt a dedicated docs bug report template, or an equivalent conditional issue-form path, that labels documentation defects correctly while preserving strict runtime fields for runtime bug reports.

## Reproduction Assessment

Not applicable: this is an issue-template/process request, not a runtime bug. Source inspection confirms current main has no docs-focused form and the generic bug form still requires runtime/model/provider fields.

## Solution Assessment

Yes: a separate docs bug form is the narrowest maintainable approach because it avoids weakening runtime bug reports while giving docs defects a correctly labeled intake path; PR #76668 is already attempting that path.
