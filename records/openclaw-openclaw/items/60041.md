## Summary

Keep open. Current main still matches the reported Twitch failure modes: the bundled Twitch config schema still generates an `allOf`/`additionalProperties: false` JSON Schema shape that can reject valid account fields, and the Twitch gateway start path can still resolve immediately after monitor setup, which the shared channel manager treats as a clean exit and restarts. The issue is also paired with open PR #74773, so it should remain open until that implementation path is reviewed and completed.

## What This Changes

Fix Twitch channel configuration, validation, onboarding diagnostics, and gateway lifecycle so a valid Twitch account connects, stays connected, and reports actionable status.

## Best Possible Solution

Land a Twitch-owned fix that flattens the channel config schema used by generated validation metadata and keeps the Twitch monitor task alive until abort/stop, with focused regression tests for both `openclaw config validate` accepting valid single-account and multi-account configs and the gateway lifecycle not treating a successful Twitch connection as a clean exit.

## Reproduction Assessment

Yes. There is a high-confidence source-level reproduction: valid Twitch account fields flow through a generated schema with conflicting `allOf` branches, and a successful monitor setup can resolve `startAccount`, causing the shared channel manager to record a clean exit and auto-restart.

## Solution Assessment

No, not fully yet. PR #74773 is a plausible narrow fix for the schema validation half, but the best maintainable solution also needs the Twitch gateway lifecycle to follow the passive-monitor pattern and stay pending until abort.
