## Summary

Keep open. Current main has addressed the Twitch schema, docs, and setup-surface parts, but the reported connection loop still matches current source: Twitch `startAccount` awaits a monitor that returns immediately, and the shared gateway manager restarts any channel task that resolves without an explicit stop.

## What This Changes

This issue reports that Twitch authenticates after upgrade but never stabilizes, with schema friction and an immediate gateway auto-restart loop.

## Best Possible Solution

Keep the flat schema/docs/setup work and land a narrow Twitch-owned lifecycle fix that keeps the monitor task pending until abort or explicit stop.

## Reproduction Assessment

Yes, source-level. A successful Twitch monitor setup returns immediately, and the shared gateway manager treats the resolved channel task as a clean exit that schedules restart attempts.

## Solution Assessment

No, the current state is only partial. The remaining fix should be in Twitch `startAccount` or `monitorTwitchProvider` lifecycle handling, not by weakening shared gateway clean-exit restart behavior.
