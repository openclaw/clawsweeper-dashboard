## Summary

Keep open. Current main has already flattened the Twitch config schema and improved docs/setup surfaces, but the reported connect-then-auto-restart loop still matches the Twitch lifecycle code and shared channel manager behavior; the related open PR covers only the schema half.

## What This Changes

The issue asks for Twitch config validation/setup diagnostics and a stable gateway connection for valid Twitch credentials instead of an immediate auto-restart loop.

## Best Possible Solution

Resolve the schema-only PR relationship, then land a Twitch-owned lifecycle fix that keeps the monitor task pending until abort or explicit stop while preserving the flat schema/docs already on main.
