## Summary

Keep open. Current main and the latest release have addressed the Twitch schema/docs/setup portions, but the reported connection loop is still source-reproducible: Twitch monitor setup returns after registering handlers, and the shared channel manager restarts any channel task that resolves without an explicit stop. The linked schema-only PR also remains open with closing syntax, so maintainers should dispose of or expand that PR before separate cleanup.

## What This Changes

The issue reports that a configured Twitch channel authenticates after an update but never reaches a stable connected state, while also requesting clearer current Twitch schema, setup, validation, and diagnostics.

## Best Possible Solution

Keep the flat schema/docs/setup work and land a narrow Twitch-owned lifecycle fix that keeps the monitor task pending until abort or explicit stop.

## Reproduction Assessment

Yes, source-level. A successful Twitch monitor setup returns immediately, and the shared channel manager has source and tests showing resolved channel tasks are treated as clean exits and auto-restarted.

## Solution Assessment

No, the current state is only a partial fix. The narrow maintainable solution is in Twitch `startAccount` or `monitorTwitchProvider` lifecycle handling, not weakening shared clean-exit restart behavior.
