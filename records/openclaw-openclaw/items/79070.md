## Best Possible Solution

## Reproduction Assessment

Yes for the review finding by source inspection: current main uses the Control UI-local JSON sender with `Cache-Control: no-cache`, while the PR diff switches that path to the common helper that does not set it. This is not a runtime failure reproduction.

## Solution Assessment

No, not yet. The shared helper is a reasonable cleanup for generic gateway handlers, but this Control UI call site needs to preserve the existing no-cache header or use a cache-aware helper.
