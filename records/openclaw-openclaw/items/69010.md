## Summary

Keep open. Current main still has the stale-session delivery precedence behavior this PR is trying to fix, but the PR head has a concrete same-channel mixed-route bug: a request that changes `to` without `accountId` can persist the new target with the old account ID. The Aisle security analysis and the latest ClawSweeper review independently identify the same route-mixing risk, so the PR needs revision before merge.

## What This Changes

The PR changes gateway agent delivery-context precedence, forwards the resolved delivery target into `runContext.currentChannelId`, adds gateway agent regression tests, and adds a changelog fix entry.

## Best Possible Solution

Revise the same-channel replacement path so route fields are updated atomically: when `to` changes, clear `accountId` and `threadId` unless the request supplies them or a channel resolver validates and derives them. Add a regression for changing `to` without `accountId`, then rebase and rerun the targeted gateway tests plus the changed gate.

## Reproduction Assessment

Yes. The original bug has a high-confidence reproduction shape from #68378 and the PR’s stale Feishu-to-Discord regression test; the remaining review finding is reproducible with a same-channel `deliver: true` request that changes `to` while omitting `accountId`.

## Solution Assessment

No. The PR targets the right gateway layer and fixes part of the stale-route problem, but the current implementation is not the best fix until target/account/thread replacement is atomic for same-channel target changes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Do not carry stale accountId onto a new target:** `src/gateway/server-methods/agent.ts:946`
  - body: When `requestTargetDiffers` is true, this branch persists `requestDeliveryHint.to` but falls back to `sessionDelivery.accountId`. A `deliver: true` request that changes `to` on the same channel without `accountId` leaves the session with a mixed `{to, accountId}` route, so later implicit delivery can use credentials from the previous target. Clear coupled fields or resolve them atomically when the target changes.
  - confidence: 0.88
