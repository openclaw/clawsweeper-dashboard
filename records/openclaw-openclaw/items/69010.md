## Summary

Keep open: this PR addresses a real gateway route-precedence bug that is not implemented on current main, but the latest head still has a P1 same-channel route safety defect and lacks after-fix real behavior proof.

## What This Changes

This PR changes gateway agent route merging to prefer concrete current request delivery context over stale session delivery fields, forwards the resolved target into run context, adds gateway regression tests, and adds a changelog entry.

## Best Possible Solution

Amend this PR or replace it with a narrow gateway fix that updates coupled same-channel route fields atomically: when to changes, clear accountId and threadId unless the request supplies them or a channel resolver validates the pair, then add the omitted-account regression and redacted real gateway/channel proof.

## Reproduction Assessment

Yes by source inspection of PR head: a same-channel deliver:true request can change to while omitting accountId, and line 946 falls back to sessionDelivery.accountId. I did not run the live gateway/channel flow in this read-only review.

## Solution Assessment

No. The PR targets the right gateway layer and covers part of the stale-route bug, but target replacement should update coupled route fields atomically or validate the pair before persisting.

## Review Findings

Overall correctness: patch is incorrect
