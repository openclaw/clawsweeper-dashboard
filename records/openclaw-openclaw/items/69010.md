Overall correctness: patch is incorrect

Full review comments:

- **[P1] Clear stale account metadata when replacing targets:** `src/gateway/server-methods/agent.ts:946`
  - body: When `requestTargetDiffers` is true, this builds the new delivery context with the request's fresh `to` but falls back to `sessionDelivery.accountId`. A request that retargets `discord` from `channel:old` to `channel:new` without an `accountId` will persist `channel:new` with the old account, so a later implicit delivery can send through the wrong configured account. Treat the target change as an atomic route replacement and clear `accountId`/`threadId` unless the request supplies replacements.
