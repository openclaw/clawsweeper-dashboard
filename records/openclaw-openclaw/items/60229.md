Overall confidence: 0.9

Full review comments:

- **[P1] Await coalesced busy flushes before returning:** `src/auto-reply/inbound-debounce.ts:207-213`
  - body: This schedules the coalesced busy flush and then returns from `enqueue()` before `onFlush` runs. Telegram awaits this enqueue while handling an update, so a restart after the handler returns but before the queued flush executes can leave that follow-up message acknowledged without being delivered.
  - confidence: 0.91
- **[P2] Preserve chat/topic ordering for text fragments:** `extensions/telegram/src/sequential-key.ts:65`
  - body: Adding `:msg:${messageId}` lets ordinary same-chat/topic updates run concurrently through the Telegram middleware. The handler keeps shared text-fragment state by chat/topic, so split long-paste fragments can interleave or miss the buffer and reach the agent as separate or out-of-order turns.
  - confidence: 0.86
