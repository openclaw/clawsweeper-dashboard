## Summary

Keep open: the PR is a useful candidate for the open Windows restart bug, and current main/latest release do not already implement the schtasks probe, but the patch is not merge-ready because it can probe a different Scheduled Task than the handoff will run and still lacks redacted real Windows proof.

## Best Possible Solution

## Solution Assessment

No/unclear: probing `schtasks` is a safer direction than detached respawn, but this implementation probes only the default task while the restart handoff can use profile-specific or custom task names.
