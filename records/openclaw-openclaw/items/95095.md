## Summary

Keep open: current main and the latest release do not solve the linked Windows env-less Scheduled Task restart issue, and this PR is still not merge-ready because it probes only the default task while the restart handoff supports profile and custom task names, with no real Windows after-fix proof.

## What This Changes

The PR adds a Windows `schtasks /Query` fallback in supervisor detection when environment markers are absent.

## Best Possible Solution

Resolve and query the same Windows task name the restart handoff will run, cover default/profile/custom cases, and require real Windows proof before merge.

## Reproduction Assessment

Yes, source-level: current main returns null for env-less win32 supervisor detection, and the linked issue describes a manually started gateway with an installed Scheduled Task but missing service env. I did not run live Windows proof in this read-only review.

## Solution Assessment

No/unclear: probing `schtasks` is a plausible safer direction, but this implementation probes only the default task while the restart path can use profile or custom names. The safer solution is to query the exact task name the handoff will run and prove it on Windows.
