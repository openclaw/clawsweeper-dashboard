## Summary

Keep open: this is a useful candidate for the linked Windows restart bug, but it is not merge-ready because the fallback probes only the default Scheduled Task while the restart handoff supports profile/custom task names, and real Windows behavior proof is still missing.

## What This Changes

## Best Possible Solution

## Solution Assessment

No/unclear: probing `schtasks` is a safer direction than detached respawn, but this implementation probes only the default task while the restart handoff can use profile-specific or custom task names.
