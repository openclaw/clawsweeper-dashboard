## Summary

Keep open: this PR is a plausible candidate for the linked Windows restart bug, but it is not merge-ready because the new fallback probes only the default Scheduled Task while the actual handoff can target profile-specific or custom task names, and the external real-behavior proof is still absent.

## What This Changes

The PR adds a Windows `schtasks /Query` fallback to supervisor detection, expands mocked supervisor-marker tests, and adds one bounded-response safe-integer test.

## Best Possible Solution

Resolve and probe the same Windows task name the restart handoff will run, cover default/profile/override cases, split unrelated test churn, and require redacted Windows `/restart` plus `/status` proof before merge.

## Reproduction Assessment

Yes, at source level: current main returns no `schtasks` supervisor for env-less Windows and then keeps ordinary restart on the in-process fallback. I did not run live Windows proof in this read-only review.

## Solution Assessment

No/unclear: probing `schtasks` is safer than detached respawn, but this implementation probes only the default task while the handoff supports profile-specific and custom task names.
