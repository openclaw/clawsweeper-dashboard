- `merge-risk: 🚨 compatibility`: The PR changes a shipped Windows fallback and can mis-handle profile/custom Scheduled Task names that current restart helpers support.

## Summary

Keep open: the schtasks-probe direction is a plausible safer replacement for detached respawn, but this PR is not merge-ready because it probes only the default task name, lacks real Windows proof, and changes an upgrade-sensitive restart fallback.

## What This Changes

The PR adds a Windows `schtasks /Query` fallback in `src/infra/supervisor-markers.ts` so env-less gateway processes can be treated as Scheduled Task supervised.

## Best Possible Solution

Use the same profile-aware task resolution for probing and restart handoff, preserve the safe fallback when that exact task is absent, add focused tests, and require real Windows proof before merge.

## Reproduction Assessment

Yes, source-level: current main returns disabled on win32 without Scheduled Task markers, while the related issue comment identifies the env-less manually started gateway with an installed task. I did not run live Windows proof in this read-only review.

## Solution Assessment

No/unclear: probing schtasks is a plausible safer direction, but this implementation probes only the default task and has no after-fix Windows proof. The safer solution is a profile-aware exact-task probe plus regression tests and live proof.
