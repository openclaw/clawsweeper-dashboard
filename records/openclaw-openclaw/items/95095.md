## Summary

Keep open: current main and the latest release still lack the env-less Windows Scheduled Task probe, but this PR is not merge-ready because it lacks real Windows proof and the fallback probes only the default task while the handoff supports profile/custom task names.

## What This Changes

The PR adds a Windows `schtasks /Query` fallback to `detectRespawnSupervisor()`, expands supervisor-marker tests, and adds a bounded-response safe-integer test.

## Best Possible Solution

Resolve and probe the same Windows task name the restart handoff will run, keep the supervisor fix focused, and require redacted Windows `/restart` plus `/status` proof before merge.

## Reproduction Assessment

Yes, at source level: current main returns no `schtasks` supervisor for env-less Windows and then keeps ordinary restart on the in-process fallback. I did not run live Windows proof in this read-only review.

## Solution Assessment

No/unclear: probing `schtasks` is a safer direction than detached respawn, but this implementation probes only the default task while the restart handoff can use profile-specific or custom task names.
