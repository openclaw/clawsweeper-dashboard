## Summary

Keep open: current main still has a source-proven overwrite path, and the linked open fixing PR should be reviewed before this issue is closed.

## What This Changes

The issue asks OpenClaw to stop stale session usage persistence from clearing post-compaction total token state.

## Best Possible Solution

Fix persistSessionUsageUpdate so stale usage updates preserve both totalTokens and totalTokensFresh when no fresh context snapshot exists, and add regression coverage for the post-compaction case.

## Reproduction Assessment

Yes, this is source-reproducible: run preflight compaction so totalTokensFresh is true, then call persistSessionUsageUpdate with nonzero usage but no lastCallUsage, promptTokens, or usageIsContextSnapshot. Current main computes totalTokens as undefined and writes totalTokensFresh=false.

## Solution Assessment

The issue’s proposed direction is correct, but the linked PR still needs review because the fetched diff preserves totalTokens while continuing to write totalTokensFresh=false in the stale branch. The safer fix is to leave both fields untouched unless a fresh context snapshot is available.
