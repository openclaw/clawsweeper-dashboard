## Summary

Keep open. Current main still lacks a `/stream` or `/streaming` command, a loaded command handler, persisted session/conversation override state, and a shared resolver path that can layer a local override ahead of channel config; the related items are adjacent streaming and command work, not a canonical implementation.

## What This Changes

Request a slash/native command to show, set, and reset preview streaming mode for the current chat/session without editing gateway config or restarting.

## Best Possible Solution

Add a shared preview-streaming override contract/resolver that supports show/set/reset through `/stream`, surfaces active overrides in `/status`, and lets capable channels opt in while unsupported channels return a clear unsupported response.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The absence is high-confidence by source inspection of the command registry, loaded handlers, session state, shared streaming resolver, docs, and Telegram/Slack/Discord preview paths.

## Solution Assessment

Unclear until maintainers choose the override scope, persistence, and channel/plugin contract. The maintainable direction is a shared resolver plus channel opt-in, not direct gateway config edits or a Telegram-only toggle.
