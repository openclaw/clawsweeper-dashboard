## Summary

Keep open. Current main still lacks a `/stream` or `/streaming` command, a session/conversation preview-streaming override, and shared channel plumbing for Telegram/Slack/Discord to layer that override over configured defaults.

## What This Changes

The issue requests a slash/native command to show, set, and reset preview-streaming mode for the current chat/session without editing gateway config or restarting.

## Best Possible Solution

Define a shared preview-streaming override contract with channel opt-in, expose show/set/reset through `/stream`, and surface active overrides in `/status` with clear unsupported-channel replies.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The absence is high-confidence from current-main inspection of command registration, handler loading, session state, docs, and Telegram/Slack/Discord preview-mode consumers.

## Solution Assessment

Unclear until maintainers choose the persistence and shared-channel API shape. The maintainable direction is a channel opt-in override resolver, not a command that rewrites gateway config or a Telegram-only toggle presented as shared behavior.
