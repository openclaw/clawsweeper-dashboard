## Summary

Keep open. Current main still does not provide a `/stream` or `/streaming` chat command, a loaded command handler, or a session/conversation-level preview-streaming override; Telegram, Slack, and Discord still resolve preview behavior from channel config rather than a local runtime override.

## What This Changes

Add a shared `/stream` or `/streaming` command to show, set, and reset the current conversation's preview-streaming mode without editing gateway config or restarting the gateway.

## Best Possible Solution

Add a shared preview-streaming override contract/resolver with channel opt-in, expose show/set/reset through `/stream` plus `/status`, and return a clear unsupported-channel response where preview overrides do not apply.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The absence is high-confidence from source inspection of the command registry, loaded handlers, session state, shared resolver, docs, and Telegram/Slack/Discord preview paths.

## Solution Assessment

Unclear until maintainers choose session-vs-conversation persistence and the channel/plugin contract shape. The maintainable direction is a shared override resolver with opt-in channel support, not a Telegram-only toggle or a command that rewrites gateway config.
