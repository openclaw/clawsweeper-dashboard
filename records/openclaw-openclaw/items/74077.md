## Summary

Keep open. Current main still controls preview streaming through channel config and channel-specific draft dispatch paths, with no built-in `/stream` or `/streaming` command, no session/conversation override field, and no shared override resolver consumed by Telegram, Slack, or Discord. The prior ClawSweeper comment and the related items are useful context, but they are adjacent command-alias and channel-streaming work rather than a canonical implementation of this request.

## What This Changes

Request a slash/native command to inspect, set, and reset preview streaming mode for the current chat/session across preview-capable channels without editing gateway config or restarting.

## Best Possible Solution

Add a shared preview-streaming override contract that resolves before channel config defaults, expose it through a built-in status/set/reset command, show active overrides in `/status`, document the command, and let preview-capable channel plugins opt in while unsupported surfaces return a clear unsupported response.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The absence is high-confidence by source inspection: current main has no `/stream` registration or handler, no session/protocol override field, and the inspected preview paths resolve from channel config inputs.

## Solution Assessment

Unclear until maintainers choose the override scope, persistence model, and public channel/plugin contract. The maintainable direction is shared override resolution plus channel opt-in, rather than a Telegram-only command or a command that cannot affect the existing dispatch paths.
