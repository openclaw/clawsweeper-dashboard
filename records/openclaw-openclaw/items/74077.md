## Summary

Keep open. Current main still lacks the requested `/stream` or `/streaming` command, a loaded command handler, session/conversation storage for a preview-streaming override, and channel resolver inputs that would layer a local override ahead of config; the related items are adjacent streaming, alias, or Telegram-specific work rather than a canonical implementation.

## What This Changes

Add a slash/native command to inspect, set, and reset preview streaming mode for the current chat/session without editing gateway config or restarting the gateway.

## Best Possible Solution

Add a shared preview-streaming override contract and resolver with channel opt-in, expose show/set/reset through `/stream` plus `/status`, and return a clear unsupported response when a channel cannot apply preview overrides.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The absence is high-confidence from source and docs inspection of command registration, handler loading, session state, shared streaming resolution, and Telegram/Slack/Discord preview consumers.

## Solution Assessment

Unclear until maintainers choose persistence scope and the channel/plugin contract. The maintainable direction is a shared override resolver with channel opt-in, not a Telegram-only toggle or a command that rewrites gateway config.
