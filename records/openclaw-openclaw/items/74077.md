## Summary

Keep open. Current main and v2026.6.6 still expose preview streaming through static channel config and related channel implementations; there is no shared `/stream` or `/streaming` command, no current-session streaming-mode override, and the related `/progress` issue is narrower and still open rather than a replacement.

## What This Changes

Request a slash/native command to show, set, and reset preview streaming mode for the current chat or session without editing gateway config or restarting the gateway.

## Best Possible Solution

Define the shared override scope and channel capability contract first, then implement one command path that keeps channel config as the fallback and returns a clear unsupported response where a channel cannot honor the mode.

## Reproduction Assessment

Not applicable; this is a feature request for a command that does not exist today. Source inspection confirms current main lacks both the command and a session-scoped streaming-mode override.

## Solution Assessment

Unclear until maintainers decide the command scope and storage model. A session-style command is plausible, but the maintainable solution needs a shared channel capability contract rather than a one-channel shortcut.
