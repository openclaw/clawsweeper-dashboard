## Summary

Keep open. Current main still exposes preview streaming through channel configuration, with no `/stream` or `/streaming` command, no loaded handler, and no session or conversation override feeding Telegram, Slack, or Discord delivery.

## What This Changes

Add a slash/native command such as `/stream` to show, set, or reset current chat/session preview streaming behavior without editing gateway config or restarting.

## Best Possible Solution

Add a shared preview-streaming override contract/resolver, persist the chosen current-scope override, expose show/set/reset through `/stream`, surface active overrides in `/status`, and let unsupported channels return an explicit unsupported response.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The absence is high-confidence from source inspection of the command registry, loaded handlers, session state, shared resolver, docs, and Telegram/Slack/Discord delivery paths.

## Solution Assessment

Unclear until maintainers choose the override scope and channel/plugin contract shape. The maintainable direction is a shared resolver plus channel opt-in, not a Telegram-only command or a command that rewrites gateway config.
