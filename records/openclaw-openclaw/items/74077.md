## Summary

Keep open. Current main still has config-driven preview streaming for Telegram, Slack, and Discord, but no `/stream` or `/streaming` built-in command, no loaded handler, no session/conversation streaming override state, and no resolver input that can layer a local override ahead of channel config.

## What This Changes

Request to add a slash/native command that shows, sets, and resets preview-streaming mode for the current chat or session without editing gateway config or restarting.

## Best Possible Solution

Add a shared preview-streaming override contract/resolver with channel opt-in, expose show/set/reset through `/stream` plus `/status`, and keep config as the default fallback.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The absence is high-confidence from inspecting the command registry, loaded handlers, session state, shared resolver, docs, and Telegram/Slack/Discord preview paths.

## Solution Assessment

Unclear until maintainers choose the persistence scope and plugin/channel contract shape. The maintainable direction is a shared override resolver with opt-in channel support, not a Telegram-only toggle or a command that rewrites gateway config.
