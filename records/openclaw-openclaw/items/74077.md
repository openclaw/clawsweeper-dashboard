## Summary

Keep open. Current main still lacks the requested `/stream` or `/streaming` command, has no loaded handler or persisted session/conversation preview-streaming override, and the inspected Telegram, Slack, and Discord delivery paths still derive preview mode from channel config rather than a local runtime override; the linked items are adjacent streaming/alias work, not a canonical implementation.

## What This Changes

The issue requests a shared slash command to inspect, set, and reset preview streaming mode for the active chat/session without editing gateway config or restarting.

## Best Possible Solution

Add a shared preview-streaming override contract/resolver with channel opt-in, expose show/set/reset through `/stream` plus `/status`, and return a clear unsupported response on channels that cannot apply preview overrides.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The absence is high-confidence from source inspection of the command registry, loaded handlers, session state, shared resolver, docs, and Telegram/Slack/Discord preview paths.

## Solution Assessment

Unclear until maintainers choose session-vs-conversation persistence and the channel/plugin contract shape. The maintainable direction is a shared override resolver with opt-in channel support, not a Telegram-only toggle or a command that rewrites gateway config.
