## Summary

Keep open. Current main still treats preview streaming as channel configuration plus channel-specific dispatch behavior, with no built-in `/stream` or `/streaming` command, no persisted per-session/conversation override, and no shared override resolver consumed by Telegram, Slack, or Discord. The request is concrete and useful, but it needs maintainer design on scope, persistence, and the channel/plugin contract before a safe narrow fix can be queued.

## What This Changes

Add a slash/native command such as `/stream` or `/streaming` to show, set, and reset preview streaming mode for the current chat/session without editing gateway config or restarting the gateway.

## Best Possible Solution

Add a shared preview-streaming override resolver that runs before channel/account config defaults, expose it through a built-in status/set/reset command, show active overrides in `/status`, and let preview-capable channels opt in while unsupported channels return a clear unsupported response.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The absence is high-confidence by source inspection: current main has no `/stream` registration or handler, no session override field, and the inspected preview paths resolve from config-only inputs.

## Solution Assessment

Unclear until maintainers choose the override scope and persistence model. The maintainable direction appears to be a shared channel override contract plus channel opt-in, rather than a Telegram-only command or a command that cannot affect the existing dispatch paths.
