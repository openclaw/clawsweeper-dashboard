## Summary

Keep open. Current main still lacks a `/stream` or `/streaming` command, a session/conversation preview-streaming override, and channel consumers that layer such an override over config; adjacent PRs cover narrower Telegram config/UI or `/progress` ideas but do not supersede this shared command request.

## What This Changes

Request a slash/native command such as `/stream` to show, set, or reset preview streaming mode for the current chat/session without gateway config edits or restart.

## Best Possible Solution

Define a shared preview-streaming override contract with channel opt-in, then expose show/set/reset through `/stream` and surface the effective override in `/status` while returning explicit unsupported-channel replies.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The absence is high-confidence from current-main source inspection of command registration, handler loading, session state, resolver inputs, docs, and Telegram/Slack/Discord preview-mode consumers.

## Solution Assessment

Unclear until maintainers choose persistence scope and shared-vs-channel-specific API shape. The maintainable direction is a channel opt-in override resolver, not a command that rewrites gateway config or a Telegram-only toggle presented as shared behavior.
