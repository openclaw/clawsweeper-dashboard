## Summary

Keep open. Current main still has no `/stream` or `/streaming` command, no command handler, no session/conversation override field, and the preview streaming paths for Telegram, Slack, and Discord still resolve from channel config plus channel-local dispatch state. The linked reports are adjacent streaming/config/observability work, not a canonical implementation of this per-chat/session command, so this needs maintainer design rather than cleanup closure.

## What This Changes

Request to add a slash/native command that shows, sets, and resets the current chat/session preview streaming mode without editing gateway config or restarting the gateway.

## Best Possible Solution

Add a shared preview-streaming override resolver that runs before channel/account config defaults, persist the selected scope intentionally, expose it through a built-in status/set/reset command, show active overrides in `/status`, and let preview-capable channels opt in while unsupported channels return a clear unsupported response.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The absence is high-confidence by source inspection: current main has no stream command registration or handler, no session override field, and inspected preview paths resolve from config-only inputs.

## Solution Assessment

Unclear until maintainers choose the override scope and persistence model. The maintainable direction appears to be a shared channel override contract plus channel opt-in, rather than a Telegram-only command or a command that edits gateway config.
