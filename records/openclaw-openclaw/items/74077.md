## Summary

Keep open. Current main still has no `/stream` or `/streaming` command and no session-scoped preview streaming override; related `/progress` work only covers tool-progress visibility and was closed unmerged, so the broader cross-channel request remains valid but needs maintainer design direction.

## What This Changes

Add a shared slash/native command to set or reset the current chat/session preview streaming mode without editing channel config or restarting the Gateway.

## Best Possible Solution

Define one shared session/conversation streaming override contract that supported channel preview-streaming implementations can honor consistently, then add command, status, docs, and tests around that contract.

## Reproduction Assessment

Yes for feature absence: source inspection shows no `/stream` or `/streaming` command in the registry and no session-entry preview streaming override field, so this is not a broken existing command.

## Solution Assessment

Unclear until maintainers choose storage scope; a command is likely the right UX, but the implementation should avoid a Telegram-only path and either align all exposed preview-streaming channels or clearly report unsupported channels.
