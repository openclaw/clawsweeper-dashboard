## Summary

Keep open. Current main still has no `/stream` or `/streaming` command, no loaded command handler for preview streaming, no session/conversation override field, and the Telegram/Slack/Discord preview paths still resolve effective streaming behavior from channel configuration plus channel-local runtime state. The adjacent linked reports cover aliases, config reloads, Telegram preview bugs, and streaming hooks, but none is a canonical implementation of this cross-channel per-chat/session control.

## What This Changes

Request a slash/native command to inspect, set, and reset preview streaming mode for the current chat/session without editing gateway config or restarting, preferably through a shared channel capability for Telegram, Slack, Discord, and similar channels.

## Best Possible Solution

Add a shared preview-streaming override resolver that runs before channel/account config defaults, persist the chosen scope intentionally, expose status/set/reset through a built-in command, show active overrides in `/status`, and let preview-capable channels opt in while unsupported channels return a clear unsupported response.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The absence is high-confidence by source inspection: current main has no command registration, handler, session override field, or resolver input for a per-chat/session preview-streaming override.

## Solution Assessment

Unclear until maintainers choose the override scope and persistence model. The maintainable direction appears to be a shared channel override contract plus channel opt-in, rather than editing gateway config from a command or adding a Telegram-only toggle.
