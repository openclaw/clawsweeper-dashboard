## Summary

Keep open. Current main still lacks a `/stream` or `/streaming` command, no session or conversation preview-streaming override is persisted or resolved, and the inspected Telegram, Slack, and Discord delivery paths still derive preview behavior from channel config; the related items are adjacent streaming, alias, and observability work rather than a canonical implementation.

## What This Changes

Add a slash or native command to show, set, and reset preview streaming mode for the current chat or session without editing gateway config or restarting the gateway.

## Best Possible Solution

Add a shared preview-streaming override contract and resolver with channel opt-in, expose show/set/reset through `/stream` plus `/status`, and return a clear unsupported response on channels that cannot apply preview overrides.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The absence is high-confidence from source inspection of the command registry, loaded handlers, session state, shared resolver, docs, and Telegram/Slack/Discord preview paths.

## Solution Assessment

Unclear until maintainers choose session-vs-conversation persistence and the channel/plugin contract shape. The maintainable direction is a shared override resolver with opt-in channel support, not a Telegram-only toggle or a command that rewrites gateway config.
