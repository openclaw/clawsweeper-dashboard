## Summary

Keep open. Current main still lacks a `/stream` or `/streaming` command, a loaded preview-streaming command handler, session/conversation override state, and a resolver input that can layer a local override ahead of channel config; the linked reports are adjacent streaming/alias issues rather than a canonical implementation of this cross-channel control.

## What This Changes

Request a built-in slash/native command to show, set, and reset preview-streaming mode for the current chat/session across preview-capable channels.

## Best Possible Solution

Add a shared preview-streaming override contract/resolver that layers an explicit session or conversation override before channel defaults, expose show/set/reset through `/stream` plus `/status`, and let capable channels opt in while unsupported channels return a clear unsupported response.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The absence is high-confidence by source inspection of the command registry, runtime handlers, session state, shared streaming resolver, docs, and Telegram/Slack/Discord preview paths.

## Solution Assessment

Unclear until maintainers choose session-vs-conversation persistence and the channel/plugin contract shape. The maintainable direction is a shared override resolver with channel opt-in, not editing gateway config from a command or adding a Telegram-only toggle.
