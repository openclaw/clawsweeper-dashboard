## Summary

Keep open. Current main and the latest stable release still expose preview streaming through static channel config only; there is no shared `/stream` command or current-chat/session streaming-mode override, and related #80989 is a narrower `/progress` slice rather than a superseding item.

## What This Changes

Add a slash/native command such as `/stream` to show, set, or reset preview streaming mode for the current chat/session without editing gateway config or restarting.

## Best Possible Solution

Define the override scope and shared channel capability contract first, then implement the command so static channel config remains the fallback and unsupported channels answer explicitly.

Not applicable; this is a feature request, and source inspection confirms current main lacks the requested command and per-session override.

## Solution Assessment

Unclear until maintainers choose the storage and capability model; a `/fast`-style session command is plausible, but cross-channel streaming support needs an explicit shared contract.
