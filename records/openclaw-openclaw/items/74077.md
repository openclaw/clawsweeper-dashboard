## Summary

Keep open. Current main still has no `/stream` or `/streaming` command, no command handler, no session/conversation override field, and the inspected Telegram, Slack, and Discord preview paths still resolve streaming mode from channel/account configuration rather than a runtime override. The request is concrete, but it needs maintainer design for persistence scope and the shared channel/plugin contract before it is a safe automated fix.

## What This Changes

Add a slash/native command to view, set, and reset preview streaming mode for the current chat or session without editing gateway config or restarting the gateway.

## Best Possible Solution

Add a shared preview-streaming override resolver that runs before channel/account config defaults, expose it through a built-in status/set/reset command, show active overrides in `/status`, and let preview-capable channels opt in while unsupported channels return a clear unsupported response.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The absence is high-confidence by source inspection: current main has no `/stream` registration or handler, no persisted override field, and the inspected preview dispatch paths resolve from config-only inputs.

## Solution Assessment

Unclear until maintainers choose the override scope and persistence model. The maintainable direction appears to be a shared channel override contract plus channel opt-in, rather than a Telegram-only command or a command that cannot affect the existing dispatch paths.
