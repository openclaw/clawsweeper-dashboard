## Summary

Keep open. Current main still has config-backed preview streaming for Telegram, Slack, Discord, Matrix, and Mattermost, but it does not have a `/stream` or `/streaming` command, a loaded handler, persisted session/conversation override state, or a shared resolver that can layer a local override ahead of channel config.

## What This Changes

Add a per-chat or per-session slash command to view, set, and reset preview streaming mode without editing gateway config or restarting the gateway.

## Best Possible Solution

Add a shared preview-streaming override contract and resolver, expose show/set/reset through `/stream` plus `/status`, and let capable channels opt in while unsupported channels return a clear unsupported response.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The absence is high-confidence by current-main inspection of the command registry, loaded handlers, session state, shared streaming resolver, docs, and Telegram/Slack/Discord delivery paths.

## Solution Assessment

Unclear until maintainers choose the override scope and plugin/channel contract. The maintainable direction is a shared resolver with channel opt-in, not a Telegram-only toggle or a command that edits gateway config.
