## Summary

Keep open. Current main still lacks a `/stream` or `/streaming` command, a command handler, a persisted session/conversation streaming override, and a shared override-aware resolver; the preview-capable channel paths still derive effective behavior from channel config and local dispatch state. The related items are useful adjacent context, but none is a canonical implementation of this requested per-chat/session command.

## What This Changes

Request a slash/native command such as `/stream` to show, set, and reset the current chat/session preview-streaming mode without editing config or restarting the gateway.

## Best Possible Solution

Add an override-aware preview-streaming resolver before channel/account config defaults, persist the selected scope intentionally, expose status/set/reset through a built-in command, show active overrides in `/status`, and let preview-capable channels opt in while unsupported channels return a clear unsupported response.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The absence is high-confidence by source inspection: current main has no stream command registration or handler, no persisted session override field, and inspected preview paths resolve from config-only inputs.

## Solution Assessment

Unclear until maintainers choose the override scope and persistence model. The maintainable direction appears to be a shared channel override contract plus channel opt-in, rather than a Telegram-only command or a command that edits gateway config.
