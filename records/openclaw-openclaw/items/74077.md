## Summary

Keep open. Current main still has preview streaming as channel configuration plus channel-specific dispatch behavior; there is no built-in `/stream` or `/streaming` command, no session/conversation override field, and no shared runtime override resolver consumed by Telegram, Slack, or Discord. The existing ClawSweeper comment and the related reports are correctly treated as adjacent context rather than a canonical implementation.

## What This Changes

The issue requests a shared slash/native command such as `/stream` to inspect, set, reset, and show per-chat or per-session preview streaming mode overrides without editing channel config or restarting the gateway.

## Best Possible Solution

Add a shared preview-streaming override resolver that runs before channel/account config defaults, expose it through a built-in status/set/reset command, show active overrides in `/status`, and let preview-capable channels opt in while unsupported channels return a clear unsupported response.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The absence is high-confidence by source inspection: current main has no `/stream` registration or handler, no persisted override field, and the inspected preview paths resolve from config-only inputs.

## Solution Assessment

Unclear until maintainers choose the override scope and persistence model. The maintainable direction appears to be a shared channel override contract plus channel opt-in, rather than a Telegram-only command or a command that cannot affect the existing dispatch paths.
