## Summary

Keep open. Current main still lacks the requested `/stream` or `/streaming` command, any loaded command handler, a persisted session/conversation preview-streaming override, and resolver inputs that would layer a local override ahead of channel config; the linked items are adjacent streaming or alias work rather than a canonical implementation.

## What This Changes

Add a shared slash/native command to show, set, and reset preview streaming mode for the current chat/session without editing gateway config or restarting.

## Best Possible Solution

Add a shared preview-streaming override contract/resolver with channel opt-in, expose show/set/reset through `/stream` plus `/status`, and return a clear unsupported response on channels that cannot apply preview overrides.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The absence is high-confidence from inspection of the command registry, loaded handlers, session state, shared resolver, docs, and Telegram/Slack/Discord preview paths.

## Solution Assessment

Unclear until maintainers choose session-vs-conversation persistence and the plugin/channel contract shape. The maintainable direction is a shared override resolver with opt-in channel support, not a Telegram-only toggle or a command that rewrites gateway config.
