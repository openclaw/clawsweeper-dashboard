## Summary

Keep open. Current main still lacks a `/stream` or `/streaming` command, a runtime command handler, and any session or conversation override field for preview streaming. The existing Telegram, Slack, and Discord preview paths resolve from channel config and dispatch-local state. The related reports are adjacent streaming/config/observability work, not a canonical implementation of this per-chat/session control.

## What This Changes

Request to add a slash/native command that can show, set, and reset preview streaming mode for the current chat or session without editing gateway config or restarting.

## Best Possible Solution

Add a shared preview-streaming override resolver ahead of channel/account defaults, persist the chosen scope intentionally, expose show/set/reset through the built-in command system, surface active overrides in `/status`, and let preview-capable channels opt in while unsupported channels return a clear unsupported response.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The absence is high-confidence by source inspection: current main has no stream command registration, no handler, no session override field, and inspected preview paths resolve from config-only inputs.

## Solution Assessment

Unclear until maintainers choose the override scope and persistence model. The narrow maintainable direction appears to be a shared channel override contract with channel opt-in, not a Telegram-only command or a command that edits gateway config.
