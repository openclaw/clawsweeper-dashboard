## Summary

Keep open. Current main still has channel-config-based preview streaming but no `/stream` or `/streaming` command, no loaded command handler, no session/conversation override field, and no resolver path that layers a local override ahead of channel config; the related items remain adjacent streaming or command work rather than a canonical implementation.

## What This Changes

Request a chat-native `/stream` or `/streaming` command to show, set, and reset preview-streaming mode for the current chat/session without editing gateway config or restarting.

## Best Possible Solution

Add a shared preview-streaming override contract that resolves before channel config, expose show/set/reset through `/stream` plus `/status`, and let preview-capable channels opt in while unsupported channels return a clear unsupported response.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The missing behavior is high-confidence by source inspection of the command registry, handler loader, session state, shared resolver, channel paths, and docs.

## Solution Assessment

Unclear until maintainers choose session-vs-conversation persistence and the plugin/channel contract. The maintainable direction is a shared override resolver with channel opt-in, not a Telegram-only toggle or command-driven config rewriting.
