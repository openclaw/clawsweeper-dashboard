## Summary

Keep open. Current main still has a single exact memory-flush write target, and allowing `SESSION_HANDOFF.md` needs an explicit write-target contract rather than prompt-derived authority.

## What This Changes

The issue asks for memory-flush writes to allow `SESSION_HANDOFF.md` or other explicitly configured prompt targets in addition to `memory/YYYY-MM-DD.md`.

## Best Possible Solution

Add a typed memory-flush write-target contract with normalized workspace-root validation, traversal and sibling-prefix rejection, explicit append versus overwrite modes, schema/docs updates, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure the flush prompt to request `SESSION_HANDOFF.md`, trigger a memory flush, and the current write wrapper rejects that path because only `activeMemoryFlushPlan.relativePath` is forwarded.

## Solution Assessment

No, not as prompt-derived permission. The maintainable solution is an explicit configured or plugin-provided allowlist with validated paths and per-target write modes.
