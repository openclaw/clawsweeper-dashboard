## Summary

Keep open. The underlying channel restart-recovery bug is plausible, but this PR is not mergeable as-is because the new guard treats internal WebChat/local route metadata as channel-backed external delivery state and would disable automatic recovery for sessions that current main intentionally resumes.

## What This Changes

The PR marks channel-backed restart-aborted main sessions failed for manual recovery and adds a Discord-shaped regression test for that path.

## Best Possible Solution

Keep the recovery decision in `src/agents/main-session-restart-recovery.ts`, but classify only external deliverable routes with real delivery targets as manual-recovery-only while preserving automatic recovery for WebChat/local main sessions, then document the user-visible behavior in the changelog.

## Reproduction Assessment

Yes. A source-level reproduction is a running `abortedLastRun` main session with a resumable tail plus persisted WebChat route/origin metadata; current main resumes it with `deliver: false`, while this PR would mark it failed before transcript recovery.

## Solution Assessment

No. The decision point is appropriate, but the predicate should normalize and require an external deliverable channel/destination, explicitly excluding WebChat/internal metadata and adding WebChat/local regression coverage.
