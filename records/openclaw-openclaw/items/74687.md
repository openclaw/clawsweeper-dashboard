## Summary

Keep open. Current main still replays restart-aborted main sessions through the synthetic gateway agent path without checking channel delivery metadata, while this PR applies a narrow guard and regression test for the channel-backed case. I found no functional or security blocker in the proposed code, but the user-visible behavior change is missing the required changelog entry.

## What This Changes

The PR adds channel-backed session detection to main-session restart recovery, marks those interrupted sessions failed for manual recovery before synthetic replay, and adds a regression test for that path.

## Best Possible Solution

Land the narrow restart-recovery guard after adding the required changelog entry and maintainer validation, keeping automatic recovery for local non-channel sessions while preventing hidden synthetic replay for sessions with durable channel delivery metadata.

## Reproduction Assessment

Yes. A high-confidence reproduction path is to seed a running `abortedLastRun` channel-backed main session with a resumable tool-result tail; current main reaches `callGateway({ method: "agent", deliver: false })`, while the PR test asserts the patched path marks it failed without replay.

## Solution Assessment

Yes, with one small process fix. The guard sits at the restart-recovery decision point before the synthetic agent turn can re-enter delivery-related session state, and the safer alternative would still need the same persisted metadata check plus focused coverage.
