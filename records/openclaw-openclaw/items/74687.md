## Summary

Keep open. Current main still allows restart-aborted channel-backed main sessions to reach the synthetic `agent` replay path, while this PR adds a narrow guard and regression coverage for the channel-backed case. The remaining concrete issue is the missing required changelog entry for a user-visible fix.

## What This Changes

The PR adds channel-backed session detection to main-session restart recovery, marks those restart-aborted sessions failed for manual recovery instead of replaying a synthetic recovery turn, and adds a regression test for Discord-style channel metadata.

## Best Possible Solution

Land the recovery guard after adding the required changelog entry and confirming CI, preserving automatic recovery for local non-channel sessions while forcing channel-backed interrupted sessions into explicit manual recovery.

## Reproduction Assessment

Yes. A high-confidence unit reproduction is to seed a running `abortedLastRun` channel-backed main session with a resumable tool-result tail; current main reaches the synthetic `agent` call, while the PR asserts the patched path marks it failed without replay.

## Solution Assessment

Yes, with one process fix. The guard is at the restart-recovery decision point before hidden replay can re-enter channel session state, and a safer alternative would still need the same persisted route-metadata check plus focused coverage.
