## Summary

Keep open. The PR targets a real restart-recovery hazard, but the patch is too broad because it treats internal WebChat/local session metadata as channel-backed delivery state, and it is also missing the required changelog entry.

## What This Changes

The PR marks channel-backed restart-aborted main sessions failed for manual recovery instead of replaying a synthetic recovery turn, and adds a Discord-shaped regression test.

## Best Possible Solution

Keep the guard at the restart-recovery decision point, but classify only routable external delivery state as manual-recovery and preserve automatic recovery for internal WebChat/local main sessions.

## Reproduction Assessment

Yes. A source-level reproduction is a running `abortedLastRun` main session with a resumable tail plus persisted WebChat origin or `lastChannel` metadata; current main resumes it with `deliver: false`, while this PR would mark it failed.

## Solution Assessment

No. The decision point is appropriate, but the predicate should require an external deliverable route with a target and should explicitly exclude internal WebChat/control-UI metadata.
