## Summary

Keep open: current main still permits a later non-terminal tool warning to replace an already streamed Feishu answer. The long-response symptom overlaps the canonical open report at https://github.com/openclaw/openclaw/issues/91159, but the overwrite behavior is a separate, narrowly actionable defect.

Likely related people: @xzq-xu introduced Feishu Card Kit streaming; @rexl2018 and @HaoHuaqing later worked on streaming merge, final deduplication, and full-content preservation (high confidence).

## What This Changes

Prevent Feishu streaming cards from truncating long replies or replacing an already displayed assistant answer with a later tool-error warning.

## Best Possible Solution

Keep the streamed assistant answer stable and suppress or deliver a later non-terminal tool warning separately, while coordinating all length-limit work through https://github.com/openclaw/openclaw/issues/91159.

## Reproduction Assessment

Yes at source level for the overwrite behavior: supported dispatch ordering can produce an assistant final followed by a tool-warning final, and Feishu replaces non-prefix final text. The CardKit truncation threshold has not been reproduced live.

## Solution Assessment

Yes for the overwrite half: a focused Feishu live-adapter policy fix is narrower and safer than changing core dispatch or adding configuration. The truncation half should remain coordinated with the canonical related issue.
