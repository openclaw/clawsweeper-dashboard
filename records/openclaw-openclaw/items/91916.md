## Label Justifications

- `P1`: Multiple users reproduce a default Feishu streaming workflow losing nearly the entire visible reply on the latest shipped release.
- `impact:message-loss`: The Feishu card truncates or replaces the intended answer, suppressing content that was otherwise generated correctly.

## Summary

Current main does not resolve the reported failure: the Feishu dispatcher still replaces accumulated streaming text with every final payload, matching reports of tail-only finals and later errors replacing the correct answer. The older open report covers truncation but does not clearly own the distinct error-overwrite behavior, so this issue remains necessary.

## What This Changes

Fix Feishu streaming cards so long replies remain complete and later tool errors do not replace an already-correct reply.

## Best Possible Solution

Treat the accumulated visible answer as canonical closeout content, replacing it only when a final payload is known to contain the complete authoritative answer, with regression coverage and live Feishu proof for both reported scenarios.

## Reproduction Assessment

Yes at source level: current main unconditionally replaces accumulated streaming text with each final payload, matching the v2026.6.5 last-character and error-replacement reports. A live current-main tenant run remains necessary as pre-landing proof.

## Solution Assessment

No, the earlier cumulative-update repair was incomplete for this report. The narrow maintainable fix belongs in the Feishu dispatcher’s final-content selection while the streaming session continues to own CardKit transport.
