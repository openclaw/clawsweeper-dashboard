The issue reports that Feishu card streaming truncates long responses and can replace a correct displayed reply with a later failed-tool error.

## Best Possible Solution

Keep this open as a Feishu streaming-card reliability bug and fix the dispatcher/finalize path so final or error events cannot replace a valid accumulated reply with tail-only or lower-priority error content.

## Reproduction Assessment

Yes for triage, but not as a live reproduction in this read-only review. The reporter matrix on shipped 2026.6.5 plus the related still-open Feishu truncation issue give a credible current reproduction path that needs maintainer/live confirmation before a fix lands.

## Solution Assessment

Unclear whether the exact implementation fix is best until the final/update path is isolated. The maintainable direction is to preserve accumulated successful reply content through Feishu streaming closeout and prevent later failed-tool errors from taking precedence over a correct user-facing reply.
