## Summary

The issue is still necessary: v2026.6.5 and current main have the same Feishu streaming-card behavior, including a finalization path that can replace accumulated text, while existing oversized-response tests do not prove Card Kit updates preserve long content. The report is concrete and bounded enough for a focused Feishu fix, but live Feishu/API-contract validation should accompany the implementation.

## What This Changes

Fix Feishu card streaming so long assistant responses remain complete and later tool-call errors do not replace an already correct reply.

## Best Possible Solution

Keep accumulated assistant text authoritative through card closeout, present later tool failures non-destructively, and add an explicit tested fallback to chunked Feishu messages/cards when Card Kit cannot safely display the complete content.

## Reproduction Assessment

Yes, at source level with high confidence for the overwrite symptom: a non-prefix final payload replaces the accumulated card text. The long-response update path is also clear, although its precise Feishu limit still needs live/API confirmation.

## Solution Assessment

Yes; a focused repair in the Feishu streaming-card and reply-dispatcher lifecycle is the narrowest maintainable solution, without new configuration or changes to core agent semantics.
