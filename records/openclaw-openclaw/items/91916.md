## Summary

This issue remains necessary: v2026.6.5 and current main retain a Feishu finalization path that can replace accumulated assistant text, while the long-card symptom still needs live Card Kit validation and is not fully owned by another report.

## What This Changes

Fix Feishu streaming cards so long replies remain complete and later tool errors do not replace already-correct assistant text.

## Best Possible Solution

Keep accumulated assistant output authoritative through closeout, show later tool failures non-destructively, and fall back to chunked Feishu delivery whenever the complete response cannot safely fit the streaming-card contract.

## Reproduction Assessment

Yes for the overwrite symptom: current source deterministically replaces accumulated streaming text with a distinct final payload. The long-response update path is also identifiable, but its exact failure mode still needs live Feishu validation.

## Solution Assessment

Yes in direction: the narrow maintainable repair belongs in the Feishu dispatcher/session boundary and should preserve answer state plus add contract-aware fallback delivery, without introducing another configuration option.
