## Summary

Keep open: the utility import cleanup is mostly harmless, but the proposed `toolStartData` sweep can delete metadata for active long-running tools and the external PR still lacks real behavior proof.

## What This Changes

The branch replaces local `clamp`/`isRecord` helpers with shared imports and adds a five-minute run-idle sweep for `toolStartData` in the embedded tool handler.

## Best Possible Solution

Rebase onto current main, keep only applicable utility dedupe, implement orphan cleanup that never deletes runs with in-flight tools, add focused regression tests, and provide redacted runtime proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path for the blocking PR defect: start one tool, leave it in flight for more than five minutes, then start another tool so the sweep deletes the first tool's start record before its end event.

## Solution Assessment

No. Timestamp-only idle detection is not the narrowest safe fix; orphan cleanup should use `countActiveToolExecutions` or an equivalent in-flight guard before deleting per-run start records.
