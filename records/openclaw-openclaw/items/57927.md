## Summary

Keep open: the shared-helper cleanup is fine, but the proposed `toolStartData` sweep can delete state for an active long-running tool call, and the external PR still lacks real behavior proof.

## What This Changes

The PR replaces two local helper implementations with shared utilities and adds a per-run idle sweep for `toolStartData` records.

## Best Possible Solution

Rebase the branch, keep the applicable helper cleanup, and implement garbage collection by skipping any run with in-flight tool calls, backed by focused tests and redacted runtime proof.

## Reproduction Assessment

Yes for the proposed-patch defect from source inspection: start one tool, let it run beyond five minutes, then start another tool so the sweep deletes the first tool's start record before its end handler runs. I did not run a live scenario in this read-only review.

## Solution Assessment

No: importing shared helpers is fine, but timestamp-only run activity is not the safest cleanup mechanism. The maintainable fix is to track in-flight tool calls per run and sweep only runs with no outstanding calls.
