## What This Changes

The branch replaces two local helper functions with shared imports and adds a five-minute run-idle sweep for `toolStartData` in the embedded tool handler.

## Best Possible Solution

Keep the utility cleanup that still applies, rework orphan cleanup to skip runs with in-flight tool records, add regression tests, and provide redacted runtime proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: start one tool, leave it in flight for more than five minutes, then start another tool so the sweep deletes the first tool's start record before its end event.

## Solution Assessment

No. Timestamp-only idle detection is not the safest fix; orphan cleanup should use `countActiveToolExecutions` or an equivalent in-flight guard before deleting per-run start records.
