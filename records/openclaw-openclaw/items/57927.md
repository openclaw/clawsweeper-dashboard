## Summary

Keep open: the memory-leak motivation remains plausible, but the PR’s idle-run sweep can delete metadata for active long-running tools and the external contributor has not provided real behavior proof.

## What This Changes

The PR replaces local utility helpers with shared imports and adds a five-minute per-run idle sweep for `toolStartData`.

## Best Possible Solution

Keep the utility cleanup where still applicable and implement orphan cleanup with an active-call-safe guard, focused regression coverage, and redacted runtime proof.

## Reproduction Assessment

Yes by source inspection for the PR-introduced defect: start one tool, leave it in flight for more than five minutes, trigger another tool start, and the sweep deletes the first tool’s start data before its end event. I did not run live proof in this read-only review.

## Solution Assessment

No. Timestamp-only run activity is not safe because active long-running calls are quiet; the safer fix is to preserve records while a run has in-flight calls, using `countActiveToolExecutions` or an equivalent guard.
