## Summary

Keep open: the utility cleanup is low-risk, but the new idle sweep can delete start metadata for an active long-running tool because run activity is stamped only on tool start/end. The PR also still lacks after-fix real behavior proof.

## What This Changes

This PR replaces local `clamp` and `isRecord` helpers with shared utilities and adds a five-minute per-run idle sweep for `toolStartData` entries.

## Best Possible Solution

Keep the utility cleanup only where it still applies to current main, rework orphan cleanup to skip runs with active tool executions, add focused regression coverage, and require redacted runtime proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction for the PR-introduced defect: start one tool, leave it in flight for more than five minutes, then start another tool so the sweep deletes the first tool's start record before its end event.

## Solution Assessment

No. Timestamp-only idle detection is not the narrowest safe fix; orphan cleanup should use `countActiveToolExecutions` or an equivalent in-flight guard before deleting per-run start records.
