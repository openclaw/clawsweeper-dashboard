## Best Possible Solution

Rework the sweep to preserve active in-flight tool calls per run, rebase onto current main’s doctor/config split, and add focused regression coverage plus redacted runtime proof.

## Reproduction Assessment

Yes for the proposed-patch defect from source inspection: start a tool, let it run past five minutes, then start any other tool so the sweep deletes the first tool’s start data before it ends. I did not run a live repro because this review is read-only.

## Solution Assessment

No. The utility imports are fine, but the timestamp-only run-idle heuristic is not the best fix because active tools do not heartbeat; the maintainable path is an active-call count or equivalent in-flight fact.
