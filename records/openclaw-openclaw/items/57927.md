## Summary

Keep open: current main has not implemented the orphan cleanup, and this PR's idle-time sweep can evict metadata for a still-running tool call. The shared utility cleanup is partly useful, but the doctor hunk is stale on current main.

## What This Changes

The PR imports shared utility helpers in the read/doctor paths and adds a per-run idle-time sweep for `toolStartData` records.

## Best Possible Solution

Implement orphan cleanup with an explicit in-flight count or equivalent per-run guard, preserve the still-valid read-tool clamp cleanup, drop the stale doctor hunk, and add focused regression coverage plus the required changelog entry.

## Reproduction Assessment

Yes. Start one tool call, advance past the five-minute idle window without ending it, trigger another tool start so the sweep runs, then end the first call and observe that `startData`-derived duration and args are missing.

## Solution Assessment

No. `lastRunActivity` alone is not the best fix because an active long-running call emits no intermediate activity; the safer solution is to sweep only runs with no outstanding tool calls.
