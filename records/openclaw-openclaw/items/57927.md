## What This Changes

The PR imports shared `clamp`/`isRecord` helpers and adds a five-minute per-run idle sweep for orphaned `toolStartData` entries in the embedded tool handler.

## Best Possible Solution

Rebase the branch, keep the helper cleanup that still applies, and replace timestamp-only cleanup with in-flight per-run tracking plus focused tests and redacted runtime proof.

## Reproduction Assessment

Yes for the proposed-patch defect from source inspection: start one tool, let it run beyond five minutes, then start another tool so the sweep deletes the first tool's start record before its end handler runs. I did not run a live scenario in this read-only review.

## Solution Assessment

No: importing shared helpers is fine, but timestamp-only run activity is not the best cleanup mechanism. The safer path is to track in-flight tool calls per run and sweep only runs with no outstanding calls.
