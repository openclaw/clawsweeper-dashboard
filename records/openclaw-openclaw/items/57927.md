## Summary

Keep open: the utility dedupe is low risk, but the proposed orphan sweep can delete start metadata for active long-running tools, and the external PR still lacks real behavior proof.

## What This Changes

The PR branch replaces duplicate local utility helpers with shared imports and adds a five-minute per-run idle sweep for `toolStartData`.

## Best Possible Solution

Rebase onto current main, keep only applicable utility cleanup, rework orphan cleanup to skip runs with active tool executions, add focused regression coverage, and post redacted runtime proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction for the PR-introduced defect: start one tool, leave it in flight for more than five minutes, then start another tool so the sweep deletes the first tool's start record before its end event.

## Solution Assessment

No. Timestamp-only idle detection is not the narrowest safe fix; orphan cleanup should use `countActiveToolExecutions` or an equivalent in-flight guard before deleting per-run start records.
