## Summary

Keep open: current main still has the relevant `toolStartData` lifecycle surface, but this PR is not merge-ready because the proposed timestamp-only sweep can evict active long-running tools, the branch is conflicting, and no real behavior proof was provided.

## What This Changes

The PR replaces local `clamp` and `isRecord` helper copies with shared utilities and adds a five-minute run-idle sweep for orphaned `toolStartData` records.

## Best Possible Solution

Track per-run in-flight tool calls and sweep only runs with no outstanding calls after the idle window, with regression coverage and redacted runtime proof for abandoned cleanup and long-running active-tool preservation.

## Reproduction Assessment

Yes for the proposed-patch defect from source inspection: start one tool, let it run beyond the five-minute idle threshold, then start another tool so the sweep deletes the first tool's start record before its end handler runs. This read-only review did not run a live scenario.

## Solution Assessment

No: the duplicate utility cleanup is fine, but timestamp-only run activity is not the best fix for orphaned `toolStartData` because active tools do not heartbeat while running. The safer solution is to track in-flight calls and sweep only inactive runs.
