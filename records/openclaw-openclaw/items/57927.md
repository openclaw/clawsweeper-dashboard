## Summary

Keep open: the cleanup goal is valid, but the PR's run-idle sweep can delete start data for an active long-running tool call, and the branch is also stale against current main.

## What This Changes

The PR replaces two local utility helpers with shared imports and adds a run-idle-based sweep for orphaned `toolStartData` entries.

## Best Possible Solution

Implement orphan cleanup with an explicit active in-flight count or equivalent guard per run, preserve the still-valid `clamp` dedupe, drop the stale doctor hunk, and add focused regression coverage plus the required changelog entry.

## Reproduction Assessment

Yes. Start one tool call, advance past the five-minute idle window without ending it, trigger another tool start so the sweep runs, then end the first call and observe that `startData`-derived duration and args are missing.

## Solution Assessment

No. The shared utility import cleanup is reasonable where it still applies, but `lastRunActivity` alone is not a safe cleanup signal because an active long-running call emits no intermediate activity.
