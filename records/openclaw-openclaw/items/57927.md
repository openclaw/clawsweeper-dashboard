## Summary

Keep open: the cleanup intent is plausible, but the proposed idle sweep can delete start data for an active long-running tool and the external PR has no after-fix real behavior proof.

## What This Changes

The branch imports shared utility helpers for `clamp`/`isRecord` and adds run-idle sweeping for `toolStartData` to clean abandoned tool execution records.

## Best Possible Solution

Update the PR to implement orphan cleanup with an explicit in-flight count or equivalent active-call guard, preserve only the still-valid utility cleanup, refresh stale hunks, and add focused regression coverage plus real runtime proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: start one tool, wait past the five-minute window without ending it, trigger another tool start so the sweep runs, then end the first tool and observe missing start-data-derived duration and args.

## Solution Assessment

No. The utility import cleanup is reasonable where it still applies, but `lastRunActivity` alone is not a safe cleanup signal because an active long-running call emits no intermediate activity.
