## Summary

Keep open: the duplicate helper cleanup is harmless, but the proposed `toolStartData` sweep is not merge-ready because source inspection shows it can delete active long-running tool state, the external PR still lacks real behavior proof, and the branch conflicts with current main.

## What This Changes

The branch replaces local `clamp` and `isRecord` helper copies with shared utilities and adds a five-minute run-idle sweep for orphaned `toolStartData` records.

## Best Possible Solution

Track active in-flight tool calls per run and sweep only runs with no outstanding calls after the idle window, with focused regression coverage and redacted runtime proof for both abandoned-run cleanup and long-running active-tool preservation.

## Reproduction Assessment

Yes for the proposed-patch defect from source inspection: start one tool, let it run beyond the five-minute idle threshold, then start another tool so the sweep deletes the first tool's start record before its end handler runs. This read-only review did not run a live scenario.

## Solution Assessment

No: the duplicate utility cleanup is fine, but timestamp-only run activity is not the best fix for orphaned `toolStartData` because active tools do not heartbeat while running. The safer solution is to track in-flight calls and sweep only inactive runs.
