## What This Changes

## Best Possible Solution

Classify only Discord Unknown Voice State, preferably `discordCode === 10065` or an explicit Unknown Voice State message, keep unrelated REST failures rejecting, and require redacted real Discord proof before merge.

## Reproduction Assessment

No live high-confidence reproduction was established. Source inspection is enough to show current main directly propagates `getGuildVoiceState` failures through `voiceStatus`, while the PR's regression coverage is mocked.

## Solution Assessment

No: normalizing explicit Unknown Voice State in the Discord helper is the right layer, but using every status-only 404 is broader than the available Discord error-code contract. The safer fix is a narrow classifier plus real Discord proof.
