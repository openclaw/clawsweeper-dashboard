## Best Possible Solution

Land a narrower Discord-code-aware classifier for code 10065 or explicit Unknown Voice State, keep other 404s failing, and include redacted real Discord voice-status proof before merge.

## Reproduction Assessment

## Solution Assessment

Is this the best way to solve the issue? No: returning an absent state for Unknown Voice State is the right direction, but matching every 404 is too broad when Discord and the local REST client expose a more specific error code.
