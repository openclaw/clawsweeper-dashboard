## Summary

Keep open. Current main has partial hook timeouts, plugin-local breaker work, and stalled-session diagnostics, but it still lacks the requested generic plugin circuit breaker, persisted breaker/status/reset policy, and hard framework timeout around plugin-owned tool execution; recent Telegram freeze context also keeps the issue active.

## What This Changes

The issue requests plugin-scoped crash-loop protection, hard plugin call timeouts, and automatic session recovery when plugin failures leave Telegram sessions stuck.

## Best Possible Solution

Design a staged runtime resilience feature that adds plugin-scoped failure accounting, abortable hook/tool timeout contracts, persisted breaker state/status/reset controls, and diagnostics-driven no-progress recovery without changing existing users to fail-closed by surprise.

## Solution Assessment

Unclear; the requested direction is reasonable, but a one-off timeout or typing-TTL restart patch would not be the best maintainable fix. The safer path is a designed runtime policy that composes with current diagnostics and the open circuit-breaker design thread.
