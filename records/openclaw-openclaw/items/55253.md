## Summary

Keep open. Current main has partial resilience for invalid plugin config, hook timeouts, memory recall timeouts, typing cleanup, and restart recovery, but it still lacks the requested plugin-scoped circuit breaker, generic hard timeout/cancellation around plugin-owned tool execution, and typing-TTL-driven session recovery.

## What This Changes

The issue asks OpenClaw to recover gracefully from plugin crash loops by adding a plugin circuit breaker, plugin tool-call timeouts, and optional session recovery when typing TTL indicates a stuck session.

## Best Possible Solution

Design this as a core plugin-runtime resilience feature with plugin-scoped breaker states, explicit timeout and AbortSignal contracts for plugin-owned tools, operator status/reset controls, and a deliberate policy for stuck-session or typing-TTL recovery.

## Reproduction Assessment

No, there is not a high-confidence live reproduction path for the reported memory-lancedb-pro crash loop from the issue alone. Source inspection does give a high-confidence gap check for the missing breaker, generic plugin tool timeout, and typing-TTL recovery behavior.

## Solution Assessment

Unclear; the requested direction is reasonable, but the best implementation needs maintainer design across plugin runtime, cancellation semantics, persistence, and session recovery policy. A narrow one-file fix would likely create inconsistent recovery behavior.
