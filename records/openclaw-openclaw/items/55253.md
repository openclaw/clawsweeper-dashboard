## Summary

Keep open. Current main has useful partial resilience for invalid plugin config, hook timeouts, memory recall timeout, and typing cleanup, but it still lacks the requested plugin-scoped circuit breaker, generic hard timeout/cancellation around plugin-owned tool execution, and typing-TTL-driven session recovery.

## What This Changes

Requests a plugin-scoped circuit breaker, typing-TTL-driven session recovery, and generic plugin tool call timeouts after a memory-lancedb-pro crash loop froze Telegram message processing.

## Best Possible Solution

Design one core plugin-runtime resilience feature with plugin-scoped breaker states, abortable timeout contracts for plugin-owned hooks/tools, operator status/reset controls, and an explicit policy for stuck-session or typing-TTL recovery.
