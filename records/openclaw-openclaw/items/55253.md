## Summary

Keep open. Current main has partial resilience for invalid plugin config, typed hook timeouts, active-memory timeouts, memory-lancedb auto-recall timeouts, and typing cleanup, but it still does not implement the requested generic plugin-scoped circuit breaker, hard timeout/cancellation around plugin-owned tool execution, or typing-TTL-driven session recovery; open #41899 covers only the circuit-breaker subset.

## What This Changes

The issue requests a plugin circuit breaker, generic plugin call timeouts, and typing-TTL-driven session recovery to prevent plugin crash loops from leaving sessions stuck.

## Best Possible Solution

Design this as a core plugin-runtime resilience contract with plugin-scoped breaker states, explicit timeout and AbortSignal semantics for plugin-owned tools, operator status/reset controls, and a deliberate stuck-session or typing-TTL recovery policy, while consolidating the breaker slice with #41899.

## Reproduction Assessment

No for the original outage: the report does not include enough memory-lancedb-pro config, logs, or rebuild steps to reproduce the zombie session on current main. Source inspection does give a high-confidence gap check for the missing generic breaker, plugin-tool timeout, and typing-TTL recovery behavior.

## Solution Assessment

Unclear; the requested direction is reasonable, but the best implementation needs maintainer design across plugin runtime policy, cancellation contracts, persistence, operator UX, and session recovery. A narrow patch in one current timeout path would not cover the full failure model.
