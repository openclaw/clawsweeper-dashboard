## Summary

Keep open. Current main still lacks the agent namespace isolation this PR is trying to add, but the PR branch is not merge-ready: it is stacked on open predecessor PRs, currently unmergeable, and the diff has blocking migration, performance, and concurrency regressions that need correction before maintainer review can proceed.

## What This Changes

The PR branch adds agent-id scoping to memory index schema/search/write paths, memory_search scope and provenance handling, QMD mcporter daemon-state isolation, memory CLI provider registration, and related regression tests.

## Best Possible Solution

Rebase the stacked memory series onto current main, keep the package memory-host SDK as the schema source of truth, implement agent_id scoping across schema migration, writes, deletes, status, keyword/vector search, and QMD daemon state, and preserve current main’s bounded vector fallback, yielding cache seeding, and non-batch concurrency semantics before running the memory-core/package validation gate.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main shared memory indexes are unscoped by agent_id, and the PR branch has deterministic branch-local failures such as Set.length on a Set, full materialization of vector fallback candidates, synchronous cache seeding, and ignored non-batch concurrency settings.

## Solution Assessment

No, not as-is. The direction is the right owner-area fix for memory isolation, but the proposed branch should be rebased and corrected so the isolation migration does not regress package schema ownership, large-index behavior, or documented provider concurrency controls.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Use Set.size for the FTS rebuild check:** `packages/memory-host-sdk/src/host/memory-schema.ts:46`
  - body: `readColumns` returns a `Set`, so `existingFtsColumns.length` is always `undefined` and this condition never rebuilds old FTS tables that lack `agent_id`. Existing package-backed databases will keep the old virtual table, and later inserts that include `agent_id` can fail after upgrade.
  - confidence: 0.98
- **[P2] Keep fallback vector scoring bounded:** `extensions/memory-core/src/memory/manager-search.ts:254`
  - body: The fallback vector path now loads every matching chunk with `.all()`, scores all rows, sorts the full array, and then slices. Current main uses an iterator with a bounded top-K accumulator, so this regresses large indexes by materializing all chunks and embeddings in memory.
  - confidence: 0.92
- **[P2] Preserve yielding during cache seeding:** `extensions/memory-core/src/memory/manager-sync-ops.ts:319`
  - body: Changing cache seeding to `.all()` plus a fully synchronous loop removes the existing `setImmediate` yield during large rebuilds. Large embedding caches can now block the event loop long enough to delay gateway health probes and other async work.
  - confidence: 0.9
- **[P2] Preserve non-batch index concurrency semantics:** `extensions/memory-core/src/memory/manager-embedding-ops.ts:103-105`
  - body: This drops `remote.nonBatchConcurrency` and the Ollama default of one non-batch worker by returning the batch concurrency whenever a configured value is present. That contradicts the current tests/docs and can overrun local Ollama or ignore explicit non-batch tuning.
  - confidence: 0.88
