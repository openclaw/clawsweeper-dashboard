## Summary

Keep open. The memory isolation direction is valid and current main still lacks agent namespace scoping in the shared SQLite memory index, but this PR should not merge as-is because the branch has blocking migration and large-index behavior regressions and is still stacked behind open prerequisite PRs.

## What This Changes

The PR branch adds agent_id scoping across memory schema, file/chunk/FTS/vector queries, memory_search provenance/scope handling, QMD mcporter daemon scoping, CLI embedding provider registration, and related regression tests.

## Best Possible Solution

Land the memory series in order only after rebasing and narrowing this branch, fix the package FTS rebuild check to use Set.size, preserve the existing non-batch concurrency contract, keep fallback vector scoring and cache seeding bounded/yielding, add the required changelog entry if this remains user-facing, and validate shared store.path isolation across schema migration, writes/deletes, keyword search, vector search, status, session state, and QMD mcporter daemon state.

## Reproduction Assessment

Yes. Source and diff inspection give high-confidence reproduction paths: current main lacks agent_id in shared memory tables and source filters, and the PR head leaves existing package FTS tables unrebuilt because Set.length is undefined.

## Solution Assessment

No, not as-is. The owner-area approach is the right direction for agent namespace isolation, but the current branch needs migration, concurrency, and large-index fixes plus stacked-series sequencing before it is the narrowest maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Use Set.size for the FTS rebuild check:** `packages/memory-host-sdk/src/host/memory-schema.ts:46`
  - body: `readColumns` returns a `Set`, so `existingFtsColumns.length` is always `undefined` and this condition never rebuilds old FTS tables that lack `agent_id`. Existing package-backed databases will keep the old virtual table, and later inserts that include `agent_id` will fail after upgrade.
  - confidence: 0.98
- **[P2] Preserve non-batch index concurrency:** `extensions/memory-core/src/memory/manager-embedding-ops.ts:94-96`
  - body: This drops `remote.nonBatchConcurrency` and the Ollama non-batch default by replacing the old `configuredNonBatchConcurrency` and `providerId` inputs with `configuredConcurrency`. Current tests require inline indexing to honor that setting and default Ollama to one worker when batch mode is disabled.
  - confidence: 0.93
- **[P2] Keep fallback vector scoring bounded:** `extensions/memory-core/src/memory/manager-search.ts:248-254`
  - body: The fallback vector path now loads every matching chunk with `.all()`, scores all rows, sorts the full array, and then slices. Current main uses an iterator with a bounded top-K accumulator, so this regresses large indexes by materializing all chunks and embeddings in memory.
  - confidence: 0.91
- **[P2] Preserve yielding during cache seeding:** `extensions/memory-core/src/memory/manager-sync-ops.ts:315-340`
  - body: Changing cache seeding to `.all()` plus a fully synchronous loop removes the existing `setImmediate` yield during large rebuilds. Large embedding caches can now block the event loop long enough to delay gateway health probes and other async work.
  - confidence: 0.87
