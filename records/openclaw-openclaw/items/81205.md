## Summary

Keep open. Current main and the latest shipped tag still bind `openclaw wiki` to one resolved memory-wiki config with no `--path`, `--wiki`, `-w`, or `OPENCLAW_WIKI_PATH` selector; the related open runtime templating PR is adjacent but does not add the direct CLI vault selector requested here.

## What This Changes

Request adding a memory-wiki CLI vault selector such as `--path`, `--wiki`, or `OPENCLAW_WIKI_PATH` so maintenance commands can target non-default vaults.

## Best Possible Solution

Define a supported memory-wiki CLI vault-selection contract, then update the plugin-owned CLI registration, config/schema behavior, docs, and focused tests while coordinating with the adjacent vault-templating work.

## Reproduction Assessment

Not applicable as a feature request, but source inspection gives a high-confidence current-main check: `registerWikiCli` closes over one resolved config and no `--path`, `--wiki`, or `OPENCLAW_WIKI_PATH` surface exists.
