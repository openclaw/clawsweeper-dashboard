## Summary

Keep open. Current `main` and `v2026.5.7` still bind `openclaw wiki` to one resolved memory-wiki config, and the related per-agent vault work does not provide the requested direct CLI/path/env selector.

## What This Changes

Request adding a supported `openclaw wiki` vault selector such as `--path`, `--wiki`, or `OPENCLAW_WIKI_PATH` so CLI commands can target office, life, or custom vault paths.

## Best Possible Solution

Define a supported memory-wiki CLI vault-selection contract, then update the plugin-owned CLI registration, config/env precedence, docs, and focused tests around that contract.

## Reproduction Assessment

Yes. Source inspection of current `main` and `v2026.5.7` shows `openclaw wiki` resolves one memory-wiki config and registers no `--path`, `--wiki`, `-w`, or `OPENCLAW_WIKI_PATH` selector.

## Solution Assessment

Unclear. A per-invocation path override is plausible, but maintainers should choose selector names, precedence, and interaction with the related per-agent vault work before implementation.
