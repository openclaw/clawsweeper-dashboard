## Summary

Keep open. Current main and v2026.5.7 still bind `openclaw wiki` to one resolved memory-wiki config with no `-p`/`--path`, `-w`/`--wiki`, or env selector; the adjacent per-agent vault work does not cover this direct CLI cron workflow.

## What This Changes

The issue requests a wiki CLI vault selector for named or custom memory-wiki vaults, such as `-w office` or `--path /custom/vault`, across commands like ingest, compile, lint, and status.

## Best Possible Solution

Define a supported memory-wiki CLI vault-selection contract, then update the plugin-owned CLI registration, config/schema behavior, docs, and focused tests around that contract.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.5.7 shows `openclaw wiki` registers no parent vault selector and all subcommands use one resolved memory-wiki config, so the reported `-w office`/`--path` flow is unsupported.

## Solution Assessment

Unclear. A CLI path or named-vault selector is plausible, but maintainers should choose the supported flag/env/config precedence and coordinate it with the existing per-agent vault-isolation work before implementation.
