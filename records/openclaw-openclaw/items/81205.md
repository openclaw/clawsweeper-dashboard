## Summary

Keep open. Current main and the latest shipped release still bind `openclaw wiki` to one resolved `memory-wiki` vault config, and the broader per-agent vault tracker plus active implementation PR do not add the direct CLI or environment selector requested here.

## What This Changes

The issue requests `openclaw wiki` support for selecting a non-default wiki vault by path, name, or environment variable for commands such as ingest, status, compile, and lint.

## Best Possible Solution

Define a plugin-owned wiki CLI vault-selection contract, then update the registrar, config/schema behavior if needed, docs, and focused tests while coordinating with https://github.com/openclaw/openclaw/issues/63829 and https://github.com/openclaw/openclaw/pull/67584.

## Reproduction Assessment

Yes, source-level. `registerWikiCli` closes over one resolved config and neither code nor docs expose `--path`, `--wiki`, a named vault selector, or `OPENCLAW_WIKI_PATH`.

## Solution Assessment

Unclear. A per-invocation raw path or named-vault selector is plausible, but maintainers need to choose the supported surface and precedence before implementation.
