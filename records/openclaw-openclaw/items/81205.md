## Summary

Keep open. Current main and the latest shipped tag still register `openclaw wiki` against one resolved memory-wiki vault config, and the related per-agent/runtime vault work does not add the requested direct CLI selector.

## What This Changes

Request adding `openclaw wiki` vault selectors such as `-p`/`--path`, `-w`/`--wiki`, or `OPENCLAW_WIKI_PATH` so CLI subcommands can operate on non-default memory-wiki vaults.

## Best Possible Solution

Define a supported memory-wiki CLI vault-selection contract, then update the plugin-owned CLI registration, config/docs, and focused tests around that contract while keeping it coordinated with the per-agent/runtime vault work.

## Reproduction Assessment

Not applicable as a strict bug reproduction because the requested CLI selector is a new feature/config surface. Source inspection does confirm current main has no `-p`/`--path`, `-w`/`--wiki`, or `OPENCLAW_WIKI_PATH` selector.

## Solution Assessment

Unclear. A per-invocation CLI path override is plausible, but maintainers should choose the supported selector names and precedence before implementation, especially relative to https://github.com/openclaw/openclaw/pull/67584 and https://github.com/openclaw/openclaw/issues/63829.
