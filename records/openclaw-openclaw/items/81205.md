## Summary

Keep open. Current main and the latest shipped tag still bind `openclaw wiki` to one resolved memory-wiki vault config, and the direct CLI selector requested here is not covered by the related per-agent/runtime vault work.

## What This Changes

The issue requests `openclaw wiki` support for selecting a non-default memory-wiki vault by path, named vault, or environment default.

## Best Possible Solution

Define a supported memory-wiki CLI vault-selection contract, then update the plugin-owned CLI registration, config/schema behavior, docs, and focused tests around that contract.

## Reproduction Assessment

Yes. Source inspection of current main and `v2026.5.7` shows `openclaw wiki` resolves one memory-wiki config and registers no `--path`, `--wiki`, or `OPENCLAW_WIKI_PATH` selector.

## Solution Assessment

Unclear. A per-invocation path override is plausible, but maintainers should choose the supported selector surface and precedence before implementation, especially relative to the existing per-agent vault-isolation work.
