## Summary

Keep open. Current `main` and latest shipped `v2026.5.18` still register `openclaw wiki` against one resolved `memory-wiki` vault config, and the adjacent per-agent vault work does not add the requested direct CLI or environment selector.

## What This Changes

Add a vault selector to `openclaw wiki` so commands can operate on non-default `memory-wiki` vaults by path, named vault, or environment default.

## Best Possible Solution

Define a memory-wiki-owned CLI vault-selection contract, then update `registerWikiCli`, lazy CLI metadata, config/docs, and focused tests in coordination with the per-agent vault-isolation work.

## Reproduction Assessment

Yes, source-level. Current main registers `openclaw wiki` with one resolved memory-wiki config and current code/docs expose no `--path`, `--wiki`, named-vault, or `OPENCLAW_WIKI_PATH` selector.

## Solution Assessment

Unclear. A per-invocation selector is plausible, but maintainers should choose the supported selector surface and precedence before implementation, especially relative to context-aware vault templating work.
