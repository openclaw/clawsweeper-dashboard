## Summary

Keep open. Current main and the latest shipped tag still expose `openclaw wiki` through one configured `memory-wiki` vault path, and the related per-agent vault-isolation work does not add the requested direct CLI or environment selector.

## What This Changes

Request adds an `openclaw wiki` vault selector such as `--path`, `--wiki`, or `OPENCLAW_WIKI_PATH` so maintenance commands can target office, life, or custom vaults.

## Best Possible Solution

Define a plugin-owned wiki CLI vault-selection contract, then update `registerWikiCli`, the lazy CLI metadata path, config/schema behavior if needed, docs, and focused CLI tests in coordination with the per-agent vault-isolation work.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection confirms current `openclaw wiki` resolves one configured vault and provides no `--path`, `--wiki`, named-vault, or `OPENCLAW_WIKI_PATH` selector.

## Solution Assessment

Unclear. A per-invocation path or named-vault override is plausible, but maintainers should choose the supported selector shape and precedence before implementation, especially relative to the context-aware vault templating work.
