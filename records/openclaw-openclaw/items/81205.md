## Summary

Keep open. Current main and the latest shipped tag still bind `openclaw wiki` to one resolved `memory-wiki` vault config, and the related per-agent vault-isolation issue/PR do not add the requested direct CLI or environment selector.

## What This Changes

The issue requests `openclaw wiki` support for selecting a non-default memory-wiki vault by path, named wiki, or environment default.

## Best Possible Solution

Define a plugin-owned wiki CLI vault-selection contract, then update `registerWikiCli`, the lazy CLI metadata path, config/schema behavior if needed, docs, and focused tests in coordination with the per-agent vault-isolation work.

## Reproduction Assessment

Yes, source-level. Current main registers `openclaw wiki` with one resolved memory-wiki config and has no `--path`, `--wiki`, named-vault, or `OPENCLAW_WIKI_PATH` selector in code or docs.

## Solution Assessment

Unclear. A per-invocation path or named-vault override is plausible, but maintainers should choose the supported selector surface and precedence before implementation, especially relative to context-aware vault templating work.
