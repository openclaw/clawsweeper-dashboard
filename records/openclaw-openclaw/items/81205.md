## Summary

Keep open. Current main and the latest release still bind `openclaw wiki` to one resolved `memory-wiki` vault config, while the related vault-isolation issue and PR do not add the requested CLI or environment selector.

## What This Changes

The issue requests a `openclaw wiki` vault selector such as `--path`, `--wiki`, or `OPENCLAW_WIKI_PATH` so cron jobs can run wiki commands against office/life vaults instead of only the default main vault.

## Best Possible Solution

Define a memory-wiki-owned CLI vault-selection contract with clear precedence, update the CLI registrar/config/docs/tests around that contract, and coordinate the UX with the open vault-isolation work.

## Reproduction Assessment

Yes, source-level. Current main registers `openclaw wiki` with one resolved memory-wiki config and exposes no CLI-level `--path`, `--wiki`, named-vault, or `OPENCLAW_WIKI_PATH` selector.

## Solution Assessment

Unclear. A per-invocation path or named-vault override is plausible, but maintainers should choose the supported selector surface and precedence before implementation, especially relative to the context-aware vault templating work.
