## Summary

Keep open. Current main and the latest shipped tag still expose `openclaw wiki` through one resolved `memory-wiki` config and do not provide the requested CLI-level vault selector; the related per-agent/runtime vault work is adjacent but does not cover direct `openclaw wiki -p/-w` usage.

## What This Changes

Request adding CLI/env vault selection so `openclaw wiki` commands can target non-main memory-wiki vaults such as office or life.

## Best Possible Solution

Define a supported memory-wiki CLI vault-selection contract, then update the plugin-owned CLI registration, config/docs, and focused tests around that contract while coordinating with the ctx-aware vault work.

## Reproduction Assessment

Yes. Source inspection shows `openclaw wiki` closes over one resolved memory-wiki config and current docs/config expose no `-p`, `--path`, `-w`, `--wiki`, or `OPENCLAW_WIKI_PATH` selector.

## Solution Assessment

Unclear. A per-invocation path override is plausible, but maintainers should choose the supported selector surface and precedence before implementation, especially relative to the open per-agent vault-templating PR.
