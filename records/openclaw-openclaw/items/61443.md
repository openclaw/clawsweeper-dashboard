Keep this PR open: the provider retry direction is useful, but the current branch still exposes a generic public retry config while only OpenAI-compatible transports consume it, lacks contributor real behavior proof, and is dirty against current main.

## What This Changes

The PR adds opt-in `models.providers.<name>.retry` config, provider retry runner plumbing, abort-aware retry support, OpenAI-compatible stream wrapping, schema/help metadata, tests, and a changelog entry.

## Best Possible Solution

Choose one canonical LLM retry contract, rebase this PR or a replacement onto current main, scope the config to transports that honor it or wire all supported provider families, preserve timeout/abort/failover semantics, and require redacted real behavior proof before merge.
