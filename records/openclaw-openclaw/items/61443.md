## Summary

Keep this PR open, but it is not ready to merge: the requested provider-level LLM retry surface is still absent from current main, while the PR adds a generic public config key that only OpenAI-compatible transports honor and still lacks after-fix real behavior proof.

## What This Changes

The PR adds opt-in `models.providers.<name>.retry` config, provider retry runner plumbing, abort-aware retry support, OpenAI-compatible transport wrapping, generated config metadata, tests, and a changelog entry.

## Best Possible Solution

Choose one canonical LLM retry contract, scope the config to transports that honor it or wire all supported provider families, preserve current abort/timeout/failover semantics, and require redacted real behavior proof before merge.

## Reproduction Assessment

Yes for the review blocker: PR-head source accepts a generic provider retry config while only OpenAI-compatible stream factories consume the runner. No for real runtime behavior: the PR has not shown an after-fix real provider run.

## Solution Assessment

No. Provider retry is a reasonable feature direction, but this implementation should not merge until the public config scope is resolved, the branch is rebased, and contributor-supplied real behavior proof is added.
