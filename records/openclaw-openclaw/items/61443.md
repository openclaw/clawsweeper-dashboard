## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Carry retry through the runtime model catalog:** `src/agents/openai-transport-stream.ts:730-733`
  - body: `getProviderRetryRunner(model.provider)` depends on module state registered during config normalization, but the current session runtime creates `ModelRegistry` from `models.json` and its load/refresh path does not register retry metadata. A normal existing catalog or process restart can leave this registry empty and silently bypass a configured provider retry policy. Carry retry on the runtime model/request metadata path or register it from `ModelRegistry` load/refresh, with coverage for an existing `models.json`.
  - confidence: 0.88
- **[P2] Account for the SDK's built-in retries:** `src/agents/openai-transport-stream.ts:725-733`
  - body: This outer retry wraps OpenAI SDK calls without changing or documenting the SDK's default retry policy, so `attempts: 3` can become multiple SDK attempts per OpenClaw attempt and SDK retry-after sleeps can happen before the outer policy sees an error. The provider retry contract needs to either disable/cap the SDK retry layer when configured or clearly account for the composed attempt and timing behavior.
  - confidence: 0.78
