## Summary

Keep open: the patch is a narrow, source-backed fix for a real commitment heartbeat prompt-scoping bug, but this external PR still needs after-fix real behavior proof before merge.

## What This Changes

The PR adds a commitment-wake prompt mode in `runHeartbeatOnce` so fan-out commitment wakes ignore broad heartbeat tasks/directives and only advance timestamps for tasks actually selected to run.

## Best Possible Solution

Land this narrow heartbeat-runner fix after adding redacted real behavior proof that a commitment fan-out wake sends only the scoped follow-up and does not surface global HEARTBEAT.md/task text.

## Reproduction Assessment

Yes. Source inspection on current main shows commitment fan-out uses `reason: "commitment"`, while the prompt resolver still lets configured heartbeat tasks/directives dominate or append to commitment prompts; the PR body also includes a focused failing-before/passing-after regression test, though I did not run it in this read-only review.

## Solution Assessment

Yes. The explicit commitment-wake mode is the narrowest maintainable fix I found because it keeps normal interval/manual heartbeat behavior intact while isolating only the fan-out wake path that exists to deliver due commitments.
