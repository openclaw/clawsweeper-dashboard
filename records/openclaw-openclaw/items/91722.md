## Summary

Keep open; the refactor is not merge-ready because source inspection shows untrusted direct-mode URL fetches still inherit ambient env proxies, and the PR removes a shipped Plugin SDK subpath without preserved compatibility or visible maintainer approval.

## What This Changes

The branch replaces the legacy in-process guarded-fetch/SSRF matrix with a centralized egress helper, migrates many HTTP fetch call sites, removes the public SSRF SDK subpath and related knobs, and updates docs, tests, and boundary scripts.

## Best Possible Solution

Keep the centralized egress direction, but make untrusted direct-mode URL helpers bypass ambient env proxies unless managed proxy mode is active, and either preserve `openclaw/plugin-sdk/ssrf-runtime` as a deprecated adapter or land the removal only after explicit maintainer-approved major-version handling.

## Solution Assessment

No. Centralizing egress is a plausible direction, but this patch is not the best safe merge shape until direct-mode untrusted URL fetches stay off ambient proxies and the shipped SDK subpath has a compatibility or explicitly approved removal path.
