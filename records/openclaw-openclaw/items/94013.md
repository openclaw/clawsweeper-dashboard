## What This Changes

The PR changes Feishu monitor startup to fall back to the local registered runtime when an injected channel runtime lacks `inbound`, and adds monitor guard tests for partial, full, and undefined runtime inputs.

## Best Possible Solution

Land the narrow monitor guard after ordinary maintainer review, then keep the broader Feishu no-response issue open until a published Feishu app or user report confirms end-to-end inbound replies.

## Reproduction Assessment

## Solution Assessment

Yes for the known partial-runtime monitor failure. The fix stays inside the Feishu plugin boundary and mirrors the already-merged bot-path guard; stricter gateway runtime validation could be a separate cleanup, but it is not required for this bounded crash-path repair.
