## Summary

Keep open: the patch is a narrow, source-backed Feishu monitor fix, but contributor-supplied proof is still a local guard script plus tests rather than a real Feishu or gateway startup run showing incoming messages are processed after the change.

## What This Changes

The branch changes Feishu monitor startup to fall back to the registered full channel runtime when an injected `channelRuntime` lacks `inbound`, and adds `monitorSingleAccount` guard tests.

## Best Possible Solution

Land the narrow monitor runtime guard after redacted real Feishu or gateway startup proof shows the partial-runtime path no longer prevents incoming message processing.

## Reproduction Assessment

Yes from source inspection: current main selects any truthy `params.channelRuntime`, while the receive handler immediately reads `channelRuntime.debounce` and the public surface only guarantees `runtimeContexts`. I did not run a live Feishu repro in this read-only review.

## Solution Assessment

Yes for the known partial-runtime failure: checking `channelRuntime?.inbound` at the monitor consumption boundary matches the already-merged Feishu bot-path guard and avoids a broader SDK contract change for this narrow bug. Merge should still wait for real behavior proof.
