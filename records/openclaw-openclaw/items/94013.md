## What This Changes

The branch changes Feishu monitor startup to fall back to the registered full channel runtime when an injected `channelRuntime` lacks `inbound`, and adds `monitorSingleAccount` guard tests.

## Best Possible Solution

Land the narrow monitor guard after redacted Feishu/gateway startup proof or an explicit maintainer proof override, then close the linked user report when the fix merges.

## Reproduction Assessment

Yes from source inspection: current main can select a partial Feishu `channelRuntime`, while the receive handler immediately needs full debounce and command runtime APIs. I did not run a live Feishu account reproduction in this read-only review.

## Solution Assessment

Yes for the known partial-runtime failure: checking `channelRuntime?.inbound` at the monitor consumption boundary matches the merged Feishu bot-path guard and keeps the repair inside the Feishu plugin. Merge should still wait for real behavior proof or maintainer override.
