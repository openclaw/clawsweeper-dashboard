## What This Changes

The branch changes Feishu monitor startup to fall back to the registered full channel runtime when an injected runtime lacks `inbound`, and adds `monitorSingleAccount` guard tests.

## Best Possible Solution

Land the narrow monitor runtime guard after redacted Feishu or gateway startup proof, or after an explicit maintainer proof override accepts the source-level evidence for this internal runtime path.

## Reproduction Assessment

## Solution Assessment

Yes for the known partial-runtime failure: guarding on `channelRuntime?.inbound` at the monitor consumption boundary matches the already-merged Feishu bot-path fix and avoids a broader SDK contract change. Merge readiness still depends on real behavior proof or maintainer proof override.
