## Summary

Keep open: the patch is a narrow source-level fix for a real Feishu partial-runtime failure and I found no code defects, but it still needs live Feishu/gateway proof or an explicit maintainer proof override before merge.

## What This Changes

The PR changes Feishu monitor startup to fall back to the registered full channel runtime when an injected runtime lacks `inbound`, and adds guard tests for partial, full, and undefined runtime inputs.

## Best Possible Solution

Land the monitor runtime guard after redacted Feishu/gateway startup proof or explicit maintainer proof override, and keep https://github.com/openclaw/openclaw/issues/92595 open until inbound processing is verified.

## Reproduction Assessment

Yes, from source inspection: current main can select a context-only `channelRuntime` at Feishu monitor startup, and `createFeishuMessageReceiveHandler` immediately dereferences `channelRuntime.debounce`. I did not run a live Feishu tenant or gateway startup.

## Solution Assessment

Yes for the known partial-runtime failure: the guard at monitor runtime selection matches the merged bot-path guard in https://github.com/openclaw/openclaw/pull/93466 and stays inside the Feishu plugin boundary. Merge readiness still depends on live/gateway proof or maintainer proof override.
