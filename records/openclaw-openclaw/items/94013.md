## What This Changes

The PR changes Feishu monitor startup to fall back to the registered full channel runtime when an injected `channelRuntime` lacks `inbound`, and adds `monitorSingleAccount` guard tests for partial, full, and undefined runtimes.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Yes for the known partial-runtime failure: checking `channelRuntime?.inbound` at the monitor consumption boundary matches the merged Feishu bot-path guard and avoids a broader SDK contract change for this narrow bug. Merge readiness still depends on proof or a maintainer proof override.
