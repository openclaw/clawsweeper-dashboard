## What This Changes

The PR adds session-header timestamp parsing and tests so session-memory filenames and headers use the session creation time instead of the `/new` or `/reset` event time.

## Best Possible Solution

Refresh this as a narrow timestamp-source fix against current main: choose the timestamp from the same transcript source used for content, format it through the current local-time helper, and preserve opt-in LLM slugging, collision suffixes, and background writes.

## Reproduction Assessment

Yes, by source inspection. Current main derives session-memory filenames and headers from `event.timestamp`, so a session created before a date boundary and archived after it can be stamped with the command date.

## Solution Assessment

No, not as-is. The requested direction is plausible, but this branch must be refreshed to preserve current local-time formatting, reset-fallback source selection, opt-in slugging, collision-safe filenames, and non-blocking command replies.

- **[P2] Preserve local-time formatting for selected timestamps:** `src/hooks/bundled/session-memory/handler.ts:327-333`
  - body: Current main and the docs use host local date/time for session-memory artifacts.   - confidence: 0.9
- **[P2] Tie reset timestamp fallback to the chosen transcript:** `src/hooks/bundled/session-memory/handler.ts:306-316`
  - body: The reset timestamp scan runs whenever the active file lacks a valid session header, even if that active transcript has content. That can stamp active content with an older reset sibling's date; choose the timestamp from the same transcript source used for content.
  - confidence: 0.87
- **[P2] Keep LLM slug generation opt-in:** `src/hooks/bundled/session-memory/handler.ts:360`
  - body: Current main and the docs only call the model when `llmSlug: true`. Changing this to `llmSlug !== false` makes ordinary enabled session-memory hooks call the provider during `/new` or `/reset`, adding unexpected latency and cost.
  - confidence: 0.9
- **[P2] Preserve collision-safe memory filenames:** `src/hooks/bundled/session-memory/handler.ts:378`
  - body: Current main adds a suffix when the target session-memory filename already exists. Building `${dateStr}-${slug}.md` directly can overwrite or lose one artifact when two captures share the same selected date and slug.
  - confidence: 0.86
- **[P2] Keep memory capture off the reply path:** `src/hooks/bundled/session-memory/handler.ts:224`
  - body: Current main starts session-memory capture in the background so `/new` and `/reset` replies are not delayed by transcript reads or optional slug generation. This branch awaits the work inside the hook handler, regressing that behavior.
  - confidence: 0.82
