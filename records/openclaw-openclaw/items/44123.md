## Summary

Keep open for maintainer judgment, but do not merge as-is: the requested session-creation-date behavior is not on current main, while this stale conflicting branch lacks real behavior proof and would regress current session-memory timestamp and reset-recovery contracts.

## What This Changes

This PR changes the bundled session-memory hook to derive memory file dates and header times from the session transcript header, with reset-file fallback tests, while keeping the HHMM slug based on the command event.

## Best Possible Solution

Define the intended timestamp source for session-memory artifacts, then port a narrow implementation onto the current handler while preserving local timezone formatting, collision suffixes, reset recovery, and off-reply-path writes.

## Reproduction Assessment

Yes, at source level. Current main derives the session-memory artifact date/header from event.timestamp, so a session created on one day and archived by /new on another is stamped with the command time; I did not run a live Day 1/Day 2 scenario because this review is read-only.

## Solution Assessment

No, not as-is. The idea may be useful, but the branch must be rebuilt around current session-memory code and maintainers still need to confirm whether session creation time should replace archive command time in the public artifact semantics.

Full review comments:

- **[P1] Preserve local-time session-memory timestamps:** `src/hooks/bundled/session-memory/handler.ts:327-333`
  - body: Current main formats session-memory filenames and headers in the gateway's local timezone. This branch switches the selected timestamp back through `toISOString()` and writes `UTC`, so merging it would regress the shipped local-time behavior; keep the existing `formatLocalSessionTimestamp` path when changing which instant is selected.
  - confidence: 0.9
- **[P2] Gate reset timestamp fallback to rotated transcripts:** `src/hooks/bundled/session-memory/handler.ts:306-324`
  - body: The `.reset.*` timestamp scan runs for every missing or invalid header timestamp, including non-empty active transcripts. If a legacy or corrupted active file has reset siblings, the memory file can be stamped with an older reset session instead of the current conversation time; reuse the same active-empty/rotated predicate as the content fallback.
  - confidence: 0.86
- **[P2] Avoid a second full transcript read:** `src/hooks/bundled/session-memory/handler.ts:148`
  - body: `getSessionCreatedAt` reads the whole transcript before `getRecentSessionContentWithResetFallback` reads it again. On large transcripts this doubles memory and I/O for every archive hook; read only the header line or carry the parsed timestamp out of the existing content read.
  - confidence: 0.78
