## Summary

Keep this PR open: current main still has the reported session-memory date behavior, but the branch is conflicting and the proposed patch has correctness issues around timezone preservation and reset transcript fallback.

## What This Changes

The PR changes the bundled session-memory hook so memory filenames and headers use the session file creation timestamp instead of the `/new` event timestamp, with focused regression tests.

## Best Possible Solution

Refresh the branch on current main, derive the selected session timestamp through the existing local-time formatter, align reset timestamp fallback with the transcript fallback predicate, and add proof for a real or clearly simulated cross-day `/new` capture.

## Reproduction Assessment

Yes, source-level reproduction is clear: create a transcript with a Day 1 session header, trigger `/new` on Day 2, and current main will name/header the memory file from the hook event timestamp. I did not run the path because this review is read-only.

## Solution Assessment

No, not as-is. The direction is maintainable, but the implementation needs to preserve current local-time formatting and avoid borrowing reset-file timestamps for non-rotated active transcripts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Preserve local-time formatting for session timestamps:** `src/hooks/bundled/session-memory/handler.ts:327-333`
  - body: Current main and the docs use the host local date/time for session-memory filenames and headers. This block formats the chosen timestamp with `toISOString()` and the header stays hard-coded to UTC, so a refreshed merge would undo the current local-time behavior for both session-created and fallback timestamps. Reuse the existing local timestamp formatter for whichever Date is selected.
  - confidence: 0.86
- **[P2] Only scan reset dates after a rotated primary:** `src/hooks/bundled/session-memory/handler.ts:306`
  - body: The reset timestamp fallback runs for every missing or invalid active session header, including non-empty legacy/corrupt active transcripts. In that case the content path still reads the active transcript, but the date can come from an older `.reset.*` sibling, stamping the current conversation with the wrong session date. Gate the reset-date fallback on the same empty/rotated condition used for transcript content fallback.
  - confidence: 0.82
