## Summary

Keep this PR open because current main still writes session-memory filenames and headers from the `/new` or `/reset` event timestamp, not the previous session's creation timestamp. The PR targets a real remaining behavior gap, but it is not merge-ready after current main's local-timezone fix and the added reset-file timestamp fallback still has a correctness hole for non-empty transcripts with invalid headers.

## What This Changes

The PR changes the bundled session-memory hook so memory filenames and headers derive their date/time from the previous session file’s session header timestamp, while retaining event time for the HHMM conflict suffix, and adds regression tests for fallback cases.

## Best Possible Solution

Refresh this PR or open a narrow replacement that reads the previous session creation instant through the transcript/session-memory helper layer, formats that instant with the current host-local timestamp helper, keeps event time only for the HHMM conflict slug, and adds regression coverage for normal headers, empty rotated transcripts, invalid non-empty headers with reset siblings, and local-timezone day boundaries.

## Reproduction Assessment

Yes. The PR body's Day 1 session plus Day 2 `/new` scenario maps directly to the current handler path where `dateStr` comes from `event.timestamp`; a focused test can create a session header timestamp and invoke the hook with a later event timestamp.

## Solution Assessment

No. The requested direction is valid, but the proposed patch should be revised to preserve current local-timezone behavior and to scan `.reset.*` timestamp fallbacks only when the active transcript is truly empty/rotated, ideally without reading the whole transcript twice.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Gate reset timestamp fallback on empty transcripts:** `src/hooks/bundled/session-memory/handler.ts:306-316`
  - body: The reset sibling scan runs for any missing or invalid session header. For a non-empty current transcript with a legacy/corrupt first line and an older `.reset.*` sibling, this stamps the current memory file with the reset file's older creation date while the content still comes from the active transcript. Match the content fallback predicate and only use reset timestamps when the active transcript is actually empty/rotated.
  - confidence: 0.88
- **[P2] Preserve local-time formatting for session timestamps:** `src/hooks/bundled/session-memory/handler.ts:327-333`
  - body: Current main intentionally formats session-memory filenames, fallback slugs, and headers in the host local timezone. This patch formats both session-created and fallback event timestamps with `toISOString()`, so a refreshed merge would bring back UTC day-boundary mismatches for this hook. Reuse the local timestamp formatter for whichever instant is selected.
  - confidence: 0.9
- **[P3] Avoid reading full transcripts twice:** `src/hooks/bundled/session-memory/handler.ts:148-149`
  - body: `getSessionCreatedAt` reads the entire transcript just to inspect the first line, then the handler reads the transcript again for message extraction. Large session files can make `/new` and `/reset` do double I/O and memory work; parse the header from a shared/streamed read or return metadata from the transcript helper.
  - confidence: 0.76
