## Summary

Keep this PR open, but not mergeable as-is: the reported timestamp-source problem is still present on current main by source inspection, while this branch regresses newer session-memory contracts and still lacks inspectable real behavior proof.

## What This Changes

The PR changes the bundled session-memory hook to derive memory filename and header dates from the session transcript creation timestamp, with added regression tests for missing, invalid, and reset-file timestamp cases.

## Best Possible Solution

Refresh this as a narrow timestamp-source fix against current main: choose the date/time from the same transcript source selected for content, format through the current local-time helper, keep explicit slug opt-in, preserve collision suffixing and background writes, and add focused regression coverage.

## Reproduction Assessment

Yes, by source inspection. Current main derives the session-memory artifact date/header from `event.timestamp`, so a session created before a date boundary and archived after it can be stamped with the command date.

## Solution Assessment

No, not as-is. The requested direction is plausible, but the current branch must preserve local-time formatting, reset-fallback source selection, explicit slug opt-in, collision-safe filenames, and non-blocking command replies.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve local-time formatting for selected timestamps:** `src/hooks/bundled/session-memory/handler.ts:327-333`
  - body: Current main and the public docs use host local date/time for session-memory artifacts. Formatting the selected session timestamp and fallback timestamp with `toISOString()` and writing `UTC` can still place artifacts on the wrong local day for non-UTC users; route the selected timestamp through the current local-time formatter.
  - confidence: 0.9
- **[P2] Gate reset timestamp fallback on the selected content source:** `src/hooks/bundled/session-memory/handler.ts:306-316`
  - body: The reset timestamp scan runs whenever the active file lacks a valid session header, even if that active transcript has content. That can stamp active content with an older reset sibling's date; choose the timestamp from the same transcript source used for content.
  - confidence: 0.86
- **[P2] Keep LLM slug generation opt-in:** `src/hooks/bundled/session-memory/handler.ts:360`
  - body: Current main and the docs only call the model when `llmSlug: true`. Changing this to `llmSlug !== false` makes ordinary enabled session-memory hooks call the provider during `/new` or `/reset`, adding unexpected latency and cost.
  - confidence: 0.9
- **[P2] Keep collision-safe memory filenames:** `src/hooks/bundled/session-memory/handler.ts:378`
  - body: Current main adds a suffix when the target session-memory filename already exists. Building `${dateStr}-${slug}.md` directly can overwrite or lose one artifact when two captures share the same selected date and HHMM or LLM slug.
  - confidence: 0.86
- **[P2] Keep memory capture off the command reply path:** `src/hooks/bundled/session-memory/handler.ts:224`
  - body: Current main starts session-memory capture in the background so `/new` and `/reset` replies are not delayed by transcript reads or slug generation. This head awaits that work inside the hook handler, regressing the documented non-blocking behavior.
  - confidence: 0.82
