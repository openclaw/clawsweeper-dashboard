Overall confidence: 0.91

Full review comments:

- **[P1] Keep memory capture off the command reply path:** `src/hooks/bundled/session-memory/handler.ts:224`
  - body: Current main starts session-memory persistence in the background so `/new` and `/reset` replies are not delayed by transcript reads or optional slug generation. This branch keeps the hook handler async and awaits the write path, so merging it would reintroduce command latency and provider-dependent stalls.
  - confidence: 0.9
- **[P1] Keep LLM slug generation opt-in:** `src/hooks/bundled/session-memory/handler.ts:360`
  - body: Current main and the docs only call the model when `llmSlug` is explicitly true. Changing this to `llmSlug !== false` makes ordinary enabled session-memory hooks call a provider during `/new` or `/reset`, adding unexpected latency, cost, and failure dependency.
  - confidence: 0.91
- **[P1] Preserve collision-safe memory filenames:** `src/hooks/bundled/session-memory/handler.ts:378`
  - body: Current main resolves an available filename and adds a numeric suffix when a session-memory artifact already exists. Building `${dateStr}-${slug}.md` directly can overwrite or lose one archive when two captures share the selected date and slug.
  - confidence: 0.88
- **[P1] Preserve local-time formatting for selected timestamps:** `src/hooks/bundled/session-memory/handler.ts:327-333`
  - body: Current main formats session-memory artifact names and headers in the host local timezone, and the docs describe that behavior. This branch formats both selected and fallback timestamps with `toISOString()`, so non-UTC users can get the wrong human-day artifact and lose the current timezone label.
  - confidence: 0.9
- **[P2] Use the same transcript source for timestamp fallback:** `src/hooks/bundled/session-memory/handler.ts:306-316`
  - body: The reset timestamp scan runs whenever the active file lacks a valid session header, even if that active transcript has content. If a reset sibling exists, the memory file can stamp active content with an older reset file's date; choose the timestamp from the same source used for captured content.
  - confidence: 0.87
