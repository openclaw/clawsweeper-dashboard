Refresh this as a narrow timestamp-source fix against current main: choose the timestamp from the same transcript source used for content, format it through the current local-time helper, preserve opt-in LLM slugging, collision suffixes, and background writes, then add focused regression coverage and real proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve local-time formatting for selected timestamps:** `src/hooks/bundled/session-memory/handler.ts:327-333`
  - body: Current main and the docs use host-local date/time for session-memory artifacts. Formatting both selected session timestamps and fallback timestamps with `toISOString()` and writing `UTC` can still put artifacts on the wrong local day for non-UTC users; route the selected timestamp through the current local-time formatter.
  - confidence: 0.9
- **[P2] Gate reset timestamp fallback on the content source:** `src/hooks/bundled/session-memory/handler.ts:306-316`
  - body: The reset timestamp scan runs whenever the active file lacks a valid session header, even if that active transcript has content. That can stamp active content with an older reset sibling's date; choose the timestamp from the same transcript source used for content.
  - confidence: 0.87
- **[P2] Keep LLM slug generation opt-in:** `src/hooks/bundled/session-memory/handler.ts:360`
  - body: Current main and the docs only call the model when `llmSlug: true`. Changing this to `llmSlug !== false` makes ordinary enabled session-memory hooks call the provider during `/new` or `/reset`, adding unexpected latency and cost.
  - confidence: 0.9
- **[P2] Preserve collision-safe memory filenames:** `src/hooks/bundled/session-memory/handler.ts:378`
  - body: Current main adds a suffix when the target session-memory filename already exists. Building `${dateStr}-${slug}.md` directly can overwrite or lose one artifact when two captures share the same selected date and slug.
  - confidence: 0.86
- **[P2] Keep memory capture off the command reply path:** `src/hooks/bundled/session-memory/handler.ts:224`
  - body: Current main starts session-memory capture in the background so `/new` and `/reset` replies are not delayed by transcript reads or optional slug generation. This branch awaits the work inside the hook handler, regressing that behavior.
  - confidence: 0.82
