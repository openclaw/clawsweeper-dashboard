## Summary

Keep open: current main still has the source-reproducible session-date bug, but this PR is not mergeable as-is because the stale branch would regress current session-memory contracts and still lacks inspectable real behavior proof.

## What This Changes

The PR changes session-memory handling so memory filenames and headers use the archived session creation timestamp instead of the `/new` event timestamp, with tests for missing or invalid headers and reset transcript cases.

## Best Possible Solution

Refresh the PR against current main so only the artifact timestamp source changes: select the session-created timestamp from the same transcript source used for content, format it with the existing local formatter, keep event-time HHMM collision suffixing and opt-in LLM slugs, and attach redacted real-run proof.

## Reproduction Assessment

Yes. Source inspection on current main shows `saveSessionMemoryNow` derives the filename date and header timestamp from `event.timestamp`, so a session started before midnight and archived by `/new` after midnight will be stamped with the command date.

## Solution Assessment

No, not as-is. The requested direction is valid, but this branch must preserve current main's local-time formatting, reset content-source gating, collision suffixing, and opt-in LLM slug behavior before it is a maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve local-time formatting for selected session timestamps:** `src/hooks/bundled/session-memory/handler.ts:327-333`
  - body: Current main and the public hook docs use the host-local date/time for memory filenames and headers. This formats the selected session timestamp with `toISOString()` and later appends `UTC`, so users outside UTC can get a memory artifact on the wrong local date; feed the chosen `Date` through the existing local formatter instead.
  - confidence: 0.9
- **[P2] Use reset timestamps only when reset content is selected:** `src/hooks/bundled/session-memory/handler.ts:306`
  - body: The reset scan runs whenever the active header is missing or invalid. If the active transcript still has usable messages and an older reset sibling exists, this can stamp active content with the reset file's creation date; timestamp selection should follow the same primary-empty predicate as content recovery.
  - confidence: 0.84
- **[P2] Keep LLM slug generation opt-in:** `src/hooks/bundled/session-memory/handler.ts:360`
  - body: Current main and the hook docs call the model only when `llmSlug` is explicitly `true`. This branch uses `hookConfig?.llmSlug !== false`, so enabling session-memory without an explicit slug setting will call the model provider on `/new` and `/reset`, slowing or failing a path that should use timestamp slugs by default.
  - confidence: 0.88
- **[P2] Keep collision-safe memory filenames:** `src/hooks/bundled/session-memory/handler.ts:378`
  - body: Current main resolves an unused filename and adds a numeric suffix for same-minute captures. This branch rebuilds `${dateStr}-${slug}.md` directly, so two sessions with the same selected date and HHMM slug can clobber or lose one of the memory artifacts instead of preserving both.
  - confidence: 0.83
