## Summary

Keep open: current main still stamps session-memory artifacts from hook dispatch time, and this PR's direction is useful but stale against the local-time and reset-transcript behavior now on main.

## What This Changes

The PR reads the previous session transcript header timestamp and uses it for session-memory filenames and headers while leaving the fallback HHMM slug on hook event time, with regression tests.

## Best Possible Solution

Refresh this as a narrow session-memory fix that selects the same transcript for content and header timestamp, formats the selected instant through the current local-time helper, keeps event time only for the conflict slug, and adds changelog/regression coverage.

## Reproduction Assessment

Yes by source inspection. Create a previous session JSONL with a Day 1 `type: "session"` header, dispatch `/new` on Day 2, and current main formats the memory filename/header from hook `event.timestamp`.

## Solution Assessment

No. The goal is valid, but the current PR uses UTC formatting and a reset timestamp fallback that is independent from the selected transcript content; the safer fix is to select one transcript source and format that source's header through `formatLocalSessionTimestamp`.

- **[P2] Preserve local-time formatting for selected timestamps:** `src/hooks/bundled/session-memory/handler.ts:329-333`
  - body: Current main intentionally formats session-memory filenames, fallback slugs, and markdown headers with the host local timezone. These selected timestamp strings are derived with `toISOString()`, and the later header is hardcoded as UTC, so users outside UTC would lose the shipped local-time behavior from #72408.
  - confidence: 0.9

  - body: The reset sibling scan runs whenever the active file has no valid header, before the code knows whether content will come from the active file or a reset fallback. If a non-empty active transcript has an invalid header and an older `.reset.*` sibling exists, the memory file will contain active content under the reset file's date.
  - confidence: 0.88

  - body: `getSessionCreatedAt` reads the entire transcript only to inspect the first line, and the handler later reads the transcript again for conversation extraction. Large transcripts make `/new` and `/reset` do avoidable duplicate full-file I/O; read the first line or return header metadata from the transcript helper.
  - confidence: 0.76
