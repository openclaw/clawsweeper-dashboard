## Summary

Keep open: current main still stamps session-memory files from the command timestamp, so the central fix is not implemented; this PR is useful but not mergeable as-is because it regresses host-local formatting/reset-source selection and lacks real behavior proof.

## What This Changes

The PR reads the session JSONL header timestamp for session-memory filename/header dates and adds tests for fallback and reset transcript cases.

## Best Possible Solution

Refresh the PR against current main, derive the memory artifact date/header from the archived session start in host-local time, choose reset timestamps only from the transcript source that supplies content, preserve collision suffix behavior, and attach redacted real-run proof.

## Reproduction Assessment

Yes. Source inspection on current main shows the filename date and header time derive from command `event.timestamp`, so a session started before midnight and archived by `/new` after midnight will be stamped with the command date.

## Solution Assessment

No, not as-is. The direction is reasonable, but the branch should preserve host-local formatting and make reset timestamp selection follow the same source as archived transcript content.
