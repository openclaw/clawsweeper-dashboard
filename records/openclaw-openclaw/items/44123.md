## Summary

Keep this PR open. Current main still uses the hook event timestamp for session-memory filenames and headers, but the PR is not merge-ready because it regresses current host-local timestamp formatting and has a reset-file timestamp fallback mismatch already identified in review comments.

## What This Changes

The PR changes the bundled session-memory hook to derive memory filenames and headers from the previous session transcript header timestamp, adds reset-file fallback handling, and adds regression tests.

## Best Possible Solution

Refresh this as a narrow session-memory fix that reads the previous transcript creation instant through the transcript layer, preserves host-local formatting for the selected instant, uses reset headers only when reset content is selected, keeps event time only for the conflict slug, and adds changelog/docs coverage.

## Reproduction Assessment

Yes. Create a previous session JSONL with a Day 1 session header, trigger the session-memory hook with a Day 2 `/new` event timestamp, and current main will name/header the memory file from Day 2.

## Solution Assessment

No. The proposed direction is right, but this patch is not the best implementation until it preserves current local-time behavior and makes timestamp fallback follow the transcript content fallback decision.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve host-local formatting for selected timestamps:** `src/hooks/bundled/session-memory/handler.ts:327-333`
  - body: Current main formats session-memory filenames, fallback slugs, and headers with the host local timezone. This patch formats both the session-created path and event fallback path with `toISOString()` and writes a `UTC` header, so rebasing it would regress the local-time behavior fixed by #72408.
  - confidence: 0.9
- **[P2] Gate reset timestamp fallback on rotated transcripts:** `src/hooks/bundled/session-memory/handler.ts:306-316`
  - body: The reset sibling scan runs whenever the header timestamp is missing or invalid. A non-empty active transcript with a corrupt or legacy header and an older reset sibling would keep active content but stamp it with the reset file's creation date; the timestamp fallback should mirror the content fallback predicate.
  - confidence: 0.88
- **[P3] Avoid a second full transcript read for the header:** `src/hooks/bundled/session-memory/handler.ts:146-149`
  - body: `getSessionCreatedAt` reads the whole transcript just to inspect the first line, and the handler then reads the transcript again for message extraction. Large session files make `/new` and `/reset` do avoidable duplicate I/O; return header metadata from the transcript helper or read only the first line.
  - confidence: 0.76
- **[P3] Add the required changelog entry:** `src/hooks/bundled/session-memory/handler.ts:327-330`
  - body: This changes user-visible memory filenames and headers, but the PR does not update `CHANGELOG.md`. Add a single-line Fixes entry with contributor credit before merge.
