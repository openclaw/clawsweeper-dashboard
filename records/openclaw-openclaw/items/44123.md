## Summary

Keep open: the reported session-memory timestamp bug is still valid on current main, but this PR needs changes before merge because it would lose current local-time behavior and can pair reset-file timestamps with the wrong transcript content.

## What This Changes

The branch reads a session transcript header timestamp and uses it for session-memory filenames and headers while keeping hook event time for fallback HHMM slugs, with added regression tests.

## Best Possible Solution

Update session-memory so the artifact date/header come from the same selected transcript's session header, format that instant through `formatLocalSessionTimestamp`, keep hook event time only for the fallback slug, and preserve collision-safe filename allocation.

## Reproduction Assessment

Yes by source inspection. Current main formats session-memory artifacts from hook dispatch time, while session transcript files already carry a creation timestamp in their first-line header.

## Solution Assessment

No. The direction is valid, but the patch should select one transcript source for both content and timestamp, use the current local-time formatter, and keep existing filename collision protection.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve local-time formatting for selected timestamps:** `src/hooks/bundled/session-memory/handler.ts:329-333`
  - body: Current main formats session-memory filenames and headers in the host local timezone. This branch derives the selected date/time with `toISOString()` and writes a hardcoded `UTC` header, so users outside UTC would lose the shipped local-time behavior when this is rebased.
  - confidence: 0.9
- **[P2] Tie reset timestamp fallback to the selected transcript:** `src/hooks/bundled/session-memory/handler.ts:306-316`
  - body: The reset sibling scan runs whenever the active file has no valid header, before the code knows whether content will come from the active file or a reset fallback. A nonempty active transcript with a legacy or corrupted header could be saved under an older reset file's date.
  - confidence: 0.88
- **[P3] Avoid rereading the full transcript for the header:** `src/hooks/bundled/session-memory/handler.ts:148-149`
  - body: `getSessionCreatedAt` reads the entire transcript only to inspect the first line, and the handler later reads transcript content again. Large transcripts make `/new` and `/reset` do avoidable duplicate full-file I/O; return header metadata from the transcript helper or read only the first line.
  - confidence: 0.76
