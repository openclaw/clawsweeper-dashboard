## Summary

Keep this PR open: current main still has the reported session-memory timestamp-source bug, but the branch is not merge-ready because it lacks inspectable real behavior proof and would regress several current main behaviors.

## What This Changes

The PR changes the bundled session-memory hook and tests so memory filenames and headers use the archived session header timestamp instead of the `/new` event timestamp.

## Best Possible Solution

Refresh the PR so only the timestamp source changes: select the session-created timestamp from the same transcript source used for content, then preserve current local formatting, filename collision handling, reset fallback semantics, and slug-generation defaults.

## Reproduction Assessment

Yes. Source inspection on current main shows the session-memory handler derives the filename date and header from `event.timestamp`, so a session created before midnight and archived by `/new` after midnight will be stamped with the command date.

## Solution Assessment

No, not as-is. The requested direction is valid, but this branch must preserve current main's local-time formatting, reset content-source selection, opt-in LLM slug behavior, and collision-safe filename resolution before it is a maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve local-time formatting for selected timestamps:** `src/hooks/bundled/session-memory/handler.ts:327-333`
  - body: Current main formats session-memory filenames and headers with the host-local formatter. This branch converts the selected session timestamp through `toISOString()` and writes `UTC`, so users outside UTC can still get an artifact on the wrong local date; pass the selected `Date` through the existing local formatter instead.
  - confidence: 0.9
- **[P2] Use reset timestamps only for selected reset content:** `src/hooks/bundled/session-memory/handler.ts:306`
  - body: The reset timestamp scan runs whenever the active header is missing or invalid. If the active transcript still has usable messages and an older reset sibling exists, the handler can stamp active content with the reset file's date; timestamp selection should follow the same primary-empty predicate as content recovery.
  - confidence: 0.84
- **[P2] Keep LLM slug generation opt-in:** `src/hooks/bundled/session-memory/handler.ts:360`
  - body: Current main only asks the model for a slug when `llmSlug` is explicitly true. This branch treats every value except `false` as enabled, so enabling session-memory without an explicit slug setting can call the model during `/new` and `/reset`; keep the existing opt-in condition.
  - confidence: 0.88
- **[P2] Keep collision-safe memory filenames:** `src/hooks/bundled/session-memory/handler.ts:378`
  - body: Current main resolves an unused memory filename and adds a numeric suffix for same-minute captures. This branch rebuilds the filename directly from the selected date and slug, so two sessions with the same selected date and HHMM slug can overwrite or lose one memory artifact.
  - confidence: 0.83
