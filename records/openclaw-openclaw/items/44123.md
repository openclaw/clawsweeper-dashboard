## Summary

Keep open: current main still does not implement session-creation-date memory filenames, but this PR is not mergeable as-is because it conflicts with later local-time behavior and lacks inspectable real behavior proof.

## What This Changes

The PR adds session JSONL header timestamp parsing and tests so session-memory filenames and headers use session creation time while keeping the HHMM slug tied to the command event timestamp.

## Best Possible Solution

Refresh this PR against current main, derive the archive timestamp from the transcript source actually saved, preserve host-local formatting and current filename collision behavior, and include redacted real-run proof.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: session-memory filenames and headers are stamped from `event.timestamp`, so a session created on one day and archived by `/new` or `/reset` on another is dated by the command time.

## Solution Assessment

No, not as-is. The direction is reasonable, but the patch should preserve local-time formatting and choose reset timestamps only from the same transcript source that provides the archived content.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Preserve local-time formatting for session timestamps:** `src/hooks/bundled/session-memory/handler.ts:327-333`
  - body: Current main and docs use the host local date/time for session-memory filenames and headers. Formatting the selected session timestamp and fallback event timestamp with `toISOString()` switches this path back to UTC, so sessions around timezone boundaries can get the wrong local date/header.
  - confidence: 0.86
- **[P2] Gate reset-date fallback on rotated content:** `src/hooks/bundled/session-memory/handler.ts:306`
  - body: The reset timestamp scan runs for any missing or invalid active session header. If the active transcript still has usable messages, this can save active content under an older reset sibling's creation date; timestamp fallback should use the same primary-first predicate as transcript content recovery.
  - confidence: 0.82
