## What This Changes

The branch changes bundled session-memory to derive memory file date/header text from the transcript session header timestamp, adds reset-file timestamp fallback lookup, and expands handler tests.

## Best Possible Solution

Make any creation-date behavior a current-main patch that preserves session-memory's local-time, collision, slugging, and async contracts, with focused tests for intact and rotated transcripts.

## Reproduction Assessment

Yes. Source inspection is enough to confirm the reported behavior: current main derives the archive filename/header date from event.timestamp, so a session created on one day and archived by /new on another is stamped with the command time. I did not run tests because this review is read-only.

## Solution Assessment

No, not as-is. The direction may be useful, but this PR must be rebuilt on current session-memory code and maintainers still need to choose whether session creation time or archive time owns the public date semantics.

Full review comments:

- **[P1] Preserve local-time memory timestamps:** `src/hooks/bundled/session-memory/handler.ts:329-333`
  - body: Current main and docs say session-memory filenames and headers use the host local date/time. This branch formats the selected timestamp with `toISOString()` and hardcodes `UTC`, so users outside UTC would lose the merged local-time behavior and can see archives land on the wrong local day.
  - confidence: 0.92
- **[P1] Keep slug generation off the command path:** `src/hooks/bundled/session-memory/handler.ts:360`
  - body: Current main only calls the model when `llmSlug === true` and schedules memory capture in the background. This branch treats LLM slugs as enabled unless explicitly false and awaits the work from the command handler, so `/new` and `/reset` can again block on transcript reads or model calls.
  - confidence: 0.9
- **[P1] Use the collision-safe filename resolver:** `src/hooks/bundled/session-memory/handler.ts:378`
  - body: Current main resolves `YYYY-MM-DD-slug.md` collisions by adding a suffix. This branch writes the literal `${dateStr}-${slug}.md`, so repeated captures with the same slug on the same date can replace or fail to preserve earlier memory archives.
  - confidence: 0.88
- **[P2] Gate reset timestamp fallback on rotated transcripts:** `src/hooks/bundled/session-memory/handler.ts:306-316`
  - body: The reset-file timestamp fallback runs whenever the primary file lacks a valid session timestamp, including non-empty legacy or corrupted headers. If stale reset siblings exist, the archive can be stamped with an older reset file's session date instead of the current event fallback.
  - confidence: 0.82
