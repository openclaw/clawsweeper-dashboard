Full review comments:

- **[P2] Preserve local-time formatting for selected timestamps:** `src/hooks/bundled/session-memory/handler.ts:327-333`
  - body: Current main formats session-memory artifact names and headers in the host local timezone. This branch formats selected and fallback timestamps with toISOString() and writes a UTC header, so non-UTC users can still get the wrong human-day artifact and lose the current documented behavior.
  - confidence: 0.9
- **[P2] Use the same transcript source for timestamp fallback:** `src/hooks/bundled/session-memory/handler.ts:306-316`
  - body: The reset timestamp scan runs whenever the active file lacks a valid session header, even when that active transcript has content. If a reset sibling exists, the memory file can stamp active content with an older reset file's date; choose the timestamp from the same source used for captured content.
  - confidence: 0.87
- **[P1] Keep LLM slug generation opt-in:** `src/hooks/bundled/session-memory/handler.ts:360`
  - body: Current main and the docs only call the model when llmSlug is true. Changing this to llmSlug !== false makes ordinary enabled session-memory hooks call a provider during /new or /reset, adding unexpected latency, cost, and provider dependency.
  - confidence: 0.9
- **[P1] Preserve collision-safe memory filenames:** `src/hooks/bundled/session-memory/handler.ts:378`
  - body: Current main adds a numeric suffix when the target session-memory filename already exists. Building `${dateStr}-${slug}.md` directly can overwrite or lose one artifact when two captures share the same selected date and slug.
  - confidence: 0.86
- **[P1] Keep memory capture off the command reply path:** `src/hooks/bundled/session-memory/handler.ts:224`
  - body: Current main starts session-memory capture in the background so /new and /reset replies are not delayed by transcript reads or optional slug generation. This branch awaits the full write path inside the hook handler, regressing that responsiveness contract.
  - confidence: 0.84
