## Summary

Keep open: current main still has the reported command-time stamping behavior, but this PR is not merge-ready because it lacks inspectable real behavior proof, is stale/conflicting, and regresses current local-time formatting, reset-fallback semantics, opt-in slug generation, and collision-safe filenames.

## What This Changes

The PR changes the bundled session-memory hook and tests so memory filename/header dates can come from a session transcript header timestamp instead of the `/new` or `/reset` event timestamp.

## Best Possible Solution

Refresh this as a narrow session timestamp-source fix: choose the timestamp from the same transcript source used for content, then preserve current local-time formatting, opt-in LLM slugs, collision-safe filename resolution, and focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows the filename date and header are derived from `event.timestamp`, so a session created before midnight and archived by `/new` after midnight will be stamped with the command date.

## Solution Assessment

No, not as-is. The requested direction is valid, but the branch needs to preserve current main's local-time, reset-fallback, slug opt-in, and filename collision contracts before it is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve local-time formatting for selected timestamps:** `src/hooks/bundled/session-memory/handler.ts:327-333`
  - body: Current main formats session-memory filenames and headers through the host-local formatter. This branch converts the selected session timestamp through `toISOString()` and writes `UTC`, so users outside UTC can still get an artifact on the wrong local date; pass the selected `Date` through the existing local formatter instead.
  - confidence: 0.9
- **[P2] Use reset timestamps only for selected reset content:** `src/hooks/bundled/session-memory/handler.ts:306-316`
  - body: The reset timestamp scan runs whenever the active header is missing or invalid. If the active transcript still has usable messages and an older reset sibling exists, the handler can stamp active content with the reset file's date; timestamp selection should follow the same primary-empty predicate as content recovery.
  - confidence: 0.84
- **[P2] Keep LLM slug generation opt-in:** `src/hooks/bundled/session-memory/handler.ts:360`
  - body: Current main only asks the model for a slug when `llmSlug` is explicitly true. This branch treats every value except `false` as enabled, so enabling session-memory without an explicit slug setting can call the model during `/new` and `/reset`; keep the existing opt-in condition.
  - confidence: 0.88
- **[P2] Keep collision-safe memory filenames:** `src/hooks/bundled/session-memory/handler.ts:378`
  - body: Current main resolves an unused memory filename and adds a numeric suffix for same-minute captures. This branch rebuilds the filename directly from the selected date and slug, so two sessions with the same selected date and HHMM slug can overwrite or lose one memory artifact.
  - confidence: 0.83
