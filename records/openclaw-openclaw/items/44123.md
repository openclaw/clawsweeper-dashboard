## What This Changes

The PR adds session transcript header timestamp parsing and tests so session-memory filenames and headers use the session creation timestamp instead of the `/new` or `/reset` event timestamp.

## Best Possible Solution

Implement this as a current-main timestamp-source fix: choose the creation timestamp from the same transcript source used for content, format through the local-time helper, preserve existing filename/provider/async contracts, and cover cross-day plus rotated-transcript cases.

## Reproduction Assessment

Yes, by source inspection. Current main builds session-memory artifact dates and headers from the command event timestamp, so a session created before a day boundary and archived after it can be stamped with the archive command date.

Full review comments:

- **[P2] Preserve local-time formatting for selected timestamps:** `src/hooks/bundled/session-memory/handler.ts:327-333`
  - body: Current main and the public docs use host local date/time for session-memory artifacts. Formatting the selected session timestamp with `toISOString()` and appending `UTC` can still place artifacts on the wrong local day for non-UTC users; route the selected timestamp through the current local-time formatter.
  - confidence: 0.9
- **[P2] Tie reset timestamp fallback to the chosen transcript source:** `src/hooks/bundled/session-memory/handler.ts:306-316`
  - body: The reset timestamp scan runs whenever the active file lacks a valid session header, even if that active transcript has content. That can stamp active content with an older reset sibling’s date; choose the timestamp from the same transcript source used for content.
  - confidence: 0.87
- **[P2] Keep model-generated slugs opt-in:** `src/hooks/bundled/session-memory/handler.ts:360`
  - body: Current main and the docs only call the model when `llmSlug: true`. Changing this to `llmSlug !== false` makes ordinary enabled session-memory hooks call the provider during `/new` or `/reset`, adding unexpected latency and cost.
  - confidence: 0.9
- **[P2] Preserve collision-safe memory filenames:** `src/hooks/bundled/session-memory/handler.ts:378`
  - body: Current main adds a suffix when the target session-memory filename already exists. Building `${dateStr}-${slug}.md` directly can overwrite or lose one artifact when two captures share the same selected date and slug.
  - confidence: 0.86
- **[P2] Keep session-memory writes off the command reply path:** `src/hooks/bundled/session-memory/handler.ts:224`
  - body: Current main starts session-memory capture in the background so `/new` and `/reset` replies are not delayed by transcript reads or optional slug generation. This branch awaits the work inside the hook handler, regressing that behavior.
  - confidence: 0.82
