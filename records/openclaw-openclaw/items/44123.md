## Summary

Keep open: current main still stamps session-memory files from the `/new` or `/reset` command timestamp, so the reported chronology problem remains, but this PR is not mergeable as-is because it regresses current local-time, reset-source, opt-in slug, and collision-safe filename contracts and still lacks real behavior proof.

## What This Changes

The PR adds session transcript header timestamp parsing and tests so session-memory filename/header dates can use session creation time, including a reset-file timestamp fallback.

## Best Possible Solution

Refresh this as a narrow timestamp-source fix that chooses the timestamp from the same transcript source used for content while preserving host-local formatting, explicit `llmSlug: true`, collision-safe filename suffixing, and focused regression coverage.

## Reproduction Assessment

Yes, by source inspection. Current main formats the session-memory filename and header from `event.timestamp`, so a session created before midnight and archived after midnight will be stamped with the command date rather than the session date.

## Solution Assessment

No, not as-is. The requested direction is valid, but this branch must preserve the current local-time, reset-fallback, slug opt-in, and filename collision contracts before it is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve local-time formatting for selected timestamps:** `src/hooks/bundled/session-memory/handler.ts:327-333`
  - body: Current main formats session-memory filenames and headers through `formatLocalSessionTimestamp`, and the docs say the hook uses the host local date. This branch formats both the selected session timestamp and fallback timestamp with `toISOString()` and writes `UTC`, so users outside UTC can still get artifacts on the wrong local date; pass the chosen `Date` through the existing local formatter instead.
  - confidence: 0.9
- **[P2] Gate reset timestamp fallback on selected content source:** `src/hooks/bundled/session-memory/handler.ts:306-316`
  - body: The branch scans `.reset.*` timestamps whenever the active file lacks a valid session header, but content loading still keeps a non-empty active transcript. A legacy or invalid-header active transcript with an older reset sibling would be stamped with the reset date while saving active content; choose the timestamp from the same transcript source used for content.
  - confidence: 0.84
- **[P2] Keep LLM slug generation opt-in:** `src/hooks/bundled/session-memory/handler.ts:360`
  - body: Current main and docs only call the model when `llmSlug: true`. Changing the condition to `llmSlug !== false` makes ordinary enabled session-memory hooks call the model during `/new` and `/reset`, adding unexpected latency and provider cost; preserve the explicit true check.
  - confidence: 0.88
- **[P2] Keep collision-safe memory filenames:** `src/hooks/bundled/session-memory/handler.ts:378`
  - body: Current main uses `resolveAvailableMemoryFilename` to add a suffix for same-minute captures. This branch constructs `${dateStr}-${slug}.md` directly, so two sessions with the same selected date and HHMM or LLM slug can overwrite or lose one memory artifact.
  - confidence: 0.83
