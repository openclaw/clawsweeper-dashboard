## Summary

Keep open. Current main still derives session-memory filenames and headers from the `/new` or `/reset` event timestamp, but this PR is not mergeable as-is because it would regress current local-time formatting, can choose a reset timestamp for active transcript content, and still lacks inspectable real behavior proof.

## What This Changes

The PR adds session JSONL header timestamp parsing and regression tests so session-memory filenames and headers use session creation time rather than the `/new` or `/reset` event time.

## Best Possible Solution

Refresh on current main, derive the archive timestamp from the transcript source actually saved, format it through `formatLocalSessionTimestamp`, preserve current collision/background/`llmSlug` behavior, and add redacted real-run proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main stamps session-memory artifacts from the command event timestamp, so a transcript created on one day and archived by `/new` or `/reset` on another is named and headed for the command time; I did not execute it because this review is read-only.

## Solution Assessment

No, not as-is. The direction is reasonable, but the patch should preserve current local-time formatting and choose reset timestamps only from the same transcript source that provides the archived content.
