## What This Changes

The PR adds numeric errno collection plus EADDRNOTAVAIL fail-fast tests so Telegram transport fallback skips IP rotation for local socket allocation failures.

## Best Possible Solution

Preserve remote-reachability fallback while narrowly failing fast for local socket allocation errors, then land one candidate after real transport proof or maintainer override.

## Solution Assessment

No: the owner module is the right layer, but collecting all numeric errnos as raw strings is too broad for the existing symbolic fallback predicate. The safer fix is to map numeric errnos intentionally or narrowly detect local socket allocation failures while preserving remote-reachability fallback.
