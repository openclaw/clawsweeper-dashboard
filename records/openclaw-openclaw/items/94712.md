## What This Changes

The PR changes Telegram fetch fallback classification so EADDRNOTAVAIL skips IP-rotation fallback and adds mocked regression tests for that behavior.

## Best Possible Solution

Preserve fallback for remote-reachability errors while suppressing local socket allocation failures, then land one reviewed candidate with real transport proof or maintainer override.

## Reproduction Assessment

Yes for source-level review: current main retries code-less `fetch failed` envelopes, while this PR changes numeric-only errno classification. I did not establish a live Telegram outage reproduction in this read-only review.

## Solution Assessment

No: the owner module is the right layer, but collecting all numeric errnos as raw strings is too broad for the existing fallback predicate. The safer fix is to map numeric errnos intentionally or narrowly detect local socket allocation failures while preserving remote-reachability fallback.
