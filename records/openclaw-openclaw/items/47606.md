## Summary

Keep open. Current main has task records, Task Flow state, task audits, ACP no-output notices, and tool-call loop detection, but it still does not implement the requested artifact-gated execution guard system with durable guard state, repeated-status blocking, escalation timers, no-idle-after-green handling, rollout flag, or structured guard telemetry.

## What This Changes

The issue requests runtime execution anti-drift guardrails for artifact-gated status updates, escalation timers, no-idle-after-green handling, durable state, feature flagging, and telemetry.

## Best Possible Solution

Define an execution-guard contract first, then implement it across task/run state, status-delivery surfaces, timer/cooldown handling, telemetry, config/docs, and focused ACP/subagent/CLI/task-flow tests.

## Reproduction Assessment

No, not as an end-to-end runtime bug: the report lacks a transcript or command sequence that reproduces a status loop on current main. Reproduction is not applicable to the feature request, though source inspection verifies the requested guard system is absent.

## Solution Assessment

No, the adjacent mechanisms are not the best complete solution because they observe or warn about narrower conditions. The maintainable path is an explicit guard contract with durable state, artifact classification, timers, telemetry, and rollout semantics.
