## Summary

Keep open: current main has adjacent task-ledger persistence, channel run heartbeats, status-reaction stall timers, CLI no-output watchdogs, and cron interim-ack retry behavior, but it still does not implement the requested cross-runtime artifact-gated status contract, persisted artifact/escalation state, escalation timers, or post-green idle enforcement.

## What This Changes

Add runtime anti-drift guardrails that persist run state and gate status updates, escalation, and post-green idle handling on concrete execution artifacts.

## Best Possible Solution

Define a versioned progress-fact contract first, then implement persisted guard state at the common task/runtime boundary with narrow adapters and tests for CLI, cron, ACP/subagent, and embedded-runner paths before enforcing thresholds.
