## Summary

Keep open. Current main has adjacent task ledgers, task audits, ACP no-output notices, status delivery, and tool-call loop detection, but it still lacks the requested durable artifact-gated execution guard system with repeated-status blocking, escalation timers, no-idle-after-green enforcement, rollout flag, and structured guard telemetry.

## What This Changes

The issue asks for runtime execution anti-drift guardrails that persist per-run state, gate progress messages on new artifacts, detect status loops, escalate stalls, prevent idle gaps after green verification, and emit telemetry behind `ENFORCEMENT_GUARDS_V1`.

## Best Possible Solution

Define a generic execution-guard contract first, then implement durable guard state and enforcement across task/run state plus ACP, subagent, CLI, and notification/status delivery surfaces with docs and targeted tests.
