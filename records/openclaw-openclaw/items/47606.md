## Summary

Keep open. Current main has adjacent task records, task audit checks, ACP no-output notices, status reactions, and tool-call loop detection, but it still lacks the requested artifact-gated execution guard system with durable guard state, repeated-status blocking, escalation timers, no-idle-after-green handling, rollout flag, and structured guard telemetry.

## What This Changes

Add enforceable runtime execution anti-drift guards for artifact-gated status updates, stall escalation, durable run state, and post-green follow-through.

## Best Possible Solution

Define an execution-guard contract first, then implement it across task/run state and status-delivery surfaces with durable guard state, explicit artifact classification, timer/cooldown semantics, structured events, docs/config alignment, and focused tests for ACP, subagent, CLI, and task notification paths.

## Reproduction Assessment

No, not as an end-to-end runtime bug: the report lacks a concrete transcript or command sequence that reproduces a status loop. Source inspection does verify that the requested guard system is not present on current main.

## Solution Assessment

No, the existing adjacent mechanisms are not a complete solution. Task audits, ACP no-output notices, status reactions, prompt guidance, and tool-loop detection do not enforce artifact-gated status updates or post-green/no-phase-change escalation.
