## What This Changes

The issue requests runtime execution anti-drift guardrails for multi-batch agent sessions, including artifact-gated status updates, durable per-run state, timer-based escalation, no-idle-after-green enforcement, a feature flag, and telemetry.

## Best Possible Solution

Define the guardrail as a product-owned execution reliability contract, use the existing diagnostics/session-state architecture with SQLite-backed durable state where persistence is required, and land it with focused proof for status-loop, timer escalation, restart resume, and post-green idle behavior.
