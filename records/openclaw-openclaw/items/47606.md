## Summary

Keep open: current main has adjacent task persistence, strict-agentic plan-only retry handling, and cron phase watchdogs, but not the requested cross-runtime artifact-gated status contract, persisted guard/escalation state, escalation timers, or no-idle-after-green enforcement. Related cleanup closed https://github.com/openclaw/openclaw/issues/82430 back into this issue, while https://github.com/openclaw/openclaw/issues/56349 tracks a different outbound pre-send boundary.

## What This Changes

Request runtime execution guardrails that persist per-run state, require artifacts for nonterminal status updates, circuit-break status loops, escalate stalls, and enforce post-green follow-through behind a feature flag with telemetry.

## Best Possible Solution

Define a shared progress-fact and guard-state contract at the task/runtime boundary, persist it with task/flow state, and enforce artifact-gated status, timers, no-idle-after-green, and telemetry behind an explicit rollout flag across all execution paths.
