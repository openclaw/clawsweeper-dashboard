## Summary

Keep open: current main has adjacent task persistence, task-flow state, cron watchdogs, stale-ack mitigation, and outbound hook surfaces, but it does not implement the requested cross-runtime artifact-gated status contract, persisted guard/escalation state, escalation timers, or no-idle-after-green enforcement. Related cleanup routed the no-progress/status-loop portion back to this issue, while the broader pre-send-policy work in https://github.com/openclaw/openclaw/issues/56349 does not cover the run-state/timer requirements here.

## What This Changes

Request runtime execution guardrails that persist run state, gate status updates on new artifacts, detect no-progress loops, enforce escalation/no-idle-after-green timers, and emit telemetry behind a feature flag.

## Best Possible Solution

Define a shared progress-fact and guard-state contract at the task/runtime boundary, persist it with task/flow state, then enforce artifact-gated status, escalation timers, and no-idle-after-green behavior across CLI, cron, ACP/subagent, and embedded-runner paths behind an explicit rollout flag with telemetry.
