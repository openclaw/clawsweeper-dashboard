## Summary

Keep open. Current main has adjacent task records, task audits, ACP no-output notices, status reactions, and tool-call loop detection, but it still does not implement the requested artifact-gated execution guard system with durable guard state, repeated-status blocking, escalation timers, no-idle-after-green policy, rollout flag, or structured guard telemetry.

## What This Changes

The issue requests enforceable runtime anti-drift guards for agent execution: durable run state, artifact-gated status updates, loop circuit breaking, escalation timers, no-idle-after-green handling, a feature flag, and telemetry.

## Best Possible Solution

Define an execution-guard contract first, then implement it across task/run state and status-delivery surfaces with durable guard state, explicit artifact classification, timer and cooldown semantics, structured guard events, config/docs alignment, and targeted tests for ACP, subagent, CLI, and task-notification paths.

## Reproduction Assessment

No, not as an end-to-end runtime bug: the report lacks a concrete transcript or command sequence that reproduces a status loop. The missing guard system is still verifiable by source, docs, and test inspection on current main.

## Solution Assessment

No, the existing adjacent mechanisms are not the best complete solution. Task audits, ACP no-output notices, status reactions, prompt guidance, and tool-loop detection do not enforce artifact-gated status updates or post-green/no-phase-change escalation.
