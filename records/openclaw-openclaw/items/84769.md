## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Keep Gemini workspace trust in the backend args:** `extensions/google/cli-backend.ts:31`
  - body: The new argv drops `--skip-trust` and only passes `--approval-mode yolo`. Upstream Gemini CLI uses `--skip-trust` to trust the workspace, then forces non-default approval modes back to `default` when the folder is not trusted, so OpenClaw's non-interactive backend can start prompting or blocking in workspaces that worked on main. Preserve `--skip-trust` in both fresh and resume args, adding approval mode only if needed.
  - confidence: 0.88
- **[P1] Use fake-IP SSRF exemptions instead of private-network allow:** `extensions/google/oauth.http.ts:9`
  - body: This unconditionally allows private/internal DNS answers for Google OAuth helper requests. The repo's fake-IP contract uses host-scoped `allowRfc2544BenchmarkRange` and `allowIpv6UniqueLocalRange` without `allowPrivateNetwork`, so this broadens an OAuth-token security boundary beyond the Clash/Surge fake-IP case the docs describe.
  - confidence: 0.84
- **[P2] Wire Gemini into the persisted-sync candidate gate:** `src/agents/auth-profiles/external-cli-sync.ts:129-134`
  - body: Adding a persisted Gemini provider here is not enough for normal unscoped persisted refreshes: `syncPersistedExternalCliAuthProfiles` returns before resolving providers unless explicit scope is supplied or the store already has one of the hard-coded Claude/MiniMax profile ids. Add Gemini to that candidate gate and cover the stale `google-gemini-cli:default` refresh path.
  - confidence: 0.78
