## Summary

## What This Changes

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

No as written: this is a plausible mitigation, but replacing all plugin providers with an empty list is too broad and leaves the sibling capability registry unresolved. A safer solution isolates the failing plugin or resolver failure while preserving unaffected provider registrations.
