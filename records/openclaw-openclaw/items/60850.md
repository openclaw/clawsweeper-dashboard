## Summary

Keep open. Current main still uses the low-alpha global selection styling in the Control UI/WebChat browser surface, and the linked PR that uses closing syntax for this issue remains open and unmerged as the active implementation candidate.

## What This Changes

The issue requests a higher-contrast or configurable WebChat/Control UI text-selection highlight because the current highlight is difficult for color-deficient users to see.

## Best Possible Solution

Land PR #60854, or an equivalent focused change in `ui/src/styles/base.css`, so browser text selection uses a clearly visible background/text token pair across the built-in themes and has targeted visual or CSS contract proof.

## Solution Assessment
