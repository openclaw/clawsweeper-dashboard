## Summary

Keep open. Current main and the latest release still use the low-alpha global selection background in the Control UI/WebChat browser surface, and PR #60854 remains the open unmerged implementation candidate with closing syntax for this issue.

## What This Changes

The issue requests a higher-contrast or configurable WebChat browser text-selection highlight for color-deficient users.

## Best Possible Solution

Land PR #60854, or an equivalent focused change in `ui/src/styles/base.css`, so browser text selection uses a clearly visible background/text token pair across built-in themes.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: selecting text in the Control UI chat tab uses the global `::selection` rule from `ui/src/styles/base.css`, which still points at the low-alpha `--accent-subtle` token.

## Solution Assessment
