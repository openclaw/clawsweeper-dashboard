## Summary

Keep open. Current main and the latest release still have the unguarded plugin descriptor-cache read path, and this PR is a focused, apparently correct maintainer bugfix with regression coverage; it should stay open for maintainer landing-order review because it is draft, protected by the `maintainer` label, and overlaps other open descriptor-hardening drafts.

## What This Changes

The PR makes plugin tool descriptor-cache capture return no cached descriptor when plugin-owned descriptor fields throw, skips caching incomplete descriptor sets, and adds focused cache/resolver regression tests.

## Best Possible Solution

Land one maintainer-selected plugin descriptor hardening path: either merge this narrow cache-capture fix or fold it into the broader resolver/descriptor-cache work in https://github.com/openclaw/openclaw/pull/89384 after maintainers choose the canonical scope.

## Reproduction Assessment

Yes, source-reproducible. Current main and v2026.6.9 read a plugin tool schema once during validation and again during descriptor-cache capture, so a getter that throws on the later read can abort resolution before the live tool is returned.

## Solution Assessment

Yes for the narrow cache-capture crash. The patch is a bounded mitigation in the owner module, but maintainers still need to decide whether this branch or a broader descriptor-hardening draft is the best canonical merge path.
