## Summary

Keep open: this is a focused bug-fix attempt, but source review shows the moved linger helper is still behind the Linux systemd availability gate, and the helper also returns on the same false availability result, so it does not address the reported no-user-bus/headless path. The PR also lacks after-fix real headless-server proof.

## What This Changes

The PR moves the non-interactive systemd linger helper to run before `service.install()` in the gateway daemon installer and adds an explanatory comment.

## Best Possible Solution

Refactor the non-interactive Linux install path so true systemd absence is separated from disabled or unreachable user linger, handle the safe linger remediation before the skip/install decision, add focused regression coverage, and verify on a fresh headless Ubuntu host.

## Reproduction Assessment

Yes from source for the PR defect: a false Linux systemd availability result returns before the moved helper, and the helper has the same false-return guard. I did not run a live headless Ubuntu smoke because this is a read-only review and the PR lacks after-fix runtime proof.

## Solution Assessment

No. Reordering the existing helper below the availability gate is not sufficient for the reported no-user-bus path; the installer needs a pre-gate linger/status path or a refined availability contract.
