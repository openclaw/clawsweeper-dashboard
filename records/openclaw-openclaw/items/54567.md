## Summary

Keep open: the branch still misses the reported headless no-user-bus path because the install flow and linger helper both gate on `isSystemdUserServiceAvailable()` before attempting linger, and the user-facing fix also needs a changelog entry.

## What This Changes

The PR moves `ensureSystemdUserLingerNonInteractive` before `service.install()` in the non-interactive daemon install path and adds an explanatory comment.

## Best Possible Solution

Make Linux non-interactive daemon install distinguish missing systemd from disabled or unreachable user linger, attempt the existing linger repair before declaring user systemd unavailable, and lock that sequence with focused regression tests plus a changelog entry.

## Reproduction Assessment

Yes for the PR defect: current source and tests show `Failed to connect to bus` makes the availability probe return false, and the moved linger call remains unreachable on that path. The original Ubuntu report has concrete steps, but this review did not run a live headless smoke.

## Solution Assessment

No. Moving the helper only above `service.install()` is not sufficient while both the caller and helper still return before linger on the unavailable user-bus path; the safer fix is to split or reorder the availability and linger checks with regression coverage.
