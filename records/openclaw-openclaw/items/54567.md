## Summary

Keep open: the branch is still not correct for the headless no-user-bus path because the moved linger call remains below the Linux user-systemd availability return and the helper self-returns on that same false probe.

## What This Changes

The PR moves `ensureSystemdUserLingerNonInteractive` before `service.install()` in `installGatewayDaemonNonInteractive` and adds a short explanatory comment.

## Best Possible Solution

Use a Linux-only daemon install path that distinguishes missing systemd from disabled linger and proves the sequencing with focused tests, or close this PR as obsolete if maintainers confirm the shipped user-bus and missing-unit fixes fully cover the report.

## Reproduction Assessment

Unclear for the live Ubuntu failure: the PR gives concrete steps but no before/after smoke log. The patch defect is source-reproducible because the new call remains unreachable whenever `isSystemdUserServiceAvailable()` returns false.

## Solution Assessment

No. Moving the helper only below the existing availability gate is not the narrowest reliable fix for a no-user-manager headless install; the safer repair is to distinguish unavailable systemd from disabled linger and handle linger before that gate.
