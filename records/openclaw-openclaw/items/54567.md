## Summary

Keep open. Current main still does not contain the PR’s intended reorder, and the PR needs maintainer follow-up because the new call is still below the user-systemd availability gate, which can skip the no-user-manager headless case described by the PR.

## What This Changes

The PR moves the existing non-interactive systemd linger helper earlier in `installGatewayDaemonNonInteractive`, before `service.install()`, and adds a short explanatory comment.

## Best Possible Solution

The desired end state is a non-interactive Linux daemon install path that either enables/checks linger before any user-bus availability gate, or has a documented reason that current systemd bus fallback and reload retry make this reorder unnecessary. If the fix is kept, add a narrow regression test for the sequencing or attach live headless Ubuntu proof.

## Reproduction Assessment

Unclear for the exact Ubuntu failure: the PR provides concrete Ubuntu 24.04 headless steps, but no live before/after proof is present. Source tracing does give a high-confidence static check for the relevant order and activation path on current main.

## Solution Assessment

No, not as written. Moving the helper before `service.install()` is directionally small, but placing it after the existing availability return path, while the helper itself also checks availability, is not the narrowest reliable fix for a no-user-manager headless install.
