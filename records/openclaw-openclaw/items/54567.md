## Summary

Keep open: the proposed reordering does not fix the reported headless path because it still runs after the user-systemd availability skip and calls a helper that returns on the same unavailable probe. The PR also lacks real after-fix headless-server proof.

## What This Changes

The PR moves `ensureSystemdUserLingerNonInteractive` before `service.install()` in the non-interactive daemon install flow and adds an explanatory comment.

## Best Possible Solution

Refactor the Linux daemon install/linger flow so disabled or unreachable linger is handled before treating user systemd as unavailable, while preserving true no-systemd skips and adding focused coverage plus redacted live headless Ubuntu output.

## Reproduction Assessment

Yes for the patch defect from source: the installer and the moved helper both gate on `isSystemdUserServiceAvailable()`, and tests classify failed user-bus access as unavailable. The intended after-fix behavior is not proven with a live headless-server run.

## Solution Assessment

No. Reordering the existing helper below the availability gate is not sufficient; the fix needs a pre-gate linger/status path or a helper split that can distinguish true systemd absence from disabled or unreachable linger.
