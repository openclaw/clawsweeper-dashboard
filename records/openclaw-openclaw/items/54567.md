## Summary

Keep open: this PR targets a real Linux daemon-install pain point, but the patch does not cover the no-user-bus path it describes because the moved linger call remains behind the current systemd availability gate and the helper has the same self-gate.

## What This Changes

The branch moves `ensureSystemdUserLingerNonInteractive` before `service.install()` in the non-interactive daemon install path and adds an explanatory comment.

## Best Possible Solution

Update the daemon install flow so Linux non-interactive setup can distinguish missing systemd from disabled/unreachable user linger, attempt the existing linger repair before the unavailable-systemd skip when appropriate, and cover that sequence with focused regression tests and a changelog entry.

## Reproduction Assessment

Yes for the patch defect: current source and tests show failed user-bus access makes `isSystemdUserServiceAvailable()` return false, and daemon install returns before the moved linger call can run. The live Ubuntu report has concrete steps but no before/after smoke log in this PR.

## Solution Assessment

No. Moving the helper only above `service.install()` is not sufficient; the maintainable fix needs to run or split the linger check before classifying user systemd as unavailable, then prove the ordering with tests.

## Repair Work Prompt

Repair PR #54567 for OpenClaw's Linux non-interactive daemon install. The observable problem is fresh/headless Ubuntu setup with disabled linger: the current PR moves `ensureSystemdUserLingerNonInteractive` before `service.install()`, but it is still unreachable when `isSystemdUserServiceAvailable()` returns false, and the helper also self-gates on that same probe. Keep the fix scoped to Linux systemd user-service install behavior: split or adjust the linger helper/probe so non-interactive setup can read `loginctl` status and attempt the existing `sudo -n loginctl enable-linger <user>` path before returning `systemd-user-unavailable` when the failure is disabled/unreachable user bus, while preserving non-Linux no-op behavior, missing-systemd handling, permission-denied handling, and graceful sudo failure. Add focused regression coverage for the ordering/unavailable-bus case, keep existing systemd machine-scope fallback behavior intact, and add a single-line Unreleased Fixes changelog entry crediting the contributor. Do not add a new CLI flag, config option, broad system service mode, dependency, or release/version change.
