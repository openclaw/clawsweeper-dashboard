## Summary

Keep open: the proposed reorder does not fix the reported no-user-bus/headless path because both the installer and the linger helper still return before linger can be enabled, and the PR also lacks after-fix real behavior proof.

## What This Changes

The PR moves the non-interactive systemd linger helper before `service.install()` and adds a comment in the gateway daemon install flow.

## Best Possible Solution

Handle disabled or unreachable user-bus/linger before classifying Linux user systemd as unavailable, preserve true no-systemd skips, and add focused coverage plus redacted headless Ubuntu proof.

## Reproduction Assessment

Yes for the PR defect from source: the installer and moved helper both gate on `isSystemdUserServiceAvailable()`, and tests classify failed user-bus access as unavailable. The intended after-fix behavior is not proven with a live headless-server run.

## Solution Assessment

No. Reordering the helper below the availability gate is not sufficient; the fix needs a pre-gate linger/status path or helper split that distinguishes true systemd absence from disabled or unreachable linger.
