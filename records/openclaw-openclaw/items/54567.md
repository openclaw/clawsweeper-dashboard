## Summary

Keep open: the PR is a focused bug-fix attempt, but the patch is incomplete because the moved linger helper is still unreachable when the Linux systemd availability probe returns false, and the helper itself returns on that same probe. The PR also lacks after-fix real headless-server proof.

## What This Changes

The PR moves `ensureSystemdUserLingerNonInteractive` before `service.install()` in the non-interactive daemon installer and adds a comment explaining the headless systemd case.

## Best Possible Solution

Separate true systemd absence from disabled or unreachable user linger, attempt or surface the linger repair before the unavailable-systemd skip where safe, add focused regression coverage, and include redacted headless Ubuntu proof.

## Reproduction Assessment

Yes for the PR defect from source: a false Linux systemd availability result returns before the moved helper, and the helper has the same false-return guard. I did not run a live headless Ubuntu smoke because this is a read-only review and the PR lacks after-fix runtime proof.

## Solution Assessment

No. Reordering the existing helper below the availability gate is not sufficient for the reported no-user-bus path; the installer needs a pre-gate linger/status path or a refactored availability contract.
