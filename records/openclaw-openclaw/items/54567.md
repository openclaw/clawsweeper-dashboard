## Best Possible Solution

Separate true systemd absence from disabled or unreachable user linger, attempt or surface the linger repair before the unavailable-systemd skip when appropriate, and prove the path with focused regression coverage plus redacted live headless Ubuntu output.

## Solution Assessment

No. Reordering the existing helper below the availability gate is not sufficient because the helper also returns on that same unavailable probe; the install flow needs a pre-gate linger/status path or a refactored helper.
