## Summary

Keep open: the proposed reorder still cannot fix the reported headless no-user-bus path because the installer skips before the moved linger call, and the helper itself returns on the same availability probe. The external PR also lacks after-fix real headless-server proof.

## What This Changes

The PR moves `ensureSystemdUserLingerNonInteractive` before `service.install()` in non-interactive gateway daemon installation and adds an explanatory comment.

## Best Possible Solution

Handle disabled or unreachable user-bus/linger before the daemon install availability skip while preserving true no-systemd behavior, then cover the ordering/classification in focused tests.

## Reproduction Assessment

Yes, source-level: when `isSystemdUserServiceAvailable()` returns false for `Failed to connect to bus`, daemon install exits before the PR's moved linger call. The repository tests cover both that availability classification and the early skip path.

## Solution Assessment

No. Reordering the call only before `service.install()` is not sufficient because it remains below the availability gate and the helper has the same self-gate; the safer fix is a pre-gate linger/status path or helper split.
