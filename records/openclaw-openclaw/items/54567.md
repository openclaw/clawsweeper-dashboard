## Summary

Keep open. The PR targets a real Linux daemon-install failure family, but the proposed reorder is not mergeable as written because the new linger call remains below the Linux user-systemd availability return path, and the linger helper itself returns on the same unavailable probe. Current main has separate systemd user-bus and missing-unit fixes, including the linked issue closure, but it does not implement a reliable pre-gate linger enable path.

## What This Changes

The PR moves the existing non-interactive systemd linger helper before `service.install()` in `installGatewayDaemonNonInteractive` and adds a short explanatory comment.

## Best Possible Solution

The desired end state is either a Linux-only daemon install path that checks or enables linger before any user-bus availability return, with tests proving the sequencing, or a maintainer decision that current main's shipped user-bus and missing-unit fixes fully supersede this branch.

## Reproduction Assessment

Unclear for the live Ubuntu failure: the PR gives concrete steps, but no before/after smoke log is attached. There is a high-confidence static reproduction of the patch flaw: when `isSystemdUserServiceAvailable()` returns false, both current main and the PR return before the linger helper can run.

## Solution Assessment

No. Moving the helper only below the existing availability gate is not the narrowest reliable fix for a no-user-manager headless install; the safer repair is to distinguish truly missing systemd from disabled linger and perform the linger check before the user-bus gate, or close as superseded if maintainers confirm the shipped fixes cover the report.
