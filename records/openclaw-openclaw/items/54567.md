## Summary

Keep open: this is a focused bug-fix attempt, but the patch does not cover the reported headless user-bus path because the moved linger call is still below the early systemd availability skip and the helper self-gates on that same availability probe. The external PR also lacks after-fix real behavior proof from a headless server.

## What This Changes

The PR moves `ensureSystemdUserLingerNonInteractive` ahead of `service.install()` in the non-interactive gateway daemon installer and adds an explanatory comment.

## Best Possible Solution

Distinguish true systemd absence from a disabled or unreachable user bus, run linger/status remediation before the early skip, and add focused seam coverage plus redacted headless Ubuntu proof.

## Reproduction Assessment

Yes, source-level. The installer and moved helper both depend on `isSystemdUserServiceAvailable()`, and current tests classify the reported user-bus failure family as unavailable, so the moved call remains unreachable on that path.

## Solution Assessment

No. Moving the helper only before `service.install()` is not the narrow maintainable fix because it remains below the early availability gate and the helper returns on the same failed probe; the fix needs pre-gate linger/status handling or a split availability contract.
