## Summary

Keep open: this is a focused gateway install fix attempt, but the moved linger call still runs after the Linux user-systemd availability skip, and the helper self-gates on the same probe. The branch also lacks real after-fix headless Ubuntu proof.

## What This Changes

The PR moves `ensureSystemdUserLingerNonInteractive` before `service.install()` in the non-interactive gateway daemon installer and adds a headless-systemd explanatory comment.

## Best Possible Solution

Distinguish true systemd absence from an unreachable user bus, perform valid linger/status remediation before any early skip, and cover the daemon install path with focused seam tests plus redacted headless Ubuntu proof.

## Reproduction Assessment

Yes, source-level. A Linux user-bus failure that makes `isSystemdUserServiceAvailable()` false returns before the PR's moved call, and the helper would also return on that same probe; I did not run a live headless server.

## Solution Assessment

No. Reordering the helper only before `service.install()` is not sufficient for the claimed unreachable-user-manager path because the earlier availability gate and helper self-gate still block the remediation.
