## Summary

Keep open. Current main still routes Linux/non-sips image resizing through the Sharp-backed media runtime without a libvips CLI fallback, and #75732 is an open closing PR for this same failure class.

## What This Changes

The issue requests a durable fallback for Sharp prebuilt binary failures on SSE2-only Common KVM Linux VMs so image uploads keep working after OpenClaw updates.

## Best Possible Solution

Land or replace #75732 with a bounded Linux fallback that preserves pixel-limit guards, temp cleanup, MIME handling, and failure semantics.

## Reproduction Assessment

Yes for the source-level path: force Sharp import or resize to throw on Linux/non-sips, then exercise `optimizeImageToJpeg()` and current main can throw `Failed to optimize image`. The exact SSE2-only KVM hardware path still needs live maintainer or contributor verification.

## Solution Assessment

Unclear. A `vipsthumbnail` fallback directly addresses the report, but the accepted fix should keep the fallback narrowly in the image backend and preserve existing media safety boundaries.
