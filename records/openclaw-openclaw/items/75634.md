## Summary

Keep open. Current main still routes Linux/non-sips image resizing through the Sharp-backed media-understanding runtime with no vipsthumbnail fallback, and #75732 is an open, unmerged closing PR for the same reported failure.

## What This Changes

Request a durable Linux image-optimization fallback when Sharp’s linux-x64 prebuilt fails on SSE2-only Common KVM CPUs.

## Best Possible Solution

Land #75732 or a safer equivalent bounded Linux fallback that preserves image guards, temp-file cleanup, MIME behavior, and failure semantics, then let the linked issue close after merge.

## Reproduction Assessment

Yes for the source-level path: force Sharp import or resize to throw on Linux/non-sips, then exercise optimizeImageToJpeg or loadWebMedia and current main can reach Failed to optimize image. The exact SSE2-only KVM hardware path still needs live maintainer or contributor verification.

## Solution Assessment

Unclear. A Linux vipsthumbnail fallback directly addresses the report, but the accepted fix should be reviewed against guard preservation, temp-file hygiene, external process behavior, and the complementary under-cap original-image fallback in #74243.
