## Summary

Keep open. Current main still routes non-sips Linux image resizing through the Sharp-backed media runtime with no `vipsthumbnail` fallback, and #75732 is an open, unmerged closing PR for this issue.

## What This Changes

Add an update-safe fallback so image optimization works when Sharp prebuilts fail on SSE2-only Linux KVM CPUs.

## Best Possible Solution

Keep this issue open until #75732 or a safer equivalent lands with bounded Linux fallback behavior and preserved media guards.

## Reproduction Assessment

Yes for the source-level path: on Linux/non-sips, force Sharp import or resize to throw, then exercise `optimizeImageToJpeg()` or `loadWebMedia()` and current main can throw `Failed to optimize image`. The exact SSE2-only KVM hardware path still needs live maintainer or contributor verification.

## Solution Assessment

Unclear. A Linux `vipsthumbnail` fallback directly addresses the report, but the accepted fix should preserve pixel-limit checks, temp-file hygiene, MIME handling, and failure semantics; #74243 remains complementary for under-cap original-image fallback policy.
