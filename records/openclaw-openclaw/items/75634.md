## Summary

Keep open. ## What This Changes

The issue asks for image optimization to gracefully handle Sharp linux-x64 prebuilt failures on SSE2-only Common KVM CPUs, preferably with a persistent libvips fallback.

## Best Possible Solution

Review, land, or replace #75732 with a bounded Linux image fallback that preserves pixel-limit guards, temp cleanup, MIME behavior, and current failure semantics.

## Reproduction Assessment

## Solution Assessment

Unclear until the open PR is reviewed. A libvips CLI fallback directly addresses the report, but the accepted fix should stay narrowly in the image backend and preserve existing media safety boundaries.
