## Summary

Keep open: current main and v2026.6.6 still have the unbounded gateway auth limiter map, and https://github.com/openclaw/openclaw/pull/77987 is an open closing PR rather than a merged fix.

## What This Changes

The issue requests a hard bound for the gateway auth failed-attempt limiter's in-memory per-scope/IP entries under unique-IP floods while preserving active lockouts.

## Best Possible Solution

Review and land or replace the linked gateway PR with an insertion-time cap that evicts oldest non-locked entries, preserves active lockouts, and includes focused regression coverage.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: repeatedly call `recordFailure` with unique non-loopback IPs before the prune interval and `entries.size` grows with each new key because there is no insert-time cap. I did not run a live flood in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is to enforce a hard cap on insert while skipping active lockouts so flooding cannot clear a lockout. The open linked PR follows that general shape, but the issue should stay open until that PR or an equivalent fix merges.
