## Summary

The report is credible and still necessary: the latest release and current main both retain the affected first-class provider path, no canonical replacement item was found, and the reported hostname currently fails DNS resolution. ## What This Changes

Make first-class `ollama-cloud/...` model calls reach Ollama Cloud successfully instead of failing DNS lookup for `ai.ollama.com`.

## Best Possible Solution

Capture the first-class provider’s final outbound request URL, repair the narrow component that derives `ai.ollama.com`, and add regression coverage for the actual host and path while preserving the working local/hybrid Ollama route.

## Reproduction Assessment

No complete current-main model call was reproduced with credentials, but the issue provides a concrete released-version path and repeated logs, and the exact failing hostname is independently confirmed as non-resolving. The runtime source path is therefore reproducible enough to investigate with high confidence, but not yet eligible for the strict automatic bug-fix lane.

## Solution Assessment

No, changing the declared cloud base URL is not the best fix because it is already `https://ollama.com`. The narrowest maintainable approach is to trace the final outbound URL and fix whichever runtime or dependency boundary introduces `ai.ollama.com`.
