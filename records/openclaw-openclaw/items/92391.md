## Summary

Keep this open because current main and the latest shipped tag still lack the provider-id doctor migration for persisted `ollama-cloud` configs using the retired `ai.ollama.com` host, and the active linked PR is the implementation path rather than a reason to close before merge.

## What This Changes

The issue reports that first-class `ollama-cloud/...` model routing can fail DNS through a persisted retired `https://ai.ollama.com` endpoint while the older local Ollama cloud route works.

## Best Possible Solution

Review and land the existing doctor migration if maintainers accept the upgrade-sensitive provider config repair; otherwise choose an equivalent migration that rewrites retired `ai.ollama.com` provider config before runtime reads it.

## Reproduction Assessment

Yes. The issue body provides exact setup/logs, current-main source routes persisted `models.providers.ollama-cloud.baseUrl` into the runtime request path, and a live DNS check shows `ai.ollama.com` does not resolve while `ollama.com` does.

## Solution Assessment

Yes. The linked PR's doctor migration is the best-shaped fix under repository policy because runtime reads canonical config only and legacy provider config belongs in `openclaw doctor --fix`, not a silent runtime fallback.
