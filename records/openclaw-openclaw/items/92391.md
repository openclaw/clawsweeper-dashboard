## What This Changes

The issue reports that first-class `ollama-cloud/...` models fail DNS lookup against `ai.ollama.com` while the older `ollama/<model>:cloud` route works.

## Best Possible Solution

Merge a provider-owned doctor migration that rewrites retired Ollama Cloud endpoint config to the canonical host and discovers provider-scoped doctor contracts from `models.providers`, then close this issue.

## Reproduction Assessment

Yes, from source and issue logs, but not from a live authenticated request in this review. Current main reads persisted `models.providers.ollama-cloud.baseUrl` before the default, and the report includes config/log evidence that the persisted value is `https://ai.ollama.com`.

## Solution Assessment

Yes. A plugin-owned doctor migration is the best fit under repository policy because it repairs retired persisted config before runtime, while a runtime fallback would silently preserve legacy configuration.
