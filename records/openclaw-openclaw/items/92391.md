## What This Changes

The issue reports that first-class `ollama-cloud/<model>` selections fail against `ai.ollama.com` even though the hybrid `ollama/<model>:cloud` route works.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/92594 or an equivalent plugin-owned doctor migration, then close this issue once the fix is merged.

## Reproduction Assessment

Yes. A persisted `models.providers.ollama-cloud.baseUrl` pointing at `https://ai.ollama.com` is enough to reproduce the source path: current main reads that value into runtime calls, and live DNS checks confirm that host does not resolve here.

## Solution Assessment

Yes. The best fix direction is the linked plugin-owned doctor migration plus provider-id discovery repair; a runtime fallback would hide stale config instead of migrating it to the canonical endpoint.
