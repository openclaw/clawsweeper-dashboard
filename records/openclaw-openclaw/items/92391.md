## What This Changes

Direct `ollama-cloud/...` model routing fails with DNS lookup errors for the retired `ai.ollama.com` endpoint even though the hybrid `ollama/<model>:cloud` path works.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/92594; the desired end state is an Ollama-owned doctor migration that rewrites the retired provider endpoint before runtime reads canonical config.

## Reproduction Assessment

Yes for source reproduction, but not a full authenticated live chat run. ## Solution Assessment

Yes. The best fix direction is the linked PR's plugin-owned doctor migration plus provider-id doctor discovery; adding a runtime fallback would hide stale config instead of repairing it.
