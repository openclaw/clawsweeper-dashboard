## Summary

Keep this PR open for revision: it prevents the reported silent loss, but does so by adding an undocumented model-facing compatibility alias where the message-tool contract deliberately exposes only the canonical `media` parameter.

## What This Changes

The PR makes top-level and structured `image` values participate in outbound media normalization and send-payload construction.

## Best Possible Solution

Keep `media` as the single documented send source and reject a supplied `image` parameter with an actionable validation error before delivery, with regression coverage proving the tool no longer returns successful text-only delivery.

## Reproduction Assessment

Yes. Current main normalizes the `image` key but omits it from final outbound media selection, giving a high-confidence source path for successful text delivery with the requested attachment discarded.

## Solution Assessment

No. The patch stops the symptom by creating an undocumented parallel alias; the narrower contract-preserving fix is to reject `image` clearly and direct the caller to the existing `media` parameter.
