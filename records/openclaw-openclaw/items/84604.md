## Summary

Keep open: this is a concrete upgrade/auth-provider report with fresh follow-up state, and current main has related Claude CLI routing and external-CLI sync pieces but no high-confidence proof that the macOS 4.x to 5.x Keychain-backed migration now creates the missing Claude CLI auth profile and recovery diagnostics automatically.

## What This Changes

The issue reports that upgrading from OpenClaw 2026.4.15 to 2026.5.18 can leave Claude CLI-backed agents without the canonical auth-profile/runtime mapping, causing claude-cli harness registration failures.

## Best Possible Solution

Add a tested doctor/upgrade repair that detects legacy Claude CLI runtime use, safely imports the existing Claude CLI Keychain credential into anthropic:claude-cli when allowed, records an explicit migration receipt, and fails with actionable auth/runtime guidance when that import is unavailable.

## Reproduction Assessment

No high-confidence live reproduction is available from this review. The report provides concrete upgrade steps and source inspection shows a plausible missing scoped-auth import path, but proving the exact failure needs a macOS 2026.4.15 Claude CLI Keychain install upgraded to current or the affected shipped release.

## Solution Assessment

Unclear. The requested migration direction is reasonable, but the maintainable fix should likely be a bounded doctor/upgrade repair with a receipt and Keychain-proofed behavior rather than unconditional gateway startup mutation or relying on lastGood semantics.
