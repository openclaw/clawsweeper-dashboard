## What This Changes

The issue reports that upgrading from OpenClaw 2026.4.15 to 2026.5.18 can leave Claude CLI auth-profile/runtime mapping absent, causing Claude CLI-backed agents to fail with `MissingAgentHarnessError`.

## Best Possible Solution

Add a maintainer-approved doctor or upgrade repair that detects legacy Claude CLI runtime use, imports valid existing Claude CLI credentials into the canonical profile path, emits an auditable receipt, and reconcile that with the linked selector fix so CLI aliases are never treated as plugin harness ids.

## Reproduction Assessment

Yes for the selector half: current source calls `selectAgentHarness` from compaction and still throws for a forced `claude-cli` runtime because it is not a plugin harness. The 4.x to 5.x auth-profile migration half is concrete from reporter state but still needs live macOS Keychain upgrade proof against current main.

## Solution Assessment

Unclear. The selector fix is narrow and already has open PR coverage, but the auth-profile repair should likely be a bounded doctor/upgrade migration with a receipt rather than an unconditional gateway startup migration, which needs maintainer confirmation.
