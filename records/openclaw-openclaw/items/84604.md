## Summary

## What This Changes

The issue reports that upgrading from OpenClaw 2026.4.15 to 2026.5.18 can leave Claude CLI auth-profile/runtime state unmigrated, causing `claude-cli` harness registration failures.

## Best Possible Solution

Add a maintainer-reviewed doctor or upgrade repair that detects legacy Claude CLI runtime use, imports valid external CLI credentials into canonical auth-profile state, records a migration receipt, and pairs that with the CLI-alias selector regression fix.

## Reproduction Assessment

Yes, at source level for the selector half: current docs say `claude-cli` is not a harness id, while current main can still pass it to `selectAgentHarness` and throw. The original macOS 4.x to 5.18 Keychain migration half still lacks a live reproduction in this review.

## Solution Assessment

Unclear as a complete fix. The requested migration direction is plausible, but the best maintainable path should be a bounded doctor/upgrade repair with a receipt plus the selector regression, not an unconditional gateway startup mutation.
