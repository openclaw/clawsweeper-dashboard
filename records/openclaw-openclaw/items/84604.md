## What This Changes

The issue reports that a 2026.4.15 to 2026.5.18 upgrade can leave Claude CLI auth-profile/runtime state unmigrated, causing Claude CLI-backed Telegram agents to return missing-harness errors.

## Best Possible Solution

Add a tested `openclaw doctor --fix` or upgrade-repair path that detects legacy Claude CLI runtime use, imports existing Claude CLI credentials into the canonical `anthropic:claude-cli` profile when safe, restores needed routing state, and emits a migration receipt or explicit failure.

## Reproduction Assessment

No, not as a high-confidence live repro yet. The report gives concrete upgrade steps and source inspection supports a migration/diagnostic gap, but proving the user-visible failure needs a macOS 2026.4.15 to current or next-release Keychain-backed upgrade run.

## Solution Assessment

Unclear. The requested profile/state migration is plausible, but the best maintainable fix should be a bounded doctor/upgrade repair with receipt and Keychain failure handling rather than an unconditional gateway startup migration.
