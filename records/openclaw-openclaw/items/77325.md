## Summary

Keep open: the reported regression is valid and this PR targets the right shared auto-reply path, but the slash-prefix predicate is too broad because it also makes text slash turns that continue into the agent bypass the group `message_tool` visibility policy.

## What This Changes

The PR adds a shared predicate for native commands and slash-prefixed text commands, wires it into source reply delivery and dispatch preference, and adds regression tests plus a changelog entry for #77260.

## Best Possible Solution

Keep a generic command-reply visibility exception, but apply it only to confirmed native or consumed text command replies while leaving normal and continuation agent output under `visibleReplies`.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction for the original regression: Matrix/Tlon group `/new` turns arrive as `CommandSource: "text"`, while current main only exempts `CommandSource: "native"` before suppressing automatic group replies under `messages.groupChat.visibleReplies: "message_tool"`.

## Solution Assessment

No, not as submitted. The shared fix direction is better than channel-specific special-casing, but the current predicate keys off raw slash prefix rather than a consumed command reply, so it can make normal agent continuations visible in groups.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep slash-prefixed continuations under the group policy:** `src/auto-reply/reply/source-reply-delivery-mode.ts:31`
  - body: This treats any text slash-prefix as an automatic command-source turn before command handling proves the turn ended in a canned command reply. Inputs like `/new summarize this` fall through from the reset handler when a tail is present, unhandled slash text returns `shouldContinue: true`, and plugin commands can explicitly continue into the agent; with this predicate those normal agent replies can bypass `messages.groupChat.visibleReplies: "message_tool"` in Matrix/Tlon groups. Please tie the automatic exemption to consumed command-result replies, or otherwise keep continuation turns in `message_tool_only` mode.
  - confidence: 0.86
- **[P3] Update the visible-replies docs:** `src/auto-reply/reply/source-reply-delivery-mode.ts:45`
  - body: This behavior change makes text command replies bypass the group `message_tool` suppression, but `docs/channels/groups.md` still says text-typed `/...` commands follow the configured group default. After merge, the public docs would describe the old behavior, so please update that visible-replies paragraph with the new command-reply exception.
  - confidence: 0.9
