## Summary

Keep open: the PR addresses a real current-main regression, but the submitted predicate is too broad and would let unknown slash-prefixed text turns bypass the configured group reply suppression.

## What This Changes

The PR adds a shared command-source predicate for native commands and text slash-prefix bodies, wires it into auto-reply source delivery and dispatch tool preference, and adds regression tests plus a changelog entry for #77260.

## Best Possible Solution

Keep the core auto-reply fix, but exempt only actual handled text slash command replies from message_tool suppression so unknown slash-prefixed agent turns still honor the configured visibility policy.

## Reproduction Assessment

Yes. Source inspection on current main shows Matrix/Tlon text slash commands enter with CommandSource: "text" while group/channel delivery only bypasses suppression for native commands.

## Solution Assessment

No as submitted. The shared predicate is the right location, but using a raw slash-prefix test widens automatic delivery beyond handled command replies.

- **[P2] Narrow the text-command bypass to handled commands:** `src/auto-reply/reply/source-reply-delivery-mode.ts:36`
