## Summary

Keep open: current main and the latest release still use the cached xAI OAuth token endpoint, and this PR is a narrow candidate fix, but it still needs after-fix OpenClaw refresh proof or a maintainer override before merge.

## What This Changes

The PR changes xAI OAuth refresh to rediscover the token endpoint when a persisted credential still points at `https://auth.x.ai/oauth/token` and adds a regression test for that path.

## Best Possible Solution

Land the xAI plugin-owned refresh mitigation after redacted terminal/live output or a maintainer override proves a stale persisted endpoint is rediscovered and normalized.

## Reproduction Assessment

Yes for the source-level path: current main posts refresh to the cached endpoint, and live xAI probes show `/oauth2/token` returns OAuth JSON while `/oauth/token` returns Cloudflare HTML. A full patched OpenClaw stale-token refresh run has not been shown.

## Solution Assessment

Yes for the narrow mitigation: the fix stays in the xAI plugin, special-cases only the known retired endpoint, and leaves other cached endpoints untouched. It may not resolve every report under https://github.com/openclaw/openclaw/issues/84018 without live stale-token proof.
