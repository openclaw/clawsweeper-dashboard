## What This Changes

The branch adds an xAI OAuth refresh compatibility path that rediscovers the current token endpoint when a saved credential contains the retired `https://auth.x.ai/oauth/token` endpoint, with regression tests for rediscovery and discovery failure.

## Best Possible Solution

Land the narrow provider-owned repair after maintainer acceptance and green exact-head gates, while keeping broader xAI renewal and entitlement questions tracked separately.

## Reproduction Assessment

Yes for the narrow stale-endpoint state: current main posts refreshes to a saved `tokenEndpoint`, live xAI discovery advertises `/oauth2/token`, and live dummy probes show `/oauth/token` returns Cloudflare HTML. The broader source of those stale saved values is still not proven.

## Solution Assessment

Yes, this is the best narrow repair for the proven state: it stays in the xAI plugin, only handles the exact retired endpoint, uses trusted discovery, and avoids sending refresh tokens to the retired endpoint when discovery fails.
