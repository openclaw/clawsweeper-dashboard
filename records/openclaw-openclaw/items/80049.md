## Summary

Keep open: this PR targets a source-reproducible raw tool-output leak and the code direction is sound, but it still needs required real behavior proof before merge and a small docs follow-up for the changed result contract.

## What This Changes

The PR changes subagent announce output selection to stop returning latest raw tool/toolResult text when assistant output is absent, and updates format e2e expectations to `(no output)`.

## Best Possible Solution

Land one canonical fix that removes raw tool/toolResult fallback across direct and refresh captures, preserves assistant and timeout partial-progress behavior, updates the public subagents result contract, and closes the linked bug after proof and CI are green.

## Reproduction Assessment

Yes for a source-level reproduction: current main's selector reaches `latestRawText` when a child transcript has no assistant text and ends in tool/toolResult output. I did not live-reproduce the Telegram duplicate-progress half in this read-only PR review.

## Solution Assessment

Yes for the central raw-output leak: returning `undefined` unconditionally is narrower and more complete than the older outcome-only guard because it also covers capture paths that omit outcome. The branch still needs real setup proof and a docs update before merge.
