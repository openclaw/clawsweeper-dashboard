## Summary

Keep open: the raw-output leak remains source-reproducible on current main and this patch is the right narrow direction, but the external PR still lacks real behavior proof and leaves public subagent Result docs stale.

## What This Changes

The branch removes the raw `latestRawText` fallback from subagent announce output selection and updates e2e expectations for tool-only child histories to `(no output)`.

## Best Possible Solution

Land one canonical fix that removes raw tool/toolResult fallback across direct and refresh captures, preserves assistant and timeout partial-progress behavior, updates the subagents Result docs, and then closes the linked bug after proof and CI are green.

## Reproduction Assessment

Yes for a source-level reproduction: current main populates `latestRawText` from tool/toolResult messages and returns it when assistant output is absent, and current tests assert that behavior. I did not live-reproduce the Telegram duplicate-progress half.

## Solution Assessment

Yes for the central raw-output leak: returning `undefined` unconditionally is narrower and more complete than the older outcome-only guard because it also covers capture paths that omit outcome. The branch still needs real setup proof and a docs update before merge.
