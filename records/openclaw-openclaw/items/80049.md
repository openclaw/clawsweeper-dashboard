## Summary

Keep open: this PR is a focused and plausible fix for the source-reproducible raw tool-output fallback, but it still needs stronger real behavior proof, a subagents docs update, and maintainer handling of the duplicate-progress part of the linked bug before merge/closure.

## What This Changes

The branch changes subagent announce output selection to return no result instead of latest raw tool/toolResult text when no assistant-produced text is available, and updates e2e expectations to `(no output)`.

## Best Possible Solution

Land one canonical raw-output fix, update the subagents result contract docs, and keep or split the duplicate-progress portion into a separate tracked item if it is not proven fixed.

## Reproduction Assessment

Yes for a source-level reproduction: current main selects `snapshot.latestRawText` when the child transcript has no assistant text and ends in tool/toolResult output. I did not establish a live Telegram reproduction in this read-only review.

## Solution Assessment

Yes for the central raw-output leak: returning `undefined` is the narrow fix because it forces assistant-reply or controlled no-output handling instead of replaying tool text. It is not enough by itself to close the broader linked bug unless the duplicate-progress half is separately resolved or tracked.
