## What This Changes

The branch adds a TUI thinking whitespace normalizer that turns single newline boundaries into spaces and adds formatter plus stream-assembler regression tests.

## Best Possible Solution

Normalize newline variants to LF inside the thinking formatter, preserve explicit paragraph breaks, add CRLF/bare-CR regression coverage, then merge after after-fix TUI proof is provided.

## Reproduction Assessment

Yes, source-reproducible: current main newline-joins thinking blocks, the stream assembler passes that text into displayed TUI output, and the current test still expects `alpha\nbeta`. I did not run an interactive TUI session because this review is read-only.

## Solution Assessment

No. The PR targets the right formatter path, but it should normalize `\r\n?` to `\n` before collapsing single LFs and add matching regressions before merge.
