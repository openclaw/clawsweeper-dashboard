## Summary

Keep open: this PR targets a real source-reproducible TUI thinking-whitespace bug that current main still does not solve, but the patch should not merge until it handles CRLF/bare-CR thinking input and has after-fix real TUI proof.

## What This Changes

The PR adds TUI thinking-whitespace normalization and updates formatter/stream-assembler tests to expect spaces between adjacent thinking fragments while preserving paragraph breaks.

## Best Possible Solution

Normalize newline variants to LF before collapsing single thinking newlines, preserve explicit paragraph breaks, add CRLF/bare-CR regression coverage, and require a real TUI run showing the fixed behavior before merge.

## Reproduction Assessment

Yes, source-reproducible: current main newline-joins thinking blocks, and the stream assembler feeds that formatter output into displayed assistant text. I did not run a live TUI session because this review is read-only.

## Solution Assessment

No, not as submitted. The PR targets the right formatter seam, but it should normalize `\r\n?` to `\n` before collapsing single newlines and add matching regressions before merge.

## Review Findings

Overall correctness: patch is incorrect
