## Summary

Keep this PR open for a narrow repair. It addresses a real current-main TUI thinking-whitespace bug, but the proposed helper only handles LF newlines and the user-visible fix still needs the required changelog entry.

## What This Changes

The PR adds a TUI thinking-whitespace normalizer, applies it to thinking extraction/rendering paths, and adds formatter plus stream-assembler regressions for adjacent thinking fragments.

## Best Possible Solution

Merge a narrow TUI fix that normalizes newline variants before collapsing single thinking-fragment boundaries, preserves paragraph breaks, adds CRLF/bare-CR coverage, includes a changelog entry, and leaves answer text plus non-thinking block joining unchanged.

## Reproduction Assessment

Yes. Current main has a unit-level reproduction where two thinking blocks render as `alpha\nbeta`, and the stream assembler passes formatter output directly to the displayed assistant message.

## Solution Assessment

No. The PR direction is right, but the current implementation should normalize `\r\n?` to `\n` before replacing single LFs and add the required changelog entry before merge.
