Full review comments:

- **[P2] Normalize CRLF before replacing thinking newlines:** `src/tui/tui-formatters.ts:250`
  - body: This helper only replaces `\n`, so a CRLF boundary like `First.\r\nSecond.` becomes `First.\r Second.`. `sanitizeRenderableText` preserves `\r`, so CRLF thinking streams will not get the intended space boundary and can still carry a carriage return into the TUI text path. Normalize `\r\n?` to `\n` before this replacement and add a regression test.
