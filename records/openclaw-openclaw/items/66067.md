## Summary

Keep open. Current main still newline-joins thinking blocks in the TUI, so the PR addresses a real remaining readability bug, but the patch needs CRLF/bare-CR normalization and a required changelog entry before it is merge-ready.

## What This Changes

The PR adds a TUI thinking-whitespace normalizer, applies it to thinking extraction/rendering paths, and adds formatter plus stream-assembler regression tests for spaces between thinking fragments.

## Best Possible Solution

Keep the change narrow to TUI thinking text: normalize all newline variants to LF before collapsing single thinking-fragment boundaries to spaces, preserve blank-line paragraph breaks, add CRLF/bare-CR regression coverage and a user-facing changelog entry, and leave assistant answer text plus non-thinking block joining unchanged.

## Reproduction Assessment

Yes. Current main has a unit-level reproduction: multiple thinking blocks render as `alpha\nbeta`, and the streaming assembler displays that formatter output without another normalization pass. The PR also supplies the intended LF-only regression path but still needs CRLF coverage.

## Solution Assessment

No. The direction is the right maintainable fix, but the proposed helper is incomplete because it handles only LF and omits the required changelog entry; normalizing `\r\n?` to `\n` before collapsing single newlines is the safer version.
