## Summary

This PR should stay open: current main still passes thinking fragments through as newline-joined text, and the branch targets the right TUI formatter path, but it still has a concrete CRLF/bare-CR handling bug and lacks after-fix real TUI proof.

## What This Changes

The PR normalizes TUI thinking-block whitespace so adjacent streamed thinking fragments render with spaces while preserving paragraph breaks, and adds formatter and stream-assembler regression tests.

## Best Possible Solution

Normalize newline variants to LF before collapsing single thinking newlines, preserve explicit paragraph breaks, add CRLF/bare-CR regression coverage, and require a redacted real TUI run showing the fixed behavior before merge.

## Reproduction Assessment

Yes, source-reproducible: current main newline-joins thinking blocks, and the stream assembler feeds that formatter output into the displayed assistant text. I did not run a live TUI session in this read-only review.

## Solution Assessment

No, not as submitted. The PR targets the right formatter seam, but it should normalize \r\n? to \n before collapsing single newlines and add matching regression coverage.

## Review Findings
