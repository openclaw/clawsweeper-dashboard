## Summary

Keep open: this PR addresses a real TUI thinking-readability bug, but the current patch is incomplete because its whitespace helper only handles LF newlines and the contributor proof is limited to tests rather than observed TUI behavior.

## What This Changes

The PR adds a TUI thinking-whitespace normalizer, applies it when extracting thinking text, and adds formatter plus stream-assembler regressions for adjacent thinking fragments.

## Best Possible Solution

Merge a narrow TUI formatter fix that normalizes newline variants to LF before converting single thinking-fragment boundaries to spaces, preserves paragraph breaks, adds CRLF/bare-CR regressions, and leaves non-thinking text unchanged.

## Reproduction Assessment

Yes, from source inspection: current main's formatter has a test expecting two thinking blocks as `alpha\nbeta`, and the stream assembler consumes that formatter output directly. I did not run tests because this cleanup review is read-only.

## Solution Assessment

No. The PR is aimed at the right code path, but it should normalize `\r\n?` to `\n` before collapsing single newlines and it needs after-fix real TUI proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Normalize CRLF before collapsing thinking newlines:** `src/tui/tui-formatters.ts:250`
  - body: The new helper only matches `\n`, so `First\r\nSecond` renders as `First\r Second`, and CRLF paragraph breaks collapse because the current sanitizer preserves `\r`. Normalize `\r\n?` to `\n` before replacing single LFs and add CRLF/bare-CR coverage.
  - confidence: 0.92
