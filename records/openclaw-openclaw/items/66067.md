## Summary

Keep open: this is a focused TUI bug-fix PR whose exact space-normalization behavior is not on current main, but it should not merge as submitted because the new helper leaves CRLF/bare-CR input unsafe and the PR still lacks after-fix real TUI proof.

## What This Changes

The PR adds TUI thinking whitespace normalization so adjacent streamed thinking fragments render with spaces while preserving paragraph breaks, plus formatter and stream-assembler regression tests.

## Best Possible Solution

Normalize newline variants to LF before collapsing single thinking newlines, preserve explicit paragraph breaks, add CRLF/bare-CR regression coverage, then require after-fix TUI proof before merge.

## Reproduction Assessment

Yes, source-reproducible: current main newline-joins thinking blocks, the stream assembler passes that formatter output into displayed assistant text, and the existing regression still expects `alpha\nbeta`. I did not run a live TUI session because this review is read-only.

## Solution Assessment

No, not as submitted. The PR targets the right formatter seam, but it should normalize `\r\n?` to `\n` before collapsing single newlines and add matching regressions before merge.
