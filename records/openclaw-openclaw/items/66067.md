## Summary

Keep this PR open: it targets a real TUI readability bug that current main has not implemented, but the proposed patch still has a concrete CRLF handling bug and lacks required after-fix real TUI proof.

## What This Changes

The PR adds a TUI thinking-whitespace normalizer, applies it to extracted thinking text, and adds formatter plus stream-assembler regressions for adjacent thinking fragments.

## Best Possible Solution

Normalize newline variants to LF before collapsing single thinking-fragment boundaries, preserve explicit paragraph breaks, add CRLF/bare-CR regressions, and include after-fix TUI proof.

## Reproduction Assessment

Yes, source-reproducible: current main still newline-joins thinking blocks and the stream assembler passes that text into the TUI assistant renderer. I did not run an interactive terminal session because this review is read-only.

## Solution Assessment

No. The PR is aimed at the right formatter path, but it should normalize `\r\n?` to `\n` before collapsing single newlines and add matching regressions before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Normalize CRLF before collapsing thinking newlines:** `src/tui/tui-formatters.ts:250`
