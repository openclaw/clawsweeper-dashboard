## Summary

Keep open: current main and the latest shipped release still have the unbounded Discord REST entity cache behavior, and this PR is the focused fix path with sufficient after-fix runtime proof and no blocking review findings found.

## What This Changes

The PR adds an insert-time expiry sweep, a 5,000-entry default cap, optional cache tuning parameters, a size getter, colocated eviction tests, and a Discord changelog entry.

## Best Possible Solution

Land a conflict-free version of this Discord-owned cap/sweep fix with the focused regression tests, leaving LRU telemetry as a follow-up if maintainers want it.

## Reproduction Assessment

Yes. Source inspection shows a stream of unique user, channel, guild, or member fetch keys all reach entries.set while expired unrelated keys are never swept and the Map has no max size on current main and v2026.5.7.

## Solution Assessment

Yes. A plugin-local insert-time expiry sweep plus a hard cap directly bounds the existing cache without changing public fetch semantics or gateway invalidation behavior.
