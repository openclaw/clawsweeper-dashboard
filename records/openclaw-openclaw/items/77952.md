## What This Changes

The PR adds a write-time expiry sweep, a 5,000-entry FIFO cap, optional constructor tuning, a diagnostic size getter, colocated tests, and a changelog entry for DiscordEntityCache.

## Best Possible Solution

## Solution Assessment

Yes. A Discord-plugin-local insert-time expiry sweep plus a hard cap directly bounds the existing cache without changing public fetch semantics or gateway invalidation behavior.
