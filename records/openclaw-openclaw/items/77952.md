## What This Changes

The branch adds a capped, write-time-swept DiscordEntityCache with a size getter, colocated eviction tests, and a Discord changelog entry.

## Best Possible Solution

Land a conflict-free version of the plugin-local cap/sweep with deterministic tests and changelog coverage, leaving LRU telemetry as a focused follow-up if maintainers want it.

## Reproduction Assessment

Yes, source-reproducible rather than locally executed: current main only expires the exact key being fetched, and the long-lived DiscordClient owns one cache instance. A stream of distinct REST entity keys can therefore grow the Map past the TTL window until another mechanism removes entries.

## Solution Assessment

Yes. A plugin-local insert-time expiry sweep plus hard cap directly bounds the existing cache without changing public fetch semantics; LRU can be a telemetry-backed follow-up instead of blocking this leak fix.
