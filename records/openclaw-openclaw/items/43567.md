## Summary

Keep open. Current main fixed the concrete pre-compaction leak by hiding it from the visible transcript, but it has not implemented the broader product request for intentionally visible runtime/system messages to render as a distinct Control UI/TUI block.

## What This Changes

The issue requests a distinct visual treatment for internal/runtime system messages so they do not look like user or assistant chat bubbles.

## Best Possible Solution

Define first-class runtime/system provenance, keep internal-only messages hidden, and render only intentionally visible system notices outside normal chat bubbles with a dedicated label/style shared by Control UI and TUI where appropriate.

## Reproduction Assessment

Partly. Source review shows current Control UI has no dedicated system/runtime notice presentation lane, and related reports provide real examples, but I did not run a live current-main visual reproduction.

## Solution Assessment

No as a UI-only patch. The maintainable solution should first settle the runtime audience/provenance contract, then make Control UI/TUI render intentionally visible notices distinctly while preserving hidden-internal filtering.
