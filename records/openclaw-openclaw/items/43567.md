## Summary

Keep open: current main and v2026.5.22 fixed the narrow pre-compaction memory-flush transcript leak, but the broader request still lacks a shared runtime/system provenance contract and a Control UI projection for generic internal system messages as distinct blocks.

## What This Changes

Request internal runtime/system messages be shown as clearly labeled system UI blocks instead of normal chat bubbles.

## Best Possible Solution

Define one shared runtime/system audience or provenance contract, then project it consistently through Gateway history, Control UI, and TUI as hidden internal context or a labeled system block according to maintainer UX policy.

## Solution Assessment

Unclear pending maintainer UX/API direction. A shared audience/provenance contract plus one display projection is safer than per-surface string filtering or a narrow patch that recognizes only pre-compaction text.
