## Summary

Keep open. Current main fixes the concrete pre-compaction memory-flush leak, but the broader request is still necessary because Control UI still lacks a generic runtime/system provenance block and the related WebChat continuation/audience problem remains open.

## What This Changes

Surface internal runtime/system messages in a distinct UI treatment instead of rendering them like ordinary user or assistant chat bubbles.

## Best Possible Solution

Define one generic runtime/system audience and provenance contract, then have Gateway projection and Control UI/TUI render intentionally visible system rows in a dedicated diagnostic block while keeping agent-only prompts out of the visible transcript.

## Reproduction Assessment

No, not as a live current-main repro. Source inspection shows current Control UI lacks a dedicated system block for `role: "system"` rows, and related WebChat discussion gives a concrete restart-continuation symptom that still needs live confirmation.

## Solution Assessment

No, a simple color-only UI patch is only a plausible partial fix. The best fix first chooses the runtime audience/provenance model so agent-only continuation prompts are hidden or dispatched correctly while intentionally visible system events get a distinct presentation.
