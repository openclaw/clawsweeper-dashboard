## Summary

Keep open: current main fixes the narrow pre-compaction memory-flush transcript leak, but the broader request still lacks a shared runtime/system audience contract and a Control UI/TUI projection that renders system-originated items as a distinct non-user block.

## What This Changes

Render runtime/internal system-originated messages in Control UI/TUI as a clearly labeled non-user/non-assistant block instead of ordinary chat bubbles.

## Best Possible Solution

Define one shared runtime/system audience or provenance contract, then project it consistently through gateway history and Control UI/TUI as hidden internal context or a labeled system block according to maintainer UX policy.

## Reproduction Assessment

Yes, at source level. Current main still formats generic queued SystemEvent entries as `System:` prompt text, and Control UI has no generic runtime/system ChatItem or projection for `internal_system` provenance.

## Solution Assessment

Unclear pending maintainer product/API direction. A shared audience/provenance contract plus one display projection is safer than a per-surface string filter or a narrow styling patch that only recognizes the pre-compaction text.
