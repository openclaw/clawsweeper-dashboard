## Summary

Keep open. Current main fixed the narrow pre-compaction memory-flush transcript leak, but the broader UI request remains: visible `system` rows still lack a dedicated Control UI system treatment, and TUI history replay does not route ordinary system-role history rows through its system-notice path; #70348 is a narrower artifact-filtering PR, not a full replacement.

## What This Changes

The issue requests that intentionally visible internal system/runtime messages render as a distinct system block in Control UI/TUI rather than looking like ordinary chat bubbles.

## Best Possible Solution

Add an explicit system/runtime display contract and render it with dedicated Control UI styles plus TUI `addSystem` replay, while keeping known internal artifacts filtered from normal chat history.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to project or render a visible `role: "system"` history row on current main: Control UI falls through to `other` styling, and TUI history replay has no ordinary system-role branch.

## Solution Assessment

Unclear for an automated fix lane. Distinct rendering is the right end state, but maintainers should reconcile it with the hide-versus-label filtering policy in #70348 before a cross-surface implementation.
