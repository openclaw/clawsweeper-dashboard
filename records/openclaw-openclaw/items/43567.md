## Summary

Keep open. Current main fixed the concrete pre-compaction memory-flush prompt leak by making that maintenance turn transcript-silent, but the broader request for a first-class runtime/system audience and distinct visible treatment is still unresolved and overlaps several open, more specific reports.

## What This Changes

The issue asks OpenClaw to render internal/runtime system messages separately from normal user or assistant chat bubbles.

## Best Possible Solution

Define a first-class runtime/system audience contract: agent-only internals stay out of user-visible history, while intentionally visible runtime notices render in a dedicated diagnostic/system treatment across Control UI/WebChat and TUI.

## Reproduction Assessment

No high-confidence live reproduction for the original pre-compaction prompt was established; that narrow leak was addressed by a merged memory-flush change. Source inspection still shows the broader absence of a generic runtime/system audience and display contract in shared projection plus UI rendering.

## Solution Assessment

No, a UI-only colored block is not the best standalone fix. The maintainable path should first decide which runtime messages are agent-only versus intentionally visible, then implement shared provenance filtering/projection and matching UI/TUI rendering.
