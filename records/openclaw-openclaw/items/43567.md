## Summary

Keep open: current main has fixed the specific pre-compaction prompt leak separately, but it still lacks a generic Control UI/TUI runtime-system block or settled provenance/audience contract for deciding which internal messages should be hidden versus visibly diagnosed.

## What This Changes

The issue requests a distinct Control UI/TUI visual treatment for internal system/runtime messages so they are not confused with user or assistant chat.

## Best Possible Solution

Define a shared provenance/audience contract, hide agent-only runtime inputs before display projection, and render intentionally visible diagnostics through one consistent Control UI/TUI system block.

## Reproduction Assessment

Not applicable as a feature/UX request. Source inspection gives high confidence that current main lacks the requested shared system/runtime block, while specific leakage examples are being handled in narrower related work.

## Solution Assessment

Unclear pending maintainer UX/API direction. A distinct block is plausible for intentionally visible diagnostics, but styling leaked internal prompts alone is not the best fix without first settling the provenance and audience boundary.
