## Summary

Keep open: current main fixed the narrow pre-compaction flush leak, but the broader UI/runtime provenance request is still not implemented. Generic system events still have only text/context metadata, still format as `System:` prompt text, and Control UI only has a compaction-specific divider rather than a generic runtime/system block; related audience and continuation work remains open.

## What This Changes

Request to render internal system/runtime messages as clearly distinct non-user/non-assistant UI items instead of ordinary chat bubbles.

## Best Possible Solution

Define one shared runtime/system audience or provenance contract, then project it consistently through gateway history and Control UI/TUI as hidden internal context or a labeled system block according to maintainer UX policy.

## Reproduction Assessment

Yes, at source level. Current main still formats generic queued `SystemEvent` entries as `System:` prompt text, and Control UI has no generic runtime/system chat item or projection branch for `internal_system` provenance.

## Solution Assessment

Unclear until maintainers choose the audience and UX boundary. A shared provenance contract plus one display projection is safer than per-surface string filters or a patch that only recognizes pre-compaction text.
