## Summary

Keep open. Current main has useful partial fixes for the concrete memory-flush leak and a compaction divider, but it still lacks a first-class, consistent runtime/system message block or audience contract across Control UI and TUI.

## What This Changes

Request to render internal/runtime-originated messages in a clearly distinct system UI treatment instead of normal chat bubbles.

## Best Possible Solution

Define typed audience/provenance for runtime/system events, hide agent-only prompts, and render intentionally visible diagnostics through a shared Control UI/TUI system block.

## Reproduction Assessment

Not applicable as a feature/UX request. Source inspection shows partial compaction and memory-flush handling, but no current contract that every visible internal/runtime message must render as a distinct system block.

## Solution Assessment

Unclear. A distinct block is plausible for intentional diagnostics, but the maintainable fix needs an audience/provenance contract first so agent-only internal turns can be hidden rather than styled after the fact.
