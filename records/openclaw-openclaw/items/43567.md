## Summary

Keep open. Current main fixes the concrete pre-compaction memory-flush leak by making that maintenance prompt runtime-only, but the broader request for a generic, distinct runtime/system UI treatment is not implemented and overlaps active product/API tracks.

## What This Changes

The issue requests that internal runtime/system messages appear in a clearly distinct UI block instead of normal chat bubbles.

## Best Possible Solution

Define a shared structural audience/provenance contract for runtime-internal input, then have Gateway history projection and Control UI either suppress those rows or render intentionally visible ones in a typed system/runtime block.

## Reproduction Assessment

Yes, at source level. The original pre-compaction example is fixed, but restart continuations still carry internal_system provenance in normal body fields and the current display/UI layers have no generic runtime/system block path.

## Solution Assessment

Unclear pending maintainer product direction. A maintainable fix should be structural, using provenance or event audience in the shared projection, rather than styling or hiding messages by broad text-prefix matches.
