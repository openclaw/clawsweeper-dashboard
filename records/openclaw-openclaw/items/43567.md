## What This Changes

Requests that internal runtime/system messages be rendered distinctly from normal user and assistant chat in Control UI/TUI.

## Best Possible Solution

Define a canonical runtime audience/provenance shape, then have gateway history, Control UI, and TUI either suppress agent-only events or render user-visible diagnostic events in a dedicated labeled system/introspection block.

## Reproduction Assessment

No high-confidence live reproduction was run for the original pre-compaction example, and that example is now narrowly fixed. Source inspection does show that generic `role: "system"` messages are not rendered as a dedicated system block, and the related WebChat continuation report remains open.

## Solution Assessment

No, a narrow close or CSS-only tweak is not the best solution. The maintainable path is to decide the runtime audience/provenance contract first, then render or suppress those events consistently across chat history surfaces.
