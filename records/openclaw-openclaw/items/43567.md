## Summary

Keep open: the narrow pre-compaction leak has been addressed separately, but current main still does not implement the requested distinct system/runtime message block; web chat mostly hides internal artifacts, and remaining `role: "system"` UI paths fall through generic rendering.

## What This Changes

Request rendering internal system/runtime messages in Control UI and TUI as clearly labeled, visually distinct blocks instead of normal chat bubbles.

## Best Possible Solution

Define a shared provenance/display contract: keep silent housekeeping hidden, and render intentionally visible `system`/runtime messages as a diagnostic System block in Control UI and TUI with focused regression coverage.

## Reproduction Assessment

No high-confidence live reproduction was run in this read-only review. Source inspection does show the remaining UI gap: `role: "system"` messages can enter the chat item stream and render through generic `other`/dim-text paths rather than a dedicated System block.

## Solution Assessment

No: current main mainly hides internal artifacts and only has generic system-message rendering, so it is not the requested solution. The safer path is a maintainer-approved display policy plus a narrow UI/TUI implementation for intentionally visible runtime messages.
