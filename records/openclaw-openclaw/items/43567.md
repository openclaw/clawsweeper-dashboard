## Summary

Keep open. Current main and the latest release have fixed the concrete pre-compaction memory-flush transcript leak, but the broader product contract for intentionally visible runtime/system messages is still unsettled, and the open WebChat continuation tracker shows the Gateway/UI audience boundary remains active.

## What This Changes

The issue requests rendering internal/runtime system messages separately from normal user and assistant chat bubbles so users do not mistake them for human-authored chat.

## Best Possible Solution

Define a canonical Gateway audience/provenance contract for runtime events, then have Control UI and TUI render intentionally visible runtime notices as labeled system/introspection blocks while keeping agent-only continuations out of human-authored chat.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows the current Gateway/UI lacks a structural visible system-message contract, and the related WebChat continuation tracker supplies current user-flow evidence for the remaining boundary.

## Solution Assessment

No, a UI-only colored bubble is not the best complete fix. The safer solution starts at the Gateway/session projection boundary so clients can distinguish user, assistant, tool, runtime-visible, and agent-only messages consistently.
