## Summary

Keep open. Current main and the latest release fix the concrete pre-compaction memory-flush prompt leak, but the broader request for a first-class runtime/system audience and distinct visible treatment is still unresolved across shared projection, Control UI, and related runtime-message paths.

## What This Changes

Request to render internal/runtime system messages in a dedicated visual treatment instead of normal user or assistant chat bubbles.

## Best Possible Solution

Define a shared runtime/system audience contract: agent-only internals stay out of visible history, while intentionally visible runtime notices render in a dedicated diagnostic/system treatment across Control UI/WebChat and TUI.

## Reproduction Assessment

No high-confidence live reproduction for the original pre-compaction leak was established on current main; https://github.com/openclaw/openclaw/pull/73380 made that maintenance prompt transcript-silent. Source inspection still shows the generic runtime/system display contract is missing.

## Solution Assessment

No, a UI-only colored block is not the best standalone fix. The maintainable solution should first decide which runtime messages are agent-only versus intentionally visible, then implement shared provenance filtering/projection and matching UI/TUI rendering.
