## Summary

Keep open. Current main fixes the concrete pre-compaction memory-flush leak and has a compaction divider, but it still lacks a first-class, shared system/runtime chat block across Control UI and TUI.

## What This Changes

Request to render visible internal/runtime messages in a clearly labeled distinct system block instead of normal chat bubbles.

## Best Possible Solution

Define a shared provenance/audience contract for runtime/system events, hide agent-only prompts before transcript/display projection, and render intentionally visible diagnostics as a consistent Control UI/TUI system block.

## Reproduction Assessment

Not applicable as a feature/UX request. Source inspection shows partial handling for compaction and memory flush, but no current contract that all visible runtime/system messages render as a distinct block.

## Solution Assessment

Unclear. A distinct block is likely right for intentional diagnostics, but the maintainable solution needs provenance and audience first so hidden internal turns are not solved by styling leaked prompts.
