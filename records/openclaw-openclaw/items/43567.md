## Summary

Keep open. Current main fixed the specific pre-compaction memory-flush prompt leak, but the broader Gateway/Control UI/TUI audience and provenance model for intentionally visible runtime/system messages is still unresolved, and a related live WebChat continuation report shows the remaining boundary is not settled.

## What This Changes

The issue requests that internal runtime/system messages be shown with a distinct non-user, non-assistant treatment instead of ordinary chat bubbles.

## Best Possible Solution

Define a canonical Gateway audience/provenance contract for runtime/system events, then have Control UI and TUI render visible runtime notices through a clearly labeled system/introspection block while keeping agent-only continuations out of human-authored chat.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only pass. Source inspection and the open related WebChat continuation report show the remaining audience/provenance gap, while the original pre-compaction leak itself is already fixed.

## Solution Assessment

No, a UI-only colored bubble is not the best complete fix. The safer solution starts at the Gateway/session projection boundary so Control UI and TUI can distinguish user, assistant, tool, runtime-visible, and agent-only messages consistently.
