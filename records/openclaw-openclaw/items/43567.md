## Summary

Keep open. Current main fixes the narrow pre-compaction memory-flush leak, but it still lacks a distinct system/runtime rendering path for intentionally visible system messages in Control UI and TUI history replay; #70348 is narrower artifact filtering, not a full replacement for this request.

## What This Changes

The issue asks for internal/runtime system messages to render as clearly distinct system-originated UI blocks instead of normal chat bubbles.

## Best Possible Solution

Define a structural display contract for intentional runtime/system notices, carry it through `chat.history`, render it with a dedicated Control UI system treatment, and route TUI history system rows through the existing system-notice path.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to project or render a visible `role: "system"` row on current main: Control UI falls through to generic `other` styling, and TUI history replay does not route ordinary system rows to `addSystem`.

## Solution Assessment

Unclear for an automated fix lane. Distinct rendering is the right end state, but maintainers should resolve hide-versus-label policy with the open #70348 filtering work before a cross-surface implementation.
