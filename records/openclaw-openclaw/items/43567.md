## Summary

Keep open. Current main fixed the concrete pre-compaction memory-flush leak by keeping that prompt runtime-only, but the broader request for a first-class system/runtime chat block is not implemented consistently across Control UI and TUI.

## What This Changes

The issue requests rendering internal runtime/system messages in a visually distinct chat treatment instead of normal user or assistant bubbles.

## Best Possible Solution

Define a first-class system/runtime chat item with explicit provenance from chat.history and render it consistently in Control UI and TUI, while leaving memory-flush prompts runtime-only.
