## Summary

Keep open: current main fixed the specific pre-compaction memory-flush transcript leak, but the broader runtime/system message display contract is still unresolved. Generic system events still have no audience/display field and drain as plain `System:` prompt text, while restart continuations still carry raw internal continuation text through normal inbound recording with only provenance metadata.

## What This Changes

Surface runtime/internal system messages in a distinct UI treatment instead of letting them appear as ordinary user or assistant chat.

## Best Possible Solution

Define one shared runtime/system audience or provenance contract, then have Gateway history projection and Control UI/TUI render the chosen hidden or labeled treatment consistently across surfaces.

## Solution Assessment

Unclear pending maintainer product/API direction. A shared provenance or audience contract with one display projection is safer than per-surface styling or string-prefix filters.
