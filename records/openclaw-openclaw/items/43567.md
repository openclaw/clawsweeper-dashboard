## Summary

Keep open. Current main fixes the concrete pre-compaction memory-flush leak and has a compaction-specific divider, but it still does not provide a shared Control UI/TUI runtime-system block or a settled audience contract for intentionally visible diagnostics.

## What This Changes

Request to render internal/runtime-originated messages in a distinct Control UI/TUI system block instead of normal chat bubbles.

## Best Possible Solution

Define a shared provenance/audience contract, hide agent-only runtime prompts before display projection, and render intentionally visible diagnostics through one consistent Control UI/TUI system block.

## Reproduction Assessment

Not applicable as a feature/UX request. Source inspection gives high confidence that current main lacks the requested shared block: Control UI has only a compaction-specific divider plus generic `other` styling, and TUI history has no shared system/runtime history projection.

## Solution Assessment

Unclear. A distinct block is plausible for intentional diagnostics, but the maintainable solution needs the provenance/audience boundary first so hidden internal prompts are not solved by merely styling leaked text.
