## Summary

Keep open: the concrete pre-compaction memory-flush example is fixed on current main, but the broader UX/API request is still unresolved because generic system-event and restart-continuation provenance is not routed through one shared hidden or distinct display treatment.

## What This Changes

The issue requests a distinct non-user/non-assistant UI treatment for internal runtime/system messages that can otherwise look like normal chat bubbles.

## Best Possible Solution

Define one shared runtime/system audience or provenance contract, then have Gateway history projection and Control UI render the chosen hidden or labeled treatment consistently across surfaces.

## Reproduction Assessment

Yes, at source level. Current main fixes the original pre-compaction prompt leak, but generic SystemEvent drains and restart continuations still lack a shared structural audience/display contract, and Control UI only special-cases compaction dividers.

## Solution Assessment

Unclear pending maintainer product/API direction. A shared provenance or audience contract with one display projection is safer than per-surface styling or string-prefix filters.
