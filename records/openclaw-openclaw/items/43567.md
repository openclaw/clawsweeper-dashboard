## Summary

Keep open. Current main fixes the narrow pre-compaction memory-flush leak, but it still lacks a distinct structural display class and rendering treatment for intentionally visible runtime/system messages across chat.history projection, Control UI grouping, and TUI history replay; the open #70348 path is a narrower artifact-filtering PR, not a complete replacement for this request.

## What This Changes

Request to render internal/runtime system messages as clearly system-originated blocks instead of normal chat bubbles.

## Best Possible Solution

Define a structural display contract for intentional runtime/system notices, carry it through `chat.history`, render it with a dedicated Control UI system treatment, and make TUI live/history display route those rows through the existing system-notice path while continuing to suppress artifacts that should never be user-visible.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by projecting or rendering a visible `role: "system"` row on current main: Control UI falls through to generic `other` styling, and TUI history replay does not route ordinary system rows to `addSystem`.

## Solution Assessment

Unclear for an automated fix lane. Distinct rendering is the right end state, but maintainers should resolve the hide-versus-label policy with the open #70348 filtering work before a cross-surface implementation.
