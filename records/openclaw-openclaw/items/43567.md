## Summary

Keep open: the concrete pre-compaction memory-flush leak was fixed, but current main still lacks a generic runtime/system provenance contract and Control UI/TUI treatment that hides or clearly separates internal messages. The open related tracks at https://github.com/openclaw/openclaw/issues/69492, https://github.com/openclaw/openclaw/pull/72201, and https://github.com/openclaw/openclaw/issues/78344 show this still needs maintainer UX/API direction rather than cleanup closure.

## What This Changes

Render internal/runtime system messages separately from normal user and assistant chat bubbles so they are visibly system-originated.

## Best Possible Solution

Define one shared provenance/audience contract for runtime-internal input, then make Gateway history projection and Control UI/TUI rendering either hide internal rows or show intentionally visible rows as a labeled runtime/system block.

## Reproduction Assessment

Yes, at source level: current main still has generic SystemEvent formatting and restart-continuation paths without a shared internal/system display branch. I did not run a live Control UI or TUI reproduction in this read-only review.

## Solution Assessment

Unclear pending maintainer product direction. A structural provenance/audience contract coordinated with the UI projection is safer than per-surface styling or text-prefix filters.
