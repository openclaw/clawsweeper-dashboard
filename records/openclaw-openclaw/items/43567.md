## Summary

Keep open: current main fixed the narrow pre-compaction memory-flush transcript leak, but the broader request still lacks a shared runtime/system audience contract and a Control UI/TUI projection that renders generic internal system messages as distinct blocks.

## What This Changes

The issue requests rendering internal system/runtime messages as labeled, visually distinct UI blocks instead of ordinary user or assistant chat bubbles.

## Best Possible Solution

Define one shared runtime/system audience or provenance contract, then project it consistently through Gateway history and Control UI/TUI as hidden internal context or a labeled system block according to maintainer UX policy.

## Reproduction Assessment

Yes, at source level. Current main still formats generic queued SystemEvent entries as `System:` prompt text, and Control UI has no generic runtime/system ChatItem or projection branch beyond the compaction divider.

## Solution Assessment

Unclear pending maintainer UX/API direction. A shared audience/provenance contract plus one display projection is safer than a per-surface string filter or a narrow styling patch that only recognizes the pre-compaction text.
