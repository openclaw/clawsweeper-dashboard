## Summary

Keep open. Current main fixed the concrete pre-compaction memory-flush transcript leak and has a special compaction divider, but it does not yet provide a first-class, consistent system/runtime chat block across Control UI and TUI.

## What This Changes

Request that internal system/runtime messages render as clearly system-originated UI blocks instead of normal user or assistant chat bubbles.

## Best Possible Solution

Define explicit chat display provenance/audience for runtime/system events, hide agent-only prompts, and render intentionally visible diagnostics consistently in Control UI and TUI.

## Reproduction Assessment

Not applicable as a feature/UX request. Source inspection shows partial handling for compaction and memory flush, but no defined current contract that every internal system message must render as a distinct block.

## Solution Assessment

Unclear. A distinct block may be right for intentional diagnostics, but the maintainable solution needs a provenance/audience contract first so hidden internal turns are not solved by ad hoc UI text matching.
