## Summary

Keep open: current main fixes the concrete pre-compaction memory-flush transcript leak, but it still lacks a shared audience/display contract for runtime-originated content, and restart continuations can still be persisted as normal user-body text with only internal provenance metadata.

## What This Changes

The issue requests rendering runtime/internal system-originated messages as clearly distinct system UI blocks instead of normal user or assistant chat bubbles.

## Best Possible Solution

Define one shared runtime/system audience or provenance contract, then have Gateway history projection and Control UI/TUI render the chosen hidden or labeled treatment consistently across surfaces.

## Reproduction Assessment

Yes, at source level. Generic SystemEvent entries still use plain System-prefixed prompt text without audience/display metadata, and restart continuations carry internal_system provenance while preserving raw continuation text in normal body fields.

## Solution Assessment

Unclear pending maintainer product/API direction. A shared provenance or audience contract with one display projection is safer than per-surface styling or string-prefix filters.
