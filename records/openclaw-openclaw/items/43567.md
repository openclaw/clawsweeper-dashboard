## Summary

The original memory-flush leak is fixed, but current main still lacks a clear, consistently rendered system/runtime authorship lane for intentionally visible events, so the broader product decision remains necessary.

## What This Changes

Render runtime-originated conversation entries distinctly from user and assistant chat bubbles.

## Best Possible Solution

Define typed provenance and audience at the transcript/gateway boundary, omit agent-only prompts, and render intentionally visible runtime events as a labeled, muted system block consistently in Control UI and TUI.

## Reproduction Assessment

Yes at source level for the remaining presentation gap: Control UI recognizes a `system` role but does not consistently label non-user content as system-originated. The issue’s original pre-compaction example no longer reproduces because it is now excluded from visible history.

## Solution Assessment

Yes, with an important boundary refinement: distinct styling is the right treatment for intentionally visible runtime events, while internal prompts such as memory flush must stay out of visible history entirely.
