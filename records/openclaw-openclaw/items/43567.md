## Summary

Keep open. Current main fixes the concrete pre-compaction memory-flush prompt leak, but the broader Control UI/Gateway audience and provenance treatment is still missing and an open WebChat continuation issue shows the remaining product boundary is not settled.

## What This Changes

The issue requests a distinct UI treatment for internal runtime/system messages so they are not rendered like normal user or assistant chat bubbles.

## Best Possible Solution

Define one runtime/system audience and provenance contract, then have Gateway projection and Control UI/TUI render intentionally visible system rows in a dedicated diagnostic block while keeping agent-only prompts out of the visible transcript or dispatching them as agent turns.

## Reproduction Assessment

No, not as a live current-main repro. Source inspection shows current Control UI lacks a dedicated system/runtime block for role:"system" rows, and the related WebChat issue provides a concrete restart-continuation symptom that still needs live confirmation.

## Solution Assessment

No, a simple colored chat bubble is only a partial fix. The maintainable solution should first settle the audience/provenance model so agent-only continuation prompts and intentionally visible system events are handled differently.
