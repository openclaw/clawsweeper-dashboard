## Summary

## What This Changes

Render runtime/internal system messages in a visibly distinct system/runtime treatment rather than normal user or assistant chat bubbles.

## Best Possible Solution

Define one shared runtime/system audience or provenance contract, then have Gateway history projection and Control UI/TUI render the chosen hidden or labeled treatment consistently across surfaces.

## Reproduction Assessment

Yes, at source level. Generic SystemEvent entries still drain as plain `System:` text and restart continuations still carry raw `internal_system` input through normal body fields; I did not run a live UI/TUI session.

## Solution Assessment

Unclear pending maintainer product/API direction. A shared provenance or audience contract with one display projection is safer than per-surface styling or string-prefix filters.
