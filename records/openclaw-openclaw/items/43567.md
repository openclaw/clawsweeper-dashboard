## Summary

Keep open: current main fixes the narrow pre-compaction memory-flush transcript leak, but it still lacks a general runtime/system audience contract and UI projection that would render system-originated items as a distinct non-user block.

## What This Changes

The issue asks OpenClaw to show runtime/internal system messages in a clearly distinct UI treatment instead of normal user or assistant chat bubbles.

## Best Possible Solution

Define one shared runtime/system audience or provenance contract, then project it consistently through gateway history and Control UI/TUI as hidden internal context or a labeled system block according to maintainer UX policy.

## Reproduction Assessment

Yes, at source level. Current main still flattens generic SystemEvent entries into `System:` prompt text and the Control UI has no generic runtime/system ChatItem or projection for `internal_system` provenance.

## Solution Assessment

Unclear pending maintainer product/API direction. A shared audience/provenance contract plus one display projection is safer than a per-surface string filter or a narrow styling patch that only recognizes the pre-compaction text.
