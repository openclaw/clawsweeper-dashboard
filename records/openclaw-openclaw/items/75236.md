## Summary

Keep open. Current `main` still lacks the requested cursor fields on both Gateway RPC surfaces, and the existing `sessions.list` limit/yield optimizations do not provide resumable pagination metadata.

## What This Changes

Request adding cursor-style pagination fields to Gateway `agents.files.list` and `sessions.list` so SDK and agent consumers can page large result sets consistently.

## Best Possible Solution

Define an additive cursor contract for `sessions.list` across schema, runtime, SDK/client types, docs, generated protocol artifacts, and tests; extend `agents.files.list` only if maintainers confirm that fixed-scope file listing should share the same API shape.

## Reproduction Assessment

Yes. Source inspection is enough: strict validators omit `cursor`, and current responses do not expose `nextCursor` or a pre-window `total` for either requested RPC.

## Solution Assessment

Unclear for the full requested scope. Cursor pagination is a plausible maintainable direction for `sessions.list`, but `agents.files.list` and the unmerged artifact precedent need protocol/product confirmation first.
