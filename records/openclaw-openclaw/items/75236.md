## Summary

Close: current main now solves the central unbounded-list problem. `sessions.list` has bounded `limit`/`offset` pagination with `totalCount`, `nextOffset`, and `hasMore`, ACP exposes opaque cursors on top of that Gateway list, and `agents.files.list` is a fixed bootstrap/workspace-file surface rather than an unbounded workspace scan.

## What This Changes

The issue requests cursor-style pagination fields for Gateway `sessions.list` and `agents.files.list` so large session or file sets can be paged efficiently.

## Best Possible Solution

Keep the shipped bounded `sessions.list` pagination contract and the fixed-scope `agents.files.list` contract; consider a new issue only if a future full workspace-file browser API needs pagination.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection verifies current main has the relevant `sessions.list` pagination contract and that `agents.files.list` is not an unbounded file enumeration surface.

Yes. The current implementation solves the user-facing scalability problem with an additive Gateway `limit`/`offset` contract while keeping `agents.files.list` narrow instead of adding unnecessary pagination to a fixed file list.
