## Summary

Keep open. Current main still lacks cursor pagination metadata on `sessions.list`, and `agents.files.list` still exposes a fixed bootstrap-file list; implementing the request would be additive Gateway protocol work that needs maintainer product/protocol judgment, especially because the referenced artifact cursor precedent is still an open PR rather than current-main behavior.

## What This Changes

Add cursor-based pagination metadata to the Gateway `agents.files.list` and `sessions.list` RPCs, following the artifact-list pattern proposed in #75208.

## Best Possible Solution

Define an additive Gateway pagination contract for `sessions.list`; extend `agents.files.list` only if maintainers want bootstrap-file listing to share that contract, then update schemas, runtime handlers, SDK types, generated protocol outputs, docs, and focused tests together.

## Reproduction Assessment

Yes. Source inspection against current main reproduces the API-shape gap: strict validators omit `cursor`, and current responses do not expose `nextCursor` or a pre-slice `total`.

## Solution Assessment

Unclear. Cursor pagination is plausible for `sessions.list`, but the artifact cursor precedent is not on main and `agents.files.list` is fixed-scope today, so the narrow maintainable fix needs a maintainer-approved protocol shape first.
