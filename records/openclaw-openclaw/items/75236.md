## Summary

Keep open. Current main still lacks cursor pagination on both requested Gateway RPC surfaces, and the referenced artifact cursor helper is not present on main because the related artifact cursor PR remains open/unmerged in the supplied context.

## What This Changes

The issue requests additive `cursor`, `limit`, `nextCursor`, and `total` support for `agents.files.list` and `sessions.list`.

## Best Possible Solution

Define an additive cursor contract for `sessions.list` across schema, runtime, SDK/native models, docs, and tests; for `agents.files.list`, either add compatible metadata deliberately or document that the fixed bootstrap-file surface is intentionally unpaginated.

## Reproduction Assessment

Yes. Source inspection is enough to reproduce the API-shape gap: both strict validators omit `cursor`, and current responses do not expose `nextCursor` or a pre-page `total`.

## Solution Assessment

Unclear for the full requested scope. Cursor pagination is a maintainable additive direction for `sessions.list`, but applying it to fixed-scope `agents.files.list` should wait for maintainer confirmation and should not depend on #75208 until that cursor contract lands or is replaced.
