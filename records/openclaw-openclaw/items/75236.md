## Summary

Keep open. Current main still lacks cursor fields and pagination metadata on both requested Gateway RPC surfaces, and the referenced artifact cursor work is still an open, unmerged PR rather than a shared contract on main.

## What This Changes

Add additive cursor pagination fields to `agents.files.list` and `sessions.list`, following the pending artifact pagination pattern.

## Best Possible Solution

Define an additive, documented Gateway pagination contract for `sessions.list`, and extend `agents.files.list` only if maintainers confirm that fixed-scope file listing should share that contract.

## Reproduction Assessment

Yes. Source inspection is enough: strict validators reject `cursor`, current responses lack `nextCursor` and `total`, and the existing `limit` path slices results rather than exposing cursor pagination.

## Solution Assessment

Unclear for the full requested scope. Cursor pagination is a maintainable direction for `sessions.list`, but copying the unmerged artifact pattern into `agents.files.list` should wait for maintainer protocol judgment.
