## Summary

Keep open: current main still lacks the requested cursor/nextCursor pagination contract for both `sessions.list` and `agents.files.list`; the related merged sessions work deliberately shipped a bounded limit/metadata contract instead of resumable pagination.

## What This Changes

Request cursor-based pagination fields for the Gateway `agents.files.list` and `sessions.list` RPC params/results.

## Best Possible Solution

Define an additive Gateway pagination contract for `sessions.list` across schema, SDK, docs, generated clients, and tests, and extend `agents.files.list` only if maintainers decide that file-list surface should become pageable.

## Reproduction Assessment

Not applicable as a bug reproduction; source inspection gives high-confidence proof of the missing feature because current main has no `cursor` or `nextCursor` fields on either requested RPC.

## Solution Assessment

Unclear. Cursor pagination is plausible for `sessions.list`, but current main deliberately chose bounded limit metadata, and `agents.files.list` may not need a pagination contract unless its scope expands.
