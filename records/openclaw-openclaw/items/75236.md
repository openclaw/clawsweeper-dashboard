## Summary

Keep open. Current main still lacks cursor pagination fields on both requested Gateway RPC surfaces, and the referenced artifact cursor precedent is not present on main, so this is real remaining feature work rather than cleanup-close material.

## What This Changes

Add cursor-style pagination metadata and parameters to `agents.files.list` and `sessions.list` Gateway RPCs using the artifact pagination pattern.

## Best Possible Solution

Define an additive Gateway pagination contract for `sessions.list` across schemas, runtime, SDK/docs, generated clients, and tests; extend `agents.files.list` only if maintainers confirm that fixed bootstrap-file listing should share the contract.

## Reproduction Assessment

Yes. Source inspection on current main is enough to reproduce the API-shape gap: strict validators omit `cursor`, and current responses do not expose `nextCursor` or a pre-slice `total`.

## Solution Assessment

Unclear. Cursor pagination is plausible for `sessions.list`, but `agents.files.list` is fixed-scope today and the referenced artifact cursor helper is not on main, so maintainers should settle the additive protocol contract first.
