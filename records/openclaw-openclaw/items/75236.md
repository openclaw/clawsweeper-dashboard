## Summary

Keep open: current main still lacks the requested cursor/nextCursor pagination contract for both `sessions.list` and `agents.files.list`, and the recent sessions work intentionally added bounded limit metadata rather than resumable cursors.

## What This Changes

Add cursor-based pagination parameters and result metadata to Gateway `agents.files.list` and `sessions.list` for consistent large-result handling.

## Best Possible Solution

Define a stable additive Gateway pagination contract for `sessions.list`, update SDK/docs/tests with that contract, and extend `agents.files.list` only if maintainers decide that fixed bootstrap-file list should paginate.

## Reproduction Assessment

Not applicable as a bug reproduction, but source inspection gives high-confidence proof of the missing feature: current main has no `cursor`/`nextCursor` fields on either requested RPC.

## Solution Assessment

Unclear. Cursor pagination is plausible for `sessions.list`, but https://github.com/openclaw/openclaw/pull/77123 deliberately stopped at limit metadata, so maintainers should choose the protocol direction before implementation.
