## Summary

Keep open. Current main still lacks cursor pagination on both requested Gateway RPC surfaces; recent sessions-list performance work bounds and yields some work, but it does not add `cursor`, `nextCursor`, or `total` protocol fields.

## What This Changes

Add additive cursor pagination fields to `agents.files.list` and `sessions.list`, following the related artifact pagination proposal.

## Best Possible Solution

Define an additive, documented cursor contract for `sessions.list` across schema, runtime, SDK/client models, generated clients, docs, and tests; only extend `agents.files.list` if maintainers confirm that fixed-scope file listing should share the same contract.

## Reproduction Assessment

Yes. Source inspection is enough to reproduce the API-shape gap: strict validators omit `cursor`, and current responses do not expose `nextCursor` or a pre-slice `total`.

## Solution Assessment

Unclear for the full requested scope. Cursor pagination is a maintainable direction for `sessions.list`, but `agents.files.list` should wait for maintainer confirmation because it returns a fixed bootstrap-file set today.
