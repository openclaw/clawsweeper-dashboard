## Summary

Keep open. Current main still lacks cursor pagination on both requested Gateway RPC surfaces, and the referenced artifact cursor work is still represented by an open related PR rather than an implemented shared contract on main.

## What This Changes

Add cursor pagination fields and behavior to `agents.files.list` and `sessions.list` for consistent bounded Gateway list RPCs.

## Best Possible Solution

Define an additive cursor contract for `sessions.list` across schema, runtime, SDK/client models, Swift generation, docs, and tests; separately decide whether `agents.files.list` should gain compatible pagination metadata or be documented as intentionally fixed-scope.

## Reproduction Assessment

Yes. Source inspection is enough to reproduce the API-shape gap: strict validators omit `cursor`, and current responses do not expose `nextCursor` or a pre-slice `total`.

## Solution Assessment

Unclear for the full requested scope. Cursor pagination is a maintainable direction for `sessions.list`, but `agents.files.list` should wait for maintainer confirmation because it returns a small fixed bootstrap-file set today.
