## Summary

Keep open: current main still lacks the requested cursor/nextCursor contract; `sessions.list` is bounded with limit/offset metadata and `agents.files.list` is a fixed bootstrap-file list, so the remaining work is an API-direction decision rather than a bugfix.

## What This Changes

Requests cursor-style pagination fields for Gateway `sessions.list` and `agents.files.list`, following the artifact-list pattern proposed in a related open PR.

## Best Possible Solution

Have maintainers choose the Gateway list pagination contract first: standardize `sessions.list` on cursor/nextCursor with SDK, docs, and tests, and only page `agents.files.list` if that fixed file surface is meant to grow.

## Reproduction Assessment

No bug reproduction is needed; source inspection confirms current main has `limit`/`offset` for `sessions.list` and only `agentId` for `agents.files.list`, with no cursor/nextCursor contract. The missing feature is therefore reproducible from the schemas and handlers.

## Solution Assessment

Unclear. Cursor pagination is plausible for `sessions.list`, but current main deliberately uses offset metadata, and `agents.files.list` may not need pagination while it returns a fixed bootstrap-file set.
