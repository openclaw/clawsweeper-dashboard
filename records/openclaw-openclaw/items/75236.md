## Summary

Keep open. Current main still lacks the requested cursor pagination on both Gateway RPC surfaces, and the requested pattern depends on protocol scoping because the cited artifact cursor helper is only in open PR #75208.

## What This Changes

Request cursor, limit, nextCursor, and total support on the Gateway agents.files.list and sessions.list RPCs, following the artifact pagination pattern proposed in #75208.

## Best Possible Solution

Define an additive, documented cursor contract for sessions.list across schema, runtime, SDK/client types, generated clients, docs, and tests; extend agents.files.list only if maintainers confirm that bootstrap file listing should share that paginated shape.

## Reproduction Assessment

Yes. Source inspection against origin/main reproduces the API-shape gap: strict validators omit cursor, and current responses do not expose nextCursor or a pre-slice total.

## Solution Assessment

Unclear for the full requested scope. Cursor pagination is a maintainable direction for sessions.list, but agents.files.list should wait for maintainer confirmation because it returns a fixed bootstrap-file set today and the artifact cursor pattern is not merged.
