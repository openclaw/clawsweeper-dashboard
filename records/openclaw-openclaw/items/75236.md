## Summary

Keep open. Current main still lacks cursor pagination on both requested Gateway RPC surfaces, and the referenced artifact cursor precedent is not present on main because the related artifact PR remains open/unmerged.

## What This Changes

Add cursor pagination fields to Gateway `agents.files.list` and `sessions.list` so callers can page large file or session result sets.

## Best Possible Solution

Define an additive Gateway pagination contract, likely starting with `sessions.list`; extend `agents.files.list` only if maintainers confirm the fixed workspace-file listing should be pageable, then update schemas, runtime, SDK types/docs, generated protocol artifacts, and regression tests together.

## Reproduction Assessment

Yes. Source inspection on current main is enough to verify the API-shape gap: strict schemas omit `cursor`, and the runtime responses omit `nextCursor` and `total` pagination metadata.

## Solution Assessment

Unclear. Cursor pagination is plausible for `sessions.list`, but current main lacks the referenced artifact cursor helper/pattern and `agents.files.list` is fixed-scope today, so maintainers should settle the additive protocol contract first.
