## Summary

Keep open: current main partially bounds `sessions.list`, but it still does not provide the requested cursor/`nextCursor` contract, and `agents.files.list` remains a fixed bootstrap-file listing.

## What This Changes

Request cursor pagination fields for Gateway `sessions.list` and `agents.files.list`, following the artifact cursor pattern proposed in a related PR.

## Best Possible Solution

Define an additive Gateway pagination contract for `sessions.list`, reconcile it with existing ACP/task cursor semantics, and extend `agents.files.list` only if maintainers decide that fixed bootstrap-file surface should paginate.

## Reproduction Assessment

Yes. Source inspection of current main shows strict validators and handlers omit `cursor`/`nextCursor` on the requested RPCs; `sessions.list` has limit metadata only and `agents.files.list` accepts only `agentId`.

## Solution Assessment

Unclear. Cursor pagination is plausible for `sessions.list`, but current main already chose bounded limit metadata and `agents.files.list` is fixed-scope today, so maintainers should settle the protocol direction first.
