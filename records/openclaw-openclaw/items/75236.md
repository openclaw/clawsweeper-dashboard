## Summary

Keep open. Current main still lacks cursor pagination on both requested Gateway RPC surfaces, and the related artifact cursor contract remains outside main, so this needs maintainer API direction rather than cleanup closure.

## What This Changes

The issue requests additive cursor, limit, nextCursor, and total support for `agents.files.list` and `sessions.list`.

## Best Possible Solution

Define a maintainer-approved additive cursor contract for `sessions.list` across schema, runtime, SDK/client types, generated models, docs, and tests; separately decide whether `agents.files.list` should gain compatible pagination metadata or be documented as intentionally unpaginated.

## Reproduction Assessment

Yes. Source inspection is enough: strict validators omit `cursor`, and current responses omit `nextCursor` and a pre-slice `total`; a cursor-bearing request would not match the current schemas.

## Solution Assessment

Unclear for the full requested scope. Cursor pagination is a plausible maintainable direction for `sessions.list`, but applying it to `agents.files.list` should wait for maintainer confirmation because that method is fixed-scope today and the referenced artifact cursor helper is not on main.
