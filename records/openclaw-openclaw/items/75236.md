## Summary

Keep open. Current main still lacks cursor parameters and pagination result metadata on both requested Gateway RPC surfaces, so the requested API consistency work remains valid and unimplemented.

## What This Changes

The issue asks to add cursor pagination fields such as cursor, limit, nextCursor, and total to Gateway agents.files.list and sessions.list, following the artifact-list pagination pattern proposed in #75208.

## Best Possible Solution

Define an additive pagination contract for sessions.list across protocol schema, runtime behavior, SDK/client types, generated Swift models, docs, and focused tests. For agents.files.list, make an explicit API decision: either add compatible pagination metadata for consistency or document that the fixed bootstrap-file surface is intentionally unpaginated.

## Reproduction Assessment

Yes. Source inspection against current main is enough: the strict validators omit cursor, and both handlers return arrays without nextCursor or a pre-slice total.

## Solution Assessment

Unclear for the full requested scope. Cursor pagination is a reasonable additive direction for sessions.list, but applying the same contract to agents.files.list should wait for maintainer confirmation because that RPC is fixed-scope today.
