## Summary

Keep this issue open. Current main still supports plaintext SKILL.md discovery, config-scoped skill entries, session skill snapshots, and install-time dangerous-code scanning, but it does not implement a signed/encrypted commercial Markdown wrapper or a public persist/trustedSources skill trust contract; open PR #74035 remains the unmerged implementation candidate and uses closing syntax for this issue.

## What This Changes

The issue requests persistent commercial skill configuration support, including signed or encrypted Markdown wrapping and trusted commercial-use skill handling that does not block legitimate commercial scenarios.

## Best Possible Solution

Keep the issue open while maintainers review, revise, land, or close PR #74035, or define a narrower signed/private ClawHub skill-bundle design that preserves readable operator review, provenance, scan auditability, and documented commercial trust boundaries.

## Reproduction Assessment

Not applicable as a feature and product/security policy request. Source inspection gives a high-confidence check of current plaintext skill loading and snapshot behavior, but the report does not provide concrete steps for reproducing skills being forgotten or ignored.

## Solution Assessment

Unclear. A paid encrypted wrapper or trusted-source scan bypass is broader than a narrow skill persistence fix; the safer maintainable path is explicit maintainer product/security review of PR #74035 or a separate signed/private bundle design before treating this as the right fix.
