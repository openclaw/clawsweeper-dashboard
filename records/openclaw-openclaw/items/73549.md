## Summary

Keep this issue open. Current main supports plaintext skill discovery, skill snapshots, watcher/config invalidation, and install-time scan blocking, but it does not implement the requested signed/encrypted Markdown wrapper, commercial trusted-source policy, or `persist` / `trustedSources` skill config surface. PR #74035 remains the open implementation candidate with closing syntax, so this issue should not be closed before that PR or an equivalent maintainer-approved design lands.

## What This Changes

Request persistent commercial skill support, including a paid signed or encrypted Markdown wrapper and trusted-source policy so commercial skills remain active and protect intellectual property.

## Best Possible Solution

Keep this issue open while maintainers review, revise, land, or close PR #74035, or define a narrower signed/private ClawHub skill-bundle design that preserves readable operator review, provenance, scan auditability, and documented commercial trust boundaries.

## Reproduction Assessment

Not applicable as a feature and product/security policy request. Source inspection gives a high-confidence check of current plaintext skill loading and snapshot behavior, but the report does not include concrete steps to reproduce skills being forgotten or ignored.

## Solution Assessment

Unclear. A paid encrypted wrapper or trusted-source scan bypass is broader than a narrow persistence fix; the safer maintainable path is maintainer product/security review of PR #74035 or an explicit signed/private ClawHub bundle design before treating this as the right fix.
