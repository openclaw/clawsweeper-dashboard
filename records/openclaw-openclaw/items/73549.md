## Summary

Keep this issue open. Current main supports plaintext skill discovery, eligible-skill snapshots, watcher/config refresh, and install-time scan blocking, but it does not implement the requested signed or encrypted Markdown wrapper, commercial trusted-source policy, or `persist` / `trustedSources` skill config surface; PR #74035 remains the open implementation candidate with closing syntax.

## What This Changes

The issue requests persistent skill/.md configuration for commercial use, including a paid signed or encrypted Markdown wrapper and trust policy so commercial skill bundles can remain active without being blocked by current security review.

## Best Possible Solution

Keep the issue open while maintainers review, revise, land, or close PR #74035, or define a narrower signed/private ClawHub skill-bundle design that preserves readable operator review, provenance, scan auditability, and documented commercial trust boundaries.

Not applicable as a feature and product/security policy request. Source and docs inspection gives a high-confidence check of current plaintext skill loading and snapshot behavior, but the report does not provide concrete steps for the claimed forgotten or ignored skills.
