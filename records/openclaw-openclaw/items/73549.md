## Summary

Keep this issue open. Current main still lacks the requested `persist`, `trustedSources`, signing, encryption, or commercial wrapper surface, and PR #74035 is an open unmerged closing implementation candidate that should be reviewed rather than pre-closing the issue.

## What This Changes

The issue asks for persistent skill configuration plus a commercial signed/encrypted Markdown wrapper and trusted-source path for commercial skill use.

## Best Possible Solution

Keep this issue open while maintainers review, land, revise, or close PR #74035, or define a narrower signed/private ClawHub bundle design that preserves readable review, provenance, scan auditability, and explicit trust boundaries.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable for the commercial wrapper feature request; no for the skill-forgetting claim because the issue lacks reset/session steps. Source inspection does verify that the requested config and wrapper surfaces are absent on current main.

## Solution Assessment

Is this the best way to solve the issue? Unclear. A paid encrypted wrapper or configurable trusted-source scan bypass is broader than a persistence fix; the safer path is maintainer product/security review of PR #74035 or an explicit signed/private ClawHub bundle design.
