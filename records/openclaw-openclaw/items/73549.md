## Summary

Keep this issue open. Current main still only supports readable plaintext skill folders, snapshot refresh, runtime eligibility gates, and scan-gated installer metadata; it does not implement signed/encrypted Markdown wrappers, `persist`, `trustedSources`, or a commercial trust policy, and open PR #74035 uses closing syntax for this issue.

## What This Changes

The issue requests persistent commercial skill configuration plus official signed or encrypted Markdown skill wrapping/trusted-source handling.

## Best Possible Solution

Keep this issue open while maintainers review, revise, land, or close PR #74035, or define a narrower signed/private ClawHub bundle design that preserves readable review, provenance, scan auditability, and operator choice.

## Reproduction Assessment

Not applicable for the commercial wrapper feature request. There is no high-confidence reproduction for skills being forgotten or ignored because the report lacks reset/session steps, but source inspection verifies current snapshot behavior and the missing requested config surface.

## Solution Assessment

Unclear. A paid encrypted wrapper or trusted-source scan bypass is broader than a persistence fix, so the maintainable path is explicit maintainer product/security review of PR #74035 or a narrower signed/private ClawHub bundle design.
