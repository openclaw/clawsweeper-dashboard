## Summary

Keep this issue open. Current main supports plaintext SKILL.md discovery, gating, watcher-based refresh, and persisted per-session skill snapshots, but it does not implement a persist flag, trustedSources policy, signed/encrypted Markdown wrapper, or commercial trust model; PR #74035 is also still the open implementation candidate using closing syntax.

## What This Changes

The issue requests persistent skill configuration plus a commercial signed or encrypted Markdown wrapper and trusted-source policy for skills.

## Best Possible Solution

Keep the issue open while maintainers review PR #74035 or define a narrower signed/private ClawHub skill-bundle design that preserves operator inspection, provenance, dangerous-code scan auditability, and documented commercial trust boundaries.

## Reproduction Assessment

Not applicable as a feature and product/security policy request. Source inspection gives a high-confidence check of current plaintext skill loading and snapshot behavior, but the report does not provide concrete steps to reproduce skills being forgotten or ignored.

## Solution Assessment

Unclear. A paid encrypted wrapper or trusted-source scan bypass is broader than a narrow persistence fix; the safer maintainable path is maintainer product/security review of PR #74035 or an explicit signed/private ClawHub bundle design.
