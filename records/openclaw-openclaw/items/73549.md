Keep this issue open. Current main still lacks signed/encrypted Markdown wrappers, `persist` or `trustedSources` skill config, and a commercial trust policy, while open PR #74035 is already the linked implementation candidate for the same work.

Request persistent skill handling plus commercial/private SKILL.md wrapping or trust controls for SKILL.md-based skills.

Unclear. A signed/private ClawHub bundle or a narrow persistence fix may be safer than a broad trusted-source scan bypass, so maintainer product and security review of #74035 is the right path.
