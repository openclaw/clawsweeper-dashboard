## Summary

Keep this issue open. Current main still supports plaintext skill loading, snapshots, and scan-gated installs, but it does not implement signed/encrypted Markdown wrappers, `persist`, `trustedSources`, or a commercial trust policy; open PR #74035 remains the linked implementation candidate.

## What This Changes

Request persistent skill/.md support for commercial use, including a paid signed/encrypted wrapper and trusted commercial-source policy.

## Best Possible Solution

Keep this issue open while maintainers review, land, revise, or close #74035, or define a narrower signed/private ClawHub skill-bundle design that preserves readable review, provenance, scan auditability, and documented trust boundaries.

## Reproduction Assessment

Not applicable for the commercial wrapper feature request. There is no high-confidence reproduction for the reported skills being forgotten or ignored because the issue lacks reset/session steps, but source inspection verifies current plaintext snapshot behavior and the missing requested config surface.

## Solution Assessment

Unclear. A paid encrypted wrapper or trusted-source scan bypass is broader than a persistence fix; the safer path is maintainer product/security review of #74035 or an explicit signed/private ClawHub bundle design.
