## Summary

Keep this issue open. Current main still supports plaintext, scan-gated skill loading and does not implement signed/encrypted Markdown wrappers, `persist`, or a configurable commercial `trustedSources` policy; open PR #74035 remains the linked implementation candidate.

## What This Changes

The issue requests persistent commercial skill configuration plus signed or encrypted Markdown wrapping and trust-policy support for paid use cases.

## Best Possible Solution

Keep the issue open while maintainers review, land, revise, or close the linked PR, or define a narrower signed/private skill-bundle design that preserves readable auditability, provenance, scan coverage, and explicit operator trust boundaries.

## Reproduction Assessment

Not applicable for the commercial wrapper feature request. There is no high-confidence reproduction path for the reported skill-forgetting behavior because the issue lacks reset/session steps, but source and docs inspection verifies the missing requested config surfaces.

## Solution Assessment

Unclear. A paid encrypted wrapper or operator-configured scan bypass is broader than a persistence fix, so the safer path is explicit product and security review of the linked PR or a narrower private skill-bundle design.
