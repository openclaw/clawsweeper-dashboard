## Summary

Keep this issue open. Current main still has plaintext, eligibility-gated skill loading and a narrow official ClawHub trust path, but it does not implement signed/encrypted Markdown wrappers, `persist`, or user-configured `trustedSources`; open PR #74035 remains the linked implementation candidate.

## What This Changes

The issue asks for persistent commercial skill support, including signed/encrypted Markdown wrappers, proprietary-skill protection, and less restrictive trust/security handling.

## Best Possible Solution

Keep this issue open while maintainers review, revise, land, or close #74035, or define a narrower signed/private ClawHub bundle design that preserves readable review, provenance, scan auditability, and documented trust boundaries.

## Reproduction Assessment

Not applicable for the commercial wrapper feature request. There is no high-confidence reproduction for the reported skills being forgotten or ignored because reset/session steps are missing, but source and docs inspection verifies the current plaintext snapshot behavior and missing config surface.

## Solution Assessment

Unclear. A paid encrypted wrapper or trusted-source scan bypass is broader than a persistence fix; the safer path is maintainer product/security review of #74035 or an explicit signed/private ClawHub bundle design.
