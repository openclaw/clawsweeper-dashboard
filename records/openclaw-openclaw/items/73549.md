## Summary

Keep this issue open. Current main already has plaintext skill discovery, per-session skill snapshots, watcher/config invalidation, and install-time scan blocking, but it does not implement a signed/encrypted commercial Markdown wrapper, a trusted commercial-source policy, or the proposed `persist` / `trustedSources` config surface; the linked implementation PR remains open and unmerged.

## What This Changes

The issue requests persistent commercial skill configuration, including a paid signed or encrypted Markdown wrapper/trust policy and less restrictive handling for legitimate commercial skill use.

## Best Possible Solution

Keep this issue open while maintainers review, revise, land, or close PR #74035, or define a narrower signed/private ClawHub skill-bundle design that preserves readable operator review, provenance, scan auditability, and documented commercial trust boundaries.

## Reproduction Assessment

Not applicable as a feature and product/security policy request. Source inspection gives a high-confidence check of current plaintext skill loading and snapshot behavior, but the report does not provide concrete steps to reproduce skills being forgotten or ignored.

## Solution Assessment

Unclear. A paid encrypted wrapper or trusted-source scan bypass is broader than a narrow persistence fix; the safer maintainable path is maintainer product/security review of PR #74035 or an explicit signed/private ClawHub bundle design before treating this as the right fix.
