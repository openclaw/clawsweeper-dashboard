## Summary

Keep this issue open. Current main still has plaintext, readable `SKILL.md` loading plus scan-gated installs, but it does not implement signed/encrypted Markdown wrappers, a `persist` skill flag, `trustedSources`, or a commercial trust policy; open PR #74035 remains the linked implementation/design candidate.

## What This Changes

The issue requests persistent skill configuration and commercial signed or encrypted Markdown wrapper support for skills.

## Best Possible Solution

Keep this issue open while maintainers review, revise, land, or close PR #74035, or define a narrower signed/private ClawHub bundle design that preserves readable review, provenance, scanning, and documented trust boundaries.
