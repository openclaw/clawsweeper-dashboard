## What This Changes

The issue reports WhatsApp login failing before the QR code because `@openclaw/whatsapp` cannot install on Windows.

## Best Possible Solution

Keep the merged managed npm-root argv fix and let affected released-version users pick up the next release or a current main/beta build rather than adding a WhatsApp-specific workaround.

## Reproduction Assessment

Yes. The canonical fix PR reports Windows Node 22.20.0/npm 10.9.3 proof reproducing the `Cannot read properties of undefined (reading 'spec')` crash with `--prefix .`; current main removes that command shape.

## Solution Assessment

Yes. Removing redundant `--prefix .` from npm commands already executed inside the managed npm root is the narrow shared installer fix, and it avoids a WhatsApp-only workaround.
