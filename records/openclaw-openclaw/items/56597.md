## Summary

Keep open. Current main and the latest tagged release still persist authorized `/exec` session fields and resolve stored session exec values before agent/global `tools.exec` defaults, while the related ACP cache fix only addressed runtime-handle freshness.

## What This Changes

The issue asks OpenClaw to stop durable Discord session exec overrides from outliving later global `tools.exec` security configuration changes.

## Best Possible Solution

Define the exec policy contract first, then align persistence, resolver precedence, reset or migration behavior, docs, and regression tests with that contract.
