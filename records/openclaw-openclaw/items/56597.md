## Summary

Keep open. Current main still persists authorized external-channel `/exec` defaults into session entries and still resolves stored session exec values ahead of configured agent/global exec policy; the merged ACP cache fix addressed a neighboring stale-runtime-handle problem, not this persisted session-default precedence question.

## What This Changes

The issue reports that Discord sessions can keep per-session exec directives after global `tools.exec` config changes, causing older session state to override intended global policy.

## Best Possible Solution

Define the intended lifetime and precedence of session-level `/exec` defaults, then update persistence/resolution, upgrade behavior, focused tests, and docs so global operator policy changes cannot be silently shadowed by stale session state.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.6.6` still persist `exec*` fields into session entries and resolve them ahead of configured global/agent exec policy for later runs.

## Solution Assessment

Unclear. The reporter's desired behavior is plausible, but current docs and tests intentionally preserve authorized external-channel session defaults, so maintainers need to choose between non-persistence, different precedence, reset/migration behavior, or a deliberate durable override contract.
