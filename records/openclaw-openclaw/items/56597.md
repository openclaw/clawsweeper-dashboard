- `impact:session-state`: The central behavior is persisted session entry state carrying `exec*` values across later turns and restarts.

## Summary

Keep open. Current main and v2026.6.6 still persist authorized session-level exec fields and apply stored session exec policy after global/agent policy, while merged related work fixed adjacent ACP cache staleness and current docs/tests intentionally describe durable `/exec` session defaults.

## What This Changes

The issue asks OpenClaw to stop stale Discord session-level exec directive state from shadowing later global exec security configuration changes.

## Best Possible Solution

Define the intended lifetime and precedence of `/exec` session defaults, then align persistence, reset or migration behavior, docs, and tests so global operator policy changes are not silently shadowed unless durable session override is explicitly the supported contract.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.6.6` still persist `exec*` fields into session entries and resolve stored session exec values ahead of configured global or agent exec policy for later runs.

## Solution Assessment

Unclear. The reporter’s desired global-policy enforcement is plausible, but current docs/tests intentionally preserve authorized external-channel session defaults, so maintainers need to choose the permanent contract before a fix PR changes persistence or precedence.
