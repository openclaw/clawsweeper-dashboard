## Summary

Keep open. Current main still persists authorized `/exec` defaults into durable session state and resolves those stored values ahead of agent/global exec defaults; tests and docs describe sticky per-session behavior, so this remains an unresolved exec-policy/security decision rather than a cleanup close.

## What This Changes

The issue asks OpenClaw to stop Discord sessions from retaining stale per-session `/exec` overrides when global `tools.exec` policy changes.

## Best Possible Solution

Define the intended admin-policy precedence for persisted `/exec` overrides, then update the shared session persistence/resolution path, docs, and focused tests so tightened operator policy takes effect predictably for existing sessions.

## Reproduction Assessment

Yes, at source level. A persisted session entry with `execSecurity: "full"` is still resolved before stricter agent/global `tools.exec` defaults on later Discord turns; no live Discord gateway run was performed in this read-only review.

## Solution Assessment

Unclear. The report identifies a real policy conflict, but the best fix depends on a maintainer decision about whether global `tools.exec` is a hard ceiling, whether persisted session overrides need config-version/reset semantics, or whether tool policy remains the hard-disable layer.
