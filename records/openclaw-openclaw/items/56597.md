## Label Justifications

- `P2`: The issue is a bounded exec/session-state security-policy problem rather than an immediate outage or confirmed broad exploit.
- `impact:security`: Persisted exec overrides can preserve a looser requested host-exec policy after an operator tightens global configuration.
- `impact:session-state`: The reported behavior is caused by durable session state carrying exec override fields across later turns and restarts.

## Summary

Keep open. Current main and the latest tagged release still persist authorized per-session `/exec` fields and resolve them before agent/global `tools.exec` settings; the related ACP cache fix addressed stale runtime-handle reuse, not this stored session override precedence.

## What This Changes

The issue asks OpenClaw to prevent durable Discord session exec overrides from outliving later global `tools.exec` policy changes.

## Best Possible Solution

Define the exec-policy precedence contract first, then align persistence, resolver precedence, docs, migration or reset behavior, and focused tests with that contract.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.5.22` both persist exec override fields into session state and resolve them before stricter agent/global exec settings.

## Solution Assessment

Unclear. The report identifies a real security-policy conflict, but current docs and tests encode sticky `/exec` session defaults, so the best fix depends on a maintainer decision about the desired contract.
