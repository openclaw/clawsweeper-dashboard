## Summary

Keep open. Current main and the latest release still persist authorized `/exec` session defaults and layer stored session exec policy after global and agent config, so tightened operator config can still be shadowed by stale session state; current docs/tests make the fix a maintainer security/product decision rather than a cleanup close.

## What This Changes

The issue asks OpenClaw to stop persisted Discord `/exec` session directives from overriding later global `tools.exec` policy changes.

## Best Possible Solution

Define the supported lifetime and precedence for session-level `/exec` defaults, then align persistence, resolution, reset or migration behavior, tests, and docs so operator policy changes cannot be silently shadowed unless durable session overrides are explicitly the intended contract.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.6.8` still persist `exec*` fields into session entries and resolve them after global and agent exec policy for later runs.

## Solution Assessment

Unclear. The reporter's desired behavior is plausible for security enforcement, but current docs and tests intentionally preserve authorized external-channel session defaults, so maintainers need to choose the durable override contract before implementation.
