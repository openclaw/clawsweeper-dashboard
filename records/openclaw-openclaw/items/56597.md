## Label Justifications

- `P2`: The issue describes a bounded but real session exec-policy drift affecting Discord/operator security posture without evidence of a current emergency.
- `impact:security`: A retained session `execSecurity` value can keep broader command-execution permission after an operator tightens global exec policy.

## Summary

Keep open. Current main and `v2026.6.6` still persist `/exec` session defaults and still resolve stored session exec values ahead of configured global/agent exec policy; the merged ACP cache fix only covers cached runtime-handle freshness, not this session-entry precedence question.

## What This Changes

The issue asks OpenClaw to stop Discord session-stored exec directives from overriding later global `tools.exec` policy changes across gateway restarts.

## Best Possible Solution

Define the intended lifetime and precedence of session-level exec overrides, then update persistence/resolution plus focused tests so operator global exec policy changes are not silently shadowed by old stored directives.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.6.6` still persist `exec*` fields into session entries and resolve them ahead of configured global/agent exec policy for later runs.

## Solution Assessment

Unclear. The likely code fixes are mechanically bounded, but choosing non-persistence, different precedence, reset/migration behavior, or a deliberate durable override contract is a security and compatibility decision.
