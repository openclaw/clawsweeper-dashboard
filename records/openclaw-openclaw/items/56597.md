## Summary

Keep open. Current main and the latest shipped release still persist authorized `/exec` defaults into durable session state and resolve those stored values ahead of agent/global exec defaults; docs and tests also describe sticky per-session behavior, so this remains security-sensitive policy work rather than cleanup-closeable drift.

## What This Changes

The issue asks OpenClaw to stop Discord/shared session-stored `/exec` overrides from outliving global `tools.exec` config changes.

## Best Possible Solution

Define the intended admin-policy precedence for persisted `/exec` overrides, then update shared session persistence/resolution, docs, and focused tests so tightened operator policy takes effect predictably for existing sessions.

## Reproduction Assessment

Yes, at source level. A persisted session entry with `execSecurity: "full"` is still resolved before stricter agent/global `tools.exec` defaults on later Discord turns, although no live Discord gateway run was performed.

## Solution Assessment

Unclear. The report identifies a real policy conflict, but the best fix depends on a maintainer/security decision about whether global `tools.exec` is a hard ceiling, whether persisted session overrides need reset/version semantics, or whether tool policy remains the hard-disable layer.
