## Summary

Keep open. Current main and v2026.6.6 still persist `/exec` session defaults and resolve persisted session exec values ahead of configured agent/global exec policy; the related ACP runtime-cache fix covers a different stale-config path and does not resolve this per-session override precedence question.

## What This Changes

The issue asks OpenClaw to stop old Discord per-session exec directives from indefinitely shadowing later global `tools.exec` security configuration changes.

## Best Possible Solution

Decide the intended lifetime and precedence of session-level exec overrides, then update the persistence/resolution path so operator global exec policy changes are not silently shadowed by old stored directives.

## Reproduction Assessment

Yes. Source inspection shows current main and v2026.6.6 still persist `exec*` fields into session entries and resolve them ahead of configured global/agent exec policy for later runs.

## Solution Assessment

Unclear. The likely fixes are mechanically small, but choosing between non-persistence, different precedence, reset/migration behavior, or a deliberate durable override contract is a product/security decision.
