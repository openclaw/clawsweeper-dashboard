## Summary

Keep open. Current main and `v2026.6.6` still persist `/exec` values into durable session entries and still layer stored session exec policy above global/agent exec config; the merged ACP cache fix only addressed runtime-handle freshness, not this session-entry lifetime and precedence problem.

## What This Changes

The issue requests that Discord `/exec` session directives stop indefinitely shadowing later global `tools.exec` policy changes.

## Best Possible Solution

Define the intended lifetime and precedence of session-level exec overrides, then update persistence/resolution plus focused tests so operator global exec policy changes are not silently shadowed by stale stored directives.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.6.6` still persist `exec*` fields into session entries and resolve stored session exec values ahead of configured global/agent exec policy for later runs.

## Solution Assessment

Unclear. The mechanical fixes are plausible, but choosing non-persistence, different precedence, reset/migration behavior, or a deliberate durable override contract is a product/security decision.
