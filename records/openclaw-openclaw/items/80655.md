## Summary

Current main has the WhatsApp group-dispatch path wired through fresh runtime config and agent-scoped mention matching, and the post-v2026.5.7 fix also covers the related group handling called out in the discussion. The fix is present on main only; the latest release in the provided context is still v2026.5.7.

## What This Changes

Report that WhatsApp group messages accepted by the gateway in v2026.5.7 no longer reach an agent session when `groupChat.mentionPatterns` should match.

## Best Possible Solution

Keep the current-main per-message config plumbing for WhatsApp group routing and ship it in the next release; users who want visible automatic group replies should use the documented `messages.groupChat.visibleReplies: "automatic"` policy.

## Reproduction Assessment

Yes, source-level: the reported release path and related reports point to stale or mismatched WhatsApp config reaching dispatch-adjacent group handling, while current main now passes a freshly loaded config through routing, group gating, and `processMessage`. I did not run a live WhatsApp account in this read-only review.

## Solution Assessment

Yes. The current-main fix is the narrow maintainable solution because it carries refreshed runtime config through the active inbound boundary instead of adding a restart-only policy or a broader target-facts refactor for this regression.
