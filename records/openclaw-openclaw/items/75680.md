## Summary

Keep open. Current main still does not fully satisfy the report: the bundled memory-lancedb auto-capture path has only partial Chinese trigger coverage, short reported CJK phrases are rejected before trigger matching, Japanese/Korean defaults and custom trigger config are absent, and the linked implementation PR #76121 is open and unmerged.

## What This Changes

The issue requests CJK/i18n support for memory-lancedb auto-capture triggers, preferably with configurable custom trigger phrases.

## Best Possible Solution

Keep the fix inside the bundled memory-lancedb plugin by adding tested CJK trigger coverage and short-CJK handling; optional customTriggers support can be included if maintainers accept the config surface already proposed in #76121.

## Reproduction Assessment

Yes. Source inspection gives a deterministic path: short CJK inputs fail the text.length < 10 gate in shouldCapture, and the auto-capture hook skips storage when shouldCapture returns false.

## Solution Assessment

Yes. A plugin-owned update to the trigger/length filter, with focused tests and docs, is the narrowest maintainable path; LLM intent detection would be broader product work.
