## Summary

Keep open: current main only partially addresses the report. The memory-lancedb auto-capture gate is still a hard-coded trigger list, has no custom trigger config surface, and one of the reporter's concrete Chinese examples is still not capturable; the related #71349 cleanup was a separate cursor-starvation fix, not an i18n trigger fix.

## What This Changes

Add CJK/i18n support or configurable trigger patterns for memory-lancedb autoCapture so non-English memory-worthy phrases can be captured reliably.

## Best Possible Solution

Implement this inside the memory-lancedb plugin: add a safe configurable trigger surface, broaden tested CJK defaults for the concrete reported phrases, document the setting, and keep the existing cursor, embedding, and duplicate-detection behavior unchanged.

## Reproduction Assessment

Yes. The auto-capture filter is a pure source-level gate: current shouldCapture only tests MEMORY_TRIGGERS, and a direct regex check shows the reported phrase 以后都用这个 still returns false before any LanceDB or embedding dependency is involved.

## Solution Assessment

No, the current partial hard-coded Chinese regex is not the best final solution. A plugin-owned configurable trigger option, preferably literal phrases or validated bounded patterns, is the narrowest maintainable fix; LLM-based intent detection would be a broader product change.
