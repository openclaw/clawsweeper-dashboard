## Summary

Keep open. Current main still only partially covers Chinese auto-capture and leaves the reported short CJK phrases, `以后都用这个`, Japanese/Korean examples, and custom trigger configuration unsupported.

## What This Changes

The issue requests broader CJK/i18n support for memory-lancedb auto-capture triggers, preferably with user-configurable custom trigger patterns.

## Best Possible Solution

Keep the fix inside the bundled memory-lancedb plugin: add safe configurable trigger literals, broaden tested default CJK coverage, and adjust short-text handling for CJK without changing core memory or LanceDB storage behavior.

## Reproduction Assessment

Yes. The source-level reproduction is deterministic: `shouldCapture` rejects reported short Chinese/Japanese/Korean phrases through the length gate or missing trigger regex before any embedding or LanceDB dependency is involved.

## Solution Assessment

Yes. A plugin-owned default-trigger and safe custom-trigger update is the narrowest maintainable solution; LLM-based intent detection is broader product work and this report does not show a missing core plugin API.
