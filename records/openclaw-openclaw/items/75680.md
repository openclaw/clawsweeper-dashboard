## Summary

Keep open. Current main still leaves memory-lancedb autoCapture with a fixed trigger list, a pre-trigger length gate that rejects the reported short Chinese examples, no Japanese/Korean defaults, and no accepted customTriggers config; #76121 is open and unmerged.

## What This Changes

Request CJK/i18n support for memory-lancedb autoCapture triggers, preferably with custom trigger config.

## Best Possible Solution

Land #76121 or an equivalent plugin-owned patch with tested CJK defaults, safe literal custom triggers, docs, and changelog coverage.

## Reproduction Assessment

Yes. The source-level path is deterministic: short reported CJK phrases such as `记住这个`, `我喜欢`, and `重要` fail the `text.length < 10` gate before trigger matching, and the auto-capture hook skips messages when shouldCapture returns false.

## Solution Assessment

Yes. A bundled plugin-owned trigger/config update is the narrowest maintainable fix; LLM-based intent detection would be broader product work, and this report does not show a missing core plugin API.
