## Summary

Keep open. Current main still leaves the reported memory-lancedb auto-capture gap: short CJK trigger phrases are rejected before trigger matching, Japanese/Korean defaults and `customTriggers` are absent, and the linked closing PR #76121 is still open and unmerged.

## What This Changes

The issue asks memory-lancedb autoCapture to recognize common CJK memory triggers or allow configurable trigger patterns.

## Best Possible Solution

Keep the fix inside the bundled memory-lancedb plugin by landing #76121 or an equivalent narrow patch with tested CJK defaults, safe literal custom triggers, docs, and changelog coverage.

## Reproduction Assessment

Yes. The source-level reproduction is deterministic: `shouldCapture` rejects reported short CJK examples such as `记住这个`, `我喜欢`, and `重要` before trigger matching, and the auto-capture hook skips rejected messages.

## Solution Assessment

Yes. A plugin-owned trigger/config update is the narrowest maintainable direction; LLM-based intent detection would be broader product work and the report does not show a missing core plugin API.
