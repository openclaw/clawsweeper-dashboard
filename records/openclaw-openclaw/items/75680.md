## Summary

Keep open. Current main still rejects several reported CJK auto-capture phrases and has no configurable custom trigger support; the open closing PR #76121 is the right place to review the proposed fix before this issue closes.

## What This Changes

The issue requests broader CJK/i18n trigger coverage or configurable custom triggers for memory-lancedb autoCapture.

## Best Possible Solution

Review and land #76121 or an equivalent plugin-owned patch that adds safe customTriggers support and tested CJK trigger handling without changing core memory storage behavior.

## Reproduction Assessment

Yes. The source-level reproduction is deterministic: reported short CJK phrases fail the length gate or missing trigger regex in shouldCapture, and agent_end then skips storage before LanceDB or embeddings are involved.

## Solution Assessment

Yes. A memory-lancedb-owned trigger/config update is the narrowest maintainable path; LLM-based intent detection would be broader product work and this report does not show a missing core API.
