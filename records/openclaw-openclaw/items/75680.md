## Summary

Keep open. Current main still has a source-level auto-capture gap for the reported short CJK phrases, lacks Japanese/Korean default triggers and `customTriggers`, and the linked closing PR #76121 is still open rather than merged.

## What This Changes

Support CJK/i18n memory auto-capture in the bundled memory-lancedb plugin, preferably through CJK trigger coverage or configurable custom triggers.

## Best Possible Solution

Keep the fix in `extensions/memory-lancedb` by landing #76121 or an equivalent narrow patch with tested short CJK handling, Japanese/Korean coverage or safe literal custom triggers, docs, and changelog coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the reported short CJK examples are rejected by the `< 10` length gate before regex matching, and the auto-capture hook skips anything `shouldCapture` rejects.

## Solution Assessment

Yes. A plugin-owned trigger/config update is the narrowest maintainable direction; LLM-based intent detection would be broader product work and no missing core plugin API is evident.
