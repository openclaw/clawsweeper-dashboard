## Summary

Keep open. Current main still only partially addresses the report: memory-lancedb has one Chinese trigger, but short reported CJK phrases are rejected before trigger matching, Japanese/Korean defaults are absent, `customTriggers` is not accepted by config or manifest schema, and the linked fix PR #76121 is open and unmerged.

## What This Changes

The issue requests CJK/i18n-aware auto-capture triggers or configurable custom trigger patterns for the bundled memory-lancedb plugin.

## Best Possible Solution

Keep the fix inside the bundled memory-lancedb plugin by landing #76121 or an equivalent narrow patch with tested CJK defaults, safe literal custom triggers, docs, and changelog coverage.

## Reproduction Assessment

Yes. The source-level reproduction is deterministic: short CJK examples such as `记住这个`, `我喜欢`, and `重要` fail the `text.length < 10` gate before trigger matching, and the auto-capture hook skips messages when shouldCapture returns false.

## Solution Assessment

Yes. A plugin-owned trigger/config update is the narrowest maintainable solution; LLM-based intent detection would be broader product work and this report does not show a missing core plugin API.
