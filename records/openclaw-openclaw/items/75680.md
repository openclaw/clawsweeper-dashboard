## Summary

Keep open. Current main only partially covers Chinese auto-capture: the plugin still uses a hard-coded trigger list, has no configurable trigger surface, short reported CJK phrases are rejected by the length gate, and the reporter's `以后都用这个` example still misses the trigger regex entirely before embedding or storage can run.

## What This Changes

The issue requests broader CJK/i18n or configurable trigger support for `memory-lancedb` autoCapture so non-English memory-worthy phrases are captured automatically.

## Best Possible Solution

Implement this inside the bundled `memory-lancedb` plugin by adding a safe custom trigger config surface, broadening tested default CJK coverage for the reported phrases, adjusting the short-text gate where needed for CJK phrases, and updating manifest hints, docs, tests, and changelog while leaving core memory, embedding, LanceDB storage, duplicate detection, and cursor semantics unchanged.

## Reproduction Assessment

Yes. The reproduction is a pure source-level check against `shouldCapture`: the reported CJK examples return false before any LanceDB, embedding provider, or live gateway dependency is involved.

## Solution Assessment

Yes. A plugin-owned default-trigger plus safe custom-trigger update is the narrowest maintainable fix; LLM-based intent detection would be broader product work, and this report does not require a new core plugin API.
