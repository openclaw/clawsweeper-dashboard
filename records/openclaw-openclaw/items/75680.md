## Summary

Keep open. Current main still rejects the reported short CJK phrases before trigger matching and has no configurable custom trigger path; the linked implementation PR #76121 is open, so this issue should stay open until that PR lands or is closed.

## What This Changes

The issue requests CJK/i18n support for memory-lancedb auto-capture triggers, preferably through plugin-owned CJK defaults or configurable custom trigger phrases.

## Best Possible Solution

Land PR #76121 or an equivalent memory-lancedb-only patch with tested CJK defaults, safe literal custom trigger config, manifest/docs coverage, and changelog entry.

## Reproduction Assessment

Yes. The source-level reproduction is deterministic: the reported short CJK phrases fail `text.length < 10` before trigger matching, and `customTriggers` is rejected by config parsing.

## Solution Assessment

Yes. A plugin-owned CJK trigger and bounded literal custom-trigger update is the narrowest maintainable fix; LLM-based intent detection would be broader product work.
