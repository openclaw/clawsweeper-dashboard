## Summary

Keep open. Current main only partially covers Chinese auto-capture with one hard-coded regex, but the reported short CJK examples still fail the length gate, one reported Chinese phrase still has no trigger match, and there is no Japanese/Korean or custom-trigger config surface.

## What This Changes

The issue asks the bundled memory-lancedb plugin's autoCapture trigger detection to support CJK/i18n memory-worthy phrases, preferably through configurable custom triggers.

## Best Possible Solution

Implement the fix inside memory-lancedb by broadening tested default CJK coverage, adjusting the short-text gate for CJK trigger phrases, adding a safe custom trigger config surface, and updating the manifest, docs, tests, and changelog while preserving core memory APIs, embedding selection, LanceDB storage, duplicate detection, and cursor semantics.

## Reproduction Assessment

Yes. The reproduction is a source-level check of shouldCapture: the reported short Chinese phrases are rejected before trigger matching, and 以后都用这个 is not covered by the current trigger regex before the agent_end hook can embed or store it.

## Solution Assessment

Yes. A plugin-owned default-trigger plus safe custom-trigger update is the narrowest maintainable path; LLM-based intent detection would be broader product work and this report does not require a new core plugin API.
