## Summary

Keep open. Current main only partially covers Chinese auto-capture and still leaves reported short CJK phrases, `以后都用这个`, Japanese/Korean examples, and custom trigger configuration unsupported.

## What This Changes

Improve the bundled memory-lancedb plugin’s autoCapture trigger matching for CJK/i18n phrases, preferably with safe user-configurable custom triggers.

## Best Possible Solution

Keep the fix inside the bundled memory-lancedb plugin: add safe configurable trigger literals, broaden tested default CJK coverage, and adjust the short-text gate for CJK without changing core memory or LanceDB storage behavior.

## Reproduction Assessment

Yes. The reproduction is a source-level shouldCapture check: the reported short Chinese phrases fail the length gate, and 以后都用这个 fails trigger matching before any embedding or LanceDB dependency is involved.

## Solution Assessment

Yes. A plugin-owned default-trigger and safe custom-trigger update is the narrowest maintainable solution; LLM-based intent detection is broader product work and this report does not show a missing core plugin API.
