## Summary

Keep open. Current main still rejects several reported short CJK auto-capture phrases and does not expose custom trigger configuration; open PR #76121 is the linked implementation candidate and has not merged.

## What This Changes

The issue requests memory-lancedb autoCapture to recognize CJK/i18n memory triggers or support configurable custom trigger patterns.

## Best Possible Solution

Keep the fix inside the bundled memory-lancedb plugin: review and land #76121 or an equivalent narrow patch with safe literal custom triggers, tested CJK defaults, and no core storage or LanceDB behavior change.

## Reproduction Assessment

Yes. The source-level reproduction is deterministic: shouldCapture rejects the reported short CJK examples before trigger matching, and the auto-capture hook skips messages when shouldCapture returns false.

## Solution Assessment

Yes. A plugin-owned trigger/config update is the narrowest maintainable solution; LLM-based intent detection would be broader product work and this report does not show a missing core plugin API.
