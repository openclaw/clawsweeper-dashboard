## Summary

Keep open. Current main still rejects the reported short CJK phrases before trigger matching and does not expose custom trigger configuration; the linked fix PR #76121 is open and unmerged.

## What This Changes

Add CJK/i18n-friendly auto-capture triggers or custom trigger configuration to memory-lancedb so non-English memory-worthy phrases can be captured.

## Best Possible Solution

Keep the fix in the bundled memory-lancedb plugin: land #76121 or an equivalent narrow patch with short CJK trigger handling, safe literal `customTriggers` config, tests, docs, and changelog coverage.

## Reproduction Assessment

Yes. The source-level reproduction is deterministic: reported short CJK examples are rejected by the length gate in `shouldCapture`, and the auto-capture hook skips any text that fails that predicate.

## Solution Assessment

Yes. A plugin-owned trigger/config update is the narrowest maintainable fix; LLM-based intent detection would be broader product work and this report does not show a missing core plugin API.
