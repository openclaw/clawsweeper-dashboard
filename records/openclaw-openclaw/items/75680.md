## Summary

Keep open. Current main has only partial Chinese trigger coverage, still rejects the reported short CJK phrases before trigger matching, has no Japanese/Korean or configurable custom trigger support, and the linked fix PR #76121 is open and unmerged.

## What This Changes

Add CJK/i18n support or configurable custom triggers to `memory-lancedb` auto-capture so non-English memory-worthy phrases are captured.

## Best Possible Solution

Keep the fix inside the bundled `memory-lancedb` plugin by landing #76121 or an equivalent narrow patch with tested CJK defaults, safe literal custom triggers, docs, and changelog coverage.

## Reproduction Assessment

Yes. The source-level reproduction is deterministic: the reported short CJK examples fail the `text.length < 10` gate before `MEMORY_TRIGGERS` is evaluated, and the auto-capture hook skips messages when `shouldCapture` returns false.

## Solution Assessment

Yes. A plugin-owned trigger/config update is the narrowest maintainable solution; LLM-based intent detection would be broader product work and this report does not show a missing core plugin API.
