## Summary

Keep open. Current main only partially addresses the report: `memory-lancedb` still gates auto-capture through a hard-coded plugin-local trigger list, has no custom trigger config, and still misses the reporter's concrete `以后都用这个` example before embedding or storage can run.

## What This Changes

The issue requests CJK/i18n support or configurable trigger patterns for the bundled `memory-lancedb` plugin's `autoCapture` heuristics.

## Best Possible Solution

Implement this inside the `memory-lancedb` plugin: add a safe custom trigger surface, broaden tested default CJK coverage for the reported phrases, update manifest/UI hints/docs/changelog, and leave core, embedding, cursor, and duplicate-detection behavior unchanged.

## Reproduction Assessment

Yes. The reproduction is a pure source-level check: evaluate current `MEMORY_TRIGGERS` through `shouldCapture`, and `以后都用这个` returns false before any LanceDB or embedding dependency is involved.

## Solution Assessment

Yes. A plugin-owned config/default-trigger update is the narrowest maintainable solution; LLM-based intent detection would be a broader product change and a new core seam is not needed for this report.
