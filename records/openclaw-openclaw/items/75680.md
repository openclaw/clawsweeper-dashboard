## Summary

Keep open. Current main still leaves the reported memory-lancedb auto-capture gap: short reported CJK trigger phrases are rejected before trigger matching, Japanese/Korean defaults and `customTriggers` are absent, and #76121 is an open unmerged closing PR.

## What This Changes

Request memory-lancedb `autoCapture` to recognize CJK/i18n memory-worthy phrases or allow configurable trigger patterns.

## Best Possible Solution

Land #76121 or an equivalent memory-lancedb plugin patch with tested short CJK defaults, any accepted safe literal custom trigger config, docs, and changelog coverage.

## Reproduction Assessment

Yes. Source inspection gives a deterministic path: reported short CJK examples are below the 10-code-unit gate in `shouldCapture`, and the auto-capture hook skips anything that returns false there.

## Solution Assessment

Yes. Keeping the fix inside the bundled memory-lancedb plugin is the narrowest maintainable direction; LLM-based intent detection would be broader product work and the report does not show a missing core plugin API.
