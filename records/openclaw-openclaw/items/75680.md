## Summary

Keep open. Current main only partially covers Chinese auto-capture: the plugin has one Chinese regex, but `shouldCapture` still rejects the reporter's short examples via the 10-character gate, misses `以后都用这个`, has no Japanese/Korean trigger coverage, and exposes no configurable trigger surface.

## What This Changes

The issue asks the bundled `memory-lancedb` autoCapture filter to support CJK/i18n memory triggers or user-configurable custom trigger patterns instead of relying on a mostly hardcoded English/Czech trigger list.

## Best Possible Solution

Implement the fix inside the bundled `memory-lancedb` plugin: add safe custom trigger configuration, broaden tested default CJK coverage for the reported phrases, adjust the short-text gate for memory-worthy CJK text, and update manifest hints, docs, tests, and changelog without changing core memory, embedding, LanceDB storage, duplicate detection, or cursor semantics.

## Reproduction Assessment

Yes. The reproduction is a source-level check against `shouldCapture`: the reported Chinese examples return false before any embedding provider, LanceDB instance, or live gateway behavior is involved.

## Solution Assessment

Yes. A plugin-owned default-trigger plus safe custom-trigger update is the narrowest maintainable fix; LLM-based intent detection would be broader product work, and arbitrary user regex support should be constrained to avoid ReDoS-style risks.
