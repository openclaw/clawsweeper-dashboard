## Repair Work Prompt

Fix the Google/Gemini LLM API-key rotation gap reported in https://github.com/openclaw/openclaw/issues/97314. Current main builds Google LLM transport headers from a single `options?.apiKey ?? getEnvApiKey(model.provider)` value in `extensions/google/transport-stream.ts`, while Google embeddings already use `collectProviderApiKeysForExecution` plus `executeWithApiKeyRotation`. Reuse the existing rotation helper for pre-SSE Google Generative AI LLM requests so rate-limit/provider quota errors from the first key retry with the next configured key. Preserve Google Vertex ADC/OAuth behavior, do not add a new config or env surface, and do not retry after user-visible SSE chunks have started. Add focused tests for multiple `GEMINI_API_KEY_*`/primary key behavior and the no-rotation Vertex ADC/OAuth path. Do not edit `CHANGELOG.md`.
