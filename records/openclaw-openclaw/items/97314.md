## Summary

Keep open: current main and latest release still use one resolved Google/Gemini LLM API key, and the open linked PR is the active candidate fix rather than a merged implementation.

## What This Changes

The issue reports that Google/Gemini LLM streaming uses one resolved API key instead of rotating across configured Gemini keys on 429 responses.

## Best Possible Solution

Review and land the linked provider-owned pre-SSE rotation fix, preserving Vertex/OAuth/custom endpoint boundaries and adding focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible: current main resolves one Google LLM API key and never collects the configured rotation key set before opening the SSE request. I did not run a live Gemini quota response with real credentials.

## Solution Assessment

Yes, the best fix is to reuse the existing shared API-key rotation helper at the pre-SSE request boundary. A custom rotator or new config knob would duplicate existing provider-auth behavior.
