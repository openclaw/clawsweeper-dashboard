- `P2`: This is a normal-priority Google provider reliability bug with a documented behavior gap but no evidence of a core runtime outage.
- `impact:auth-provider`: The issue concerns provider credential selection and API-key rotation for Google/Gemini model requests.

## Summary

Keep open: current main and the latest release still build the Google LLM transport request with a single API key, while the documented rotation helper is only wired into adjacent embedding/media paths.

## What This Changes

Report that Google/Gemini LLM streaming uses only the first configured API key instead of rotating across multiple GEMINI_API_KEY values on rate-limit failures.

## Best Possible Solution

Wire the provider-owned Google LLM transport through the existing pre-request API-key rotation helper for rate-limit responses, with focused tests and explicit protection for Vertex ADC and already-started streams.

## Reproduction Assessment

Yes, source-reproducible: current main and v2026.6.10 build Google LLM request headers from a single resolved key and never collect the configured rotation key set. I did not run a live 429 repro with real Gemini keys.

## Solution Assessment

Yes, the best fix is to reuse the existing generic rotation helper at the pre-SSE request boundary. A separate custom rotator or new config surface would duplicate current provider-auth behavior.

## Repair Work Prompt

Fix the Google/Gemini LLM transport API-key rotation gap reported in https://github.com/openclaw/openclaw/issues/97314. In current main, extensions/google/transport-stream.ts builds one apiKey from options.apiKey or getEnvApiKey(model.provider) and calls openGoogleSseChunks once; docs and the existing helper say Google requests should rotate across GEMINI_API_KEYS/GEMINI_API_KEY_* on rate-limit errors. Reuse collectProviderApiKeysForExecution and executeWithApiKeyRotation rather than adding a new rotator or config option. Apply rotation only where it is safe before SSE output starts; preserve option-supplied keys, model/option headers, Gemini 3 first-response retry behavior, abort handling, and Google Vertex ADC/OAuth semantics. Either update the legacy src/llm/providers/google.ts path too or prove it is no longer runtime-reachable. Add focused regression coverage in extensions/google/transport-stream.test.ts for a 429/resource-exhausted first attempt using key 1 followed by a successful attempt using key 2. Do not edit CHANGELOG.md.
