## What This Changes

The issue reports that Google/Gemini LLM streaming does not rotate configured Gemini API keys on 429 responses, while the adjacent embedding path already uses the shared key-rotation helper.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/97328, then close this issue only after the provider-owned Google transport fix reaches main.

## Reproduction Assessment

Yes, source-reproducible: current main resolves one Google LLM API key and never collects the configured rotation key set before opening SSE. I did not run a live Gemini 429 with real credentials in this read-only review.

## Solution Assessment

Yes, the best fix is the provider-owned pre-SSE rotation approach in the open candidate PR, with explicit credential-boundary tests. A new config knob or legacy core wrapper would be a broader and less maintainable solution.
