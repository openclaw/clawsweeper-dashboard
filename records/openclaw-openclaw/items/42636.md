## Summary

This PR is a valid narrow UI performance direction, but it is stale against current main and would miss the active chat renderer. Current main still fetches and renders 200 messages, while the PR updates an old view-local builder path instead of the extracted `build-chat-items` module, so it needs a refresh before merge.

## What This Changes

The PR reduces the Control UI initial `chat.history` request and chat render window from 200 to 80 messages by adding a shared UI constant and updating controller/view tests.

## Best Possible Solution

Land a refreshed narrow UI patch that centralizes the initial chat history window in the active history-limit surface, applies it to `loadChatHistory()` and `buildChatItems()`, deliberately preserves or updates input-history alignment, adds direct controller and builder tests, and records the user-facing fix in `CHANGELOG.md`.
