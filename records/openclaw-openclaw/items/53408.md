## Summary

Keep open. Current main still does not cover the reported custom Anthropic-compatible long-session argument-loss path: malformed argument repair is scoped to Kimi-on-Anthropic and OpenAI-compatible transports, while the Anthropic transport still parses streamed tool JSON directly for other custom `anthropic-messages` providers.

## What This Changes

The issue asks OpenClaw to reliably preserve `write`, `exec`, and related tool arguments in long, tool-heavy sessions for custom Anthropic-compatible providers instead of delivering missing or empty argument objects.

## Best Possible Solution

Keep this as the canonical reliability bug until maintainers add a captured or synthetic custom `anthropic-messages` stream fixture with large fragmented `write`, `exec`, and preferably `sessions_spawn` arguments, then add a scoped repair or explicit malformed-argument diagnostic for that provider class.

## Reproduction Assessment

No. The report and related #55224 provide credible field reproduction details, but current main does not include a deterministic fixture or live replay that reproduces the custom Anthropic-compatible long-fragment stream path.

## Solution Assessment

Unclear. The existing repair wrapper is a reasonable narrow fix for Kimi/OpenAI-compatible malformed fragments, but it is not sufficient evidence for this issue; the safer path is fixture-backed custom Anthropic-compatible repair or diagnostics rather than a blind provider-wide toggle.
