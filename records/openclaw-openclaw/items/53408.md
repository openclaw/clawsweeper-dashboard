## Summary

Keep open. Current main still leaves the reported custom Anthropic-compatible long-session argument-loss path uncovered: malformed argument repair is only enabled for Kimi on `anthropic-messages` and for `openai-completions`, while custom `anthropic-messages` proxies still go through direct streamed JSON parsing without a fixture proving large fragmented `write`/`exec` arguments survive.

## What This Changes

The issue asks OpenClaw to reliably preserve long `write`, `exec`, and related tool-call arguments in extended sessions, especially through custom Anthropic-compatible provider proxies.

## Best Possible Solution

Keep this as the canonical reliability bug until maintainers add or replay a custom `anthropic-messages` stream fixture with large fragmented `write`, `exec`, and preferably `sessions_spawn` arguments, then implement a scoped repair or explicit diagnostic for that provider class.

## Reproduction Assessment

No. The issue and related field reports describe a plausible long-session reproduction, but current main does not include a deterministic test fixture or live replay that reproduces the custom Anthropic-compatible long-fragment stream path.

## Solution Assessment

Unclear. The existing Kimi/OpenAI-compatible malformed-argument repair is reasonable for its covered providers, but it is not sufficient for this report; the safer path is fixture-backed custom Anthropic-compatible repair or diagnostics rather than a blind provider-wide toggle.
