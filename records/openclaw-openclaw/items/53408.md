## Summary

Keep open: current main still scopes malformed tool-call argument repair away from the reported custom Anthropic-compatible proxy path, while the stream parser can still turn incomplete streamed arguments into `{}` and the `write`/`exec` validators correctly reject the missing required fields.

## What This Changes

The issue reports long-context `write` and `exec` tool calls arriving with missing `content` or `command` arguments, especially with large Anthropic-compatible proxy payloads.

## Best Possible Solution

Keep this as the canonical custom Anthropic-compatible and long-context argument-loss bug, collect a redacted current-main trace, and make a narrowly gated repair or fail-closed transport fix with regression coverage while leaving staged/file-backed arguments to the broader feature tracker.

## Reproduction Assessment

No high-confidence live reproduction was established for the reported copilot-api/webchat setup. Source inspection shows the custom `anthropic-messages` route still bypasses repair, while incomplete streamed arguments can become `{}` before required-parameter validation.

## Solution Assessment

Unclear. Current main is not sufficient for the reported route, but the safest fix is trace-backed provider or transport policy work rather than unconditionally enabling malformed-argument repair for every Anthropic-compatible provider.
