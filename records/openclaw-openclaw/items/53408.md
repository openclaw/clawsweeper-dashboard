## Summary

Keep open. Current main still does not show a completed fix for the reported custom Anthropic-compatible long-session tool-argument loss path; the existing malformed-argument repair is gated to Kimi on `anthropic-messages`, OpenAI Completions, and selected Responses transports, while custom `copilot-api`-style Anthropic routes still use the direct streaming JSON parse path.

## What This Changes

The issue reports that long webchat sessions with a custom Anthropic-compatible `copilot-api` provider can cause `write`, `exec`, and related tool arguments to arrive empty or with required fields missing.

## Best Possible Solution

Keep this as the canonical custom Anthropic-compatible tool-argument reliability bug until maintainers add a captured or synthetic long fragmented stream fixture and then apply a scoped repair or explicit diagnostic policy for that provider class.

## Reproduction Assessment

No. Source inspection shows a credible path to empty arguments for malformed custom Anthropic-compatible streamed JSON, but there is no deterministic current-main fixture or live replay that reproduces the long-session `copilot-api` failure.

## Solution Assessment

Unclear. The existing repair wrapper is a reasonable narrow fix for Kimi and some OpenAI-compatible malformed fragments, but this issue needs fixture-backed custom `anthropic-messages` coverage before widening the repair or changing provider policy.
