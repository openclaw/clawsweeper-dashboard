## Summary

Keep open. Current main still does not prove or cover the reported custom Anthropic-compatible long-session argument loss: malformed-argument repair is limited to Kimi on `anthropic-messages` and `openai-completions`, while custom `anthropic-messages` streams still parse incomplete `partial_json` into `{}`.

## What This Changes

The issue reports that long, tool-heavy sessions can deliver empty or partial argument objects to `write`, `exec`, `edit`, and related tools, especially through Anthropic-compatible custom/proxy providers.

## Best Possible Solution

Keep this as the canonical reliability bug until maintainers capture or synthesize a custom Anthropic-compatible stream fixture with large fragmented `write`, `exec`, and preferably `sessions_spawn` arguments, then add provider-safe repair or explicit diagnostics for malformed live tool-argument assembly.

## Reproduction Assessment

No. The issue and related duplicate give credible long-session reproduction descriptions, but current main does not include a deterministic fixture or live replay for the reported custom `anthropic-messages` stream path.

## Solution Assessment

Unclear. The current scoped repair is appropriate for known Kimi and OpenAI-compatible regressions, and custom endpoint header suppression addresses a related proxy class, but the safest remaining path is fixture-backed Anthropic-compatible repair or diagnostics rather than a blind provider-wide toggle.
