## Summary

Keep open. Current main still matches the reported TUI shutdown gap, and an open, unmerged PR already references this issue with closing syntax, so the issue should remain open until that PR or an equivalent focused fix lands.

## What This Changes

The issue reports that `openclaw tui` can become unrecoverable after a gateway WebSocket close, ignoring Ctrl+C, Ctrl+D, and SIGINT until SIGKILL.

## Best Possible Solution

Land or replace the linked focused TUI lifecycle fix so disconnected-state exit is explicitly handled: Ctrl+C/SIGINT should have a reliable escape path after gateway disconnect, Ctrl+D shutdown should complete through bounded cleanup/fallback behavior, and regression tests should cover the disconnected state rather than only generic shutdown helpers.

## Reproduction Assessment

Yes. The report gives concrete gateway-restart steps, and current main has matching disconnect, Ctrl+C/SIGINT, Ctrl+D, and shutdown-helper paths; this review did not run the macOS live repro.

## Solution Assessment

Yes, the narrow maintainable solution belongs in `src/tui/tui.ts` and its colocated tests. Because #75381 is already open and references this issue with closing syntax, the right maintainer path is to review/land that PR or an equivalent focused fix before closing this issue.
