## Summary

Keep open. Current main still matches the reported TUI lifecycle gap: gateway disconnect only updates status, while Ctrl+C/SIGINT remain on the normal double-press path and Ctrl+D uses shutdown without an exit-only fallback. The open closing PR #75381 is the right implementation vehicle, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that `openclaw tui` can become unrecoverable after a gateway WebSocket close and asks for disconnected-state exit handling plus bounded shutdown fallback.

## Best Possible Solution

Land #75381 or an equivalent focused TUI lifecycle patch that makes disconnected-state exit reliable, preserves normal active-session Ctrl+C behavior outside that state, adds a bounded/finally/timeout shutdown path, and includes regression coverage for gateway-disconnected Ctrl+C/Ctrl+D/SIGINT behavior.

## Reproduction Assessment

Yes. The report gives a concrete `openclaw tui` plus `openclaw gateway restart` path, and current main has matching disconnected-state, Ctrl+C/SIGINT, and Ctrl+D shutdown code paths; this review did not execute the macOS live repro.

## Solution Assessment

Yes. A narrow TUI lifecycle fix is the best path: handle disconnected-state exit in `src/tui/tui.ts` and cover it in TUI tests, rather than changing gateway WebSocket behavior or broad terminal infrastructure first.
