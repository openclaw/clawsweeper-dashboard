## Summary

Keep open: current main still has a source-backed path where accepted `sessions_spawn` results already marked `isError: true` are collected as compaction tool failures, and the earlier accepted-spawn terminal-success work does not cover this legacy compaction noise.

## What This Changes

The issue reports that accepted `sessions_spawn` tool results can be persisted as errors and later summarized as `## Tool Failures`.

## Best Possible Solution

Add a narrow canonical classification and compaction guard: accepted `sessions_spawn` payloads should be non-errors, forbidden/error statuses should remain failures, and legacy accepted records should be skipped by the compaction failure summary.

## Reproduction Assessment

Yes, source-reproducible: pass a `sessions_spawn` tool result with accepted payload data and `isError: true` through compaction, and current code will include it in `## Tool Failures`. I did not execute the test in this read-only review.

## Solution Assessment

Yes; the best fix is a narrow tool-result classification and compaction guard using the existing accepted-spawn predicate, not a config option or a broad exemption for all tool failures.
