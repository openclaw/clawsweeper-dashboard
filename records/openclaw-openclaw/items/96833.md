## Summary

Keep open: current main still records accepted `sessions_spawn` results as failures in the source-backed paths, and the complete closing PR is open but not merged.

## What This Changes

The issue requests that accepted `sessions_spawn` results be recorded as non-errors and ignored by compaction failure summaries while genuine forbidden/error spawns remain failures.

## Best Possible Solution

Land or revise https://github.com/openclaw/openclaw/pull/96851 as the complete source-and-read-boundary fix, then close this issue after main no longer marks accepted spawns as failures.

## Reproduction Assessment

Yes. A current-main `toolResult` with `toolName: "sessions_spawn"`, `isError: true`, and accepted spawn details will be collected as a compaction failure because the collector only checks `isError`; I did not run tests because this review is read-only.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the accepted-spawn predicate at both the embedded-runner classification boundary and the compaction read boundary; https://github.com/openclaw/openclaw/pull/96851 appears to follow that shape.
