## Summary

Keep open: current main and the latest release still classify accepted `sessions_spawn` results marked `isError: true` as tool failures, and an open closing PR now owns the complete repair.

## What This Changes

This issue asks for accepted `sessions_spawn` results to be treated as successful in persisted/forwarded results and compaction, while forbidden/error spawns remain failures.

## Best Possible Solution

Land a narrow fix that uses the existing accepted-spawn predicate at both the persisted/forwarded result boundary and the compaction failure-summary boundary, while preserving forbidden/error spawn failures.

## Reproduction Assessment

Yes. Source inspection shows that passing an accepted `sessions_spawn` tool result with `isError: true` through `collectToolFailures` will include it in `## Tool Failures`; I did not run tests in this read-only review.

## Solution Assessment

Yes. The best fix is the narrow classification and read-boundary guard using `normalizeAcceptedSessionSpawnResult`; https://github.com/openclaw/openclaw/pull/96851 appears to cover both surfaces, while https://github.com/openclaw/openclaw/pull/96842 covers only the read-side symptom.
