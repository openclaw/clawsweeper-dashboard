The issue asks OpenClaw to stop treating accepted `sessions_spawn` results as tool failures in persisted/forwarded results and compaction summaries.

## Best Possible Solution

Review and land one complete fix, likely the live closing PR, that normalizes accepted `sessions_spawn` as success at the source and filters legacy accepted records from compaction while preserving forbidden/error failures.

## Reproduction Assessment

Yes. Source inspection shows an accepted `sessions_spawn` tool result with `isError: true` will pass into `collectToolFailures`, and the current source-side classifiers do not yet special-case the accepted spawn contract.

## Solution Assessment

Yes. The best fix is the narrow classification plus compaction guard described by the issue, using the existing accepted-spawn predicate rather than a broad suppression of tool failures.
