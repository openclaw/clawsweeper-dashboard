## What This Changes

The branch adds macOS `ps`-based transitive parent-PID walking to the stale-gateway ancestry guard, threads caller timeouts into that lookup, and adds regression coverage plus a changelog entry.

## Best Possible Solution

Land the focused ancestry patch after maintainer validation of the intentional fail-closed behavior, preferably with redacted macOS LaunchAgent proof showing refusal before gateway stop.

## Reproduction Assessment

No high-confidence live current-main macOS repro was run in this read-only review. Source inspection plus the linked issue logs give a credible path: current main stops macOS ancestry at `process.ppid`, so a nested gateway-owned updater can miss the gateway PID and reach managed-service stop.

## Solution Assessment

Yes, for the refusal side of the acceptance criteria. Extending the existing ancestry guard to macOS with bounded `ps` probes is narrower than adding a new durable launchd handoff in this PR, while still preserving the existing safety contract.
