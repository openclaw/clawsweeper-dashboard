## Summary

Keep this PR open. Current main still has the reported persisted-session repair gap, but the PR head is not ready to merge because its tail-drop pass stops at metadata rows that SessionManager still uses as the active leaf, leaving the errored assistant as the final replayed model turn.

## What This Changes

The PR changes session-file repair to drop trailing errored assistant entries, adds a `droppedTrailingErrorEntries` report/log counter, expands repair regression tests, and updates transcript-hygiene docs.

## Best Possible Solution

Revise the repair contract so metadata rows are preserved but replay-invisible tail metadata is transparent when finding the last provider-visible turn; drop trailing errored assistant entries underneath it, add metadata-tail regression coverage, correct the docs text, and add the required changelog entry.

## Reproduction Assessment

Yes. A source-level reproduction is a session JSONL ending `user -> errored assistant -> model_change` or `thinking_level_change`: the PR stops at the metadata row, while SessionManager replays the assistant as the final provider message.

## Solution Assessment

No. Disk repair is the right narrow fix location and the counter bookkeeping is reasonable, but treating every non-message tail row as terminal does not match the dependency's replay semantics.
