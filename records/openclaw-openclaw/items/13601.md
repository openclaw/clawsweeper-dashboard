## Summary

Keep open. Current main has persistent JSONL-backed cron run history, but the requested timestamp filtering, richer CLI history commands, export/follow behavior, success/failure aggregates, and manual/day-based retention workflow are still not implemented.

## What This Changes

The issue requests durable cron execution history with CLI filtering, export/follow, aggregate reporting, and retention controls.

## Best Possible Solution

Extend the existing JSONL-backed run history with additive timestamp filters through the reader, protocol, Gateway, CLI/UI, and docs, then separately scope aliases, aggregates, export/follow, and day-based/manual retention controls.
