## Summary

Keep open: current main and v2026.6.10 already have durable SQLite cron run history with status, delivery, run-id, query, paging, and UI filters, but the requested CLI logs/history command surface, timestamp range filters, follow/export behavior, success-rate aggregates, and explicit retention/prune/vacuum workflow are still missing.

## What This Changes

The issue asks for persistent cron execution history plus CLI filtering, time ranges, JSON/follow export, list aggregates, and retention/prune controls.

## Best Possible Solution

Use the existing SQLite run-log store as the backend, then add an explicit additive CLI/protocol/docs design for logs/history status and time filters, JSON/follow semantics, aggregates, and secure retention/redaction behavior.
