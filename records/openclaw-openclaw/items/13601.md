## Summary

Request persistent cron execution history with status/time filtering, CLI logs/export/follow commands, success-rate aggregates, and configurable retention/prune controls.

## Best Possible Solution

Add an explicitly scoped design on top of the existing JSONL-backed `cron.runs` contract: timestamp filters, a supported CLI history/logs surface, JSON export/follow semantics, summary aggregates, docs, and clear export/redaction/retention rules.
