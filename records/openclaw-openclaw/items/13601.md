Keep open. Current main implements durable JSONL-backed cron run history plus status, delivery, text-query, sort, and pagination filters, but the requested timestamp/range filters, richer `cron logs` or `cron history` CLI, follow/export behavior, success-rate aggregates, and day/manual retention controls remain unimplemented and need product scoping.

## What This Changes

The issue requests persistent cron execution history with filtering, CLI log/history commands, export/follow behavior, list aggregates, and retention controls.

## Best Possible Solution
