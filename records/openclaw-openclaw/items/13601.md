## Summary

Keep open: current main and v2026.6.6 have durable SQLite cron run history and UI/Gateway filters, but the requested CLI time-range audit workflow, aggregates, follow/export behavior, and manual maintenance surface remain missing.

## What This Changes

## Best Possible Solution

Keep the SQLite-backed run history foundation, then split approved follow-ups: expose current filters in CLI, add timestamp bounds through protocol/Gateway/UI/CLI, and decide the security policy for export, follow, aggregates, and manual maintenance before implementation.
