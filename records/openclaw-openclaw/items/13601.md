## Summary

Keep open. Current main and the latest release include durable JSONL-backed cron run history with runId, status, delivery, query, pagination, and pruning, but they still lack timestamp/range filters, a richer `cron logs` or `cron history` CLI with status/time/follow/export options, success-rate/failure aggregates, and explicit history-retention/manual-prune semantics.

## What This Changes

The issue requests persistent cron execution history with status and time filtering, CLI log/history commands, JSON export/follow support, list aggregates, and retention controls.

## Best Possible Solution

Scope an additive design on top of the existing JSONL-backed `cron.runs` contract: add timestamp filters through the reader, protocol, gateway, UI, CLI, and docs; expose a supported `cron logs` or `cron history` command; add summary aggregates; and define retention, pruning, export, and redaction semantics.
