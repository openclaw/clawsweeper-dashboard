## What This Changes

The issue reports that ACPX/Claude ACP sessions can fail with a TypeError when a primitive adapter frame reaches code that checks for message fields with the `in` operator.

## Best Possible Solution

Land a narrow OpenClaw update that either consumes the upstream acpx parser guard or keeps a compatibility wrapper mitigation for bundled older acpx, with regression coverage for primitive/null/array adapter stdout frames.

## Reproduction Assessment

Yes, by source inspection: current main pins acpx 0.10.0, that version enqueues primitive parsed JSON values, and claude-agent-acp 0.39.0 uses `in` checks on session messages. I did not run a live `sessions_spawn(runtime="acp")` smoke in this read-only review.

## Solution Assessment

Yes for the current direction: the shared parser guard belongs upstream in acpx, and OpenClaw should either bump to that fixed acpx version or land the linked wrapper mitigation while older bundled acpx is still in use.
