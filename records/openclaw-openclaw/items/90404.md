The issue asks OpenClaw to handle non-object ACP adapter messages so `sessions_spawn(runtime="acp")` with Claude ACP does not fail with a TypeError and empty output.

## Best Possible Solution

Fix the non-object JSON-RPC message guard at the ACP SDK/acpx boundary, consume the fixed package in `@openclaw/acpx`, and cover it with a focused regression plus a live Claude ACP smoke.

## Reproduction Assessment

Yes, source-reproducible: the pinned ACP SDK dispatches on `"method" in message` without an object guard, and a numeric message produces the same TypeError text. I did not establish a full live Claude ACP reproduction in this read-only pass.

## Solution Assessment

Yes, the reporter's suggested type guard is the right shape, but the best fix is at the dependency boundary rather than catching the crash after it becomes an unhandled ACP runtime failure.
