## Label Justifications

- `P1`: A released ACP `sessions_spawn` workflow can fail before delivering Claude ACP session output for affected users.
- `impact:message-loss`: The reported TypeError leaves the spawned ACP session with empty messages instead of returning runtime output.

## Summary

Keep open: current main and v2026.6.8 still carry the ACPX dependency path that can pass primitive JSON stdout frames into the ACP SDK dispatcher, while https://github.com/openclaw/openclaw/pull/95130 is now the active open candidate fix.

## What This Changes

Prevent ACPX/Claude ACP sessions spawned through `sessions_spawn(runtime="acp")` from failing when adapter stdout contains primitive JSON frames such as `1`.

## Best Possible Solution

Review and either land or revise https://github.com/openclaw/openclaw/pull/95130; if that PR is rejected, fix the object-shape validation at the acpx/ACP SDK boundary with regression coverage for primitive, null, string, array, request, response, and notification frames.

## Reproduction Assessment

Yes. Source inspection shows `acpx@0.10.0` enqueues parsed primitive stdout values and `@agentclientprotocol/sdk@0.22.1` applies `"method" in message` before object narrowing; I did not run a live Claude ACP session.

## Solution Assessment

Yes on direction, not yet on completion. The right fix is to validate ACP message shape before dispatch; the active question is whether maintainers accept the OpenClaw wrapper mitigation in https://github.com/openclaw/openclaw/pull/95130 or require the guard in acpx/ACP SDK.
