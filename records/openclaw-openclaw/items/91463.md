The PR changes the Browser plugin tab-selection resolver to preserve a newly opened target, poll for delayed local-managed `wsUrl` discovery, fall back to selectable unfiltered tabs, and add regression tests.

## Best Possible Solution

Harden the resolver so polling keeps the latest usable nonempty snapshot, tolerates transient list failures, and then verify the local-managed Chrome/CDP timing path with redacted live output.

## Solution Assessment

No, not yet: the resolver is the right layer, but the current polling loop should keep the latest usable tab snapshot and tolerate transient list failures before this is the best fix.
