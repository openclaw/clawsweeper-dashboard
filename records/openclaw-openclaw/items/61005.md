## Summary

Keep open: current main and v2026.5.7 still block the final Android onboarding step when only the node is connected. The focused target-repo fix at https://github.com/openclaw/openclaw/pull/61269 was stale-closed, and the remaining open same-head PR is only in a fork.

## What This Changes

Allow Android onboarding to finish once the device node is paired and connected, even when the operator session is offline.

## Best Possible Solution

Ship a small Android app fix that treats node connectivity as sufficient for final onboarding completion, keeps disconnected-node states blocked, and updates the focused logic test.

## Reproduction Assessment

Yes. Current main has a pure helper path where `isConnected = false` and `isNodeConnected = true` returns false, while `NodeRuntime` displays that same runtime condition as `Connected (operator offline)`.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Android onboarding finish gate and its unit test; there is no evidence that gateway pairing, auth, or protocol behavior needs to change.
