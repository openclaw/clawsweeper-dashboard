## Summary

Keep open: current main and v2026.5.12 still require both operator and node connectivity before Android onboarding can finish, so the reported `Connected (operator offline)` state remains blocked. The focused target-repo fix at https://github.com/openclaw/openclaw/pull/61269 was stale-closed unmerged, and the only open same-head PR is in a fork rather than the target repository.

## What This Changes

Allow Android onboarding to finish when the device node is connected even if the operator session is offline.

## Best Possible Solution

Ship a small Android app fix that treats node connectivity as sufficient for final onboarding completion, keeps disconnected-node states blocked, updates focused unit coverage, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes, source-level: current main returns false for `isConnected = false` and `isNodeConnected = true`, while `NodeRuntime` can display that same state as `Connected (operator offline)`. I did not run a device/emulator onboarding pass in this read-only sweep.
