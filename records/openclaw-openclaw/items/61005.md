## Summary

Keep open: current `main` and the latest release still require both operator and node connectivity before Android onboarding can finish, so the reported `Connected (operator offline)` state remains blocked. There is an open closing PR with the same focused fix path, so the issue should stay open until that fix is reviewed, landed, or explicitly rejected.

## What This Changes

The issue reports Android onboarding staying stuck on the final Connect step when the node is paired and connected but the operator session is offline.

## Best Possible Solution

Land or replace the focused Android onboarding fix so node connectivity is sufficient for final onboarding completion, disconnected-node states remain blocked, and `OnboardingFlowLogicTest` locks that contract.
