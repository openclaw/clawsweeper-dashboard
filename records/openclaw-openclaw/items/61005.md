## Summary

Keep open: current main still blocks final Android onboarding when the node is connected but the operator session is offline, and open PR #61269 is the focused unmerged fix candidate.

## What This Changes

Allow Android onboarding to complete when the device node is paired and connected even if the operator session is offline.

## Best Possible Solution

Land a rebased #61269 or an equivalent Android app change that treats node connectivity as sufficient for final onboarding while keeping disconnected-node states blocked and updating `OnboardingFlowLogicTest`.

## Reproduction Assessment

Yes. Current main has a pure helper path where `isConnected = false` and `isNodeConnected = true` returns false, while `NodeRuntime` explicitly displays `Connected (operator offline)` for that runtime condition.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Android onboarding finish gate and its unit test; the evidence does not point to a gateway, pairing, or protocol change.
