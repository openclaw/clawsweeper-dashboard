## Summary

Keep open. Current main still reproduces the Android onboarding blocker: the finish gate requires both operator and node connectivity, while the runtime explicitly models the reported node-connected/operator-offline state. The focused closing PR #61269 remains open and unmerged, so this issue should stay open until that PR is rebased/landed or replaced by an equivalent fix.

## What This Changes

The issue asks to unblock Android onboarding completion when a paired Android node is connected to the gateway but the operator session is offline.

## Best Possible Solution

Land a rebased #61269 or an equivalent Android app change that treats node connectivity as sufficient for final onboarding completion, keeps truly disconnected states blocked, and updates `OnboardingFlowLogicTest` to lock in the node-connected/operator-offline contract.

## Reproduction Assessment

Yes. The focused current-main reproduction is the pure helper state `isConnected = false` and `isNodeConnected = true`: it returns false today, and `NodeRuntime` can produce `Connected (operator offline)` for the same state.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Android onboarding completion gate and its unit test; no gateway, pairing, or protocol change is needed for the reported blocker.
