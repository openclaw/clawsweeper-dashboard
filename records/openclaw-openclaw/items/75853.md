## Summary

Keep open. Current main intentionally fail-closes external hook transform paths, but the report still identifies a narrow diagnostics/docs gap because this invalid config is accepted by config schema and daemon/doctor preflight before aborting Gateway runtime resolution.

## What This Changes

Add a recovery diagnostic for hook mapping configs whose `hooks.transformsDir` points outside the canonical hook transforms directory.

## Best Possible Solution

Preserve the dynamic-import containment rule and add a preflight/doctor/status hint that tells users to move transform modules under `~/.openclaw/hooks/transforms` or use the appropriate plugin webhook flow.

## Reproduction Assessment

Yes. A config with hooks enabled, at least one mapping transform, and `hooks.transformsDir` outside `<configDir>/hooks/transforms` reaches `resolveHookMappings` during Gateway startup and throws the same `Hook transformsDir` containment error described in the report.

## Solution Assessment

Yes for the narrowed direction. The maintainable fix is a targeted diagnostic and docs update, not relaxing compatibility for arbitrary workspace skill paths in dynamic imports.
