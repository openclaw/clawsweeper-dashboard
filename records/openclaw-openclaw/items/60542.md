## Summary

Keep open. Current main still reuses a finite `updatedAt` main-session entry without checking whether a terminal row is stale relative to its transcript; the shipped adjacent stale-running fix does not cover this terminal-row divergence, and the targeted PR for this shape was closed unmerged.

## What This Changes

The issue asks OpenClaw to reconcile a persisted main-session registry entry when it is terminal but stale relative to its transcript and blocks new input.

## Best Possible Solution

Add a narrow main-session initialization reconciliation that detects terminal rows older than their transcript or lifecycle tail and safely rolls forward or rotates without changing normal terminal sessions or explicit reset behavior.

## Reproduction Assessment

Yes, as a source-level regression path. Seed `agent:main:main` with a terminal row whose `updatedAt` or `endedAt` is older than its transcript, call `initSessionState`, and current main will reuse the row instead of reconciling transcript freshness.

## Solution Assessment

Yes. A targeted `initSessionState` or session-file helper fix with regression coverage is the narrowest maintainable path; a broad startup migration or generic store sweep would carry more risk.
