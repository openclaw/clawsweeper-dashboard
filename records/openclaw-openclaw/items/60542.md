## Summary

Keep open. Current main still has the terminal-row divergence gap: main-session initialization reuses a persisted row based on `sessionId` plus finite `updatedAt` without checking whether a terminal row is stale relative to the transcript, and the only targeted PR for this exact shape was closed unmerged.

## What This Changes

The issue asks OpenClaw to reconcile stale persisted main-session rows whose terminal registry snapshot is older than the linked transcript so new prompts do not wedge after restart or recovery.

## Best Possible Solution

Add a targeted main-session initialization reconciliation with regression coverage that detects terminal rows older than their transcript or lifecycle tail and safely rolls forward or rotates the session.

## Reproduction Assessment

Yes, source-level with high confidence. Seed `agent:main:main` with a terminal row whose `updatedAt` or `endedAt` predates its transcript, then `initSessionState` will reuse it because current freshness checks ignore transcript tail freshness.

## Solution Assessment

Yes. A narrow fix in `initSessionState` or the session-file helper is the most maintainable path; a broad startup migration or generic store sweep would be riskier than reconciling only the stale terminal main-session case.
