## Summary

Keep open. Current main still has adjacent lifecycle and restart-recovery fixes, but I did not find a repair for the specific stale terminal main-session row being older than its transcript; the targeted PR for this shape was closed unmerged.

## What This Changes

The issue requests reconciliation when a persisted main-session registry entry becomes stale relative to its transcript and wedges new input across surfaces.

## Best Possible Solution

Add a narrow main-session initialization reconciliation that detects terminal rows older than their transcript/lifecycle tail and rolls them forward or rotates safely without changing normal terminal sessions or explicit reset behavior.

## Reproduction Assessment

Yes. A focused regression can seed `agent:main:main` with a terminal row whose `updatedAt`/`endedAt` is older than its transcript, call `initSessionState`, and assert current main reuses the stale row instead of reconciling it.

## Solution Assessment

Yes. A targeted initializer/helper fix with regression coverage is the narrowest maintainable path; a broad startup migration or generic store sweep would be riskier.
