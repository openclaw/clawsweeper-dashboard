## Summary

Keep open. Current main still reuses a fresh-enough persisted main-session row without reconciling terminal row freshness against the referenced transcript, the latest shipped tag shows the same behavior, and the exact stale-terminal-row PR was closed unmerged.

## What This Changes

The issue asks OpenClaw to keep persisted main-session registry rows consistent with their linked transcripts so stale terminal rows cannot wedge new input after restart or recovery.

## Best Possible Solution

Add targeted main-session initialization or session-file reconciliation that detects terminal rows older than their referenced transcript or lifecycle tail and safely rotates to a fresh main session with focused regression coverage.

## Reproduction Assessment

Yes, at source level: seed a main-session store row with terminal status and an older updatedAt/endedAt than the linked transcript tail, and current initSessionState can still reuse that row without transcript freshness reconciliation. I did not execute a live gateway repro in this read-only pass.

## Solution Assessment

Yes, a narrow reconciliation in initSessionState or the session-file helper is the maintainable fix. The stale-running PR alone is not sufficient unless it is explicitly extended to cover terminal rows stale relative to transcripts.
