## Summary

Keep open. Current main still reuses a persisted main-session row from session id plus freshness timestamps without reconciling whether a terminal row is stale relative to its transcript, and the only exact fix PR was closed unmerged.

## What This Changes

The issue asks OpenClaw to reconcile stale persisted main-session rows that diverge from their transcript so new input does not wedge after restart or recovery.

## Best Possible Solution

Add targeted main-session initialization reconciliation that detects terminal rows older than their transcript or lifecycle tail and safely rolls forward or rotates the session with regression coverage.

## Reproduction Assessment

Yes, source-level with high confidence. Seed the main session store with a terminal row whose updatedAt or endedAt predates newer transcript content, then initSessionState can reuse it because current freshness checks ignore transcript tail freshness.

## Solution Assessment

Yes. A narrow reconciliation in initSessionState or the session-file helper is the maintainable fix; the closed PR's core idea should be reimplemented cleanly without unrelated files and with focused regression coverage.
