## Summary

Keep open. Current main has the narrower stale-running preservation fix, but it still does not reconcile a terminal main-session row that is stale relative to its transcript, and the direct PR for this shape was closed unmerged.

## What This Changes

The issue asks OpenClaw to keep the persisted main-session registry row consistent with its transcript so stale terminal rows do not wedge new input after restart or recovery.

## Best Possible Solution

Add a targeted main-session initialization reconciliation that detects terminal registry rows older than their linked transcript or lifecycle tail and safely rolls forward or rotates without changing normal terminal sessions.

## Reproduction Assessment

Yes. Source inspection gives a focused current-main regression path: seed `agent:main:main` with a fresh terminal row older than its transcript, call `initSessionState`, and it reuses the row instead of reconciling or rotating it.

## Solution Assessment

Yes. A narrow initializer/session-file helper fix with regression coverage is the maintainable path; a broad startup migration or whole-store sweep would carry more risk.
