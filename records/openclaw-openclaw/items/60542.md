## Summary

Keep open. Current main still has adjacent lifecycle persistence fixes, but I did not find reconciliation for the reported stale terminal main-session row being older than its transcript, and the targeted PR for this shape was closed unmerged.

## What This Changes

The issue asks OpenClaw to keep the persisted main-session registry row consistent with its transcript so stale terminal rows cannot wedge new input after restart or recovery.

## Best Possible Solution

Add a narrow main-session initialization reconciliation that detects terminal rows older than their transcript or lifecycle tail and safely rolls forward or rotates without changing normal terminal sessions or explicit reset behavior.

## Reproduction Assessment

Yes. A focused regression can seed `agent:main:main` with a terminal row whose `updatedAt` or `endedAt` is older than its transcript, call `initSessionState`, and assert current main reuses the stale row instead of reconciling it.

## Solution Assessment

Yes. A targeted initializer/session-file helper fix with regression coverage is the narrowest maintainable path; a broad startup migration or generic store sweep would be riskier.
