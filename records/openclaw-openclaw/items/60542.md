## Summary

Keep open. Current main still has the terminal-row divergence gap: main-session initialization reuses a finite `updatedAt` entry without checking whether a terminal row is stale relative to its transcript, and the earlier targeted PR for this exact shape was closed unmerged.

## What This Changes

The issue asks OpenClaw to reconcile stale persisted main-session registry rows that can diverge from their transcript and wedge new input across surfaces.

## Best Possible Solution

Add a narrow main-session initialization reconciliation that detects terminal rows older than their transcript or lifecycle tail and safely rolls forward or rotates the session without changing normal terminal sessions or explicit reset behavior.

## Reproduction Assessment

Yes, source-level rather than live. Seed `agent:main:main` with a terminal row whose `updatedAt` or `endedAt` is older than its transcript, call `initSessionState`, and current main will reuse the row instead of reconciling transcript freshness.

## Solution Assessment

Yes. A targeted fix in `initSessionState` or the session-file helper with regression coverage is the narrowest maintainable path; a broad startup migration or generic store sweep would carry more risk.
