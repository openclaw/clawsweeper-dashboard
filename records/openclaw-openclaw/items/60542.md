## Summary

Keep open. Current main has adjacent lifecycle and restart-recovery fixes, but the specific stale terminal main-session row that is older than its transcript is still not reconciled during session initialization, and the related targeted PR was closed unmerged.

## What This Changes

Ensure persisted main-session registry rows stay consistent with newer transcript/lifecycle state so new prompts remain accepted after restart or recovery.

## Best Possible Solution

Add a focused recovery path in session initialization, or a small helper it calls, that detects main-session terminal rows whose transcript or lifecycle tail is newer than the registry snapshot and safely reconciles or rotates them into an input-accepting state while preserving normal terminal sessions, explicit resets, provider-owned sessions, and running-session restart recovery.

## Reproduction Assessment

Yes. A focused regression can seed `agent:main:main` with terminal status and older updatedAt/endedAt, pair it with a newer transcript containing a later lifecycle or timeout tail, then call `initSessionState` and assert current main reuses/preserves the stale row instead of reconciling it; I did not run it because this review is read-only.

## Solution Assessment

Yes. The narrowest maintainable fix is targeted main-session stale-row reconciliation in the owner session-init path, informed by the closed PR, rather than a broad startup migration or generic store sweep.
