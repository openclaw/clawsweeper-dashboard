## Summary

Keep open for maintainer review: this PR does not add the promised WhatsApp `onReasoningStream` sink, and its shared runner-level filtering risks removing reasoning payloads before channel-specific lanes can handle them.

## What This Changes

The PR changes shared auto-reply runner and payload handling to filter `isReasoning` payloads, adds focused tests, and adds a changelog entry for WhatsApp reasoning suppression.

## Best Possible Solution

Use the narrower WhatsApp-owned callback fix from the reviewed #75258 shape, preserve shared reasoning lanes, and keep #74886 open for fallback and stability follow-up.

## Reproduction Assessment

Yes for the PR defect: the patch and file list show no WhatsApp dispatch change, while current main still lacks `replyOptions.onReasoningStream`. A safe live WhatsApp reproduction was not attempted because the reported behavior can leak private reasoning to an external channel.

## Solution Assessment

No. The current diff suppresses reasoning in shared runner/final-payload paths and omits the WhatsApp callback; the safer fix is to add the WhatsApp no-op sink and regression coverage without globally stripping channel-specific reasoning payloads.
