## What This Changes

The issue asks for `channels.telegram.pollingStallThresholdMs` to be honored by Telegram's default isolated polling ingress instead of only the embedded polling path.

## Best Possible Solution

Land or replace the focused linked PR so isolated ingress tracks worker getUpdates liveness and restarts on the existing configured threshold, with regression tests and runtime proof.
