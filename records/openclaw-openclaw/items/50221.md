## Summary

Keep open because the branch targets a real security boundary but currently regresses least-privilege command access and is stale against current plugin runtime config APIs.

## What This Changes

The branch adds operator.admin gating for internal chat command authorization, /exec default persistence, and plugin runtime config writes, plus scope propagation into plugin commands and tests.

## Best Possible Solution

Model internal command capabilities centrally so read-only commands remain available to operator.write, mutating command and plugin config writes require operator.admin, and the runtime config guard covers every current mutation API.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the branch makes operator.write chat.send callers unauthorized before command-specific gates run, and the current runtime config API shows the sink guard is incomplete.

## Solution Assessment

No. The security goal is valid, but the branch should be reworked around command capability policy plus complete runtime config mutation coverage rather than a blanket admin gate on internal chat commands.
