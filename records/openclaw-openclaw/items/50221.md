## Summary

Keep open: the hardening direction is valid, but the current patch is not safe to merge because it blanket-gates internal chat commands before command-specific policy can preserve read-only access, and it misses current main's primary plugin runtime config mutation APIs.

## What This Changes

The branch adds admin-scope enforcement for internal `chat.send` command authorization, `/exec` directive persistence, and plugin runtime config writes, with plugin command scope propagation and regression tests.

## Best Possible Solution

Model internal command capabilities centrally so read-only internal commands remain available to `operator.write`, mutating command/runtime config writes require `operator.admin`, and every plugin runtime config sink shares one approved guard.

## Reproduction Assessment

Yes. Source inspection against the PR diff and current main is enough: the `CommandAuthorized` change rejects commands before per-command scope gates, and current main exposes runtime config mutation sinks the PR does not guard.

## Solution Assessment

No. The security hardening goal is valid, but this patch should be redesigned around a maintainer-approved command capability model and complete runtime config sink coverage.
