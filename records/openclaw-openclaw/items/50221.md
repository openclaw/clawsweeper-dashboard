## Summary

Keep open: the hardening goal is valid, but the current branch is not safe to merge because it conflates admin scope with all command authorization and misses current plugin config mutation sinks.

## What This Changes

The PR adds admin-scope enforcement for internal chat command authorization, /exec persistence, and plugin runtime config writes, plus scope propagation through plugin commands.

## Best Possible Solution

Model internal command capabilities centrally so read-only commands remain available to operator.write, mutating commands require operator.admin, and every runtime config write sink shares one approved guard.

## Reproduction Assessment

Yes. Source inspection on the PR diff and current main shows the blanket chat.send gate runs before command-specific read/write gates, and current main exposes runtime config write APIs the PR does not guard.

## Solution Assessment

No. The hardening direction is valid, but the current patch should be revised around a maintainer-approved command capability model and complete runtime config sink coverage.
