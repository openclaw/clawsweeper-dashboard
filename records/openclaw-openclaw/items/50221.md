## Summary

Keep open; the hardening goal is real, but the current diff blocks allowed read-only internal commands, drops allowed mixed directive updates, and only guards the deprecated runtime config write API while current mutation APIs remain outside the proposed sink guard.

## What This Changes

The PR requires `operator.admin` for internal `chat.send` command authorization, `/exec` default persistence, and plugin runtime config writes by propagating gateway scopes into directive and plugin command paths.

## Best Possible Solution

Model internal command capabilities centrally so read-only commands remain available to `operator.write`, mutating operations require `operator.admin`, and every approved runtime config write API shares the same guard.

## Reproduction Assessment

Yes. Source inspection of the PR head and current main gives a high-confidence path: an `operator.write` `chat.send` caller would be denied before `/config show` policy runs, mixed directive-only messages can lose allowed updates, and current runtime config mutation APIs are not covered by the PR guard.

## Solution Assessment

No. The patch is not the best fix because it uses a coarse admin gate and incomplete sink coverage where the current code needs command-level capability policy and complete runtime config write enforcement.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve read-only internal commands for write-scoped clients:** `src/gateway/server-methods/chat.ts:1308`
  - body: This makes every internal `chat.send` caller without `operator.admin` set `CommandAuthorized=false`. `resolveCommandSenderAuthorization` treats that as a top-level gate, so read-only commands intentionally available to `operator.write` clients, such as `/config show`, are rejected before command-specific policy can allow them.
  - confidence: 0.93
- **[P2] Let allowed directives survive rejected exec persistence:** `src/auto-reply/reply/directive-handling.impl.ts:279-283`
  - body: This early return fires for directive-only messages with any rejected `/exec` options before later code persists the other directive fields. A message like `/think high /exec host=gateway` would drop the allowed thinking update along with the rejected exec persistence.
  - confidence: 0.88
- **[P2] Guard all runtime config write APIs:** `src/plugins/runtime/runtime-config.ts:9-20`
  - body: The guard wraps only deprecated `writeConfigFile`, while current plugin runtime writes primarily use `mutateConfigFile` and `replaceConfigFile`. If sink-level hardening is approved, those active mutation APIs need equivalent enforcement.
  - confidence: 0.86
