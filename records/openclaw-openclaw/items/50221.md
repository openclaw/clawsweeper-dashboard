## Summary

Keep open: the hardening goal is valid, but the patch currently blocks least-privilege read-only commands and misses the current primary runtime config mutation APIs. The author has also asked for human direction on the command capability policy, so this should not be closed or auto-repaired as conservative cleanup.

## What This Changes

The PR adds internal `operator.admin` checks for `chat.send` command authorization, `/exec` default persistence, and plugin runtime config writes, plus plugin command scope propagation and regression tests.

## Best Possible Solution

Model internal command capabilities centrally so read-only commands remain available to `operator.write`, mutating command and plugin config writes require `operator.admin`, and every runtime config mutation API is covered by one shared policy.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the PR makes `operator.write` `chat.send` callers unauthorized before command-specific gates run, and current main exposes unguarded `mutateConfigFile` and `replaceConfigFile` runtime mutation paths.

## Solution Assessment

No. The hardening direction is sound, but this patch should be reworked around command capability policy plus complete runtime config mutation coverage rather than a blanket admin gate on internal chat commands.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve read-only commands for write-scoped clients:** `src/gateway/server-methods/chat.ts:1308`
  - body: This makes internal `chat.send` set `CommandAuthorized=false` unless the gateway client has `operator.admin`. That rejects all slash commands before command-specific gates can allow read-only paths such as `/config show` or `/acp sessions`, so least-privilege clients would need admin for inspection commands.
  - confidence: 0.93
- **[P2] Let allowed directives survive rejected exec persistence:** `src/auto-reply/reply/directive-handling.impl.ts:279-283`
  - body: The early return for empty cleaned directive-only messages runs before the later per-field persistence logic. A mixed message such as `/think high /exec host=gateway` from an internal `operator.write` client reports the exec denial and skips the allowed thinking update.
  - confidence: 0.88
- **[P2] Guard every runtime config mutation API:** `src/plugins/runtime/runtime-config.ts:10-21`
  - body: The scope check wraps only deprecated `writeConfigFile`, but current plugin runtime exposes `mutateConfigFile` and `replaceConfigFile` as the primary mutation APIs. Plugin commands using those APIs can still mutate config without this admin check after the branch is rebased.
  - confidence: 0.84
