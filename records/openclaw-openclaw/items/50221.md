## Summary

Current main has shipped narrower internal-scope gates that cover the observable cleanup without the PR branch's blanket chat.send admin gate, so this stale branch should close rather than merge.

## What This Changes

The PR branch gates internal chat command authorization, exec default persistence, and plugin runtime config writes behind admin scope, propagating gateway scopes into plugin command execution with tests.

## Best Possible Solution

Keep the shipped current-main authorization model: chat.send carries gateway scopes, directive and owner-specific mutation sinks enforce admin, and plugin commands use explicit requiredScopes metadata instead of a blanket admin default.

## Reproduction Assessment

Yes. Source inspection of the PR diff is enough: setting CommandAuthorized=false for operator.write chat.send callers makes generic command authorization reject before narrower command gates can run, while current main preserves that context separately.

## Solution Assessment

No. The PR branch is not the best solution because its blanket chat.send gate weakens least privilege; current main's shipped targeted gates and requiredScopes contract are the narrower maintainable path.
