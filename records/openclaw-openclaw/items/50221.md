## Summary

Keep open for maintainer review. The security hardening goal is plausible and the PR includes real behavior proof, but the branch still introduces a least-privilege regression for internal slash commands and misses current runtime config mutation APIs after main moved on.

## What This Changes

The PR adds internal gateway scope propagation and `operator.admin` checks for `chat.send` command authorization, `/exec` default persistence, and plugin runtime config writes.

## Best Possible Solution

Model internal command capabilities centrally so write-scoped clients retain read-only commands, mutating slash-command paths require `operator.admin`, and every plugin runtime config mutation API shares one internal-scope gate.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the PR makes write-scoped internal `chat.send` callers unauthorized before command-specific gates run, and current main exposes runtime config mutation APIs that the patch does not guard.

## Solution Assessment

No. The hardening direction is sound, but this patch should be reworked around command capability policy and complete runtime config mutation coverage instead of a blanket admin gate on internal chat commands.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve read-only commands for write-scoped clients:** `src/gateway/server-methods/chat.ts:1308`
  - body: This makes internal `chat.send` set `CommandAuthorized=false` unless the client has `operator.admin`, so command authorization rejects slash commands before per-command gates can allow read-only paths such as `/config show`, `/acp sessions`, `/status`, or `/whoami`. Least-privilege clients would need admin credentials for inspection commands.
  - confidence: 0.93
- **[P2] Let allowed directives survive rejected exec persistence:** `src/auto-reply/reply/directive-handling.impl.ts:279-283`
  - body: This early return runs before per-field persistence. A directive-only message such as `/think high /exec host=gateway` leaves `cleaned` empty after parsing, so the allowed thinking update is dropped along with the rejected `/exec` persistence.
  - confidence: 0.88
- **[P2] Guard every runtime config mutation API:** `src/plugins/runtime/runtime-config.ts:10-21`
  - body: The scope check wraps only deprecated `writeConfigFile`, but current plugin runtime mutations use `mutateConfigFile` and `replaceConfigFile`. Plugin commands using those APIs can still mutate config without this admin check after the branch is rebased.
  - confidence: 0.84
