## Summary

Keep open: the hardening goal and live proof are useful, but the patch is not mergeable as-is because it blocks read-only internal slash-command flows and misses current runtime config mutation APIs.

## What This Changes

The PR adds internal `operator.admin` checks for gateway `chat.send` command authorization, `/exec` default persistence, plugin runtime config writes, and plugin command scope propagation.

## Best Possible Solution

Model internal command capabilities centrally so read-only commands remain available to `operator.write`, mutating commands require `operator.admin`, and all plugin runtime config write APIs share one approved sink-level guard.

## Reproduction Assessment

Yes. Source inspection on the PR head and current main shows the blanket `chat.send` gate runs before command-specific read/write gates, and current main exposes runtime config write APIs that the PR does not guard.

## Solution Assessment

No. The hardening direction is valid, but the patch should be replaced or revised around a maintainer-approved command capability model and complete runtime config sink coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve read-only commands for write-scoped clients:** `src/gateway/server-methods/chat.ts:1311`
  - body: This makes internal `chat.send` set `CommandAuthorized=false` unless the client has `operator.admin`, so command authorization rejects slash commands before per-command gates can allow read-only paths such as `/config show`, `/status`, or `/whoami`. Least-privilege clients would need admin credentials for inspection commands.
  - confidence: 0.93
- **[P2] Let allowed directives survive rejected exec persistence:** `src/auto-reply/reply/directive-handling.impl.ts:282-285`
  - body: This early return runs before the per-field persistence block. A directive-only message such as `/think high /exec host=gateway` leaves `cleaned` empty after parsing, so the allowed thinking update is dropped along with the rejected `/exec` persistence.
  - confidence: 0.88
- **[P2] Guard current runtime config mutation APIs:** `src/plugins/runtime/runtime-config.ts:9-20`
  - body: The scope check wraps only the deprecated `writeConfigFile` API from the branch base, but current main plugin runtime mutations use `mutateConfigFile` and `replaceConfigFile`. Those APIs need the same internal-scope enforcement if sink-level hardening is chosen.
  - confidence: 0.84
