## Summary

Current main already covers the useful authorization hardening in a safer shape: it preserves least-privilege read-only command paths while admin-gating concrete mutations and forwarding gateway scopes into plugin command execution. The PR branch should not merge because its blanket `chat.send` command gate regresses least privilege and its runtime-config sink guard covers only a deprecated API.

## What This Changes

The PR changes internal gateway command scope handling so `operator.admin` is required for `chat.send` slash authorization, `/exec` default persistence, and plugin runtime config writes while propagating gateway scopes into plugin command execution.

## Best Possible Solution

Keep the current-main command-level scope model: preserve read-only internal command access, require `operator.admin` for concrete mutations, and only revisit generic plugin config-write sinks through a separate maintainer-approved security policy.

## Reproduction Assessment

Yes. Source inspection of the PR diff against current main gives a high-confidence path for the branch regressions and confirms current main now covers the useful hardening paths.

## Solution Assessment

No for the PR branch. Current main's per-command/path scope checks are narrower and more maintainable than the branch's blanket `chat.send` admin gate plus deprecated-only runtime-config wrapper.

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve read-only commands for write-scoped clients:** `src/gateway/server-methods/chat.ts:1308`
  - body: Setting `CommandAuthorized=false` for every non-admin internal `chat.send` caller rejects slash commands before command-specific policy can allow read-only paths such as `/config show`. That pushes least-privilege clients toward admin credentials for inspection workflows.
  - confidence: 0.93
- **[P2] Keep allowed directives when rejecting exec persistence:** `src/auto-reply/reply/directive-handling.impl.ts:279-283`
  - body: This early return fires for directive-only messages that include `/exec` options before later persistence can save other valid directives. A mixed message such as `/think high /exec host=gateway` would reject exec and also drop the allowed thinking change.
  - confidence: 0.88
- **[P2] Guard active runtime config write APIs:** `src/plugins/runtime/runtime-config.ts:9-20`
  - body: The branch wraps deprecated `writeConfigFile`, but current runtime config exposes and plugins use `mutateConfigFile` and `replaceConfigFile`. If sink-level hardening is accepted, those active write paths need equivalent enforcement.
  - confidence: 0.86
