## Summary

Keep open: the hardening goal is legitimate, and the PR has real behavior proof, but the current diff is not mergeable because it makes internal chat.send slash commands admin-only before command-specific policy can preserve read-only commands and it does not cover the current runtime config mutation APIs.

## What This Changes

The PR adds operator.admin checks for internal chat.send command authorization, /exec default persistence, and plugin runtime config writes, plus gateway-scope propagation into plugin command execution.

## Best Possible Solution

Model internal command capabilities centrally so read-only commands stay available to operator.write, mutating command and plugin config writes require operator.admin, and every runtime config mutation API shares one internal-scope gate.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the PR makes write-scoped internal chat.send callers unauthorized before command-specific gates run, and current main exposes runtime config mutation APIs that the patch does not guard.

## Solution Assessment

No. The hardening direction is sound, but this patch needs a maintainer-approved command capability model and complete runtime config sink coverage instead of a blanket chat.send admin gate.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve read-only commands for write-scoped clients:** `src/gateway/server-methods/chat.ts:1308`
  - body: This makes internal chat.send set CommandAuthorized=false unless the client has operator.admin, so command authorization rejects slash commands before per-command gates can allow read-only paths such as /config show, /acp sessions, /status, or /whoami.   - confidence: 0.93

  - confidence: 0.88
- **[P2] Guard every runtime config mutation API:** `src/plugins/runtime/runtime-config.ts:10-21`
  - body: The scope check wraps only deprecated writeConfigFile, but current plugin runtime mutations use mutateConfigFile and replaceConfigFile. Plugin commands using those APIs can still mutate config without this admin check after the branch is rebased.
  - confidence: 0.84
