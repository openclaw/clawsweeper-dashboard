## Summary

Keep open: the PR addresses a real internal-scope hardening area, but the branch still has blocking authorization and security-design issues and overlaps with newer current-main work without cleanly replacing it.

## What This Changes

The PR changes internal gateway chat, directive persistence, plugin command execution, runtime config writes, and tests so selected internal-channel mutations require `operator.admin` and plugin command scopes propagate into runtime context.

## Best Possible Solution

Model internal slash-command capabilities centrally or per command, preserve read-only `operator.write` inspection paths, require `operator.admin` for mutating operations, and enforce any approved plugin config-write policy through active APIs or explicit command metadata.

## Reproduction Assessment

Yes. Source inspection of the PR diff against current main gives a high-confidence path: non-admin internal `chat.send` commands are broadly denied, mixed directive-only updates can be dropped, and the runtime-config guard misses active write APIs.

## Solution Assessment

No. The PR hardens real surfaces, but the blanket chat gate is too coarse and the runtime-config sink guard is incomplete; the safer path needs maintainer-approved command capability policy plus complete active API enforcement.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve read-only commands for write-scoped clients:** `src/gateway/server-methods/chat.ts:1308`
  - body: This makes every non-admin internal `chat.send` slash command fail before command-specific policy runs. Current main intentionally allows read-only internal paths such as `/config show` and read-only ACP commands for `operator.write`, so this would force admin credentials for inspection-only workflows.
  - confidence: 0.93
- **[P2] Keep allowed directives when rejecting exec persistence:** `src/auto-reply/reply/directive-handling.impl.ts:279-283`
  - body: This early return runs before persistence of other directive-only updates once `/exec` has options and `cleaned` is empty. Because parsing removes all directives from `cleaned`, input like `/think high /exec host=gateway` drops the allowed thinking update instead of only rejecting exec persistence.
  - confidence: 0.88
- **[P2] Guard active runtime config write APIs:** `src/plugins/runtime/runtime-config.ts:9-20`
  - body: The new guard wraps only deprecated `writeConfigFile`, but current runtime config mutations use `mutateConfigFile` and `replaceConfigFile`. If sink-level hardening is intended, the active APIs need equivalent enforcement.
  - confidence: 0.86
