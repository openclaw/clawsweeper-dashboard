## Summary

Keep open: this security-sensitive PR has useful proof, but the current diff still has blocking policy/correctness issues and the author has asked for maintainer direction on command capability modeling.

## What This Changes

The branch propagates gateway scopes into plugin command/runtime paths and requires `operator.admin` for internal `chat.send` command authorization, `/exec` persistence, and plugin runtime config writes.

## Best Possible Solution

Model internal command capabilities centrally or per command, preserve read-only `operator.write` inspection paths, require `operator.admin` for mutating operations, and apply any approved plugin config-write policy consistently to active APIs or explicit plugin command metadata.

## Reproduction Assessment

Yes. Source inspection of the PR diff against current main gives a high-confidence path for the three blockers: broad command denial, mixed directive persistence loss, and incomplete runtime-config sink coverage.

## Solution Assessment

No. The PR hardens real surfaces, but the blanket chat gate is too coarse and the sink guard is incomplete; the safer path needs maintainer-approved command capability policy plus complete enforcement.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve read-only commands for write-scoped clients:** `src/gateway/server-methods/chat.ts:1308`
  - body: This makes every non-admin internal `chat.send` slash command fail before command-specific policy runs. Current main still allows read-only internal paths such as `/config show` for `operator.write` while requiring admin for writes, so this would force admin credentials for inspection workflows.
  - confidence: 0.93
- **[P2] Keep allowed directives when rejecting exec persistence:** `src/auto-reply/reply/directive-handling.impl.ts:279-283`
  - body: This early return runs before persistence of other directive-only updates once `/exec` has options and `cleaned` is empty. Because directive parsing strips all directives from `cleaned`, input like `/think high /exec host=gateway` returns before saving the allowed thinking update.
  - confidence: 0.88
- **[P2] Guard active runtime config write APIs:** `src/plugins/runtime/runtime-config.ts:9-20`
  - body: The guard wraps only deprecated `writeConfigFile`. Current runtime config API exposes `mutateConfigFile` and `replaceConfigFile` as active write paths, so sink-level hardening would still be bypassed by current plugin code.
  - confidence: 0.86
