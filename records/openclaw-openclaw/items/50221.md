## Summary

Keep open: the hardening target is plausible, but the current diff blocks read-only internal commands, drops allowed directive updates in mixed `/exec` messages, and does not cover current runtime config mutation APIs.

## What This Changes

The PR requires `operator.admin` for internal `chat.send` command authorization, `/exec` default persistence, and plugin runtime config writes, with plugin command scope propagation and regression tests.

## Best Possible Solution

Model internal command capabilities centrally so read-only commands remain available to `operator.write`, mutating command and plugin config writes require `operator.admin`, and every runtime config mutation API uses one shared internal-scope gate.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the PR makes write-scoped internal `chat.send` callers unauthorized before command-specific gates run, and current main exposes runtime config mutation APIs that the patch does not guard.

## Solution Assessment

No. The hardening direction is sound, but this patch should be reworked around command capability policy and complete runtime config mutation coverage instead of a blanket admin gate on internal chat commands.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve read-only commands for write-scoped clients:** `src/gateway/server-methods/chat.ts:1308`
  - body: This makes internal `chat.send` set `CommandAuthorized=false` unless the client has `operator.admin`, so command authorization rejects every slash command before command-specific gates can allow read-only paths such as `/config show` or `/acp sessions`.   - confidence: 0.93

  - confidence: 0.88
- **[P2] Guard every runtime config mutation API:** `src/plugins/runtime/runtime-config.ts:10-21`
  - body: The scope check wraps only deprecated `writeConfigFile`, but current plugin runtime mutations use `mutateConfigFile` and `replaceConfigFile`. Plugin commands using those APIs can still mutate config without this admin check after the branch is rebased.
  - confidence: 0.84
