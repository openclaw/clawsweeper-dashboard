## Summary

Keep open: the hardening direction is useful, and current main already covers part of the `/exec` persistence concern, but this PR still has blocking authorization-policy issues before it can merge.

## What This Changes

The branch propagates gateway client scopes into plugin command runtime context and requires `operator.admin` for internal `chat.send` slash authorization, `/exec` persistence, and plugin runtime config writes.

## Best Possible Solution

Model internal command capabilities centrally or per command, preserve read-only `operator.write` inspection paths, require `operator.admin` for mutations, and apply the approved plugin config-write policy consistently to active runtime config write APIs.

## Reproduction Assessment

Yes. Source inspection of the PR diff against current main gives a high-confidence path for the command gate, mixed directive, and runtime-config sink blockers.

## Solution Assessment

No. The PR hardens real surfaces, but the blanket chat gate is too coarse and the runtime-config guard is incomplete; the safer path needs maintainer-approved command capability and plugin write policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve read-only commands for write-scoped clients:** `src/gateway/server-methods/chat.ts:1308`
  - body: This sets `CommandAuthorized=false` for every non-admin internal `chat.send` caller, so command-auth rejects slash commands before command-specific policy can allow read-only paths such as `/config show`. Least-privilege UI or automation clients would need admin credentials for inspection workflows.
  - confidence: 0.93
- **[P2] Keep allowed directives when rejecting exec persistence:** `src/auto-reply/reply/directive-handling.impl.ts:279-283`
  - body: This early return fires for directive-only messages that include `/exec` options before the later persistence code can save other valid directives. A message like `/think high /exec host=gateway` would reject exec and also drop the allowed thinking change.
  - confidence: 0.88
- **[P2] Guard active runtime config write APIs:** `src/plugins/runtime/runtime-config.ts:9-20`
  - body: The guard wraps only deprecated `writeConfigFile`, but current runtime config exposes and plugins use `mutateConfigFile` and `replaceConfigFile`. If sink-level hardening is the policy, those active write paths bypass it.
  - confidence: 0.86
