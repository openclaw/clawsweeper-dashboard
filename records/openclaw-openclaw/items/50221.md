## Summary

Keep this PR open: the security-hardening direction is useful, but the current patch still blocks read-only internal commands and misses the current plugin runtime config mutation sinks, so it needs maintainer direction and rework before merge.

## What This Changes

The PR adds internal `operator.admin` gates for `chat.send` command authorization, `/exec` persistence, and plugin runtime config writes, plus gateway scope propagation into plugin command execution.

## Best Possible Solution

Model internal slash-command capabilities centrally so read-only commands stay available to `operator.write`, mutating commands require `operator.admin`, and all plugin runtime config write sinks share one approved guard.

## Reproduction Assessment

Yes. Source inspection of the PR head and current main shows the blanket `chat.send` gate runs before command-specific read/write gates, and current main exposes runtime config write APIs that the PR does not guard.

## Solution Assessment

No. The hardening goal is valid, but this patch gates too broadly and targets a stale runtime config surface; the safer path is centralized command capability policy plus complete sink coverage.
