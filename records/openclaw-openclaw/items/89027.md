## Summary

Keep open: the direction is useful, but the PR gates completed empty CLI turns on `silentReplyPromptMode` instead of the existing silent-reply policy, so supported group silent-reply paths can still throw `empty_response` and rerun on a fallback model. ## What This Changes

The branch adds a `completedTurn` marker to Claude CLI JSONL result parsing and uses it in CLI runner empty-response handling to skip fallback for completed empty turns when silence is treated as allowed.

## Best Possible Solution

Keep the completed-turn parser signal, pass the existing `allowEmptyAssistantReplyAsSilent` policy into the CLI runner, gate completed empty turns on that policy, add runner tests for both branches, and require redacted real claude-cli/gateway proof.
