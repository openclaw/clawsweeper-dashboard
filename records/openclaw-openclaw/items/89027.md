## What This Changes

The PR adds a `completedTurn` flag to Claude JSONL parsing, skips CLI `empty_response` failover for completed empty result events, and updates parser expectations.

## Best Possible Solution

Gate completed empty CLI results on the existing `allowEmptyAssistantReplyAsSilent` policy, preserve `empty_response` for silent-disallowed contexts, and add real redacted claude-cli proof before merge.

## Reproduction Assessment

Yes, at source level. The PR head can return a successful `EmbeddedAgentRunResult` with no payloads for an empty completed Claude result, and current silent-reply policy is not consulted in that new branch.

## Solution Assessment

No. Carrying completion state is plausible, but the best fix is narrower: use the existing silent-reply policy so only allowed silent contexts suppress `empty_response`.
