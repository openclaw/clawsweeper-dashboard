## Summary

Keep open. The PR targets a real linked memory-core isolation bug, but the proposed patch is not mergeable yet: agent-scoped corpus paths are dropped before recall tracking, shared read-path filtering still falls open without an agent id, and the added Vitest coverage currently contains runtime/type errors.

## What This Changes

This draft PR adds shared-workspace detection, threads currentAgentId through memory-core dreaming, writes shared-workspace session corpus files under per-agent subdirectories, records promotion provenance, and adds isolation tests.

## Best Possible Solution

Implement an explicit agent-scoped dreaming contract that filters both write and read paths, recognizes per-agent corpus paths throughout recall/promotion/source checks, handles legacy pooled corpus data deliberately, and proves the behavior with a two-agent shared-workspace regression.

## Reproduction Assessment

Yes. A source-level reproduction is a shared workspace where currentAgentId writes memory/.dreams/session-corpus/<agent>/<day>.txt; the existing recall-path regex rejects that path, and a missing currentAgentId lets shared recall reads pass through unfiltered.

## Solution Assessment

No. The direction is plausible, but the current patch is not the narrowest maintainable fix because it changes corpus layout without updating the recall contract and leaves a fail-open read path for shared workspaces without agent identity.
