## Summary

Keep open. The PR is aimed at a real shared-workspace memory-isolation bug, but the latest diff still leaves manual promotion/REM ranking unscoped and makes a public SDK type incompatible for existing plugin consumers, so it is not ready to merge.

## What This Changes

The PR adds shared-workspace metadata, agent/session identity threading, per-agent dreaming corpus/provenance handling, rank filtering, isolation tests, and an audit report for memory-core dreaming.

## Best Possible Solution

Make the agent-boundary contract mandatory or derived for every durable ranking/promotion reader and writer, keep the exported workspace type backward-compatible, add the changelog entry, then validate the focused memory-core and SDK tests before merge.

## Reproduction Assessment

Yes. A source-level reproduction is to seed a shared workspace recall store with another agent's `memory/.dreams/session-corpus/<agent>/<day>.txt` entry and exercise `openclaw memory promote --agent <id> --apply` or REM preview, whose call sites are not updated by the provided PR diff.

## Solution Assessment

No. The PR direction is right for managed cron/heartbeat dreaming, but the best fix must cover manual promote/promote-explain/REM ranking and preserve public SDK compatibility before merge.
