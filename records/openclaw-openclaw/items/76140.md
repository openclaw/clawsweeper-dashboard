## Summary

Keep open. The PR improves shared-workspace ingest/read isolation, but the durable promotion path still ranks and applies workspace-wide recall candidates, so another agent's corpus entry can still be promoted into `MEMORY.md`; the user-facing memory fix also still needs a changelog entry.

## What This Changes

This PR adds shared-workspace metadata, threads agent/session identity through memory-core dreaming, writes agent-scoped corpus/provenance files, and adds isolation plus audit coverage.

## Best Possible Solution

Extend the shared-workspace contract through rank and promote so only the validated current agent's scoped corpus can become durable memory, handle legacy pooled entries deliberately, add regression coverage, and document the user-facing fix in `CHANGELOG.md`.

## Reproduction Assessment

Yes. Seed a shared workspace short-term recall store with another agent's agent-scoped or legacy pooled corpus entry, trigger deep dreaming, and PR head still lets that entry be ranked for durable promotion because the rank/apply path has no active-agent filter.

## Solution Assessment

No. The direction is appropriate, but it is incomplete until the final durable promotion candidate set is scoped to the validated agent instead of only adding provenance metadata.
