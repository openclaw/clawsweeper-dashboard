## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Do not synthesize an agent for shared ranking:** `extensions/memory-core/src/dreaming.ts:619`
  - body: When no real `currentAgentId` is available, this fallback passes the first configured agent into `rankShortTermPromotionCandidates`. In a shared workspace that bypasses the fail-closed contract and can still promote that agent's existing corpus entries, so pass only the actual current identity and let shared ranking return no candidates without it.
  - confidence: 0.92
- **[P1] Require shared-workspace scope for promotion callers:** `extensions/memory-core/src/short-term-promotion.ts:1247-1253`
  - body: The new rank filter only runs when callers opt in with `isShared`, but `memory promote`, `promote-explain`, and REM preview still call this API without that metadata. Those user-facing paths can still rank shared-workspace recall entries broadly, so derive and pass the selected agent plus shared-workspace status at the callers or make the API require it.
  - confidence: 0.89
- **[P2] Keep MemoryDreamingWorkspace source-compatible:** `src/memory-host-sdk/dreaming.ts:149`
  - body: `MemoryDreamingWorkspace` is exported through the plugin SDK subpath, and making `shared` required breaks external code that constructs the previous `{ workspaceDir, agentIds }` shape. Make the field optional or keep a narrower internal type while still returning `shared` from the resolver.
  - confidence: 0.84
