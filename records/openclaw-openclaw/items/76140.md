## Summary

Keep open. The latest PR head addresses the earlier ingest/read-path blockers, but durable promotion still ranks and applies the whole shared recall store, so the main cross-agent contamination risk is not fully closed.

## What This Changes

The PR adds shared-workspace metadata, threads agent/session identity through memory-core dreaming, writes agent-scoped corpus/provenance files, and adds isolation tests plus an audit report.

## Best Possible Solution

Implement one validated shared-workspace contract across ingest, read, rank, and promote: only the active configured agent's scoped corpus can become durable memory, legacy pooled entries are handled deliberately, and coverage plus changelog document the behavior.

## Reproduction Assessment

Yes. A source-level reproduction is available by using a shared workspace whose short-term recall store contains another agent's agent-scoped or legacy pooled corpus entry, then triggering dreaming and observing that PR head still ranks that entry for promotion.

## Solution Assessment

No. The PR direction is appropriate, but it is not yet the best fix because it isolates ingest/read phases while leaving the final durable promotion candidate set unscoped.
