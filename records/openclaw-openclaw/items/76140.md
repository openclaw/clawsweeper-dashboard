## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Do not synthesize an agent for shared ranking:** `extensions/memory-core/src/dreaming.ts:619-620`
  - body: When params.currentAgentId is absent, this fallback passes the first configured agent into rankShortTermPromotionCandidates. In a shared workspace that defeats the fail-closed rule: the sweep can still rank and promote that first agent's corpus entries without a real hook identity. Pass the actual currentAgentId through and let shared ranking return no candidates, or skip the shared workspace before ranking.
  - confidence: 0.9
- **[P1] Scope manual promotion callers too:** `extensions/memory-core/src/short-term-promotion.ts:1247-1253`
  - body: The new rank filter only runs when callers pass isShared/currentAgentId, but openclaw memory promote, promote-explain, and rem-harness still call this function without those options. After this PR teaches the ranker to accept agent-scoped session-corpus paths, those user-facing paths can rank another agent's shared-workspace entries and promote them with --apply. Derive the shared-workspace metadata at those callers or make the boundary mandatory in the rank API.
  - confidence: 0.88
- **[P2] Keep MemoryDreamingWorkspace backward compatible:** `src/memory-host-sdk/dreaming.ts:147-149`
  - body: MemoryDreamingWorkspace is exported through the plugin SDK, so making shared a required field breaks plugin code that constructs the existing { workspaceDir, agentIds } shape. Make the new field optional or use a narrower internal type while still returning the value from resolveMemoryDreamingWorkspaces.
  - confidence: 0.84
- **[P3] Add the memory isolation fix to the changelog:** `extensions/memory-core/src/dreaming.ts:523-530`
  - body: This is a user-facing fix(memory) behavior change, but the PR does not change CHANGELOG.md. Add a single-line entry under the active Unreleased Fixes section before merge.
  - confidence: 0.82
