  - confidence: 0.94
- **[P1] Make per-agent corpus paths rankable:** `extensions/memory-core/src/dreaming-phases.ts:754-760`
  - body: Shared workspaces now write results to `memory/.dreams/session-corpus/<agent>/<day>.txt`, but the PR-head short-term path recognizer still only accepts `memory/.dreams/session-corpus/<day>.txt`. Those new results are passed to `recordShortTermRecalls`, so they can be skipped before ranking and promotion ever see the isolated corpus.
  - confidence: 0.92
- **[P1] Apply agent scope inside promotion ranking:** `extensions/memory-core/src/dreaming.ts:616-618`
  - body: This call passes `currentAgentId` and `isShared`, but `RankShortTermPromotionOptions` does not define those fields and the ranker still iterates the workspace-level store without filtering by agent. That leaves promotion able to rank cross-agent or legacy shared entries and should also fail type checking at the call sites.
  - confidence: 0.93
- **[P1] Fail closed for shared recall without identity:** `extensions/memory-core/src/dreaming-phases.ts:672-674`
  - body: When `isShared` is true but `currentAgentId` is absent, this helper returns every recall entry instead of none. An identity-less shared-workspace run can therefore feed existing shared or contaminated recall entries into light/REM dreaming even though ingestion tries to fail closed.
  - confidence: 0.9
- **[P2] Use canonical plugin state for provenance:** `extensions/memory-core/src/short-term-promotion.ts:1602`
  - body: The new provenance sidecar is written as ad hoc JSON under `memory/.dreams`, while current main has moved memory-core dreaming state to the SQLite-backed plugin-state helpers. Either drop this sidecar or move it through the canonical guarded state/migration path before adding new persistent state.
