## Summary

Keep open. The PR improves shared-workspace ingestion and light/REM reads, but the durable promotion path still consumes unscoped recall-store candidates and the user-facing memory fix lacks a changelog entry.

## What This Changes

The PR adds shared-workspace metadata, threads agent/session identity through memory-core dreaming, writes agent-scoped session corpus/provenance data, and adds isolation plus audit tests.

## Best Possible Solution

Complete one isolation contract across ingest, read, rank, and promote so only the validated active agent's scoped corpus can become durable memory, with legacy pooled entries handled deliberately and documented in the changelog.

## Reproduction Assessment

Yes. Seed a shared workspace short-term recall store with an entry from another agent-scoped corpus path or the legacy pooled session-corpus path, make it meet promotion thresholds, then trigger dreaming for a different agent; the PR head can still rank and apply that entry into MEMORY.md.

## Solution Assessment

No. The direction is appropriate, but it is not the best fix until durable promotion candidates are scoped before applyShortTermPromotions and the user-facing behavior change is recorded in CHANGELOG.md.
