Full review comments:

- **[P1] Restore the resolver options contract:** `src/memory-host-sdk/dreaming.ts:608`
  - body: `resolveMemoryDreamingWorkspaces` is exported through the plugin-facing memory-core host status surface, and current runtime/doctor callers still pass `primaryWorkspaceDir` or `env`. Removing that parameter makes those calls fail or silently drop fallback workspace behavior, so keep a compatible options object while adding the shared-workspace flag.
  - confidence: 0.94
- **[P1] Make per-agent corpus paths rankable:** `extensions/memory-core/src/dreaming-phases.ts:754-760`
  - body: Shared workspaces now write corpus entries to `memory/.dreams/session-corpus/<agent>/<day>.txt`, but the short-term promotion recognizer only accepts `memory/.dreams/session-corpus/<day>.txt`. Those new results are recorded through `recordShortTermRecalls`, so they can be skipped before ranking and promotion see the isolated corpus.
  - confidence: 0.92
- **[P1] Wire agent scope into the promotion ranker:** `extensions/memory-core/src/dreaming.ts:616-617`
  - body: This call passes `currentAgentId` and `isShared`, but `RankShortTermPromotionOptions` does not define those fields and the ranker still iterates the whole workspace store without filtering by agent. That leaves promotion able to rank cross-agent entries and should also fail type checking at this object literal.
  - confidence: 0.93
- **[P1] Fail closed for shared recall without identity:** `extensions/memory-core/src/dreaming-phases.ts:672-674`
  - body: When `isShared` is true but `currentAgentId` is absent, this helper returns every recall entry instead of none. An identity-less shared-workspace run can therefore feed existing shared or contaminated recall entries into light/REM dreaming even though ingestion tries to fail closed.
  - confidence: 0.9
- **[P2] Move provenance to canonical plugin state:** `extensions/memory-core/src/short-term-promotion.ts:1602`
  - body: The new provenance store is written as ad hoc JSON under `memory/.dreams`, while current OpenClaw state policy and memory-core dreaming state use SQLite-backed plugin state. Either drop the sidecar or move it through the canonical guarded state and migration path before adding new persisted state.
  - confidence: 0.86
