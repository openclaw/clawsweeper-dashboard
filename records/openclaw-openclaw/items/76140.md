## Summary

Keep open. The PR is pointed at a real memory isolation bug and the latest head appears to address the earlier managed-dreaming promotion gap, but it still leaves user-facing manual promotion paths unscoped, makes a public SDK type required in a breaking way, and is missing the required changelog entry.

## What This Changes

The PR adds shared-workspace metadata, threads agent/session identity through memory-core dreaming, scopes shared session corpus/provenance paths, and adds isolation tests plus an audit note.

## Best Possible Solution

Use one agent-boundary contract for managed dreaming, manual promote/explain, REM preview, and public SDK compatibility, then land it with focused isolation coverage and a changelog entry.

## Reproduction Assessment

Yes. A source-level reproduction is available by seeding a shared workspace recall store with another agent's `memory/.dreams/session-corpus/<agent>/...` entry and exercising the manual `memory promote --agent <id> --apply` path, whose current call site does not pass the new isolation flags.

## Solution Assessment

No. The PR direction is right for cron/heartbeat dreaming, but the best fix must scope every durable promotion reader/writer and keep the exported workspace type backward-compatible.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Scope manual promotion callers too:** `extensions/memory-core/src/short-term-promotion.ts:1240-1248`
  - body: The new rank filter only runs when callers pass `isShared` and `currentAgentId`, but `openclaw memory promote --agent`, `promote-explain`, and the REM harness still call `rankShortTermPromotionCandidates` without those flags. In a shared workspace, those user-facing paths can still rank another agent's `session-corpus/<agent>/...` entry and `--apply` can promote it to `MEMORY.md`; derive the shared-workspace metadata at these call sites or make the boundary mandatory in the rank API.
  - confidence: 0.88
- **[P2] Keep MemoryDreamingWorkspace backward compatible:** `src/memory-host-sdk/dreaming.ts:147`
  - body: `MemoryDreamingWorkspace` is exported through `openclaw/plugin-sdk/memory-core-host-status`, so making `shared` a required field breaks plugin code that constructs the existing `{ workspaceDir, agentIds }` shape. Make the new field optional or introduce a versioned/narrower internal type while still returning the value from `resolveMemoryDreamingWorkspaces`.
  - confidence: 0.83
- **[P3] Add the memory isolation fix to the changelog:** `extensions/memory-core/src/dreaming.ts:904-912`
  - body: This is a user-facing `fix(memory)` behavior change, but the PR does not touch `CHANGELOG.md`. Add a single-line entry under the active Unreleased Fixes section before merge.
