## Summary

Keep open. The latest PR head improves ingest/read isolation, but durable promotion still ranks and applies the whole shared workspace recall store, so cross-agent memory can still enter `MEMORY.md`; the PR is also draft and missing the required user-facing changelog entry.

## What This Changes

The PR adds shared-workspace metadata, threads agent/session identity through memory-core dreaming, writes agent-scoped session corpus/provenance, and adds isolation tests plus an audit report.

## Best Possible Solution

Enforce one shared-workspace contract across ingest, read, rank, and promote: fail closed without a validated active agent, only the active agent's scoped corpus can become durable memory, legacy pooled entries are handled deliberately, and the changelog records the user-facing fix.

## Reproduction Assessment

Yes. A source-level reproduction is to seed a shared workspace recall store with another agent's `session-corpus/<agent>/...` entry, trigger dreaming as the current agent, and observe PR head still ranks that entry for durable promotion.

## Solution Assessment

No. The PR direction is appropriate, but the current patch is not the best fix until the durable promotion candidate set is scoped before apply and the changelog is added.
