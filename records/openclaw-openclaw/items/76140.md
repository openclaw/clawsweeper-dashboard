## Summary

Keep open. The PR moves the memory dreaming fix in the right direction, but the latest head still has blocking shared-workspace isolation gaps, a public SDK compatibility break, and the required changelog entry is missing.

## What This Changes

The PR adds shared-workspace metadata, threads agent/session identity through memory-core dreaming, scopes shared session corpus and provenance paths, filters shared promotion ranking, and adds isolation tests plus an audit note.

## Best Possible Solution

Use one validated agent-boundary contract for managed dreaming, manual promote/promote-explain, REM preview, and public SDK compatibility, then land it with focused isolation coverage and a changelog entry.

## Reproduction Assessment

Yes. A source-level reproduction is available by seeding a shared workspace recall store with another agent's session-corpus entry and running a durable promotion path that lacks isShared/currentAgentId, or by calling managed promotion without currentAgentId and observing the PR head fallback to the first configured agent.

## Solution Assessment

No. The direction is appropriate, but the current patch makes isolation opt-in at the rank API, leaves manual durable promotion unscoped, bypasses fail-closed behavior in the managed promotion wrapper, and tightens a public SDK type.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Require identity before shared managed promotion:** `extensions/memory-core/src/dreaming.ts:619`
  - body: The managed promotion call falls back to the first configured workspace agent when `params.currentAgentId` is absent. In a shared workspace, that prevents `rankShortTermPromotionCandidates` from failing closed and can rank/promote one agent's corpus during an unidentified run; skip shared promotion or pass no identity unless the active agent is validated.
  - confidence: 0.87
- **[P1] Scope manual promotion callers too:** `extensions/memory-core/src/short-term-promotion.ts:1247-1253`
  - body: The new rank filter only runs when callers pass `isShared` and `currentAgentId`, but `memory promote`, `promote-explain`, and the REM harness still call this API without those flags. In shared workspaces, `memory promote --apply` can still promote another agent's `session-corpus/<agent>/...` entry; derive the same workspace identity metadata at those call sites or make the boundary mandatory.
  - confidence: 0.9
- **[P2] Keep MemoryDreamingWorkspace backward compatible:** `src/memory-host-sdk/dreaming.ts:147-149`
  - body: `MemoryDreamingWorkspace` is exported through `openclaw/plugin-sdk/memory-core-host-status`, so making `shared` required breaks plugin code that constructs the existing `{ workspaceDir, agentIds }` shape. Make the new field optional or introduce an internal result type while still returning `shared` from the resolver.
  - confidence: 0.84
- **[P3] Add a changelog entry for the memory fix:** `extensions/memory-core/src/dreaming.ts:967-968`
  - body: This is a user-facing `fix(memory)` behavior change, but the PR does not update `CHANGELOG.md`. Add a single-line entry under the active Unreleased Fixes section before merge.
  - confidence: 0.82
