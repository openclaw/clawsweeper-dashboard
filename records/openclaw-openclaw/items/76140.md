## Summary

Keep open. This draft PR targets the real shared-workspace dreaming contamination bug, but the current patch still has blocking isolation and test defects before it is safe to merge.

## What This Changes

The PR adds shared-workspace detection, threads currentAgentId/sessionKey through memory-core dreaming, writes per-agent session corpus files and promotion provenance, and adds isolation tests.

## Best Possible Solution

Implement an explicit agent-scoped dreaming contract end to end: validate the active agent against the matched workspace, accept and rehydrate agent-scoped corpus paths, fail closed when identity is unavailable, handle legacy pooled corpus data deliberately, and prove it with a two-agent shared-workspace regression.

## Reproduction Assessment

Yes. Current main can be reproduced at source level with two configured agents sharing one workspace: the resolver groups both agents, ingestion iterates both session directories, and dreaming records unscoped session corpus recall entries.

## Solution Assessment

No. The PR direction is plausible, but this implementation is not yet the best fix because it changes corpus layout without updating all recall contracts, leaves a shared read path fail-open, and adds tests that will not run under the repo's Vitest setup.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Accept agent-scoped corpus paths before recording recalls:** `extensions/memory-core/src/dreaming-phases.ts:753-760`
  - body: `appendSessionCorpusLines` now returns paths like `memory/.dreams/session-corpus/emmi/2026-05-02.txt`, but `recordShortTermRecalls` still keeps only paths accepted by `SHORT_TERM_SESSION_CORPUS_RE`, which matches only `session-corpus/<day>.txt`. In shared workspaces the snippets are written to disk but dropped before the short-term recall store, so light/REM/deep dreaming will not see the agent-scoped corpus.
  - confidence: 0.94
- **[P1] Fail closed shared recall reads without an agent id:** `extensions/memory-core/src/dreaming-phases.ts:672-674`
  - body: For shared workspaces, `filterRecallEntriesForAgentIsolation` returns all entries when `currentAgentId` is missing. That preserves the contamination path for hook or cron contexts that lack agent identity: transcript ingestion skips, but existing pooled recall entries can still feed dreaming.
  - confidence: 0.9
- **[P2] Validate the shared-workspace agent id before ingesting:** `extensions/memory-core/src/dreaming-phases.ts:729-730`
  - body: When a workspace is shared, the helper returns `[currentAgentId]` without checking that the ID belongs to the matched workspace. A stale or mismatched identity can cause the sweep for one workspace to ingest another agent's sessions instead of failing closed, which contradicts the PR's adversarial no-cross-contamination goal.
  - confidence: 0.84
- **[P2] Scope shared-workspace read filtering to the current workspace:** `extensions/memory-core/src/dreaming-phases.ts:1647-1651`
  - body: `workspaceIsShared` is computed with `.some(...)` across all configured workspaces, not the workspace currently being swept. If any workspace is shared, an isolated workspace with a `currentAgentId` will filter its normal short-term entries as though it were shared, dropping non-agent-scoped paths from light dreaming.
  - confidence: 0.82
- **[P2] Use Vitest-compatible module access in tests:** `extensions/memory-core/src/dreaming-phases.test.ts:2803-2804`
  - body: This test file imports from Vitest and has no Jest global, but the added tests call `jest.requireActual`. The targeted test run will throw before the isolation assertions execute; use the imported `__testing` surface or Vitest-compatible module helpers.
  - confidence: 0.93
- **[P2] Use fs.promises-compatible existence checks:** `extensions/memory-core/src/dreaming-phases.test.ts:2860-2862`
  - body: `fs` is imported from `node:fs/promises`, which does not provide `pathExists`. These assertions will fail at runtime or typecheck; use `fs.access`, `stat`, or an existing test helper.
  - confidence: 0.93
