## Summary

Keep open. The PR is pointed at a real current-main contamination path, but the latest patch still has blocking functional and security gaps in the shared-workspace isolation flow.

## What This Changes

The PR adds shared-workspace metadata, threads agent/session identity through memory-core dreaming, writes agent-scoped session corpus and provenance data, and adds isolation tests.

## Best Possible Solution

Implement a validated agent-scoped dreaming contract end to end: accept and rehydrate agent-scoped corpus paths, fail closed without a matching active agent, handle legacy pooled corpus deliberately, add two-agent integration coverage, and include a changelog note.

## Reproduction Assessment

Yes. Current main has a source-level reproduction path: configure multiple agents with the same workspace and the dreaming resolver returns all matched agent IDs into one unscoped session corpus and recall store.

## Solution Assessment

No. The PR direction is plausible, but this patch is not yet the best fix because the new corpus path is not accepted by the recall store and shared reads still lack a strict agent boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Accept agent-scoped corpus paths before recording recalls:** `extensions/memory-core/src/dreaming-phases.ts:754-761`
  - body: When `isShared` is true, this returns `memory/.dreams/session-corpus/<agent>/<day>.txt`, but `recordShortTermRecalls` still filters through `isShortTermMemoryPath`, whose session-corpus regex only accepts `<day>.txt` directly under `session-corpus`. The isolated corpus file is written but never enters the short-term recall store.
  - confidence: 0.95
- **[P1] Fail closed shared recall reads without an agent id:** `extensions/memory-core/src/dreaming-phases.ts:672-674`
  - body: In shared mode with no `currentAgentId`, this helper returns every stored recall entry. That preserves the contamination path for any read phase that reaches shared-workspace recall without an active agent identity.
  - confidence: 0.92
- **[P2] Reject agent ids outside the matched shared workspace:** `extensions/memory-core/src/dreaming-phases.ts:729-730`
  - body: For shared workspaces this trusts any non-empty `currentAgentId` and returns it without checking `match.agentIds`. A stale or mismatched identity can ingest another agent's sessions into this workspace instead of failing closed.
  - confidence: 0.86
- **[P2] Check the current workspace before applying shared filtering:** `extensions/memory-core/src/dreaming-phases.ts:1647-1651`
  - body: This computes `workspaceIsShared` with `.some(...)` across all configured workspaces rather than the workspace being swept. If any workspace is shared, isolated workspaces filter their normal recall entries as shared and drop non-agent-scoped paths.
  - confidence: 0.84
- **[P3] Add the memory isolation fix to the changelog:** `extensions/memory-core/src/dreaming.ts:927-928`
  - body: This is a user-facing `fix(memory)` behavior change, but the PR does not include a `CHANGELOG.md` entry. OpenClaw policy requires a changelog note for user-facing fixes before merge.
  - confidence: 0.82
