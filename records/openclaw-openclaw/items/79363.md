## What This Changes

The PR adds an opt-in `tools.exec.safeBuiltins` config, allowlist/runtime support, tests, docs, generated config baseline updates, and changelog coverage for auto-allowing selected POSIX shell builtins.

## Best Possible Solution

Ship the closed-set opt-in only after `safeBuiltins` is consistently threaded through gateway, node precheck/final policy, shared types, config metadata, and a redacted configured exec-flow proof for the linked issue.

## Reproduction Assessment

Yes, source-reproducible. Current main has no builtin satisfaction branch and pathless command segments cannot match normal allowlist paths, so a command like `cd /tmp && git status` cannot satisfy allowlist evaluation for the `cd` segment without a new mechanism.

## Solution Assessment

No, not yet. The closed-set opt-in is a reasonable direction, but this patch should first carry the setting through `host=node`, repair the widened satisfaction type contract, add config help/label metadata, and include configured runtime proof.
