This PR is a plausible fix path for the linked exec allowlist report, but it should not merge yet because the new approval bypass knob accepts arbitrary configured builtin names and the public docs do not match the implemented defaults.

## What This Changes

The PR adds an optional `tools.exec.safeBuiltins` config path, allowlist satisfaction branch, runtime plumbing, docs, changelog, and regression tests for POSIX shell builtins in exec allowlist mode.

## Best Possible Solution

Merge after restricting `safeBuiltins` to a closed supported set, documenting the real defaults and opt-in risks, and keeping the linked issue open until the fixed PR lands.

Yes. Current-main source shows pathless builtin segments cannot satisfy allowlist, safeBins, or skill auto-allow, and the linked report plus PR body provide concrete `cd ~/` and `cd /tmp && git status` examples.

## Solution Assessment

Not yet. The opt-in config direction is plausible, but this implementation should fail closed for supported builtin names and align the docs/changelog before merge.
