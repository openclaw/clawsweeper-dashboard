## Summary

Keep open: the PR is a plausible opt-in direction for the linked exec allowlist problem, but it is not merge-ready because the node-host precheck still misses the new policy and the public config metadata is incomplete; the supplied proof is useful terminal evidence but does not exercise a configured OpenClaw exec approval flow.

## What This Changes

The PR adds an opt-in `tools.exec.safeBuiltins` config, allowlist evaluator support, runtime threading, tests, docs, generated config baseline updates, and a changelog entry for POSIX shell builtins in exec allowlist mode.

## Best Possible Solution

Land a narrow opt-in implementation that applies builtin handling consistently across gateway, sandbox, node precheck, and node final evaluation, keeps the supported set closed and off by default, updates config metadata, and includes redacted real exec-flow proof.

## Reproduction Assessment

Yes, source-reproducible. Current main requires resolved executable paths for allowlist matching and has no builtin satisfaction branch, which matches the linked `cd ~/` and `cd /path && git status` behavior.

## Solution Assessment
