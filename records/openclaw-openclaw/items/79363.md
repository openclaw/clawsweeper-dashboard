## Summary

Keep open: the PR addresses a real source-reproducible allowlist gap, but it still misses the node-host gateway precheck and config metadata, and the supplied proof does not show the configured OpenClaw exec-flow path.

## What This Changes

The PR adds an opt-in `tools.exec.safeBuiltins` config, allowlist/runtime support, tests, docs, generated config baseline hashes, and changelog coverage for selected POSIX shell builtins in exec allowlist mode.

## Best Possible Solution

Land the opt-in only after safeBuiltins is carried through gateway and node precheck/final policy paths, config help/label metadata and baselines are aligned, and redacted configured exec-flow proof is supplied.

## Reproduction Assessment

Yes, source-reproducible. Current main has no builtin satisfaction branch and pathless command segments cannot match normal allowlist paths, so `cd /tmp && git status` cannot satisfy the allowlist for the `cd` segment.

## Solution Assessment

No, not yet. The closed-set opt-in design is plausible, but the implementation should carry the same resolved safeBuiltins set through the node precheck and add public config metadata before merge.
