## What This Changes

The PR adds `tools.exec.safeBuiltins` config/schema/docs/tests and allowlist/runtime plumbing so opted-in POSIX shell builtins can satisfy exec allowlist evaluation.

## Best Possible Solution

Merge only after `safeBuiltins` is restricted to a closed supported set, propagated through gateway and node approval analyzers, docs/changelog state the actual defaults, and maintainers accept the policy knob.

## Reproduction Assessment

Yes. Current-main source shows pathless builtin segments cannot satisfy allowlist, safeBins, or skill auto-allow, matching the linked `cd ~/` and `cd /tmp && git status` examples plus the PR body's before/after output.

## Solution Assessment

No. The opt-in config direction is plausible, but this implementation should fail closed for supported builtin names, cover host=node approval prechecks, and align public docs before merge.
