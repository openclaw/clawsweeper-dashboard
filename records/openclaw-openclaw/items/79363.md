## What This Changes

The PR adds opt-in `tools.exec.safeBuiltins` config support, threads it through gateway/node exec allowlist evaluation, documents it, updates generated baselines, and adds safe-builtin tests.

## Best Possible Solution

Merge only after the safe-builtin evaluator fails closed for unsupported/code-evaluating builtins, the docs match the actual default/off-by-default behavior, and maintainers accept the new exec approval policy knob.

## Reproduction Assessment

Yes. Current-main source shows pathless builtin segments cannot satisfy allowlist, safeBins, or skills, and the linked issue plus PR body provide concrete `cd ~/` and `cd /tmp && git status` examples.

## Solution Assessment

No. The opt-in config direction is plausible, but this diff should first enforce a closed safe-builtin set or hard-deny code-evaluating builtins and fix the docs/default mismatch.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Reject unsafe builtin names before trusting safeBuiltins:** `src/infra/exec-safe-builtins.ts:63`
  - body: `isSafeBuiltinSegment` trusts any configured builtin token once it has no resolved filesystem path. If `eval`, `source`, or `.` is present in `tools.exec.safeBuiltins`, that pathless code-evaluating builtin is marked `safeBuiltins` and bypasses allowlist approval, contradicting the PR's security section. Please filter to a closed supported set or hard-deny these names before returning true.
  - confidence: 0.88
- **[P3] Align the documented default builtin set:** `docs/tools/exec-approvals-advanced.md:153-155`
  - body: The docs call `cd`, `export`, and `unset` part of the conservative default set, but the code's `DEFAULT_SAFE_BUILTINS` intentionally excludes those state-mutating builtins and the option is off when omitted. That can steer users toward a riskier list while thinking it is the default; please make the docs/example match the stateless default or clearly label these as explicit opt-ins.
  - confidence: 0.82
