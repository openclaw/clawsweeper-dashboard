## What This Changes

The issue reports stale foreground WhatsApp auto-reply finalization after a newer inbound message arrives in the same chat/session.

## Best Possible Solution

Land #76970 or an equivalent shared auto-reply freshness guard that suppresses superseded foreground finals while preserving different-target and background-completion behavior.

## Reproduction Assessment

No live high-confidence reproduction was established here. Source inspection shows a credible current-main path: a newer inbound message can steer an active run while the older run can still route its final payload without a run/message freshness check.
