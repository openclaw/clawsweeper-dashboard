## Summary

Keep open: current main still lacks a foreground final-reply freshness guard, and open PR #76970 is already paired with this issue using closing syntax.

## What This Changes

The issue reports stale WhatsApp foreground final replies from an older tool-backed run being delivered after a newer inbound message in the same chat.

## Best Possible Solution

Review and land #76970, or an equivalent shared auto-reply freshness guard with regression coverage, then close this issue from the merged fix.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows a credible current-main path: newer inbound messages can steer an active run while the older run can still enqueue its final reply without a freshness check.

## Solution Assessment

Yes, directionally: the narrowest maintainable solution is a shared foreground final-delivery freshness guard in auto-reply dispatch, not a WhatsApp-specific workaround.
