## Summary

Keep open: current main still has the source-level stale foreground finalization path, and #76970 is the open closing PR for the focused fix.

## What This Changes

Prevent older tool-backed WhatsApp foreground final replies from being delivered after a newer inbound message starts in the same chat/session.

## Best Possible Solution

Land a shared auto-reply freshness guard with regression coverage that suppresses superseded foreground finals while preserving unrelated targets and background completion semantics.

## Reproduction Assessment

No live high-confidence reproduction was established here. Source inspection shows a credible current-main path: a newer inbound message can steer the active run while the older run can still enqueue or route an unguarded final reply.

## Solution Assessment

Yes, directionally: a shared foreground-final freshness guard is the narrowest maintainable fix, and #76970 is already the open PR for that path. A WhatsApp-only workaround would leave the same shared auto-reply race available to other messaging channels.
