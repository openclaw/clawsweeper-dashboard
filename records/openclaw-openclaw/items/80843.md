## Summary

Keep open. Current main still lacks the requested explicit `web_search` fallback chain, recoverable-error contract, logs, and aggregated failure reporting; an open implementation PR exists, but it has not merged and needs product/API review and rework before it can close this issue.

## What This Changes

Request an ordered `web_search` provider failover chain for explicit and auto-selected providers, with recoverable-error retry, DuckDuckGo last, fallback logging, and aggregated final errors.

## Best Possible Solution

Keep this as the canonical design issue, settle a backward-compatible fallback contract, then rework or replace the open implementation PR so provider loading, SecretRefs, validation, cancellation, logging, and aggregated errors are covered together.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection gives high confidence that current main lacks the requested fallback config fields and deliberately keeps explicit provider failures fail-fast.

## Solution Assessment

No current implementation fully solves it. The safest path is a maintainer-approved public config and error-classification contract, followed by a focused runtime/config/docs/test implementation or a reworked version of the open PR.
