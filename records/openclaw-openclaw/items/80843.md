## Summary

Keep open. Current main still lacks the requested explicit `web_search` fallback chain and recoverable-error contract; the related implementation PR is open but conflicting and not yet ready to land.

## What This Changes

The issue requests an opt-in `web_search` fallback chain across configured providers for recoverable quota and network failures, with DuckDuckGo as final fallback and aggregated diagnostics.

## Best Possible Solution

Define a backward-compatible fallback contract, validate fallback provider ids through plugin metadata, normalize recoverable provider failures, document query fan-out and diagnostics, then land a repaired version of the existing implementation PR or a narrow replacement.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that the requested config fields are absent and explicit provider failures remain fail-fast on current main.

## Solution Assessment

No current solution fully solves the request. The existing auto-selected fallback path is useful, and https://github.com/openclaw/openclaw/pull/63571 is the relevant implementation candidate, but the public config shape, error taxonomy, validation, docs, and proof still need to be resolved before merge.
