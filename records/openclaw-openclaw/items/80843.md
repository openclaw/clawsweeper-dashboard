## Summary

Keep open. Current main has limited auto-mode retry behavior, but it still lacks the requested public fallback-chain config, explicit-provider failover, recoverable-error contract, fallback-attempt logging, and aggregated all-provider failure reporting; the older fallback PRs are closed and the merged related work only added Tavily as another provider.

## What This Changes

Request an ordered `web_search` provider failover chain for explicit and auto-selected providers, with recoverable-error gating, DuckDuckGo as final fallback, logging, and aggregated diagnostics.

## Best Possible Solution

Define a backward-compatible web-search fallback contract in config/schema/docs, normalize provider failure reasons, and implement ordered runtime retries with focused coverage for explicit and auto modes.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection confirms the requested config keys are absent and explicit provider failures are intentionally not retried today.

## Solution Assessment

No current solution fully solves the request. The existing auto-mode fallback and merged Tavily support are useful pieces, but the maintainable path needs an agreed config shape and provider error contract before implementation.
