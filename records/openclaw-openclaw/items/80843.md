## Summary

Keep open. Current main still has only single-provider explicit selection plus auto-detect fallback behavior, while the requested explicit `web_search` fallback chain, retryable-error taxonomy, logging, and aggregate failure reporting remain unshipped; an open implementation PR exists but is conflicting and previously reviewed as needing rework and real behavior proof.

## What This Changes

Request to add a documented `web_search` provider failover chain for recoverable quota, rate-limit, timeout, network, and provider-side failures, with DuckDuckGo as the final fallback.

## Best Possible Solution

Review and rework https://github.com/openclaw/openclaw/pull/63571 or a successor into a backward-compatible fallback contract that validates provider ids, preserves plugin-owner loading and SecretRef behavior, documents query fan-out, and proves the real fallback flow.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main lacks `tools.web.search.fallbacks` and keeps explicit provider failures fail-fast.

## Solution Assessment

No current implementation on main solves the request. The active open PR is the closest path, but it needs maintainer API review, branch rework, and real behavior proof before it can be the best fix.
