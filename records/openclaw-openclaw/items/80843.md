## Summary

Keep open. Current main still lacks the requested explicit `web_search` fallback chain: the runtime only retries non-explicit auto-selection failures, the config schema/docs expose no fallback fields, and the related historical request and implementation PR are closed/unmerged.

## What This Changes

Request a `web_search` provider failover chain with explicit fallbacks, auto configured-provider fallback, recoverable-error gating, logging, and aggregated failure diagnostics.

## Best Possible Solution

Define a backward-compatible fallback config and normalized provider error contract, keep single-provider behavior as the default, then implement ordered retries with docs and focused regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection on current main confirms the requested fallback config is absent and explicit provider runtime failures still do not retry.

## Solution Assessment

No current solution fully solves the request. The existing auto-selected fallback path is useful, but it is not the requested explicit ordered failover chain with recoverable-error gating and aggregated diagnostics.
