## Summary

Keep open. Current main still implements only the existing auto-selected-provider fallback path and keeps explicit `tools.web.search.provider` failures fail-fast; the requested fallback config, recoverable-error contract, logging, and aggregated diagnostics remain unimplemented, with https://github.com/openclaw/openclaw/pull/63571 as an open implementation candidate rather than a merged fix.

## What This Changes

Request a `web_search` provider fallback chain that retries configured providers on quota/rate-limit/network failures and uses DuckDuckGo only as the final fallback.

## Best Possible Solution

Define a backward-compatible fallback config and provider error contract, then land or replace https://github.com/openclaw/openclaw/pull/63571 with validation, docs, logging, aggregated errors, and focused runtime coverage.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection confirms the requested config fields are absent and explicit provider failures remain fail-fast on current main.

## Solution Assessment

Unclear until maintainers settle the config and error-contract details. The explicit fallback direction is plausible, but the open implementation PR still needs review against current provider loading, SecretRef, cancellation, and documentation contracts.
