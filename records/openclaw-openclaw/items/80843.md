## Summary

Keep open. Current main has limited auto fallback when no provider is explicitly selected, but it still lacks the requested ordered fallback config, recoverable-error contract, explicit-provider failover, fallback-attempt logging, and aggregated all-provider diagnostics.

## What This Changes

The issue requests an ordered web_search provider fallback chain for recoverable quota, rate-limit, and network failures, with DuckDuckGo as the final fallback and aggregated diagnostics.

## Best Possible Solution

Define a backward-compatible web_search fallback contract, normalize provider failure reasons, keep current single-provider behavior by default, and document explicit plus auto fallback modes before adding runtime retries.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report. Source inspection confirms the requested config fields are absent and explicit provider failures are intentionally not retried today.

## Solution Assessment

Unclear until maintainers choose the public config shape and provider error contract. The existing auto-selected fallback path is useful, but it is not the requested explicit ordered fallback chain with recoverable-error gating and aggregated diagnostics.
