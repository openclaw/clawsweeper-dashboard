## Summary

Keep open. Current main and the latest shipped web-search docs still support only single explicit provider selection plus limited auto-selected fallback behavior; the requested explicit fallback chain, recoverable-error contract, logging, and aggregated failure reporting remain unimplemented.

## What This Changes

Request adds a `web_search` provider fallback chain with explicit `tools.web.search.fallbacks`, auto configured-provider fallback, retryable-error handling, fallback logs, and aggregated final errors.

## Best Possible Solution

Define a documented, backward-compatible web-search fallback contract, including provider loading/validation, SecretRef activation, retryable error classification, logs, aggregated errors, and focused runtime/config/docs coverage.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high confidence that current main lacks the requested fallback config fields and keeps explicit provider failures fail-fast.

## Solution Assessment

No current implementation fully solves this yet. The direction is plausible, but https://github.com/openclaw/openclaw/pull/63571 needs maintainer API/product review and rework before it can become the fix.
