## Summary

Keep open. Current main and the latest shipped release still support single explicit web-search provider selection plus limited auto-detect fallback; the requested explicit/auto fallback chain, recoverable-error taxonomy, logging, and aggregated failure reporting are not implemented. The open related PR covers part of the idea but has review blockers and does not supersede this issue.

## What This Changes

Request an ordered `web_search` provider fallback chain for recoverable quota, rate-limit, and network failures.

## Best Possible Solution

Define a documented, backward-compatible fallback contract first, then rework or land https://github.com/openclaw/openclaw/pull/63571 with provider validation, error classification, SecretRef semantics, docs, and focused tests aligned.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main lacks the requested fallback fields and intentionally keeps explicit provider failures fail-fast.

## Solution Assessment

No current implementation fully solves the issue. The related PR is a useful starting point, but the maintainable solution needs an owner-approved public config and provider error contract before merge.
