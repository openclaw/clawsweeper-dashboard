## Summary

Keep open. Current main has some auto-selected provider fallback, but it does not implement the requested explicit ordered fallback chain, recoverable-error policy, fallback-attempt diagnostics, or aggregated all-provider failure reporting.

## What This Changes

Request a backward-compatible `web_search` provider fallback chain for quota, rate-limit, and transient network failures, with explicit and auto fallback modes.

## Best Possible Solution

Define a stable fallback config and normalized recoverable-error contract, preserve existing single-provider behavior by default, then implement ordered retries with focused runtime/config/docs coverage.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection confirms the proposed config fields are absent and explicit provider failures are intentionally not retried today.

## Solution Assessment

Unclear until maintainers choose the public config shape and provider error contract. The narrow maintainable path is to design that contract first, then implement retries and diagnostics against it.
