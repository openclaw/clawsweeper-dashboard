## Summary

Keep this open: current main has auto-detect fallback for unpinned web_search providers, but the requested explicit fallback chain, retryable-error policy, DuckDuckGo-last policy, and aggregated failure reporting are not implemented. There is an active open implementation candidate at https://github.com/openclaw/openclaw/pull/63571, so the issue should track review/product alignment rather than start a separate repair lane.

## What This Changes

The issue requests ordered web_search provider failover for recoverable quota, rate-limit, and network failures.

## Best Possible Solution

Review and align the explicit fallback-chain design in the active PR or a narrow replacement, preserving current single-provider behavior when no fallback chain is configured.

## Reproduction Assessment

Not applicable: this is a feature/config request, not a broken existing behavior report. Source and tests show current main intentionally supports only auto-candidate fallback when no provider is explicitly selected, while explicit provider selection remains fail-fast.

## Solution Assessment

Unclear until maintainers choose the public config and retry policy. The requested direction is plausible, but the safest path is to review the active PR against current runtime semantics instead of closing this as implemented or spawning another repair branch.
