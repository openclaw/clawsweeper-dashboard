## Summary

Keep open. Current main still lacks the requested explicit web_search fallback chain, auto-mode fallback over configured providers, recoverable-error taxonomy, fallback-attempt logging, and aggregated all-provider error reporting; an open implementation PR overlaps with the request but still needs review/proof rather than issue closure.

## What This Changes

Add an ordered web_search provider fallback chain for quota, rate-limit, and transient network/provider failures, with DuckDuckGo as the final fallback and aggregated diagnostics.

## Best Possible Solution

Define a backward-compatible fallback config and normalized retryable-error contract, document query fan-out and diagnostics, then land a reviewed implementation that preserves current fail-fast defaults unless fallback is explicitly enabled.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection confirms current main lacks the requested fallback config fields and explicit provider failures remain fail-fast.

## Solution Assessment

No current solution fully solves the issue. The open implementation direction is plausible, but it needs maintainer agreement on config shape/error taxonomy and must be repaired/proven against current provider-load, SecretRef, and abort-signal contracts.
