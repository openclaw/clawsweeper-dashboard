## Summary

Keep open. Current main only has implicit fallback for auto-selected providers; it still lacks the requested explicit ordered fallback config, recoverable-error taxonomy, fallback-attempt logging, and aggregated all-provider failure reporting, and the prior related issue https://github.com/openclaw/openclaw/issues/35647 was closed as stale rather than replaced by an open canonical tracker.

## What This Changes

The issue requests a configurable web_search provider fallback chain for quota, rate-limit, and transient network failures, with DuckDuckGo as the final fallback.

## Best Possible Solution

Define a backward-compatible web_search fallback contract, normalize retriable provider failures, preserve existing single-provider behavior by default, and document/test explicit and auto fallback modes.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection confirms the requested config fields are absent and explicit provider failures are intentionally not retried today.

## Solution Assessment

No current implementation fully solves the request. The existing auto-selected fallback path is useful, but an explicit ordered fallback chain with recoverable-error gating and aggregated diagnostics needs a new maintained contract.
