## Summary

Keep open. Current main has auto fallback when no provider is explicitly selected, but it does not implement the requested explicit fallback chain, recoverable-error taxonomy, logging, or aggregated all-provider failure reporting; the prior matching issue was closed as stale and the related implementation PRs are closed/unmerged.

## What This Changes

Add an ordered `web_search` provider failover chain for quota, rate-limit, and network failures, with DuckDuckGo as the final fallback and aggregated failure reporting.

## Best Possible Solution

Define a backward-compatible fallback contract, normalize provider failure reasons, preserve single-provider behavior by default, and document the explicit and auto fallback modes before implementing runtime retries.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection confirms the requested config fields are absent and explicit provider failures are intentionally not retried today.

## Solution Assessment

No current solution fully solves the request. The existing auto-selected fallback path is useful but not the requested explicit ordered fallback chain with recoverable-error gating and aggregated diagnostics.
