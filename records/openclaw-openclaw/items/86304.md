## What This Changes

The issue reports Dashboard history session mapping corruption after upgrade, where a Dashboard session key in `sessions.json` points to the wrong historical transcript id while the original session file remains on disk.

## Best Possible Solution

Add a maintainer-approved diagnostic or repair path that can validate a corrupted Dashboard key against redacted store/transcript fixtures before changing mappings, and add regression coverage for the chosen recovery rule.

## Reproduction Assessment

No high-confidence end-to-end upgrade reproduction is available. Source inspection does show that if `sessions.json` maps a Dashboard key to the wrong `sessionId`, current `sessions.list` and `chat.history` will use that wrong mapping.

## Solution Assessment
