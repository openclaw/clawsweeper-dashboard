## Summary

Keep open: current main still trusts the stored `SessionEntry.sessionId` for Dashboard session rows, previews, and `chat.history`, and neither current main nor the latest release adds a safe repair for corrupted Dashboard key mappings.

## What This Changes

The issue reports that after upgrade, Dashboard history entries can point to the wrong preserved session trajectory, so clicking an old item loads mismatched history.

## Best Possible Solution

Add a maintainer-approved diagnostic or doctor repair that validates corrupted Dashboard mappings against store, transcript header, and trajectory evidence, and refuses ambiguous relinks.

## Reproduction Assessment

No live end-to-end upgrade fixture is available, but the source path is reproducible from code: if `sessions.json` maps the Dashboard key to the wrong `sessionId`, current main lists and loads history through that stored entry.

## Solution Assessment

Unclear. Repairing the stored mapping may be right, but the durable rule for choosing the correct preserved orphan transcript or trajectory file needs maintainer review before rewriting user history.
