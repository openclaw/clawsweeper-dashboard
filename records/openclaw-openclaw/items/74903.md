## Summary

Keep open. Current main still matches the reported Feishu topic-session routing bug, and open closing PR #75364 plus overlapping PR #75159 should be reviewed or consolidated before this issue is closed.

## What This Changes

Fix Feishu group_topic/group_topic_sender sessions so message(action=send) replies stay in the originating topic thread instead of posting to the group root.

## Best Possible Solution

Merge a Feishu-owned runtime action-handler fix such as #75364, then close this issue after focused channel-action regression coverage and a changelog entry land.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: the group prompt directs visible output to `message(action=send)`, while current Feishu `send` leaves `replyToMessageId` undefined and `replyInThread` false.

## Solution Assessment

Yes. Handling topic-scoped `send` in Feishu action dispatch is narrower and more reliable than only changing prompt wording because it preserves topic context even when the model follows the existing `action=send` instruction.
