## Summary

Keep open. Current main still matches the reported Feishu topic-thread bug, and an open closing PR plus an overlapping prompt-only PR should be reviewed or consolidated before this issue closes.

## What This Changes

The issue asks for Feishu `group_topic` and `group_topic_sender` message-tool sends to post back into the triggering topic thread instead of the group root.

## Best Possible Solution

Land or consolidate the Feishu-owned fix so topic-scoped `send` actions inherit the inbound topic reply target while preserving plain group behavior.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: the group prompt sends the model to `message(action=send)`, while current Feishu `send` leaves `replyToMessageId` undefined and `replyInThread` false.

## Solution Assessment

Yes, the runtime action-handler direction in #75364 appears narrower and more robust than prompt-only guidance because it preserves topic delivery even when the model follows the existing `action=send` instruction.
