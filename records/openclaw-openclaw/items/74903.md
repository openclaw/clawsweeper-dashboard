## Summary

Keep open. Current main still prompts topic-scoped Feishu group sessions toward `message(action=send)`, and the Feishu `send` action still posts without a topic reply target; the linked open closing PR should be reviewed or consolidated before this issue closes.

## What This Changes

The issue requests that Feishu `group_topic` and `group_topic_sender` sessions keep assistant-visible replies inside the triggering topic thread instead of sending them as root group messages.

## Best Possible Solution

Land or consolidate a Feishu-owned runtime fix so topic-scoped `send` actions inherit the inbound topic reply target while preserving existing plain group behavior.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: the group prompt sends the model to `message(action=send)`, while current Feishu `send` leaves `replyToMessageId` undefined and `replyInThread` false.

## Solution Assessment

Yes. A Feishu action-handler fix like #75364 is the narrowest maintainable solution because it preserves topic delivery even when the model follows the current `action=send` instruction; prompt guidance can be optional follow-up copy.
