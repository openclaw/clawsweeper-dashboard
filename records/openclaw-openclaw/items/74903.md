## Summary

Keep open. Current main still routes Feishu `message(action=send)` as a root chat send in topic-scoped sessions, and open closing PR #75364 plus overlapping PR #75159 should be reviewed or consolidated before this issue closes.

## What This Changes

Feishu `group_topic` and `group_topic_sender` message-tool sends should post inside the triggering topic thread instead of as normal group messages.

## Best Possible Solution

Land or consolidate a Feishu-owned runtime fix so topic-scoped `send` actions inherit the inbound topic reply target while preserving existing plain group behavior.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: the group prompt sends the model to `message(action=send)`, while current Feishu `send` leaves `replyToMessageId` undefined and `replyInThread` false.

## Solution Assessment

Yes. A Feishu action-handler fix is the narrowest maintainable direction because it preserves topic delivery even when the model follows the current `action=send` instruction; prompt guidance can be optional follow-up copy.
