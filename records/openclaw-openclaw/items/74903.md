## Summary

Keep open. Current main still routes Feishu topic-scoped `message(action=send)` as a root group send, and the issue is already paired with open PR #75364 plus overlapping prompt-only PR #75159 that should be reviewed or consolidated before closure.

## What This Changes

The issue asks for Feishu `group_topic` and `group_topic_sender` visible replies from the message tool to stay in the originating topic thread instead of posting as plain group messages.

## Best Possible Solution

Land or consolidate a Feishu-owned runtime fix so topic-scoped `send` actions inherit the inbound topic reply target while preserving existing plain group behavior.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: the group prompt sends the model to `message(action=send)`, while current Feishu `send` leaves `replyToMessageId` undefined and `replyInThread` false.

## Solution Assessment

Yes. A Feishu action-handler fix is the narrowest maintainable solution because it preserves topic delivery even when the model follows the current `action=send` instruction; prompt guidance can be optional follow-up copy.
