## Summary

Keep open. Current main still has the reported Feishu topic/tool-send mismatch, and this issue is explicitly covered by open closing PRs #75364 and #75159, so it should remain open until maintainers land or replace one of those fixes.

## What This Changes

The issue reports that Feishu `group_topic` and `group_topic_sender` sessions steer the assistant to `message(action=send)`, causing visible replies to post as normal group messages instead of topic-thread replies.

## Best Possible Solution

Land a Feishu-owned runtime fix that makes same-conversation `message(action=send)` from `group_topic` and `group_topic_sender` sessions carry the inbound topic root/current message into Feishu `reply_in_thread` transport, while preserving plain group send behavior and adding focused channel action regression tests.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: in a Feishu topic-scoped message-tool-only group turn, the prompt asks for `message(action=send)`, and the Feishu action handler sends that action with no `replyToMessageId` and `replyInThread: false`.

## Solution Assessment

Yes. The best direction is a narrow Feishu plugin runtime fix like #75364 because it preserves the delivery invariant even when the model follows the existing generic `action=send` prompt; a prompt-only nudge like #75159 is more model-dependent.
