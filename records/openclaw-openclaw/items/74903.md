## Summary

Keep open. Current main still routes Feishu topic-scoped `message(action=send)` through the plain group send path, and this issue is paired with open closing PR #75364 plus overlapping PR #75159, so it should remain open until the fix lands or the PRs are consolidated.

## What This Changes

Feishu `group_topic` and `group_topic_sender` sessions should post assistant message-tool replies back into the originating topic thread instead of the group root.

## Best Possible Solution

Land or consolidate a Feishu-owned runtime fix so topic-scoped message-tool sends inherit the inbound topic reply target while preserving existing plain group behavior.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: the group prompt directs the model to `message(action=send)`, while the current Feishu `send` handler leaves `replyToMessageId` undefined and `replyInThread` false.

## Solution Assessment

Yes. A Feishu action-handler fix is the narrowest maintainable solution because it preserves topic delivery even when the model follows the current `action=send` instruction; prompt guidance can be optional follow-up copy.
