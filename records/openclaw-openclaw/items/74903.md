## Summary

Keep open. Current main still matches the reported Feishu topic-session routing bug: the shared group prompt points visible tool replies at `message(action=send)`, but the Feishu action handler only sends thread replies when the action is explicitly `thread-reply`. An open closing PR (#75364) and overlapping prompt-only PR (#75159) should be reviewed or consolidated before this issue closes.

## What This Changes

The issue asks for Feishu `group_topic` and `group_topic_sender` message-tool sends to stay in the originating topic thread instead of posting as plain group messages.

## Best Possible Solution

Land or replace the Feishu-owned runtime fix from #75364 so same-conversation `message(action=send)` in topic-scoped sessions inherits the inbound topic reply target. A prompt hint like #75159 can be added as defense in depth, but delivery should not depend on model compliance.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: Feishu `group_topic` message-tool-only turns are prompted to call `message(action=send)`, and current main sends that action with no `replyToMessageId` and `replyInThread: false`.

## Solution Assessment

Yes. The narrow Feishu plugin runtime direction is the best fix because the action handler has the session key and current inbound message id, so it can preserve topic threads without relying on the model to choose `thread-reply`.
