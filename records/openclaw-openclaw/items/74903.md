## Summary

Keep open. Current main still leaves Feishu `group_topic` and `group_topic_sender` message-tool `send` actions as root group sends, and this issue is already paired with open closing PR #75364 plus overlapping PR #75159 that should be reviewed or consolidated before closure.

## What This Changes

The issue reports that Feishu topic-scoped group sessions post assistant `message(action=send)` replies as normal group messages instead of topic-thread replies.

## Best Possible Solution

Review and consolidate the open Feishu PRs, preferably landing a Feishu-owned runtime action-handler fix that makes topic-scoped `send` inherit the inbound topic reply target while preserving plain group behavior.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: the group prompt sends the model to `message(action=send)`, while current Feishu `send` leaves `replyToMessageId` undefined and `replyInThread` false.

## Solution Assessment

Yes. A Feishu action-handler fix is the narrowest maintainable solution because it preserves topic delivery even when the model follows the current `action=send` instruction; prompt guidance can be complementary.
