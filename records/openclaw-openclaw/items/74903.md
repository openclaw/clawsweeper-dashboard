## Summary

Keep open. Current main still routes Feishu topic-scoped `message(action="send")` through the non-threaded send path, and this issue is already paired with open closing PR https://github.com/openclaw/openclaw/pull/75364 plus overlapping PR https://github.com/openclaw/openclaw/pull/75159.

## What This Changes

The issue asks Feishu `group_topic` and `group_topic_sender` message-tool replies to stay inside the triggering topic thread instead of posting as plain group messages.

## Best Possible Solution

Ship a Feishu-owned runtime fix so topic-scoped `send` actions inherit the inbound topic reply target across text, card, and media paths while preserving plain group behavior.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: message-tool-only group prompts direct the model to `message(action=send)`, while Feishu `send` leaves `replyToMessageId` undefined and `replyInThread` false on current main.

## Solution Assessment

Yes. A Feishu action-handler fix is the narrowest maintainable solution because it preserves topic delivery even when the model follows the current `action=send` instruction; prompt guidance can be optional follow-up copy.
