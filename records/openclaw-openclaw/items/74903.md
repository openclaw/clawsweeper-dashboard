## Summary

Keep open. Current main still has the reported Feishu topic-session message-tool mismatch, and open closing PRs #75364 and #75159 already target the same issue, so this should remain open until one fix lands or is replaced.

## What This Changes

The report asks Feishu `group_topic` and `group_topic_sender` sessions to post visible assistant replies into the original topic thread instead of sending `message(action=send)` to the group root.

## Best Possible Solution

Land or replace a narrow Feishu plugin runtime fix that makes same-conversation `message(action=send)` from `group_topic` and `group_topic_sender` sessions inherit the inbound topic root/current message as a thread reply, while preserving plain group sends and adding focused action-handler regression tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: a message-tool-only Feishu group prompt asks for `message(action=send)`, and current Feishu `send` leaves `replyToMessageId` undefined with `replyInThread: false` even when the session is topic-scoped.

## Solution Assessment

Yes. The best repair direction is a Feishu-owned runtime fix like #75364 because it enforces the delivery invariant even if the model follows the current generic `action=send` prompt; #75159's prompt nudge is useful but more model-dependent.
