## Summary

Keep open. Current main still has the reported Feishu topic-session mismatch, and an open closing PR plus an overlapping prompt-only PR are already attached, so the issue should remain open until maintainers land or consolidate the fix.

## What This Changes

Feishu `group_topic` and `group_topic_sender` sessions should keep assistant `message(action=send)` replies inside the triggering topic thread instead of sending a plain group message.

## Best Possible Solution

Land or replace the Feishu-owned runtime fix that makes same-conversation `message(action=send)` from `group_topic` and `group_topic_sender` sessions use the inbound topic root or current message as the Feishu thread reply target, while preserving plain group `send` behavior and adding focused channel-action regression tests.

## Reproduction Assessment

Yes. The source path is high-confidence: the prompt tells the model to use `action=send`, and the current Feishu action handler sends that action with no reply target and `replyInThread: false`.

## Solution Assessment

Yes. The narrowest maintainable solution is a Feishu plugin runtime fix like #75364 because the action handler has the session/tool context needed to preserve the topic thread; #75159's prompt-only nudge is useful only as defense in depth.
