## Summary

Keep open. Current main still matches the reported Feishu topic-session mismatch, and an open closing PR plus an overlapping prompt-only PR are already attached, so repository policy says to leave this issue open until maintainers land or consolidate the fix.

## What This Changes

Fix Feishu `group_topic` and `group_topic_sender` sessions so assistant-visible `message(action=send)` replies stay in the triggering topic thread instead of posting as normal group messages.

## Best Possible Solution

Land or replace the Feishu-owned runtime fix that makes same-conversation `message(action=send)` from `group_topic` and `group_topic_sender` sessions use the inbound topic root/current message as the Feishu thread reply target, while preserving plain group `send` behavior and adding focused action-handler regression tests. A prompt hint can be added as defense in depth, but delivery should not depend on model compliance.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: the message-tool-only group prompt asks for `action=send`, and current Feishu `send` resolves no reply target and sets `replyInThread: false`; the focused channel-action tests do not cover topic-scoped send inheritance.

## Solution Assessment

Yes. The narrowest maintainable direction is the Feishu plugin runtime fix represented by #75364 because the bug is Feishu-specific delivery routing and the action handler has the session/tool context needed to preserve topic threads; #75159's prompt-only approach is useful only as an additional nudge.
