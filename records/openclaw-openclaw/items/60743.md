## Summary

Keep open. Current main does not implement the requested recent-continuity layer, and this external XL memory PR still has blocking correctness, real-proof, and data-access issues rather than being obsolete or closeable.

## What This Changes

The PR adds a `memory/recent` continuity snapshot module and wires it into session-memory reset/new handling, compact-before handling, compaction safeguard context, and post-compaction recovery context.

## Best Possible Solution

Rebase and narrow this into a maintainer-approved continuity slice that preserves current latency, local-time, collision, and post-compaction contracts; make provider extraction explicitly opt-in; and include real workspace-file proof.

## Reproduction Assessment

Yes for the review findings: source inspection of PR head and current main shows the default provider extraction path, archive overwrite regression, compact lifecycle mismatch, stale post-compaction signature, and stale summary-only snapshot path. The central recent-continuity layer is a feature, so there is no current-main bug reproduction required for the main ask.

## Solution Assessment

No as-is. The direction may be viable, but this patch is not the narrowest maintainable solution until it preserves current contracts, fixes compact lifecycle wiring, documents any new config/data boundary, and provides real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Keep LLM extraction opt-in and off reset replies:** `src/hooks/bundled/session-memory/handler.ts:239-243`
  - body: `allowLlmSlug` and `allowLlmExtract` both default to enabled and the handler now awaits provider-backed extraction on `/new` and `/reset`. Current behavior keeps reset replies responsive and only calls the model for slugs when `llmSlug: true`, so this can delay command replies and send transcript text to a provider without a new opt-in.
  - confidence: 0.95
- **[P2] Preserve unique archive filenames:** `src/hooks/bundled/session-memory/handler.ts:270`
  - body: The reset/new archive path now writes directly to `${dateStr}-${slug}.md`, dropping current main's collision loop. Two resets in the same minute, or two sessions with the same generated slug, will overwrite the earlier memory archive instead of creating a suffixed file.
  - confidence: 0.93
- **[P2] Wire compact snapshots through the real hook contract:** `src/hooks/bundled/session-memory/handler.ts:131`
  - body: The new `session:compact:before` branch is not registered by `HOOK.md`, and current compaction events do not include `workspaceDir`, `cfg`, or `sessionFile` for this handler to read. The advertised compact-before snapshot path will not run through normal loading, and simply adding the metadata would still produce an empty/default-workspace snapshot.
  - confidence: 0.94
- **[P2] Preserve the post-compaction options signature:** `src/auto-reply/reply/post-compaction-context.ts:70-72`
  - body: Current main calls `readPostCompactionContext(workspaceDir, { cfg, agentId })` so the helper can apply agent-specific context limits. This branch changes the signature back to positional `cfg`/`nowMs`, which will fail or silently ignore the current options object after rebase.
  - confidence: 0.91
- **[P2] Include summaries in material-change checks:** `packages/memory-host-sdk/src/host/continuity.ts:417-438`
  - body: When extraction is disabled or fails, `conversationSummary` can be the only meaningful state in a recent snapshot. `hasMaterialContinuityChange()` omits that field from both canonical objects, so a later session with different transcript text but the same fallback fields can be skipped and leave `latest.md` stale.
  - confidence: 0.9
