## Summary

Keep open. This PR is still the canonical implementation path for the proposed recent continuity feature, but the current patch has blocking correctness, data-access, lifecycle-registration, and real behavior proof gaps that need maintainer review before merge.

## What This Changes

The PR adds a `memory/recent` continuity snapshot layer and wires it into session-memory reset/new handling, compact-before handling, and post-compaction recovery context.

## Best Possible Solution

Rebase and narrow this into a maintainer-approved continuity layer that preserves current session-memory latency and local-time contracts, makes provider extraction explicitly opt-in, registers lifecycle metadata correctly, updates schema/docs/tests, and includes real behavior proof.

## Reproduction Assessment

Yes for the review findings: source inspection of PR head and current main shows the invalid type import, default provider extraction path, missing compact event registration, UTC archive regression, and stale summary-only snapshot path. The requested recent continuity layer itself is a feature, so there is no current-main bug reproduction required for the central ask.

## Solution Assessment

No as-is. The additive continuity direction may be viable, but this patch is not the narrowest maintainable solution until the data boundary, lifecycle registration, current behavior preservation, and proof gaps are addressed.

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Import OpenClawConfig from the exported type module:** `src/hooks/bundled/session-memory/handler.ts:21`
  - body: This branch imports `OpenClawConfig` from `src/config/config.js`, but that module does not export the type at PR head; the exported type lives in `src/config/types.openclaw.ts`. The changed files will fail type checking before the behavior can be exercised.
  - confidence: 0.97
- **[P1] Keep provider extraction opt-in and off reset replies:** `src/hooks/bundled/session-memory/handler.ts:239-243`
  - body: `allowLlmExtract` defaults to enabled and is awaited inside the hook, so default `/new` and `/reset` handling can send transcript content to the configured provider and wait on the embedded run. Preserve the current local/background behavior unless users explicitly opt in.
  - confidence: 0.95
- **[P2] Register the compact event before handling it:** `src/hooks/bundled/session-memory/handler.ts:131`
  - body: The new `session:compact:before` branch will not run when the bundled hook is loaded normally because `HOOK.md` still lists only `command:new` and `command:reset`, and the loader registers only metadata-listed events.
  - confidence: 0.96
- **[P2] Preserve local-time archive naming:** `src/hooks/bundled/session-memory/handler.ts:161`
  - body: Current main intentionally names session-memory archive files and headings in the host local timezone. This branch derives the archive date, fallback slug, and header time from `toISOString()`, which would return reset/new artifacts to UTC after rebase.
  - confidence: 0.93
- **[P2] Compare conversation summaries before skipping writes:** `packages/memory-host-sdk/src/host/continuity.ts:417-438`
  - body: When extraction is disabled or fails, `conversationSummary` can be the only meaningful recent state. `hasMaterialContinuityChange()` omits it from both canonical objects, so later sessions with different conversation text but the same fallback fields can be skipped and leave `latest.md` stale.
  - confidence: 0.91
- **[P3] Use the first non-empty fallback list section:** `packages/memory-host-sdk/src/host/continuity.ts:361-366`
  - body: `sectionToList()` always returns an array, so the nullish coalescing here never reaches the `Blockers` or singular `Next Step` fallback. User-authored continuity docs using those headings parse as empty lists; choose the first non-empty parsed list instead.
  - confidence: 0.82
