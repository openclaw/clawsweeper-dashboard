## Summary

Keep open, but the current PR head is not mergeable as-is: current main still lacks the proposed `memory/recent` layer, while the branch regresses existing session-memory contracts, widens transcript data access, and lacks required real behavior proof.

## What This Changes

The PR adds a `memory/recent` continuity snapshot layer, wires it into `/new`, `/reset`, compact-before, and post-compaction recovery, and adds continuity parsing/rendering tests.

## Best Possible Solution

Rework this into a maintainer-approved continuity slice that preserves existing session-memory contracts, makes provider extraction explicitly opt-in, uses the real compaction lifecycle contract, documents any new config, and includes redacted runtime file proof.

## Reproduction Assessment

Yes for the review findings: source inspection of current main and PR head shows the default provider extraction path, archive filename/timezone regressions, unreachable or under-contextual compact handling, post-compaction signature drift, and summary-only snapshot skip. The central recent-continuity layer is a new feature, not a current-main bug repro.

## Solution Assessment
