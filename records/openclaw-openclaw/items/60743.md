## Summary

Keep open, but do not merge as-is. Current main does not implement the proposed recent-continuity layer, and the linked feature issue was closed in favor of this PR, but the PR is an external XL core memory feature with missing real behavior proof, a default-on provider data-access expansion, and several current-main contract regressions.

## What This Changes

This PR adds a `memory/recent` continuity snapshot layer, LLM extraction helper, `/new`/`/reset` and compact-before write paths, post-compaction recovery reads, and focused tests across memory-host, session-memory, and compaction context code.

## Best Possible Solution

Rework this into a smaller maintainer-approved continuity slice that preserves current session-memory contracts, uses the real compaction lifecycle with the needed context, makes provider extraction explicitly opt-in, and includes real workspace-file proof.

## Reproduction Assessment

Yes for the review findings: PR-head source inspection against current main shows the default provider extraction path, filename collision regression, compact lifecycle mismatch, post-compaction signature drift, and summary-only material-change omission. The central recent-continuity layer is a feature, not a current-main bug.

## Solution Assessment

No as-is. The direction may be viable, but the current patch is not the narrowest maintainable solution until it preserves existing contracts, documents and gates the data-access boundary, and provides real behavior proof.
