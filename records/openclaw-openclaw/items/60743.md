## Summary

Keep open: current main still lacks the requested `memory/recent` continuity layer, and the paired feature issue was already consolidated into this PR, but the branch is not merge-ready because it changes core memory/data-access behavior and still has blocking correctness, security, and proof gaps.

## What This Changes

The PR adds a `memory/recent` snapshot protocol, writes snapshots from session-memory reset/new and compact-before paths, reads them during post-compaction recovery, and adds focused tests.

## Best Possible Solution

Rework this into a smaller maintainer-approved continuity slice that preserves current session-memory contracts, uses the real compaction lifecycle with the needed context, gates provider extraction explicitly, and includes real workspace-file proof.

## Reproduction Assessment

Yes for the review findings: PR-head source inspection against current main shows the default provider extraction path, filename collision regression, compact lifecycle mismatch, post-compaction signature drift, and summary-only material-change omission. The central recent-continuity layer is a feature, not a current-main bug.

## Solution Assessment

No as-is. The direction may be viable, but this patch is not the narrowest maintainable solution until it preserves existing contracts, documents and gates the data-access boundary, and provides real behavior proof.
