## Summary

Current main still lacks the requested recent-continuity layer, and the PR is not mergeable as-is: the branch is dirty against main and introduces a security-sensitive LLM opt-out regression plus several correctness regressions around hook registration, local-time archive naming, and snapshot freshness.

The branch adds a `memory/recent` snapshot format and helper APIs, writes recent continuity files from session-memory on reset/new and pre-compaction, reads them during post-compaction recovery context, and adds focused tests.

Full review comments:

- **[P1] Respect the existing LLM opt-out before extracting continuity:** `src/hooks/bundled/session-memory/handler.ts:240-243`
  - body: Current session-memory lets operators set `llmSlug: false` to avoid sending transcript content through the embedded model call. This branch adds a separate `llmExtract` path that defaults to true, so the hook can still send the same session excerpt to the configured model even when the existing opt-out is set; gate extraction behind the existing opt-out or make the new behavior explicitly opt-in/migrated.
  - confidence: 0.86
- **[P2] Register the compaction event for session-memory:** `src/hooks/bundled/session-memory/handler.ts:131`
  - body: The new `session:compact:before` branch will not run when the bundled hook is loaded normally: `HOOK.md` still registers only `command:new` and `command:reset`, and the loader registers handlers only for metadata-listed events. Add `session:compact:before` to the hook metadata/docs or remove the advertised compaction path.
  - confidence: 0.96
- **[P2] Preserve local-time archive naming:** `src/hooks/bundled/session-memory/handler.ts:161`
  - body: Current main intentionally names session-memory archives and headers in the host local timezone, with regression coverage for non-UTC hosts. This branch derives the archive date, fallback slug, and header from `toISOString()`, so `/new` and `/reset` would go back to UTC-dated files after rebase unless the local-time path is kept.
  - confidence: 0.93
- **[P2] Compare the summary before skipping snapshot writes:** `packages/memory-host-sdk/src/host/continuity.ts:417-438`
  - body: When LLM extraction is disabled or fails, `conversationSummary` can be the only meaningful recent state. `hasMaterialContinuityChange()` omits it from both canonical objects, so later sessions with different conversation text but the same fallback fields can be skipped and leave `latest.md` stale.
  - confidence: 0.9
- **[P3] Honor fallback section headings:** `packages/memory-host-sdk/src/host/continuity.ts:361-366`
  - body: `sectionToList()` always returns an array, so the nullish coalescing here never reaches the `Blockers` or singular `Next Step` fallback. User-authored continuity docs using those headings parse as empty lists; choose the first non-empty parsed list instead.
  - confidence: 0.82
- **[P3] Add the required changelog entry:** `src/hooks/bundled/session-memory/handler.ts:301`
  - body: This is a user-facing feature branch, but the PR does not touch `CHANGELOG.md`. OpenClaw policy requires user-facing `feat` changes to add a single-line active-version changelog entry with appropriate credit before merge.
  - confidence: 0.88
