## Summary

Keep open: current main does not contain the recent-continuity feature, but this PR is not mergeable as-is because it widens transcript provider access by default, regresses existing session-memory contracts, and lacks required real behavior proof.

## What This Changes

This PR adds a `memory/recent` continuity snapshot layer, an LLM extraction helper, and recovery wiring for `/new`, `/reset`, compact-before, compaction safeguard, and post-compaction context.

## Best Possible Solution

Rebase into a smaller maintainer-approved continuity slice that preserves async reset behavior, filename uniqueness, compaction hook contracts, agent-specific context limits, and makes provider extraction explicitly opt-in with real workspace-file proof.

## Reproduction Assessment

Yes for the review findings: source inspection of PR head against current main shows the default provider extraction path, archive overwrite regression, compact lifecycle mismatch, stale post-compaction signature, and summary-only snapshot skip. The central recent-continuity layer is a feature, not a current-main bug.

## Solution Assessment

No as-is. The direction may be viable, but this patch is not the narrowest maintainable solution until it preserves current contracts, fixes compact lifecycle wiring, documents the data/config boundary, and provides real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Keep LLM extraction opt-in and off reset replies:** `src/hooks/bundled/session-memory/handler.ts:239-243`
  - body: `allowLlmSlug` and `allowLlmExtract` now default to enabled, and the async handler awaits `generateContinuitySnapshotViaLLM`. Current main keeps session-memory work off the command reply path and only calls the model when `llmSlug: true`, so this can delay `/new` or `/reset` replies and send transcript content to the configured provider without opt-in.
  - confidence: 0.95
- **[P2] Preserve unique archive filenames:** `src/hooks/bundled/session-memory/handler.ts:270`
  - body: The branch writes reset/new archives directly to `${dateStr}-${slug}.md`, dropping current main's collision suffix loop. Two writes in the same minute, or two sessions with the same model slug, will overwrite the earlier memory archive.
  - confidence: 0.93
- **[P2] Register compact snapshots on the real hook contract:** `src/hooks/bundled/session-memory/handler.ts:131`
  - body: The handler adds a `session:compact:before` branch, but `HOOK.md` still registers only `command:new` and `command:reset`, and the current compaction event only provides counts/session ids. The advertised compact-before snapshot path will not run through normal loading or would lack the workspace/session file it needs.
  - confidence: 0.94
- **[P2] Preserve the post-compaction options signature:** `src/auto-reply/reply/post-compaction-context.ts:69-72`
  - body: Current main calls `readPostCompactionContext(workspaceDir, { cfg, agentId })` so agent-specific limits apply. This branch changes the helper back to positional `cfg`/`nowMs`; after rebase, callers will pass the options object as config and lose `agentId`/limits or fail typing.
  - confidence: 0.91
- **[P2] Include summaries in material-change checks:** `packages/memory-host-sdk/src/host/continuity.ts:417-438`
  - body: When extraction is disabled or fails, `conversationSummary` can be the only useful change, but `hasMaterialContinuityChange()` excludes it from both canonical comparisons. A later transcript-only update can be skipped and leave `latest.md` stale.
  - confidence: 0.9
