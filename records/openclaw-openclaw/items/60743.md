## Summary

Keep open: current main still lacks the proposed `memory/recent` continuity layer, but the PR is not merge-ready because it changes core memory/data-access behavior, has source-proven correctness regressions, and still lacks real behavior proof.

## What This Changes

The PR adds a `memory/recent` snapshot format, writes recent continuity files from session-memory reset/new and compaction paths, reads them during post-compaction recovery, and adds focused tests.

## Best Possible Solution

Rework this into a smaller maintainer-approved continuity slice that preserves current session-memory contracts, gates provider extraction explicitly, uses the real `session:compact:before` context, and includes real workspace-file proof.

## Reproduction Assessment

Yes for the review findings: source inspection against current main and PR head shows the default provider extraction path, archive filename regression, compact lifecycle mismatch, post-compaction signature drift, and summary-only material-change omission. The central recent-continuity layer is a new feature, not a current-main bug.

## Solution Assessment

No as-is. The direction may be viable, but this patch is not the narrowest maintainable solution until it preserves existing contracts, documents and gates the data-access boundary, and provides real behavior proof.
