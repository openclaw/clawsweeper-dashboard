Current main calls a provider only when `llmSlug: true` and keeps capture off the command reply path, so this can delay `/new` or `/reset` acknowledgements and send transcript text to a provider without opt-in.
  - confidence: 0.95

  - body: The branch derives `dateStr`, fallback slug, and the archive heading from `toISOString()`, while current main formats session-memory artifacts in the user's local timezone and the docs promise the host local date. Merging this would move archives around UTC boundaries and make `/new`/`/reset` files inconsistent with current behavior.
  - confidence: 0.91
- **[P2] Preserve unique archive filenames:** `src/hooks/bundled/session-memory/handler.ts:270`
  - body: The archive path is now `${dateStr}-${slug}.md`, dropping current main's suffix loop. Repeated reset/new captures in the same minute or with the same generated slug can overwrite earlier memory archives.
  - confidence: 0.93
- **[P2] Register compact snapshots on a real compaction contract:** `src/hooks/bundled/session-memory/handler.ts:131`
  - body: The new `session:compact:before` branch is not reachable through bundled hook metadata, which still lists only `command:new` and `command:reset`; the current compact-before event also lacks the session file/config/workspace context this branch needs for a useful transcript-backed snapshot.
  - confidence: 0.94
