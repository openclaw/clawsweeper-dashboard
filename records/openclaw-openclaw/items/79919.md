## Summary

Keep open: the PR targets a real voice-call route-boundary bug, but the proposed helper introduces a regression for an allowed root stream path and the external proof is only targeted test output.

## What This Changes

The PR changes the voice-call realtime WebSocket upgrade classifier from raw prefix matching to exact-or-slash-child matching and adds a regression test for sibling stream paths.

## Best Possible Solution

Keep the focused route-boundary fix, but adjust the helper and tests so root stream paths still route tokenized child upgrades, then add real after-fix runtime proof before landing.

## Reproduction Assessment

Yes. The original bug is source-reproducible from current main's raw `startsWith` classifier, and the PR regression is source-reproducible from allowed `realtime.streamPath: "/"` plus tokenized TwiML child paths.

## Solution Assessment

No. Exact-or-slash-child matching is the right direction for normal stream paths, but this implementation needs to preserve or explicitly handle the root stream path before it is the narrowest maintainable fix.

## Review Findings

- **[P2] Preserve root stream child upgrades:** `extensions/voice-call/src/webhook.ts:182-183`
  - body: When `realtime.streamPath` is configured as `/`, `getStreamPathPattern()` returns `/` and TwiML appends `/${token}` for the WebSocket URL.
