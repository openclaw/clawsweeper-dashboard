## Summary

Keep open: this is a new Telegram notification feature/config request, not a stale bug or already-implemented behavior. Current main has the building blocks for cron delivery, image generation, and Telegram image sending, but it does not define a notification-level image template/cache/fallback contract.

## What This Changes

Add a notification-level way for Telegram recurring/proactive notifications to generate an image, attach it, reuse/cache it when practical, and fall back to text on image-generation failure.

## Best Possible Solution

Define the product boundary first: either document a cron plus image_generate/message-tool recipe, or add a plugin-owned notification image template/cache contract with fallback behavior and tests.
