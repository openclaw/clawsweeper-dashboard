## Summary

Keep open. Current main prevents unsafe mixed-vector search by pausing/rebuilding a single index, but it still lacks the requested first-class per-model indexes, active/fallback index routing, per-index ingestion state, and observability.

## What This Changes

Request multi-index embedding memory support with model-aware failover so different embedding models do not share one vector space.

## Best Possible Solution

Keep this as the canonical memory-core design tracker and implement a maintainer-approved, backward-compatible per-index schema, ingestion state machine, query router, status surface, and doctor/migration path inside the memory plugin/API boundary.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/design request. Source inspection confirms the current behavior remains one resolved memorySearch provider/model/store with mismatch guards rather than multiple active indexes.

## Solution Assessment

No, not as an autonomous fix yet. The requested direction is plausible, but the best solution needs maintainer-approved memory-core/plugin API design for config compatibility, schema migration, per-index ingestion, routing, and observability.
