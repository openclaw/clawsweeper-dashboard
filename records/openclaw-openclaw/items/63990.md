## Summary

Keep open. Current main and v2026.6.8 still have a single resolved memorySearch provider/model/fallback/store and guard against mixed vector spaces by pausing mismatched indexes, but they do not implement per-model indexes, coordinated dual ingestion, active/fallback index routing, or safe failover without rebuild.

## What This Changes

Request first-class multi-index embedding memory with one vector index per embedding model, model-aware query failover, status visibility, and backward-compatible config.

## Best Possible Solution

Keep this as the canonical memory-core design tracker for maintainer-approved per-model index storage, dual ingestion/backfill, active/fallback routing, status metrics, and doctor migration.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature proposal. Source and release inspection confirm current main still has scalar memorySearch provider/model/fallback/store behavior rather than multi-index failover.

## Solution Assessment

Unclear as a final design. Per-model indexes are directionally safer than single-index fallback, but the maintainable solution needs maintainer-approved config, storage, plugin/API, status, and migration boundaries.
