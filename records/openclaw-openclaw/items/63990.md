## Summary

Keep open. Current main still has one resolved memory embedding provider/model/store and fallback triggers a full reindex, while the requested multi-index failover is a broad public config, storage, ingestion, query routing, observability, migration, and docs feature that needs maintainer design.

## What This Changes

Request first-class per-model memory embedding indexes with active/fallback routing, independent ingestion state, status/metrics, and backward-compatible single-index config.

## Best Possible Solution

Design an additive memory contract that preserves current single-index installs while adding explicit per-model indexes, canonical chunk storage, per-index backfill/status, single-index query routing with controlled failover, validation, and observability.

## Solution Assessment
