## Summary

Keep open. Current main and v2026.6.10 still expose one resolved memorySearch provider/model/fallback/store and one canonical memory index, with safety guards that avoid mixed vector spaces, but they do not implement per-model indexes, coordinated dual ingestion, active/fallback index routing, or safe provider failover without rebuild.

## What This Changes

The issue requests multi-index embedding memory with model-aware failover, one vector index per embedding model, dual ingestion, explicit active/fallback index routing, and observability.

## Best Possible Solution

Keep this as the canonical maintainer design tracker for memory-core multi-index failover rather than queueing an automated fix.

Not applicable as a bug reproduction; this is a feature request. Current-main and v2026.6.10 source inspection confirm the requested multi-index behavior is absent.

## Solution Assessment

Unclear as a final design. Per-model indexes are directionally safer than single-index fallback, but the config, storage, plugin/API, status, and migration shape require maintainer product review.
