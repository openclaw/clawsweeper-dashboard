- `impact:session-state`: The request would change persisted memory index state, ingestion consistency, active memory recall, and vector-search behavior.

## Summary

Keep open. Current main and v2026.6.10 still expose one resolved memorySearch provider/model/fallback and one canonical memory index, with guards that avoid mixed vector-space search, but they do not implement per-model indexes, coordinated dual ingestion, active/fallback index routing, or safe provider failover without rebuild.

## What This Changes

The issue requests first-class multi-index embedding memory with per-model indexes, coordinated ingestion, model-aware querying, safe failover, observability, and backward-compatible config.

## Best Possible Solution

Keep this as the canonical maintainer design tracker for memory-core multi-index failover, define the config/storage/migration/status contract first, and let https://github.com/openclaw/openclaw/issues/96534 handle the narrower fallback recovery bug separately.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/design request with production context. Current-main and v2026.6.10 source inspection confirm the requested multi-index behavior is absent rather than failing to reproduce.

## Solution Assessment

Unclear as a final implementation design. Per-model indexes are directionally safer than single-index fallback, but the config, storage, plugin/API, status, and migration shape require maintainer product review before a fix PR.
