## Summary

Keep open. Current main and v2026.6.1 still expose one memorySearch provider/model/fallback surface and one active vector index, while the linked duplicate cleanup explicitly made this issue the canonical tracker for the remaining multi-index failover design.

## What This Changes

Requested feature: add multi-index embedding memory with per-model indexes and model-aware failover so OpenClaw can fail over embedding providers without mixing vector spaces.

## Best Possible Solution

Keep this as the canonical memory-core design tracker and implement a maintainer-approved, backward-compatible per-index schema, ingestion state, query router, status surface, and doctor/migration story inside the memory plugin/API boundary.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature proposal. Source and release inspection confirm the current behavior remains a single active memorySearch provider/model/index, with provider/model changes treated as index identity mismatches that require rebuild.

## Solution Assessment

No, not as an autonomous fix yet. The right solution needs maintainer-approved memory-core/plugin API design for per-index storage, ingestion status, routing, observability, and upgrade behavior rather than a narrow code patch.
