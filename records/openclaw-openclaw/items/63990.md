## Summary

Keep open. Current main and v2026.6.8 still expose a single resolved memorySearch provider/model/fallback/store and protect vector correctness by pausing mismatched indexes, but they do not implement per-model indexes, coordinated ingestion, active-index routing, or safe embedding failover without rebuild.

## What This Changes

Request first-class multi-index embedding memory with separate per-model vector indexes, model-aware query failover, status observability, and backward-compatible memorySearch config.

## Best Possible Solution

Keep this open as the canonical memory-core design tracker for maintainer-approved per-index storage/config, ingestion and backfill, active/fallback routing, observability, and doctor migration.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature proposal. Source and docs confirm current main still has scalar memorySearch provider/model/fallback/store behavior rather than multi-index failover.

## Solution Assessment

Unclear as a final design. Per-model indexes are directionally safer than single-index fallback, but the best solution needs maintainer-approved config, storage, plugin/API, status, and migration boundaries.
