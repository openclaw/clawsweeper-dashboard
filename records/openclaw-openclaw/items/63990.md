## Summary

Keep open. Current main still exposes one resolved memory embedding provider/model/store and handles fallback by switching the provider followed by reindexing, while this request is a larger multi-index config, storage, ingestion, routing, observability, and migration feature.

## What This Changes

Add first-class multi-index embedding memory so each embedding model has its own index, coordinated ingestion state, model-aware query routing, and controlled failover without mixing vector spaces.

## Best Possible Solution

Design an additive memory contract that preserves current single-index installs while adding canonical chunks, per-model indexes, per-index backfill/status, single-index query routing with controlled failover, dimension validation, and observability.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs give high-confidence evidence that current main resolves one memory provider/model/store and treats provider/model changes as full-reindex events.

## Solution Assessment

Unclear for the exact proposed field names, but the multi-index direction is sound. The maintainable path is an approved additive contract across config, storage, ingestion, routing, observability, migration, and docs rather than a narrow fallback-only patch.
