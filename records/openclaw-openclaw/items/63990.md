## Summary

Keep open. Current main still has one active memory embedding provider/model/store, uses full reindexing when provider or model identity changes, and has no first-class per-model index routing or per-index status; the related fallback issues are already closed in favor of this canonical design request.

## What This Changes

The issue requests multi-index memory embeddings with per-model indexes, active/fallback routing, per-index ingestion state, observability, and backward-compatible config.

## Best Possible Solution

Design an additive memory contract that preserves single-index installs while adding canonical chunks, per-model indexes, per-index backfill/status, single-index query routing with controlled failover, dimension validation, and status/metrics/docs.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, and live issue context give high-confidence evidence that current main still has one active memory provider/model/store and treats provider/model changes as full-reindex events.

## Solution Assessment

Unclear for the exact proposed field names, but the multi-index direction is sound. The maintainable path is an approved additive contract across config, storage, ingestion, routing, observability, migration, tests, and docs rather than a narrow fallback-only patch.
