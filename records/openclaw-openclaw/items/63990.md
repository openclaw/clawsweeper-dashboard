## Summary

Keep open. Current main and the latest release guard against unsafe mixed-vector search, but memorySearch still resolves to one provider/model/store and does not provide the requested per-model indexes, active/fallback index routing, per-index ingestion state, or failover observability.

## What This Changes

Request first-class per-model memory embedding indexes with model-aware active/fallback routing, coordinated ingestion, status output, and backward-compatible single-index config.

## Best Possible Solution

Keep this as the canonical memory-core design tracker and implement a maintainer-approved, backward-compatible per-index schema, ingestion pipeline, query router, status surface, and doctor migration inside the memory plugin/API boundary.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/design request. Source inspection gives high confidence that current main remains a single-index memorySearch implementation with mismatch guards rather than first-class multi-index failover.

## Solution Assessment

No, not as an autonomous fix yet. The requested direction is plausible, but the best solution needs maintainer-approved memory-core/plugin API design for config compatibility, schema migration, per-index ingestion, routing policy, and observability.
