## Summary

Keep open. Current main and v2026.6.6 still expose one resolved memorySearch provider/model/store and protect vector correctness by pausing mismatched indexes, but they do not implement model-aware multi-index ingestion, active-index routing, or failover without rebuild.

## What This Changes

The issue requests first-class multi-index embedding memory with per-model indexes, active-index routing, independent ingestion status, failover, observability, and backward-compatible single-index behavior.

## Best Possible Solution

Keep this as the canonical memory-core design tracker for a maintainer-approved per-index schema, ingestion state machine, active/fallback query router, status surface, docs, and doctor/migration plan.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature proposal. Source, docs, and latest-release inspection confirm current behavior remains a single active memorySearch provider/model/store with index-identity mismatch protection rather than multi-index failover.

## Solution Assessment

Unclear as a final design. The proposed architecture is directionally sound, but the best solution needs maintainer-approved memory-core/plugin API design for per-index storage, compatibility, migration, routing, and observability.
