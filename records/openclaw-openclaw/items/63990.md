## Summary

Keep open. Current main and v2026.6.10 still have a scalar memorySearch provider/model/fallback and one built-in SQLite index with identity guards, but they do not implement per-model indexes, coordinated dual ingestion, active/fallback index routing, or safe failover without rebuild.

## What This Changes

The issue requests first-class multi-index embedding memory with per-model indexes, dual ingestion, explicit active/fallback routing, observability, and backward-compatible single-index defaults.

## Best Possible Solution

Keep this issue as the canonical maintainer design tracker for memory-core multi-index failover, then implement only after the config, storage, routing, observability, and migration contract is approved.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Current-main and v2026.6.10 source inspection confirm the requested multi-index behavior is absent.

## Solution Assessment

Unclear as a final design. Per-model indexes are directionally safer than single-index fallback, but the config, storage, plugin/API, status, and migration shape require maintainer product review.
