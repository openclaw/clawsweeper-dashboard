## Summary

Keep open. Current main and v2026.6.10 still expose a single resolved memorySearch provider/model/fallback/store and guard against mixed vector spaces by pausing mismatched indexes, but they do not implement per-model indexes, coordinated dual ingestion, active/fallback index routing, or safe failover without rebuild.

## What This Changes

Issue requests first-class multi-index memory embeddings with per-model indexes, model-aware active/fallback routing, coordinated ingestion, status visibility, and backward-compatible config.

## Best Possible Solution

Keep this as the canonical maintainer design tracker for memory-core multi-index failover rather than queueing an automated fix.

Not applicable as a bug reproduction; this is a feature request. Current-main and v2026.6.10 source inspection confirm the requested multi-index behavior is absent.

## Solution Assessment

Unclear as a final design. Per-model indexes are directionally safer than single-index fallback, but the config, storage, plugin/API, status, and migration shape require maintainer product review.
