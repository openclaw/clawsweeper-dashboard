- **[P1] Preserve fs-safe run-log writes:** `src/cron/run-log.ts:210-213`
  - body: Current main appends run logs through appendRegularFile with rejectSymlinkParents and reads all-job logs through an fs-safe root. This head switches the write path to bare fs.appendFile and the all-jobs read path to bare fs.readdir/fs.readFile, so merging it would drop existing private run-log fs-safety hardening.
  - confidence: 0.86
- **[P1] Preserve current run-log filters and search fields:** `src/cron/run-log.ts:437`
  - body: Current main parses runId, diagnostics, delivery, and failureNotificationDelivery, and cron.runs searches diagnostics and delivery channels. This head removes those parsed fields and narrows query text, so existing runId filters and diagnostics/delivery searches would stop working after merge.
  - confidence: 0.88
- **[P2] Grow tail reads until keepLines are complete:** `src/cron/run-log.ts:166`
  - body: The prune window is capped at maxBytes or keepLines * 2048, so recent JSONL entries larger than 2 KB can leave fewer than keepLines complete records available before rewriting. Expand the tail in bounded chunks until enough complete lines are collected or BOF is reached.
  - confidence: 0.9
- **[P2] Drop the first streamed line only when partial:** `src/cron/run-log.ts:184-185`
  - body: The prune path drops the first streamed line whenever startPos > 0, but startPos can land immediately after a newline. In that case the line is complete, so pruning loses a valid recent entry.
  - confidence: 0.87
