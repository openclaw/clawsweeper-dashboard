## Summary

Close this PR as not actionable for the OpenClaw source repo: it is a custom downstream Docker image/workspace seed with blocking build and security issues, not a generic upstream Docker capability.

## What This Changes

The PR modifies the root Docker image and ignore files to bake a custom agent workspace, install a Python venv with websockets, add first-run config/auth/USER templates, and run a custom entrypoint.

## Best Possible Solution

Keep this as a downstream image; upstream any need as a generic, opt-in Docker config-seeding hook with reviewed inputs, tests, docs, and no embedded personal workspace or provider-specific defaults.

## Reproduction Assessment

Yes for the PR-introduced failures: source inspection shows the documented Docker build path leaves OPENCLAW_DOCKER_APT_PACKAGES empty while the PR runs python3 -m venv unconditionally and later calls envsubst from the entrypoint. I did not run Docker because this was a read-only review.

## Solution Assessment

No. Baking a custom workspace, templates, unpinned Python dependency, and startup policy into the upstream Dockerfile is not the narrow maintainable solution; a downstream image or generic opt-in seeding mechanism is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P1] Gate Python venv setup behind installed dependencies:** `Dockerfile:262-264`
  - body: This RUN now executes for every root Docker image, but current main only installs python3 by default and OPENCLAW_DOCKER_APT_PACKAGES defaults to empty. Debian slim images need python3-venv for `python3 -m venv`, so the documented `docker build -f Dockerfile .` path breaks unless the user knows to pass extra packages.
  - confidence: 0.94
- **[P1] Do not COPY a gitlink workspace:** `Dockerfile:269`
  - body: `openclaw-team-workspace` is added only as a subproject commit, while this COPY expects actual reviewed files in the build context. Normal checkouts will not have those contents without submodule metadata, and baking external unreviewed workspace contents into the root image is not reproducible.
  - confidence: 0.96
- **[P2] Install envsubst before using the entrypoint:** `scripts/openclaw-entrypoint.sh:41`
  - body: The new entrypoint calls `envsubst`, but the runtime image does not install gettext-base unless the caller supplies a custom apt package build arg. That means images built through the documented defaults can build past this file but fail immediately on first start.
  - confidence: 0.9
- **[P2] Generate seeded config with a JSON encoder:** `scripts/openclaw-entrypoint.sh:41`
  - body: The entrypoint writes JSON templates by piping raw environment variables through `envsubst`. Values such as names, timezones, tokens, or API keys containing quotes, backslashes, or newlines will corrupt the JSON or change its structure; use the existing config CLI or a JSON-aware generator instead.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The diff introduces concrete Docker supply-chain and startup-policy concerns in addition to the functional build failures.

Concerns:

- **[high] Unreviewed gitlink is copied into the image:** `Dockerfile:269`
  - body: The PR adds `openclaw-team-workspace` as a subproject commit and copies it into `/app/agent-workspace` without including reviewed content or `.gitmodules` metadata, making the image depend on external workspace contents outside the repo diff.
  - confidence: 0.96
- **[medium] Unpinned PyPI package is installed at image build time:** `Dockerfile:263`
  - body: The Dockerfile installs `websockets` from PyPI without a version, hash, lockfile, or internal package policy, which weakens the repository's reproducible Docker build and supply-chain posture.
  - confidence: 0.92
- **[low] Insecure Control UI flag becomes a baked default:** `templates/openclaw.template.json:49`
  - body: The seeded template enables `gateway.controlUi.allowInsecureAuth=true`; current docs and audit code treat this as an operator-selected compatibility toggle that should remain unset by default.
  - confidence: 0.9
- **[medium] Raw envsubst writes secret-bearing config JSON:** `scripts/openclaw-entrypoint.sh:41`
  - body: The entrypoint substitutes user-provided environment variables directly into JSON templates for gateway config and auth profiles, which can corrupt or inject config structure and stores provider keys as inline plaintext rather than using the existing config/SecretRef paths.
  - confidence: 0.86
