## Summary

Keep open: current main and the latest release still advertise MiniMax music `durationSeconds` support while the provider only embeds duration as prompt text and sends no structured duration field. The issue is already paired with open closing PRs, so it should remain open until maintainers choose and land the contract-safe fix path.

## What This Changes

MiniMax music generation should stop silently advertising nonfunctional `durationSeconds` control, either by honoring a documented vendor request field or by removing the unsupported capability and docs claims.

## Best Possible Solution

Land the contract-aligned linked fix that removes unsupported MiniMax duration advertising unless MiniMax supplies proof of a documented structured duration request field, then close this issue after merge.

## Reproduction Assessment

Yes for the source-level mismatch: current main advertises MiniMax duration support and docs list `durationSeconds`, while the provider sends no structured duration field. I did not independently run a live MiniMax generation to prove the default-length output.

## Solution Assessment
