## Summary

Keep open: current main still advertises MiniMax music duration support while the provider only embeds the requested duration in prompt text, and the current MiniMax OpenAPI schema does not show a structured request duration field to pass through safely.

## What This Changes

The issue asks OpenClaw to make `durationSeconds` actually affect MiniMax `music_generate` requests instead of silently producing default-length tracks.

## Best Possible Solution

Align MiniMax music provider capabilities with the verified MiniMax API contract: either pass the official duration field if MiniMax documents one, or stop advertising duration support so OpenClaw warns that `durationSeconds` is ignored.

## Reproduction Assessment

Yes for the adapter mismatch: current source advertises duration support and forwards `durationSeconds` only as prompt text, while the upstream OpenAPI schema lacks a request duration field. No live MiniMax output-length run was performed in this read-only review.

## Solution Assessment

No to the proposed pass-through field as written until MiniMax documents the exact request key. The safer narrow fix is to correct provider capabilities and tests, or pass only a field confirmed by the official API contract.
