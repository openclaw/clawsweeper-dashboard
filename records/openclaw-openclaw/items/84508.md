## Summary

Keep open: current main still advertises MiniMax music duration support while only placing durationSeconds in prompt text, but the proposed structured-field fix is not safe to apply without MiniMax contract confirmation because the current official OpenAPI schema does not list a duration request field.

## What This Changes

The issue asks for MiniMax music generation to honor durationSeconds by sending a structured MiniMax API duration parameter instead of relying on a prompt hint.

## Best Possible Solution

First confirm MiniMax's authoritative contract for music duration control; then either send the documented structured field with regression coverage, or stop advertising MiniMax duration support and update docs/tests so durationSeconds is reported as unsupported for this provider.

## Reproduction Assessment

Yes for the source-level mismatch: current main advertises MiniMax duration support and forwards durationSeconds only as prompt text. No for the exact generated-track length, because this review did not run a live MiniMax request with credentials.

## Solution Assessment

No as written. The proposed structured duration field needs upstream confirmation because the current MiniMax OpenAPI schema does not list a duration request parameter, so the maintainable fix may be capability/docs correction rather than adding an undocumented body field.
