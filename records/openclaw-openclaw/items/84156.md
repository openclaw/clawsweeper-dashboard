## Summary

Keep open for maintainer review. The PR appears to repair the cron all-error-payload recovery path with focused tests and sufficient live-output proof, but it intentionally changes persisted cron status semantics for recovered tool-error runs.

## What This Changes

The branch updates `resolveCronPayloadOutcome` and helper tests so all-error tool payloads can be marked recovered by terminal assistant-visible text when no fatal run signal, denial, or structured delivery payload exists.

## Best Possible Solution

Land the resolver-level fix once maintainers accept the recovered-terminal-assistant status contract, keeping typed run errors, fatal failure signals, denial markers, structured delivery, and unrecovered trailing errors fatal.

## Reproduction Assessment

Yes. Current main has a focused helper test and resolver logic showing all-error payloads with final assistant text still select the last error, and `finalizeCronRun` maps that fatal helper result to persisted cron `status: "error"`.

## Solution Assessment

Yes, with a maintainer policy check. The PR fixes the implicated resolver instead of only changing summary text, while preserving typed run failures, fatal failure signals, denial markers, structured delivery safeguards, and unrecovered trailing errors.
