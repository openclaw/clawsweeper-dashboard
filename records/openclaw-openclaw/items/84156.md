## What This Changes

The branch updates the isolated cron payload outcome resolver and tests so recovered `isError` payloads can resolve nonfatal when terminal assistant-visible text or other recovery evidence exists.

## Best Possible Solution

Land the resolver-level recovery classification if maintainers accept the recovered-terminal-assistant status contract, preserving the existing fatal guardrails and focused regression coverage.

## Solution Assessment

Yes, if maintainers accept the status contract. The PR fixes the central resolver instead of adding a new config path, and keeps run-level errors, fatal failure signals, denial markers, structured delivery, and unrecovered trailing errors fatal.
