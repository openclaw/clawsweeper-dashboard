## Summary

Keep open: current main implements the status-indicator subset, but the room/channel/title/delivery/cost/lineage context work is still unique. This PR should not merge as-is because the branch needs refresh, focused correctness fixes, and real behavior proof.

## What This Changes

The PR expands Control UI Sessions and Chat to request and render richer session metadata including derived titles, previews, channel/room context, Guardian state, delivery route, cost, token breakdown, lineage, model, runtime, and status badges.

## Best Possible Solution

Port the useful metadata display onto current main, integrate with the existing Status/Runtime columns and context-notice helper, fix the token and duration details, and add focused UI coverage.

## Reproduction Assessment

Yes for the PR blockers: source inspection shows the input/output fallback remains behind the stale-total guard, and duration labels use rounding that overstates elapsed time. The remaining feature gap on main is also source-visible because Control UI does not request or type the richer session metadata fields.

## Solution Assessment

Partly. Surfacing existing gateway session metadata in Control UI is the right direction, but this branch should be refreshed or replaced on current main and aligned with the newer Status/Runtime UI before merge.
