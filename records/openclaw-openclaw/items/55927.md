## Summary

Keep this PR open. Current main already exposes the richer gateway session-row metadata, but the Control UI still does not request or render most of it, so the PR is not obsolete; however, the branch needs a rebase and focused fixes before it is mergeable.

## What This Changes

This PR extends the Control UI sessions and chat views to request and render room, channel, title, delivery, token, cost, lineage, status, and policy metadata from gateway session rows.

## Best Possible Solution

Keep this PR as the implementation candidate, rebase it onto current main, move the token fallback into the current context-notice helper, request the derived session fields from the controller, render the gateway-backed metadata in Sessions and Chat, fix the duration math, and remove unrelated formatting-only churn before targeted UI validation.

## Reproduction Assessment

Yes. Current-main code inspection gives a high-confidence reproduction path: `sessions.list` requests omit the derived-field flags, the UI row type and renderers do not expose the richer gateway fields, and `getContextNoticeViewModel` still returns null for high `inputTokens` when `totalTokens` is absent.

## Solution Assessment

No, not as-is. The product direction is right because the backend contract already exposes the data, but the branch needs to be rebased onto current main and fixed in the current helper/test structure before it is the narrowest maintainable solution.
