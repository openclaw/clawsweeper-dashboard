## Summary

Keep open: current main still does not consume the richer gateway session metadata in Control UI, so the central feature remains useful, but this external PR is not merge-ready because it lacks real behavior proof, conflicts with newer Sessions UI work, and still has source-visible token and duration defects.

## What This Changes

The branch extends Control UI Sessions and Chat to request and render richer gateway session metadata, including room/channel/title/preview, Guardian policy, delivery, cost, token, lineage, runtime, and status context, plus focused UI tests.

## Best Possible Solution

Refresh or replace the branch on current main, preserve the existing Status and Runtime columns, port the useful metadata display into the current Control UI structure, fix the token and duration defects, and attach redacted Control UI proof.

## Reproduction Assessment

Yes for the PR blockers: PR-head source shows the context notice fallback is still gated behind totalTokensFresh=false, and the duration label uses rounding that can overstate elapsed time. The feature gap on current main is also source-visible because Control UI does not request or type the richer session metadata.

## Solution Assessment

Partly. Surfacing existing gateway session metadata in Control UI is a reasonable direction, but this branch should be refreshed against current main, the two correctness defects should be fixed, and real behavior proof should be attached before merge.
