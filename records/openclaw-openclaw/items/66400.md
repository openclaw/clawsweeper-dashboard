## What This Changes

The PR pre-registers same-session agent run IDs in the TUI agent-event handler before filtering and adds optional `sessionKey` to the TUI `AgentEvent` type.

## Best Possible Solution

Land this PR or an equivalent TUI fix after adding an Unreleased Fixes changelog entry and handler tests for tool-first same-session adoption, cross-session rejection, BTW guards, and optimistic-message bookkeeping.

## Reproduction Assessment

Yes. On current main, a focused handler check can call `handleAgentEvent` with a same-session tool `start` event before any `handleChatEvent`; it returns at the known-run filter before `chatLog.startTool`, matching the reported missing live `/verbose full` tool card.

## Solution Assessment

Yes for the code direction. Same-session pre-registration is the narrowest maintainable fix because it uses gateway-provided `sessionKey` and the latest diff mirrors `handleChatEvent`'s BTW and optimistic-message guards; merge readiness still needs changelog and regression coverage.
