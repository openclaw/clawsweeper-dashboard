## Review Findings

Overall correctness: patch is correct

Overall confidence: 0.84

Full review comments:

- **[P1] Only consume optimistic state for the submitted run:** `src/tui/tui-event-handlers.ts:335-340`
  - body: This block clears `pendingOptimisticUserMessage` for any untracked event whose `sessionKey` matches. The existing chat path deliberately only consumes that state when the event is for `pendingChatRunId` or a known local run; otherwise an unrelated same-session run that happens to emit first can become `activeChatRunId`, clear the pending flag, and stop `sendMessage` from recording the real submitted run as pending when `sendChat` returns.
  - confidence: 0.83
- **[P1] Carry agentId before adopting global agent events:** `src/tui/tui-types.ts:53`
  - body: Gateway agent payloads already include `agentId`, and `handleChatEvent`/`handleBtwEvent` use it to reject `global` events from non-selected agents. Adding only `sessionKey` means a TUI attached to `global` can adopt another agent's early tool event as the active run before any chat event has a chance to apply the selected-agent filter.
  - confidence: 0.78
