## Summary

Keep open: the Feishu catchup direction is useful, but this branch is not mergeable because the new module is standalone, not started by the monitor lifecycle, uses invalid current-main Feishu helpers/client/API assumptions, and lacks live recovery proof.

## What This Changes

The PR adds a standalone Feishu message recovery module that polls chats every 30 seconds and tries to replay unprocessed bot-mention messages.

## Best Possible Solution

Implement Feishu missed-message catchup inside the monitor lifecycle behind an explicit safe policy/config surface, using current SDK APIs, structured mentions, shared dedupe, shared queueing, and live redacted Feishu proof.

## Reproduction Assessment

Yes. Static source review against current main gives a high-confidence reproduction path for the PR defects: the module imports unavailable helpers, is never started, expects account.client, and calls SDK methods/fields that do not match current Feishu contracts.

## Solution Assessment

No. The feature direction is useful, but a standalone interval scanner that bypasses the Feishu monitor lifecycle, client factory, dedupe/queue contracts, shared chat history, and structured mention policy is not the narrow maintainable solution.
