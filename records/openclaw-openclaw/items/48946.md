## Summary

This PR should stay open but is not mergeable as-is: the proposed catchup feature still appears useful for Feishu, but the branch is unintegrated, incompatible with current Feishu contracts, and lacks live behavior proof.

## What This Changes

Adds a standalone Feishu message recovery module that polls Feishu chat history and replays missed bot-mentioned messages after downtime.

## Best Possible Solution

Rework Feishu catchup inside the monitor lifecycle behind an explicit opt-in config, using `createFeishuClient`, `client.im.chat.list`/`client.im.message.list`, structured mentions, existing dedupe and shared history, bounded cursors, and redacted live Feishu proof.

## Reproduction Assessment

Yes for the PR defects: source inspection shows the module is never started, imports helpers that current `dedup.ts` does not export, assumes a nonexistent `account.client`, and calls SDK methods absent from the pinned Lark SDK. No live Feishu tenant proof was established for the original missed-message symptom.

## Solution Assessment

No. A standalone interval scanner that bypasses monitor lifecycle, shared state, queueing, structured mention policy, and opt-in privacy controls is not the best maintainable solution.
