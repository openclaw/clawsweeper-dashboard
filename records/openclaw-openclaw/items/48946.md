## Summary

Keep open: the Feishu catchup capability is still not implemented on current main, but this external PR is not mergeable because the added module is standalone, conflicts with current Feishu/Lark contracts, and lacks required live behavior proof.

## What This Changes

The branch adds a standalone Feishu message recovery module that polls chats every 30 seconds, scans recent messages, and tries to replay missed bot-mention messages.

## Best Possible Solution

Implement missed-message catchup inside the Feishu monitor lifecycle using SDK chat/message list APIs, structured mention filtering, shared dedupe, sequential queueing, and shared chat history, then validate it with redacted live Feishu recovery proof.

## Reproduction Assessment

Yes. Static source review of the PR diff plus current main contracts gives a high-confidence reproduction path for the PR defects: it imports non-exported helpers, never starts, expects account.client, and calls SDK methods/fields that do not exist.

## Solution Assessment

No. The feature direction is useful, but a standalone scanner that bypasses the Feishu monitor lifecycle, client factory, dedupe/queue contracts, shared chat history, and structured mention policy is not the narrow maintainable solution.
