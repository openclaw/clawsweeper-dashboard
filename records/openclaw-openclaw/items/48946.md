## Summary

Keep this PR open, but it is not merge-ready: the Feishu catch-up idea is useful, while the current branch has no real tenant proof and the added module is standalone, unwired, and inconsistent with current Feishu monitor, dedupe, and pinned Lark SDK contracts.

## What This Changes

The branch adds a new `extensions/feishu/src/feishu-message-recovery.ts` module that polls Feishu chats every 30 seconds and tries to replay missed bot-mentioned messages.

## Best Possible Solution

Design Feishu catch-up as an opt-in monitor-lifecycle feature using current SDK list APIs, persistent cursors/dedupe, structured mentions, shared queue/history state, focused tests, and redacted live proof.

## Reproduction Assessment

Yes for the PR defects: source inspection plus the pinned SDK type contract show the lifecycle, import, API, field-path, and shared-state problems. No live Feishu tenant proof establishes the original offline missed-message symptom or the proposed recovery behavior.

## Solution Assessment

No. A standalone interval scanner that is not wired into the monitor and bypasses existing queue, dedupe, structured mention, shared history, and privacy controls is not the maintainable solution.
