- `P2`: This is a normal-priority Feishu reliability improvement with channel-limited scope, not an urgent core runtime outage.

## Summary

Keep open: current main still does not implement Feishu missed-message catchup, but this PR is not merge-ready because the added recovery module is standalone, unwired, SDK-incompatible, and unsafe around message filtering/state.

## What This Changes

The PR adds `extensions/feishu/src/feishu-message-recovery.ts`, a periodic Feishu chat-history polling module intended to recover bot-mentioned messages missed while the gateway was offline.

## Best Possible Solution

Design Feishu catch-up as an opt-in monitor-lifecycle feature using current Lark SDK APIs, structured mentions, persistent cursors/dedupe, shared dispatch state, focused tests, and redacted real-tenant proof.

## Reproduction Assessment

Yes for the PR defects: source inspection plus the pinned SDK contract show the lifecycle, import, API, field-path, and shared-state problems. No live Feishu tenant proof establishes the original offline missed-message symptom or this branch's proposed recovery behavior.

## Solution Assessment

No. A standalone interval scanner that is not wired into the monitor and bypasses existing queue, dedupe, structured mention, shared history, and privacy controls is not the maintainable solution.
