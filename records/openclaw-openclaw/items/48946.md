- `P2`: This is a normal-priority Feishu reliability improvement with limited channel scope, not an urgent core runtime regression.

## Summary

Keep this PR open but not merge-ready: the Feishu catchup goal is useful and still not implemented on current main, but this branch is a standalone scanner with missing proof, lifecycle gaps, SDK-contract errors, and privacy/session-state risks.

## What This Changes

The PR adds `extensions/feishu/src/feishu-message-recovery.ts`, a Feishu module that polls recent chats and tries to replay missed bot-directed messages.

## Best Possible Solution

Design Feishu catchup as an opt-in monitor-lifecycle feature that uses the current client factory, SDK list APIs, structured mentions, persistent dedupe, shared queue/history state, bounded cursors, tests, and redacted live proof.

## Reproduction Assessment

Yes for the PR defects: source inspection and the pinned SDK contract show the lifecycle, import, client/API, field-path, and shared-state problems. No live Feishu tenant proof establishes the original offline missed-message symptom or the proposed recovery behavior.

## Solution Assessment

No. A standalone interval scanner that is not wired into the monitor and bypasses existing queue, dedupe, structured mention, shared history, and privacy controls is not the maintainable solution.
