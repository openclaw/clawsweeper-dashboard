## Summary

Keep this PR open for maintainer review: the Feishu missed-message recovery goal is useful, but the current patch is a standalone module that is not wired into current main and conflicts with current Feishu lifecycle, dedupe, client, SDK, and message-shape contracts.

## What This Changes

The PR adds a new Feishu plugin message-recovery module that polls recent chats every 30 seconds and tries to replay missed bot-mention messages after offline periods.

## Best Possible Solution

Implement Feishu missed-message catchup inside the Feishu monitor lifecycle using shared inbound handler contracts and explicit recovery policy, preserving dedupe, structured mentions, group access, queueing, and shared chat history.

## Reproduction Assessment

Yes. Static source review gives a high-confidence reproduction path for the PR defects: the added file imports non-exported helpers, is never started from current monitor lifecycle, reads a non-existent `account.client`, and uses SDK/message shapes that conflict with current main and the Lark SDK types.

## Solution Assessment

No. The feature direction is useful, but this patch is not the narrow maintainable solution because it bypasses current lifecycle, client factory, structured mention policy, dedupe claims, sequential queueing, and shared conversation history.

  - body: `resolveFeishuAccount` returns resolved credentials/config, not an object with `account.client`; casting `account.client as Lark.IM` leaves recovery without a usable SDK client. Follow the existing `createFeishuClient(account)` path.
  - confidence: 0.96

  - confidence: 0.94
- **[P2] Filter recovered messages with structured mentions:** `extensions/feishu/src/feishu-message-recovery.ts:132-135`
  - body: The raw `content.includes("@")` fallback treats emails, pasted handles, and broadcast mentions as bot-directed recovery candidates. Use the structured `mentions` data and existing bot-mention rules to avoid replaying unrelated group messages.
  - confidence: 0.86

  - confidence: 0.92

  - confidence: 0.89
