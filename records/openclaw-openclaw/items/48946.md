## Summary

Keep open: the Feishu catchup goal is valid, but this PR is not merge-ready because it lacks real Feishu reconnect proof and the patch has blocking lifecycle, import, SDK-contract, state, and privacy defects.

## What This Changes

The PR adds a standalone Feishu message recovery module that polls recent chats every 30 seconds and tries to replay unprocessed bot-directed messages.

## Best Possible Solution

Design Feishu catchup as an opt-in monitor-lifecycle feature that uses the current client factory, SDK list APIs, structured mentions, persistent cursors/dedupe, shared dispatch state, focused tests, and redacted live proof.

## Reproduction Assessment

Yes for the PR defects: source inspection and the pinned SDK contract show the lifecycle, import, API, field-path, and shared-state problems. No live Feishu tenant proof establishes the original offline missed-message symptom or the proposed recovery behavior.

## Solution Assessment

No. A standalone interval scanner that is not wired into the monitor and bypasses existing queue, dedupe, structured mention, shared history, and privacy controls is not the maintainable solution.
