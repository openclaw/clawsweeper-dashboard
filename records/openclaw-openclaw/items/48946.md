## Summary

Keep open for maintainer review: the Feishu catchup goal is useful, but this PR is not merge-ready because the added module is not wired into the monitor lifecycle, does not match current Feishu exports or the pinned Lark SDK shape, and can replay unrelated group history through the agent path.

## What This Changes

The branch adds a Feishu `feishu-message-recovery.ts` module that polls recent chats every 30 seconds and attempts to replay missed bot-mentioned messages.

## Best Possible Solution

Design Feishu catchup as an opt-in monitor-lifecycle feature using the current client factory, SDK list APIs, structured mentions, persistent cursors/dedupe, shared dispatch state, focused tests, and redacted live proof.

## Reproduction Assessment

Yes for the PR defects: source inspection plus the pinned SDK contract show the lifecycle, import, API, field-path, and shared-state problems. No live Feishu tenant proof establishes the original offline missed-message symptom or the proposed recovery behavior.

## Solution Assessment

No. A standalone interval scanner that is not wired into the monitor and bypasses existing queue, dedupe, structured mention, shared history, and privacy controls is not the maintainable solution.
