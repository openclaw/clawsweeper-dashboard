Full review comments:

- **[P1] Treat accepted retry-key 409s as delivered:** `extensions/line/src/retry.ts:60-62`
  - body: LINE's retry-key contract returns 409 when the same retry-key request was already accepted; this code classifies every 4xx as failure, so a push that succeeded before a lost response can still bubble as failed and later be retried with a new key. Handle that accepted response as delivery success or a distinct delivered result. ([developers.line.biz](https://developers.line.biz/en/reference/messaging-api/nojs/))
  - confidence: 0.91
- **[P2] Fix the retry body narrowing lint error:** `extensions/line/src/retry.ts:42`
  - body: Live `check-lint` and `check-additional-extension-bundled` still fail on this cast with `typescript(no-unnecessary-type-conversion)`, so the current head cannot merge until the unnecessary conversion is removed or the body is narrowed differently.
  - confidence: 0.98
- **[P2] Reuse the existing LINE loading keepalive:** `extensions/line/src/monitor.ts:255-262`
  - body: The outer keepalive already starts before `inbound.run` and stops in the outer `finally`; starting another keepalive inside delivery doubles loading-animation calls during long replies. Transfer or reuse the existing keepalive instead of running both intervals.
  - confidence: 0.9
- **[P2] Split the status fallback changes out of this LINE PR:** `src/status/status-text.ts:510-514`
