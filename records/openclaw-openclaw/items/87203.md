## Summary

Current main already covers both reported failure modes: managed inbound media refs are hydrated before non-sandbox workspace path resolution, and date-only daily memory reads now return optional not-found context instead of surfacing ENOENT.

## What This Changes

The issue requests workspace-independent handling for `media://inbound/*` refs on non-sandbox agents with custom workspaces, plus graceful handling for missing daily memory files.

## Best Possible Solution

Close this as fixed on current main and let the next release carry the media URI and daily-memory read behavior to users.

## Reproduction Assessment

Yes. Source inspection of the latest release shows non-sandbox native image refs fell through to workspace-relative resolution, while current main adds the pre-workspace media URI hydration and focused tests for the scenario.

## Solution Assessment

Yes. The current main fix is the narrowest maintainable direction: resolve managed media refs through the canonical media-reference helper before workspace path logic, and treat missing date-only memory reads as optional context.
