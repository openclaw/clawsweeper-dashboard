## Summary

Keep open. Current main still computes `sessions.usage` totals from the limited page, while this mergeable PR applies the narrow source fix, adds regression coverage, and has no blocking review findings.

## What This Changes

The PR changes `sessions.usage` so aggregate totals and breakdowns are computed across all matching sessions while `limit` only caps the returned `sessions` rows, with a regression test and changelog entry.

## Best Possible Solution

Land this PR or an equivalent gateway fix that keeps `limit` scoped to returned session rows while aggregating all matching sessions, with the regression test and changelog entry preserved.

## Reproduction Assessment

Yes. Current main applies `limit` before the aggregation loop, so any date range with more matching sessions than the requested limit undercounts totals and aggregate breakdowns; the PR test models this with two sessions and `limit: 1`.

## Solution Assessment

Yes. The PR uses the narrow maintainable fix: aggregate over `mergedEntries` and use the limited-key set only to decide which per-session rows to return.
