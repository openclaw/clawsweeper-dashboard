## Label Justifications

- `P2`: This is a current source-backed reliability bug with meaningful but intermittent impact limited to particular provider-error outcomes and delivery paths.
- `impact:message-loss`: The reported terminal outcome can suppress both the assistant response and a user-visible failure notification.
- `impact:auth-provider`: The affected result can bypass the configured model fallback chain after a provider stream error.

## Summary

Current main still leaves the reported gap: an errored assistant turn with hidden thinking content or positive output usage bypasses silent-error retry, while reasoning-only retry explicitly excludes error turns. The broader fallback and isolated-delivery behavior crosses intentional safety and delivery-policy boundaries, so this needs maintainer direction rather than closure or an automatic patch.

## What This Changes

Fix provider-error turns containing only hidden reasoning so multi-step tasks retry, fail over, or surface a safe visible failure instead of silently ending.

## Best Possible Solution

Define a shared invisible-provider-error outcome that distinguishes hidden reasoning from user-visible output, permits bounded replay only when replay metadata proves no side effects or delivery, feeds exhausted outcomes into the existing fallback classifier, and delivers only a sanitized generic failure on isolated channel paths.

## Reproduction Assessment

Yes at source level: current main and its focused test deterministically prove that positive output or a thinking content block excludes an error turn from silent-error retry, while the sibling reasoning-only path excludes error stops. This read-only review did not inject a live Vertex stream to reproduce the final isolated-channel suppression.

## Solution Assessment

No, merely deleting the output-token and content-length guards is not the safest complete fix. The narrow maintainable solution should use explicit visible-output and replay-safety semantics, then align fallback and sanitized terminal delivery with that result.
