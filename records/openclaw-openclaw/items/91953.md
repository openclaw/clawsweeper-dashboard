## Summary

Keep this open: current main and the latest release still have the retry gap, and the open linked fix PR explicitly owns the implementation path but has not merged yet.

## What This Changes

The issue requests retry or visible recovery for replay-safe `stopReason="error"` turns that emit only thinking/non-visible output before failing.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/92191, or an equivalent bounded runner fix that retries replay-safe non-visible `stopReason="error"` turns before failover while preserving side-effect and auth/rate-limit/billing guards.

## Reproduction Assessment

Yes at source level: current main has guards that skip both retry paths for `stopReason="error"` plus thinking/non-zero output, and an existing test asserts no retry for errored reasoning-only turns. I did not run the intermittent live provider reproduction.

## Solution Assessment

Yes, the best direction is a narrow runner fix that treats thinking-only errored output as non-visible progress only when replay metadata proves the attempt is safe. The open linked PR appears to target that boundary with focused unit and QA coverage, so the issue should remain open until it merges or is rejected.
