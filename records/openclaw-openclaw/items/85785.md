## Summary

Keep open: this is a focused fix for a source-reproducible plugin gateway timeout, but the current PR still fails CI and changes plugin gateway return semantics without updating the public plugin-facing contract.

## What This Changes

The PR wraps plugin-registered gateway handlers so non-undefined return values are converted into gateway RPC success responses, adds loader regression tests, and adds a changelog entry.

## Best Possible Solution

Land the registry adapter only after CI is green, the plugin-facing handler type/docs describe return-value completion and respond precedence, and redacted live gateway-call proof shows the timeout is fixed.

## Reproduction Assessment

Yes, source-reproducible. On current main, a plugin gateway method that returns a payload without calling respond is stored raw, and the WebSocket response path only emits frames through respond, so the CLI can wait until its timeout.

## Solution Assessment

No, not yet. The registry adapter is the right narrow boundary, but this PR still needs typed test fixes, SDK/docs contract alignment, and real gateway-call proof.
