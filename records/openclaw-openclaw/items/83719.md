## Best Possible Solution

Keep the report-only runner, but make its default path share the existing memory-core dreaming day/timezone formatter and add coverage before merge.

Yes. Source inspection shows the new helper formats paths with UTC `toISOString()`, while current memory-core paths and tests use `formatMemoryDreamingDay(..., timezone)`.

No. The report-only boundary is the right shape, but the default path should reuse the existing dreaming day-bucketing contract before merge.
