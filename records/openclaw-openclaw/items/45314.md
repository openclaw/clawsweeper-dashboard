Keep open: current main and the latest release still let fast-abort replies reach prefix normalization before model/thinking context is populated, and the formally linked fixing PR remains open rather than merged.

## What This Changes

The issue reports that early abort replies send unresolved `responsePrefix` template tokens such as `{model}` and `{thinkingLevel}` instead of populated values.

## Best Possible Solution

Repair and land https://github.com/openclaw/openclaw/pull/45315, or close that PR and keep this issue as the canonical bug until a narrower active-state fix lands.

## Reproduction Assessment

Yes, at source level. Current main sends fast-abort replies before `onModelSelected` populates dynamic prefix context, and normalization can preserve unresolved template variables.

## Solution Assessment

No, the currently linked default-seeding PR is only a partial path. The best fix should carry canonical active session/channel/model/thinking facts into fast-abort prefix rendering, with a deliberate fallback when no active facts exist.
