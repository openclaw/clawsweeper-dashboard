## Summary

Keep open. Current main still has a source-reproducible path where early abort and routed replies can apply a templated `responsePrefix` before model/thinking context is populated, and the same reporter has an open paired PR with closing syntax at https://github.com/openclaw/openclaw/pull/45315.

## What This Changes

The issue reports that early abort replies can send unresolved `responsePrefix` template variables such as `{model}` and `{thinkingLevel}` instead of interpolated values.

## Best Possible Solution

Thread a populated response-prefix context into fast abort and direct routed reply paths, preferably from active session/runtime metadata, then land focused regression coverage through the existing paired PR or a narrow replacement.

## Reproduction Assessment

Yes, at source level. Configure `messages.responsePrefix` with `{model}` or `{thinkingLevel}` and hit the fast-abort or routed reply path before `onModelSelected`; current main can normalize with missing prefix context values and preserve the raw tokens.

## Solution Assessment

No current-main fix is present. The best fix is a narrow context plumbing change for early abort and routed replies, not a new config option or broad template-policy change.
