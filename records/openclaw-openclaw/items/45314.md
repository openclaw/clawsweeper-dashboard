## What This Changes

The issue reports unresolved `{model}` and `{thinkingLevel}` placeholders in early abort replies when `messages.responsePrefix` uses template variables.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/45315 or a focused replacement that carries active session/channel/runtime model and thinking context into early abort and routed reply normalization, with regression tests for templated abort prefixes.

## Reproduction Assessment

Yes, at source level. Configure `messages.responsePrefix` with `{model}` or `{thinkingLevel}`, trigger `stop` or `/stop` before `onModelSelected` runs, and the fast-abort/routed reply path can normalize the abort reply with missing context while the resolver preserves placeholders.

## Solution Assessment

Not applicable as a patch verdict for this issue. The maintainable fix is to provide active prefix context to early abort and routed reply normalization, not to drop prefixes or rely only on raw config defaults.
