## Summary

Keep open. Current main still has the fast-abort response-prefix context gap, and open closing PRs already target this issue, so it should stay open until one fix lands or the PRs are explicitly superseded.

## What This Changes

Stop and early-abort replies should interpolate configured `responsePrefix` template variables instead of sending literal placeholders.

## Best Possible Solution

Land or supersede the linked PRs with one narrow fix that derives safe runtime prefix context for fast abort and routed replies, preserves static prefixes, and keeps the normal `onModelSelected` override path intact.

## Reproduction Assessment

Yes. Configure `messages.responsePrefix` with `{model}` or `{thinkingLevel}` and send `stop` or `/stop` during an active turn; current source shows the abort payload is emitted before model-selection context is populated, and missing values remain literal.

## Solution Assessment

Unclear which open PR is the best final patch without maintainer review. The safer direction is to use actual session/runtime model metadata where available and pass prefix context into routed delivery instead of relying only on raw config defaults.
