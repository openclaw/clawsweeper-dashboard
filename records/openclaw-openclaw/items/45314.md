## Summary

Keep open. Current main still has a source-reproducible gap where early abort/direct-route replies can apply a templated `responsePrefix` without a populated prefix context, and an open same-author PR already explicitly tracks the fix.

## What This Changes

The issue reports that early stop/abort replies leak raw `responsePrefix` template variables such as `{model}` and `{thinkingLevel}` instead of sending populated values.

## Best Possible Solution

Thread a populated response-prefix context into early abort and direct route reply paths, preferably from active session/runtime metadata, and land focused regression coverage through the open fix PR or a narrow replacement.

## Reproduction Assessment

Yes, at source level. Configure a `messages.responsePrefix` with `{model}` or `{thinkingLevel}` and hit the fast-abort/direct-route path before `onModelSelected`; current main can normalize with the raw prefix and no template context.

## Solution Assessment

No current-main fix is present. The narrow maintainable direction is to preserve useful prefixes while supplying session/runtime prefix context for early aborts, then review or replace the existing conflicting PR.
