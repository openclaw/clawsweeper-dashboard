## Summary

Keep open. Current main and v2026.5.7 still have a source-reproducible fast-abort path where templated response prefixes can be normalized without populated model/thinking context, and the reporter has an open paired fix PR with a closing reference.

## What This Changes

Populate responsePrefix template variables such as {model} and {thinkingLevel} for early stop or abort replies instead of sending raw placeholders.

## Best Possible Solution

Land a focused fix that supplies a populated, session/runtime-aware ResponsePrefixContext to both direct dispatcher delivery and routed fast-abort delivery, with regression tests for both paths.

## Reproduction Assessment

Yes, at source level. Configure messages.responsePrefix with {model} or {thinkingLevel}, start a run, then send stop or /stop before onModelSelected; current main can normalize the abort reply with a raw templated prefix and missing context.

## Solution Assessment

No current-main fix is present. The open PR's config-default preseed is a useful start, but the maintainable fix should also carry prefix context into routed reply normalization so all abort delivery paths resolve templates.
