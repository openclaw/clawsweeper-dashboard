## What This Changes

The issue reports that early abort replies leak unresolved `responsePrefix` template variables such as `{model}` and `{thinkingLevel}`.

## Best Possible Solution

Land a focused fix, likely through https://github.com/openclaw/openclaw/pull/45315 or a replacement, that carries populated runtime/session-aware prefix context into both routed fast-abort delivery and dispatcher fallback, with regression coverage.

## Reproduction Assessment

Yes, at source level. Configure `messages.responsePrefix` with `{model}` or `{thinkingLevel}`, trigger `stop` or `/stop` before `onModelSelected` runs, and current main can normalize the abort reply without model/thinking context.

## Solution Assessment

No current-main solution is present. The narrow maintainable fix is to pass a populated `ResponsePrefixContext` through the fast-abort routed path and the dispatcher fallback rather than only changing template fallback behavior.
