## Summary

Keep open. Current main and the latest release still have a source-reproducible path where fast abort or routed replies can apply a templated response prefix without a populated prefix context, and the same reporter has an open paired fix at https://github.com/openclaw/openclaw/pull/45315 with a closing reference.

## What This Changes

Early abort replies should interpolate responsePrefix template variables such as {model} and {thinkingLevel} instead of sending them literally.

## Best Possible Solution

Thread a populated response-prefix context into fast abort and direct routed reply paths, preferably from active session/runtime metadata, then land focused regression coverage through https://github.com/openclaw/openclaw/pull/45315 or a narrow replacement.

## Reproduction Assessment

Yes, at source level. Configure messages.responsePrefix with {model} or {thinkingLevel} and hit the fast-abort or routed reply path before onModelSelected; current main can normalize with the raw prefix and no template context.

## Solution Assessment

No current-main fix is present. The narrow maintainable solution is to preserve useful prefixes while supplying session/runtime prefix context for early aborts, then review or replace the existing paired fix.
