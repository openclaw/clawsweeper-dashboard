## What This Changes

Abort/stop replies should interpolate `messages.responsePrefix` template variables instead of sending raw `{model}` or `{thinkingLevel}` placeholders.

## Best Possible Solution

Land or replace the linked PR with a fix that derives abort-prefix context from active session/channel model state and carries that context through routed delivery, then close this issue from the merged fix.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: configure a templated `messages.responsePrefix`, trigger `stop` or `/stop` while a run is active, and the fast-abort reply can be normalized before model context is populated.
