## What This Changes

The issue requests that early stop or abort replies populate responsePrefix template variables such as {model} and {thinkingLevel} instead of sending raw placeholders.

## Best Possible Solution

Repair and land https://github.com/openclaw/openclaw/pull/45315, or close that PR and keep this issue as the canonical bug until a narrower active-state fix lands.

## Reproduction Assessment

Yes, at source level. Current main sends fast-abort replies before onModelSelected populates dynamic prefix context, and the issue includes WhatsApp screenshots showing literal placeholders.

## Solution Assessment

No, current main is not the best solution because it still permits raw placeholders. The best fix should carry effective active model/provider/thinking facts into fast-abort prefix rendering, with a deliberate fallback when no active facts exist.
