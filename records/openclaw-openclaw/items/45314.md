## Summary

Keep open because current main still has a source-level path where early abort replies can apply a templated response prefix without model/thinking context, and the same reporter has an open closing PR at https://github.com/openclaw/openclaw/pull/45315.

## What This Changes

Early stop or abort replies should resolve `messages.responsePrefix` template variables instead of sending raw `{model}` or `{thinkingLevel}` text.

## Best Possible Solution

Review, repair, and land the linked PR or a focused replacement that supplies populated response-prefix context to both dispatcher fallback and routed fast-abort delivery, with regression coverage for abort replies and normal `onModelSelected` override behavior.

## Reproduction Assessment

Yes, at source level. Configure `messages.responsePrefix` with `{model}` or `{thinkingLevel}`, trigger `stop` or `/stop` before `onModelSelected` runs, and current main can normalize the abort reply without model/thinking context.

## Solution Assessment

Not applicable as a patch verdict for this issue. The maintainable fix is to carry populated prefix context through early abort routing rather than dropping prefixes or relying only on raw config defaults.
