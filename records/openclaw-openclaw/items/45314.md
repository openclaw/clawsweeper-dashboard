## Summary

Keep open: current main still has a source-level path where early abort replies can apply a templated responsePrefix without model/thinking context, and the active closing PR is still open and conflicting.

## What This Changes

The issue reports that early stop/abort replies send literal responsePrefix template variables such as `{model}` and `{thinkingLevel}` instead of populated values.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/45315 or a focused replacement that supplies active provider/model/thinking reply-prefix context to early abort and routed delivery, with regression coverage for session overrides and normal `onModelSelected` override behavior.

## Reproduction Assessment

Yes, at source level. Configure `messages.responsePrefix` with `{model}` or `{thinkingLevel}`, trigger `stop` or `/stop` before `onModelSelected` runs, and the fast-abort path can normalize the abort reply with no model/thinking context while the resolver preserves unresolved placeholders.

## Solution Assessment

Not applicable as a patch verdict for this issue. The maintainable fix is to carry populated prefix context from session/runtime state into early abort and routed reply normalization, rather than dropping prefixes or relying on raw config strings.
