- `impact:session-state`: The bug is caused by missing model/thinking runtime context when an early abort reply is normalized.

## Summary

Keep open: current main still has a source-level path where early abort replies can apply a templated `responsePrefix` without model/thinking context, and https://github.com/openclaw/openclaw/pull/45315 remains the active closing PR for this issue.

## What This Changes

The issue asks for early stop/abort replies to interpolate `responsePrefix` variables like `{model}` and `{thinkingLevel}` instead of sending raw placeholders.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/45315 or a focused replacement that carries populated reply-prefix context through both dispatcher fallback and routed fast-abort delivery, with regression coverage for abort replies and normal `onModelSelected` override behavior.

## Reproduction Assessment

Yes, at source level. Configure `messages.responsePrefix` with `{model}` or `{thinkingLevel}`, trigger `stop` or `/stop` before `onModelSelected` runs, and current routed fast-abort code can normalize the abort reply without model/thinking context.

## Solution Assessment

Not applicable as a patch verdict for this issue. The maintainable fix is to carry populated prefix context into early abort routing rather than dropping prefixes or relying only on raw config defaults.
