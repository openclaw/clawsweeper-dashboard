## Summary

Keep open. Current main still has a source-level fast-abort path where templated response prefixes can be normalized without model/thinking context, and the same reporter has an open closing PR at https://github.com/openclaw/openclaw/pull/45315.

## What This Changes

The issue reports that templated `messages.responsePrefix` values such as `{model}` and `{thinkingLevel}` are sent literally in early abort replies like `stop` or `/stop`.

## Best Possible Solution

Land or replace the linked PR with a focused fix that supplies populated session/runtime `ResponsePrefixContext` to dispatcher fallback and routed fast-abort delivery, with regression coverage for abort replies and normal `onModelSelected` override behavior.

## Reproduction Assessment

Yes, at source level. Configure `messages.responsePrefix` with `{model}` or `{thinkingLevel}`, trigger `stop` or `/stop` before `onModelSelected` runs, and current main can normalize the abort reply without model/thinking context.

## Solution Assessment

No current-main solution is present. The narrow maintainable path is to carry a populated prefix context through early abort routing and dispatcher fallback rather than only changing template fallback behavior.
