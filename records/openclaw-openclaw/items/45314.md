## Summary

Keep open. Current main still has a source-reproducible fast-abort path where a templated response prefix can be normalized without model/thinking context, and the reporter has an open paired PR with closing syntax at https://github.com/openclaw/openclaw/pull/45315.

## What This Changes

The issue reports that early `stop` or `/stop` abort replies can show literal `{model}` and `{thinkingLevel}` tokens in `messages.responsePrefix` instead of resolved values.

## Best Possible Solution

Land or replace the linked fix PR with a focused change that derives runtime/session-aware prefix context and passes it through both direct and routed fast-abort delivery with regression tests.

## Reproduction Assessment

Yes, at source level. Configure `messages.responsePrefix` with `{model}` or `{thinkingLevel}`, start a run, then send `stop` or `/stop` before `onModelSelected` runs; current main can normalize that abort reply without the needed model/thinking context.

## Solution Assessment

No complete current-main solution is present. The maintainable fix should carry populated prefix context into routed delivery as well as direct dispatcher delivery, preferably using runtime/session metadata where available.
