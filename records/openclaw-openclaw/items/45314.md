## Summary

Keep open: current main and the latest release still have the source-level path where an early abort can normalize a templated responsePrefix before model/thinking context is available, and the open linked PR remains the active implementation candidate.

## What This Changes

The issue asks for early abort replies such as `stop` or `/stop` to populate responsePrefix template variables instead of sending raw `{model}` and `{thinkingLevel}` placeholders.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/45315 or a focused replacement that carries active session/channel/runtime model and thinking context into early abort and routed reply normalization, with regression tests for templated abort prefixes.

## Reproduction Assessment

Yes, at source level. Configure `messages.responsePrefix` with `{model}` or `{thinkingLevel}`, trigger `stop` or `/stop` before `onModelSelected` runs, and the fast-abort/routed reply path can normalize the abort reply with missing context while the resolver preserves placeholders.

## Solution Assessment

Not applicable as a patch verdict for this issue. The maintainable fix is to provide populated active prefix context to early abort and routed reply normalization, not to drop prefixes or rely only on raw config defaults.
