## Summary

Keep open. Current main still has the fast-abort prefix context gap, and open closing PRs already target this issue, so maintainers should review or supersede those instead of closing this issue or queueing a duplicate repair.

## What This Changes

The issue reports that `messages.responsePrefix` template variables such as `{model}` and `{thinkingLevel}` leak literally in early abort replies from `stop` or `/stop`.

## Best Possible Solution

Land or supersede the linked PRs with one narrow fix that provides safe response-prefix context for fast abort and routed replies while preserving static prefixes and the normal `onModelSelected` override path.

## Reproduction Assessment

Yes. Configure `messages.responsePrefix` with `{model}` or `{thinkingLevel}` and send `stop` or `/stop` during an active turn; current source shows the abort payload can be emitted before `onModelSelected` fills model context, and unresolved known variables remain literal.

## Solution Assessment

Unclear which open PR is the best final patch without maintainer review. The safer direction appears to use actual session/runtime model metadata where available and pass prefix context into routed delivery instead of relying only on raw config defaults.
