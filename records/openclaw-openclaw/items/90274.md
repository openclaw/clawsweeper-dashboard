## What This Changes

The PR changes tool-definition wrapping to forward `parameters` through an enumerable getter and adds wrapper regression tests for throwing parameter getters.

## Best Possible Solution

Land the lazy-forwarding wrapper fix after normal maintainer review and changed-gate proof, then reconcile the overlapping fallback-schema PR so only one tool-wrapper behavior remains.

## Reproduction Assessment

Yes from source inspection: current main reads `definition.parameters` and `tool.parameters` during wrapper construction, so the throwing getters used in the PR tests would fail before quarantine. I did not run tests because this review was required to keep the checkout read-only.

## Solution Assessment

Yes. Deferring the schema read is the narrowest maintainable fix because it preserves the original unreadable schema for the existing quarantine/diagnostic path instead of advertising a fake empty schema.
