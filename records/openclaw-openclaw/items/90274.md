## Summary

Keep open: current main and the latest release still eagerly read wrapped tool `parameters`, while this maintainer draft preserves throwing schema getters for the downstream quarantine path with focused regression coverage and no blocking code-review findings.

## What This Changes

The PR adds a forwarded `parameters` accessor in the session tool-definition adapter and tests unreadable schema getters for both ToolDefinition-to-AgentTool and AgentTool-to-ToolDefinition wrapping.

## Best Possible Solution

Land this schema-preserving adapter after final-head maintainer validation, leaving broader descriptor/name hardening to the related drafts if maintainers still want that scope.

## Reproduction Assessment

Yes. Current main and `v2026.6.8` directly read `parameters` while constructing wrappers, so a throwing getter can abort before the existing schema quarantine path sees it; this review verified the path from source rather than executing tests.

## Solution Assessment

Yes. Forwarding `parameters` through an accessor is the narrowest maintainable fix for this crash class because it avoids construction-time reads while preserving the later quarantine diagnostic boundary.
