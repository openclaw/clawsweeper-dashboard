## What This Changes

The issue asks for `agents.create` and device-token additions to avoid triggering a full Gateway restart that drops in-flight HTTP/WS calls.

## Best Possible Solution

Keep the current main behavior: agent-list writes and paired-device token mutations should remain hot/dynamic, while real `gateway.*` auth or exposure edits continue to require an explicit Gateway restart.
