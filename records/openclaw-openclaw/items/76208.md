## Summary

Current main already avoids the reported self-restart path: `agents.create` persists agent config, `agents.list` changes hot-reload heartbeat state instead of restarting the Gateway, and in-process writes are compared source-to-source so runtime-materialized `gateway.auth.token` does not become a phantom restart reason.

## What This Changes

The issue asks OpenClaw to stop `agents.create` from triggering a Gateway restart and brief port outage while follow-up RPC/HTTP calls are in flight.

## Best Possible Solution

Keep the current source-config reload planning and hot `agents.list` behavior; if this recurs, reopen with a current-main SHA and logs showing a non-`agents.list` restart reason.

## Reproduction Assessment

Yes. The report gives a concrete WS sequence and timestamped Gateway logs for the older `47009dd7` image, and the current-main source/test path maps that failure to reload planning that no longer restarts for agent-list/runtime-default diffs.

## Solution Assessment

Yes. Avoiding the restart at the reload-plan level is narrower than draining or socket handoff for routine agent creation, while the existing restart deferral remains the fallback for changes that truly require SIGUSR1.
