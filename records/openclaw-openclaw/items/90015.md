- `P1`: The issue describes a provider-controlled unbounded buffer in core agent streaming that can affect live gateway availability.
- `impact:crash-loop`: The reported failure mode is unbounded gateway memory growth that can crash or restart the gateway process.

## Summary

Keep open: current main and v2026.6.9 still have the unbounded opened-but-unclosed DeepSeek DSML recovery buffer, and the open linked PR is the candidate fix but has not merged.

## What This Changes

The issue asks for a 256 KB cap on `createDeepSeekDsmlToolCallRecoverer()` so unclosed DeepSeek DSML blocks cannot grow gateway memory without bound.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/86637 with a UTF-8 byte-counted DSML recoverer cap aligned with the sibling stream caps and focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows the DSML recoverer retains an opened block across chunks and only clears it when the close token arrives or final flush runs, with no size cap before then.

## Solution Assessment

Yes in direction. The existing recoverer is the right owner path and a sibling-aligned cap is the right fix shape, but the issue should stay open until the linked PR or a replacement actually lands.
