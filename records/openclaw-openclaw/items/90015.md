## Summary

Keep open: current main and v2026.6.8 still leave the DeepSeek DSML recovery buffer uncapped after an open token without a close token, and the open linked fix PR has not merged.

## What This Changes

The issue asks for a bounded DSML recovery buffer so unclosed DSML blocks cannot grow gateway memory indefinitely.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/86637 with a byte-counted DSML recoverer cap and focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible: after a DeepSeek DSML open token, current main appends to `buffer` across chunks and returns without clearing it when no close token is present. I did not run a synthetic stream because this review is read-only.

## Solution Assessment

Yes in direction: the narrow maintainable fix is a byte-counted cap in the existing DSML recoverer, aligned with sibling stream caps. The issue should stay open until the linked PR lands or a replacement provides that fix.
