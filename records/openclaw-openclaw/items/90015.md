## What This Changes

The issue requests a bounded DeepSeek DSML recovery buffer that discards an unclosed oversized DSML block as plain text instead of retaining it indefinitely.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/86637 with a UTF-8 byte-counted DSML recoverer cap aligned with sibling stream caps and focused regression coverage.

## Reproduction Assessment

Yes. Source inspection shows current main appends to the DSML recovery buffer and only clears an opened-but-unclosed DSML block at final flush, with no cap before then.

## Solution Assessment

Yes in direction. The existing recoverer is the right owner path, but the final fix should use UTF-8 byte accounting to match sibling caps and should remain open until the linked PR or a replacement lands.
