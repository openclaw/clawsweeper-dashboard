## Summary

Keep open. The PR fixes a real empty-reply fallback problem, but it also silences Gateway timeout/aborted final responses with empty payloads, so the patch needs a narrow adjustment before maintainer review/landing.

## What This Changes

The PR removes fallback logging for empty Gateway and embedded agent payload lists, adds regression tests for Gateway, ACP, and direct delivery empty replies, and adds a changelog entry.

## Best Possible Solution

Suppress only successful intentional empty replies, preserve non-ok Gateway terminal summaries such as timeouts, and keep the added regression coverage plus changelog aligned with that boundary.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main logs for `status: "ok"` with `payloads: []`, while the patched Gateway branch would also silence `status: "timeout"`/`summary: "aborted"` empty responses because it checks only payload length.

## Solution Assessment

No. The patch is directionally right, but the best fix should gate silence to successful empty replies and continue surfacing Gateway timeout/error summaries.
