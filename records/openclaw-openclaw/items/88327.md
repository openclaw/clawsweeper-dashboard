## What This Changes

This PR removes Control UI's hardcoded `chat.history` `maxChars` request override so the gateway can apply `gateway.webchat.chatHistoryMaxChars`.

## Best Possible Solution

Land the narrow override removal after real Control UI/Gateway proof shows a long WebChat history load respects `gateway.webchat.chatHistoryMaxChars` without sending a client cap.

## Reproduction Assessment

Yes, source-reproducible: current main always sends `maxChars: 4000` from Control UI while gateway source, tests, and docs show RPC `maxChars` overrides `gateway.webchat.chatHistoryMaxChars`. I did not execute a live browser/Gateway repro in this read-only review.

## Solution Assessment

Yes, the proposed code path is the narrowest maintainable fix: remove the UI's per-request override and let the existing gateway config/default precedence apply. The merge still needs real behavior proof, not a larger full-message-reader product change.
