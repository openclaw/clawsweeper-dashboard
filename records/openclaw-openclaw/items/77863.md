## Summary

Keep open: this targets a real memory regression report, but the PR is not merge-ready because the resolved-path import is not portable and the submitted proof does not show patched OpenClaw running in the reported global-install setup.

## What This Changes

This PR changes sqlite-vec loading to resolve through `createRequire(import.meta.url)` with a bare-import fallback and adds sqlite-vec loader unit tests.

## Best Possible Solution

Revise the PR to import a resolved sqlite-vec path via a file URL, keep tests hermetic around optional native dependencies, add the required changelog entry, and attach a packaged/global-install smoke that shows vector loading from outside the install root.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Partially: the related report has concrete 2026.5.4 logs and current source has a clear degraded path when sqlite-vec cannot be imported, but I did not live-reproduce the global npm cwd failure against current main.

## Solution Assessment

Is this the best way to solve the issue? No as written: module-relative resolution is a plausible direction, but the resolved path must be converted to a file URL, tests should not depend on optional native install state, and the fix needs real packaged-install proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Convert resolved paths to file URLs before importing:** `packages/memory-host-sdk/src/host/sqlite-vec.ts:21`
  - body: `require.resolve()` returns a native filesystem path. On Windows that is a `C:\...` string, and `import(resolved)` is parsed by Node as an unsupported `c:` URL; the catch then falls back to the old bare import path, so this fix would not actually cover Windows global installs. Convert the resolved path with `pathToFileURL(resolved).href` before importing it.
  - confidence: 0.92
- **[P2] Keep sqlite-vec loader tests hermetic:** `packages/memory-host-sdk/src/host/sqlite-vec.test.ts:14-17`
  - body: The new no-`extensionPath` tests exercise the real `sqlite-vec` optional native package. Since OpenClaw intentionally keeps `sqlite-vec` optional and supports installs where optional native dependencies are omitted, these tests can fail in supported package states instead of validating the loader logic. Mock `sqlite-vec` for the success path or inject the resolver so the test does not require the optional package.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `packages/memory-host-sdk/src/host/sqlite-vec.ts:15`
  - body: This is a user-facing memory regression fix, but the PR only changes the loader and tests. OpenClaw policy requires user-facing fixes to add a `CHANGELOG.md` entry before merge.
  - confidence: 0.88
