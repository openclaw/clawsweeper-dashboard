## Summary

Keep open: current main and the latest release still lack a root `sharp` dependency declaration while bundled image processing imports `sharp`, and https://github.com/openclaw/openclaw/pull/83403 is an open PR that explicitly closes this issue.

## What This Changes

Fresh packaged installs should include Sharp so image attachment resizing and media-understanding image operations do not fail with the missing optional dependency error.

## Best Possible Solution

Land a root `optionalDependencies` entry for exact `sharp: 0.34.5` with matching `pnpm-lock.yaml` importer and dependency checks, then close this issue and sweep the related Sharp image-processing reports.

## Reproduction Assessment

Yes, source-level. Current main's root package and lockfile do not declare `sharp`, while the bundled image-ops module imports it and throws the reported missing optional dependency error; I did not run a fresh global install in this read-only review.

## Solution Assessment

Yes in direction. Root `optionalDependencies` is the narrow package-boundary fix for a best-effort native dependency, but it should be exact-pinned and paired with a lockfile update before merge. ([docs.npmjs.com](https://docs.npmjs.com/cli/v11/configuring-npm/package-json))
