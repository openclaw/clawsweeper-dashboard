## Repair Work Prompt

Fix the external-channel cron announce resolution gap from https://github.com/openclaw/openclaw/issues/78754. Current `src/infra/outbound/channel-resolution.ts` rejects raw channel IDs before plugin lookup when `normalizeDeliverableOutboundChannel` cannot find an external channel in registered metadata, and `src/infra/outbound/targets.ts` then reports `Unsupported channel: openclaw-weixin`. Add a generic, non-Weixin-specific path that lets configured, loaded, or active external channel plugins resolve for isolated cron announce delivery without accepting internal webchat or arbitrary non-plugin channels. Preserve existing best-effort partial-failure accounting in `src/cron/isolated-agent/delivery-dispatch.ts`; do not add config flags, hard-coded channel ids, or CHANGELOG edits. Add focused regression tests around `resolveOutboundChannelPlugin`/target resolution for an installed or active external channel not yet in the deliverable list, plus a negative test for truly unsupported channels.
