## Summary

Keep open: the report matches a current core routing gap where cron announce/failure delivery resolves an external channel target before the outbound path can bootstrap the plugin, so a valid `openclaw-weixin` channel can still surface as `Unsupported channel`.

## What This Changes

Cron isolated-session announce/failure delivery to the external `openclaw-weixin` channel fails with `Unsupported channel` even though isolated agent message sends can reach Weixin.

## Best Possible Solution

Add a generic bootstrap-aware external-channel target resolution path for cron announce/failure delivery, with regression coverage using a fake external channel and no Weixin-specific branch.

## Reproduction Assessment

Yes, at source level: a valid external channel id can flow through cron failure delivery into `resolveOutboundTarget`, which returns `Unsupported channel` if the external plugin has not already been registered. I did not run a live Weixin cron job in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to reuse the existing plugin bootstrap seam during target resolution or before cron calls it. A Weixin-specific workaround or a persisted config migration would be the wrong boundary for this symptom.
