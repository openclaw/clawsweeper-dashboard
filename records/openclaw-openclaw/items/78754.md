## Summary

## What This Changes

The issue requests that isolated cron announce delivery resolve and send to the external `openclaw-weixin` channel instead of failing with `Unsupported channel: openclaw-weixin`.

## Best Possible Solution

Add a narrow generic external-channel lookup/bootstrap path for cron announce target resolution, with regression coverage and later redacted live proof for a configured `openclaw-weixin` job.

## Reproduction Assessment

Yes at source level, but not as a live Weixin run. Current main can reject an external channel id before plugin lookup, and that missing plugin path returns the reported `Unsupported channel` error.

## Solution Assessment

Yes for the fix direction: the maintainable repair is a generic plugin-channel resolver/bootstrap fallback, not a Weixin hard-code. The separate false-success and stale Weixin runtime paths should remain in their existing follow-up tracks.
