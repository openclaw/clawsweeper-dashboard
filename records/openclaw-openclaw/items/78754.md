## Summary

Keep open: this is a plausible regression in the documented external-channel cron announce path, and current main has adjacent fixes, but there is not yet high-confidence proof that openclaw-weixin cron announce or failure-destination delivery is fixed on current main.

## What This Changes

Cron jobs targeting isolated sessions with announce delivery to the external openclaw-weixin channel should resolve and deliver instead of logging Unsupported channel.

## Best Possible Solution

Make cron announce and failure-destination resolution use the same loaded external channel plugin/runtime path that successful isolated message sends use, with a generic regression test for non-bundled channel ids and no hard-coded Weixin behavior.

## Reproduction Assessment

No high-confidence current-main live reproduction is available. The log maps to the source path where cron failure-destination delivery fails if the outbound plugin cannot be resolved, but the Weixin plugin setup and credentials were not exercised here.

## Solution Assessment

Unclear. The maintainable direction is a generic external-channel resolution fix, but recent merged work in https://github.com/openclaw/openclaw/pull/77799 and https://github.com/openclaw/openclaw/pull/78644 needs to be tested against this exact Weixin cron path before closing or patching further.
