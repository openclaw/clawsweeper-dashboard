## What This Changes

## Best Possible Solution

Move the Mattermost API URL/path validation into a shared helper used by both `client.request` and direct authenticated file downloads, with regression coverage and redacted real Mattermost proof before merge.

## Solution Assessment

No. The shared client-boundary direction is right, but this implementation stops at `client.request` and leaves `resolveMattermostMedia` building authenticated `/files/{fileId}` URLs directly.
