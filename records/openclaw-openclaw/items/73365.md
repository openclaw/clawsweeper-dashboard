## What This Changes

## Reproduction Assessment

Yes, at source level. With queue mode `steer`, current main rejects or defers a message sent while the active run is in a non-streaming tool phase because the embedded, reply-runner, and reply-registry gates still depend on `isStreaming`.

## Solution Assessment

No, not as written. Lifecycle-aware active/stopped checks are the right fix shape, but this branch must be ported to current runner paths and must not add a default throttle that drops burst steering unless maintainers explicitly accept that behavior change.
