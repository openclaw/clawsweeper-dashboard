## Summary

Keep open: the lifecycle-aware steering direction is useful, but this branch is not merge-ready because it changes the old `pi-embedded-runner` path while current main uses `embedded-agent-runner`, and it adds a default production throttle that can drop documented burst steering. The maintainer label and member-authored status also make this a human-owned rebase or replacement path, not cleanup-close material.

## What This Changes

This PR changes active-run steer admission from streaming-only checks to lifecycle/injectability checks for embedded and reply runs, adds steer injection validation, and updates focused tests.

## Best Possible Solution

Land a rebased or replacement lifecycle-aware steering fix on the current `src/agents/embedded-agent-runner` and reply-run paths, preserving lossless burst steering plus stopped-run and compaction safeguards.

## Reproduction Assessment

Yes, at source level. With queue mode `steer`, current main still rejects or defers a message sent while the active run is in a non-streaming tool phase because the embedded, reply-agent, and reply-registry gates depend on `isStreaming`.

## Solution Assessment

No, not as written. Lifecycle-aware active/stopped checks are the right fix shape, but this branch must be ported to current runner paths and must not add a default throttle that drops burst steering unless maintainers explicitly accept that behavior change.
