## Summary

Keep open. The underlying active-run steering bug is still present on current main and the latest release, but this protected maintainer PR is not merge-ready because it targets stale embedded-runner paths and adds a lossy default steer throttle.

## What This Changes

The PR changes steer-message admission for embedded and reply runs from streaming-only checks to active/stopped lifecycle checks, adds shared steer injection bounds, updates tests, and edits the release-owned changelog.

## Best Possible Solution

Land a rebased or replacement lifecycle-aware steering fix on the current `embedded-agent-runner` and reply-run paths, keep burst steering lossless, and leave release-note context in PR metadata instead of `CHANGELOG.md`.

## Reproduction Assessment

Yes, at source level. With queue mode `steer`, current main rejects or defers a message sent while the active run is in a non-streaming tool phase because the embedded, reply-runner, and reply-registry gates still depend on `isStreaming`.

## Solution Assessment

No, not as submitted. Lifecycle-aware active/stopped checks are the right fix shape, but this branch must be ported to current runner paths and must not add a default throttle that drops burst steering.
