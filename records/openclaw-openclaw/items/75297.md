## Summary

Close as implemented on current main. The post-v2026.4.29 mainline now has targeted Gateway mitigations for the reported slow `models.list`/`sessions.list` catalog path plus the related runtime-deps reinstall pressure, with tests and an unreleased changelog entry tying the fix to this report.

## What This Changes

The issue reports a post-2026.4.23 Gateway regression where `sessions.list`, `models.list`, and related Control UI surfaces become very slow under large local state, while rollback to 2026.4.23 restores responsiveness.

## Best Possible Solution

Keep the current main mitigations as the resolution for this concrete v2026.4.29 rollback report, ship them in the next release, and leave broader Gateway saturation and pagination follow-ups on the narrower existing trackers.

## Reproduction Assessment

Yes. The issue provides a high-confidence version-to-version reproduction shape with the same local state: compare post-2026.4.23 builds against v2026.4.23 using `sessions.list`, `models.list`, liveness diagnostics, and CPU; current tests now cover the slow catalog paths that matched the strongest source-level failure mode.

## Solution Assessment

Yes. The current fix is a narrow maintainable response: avoid waiting on slow provider catalog discovery in hot list views and avoid repeated runtime-deps materialization churn, without changing the exact full-catalog `models.list --all` semantics.
