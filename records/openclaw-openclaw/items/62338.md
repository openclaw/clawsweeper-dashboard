## Summary

Keep open. Current main still lacks the doctor-facing FTS5 warning, and this PR is useful, but it needs a current-main refresh plus release-note/docs cleanup before landing or replacement.

## What This Changes

The PR adds FTS status to the gateway doctor memory payload, forwards it through the CLI doctor probe, and emits an OpenClaw doctor warning when SQLite FTS5 is unavailable.

## Best Possible Solution

Land an updated diagnostic PR that exposes FTS availability in doctor status, warns in doctor output, preserves current skipped-probe semantics, documents the additive field as needed, and treats #62328 as a broader follow-up unless maintainers choose otherwise.

## Reproduction Assessment

Yes. On current main, source inspection shows manager.status() exposes fts enabled/available/error while doctor.memory.status and noteMemorySearchHealth do not forward or render it; an FTS-unavailable manager status therefore remains invisible in openclaw doctor.

## Solution Assessment

Partially. The code direction is a good narrow diagnostic fix, but it should be rebased onto the current probe contract and should not present itself as the full fix for the broader keyword fallback issue unless maintainers explicitly accept that scope.
