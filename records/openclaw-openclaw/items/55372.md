## Summary

Keep open. Current main still has routed restart-sentinel notice delivery with no config guard, and the config schema/docs do not expose a notification suppress flag; the broader #70296 context does not supersede this narrower tracker.

## What This Changes

Request a config option to suppress restart-sentinel outbound notifications while preserving session wake and heartbeat behavior.

## Best Possible Solution

Add a gateway-owned boolean that defaults to current behavior and gates only outbound restart-sentinel notice delivery, leaving sentinel consumption, system-event wake, heartbeat wake, continuation enqueue/drain, and fallback session-only wake intact.

## Reproduction Assessment

Yes. Source inspection on current main shows a routable sentinel reaches deliverRestartSentinelNotice after enqueueRestartSentinelWake, and the focused test asserts both outbound delivery and wake with no suppress-config path.

## Solution Assessment

Yes. A gateway config boolean around only outbound notice delivery is the narrowest maintainable solution; hooks.internal.entries is not the right surface because the restart sentinel runs as a post-ready sidecar, not a hook entry.
