## Summary

Keep open. Current main still has unconditional routed restart-sentinel notice delivery and no schema/docs option to suppress it; related restart fixes cover wake and continuation durability, not a notify opt-out.

## What This Changes

The issue requests a config option to suppress restart-sentinel outbound notifications while preserving session wake behavior.

## Best Possible Solution

Add a gateway-owned boolean that defaults to current behavior and gates only restart-sentinel outbound notice delivery, while preserving sentinel consumption, system-event wake, heartbeat wake, durable continuation enqueue/drain, and fallback session-only wake.

## Reproduction Assessment

Yes. Source inspection shows a routable restart sentinel still reaches deliverRestartSentinelNotice without a config guard, and the focused test locks in outbound delivery plus wake; I did not run tests because this is a read-only sweep and pnpm is unavailable in this shell.

## Solution Assessment

Yes. A first-class gateway config flag around only the outbound notice is the narrowest maintainable solution; putting this under hooks.internal.entries would be misleading because the restart sentinel is a gateway startup sidecar, not an internal hook entry.
