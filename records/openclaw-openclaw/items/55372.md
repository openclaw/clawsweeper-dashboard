## What This Changes

The issue requests a config option to suppress restart-sentinel outbound notifications while preserving restart wake/heartbeat behavior.

## Best Possible Solution

Add a gateway-owned boolean that defaults to current visible-notice behavior and gates only outbound restart-sentinel notice delivery while preserving wake and continuation handling.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: a routable restart sentinel reaches `deliverRestartSentinelNotice` without a config guard, and focused tests assert outbound delivery plus wake on current main. I did not run a live channel restart in this read-only review.

## Solution Assessment

Yes, a notice-only gateway config flag is the narrowest maintainable direction. The proposed `hooks.internal.entries` shape should not be used because restart-sentinel handling is a gateway startup sidecar, not a hook entry.
