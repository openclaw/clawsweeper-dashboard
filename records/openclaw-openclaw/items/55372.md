## Summary

Keep open. Current main still has no config/schema/docs opt-out, and the restart-sentinel startup path still sends a visible outbound notice whenever it resolves a channel route.

## What This Changes

Add a config option to suppress restart-sentinel outbound channel notifications while preserving restart wake and continuation behavior.

## Best Possible Solution

Add a gateway-owned boolean that defaults to current visible-notice behavior and gates only outbound restart-sentinel notice delivery, with schema, help, labels, docs, tests, and changelog aligned.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a routable restart sentinel reaches deliverRestartSentinelNotice without a config guard, and the focused tests assert outbound delivery plus wake on current main.

## Solution Assessment

Yes. A gateway-owned notice-only flag is the narrowest maintainable direction; the proposed hooks.internal.entries path should not be used because restart-sentinel handling is a gateway sidecar, not a hook entry.
