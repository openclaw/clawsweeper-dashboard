## What This Changes

Request a config option to suppress outbound restart-sentinel notifications while preserving restart wake and continuation behavior.

## Best Possible Solution

Add a gateway-owned boolean that defaults to current visible-notice behavior and gates only outbound restart-sentinel notice delivery while preserving wake, continuation, and queue recovery.

## Reproduction Assessment

Yes, source-level. Current main reaches deliverRestartSentinelNotice for any routed restart sentinel with no config guard, while tests assert notification delivery and wake together.

## Solution Assessment

Yes, if implemented as a notice-only gateway config option. The proposed hooks.internal.entries shape is not the best fit because restart-sentinel handling is a gateway startup sidecar, not a hook entry.
