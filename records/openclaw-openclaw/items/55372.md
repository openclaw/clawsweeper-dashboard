## Summary

Keep open. Current main still has no config/schema/docs opt-out, and the restart-sentinel startup path still sends routed channel notices whenever a route resolves; older related reports were closed stale or as duplicates, while the broader open report points back to this narrower request.

## What This Changes

The issue requests a user-facing config option to suppress restart-sentinel outbound notifications while preserving session wake/heartbeat behavior.

## Best Possible Solution

Add a gateway-owned boolean that defaults to current behavior and gates only visible restart-sentinel notice delivery, with schema/docs/tests/changelog aligned.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a routable restart sentinel reaches deliverRestartSentinelNotice without a config guard, and the focused tests assert outbound delivery plus wake on current main.

## Solution Assessment

Yes, but not via the hooks path suggested in the issue body. A gateway-owned notice-only flag is the narrowest maintainable fix because it preserves the wake/continuation contract while removing the unsolicited channel message.
