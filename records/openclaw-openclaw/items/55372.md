## Summary

Keep open. Current main still has no config/schema/docs opt-out and the restart-sentinel startup path still sends routed channel notices whenever a route resolves; the broader related issue overlaps but points back to this narrower sentinel request, so this should remain the canonical item for the opt-out.

## What This Changes

Request a configuration option to suppress restart-sentinel outbound channel notifications while preserving wake and heartbeat behavior.

## Best Possible Solution

Add a gateway-owned boolean that defaults to current behavior and gates only outbound restart-sentinel notice delivery.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a routable restart sentinel reaches deliverRestartSentinelNotice without a config guard, and the focused tests assert outbound delivery plus wake on current main.

## Solution Assessment

Yes. A first-class gateway config flag around only the outbound notice is the narrowest maintainable direction; hooks.internal.entries is the wrong surface because restart-sentinel handling is a gateway startup sidecar, not a bundled hook entry.
