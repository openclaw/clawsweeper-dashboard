## Summary

Keep open. Current main still has no restart-sentinel outbound-notification opt-out: the startup path always calls the notice delivery helper when routing resolves, existing tests lock in notice-plus-wake behavior, and the config/docs surfaces only expose generic hooks and unrelated restart settings.

## What This Changes

The issue requests a configuration option to suppress the restart-sentinel outbound notification while preserving wake/heartbeat behavior.

## Best Possible Solution

Add a gateway-owned config flag that defaults to current behavior and gates only restart-sentinel outbound notice delivery; preserve sentinel consumption, continuation queueing/draining, system-event wake, and heartbeat wake, with schema/help/docs and focused regression tests.

## Reproduction Assessment

Yes. A routable restart sentinel exercises scheduleRestartSentinelWake and current tests already prove the observable path: outbound delivery is attempted and the session wake is enqueued, with no config value available to suppress only the notice.

## Solution Assessment

Yes. A first-class gateway restart-sentinel config flag is the narrowest maintainable fix; putting this under hooks.internal.entries would be misleading because the restart sentinel is a gateway startup sidecar, not an internal hook.
