## Summary

Keep open. Current main still does not provide the requested global setup/onboarding hook for credentialed tool-only plugins, and the related open metadata PR does not implement setup-time prompting or plugin-owned credential persistence.

## What This Changes

The issue requests a first-class global setup hook so credentialed tool-only plugins can participate in `openclaw setup` and persist plugin-owned credentials/config.

## Best Possible Solution

Define a backward-compatible, manifest-first tool-plugin setup contract that global setup can discover cheaply, prompt through SecretInput-safe plumbing, persist under plugin-owned config, and document/test as public SDK behavior.

## Reproduction Assessment

Not applicable as a runtime bug; this is a public SDK/setup feature gap. Source inspection on current main confirms the requested `setup.toolPlugins` descriptor, setup registry entry, setup API hook, and sensitive credential prompt path are absent.

## Solution Assessment

Unclear pending maintainer API design. A dedicated descriptor-first tool-plugin setup contract fits the plugin boundary, but it should be coordinated with https://github.com/openclaw/openclaw/pull/80774 and the broader credential-provider discussion at https://github.com/openclaw/openclaw/issues/59165.
