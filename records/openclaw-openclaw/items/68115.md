## Summary

Keep open. Current main still lacks a first-class setup/onboarding hook for credentialed tool-only plugins, and the related open metadata PR does not implement or supersede the requested global setup flow.

## What This Changes

Add a setup/onboarding hook so credentialed tool-only plugins can declare setup metadata, prompt during global `openclaw setup`, and persist plugin-owned credentials/config.

## Best Possible Solution

Define one backward-compatible, manifest-first tool-plugin credential setup contract that the global wizard can consume without broad runtime execution, then document and test the SDK/config contract.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature/API gap. Source inspection on current main shows no manifest `setup.toolPlugins` descriptor, setup registry entry type, API handler, or wizard path for sensitive tool-plugin credentials.

## Solution Assessment

Unclear pending maintainer API design. The manifest-first direction fits the plugin boundary, but it should be coordinated with https://github.com/openclaw/openclaw/pull/80774 so auth metadata, SecretInput handling, and plugin-owned persistence do not split into competing contracts.
