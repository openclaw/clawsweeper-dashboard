## Summary

Keep open. Current `main` still lacks a first-class global setup hook for credentialed tool-only plugins, and the related #61551 runtime hook-runner issue is closed but does not cover this public SDK/setup API gap.

## What This Changes

Add a global setup/onboarding SDK hook and manifest descriptor for credentialed tool-only plugins so `openclaw setup` can prompt for and persist plugin-owned configuration.

## Best Possible Solution

Define an additive, manifest-first tool-plugin setup contract that registers setup-time credential/config handlers, reuses SecretInput-safe prompting, persists under plugin-owned config, and updates SDK docs plus contract tests.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature/API gap. The absence is high-confidence from source review: current setup descriptors, registry collection, public plugin API, and setup wizard have no credentialed tool-plugin setup path.

## Solution Assessment

Yes, keeping this open for maintainer design is the best current path. The requested direction aligns with the plugin API expansion guidance, but the exact credential and discovery semantics should be settled before an automated implementation PR.
