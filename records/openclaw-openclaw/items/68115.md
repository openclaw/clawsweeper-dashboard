## Summary

Keep open. Current main still lacks the requested first-class setup/onboarding hook for credentialed tool-only plugins: manifest setup metadata, setup descriptor lookup, setup registry collection, public plugin API, and the global setup wizard remain provider/CLI/generic-config shaped, and the related open draft metadata PR does not close or supersede this setup API gap.

## What This Changes

Request a first-class global setup hook so credentialed tool-only plugins can participate in `openclaw setup` and persist plugin-owned API-key/config values.

## Best Possible Solution

Define one additive manifest-first tool-plugin setup contract that declares credential/config needs cheaply, reuses SecretInput-safe prompting, persists under plugin-owned config, and wires the global setup wizard, docs, SDK baseline, and contract tests without executing every plugin during discovery.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature/API gap. The absence is high-confidence from source inspection of current main’s manifest setup fields, setup descriptor lookup, setup registry handlers, public plugin API, and setup wizard behavior.

## Solution Assessment

Unclear pending maintainer API design. A manifest-first setup hook is the right boundary, but it should be coordinated with the open `authRequirements` metadata work so global setup consumes one stable credential contract.
