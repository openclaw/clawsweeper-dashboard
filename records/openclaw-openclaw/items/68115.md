## Summary

Keep open. Current main still lacks a first-class global setup hook for credentialed tool-only plugins, and the related open metadata PR does not implement or supersede the requested setup-time credential collection/persistence path.

## What This Changes

Add a first-class setup/onboarding hook so credentialed tool-only plugins can participate in the global `openclaw setup` wizard.

## Best Possible Solution

Define one backward-compatible, manifest-first tool-plugin credential setup contract that global setup can discover cheaply, prompt through SecretInput-safe plumbing, persist under plugin-owned config, and document/test as a public SDK surface.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature/API gap. Source inspection on current main confirms no `setup.toolPlugins` descriptor, setup registry entry, setup API handler, or wizard path that prompts and persists sensitive tool-plugin credentials.

## Solution Assessment

Unclear pending maintainer API design. A manifest-first descriptor plus setup-time credential handler fits the plugin boundary, but it should be coordinated with https://github.com/openclaw/openclaw/pull/80774 so metadata and credential persistence do not become competing contracts.
