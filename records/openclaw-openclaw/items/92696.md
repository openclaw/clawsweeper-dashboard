## Summary

Keep this PR open, but it is not merge-ready. The cleanup gap is real, yet the submitted ordering can dispose the session MCP runtime while the old embedded run is still registered and potentially executing MCP tool work.

## What This Changes

The PR moves `retireSessionMcpRuntime()` before `waitForEmbeddedAgentRunEnd()` in gateway session cleanup and adds a verbose log for the reordered MCP retirement.

## Best Possible Solution

Use a lifecycle-safe cleanup contract that retires MCP runtime only after the run drains, is explicitly force-cleared, or has deferred retirement tied to lease release, with focused reset and delete timeout coverage.

## Reproduction Assessment

Yes at source level: current main skips MCP runtime retirement when the embedded run wait returns false after reset/delete abort. I did not reproduce the live file-locking MCP process scenario.

## Solution Assessment

No, not as submitted. The PR targets a real cleanup gap, but moving retirement before the run-drain result conflicts with the active-run and MCP lease contracts.
