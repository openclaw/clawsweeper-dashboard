## Summary

## What This Changes

The PR moves `retireSessionMcpRuntime()` before `waitForEmbeddedAgentRunEnd()` in gateway session reset cleanup.

## Best Possible Solution

## Solution Assessment

Partly. The PR is a plausible narrow fix for the skipped cleanup path, but it is not yet proven as the best fix because it tears down MCP runtime before the aborting run has necessarily drained.
