## Summary

Keep this PR open: the source-level bug is plausible and the patch is narrow, but it still needs exact redacted real-process proof and maintainer acceptance of retiring MCP runtime before a reset attempt has actually drained.

## What This Changes

The PR moves MCP runtime retirement earlier during gateway session reset cleanup so old session MCP processes are disposed immediately after aborting the embedded run rather than only when the run ends within the grace window.

## Best Possible Solution

Land this gateway reset cleanup only after exact redacted process/log proof and maintainer agreement that abort-time MCP teardown is intended; keep the manager-level rebind fix as its separate related path.
