## Summary

Keep open for maintainer product review: current main and the latest release still always materialize sandbox skills and have no `skipSkillsSync` config, but the requested opt-out changes config surface and sandbox skill-read invariants, so it needs sandbox/config direction before implementation.

## What This Changes

Request to add an opt-in `agents.defaults.skipSkillsSync` setting that skips per-session sandbox skill copying for custom backends with preinstalled skills.

## Best Possible Solution

Have sandbox owners decide whether the opt-out belongs as a backend capability or explicit config, then implement it with docs, schema, and regression coverage that preserves default skill materialization.

## Reproduction Assessment

Yes, at source level: current main calls sandbox skill sync for both non-`rw` and `rw` sandbox layouts, and adjacent tests assert those calls. The reported scale cost was not independently measured here.

## Solution Assessment

Unclear: `agents.defaults.skipSkillsSync` is plausible, but a backend-declared capability or sandbox-owned config may be the safer owner boundary because current sync protects readable sandbox skill paths.
