## Summary

Keep open: current main still syncs skills into sandbox-readable locations, and the request is a real operational optimization but needs maintainer design judgment because skipping that path can break sandbox-rooted skill reads.

## What This Changes

The issue requests a new `agents.defaults.skipSkillsSync` config option so sandbox workspaces can skip copying skills when a custom backend already provides them.

## Best Possible Solution

Keep this open for a maintainer decision on a sandbox-scoped or backend-aware capability that preserves in-sandbox skill readability while avoiding redundant copies for pre-baked custom backends.

## Reproduction Assessment

Yes for the current behavior from source inspection: current main calls skill synchronization during sandbox workspace setup and the docs/tests confirm sandbox-readable skill materialization is intentional. I did not reproduce the reported large-deployment disk or latency cost in a live multi-session setup.

## Solution Assessment

No; a broad `agents.defaults.skipSkillsSync` boolean is only a plausible fix because it can bypass the documented sandbox-readable skill path. A safer design would tie the skip to a sandbox/backend capability or policy that proves equivalent in-container skill paths.
