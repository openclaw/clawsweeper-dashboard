## Summary

Keep open: current main does not have this repair, and the latest PR head still has a correctness blocker because it treats a configured Codex OAuth PI model as permission to retain native Codex runtime and CLI session state. The PR is also draft, dirty, and maintainer-labeled.

## What This Changes

This PR adds a plugin-owned doctor session-state cleanup contract, registers Codex route-state ownership, wires it into doctor state integrity, and updates tests, docs, and the changelog for stale Codex session routes.

## Best Possible Solution

Keep the plugin-owned doctor repair direction, but make route allowance state-kind aware so openai-codex PI routes preserve valid provider model state while still clearing stale native Codex runtime, harness, and CLI bindings; then regenerate the Plugin SDK API baseline.

## Reproduction Assessment

Yes, source-level. A session configured for openai-codex/gpt-5.4 under PI with stale agentHarnessId: "codex" and codex-cli bindings reaches routeAllowsOwnerState and skips the cleanup branch, contrary to the documented provider/runtime split.

## Solution Assessment

No. The repair seam is maintainable, but the current predicate is too coarse; provider-route allowance needs to be separate from runtime, harness, and CLI cleanup, and the SDK baseline hash needs to be updated for the new public type.
