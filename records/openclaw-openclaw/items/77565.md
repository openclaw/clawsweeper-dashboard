## Summary

Keep open. The PR is useful source work for the stale Codex session-route bug, but the current head still conflates `openai-codex/*` PI provider routes with native Codex runtime state and also misses the required Plugin SDK API baseline update.

## What This Changes

The PR adds a plugin-owned doctor session-state repair contract and Codex implementation, wires it into doctor state integrity, and updates tests, docs, and changelog for stale Codex session cleanup.

## Best Possible Solution

Keep the plugin-owned repair direction, make route allowance state-kind aware so PI provider state does not preserve stale native runtime/harness/CLI bindings, then regenerate the SDK API baseline and rerun the targeted doctor/Codex tests plus changed gate.

## Reproduction Assessment

Yes, source-level. A session configured for `openai-codex/gpt-5.4` under PI with stale `agentHarnessId: "codex"` and `codex-cli` bindings reaches `routeAllowsOwnerState` and skips the cleanup branch.

## Solution Assessment

No. The repair seam is maintainable, but the current predicate is too coarse and the new public SDK export needs its baseline regenerated.
