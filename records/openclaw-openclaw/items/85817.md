## Summary

Keep this PR open for maintainer/security review. The scoped Policy design is useful and well covered, but the current implementation can silently skip a scoped `agentIds` rule when the targeted runtime agent is not present in `agents.list`, even though OpenClaw can route explicit unknown agent ids through default/global posture.

## What This Changes

Adds named Policy `scopes.<scopeName>` overlays for `agentIds`, scoped `tools` and `agents.workspace` conformance, `tools.alsoAllow.expected`, docs, focused policy tests, and small agent test/model metadata fixes.

## Best Possible Solution

Land a scoped Policy contract that either evaluates inherited default/global posture for every declared `agentIds` target or rejects scoped selectors with no enforceable evidence, with docs and tests for the chosen behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: add a scope for a non-main runtime agent id that is not in `agents.list`, configure default/global posture that violates the scoped rule, and the scoped filter has no evidence row to report.

## Solution Assessment

No. The overlay direction is promising, but the implementation needs unmatched-selector handling before this becomes the public Policy audit contract.
