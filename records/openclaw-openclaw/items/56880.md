Keep open. Current main and the latest release still serialize normal same-session agent turns; `/btw` is only an explicit ephemeral side-question path, and the linked implementation PR remains open, conflicting, and labeled for proof/risk review.

## What This Changes

The issue requests concurrent same-session message handling so short follow-ups can run while a long agent turn is still active.

## Best Possible Solution

Keep this issue open while maintainers review or replace https://github.com/openclaw/openclaw/pull/57239 and decide whether to fold the broader scheduler design into https://github.com/openclaw/openclaw/issues/53023.

## Reproduction Assessment

Yes, at source level. Current main routes normal same-session follow-up work through steering, enqueue/wait behavior, and session lanes capped at one active run rather than spawning a parallel normal turn.

## Solution Assessment

Unclear. A fast-lane or concurrent-turn implementation could solve the latency problem, but maintainers need to decide ordering, tool safety, session-state, and whether this belongs in `/btw`, queue policy, or configurable lane concurrency.
