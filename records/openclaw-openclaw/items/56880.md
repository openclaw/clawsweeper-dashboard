## Summary

Keep open. Current main still has one normal tool-capable run per session, the explicit `/btw` side path is narrower than the requested concurrent turns, and an open PR already references this issue with closing syntax.

## What This Changes

The issue requests concurrent or fast-lane message handling within one session so short follow-ups are not blocked by a long-running agent turn.

## Best Possible Solution

Keep this issue open while maintainers review, replace, or close the linked PR; the durable end state should be either a bounded quick-reply path with hard isolation or a documented per-session concurrency design with tests for tool safety and ordering.

## Reproduction Assessment

Yes. A high-confidence source path is a long normal run in one session followed by another normal message in that same session; current main steers, enqueues, interrupts-and-waits, or uses explicit `/btw`, but does not spawn an independent normal tool-capable turn.

## Solution Assessment

Unclear. The current `/btw` path is a useful explicit workaround, but automatic fast lanes or general concurrent turns need maintainer product and safety decisions before they are the best solution.
