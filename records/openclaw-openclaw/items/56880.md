## Summary

Keep open. Current main still serializes normal tool-capable turns per session and routes active-session messages through steering, followup, or interrupt behavior rather than spawning independent concurrent turns; an open PR with closing syntax already targets a narrower fast-lane mitigation, so the issue should wait for maintainer handling there.

## What This Changes

The issue requests concurrent or parallel message handling within a session so quick follow-up messages can get timely responses while a long agent turn is still running.

## Best Possible Solution

Keep the issue open while maintainers review the linked fast-lane PR as the narrow implementation candidate. The durable end state should either land a bounded, tested quick-reply path for active sessions or explicitly design safe per-session concurrency with isolation, transcript consistency, tool conflict handling, and rate-limit controls.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a long normal run in one Discord or other channel session key followed by another normal message in the same session; current main steers or queues the second message instead of spawning an independent tool-capable turn.

## Solution Assessment

Unclear. The linked fast-lane PR is a plausible narrow mitigation, but full concurrent agent turns need a deeper safety design; reviewing, replacing, or closing that PR is better than closing this issue or queueing duplicate repair work.
