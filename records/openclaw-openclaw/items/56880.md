## Summary

Keep open. Current main and v2026.5.20 still serialize normal same-session agent runs, `/btw` is only an explicit ephemeral side-question path, and https://github.com/openclaw/openclaw/pull/57239 is an open implementation candidate with closing syntax for this issue.

## What This Changes

The issue requests concurrent or fast-lane same-session message handling so short follow-ups are not blocked behind a long-running agent turn.

## Best Possible Solution

Keep this issue open while maintainers review https://github.com/openclaw/openclaw/pull/57239 or consolidate the design with https://github.com/openclaw/openclaw/issues/53023, preserving default per-session serialization unless an explicit tested fast-lane policy is approved.

## Reproduction Assessment

Yes, at source level. Current main routes ordinary same-session follow-up work through enqueue/wait behavior and per-session lanes rather than starting a second normal tool-capable turn.

## Solution Assessment

Unclear. A fast-lane or concurrent-turn path could solve the latency problem, but maintainers need to decide ordering, session state, tool safety, and whether this should be `/btw`, a configurable lane policy, or a narrow quick-reply mode.
