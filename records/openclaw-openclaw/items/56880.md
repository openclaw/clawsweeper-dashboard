## Summary

Keep open. Current main still preserves one normal tool-capable agent run per session, the existing quick side path is the explicit tool-less `/btw` command, and the open implementation candidate at https://github.com/openclaw/openclaw/pull/57239 references this issue with closing syntax.

## What This Changes

The issue requests concurrent or fast-lane message handling so short follow-up messages are not blocked by a long-running turn in the same chat session.

## Best Possible Solution

Keep the issue open while maintainers review, replace, or close the linked PR; the durable end state should be either a bounded quick-reply path with hard isolation or a documented per-session concurrency design with tests for tool safety and ordering.

## Reproduction Assessment

Yes. A high-confidence source-level path is a long normal run in one session followed by another normal message in the same session; current main steers or enqueues the new message rather than spawning an independent tool-capable turn.

## Solution Assessment

Unclear. The current `/btw` path is a useful explicit workaround, but automatic normal-message fast lanes or general concurrent turns need maintainer product and safety decisions before they are the best solution.
