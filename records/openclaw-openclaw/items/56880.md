## Summary

Keep open. Current main still serializes normal tool-capable turns per session and routes active-session messages through steering or follow-up queue behavior; the only implemented quick side path is the narrower tool-less `/btw` command. PR #57239 remains an open implementation candidate that references this issue with closing syntax, so this issue should stay open until that PR is reviewed, merged, replaced, or closed.

## What This Changes

Request to allow concurrent or fast-lane per-session message handling so quick Discord or chat-channel follow-ups can run without waiting behind a long active agent turn.

## Best Possible Solution

Keep the issue open while maintainers review PR #57239 as the narrow candidate. The durable end state should either land a bounded, tested quick-reply path for busy sessions or explicitly design safe per-session concurrency with isolation, transcript consistency, tool conflict handling, and rate-limit controls.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a long normal run in one session followed by another normal message in the same session; current main steers or enqueues that message rather than creating an independent tool-capable turn.

## Solution Assessment

Unclear. The current `/btw` path is a useful tool-less workaround and PR #57239 may be a plausible mitigation, but full concurrent agent turns require broader safety design before the general request can be considered solved.
