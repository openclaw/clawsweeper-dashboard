## What This Changes

Allow quick same-session follow-up messages, especially on Discord, to run concurrently or fast-lane instead of waiting behind a long active agent turn.

## Best Possible Solution

Keep this issue open while maintainers review https://github.com/openclaw/openclaw/pull/57239 or fold the work into https://github.com/openclaw/openclaw/issues/53023; the durable outcome should provide an explicit quick-reply or concurrency policy without weakening session and tool-safety guarantees.

## Reproduction Assessment

Yes, at source level. A long active same-session run followed by another normal message still reaches queue policy, followup enqueue/wait behavior, or explicit `/btw`, but current main does not spawn a second normal tool-capable turn.

## Solution Assessment

Unclear. `/btw` is a useful explicit side path, but automatic fast lanes or general concurrent turns require maintainer decisions around isolation, ordering, and tool safety.
