## Summary

Keep open. Current main still preserves one normal active agent run per session; `/btw` covers explicit ephemeral side questions, but it does not implement normal concurrent message handling, and the remaining work needs maintainer scheduler/product review alongside the open umbrella and partial PR.

## What This Changes

Request concurrent or fast-lane same-session message handling so short follow-ups can run while a long agent turn is active.

## Best Possible Solution

Use the open umbrella at https://github.com/openclaw/openclaw/issues/53023 for the scheduler design, review the partial concurrency PR separately, and keep the default serialized session safety model unless maintainers approve a tested fast-lane or opt-in concurrency policy.

## Reproduction Assessment

Yes, at source level. Current main still sends normal embedded runs through a per-session lane capped at one active task, while `/btw` is an explicit ephemeral side-query path rather than normal concurrent message handling.

## Solution Assessment

Unclear. A fast-lane, configurable session-lane concurrency, or user-priority scheduler could each solve part of the latency problem, but maintainers need to choose transcript, tool-safety, and default behavior semantics before implementation.
