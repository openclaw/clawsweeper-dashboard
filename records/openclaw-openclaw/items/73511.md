## Summary

Keep open. Current main still rejects allowlisted hosted custom-scheme browser origins, while this PR is a narrow Gateway authorization-boundary fix with matching regression coverage and no discrete blocking review finding.

## What This Changes

The PR updates Gateway origin normalization so hosted non-special browser origins can match explicit Control UI `allowedOrigins` entries, with regression tests and a changelog entry.

## Best Possible Solution

Land this narrow Gateway normalization fix after maintainer review and exact-head validation, preserving explicit allowlist enforcement for custom-scheme origins.

## Reproduction Assessment

Yes. Current main plus the Node URL parser check reproduces the failure path: `tauri://localhost` yields `URL.origin === "null"`, so `checkBrowserOrigin` cannot match an explicit `tauri://localhost` allowlist entry.

## Solution Assessment

Yes. Reconstructing `scheme://host` only for opaque URL origins that still have a host is the narrowest maintainable fix because it feeds the existing allowlist path instead of adding a desktop-wrapper-specific bypass.
