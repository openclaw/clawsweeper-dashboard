## Summary

Keep open. Current main still rejects the explicit custom-scheme allowlist case, and this PR is the active narrow implementation for the linked Gateway bug; the patch matches the traced root cause and keeps explicit allowlist matching as the authorization gate, but it still needs maintainer review and CI resolution before merge.

## What This Changes

The PR changes Gateway origin normalization to preserve `scheme://host` for non-special URL schemes, adds allowlisted and unlisted custom-scheme origin tests, and adds a changelog entry.

## Best Possible Solution

Land the focused Gateway normalization change after CI/rebase and maintainer confirmation of the authorization semantics: custom desktop-wrapper schemes should match only explicit allowlist entries, while wildcard, host-header fallback, local-loopback, literal-null, malformed-origin, and unlisted-origin behavior stays unchanged.

## Reproduction Assessment

Yes. Current main plus the Node URL check gives a high-confidence reproduction: `tauri://localhost` parses with `url.origin === "null"`, so `checkBrowserOrigin` cannot match an explicit `tauri://localhost` allowlist entry.

## Solution Assessment

Yes. Reconstructing `scheme://host` only for `url.origin === "null"` with a host is the narrowest maintainable fix because it reuses the existing lowercase allowlist path instead of adding a Tauri/Electron-specific bypass or recommending wildcard origins.
