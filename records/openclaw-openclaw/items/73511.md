## What This Changes

The PR preserves `scheme://host` for opaque custom-scheme Gateway Control UI `Origin` headers, adds allow/reject origin-check tests, and adds a manual changelog entry.

## Best Possible Solution

Land the parser-local opaque-origin fallback after real Gateway/custom-wrapper proof is added, keep scheme wildcard policy in https://github.com/openclaw/openclaw/issues/39739, and leave changelog text to release generation.

## Reproduction Assessment

## Solution Assessment

Yes, mostly. The parser-local fallback is the narrow maintainable fix and avoids the broader allowlist reparsing risk seen in https://github.com/openclaw/openclaw/pull/38290, but this PR still needs real Gateway proof and the manual changelog entry removed before merge.
