## Summary

Keep open. Current main still rejects explicit `gateway.controlUi.allowedOrigins` entries for hosted custom-scheme browser origins, while this PR is a narrow Gateway fix with targeted regression coverage and no concrete correctness finding.

## What This Changes

The PR preserves `scheme://host` for hosted opaque custom-scheme origins in Gateway origin normalization, adds custom-scheme allow/reject unit cases, and adds a changelog entry.

## Best Possible Solution

Land this narrow Gateway normalization fix after maintainer review and green changed-surface validation, while preserving explicit allowlist enforcement for custom-scheme origins.

## Reproduction Assessment

Yes. Current main plus the Node URL parser check gives a high-confidence reproduction: `tauri://localhost` parses with `URL.origin === "null"`, so `checkBrowserOrigin` compares `"null"` to an explicit `tauri://localhost` allowlist entry and rejects it.

## Solution Assessment

Yes. Reconstructing `scheme://host` only for opaque URL origins that still have a host is the narrowest maintainable fix because it feeds the existing allowlist path instead of adding a Tauri/Electron-specific bypass.
