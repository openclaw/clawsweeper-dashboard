## Summary

Keep open. The patch is a narrow and plausible fix for the custom-scheme origin bug, but it is not merge-ready because after-fix real behavior proof is missing and maintainers still need to consolidate it with the overlapping Gateway origin PR.

## What This Changes

This PR changes Gateway origin parsing so hosted non-special scheme origins can match explicit `gateway.controlUi.allowedOrigins` entries, adds unit cases, and adds a changelog entry.

## Best Possible Solution

Land one exact opaque-origin normalization fix with focused unit and Gateway handshake coverage, choosing between this PR and https://github.com/openclaw/openclaw/pull/38290 while leaving scheme wildcard semantics to https://github.com/openclaw/openclaw/issues/39739.

## Reproduction Assessment

Yes, at source level. Node parses hosted custom schemes with `URL.origin === "null"`, and current `checkBrowserOrigin` compares that parsed value against the exact allowlist entry; I did not run Vitest in this read-only review.

## Solution Assessment

Yes, the proposed parser-level fix is the narrow maintainable direction because it preserves explicit allowlist enforcement instead of adding a desktop-wrapper bypass. Maintainers should still consolidate it with the overlapping PR and keep or borrow handshake coverage before landing.
