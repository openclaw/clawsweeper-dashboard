## Summary

Keep open. Current main still has the reported custom-scheme origin mismatch, and this PR is a narrow, relevant fix with regression coverage and no concrete correctness or security finding; it still needs normal maintainer review plus current-head CI/rebase attention before merge.

## What This Changes

The PR updates Gateway origin normalization to preserve hosted non-special scheme origins for explicit `gateway.controlUi.allowedOrigins` matching, adds origin-check regression cases, and adds a changelog fix entry.

## Best Possible Solution

Land this narrow Gateway normalization fix after refreshing the branch and validation, preserving explicit allowlist enforcement for custom-scheme origins while keeping literal `Origin: null`, malformed origins, wildcard, Host-header fallback, and local-loopback behavior bounded by the existing rules.

## Reproduction Assessment

Yes. Current main plus the Node URL check provides a high-confidence reproduction: `tauri://localhost` parses with `URL.origin === "null"`, so `checkBrowserOrigin` compares `"null"` to an explicit `tauri://localhost` allowlist entry and rejects it.

## Solution Assessment

Yes. Reconstructing `scheme://host` only for opaque URL origins that still have a host is the narrowest maintainable fix because it feeds the existing allowlist path instead of adding a Tauri/Electron-specific bypass or encouraging wildcard origins.
