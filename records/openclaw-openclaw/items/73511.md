## What This Changes

The PR changes Gateway origin parsing so hosted custom-scheme Control UI origins such as `tauri://localhost` can match exact `gateway.controlUi.allowedOrigins` entries.

## Best Possible Solution

Land this or an equivalent current-main parser fix after redacted Gateway proof, preserving exact allowlist semantics while keeping wildcard-pattern policy in the separate related issue.

## Reproduction Assessment

Yes at source level. Current main compares allowlist entries against `URL.origin`, and a Node probe shows hosted custom schemes serialize that origin as `null` before exact allowlist matching.

## Solution Assessment

Yes for the parser layer, with merge-readiness caveats. Reconstructing only hosted opaque origins before exact allowlist matching is narrower than wildcard, config, or Host-header policy changes.
