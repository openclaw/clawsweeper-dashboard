## What This Changes

This PR changes Gateway origin normalization so hosted non-special scheme origins can match explicit `gateway.controlUi.allowedOrigins` entries, adds unit cases, and adds a changelog entry.

## Best Possible Solution

Land one exact opaque-origin normalization fix with focused unit and Gateway handshake coverage, choosing between this PR and https://github.com/openclaw/openclaw/pull/38290 while leaving scheme wildcard semantics to https://github.com/openclaw/openclaw/issues/39739.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.5.7` parse hosted custom schemes with `URL.origin === "null"`, and `checkBrowserOrigin` compares that parsed value against the exact allowlist entry.

## Solution Assessment

Yes. Reconstructing `scheme://host` only for opaque URL origins that still have a host is a narrow maintainable fix because it keeps explicit allowlist enforcement instead of adding a Tauri/Electron-specific bypass.
