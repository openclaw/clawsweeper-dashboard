## Summary

Keep open: this is a fresh, detailed behavior bug against the subagent completion contract, and current main does not have an exact regression proving that a main-session requester cannot be displaced by a parallel Telegram-bound sibling session.

## What This Changes

The issue reports that a subagent completion announce launched from `agent:main:main` is delivered into a parallel Telegram session instead of the originating parent session.

## Best Possible Solution

Add a focused regression for a main parent session with a parallel Telegram-bound sibling, then enforce that completion handoff is keyed by the recorded requester session before applying external route hints.

## Reproduction Assessment

No high-confidence current-main reproduction was established. The issue has detailed OpenClaw v2026.5.7 field evidence, but local inspection only found a plausible route-hint boundary and missing exact coverage, not a failing test or live Telegram proof.

## Solution Assessment

Unclear until the regression is written. The likely maintainable direction is a narrow requester-session invariant in subagent announce delivery, not a broad Telegram transport rewrite or the separate parent `announceTarget` feature.
