## What This Changes

The issue reports that subagent completion announces spawned from `agent:main:main` are routed to a parallel Telegram-bound session instead of the originating parent session.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/80242; the durable fix should record the originating requester session and prefer requester bindings before child bindings for completion delivery.

## Reproduction Assessment

No live current-main Telegram reproduction was established. Source inspection shows the plausible current-main path: `runSessionKey` is available but not used by `sessions_spawn`, and completion-origin binding resolution checks the child before the requester.

## Solution Assessment

Yes, the linked PR's direction appears to be the narrow maintainable solution: use the existing live run-session key for spawn ownership and make requester binding priority explicit. It still needs real Telegram proof and normal review before merge.
