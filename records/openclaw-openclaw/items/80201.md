## What This Changes

Route subagent completion announces back to the session that issued `sessions_spawn` instead of a parallel Telegram-bound session for the same agent.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/80242; the fix should preserve the recorded requester session as completion owner while keeping sandbox policy tied to the channel peer session.

## Reproduction Assessment

Yes, this is source-reproducible with high confidence, though not live-reproduced here. Current main passes only the channel/session key into spawn ownership and resolves child bindings before requester bindings, matching the reported wrong-session completion route.

## Solution Assessment

Yes, the linked PR’s direction is the narrow maintainable path: separate completion ownership from sandbox policy and prefer requester binding before child fallback. The issue should remain open until that implementation is merged or explicitly rejected.
