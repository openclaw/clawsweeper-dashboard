## What This Changes

The issue reports subagent completion announces from `agent:main:main` being delivered to a parallel Telegram-bound session instead of the originating parent session.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Yes, the linked PR direction is the narrow maintainable fix: separate completion ownership from sandbox identity and prefer requester binding before child fallback. The issue should remain open until that fix is merged or superseded.
