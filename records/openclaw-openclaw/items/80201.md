## What This Changes

The issue reports that a subagent completion announce from `agent:main:main` can be delivered to a parallel Telegram-bound session instead of the originating parent session.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/80242 with redacted real Telegram or equivalent runtime proof, then close this issue only after the fix merges.

## Reproduction Assessment

Yes, source-reproducible: current main still passes the sandbox peer key to `sessions_spawn` while carrying the live parent as `runSessionKey`, then resolves completion origin through a child-first bound route. I did not run a live Telegram E2E reproduction.

## Solution Assessment
