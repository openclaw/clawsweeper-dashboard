## Summary

Keep this PR open: the runtime change is the right narrow fix for #70110, but the diff still exposes add-only Nextcloud Talk reactions while the public docs describe shared add/remove behavior and omit Nextcloud Talk from the `react` channel list.

## What This Changes

The branch adds a Nextcloud Talk `ChannelMessageActionAdapter`, registers it on the plugin, adds adapter/sender tests, and records the fix in the changelog.

## Best Possible Solution

Land this adapter after documenting Nextcloud Talk add-only reaction support, then close #70110 and mark #72679 as superseded by the canonical merge.

## Reproduction Assessment

Yes. On current main, Nextcloud Talk advertises reactions and has a sender, but no `actions.handleAction` is registered, so the shared dispatcher returns `null` for `react` before any HTTP request can be sent.

## Solution Assessment

Yes for the runtime fix. A plugin-owned action adapter is the narrow maintainable seam; the remaining safer completion is docs that say Nextcloud Talk supports adding reactions but not removal yet.
