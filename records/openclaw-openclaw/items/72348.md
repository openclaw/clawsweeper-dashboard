## Summary

Keep open: current main still has the Nextcloud Talk reaction dispatch gap, and this PR is a focused implementation candidate for the linked bug. The runtime approach looks correct, but the docs still need to describe Nextcloud Talk as add-only for shared reactions before merge.

## What This Changes

The PR adds and registers a Nextcloud Talk `react` message-action adapter, adds adapter and sender unit tests, and records the user-facing fix in `CHANGELOG.md`.

## Best Possible Solution

Land this PR or an equivalent adapter after the docs reflect Nextcloud Talk add-only reaction support, then close #70110 and mark #72679 as superseded by the canonical merge.

## Reproduction Assessment

Yes. On current main, a Nextcloud Talk `react` message action reaches shared dispatch with no plugin `actions.handleAction`, so the existing sender is unreachable and dispatch falls through as unsupported.

## Solution Assessment

Yes for the runtime fix. A plugin-owned `ChannelMessageActionAdapter` is the narrow maintainable seam; the safer completion is a docs update for the unsupported remove path.
