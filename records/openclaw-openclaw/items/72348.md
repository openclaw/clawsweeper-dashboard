## Summary

Keep open. Current main still lacks the Nextcloud Talk action adapter, and the PR is the right implementation direction for #70110; the remaining issue is a narrow docs follow-up because the newly exposed action is add-only while the public reaction/CLI docs do not mention Nextcloud Talk or its removal limitation.

## What This Changes

The PR adds and registers a Nextcloud Talk `ChannelMessageActionAdapter` for `react`, adds adapter and sender tests, and records the user-facing fix in the changelog.

## Best Possible Solution

Land this PR or an equivalent Nextcloud Talk-owned adapter after updating the public message/reaction docs for add-only Nextcloud Talk support, then close #70110 and mark the overlapping #72679 implementation as superseded by the canonical merge.

## Reproduction Assessment

Yes. On current main, a Nextcloud Talk `message` tool `react` action reaches `dispatchChannelMessageAction`, but the plugin has no `actions.handleAction`, so the existing reaction sender is never reachable from the shared action path.

## Solution Assessment

Yes for the runtime fix. A plugin-owned `ChannelMessageActionAdapter` is the narrow maintainable seam already used by other channels; the safer completion is to document that Nextcloud Talk now supports adding reactions but not reaction removal.
