## Summary

Keep open: this is a plausible bug-fix direction for the open zero-byte session-store report, but the current head still has merge-blocking correctness and security defects, and the terminal proof does not demonstrate the current branch against the real writer contract.

## What This Changes

The branch adds load-time session-store recovery from `.bak` and `.tmp` artifacts, recovery tests, a validation helper, and a changelog entry.

## Best Possible Solution

Keep a narrow recovery change that runs only after failed or zero-byte primary loads, matches actual current writer artifacts plus supported legacy artifacts, validates regular sibling candidates before reading, self-heals atomically with 0600, and includes credible live proof.

## Reproduction Assessment

Yes. Current main is source-reproducible by making `sessions.json` malformed or zero bytes: `loadSessionStore` catches the read/parse failure and returns an empty store instead of recovering or surfacing the failure.

## Solution Assessment

No. Load-time recovery is a reasonable narrow direction, but this diff restores valid empty stores, misses the actual fs-safe temp prefix, follows linked candidates, and rewrites the healed store non-atomically.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P2] Gate recovery on a failed primary read:** `src/config/sessions/store-load.ts:170`
  - body: This condition treats a valid `{}` store the same as a failed or zero-byte read. With a stale `.bak` nearby, an intentional reset revives old sessions, and the new valid-empty-store test fails.
  - confidence: 0.98
- **[P2] Match the actual fs-safe temp prefix:** `src/config/sessions/store-load.ts:43`
  - body: Current session writes call `writeTextAtomic` -> `replaceFileAtomic` without a `tempPrefix`; pinned fs-safe creates `.fs-safe-replace.<pid>.<uuid>.tmp`. This matcher only accepts `sessions.json.*.tmp` and `.sessions.json.*.tmp`, so actual current residual writer artifacts are ignored.
  - confidence: 0.98
- **[P2] Reject linked artifacts before reading them:** `src/config/sessions/store-load.ts:194-200`
  - body: The scan filters by name, then uses `statSync` and `readFileSync`, both of which follow symlinks. A matching symlink can import another readable JSON file as session data; validate `lstatSync` regular single-link siblings before parsing.
  - confidence: 0.91
- **[P2] Rewrite healed stores through an atomic private path:** `src/config/sessions/store-load.ts:225`
  - body: `writeFileSync` on the existing corrupt store is not atomic, and `mode` does not tighten permissions on an existing file. A crash mid-write can leave another truncated store; self-heal should use an atomic 0600 replacement path.
  - confidence: 0.9
