## Summary

Keep open: the linked bug is valid and the PR targets the right loader boundary, but the current patch still has blocking recovery-gating, artifact-validation, fresh-temp, and test-coverage defects.

## What This Changes

The PR adds load-time session-store recovery from `.bak` and temp artifacts, self-healing writeback, focused recovery tests, and a changelog entry.

## Best Possible Solution

Keep recovery in `loadSessionStore`, but make it a corrupt-existing-primary fallback with strict recovery-candidate validation, a defined fresh-temp policy, self-heal writeback, and focused regression tests.

## Reproduction Assessment

Yes. Source inspection shows current main returns an empty store when a zero-byte or malformed `sessions.json` is read, and https://github.com/openclaw/openclaw/issues/77783 provides a concrete Linux incident.

## Solution Assessment

No. The loader boundary is the right place, but this patch is not yet the best fix because it recovers outside proven corrupt-primary cases and trusts recovery artifacts too broadly.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P2] Gate recovery to existing corrupt primary stores:** `src/config/sessions/store-load.ts:176`
  - body: The recovery gate still includes `Object.keys(store).length === 0`, and missing files also set `primaryReadFailed`, so a valid empty `{}` store or a deleted primary can restore stale `.bak` data. The new tests for those cases would fail against this implementation.
  - confidence: 0.98
- **[P2] Validate backup artifacts before parsing:** `src/config/sessions/store-load.ts:182-183`
  - body: `sessions.json.bak` is read and parsed before any `lstat`, regular-file, symlink, or hardlink validation. Since this imports private session metadata as trusted recovery state, apply the same artifact validation before reading the backup.
  - confidence: 0.94
- **[P2] Apply the fresh-temp age guard:** `src/config/sessions/store-load.ts:218-221`
  - body: The PR body says fresh temp files are ignored to avoid racing active writers, but the code adds every matching temp candidate after basic stat checks. A transient primary failure can still import an in-flight fresh temp when it is the only candidate.
  - confidence: 0.88
- **[P2] Cover the actual fs-safe temp prefix in tests:** `src/config/sessions/store-load.recovery.test.ts:87-88`
  - body: These fixtures use `.sessions.json.<pid>.<uuid>.tmp`, while the PR matcher and pinned `@openclaw/fs-safe` helper use `.fs-safe-replace.<pid>.<uuid>.tmp`. The advertised fs-safe tests currently miss the intended branch and fail against the matcher.
  - confidence: 0.97
- **[P2] Require valid session entries before accepting recovery:** `src/config/sessions/store-load.ts:184`
  - body: `isSessionStoreRecord` accepts any object, so a backup with malformed entries can block tmp fallback and self-heal invalid session data. Use the added entry validator, or equivalent, before accepting a recovery candidate.
  - confidence: 0.86
