## Label Justifications

- `P2`: The issue affects external-channel session visibility for Codex runtime users but has a bounded Codex app-server fix surface.
- `impact:session-state`: The durable session transcript is delayed relative to the accepted inbound message, causing WebChat/Control UI session state to lag.
- `impact:message-loss`: Inbound channel messages are accepted but effectively hidden from the UI transcript until the turn finalizes, which looks like suppressed message delivery to operators.

## Summary

Keep open: current main still mirrors the Codex app-server user prompt only through the end-of-turn transcript snapshot, and the linked open fix PR uses closing syntax, so this issue should remain open until that PR lands or is rejected.

## What This Changes

Append the accepted Codex app-server user prompt to the durable session transcript immediately after a stable turn id exists so WebChat/Control UI can show external-channel inbound messages before turn completion.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84652, or an equivalent narrow Codex app-server fix that mirrors only the accepted prompt after turn acceptance using the existing `${turnId}:prompt` identity and final-mirror dedupe contract.

## Reproduction Assessment

Yes. Current main still builds the Codex user prompt inside the final `buildResult` snapshot and mirrors it only after turn completion, and the reporter supplied Feishu plus Telegram timing evidence that matches that source path.

## Solution Assessment

Yes. The narrow maintainable fix is to mirror the accepted prompt once a stable Codex turn id exists, using the existing mirror identity and idempotency path rather than adding channel-specific transcript writes.
