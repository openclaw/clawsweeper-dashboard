## Label Justifications

- `P2`: This is a normal-priority Codex runtime transcript visibility bug with a clear, limited implementation surface.
- `impact:message-loss`: Accepted inbound channel messages are not visible to WebChat/Control UI through the durable transcript until the turn ends.
- `impact:session-state`: The session transcript state and user-message timestamp lag behind the actual inbound message acceptance.

## Summary

Keep open: current main still mirrors the Codex user prompt only from the final transcript snapshot, and https://github.com/openclaw/openclaw/pull/84652 is an open closing PR that should be reviewed and landed before this issue closes.

## What This Changes

The issue asks Codex app-server turns to append the accepted inbound user prompt to the durable session transcript immediately instead of waiting for end-of-turn mirroring.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/84652; if it lands, close this issue as implemented by that PR.

## Reproduction Assessment

Yes. Source inspection shows current main creates the Codex user transcript row only in the completion snapshot, and the discussion includes Feishu plus Telegram timing evidence that matches that path.

## Solution Assessment

Yes. The narrow maintainable fix is to mirror only the accepted prompt once the Codex turn has a stable `turnId`, reusing the existing `${turnId}:prompt` identity and final-mirror dedupe.
