## Summary

Keep open: current main still mirrors the Codex app-server user prompt only after completion, and the related Telegram report confirms the same symptom outside Feishu. No open PR appears to close this specific inbound-transcript gap.

## What This Changes

The issue asks for external-channel Codex inbound user messages to be written to the durable session transcript immediately instead of only at turn finalization.

## Best Possible Solution

Add an early Codex app-server prompt-only transcript mirror after `turn/start` returns a stable turn id, using the existing `${turnId}:prompt` identity and idempotency scope; leave assistant/tool observer streaming to https://github.com/openclaw/openclaw/issues/82611.

## Reproduction Assessment

Yes at source level, though not via a live current-main Feishu run here. Starting a Codex app-server turn and inspecting transcript before completion should show no new user row because the current code awaits completion before building and mirroring the prompt snapshot.

## Solution Assessment

Yes. The narrow maintainable fix is to mirror only the prompt once a stable Codex turn id exists, using the existing mirror identity/dedupe contract instead of changing channel metadata recording or the broader external observer-stream policy.
