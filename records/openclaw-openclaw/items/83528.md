## What This Changes

## Best Possible Solution

Add a narrow Codex app-server early prompt mirror immediately after turn/start returns a stable turn id, reusing the existing `${turnId}:prompt` identity and thread-stable idempotency scope so the final mirror remains the canonical dedupe pass.
