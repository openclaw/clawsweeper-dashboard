## Summary

Keep open: current main still mirrors the Codex app-server user prompt only after the Codex turn completes, and the shared inbound channel record path only updates session metadata/route state, so WebChat/Control UI still lacks a durable inbound message during the turn.

## What This Changes

The issue requests that external-channel Codex sessions append the accepted inbound user message to the durable session transcript immediately instead of waiting for end-of-turn mirroring.

## Best Possible Solution

Add a narrow Codex app-server prompt mirror immediately after `turn/start` returns a stable turn id, using the existing `${turnId}:prompt` identity and thread-stable idempotency scope so the final mirror remains a deduped no-op for that prompt.

## Reproduction Assessment

Yes, at source level with supporting live shipped-version evidence. Start an external-channel Codex app-server turn and inspect the transcript before completion: current main records metadata first, while the user prompt is only built and mirrored after `completion` resolves.

## Solution Assessment

Yes, the proposed runtime-side early prompt mirror is the narrow maintainable fix. It uses the existing mirror identity/dedupe contract and avoids turning the generic channel metadata record path into a transcript writer.
