## Summary

Keep open: current main still records the inbound channel turn as session metadata first, but the Codex app-server user prompt is only appended to the transcript in the final mirror after turn completion, so WebChat/Control UI has no durable user row to display immediately.

## What This Changes

The issue requests immediate durable transcript mirroring of accepted inbound external-channel user prompts for Codex app-server sessions so WebChat/Control UI can show them before the assistant turn finishes.

## Best Possible Solution

Append the Codex app-server prompt immediately after `turn/start` returns a stable turn id, using the existing `${turnId}:prompt` mirror identity and thread-stable idempotency scope so the final mirror remains a deduped no-op for that prompt.

## Reproduction Assessment

Yes at source level, though not from a live Feishu run. Start a Codex app-server turn and inspect the session transcript before completion: current main records session metadata first, while the user prompt is only mirrored after `completion` resolves.

## Solution Assessment

Yes, the suggested direction is the narrow maintainable fix. Mirroring only the prompt after `turn/start` has a stable id uses the existing mirror identity/dedupe contract and avoids moving channel-owned record logic into core.
