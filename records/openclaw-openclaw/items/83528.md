## Summary

Keep open: current main still mirrors the Codex app-server inbound user prompt only after the turn completes, while the channel record path only updates session metadata/route state. The added Telegram evidence also confirms the delayed transcript symptom is not Feishu-specific.

## What This Changes

The issue requests immediate durable transcript writes for external-channel user messages in Codex-runtime sessions so WebChat and Control UI can show inbound messages before the assistant turn finishes.

## Best Possible Solution

Add a narrow Codex app-server prompt mirror immediately after turn/start returns a stable turn id, using the existing `${turnId}:prompt` identity and the same thread-stable idempotency scope so the final mirror remains a deduping no-op for that prompt.
