## What This Changes

The PR canonicalizes main-session requester/controller keys for subagent list/control callers and adds tool plus slash-command regression tests.

## Best Possible Solution

Normalize equivalent main-session requester/controller keys symmetrically at registry write and read boundaries, or make list/control lookups alias-compatible, with reciprocal raw-main and canonical-main regression coverage.

## Reproduction Assessment

Yes, at source level. A subagent run can be stored under one valid main-session key shape while list/control resolves an equivalent but different key, and current main compares the keys by exact string equality.

## Solution Assessment

No. The PR is pointed at the right alias boundary, but the current diff canonicalizes only read-side callers and does not prove or cover runs written under raw main aliases.
