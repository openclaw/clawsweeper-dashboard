## Summary

Keep this PR open: it targets a real subagent visibility bug linked from https://github.com/openclaw/openclaw/issues/75593, but the current patch only canonicalizes read-side keys and can still miss runs registered under the raw `main` alias.

## What This Changes

The PR canonicalizes main-session requester/controller keys for subagent list/control paths and adds tool and slash-command regression tests.

## Best Possible Solution

Normalize equivalent main-session requester/controller keys symmetrically at registry write and read boundaries, or make lookups alias-compatible, with reciprocal tests for raw-main and canonical-main records.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: a spawn registered under raw `main` can be listed through a canonicalized `agent:main:main` caller, and exact-key filtering drops the run. I did not execute tests because this review is read-only.

## Solution Assessment

No. The PR is aimed at the right boundary, but the read-side-only fix is not symmetric enough; writes must be canonicalized too, or reads must include equivalent legacy aliases.
