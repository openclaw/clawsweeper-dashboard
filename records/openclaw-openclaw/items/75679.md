Keep open. The PR targets a real subagent list/control bug, but the patch remains one-directional: raw `main` requester/controller writes from spawn paths can still miss canonicalized reads, and the user-facing fix lacks the required changelog entry.

## What This Changes

The PR canonicalizes main-session requester/controller keys for subagent list/control read paths and adds tool and slash-command regression tests.

## Best Possible Solution

Normalize requester/controller keys symmetrically at registry write and read boundaries, or make lookups alias-compatible for legacy raw-main records, with reciprocal tests and a single-line changelog entry.

Yes. Source inspection gives a high-confidence path: spawn or register a subagent with `agentSessionKey: "main"` under per-sender config, then list/control through a path resolving the caller to `agent:main:main`; exact-key matching misses the raw-main record.

## Solution Assessment

No. The PR fixes the canonical-write/read-from-raw direction, but the maintainable fix must also preserve raw-main spawned records by canonicalizing writes or querying equivalent aliases.
