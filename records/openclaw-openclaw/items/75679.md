## Summary

Keep open. This PR addresses the valid #75593 subagent-list bug, but the diff only canonicalizes read-side list/control keys, so raw `main` writes can still be missed; it also needs the required user-facing changelog entry.

## What This Changes

This PR canonicalizes requester keys at subagent list/control read boundaries and adds tests for listing canonical main-owned runs through `main` aliases.

## Best Possible Solution

Normalize requester/controller keys symmetrically at registry write and read boundaries, or make read lookups alias-compatible for legacy raw-main records, with reciprocal regression tests and a single-line changelog fix entry.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to register or spawn a subagent with `agentSessionKey: "main"` under per-sender config, then list/control through a path that resolves the caller to `agent:main:main`; exact-key filtering misses the raw-main record.

## Solution Assessment

No. The proposed patch fixes the canonical-write/read-from-raw direction, but the narrower maintainable fix must also preserve raw-main spawned records or canonicalize writes before exact-key storage.
