## Summary

Keep open: current main and v2026.6.5 route plugin-owned Slack modal submissions and closes, but they still do not expose a supported open/update/close/validation modal action or plugin API. A prior modal lifecycle action PR was closed unmerged, and the open Slack shortcut issue is related trigger work rather than a replacement for this modal lifecycle contract.

## What This Changes

Request to add first-class Slack modal lifecycle primitives so workflows or plugins can open, update, close, validate, and receive structured Slack modal submissions.

## Best Possible Solution

Define a Slack plugin-owned modal lifecycle contract that reuses current plugin modal submission routing, supports open/update/close plus validation ACK responses, documents ownership and callback routing, and coordinates with Slack shortcut trigger work where needed.

## Reproduction Assessment

Not applicable: this is a missing workflow/plugin API surface, not a broken existing behavior report. Source inspection verifies the absence of modal lifecycle actions, but there is no failing current behavior to reproduce.

## Solution Assessment

Unclear until maintainers choose the API boundary. The likely maintainable direction is a Slack plugin-owned contract built on current callback routing, not an untyped raw Slack API escape hatch.
