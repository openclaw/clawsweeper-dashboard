## Summary

Keep open: current main and the latest release already handle Slack modal submission/close callbacks for plugin-owned modals, but they do not expose the requested first-class workflow surface for opening, updating, closing, and validating Slack modals.

## What This Changes

The issue requests first-class Slack modal support for structured interactive workflows using Slack Block Kit and `views.open`/`views.update`/`views.close`.

## Best Possible Solution

Design a Slack-plugin-owned modal workflow surface that reuses the existing modal submission routing, and extend shared plugin SDK contracts only if maintainers want cross-plugin modal workflow semantics.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source inspection shows current modal submission handling exists, while first-class modal open/update/close support is still missing.

## Solution Assessment
