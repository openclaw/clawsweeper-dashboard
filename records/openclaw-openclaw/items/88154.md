## Summary

Keep open: current main and the latest release support plugin-owned Slack modal submission/close routing, but they still do not expose the requested first-class workflow/plugin surface for opening, updating, closing, options callbacks, or validation ACKing Slack modals.

## What This Changes

Request a Slack plugin/workflow API for native modal lifecycle operations and structured modal submission handling.

## Best Possible Solution

Define a Slack plugin-owned modal lifecycle contract that reuses current callback routing, supports open/update/push/close plus validation ACK and options callbacks, documents ownership, and ships focused tests plus live Slack proof.

## Reproduction Assessment

Not applicable: this is a missing workflow/plugin API surface, not a broken existing behavior report. Source inspection verifies the current receive-side behavior and the absence of modal lifecycle actions.

## Solution Assessment

Unclear until maintainers choose the API boundary. A Slack plugin-owned contract built on current callback routing looks maintainable, while a raw Slack API escape hatch would be a weaker long-term surface.
