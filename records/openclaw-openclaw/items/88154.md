## Summary

Keep open: current main and v2026.6.5 handle Slack modal submission/close callbacks for OpenClaw-routed or plugin-owned modals, but they still do not expose the requested first-class workflow/plugin API for opening, updating, closing, and validating modals.

## What This Changes

Request a first-class Slack modal lifecycle surface for workflows/plugins to open, update, close, validate, and receive structured Slack modal submissions.

## Best Possible Solution

Define a Slack plugin-owned modal contract that reuses the existing modal lifecycle routing, supports open/update/close plus validation ACK responses, documents ownership, and keeps core plugin-agnostic.

## Reproduction Assessment

Not applicable: this is a missing workflow/plugin capability rather than a broken existing behavior report. Source inspection shows current main handles modal submissions/closes but lacks the requested lifecycle action surface.

## Solution Assessment

Unclear until maintainers choose the API boundary. The likely maintainable path is a Slack plugin-owned or Plugin SDK modal contract that builds on existing callback routing, not an ad hoc raw Slack API escape hatch.
