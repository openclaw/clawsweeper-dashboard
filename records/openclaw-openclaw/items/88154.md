## Summary

Keep open: current main and v2026.6.5 handle plugin-owned Slack modal submission/close callbacks, but they still do not expose the requested first-class workflow/plugin API for opening, updating, closing, validating, and owning Slack modals.

## What This Changes

The issue requests first-class Slack modal workflow/plugin support for opening, updating, closing, validating, and routing native Slack modal submissions.

## Best Possible Solution

Define a Slack plugin-owned modal lifecycle contract that builds on the existing modal callback routing, supports open/update/push or documented close semantics plus validation ACK responses, and ships with docs, regression tests, and real Slack proof.

## Solution Assessment

Unclear until maintainers choose the API boundary. The likely maintainable direction is a Slack plugin-owned or Plugin SDK modal contract that reuses existing callback routing, not an ad hoc raw Slack API escape hatch.
