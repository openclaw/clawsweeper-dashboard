## Summary

Keep open: current main and v2026.6.9 support Slack modal submission/close routing into plugin handlers, but they still do not expose the requested first-class Slack modal lifecycle surface for opening, updating, closing, options callbacks, or validation ACK behavior.

## What This Changes

The issue requests first-class Slack modal lifecycle support for workflows/plugins, including open/update/close actions, modal submissions, validation ACKs, and documentation.

## Best Possible Solution

Define a Slack plugin-owned modal lifecycle contract that reuses current callback routing, supports open/update/push/close plus validation ACK and options callbacks, documents ownership, and ships focused tests plus live Slack proof.

## Reproduction Assessment

Not applicable: this is a missing workflow/plugin API surface, not a broken existing behavior report. Source, docs, related PRs, and latest-release inspection verify the current receive-side behavior and the absence of outbound modal lifecycle actions.

## Solution Assessment

Unclear until maintainers choose the API boundary. A Slack plugin-owned contract built on current callback routing looks maintainable, while a raw Slack API escape hatch would be a weaker long-term surface.
