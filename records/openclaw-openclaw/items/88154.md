## Summary

Keep open: current main and latest release support plugin-owned Slack modal submission/close routing, but they still do not expose the requested first-class workflow/plugin surface for opening, updating, closing, and validation-ACKing Slack modals.

## What This Changes

The issue requests Slack modal lifecycle support so workflows/plugins can open, update, close, validate, and receive native Slack modal submissions.

## Best Possible Solution

Define a Slack plugin-owned modal lifecycle contract that reuses current modal callback routing, supports open/update/push/close plus validation ACK responses, documents ownership, and includes focused tests plus live Slack proof.

## Reproduction Assessment

Not applicable: this is a missing workflow/plugin API surface, not a broken existing behavior report. Source inspection verifies the current receive-side behavior and the absence of modal lifecycle actions.

## Solution Assessment

Unclear until maintainers choose the API boundary. A Slack plugin-owned contract built on current callback routing looks like the maintainable direction, while a raw Slack API escape hatch would be weaker.
