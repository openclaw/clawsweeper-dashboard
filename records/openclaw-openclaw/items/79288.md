## What This Changes

The branch adds a 24-hour stale-running reconciliation path in task maintenance, updates background task docs, and adds regression tests for stale child-session-backed running tasks.

## Best Possible Solution

Keep the stale-running reconciliation in task maintenance, but make the 24-hour fresh-evidence check reuse the existing runtime-specific backing semantics for legacy CLI tasks and document the shipped fix in the changelog.

## Reproduction Assessment

Yes, source-reproducible. Current main can leave stale `running` rows active when an old persisted child/session row is still treated as backing state, and the PR regression is visible from the new CLI fresh-evidence branch.

## Solution Assessment

No, not as-is. Task maintenance is the right surface, but the stale-age check should preserve legacy CLI child-session freshness semantics and add the required changelog entry before merge.
