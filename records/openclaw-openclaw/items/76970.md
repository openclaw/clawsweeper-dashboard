## Summary

Keep this PR open: it targets a real source-reproducible stale foreground auto-reply bug, but the current head still has a stale-delivery window after async delivery hooks.

## What This Changes

The PR adds a shared foreground auto-reply freshness fence in `src/auto-reply/dispatch.ts`, regression coverage in `src/auto-reply/dispatch.freshness.test.ts`, and a changelog entry for #76905.

## Best Possible Solution

Keep the shared dispatcher fence, recheck supersession immediately after async `beforeDeliver` hook resolution, and cover that hook race in a focused regression.

## Reproduction Assessment

Yes, source-reproducible: WhatsApp records stable session/account/origin target fields and routes through the shared buffered dispatcher, while current main has no generation guard. The PR models the same-session race; this read-only review did not run tests.

## Solution Assessment

No, not yet: the shared dispatcher fence is the right maintainable fix, but it must recheck freshness after `configuredBeforeDeliver` resolves before allowing delivery.
