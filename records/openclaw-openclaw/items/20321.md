## Summary

Keep open: current main still lacks the requested Telegram `members` group policy, the related implementation PRs are closed unmerged, and the external chat-only-trigger plugin is useful prior art rather than an equivalent bundled all-members authorization path.

## What This Changes

Add a Telegram `groupPolicy: "members"` mode so new Telegram groups can be authorized by trusted participants instead of manually allowlisting every group chat ID.

## Best Possible Solution

Land an explicit bundled Telegram `members` policy with config type/schema/generated metadata, inbound and native-command authorization, fail-closed member verification, docs, security-audit behavior, and focused tests while preserving existing policies.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Static current-main inspection shows `members` is absent from config, schema, generated metadata, Telegram inbound/native-command authorization, docs, and security audit handling.

## Solution Assessment

Unclear until maintainers define the authorization contract. If accepted, the durable implementation belongs in the bundled Telegram plugin plus shared config/docs/security surfaces; the chat-only-trigger plugin is helpful prior art but not equivalent.
