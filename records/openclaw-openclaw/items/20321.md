## Summary

Keep open: current main and the latest release still lack the requested Telegram `groupPolicy: "members"` policy. The closed implementation attempts and the community chat-only-trigger plugin are useful prior art, but the remaining work changes Telegram group authorization and security-audit semantics, so it needs maintainer/security direction before implementation.

## What This Changes

Add a Telegram group policy that authorizes new groups based on trusted current members instead of requiring each group chat ID to be allowlisted.

## Best Possible Solution

Define and land a bundled Telegram `members` policy after maintainer approval, with fail-closed membership verification, config/schema/generated metadata, inbound and native-command authorization, security-audit behavior, docs, and focused tests.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Static inspection of current main and the latest release gives high confidence that `members` is absent from the config contract, Telegram authorization paths, docs, and security audit.

## Solution Assessment

Unclear until maintainers define the authorization contract. If accepted, the maintainable path is a bundled Telegram channel policy rather than a ClawHub-only workaround because the behavior changes core channel access control and audit semantics.
