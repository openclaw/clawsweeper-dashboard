## Summary

Keep this open. Current main and the latest release still support only `open`, `disabled`, and `allowlist` for Telegram group policy; the closed implementation attempts and the Chat Only Trigger plugin are useful prior art or workarounds, but they do not provide the requested bundled, fail-closed all-members authorization mode.

## What This Changes

Request a new Telegram `groupPolicy: "members"` mode that authorizes new groups when every current participant is trusted by user allowlist.

## Best Possible Solution

Define a Telegram-owned, opt-in `members` policy with fail-closed all-participant verification, matching inbound and native-command enforcement, config/schema/docs/audit coverage, and no silent cross-channel normalization to open access.

## Reproduction Assessment

Not applicable: this is a request for a new authorization mode, not a broken existing contract. Current-main source inspection gives high-confidence evidence that the requested mode is absent.

## Solution Assessment

No: the earlier PRs are useful design input, but this should not be landed unchanged without an explicit Telegram-specific fail-closed contract and owner-boundary decision.
