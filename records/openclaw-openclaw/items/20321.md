## Summary

Keep open: current main and the latest release still do not include the requested Telegram `groupPolicy: "members"` mode. The closed implementation attempts and the chat-only-trigger plugin are useful prior art, but this remains a security-sensitive Telegram authorization feature that needs maintainer contract approval before implementation.

## What This Changes

Add a Telegram group policy that authorizes new groups based on trusted current members rather than manually allowlisting each group chat ID.

## Best Possible Solution

Define and land a bundled Telegram `members` policy with config/schema/generated metadata, inbound and native-command authorization, fail-closed membership verification, security-audit behavior, docs, and focused tests after maintainer approval of the access-control contract.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Static inspection of current main gives high confidence that `members` is absent from config, schema, Telegram inbound authorization, native-command authorization, and release behavior.

## Solution Assessment

Unclear until maintainers define the authorization contract. If accepted, the maintainable path is a bundled Telegram policy plus config/docs/security updates; the linked community plugin is useful prior art but not equivalent.
