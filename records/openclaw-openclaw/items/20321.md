## Summary

Current main and v2026.6.5 still lack the requested Telegram all-members authorization mode; the closed implementation attempts and the Chat Only Trigger plugin are useful prior art, but neither provides the bundled fail-closed behavior requested here. Keep this open for an explicit maintainer/security decision because the feature changes who may invoke an agent and how Telegram API uncertainty is handled.

## What This Changes

Add a Telegram group policy that automatically admits newly created groups only when every current participant is present in a trusted user allowlist.

## Best Possible Solution

Define a Telegram-owned, opt-in `members` policy whose canonical invariant is that every participant is positively accounted for by stable trusted IDs, with fail-closed API/cache semantics, clear diagnostics, and identical enforcement for inbound messages and native commands while preserving all existing policy defaults.

## Reproduction Assessment

Not applicable: this is a request for a new authorization mode, and current main provides a high-confidence source check showing that the mode is absent rather than a failing existing contract.

## Solution Assessment

No, the earlier proposals are useful design input but should not be adopted unchanged without a maintainer-approved Telegram-specific failure contract; in particular, cross-channel normalization to `open` would be an unsafe competing semantic.

## Risks / Open Questions

- The security contract is unresolved: API errors, missing privileges, unknown or anonymous participants, nonnumeric allowlist entries, bots, membership-change events, cache TTL/invalidation, and member-count mismatches must all have explicit fail-closed behavior.
- Adding `members` to the shared GroupPolicy union could accidentally create unsupported semantics in non-Telegram channels; repository ownership rules favor a Telegram-owned policy surface or an explicitly designed generic capability rather than silently normalizing it to `open`.
- The proposed behavior can stop an existing group workflow when Telegram membership verification becomes unavailable, so defaults, audit messaging, diagnostics, and operator recovery need product approval.
