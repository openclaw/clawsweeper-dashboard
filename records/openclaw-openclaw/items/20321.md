## Summary

Keep this issue open. Current main and v2026.6.6 still lack the requested Telegram all-members authorization mode, while the prior implementation PRs are closed unmerged and the community plugin provides a related but materially different chat-only workaround. The remaining work needs explicit product and security decisions because Telegram membership lookup failures, stale caches, and command-path inconsistencies could change the agent access boundary.

## What This Changes

Add a Telegram group policy that permits newly created groups only when every current participant is a trusted user.

## Best Possible Solution

Define a Telegram-owned `members` policy with fail-closed all-participant verification, bounded and correctly invalidated caching, consistent inbound and native-command enforcement, accurate security-audit treatment, updated config metadata/docs, and focused tests while preserving all existing policies.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new capability request. Source and release inspection do provide high-confidence confirmation that the requested policy is currently absent.

## Solution Assessment

No final implementation path is approved yet. A narrow Telegram-only policy is preferable to the prior cross-channel normalization approaches, but maintainers must first define the membership and fail-closed security contract.
