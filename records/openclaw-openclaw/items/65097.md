## Summary

Keep open. Current main still has the reported Dream Diary language gap: the narrative prompt and prompt builder are English-only, the narrative runner and phase call sites only carry timezone/model, the resolved Dreaming config has no language or locale field, the strict memory-core manifest would reject the requested key, and public docs do not expose a language setting. The related closed #69354 is duplicate user signal, while open #69533 is broader custom-prompt work and does not supersede this narrower config request.

## What This Changes

Add a language or locale option under memory-core Dreaming config so Dream Diary narrative generation can produce narrative output in the user's preferred language.

## Best Possible Solution

Implement this in memory-core as a documented, schema-backed explicit Dream Diary language hint, thread it through the memory-host config resolver into light/REM/deep narrative generation, add a concise output-language directive in both subagent and fallback prompt paths, and leave automatic locale inference plus full prompt-template customization to separate product work such as #69533.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a runtime bug reproduction because this is a feature request; the gap is high-confidence from source inspection because current schema, resolver, call sites, prompts, tests, and docs have no language or locale carrier.

## Solution Assessment

Is this the best way to solve the issue? Yes, with scope narrowed to an explicit configured language hint. That is the smallest maintainable path; `auto` inference and user prompt templates need separate design because they affect broader context and prompt contracts.
