## Summary

Keep open. Current main still has no Dream Diary language or locale configuration: the prompt is fixed English, the narrative API and phase call sites carry only timezone/model, the strict memory-core schema omits the requested key, and docs expose only enabled/frequency/model.

## What This Changes

Request to add a schema-backed Dreaming narrative output language or locale setting for memory-core Dream Diary generation.

## Best Possible Solution

Add one documented, schema-backed explicit Dream Diary output-language hint in memory-core and keep automatic locale inference plus full prompt-template customization as separate follow-up work.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a crash-style bug, but the feature gap is high-confidence from source inspection of the schema, resolver, prompts, call sites, tests, and docs.

## Solution Assessment

Is this the best way to solve the issue? Yes, if scoped to an explicit configured language hint; `auto` inference and custom prompt templates are broader prompt/context contracts and should stay separate from this first step.
