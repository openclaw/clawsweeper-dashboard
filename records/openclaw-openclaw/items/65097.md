## Summary

Keep open. Current main still lacks a Dreaming language/locale config path: the Dream Diary prompt is fixed English, the narrative runner and phase callers carry only timezone/model, the strict memory-core schema rejects language/locale keys, and docs only document the existing settings. Related https://github.com/openclaw/openclaw/issues/69354 was closed as a duplicate of this issue, while https://github.com/openclaw/openclaw/issues/69533 and https://github.com/openclaw/openclaw/issues/79223 cover broader prompt-customization variants rather than superseding this narrower canonical request.

## What This Changes

Add a memory-core Dreaming language or locale config so Dream Diary narrative generation can produce non-English diary entries.

## Best Possible Solution

Add one documented explicit Dream Diary language hint in memory-core, wire it through the resolved Dreaming config and narrative prompt paths, and leave auto-detection plus full prompt-template customization to separate follow-up design.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a runtime bug, but the feature gap is high-confidence from source inspection: current main has no language/locale schema, resolver, runner, prompt, docs, or test surface for Dream Diary output.

## Solution Assessment

Is this the best way to solve the issue? Yes if scoped to an explicit `dreaming.language` hint; `auto` inference and custom prompt templates are broader contracts and should remain separate from the first implementation.
