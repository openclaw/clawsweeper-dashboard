## Summary

Keep open. Current main still has the reported Dream Diary language gap: the narrative prompt is English-only, runtime plumbing carries timezone/model but no language hint, strict memory-core config schema has no language or locale key, and docs expose only enabled/frequency/model; #69354 is duplicate user signal and #69533 is broader custom-prompt work rather than a replacement for this narrower config request.

## What This Changes

Add a configurable language or locale option for memory-core Dreaming narrative output so Dream Diary entries can use the user's preferred language.

## Best Possible Solution

Add a documented, schema-backed explicit Dream Diary output-language hint in memory-core, and leave automatic inference plus custom prompt templates to separate follow-up work such as #69533.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a crash-style runtime bug, but the feature gap is high-confidence from source inspection: current schema, resolver, prompts, call sites, tests, and docs have no accepted language or locale carrier.

## Solution Assessment

Is this the best way to solve the issue? Yes, if scoped to an explicit configured language hint. That is the narrowest maintainable first step; `auto` inference and full prompt customization should stay separate because they affect broader prompt and context contracts.
