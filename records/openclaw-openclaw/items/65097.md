## Summary

Keep open. Current main still has no Dreaming language or locale config path: the prompt is English, the runner only carries timezone/model, the manifest schema rejects language/locale keys, and docs do not document a language setting.

## What This Changes

Request adds a language/locale setting under memory-core Dreaming config so Dream Diary narrative generation can use a preferred output language.

## Best Possible Solution

Add a documented explicit `dreaming.language` hint in memory-core, thread it through config resolution and Dream Diary generation, and leave full prompt-template customization to https://github.com/openclaw/openclaw/issues/69533.

## Reproduction Assessment

Not applicable as a runtime bug. Source inspection gives a high-confidence feature-gap proof because current main has no schema, resolver, docs, prompt, or narrative call path for a Dream Diary language hint.

## Solution Assessment

Yes, if narrowed to an explicit `dreaming.language` hint. `auto` language inference and arbitrary prompt templates are broader contracts and should remain separate unless maintainers choose to expand scope.
