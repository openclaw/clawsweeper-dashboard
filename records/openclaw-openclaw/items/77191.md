## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Fallback missing default agent IDs to main:** `src/cron/service/ops.ts:293`
  - body: When `CronServiceState` is constructed without `deps.defaultAgentId`, `normalizeOptionalAgentId(state.deps.defaultAgentId)` returns `undefined`; jobs with no `agentId` then get `effectiveOwner === ""` and are excluded from `cron.list({ agentId: "main" })`. Other cron paths treat a missing default as `main`, so embedded/non-gateway callers can still lose default-agent jobs. Use the same `normalizeAgentId`/`DEFAULT_AGENT_ID` fallback before comparing.
