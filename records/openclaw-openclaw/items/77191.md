## What This Changes

The PR adds an optional `agentId` filter to `cron.list`/`listPage`, threads it through the gateway protocol and server handler, refreshes Swift protocol models, and adds regression tests plus a changelog entry.

## Best Possible Solution

Land the additive gateway-side filter after it compares requested and stored owners through the existing agent-id normalizer, while leaving CLI `--agent` and tool-layer auto-fill as separate follow-ups.

## Reproduction Assessment

Yes. Source inspection of current main shows `cron.list`/`listPage` has no `agentId` parameter or filter, and the current PR diff shows the new comparison uses raw trimmed strings while create/update canonicalize agent IDs.

## Solution Assessment

No, not yet. The API boundary is the narrow additive direction for #77118, but the implementation should normalize the requested filter and effective owner with `normalizeAgentId` before comparing.
