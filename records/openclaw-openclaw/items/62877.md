## Summary

Keep open: current main still emits the raw `/btw <side question>` usage text, and the direct outbound sanitizer strips the placeholder before delivery on plain-text/markdown-style channel paths. This is a narrow producer-side bug with a clear fix and regression-test path.

## What This Changes

The issue requests markdown-safe `/btw` missing-question usage text so the `<side question>` placeholder is not stripped by markdown or HTML-rendering channels.

## Best Possible Solution

Use a non tag-shaped `/btw` usage placeholder at the producer, add a regression that the delivered or sanitized usage keeps the placeholder visible, and add a changelog entry.

## Reproduction Assessment

Yes. On current main, `/btw` without a question reaches the empty-question branch returning `BTW_USAGE`, and importing the current sanitizer shows `Usage: /btw <side question>` is transformed to `Usage: /btw `.

## Solution Assessment

Yes. A producer-owned usage-string fix is the narrowest maintainable path; backticks alone are not sufficient unless sanitizer behavior changes, because the current sanitizer still strips `<side question>` inside inline-code text.
