## Summary

Keep open: current main still emits the raw `Usage: /btw <side question>` text for an empty `/btw`, and the shared plain-text sanitizer can strip the placeholder as an HTML-like tag before delivery.

## What This Changes

The issue asks for slash-command usage placeholders, especially `/btw <side question>`, to survive markdown or HTML-rendering channel delivery.

## Best Possible Solution

Emit producer-owned usage text that contains no raw tag-shaped placeholder, update the focused tests and changelog, and preserve sanitizer autolink behavior unless a narrowly tested sanitizer improvement is needed.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: `/btw` with no question returns the raw usage string, and the outbound sanitizer path can remove `<side question>` before direct channel delivery; the issue also reports a live Dingtalk reproduction.

## Solution Assessment

Yes. Fixing the producer text is the narrowest maintainable path because the producer knows this is a placeholder, while channel-wide escaping would risk breaking valid autolinks and platform formatting.
