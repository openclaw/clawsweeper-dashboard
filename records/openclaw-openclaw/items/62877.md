## Summary

Keep open: current main still emits the raw `/btw <side question>` usage text for an empty `/btw`, and the focused test still locks that unsafe output. The shared outbound sanitizer also strips tag-shaped angle-bracket text while preserving autolinks, so this remains a narrow producer-side channel-rendering bug.

## What This Changes

Make the missing-question `/btw` usage text safe for markdown/HTML-rendering channels so the placeholder is not stripped.

## Best Possible Solution

Change the BTW producer text so no raw tag-shaped placeholder reaches channel sanitizers or markdown renderers, update the focused test and changelog, and preserve the shared sanitizer's autolink behavior unless a separately tested sanitizer improvement is required.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: `/btw` with no side question reaches the empty-question branch and returns the raw usage string, while the outbound sanitizer strips tag-shaped angle-bracket text.

## Solution Assessment

Yes, a producer-owned usage-string fix is the narrowest maintainable direction. The implementation should avoid relying on backticks alone around `<side question>` unless it also updates and tests sanitizer behavior for inline code spans.
