## Summary

Keep open: current main still emits the raw tag-shaped `/btw <side question>` usage text, and the shared markdown/plain-text delivery path still strips unknown HTML-like tags. A referenced fix commit exists in the issue timeline, but it is not on current main and no open closing PR was found.

## What This Changes

The issue asks OpenClaw to keep the `/btw` no-question usage placeholder visible in markdown/plain-text channels instead of emitting a tag-shaped `<side question>` token that renderers strip.

## Best Possible Solution

Change the producer-side `/btw` usage text to a non tag-shaped placeholder, add regression coverage through the command and sanitizer/delivery path, and add a changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows `/btw` with no question returns `BTW_USAGE` containing `<side question>`, and the shared sanitizer strips unknown HTML-like tags after preserving autolinks.

## Solution Assessment

Yes. A producer-owned non tag-shaped usage string is the narrowest maintainable fix; channel-wide escaping would break legitimate autolinks, and backticks alone are not enough unless sanitizer behavior also changes.
