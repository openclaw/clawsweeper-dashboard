## Summary

Keep open. Current main still omits `rerank:false` for QMD v2 mcporter `query` calls when OpenClaw is in `search` or `vsearch` mode, and the PR is a narrow, well-proven fix for that remaining bug.

## What This Changes

The PR adds `rerank:false` to memory-core's QMD v2 mcporter `query` call arguments for `search` and `vsearch` modes and adds focused unit coverage for `vsearch`.

## Best Possible Solution

Land this narrow call-argument fix, or an equivalent maintainer-owned patch, so QMD mcporter `search` and `vsearch` keep their lexical/vector-only contract under the unified `query` tool.

## Reproduction Assessment

Yes, by source inspection. Current main builds unified QMD `query` MCP call args for `search` and `vsearch` without `rerank:false`, while upstream QMD's `query` tool defaults `rerank` to true.

## Solution Assessment

Yes. Setting `rerank:false` at the unified QMD v2 mcporter call site is the narrowest maintainable fix because direct CLI `search`/`vsearch` already use non-reranking QMD commands and full `query` mode keeps its config-controlled rerank behavior.
