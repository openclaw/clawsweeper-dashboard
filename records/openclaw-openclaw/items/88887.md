## Summary

This PR is still necessary: current main has the query-mode rerank toggle from https://github.com/openclaw/openclaw/pull/90304, but `search` and `vsearch` still reach QMD's unified MCP `query` tool without overriding QMD's default `rerank:true`. The patch is a focused fix with sufficient terminal proof and no blocking review findings.

## What This Changes

This PR adds `rerank:false` to memory-core's QMD v2 mcporter `query` call arguments for `search` and `vsearch` modes and adds focused vsearch argument coverage.

## Best Possible Solution

Land this focused runtime/test fix after required checks are green, keeping query-mode rerank configuration and non-query QMD tool fallbacks intact.

## Reproduction Assessment

Yes. By source inspection, configure QMD mcporter with `searchMode: "search"` or `"vsearch"`; current main routes to QMD's unified `query` MCP tool without `rerank:false`, while upstream QMD defaults `rerank` to true.

## Solution Assessment

Yes. Setting `rerank:false` at the unified QMD v2 mcporter call site is the narrowest maintainable fix because direct CLI and QMD v1 tool paths already use non-reranking search/vector tools, and full `query` mode keeps its config control.
