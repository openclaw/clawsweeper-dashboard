## Summary

Keep open. The PR still carries a focused memory-core bug fix: current main has a query-mode `memory.qmd.rerank` toggle, but `search` and `vsearch` mcporter calls still omit `rerank:false` when using QMD's unified `query` tool.

## What This Changes

The PR adds `rerank:false` to QMD unified mcporter `query` calls for `search` and `vsearch` modes and adds a unit test for the vsearch call args.

## Best Possible Solution

Land the narrow mcporter argument fix after normal maintainer and current-base CI review, keeping full `query` mode governed by the existing `memory.qmd.rerank` behavior.

## Reproduction Assessment

Yes, source-level. On current main, QMD mcporter `searchMode: "search"` or `"vsearch"` still sends unified `query` args without `rerank`, while upstream QMD defaults that field to true.

## Solution Assessment

Yes. Setting `rerank:false` at the unified QMD v2 mcporter call site is the narrowest fix because direct CLI and QMD v1 tool paths already avoid this unified reranker path, and full `query` mode remains separately configurable.
