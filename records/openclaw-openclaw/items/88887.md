## Summary

Keep open for maintainer review and merge: current main still omits `rerank:false` for QMD v2 mcporter `search`/`vsearch` calls, and this PR applies the narrow dependency-backed fix with focused coverage and sufficient live proof.

## What This Changes

The PR adds `rerank:false` to QMD v2 mcporter `query` calls for `search` and `vsearch` modes and adds a vsearch argument test.

## Best Possible Solution

Merge the narrow mcporter argument fix after normal maintainer and CI review, keeping full query-mode no-rerank as the already-merged config behavior from the related PR.

## Reproduction Assessment

Yes. Source inspection reproduces current-main behavior: `search` and `vsearch` build lexical/vector-only QMD v2 searches, but the mcporter unified `query` call omits `rerank:false` and QMD defaults that option to true.

## Solution Assessment

Yes. Setting `rerank:false` at the unified QMD v2 mcporter call site for `search` and `vsearch` is the narrowest maintainable fix because direct CLI and QMD v1 tool paths already avoid this reranker path, while full `query` mode keeps its separate config-controlled behavior.
