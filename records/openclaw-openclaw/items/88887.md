## Summary

Keep open: this is a focused, still-needed bug fix for the QMD mcporter warm path. Current main sends QMD v2 `query` calls for `search`/`vsearch` without `rerank`, QMD defaults that field to true, and the PR adds the narrow `rerank:false` argument plus targeted coverage with no blocking patch defects found.

## What This Changes

The PR adds `rerank:false` to QMD v2 mcporter `query` calls for `search` and `vsearch` modes and adds a unit test that captures the vsearch call arguments.

## Best Possible Solution

Land this narrow mcporter argument fix after normal maintainer and CI review, while leaving full query-mode no-rerank as separate product/config work tracked at https://github.com/openclaw/openclaw/issues/61834.

## Reproduction Assessment

Yes. Source inspection reproduces the current-main path: configure QMD mcporter with `searchMode: "search"` or `"vsearch"`, OpenClaw sends unified `query` args without `rerank`, and QMD defaults `rerank` to true.

## Solution Assessment

Yes. Setting `rerank:false` at the unified QMD v2 mcporter call site is the narrowest maintainable fix because direct CLI/v1 tool paths already avoid this reranker path and full `query` mode keeps its existing behavior.
