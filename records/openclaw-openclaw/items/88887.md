## Summary

Keep this PR open: current main and the latest release still omit `rerank:false` for QMD mcporter `search`/`vsearch`, while the PR applies the narrow runtime/test fix and has sufficient live-output proof.

## What This Changes

The PR adds `rerank:false` to memory-core's QMD unified mcporter `query` arguments for `search` and `vsearch` modes, plus focused unit coverage for `vsearch`.

## Best Possible Solution

Land this narrow runtime/test fix, or an equivalent maintainer-owned patch, so QMD mcporter `search` and `vsearch` preserve their lexical/vector-only contract on current QMD.

## Reproduction Assessment

Yes, by source inspection. Current main builds QMD unified `query` MCP call args for `search` and `vsearch` without `rerank:false`, while current upstream QMD's `query` tool defaults `rerank` to true.

## Solution Assessment

Yes. Setting `rerank:false` at the unified QMD mcporter call site for `search` and `vsearch` is the narrowest maintainable fix because direct CLI modes already use dedicated commands and full `query` mode keeps its config-controlled behavior.
