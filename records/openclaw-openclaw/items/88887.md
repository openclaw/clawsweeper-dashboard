## Summary

Keep open. Current main and the merged rerank-toggle work at https://github.com/openclaw/openclaw/pull/90304 cover configurable no-rerank for `searchMode: "query"`, but `search` and `vsearch` still reach QMD's unified MCP `query` tool without `rerank:false`; this branch is the focused runtime/test fix and I found no blocking patch defect.

## What This Changes

This PR adds `rerank:false` to memory-core's QMD v2 mcporter `query` call arguments for `search` and `vsearch` modes and adds focused vsearch argument coverage.

## Best Possible Solution

Close this PR as superseded by https://github.com/openclaw/openclaw/pull/90304.

## Reproduction Assessment

Yes, by source inspection. Configure QMD mcporter with `searchMode: "search"` or `"vsearch"`; current main builds unified QMD `query` arguments without `rerank:false`, while upstream QMD defaults that argument to true.

## Solution Assessment

Yes. Setting `rerank:false` at the unified QMD v2 mcporter call site is the narrowest maintainable fix because direct CLI and QMD v1 tool paths already use non-reranking search/vector tools, and full `query` mode keeps its explicit config control.
