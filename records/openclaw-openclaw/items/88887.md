## What This Changes

The PR sends `rerank:false` on memory-core QMD v2 mcporter `query` calls for `search` and `vsearch` modes and adds a vsearch argument-capture unit test.

## Best Possible Solution

Land this narrow mcporter argument fix after normal maintainer and CI refresh, while keeping full query-mode rerank control in the already-merged config path.

## Reproduction Assessment

Yes. Source inspection reproduces the current-main path: configure QMD mcporter with `searchMode: "search"` or `"vsearch"`, and OpenClaw sends unified `query` args without `rerank` even though QMD defaults reranking to true.

## Solution Assessment

Yes. Setting `rerank:false` at the unified QMD v2 mcporter call site for the non-query modes is the narrowest maintainable fix because direct CLI and v1 tool fallback paths already avoid this reranker path.
