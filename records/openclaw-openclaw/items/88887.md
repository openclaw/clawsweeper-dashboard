## Summary

Keep open. Current main still omits `rerank:false` for QMD v2 mcporter `search`/`vsearch` calls, while this PR is a focused fix with supplied live terminal proof and no actionable patch defects found.

## What This Changes

The PR adds `rerank:false` to memory-core's QMD unified MCP `query` call for `search` and `vsearch` modes and adds a focused vsearch argument test.

## Best Possible Solution

Land this narrow mcporter argument fix after normal maintainer and CI review; keep broader reranker product work in separate items such as https://github.com/openclaw/openclaw/issues/64438.

## Reproduction Assessment

Yes. Source inspection reproduces the current-main path: configure QMD mcporter with `searchMode: "search"` or `"vsearch"`, and OpenClaw sends unified `query` args without `rerank`, leaving QMD's default reranking enabled.

## Solution Assessment

Yes. Setting `rerank:false` at the unified QMD v2 mcporter call site is the narrowest maintainable fix because direct CLI and v1 fallback paths already avoid this reranker path, while full `query` mode keeps existing behavior.
