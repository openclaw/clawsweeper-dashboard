## Summary

Keep open; the PR is still necessary because current main only exposes `memory.qmd.rerank` for full QMD `query` mode and still lets mcporter-backed `search`/`vsearch` flow through QMD's default reranker. The merge result is narrow, keeps full `query` behavior unchanged, and I found no actionable patch defect.

## What This Changes

The PR adds `rerank:false` to memory-core's QMD v2 mcporter `query` calls for `search` and `vsearch` modes and adds a unit test for the vsearch call arguments.

## Best Possible Solution

Land this narrow mcporter argument fix after normal maintainer and CI review; keep full query-mode no-rerank handled by the already-merged https://github.com/openclaw/openclaw/pull/90304.

## Reproduction Assessment

Yes. Source inspection reproduces the current-main path: configure QMD mcporter with `searchMode: "search"` or `"vsearch"`, OpenClaw sends unified `query` args without `rerank:false`, and QMD 2.1+ defaults `rerank` to true.

## Solution Assessment

Yes. Setting `rerank:false` at the unified QMD v2 mcporter call site is the narrowest maintainable fix because direct CLI and v1 tool paths already avoid this reranker path, and full `query` mode keeps its existing explicit `memory.qmd.rerank` control.
