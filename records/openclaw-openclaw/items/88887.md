## Summary

No blocking findings; current main still violates the documented no-reranker behavior for QMD `search`/`vsearch` when using the mcporter v2 `query` path, and the PR's small call-argument change matches QMD's current rerank contract.

## What This Changes

The PR adds `rerank:false` to QMD v2 mcporter `query` calls when OpenClaw search mode is `search` or `vsearch`, plus a unit test for vector-only mode.

## Best Possible Solution

Land the focused mcporter call-argument fix after normal CI because it restores documented `search`/`vsearch` semantics without adding config or touching unrelated QMD paths.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: configure QMD mcporter with `searchMode: "search"` or `"vsearch"` on current main and the unified `query` call omits `rerank`, while QMD defaults it to true.

## Solution Assessment

Yes. Setting `rerank:false` at the unified QMD query call site is the narrowest maintainable fix because `buildV2Searches` already encodes searchMode and the CLI/v1 paths do not rely on QMD's reranking default.
