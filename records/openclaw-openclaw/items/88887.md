## Summary

Keep open: current main and the latest release still omit the `rerank:false` mcporter argument for QMD `search`/`vsearch`, and the PR is a focused fix with sufficient real behavior proof and no blocking findings.

## What This Changes

The PR adds `rerank:false` to memory-core's QMD unified mcporter `query` arguments for `search` and `vsearch` modes and adds focused unit coverage for `vsearch`.

## Best Possible Solution

Land this narrow runtime/test fix, or an equivalent maintainer-owned patch, so QMD mcporter `search` and `vsearch` preserve their lexical/vector-only contract on current QMD.

## Reproduction Assessment

Yes, by source inspection. Current main builds unified QMD `query` MCP call args for `search` and `vsearch` without `rerank:false`, while current upstream QMD's `query` tool defaults `rerank` to true.

## Solution Assessment

Yes. Setting `rerank:false` at the unified QMD mcporter call site for `search` and `vsearch` is the narrowest maintainable fix because direct CLI modes already use dedicated non-reranking commands and full `query` mode keeps its config-controlled behavior.
