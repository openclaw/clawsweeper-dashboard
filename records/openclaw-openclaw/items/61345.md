## Summary

Keep open: this PR is a focused fix for the reconnect/login-gate split, but it still needs a small correctness fix for an existing protocol-mismatch path and inspectable real behavior proof before merge.

## What This Changes

The PR adds a Control UI login-gate predicate, shows a topbar `Connecting...` pill for disconnected dashboard states, and keeps that pill label visible on mobile.

## Best Possible Solution

Preserve the narrow reconnect/dashboard-shell behavior, add the existing protocol-mismatch/config failure to the gate predicate with focused coverage, then merge after inspectable real behavior proof is added.

## Reproduction Assessment

Yes, at source level. Current main gates every disconnected state, and the PR-introduced protocol-mismatch regression follows from `connected=false`, a configured gateway URL, `lastError='protocol mismatch'`, and no structured code returning false from the new predicate.

## Solution Assessment

No, not yet. Splitting transient reconnects from login-required states is the right boundary, but the predicate should keep existing high-confidence protocol-mismatch recovery on the login gate and the PR still needs real behavior proof.
