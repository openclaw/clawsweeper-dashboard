## Summary

Keep this PR open. Current main still renders the login gate for every disconnected Control UI state, while the PR is a narrow fix for separating transient reconnects from true login-required states; the broader open login-gate issue should not supersede this concrete bugfix before it merges.

## What This Changes

The PR changes Control UI render logic to use a new login-gate predicate, keeps the dashboard shell visible with a Connecting status during transient disconnects, and adds a mobile CSS override so that status label remains visible.
