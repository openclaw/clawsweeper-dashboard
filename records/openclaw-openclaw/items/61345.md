## Summary

Keep open: the PR is directionally useful for the Control UI reconnect UX, but the current branch would drop existing protocol-mismatch recovery guidance and still lacks inspectable after-fix real behavior proof.

## What This Changes

The branch changes the Control UI render gate to keep generic disconnected states in the dashboard shell with a `Connecting...` topbar pill, only showing the login gate for missing config or selected auth/config/device errors, plus a mobile CSS override for the pill label.

## Best Possible Solution

Keep the dashboard-shell reconnect split, but route existing high-confidence failures such as protocol mismatch through the login gate, add focused coverage, and merge only after redacted real behavior proof is attached.

## Reproduction Assessment

Yes, at source level. In the PR, `connected=false`, a configured `gatewayUrl`, `lastError='protocol mismatch'`, and `lastErrorCode=null` returns false from the new predicate, while current main has protocol-mismatch login-gate recovery and tests.

## Solution Assessment

No, not yet. Splitting transient reconnects from login-required states is the right local surface, but the predicate must preserve protocol-mismatch recovery and the PR still needs inspectable real behavior proof.
