## Summary

Keep open. Current main still has the reported source-level path: isolated cron announce awareness is queued as untrusted, heartbeat converts pending untrusted events into `ForceSenderIsOwnerFalse`, command auth hard-overrides owner identity, and owner-only tools are then filtered.

## What This Changes

The issue asks OpenClaw to stop owner-created isolated cron announce delivery from temporarily removing owner-only tools on the next owner DM.

## Best Possible Solution

Add a narrow provenance- or audience-aware internal awareness path for delivered isolated cron output while preserving owner downgrades for arbitrary untrusted system events.

## Reproduction Assessment

Yes, source-reproducible. The issue includes a live trace, and current main still queues cron awareness as untrusted, maps pending untrusted events to ForceSenderIsOwnerFalse, hard-overrides owner identity, and filters owner-only tools.

## Solution Assessment

Unclear. Trusting all owner-created cron output may be too broad; the safer maintainable direction is a scoped provenance or internal-audience exception such as #72201 or an equivalent narrow fix.
