## Summary

Keep open. Current main still has the reported source-level path from untrusted isolated cron announce awareness to owner downgrade and owner-only tool filtering, and the overlapping #72201 audience/provenance proposal is still an open PR rather than merged behavior.

## What This Changes

This issue asks OpenClaw to stop cron-fired isolated announce delivery awareness from stripping owner-only tools from the next verified owner DM.

## Best Possible Solution

Add a narrow provenance- or audience-aware lane for delivered isolated cron announce awareness so the main session gets hidden runtime context without downgrading the next verified owner DM, while preserving owner downgrades for arbitrary untrusted events.

## Reproduction Assessment

Yes. The issue includes a detailed live trace, and current main still shows the source path: untrusted cron awareness is queued, heartbeat sets ForceSenderIsOwnerFalse, command auth clears senderIsOwner, and owner-only tool policy filters cron/gateway/nodes.

## Solution Assessment

Unclear. The narrowest maintainable direction appears to be an internal-audience or provenance exception like #72201, rather than globally trusting cron output or weakening ForceSenderIsOwnerFalse for every verified owner turn.
