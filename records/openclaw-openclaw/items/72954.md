## Summary

Keep open because current main still has the reported source-level chain: isolated cron announce awareness is queued as untrusted, heartbeat converts untrusted pending events into a forced owner downgrade, command auth honors that downgrade before owner identity, and owner-only tools are then filtered out.

## What This Changes

Prevent isolated cron announce delivery awareness from stripping owner-only `cron`, `gateway`, and `nodes` tools on the next verified owner DM.

## Best Possible Solution

Add a scoped provenance or internal-audience lane for isolated cron announce awareness so owner-created cron output can inform the next verified owner turn without weakening the general untrusted-event owner downgrade.

## Reproduction Assessment

Yes. The report has a concrete live trace, and current main still provides a source-level reproduction path through untrusted cron awareness queueing, pending-event inspection, hard owner downgrade, and owner-only tool filtering.

## Solution Assessment

Unclear. Trusting owner-created cron output may be valid for this producer, but a narrow cron-awareness provenance or internal-audience event lane is safer than generally weakening `ForceSenderIsOwnerFalse`.
