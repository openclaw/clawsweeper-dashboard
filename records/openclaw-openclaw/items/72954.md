## Summary

Keep open. Current main still has the reported source path: isolated cron awareness is queued as untrusted, heartbeat maps pending untrusted events to `ForceSenderIsOwnerFalse`, command auth hard-downgrades owner identity, and owner-only tool policy removes the named tools; the related #72201 proposal is still an open design path rather than shipped code.

## What This Changes

The issue asks OpenClaw to stop cron-fired isolated announce awareness from stripping owner-only tools on the next verified owner DM.

## Best Possible Solution

Add a narrow provenance- or audience-aware path so owner-created isolated cron announce awareness can inform the next verified owner turn without granting owner-only tools to arbitrary untrusted system events.

## Reproduction Assessment

Yes. The issue includes a concrete live trace, and current main still has a source-level reproduction path through untrusted cron awareness queueing, pending-event inspection, hard owner downgrade, and owner-only tool filtering.

## Solution Assessment

Unclear. The suggested owner-created-cron trust change may be too broad; an internal-audience or provenance exception like #72201 appears safer than weakening `ForceSenderIsOwnerFalse` globally.
