## What This Changes

The issue asks OpenClaw to prevent owner-created isolated cron announce delivery from stripping owner-only tools on the next owner DM.

## Best Possible Solution

Add a narrow provenance- or audience-aware path for delivered isolated cron awareness so the main session receives hidden runtime context without disabling owner-only tools for the next verified owner turn.

## Reproduction Assessment

Yes. The issue includes a detailed live trace, and current main still has the source-level path from untrusted cron awareness through ForceSenderIsOwnerFalse to owner-only tool filtering.

## Solution Assessment

Unclear. Trusting all owner-created cron output may be too broad; the safer maintainable direction is a narrow internal-audience or provenance exception that preserves owner downgrades for arbitrary untrusted events.
