Keep open. Current main still has the reported trust-to-owner-downgrade chain for isolated cron awareness, and the active #72201 event-audience proposal is not merged into main.

## What This Changes

Prevent isolated cron announce awareness from causing the next verified owner DM to lose owner-only tools such as cron, gateway, and nodes.

## Best Possible Solution

Add a narrow provenance- or audience-aware lane for isolated cron announce awareness so owner-created cron output can inform the next verified owner turn without granting owner-only tools to arbitrary untrusted system events.

## Reproduction Assessment

Yes. The issue includes a concrete live trace, and current main still has the source-level path from untrusted cron awareness queueing to owner downgrade and owner-only tool filtering.

## Solution Assessment

Unclear. A scoped internal-audience or provenance exception, like the direction in #72201, looks safer than generally trusting cron output or letting verified owner identity bypass all untrusted-event downgrades.
