## Summary

Keep open. Current main still has the reported source-level chain from delivered cron awareness through untrusted-event owner downgrade and owner-only tool filtering, while the related #72201 audience/provenance direction is not present on main.

## What This Changes

Owner-created isolated cron `agentTurn` announce delivery should not cause the next owner DM to lose owner-only tools through the untrusted system-event downgrade path.

## Best Possible Solution

Add a narrow provenance- or audience-aware internal awareness path for delivered isolated cron output while preserving owner downgrades for arbitrary untrusted system events.

## Reproduction Assessment

Yes, source-reproducible. The report includes a live trace, and current main still queues cron awareness as untrusted, maps untrusted pending events to `ForceSenderIsOwnerFalse`, hard-overrides owner identity, and filters owner-only tools.

## Solution Assessment

Unclear. Trusting all owner-created cron output may be too broad; the safer maintainable direction is a narrow provenance or internal-audience exception, such as #72201 or an equivalent scoped fix.
