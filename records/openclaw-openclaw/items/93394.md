## Summary

Keep this PR open: it is a useful targeted fix, but it still misses the grouped direct-QMD search path, so the orphaned subprocess leak remains for supported multi-collection and mixed memory/session configurations.

## What This Changes

The PR adds optional AbortSignal handling to QMD memory search and the exported QMD process helper so direct qmd subprocesses are killed when memory_search times out.

## Best Possible Solution

Thread the same deadline signal through grouped and fallback direct-QMD searches, add grouped abort coverage, and keep the additive runCliCommand signal behavior only with maintainer acceptance of that exported-helper contract.

## Reproduction Assessment

Yes. Source inspection shows current memory_search supplies a deadline signal to manager.search, while the PR head still drops that signal in grouped direct-QMD searches; I did not run a live QMD timeout reproduction in this read-only review.

## Solution Assessment

No. Direct subprocess cancellation is the right layer, but the PR is not the best complete fix until grouped direct-QMD searches and their fallback path receive the same signal.
