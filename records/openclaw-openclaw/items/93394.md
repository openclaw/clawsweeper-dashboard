## Summary

Keep this PR open: the direct qmd subprocess cancellation is useful, but the patch still misses the supported grouped direct-QMD search path, so the orphaned subprocess leak remains for multi-collection or mixed-source QMD searches.

## What This Changes

The PR adds optional AbortSignal propagation from QMD memory search into qmd subprocess execution and adds regression tests for aborted QMD searches.

## Best Possible Solution

Thread the same deadline signal through runQueryAcrossCollectionGroups and its fallback path, add grouped multi-collection abort coverage, and keep the additive runCliCommand signal behavior if maintainers accept that exported-helper contract.

## Reproduction Assessment

Yes, source-reproducible: current main passes a memory_search deadline signal into manager.search, and QMD grouped direct searches still run qmd subprocesses without that signal. I did not run a live QMD timeout reproduction in this read-only review.

## Solution Assessment

No. Direct subprocess cancellation is the right layer, but the PR is not yet the best complete fix because it misses the grouped direct-QMD helper and fallback path.
