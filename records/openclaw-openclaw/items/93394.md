## What This Changes

The PR adds optional AbortSignal plumbing from QMD memory search into qmd/mcporter subprocess execution and tests that aborted searches kill or skip child processes.

## Best Possible Solution

Land the bounded cancellation plumbing after maintainer acceptance of the additive exported-helper cancellation contract and refreshed merge checks, while preserving existing no-signal behavior.

## Reproduction Assessment

Yes. Current main creates a 15s memory_search deadline signal and passes it into manager.search, but QmdMemoryManager and the QMD process helpers on main do not consume it, so a QMD search subprocess can outlive the tool deadline until the qmd command timeout.

## Solution Assessment

Yes. Forwarding the existing deadline signal to the shared qmd/mcporter process helper is the narrow owner-boundary fix; wrapping only memory_search would leave backend subprocess callers and tests drifting.
