## Summary

Keep open: the direct qmd subprocess cancellation is useful and has real subprocess proof, but the patch is still a partial fix because grouped multi-collection direct QMD searches and their fallback path do not receive the deadline signal.

## What This Changes

The PR adds optional AbortSignal plumbing from QMD memory search into runCliCommand, kills qmd subprocesses on abort, and adds QMD manager/process tests.

## Best Possible Solution

Thread the same deadline signal through every direct-QMD search subprocess path, including grouped collection searches and fallback, while keeping the exported helper cancellation contract explicit and tested.

## Reproduction Assessment

Yes, source-reproducible: memory_search passes a deadline AbortSignal into manager.search, and PR head still leaves grouped direct-QMD searches without that signal. I did not run a live QMD timeout reproduction in this read-only review.

## Solution Assessment

No. Direct subprocess cancellation is the right layer, but the current patch is too narrow until grouped direct-QMD initial and fallback paths carry the same signal.
