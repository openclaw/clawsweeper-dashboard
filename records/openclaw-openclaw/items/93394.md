## Summary

Keep open: this is a useful focused fix, but it is not merge-ready because supported grouped direct-QMD searches still drop the deadline AbortSignal and can leave qmd children running after memory_search times out.

## What This Changes

The PR adds optional AbortSignal plumbing to QMD memory search and runCliCommand with tests so direct qmd subprocesses are killed when memory_search times out.

## Best Possible Solution

Thread the deadline signal through every direct-QMD search path, including grouped and query-fallback paths, add grouped abort coverage, and keep the exported runCliCommand signal option only with maintainer acceptance.

## Reproduction Assessment

Yes. Source inspection shows memory_search passes a deadline signal into manager.search, while PR head still drops that signal in grouped direct-QMD searches; existing tests prove the grouped path is supported.

## Solution Assessment

No. Direct subprocess cancellation is the right layer, but the branch is incomplete until grouped direct-QMD searches and their fallback path receive the same signal.
