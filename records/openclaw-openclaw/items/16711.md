Keep open: current main and v2026.5.20 still have an inline-only Control UI composer with no Expand button, large draft modal, Apply/Cancel/Send workflow, or modal-only Enter-newline plus Ctrl/Cmd+Enter-send behavior. ## What This Changes

Request an Expand control for the Control UI chat composer that opens a larger editor modal with Apply, Cancel/Close, Send, and modal-specific keyboard behavior.

## Best Possible Solution

If accepted, implement this in the Control UI chat composer using the existing modal-dialog primitive, separate modal draft state, Apply/Cancel/Send semantics, modal-only keyboard handling, focus restoration, preserved attachment/queue behavior, and focused UI tests.
