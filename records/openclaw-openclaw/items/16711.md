## Summary

Keep this issue open for maintainer product follow-up. Current main and v2026.6.6 provide an auto-growing inline composer capped at 150px, but neither implements the requested large editor, Expand action, Apply/Cancel/Send flow, or alternate modal keyboard semantics; no canonical related issue or merged PR owns that remaining Control UI work.

## What This Changes

Add an Expand control to the Control UI chat composer that opens a larger draft editor with cancel, apply, send, and dedicated keyboard semantics.

## Best Possible Solution

Choose one accessible, responsive long-prompt editing pattern for the built-in composer, preserve a single canonical draft and attachments, and cover apply/cancel/send, focus restoration, mobile layout, and both inline and expanded keyboard contracts in focused UI tests.

## Reproduction Assessment

Not applicable: this requests a new editing mode rather than reporting broken established behavior; source inspection confirms only the bounded inline auto-grow behavior currently exists.

## Solution Assessment

Unclear: a modal is plausible, but maintainers should compare it with enlarging the existing responsive auto-grow behavior and explicitly settle keyboard, draft, attachment, focus, mobile, and accessibility semantics.
