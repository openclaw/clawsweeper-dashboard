## Summary

Keep open: current main still has only the bounded inline Control UI composer, with no Expand button, large draft modal, Apply/Cancel/Send workflow, or modal-specific keyboard behavior. This is a valid UX feature request rather than a closeable cleanup item.

## What This Changes

Add an Expand control for the Control UI chat composer that opens a larger prompt editor with Apply, Cancel/Close, Send, and modal-specific Enter/Ctrl+Enter behavior.

## Best Possible Solution

Keep this as a Control UI enhancement; if accepted, implement a chat-local expanded composer modal using the existing modal primitive and current draft/send path, with focused UI tests for draft ownership, Apply, Cancel, Send, keyboard handling, and focus restoration.
