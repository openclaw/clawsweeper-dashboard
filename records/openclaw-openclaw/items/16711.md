## Summary

Keep open: current main still has a bounded inline Control UI composer with no Expand control or large-editor modal, and no canonical duplicate or merged implementation replaces the requested UX feature.

## What This Changes

Add an Expand control to the chat composer that opens a larger draft editor with Apply, Cancel/Close, Send, and modal-specific Enter/Ctrl-or-Cmd+Enter behavior.

## Best Possible Solution

If accepted, implement a Control UI-local expanded composer using the existing modal primitive and draft/send APIs, with focused tests for draft apply/cancel/send, keyboard handling, attachments, focus, and queued sends.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source inspection on current main shows the existing composer is a single bounded inline textarea with no expand control or editor modal.

## Solution Assessment

Unclear until maintainers choose the UX direction. The narrow maintainable path would be a chat-local modal around the existing draft/send flow rather than Gateway, protocol, or config changes.
