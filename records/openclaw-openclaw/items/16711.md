## Repair Work Prompt

Implement the expanded chat composer requested in openclaw/openclaw#16711. Add an Expand icon/button to the Control UI chat composer near the existing toolbar controls. Opening it should show a large editor modal seeded from the current draft. The modal must keep separate draft state until the user chooses Apply or Send. Close/Cancel should dismiss without changing the inline draft. Apply should copy the modal text back to the inline composer without sending. Send should apply the modal text and submit through the existing chat send path. Inside the modal, Enter inserts a newline and Ctrl+Enter/Cmd+Enter sends. Preserve existing inline composer behavior, including Enter-to-send, attachments, queued sends, abort/busy state, focus restoration, and disabled/connected handling. Keep changes scoped to the Control UI; do not change Gateway chat.send semantics or unrelated session behavior. Add focused UI tests for render controls, modal draft ownership, Apply/Cancel/Send, and keyboard behavior. Add a user-facing CHANGELOG.md entry under the active Changes section with Thanks @dragidude.
