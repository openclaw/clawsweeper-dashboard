## Summary

Keep open: current main still lacks the requested expanded composer modal, and this is a new Control UI UX feature that needs maintainer product direction rather than cleanup closure.

## What This Changes

Add an Expand button for the chat composer that opens a larger modal editor with Apply, Cancel, and Send behavior for long prompts.

## Best Possible Solution

If accepted, implement this as a Control UI-local expanded composer using the existing modal primitive and current draft/send APIs, without Gateway protocol or config changes.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing contract. Source inspection on current main gives high confidence that only the bounded inline composer exists today.

## Solution Assessment

Unclear until maintainers choose the UX direction. The narrow maintainable path is a chat-local modal with separate draft state, Apply/Cancel/Send semantics, and focused UI tests; the auto-grow alternative is adjacent but not the requested workflow.
