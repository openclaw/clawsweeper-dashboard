This PR strips internal sender metadata from Control UI chat history and live stream rendering, suppresses metadata-only chat items, and stops operator UI `chat.send` requests from populating sender identity fields, with regression tests.

## Best Possible Solution

Keep the metadata stripping and operator sender suppression, but preserve canvas-only assistant anchors and add a concise changelog fix entry before merge.

## Reproduction Assessment

Yes, source-reproducible. Current main renders active stream text raw, only strips inbound metadata for user-role history, and still sends operator UI client identity fields through `chat.send`.

## Solution Assessment

No. The fix direction is correct, but the empty-message filter should be delayed until after canvas lifting or narrowed to exact metadata-only entries, and the user-facing fix needs a changelog entry.
