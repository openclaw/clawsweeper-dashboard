## Summary

Keep open: current main and v2026.6.9 still count historical cron errors in the Overview card and attention item without excluding disabled jobs, while the live related issues do not own this exact disabled-job aggregation bug.

## What This Changes

The issue asks for disabled cron jobs with no next scheduled run to stop contributing to the top-level failed cron summary purely because their last run ended in error.

## Best Possible Solution

Add one shared UI predicate for actionable cron failures and use it in both the Overview card and attention items, while keeping disabled jobs' historical error state visible in cron detail/list views.

## Reproduction Assessment

Yes, source-level: current main loads disabled cron jobs into the Overview input and counts any `lastRunStatus === "error"` as failed. I did not run a live UI scenario in this read-only pass.

## Solution Assessment

Yes: the narrow maintainable fix is to centralize the Overview-only actionable-failure predicate, not to erase historical cron error state or change scheduler persistence.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/95716. Current Control UI Overview loads all cron jobs by default, including disabled jobs, and both `renderOverviewCards` and `buildAttentionItems` count any `resolveCronJobLastRunStatus(job) === "error"` as a current failure. Add a small shared UI helper/predicate, likely in `ui/src/ui/cron-status.ts`, that treats disabled jobs with no scheduled next run as historical/non-actionable for Overview failure counts while preserving historical `lastRunStatus` in cron list/detail filters. Use the helper in `ui/src/ui/views/overview-cards.ts` and `ui/src/ui/app-settings.ts`. Add focused regression coverage for a disabled failed job not appearing in the Overview failed count/attention item, plus an enabled failed job still appearing. Do not change cron persistence, scheduler retry semantics, failure notifications, or the adjacent issues https://github.com/openclaw/openclaw/issues/90595 and https://github.com/openclaw/openclaw/issues/49258. No CHANGELOG.md edit.
