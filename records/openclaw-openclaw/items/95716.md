## Summary

Keep open: current main and the latest release still count disabled cron jobs with historical `lastRunStatus: "error"` in the Control UI Overview failed summaries, and the linked fix PR is open rather than merged.

## What This Changes

The issue reports that the Control UI Overview treats disabled cron jobs with historical `lastRunStatus: "error"` as current failed cron jobs.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/95723, or an equivalent small UI fix that centralizes active cron failure detection for the Overview card and attention item while preserving historical status in detail views.

## Reproduction Assessment

Yes, at source level: current main's Overview card and attention builder both filter solely on resolved `lastRunStatus === "error"`, so a disabled job retaining historical error state enters both summaries. I did not run a browser UI scenario in this read-only pass.

## Solution Assessment

Yes, the best path is the linked small UI fix or equivalent: share an active-failure predicate for Overview current-actionability surfaces without changing scheduler state or hiding historical error details.
