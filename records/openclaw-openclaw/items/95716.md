## Summary

Keep open: current main and the latest release still count disabled cron jobs as failed from historical `lastRunStatus: "error"`, and the direct candidate fix PR remains open and unmerged.

## What This Changes

The issue reports that Control UI Overview treats disabled cron jobs with retained historical error state as active failed cron jobs.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/95723, or an equivalent focused UI patch, then close this issue once the fix is reachable from main.

## Reproduction Assessment

Yes, at source level: both current main and v2026.6.9 count cron jobs as failed when `resolveCronJobLastRunStatus(j) === "error"` without checking `enabled` or `nextRunAtMs`. I did not run a browser UI scenario in this read-only review.

## Solution Assessment

Yes, the linked PR's shared active-failure predicate is the best narrow fix because it updates both affected Overview summaries while preserving historical last-run status for detail views.
