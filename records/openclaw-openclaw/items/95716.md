## Summary

Keep open: current main and v2026.6.9 still count cron jobs as failed from historical `lastRunStatus: "error"` without checking whether the job is disabled, and the linked fix PR is still open.

## What This Changes

Overview should stop counting disabled cron jobs with historical error status as current failed cron jobs.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/95723, or an equivalent small UI fix that centralizes an active-cron-failure predicate for both Overview failed-count surfaces while preserving historical status in detail views.

## Reproduction Assessment

Yes, at source level: current main's Overview card and attention builder filter solely on resolved `lastRunStatus === "error"`, so a disabled `CronJob` retaining historical error state enters both summaries. I did not run a browser UI scenario in this read-only pass.

## Solution Assessment

Yes, the best path is the linked small UI fix or equivalent: use one active-failure predicate shared by the Overview card and attention list, without changing scheduler state or hiding historical error details.
