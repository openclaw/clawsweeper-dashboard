## Summary

Keep open. Current `node.list` no longer directly scans agent directories, but configured-only `sessions.list` still loads the broad disk-discovered combined store before filtering, and `doctor` still only warns about orphaned agent directories instead of offering an automatic cleanup path; the overlapping cache PR is open, conflicting, and not a safe canonical fix.

## What This Changes

The issue requests that Gateway list RPCs avoid blocking on orphaned agent directories and that `openclaw doctor --fix` provide a cleanup path for stale agent directories.

## Best Possible Solution

Keep this as the canonical issue and repair `sessions.list` so configured-only callers use configured-only store loading, while preserving explicit broad discovery for ACP/retired sessions and making a maintainer decision on stale agent-directory cleanup.

## Reproduction Assessment

No live high-confidence timing reproduction was run. Source inspection does show a configured-only `sessions.list` request still loading the broad combined store before filtering, but the reported stall magnitude depends on slow storage and many stale directories.

## Solution Assessment

No, this should not be closed as solved yet. The narrow runtime fix appears to be passing the existing configured-only option into the combined-store loader, while automatic stale-directory cleanup remains a maintainer compatibility decision.
