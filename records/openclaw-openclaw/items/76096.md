## Summary

Keep open: the exact `node.list` claim is partly obsolete, but current main and the latest release still leave adjacent `agents.list` and `sessions.list` hot paths plus orphan-dir cleanup policy unresolved.

## What This Changes

The issue asks Gateway list RPCs to avoid blocking on orphaned agent directories and asks `openclaw doctor --fix` to provide a safe cleanup path.

## Best Possible Solution

Keep the current `node.list` behavior, narrow Gateway list discovery where configured scope is already known, and define a deliberate doctor archive or cleanup policy for verified orphan agent directories.

## Reproduction Assessment

No high-confidence live current-main reproduction was run. Source inspection gives a medium-confidence current path: orphan or disk-only agent state can still make `agents.list` synchronously enumerate `state/agents`, and `sessions.list` can load broad stores before configured-agent filtering, while `node.list` itself no longer scans agent directories.

## Solution Assessment

Unclear. The runtime side likely belongs in existing configured-agent and agentId list seams, but `doctor --fix` cleanup needs a maintainer policy for archive/delete safety before implementation.
