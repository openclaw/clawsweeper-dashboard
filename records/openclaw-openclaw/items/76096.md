## Summary

Keep open: current `node.list` no longer scans agent directories, but current `main` and the latest release still have source-backed blocking discovery in adjacent `sessions.list`/`agents.list` paths, and automatic orphan-agent cleanup still needs a maintainer policy decision.

## What This Changes

The issue requests that Gateway list RPCs avoid scanning orphaned agent directories and that `doctor --fix` safely clean up or repair those stale directories.

## Best Possible Solution

Make Gateway list paths avoid loading unrelated disk-only stores while preserving intentional ACP/retired-session discovery, then add a deliberate doctor archive or cleanup policy for true orphan agent directories.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection gives a medium-confidence current-main path: create many disk-only agent/session stores, then call `sessions.list` with `configuredAgentsOnly` or `agents.list`; `node.list` itself no longer scans agent directories.

## Solution Assessment

Unclear. Narrowing configured-agent discovery is likely the right runtime direction, but `doctor --fix` cleanup needs an explicit maintainer policy because orphan dirs can contain recoverable auth and session state.
