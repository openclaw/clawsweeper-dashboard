## Summary

Keep open: the exact `node.list` path on current `main` no longer scans agent directories, but the underlying low-power-host performance concern is still source-backed through `agents.list` and `sessions.list` disk discovery, and the requested automatic orphan-dir cleanup is not implemented.

## What This Changes

The issue asks OpenClaw to stop Gateway list RPCs from blocking on orphaned agent directories and to provide a safe `doctor --fix` cleanup path.

## Best Possible Solution

Make configured-agent Gateway list paths avoid scanning or loading unrelated disk-only agent stores, preserve intentional ACP/disk-only discovery where it is required, and add a deliberate safe cleanup/archive policy if maintainers want `doctor --fix` to repair orphan dirs.

## Reproduction Assessment

No, I did not run a live low-power reproduction. Source inspection gives a medium-confidence repro path: create many disk-only agent/session directories, call `agents.list` or `sessions.list` with configured-agent intent, and observe synchronous directory/session-store discovery before filtering.

## Solution Assessment

Unclear. The open caching PR may reduce repeated list-call pain, but the durable fix should narrow configured-agent discovery and separately decide the safety policy for automatic orphan-dir cleanup.
