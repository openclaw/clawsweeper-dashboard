## Summary

Keep open: current main no longer makes `node.list` scan agent directories directly, but the same Control UI startup path still invokes `agents.list` and `sessions.list` discovery code that synchronously enumerates agent/session directories, and `doctor` still only warns about orphaned agent directories.

## What This Changes

The issue asks OpenClaw to stop list RPCs from blocking on orphaned agent directories and to add a safe `doctor --fix` cleanup path for stale agent directories.

## Best Possible Solution

Bound or cache/yield agent and session-store discovery while preserving intentional disk-only session visibility, then add a guarded `doctor --fix` path that previews and safely archives or removes confirmed stale agent directories.

## Reproduction Assessment

Unclear for the exact `node.list` claim because current `node.list` only aggregates pairing state and connected nodes. Source inspection does show a high-confidence broader Control UI stall path through synchronous `agents.list` and `sessions.list` directory discovery during the same startup flow.

## Solution Assessment

No for a `node.list`-only fix or a blanket skip of orphan directories. The maintainable path is to repair shared agent/session discovery and doctor cleanup while preserving disk-only ACP/session contracts.
