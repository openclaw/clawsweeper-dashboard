## Summary

Keep open: current main has partial latency work, but synchronous agent/session directory discovery and the orphan-agent doctor warning remain in the hot path. The exact `node.list` attribution looks indirect because `node.list` itself does not scan agent directories, but the reported Control UI stall class is still source-backed.

## What This Changes

The issue asks to prevent Control UI list RPCs from synchronously scanning orphaned agent/session directories and to add a `doctor --fix` cleanup path for stale agent dirs.

## Best Possible Solution

Make list discovery bounded, cached, async/yielded, or scoped while preserving intentional disk-only session visibility, and add a conservative doctor preview/confirmation repair for stale agent directories.

## Reproduction Assessment

Yes for the broader stall class: current main still has synchronous `readdirSync`, realpath/lstat validation, and discovered-store enumeration reachable from Control UI list refreshes with many agent dirs. Unclear for the exact `node.list` attribution because the current `node.list` handler does not scan agent directories directly.

## Solution Assessment

No for a `node.list`-only fix: the narrow maintainable repair belongs in agent/session discovery and guarded doctor cleanup, with `node.list` treated as a delayed victim unless new profiling proves otherwise.
