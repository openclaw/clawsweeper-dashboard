## Summary

Keep open: current `node.list` is not the direct agent-directory scanner, but current `main` still has synchronous `agents.list` and `sessions.list` discovery paths during the same Control UI startup flow, and `doctor` still only warns about orphaned agent directories.

## What This Changes

The issue requests that Control UI startup list RPCs avoid blocking the event loop on orphaned agent directories and that `doctor` offer a safe cleanup path.

## Best Possible Solution

Preserve disk-only ACP/custom session visibility while making agent/session discovery bounded, cached, or async/yielded, then add a guarded doctor cleanup or archival path for confirmed stale agent directories.

## Reproduction Assessment

Unclear for the exact `node.list` attribution because the current handler does not scan agent directories. Source inspection does show a high-confidence broader Control UI startup stall path through synchronous `agents.list` and `sessions.list` discovery over disk-only agent/session directories.

## Solution Assessment

No for a `node.list`-only repair or a blanket orphan-dir skip. The safer fix belongs in shared agent/session discovery and guarded doctor cleanup while preserving disk-only ACP/custom session contracts.
