## Summary

Keep open: current main still has synchronous orphan-agent directory discovery on adjacent Control UI list paths, while the exact `node.list` attribution appears likely to be cross-RPC delay rather than that handler scanning agent directories itself.

## What This Changes

Avoid event-loop stalls from orphaned agent directories during Control UI list RPCs and add a guarded cleanup path for stale agent dirs.

## Best Possible Solution

Bound or make asynchronous the configured-agent and session-store discovery paths, while adding only a conservative doctor preview/confirmation path for stale agent directories.

## Reproduction Assessment

Yes for the broader stall class: create many stale directories under the state `agents` directory and hit the Control UI list paths that call `agents.list` and `sessions.list`. The exact `node.list` reproduction is unclear because current `node.list` reads pairing JSON and in-memory registry state rather than agent directories.

## Solution Assessment

No, changing `node.list` alone would miss the root cause visible in current main. The safer fix is a configured-agent fast path for `agents.list`, async/yielded or cached session-store discovery for `sessions.list`, and non-destructive doctor cleanup guidance or confirmation.
