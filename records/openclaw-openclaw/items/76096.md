## Summary

Keep open: current `node.list` does not directly scan agent directories, but current `main` still has synchronous `agents.list` and `sessions.list` discovery paths that enumerate on-disk agent/session directories during the Control UI startup flow, and `doctor` still only warns about orphaned agent dirs.

## What This Changes

The issue reports severe gateway stalls during Control UI list RPCs when many stale agent directories remain under the state agents directory.

## Best Possible Solution

Keep the issue open and make agent/session discovery bounded, cached, async/yielded, or scoped while preserving intentional disk-only session visibility, then add guarded doctor remediation for stale agent directories.

## Reproduction Assessment

Unclear for the exact `node.list` attribution because the current handler does not scan agent directories. Source inspection does show a high-confidence broader Control UI stall path through synchronous `agents.list` and `sessions.list` discovery on current `main`.

## Solution Assessment

No for a `node.list`-only fix. The maintainable repair belongs in shared agent/session-store discovery and doctor cleanup while preserving the disk-only session-store contracts from #32831 and #44176.
