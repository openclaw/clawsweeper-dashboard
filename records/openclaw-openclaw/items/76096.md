## Summary

Keep open: current main still has source-backed synchronous agent/session directory discovery reachable from Control UI list refreshes, and the orphan-agent doctor path still warns without a repair. The literal `node.list` handler is not the direct agent-dir scanner now, so the remaining bug should be framed as adjacent list-path event-loop starvation rather than a `node.list`-only defect.

## What This Changes

The issue reports gateway list RPC stalls from orphaned `~/.openclaw/agents` directories and asks for bounded/non-blocking list discovery plus an orphan-agent `doctor --fix` cleanup path.

## Best Possible Solution

Make gateway list discovery bounded, cached, async/yielded, or scoped while preserving disk-only ACP session visibility, and add a conservative doctor preview/confirmation or archive repair for stale agent dirs.

## Reproduction Assessment

Yes, source-reproducible for the broader event-loop stall class: Control UI list refreshes reach synchronous agent-dir and session-store discovery on current main. It is not a high-confidence literal `node.list` direct-scanner reproduction because that handler now only reads pairing/registry state.

## Solution Assessment

No for a `node.list`-only fix: the narrow maintainable repair belongs in `agents.list` and session-store discovery, with guarded doctor cleanup for stale agent dirs. The current source already separates node pairing from the costly disk scans, so fixing the shared list hot paths is the safer direction.
