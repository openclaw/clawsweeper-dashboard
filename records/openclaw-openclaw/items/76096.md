## Summary

Keep open: current main has adjacent `sessions.list` performance fixes and `node.list` itself no longer scans agent directories, but the Control UI startup path still reaches synchronous agent/session discovery over on-disk agent roots, and doctor still only warns about orphaned agent directories.

## What This Changes

The issue reports severe Control UI/RPC event-loop stalls from synchronous discovery over orphaned agent directories and asks for bounded discovery plus a doctor cleanup path.

## Best Possible Solution

Keep the issue open and make agent/session discovery bounded, cached, async/yielded, or otherwise non-blocking while preserving disk-only ACP/retired session visibility, then add a guarded doctor repair path for confirmed stale agent directories.

## Reproduction Assessment

Unclear for the exact `node.list` timing, because the current handler does not scan agent directories. Yes from source for the broader Control UI stall class: startup still reaches synchronous `agents.list` and `sessions.list` discovery over on-disk agent roots.

## Solution Assessment

No for the issue's simple "skip orphaned directories" direction, because current tests and linked PR history intentionally preserve disk-only ACP/retired session discovery. The safer fix is a bounded or cached discovery path plus guarded cleanup semantics.
