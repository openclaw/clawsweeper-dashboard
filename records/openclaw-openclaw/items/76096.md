## Summary

Keep open: current `node.list` itself no longer scans agent directories, but current `main` still has synchronous `agents.list` and `sessions.list` discovery paths that enumerate agent/session directories during the same Control UI startup flow, and `doctor` still only warns about orphaned agent directories.

## What This Changes

Make Control UI list RPCs stay responsive when orphaned agent directories exist, and add a safe cleanup path for stale agent directories.

## Best Possible Solution

Keep the issue open and make list discovery bounded, cached, async/yielded, or scoped while preserving disk-only session visibility, then add a guarded doctor preview/confirmation cleanup for stale agent directories.

Unclear for the exact `node.list` attribution because the current handler does not scan agent directories. ## Solution Assessment
