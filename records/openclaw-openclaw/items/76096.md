## Summary

Keep open: the exact node.list path no longer scans agent directories, but current main and the latest release still have source-backed blocking work in adjacent configured-agent list paths, and automatic orphan-dir cleanup still needs a maintainer policy decision.

## What This Changes

The issue requests that Gateway list RPCs avoid scanning orphaned agent directories and that doctor provide a safe fix path for stale agent state.

## Best Possible Solution

Make configured-agent Gateway list paths avoid loading unrelated disk-only stores, preserve intentional ACP/session discovery where required, and add a deliberate doctor archive or cleanup policy for true orphan agent dirs.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection gives a medium-confidence path: create many disk-only agent/session stores, call sessions.list with configuredAgentsOnly or agents.list, and observe synchronous discovery/loading before filtering.

## Solution Assessment

Unclear. Narrowing configured-agent discovery is likely the right runtime fix, but doctor --fix cleanup needs an explicit maintainer policy because orphan dirs can contain recoverable auth and session state.
