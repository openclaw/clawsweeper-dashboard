## Summary

Keep open: current main has improved some sessions-list row work, but it still has synchronous agent/session discovery over disk-only agent directories during the same Control UI startup flow, and doctor still only warns about orphaned agent directories instead of offering a guarded cleanup path.

## What This Changes

The issue asks OpenClaw to stop list RPCs from stalling on large numbers of orphaned agent directories and to add a doctor cleanup path for those directories.

## Best Possible Solution

Keep the disk-only session contract, but make gateway list discovery bounded, cached, async/yielded, or scoped so stale directories cannot stall the event loop, then add an explicit doctor preview/confirmation repair for orphaned agent directories.

## Reproduction Assessment

Yes for the broader source path: current Control UI startup can issue agents/sessions/nodes list work together, and agents.list plus sessions.list still use synchronous directory discovery. No for the exact node.list-only attribution, because the current node.list handler itself does not scan agent directories.

## Solution Assessment

No for a node.list-only repair. The maintainable fix belongs in agent/session discovery and doctor cleanup while preserving intentional disk-only session visibility.
