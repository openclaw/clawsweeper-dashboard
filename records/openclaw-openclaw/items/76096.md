## Summary

Keep open: current main only partially addresses the reported stall class. `node.list` itself does not scan agent directories, but adjacent Control UI list RPCs still have synchronous orphan-agent-directory discovery paths, and doctor still only warns instead of offering a guarded fix.

## What This Changes

The issue asks for Gateway/Control UI list RPCs to stop blocking on orphaned `~/.openclaw/agents` directories and for `doctor --fix` to provide a safe cleanup path.

## Best Possible Solution

Make `agents.list` and `sessions.list` bounded or non-blocking on orphan-heavy state, while adding a conservative doctor cleanup/preview flow that preserves disk-only session visibility unless the operator explicitly confirms removal.

## Reproduction Assessment

Yes for the broader gateway stall path: current main still has synchronous agent-dir and session-store discovery reachable from Control UI list RPCs. Unclear for `node.list` specifically, because the current `node.list` handler itself does not scan agent directories.

## Solution Assessment

No: changing `node.list` alone would miss the likely root cause. The safer repair is to fast-path configured-agent listing, async/cache or bound session-store discovery, and add guarded doctor repair for stale agent directories.

## Repair Work Prompt

Fix openclaw/openclaw#76096 on current main. The observable bug is that hosts with many orphaned `~/.openclaw/agents` directories can stall Control UI list RPCs; current source suggests `node.list` is delayed by event-loop starvation from adjacent `agents.list`/`sessions.list`, not by scanning inside `node.list` itself. Keep `node.list` pairing/registry semantics unchanged. In `src/gateway/session-utils.ts`, avoid scanning disk-only agent dirs when `agents.list` explicitly scopes the agent list, while preserving fallback behavior when no explicit list exists. For `sessions.list`, replace or bound the sync combined-store target discovery/load path with async/yielded or cached discovery so orphan-heavy agent roots cannot monopolize the event loop. Preserve existing disk-only ACP/retired session visibility covered by `src/gateway/session-utils.subagent.test.ts` and `src/config/sessions/targets.test.ts`. Add a conservative `doctor --fix`/repair path for orphan agent dirs with preview/confirmation; do not delete configured dirs, active dirs, or session/auth state silently. Update `CHANGELOG.md` for the user-facing fix and keep wording to plugins/agents terminology.
