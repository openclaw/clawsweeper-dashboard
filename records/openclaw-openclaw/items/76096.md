## Summary

Keep open: current main still has synchronous agent-directory/session-store discovery reachable from Control UI list RPCs, and `doctor --fix` still only warns about orphaned agent directories. The exact `node.list` attribution is likely indirect, but the reported gateway stall class remains source-backed.

## What This Changes

The issue asks OpenClaw to stop orphaned `~/.openclaw/agents/*` directories from stalling list RPCs and to add a guarded cleanup path for stale agent directories.

## Best Possible Solution

Make gateway list discovery bounded, cached, async/yielded, or scoped while preserving intentional disk-only session visibility, and add a conservative doctor preview/confirmation flow for stale agent directories.

## Reproduction Assessment

Yes for the broader stall class: current main has synchronous `fs.readdirSync`, realpath/lstat validation, and discovered-store enumeration reachable from Control UI list RPCs with many agent dirs. Unclear for `node.list` itself, because that handler does not scan agent directories on current main.

## Solution Assessment

No for a `node.list`-only fix: the narrow maintainable repair belongs in `agents.list` and session-store discovery, with guarded doctor cleanup rather than silent deletion.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/76096 with a narrow performance PR. The observable bug is that orphaned directories under the OpenClaw agents state root can make Control UI list RPCs stall the gateway event loop. Do not treat `node.list` as the primary fix unless new evidence shows it scans those dirs; current source points to `agents.list` and `sessions.list`/combined session-store discovery. Make `agents.list` avoid unbounded synchronous scanning of all orphaned dirs when explicit `agents.list` config exists, and make gateway session-store discovery bounded, cached, async/yielded, or otherwise list-safe without losing intentional disk-only/retired/ACP session visibility. Add a conservative `doctor --fix` path only if it previews or confirms stale agent-dir cleanup and does not delete recoverable auth/session state silently. Update `CHANGELOG.md` because this is user-facing. Do not remove the existing disk-only session visibility contract covered by `src/gateway/session-utils.subagent.test.ts`.
