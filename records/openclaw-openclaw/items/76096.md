## Summary

Keep open: current main does not yet address the orphan-agent-directory stall class, even though the exact `node.list` attribution looks likely misidentified because that handler does not scan agent dirs.

## What This Changes

The issue asks OpenClaw to stop Control UI/RPC listing from synchronously scanning orphaned `~/.openclaw/agents` directories and to add a safe doctor cleanup path.

## Best Possible Solution

Make the listing hot paths bounded or non-blocking while preserving intentional disk-only session visibility, and add only a guarded cleanup workflow for stale agent directories.

## Reproduction Assessment

Yes for the broader gateway stall: current code has synchronous `fs.readdirSync` and store-discovery paths reachable from Control UI list RPCs with many agent dirs. Unclear for `node.list` specifically, because that handler does not scan agent directories on current main.

## Solution Assessment

No: changing `node.list` alone is unlikely to solve the root cause. The safer fix is a configured-agent fast path for `agents.list`, async/yielded or cached discovery for `sessions.list`, and a conservative doctor cleanup/preview path rather than automatic deletion.

## Repair Work Prompt

Fix the orphan-agent-directory event-loop stall reported in https://github.com/openclaw/openclaw/issues/76096. Focus on the current Control UI list hot paths, not the node device catalog: `node.list` itself should keep its paired-device/connected-node semantics. Add a configured-agent fast path so `agents.list` does not synchronously enumerate every state `agents` directory when `agents.list` is explicit. For `sessions.list` with templated per-agent stores, remove or bound the synchronous discovered-agent scan by using async/yielded/cached discovery or another conservative design that preserves existing ACP/deleted/retired disk-only session visibility tests. Do not automatically delete agent directories in normal gateway startup or list RPCs. For doctor, either add an explicit confirmed cleanup/repair path with safe preview/backups or improve the fix guidance if automatic removal is too risky. Add focused regression coverage for many disk-only agent dirs, preserve current tests for disk-only session visibility, and add a user-facing changelog entry unless the final change is test-only/internal.
