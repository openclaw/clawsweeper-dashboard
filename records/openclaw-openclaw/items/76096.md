## Summary

Keep open: current `node.list` no longer scans agent directories, but current `main` and v2026.6.6 still leave source-backed `agents.list`/`sessions.list` disk-discovery work and `doctor` only warns about orphan agent directories instead of repairing them.

## What This Changes

The issue asks for Gateway list RPCs to avoid blocking on orphaned agent directories and for `openclaw doctor --fix` to provide a safe cleanup path.

## Best Possible Solution

Narrow configured-agent Gateway list paths so they avoid unrelated disk-only stores, preserve intentional ACP/retired-store discovery where required, and add an explicit safe archive/delete policy if `doctor --fix` should repair orphan agent directories.

## Reproduction Assessment

No high-confidence live reproduction was established for the exact `node.list` title. Source inspection supports the remaining path: `agents.list` still synchronously enumerates agent dirs before filtering, and `sessions.list` loads combined stores before applying `configuredAgentsOnly`.

## Solution Assessment

No, not as a simple blanket skip/delete of orphan directories. The maintainable fix needs to separate configured-agent UI hot paths from intentional disk-only ACP/retired-session discovery and decide a safe doctor archive/delete policy.
