## Summary

Keep open: current `main` has moved the exact `node.list` handler away from agent-directory scanning, but the related Control UI `sessions.list` path still loads broad session-store discovery before filtering, and `doctor` still warns about orphaned agent directories without an automatic repair policy.

## What This Changes

The issue requests that Gateway list calls avoid long event-loop stalls from orphaned agent directories and that `openclaw doctor --fix` provide a safe cleanup path.

## Best Possible Solution

Narrow configured-agent Gateway list paths so they do not synchronously scan or load unrelated orphan stores, preserve intentional ACP/disk-only discovery where required, and add only a deliberate safe doctor cleanup/archive policy.

## Reproduction Assessment

No high-confidence live reproduction was run for the exact 30-48 second stall on comparable hardware. Source inspection does show a credible current-main path through Control UI `sessions.list`, synchronous broad session-store discovery, and warning-only orphan-dir doctor behavior.

## Solution Assessment

Unclear as a single automated fix. The best repair likely combines a bounded Gateway discovery/performance change with a separate maintainer decision on safe orphan-dir cleanup semantics.
