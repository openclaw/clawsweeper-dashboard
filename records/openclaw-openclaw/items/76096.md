## Summary

Keep open: current main and the latest release no longer route `node.list` through agent-directory discovery, but related configured-agent `agents.list`/`sessions.list` paths still synchronously inspect disk-only agent/session state and the requested `doctor --fix` orphan-dir cleanup remains a maintainer policy decision.

## What This Changes

The issue reports severe Gateway event-loop stalls from orphaned agent directories and asks OpenClaw to skip those directories in hot RPC paths or provide an automatic doctor cleanup path.

## Best Possible Solution

Make configured-agent RPC paths avoid scanning or loading unrelated disk-only agent stores while preserving explicit recovery/discovery modes, then add a safe orphan-dir archive or repair policy only if maintainers choose that doctor behavior.

## Reproduction Assessment

No live high-confidence reproduction was run in this read-only review. Source inspection gives a medium-confidence path: create many disk-only agent/session directories, call configured-agent list views, and observe synchronous directory/session-store discovery before filtering.

## Solution Assessment

Unclear. A durable runtime fix should narrow configured-agent discovery rather than rely only on caching, while the requested `doctor --fix` cleanup needs a maintainer-approved archive or deletion policy.
