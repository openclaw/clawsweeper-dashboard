## Summary

Keep this PR open because it addresses a real remaining plugin command context gap: current main receives an optional command-level agent id but does not expose it to plugin command handlers. The patch is narrowly scoped, but it is not merge-ready because the public Plugin SDK API baseline and required changelog entry are missing, and maintainers still need to accept the stable public field name.

## What This Changes

The PR adds optional `agentId` metadata to `PluginCommandContext`, forwards `HandleCommandsParams.agentId` through plugin command execution, and adds tests for present and omitted values.

## Best Possible Solution

Land the additive command-context metadata after public API approval, with the generated SDK baseline hash, a user-facing changelog entry, and focused command-context tests committed together.

## Reproduction Assessment

Yes. Current main can be checked directly: `HandleCommandsParams.agentId` exists, but `handlePluginCommand`, `executePluginCommand`, and `PluginCommandContext` omit it; a targeted `executePluginCommand` unit test can reproduce the missing `ctx.agentId`.

## Solution Assessment

Mostly yes. The PR's optional field and pass-through are the narrowest maintainable shape for this command-context gap, but the patch needs generated API baseline and changelog follow-through before it is the best mergeable solution.
