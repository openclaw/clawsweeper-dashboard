## Summary

Keep open: current main still lacks structured sandbox-denial classification on the Codex JSONL success path; the focused prior fix is closed unmerged, and the open artifact-finalization PR is complementary rather than a failure-classification fix.

## What This Changes

The issue asks OpenClaw to treat exit-0 Codex CLI JSONL sandbox-denial events as failed runs instead of completed successful subagent work.

## Best Possible Solution

Add structured Codex JSONL sandbox-denial classification before success result construction, map it to a local failover reason/status, exclude it from auth-profile health, and cover both detection and false-positive cases.

## Reproduction Assessment

Yes, source-reproducible. Current main routes exit-0 `codex-cli` JSONL output past the nonzero-exit failover branch into parsing and then records `result: "success"` / `stopReason: "completed"`; I did not run a live Codex command in this read-only pass.

## Solution Assessment

Yes. A structured-field detector in the JSONL parser/runner path, like the closed unmerged implementation at https://github.com/openclaw/openclaw/pull/39460, is the narrowest maintainable fix; output-last-message artifact work is complementary and should not close this bug.
