## Repair Work Prompt

Fix #39317 on current main. Current `codex-cli` exit-0 JSONL output goes through `parseCliOutput` and `buildCliRunResult` as success even when Codex reports a sandbox/tool denial in structured JSONL. Add a focused structured failure-classification path for Codex JSONL before success result construction, preferably by returning failure metadata from `parseCliJsonl`/`parseCliOutput` or a nearby helper and throwing `FailoverError` from the CLI runner. Detect structured error/tool/command/status fields only; do not match ordinary assistant message text. Add a `sandbox_denied` failover/status/auth-profile policy contract only if needed, and ensure local sandbox denial does not mark provider auth profiles unhealthy. Do not change Codex sandbox defaults, output-last-message artifact finalization, plugin boundaries, or unrelated CLI behavior. Add focused regression tests and a single-line `CHANGELOG.md` fix entry for the user-visible behavior change.
