## Summary

Keep open: current main still lets an exit-0 Codex JSONL run bypass CLI failure handling, parses the output normally, and records the CLI attempt as successful. The focused fix PR #39460 was closed unmerged, while #66081/#65074 cover final-message artifact selection rather than semantic failure classification.

## What This Changes

Detect Codex CLI sandbox-denied JSONL failures even when `codex exec --json` exits 0, and propagate the run as failed.

## Best Possible Solution

Add a focused structured Codex JSONL failure-classification path before CLI success result construction, with a `sandbox_denied` failover reason, auth-profile exclusion, regression tests, and no change to sandbox defaults or artifact finalization scope.

## Reproduction Assessment

Yes. The issue provides a concrete `codex exec --json --sandbox read-only` reproduction, and current source shows exit-0 JSONL output bypasses failure handling and is recorded as success.

## Solution Assessment

Yes. A #39460-style structured-field detector is the narrowest maintainable fix; matching normal assistant text would be noisy, and #66081's artifact work is complementary rather than a fix for this bug.
