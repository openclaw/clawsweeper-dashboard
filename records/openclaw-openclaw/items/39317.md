## Summary

Keep open: current main still lacks semantic `sandbox_denied` classification for exit-0 Codex JSONL failures; the focused fix was closed unmerged, and the open artifact-finalization PR is complementary rather than a failure-classification fix.

## What This Changes

Classify Codex `exec --json` sandbox-denied or write-denied JSONL events as failed even when the Codex process exits 0.

## Best Possible Solution

Add structured Codex JSONL sandbox-denial classification before success result construction, with a `sandbox_denied` failover reason/status policy and focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible. The reported exit-0 `codex exec --json --sandbox read-only` path maps to current main where exit-0 output bypasses failover, JSONL parsing has no semantic failure flag, and the runner records success; I did not run a live Codex binary during this read-only review.

## Solution Assessment

Yes. A structured-field detector like the approach in https://github.com/openclaw/openclaw/pull/39460 is the narrowest maintainable fix; https://github.com/openclaw/openclaw/pull/66081 improves final-message selection but does not classify semantic failures.
