Fix https://github.com/openclaw/openclaw/issues/39317 on current main. OpenClaw currently treats exit-0 `codex-cli` JSONL runs as success even when structured Codex output indicates sandbox/write denial or failed tool/action execution. Add structured JSONL failure detection before `buildCliRunResult`, propagate a `FailoverError` reason such as `sandbox_denied` with an appropriate status, and keep local sandbox denial out of auth-profile health tracking. Match structured records only, not ordinary assistant message text, to avoid false positives when the assistant merely discusses sandboxing. Use #39460 as source context but adapt to current files; do not implement or depend on #66081's `--output-last-message` artifact finalization and do not change Codex sandbox defaults. Add/update regression coverage for parser detection, runner exit-0 failure propagation, false-positive prevention, failover status/reason typing, and auth-profile failure policy. Add a user-facing changelog fix entry if behavior changes are committed.
