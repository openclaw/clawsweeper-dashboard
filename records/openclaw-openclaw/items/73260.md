## Summary

Keep open: current main still uses mtime-based `auth-profiles.json` and `models.json` cache inputs, while the PR head still has a size-only oversize sentinel that can preserve cache hits after same-length over-cap rewrites.

## What This Changes

The PR replaces mtime-based models-config cache fingerprint inputs with stable content hashes, adds `modelsJsonHash` drift checks, hardens file reads/chmod, and adds fingerprint-cache tests.

## Best Possible Solution

Keep the content-hash direction, but make over-cap files force a miss or use a bounded content digest with size/truncation metadata, add same-size over-cap regression coverage, and add the required changelog entry.

## Reproduction Assessment

Yes. On PR head, create an over-cap `models.json` or `auth-profiles.json`, warm the ready cache, then rewrite different content with the same byte length; `safeHashRegularFile` can return the same `oversize:<size>` value and preserve the cache hit.

## Solution Assessment

No. The proposed design is the right direction, but the over-cap sentinel is not the narrowest maintainable implementation for drift detection because it is deterministic without reading content.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Force a miss for oversized file hashes:** `src/agents/models-config.ts:115`
  - body: When a file is over the cap, `safeHashRegularFile()` returns `oversize:${lst.size}` without hashing any bytes. Since the cache hit path compares this value directly, a same-size rewrite of an oversized `models.json` or `auth-profiles.json` can keep the ready cache hot and skip re-planning. Return `null`, or include bounded content plus size/truncation metadata in the digest.
  - confidence: 0.9
- **[P3] Add the required changelog entry:** `src/agents/models-config.ts:267`
  - body: This is a user-facing `perf` change: OAuth token refreshes and `models.json` writes should no longer repeatedly rerun provider discovery. Repo policy requires a `CHANGELOG.md` entry for user-facing perf/fix/feat changes, but this diff only updates source and tests.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: Security review needs attention because the PR improves file handling but leaves a concrete cache-integrity bypass in the over-cap hash path.

Concerns:

- **[medium] Size-only oversize hash can bypass drift detection:** `src/agents/models-config.ts:115`
  - body: For over-cap `models.json` or `auth-profiles.json`, the helper returns a deterministic `oversize:<size>` value without content. An external writer that changes an oversized file while preserving byte length can keep the cache hash unchanged and bypass the new re-plan trigger.
  - confidence: 0.9
