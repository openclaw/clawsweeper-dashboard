## Summary

Keep this PR open: current main has not implemented the proposed content-hash/drift-detection behavior, and the PR head is still not merge-ready because its over-cap file hash path can preserve cache hits based only on byte length.

## What This Changes

The PR replaces mtime-based auth-profile and models.json cache fingerprinting with stable content hashes, stores a `modelsJsonHash` in the ready cache, hardens models.json file reads/chmod, and adds fingerprint-cache tests.

## Best Possible Solution

Keep the content-hash direction, but change the over-cap path so oversized files cannot produce stable cache hits from byte length alone, then add regression coverage for same-size over-cap edits to both `models.json` and `auth-profiles.json`.

## Reproduction Assessment

Yes. On PR head, create an over-cap `models.json` or `auth-profiles.json`, warm the ready cache, then rewrite different content with the same byte length; `safeHashRegularFile` can return the same `oversize:<size>` value and preserve the cache hit.

## Solution Assessment

No. The proposed content-hash design is the right direction, but the current oversize sentinel is not the best implementation for a drift-detection path because it is deterministic without reading content.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Force a miss for oversized file hashes:** `src/agents/models-config.ts:115`
  - body: When a file is over the cap, `safeHashRegularFile()` returns `oversize:${lst.size}` without hashing any bytes. Since the cache hit path compares this value directly, a same-size rewrite of an oversized `models.json` or `auth-profiles.json` can keep the ready cache hot and skip re-planning. Return `null` for oversize, or include bounded content plus size/truncation metadata in the digest.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The PR improves several file-handling paths, but the size-only over-cap hash is a concrete cache-integrity weakness in the proposed drift-detection mechanism.

Concerns:

- **[medium] Size-only oversize hash can bypass drift detection:** `src/agents/models-config.ts:115`
  - body: For over-cap `models.json` or `auth-profiles.json`, the helper returns a deterministic `oversize:<size>` value without content. An external writer that changes an oversized file while preserving byte length can keep the cache hash unchanged and bypass the new re-plan trigger.
