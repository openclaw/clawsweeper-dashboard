## Summary

Keep this PR open: current main still has the mtime-based models-config ready-cache fingerprint, so the PR is not obsolete, but the proposed over-cap hash path still leaves a concrete cache-integrity hole that should be resolved before merge.

## What This Changes

The PR replaces mtime-based `auth-profiles.json` and `models.json` cache inputs with stable content hashes, stores a `modelsJsonHash` in the ready cache, hardens models file handling, and adds fingerprint-cache tests.

## Best Possible Solution

Keep the content-hash direction, but change the over-cap path so oversized files cannot produce stable cache hits from byte length alone, then add regression coverage for same-size over-cap edits to both `models.json` and `auth-profiles.json`.

## Reproduction Assessment

Yes. On PR head, an over-cap `models.json` or `auth-profiles.json` can be warmed into the ready cache and then rewritten with different same-length content; the size-only `oversize:<size>` value can remain unchanged and preserve the cache hit.

## Solution Assessment

No. The PR’s overall content-hash design is the right direction, but the size-only over-cap sentinel is not the best implementation for a drift-detection path because it is deterministic without reading any content.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Force a miss for oversized file hashes:** `src/agents/models-config.ts:115`
  - body: When a file is over the cap, `safeHashRegularFile()` returns `oversize:${lst.size}` without hashing any bytes. Since the ready-cache path compares this value directly, same-length edits to oversized `models.json` or `auth-profiles.json` can keep the cache hot and skip re-planning. Return `null` for oversize, or include bounded content plus size/truncation metadata in the digest.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The PR improves several file-handling paths, but the size-only over-cap hash is a concrete cache-integrity weakness in the proposed drift-detection mechanism.

Concerns:

- **[medium] Size-only oversize hash can bypass drift detection:** `src/agents/models-config.ts:115`
  - body: For over-cap `models.json` or `auth-profiles.json`, the helper returns a deterministic `oversize:<size>` value without content. An external writer that changes an oversized file while preserving byte length can keep the cache hash unchanged and bypass the new re-plan trigger.
  - confidence: 0.87
