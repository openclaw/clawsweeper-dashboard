## Summary

Keep this PR open. Current main still uses mtime inputs for the `ensureOpenClawModelsJson` ready cache, so the performance problem is real and not implemented on main; however the PR head still returns a size-only oversize sentinel for the new file-hash path, which can preserve a cache hit across same-size over-cap edits and undercuts the drift-detection/security-hardening claim.

## What This Changes

This PR replaces mtime-based models-config ready-cache inputs with content hashes, stores a `modelsJsonHash` alongside ready-cache entries, hardens `models.json` hashing/chmod paths, and adds fingerprint-cache tests.

## Best Possible Solution

Keep the content-hash direction, but change the oversize branch so over-cap files cannot produce stable cache hits based only on length. The narrow fix is to fail closed as a cache miss for oversize files, or compute a bounded digest that incorporates actual bytes plus size/truncation metadata, with regression coverage for same-size over-cap edits.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: create an over-cap `models.json` or `auth-profiles.json`, warm the readyCache, then rewrite different content with the same byte length; the PR head can return the same `oversize:<size>` value and preserve the cache hit.

## Solution Assessment

No. The content-hash approach is the right maintainable direction, but the current oversize handling is not the best implementation because it keeps a deterministic non-content hash in a path intended to detect content drift.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Force a miss for oversized file hashes:** `src/agents/models-config.ts:115`
  - body: When a file is already over the cap, `safeHashRegularFile()` returns `oversize:${lst.size}` without reading any content. Since cache hits compare this value to the captured `modelsJsonHash`/fingerprint input, a same-size edit to an oversized `models.json` or `auth-profiles.json` can keep the readyCache hot and skip re-planning. Return `null` for oversize or include bounded content in the digest so content drift cannot collide by size alone.
  - confidence: 0.88

## Security Review
