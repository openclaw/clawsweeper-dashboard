## Summary

Keep this PR open: the cache-fingerprint design looks like the right fix and I found no current code-review finding, but the latest token-profile expiry change is only backed by Vitest/type output rather than current-head real behavior proof.

## What This Changes

The PR changes models-config ready-cache fingerprinting from auth/models mtime inputs to content/outcome hashes, adds fail-closed handling for uncacheable auth-profiles.json and models.json, hardens file reads/chmod, and adds regression coverage plus changelog text.

## Best Possible Solution

Land this PR after current-head real behavior proof covers token-profile expiry invalidation and the models-config/auth owners are comfortable with the cache contract.

## Reproduction Assessment

Yes, source-reproducible. Current main fingerprints auth-profiles.json and generated models.json by mtime, while successful auth-profile use rewrites the auth store and can churn the fingerprint.

## Solution Assessment

Yes for the code direction. The content/outcome fingerprint design is the narrow maintainable fix because it removes self-invalidating mtimes while preserving semantic auth changes and fail-closed handling for uncacheable files.
