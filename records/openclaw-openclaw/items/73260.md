## Summary

Keep open. Current main still uses mtime-based models-config ready-cache inputs, so the PR is not obsolete, but the PR head is not merge-ready because its over-cap hash path still returns a deterministic size-only sentinel that can preserve cache hits across same-size content edits. The discussion from Aisle, Greptile, and the prior ClawSweeper review all identified the same cache-integrity gap, and the PR head also has failing node check-runs.

## What This Changes

The PR replaces mtime-based ready-cache inputs in `ensureOpenClawModelsJson` with content hashes, stores `modelsJsonHash` in ready-cache entries, hardens model/auth file reads and chmod handling, and adds fingerprint-cache tests.

## Best Possible Solution

Keep the content-hash direction, but change the over-cap path so oversized files cannot produce stable cache hits based only on byte length. A narrow maintainable fix is to return `null` for over-cap files to force a cache miss, or compute a bounded digest that includes actual bytes plus size/truncation metadata, with regression coverage for same-size over-cap edits.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: warm the ready cache with an over-cap `models.json` or `auth-profiles.json`, then rewrite different content with the same byte length; the PR head can return the same `oversize:<size>` value and keep the cache hit.

## Solution Assessment

No. The overall content-hash approach is a good direction, but this implementation is not the best fix while the oversize branch uses a deterministic non-content sentinel in a path meant to detect content drift.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Force a miss for oversized file hashes:** `src/agents/models-config.ts:115`
  - body: When a file is already over the cap, `safeHashRegularFile()` returns `oversize:${lst.size}` without reading any content. Since cache hits compare this value to the captured `modelsJsonHash` or fingerprint input, a same-size edit to an oversized `models.json` or `auth-profiles.json` can keep the ready cache hot and skip re-planning. Return `null` for oversize or include bounded content in the digest so content drift cannot collide by size alone.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The PR improves several file-handling paths, but the size-only oversize hash is a concrete cache-integrity weakness in the proposed drift-detection mechanism.

Concerns:

An attacker or external process that can rewrite an oversized file to different content with the same length can preserve the cache hash and bypass the new re-plan trigger.
