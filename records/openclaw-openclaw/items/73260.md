## Summary

Keep this PR open: current main still has the mtime-based cache fingerprint behavior the PR is trying to improve, but the PR merge ref still contains a size-only oversize hash that can bypass the new drift-detection path, and the exact head has failing check runs.

## What This Changes

The PR replaces `ensureOpenClawModelsJson` mtime-based ready-cache inputs with auth-profile content hashing plus a stored `models.json` hash check, hardens related file reads/chmod, and adds fingerprint-cache tests.

## Best Possible Solution

Keep the content-hash direction, but change oversize handling so over-cap files cannot produce stable cache hits from size alone: either fail closed as a cache miss or compute a bounded content digest with explicit truncation and size metadata, then add focused regression coverage for same-size over-cap edits before rerunning the touched models-config checks.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: warm the readyCache with an over-cap `models.json` or `auth-profiles.json`, rewrite it to different same-length content, and the PR merge ref still returns the same `oversize:<size>` value before the equality-based cache hit.

## Solution Assessment

No. The proposed content-hash approach is the right narrow direction, but the current oversize sentinel is not the best implementation because it preserves a deterministic non-content hash in the drift-detection path the PR adds.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Force a miss for oversized file hashes:** `src/agents/models-config.ts:115`
  - body: When a file is already over the cap, `safeHashRegularFile()` returns `oversize:${lst.size}` without reading any content. Since cache hits compare this value to the captured `modelsJsonHash`, a same-size edit to an oversized `models.json` or `auth-profiles.json` can keep the readyCache hot and skip re-planning. Return `null` for oversize or include bounded content in the digest so content drift cannot collide by size alone.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The PR improves several file-handling paths, but the new size-only oversize hash is a concrete cache-integrity weakness in the proposed drift-detection mechanism.

Concerns:

- **[medium] Size-only oversize hash can bypass drift detection:** `src/agents/models-config.ts:115`
  - body: For over-cap `models.json` or `auth-profiles.json`, the helper returns a deterministic `oversize:<size>` value without content. An attacker or external process that can rewrite an oversized file to different content with the same length can preserve the cache hash and bypass the new re-plan trigger.
