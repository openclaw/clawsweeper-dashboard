## Summary

## What This Changes

The issue requests Codex-OAuth-only media understanding to use a Codex-backed vision route instead of requiring `OPENAI_API_KEY` for an OpenAI image/vision fallback.

## Best Possible Solution

Codify the intended media route: either support Codex OAuth fallback for `openai/gpt-*` image understanding with tests, or document and diagnose that Codex-only installs must select `codex/gpt-*` or another Codex-auth provider explicitly.

## Solution Assessment

Unclear; routing through a Codex media provider may be the right fix, but current docs distinguish `openai/gpt-*` from `codex/gpt-*`. A maintainer should choose that contract before automation changes auth fallback behavior.
