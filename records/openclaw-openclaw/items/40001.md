## Label Justifications

- `P1`: Ordinary agent and isolated cron writes can silently destroy persistent workspace content in active user workflows.
- `impact:data-loss`: The reported behavior truncates existing files without a recovery path inside OpenClaw.
- `impact:session-state`: Affected files include durable memory and operational context used across agent sessions.

## Summary

Keep open: current `main` and release `v2026.6.5` still expose ordinary agent `write` as overwrite-only, while append-only handling is limited to memory-flush runs. Multiple independent reports confirm silent truncation, and the open closing PR remains the active implementation path but still has a path-normalization review finding and requires maintainer approval of the model-facing API.

## What This Changes

Add a safe append-capable agent file-writing operation so ordinary and isolated sessions can extend shared workspace files without truncating existing content.

## Best Possible Solution

Land a single optional append mode on the existing root-scoped `write` tool after reusing the complete overwrite-path normalization contract, preserving overwrite as the default, and retaining serialized same-file mutations across host and sandbox paths.

## Reproduction Assessment

Yes at source level: current ordinary writes have overwrite semantics, and append-only behavior is confined to the memory-flush wrapper. Multiple independent production reports provide matching truncation and byte-shrinkage evidence, although this read-only review did not perform another destructive live run.

## Solution Assessment

Not yet in its present branch state. An optional append mode on the existing write tool is the narrowest maintainable direction, but the active PR must first share the established path-normalization behavior and receive maintainer approval for the public tool-contract change.
