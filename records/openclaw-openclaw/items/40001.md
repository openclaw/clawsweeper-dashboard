## Summary

Keep this open. Current main still exposes ordinary host and sandbox `write` as overwrite-oriented outside the memory-flush wrapper, and the open closing PR is the implementation candidate rather than a reason to close the issue before merge.

## What This Changes

The issue requests an append-safe file write primitive for ordinary agent and isolated cron sessions so shared workspace files are not overwritten when the agent intends to append.

## Best Possible Solution

Land a maintainer-approved append-capable primitive that preserves workspace and sandbox guards, validates malformed append intent safely, and then close this issue after merge.

## Reproduction Assessment

Yes. Source inspection shows ordinary host and sandbox `write` still routes through overwrite-style operations, while append-only behavior is scoped to memory-flush runs; the issue thread adds production cron evidence on the same path.

## Solution Assessment

Yes as a direction, but not complete yet. A first-class append-capable file primitive is the narrowest durable fix boundary, and the exact public API shape should be settled through the active PR review.
