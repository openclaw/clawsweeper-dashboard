## Summary

Keep this open. Current main still exposes ordinary host and sandbox `write` as overwrite-only outside the memory-flush wrapper, and the linked open implementation PR should be reviewed or rejected before this issue is closed.

## What This Changes

The issue requests a safe append-capable file write path for ordinary and isolated cron agent sessions so shared workspace files are not silently overwritten.

## Best Possible Solution

Review and land or explicitly reject https://github.com/openclaw/openclaw/pull/77127, then close this issue only after ordinary writes have safe append semantics on main.

## Reproduction Assessment

Yes. Source inspection shows ordinary host and sandbox `write` still overwrite on current main, while the issue and duplicate reports provide concrete production cron evidence for the same path.

## Solution Assessment

Yes as a direction, but not complete yet. A first-class append-capable file primitive that preserves workspace and sandbox guards is the maintainable fix; the exact API shape should be settled in the active PR review.
