## Summary

Keep this open. Current main still exposes ordinary agent `write` calls as overwrite-only outside the memory-flush wrapper, and https://github.com/openclaw/openclaw/pull/77127 remains open as the closing implementation path for maintainer API review.

## What This Changes

The issue requests a first-class append-safe write path so isolated cron sessions can add to shared memory files without destructive full-file overwrites.

## Best Possible Solution

Land a reviewed append-capable write path that preserves existing workspace, sandbox, validation, abort, and path-guard behavior, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes. Current main is source-reproducible: ordinary host, workspace, and sandbox `write` calls still overwrite, while isolated cron runs use the `cron` trigger that bypasses the memory-flush append-only wrapper.

## Solution Assessment

Yes as a direction, but not yet as a completed repository state. A first-class append-capable write path is the narrow durable fix; maintainers still need to settle and review the API shape in the open implementation PR.
