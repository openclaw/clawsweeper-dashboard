## Summary

Keep open: current main and v2026.6.6 still expose the general agent `write` tool as path/content overwrite-only, while the only append behavior is the memory-flush-specific wrapper. The open linked PR is the active implementation path and should be reviewed, landed, replaced, or rejected before this issue closes.

## What This Changes

The issue requests a first-class append path for the agent `write` tool so isolated cron and normal agent sessions can append to shared workspace files without overwriting existing content.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/77127, or replace it with an equivalent maintainer-approved core append primitive that preserves workspace and sandbox guards plus the exported tool-call contract.

## Reproduction Assessment

Yes, at source level. Current main's write schema/input only accept `path` and `content`, ordinary execution overwrites via `ops.writeFile`, and append behavior is limited to memory-flush runs; I did not run a mutating cron repro in this read-only review.

## Solution Assessment

Yes in direction, but not yet as a completed repository outcome. A first-class append primitive is a better fix than prompt guards or `exec >>`, but maintainers still need to approve the exact API shape and land or replace the linked PR.
