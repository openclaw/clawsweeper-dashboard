## Summary

Keep this issue open: current main and v2026.6.10 still expose ordinary `write` as overwrite-only, while https://github.com/openclaw/openclaw/pull/77127 is the active open implementation path rather than a reason to close before merge.

## What This Changes

Request an append-capable write primitive or mode so isolated cron and ordinary agent writes can append to shared workspace and memory files without truncating existing content.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/77127 or replace it with an equally narrow append-capable write contract that preserves existing overwrite defaults and workspace/sandbox guards.

## Reproduction Assessment

Yes. Current main is source-reproducible: ordinary `write` accepts only `path` and `content`, calls `ops.writeFile`, and isolated cron uses `trigger: "cron"` rather than the memory-only append wrapper.

## Solution Assessment

Yes as a direction, but not as completed work. A first-class append mode is the maintainable boundary because prompt-only guards and read-concatenate-overwrite flows are racy; the linked PR should be reviewed, landed, or explicitly replaced.
