## Summary

Keep open: current main and the latest release still expose ordinary `write` as overwrite-only, and the open linked PR https://github.com/openclaw/openclaw/pull/77127 is the active fix path for this canonical data-loss report.

## What This Changes

The issue requests an append-capable write path so agent and cron writes can add to shared workspace files without truncating existing content.

## Best Possible Solution

Review and either land or explicitly reject https://github.com/openclaw/openclaw/pull/77127; if rejected, replace it with an equivalent append primitive that preserves overwrite as the default and keeps workspace, sandbox, and mutation-queue guards intact.

## Reproduction Assessment

Yes, at source level. Current main's ordinary write tool still accepts only `path` and `content` and calls the overwrite operation, while the issue and duplicate reports include concrete cron and single-agent truncation examples.

## Solution Assessment

Yes for keeping it open, not for closing it now. A first-class append primitive is the narrow durable fix compared with prompt guards or `exec >>`, and the open linked PR is the current implementation candidate pending maintainer API acceptance.
