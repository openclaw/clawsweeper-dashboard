## Summary

Keep this issue open: current `main` and the latest release still expose ordinary `write` as create/overwrite only, while the append behavior is limited to the memory-flush wrapper. The issue already has an open linked implementation PR, so this should remain the canonical tracker until that PR lands or maintainers choose a different append-tool contract.

## What This Changes

The issue requests append-capable agent file writes so isolated cron or agent sessions do not overwrite shared workspace memory files.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/77127 or choose a different explicit append-tool contract that preserves overwrite-by-default behavior plus host, workspace, and sandbox guards.

## Solution Assessment

Yes, a first-class append path at the write-tool contract is the right layer; current memory-flush-only wrapping proves prompt-only or subsystem-only fixes leave ordinary agent/cron writes exposed.
