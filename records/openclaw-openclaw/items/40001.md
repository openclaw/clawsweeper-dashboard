## Summary

Keep open: current main and the latest release still expose ordinary `write` as create-or-overwrite only, while the append-only wrapper is scoped to memory-flush runs. The open linked pull request already owns the proposed append primitive, so this issue should stay open as the canonical tracker until that PR lands, is replaced, or maintainers choose a different API shape.

## What This Changes

Add a first-class append-capable file write primitive or mode so ordinary agent and isolated cron writes can add to shared files without truncating existing content.

## Best Possible Solution

Review and land, replace, or deliberately reject https://github.com/openclaw/openclaw/pull/77127 with a maintainer-approved append primitive that keeps overwrite as the default and preserves workspace/sandbox guards plus same-file serialization.

## Reproduction Assessment

Yes, at source level. Current main exposes ordinary `write` as overwrite-only and gates append behavior behind the memory-flush trigger; I did not run a destructive live cron reproduction in this read-only review.

## Solution Assessment

Unclear until maintainers accept the API shape. A first-class append primitive is the maintainable direction compared with prompt guards or `exec >>`, but the linked PR's public write-contract change needs maintainer approval before merge.
