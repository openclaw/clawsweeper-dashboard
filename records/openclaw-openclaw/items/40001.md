## Summary

Keep this open. Current main still exposes ordinary host and sandbox `write` as overwrite-only outside the memory-flush wrapper, and the open closing PR is the implementation candidate rather than a reason to close before merge.

## What This Changes

Add a first-class append-capable path for agent file writes so isolated cron and monitor workflows do not have to emulate append with destructive read-concatenate-write behavior.

## Best Possible Solution

Review and land or explicitly reject https://github.com/openclaw/openclaw/pull/77127 after maintainer API review, then close this issue only once ordinary writes have safe append semantics on main.

## Reproduction Assessment

Yes. Current source shows ordinary host and sandbox `write` still overwrite, while the issue comments provide production cron evidence for the same path.

## Solution Assessment

Yes as a direction, but not complete yet. A first-class append-capable file primitive that preserves workspace and sandbox guards is the maintainable fix; the exact API shape should be settled in the active PR review.
