## Summary

Keep this open. Current main still exposes ordinary agent and cron `write` calls as create-or-overwrite, while append-safe behavior is limited to memory-flush runs; the active closing PR is the right implementation candidate rather than a reason to close this issue before merge.

## What This Changes

The issue requests a model-facing append capability for file writes so cron and ordinary agent runs can add to shared memory/log files without truncating existing content.

## Best Possible Solution

Review and land a maintainer-approved append-capable file primitive through https://github.com/openclaw/openclaw/pull/77127 or a revised equivalent, preserving workspace and sandbox path policy, then close this issue after merge.

## Reproduction Assessment

Yes, from source inspection: ordinary `write` still routes to overwrite-only operations, and the append wrapper is only enabled for memory-flush runs. I did not run a live cron because this was a read-only review.

## Solution Assessment

Unclear as a final API choice. A first-class append surface is the right fix boundary, but maintainers still need to choose or approve the public shape and finish review of the active PR.
