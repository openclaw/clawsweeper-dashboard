## Summary

Keep this issue open: current main and the latest release still expose ordinary `write` as overwrite-only, while the open linked PR is the active append-mode implementation path rather than a reason to close before merge.

## What This Changes

Requests a first-class append-capable write path so isolated cron and agent sessions can add to shared memory/workspace files without overwriting existing content.

## Best Possible Solution

Land a narrow append-capable write contract through the linked PR or a direct successor while preserving overwrite as the default and keeping workspace/sandbox guards intact.

## Reproduction Assessment

Yes, source-level. Current main defines ordinary `write` as `path` plus `content` and executes overwrite-style `writeFile`; I did not run a live cron repro in this read-only review.

## Solution Assessment

Yes as a direction, but not as completed work. A first-class append mode is the maintainable boundary because prompt-only guards and read-concatenate-overwrite flows are racy; the linked PR should be reviewed, landed, or explicitly replaced.
