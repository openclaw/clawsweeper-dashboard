## Summary

Keep this issue open: current main and the latest release still expose the normal write tool as create/overwrite only, while the open linked PR is an implementation candidate rather than a merged fix.

## What This Changes

The issue asks for a safe append-capable write primitive so agent and isolated cron writes do not truncate shared workspace or memory files.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/77127 or an equivalent maintainer-approved append API that keeps overwrite as the default while preserving workspace, sandbox, and file-mutation guards.

## Reproduction Assessment

Yes. Source inspection is enough to reproduce the contract gap: normal write accepts only path/content, calls overwrite-style write operations, and cron-triggered runs do not activate the memory-flush append wrapper.

## Solution Assessment

Yes in direction, with one open product detail: an explicit append mode or append tool is the narrow maintainable fix, but maintainers still need to settle the exact API shape before the linked PR lands.
