## Summary

Keep this open. Current main still exposes ordinary agent and cron `write` as overwrite-oriented outside the memory-flush wrapper, and the open closing PR is an implementation candidate rather than a reason to close the issue before merge.

## What This Changes

Add an append-capable file write path so isolated cron and ordinary agent runs can add to shared workspace logs or memory files without truncating existing content.

## Best Possible Solution

Land or replace the active append-mode PR with a maintainer-approved primitive that preserves workspace and sandbox guards, rejects malformed append intent safely, and then close this issue after merge.

## Reproduction Assessment

Yes, at source level. Ordinary host and sandbox `write` paths still route through overwrite-style operations while append-only behavior is scoped to memory-flush runs; the issue thread adds production cron evidence on the same path.

## Solution Assessment

Yes as a direction, but not complete yet. A first-class append-capable file primitive is the narrowest durable fix boundary, and the exact public API shape should be settled through the active PR review.
