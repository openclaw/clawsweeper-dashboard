## Summary

Keep this open. Current main still exposes normal agent `write` as overwrite-only outside the memory-flush wrapper, and the linked implementation PR is still open as the closing path.

## What This Changes

The issue requests an append-capable write primitive so cron and isolated agent sessions can add to shared workspace files without overwriting existing content.

## Best Possible Solution

Land a reviewed append-capable write path that preserves existing workspace, sandbox, validation, abort, and path-guard behavior, then close this issue after it reaches main.

## Reproduction Assessment

Yes. Current main is source-reproducible: normal host, workspace, and sandbox write paths still overwrite, while cron runs do not enter the memory-flush append-only wrapper.

## Solution Assessment

Yes as a direction, but not yet as completed repository state. A first-class append-capable write path is the narrow durable fix; maintainer review should settle the exact public API shape in the open implementation PR.
