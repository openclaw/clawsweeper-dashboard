## Summary

Keep open. Current main and v2026.6.6 still expose the ordinary agent `write` tool as create/overwrite only, and the linked open PR is the active fix path rather than a merged implementation.

## What This Changes

The issue requests safe append semantics for the agent write tool so isolated cron and ordinary agent sessions can add to shared workspace files without truncating existing content.

## Best Possible Solution

Review and land the linked append-mode PR, or choose a different maintainer-approved append primitive, while keeping overwrite as the default and preserving workspace, sandbox, and validation behavior.

## Reproduction Assessment

Yes, at source level. Current main's ordinary `write` tool accepts `path` and `content` and overwrites via `writeFile`; only memory-triggered flush writes get a dedicated append-only wrapper.

## Solution Assessment

Yes for keeping this issue open. The linked append-mode PR is the right review target because prompt guards and `exec >>` workarounds do not preserve the write tool's workspace policy boundary, but maintainers still need to accept the exact API shape.
