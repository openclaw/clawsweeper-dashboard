## Summary

Keep this open: current main and the latest release still expose ordinary agent and cron `write` as overwrite-only, while the append-capable fix PR that closes this issue is still open and unmerged.

## What This Changes

The issue requests append-safe file write semantics for ordinary agent and cron `write` calls so shared memory and log files are not overwritten.

## Best Possible Solution

Review and land a maintainer-approved append-capable write path, likely through the linked PR or a replacement, while preserving overwrite defaults, workspace and sandbox guards, and focused regression coverage.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence. Current main accepts only `path` and `content` for ordinary `write`, routes those calls to overwrite operations, and cron runs do not activate the memory-flush append wrapper.

## Solution Assessment

Yes as a direction, but not as completed main behavior. A first-class append-capable write path is the durable fix; maintainers still need to approve the exact public tool API in the linked PR.
