## Summary

Keep this issue open: current main and the latest release still expose ordinary `write` as overwrite-only, while the open linked PR is the active fix path and needs maintainer API review before this can be closed.

## What This Changes

The issue requests a first-class append path for agent file writes so cron and isolated sessions do not silently overwrite shared workspace memory files.

## Best Possible Solution

Review and either land https://github.com/openclaw/openclaw/pull/77127 or replace it with a maintainer-approved append primitive that keeps overwrite as the default and preserves workspace, sandbox, and mutation-queue guards.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.8 expose ordinary `write` as `path` plus `content` and execute it through overwrite operations, while the append-only wrapper is restricted to memory-triggered flush runs.

## Solution Assessment

Mostly yes, pending maintainer API acceptance. A first-class append primitive is narrower and safer than prompt guards or shell redirection, but the exact public write-tool shape should be confirmed before merge.
