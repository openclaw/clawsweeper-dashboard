## Summary

Keep this open. Current main and the latest v2026.5.12 release still expose ordinary `write` as create/overwrite outside the memory-flush wrapper, and the open closing PR is the implementation candidate rather than a reason to close this issue before merge.

## What This Changes

Request an append-capable file primitive or mode so ordinary agent and cron writes can append to shared workspace files without destructive rewrites.

## Best Possible Solution

Review and land or explicitly reject the existing append-mode PR after maintainer API review, then close this issue only after ordinary writes have safe append semantics on main.

## Reproduction Assessment

Yes. Source inspection shows ordinary host and sandbox `write` still uses upstream overwrite-style write operations, while the issue comments add production cron evidence on the same path.

## Solution Assessment

Yes as a direction, but it is not complete yet. A first-class append-capable file primitive preserving workspace and sandbox guards is the narrow maintainable fix; the exact public API shape should be settled in the active PR review.
