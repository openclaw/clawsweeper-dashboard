## Label Justifications

- `P1`: The issue describes repeated silent data loss in real cron and agent workflows, and current main still lacks a general append primitive.
- `impact:data-loss`: Overwrite-only `write` calls can replace existing memory or workspace files without built-in recovery.
- `impact:session-state`: The affected shared memory files preserve operational and session context across agent runs.

## Summary

Keep open: current main and v2026.5.19 still expose general `write` as overwrite-only, and the issue is paired with an open closing PR for the append contract.

## What This Changes

The issue requests a safe append-capable write primitive for agent and cron workflows so shared workspace and memory files can accumulate content without being replaced.

## Best Possible Solution

OpenClaw should expose a maintainer-approved append-capable file primitive that preserves overwrite defaults, workspace and sandbox path guards, mutation-queue behavior, docs, and focused regression coverage.

## Reproduction Assessment

Yes. Source inspection shows ordinary `write` still accepts only `path` and `content` and routes to overwrite operations, while cron runs use the normal tool path outside the memory-flush append-only wrapper.

## Solution Assessment

Yes as a direction, but not as completed main behavior. A first-class append-capable write path is the durable fix; maintainers still need to approve the exact API shape and concurrency behavior in the linked implementation path.
