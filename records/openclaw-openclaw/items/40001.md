## Summary

Keep this open. Current main still exposes normal host and sandbox `write` as overwrite-only outside the memory-flush wrapper, and the open closing PR is the implementation candidate rather than a reason to close the issue before merge.

## What This Changes

The issue requests append-safe file writing for ordinary agent and isolated cron runs so shared workspace files are not silently overwritten.

## Best Possible Solution

Review and land or explicitly reject https://github.com/openclaw/openclaw/pull/77127, then close this issue only after ordinary writes have safe append semantics on main.

## Reproduction Assessment

Yes. Current source shows normal host and sandbox `write` still overwrite, while the issue comments and duplicate thread provide production cron evidence for the same path.

## Solution Assessment

Yes as a direction, but not complete yet. A first-class append-capable file primitive that preserves workspace and sandbox guards is the maintainable fix; the exact API shape should be settled in the active PR review.
