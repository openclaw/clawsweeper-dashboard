## Summary

Keep this open. Current main still exposes ordinary agent and cron `write` as an overwrite-only file tool, and the open closing PR is the right implementation path to review before this issue can close.

## What This Changes

The issue requests an append-capable write path so isolated cron and agent sessions can update shared memory or log files without overwrite-style data loss.

## Best Possible Solution

Land a reviewed append-capable file primitive that preserves overwrite as the default plus existing workspace, sandbox, validation, abort, and queued-mutation behavior, then close this issue after it reaches main.

## Reproduction Assessment

Yes. Current main is source-reproducible: ordinary host and sandbox write paths still use replacement writes with only `path` and `content`, while append-only behavior is gated to memory-flush runs.

## Solution Assessment

Yes as a direction, but not yet as completed repository state. A first-class append-capable write path is the narrow durable fix; maintainer review should settle the exact public API shape in the open implementation PR.
