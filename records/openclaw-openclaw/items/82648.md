## Summary

Keep open. The PR addresses a plausible current-main media resolution gap, but the proposed resolver needs security/product review for cross-conversation state-dir reads and CI is currently blocked by an introduced lint failure.

## What This Changes

The PR adds an LCM `file_<16hex>` image resolver, wires it into prompt image loading and the image tool, and adds focused tests for managed `lcm-files` images.

## Best Possible Solution

Decide whether core should own this LCM media contract; if yes, scope the lookup to a trusted conversation/session or explicitly document token-as-capability semantics, then fix lint and keep the focused resolver tests.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main has no prompt `lcm-file-ref` detector and the image tool treats opaque refs as ordinary image sources. I did not run a live LCM setup.

## Solution Assessment

Unclear. Resolving managed refs before workspace fallback is plausible, but the current global `lcm-files` scan and core-specific LCM contract need maintainer validation before this is the best maintainable fix.
