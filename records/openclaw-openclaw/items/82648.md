## Summary

Keep open: the PR targets a plausible real LCM image-ref failure, but the proposed fix hard-codes Lossless storage into core media resolution and misses several image formats already accepted by the prompt image path.

## What This Changes

The PR adds a core LCM `file_<16hex>` image resolver under `src/media`, wires it into prompt image loading and the image tool, and adds focused tests.

## Best Possible Solution

Resolve externalized LCM images through a generic or plugin-owned managed-media reference contract shared by prompt image loading and tools, reusing the existing image type policy.

## Reproduction Assessment

Yes, source-reproducible. Current main has no strict `file_<16hex>` managed-media resolver, so those tokens can fall through to workspace/path handling; the PR body also includes after-fix terminal output for the loader path.

## Solution Assessment

No, not as written. The branch demonstrates the failing path, but the maintainable fix should avoid baking `lcm-files` into core and should share the current supported image-format policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep LCM storage out of core media resolution:** `src/media/lcm-file-reference.ts:25`
  - body: This roots a shared core resolver at `stateDir/lcm-files`, which makes the Lossless/LCM storage layout part of core media resolution. Core policy keeps owner-specific storage behavior in plugins or generic contracts, and the plugin API already has hosted media resolver seams; route this through a generic/plugin-owned media-reference contract instead.
  - confidence: 0.84
- **[P2] Reuse the existing image extension policy:** `src/media/lcm-file-reference.ts:7`
  - body: The resolver only searches `.webp`, `.png`, `.jpg`, `.jpeg`, `.gif`, and `.svg`, while the existing prompt-image path also accepts `.bmp`, `.tiff`, `.tif`, `.heic`, and `.heif`. LCM refs for those already-supported uploaded images would still miss this resolver and fall back to the broken workspace lookup.
  - confidence: 0.79
