## Summary

Keep this PR open. ## What This Changes

The PR changes plugin install/update, onboarding repair, doctor repair, and legacy migration code so managed official npm plugin install records store exact resolved registry specs and equivalent legacy bare records do not conflict.

## Best Possible Solution

Land the exact-spec canonicalization only if maintainers agree that managed official npm install records should store the resolved artifact identity while user-authored third-party moving specs remain warning-worthy.

## Solution Assessment

Yes. Centralizing the spec decision in a small helper and applying it to each managed official writer is a cleaner fix than weakening the security audit, and the added tests cover third-party moving specs, manual exact pins, and migration conflicts.
