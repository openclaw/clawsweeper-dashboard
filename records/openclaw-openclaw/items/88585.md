## Summary

Keep open: the useful pinning fix is not merge-ready because this PR makes manual `openclaw plugins update <id>` ignore documented exact official npm pins and still lacks after-fix real behavior proof.

## What This Changes

The PR changes plugin install/update, doctor repair, onboarding, and legacy state migration paths so official npm plugin records store exact resolved specs and equivalent legacy records do not warn.

## Best Possible Solution

Keep exact resolved-spec persistence for automatic official update/doctor/onboarding writes, but preserve stored exact pins for manual id-based plugin updates unless the user supplies a new spec.

## Reproduction Assessment

Yes, source-reproducible: PR head passes `syncOfficialPluginInstalls: true` from the manual update command, and `updateNpmInstalledPlugins` uses the official override before the stored record spec. I did not run a live patched update/status flow.

## Solution Assessment

No. Exact resolved-spec persistence is the right direction, but enabling official sync in the manual plugins update command is too broad because id-based updates should reuse tracked exact specs.
