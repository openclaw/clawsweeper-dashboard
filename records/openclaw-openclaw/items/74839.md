The PR adds an OpenRouter `resolveThinkingProfile` hook that exposes `off`, `low`, `medium`, `high`, and `xhigh` for cached reasoning-capable OpenRouter models while excluding unsupported Hunter Alpha proxy routes.

## Best Possible Solution

Land the provider-local thinking-profile hook with a single-line changelog entry and targeted OpenRouter tests that cover a reasoning model and the unsupported Hunter Alpha boundary.

## Solution Assessment

Yes, with small completion. A provider-owned `resolveThinkingProfile` hook is the narrowest maintainable solution because core already delegates provider-specific thinking level sets to provider plugins.
