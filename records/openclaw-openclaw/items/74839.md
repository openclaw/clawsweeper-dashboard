The PR adds an OpenRouter provider `resolveThinkingProfile` hook that advertises `off`, `low`, `medium`, `high`, and `xhigh` for cached reasoning-capable OpenRouter models while excluding the Hunter Alpha proxy route.

## Best Possible Solution

## Reproduction Assessment

Yes. Current main can be reproduced by code inspection: without an OpenRouter `resolveThinkingProfile` hook, explicit `/think xhigh` validation falls back to the base profile that stops at `high`.

## Solution Assessment

Yes, with small merge-readiness cleanup. A provider-owned `resolveThinkingProfile` hook is the narrowest maintainable fix because the core resolver already delegates custom level sets to provider plugins.
