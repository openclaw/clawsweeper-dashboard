## Summary

Keep open. The PR targets a valid OpenRouter bug and uses the right provider-owned seam, but the proposed hook depends on the synchronous OpenRouter capability cache, so the reported `/think xhigh` path can still fail on a cold cache before dynamic model metadata is prepared.

## What This Changes

The PR adds an OpenRouter `resolveThinkingProfile` hook that exposes `off`, `low`, `medium`, `high`, and `xhigh` for cached reasoning-capable OpenRouter models while excluding unsupported Hunter Alpha proxy routes.

## Best Possible Solution

Keep the fix provider-owned, but make OpenRouter xhigh support deterministic for the reported DeepSeek V4 refs even when the capability cache is cold, preserve the Hunter Alpha exclusion and non-reasoning fallback, add focused OpenRouter thinking-profile coverage, and record the user-facing fix in the Unreleased changelog.

## Reproduction Assessment

Yes. Current-main inspection gives a high-confidence reproduction path: OpenRouter has no thinking profile hook, and `/think xhigh` validation only accepts `xhigh` when a provider profile, catalog compat, or legacy hook advertises it. The PR branch should also be checked with a cold OpenRouter capability cache because the new hook reads only the sync cache.

## Solution Assessment

No. `resolveThinkingProfile` is the right seam, but this implementation is not the best fix until it avoids the cold-cache miss and has targeted tests for the reported reasoning model, a non-reasoning model, and the Hunter Alpha exclusion.
