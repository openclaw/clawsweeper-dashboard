## What This Changes

The PR adds an OpenRouter `resolveThinkingProfile` hook that exposes `off`, `low`, `medium`, `high`, and `xhigh` for cached reasoning-capable OpenRouter models while excluding the Hunter Alpha proxy route.

## Best Possible Solution

Land the provider-owned OpenRouter thinking profile after adding the changelog entry and targeted tests, without changing the core thinking API.

## Reproduction Assessment

Yes. Current main can be reproduced by code inspection: the core resolver asks for a provider `resolveThinkingProfile`, but OpenRouter does not register one, so `/think xhigh` falls back to levels that stop at `high`.

## Solution Assessment

Yes, with small merge-readiness cleanup. A provider-owned `resolveThinkingProfile` is the narrowest maintainable fix because OpenRouter already owns dynamic capability detection and core already delegates custom thinking levels to provider hooks.
