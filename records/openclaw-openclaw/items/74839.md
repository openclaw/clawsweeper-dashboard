## Summary

Keep open. The PR targets a valid OpenRouter provider bug and the provider-owned hook is the right shape, but it still needs merge-readiness cleanup: a required changelog entry and focused regression coverage for the new thinking profile behavior.

## What This Changes

The PR adds an OpenRouter `resolveThinkingProfile` hook that exposes `off`, `low`, `medium`, `high`, and `xhigh` for cached reasoning-capable OpenRouter models while excluding the Hunter Alpha proxy route.

## Best Possible Solution

Land the provider-owned OpenRouter thinking profile with a changelog entry and targeted tests that prove xhigh is exposed only for reasoning-capable OpenRouter models and remains excluded for known unsupported proxy routes.

## Reproduction Assessment

Yes, by source inspection. Current main has no OpenRouter `resolveThinkingProfile` hook, so explicit `xhigh` validation uses the shared fallback path unless catalog compat already advertises xhigh.

## Solution Assessment

Yes, with small cleanup. A provider-owned `resolveThinkingProfile` hook is the narrowest maintainable fix because core already delegates provider-specific thinking level sets to provider plugins.
