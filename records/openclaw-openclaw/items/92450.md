## Summary

This is a valid, newly actionable feature request. Telegram Bot API 10.1 added bot-native rich messages and table blocks on June 11, 2026, while current OpenClaw still converts markdown tables through its existing HTML/code fallback renderer. Keep the issue open for a Telegram-owned design covering normal sends, edits, streaming drafts, replies, and safe fallback behavior.

## What This Changes

Add native Telegram rich-message table rendering for outbound markdown tables while retaining existing delivery fallbacks.

## Best Possible Solution

Extend the Telegram-owned markdown renderer to produce Bot API 10.1 rich table blocks for eligible text replies, with shared send/edit/draft semantics and an atomic fallback to the existing table-mode HTML path.

## Reproduction Assessment

Yes for the capability gap: current main has explicit table fallback conversion but no rich-message API path, while Telegram’s official Bot API now exposes native table blocks.

## Solution Assessment

Yes in direction, but the maintainable solution is to extend the canonical Telegram renderer and all applicable delivery operations rather than add one-off table detection around a single sendMessage call.
