## Summary

Keep open. The PR moves the live-session model-switch check in the right direction, but it still leaves two fallback-capable embedded runner entrypoints using the active fallback candidate as the live default instead of the caller-selected model.

## What This Changes

The PR adds liveModelDefaultProvider/liveModelDefaultModel to embedded PI runs, changes live-session model selection, threads the defaults through cron and the main auto-reply runner path, adds regression tests, and updates the changelog.

## Best Possible Solution

Complete the stable live-default contract for every fallback-capable embedded runner entrypoint, with the original caller-selected provider/model passed separately from the active fallback candidate and covered by focused tests.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a queued follow-up or direct `openclaw agent --model` run where model fallback invokes `runEmbeddedPiAgent` with a different fallback candidate while `liveModelSwitchPending` is set.

## Solution Assessment

No. The new runner params are the narrow maintainable mechanism, but the current PR is incomplete because two entrypoints still omit those params and therefore keep the original bug on those paths.
