## Summary

Keep open: this PR targets a real live-session model-selection bug, but the current diff still leaves fallback-capable embedded runner entrypoints that do not preserve the original caller-selected model.

## What This Changes

The PR adds live-model default parameters to the embedded runner, forwards them through cron and the main auto-reply execution helper, adds focused live-switch tests, and adds a changelog entry.

## Best Possible Solution

Complete one stable live-default contract across every fallback-capable embedded runner entrypoint, with regression coverage for direct agent commands and queued follow-ups as well as the cron and heartbeat paths already touched.

## Reproduction Assessment

Yes, at source level. The remaining queued follow-up and direct agent command paths call runEmbeddedPiAgent from runWithModelFallback with only the active fallback provider/model, so the new live-default fields would default to the wrong model there.

## Solution Assessment

No. The new live-default fields are the right narrow mechanism, but the patch is not complete until all fallback callsites pass the original caller-selected provider/model into the embedded runner.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
