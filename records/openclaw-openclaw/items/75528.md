## Summary

Keep open. The PR is moving in the right direction, but the latest head still leaves two fallback-capable embedded-runner callsites without the stable caller-selected model, so explicit model runs can still be evaluated against a fallback candidate.

## What This Changes

The branch adds live-model default parameters to embedded agent runs, changes live-session switch resolution to use caller defaults, threads them through cron and auto-reply runner execution, adds regression tests, and updates the changelog.

## Best Possible Solution

Complete the stable live-default contract across every fallback-capable embedded runner entrypoint, with focused tests for cron, heartbeat/auto-reply runner execution, queued follow-ups, direct agent commands, and the core live switch helper.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a queued follow-up or direct `openclaw agent --model` run where the fallback candidate differs from the caller-selected model while `liveModelSwitchPending` is set; the remaining callsites still pass only the active fallback provider/model.

## Solution Assessment

No. The new runner fields are a narrow maintainable direction, but the patch is not complete until all fallback-capable callsites pass the original caller-selected provider/model separately from the active fallback candidate.
