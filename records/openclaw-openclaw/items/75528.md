## Summary

Keep open: the PR addresses a real live-session model-default bug, but the current patch is incomplete because two fallback-capable embedded runner entrypoints still do not preserve the original caller-selected model separately from the active fallback candidate.

## What This Changes

The PR adds live-session default model plumbing to the embedded runner, threads it through cron and shared auto-reply run params, adds focused tests, and updates the changelog.

## Best Possible Solution

Use one stable caller-default contract across embedded runner entrypoints, with original provider/model propagated separately from active fallback candidates and covered by focused regression tests.

## Reproduction Assessment

Yes, at source level. A fallback attempt from a caller-selected model in queued followups or `openclaw agent --model` would call `runEmbeddedPiAgent` with only the fallback candidate, so the PR's new live defaults collapse to the wrong model there.

## Solution Assessment

No. The live-default mechanism is the right narrow direction, but the patch is not complete until all fallback-capable embedded runner entrypoints pass the original caller-selected provider/model.
