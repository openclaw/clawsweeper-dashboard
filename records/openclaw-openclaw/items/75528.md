## Summary

Keep open. The PR targets a valid live-session model-selection bug, but the resolver-only patch is not safe to merge because the active runner still passes product defaults into the switch check; the branch also lacks the required changelog entry for a user-visible agents fix.

## What This Changes

The PR modifies `src/agents/live-model-switch.ts` so live-session model selection prefers caller-supplied default provider/model values over resolved agent defaults when they differ.

## Best Possible Solution

Thread the actual selected run provider/model into `shouldSwitchToLiveModel`, then make `resolveLiveSessionModelSelection` honor those explicit defaults without guessing from a mismatch against the agent primary; add focused live-switch regression coverage and the required `CHANGELOG.md` fix entry.

## Reproduction Assessment

Yes. A focused unit path can set an agent primary different from `openai/gpt-5.5`, mark `liveModelSwitchPending`, and call the switch check through the runner contract; with this patch shape, the product default can be resolved as the pending target instead of the caller-selected run model.

## Solution Assessment

No. The proposed resolver heuristic is not the best fix because the active caller still supplies ambient product defaults; the safer solution is to pass the real run selection across the existing switch-check boundary and test that contract.
