## Summary

Keep open. The re-review shows the latest head fixes part of the live-default threading, including the cron path, but two fallback-capable embedded runner entrypoints still omit the original caller-selected provider/model, so the patch can still clear or compare a pending live switch against a fallback candidate.

## What This Changes

The PR adds stable live-session default provider/model parameters to the embedded runner, wires them through selected auto-reply and cron paths, adds focused tests, and updates the changelog for explicit model handling during live session switch checks.

## Best Possible Solution

Finish the stable live-default contract for every fallback-capable embedded runner callsite by carrying the original selected provider/model separately from each fallback attempt, then lock the queued follow-up and direct agent command paths with focused regression tests.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a queued follow-up or direct `openclaw agent --model` run where the original selected model differs from a fallback candidate while `liveModelSwitchPending` is set; the latest head still enters `runEmbeddedPiAgent` from those two paths without stable live defaults.

## Solution Assessment

No. The overall direction is the narrow maintainable fix, but the current PR is incomplete until the queued follow-up runner and direct agent command attempt path pass the original selected provider/model and have coverage.
