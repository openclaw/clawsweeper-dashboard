## Summary

Keep open. The PR moves the live-session switch check in the right direction, but the latest diff still leaves two fallback-capable embedded runner entrypoints without the original caller-selected provider/model, so explicit model choices are not consistently preserved during fallback.

## What This Changes

The PR adds live-model default parameters to embedded agent runs, uses them in live-session switch resolution, threads them through cron and auto-reply runner execution, adds regression tests, and updates the changelog.

## Best Possible Solution

Complete the stable live-default contract across every embedded fallback runner entrypoint: the helper should use caller defaults, and cron, heartbeat/auto-reply, queued follow-ups, and direct agent commands should all pass the original selected provider/model with focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a queued follow-up or direct openclaw agent --model run where fallback selects a different provider/model while liveModelSwitchPending is set; those remaining callsites still pass only the active fallback candidate to runEmbeddedPiAgent.

## Solution Assessment

No. The new runner fields are a narrow maintainable seam, but the proposed patch is incomplete until all fallback-capable callsites pass the original caller-selected provider/model separately from the active fallback attempt.
