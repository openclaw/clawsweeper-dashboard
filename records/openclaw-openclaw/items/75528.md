## Summary

Keep open. The PR moves the live-session default model handling in the right direction and now covers the direct reply, memory-helper, and cron paths, but it still misses the queued follow-up runner path that was called out in the earlier ClawSweeper review.

## What This Changes

The PR threads caller-selected live model defaults into embedded agent live-switch checks for several fallback paths, adds focused regression tests, and adds a changelog fix entry.

## Best Possible Solution

Complete the live-default contract for every fallback-capable embedded runner entrypoint by passing the original caller-selected `run.provider` and `run.model` through `createFollowupRunner`, then lock it with a focused queued follow-up regression test.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a queued follow-up whose selected run model differs from a fallback candidate while `liveModelSwitchPending` is true; the current PR still lets that path enter `runEmbeddedPiAgent` without stable live defaults.

## Solution Assessment

No. The proposed direction is maintainable, but the implementation is incomplete because it updates direct reply and cron paths while leaving a production queued follow-up fallback caller on the new active-attempt default.
