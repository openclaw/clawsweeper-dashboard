The PR adds liveModelDefaultProvider/liveModelDefaultModel plumbing for live-session model switch checks, updates cron and auto-reply execution paths, adds focused tests, and records a changelog fix.

## Best Possible Solution

Yes. A high-confidence code-level reproduction is a queued follow-up or direct `openclaw agent --model` run where fallback tries a different provider/model while `liveModelSwitchPending` is set; those callsites still pass only the active candidate.

No. The new runner fields are a maintainable direction, but the patch is incomplete until every fallback-capable callsite passes the original caller-selected provider/model separately from the active fallback candidate.
