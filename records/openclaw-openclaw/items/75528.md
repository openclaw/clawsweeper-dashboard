## Summary

Keep open. The PR targets a real live-session model bug, but the latest diff still reuses the current fallback candidate as the switch-check default, so a reset-to-default during fallback can clear `liveModelSwitchPending` instead of restarting on the agent default.

## What This Changes

The PR changes live-session model selection to prefer caller-supplied run models, passes the embedded runner provider/model into the switch check, adds focused tests, and adds a changelog fix entry.

## Best Possible Solution

Keep the explicit cron/heartbeat model support, but pass a stable original run default or explicit caller-model signal into the live switch resolver separately from the current fallback attempt's provider/model.

## Reproduction Assessment

Yes. A deterministic reproduction is an active fallback attempt where `sessions.patch` with `model: null` or the session status tool's default reset removes overrides and sets `liveModelSwitchPending`; with this PR, the switch check can resolve the current fallback candidate as the default and clear the pending switch.

## Solution Assessment

No. The PR is close, but the maintainable fix is to distinguish original caller-selected defaults from transient fallback candidates instead of treating any provider/model that differs from the agent primary as caller intent.
