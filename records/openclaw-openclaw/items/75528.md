## Summary

Keep open: the PR fixes part of the live-session model-default path, but two fallback-capable embedded runner entrypoints still omit the stable caller provider/model, so fallback attempts can still compare a pending live reset against the fallback candidate instead of the caller-selected model.

## What This Changes

The PR adds live-model default parameters to embedded agent runs, forwards them through cron and the main auto-reply runner, updates focused live-switch tests, and adds a changelog fix.

## Best Possible Solution

Complete the live-default plumbing across all fallback-capable embedded runner entrypoints and keep focused regression coverage for cron, main auto-reply, queued follow-ups, direct agent commands, and the core live-switch helper.

## Reproduction Assessment

Yes, at source level. Queued follow-ups and direct agent commands both run through runWithModelFallback but still pass only the active fallback provider/model into runEmbeddedPiAgent, so the PR's omitted live defaults default to the fallback candidate in those paths.

## Solution Assessment

No. The new live-default fields are the right narrow mechanism, but this PR is incomplete until the original caller provider/model is threaded through the remaining fallback callsites and covered by tests.
