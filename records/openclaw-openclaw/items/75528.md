## Summary

Keep open. The PR is moving in the right direction, but the live-default contract is still incomplete for fallback-capable embedded calls, so pending live model resets can still be compared against a fallback candidate instead of the caller-selected model.

## What This Changes

The PR adds embedded-runner live default provider/model parameters, wires them through the main reply and cron fallback paths, and adds focused tests plus a changelog entry for live session model switching.

## Best Possible Solution

Complete the stable live-default contract for every fallback-capable embedded runner entrypoint by carrying the original selected provider/model separately from each fallback attempt, then lock the queued follow-up and direct agent command paths with focused regression tests.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a queued follow-up or direct agent command whose original selected model differs from a fallback candidate while liveModelSwitchPending is true; the remaining unthreaded callers enter runEmbeddedPiAgent without stable live defaults.

## Solution Assessment

No. The proposed approach is the narrow maintainable fix, but the current patch is incomplete until the queued follow-up runner and direct agent command attempt path pass the original selected provider/model and have coverage.
