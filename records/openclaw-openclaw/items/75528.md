## Summary

Keep open. The latest PR fixes the earlier missing changelog/test coverage and starts threading the run model into the live-switch check, but it now conflates the current fallback candidate with the session's default selection, which can drop a user-requested reset-to-default live switch during fallback attempts.

## What This Changes

The PR updates live-session model resolution to prefer caller-supplied defaults, passes the embedded runner's provider/model into the switch check, adds focused tests, and adds a changelog fix entry.

## Best Possible Solution

Keep the explicit cron/heartbeat model fix, but separate the original caller-selected default from the current fallback candidate so resets to the agent default still interrupt active fallback attempts correctly.

## Reproduction Assessment

Yes. A focused reproduction is an active fallback attempt where `sessions.patch` or `session_status` resets the model to default, clearing overrides and setting `liveModelSwitchPending`; with this PR the switch resolver can resolve back to the fallback candidate and clear the flag instead of restarting on the agent default.

## Solution Assessment

No. The PR is close, but the safest maintainable solution is to thread a distinct original run default or explicit caller-selection signal into the switch check rather than reusing the current attempt's provider/model as both default and current state.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep reset-to-default switches on the agent default:** `src/agents/pi-embedded-runner/run.ts:1228-1229`
  - body: These values are the current model-fallback candidate, not necessarily the session default. When a live session is reset to default, override fields are removed and `liveModelSwitchPending` is set; during a fallback attempt this makes the resolver select the fallback candidate, so the pending switch is cleared instead of restarting on the agent default. Thread the original selected run default, or a separate explicit-model signal, instead of reusing the current candidate here.
