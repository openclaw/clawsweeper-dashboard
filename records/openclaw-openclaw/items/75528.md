## Summary

Keep this PR open: the live-default threading is the right direction, but the latest patch still leaves fallback-capable queued follow-up and direct agent command paths without the stable caller-selected model, so the bug can still reproduce there.

## What This Changes

The PR adds stable live-model default parameters to embedded runner live-session switch checks, wires them through cron and an auto-reply helper path, adds focused tests, and records a changelog fix.

## Best Possible Solution

Complete stable live-default propagation across every fallback-capable embedded runner entrypoint, with focused coverage for cron, heartbeat/auto-reply execution, queued follow-ups, direct agent commands, and the core live switch helper.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a queued follow-up or direct openclaw agent --model run where model fallback tries a different provider/model while liveModelSwitchPending is set; the two remaining callsites still pass only the active fallback candidate.

## Solution Assessment

No. The new live-default parameters are a maintainable fix direction, but this patch is incomplete until all fallback-capable runEmbeddedPiAgent callsites pass the original caller-selected provider/model separately from the active fallback attempt.
