## Summary

Keep this PR open. The reported gateway display-name bug is still plausible on current `main`, but the branch patches an older `listSessionsFromStore` path while current code projects both list results and live change snapshots through `buildGatewaySessionRow`.

## What This Changes

The PR changes `src/gateway/session-utils.ts` so `listSessionsFromStore` treats configured per-agent and global main session keys as stable main sessions, preserving explicit `entry.displayName` and otherwise returning `Main Session` instead of transient origin labels.

## Best Possible Solution

Move the stabilization into `buildGatewaySessionRow` or a shared helper used by both `listSessionsFromStore` and `loadGatewaySessionRow`, preserve explicit `entry.displayName`, recognize both `resolveAgentMainSessionKey` and `resolveMainSessionKey`, and add focused regression tests for default-agent main, configured main keys, global main, and stale heartbeat origin labels.

## Reproduction Assessment

Yes. A deterministic reproduction is to project a main session entry with no explicit label/displayName and `origin.label: "heartbeat"` through `buildGatewaySessionRow`; the row can still carry `displayName: "heartbeat"`, and the UI prefers that row value before its `Main Session` fallback.

## Solution Assessment

No. The intended fix is correct, but the current PR shape patches only the older list projection path; the narrow maintainable fix is a shared gateway row-projection guard plus regression coverage for both list and event snapshot callers.
