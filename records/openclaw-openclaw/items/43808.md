Keep this PR open: current main still has a source-proven cron failure-destination gap for threaded Telegram deliveries, and the patch direction looks narrow and correct, but it still needs live Telegram/equivalent topic proof and normal mergeability handling before landing.

## What This Changes

The PR changes cron failure-destination dedupe and failure notification target resolution so same-chat Telegram failure alerts for threaded primary deliveries are preserved and do not inherit the primary session thread.

## Best Possible Solution

Land the narrow cron fix after live Telegram/equivalent proof and rebase/CI cleanup, keeping failure destinations threadless unless maintainers separately design explicit failure-topic support.
