## Summary

Keep open. Current main still has the cron failure-destination bug, and this PR is a focused fix with sufficient terminal proof; it needs maintainer rebase/review against the newer cron delivery-target code rather than cleanup closure.

## What This Changes

This PR makes cron failure-destination dedupe thread-aware for Telegram announce delivery, suppresses session-derived thread carry for failure notifications, and adds focused cron/target-resolution tests plus a changelog entry.

## Best Possible Solution

Land a rebased narrow fix that makes failure-destination duplicate checks thread-aware and suppresses inherited session thread IDs only for failure notifications while preserving current main delivery-context behavior.

## Reproduction Assessment

Yes, at source level. Current main carries `delivery.threadId` into primary cron delivery, but failure-destination dedupe ignores it and failure notification target resolution can still reuse session thread context.

## Solution Assessment

Yes. The PR fixes the bug at the cron delivery-planning and failure-notification target-resolution seams without expanding cron schema, Telegram transport, or plugin contracts; the safe landing path is a rebase that preserves newer main behavior.
