## Summary

Keep open: current main improves the supported `openclaw update` path and related missing-plugin channel-list diagnostics, but the reported direct package-manager core upgrade with an older installed official plugin can still bypass plugin convergence and lacks a clear core/plugin incompatibility warning.

## What This Changes

The issue asks OpenClaw to realign or clearly diagnose stale installed plugin packages after a core upgrade so channels do not silently appear disabled or broken.

## Best Possible Solution

Add a narrow, explicit diagnostic for active official external plugins whose installed package is stale or incompatible with the running core, and keep package-manager mutation in `openclaw update`, plugin update, or doctor repair flows rather than Gateway startup.

## Reproduction Assessment

No high-confidence live reproduction was established. The source path is credible because direct package-manager core upgrades bypass `openclaw update` plugin convergence, while current compatibility checks only enforce a plugin's minimum host version.

## Solution Assessment

Unclear. Auto-upgrading plugins during startup conflicts with the documented verify-only startup model, so the maintainable direction is a policy-approved warning/refusal plus explicit update or doctor repair guidance.
