- `impact:message-loss`: The reported failure mode leaves Discord disabled or non-functional, so channel messages may not be received or delivered.

## Summary

Keep open: current main has a strong `openclaw update` plugin-sync/convergence path, but the reported direct package-manager upgrade path can still bypass that path and the inspected startup/registry diagnostics do not clearly warn on an older active official plugin package before Discord degrades.

## What This Changes

The issue reports plugin/core version drift after a core upgrade causing Discord to appear disabled or non-functional without an actionable compatibility warning.

## Best Possible Solution

Add a narrow diagnostic for active official external plugins whose installed package is stale or incompatible with the host, and make the repair path explicit without running package managers during Gateway startup.

## Reproduction Assessment

No high-confidence live reproduction was established. The source path is credible because direct package-manager core upgrades bypass `openclaw update` plugin sync/convergence, while startup is documented as verify-only and the current registry check only enforces a plugin's minimum host version.

## Solution Assessment

Unclear until maintainers choose the policy for manual global upgrades and exact official plugin pins. Auto-upgrading during startup would conflict with the verify-only startup model, so the safer direction is explicit update/doctor convergence plus clear warnings or refusal for active incompatible official plugin drift.
