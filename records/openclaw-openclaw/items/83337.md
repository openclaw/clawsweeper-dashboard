## Label Justifications

- `P1`: The report describes a post-upgrade Discord channel becoming disabled or non-functional with no actionable warning, which can break a real channel workflow.
- `impact:message-loss`: The affected failure mode can leave Discord messages undelivered because the channel is present but disabled or broken after upgrade.

## Summary

Keep open: this is still a credible upgrade-diagnostics gap. Current main has guarded plugin convergence for `openclaw update`, but the reported manual global package upgrade path can bypass it, and the runtime compatibility gate still does not clearly warn when an older active official plugin package lags behind the upgraded core.

## What This Changes

The issue asks OpenClaw to keep official plugin packages aligned with core upgrades or emit an actionable incompatibility warning when plugin/core version drift breaks a channel.

## Best Possible Solution

Decide the policy for exact official plugin pins and manual global upgrades, then add a narrow diagnostic or convergence check for active official external plugins without running package managers during gateway startup.

## Reproduction Assessment

No high-confidence live reproduction was established. The source path is credible because direct global package upgrades bypass `openclaw update` plugin convergence, while current compatibility checks only enforce a plugin's minimum host version.

## Solution Assessment

Unclear. Auto-upgrading every plugin during startup would conflict with the documented verify-only startup model, so maintainers should first choose the manual-upgrade and exact-pin policy, then add explicit warnings, update convergence, or refusal for active incompatible official plugin drift.
