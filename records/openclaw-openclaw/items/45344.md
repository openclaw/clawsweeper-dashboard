## What This Changes

The PR adds configured-agent allowlist validation for explicit OpenAI-compatible gateway agent selectors and returns 400 `invalid_request_error` responses for unknown IDs.

## Best Possible Solution

Rebase the validation onto current gateway code, reject only unknown explicit agent selectors, preserve configured default routing, `openclaw/default`, header precedence, and `x-openclaw-model`, then add focused regression tests plus redacted real gateway proof.

## Reproduction Assessment

Yes, source-reproducible. Current main returns explicit header/model agent IDs without a configured-agent allowlist check, and both OpenAI-compatible HTTP handlers use those resolved IDs before dispatch.

## Solution Assessment

No. The validation direction is right, but this branch is not the best merge shape until it is rebased, preserves current default-agent and model-override contracts, clarifies the allowlist contract, and includes after-fix real gateway proof.

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve configured default-agent routing:** `src/gateway/http-utils.ts:105`
  - body: The branch falls back to literal `main` when no explicit selector is parsed, but current main routes unselected `openclaw` requests through `resolveDefaultAgentId(cfg)`. In configs where the configured default agent is not `main`, this would send Chat Completions and Responses turns to the wrong agent.
  - confidence: 0.9
- **[P2] Keep openclaw/default mapped to the default agent:** `src/gateway/http-utils.ts:66-68`
  - body: This parser treats `openclaw/default` as an explicit agent ID named `default` and validates it against configured IDs. Current docs and tests expose `openclaw/default` as the stable alias for the configured default agent, so existing clients would get a 400 unless an agent named `default` exists.
  - confidence: 0.89
- **[P3] Clarify omitted allowlist behavior:** `src/gateway/http-utils.ts:84`
  - body: `knownAgentIds` is optional on the resolver parameters, but `!knownAgentIds?.includes(...)` throws for every explicit selector when it is omitted. Make the allowlist required on validating paths or treat `undefined` as no validation so direct callers do not get surprising rejections.
  - confidence: 0.84
