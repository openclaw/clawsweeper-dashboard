  - confidence: 0.9

  - body: This keeps `useMessageChannelHeader` disabled for `/v1/responses`, but current main, docs, and tests support `x-openclaw-message-channel` there. Preserving this line during rebase would drop channel-aware prompt and policy context for Responses clients.

  - confidence: 0.84
