## Summary

Keep open. Current main still lacks unknown-agent validation on the OpenAI-compatible HTTP paths, but this PR branch is conflicting, lacks real after-fix gateway proof, and still introduces two compatibility regressions that should be fixed before merge.

## What This Changes

The branch adds `InvalidGatewayAgentIdError`, validates explicit agent IDs in Chat Completions and Responses request handling, maps unknown IDs to 400 responses, and adds focused gateway tests.

## Best Possible Solution

Validate explicit gateway agent selections against the current configured plus effective-default agent set in the shared resolver, preserve default aliases and header/model precedence, then land with focused regression tests and real gateway proof.

## Reproduction Assessment

Yes, from source inspection. Current main normalizes explicit header and model agent selections without checking configured agents, so an unknown ID reaches downstream gateway dispatch instead of returning 400; I did not run a live gateway because this review is read-only.

## Solution Assessment

No. The proposed direction is right, but the branch should preserve explicit default-agent routing, avoid changing omitted-allowlist behavior by accident, and be rebased onto current default-agent routing before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Preserve explicit default-agent routing:** `src/gateway/http-utils.ts:84`
  - body: The new allowlist rejects `x-openclaw-agent-id: main` / `openclaw:main` whenever `knownAgentIds` lacks `main`, while this branch still lets an unselected request fall through to `main` and the HTTP docs show explicit `main`. Include the effective default alias in the allowed set or handle it before rejecting.
  - confidence: 0.87
- **[P3] Preserve callers that omit knownAgentIds:** `src/gateway/http-utils.ts:84`
  - body: `knownAgentIds` is optional, but `!knownAgentIds?.includes(...)` is true when the option is omitted, so direct callers that do not opt into validation would reject every explicit header/model agent selection. Either require the list on validating paths or guard the check with `knownAgentIds !== undefined`.
  - confidence: 0.82
