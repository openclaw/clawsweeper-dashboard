## Summary

Keep this PR open. Current main still lacks configured-agent validation on the OpenAI-compatible chat and Responses HTTP paths, so the central security/behavior fix is still useful, but the submitted patch is not merge-ready because it needs a current-main rebase and fixes for the helper contract and explicit default-agent compatibility.

## What This Changes

The PR adds configured-agent allowlist validation for explicit gateway agent selectors, maps unknown selections to 400 errors on chat and Responses HTTP handlers, and adds regression tests for unknown header/model selectors.

## Best Possible Solution

Land a rebased version that validates only explicit agent selectors against the current runtime config, returns `400 invalid_request_error` for unknown explicit selectors on `/v1/chat/completions` and `/v1/responses`, preserves implicit default routing plus explicit `main` and `openclaw/default` compatibility, and adds focused regression tests for known, unknown, and default selectors.

## Reproduction Assessment

Yes. The PR body gives concrete HTTP requests, and current-main code inspection confirms those explicit selectors are normalized and used without configured-agent validation before dispatch; I did not run live tests because this review was read-only.

## Solution Assessment

No, not as submitted. The direction is the right narrow fix, but the patch should be revised to use the current runtime config boundary and to avoid breaking omitted-validation call sites or explicit default-agent selectors.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Skip validation when no known-agent list is supplied:** `src/gateway/http-utils.ts:84`
  - body: `knownAgentIds` is still optional, but `!knownAgentIds?.includes(...)` throws whenever a caller omits it. Existing compat surfaces such as embeddings call `resolveAgentIdForRequest` without a list for valid agent-scoped requests, so rebasing this helper would turn those requests into uncaught `InvalidGatewayAgentIdError`s. Guard only when the list is present, or make the parameter required and update every caller.
  - confidence: 0.88
- **[P2] Preserve explicit default-agent selectors:** `src/gateway/http-utils.ts:84`
  - body: The allowlist only accepts IDs returned by `listAgentIds(loadConfig())`. In multi-agent configs without a literal `main` entry, explicit `x-openclaw-agent-id: main` or `model: openclaw:main` returns 400 even though the same path still routes implicitly to `main`; current main also treats `openclaw/default` as the configured default. Include the resolved default aliases before rejecting.
  - confidence: 0.84
