## Summary

Keep open: the PR targets a real gateway bug, but it needs branch updates, contributor real-behavior proof, and at least one correctness fix before merge.

## What This Changes

The PR adds allowlist validation for explicit agent IDs on OpenAI Chat Completions and OpenResponses HTTP requests and returns 400 for unknown selections.

## Best Possible Solution

Validate explicit gateway agent selections against the current configured/default agent set in the shared request-context resolver, preserve documented default-agent aliases, and cover both HTTP handlers with focused regression tests.

## Reproduction Assessment

Yes. Source inspection on current main shows explicit header and model agent selections are normalized and used without an existence check; I did not run a live gateway because this review is read-only.

## Solution Assessment

No. The proposed direction is right, but this branch needs to be rebased onto current gateway routing and fixed so explicit default-agent selection and omitted allowlist behavior remain compatible.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Allow explicit main to match implicit fallback:** `src/gateway/http-utils.ts:82-84`
  - body: The new allowlist rejects any explicit selection not returned by `knownAgentIds`, but this patch still lets an implicit request fall back to `main` when the list is `["alpha", "beta"]`. That means `x-openclaw-agent-id: main` or `model: "openclaw:main"` can fail with 400 even though the same target is accepted implicitly and is still shown in the HTTP docs; include the default/main alias in the allowed set or handle it before rejecting.
  - confidence: 0.87
- **[P3] Preserve behavior when knownAgentIds is omitted:** `src/gateway/http-utils.ts:82-84`
  - body: `knownAgentIds` is optional, but `!knownAgentIds?.includes(...)` throws when it is omitted, so existing direct callers that do not opt into validation would reject every explicit agent selection. Either make the parameter required for validating callers or guard the check with `knownAgentIds !== undefined`.
  - confidence: 0.82
