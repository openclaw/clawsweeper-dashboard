## Summary

Keep open. The gateway bug is still source-reproducible and this PR targets the right boundary, but the branch is not merge-ready because the validation helper has blocking edge cases, the branch conflicts with newer agent-first gateway code, and the external PR only provides test proof.

## What This Changes

The PR adds explicit-agent validation to the OpenAI Chat Completions and OpenResponses HTTP handlers so unknown `x-openclaw-agent*` headers or `openclaw:`/`agent:` model selectors return 400, with gateway tests.

## Best Possible Solution

Rebase a narrow gateway fix over current main that validates explicit agent selections against configured/default agents, maps unknown selections to 400 on Chat Completions and Responses, preserves the current model-override contract, and includes focused regression tests plus real behavior proof.

## Reproduction Assessment

Yes, from source inspection rather than a live command. Current main returns normalized explicit agent ids from headers or `openclaw:` / `agent:` model selectors before any configured-agent validation, so an unknown `ghost` selector can reach dispatch instead of a 400.

## Solution Assessment

No, not as currently proposed. The intent is right, but the branch needs a current-main rebase and config-aware validation fixes before it is the narrowest maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Keep optional agent validation optional:** `src/gateway/http-utils.ts:84-85`
  - body: `knownAgentIds` is still typed as optional, but `!knownAgentIds?.includes(...)` throws for every explicit selector when a caller omits it. Current and future callers that use `resolveAgentIdForRequest` without an allowlist would turn valid explicit agent selections into unexpected errors; either make the parameter required everywhere or gate the check on `knownAgentIds !== undefined`.
  - confidence: 0.86
- **[P2] Validate the default agent consistently:** `src/gateway/http-utils.ts:84-85`
  - body: This allowlist is built from configured agent ids, while the same helper still falls back to `main` when there is no explicit selector. In a config whose list omits `main`, an implicit request routes to `agent:main` but `x-openclaw-agent-id: main` or `openclaw:main` returns 400, breaking clients that pin the default selector; include the resolved default agent or move to the current config-aware default-agent contract.
  - confidence: 0.82
