## Summary

Keep open. The underlying gateway bug is still real on current main, but this PR head is not merge-ready because it regresses configured default-agent routing and the documented `openclaw/default` alias, and it still lacks after-fix real gateway proof from the contributor.

## What This Changes

The PR adds configured-agent allowlist validation for explicit OpenAI-compatible gateway agent selectors and maps unknown IDs to 400 responses in Chat Completions and Responses, with focused tests.

## Best Possible Solution

Rebase the validation onto current gateway code so only explicit unknown agent selectors are rejected while preserving configured default routing, `openclaw/default`, header precedence, and `x-openclaw-model`, then add focused regressions and redacted live gateway proof.

## Reproduction Assessment

Yes, source-level. Current main returns explicit header/model agent IDs without checking configured agents, and both OpenAI-compatible HTTP handlers use that result before dispatch; I did not run a live gateway in this read-only review.

## Solution Assessment

No. The validation direction is right, but this branch is not the best merge shape until it is rebased, preserves the current default-agent and model-alias contracts, and includes after-fix real behavior proof.

- **[P2] Preserve configured default-agent routing:** `src/gateway/http-utils.ts:105`
  - body: This fallback hard-codes `main`, but current main uses `resolveDefaultAgentId(cfg)` for unselected `openclaw` or missing selectors. In configs whose default agent is not `main`, Chat Completions and Responses would route to the wrong agent.
  - confidence: 0.9
- **[P2] Keep `openclaw/default` routed to the default agent:** `src/gateway/http-utils.ts:66-68`
  - body: The parser now treats `openclaw/default` as an explicit agent id named `default` and validates it against configured ids. Current docs and `/v1/models` expose `openclaw/default` as the stable alias for the configured default agent, so existing clients would get a 400 unless an agent named `default` exists.
  - confidence: 0.89
- **[P3] Clarify omitted allowlist behavior:** `src/gateway/http-utils.ts:84`
  - body: `knownAgentIds` is optional on the resolver params, but this condition rejects every explicit selector when it is omitted. Make the allowlist required on validating paths or treat `undefined` as no validation so direct callers have a coherent contract.
  - confidence: 0.84
