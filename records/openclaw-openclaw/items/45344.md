Full review comments:

- **[P2] Preserve configured default-agent routing:** `src/gateway/http-utils.ts:105`
  - body: This fallback hard-codes `main`, but current main routes unselected `openclaw` or missing selectors through the configured default agent. In configs whose default agent is not `main`, this would send requests to the wrong agent.
  - confidence: 0.91
- **[P2] Keep openclaw/default mapped to the default agent:** `src/gateway/http-utils.ts:66-68`
  - body: The parser now treats `openclaw/default` as an explicit agent id named `default` and validates it against configured ids. Current docs and tests expose `openclaw/default` as the stable alias for the configured default agent, so existing clients would get a 400 unless an agent named `default` exists.
  - confidence: 0.9
- **[P2] Keep Responses message-channel header support:** `src/gateway/openresponses-http.ts:440`
  - body: This keeps `useMessageChannelHeader` disabled for `/v1/responses`, but current main, docs, and tests support `x-openclaw-message-channel` there. Preserving this line during rebase would drop channel-aware prompt and policy context for Responses clients.
  - confidence: 0.86
- **[P3] Make the allowlist contract explicit:** `src/gateway/http-utils.ts:84`
  - body: `knownAgentIds` is optional on the resolver params, but this condition rejects every explicit selector when it is omitted. Make the allowlist required on validating paths or treat `undefined` as no validation so direct callers have a coherent contract.
  - confidence: 0.84
