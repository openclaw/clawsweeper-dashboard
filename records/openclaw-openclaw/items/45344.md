## Summary

Keep open: current main still accepts unknown explicit gateway agent IDs, and this PR points at the right bug but would regress current default-agent and default-alias behavior before it can merge.

## What This Changes

The PR adds validation for explicit OpenAI-compatible gateway agent selectors, maps unknown IDs to 400 errors, and adds focused Chat Completions and Responses tests.

## Best Possible Solution

Validate explicit gateway agent selections against configured agents in the shared resolver, preserve `openclaw` and `openclaw/default` as configured-default aliases, and return 400 from both HTTP endpoints with tests and real gateway proof.

## Reproduction Assessment

Yes. Source inspection of current main shows explicit header/model agent IDs are returned before any configured-agent validation, and both HTTP endpoints use that unchecked path before dispatch.

## Solution Assessment

No. The direction is right, but the branch should be rebased to preserve current configured-default routing and `openclaw/default`, then backed by after-fix real gateway proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve configured default-agent routing:** `src/gateway/http-utils.ts:105`
  - body: Current main routes unselected `openclaw` requests through `resolveDefaultAgentId(cfg)`, but this branch falls back to literal `main`. In configs where the default agent is not `main`, no-selector Chat Completions and Responses requests would route to the wrong agent.
  - confidence: 0.88
- **[P2] Preserve openclaw/default as the default alias:** `src/gateway/http-utils.ts:84`
  - body: Current docs and tests treat `model: "openclaw/default"` as the stable alias for the configured default agent. This branch parses it as an explicit agent id named `default` and rejects it unless such an agent exists, breaking existing clients that use the alias.
  - confidence: 0.86
- **[P3] Clarify omitted allowlist behavior:** `src/gateway/http-utils.ts:84`
  - body: `knownAgentIds` is optional on the resolver parameters, but the new check throws for every explicit header/model selection when it is omitted. Make the option required on validating paths or guard `undefined` as no validation so future direct callers do not get surprising rejections.
  - confidence: 0.82
