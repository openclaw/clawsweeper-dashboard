## Summary

Keep open: current main still has the unknown-agent validation gap, but this PR is stale against the newer gateway default-agent contract and would regress documented routing before it can merge.

## What This Changes

The PR adds configured-agent validation for explicit OpenAI-compatible gateway agent selectors in Chat Completions and Responses, maps unknown IDs to 400 errors, and adds focused gateway tests.

## Best Possible Solution

Validate explicit gateway agent selections against configured IDs after preserving configured default aliases, return 400 from both HTTP endpoints, and prove it with focused tests plus real gateway output.

## Reproduction Assessment

Yes, from source inspection. Current main returns explicit header/model agent IDs without checking configured agents, and both HTTP endpoints use that unchecked path before dispatch.

## Solution Assessment

No. The direction is right, but this branch needs to be rebased and adjusted to preserve `resolveDefaultAgentId`/`openclaw/default` behavior and clarify the `knownAgentIds` contract before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Preserve configured default-agent routing:** `src/gateway/http-utils.ts:105`
  - body: Current main routes unselected `openclaw` requests through `resolveDefaultAgentId(cfg)`, but this branch falls back to literal `main`. In configs where the default agent is not `main`, no-selector Chat Completions and Responses requests would route to the wrong agent.
  - confidence: 0.89
- **[P2] Preserve openclaw/default as the default alias:** `src/gateway/http-utils.ts:84`
  - body: Current docs and tests treat `model: "openclaw/default"` as the stable alias for the configured default agent. This branch parses it as an explicit agent id named `default` and rejects it unless such an agent exists, breaking existing clients that use the alias.
  - confidence: 0.87
- **[P3] Clarify omitted allowlist behavior:** `src/gateway/http-utils.ts:84`
  - body: `knownAgentIds` is optional on the resolver parameters, but the new check throws for every explicit header/model selection when it is omitted. Make the option required on validating paths or guard `undefined` as no validation so future direct callers do not get surprising rejections.
  - confidence: 0.83
