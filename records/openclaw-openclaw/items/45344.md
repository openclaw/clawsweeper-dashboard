## Summary

Keep open. The underlying gateway bug is still present on current main and this PR is useful, but the branch is stale/conflicting, regresses the configured default-agent contract, breaks the `openclaw/default` alias, and still lacks after-fix real gateway proof.

## What This Changes

The branch adds configured-agent validation and 400 `invalid_request_error` handling for explicit OpenAI Chat Completions and Responses agent selectors, with focused gateway tests.

## Best Possible Solution

Validate explicit gateway agent selectors against configured agents while preserving `resolveDefaultAgentId` for `openclaw` and `openclaw/default`, integrate with current model-override logic, and prove the result with focused tests plus real gateway output.

## Reproduction Assessment

Yes. Source inspection of current main shows explicit header/model agent IDs are returned without a configured-agent allowlist check, and both HTTP endpoints use that resolver before dispatch.

## Solution Assessment

No. The direction is right, but this branch is not the best merge shape until it preserves configured-default routing, preserves `openclaw/default`, clarifies the allowlist contract, and adds real after-fix proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Preserve configured default-agent routing:** `src/gateway/http-utils.ts:105`
  - body: Current main routes unselected `openclaw` requests through `resolveDefaultAgentId(cfg)`, but this branch falls back to literal `main`. In configs where the default agent is not `main`, no-selector Chat Completions and Responses requests would route to the wrong agent.
  - confidence: 0.89
- **[P2] Preserve openclaw/default as the default alias:** `src/gateway/http-utils.ts:84`
  - body: Current docs and tests treat `model: "openclaw/default"` as the stable alias for the configured default agent. This branch parses it as an explicit agent id named `default` and rejects it unless such an agent exists, breaking existing clients that use the alias.
  - confidence: 0.87
- **[P3] Clarify omitted allowlist behavior:** `src/gateway/http-utils.ts:84`
  - body: `knownAgentIds` is optional on the resolver parameters, but the new check throws for every explicit header/model selection when it is omitted. Make the option required on validating paths or guard `undefined` as no validation so future direct callers do not get surprising rejections.
  - confidence: 0.83
