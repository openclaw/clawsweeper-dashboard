Adds configured-agent allowlist validation and 400 invalid_request_error responses for explicit OpenAI-compatible gateway agent selectors, with focused tests for Chat Completions and Responses.

## Best Possible Solution

## Reproduction Assessment

Yes, source-level. Current main and v2026.5.12 both return explicit header/model agent IDs without checking configured agents, and both OpenAI-compatible HTTP handlers use that result before dispatch.

## Solution Assessment

No. The validation direction is right, but this branch is not the best merge shape until it is rebased onto current main, preserves the current default-agent/model-alias contracts, and includes after-fix real behavior proof.
