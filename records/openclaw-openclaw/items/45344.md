## Summary

Keep open. The gateway bug is still source-reproducible and this PR targets the right surface, but the branch is conflicting, lacks required after-fix real gateway proof, and regresses current default-agent and `openclaw/default` contracts.

## What This Changes

The PR adds configured-agent allowlist validation for explicit OpenAI-compatible gateway agent selectors and maps unknown IDs to 400 `invalid_request_error` responses in Chat Completions and Responses.

## Best Possible Solution

Rebase onto current gateway code and validate only explicit unknown agent selectors while preserving configured default routing, `openclaw/default`, header precedence, and `x-openclaw-model`, with focused regression tests and redacted live gateway proof.

## Reproduction Assessment

Yes. Current main resolves explicit header/model agent IDs and both OpenAI-compatible HTTP handlers use the result before dispatch without checking that the agent is configured; I did not run a live gateway because this was a read-only review.

## Solution Assessment

No. The validation direction is right, but this branch is not the best merge shape until it is rebased, preserves current default-agent and model-override contracts, and includes after-fix real gateway proof.

  - confidence: 0.84
