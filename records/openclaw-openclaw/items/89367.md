## Summary

Keep this PR open: it is the strongest focused wait-layer candidate I found for the pending timeout snapshot race, with current main and the latest release still showing the asymmetric fallback. The patch is narrow, has hard/soft timeout regression coverage, and does not try to close the linked broader delivery issue by itself.

## What This Changes

The PR changes `waitForAgentJob` so its outer fallback timer forwards a pending hard-timeout snapshot after pending-error precedence, and adds regression tests for hard provider timeout and soft queue-timeout recovery.

## Best Possible Solution

Land this narrow wait-layer fix if maintainers accept the bounded proof, and keep the no-output delivery concern tracked separately under https://github.com/openclaw/openclaw/issues/89095.

## Reproduction Assessment

Yes. Source inspection shows current main schedules pending timeout snapshots but the outer `waitForAgentJob` timer only forwards pending errors, and the PR adds fake-timer coverage for the exact race; I did not run tests in this read-only review.

## Solution Assessment

Yes for the wait-layer bug. The proposed fix is at the fallback that drops timeout snapshots, preserves pending-error precedence, uses the shared hard-timeout phase contract, and leaves the broader no-output delivery concern outside this PR.
