## Summary

Keep open: this is a focused wait-layer bug-fix PR, but the current gate is narrower than OpenClaw's normalized hard-timeout contract and can still drop provider-started timeout snapshots without a timeoutPhase.

## What This Changes

The PR exports the hard timeout phase predicate, forwards pending hard-timeout snapshots from waitForAgentJob's fallback timer, and adds hard/soft timeout regression tests.

## Best Possible Solution

Align the fallback with the normalized terminal hard-timeout contract, keep soft queue/gateway-draining phases recoverable, and leave zero-output delivery as separate canonical issue work unless this PR adds live proof for it.

## Reproduction Assessment

Yes, source-reproducible: current main stores pending timeout snapshots but the outer waitForAgentJob timer only forwards pending errors, so a hard-timeout snapshot can lose endedAt before downstream parent completion handling sees it. I did not run a live parent/subagent timeout in this read-only review.

## Solution Assessment

No, not as currently written: the wait-layer location is right, but the fallback should reuse the normalized hard-timeout outcome contract instead of only checking timeoutPhase. The broader zero-output delivery issue should stay separate unless this PR explicitly proves and fixes it.
