## Summary

Keep open: the bug is valid and the PR is in the right wait-layer area, but the current patch can still prematurely resolve a recoverable aborted run as a terminal timeout before the retry grace expires.

## What This Changes

The PR changes `waitForAgentJob`'s outer timeout fallback to return pending timeout snapshots with `endedAt` and adds focused fake-timer tests for timeout and transient-abort behavior.

## Best Possible Solution

Forward pending timeout snapshots from the outer fallback only when the snapshot is proven to be a hard timeout by the shared timeout attribution contract, then cover both provider hard-timeout forwarding and `aborted` plus `endedAt` recovery in wait-layer tests.

## Reproduction Assessment

Yes. Current main source shows the outer wait timer drops pending timeout snapshots, and the PR proof plus existing retry-grace tests give a high-confidence source reproduction path without needing a new product mode.

## Solution Assessment

No. The fix is in the right function, but `endedAt` is not the best terminal discriminator because existing retry-grace coverage treats `aborted` events with `endedAt` as recoverable until grace expires.
