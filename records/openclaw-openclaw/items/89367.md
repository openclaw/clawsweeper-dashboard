## Summary

Keep open: the PR is in the right wait-layer function and has useful proof for the reported race, but the latest guard still does not match OpenClaw's shared hard-timeout contract and can mis-handle session terminal state before the retry grace expires.

## What This Changes

The PR changes `waitForAgentJob` so the outer fallback timer can return pending timeout snapshots, and adds fake-timer regression coverage for provider timeout and transient-abort recovery cases.

## Best Possible Solution

Keep the wait-layer fix, but make the fallback use the same hard-timeout predicate as `buildAgentRunTerminalOutcome` and cover both hard and soft timeout attribution in regression tests.

## Reproduction Assessment

Yes. Current source and the linked issue show the outer wait timer can drop pending timeout snapshots; the remaining PR defect is source-reproducible by using a pending soft timeout phase such as `gateway_draining` before the retry grace expires.

## Solution Assessment

No. The fix belongs in `waitForAgentJob`, but the discriminator should reuse the shared hard-timeout contract rather than treating any `timeoutPhase` string as hard.
