## What This Changes

The PR updates `waitForAgentJob` to return a pending hard timeout snapshot when its outer wait timer fires and adds regression tests for hard provider timeout and soft queue-timeout recovery.

## Best Possible Solution

Land the narrow wait-layer fix after required checks if maintainers accept the bounded proof, and keep the related no-output delivery question tracked separately until it is fixed or disproven.

## Reproduction Assessment

Yes: source inspection shows current main schedules pending timeout snapshots but the outer wait timer only forwards pending errors, so a hard timeout can resolve as null before timeout grace publishes it. I did not execute a live parent/sub-agent timeout in this read-only review.

## Solution Assessment

Yes: this is the best narrow fix I found because it preserves pending-error precedence, uses the shared hard-timeout phase predicate, and adds hard/soft timeout regression coverage. The separate no-output delivery path should remain outside this PR unless a follow-up proves and fixes it.
