## What This Changes

The PR changes `waitForAgentJob`'s outer timeout callback to forward pending hard timeout snapshots and adds fake-timer regression tests for provider timeout and soft queue-timeout recovery.

## Best Possible Solution

Land the narrow wait-layer fix after required checks if maintainers accept the remaining live-delivery proof gap; otherwise ask for one live sub-agent timeout smoke before merge.

## Reproduction Assessment

Yes, by source inspection: current `main` schedules pending timeout snapshots but the outer timer only forwards pending errors, so a hard timeout can resolve as null before the grace timer publishes it. I did not execute the repro locally because this review is read-only.

## Solution Assessment

Yes, this is the best narrow fix: it preserves pending-error behavior, uses the existing hard-timeout predicate, and adds coverage for both provider hard timeout and soft queue recovery. The only remaining gap is live parent-notification proof, not a clearer code fix.
