## Summary

Keep open: this is a focused wait-layer bugfix with useful proof and a regression test, but the current patch forwards every pending timeout snapshot before the retry grace window, which can make transient aborted runs look terminal for short `agent.wait` callers.

## What This Changes

The PR changes `waitForAgentJob` to return a pending timeout snapshot from the outer fallback timer and adds a fake-timer regression test for that race.

## Best Possible Solution

Keep the wait-layer repair, but forward a pending timeout snapshot early only when it is a hard run timeout; preserve retry-grace behavior for correctable timeout-shaped lifecycle events and leave the remaining delivery failure tracked by https://github.com/openclaw/openclaw/issues/89095.

## Reproduction Assessment

Yes, by source inspection: current main can create a pending timeout snapshot, then the outer `waitForAgentJob` timer resolves `null` because it checks only pending errors. I did not run tests in this read-only checkout.

## Solution Assessment

No as written: the narrow fix belongs in `waitForAgentJob`, but it should distinguish hard timeout snapshots from retry-grace timeout-shaped events before returning terminal fields early.
