## What This Changes

The issue requests a supported `tools.exec.shell` configuration option for choosing the shell used by exec tool invocations.

## Best Possible Solution

Define one maintained exec shell-selection contract covering precedence, executable-plus-argument semantics, validation timing, Windows/WSL cwd and PATH behavior, approval and safe-bin interactions, schema/help/docs, and targeted Windows plus POSIX regression proof.

## Reproduction Assessment

## Solution Assessment

Unclear. A config-backed shell override is plausible, but the maintainable solution needs a full exec contract rather than a simple string passthrough into one spawn path.
