## What This Changes

The issue requests a configurable `tools.exec.shell` option so exec commands can use a user-selected shell such as WSL bash or `pwsh` instead of the platform default.

## Best Possible Solution

Define one maintainer-approved shell-selection contract for exec, then update config types/schema/help/docs and thread it consistently through gateway, PTY, sandbox, node, approval, PATH, WSL, and safe-bin behavior with focused tests.

## Reproduction Assessment

Yes, source inspection reproduces the missing capability: current main has no `tools.exec.shell` config type, strict schema key, docs entry, or runtime config threading. This is not a failing-runtime bug reproduction because the request is for a new config feature.

## Solution Assessment

Unclear. A config-backed shell override is plausible, but the best design needs maintainer decisions on shell arguments, WSL cwd/PATH semantics, validation timing, and approval/safe-bin policy.
