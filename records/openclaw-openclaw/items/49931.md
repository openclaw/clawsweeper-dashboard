Define a supported exec shell-selection contract covering precedence, executable-plus-argument semantics, validation, Windows/WSL cwd and PATH behavior, approval and safe-bin interactions, gateway/node/sandbox/PTY behavior, schema/help/docs, and targeted Windows plus POSIX proof.

Unclear. A config-backed shell override is plausible, but the maintainable solution needs a full exec contract rather than a simple string passthrough into one spawn path.
