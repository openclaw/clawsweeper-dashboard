## Best Possible Solution

Define a supported core exec shell-selection contract covering precedence, executable-plus-argument semantics, validation, WSL cwd/PATH behavior, approval and safe-bin interactions, gateway/node/sandbox/PTY behavior, schema/help/docs, and targeted Windows plus POSIX proof.

## Solution Assessment

Unclear. A config-backed override is plausible, but the maintainable solution needs a complete exec shell contract rather than a simple string passthrough into one spawn path.
