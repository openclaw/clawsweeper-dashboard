## Summary

Keep open: the underlying outbound queue recovery bug is source-reproducible, but this branch is not mergeable because it adds malformed gateway startup/readiness code, drops other queued replay fields during normalization, and lacks after-fix real behavior proof.

## What This Changes

This PR adds legacy delivery-queue normalization/tests and also adds gateway agents.create readiness/runtime config refresh changes.

## Best Possible Solution

Land a current-main outbound-only queue storage normalizer that preserves every QueuedDelivery replay field with focused storage/recovery regression tests, and split gateway readiness/runtime refresh work into a separate PR.

## Reproduction Assessment

Yes, by source inspection. Current main reads queue JSON as QueuedDelivery and only backfills lastAttemptAt, while recovery replays fields that a legacy-entry normalizer must preserve.

## Solution Assessment

No. Queue-storage normalization is the right boundary, but this PR is not the best mergeable solution because it bundles malformed gateway runtime refresh code and still does not preserve the full current queue replay surface.
