## Summary

Keep open: current main still has the legacy queue-normalization gap, but this PR is not mergeable because it mixes that outbound fix with malformed gateway readiness/runtime-secret refresh code and still drops queued replay metadata.

## What This Changes

The PR adds legacy delivery-queue normalization and tests for replay metadata, plus unrelated gateway agents.create runtime-readiness refresh code and tests.

## Best Possible Solution

Land an outbound-storage-only normalizer that preserves every queued replay field, and split any gateway readiness or runtime-secrets refresh work into a separate owner-reviewed PR.
