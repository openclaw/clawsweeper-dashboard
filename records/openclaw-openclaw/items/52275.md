## Summary

Keep this PR open: the outbound queue metadata bug is still source-reproducible on current main, but this branch is not mergeable because it bundles malformed gateway runtime refresh code, omits current replay fields from the normalizer, and lacks real behavior proof.

## What This Changes

The PR adds legacy outbound delivery queue normalization and tests, plus gateway `agents.create` runtime refresh/readiness changes.

## Best Possible Solution

Land a current-main outbound-only queue storage normalizer that preserves every `QueuedDelivery` replay field with focused storage/recovery regression tests, and split gateway readiness/runtime refresh work into a separate PR.

## Reproduction Assessment

Yes. Source inspection shows current main reads queue JSON directly as `QueuedDelivery` and only backfills `lastAttemptAt`, while recovery forwards metadata that a legacy-entry normalizer must preserve.

## Solution Assessment

No. Queue-storage normalization is the right boundary, but this PR is not the best mergeable solution because it bundles malformed gateway runtime refresh code and still does not preserve the full current queue surface.
