## Summary

Keep open: this PR targets a real source-reproducible outbound delivery-queue recovery bug, but the current branch is not mergeable because it includes malformed unrelated gateway startup/runtime refresh changes and the queue normalizer still drops replay metadata.

## What This Changes

The PR normalizes legacy outbound delivery queue entries and tests, and also adds gateway agent-create runtime readiness refresh code and tests.

## Best Possible Solution

Land this as an outbound storage/recovery-only fix that canonicalizes legacy entries while preserving every queued replay field, and split any gateway readiness or runtime-secrets refresh work into a separate owner-reviewed PR if still needed.

## Reproduction Assessment

Yes, by source inspection. Current main reads legacy queue JSON as QueuedDelivery and failDelivery/loadPendingDeliveries do not remap payload/target/attempt/createdAt before retry bookkeeping or recovery replay.

## Solution Assessment

No. The storage normalizer is the right boundary, but this PR is not the best mergeable solution because it mixes in unrelated gateway runtime refresh code and still drops queued replay options.
