## Summary

Keep open: the outbound recovery bug is real, but this branch is not mergeable because it is conflicting, contains malformed and unrelated gateway runtime-refresh code, drops current queued replay fields, and lacks real behavior proof.

## What This Changes

The branch adds legacy delivery-queue normalization and tests to preserve replay metadata, plus gateway agents.create runtime-refresh/readiness changes.

## Best Possible Solution

Land a current-main outbound-only queue-storage normalizer that preserves the full QueuedDelivery replay surface with focused storage/recovery tests, and split gateway runtime refresh into a separate owner-reviewed PR.

## Reproduction Assessment

Yes, by source inspection: current main reads queued entries as QueuedDelivery and only backfills lastAttemptAt, while recovery replays metadata that a legacy-entry normalizer must preserve. I did not run a live crash-recovery repro in this read-only review.

## Solution Assessment

No. Queue-storage normalization is the right boundary, but this PR mixes in malformed gateway runtime-refresh code and normalizes by rebuilding a partial record that drops current replay fields.
