## Summary

This PR should stay open because the proposed branch is still structurally broken and mixes the outbound queue recovery fix with unrelated gateway readiness/runtime-secret refresh changes. The outbound recovery bug is valid by source inspection, but the current patch is not mergeable as-is and still loses queued replay metadata during normalization.

## What This Changes

The PR adds legacy delivery-queue normalization/tests for recovered outbound entries and also adds gateway agents.create readiness/runtime config refresh changes.

## Best Possible Solution

Land a current-main outbound-only queue storage normalizer that preserves every `QueuedDelivery` replay field with focused storage/recovery regression tests, and split gateway readiness/runtime refresh work into a separate PR.

## Reproduction Assessment

Yes, by source inspection. Current main reads queue JSON directly as `QueuedDelivery` and only backfills `lastAttemptAt`, while recovery replays metadata that a legacy-entry normalizer must preserve.

## Solution Assessment

No. Queue-storage normalization is the right boundary, but this PR is not the best mergeable solution because it bundles malformed gateway runtime refresh code and still does not preserve the full current queue surface.
