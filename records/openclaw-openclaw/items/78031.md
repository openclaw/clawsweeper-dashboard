## Summary

Keep open: the proposed compact bootstrap guidance is not already on current main and may be useful, but the PR still has correctness blockers around `continuation-skip`, overstates automatic savings, and lacks inspectable after-fix proof.

## What This Changes

The branch adds compact-writing rules and examples to `docs/reference/templates/BOOTSTRAP.md` and a `CHANGELOG.md` entry advertising shorter first-run workspace files.

## Best Possible Solution

Narrow the template wording to match runtime context-injection semantics, qualify savings as expected only after successful bootstrap rewriting, add inspectable real-run proof, and then let maintainers decide the compact-style tradeoff.

## Reproduction Assessment

Yes for the review findings: the PR head contains the `continuation-skip` startup exception and current-main source/tests show first-turn injection still occurs. No high-confidence reproduction exists for the claimed 83% generated-file savings because the PR body does not include inspectable output.

## Solution Assessment

No. The narrower maintainable path is to remove `continuation-skip` from the startup exception, qualify the savings claim, add inspectable proof, and leave the compact-template style decision to maintainers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P2] Remove continuation-skip from the startup exception:** `docs/reference/templates/BOOTSTRAP.md:18`
  - body: The new BOOTSTRAP rule says startup injection is skipped when `contextInjection` is `continuation-skip`, but current main only skips when mode is `never` or after a completed turn. This can mislead users about first-turn token costs; remove `continuation-skip` from this startup exception or reword it as a later-turn reinjection mode.

Make the savings conditional or expected rather than an automatic release result.
  - confidence: 0.9
