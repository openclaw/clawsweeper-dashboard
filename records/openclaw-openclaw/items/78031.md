## What This Changes

The PR adds compact-writing guidance and examples to `docs/reference/templates/BOOTSTRAP.md` plus a changelog note claiming shorter recurring prompt load from first-run workspace files.

## Best Possible Solution

Land a narrowed template update that accurately describes startup versus continuation behavior, scopes compact style to agent-generated instruction files, and presents token savings as conditional on generated output actually being compacted.

## Reproduction Assessment

Yes for the review findings: the PR diff contains the disputed wording, and current-main source plus tests show the actual `continuation-skip` behavior. No high-confidence live reproduction exists for the claimed generated-file savings because the PR body lacks inspectable after-fix output.

## Solution Assessment

No. The safer solution is to remove or qualify the startup `continuation-skip` exception, qualify the savings claim, add inspectable real proof, and let maintainers decide the compactness versus onboarding-tone tradeoff.
