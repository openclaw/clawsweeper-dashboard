## What This Changes

The PR adds compact-writing guidance and SOUL/AGENTS before-and-after examples to `docs/reference/templates/BOOTSTRAP.md` so first-run agents write shorter workspace files.

## Best Possible Solution

Keep the compact-writing direction only after the examples are rule-equivalent to current startup-context semantics and backed by real generated-workspace/token proof.

## Reproduction Assessment

Yes for the review finding: source inspection of the PR head shows the compact AGENTS example treats daily memory as injected context, while current docs and schema scope daily memory to a one-shot `/new` or `/reset` startup prelude.

## Solution Assessment
