## What This Changes

This PR changes `src/agents/sandbox/backend.ts` to store sandbox backend registrations in a `Symbol.for` process-global `resolveGlobalMap` instead of a module-local `Map`.

## Best Possible Solution

Land this PR or an equivalent process-global sandbox backend registry fix with split-module/Jiti coverage and redacted external backend proof, then close the canonical bug report.

## Reproduction Assessment

Yes, source-level with high confidence. Plugin code registers through `openclaw/plugin-sdk/sandbox` into a module-local map, while sandbox runtime lookup can happen through a separate module graph that calls `requireSandboxBackendFactory`.

## Solution Assessment

Yes on code direction. A `Symbol.for` process-global registry is the narrow maintainable fix, but merge should wait for real external-backend proof and focused split-module coverage.
