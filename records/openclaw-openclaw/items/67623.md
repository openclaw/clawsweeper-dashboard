## Summary

Keep this PR open. Current main still uses a module-local sandbox backend registry, the paired same-author bug remains open, and the branch is a narrow implementation candidate rather than obsolete cleanup.

## What This Changes

The PR changes the sandbox backend registry in `src/agents/sandbox/backend.ts` from a module-local `Map` to a `Symbol.for`/`resolveGlobalMap` process-global map.

## Best Possible Solution

Land this PR or an equivalent registry-scoping fix after adding the required changelog entry and preferably a focused duplicate-module regression test, while keeping the runtime change limited to the sandbox backend registry surface.

## Reproduction Assessment

Yes. A high-confidence seam-level reproducer is to import two fresh instances of `src/agents/sandbox/backend.ts`, register a unique backend in one instance, and require it from the other; current main's module-local `Map` makes that registration invisible across instances.

## Solution Assessment

Yes. Using the existing `resolveGlobalMap` helper with a stable `Symbol.for` key is the narrowest maintainable fix for process-local duplicate module instances; the safer follow-up is test coverage, not a broader plugin loader change.
