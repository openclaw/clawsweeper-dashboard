## Summary

Keep open: current main still uses a module-local sandbox backend registry, and the PR is a narrow fix for the linked plugin-facing registration bug, but it still needs the required changelog entry before merge.

## What This Changes

The PR changes the sandbox backend registry in `src/agents/sandbox/backend.ts` from a module-local `Map` to a `resolveGlobalMap` process-global map keyed by `Symbol.for`.

## Best Possible Solution

Land this PR or an equivalent registry-scoping fix after adding release-note coverage and a duplicate-module regression test, keeping the change limited to the generic sandbox backend registry.

## Reproduction Assessment

Yes. The linked report gives a concrete plugin registration path, and code inspection shows the failure mode: one module instance can register a backend while another instance's module-local map is used for `requireSandboxBackendFactory`.

## Solution Assessment

Yes. A `Symbol.for` process-global map through the existing `resolveGlobalMap` helper is the narrowest maintainable fix for duplicate module instances; the safer follow-up is focused coverage, not a broader plugin loader rewrite.
