## Summary

Keep open. The PR targets a source-reproducible plugin-loader bug that is still present on current main, but it still needs contributor real-behavior proof and maintainer approval of the non-activating discovery/config-validation boundary before merge.

## What This Changes

The PR skips plugin config schema validation for undefined config during non-activating snapshot and CLI metadata loads, preserves normalized/defaulted pluginConfig, adds loader regression tests, and includes SwiftFormat-only macOS brace changes.

## Best Possible Solution

Land this PR or an equivalent context-aware loader fix after maintainer approval and real runtime proof, keeping activation and explicit-config validation strict and leaving https://github.com/openclaw/openclaw/issues/64946 separately tracked.

## Reproduction Assessment

Yes. Source inspection on current main shows non-activating snapshot and CLI metadata paths pass undefined entry config into validatePluginConfig, which validates it as {}, matching the linked issue’s strict-schema false-positive path; I did not run a live external plugin in this read-only review.

## Solution Assessment

Unclear until maintainer security review. Functionally the PR is narrow and preserves activation plus explicit-config validation, but the repository owners should confirm that discovery registration may proceed when required runtime config is omitted.
