## Summary

Keep this PR open. Current main still validates omitted plugin config as an empty object in the non-activating snapshot and CLI metadata paths, so the branch addresses a real bug; however, the diff needs a required changelog entry and maintainer signoff on the discovery-mode validation boundary before merge.

## What This Changes

The PR defers schema validation for undefined plugin config in non-activating snapshot and CLI metadata loads, preserves normalized/defaulted pluginConfig, adds focused loader regression tests, and includes two SwiftFormat-only macOS brace changes.

## Best Possible Solution

Land this PR or an equivalent loader change after maintainer review confirms the discovery/CLI metadata boundary is safe, with activation and explicit-config validation remaining strict, schema defaults still applied for pluginConfig, a required changelog entry added, and #64946 left as a separate follow-up.

## Reproduction Assessment

Yes. A focused reproduction is a plugin with a strict top-level required configSchema loaded through loadOpenClawPlugins({ activate:false, loadModules:false }) or loadOpenClawPluginCliRegistry without plugins.entries.<id>.config; current main validates undefined as {} before returning metadata.

## Solution Assessment

Unclear pending maintainer security review. Functionally the patch is narrow and preserves activation plus explicit-config validation, but maintainers should approve whether discovery registration may proceed when required runtime config is omitted and represented by defaults or {}.
