## What This Changes

The branch defers config validation for undefined plugin config in non-activating plugin snapshot and CLI metadata loads, preserves normalized/defaulted pluginConfig, adds focused loader tests, and applies two SwiftFormat brace-only macOS edits.

## Best Possible Solution

Land this PR or an equivalent context-aware loader fix after maintainer approval: omitted config should be deferred only in non-activating metadata/preflight paths, activation and explicit-config validation should remain strict, schema defaults should still populate pluginConfig, and the sibling issue should stay separately tracked.

## Reproduction Assessment

Yes. From source inspection, a plugin with strict required root fields and no entry config reaches validatePluginConfig through loadOpenClawPlugins({ activate: false, loadModules: false }) or loadOpenClawPluginCliRegistry, where current main validates undefined as {} before returning metadata.

## Solution Assessment

Unclear pending maintainer security review. Functionally the PR is narrow and preserves activation plus explicit-config validation, but maintainers should confirm discovery registrations may proceed with omitted required runtime config represented by a normalized/defaulted object.
