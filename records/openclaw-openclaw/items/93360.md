## What This Changes

## Best Possible Solution

Land either this narrow permission-tool gate or the broader family-gate PR after maintainer scope choice, explicit acceptance of the fail-closed behavior, and refreshed green checks.

## Reproduction Assessment

Yes. Configure two Feishu accounts where account `a` has `tools.perm: false` and account `b` has `tools.perm: true`; current main registers `feishu_perm` from account `b` and then creates a client for selected account `a` without checking account `a`'s permission gate.

## Solution Assessment

Mostly yes for the narrow bug. The patch mirrors the existing Bitable selected-account gate, but maintainers should decide whether this narrow PR or the broader Feishu family-gate PR is the better final landing path.
