## Summary

Keep open: the focused patch appears correct and the contributor has now supplied sufficient redacted runtime proof, but the PR has a protected maintainer label and a real compatibility/scope decision against the broader Feishu family-gate PR remains.

## What This Changes

The PR resolves the selected Feishu account during `feishu_perm` execution, rejects selected accounts with permission tooling disabled, and adds contextual plus explicit-account routing regression tests.

## Best Possible Solution

Land exactly one account-gating path: either this narrow permission-only fix or the broader shared family-gate PR, with maintainers explicitly accepting the fail-closed selected-account behavior.

## Reproduction Assessment

Yes. Current main and v2026.6.6 register `feishu_perm` from any enabled account and create a client for the resolved account without checking that account's `tools.perm`; the PR adds focused regression tests for the contextual and explicit disabled-account cases.

## Solution Assessment

Mostly yes for the narrow permission-tool bug. The patch mirrors the existing Bitable selected-account gate, but maintainers still need to decide whether the broader shared family-gate PR is the better final shape.
