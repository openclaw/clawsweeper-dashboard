## Summary

Keep open: current main still emits plain generated Telegram blockquotes, while explicit raw `<blockquote expandable>` HTML is preserved. The remaining question is the maintainer-approved authoring policy for generated collapsible quotes, not a cleanup-close candidate.

## What This Changes

The issue requests that Telegram rich-message blockquote generation emit an expandable blockquote form for collapsible long quotes instead of only preserving that attribute in raw HTML.

## Best Possible Solution

Choose one maintainer-approved authoring policy for collapsible Telegram quote content, then implement it in the Telegram formatter with focused coverage and live proof while preserving normal blockquotes by default unless maintainers explicitly choose otherwise.

## Reproduction Assessment

Yes. Source and test inspection reproduce the current behavior: Markdown blockquotes render as plain `<blockquote>`, while raw `<blockquote expandable>` is preserved; no live Telegram send was run in this read-only review.

## Solution Assessment

Unclear until maintainers choose the authoring policy. A one-line universal expandable marker is not the best fix because it changes all existing quote presentation; an approved explicit marker, heuristic, or docs-confirmed rich HTML path would be safer.
