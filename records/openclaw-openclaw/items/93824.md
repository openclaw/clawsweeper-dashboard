## Summary

Keep open: current main and the latest release still generate plain Telegram `<blockquote>` for Markdown blockquotes, while only explicit raw `<blockquote expandable>` HTML is preserved. The remaining work is a product-policy decision about the generated authoring surface, not a cleanup close candidate.

## What This Changes

The issue asks for Telegram markdown blockquotes to emit Telegram's expandable blockquote attribute so long quotes can collapse in Telegram clients.

## Best Possible Solution

Choose one maintainer-approved Telegram authoring policy for collapsible quote content, then implement that policy with focused formatter/send coverage and live Telegram proof while preserving normal Markdown blockquotes by default unless maintainers explicitly choose otherwise.

## Reproduction Assessment

Yes. Source inspection shows Markdown blockquotes flow through `renderTelegramHtml` and emit plain `<blockquote>`, while explicit raw `<blockquote expandable>` is preserved; no live Telegram send was run in this read-only review.

## Solution Assessment

Unclear until maintainers choose the authoring policy. The one-line global marker change is not the best fix because it changes all generated blockquotes; a safer solution is an approved explicit marker, documented rich HTML path, or approved heuristic with tests and Telegram proof.
