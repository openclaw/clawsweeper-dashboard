## Summary

Keep open: current main and the latest release still generate plain Telegram blockquotes from Markdown while preserving explicit expandable HTML, and changing generated output needs maintainer direction because current guidance uses `<details>` for collapsible Telegram rich content.

## What This Changes

The issue asks for Telegram rich-message Markdown blockquotes to emit an expandable Telegram blockquote form instead of only accepting that attribute in explicit HTML.

## Best Possible Solution

Maintain the shipped explicit-HTML passthrough and choose one product policy for generated collapsible quotes before adding formatter behavior, tests, and live Telegram proof.

## Reproduction Assessment

Yes. Source inspection shows generated Markdown blockquotes still emit plain `<blockquote>` while explicit `<blockquote expandable>` HTML is preserved; no live Telegram client repro was run in this read-only review.

## Solution Assessment

Unclear. The requested behavior is plausible, but current guidance favors `<details>` for collapsible content, so maintainers should pick the authoring and default policy before code changes.
