## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Handle null parent-watch filenames:** `extensions/memory-core/src/memory/manager-sync-ops.ts:623-624`
  - body: Node documents that `fs.watch` may call listeners with `filename` set to null. The main watcher handles that, but this parent watcher returns unless the filename equals the watched basename, so a root replacement reported with a null filename leaves the native watcher bound to the old inode and later files under the recreated `memory/` tree may not drive sync. Treat null as an unknown parent event and re-check the watched directory inode.
  - confidence: 0.91
- **[P3] Close paired parent watchers on native errors:** `extensions/memory-core/src/memory/manager-sync-ops.ts:600-609`
  - body: When the main native watcher emits `error`, this path removes only the main watcher and adds chokidar fallback. The paired parent watcher for the same directory remains live, so a later basename event can reattach native coverage while chokidar is already watching the directory, adding duplicate handles and events. Close/remove the paired parent watcher before falling back.
  - confidence: 0.8
