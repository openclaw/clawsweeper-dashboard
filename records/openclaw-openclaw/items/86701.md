## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Gate native recursive watches off Linux:** `extensions/memory-core/src/memory/manager-sync-ops.ts:494-496`
  - body: On Linux and other non-macOS/non-Windows platforms, Node routes `fs.watch({ recursive: true })` through `internal/fs/recursive_watch`, which walks the tree and calls `fs.watch(file)` for discovered entries. That means this path does not provide the constant watcher profile the PR relies on, and large memory trees can still allocate O(n) watchers without hitting the creation-failure fallback. Keep those directories on the existing fallback unless a bounded Linux backend is proven.
  - confidence: 0.91
- **[P2] Restore directory coverage after native watcher errors:** `extensions/memory-core/src/memory/manager-sync-ops.ts:521-537`
  - body: This handler closes and removes the native watcher, then only schedules one broad sync. Since `intervalMinutes` defaults to 0, later changes under that directory will no longer produce watch-driven reindexing until the manager is recreated, so memory search can go stale after a single backend error. Recreate the native watcher or attach that directory to the chokidar fallback and test a subsequent event after the error.
  - confidence: 0.9
