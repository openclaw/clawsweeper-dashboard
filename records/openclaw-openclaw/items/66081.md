## Summary

Keep open: current main still lacks the Codex final-message artifact path, so this PR addresses a real CLI runner gap, but the proposed patch is not merge-ready because it broadens JSONL reply extraction beyond the stated `codex-cli` scope, misses command-log cleanup, and lacks the required changelog entry.

## What This Changes

The PR adds Codex CLI `--output-last-message` temp artifact creation, readback, cleanup, artifact-over-stdout final reply selection, and focused parser/runner tests for the `codex-cli` path.

## Best Possible Solution

Land the artifact-preferred Codex CLI finalization path after narrowing the JSONL content fallback to `codex-cli` or an explicit dialect, masking the generated artifact path in verbose argv logs, placing artifact allocation under the cleanup scope, and adding the required changelog entry with focused tests for artifact precedence, empty-artifact fallback, and non-Codex JSONL behavior.

## Reproduction Assessment

Yes. Current main inspection reproduces the missing behavior because the runner finalizes from stdout and the bundled `codex-cli` backend does not pass `--output-last-message`; the PR tests simulate the artifact-vs-stdout precedence case without needing a live Codex run.

## Solution Assessment

No, not as proposed. The artifact strategy is the right direction, but the current diff changes generic JSONL backend parsing and needs small command-execution hygiene, cleanup, and changelog fixes before it is the narrowest maintainable solution.

  - body: The new fallback extracts `message`, `content`, `result`, and `response` from every JSONL backend.   - confidence: 0.9

  - confidence: 0.86
- **[P3] Create the artifact inside the cleanup scope:** `src/agents/cli-runner/execute.ts:272-279`
  - body: The PR awaits Codex artifact preparation before the outer cleanup `finally` is active. If that new temp-file allocation fails after earlier prompt/image/plugin resources were prepared, those resources can be left behind.   - confidence: 0.72

  - body: Adding `--output-last-message <temp-file>` means verbose `cli argv` logs can include the generated temp directory and artifact filename. That path is not a credential, but it is command-execution log hygiene and should be masked like existing system prompt and image path arguments.
  - confidence: 0.82
