## Summary

Keep this PR open, but it is not merge-ready: the artifact direction is useful, yet the branch is conflicting, lacks real Codex CLI proof, and currently broadens generic JSONL parsing beyond the stated `codex-cli` scope.

## What This Changes

The PR wires direct `codex-cli` runs to pass an `--output-last-message` temp file, prefer its text over stdout, expands JSONL text extraction, and adds focused parser/runner tests.

## Best Possible Solution

Decide whether direct/custom `codex-cli` backends should still receive artifact finalization; if yes, rebase this into provider-scoped artifact support with masked logs, guarded cleanup, live proof, and leave broader JSONL terminal semantics to https://github.com/openclaw/openclaw/pull/59920.

## Reproduction Assessment

Yes at source level: current main still finalizes the generic CLI runner from streamed JSONL or parsed stdout and has no artifact readback. Live reproduction is less direct because current docs say bundled `codex-cli` is no longer registered by default.

## Solution Assessment

No, not as proposed. The artifact approach is plausible for an explicitly supported direct `codex-cli` backend, but the current diff changes generic JSONL parsing, needs temp-file log/cleanup fixes, lacks live proof, and needs maintainer direction after the app-server migration.

  - confidence: 0.72
