## Summary

Keep open: the requested artifact-preferred Codex CLI finalization is not on the generic CLI runner in current main, but the PR still needs real behavior proof and code changes before it is mergeable. Current main has also moved bundled Codex agent runs to the Codex app-server route, so this should be retargeted deliberately rather than swept closed.

## What This Changes

The PR adds Codex CLI `--output-last-message` temp artifact wiring so `codex-cli` final replies prefer that artifact over stdout, with stdout parsing as fallback.

## Best Possible Solution

Rebase or replace this with a narrow, current-main-aware patch only if maintainers still want explicit Codex CLI backend artifact finalization outside the Codex app-server route.

## Reproduction Assessment

Yes, source-level. Current main's generic CLI runner still finalizes successful CLI runs from streamed JSONL or parsed stdout, while the PR body also confirms no live Codex CLI after-fix run was verified.

## Solution Assessment

No, not as proposed. The artifact strategy is plausible for an explicit Codex CLI backend, but this patch changes generic JSONL parsing, needs log/cleanup fixes, and targets a legacy bundled backend surface after current main moved Codex agent runs to the app-server route.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Scope JSONL fallback to Codex:** `src/agents/cli-output.ts:486-493`
  - body: The new fallback extracts `message`, `content`, `result`, and `response` from every JSONL backend. Custom JSONL backends are documented public surface, so a non-Codex status or tool-result record can now be folded into the final assistant reply; guard this by `providerId === "codex-cli"` or an explicit dialect and cover a non-Codex regression case.
  - confidence: 0.9
- **[P3] Mask the generated artifact path in argv logs:** `src/agents/cli-runner/execute.ts:278-279`
  - body: Adding `--output-last-message <temp-file>` means verbose `cli argv` logs can include the generated temp directory and filename. Command-execution logs already mask injected prompt and image paths, so add a masking case for this generated artifact path too.
  - confidence: 0.82
- **[P3] Move artifact allocation under the cleanup guard:** `src/agents/cli-runner/execute.ts:272-275`
  - body: `prepareCodexLastMessageArtifact` is awaited before the outer cleanup `finally` starts. If that temp allocation fails after earlier prompt image, system prompt, or plugin resources were prepared, those resources can skip cleanup; allocate this inside the guarded block or start the cleanup guard before the new I/O point.
  - confidence: 0.72
