## Summary

Keep open: the Codex artifact-finalization direction is useful and still missing on current main, but this branch still has a blocking generic JSONL parsing regression plus missing real behavior proof before merge.

## What This Changes

This PR wires direct `codex-cli` runs to allocate a temporary `--output-last-message` file, prefer non-empty artifact text over stdout, extend JSONL content parsing, and add runner/parser tests.

## Best Possible Solution

Land a narrow `codex-cli` artifact-finalization change after scoping JSONL parsing to Codex or an explicit dialect, masking the artifact path, moving artifact setup under cleanup protection, reconciling parser overlap with https://github.com/openclaw/openclaw/pull/59920, and adding redacted real Codex CLI proof.

## Reproduction Assessment

Yes, source-level. Current main's bundled `codex-cli` args omit `--output-last-message`, and `executePreparedCliRun` still finalizes successful runs from parsed stdout/JSONL rather than an artifact; I did not run a live Codex finalization path in this read-only review.

## Solution Assessment

No, not as proposed. The artifact strategy is the right direction, but the patch should stay provider-scoped, preserve generic JSONL backend behavior, mask the generated artifact path, protect cleanup, and include real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Scope JSONL content fallback to Codex:** `src/agents/cli-output.ts:486-493`
  - body: This fallback now extracts `message`, `content`, `result`, and `response` from every JSONL backend. OpenClaw documents custom JSONL CLI backends, so status or tool-result records from non-Codex backends can be folded into the final assistant reply; guard this with `providerId === "codex-cli"` or an explicit dialect and cover a non-Codex regression case.
  - confidence: 0.9
- **[P3] Mask the output-last-message path in argv logs:** `src/agents/cli-runner/execute.ts:278-279`
  - body: Adding `--output-last-message <temp-file>` means verbose `cli argv` logs can include the generated temp directory and artifact filename. The path is not a credential, but command-execution logs already mask injected prompt/image paths, so add a masking case for this generated artifact path too.
  - confidence: 0.82
- **[P3] Create the artifact inside the cleanup scope:** `src/agents/cli-runner/execute.ts:272-275`
  - body: `prepareCodexLastMessageArtifact` is awaited before the outer cleanup `finally` starts. If that temp allocation fails after earlier prompt image, system prompt, or plugin resources were prepared, their cleanup will not run; move artifact setup under the guarded cleanup scope or start the cleanup `try` before this I/O point.
  - confidence: 0.72
