## Summary

Keep open: the core artifact-finalization idea is useful and not on current main, but the patch is not merge-ready because it broadens generic JSONL parsing, has small runner hygiene issues, and lacks external real behavior proof.

## What This Changes

The PR wires `codex-cli` runs to create/read/cleanup a temp `--output-last-message` artifact, prefer non-empty artifact text over stdout, and adds JSONL parser plus runner tests.

## Best Possible Solution

Land the Codex CLI artifact-finalization path after scoping JSONL parsing to Codex or an explicit dialect, masking the artifact path, moving setup under cleanup protection, reconciling parser overlap with https://github.com/openclaw/openclaw/pull/59920, and adding redacted real Codex CLI proof.

## Reproduction Assessment

Yes. Current-main source shows the bundled `codex-cli` backend does not pass `--output-last-message`, and `executePreparedCliRun` still finalizes successful CLI runs from stdout/JSONL parsing rather than a Codex artifact.

## Solution Assessment

No, not as proposed. The artifact strategy is the right direction, but the patch currently changes generic JSONL behavior and needs runner log/cleanup fixes plus real behavior proof before it is the narrow maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Scope JSONL content fallback to Codex:** `src/agents/cli-output.ts:486-493`
  - body: The new fallback extracts `message`, `content`, `result`, and `response` from every JSONL backend. OpenClaw documents custom JSONL CLI backends, so status or tool-result records from non-Codex backends can now be folded into the final assistant reply; guard this with `providerId === "codex-cli"` or an explicit dialect and cover a non-Codex regression case.
  - confidence: 0.9
- **[P3] Mask the output-last-message path in argv logs:** `src/agents/cli-runner/execute.ts:278-279`
  - body: Adding `--output-last-message <temp-file>` means verbose `cli argv` logs can include the generated temp directory and artifact filename. The path is not a credential, but command-execution logs already mask injected prompt/image paths, so add a masking case for this generated artifact path too.
  - confidence: 0.82
- **[P3] Create the artifact inside the cleanup scope:** `src/agents/cli-runner/execute.ts:272-275`
  - body: `prepareCodexLastMessageArtifact` is awaited before the outer cleanup `finally` starts. If that new temp allocation fails after earlier prompt image, system prompt, or plugin resources were prepared, their cleanup will not run; move artifact setup under the guarded cleanup scope or start the cleanup `try` before this I/O point.
  - confidence: 0.72
