## Summary

Keep this PR open: the Codex artifact direction matches the linked reliability request, but the current patch still broadens generic JSONL parsing and has narrow command-execution hygiene and changelog blockers before merge.

## What This Changes

The PR wires `codex-cli` runs to create/read a `--output-last-message` temp artifact, prefer artifact text over stdout, extend JSONL parsing for Codex content records, and add focused runner/parser tests.

## Best Possible Solution

Land the artifact-preferred Codex CLI finalization path after scoping JSONL fallback to Codex or an explicit dialect, moving artifact allocation under cleanup coverage, masking the generated path in verbose logs, adding the changelog entry, and rerunning focused tests plus the changed gate.

## Reproduction Assessment

Yes. Current main inspection shows `codex-cli` finalization still depends on stdout and the bundled backend does not pass `--output-last-message`; the PR’s tests simulate the artifact-versus-stdout precedence path without needing a live Codex run.

## Solution Assessment

No, not as proposed. The artifact strategy is the right narrow direction, but the current diff changes generic JSONL backend parsing and needs small cleanup/logging/changelog repairs before it is the maintainable fix.
