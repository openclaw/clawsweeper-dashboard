## Summary

Keep open: current main still lacks Codex CLI final-message artifact finalization, and this PR is the focused implementation path for the closed enhancement request, but the patch still needs changes before merge and external real behavior proof is only mock/test-based so far.

## What This Changes

The PR adds Codex CLI `--output-last-message` temp artifact creation/readback/cleanup, prefers non-empty artifact text over stdout for `codex-cli` final replies, and adds parser/runner tests.

## Best Possible Solution

Land the Codex CLI artifact-finalization path after scoping the JSONL fallback to `codex-cli` or an explicit dialect, masking the artifact path in verbose argv logs, covering artifact cleanup on setup failures, and adding redacted real Codex CLI proof.

## Reproduction Assessment

Yes. Current-main source has a high-confidence reproduction of the missing behavior: the bundled `codex-cli` backend does not pass `--output-last-message`, and `executePreparedCliRun` finalizes from stdout/JSONL parsing. The PR's tests simulate artifact precedence, but the PR body does not show a live Codex CLI run.

## Solution Assessment

No, not as proposed. The artifact strategy is the right direction, but the patch currently broadens generic JSONL parsing and needs small runner hygiene plus real behavior proof before it is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Scope JSONL content fallback to Codex:** `src/agents/cli-output.ts:486-492`
  - body: The new fallback extracts `message`, `content`, `result`, and `response` from every JSONL backend. OpenClaw documents custom JSONL CLI backends, so status or tool-result records from non-Codex backends can now be folded into the final assistant reply; guard this with `providerId === "codex-cli"` or an explicit dialect and cover a non-Codex regression case.
  - confidence: 0.9
- **[P3] Mask the output-last-message path in argv logs:** `src/agents/cli-runner/execute.ts:278`
  - body: Adding `--output-last-message <temp-file>` means verbose `cli argv` logs can include the generated temp directory and artifact filename. The path is not a credential, but command-execution logs already mask injected prompt/image paths, so add a masking case for this generated artifact path too.
  - confidence: 0.82
- **[P3] Create the artifact inside the cleanup scope:** `src/agents/cli-runner/execute.ts:272-275`
  - body: `prepareCodexLastMessageArtifact` is awaited before the outer cleanup `finally` starts. If that new temp allocation fails after earlier prompt image, system prompt, or plugin resources were prepared, their cleanup will not run; move artifact setup under the guarded cleanup scope or start the cleanup `try` before this I/O point.
  - confidence: 0.72
