## Summary

Keep open: current main still lacks Codex `--output-last-message` finalization, and this PR remains the canonical implementation path, but the current head is not merge-ready because it broadens generic JSONL parsing, exposes the generated artifact path in verbose argv logs, allocates the artifact before the cleanup guard, and lacks real behavior proof.

## What This Changes

The PR adds Codex CLI `--output-last-message` temp artifact wiring, prefers non-empty artifact text over stdout, and adds JSONL parser plus runner tests.

## Best Possible Solution

Land a narrow Codex CLI artifact-finalization patch after scoping parser changes to Codex or an explicit dialect, fixing log masking and cleanup ordering, reconciling https://github.com/openclaw/openclaw/pull/59920, and adding redacted real Codex CLI proof.

## Reproduction Assessment

Yes, source-level. Current main's bundled `codex-cli` backend does not pass `--output-last-message`, and `executePreparedCliRun` still finalizes successful CLI runs from parsed stdout/JSONL rather than a Codex artifact.

## Solution Assessment

No, not as proposed. The artifact strategy is the right direction, but the current patch changes generic JSONL behavior, needs log/cleanup fixes, and lacks real behavior proof before it is the narrow maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Scope JSONL fallback to Codex:** `src/agents/cli-output.ts:486-493`
  - body: The new fallback extracts `message`, `content`, `result`, and `response` from every JSONL backend. Custom JSONL backends are documented public surface, so a non-Codex status or tool-result record can now be folded into the final assistant reply; guard this by `providerId === "codex-cli"` or an explicit dialect and cover a non-Codex regression case.
  - confidence: 0.9
- **[P3] Mask the generated artifact path in argv logs:** `src/agents/cli-runner/execute.ts:278-279`
  - body: Adding `--output-last-message <temp-file>` means verbose `cli argv` logs can include the generated temp directory and filename. Command-execution logs already mask injected prompt and image paths, so add a masking case for this generated artifact path too.
  - confidence: 0.82
- **[P3] Move artifact allocation under the cleanup guard:** `src/agents/cli-runner/execute.ts:272-275`
  - body: `prepareCodexLastMessageArtifact` is awaited before the outer cleanup `finally` starts. If that temp allocation fails after earlier prompt image, system prompt, or plugin resources were prepared, those resources skip cleanup; start the cleanup guard before this new I/O point or allocate inside the guarded block.
  - confidence: 0.72
