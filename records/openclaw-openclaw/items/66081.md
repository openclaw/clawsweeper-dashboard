## Summary

Keep this PR open: current main still lacks direct CLI-runner final-message artifact readback, and the linked enhancement was closed in favor of this PR, but the branch is not merge-ready because it broadens generic JSONL parsing, has temp-file log/cleanup issues, lacks real Codex CLI proof, and now needs a maintainer decision after the Codex app-server migration.

## What This Changes

The PR adds `codex-cli` `--output-last-message` temp-file wiring, prefers that artifact over stdout for final replies, extends JSONL text extraction, and adds focused runner/parser tests.

## Best Possible Solution

Decide whether explicit custom/direct `codex-cli` backends should still receive artifact finalization after the app-server migration; if yes, rebase this into provider-scoped artifact support with live proof and leave broader JSONL terminal semantics to https://github.com/openclaw/openclaw/pull/59920.

## Reproduction Assessment

Yes at source level: current main still finalizes generic CLI runs from streamed JSONL or parsed stdout and has no artifact readback in `executePreparedCliRun`. Live reproduction is less direct because current docs say bundled `codex-cli` is no longer registered by default.

## Solution Assessment

No, not as proposed. The artifact approach is plausible for an explicitly supported direct `codex-cli` backend, but the current diff changes generic JSONL parsing, needs temp-file log/cleanup fixes, lacks live proof, and needs maintainer direction after the app-server migration.
