## Summary

Keep open: current main still uses hidden system-prompt delivery for the native subagent task and exposes no supported visible task-envelope option or hook. The adjacent `systemPromptOverride` composition path is now covered, but the transcript-visible delivery request remains a feature/API decision.

## What This Changes

Request an opt-in visible/auditable native `sessions_spawn` task-delivery envelope or hook/config surface so child transcripts can contain the delegated task.

## Best Possible Solution

Design an explicit opt-in native `sessions_spawn` task-delivery mode or transform hook, keep the token-saving system-only default unless maintainers choose otherwise, and add regression coverage for transcript-visible `TASK_ID` delivery.

## Reproduction Assessment

Yes for the requested surface: source inspection shows current main builds a generic first user message and passes the delegated task only through `extraSystemPrompt`. I did not live-reproduce the configured-agent startup override, but the discussion adds field evidence from OpenClaw 2026.5.5.

## Solution Assessment

Unclear until maintainers choose the API shape. An opt-in visible mode or hook is plausible, while changing the default would risk reintroducing the duplicate-token behavior fixed by https://github.com/openclaw/openclaw/issues/72019.
