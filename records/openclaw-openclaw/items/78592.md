## Summary

Keep open: current main still has hidden-only native subagent task delivery with no visible-envelope tool, config, or hook surface, and the linked open PR is the active implementation candidate rather than a reason to close the issue before merge.

## What This Changes

Requests a supported native `sessions_spawn` mode, config, or hook that makes the delegated task visible and auditable in the child transcript.

## Best Possible Solution

Review and land a narrowly scoped opt-in visible task-envelope feature, or choose an equivalent UI/hook design, while preserving hidden system-prompt-only delivery as the default.

## Reproduction Assessment

Yes for the visible transcript gap: current source and tests show the child’s first visible user message omits the delegated task and points to `**Your Role**`. I did not locally reproduce the configured-agent startup wandering, but the issue comments add field reports on OpenClaw `2026.5.5` and `v2026.5.7`.

## Solution Assessment

Unclear until maintainers accept the public API shape. The linked opt-in boolean is a narrow, backwards-compatible direction, but this should stay open until that PR is reviewed, merged, or replaced by another maintainer-approved design.
