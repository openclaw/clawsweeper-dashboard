## Summary

Keep open: current main intentionally sends the native subagent task through `extraSystemPrompt` with only a generic visible kickoff message, and existing tool/config/hook surfaces do not provide the requested visible task-envelope control. This is a feature/API decision rather than conservative cleanup.

## What This Changes

Request a supported native `sessions_spawn` mode, config, or hook that can put the delegated task envelope in the child’s first visible message for auditability.

## Best Possible Solution

Add a deliberately scoped native `sessions_spawn` task-delivery mode or hook with regression coverage, while preserving the token-saving hidden system-only delivery as the default unless maintainers choose otherwise.

## Reproduction Assessment

No high-confidence live reproduction of the startup override failure was established. Source inspection does confirm the current visible child message omits the task and points to `**Your Role**` in the system prompt.

## Solution Assessment

Unclear until maintainers choose the API shape. A visible delivery option is plausible, but it should be designed against the existing token-duplication fix rather than changing the default blindly.
