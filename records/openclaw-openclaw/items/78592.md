## Summary

Keep open: current main still intentionally hides the delegated task from the native subagent's first visible user message, and the linked implementation PR that closes this issue is still open. This is a feature/API decision around an opt-in visible task-delivery mode, not a cleanup-close candidate.

## What This Changes

The issue requests an opt-in visible task envelope or hook/config surface for native `sessions_spawn` so delegated tasks can be audited in child transcripts.

## Best Possible Solution

Review and land a narrow opt-in visible task-delivery surface, or choose an explicit transcript/UI metadata rendering path, while preserving hidden system-only delivery as the default unless maintainers decide otherwise.

## Reproduction Assessment

Yes for the central auditability gap: current source and tests show the child first visible message omits the delegated task while the system prompt carries it. The broader configured-agent startup override remains field-reported rather than live-reproduced in this review.

## Solution Assessment

Unclear until maintainers choose the API shape. The open opt-in `visibleTaskEnvelope` direction looks narrow and compatible with the token-saving default, but it still needs review as a user-facing tool/schema contract.
