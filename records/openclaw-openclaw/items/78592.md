## Summary

Keep open: current main still uses hidden-only native subagent task delivery, and the remaining visible/auditable task-envelope work is already paired with the open implementation PR at https://github.com/openclaw/openclaw/pull/78992. Because that PR uses closing syntax for this issue, this issue should stay open until the PR lands or maintainers choose a different API direction.

## What This Changes

The issue requests an opt-in visible task envelope or hook/config surface so native `sessions_spawn` can expose the delegated task in the child transcript for auditability.

## Best Possible Solution

Review, land, or revise https://github.com/openclaw/openclaw/pull/78992 as the canonical implementation path while preserving the hidden system-prompt-only default unless maintainers choose otherwise.

## Reproduction Assessment

Yes for the visible-transcript symptom: current source and tests show the delegated task is intentionally absent from the first visible user message, and commenters reported the same behavior on v2026.5.5 and v2026.5.7. I did not live-reproduce the configured-agent startup override path.

## Solution Assessment

Yes, keeping this issue open is the best cleanup outcome because https://github.com/openclaw/openclaw/pull/78992 is the active closing implementation candidate. The narrower solution is an opt-in native task-delivery mode rather than reverting the token-dedupe behavior from https://github.com/openclaw/openclaw/issues/72019 by default.
