## Summary

Keep open. The Prometheus quick start still has a real docs gap, but this PR's warning wording should be tightened before merge because it can steer controlled deployments away from explicit plugin inventories and overstates allowlist effects around selected-slot and bundled-channel exceptions.

## What This Changes

The PR adds a warning callout to the Prometheus quick start explaining that `plugins.allow` behaves as a strict allowlist and can disable other configured plugins if copied into an existing gateway config.

## Best Possible Solution

Land a tightened warning that tells existing allowlist users to add `diagnostics-prometheus` to their current list, describes omitted plugins as potentially disabled, and preserves explicit plugin-inventory guidance.

## Solution Assessment

No, not as written. An inline warning beside the quick-start config is the right shape, but the wording should preserve existing allowlists and avoid exception-sensitive absolute examples.
