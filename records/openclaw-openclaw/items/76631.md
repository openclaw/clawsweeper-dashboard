## Summary

Keep open. The docs gap is still present on current main, but this PR needs a small wording adjustment and after-fix docs proof before merge because the new warning overstates the current memory-slot and bundled-channel exceptions.

## What This Changes

The PR adds a Warning callout to the Prometheus quick start explaining that `plugins.allow` is a strict allowlist and can disable other configured plugins.

## Best Possible Solution

Merge a precise warning beside the Prometheus snippet that says omitted plugins may be disabled, names or avoids the current slot/channel exceptions, and links readers to the current plugin configuration reference rather than relying on a closed discussion thread.

## Reproduction Assessment

Yes. Current main still has the Prometheus quick-start allowlist snippet without an adjacent warning, and source/tests show that a non-empty `plugins.allow` disables omitted plugin ids except for narrow documented exceptions.

## Solution Assessment

Mostly yes. An additive Warning next to the risky snippet is the narrowest docs fix, but the current wording should be softened so it does not state that selected memory slots or all channel plugins are always disabled.
