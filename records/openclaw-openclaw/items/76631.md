## Summary

Keep open: current main still has the Prometheus quick-start allowlist snippet without this warning, and the PR is a useful docs-only fix; the remaining issue is precision in the new warning text around known allowlist exceptions.

## What This Changes

The branch adds a `docs/gateway/prometheus.md` Warning callout explaining that `plugins.allow` is a strict allowlist and can disable other configured plugins when they are omitted.

## Best Possible Solution

Merge a precise warning beside the Prometheus snippet that explains strict allowlist behavior without promising more than the activation code enforces.

## Reproduction Assessment

Yes. The docs gap is visible on current main, and the allowlist behavior is source-reproducible through `resolvePluginActivationDecisionShared`; I did not run a live gateway or docs preview in this read-only review.

## Solution Assessment

Mostly yes. An additive Warning is the narrowest maintainable fix for the immediate docs pitfall, but the wording should be softened or exception-aware before merge.
