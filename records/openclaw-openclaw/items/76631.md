## Summary

Keep open. The PR addresses a real current docs gap: the Prometheus quick start shows a restrictive `plugins.allow` example without explaining that omitted plugin ids are excluded, but the added warning should be tuned so it does not overstate the memory-slot and bundled-channel exceptions already present in the activation logic.

## What This Changes

The PR adds a `Warning` callout to `docs/gateway/prometheus.md` explaining that `plugins.allow` is strict-mode and existing multi-plugin gateways should include other desired plugin ids or omit it.

## Best Possible Solution

Merge a precise warning beside the Prometheus quick start that explains restrictive allowlist behavior while avoiding blanket claims about plugins covered by slot or channel exceptions.

## Reproduction Assessment

Yes. Current `main` shows the risky Prometheus `plugins.allow` snippet without adjacent warning text, and source inspection shows omitted plugin ids are disabled under a non-empty allowlist; I did not run a live gateway or docs preview in this read-only pass.

## Solution Assessment

Mostly yes. An additive docs warning is the narrowest maintainable fix for the immediate pitfall, but the wording should be softened to match selected memory/context slot and conditional bundled-channel exceptions.
