## Best Possible Solution

Land a docs-only warning beside the Prometheus snippet after tightening it to say omitted plugins may be disabled, naming or avoiding the exceptions, and linking current plugin/config documentation.

## Reproduction Assessment

Yes. Current main and `v2026.6.6` both show the Prometheus `plugins.allow` example without an adjacent warning, and source/tests show restrictive allowlists disable omitted plugins except for documented exceptions.

## Solution Assessment

Mostly yes. An inline Warning is the narrowest maintainable docs fix, but this version should be made precise and should point at current config docs now that https://github.com/openclaw/openclaw/pull/77194 has landed.
