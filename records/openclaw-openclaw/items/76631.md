## Summary

Keep open: the docs-only PR addresses a real gap in current main, where the Prometheus quick start shows a strict `plugins.allow` snippet without warning existing multi-plugin gateways. The patch is directionally correct, with one low-priority wording precision issue around current slot/channel exceptions.

## What This Changes

The PR adds a `<Warning>` callout to `docs/gateway/prometheus.md` explaining that `plugins.allow` is strict and existing gateways should include other desired plugin ids or omit it.

## Best Possible Solution

Merge a precise warning beside the Prometheus snippet that names strict allowlist behavior and avoids promising behavior around memory-slot or channel exceptions.

## Reproduction Assessment

Yes for the docs gap and source behavior: current main shows the risky Prometheus snippet without an adjacent warning, and the activation code disables omitted plugin ids under a non-empty allowlist. I did not run a live gateway or docs preview in this read-only pass.

## Solution Assessment

Mostly yes. An additive Warning is the narrowest maintainable docs fix for the immediate pitfall, but the sentence listing disabled plugins should be softened to match the existing slot and channel exceptions.
