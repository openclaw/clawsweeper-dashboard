## Summary

Keep open. Current main still treats `host.orb.internal` as remote for local-marker auth, while the issue shows a host-backed Ollama endpoint reachable from the VM, so this is a narrow valid auth-provider regression rather than stale or duplicate cleanup.

## What This Changes

Fix local Ollama in an Orb VM so a reachable `host.orb.internal` Ollama base URL works after upgrade without requiring a real API key.

## Best Possible Solution

Add a narrow OrbStack host classification for local Ollama auth and discovery, with regression tests and docs, while keeping public or arbitrary remote Ollama hosts credential-required.

## Reproduction Assessment

Yes, at source level: `host.orb.internal` is reachable in the report but is not classified as local by either current auth classifier, so `ollama-local` can fall through to the missing-key error. A live OrbStack run would still be useful before landing.

## Solution Assessment

Yes: the maintainable fix is to align the core and Ollama plugin local-host classifiers for the exact OrbStack host case, with tests, rather than adding a new config knob or accepting broad `.internal` hosts.
