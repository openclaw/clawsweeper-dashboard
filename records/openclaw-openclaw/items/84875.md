## What This Changes

Local Ollama configured through the Orb VM host alias should keep working after upgrade without requiring a real API key or auth profile.

## Best Possible Solution

Recognize `host.orb.internal` as an explicit local/self-hosted Ollama no-auth path, add regression coverage for that alias, and preserve the current credential requirement for public remote hosts.

## Repair Work Prompt

Current main rejects `baseUrl: "http://host.orb.internal:11434"` for local Ollama no-auth behavior because both `src/agents/model-auth.ts` and `extensions/ollama/src/discovery-shared.ts` treat dotted non-`.local` hostnames as remote.
