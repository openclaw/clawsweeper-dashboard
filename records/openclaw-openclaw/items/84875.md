## What This Changes

The issue reports an Ollama regression where a VM-to-host base URL using `host.orb.internal` is treated as needing a real provider credential instead of local no-auth Ollama access.

## Best Possible Solution

Support known VM/container host-gateway aliases such as `host.orb.internal` and the already documented `host.docker.internal` through the existing local Ollama no-auth path, with tests proving public remote hosts still require credentials.

## Reproduction Assessment

## Solution Assessment

Yes: the best path is a narrow local-host-alias repair with regression tests, not a broad rule that accepts `ollama-local` for arbitrary public dotted hosts.
