## What This Changes

The issue requests restoring local Ollama no-auth behavior for an OpenClaw-in-Orb VM setup using `host.orb.internal` after upgrading to v2026.5.19.

## Best Possible Solution

Add a narrow, tested local-host policy update for known VM host aliases such as `host.orb.internal` while preserving real-credential requirements for public remote Ollama endpoints.

## Solution Assessment

Yes, the best path is a narrow local-host alias fix with regression tests; broadening all dotted or `.internal` hostnames would be a riskier auth-boundary change.
