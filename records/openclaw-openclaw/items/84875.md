## Repair Work Prompt

Fix the Ollama local-auth regression for Orb VM host-to-host setups. The reported config uses `models.providers.ollama.baseUrl: "http://host.orb.internal:11434"` with a local/no-auth Ollama marker and no real API key; current auth predicates do not classify that host alias as local, so runtime can throw `No API key found for provider "ollama"`. Keep the fix narrow: support the Orb host alias or an explicit local Ollama marker without allowing Ollama Cloud/public remotes to bypass real credentials. Add regression coverage for runtime auth and Ollama discovery/local-host classification, and check setup behavior around `/api/tags` separately from `/api/generate`. Update docs/changelog if the user-visible auth rule changes. Do not change provider routing, auth profile storage format, or broad SSRF policy.
