## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/84875 on current main. The reported regression is that OpenClaw inside an Orb VM cannot use a host Ollama server at `http://host.orb.internal:11434` with the non-secret `ollama-local` marker and falls through to `No API key found for provider "ollama"`. Keep the fix inside the Ollama/local-auth boundary: support `host.orb.internal` as a local/self-hosted Ollama host alias in both the core custom-provider marker path and the Ollama plugin synthetic-auth path, but do not allow arbitrary public dotted hosts to bypass real credentials. Add regression tests for `host.orb.internal` and for public remote hosts still requiring credentials. Also verify whether the setup wizard's `/api/tags` reachability failure is resolved by the same alias policy or needs a focused probe/SSRF test. Do not add a config option or broaden cloud/remote credential behavior. Update docs only if the accepted local-host alias list changes in user-visible Ollama docs.
