## Summary

Keep open. Current main still has the reported recovery gap for billing-disabled fallback chains that share the same auth profile, and plugin/configure auth refresh paths still upsert fresh credentials without clearing stale cooldown state.

## What This Changes

The issue asks OpenClaw to recover from persisted provider/profile cooldowns after billing or credential recovery without requiring manual `auth-state.json` edits.

## Best Possible Solution

Keep persisted cooldown protection, but add targeted recovery for same-auth billing-disabled chains and clear matching usage state after successful configure/plugin auth refresh.

## Reproduction Assessment

Yes, source-level. The issue gives concrete billing and re-auth repros, and current main still shows the same-auth fallback suspension and plugin/configure auth cleanup gaps without live provider credentials.

## Solution Assessment

Yes. The maintainable fix is targeted recovery that preserves persisted cooldowns while probing when fallbacks do not provide a real alternate auth path and clearing stale state after deliberate credential refresh.
