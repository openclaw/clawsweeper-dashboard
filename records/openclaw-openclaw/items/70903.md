Keep open. Current main and the latest release still contain the source-level recovery gap, and two open PRs with closing syntax are active fix candidates, so this issue should remain the canonical tracker until a fix lands.

## What This Changes

The issue asks OpenClaw to recover auth profiles from persisted billing/cooldown state after billing or re-authentication succeeds, instead of blocking provider selection for hours.

## Best Possible Solution

Land or consolidate a canonical fix that reuses the existing probe throttle and `clearAuthProfileCooldown` cleanup path for same-auth billing recovery and successful plugin/configure re-auth, then close this issue after merge.

## Reproduction Assessment

Yes, source-level: current main still suppresses far-from-expiry billing probes when configured fallback candidates exist, and plugin/configure auth can save refreshed credentials without clearing stale usage state. I did not run a live paid-provider billing recovery path in this read-only review.

## Solution Assessment

Yes. The best fix is the narrow recovery path already suggested by current code: keep persisted cooldown protection, but probe same-auth billing-disabled chains and clear matching usage state after successful re-auth rather than adding a broad reset workflow first.
