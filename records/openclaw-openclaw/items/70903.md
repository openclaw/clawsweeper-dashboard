## Summary

Keep open. Current main still has the core recovery gap: billing-disabled profiles remain persisted for long windows, same-auth fallback chains can avoid far-from-expiry probes, and configure/plugin auth refresh paths can upsert fresh credentials without clearing stale usage state.

## What This Changes

The issue asks OpenClaw to recover from persisted provider auth cooldown state after billing recovery or credential refresh without manual auth-state editing.

## Best Possible Solution

Keep persisted cooldown protection, but add targeted recovery for billing-disabled chains without a real alternate auth path and clear matching provider/profile usage state after successful configure/plugin auth refresh.

## Reproduction Assessment

Yes, source-level. The issue gives concrete billing and re-auth repros, and current main still shows the same-auth fallback suspension plus configure/plugin-auth cleanup gap without needing live provider credentials.

## Solution Assessment

Yes. Targeted recovery is the narrow maintainable direction: preserve persisted cooldowns and auth protection while probing when fallbacks do not provide a real alternate path and clearing stale state after deliberate credential refresh.
