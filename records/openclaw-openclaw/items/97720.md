## Summary

Keep open: current main still drops Telegram member join/leave service messages before an agent-visible event can be built, and the remaining decision is whether to add a Telegram-plugin event toggle or a reusable channel event seam rather than closing this as a plugin-only request.

## What This Changes

Request an opt-in, default-off Telegram group setting that turns member join/leave service messages into agent-visible events for skills.

## Best Possible Solution

Keep this open for maintainer direction: either add a narrow opt-in Telegram plugin event using existing system-event semantics, or first add a reusable channel event SDK seam if maintainers want this externalized.

## Reproduction Assessment

Yes, at source level: Telegram member service messages have no text/media placeholder path, so the inbound body builder returns null and the message processor skips them. This is a feature gap rather than a broken existing contract.

## Solution Assessment

Unclear as proposed: the reaction notification system-event path is a plausible implementation model, but maintainers should first decide whether this belongs in Telegram plugin config or a reusable channel event API.
