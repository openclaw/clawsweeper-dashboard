## What This Changes

The PR adds optional `detailCode` preservation to `AcpRuntimeError`, forwards it through ACP turn event/result error seams, and changes stale persistent-session recovery to match acpx `SESSION_RESUME_REQUIRED` instead of backend reason text.

## Best Possible Solution

Land the structured `SESSION_RESUME_REQUIRED` recovery after maintainer acceptance of the compatibility, session-state, and message-delivery risk, then close the linked stale Kiro ACP session issue once the fix reaches main.

## Solution Assessment

Yes, with a maintainer compatibility choice: preserving and matching acpx's structured `detailCode` is narrower and more maintainable than expanding backend reason-text regexes, and the PR covers both event and failed-result seams.
