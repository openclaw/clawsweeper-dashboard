## What This Changes

The issue reports that memory-core Dreaming on 2026.6.1 ingests archived `.jsonl.deleted.*` session paths into `session-corpus` and writes fallback DREAMS.md text even when the model appears to return prose.

## Best Possible Solution

Land a focused memory-core fix that prevents archived/generated session transcripts from polluting Dreaming corpus and makes narrative readback use the gateway-supported assistant output shape before falling back.

## Reproduction Assessment

No full live current-main reproduction was run in this read-only review. Source inspection shows a credible path for archived transcript paths to enter Dreaming corpus and for narrative readback to fall back when `sessions.get` lacks extractable assistant text.

## Solution Assessment

No, closing as duplicate is not the best solution because the related archive-ingestion issue does not cover the Dream Diary no-text fallback. The best fix should share archive-ingestion work with the related open issue while adding narrative readback coverage for valid prose responses.
