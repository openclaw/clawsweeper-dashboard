## Summary

Keep open: current main and latest release still match the narrowed report. Runtime/config docs already describe bounded safe deferral, but daemon/gateway CLI help and nearby docs still present `--safe` as drain-only without pointing operators to `gateway.reload.deferralTimeoutMs=0` for indefinite waits.

## What This Changes

The issue asks for safe daemon restart behavior or documentation to clarify that bounded safe deferral can age into a forced restart while active work remains.

## Best Possible Solution

Keep the current bounded restart implementation unless maintainers choose otherwise, and clarify daemon/gateway CLI help and docs that `--safe` is bounded by `gateway.reload.deferralTimeoutMs` with `0` as the current indefinite-wait config.

## Reproduction Assessment

Yes. Current main and the latest release reproduce the narrowed issue as a docs/help mismatch: safe restart can time out into a forced intent, while CLI docs/help emphasize draining without stating the bounded default.

## Solution Assessment

Yes for the narrowed request: a wording-only clarification is the narrowest maintainable fix unless maintainers want to change the default behavior or add a new CLI option.
