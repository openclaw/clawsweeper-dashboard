## Summary

Keep this issue open as a narrow CLI/docs accuracy task. The reporter correctly narrowed the original incident: bounded deferral and force-after-timeout are intended and configurable, but the shipped `--safe` help and lifecycle docs still imply that restart always waits until active work drains.

## What This Changes

Clarify the safe-restart CLI and documentation contract so operators know the default deferral is bounded and can force restart, while `gateway.reload.deferralTimeoutMs=0` waits indefinitely.

## Best Possible Solution

Preserve the existing restart semantics and update command help plus gateway/daemon lifecycle docs to state the bounded default, possible force-after-timeout, and `gateway.reload.deferralTimeoutMs=0` indefinite-wait control.
