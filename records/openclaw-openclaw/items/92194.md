## Label Justifications

- `P1`: The latest stable release can terminate active agent work after an operator invokes a command documented as safe.
- `impact:session-state`: The timeout path aborts embedded runs and marks active sessions as interrupted by gateway restart.

## Summary

This remains a valid, shipped lifecycle bug: `--safe` promises drain-before-restart, but the default five-minute deferral deliberately becomes a force intent that skips the final drain and aborts active runs. The policy choice between indefinite waiting, cancellation without restart, and explicit force-after-timeout needs lifecycle-owner judgment, so this should stay open for a focused design and fix.

## What This Changes

Make safe restart avoid unexpectedly forcing after the bounded deferral window, or explicitly expose and document that operator choice.

## Best Possible Solution

Give explicit safe restart a distinct non-forcing request policy: wait until idle or end without restarting at its requested deadline, and require a separately named operator action for force-after-timeout; then align CLI help, gateway/config docs, RPC intent, and lifecycle tests.

## Reproduction Assessment

Yes, at source level with high confidence. Current main deterministically converts the default safe deferral expiry into a force intent, and the reporter's v2026.6.5 journal sequence matches the tested drain-bypass path.

## Solution Assessment

The report identifies the right failure, but changing the shared timeout or only documenting the surprise is not the best complete fix. Explicit safe-restart semantics should be separated from automatic reload recovery and explicit force behavior.

## Risks / Open Questions

- Changing `gateway.reload.deferralTimeoutMs` globally could also alter automatic config-reload and channel-reload recovery, so explicit safe-restart intent should not be fixed by an undifferentiated shared-default change.
- Making safe restart wait indefinitely by default can leave an operator restart pending forever when task state is genuinely stuck; the product contract needs a visible cancellation or explicit force escape path.
