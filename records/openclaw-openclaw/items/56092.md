## Summary

Keep open. Current main still has the source-reproducible configured heartbeat-session precedence problem, and this PR is a plausible fix direction, but the branch still needs real behavior proof, a rebase, and a source-session canonicalization fix before merge.

## What This Changes

The PR reroutes system-event-triggered heartbeat runs to the configured heartbeat session, migrates/restores system events between session queues, tags wake-event sources, extends system-event options, and adds heartbeat session-routing tests.

## Best Possible Solution

Rebase onto current main and implement configured heartbeat-session precedence with canonical same-agent event migration/restore that preserves trust, delivery context, dedup, and current typed wake behavior.

## Reproduction Assessment

Yes. Source inspection shows exec completion can enqueue a system event under an originating session and the heartbeat runner still resolves forcedSessionKey before heartbeat.session on current main.

## Solution Assessment

No. The priority reorder is the right direction, but the proposed migration drains a raw forced session key before applying the canonical same-agent validation used for routing.

## Review Findings
