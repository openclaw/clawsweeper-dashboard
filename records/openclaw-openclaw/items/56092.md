## Summary

Keep this PR open. Current main still does not implement the intended heartbeat-session routing behavior, but the PR is not ready to merge because its migration path still drains the raw forced session key instead of using the same canonicalization and agent-scope checks as session resolution.

## What This Changes

The PR moves configured `heartbeat.session` resolution ahead of the forced wake session, adds cross-session system-event migration and skip-restore logic, tags several wake-event sources, and adds heartbeat session-routing tests.

## Best Possible Solution

Keep this PR as the implementation candidate for #41165, but require a focused update that canonicalizes and agent-scopes the source queue before migration, rebases onto current main without dropping newer heartbeat behavior, and keeps the regression tests centered on configured-session precedence plus safe event migration/restore.

## Reproduction Assessment

Yes. The current-main path is visible statically: exec `notifyOnExit` queues under the originating session and sends a scoped wake, while `resolveHeartbeatSession()` returns that forced session before evaluating configured `heartbeat.session`.

## Solution Assessment

No, not yet. The direction is the right narrow fix, but the current PR should not merge until the raw source-session drain is replaced with canonicalized, agent-scoped migration and the branch is updated against current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Canonicalize the source session before draining events:** `src/infra/heartbeat-runner.ts:463-468`
  - body: The migration path drains `params.forcedSessionKey?.trim()` directly. `resolveHeartbeatSession()` normalizes aliases and rejects cross-agent/subagent keys before routing, so a wake carrying an alias or mismatched agent key can drain the wrong raw queue and move or lose system events across session boundaries. Reuse the same canonicalization and agent-id check before any drain/re-enqueue.
  - confidence: 0.86
