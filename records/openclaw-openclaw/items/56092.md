## Summary

Keep open: the reported heartbeat/session precedence bug is still source-reproducible on current main, and this PR remains a plausible implementation candidate, but it is not merge-ready because the migration path still drains the raw forced session key and the contributor has not provided real after-fix behavior proof.

## What This Changes

The PR reorders heartbeat session resolution, adds cross-session system-event migration and skip-restore logic, tags wake-event sources, extends system-event options, and adds heartbeat session-routing tests.

## Best Possible Solution

Rebase or replace this PR with a narrow fix that lets configured heartbeat.session win while migrating or restoring only canonical same-agent source queues, then add focused regression coverage and real Telegram or terminal proof.

## Reproduction Assessment

Yes, source-level. Exec completion can enqueue and wake heartbeat with the originating session key, and current main resolves that forced key before the documented heartbeat.session configuration.

## Solution Assessment

No. The precedence reorder is the right direction, but the current migration path drains the raw forced session key before applying the same canonical same-agent validation used for routing.
