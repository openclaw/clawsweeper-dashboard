## Summary

Keep open. Current main still has the source-reproducible heartbeat/session precedence bug and this PR is the active implementation candidate, but the latest head still has a blocking cross-session migration issue and lacks required real behavior proof.

## What This Changes

The branch reorders heartbeat session resolution so configured `heartbeat.session` wins over forced wake sessions, migrates/restores system events across queues, tags wake-event sources, extends system-event options, and adds session-routing tests.

## Best Possible Solution

Rebase onto current main and implement configured heartbeat-session precedence with canonical same-agent event migration/restore that preserves trust, delivery context, dedup, and current typed wake behavior.

## Reproduction Assessment

Yes. Source inspection shows exec completion queues a system event in the originating session and the wake scheduler forwards that key, while current `resolveHeartbeatSession()` returns the forced key before evaluating `heartbeat.session`.

## Solution Assessment

No. The priority reorder is the right direction, but this head drains `params.forcedSessionKey` raw before applying the canonical same-agent validation used for session routing.
