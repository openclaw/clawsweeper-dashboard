## Summary

Keep open. Current main still has the reported heartbeat-session precedence bug, and this PR is the active implementation candidate, but the submitted patch is not merge-ready because its queue migration drains a raw forced session key, lacks after-fix real behavior proof, and predates newer heartbeat/system-event contracts.

## What This Changes

The PR reorders heartbeat session resolution, migrates/restores pending system events between sessions, tags wake-triggered system events, extends system-event queue metadata, and adds heartbeat session-routing tests.

## Best Possible Solution

Land a rebased narrow fix where configured heartbeat.session wins for system-event-triggered wakes, source queues are canonicalized and same-agent scoped before migration, system-event metadata is preserved, and redacted real behavior proof is attached.

## Reproduction Assessment

Yes, source-level. Current main lets exec notify-on-exit queue and wake heartbeat with the originating session key, then resolveHeartbeatSession accepts that forced key before the documented heartbeat.session setting.

## Solution Assessment

No, not as submitted. The precedence reorder is the right direction, but queue migration must canonicalize and reject mismatched source session keys before any drain, and the branch still needs real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Canonicalize source session before migrating queues:** `src/infra/heartbeat-runner.ts:463-468`
