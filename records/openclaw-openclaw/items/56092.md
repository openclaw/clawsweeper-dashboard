## Summary

Keep open. The PR targets a real source-reproducible heartbeat routing bug, but it should not merge as submitted because the migration source still uses a raw forced session key, after-fix real behavior proof is missing, and the branch must be reconciled with current main's wake and system-event contracts.

## What This Changes

The branch makes configured `heartbeat.session` take precedence over system-event forced sessions, migrates/restores pending system events between queues, tags wake sources, extends system-event options, and adds session-routing tests.

## Best Possible Solution

Rebase or replace this with a narrow fix where configured `heartbeat.session` wins, migrated source queues are canonicalized and same-agent scoped, trust/delivery metadata is preserved, and redacted real behavior proof is added.

## Reproduction Assessment

Yes, source-level. Current main lets exec completion enqueue and wake heartbeat with the originating session key, and `resolveHeartbeatSession()` can consume that forced key before the documented `heartbeat.session` configuration.

## Solution Assessment

No, not as submitted. The precedence reorder is the right direction, but queue migration must canonicalize and reject mismatched source session keys before draining, and the branch still needs real behavior proof.

## Review Findings

Overall correctness: patch is incorrect
