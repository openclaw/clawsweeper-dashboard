## Summary

Keep open. The PR targets a valid source-reproducible `heartbeat.session` precedence bug, but the current head still has an unsafe raw source-queue migration path and lacks required real behavior proof.

## What This Changes

The PR reorders heartbeat session resolution, migrates/restores system events between sessions, tags wake-triggered system events, extends system-event queue options, and adds heartbeat session-routing tests.

## Best Possible Solution

Land a rebased narrow fix where configured `heartbeat.session` wins, source queues are canonicalized and same-agent scoped before migration, system-event metadata is preserved, and redacted real behavior proof is attached.

## Reproduction Assessment

Yes, source-level. Current main lets exec completion enqueue and wake heartbeat with the originating session key, and `resolveHeartbeatSession()` can consume that forced key before the documented `heartbeat.session` configuration.

## Solution Assessment

No, not as submitted. The precedence reorder is the right direction, but queue migration must canonicalize and reject mismatched source session keys before draining, and the branch still needs real behavior proof.
