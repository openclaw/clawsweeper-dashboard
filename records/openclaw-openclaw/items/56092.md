## Summary

Keep this PR open for maintainer follow-up. The bug is source-reproducible and the precedence reorder is aimed at the right path, but the PR still drains a raw forced session key before canonical same-agent validation and lacks real after-fix runtime proof.

## What This Changes

The PR makes configured `heartbeat.session` take precedence over wake-originating sessions, migrates/restores system events across queues, tags wake-event sources with `contextKey`, extends system-event options, and adds heartbeat session-routing tests.

## Best Possible Solution

Rebase and implement configured heartbeat-session precedence with canonical same-agent event migration/restore that preserves delivery context, trust, deduplication, and skip-restore behavior, then prove it in a real heartbeat/Telegram setup.

## Reproduction Assessment

Yes. Source inspection shows an exec completion can enqueue a system event under the originating session and wake heartbeat with that session key, while current main resolves the forced key before the configured `heartbeat.session`.

## Solution Assessment

No. The precedence reorder is the right direction, but this PR’s migration path drains the raw forced session key before canonical same-agent validation, so it is not yet the safest maintainable fix.
