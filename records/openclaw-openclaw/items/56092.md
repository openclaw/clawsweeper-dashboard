## Summary

Keep open: current main still has the reported heartbeat routing problem, and this PR is useful but not mergeable as-is because its cross-session migration path can drain a raw, unscoped source queue.

## What This Changes

The PR prioritizes configured `heartbeat.session` over forced wake sessions, migrates system events between origin and resolved heartbeat queues with restore/dedup/trust handling, tags wake events, and adds session-routing tests.

## Best Possible Solution

Rebase or replace the PR with a narrow fix where configured `heartbeat.session` wins and any migration uses canonical same-agent source keys while preserving delivery context, trust, dedup identity, and skip-restore coverage.

## Reproduction Assessment

Yes. Source inspection shows `notifyOnExit` queues into the originating session and wakes heartbeat with that key, while current main resolves `forcedSessionKey` before `heartbeat.session` and peeks only the resolved queue.

## Solution Assessment

No. The priority reorder is the right direction, but moving prompt-visible system events must be sourced from a canonical same-agent session key before this patch is safe to merge.
