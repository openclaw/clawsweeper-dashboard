## Summary

Keep open. This PR targets a real source-reproducible heartbeat/session bug, but the branch still drains a raw forced session key before canonical same-agent validation, lacks contributor real behavior proof, and needs maintainer review rather than cleanup closure.

## What This Changes

The PR makes configured `heartbeat.session` take precedence for system-event-triggered heartbeats, migrates/restores system events between session queues, tags wake-event sources, extends system-event options, and adds session-routing tests.

## Best Possible Solution

Rebase onto current main and implement configured heartbeat-session precedence with canonical same-agent event migration/restore that preserves delivery context, trust, deduplication, skip-restore behavior, and current subagent/isolated-session guards, then prove it in a real heartbeat/Telegram setup.

## Reproduction Assessment

Yes, by source inspection. Current main enqueues exec completion events under the originating session, forwards that session through heartbeat wake, and resolves the forced key before `heartbeat.session`.

## Solution Assessment

No. The precedence reorder is the right direction, but the migration path must canonicalize and same-agent-check the source session before draining queues, then be rebased and proven in a real setup.
