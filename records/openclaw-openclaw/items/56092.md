## Summary

Keep open: the reported bug is source-reproducible and the PR takes the right direction, but the current PR head still migrates prompt-visible system events from a raw forced session key before applying the canonicalization and same-agent checks used by session routing.

## What This Changes

The PR reorders heartbeat session resolution, migrates and restores system-event queues across sessions, tags wake events with context keys, extends system-event metadata, and adds heartbeat session-routing tests.

## Best Possible Solution

Keep the PR open and rework the migration path so configured heartbeat sessions win while any queue migration uses canonical same-agent source keys and preserves current wake-source, subagent, trust, dedup, and skip-restore behavior.

## Reproduction Assessment

Yes. Source inspection shows exec completion queues a system event in the originating session, requests a scoped heartbeat with that same key, and current main resolves forcedSessionKey before heartbeat.session.

## Solution Assessment

No, not yet. The priority reorder is the right direction, but the migration source must be canonicalized and same-agent checked before draining prompt-visible queues.
