## Summary

Keep open: the UX idea is plausible, but this branch is not mergeable as-is because it targets the old selector helper, bypasses current catalog-label contracts, lacks real browser proof, and overlaps active model-label product work.

## What This Changes

The PR changes the Control UI chat model selector helper to group catalog models under provider optgroups with hard-coded provider display labels and per-group sorting.

## Best Possible Solution

Rebuild the feature on `ui/src/ui/chat-model-select-state.ts` and `ui/src/ui/chat/session-controls.ts`, preserve catalog display disambiguation, derive provider labels from provider/catalog metadata or a shared formatter, and add focused tests plus redacted browser proof.

## Reproduction Assessment

Not applicable: this is a feature PR, not a bug report. Source inspection shows current main still renders the chat model selector from a flat `ChatModelSelectState.options` list.

## Solution Assessment

No. Provider grouping may be a reasonable UX direction, but this implementation should be rebuilt on the current selector split and preserve catalog-label contracts instead of hard-coding provider labels in the old helper.
