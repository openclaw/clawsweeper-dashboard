## Summary

Keep open: the feature is still useful, but the branch is dirty and applies grouping to `app-render.helpers.ts` while current `main` now builds model selector state in `chat-model-select-state.ts` and renders it in `chat/session-controls.ts`; the patch also bypasses catalog label disambiguation and lacks the required changelog entry.

## What This Changes

The PR changes the Control UI chat model selector from a flat option list into provider-grouped optgroups with hard-coded provider display labels.

## Best Possible Solution

Implement provider grouping in `chat-model-select-state.ts` and render provider optgroups from `chat/session-controls.ts`, preserving catalog-derived display labels, provider/model option values, focused regression coverage, and an `Unreleased` changelog entry.

## Reproduction Assessment

Not applicable: this is a feature PR rather than a bug report. Source inspection gives a high-confidence current-main path showing the active chat model picker still uses a flat option array.

## Solution Assessment

No. The requested grouping is reasonable, but the current patch should be rebased into the active selector state/render split and reuse catalog/provider metadata instead of editing the old helper with hard-coded labels.
