## Summary

Keep open for maintainer review: the requested grouped model picker is not on current main, but this PR is not merge-ready because it targets an obsolete helper path, conflicts with the active selector split, drops current catalog label behavior, and still lacks after-fix real behavior proof.

## What This Changes

The PR changes the Control UI chat model selector in `ui/src/ui/app-render.helpers.ts` to build provider groups with hard-coded friendly labels and render grouped `<optgroup>` options.

## Best Possible Solution

Decide the model-picker grouping and label policy first, then implement it in `ui/src/ui/chat-model-select-state.ts` and `ui/src/ui/chat/session-controls.ts` while preserving catalog metadata and duplicate-label disambiguation.

## Reproduction Assessment

Not applicable: this is a feature PR, not a bug report. Source inspection of current main shows the model selector still consumes a flat options array, while the PR provides no runtime proof of the grouped UI.

## Solution Assessment

No. Grouping may be reasonable, but this implementation should move to the active selector state/render split, preserve the catalog-backed label contract, and wait for product direction on grouped labels versus exact refs.
