## Summary

Keep open: the provider grouping is a plausible UI feature, but the branch targets the pre-split selector helper, loses existing catalog-label disambiguation, and has no real browser proof attached.

## What This Changes

The PR changes the Control UI chat model selector helper to render provider-grouped optgroups with hard-coded friendly provider labels and sorted model options.

## Best Possible Solution

Implement provider grouping in `ui/src/ui/chat-model-select-state.ts` and `ui/src/ui/chat/session-controls.ts`, reuse catalog display lookup labels, derive provider names from metadata or a shared formatter, and add focused state/render coverage.

## Reproduction Assessment

Not applicable: this is a feature PR, not a bug report. Source inspection shows current main still builds a flat model option array and renders flat `<option>` children in the active selector path.

## Solution Assessment

No. Grouping by provider is a reasonable direction, but this patch should target the current selector state/render split and preserve existing catalog-label disambiguation instead of hard-coding provider labels in a stale helper.
