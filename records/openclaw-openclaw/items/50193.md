## Summary

Keep this PR open. Current main still renders the Control UI chat model picker as a flat list, so the requested provider grouping is not implemented, but the branch is stale against the current split model-select owner and the proposed diff would regress catalog label disambiguation.

## What This Changes

The PR changes the Control UI chat model selector from flat `<option>` entries to provider `<optgroup>` sections in `ui/src/ui/app-render.helpers.ts`, adds a hard-coded provider label table, and sorts models within each group.

## Best Possible Solution

Provider grouping should be represented in the current `chat-model-select-state.ts` model-select state, rendered by `chat/session-controls.ts` with `<optgroup>`, and built on the existing catalog display lookup so selected and duplicate model labels remain unambiguous while submitted `sessions.patch` values stay unchanged.

## Reproduction Assessment

Yes. A high-confidence read-only reproduction path exists: current main shows `options: ChatModelSelectOption[]` and renders those entries directly as `<option>`, while existing duplicate-name tests demonstrate the label cases that the PR's `entry.name` path would collapse.

## Solution Assessment

No. The proposed direction is useful, but this patch is no longer the best implementation because it targets the old owner file and bypasses the current catalog display helper; a narrow rework in the current state/render split is the safer solution.
