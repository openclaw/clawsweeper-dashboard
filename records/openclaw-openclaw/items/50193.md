## Summary

Keep open: the requested UI grouping is useful, but this patch is not merge-ready because current main now owns the active selector in a separate state/render split, the PR drops catalog label disambiguation, and contributor real behavior proof is missing.

## What This Changes

The PR changes the Control UI chat model selector from flat options to provider-grouped optgroups with a hard-coded provider label map in `ui/src/ui/app-render.helpers.ts`.

## Best Possible Solution

Rebase the feature into `ui/src/ui/chat-model-select-state.ts` and `ui/src/ui/chat/session-controls.ts`, preserve `buildCatalogDisplayLookup` labels, derive provider labels from metadata or shared formatting, and add focused state/render coverage.

## Reproduction Assessment

Not applicable: this is a feature PR, not a bug report. Source inspection gives high confidence that current main still renders the active chat model picker as a flat option list.

## Solution Assessment

No. Grouping by provider is a reasonable direction, but this patch should move into the current selector state/render split and reuse catalog display helpers instead of hard-coded provider labels and name-only option text.
