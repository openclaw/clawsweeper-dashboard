## Summary

Keep open: the requested provider grouping is not implemented on current main, but this branch is not merge-ready because it targets the old helper path, drops catalog label disambiguation, uses a hard-coded provider label table, and lacks real behavior proof.

## What This Changes

This PR changes the Control UI chat model selector helper to build provider-grouped optgroups with hard-coded friendly provider labels and sorted model options.

## Best Possible Solution

Implement provider grouping in `ui/src/ui/chat-model-select-state.ts` and `ui/src/ui/chat/session-controls.ts`, reuse catalog display lookup labels, derive provider names from metadata or a shared formatter, and add focused state/render coverage.

## Reproduction Assessment

Not applicable: this is a feature PR, not a bug report. Source inspection gives high confidence that current main still renders the active model picker as a flat option list.

## Solution Assessment

No. Grouping by provider is a reasonable direction, but this patch should move into the current selector state/render split and preserve the existing catalog-label disambiguation instead of hard-coding provider labels in the stale helper.
