## Summary

Keep open: the feature is still useful, but the patch is stale against the current Control UI model selector split and would regress existing catalog label behavior before it is mergeable.

## What This Changes

The PR changes the Control UI chat model selector from a flat option list into provider-grouped optgroups with hard-coded provider display labels.

## Best Possible Solution

Add provider grouping in `chat-model-select-state.ts` and render it from `chat/session-controls.ts`, while preserving catalog-derived labels, provider/model values, focused tests, and a changelog entry.

## Reproduction Assessment

## Solution Assessment

No. The requested grouping is reasonable, but the patch should be rebased into the current selector state/render split and reuse the catalog display lookup instead of inlining model and provider labels in the old helper.
