Implement grouping in `ui/src/ui/chat-model-select-state.ts` and `ui/src/ui/chat/session-controls.ts` using the existing catalog display lookup, with tests for aliases, duplicate names, and provider grouping after maintainer direction on provider group labels.

Not applicable as a user bug reproduction because this is a feature PR. The blocking review findings are source-reproducible by comparing the PR head with current main's active model selector and catalog label tests.

## Solution Assessment

No. The maintainable path is to add grouping to the active picker state/render code and reuse catalog label resolution, rather than reviving a stale helper and adding a separate provider-label policy.
