## Summary

Keep open. Current main still exposes `before_model_resolve` as an override-only hook and has no typed, merged, or propagated blocked-model outcome, so the reported budget-guard use case remains a real core plugin API gap rather than ClawHub-only plugin work.

## What This Changes

Add `block` and `blockReason` to `PluginHookBeforeModelResolveResult` so plugins can stop a model call before provider dispatch.

## Best Possible Solution

Add an additive core plugin API change: extend the public model-resolve result with `block?: boolean` and `blockReason?: string`, merge it with terminal sticky-block semantics matching `before_tool_call`, propagate a blocked model-selection outcome before `resolveModelAsync`, surface the reason to the user, and update SDK baseline, docs, changelog, and focused hook/model-selection tests.

## Reproduction Assessment

Yes. A focused source-level reproduction is high-confidence: a `before_model_resolve` handler returning `{ block: true, blockReason: "budget exhausted" }` would have the field omitted by the current merger, and a thrown budget error is caught fail-open in `resolveHookModelSelection`.

## Solution Assessment

Yes, the requested direction is the narrowest maintainable fix if scoped to the existing plugin hook contract. It should reuse the established `before_tool_call` terminal-block semantics rather than inventing a separate budget-specific core path.
