## Summary

Keep open. Current main still exposes `before_model_resolve` as an override-only plugin hook, so the budget-guard use case remains a real public plugin API gap rather than a ClawHub-only plugin request.

## What This Changes

Add `block` and `blockReason` support to `PluginHookBeforeModelResolveResult` so plugins can abort model calls before provider/model resolution.

## Best Possible Solution

Extend the core plugin hook contract with model-resolve block semantics matching `before_tool_call`, then propagate a blocked model-selection outcome before provider resolution.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a `before_model_resolve` handler returning `{ block: true, blockReason: "budget exhausted" }` has no typed field, is dropped by the override-only merger, and no blocked branch exists before `resolveModelAsync`.

## Solution Assessment

Yes. Reusing the existing `before_tool_call` terminal-block pattern is the narrowest maintainable direction for this plugin API gap; a budget-specific core path would be less general.
