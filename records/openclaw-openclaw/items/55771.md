## Summary

Close. Current main implements the central need through the dedicated `before_agent_run` lifecycle gate, which lets plugins block before provider/model submission; `before_model_resolve` remains selection-only, but budget/security plugins now have a supported pre-model block path.

## What This Changes

Add a supported way for plugins to block model calls with a user-visible reason, originally proposed as `block` and `blockReason` on `PluginHookBeforeModelResolveResult`.

## Best Possible Solution

Keep `before_model_resolve` focused on provider/model selection and use the documented `before_agent_run` gate for budget, security, and other policy decisions that must block model submission.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection confirm current main now has a supported pre-model blocking hook for the reported budget-guard use case.

## Solution Assessment

Yes. A dedicated lifecycle gate is the cleaner solution because it runs after prompt construction/model resolution, blocks before provider submission, and redacts blocked content instead of overloading the model-selection hook.
