## Summary

Keep open. Current main still has the provider-inventory visibility path, while this PR addresses the real #75166 leak by tagging image/video inventory output and suppressing its text before shared-surface delivery and tool TTS; I did not find a blocking correctness or supply-chain issue in the proposed diff.

## What This Changes

The PR adds an internal provider-inventory marker for image/video tool output, suppresses that tagged text on non-direct dispatch before TTS while preserving media and direct diagnostics, and updates tests, docs, and changelog.

## Best Possible Solution

Land one canonical fix that treats image/video provider inventory as internal on shared chat surfaces before text delivery and TTS, while preserving generated media delivery and direct-chat diagnostics; this PR is a credible merge vehicle for that policy if CI stays green.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by returning an `image_generate` or `video_generate` list result with `details.providers` on current main, where the compact-output bypass emits tool text and Discord-style `ChatType: "channel"` delivery remains eligible.

## Solution Assessment

Yes. The PR's typed payload marker plus dispatcher filtering before TTS is narrower and more maintainable than deleting provider inventory entirely, because it keeps direct diagnostics and media behavior intact while closing the shared-surface leak.
