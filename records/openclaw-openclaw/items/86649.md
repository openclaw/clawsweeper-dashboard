## What This Changes

The PR adds a Claude stream-json assistant-record fallback that emits incremental deltas only when `message.stop_reason` is explicitly `null`, plus parser regression tests.

## Best Possible Solution

Land the parser fallback only after redacted live proof confirms real Claude CLI assistant partial records with `stop_reason: null` produce progressive deltas without duplicate finals across the Gateway/UI path.

## Reproduction Assessment

Yes, source-reproducible: current main only relays `stream_event` text deltas, while the related issue gives concrete WebChat/TUI steps and logs showing raw Claude CLI JSONL lines with no visible partials. I did not run a live Claude CLI session in this read-only review.

## Solution Assessment

Yes, if the upstream shape is confirmed: the shared CLI JSONL parser is the narrow owner boundary used by both live and spawned CLI paths. The safer merge state is to pair this code with redacted live/raw-shape proof rather than relying on synthetic fixtures alone.
