## Summary

Keep this PR open, but it is not merge-ready: the patch treats Claude `assistant` records as accumulated partial deltas even though the Claude Agent SDK streaming contract says partials are `stream_event` records and `AssistantMessage` objects are complete messages after a response. ([code.claude.com](https://code.claude.com/docs/en/agent-sdk/streaming-output?utm_source=openai))

## What This Changes

The PR adds a Claude CLI assistant-message parser that treats `type: "assistant"` JSONL records as accumulated streaming text and adds parser tests for deltas, deduplication, and filtering.

## Best Possible Solution

Diagnose the live raw Claude CLI stream for the linked bug, keep `stream_event`/`text_delta` as the streaming source unless a documented accumulated assistant-partial shape is proven, and add regression coverage for multi-message/tool turns.

## Reproduction Assessment

Yes for the review finding: official docs plus source inspection show `assistant` records are complete messages, and the new parser would emit them as deltas. The linked Windows live symptom still needs redacted raw JSONL to reproduce its root cause.

## Solution Assessment

No. The current PR fixes the symptom by treating `assistant` as partials, but the safer maintainable fix is to prove why live `stream_event` deltas are absent or unrelayed and preserve Claude's documented message boundary.
