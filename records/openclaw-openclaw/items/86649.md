## Label Justifications

- `P1`: The PR targets a live Claude CLI agent streaming regression that can make WebChat and TUI turns appear hung until final output arrives.
- `merge-risk: 🚨 compatibility`: The diff changes behavior for the custom CLI backend dialect `claude-stream-json`, not only the bundled Claude CLI provider.
- `merge-risk: 🚨 message-delivery`: The diff changes which JSONL assistant records are relayed as user-visible streaming deltas, where duplicate or false partial delivery is the key failure mode.

## Summary

Keep this PR open for maintainer review: the parser patch is narrowly placed and plausibly fixes the linked streaming gap, but the current proof is still a synthetic production-parser exercise rather than live Claude CLI/Gateway/WebChat/TUI behavior, and the fallback changes the public `claude-stream-json` dialect semantics.

## What This Changes

The PR adds a Claude CLI JSONL fallback that converts assistant records with explicit `message.stop_reason: null` into streaming deltas and adds parser tests for partial, final, ambiguous, and mixed-content records.

## Best Possible Solution

Land the parser fallback only after redacted live proof confirms real Claude CLI assistant partial records produce progressive Gateway/UI deltas without duplicate final text, or deliberately narrow the fallback to bundled Claude CLI if maintainers do not want custom dialect backends included.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. The parser-level gap is source-reproducible because current main ignores assistant records, but the linked issue has a maintainer non-repro comment and still needs redacted raw Claude stream-json proof.

## Solution Assessment

Yes, if the raw Claude CLI shape is confirmed: the shared CLI JSONL parser is the narrow owner boundary used by spawned and live Claude CLI paths. The safer merge state is to pair this code with redacted live/raw-shape proof or narrow the fallback to bundled Claude CLI.
