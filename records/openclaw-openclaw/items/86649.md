## Summary

Keep open: the parser change is focused and useful, but the branch still lacks real Claude CLI Gateway/WebChat/TUI proof and changes shared Claude JSONL delivery semantics that maintainers should approve before merge.

## What This Changes

The PR adds Claude CLI assistant partial-message parsing to the CLI JSONL streaming parser and tests that explicit `stop_reason: null` assistant records become incremental deltas while complete or ambiguous records are skipped.

## Best Possible Solution

Capture a redacted live Claude CLI stream-json sample and WebChat or TUI output, then land the narrow parser rule only if top-level assistant records are the observed partial shape and the shared-dialect compatibility impact is intentional.

## Reproduction Assessment

No high-confidence live reproduction of the reported WebChat/TUI buffering is available in this review. The parser-level fixture is source-reproducible, but linked current-main and reported-package repro attempts saw pre-final deltas, so raw failing stream data is still missing.

## Solution Assessment

Unclear. The guarded fallback is plausible only if real Claude CLI output contains top-level assistant records with explicit `stop_reason: null`; the safer path is to prove that raw shape before expanding shared parser semantics.
