## Summary

Keep this PR open, but not merge-ready: the patch handles Claude `assistant` JSONL records as accumulated partial text even though the upstream streaming contract identifies them as complete assistant messages, and the supplied proof is still a simulated parser check rather than live Claude CLI output.

## What This Changes

The PR adds a Claude CLI assistant-message fallback parser in `src/agents/cli-output.ts` and parser tests for assistant-message deltas, deduplication, and filtering.

## Best Possible Solution

Keep the parser aligned to Claude's `stream_event` delta contract, or add a live-output-backed fallback for the exact raw JSONL shape that Claude CLI actually emits and cover multi-message/tool-turn cases.

## Reproduction Assessment

Unclear: the linked issue has strong user logs showing zero gateway deltas during Claude CLI turns, but this review did not get raw live Claude CLI JSONL proving that top-level `assistant` messages are accumulated partials rather than complete messages.

## Solution Assessment

No: the current PR is not the best fix because it relies on complete `AssistantMessage` records as if they were partial deltas; the safer path is contract-backed stream-event parsing plus live raw-output proof for any fallback.
