## Summary

Keep open: the parser change is narrow and plausible, but it is not merge-ready until live Claude CLI/Gateway proof confirms that assistant records with explicit `stop_reason: null` are the real partial-output shape, because current official Claude docs describe partial output as `StreamEvent` raw events rather than assistant records.

## What This Changes

Adds a Claude stream-json parser path that treats assistant records with explicit `message.stop_reason: null` as cumulative partials and emits incremental assistant deltas, with regression tests.

## Best Possible Solution

Require redacted live Claude CLI stream-json or Gateway/WebChat/TUI proof for the exact event sequence; if the raw shape matches, land this parser-level fix, otherwise adjust the parser to the documented StreamEvent contract.
