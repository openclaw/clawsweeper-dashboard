## Label Justifications

- `P1`: The PR targets a broken Claude CLI streaming workflow where WebChat/TUI users can lose in-turn progress visibility for long agent turns.
- `merge-risk: 🚨 compatibility`: The parser changes how existing Claude stream-json compatible backends interpret `assistant` records during upgrades.
- `merge-risk: 🚨 message-delivery`: The changed parser can emit additional assistant deltas from records that were previously ignored, affecting WebChat/TUI delivery behavior.

## Summary

Keep open, but not merge-ready: the patch is narrow and relevant, but it changes Claude JSONL delivery semantics without live Claude CLI/WebChat/TUI proof, and the pinned Claude Agent SDK types still describe partial assistant output as `stream_event` records.

## What This Changes

The PR adds a Claude-compatible JSONL parser fallback that turns `assistant` records with absent or null `stop_reason` into streaming deltas, with parser regression tests.

## Best Possible Solution

Land this fallback only after redacted raw Claude CLI stream-json or live WebChat/TUI proof confirms `assistant` records with absent or null `stop_reason` are true partial snapshots in the affected setup.

## Reproduction Assessment

No high-confidence current-main reproduction is available. The source path shows current main ignores `assistant` records, but related live current-main and Windows/npm checks saw pre-final deltas, and this PR lacks raw failing Claude JSONL or live after-fix proof.

## Solution Assessment

Unclear. The patch is bounded, but the safest solution needs live or dependency proof before treating `assistant` records as partial snapshots rather than complete-message records.
