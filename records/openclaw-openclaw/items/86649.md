## Summary

Keep this PR open: the parser change is narrow and I found no blocking code defect, but the merge still needs real Claude CLI/Gateway behavior proof because the current evidence only exercises synthetic JSONL records.

## What This Changes

Adds parsing and tests so Claude stream-json `assistant` records with explicit `message.stop_reason: null` are emitted as incremental `CliStreamingDelta`s after the existing `stream_event` text-delta path.

## Best Possible Solution

Land the narrow parser support only after redacted live Claude CLI stream-json and WebChat/TUI or Gateway proof confirm that explicit-null assistant partial records are the missing leg and do not duplicate final text.

## Reproduction Assessment

Source-reproducible for the proposed parser shape: current main ignores assistant records as text deltas because only `stream_event.content_block_delta` is parsed. I do not have a high-confidence live reproduction of the full Claude CLI/WebChat or TUI issue from this read-only review.

## Solution Assessment

This looks like an acceptable narrow fix if real Claude CLI output contains explicit-null assistant partial records and no matching stream_event deltas. It is not proven as the best fix yet because the linked issue's requested raw stream-json and live user-path proof are still missing.
