## What This Changes

Request to add an optional `CliBackendPlugin.estimateUsage` SDK hook so text-output CLI backends can supply heuristic usage counts when stdout has no structured usage data.

## Best Possible Solution

Settle the usage-estimate API semantics, then land or replace the linked implementation with SDK exports, runner wiring, docs/baseline coverage, session/status estimate labeling, and focused tests.

## Reproduction Assessment

Yes for the source-level gap: current main lacks the requested hook and text-output parsing returns no usage. No live current-main Antigravity reproduction is available because the consuming backend work is still open.

## Solution Assessment

Unclear until maintainers decide the public SDK contract. A backend-owned hook fits the plugin boundary, but the best implementation must preserve estimated-vs-exact semantics through SDK, session, status, and UI consumers.
