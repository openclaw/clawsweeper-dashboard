## Summary

Keep open: current main and the latest release still have no CLI-backend usage-estimation hook, and the linked implementation PR is open, draft, and waiting on maintainer API/UX approval rather than merged.

## What This Changes

The issue requests an optional `CliBackendPlugin.estimateUsage` SDK hook so text-output CLI backends can provide estimated token usage when stdout has no structured usage data.

## Best Possible Solution

Approve a narrow estimated-usage SDK contract with explicit estimated/exact semantics, land `https://github.com/openclaw/openclaw/pull/91477` or an equivalent implementation, then add backend-specific estimators in separate plugin PRs.

## Reproduction Assessment

Yes, source-level: current main’s text-output parser returns no usage and the runner has no backend estimation callback. There is not yet a live current-main Antigravity repro because that backend remains in an open PR.

## Solution Assessment

Unclear until maintainers decide the public SDK contract. A backend-owned hook is the likely right layer, but the final design needs explicit estimated/exact usage semantics and focused SDK, runner, docs, and regression coverage.
