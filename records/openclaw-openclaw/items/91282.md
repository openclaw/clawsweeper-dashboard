## Summary

Keep open: current main and v2026.6.8 still do not provide a CLI-backend usage-estimation hook, and the linked implementation PR is open, draft, and conflicting while waiting on maintainer API/UX approval.

## What This Changes

Request to add an optional `CliBackendPlugin.estimateUsage` SDK hook so text-output CLI backends can supply heuristic token usage when stdout has no structured usage fields.

## Best Possible Solution

Approve the estimated-usage SDK contract, land https://github.com/openclaw/openclaw/pull/91477 or an equivalent implementation with durable estimated/exact semantics, then add backend-specific estimators in separate plugin PRs.

## Reproduction Assessment

Yes, source-level: current main’s text-output parser returns no usage and the runner has no backend estimation callback. A live current-main Antigravity repro is not available because that backend remains in an open PR.

## Solution Assessment

Unclear until maintainers decide the public SDK contract. A backend-owned hook is the likely right layer, but the final design needs explicit estimated/exact usage semantics plus focused SDK, runner, docs, and regression coverage.
