## Summary

Keep open: current main still has no CLI backend usage-estimation hook, and the linked implementation PR is open, draft, conflicting, and still waiting on the public API/product decision.

## What This Changes

Add an optional `CliBackendPlugin.estimateUsage` hook so text-output CLI backends can provide heuristic token usage for session/TUI usage displays.

## Best Possible Solution

Keep this as the canonical API decision issue, then either land a repaired https://github.com/openclaw/openclaw/pull/91477 with durable estimate semantics and proof, or close both if maintainers reject the hook.

## Reproduction Assessment

Yes for the source-level gap: current main has no estimator hook, text-mode CLI parsing returns no usage, and the runner has no estimation path. A live current-main reproduction of the Antigravity UX is not available because the consuming backend is still an open PR.

## Solution Assessment

Unclear until maintainers approve the public SDK and UI semantics. A backend-owned hook fits the plugin boundary, but the best solution must preserve estimated-vs-exact usage through downstream session/status/cost consumers.
