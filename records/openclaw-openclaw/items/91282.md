## Summary

Keep open: current main and v2026.6.6 still lack a backend-owned usage-estimation hook, and the linked open implementation PR is the right place to review the API semantics before this issue can close.

## What This Changes

Request to add an optional `CliBackendPlugin.estimateUsage` hook so text-output CLI backends can report estimated token usage.

## Best Possible Solution

Decide the public SDK semantics in the linked implementation PR; if approved, land a narrow API, runner, docs, API-baseline, and test change with an explicit estimated-usage marker, then add backend-specific opt-ins separately.

## Reproduction Assessment

Yes for source-level reproduction: current main has no usage-estimation hook, text-mode parsing returns no usage, and the runner has no estimator callback. No high-confidence live TUI reproduction is available on current main because the Antigravity backend that would expose the symptom is still an open PR.

## Solution Assessment

Unclear until maintainers decide the public SDK contract. The backend-owned hook with an explicit estimated marker is plausible and aligned with existing backend hooks, but the best final shape should be settled in https://github.com/openclaw/openclaw/pull/91477.
