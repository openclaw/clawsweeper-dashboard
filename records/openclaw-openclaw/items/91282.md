## What This Changes

Request to add an optional `CliBackendPlugin.estimateUsage` hook so text-output CLI backends can provide heuristic token usage for session/TUI usage displays.

## Best Possible Solution

Approve the heuristic-usage API semantics, then land the linked PR or a successor that carries the estimate marker through SDK exports, docs, session/status/TUI consumers, and focused tests before any backend opt-in ships.

## Reproduction Assessment

Yes for source-level behavior: current main has no estimator hook, and `outputMode: "text"` parsing returns no usage. No high-confidence live TUI reproduction is available on current main because the Antigravity CLI backend work remains open.

## Solution Assessment

Unclear until maintainers decide the public API/UX contract. A backend-owned hook fits the plugin boundary, but the best implementation must preserve estimated-versus-exact semantics through downstream usage displays and public SDK surfaces.
