## Summary

Keep open: current main still has no plugin SDK or CLI runner hook for backend-owned usage estimates, while the request is a concrete public plugin API extension that fits the project direction but needs maintainer API approval before implementation.

## What This Changes

Add an optional `CliBackendPlugin.estimateUsage` hook so text-output CLI backend plugins can supply backend-owned token estimates when stdout has no usage payload.

## Best Possible Solution

Approve a narrow SDK-owned usage-estimation contract if maintainers accept heuristic usage semantics, then wire it through CLI backend resolution and runner execution with focused docs, API-baseline coverage, and parser/runner tests.

## Reproduction Assessment

No high-confidence runtime reproduction on current main because the dependent Antigravity CLI backend is still an open PR. Source inspection does show the current text-output CLI path cannot attach usage and the plugin SDK has no hook to supply it.

## Solution Assessment

Unclear until maintainers decide the public SDK contract. The proposed backend-owned hook is plausible and aligns with existing provider-owned CLI backend hooks, but the final design should avoid presenting heuristic estimates as exact provider billing data.
