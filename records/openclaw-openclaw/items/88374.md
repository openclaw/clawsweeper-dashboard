## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.78

Full review comments:

- **[P2] Keep persisted custom node ids signed:** `src/node-host/runner.ts:272`
  - body: `runNodeHost` persists a CLI override into node.json and later reuses `config.nodeId` as `instanceId`, but this line signs the id only when the current invocation passed `--node-id`. On a later foreground `openclaw node run` without the flag, the gateway sees a v3 device-auth payload, does not trust `client.instanceId`, and the node can fall back to the raw device id again. Please either sign user-configured node ids after restart or stop persisting/claiming that id without an explicit source marker.
  - confidence: 0.82
