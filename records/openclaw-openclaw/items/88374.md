## Repair Work Prompt

On OpenClaw PR #88374, fix the owner-mismatch metadata bug in the signed custom node-id path. When a signed `client.instanceId` is already paired to another `ownerDeviceId`, `reconcileNodePairingOnConnect` returns `registrationNodeId` as the authenticated device id, but `src/gateway/server/ws-connection/message-handler.ts` later updates paired metadata for both `nodeSession.nodeId` and raw `client.instanceId`. Change that post-register metadata update so it only writes to the registered/adopted node identity, or explicitly verifies the paired record owner before updating a second raw instance id. Add a regression test that sets up an existing paired custom node owned by `victim-device`, connects a second signed device using the same custom instance id, and verifies the victim paired node's `lastConnectedAtMs` is not refreshed by the second device. Do not broaden the device-auth protocol, do not change docs or CHANGELOG, and keep the v3 fallback behavior intact.
