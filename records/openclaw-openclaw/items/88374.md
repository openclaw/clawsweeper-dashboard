## What This Changes

The PR signs explicit node-host instance IDs, resolves a canonical gateway node identity from that signed value, and carries it through node pairing, catalog/listing, pending work, events, invoke results, and node-host restart state.

## Best Possible Solution

Land the broad identity fix after gateway/security owners accept the signed-instance trust boundary and legacy pairing adoption semantics, keeping the compatibility defaults and regression coverage in this PR.
