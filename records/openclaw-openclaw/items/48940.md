Full review comments:

- **[P1] Store ACP runtime state in SQLite:** `src/acp/store/store.ts:111-112`
  - body: This persists ACP sessions, runs, events, checkpoints, leases, and idempotency in a JSON sidecar. OpenClaw-owned runtime state belongs in SQLite with clear migration or doctor ownership, otherwise upgrades and recovery can drift from the canonical state store.
  - confidence: 0.95
- **[P1] Return the existing projection promise:** `src/auto-reply/reply/dispatch-acp-replay.ts:456`
  - body: `ensureProjection` calls `existing.start()`, but `start()` launches another polling loop each time. Re-ensuring the same target can run parallel projection loops that duplicate sends or race checkpoint writes.
  - confidence: 0.92
- **[P1] Await reconnect reconciliation before worker frames:** `src/gateway/server/ws-connection/message-handler.ts:1024-1038`
  - body: The connect handler starts ACP lease reconciliation in the background, so worker events can arrive while the lease is still suspect and be rejected. That can leave a completed node run stuck in gateway recovery.
  - confidence: 0.9
- **[P1] Gate ACP node commands explicitly:** `src/gateway/node-command-policy.ts:56`
  - body: Adding ACP session and turn commands to `SYSTEM_COMMANDS` makes them part of the default desktop node allowlist. This expands a command/control surface without an ACP-specific authorization decision.
  - confidence: 0.9
- **[P1] Reject cross-session run ID reuse:** `src/acp/store/store.ts:532-534`
  - body: `startRun` returns any existing `store.runs[runId]` without checking the session and request. A run-id collision can alias two sessions and then strand or corrupt delivery bookkeeping.
  - confidence: 0.88
- **[P1] Propagate ACP event transport failures:** `src/node-host/invoke.ts:425-426`
  - body: ACP worker events are sent through the existing best-effort `sendNodeEvent` helper, which swallows gateway request failures. A node can settle locally while the gateway never records the terminal, leaving recovery and replay inconsistent.
  - confidence: 0.86
- **[P1] Stop projection runner on early turn-start failure:** `src/auto-reply/reply/dispatch-acp.ts:395-396`
  - body: When `runTurn` fails before the durable run exists, the code rethrows without stopping the projection runner started with `waitForRunStart`, leaving a background poller waiting for a run that will never appear.
  - confidence: 0.84
