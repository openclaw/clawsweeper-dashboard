## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Use shared SQLite for ACP runtime state:** `src/acp/store/store.ts:112`
  - body: This new store is the durable source of truth for ACP sessions, runs, events, leases, delivery targets, and checkpoints, but it lives at `state/acp/gateway-node-runtime-store.json`. OpenClaw runtime state is supposed to live in the shared SQLite state DB, so landing this sidecar would create a second migration and recovery path for core session state.
  - confidence: 0.93
- **[P1] Reject cross-session run id collisions:** `src/acp/store/store.ts:532-535`
  - body: `startRun` returns an existing `store.runs[runId]` before checking `sessionKey` or `requestId`. If two sessions produce the same run id, the second session will reuse the first run record and later worker events or delivery targets validate against the wrong session.
  - confidence: 0.91
- **[P1] Do not restart an active projection runner:** `src/auto-reply/reply/dispatch-acp-replay.ts:455-456`
  - body: When a projection for the same target is already active, `ensureProjection` calls `existing.start()`, but the runner start path launches another polling loop. A second ensure on the same target can therefore race checkpoint updates or duplicate outward sends instead of deduplicating active work.
  - confidence: 0.9
- **[P2] Stop the projection runner after early turn-start failure:** `src/auto-reply/reply/dispatch-acp.ts:395-396`
  - body: On the durable ACP path, `ensureProjection(... waitForRunStart: true)` can start before `runTurn` creates a durable run. If `runTurn` fails first, this branch rethrows without stopping the projection service, leaving a background poll loop waiting for a run that will never appear.
  - confidence: 0.87
- **[P1] Propagate ACP worker event delivery failures:** `src/node-host/invoke.ts:425-426`
  - body: The ACP node command handler awaits `sendNodeEvent`, but the helper catches and ignores `client.request("node.event", ...)` failures. A worker event or terminal can be lost at the gateway RPC boundary while the node settles locally, leaving the gateway run stuck in recovery.
  - confidence: 0.9
- **[P1] Await ACP reconnect reconcile before worker events:** `src/gateway/server/ws-connection/message-handler.ts:1024-1038`
  - body: `handleNodeConnected` runs fire-and-forget after node registration, so a reconnecting node can deliver ACP worker events before its suspect lease is reactivated. Those events can be rejected as stale/non-active progress and the node-host path does not retry them as durable recovery.
  - confidence: 0.86
