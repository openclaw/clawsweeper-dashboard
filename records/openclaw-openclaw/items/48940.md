Full review comments:

- **[P1] Store ACP runtime state in SQLite:** `src/acp/store/store.ts:112`
  - body: This adds OpenClaw-owned durable ACP runtime state as `state/acp/gateway-node-runtime-store.json`. Runtime state belongs in the shared or agent SQLite stores with a clear migration/doctor owner; adding a JSON sidecar creates an upgrade-sensitive second state path outside the canonical storage model.
  - confidence: 0.9
- **[P1] Reject run IDs that belong to another session:** `src/acp/store/store.ts:532-534`
  - body: `startRun` treats any existing `runId` as idempotent without checking `sessionKey` or `requestId`. A collision can make one ACP session reuse another session's run record, after which worker events fail validation or corrupt delivery bookkeeping.
  - confidence: 0.92
- **[P1] Return the active projection instead of starting it again:** `src/auto-reply/reply/dispatch-acp-replay.ts:455-456`
  - body: When a target is already active, `ensureProjection` calls `existing.start()`, but `start()` launches `run()` every time. A second ensure for the same target can create parallel polling/delivery loops and race checkpoint updates or duplicate outbound sends.
  - confidence: 0.9
- **[P1] Stop the runner when turn creation fails early:** `src/auto-reply/reply/dispatch-acp.ts:394-396`
  - body: The durable path starts the projection runner before `runTurn`, then rethrows `turnError` when no durable run exists. That leaves the `waitForRunStart` runner polling for a run that can never be created, leaking background work for each early failure.
  - confidence: 0.88
- **[P1] Propagate ACP worker event transport failures:** `src/node-host/invoke.ts:425-427`
  - body: The ACP invoke handler sends worker events through the generic best-effort `sendNodeEvent`, whose implementation catches and ignores failed `node.event` RPCs. ACP terminals and worker events are durable protocol inputs, so treating a dropped send as success can leave the gateway store missing terminal state while the node believes the run completed.
  - confidence: 0.87
- **[P1] Await reconnect reconciliation before accepting worker events:** `src/gateway/server/ws-connection/message-handler.ts:1024-1034`
  - body: Node connect launches ACP lease reconciliation in the background. A worker event arriving before that reconciliation finishes can still see the lease as suspect and get rejected, leaving an otherwise completed node run stuck in gateway recovery.
  - confidence: 0.86
- **[P1] Gate ACP node commands explicitly:** `src/gateway/node-command-policy.ts:55-57`
  - body: Adding ACP session and turn commands to the default system node command set makes paired nodes eligible for new execution controls without an ACP-specific policy decision. These commands should be behind explicit capability/policy gating before broadening the gateway-node trust boundary.
  - confidence: 0.84
