## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Propagate ACP worker event send failures:** `src/node-host/invoke.ts:426`
  - body: `sendNodeEvent` swallows `client.request("node.event", ...)` failures, so `handleAcpInvokeCommand` can treat dropped ACP worker events or terminals as successful delivery. ACP needs a non-best-effort event sender here so the node does not finish locally while the gateway never records the durable terminal.
  - confidence: 0.86
- **[P1] Return the existing projection promise:** `src/auto-reply/reply/dispatch-acp-replay.ts:456`
  - body: When a target is already active, `ensureProjection` calls `existing.start()`, but `start()` launches the runner again. A second ensure for the same target can therefore create duplicate polling and delivery loops that race outward sends and checkpoint writes.
  - confidence: 0.83
- **[P1] Stop the projection runner when early turn start fails:** `src/auto-reply/reply/dispatch-acp.ts:395-396`
  - body: The durable ACP path starts a projection with `waitForRunStart: true`, then rethrows when `runTurn` fails before a durable run exists. That branch never stops the runner, so it can keep polling forever for a run that will never appear.
  - confidence: 0.81
- **[P1] Await ACP reconnect reconciliation before worker events:** `src/gateway/server/ws-connection/message-handler.ts:1024`
  - body: `handleNodeConnected` runs fire-and-forget after node registration, leaving a window where incoming worker events can hit a still-suspect lease and be rejected. A transient reconnect can strand a completed run in recovery if the node does not replay the rejected event afterward.
  - confidence: 0.82
- **[P1] Reject cross-session run ID reuse:** `src/acp/store/store.ts:532-534`
  - body: `startRun` returns any existing `store.runs[runId]` without verifying the session or request. If two sessions collide on a run id, the second session aliases the first run and can corrupt lease validation and delivery bookkeeping.
  - confidence: 0.85
