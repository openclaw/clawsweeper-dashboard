  - body: This creates `state/acp/gateway-node-runtime-store.json` for sessions, runs, events, leases, checkpoints, delivery targets, and idempotency.   - confidence: 0.93
- **[P1] Reject run IDs that belong to another session:** `src/acp/store/store.ts:532-535`
  - body: `startRun` returns any existing run for the raw `runId` before checking `sessionKey` or `requestId`. If two ACP sessions produce the same request-derived run id, the second session aliases the first run and later worker events can fail validation or corrupt delivery bookkeeping.
  - confidence: 0.91
- **[P1] Use reliable ACP worker event transport:** `src/node-host/invoke.ts:425-426`
  - body: The ACP command path passes worker progress through `sendNodeEvent`, but that helper still swallows `client.request("node.event")` failures. ACP terminals/events must surface transport failures so the node does not settle locally while the gateway never records the accepted event or terminal.
  - confidence: 0.9
- **[P1] Return the existing projection promise:** `src/auto-reply/reply/dispatch-acp-replay.ts:454-456`
  - body: When a target is already active, `ensureProjection` calls `existing.start()`, and `start()` launches another run loop. Re-entering projection for the same target can duplicate outbound delivery work and race checkpoint updates instead of deduplicating the active runner.
  - confidence: 0.89
- **[P1] Await node reconnect reconciliation:** `src/gateway/server/ws-connection/message-handler.ts:1024-1038`
  - body: The connect handler starts ACP status reconciliation in the background. Worker events arriving before that reconciliation reactivates a suspect lease can be rejected as stale/missing even though the same node legitimately completed the run.
  - confidence: 0.87
- **[P1] Keep ACP commands behind explicit node policy:** `src/gateway/node-command-policy.ts:54-57`
  - body: Adding ACP session and turn commands to the broad system command list expands paired-node authority without a distinct default-off or dangerous-command policy. These commands should be explicitly gated before they can control runtime execution over the node trust boundary.
  - confidence: 0.86
