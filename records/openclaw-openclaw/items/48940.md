## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Use SQLite for durable ACP runtime state:** `src/acp/store/store.ts:1`
  - body: This store becomes the source of truth for sessions, runs, events, leases, delivery targets, projector checkpoints, and idempotency, but it writes a JSON sidecar under the state directory. OpenClaw runtime state, queues, checkpoints, and caches are supposed to live in SQLite via the shared state DB/plugin KV/agent DB path, so landing this would add a second durable store that doctor/migrations and recovery paths do not own.
  - confidence: 0.93
- **[P1] Reject cross-session run id collisions:** `src/acp/store/store.ts:1`
  - body: `startRun` treats an existing `store.runs[runId]` as idempotent without requiring the same session and request. If two sessions produce the same run id, the second turn can reuse the first session's durable run and then fail lease/session validation or corrupt replay bookkeeping; namespace run ids by session or require the existing session/request to match before returning it.
  - confidence: 0.89
- **[P1] Do not restart active projection runners:** `src/auto-reply/reply/dispatch-acp-replay.ts:1`
  - body: `ensureProjection` restarts an existing runner for the same target instead of treating the active runner as already ensured. A second ensure on the same target can create concurrent polling and delivery loops over one run, racing checkpoint updates and duplicating outbound sends.
  - confidence: 0.86
- **[P1] Propagate ACP worker event delivery failures:** `src/node-host/invoke-acp.ts:1`
  - body: The ACP node-host path must not treat best-effort `node.event` delivery as successful ACP worker event or terminal delivery. If the gateway RPC fails after the worker completes locally, the gateway may never record the terminal and the run can be stranded in recovery while the node thinks it is done.
  - confidence: 0.84
- **[P1] Await reconnect reconcile before accepting worker events:** `src/gateway/server/ws-connection/message-handler.ts:1021`
  - body: The gateway starts ACP reconnect reconciliation as fire-and-forget, leaving a window where worker events after reconnect can arrive while the lease is still suspect and be rejected as stale. Reconnect handling needs to serialize or gate ACP worker ingress so valid late terminals are not dropped during recovery.
  - confidence: 0.84
- **[P1] Add real paired-node ACP proof before merge:** `src/acp/runtime/acp-node.ts:1`
  - body: The PR body lists harness tests, tsgo, extension tests, and CI, but explicitly says no fresh real node interactive end-to-end run was performed. This runtime changes gateway/node execution, restart/replay, reconnect, cancel, and outward delivery, so it needs redacted terminal/log/recording proof from a real paired-node ACP run after the fixes.
  - confidence: 0.96
