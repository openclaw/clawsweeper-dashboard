## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Store ACP runtime state in SQLite:** `src/acp/store/store.ts:111-113`
  - body: This new durable store writes sessions, runs, events, leases, checkpoints, delivery targets, and idempotency data to `state/acp/gateway-node-runtime-store.json`. OpenClaw's current runtime-state owner is SQLite with doctor-owned legacy import, and current main already has ACP state/replay tables, so this sidecar creates a second persistent runtime source of truth with no schema or migration owner. Move the ACP gateway runtime store into shared SQLite before merge.
  - confidence: 0.92
- **[P1] Return the existing projection promise:** `src/auto-reply/reply/dispatch-acp-replay.ts:454-457`
  - body: When a target is already active, `ensureProjection` calls `existing.start()`, but `start()` unconditionally launches `this.run()` again. Re-ensuring the same target can therefore start another polling/delivery loop over the same checkpoint and duplicate outbound sends or race checkpoint updates. Make `start()` idempotent or return the original settled promise.
  - confidence: 0.9
- **[P1] Stop projection polling on early turn-start failure:** `src/auto-reply/reply/dispatch-acp.ts:394-397`
  - body: The durable path starts projection with `waitForRunStart`, then rethrows if `runTurn` fails before a durable run exists. Because the runner keeps polling while the run is absent and this branch never stops it, each early start failure leaks a background projection loop for a run that will never appear. Stop/cancel the runner on this branch or start projection only after the run record exists.
  - confidence: 0.88
- **[P1] Await reconnect reconciliation before worker frames:** `src/gateway/server/ws-connection/message-handler.ts:1024-1038`
  - body: Node registration fires `handleNodeConnected` in the background, so ACP worker events or terminals can be processed while the lease is still `suspect`; the store then rejects progress as `ACP_NODE_ACTIVE_LEASE_MISSING`, and the node may not resend the terminal frame. Gate ACP worker ingress until reconciliation completes, or queue frames for the connection.
  - confidence: 0.85
- **[P1] Reject cross-session run ID reuse:** `src/acp/store/store.ts:532-535`
  - body: `startRun` returns any existing `store.runs[runId]` without verifying that it belongs to the same `sessionKey` and `requestId`. If two sessions produce the same request/run id, the second session aliases the first run and subsequent worker events can fail lease/session validation or corrupt delivery bookkeeping. Require the existing run's session and request to match, or namespace run ids by session.
  - confidence: 0.9
- **[P1] Gate ACP node commands explicitly:** `src/gateway/node-command-policy.ts:54-58`
  - body: Adding ACP commands directly to `SYSTEM_COMMANDS` makes ACP session and turn control part of the default desktop node command set. That expands the paired-node command execution boundary without a dedicated capability/policy gate or approval proof. Keep these commands behind explicit ACP capability and policy checks with focused tests before default-enabling them.
  - confidence: 0.82
