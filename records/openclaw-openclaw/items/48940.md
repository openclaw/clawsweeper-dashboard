## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Store ACP runtime state in SQLite:** `src/acp/store/store.ts:111`
  - body: The new durable ACP gateway store writes OpenClaw-owned runtime state to `state/acp/gateway-node-runtime-store.json`. Runtime state belongs in the shared state DB or a dedicated SQLite-backed schema, otherwise upgrades, backup behavior, locking, and migrations get a second unreviewed source of truth.
  - confidence: 0.92
- **[P1] Propagate ACP worker event failures:** `src/node-host/invoke.ts:426`
  - body: `handleAcpInvokeCommand` is wired through `sendNodeEvent`, but that helper catches and ignores `node.event` request failures. ACP turns can therefore mark worker events or terminals as delivered locally while the gateway store never records them, leaving durable runs stuck in recovery.
  - confidence: 0.91
- **[P1] Return the existing projection instead of starting another loop:** `src/auto-reply/reply/dispatch-acp-replay.ts:456`
  - body: When a projection already exists, `ensureProjection` calls `existing.start()`, and `start()` launches a new `run()` every time. A duplicate ensure for the same target can create parallel polling/delivery loops that race checkpoint updates and duplicate outbound sends.
  - confidence: 0.89
- **[P2] Stop projection polling when turn start fails early:** `src/auto-reply/reply/dispatch-acp.ts:396`
  - body: If `acpManager.runTurn` fails before a durable run record exists, this branch rethrows while the wait-for-run-start projection keeps polling for a run that will never appear. That leaks a background recovery loop for each early start failure.
  - confidence: 0.86
- **[P1] Await reconnect reconciliation before accepting worker progress:** `src/gateway/server/ws-connection/message-handler.ts:1024`
  - body: Node reconnect reconciliation is launched fire-and-forget, so incoming ACP worker events can be processed while the store still marks the lease as suspect. Those events can be rejected as missing an active lease even though the node has reconnected, leaving completed runs stuck.
  - confidence: 0.88
- **[P1] Reject cross-session durable run ID reuse:** `src/acp/store/store.ts:533`
  - body: `startRun` returns any existing `runs[runId]` as idempotent without checking the session or request identity. If two sessions collide on a provider/request-derived run ID, the second session aliases the first run and later worker events can corrupt or fail delivery bookkeeping.
  - confidence: 0.87
- **[P1] Gate ACP node commands behind explicit policy:** `src/gateway/node-command-policy.ts:56`
  - body: Adding all ACP commands to the default node command set lets the gateway ask connected nodes to perform ACP session and turn operations by default. This expands a security boundary and should be behind an explicit capability or policy decision with upgrade proof.
  - confidence: 0.84
