## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Propagate ACP worker event send failures:** `src/node-host/invoke.ts:426`
  - body: The ACP bridge awaits `sendNodeEvent`, but that helper catches and ignores failed `node.event` RPCs. A node can then finish a run locally while the gateway never records the worker event or terminal, leaving durable replay and cancel state inconsistent.
  - confidence: 0.92
- **[P1] Return the existing projection promise:** `src/auto-reply/reply/dispatch-acp-replay.ts:456`
  - body: When a projection target is already active, `ensureProjection` calls `existing.start()`, and `start()` launches another `run()` loop. Re-entering this path can duplicate outbound sends and race checkpoint updates for the same target.
  - confidence: 0.9
- **[P1] Stop the projection runner when turn creation fails:** `src/auto-reply/reply/dispatch-acp.ts:395-396`
  - body: If `runTurn` fails before creating a durable run, the code rethrows `turnError` while the wait-for-run-start projection loop remains active. That leaves a background poller waiting forever for a run that cannot appear.
  - confidence: 0.88
- **[P1] Await ACP reconnect reconciliation before worker events:** `src/gateway/server/ws-connection/message-handler.ts:1024`
  - body: `handleNodeConnected` runs in the background, so a reconnecting worker can send events while its lease is still `suspect`; those events are rejected as not active and the worker path has no proven fallback to recover the dropped terminal.
  - confidence: 0.86
- **[P1] Reject cross-session run ID reuse:** `src/acp/store/store.ts:533-534`
  - body: `startRun` treats any existing `runId` as idempotent without checking `sessionKey` or `requestId`. If two sessions collide on a request-derived run id, the second session reuses the first run and later worker events fail or corrupt delivery bookkeeping.
  - confidence: 0.91
- **[P3] Reject array payloads as non-object worker envelopes:** `src/acp/store/gateway-events.ts:50-52`
  - body: This `isRecord` helper accepts arrays, unlike the sibling ACP node parser, so malformed worker payloads reach field-level validation and produce misleading errors instead of failing as a non-object envelope.
  - confidence: 0.78
