## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Store ACP runtime state in SQLite:** `src/acp/store/store.ts:111-113`
  - body: This creates `state/acp/gateway-node-runtime-store.json` and writes sessions, runs, events, leases, checkpoints, delivery targets, and idempotency through JSON. New OpenClaw-owned runtime state should live in the shared SQLite/Kysely state model, otherwise upgrades and recovery have a second persistence contract.
  - confidence: 0.94
- **[P1] Propagate ACP worker event send failures:** `src/node-host/invoke.ts:425-427`
  - body: The ACP command handler awaits `sendNodeEvent`, but that helper catches and ignores `node.event` request failures. A node can therefore treat a worker event or terminal as delivered while the gateway never persisted it, leaving the run stuck or replaying from inconsistent state.
  - confidence: 0.92
- **[P1] Return the active projection promise:** `src/auto-reply/reply/dispatch-acp-replay.ts:455-457`
  - body: When a target already has a runner, `ensureProjection` calls `existing.start()`, and `start()` launches `run()` again each time. Re-entering the same runner can create duplicate polling/delivery loops racing the same checkpoint instead of deduping the active projection.
  - confidence: 0.91
- **[P1] Reject run IDs that belong to another session:** `src/acp/store/store.ts:532-535`
  - body: `startRun` returns any existing `store.runs[runId]` without checking `sessionKey` or `requestId`. Since the PR derives the run id from message ids when present, two sessions with the same provider message id can alias one durable run and corrupt delivery/recovery bookkeeping.
  - confidence: 0.9
- **[P1] Await reconnect reconciliation before worker events:** `src/gateway/server/ws-connection/message-handler.ts:1024-1038`
  - body: Node connect starts `handleNodeConnected` in the background, so ACP worker traffic can arrive while the persisted lease is still `suspect`. Valid same-node progress can be rejected before reconciliation completes, which can strand an otherwise healthy run in recovery.
  - confidence: 0.88
- **[P1] Gate ACP node commands explicitly:** `src/gateway/node-command-policy.ts:54-59`
  - body: Adding all ACP worker-control commands to `SYSTEM_COMMANDS` makes them part of default desktop/server node command policy. These commands start, cancel, close, and status-check ACP execution, so they need a dedicated ACP capability/trust-boundary gate instead of inheriting generic system command defaults.
  - confidence: 0.9
- **[P3] Replace the unreachable handled guard:** `src/gateway/server-node-events.ts:268-270`
  - body: After throwing when `handled` is false, the following `if (handled)` branch is always true. This is not the merge blocker, but it should be reduced to a direct return while touching this path.
  - confidence: 0.96

## Security Review

Status: needs_attention

Summary: The diff expands gateway-to-node command authority and durable runtime data access, so the security boundary needs explicit maintainer review before merge.
