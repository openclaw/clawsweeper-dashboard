Full review comments:

- **[P1] Store ACP runtime state in SQLite:** `src/acp/store/store.ts:111-113`
  - body: This new durable store writes ACP sessions, runs, events, leases, checkpoints, delivery targets, and idempotency to a JSON file under state. OpenClaw-owned runtime state belongs in the SQLite-backed state stores, so this sidecar creates an unsupported persistent store with no migration or doctor owner.
  - confidence: 0.94
- **[P1] Reject run IDs from other sessions:** `src/acp/store/store.ts:532-535`
  - body: `startRun` returns any existing run keyed by `runId` before checking `sessionKey` or `requestId`. If two sessions collide on a run ID, the second session aliases the first run and later worker events can reject or corrupt durable progress.
  - confidence: 0.91
- **[P1] Return the active projection promise:** `src/auto-reply/reply/dispatch-acp-replay.ts:455-456`
  - body: When a target is already active, `ensureProjection` calls `existing.start()`, but `start()` launches another polling loop. A second ensure call can therefore duplicate outward sends and race checkpoint advancement for the same target.
  - confidence: 0.93
- **[P1] Stop the runner when turn creation fails:** `src/auto-reply/reply/dispatch-acp.ts:394-397`
  - body: The durable path starts a projection runner waiting for a run, then throws `turnError` when no durable run exists without stopping that runner. That leaves a background loop polling forever for a run that will never be created.
  - confidence: 0.88
- **[P1] Await reconnect reconciliation before worker events:** `src/gateway/server/ws-connection/message-handler.ts:1024-1038`
  - body: The connect handler starts `handleNodeConnected` fire-and-forget, so ACP worker frames can be processed while the store still marks the lease suspect. Those frames can be rejected as missing an active lease, leaving a completed node turn stuck in recovery.
  - confidence: 0.84
- **[P1] Gate ACP node commands explicitly:** `src/gateway/node-command-policy.ts:54-57`
  - body: Adding `NODE_ACP_COMMANDS` to `SYSTEM_COMMANDS` makes ACP session and turn controls part of the default node allowlists whenever a node declares them. This new worker-control surface needs an explicit ACP policy or maintainer signoff instead of inheriting generic system command defaults.
  - confidence: 0.86
