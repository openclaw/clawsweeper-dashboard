## Review Findings

Full review comments:

- **[P1] Store ACP runtime state in SQLite:** `src/acp/store/store.ts:112`
  - body: This new durable runtime store writes sessions, runs, events, leases, checkpoints, delivery targets, and idempotency to `state/acp/gateway-node-runtime-store.json`. OpenClaw-owned runtime state and checkpoints belong in SQLite, so this sidecar would create unmanaged persistent state with no normal migration/doctor ownership.
  - confidence: 0.94
- **[P1] Reject cross-session run ID reuse:** `src/acp/store/store.ts:532-535`
  - body: `startRun` returns any existing `store.runs[runId]` without checking that it belongs to the same `sessionKey` and `requestId`. Since the dispatch path uses message-derived request IDs as run IDs, a collision can alias two sessions to one durable run and corrupt later worker-event and delivery bookkeeping.
  - confidence: 0.91
- **[P1] Return the active projection promise:** `src/auto-reply/reply/dispatch-acp-replay.ts:454-456`
  - body: When a projection for the same `targetKey` is already active, calling `existing.start()` launches `this.run()` again because `start()` always starts a new loop. Repeated `ensureProjection` calls can therefore run multiple projectors for one target and race or duplicate outbound delivery instead of deduplicating it.
  - confidence: 0.9
- **[P1] Stop projection when turn creation fails:** `src/auto-reply/reply/dispatch-acp.ts:395-396`
  - body: If `acpManager.runTurn` fails before a durable run exists, this branch rethrows `turnError` but never stops the projection runner started with `waitForRunStart: true`. That runner keeps polling forever for a run that will not appear, leaking background work for each early dispatch failure.
  - confidence: 0.88
- **[P1] Propagate ACP worker event delivery failures:** `src/node-host/invoke.ts:425-426`
  - body: The ACP worker bridge routes events through the generic `sendNodeEvent` helper, which catches and ignores `client.request("node.event", ...)` failures. A node can therefore treat worker events or terminals as delivered even when the gateway never persisted them, leaving runs stuck in recovery or with lost terminal state.
  - confidence: 0.92
- **[P1] Await reconnect reconcile before worker frames:** `src/gateway/server/ws-connection/message-handler.ts:1024-1038`
  - body: `handleNodeConnected` is launched fire-and-forget, so worker `acp.worker.*` frames can arrive while the gateway store still marks the lease `suspect`. Those frames can be rejected before status reconcile reactivates the lease, stranding an otherwise completed run in recovery.
  - confidence: 0.86
- **[P1] Gate ACP node commands explicitly:** `src/gateway/node-command-policy.ts:54-58`
  - body: Adding all ACP commands to `SYSTEM_COMMANDS` makes them part of the default macOS, Linux, and Windows node allowlists when a node declares them. This expands paired-node execution from generic system commands to ACP session and turn control without an ACP-specific opt-in or owner policy decision.
  - confidence: 0.9
