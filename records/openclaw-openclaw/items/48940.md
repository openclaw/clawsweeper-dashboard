Full review comments:

- **[P1] Store ACP runtime state in SQLite:** `src/acp/store/store.ts:112`
  - body: This creates `state/acp/gateway-node-runtime-store.json` for durable sessions, runs, events, leases, checkpoints, delivery targets, and idempotency. OpenClaw-owned runtime state belongs in the shared SQLite state owner with migration/doctor coverage, so this sidecar would create an upgrade and backup path outside the canonical store.
  - confidence: 0.93
- **[P1] Reject cross-session run ID reuse:** `src/acp/store/store.ts:532-534`
  - body: `startRun` returns any existing `store.runs[runId]` without checking that it belongs to the same session and request. Because the durable path derives `runId` from message ids when available, two sessions can alias to one run record and leave the second session's worker events failing lease/session validation.
  - confidence: 0.91
- **[P1] Return the active projection promise:** `src/auto-reply/reply/dispatch-acp-replay.ts:456`
  - body: When a target already has an active projection runner, `ensureProjection` calls `existing.start()`, but `start()` launches `run()` again. A second ensure for the same target can therefore create duplicate polling/delivery loops racing checkpoints and outward sends instead of deduplicating the active projection.
  - confidence: 0.9
- **[P1] Stop projection when turn creation fails:** `src/auto-reply/reply/dispatch-acp.ts:395-396`
  - body: The durable dispatch starts a projection runner with `waitForRunStart: true`; if `runTurn` fails before a durable run is created, this branch throws the turn error but never stops the runner. That leaves a background loop polling forever for a run record that will never appear.
  - confidence: 0.88
- **[P1] Propagate ACP worker event delivery failures:** `src/node-host/invoke.ts:426`
  - body: The ACP command handler passes worker events through the existing node-event helper, which catches and ignores `node.event` request failures. ACP event and terminal delivery is protocol-critical; if those sends are dropped, the node can settle locally while the gateway never records the canonical terminal, leaving the run in recovery or replay drift.
  - confidence: 0.89
