## Review Findings

Overall confidence: 0.9

Full review comments:

- **[P1] Store ACP runtime state in SQLite:** `src/acp/store/store.ts:111-113`
  - body: This new durable store writes sessions, runs, events, leases, checkpoints, delivery targets, and idempotency to `state/acp/gateway-node-runtime-store.json`. OpenClaw-owned runtime state belongs in the SQLite-backed state stores, so this sidecar creates a second persistent runtime store with no migration or doctor owner.
  - confidence: 0.94
- **[P1] Reject run IDs that belong to another session:** `src/acp/store/store.ts:532-535`
  - body: `startRun` returns any existing `store.runs[params.runId]` before checking `sessionKey` or `requestId`. If two sessions collide on a run ID, the second session aliases the first run and later worker events will corrupt or reject durable progress.
  - confidence: 0.91
- **[P1] Return the existing projection promise:** `src/auto-reply/reply/dispatch-acp-replay.ts:455-456`
  - body: When a target is already active, `ensureProjection` calls `existing.start()`, but `start()` launches another `run()` loop each time. A second ensure call can therefore duplicate outward sends and race checkpoint advancement for the same target.
  - confidence: 0.93
- **[P1] Stop the runner when early turn creation fails:** `src/auto-reply/reply/dispatch-acp.ts:394-397`
  - body: The durable dispatch path starts a projection runner with `waitForRunStart`, then throws `turnError` when no durable run exists without stopping the runner. That leaves a background loop polling for a run that will never be created.
  - confidence: 0.88
- **[P1] Use reliable ACP worker event transport:** `src/node-host/invoke-acp.ts:865-874`
  - body: The ACP worker path awaits `sendNodeEvent` and advances delivery state, but the underlying node-host helper treats `node.event` as best-effort and swallows RPC failures. A dropped worker event or terminal can look successful locally while the gateway store never records it.
  - confidence: 0.9
- **[P1] Await reconnect reconciliation before worker events:** `src/gateway/server/ws-connection/message-handler.ts:1024-1038`
  - body: The connect handler starts `handleNodeConnected` fire-and-forget, so subsequent ACP worker frames can be processed while the store still marks the lease suspect. Those frames can be rejected as missing an active lease, leaving a completed node turn stuck in recovery.
  - confidence: 0.84
- **[P1] Gate ACP node commands explicitly:** `src/gateway/node-command-policy.ts:54-57`
  - body: Adding `NODE_ACP_COMMANDS` to `SYSTEM_COMMANDS` makes ACP session and turn controls part of the default desktop/server node allowlists whenever a node declares them. This new worker-control surface needs an explicit ACP policy or maintainer signoff rather than inheriting generic system command defaults.
  - confidence: 0.86
