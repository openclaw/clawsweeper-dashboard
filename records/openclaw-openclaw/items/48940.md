Full review comments:

- **[P1] Store ACP runtime state in SQLite:** `src/acp/store/store.ts:111-112`
  - body: This adds durable OpenClaw runtime state as `state/acp/gateway-node-runtime-store.json`. Root policy makes SQLite the default owner for runtime state, so sessions, runs, events, leases, checkpoints, targets, and idempotency need to live in an approved SQLite owner with upgrade/doctor proof instead of a new JSON sidecar.
  - confidence: 0.95
- **[P1] Gate ACP node commands explicitly:** `src/gateway/node-command-policy.ts:54-57`
  - body: Adding `NODE_ACP_COMMANDS` to `SYSTEM_COMMANDS` makes ACP session/turn control part of the default desktop node command surface once declared. That broadens the gateway/node trust boundary; ACP worker control should be separately authorized or explicitly opted in rather than inherited from system command defaults.
  - confidence: 0.92
- **[P1] Propagate ACP worker event delivery failures:** `src/node-host/invoke.ts:425-426`
  - body: The ACP invoke path awaits `sendNodeEvent`, but that helper catches and ignores `client.request("node.event")` failures. A node can therefore treat worker events or terminals as delivered while the gateway never records them, leaving durable runs stuck in recovery or missing outward delivery.
  - confidence: 0.94
- **[P1] Return the active projection promise:** `src/auto-reply/reply/dispatch-acp-replay.ts:455-456`
  - body: When a projection already exists for the target, `ensureProjection` calls `existing.start()`, and `start()` launches another `run()` loop. A second ensure for the same target can therefore duplicate outbound sends and race checkpoint writes instead of joining the existing runner.
  - confidence: 0.93
- **[P1] Stop the projection runner on early turn-start failure:** `src/auto-reply/reply/dispatch-acp.ts:394-396`
  - body: If `runTurn` fails before the durable run exists, this branch rethrows after starting a projection with `waitForRunStart: true`. Because the runner is never stopped, it keeps polling for a run that will never appear and leaks background work after each early dispatch failure.
  - confidence: 0.9
- **[P1] Await ACP reconnect reconciliation before worker frames:** `src/gateway/server/ws-connection/message-handler.ts:1024-1034`
  - body: `handleNodeConnected` runs fire-and-forget during node registration, so `acp.worker.event` or `acp.worker.terminal` frames can be handled before suspect lease reconciliation completes. In that window the store still rejects progress, which can strand an otherwise completed worker run in recovery.
