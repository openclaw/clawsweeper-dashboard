Full review comments:

- **[P1] Store ACP runtime state in SQLite:** `src/acp/store/store.ts:111-112`
  - body: `AcpGatewayStore` persists sessions, runs, events, leases, checkpoints, delivery targets, and idempotency to a JSON sidecar. OpenClaw-owned runtime state belongs in SQLite with migration/doctor ownership, otherwise upgrade and recovery behavior drift from the canonical store.
  - confidence: 0.95
- **[P1] Gate ACP node commands explicitly:** `src/gateway/node-command-policy.ts:56`
  - body: Adding `NODE_ACP_COMMANDS` to `SYSTEM_COMMANDS` makes all ACP session and turn commands default-allowed for desktop nodes once declared. That expands the gateway/node command-control surface without an ACP-specific capability, operator, or policy gate.
  - confidence: 0.9
- **[P1] Propagate ACP worker event delivery failures:** `src/node-host/invoke-acp.ts:865-874`
  - body: The new ACP worker path awaits the same best-effort event sender that swallows `node.event` request failures, then advances `deliveredSeq` and can settle the local turn. A dropped terminal can leave the gateway store stuck in recovery while the node thinks the run completed.
  - confidence: 0.88
- **[P1] Return the existing projection promise:** `src/auto-reply/reply/dispatch-acp-replay.ts:456`
  - body: `ensureProjection` calls `existing.start()`, but `start()` launches a new `run()` every time. Re-ensuring the same target can run parallel projection loops that duplicate sends or race checkpoint writes.
  - confidence: 0.92
- **[P1] Await reconnect reconciliation before worker frames:** `src/gateway/server/ws-connection/message-handler.ts:1024-1038`
  - body: The node register handler starts ACP lease reconciliation in the background, so worker events or terminals can arrive while the lease is still suspect and be rejected. That can strand a completed node run in gateway recovery after reconnect.
  - confidence: 0.9
- **[P1] Reject cross-session run ID reuse:** `src/acp/store/store.ts:532-534`
