## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Propagate ACP worker event send failures:** `src/node-host/invoke.ts:425-427`
  - body: The ACP worker bridge sends durable worker events through a best-effort node-event helper that swallows `node.event` request failures. If a terminal or event is dropped, the node can treat the turn as settled while the gateway never records the state transition, leaving recovery/replay inconsistent.
  - confidence: 0.93
- **[P1] Return the active projection promise without restarting it:** `src/auto-reply/reply/dispatch-acp-replay.ts:454-456`
  - body: When a projection target is already active, `ensureProjection()` calls `existing.start()`, but `start()` launches another run loop each time. A second ensure for the same target can create duplicate polling/delivery loops and race checkpoint updates.
  - confidence: 0.91
- **[P2] Stop the projection runner when run creation fails:** `src/auto-reply/reply/dispatch-acp.ts:394-396`
  - body: The durable dispatch path starts a projection runner before `runTurn()`, then rethrows if no durable run record exists. That leaves the runner polling forever for a run that will never appear after early start failures.
  - confidence: 0.89
- **[P1] Reject run ID collisions across sessions:** `src/acp/store/store.ts:532-534`
  - body: `startRun()` treats any existing `runId` as an idempotent hit without verifying the same `sessionKey` and `requestId`. Cross-session request ID reuse can alias one session onto another run and break later lease/session validation.
  - confidence: 0.92
- **[P1] Await reconnect reconciliation before worker events arrive:** `src/gateway/server/ws-connection/message-handler.ts:1024-1038`
  - body: The reconnect path launches ACP lease reconciliation fire-and-forget, then continues normal node setup. A reconnecting worker can send events before a suspect lease is reactivated, causing valid progress or terminal events to be rejected and the run to remain in recovery.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: Security review needs attention because the PR expands gateway-to-node ACP command/event execution and classifies ACP runtime commands as default system node commands.

Concerns:

- **[medium] Classify ACP node commands outside the default system bucket:** `src/gateway/node-command-policy.ts:56`
  - body: The PR adds `NODE_ACP_COMMANDS` to the default `SYSTEM_COMMANDS` set, making ACP session/turn control commands available wherever normal system commands are default-allowed. Because these commands start and control node-host runtime execution over the gateway-node boundary, they need explicit policy classification, documentation, and tests instead of inheriting the existing system command bucket.
  - confidence: 0.74
