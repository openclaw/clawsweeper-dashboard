## Review Findings

Overall confidence: 0.91

Full review comments:

- **[P1] Store ACP runtime state in SQLite:** `src/acp/store/store.ts:111-112`
  - body: This adds durable OpenClaw runtime state, event logs, checkpoints, delivery targets, leases, and idempotency as `acp/gateway-node-runtime-store.json`. Repository policy and current state ownership require OpenClaw-owned runtime state to live in SQLite, so this sidecar would create an unsupported parallel durable store and upgrade path.
  - confidence: 0.97
- **[P1] Propagate ACP worker event failures:** `src/node-host/invoke.ts:425-426`
  - body: The ACP worker bridge uses `sendNodeEvent`, but that helper swallows `client.request("node.event")` failures. A lost `acp.worker.event` or `acp.worker.terminal` can therefore look successful to the node while the gateway never records it, leaving the durable run stuck or replaying the wrong state.
  - confidence: 0.95
- **[P1] Return the active projection promise:** `src/auto-reply/reply/dispatch-acp-replay.ts:454-456`
  - body: When a projection is already active, `ensureProjection` calls `existing.start()`, and `start()` launches `run()` again. A second caller for the same target can create another delivery loop racing the same checkpoints and duplicating outward effects instead of joining the existing runner.
  - confidence: 0.94
- **[P1] Stop projection when early turn creation fails:** `src/auto-reply/reply/dispatch-acp.ts:374-381`
  - body: The durable path starts a projection runner with `waitForRunStart: true` before `acpManager.runTurn`. If turn creation fails before a durable run is created, the code rethrows without stopping the runner, leaving a polling loop waiting forever for a run that will never appear.
  - confidence: 0.91
- **[P1] Reconcile ACP leases before accepting worker events:** `src/gateway/server/ws-connection/message-handler.ts:1024-1034`
  - body: The reconnect hook starts `handleNodeConnected` fire-and-forget. Worker events can arrive before suspect-lease reconciliation finishes, so the store may reject legitimate output or terminals as missing an active lease and strand an otherwise healthy run in recovery.
  - confidence: 0.9
- **[P1] Reject run IDs from other sessions:** `src/acp/store/store.ts:532-535`
  - body: `startRun` returns any existing row for the same `runId` without checking `sessionKey` or `requestId`. Since dispatch can use message IDs as run IDs, two sessions can alias the same durable run and corrupt lease validation, recovery, and delivery bookkeeping.
  - confidence: 0.93
- **[P1] Gate ACP node commands outside system defaults:** `src/gateway/node-command-policy.ts:54-58`
  - body: Adding all ACP session and turn commands to `SYSTEM_COMMANDS` makes them default desktop/server node commands. These commands control runtime sessions and execution, so they need an explicit ACP capability or command-policy gate instead of inheriting the broad system-command allowlist.
  - confidence: 0.88
