Full review comments:

- **[P1] Move ACP runtime state into SQLite:** `src/acp/store/store.ts:112`
  - body: This stores OpenClaw-owned ACP runtime truth in `state/acp/gateway-node-runtime-store.json`, including runs, events, leases, checkpoints, delivery targets, and idempotency. Runtime state belongs in SQLite, otherwise this adds a second locking, migration, backup, and recovery path for gateway truth.
  - confidence: 0.96
- **[P1] Reject cross-session run ID reuse:** `src/acp/store/store.ts:533-534`
  - body: `startRun` returns any existing `store.runs[runId]` before checking `sessionKey` or `requestId`. If two sessions produce the same run ID, the second session will alias the first run and later worker events/terminals can fail validation or corrupt delivery bookkeeping.
  - confidence: 0.95
- **[P1] Do not restart an active projection runner:** `src/auto-reply/reply/dispatch-acp-replay.ts:455-456`
  - body: When a target already has an active runner, `ensureProjection` calls `existing.start()`, but `start()` launches another `run()` loop every time. A repeated ensure for the same target can therefore run duplicate poll/delivery loops and race checkpoint updates.
  - confidence: 0.94
- **[P2] Stop projection when turn start fails early:** `src/auto-reply/reply/dispatch-acp.ts:395-396`
  - body: If `runTurn` fails before a durable run exists, this branch rethrows after starting a projection runner with `waitForRunStart: true`. That runner is never stopped and will continue polling for a run that will never appear.
  - confidence: 0.9
- **[P1] Propagate ACP worker event delivery failures:** `src/node-host/invoke.ts:425-426`
  - body: ACP turn processing awaits `sendNodeEvent`, but the helper catches and ignores `node.event` transport failures. A node can finish locally while the gateway never records the event or terminal, leaving durable recovery and replay state wrong.
  - confidence: 0.93
- **[P1] Await ACP reconnect reconciliation before worker events:** `src/gateway/server/ws-connection/message-handler.ts:1024`
  - body: The connect path starts `handleNodeConnected` in the background. A reconnecting node can send `acp.worker.event` or `acp.worker.terminal` before the suspect lease is reactivated, causing valid progress to be rejected and the run to remain in recovery.
  - confidence: 0.91
- **[P1] Gate ACP node commands explicitly:** `src/gateway/node-command-policy.ts:56`
  - body: Adding `NODE_ACP_COMMANDS` to the default system command group makes six new worker-control commands allowlisted on desktop node platforms when a node declares them. These commands expand the gateway-to-node execution boundary and should be default-off or explicitly approved before merge.
  - confidence: 0.9
