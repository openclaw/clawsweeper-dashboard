## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Propagate ACP worker event send failures:** `src/node-host/invoke.ts:426`
  - body: The ACP command path sends worker events and terminals through `sendNodeEvent`, but that helper catches and ignores `node.event` request failures. A node can settle locally while the gateway never records the durable terminal/event, so this path needs an ACP sender that propagates failures or moves the node-host session into explicit error state.
  - confidence: 0.93
- **[P1] Return the active projection promise:** `src/auto-reply/reply/dispatch-acp-replay.ts:456`
  - body: When a target is already active, `ensureProjection` calls `existing.start()`, and `start()` launches `this.run()` every time. Re-ensuring the same target can start duplicate polling/delivery loops and race checkpoint writes instead of deduplicating work.
  - confidence: 0.91
- **[P2] Stop the projection runner when run creation fails:** `src/auto-reply/reply/dispatch-acp.ts:395-396`
  - body: The durable ACP path starts a wait-for-run projection before `runTurn`, then rethrows early failures when no durable run exists. That branch leaves the runner polling forever for a run that will never be created.
  - confidence: 0.89
- **[P1] Reject run ID collisions across sessions:** `src/acp/store/store.ts:533-534`
  - body: `startRun` returns any existing `runId` as idempotent without checking `sessionKey` or `requestId`. A reused request-derived ID from another session can alias to the wrong durable run and break lease/session validation.
  - confidence: 0.92
- **[P1] Await ACP reconnect reconciliation before worker events:** `src/gateway/server/ws-connection/message-handler.ts:1024-1034`
  - body: The node reconnect path starts ACP lease reconciliation in the background and then continues setup. A reconnecting worker can send progress before the suspect lease is reactivated, causing valid events or terminals to be rejected.
  - confidence: 0.86
- **[P1] Classify ACP node commands with explicit policy:** `src/gateway/node-command-policy.ts:56`
  - body: Adding `NODE_ACP_COMMANDS` to `SYSTEM_COMMANDS` makes ACP session and turn controls available wherever ordinary system commands are default-allowed. These commands need an ACP-specific authorization contract and tests before expanding the gateway-node execution boundary.
  - confidence: 0.86
- **[P3] Reject array worker envelopes at the object boundary:** `src/acp/store/gateway-events.ts:50-51`
  - body: The ACP worker envelope `isRecord` helper accepts arrays, so malformed array payloads fall through to misleading field-level validation instead of being rejected as a non-object envelope.
  - confidence: 0.78
