## Summary

Keep open. Current main still preserves the reported split: status/cron paths can see or use persisted WhatsApp direct-session routing, while agent session tools hide same-agent sessions outside the default `tree` visibility boundary unless configuration is broadened.

## What This Changes

Report asks for WhatsApp direct sessions that appear in status and cron delivery to be consistently discoverable and targetable by agent session tools.

## Best Possible Solution

Make an explicit maintainer policy choice, then either add a generic deliverable-session discovery/targeting path that preserves visibility boundaries or document the required `tools.sessions.visibility` setting as the supported operator path.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to persist a same-agent WhatsApp direct session that was not spawned by the requester, leave `tools.sessions.visibility` unset so it defaults to `tree`, then compare `openclaw status --json` with `sessions_list` or `sessions_send` from a different same-agent requester session.

## Solution Assessment

Unclear as a product decision. Making WhatsApp DMs unconditionally visible is not the narrowest safe fix; a generic visibility-aware deliverable-session path or explicit configuration guidance better preserves the current privacy model.
