## Summary

Keep open. Current main still routes `/v1/chat/completions` through the normal Gateway agent command path, generates a session key for anonymous calls, and has no Chat Completions passthrough or no-persistence config.

## What This Changes

The issue requests a lightweight `/v1/chat/completions` mode that keeps provider routing, fallback, and auth while skipping session metadata and transcript persistence.

## Best Possible Solution

Define a first-class, documented HTTP relay or ephemeral-session contract for provider routing/fallback/auth that explicitly states how streaming, tools, prompts, memory, usage, and existing OpenAI clients behave without persistence.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: send `/v1/chat/completions` without `user` or `x-openclaw-session-key`, and the handler generates an `agent:<id>:openai:<uuid>` session key before the normal persistence path. I did not run a live request because this read-only review must not create session artifacts.

## Solution Assessment

Unclear. Passthrough mode is a plausible product direction, but maintainers should choose between raw provider relay, ephemeral agent sessions, and cleanup/TTL semantics before changing the public Gateway contract.
