## Summary

Keep open. Current main still treats OpenAI-compatible Chat Completions as a normal agent run with generated session keys and durable session/transcript paths, and the requested no-persistence passthrough mode is a new public Gateway mode/config contract rather than conservative cleanup.

## What This Changes

Add a lightweight OpenAI-compatible Chat Completions relay mode that preserves routing, fallback, and auth while skipping agent sessions and all session persistence.

## Best Possible Solution

Define a first-class documented relay or ephemeral-session contract for OpenAI-compatible HTTP calls, covering routing, fallback, auth, prompts, tools, memory, usage, streaming, and persistence lifecycle.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: send /v1/chat/completions without user or x-openclaw-session-key and current main generates an agent:<id>:openai:<uuid> session key before dispatching through agentCommandFromIngress; I did not run it live because this read-only review must not create session artifacts.

## Solution Assessment

Unclear. Passthrough mode is plausible, but maintainers should choose between raw provider relay, ephemeral agent sessions, cleanup/TTL behavior, and REST session-management semantics before changing the public Gateway contract.
