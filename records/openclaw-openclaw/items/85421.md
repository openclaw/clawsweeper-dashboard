## What This Changes

The issue asks OpenClaw to apply existing control-reply suppression to cron announce delivery so exact `REPLY_SKIP` and `ANNOUNCE_SKIP` replies do not post to external channels.

## Best Possible Solution

Land one canonical fix that reuses the shared control-reply suppression predicate in cron final delivery, adds exact-token regressions for text and structured cron paths, and then close this issue from the merged PR.

## Reproduction Assessment

Yes. Source inspection on current main shows `REPLY_SKIP` remains non-empty after cron normalization and reaches the outbound delivery path; the reporter also supplied repeated Discord observations.

## Solution Assessment

Yes for the exact-token leak: the narrow maintainable solution is to reuse the existing control-reply suppression predicate in cron delivery before outbound send. Markdown-wrapped variants should be a separately tested behavior decision.
