## What This Changes

Issue asks for cron announce delivery to suppress control replies such as `REPLY_SKIP` and `ANNOUNCE_SKIP` instead of posting them to channels.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/85471 or an equivalent current-main patch that reuses the shared control-reply predicate in cron delivery, then close this issue as implemented.

## Reproduction Assessment

Yes, source-reproducible: current main's cron delivery normalizer only handles the silent token, while `REPLY_SKIP` and `ANNOUNCE_SKIP` are recognized by the gateway control-reply predicate. I did not run a live Discord cron in this read-only review.

## Solution Assessment

Yes, the narrow best fix is to reuse the existing shared control-reply predicate in cron outbound normalization; https://github.com/openclaw/openclaw/pull/85471 already implements that shape. Markdown-wrapped variants should stay a separate shared-contract decision if maintainers want them.
