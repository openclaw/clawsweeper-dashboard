## Summary

Keep open for maintainer acceptance: the patch appears correct and well-proven, but it intentionally expands automatic persisted binding deletion for ACP session state.

## What This Changes

The PR adds a cwd existence check to ACP dispatch stale-session cleanup so generic ACP_SESSION_INIT_FAILED errors unbind when the persisted ACP runtime cwd is missing, plus regression tests for missing and present cwd.

## Best Possible Solution

Land after a maintainer explicitly accepts cwd-confirmed ACP binding cleanup semantics, or narrow the guard if session-key-wide unbinding is too aggressive.

## Reproduction Assessment

Yes. Current main's stale classifier is message-only, while the PR proof and tests exercise the generic ACP_SESSION_INIT_FAILED plus missing persisted cwd path.

## Solution Assessment

Yes, subject to maintainer acceptance. The implementation extends the existing dispatch cleanup helper with the canonical cwd resolver and focused positive/negative tests instead of adding a parallel recovery path.
