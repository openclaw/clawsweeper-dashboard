## Summary

Keep open: the PR fixes a real ACP stale-binding gap, but the current catch-path still lets a pre-runtime `acp.allowedAgents` policy rejection remove persisted bindings when the stored cwd is missing.

## What This Changes

The branch extends ACP dispatch stale-session cleanup so a generic `ACP_SESSION_INIT_FAILED` can unbind a bound session when its persisted runtime cwd is filesystem-confirmed missing, with focused dispatch tests.

## Best Possible Solution

Keep the missing-cwd recovery, but only invoke cwd-based unbinding for actual ACP runtime initialization failures after turn startup, with a negative policy-rejection regression and explicit maintainer acceptance of the session-state cleanup semantics.

## Reproduction Assessment

Yes. The PR body and inspected gist provide a live staging path for the intended missing-cwd cleanup, and source inspection shows a high-confidence current-branch path where `acp.allowedAgents` rejection can hit the same cwd-based unbind before `runTurn`.

## Solution Assessment

No, not yet. The cwd probe is the right signal for the reported runtime failure, but it must be scoped to runtime initialization failures rather than every `ACP_SESSION_INIT_FAILED` thrown in the dispatch try block.
