## Summary

Keep open. Current main still misses the dispatch-side missing-cwd stale ACP binding cleanup, and this PR is a useful replacement for the earlier closed unmerged #60720, but the reset-in-place part needs a small current-main adjustment because `closeSession` can now return a recoverable `runtimeNotice` instead of throwing.

## What This Changes

This PR extracts ACP stale-session classification into a shared helper, adds missing-cwd matching, uses it in dispatch and reset-in-place cleanup, and adds ACP dispatch/lifecycle regression tests.

## Best Possible Solution

Land a rebased version that keeps the shared stale-session predicate and dispatch missing-cwd cleanup, updates reset-in-place cleanup to handle both thrown stale init errors and recoverable `closeSession.runtimeNotice`, and includes the required changelog plus targeted ACP regressions.

## Reproduction Assessment

Yes. Current main has a deterministic mocked reproduction path: an `ACP_SESSION_INIT_FAILED` message containing `working directory does not exist` reaches dispatch stale cleanup but is not classified, while reset-in-place ignores recoverable close notices that current manager tests show are returned for discard reset init failures.

## Solution Assessment

No. The shared classifier and dispatch change are the right narrow shape, but the reset cleanup should also inspect a successful `closeSession` result's `runtimeNotice`; otherwise the missing-cwd reset path can still skip the new unbind logic on current main.
