## Summary

Keep open: the missing-cwd recovery is useful and has real behavior proof, but the current patch can remove bound-conversation state for pre-runtime ACP policy failures that also use `ACP_SESSION_INIT_FAILED`. That makes the PR not merge-ready until the cwd-based cleanup is scoped to actual runtime initialization failures.

## What This Changes

The PR adds a filesystem-confirmed missing persisted ACP cwd signal to dispatch stale-binding cleanup and covers generic init-failure cwd cases in dispatch tests.

## Best Possible Solution

Keep the missing-cwd recovery, but invoke cwd-based unbinding only for actual `runTurn` runtime-initialization failures and add the missing negative policy-rejection regression.

## Reproduction Assessment

Yes. The PR body and inspected gist provide a live staging path for the intended missing-cwd cleanup, and source inspection shows the review blocker through `resolveAcpAgentPolicyError` before `runTurn`.

## Solution Assessment

No, not yet. The cwd probe is the right signal for the reported runtime failure, but it must be scoped to runtime initialization failures rather than every `ACP_SESSION_INIT_FAILED` thrown in the dispatch try block.
