## Summary

Keep open: current main still lacks the missing-cwd ACP dispatch recovery, but this PR is not merge-ready because its broad catch can remove bindings for a pre-runtime policy rejection rather than a stale runtime session.

## What This Changes

The branch adds a persisted-cwd existence check to ACP dispatch stale-session cleanup and regression tests for generic init failures with missing or existing cwd metadata.

## Best Possible Solution

Land a narrowed dispatch-only fix that uses the missing-cwd signal only for actual ACP runtime initialization failures, preserves policy-rejection bindings, and has maintainer acceptance for the remaining session-state semantics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence PR-introduced path: a ready ACP session with a missing persisted cwd and disallowed acp.allowedAgents throws ACP_SESSION_INIT_FAILED before runTurn, then the PR catch can unbind it.

## Solution Assessment

No. The missing-cwd recovery is the right direction, but the cwd signal should be limited to actual runtime initialization failures and backed by a policy-rejection negative regression.
