## Summary

Keep this PR open: it is still the active useful fix for missing-cwd ACP bound-session cleanup, but the current patch is not merge-ready because the new cwd-based unbind can also run for pre-runtime policy rejections.

## What This Changes

The branch broadens ACP dispatch stale-binding cleanup to unbind a bound session on generic ACP session init failure when the persisted runtime cwd is missing, with dispatch regression tests.

## Best Possible Solution

Preserve the missing-cwd recovery, but only delete bindings after actual runtime-init/runTurn stale failures and make the remaining session-state deletion semantics an explicit maintainer decision.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: ready ACP session metadata with missing cwd plus an acp.allowedAgents rejection throws ACP_SESSION_INIT_FAILED before runTurn and then reaches the new cwd-based cleanup.

## Solution Assessment

No. The recovery direction is right, but the current catch is too broad; the best fix is to limit cwd-based unbind to actual runtime-init/runTurn failures and add the policy-rejection negative test.
