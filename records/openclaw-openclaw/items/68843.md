## Summary

Keep this PR open: the missing-cwd recovery is useful and has real behavior proof, but the current patch still widens stale-binding deletion into the broad dispatch catch where ACP agent-policy failures also flow before runtime init.

## What This Changes

This PR extends ACP dispatch stale-binding cleanup to treat generic ACP init failures as stale when the persisted session cwd is missing, with focused tests for missing versus existing cwd.

## Best Possible Solution

Land a narrowed dispatch-only fix that applies missing-cwd cleanup only to actual ACP runtime init failures, preserves policy-rejected bindings, and has maintainer acceptance for the remaining deletion semantics.

## Reproduction Assessment

Yes. Source inspection shows current main lacks the missing-cwd stale predicate, and the PR's linked live proof shows the intended after-fix generic init-failure cleanup; the introduced policy-rejection blocker is also source-reproducible.

## Solution Assessment

No, not as-is. The cwd probe should be gated to actual ACP runtime initialization failures rather than the whole dispatch catch, then maintainers can decide the remaining session-state semantics.
