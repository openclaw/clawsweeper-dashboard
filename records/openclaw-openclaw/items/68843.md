## Summary

Keep open: the missing-cwd recovery is useful and well evidenced, but the current patch can delete persisted ACP conversation bindings for pre-runtime agent-policy failures because that path also uses `ACP_SESSION_INIT_FAILED`.

## What This Changes

The branch adds a persisted ACP session cwd existence check to dispatch stale-session cleanup and tests generic ACP init failures with missing versus existing cwd metadata.

## Best Possible Solution

Keep the missing-cwd recovery, but gate the cwd-based unbind to actual runtime init/runTurn failures and preserve bindings for policy/config failures.
