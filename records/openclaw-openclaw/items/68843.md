## Summary

Keep this PR open. The patch appears to fix a real ACP stale-binding failure and has focused tests plus live log proof, but it intentionally removes persisted bound-conversation state and still needs ACP/session owner acceptance of that session-state tradeoff before merge.

## What This Changes

The PR extends ACP dispatch stale-binding cleanup so a generic `ACP_SESSION_INIT_FAILED` unbinds a bound session when its persisted `runtimeOptions.cwd` is missing, with regression coverage for missing and existing cwd cases.

## Best Possible Solution

Land this after an ACP/session owner accepts the missing-cwd unbind semantics and targetSessionKey-wide cleanup scope, or narrow the cleanup before merge if that state-loss tradeoff is not acceptable.
