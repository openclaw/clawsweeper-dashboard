## Summary

Keep this PR open: current main does not implement the missing-persisted-cwd cleanup, and the PR is the active narrow fix candidate, but the current patch can unbind persisted ACP conversation bindings for a pre-runtime policy rejection rather than an actual runtime init failure.

## What This Changes

The PR changes ACP dispatch so generic ACP session-init failures can unbind stale bound conversations when the persisted ACP runtime cwd is confirmed missing, with dispatch regression coverage.

## Best Possible Solution

Constrain cwd-confirmed cleanup to actual ACP runtime init or `runTurn` failures, add a negative policy regression, then get ACP/session owner acceptance for the targetSessionKey-wide unbind semantics.

## Reproduction Assessment

Yes. Source inspection shows the broad catch path can convert a pre-runtime allowed-agent policy rejection into cwd-based unbind, and the PR body supplies live logs for the intended missing-cwd runtime cleanup path.

## Solution Assessment

No. Persisted cwd is a useful runtime-staleness signal, but passing it through the broad dispatch catch is too wide; the best fix is to scope cwd-based unbind to the actual runtime init/runTurn failure path.
