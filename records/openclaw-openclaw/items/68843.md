## What This Changes

The branch adds a cwd-existence check to ACP dispatch stale-session classification and tests that missing persisted cwd unbinds stale bound conversations while an existing cwd does not.

## Best Possible Solution

Have an ACP/session owner accept the missing-cwd stale trigger and target-session-key-wide unbind semantics, then land this focused patch with its regression tests and supplied proof if that behavior is intended.

## Reproduction Assessment

## Solution Assessment

Mostly yes. The PR uses the persisted ACP cwd as a narrow signal instead of matching generic error text, but the target-session-key-wide unbind semantics need explicit owner acceptance before merge.
