## Summary

Keep this PR open: current main still lacks the missing-cwd stale-binding recovery and the supplied live log proof shows the intended behavior, but the latest patch can also unbind a session after a pre-runtime ACP agent policy rejection because that path uses the same `ACP_SESSION_INIT_FAILED` code.

## What This Changes

The PR extends ACP dispatch stale-session cleanup to treat generic ACP init failures as stale when the resolved ACP session metadata has a filesystem-missing persisted cwd, and adds focused dispatch tests.

## Best Possible Solution

Keep the missing-cwd recovery, but gate cwd-based unbinding to real runtime init failures and then get explicit maintainer acceptance for the remaining session-state deletion semantics.

## Reproduction Assessment

Yes. Source inspection shows current main only unbinds selected message-matched ACP init failures, while the PR proof shows the missing-cwd generic init-failure path clearing a stale binding.

## Solution Assessment

No. The intended fix is in the right dispatch area, but the current catch-site implementation is too broad because it can treat a pre-runtime policy rejection as a stale runtime init failure.
