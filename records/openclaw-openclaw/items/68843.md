## Summary

Keep this PR open for maintainer review: current main and the latest release still use only message-based ACP stale-session detection, while this branch adds the missing persisted-cwd signal with focused tests and live proof. I found no line-level correctness defect, but the new cleanup deletes persisted session bindings and needs explicit ACP/session-state owner acceptance before merge.

## What This Changes

The PR extends ACP dispatch stale-binding cleanup to unbind a bound session after a generic ACP init failure when the persisted runtime cwd is missing, with regression tests for missing and existing cwd cases.

## Best Possible Solution

Land this only after an ACP/session-state owner confirms the missing-cwd stale trigger and target-session-key-wide unbind semantics; otherwise choose a narrower binding-removal policy before merge.

## Solution Assessment

Mostly yes, pending owner approval: filesystem-confirmed missing cwd is a narrow signal for an unrecoverable ACP runtime, but target-session-key-wide unbind is a session-state policy decision.
