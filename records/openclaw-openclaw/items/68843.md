## Summary

Keep open: this is the right ACP recovery direction and the PR includes convincing live proof, but the implementation still routes the new missing-cwd deletion signal through the broad dispatch catch, so a pre-runtime allowed-agent policy failure can remove persisted bindings.

## What This Changes

The PR extends ACP dispatch stale-binding cleanup so generic `ACP_SESSION_INIT_FAILED` errors unbind a bound session when the persisted ACP runtime cwd is missing, with focused dispatch tests.

## Best Possible Solution

Keep the missing-cwd recovery, but gate cwd-based cleanup to actual runtime init or `runTurn` failures, add a policy-denial regression, and have maintainers accept the remaining target-session-wide unbind semantics.

## Reproduction Assessment

Yes. The intended bug has live staging log proof, and the remaining over-unbind blocker is source-reproducible from the outer catch plus `resolveAcpAgentPolicyError` using `ACP_SESSION_INIT_FAILED`.

## Solution Assessment

No, not yet. The fix belongs at the runtime-init or `runTurn` failure boundary; passing cwd into the broad outer catch makes pre-runtime policy failures delete session bindings.
