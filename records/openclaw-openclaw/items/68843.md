## Summary

Keep this PR open: the missing-cwd recovery is useful and the live proof is credible, but the current patch can also unbind persisted ACP bindings for an agent-policy rejection before runtime init starts.

## What This Changes

The PR adds a cwd-existence check to ACP dispatch stale-session cleanup and tests generic `ACP_SESSION_INIT_FAILED` behavior for missing versus existing persisted cwd.

## Best Possible Solution

Keep the missing-cwd recovery, but first restrict cwd-based cleanup to runtime-init failures or add an equivalent guard/test proving policy rejections cannot delete bindings, then get explicit session-state acceptance for the target-session unbind scope.

## Reproduction Assessment

Yes. Source inspection shows the reported current-main gap: stale cleanup keys on selected messages only, while real runtime init failures can surface as generic `ACP_SESSION_INIT_FAILED`; the PR also supplies live log proof for the intended missing-cwd path.

## Solution Assessment

No, not yet. The cwd probe is a good narrow repair for the runtime-init failure, but the current catch boundary also includes pre-runtime agent-policy rejection, so the best fix needs a guard or regression for that case before merge.
