## Summary

This PR should stay open, but it is not merge-ready: the intended missing-cwd recovery is useful and well proven, yet the new cwd-based unbind signal is applied to the broad ACP dispatch catch path, so an agent-policy rejection can delete session bindings before runtime initialization is attempted.

## What This Changes

The PR extends ACP dispatch stale-binding cleanup to unbind a bound session when a generic ACP session init failure is paired with a filesystem-confirmed missing persisted cwd, with focused dispatch tests.

## Best Possible Solution

Keep the missing-cwd recovery, but narrow the cwd-based unbind to actual runtime/session initialization failures after ACP dispatch reaches `runTurn`, preserve policy rejections as non-unbinding errors, and add a regression for the disallowed-agent plus missing-cwd case before maintainer acceptance of the session-state scope.

## Reproduction Assessment

Yes for the review blocker: source inspection shows `resolveAcpAgentPolicyError` emits `ACP_SESSION_INIT_FAILED` before `runTurn`, and the PR passes missing persisted cwd into the broad catch-path unbind classifier. The intended positive missing-cwd recovery also has linked live log proof.

## Solution Assessment

No. The requested recovery is the right area, but the implementation needs to distinguish runtime-init failures from pre-runtime policy failures before it is a safe session-state fix.
