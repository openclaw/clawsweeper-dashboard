## Summary

Keep this PR open, but it still needs changes before merge: the intended missing-cwd ACP recovery is useful and well-proven, but the current catch boundary can also unbind on pre-runtime agent-policy rejection because that path shares `ACP_SESSION_INIT_FAILED`.

## What This Changes

The PR extends ACP dispatch stale-binding cleanup to unbind a bound ACP session when a generic init failure is paired with a filesystem-confirmed missing persisted cwd.

## Best Possible Solution

Preserve the missing-cwd runtime-init recovery, add a guard and regression proving policy rejection cannot unbind, then have a maintainer accept the target-session unbind scope before merge.

Yes. The intended missing-cwd path is shown by PR proof and source inspection, and the policy-rejection defect is source-reproducible from the shared catch boundary plus shared error code.

## Solution Assessment

No, not yet. The cwd probe is a narrow repair for runtime init failure, but the current implementation applies it to an error boundary that also includes pre-runtime agent-policy rejection.
