## Summary

Keep open: the missing-cwd ACP recovery is useful and the live proof is sufficient, but the latest diff still lets a pre-runtime agent-policy failure use the new missing-cwd cleanup path and delete persisted bindings.

## What This Changes

The PR adds a filesystem-confirmed missing persisted ACP cwd condition to dispatch stale-binding cleanup and adds dispatch tests for missing versus existing cwd generic init failures.

## Best Possible Solution

Keep the missing-cwd recovery, but gate cwd-based cleanup to actual ACP runtime initialization failures and preserve bindings for policy/config failures.

## Solution Assessment

No, not yet. The fix direction is right, but the implementation is too broad because it lets pre-runtime policy errors use the missing-cwd cleanup path; the safer solution is a runtime-init-only gate plus a policy-denial regression.
