## Summary

Keep open: the dispatch-side classifier change looks useful, but this PR does not wire stale missing-cwd cleanup into the current bound `/new` and `/reset` path, and the supplied proof is still mocked/test-runner output rather than live ACP binding evidence.

## What This Changes

The PR shares ACP stale-session detection, adds missing-cwd stale cleanup in dispatch and lifecycle reset handling, and adds focused regression tests.

## Best Possible Solution

Keep the shared classifier and dispatch cleanup, but wire the stale missing-cwd cleanup through the actual ACP stateful target reset/gateway reset path and require redacted live ACP binding proof before merge.

## Reproduction Assessment

Yes, from source inspection: current main's stale dispatch classifier omits missing-cwd messages, and the current bound reset command path goes through the ACP stateful target driver and gateway reset service rather than the lifecycle helper changed by this PR. ## Solution Assessment

No. The shared classifier and dispatch fix are the right shape, but the reset half should be wired into `resetConfiguredBindingTargetInPlace`/the ACP stateful target driver or gateway reset authority rather than only updating a helper that is not on the current bound reset path.
