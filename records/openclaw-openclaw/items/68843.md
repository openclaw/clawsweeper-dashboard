## Summary

Keep open: the PR is a focused ACP stale-binding bugfix with good proof, but the new cwd-based stale trigger is wired through the whole dispatch catch and can unbind on non-runtime policy errors that also use ACP_SESSION_INIT_FAILED.

## What This Changes

The PR extends ACP dispatch stale-session cleanup to check the persisted ACP session cwd and unbind target-session-key bindings on generic ACP_SESSION_INIT_FAILED when that cwd is missing, with positive and negative dispatch tests.

## Best Possible Solution

Constrain the missing-cwd stale trigger to actual ACP runtime init or runTurn failures, keep policy/config failures from unbinding, then get ACP/session owner acceptance for the targetSessionKey-wide cleanup semantics.

## Reproduction Assessment

Yes. Current main's stale classifier is message-only, the supplied redacted staging log shows the after-fix cleanup path, and source inspection shows the new broad-catch policy-error path.

## Solution Assessment

No. Persisted cwd is a good signal for runtime init failure, but passing it through the whole dispatch catch is too broad; the safer fix is to restrict cwd-based stale classification to the runtime init/runTurn failure path.
