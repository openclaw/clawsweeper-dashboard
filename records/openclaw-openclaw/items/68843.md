## Summary

Keep this PR open for maintainer review: the patch appears technically correct and well proven, but it intentionally changes when persisted ACP conversation bindings are removed, so the session-state semantics need owner acceptance before merge.

## What This Changes

The PR extends ACP dispatch stale-session cleanup so a generic `ACP_SESSION_INIT_FAILED` with a filesystem-confirmed missing persisted cwd unbinds the stale ACP conversation binding, with positive and negative regression tests.

## Best Possible Solution

Have an ACP/session owner accept the missing-cwd stale trigger and target-session-key-wide unbind semantics, then land this focused patch with its regression tests and supplied proof if that behavior is intended.

## Reproduction Assessment

Yes. Current main source shows the generic `ACP_SESSION_INIT_FAILED` plus missing persisted cwd path is not classified as stale, and the supplied redacted staging log shows the after-fix cleanup removing the stale binding.

## Solution Assessment

Mostly yes. The PR uses persisted ACP cwd as a narrow signal instead of matching generic error text, but target-session-key-wide unbind needs explicit owner acceptance before merge.
