## Summary

Keep open: this PR targets a valid source-reproducible Control UI session-picker bug, but the patch removes native click activation from the picker buttons and leaves only mouse-down handlers, which regresses keyboard activation. It also needs real after-fix Control UI proof before merge.

## What This Changes

Replaces the session picker search, clear, option, and load-more button click handlers in `ui/src/ui/chat/session-controls.ts` with left-button mouse-down handlers that prevent focus loss before running each action.

## Best Possible Solution

Land a narrow picker event-handling fix that prevents the blur race while preserving native click and keyboard activation for all four controls, with focused regression coverage and real Control UI proof.

## Reproduction Assessment

Yes. Source inspection shows the reported blur path clears picker results before click, and the patch-introduced keyboard regression follows directly from removing click handlers from native buttons.

## Solution Assessment

No. Moving the actions from click to mouse-down addresses the mouse blur race but is not the best fix because it drops native keyboard activation; the safer path is to suppress the focus-shift on mouse-down while keeping click or keyboard activation.
