## Summary

Keep open. The PR targets a plausible Telegram delivery bug, but the patch is not mergeable as-is: it removes an existing formatter helper that is still referenced, narrows explicit HTML mode below Telegram's supported markup contract, and lacks real Telegram after-fix proof.

## What This Changes

The PR adds a Telegram HTML sanitizer, applies it before `sendTelegramText` calls `sendMessage` with `parse_mode: "HTML"`, and adds formatter and delivery tests for unsupported angle-bracket tags.

## Best Possible Solution

Revise the sanitizer to preserve the existing formatter helper and the full Telegram Bot API HTML contract, add a direct `textMode: "html"` boundary regression test, and provide redacted live Telegram proof for a literal unsupported tag.

## Reproduction Assessment

Yes for the PR defects: source inspection shows `escapeHtmlAttr` is deleted while `buildTelegramLink` still references it, and the whitelist omits documented Telegram HTML forms. The reported Telegram truncation still needs live Telegram proof on this branch.

## Solution Assessment

No. A send-boundary sanitizer can be a reasonable direction, but this implementation breaks the formatter and only partially models Telegram's HTML subset; the safer fix should preserve existing helpers and the documented explicit-HTML contract.
