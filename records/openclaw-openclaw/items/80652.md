## Summary

Keep open. The PR targets a plausible Telegram HTML boundary, but the patch is not mergeable yet because the sanitizer mishandles valid spoiler span markup, whitelists an unsupported spoiler tag, regresses link attribute escaping, and the supplied proof only exercises a standalone copy of the sanitizer rather than OpenClaw or Telegram delivery.

## What This Changes

The PR adds a Telegram HTML sanitizer, applies it before several parse_mode=HTML sends, and adds formatter plus delivery tests for unknown tag escaping.

## Best Possible Solution

Fix the sanitizer against Telegram’s exact HTML contract, preserve existing escaping guarantees, add focused tests for the broken cases, and provide real Telegram or OpenClaw delivery proof before using this to close the linked issue.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: `sanitizeTelegramHtml('<span class="tg-spoiler">s</span>')` will escape the closing span because the closing tag has no class attribute; I did not run tests or live Telegram in this read-only review.

## Solution Assessment

No. Sanitizing HTML before Telegram delivery is a plausible direction, but this implementation is not yet the narrow correct fix and it does not address the linked issue’s documented remaining reasoning-strip path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve tg-spoiler closing span tags:** `extensions/telegram/src/format.ts:51-54`
  - body: The span branch requires `class="tg-spoiler"` on every `<span>` token, so the closing `</span>` in `<span class="tg-spoiler">s</span>` is escaped as `&lt;/span&gt;`. That breaks the valid Telegram spoiler span form and contradicts the test this PR adds; allow closing span tags while still gating opening spans.
  - confidence: 0.94
- **[P2] Escape unsupported spoiler tags:** `extensions/telegram/src/format.ts:31-33`
  - body: Telegram HTML supports `<tg-spoiler>` and `<span class="tg-spoiler">`, not a bare `<spoiler>` tag. Keeping `spoiler` in the allowed set lets an unsupported tag pass through, preserving the same class of unrecognized tag this patch is meant to neutralize.
  - confidence: 0.88
- **[P2] Keep escaping angle brackets in href attributes:** `extensions/telegram/src/format.ts:22-24`
  - body: The old `escapeHtmlAttr` escaped `&`, `<`, `>`, and quotes via `escapeHtml`; this replacement only escapes `&` and quotes. Markdown link hrefs containing raw angle brackets can now be emitted inside `<a href="...">`, regressing the formatter’s existing Telegram-safe attribute escaping.
  - confidence: 0.82
