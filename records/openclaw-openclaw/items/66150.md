## What This Changes

The branch adds an operational-alert filter to transcript mirroring and regression tests that suppress backup/git alerts while preserving ordinary Spanish error text.

## Best Possible Solution

Limit suppression to explicit backup/git operational alert signatures or typed owner metadata, add negative coverage for ordinary pull-rebase troubleshooting text, and leave broader delivery replay policy with https://github.com/openclaw/openclaw/issues/51628.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path for the review finding: pass text beginning with `Se han producido conflictos durante el pull --rebase` through the PR resolver and append path, and it returns null as empty text. I did not run tests because this review is read-only.

## Solution Assessment

No, not as submitted: filtering explicit backup/git alert signatures is an acceptable narrow mitigation, but the standalone phrase matcher is too broad for a shared transcript mirror gate. The safer repair is to prefix-gate or remove that matcher and keep the broader delivery replay policy separate.
