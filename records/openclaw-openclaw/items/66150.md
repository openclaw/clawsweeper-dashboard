## Summary

This PR is a plausible narrow hardening fix, but it is not ready to merge because the generic alert filter can still silently drop legitimate warning-prefixed Spanish mirror text, the branch is conflicting, and no after-fix real behavior proof is posted.

## What This Changes

The PR adds operational-alert pattern filtering to `resolveMirroredTranscriptText` and regression tests so selected backup/git alert mirror text is not appended to session transcripts.

## Best Possible Solution

Land a narrower suppression keyed to a backup/git conflict signature or producer-owned non-transcript marker, preserve normal delivery mirrors, and leave broader replay/dedupe behavior to https://github.com/openclaw/openclaw/issues/51628.

## Reproduction Assessment

Yes at source level: current main would persist the reported alert string if it arrives through the generic mirror append path. I do not have high-confidence end-to-end proof that the backup/git alert producer is currently emitting that mirror in a live setup.

## Solution Assessment

No, not as written. Filtering non-user-facing operational alerts is reasonable, but the generic resolver should be constrained to backup/git conflict text or a producer-owned marker before merge.
