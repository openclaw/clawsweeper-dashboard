## Summary

Keep open: current main still persists arbitrary delivery-mirror text, but this PR can still silently drop legitimate warning-prefixed Spanish mirror text and the external PR lacks after-fix real behavior proof.

## What This Changes

The PR adds operational-alert regex filtering to `resolveMirroredTranscriptText` and regression tests for skipping backup/git alert mirrors.

## Best Possible Solution

Revise this into a narrow, regression-tested suppression for producer-owned backup/git operational alerts, preferably using a tighter signature or producer marker while leaving broader replay work to the open canonical issue.

## Reproduction Assessment

Yes, at source level: current main passes arbitrary non-empty mirror text through `resolveMirroredTranscriptText` and then persists it as a `delivery-mirror` assistant record. I did not run a live outbound or replay flow in this read-only review.

## Solution Assessment

No, not as written. Filtering known operational alert mirrors is reasonable, but the Spanish warning match should be constrained to the backup-git/conflict signature or replaced with a producer-owned marker before merge.
