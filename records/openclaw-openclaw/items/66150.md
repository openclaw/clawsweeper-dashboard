## Summary

Keep open: current main still lacks operational-alert suppression, but this PR is not merge-ready because the generic warning-prefixed Spanish filter can drop legitimate mirrored assistant text and the external PR still has no after-fix real behavior proof.

## What This Changes

The PR adds pattern-based suppression of known backup/git operational alert mirror text in `resolveMirroredTranscriptText` and tests that such mirrors are skipped while normal mirror text remains.

## Best Possible Solution

Revise this PR into a narrowly scoped suppression keyed to backup/git operational alerts or a producer-owned marker, preserve normal delivery mirrors, and leave broader replay/dedupe work to https://github.com/openclaw/openclaw/issues/51628.

## Reproduction Assessment

Yes, at source level: current main resolves arbitrary non-empty mirror text and appends it as a `delivery-mirror` assistant transcript entry. I did not run a live backup/git alert scenario in this read-only review.

## Solution Assessment

No, not as written. Filtering non-user-facing operational alerts is reasonable, but the generic resolver should match a backup/git conflict signature or producer-owned marker and the contributor needs to post after-fix real behavior proof.
