## Summary

Keep open. Current main still has the transcript-contamination path the PR targets, but the proposed filter is still too broad because it drops any warning-prefixed Spanish `Error en...` mirrored assistant text, not only the known backup/git operational alert.

## What This Changes

The PR adds operational-alert text filtering to the transcript mirror resolver and regression coverage so known backup/git alert mirrors are skipped while ordinary mirror text remains appendable.

## Best Possible Solution

Land a narrow, regression-tested suppression that only identifies producer-owned backup/git operational alerts, preferably via metadata or a tightly constrained backup-git/conflict signature, while leaving broader delivery replay work to #51628.

## Reproduction Assessment

Yes. Current main source traces any nonempty mirror text into a `delivery-mirror` transcript append, and a local regex probe shows the PR's first pattern also matches a plausible legitimate warning-prefixed Spanish assistant message.

## Solution Assessment

No. Filtering operational alert mirrors can be a reasonable hardening direction, but this PR should first constrain the Spanish warning pattern to the known backup/git alert or move the signal closer to the alert producer.
