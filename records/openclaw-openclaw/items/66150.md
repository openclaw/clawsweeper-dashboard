## Summary

Keep open. Current main still lacks this suppression, but this PR should not merge as-is because its generic alert regex can silently drop legitimate mirrored assistant text and the external contributor has not posted after-fix real behavior proof.

## What This Changes

The PR adds operational-alert pattern filtering to `resolveMirroredTranscriptText` and regression tests so backup/git alert mirrors are skipped while normal mirror text remains.

## Best Possible Solution

Land a narrow, regression-tested suppression only after the match is constrained to the backup/git conflict signature or a producer-owned marker, while leaving broader delivery replay work with https://github.com/openclaw/openclaw/issues/51628.

## Reproduction Assessment

Yes, at source level: current main returns arbitrary non-empty mirror text from `resolveMirroredTranscriptText`, and `appendAssistantMessageToSessionTranscript` persists it as `delivery-mirror` assistant content. I did not run a live backup/git alert scenario in this read-only review.

## Solution Assessment

No, not as written. Filtering non-user-facing operational alerts is reasonable, but the generic resolver should match a backup/git conflict signature or producer-owned marker instead of every warning-prefixed Spanish `Error en` mirror.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88
