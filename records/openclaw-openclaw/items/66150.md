## Summary

Keep open: current main still lacks this narrow suppression, but the PR should not merge as-is because the generic warning-prefixed Spanish regex can silently drop legitimate mirrored assistant text and the external PR has no after-fix real behavior proof.

## What This Changes

The PR adds operational-alert pattern filtering to `resolveMirroredTranscriptText` and regression tests so backup/git alert mirrors are not persisted as assistant transcript text.

## Best Possible Solution

Land a narrow, regression-tested suppression for non-user-facing operational alert mirrors, preferably matching a backup-git/conflict signature or producer-owned marker while leaving broad replay recovery to https://github.com/openclaw/openclaw/issues/51628.

## Reproduction Assessment

Yes, at source level: current main returns arbitrary non-empty mirror text from `resolveMirroredTranscriptText`, and `appendAssistantMessageToSessionTranscript` persists it as a `delivery-mirror` assistant record. I did not run tests or establish a live backup/git alert reproduction in this read-only review.

## Solution Assessment

No, not as written. Filtering known operational alert mirrors is reasonable, but the Spanish warning match should be constrained to the backup-git/conflict signature or replaced with a producer-owned marker before merge.
