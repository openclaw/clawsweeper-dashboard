## Summary

Keep open: current main still persists arbitrary non-empty mirror text, and this PR is a useful narrow hardening patch but should not merge as-is because its Spanish warning pattern can silently drop legitimate mirrored assistant content.

## What This Changes

The PR adds operational-alert filtering to transcript mirror resolution and tests for skipping stale backup/git alert text while preserving ordinary delivery mirrors.

## Best Possible Solution

Land a narrow, regression-tested suppression for non-user-facing backup/git operational alerts, preferably by matching a producer-owned backup-git/conflict signature while leaving general delivery replay work to #51628.

## Reproduction Assessment

Yes. Current main can be traced from outbound mirror delivery into `appendAssistantMessageToSessionTranscript`, and the PR regex can be checked directly against warning-prefixed non-backup Spanish assistant text that would be incorrectly discarded.

## Solution Assessment

No, not as written. Filtering operational alert mirrors is a reasonable hardening direction, but the first pattern should be constrained to the known backup-git/conflict alert or replaced with a producer-owned signal.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Constrain the Spanish warning match to backup-git alerts:** `src/config/sessions/transcript-mirror.ts:35`
  - body: The resolver runs for every delivery mirror before append. With `/^⚠️\s*error en\b/i`, legitimate mirrored assistant text such as `⚠️ Error en la API: reintenta luego` is classified as an operational alert, so `appendAssistantMessageToSessionTranscript` returns `empty text` and silently omits visible content from the transcript. Please require the known backup-git/conflict signature or a producer-owned marker instead.
