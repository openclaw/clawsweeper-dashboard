## Summary

Keep open. Current main still lacks operational-alert suppression, but this PR can silently drop legitimate warning-prefixed Spanish mirror text and the external contributor has not provided after-fix real behavior proof.

## What This Changes

The PR adds regex-based operational-alert suppression to `resolveMirroredTranscriptText` and tests that backup/git alert mirrors are skipped while normal delivery-mirror text remains.

## Best Possible Solution

Revise this into a narrow, regression-tested suppression for producer-owned backup/git operational alerts, preferably using a tighter signature or marker while leaving broad replay recovery to https://github.com/openclaw/openclaw/issues/51628.

## Reproduction Assessment

Yes, at source level: current main passes arbitrary non-empty mirror text through `resolveMirroredTranscriptText` and then persists it as a `delivery-mirror` assistant record. I did not establish a live outbound or replay reproduction in this read-only review.

## Solution Assessment

No, not as written. Filtering known operational alert mirrors is reasonable, but the Spanish warning match should be constrained to the backup-git/conflict signature or replaced with a producer-owned marker before merge.
