## Summary

Keep open: the PR targets a real transcript-contamination path, but the proposed generic warning-prefix regex can still silently drop legitimate mirrored assistant text and the external contributor has not provided after-fix real behavior proof.

## What This Changes

The PR adds operational-alert pattern filtering to transcript mirror resolution and regression tests that skip backup/git alert mirror text while preserving normal delivery mirrors.

## Best Possible Solution

Revise this into a narrow, regression-tested suppression for producer-owned backup/git operational alerts, while leaving broader delivery replay behavior to https://github.com/openclaw/openclaw/issues/51628.

## Reproduction Assessment

Yes, at source level. Current main can be traced from outbound mirror delivery into `appendAssistantMessageToSessionTranscript`, and the PR regex can be checked directly against warning-prefixed non-backup Spanish assistant text that would be discarded.

## Solution Assessment

No, not as written. Filtering known operational alert mirrors is reasonable, but the Spanish warning match should be constrained to the backup-git/conflict signature or replaced with a producer-owned signal.

## Review Findings
