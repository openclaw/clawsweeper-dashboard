Keep open: the PR targets a real source-reproducible agent file-path gap, but the implementation puts document-extension correction in a shared helper that is also used for free-form exec commands, so it can silently change shell text before execution.

## What This Changes

The PR adds document-extension correction for known `.docodex`/Office-extension hallucinations in agent tool parameter normalization, read workspace guarding, and focused tests.

## Best Possible Solution

Keep document-extension correction on structured read/write/edit path parameters while leaving exec command and routing normalization XML-only unless maintainers approve a parsed shell-command repair.

## Reproduction Assessment

## Solution Assessment

No. The path correction is plausible, but putting it inside a helper used by exec command and routing strings is broader than the maintainable file-path boundary.
