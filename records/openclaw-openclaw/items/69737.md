## Summary

Keep open. Current main partially fixes a neighboring provider-internal-error shape through https://github.com/openclaw/openclaw/pull/49401, but the exact reported OpenAI wording starting with “The server had an error processing your request” is still not matched by the new classifier and can still fall through to the raw formatter fallback before being pushed as a channel-visible error reply.

## What This Changes

The issue asks OpenClaw to stop posting raw unclassified assistant errorMessage text to user-facing delivery channels when stopReason is error.

## Best Possible Solution

Add a narrow regression that exercises the reported OpenAI “The server had an error processing your request” help-center/request-ID text through the formatter and payload path, then genericize that truly internal provider diagnostic while preserving existing actionable diagnostics.

## Reproduction Assessment
