## Summary

Keep open: the reported IRC proactive-send bug still exists on current main, and this PR is useful context but should not merge as-is because the process-local registry misses the CLI-to-gateway path and can publish clients before IRC channel joins are confirmed.

## What This Changes

The PR adds an IRC active-client registry, registers monitor clients, and makes `sendMessageIrc` prefer a ready registered client before falling back to a transient connection.

## Best Possible Solution

Implement proactive IRC channel sends through the gateway-owned joined connection, expose send readiness only after JOIN confirmation, preserve a no-monitor fallback, and add focused regression plus real IRC proof.

## Reproduction Assessment

Yes, by source inspection rather than a live IRC run: `openclaw message send --channel irc --target #room` runs in the CLI process while the joined monitor socket lives in the gateway process, so this branch’s module-local registry is empty and the transient unjoined send path remains.

## Solution Assessment

No. Reusing a monitor client is directionally useful for same-process sends, but the current PR is not the best fix because it misses the CLI/gateway process boundary and JOIN-confirmed readiness.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Route proactive IRC sends through the gateway connection:** `extensions/irc/src/send.ts:72-74`
  - body: The new lookup only sees monitor clients in the current Node process. CLI `message send` runs outside the gateway monitor process, so `getActiveClient(...)` stays empty there and channel sends still fall back to the transient unjoined `PRIVMSG` path.
  - confidence: 0.88
- **[P2] Publish monitor clients only after channel joins are confirmed:** `extensions/irc/src/monitor.ts:140`
  - body: `connectIrcClient()` resolves readiness on numeric `001` and sends JOIN commands afterward without waiting for confirmed membership. Registering the client immediately can still route startup sends through a socket that has not joined the target channel.
  - confidence: 0.84
