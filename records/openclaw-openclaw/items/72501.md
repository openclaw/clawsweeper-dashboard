Full review comments:

- **[P2] Keep registry write failures from failing sent polls:** `extensions/telegram/src/action-runtime.ts:620`
  - body: This awaits the new registry write after `sendPollTelegram` has already succeeded. If the state dir is unwritable or full, OpenClaw will report the poll action as failed even though Telegram already received the poll, leaving the user with an error for a delivered message and no registry; wrap/log this as non-fatal before returning the successful send result.
  - confidence: 0.88
- **[P2] Handle empty poll answers as retractions:** `extensions/telegram/src/bot-handlers.runtime.ts:1729-1730`
  - body: Telegram can send an empty `option_ids` array when a user retracts a poll vote, but this path joins it into an empty string and enqueues `voted:` with no option. That gives the agent misleading vote text instead of a clear cleared/retracted-vote event; add a retraction label and regression test.
  - confidence: 0.91
