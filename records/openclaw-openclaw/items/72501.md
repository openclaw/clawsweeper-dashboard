## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Mark poll votes untrusted:** `extensions/telegram/src/bot-handlers.runtime.ts:1584-1587`
  - body: The handler builds event text from Telegram voter identity plus poll question and option labels, then enqueues it without `forceSenderIsOwnerFalse: true` or `trusted: false`. Omitted trust metadata is treated as trusted system-event context, so external vote text can be promoted into trusted prompt context.
  - confidence: 0.94
- **[P2] Preserve topic targets in the poll registry:** `extensions/telegram/src/action-runtime.ts:502`
  - body: Only the explicit `messageThreadId` parameter is stored, but Telegram poll sends also accept topic IDs embedded in `to` such as `-100...:topic:42`. Those polls are sent to the topic, then later votes are registered without topic 42 and can route to the wrong forum session.
  - confidence: 0.9
- **[P2] Keep successful poll sends successful:** `extensions/telegram/src/action-runtime.ts:497-505`
  - body: This registry write runs after Telegram has already created the poll. If local persistence throws, callers see a failed action and may retry a non-idempotent send, creating duplicate polls; catch and log the registry failure before returning the successful send result.
  - confidence: 0.91
