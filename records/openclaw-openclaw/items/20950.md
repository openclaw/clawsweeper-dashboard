## Summary

Keep this issue open. Current main still lacks directory-backed cron job definitions, and the discussion already points to https://github.com/openclaw/openclaw/pull/41206 as the open implementation candidate rather than a shipped fix.

## What This Changes

Add optional directory-backed cron job storage so each cron job can live in its own JSON file for GitOps/NixOS-style management.

## Best Possible Solution

Keep this as the canonical tracker and land, rework, or replace https://github.com/openclaw/openclaw/pull/41206 with a backward-compatible directory-store mode that preserves jobs.json users and the runtime-state sidecar.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high confidence that current main lacks the requested directory-backed cron store because config, schema, docs, store, Gateway startup, and service persistence all still use one store file.

## Solution Assessment

Unclear for the exact config, symlink, precedence, and migration shape, but an additive directory-backed mode is the maintainable direction. The safest path is maintainer review and rework of https://github.com/openclaw/openclaw/pull/41206 or an equivalent replacement that preserves current jobs.json and jobs-state.json contracts.
