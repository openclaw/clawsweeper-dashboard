## Summary

Keep this issue open. Current main still exposes only a single cron store file plus a derived runtime-state sidecar, and the related open PR #41206 is the active implementation candidate rather than a reason to close this tracker.

## What This Changes

The issue requests an additive per-job cron directory store for declarative NixOS/GitOps setups alongside the current aggregate cron jobs file.

## Best Possible Solution

Keep this as the canonical tracker and land, rework, or replace #41206 with an additive directory-store mode that preserves existing `jobs.json` behavior and updates config schema/help, doctor/docs, and focused cron store/Gateway tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high confidence that current main lacks the requested directory-backed cron store path because config, schema, docs, store, and Gateway startup all still point at a single store file.

## Solution Assessment

Unclear for the exact config shape, but an additive directory-backed mode is the maintainable direction. The safest next step is to review, land, rework, or replace #41206 while preserving the current file-store and `jobs-state.json` sidecar contracts.
