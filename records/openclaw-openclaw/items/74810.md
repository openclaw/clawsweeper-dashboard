## Summary

Keep open: current main still has the reported no-tool run failure shape, and this issue adds the generic gateway `modelRun` path beyond the related `llm-task` workflow report. The discussion includes two independent reproductions and validated local patching, while open PRs #73031 and #74139 are still implementation candidates rather than landed fixes.

## What This Changes

The issue asks for intentionally LLM-only model runs to ignore inherited tool allowlists instead of failing with an empty-callable-tools guard error.

## Best Possible Solution

Land one focused fix covering both generic gateway `modelRun` and plugin-owned `llm-task` LLM-only runs, with regression tests and preserved fail-closed behavior for ordinary explicit allowlist misses.

## Reproduction Assessment

Yes. The issue provides a concrete `openclaw infer model run --gateway` repro, the follow-up comment provides a raw `/tools/invoke` repro, and current-main source inspection shows the same empty tools plus inherited allowlist precheck path before provider execution.

## Solution Assessment

Yes at the product level, but the implementation scope matters. The safest solution is to neutralize inherited allowlist checks only for intentional LLM-only runs while keeping ordinary explicit allowlist validation fail-closed.
