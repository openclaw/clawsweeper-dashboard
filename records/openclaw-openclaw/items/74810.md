## Summary

Keep open: current main still has the reported pre-provider failure shape, and the related fix PRs remain open or closed-unmerged rather than shipped.

## What This Changes

Intentional no-tool `modelRun` and `llm-task` runs should not fail because inherited global tool allowlists resolve to zero callable tools.

## Best Possible Solution

Land one focused fix that treats intentional LLM-only provider calls as no-tool runs while preserving empty-allowlist validation for ordinary tool-enabled runs, with regression coverage for both one-shot `modelRun` and `llm-task`.

## Reproduction Assessment

Yes. The issue supplies CLI and curl repros, and source inspection shows `modelRun` reaches the embedded runner with no callable tools while inherited allowlist sources still trigger the precheck guard before provider submission.

## Solution Assessment

Yes at the product level, but the implementation scope needs care. The safest fix is to exempt intentional LLM-only call shapes from inherited allowlist emptiness checks without weakening validation for genuine tool-enabled allowlist mismatches.
