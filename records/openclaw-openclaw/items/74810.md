## What This Changes

The issue requests that LLM-only modelRun and related disabled-tool runs ignore inherited ambient tool allowlists instead of failing before provider execution.

## Best Possible Solution

Land one focused fix that treats inherited ambient allowlists as irrelevant for intentional no-tool runs while preserving runtime toolsAllow contradiction checks.

## Reproduction Assessment

Yes. The issue and follow-up comment provide concrete CLI and /tools/invoke repros, and current-main source inspection shows the same empty-tools plus inherited-allowlist precheck before provider execution.

## Solution Assessment

Yes at the product level, but the implementation boundary matters. The narrow maintainable fix is to neutralize inherited allowlist checks only for intentional no-tool runs while keeping ordinary allowlist mistakes fail-closed.
