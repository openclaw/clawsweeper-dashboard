## Summary

Keep open: current main still has the reported failure shape, and this issue adds the generic one-shot `modelRun` path beyond the related `llm-task` workflow report.

## What This Changes

The issue asks for intentional no-tool `modelRun` and `llm-task` embedded runs to avoid failing on inherited tool allowlists.

## Best Possible Solution

Land one focused fix, likely by reconciling #73031 and #74139, that treats intentional no-tool provider calls as no-tool runs while preserving fail-closed allowlist validation for ordinary tool-enabled runs.

## Reproduction Assessment

Yes. The issue and follow-up comment provide concrete CLI and `/tools/invoke` repros, and current-main source inspection shows the same empty tools plus inherited allowlist precheck path before provider execution.

## Solution Assessment

Yes at the product level, but the implementation boundary matters. The safest solution is to neutralize inherited allowlist checks only for intentional LLM-only runs and add regression coverage for both gateway `modelRun` and `llm-task`.
