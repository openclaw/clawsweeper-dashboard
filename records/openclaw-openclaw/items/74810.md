## What This Changes

This issue asks for LLM-only `modelRun` and plugin-owned disabled-tool embedded runs to ignore inherited ambient tool allowlists instead of failing before provider execution.

## Best Possible Solution

Land one focused fix that ignores inherited ambient allowlist sources for intentional no-tool runs while keeping runtime `toolsAllow` contradictions fail-closed, reconciling #73031 and #74139 if needed.

## Reproduction Assessment

Yes. I did not run a live provider probe in this read-only review, but target-main source shows `modelRun` and `llm-task` can produce zero callable tools while inherited allowlist sources remain non-empty and still reach the guard.

## Solution Assessment

Yes, the issue’s requested direction is the narrow maintainable fix. The final patch should distinguish inherited ambient allowlists from explicit per-run `toolsAllow` so no-tool calls can proceed without weakening ordinary tool-policy validation.
