## Summary

Keep open: current main still has the reported failure shape for intentional no-tool model runs when inherited tool allowlist sources are non-empty, and the related open fix PRs have not landed.

## What This Changes

Allow intentional LLM-only modelRun and llm-task runs to proceed without the empty-callable-tools guard firing on inherited tool allowlists.

## Best Possible Solution

Review and land one focused fix that ignores inherited ambient allowlist sources for intentional no-tool runs while keeping explicit runtime toolsAllow conflicts fail-closed.

## Reproduction Assessment

Yes. I did not run a live provider probe in this read-only review, but current main source shows modelRun and llm-task can produce zero callable tools while still passing inherited allowlist sources into the guard.

## Solution Assessment

Yes, the requested direction is correct. The narrow maintainable fix is to neutralize inherited allowlist checks only for intentional no-tool call shapes and keep ordinary tool-enabled and runtime toolsAllow failures fail-closed.
