## Summary

Keep open: current main still has the reported failure shape for intentional no-tool model runs when inherited allowlist sources are non-empty, and the related open fix PRs have not landed.

## What This Changes

Allow intentional LLM-only modelRun and llm-task runs to proceed without the empty-callable-tools guard firing on inherited tool allowlists.

## Best Possible Solution

Land one focused guard-level fix that ignores inherited ambient allowlist sources for intentional no-tool model runs while preserving fail-closed validation for explicit runtime toolsAllow conflicts.

## Reproduction Assessment

Yes. The issue and follow-up comment provide concrete CLI and /tools/invoke repros, and current-main source shows modelRun and llm-task can produce zero callable tools while inherited allowlist sources still reach the guard.

## Solution Assessment

Yes, the requested direction is correct. The safest implementation should neutralize inherited allowlist checks only for intentional no-tool call shapes and keep ordinary tool-enabled and runtime toolsAllow failures fail-closed.
