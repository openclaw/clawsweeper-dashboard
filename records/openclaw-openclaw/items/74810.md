## Summary

Keep open: current main still has the reported failure shape for LLM-only runs when an inherited allowlist source is non-empty, and the related open PRs have not landed.

## What This Changes

LLM-only gateway model runs and llm-task embedded runs should proceed as no-tool calls even when the parent tool policy has an inherited allowlist.

## Best Possible Solution

Land one focused fix, likely by reconciling #73031 and #74139, that treats intentional no-tool provider calls as no-tool runs while preserving fail-closed allowlist validation for ordinary agent/tool runs.

## Reproduction Assessment

Yes. The issue and follow-up comment provide concrete CLI and /tools/invoke repros, and current-main source inspection shows the same empty tools plus inherited allowlist precheck path before provider execution.

## Solution Assessment

Yes at the product level, but the implementation boundary matters. The safest fix is to neutralize inherited allowlist checks only for intentional LLM-only runs and add regression coverage for both gateway modelRun and llm-task.
