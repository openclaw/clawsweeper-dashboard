## Summary

Keep open: current main and the latest release still render the run-scoped isolated-cron session identity into the Runtime prompt line, and the linked fix PR is open but not merged.

## What This Changes

Report that isolated cron runs include a per-run `:run:<UUID>` in Runtime prompt session fields, breaking byte-stable prompt prefix caching after v2026.6.8.

## Best Possible Solution

Land or replace the linked prompt-rendering fix so prompt-facing cron identity is stable while routing, transcript, cleanup, delivery, and subagent state remain per-run.

## Reproduction Assessment

Yes at source level: current main constructs a `:run:` isolated-cron session key, passes it into embedded execution, and renders it in the Runtime prompt line. I did not independently run a live metered-provider token trace.

## Solution Assessment

Yes, the narrow fix direction is to keep run identity internal and stabilize prompt-facing identity. Reusing one session across cron runs would weaken isolation, and a new config knob is not needed for the established bug.
