## Summary

Keep open: current main and v2026.6.10 still source-reproduce the reported volatile isolated-cron `Runtime:` line, and the issue is already paired with an open mergeable candidate fix PR that closes it.

## What This Changes

The issue reports that isolated cron runs render a per-run `:run:<UUID>` session suffix into the system prompt `Runtime:` line, breaking byte-stable prompt prefix caching across runs.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/96686 if it passes maintainer review, then close this issue as implemented by that merged commit and ship it in the next release train.

## Reproduction Assessment

Yes. Source inspection shows isolated cron creates a fresh run session id, appends it to the cron session key, forwards that key into runtime prompt params, and renders it verbatim in `Runtime:`; I did not run a live provider-cost repro in this read-only pass.

## Solution Assessment

Yes. The best fix is to keep cron delivery/session routing run-scoped while rendering only cache-stable identity in the prompt; the linked open PR appears to target exactly that boundary, so the issue should stay open until it lands or is rejected.
