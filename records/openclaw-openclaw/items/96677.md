## Summary

Keep open: current main and v2026.6.10 still source-reproduce the volatile isolated-cron Runtime-line session key, and the linked open fix PR is the active candidate. This issue should close only after that PR merges or a replacement lands.

## What This Changes

The issue reports that isolated cron run-scoped session keys leak a per-run UUID into the cached system prompt Runtime line, invalidating prompt prefix caching across runs.

## Best Possible Solution

Keep this issue as the canonical tracker until https://github.com/openclaw/openclaw/pull/96686 or a replacement lands, then close it as implemented with release or main-commit proof.

## Reproduction Assessment

Yes, at source level: current main constructs a :run: isolated-cron session key, passes it into embedded execution, and renders it in the Runtime prompt line. I did not run a live metered-provider billing comparison.

## Solution Assessment

Yes, the best direction is to keep the per-run cron identity internal for routing and cleanup while rendering a stable prompt/cache-facing cron identity. The open candidate PR targets that boundary, so the issue should remain open until the fix is merged or replaced.
