## Summary

Keep open: current main and v2026.6.10 still render the run-scoped isolated-cron session key in the Runtime prompt line, and the canonical fix PR is open but not merged.

## What This Changes

The issue reports that isolated cron runs inject a per-run `:run:<UUID>` into the system-prompt Runtime line, breaking prompt prefix caching and increasing metered input tokens.

## Best Possible Solution

Land a narrow prompt-rendering fix that keeps isolated-cron routing and transcripts run-scoped while removing only duplicated volatile run identity from the model-visible Runtime line.

## Reproduction Assessment

Yes at source level: current main creates `:run:` isolated-cron session keys, passes them into embedded execution, and renders them in the Runtime prompt line. I did not run a live metered-provider token trace.

## Solution Assessment

Yes. Prompt-facing normalization at `buildRuntimeLine` is the narrowest maintainable fix for this bug; broader volatile prompt-placement work remains separate in https://github.com/openclaw/openclaw/issues/71973.
