## Summary

Keep open: current main and the latest release still render the run-scoped isolated-cron session key in the Runtime prompt line, and the linked fix candidate is open but not merged.

## What This Changes

The issue reports that isolated cron runs leak a per-run `:run:<UUID>` session key into the system prompt, breaking byte-stable prompt prefix caching and increasing metered input cost.

## Best Possible Solution

Review, refresh, and land https://github.com/openclaw/openclaw/pull/96686 or a narrow replacement that keeps per-run cron state internal while rendering stable prompt-facing identity.

## Reproduction Assessment

Yes, source-reproducible: current main constructs a `:run:` isolated-cron session key, passes it into embedded execution, and renders that identity in the Runtime prompt line. I did not run a live provider token trace.

## Solution Assessment

Yes, the narrow fix direction is to keep per-run cron identity for routing/transcripts while normalizing the prompt-facing identity. Reusing one cron session across runs would weaken isolation, and a new config option is not required for the core bug.
