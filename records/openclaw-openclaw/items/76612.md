## Summary

Keep open. Current main still has a source-level path that can send Anthropic `cache_control` markers to Kimi Code, and #76741 is an open closing PR for the narrow Kimi-owned fix.

## What This Changes

Requests that OpenClaw avoid sending Anthropic `cache_control` prompt-cache markers to Kimi Code because they correlate with successful stop responses that contain no assistant content.

## Best Possible Solution

Review and land a Kimi-owned payload patch that strips `cache_control` only for Kimi Code, with regression coverage, while preserving shared Anthropic/OpenRouter/Bedrock cache behavior.

## Reproduction Assessment

Yes for a source-level reproduction, but not a live provider run. A Kimi `anthropic-messages` request with inherited or explicit `cacheRetention` reaches the shared Anthropic payload policy, which can add `cache_control` before the Kimi wrapper sends the payload.

## Solution Assessment

Yes. The narrow maintainable solution is the Kimi provider boundary that #76741 targets; disabling shared Anthropic-compatible caching globally would regress providers where prompt-cache behavior is intended.
