## Summary

Keep open. Current main still allows explicit or inherited `cacheRetention` to become Anthropic `cache_control` markers on the Kimi `anthropic-messages` path, and the linked open PR #76741 is the active fix candidate, so this issue should not be closed before that PR merges.

## What This Changes

The issue asks OpenClaw to prevent Kimi Code requests from carrying Anthropic `cache_control` prompt-cache markers that can yield empty successful responses.

## Best Possible Solution

Land #76741 or an equivalent Kimi-local payload patch that strips or suppresses `cache_control` only for Kimi, then close this issue after the fix merges.

## Reproduction Assessment

Yes for a source-level reproduction, but not a live provider call in this review. A Kimi `anthropic-messages` model with explicit or inherited `cacheRetention` reaches `buildAnthropicParams`, which applies `cache_control` before the Kimi wrapper sends the request and does not strip it.

## Solution Assessment

Yes. The narrow maintainable fix is at the Kimi provider boundary so Anthropic/OpenRouter/custom-provider cache behavior remains intact, and #76741 is already pointed at that boundary.
