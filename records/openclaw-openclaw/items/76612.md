## Summary

Keep open. Current main still lets Kimi Code requests receive Anthropic `cache_control` markers, and the linked fix PR #76741 is open with closing syntax, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Suppress or strip Anthropic `cache_control` prompt-cache markers for Kimi Code requests so successful Kimi responses do not become empty `incomplete_result` runs.

## Best Possible Solution

Review and land PR #76741, or an equivalent Kimi-provider-boundary fix, with regression coverage for system, user, and tool-result blocks plus a changelog entry.

## Reproduction Assessment

Yes for source-level reproduction, not live provider reproduction. A Kimi `anthropic-messages` model with explicit or inherited `cacheRetention` reaches `buildAnthropicParams`, receives `cache_control`, and the current Kimi wrapper does not remove it.

## Solution Assessment

Yes. A provider-local strip in `extensions/kimi-coding/stream.ts` is the narrowest maintainable repair; changing the generic Anthropic payload policy would affect other custom providers with tested cache-retention behavior.
