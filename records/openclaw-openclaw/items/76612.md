## Summary

Keep open. Current main still has a source-level path that can send Anthropic `cache_control` markers to Kimi Code, and #76741 is already open as the linked fix candidate.

## What This Changes

Suppress or strip Anthropic prompt-cache `cache_control` markers before Kimi Code requests so successful Kimi runs do not finish as empty content.

## Best Possible Solution

Review and land #76741 or an equivalent Kimi-owned wrapper fix that strips `cache_control` locally while preserving generic Anthropic-compatible cache behavior for providers that support it.

## Reproduction Assessment

Yes for source-level reproduction, but not live-provider reproduction in this review. A Kimi `anthropic-messages` model with explicit or inherited `cacheRetention` reaches `buildAnthropicParams`, which applies `cache_control` before the current Kimi wrapper sends the payload.

## Solution Assessment

Yes. A Kimi-owned payload strip is the narrowest maintainable fix because custom `anthropic-messages` cache retention is already documented and tested for other providers.
