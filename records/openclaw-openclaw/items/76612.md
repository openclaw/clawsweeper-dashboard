## Summary

Keep open. Current main still has the source path that can send Anthropic `cache_control` markers to Kimi, and open PR #76741 is already paired to this issue with a narrow fix.

## What This Changes

Suppress or strip Anthropic `cache_control` prompt-cache markers before requests reach Kimi Code's `api.kimi.com` Anthropic-compatible endpoint.

## Best Possible Solution

Review and land #76741, or an equivalent Kimi-owned wrapper fix, while preserving prompt-cache behavior for Anthropic, OpenRouter, Bedrock, and other compatible endpoints.

## Reproduction Assessment

Yes for a source-level reproduction, but not a live provider request in this review. A Kimi `anthropic-messages` model with explicit cache retention can reach shared Anthropic payload building, which applies `cache_control` before the current Kimi wrapper sends the request.

## Solution Assessment

Yes. The narrow maintainable solution is at the Kimi provider boundary; disabling cache markers globally would regress documented and tested cache behavior for other providers.
