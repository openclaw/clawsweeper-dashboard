## Summary

Keep open. Current main still lets inherited or explicit Anthropic `cacheRetention` reach the Kimi Code `anthropic-messages` request path, the Kimi wrapper does not remove `cache_control`, and the open closing PR #76741 should be reviewed or landed before this issue closes.

## What This Changes

Suppress Anthropic `cache_control` markers for Kimi Code requests so successful Kimi responses do not come back as empty `content: []` and get classified as incomplete.

## Best Possible Solution

Review and land #76741, or an equivalent Kimi-owned strip of `cache_control` in `extensions/kimi-coding/stream.ts`, while preserving generic Anthropic-compatible cache behavior for other providers.

## Reproduction Assessment

Yes for a source-level reproduction, but not a live request in this review. A Kimi `anthropic-messages` model with explicit or inherited `cacheRetention` reaches the shared Anthropic payload policy, which can attach `cache_control`, and the current Kimi wrapper does not strip it.

## Solution Assessment

Yes. The narrow maintainable fix is at the Kimi provider boundary; disabling prompt-cache markers globally would conflict with existing custom `anthropic-messages` cache-retention behavior and tests.
