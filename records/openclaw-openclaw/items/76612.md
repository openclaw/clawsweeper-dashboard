## Summary

Keep open. Current `main` still has a source-level path where explicit or inherited Kimi `cacheRetention` can become Anthropic `cache_control` markers, and the linked fix PR #76741 is open and unmerged, so this issue should wait for that PR or an equivalent fix to land.

## What This Changes

Kimi Code should stop receiving Anthropic `cache_control` prompt-cache markers because they can cause successful empty-content replies.

## Best Possible Solution

Land #76741 or an equivalent Kimi-owned payload scrub that removes Anthropic cache markers before requests reach `api.kimi.com`, then close this issue after the fix is merged and validated.

## Reproduction Assessment

Yes, at source level. A Kimi `anthropic-messages` model with explicit or inherited `cacheRetention` reaches `buildAnthropicParams`, which applies `cache_control` before the current Kimi wrapper sends the request.

## Solution Assessment

Yes. The narrow maintainable fix is to strip or suppress cache markers at the Kimi provider boundary; a global change would conflict with existing tests that preserve explicit cache retention for other custom Anthropic-compatible providers.
