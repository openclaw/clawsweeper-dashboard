## Summary

Keep open. Current main still lets Anthropic `cache_control` markers reach the Kimi `anthropic-messages` request path, and the open linked PR #76741 is the active implementation candidate, so this issue should remain open until that PR lands or is replaced.

## What This Changes

The issue asks OpenClaw to suppress or strip Anthropic `cache_control` markers before requests reach Kimi Code's Anthropic-compatible endpoint.

## Best Possible Solution

Land or replace #76741 with a Kimi-owned payload fix that removes `cache_control` only for Kimi Code, with regression coverage and a changelog entry.

## Reproduction Assessment

Yes for a source-level reproduction, but not a live provider run. A Kimi `anthropic-messages` model with explicit or inherited `cacheRetention` reaches `buildAnthropicParams`, gets `cache_control` markers, and the current Kimi wrapper forwards them without stripping.

## Solution Assessment

Yes. The narrow maintainable fix is at the Kimi provider boundary so Anthropic, Vertex, Bedrock, OpenRouter, and explicitly configured custom `anthropic-messages` cache behavior remain unchanged.
