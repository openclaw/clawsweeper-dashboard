## Summary

Keep open. Current main and the latest release still have a source-level path where Kimi requests can receive Anthropic `cache_control` markers, and the linked open PR is the active fix path rather than a reason to close the issue early.

## What This Changes

The issue requests that OpenClaw stop sending Anthropic `cache_control` prompt-cache markers to Kimi Code’s Anthropic-compatible endpoint because Kimi can return an empty successful response when those markers are present.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/76741; the final fix should stay provider-local to Kimi and avoid changing cache behavior for Anthropic-compatible providers that support these markers.

## Reproduction Assessment

Yes for a high-confidence source-level reproduction, but not a live Kimi request in this pass. A Kimi `anthropic-messages` model with inherited/default cache retention reaches shared Anthropic payload policy, which can add `cache_control`, and current main has no Kimi strip before send.

## Solution Assessment

Yes. The linked provider-local Kimi wrapper cleanup is the narrowest maintainable solution because disabling Anthropic prompt-cache markers globally would risk providers that intentionally support them.
