## Summary

Keep open. Current main still lets Kimi Code use the shared Anthropic Messages cache-marker path without a Kimi-specific strip or opt-out, and an open closing PR already targets the narrow provider-owned fix.

## What This Changes

Suppress Anthropic `cache_control` markers for Kimi Code requests so successful Kimi responses do not collapse into empty-content `incomplete_result` runs.

## Best Possible Solution

Review and land the linked Kimi provider-boundary fix that strips `cache_control` only for Kimi, while preserving other Anthropic-compatible prompt-cache behavior.

## Reproduction Assessment

Yes for source-level reproduction. Kimi declares `anthropic-messages`, `buildAnthropicParams` enables cache control, the payload policy applies markers, and the Kimi wrapper does not strip them; I did not make a live Kimi request.

## Solution Assessment

Yes. The narrow maintainable fix is at the Kimi provider boundary; changing the global Anthropic-compatible cache policy would risk providers whose explicit cache-retention behavior is already covered.
