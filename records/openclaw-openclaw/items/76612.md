## Summary

Keep open. Current main still has a source-level path where explicit or inherited `cacheRetention` can become Anthropic `cache_control` markers on Kimi requests, and the open fix at https://github.com/openclaw/openclaw/pull/76741 uses closing syntax for this issue, so this should not be closed before that PR is landed or otherwise resolved.

## What This Changes

Kimi Code should avoid Anthropic `cache_control` prompt-cache markers because they can produce successful Kimi responses with empty terminal content.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/76741; the fix should stay provider-local to Kimi and avoid changing Anthropic/OpenRouter/Vertex/Bedrock cache behavior.

## Reproduction Assessment

Yes for a source-level reproduction, but not a live request here. A Kimi `anthropic-messages` model with explicit or inherited `cacheRetention` reaches the shared Anthropic payload policy, which can add `cache_control`, and the current Kimi wrapper does not strip it.

## Solution Assessment

Yes. The best path is the existing provider-local Kimi payload cleanup in https://github.com/openclaw/openclaw/pull/76741, because a global cache-policy change would risk providers that intentionally support Anthropic-compatible cache markers.
