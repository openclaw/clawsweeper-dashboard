## Summary

Keep open. Current main and the latest release still have a source-level path where Kimi requests can receive Anthropic `cache_control` markers, and the open linked PR is the active fix path rather than a reason to close this issue early.

## What This Changes

The issue asks OpenClaw to stop sending Anthropic prompt-cache `cache_control` markers to Kimi Code's Anthropic-compatible endpoint because they correlate with empty successful responses.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/76741; the final fix should remain provider-local to Kimi and avoid changing cache behavior for Anthropic-compatible providers that intentionally support these markers.

## Reproduction Assessment

Yes for a source-level reproduction, but not a live Kimi request in this pass. A Kimi `anthropic-messages` model with inherited or explicit cache retention reaches shared Anthropic payload policy, which can add `cache_control`, and current main has no Kimi strip before send.

## Solution Assessment

Yes. The provider-local Kimi wrapper cleanup proposed in https://github.com/openclaw/openclaw/pull/76741 is the narrowest maintainable fix because a global cache-policy change would risk providers that support Anthropic cache markers.
