## Summary

Keep open. Current `main` still lets Kimi Coding use the shared Anthropic Messages cache-marker path, and this issue is already paired with open PR #76741 using closing syntax, so it should remain open until that PR lands or is rejected.

## What This Changes

Suppress or strip Anthropic `cache_control` prompt-cache markers for Kimi Code requests so successful Kimi responses do not end as empty `incomplete_result` runs.

## Best Possible Solution

Land a Kimi-owned payload policy fix that removes Anthropic cache markers only for Kimi Coding, with regression coverage for inherited and explicit `cacheRetention`, then let the linked PR close the issue.

## Reproduction Assessment

Yes for a source-level reproduction, but not a live provider reproduction here. A Kimi `anthropic-messages` request with explicit or inherited `cacheRetention` reaches the shared Anthropic payload policy, which can add `cache_control` before the current Kimi wrapper sends the request unchanged.

## Solution Assessment

Yes, the narrowest maintainable fix is at the Kimi provider boundary or an equivalent Kimi-specific payload policy guard. A global change to custom Anthropic-compatible cache behavior would risk regressing providers whose explicit cache-retention behavior is currently tested.
