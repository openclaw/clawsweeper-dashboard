## Summary

Keep open. Current main still has a source path where explicit or inherited `cacheRetention` on a Kimi `anthropic-messages` run becomes Anthropic `cache_control` markers, while the Kimi provider wrapper only adjusts thinking/tool-call compatibility and does not strip those markers.

## What This Changes

The issue asks OpenClaw to avoid sending Anthropic prompt-cache `cache_control` markers to Kimi Code because Kimi can return an empty successful terminal response when they are present.

## Best Possible Solution

Land a Kimi-owned compatibility fix that strips or suppresses Anthropic `cache_control` for Kimi Code while preserving generic Anthropic, OpenRouter, Vertex, Bedrock, and custom-provider cache behavior.

## Reproduction Assessment

Yes for a source-level reproduction, but not for a live upstream call in this review. A Kimi `anthropic-messages` model with explicit or inherited `cacheRetention` reaches the Anthropic transport, which applies `cache_control` before Kimi's wrapper sends the payload.

## Solution Assessment

Yes. The narrow maintainable fix belongs at the Kimi provider boundary, where Kimi-specific payload compatibility already lives, instead of disabling prompt-cache behavior for all custom Anthropic-compatible providers.
