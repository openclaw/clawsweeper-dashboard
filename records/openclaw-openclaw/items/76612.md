## Summary

Keep open. Current main still has the Kimi Anthropic Messages path that can receive inherited or explicit `cacheRetention`, add Anthropic `cache_control` markers, and then send the payload through the Kimi wrapper without stripping them; the linked open PR is the right place to finish the fix before this issue closes.

## What This Changes

Suppress or strip Anthropic `cache_control` prompt-cache markers for Kimi Code requests so Kimi no longer returns successful empty terminal responses.

## Best Possible Solution

Review and land PR #76741 or an equivalent Kimi-owned payload fix that removes `cache_control` before requests reach `api.kimi.com`, with focused regression coverage for inherited and explicit cache retention.

## Reproduction Assessment

Yes, for a source-level reproduction. A Kimi `anthropic-messages` model with explicit or inherited `cacheRetention` reaches `buildAnthropicParams`, receives `cache_control`, and the current Kimi wrapper has no stripping path.

## Solution Assessment

Yes. The narrow maintainable solution is to suppress or strip Anthropic cache markers at the Kimi provider boundary, while preserving shared Anthropic/OpenRouter/Vertex/custom-provider cache semantics elsewhere.
