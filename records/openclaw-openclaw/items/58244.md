## Summary

Keep this PR open. The underlying queue Map growth is source-reproducible and the pruning direction is useful, but the submitted patch introduces correctness/typecheck regressions and still lacks required after-fix real behavior proof.

## What This Changes

The branch adds pruning of idle dynamic entries from the process command queue lane Map plus regression tests.

## Best Possible Solution

Land a narrow queue fix that prunes only idle default-concurrency dynamic lanes while preserving configured, suspended, and persistent lane state plus waiter notifications, with regression coverage and real gateway/auth-probe proof.

## Reproduction Assessment

Yes, by source inspection. Current main creates dynamic `session:*` and `auth-probe:*` lanes in a process-global Map, and no production path removes idle lane entries; I did not run a live memory reproduction.

## Solution Assessment

No, not as submitted. Idle dynamic-lane pruning is the right direction, but this patch needs to preserve waiter notifications, existing test helper bindings, zero-concurrency suspension state, and all configured persistent lanes before it is maintainable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P2] Preserve suspended dynamic lanes:** `src/process/command-queue.ts:329`
  - body: The pruning guard deletes lanes with `maxConcurrent <= 1`, which includes lanes suspended with concurrency `0`. When `suspendSession()` pauses a session lane, completion, clear, or reset can delete that lane state, and the next enqueue recreates it at default concurrency, bypassing the intended suspension until auto-resume.
  - confidence: 0.9
- **[P2] Notify waiters after rejected lane tasks:** `src/process/command-queue.ts:308`
  - body: The rejection path now removes the active task id and calls `pump(true)` without notifying active-task waiters. `waitForActiveTasks()` relies on those notifications, so restart or recovery waits can sleep until timeout even though the rejected task has already drained.
  - confidence: 0.91
- **[P2] Restore the snapshot helper bindings:** `src/process/command-queue.test.ts:30`
  - body: The test module no longer binds `getCommandLaneSnapshot` or `getCommandLaneSnapshots`, but the existing helper and tests still call them. This leaves undefined identifiers in the PR head and blocks test typechecking.
  - confidence: 0.97
- **[P3] Include cron-nested in persistent lanes:** `src/process/command-queue.ts:125-130`
  - body: `CommandLane.CronNested` is configured as a startup lane alongside `cron`, `main`, and `subagent`, but this persistent-lane set omits it. Default-concurrency cron-nested state can be pruned even though the policy is meant to preserve well-known gateway lanes.
  - confidence: 0.84
