## Summary

Keep open: current main still has the source-reproducible lane Map leak, but this PR is not mergeable as-is because it changes active-task waiter behavior, removes helper bindings still used by tests, and can delete suspended lane state; external real behavior proof is also missing.

## What This Changes

The PR adds idle dynamic lane pruning to the process command queue and regression tests around lane count behavior.

## Best Possible Solution

Land a narrow queue fix that prunes idle default-concurrency dynamic lanes without losing suspended, custom, or persistent lane state, preserves waiter notifications, and includes regression coverage plus real gateway/auth-probe proof.

## Reproduction Assessment

Yes, by source inspection. Current main creates dynamic `session:*` and `auth-probe:*` lanes in a process-global Map, and no production path removes idle lane entries.

## Solution Assessment

No, not as submitted. Idle dynamic-lane pruning is the right direction, but this patch needs to preserve waiter notifications, existing test helper bindings, zero-concurrency suspension state, and all configured persistent lanes before it is maintainable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P2] Preserve suspended dynamic lanes:** `src/process/command-queue.ts:329`
  - body: The merged result prunes lanes with `maxConcurrent <= 1`, which includes lanes suspended with concurrency `0`. When `suspendSession()` pauses a session lane, completion, clear, or reset can delete that lane state and the next enqueue recreates it at default concurrency, bypassing the intended suspension until auto-resume.
  - confidence: 0.9
- **[P2] Notify waiters after rejected lane tasks:** `src/process/command-queue.ts:308`
  - body: The rejection path now removes the active task id and calls `pump(true)` without notifying active-task waiters. `waitForActiveTasks()` relies on those notifications, so restart or recovery waits can sleep until timeout even though the rejected task has already drained.
  - confidence: 0.91
- **[P2] Restore the snapshot helper bindings:** `src/process/command-queue.test.ts:30`
  - body: The test module no longer binds `getCommandLaneSnapshot` or `getCommandLaneSnapshots`, but the existing helper and tests still call them. This leaves undefined identifiers in the PR head and blocks test typechecking.
  - confidence: 0.97
- **[P3] Include cron-nested in persistent lanes:** `src/process/command-queue.ts:125-130`
  - body: `CommandLane.CronNested` is configured as a startup lane alongside `cron`, `main`, and `subagent`, but this persistent-lane set omits it. Default-concurrency cron-nested state can be pruned even though the policy is meant to preserve well-known gateway lanes.
  - confidence: 0.84
