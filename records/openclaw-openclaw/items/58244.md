## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Preserve suspended dynamic lanes:** `src/process/command-queue.ts:435`
  - body: Current main allows non-probe lanes to be suspended with `setCommandLaneConcurrency(lane, 0)`. This pruning condition also deletes idle non-persistent lanes with `maxConcurrent === 0`, so a suspended dynamic lane can be recreated at default concurrency and resume work unexpectedly.
  - confidence: 0.9
- **[P2] Notify active-task waiters after failures:** `src/process/command-queue.ts:298`
  - body: The failure path removes the active task id but now calls only `pump(true)`. `waitForActiveTasks()` depends on `notifyActiveTaskWaiters()`, so rejected or timed-out tasks can leave restart/recovery waiters unresolved until timeout.
  - confidence: 0.93
- **[P2] Keep the current snapshot helper bindings:** `src/process/command-queue.test.ts:30`
  - body: This removes the `getCommandLaneSnapshot` and `getCommandLaneSnapshots` bindings, but the PR head still calls those helpers later in the same test file. The command-queue tests will fail to typecheck or run until those bindings stay in place.
  - confidence: 0.97
- **[P2] Include cron-nested in persistent lanes:** `src/process/command-queue.ts:115-120`
  - body: Current main defines and configures `CommandLane.CronNested` as a well-known lane. Leaving it out of `PERSISTENT_LANES` means idle cron-nested state can be pruned even though gateway startup explicitly manages its concurrency.
  - confidence: 0.86
