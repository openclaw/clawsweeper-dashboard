## Summary

Keep open. The leak report is valid and the PR targets the right owner surface, but the exact head still has blocking rejected-task waiter and test-binding regressions, plus cron-nested persistence and changelog gaps; current main still lacks idle dynamic-lane pruning.

## What This Changes

The PR adds command-queue pruning for idle dynamic lanes in task completion, clear, and reset paths, plus a test-only lane count helper and regression tests.

## Best Possible Solution

Land a narrow command-queue fix that prunes only fully idle default-concurrency dynamic lanes while preserving active-task waiter notifications, snapshot helper tests, all well-known lanes including cron-nested, and release-note coverage.

## Reproduction Assessment

Yes. On current main, enqueueing work on a unique session:* or auth-probe:* lane creates a Map entry with no production idle delete path; the PR regressions are statically visible from the exact head diff and raw files.

## Solution Assessment

No, not as submitted. Pruning in src/process/command-queue.ts is the right maintainable fix, but the patch must preserve existing waiter semantics, restore test helper bindings, include cron-nested as persistent, and add the required changelog entry.
