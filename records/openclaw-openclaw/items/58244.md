Full review comments:

- **[P2] Preserve suspended dynamic lanes:** `src/process/command-queue.ts:540`
  - body: Current main suspends lanes with `setCommandLaneConcurrency(params.laneId, 0)`. This condition treats `maxConcurrent === 0` as disposable, so `clearCommandLane()` or `resetAllLanes()` can delete a suspended lane and the next enqueue recreates it at default concurrency before the scheduled resume.
  - confidence: 0.91

  - body: The rejection path removes the active task id but no longer calls `notifyActiveTaskWaiters()`. `waitForActiveTasks()` relies on that notification, so restart or recovery waits can sit until timeout after a rejected or timed-out task has already drained.
  - confidence: 0.9

  - body: The PR removes the `getCommandLaneSnapshot` and `getCommandLaneSnapshots` bindings, but the existing tests still call both helpers later in this file. That leaves the command-queue test file failing to typecheck or run after merge.
  - confidence: 0.88
