## Summary

## What This Changes

The PR adds `scripts/janitor/clean_claude_sessions.py`, a dry-run-by-default Python janitor for stale Claude CLI session JSONL files with explicit live deletion and summary logging.

## Best Possible Solution

Keep the dry-run-first janitor shape, but revalidate each candidate's current file state immediately before deletion and skip files that were touched, replaced, moved, or changed type since collection.

## Reproduction Assessment

Yes for the PR defect from source inspection: a file can be collected as stale, receive a new write before live deletion, and still be unlinked because deletion uses the old `Candidate` snapshot. I did not run destructive cleanup in this read-only review.

## Solution Assessment

No, not yet; the janitor's dry-run-first design is the right direction, but the live delete path should revalidate file age and identity at unlink time before this is the best mergeable form.
