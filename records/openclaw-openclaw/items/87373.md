## What This Changes

The PR adds a Python janitor script under `scripts/janitor` that dry-runs or explicitly deletes stale Claude CLI JSONL session files and appends live cleanup summaries to `memory/janitor.log`.

## Best Possible Solution

Keep the janitor dry-run-by-default, but make live deletion revalidate each candidate immediately before unlink and skip anything that is missing, replaced, outside the root, no longer a regular JSONL file, or newer than the cutoff.

## Reproduction Assessment

## Solution Assessment

No. The script shape is plausible, but the best fix is to revalidate the path, file type, root containment, and current mtime immediately before live deletion.
