## Summary

Keep this PR open: the useful janitor idea still has a concrete session-state blocker because live mode deletes candidates from an earlier scan snapshot without rechecking whether Claude has written to that transcript before unlink.

## What This Changes

The branch adds `scripts/janitor/clean_claude_sessions.py`, a dry-run-first Python janitor for stale Claude CLI project JSONL files with explicit live deletion and summary logging.

## Best Possible Solution

Keep the dry-run-first janitor shape, but make live deletion revalidate file freshness and identity immediately before unlinking, with focused proof for the updated-after-scan race.

## Reproduction Assessment

Yes, from source inspection: `collect_candidates()` snapshots stale files, then `delete_candidates()` later unlinks the same path without checking whether the current file is still stale. A focused temporary-directory fixture can reproduce this by updating a candidate after scan and before unlink.

## Solution Assessment

No, not yet. The maintainable fix is to re-stat each candidate immediately before deletion, skip updated or replaced files, and prove that race with a focused fixture or scripted proof.
