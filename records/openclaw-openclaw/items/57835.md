## Summary

This focused gateway PR still carries useful, unmerged behavior: current main returns top-level agent summary `name` from `agents.list[].name` only, while the PR consistently falls back to `agents.list[].identity.name` and covers precedence plus missing-name cases. I found no blocking correctness or security findings, and the existing automerge lane can continue gating the exact head before merge.

## What This Changes

The PR updates gateway agent list helpers to use `identity.name` when top-level `name` is unset and adds regression tests for fallback, explicit-name precedence, and unnamed agents.

## Best Possible Solution

Land the shared gateway fallback with explicit top-level `name` precedence preserved, keeping the regression coverage for both summary helpers.

## Reproduction Assessment

Yes, from source inspection. On current main, both gateway summary helpers set the row-level `name` from `entry.name` only, so an identity-only agent leaves that summary field unset.

## Solution Assessment

Yes. Updating the shared gateway summary helpers is the narrow maintainable fix because it preserves explicit configured names and avoids patching each downstream consumer separately.
