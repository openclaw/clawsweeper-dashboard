## Summary

Keep open. The PR remains a focused, useful bug fix: current main still leaves the top-level gateway summary name unset for identity-only configured agents, while the branch adds the configured-name-first fallback, regression coverage, sufficient terminal proof, and green checks.

## What This Changes

The branch changes `src/gateway/agent-list.ts` and `src/gateway/session-utils.ts` to populate summary `name` from trimmed `identity.name` when top-level `name` is absent, with colocated regression tests.

## Best Possible Solution

Land the focused gateway fallback after normal maintainer merge review, preserving configured-name precedence and the added regression tests.

## Reproduction Assessment

Yes, from source inspection. On current main, both gateway summary helpers set the top-level row `name` from `entry.name` only, so an agent with only `identity.name` leaves that summary field unset.

## Solution Assessment

Yes. The PR updates the shared gateway summary helpers directly with the same configured-name-first precedence already used by UI label normalization, which is narrower than making every consumer repeat the fallback.
