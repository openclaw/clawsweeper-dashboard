## Summary

Keep this PR open for normal maintainer merge/check handling. Current main still has the reported gateway summary gap, the PR fixes the right helper paths, the after-fix proof is sufficient, and I did not find a blocking correctness issue.

## What This Changes

The PR changes the basic and rich gateway agent-summary helpers to use `agents.list[].identity.name` when `agents.list[].name` is absent, with focused regression tests.

## Best Possible Solution

Land this focused gateway fallback after required checks are green or appropriately rerun, keeping the existing UI-side fallback as an additional rendering safeguard.

## Reproduction Assessment

Yes, from source inspection. On current main, an agent with `identity.name` but no top-level `name` flows through both gateway summary helpers with the returned row `name` taken only from `entry.name`.

## Solution Assessment

Yes. The PR's configured-name-then-identity-name precedence is the narrowest maintainable fix because explicit configured names remain authoritative while existing identity metadata fills the missing display-name field.
