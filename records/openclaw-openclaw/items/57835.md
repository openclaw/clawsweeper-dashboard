## Summary

Keep open: current main still lacks the PR’s unique top-level gateway summary fallback. The linked UI issue was closed because nested identity rendering already works, but this focused gateway contract change is not redundant and has sufficient real behavior proof with no blocking review findings.

## What This Changes

The PR updates gateway agent summary helpers to fall back from `agents.list[].name` to trimmed `agents.list[].identity.name`, with regression tests for fallback, explicit-name precedence, and missing-name cases.

## Best Possible Solution

Land the focused gateway fallback after normal maintainer merge review, preserving explicit top-level name precedence and the added regression coverage.

## Reproduction Assessment

Yes, from source inspection. On current main, both gateway summary helpers set the top-level row `name` from `entry.name` only, so an identity-only agent leaves that summary field unset.

## Solution Assessment

Yes. Updating the shared gateway summary helpers is the narrow maintainable fix and matches the existing UI label precedence of configured name, then identity name, then id.
