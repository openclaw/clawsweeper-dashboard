## Summary

Keep open: current main still leaves the top-level gateway agent summary name unset when only `identity.name` is configured, while this focused PR addresses that path with tests, real terminal proof, and green checks. I found no blocking correctness or security findings.

## What This Changes

The branch updates the basic and rich gateway agent summary helpers to fall back to trimmed `identity.name` when `name` is absent, with focused Vitest coverage.

## Best Possible Solution

Land this focused gateway fallback after ordinary maintainer merge review, keeping explicit configured names authoritative and retaining the regression coverage.

## Reproduction Assessment

Yes, from source inspection. On current main, an `agents.list` entry with `identity.name` and no top-level `name` leaves the top-level `name` unset in both gateway summary helpers.

## Solution Assessment

Yes. The PR's explicit `configuredName ?? identity.name` fallback is the narrow maintainable fix because it updates the shared gateway summary paths without changing configured-name precedence.
