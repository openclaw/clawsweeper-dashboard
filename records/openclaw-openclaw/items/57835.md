## Summary

Keep open: current main still leaves the top-level gateway/status agent `name` unset when only `identity.name` is configured, and this PR is a focused fix, but it still lacks contributor-supplied real behavior proof before merge.

## What This Changes

The PR changes gateway agent-list/session summary helpers to use trimmed `identity.name` as the top-level row name when `agents.list[].name` is absent, with regression tests for fallback and missing-name behavior.

## Best Possible Solution

Land the focused fallback after the contributor adds redacted real gateway/status output and the branch is refreshed and currently validated.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: configure an agent with `identity.name` but no top-level `name`, then `listGatewayAgentsBasic` and `listAgentsForGateway` leave the top-level row `name` undefined.

## Solution Assessment

Yes. The PR's `configuredName ?? identityName` fallback is the narrowest maintainable fix because explicit configured names still win while existing identity metadata fills the missing display-name field.
