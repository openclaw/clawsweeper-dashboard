## Summary

Keep open. Current main still omits the `identity.name` fallback in the gateway summary helpers, while this PR applies the narrow fallback in both affected paths with focused tests and sufficient terminal proof.

## What This Changes

The branch adds an `identity.name` fallback for gateway agent summaries in `listGatewayAgentsBasic` and `listAgentsForGateway`, plus regression tests for fallback, explicit-name precedence, and missing-name cases.

## Best Possible Solution

Land this focused gateway fallback after required checks pass, keeping explicit configured names authoritative and using the existing UI fallback as rendering-side safety.

## Reproduction Assessment

Yes, from source inspection. On current main, configure an `agents.list` entry with `identity.name` and no top-level `name`; both gateway summary helpers set the row name from `entry.name` only, leaving it unset.

## Solution Assessment

Yes. The PR's `configuredName ?? identity.name` fallback is the narrowest maintainable fix because it updates the shared gateway summaries while preserving explicit configured-name precedence.
