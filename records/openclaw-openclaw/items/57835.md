## Summary

Keep open. Current main still leaves the top-level gateway agent `name` sourced only from `agents.list[].name` in both summary helpers, and this PR's fallback is narrow, tested, and backed by terminal proof; the linked Web UI issue being closed does not make this gateway API consistency change obsolete.

## What This Changes

The PR updates `listGatewayAgentsBasic` and `listAgentsForGateway` to use trimmed `identity.name` when top-level agent `name` is absent and adds gateway regression tests for fallback, explicit-name precedence, and missing names.

## Best Possible Solution

Land the focused gateway fallback after normal maintainer review, preserving explicit top-level name precedence and the added regression coverage.

## Reproduction Assessment

Yes, from source inspection. On current main, both gateway summary helpers set the top-level row `name` from `entry.name` only, so an identity-only agent leaves that summary field unset.

## Solution Assessment

Yes. Updating the shared gateway summary helpers is the narrow maintainable fix and avoids patching each consumer that expects `agent.name` to be display-ready.
