## Summary

Keep this PR open: current main still leaves the top-level gateway agent name unset when only `identity.name` is configured, and the proposed code is a narrow, correct fallback. The merge blocker is missing after-fix real behavior proof from the external contributor, not a functional code finding.

## What This Changes

The PR updates gateway agent-list and session summary helpers to use `agents.list[].identity.name` as the displayed agent name when `agents.list[].name` is unset, with regression tests for fallback and missing-name cases.

## Best Possible Solution

Land the focused fallback after the contributor adds redacted real behavior proof and the branch is refreshed against current main.

## Reproduction Assessment

Yes. A config with `agents.list[]` containing `identity.name` but no top-level `name` follows a deterministic current-main path where both gateway helpers set top-level `name` only from `entry.name`.

## Solution Assessment

Yes. The `configuredName ?? identityName` fallback is the narrowest maintainable fix because it preserves explicit configured names while filling the missing display-name field from already-present identity metadata.
