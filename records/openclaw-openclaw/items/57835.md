Yes. Source inspection on current main shows that both gateway summary helpers leave the top-level `name` unset when `agents.list[].name` is absent even if `identity.name` is present.

## Solution Assessment

Yes for the code change. The `configuredName ?? identityName` fallback is the narrowest maintainable fix because explicit configured names still win while existing identity metadata fills the missing display-name field; the remaining work is merge-gate cleanup, not a different implementation.
