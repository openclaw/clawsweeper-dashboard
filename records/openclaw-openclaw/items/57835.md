Keep open: this is a focused bug-fix PR against a gateway summary gap that still exists on current main, and the latest diff is narrow, proved, and has no blocking review findings.

## What This Changes

The PR makes gateway agent summaries fall back from `agents.list[].name` to `agents.list[].identity.name` and adds regression tests for fallback, explicit-name precedence, and missing-name behavior.

## Best Possible Solution

Land the focused gateway fallback after maintainer merge review, preserving explicit configured-name precedence and keeping UI-side fallback as consumer safety.

## Reproduction Assessment

Yes, from source inspection. On current main, an `agents.list` entry with `identity.name` but no top-level `name` leaves `result.agents[].name` unset in both gateway summary helpers.

## Solution Assessment

Yes. The PR's configured-name-first fallback updates the shared gateway summaries directly and is narrower than relying on each UI/API consumer to repeat the same fallback.
