## Best Possible Solution

Land this focused gateway fallback after the branch is current enough for required checks, keeping the existing UI fallback as the rendering-side safety net.

## Solution Assessment

Yes. The PR's `configuredName ?? identityName` path is the narrowest maintainable fix because explicit configured names still win while existing identity metadata fills the missing display-name field.
