## What This Changes

This PR changes gateway agent listing helpers so `identity.name` becomes the top-level summary name when `agents.list[].name` is absent, with regression coverage for fallback, explicit-name precedence, and missing-name cases.

## Best Possible Solution

Land the focused gateway fallback after maintainer review, preserving explicit top-level name precedence and the added regression coverage.

## Reproduction Assessment

Yes, from source inspection. On current main, both gateway summary helpers set the top-level row `name` from `entry.name` only, so an identity-only agent leaves that summary field unset.

## Solution Assessment

Yes. Updating the shared gateway summary helpers is the narrow maintainable fix and avoids patching every consumer that expects `agent.name` to be display-ready.
