## What This Changes

The PR makes gateway agent list helpers populate top-level summary names from `identity.name` when configured `name` is missing, with regression tests for fallback, precedence, and unnamed cases.

## Best Possible Solution

Land the focused gateway fallback after normal merge gates, preserving explicit configured-name precedence and the added regression coverage.

## Reproduction Assessment

Yes, from source inspection. On current main, both gateway summary helpers leave top-level `name` sourced only from `agents.list[].name`, so an identity-only agent does not get `identity.name` promoted into the summary field.

## Solution Assessment

Yes. Updating the shared gateway summary helpers is the narrow maintainable fix and avoids patching each consumer that expects `agent.name` to be display-ready.
