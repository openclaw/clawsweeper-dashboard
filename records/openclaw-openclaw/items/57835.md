## Summary

Keep open for normal maintainer review: current main still leaves the top-level gateway summary name unset when only `identity.name` exists, and this PR adds the narrow fallback with focused coverage. The linked Web UI issue is already closed because the UI can read nested `identity.name`, but this branch still improves the gateway summary shape itself.

## What This Changes

The PR updates gateway agent summary helpers to fall back from `agents.list[].name` to `agents.list[].identity.name` and adds regression tests for fallback, precedence, and missing-name cases.

## Best Possible Solution

Land the focused gateway fallback after normal maintainer merge review, preserving configured-name precedence and the added regression coverage.

## Reproduction Assessment

Yes, from source inspection. On current main, both gateway summary helpers populate top-level `name` from `entry.name` only, so an agent with only `identity.name` leaves that summary field unset.

## Solution Assessment

Yes. The PR fixes the shared gateway summary helpers directly with the same configured-name-first precedence used by UI label normalization, which is narrower than making each consumer repeat the fallback.
