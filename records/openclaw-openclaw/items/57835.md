## Summary

Keep this PR open: current main still leaves the gateway agent summary `name` unset when only `agents.list[].identity.name` is configured, and the proposed patch is a focused fix with suitable regression coverage and real terminal proof.

## What This Changes

The PR updates gateway agent summary helpers to fall back to `agents.list[].identity.name` when top-level `agents.list[].name` is unset, while preserving explicit-name precedence and adding regression tests.

## Best Possible Solution

Land this focused gateway fallback after required checks are acceptable, keeping explicit configured names authoritative and retaining UI-side identity fallback as rendering safety.

## Reproduction Assessment

Yes. From current-main source inspection, configure an `agents.list` entry with `identity.name` and no top-level `name`; `listGatewayAgentsBasic` and the top-level `listAgentsForGateway` row name remain unset.

## Solution Assessment

Yes. The PR's `configuredName ?? identity.name` fallback is the narrowest maintainable fix because it updates the shared gateway summary helpers while preserving explicit configured-name precedence.
