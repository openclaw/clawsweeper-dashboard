## Summary

Keep open. Current main still leaves the top-level gateway summary name unset when only `identity.name` is configured, while this PR is focused, appears correct, and has sufficient terminal proof; the remaining action is ordinary maintainer merge/check review, not cleanup closure.

## What This Changes

The PR updates gateway agent summary helpers to fall back to trimmed `identity.name` when the top-level agent `name` is absent, with tests for fallback, explicit-name precedence, and missing-name cases.

## Best Possible Solution

Land the focused gateway fallback after normal maintainer review, preserving explicit top-level name precedence and the added regression coverage.

## Reproduction Assessment

Yes, from source inspection. On current main, both gateway summary helpers set the top-level row `name` from `entry.name` only, so an identity-only agent leaves that summary field unset.

## Solution Assessment

Yes. Updating the shared gateway summary helpers is the narrow maintainable fix and avoids patching each consumer that expects `agent.name` to be display-ready.
