## What This Changes

The PR makes gateway config hot reload cancellation-aware during shutdown/restart, moves config reloader stop before channel teardown, and adds gateway regression coverage plus drifted command-test updates.

## Best Possible Solution

Resolve the branch conflicts, keep the cancellation-aware reloader and shutdown ordering, then land after focused gateway tests and normal CI remain green.

## Reproduction Assessment

Yes. Current main shows a high-confidence source path: a channel hot reload can defer behind active work while gateway shutdown later stops channels before stopping or canceling the running config reloader.

## Solution Assessment

Yes. Stopping/aborting the config reloader before channel teardown and making deferred channel reloads observe that cancellation is the narrow lifecycle fix; process-wide channel single-flight would be broader and less direct.
