## Summary

Keep open: this is an active bug-fix PR for a linked gateway race, and current main still has the described deferred channel reload path without shutdown/restart cancellation. The PR’s approach matches the implicated lifecycle boundary, and I found no blocking review findings in the current diff.

## What This Changes

The PR adds a gateway config reload cancellation context, aborts and drains active reloads during stop, skips deferred channel restart work after cancellation, stops the config reloader before channel teardown, and adds focused gateway regression coverage plus related test expectation updates.

## Best Possible Solution

Land the focused gateway cancellation fix after required checks pass so https://github.com/openclaw/openclaw/issues/79487 closes through the PR merge.

## Reproduction Assessment

Yes, at source level. Current main defers channel reloads until active work drains but provides no cancellation signal to that deferred work, and shutdown currently tears down channels before stopping the config reloader, matching the linked issue timeline.

## Solution Assessment

Yes. Cancelling and draining the config reloader, then checking that cancellation before deferred channel stop/start, is narrower than introducing global per-channel single-flight semantics and stays in the gateway lifecycle owner boundary.
