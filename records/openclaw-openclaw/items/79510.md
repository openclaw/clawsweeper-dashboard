## Summary

No blocking findings. The PR targets a real gateway lifecycle race: current main can let a deferred channel reload continue into shutdown/restart, while the branch adds cancellation/drain state at the config reloader boundary and skips stale channel restart work.

## What This Changes

The PR adds abortable gateway config reload context, cancels deferred channel reloads during gateway shutdown/restart, moves reloader stop before channel teardown, and updates focused regression, test, and changelog expectations.

## Best Possible Solution

Land this focused gateway lifecycle fix after normal maintainer review and required checks, so https://github.com/openclaw/openclaw/issues/79487 is closed by the merge.

## Reproduction Assessment

Yes, source-reproducible. Current main defers channel reloads behind active work without a shutdown/restart cancellation context and stops the config reloader after channel teardown, matching the linked report’s timeline.

## Solution Assessment

Yes. Passing cancellation/drain state from the config reloader into hot reload handlers and stopping the reloader before channel teardown is the narrow owner-boundary fix for this race.
