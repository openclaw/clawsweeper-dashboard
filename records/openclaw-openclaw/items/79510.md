## Summary

Keep open: the patch is a narrow, source-backed fix for the gateway deferred channel reload restart race, with sufficient live-output proof and focused regression coverage. The remaining blocker is maintainer-side validation/rebase handling for exact-head CI, not a PR-local code finding.

## What This Changes

The branch adds cancellation-aware gateway config reload context, drains active reloads on stop, stops the config reloader before channel teardown, skips cancelled deferred channel restarts, and adds gateway regression tests plus a changelog entry.

## Best Possible Solution

Rebase onto current main if needed, rerun exact-head CI, and merge the cancellation-aware reload fix with its focused regression coverage and changelog entry preserved.

## Reproduction Assessment

Yes, with source-level confidence rather than a live Telegram webhook run. Current main defers channel hot reload behind active work, stops the config reloader late, and does not abort or drain active reload work during restart shutdown.

## Solution Assessment

Yes. Passing a cancellation context through the config reloader, draining active reloads on stop, and stopping the reloader before channel teardown is the narrow lifecycle fix; a process-wide channel single-flight would be broader and would mask stale reload work instead of cancelling it.
