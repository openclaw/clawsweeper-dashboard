## Summary

Keep this PR open: current main still has the deferred channel-reload/restart race, and the PR's cancellation fence is a narrow fix with focused tests, supplied real behavior proof, and no blocking review findings.

## What This Changes

Adds a gateway config-reload cancellation context, drains active reloads on stop, stops the reloader before channel teardown, skips cancelled deferred channel reloads, and updates focused gateway plus incidental test assertions.

## Best Possible Solution

Land this cancellation-fence approach or an equivalent core gateway lifecycle fix, then close https://github.com/openclaw/openclaw/issues/79487 as implemented.

## Reproduction Assessment

Yes. The source path on current main is high-confidence: a deferred channel reload can wait through active work while shutdown/restart begins, and the reloader is stopped only after channel teardown.

## Solution Assessment

Yes. Passing a stop/abort context through the existing reloader and cancelling deferred channel restart work is narrower than adding process-wide channel single-flight and directly addresses stale lifecycle work.
