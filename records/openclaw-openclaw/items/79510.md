Keep open: current main still has the gateway reload/restart race, and this PR addresses the implicated lifecycle path without a blocking code-review finding. Exact-head CI and normal maintainer review should still gate merge.

## What This Changes

The PR adds abortable gateway config reload context, cancellation checks for deferred channel reloads, earlier config-reloader shutdown, regression tests, command test expectation updates, and a changelog entry.

## Best Possible Solution

Land this focused gateway lifecycle fix after exact-head CI is green or current failures are confirmed unrelated, so https://github.com/openclaw/openclaw/issues/79487 closes through the merge.

## Reproduction Assessment

Yes, at source level. Current main defers channel reloads behind active work without a shutdown/restart cancellation context and closes channels before stopping the config reloader, matching the linked issue timeline.

## Solution Assessment

Yes. Passing cancellation and drain state from the config reloader into hot reload handlers, then stopping the reloader before channel teardown, is narrower than a process-wide channel single-flight guard.
