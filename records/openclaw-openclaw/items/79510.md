The PR adds cancellation-aware gateway config reload context, drains active reloads during shutdown, stops the config reloader before channel teardown, guards deferred channel restarts, and updates focused tests, changelog, and drifted command tests.

## Best Possible Solution

## Solution Assessment

Yes. Passing a stop/abort context through the config reloader, draining active reloads on stop, and stopping the reloader before channel teardown is narrower than adding broader process-wide channel single-flight behavior.
