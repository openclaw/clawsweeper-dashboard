## Summary

Keep this open. Current main still omits `moltbot.json` from active config discovery and doctor preflight recovery, so the reported `.openclaw/moltbot.json` sibling-layout upgrade failure remains valid; the related #54201 symptom work improved the gateway error wording, but it does not locate or repair the legacy-named config file.

## What This Changes

The issue requests migration, diagnostics, and changelog/docs coverage for installs where the real config remained at `.openclaw/moltbot.json` while `openclaw.json` was missing or skeletal after upgrade.

## Best Possible Solution

Add a narrow recovery path that detects `.openclaw/moltbot.json` when canonical `openclaw.json` is missing, empty, or skeletal; surfaces a clear gateway/doctor hint naming that file; offers a backup-assisted `openclaw doctor --fix` copy or merge; and updates migration/troubleshooting docs plus the changelog.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a home directory with `.openclaw/openclaw.json` as `{}` or otherwise missing `gateway.mode`, plus a valid `.openclaw/moltbot.json`; current path resolution selects the canonical file, doctor preflight returns early, and gateway startup reports missing `gateway.mode` without naming the sibling config.

## Solution Assessment

Yes. The best maintainable fix is targeted doctor/diagnostic recovery for this retired filename, not restoring broad legacy runtime probing that the v2026.3.22 breaking change intentionally removed.
