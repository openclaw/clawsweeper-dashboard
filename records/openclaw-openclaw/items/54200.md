## Summary

## What This Changes

Recover or diagnose upgrades where canonical config is skeletal but a sibling legacy `moltbot.json` still contains the user's real configuration.

## Best Possible Solution

Add a targeted doctor/startup recovery path that detects a sibling legacy `moltbot.json` only when canonical `openclaw.json` is missing, empty, or skeletal, while preserving `openclaw.json` as the canonical runtime contract.

## Solution Assessment

Yes. The narrow maintainable fix is doctor/startup recovery or guidance for this upgrade state; re-adding `moltbot.json` as a normal config fallback would be broader than the compatibility repair.
