## Summary

Keep open: current main and the latest tagged release still ignore a sibling `~/.openclaw/moltbot.json` when a skeletal `openclaw.json` already exists, and the related closed issue only improved the missing-`gateway.mode` wording, not this migration recovery gap.

## What This Changes

The issue asks OpenClaw to detect and recover from upgrades where the real config remains in `~/.openclaw/moltbot.json` while `~/.openclaw/openclaw.json` is empty or skeletal.

## Best Possible Solution

Add a narrow `doctor --fix` recovery and gateway diagnostic path for sibling `~/.openclaw/moltbot.json` only when canonical `openclaw.json` is missing, empty, or skeletal, while keeping `openclaw.json` canonical.
