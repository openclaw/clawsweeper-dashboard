## Summary

Keep this issue open. Current main and the latest release still have a source-reproducible missing-schedule cron row path, and the linked closing PR is open rather than merged.

## What This Changes

The issue asks for the Control UI cron dashboard to tolerate malformed persisted cron jobs and keep showing valid scheduled jobs.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/75970 with store-boundary validation/quarantine that preserves malformed rows for repair plus defensive UI schedule guards.

## Reproduction Assessment

Yes. A persisted cron row with a valid payload but missing or non-object `schedule` can pass the Control UI payload-only filter and then reach `job.schedule.kind` or `formatCronSchedule` on current main.

## Solution Assessment

Yes. Store-boundary quarantine/preservation plus a defensive UI schedule guard is the narrow maintainable fix; broad optional chaining in built artifacts would leave malformed runtime state circulating.
