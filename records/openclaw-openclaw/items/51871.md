## Summary

Current main and the shipped v2026.5.18 release now filter malformed persisted cron rows before they reach cron runtime or Control UI consumers, covering the missing schedule/payload shapes identified in the issue discussion.

## What This Changes

The issue asks for the Control UI cron dashboard to keep displaying valid cron jobs when a persisted jobs.json row has a missing or malformed schedule or payload.

## Best Possible Solution

Keep the shipped store-boundary validation and doctor repair path; any remaining work on the stale linked PR should be handled as PR cleanup rather than keeping this issue open.

## Reproduction Assessment

No current-main failure path remains for the reported malformed jobs.json shape. The older failure was high-confidence from console evidence and comments, but current source filters missing or invalid schedule/payload rows during cron store load before they can reach the dashboard.

## Solution Assessment

Yes. Store-boundary validation with a warning and `openclaw doctor --fix` repair path is the maintainable fix because it prevents malformed persisted rows from circulating into scheduler and UI consumers instead of masking crashes with broad optional chaining.
