## Summary

Close: the Control UI blank cron dashboard path is fixed on current main and shipped in v2026.5.18; malformed persisted cron rows are now rejected at the cron store boundary before UI code can dereference missing schedule or payload fields.

## What This Changes

The issue reports that Control UI cron jobs rendered as an empty dashboard on 2026.3.13 when persisted cron rows had incomplete schedule or payload data.

## Best Possible Solution

Keep the shipped store-boundary validation and retire the now-superseded open fix PR rather than adding UI-only optional chaining around malformed runtime state.

## Reproduction Assessment

Yes for the affected releases: a persisted `jobs.json` row missing a valid schedule or payload shape could blank the cron page. Current main no longer reproduces that path by source inspection because cron store loading skips those rows before list/UI consumers see them.

## Solution Assessment

Yes. Store-boundary validation is the maintainable fix because it prevents malformed persisted rows from circulating into CLI, scheduler, and Control UI consumers while preserving the file for explicit repair.
