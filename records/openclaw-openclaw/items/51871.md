## Summary

Current main and the latest release now skip malformed persisted cron rows before they can enter runtime/UI cron lists, which addresses the dashboard-blank failure described in the issue. The still-open related PR is conflicting and appears superseded for this issue’s central failure mode.

## What This Changes

The issue reports that malformed persisted cron jobs can make the Control UI cron dashboard render blank even though valid cron jobs still run and appear in CLI output.

## Best Possible Solution

Keep the shipped store-boundary skip for malformed persisted rows and handle any extra raw-row preservation desired by the conflicting follow-up PR separately.

## Reproduction Assessment

Yes for the historical bug: the comments and previous source path identify a persisted cron row with missing `schedule` or incomplete `payload` as the failing shape. Current main should no longer reproduce it because invalid persisted rows are filtered during cron store load.

## Solution Assessment

Yes. The shipped fix addresses the durable boundary by keeping malformed persisted rows out of runtime/UI cron consumers instead of adding scattered optional chaining in render paths.
