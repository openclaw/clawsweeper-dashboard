## Summary

Keep this issue open. Current main still has a source-reproducible cron dashboard crash path for a persisted row with a valid payload but missing or malformed schedule, and the linked closing PR is open/unmerged rather than shipped.

## What This Changes

The issue asks for the Control UI cron dashboard to list jobs instead of blanking when persisted cron jobs contain malformed or missing schedule/payload fields.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/75970 with store-boundary validation/quarantine that preserves raw malformed rows for repair, plus defensive UI schedule guards.

## Reproduction Assessment

Yes. A persisted cron row with a valid payload but missing/non-object `schedule` can still pass current UI payload filtering and then hit `job.schedule.kind` in filtering, edit hydration, or rendering on current main.

## Solution Assessment

Yes. Store-boundary quarantine/validation with UI guards is the narrow maintainable direction; broad optional chaining in built assets would be weaker and could leave malformed runtime state circulating.
