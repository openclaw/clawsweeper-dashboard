Keep this issue open. Current main still has a source-reproducible cron dashboard/read failure path for malformed persisted cron rows, and open PR https://github.com/openclaw/openclaw/pull/75970 is the active closing implementation rather than a reason to close before merge.

## What This Changes

The issue asks for Control UI cron dashboards to keep displaying jobs when a persisted cron row is incomplete or missing `schedule`/`payload` fields.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/75970 so malformed persisted cron rows are quarantined or validated at the store boundary, with UI guards as a second line of protection.

## Reproduction Assessment

Yes. A persisted cron row with a valid payload but missing or non-object `schedule` can still pass current UI payload filtering and then hit `job.schedule.kind` in cron read maintenance or Control UI filtering/edit hydration.

## Solution Assessment

Yes. The best path is store-boundary validation/quarantine that preserves raw user data for repair, plus defensive UI guards; the open PR follows that direction.
