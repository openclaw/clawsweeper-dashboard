## Summary

Keep this open: current main and v2026.6.8 still load and report only configured skill roots, while the bundled skill-creator can write to any required `--path`; the remaining behavior needs maintainer direction on scan scope and diagnostic surface.

## What This Changes

Request a guard or diagnostic so skill-like directories created outside configured OpenClaw skill roots are defaulted, warned about, or reported as orphans.

## Best Possible Solution

Add a bounded explicit diagnostic in `openclaw skills check` or doctor that reports orphan `SKILL.md` directories outside configured roots with remediation into canonical roots; only add startup or broad home-directory scanning after maintainer approval.
