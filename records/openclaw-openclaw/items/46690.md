## Summary

Keep open. Current main still loads and reports only documented/configured skill roots, while the requested orphan `SKILL.md` inventory would add a new diagnostic/scaffold policy that needs maintainer decisions on scan scope, redaction, performance, false positives, and intentional external skill layouts.

## What This Changes

The issue requests warnings or diagnostics when skill directories with `SKILL.md` are created outside the documented OpenClaw skill roots.

## Best Possible Solution

Add a bounded orphan-skill diagnostic in `openclaw skills check` or doctor, then align skill-creator warnings once maintainers define scan roots, path redaction, duplicate handling, and override policy.

## Reproduction Assessment

Yes, source-reproducible. A valid `SKILL.md` under an unconfigured directory such as `~/my-new-skill` is outside the documented roots and is not reached by the loader, skills status report, doctor workspace status, or workspace skill audit.

## Solution Assessment

Unclear. A home-depth scan would catch the reported example, but the maintainable solution needs maintainer policy for scan bounds, privacy redaction, performance, duplicate handling, and scaffold override behavior.
