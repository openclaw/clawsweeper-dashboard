## Summary

Keep open. Current main still limits skill loading, listing, checking, doctor status, and security audit paths to documented/configured skill roots, while the bundled skill creator still accepts arbitrary output directories, so the requested orphan-skill guard is not implemented.

## What This Changes

The issue asks OpenClaw to warn about or prevent skill directories created outside supported skill roots.

## Best Possible Solution

Add a bounded orphan-skill diagnostic, likely in `openclaw skills check` and/or doctor, that compares candidate `SKILL.md` folders against documented load roots; align skill-creator defaults or warnings after maintainers choose the scan and override policy.

## Reproduction Assessment

Yes. A source-level reproduction is clear: place a valid `SKILL.md` outside `<workspace>/skills`, `<workspace>/.agents/skills`, `~/.agents/skills`, `~/.openclaw/skills`, bundled/plugin roots, and `skills.load.extraDirs`; current load/status/check paths only enumerate those roots.

## Solution Assessment

Unclear. The broad home-directory or startup scan suggested in the issue would catch the example, but the maintainable fix needs maintainer-approved privacy and performance bounds, probably as an explicit diagnostic before any implicit startup warning.
