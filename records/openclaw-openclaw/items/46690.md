## Summary

Keep open. Current main still documents and enumerates only known/configured skill roots; recent symlink support helps intentional managed/personal layouts, but it does not detect arbitrary orphan `SKILL.md` folders or warn when skill-creator scaffolds outside a load root.

## What This Changes

The issue requests diagnostics and scaffold guardrails for skill folders created outside OpenClaw's canonical/configured skill roots.

## Best Possible Solution

Add a bounded orphan-skill diagnostic in `openclaw skills check` or doctor, then align skill-creator path warnings after maintainers define scan scope, redaction, performance, and override policy.

## Reproduction Assessment

Yes, source-reproducible. A valid `SKILL.md` under an arbitrary directory such as `~/my-skill` is outside the documented roots and is not reached by loader, skills list/check, status, doctor, or the current workspace-only audit paths.

## Solution Assessment

Unclear. The suggested home-depth scan would catch the example, but the maintainable fix needs maintainer decisions on scan bounds, privacy/redaction, performance, and scaffold override behavior.
