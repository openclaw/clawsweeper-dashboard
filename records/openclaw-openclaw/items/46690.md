## Summary

Keep open. Current main still treats arbitrary home-directory skill folders as outside the supported/configured skill roots, and adding orphan-skill discovery or scaffold guardrails is a new diagnostic policy that still needs maintainer direction.

## What This Changes

The issue requests warnings or diagnostics for `SKILL.md` directories created outside canonical skill roots, plus safer defaults for skill scaffolding tools.

## Best Possible Solution

Define a bounded orphan-skill diagnostic for `openclaw skills check` or doctor, update skill-creator guidance/guards for canonical roots, and keep normal skill loading limited to configured roots.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: create a valid `SKILL.md` under an unconfigured directory such as `~/my-skill`, and current loader/status/check paths will not load or report it; I did not create files because this review is read-only.

## Solution Assessment

Unclear. The requested home-depth scan would catch the example, but the best implementation needs maintainer policy for scan scope, privacy redaction, duplicate handling, and any scaffold override flag semantics.
