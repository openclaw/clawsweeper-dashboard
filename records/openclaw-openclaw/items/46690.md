## Summary

Keep open. Current main and the latest release still load and report only configured skill roots; an arbitrary valid SKILL.md directory elsewhere remains invisible, and the remaining fix needs a maintainer decision on safe scan scope and diagnostic surface.

## What This Changes

Request a guard or diagnostic that warns when valid SKILL.md directories are created outside supported OpenClaw skill roots and steers skill creation toward the active workspace.

## Best Possible Solution

Define a bounded orphan-skill diagnostic in openclaw skills check or doctor, keep runtime loading limited to documented roots, and align skill creation flows with the active workspace skills directory.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a valid SKILL.md directory outside every supported skill root is not loaded or reported by skills list, skills check, status, or the current workspace audit.

## Solution Assessment

Unclear. The warning is useful, but maintainers should first choose privacy-safe scan roots, depth limits, redaction, duplicate policy, and whether the surface belongs in skills check, doctor, startup diagnostics, or skill-creator.
