## Summary

Keep open. Current main has staged npm installs, dist-inventory verification/pruning, post-update doctor, and immediate no-deferral `update.run` restarts, but the live Gateway RPC path still swaps the global package root before the Gateway restart is scheduled, so the reported macOS npm-global TUI hang is not proven fixed.

## What This Changes

The issue asks OpenClaw to make `openclaw gateway update.run` upgrades recover safely instead of leaving TUI WebSocket handshakes hung by missing hashed dist files.

## Best Possible Solution

Make package-manager `update.run` fail-safe by either refusing live Gateway RPC package swaps with clear shell recovery guidance, or handing off to a detached updater that stops/restarts the service and verifies `/health` plus TUI connectivity before success.

## Reproduction Assessment

Yes, source-level. The report has concrete macOS/npm-global/LaunchAgent steps and current main still exposes the `update.run` to global package swap path, but I did not establish a live current-main macOS failure.

## Solution Assessment

Unclear. Staged installs and dist inventory checks address stale files, but the safest maintainable fix likely needs a maintainer decision between blocking RPC package updates and adding an external handoff updater.
