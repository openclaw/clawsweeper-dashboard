## Summary

Keep open. Current main has useful stale-dist mitigations, but the control-plane package update path still performs the package-manager update from inside the live Gateway handler and only schedules restart after the package swap, so the reported macOS npm-global TUI hang is not proved fixed.

## What This Changes

The issue asks for `openclaw gateway update.run` to safely update npm-global installs so `openclaw tui` reconnects instead of hanging after an upgrade.

## Best Possible Solution

Make package-manager `update.run` service-safe by handing off outside the live Gateway process or refusing live package swaps with clear recovery guidance, then verify `/health` and `openclaw tui` after a macOS LaunchAgent upgrade.

## Reproduction Assessment

Yes, source-reproducible. The issue gives a concrete macOS npm-global LaunchAgent upgrade path, and current main still contains the live Gateway `update.run` path that replaces the package tree before restart; I did not run the macOS upgrade live.

## Solution Assessment

Unclear. The current staged install, dist inventory verification, and immediate restart are good guardrails, but they do not prove that replacing the active package tree from the live Gateway request path is the narrowest maintainable fix.
