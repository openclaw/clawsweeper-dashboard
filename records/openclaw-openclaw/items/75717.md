## Summary

Keep open. This is a fresh, concrete latest-release regression, and current main still allows the live Gateway control-plane `update.run` path to run package-manager replacement from inside the running Gateway before restart. Existing staged npm install, dist-inventory verification, and hard-respawn mitigations are relevant, but they do not prove the reported macOS npm-global LaunchAgent TUI hang is fixed.

## What This Changes

Make `openclaw gateway update.run` safe after 2026.4.29 npm-global upgrades so TUI sessions connect instead of hanging on missing hashed Control UI chunks and WebSocket handshake timeouts.

## Best Possible Solution

Make package-manager `update.run` fail-safe for managed Gateway installs: either hand off to an external updater that stops the managed service before replacing the package tree, or refuse live package-manager swaps with a clear recovery path, then verify post-restart health and TUI connection against the packaged artifact.

## Reproduction Assessment

Yes. The issue provides a high-confidence reproduction path: macOS npm-global LaunchAgent install, upgrade from 2026.4.11 to 2026.4.29 via `openclaw gateway update.run`, then run `openclaw tui` and inspect Gateway logs for the missing hashed Control UI file plus WebSocket handshake timeout.

## Solution Assessment

Unclear. The proposed pre-clean step is plausible but not the best standalone fix because current main already stages and verifies npm installs; the safer solution likely needs an out-of-process handoff or explicit refusal of live package-manager swaps.
