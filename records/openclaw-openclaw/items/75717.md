## Summary

Keep open. Current main has useful staged dist-inventory verification and a separate shell-update guard, but the Gateway RPC `update.run` path still runs the package updater inside the live Gateway handler and schedules restart only after a successful package swap, so the reported npm-global TUI handshake hang is not proved fixed.

## What This Changes

The issue reports that upgrading from 2026.4.11 to 2026.4.29 via Gateway `update.run` can leave missing hashed dist files and make all TUI WebSocket handshakes hang until rollback.

## Best Possible Solution

Make package-manager `update.run` fail-safe by moving package replacement outside the live Gateway process or refusing that RPC path with clear shell-update recovery guidance, then prove `/health` and `openclaw tui` after a managed macOS npm-global upgrade.

## Reproduction Assessment

Yes. The report gives a high-confidence reproduction path with version range, platform, install method, logs, and rollback result; current source still contains the Gateway RPC package-swap shape needed to exercise the failure mode.

## Solution Assessment

Unclear. Current staged verification and the #75729 shell-update guard are useful partial mitigations, but the narrow maintainable fix likely needs to address package-manager replacement from the Gateway control-plane `update.run` path itself.
