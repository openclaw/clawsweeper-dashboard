## Summary

Keep open. The report is fresh, concrete, and targets the latest shipped release; current main has several update hardening layers, but it is not proven to fix this managed macOS npm-global `update.run` failure because the live Gateway still performs the package-manager update before scheduling the restart.

## What This Changes

The issue asks for `update.run` to stop leaving npm-global managed Gateway installs with stale or missing hashed dist files that make `openclaw tui` WebSocket handshakes time out after upgrading to 2026.4.29.

## Best Possible Solution

Make package-manager `update.run` fail-safe for managed Gateway installs by handing off to an external updater that stops or restarts the service before replacing the package tree, or by refusing live package swaps with a clear recovery path. The finished fix should validate packaged dist inventory and post-restart `/readyz` plus TUI connectivity on a macOS LaunchAgent npm-global upgrade.

## Reproduction Assessment

Yes. The issue provides a high-confidence reproduction path with concrete versions, install method, macOS service mode, observed TUI status, handshake timeout, `/health` failure, and missing dist-module log; I did not run it because this review was read-only.

## Solution Assessment

Unclear. The suggested pre-clean is plausible but not clearly sufficient because current main already has staged npm installs, dist inventory verification, postinstall pruning, and special update respawn; the safer solution likely needs an out-of-process managed-service handoff or an explicit no-live-swap policy.
