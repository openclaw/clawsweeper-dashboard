## Summary

Keep open. This is a fresh, concrete latest-release regression, and current main still contains the live control-plane package-update shape that can replace the global package tree before the running Gateway exits; existing staging, dist-inventory verification, and update-respawn mitigations are relevant but not enough to prove the TUI hang and missing Control UI chunk are fixed.

## What This Changes

The issue reports that upgrading an npm-global managed Gateway from 2026.4.11 to 2026.4.29 via `update.run` leaves `openclaw tui` unable to complete the Gateway WebSocket handshake because the Gateway crashes on a missing hashed `dist/control-ui` file.

## Best Possible Solution

Make package-manager `update.run` fail-safe for managed Gateway installs: either hand off to an external updater that stops the managed service before replacing the package tree, or refuse live package-manager swaps with a clear recovery path, then verify post-restart health and TUI connection against the packaged artifact.

## Reproduction Assessment

Yes. The issue provides a high-confidence reproduction path: upgrade a macOS npm-global LaunchAgent install from 2026.4.11 to 2026.4.29 via `update.run`, then run `openclaw tui` and inspect Gateway logs for the missing hashed Control UI file plus WebSocket handshake timeout.

## Solution Assessment

Unclear. The proposed pre-clean is plausible but not sufficient as the best standalone fix, because current main already uses staged npm install plus dist inventory verification; the safer solution likely needs an out-of-process handoff or an explicit refusal of live package-manager swaps.
