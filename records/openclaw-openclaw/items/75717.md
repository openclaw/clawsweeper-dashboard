## Summary

Current main now fixes the reported stale dist chunk/update-run failure mode: package-manager `update.run` swaps are verified from a staged install and then force an immediate no-deferral/no-cooldown Gateway restart, so the old process is not left serving or lazy-loading from a replaced package tree.

## What This Changes

The issue reports that upgrading via `openclaw gateway update.run` to v2026.4.29 left stale hashed `dist` files behind, causing Gateway health failures and TUI WebSocket handshake timeouts.

## Best Possible Solution

Keep the current staged install/dist-inventory verification plus immediate no-cooldown Gateway handoff for package-manager `update.run`; open a fresh issue only if a current-main or post-fix release still reproduces TUI handshakes after upgrade.

## Reproduction Assessment

Yes. The issue gives a concrete macOS npm-global LaunchAgent upgrade path and logs for `/health` 500, missing hashed `dist/control-ui-*` import, and WebSocket handshake timeout; related #75206 reproduced the same stale-chunk class on another update path.

## Solution Assessment

Yes. The current fix addresses the root update lifecycle window without adding runtime import fallbacks: verify a clean staged package, swap it, then restart the old Gateway process immediately instead of letting it lazy-load removed chunks.
