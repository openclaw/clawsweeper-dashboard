## Summary

Keep open. Current main has useful stale-dist verification and pruning, but the control-plane `update.run` path still performs package-manager updates from inside the live Gateway process and swaps the active global package root before the Gateway restart is scheduled, so the reported macOS npm-global TUI hang is not proven fixed.

## What This Changes

The issue reports that upgrading a macOS LaunchAgent npm-global install with `openclaw gateway update.run` leaves TUI WebSocket handshakes hanging after a missing hashed `dist/control-ui-*` file error.

## Best Possible Solution

Make package-manager `update.run` fail-safe by refusing live Gateway RPC package swaps with clear shell recovery guidance, or by handing off to a detached service-aware updater that verifies post-restart `/health` and TUI connectivity.

## Reproduction Assessment

Yes, source-level. The report provides a detailed macOS npm-global LaunchAgent path, and current main still contains the server-side `update.run` to global package-root swap path needed to exercise the failure mode.

## Solution Assessment

Unclear. Staged clean installs and dist inventory checks are good guardrails, but the safest maintainable fix likely needs a maintainer decision between blocking RPC package updates and adding an external handoff updater.
