## Summary

Keep open. Current main has useful mitigations, especially #75729 for shell `openclaw update`, but the Gateway RPC `update.run` path still performs the package-manager update inside the live Gateway handler and swaps the global package tree before scheduling restart, so the reported macOS npm-global TUI/WebSocket failure is not proved fixed.

## What This Changes

The issue asks OpenClaw to prevent npm-global `openclaw gateway update.run` upgrades from leaving Gateway/TUI sessions broken by stale or missing hashed `dist/control-ui` chunks.

## Best Possible Solution

Make package-manager `update.run` fail-safe by handing off to an updater outside the live Gateway process or refusing the RPC with clear shell-update recovery guidance, while preserving staged dist verification.

## Reproduction Assessment

Yes. The report provides concrete upgrade steps and logs, and current main still has the same source-level shape needed to exercise the failure mode: live Gateway RPC calls `runGatewayUpdate()`, swaps the package root, then restarts.

## Solution Assessment

Unclear. Staged install verification and immediate restart reduce stale-file risk, but the narrow maintainable fix likely needs to remove or explicitly reject live package-manager swaps from the Gateway control-plane path.
