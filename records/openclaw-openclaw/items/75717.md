## Summary

Current main already addresses the central 2026.4.29 `update.run` failure: npm global updates install into a clean staged prefix, verify the packaged `dist` inventory, swap only after verification, and force an immediate restart for package swaps. The proof commit is on main only; no release tag contains it yet, so affected `v2026.4.29` users still need the downgrade workaround or a newer build.

## What This Changes

The issue reports that `openclaw gateway update.run` from 2026.4.11 to 2026.4.29 leaves stale hashed Control UI `dist` files, causing gateway 500s and TUI WebSocket handshake timeouts for all agents.

## Best Possible Solution

Keep the staged package-update implementation as the source of truth: install into a clean temporary prefix, verify the full packaged `dist` inventory, swap atomically, and restart only after a successful package swap.

## Reproduction Assessment

Yes. The issue has a high-confidence reproduction path via `openclaw gateway update.run` from 2026.4.11 to 2026.4.29 plus concrete missing hashed `dist` logs; this read-only review verified the fix through current source and tests rather than running a live macOS global npm upgrade.

## Solution Assessment

Yes. The current-main approach is the narrow maintainable fix because it avoids cleaning a live package tree in place, fails before restart when packaged `dist` is inconsistent, and prevents the old gateway process from lazy-loading removed chunks after a package swap.
