## Summary

Keep open. Current main has useful stale-dist and staged-install mitigations, but the live Gateway `update.run` path still awaits a package-manager update in the running Gateway process and only schedules restart after the package tree has already been swapped, so the reported macOS npm-global TUI hang is not proved fixed.

## What This Changes

The issue requests fixing an upgrade regression where `openclaw gateway update.run` to 2026.4.29 can leave missing or stale dist chunks and make `openclaw tui` WebSocket handshakes hang.

## Best Possible Solution

Make package-manager `update.run` fail-safe by handing package replacement to an updater outside the live Gateway process, or by refusing live package swaps with clear shell-update recovery guidance and post-restart health/TUI verification.

## Reproduction Assessment

Yes. The issue provides a high-confidence macOS npm-global LaunchAgent upgrade path, and current source still contains the live Gateway package-swap path needed to exercise the failure mode; I did not run the macOS upgrade locally.

## Solution Assessment

Unclear. The current staged-install verification, stale-dist pruning, and immediate restart are useful, but the safer maintainable direction is to avoid replacing the active package tree from inside the live Gateway request path.
