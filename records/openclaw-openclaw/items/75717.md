## Summary

Keep open. Current main has useful mitigations for adjacent stale-dist and shell-update paths, but the reported live Gateway `update.run` package-manager path still swaps the package tree from inside the running Gateway before scheduling restart, so the macOS npm-global TUI hang is not proved fixed.

## What This Changes

The issue reports that upgrading a macOS npm-global LaunchAgent install from 2026.4.11 to 2026.4.29 via `openclaw gateway update.run` leaves all TUI sessions hanging on WebSocket handshake timeouts after missing hashed dist chunks.

## Best Possible Solution

Make package-manager `update.run` fail-safe by refusing or handing off live Gateway package swaps to a service-aware updater, then prove restart health and TUI reconnect on a real npm-global upgrade.

## Reproduction Assessment

Yes. The issue provides a high-confidence macOS npm-global LaunchAgent upgrade path, and current main still contains the live `update.run` package-swap shape needed to exercise the failure mode.

## Solution Assessment

Unclear. Staged dist verification and the merged shell-update guard reduce adjacent failures, but they do not appear to be the narrowest complete solution for package-manager updates triggered from the live Gateway control plane.
