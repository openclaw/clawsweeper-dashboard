## Summary

Keep open. Current main has useful mitigations, but the live Gateway `update.run` path still replaces the npm package tree inside the request handler and only schedules restart after the swap, so the reported macOS npm-global TUI hang is not proved fixed.

## What This Changes

Prevent `openclaw gateway update.run` npm-global upgrades from leaving the Gateway/TUI broken by stale or missing packaged `dist` chunks after upgrading to v2026.4.29.

## Best Possible Solution

Make package-manager `update.run` fail-safe by handing off to a service-aware updater outside the live Gateway process, or by refusing live package swaps with clear shell-update recovery guidance and post-restart health/TUI verification.

## Reproduction Assessment

Yes. The issue provides a high-confidence macOS npm-global LaunchAgent upgrade path, and current source still contains the live RPC package-swap shape needed to exercise the failure mode.

## Solution Assessment

Unclear. The reporter's pre-clean idea and current staged verification address stale files, but the narrower maintainable fix likely needs to avoid or explicitly reject package-manager replacement from the live Gateway control-plane path.
