## Summary

Keep open. Current main has staged npm installs, dist-inventory verification, and immediate restart guardrails, but the Gateway RPC `update.run` path still performs the package-manager swap inside the live Gateway request before scheduling restart, so the reported macOS npm-global TUI hang is not proven fixed.

## What This Changes

The issue reports that `openclaw gateway update.run` from 2026.4.11 to 2026.4.29 can leave TUI sessions hanging with WebSocket handshake timeouts and missing hashed `dist` chunks after a macOS npm-global upgrade.

## Best Possible Solution

Make package-manager `update.run` fail-safe by either refusing live Gateway RPC package replacement with clear shell recovery guidance, or handing off to a detached service-aware updater that verifies `/health` and TUI connectivity after restart.

## Reproduction Assessment

Yes, source-level: the report gives detailed macOS npm-global LaunchAgent steps, and current main still contains the server-side `update.run` path that can swap a global package root from inside the live Gateway RPC transaction. I did not run a live macOS package-artifact reproduction.

## Solution Assessment

Unclear. The staged install and dist-inventory checks are good mitigations for stale hashed files, but the narrowest maintainable fix for this report likely needs a maintainer decision on whether control-plane package updates should be blocked or delegated outside the live Gateway process.
