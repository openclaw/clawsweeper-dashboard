## Summary

Keep open. Current main has staged npm install and dist-inventory verification, but the live Gateway `update.run` path still runs the package update from the request handler and schedules restart only after the package swap, so the reported TUI hang/missing chunk failure mode is not proved fixed.

## What This Changes

The issue asks OpenClaw to make npm-global `update.run` upgrades safe after v2026.4.29 so TUI reconnects instead of hanging on WebSocket handshake timeout after a missing hashed Control UI dist chunk.

## Best Possible Solution

Make package-manager `update.run` fail-safe for managed Gateway installs by handing off to a service-aware external updater before replacing the package tree, or by refusing live package-manager swaps with a clear shell-update recovery path and post-restart health/TUI verification.

## Reproduction Assessment

Yes. The issue provides a high-confidence reproduction path for macOS npm-global LaunchAgent installs, and current code still contains the live request-handler package-swap shape needed to exercise it.

## Solution Assessment

Unclear. The proposed pre-clean is only a partial fix because current main already stages and verifies npm installs; the safer solution is to avoid or explicitly reject package-manager swaps from the live Gateway control-plane path.
