## Summary

Keep open. Current main has useful stale-dist and service-update guardrails, but the reported control-plane package update still runs `runGatewayUpdate()` inside the live Gateway request handler and swaps the active npm package tree before scheduling restart, so the macOS npm-global TUI hang is not proved fixed.

## What This Changes

The issue asks for `openclaw gateway update.run` to stop leaving upgraded npm-global macOS gateways and TUI clients hung on WebSocket handshake timeouts from stale or missing bundled dist files.

## Best Possible Solution

Make package-manager `update.run` fail-safe by handing off to a service-aware updater outside the live Gateway process, or by refusing live package swaps with clear shell-update recovery guidance and post-restart health/TUI verification.

## Reproduction Assessment

Yes. The issue provides a high-confidence macOS npm-global LaunchAgent upgrade path, and current source still contains the server-side `update.run` to package-root swap path needed to exercise the failure mode.

## Solution Assessment

Unclear. The current stale-dist verification and pruning are useful, but they do not settle whether replacing the active package tree from the live Gateway request path is safe; an external handoff or explicit refusal is the safer direction.
