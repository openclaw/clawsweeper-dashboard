Full review comments:

- **[P2] Clear stale systemd substate when is-active wins:** `src/daemon/systemd.ts:896`
  - body: When `systemctl show` returns `ActiveState=failed`/`SubState=failed` but `is-active --quiet` succeeds, this sets status/state to running/active while still returning the stale `subState`. `formatRuntimeStatus` always prints `subState`, so `openclaw status` can still show a running service with `sub failed`, preserving the misleading failed text this patch is meant to remove. Clear or replace stale `subState` when the probe overrides a non-active show state, and assert that in the regression test.
