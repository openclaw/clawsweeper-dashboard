## Summary

Keep open. Current main still lacks WhatsApp 428-specific handling in the opening and post-open close paths, and open PR #75773 is already the linked implementation candidate.

## What This Changes

Reports a WhatsApp Web regression where HTTP 428 `Precondition Required / Connection Terminated` after upgrading to 2026.4.29 exits or restarts the channel without clear relink handling.

## Best Possible Solution

Review and land or replace PR #75773 with WhatsApp-owned 428 handling for both opening errors and listener close decisions, plus regression tests that preserve existing 401 and 440 behavior.

## Reproduction Assessment

Yes for a source-level path: inject a Boom-like status 428 rejection from `waitForWaConnection` or a listener close with status 428, and current main rethrows or retries instead of stopping with relink guidance. The recurring live timing is supported by issue comments but was not live-reproduced in this read-only pass.

## Solution Assessment

No for current main. The linked PR direction is the narrowest maintainable approach because it keeps the fix in the WhatsApp monitor/controller boundary rather than adding core channel-manager policy.
