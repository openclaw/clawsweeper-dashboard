## What This Changes

The issue reports recurring WhatsApp channel exits with HTTP 428 after upgrading to v2026.4.29, leaving intermittent no-listener windows for outbound sends.

## Best Possible Solution

Keep this issue open until #75773 or an equivalent WhatsApp-local fix lands. The end state should handle 428 in the WhatsApp monitor/controller path for both opening and post-open closes, add focused regression coverage, preserve 401 logged-out and 440 conflict behavior, and include the required user-facing changelog entry if behavior changes.
