## Summary

Keep open. Current main still has a credible Homebrew/package-managed gateway auto-update failure path, and the linked open PR with closing syntax should be reviewed, landed, or replaced before this issue is closed.

## What This Changes

Homebrew users report that the gateway detects a package update with auto-update enabled but never applies it after the stable delay.

## Best Possible Solution

Define the Homebrew auto-update contract explicitly, then land either a safe updater that runs outside the live gateway service or a Homebrew skip-and-hint path with aligned docs, tests, and changelog.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a Homebrew/package-managed gateway service with a due stable auto-update: startup spawns the update CLI from the service environment, and the package update guard refuses replacement from the live gateway service path.

## Solution Assessment

Unclear. PR #74870 is a plausible narrow fix if maintainers want Homebrew to be manual with a clear hint, but it intentionally changes the user expectation from automatic install to a documented skip path rather than implementing an external Homebrew updater.
