## Summary

Keep open. Current main still has a credible Homebrew/package-managed gateway auto-update failure path, and the linked open PR with closing syntax should be reviewed, landed, or replaced before this issue is closed.

## What This Changes

Report requests that Homebrew-installed gateways with auto-update enabled install a detected stable update after the configured delay instead of remaining on the old version.

## Best Possible Solution

Define the Homebrew auto-update contract, then land a corrected Homebrew-aware updater or skip-and-hint path with aligned docs, tests, and changelog.

## Reproduction Assessment

Yes, from source-level inspection on current main. A due package auto-update spawned from a gateway service inherits service env and reaches the package-update guard that blocks updates from inside the gateway service; I did not run a live Homebrew gateway.

## Solution Assessment

Unclear. The linked PR's skip-and-hint direction is plausible if Homebrew remains manual, but maintainers still need to decide whether Homebrew installs should auto-run brew or intentionally skip with explicit guidance.
