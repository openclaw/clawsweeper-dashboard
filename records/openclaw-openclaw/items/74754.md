## Summary

Keep open. Current main still has a credible Homebrew/package-managed auto-update failure path, and open PR #74870 is already the active closing implementation candidate, so closing this issue before that PR is reviewed or merged would orphan a valid report.

## What This Changes

The issue requests that a Homebrew-installed gateway with auto-update enabled actually install a detected stable update after the configured delay and restart itself.

## Best Possible Solution

Resolve the Homebrew auto-update contract explicitly: either implement a safe Homebrew-aware apply flow outside the live gateway service process, or land the skip-and-hint approach from PR #74870 with aligned docs, tests, and changelog coverage.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a package-managed gateway service with auto-update due: the auto-update child inherits gateway service markers, then `openclaw update` refuses package replacement inside that service process.

## Solution Assessment

Unclear. PR #74870 is a plausible narrow fix if maintainers choose manual `brew upgrade openclaw` guidance for Homebrew installs, but it does not implement the originally expected auto-install behavior.
