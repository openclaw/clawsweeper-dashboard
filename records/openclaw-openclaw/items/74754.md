## Summary

Keep open. Current main still has a credible Homebrew auto-update gap, and PR #74870 is an open, unmerged implementation candidate with closing syntax for this issue.

## What This Changes

The issue asks for Homebrew-managed gateway auto-update to install a detected newer stable OpenClaw release after the configured delay.

## Best Possible Solution

Define the Homebrew update contract explicitly: either land a Homebrew-aware skip-and-hint path with docs/tests/changelog, or implement a safe updater that runs outside the live gateway service and can replace the Homebrew-managed install.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a Homebrew-managed package install with auto-update due after `stableDelayHours` plus jitter: current startup spawns the package update CLI, whose global update path only supports npm, pnpm, and bun rather than `brew upgrade openclaw`.

## Solution Assessment

Unclear. PR #74870 is a plausible narrow fix if maintainers choose manual Homebrew upgrade hints, but automatic Homebrew installation would need a different outside-service apply design.
