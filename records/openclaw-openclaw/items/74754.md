## Summary

Keep open. Current main still has the credible Homebrew/package-managed gateway auto-update failure path, and the linked open PR with closing syntax should be reviewed, landed, or replaced before this issue is closed.

## What This Changes

The issue reports that Homebrew-installed gateways detect an available stable update but never auto-install it after the configured delay.

## Best Possible Solution

Establish one Homebrew-managed updater contract: either safely invoke Homebrew outside the live gateway service or explicitly skip auto-apply with a one-time brew upgrade hint, with tests, docs, and changelog aligned.

## Reproduction Assessment

Yes, at source level. A package-managed gateway with a due stable auto-update spawns the update CLI from the service environment, and the package update guard can refuse replacement from the live gateway service path.

## Solution Assessment

Unclear. PR #74870 is a plausible narrow fix if maintainers want Homebrew auto-update to become a documented skip-and-hint path, but it does not implement the originally documented auto-apply behavior.
