## Summary

Keep open. Current main still has a credible Homebrew/package-managed gateway auto-update failure path, and open PR #74870 is the active closing implementation candidate rather than a merged fix.

## What This Changes

This issue requests that Homebrew-installed gateway auto-update actually install a detected newer stable version after the configured rollout delay.

## Best Possible Solution

Define the Homebrew update contract explicitly: either land a Homebrew-aware skip-and-hint path with aligned tests/docs/changelog, or implement a safe updater that runs outside the live gateway service before replacing a package-managed install.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a Homebrew/package-managed gateway service with auto-update due after delay plus jitter: startup spawns the update CLI from the service environment, and the update command blocks package replacement from that live service process.

## Solution Assessment

Unclear. The linked PR is a plausible narrow fix if maintainers choose manual Homebrew upgrade hints, but it intentionally changes the user expectation from automatic install to explicit Homebrew-managed skip behavior.
