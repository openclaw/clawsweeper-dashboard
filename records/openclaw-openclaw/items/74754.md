## Summary

Keep open. Current main still has the credible Homebrew/package-managed auto-update failure path, and open PR #74870 references this issue with closing syntax, so the issue should stay open until that PR lands or maintainers choose a different update contract.

## What This Changes

The issue requests that a Homebrew-installed gateway automatically install a detected stable update after the configured rollout delay.

## Best Possible Solution

Define the Homebrew update contract explicitly: either land the Homebrew-aware skip-and-hint behavior with aligned docs/tests/changelog, or implement a safe updater flow that runs outside the live gateway service before replacing a package-managed install.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a package-managed gateway service with auto-update due: the runner launches the update CLI from the service-marked environment, and the update command refuses package replacement inside that service process.

## Solution Assessment

Unclear. The linked PR is a plausible narrow fix if maintainers choose manual Homebrew upgrade hints, but it intentionally does not implement the originally expected automatic install behavior.
