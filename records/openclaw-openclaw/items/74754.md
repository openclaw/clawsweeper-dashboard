## What This Changes

The issue reports that a Homebrew-installed gateway detects a newer package version but never applies the configured automatic stable update.

## Best Possible Solution

Yes. A high-confidence source-level reproduction is a Homebrew/package-managed gateway service with a due stable auto-update: startup spawns the update CLI from the service environment, and the package update guard refuses replacement from the live gateway service path.

## Solution Assessment

Unclear. PR #74870 is a plausible narrow skip-and-hint fix if Homebrew should remain manually upgraded, but implementing a safe external Brew updater would better match the currently documented auto-install expectation.
