## Summary

Keep open. Current main still has the Homebrew/package-managed auto-update failure path, and open PR #74870 remains unmerged with closing syntax for this issue, so cleanup should wait for that PR or a maintainer-chosen update contract.

## What This Changes

The issue asks OpenClaw to auto-install a detected newer stable version for Homebrew installs after the configured rollout delay instead of only recording update availability.

## Best Possible Solution

Define the Homebrew update contract explicitly, then align code, docs, tests, and changelog around either a Homebrew-aware skip-and-hint path or a safe out-of-process package updater.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a package-managed gateway service with auto-update due: the runner launches the update CLI from the service-marked environment, and the update command refuses package replacement inside that service process.

## Solution Assessment

Unclear. PR #74870 is a plausible narrow fix if maintainers choose manual Homebrew upgrade hints, but it intentionally does not implement the originally expected automatic install behavior.
