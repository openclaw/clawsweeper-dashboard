## Summary

Keep open. Current main still has a credible package-managed gateway auto-update failure path, and the linked open PR that targets this issue should be reviewed, landed, or replaced before cleanup closes it.

## What This Changes

The issue reports that a Homebrew-managed gateway detects a newer stable OpenClaw version but never auto-installs it after the configured auto-update delay.

## Best Possible Solution

Define the Homebrew auto-update contract explicitly, then land a safe updater that runs outside the live gateway service or a documented Homebrew skip-and-hint path with aligned tests and changelog.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a package-managed gateway service with a due auto-update launches the update CLI from the live service process, and the CLI refuses unsafe in-service package replacement.

## Solution Assessment

Unclear. PR #74870 is a plausible narrow fix if maintainers want Homebrew to be manual with a clear hint, but it changes the expectation from automatic install to an explicit skip path rather than implementing external Homebrew updating.
