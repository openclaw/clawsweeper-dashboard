## Summary

Keep open. Current main still has the credible Homebrew/package-managed auto-update failure path, and the linked open PR #74870 should be reviewed or replaced before this issue is closed.

## What This Changes

Homebrew-installed gateways should automatically apply stable package updates after the configured delay instead of only recording available-update state.

## Best Possible Solution

Define the Homebrew update contract explicitly, then land or replace PR #74870 with aligned code, tests, docs, and changelog before closing this issue.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a Homebrew/package-managed gateway service with auto-update due after delay plus jitter: startup spawns the update CLI from the service environment, and the update command blocks package replacement from that live service process.

## Solution Assessment

Unclear. PR #74870 is a plausible narrow fix if maintainers choose manual Homebrew upgrade hints, but it intentionally changes the expectation from automatic install to an explicit Homebrew-managed skip path.
