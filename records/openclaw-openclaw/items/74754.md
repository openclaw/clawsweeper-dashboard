## Summary

Keep open. Current main still lacks a Homebrew-specific auto-update contract, and PR #74870 uses closing syntax for this issue but remains open and unmerged, so the issue should track that PR or a replacement until it lands.

## What This Changes

The issue asks the gateway auto-updater to install a detected stable update for Homebrew-installed OpenClaw after `stableDelayHours` instead of only recording availability.

## Best Possible Solution

Define and document the Homebrew-managed update contract, then land PR #74870 or a replacement that implements that contract with aligned runtime logs, docs, tests, and changelog coverage.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a Homebrew Cellar/opt package root with auto-update due: current main invokes the generic `openclaw update` path and has no Homebrew detection, skip, or `brew upgrade` apply path.

## Solution Assessment

Unclear. The linked skip-and-hint PR is a plausible narrow fix if maintainers choose manual Homebrew upgrades, but it changes the user expectation from automatic install to explicit Homebrew-managed guidance.
