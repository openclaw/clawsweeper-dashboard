## Summary

Keep open. Current main still has a credible package-managed gateway auto-update failure path, and open PR #74870 already references this issue with closing syntax, so this should remain open until that PR lands or maintainers choose a different Homebrew update contract.

## What This Changes

The issue reports that Homebrew-installed gateways detect a newer OpenClaw version but never perform the configured automatic update after the stable delay.

## Best Possible Solution

Resolve the Homebrew auto-update contract explicitly: either land the Homebrew-aware skip-and-hint behavior with aligned docs/tests/changelog, or implement a safe updater flow that runs outside the live gateway service before replacing a package-managed install.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a package-managed gateway service with auto-update due: the runner launches the update CLI, the child inherits gateway service markers, and the update command refuses package replacement inside that service process.

## Solution Assessment

Unclear. PR #74870 is a plausible narrow fix if maintainers choose manual Homebrew upgrade hints, but it intentionally does not implement the originally expected automatic install and restart behavior.
