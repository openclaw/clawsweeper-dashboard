## Summary

Keep open. Current main still has a credible Homebrew/package-managed auto-update failure path, and an open PR explicitly uses closing syntax for this issue, so the issue should remain open until that implementation path is reviewed and merged or replaced.

## What This Changes

The issue requests that gateway auto-update actually install a detected stable update for Homebrew-managed OpenClaw installations after the configured stable delay.

## Best Possible Solution

Define the Homebrew update contract explicitly: either land the Homebrew-aware skip-and-hint behavior with aligned tests/changelog/docs, or implement a safe updater flow that runs outside the live gateway service before replacing a package-managed install.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a package-managed gateway service with auto-update due: the startup runner spawns the update CLI from the service environment, and the update command refuses package replacement inside that service process.

## Solution Assessment

Unclear. The linked PR is a plausible narrow fix if maintainers choose manual Homebrew upgrade hints, but it intentionally changes the expectation from automatic install to an explicit Homebrew-managed skip path.
