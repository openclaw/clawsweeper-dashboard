## Summary

Keep open: this PR has a protected maintainer label and the latest head still has a P2 functional blocker where narrowed plugin/group allowlists can drop auth-gated plugin tools.

## What This Changes

The PR refactors embedded-runner tool allowlist planning, narrows constructed tool families, updates update_plan and bundled MCP/LSP gating, and adds tests plus a changelog entry.

## Best Possible Solution

Keep the construction-plan refactor, but make plugin-only tool resolution receive the same auth profile context as the normal OpenClaw tool factory and land with auth-gated allowlist coverage.

## Reproduction Assessment

Yes, for the review finding by source inspection. The latest PR head routes group/plugin-only allowlists through the plugin-only resolver while that options object omits authProfileStore, and the resolver only honors profile-backed auth when that store is present.

## Solution Assessment

No. The PR direction is narrow and maintainable, but it is not complete until the plugin-only path forwards authProfileStore and covers auth-gated group/plugin allowlists.
