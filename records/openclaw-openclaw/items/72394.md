## Summary

Keep open. Current main still has the isolated-heartbeat queue boundary: preflight peeks base-session events, the run switches to the `:heartbeat` session key, and active-session event inspection reads that isolated key without transferring queued base events; changing this needs a contract decision because existing trust and routing tests rely on some base-session events not crossing.

## What This Changes

Request transfer or explicit visibility of pending base-session system event entries when `heartbeat.isolatedSession` switches the heartbeat run to the `:heartbeat` session key.

## Best Possible Solution

Define and document the isolated-heartbeat event visibility contract, then implement selective transfer or explicit base-key lookup with regression coverage for trust, delivery-context, and single-consumption semantics.

## Reproduction Assessment

Yes, by source inspection. Enqueue a system event on the base heartbeat session, enable `heartbeat.isolatedSession`, and active-session consumers that call `peekSystemEventEntries` on the `:heartbeat` key will read a separate empty queue because queues are session-key scoped and no transfer is performed.

## Solution Assessment

Unclear. Mirroring all entries is plausible, but existing tests intentionally keep some base-session hook and untrusted events from active isolated classification, so the maintainable fix needs a maintainer-owned visibility contract first.
