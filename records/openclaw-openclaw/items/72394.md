- `impact:session-state`: Pending event state is visible on the base session key but not on the isolated heartbeat run key.

## Summary

Keep open. Current main still peeks pending system events on the base heartbeat session, switches the model run to the isolated `:heartbeat` key, and consumes selected events from the base queue after the run; that leaves no general active-session event visibility contract for future isolated-session consumers.

## What This Changes

The issue requests transferring or sharing pending system event entries when heartbeat crosses from a base session into an isolated heartbeat session.

## Best Possible Solution

Define one explicit heartbeat system-event visibility contract, then update the runner and focused regression tests so base and isolated session keys behave predictably without preserving stale routing context.

## Reproduction Assessment

No high-confidence live reproduction is established. Source inspection does reproduce the latent invariant: preflight peeks base-session events, the model run switches to `:heartbeat`, and general isolated-session queue readers would see an empty queue.

## Solution Assessment

Unclear as an implementation choice. The issue's clone/move and shared-read options are both plausible, but maintainers should choose the durable session/event ownership contract before implementation.
