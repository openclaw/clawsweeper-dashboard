## Summary

Keep open. Current main still peeks base-session events before isolation but only peeks the active `:heartbeat` queue after isolation; there is no generic transfer or alias, and existing tests encode trust-sensitive exceptions.

## What This Changes

The issue asks OpenClaw to make pending base-session system-event entries visible across the isolated heartbeat session boundary.

## Best Possible Solution

Define a documented pre-isolation event visibility rule, then cover transfer or explicit base-key reads with tests for ordering, single consumption, duplicates, delivery context, and untrusted-event handling.

## Reproduction Assessment

Yes. A focused regression can enqueue an event on the base heartbeat session, run an isolated heartbeat, and assert that `peekSystemEventEntries` on the `:heartbeat` key remains empty while active consumers read that key.

## Solution Assessment

Unclear. Mirroring the session-entry transfer is plausible, but current code intentionally treats some base-session events differently under isolated sessions, so maintainers should choose the trust and delivery semantics first.
