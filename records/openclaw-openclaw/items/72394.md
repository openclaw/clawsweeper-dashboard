## Summary

Keep open. Current main still leaves pending base-session system-event entries key-scoped when an isolated heartbeat switches to the `:heartbeat` run key, and existing tests show some base-session hook and untrusted events are intentionally not treated as active isolated-session events.

## What This Changes

Request to transfer or otherwise preserve visibility for pending system-event entries across the `heartbeat.isolatedSession` base-to-`:heartbeat` session boundary.

## Best Possible Solution

Define and document the isolated-heartbeat event visibility contract, then implement selective transfer or explicit base-key reads with regression coverage for delivery context, trust downgrades, and single-consumption behavior.

## Reproduction Assessment

Yes, by source inspection. Enqueue a system event on the base heartbeat session, enable `heartbeat.isolatedSession`, and the runner switches active inspection to the `:heartbeat` key while queues remain keyed independently.

## Solution Assessment

Unclear. Mirroring the cron session-entry transfer is plausible, but existing trust and hook-routing tests show maintainers should first decide which event classes are supposed to cross the isolated boundary.
