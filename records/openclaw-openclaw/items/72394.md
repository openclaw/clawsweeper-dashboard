## Summary

Keep open. Current main still lacks a generic transfer or alias for base-session system-event entries into the isolated `:heartbeat` queue; existing code covers selected prompt consumers but leaves the boundary contract unresolved.

## What This Changes

Request that pending system events queued on the base heartbeat session remain visible or be transferred when heartbeat runs in an isolated `:heartbeat` session.

## Best Possible Solution

Define and document the pre-isolation event visibility rule, then implement either selective transfer or explicit base-key reads with tests for ordering, single consumption, duplicate handling, delivery context, and trust classification.

## Reproduction Assessment

Yes. A focused regression can enqueue a system event on the base heartbeat session, run an isolated heartbeat, and assert that `peekSystemEventEntries` on the `:heartbeat` key remains empty while active-key consumers read that key.

## Solution Assessment

Unclear. Mirroring the cron session-entry transfer is plausible, but current tests intentionally keep some base-session events out of isolated active classification, so the narrowest maintainable fix depends on a maintainer decision about trust and delivery semantics.
