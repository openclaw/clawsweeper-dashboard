## Summary

Keep open. Current main and the latest release still let persisted session model state win over the configured default, while related closed items route the remaining model-refresh policy back to this issue instead of resolving it.

## What This Changes

The issue asks for existing persistent sessions to refresh, migrate, or clearly reset their stored model selection after `openclaw models set` changes the global default model.

## Best Possible Solution

Keep this as the canonical active-session default-model refresh tracker and define one source-aware contract for explicit, legacy, runtime, and auto model state across `models set`, status, doctor, reset, and any migrate or inherit command.

## Reproduction Assessment

Yes, at source level. Persist a session with old `model`/`modelProvider` or model override metadata, run `openclaw models set`, and current resolution still prefers stored session state before the configured default.

## Solution Assessment

Unclear. A refresh or migration mechanism is plausible, but maintainers need to choose whether default changes should auto-propagate, require an opt-in setting, use an explicit reset/migrate command, or focus on status/doctor guidance.
