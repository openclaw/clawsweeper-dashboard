## Summary

Keep open. Current main still updates only the configured default model, while persisted session runtime and override metadata can continue to determine the model shown or used until an explicit session reset or model patch occurs.

## What This Changes

Requests that persistent chat sessions refresh or offer migration after `openclaw models set` changes the global default model.

## Best Possible Solution

Add an intentional refresh or migration path so stale default-derived session state can move to the new default while explicit user-selected session models remain pinned unless reset or migrated.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to persist a session with old `model`/`modelProvider` metadata, run `models set` to update the configured default, and observe that session resolution/listing can still prefer the stored metadata until an explicit reset or patch clears it.

## Solution Assessment

Unclear. A refresh or migration mechanism is the right direction, but the narrowest maintainable fix needs maintainer input on whether `models set` should mutate existing sessions automatically or expose an explicit reset/migrate action.
