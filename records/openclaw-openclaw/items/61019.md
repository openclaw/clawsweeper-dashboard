## Summary

Keep open. Current main still changes the configured default model without refreshing persisted session runtime or override metadata, and both Gateway/session listing paths can continue to resolve an existing session from stored model fields rather than the new global default.

## What This Changes

The issue requests that active/persistent sessions inherit or can deliberately migrate to a new global default after `openclaw models set <model>`.

## Best Possible Solution

Add an intentional refresh/migration path for sessions after a global default change: stale default-derived runtime state should move to the new default, while explicit user selections remain pinned unless the user resets or migrates them.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to create a stored session with old `model`/`modelProvider` metadata, run `models set` to update the default, and observe that session resolution/listing can still prefer the stored metadata until an explicit reset or patch clears it.

## Solution Assessment

Unclear. A refresh or migration mechanism is the right direction, but the narrowest maintainable fix needs maintainer input on whether `models set` should mutate existing sessions automatically or expose an explicit reset/migrate action.
