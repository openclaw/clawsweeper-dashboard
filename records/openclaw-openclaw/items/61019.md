Keep open. Current main still preserves persisted session model state ahead of the new global default, and the duplicate/superseded related issues route the remaining work here as a model-refresh policy decision rather than a cleanup close.

## What This Changes

The issue requests a policy or mechanism for active or persistent sessions to pick up a new global default model after `openclaw models set`.

## Best Possible Solution

Define one public model-refresh contract, then implement the chosen propagation, migrate/reset, or status/doctor guidance while preserving explicit user selections and coordinating narrower cache/override fixes.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: write a new global default with `openclaw models set`, keep a session entry with stored `modelProvider`/`model` or an explicit override, and model resolution will choose the stored session state before the new default.

## Solution Assessment

Unclear. A refresh or migration path is plausible, but maintainers need to choose automatic propagation, an opt-in setting/command, or explicit reset/status guidance before this is a narrow safe fix.
