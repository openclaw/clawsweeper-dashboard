## Summary

Keep open. Current main still has the reported session-continuity behavior, and the related config-option issue was closed into this one, so the remaining work is a product/API policy decision rather than a cleanup close.

## What This Changes

The issue asks for active sessions to refresh or expose a migration path when the global default model changes.

## Best Possible Solution

Choose a deliberate refresh policy and implement it where default-model updates, config reload, and session metadata meet, while preserving `modelOverrideSource: "user"` selections and documenting `models set` behavior.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: persist a session with old `model`/`modelProvider` or override metadata, run `openclaw models set`, and session resolution/listing will still prefer the stored session state. I did not run a live CLI session because this review is read-only.

## Solution Assessment

Unclear. A refresh or migration mechanism is plausible, but maintainers need to choose automatic propagation, an opt-in config/command, or explicit reset/status guidance before this is a narrow maintainable fix.
