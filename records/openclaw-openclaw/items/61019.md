## Summary

Keep open. Current main still updates global model defaults without migrating or clearing persisted session model state, so active sessions can continue resolving to their stored provider/model until an explicit reset or patch path runs; the related closed items route the remaining behavior back to this policy decision rather than a shipped fix.

## What This Changes

The issue asks for active sessions to pick up global default-model changes, or for an explicit refresh/migration path when the global model changes.

## Best Possible Solution

Keep this as the canonical active-session default-model refresh tracker and choose one source-aware policy: explicit reset/migrate UX, opt-in propagation, or clearer status/doctor guidance that preserves intentional user selections.

## Reproduction Assessment

Yes, at source level. Persist a session with old `model`/`modelProvider` or override metadata, run `openclaw models set`, and current session resolution/listing still prefers stored session state before falling back to the new configured default.

## Solution Assessment

Unclear. A refresh or migration mechanism is plausible, but the best fix depends on a maintainer policy choice between automatic propagation, opt-in configuration, explicit reset/migrate commands, and status/doctor guidance.
