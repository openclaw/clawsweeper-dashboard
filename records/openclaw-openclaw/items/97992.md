## Summary

Keep open: current main and v2026.6.10 intentionally keep the macOS LaunchAgent PATH canonical and omit Nix/Bun/user-bin directories, so this remains a product and security-policy decision rather than cleanup.

## What This Changes

Request to include or make configurable macOS Gateway daemon PATH entries for user package-manager directories such as Nix, Bun, and local bins.

## Best Possible Solution

Decide the macOS service PATH policy first; if supporting these dirs, prefer a maintainer-approved opt-in service PATH setting with doctor, docs, and regression coverage over changing the canonical default silently.

## Reproduction Assessment

Yes for the service PATH behavior: current main and v2026.6.10 skip Darwin user dirs by default, and tests assert Nix paths are omitted. The user-visible exec failure itself was not live-reproduced here.

## Solution Assessment

Unclear: flipping Darwin user dirs on by default is not clearly the best fix because current docs/tests encode a canonical PATH for service hardening. A maintainer-approved opt-in or clearer use of existing exec PATH controls is safer.
