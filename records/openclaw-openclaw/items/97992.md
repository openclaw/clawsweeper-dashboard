## Summary

Keep open: current main and v2026.6.10 still intentionally keep macOS LaunchAgent PATH canonical and omit general user-bin/Nix/Bun directories, so the remaining work is a product and security-policy decision rather than conservative cleanup.

## What This Changes

The issue requests that macOS Gateway daemon PATH handling include or preserve user-level binary directories such as Nix, Bun, and local bins.

## Best Possible Solution

Decide the macOS daemon PATH security policy first; if support is desired, prefer a maintainer-approved opt-in service PATH setting or documented exec-level path configuration with doctor/docs/tests over silently changing the default.

## Reproduction Assessment

Yes for the source-level service PATH behavior: current main and v2026.6.10 skip Darwin user dirs by default, and tests assert Nix/profile paths are omitted. I did not live-reproduce the user-visible macOS exec failure in this read-only review.

## Solution Assessment

Unclear: flipping Darwin user dirs on by default is not clearly the best fix because current docs/tests encode a hardened canonical service PATH. A maintainer-approved opt-in or existing exec PATH control may be safer.
