## Summary

Keep open: current source, tests, and docs confirm macOS LaunchAgent PATH is intentionally canonical and excludes user package-manager dirs, so the requested Nix/Bun/user-bin behavior remains an unresolved product and security-policy decision rather than already implemented cleanup.

## What This Changes

Request to include or opt into macOS Gateway service PATH entries for Nix, Bun, local, and other user-bin directories instead of regenerating only the canonical LaunchAgent PATH.

## Best Possible Solution

Decide the macOS service PATH policy first; if maintainers want Nix/user-bin support, add a narrow explicit opt-in with daemon install, doctor audit, docs, and regression coverage rather than silently copying user dirs by default.

## Reproduction Assessment

Yes for the service PATH behavior from source: current main and v2026.6.10 both skip Darwin user dirs by default, and tests assert `.nix-profile/bin` is omitted. I did not establish a live macOS launchd exec transcript, so the user-visible exec failure is source-reproducible rather than live-reproduced.

## Solution Assessment

Unclear: flipping `includeUserDirs` on Darwin is not clearly the best fix because current docs/tests encode a canonical PATH to prevent version-manager dirs from changing daemon child process resolution. A safer path is maintainer-approved explicit opt-in or clearer guidance around `tools.exec.pathPrepend` and login-shell PATH for exec-only needs.
