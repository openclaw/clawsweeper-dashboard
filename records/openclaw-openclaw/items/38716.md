## Summary

Keep open. Current main still lacks Slack RTS/action-token search: there is no `assistant.search.context` call path, no `action_token` propagation from Slack inbound events, and no Slack `search` message action; the open `search.messages` PR is adjacent but does not satisfy the requested RTS contract.

## What This Changes

Add Slack Real-Time Search support by extracting Slack assistant action tokens and exposing an `assistant.search.context`-backed message or dedicated tool action.

## Best Possible Solution

Implement Slack-plugin-owned RTS support with an explicit ephemeral-token contract, a typed read-only `assistant.search.context` action, documented Slack AI/search scope limits, and deliberate coordination with `search.messages` work.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence current-main gap check: RTS symbols, action-token propagation, and Slack RTS action dispatch are absent.

## Solution Assessment

Unclear until maintainers define the Slack RTS product and security contract. The maintainable direction is Slack-plugin-owned RTS support based on Slack's current `assistant.search.context` contract, not treating user-token `search.messages` as a full substitute.

## Security Review

Status: needs_attention

Summary: The requested feature is security-sensitive because it would handle ephemeral Slack action tokens and potentially broad workspace search results.

Concerns:

- **[medium] Define RTS action-token boundaries:**
  - body: A future implementation should avoid persisting or logging Slack `action_token` values, bind each token to the triggering Slack turn, and keep tokens out of generic tool output and stored transcripts.
  - confidence: 0.9
