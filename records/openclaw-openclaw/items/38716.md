## Summary

Keep open. Current main still lacks Slack RTS/action-token support, and the related open search work at https://github.com/openclaw/openclaw/pull/65691 implements user-token `search.messages`, not `assistant.search.context` with ephemeral action tokens.

## What This Changes

Request Slack plugin support for extracting Slack assistant action tokens and exposing `assistant.search.context` Real-Time Search as a message or dedicated search action.

## Best Possible Solution

Implement Slack-plugin-owned RTS support with a typed read-only action that keeps ephemeral action tokens turn-scoped, calls `assistant.search.context`, and documents scopes, privacy limits, rate limits, and coexistence with user-token `search.messages`.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection on current main gives a high-confidence verification path: no action-token propagation, RTS helper, `assistant.search.context` call, or Slack search action exists.

## Solution Assessment

Unclear until maintainers decide the security and API shape. The maintainable direction is Slack-plugin-owned RTS support; treating the user-token `search.messages` PR as the fix would not cover the requested action-token API.
