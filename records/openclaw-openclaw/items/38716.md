## Summary

Keep open. Current main still lacks Slack RTS support: no event-scoped `action_token` propagation, no `assistant.search.context` call path, and no Slack `search` message action; the related open `search.messages` PRs are adjacent but do not satisfy the assistant action-token RTS request.

## What This Changes

The issue requests Slack RTS support by capturing Slack assistant/action tokens and exposing `assistant.search.context` through a message or dedicated search action.

## Best Possible Solution

Implement Slack-plugin-owned RTS support with an explicit ephemeral-token contract, a typed read-only `assistant.search.context` action, documented `search:read.*` scope limits, and clear separation or consolidation with user-token `search.messages` work.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence current-main gap check: RTS symbols, event action-token propagation, and Slack RTS action dispatch are absent.

## Solution Assessment

Unclear until maintainers define the Slack RTS product and security contract. The maintainable direction is Slack-plugin-owned RTS support based on Slack’s current `assistant.search.context` contract, not treating legacy `search.messages` as a full substitute.
