## Summary

Keep open. Current main still lacks Slack RTS support: no `assistant.search.context` call path, no captured `action_token`, and no Slack `search` message action on the bundled Slack plugin; the open adjacent Slack search PRs use `search.messages`, which does not satisfy the event-scoped RTS contract.

## What This Changes

Add Slack RTS support by capturing Slack assistant action tokens and exposing `assistant.search.context` through a message action or dedicated tool.

## Best Possible Solution

Implement Slack-plugin-owned RTS search with event-scoped action-token capture, a read-only `assistant.search.context` action, privacy/scope docs, focused tests, and redacted live Slack proof, coordinated with but distinct from `search.messages` work.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence verification path: current main lacks RTS symbols, event action-token context, and Slack RTS action dispatch.

## Solution Assessment

Unclear until maintainers define the Slack RTS product and security contract. The likely maintainable direction is a Slack-plugin-owned, event-scoped `assistant.search.context` action rather than treating user-token `search.messages` as the same feature.
