## Summary

Keep open. Current main and the latest release still do not implement Slack RTS/action-token search: Slack assistant events are handled, but no `action_token` is captured, no `assistant.search.context` call exists, and the related open Slack search PRs use `search.messages` instead of the RTS action-token contract.

## What This Changes

Request support for Slack Real-Time Search by capturing Slack assistant action tokens and exposing `assistant.search.context` through a Slack message/tool action.

## Best Possible Solution

Implement a Slack-plugin-owned RTS action that carries per-turn Slack action tokens only through the authorized assistant/message context, calls `assistant.search.context`, and documents scope, privacy, and rate-limit behavior alongside any `search.messages` surface.

## Reproduction Assessment

Not applicable as a user bug reproduction, but the feature gap is source-reproducible: current main and `v2026.5.22` have no `assistant.search.context`, `action_token`, or Slack `search` action path.

## Solution Assessment

Unclear until maintainers define the Slack RTS product and security contract. The requested direction is useful, but it should follow Slack's action-token contract instead of treating `search.messages` as an RTS substitute.
