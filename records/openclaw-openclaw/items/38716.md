## Summary

Keep open. Current main and v2026.6.8 still do not implement Slack RTS/action-token search: Slack assistant events preserve thread context only, Slack action discovery does not advertise `search`, and the runtime read path uses conversation history/replies rather than `assistant.search.context`. The open Slack action-parity PR is adjacent user-token `search.messages` work, but it does not supersede this action-token RTS request.

## What This Changes

Request Slack RTS support by capturing Slack assistant/event `action_token` values and exposing `assistant.search.context` through a Slack message/tool action.

## Best Possible Solution

Add a Slack-plugin-owned RTS search operation that privately consumes a thread/user/account-bound action token, documents required AI/search scopes, and proves live Slack behavior while keeping it distinct from user-token search unless maintainers deliberately consolidate them.

## Reproduction Assessment

Yes. The source-repro path is current main: Slack assistant events are handled, but there is no `action_token` model, no `assistant.search.context` call, and no Slack `search` action advertised or dispatched.

## Solution Assessment

No, not as a generic token exposure. The safer solution is a typed Slack RTS search operation that privately consumes the short-lived token inside the Slack plugin boundary.
