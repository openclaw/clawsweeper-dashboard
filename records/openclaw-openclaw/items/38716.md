## Summary

Keep open. Current main still lacks Slack RTS support: there is no event-scoped `action_token` propagation, no `assistant.search.context` call path, and no Slack search/RTS message action; the remaining work needs product and security review before implementation.

## What This Changes

Add Slack RTS workspace search by carrying Slack event action tokens and exposing `assistant.search.context` through the Slack plugin action surface.

## Best Possible Solution

Implement this in the bundled Slack plugin as a typed read-only RTS action that follows Slack's current `message`/`app_mention` action-token contract, keeps tokens ephemeral, documents `search:read.*` scope limits, and coordinates naming with the separate user-token search work.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence current-main gap check: RTS symbols, event action-token propagation, and Slack RTS action dispatch are absent.

## Solution Assessment

Unclear until maintainers define the Slack RTS product and security contract. The maintainable direction is Slack-plugin-owned RTS support based on Slack's current `assistant.search.context` docs, not a generic legacy `search.messages` substitute.

## Security Review

Status: needs_attention

Summary: The requested feature is security-sensitive because it would handle ephemeral Slack action tokens and broad workspace search results.

Concerns:

- **[medium] Define RTS action-token boundaries:**
  - body: A future implementation should avoid persisting or logging Slack `action_token` values, bind each token to the triggering Slack turn, and keep tokens out of generic tool output and stored transcripts.
  - confidence: 0.9
- **[medium] Gate workspace-wide search exposure:**
  - body: RTS can expose messages, files, users, and channels depending on token type and `search:read.*` scopes, so maintainers should define consent, redaction, rate-limit, and scope behavior before landing the feature.
  - confidence: 0.86
