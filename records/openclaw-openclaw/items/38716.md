## Summary

Keep open. Current main and the latest release still do not implement Slack RTS/action-token search, and the related Slack `search.messages` work does not satisfy the requested `assistant.search.context` action-token contract.

## What This Changes

The issue requests Slack RTS support by capturing Slack assistant action tokens and exposing `assistant.search.context` as a Slack message/tool action.

## Best Possible Solution

Implement a Slack-plugin-owned RTS action that captures per-turn action tokens only from authorized Slack assistant/message context, calls `assistant.search.context`, and documents scopes, privacy, consent, rate limits, and coexistence with `search.messages`.

## Reproduction Assessment

Yes for a source-level feature gap, not for a broken existing behavior: current main has Slack assistant events and read actions, but no `action_token`, `assistant.search.context`, or Slack `search` action path.

## Solution Assessment

Unclear until maintainers define the RTS security and product contract. The requested direction is useful, but it should be implemented as a Slack-owned RTS path rather than treated as covered by user-token `search.messages`.
