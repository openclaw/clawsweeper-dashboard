## Summary

Keep open. Current main still lacks the requested Slack RTS path: no Slack `action_token` is captured, no `assistant.search.context` call exists, and the adjacent open Slack search PRs implement user-token `search.messages`, not assistant action-token Real-Time Search.

## What This Changes

The issue requests Slack Real-Time Search support by extracting assistant action tokens from Slack assistant events and exposing `assistant.search.context` through a message action or dedicated tool.

## Best Possible Solution

Implement Slack-plugin-owned RTS support with an explicit contract: capture the inbound event `action_token`, keep it scoped to the triggering assistant/message turn, call `assistant.search.context` through a typed read-only action, and document scopes, limits, and privacy behavior.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection on current main gives a high-confidence verification path for the missing capability: no action-token field, no RTS helper, and no Slack RTS search action exist.

## Solution Assessment

Unclear until maintainers decide the security and API shape. The maintainable direction is Slack-plugin-owned RTS support; treating user-token `search.messages` work as the fix would not cover the requested action-token API.
