## Summary

Keep open. Current main and the latest release still do not implement Slack RTS/action-token search, and the related open Slack search PRs use legacy `search.messages` rather than Slack's `assistant.search.context` action-token contract.

## What This Changes

The issue requests Slack RTS support by capturing Slack assistant/action tokens and exposing an `assistant.search.context`-backed search action or tool.

## Best Possible Solution

Implement a Slack-plugin-owned RTS contract that captures ephemeral per-turn action tokens, exposes a typed read-only `assistant.search.context` action, documents scope/privacy limits, and deliberately separates or consolidates it with `search.messages`.

## Reproduction Assessment

Not applicable as a user bug reproduction, but yes for a source-level feature gap: current main and `v2026.5.22` have no `assistant.search.context`, `action_token`, or Slack `search` action path.

## Solution Assessment

Unclear until maintainers define the Slack RTS product and security contract. The requested direction is useful, but the implementation should follow Slack's action-token source and avoid treating `search.messages` as an RTS substitute.
