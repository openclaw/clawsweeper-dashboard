## Summary

Keep open. Current main still lacks Slack RTS support: there is no `assistant.search.context` call path, no `action_token` capture or propagation, and Slack's advertised message actions still exclude a Slack search action; the adjacent `search.messages` work does not cover the requested assistant action-token RTS flow.

## What This Changes

Request Slack plugin support for Slack RTS `assistant.search.context` using per-interaction action tokens and exposure through a Slack message/search tool action.

## Best Possible Solution

Implement this in the bundled Slack plugin only after settling the RTS contract: capture supported Slack action tokens from inbound user events, keep them scoped to the active assistant/message context, expose a typed read-only RTS search action, document the AI/search scopes and privacy limits, and decide how it coexists with user-token `search.messages`.

## Reproduction Assessment

Not applicable as a traditional bug reproduction. The gap is high-confidence from current-main static verification: the Slack plugin has no `assistant.search.context`, `action_token`, or Slack search action path to exercise.

## Solution Assessment

Unclear. Supporting Slack RTS is a plausible product direction, but the best implementation needs maintainer design around event source, ephemeral token lifetime, user-token fallback, permission gating, and overlap with #65691 before an automated fix PR is safe.
