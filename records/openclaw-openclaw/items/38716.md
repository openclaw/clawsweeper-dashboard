## Summary

Keep open. Current main still lacks Slack RTS/action-token search: there is no `assistant.search.context` call path, no `action_token` propagation from inbound Slack events, and no Slack `search` message action; the related open Slack `search.messages` PRs are adjacent but do not satisfy the RTS contract.

## What This Changes

Request Slack plugin support for Slack Real-Time Search by capturing assistant/action tokens and exposing `assistant.search.context` through a typed message action or tool.

## Best Possible Solution

Implement Slack-plugin-owned RTS support with an explicit ephemeral-token contract, a typed read-only `assistant.search.context` action, documented `search:read.*` scope limits, and deliberate separation or consolidation with `search.messages` work.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection provides a high-confidence current-main gap check: RTS symbols, action-token propagation, and Slack RTS action dispatch are absent.

## Solution Assessment

Unclear until maintainers define the Slack RTS product and security contract. The maintainable direction is Slack-plugin-owned RTS support based on Slack's current `assistant.search.context` contract, not treating legacy `search.messages` as a full substitute.
