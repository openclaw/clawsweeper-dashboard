## Summary

Keep open. Current main and the latest release still lack Slack RTS/action-token search, and the adjacent open `search.messages` PR does not satisfy the requested `assistant.search.context` action-token contract.

## What This Changes

Request support in the Slack plugin for capturing Slack assistant action tokens and exposing an RTS-backed workspace search action.

## Best Possible Solution

Implement a Slack-plugin-owned RTS action that carries per-turn Slack action tokens only through the authorized assistant/message context, calls `assistant.search.context`, and documents scope, privacy, and rate-limit behavior alongside any `search.messages` surface.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainers define the Slack RTS product and security contract. The requested direction is useful, but it should follow Slack's action-token contract instead of treating `search.messages` as an RTS substitute.
