## Summary

Keep open. Current main still lacks Slack RTS/action-token support, and the related Slack search work is the older user-token `search.messages` path rather than `assistant.search.context` with event-scoped `action_token`.

## What This Changes

Request support in the Slack plugin for extracting Slack assistant event `action_token` values and exposing `assistant.search.context` Real-Time Search to agents.

## Best Possible Solution

Implement this as Slack-plugin-owned RTS support with event-scoped action-token handling, a typed read-only search action, privacy/scope docs, and tests kept distinct from user-token `search.messages` unless maintainers intentionally consolidate them.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence verification path: current main has no RTS symbols, no action-token context, and no Slack search action dispatch.

## Solution Assessment

Unclear until maintainers settle the Slack RTS security and product contract. The maintainable direction is a Slack-plugin-owned `assistant.search.context` action; treating the adjacent `search.messages` PR as the fix would leave the requested event-scoped token behavior missing.
