## Summary

Keep open. Current main still lacks Slack RTS/action-token support, and the active adjacent Slack search work implements the older user-token `search.messages` path rather than `assistant.search.context` with event-scoped `action_token` handling.

## What This Changes

Request Slack Real-Time Search support by extracting assistant event `action_token` values and exposing `assistant.search.context` through a Slack message action or dedicated tool.

## Best Possible Solution

Implement Slack-plugin-owned RTS support with an explicit action-token contract, a typed read-only `assistant.search.context` action, focused docs/tests, and privacy/scope safeguards, either as its own follow-up or folded into an approved Slack search/parity PR.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence verification path: current main has no Slack RTS symbols, no action-token context, and no Slack `search` action dispatch.

## Solution Assessment

Unclear until maintainers decide the Slack RTS API and security shape. The maintainable direction is Slack-plugin-owned `assistant.search.context` support; treating the adjacent `search.messages` PRs as the fix would leave the requested action-token behavior missing.
