## Summary

Keep open. Current main still lacks Slack RTS/action-token search: assistant thread events preserve routing context but no `action_token`, Slack actions do not advertise or dispatch search, and existing Slack reads use conversations history/replies rather than `assistant.search.context`.

## What This Changes

The issue requests Slack Real-Time Search support by capturing assistant action tokens and exposing `assistant.search.context` through a Slack message action or dedicated tool.

## Best Possible Solution

Implement Slack-plugin-owned RTS support with an explicit ephemeral-token contract, a typed read-only `assistant.search.context` action, documented AI/search scope limits, and coordination with the existing `search.messages` work.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request, but source inspection gives a high-confidence current-main gap check: RTS symbols, action-token propagation, and Slack RTS action dispatch are absent.

## Solution Assessment

Unclear until maintainers define the Slack RTS product and security contract. The maintainable direction is Slack-plugin-owned RTS support based on Slack's current `assistant.search.context` contract, not treating user-token `search.messages` as a full substitute.
