## Summary

Keep open. Current main still lacks a Slack RTS/action-token search path: the Slack plugin does not capture `action_token`, call `assistant.search.context`, or advertise/dispatch a Slack `search` message action; the related `search.messages` work is adjacent but does not satisfy this assistant-token RTS request.

## What This Changes

The issue requests Slack plugin support for Slack Real-Time Search by carrying Slack assistant action tokens into an `assistant.search.context` message/search action.

## Best Possible Solution

Implement this in the bundled Slack plugin with a typed read-only RTS action that carries the ephemeral Slack action token only through the authorized inbound context, calls `assistant.search.context`, documents AI/search scopes and rate limits, and deliberately relates or separates it from user-token `search.messages`.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high-confidence negative proof that current main does not yet contain the requested `action_token` capture or `assistant.search.context` action path.

## Solution Assessment

Unclear. The Slack plugin is the right boundary, but the best implementation needs maintainer decisions on ephemeral token handling, privacy/scope policy, rate limits, and whether to converge with the open user-token search work.
