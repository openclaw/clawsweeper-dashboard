## Summary

Keep open. Current main and the latest release still lack the requested Slack RTS path: no `action_token` capture, no `assistant.search.context` call, and no Slack RTS/search action wired through message actions; the open Slack `search.messages` PRs are adjacent user-token search work, not the event-scoped assistant token contract.

## What This Changes

Request Slack RTS support by capturing assistant event action tokens and exposing `assistant.search.context` through a Slack message action or dedicated tool.

## Best Possible Solution

Implement a Slack-plugin-owned RTS contract that captures event-scoped action tokens only for the active turn, exposes a typed read-only `assistant.search.context` action, documents scopes/privacy limits, and keeps it distinct from user-token `search.messages` unless maintainers deliberately consolidate them.

Not applicable as a bug reproduction because this is a feature request. Source inspection provides a high-confidence current-main check: RTS symbols, action-token propagation, and Slack RTS action dispatch are absent.

## Solution Assessment

Unclear until maintainers define the Slack RTS product and security contract. The maintainable direction is event-scoped `assistant.search.context` support in the Slack plugin, coordinated with but distinct from `search.messages` work.

## Security Review

Status: needs_attention

Summary: The requested feature is security-sensitive because it would process ephemeral Slack action tokens and potentially broad workspace search results.

Concerns:

- **[medium] Define RTS action-token boundaries:**
  - body: A future implementation should avoid persisting or logging Slack `action_token` values, bind each token to the triggering Slack turn, and keep tokens out of generic tool output and stored transcripts.
  - confidence: 0.88
- **[medium] Gate organization-wide search exposure:**
  - body: RTS can return broad workspace context depending on authorization and scopes, so maintainers should define consent, privacy, redaction, and rate-limit behavior before landing the feature.
