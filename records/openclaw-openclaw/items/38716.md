## Summary

Keep open. Current main still lacks Slack RTS support through `assistant.search.context` and does not capture event-scoped Slack `action_token` values; the related open Slack search PR covers `search.messages` with user-token semantics, which is adjacent but not the requested RTS contract.

## What This Changes

The issue requests Slack RTS support by extracting assistant action tokens from Slack assistant events and exposing `assistant.search.context` search through the Slack message/tool surface.

## Best Possible Solution

Implement Slack-plugin-owned RTS search with typed event-scoped action-token capture, a read-only `assistant.search.context` action, privacy/scope documentation, focused tests, and redacted live Slack proof, coordinated with but distinct from `search.messages` work.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection provides a high-confidence verification path: current main lacks RTS symbols, action-token context, and Slack RTS action dispatch.

## Solution Assessment

Unclear until maintainers decide the Slack RTS product and security contract. The likely narrow solution is to keep this inside the Slack plugin as event-scoped `assistant.search.context` support rather than treating user-token `search.messages` as the same feature.
