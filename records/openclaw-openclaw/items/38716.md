## Summary

Keep open. Current main still does not implement Slack RTS via `assistant.search.context` or event-scoped `action_token`; the related Slack search PR uses the older user-token `search.messages` API and does not satisfy this request.

## What This Changes

Add Slack RTS support by extracting event `action_token` values and exposing `assistant.search.context` workspace search through the Slack message tool or a dedicated tool.

## Best Possible Solution

Implement this inside the Slack plugin with event-scoped `action_token` capture, a typed read-only RTS search action, privacy/scope docs, focused tests, and live redacted Slack proof, while keeping it distinct from or deliberately coordinated with the user-token `search.messages` work in https://github.com/openclaw/openclaw/pull/65691.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence verification path: current main has no RTS symbols, no `action_token` context, and no Slack RTS search action dispatch.

## Solution Assessment

Unclear until maintainers settle the Slack RTS product and security contract. The likely maintainable direction is Slack-plugin-owned `assistant.search.context` support; treating the adjacent `search.messages` PR as the fix would leave event-scoped token behavior missing.
