## Summary

Keep open. Current main still lacks Slack RTS support: there is no `assistant.search.context` implementation, no captured Slack `action_token` on inbound assistant events, and no Slack `search` message action; the open `search.messages` PR is adjacent but does not satisfy the event-scoped RTS contract.

## What This Changes

Add Slack Real-Time Search support by capturing assistant event action tokens and exposing an `assistant.search.context`-backed message action.

## Best Possible Solution

Implement a Slack-plugin-owned RTS path that captures event-scoped action tokens, exposes a typed read-only `assistant.search.context` action, documents scope/privacy limits, and keeps it distinct from user-token `search.messages` unless maintainers deliberately consolidate them.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence verification path: current main lacks RTS symbols, event action-token context, and Slack RTS action dispatch.

## Solution Assessment

Unclear until maintainers define the Slack RTS product and security contract. The likely maintainable direction is a Slack-plugin-owned, event-scoped `assistant.search.context` action rather than treating user-token `search.messages` as the same feature.

## Security Review

Status: needs_attention

Summary: The requested feature is security-sensitive because it would handle ephemeral Slack action tokens and organization-wide search results.
