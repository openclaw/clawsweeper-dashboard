## Summary

Keep open. Current main still lacks Slack RTS/action-token support, and the adjacent Slack search PRs cover the older user-token `search.messages` path rather than `assistant.search.context` with event-scoped action tokens.

## What This Changes

Request Slack Real-Time Search support by capturing Slack assistant/action tokens from inbound events and exposing `assistant.search.context` through a Slack message action or dedicated tool.

## Best Possible Solution

Implement Slack-plugin-owned RTS support with event-scoped `action_token` capture, a typed read-only `assistant.search.context` action, docs for AI/search scopes and privacy limits, and focused tests kept distinct from user-token `search.messages` unless maintainers intentionally consolidate them.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence verification path: current main has no Slack RTS symbols, no action-token context, and no Slack `search` message-action path.

## Solution Assessment

Unclear until maintainers decide the Slack RTS API and security shape. The maintainable direction is Slack-plugin-owned `assistant.search.context` support; treating adjacent `search.messages` PRs as the fix would leave the requested event-scoped action-token behavior missing.
