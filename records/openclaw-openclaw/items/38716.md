## Summary

Keep open. Current main still does not implement Slack RTS through `assistant.search.context` or event-scoped Slack `action_token` handling, and the related Slack search work tracks `search.messages` with user-token semantics instead of this assistant action-token contract.

## What This Changes

Add Slack plugin support for capturing Slack assistant action tokens from inbound assistant events and using them to expose RTS workspace search through `assistant.search.context`.

## Best Possible Solution

Implement Slack-plugin-owned RTS search with typed event-scoped action-token capture, a read-only `assistant.search.context` action, privacy/scope documentation, focused tests, and redacted live Slack proof, coordinated with but not replaced by user-token `search.messages` work.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection provides a high-confidence verification path: current main lacks RTS symbols, action-token context, and Slack RTS action dispatch.

## Solution Assessment

Unclear until maintainers decide the Slack RTS product and security contract. The likely narrow solution is to keep this inside the Slack plugin as event-scoped `assistant.search.context` support rather than treating user-token `search.messages` as the same feature.
