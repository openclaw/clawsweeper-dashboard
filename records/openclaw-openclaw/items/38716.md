## Summary

Keep open. Current main still lacks Slack RTS/action-token search: the Slack plugin has no `assistant.search.context` call, no `action_token` field in inbound Slack event types or tool context, and no `search`/RTS message action; the open `search.messages` work is adjacent but does not satisfy the requested bot-token RTS behavior.

## What This Changes

Request support in the Slack plugin for capturing assistant action tokens from Slack events and using them with `assistant.search.context` as a message search action or dedicated tool.

## Best Possible Solution

Implement this as Slack-plugin-owned RTS support: capture the inbound event `action_token`, pass it only through the relevant assistant/message context, expose a typed read-only RTS search action, and document required Slack AI/search scopes and limits.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection on current main gives a high-confidence verification path for the missing capability: no action-token field, no RTS helper, and no Slack search action exist in the plugin.

## Solution Assessment

The requested direction belongs in the bundled Slack plugin rather than core, but it needs maintainer design before implementation. Treating the user-token `search.messages` PR as the fix would not be the best solution because Slack RTS has different token, scope, rate-limit, and privacy semantics.
