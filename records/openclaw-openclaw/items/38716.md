## Summary

Keep open. Current main has Slack assistant-thread event handling, but still has no Slack RTS implementation: no `action_token` capture, no `assistant.search.context` call path, and no Slack `search` message action; related open Slack search PRs are `search.messages` work and do not cover the event-scoped RTS contract.

## What This Changes

The issue requests Slack RTS support by capturing assistant event `action_token` values and exposing `assistant.search.context` through a message action or dedicated tool.

## Best Possible Solution

Implement this in the Slack plugin with an explicit RTS contract: capture event-scoped action tokens, expose a typed read-only `assistant.search.context` action, add opt-in/gating and docs for scopes and privacy, and coordinate naming with the existing `search.messages` work.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence verification path: current main lacks RTS symbols, action-token context, and Slack RTS action dispatch.

## Solution Assessment

Unclear until maintainers define the Slack RTS product and security contract. The maintainable direction is Slack-plugin-owned, event-scoped `assistant.search.context` support rather than treating user-token `search.messages` as the same feature.

## Security Review

Status: needs_attention

Summary: The requested feature is security-sensitive because it would handle ephemeral Slack action tokens and organization-wide search results.

Concerns:

- **[medium] Define RTS action-token boundaries:**
  - body: A future implementation should avoid persisting or logging `action_token` values, bind each token to the triggering Slack turn, and keep tokens out of generic tool output and stored transcripts.
  - confidence: 0.88
- **[medium] Gate organization-wide search exposure:**
  - body: Slack RTS can expose broad workspace context depending on authorization and scopes, so maintainers should define consent, privacy, redaction, and rate-limit behavior before landing it.
