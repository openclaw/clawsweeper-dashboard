## Summary

Keep this PR open, but not merge-ready: the fix targets the right Feishu transport seam and has useful live proof, yet the current patch preserves request-level agents while disabling axios proxy handling, which can bypass OpenClaw managed proxy mode.

## What This Changes

The PR changes the Feishu client HTTP adapter and tests so Lark SDK REST calls and WS bootstrap requests use OpenClaw's ambient proxy agent with axios proxy auto-detection disabled.

## Best Possible Solution

Merge after the Feishu HTTP adapter routes SDK REST and WS bootstrap through OpenClaw's ambient proxy while overriding custom agents during managed proxy mode, with focused tests for managed and non-managed proxy cases.

## Reproduction Assessment

Yes, at source level: current main delegates Feishu SDK REST and WS bootstrap HTTP to axios-backed Lark HTTP instances, and axios uses proxy env unless config.proxy is false. The PR body also reports real WSL2 gateway/probe recovery after an equivalent installed-plugin hotfix.

## Solution Assessment

No, not yet: routing Lark SDK HTTP through OpenClaw's ambient proxy is the right seam, but preserving caller agents during managed proxy mode conflicts with the documented operator proxy boundary. The safer fix is to override agents in managed mode and test that behavior explicitly.
