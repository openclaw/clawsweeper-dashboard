## Summary

## What This Changes

The branch replaces Feishu's timeout-only Lark SDK HTTP wrapper with a proxy-aware wrapper for REST/token calls and WSClient bootstrap, plus tests for ambient and managed proxy agent behavior.

## Best Possible Solution

Land only after maintainers accept the Feishu policy that SDK HTTP/token and WS bootstrap follow ambient proxy routing, while managed proxy mode overrides request agents; keep per-channel direct/bypass policy as separate follow-up work.

## Reproduction Assessment

## Solution Assessment

Yes, with maintainer policy acceptance. Reusing the Lark SDK httpInstance seam inside the Feishu plugin is the narrow owner-boundary fix; a channel-level proxy mode is broader follow-up for mixed direct/proxy setups.
