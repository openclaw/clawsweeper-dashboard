## Summary

Keep open. Current main still has a source-level path where memory-core dreaming narrative runs can arrive at Active Memory as user-triggered webchat sessions, and Active Memory classifies unknown webchat session keys as direct chats instead of rejecting synthetic non-interactive keys.

## What This Changes

The issue asks Active Memory to reject memory-core dreaming or cron session keys before dispatch so it does not spend the embedded-run lane budget on non-interactive narrative runs.

## Best Possible Solution

Tighten Active Memory's existing eligibility or chat-type classification so only known interactive chat session-key shapes can become direct/group/channel/explicit, with a regression test for memory-core dreaming narrative keys.

## Reproduction Assessment

Yes, at source level. Memory-core builds `dreaming-narrative-*` subagent keys, the gateway can canonicalize them under `agent:<id>:...`, the agent attempt path uses `trigger: "user"`, and Active Memory maps unknown webchat keys to direct chat.

## Solution Assessment

Yes. The narrow maintainable fix is to reject synthetic non-interactive keys in the existing eligibility/classification gate; adding `excludeSessionKeyPatterns` would be broader config surface than this bug requires.
