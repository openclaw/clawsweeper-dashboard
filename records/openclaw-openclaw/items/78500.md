## Summary

Keep open. Current main still has a source-level path where memory-core dreaming narrative subagent runs can look like `trigger="user"` webchat runs to Active Memory, and Active Memory then classifies unknown webchat session keys as direct chats instead of rejecting non-interactive session keys.

## What This Changes

Active Memory should reject memory-core dreaming narrative and other non-interactive session keys before dispatching pre-prompt recall.

## Best Possible Solution

Tighten Active Memory's eligibility/session-type classification so only known interactive chat session-key shapes can become direct/group/channel/explicit, with a focused regression test for memory-core dreaming narrative keys.

## Reproduction Assessment

Yes, at source level. The current gateway/agent path hardcodes `trigger: "user"` for embedded attempts, and Active Memory maps unknown webchat session keys to `direct`, so a dreaming narrative key can pass the default direct-chat allowlist without a live cron run.

## Solution Assessment

Yes. The surgical fix is to reject non-interactive synthetic session keys in the existing eligibility/classification gate; adding an operator `excludeSessionKeyPatterns` config would be broader surface area than this bug needs.
