- `P2`: This is a normal-priority bug with limited blast radius: it wastes compute and creates noisy active-memory lane timeouts during dreaming cron, but does not appear to break ordinary conversations directly.
- `impact:session-state`: The bug is about non-interactive dreaming session keys being misclassified as eligible chat sessions for active-memory recall.

## Summary

Keep open: current main still routes memory-core dreaming subagent sessions into active-memory as user/webchat direct sessions, matching the reported timeout path, and no duplicate or merged fix owns this exact eligibility gap.

## What This Changes

Reject non-interactive memory-core dreaming or cron session keys from active-memory recall, or provide an operator exclude pattern.

## Best Possible Solution

Tighten active-memory eligibility so non-interactive system/plugin subagent session keys are rejected before recall dispatch, with a regression test for memory-core dreaming keys and no new config surface unless the bounded source fix proves insufficient.

## Reproduction Assessment

Yes, source-reproducible: memory-core dreaming uses `dreaming-narrative-*` plugin subagent sessions that reach active-memory as `trigger: user` and `messageProvider: webchat`, where current eligibility resolves them as direct chats.

## Solution Assessment

Yes: the narrow source fix is to reject non-chat session keys in active-memory eligibility; adding `excludeSessionKeyPatterns` would expand public plugin config and should be avoided unless the bounded code fix cannot cover real system-session keys.
