## Best Possible Solution

Apply the table-limit decision before starting or using Feishu streaming/static cards, then cover outbound, static reply, and explicit-card streaming-default reply behavior with focused tests.

No, not yet. Falling back to post mode is the narrow maintainable direction, but the implementation must apply the same constraint before streaming-card startup/use or explicitly scope that path out.
