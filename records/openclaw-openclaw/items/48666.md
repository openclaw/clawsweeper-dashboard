## Review Findings

Full review comments:

- **[P1] Apply body-size guard before consuming Slack requests:** `src/slack/monitor/provider.ts:234`
  - body: The new HTTP path calls `readRequestBody(req)` before installing `installRequestBodyLimitGuard`, so an unauthenticated Slack endpoint can buffer an oversized or slow POST fully in memory before `SLACK_WEBHOOK_MAX_BODY_BYTES` has any chance to trip. Keep the guard on the original stream or use a bounded body reader before parsing URL verification requests.
