## Summary

Keep open. Current main still lacks the PR's central configured-agent dropdown behavior, but the PR branch is not safe to merge because it mixes the focused UI fix with unrelated stale Slack/infra/UI changes and concrete security regressions.

## What This Changes

The PR seeds Control UI session options from configured agents and adds dropdown regression tests, while also changing Slack URL verification handling, json-file retry logic, mobile chat controls, model/theme UI, and adding a runtime log artifact.

## Best Possible Solution

Recreate the useful dropdown fix as a narrow current-main patch in the Control UI session selector with focused regression coverage for configured agents missing from sessionsResult.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to call resolveSessionOptionGroups with agentsList containing main and test-agent while sessionsResult contains only agent:main:main; current main returns only the listed row plus the active sessionKey, so the missing configured agent remains absent.

## Solution Assessment

No. The central configured-agent seeding idea is the right direction, but this branch is not the best solution because it bundles unrelated stale UI, Slack, infra, and log changes; a small current-main UI patch is safer.

  - body: The patch removes the session `<select>` from `renderChatControls` and adds `renderChatSessionSelect`, but the diff never renders that new helper from the chat layout. Keep the guard on the original stream or use a bounded body reader before parsing URL verification requests.
  - confidence: 0.94

  - body: This PR adds a local OpenClaw runtime log containing host-specific paths and connection telemetry. It is unrelated to the dropdown fix, adds repository noise, and should be removed from the branch rather than committed.
