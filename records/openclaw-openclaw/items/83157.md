## Label Justifications

- `P1`: The issue affects Claude CLI session recovery after `session_expired` by dropping recent context in a user-visible agent workflow.
- `impact:session-state`: The reported defect causes reseeded CLI sessions to lose recent transcript context and recover with stale conversation state.

## Summary

Current main still contains the prefix slice that drops the newest rendered reseed history, and the issue has a same-author open closing PR that targets the narrow fix, so it should stay open until that PR lands or is replaced.

## What This Changes

The issue reports that `buildCliSessionHistoryPrompt` keeps the oldest rendered reseed history when the transcript exceeds `maxHistoryChars`, causing the recent assistant context to be omitted during CLI session recovery.

## Best Possible Solution

Review and land the linked PR or an equivalent suffix-slice fix with regression coverage, then close this issue once the corrected reseed prompt behavior is on main.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to pass chronological rendered history longer than `maxHistoryChars` with a latest marker near the end; current main joins in input order and slices from the head, so the latest marker is omitted.

## Solution Assessment

Yes. Keeping the recent tail and moving the truncation marker to describe older dropped turns is the narrow maintainable fix, and the open linked PR implements that direction with regression tests.
