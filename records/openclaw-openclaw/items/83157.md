## Label Justifications

- `P1`: The issue describes a broken session-recovery workflow that can drop the newest Claude CLI context during reseed for real users.
- `impact:session-state`: The affected behavior is loss of recent transcript context in the session reseed prompt.

## Summary

Keep open: current main and the latest release still prefix-slice reseed history, so long histories can drop the newest assistant turns; the linked open PR is the right fix path and should land or be replaced before this issue closes.

## What This Changes

The issue requests changing CLI session reseed history truncation to retain the most recent rendered turns instead of prefix-slicing older history.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83117, or an equivalent replacement, then close this issue once tail-slice behavior and regression coverage are on main; keep the cap-size request in https://github.com/openclaw/openclaw/issues/83985 separate.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to pass chronological rendered history longer than `maxHistoryChars`; current main slices from the head, so the latest rendered turns fall out of the reseed prompt.

## Solution Assessment

Yes. Keeping the recent tail and moving the marker to describe older dropped turns is the narrow maintainable fix, and the paired open PR implements that direction with regression tests.
