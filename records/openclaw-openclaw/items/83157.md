## Summary

Keep open: current main still uses a prefix slice in the CLI reseed prompt builder, and the linked open PR is the right implementation path for this valid session-state bug.

## What This Changes

The issue reports that CLI session reseed prompt truncation keeps the oldest rendered history and drops the newest turns when `maxHistoryChars` is exceeded.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83117, or an equivalent suffix-slice regression fix, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to pass chronological messages whose rendered history exceeds `maxHistoryChars` and includes a final marker; current main slices from the head, so that newest marker is dropped.

## Solution Assessment

Yes. Keeping the recent tail and moving the marker to describe older dropped turns is the narrow maintainable fix, and the open paired PR implements that direction with regression coverage.
