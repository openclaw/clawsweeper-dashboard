- `P1`: The bug can drop recent context during Claude CLI session recovery, producing a broken real-user agent workflow after reseed.
- `impact:session-state`: The reported failure is loss of recent transcript context in the reseed prompt, causing session state drift after recovery.

## Summary

Keep open: current main and the latest release still prefix-slice capped reseed history, which can drop the newest assistant turns, and the open paired fix at https://github.com/openclaw/openclaw/pull/83117 is the right path to resolve it.

## What This Changes

Fix `buildCliSessionHistoryPrompt` so capped reseed history preserves the newest turns instead of prefix-slicing the oldest content.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83117 or an equivalent suffix-slice fix with latest-turn regression coverage, then close this issue after merge.

## Reproduction Assessment

Yes. Source inspection shows chronological rendered history is prefix-sliced when it exceeds `maxHistoryChars`, and the reporter supplied a redacted real Claude CLI reseed capture showing the newest turn cut off at the truncation marker.

## Solution Assessment

Yes. Keeping the recent tail and moving the marker to describe older dropped turns is the narrow maintainable fix; the paired PR adds targeted regression coverage for that behavior.
