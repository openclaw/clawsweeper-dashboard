## What This Changes

The issue reports that `buildCliSessionHistoryPrompt` truncates long CLI reseed history from the end, dropping the newest assistant/user context instead of older turns.

## Best Possible Solution

Review and land the paired suffix-slice fix, or an equivalent regression-tested fix, then close this issue after the corrected reseed prompt behavior reaches main.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: pass chronological rendered history longer than `maxHistoryChars` with latest-turn markers, and current main slices from the head so those latest markers are excluded from the reseed history.

## Solution Assessment

Yes. Keeping the recent tail and moving the marker to describe older dropped turns is the narrow maintainable fix; the open paired PR implements that direction, so the issue should remain open until that fix or an equivalent lands.
