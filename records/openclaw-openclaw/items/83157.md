## Summary

Keep open: current main still prefix-slices the rendered CLI reseed history, and the same-author paired PR https://github.com/openclaw/openclaw/pull/83117 is the right path to land or replace before closing this issue.

## What This Changes

The issue reports that over-cap CLI reseed history keeps the oldest rendered transcript and drops the most recent assistant turns from the recovery prompt.

## Best Possible Solution

Land or replace the focused suffix-slice fix with regression coverage, then close this issue after the fix is on main.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to pass chronological rendered history longer than `maxHistoryChars` with a final marker; current main slices from the head, so the final marker is absent from the reseed history.

## Solution Assessment

Yes. Keeping the recent tail and moving the truncation marker to describe older dropped turns is the narrow maintainable fix; the paired PR also handles the compaction-summary edge case before this issue should close.
