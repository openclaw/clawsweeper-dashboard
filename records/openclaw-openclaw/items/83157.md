## Summary

Keep open: current main and v2026.5.18 still use the prefix slice, while the open paired PR is the right implementation path and references this issue with closing syntax.

## What This Changes

The issue reports that `buildCliSessionHistoryPrompt` keeps the oldest reseed history when `maxHistoryChars` is exceeded, dropping recent assistant turns from session recovery context.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/83117, then close this issue once suffix-slice behavior and regression coverage are on main.

## Reproduction Assessment

Yes, there is a high-confidence source reproduction path: pass chronological rendered history longer than `maxHistoryChars` with a latest-turn marker, and current main slices from the head so that marker is dropped. I did not run a live CLI reseed in this read-only review.

## Solution Assessment

Yes, keeping the recent tail and moving the truncation marker to describe older dropped turns is the narrow maintainable fix. The paired PR implements that direction with regression tests, so the issue should stay open until that PR merges or is replaced.
