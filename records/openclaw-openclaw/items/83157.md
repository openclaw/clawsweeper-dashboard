## What This Changes

The issue requests changing CLI reseed history truncation to keep recent turns rather than the oldest content when `maxHistoryChars` is exceeded.

## Best Possible Solution

## Solution Assessment

Yes. Keeping the recent tail and moving the marker to describe older dropped turns is the narrow maintainable fix, and the open paired PR implements that direction with regression coverage.
