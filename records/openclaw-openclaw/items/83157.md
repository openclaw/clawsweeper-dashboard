## Summary

Keep open: current main still prefix-slices chronological reseed history, and the open same-author PR is the right implementation path for this valid session-state bug.

## What This Changes

The issue reports that `buildCliSessionHistoryPrompt` keeps the oldest rendered transcript and drops recent assistant turns when reseed history exceeds `maxHistoryChars`.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/83117, then close this issue once suffix-slice behavior and regression coverage are on main.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to pass chronological messages longer than `maxHistoryChars` with a unique latest marker; current main renders in order and prefix-slices, so the latest marker is dropped.

## Solution Assessment

Yes. Keeping the recent suffix and moving the truncation marker to describe older dropped turns is the narrow maintainable fix; the open paired PR implements that direction.
