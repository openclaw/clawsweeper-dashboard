## Label Justifications

- `P1`: The bug can cause recovered Claude CLI sessions to lose the latest conversation context during session reseed.
- `impact:session-state`: The affected behavior drops recent transcript context from the reseed prompt while stored session state still contains it.

## Summary

Keep open: current main and the latest shipped tag still prefix-slice the reseed history, and the issue is already paired with an open closing PR that targets the narrow fix.

## What This Changes

The issue reports that `buildCliSessionHistoryPrompt` keeps the oldest over-cap reseed transcript content and drops recent assistant turns from recovered CLI session context.

## Best Possible Solution

Land the linked PR or an equivalent suffix-slice regression-tested fix, then close this issue as implemented.

## Reproduction Assessment

Yes. Passing chronological rendered history longer than `maxHistoryChars` through current main source drops the newest suffix because the function slices from the head; the reporter also supplied a redacted real reseed capture showing the same failure.

## Solution Assessment

Yes. Keeping the recent transcript tail and moving the marker to describe older dropped turns is the narrow maintainable fix; the open paired PR implements that direction with regression tests.
