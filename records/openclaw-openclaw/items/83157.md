## What This Changes

The issue reports that CLI session reseed history keeps the oldest over-cap transcript content and drops the newest assistant turns.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83117 or an equivalent suffix-slice fix with regression coverage, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes. A high-confidence source repro is to pass chronological rendered history longer than maxHistoryChars with a final marker; current main joins in order and prefix-slices, so the final marker is dropped.

## Solution Assessment

Yes. Keeping the recent suffix and moving the marker to describe older dropped turns is the narrow maintainable fix; the paired PR implements that direction with regression tests.
