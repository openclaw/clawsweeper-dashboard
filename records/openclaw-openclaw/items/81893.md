## Summary

Keep open: current main only supports the relative memory-flush token trigger, and adding a model-independent absolute trigger would be a new public configuration/API behavior that needs maintainer product judgment rather than cleanup closure.

## What This Changes

The issue requests an optional `absoluteThresholdTokens` sibling under `agents.defaults.compaction.memoryFlush` so memory flush can trigger at an absolute token count independent of model context window.

## Best Possible Solution

If maintainers accept the API, add an optional absolute token trigger to the memory-flush contract, evaluate it alongside the existing relative and transcript-byte gates, and document/test first-trigger-wins behavior without changing current defaults.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a request for a new public configuration mode. Source inspection does confirm the current relative behavior and the absence of an absolute threshold field on current main.

## Solution Assessment

Unclear pending maintainer API judgment: the proposed sibling field is a plausible narrow design, but the repository should first settle naming, precedence, and documentation for the public config contract.
