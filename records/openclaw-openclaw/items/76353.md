## Summary

Keep open: current main still warns unconditionally when compile-cache pruning hits permission errors, and open PR #76362 is the active closing fix candidate rather than a reason to close before merge.

## What This Changes

The issue asks for postinstall compile-cache pruning to suppress expected EACCES/EPERM warnings while preserving warnings for unexpected failures.

## Best Possible Solution

Review and land PR #76362, or an equivalent narrow patch, with regression coverage for permission-denied prune failures.

## Reproduction Assessment

Yes. Source inspection on current main shows permission-denied `readdirSync` or `rmSync` failures still flow into unconditional postinstall warnings; I did not run the root/non-root install sequence.

## Solution Assessment

Yes. A narrow errno guard in the compile-cache prune catch paths is the maintainable fix boundary; changing cache locations or ownership policy would be broader than this cosmetic postinstall bug requires.
