## What This Changes

Suppress EACCES/EPERM warnings when postinstall cannot prune a shared Node compile cache while preserving warnings for unexpected prune failures.

## Best Possible Solution

Review and land PR #76362 or an equivalent narrow patch with regression coverage for EACCES/EPERM suppression while keeping warnings for unexpected prune failures.

## Reproduction Assessment

Yes. A current-main Node invocation with a mocked EACCES rmSync produces the reported warning, and source inspection shows no permission-error guard in either prune catch path.

## Solution Assessment

Yes. The narrowest maintainable fix is an errno guard for EACCES/EPERM in the compile-cache prune warning path; changing cache location or ownership policy would be broader than this cosmetic postinstall bug requires.
