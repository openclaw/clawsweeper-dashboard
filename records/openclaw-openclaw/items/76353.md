## Summary

Keep open: current main still warns unconditionally on compile-cache prune failures, and PR #76362 is the open closing fix candidate rather than a reason to close this issue before merge.

## What This Changes

The issue requests suppressing expected EACCES/EPERM warnings when postinstall prunes a shared OpenClaw compile cache.

## Best Possible Solution

Land PR #76362 or an equivalent narrow patch with regression coverage for EACCES/EPERM suppression while preserving warnings for unexpected prune failures.

## Reproduction Assessment

Yes. Current source shows permission-denied `readdirSync` or `rmSync` failures reach unconditional warning calls, and postinstall invokes that helper; I did not run the root/non-root global install sequence in this read-only pass.

## Solution Assessment

Yes. A narrow errno guard in the compile-cache prune catch path is the maintainable fix boundary; changing cache locations or ownership policy would be broader than this cosmetic postinstall bug requires.
