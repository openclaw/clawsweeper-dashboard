## Summary

Keep open: current main still warns unconditionally when compile-cache pruning hits permission errors, and open PR #76362 is the active closing fix candidate rather than a reason to close before merge.

## What This Changes

Suppress misleading postinstall warnings when shared OpenClaw compile-cache pruning hits EACCES or EPERM.

## Best Possible Solution

Land PR #76362 or an equivalent narrow patch with regression coverage that suppresses EACCES/EPERM while preserving warnings for unexpected prune failures.

## Reproduction Assessment

Yes. Current main can reproduce the warning path by making pruneOpenClawCompileCache hit EACCES or EPERM from readdirSync or rmSync; the reported root/non-root global install is the real-world version of that source-backed path.

## Solution Assessment

Yes. A narrow errno guard in the compile-cache prune catch path is the maintainable fix boundary; changing cache ownership or default cache location would be broader than this cosmetic postinstall bug requires.
