## Summary

Keep open: current main still sends permission-denied compile-cache prune failures to the generic postinstall warning path, and open PR #76362 is the active closing fix candidate rather than a reason to close this before merge.

## What This Changes

Suppress EACCES/EPERM warnings from postinstall compile-cache pruning while preserving warnings for unexpected prune failures.

## Best Possible Solution

Review and land PR #76362, or an equivalent narrow patch with regression coverage for EACCES/EPERM suppression in the compile-cache prune path.

## Reproduction Assessment

Yes. Current source shows permission-denied `readdirSync` or `rmSync` failures reach unconditional warning calls during postinstall; I did not run the root/non-root global install sequence in this read-only pass.

## Solution Assessment

Yes. A narrow errno guard in the compile-cache prune catch paths is the maintainable fix boundary; changing cache location or ownership policy would be broader than this cosmetic postinstall bug requires.
