## Summary

Keep open: current main still emits the noisy postinstall warning for compile-cache EACCES/EPERM failures, and open PR #76362 is the active closing fix candidate rather than a reason to close this issue before merge.

## What This Changes

Suppress expected EACCES/EPERM warnings when bundled-plugin postinstall prunes shared Node compile-cache directories.

## Best Possible Solution

Land PR #76362 or an equivalent narrow patch with regression coverage for both base-directory and per-entry permission-denied prune failures.

## Reproduction Assessment

Yes. Source inspection on current main shows `runBundledPluginPostinstall` reaches `pruneOpenClawCompileCache`, and both the `readdirSync` and `rmSync` catch paths warn unconditionally; I did not run the root/non-root global install sequence in this read-only review.

## Solution Assessment

Yes. A narrow errno guard around the compile-cache prune warning path is the best fix boundary; changing the cache location or ownership policy would be broader than needed for this cosmetic postinstall bug.
