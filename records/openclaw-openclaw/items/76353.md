## Summary

Keep open: current main still warns for every compile-cache prune failure, and open PR #76362 is the active implementation candidate that references this issue with closing syntax. The issue should stay open until that PR lands or is replaced.

## What This Changes

Suppress harmless EACCES/EPERM warnings when postinstall prunes shared Node compile-cache directories.

## Best Possible Solution

Land a narrow postinstall fix, likely via #76362, that treats EACCES/EPERM as expected shared-cache cleanup misses while preserving warnings and regression coverage for other failures.

## Reproduction Assessment

Yes. Current-main source inspection shows both rmSync and readdirSync failures produce the reported warning, and the existing test seam already asserts warning behavior for a mocked prune failure; I did not rerun the root/non-root global install sequence.

## Solution Assessment

Yes. A focused errno guard inside pruneOpenClawCompileCache is the narrow maintainable fix; changing the default cache path to be per-user would be broader than the reported diagnostic bug requires.
