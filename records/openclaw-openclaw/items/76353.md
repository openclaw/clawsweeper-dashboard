## Summary

Keep open: current main still emits the misleading compile-cache prune warning for EACCES/EPERM, and PR #76362 is the linked open fix candidate rather than a reason to close this issue before merge.

## What This Changes

Suppress misleading postinstall warnings when shared compile-cache pruning hits EACCES or EPERM.

## Best Possible Solution

Land PR #76362 or an equivalent narrow patch with regression coverage for EACCES/EPERM suppression while preserving warnings for unexpected prune failures.

## Reproduction Assessment

Yes. Current main reproduces with an in-memory postinstall call: mocked `rmSync` or `readdirSync` throwing EACCES/EPERM reaches the warning path.

## Solution Assessment

Yes. A narrow errno guard in `pruneOpenClawCompileCache` is the maintainable fix boundary; changing cache locations or ownership policy would be broader than this cosmetic postinstall bug requires.
