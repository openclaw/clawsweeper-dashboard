## Summary

Keep open: current main still sends permission-denied compile-cache prune failures to the warning path, and open PR #76362 is the active closing fix candidate rather than a reason to close before merge.

## What This Changes

Suppress expected EACCES/EPERM warnings during postinstall compile-cache pruning when shared cache directories are not writable.

## Best Possible Solution

Review and land PR #76362, or an equivalent narrow patch with regression coverage for EACCES/EPERM suppression while preserving warnings for unexpected prune failures.

## Reproduction Assessment

Yes. Current source shows permission-denied `readdirSync` or `rmSync` failures reach unconditional warning calls during postinstall; I did not run the root/non-root install sequence in this read-only pass.

## Solution Assessment

Yes. A narrow errno guard in the compile-cache prune catch path is the maintainable fix boundary; changing cache locations or ownership policy would be broader than this cosmetic postinstall bug requires.
