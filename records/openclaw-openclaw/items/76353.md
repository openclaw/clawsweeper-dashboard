## Summary

Keep open: current main still unconditionally warns when compile-cache pruning hits permission errors, and open PR #76362 is the active closing fix candidate rather than a reason to close before merge.

## What This Changes

The issue requests suppressing EACCES/EPERM warnings when postinstall prunes the shared OpenClaw Node compile cache.

## Best Possible Solution

Land PR #76362 or an equivalent narrow patch with regression coverage for EACCES/EPERM suppression while preserving warnings for unexpected prune failures.

## Reproduction Assessment

Yes. Current source shows permission-denied `readdirSync` or `rmSync` failures reach unconditional warning calls, and postinstall invokes that helper; I did not run the root/non-root install sequence in this read-only pass.

## Solution Assessment

Yes. A narrow errno guard in the compile-cache prune warning path is the maintainable fix boundary; changing the shared cache location or ownership policy would be broader than this cosmetic postinstall bug requires.
