## Summary

Keep open: current main still emits the compile-cache prune warning for permission-denied cleanup, and open PR #76362 is the active closing fix candidate rather than a reason to close this issue before merge.

## What This Changes

The issue asks for postinstall compile-cache pruning to skip EACCES/EPERM warnings when a shared Node compile cache is not writable by the current user.

## Best Possible Solution

Land PR #76362 or an equivalent narrow patch with regression coverage for EACCES/EPERM suppression while preserving warnings for unexpected prune failures.

## Reproduction Assessment

Yes. Current source shows permission-denied `readdirSync` or `rmSync` failures reach unconditional warning calls during postinstall; I did not run the root/non-root global install sequence in this read-only pass.

## Solution Assessment

Yes. A narrow errno guard in the compile-cache prune catch paths is the maintainable fix boundary; changing cache locations or ownership policy would be broader than this cosmetic postinstall bug requires.
