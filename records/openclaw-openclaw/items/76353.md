## Summary

Keep open: current main still emits the misleading compile-cache prune warning for permission-denied cleanup failures, and open PR #76362 is the active closing fix candidate.

## What This Changes

The issue asks to suppress harmless EACCES/EPERM warnings when postinstall prunes shared Node compile-cache directories.

## Best Possible Solution

Land or close #76362, or an equivalent narrow patch, so EACCES/EPERM prune failures are skipped while unexpected cleanup errors still surface.

## Reproduction Assessment

Yes. A current-main Node probe that injects an `EACCES` `rmSync` failure reproduces the warning path, and source inspection shows the same path is used during postinstall.

## Solution Assessment

Yes. A narrow errno guard in `pruneOpenClawCompileCache` is the best fix boundary; changing the cache location or ownership model would be broader than this cosmetic postinstall bug requires.
