## What This Changes

The issue asks to suppress noisy EACCES/EPERM warnings when the postinstall script cannot prune a shared OpenClaw compile cache.

## Best Possible Solution

Review and land the linked narrow PR that skips EACCES/EPERM compile-cache prune warnings while keeping unexpected prune failures visible.

## Reproduction Assessment

Yes. Source inspection shows any rmSync/readdir failure in the compile-cache prune helper still emits the reported warning, and the existing mocked failure test covers that warning path; I did not run the root/non-root global install sequence.

## Solution Assessment

Yes. The narrow errno guard proposed in the linked PR is the best fit because it changes only the diagnostic path for expected shared-cache permission failures instead of changing cache ownership or location policy.
