## Summary

Keep open: current main still warns unconditionally for permission-denied compile-cache prune failures, and open PR #76362 is the active closing fix candidate rather than a reason to close this issue before merge.

## What This Changes

Suppress misleading postinstall warnings when pruning shared Node compile-cache directories fails with EACCES or EPERM.

## Best Possible Solution

Land PR #76362 or an equivalent narrow patch with regression coverage for EACCES/EPERM suppression and generic-error warnings.

## Reproduction Assessment

Yes. Source inspection on current main shows permission-denied `readdirSync` or `rmSync` failures would reach the unconditional compile-cache warning path; I did not run the root/non-root global install sequence in this read-only review.

## Solution Assessment

Yes. A narrow errno guard in the compile-cache prune catch paths is the maintainable fix boundary; changing cache locations or ownership policy would be broader than this cosmetic postinstall bug requires.
