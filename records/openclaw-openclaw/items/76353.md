## Summary

Keep open: current main still sends permission-denied compile-cache prune failures to the postinstall warning path, and open PR #76362 is the active closing fix candidate rather than a reason to close this issue before merge.

## What This Changes

The issue asks for postinstall compile-cache pruning to silently skip shared-cache EACCES/EPERM failures while preserving warnings for unexpected prune failures.

## Best Possible Solution

Land PR #76362 or an equivalent narrow patch with regression coverage for permission-denied prune suppression and unchanged warnings for unexpected failures.

## Reproduction Assessment

Yes. Source inspection shows EACCES/EPERM from `readdirSync` or `rmSync` would hit unconditional warning calls on current main; I did not run the root/non-root global install sequence in this read-only pass.

## Solution Assessment

Yes. A narrow errno guard in the compile-cache prune catch path is the maintainable fix boundary; changing the cache location or ownership policy would be broader than this cosmetic postinstall bug requires.
