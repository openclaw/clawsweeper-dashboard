## Summary

Keep open: current main still emits the noisy compile-cache prune warning on permission failures, and open PR #76362 is the active closing fix candidate rather than a reason to close before merge.

## What This Changes

The issue asks postinstall compile-cache pruning to suppress expected EACCES/EPERM permission warnings while preserving warnings for unexpected failures.

## Best Possible Solution

Land PR #76362 or an equivalent narrow patch with regression coverage for EACCES/EPERM suppression while preserving warnings for unexpected prune failures.

## Reproduction Assessment

Yes. Current main shows permission-denied `readdirSync` or `rmSync` failures reach unconditional warning calls, and postinstall invokes that helper; I did not run the root/non-root install sequence in this read-only pass.

## Solution Assessment

Yes. A narrow errno guard in the compile-cache prune catch path is the maintainable fix boundary; changing cache locations or ownership policy would be broader than this cosmetic postinstall bug requires.
