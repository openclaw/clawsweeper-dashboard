## Summary

Keep open: current main still warns for every compile-cache prune failure, and the linked open #76362 uses closing syntax for this issue, so cleanup should wait for that PR to be reviewed and merged or closed.

## What This Changes

Suppress misleading postinstall warnings when compile-cache pruning hits EACCES or EPERM in shared cache directories.

## Best Possible Solution

The postinstall helper should treat permission-denied shared compile-cache cleanup as expected while preserving diagnostics for other prune failures.

## Reproduction Assessment

Yes. Source reproduction is high-confidence by injecting EACCES or EPERM from rmSync or readdirSync in pruneOpenClawCompileCache; current main catches it and logs the reported warning.

## Solution Assessment

Yes. A narrow errno guard, as described by #76362, is the maintainable direction; switching to a per-user cache path would be a broader behavior change for a cosmetic warning.
