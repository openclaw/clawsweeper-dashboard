## Summary

Keep open: current main still warns for all compile-cache prune failures, and the linked open PR #76362 is the active implementation candidate for this issue.

## What This Changes

Suppress EACCES/EPERM warnings during postinstall compile-cache pruning while preserving warnings for unexpected prune failures.

## Best Possible Solution

Review and land PR #76362, or an equivalent narrow guard, with regression coverage for EACCES/EPERM and preserved warnings for other errors.

## Reproduction Assessment

Yes. Current main has a clear source reproduction path: any `readdirSync` or `rmSync` failure in `pruneOpenClawCompileCache` logs the warning, and the existing test already proves the removal-failure branch warns.

## Solution Assessment

Yes. A narrow EACCES/EPERM guard in the existing prune helper is the most maintainable fix; changing the cache path to be per-user would be a broader behavior change.
