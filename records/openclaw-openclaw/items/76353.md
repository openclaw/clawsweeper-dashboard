## Summary

Keep open: current main still warns on permission-denied compile-cache prune failures, and open PR #76362 is the active closing fix candidate rather than a reason to close this issue before merge.

## What This Changes

Suppress misleading postinstall WARN output when compile-cache pruning fails with EACCES or EPERM in shared or root-owned cache layouts.

## Best Possible Solution

Land PR #76362 or an equivalent narrow patch with regression coverage for EACCES/EPERM suppression while preserving warnings for unexpected failures.

## Reproduction Assessment

Yes. Source inspection on current main shows permission-denied `readdirSync` or `rmSync` failures would reach the unconditional compile-cache warning path; I did not run the root/non-root global install sequence in this read-only review.

## Solution Assessment

Yes. A narrow errno guard in the compile-cache prune catch path is the maintainable fix boundary; changing cache locations or ownership policy would be broader than this cosmetic postinstall bug requires.
