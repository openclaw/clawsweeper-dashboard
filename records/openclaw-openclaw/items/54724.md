## Summary

Keep open: current main already carries the keyed/select-stability subset, but the Agents Overview catalog-hydration path is not fully implemented there, and this stale PR would regress the current configured model-picker contract.

## What This Changes

The PR remounts Agents Overview per selected agent, adds shared model-catalog refresh helpers and loading UI, wires Agents tab paths to load models, and adds UI/controller tests.

## Best Possible Solution

Rebase or replace the PR so it only adds the missing Agents Overview catalog-hydration behavior, preserves the current configured model-list contract, and includes the required changelog and regression coverage.

## Reproduction Assessment

Source-level yes for the remaining concern: current main's Agents tab refresh loads agents/config but not the model catalog, so an overview-only first load can render with an empty chatModelCatalog. I did not run browser tests because this review was read-only.

## Solution Assessment

No. The branch is directionally useful, but its shared requestModels helper calls models.list with {} instead of the current configured picker view; the safer fix is to rebase onto current main and preserve current model-list semantics while adding the missing hydration path.
