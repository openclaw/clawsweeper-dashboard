## Summary

Keep open: current main still omits install records from channel catalog discovery, so the core fix is useful, but this PR should not merge as written because the trusted Weixin catalog entry is floating and the user-facing change lacks a changelog entry.

## What This Changes

The PR lazy-loads and forwards persisted plugin install records into channel catalog discovery, adds regression coverage, and adds Weixin to the official external channel catalog.

## Best Possible Solution

Land the install-record forwarding with focused regression coverage, add the required changelog entry, and keep Weixin official catalog metadata only if it is pinned with vetted integrity or split to a separate maintainer-reviewed follow-up.

## Reproduction Assessment

Yes. Source inspection shows the catalog path reaches `discoverOpenClawPlugins` without `installRecords`, while discovery only visits persisted install-record paths when that parameter is present.

## Solution Assessment

No, not as written. The install-record forwarding is the narrow fix for the reported channel resolution bug, but the floating trusted Weixin catalog entry and missing changelog should be corrected before merge.
