## Summary

This PR fixes a real source-visible channel catalog bug, but it is not merge-ready because the new third-party Weixin official catalog entry violates the existing exact-version-plus-integrity policy and would fail the catalog policy test.

## What This Changes

The PR forwards persisted plugin install records into channel catalog discovery, adds coverage for that behavior, and adds Weixin to the official external channel catalog.

## Best Possible Solution

Keep the install-record forwarding fix and regression tests, but pin or remove the Weixin official catalog entry and add the required changelog entry before merge.

## Reproduction Assessment

Yes, from source inspection: current main routes channel catalog listing through `listChannelCatalogEntries`, which omits `installRecords`, while discovery only scans persisted installed plugin paths when that map is supplied. I did not run the CLI because this review is read-only.

## Solution Assessment

No, not as-is. The core forwarding approach is narrow and matches sibling discovery callers, but the bundled catalog addition needs an exact npm version plus `expectedIntegrity` or should be split out.
