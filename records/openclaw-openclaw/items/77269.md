## Summary

Keep open: the core fix is well targeted and the earlier catalog pin/changelog blockers are addressed, but the force-pushed patch now introduces a concrete import-cycle regression that is already failing PR-head CI.

## What This Changes

The PR lazy-loads installed plugin records into channel catalog discovery, adds unit coverage and changelog entries, and adds a pinned Weixin external channel catalog entry.

## Best Possible Solution

Keep the install-record forwarding and pinned catalog entry, but read the ledger through a read-only module that does not pull channel/config initialization into channel-catalog-registry.

## Reproduction Assessment

Yes. Source inspection shows current main omits installRecords on the channel catalog discovery path, and PR-head CI reproduces the new import-cycle failure after the attempted fix.

## Solution Assessment

No, not as-is. The forwarding approach is the right narrow fix, but the new import should avoid the installed-plugin-index-records barrel and use a read-only ledger reader seam instead.
