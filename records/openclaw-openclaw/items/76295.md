Keep open. Multiple fresh Docker reports still show 8-16s in the named prep stage through v2026.5.2, and current main still builds core, OpenClaw, and plugin tools synchronously before `stream-ready`; the existing cache fixes are relevant but not enough proof to close this as implemented.

The issue reports a regression where `core-plugin-tools` adds roughly 8 seconds or more of prep latency per agent turn on Docker installs since v2026.4.24.

## Best Possible Solution

Keep this open until a focused performance fix or benchmark proves `core-plugin-tools` no longer repeats unsafe plugin runtime/config work on every turn while preserving plugin tool availability and execution behavior.
