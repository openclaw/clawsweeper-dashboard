This PR adds gateway auth dangerous-config flag checks for `gateway.auth.mode="none"` and trusted-proxy allow-all states, plus a Zod refinement rejecting whitespace-only `gateway.auth.trustedProxy.userHeader` values.

## Best Possible Solution

Rebase the PR onto current main, add the gateway auth checks in the current core dangerous-flag collection path, cover both absent and empty trusted-proxy allowlists, and add focused collector/startup-warning tests. Keep the `userHeader` tightening only with the required generated-schema and docs consistency checks.
