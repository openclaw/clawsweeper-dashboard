## What This Changes

The issue asks OpenClaw to prevent leaked Gateway session write locks from blocking later session operations for 30+ minutes, with heartbeat, shorter stale timeout, and exception-cleanup mitigations.

## Best Possible Solution

Consolidate this report under https://github.com/openclaw/openclaw/issues/13744, preserve the field evidence there, and implement one agreed stale/max-hold/env policy plus a designed narrower or single-writer transcript lock lifecycle.

## Reproduction Assessment

Yes.
