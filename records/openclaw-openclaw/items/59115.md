The branch appends Slack shared/forwarded attachment text or fallback content to thread history output and adds Slack monitor regression tests.

## Best Possible Solution

Rebase onto current main, implement one capped renderable-share extraction path in `extensions/slack/src/monitor/thread.ts`, preserve the existing forwarded-message heading format, add focused regression coverage, and attach redacted Slack proof.

## Reproduction Assessment

Yes, at source level. Mock `conversations.replies` with a thread reply whose `text` is empty and whose `attachments` contains an `is_share` item with `text` or `fallback`; current main drops it because the runtime helper retains only text or file messages.

## Solution Assessment

No as-is. The requested fix is narrow and appropriate, but this branch must move the logic to `thread.ts` and derive retention from the same capped rendered-share subset used for output.
