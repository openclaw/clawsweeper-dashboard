## Summary

Keep open: current main still drops forwarded/shared Slack attachment text from thread history, and this PR targets the right bug, but the branch is not merge-ready because it edits the old helper location, still has a retain/render mismatch, and lacks attached real Slack proof.

## What This Changes

The PR changes Slack monitor media/thread-history code and tests to append readable `is_share` attachment text or fallback content to Slack thread history entries.

## Best Possible Solution

Apply one capped renderable-share extraction path in `extensions/slack/src/monitor/thread.ts`, preserve the existing forwarded-message heading format, add focused regression coverage, and require redacted Slack proof before merge.

## Reproduction Assessment

Yes, at source level. Mock `conversations.replies` with a thread reply whose top-level `text` is empty and whose `attachments` contains an `is_share` item with `text` or `fallback`; current `thread.ts` drops it at the text/files filter.

## Solution Assessment

No as-is. The requested fix is narrow and appropriate, but the branch must move the logic to `thread.ts` and derive retention from the same capped/renderable share subset used for output.
