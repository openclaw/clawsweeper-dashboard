## Summary

Keep open: the Slack context bug is real and the PR is pointed in the right direction, but current main moved the active thread-history helper to `thread.ts`, the branch still patches the old `media.ts` implementation, real Slack proof is missing, and one retain/render mismatch remains.

## What This Changes

The PR adds Slack thread-history parsing and tests so forwarded/shared `is_share` attachment text or fallback is appended to retained thread context.

## Best Possible Solution

Port the change into `extensions/slack/src/monitor/thread.ts`, derive retention and rendering from one capped readable forwarded-attachment subset, keep focused tests, and attach redacted Slack proof before merge.

## Reproduction Assessment

Yes, at source level. A mocked `conversations.replies` response whose only readable history content is an `is_share` attachment will be skipped by current main's `thread.ts` because it ignores `attachments`.

## Solution Assessment

No. The requested behavior is the right narrow fix, but this branch needs to target the active `thread.ts` helper and use one capped renderable attachment subset for both retention and output.
