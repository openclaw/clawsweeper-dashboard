## Summary

Keep open: the Slack context-loss bug is valid, but the branch is not mergeable as-is because current main moved the runtime helper to `thread.ts`, the PR still edits the old `media.ts` helper, the retain/render share logic can still emit a blank retained entry, and external real Slack proof is missing.

## What This Changes

The PR adds Slack forwarded/shared attachment text extraction to thread history rendering and Slack monitor regression tests.

## Best Possible Solution

Apply one capped renderable-share extraction path in `extensions/slack/src/monitor/thread.ts`, preserve the existing forwarded-message heading format, add focused regression coverage, and require redacted Slack proof before merge.

## Reproduction Assessment

Yes, at source level. Mock `conversations.replies` with a thread reply whose `text` is empty and whose `attachments` contains an `is_share` item with `text` or `fallback`; current main drops it because `thread.ts` retains only text or file messages.

## Solution Assessment

No as-is. The requested fix is narrow and appropriate, but this branch must move the logic to `extensions/slack/src/monitor/thread.ts` and derive retention from the same capped rendered-share subset used for output.
