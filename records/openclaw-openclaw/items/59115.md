## Summary

Keep open: the Slack bug is source-reproducible on current main, but this branch is stale against the helper move and still lacks redacted real Slack behavior proof.

## What This Changes

This PR adds Slack thread-history handling and tests for readable `is_share` attachments, appending forwarded text or fallback content to history entries.

## Best Possible Solution

Rebase and implement one capped renderable-share extraction path in `extensions/slack/src/monitor/thread.ts`, mirroring `resolveSlackAttachmentContent`, with focused regression tests and redacted real Slack proof.

## Reproduction Assessment

Yes, at source level. Mock `conversations.replies` with an attachment-only thread reply whose `attachments` contains a readable `is_share` item; current main drops it at the text/files filter in `thread.ts`.

## Solution Assessment

No as-is. The requested fix is narrow and appropriate, but the branch must move the logic to `thread.ts` and derive retention from the same capped/renderable share subset used for output.
