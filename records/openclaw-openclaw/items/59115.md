## Summary

Keep open. Current main still drops Slack forwarded/share-only attachments from thread history, and the PR is useful but not merge-ready because it needs adaptation to the current `thread.ts` implementation, one capped-retention logic fix, and a changelog entry.

## What This Changes

The PR branch adds Slack forwarded/shared attachment text or fallback content to thread history entries and adds unit coverage for forwarded attachments and body-less shares.

## Best Possible Solution

Adapt the source PR onto `extensions/slack/src/monitor/thread.ts`, compute one capped forwarded-share subset for both retention and rendering, add focused regression tests, and add the required user-facing changelog entry.

## Reproduction Assessment

Yes. Mock `conversations.replies` with a thread message whose `text` is empty and whose `attachments` include an `is_share` item with `text` or `fallback`; current main filters it out because `resolveSlackThreadHistory` only retains text or file messages.

## Solution Assessment

No as-is. The direction is the narrowest maintainable fix, but the branch needs to target the current helper location and make the retain predicate match the capped rendered attachment subset before it should merge.
