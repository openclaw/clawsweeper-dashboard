## Summary

Keep this PR open. Current main still drops Slack forwarded/shared attachment content from thread history, so the user-visible bug is real and the PR is aimed at the right Slack-owned surface, but the branch is stale against the current `thread.ts` refactor and still has a capped-render mismatch that can retain a blank history item.

## What This Changes

The PR branch adds forwarded/shared Slack attachment text or fallback content to thread history entries and adds unit coverage for forwarded attachments, forwarded-only messages, ignored non-share attachments, and body-less shares.

## Best Possible Solution

Adapt the source PR to the current `extensions/slack/src/monitor/thread.ts` implementation, preserve existing file-placeholder/thread-history semantics, derive one capped renderable forwarded-attachment subset for both retention and formatting, add focused regression tests, and include the required changelog entry.

## Solution Assessment
