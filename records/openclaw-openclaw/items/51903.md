The issue requests fixing the `nodes` tool so schema-valid `describe` calls without `node` do not fail with `node required`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/51926, or replace it with a maintained patch that accepts omitted `node` for read-only `describe`, preserves explicit-node requirements for targeted actions, and adds focused regression coverage.

## Reproduction Assessment

Yes. Current `main` accepts omitted `node` in the flattened schema, then the `describe` branch calls `readStringParam(params, "node", { required: true })`, which deterministically throws `node required`.

## Solution Assessment
