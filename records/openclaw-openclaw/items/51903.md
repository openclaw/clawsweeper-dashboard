## What This Changes

The issue asks for the agents `nodes` tool `describe` action to handle omitted `node` consistently with its optional schema instead of failing with `node required`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/51926, or replace it with a maintained patch that intentionally accepts omitted `node` for read-only `describe` while preserving explicit-node requirements for targeted actions.

## Reproduction Assessment

Yes. Source inspection shows `{ "action": "describe" }` is allowed by the schema, then the implementation immediately requires `node` and throws `node required`; no live Gateway run was attempted in this read-only review.

## Solution Assessment
