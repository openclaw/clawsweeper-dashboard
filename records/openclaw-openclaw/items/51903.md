## Summary

Keep open. Current main still has the reported schema/runtime mismatch, and https://github.com/openclaw/openclaw/pull/51926 is an open same-author closing PR, so the issue should stay open until that PR is reviewed, landed, replaced, or closed.

## What This Changes

The issue asks for the agents `nodes` tool to stop rejecting `action: "describe"` calls that omit `node`, because the schema presents `node` as optional.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/51926, or replace it with a maintained patch that intentionally accepts omitted `node` for read-only `describe` while preserving explicit-node requirements elsewhere.

## Reproduction Assessment

Yes, by source inspection. Current main still accepts omitted `node` in the tool schema and then the `describe` branch requires it before default-node resolution can run; I did not run live tests because this review is read-only.

## Solution Assessment

Unclear as a product choice. Accepting omitted `node` for read-only `describe` is the right surface, but maintainers still need to choose whether it should default to a node or return discovery/list data in multi-node setups.
