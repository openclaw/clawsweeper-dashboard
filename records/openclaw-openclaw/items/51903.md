## What This Changes

The issue reports that the agents `nodes` tool allows omitted `node` in schema but rejects `action: "describe"` at runtime with `node required`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/51926, or replace it with a maintained patch that accepts omitted `node` for read-only `describe` while preserving explicit-node requirements for targeted actions.

## Reproduction Assessment

Yes. Source inspection on current main shows the schema permits omitted `node`, while the `describe` branch hard-requires it and throws `node required` before default-node resolution can run.

## Solution Assessment

Unclear. Reusing the existing default-node resolver is a narrow implementation path, but maintainers still need to confirm whether omitted-node `describe` should default, list/discover, or use hybrid semantics.
