## Summary

Keep open. Current main still has the reported schema/runtime mismatch, and https://github.com/openclaw/openclaw/pull/51926 remains an open same-author closing implementation path.

## What This Changes

The issue asks for the agent `nodes` tool to stop rejecting `action: "describe"` calls that omit `node`, because the tool schema currently marks `node` optional.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/51926, or replace it with a maintained patch that accepts omitted `node` for read-only `describe` and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows the schema permits omitted `node`, while the `describe` branch hard-requires it and throws `node required` before default-node resolution can run.

## Solution Assessment

Unclear. Reusing the existing default-node resolver is a narrow fix for the proposed behavior, but maintainers still need to confirm whether omitted-node `describe` should default, list/discover, or use hybrid semantics.
