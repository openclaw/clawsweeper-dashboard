## Summary

Keep open. Current `main` and release `v2026.5.20` still have the reported `nodes describe` schema/runtime mismatch, and the same-author closing PR at https://github.com/openclaw/openclaw/pull/51926 remains open, so this issue should stay open until that PR is reviewed, landed, or replaced.

## What This Changes

The issue asks for the agents `nodes` tool to handle `action: "describe"` without `node` consistently with its optional schema instead of throwing `node required`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/51926, or replace it with a maintained patch that accepts omitted `node` for read-only `describe`, preserves explicit-node requirements for targeted actions, and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current `main` shows the schema accepts omitted `node`, then the `describe` branch requires `node` and would throw `node required` before gateway dispatch.

## Solution Assessment

Unclear. Accepting omitted `node` for read-only `describe` is the right fix boundary, but maintainers still need to choose whether omitted-node behavior should default to a node, return discovery/list output, or combine those behaviors.
