## What This Changes

The issue reports that the agents `nodes` tool schema allows `action: "describe"` without `node`, while the implementation rejects that call with `node required`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/51926, or replace it with a maintained patch that accepts omitted `node` for read-only `describe`, preserves explicit-node requirements for targeted actions, and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current `main` shows the schema accepts omitted `node`, then the `describe` branch requires `node` and would throw `node required`.

## Solution Assessment

Unclear. Accepting omitted `node` for read-only `describe` is the right fix boundary, but maintainers still need to choose whether omitted-node behavior should default to a node or return discovery/list output.
