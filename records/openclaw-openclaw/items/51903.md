## Summary

Keep open. Current main still permits omitted `node` in the flat `nodes` tool schema but the `describe` branch hard-requires `node`, and the same-author open implementation PR at https://github.com/openclaw/openclaw/pull/51926 remains the active closing path.

## What This Changes

Make the agents `nodes` tool handle `describe` without `node` consistently with its schema instead of throwing `node required`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/51926, or replace it with a maintained patch that accepts omitted `node` for read-only `describe` while preserving explicit-node requirements for targeted actions and adding focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows the schema permits `{ "action": "describe" }`, then the `describe` branch hard-requires `node` and throws before default-node resolution can run.

## Solution Assessment

Yes, if maintainers accept default-node semantics for `describe`. Reusing `resolveNodeId(..., true)` is the narrowest code path, while returning list-style discovery output would be a broader product choice.
