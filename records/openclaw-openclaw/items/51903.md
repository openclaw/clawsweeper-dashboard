## Summary

Keep open. Current main still has the reported schema/runtime mismatch, and same-author open PR #51926 remains the active closing implementation path, so the issue should wait for PR review, landing, or replacement rather than be closed or split into a separate repair job.

## What This Changes

The issue asks for the agents `nodes` tool to handle `action: "describe"` without failing on an omitted `node` parameter when the tool schema permits that shape.

## Best Possible Solution

Review and land PR #51926, or replace it with a maintained patch in `src/agents/tools/nodes-tool.ts` that intentionally accepts omitted `node` for the read-only `describe` action while preserving explicit-node requirements for targeted actions and adding focused coverage in `src/agents/tools/nodes-tool.test.ts`.

Yes. The source path is enough to reproduce with high confidence: `nodes` `describe` without `node` is schema-valid, then `readStringParam(params, "node", { required: true })` throws before resolver or gateway behavior can run.

## Solution Assessment

Unclear. Making omitted-node `describe` intentional is the right narrow boundary, but maintainers still need to decide whether PR #51926's default-node behavior is preferable to discovery/list output or combined semantics.
