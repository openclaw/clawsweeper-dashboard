## Summary

Keep open. Current main still has the reported schema/runtime mismatch, and https://github.com/openclaw/openclaw/pull/51926 is an open same-author implementation path with closing syntax, so this issue should not be closed before that PR is reviewed, landed, or replaced.

## What This Changes

The issue asks for the agents `nodes` tool to stop rejecting `action: "describe"` when `node` is omitted despite the schema marking it optional.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/51926, or replace it with a maintained patch that accepts omitted `node` for read-only `describe` while preserving explicit-node requirements for targeted actions and adding focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible with high confidence. Current main accepts omitted `node` in the tool schema, but the `describe` branch calls `readStringParam` with `{ required: true }`, which deterministically throws `node required` before default resolution can run.

## Solution Assessment

Unclear. Accepting omitted `node` for `describe` is the right bug boundary, but maintainers still need to choose whether the response should default to a node, return discovery/list data, or combine those behaviors.
