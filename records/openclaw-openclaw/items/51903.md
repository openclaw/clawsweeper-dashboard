## What This Changes

The issue requests fixing the agents `nodes` tool so `action: "describe"` no longer accepts omitted `node` in the schema only to fail at runtime with `node required`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/51926, or replace it with a maintained patch that accepts omitted `node` for read-only `describe`, preserves explicit-node requirements for targeted actions, and adds focused regression coverage.

## Reproduction Assessment

Yes, by source inspection. Current main accepts omitted `node` in the flattened schema, then the `describe` branch calls `readStringParam(params, "node", { required: true })`, which deterministically throws `node required`.

## Solution Assessment

Unclear. Accepting omitted `node` for read-only `describe` is the right fix boundary, but maintainers still need to choose whether the no-node behavior should select a default node, return discovery/list output, or combine those semantics.
