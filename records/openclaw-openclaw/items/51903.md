The issue requests fixing the agents `nodes` tool so schema-valid `describe` calls without `node` no longer fail with `node required`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/51926, or replace it with a maintained patch that defines omitted-node `describe` semantics, keeps targeted actions explicit, and adds focused regression coverage.

## Reproduction Assessment

Yes, by source inspection. Current main accepts omitted `node` in the flat schema, then the `describe` branch requires `node` and deterministically throws `node required` before gateway dispatch.

## Solution Assessment

Unclear. Accepting omitted `node` for read-only `describe` is the right repair boundary, but maintainers still need to choose whether omitted input should select a default node or return discovery/list output.
