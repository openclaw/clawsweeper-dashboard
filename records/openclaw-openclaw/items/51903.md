## What This Changes

The issue requests fixing the agents `nodes` tool so `action: "describe"` without `node` no longer fails with `node required` while the schema permits omitted `node`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/51926, or replace it with a maintained patch in `src/agents/tools/nodes-tool.ts` that intentionally handles omitted `node` for read-only `describe` and adds focused regression coverage in `src/agents/tools/nodes-tool.test.ts`.

## Reproduction Assessment

Yes, by source inspection. Current `main` accepts omitted `node` in the flat schema, then the `describe` branch requires `node` and `readStringParam` deterministically throws `node required` before default-node resolution can run.

## Solution Assessment

Unclear. Handling omitted `node` for read-only `describe` is the right area, but maintainers still need to choose default-node versus discovery/list semantics; the linked PR implements the default-node path but lacks direct regression coverage.
