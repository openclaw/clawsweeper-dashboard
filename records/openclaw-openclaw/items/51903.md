## Summary

Keep open. Current main still has the reported schema/runtime mismatch, and same-author open PR #51926 remains the active closing implementation path, so this issue should wait for PR review, landing, or replacement.

## What This Changes

The issue asks for the agents `nodes` tool to stop rejecting `action: "describe"` when `node` is omitted even though the flattened schema marks `node` optional.

## Best Possible Solution

Review and land PR #51926, or replace it with a maintained patch that intentionally accepts omitted `node` for the read-only `describe` action while preserving explicit-node requirements for targeted actions and adding focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main is enough: the schema permits omitted `node`, the `describe` branch hard-requires it, and the shared parameter reader throws the reported `node required` error before gateway resolution.

## Solution Assessment

Unclear. Allowing omitted `node` for `describe` is the right narrow boundary, but maintainers still need to decide whether the best behavior is default-node selection, discovery/list output, or combined semantics.
