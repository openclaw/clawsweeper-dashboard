## Summary

Keep open. Current main still has the reported schema/runtime mismatch, and the same-author open PR #51926 remains the active closing implementation path, so this should wait for normal PR review, landing, or replacement rather than be closed or split into a separate repair job.

## What This Changes

The issue reports that the agents `nodes` tool schema allows `action: "describe"` without `node`, while the implementation still requires `node` and throws `node required`.

## Best Possible Solution

Review and land PR #51926, or replace it with a maintained patch in `src/agents/tools/nodes-tool.ts` that intentionally accepts omitted `node` for the read-only `describe` action while preserving explicit-node requirements for targeted actions, with focused coverage in `src/agents/tools/nodes-tool.test.ts`.

## Reproduction Assessment

Yes. The source path is enough to reproduce with high confidence: `nodes` `describe` without `node` is schema-valid, then `readStringParam(params, "node", { required: true })` throws before resolver or gateway behavior can run.

## Solution Assessment

Unclear. Supporting omitted-node `describe` is the right narrow boundary, but maintainers still need to decide whether PR #51926's default-node behavior is preferable to discovery/list output or combined semantics.
