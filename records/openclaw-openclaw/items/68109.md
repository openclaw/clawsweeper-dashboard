## What This Changes

The issue requests that successful Linux/systemd `openclaw gateway restart` calls return a reliable success exit instead of `exit=null` when invoked from the managed service context.

## Best Possible Solution

Continue review on https://github.com/openclaw/openclaw/pull/66735, land a hardened systemd self-restart handoff with live user-systemd proof, and keep this report linked there as user evidence.

## Reproduction Assessment

Yes, at source level. The reporter provided a concrete Ubuntu/user-systemd log, and current main plus the latest release still await `systemctl --user restart` inside a `KillMode=control-group` unit, which explains the null exit path without needing a new product decision.

## Solution Assessment

Yes, the best solution is the focused systemd self-restart handoff being tracked in the canonical PR. A late `process.exitCode = 0` workaround would not address the caller being terminated before JavaScript continuation runs.
