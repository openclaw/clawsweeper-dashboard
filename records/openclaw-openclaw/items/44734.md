## Summary

Keep open. Current main and v2026.5.12 still advertise `.ogg` and `.opus` as multimodal memory audio, preserve the detected raw MIME/data, and forward it unchanged to Gemini embedding requests; the broader open Gemini multimodal tracker is related, but this narrower provider-format bug remains distinct.

## What This Changes

The issue reports that memory multimodal indexing accepts OGG/Opus audio files but Gemini embeddings rejects them when OpenClaw sends them as raw inline data.

## Best Possible Solution

Define a provider-aware multimodal audio contract: transcode unsupported OGG/Opus into a Gemini-supported representation with proof, or stop advertising/indexing those formats as raw Gemini audio and emit a clear skip or warning.

## Reproduction Assessment

Yes, at source level. Enabling multimodal audio indexing for `.ogg` or `.opus` classifies the file, preserves its detected audio MIME, sends it to Gemini as raw `inlineData`, and rethrows non-size provider errors.

## Solution Assessment

Unclear. Auto-transcoding is the most complete user-visible fix but adds dependency and runtime policy, while provider-aware filtering plus docs is narrower but changes the advertised support contract.
