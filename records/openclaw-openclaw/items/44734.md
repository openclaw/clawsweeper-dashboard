## Summary

Keep open: current main and the latest release still advertise `.ogg` and `.opus` as Gemini multimodal memory audio inputs, preserve the detected MIME/raw bytes, and forward them unchanged to Gemini inline data. The root cause remains source-reproducible, but maintainers need to choose whether OpenClaw should preserve the advertised formats through transcoding or narrow the provider-specific contract with skip/warn behavior and docs.

## What This Changes

The issue reports that Gemini-backed multimodal memory indexing accepts OGG/Opus audio as indexable but sends it to Gemini in a format the provider rejects.

## Best Possible Solution

Define a provider-specific Gemini multimodal audio contract, then add tests and docs for either converting unsupported audio to MP3/WAV before upload or rejecting/skipping unsupported formats with a clear warning.

## Reproduction Assessment

Yes, at source level. Current main classifies `.ogg` and `.opus` as multimodal audio, preserves the detected MIME/raw bytes, and forwards them unchanged to Gemini inline data; I did not run a live Gemini request.

## Solution Assessment

Unclear. The defect is narrow, but maintainers need to choose whether OpenClaw should transcode unsupported audio or narrow the provider-specific supported-format contract with skip/warn behavior and docs.
