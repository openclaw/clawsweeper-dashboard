## Label Justifications

- `P2`: This affects an opt-in Gemini multimodal memory indexing path with a conversion workaround rather than core runtime availability.
- `impact:session-state`: The failure can prevent selected audio files from being embedded into searchable memory state and may abort an indexing run.

## Summary

Keep open: current main and v2026.5.19 still advertise OGG/Opus as multimodal memory audio, preserve the detected raw MIME/data, and pass it through to Gemini without provider-aware filtering or transcoding.

## What This Changes

The issue reports that Gemini multimodal memory indexing accepts OGG/Opus audio but forwards it to Gemini as raw unsupported audio, causing provider 400 errors.

## Best Possible Solution

Define a provider-aware multimodal audio contract: either transcode advertised-but-provider-unsupported audio into a Gemini-accepted representation, or stop advertising those formats as Gemini-indexable and skip them with clear warnings and docs.

## Reproduction Assessment

Yes, at source level. Current main classifies `.ogg` and `.opus` as multimodal audio, preserves detected MIME/raw bytes, forwards them unchanged to Gemini inline data, and rethrows non-size provider errors; I did not run a live Gemini request.

## Solution Assessment

Unclear. Auto-transcoding is the most complete user-visible fix, but maintainers need to decide whether OpenClaw should add bounded transcode behavior or narrow the provider-specific supported-format contract.
