## Summary

Keep open: current main and the latest release still advertise OGG/Opus as Gemini multimodal memory audio, preserve the detected MIME/raw bytes, and send them to Gemini unchanged. The source-level issue remains clear, but the fix needs a maintainer decision between transcoding support and provider-aware skip/warn behavior with docs.

## What This Changes

Fix Gemini multimodal memory indexing so OGG/Opus audio under `memorySearch.extraPaths` is either embedded safely or no longer advertised and sent as unsupported raw media.

## Best Possible Solution

Define a provider-specific Gemini multimodal audio contract, then add tests and docs for either converting unsupported audio to MP3/WAV or rejecting/skipping it before provider upload with a clear warning.

## Reproduction Assessment

Yes, at source level. Current main classifies `.ogg` and `.opus` as multimodal audio, preserves the detected MIME/raw bytes, and forwards them unchanged to Gemini inline data; I did not run a live Gemini request.

## Solution Assessment

Unclear. The defect is narrow, but maintainers need to choose whether OpenClaw should transcode unsupported audio or narrow the provider-specific supported-format contract with skip/warn behavior and docs.
