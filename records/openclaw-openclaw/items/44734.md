## Label Justifications

- `P2`: This affects an opt-in Gemini multimodal memory indexing path with a conversion workaround rather than core runtime availability.
- `impact:session-state`: The failure prevents selected files from being embedded into searchable memory state and can abort the memory indexing run.

## Summary

Keep open: current main and shipped v2026.5.18 still advertise OGG/Opus as Gemini multimodal memory audio, preserve the detected MIME/raw bytes, forward them unchanged to Gemini, and only skip multimodal provider errors that look size-related.

## What This Changes

Fix or document Gemini multimodal memory indexing for OGG/Opus audio files so unsupported raw audio does not abort indexing.

## Best Possible Solution

Define a provider-aware multimodal audio contract: either transcode OGG/Opus into a Gemini-accepted representation with bounded dependency behavior, or stop advertising those formats as Gemini-indexable and skip them with clear warnings and docs.

## Reproduction Assessment

Yes, at source level. Current main classifies `.ogg` and `.opus` as multimodal audio, preserves their detected MIME and bytes, forwards them unchanged to Gemini inline data, and rethrows non-size provider errors; I did not run a live Gemini request.

## Solution Assessment

Unclear. Auto-transcoding is the most complete user-visible fix, but maintainers need to decide whether OpenClaw should add transcode dependency behavior or instead narrow the provider-specific supported-format contract.
