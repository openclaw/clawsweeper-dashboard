## Label Justifications

- `P2`: This affects an opt-in Gemini multimodal memory indexing path with a conversion workaround rather than core runtime availability.
- `impact:session-state`: The failure can prevent selected audio files from being embedded into searchable memory state and may abort an indexing run.

## Summary

Keep open: current main and v2026.5.19 still advertise OGG/Opus as Gemini multimodal memory audio, while the indexing path preserves the detected raw MIME/data and forwards it to Gemini without provider-aware filtering or transcoding.

## What This Changes

Make Gemini multimodal memory indexing handle OGG/Opus audio safely, or narrow the documented supported-format contract so unsupported audio is skipped or warned before provider rejection.

## Best Possible Solution

Define a provider-aware multimodal audio contract: transcode unsupported audio into a Gemini-accepted representation, or stop advertising those formats as Gemini-indexable and skip them with clear warnings and docs.

## Reproduction Assessment

Yes, at source level. Current main classifies `.ogg` and `.opus` as multimodal audio, preserves detected MIME/raw bytes, forwards them unchanged to Gemini inline data, and rethrows non-size provider errors; I did not run a live Gemini request.

## Solution Assessment

Unclear. Auto-transcoding is the most complete user-visible fix, but maintainers need to decide whether OpenClaw should add bounded transcode behavior or narrow the provider-specific supported-format contract.
