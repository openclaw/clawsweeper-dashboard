## Summary

Keep open. Current main and the latest release still allow OGG/Opus multimodal memory audio, preserve the detected MIME type, and pass it through to Gemini inlineData; the remaining fix needs a maintainer decision between narrowing/skipping unsupported audio and owning transcoding.

## What This Changes

Issue reports Gemini multimodal memory indexing failing for OGG/Opus audio accepted from memorySearch.extraPaths and sent to Gemini as raw inline data.

## Best Possible Solution

Make Gemini multimodal memory provider-aware: reject or skip unsupported audio with clear diagnostics and aligned docs, or add an explicitly approved and tested transcode path if maintainers want OGG/Opus to remain user-supported.

## Reproduction Assessment

Yes, at source level: current main accepts OGG/Opus as multimodal audio, stores the detected MIME type, and forwards it unchanged to Gemini inlineData. A live Gemini run with redacted credentials would still be useful before implementation.

## Solution Assessment

No automatic fix is clearly best yet. Maintainers should choose whether the product should reject/skip unsupported Gemini audio, narrow the documented supported formats, or own transcoding as a new dependency-backed behavior.
