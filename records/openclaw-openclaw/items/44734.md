## Summary

Keep open. Current main still advertises OGG/Opus as multimodal memory audio formats and forwards their detected MIME/data to Gemini unchanged, while Gemini Embedding 2 documents audio support as MP3/WAV only.

## What This Changes

The issue requests that Gemini multimodal memory indexing handle OGG/Opus audio files instead of failing with a provider 400.

## Best Possible Solution

Define Gemini-specific multimodal audio handling: either approved optional transcode to MP3/WAV before embedding, or restrict advertised/accepted audio formats and skip unsupported files with a clear warning so indexing continues.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main classifies `.ogg`/`.opus`, preserves their MIME and bytes, and forwards them unchanged to Gemini where the documented embedding audio formats are MP3/WAV. I did not run a live provider call because no Google/Gemini key was available.

## Solution Assessment

Unclear. The narrowest safe code change is provider-aware validation plus docs and warning behavior, but actually preserving OGG/Opus support requires a maintainer-approved transcoding policy.
