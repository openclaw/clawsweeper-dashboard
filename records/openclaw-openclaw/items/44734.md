## Summary

Keep open. Current main still advertises OGG/Opus as multimodal memory audio, preserves the detected MIME and original bytes, forwards them unchanged to Gemini inlineData, and only treats oversized multimodal errors as non-fatal. The reported Gemini OGG/Opus failure path remains unresolved on main and in the latest shipped tag.

## Best Possible Solution

Keep this open and fix the Gemini multimodal memory path. The safest narrow fix is provider-aware handling for unsupported Gemini audio inputs: do not send raw OGG/Opus to Gemini, do not abort the whole index when one multimodal file is rejected for unsupported media format, and update docs/tests to describe supported raw formats and the conversion workaround. A separate follow-up can add trusted ffmpeg transcoding if maintainers want true OGG/Opus embed-success behavior.
