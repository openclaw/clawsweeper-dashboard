## Summary

Keep open: current main still selects `.ogg` and `.opus` as multimodal memory audio, preserves their detected audio MIME types, and forwards inline data unchanged to the Gemini embedding request path, so the reported provider rejection remains unresolved.

## What This Changes

Make multimodal memory indexing handle OGG/Opus audio with Gemini instead of failing, or document and filter unsupported audio formats.

## Best Possible Solution

Define the Gemini multimodal audio contract, then implement provider-aware handling that either converts unsupported audio before embedding or skips it with clear diagnostics and aligned docs.

## Reproduction Assessment

Yes, from source inspection and the confirming issue comment. Current main selects `.ogg`/`.opus`, resolves them to audio MIME types, and sends those MIME types unchanged to Gemini inline embedding data.

## Solution Assessment

Unclear. The code path is narrow, but the maintainable fix depends on whether OpenClaw should add transcoding for advertised OGG/Opus support or narrow the Gemini contract with provider-specific filtering and docs.
