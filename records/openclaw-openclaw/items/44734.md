## Summary

Keep open. Current main and the latest release still advertise `.ogg` and `.opus` as multimodal memory audio, preserve the detected raw MIME/data, forward it unchanged to Gemini embedding requests, and only skip size-limit provider failures while current Gemini docs list audio embedding support as MP3/WAV.

## What This Changes

Handle OGG/Opus audio in Gemini multimodal memory indexing without 400s, or correct the public supported-format contract.

## Best Possible Solution

Define a provider-aware multimodal audio contract: either transcode unsupported OGG/Opus into a Gemini-supported representation with proof, or stop advertising/indexing those formats as raw Gemini audio and emit a clear skip or warning.

## Reproduction Assessment

Yes, at source level. Enabling multimodal audio indexing for `.ogg` or `.opus` classifies the file, preserves its detected audio MIME, sends it to Gemini as raw `inlineData`, and rethrows non-size provider errors; I did not run a live Gemini request.

## Solution Assessment

Unclear. Auto-transcoding is the most complete user-visible fix but adds dependency and runtime policy, while provider-aware filtering plus docs is narrower but changes the advertised support contract.
