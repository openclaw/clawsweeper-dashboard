## Summary

Keep open: current main and v2026.5.20 still classify `.ogg` and `.opus` as supported Gemini multimodal memory audio, preserve the detected MIME and original bytes, and forward them unchanged to Gemini inline data. The remaining fix is real, but maintainers need to choose whether OpenClaw should transcode unsupported audio or narrow the provider-specific support contract with skip/warn behavior and docs.

## What This Changes

The issue reports that Gemini multimodal memory indexing accepts OGG/Opus audio files but sends them raw to Gemini, which rejects those files.

## Best Possible Solution

Define a provider-specific Gemini multimodal audio contract, then add tests and docs for either converting unsupported audio to MP3/WAV or rejecting/skipping it before provider upload with a clear warning.
