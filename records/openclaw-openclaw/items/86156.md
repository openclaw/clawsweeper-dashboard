Full review comments:

- **[P1] Export the channel broker runtime setter:** `extensions/channel-broker/index.ts:17-18`
  - body: The bundled entry tells OpenClaw to load `setChannelBrokerRuntime` from `./runtime-api.js`, but that file only re-exports SDK helpers and never exports the setter. When the channel entry is runtime-bound, this new plugin will not expose the configured runtime hook its metadata declares.
  - confidence: 0.93
- **[P2] Align the capability schema with the SDK contract:** `extensions/channel-broker/src/config-schema.ts:39`
  - body: `ChannelBrokerProviderConfig` is typed against `BrokerPlatformCapabilities`, whose shape is `delivery`/`live`/`receive`/`native`, but the schema accepts only flat fields like `inbound`, `outbound`, and `threads`. Valid SDK-shaped provider capability config would be rejected, while accepted flat config will not drive the broker capability helpers correctly.
  - confidence: 0.88
