## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Export the configured channel runtime setter:** `extensions/channel-broker/index.ts:17-18`
  - body: `index.ts` declares `runtime.exportName: "setChannelBrokerRuntime"`, but `runtime-api.ts` does not export that symbol. The bundled entry loader throws on a missing named export when it wires `setChannelRuntime`, so the new channel can fail registration before it is usable.
  - confidence: 0.94
- **[P2] Infer broker-prefixed Discord DMs as direct:** `extensions/channel-broker/src/channel.ts:55-65`
  - body: `broker:discord:user:123...` parses as platform `broker` with conversation id `discord:user:123...`, so this helper sees `discord` as the first segment and returns `channel`. Generic target resolution trusts `inferTargetChatType`, which can give the advertised broker DM route channel semantics outside the later route resolver.
  - confidence: 0.9
- **[P2] Align capability config with the SDK contract:** `extensions/channel-broker/src/config-schema.ts:5-39`
  - body: The schema accepts flat fields such as `draftPreview`, but the SDK capability checks read nested `delivery`, `live`, `receive`, and `native` groups and account normalization passes the object through. That rejects SDK-shaped config and makes accepted flat config ineffective for `brokerPlatformSupports`.
  - confidence: 0.88
