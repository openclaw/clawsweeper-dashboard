## Best Possible Solution

Land the QQBot-owned sanitizer hook with regression coverage, and keep any broader default-sanitizer policy for plugins as separate follow-up discussion.

## Reproduction Assessment

## Solution Assessment

Yes, this is the best narrow fix for the linked QQBot regression: the QQBot plugin owns its outbound descriptor and can use the existing SDK-exported delivery sanitizer without adding config, SDK, or core fallback behavior. A core default sanitizer would be broader product/API policy work and is not needed for this repair.
