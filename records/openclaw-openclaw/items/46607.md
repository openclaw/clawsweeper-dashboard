## Summary

Keep open: current main still lacks the requested Google/Gemini audio `@file` to `inlineData` conversion, and this PR remains the canonical implementation thread, but the patch is not merge-ready because it still has workspace-boundary and Gemini MIME contract defects plus missing real behavior proof.

## What This Changes

The PR adds Google/Gemini payload rewriting and regression tests to convert local audio `@file` refs in user text parts into Gemini `inlineData` parts before dispatch.

## Best Possible Solution

Rebase into the current shared Google wrapper, pass the session `workspaceDir` into audio ref resolution, emit only Gemini-supported audio MIME values or preserve unsupported refs, keep focused regression coverage, and require redacted real Gemini headless proof before merge.

## Reproduction Assessment

Yes for source-level reproduction: current main's Google wrapper only sanitizes thinking payloads and has no audio `@file` to `inlineData` conversion. No live Gemini run was performed in this read-only review.

## Solution Assessment

No, not as-is. The implementation direction is reasonable, but it needs current-main rebase, workspace-bound path resolution, Gemini MIME contract fixes, and real behavior proof before it is a maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Resolve inline refs from the session workspace:** `src/agents/pi-embedded-runner/extra-params.ts:251`
  - body: `resolveGoogleInlineAudioPath` binds local file reads to `process.cwd()`, but embedded sessions already carry a `workspaceDir` that can differ from the OpenClaw process cwd. When they differ, prompt-selected audio refs can inline bytes from the wrong local boundary; pass the runtime workspace into the wrapper and validate resolved realpaths there.
  - confidence: 0.87
- **[P2] Emit only Gemini-supported audio MIME types:** `src/agents/pi-embedded-runner/extra-params.ts:327`
  - body: The sniffer can return `audio/opus` and `audio/mp4`, and this line sends that value directly as Gemini `inlineData`. Google's current audio docs list supported Gemini audio MIME types as WAV, MP3, AIFF, AAC, OGG Vorbis, and FLAC, so unsupported values should stay as text or be converted through a supported representation before dispatch. ([ai.google.dev](https://ai.google.dev/gemini-api/docs/audio))
  - confidence: 0.84
