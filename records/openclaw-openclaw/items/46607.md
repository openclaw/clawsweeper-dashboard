## Summary

Keep open. The feature direction is valid and current main still does not inline Google/Gemini audio @file refs, but the PR is not mergeable as-is because it patches the fallback wrapper instead of the active Google provider wrapper and uses process cwd as the trust boundary for prompt-controlled file reads.

## What This Changes

The PR adds a Google/Gemini payload rewrite that converts resolvable local audio @file references into inlineData parts, plus regression tests for resolvable, missing, absolute, and oversized refs.

## Best Possible Solution

Implement the conversion in the active Google provider wrapper or a shared Google transport seam, resolve refs against the active workspace or sandbox media root with realpath boundary checks, reuse shared media MIME detection where practical, and cover valid audio, cwd-versus-workspace, sandbox escape, oversized, unresolved, and common M4A cases.

## Reproduction Assessment

Yes. On current main, a Google/Gemini string user prompt containing @sample.ogg is converted by convertGoogleMessages into a text part, while the PR diff demonstrates the intended inlineData rewrite path.

## Solution Assessment

No. The requested behavior is valid, but this PR is not the best way yet because it patches the fallback wrapper, uses process.cwd() as the file-read boundary, and duplicates a narrow audio sniff instead of fitting the active provider and media seams.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Use the active workspace for audio refs:** `src/agents/pi-embedded-runner/extra-params.ts:251-256`
  - body: resolveGoogleInlineAudioPath uses process.cwd() as the trust root. Embedded/headless runs already thread workspaceDir into applyExtraParamsToAgent, so when the gateway cwd differs from the session workspace this can miss valid refs or inline files from the wrong root; pass the active workspace or sandbox root and realpath-check against it.
  - confidence: 0.88
- **[P2] Move the rewrite into the active Google wrapper:** `src/agents/pi-embedded-runner/extra-params.ts:368`
  - body: This call is inside the fallback Google payload wrapper, but Gemini CLI uses the plugin provider wrapper and current main skips the fallback wrapper once a provider wrapper handled the stream. Put the payload patch in the active Google provider wrapper or shared Google transport seam so it actually runs for google-gemini-cli.
  - confidence: 0.84
- **[P2] Accept common MP4 brands for M4A audio:** `src/agents/pi-embedded-runner/extra-params.ts:204`
  - body: The MP4 sniff only accepts audio-specific major brands, so valid .m4a files with common ftyp brands such as isom or mp42 are left as plain text refs. Reuse shared MIME detection or inspect compatible brands before rejecting these files.
  - confidence: 0.78
