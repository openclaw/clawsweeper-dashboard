## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Resolve audio refs from the runtime workspace:** `src/agents/pi-embedded-runner/extra-params.ts:251-253`
  - body: resolveGoogleInlineAudioPath anchors prompt-selected file reads to process.cwd(), while current main passes an effective workspace into applyExtraParamsToAgent. When cwd differs from the session workspace or sandbox, an @audio ref can read or skip the wrong local file before sending bytes to Google; thread the runtime workspace/sandbox root into this resolver instead.
  - confidence: 0.88
- **[P2] Budget for base64 inline request size:** `src/agents/pi-embedded-runner/extra-params.ts:315`
  - body: The guard allows any raw file up to 20 MiB, then base64-encodes it into inlineData. Google's inline audio limit is the total request size, including inline files, so a near-limit raw file can become an oversized JSON request before text/system content is counted; leave headroom, check encoded payload size, or keep the ref as text/use Files API. ([ai.google.dev](https://ai.google.dev/gemini-api/docs/audio))
  - confidence: 0.82
