## Summary

Keep open. Current main still lacks inline Google/Gemini audio @file handling, but this PR should not merge as-is because the implementation reads from process.cwd(), lands in the wrong/stale wrapper path for current provider hooks, and rejects common valid M4A files.

## What This Changes

This PR adds a Google/Gemini payload rewrite that converts resolvable local audio @file refs in text parts into inlineData parts, with regression tests for resolvable, missing, absolute, and oversized refs.

## Best Possible Solution

Implement the conversion in the active Google provider wrapper or shared Google transport seam, resolve refs against the active workspace or sandbox media root with realpath boundary checks, and reuse shared media MIME detection with targeted tests.

## Reproduction Assessment

Yes. Source inspection shows current main turns a Google/Gemini string prompt containing an audio @file ref into a text part, and there is no current inline-audio rewrite path.

## Solution Assessment

No. The requested behavior is useful, but this PR is not the best implementation because it patches a stale/fallback seam, uses process cwd as the trust root, and duplicates narrow audio sniffing instead of the active provider/media seams.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Resolve audio refs from the active workspace:** `src/agents/pi-embedded-runner/extra-params.ts:251-256`
  - body: resolveGoogleInlineAudioPath uses process.cwd() as the trust root. Embedded/headless runs already pass workspaceDir through applyExtraParamsToAgent and provider wrapper context, so this can miss valid refs or inline files from the wrong root when the gateway cwd differs from the session workspace; use the active workspace or sandbox root with a realpath boundary check.
  - confidence: 0.88
- **[P2] Install the rewrite in the current Google stream hook:** `src/agents/pi-embedded-runner/extra-params.ts:368`
  - body: This hooks the rewrite into the old extra-params Google payload wrapper, but current main routes google-gemini-cli through provider-owned stream hooks and skips fallback wrappers once a provider wrapper handles the stream. Move the payload rewrite to the active Google provider wrapper or shared Google transport seam so it runs after rebase.
  - confidence: 0.84
- **[P2] Recognize common M4A MP4 brands:** `src/agents/pi-embedded-runner/extra-params.ts:243-244`
  - body: The MP4 sniff only accepts audio-specific major brands, so valid .m4a files with common ftyp brands such as isom or mp42 remain plain text refs. Reuse shared MIME detection or inspect compatible brands before rejecting these files.
  - confidence: 0.78
