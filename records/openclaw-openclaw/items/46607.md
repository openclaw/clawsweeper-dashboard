## Summary

Keep open. Current main still lacks the requested Google/Gemini headless audio `@file` to `inlineData` conversion, but this PR should not merge as-is because it still has a concrete `.m4a` sniffing gap and lacks required real-behavior proof from an external contributor.

## What This Changes

The PR adds Google/Gemini headless payload rewriting that turns supported inline local audio `@file` references into Gemini `inlineData` parts, plus regression tests for resolved, unresolved, unsafe, oversized, and MIME-checked refs.

## Best Possible Solution

Rebase the feature onto the current shared Google wrapper, fix `.m4a` MP4-brand handling, preserve the path/size/MIME guards, and require redacted real-behavior proof before merge.

## Reproduction Assessment

No live high-confidence reproduction was run in this read-only review. Source inspection does show current main lacks the requested payload rewrite, and the PR tests describe the missing conversion path from text refs to Google `inlineData`.

## Solution Assessment

No, not as-is. The direction is maintainable, but the patch needs the `.m4a` sniffing fix, rebase onto the current wrapper location, and real-behavior proof before it is the best landing shape.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Accept compatible MP4 brands for .m4a audio:** `src/agents/pi-embedded-runner/extra-params.ts:242-244`
  - body: The `.m4a` path only returns `audio/mp4` when the file's major brand is in the small `GOOGLE_INLINE_AUDIO_MP4_BRANDS` set. An audio-only `.m4a` with a compatible generic MP4 brand will be left as a text ref even though `.m4a` is advertised as supported, so the feature can silently fail for valid inputs. Please accept compatible MP4 brands for `.m4a` or use shared MIME detection with the filename fallback.
  - confidence: 0.86

## Security Review

Status: cleared

That can send bytes from the wrong local directory when cwd and workspace differ, so the rebase should use the current workspaceDir/path-policy boundary before dispatching audio data.
