## Summary

Keep open. Current main still leaves Google/Gemini string @audio refs as text, and this PR is not merge-ready because the rewrite is wired into a fallback wrapper skipped by the active Gemini CLI provider path and uses the process cwd as the file-read boundary.

## What This Changes

The PR adds a Google/Gemini payload rewrite that converts resolvable local audio @file refs into inlineData parts, plus regression tests for resolvable, missing, absolute, and oversized refs.

## Best Possible Solution

Implement the conversion in the active Google provider wrapper or shared Google transport seam, resolve refs against the active workspace or sandbox media root with realpath boundary checks, and reuse shared media MIME detection with targeted tests.

## Reproduction Assessment

Yes. Source inspection shows current main turns a Google/Gemini string prompt containing @sample.ogg into a text part, while the PR diff demonstrates the intended inlineData rewrite path.

## Solution Assessment

No. The requested behavior is valid, but this PR is not the best implementation yet because it patches the fallback wrapper, uses process.cwd() for local reads, and duplicates a narrow audio sniff instead of using the active provider/media seams.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Use the active workspace for audio refs:** `src/agents/pi-embedded-runner/extra-params.ts:251-256`
  - body: resolveGoogleInlineAudioPath uses process.cwd() as the trust root. Embedded/headless runs already thread workspaceDir into applyExtraParamsToAgent, so when the gateway cwd differs from the session workspace this can miss valid refs or inline files from the wrong root; pass the active workspace or sandbox root and realpath-check against it.
  - confidence: 0.88
- **[P2] Move the rewrite into the active Google wrapper:** `src/agents/pi-embedded-runner/extra-params.ts:368`
  - body: This call is inside the fallback Google payload wrapper, but Gemini CLI uses the plugin provider wrapper and current main skips the fallback wrapper once a provider wrapper handled the stream. Put the payload patch in the active Google provider wrapper or shared Google transport seam so it actually runs for google-gemini-cli.
  - confidence: 0.84
- **[P2] Accept common MP4 brands for M4A audio:** `src/agents/pi-embedded-runner/extra-params.ts:244`
  - body: The MP4 sniff only accepts audio-specific major brands, so valid .m4a files with common ftyp brands such as isom or mp42 are left as plain text refs. Reuse shared MIME detection or inspect compatible brands before rejecting these files.
  - confidence: 0.78
