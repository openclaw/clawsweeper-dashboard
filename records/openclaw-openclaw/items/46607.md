## What This Changes

The PR rewrites Google/Gemini headless payload text parts so resolvable local audio @file references become Gemini inlineData parts, with regression tests for resolved, missing, absolute, and oversized refs.

## Best Possible Solution

Rebase onto current main, move the inliner to the current Google wrapper/runtime surface, bind file reads to the effective workspace or sandbox, enforce encoded total request limits and supported MIME behavior, preserve unresolved refs as text, and add redacted real Gemini proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Resolve audio refs from the runtime workspace:** `src/agents/pi-embedded-runner/extra-params.ts:251-253`
  - body: resolveGoogleInlineAudioPath anchors prompt-selected file reads to process.cwd(), while current main passes an effective workspace into applyExtraParamsToAgent. When cwd differs from the session workspace or sandbox, an @audio ref can read or skip the wrong local file before sending bytes to Google; thread the runtime workspace/sandbox root into this resolver instead.
  - confidence: 0.88
- **[P2] Budget encoded audio against the total request limit:** `src/agents/pi-embedded-runner/extra-params.ts:315-327`
  - body: The guard allows any raw file up to 20 MiB, then base64-encodes it into inlineData alongside text and system content. Gemini's inline audio limit is the total request size, so near-limit raw files can still build oversized requests; leave headroom, check encoded payload size, or keep the ref as text/use the Files API.
  - confidence: 0.84
- **[P2] Accept common m4a MP4 brands before rejecting audio:** `src/agents/pi-embedded-runner/extra-params.ts:242-244`
  - body: The MP4 sniff only accepts a narrow m4a/f4a brand set, so valid .m4a files with common ISO brands such as isom or mp42 are left as plain text even though the PR promises .m4a support. Broaden the compatible-brand handling or rely on a shared MIME detector before rejecting.
  - confidence: 0.74
