## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Resolve audio refs from the runtime workspace:** `src/agents/pi-embedded-runner/extra-params.ts:251-256`
  - body: `resolveGoogleInlineAudioPath` anchors trust at `process.cwd()`, while current embedded turns already pass `effectiveWorkspace` into `applyExtraParamsToAgent`. In sandboxed or headless sessions cwd can differ from the session root, so this can read or skip the wrong local file before sending bytes to Google; thread the runtime workspace or sandbox root into this resolver.
  - confidence: 0.88
- **[P2] Cap the provider request, not raw file bytes:** `src/agents/pi-embedded-runner/extra-params.ts:315-327`
  - body: The guard allows any raw file up to 20 MiB and then base64-encodes it into `inlineData`. Google's audio contract frames this as total request size including files, prompt text, and system instructions, so a near-limit raw file can still produce an oversized request; leave headroom, check encoded total size, or use the Files API. ([ai.google.dev](https://ai.google.dev/gemini-api/docs/audio?utm_source=openai))
  - confidence: 0.84
- **[P2] Reuse shared audio MIME detection for M4A:** `src/agents/pi-embedded-runner/extra-params.ts:242-244`
  - body: The `.m4a` branch accepts only a small local MP4 major-brand allowlist, so valid audio that the shared media detector would classify as `audio/mp4` can remain a plain text ref. Reuse the existing media MIME detector instead of maintaining a Google-only sniffer.
  - confidence: 0.76
