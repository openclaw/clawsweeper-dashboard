## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Resolve audio refs from the runtime workspace:** `src/agents/pi-embedded-runner/extra-params.ts:251-254`
  - body: `resolveGoogleInlineAudioPath` fixes its trust boundary at `process.cwd()`, while current main passes an effective workspace into this provider surface. In headless or sandboxed sessions cwd can differ from the session root, so an `@audio` ref can read or skip the wrong local file before sending bytes to Google; thread the runtime workspace or sandbox root into the resolver.
  - confidence: 0.88
- **[P2] Budget inline audio using the provider request contract:** `src/agents/pi-embedded-runner/extra-params.ts:315`
  - body: The guard allows raw files up to 20 MiB, then base64-encodes them into `inlineData` without accounting for the full request or using the Files API fallback. Google documents inline audio limits as a total request budget including text, system instructions, and inline files, so this can still produce oversized requests or enforce a stale local limit.
  - confidence: 0.82
- **[P2] Accept valid M4A MP4 brands:** `src/agents/pi-embedded-runner/extra-params.ts:242-244`
  - body: The `.m4a` sniff only accepts a small major-brand allowlist, so valid AAC-in-MP4 audio with common brands like `isom` or `mp42` remains a plain text ref even though the PR promises `.m4a` support. Use the shared MIME detector or compatible-brand handling before rejecting the file.
  - confidence: 0.76
