## Summary

Keep open: current main still lacks this narrow inline-audio path, but this PR is not ready because it reads prompt-selected files from the process cwd boundary, can emit MIME types outside Gemini's audio contract, and has no after-fix real Gemini proof.

## What This Changes

The PR adds Google/Gemini payload rewriting and tests to convert resolvable local audio `@file` refs into `inlineData` parts before dispatch.

## Best Possible Solution

Land a refreshed implementation in the current shared Google wrapper that resolves refs against the session workspace, emits only Gemini-supported audio parts or preserves text, covers regressions, and includes redacted real Gemini headless proof.

## Reproduction Assessment

Yes, at source level: current main's Google wrapper does not inline audio refs, and the current Gemini CLI media path still uses `read_many_files`. I did not run a live Gemini audio session in this read-only review.

## Solution Assessment

No, not as-is. The direction is useful, but the PR should be reworked into the current shared wrapper with `workspaceDir`-bound path resolution, a Gemini-supported MIME contract, and real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Resolve inline refs from the session workspace:** `src/agents/pi-embedded-runner/extra-params.ts:251-256`
  - body: `resolveGoogleInlineAudioPath` binds local file reads to `process.cwd()`, but embedded sessions already carry `workspaceDir` and that can differ from the OpenClaw process cwd. Prompt-selected audio refs should resolve and realpath-check against the session workspace before bytes are sent to the provider.
  - confidence: 0.88
- **[P2] Emit only Gemini-supported audio MIME types:** `src/agents/pi-embedded-runner/extra-params.ts:327`
  - body: The sniffer can return `audio/opus` and `audio/mp4`, and this line sends that value directly as Gemini `inlineData`. Unsupported values should stay as text or be converted/mapped through a proven Gemini-supported representation before dispatch. ([ai.google.dev](https://ai.google.dev/gemini-api/docs/audio?utm_source=openai))
  - confidence: 0.84
