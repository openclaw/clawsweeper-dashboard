## Summary

Keep open: this targets a real Google/Gemini headless audio gap that current main still does not implement, but the PR is not mergeable as-is because real behavior proof is missing and the diff still has workspace-boundary and MIME-contract blockers.

## What This Changes

The PR adds Google/Gemini payload rewriting and tests to convert local audio `@file` refs in user text parts into `inlineData` parts during headless dispatch.

## Best Possible Solution

Rebase into the current shared Google wrapper, pass the session `workspaceDir` into audio ref resolution, emit only Gemini-supported audio MIME values or preserve unsupported refs, keep focused regression coverage, and require redacted real Gemini headless proof before merge.

## Reproduction Assessment

Yes, source-level: current main's Google wrapper only sanitizes thinking payloads and the documented Gemini CLI audio fallback still uses `read_many_files`. I did not run a live Gemini session in this read-only review.

## Solution Assessment

No, not as-is. The implementation direction is reasonable, but the PR must use the session workspace boundary, honor the Gemini MIME contract, rebase cleanly, and provide real behavior proof.

## Review Findings

  - confidence: 0.84
