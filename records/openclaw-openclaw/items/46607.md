Full review comments:

- **[P2] Resolve audio refs from the runtime workspace:** `src/agents/pi-embedded-runner/extra-params.ts:250-256`
  - body: `resolveGoogleInlineAudioPath` anchors trust at `process.cwd()`, while current runs pass `effectiveWorkspace` into `applyExtraParamsToAgent`. Sandboxed or headless sessions can read or skip files relative to the wrong root before bytes are sent to Google, so thread the runtime workspace or sandbox root into this resolver.
  - confidence: 0.88
- **[P2] Cap the total Gemini request, not raw file bytes:** `src/agents/pi-embedded-runner/extra-params.ts:314-327`
  - body: The guard admits any raw file up to 20 MiB and then base64-encodes it into `inlineData`. Gemini's audio docs define the inline limit as total request size including prompts, system instructions, and inline files, so near-limit files can still produce rejected requests; leave headroom, check the encoded payload, or use the Files API. ([ai.google.dev](https://ai.google.dev/gemini-api/docs/audio?utm_source=openai))
  - confidence: 0.84
- **[P2] Reuse shared MIME detection for M4A:** `src/agents/pi-embedded-runner/extra-params.ts:242-244`
  - body: The local MP4 sniffer only accepts a small major-brand allowlist, so valid `.m4a` files that the shared detector accepts as `audio/mp4` can remain plain text refs. Use or extend the shared media helpers instead of a Google-only sniffer.
  - confidence: 0.76
