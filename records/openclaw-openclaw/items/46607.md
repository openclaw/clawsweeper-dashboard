## Summary

Keep open: current main still lacks the requested inline Google/Gemini audio attachment behavior, but this PR is not merge-ready because it patches a stale fallback wrapper and adds prompt-controlled local file reads rooted at `process.cwd()` instead of the active workspace/sandbox boundary. The prior review comments correctly identify the useful direction and the security-sensitive parts that need maintainer approval.

## What This Changes

The PR rewrites Google/Gemini headless text payload parts so resolvable local audio `@file` references become Gemini `inlineData` parts, with regression coverage for resolvable, missing, absolute, and oversized refs.

## Best Possible Solution

Implement the conversion in the active Google provider wrapper or shared Google transport seam, resolve refs against the active workspace/sandbox media root with realpath boundary checks, and reuse shared MIME/size helpers with focused tests.

## Reproduction Assessment

Yes. Source inspection on current main shows string Google user content becomes a text part, and current Google providers use provider-owned stream hooks that skip the fallback extra-params wrapper once handled.

## Solution Assessment

No. The requested capability is useful, but this patch is not the best implementation because it patches a stale fallback seam, uses `process.cwd()` as the trust root, and duplicates narrow audio sniffing instead of using the active provider/media seams.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Resolve audio refs from the active workspace:** `src/agents/pi-embedded-runner/extra-params.ts:251-256`
  - body: `resolveGoogleInlineAudioPath` uses `process.cwd()` as the trust root. Embedded/headless runs already pass `workspaceDir` through the wrapper context, so this can miss valid refs or inline files from the wrong root when the gateway cwd differs from the session workspace; use the active workspace or sandbox media root with a realpath boundary check.
  - confidence: 0.88
- **[P2] Install the rewrite in the active Google stream hook:** `src/agents/pi-embedded-runner/extra-params.ts:368`
  - body: This adds the rewrite to the fallback Google payload wrapper, but current main routes Google/Gemini providers through provider-owned stream hooks and skips that fallback once a provider wrapper handles the stream. Move the payload rewrite to the active Google provider wrapper or shared Google transport seam so it actually runs after rebase.
  - confidence: 0.84
- **[P2] Accept common MP4 brands for M4A audio:** `src/agents/pi-embedded-runner/extra-params.ts:204`
  - body: The MP4 sniff only accepts a small set of audio-specific major brands, so valid `.m4a` files whose `ftyp` major brand is `isom` or `mp42` can remain plain text refs. Reuse shared MIME detection or inspect compatible brands before rejecting these files.
  - confidence: 0.78
