Overall confidence: 0.86

Full review comments:

- **[P2] Resolve audio refs from the runtime workspace:** `src/agents/pi-embedded-runner/extra-params.ts:250-257`
  - body: resolveGoogleInlineAudioPath fixes its trust boundary at process.cwd(), while the current runner already passes effectiveWorkspace into applyExtraParamsToAgent. In headless or sandboxed sessions cwd can differ from the session root, so an @audio ref can read or skip the wrong local file before sending bytes to Google; thread the runtime workspace or sandbox root into this resolver.
  - confidence: 0.88
- **[P2] Cap the provider request, not raw file bytes:** `src/agents/pi-embedded-runner/extra-params.ts:315-327`
  - body: The guard allows any raw file up to 20 MiB and then base64-encodes it into inlineData. Google's audio docs define the inline limit as total request size, including prompts, system instructions, and files, so a near-limit raw file can still produce an oversized request; leave headroom, check encoded total size, or use the Files API. ([ai.google.dev](https://ai.google.dev/gemini-api/docs/audio))
  - confidence: 0.84
- **[P2] Reuse shared audio MIME detection for M4A:** `src/agents/pi-embedded-runner/extra-params.ts:242-244`
  - body: The .m4a branch accepts only a small local MP4 major-brand allowlist, so valid MP4-family audio that the shared media detector would classify can remain a plain text ref. Reuse the existing media MIME detector or expand through that shared contract instead of maintaining a Google-only sniffer.
  - confidence: 0.76
