## Summary

## What This Changes

The PR changes Codex runtime extra-param resolution to look up canonical `openai/*` model params before `openai-codex/*` and adds resolver/status tests for `textVerbosity`.

## Best Possible Solution

Resolve Codex-runtime params from the documented `openai/*` model ref as a fallback when provider-specific `openai-codex/*` params are absent, with tests for both precedence cases and redacted real-route proof.

## Reproduction Assessment

Yes. Source inspection shows current main resolves params by active provider/model, so an `openai-codex` runtime session misses canonical `openai/gpt-5.5` params and then applies the GPT-5 `low` verbosity default; I did not run live Codex OAuth.

## Solution Assessment

No, not as written. Falling back to canonical `openai/*` aligns with the docs, but making it win over explicit `openai-codex/*` params is not the safest compatibility-preserving fix.
