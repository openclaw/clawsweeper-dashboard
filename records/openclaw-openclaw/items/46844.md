## Summary

Keep open. Current main still lacks a between-turn Talk Mode idle timeout: Voice Wake can activate Talk directly, but Talk only finalizes non-empty transcripts on the short silence window and then resumes listening after assistant replies or failures.

## What This Changes

Add a configurable Talk Mode idle timeout that auto-deactivates wake-triggered Talk after a period with no user interaction.

## Best Possible Solution

Add an additive Talk lifecycle setting that is validated in config, exposed through talk.config, documented on the public Talk docs, and honored by macOS Talk so idle sessions return to wake listening while keeping silenceTimeoutMs scoped to transcript finalization pauses.

## Reproduction Assessment

Yes. Enable Voice Wake with Trigger Talk Mode, wake into Talk, complete a turn, and then remain silent; source review shows empty between-turn silence is ignored and the runtime resumes recognition after replies or errors.

## Solution Assessment

Unclear as proposed. The direction is narrow and maintainable, but a maintainer should define the public Talk lifecycle contract before implementation chooses naming, default duration, bounds, disabled behavior, and reset triggers.
