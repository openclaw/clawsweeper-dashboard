## Summary

Keep open. Current main still lacks a between-turn Talk Mode idle timeout after Voice Wake activates Talk, and implementing it would add a new public Talk lifecycle/config/protocol contract that needs maintainer product/API direction.

## What This Changes

The issue requests a configurable Talk Mode idle timeout that auto-deactivates a Voice Wake-triggered Talk session after a period of no user interaction.

## Best Possible Solution

Add an additive, maintainer-approved Talk lifecycle setting across config/schema/protocol/docs and honor it in macOS Talk so idle sessions return to wake listening while keeping silenceTimeoutMs scoped to transcript finalization.

## Reproduction Assessment

Yes, source-reproducible for the missing capability. Enable Voice Wake with Trigger Talk Mode, complete one Talk exchange, then remain silent; current main has no accepted idle-timeout field and macOS Talk ignores empty between-turn silence while resuming listening after replies or errors.

## Solution Assessment

Unclear. The requested direction is plausible and bounded, but maintainers should decide the public Talk lifecycle contract before implementation commits to scope, naming, defaults, bounds, and reset semantics.
