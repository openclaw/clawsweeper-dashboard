## Summary

Keep open. Current main and the latest release still lack a between-turn Talk Mode idle timeout after Voice Wake activates Talk, and the requested behavior is a new public Talk lifecycle/config contract that needs maintainer product/API direction before implementation.

## What This Changes

Request a configurable Talk Mode idle timeout that auto-deactivates Voice Wake-triggered Talk sessions after between-turn inactivity.

## Best Possible Solution

Add an additive Talk lifecycle setting, validate and expose it through Talk config/protocol, document it, and honor it in macOS Talk so idle sessions return to wake listening while keeping silenceTimeoutMs scoped to transcript finalization.

## Reproduction Assessment

Yes, source-reproducible. Enable Voice Wake with Trigger Talk Mode, complete one Talk exchange, then remain silent; current source ignores empty between-turn silence and resumes listening after replies or failures.

## Solution Assessment

Unclear. The direction is plausible and bounded, but maintainers should decide the public Talk lifecycle contract before implementation commits to scope, field names, defaults, bounds, and reset semantics.
