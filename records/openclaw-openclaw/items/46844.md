## Summary

Keep open. Current main still lacks a between-turn Talk Mode idle timeout after Voice Wake activates Talk Mode; the request is valid and source-reproducible, but it adds a new public Talk lifecycle/config contract that needs maintainer product/API direction before implementation.

## What This Changes

Add a configurable Talk Mode idle timeout that auto-deactivates Voice Wake-triggered Talk sessions after a period without user interaction.

## Best Possible Solution

Add an additive Talk lifecycle setting, validate and expose it through Talk config/protocol, document it on the public Talk docs, and honor it in macOS Talk so idle sessions return to wake listening while keeping silenceTimeoutMs scoped to transcript finalization.

## Reproduction Assessment

Yes, source-reproducible. Enable Voice Wake with Trigger Talk Mode, complete one Talk exchange, then stay silent; current source ignores empty between-turn silence and resumes recognition after replies or failures.

## Solution Assessment

Unclear. The requested idle timer is plausible and bounded, but maintainers should choose the public Talk lifecycle contract before implementation commits to field names, defaults, bounds, scope, and reset behavior.
