## Summary

Keep open. Current main and v2026.6.10 still have Voice Wake-to-Talk activation and intra-turn Talk silence handling, but no between-turn idle deactivation or public Talk idle-timeout config contract.

## What This Changes

The issue requests a configurable idle timeout that deactivates Talk Mode after Voice Wake starts it and no user interaction occurs for a configured period.

## Best Possible Solution

Keep this as the canonical Talk idle-timeout tracker, choose the public lifecycle/config contract, then implement one documented idle-deactivation path across config validation, gateway protocol, macOS Talk runtime, docs, and focused tests.

## Reproduction Assessment

Yes, at source level. Current main can enable Talk Mode from Voice Wake, but the Talk silence monitor ignores empty between-turn silence and the runtime resumes listening after responses or errors instead of deactivating.

## Solution Assessment

No implementation path is ready yet. A 30-second default is plausible, but maintainers still need to choose the config name, scope, reset events, disabled value, limits, and whether this applies to all Talk sessions or only Voice Wake-triggered Talk Mode.
