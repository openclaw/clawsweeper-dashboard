## Summary

Keep open. Current main still lacks a between-turn Talk Mode idle timeout after Voice Wake activation; this is a valid feature request, but it adds a new public Talk lifecycle/config contract that needs maintainer product/API direction before implementation.

## What This Changes

The issue requests a configurable between-turn Talk Mode idle timeout that auto-deactivates Talk Mode after Voice Wake-triggered conversations go idle.

## Best Possible Solution

Add an additive Talk lifecycle setting, validate and expose it through Talk config/protocol, document it, and honor it in macOS Talk so idle sessions return to wake listening while keeping silenceTimeoutMs scoped to transcript finalization.

## Reproduction Assessment

Yes. Source review gives a high-confidence path: enable Voice Wake with Trigger Talk Mode, complete one Talk exchange, then remain silent; empty between-turn silence is ignored and Talk resumes listening.

## Solution Assessment

Unclear. The direction is plausible and bounded, but maintainers should choose the public Talk lifecycle contract before implementation commits to field names, defaults, bounds, scope, and reset behavior.
