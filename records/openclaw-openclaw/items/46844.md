## Summary

Keep open. Current main and the latest release still only implement intra-turn Talk silence handling; the requested between-turn idle deactivation after Voice Wake-triggered Talk Mode is still missing and needs a maintainer decision on the public Talk lifecycle/config contract.

## What This Changes

The issue requests a configurable Talk Mode idle timeout after Voice Wake activation so inactive voice sessions automatically return to wake-word waiting before the hard session cap.

## Best Possible Solution

Keep this as the canonical Talk idle-timeout tracker, choose the public lifecycle/config contract, then implement one documented idle-deactivation path across config validation, gateway protocol, macOS Talk runtime, docs, and focused tests.

## Reproduction Assessment

Yes, at source level. Voice Wake can enable Talk Mode, the Talk silence monitor only finalizes non-empty transcripts, and current main plus v2026.6.10 resume listening after turns instead of disabling Talk Mode on idle.

## Solution Assessment

No implementation path is ready yet. A 30-second default is plausible, but maintainers still need to choose the config name, scope, reset events, disabled value, and whether this applies to all Talk sessions or only Voice Wake-triggered Talk Mode.
