## Summary

Keep open. Current main and the latest release still expose only intra-turn Talk silence handling; the requested between-turn idle deactivation after Voice Wake activation is not implemented and needs a maintainer decision on the public Talk lifecycle/config contract.

## What This Changes

The issue requests a configurable between-turn Talk Mode idle timeout that deactivates voice interaction after inactivity following Voice Wake activation.

## Best Possible Solution

Keep this as the canonical Talk idle-timeout tracker, choose the public Talk lifecycle/config contract, then implement one documented idle-deactivation path across config, protocol, runtime, docs, and focused tests.

## Reproduction Assessment

Yes, at source level. Voice Wake enables Talk Mode, Talk silence handling only finalizes non-empty transcripts, and current main plus v2026.6.8 resume listening after each turn instead of disabling Talk Mode on idle.

## Solution Assessment

No implementation path is ready yet. A 30-second timeout is plausible, but maintainers still need to choose the config name, default, reset events, disabled value, and whether the behavior applies to all Talk sessions or only Voice Wake-triggered Talk Mode.
