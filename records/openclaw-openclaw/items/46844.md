## Summary

Keep open. Current main still lacks a between-turn idle timeout for Voice Wake-triggered Talk Mode; the existing Talk silence handling only finalizes non-empty transcripts and then resumes listening, so this remains a valid feature request that needs Talk lifecycle/config product direction.

## What This Changes

Add a configurable between-turn idle timeout that auto-deactivates Voice Wake-triggered Talk Mode after inactivity.

## Best Possible Solution

Define an additive Talk lifecycle setting, validate and expose it through talk.config, document it, and honor it in macOS Talk so idle sessions return to wake listening while preserving silenceTimeoutMs as transcript-finalization timing.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: enable Voice Wake with Trigger Talk Mode, complete one Talk exchange, then remain silent; empty between-turn silence is ignored and Talk resumes listening.

## Solution Assessment

Unclear as proposed. The direction is narrow, but maintainers should choose the public lifecycle contract before implementation commits to field names, defaults, bounds, scope, and reset points.
