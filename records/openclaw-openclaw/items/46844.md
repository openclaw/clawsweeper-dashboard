## Summary

Keep open. Current main still lacks a Talk Mode between-turn idle timeout: Voice Wake can directly activate Talk, but Talk only uses the short silence window to finalize non-empty transcripts and then resumes listening after replies or failures.

## What This Changes

Add a configurable idle timeout that deactivates Voice Wake-triggered Talk Mode after a period of no user interaction.

## Best Possible Solution

Add an additive Talk lifecycle setting that is validated in config, exposed through talk.config, documented on the public Talk docs, and honored by macOS Talk so idle sessions return to wake listening while silenceTimeoutMs remains scoped to transcript finalization.

## Reproduction Assessment

Yes. Enable Voice Wake with Trigger Talk Mode, wake into Talk, complete one exchange, then stay silent; source review shows empty between-turn silence is ignored and the runtime resumes recognition after replies or failures.

## Solution Assessment

Unclear as proposed. The requested direction is narrow and maintainable, but a maintainer should define the public Talk lifecycle contract before implementation chooses naming, default duration, bounds, disabled behavior, and reset triggers.
