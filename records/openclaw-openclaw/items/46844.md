## Summary

Keep open. Current main and the latest v2026.6.6 release still only implement intra-turn Talk silence handling; the requested between-turn auto-deactivation remains missing and needs a maintainer product/API decision before implementation.

## What This Changes

Requests a configurable Talk Mode inactivity timeout that deactivates voice-wake-started Talk sessions after the user stops interacting.

## Best Possible Solution

Define a maintainer-approved Talk lifecycle contract, then implement one additive idle timer at the macOS Talk/Voice Wake boundary with documented defaults, reset events, and config/schema/protocol coverage.

## Reproduction Assessment

Yes, at source level: Voice Wake can enable Talk Mode directly, and TalkModeRuntime then keeps restarting listening after turns while checkSilence only handles non-empty transcripts. This proves the missing between-turn idle shutdown without a live macOS audio run.

## Solution Assessment

Unclear as proposed: an idle timer is the right class of solution, but maintainers still need to choose voice-wake-only versus all Talk sessions, reset events, field name, units, default, limits, and disabled behavior.
