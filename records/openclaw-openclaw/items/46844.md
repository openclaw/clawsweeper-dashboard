## Summary

Keep open. Current main still lacks a between-turn Talk idle timeout: Voice Wake can activate Talk Mode, but native Talk only has a short transcript-finalization silence window and then resumes listening after replies or failures.

## What This Changes

Request a configurable Talk Mode idle timeout that deactivates Voice Wake-triggered Talk after a period without user interaction.

## Best Possible Solution

Define an additive Talk lifecycle setting, validate and expose it through `talk.config`, document it publicly, and have native Talk return idle Voice Wake-triggered sessions to wake listening while keeping `silenceTimeoutMs` scoped to transcript-finalization pauses.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes from source inspection: current macOS Talk has no between-turn idle timer and resumes listening after replies or failures, while strict config/protocol/docs expose only `silenceTimeoutMs` for intra-turn transcript finalization.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose the public lifecycle contract; an additive Talk idle setting is plausible, but an automated fix should not pick scope, defaults, limits, or reset semantics on its own.
