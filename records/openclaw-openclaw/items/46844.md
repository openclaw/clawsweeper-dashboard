## Summary

Keep open. Current main and v2026.6.6 still lack the requested between-turn Talk idle deactivation: Voice Wake can activate Talk Mode directly, but Talk config, schema, docs, and macOS runtime only expose intra-turn silence handling.

## What This Changes

Add a configurable Talk-mode idle timeout that deactivates hands-free Voice Wake-triggered Talk sessions after no user interaction.

## Best Possible Solution

Define the Talk lifecycle contract first, then add one additive validated config/protocol/docs path and runtime handling that returns idle Voice Wake-triggered Talk sessions to wake listening without changing silenceTimeoutMs semantics.

## Reproduction Assessment

Yes, at source level. Current main shows Voice Wake can enable Talk Mode directly, while TalkModeRuntime only finalizes non-empty transcripts after the intra-turn silence window and then resumes listening between turns.

## Solution Assessment

Unclear as proposed. An idle timer is the right class of solution, but maintainers still need to choose scope, field name, units, default, limits, disabled behavior, and reset events before implementation is the best fix.
