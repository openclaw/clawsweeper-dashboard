## Summary

Keep open. Current main and v2026.5.12 still lack a between-turn Talk Mode idle timeout after Voice Wake activates Talk, and the remaining work is a new public Talk lifecycle/config contract that needs maintainer product/API direction.

## What This Changes

Add a configurable Talk Mode idle timeout that deactivates Voice Wake-triggered Talk sessions after inactivity and returns to wake-word listening.

## Best Possible Solution

Add an additive, maintainer-approved Talk lifecycle setting across config/schema/protocol/docs and honor it in macOS Talk so idle sessions return to wake listening while keeping silenceTimeoutMs scoped to transcript finalization.

## Reproduction Assessment

Yes, source-reproducible. Enable Voice Wake with Trigger Talk Mode, complete one Talk exchange, then remain silent; current main has no accepted idle-timeout field and the macOS Talk runtime ignores empty between-turn silence while resuming listening after replies or errors.

## Solution Assessment

Unclear. The requested direction is plausible and bounded, but maintainers should decide the public Talk lifecycle contract before implementation commits to scope, names, defaults, limits, and reset semantics.
