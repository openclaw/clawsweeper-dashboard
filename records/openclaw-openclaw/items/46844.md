## Summary

Keep open. Current main still lacks a between-turn idle timeout for Voice Wake-triggered Talk Mode, and the request is a new Talk lifecycle/config contract rather than an already-defined bug fix.

## What This Changes

Request a configurable idle timeout that auto-deactivates Talk Mode after inactivity when Voice Wake starts a Talk session.

## Best Possible Solution

Define an additive Talk lifecycle setting, validate and expose it through talk.config, document it, and honor it in macOS Talk so idle sessions return to wake listening without changing silenceTimeoutMs transcript-finalization semantics.

## Reproduction Assessment

Yes. Source review gives a high-confidence path: enable Voice Wake with Trigger Talk Mode, complete one Talk exchange, then remain silent; empty between-turn silence is ignored and Talk resumes listening.

## Solution Assessment

Unclear as proposed. The direction is narrow, but maintainers should choose the public lifecycle contract before implementation commits to idleTimeoutS/reset flags, defaults, bounds, or reset points.
