## Summary

Keep open. Current main still lacks a Talk between-turn idle timeout, and the related heartbeat/session cleanup items do not cover this Voice Wake-triggered Talk lifecycle gap.

## What This Changes

Request a configurable Talk Mode idle timeout that auto-deactivates Voice Wake-triggered Talk after no user interaction.

## Best Possible Solution

Define an additive Talk idle-timeout contract, validate and expose it through talk.config, document it, and have macOS Talk return to Voice Wake after idle without changing silenceTimeoutMs transcript-finalization semantics.

## Reproduction Assessment

Yes. Source review gives a high-confidence path: enable Voice Wake with Trigger Talk Mode, complete one Talk exchange, then remain silent; empty between-turn silence is ignored and the runtime resumes listening.

## Solution Assessment

Unclear as proposed. The direction is narrow and maintainable, but maintainers should choose the public lifecycle contract before an automated fix commits to idleTimeoutS/reset flags, defaults, or reset semantics.
