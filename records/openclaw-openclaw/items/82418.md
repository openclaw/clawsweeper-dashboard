## What This Changes

The PR changes Slack direct-message thread replies to route to the main DM session, adds verbose logging for unresolved assistant DM message edits, and adds one routing unit test.

## Best Possible Solution

Make an explicit Slack DM thread-routing decision, update routing/prepared-message tests and Slack docs to match it, land with redacted live Slack proof, and keep the broader Assistant App dispatch work under its canonical follow-up.

## Reproduction Assessment

## Solution Assessment

No, not in the current PR form: flattening DM thread replies may be the right product behavior, but the patch must align existing contract tests/docs and provide real Slack proof before it is maintainable.
