## Summary

Keep open: this is a plausible Slack bug fix, but the patch still conflicts with current Slack thread-session tests/docs, lacks after-fix real Slack proof, and would auto-close a broader linked bug that it explicitly does not fully fix.

## What This Changes

The PR changes Slack DM thread replies to route through the main DM session, adds a verbose log for unresolved assistant message edits, and adds one routing unit test.

## Best Possible Solution

Make an explicit Slack routing decision for DM thread replies, update the prepared-message/dispatch tests and Slack docs to match it, avoid auto-closing the broader assistant_app_thread issue unless that case is fixed or split, and land with redacted live Slack proof or a maintainer proof override.

## Reproduction Assessment

Yes at source level: current main maps direct-message thread replies to `threadTs`, routes through thread session key generation, and existing tests assert thread-scoped DM session keys. I did not run a live Slack Socket Mode reproduction in this read-only review.

## Solution Assessment

Unclear: flattening DM thread replies may be the right fix, but this PR is not yet the best maintainable form because it leaves current tests/docs and the broader linked issue inconsistent, and it lacks real Slack proof.
