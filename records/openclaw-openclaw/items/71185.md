## Summary

Keep this PR open: current main still has the brittle trajectory support-capture path, and the PR is a focused defensive fix with targeted regression coverage and no blocking review findings. The remaining action is maintainer review and landing, not cleanup closure.

## What This Changes

The PR widens support path redaction to accept nullish inputs, filters or falls back malformed skill snapshot entries during trajectory metadata capture, and adds regression tests for those cases.

## Best Possible Solution

Land the focused consumer-side hardening in trajectory metadata and support redaction, then consider a separate producer-side cleanup in the skills loader if maintainers want to normalize rejected skill entries at the source.

## Reproduction Assessment

Yes. A constructed `SkillSnapshot` containing a `resolvedSkills` entry with undefined `name`, `filePath`, or `baseDir` reaches current main's metadata builder, where undefined paths hit `file.startsWith` and undefined names can hit `localeCompare`.

## Solution Assessment

Yes. The PR is the narrowest maintainable fix for the support-capture crash because it hardens the consumer without weakening the symlink-escape rejection path; producer normalization can remain a follow-up.
