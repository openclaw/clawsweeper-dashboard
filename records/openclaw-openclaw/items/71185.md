## Summary

This PR should stay open for normal maintainer landing review: current main still has the nullable skill-snapshot crash path, and the branch applies a focused consumer-side guard with regression tests and sufficient live-log proof while leaving the symlink-escape security guard unchanged.

## What This Changes

The branch makes trajectory support metadata tolerate malformed skill snapshot entries by widening path redaction for nullish input, filtering invalid resolved skills with fallback to the lightweight skill list, and adding regression coverage.

## Best Possible Solution

Land the focused consumer hardening once required checks are green or unrelated failures are confirmed, and keep producer-side normalization of rejected skill entries as a separate cleanup if maintainers want a stricter contract.

## Solution Assessment

Yes. Defensive tolerance in the support-capture consumers is the narrowest immediate fix for preventing diagnostic metadata from failing agent turns while preserving the symlink-escape rejection behavior.
