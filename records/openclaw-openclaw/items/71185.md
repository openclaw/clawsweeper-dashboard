Keep open for normal PR merge handling: current main still has the unsafe trajectory support-capture path, while this PR now contains the focused defensive fix, regression tests, real behavior proof, and the previously missing changelog entry. I found no blocking correctness or security findings.

## What This Changes

The PR hardens trajectory support metadata capture against partial skill snapshot entries, widens support path redaction for nullish inputs, adds regression tests, and documents the user-facing fix in the changelog.

## Best Possible Solution

Land the focused consumer-side hardening once required checks pass, and leave producer-side skill snapshot normalization as an optional follow-up if maintainers want a cleaner source fix.

## Reproduction Assessment

Yes. By source inspection, a `SkillSnapshot` with a partial `resolvedSkills` entry reaches current main’s metadata builder, where undefined paths hit `file.startsWith` and undefined names can hit `localeCompare`; the PR body also supplies matching live crash logs.

## Solution Assessment

Yes. Hardening the support-capture consumer is the narrowest maintainable fix for this failure path because it preserves the symlink-escape rejection behavior while preventing malformed diagnostic metadata from failing the whole agent turn.
