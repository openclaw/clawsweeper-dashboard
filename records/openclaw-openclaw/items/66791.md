## Summary

Keep this issue open. Current main still exposes Discord scheduled-event list/create only; event-edit and event-delete are absent from the shared message-tool contract, Discord discovery, parser/runtime dispatch, REST helpers, CLI/docs, and targeted tests, while the reporter's follow-up matches the existing bundled Discord implementation path.

## What This Changes

Add Discord message-tool actions for editing and deleting existing scheduled events by event ID.

## Best Possible Solution

Implement the missing lifecycle actions in the bundled Discord scheduled-event path: add shared action/schema/display/target metadata, expose both actions under the existing Discord `events` gate, add parser/runtime/send/REST helpers for get/patch/delete, preserve event-create behavior, update CLI/docs/changelog, and cover edit merge semantics plus delete dispatch with focused tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to inspect current main: the action enum, event schema, Discord discovery, parser/runtime, REST helpers, CLI, and docs all stop at event-list/event-create, so an agent cannot currently discover or dispatch event-edit or event-delete.

## Solution Assessment

Yes. The narrowest maintainable solution is to extend the existing bundled Discord events action group rather than move this to ClawHub; event-edit should merge only supplied fields against the current scheduled event before PATCH, and event-delete should remain behind the existing Discord events/admin gate.
