## Summary

Keep open. Current main only exposes Discord scheduled events through `event-list` and `event-create`; the requested edit/delete actions are absent from discovery, dispatch, runtime, REST helper, CLI, and docs surfaces. The reporter's follow-up accurately identifies a narrow Discord plugin-owned implementation path, so this is a good focused repair candidate.

## What This Changes

Add Discord message-tool actions to edit and delete scheduled events after creation.

## Best Possible Solution

Implement `event-edit` and `event-delete` inside the existing Discord plugin event action group, with typed REST helpers, partial-update behavior that preserves omitted fields, CLI/docs/tests/changelog coverage, and no new core Discord-specific seam.

## Solution Assessment

Yes. Extending the existing Discord event pipeline is the narrowest maintainable path; the reporter's suggestion to merge only user-provided edit fields is the safest implementation detail to validate against Discord's scheduled-event constraints.
