## Summary

Keep this open. Current main still exposes Discord scheduled events only for listing and creation; the requested edit/delete actions are absent from shared action names, Discord discovery, action dispatch, runtime handling, REST wrappers, docs, and tests. The reporter's follow-up comment accurately identified the existing implementation path, making this a narrow Discord plugin follow-up rather than a stale or duplicate cleanup item.

## What This Changes

Request Discord message-tool support for editing and deleting scheduled events after creation.

## Best Possible Solution

Add `event-edit` and `event-delete` to the official Discord plugin under the existing `events` action gate, with typed REST helpers, focused tests, Discord docs, and a changelog entry.

Yes for the feature gap. Source inspection shows no `event-edit` or `event-delete` action names, discovery entries, parser branches, runtime cases, or REST helpers, so the requested tool actions cannot be used on current main.
