## Summary

Keep this issue open. Current main still exposes Discord scheduled-event list/create only, and event-edit/event-delete are absent from the shared message-tool contract, Discord discovery, parser/runtime dispatch, REST helpers, CLI/docs, and tests; the reporter's follow-up accurately maps the existing bundled Discord implementation path.

## What This Changes

Add Discord message-tool actions for editing and deleting existing scheduled events, including eventId-based schema fields and Discord REST support.

## Best Possible Solution

Implement event-edit and event-delete as a focused extension of the existing bundled Discord scheduled-event lifecycle: update the shared action list/target metadata/schema/display metadata, add eventId support, expose actions from Discord discovery, add parser/runtime/REST helpers for get/patch/delete, update CLI/docs/changelog, and cover payload merging plus delete routing with targeted tests.
