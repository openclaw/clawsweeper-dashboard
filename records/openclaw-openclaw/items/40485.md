The PR changes legacy Telegram native-command code so direct-chat `/new` and `/reset` replies force `replyToMode: "off"`, and adds a legacy `/new` regression test.

## Best Possible Solution

Implement the direct-chat `/new` and `/reset` no-reply override in the active Telegram plugin, cover both commands, and add a one-line Unreleased changelog entry.

## Solution Assessment

No. The behavioral direction is narrow, but this PR applies it to removed legacy files; the maintainable fix is to port the override and tests to `extensions/telegram/src` and add the required changelog entry.
