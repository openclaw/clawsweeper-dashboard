## Summary

Keep open: the reported Telegram reply-context bug is still source-traceable, but this PR changes removed legacy `src/telegram` files, so the proposed patch would not affect the active Telegram plugin and it still lacks required real Telegram proof.

## What This Changes

The PR adds a command-specific `replyToMode` override and a `/new` regression test under legacy `src/telegram` files to send direct-chat native `/new` and `/reset` replies without Telegram reply threading.

## Best Possible Solution

Port the direct-chat `/new` and `/reset` reply suppression to `extensions/telegram/src/bot-native-commands.ts`, cover both commands in active plugin tests, preserve group/topic threading, and require redacted live Telegram proof before merge.

## Reproduction Assessment

Yes, at source level. With Telegram `replyToMode` set to `first` or `all`, native `/new` or `/reset` in a private chat reaches the active plugin path, gets the command `message_id` injected, and delivery only drops that target when mode is `off`.

## Solution Assessment

No. The behavioral direction is narrow, but this PR applies it to deleted legacy `src/telegram` paths instead of the active `extensions/telegram` plugin, so the shipped runtime would not receive the fix.
