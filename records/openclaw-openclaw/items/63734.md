## Summary

Keep open. Current main still has deterministic exec security/ask modes, allowlists, safeBins, durable approvals, strict inline-eval checks, and display-only command highlighting, but not the requested built-in L1-L4 risk classifier; implementing it needs security/product design rather than conservative cleanup.

## What This Changes

Request to add a built-in L1-L4 command risk classifier to exec approvals so safe commands can auto-run and dangerous commands can be denied.

## Best Possible Solution

Define an opt-in, parser-backed exec policy surface that coordinates safe argv-shape auto-allow and hard-deny patterns with existing allowlist, safeBins, durable approval, and strictInlineEval semantics, then document and test it.

## Reproduction Assessment

Not applicable as a bug reproduction. Source, docs, tests, live issue context, and related searches show the requested classifier is absent from current main and is a new policy capability.

## Solution Assessment

Unclear. The L1-L4 direction addresses real approval fatigue, but command-text classification is not yet proven to be the safest maintainable design; a parsed, opt-in policy coordinated with denylist work is safer.
