## Summary

Keep this PR open: current main and the latest release still have the local embedded TUI slash-command fallthrough, and this draft PR is a focused fix with sufficient PTY proof and no blocking code findings. The remaining decision is maintainer acceptance of the user-visible local command compatibility change before it lands.

## What This Changes

The PR changes local embedded TUI slash-command parsing, discovery, command handling, and regression tests so known unsupported shared slash commands are hidden or answered locally instead of being sent to the model.

## Best Possible Solution

Land the focused TUI-local interception once the draft is ready and maintainers accept the local unsupported-command contract, while preserving gateway-mode forwarding and unknown slash-command prompts.

## Reproduction Assessment

Yes. Current main source shows slash input goes to `handleCommand()`, local command discovery includes shared commands, and unmatched commands fall through to `sendMessage(raw)` and then `agentCommandFromIngress()`.

## Solution Assessment

Yes. The TUI command boundary is the narrowest maintainable place to stop local embedded slash fallthrough before `sendChat`; implementing full local shared-command execution would be larger product work, and the linked issue explicitly accepts unsupported local responses.
