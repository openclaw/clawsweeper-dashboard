## Summary

Keep open: the performance idea is still potentially useful, but the current patch changes auth repair ordering in a way that can break first-run OAuth cleanup and the external PR still lacks real patched `doctor --fix` proof.

## What This Changes

The PR changes `runDoctorRepairSequence` so plugin dependency cleanup, legacy OAuth sidecar repair, and stale OAuth shadow repair run together via `Promise.all`.

## Best Possible Solution

Keep dependent OAuth repairs serial, parallelize only truly independent cleanup work, add focused repair sequencing coverage, and require redacted patched-run `doctor --fix` proof before merge.

## Reproduction Assessment

No high-confidence live reproduction of the reported wall-clock regression was run. The blocking PR bug is source-reproducible from current main: the ordering test expects sidecar repair before stale-shadow cleanup, and the scanner skips stores that still contain legacy sidecar refs.

## Solution Assessment

No; this is not the best merge shape because it parallelizes auth repairs that share persisted auth profile state. The maintainable fix is to keep the auth dependency ordered while only parallelizing independent cleanup steps.
