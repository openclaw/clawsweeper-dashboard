## Summary

Keep open: the performance idea is plausible, but the current patch is not merge-ready because it parallelizes OAuth repairs that current main and the latest release keep ordered for first-run credential cleanup.

## What This Changes

The PR changes `runDoctorRepairSequence` to run plugin dependency cleanup, legacy OAuth sidecar repair, and stale OAuth shadow cleanup together via `Promise.all`.

## Best Possible Solution

Keep dependent OAuth repairs serial, parallelize only repair steps that do not read or write the same auth profile stores, and require patched-run doctor timing proof before merge.

## Reproduction Assessment

Yes for the patch defect from source inspection: current-main code and tests show stale-shadow cleanup must run after sidecar migration. I did not reproduce the original wall-clock slowdown live in this read-only review.

## Solution Assessment

No; this is not the best merge shape because it parallelizes auth repairs that share persisted auth profile state. The safer solution is to keep that dependency ordered and parallelize only independent cleanup work.
