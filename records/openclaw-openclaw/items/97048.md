## Summary

Keep open: current main still has a split contract where generated plugin catalog shards can preserve `audio`/`video` inputs, but the session model registry validator rejects those values and drops the affected rows. Three open PRs explicitly target this issue, so the right cleanup path is to review and land one fix before closing.

## What This Changes

The issue reports that OpenClaw-generated MiniMax and NVIDIA model catalog shards with `audio` or `video` inputs are rejected, causing those primary models to be dropped and fallback to another model.

## Best Possible Solution

Land one narrow schema/type-alignment PR, preferably the clean proof-backed candidate or an equivalent maintainer revision, then close this issue while leaving broader fallback guardrails to the separate follow-up issues.

## Reproduction Assessment

Yes, as source-reproducible: generated plugin catalog files are loaded through `validateModelsConfig`, whose current TypeBox schema rejects the reported `audio` and `video` values. I did not execute a live provider fetch in this read-only review.

## Solution Assessment

Yes, the best fix is generic schema/type alignment plus generated-catalog regression coverage. Provider-specific clamping for MiniMax or NVIDIA would leave the split validator contract in place.
