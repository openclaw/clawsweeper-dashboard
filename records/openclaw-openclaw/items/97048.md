## Summary

Keep open: current main still rejects generated plugin catalog rows that include audio or video inputs in the session model registry, and an open current-base PR is already the active fix candidate. The issue should close only after one canonical fix PR lands.

## What This Changes

The issue reports generated MiniMax/NVIDIA model catalogs with audio/video inputs being rejected by the model registry schema, causing affected primary models to be dropped and fallback to occur.

## Best Possible Solution

Land one canonical narrow fix that preserves audio/video model inputs through generated catalog loading, registry model typing, and model-list display, then close this issue from that merged PR.

## Reproduction Assessment

Yes from source inspection, though I did not run a live gateway repro in this read-only review. A generated plugin catalog row with `input: ["text", "image", "video"]` or `input: ["text", "image", "audio"]` reaches a registry schema that still only accepts `text` and `image`.

## Solution Assessment

Yes: the best fix is narrow schema/type/list alignment with the generated catalog contract. The active current-base PR appears to be the right place to finish that work, so this issue should remain open until that PR lands or is replaced.
