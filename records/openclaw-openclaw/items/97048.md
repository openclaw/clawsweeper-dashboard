## What This Changes

The issue reports that generated MiniMax and NVIDIA model catalog shards include `audio` or `video` inputs that the session model registry rejects, dropping those models and causing fallback.

## Best Possible Solution

Land one narrow candidate fix that aligns the session registry schema and public model/list type surfaces with the generated catalog input contract, with regression coverage for generated plugin catalog shards carrying `audio` and `video`.

## Reproduction Assessment

Yes by source inspection: generated catalog normalization can emit `audio` and `video`, while the current session registry TypeBox schema only accepts `text` and `image`, matching the reported gateway validation errors. I did not run a live gateway/provider reproduction because this review is read-only.

## Solution Assessment

Yes. The best fix is narrow schema/type alignment plus a generated-catalog regression test in the session model registry path, not a provider-specific clamp or a new config option.
