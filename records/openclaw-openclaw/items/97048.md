## What This Changes

The issue reports that generated MiniMax/NVIDIA plugin model catalogs can include `audio` or `video` inputs that the session model registry rejects, dropping those primary models and causing fallback.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/97252 or an equivalent schema/type-alignment fix, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes. Source inspection shows generated catalog rows with `audio` or `video` reach the TypeBox validator that only accepts `text` and `image`; the issue body includes matching gateway log errors, but I did not run a live provider fetch.

## Solution Assessment

Yes. The narrow maintainable solution is generic schema/type alignment plus regression coverage; closing now would be premature because the strongest fix PR is still open, not merged.
