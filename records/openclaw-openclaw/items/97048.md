## Summary

Keep open: current `main` and the latest release still reject generated plugin catalog rows whose `input` includes `audio` or `video`, and three open PRs are already linked as candidate fixes. The issue should remain open until one of those PRs lands or maintainers choose a different fix path.

## What This Changes

The issue reports that generated MiniMax/NVIDIA model catalog shards with `audio` or `video` inputs fail session registry validation, causing those models to be dropped and triggering fallback.

## Best Possible Solution

Review and land the best linked schema/type-alignment PR, with regression coverage for generated plugin catalog shards carrying `audio` and `video` inputs.

## Reproduction Assessment

Yes. Source inspection shows a generated catalog row with `input: ["text", "image", "video"]` or `audio` reaches the session registry TypeBox validator, which still only accepts `text` and `image`.

## Solution Assessment

Yes. The best fix is generic schema/type alignment plus generated-catalog regression coverage; provider-specific clamping would leave the split validator contract in place.
