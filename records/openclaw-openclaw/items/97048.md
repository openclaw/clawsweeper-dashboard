## Summary

Keep open: current main and v2026.6.10 still reject generated model catalog rows whose `input` includes `audio` or `video`, and several open PRs are candidate fixes rather than merged implementation proof.

## What This Changes

The issue asks OpenClaw to stop rejecting its own generated provider catalog entries when MiniMax/NVIDIA rows include audio or video input modalities.

## Best Possible Solution

Land one canonical additive schema/type/catalog alignment that preserves `audio` and `video` through generated catalog loading, registry models, and model-list output with regression coverage for the reported MiniMax and NVIDIA rows.

## Reproduction Assessment

Yes. A generated catalog row with `input: ["text", "image", "video"]` or `audio` reaches `validateModelsConfig`, whose current TypeBox schema only allows `text` and `image`, causing the shard to be dropped.

## Solution Assessment

Yes. A generic schema/type alignment with generated-catalog regression coverage is the best fix; provider-specific clamping would leave OpenClaw-generated catalog rows and registry validation out of sync.
