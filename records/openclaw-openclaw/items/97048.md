## Summary

Keep open: current main and the latest release still reject generated plugin catalog rows containing `audio` or `video`, and two open PRs are candidate fixes that should be reviewed before this issue closes.

## What This Changes

The issue reports that OpenClaw-generated MiniMax/NVIDIA plugin catalog shards can include `audio` or `video` input modalities that the session model registry rejects, dropping models and causing fallback.

## Best Possible Solution

Review and land one focused candidate PR that aligns the generated catalog registry schema and related model input types with the existing audio/video catalog contract, with regression coverage for generated plugin catalog shards.

## Reproduction Assessment

Yes. Source inspection shows generated catalog rows flow through `validateModelsConfig`, whose current registry schema still only accepts `text` and `image`; I did not run a live provider fetch in this read-only review.

## Solution Assessment

Yes. The best fix is generic registry/schema/type alignment with a generated-catalog regression test; provider-specific clamping for only MiniMax or NVIDIA would leave the inconsistent contract in place.
