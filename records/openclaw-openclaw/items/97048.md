## Summary

Keep open: current main and the latest release still reject generated plugin catalog rows whose `input` includes `audio` or `video`, and two open PRs are linked as candidate fixes but neither has merged. The issue should remain open until one focused fix lands.

## What This Changes

The issue reports that OpenClaw-generated MiniMax/NVIDIA plugin model catalogs can contain `audio` or `video` inputs that the session model registry rejects, dropping those models and causing fallback.

## Best Possible Solution

Review and land one focused fix that aligns generated catalog validation/types with the existing media input contract and adds regression coverage, then close this issue after merge.

## Reproduction Assessment

Yes. Source inspection shows a generated catalog row with `input: ["text", "image", "video"]` or `audio` reaches the session registry validator, whose schema still only allows `text` and `image`; no live provider fetch was run in this read-only review.

## Solution Assessment

Yes. The best fix is generic schema/type alignment plus a regression test for generated plugin catalog shards; provider-specific clamping would leave OpenClaw's generated catalog contract split across paths.
