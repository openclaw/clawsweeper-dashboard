## Best Possible Solution

Review and land one narrow additive schema/type-alignment PR, preferably the latest-head candidate at https://github.com/openclaw/openclaw/pull/97252 if review clears, then close this issue after merge.

Yes. A generated catalog row with `input: ["text", "image", "video"]` or `audio` reaches `validateModelsConfig`, whose current registry schema only allows `text` and `image`; I did not run a live provider fetch in this read-only review.

## Solution Assessment

Yes. The best fix is generic schema/type alignment with generated-catalog regression coverage; provider-specific clamping for MiniMax or NVIDIA would leave the split catalog contract in place.
