## Summary

Keep open: current main and v2026.6.10 still reject generated model catalog rows whose `input` includes `audio` or `video`, and the issue is now paired with open fixing PRs rather than implemented on main.

## What This Changes

The issue asks OpenClaw to stop dropping generated MiniMax/NVIDIA catalog models when provider-reported input modalities include audio or video.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/97252 or an equivalent canonical schema/type alignment with regression coverage, then close this issue and supersede the sibling fix PRs.

## Reproduction Assessment

Yes. Source inspection shows generated plugin catalog rows with `audio` or `video` reach `validateModelsConfig`, whose current main and v2026.6.10 registry schema only accepts `text` and `image`.

## Solution Assessment

Yes. The best fix is generic schema/type alignment plus a generated-catalog regression test; provider-specific clamping would preserve the drift and discard useful provider metadata.
