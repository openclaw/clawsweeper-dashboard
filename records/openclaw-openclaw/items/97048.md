## Summary

Keep open: current main and the latest release still reject generated plugin catalog rows whose model `input` includes `audio` or `video`, and the two linked closing PRs are still open rather than merged.

## What This Changes

The issue requests that OpenClaw accept or safely normalize generated provider model catalog entries whose input modalities include `audio` or `video` so affected primary models are not dropped and replaced by fallback selection.

## Best Possible Solution

Keep this issue open as the canonical report, review or land the stronger linked candidate PR, then close only after the generated-catalog registry validator and related type surfaces are aligned on main.

## Reproduction Assessment

Yes, at source level: generated plugin catalog files are loaded through `loadCustomModels`, and `validateModelsConfig` still rejects any model `input` outside `text` or `image`. I did not run a live MiniMax/NVIDIA fetch in this read-only review.

## Solution Assessment

Yes. The best fix is generic registry/schema/type alignment plus generated-catalog regression coverage; a provider-specific clamp for only MiniMax or NVIDIA would leave the split contract in place.
