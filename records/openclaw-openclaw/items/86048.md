## What This Changes

The issue reports a WSL2 NVIDIA GPU-PV lockup after a llama-server D-state crash, with OpenClaw agent runs stalling until the WSL VM is restarted.

## Best Possible Solution

Decide the ownership boundary first: document this as a WSL2/NVIDIA/llama.cpp operational limitation with OpenClaw timeout/fallback/localService guidance, or design an explicit host-side watchdog/advisory surface if maintainers want OpenClaw to own more than transport failure handling.
