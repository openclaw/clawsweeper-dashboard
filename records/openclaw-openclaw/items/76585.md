## What This Changes

The issue asks OpenClaw to stop cron jobs from timing out under event-loop saturation caused by large binary files in an agent workspace.

## Best Possible Solution

Keep this issue as the narrow workspace-bloat diagnostic thread under the broader #76562 regression family, add a minimal current-main repro, then cap, skip, yield, cache, or async-offload the specific blocking prep work rather than only increasing timeouts.
