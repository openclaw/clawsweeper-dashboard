## Summary

Keep open: the PR is not merge-ready because it conflicts with existing WebChat reload and session-store contracts, and its cron recovery path can mark legitimate long-running work as done.

## What This Changes

The PR changes WebChat final-event reload handling, gateway session entry resolution, and cron session reaping/tests to suppress no-message final reloads, prefer exact session keys, and mark old running cron run rows done.

## Best Possible Solution

Rework the branch as narrower fixes that preserve current final-event reconciliation, fix archived-session selection without breaking duplicate-alias freshness, and recover cron runs only with runtime ownership or timeout-aware lifecycle proof.

## Reproduction Assessment

Yes. The PR-introduced regressions are reproducible from source-level contracts and tests: no-message final reload coverage, duplicate-row session lookup/history tests, and cron timeout plus lifecycle timestamp behavior all show unsafe paths.

## Solution Assessment

No. The proposed changes use broad heuristics where the safer path is to distinguish command finals, target archived session keys precisely, and prove cron runs are no longer active before marking them terminal.

Full review comments:

- **[P2] Preserve no-message final history reloads:** `ui/src/ui/chat-event-reload.ts:10-12`
  - body: Returning false whenever `payload.message` is absent disables the fallback reload path for final events that currently reconcile deferred `session.message` updates or no-payload terminal events. Current main has explicit coverage expecting these finals to reload history.
  - confidence: 0.9
- **[P1] Keep freshest duplicate session rows:** `src/gateway/session-utils.ts:400-408`
  - body: `loadSessionEntry` now prefers `store[trimmedKey]` or `store[canonicalKey]` before checking duplicate store keys. That can return stale metadata and history when a fresher legacy or case-variant duplicate exists, which current gateway tests cover.
  - confidence: 0.92
- **[P1] Prove cron runs are unowned before marking done:** `src/cron/session-reaper.ts:106-109`
  - body: The reaper marks any running cron row older than two hours as done without checking runtime ownership. Agent-turn cron jobs can have long or disabled timeouts, and lifecycle `updatedAt` is not a heartbeat, so active runs can be finalized and later pruned.
  - confidence: 0.93
- **[P2] Avoid persisting epoch endedAt:** `src/cron/session-reaper.ts:107-109`
  - body: When `entry.updatedAt` is missing, `updatedAt` becomes 0 and the recovered row is persisted with `endedAt: 0`, which would display the Unix epoch instead of the recovery time.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `ui/src/ui/chat-event-reload.ts:7`
  - body: This is a user-facing fix PR, but the changed files do not include `CHANGELOG.md`. Repo policy requires a changelog entry for user-facing fix changes before merge.
