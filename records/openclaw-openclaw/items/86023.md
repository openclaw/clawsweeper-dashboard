## Summary

Keep open: current main still has the fixed Codex app-server native-token startup guard and still invalidates Codex bindings after context-engine-owned compaction. The open related PRs at https://github.com/openclaw/openclaw/pull/85978 and https://github.com/openclaw/openclaw/pull/86069 cover narrower slices, but they do not implement the requested rotation diagnostics, semantic binding ownership across compaction/rollover, workspace bootstrap fingerprints, or doctor/status distinction.

## What This Changes

The issue asks for Codex app-server native thread reuse to be governed by semantic context-engine/bootstrap ownership, rotation diagnostics, and compaction or rollover compatibility instead of a blunt native-token guard.

## Best Possible Solution

Keep this as the canonical architecture follow-up: review the narrower open PRs separately, then implement reason-coded Codex native-thread rotation diagnostics and semantic binding ownership in small owner-reviewed changes.
