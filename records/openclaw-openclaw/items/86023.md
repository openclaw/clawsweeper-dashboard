## Summary

Keep open: current main has addressed the old fixed-70k startup-rotation slice with a model-aware native-thread fuse, but the broader semantic ownership request is still live because current compaction clears the stored `thread_bootstrap` projection and the lifecycle diagnostics/preservation work remains in open PRs.

## What This Changes

The issue asks Codex long-running sessions to own native thread/bootstrap cache rotation semantically across context-engine projection, compaction, rollover, and diagnostics instead of relying on blunt token or session-file checks.

## Best Possible Solution

Keep this issue as the canonical tracker until maintainers settle the diagnostics and semantic-preservation design, then split any confirmed Pi/reload compute-cache work into a narrower follow-up if it is distinct.

## Reproduction Assessment

Yes. Source inspection on current main shows post-context-engine native compaction clears `contextEngine.projection`, and the adjacent test asserts that projection becomes undefined; I did not run a live long Discord/Codex session in this read-only review.

## Solution Assessment

Unclear as a final design. Semantic preservation matches the `thread_bootstrap` contract, but the lifecycle diagnostics, status surface, and possible Pi compute-cache scope need maintainer sequencing through the open PRs rather than a fresh side fix.
