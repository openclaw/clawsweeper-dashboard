## Summary

Keep open for revision: this is a focused Ollama bug-fix attempt, but the patch does not cover the linked `glm-5.2:cloud` repro and there is no after-fix real Ollama or Telegram behavior proof.

## What This Changes

The PR changes the Ollama provider policy resolver to classify unspecified-reasoning models as reasoning-capable when their model id matches the existing Ollama reasoning-name heuristic.

## Best Possible Solution

Feed live Ollama `/api/show` capability metadata into the native `/think` menu/catalog context for live-discovered models, preserve explicit `reasoning: false`, and cover `glm-5.2:cloud` with focused tests before merge.

## Reproduction Assessment

Yes from source, but not from a live run. Telegram native `/think` menus use configured catalog metadata by default, while the linked live-discovered `glm-5.2:cloud` model lacks configured reasoning and the PR fallback regex does not match it.

## Solution Assessment

No. The patch is a plausible fallback for some unknown reasoning model names, but the best fix should pass live Ollama capability metadata into the menu/profile decision and add a regression for the reported model.
