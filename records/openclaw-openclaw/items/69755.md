## Summary

Keep open. Current main and the latest stable release still format restart sentinel notices with a direct `Run: openclaw doctor --non-interactive` hint and reuse that same text through system-event, heartbeat, and outbound delivery paths, while the related doctor-summary PR is still open and does not cover this actionability problem.

## What This Changes

The issue asks for post-restart doctor guidance to distinguish a recommended follow-up from whether it is actionable in the current heartbeat or approval-constrained surface.

## Best Possible Solution

Make restart-sentinel doctor guidance context-aware or conservatively worded so constrained heartbeat/outbound notices route users to a terminal or approvals-capable UI while preserving profile-aware command formatting.

## Reproduction Assessment

Yes by source inspection. Current main builds restart sentinel payloads with a direct doctor hint and delivers the same formatted text through heartbeat/system-event and outbound paths where approval completion may be unavailable.

## Solution Assessment

Yes. The narrow maintainable fix is to adjust restart-sentinel hint generation or delivery formatting, not to change doctor behavior, weaken exec approvals, or build broader self-healing.
