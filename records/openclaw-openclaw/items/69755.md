## Summary

Keep open. Current main still emits a direct `Run: openclaw doctor --non-interactive` restart-sentinel hint and sends the same message through heartbeat/outbound paths without checking whether that surface can complete approval-gated execution.

## What This Changes

The issue requests context-aware post-restart doctor guidance so constrained heartbeat or approval-limited surfaces frame `openclaw doctor --non-interactive` as a terminal or approvals-capable follow-up.

## Best Possible Solution

Make restart sentinel follow-up text or payload formatting distinguish terminal/approval-capable execution from outbound or heartbeat delivery while preserving the doctor command where it is genuinely actionable.

## Reproduction Assessment

Yes. Current main statically reproduces the loop: restart sentinel payloads carry a direct doctor command, startup forwards that text through heartbeat/outbound delivery, and exec approvals can deny timed-out approval attempts as reported.

## Solution Assessment

Yes. The narrow maintainable fix is restart-sentinel follow-up copy/context handling, not weakening exec approvals, changing doctor behavior, or adding a broad recovery loop.
