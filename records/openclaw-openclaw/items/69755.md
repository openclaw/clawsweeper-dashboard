## Summary

Keep open. Current main still formats the restart-sentinel doctor follow-up as a direct `Run:` command and forwards that same text through wake/outbound delivery paths without distinguishing approval-capable contexts from heartbeat or constrained surfaces.

## What This Changes

Requests context-aware restart/config-patch follow-up messaging so `openclaw doctor --non-interactive` is not presented as immediately actionable where approval-gated execution cannot complete.

## Best Possible Solution

Make restart-sentinel follow-up text distinguish terminal or approvals-capable follow-up from constrained heartbeat/outbound delivery while preserving the doctor command where it is genuinely actionable.

## Reproduction Assessment

Yes by source inspection, though not by a live macOS session: current main writes direct doctor hints into restart sentinel payloads, formats them into a single message, and forwards that message through heartbeat/outbound delivery while approval timeouts can deny the suggested command.

## Solution Assessment

Yes. The narrow maintainable fix is restart-sentinel follow-up copy or payload formatting that accounts for delivery/actionability context, not changing doctor behavior, weakening exec approvals, or implementing a broad self-healing workflow.
