## Summary

Keep open. Current main still emits the restart-sentinel doctor follow-up as a direct `Run:` command and forwards that same formatted text through wake/outbound delivery without knowing whether the receiving surface can complete exec approvals.

## What This Changes

The issue asks for post-restart doctor follow-up text to distinguish recommended terminal or approvals-capable follow-up from constrained heartbeat/outbound contexts.

## Best Possible Solution

Adjust restart-sentinel follow-up copy or formatting so constrained wake/outbound contexts direct the user to a terminal or approvals-capable UI, while preserving the doctor command where it is actually actionable.

Yes by source inspection, though not by a live macOS session: current main writes direct doctor hints into restart sentinel payloads and forwards the same text through wake/outbound delivery while approval docs define unreachable-UI fallback denial paths.

## Solution Assessment

Yes. The narrow maintainable fix is context-aware restart-sentinel follow-up wording or payload formatting, not changing doctor behavior, weakening exec approvals, or adding a broad self-healing workflow.
