## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Extend the strict model schema:** `src/config/types.agents-shared.ts:39`
  - body: `fallbackOnErrors` is added to `AgentModelConfig`, but `AgentModelSchema` is still `.strict()` and only accepts the existing keys. A user following the PR examples would hit an unrecognized-key validation error before fallback routing can read the setting.
  - confidence: 0.92
- **[P1] Honor default-level fallbackOnErrors:** `src/agents/agent-scope.ts:259-263`
  - body: This resolver only reads the selected agent's `model`, so the documented `agents.defaults.model.fallbackOnErrors` example is ignored for agents without a per-agent model object. The option needs the same effective-default resolution as fallback lists, or the advertised scope should be narrowed.
  - confidence: 0.9
- **[P2] Pass fallbackOnErrors into queued follow-up runs:** `src/auto-reply/reply/agent-runner-utils.ts:168-173`
  - body: `resolveModelFallbackOptions()` now returns `fallbackOnErrors`, but `createFollowupRunner()` still builds its own `runWithModelFallback()` arguments and only forwards `fallbacksOverride`. Queued follow-ups would keep the old retry policy for the same agent config.
  - confidence: 0.82
- **[P2] Stop image fallback on disallowed errors:** `src/agents/model-fallback.ts:838`
  - body: The image/PDF fallback path passes `fallbackOnErrors` into each attempt, but after a disallowed error comes back the loop still records it and continues to later candidates. With an allowlist such as `[429]`, a 400 or 401 would still try later image/PDF models instead of surfacing the disallowed error.
  - confidence: 0.84
- **[P2] Preserve fallbackOnErrors during model scans:** `src/config/types.agents-shared.ts:39`
  - body: After this becomes persisted config, `openclaw models scan` still rebuilds model defaults from only `primary` and `fallbacks`. Running the scan can silently delete the new policy unless the rewrite path preserves the field or this public config surface is narrowed.
  - confidence: 0.78
- **[P2] Narrow fallbackOnErrors to supported model configs:** `src/config/types.agents-shared.ts:39`
  - body: `AgentModelConfig` is reused for `imageModel`, `pdfModel`, and generation model defaults, but those paths either coerce configs down or do not pass this setting through consistently. Exposing the key on the shared type makes it look supported outside chat fallback unless those paths are wired and tested or the type/schema is split.
  - confidence: 0.74
