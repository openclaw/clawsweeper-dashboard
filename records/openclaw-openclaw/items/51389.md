## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Extend the strict model schema:** `src/config/types.agents-shared.ts:39`
  - body: `fallbackOnErrors` is added to `AgentModelConfig`, but `AgentModelSchema` is strict and still accepts only the existing model keys on current main. A user following the PR examples would hit an unrecognized-key validation error before the fallback runner can read the setting.
  - confidence: 0.92
- **[P1] Honor defaults.model fallbackOnErrors:** `src/agents/agent-scope.ts:259-263`
  - body: This resolver only reads the per-agent `model`, so the documented `agents.defaults.model.fallbackOnErrors` example is ignored for agents without a per-agent model object. The option needs the same effective-default resolution as the fallback list, or the docs/config surface should be narrowed.
  - confidence: 0.89
- **[P2] Fail fast in image fallback for disallowed errors:** `src/agents/model-fallback.ts:834-839`
  - body: The image fallback path passes `fallbackOnErrors` into each attempt, but after a disallowed error comes back the loop records it and continues to the next candidate. With a custom allowlist like `[429]`, a 400 or 401 would still burn through later image/PDF models instead of surfacing the disallowed error.
  - confidence: 0.84
- **[P2] Preserve fallbackOnErrors during model scans:** `src/config/types.agents-shared.ts:39`
  - body: After this field becomes persisted config, `openclaw models scan` still rebuilds model defaults from only `primary` and `fallbacks`. Running the scan can silently delete the new policy unless the rewrite path preserves the field or this public config surface is narrowed.
  - confidence: 0.78
- **[P2] Scope fallbackOnErrors away from unsupported model configs:** `src/config/types.agents-shared.ts:39`
  - body: `AgentModelConfig` is reused for `imageModel`, `pdfModel`, and generation model defaults, but those tool paths currently coerce configs down to primary/fallback/timeout or use separate runtimes. Exposing the key on the shared type makes it look supported outside chat fallback unless the media paths are wired and tested or the type/schema is split.
  - confidence: 0.74
