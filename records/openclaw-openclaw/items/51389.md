## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Extend the strict model schema:** `src/config/types.agents-shared.ts:39`
  - body: `fallbackOnErrors` is added to `AgentModelConfig`, but `AgentModelSchema` is still `.strict()` and only accepts `primary`, `fallbacks`, and `timeoutMs`. A user following the PR examples would hit an unrecognized-key validation error before the fallback runner can read the setting.
  - confidence: 0.92
- **[P1] Honor default-level fallbackOnErrors:** `src/agents/agent-scope.ts:259-263`
  - body: This resolver only reads the per-agent `model`, so the documented `agents.defaults.model.fallbackOnErrors` example is ignored for agents without a per-agent model object. The option needs the same effective-default resolution as the fallback list, or the docs/config surface should be narrowed.
  - confidence: 0.9
- **[P2] Stop image fallback on disallowed errors:** `src/agents/model-fallback.ts:838-839`
  - body: The image/PDF fallback path passes `fallbackOnErrors` into each attempt, but after a disallowed error comes back the loop records it and continues to the next candidate. With an allowlist like `[429]`, a 400 or 401 would still try later image/PDF models instead of surfacing the disallowed error.
  - confidence: 0.84
- **[P2] Preserve fallbackOnErrors during model scans:** `src/config/types.agents-shared.ts:39`
  - body: After this becomes persisted config, `openclaw models scan` still rebuilds model defaults from only `primary` and `fallbacks`. Running the scan can silently delete the new policy unless the rewrite path preserves the field or this public config surface is narrowed.
  - confidence: 0.78
- **[P2] Narrow fallbackOnErrors to supported model config surfaces:** `src/config/types.agents-shared.ts:39`
  - body: `AgentModelConfig` is reused for `imageModel`, `pdfModel`, and generation model defaults, but those tool paths either coerce configs down to primary/fallback/timeout or use separate runtimes. Exposing this key on the shared type makes it look supported outside chat fallback unless those paths are wired and tested or the type/schema is split.
  - confidence: 0.74
