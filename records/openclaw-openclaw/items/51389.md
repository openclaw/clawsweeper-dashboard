Current main still does not implement the public `fallbackOnErrors` model config surface, so the PR is not obsolete, but the branch is not merge-ready because the proposed config is not accepted by the strict schema and the documented default-level setting is not resolved by the new wiring.

## What This Changes

The PR adds a `fallbackOnErrors` model config type/resolver and threads a configurable HTTP-status fallback policy through parts of chat and cron model fallback handling.

## Best Possible Solution

If OpenClaw wants this public surface, land a current-main implementation that extends the type, strict schema, config help/docs, and generated metadata together; resolves default and per-agent policy consistently; preserves the field in config rewrite commands; explicitly scopes or rejects image/PDF/media model policy; fails fast on disallowed custom statuses; and adds focused fallback tests for `default`, `all`, custom arrays, queued follow-ups, cron, and any supported media path.

  - confidence: 0.88

Mirror the chat fallback fail-fast check here, or avoid exposing the option on this path until it is supported.
