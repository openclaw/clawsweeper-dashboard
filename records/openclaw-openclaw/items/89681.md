## What This Changes

The PR changes the Control UI Skills view to render skill groups and skill rows with keyed Lit `repeat()` calls and adds a jsdom regression test for checkbox state after filtering.

## Best Possible Solution

Land the keyed `repeat()` fix with the regression test after normal maintainer review and required checks.

## Reproduction Assessment

Yes. Current main has a clear source-level reproduction path: toggle a disabled skill so it leaves the Disabled tab, then the next positional row can inherit the mutated checkbox DOM state.

## Solution Assessment

Yes. Keying the filtered group and skill rows at the rendering owner is the narrow maintainable fix; resetting checkbox state locally would treat the symptom while leaving positional DOM reuse intact.
