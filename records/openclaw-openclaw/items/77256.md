## Summary

Keep open: current main does not have the new hook, but the PR is not merge-ready because the Pi path queues next-turn runtime context before the hook decision, real behavior proof is missing, and the new prompt-access API needs maintainer review.

## What This Changes

The PR adds a typed fail-closed `before_model_call` plugin hook across plugin contracts, Pi/CLI/Codex submission paths, docs, generated baselines, tests, and changelog.

## Best Possible Solution

Land a generic final-submission hook only after the API boundary is accepted, blocked turns leave no queued hidden context, and exact-head real behavior proof shows both allow and block paths.

## Reproduction Assessment

Not applicable for the requested feature because this is a new public plugin hook, not a bug report. The blocking review finding is source-reproducible from the PR branch: the Pi path queues hidden next-turn runtime context before the preflight hook can block.

## Solution Assessment

No as submitted: the proposed hook shape may be useful, but the Pi path should move `queueRuntimeContextForNextTurn` until after `before_model_call` continues and add regression coverage for blocked turns with runtime context.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Queue runtime context only after preflight passes:** `src/agents/pi-embedded-runner/run/attempt.ts:3012-3015`
  - body: The Pi path queues hidden runtime context before `before_model_call` runs. Since that helper sends the context with `deliverAs: "nextTurn"`, a blocked or failed preflight skips the current model call but can still leak stale hidden context into a later turn. Move this queueing into the successful preflight branch and cover a blocked runtime-context turn.
  - confidence: 0.89
