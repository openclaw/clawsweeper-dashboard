## Summary

Keep this PR open, but it is not ready to merge: the proposed public hook is a new feature, current main does not already implement it, and the PR head still has source-reproducible security and side-effect blockers plus no inspectable real behavior proof.

## What This Changes

The PR adds a fail-closed `before_model_call` plugin hook across Pi, CLI, and Codex model submission paths, with docs, tests, config baselines, and SDK API baseline updates.

## Best Possible Solution

Land one coherent model-call hook API only after preserving the existing hook catalog and access gates, making blocked turns side-effect-free, resolving the overlap with `before_agent_run` and https://github.com/openclaw/openclaw/pull/58360, and proving real allow/block behavior.

## Reproduction Assessment

No high-confidence current-main bug reproduction is applicable because this is a new public plugin hook. The blocking PR issues are source-reproducible by comparing PR head `src/plugins/hook-types.ts` and `src/agents/pi-embedded-runner/run/attempt.ts` with current main.

## Solution Assessment

No, not as submitted. The API idea may be useful, but this branch must first preserve current conversation-access gates, move runtime-context queueing until after an allowed preflight, and get maintainer API/security review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve current conversation hook gates:** `src/plugins/hook-types.ts:171-177`
  - body: Current main gates `before_model_resolve`, `before_agent_reply`, and `before_agent_run` behind `allowConversationAccess`, but this branch's conversation-hook list drops those names while adding `before_model_call`. Rebase and add the new hook without removing the current raw conversation hooks, otherwise non-bundled plugins can bypass or lose existing protected hook behavior.
  - confidence: 0.9
- **[P2] Queue runtime context after preflight passes:** `src/agents/pi-embedded-runner/run/attempt.ts:3012-3015`
  - body: `queueRuntimeContextForNextTurn` runs before `before_model_call`, so a blocked or failed preflight can still leave hidden runtime context queued for a later model request. Move that queueing into the allowed branch after the hook succeeds and cover a blocked turn with runtime context.
  - confidence: 0.9
