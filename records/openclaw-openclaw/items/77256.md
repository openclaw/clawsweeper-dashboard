## Summary

This PR should stay open, but not as-is: current main still lacks `before_model_call`, the PR is a broad new plugin API surface, and source review found a blocking Pi-side side effect when the hook blocks. It also still needs maintainer API/product review and contributor real behavior proof before merge.

## What This Changes

The PR adds a fail-closed `before_model_call` plugin hook with conversation-access gating, Pi/CLI/Codex preflight wiring, tests, docs, generated baselines, and a changelog entry.

## Best Possible Solution

Land a single generic final-submission hook only after the API boundary is accepted, blocked turns are side-effect-free, and exact-head real behavior proof shows the hook blocking and allowing real model submissions.

## Reproduction Assessment

Not applicable for the requested feature because this is a new public plugin hook, not a bug report. The blocking review finding is source-reproducible from the PR branch: the Pi path queues hidden next-turn runtime context before the preflight hook can block.

## Solution Assessment

No as submitted: the proposed hook shape may be useful, but the Pi path should move `queueRuntimeContextForNextTurn` until after `before_model_call` continues and add regression coverage for blocked turns with runtime context.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Queue runtime context only after preflight passes:** `src/agents/pi-embedded-runner/run/attempt.ts:3012-3015`
  - body: In the Pi path, hidden runtime context is queued with `deliverAs: "nextTurn"` before `before_model_call` runs. If the hook blocks or fails, the current prompt is skipped, but that queued context remains and can be sent with a later model call despite the veto. Move the queueing into the non-blocked branch after the preflight succeeds.
  - confidence: 0.89

## Security Review

Status: needs_attention

Summary: The diff adds a sensitive prompt-access capability and currently lets the Pi path queue hidden runtime context before a blocking preflight decision.

Concerns:

- **[medium] Blocked Pi preflight can leave hidden context queued:** `src/agents/pi-embedded-runner/run/attempt.ts:3012`
  - body: Because `queueRuntimeContextForNextTurn` runs before `before_model_call`, a blocked or failed preflight can still leave internal runtime context queued for a later turn. That weakens the fail-closed model-call gate and can expose stale hidden context to a later provider request.
  - confidence: 0.86
