## Summary

Keep this PR open, but do not merge it as submitted. The branch adds a plausible core hook surface, but source review found blocking security/correctness issues in the hook access policy and Pi blocked-turn ordering, and the external contributor still has not provided inspectable real behavior proof.

## What This Changes

The PR adds a fail-closed `before_model_call` plugin hook across Pi, CLI, and Codex model submission paths, updates conversation-access policy/docs/baselines, and adds related tests.

## Best Possible Solution

Land one coherent model-call hook API only after preserving existing conversation-access gates, making blocked turns side-effect-free, and proving real allow/block behavior at the exact head.

## Reproduction Assessment

Not applicable for the requested feature because this is a new public plugin hook, not a current-main bug. The blocking review findings are source-reproducible from the PR head: the hook list drops current-main gated names and the Pi path queues next-turn runtime context before the preflight decision.

## Solution Assessment

No as submitted. The idea may be useful, but the implementation should first preserve the existing plugin access contract, move Pi runtime-context queueing after a successful preflight, and resolve the API overlap with https://github.com/openclaw/openclaw/pull/58360.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve existing conversation hook gates:** `src/plugins/hook-types.ts:171-177`
  - body: Current main gates `before_model_resolve`, `before_agent_reply`, and `before_agent_run` behind `allowConversationAccess`, but this branch's `CONVERSATION_HOOK_NAMES` drops those names while adding `before_model_call`. That would let non-bundled plugins register existing raw prompt/session hooks without the required trust opt-in. Add the new hook without removing the existing conversation hook names and restore the loader coverage.
  - confidence: 0.88
- **[P2] Queue runtime context only after preflight passes:** `src/agents/pi-embedded-runner/run/attempt.ts:3012-3021`
  - body: In the Pi path, hidden runtime context is queued with `deliverAs: "nextTurn"` before `before_model_call` runs. If the hook blocks or fails, the model request is skipped but the queued context remains and can be sent with a later request. Move this queueing into the allowed branch after preflight succeeds, and cover a blocked turn with runtime context.
  - confidence: 0.9

## Security Review
