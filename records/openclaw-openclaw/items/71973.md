## Summary

Keep open. Current main still appends volatile exact wall-clock text into heartbeat, isolated cron, session reset, post-compaction, and memory-flush prompt bodies; the merged Reference UTC change only changed formatting, and the broader cron cache-affinity issue remains separate.

## What This Changes

Request to move exact `Current time:` injection out of cache-sensitive recurring prompt prefixes while keeping stable timezone context available.

## Best Possible Solution

Keep stable timezone-only context cacheable, move exact wall-clock values to `session_status` or late turn-specific text only where required, and leave provider cache-affinity work to the separate cron cache issue.

## Reproduction Assessment

Yes. Source inspection on current main shows the helper formats fresh exact time and the heartbeat, isolated cron, reset, post-compaction, and memory-flush builders still append it; committed tests assert those lines.

## Solution Assessment

Yes. Targeting the shared helper contract and named prompt builders is the narrow maintainable direction; cross-run provider affinity remains tracked separately at https://github.com/openclaw/openclaw/issues/62809.
