## Best Possible Solution

Update the per-turn payload builders so exact wall-clock time is omitted, delayed, or tool-provided for heartbeat, isolated cron, and post-compaction flows, while keeping timezone-only system prompt context and existing date-placeholder semantics intact.

Fix https://github.com/openclaw/openclaw/issues/71973. Current main still appends exact `Current time:` wall-clock lines into cache-sensitive per-turn payloads for heartbeat, isolated cron, and post-compaction refresh. Remove, delay, or replace those exact-time prompt-prefix injections so repeated runs keep stable prefixes, while preserving timezone-only system prompt behavior, `YYYY-MM-DD` substitution/date awareness, and access to current time through `session_status` or an equivalent late-bound runtime source.
