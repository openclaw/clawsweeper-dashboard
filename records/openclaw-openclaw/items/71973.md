## Summary

Keep open. Current main still injects fresh exact wall-clock text into heartbeat, isolated cron, session reset, and post-compaction prompt bodies, while the stable system prompt intentionally carries only timezone metadata for cache stability. The merged related PR at https://github.com/openclaw/openclaw/pull/42654 only split the local and UTC time format, and the broader cache-affinity issue at https://github.com/openclaw/openclaw/issues/62809 does not cover this narrower prompt-prefix cleanup.

## What This Changes

The issue asks OpenClaw to keep exact wall-clock time out of cache-sensitive recurring prompt prefixes while preserving stable timezone context.

## Best Possible Solution

Move or omit exact wall-clock text from the affected per-turn prompt builders unless the turn truly needs it, while keeping timezone-only system context and `session_status` as the exact-time path.

## Reproduction Assessment

Yes. Source inspection on current main shows the helper formats fresh exact time and the heartbeat, isolated cron, session reset, and post-compaction builders append it into prompt bodies; committed tests still assert those lines.

## Solution Assessment

Yes. Targeting the per-turn prompt builders and shared helper contract is the narrow maintainable direction; provider cache-key affinity remains separately tracked at https://github.com/openclaw/openclaw/issues/62809.
