## Summary

Keep open. Current main still appends a fresh `Current time:` line into the heartbeat body, isolated cron command body, and post-compaction refresh block, while the existing cache-boundary docs/source only prove the stable system prompt is timezone-only.

## What This Changes

Move exact wall-clock current-time lines out of recurring cache-sensitive heartbeat, cron, and post-compaction prompt bodies while retaining necessary date grounding.

## Best Possible Solution

Update the owner call sites so exact wall-clock time is late-bound, conditional, or available through tools/status where possible, while preserving stable timezone metadata and date-dependent startup semantics.

## Reproduction Assessment

Yes. Source inspection and committed tests show the named heartbeat, isolated cron, and post-compaction flows still include a freshly formatted `Current time:` line in the actual agent turn payloads.

## Solution Assessment

Yes. The requested direction is the narrow maintainable fix because the system-prompt cache boundary already exists; the remaining work belongs in the per-turn payload builders rather than broad prompt-cache machinery.
