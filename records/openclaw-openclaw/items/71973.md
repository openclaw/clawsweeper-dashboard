## Summary

Keep open: current main still appends exact wall-clock `Current time` / `Reference UTC` text into heartbeat, isolated cron, session-reset, and post-compaction prompt bodies, while the merged stale-time refresh work only makes that timestamp fresher rather than moving it out of cache-sensitive model-bound content.

## What This Changes

The issue requests moving volatile exact current-time prompt text out of cache-sensitive prompt prefixes for recurring automation and context-refresh turns.

## Best Possible Solution

Define one model-visible timestamp and cache-boundary contract, then update the shared current-time helper plus heartbeat, cron, reset, post-compaction callers and regression tests together.

## Reproduction Assessment

Yes, at source level. Current main still routes exact `Current time` / `Reference UTC` blocks into heartbeat, cron, reset, and post-compaction model-bound prompt text; I did not run a live provider cache experiment.

## Solution Assessment

Unclear. Moving exact time later is plausible, but the best fix needs a maintainer decision about whether exact time belongs in late runtime metadata, user message text, `session_status`, or a separate trigger/current timestamp contract.
