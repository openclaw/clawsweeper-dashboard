## Summary

Keep open. Current main still injects fresh exact wall-clock time into heartbeat, isolated cron, session reset, and post-compaction prompt bodies; the merged Reference UTC change only split the time format, and the related open cron cache-affinity issue tracks a broader separate problem.

## What This Changes

The issue requests moving volatile exact current-time text out of cache-sensitive prompt prefixes while keeping stable timezone context available.

## Best Possible Solution

Move or omit exact wall-clock text from the affected per-turn prompt builders unless that turn truly needs it, while keeping timezone-only system context and `session_status` as the exact-time path.

## Reproduction Assessment

Yes. Source inspection on current main shows the helper formats fresh exact time and the heartbeat, isolated cron, and post-compaction builders append it into turn bodies; committed tests still assert those lines.

## Solution Assessment

Yes. Targeting the per-turn prompt builders and shared helper contract is the narrowest maintainable direction; provider cache machinery and the broader cross-run cache-affinity issue can stay separate.
