## Summary

Keep open. Current main still matches the report: exact wall-clock `Current time:` text is generated and inserted into heartbeat, isolated cron, post-compaction refresh, and session reset prompt paths, while the stable system prompt only carries timezone metadata. The related #62809 is broader cache-affinity work and does not supersede this narrower timestamp-placement issue.

## What This Changes

The issue requests moving exact wall-clock current-time text out of cache-sensitive prompt prefixes while keeping stable timezone context.

## Best Possible Solution

Make a provider-aware prompt composition change that preserves stable timezone metadata in cacheable system context, keeps exact wall-clock time only on turns that require it, and places volatile timestamps below provider cache checkpoints or in late-bound event/runtime content. Regression coverage should cover heartbeat, isolated cron, post-compaction refresh, session reset, and Anthropic/OpenAI-family payload shaping.

## Reproduction Assessment

Yes. A read-only source check on current main follows the reported helper and finds `Current time:` injected into heartbeat, isolated cron, post-compaction refresh, and session reset paths, with tests asserting those lines.

## Solution Assessment

Unclear for a mechanical fix. The requested direction is likely right, but deciding which runtime turns still need exact time and where it belongs relative to provider cache boundaries needs prompt/cache owner design.
