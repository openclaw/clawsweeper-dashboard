## Summary

Keep open: current main still injects exact per-run wall-clock text into heartbeat, isolated cron, reset, post-compaction, and memory-flush prompt bodies, while related open items cover broader cache affinity or stale timestamp refresh rather than this volatile-prefix cleanup.

## What This Changes

Move exact per-turn current-time text out of cache-sensitive prompt prefixes while preserving stable timezone and date-awareness behavior.

## Best Possible Solution

Move exact wall-clock values to `session_status` or another late, turn-specific surface only where needed, keep timezone-only system context cacheable, and leave broader cross-run cache affinity to https://github.com/openclaw/openclaw/issues/62809.

## Reproduction Assessment

Yes for source-level reproduction. Current main still formats fresh exact time and injects or preserves it in named heartbeat, cron, reset, post-compaction, and memory-flush prompt builders; I did not run live provider cache traces in this read-only review.

## Solution Assessment

Yes. The narrow maintainable direction is to adjust the shared current-time contract and named prompt builders so exact time is late or on-demand, while preserving timezone/date context and not folding in the broader cache-affinity work.
