## Summary

Keep open: current main and latest release still append exact `Current time` / `Reference UTC` text into heartbeat, cron, session-reset, and post-compaction prompt bodies. The merged UTC-split work changed formatting, not cache-sensitive placement, and I did not find a canonical merged or open duplicate that owns this exact cleanup.

## What This Changes

Move volatile exact current-time injection out of cache-sensitive heartbeat, cron, and post-compaction prompt prefixes while preserving stable timezone context.

## Best Possible Solution

Define one timestamp/cache-boundary contract, then update the shared current-time helper and heartbeat, cron, reset, memory, and post-compaction callers/tests so exact wall-clock time is late-bound only where needed.

## Reproduction Assessment

Yes. Source inspection on current main shows exact `Current time` / `Reference UTC` text still reaches heartbeat, cron, reset, and post-compaction prompt bodies; I did not perform a live prompt-cache hit experiment.

## Solution Assessment

Unclear. The direction is plausible, but the best fix depends on a maintainer decision about whether exact time belongs in late runtime metadata, the user message, `session_status`, or separate trigger/current timestamp fields.
