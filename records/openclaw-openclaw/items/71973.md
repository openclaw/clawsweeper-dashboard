## Summary

Keep open. Current main still appends a fresh `Current time:` line into heartbeat payloads, isolated cron `commandBody`, and post-compaction refresh content; the already-shipped system-prompt cache-boundary work only fixes the stable system prompt layer, and the related #62809 remains the broader cache-affinity item rather than a superseding fix.

## What This Changes

Move exact per-turn `Current time:` wall-clock injection out of cache-sensitive heartbeat, cron, and post-compaction prompt prefixes while preserving stable timezone/date context.

## Best Possible Solution

Keep the timezone-only system prompt contract, then update the owner call sites so exact wall-clock time is omitted, delayed, date-only, or tool-provided unless the specific turn truly requires it.

Yes. Source inspection of current main and committed tests gives a high-confidence reproduction path: build heartbeat, isolated cron, or post-compaction payloads and observe the exact `Current time:` line in the agent turn content.

## Solution Assessment

Yes. The requested direction is the narrow maintainable fix because current main already has a system-prompt cache boundary; changing the per-turn payload builders is safer than reworking provider cache machinery or the broader #62809 cache-affinity work.
