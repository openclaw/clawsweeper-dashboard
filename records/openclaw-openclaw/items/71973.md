## Summary

Keep open: current main and the latest release still inject exact wall-clock `Current time` / `Reference UTC` text into several automation and refresh prompt bodies. The related cron cache-affinity issue and stale-timestamp PR cover adjacent problems, not this volatile prompt-body placement cleanup.

## What This Changes

Request to keep exact wall-clock current-time text out of cache-sensitive heartbeat, cron, reset, post-compaction, and memory-flush prompt bodies while retaining stable timezone context.

## Best Possible Solution

Move exact wall-clock values to `session_status` or another late, turn-specific surface only where required, keep timezone-only system context cacheable, and leave broader cron cache affinity to https://github.com/openclaw/openclaw/issues/62809.

## Reproduction Assessment

Yes, at source level. The helper formats fresh exact time from `nowMs`, and heartbeat, isolated cron, reset, post-compaction, and memory-flush builders still append or preserve that text while tests assert the behavior.

## Solution Assessment

Yes. The narrow maintainable direction is to change the shared helper contract and named prompt builders so exact time is late or on-demand, while preserving timezone context and not treating the broader cron cache-affinity issue as the same fix.
