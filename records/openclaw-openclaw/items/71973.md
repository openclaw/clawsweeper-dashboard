## Summary

Keep open: current main still injects exact per-run wall-clock text into several automation and refresh prompt bodies, while the existing cache-stable system prompt only carries timezone metadata and points agents to `session_status`. The related cache-affinity issue and stale-timestamp PR cover adjacent problems, not this volatile-prefix cleanup.

## What This Changes

The issue requests moving exact `Current time` injection out of cache-sensitive heartbeat, cron, post-compaction, reset, and memory-flush prompt content while preserving stable timezone awareness.

## Best Possible Solution

Move exact wall-clock values to `session_status` or another late, turn-specific surface only where required, keep timezone-only system context cacheable, and leave cross-run provider affinity to https://github.com/openclaw/openclaw/issues/62809.

## Reproduction Assessment

Yes. Source inspection on current main shows the helper formats fresh exact time and heartbeat, isolated cron, reset, post-compaction, and memory-flush builders still append or preserve that text; current tests also assert those lines.

## Solution Assessment

Yes. The narrow maintainable direction is to change the shared helper contract and named prompt builders so exact time is late or on-demand, while preserving stable timezone context and not treating the broader cron cache-affinity issue as the same fix.
