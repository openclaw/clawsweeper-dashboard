## Summary

Keep open. Current main still appends fresh exact wall-clock text into heartbeat, isolated cron, session reset, and post-compaction prompt bodies, while the merged Reference UTC change only changed formatting and the broader cron cache-affinity issue remains separate.

## What This Changes

The issue requests moving exact wall-clock time out of cache-sensitive heartbeat, cron, and post-compaction prompt prefixes while retaining stable timezone context.

## Best Possible Solution

Keep the stable timezone-only system prompt, stop placing fresh exact wall-clock lines in early heartbeat/cron/post-compaction bodies, and preserve exact time through session_status or late turn-specific text when required.

## Reproduction Assessment

Yes. Source inspection on current main shows the helper formats fresh exact time and the heartbeat, isolated cron, session reset, and post-compaction builders still append it into turn bodies; committed tests still assert those lines.

## Solution Assessment

Yes. Targeting the per-turn prompt builders and shared helper contract is the narrowest maintainable direction; provider cache-key affinity can stay in the separate open cron-cache issue.
