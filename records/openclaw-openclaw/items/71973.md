## Summary

Keep open. Current main still appends exact wall-clock `Current time:` lines into heartbeat, isolated cron, post-compaction refresh, and bare reset prompt bodies; related #62809 and the current-time formatting PRs are adjacent, but they do not resolve this cache-placement contract.

## What This Changes

Request moves exact wall-clock current-time injection out of cache-sensitive heartbeat, cron, and post-compaction prompt prefixes while keeping stable timezone metadata.

## Best Possible Solution

Define a provider-aware prompt timestamp contract that keeps exact wall-clock time below cache checkpoints or in late-bound runtime/event content only when needed, while preserving a non-cache-busting way for startup and daily-memory flows to obtain the current date.

## Reproduction Assessment

Yes. Source inspection on current main deterministically follows `resolveCronStyleNow()` into heartbeat, isolated cron, post-compaction refresh, and bare reset prompt paths, and existing focused tests assert `Current time:` in those prompts; tests were not run because this review was read-only.

## Solution Assessment

Unclear for a mechanical fix. The requested direction is likely right for cache efficiency, but the maintainable solution needs prompt/cache owner judgment about which runtime turns still require exact time and where each provider can receive volatile content without reducing prefix-cache reuse.
