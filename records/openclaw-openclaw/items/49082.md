## Summary

Close as implemented on main. The user-visible Feishu local-media reply failure is now handled by shared reply-media normalization and managed-media staging before the Feishu dispatcher sends attachments, and that shipped in v2026.4.7.

## What This Changes

The PR threads agent-scoped `mediaLocalRoots` from Feishu bot dispatcher construction into `reply-dispatcher.ts` `sendMediaFeishu` calls.

## Best Possible Solution

Keep the shipped shared media-staging implementation and close this Feishu-specific PR rather than merging another root-threading path.

## Reproduction Assessment

No on current main. Source inspection and regression coverage show workspace-local reply media is now staged into managed outbound media before Feishu delivery, so the original raw-path fallback path is no longer the expected behavior.

## Solution Assessment

Yes. The current main solution is better than a Feishu-only parameter pass because it centralizes local reply-media normalization before all channel dispatchers while preserving the post-CVE local-file guardrails.
