## Summary

Close as implemented on main. The PR’s Feishu-specific mediaLocalRoots plumbing is no longer needed for the observable user problem because current main normalizes and stages local reply media through the shared reply-media path before the Feishu dispatcher sends it, and that shipped in v2026.4.15.

## What This Changes

The PR adds mediaLocalRoots to the Feishu reply dispatcher parameters, derives agent-scoped roots at both Feishu bot dispatcher construction paths, and forwards those roots to sendMediaFeishu.

## Best Possible Solution

Keep the shipped shared reply-media staging path as the product direction and close this Feishu-specific PR as redundant; overlapping Feishu mediaLocalRoots PRs should be reviewed against the same current-main behavior before adding channel-local root plumbing.

## Reproduction Assessment

Yes. The original failure path is an agent reply carrying a local media path to Feishu; current main routes that reply through dispatchReplyFromConfig, which normalizes and stages local media with agent-scoped access before Feishu delivery, and focused tests cover the staging and blocked-host cases.

## Solution Assessment

Yes. The current shared normalizer is the better solution because it centralizes local-file validation and converts local reply paths into managed media before channel dispatch, which is narrower and easier to audit than passing filesystem roots through each channel dispatcher.
