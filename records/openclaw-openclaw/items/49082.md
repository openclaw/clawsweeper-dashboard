The PR threads agent-scoped media roots from the Feishu bot into the reply dispatcher and forwards them to `sendMediaFeishu`.

## Best Possible Solution

Keep the shipped shared reply-media staging design and close this Feishu-specific dispatcher-plumbing PR as redundant; any remaining account-level Feishu media-root semantics should be tracked separately.

## Reproduction Assessment

No failing current-main reproduction remains. The original Feishu auto-reply local-media path is covered by current reply-media staging tests, and Feishu production replies run through that normalizer before channel delivery.

## Solution Assessment

No for merging this PR now. The PR's root-threading approach was reasonable for the older dispatcher gap, but current main's shared normalizer is the narrower maintainable solution because local-file validation and staging happen once before channel-specific senders run.
