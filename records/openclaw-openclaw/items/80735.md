## Summary

Close as duplicate/superseded: the same top-level MCP SecretRef gap is already tracked by https://github.com/openclaw/openclaw/issues/76493, and the open implementation PR https://github.com/openclaw/openclaw/pull/69417 covers the requested env/header schema, runtime resolution, target registry, secrets audit/configure behavior, docs, and tests.

## What This Changes

Support SecretRef values in `mcp.servers.*.env.*` and include that path in secrets audit so MCP credentials do not have to stay as plaintext in `openclaw.json`.

## Best Possible Solution

Keep https://github.com/openclaw/openclaw/issues/76493 and https://github.com/openclaw/openclaw/pull/69417 as the single canonical path for schema, runtime, audit/configure, docs, and test coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows the schema and public type still accept only scalar MCP env values, the runtime drops object env values, and the supported SecretRef surface does not list top-level `mcp.servers.*.env.*`.

## Solution Assessment

Yes. Closing this duplicate is the best cleanup path because the remaining work is already owned by the older canonical issue and the open implementation PR that covers the audit behavior as well as SecretRef resolution.
