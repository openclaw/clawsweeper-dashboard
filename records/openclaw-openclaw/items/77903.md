## Summary

Keep open: the reported warning is source-reproducible on current main, but the proposed patch only teaches the helper about predeclared plugin tools and does not feed manifest tool names into production policy calls.

## What This Changes

The PR adds `knownPluginToolNames` to `applyToolPolicyPipeline` and unit tests that verify declared plugin tool names avoid unknown-allowlist warnings while true typos still warn.

## Best Possible Solution

Thread enabled plugin manifest `contracts.tools` into every real policy-pipeline path, suppress only declared plugin-tool warnings while preserving typo warnings, add a changelog entry, and attach real gateway/plugin proof or a maintainer proof override.

## Reproduction Assessment

Yes, at source level. Current main classifies allowlist entries as unknown when they are absent from `pluginGroups.all`, and `llm-task` is a manifest-declared optional plugin tool that docs tell users to allow with `tools.alsoAllow`.

## Solution Assessment

No. The seam is plausible, but without wiring manifest-declared tool names into the real callers, the gateway/agent warning path still sees only the currently registered tool catalog.
