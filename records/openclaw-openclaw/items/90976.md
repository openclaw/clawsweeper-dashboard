## Summary

Keep this PR open, but not merge-ready yet: it fixes a real background-session name-derivation ReDoS vector, while the proposed single-quote branch changes labels for a valid shell case where a single-quoted argument with spaces ends in a literal backslash.

## What This Changes

The PR changes `tokenizeCommand` in background-session label derivation to remove ambiguous quoted-span regex matching and adds `deriveSessionName` regression tests.

## Best Possible Solution

Keep the ReDoS hardening, but preserve shell single-quote semantics by treating backslash as literal inside the single-quoted branch and adding a regression test for that case before merge.

## Reproduction Assessment

Yes. Current source shows the ReDoS-prone regex in the reachable background-session label path, and a local probe reproduced the introduced single-quote label regression against the proposed regex.

## Solution Assessment

No. The one-line mitigation is the right layer for the ReDoS, but the single-quoted alternative should use shell semantics such as `'[^']*'` instead of making backslash escape the closing single quote.
