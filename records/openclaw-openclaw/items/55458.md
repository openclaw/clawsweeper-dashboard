## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve explicit OpenAI provider selections:** `scripts/sync-runtime-config.mjs:164`
  - body: `availableProviders` skips `openai` after `openai-codex` consumes `OPENAI_API_KEY`, so an explicit `openai/...` primary model is treated as unavailable and rewritten to Codex on startup. That violates the script's stated preserve-user-overrides behavior.
  - confidence: 0.91
- **[P1] Restore the non-root runtime user:** `Dockerfile:262`
  - body: The Dockerfile removes the existing `USER node` hardening and adds a root entrypoint without switching back, so the gateway and entrypoint run as root. That broadens the impact of any gateway, plugin, or dependency compromise.
  - confidence: 0.9
- **[P2] Fix Docker GPG awk quoting:** `Dockerfile:238`
  - body: The awk program is single-quoted, so the embedded escaped quotes are passed literally and can cause an awk parse error when `OPENCLAW_INSTALL_DOCKER_CLI` is enabled. That aborts the Docker build before key fingerprint verification completes.
  - confidence: 0.87
- **[P3] Keep empty-message probe failures visible:** `src/commands/status.scan.ts:183`
  - body: The conditional spread drops `probeError` when the probe throws `new Error("")`, so that failure remains hidden. Preserve the field based on presence, and render based on presence, so every thrown probe failure is visible.
  - confidence: 0.78
