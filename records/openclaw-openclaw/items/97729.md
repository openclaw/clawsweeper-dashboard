Overall correctness: patch is incorrect

Full review comments:

- **[P1] Preserve exec-approval commands before preflighting:** `src/gateway/server-methods/exec-approvals.ts:124`
  - body: This guard checks `nodeSession.commands`, but current main stores the effective command list after `resolveNodePairingCommandAllowlist`, and the desktop host allowlist does not include `system.execApprovals.get/set`. A node that actually implements the dedicated approvals RPCs can therefore be rejected before `nodeRegistry.invoke`; preserve those commands in the effective surface or gate on a reliable declared-capability source first.
  - confidence: 0.92
- **[P2] Keep missing sessions on the invoke error path:** `src/gateway/server-methods/exec-approvals.ts:116-123`
  - body: When `nodeRegistry.get` returns empty, this early return bypasses the current `NodeRegistry.invoke` error mapping, so JSON callers lose the `details.nodeError` payload for `NOT_CONNECTED`. The reported bug is the connected unsupported-command case; absent or cold nodes should continue through the existing invoke/error path.
  - confidence: 0.86
