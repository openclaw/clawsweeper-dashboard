## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Gate exec-approval invokes on approved commands:** `src/gateway/server-methods/exec-approvals.ts:116`
  - body: This checks `declaredCommands`, but that field includes a node's newly reported command surface before pairing reapproval is accepted, while `commands` is the effective approved surface. A paired node that starts declaring `system.execApprovals.get/set` can therefore pass this guard and receive read/write approval-policy RPCs before that command upgrade is approved; use the approved command surface after fixing the policy defaults, and add a pending-reapproval regression test.
  - confidence: 0.92
- **[P1] Add exec approvals to the pairing allowlist:** `src/gateway/node-command-policy.ts:65`
  - body: Adding the commands only to `DESKTOP_HOST_COMMANDS` is not enough for first-time desktop node pairing: `resolveNodePairingCommandAllowlist` starts from `SYSTEM_COMMANDS`/platform defaults, and this patch leaves those unchanged. Capable desktop nodes can still have exec-approval commands filtered out before `declaredCommands` is stored, so the new preflight may still reject capable nodes; include the commands in the desktop pairing defaults and cover reconciliation in tests.
  - confidence: 0.9
