Full review comments:

- **[P1] Gate exec-approval invokes on approved commands:** `src/gateway/server-methods/exec-approvals.ts:116`
  - body: This checks nodeSession.declaredCommands, but registered sessions keep declaredCommands as the node's reported pre-approval surface while commands is the effective approved surface. A paired node that newly declares system.execApprovals.get/set can pass this guard and receive policy read/write RPCs before the command upgrade is approved; use the approved surface while preserving the unknown-node fall-through.
  - confidence: 0.92
- **[P1] Add exec approvals to the pairing defaults:** `src/gateway/node-command-policy.ts:65`
  - body: Adding the commands only to DESKTOP_HOST_COMMANDS makes previously approved exec-approval commands eligible at runtime, but resolveNodePairingCommandAllowlist still starts from PLATFORM_DEFAULTS/SYSTEM_COMMANDS and those do not include NODE_EXEC_APPROVALS_COMMANDS. First-time desktop node pairing can still filter the commands out before declaredCommands is stored, so capable nodes may be rejected by the new preflight.
  - confidence: 0.9
