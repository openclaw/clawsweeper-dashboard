## What This Changes

## Best Possible Solution

Land the shared owner-match guard after normal required checks so owner-qualified installs cannot cross-owner overwrite tracked skill state.

## Reproduction Assessment

Yes, at source level with supporting live CLI proof from the contributor. Current main can parse @bob/weather to weather, let --force bypass the existing-target check, and then write Bob's ownerHandle over Alice's origin/lock state after replacement.

## Solution Assessment

Yes. Reusing the update-path owner invariant in the ClawHub lifecycle is the narrowest maintainable fix; the CLI and archive installer are weaker layers because owner tracking is owned by the ClawHub lifecycle.
