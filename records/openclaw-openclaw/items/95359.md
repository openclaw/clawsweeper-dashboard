## Summary

Keep this PR open with changes requested: the known-owner overwrite fix is well proven, but the latest head still permits owner-qualified `--force` installs over legacy ownerless ClawHub tracking and leaves the documented `--force` overwrite contract out of sync.

## What This Changes

The PR adds a shared ClawHub owner-match guard for install/update/verify and regression coverage for refusing owner-qualified forced installs over a skill tracked to a different owner.

## Best Possible Solution

Keep the shared owner guard, make owner-qualified installs treat missing tracked owner metadata as a mismatch unless maintainers explicitly choose and document the legacy overwrite exception, and update the skills docs to describe the new force-install contract.

## Reproduction Assessment

Yes. The current-main source path is clear, and the contributor provided live before/after CLI output for the known-owner Alice/Bob overwrite case; I did not run tests in this read-only review.

## Solution Assessment

No, not yet. The shared helper is the right boundary, but install should not remain weaker than update/verify for ownerless tracked state, and the user-facing force-install docs need to match the changed behavior.
