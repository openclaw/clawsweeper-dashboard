## Summary

Keep open for maintainer review. Current main and the latest release still allow an owner-qualified `skills install --force` to flow into the bare local slug install path without the update path's owner-mismatch guard, while this PR adds the guard at the shared lifecycle boundary and covers it with a focused regression test plus live CLI proof.

## What This Changes

The branch adds a shared ClawHub owner-match guard to the skills lifecycle install/update paths and a regression test for owner-qualified cross-owner force installs.

## Best Possible Solution

Land the lifecycle owner-match guard once maintainers accept that owner-qualified installs must not replace a skill tracked under a different owner, even with `--force`.

Yes, source-reproducible. Current main parses `@bob/weather` to local slug `weather`, lets `--force` bypass the existing-target refusal, and then writes requested owner metadata; the PR body also includes live before/after CLI output for that path.

## Solution Assessment
