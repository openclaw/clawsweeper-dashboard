## What This Changes

This PR factors the ClawHub owner-mismatch check into a shared helper, applies it to owner-qualified installs before overwrite, and adds a regression test for forced cross-owner installs.

## Best Possible Solution

Land the shared lifecycle guard once maintainers accept the fail-closed cross-owner `--force` behavior and required checks are green.

## Solution Assessment

Yes. The lifecycle layer owns ClawHub origin and lock tracking, so refusing before fetch/download/install is narrower and cleaner than adding caller-specific checks in the CLI, Gateway, or archive installer.
