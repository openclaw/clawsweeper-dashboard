## What This Changes

The PR adds a shared ClawHub owner-match guard for install/update lifecycle paths and a regression test for refusing owner-qualified forced installs over a differently owned tracked skill.

## Best Possible Solution

Land the lifecycle owner-match guard after maintainers accept the fail-closed cross-owner `--force` behavior and required checks are green.

## Reproduction Assessment

Yes. Source inspection shows current main parses `@bob/weather` to `weather`, lets `--force` bypass the existing-target check, and then can rewrite origin/lock owner tracking; the PR also provides live before/after CLI output for the same path.

## Solution Assessment

Yes. The lifecycle layer owns ClawHub origin and lock tracking, so refusing before fetch/download/install is narrower and cleaner than adding caller-specific checks in the CLI, Gateway, or archive installer.
