## Summary

Keep this PR open for maintainer review: current main still allows the destructive cross-owner `--force` path, while the patch adds the guard at the ClawHub lifecycle boundary with focused regression coverage and sufficient live CLI proof.

## What This Changes

The PR adds a shared ClawHub owner-match guard for owner-qualified skill installs and updates, plus a regression test for forced installs over a skill tracked under another owner.

## Best Possible Solution

Land the lifecycle-level owner-match guard after maintainer acceptance and required checks so cross-owner ClawHub installs cannot silently overwrite tracked skill state.

## Reproduction Assessment

Yes. Source inspection shows current main can parse `@bob/weather` to the `weather` target, let `--force` bypass the existing-target check, and then rewrite origin/lock tracking; the contributor also provided live before/after CLI output.

## Solution Assessment

Yes. Reusing the existing update-path owner invariant inside the ClawHub lifecycle is the narrowest maintainable fix; the CLI, gateway, and archive installer are weaker layers because owner tracking is owned by ClawHub lifecycle state.
