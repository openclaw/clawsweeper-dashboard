## Review Findings

Overall correctness: not a patch

Overall confidence: 0.87

Full review comments:

- **[P1] Move the hook before route-owned side effects:** `src/channels/turn/kernel.ts:459-463`
  - body: This hook is advertised as pre-routing, but the call is inserted after `adapter.resolveTurn(...)` has already produced the route session and after some channel adapters have done route-specific work before entering the kernel. A redirect/suppress result therefore cannot reliably prevent the original route’s side effects; move the seam earlier in the route/preflight contract or narrow the public hook name and docs to post-resolution behavior.
  - confidence: 0.88
- **[P1] Add the default fail-open timeout:** `src/plugins/hooks.ts:989-1006`
  - body: `runClaimingHook` only gets a default budget when the hook name is in the modifying-hook timeout map, and this PR does not add one for the new hook. A slow or never-settling routing plugin can block inbound message handling indefinitely unless every plugin author sets `timeoutMs`, which violates the fail-open contract discussed for this hook.
  - confidence: 0.91
