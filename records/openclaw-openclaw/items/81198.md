## Best Possible Solution

  - body: The new import names are not exported by `hook-message.types.ts`, the hook name is not registered in `PluginHookName`/`PluginHookHandlerMap`, and the runner has no `runBeforeRouteInboundMessage` method. This fails typechecking and leaves plugin authors unable to register the hook.

  - confidence: 0.95
