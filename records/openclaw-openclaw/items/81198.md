## Summary

Keep open. Current main does not implement the requested hook, and this active feature PR still has a source-proven compatibility-path redirect bug, mock-only proof, and an unresolved trust-boundary decision for plugin-controlled redirect/suppression.

## What This Changes

The branch adds a `before_route_inbound_message` hook to plugin hook types and runner APIs, wires it into channel turn and compatibility dispatch paths, and adds kernel tests for redirect and suppression behavior.

## Best Possible Solution

Define the hook as a documented SDK contract with one consistent routing point, keep `routeSessionKey` and `ctxPayload.SessionKey` synchronized across modern and compatibility paths, add coverage for both paths, and land only with real bridge proof plus maintainer-approved scoping.

## Reproduction Assessment

Yes for the PR blocker: source inspection shows the compatibility path updates only `params.route.sessionKey`, while current recording uses `ctxPayload.SessionKey` first. For the linked user request itself, reproduction is not a bug reproduction because this is a new plugin API capability.

## Solution Assessment

No. A pre-routing hook may be the right direction, but this patch is not yet the maintainable solution until compatibility redirects update the actual recorded payload, coverage proves that path, and maintainers approve the redirect/suppression trust model.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep the compatibility payload session key in sync:** `src/plugin-sdk/inbound-reply-dispatch.ts:197-200`
  - body: `recordChannelMessageReplyDispatch` ultimately records with `ctxPayload.SessionKey ?? routeSessionKey`, but this redirect block only replaces `params.route.sessionKey`. When compatibility callers pass a finalized context that already has `SessionKey`, the hook still records and dispatches the original session, so the channel-bridge redirect does not actually take effect on this path.

  - confidence: 0.86
