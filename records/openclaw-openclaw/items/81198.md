## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Bound the pre-routing hook before awaiting it:** `src/plugins/hooks.ts:1001-1008`
  - body: `runBeforeRouteInboundMessage` delegates to the generic claiming hook, but this new hook is not given a default timeout. Because the channel turn awaits it before record/dispatch, any handler without its own `timeoutMs` can stall inbound delivery instead of failing open.
  - confidence: 0.88
- **[P2] Keep compatibility redirects in sync with payload sessions:** `src/plugin-sdk/inbound-reply-dispatch.ts:197-200`
  - body: This redirect path only replaces `params.route.sessionKey`, but current recording prefers `ctxPayload.SessionKey` when present. Compatibility callers carrying a finalized payload can still record and dispatch under the original session after a redirect.
  - confidence: 0.91
- **[P2] Document the public routing hook contract:** `src/plugins/hook-types.ts:111`
  - body: Adding this hook name exposes a public plugin API, but the hook docs still list only the existing message hooks. The PR should document timing, redirect/suppress semantics, timeout/error behavior, and trust boundaries before plugin authors depend on it.
  - confidence: 0.82
- **[P3] Assert redirects reach downstream session use:** `src/channels/turn/kernel.test.ts:953-956`
  - body: The redirect test proves the hook was called and the turn dispatched, but it never asserts the recorded session, returned route key, or payload session key. Add a downstream session assertion so the test fails when a redirect is ignored after the hook returns.
  - confidence: 0.84
