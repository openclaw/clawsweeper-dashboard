## Summary

Keep open: this PR targets a real open feature request at https://github.com/openclaw/openclaw/issues/81061, but the patch is incomplete, currently fails type/lint checks, has only mock/unit proof, and adds security-sensitive cross-session redirect/suppression semantics that need maintainer API review.

## What This Changes

The branch adds a pre-routing inbound-message hook call in the channel turn kernel plus mocked redirect and suppressDelivery tests.

## Best Possible Solution

Define a complete additive SDK contract with hook types, runner wiring, docs, API baselines, focused channel-turn/plugin tests, and explicit redirect/suppression guardrails before merging this capability.

## Reproduction Assessment

Yes for the PR blockers: source inspection and CI reproduce the missing hook contract, missing runner method, and turn-contract type errors. For the linked user need, this is a new plugin API capability request rather than a broken documented behavior.

## Solution Assessment

No. A pre-routing hook may be a reasonable direction, but this patch only inserts a kernel call and mocked tests without the public SDK contract, runner registration, real behavior proof, or security guardrails needed for a routing API.

  - body: The production change imports PluginHookBeforeRouteInboundMessage* types and calls runBeforeRouteInboundMessage, but the PR does not add those exports, the hook-name registry entry, the handler-map entry, or the runner method. After the compile errors are fixed, redirects with an existing resolved payload would still use the original session unless resolved.ctxPayload.SessionKey is updated.
  - confidence: 0.9

  - confidence: 0.94
