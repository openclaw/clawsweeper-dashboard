## Summary

## What This Changes

The PR branch adds a channel-turn kernel call and tests for a new `before_route_inbound_message` hook that redirects or suppresses inbound messages.

## Best Possible Solution

Carry the full hook contract through `src/plugins/*`, SDK exports, docs, and tests, then wire it at the correct resolved-turn boundary with validated redirect/suppress semantics and real channel-bridge proof.

## Reproduction Assessment

Not applicable: this PR proposes a new plugin API capability rather than fixing existing documented behavior. The current branch failure is source/CI-reproducible because the PR does not typecheck.

## Solution Assessment

No: the current patch is not mergeable because it dropped the hook registry/type/runner changes and reads fields that are not on the generic turn params. A safer fix is to design the hook contract first, then wire it through resolved turn data with security and audit rules.
