## Summary

Keep open. Current main and the latest release still encode Discord bot-owned autoThread and bound-thread session paths that can bypass configured mention requirements, while the newer related issues are open coordination points rather than merged or shipped fixes.

## What This Changes

The issue asks Discord thread follow-up messages to honor `requireMention` instead of staying active after an initial mention or bot-created thread.

## Best Possible Solution

Define an additive Discord thread-reply mention policy, wire it through config/schema/preflight/docs/tests, and coordinate it with the broader strict implicit-mention design so existing thread continuity remains compatible by default.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create a bot-owned autoThread or active bound Discord thread/session with `requireMention: true`, then send an unmentioned follow-up; current preflight can clear `shouldRequireMention` before the no-mention drop branch.

## Solution Assessment

Unclear. A strict per-message rule would address the report, but the durable fix needs maintainer agreement on Discord thread-reply defaults and whether the config should align with the broader strict implicit-mention policy.
