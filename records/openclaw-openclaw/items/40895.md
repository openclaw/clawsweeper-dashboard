## Summary

Keep open. Current main still lowers Discord mention requirements for bot-owned autoThread threads and active bound thread sessions before the no-mention drop path, and the overlapping related issues have not replaced this report with a terminal fix.

## What This Changes

The issue asks Discord thread replies to honor `requireMention: true` per message, or to add an explicit thread-specific mention policy.

## Best Possible Solution

Adopt an explicit, documented Discord thread-reply mention policy, likely mirroring the Slack explicit-thread-mention seam, then update Discord preflight, config schema/types/metadata, docs, changelog, and regression tests together.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: with `autoThread: true`, bot-owned thread ownership makes `resolveDiscordShouldRequireMention` return false, and with an active bound thread session preflight clears the requirement before the no-mention drop branch.

## Solution Assessment

Unclear. A strict per-message rule would stop the unwanted replies, but current tests encode continuity for bot-owned and bound threads, so the safest solution needs an explicit Discord policy/config decision rather than an unannounced default flip.
