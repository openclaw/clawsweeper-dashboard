## Summary

Keep open. Current main still has the Discord thread paths that can bypass configured mention requirements, and the overlapping follow-up issues are still open rather than merged or shipped.

## What This Changes

The issue asks Discord thread replies to honor `requireMention` per message, or expose an explicit thread-specific mention policy.

## Best Possible Solution

Define a Discord-owned explicit thread-reply mention policy, document the default, wire it through config/schema/preflight before model invocation, and keep compatibility with existing thread continuity unless maintainers choose a default change.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: bot-owned autoThread threads and active bound thread sessions clear `shouldRequireMention` before the no-mention drop branch.

## Solution Assessment

Unclear. A strict per-message rule would address the report, but the maintainable fix needs a Discord policy/config decision because current tests encode implicit thread continuity and Slack uses an explicit opt-in option.
