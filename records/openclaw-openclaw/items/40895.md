## Summary

Keep open. Current main still bypasses Discord mention requirements for bot-owned autoThread threads and active bound thread sessions, and the related thread-mention follow-ups are still open rather than merged or shipped.

## What This Changes

Discord thread replies should honor `requireMention` instead of treating an initial mention or bot-owned thread as persistent activation.

## Best Possible Solution

Define and document the Discord thread-reply mention policy, likely by adding a Discord-specific explicit-thread-mention option that uses the existing inbound implicit-mention allowlist seam, then update config schema, docs, changelog, and regression tests together.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: bot-owned autoThread threads and active bound thread sessions both clear the effective mention requirement before the no-mention drop branch.

## Solution Assessment

Unclear. A strict per-message rule would stop the unwanted replies, but the maintainable path needs a Discord policy/config decision because current tests preserve implicit thread continuity and Slack solved the analogous case with an explicit option.
