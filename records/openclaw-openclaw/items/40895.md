## Summary

Keep open. Current `main` and the latest release still lower Discord mention requirements for bot-owned autoThread threads and active bound thread sessions before the no-mention drop path, while related #52568 and #62066 track overlapping but not terminally identical policy work.

## What This Changes

Discord thread follow-up messages should honor `requireMention` instead of staying implicitly active after an initial mention, bot-owned autoThread thread, or bound thread session.

## Best Possible Solution

Define an explicit Discord thread-reply mention policy, coordinate naming and scope with #52568 and #62066, then update Discord preflight, autoThread ownership handling, bound-thread behavior, config types/schema/metadata, docs, changelog, and regression tests together.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: create a bot-owned autoThread or active bound Discord thread session with `requireMention: true`, send a follow-up without a mention, and current preflight resolves `shouldRequireMention` to false before the no-mention drop branch.

## Solution Assessment

Unclear. A strict per-message rule would stop the bypass, but current tests encode continuity for bot-owned and bound threads, so the maintainable fix needs an explicit documented Discord policy rather than a silent behavior flip.
