## Summary

Keep open. Current main still has source-reproducible Discord thread paths that bypass mention requirements, and the related thread-mention follow-up remains open rather than merged or shipped.

## What This Changes

The issue requests Discord thread messages to honor `requireMention` per message, or gain a thread-specific mention setting so follow-up thread replies do not trigger unexpectedly.

## Best Possible Solution

Define and document the Discord thread-reply mention policy, then implement it in the Discord plugin with aligned config schema, docs, changelog, and regression tests, using #52568 if maintainers want the explicit middle-ground mode.

## Reproduction Assessment

Yes. Source inspection on current main shows bot-owned autoThread threads return `false` from `resolveDiscordShouldRequireMention`, and bound thread sessions clear the effective mention requirement before the no-mention drop branch.

## Solution Assessment

Unclear. A strict per-message gate would address the report, but the safer maintainable path needs maintainer-approved Discord thread policy because current tests encode implicit thread continuity and Slack solved the analogous case with an explicit option.
