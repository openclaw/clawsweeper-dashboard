## Summary

Keep open. Current main still has the reported timeout path: plugin gateway handlers are awaited, but returned payloads are not converted into RPC responses, so a successful return-only handler can leave `openclaw gateway call` waiting until its timeout.

## What This Changes

Plugin gateway methods that complete successfully and return a value should send that value back to `openclaw gateway call` instead of timing out.

## Best Possible Solution

Add a backwards-compatible gateway method result adapter that preserves existing `respond(...)` handlers while sending a successful RPC payload when a plugin handler returns a JSON-compatible value without responding.

## Reproduction Assessment

Yes. Register a plugin gateway method that returns a payload without calling `respond(...)`, then call it through `openclaw gateway call`; current source awaits the handler but never emits a response frame.

## Solution Assessment

Yes. The narrow maintainable fix is in the gateway method dispatch/SDK contract, with docs and regression coverage, rather than changing auth, scopes, or timeout behavior.
