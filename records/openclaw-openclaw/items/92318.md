## Summary

## What This Changes

The PR adds an internal source-delivery plan flag and enables it for cron announce delivery so target-less message-tool sends no longer satisfy the configured cron target, with focused regression tests.

## Best Possible Solution

Land the explicit-target-evidence requirement at the cron source-delivery boundary after maintainers accept or prove the pending sessions_send duplicate-delivery edge.

## Reproduction Assessment

Yes. Current main and v2026.6.6 synthesize the planned target from didSendViaMessageTool=true with an empty target list, which can satisfy cron source delivery and skip fallback without explicit target proof.

## Solution Assessment

Yes, this is the best scoped fix. It adds an opt-in evidence requirement to the existing source-delivery plan and enables it only for cron announce, avoiding a broader change to legacy message-tool-owned flows.
