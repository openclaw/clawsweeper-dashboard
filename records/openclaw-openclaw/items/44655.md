## Summary

Keep open. Current main and the latest release still expose Z.AI through a manifest-backed static catalog plus GLM-5 forward-compat synthesis, but they do not fetch Z.AI/Zhipu `/models`; this is a provider-owned feature that needs endpoint and Coding Plan contract confirmation before implementation.

## What This Changes

Request dynamic Z.AI/Zhipu model discovery from a provider `/models` endpoint, with stable fallback rows and custom model-id support when discovery fails.

## Best Possible Solution

Add a provider-owned Z.AI live catalog/refresh path that uses guarded authenticated `/models` discovery, tolerant schema parsing, static manifest fallback, preserved GLM-5 forward compatibility, and updated Z.AI docs/tests.

## Reproduction Assessment

Not applicable as a bug reproduction. The absence is source-verifiable: current main uses static manifest rows plus GLM-5 synthesis and has no Z.AI `/models` fetch path.

## Solution Assessment

Unclear until provider semantics are confirmed. The maintainable direction is a Z.AI-owned live catalog/cache hook with static fallback, not a frontend-only dropdown change.
