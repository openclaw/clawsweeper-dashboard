## Summary

Keep open. Current main and the latest release still do not implement Z.AI `/models` discovery; they use a static manifest catalog, endpoint probing for a few defaults, and GLM-5 synthesis only, so this remains a provider feature and product/API decision rather than closeable cleanup.

## What This Changes

Request Z.AI/Zhipu provider model discovery from the provider `/models` endpoint, dynamic model picker population, and fallback/default or custom model behavior.

## Best Possible Solution

Add a Z.AI-owned live catalog path that authenticates against the selected region/plan, defensively parses the provider model response, falls back to the manifest catalog, preserves GLM-5 forward compatibility, and documents refresh/failure behavior.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection on current main verifies the requested live `/models` discovery path is absent: Z.AI is static manifest catalog plus GLM-5 forward-compatible synthesis.

## Solution Assessment

Unclear until the provider contract is confirmed. A frontend-only dropdown change is not the best boundary; the maintainable path is a provider-owned live catalog hook with authenticated fetch, static fallback, and documented failure behavior.
