## Summary

Keep open. Current main still uses manifest-backed Z.AI catalog rows, fixed chat-completions endpoint probes, and GLM-5 forward-compatible synthesis, but it does not register an authenticated Z.AI `/models` live catalog or populate the picker from that endpoint.

## What This Changes

Request dynamic Z.AI/Zhipu model discovery from authenticated `/models` endpoints, with stable fallback behavior for static or custom model IDs.

## Best Possible Solution

Add a Z.AI-owned `liveCatalog` through `registerModelCatalogProvider`, preserving the static manifest fallback and covering endpoint selection, parsing, caching, failure behavior, docs, and focused provider tests.

## Reproduction Assessment

No high-confidence bug reproduction is applicable because this is a feature request. Source inspection gives high confidence that current main still lacks Z.AI `/models` live catalog discovery.

## Solution Assessment

Unclear until maintainers confirm the Z.AI endpoint contract and policy for authenticated discovery, caching, and fallback. The maintainable path is provider-owned live catalog discovery through the plugin SDK seam, not a frontend-only dropdown patch.
