Keep open. Current main still has static manifest-backed Z.AI model rows, GLM-5 forward-compatible metadata synthesis, and chat-completions endpoint probes, but no authenticated Z.AI `/models` live catalog feeding the picker.

## What This Changes

The issue requests dynamic Z.AI/Zhipu model discovery from the vendor `/models` endpoint, with stable fallback rows and custom-model fallback when discovery fails.

## Best Possible Solution

Add a Z.AI-owned live catalog through `registerModelCatalogProvider`, preserving the static manifest fallback and covering endpoint selection, parsing, caching, failure behavior, docs, and focused provider tests.
