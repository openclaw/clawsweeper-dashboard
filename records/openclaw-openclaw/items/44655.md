## What This Changes

The issue requests dynamic Z.AI/Zhipu model discovery from provider `/models` endpoints instead of relying only on static bundled model rows.

## Best Possible Solution

Add Z.AI-owned live catalog discovery through the existing model catalog provider seam or explicit catalog refresh path, preserving the manifest fallback and covering endpoint selection, parsing, caching, failure behavior, docs, and focused Z.AI tests.

## Reproduction Assessment

No high-confidence bug reproduction is applicable because this is a feature request, not broken documented behavior. Source inspection gives high confidence that current main still lacks authenticated Z.AI `/models` live catalog discovery.

## Solution Assessment

Unclear until maintainers confirm the Z.AI endpoint contract and policy for authenticated discovery, caching, and fallback. The maintainable path is provider-owned catalog discovery through the plugin/model-catalog seam, not a frontend-only dropdown patch.
