## Summary

Keep open. Current main and v2026.5.12 still use a static Z.AI manifest catalog plus chat-completion endpoint probing and GLM-5 forward-compat synthesis; the requested authenticated `/models` discovery and dynamic picker behavior is still missing and needs provider/API review.

## What This Changes

The issue requests live Zhipu/Z.AI model discovery from the provider `/models` endpoint with fallback/custom-model behavior instead of relying on a static catalog.

## Best Possible Solution

Add a Z.AI-owned live catalog path that authenticates against the selected endpoint, defensively parses the provider response, merges or falls back to the manifest catalog, preserves GLM-5 forward compatibility, and documents refresh/failure behavior.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection on current main verifies the requested live `/models` discovery path is absent: Z.AI is static manifest catalog plus chat-completion endpoint probing and GLM-5 forward-compatible synthesis.

## Solution Assessment

Unclear until the provider contract is confirmed. A frontend-only dropdown change would be the wrong boundary; the maintainable path is a provider-owned live catalog hook with authenticated fetch, static fallback, and documented failure behavior.
