## Summary

Keep open. The example `glm-4.7-flash` is now in the static Z.AI catalog, and GLM-5 forward compatibility exists, but current main still does not implement authenticated Z.AI `/models` discovery or dynamic picker population; the remaining work needs provider/product review rather than conservative cleanup.

## What This Changes

The issue requests dynamic Zhipu/Z.AI model discovery from the provider `/models` endpoint, dynamic model picker population, and stable/custom-model fallback behavior.

## Best Possible Solution

Add a Z.AI-owned live catalog implementation using the existing model-catalog hook, defensively parse authenticated provider results, merge or fall back to the manifest catalog, preserve GLM-5 forward compatibility, and document refresh/failure behavior.

## Reproduction Assessment

No high-confidence bug reproduction is applicable because this is a feature request. The missing live `/models` discovery path is source-reproducible on current main from the static manifest, chat-completion probing, and GLM-5-only synthesis code paths.

## Solution Assessment

Unclear until provider owners confirm the Z.AI endpoint contract and desired fallback UX. A frontend-only dropdown change is not the best boundary; the maintainable direction is a provider-owned live catalog hook with static fallback and focused tests.
