Keep open. Current main still uses a static Z.AI manifest catalog plus chat-completion endpoint probing and GLM-5 forward-compatible synthesis; authenticated `/models` discovery and dynamic picker population are not implemented and need provider/product review.

## What This Changes

Request Z.AI provider model selection to come from an authenticated `/models` endpoint with stable fallback/default handling instead of only a static manifest list.

## Best Possible Solution

Add a Z.AI-owned live catalog path that authenticates against the selected endpoint, defensively parses `/models`, merges or falls back to the manifest catalog, preserves GLM-5 forward compatibility, and documents refresh/failure behavior.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection on current main verifies the requested live `/models` discovery path is absent: Z.AI currently uses static manifest rows, chat-completion endpoint probing, and GLM-5-only forward-compatible synthesis.

## Solution Assessment

Unclear until the provider contract is confirmed. A frontend-only dropdown change would be the wrong boundary; the maintainable path is a provider-owned live catalog hook with authenticated fetch, static fallback, and documented failure behavior.
