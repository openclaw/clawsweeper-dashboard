Keep open. ## What This Changes

The issue requests dynamic model discovery for the Zhipu/Z.AI provider through the provider `/models` endpoint, with static fallback and a way to use newly released models without code updates.

## Best Possible Solution

Add a Z.AI-owned live catalog provider using the existing `registerModelCatalogProvider`/`liveCatalog` seam, authenticated endpoint selection, defensive response parsing, explicit caching/fallback behavior, static manifest preservation, and focused tests/docs for refresh and failure modes.
