## Summary

Keep this PR open for maintainer review: the patch is mechanically consistent and has focused proof, but the 7-day default and 30-day maximum intentionally change Discord callback lifetime policy and need explicit compatibility/security acceptance before merge.

## What This Changes

This PR raises the Discord component callback default TTL to 7 days, increases the `agentComponents.ttlMs` cap to 30 days, and updates docs, generated metadata, UI hints, and focused tests.

## Best Possible Solution

Merge only after maintainers explicitly accept the default/max TTL policy; otherwise revise the default/cap or leave longer callback lifetimes as an explicit operator opt-in.

## Reproduction Assessment

Yes for the source-level behavior: current main falls back to a 30-minute Discord component TTL, and the PR body shows terminal proof against the real registry module at the 7-day boundary. No live multi-day Discord click was run.

## Solution Assessment

Unclear as a product choice: the implementation is narrow and consistent, but the 7-day default and 30-day maximum are not clearly the best policy without maintainer acceptance. A safer alternative is keeping the current default and requiring explicit `ttlMs` opt-in for longer-lived callbacks.
