## What This Changes

The PR raises Discord component callback default TTL from 30 minutes to 7 days, raises the `agentComponents.ttlMs` maximum from 24 hours to 30 days, and updates docs, generated metadata, UI hints, and focused tests.

## Best Possible Solution

Merge as-is only if maintainers intentionally accept the 7-day default and 30-day cap; otherwise keep the old default and rely on explicit `agentComponents.ttlMs` overrides or a revised smaller policy.
