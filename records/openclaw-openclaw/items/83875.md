## Summary

Keep open: the PR targets the right managed-update `uv_cwd` failure, but the proposed fix still lets the real `update.run` path call `process.cwd()` unguarded inside the update runner before the handoff starts. The supplied terminal smoke is useful, but it bypasses that full path and does not prove the macOS LaunchAgent update sequence.

## What This Changes

The PR hardens managed update handoff cwd selection, adds an `isMainModule` cwd fallback, updates regression tests, and adds a changelog note.

## Best Possible Solution

Keep the durable cwd approach, but thread a single safe cwd through package-root/install-surface discovery and handoff, with regression coverage that exercises the real `update.run` path and redacted managed-update proof after the fix.

## Reproduction Assessment

## Solution Assessment

No, not yet. The direction is right, but the best fix must also remove or guard the update runner's remaining cwd read and prove the full managed-update path rather than only the helper pieces.
