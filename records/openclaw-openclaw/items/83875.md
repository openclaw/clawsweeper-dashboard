## Summary

Keep open for maintainer review: the bug is source-backed and the patch looks correct, but the PR intentionally changes managed-service updater cwd semantics and still lacks a live managed-service platform smoke.

## Best Possible Solution

Land the cwd hardening once maintainers accept the cwd contract change or have the managed-service smoke proof they want for supported service platforms.

## Solution Assessment

Yes, with an upgrade-proof caveat: preparing an existing cwd before helper/updater spawn is a narrow maintainable fix for uv_cwd during managed handoff. The remaining question is platform proof and cwd-contract acceptance, not an obvious code defect.
