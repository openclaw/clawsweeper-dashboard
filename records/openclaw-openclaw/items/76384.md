## Summary

Current main fixes the reported 2026.5.2 history-loss path by preserving delivered assistant replies during session-file repair; the fix is main-only so far and is not in v2026.5.2.

## What This Changes

The issue asks for Control UI/WebChat to keep full visible chat history instead of intermittently showing only the latest message after history refreshes in 2026.5.2.

## Best Possible Solution

Keep the main-branch repair behavior, ship it in the next release, and use the existing `.bak-*` transcripts for any already-affected local sessions that need manual recovery.

## Reproduction Assessment

Yes for the affected release/root cause: the issue comments describe transcript rewrites with `.bak-*` backups, and the main fix specifically removes the session-file repair path that could drop delivered assistant replies. I did not run a live UI reproduction because this was a read-only review.

## Solution Assessment

Yes. The main fix addresses the disk-level loss at session-file repair, which is narrower and safer than broad Control UI optimistic-merge workarounds that the follow-up comments said did not resolve the disappearing replies.
