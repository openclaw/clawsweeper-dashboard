## What This Changes

The PR lazily republishes freshly loaded plugin metadata snapshots into the Gateway current-snapshot slot, adds focused regression tests and a real-runtime proof script, and updates changelog/generated SDK baseline metadata.

## Best Possible Solution

Land the lazy Gateway refill after normal maintainer validation, preserving CLI per-call freshness and the existing invalidation-on-index-write contract.

## Reproduction Assessment

Yes. Source inspection shows current main clears the slot on persisted-index writes and then reloads plugin metadata without republishing it; the PR body also provides terminal proof of the cold-slot disk-open pattern and after-fix warm-slot behavior.

## Solution Assessment

Yes. Reusing the existing Gateway-owned single-slot handoff only when an active plugin-registry workspace exists is the narrow maintainable fix, and the tests cover the CLI non-publish freshness boundary.
