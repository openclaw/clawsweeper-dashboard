## What This Changes

The PR updates manifest model-id normalization to lazily republish a freshly loaded plugin metadata snapshot into the current-snapshot slot for active gateway workspaces, with tests, a proof script, and a changelog entry.

## Best Possible Solution

Land one coherent plugin-metadata snapshot reuse fix that preserves installed-index invalidation, reuses or refills the Gateway-owned single-slot snapshot only in workspace-safe flows, and reconciles the overlapping open model-normalization work.

## Reproduction Assessment

Yes, source-level: current main clears the current metadata snapshot on persisted installed-index writes, and manifest model-id normalization does not republish after a fallback load, so active gateway flows can repeatedly re-enter the disk metadata scan after the slot is cleared.
