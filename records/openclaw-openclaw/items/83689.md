## Best Possible Solution

Fix the duplicate-`partialArgs` bug by accepting only exact duplicate or known cumulative complete argument payloads, rejecting divergent concatenations, covering the stream/repair boundary with focused tests, and requiring redacted real run proof.

## Solution Assessment

No. The right direction is narrow malformed-argument recovery, but this patch accepts any balanced JSON suffix and chooses the first object, so divergent concatenations should stay fail-closed unless a maintainer explicitly approves that policy.
