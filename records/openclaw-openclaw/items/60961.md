## Summary

Keep open. Current main still routes OpenShell mirror-mode nested file writes through a direct single-file `openshell sandbox upload` of the original host path, while the full-workspace sync path already stages uploads through a flat temporary directory; the issue remains a narrow valid bug with no closing PR found in the provided/live context.

## What This Changes

OpenShell mirror-mode `write` should upload nested workspace files into existing remote subdirectories without tar failing on an already-existing directory entry.

## Best Possible Solution

Stage regular files in `syncLocalPathToRemote` through a temporary directory containing only `path.basename(localPath)`, keep the existing delete/symlink/directory branches, and add focused coverage for the resulting `openshell sandbox upload` arguments.

## Reproduction Assessment

Yes, source-reproducible with high confidence: current main still has `writeFile` -> `syncLocalPathToRemote` -> direct `openshell sandbox upload <nested-host-file> <remote-parent>`, matching the reporter's gateway logs. I did not run a live OpenShell sandbox in this read-only review.

## Solution Assessment

Yes. A flat basename-only temporary staging directory in `syncLocalPathToRemote` is the narrowest maintainable fix because it reuses the plugin's existing full-workspace upload pattern and avoids core sandbox contract changes.

Add focused regression coverage that proves nested writes upload from the staged basename-only directory and still target the remote parent directory.
