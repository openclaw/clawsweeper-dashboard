## What This Changes

The issue requests fixing OpenShell mirror-mode write uploads so nested workspace files are written to the matching remote subdirectory instead of failing on an existing directory.

## Best Possible Solution

Stage regular files in syncLocalPathToRemote through a basename-only temporary directory before invoking openshell sandbox upload, with regression coverage for nested-file writes while preserving the existing missing-path, symlink, and directory branches.

In OpenShell mirror mode, regular-file syncLocalPathToRemote uploads a nested host file directly to the remote parent, causing openshell sandbox upload/tar to fail when the remote subdirectory already exists. Change extensions/openshell/src/backend.ts so regular files are staged through a temporary directory with only path.basename(localPath) before calling openshell sandbox upload to path.posix.dirname(remotePath), reusing the existing resolveOpenShellTmpRoot/withTempWorkspace style where possible. Preserve the current missing-path removal, symlink removal, directory mkdir behavior, sandbox ensure/seed ordering, cwd, and --no-git-ignore behavior. Add focused regression coverage for a nested file upload into an existing remote parent; update or add backend tests without reaching into core internals.
