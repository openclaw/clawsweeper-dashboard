## Summary

Keep open. Current main and the latest stable tag still send OpenShell mirror-mode single-file writes through a nested host-path upload, so the reported tar collision remains source-reproducible and there is no open or merged closing PR for the issue.

## What This Changes

The issue requests that OpenShell mirror-mode file writes into existing workspace subdirectories upload only the intended basename instead of re-uploading the nested host path component.

## Best Possible Solution

Stage regular files in syncLocalPathToRemote through a basename-only temporary directory before invoking openshell sandbox upload, with regression coverage for nested-file writes while preserving the existing missing-path, symlink, and directory handling.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: mirror-mode writeFile passes a nested host file to syncLocalPathToRemote, which still uploads that nested path directly into the remote parent.

## Solution Assessment

Yes. A basename-only staging directory for regular files is the narrowest maintainable fix because it matches the existing full-workspace staging pattern without changing core sandbox contracts.
