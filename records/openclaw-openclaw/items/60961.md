## Summary

Keep open. Current main and the latest release still upload the original nested host path in OpenShell mirror-mode single-file sync, while OpenShell upload preserves directory basenames; the referenced standalone fix commit is not associated with a PR or branch on main.

## What This Changes

The issue reports that OpenShell mirror mode fails write-tool uploads into existing sandbox subdirectories because single-file sync uploads the nested host path directly.

## Best Possible Solution

Stage regular files in syncLocalPathToRemote through a basename-only temporary directory before upload, while preserving the existing missing-path, symlink, and directory branches with focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible with high confidence: current main routes mirror-mode writeFile through syncLocalPathToRemote and still uploads the nested host file directly to the remote parent. I did not run a live OpenShell sandbox during this read-only review.

## Solution Assessment

Yes. A basename-only staging directory in syncLocalPathToRemote is the narrowest maintainable fix because it reuses the plugin's existing staged upload pattern without changing core sandbox contracts.
