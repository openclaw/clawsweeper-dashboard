## Summary

Keep open. Current main still sends mirror-mode write-tool updates through `syncLocalPathToRemote` with the original nested host file path, so the reported `tar: <dir>: Cannot open: File exists` failure remains a valid narrow OpenShell plugin bug.

## What This Changes

Fix OpenShell mirror-mode write uploads so writing a file under an existing remote subdirectory no longer fails during `openshell sandbox upload` extraction.

## Best Possible Solution

Keep the fix inside the OpenShell plugin: for regular-file mirror writes, stage the file into a temporary flat directory using its basename, upload that staging directory to `path.posix.dirname(remotePath)`, clean it up, and add a focused regression test plus changelog entry.

## Reproduction Assessment

Yes. The issue gives concrete configuration, steps, and gateway logs, and current main still has the static `writeFile` -> `syncLocalPathToRemote` -> `openshell sandbox upload <localPath> <dirname(remotePath)>` path for nested mirror writes.

## Solution Assessment

Yes. The basename-only temporary staging approach is the narrowest maintainable fix because it matches the existing workspace upload staging pattern without changing core sandbox contracts.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/60961 in the OpenShell plugin only. Current `extensions/openshell/src/fs-bridge.ts` calls `syncLocalPathToRemote(hostPath, target.containerPath)` after mirror-mode writes, and `extensions/openshell/src/backend.ts` uploads the original nested `localPath` into `path.posix.dirname(remotePath)`, which can make `openshell sandbox upload` tar a parent directory that already exists remotely. In `extensions/openshell/src/backend.ts`, keep the existing missing-file, symlink, directory, remote-parent creation, and root-boundary behavior, but for regular files create a temp directory under `resolveOpenShellTmpRoot()`, copy the file into it as `path.basename(localPath)`, upload that temp directory to `path.posix.dirname(remotePath)`, and always clean up. Do not change core sandbox contracts or OpenShell remote-mode behavior. Add focused regression coverage, preferably in `extensions/openshell/src/backend.test.ts` or the smallest existing OpenShell test surface, that proves nested regular-file sync uploads a flat staging directory rather than the original nested file path and preserves cleanup/error handling. Add a single-line `CHANGELOG.md` fix entry under Unreleased/Fixes, crediting the reporter if appropriate and avoiding bot/maintainer thanks.
