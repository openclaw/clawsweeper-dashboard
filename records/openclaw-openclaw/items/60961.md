## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/60961 in the OpenShell plugin. Current main's mirror-mode `write` path writes the host file and calls `syncLocalPathToRemote` with the nested host path, then `syncLocalPathToRemote` passes that path directly to `openshell sandbox upload`, which can make tar try to create an already-existing directory under the remote parent. Update only the OpenShell regular-file sync path so it creates a temp staging directory under `resolveOpenShellTmpRoot()`, copies the file into that directory using `path.basename(localPath)`, uploads the staging directory to `path.posix.dirname(remotePath)`, and always cleans up. Preserve the existing missing-path, symlink, directory, remote-mode, and core sandbox behavior; do not change core sandbox APIs. Add focused regression coverage that would fail if a nested mirror write uploads the original nested host path instead of a basename-only staging directory. Add a user-facing CHANGELOG fix entry for #60961 without bot attribution. Use PR #57693's workspace upload staging as the implementation pattern and keep PR #69797's root-pinned write hardening intact.
