## Summary

Keep open. Current main and the latest stable tag still route OpenShell mirror-mode single-file writes through a direct nested host-path upload, and no open closing PR was found for the reported failure.

## What This Changes

Fix OpenShell mirror-mode writes so writing a file inside an existing remote subdirectory succeeds instead of failing during `openshell sandbox upload`.

## Best Possible Solution

Stage regular files in `syncLocalPathToRemote` through a basename-only temporary directory before invoking `openshell sandbox upload`, while preserving the existing missing-path, symlink, and directory branches.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: mirror-mode `writeFile` passes a nested host file to `syncLocalPathToRemote`, which still uploads that nested path directly into the remote parent.

## Solution Assessment

Yes. A basename-only staging directory for regular files is the narrowest maintainable fix because it matches the existing full-workspace staging pattern without changing core sandbox contracts.
