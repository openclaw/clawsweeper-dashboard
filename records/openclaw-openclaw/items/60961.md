## Summary

Keep open. Current main still routes OpenShell mirror-mode write-tool updates through the single-file upload path that passes the original nested host path to `openshell sandbox upload`, so the reported existing-remote-directory failure remains plausible and narrow to fix.

## What This Changes

Fix OpenShell mirror-mode file writes so writing into an existing remote subdirectory uploads a basename-only staged file into the remote parent directory.

## Best Possible Solution

Keep the fix inside the OpenShell plugin: for regular-file mirror writes, copy the file into a temporary flat staging directory under the OpenClaw temp root, upload that staging directory to `path.posix.dirname(remotePath)`, and clean it up while leaving the generic sandbox contracts unchanged.

## Reproduction Assessment

Yes. The issue includes concrete steps and logs, and current main still has the static path `writeFile` -> `syncLocalPathToRemote` -> `openshell sandbox upload <localPath> <dirname(remotePath)>` for nested mirror writes.

## Solution Assessment

Yes. The basename-only staging approach is the narrowest maintainable fix because it matches the existing full-workspace staging pattern while avoiding core sandbox changes.
