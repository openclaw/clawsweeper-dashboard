## Summary

Keep open. Current main still sends OpenShell mirror-mode nested writes through a direct `openshell sandbox upload` of the original nested host path, while the safer full-workspace path stages uploads first; no closing PR or duplicate owner was found.

## What This Changes

The issue requests fixing OpenShell mirror-mode writes so nested workspace files upload into existing remote subdirectories instead of failing during tar extraction.

## Best Possible Solution

Stage regular files in `syncLocalPathToRemote` through a basename-only temporary directory before `openshell sandbox upload`, preserving the existing delete, symlink, and directory branches with focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible with high confidence: current main still routes `writeFile` through `syncLocalPathToRemote` and directly uploads the nested host file to the remote parent directory. I did not run a live OpenShell sandbox in this read-only review.

## Solution Assessment

Yes. A basename-only staging directory in `syncLocalPathToRemote` is the narrowest maintainable fix because it reuses the plugin's existing staged upload approach without changing core sandbox contracts.
