## Summary

Keep open: current main supports the managed proxy path, but the Feishu HTTP call sites named here still use guarded fetch in strict mode and therefore do not honor raw proxy environment variables the way the Feishu WebSocket path does.

## What This Changes

Request Feishu plugin HTTP API calls to honor proxy environment variables in addition to the existing WebSocket proxy handling.

## Best Possible Solution

Add a Feishu-owned HTTP helper that opts trusted Feishu/Lark API requests into env-proxy guarded fetch while preserving hostname allowlists, SSRF checks, and the core strict default.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: set HTTPS_PROXY without proxy.enabled, then Feishu streaming-card or app-registration fetchWithSsrFGuard calls remain strict while WSClient gets an ambient proxy agent. I did not run a live Feishu account/proxy scenario.

## Solution Assessment

Yes, the narrow maintainable solution is plugin-local proxy parity for trusted Feishu/Lark API hosts using the existing guarded-fetch proxy mode; changing the core strict default would be too broad.
