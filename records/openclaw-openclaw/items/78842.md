## What This Changes

Request Feishu HTTP API calls to honor HTTP(S)_PROXY like the Feishu WebSocket path in proxy-only environments.

## Best Possible Solution

Track the remaining work in https://github.com/openclaw/openclaw/issues/72595: one Feishu-owned proxy policy should cover SDK HTTP, guarded-fetch HTTP, WebSocket, docs, and tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set HTTPS_PROXY without managed proxy enabled, then Feishu streaming-card and app-registration fetchWithSsrFGuard calls remain strict while the existing WebSocket path can use an ambient proxy agent; I did not run a live Feishu account/proxy setup.

## Solution Assessment

No, not as a standalone unconditional HTTP env-proxy change. Related open reports show Feishu needs one explicit direct/ambient proxy policy, so the broader canonical issue is the safer maintainable path.
