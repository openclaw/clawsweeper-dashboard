## Summary

Keep open: current main still leaves the Feishu streaming-card and app-registration HTTP calls in default strict guarded-fetch mode, so raw `HTTP_PROXY`/`HTTPS_PROXY` environments still do not match the Feishu WebSocket proxy path. The exact fix attempt in https://github.com/openclaw/openclaw/pull/78849 was closed unmerged after the author could not provide live Feishu proxy proof.

## What This Changes

Request Feishu streaming-card and app-registration HTTP calls honor proxy environment variables instead of using the default strict guarded-fetch mode.

## Best Possible Solution

Add a Feishu-local guarded-fetch helper for trusted Feishu/Lark API hosts, use trusted env-proxy mode only on the streaming-card and app-registration HTTP calls, and keep core strict mode unchanged.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set `HTTPS_PROXY` without `proxy.enabled`, then Feishu streaming-card/app-registration calls omit `mode` and stay strict while WSClient resolves an ambient proxy agent.

## Solution Assessment

Yes. The narrow maintainable direction is plugin-local trusted env-proxy routing for fixed Feishu/Lark API hosts; changing the core guarded-fetch default or folding in the broader Feishu SDK/WebSocket proxy policy would be too broad.
