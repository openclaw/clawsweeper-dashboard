## Summary

Keep open: current main still has an unbounded `channels capabilities` hook path, and explicit channel lookup can still enter plugin install/config refresh work matching the reported staging symptom.

## What This Changes

The issue reports that `openclaw channels capabilities --channel discord` hangs, leaves CPU-bound `openclaw-channels` children, and should instead return quickly or fail within a bounded timeout.

## Best Possible Solution

Make the capability diagnostic path bounded end to end, keep read-only inspection off install/registry mutation paths unless setup is explicit, and add focused regression coverage.

## Reproduction Assessment

Yes, from source: a configured channel plugin whose probe or diagnostics hook never settles will keep `channels capabilities` waiting because the command awaits those hooks directly. The specific Discord CPU-bound child process was not live-reproduced.

## Solution Assessment

Yes, a narrow repair is the best path: enforce the existing CLI timeout around all capability hook work and avoid read-only capability commands triggering install/registry refresh work.
