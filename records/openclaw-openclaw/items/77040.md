## What This Changes

The issue asks for `openclaw channels capabilities --channel discord` to return Discord capability diagnostics quickly or fail with a bounded timeout instead of hanging and leaving a CPU-bound channel process.

## Best Possible Solution

Keep the capability diagnostic bounded end to end: apply the requested timeout across plugin resolution/setup, channel probes, and capability diagnostics, and make any dependency-staging side effects either bounded and cleaned up or explicitly moved out of this read-only probe path by maintainer decision.
