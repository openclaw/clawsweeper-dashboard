## Summary

Keep open: current main still has the risky diagnostic shape. The command is documented for Discord voice permission checks, but the current path can still load configured channel plugins and the Discord targeted diagnostics await permission work without carrying the CLI timeout through that call.

## What This Changes

The issue reports that `openclaw channels capabilities --channel discord` can hang, spawn a CPU-bound `openclaw-channels` child process, and fail to return bounded Discord capability diagnostics.

## Best Possible Solution

Make `channels capabilities` a bounded read-only diagnostic path: avoid full configured plugin activation when metadata/setup artifacts are enough, and carry the CLI timeout through Discord capability and permission checks with focused regression coverage.

## Reproduction Assessment

No live high-confidence reproduction was established in this read-only review. Source inspection still shows current main preserving the likely failure path: configured channel plugin loading plus Discord targeted diagnostics without a caller-controlled timeout.

## Solution Assessment

Yes, a narrow repair is the right direction. The maintainable fix is to keep this diagnostic read-only and bounded rather than adding a new mode or broad compatibility layer.
