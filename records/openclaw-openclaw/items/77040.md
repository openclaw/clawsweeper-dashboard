## Summary

Keep open: the current channel capabilities path still awaits plugin probe and diagnostics hooks without enforcing the CLI timeout, and explicit-channel resolution can still enter plugin install/registry refresh work that matches the reported dependency-staging symptom.

## What This Changes

The issue asks for `openclaw channels capabilities --channel discord` to return quickly or fail with a bounded timeout instead of hanging and leaving CPU-bound channel child processes.

## Best Possible Solution

Bound the diagnostic path end to end and keep capability inspection on a read-only plugin path unless the user explicitly runs setup or install.

## Reproduction Assessment

Yes, from source: a configured channel plugin whose probe or diagnostics hook never settles will keep `channels capabilities` waiting because those hooks are awaited directly. The exact Discord CPU-bound child process was not live-reproduced.

## Solution Assessment

Yes, a narrow repair is appropriate: enforce the existing CLI timeout around all capability hook work and avoid read-only capability commands triggering install or registry refresh work.
