## Summary

Keep open: current main still lacks a command-level timeout around channel capability probe/diagnostics hooks, and the explicit-channel path can still enter plugin install/registry refresh work before rendering capabilities.

## What This Changes

The issue reports that `openclaw channels capabilities --channel discord` can hang, emit plugin dependency staging output, and leave a CPU-bound `openclaw-channels` child process.

## Best Possible Solution

Bound the capabilities command end to end under `--timeout`, report hook timeouts as actionable diagnostics, and keep setup/install work out of read-only inspection unless explicitly requested.

## Reproduction Assessment

Yes, from source: a plugin whose `probeAccount` or `buildCapabilitiesDiagnostics` promise never settles will keep `channels capabilities` awaiting indefinitely. I did not live-reproduce the production Discord CPU-bound child process.

## Solution Assessment

Yes, a narrow repair is appropriate: enforce the existing CLI timeout around capability hook work and add focused regression coverage. Any change to optional install-on-demand should preserve or deliberately update the existing tested behavior.

## Repair Work Prompt

Fix openclaw/openclaw#77040. On current main, `openclaw channels capabilities --channel discord` is not bounded by the CLI timeout across plugin status hooks. Wrap `probeAccount` and `buildCapabilitiesDiagnostics` in `src/commands/channels/capabilities.ts` with command-level timeout/error conversion so the command returns promptly even if a plugin hook never settles. Audit the explicit-channel resolution path so already configured/installed channels do not unnecessarily trigger plugin install/registry refresh work; do not remove existing optional install-on-demand behavior without updating its tests/docs/changelog or stopping for maintainer review. Add regression tests in `src/commands/channels/capabilities.test.ts` for hanging probe and diagnostics hooks, and update `src/channels/plugins/read-only.test.ts` only if read-only plugin resolution behavior changes. Add a single-line `CHANGELOG.md` fix entry if user-facing behavior changes. Do not introduce a new config option or broad plugin-loader architecture change.
