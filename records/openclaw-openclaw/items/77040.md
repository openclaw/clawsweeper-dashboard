## Summary

Keep open: current main still has an existing `channels capabilities` diagnostic path that can await plugin capability hooks without a command-level timeout and can enter install/registry-refresh behavior for an explicit channel.

## What This Changes

Bound `openclaw channels capabilities --channel discord` so it returns quickly or times out instead of leaving CPU-bound channel/plugin work behind.

## Best Possible Solution

Keep the diagnostic bounded end to end and keep capability inspection on a read-only path, returning actionable setup guidance instead of doing install or registry mutation work.

## Reproduction Assessment

No live high-confidence reproduction was established in this read-only review. Source inspection does show a clear current-main path where the documented timeout does not bound plugin hooks and the explicit channel path can enter install/registry refresh work.

## Solution Assessment

Yes, a narrow fix is the right direction: enforce the command timeout around capability hooks and prevent read-only capability inspection from performing install or registry mutation work. This should preserve existing channel semantics while making the diagnostic safe to run on a live gateway.
