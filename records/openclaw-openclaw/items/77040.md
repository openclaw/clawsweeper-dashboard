## Summary

Keep open: current main still has an existing diagnostic command whose documented `--timeout` is only passed into some plugin hooks, while the surrounding capability discovery, setup fallback, explicit install, and registry refresh paths are not bounded end to end.

## What This Changes

The issue reports that `openclaw channels capabilities --channel discord` hangs, stages plugin runtime dependencies, and leaves a CPU-bound `openclaw-channels` child process instead of returning bounded diagnostics.

## Best Possible Solution

Keep `channels capabilities` bounded end to end: enforce timeout handling around capability hooks and loader/install work, and avoid unnecessary setup/runtime loading for already configured Discord diagnostics.

## Reproduction Assessment

Partially: source inspection gives a clear current-main path where `channels capabilities --channel discord` can wait beyond its command timeout around capability hooks and plugin discovery/install work. The exact production CPU peg was not live-reproduced in this review.

## Solution Assessment

Yes, the narrow maintainable fix is to add command-level timeout coverage and regression tests around the current capability path while preserving existing channel/plugin install semantics unless maintainers choose to change them.
