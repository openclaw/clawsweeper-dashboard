## What This Changes

The PR adds Telegram middleware that answers callback queries before per-chat/topic sequentialization and skips the downstream callback acknowledgement after early success.

## Best Possible Solution

Land the focused Telegram-layer acknowledgement fix after redacted live Telegram proof shows the callback spinner clears while callback business logic remains serialized behind the busy chat/topic lane.

## Reproduction Assessment

Yes, source-reproducible: current main and v2026.6.1 answer callback queries only inside the callback handler that runs after `sequentialize`, and grammY's sequentializer waits before calling downstream middleware. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

Yes, this appears to be the best narrow fix shape: acknowledge the Telegram callback before the queue while keeping callback handling itself serialized. Broader alternatives like moving all callback handling to a separate lane would risk changing ordering semantics, while approval-only lane fixes are too narrow for generic inline buttons.
