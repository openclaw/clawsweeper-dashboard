## Summary

Keep open: current main still has misleading generic hover affordances, but this PR cannot merge as-is because it targets a stale Overview path, mixes unrelated generated/tooling artifacts, and lacks real behavior proof.

## What This Changes

The branch scopes generic card hover styles to interactive modifiers, wires old Overview stat-card divs for click/keyboard navigation, adds an Overview test, adjusts hook-loader tests, and adds large Spryng routing/data/docs artifacts.

## Best Possible Solution

Land a focused Web Control UI patch that scopes hover/focus affordances to real interactive elements, keeps the current native `button.ov-card` Overview path, adds narrow coverage, and omits unrelated artifacts.

## Reproduction Assessment

Yes. Source inspection on current main shows generic `.card:hover`, `.stat:hover`, and `.account-card:hover` styling on containers, while generic account cards render as plain non-clickable `div.account-card` elements; I did not run a browser session.

## Solution Assessment

No. The UI direction is reasonable, but this branch should be respun as a narrow patch against the current `overview-cards.ts` and `components.css` surfaces with unrelated artifacts removed.
