Keep open: current main exposes session and memory primitives, but the inspected code keeps recall visibility-scoped and explicit rather than automatically recovering ambiguous cross-channel context, and the remaining work needs a privacy-aware product decision.

## What This Changes

The issue requests cross-channel context discovery or handoff so a WebChat approval turn can recover recent WhatsApp context for the same user identity.

## Best Possible Solution

Define a scoped, opt-in cross-surface handoff or recall policy that reuses existing session tools and session-memory under identity and visibility constraints, then cover WhatsApp-to-WebChat approval handoffs with regression tests.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection does show the plausible current-main path: separate sessions are not automatically attached, and session and memory recall are constrained by configured visibility.

## Solution Assessment

Unclear as a code fix: a prompt-only nudge may help but is probably not sufficient unless paired with explicit identity and visibility policy. A maintainable solution should decide between opt-in auto-attach, handoff tokens, or session unification rather than silently broadening recall.
