## Summary

Keep open: current main has the session and memory primitives, but not automatic cross-channel recall for an ambiguous WebChat handoff, and the remaining work needs a privacy-aware product decision.

## What This Changes

Request automatic recall of recent conversation context when the same user moves a task from WhatsApp to WebChat for approval.

## Best Possible Solution

Define a scoped, opt-in cross-surface handoff or recall policy that reuses existing session tools/session-memory under identity and visibility constraints, then cover WhatsApp-to-WebChat approval handoffs with regression tests.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection does show the plausible current-main path: separate sessions are not automatically attached, and session/memory recall is constrained by configured visibility.

## Solution Assessment

Unclear as a code fix: a prompt-only nudge may help but is probably not sufficient unless paired with explicit identity and visibility policy. A maintainable solution should decide between opt-in auto-attach, handoff tokens, or session unification rather than silently broadening recall.
