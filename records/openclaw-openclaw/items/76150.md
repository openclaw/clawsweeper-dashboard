## Summary

Keep open: current main still has the reported health shape, where an update-triggered restart can report the Gateway as unhealthy when the service is active but the Gateway port is free, and the available issue data is not enough to prove this is fixed by the newer package-update guard or the related Linux/systemd handoff work.

## What This Changes

The issue reports that after updating OpenClaw, Gateway restart verification reports a version mismatch/unavailable gateway while the Linux service is active and the Gateway port is free.

## Best Possible Solution

Keep this open until the reporter or a maintainer attaches the deep status, restart log, and Linux journal; then route the fix to the specific failing layer, whether that is systemd handoff, package-update restart orchestration, service startup, or port diagnostics.

## Reproduction Assessment

No for the full WSL2 OpenEuler npm-update failure. Source inspection and existing tests cover the running-service/free-port health shape, but the actual reason the active service is not listening is still missing.

## Solution Assessment

Unclear. A #66735-style systemd handoff is likely best if logs show an in-service restart, while a package-update guard, service startup fix, or port/probe diagnostic fix would be narrower if the logs point elsewhere.
