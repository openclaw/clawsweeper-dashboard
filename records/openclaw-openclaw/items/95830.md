## Summary

## What This Changes

The PR records public Telegram poll origins in plugin keyed state, adds a `poll_answer` handler that enqueues vote events, pins the update allowlist defensively, and adds Telegram registry/send/handler tests.

## Best Possible Solution

Keep the sidecar-free plugin-state registry, but route poll answers through the canonical Telegram conversation route and add live Telegram proof for direct plus topic/bound-session delivery before merge.
