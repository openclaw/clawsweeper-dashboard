## Summary

Keep open: this is a focused bug-fix PR that is still necessary because current main and the latest release still reject short tweakcn IDs before fetch, while the patch preserves the existing importer safety checks and adds targeted regression coverage.

## What This Changes

The PR lowers the Control UI tweakcn theme ID validator from an 8-character minimum to 4 characters and adds regression tests for short built-in IDs plus malformed short IDs.

## Best Possible Solution

Merge this narrow validator/test fix once the draft state and normal checks are resolved, keeping the importer’s existing host, redirect, payload-size, schema, and CSS-token safeguards intact.

## Reproduction Assessment

Yes. Current main’s exact regex rejects `claude`, `zinc`, `slate`, and `twitter` before fetch, and live tweakcn probes returned valid JSON for short IDs `claude` and `twitter`.

## Solution Assessment

Yes. Relaxing only the existing ID-length validator while preserving the existing host/path/fetch/schema/CSS checks is the narrowest maintainable fix; no new UI, config, storage, or fallback path is needed.
