## Summary

Keep open: the PR is a meaningful candidate fix for the source-reproducible #95506 MCP runtime fan-out issue, but it adds a new config/security boundary and still lacks contributor real-behavior proof from a real OpenClaw setup.

## What This Changes

The PR adds an opt-in `mcp.runtimeScope: "shared"` mode, rewires bundled MCP runtime ownership to a workspace/config runtime key, and updates tests, schema/help labels, generated config hashes, and docs.

## Best Possible Solution

Keep default session isolation and land shared mode only after maintainers accept the single-tenant isolation contract, the PR provides redacted real runtime proof for both scopes, and the #79882 overlap is resolved.

## Reproduction Assessment

Yes at source level: current main and v2026.6.9 key bundled MCP runtimes by session id, and #95506 includes a focused stdio proof showing duplicate runtimes, catalogs, processes, and `tools/list` calls for identical workspace/config sessions.

## Solution Assessment

Partially: workspace/config-keyed sharing is a bounded implementation layer for the reported fan-out, but the new config surface and cross-session MCP state boundary need maintainer acceptance and real runtime proof before it is the best fix.
