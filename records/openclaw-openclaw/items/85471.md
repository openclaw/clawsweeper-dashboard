Keep open. The patch is the right narrow fix for the exact-token cron announce leak, but the visible real-behavior proof is still not inspectable enough to clear the external-PR proof gate.

## What This Changes

The PR changes cron isolated-agent delivery to use the shared suppressed-control-reply predicate and expands cron delivery tests for token-only `NO_REPLY`, `ANNOUNCE_SKIP`, and `REPLY_SKIP`.

## Best Possible Solution

Use this PR as the canonical exact-token fix after an inspectable redacted cron-to-channel proof artifact is added or maintainers explicitly accept the existing Crabbox proof record; handle markdown-wrapped variants separately.

## Reproduction Assessment

Yes. Source inspection on current main shows cron delivery only suppresses `NO_REPLY`, while the shared predicate already suppresses `ANNOUNCE_SKIP` and `REPLY_SKIP`; the linked issue also reports repeated Discord cron leaks.

## Solution Assessment

Yes for the exact-token leak. Reusing the shared suppression predicate in cron delivery is the narrow maintainable solution; markdown-wrapped variants should only be added with explicit tests and maintainer agreement.
