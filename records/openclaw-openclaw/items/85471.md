## What This Changes

The PR changes cron announce/direct delivery to suppress token-only `NO_REPLY`, `ANNOUNCE_SKIP`, and `REPLY_SKIP` before outbound sends, with regression coverage for direct and text delivery.

## Best Possible Solution

Land the exact-token cron suppression fix and track markdown-wrapped token handling separately if maintainers want to broaden the control-reply contract.

## Reproduction Assessment

Yes. Current main and `v2026.5.20` show cron delivery still suppresses only `NO_REPLY`, while the existing shared predicate already covers `ANNOUNCE_SKIP` and `REPLY_SKIP`; the linked issue and PR proof cover the real external-channel symptom.

## Solution Assessment

Yes for the exact-token leak. Reusing the shared predicate in the cron delivery normalizer is the narrow maintainable fix, while markdown-wrapped variants are a separate policy change needing their own tests.
