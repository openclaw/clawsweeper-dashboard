## Summary

Keep open. Current main has CLI JSONL delta parsing into the global agent-event stream, but the auto-reply CLI path still does not feed those deltas into channel preview callbacks, and the requested opt-in config, capability metadata, and diagnostics are still absent.

## What This Changes

Add opt-in incremental streaming for CLI backends so channel previews can receive progressive updates instead of only final buffered output.

## Best Possible Solution

Define a capability-gated CLI incremental preview contract that keeps buffered behavior as default, forwards eligible CLI delta snapshots to existing channel `onPartialReply` hooks with coalescing and final dedupe, and documents diagnostics for produced-versus-dropped incremental output.

## Reproduction Assessment

Yes. A source-level reproduction is clear: a Claude stream-json CLI backend can emit assistant deltas into agent events, but the auto-reply CLI branch has no `onPartialReply` path, so Telegram and Feishu preview hooks remain final-only for CLI runs.

## Solution Assessment

Unclear. The requested opt-in direction is reasonable, but the exact backend config, capability metadata, diagnostics, and cross-channel dedupe behavior need maintainer agreement before a repair patch should wire deltas into previews.
