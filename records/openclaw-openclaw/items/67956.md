## Summary

Keep open. Current main has partial CLI JSONL delta plumbing into the global agent-event stream, but the auto-reply CLI path still does not bridge those deltas into channel preview `onPartialReply` hooks, and the requested opt-in config/capability/diagnostic contract is still absent.

## What This Changes

The issue requests an optional incremental streaming mode for CLI backends so channels can show progressive preview updates during long CLI-backed agent runs instead of waiting for final buffered output.

## Best Possible Solution

Add a capability-gated CLI incremental preview contract: keep buffered behavior as the default, let eligible CLI deltas feed existing channel `onPartialReply` preview hooks with coalescing and final-message dedupe, add diagnostics for produced-versus-dropped incremental output, and update config docs/schema/tests together.

## Reproduction Assessment

Yes. A code-level reproduction is clear: a Claude stream-json CLI backend can emit assistant deltas into agent events, but the auto-reply CLI branch has no `onPartialReply` path, so Telegram/Feishu preview hooks stay final-only for CLI runs.

## Solution Assessment

Unclear. The opt-in direction is reasonable, but the exact public backend config, capability metadata, and cross-channel dedupe semantics should be settled before an automated patch forwards CLI assistant events into previews.
