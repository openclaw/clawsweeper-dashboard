## Summary

Keep open for maintainer review: the patch is a focused fix for the linked CDP attachment-download bug, and I found no concrete line-level defect, but it intentionally changes the browser `/navigate` download persistence and response boundary.

## What This Changes

The PR changes Playwright-backed browser `/navigate` to capture CDP attachment downloads, validate the captured URL before managed persistence, and return optional download metadata.

## Best Possible Solution

Land after browser maintainers accept the guarded download persistence boundary and additive `/navigate` response payload; no automated repair is needed from this pass.

## Reproduction Assessment

Yes. Source inspection plus Playwright v1.60.0 source gives a high-confidence path: current main has no `/navigate` download-success handling, and Playwright Chromium throws `Download is starting` when CDP navigation reports a download.

## Solution Assessment

Yes. The fix is at the existing Playwright navigation seam, uses the current route's result forwarding, and rejects blocked captured URLs before `saveAs`; lower-level CDP-only or explicit-action-only alternatives would miss the direct `/navigate` failure.
