## Summary

## What This Changes

The PR changes Playwright-backed browser `/navigate` to capture attachment downloads, validate captured download URLs before saving bytes, return optional download metadata, and add focused regression tests.

## Best Possible Solution

Land this after browser maintainers accept the policy-before-save boundary and additive response contract, with optional contract/docs polish if they want the new payload advertised before merge.

## Reproduction Assessment

Yes. Source inspection shows current main rethrows Playwright's download-starting navigation abort, and Playwright v1.60.0 Chromium throws that abort when CDP `Page.navigate` reports a download.

## Solution Assessment

Yes, this is the right layer: the browser route already forwards `navigateViaPlaywright` results, and the patch validates captured download URLs before saving. A lower-level CDP-only hook would be broader than needed for this focused `/navigate` seam.
