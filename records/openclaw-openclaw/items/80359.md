## Summary

Keep open because the branch still uses token equality as the token-only reconnect signal, which can make ordinary `openclaw dashboard` launches with a restored same-gateway token skip the reset-to-main behavior established by the related session-scoping work. Maintainer review should choose an explicit session or reconnect marker before this merges.

## What This Changes

This PR changes Control UI URL hydration so token-only WebChat URLs keep the current session when the URL token matches the stored token, and adds focused `app-settings` tests.

## Best Possible Solution

Preserve reconnect sessions through an explicit `session=` URL or a distinct reconnect marker while keeping ordinary token-only dashboard launches as fresh opens to `main`.

## Reproduction Assessment

Yes for the blocking review finding by source inspection: dashboard emits token-only URLs, settings can restore the same token and a non-main session before URL hydration, and the PR guard then skips the reset. The full Gateway crash/restart symptom was not independently reproduced.

## Solution Assessment

No. Token equality alone is not the narrowest maintainable reconnect signal because ordinary dashboard launches can carry the same token; explicit session intent or a separate reconnect marker is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep same-token dashboard launches on main:** `ui/src/ui/app-settings.ts:214-216`
  - body: The new guard treats any token-only URL whose token equals `host.settings.token` as a reconnect. `openclaw dashboard` also emits token-only URLs from the configured gateway token, and settings load can restore that same token from `sessionStorage` along with a persisted non-main session before URL hydration runs, so an ordinary dashboard launch can skip the reset that the related session-scoping work intentionally requires. Please require an explicit `session=` or distinct reconnect signal instead of token equality alone.
  - confidence: 0.88
