## Summary

Keep open: the PR targets a source-reproducible WebChat reconnect/session-loss path and now includes real browser proof, but the patch removes the deliberate token-only launch reset guardrail from the Control UI URL hydrator and can reintroduce stale session restoration for fresh tokenized opens.

## What This Changes

The branch removes the Control UI token-only session reset, updates URL settings tests to preserve the stored session, and adds a WebChat reconnect changelog entry.

## Best Possible Solution

Keep the fresh token-only launch reset and add a narrower reconnect/session-known path, preferably by carrying an explicit session on reconnect URLs, with tests covering both semantics.

## Reproduction Assessment

Yes for the narrow URL-hydration path: current main source and a focused test show token-only URLs without session reset selection to main. The full Windows Gateway crash/restart flow is not reproduced here.

## Solution Assessment

No. Deleting the reset fixes the reported preservation case but removes a deliberate launch guardrail; the safer fix should distinguish reconnect from a fresh tokenized open.
