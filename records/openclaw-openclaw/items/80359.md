## Summary

Keep open; this is a plausible user problem, but the PR should not merge as-is because the reconnect heuristic is ambiguous and can undo the token-only fresh-launch behavior established by https://github.com/openclaw/openclaw/pull/47453.

## What This Changes

The branch changes Control UI URL hydration so same-gateway token-only URLs skip the session reset when the URL token matches the already-loaded settings token, adds focused app-settings tests, and adds a changelog entry.

## Best Possible Solution

Keep this open until maintainers choose an unambiguous reconnect contract, then preserve sessions through explicit `session=` URLs or a separate reconnect marker while retaining token-only fresh-launch reset coverage.

## Reproduction Assessment

Yes for the review finding: source inspection of URL hydration, token storage, and dashboard URL construction shows same-token token-only launches can skip reset under the PR. No full live Gateway crash/restart reproduction was established; the proof covers URL hydration in a real browser.

## Solution Assessment

No. Token equality alone is not the narrowest maintainable fix because ordinary dashboard launches can use the same token; the safer contract is explicit `session=` or a separate reconnect/session-known marker.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep same-token dashboard launches on main:** `ui/src/ui/app-settings.ts:214-216`
  - body: The new guard treats any token-only URL whose token equals `host.settings.token` as a reconnect. `openclaw dashboard` also builds token-only URLs from the configured gateway token, and `loadSettings()` restores that same token from sessionStorage along with a persisted non-main session. A normal dashboard launch can therefore skip the reset that https://github.com/openclaw/openclaw/pull/47453 intentionally requires and reopen a stale session. Please require an explicit session or reconnect signal instead of token equality alone.
  - confidence: 0.88
