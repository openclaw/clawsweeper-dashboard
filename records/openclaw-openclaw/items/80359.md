## Summary

Keep open: this targets a plausible WebChat restore bug, but the patch is too broad because it removes an intentional token-only launch reset guardrail from the prior session-scoping work, and the external PR only provides test/check claims as proof.

## What This Changes

The branch removes token-only session reset from Control UI URL settings, updates URL/session tests to preserve persisted session selection, and adds a changelog entry.

## Best Possible Solution

Keep the prior fresh-token launch guardrail while adding a narrower same-gateway reconnect restoration path, with tests covering both semantics and real proof for the crash/restart flow.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection and the existing focused test show the token-only URL reset path on current main, but the full Windows WebChat Gateway crash/restart flow is not proven by this PR.

## Solution Assessment

No. Deleting the reset entirely solves the reported test case but also removes the prior token-only launch guardrail; the safer fix should distinguish a reconnect to an existing scoped session from a fresh tokenized launch.
