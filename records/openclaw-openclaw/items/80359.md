## Summary

Keep this PR open, but the current patch should not merge as-is. The follow-up restored the different-token reset case, but token equality still conflates a reconnect with a fresh same-token dashboard launch, which weakens the token-only launch contract from the prior session-scoping change.

## What This Changes

The branch changes Control UI URL hydration so token-only URLs skip the session reset when the supplied token matches the loaded same-gateway token, with focused settings tests and a changelog entry.

## Best Possible Solution

Keep the fresh token-only launch reset unless the URL carries an explicit session or another unambiguous reconnect/session-known signal, with tests covering matching reconnect, same-token fresh launch, different-token fresh launch, and explicit session URLs.

## Reproduction Assessment

Yes for the narrow source-level URL-hydration path and the review finding: current main and the PR diff show exactly when token-only URLs reset or preserve session. No live Windows Gateway crash/restart replay was established.

## Solution Assessment

No. Token equality is an ambiguous reconnect signal because normal dashboard links reuse the same gateway token; the safer fix should keep fresh same-token launches on main unless a distinct reconnect/session-known signal proves otherwise.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Keep same-token fresh launches on main:** `ui/src/ui/app-settings.ts:215`
  - body: The new guard treats any token-only URL whose token equals `host.settings.token` as a reconnect. `openclaw dashboard` opens and copies tokenized URLs using the configured gateway token, so a fresh dashboard launch after selecting an old session can now preserve that stale session instead of honoring the existing token-only launch contract from https://github.com/openclaw/openclaw/pull/47453. Please add a distinct reconnect/session-known signal or explicit `session=` path, and keep same-token fresh launches covered by a reset test.
  - confidence: 0.85
