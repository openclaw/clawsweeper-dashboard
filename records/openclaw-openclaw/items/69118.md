## Summary

Keep open. Current main still has the reported session-continuity bug: groupIntro is first-turn/re-intro text, but it remains part of the static prompt identity used for CLI reuse, and the current Claude live-session fingerprint also changes when the full prompt changes.

## What This Changes

The issue reports that group-channel Claude CLI sessions reset on follow-up turns because turn-scoped group intro text changes CLI session prompt identity.

## Best Possible Solution

Keep first-turn group guidance in the actual prompt, but exclude turn-scoped groupIntro from stable CLI session identity and Claude live-session restart identity, with regression coverage for first-to-second group turns.

## Reproduction Assessment

Yes. The source-level path is high confidence: a first group turn includes `groupIntro`, the follow-up omits it, and the current reuse/live-session identity checks treat that drift as a new prompt identity.

## Solution Assessment

Yes for a narrow prompt-identity fix. Dropping all prompt and MCP invalidation keys would be broader policy work; this issue can be fixed by separating turn-scoped group instructions from durable session identity.
