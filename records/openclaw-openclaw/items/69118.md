## Summary

Keep open. Current main still hashes first-turn groupIntro as part of CLI session reuse identity, then strictly invalidates the stored Claude CLI session when the ordinary follow-up turn omits that intro.

## What This Changes

The issue requests preserving Claude CLI session reuse across group-channel follow-up turns by preventing first-turn group intro text from drifting extraSystemPromptHash.

## Best Possible Solution

Keep group activation guidance in the actual CLI system prompt while making the reuse identity stable across ordinary group follow-up turns and covering the missing-static-prompt comparator edge with focused regression tests.

## Reproduction Assessment

Yes, at source level. A first group turn includes groupIntro in extraSystemPromptStatic, a follow-up group turn omits it, and resolveCliSessionReuse currently invalidates on that hash mismatch.

## Solution Assessment

Yes. The narrow maintainable fix is at the static prompt identity and reuse-comparison boundary; dropping all prompt/MCP reuse keys would be broader policy work unless maintainers explicitly choose that route.
