## Summary

Keep open. Current main still contains the reported false invalidation path, the latest release still has the strict `extraSystemPromptHash` comparator, and recent comments add live v2026.5.7 evidence for Discord and subagent-announcement resets.

## What This Changes

The issue reports Claude CLI session resets in group-style channels because turn-scoped prompt text changes the `extraSystemPromptHash` used for CLI session reuse.

## Best Possible Solution

Keep transient prompt overlays out of durable CLI session identity while preserving rotation for auth, MCP, model, workspace, skills, and genuinely stable prompt changes.

## Reproduction Assessment

Yes, this is source-reproducible on current main. A first group turn can include `groupIntro` in `extraSystemPromptStatic`, a follow-up can omit it, and `resolveCliSessionReuse` invalidates the stored CLI session when that hash changes.

## Solution Assessment

Yes, the requested direction is the right bug boundary. The safer implementation should exclude or normalize transient group/direct/inter-session overlays rather than special-casing one channel, while keeping durable identity checks strict.
