## Summary

Keep open. Current main still builds the CLI session reuse hash from turn-scoped group prompt material and still invalidates Claude CLI session reuse on any stored/current `extraSystemPromptHash` drift; recent comments add subagent-announce evidence for the same false `system-prompt` invalidation path.

## What This Changes

The issue reports Claude CLI session resets in group-channel and subagent-announcement flows when transient prompt overlays change `extraSystemPromptHash`.

## Best Possible Solution

Keep the behavioral prompt overlays in the actual CLI prompt, but remove transient group-intro and inter-session caller state from the stable session-reuse identity with regression coverage for group follow-up turns and subagent announcements.

## Reproduction Assessment

Yes, at source level. A first group turn can include `groupIntro` in `extraSystemPromptStatic`, a normal follow-up can omit it after `systemSent` is true, and the resolver invalidates the stored Claude CLI session when the resulting hash changes.

## Solution Assessment

Yes, the narrowest maintainable direction is to keep sending transient prompt context to the model while excluding it from session-reuse identity. Dropping every prompt or MCP reuse key would be broader than necessary unless maintainers explicitly choose that policy.
