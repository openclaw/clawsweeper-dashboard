## Summary

Keep open. Current main still hashes turn-scoped group intro and other static prompt overlays into Claude CLI session reuse identity, then invalidates the saved session on any stored/current prompt-hash drift; recent comments also confirm the same failure on v2026.5.7.

## What This Changes

The issue reports that Claude CLI group-channel follow-up turns recompute a different `extraSystemPromptHash` because transient group-intro/static prompt content changes between turns, causing session resets and lost context.

## Best Possible Solution

Separate delivered prompt overlays from durable CLI session identity, keep `groupIntro` and inter-session context in the actual model prompt, and add regression coverage for group follow-up turns plus subagent-announce/static-prompt gaps.

## Reproduction Assessment

Yes, at source level. A first group turn includes `groupIntro` in `extraSystemPromptStatic`, a normal follow-up can omit it, and the resolver invalidates the saved Claude CLI session when that hash changes.

## Solution Assessment

Yes. The narrowest maintainable solution is to stop using transient prompt overlays as hard session-reuse identity while retaining strict auth and real durable-context checks.
