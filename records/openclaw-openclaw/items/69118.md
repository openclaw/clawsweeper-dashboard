## Label Justifications

- `P1`: The report describes current multi-turn Claude CLI context loss in real Discord/Telegram-style workflows, which breaks active agent/channel sessions.
- `impact:session-state`: The false `system-prompt` reset changes the active CLI session binding and drops conversation continuity for affected turns.

## Summary

Keep open. Current main and latest release still hash turn-scoped group/direct prompt context into the Claude CLI session identity, and the resolver still invalidates on that hash drift; recent comments add live v2026.5.7 evidence for Discord, subagent announce, and cross-channel direct-chat triggers.

## What This Changes

The issue asks OpenClaw to stop Claude CLI session reuse from being invalidated when transient group, direct, or inter-session prompt context changes between turns.

## Best Possible Solution

Keep transient group, direct, and inter-session prompt overlays out of durable CLI session identity while preserving invalidation for auth, MCP, tool surface, workspace/model, and genuinely stable prompt changes.

## Reproduction Assessment

Yes. This is source-reproducible on current main: a first group turn can include `groupIntro` in `extraSystemPromptStatic`, a follow-up can omit it, `prepareCliRunContext` hashes the changed static prompt, and `resolveCliSessionReuse` invalidates the binding.

## Solution Assessment

Yes, the requested direction is the right bug boundary. The maintainable fix should normalize or exclude transient prompt overlays from the reuse hash rather than special-casing one channel or dropping all stable prompt invalidation.

## Repair Work Prompt

Fix the Claude CLI session continuity bug tracked by https://github.com/openclaw/openclaw/issues/69118. Keep transient prompt overlays such as group first-turn intro, direct-channel provider labels when `session.dmScope:"main"` shares one logical DM, inter-session/subagent announce envelopes, and missing/empty caller static prompt cases out of durable `extraSystemPromptHash` session identity. Preserve strict invalidation for auth profile/epoch, MCP resume/config identity, prompt tool surface, and genuinely stable prompt changes. Avoid a channel-specific workaround and do not fold the separate Claude live-session `systemPromptHash` PR into this fix unless the stable identity boundary must be shared. Add focused regression coverage for group turn 1 -> turn 2 reuse, subagent announce/direct agent calls, and cross-channel direct chat under `dmScope:"main"`; update `CHANGELOG.md` because this is user-visible.
