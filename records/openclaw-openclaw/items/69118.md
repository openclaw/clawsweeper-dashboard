## Summary

Keep open. Current main still includes the turn-scoped group intro in the CLI session reuse prompt hash, and the resolver still treats any stored/current prompt-hash drift as `system-prompt` invalidation; the merged adjacent PR explicitly did not cover this group-intro case.

## What This Changes

The issue requests that Claude CLI session reuse stop resetting across group-channel follow-up turns when `groupIntro` appears only on the first or re-intro turn.

## Best Possible Solution

Keep the first-turn group intro in the actual CLI system prompt, but remove turn-scoped or absent caller state from the stable reuse identity and add regression coverage for group follow-up turns and the related missing-static-prompt caller case.

## Reproduction Assessment

Yes, at source level. A first group turn includes `groupIntro` in `extraSystemPromptStatic`, a normal follow-up omits it after `systemSent` is true, and the resolver invalidates the stored CLI session when the hash changes.

## Solution Assessment

Yes. The narrowest maintainable fix is to keep delivering `groupIntro` to the model while excluding it, and similarly absent static context, from session-reuse identity; dropping all prompt/MCP reuse keys would be a broader policy change.
