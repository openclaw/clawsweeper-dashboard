## Summary

Keep open. Current main still includes the reported false invalidation path: first-turn groupIntro text is part of the static prompt identity, while follow-up group turns omit it and the CLI session resolver still resets on prompt-hash mismatch.

## What This Changes

The issue reports that Claude CLI group-channel sessions reset between consecutive turns because first-turn-only groupIntro text changes the extraSystemPromptHash used for CLI session reuse.

## Best Possible Solution

Keep groupIntro in the actual first-turn/re-intro system prompt, but exclude turn-scoped groupIntro from the stable prompt identity used for CLI session reuse and add a focused group turn-transition regression test.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: first group turns include `groupIntro` in `extraSystemPromptStatic`, follow-up group turns omit it, and the resolver returns `invalidatedReason: "system-prompt"` when hashes differ.

## Solution Assessment

Yes. A narrow fix at the stable prompt identity boundary is the best maintainable path for this issue; dropping all prompt/MCP reuse keys would be broader policy work.
