## Summary

Keep open: current main still puts turn-scoped group intro and other caller-dependent static prompt material into the CLI session reuse hash, and the resolver still invalidates on stored/current prompt-hash drift. Recent issue comments add live 2026.5.7 Discord and subagent-announce evidence, and the related open PR only addresses a separate live-session fingerprint issue.

## What This Changes

The issue requests preserving Claude CLI session continuity when group-channel or subagent-announcement turns change transient `extraSystemPromptHash` inputs.

## Best Possible Solution

Define a stable CLI session reuse identity that excludes transient group intro, inter-session announce overlays, and missing caller static context while preserving real auth/account invalidation, and add focused regression tests for group follow-up and subagent-announce turns.

## Reproduction Assessment

Yes, at source level: a first group turn includes `groupIntro` in `extraSystemPromptStatic`, a normal follow-up can omit it, and the resolver invalidates the stored CLI session on that hash drift. Recent comments also provide live 2026.5.7 logs for Discord and subagent-announce triggers, but I did not run a live gateway repro in this review.

## Solution Assessment

Yes. The narrow maintainable direction is to keep transient prompt overlays in the actual prompt sent to Claude while excluding them, and absent caller state, from the session-reuse identity; dropping unrelated identity checks would be broader than necessary.
