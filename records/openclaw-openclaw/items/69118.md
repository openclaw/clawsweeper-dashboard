## Summary

Keep open. Current main still has a source-proven `extraSystemPromptHash` false-invalidation path for Claude CLI sessions, and the latest discussion adds fresh v2026.5.7 evidence plus an open partial PR that only covers the direct-channel label trigger.

## What This Changes

Ensure Claude CLI sessions keep continuity across follow-up group, direct, and inter-session turns instead of resetting when transient prompt context changes `extraSystemPromptHash`.

## Best Possible Solution

Keep transient group, direct-channel, inter-session, and per-turn prompt overlays out of durable CLI session identity while preserving strict auth and genuinely durable runtime identity checks, with regressions for group turn 1 to turn 2, subagent announcements, and dmScope=main channel switches.

## Reproduction Assessment

Yes, at source level. A first group turn can include `groupIntro` in `extraSystemPromptStatic`, a follow-up can omit it, and the resolver invalidates on the resulting prompt hash mismatch; recent comments add matching live logs.

## Solution Assessment

Yes, the requested direction is the right bug boundary. The narrow maintainable solution is to normalize or exclude transient prompt overlays from the reuse hash rather than special-case one channel trigger.
