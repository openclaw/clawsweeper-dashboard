## Summary

Keep open. Current main still includes turn-scoped group and inter-session prompt overlays in the Claude CLI session-reuse identity path, and the resolver still invalidates saved sessions on `extraSystemPromptHash` drift; recent issue comments add live v2026.5.7 evidence for the same user-visible reset.

## What This Changes

The issue asks OpenClaw to preserve Claude CLI session continuity when group-channel or inter-session prompt overlays change between turns.

## Best Possible Solution

Separate delivered per-turn prompt overlays from durable Claude CLI session identity while preserving invalidation for auth, MCP, model, workspace, skills, and genuinely stable system-prompt changes.

## Reproduction Assessment

Yes, by source inspection rather than a live run. A first group turn can include `groupIntro` in `extraSystemPromptStatic`, a follow-up can omit it, and current `resolveCliSessionReuse` invalidates the saved CLI session when that hash changes.

## Solution Assessment

Yes, the requested direction is the right bug boundary. The maintainable fix should exclude transient group/direct/inter-session overlays from session identity while keeping real stable prompt and runtime identity changes as rotation triggers.
