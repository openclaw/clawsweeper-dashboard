## Summary

Keep open. Current main still puts turn-scoped group intro and other static prompt overlays into the Claude CLI session-reuse hash, and the resolver still drops the saved session when that stored hash differs from the next turn; recent comments confirm the same user-visible reset on v2026.5.7.

## What This Changes

The issue asks OpenClaw to stop resetting Claude CLI sessions when group-channel follow-up turns legitimately change `extraSystemPromptHash`.

## Best Possible Solution

Separate delivered prompt overlays from durable Claude CLI session identity, keep group/direct/inter-session guidance in the actual model prompt, and add regression coverage for group follow-up turns plus subagent-announce delivery.

## Reproduction Assessment

Yes, at source level. A first group turn includes `groupIntro` in `extraSystemPromptStatic`, a normal follow-up can omit it, and the resolver invalidates the saved Claude CLI session when that hash changes.

## Solution Assessment

Yes. The narrowest maintainable fix is to stop treating transient prompt overlays as hard CLI session identity while keeping strict checks for auth and genuinely durable runtime context changes.
