## Summary

Keep open. The intended glossary update is plausible but still needs terminology review, and the branch also contains an unrelated git-plugin install change that breaks explicit ref checkouts.

## What This Changes

This PR updates Chinese docs glossary JSON entries and also changes git plugin ref checkout/tests to insert a `--` separator before the requested ref.

## Best Possible Solution

Split or remove the unrelated git-install commit, let https://github.com/openclaw/openclaw/pull/79901 handle git ref hardening, and align glossary edits with the docs-i18n terminology policy before merge.

## Reproduction Assessment

Yes for the main-reported glossary state: current main shows the zh-CN and zh-TW mappings described by https://github.com/openclaw/openclaw/issues/79935. The PR-introduced git checkout regression is also reproducible in a temporary Git repo.

## Solution Assessment

No. The glossary direction needs terminology alignment, and the included `git checkout --detach -- <ref>` change is not the safe git-ref fix; the safer path is to split it out and use the `git switch --detach -- <ref>` direction in https://github.com/openclaw/openclaw/pull/79901.

## Review Findings
