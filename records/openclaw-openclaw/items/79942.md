The branch updates Chinese docs glossary JSON entries and also changes git plugin ref checkout/tests to add an option separator.

## Best Possible Solution

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use a valid ref-only git checkout form:** `src/plugins/git-install.ts:290`
  - body: The inserted `--` is in the pathspec separator position for `git checkout`, not a revision separator. In a temporary repo, `git checkout --detach -- v1.2.3` exits 128 with `--detach does not take a path argument`, so any `git:<repo>@v1.2.3` install would now fail. Use the verified `git switch --detach -- <ref>` direction from the focused git-install PR, or remove this unrelated change here.
  - confidence: 0.97
- **[P2] Keep Gateway terminology rules consistent:** `docs/.i18n/glossary.zh-CN.json:84`
  - body: This changes the zh-CN glossary target for `Gateway` to `网关`, but the docs-i18n prompt still requires `Gateway 网关` for the OpenClaw Gateway. That leaves the translation pipeline with conflicting mandatory terminology rules, so the prompt and glossary should be changed together only after the terminology decision is made.
  - confidence: 0.86

## Security Review
