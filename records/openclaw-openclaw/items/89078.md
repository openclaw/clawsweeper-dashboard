## Summary

Keep open: the PR is a useful direction, but it only updates `sendMessage()` while normal typed TUI prompts are still blocked by the unchanged submit gate before they reach that function, and the new session row value does not represent the Gateway's effective queue mode.

## What This Changes

The PR adds `queueMode` to Gateway session rows and TUI session state, then gates the TUI `sendMessage` busy check on `queueMode === "steer"`.

## Best Possible Solution

Let both typed-submit and slash-command TUI paths reach Gateway queue handling for active-run prompts while keeping only duplicate-submit and stop/Esc protections local.

## Reproduction Assessment

Yes. Source inspection shows normal typed messages still return false from `canSubmitTuiChatMessage` for gateway-mode active runs before `sendMessage` can apply the PR's queue-mode-aware guard.

## Solution Assessment

No. Updating `sendMessage` is only a partial fix because the caller-side submit gate remains unchanged and the propagated field is not the Gateway's effective queue mode.
