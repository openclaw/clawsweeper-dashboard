## Summary

Current `main` still passes unresolved configuration directly into the Nextcloud Talk doctor preview, which dereferences `botSecret` and can throw the exact reported error. The latest release, `v2026.6.5`, contains the same path, and no separate canonical fix was found.

**Likely related people**
- Vincent Koc — high-confidence recent contributor to the Nextcloud Talk doctor/account path.
- Josh Avant — medium-confidence contributor to the shared SecretRef coverage.
- Peter Steinberger — medium-confidence contributor to shared command configuration resolution.

## What This Changes

Fix `openclaw doctor --non-interactive` so a working file-backed Nextcloud Talk `botSecret` SecretRef does not abort the diagnostic run.

## Best Possible Solution

Provide channel doctor previews with a configuration snapshot resolved through the generic command/gateway SecretRef boundary, while retaining clear errors when a required secret is genuinely unavailable and avoiding channel-specific core exceptions.

## Reproduction Assessment

Yes at source level. Core doctor passes raw config to the Nextcloud Talk preview, which resolves the account credential and triggers the exact unresolved-SecretRef exception reported by the user.

## Solution Assessment

Yes in direction, but the generic boundary should own the fix. Reusing command-scoped gateway resolution is narrower and more maintainable than adding a Nextcloud-specific exception or treating unresolved credentials as valid strings.
