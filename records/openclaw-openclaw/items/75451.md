## Summary

Keep this PR open. Current main still repeats plugin discovery in the scoped capability retry path, while the PR’s explicit caller-owned discovery threading matches the documented plugin cache boundary and is not already implemented on main.

## What This Changes

The PR branch adds optional `PluginDiscoveryResult` plumbing to bundled plugin helper APIs, reuses one discovery result across scoped capability retry attempts, and adds a plugin performance changelog entry.

## Best Possible Solution

Land the explicit current-flow discovery reuse after CI, keeping `discoverOpenClawPlugins()` stateless and avoiding any module-level or time-window cache; a small retry-reuse assertion would make the performance contract easier to preserve.

## Reproduction Assessment

Yes. On current main, the static reproduction is to force the first scoped capability runtime load to miss or fail; `loadScopedCapabilityRuntimeRegistryEntries` retries and calls `loadBundledCapabilityRuntimeRegistry` twice, and that helper performs discovery internally each time.

## Solution Assessment

Yes. The PR’s optional parameter and function-scoped reuse is the narrow maintainable solution because it removes duplicate scans for callers that already own current-flow discovery without changing freshness semantics or adding a hidden cache.
