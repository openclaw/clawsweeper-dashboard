## Summary

Keep this PR open. Current main still performs plugin discovery inside the bundled helpers and repeats the bundled runtime loader inside the scoped contract retry path, so the proposed perf change is not already implemented or superseded; the final diff shape is narrow and I found no blocking review finding in the provided patch plus current-main inspection.

## What This Changes

The PR adds optional caller-provided `PluginDiscoveryResult` parameters to bundled plugin helper APIs, reuses one function-scoped discovery result across scoped capability retry attempts, and adds a changelog entry.

## Best Possible Solution

Land a narrow explicit-discovery threading change that preserves stateless discovery, keeps reuse scoped to caller-owned current-flow data, covers the contract retry path with focused tests or existing CI, and keeps the user-facing changelog entry.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: force the first scoped capability load to miss or error, and the retry loop calls `loadBundledCapabilityRuntimeRegistry` twice; each call performs its own `discoverOpenClawPlugins` scan.

## Solution Assessment

Yes. The explicit optional parameter and function-scoped retry reuse are the narrow maintainable solution because they avoid a hidden module-level cache while still letting a caller share one current-flow discovery result.
