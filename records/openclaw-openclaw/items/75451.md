## Summary

Keep open. Current main still performs redundant plugin discovery in the scoped capability retry path, and this PR is a narrow explicit-object reuse that matches the documented cache boundary; I did not find a discrete diff bug, but it still needs maintainer review and exact-head validation before merge.

## What This Changes

The branch threads an optional `PluginDiscoveryResult` through bundled plugin source, channel catalog, and capability runtime helpers, then reuses one discovery result across scoped capability registry retry attempts.

## Best Possible Solution

Land the explicit discovery-result plumbing once exact-head CI and changed-gate proof are green, preserving stateless discovery and avoiding hidden metadata caches.

## Reproduction Assessment

Yes. On current main, the scoped capability retry loop can call `loadBundledCapabilityRuntimeRegistry` twice, and that helper performs its own `discoverOpenClawPlugins` scan each time.

## Solution Assessment

Yes. Passing an explicit per-flow discovery result is the narrow maintainable fix because it avoids a hidden cache while removing duplicate filesystem scans in the retry path.
