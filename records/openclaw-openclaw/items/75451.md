## What This Changes

The branch adds optional `PluginDiscoveryResult` parameters to three plugin helper APIs and reuses one discovery result across the scoped capability registry retry loop, plus a changelog entry.

## Best Possible Solution

Land the function-scoped discovery threading after the contributor adds redacted real setup proof and the exact PR head passes the relevant plugin registry gates.

## Reproduction Assessment

Yes. Source inspection on current main shows the scoped capability retry loop can call the bundled capability registry twice, and each registry load performs plugin discovery.

## Solution Assessment

Yes. Passing an explicit `PluginDiscoveryResult` through short-lived helper parameters matches the documented cache boundary and avoids a hidden global discovery cache; the remaining blocker is proof, not a better design found in review.
