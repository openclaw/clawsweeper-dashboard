## Summary

Keep this PR open. Current main still repeats plugin discovery in the scoped capability retry path, and the PR is a narrow, backward-compatible performance change rather than obsolete cleanup; I found no blocking correctness or security issue from the provided diff and local current-main inspection.

## What This Changes

The PR adds optional `PluginDiscoveryResult` plumbing to bundled runtime, bundled source, and channel catalog helpers, reuses one discovery result across scoped capability retry attempts, and adds a changelog entry.

## Best Possible Solution

Land the explicit current-flow discovery reuse after CI, keeping `discoverOpenClawPlugins()` stateless and avoiding any module-level discovery cache; a small retry-reuse regression test would be a useful maintainer choice but is not a blocking defect in this XS patch.

## Reproduction Assessment

Yes. On current main, force the first scoped capability runtime load to miss or fail; `loadScopedCapabilityRuntimeRegistryEntries` calls `loadBundledCapabilityRuntimeRegistry` twice, and that helper performs its own `discoverOpenClawPlugins` call each time.

## Solution Assessment

Yes. The proposed optional `PluginDiscoveryResult` parameter is the narrow maintainable path because it lets callers share a current-flow object while preserving fresh discovery for callers that do not pass one, matching the documented plugin cache boundary.
