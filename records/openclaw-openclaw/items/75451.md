## Summary

Keep open for normal PR review: the explicit per-flow discovery threading fits the plugin cache-boundary policy, but the contributor proof is still only typecheck/Vitest output and the new channel-catalog fast path should avoid an unused installed-index read.

## What This Changes

The branch adds optional `PluginDiscoveryResult` parameters to bundled plugin helpers, reuses one discovery result in the scoped capability runtime retry loop, and adds a changelog entry.

## Best Possible Solution

Land the explicit function-scoped discovery reuse after redacted runtime or instrumentation proof is added and the supplied-discovery path avoids unnecessary ledger reads.

## Reproduction Assessment

Yes, at source level: current main's scoped retry path can call `loadBundledCapabilityRuntimeRegistry` twice, and that helper performs its own `discoverOpenClawPlugins` call. I did not run a startup benchmark because this review is read-only.

## Solution Assessment

Mostly yes: explicit caller-owned discovery matches the documented cache boundary and avoids hidden caches. The patch should also skip channel-catalog ledger lookup when `discovery` is supplied and needs real behavior proof before merge.

## Review Findings

Overall correctness: patch is correct
