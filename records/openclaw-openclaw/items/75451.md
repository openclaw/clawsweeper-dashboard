## Summary

Keep this PR open: current main still repeats plugin discovery through the scoped capability retry path, and the PR's explicit current-flow discovery threading is a narrow fit for the documented plugin cache boundary.

## What This Changes

The PR adds optional `PluginDiscoveryResult` parameters to bundled plugin helper APIs, reuses one discovery result across scoped capability retry attempts, and adds a changelog entry for the plugin performance improvement.

## Best Possible Solution

Land the explicit current-flow discovery reuse approach after rebase/CI, keeping `discoverOpenClawPlugins()` stateless and avoiding any module-level discovery cache.

## Reproduction Assessment

Yes. The static reproduction is current main's scoped retry helper: force the first bundled capability load to miss or fail, and the loop calls `loadBundledCapabilityRuntimeRegistry` twice; that helper performs its own discovery scan on each call.

## Solution Assessment

Yes. The PR's optional parameter and function-scoped reuse are the narrow maintainable fix; a hidden cache would violate the documented plugin cache boundary.
