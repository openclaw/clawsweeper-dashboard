## Summary

Keep this PR open: current main still repeats plugin discovery through the scoped capability retry path, and the proposed explicit discovery threading is narrow, backwards-compatible, and aligned with the documented plugin cache boundary. I found no blocking review findings in the provided diff, but rebase and CI should still gate merge.

## What This Changes

The PR adds optional `PluginDiscoveryResult` plumbing to bundled plugin helper APIs and reuses one discovery result across scoped capability retry attempts, plus a changelog entry.

## Best Possible Solution

Land the explicit current-flow discovery reuse after rebase and CI, keeping `discoverOpenClawPlugins()` stateless and avoiding any module-level discovery cache.

## Reproduction Assessment

Yes. The static reproduction is current main's scoped capability retry helper: force the first bundled capability load to miss or fail, and the retry loop calls the bundled runtime loader twice; that loader performs discovery internally each time.

## Solution Assessment

Yes. The PR's optional parameter and function-scoped reuse are the narrow maintainable fix because they avoid hidden cache state while preserving backwards-compatible helper calls.
