## Summary

Keep open: current main still repeats plugin discovery in the scoped capability retry path, and this PR's explicit reuse matches the documented plugin cache boundary; I found no blocking correctness or security findings.

## What This Changes

The PR adds optional `PluginDiscoveryResult` plumbing to bundled plugin helper APIs and reuses one discovery result across scoped capability retry attempts, with a changelog entry.

## Best Possible Solution

Land the explicit current-flow discovery reuse after maintainer review and CI, keeping `discoverOpenClawPlugins()` stateless and avoiding any module-level discovery cache.

## Reproduction Assessment

Yes. The static reproduction is current main's scoped capability retry helper: force the first scoped provider/web capability load to miss or fail, and the loop calls the bundled runtime loader twice; that loader performs discovery internally on each call.

## Solution Assessment

Yes. The optional parameter and function-scoped reuse are the narrow maintainable fix for this path, while a hidden discovery cache would conflict with the documented plugin cache boundary.
