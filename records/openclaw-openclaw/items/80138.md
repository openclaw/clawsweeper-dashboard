## Summary

Keep this PR open: it addresses a real Codex auth repair bug, but the latest head still introduces a blocking model-visibility regression by using `agents.defaults.models` as runtime-policy storage.

## What This Changes

The PR rewrites legacy `openai-codex/*` doctor repairs to canonical `openai/*`, adds model-scoped Codex runtime policy, preserves model-map metadata, and updates tests, docs, and changelog.

## Best Possible Solution

Preserve Codex auth during doctor repair while keeping model visibility unchanged, either by separating runtime-policy storage from visibility or by making visibility semantics ignore runtime-only entries.

## Reproduction Assessment

Yes, source-reproducible: PR head creates `agents.defaults.models` for a bare default-agent repair, and current main treats any default model-map entry as an allowlist. I did not run tests because this review is read-only.

## Solution Assessment

No. Preserving Codex auth intent is the right direction, but storing that intent by creating `agents.defaults.models` changes model visibility and is not the narrowest maintainable fix.
