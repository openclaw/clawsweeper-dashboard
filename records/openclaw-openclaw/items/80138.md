## Summary

Keep open. The latest head has sufficient real CLI proof and fixed the earlier channel-only repair problem, but it still stores default-agent Codex runtime policy by creating `agents.defaults.models` entries that current main treats as a model allowlist, and it can still overwrite an explicit canonical per-model runtime policy during repair.

## What This Changes

The PR changes doctor route repair to canonicalize legacy `openai-codex/*` refs to `openai/*`, add Codex runtime policy where needed, preserve model-map metadata, and update related tests, docs, and changelog.

## Best Possible Solution

Keep the Codex auth-preserving repair, but store runtime intent without silently narrowing `agents.defaults.models` visibility and avoid overwriting explicit canonical model runtime policy; then rebase and land with the existing doctor/runtime coverage.

## Reproduction Assessment

Yes for the PR-introduced regression: source inspection shows the helper can create `agents.defaults.models` for a bare legacy default-agent ref, and current main treats any default model-map key as a configured allowlist. I did not run tests because this review is read-only.

## Solution Assessment

No. Preserving Codex auth intent is the right direction and the latest head fixed the channel-only path, but default-agent repair should not use hidden allowlist state or clobber explicit per-model runtime policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Avoid creating an implicit model allowlist:** `src/commands/doctor/shared/codex-route-warnings.ts:490-496`
  - body: When a config only has a legacy `agents.defaults.model`, this helper creates `agents.defaults.models` solely to store Codex runtime policy. Current model selection treats any default model-map key as an explicit allowlist, so `doctor --fix` can unexpectedly make previously valid `/model` switches or session overrides outside the repaired OpenAI keys unavailable.
  - confidence: 0.93
- **[P2] Preserve explicit canonical model runtime policy:** `src/commands/doctor/shared/codex-route-warnings.ts:498-504`
  - body: After a legacy ref maps to an existing canonical model entry, this always writes `agentRuntime.id = "codex"` unless it was already Codex. If the canonical row explicitly selected `pi`, doctor silently flips a user-authored provider/model policy; preserve that policy or leave the contradictory legacy ref for manual repair.
  - confidence: 0.86
