## Summary

This PR should stay open because it addresses a real Codex auth-routing repair bug, but the current patch introduces a blocking model-visibility regression by materializing `agents.*.models` maps during repair. The existing automated P1 review comment matches the current source contracts, so the branch needs a focused fix before merge.

## What This Changes

The PR rewrites legacy `openai-codex/*` agent refs to canonical `openai/*` while adding model-scoped Codex runtime policy, preserving copied model-map metadata, and updating doctor/runtime docs plus the changelog.

## Best Possible Solution

Keep the Codex auth-preservation behavior, but store or resolve the repaired Codex runtime policy without turning an absent model map into an explicit model allowlist; add regression coverage for the unrestricted-config case.

## Reproduction Assessment

Yes for the PR-introduced regression: the diff creates model-map entries, and current main treats any default model-map entry as configured visibility. I did not run a live command because this review is read-only and doctor/test commands may write artifacts.

## Solution Assessment

No. Preserving Codex auth intent is the right direction, but using `agents.*.models` as the unconditional storage vehicle is not the narrowest safe fix because that field also controls model visibility.
