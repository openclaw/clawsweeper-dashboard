## Summary

Keep open: this fixes a real Codex route-repair bug and has sufficient terminal proof, but the latest head still stores new runtime intent by creating `agents.defaults.models` for bare default-agent repairs, which changes documented model visibility because that map is the `/model` and session override allowlist.

## What This Changes

The PR changes doctor route repair to canonicalize legacy `openai-codex/*` agent model refs to `openai/*`, add Codex runtime policy, preserve copied model-map metadata, and update related tests, docs, and changelog.

## Best Possible Solution

Preserve Codex auth during doctor repair without silently creating model visibility restrictions, then align docs and tests with the chosen canonical route contract before closing the linked bugs.

## Reproduction Assessment

Yes for the PR-introduced regression: PR head creates `agents.defaults.models` for a bare legacy default-agent ref, and current main treats any default model-map entry as an allowlist. I did not run tests because this review is read-only.

## Solution Assessment

No. Preserving Codex auth intent is the right direction and the latest head fixed the explicit-runtime overwrite, but storing new runtime intent in `agents.defaults.models` is not the narrowest maintainable fix because it changes model visibility.
