## What This Changes

The issue requests a Docker/Compose-friendly way to configure `gateway.controlUi.allowedOrigins` for Portainer-style remote deployments without running an extra `config set` command.

## Best Possible Solution

Choose a maintainer-approved Docker/Compose contract, preferably strict JSON with fail-closed validation, clear precedence, no silent overwrite of existing origins, and updated Docker/security docs.

## Reproduction Assessment

Yes, source-level: current main exposes no Compose/env input for non-localhost `gateway.controlUi.allowedOrigins`, while startup still relies on configured origins or deliberate fallback. I did not run Docker because this review is read-only.

## Solution Assessment

Unclear until maintainers choose the contract. A setup-time strict-JSON persistence path looks safer than a runtime shim, but the final shape needs security and config-surface approval.
