## Summary

Keep open. Current main still rejects the `/activate` alias in both the activation parser and the command detection registry, and this issue is already paired with an open fix PR using closing syntax, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks for WhatsApp group activation commands to accept `/activate mention` and `/activate always` as aliases for `/activation mention` and `/activation always`.

## Best Possible Solution

Land or replace the paired PR with a narrow fix that accepts `/activate` and `/activation`, covers both parser and control-command detection behavior, preserves current `/activation` behavior, and adds focused tests for the alias in group-command routing.

## Reproduction Assessment

Yes. Current main has a direct source-level reproduction: `/activation mention` matches the parser and registry path, while `/activate mention` has neither a parser match nor a registered control-command alias.

## Solution Assessment

Yes. Supporting `/activate` as a compatibility alias is the narrow maintainable solution, but the implementation should cover both `parseActivationCommand` and the command alias/detection path used before WhatsApp group mention gating.
