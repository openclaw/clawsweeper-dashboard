## Summary

Keep this PR open: it addresses a real source-backed Codex local probe failure, but the current patch is still too broad because it adds instructions to every local provider probe instead of only the OpenAI Codex Responses path.

## What This Changes

The PR adds a minimal system prompt to local `infer model run` calls, includes provider `errorMessage` details in empty-output errors, updates CLI tests, and adds a changelog entry.

## Best Possible Solution

Keep the provider error surfacing, but add instructions only when the prepared local model uses the OpenAI Codex Responses provider/API and preserve raw local contexts for other providers.

## Reproduction Assessment

Yes, source-backed: current main passes only `messages` to local simple completion, while the Codex Responses transport emits `instructions` only from `context.systemPrompt`; the linked issue supplies the live backend `Instructions are required` result. I did not rerun the OAuth-backed live probe in this read-only review.

## Solution Assessment

No, not as written. Supplying instructions fixes the Codex path, but applying the prompt to all local providers is broader than the reported bug and conflicts with the documented raw local probe behavior.
