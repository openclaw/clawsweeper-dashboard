- **[P1] Narrow artifact matching to protocol-only markers:** `src/auto-reply/tokens.ts:24`
  - body: This regex treats any standalone `---`, `___`, `***`, `<tag>`, `</tag>`, or `<br/>` as an internal artifact. Because `normalizeReplyPayload` returns `null` for matches, a valid assistant reply containing only one of those strings would be silently dropped on every channel, not only Telegram. Limit this to observed internal markers or add provider/runtime context before suppressing ambiguous text.
  - confidence: 0.87

  - body: The bug fix does not need a new public plugin API, but exporting `isInternalFormattingArtifact` from the broad root SDK makes this helper a supported plugin contract. The scoped SDK guide requires narrow documented entrypoints and API-baseline alignment for new surface; remove this export unless maintainers explicitly approve and document the contract.
  - confidence: 0.84
