## Summary

Keep this PR open for maintainer review rather than merging or closing it in this sweep: current main already addresses the linked model-catalog hot-reload bug, while this broader registry still has a context-window stale-write race and lacks real behavior proof.

## What This Changes

The PR adds a config-derived cache registry and wires Gateway hot reload to invalidate model catalog, models.json readiness, and context-window caches on model config changes.

## Best Possible Solution

Keep current main's targeted model-catalog hot-reload handling unless maintainers explicitly want a broader registry; any registry should add generation/cancellation protection and include real hot-reload proof.

## Reproduction Assessment

Yes, by source inspection of the PR diff against current main. The new invalidator clears shared context state while an existing ensureContextWindowCacheLoaded task can still resume after awaits and write stale data.

## Solution Assessment

No, not in this form. Current main already covers the original model-catalog bug, and broadening invalidation to the context-window cache needs a generation or cancellation guard first.
