## Summary

Keep open. The runtime change is the right narrow fix for the linked gateway/send-policy bug, but the PR appears to introduce failing regression tests because the new assertions use `mocks.agentCommand` call counts without clearing the mock in a suite that does not reset that mock between tests.

## What This Changes

The PR gates the gateway `agent` request handler's `sendPolicy=deny` rejection behind `request.deliver === true`, adds two gateway regression tests, and adds a changelog fix entry for #73381.

## Best Possible Solution

Keep the PR's gateway/runtime contract alignment, but adjust the regression tests so they isolate `mocks.agentCommand` call history before asserting call counts; after that, validate the focused gateway test and changed gate before merge so #73381 closes through the PR relationship.

## Reproduction Assessment

Yes. Current main can be reproduced by forcing `resolveSendPolicy` to return `deny` for a non-delivery gateway `agent` invocation; source inspection shows the gateway rejects before computing delivery intent, and #73381 includes concrete CLI reproduction details.

## Solution Assessment

Mostly yes, but not as submitted. The runtime code change is the narrow maintainable fix, while the PR tests need a small isolation repair so their assertions check only the invocation under test.
