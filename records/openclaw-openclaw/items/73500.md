The PR scopes gateway `agent` send-policy denial to `request.deliver === true`, makes `resolveSendPolicy` mockable in gateway agent tests, adds allow/deny regression coverage, and adds a changelog fix entry for #73381.

## Best Possible Solution

Keep the production guard that scopes gateway send-policy denial to explicit delivery, fix the new test by clearing or snapshotting `mocks.agentCommand` before the denial assertion, then run the targeted gateway test and the normal changed gate before merge.

Yes. Current main can reproduce the reported bug by forcing `resolveSendPolicy` to return `deny` for a non-delivery gateway `agent` request; the handler rejects before it computes delivery intent, and #73381 includes matching CLI steps and a diagnostic patch.
