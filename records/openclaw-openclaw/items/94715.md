## What This Changes

The PR changes Nextcloud Talk webhook decoding to parse JSON before strict Note schema validation and adds replay tests for malformed JSON, Document events, and setup pings.

## Best Possible Solution

Keep the JSON-before-schema split, but classify supported chat messages by top-level ActivityStreams type, acknowledge valid non-message hooks including Like, Undo, and setup without dispatch, preserve 400 for malformed JSON or malformed chat-message payloads, and require signed-webhook or live Talk proof.

## Reproduction Assessment

## Solution Assessment

No, not yet. JSON-before-schema is the right layer, but the nested object.type classifier is too broad for official Like/Undo reaction hooks; the durable fix should classify by top-level activity and cover signed reaction fixtures.
