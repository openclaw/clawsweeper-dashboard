## Summary

Keep open: the PR fixes the Document/setup-ping part of the Nextcloud Talk webhook bug, but the new classifier still returns HTTP 400 for valid reaction-added hooks that are top-level `Like` activities wrapping a `Note`, and the supplied proof does not exercise the patched signed webhook server or a live Talk server.

## What This Changes

The PR changes the Nextcloud Talk webhook monitor to parse JSON before schema validation, return 200/no dispatch for non-Note payloads, and add signed replay tests for malformed JSON, malformed Note payloads, Document events, and setup pings.

## Best Possible Solution

Keep the two-stage parse, classify supported chat messages by top-level Create plus Note, return 200/no dispatch for other valid signed Talk ActivityStreams hooks, and add signed Like-hook real-path proof.

## Reproduction Assessment

Yes, source-level: current main and v2026.6.8 map signed schema misses to HTTP 400 before ignore handling. PR head still sends official Like reaction-added hooks to that 400 branch because they fail the strict schema while carrying object.type Note.

## Solution Assessment

No, not yet. JSON-before-schema is the right layer, but the durable fix should classify supported chat messages by top-level Create plus Note and acknowledge other valid signed ActivityStreams hooks while preserving 400 for malformed JSON or malformed message-shaped payloads.
