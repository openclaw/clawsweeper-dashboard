Keep open: the PR is a useful narrow fix for the linked Nextcloud Talk webhook bug, but the latest head still rejects documented reaction webhook shapes and the supplied proof does not run the real signed webhook server or a live Nextcloud Talk path.

## What This Changes

The branch changes Nextcloud Talk webhook decoding to parse signed JSON before schema validation, return 200/no dispatch for valid non-Note events, and add replay tests for malformed JSON, malformed Note payloads, file-share events, and setup pings.

## Best Possible Solution

Keep the JSON-before-schema split, but classify supported chat messages by top-level ActivityStreams shape so non-message hooks return 200/no dispatch while malformed supported messages still return 400, with signed fixtures and real webhook-path proof.

## Reproduction Assessment

Yes, source-level. Current main and v2026.6.8 parse signed webhook bodies through the strict Note schema and write HTTP 400 for schema misses before any ignore branch; I did not run a live Nextcloud server in this read-only review.

## Solution Assessment

No, not yet. JSON-before-schema is the right layer, but object.type alone is too broad for malformed-message detection because documented reaction hooks embed Note objects inside non-message activities.
