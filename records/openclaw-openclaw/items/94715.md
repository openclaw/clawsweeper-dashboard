## Summary

Keep open: the patch is aimed at the right parser boundary, but it still rejects official reaction-added non-message webhooks and the supplied proof does not exercise the real signed webhook path.

## What This Changes

The PR changes Nextcloud Talk webhook decoding to parse JSON before schema validation, return 200/no dispatch for valid non-Note events, and add replay tests for malformed JSON, malformed Note payloads, Document events, and setup pings.

## Best Possible Solution

Keep the JSON-before-schema split, classify by top-level ActivityStreams event so known non-message hooks return 200/no dispatch while malformed supported messages still return 400, add signed fixtures, and require real webhook-path proof before merge.

## Reproduction Assessment

Yes, source-level. Current main and v2026.6.8 parse signed webhook bodies through the strict Note schema and write HTTP 400 for schema misses before any ignore branch; I did not run a live Nextcloud server in this read-only review.

## Solution Assessment

No, not yet. JSON-before-schema is the right layer, but object.type alone is too broad for malformed-message detection because official reaction hooks embed Note objects inside non-message activities.
