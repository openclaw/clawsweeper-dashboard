## Summary

Keep open: the JSON-before-schema split is the right layer, but the current classifier still rejects documented reaction-added hooks and the supplied proof does not exercise the real signed webhook server or a live Nextcloud Talk path.

## What This Changes

The PR changes the Nextcloud Talk webhook decoder to parse JSON before schema validation, return 200/no dispatch for valid non-message events, and add signed replay tests for malformed JSON, malformed Note payloads, Document events, and setup pings.

## Best Possible Solution

Keep the JSON-before-schema split, classify by top-level ActivityStreams event so known non-message hooks return 200/no dispatch while malformed supported messages still return 400, add signed fixtures, and require real webhook-path proof before merge.

## Reproduction Assessment

Yes, source-level: current main parses signed webhook bodies through the strict Note schema and writes HTTP 400 for schema misses before any ignore branch. I did not run a live Nextcloud server in this read-only review.

## Solution Assessment

No, not yet. JSON-before-schema is the right layer, but object.type alone is too broad for malformed-message detection because documented reaction hooks embed Note objects inside non-message activities.
