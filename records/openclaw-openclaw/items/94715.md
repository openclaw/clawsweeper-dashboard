## Summary

## What This Changes

The PR parses Nextcloud Talk webhook bodies as JSON before schema validation, returns 200/no dispatch for non-Note events, preserves 400 for malformed JSON or Note-shaped schema misses, and adds signed replay tests for Document/setup payloads.

## Best Possible Solution

Keep the JSON-before-schema split, classify supported chat messages by top-level Create plus Note shape, acknowledge other signed ActivityStreams hooks with 200/no dispatch, and add signed Like-hook runtime proof.

## Reproduction Assessment

Yes, source-level. Current main and v2026.6.8 map signed schema misses to HTTP 400, and PR head still sends official Like reaction-added hooks to the malformed-message 400 branch; I did not execute a live Nextcloud server in this read-only review.

## Solution Assessment

No, not yet. JSON-before-schema is the right layer, but the nested object.type classifier is too broad for official Like reaction hooks; the durable fix should classify by top-level activity and cover signed reaction fixtures.
