## Summary

Keep open: this is a useful candidate for the linked Nextcloud Talk webhook bug, but it is not merge-ready because the new classifier still sends valid reaction-added webhooks to HTTP 400 and the supplied proof does not exercise the patched signed webhook path.

## What This Changes

The PR changes Nextcloud Talk webhook decoding to parse JSON before schema validation, ignore valid non-Note webhook events with HTTP 200, preserve HTTP 400 for malformed JSON/message payloads, and add replay tests for malformed JSON, Document events, and setup pings.

## Best Possible Solution

Keep the JSON-before-schema split, classify by top-level ActivityStreams type so signed non-message events including Like reactions return 200 without dispatch, preserve 400 for malformed JSON or malformed Create/Note messages, and require signed-webhook or live Talk proof.

## Reproduction Assessment

Yes, source-level: current main and v2026.6.8 map signed schema misses to HTTP 400 before classifying valid non-message events, and PR head still sends top-level Like reaction hooks with embedded Note objects to HTTP 400. I did not run tests in this read-only review.

## Solution Assessment

No, not yet. JSON-before-schema is the right layer, but the nested object.type classifier is too broad for official reaction-added hooks; the durable fix should classify by top-level activity and cover signed Like fixtures.
