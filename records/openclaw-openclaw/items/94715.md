## Summary

## What This Changes

The PR changes Nextcloud Talk signed webhook parsing to parse JSON before strict message schema validation and adds replay tests for malformed JSON, malformed Note payloads, Document events, and setup pings.

## Best Possible Solution

Keep the JSON-before-schema split, classify supported chat messages by top-level ActivityStreams event plus strict schema, acknowledge valid non-message hooks including Like/Undo/setup without dispatch, preserve 400 for malformed JSON or malformed chat-message payloads, and require production-path proof.

## Reproduction Assessment

Yes, source-level: current main and v2026.6.8 map signed schema misses to HTTP 400 before classifying valid non-message events. PR-head source also shows Like-with-embedded-Note reaction hooks would still hit the new 400 guard.

## Solution Assessment

No, not yet. JSON-before-schema is the right layer, but the nested object.type classifier is too broad for official Like/Undo reaction hooks; the durable fix should classify by top-level ActivityStreams type and cover signed reaction fixtures.
