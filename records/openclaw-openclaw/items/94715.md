This PR changes Nextcloud Talk webhook decoding to parse signed JSON before strict Note validation, return 200/no dispatch for selected non-message events, and add replay-harness tests for malformed JSON, invalid Note payloads, Document events, and setup pings.

## Best Possible Solution

Keep the JSON-before-schema split, classify non-message activity types such as reactions before rejecting malformed message-shaped payloads, and require real runtime proof before normal maintainer merge.
