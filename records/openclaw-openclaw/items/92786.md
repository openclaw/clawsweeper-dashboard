## Summary

Keep this PR open as the active implementation candidate. Current main still uses raw `chmodSync` on the per-agent database open path, while the PR reuses the already-merged shared-state chmod policy and adds focused regression coverage; I found no blocking correctness finding, but the security-sensitive degraded-mode tradeoff still needs normal maintainer review.

## What This Changes

The PR exports the shared best-effort chmod helper, applies it to agent database directory and sidecar permission hardening, and adds agent database chmod-failure tests.

## Best Possible Solution

Merge the narrow helper reuse once maintainers accept applying the already-approved shared chmod policy to the agent DB and exact-head gates remain green.

## Reproduction Assessment

Yes. Current main calls raw `chmodSync` before the agent DB opens, and the contributor supplied before/after terminal proof through the real agent DB and auth-profile reader paths with chmod failures injected at the libc syscall boundary.

## Solution Assessment

Yes. Reusing the merged shared-state helper is the narrowest maintainable fix because it keeps one chmod policy, preserves fail-closed behavior for unexpected errors, and avoids a new env/config workaround or duplicated caller catches.
