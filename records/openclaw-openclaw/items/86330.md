## Summary

## Best Possible Solution

Land the effective-policy fix only after Claude live policy resolution uses a side-effect-free approval snapshot and preserves tests for default, restrictive, session, and per-agent policy cases.

Yes. Current main source has no Claude live `control_request` response path, and the PR body plus added tests show the before/after behavior; the PR-induced approval-file side effect is also source-reproducible from the resolver call chain.

## Solution Assessment

No, not as currently implemented. The effective-policy direction is plausible and well-covered, but policy-only resolution should not call the resolver that creates or repairs exec-approval state during Claude live launch.
