## Summary

Keep open: the PR is still useful and not implemented on current main. Current main and v2026.6.9 ignore configured `session.store` for `sessions export-trajectory` when `--store` is omitted, while sibling session commands use config-aware store targets; this patch is narrow, has sufficient terminal proof, and has no blocking findings.

## What This Changes

The PR changes `sessions export-trajectory` so the implicit session store uses configured `session.store` when `--store` is omitted, updates the help text, and adds focused coverage.

## Best Possible Solution

Land the focused resolver change after maintainer acceptance of the configured-store default alignment and normal merge checks, keeping explicit `--store` as the override.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure `session.store`, list a session through the sibling sessions path, then run export without `--store`; current main reads the default agent store instead.

## Solution Assessment

Yes. The PR reuses the existing `getRuntimeConfig` plus `resolveStorePath` contract and preserves explicit `--store` precedence; routing through broader target selection would add unrelated selector semantics to a single-session export path.
