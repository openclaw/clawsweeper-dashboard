## Summary

Keep open for maintainer review: this is a focused candidate fix for a real current-main CLI store-resolution mismatch, with no blocking code findings, but the implicit default and explicit `--store` behavior are compatibility-sensitive.

## What This Changes

The PR changes `sessions export-trajectory` to resolve implicit and explicit session stores through runtime config and the shared `resolveStorePath` helper, updates help text, and adds command/resolver tests.

## Best Possible Solution

Land the focused resolver alignment after maintainers accept the CLI upgrade behavior and required checks pass, with https://github.com/openclaw/openclaw/issues/95568 remaining canonical until merge.

## Reproduction Assessment

Yes. Source inspection on current main shows export without `--store` bypasses configured `session.store` while sibling session commands use the shared resolver; I did not run a local CLI repro because this review is read-only.

## Solution Assessment

Yes. Reusing `getRuntimeConfig` and `resolveStorePath` in the single export command is the narrowest maintainable fix; broader multi-target session discovery would add list/tail semantics this command does not need.
