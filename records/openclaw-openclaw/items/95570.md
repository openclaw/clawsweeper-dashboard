## Summary

Keep open: the patch looks like the correct active fix for the linked session-store bug, but it changes compatibility-sensitive CLI store-resolution behavior and should wait for maintainer acceptance rather than cleanup closure.

## What This Changes

The PR makes `openclaw sessions export-trajectory` resolve implicit and explicit session stores through runtime config and `resolveStorePath`, updates help text, and adds focused tests.

## Best Possible Solution

Land the focused resolver alignment once maintainers accept the CLI compatibility change, with https://github.com/openclaw/openclaw/issues/95568 remaining canonical until merge.

## Reproduction Assessment

Yes. Source inspection on current main shows export without `--store` bypasses configured `session.store` while sibling session commands use the shared resolver; no local CLI repro was run because this review is read-only.

## Solution Assessment

Yes. Reusing `getRuntimeConfig` and `resolveStorePath` in the single export command is the narrowest maintainable fix; broader session target discovery or auth changes are unnecessary for this CLI path.
