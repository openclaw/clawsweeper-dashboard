The PR changes `sessions export-trajectory` so its implicit store resolves from configured `session.store` when `--store` is omitted, updates the help text, and adds focused unit coverage.

## Best Possible Solution

## Reproduction Assessment

Yes. Source inspection on current main shows export without `--store` uses the default agent session store while sibling session commands resolve configured `session.store`; the linked issue also describes a temporary configured-store repro.

## Solution Assessment

Yes. The PR reuses the existing `getRuntimeConfig` plus `resolveStorePath` contract and preserves explicit `--store` precedence; routing through broader target selection would add unrelated selector validation semantics to this single-session export path.
