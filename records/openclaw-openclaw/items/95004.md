## Summary

Keep open: current main and the latest release still suppress short first Telegram previews and answer-lane partials in progress mode, while Telegram maintainer notes make changing the debounce/placeholder policy a product decision. The linked open PR https://github.com/openclaw/openclaw/pull/95007 is adjacent progress-draft formatting work and explicitly does not address this issue's draft-materialization policy.

## What This Changes

The issue requests a Telegram streaming policy that shows a predictable draft or progress placeholder when upstream draft state exists instead of suppressing short initial previews or answer deltas in progress mode.

## Best Possible Solution

Maintainers should choose a Telegram-owned policy: keep the debounce and document the Web/Telegram difference, or add a tested single-preview placeholder or delayed materialization path that preserves one-preview/no-spam behavior.

## Reproduction Assessment

Yes at source level: current code and tests prove the short first preview is suppressed below minInitialChars, and progress mode drops answer-lane partial updates before they reach the Telegram draft stream. I did not run live Telegram proof in this read-only review.

## Solution Assessment

No as an automatic fix: the report lists several acceptable outcomes, and Telegram maintainer notes intentionally keep the first-preview debounce. The maintainable path needs maintainer choice before code changes.
