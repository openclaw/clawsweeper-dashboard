## Summary

Keep open. Current main matches the reported Telegram draft-suppression behavior, but the first-preview debounce is intentional and progress mode is documented as status-oriented rather than answer-text streaming, so this needs maintainer product/docs direction rather than a strict bug close.

## What This Changes

The issue requests more predictable Telegram preview visibility for short, fast, or progress-mode turns, or clear documentation of the intentional Telegram-Web tradeoff.

## Best Possible Solution

Decide whether to preserve the current Telegram no-spam tradeoff with clearer public docs or approve a narrow placeholder/delay behavior that proves it will not create duplicate preview spam.

## Reproduction Assessment

Yes. Source and existing tests show the first preview below minInitialChars is suppressed, and dispatch source shows answer partials are skipped in progress mode; I did not run tests because this is a read-only review.

## Solution Assessment

Unclear. Removing the debounce or streaming answer text in progress mode is not clearly the best fix because current docs and maintainer notes treat these as no-spam/status-mode tradeoffs; a docs clarification or maintainer-approved placeholder path is safer.
