## Summary

Keep open: current `main` and latest release `v2026.6.8` still let terminal `openclaw workboard list` print archived cards by default, while the Workboard tool and slash-command list paths already hide archived cards. No same-root open fix PR was found, so this remains a narrow Workboard CLI bug.

## What This Changes

The issue asks for `openclaw workboard list` to hide archived cards by default and provide an explicit way to include them.

## Best Possible Solution

Update the Workboard terminal CLI list path to use the same archived-card default as the tool and slash command, add an explicit include-archived CLI flag, and align CLI docs/tests without changing store-level list semantics.

## Reproduction Assessment

Yes, from source inspection: the terminal CLI list path reads all cards and filters only by status, while the tool and slash-command paths hide archived cards by default. I did not run a live CLI reproduction because this review was constrained to read-only inspection.

## Solution Assessment

Yes, the narrow maintainable fix is to filter archived cards in the terminal CLI list path and add an explicit include-archived flag matching the existing tool parameter. `WorkboardStore.list` should remain unfiltered because current callers already own their visibility policy.
