## Summary

Keep open: current main and latest release still let the terminal Workboard CLI list archived cards, while the Workboard tool and slash-command list paths hide them by default. Multiple open fix PRs now reference this issue, so this issue should stay as the canonical tracker until one lands.

## What This Changes

The issue asks for `openclaw workboard list` to hide soft-archived cards by default and expose an explicit include-archived path so CLI behavior matches the Workboard tool and slash command.

## Best Possible Solution

Land one focused Workboard fix that filters archived cards in the CLI presentation layer, adds an explicit include-archived flag, updates CLI docs/tests, and leaves `WorkboardStore.list` unfiltered.

## Reproduction Assessment

Yes. Source inspection shows the CLI list path only filters by status, while the Workboard tool and slash-command paths filter archived cards by default; I did not run a live CLI repro because this review was read-only.

## Solution Assessment

Yes. The narrow maintainable fix is to put archived-card visibility in the CLI list presentation layer with an explicit include-archived flag, not in the store, because current callers already own their visibility policy.
