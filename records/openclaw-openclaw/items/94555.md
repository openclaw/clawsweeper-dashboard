Keep open: current main and latest release still have the CLI/API archive-list inconsistency, and several open PRs already target the fix. The issue should remain canonical until one vetted Workboard CLI PR lands.

## What This Changes

Request to make `openclaw workboard list` hide archived Workboard cards by default, with an explicit way to include archived cards, matching the Workboard tool and slash-command defaults.

## Best Possible Solution

Land one canonical Workboard CLI fix that hides archived cards by default, keeps an explicit include-archived path, and documents the chosen JSON/text compatibility contract.

## Reproduction Assessment

Yes. Source inspection of current main and `v2026.6.9` shows the terminal list path does not filter `metadata.archivedAt`, while the tool and slash-command list paths do.

## Solution Assessment

Yes, with a compatibility choice. The maintainable fix is at the CLI presentation boundary, leaving `WorkboardStore.list` as the full-card store boundary and adding an explicit include-archived path for the old visibility.
