## Summary

Keep this issue open: current `main` still has the Workboard terminal CLI listing every card from the store and filtering only by status, while the Workboard tool and slash-command surfaces already hide archived cards by default. The requested fix is narrow, source-reproducible, and not owned by an existing duplicate or fix PR.

## What This Changes

The issue reports that `openclaw workboard list` shows archived cards by default while the `workboard_list` API hides them unless archived cards are requested.

## Best Possible Solution

Bring the terminal CLI list default in line with the existing Workboard tool and slash-command archived-card behavior, and update docs/tests for the chosen include-archived story.

## Reproduction Assessment

Yes from source inspection: the CLI list path reads all cards and lacks an archived-card filter, while the tool and slash command paths hide archived cards by default. I did not run a live CLI reproduction because this review was read-only and avoided checkout mutations.

## Solution Assessment

Yes, the report's core direction is the narrow maintainable fix: filter archived cards in the terminal CLI list path to match the established tool/slash behavior. Adding an `--include-archived` CLI flag would mirror the existing tool parameter and preserve explicit archived-card inspection.
