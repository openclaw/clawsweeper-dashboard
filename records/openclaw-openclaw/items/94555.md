## Summary

Keep open: current `main` and latest release `v2026.6.8` still have the Workboard CLI/API mismatch, and several open linked PRs are candidate fixes. Because those PRs use closing syntax for this issue, the right cleanup is to review and land one canonical fix before closing the issue.

## What This Changes

The issue requests that `openclaw workboard list` hide soft-archived Workboard cards by default and provide an explicit way to include them, matching the Workboard tool and slash-command defaults.

## Best Possible Solution

Land one canonical linked fix PR, preferably a candidate with docs, focused CLI regression tests, and real CLI proof such as https://github.com/openclaw/openclaw/pull/94562; then close this issue from the merged implementation and clean up duplicate PRs.

## Reproduction Assessment

Yes. Source inspection on current `main` shows the terminal CLI list path does not filter archived cards, while the Workboard tool and slash-command list paths do; I did not run a live CLI reproduction because this review is read-only.

## Solution Assessment

Yes. The narrow maintainable fix is to filter archived cards in the terminal CLI list path, add an explicit `--include-archived` option matching the existing tool parameter, and leave `WorkboardStore.list` unfiltered because callers own visibility policy.
