## What This Changes

The issue asks for `openclaw workboard list` to hide soft-archived cards by default, matching the Workboard tool and slash-command list behavior, while preserving an explicit way to include archived cards.

## Best Possible Solution

Land one focused Workboard plugin PR that filters archived cards from terminal `list` output by default, adds `--include-archived`, and updates CLI tests/docs; then close this issue and duplicate PRs.

## Reproduction Assessment

Yes, from source inspection: the terminal CLI list path reads all cards and filters only by status, while the Workboard tool and slash-command paths hide archived cards by default. I did not run a live CLI reproduction because this review is read-only.

## Solution Assessment

Yes, the narrow maintainable fix is to change the Workboard terminal CLI list path while leaving `WorkboardStore.list` unfiltered because sibling callers already own visibility policy. The existing open PRs should be reviewed against that shape rather than queueing another duplicate fix.
