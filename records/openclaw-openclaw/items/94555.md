## Summary

Keep open: current main and the latest release still leave the terminal Workboard CLI list path unfiltered for archived cards, while the Workboard tool and slash-command paths hide archived cards by default. Several open linked PRs already target this exact bug, so this issue should stay as the canonical tracker until one fix lands.

## What This Changes

Make `openclaw workboard list` hide soft-archived Workboard cards by default, with an explicit include-archived CLI path to match the tool and slash-command behavior.

## Best Possible Solution

Land one focused Workboard PR that filters archived cards in the CLI presentation layer, preserves an explicit include-archived escape hatch, updates focused tests/docs, and leaves `WorkboardStore.list` as the full-card store boundary.

## Reproduction Assessment

Yes. Source inspection shows the CLI list path only filters by status, while the Workboard tool and slash-command paths filter archived cards by default; I did not run a live CLI repro because this review is read-only.

## Solution Assessment

Yes. The narrow maintainable fix is to put archived-card visibility in the CLI list presentation layer with an explicit include-archived flag, not in the store, because current callers already own their visibility policy.
