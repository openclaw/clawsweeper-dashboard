## Summary

Keep open: current `main` and latest release `v2026.6.9` still lack archived-card filtering in the terminal Workboard CLI list path, and multiple open PRs now target this same issue with closing syntax.

## What This Changes

The issue asks for `openclaw workboard list` to hide soft-archived Workboard cards by default and provide an explicit way to include them, matching the Workboard tool and slash-command behavior.

## Best Possible Solution

Land one focused Workboard CLI fix that hides archived cards by default, preserves an explicit `--include-archived` path, updates CLI tests/docs, and keeps store-level list semantics unchanged.

## Reproduction Assessment

Yes. Source inspection is enough: persist a card with `metadata.archivedAt`, then current `openclaw workboard list` will still render it because the CLI list handler only filters by status.

## Solution Assessment

Yes. The narrow maintainable fix is in the Workboard CLI list presentation path; changing `WorkboardStore.list` would be too broad because tool, slash-command, dispatch, and show/read callers intentionally own their own visibility rules.
