## Label Justifications

- `P2`: This is a normal-priority bundled Workboard CLI/API consistency bug with limited blast radius.
- `impact:other`: Archived Workboard cards can remain visible in CLI output, misleading operators into thinking archive actions failed.

## Summary

Keep open: current `main` and latest `v2026.6.8` still have the Workboard CLI/API archived-card mismatch, and this issue now has multiple open PRs with closing syntax that should be reviewed or closed before the issue is resolved.

## What This Changes

Make `openclaw workboard list` hide soft-archived cards by default like `workboard_list`, with an explicit CLI way to include archived cards.

## Best Possible Solution

Land one narrow Workboard CLI fix that matches the API and slash-command archive default, keeps `WorkboardStore.list` unfiltered, and updates CLI docs and regression tests.

## Reproduction Assessment

Yes. Source inspection on current `main` shows the CLI list path filters only status, while the tool and slash-command paths hide `metadata.archivedAt` cards by default; no live CLI run was performed because this review was read-only.

## Solution Assessment

Yes. The maintainable fix belongs in the terminal CLI list surface with an `--include-archived` flag matching the existing tool option, not in `WorkboardStore.list`, because callers already own visibility policy.
