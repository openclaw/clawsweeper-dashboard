## What This Changes

The issue requests that terminal Workboard list output hide soft-archived cards by default, preferably with an explicit archived-card opt-in, to match the Workboard API/tool behavior.

## Best Possible Solution

Land one canonical Workboard CLI fix that filters archived cards at the CLI presentation boundary, provides an explicit archived-card opt-in, updates docs/tests, and leaves `WorkboardStore.list` unfiltered.

## Reproduction Assessment

Yes. Source inspection on current main shows the CLI list path only filters by status, while the Workboard tool and slash-command list paths filter archived cards by default; I did not run a live CLI repro because this review is read-only.

## Solution Assessment

Yes. The narrow maintainable fix is to apply archived-card visibility in the CLI list presentation layer with an explicit opt-in for archived cards, rather than changing the store contract shared by other callers.
