## Summary

Keep open: current main and v2026.6.9 still leave terminal Workboard CLI listing unfiltered for archived cards, and several open PRs with closing syntax are already candidate fixes.

## What This Changes

The issue requests that `openclaw workboard list` hide archived Workboard cards by default, matching the Workboard tool behavior, with an explicit way to include archived cards.

## Best Possible Solution

Land one canonical Workboard CLI fix that applies archived-card visibility at the CLI presentation boundary, keeps `WorkboardStore.list` unfiltered, and covers the default plus include-archived behavior with focused tests/docs.

## Reproduction Assessment

Yes. Source inspection of current main and v2026.6.9 shows terminal `openclaw workboard list` filters only by status, while the Workboard tool and slash-command list paths hide archived cards by default.

## Solution Assessment

Yes. The narrow maintainable fix is to put archived-card visibility in the CLI list presentation layer with an explicit include-archived flag, not in the store, because current callers already own visibility policy.
