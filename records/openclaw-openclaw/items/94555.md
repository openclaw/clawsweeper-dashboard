## What This Changes

The issue requests that `openclaw workboard list` hide soft-archived Workboard cards by default, preferably with an explicit `--include-archived` escape hatch.

## Best Possible Solution

Land one canonical Workboard CLI fix that filters archived cards in the presentation layer, preserves an explicit include-archived path, adds focused CLI coverage, and keeps `WorkboardStore.list` unfiltered.

## Reproduction Assessment

Yes, by source inspection: the current CLI list path only filters by status, while the tool and slash-command list paths filter archived cards by default. I did not run a live CLI reproduction because this review is read-only.

## Solution Assessment

Yes, the best fix is a narrow CLI presentation-layer change with an explicit include-archived flag and regression coverage. Moving the filter into `WorkboardStore.list` would be too broad because current callers intentionally own their visibility policy.
