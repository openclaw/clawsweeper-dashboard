## Summary

Keep open: current `origin/main` and latest release `v2026.6.9` still leave terminal `openclaw workboard list` unfiltered for archived cards, while the Workboard tool and slash command already hide archived cards by default. Several open PRs now target this exact bug, so the issue should stay open until one candidate lands.

## What This Changes

The issue asks for `openclaw workboard list` to hide soft-archived Workboard cards by default like the Workboard tool/slash command and expose an explicit include-archived escape hatch.

## Best Possible Solution

Land one focused Workboard CLI fix that hides archived cards from default list output, preserves an explicit include-archived path for old visibility, updates CLI docs/tests, and leaves `WorkboardStore.list` as the full-card store boundary.

## Reproduction Assessment

Yes. The current-main CLI source still lists all store cards and filters only by status, while the tool and slash-command paths filter `metadata.archivedAt`; I did not run a live CLI reproduction in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Workboard CLI list presentation path with an include-archived flag and tests/docs; moving the filter into `WorkboardStore.list` would break callers that intentionally need full-card state.
