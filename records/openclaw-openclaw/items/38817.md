## Summary

Keep open. Current main still depends on resolved model `compat.unsupportedToolSchemaKeywords` for stripping provider-rejected tool-schema keywords, while the Volcengine/Ark Kimi catalog rows still lack that compat metadata; the direct fix PRs are closed unmerged and #44888 tracks a different Kimi schema-compaction failure.

## What This Changes

The issue asks OpenClaw to strip unsupported JSON Schema validation keywords such as `minLength`, `maxLength`, and `minItems` for Volcengine-hosted Kimi/Ark models so tool calls pass provider validation.

## Best Possible Solution

Add provider-owned `compat.unsupportedToolSchemaKeywords` metadata to the affected Volcengine/Ark/Kimi catalog rows, include sibling BytePlus rows only if the same backend behavior is confirmed, and keep core normalization generic.
