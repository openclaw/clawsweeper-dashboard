## Summary

Keep open. Current main still strips unsupported tool-schema keywords only from resolved `modelCompat`, and the Volcengine/BytePlus Ark/Kimi manifest rows still do not declare `compat.unsupportedToolSchemaKeywords`; the direct closing PRs are closed unmerged while #44888 tracks a different Kimi schema-size failure.

## What This Changes

The issue asks OpenClaw to stop sending unsupported validation keywords such as `minLength` in tool schemas for Volcengine-hosted Kimi models.

## Best Possible Solution

Add provider-owned `compat.unsupportedToolSchemaKeywords` metadata to the affected Volcengine Ark/Kimi manifest rows, with focused tests proving the metadata reaches tool normalization; keep core normalization generic.

## Reproduction Assessment

Yes. A high-confidence code-path reproduction is to resolve a current Volcengine Ark/Kimi model row, normalize a tool schema containing `minLength`, and observe that current main leaves the keyword because the row has no compat metadata; the live provider rejection was reported but not rerun here.

## Solution Assessment

No for the literal core substring check proposed in the issue; the narrow maintainable fix is provider-owned manifest compat metadata feeding the existing generic normalization seam.
