## Summary

Keep open. Current main still depends on resolved model `compat.unsupportedToolSchemaKeywords` to strip provider-rejected tool-schema keywords, but the Volcengine/Ark Kimi catalog rows do not declare that compat metadata; the two direct fix PRs are closed unmerged, and #44888 tracks a different Kimi schema-compaction failure.

## What This Changes

The issue reports Volcengine-hosted Kimi models rejecting tool parameter schemas that include unsupported JSON Schema validation keywords such as `minLength`.

## Best Possible Solution

Add provider-owned `compat.unsupportedToolSchemaKeywords` metadata to affected Volcengine/Ark Kimi catalog rows and cover it with focused provider catalog and schema-normalization tests, keeping core normalization generic.
