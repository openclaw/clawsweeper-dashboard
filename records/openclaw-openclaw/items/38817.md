## Summary

Keep open. Current main still strips unsupported tool-schema keywords only from resolved model compat metadata, and the Volcengine Ark/Kimi catalog rows still do not declare `compat.unsupportedToolSchemaKeywords`; the direct fix PRs #38883 and #46887 are closed unmerged, while #44888 tracks a different Kimi schema-size failure.

## What This Changes

The issue asks OpenClaw to avoid sending unsupported JSON Schema validation keywords such as `minLength` to Volcengine-hosted Kimi/Ark models.

## Best Possible Solution

Add provider-owned `compat.unsupportedToolSchemaKeywords` metadata to the affected Volcengine/Ark/Kimi catalog rows, include BytePlus only if the same backend behavior is confirmed, and keep core normalization generic.

## Reproduction Assessment

Yes. A high-confidence code-path reproduction is to resolve a current Volcengine Ark/Kimi model row, normalize a tool schema containing `minLength`, and observe that current main leaves the keyword because the row has no compat metadata; the live provider rejection was reported but not rerun here.

## Solution Assessment

No for the literal provider/model substring check proposed in the issue. The narrower maintainable fix is provider-owned manifest compat metadata feeding the existing generic normalization seam.
