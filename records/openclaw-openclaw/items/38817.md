## Summary

Keep open. Current main still has the generic compat-driven schema cleanup path, but the affected Volcengine/Ark/Kimi catalog rows do not declare the unsupported keyword metadata needed for that path to strip `minLength` and related validation keywords.

## What This Changes

The issue requests that Volcengine-hosted Kimi/Ark model tool schemas drop unsupported JSON Schema validation keywords before provider requests are sent.

## Best Possible Solution

Add provider-owned `compat.unsupportedToolSchemaKeywords` metadata to the affected Volcengine/Ark/Kimi manifest rows, and include sibling BytePlus rows only if they share the same backend behavior, while preserving the existing generic compat-driven core path.

## Reproduction Assessment

Yes, there is a high-confidence source-level reproduction path: Volcengine/Kimi tool requests can include `minLength`, current catalog rows lack compat metadata, and normalization only strips unsupported keywords from `modelCompat`. I did not run a live Volcengine API call in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is provider catalog compat metadata, not a core provider/model string fallback, because the generic normalization seam already exists and manifest rows already propagate compat.
