## Summary

Keep open. Current main has the generic compat-driven schema stripper, but the Volcengine/Ark/Kimi catalog rows still do not provide the compat metadata that would trigger it, so the reported provider rejection remains a valid narrow bug.

## What This Changes

The issue requests that Volcengine-hosted Kimi and Ark models strip unsupported JSON Schema validation keywords such as `minLength` before tool calls are sent.

## Best Possible Solution

Add provider-owned `compat.unsupportedToolSchemaKeywords` metadata to the affected Volcengine/Ark/Kimi catalog rows, include BytePlus only if the same backend behavior is confirmed, and keep the core normalizer generic.

## Reproduction Assessment

No live high-confidence reproduction was run in this read-only review. Source inspection shows a clear reproduction path: select a current Volcengine Kimi/Ark model whose catalog row lacks compat metadata, then send any tool schema containing `minLength`, which current normalization will not strip for that model.

## Solution Assessment

No, the issue body's provider/model substring fallback is not the best maintainable fix. The repo now has a generic `modelCompat.unsupportedToolSchemaKeywords` seam, so the narrow fix should be provider-owned catalog metadata plus regression coverage.
