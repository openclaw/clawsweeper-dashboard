## What This Changes

The issue asks OpenClaw to strip unsupported JSON Schema validation keywords such as `minLength`, `maxLength`, and `minItems` for Volcengine/Ark-hosted Kimi tool calls.

## Best Possible Solution

Add provider-owned `compat.unsupportedToolSchemaKeywords` metadata to the affected Volcengine/Ark/Kimi catalog rows, cover the manifest-to-runtime compat path with focused tests, and keep core schema normalization provider-agnostic.

## Solution Assessment
