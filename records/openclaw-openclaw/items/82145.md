Full review comments:

- **[P2] Do not cache budget-clamped probe failures:** `src/cron/isolated-agent/model-preflight.runtime.ts:278-279`
  - body: When the shared chain deadline leaves less time than the configured probe timeout, this code passes that reduced value into `fetchWithSsrFGuard`. If that tiny timeout aborts the request, `budgetExhausted` remains false, so the failure is cached for five minutes and the same endpoint can be skipped on the next cron run even though it only lost the previous chain budget race. Mark reduced-budget probe failures as non-cacheable, or otherwise avoid writing them to the endpoint health cache.
