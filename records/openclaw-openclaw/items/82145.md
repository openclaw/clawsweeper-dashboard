## Summary

Keep open: the PR is a plausible config-based improvement, but it is draft, lacks after-fix real behavior proof, and accepts retry windows that can exceed cron's 60s setup watchdog before the agent runner starts.

## What This Changes

The PR adds configurable `cron.modelPreflight` retry/timeout settings, wires them into isolated cron local-provider preflight, and updates config, docs, and tests.

## Best Possible Solution

Bound or coordinate the configured preflight retry window with cron's setup watchdog and abort model, prove a real local-provider wake-up run, and keep chain-aware fallback handling tracked in https://github.com/openclaw/openclaw/issues/79329.

## Reproduction Assessment

Yes for the review finding: source shows preflight runs before `onExecutionStarted` while the 60s setup watchdog is active, and the PR schema accepts total retry windows above 60s. I did not run a live Ollama cold-start reproduction.

## Solution Assessment

No as written. A retry config may help cold local providers, but accepted settings need to be bounded or abort-aware before the docs can promise a configurable wake window.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Keep preflight retry windows under the setup watchdog:** `src/config/zod-schema.ts:713-715`
  - body: The schema accepts `timeoutMs: 60000`, `maxAttempts: 10`, and `retryDelayMs: 60000`, but isolated cron keeps a 60s setup watchdog active until `onExecutionStarted`. Because this preflight runs before that callback, a valid config such as two 60s attempts can abort job setup before the advertised wake window completes; bound the total window or make the watchdog/preflight abort path aware of it.
  - confidence: 0.86
