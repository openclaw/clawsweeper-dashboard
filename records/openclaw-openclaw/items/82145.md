## Summary

Keep open: this is a recent external draft PR that adds a new cron config surface, lacks after-fix real behavior proof, and still has a blocking correctness issue because accepted retry windows can exceed cron's 60s setup watchdog before the runner starts.

## What This Changes

The PR adds `cron.modelPreflight` timeout/retry settings for isolated cron local-provider preflight, applies them in the preflight retry loop, and updates docs plus config/runtime tests.

## Best Possible Solution

Bound or coordinate the preflight retry window with cron's setup watchdog and abort model, then prove it with a real local-provider wake-up cron run while leaving chain-aware fallback preflight tracked separately.

## Reproduction Assessment

Yes for the review finding: source inspection shows a valid config can make preflight wait longer than the 60s setup watchdog before `onExecutionStarted` clears it. I did not establish a live Ollama cold-start reproduction.

## Solution Assessment

No as written. A retry knob can be useful, but the patch should first make accepted retry settings fit cron's setup timeout and abort semantics so the documented wake window can actually complete.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Keep preflight retry windows under the setup watchdog:** `src/config/zod-schema.ts:713-715`
  - body: The schema accepts `timeoutMs: 60000`, `maxAttempts: 10`, and `retryDelayMs: 60000`, but isolated cron's setup watchdog fires after 60s until `onExecutionStarted`, and preflight runs before that callback. A valid config such as two 60s attempts can time out setup before the advertised wake window finishes; bound the total preflight window or make the watchdog/preflight abort path aware of this wait.
  - confidence: 0.86
