## Summary

Keep open. Current main still exposes only TTL-gated cache pruning, fresh TTL windows still return before pruning, and the open rolling-mode PR is not a merged/current-main solution for this request.

## What This Changes

Add active-session context pruning so old tool results can be trimmed by turn age or a rolling trigger instead of only after cache TTL inactivity.

## Best Possible Solution

Choose one active-session pruning trigger API, then implement it in the active `pi-hooks` context-pruning path with aligned config types, strict schema, generated/public docs, changelog, and regression tests for continuous turns inside the TTL window.

## Reproduction Assessment

Yes, at source level. Configure cache-ttl pruning and keep sending eligible turns inside `ttlMs`; current main returns before pruning while the fresh cache-touch timestamp keeps the TTL window alive.

## Solution Assessment

Unclear. The request belongs in core because pruning runs inside context assembly, but maintainers still need to choose the public trigger API across rolling, turn-age, and threshold bypass designs.
