## Summary

Keep open. Current main still exposes only TTL-gated context pruning, active sessions still return before pruning while the cache TTL window is fresh, and the related rolling-mode implementation at https://github.com/openclaw/openclaw/pull/43238 remains open and needs rework/API review before it can satisfy this issue.

## What This Changes

Add turn-age or rolling session pruning so active sessions can trim old tool results every turn instead of waiting for cache TTL inactivity.

## Best Possible Solution

Choose the pruning trigger API, then implement it in the active pi-hooks context-pruning path with aligned config types/schema, generated and public docs, changelog, and focused tests for consecutive active turns inside the TTL window.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: configure cache-ttl pruning, keep sending eligible turns inside ttlMs, and the hook returns before pruneContextMessages while fresh cache-touch timestamps continue to be appended.

## Solution Assessment

Unclear. The requested direction belongs in core because pruning is inside context assembly, but maintainers still need to choose the public API shape and decide whether to repair or replace the existing rolling-mode PR.
