## Summary

## What This Changes

The PR adds installer SIGINT/SIGTERM abort handlers, preserves signal exit codes through doctor paths, suppresses dashboard launch after doctor failure, changes shared Clack cancellation to exit 130, and adds install.sh regression tests.

## Best Possible Solution

Keep the installer traps and doctor/dashboard guards, leave generic Clack cancellation as ordinary nonzero cancellation, translate only the installer doctor interrupt path to 130, and require exact redacted upgrade proof before merge.

## Reproduction Assessment

Yes by source inspection: current main lacks INT/TERM traps, normalizes run_quiet_step failures to 1, ignores migration-doctor failure, and can leave dashboard opening enabled after doctor cancellation. I did not execute the live curl-pipe upgrade flow in this read-only review.

## Solution Assessment

No. The installer trap and doctor status handling are the right direction, but mapping every shared Clack cancel symbol to exit 130 is too broad because Clack uses the same cancel symbol for Escape and Ctrl+C.
