## Summary

Keep open: current main and the latest release still have the installer Ctrl+C/dashboard path, but this PR is not merge-ready because the interactive doctor-cancel case can still be treated as success and one exit-130 check is masked by the quiet wrapper.

## What This Changes

The PR modifies `scripts/install.sh` to trap INT/TERM, propagate doctor interrupt/failure state, and open the dashboard only after successful doctor runs.

## Best Possible Solution

Keep the dashboard gating, but launch doctor so a real interactive Ctrl+C aborts the installer even when doctor exits 0, preserve the child status where 130 matters, and require redacted real TTY upgrade proof before merge.

## Reproduction Assessment

Yes by source inspection: current main lacks an INT/TERM trap, doctor prompt cancellation exits 0 through `guardCancel`, and the installer can still continue into dashboard/plugin-update flow. I did not run the live `curl | bash` interactive upgrade path in this read-only review.

## Solution Assessment

No. The dashboard gating is the right direction, but this patch still relies on exit-130 paths that miss doctor-caught Ctrl+C exit 0 and one default path loses the child status behind `run_quiet_step`.
