## What This Changes

## Best Possible Solution

Scope SIGINT-style exit 130 to update/doctor cancellation under the installer contract, preserve ordinary prompt-cancel semantics unless maintainers intentionally change them, and add real TTY proof for the interactive doctor path.

## Reproduction Assessment

Yes, source-reproducible: current main can treat Clack cancellation as success through `guardCancel` exit 0 and the installer can continue after doctor success. I did not run a live TTY upgrade reproduction in this read-only review.

## Solution Assessment

No, not as currently shaped. The installer interruption fix is needed, but the best repair should avoid changing every shared Clack cancellation into exit 130 unless maintainers explicitly want that broader compatibility change.
