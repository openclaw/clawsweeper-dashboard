## Summary

Keep open: the installer bug is real on current main, but this patch is not merge-ready because it changes `run_doctor` to return failures while one caller still runs under `set -e`, and it changes shared prompt-cancel exit semantics beyond the installer path without caller proof.

## What This Changes

The branch adds installer INT/TERM traps and doctor exit handling, suppresses dashboard/plugin follow-up on doctor failure, and changes shared Clack cancellation to exit nonzero.

## Best Possible Solution

Land a narrowed installer fix that preserves best-effort doctor behavior except for explicit interruption, guards every `run_doctor` caller, scopes or fully proves any shared Clack exit-code change, and includes exact upgrade-doctor real behavior proof.

## Reproduction Assessment

Yes, source-level: current main lacks an INT trap, `run_quiet_step` loses signal exit codes, and the upgrade path can still proceed to dashboard follow-up after doctor failure. I did not run a live installer reproduction in this read-only review.

## Solution Assessment

No as submitted. The installer signal handling is a plausible direction, but the patch should avoid broad shared cancellation semantics and guard all `run_doctor` callers before it is the best fix.
