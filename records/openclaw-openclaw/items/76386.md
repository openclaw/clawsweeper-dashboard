## Summary

Keep open: the reported upgrade-interrupt bug is real on current main and the PR is directionally useful, but the proposed patch still turns one interrupted doctor path into a successful installer exit.

## What This Changes

The PR adds installer SIGINT/SIGTERM handling, rewrites the upgrade doctor exit-code check, moves dashboard auto-open behind a successful interactive doctor run, and adds a changelog entry.

## Best Possible Solution

Land a narrow installer fix that consistently exits 130 on interruption, suppresses dashboard opening after cancelled or failed interactive doctor runs, and preserves the intended post-install dashboard behavior for non-cancelled paths.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes from source: run the installer on an upgrade with a TTY and unavailable skills so `openclaw doctor` prompts, then press Ctrl+C; @clack maps that to cancel and OpenClaw exits doctor successfully while the installer has already set the dashboard flag.

## Solution Assessment

Is this the best way to solve the issue? No; adding installer-level interrupt handling is the right boundary, but the explicit doctor SIGINT branch should terminate with 130 instead of returning success.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86
