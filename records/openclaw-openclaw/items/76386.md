## Summary

Keep open: the PR fixes part of the installer dashboard problem, but the interactive Clack cancellation path still maps to exit 1 and this installer block only aborts on 130, so Ctrl+C/Escape inside upgrade doctor can still continue instead of exiting as an interrupted install.

## What This Changes

The PR adds installer SIGINT/SIGTERM traps, preserves signal-like exit codes through `run_quiet_step`, guards doctor/dashboard paths, changes shared Clack cancellation from exit 0 to exit 1, and adds focused install-script tests.

## Best Possible Solution

Make the installer treat interactive doctor cancellation as an interrupted install, preferably by scoping cancellation handling to the doctor/update path or by explicitly proving and accepting any broader CLI cancellation exit-code change.

## Reproduction Assessment

Yes from source inspection: Clack returns a cancel symbol for Ctrl+C/Escape, the PR maps that to exit 1, and the installer only aborts on doctor exit 130. I did not run the full live upgrade doctor flow.

## Solution Assessment

No: the installer-side traps and dashboard guard are useful, but the proposed patch still does not abort the Clack cancellation path and broadens shared CLI cancellation semantics without enough proof.
