## Summary

Keep open: the second commit fixes the earlier exit-130 branch, but the current PR head still does not cover the reported Clack raw-mode Ctrl+C path because `openclaw doctor` cancellation exits 0 and the installer then treats doctor as successful.

## What This Changes

The PR adds SIGINT/SIGTERM handling to `scripts/install.sh`, rewrites the upgrade doctor exit-code handling, moves dashboard opening behind a successful interactive doctor run, and adds a changelog entry.

## Best Possible Solution

Land a narrow installer/update-mode cancellation fix that makes interactive doctor cancellation propagate as interruption, exits 130, and never updates plugins or opens the dashboard after the user cancels.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes from source: in an upgrade with a TTY and unavailable skills, the doctor prompt uses Clack confirm; Ctrl+C returns Clack cancel, `guardCancel` exits doctor 0, and the installer treats that as success.

## Solution Assessment

Is this the best way to solve the issue? No; installer-level traps and exit-130 handling are useful, but the fix must also make the Clack-cancelled doctor path visible to the installer instead of accepting exit 0 as success.

## Review Findings
