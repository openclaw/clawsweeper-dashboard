## Summary

Keep this PR open: current main and the latest release still have the reported installer Ctrl+C behavior, and the PR is a focused fix, but the latest head introduces a compatibility blocker by making ordinary `run_doctor` failures nonfatal in some paths and fatal in the existing-config path under `set -e`.

## What This Changes

The PR changes `scripts/install.sh` to add SIGINT/SIGTERM abort traps, preserve doctor exit status, and gate dashboard opening on successful doctor completion.

## Best Possible Solution

Keep the INT/TERM abort behavior, but make every doctor caller status-aware: only signal 130/143 should abort the installer, ordinary doctor failures should suppress dashboard/plugin follow-up without failing compatible install paths, and focused installer coverage should lock that in.

## Reproduction Assessment

Yes by source inspection: current main and v2026.5.22 lack INT/TERM traps, ignore `run_doctor` failures, and set dashboard launch intent after doctor paths that may not have completed successfully. I did not run a live interactive upgrade flow in this read-only review.

## Solution Assessment

No, not as-is: the requested fix direction is right, and the latest head fixed the prior git dashboard regression, but the new `run_doctor` return contract still breaks the existing-config caller. The safer solution is a status-aware doctor helper or guarded call sites that distinguish interrupts from ordinary doctor failures.
