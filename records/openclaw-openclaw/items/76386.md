## Summary

Keep open. Current main still has the installer cancellation bug, but this patch is not merge-ready because interactive doctor cancellation can still surface as exit 0 and be treated as success, and the shared quiet-step exit-code change needs compatibility proof.

## What This Changes

The branch adds SIGINT/SIGTERM handling to `scripts/install.sh`, preserves quiet-step child exit codes, and checks doctor exit 130 during install/upgrade doctor runs.

## Best Possible Solution

Fix the interactive doctor cancellation contract before merge, preferably by making cancellation distinguishable from successful doctor completion and keeping quiet-step exit-code changes scoped unless maintainers deliberately accept the broader behavior.

## Reproduction Assessment

Yes from source inspection: current upgrade install runs interactive `openclaw doctor`, and doctor prompt cancellation exits 0 through `guardCancel`. The PR body does not prove the exact interactive prompt flow after the patch.

## Solution Assessment

No. The proposed exit-130 checks are too narrow because current doctor prompt cancellation can exit 0 and be marked as success; the safer fix should make cancellation distinguishable from successful doctor completion and keep shared helper semantics narrow unless deliberately accepted.
