## What This Changes

The PR modifies `scripts/install.sh` to trap INT/TERM, propagate doctor status, and gate dashboard/plugin-update follow-up on successful doctor runs during install/upgrade finalization.

## Best Possible Solution

Land the focused installer change if maintainers agree that failed or interrupted doctor should block dashboard/plugin follow-up; otherwise preserve the old dashboard fallback for non-interrupt doctor failures while still aborting on SIGINT.

## Reproduction Assessment

## Solution Assessment

Yes with maintainer signoff: trapping INT/TERM and gating dashboard/plugin follow-up on doctor success is a narrow fix for the reported behavior. The only open judgment is whether ordinary doctor failure should also fail closed during upgrades.
