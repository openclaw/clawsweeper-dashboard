## Summary

Keep this PR open, but it should not merge as-is: current main still has the installer interrupt bug, while this patch likely regresses successful git installs by no longer opening the dashboard and its proof does not cover the reported interactive upgrade-doctor path.

## What This Changes

The PR adds INT/TERM handling to `scripts/install.sh`, preserves `openclaw doctor` exit status, gates dashboard opening on doctor success, and adds a changelog entry.

## Best Possible Solution

Keep the installer-level interrupt handling, but make the doctor helper preserve interrupt/failure status so dashboard opening remains gated on successful doctor completion for both upgrade and git-install paths.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: run the installer as an upgrade with a TTY, enter interactive `openclaw doctor`, press Ctrl+C, and current main has only an EXIT trap plus `&& doctor_ok=1` doctor handling; I did not execute the live TTY upgrade in this read-only review.

## Solution Assessment

No. The signal-trap direction is right, but the current placement of `should_open_dashboard=true` drops successful git-install dashboard opening and the supplied proof should cover the actual upgrade-doctor path.
