## Summary

Keep open: this targets a real installer bug, but the patch still marks a zero-exit doctor run as success, while the actual Clack Ctrl+C path can return cancel and OpenClaw's guard exits 0. It also still needs final-head real upgrade proof for the dashboard-gating behavior.

## What This Changes

The PR modifies `scripts/install.sh` to add INT/TERM traps, preserve doctor exit status, and gate dashboard opening on successful doctor runs.

## Best Possible Solution

Make doctor cancellation observable to the installer, preferably as exit 130 in the update/installer context, then gate plugin updates and dashboard launch only on true doctor success with final-head terminal proof.

## Reproduction Assessment

Yes, by source inspection: current main lacks an INT/TERM installer trap, Clack maps Ctrl+C to cancel, OpenClaw `guardCancel` exits 0, and the installer marks zero-exit doctor runs as success. I did not execute a live `curl | bash` TTY upgrade in this read-only review.

## Solution Assessment

No. The PR handles process exit 130, but the real Clack cancel path can exit 0, so the installer still cannot distinguish a user abort from successful doctor completion.
