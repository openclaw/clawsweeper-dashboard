## What This Changes

The branch adds installer SIGINT/SIGTERM handling, propagates doctor cancellation exit codes, guards dashboard launch after doctor failures, changes shared Clack cancellation to exit 130, and adds focused installer tests.

## Best Possible Solution

Keep the installer SIGINT cleanup and dashboard guard, but localize 130 handling to the installer/doctor upgrade path or an explicit doctor cancellation contract, then prove Ctrl+C in a real interactive upgrade-doctor run.

## Reproduction Assessment

Yes from source inspection: current main lacks INT/TERM installer handling, swallows run_doctor failures, and can continue toward dashboard opening after doctor cancellation. I did not run the live upgrade installer path in this read-only review.

## Solution Assessment

No, not as written: the installer-side fix direction is right, but changing the shared Clack guard makes a narrow installer bug into a broader CLI cancellation contract change. The safer fix is to keep shared cancellation stable and handle upgrade-doctor abort semantics locally or through an explicit doctor contract.
