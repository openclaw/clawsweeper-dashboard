## Summary

The PR still needs maintainer review before merge: it addresses a real installer cancellation bug, but the patch broadens cancellation semantics through the shared Clack helper and turns doctor failure/missing-openclaw cases into dashboard-blocking install outcomes, which are compatibility-sensitive changes that current docs/history do not clearly approve. The updated body supplies useful terminal proof for the sourced installer logic and timeout-driven upgrade path, but it still does not show the live TTY Ctrl+C path at an actual Clack doctor prompt that motivated the shared helper change.

## What This Changes

The PR adds INT/TERM abort traps and status-aware doctor/dashboard gating in `scripts/install.sh`, and changes shared Clack cancellation handling in `src/commands/onboard-helpers.ts` from exit 0 to exit 130.

## Best Possible Solution

Keep the installer Ctrl+C abort behavior, but either scope SIGINT-style exit handling to the doctor/installer cancellation path or get maintainer approval plus upgrade proof for the broader shared-helper and fail-closed doctor/dashboard semantics.

## Reproduction Assessment

Yes for the underlying source path, but not as a fully reproduced live path in this review: current main ignores nonzero doctor failures in `run_doctor`, the shared Clack helper exits 0 on cancel, and the PR body describes the user-facing Ctrl+C scenario. The exact live TTY Ctrl+C at an interactive Clack doctor prompt is still not demonstrated after the patch.

## Solution Assessment

Unclear; the installer-side SIGINT trap and doctor exit-code handling are directionally appropriate, but the shared `guardCancel` change and broad dashboard-blocking semantics may not be the narrowest maintainable fix. A safer path would scope SIGINT-style behavior to the doctor/installer cancellation path or document and approve the broader compatibility change.
