## Summary

Keep open. Source inspection confirms the reported Ctrl+C upgrade problem and the PR takes the right installer-level direction, but the diff appears to regress the existing source/git install dashboard path by removing the only flag assignment that path can reach.

## What This Changes

The branch updates `scripts/install.sh` to trap INT/TERM, handle doctor exit codes explicitly, gate dashboard opening after the interactive upgrade doctor succeeds, and add a changelog fix entry.

## Best Possible Solution

Keep the installer-level interrupt fix, but preserve the existing source/git install dashboard behavior while gating npm upgrade dashboard launch on successful interactive doctor completion.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes from source: on an upgrade with a TTY, the interactive doctor prompt can convert Ctrl+C into a cancel path that exits 0 while current main has already enabled dashboard opening.

## Solution Assessment

Is this the best way to solve the issue? Not quite; installer-level signal and exit-code handling is the right boundary, but the current diff should retain the git/source install dashboard flag path instead of moving all dashboard gating into the npm upgrade doctor branch.
