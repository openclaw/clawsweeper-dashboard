## Summary

Keep open: the installer Ctrl+C bug is still real and this PR is useful, but it is not merge-ready because the final patch maps all shared Clack cancellations to exit 130 and the supplied proof still does not show the exact interactive upgrade-doctor/no-dashboard path.

## What This Changes

This PR adds installer INT/TERM traps, preserves signal exit codes through doctor paths, suppresses dashboard launch after doctor failure, changes shared Clack cancellation to exit 130, and adds install.sh regression tests.

## Best Possible Solution

Keep the installer traps and doctor/dashboard guards, leave shared Clack cancellation as generic nonzero cancellation, translate SIGINT only in the installer doctor path, and require exact redacted upgrade proof before merge.

## Reproduction Assessment

Yes, by source inspection: current main lacks INT/TERM traps, normalizes run_quiet_step failures to 1, ignores migration-doctor failure, and can leave dashboard opening enabled after doctor cancellation. I did not execute a live TTY upgrade in this read-only review.

## Solution Assessment

No. The installer-local traps and doctor/dashboard handling are the right direction, but mapping every shared Clack cancel symbol to exit 130 is too broad because Clack uses the same cancel symbol for Escape and Ctrl+C.
