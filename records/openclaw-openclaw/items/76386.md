## Summary

Keep open: this is a useful installer bug-fix attempt, but source review shows it still misses the interactive doctor cancellation path that exits 0, and the supplied proof still does not show Ctrl+C at the upgrade-doctor prompt. The shared installer wrapper exit-code change also needs maintainer compatibility review before merge.

## What This Changes

The branch adds INT/TERM traps to `scripts/install.sh`, preserves `run_quiet_step` child exit codes, and aborts doctor paths when the doctor subprocess exits 130.

## Best Possible Solution

Make installer-run doctor cancellation produce an unambiguous interrupt outcome, keep ordinary doctor failures best-effort, and add exact upgrade-doctor Ctrl+C proof before merge.

## Reproduction Assessment

Yes, source review gives a high-confidence path for the remaining defect: doctor prompts route Clack cancellation through `guardCancel`, which exits 0, while the PR only aborts on 130. I did not run the full installer doctor flow, but the Bash signal probe confirms the parent trap is not enough when the child handles SIGINT and exits 0.

## Solution Assessment

No. The PR is a plausible partial fix for subprocesses that exit 130, but the interactive doctor prompt path can still exit 0 on Clack cancel; the safer fix is to make installer-run doctor cancellation explicit before treating exit 0 as success.
