## What This Changes

The PR adds installer INT/TERM abort handlers, preserves signal exit codes through quiet doctor execution, gates dashboard launch on doctor success, changes shared Clack cancellation to exit 130, and adds installer-script regression assertions.

## Best Possible Solution

Land this focused installer fix after maintainers explicitly accept the shared cancellation exit-code compatibility change, or narrow the 130 mapping to the doctor/upgrade path while keeping the installer trap, signal propagation, and dashboard gating behavior.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.6.8 shows the installer lacks an INT/TERM trap, loses child signal status in run_quiet_step, ignores run_doctor failure, and can set dashboard launch after doctor failure.

## Solution Assessment

Yes for the installer-owned parts: trapping INT/TERM, preserving signal exit status, returning doctor failure, and gating dashboard launch are the narrow fix. The only unresolved solution-fit question is whether changing the shared guardCancel helper to exit 130 everywhere is acceptable or should be narrowed.
