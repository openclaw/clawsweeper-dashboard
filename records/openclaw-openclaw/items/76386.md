## Summary

Keep open: this is a focused installer bug-fix PR, but the submitted patch does not yet cover the reported real TTY Ctrl+C path where `openclaw doctor` can consume SIGINT and exit successfully, and one new SIGINT branch cannot observe exit 130 through the quiet-step wrapper.

## What This Changes

The PR updates `scripts/install.sh` to add INT/TERM abort traps, make doctor failures affect dashboard/plugin-update flow, and only open the dashboard after successful doctor runs.

## Best Possible Solution

Keep the dashboard gating, but implement the doctor launch so a real Ctrl+C during interactive doctor aborts the installer even when doctor exits 0, then prove that path in a real TTY upgrade run before merge.

## Reproduction Assessment

Yes for the review blocker: current source plus a process-group SIGINT shell check show that a foreground child can consume SIGINT and let the parent continue. I did not run the full OpenClaw installer upgrade flow.

## Solution Assessment

No. The dashboard gating is the right direction, but the patch still relies on parent-trap and exit-130 paths that do not cover child-consumed SIGINT, and one exit-130 check is hidden behind a wrapper that collapses statuses.
