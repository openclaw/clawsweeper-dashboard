## Best Possible Solution

Make doctor cancellation surface as an installer-visible abort or non-success state, preserve current installer TTY handling, and only open the dashboard after a verified completed doctor run.

Yes for source-level reproduction: current main runs interactive doctor through guarded Clack prompts that cancel via `runtime.exit(0)`, while the installer treats exit 0 as success. I did not run a live `curl | bash` upgrade-doctor session in this read-only review.
