## Summary

## What This Changes

The PR adds installer INT/TERM traps, propagates doctor and signal exits to suppress dashboard/plugin follow-up after failed doctor runs, and changes shared Clack prompt cancellation to exit nonzero.

## Best Possible Solution

Land a narrowed installer fix that preserves best-effort doctor behavior except for explicit interruption, guards every run_doctor caller, scopes or fully proves any shared Clack exit-code change, and includes exact upgrade-doctor real behavior proof.

## Reproduction Assessment

Yes from source inspection, though I did not run a live installer: current main lacks an INT trap, run_quiet_step loses signal exit codes, and the upgrade path can still proceed to dashboard follow-up after doctor failure.

## Solution Assessment

No as submitted. The installer signal handling is a plausible direction, but the patch should avoid broad shared cancellation semantics and guard all run_doctor callers before it is the best fix.
