## Summary

Keep open: the PR has a useful signal-handling direction, but source review shows the interactive upgrade-doctor cancellation path can still continue to verification and dashboard launch when Clack cancellation exits 1 instead of 130.

## What This Changes

The PR adds installer SIGINT/SIGTERM traps, preserves signal exits through `run_quiet_step`, handles doctor exit 130, and changes shared Clack prompt cancellation to exit nonzero.

## Best Possible Solution

Make installer doctor cancellation a closed contract: both signal exits and Clack cancellation exits should abort the installer before plugin updates, verification, or dashboard launch, with focused proof for that upgrade path.

## Reproduction Assessment

Yes from source: on PR head, Clack cancellation exits 1, the installer aborts only for doctor exit 130, and dashboard launch remains armed by the earlier upgrade doctor step.

## Solution Assessment
