## Summary

Keep open: the PR is a useful partial installer fix, but the latest head can still treat an interactive doctor cancellation as success when doctor catches Ctrl+C and exits 0, and the supplied proof does not show the real foreground-child TTY path.

## What This Changes

The PR adds SIGINT/SIGTERM traps to `scripts/install.sh`, propagates doctor exit 130 as an installer interrupt, and gates dashboard opening on successful doctor runs.

## Best Possible Solution

Make doctor cancellation observable to the installer so real terminal Ctrl+C cannot be mistaken for success, then explicitly accept or narrow the failed-doctor dashboard gating before merge.

## Reproduction Assessment

Yes, source-reproducible: current main ignores migration doctor status, doctor cancellation can exit 0, and a shell probe shows the parent trap is skipped when the foreground child handles SIGINT and exits 0. I did not run a full live `curl | bash` upgrade reproduction.

## Solution Assessment

No; this is a useful partial fix, but it does not solve the issue's doctor-catches-SIGINT-and-exits-0 path. The safer fix is to make cancellation a non-successful doctor outcome before dashboard/plugin finalization can proceed.
