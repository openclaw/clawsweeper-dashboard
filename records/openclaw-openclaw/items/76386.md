## What This Changes

This PR adds installer INT/TERM abort handling, preserves signal exit codes through `run_quiet_step` and `run_doctor`, gates dashboard launch on doctor success, changes shared Clack cancellation to exit 130, and adds installer-script coverage.

## Best Possible Solution

Land this PR or an equivalent narrow installer fix after maintainers accept the shared cancellation exit-code behavior, then close the two linked installer Ctrl+C issues as implemented.

## Reproduction Assessment

Yes by source inspection: current main still lacks INT/TERM installer traps, flattens `run_quiet_step` failures to 1, ignores doctor failure in `run_doctor`, and exits 0 on shared Clack cancellation. I did not run a live TTY upgrade in this read-only review.

## Solution Assessment

Yes, with one compatibility caveat: installer-level traps, signal-status preservation, and dashboard gating are the narrow owner-boundary fix. The shared `guardCancel` move to 130 is plausible and useful for this bug, but maintainers should explicitly accept that broader cancellation semantics change.
