## What This Changes

The PR adds SIGINT/SIGTERM abort handlers in `scripts/install.sh`, propagates doctor exit status, and gates dashboard/plugin update continuation on doctor success.

## Best Possible Solution

Land the scoped trap and status-gating behavior if maintainers want failed or interrupted doctor runs to stop follow-up dashboard/plugin actions; otherwise preserve legacy continuation for ordinary failures while still aborting on SIGINT.

## Reproduction Assessment

Yes. Current main's source path has no INT/TERM abort trap, swallows `run_doctor` failures, and sets the dashboard flag after doctor; the linked report and PR body provide concrete upgrade-flow steps and after-fix terminal proof.

## Solution Assessment

Yes, if maintainers accept the compatibility change. Installer-level traps plus status-aware doctor/dashboard gating are narrow and maintainable; the safer alternative is preserving legacy continuation for ordinary non-interrupt doctor failures.
