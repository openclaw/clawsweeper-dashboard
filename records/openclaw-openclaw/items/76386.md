## What This Changes

The PR adds installer INT/TERM interruption handling, gates upgrade doctor exit-code handling explicitly, moves dashboard opening behind successful interactive doctor completion, and adds a changelog fix entry.

## Best Possible Solution

Land the narrow installer fix after normal CI/maintainer review so interrupted upgrade doctor runs exit 130 and failed or cancelled doctor runs do not open a dead dashboard.

## Solution Assessment

Is this the best way to solve the issue? Yes; installer-level signal trapping plus explicit doctor exit-code handling is the narrow maintainable boundary, while a future doctor-level Quit option can remain a separate UX improvement.
