## Summary

Keep open: the installer direction is useful, but source and dependency inspection show the exact interactive upgrade-doctor Ctrl+C path can still return doctor exit 1 and let the installer continue, while the shared Clack cancellation helper change broadens CLI exit-code behavior beyond the reported installer bug.

## What This Changes

The PR adds installer INT/TERM traps, preserves signal exit codes through `run_quiet_step`, guards doctor/dashboard launch paths, changes shared Clack cancellation to exit non-zero, and adds focused installer tests.

## Best Possible Solution

Keep the installer fix focused: make Ctrl+C inside interactive upgrade doctor propagate an abort/130 and suppress dashboard launch, while preserving or explicitly approving broader shared CLI cancellation semantics.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes from source and dependency inspection: Clack maps Ctrl+C/Escape to a cancel symbol, the PR maps that to exit 1, and the installer only aborts on doctor exit 130.

## Solution Assessment

Is this the best way to solve the issue? No; the installer signal handling is a good direction, but the branch should distinguish user cancellation from ordinary doctor failure and avoid an unapproved shared exit-code change.
