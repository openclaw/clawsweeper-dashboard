## Summary

Keep open: current main still keeps session-store mutations and durable writes inside the store lock, plugin cleanup remains registered-hook scoped, and the checked-in tree does not contain the reported Manifest/clawrouter listener needed to verify the port-bind fix. The latest discussion also reports the v2026.4.29 starvation shape still reproducing, so this is active maintainer work rather than stale cleanup.

## What This Changes

The issue asks OpenClaw to fix a gateway restart leak triad: Manifest port rebinding, process signal-handler accumulation, and session persistence stalls that starve WebSocket handshakes.

## Best Possible Solution

Keep the issue open and split the root causes into maintainer-owned fixes: make Manifest listener startup idempotent, add a cleanup/dedup lifecycle for plugin-owned process resources, and redesign session persistence so slow durable writes cannot block Gateway handshakes or session lanes.

## Reproduction Assessment

Partial, not complete. The discussion provides high-confidence field reproduction for session-write-lock and WS starvation on 2026.4.26 through v2026.4.29, but there is no deterministic current-main triad reproduction and the Manifest listener implementation is not in the checked-in tree.

## Solution Assessment

No single current-main change is the best complete solution. The safer path is to preserve the useful mitigations while making targeted lifecycle and persistence changes after maintainers choose the durability/latency boundary.
