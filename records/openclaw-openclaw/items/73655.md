## Summary

Keep this open: current main still holds session-store writes through durable fsync work, plugin cleanup remains hook-scoped, the reported Manifest/clawrouter listener source is not present to verify, and fresh issue comments report the starvation symptom on v2026.4.29.

## What This Changes

The issue asks OpenClaw to fix a gateway restart starvation cluster involving Manifest port retries, accumulating process signal handlers, and slow session-store JSON writes blocking WebSocket handshakes.

## Best Possible Solution

Split the remaining work into owned fixes: make the Manifest/clawrouter listener idempotent, add or enforce plugin lifecycle cleanup for process-level resources, and redesign session persistence so slow durable writes cannot block gateway handshakes or session lanes.

## Reproduction Assessment

Partly: the issue discussion has high-confidence field reproduction of the symptom class on v2026.4.26 and v2026.4.29, including WS probe timeouts, CPU peg, and SessionWriteLockTimeoutError. A deterministic current-main reproduction for all three root causes together is still unclear, especially because the Manifest listener source is not in this checkout.

## Solution Assessment

No. Current main contains useful mitigations and cleanup surfaces, but it does not yet remove the durable-write lock hold, prove Manifest listener idempotence, or centrally deduplicate arbitrary plugin-owned process resources.
