## Summary

Keep open. Current main has useful mitigations for session locks, transcript I/O, runtime-deps churn, and gateway handshakes, but the reported root causes are not fully closed: session-store saves still await durable writes while inside the store lock, plugin replacement cleanup remains registered-hook scoped, and no checked-in Manifest/clawrouter listener source was found to verify an idempotent 2099/2100 bind fix.

## What This Changes

The issue asks OpenClaw to fix gateway restart starvation caused by Manifest port-bind retries, accumulated process signal handlers, and session JSON persistence stalls.

## Best Possible Solution

Split the remaining work into targeted maintainer-owned fixes: make the Manifest/clawrouter listener startup idempotent, add or enforce cleanup/dedup for plugin-owned process resources, and redesign session-store persistence so slow durable writes cannot starve Gateway handshakes or session lanes.

## Reproduction Assessment

Unclear for the complete triad. The issue has strong logs and multiple v2026.4.29 confirmations, but I did not find a deterministic current-main repro or a checked-in Manifest listener implementation to exercise locally.

## Solution Assessment

No, current main is not yet the complete solution. The existing fixes reduce related failure modes, but the root-cause fix needs narrower lifecycle and persistence changes rather than only recovery after stalls.
