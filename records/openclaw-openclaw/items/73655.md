Keep this open. ## What This Changes

Fix the gateway restart leak triad that can starve WebSocket handshakes by addressing Manifest port rebinding, signal-handler accumulation, and session-store atomic-write lock stalls.

## Best Possible Solution

Keep this issue as the umbrella until maintainers either split or land the root-cause fixes: review the session-store fsync opt-out, locate and make Manifest/clawrouter listener startup idempotent, and add a generic plugin resource cleanup/dedup contract for process-level resources.

## Reproduction Assessment

## Solution Assessment

No single narrow fix is enough for the whole issue. The linked #76881 direction appears appropriate for the fsync branch, but the Manifest listener and signal-handler lifecycle branches still need maintainer-owned source identification and cleanup semantics.
