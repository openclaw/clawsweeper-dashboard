## Summary

Keep this open. Current main still shows the session-store writer awaiting durable atomic writes, plugin replacement cleanup remains limited to registered host-hook surfaces, and the reported Manifest/clawrouter listener source is not present in this checkout; the new #76881 PR is a useful candidate for the fsync/session-store slice but does not close the full triad.

## What This Changes

The issue reports a gateway restart/load regression where Manifest port collisions, signal-listener accumulation, and session-store write locking can starve WebSocket handshakes while HTTP health checks stay responsive.

## Best Possible Solution

Split the umbrella: review/land the #76881 session-store durability fix if acceptable, then track separate maintainer-owned fixes for Manifest listener idempotency and plugin/process-resource cleanup.
