## Summary

Keep open. The discussion now includes a targeted open fix and positive retest for the session-store fsync contributor, but current main in this checkout still awaits durable atomic fsync from the session-store writer and does not prove the Manifest port-bind or signal-listener roots are fixed.

## What This Changes

The issue reports a gateway restart/load regression where Manifest port rebinding, process signal listener growth, and session-store JSON persistence can starve WebSocket handshakes while HTTP health checks still pass.

## Best Possible Solution

Review and land or supersede #76881 for the session-store fsync contributor, then split the remaining Manifest port-bind and signal-handler leak roots into explicit owner-scoped fixes with regression coverage.

## Reproduction Assessment

Partial yes. The issue comments provide multiple real 2026.4.26/2026.4.29 reproductions, and current source still shows session-store writes awaiting atomic fsync inside the writer; I did not establish a live current-main reproduction for the full Manifest/signal/session triad.

## Solution Assessment

No. Current main has useful mitigations, and #76881 appears to target one confirmed contributor, but closing this issue before that fix is reviewed and the other two roots are fixed or split would be premature.
