## Summary

Keep this open. Current main has useful mitigations around gateway readiness, session listing, stuck-session recovery, and plugin runtime-deps churn, but the core root causes reported here are not proven fixed: session-store persistence still runs durable atomic write work inside the store writer critical section, plugin cleanup is still registered-hook scoped, and the Manifest/clawrouter listener source is not present in this checkout to verify an idempotent 2099/2100 bind fix.

## What This Changes

The issue requests fixes for gateway restart starvation caused by a Manifest port-bind retry loop, process signal-handler accumulation, and long session-store write-lock holds.

## Best Possible Solution

Split the work into maintainer-owned fixes: make the Manifest/clawrouter listener startup idempotent, add or enforce cleanup/dedup for plugin-owned process resources, and redesign session-store persistence so slow durable writes cannot starve gateway handshakes or session lanes.

## Reproduction Assessment

Partial yes. The latest comments give a high-confidence reproduction path for the session-store starvation symptom on v2026.4.29, but the full Manifest/signal/session triad still does not have a deterministic current-main reproducer.

## Solution Assessment

No. Current main contains mitigations and diagnostics, but it does not yet prove or implement the narrow root-cause fixes for all three reported resource leaks.
