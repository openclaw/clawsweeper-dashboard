## Summary

Keep this open. Current main has several useful mitigations around Gateway readiness, session-list responsiveness, runtime-deps churn, and stale-session diagnostics, but the reported root causes are not proven fixed: session-store writes still hold the store lock through durable atomic persistence, plugin restart cleanup is still hook-scoped, and the Manifest/clawrouter listener described in the report is not present in the checked-in plugin tree to verify an idempotent port bind. The latest issue comments also report v2026.4.29 reproductions with same-PID session-lock timeouts and event-loop saturation, so this is an active stability bug rather than stale cleanup.

## What This Changes

The issue asks OpenClaw to fix a Gateway restart/resource-leak cluster involving Manifest port rebinding, accumulating process signal handlers, and session JSON persistence lock contention that can starve WebSocket handshakes.

## Best Possible Solution

Split the umbrella into maintainer-owned root-cause fixes: make the Manifest/clawrouter listener startup idempotent, add or enforce a plugin lifecycle cleanup path that deduplicates process-level resources such as signal handlers and sockets, and redesign session-store persistence so slow durable writes cannot monopolize Gateway handshakes or session lanes.

## Reproduction Assessment

Unclear for the complete triad. The issue has strong field evidence and a clear test plan for listener counts, port ownership, and session-lock timing, but current main has not been exercised in a deterministic local reproduction and one reported component is not visible in the checked-in source.

## Solution Assessment

No current patch is under review, and current main is not yet the complete solution. The existing mitigations help diagnose or soften starvation, but the maintainable fix needs targeted lifecycle and persistence changes rather than only longer timeouts or post-stall recovery.
