## Summary

Keep this open. Current main has several relevant mitigations for startup readiness, event-loop diagnostics, runtime-deps churn, and stuck-session recovery, but the specific reported root causes are not fully closed: session-store persistence still runs inside the session-store lock, plugin replacement cleanup remains limited to registered host-hook/lifecycle cleanup, and the reported Manifest/clawrouter listener source is not present in the checked-in plugin tree.

## What This Changes

The issue asks OpenClaw to fix a gateway restart leak triad involving Manifest port rebinding on 127.0.0.1:2099/2100, accumulating process signal handlers, and slow session JSON persistence that can starve WebSocket handshakes while HTTP health remains green.

## Best Possible Solution

Split the remaining work into maintainer-owned fixes: make the Manifest/clawrouter listener startup idempotent wherever that packaged plugin lives, add or require a generic plugin lifecycle cleanup/dedup surface for process-level resources, and redesign session persistence so slow durable writes or maintenance cannot block latency-sensitive Gateway handshakes and session lanes.

## Reproduction Assessment

Unclear for the full triad. The report gives concrete logs, listener-count/lsof probes, and independent Linux confirmations, but I did not find a deterministic current-main reproduction path for all three root causes together.

## Solution Assessment

No, current main is not the complete solution. The shipped mitigations improve observability and reduce some startup/handshake pain, but the remaining fixes need source-level Manifest idempotence, broader plugin resource cleanup semantics, and a session-store persistence design decision.
