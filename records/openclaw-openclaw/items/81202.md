## Summary

Keep open: the PR is a plausible bug-fix direction, but the branch is not merge-ready because it reads untyped and unwritten `replyTurn*` session state and lacks real forced Telegram recovery proof.

## What This Changes

The PR adds diagnostic stale reply-turn recovery that scans session stores, sends a Telegram fallback via `message.action`, marks the turn failed, and covers it with a regression test.

## Best Possible Solution

Add or reuse a durable typed turn-state contract with a production writer, send the fallback through the current `message.action` schema, then prove a forced stale Telegram topic recovery in a real setup.

## Reproduction Assessment

No for the reported live Telegram stale-turn silence: the PR body provides checks and status only, not a forced recovery. Yes for the review finding: source inspection shows the branch reads `replyTurn*` fields that current `SessionEntry` and production writers do not define.

## Solution Assessment

No. The nested `message.action` schema is the right direction, but recovery must be connected to real persisted turn state instead of test-only fields before this can be the maintainable fix.
