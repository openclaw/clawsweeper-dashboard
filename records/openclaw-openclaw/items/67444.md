## Summary

Keep open: current main still suppresses `ANNOUNCE_SKIP` completion delivery, and this PR targets the right path but is not mergeable as-is because the fallthrough forwards the `ANNOUNCE_SKIP` control token as completion content and leaves stale test/docs/changelog work.

## What This Changes

This PR changes subagent announce handling so `ANNOUNCE_SKIP` no longer returns early before inter-session completion delivery and updates announce tests around that behavior.

## Best Possible Solution

Rebase the branch, preserve `NO_REPLY` and non-completion `ANNOUNCE_SKIP` suppression, clear or replace the `ANNOUNCE_SKIP` control token before building completion findings, and add positive payload coverage plus docs/changelog updates.

## Reproduction Assessment

Yes. The source-level path is high confidence: current main with `expectsCompletionMessage: true` and `roundOneReply: "ANNOUNCE_SKIP"` returns before delivery, while the PR head reaches delivery but would build the completion result from the unchanged control token.

## Solution Assessment

No. The PR chooses the right owner module and the right early-return seam, but the implementation needs to clear the control token and align tests/docs/changelog before it is the narrow maintainable fix.
