## Summary

Keep this PR open because current main still suppresses ANNOUNCE_SKIP before the completion-delivery call, so the branch is not obsolete. The submitted patch is not mergeable as-is because its fallthrough path can forward the literal ANNOUNCE_SKIP control token as completion content, and the user-facing docs/changelog are not aligned with the changed behavior.

## What This Changes

This PR changes subagent announce handling so ANNOUNCE_SKIP no longer returns early before completion delivery and updates announce tests around ANNOUNCE_SKIP and NO_REPLY delivery.

## Best Possible Solution

Rebase the branch, keep NO_REPLY and non-completion ANNOUNCE_SKIP suppression intact, allow ANNOUNCE_SKIP to skip only the soft announce step when deterministic completion delivery is required, clear or replace the control token before building findings, and add positive payload coverage plus docs/changelog updates.

Yes. The source-level path is high confidence: runSubagentAnnounceFlow with expectsCompletionMessage true and roundOneReply ANNOUNCE_SKIP reaches the current main early return before delivery, and current tests assert no agent call.

## Solution Assessment

No. The PR targets the right code path, but the current implementation is not the best fix because it decouples the early return without clearing the control token or updating the public behavior contract.
