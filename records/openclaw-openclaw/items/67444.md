## Summary

Keep this PR open: current main still suppresses `ANNOUNCE_SKIP` completion delivery, and no merged replacement was found, but the PR is not mergeable as-is because the branch can forward the control token as completion result content and still lacks real behavior proof.

## What This Changes

This PR changes subagent announce handling so completion-mode `ANNOUNCE_SKIP` falls through to inter-session delivery and updates announce-related tests.

## Best Possible Solution

Clear or replace the `ANNOUNCE_SKIP` control token before completion result construction, preserve `NO_REPLY` and non-completion suppression, add positive delivery-payload coverage, and align the public docs with the completion-mode exception.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main returns before delivery for `expectsCompletionMessage: true` plus `roundOneReply: "ANNOUNCE_SKIP"`, while the PR head reaches delivery with `reply` still available for result construction.

## Solution Assessment

No. The owner module is the right place, but this patch needs to remove the control token from delivered result content, repair the e2e/dependency-injection coverage, and document the completion-mode exception before it is the narrow maintainable fix.
