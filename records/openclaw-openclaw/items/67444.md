## Summary

Keep open: the PR targets a plausible subagent completion-delivery bug that is not fixed on current main, but the current patch still forwards the `ANNOUNCE_SKIP` control token into delivery content and lacks real after-fix behavior proof.

## What This Changes

This PR changes subagent announce handling so completion-mode `ANNOUNCE_SKIP` falls through to inter-session delivery and updates announce-related tests.

## Best Possible Solution

Clear or replace the `ANNOUNCE_SKIP` control token before building completion findings, preserve `NO_REPLY` and non-completion suppression, add positive delivery payload coverage, and update the docs for the completion-mode exception.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main returns before delivery for `expectsCompletionMessage: true` plus `roundOneReply: "ANNOUNCE_SKIP"`, while the PR head reaches delivery with the unchanged control token still available as `reply`.

## Solution Assessment

No. The owner module and early-return seam are the right place, but this patch needs to clear the control token, repair the e2e/dependency-injection tests, and align the docs before it is the narrow maintainable fix.
