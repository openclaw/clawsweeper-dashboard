## Summary

Keep open. Current main still suppresses `ANNOUNCE_SKIP` completion delivery, so the PR is useful, but the PR head is not mergeable as-is because the fallthrough would forward the control token as completion content and the test/docs/changelog changes are incomplete.

## What This Changes

This PR changes subagent announce handling so `ANNOUNCE_SKIP` no longer returns early before inter-session completion delivery and updates announce tests around that behavior.

## Best Possible Solution

Land a narrow rebased fix in the subagent announce owner module that preserves `NO_REPLY` and non-completion suppression, clears `ANNOUNCE_SKIP` before completion findings, and updates positive coverage plus docs and changelog.

## Reproduction Assessment

Yes. The source-level path is high confidence: current main with `expectsCompletionMessage: true` and `roundOneReply: "ANNOUNCE_SKIP"` returns before delivery, while the PR head reaches delivery but would build the completion result from the unchanged control token.

## Solution Assessment

No. The PR changes the right seam, but it needs to clear or replace the control token and align tests/docs/changelog before it is the narrow maintainable solution.
