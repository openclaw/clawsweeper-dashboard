## Summary

Keep open: current main still suppresses completion delivery for `ANNOUNCE_SKIP`, and this PR targets the right announce-flow seam, but the PR head forwards the control token as completion content, leaves stale/weak test coverage, and lacks real after-fix behavior proof.

## What This Changes

The PR changes subagent announce handling so completion-mode `ANNOUNCE_SKIP` falls through to inter-session delivery and updates announce-related tests.

## Best Possible Solution

Update the PR to clear or replace the `ANNOUNCE_SKIP` control token before building completion findings, preserve `NO_REPLY` and non-completion suppression, add positive delivery payload coverage, and document the completion-mode exception.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main returns before delivery for `expectsCompletionMessage: true` plus `roundOneReply: "ANNOUNCE_SKIP"`, while the PR head reaches delivery with the unchanged control token still available as `reply`.

## Solution Assessment

No. The owner module and early-return seam are appropriate, but this patch needs to clear the control token and repair the affected tests before it is the narrow maintainable fix.
