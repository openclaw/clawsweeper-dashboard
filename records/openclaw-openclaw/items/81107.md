## Summary

Keep open: current main still builds Discord native skill commands on the provider startup path, and duplicate skill names can be uniqued/logged before the final by-skill-name dedupe runs. The broader READY reports are useful context, but this issue adds a distinct startup-load path that is not clearly fixed or safely duplicate-triaged.

## What This Changes

The issue requests fixing a v2026.5.7 Discord startup path where native skill-command deduplication loops over repeated `/discord_N` names, saturates CPU, and blocks gateway readiness.

## Best Possible Solution

Collapse duplicate skill identities before Discord native command-name uniquing and deploy-count work, while preserving per-agent allowlists, shared-workspace behavior, reserved built-in names, and the generic `/skill` fallback.
