## Repair Work Prompt

Fix the Discord native skill command startup-load bug reported in https://github.com/openclaw/openclaw/issues/81107. On current main, Discord resolves native command specs before lifecycle startup, calls `listSkillCommandsForAgents({ cfg })`, and `listSkillCommandsForAgents` builds suffix-uniqued workspace command specs before the final `dedupeBySkillName` filter. Collapse duplicate skill identities before command-name uniquing/native command generation, or otherwise avoid generating/logging duplicate command specs for the same `skillName`, while preserving per-agent skill allowlists, shared-workspace merge semantics, and the generic `/skill` fallback. Add focused regression coverage proving duplicate skills across workspaces do not inflate native command specs or emit suffix de-dupe work before the final list. Do not change Discord gateway READY lifecycle, plugin install flow, or add a new config option. Add a changelog entry for the user-visible startup fix.
