## Summary

Keep open: current main still runs Discord native skill discovery on the Discord startup path, and duplicate skill names are still uniqued/logged before final by-skillName de-dupe removes them. The related Discord READY and 1006 reports are useful context, but they do not specifically own this remaining skill-command startup-load path.

## What This Changes

The issue reports that external Discord skill command discovery repeatedly suffixes duplicate skill command names during startup, saturating CPU and preventing Discord gateway readiness on v2026.5.7.

## Best Possible Solution

Collapse duplicate skill identities before slash-command name uniquing and native Discord command generation, while preserving per-agent allowlists and the generic `/skill` fallback.

## Reproduction Assessment

No for the full live CPU-saturation plus Discord WebSocket hang on current main. Source inspection does show the current-main path where duplicate skill names are uniqued and logged before final by-skillName de-dupe removes them.

## Solution Assessment

Yes: the maintainable fix direction is a narrow source change that avoids duplicate skill-command work before Discord startup deploys native commands. A config workaround or broader gateway lifecycle change would not address the duplicate command-generation path directly.
