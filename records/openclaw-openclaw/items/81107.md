Keep open: current main still has the source path where Discord native skill discovery runs during startup and duplicate skill command names are uniqued/logged before the final by-skillName de-dupe. ## What This Changes

The issue reports that the external Discord plugin on v2026.5.7 repeatedly generates duplicate native skill command names, saturating CPU and preventing Discord gateway readiness.

## Best Possible Solution

Collapse duplicate skill identities before slash-command name uniquing and native Discord command generation, while preserving per-agent allowlists and the generic `/skill` fallback.

## Reproduction Assessment

No for the full live CPU-saturation plus Discord WebSocket hang on current main. Source inspection does show the current-main path where duplicate skill names are uniqued and logged before final by-skillName de-dupe removes them.

## Solution Assessment

Yes: the maintainable fix direction is a narrow skill-command source change that avoids duplicate command work before Discord startup deploys native commands. A config workaround or broader gateway lifecycle change would not address this duplicate generation path directly.
