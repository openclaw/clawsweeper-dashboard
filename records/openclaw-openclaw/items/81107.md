## Summary

Keep open: current main and the latest shipped release still keep Discord native skill discovery on the Discord startup path and still perform command-name uniquing/logging before the final by-skillName de-dupe. The older READY reports are useful context, but no canonical open item or merged fix owns this remaining skill-command startup-load path.

## What This Changes

The issue reports that the external Discord plugin on v2026.5.7 repeatedly generates duplicate native skill command names during startup, saturating CPU and preventing Discord gateway readiness.

## Best Possible Solution

Deduplicate skill identities before per-workspace command-name uniquing and native Discord command generation, while preserving per-agent allowlists, shared-workspace merging, reserved command names, and the generic `/skill` fallback.

## Reproduction Assessment

No for the full live CPU-saturation plus Discord WebSocket hang on current main. Source inspection gives a medium-confidence deterministic path where duplicate skill names are uniqued and logged before final by-skillName de-dupe removes them.

## Solution Assessment

Yes: the narrow maintainable fix is to avoid duplicate skill-command work before Discord startup deploys native commands. A config workaround or broader gateway lifecycle change would not directly address the duplicate command generation path.
