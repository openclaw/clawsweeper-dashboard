## Summary

Keep open: current main and latest release v2026.5.12 still run Discord native skill discovery through the global skill-command lister before Discord lifecycle startup, and duplicate skill names are still uniqued/logged before the final by-skillName de-dupe. The related READY fixes are useful context, but they do not fully own this remaining startup-load duplicate skill path.

## What This Changes

The issue requests a fix for Discord native skill-command duplicate generation that can saturate startup and block gateway readiness after installing the external Discord plugin.

## Best Possible Solution

Collapse duplicate skill identities before command-name uniquing and native Discord command generation, while preserving per-agent allowlists and the generic `/skill` fallback.

## Reproduction Assessment

No for the full live CPU-saturation plus Discord WebSocket hang on current main. Source inspection does confirm the current-main and v2026.5.12 path where native Discord startup calls global skill command listing and duplicate command names are uniqued/logged before final de-dupe.

## Solution Assessment

Yes: the narrow maintainable fix is to remove duplicate skill-command work before Discord native command generation. A broader Discord lifecycle change would not address the duplicate skill-name generation path directly.
