## Summary

One merge-blocking security defect remains in the trusted commenter: it downloads and buffers a pull-request-produced artifact before enforcing any compressed archive-size limit. The Swift cleanup, Periphery scan, exact-head gates, and post-download ZIP validation otherwise have strong source and real-run support.

## What This Changes

The PR removes unused iOS declarations and tests, adds a Periphery dead-code gate for iOS pull requests, and adds a trusted workflow that comments detected symbols on the pull request.

## Best Possible Solution

Reject artifacts above a conservative compressed-size cap using `artifact.size_in_bytes` before download, retain the existing identity and ZIP-entry checks, and add an oversized-artifact rejection case before landing the verified cleanup.

## Reproduction Assessment

Yes, from source: pull-request-controlled workflow code can upload a large artifact under the expected run-id name, and the trusted job reaches `downloadArtifact` and `Buffer.from` before enforcing any archive-size limit.

## Solution Assessment

Mostly yes: Periphery is a suitable whole-project Swift dead-code gate and the cleanup is well supported, but the safest implementation must bound artifact transfer before download rather than relying only on post-download ZIP checks.
