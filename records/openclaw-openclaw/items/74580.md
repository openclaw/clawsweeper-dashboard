## Summary

Keep open. Current main still gates non-bundled `agent_end` behind `allowConversationAccess`, and that payload still contains raw `messages`; the remaining request is a valid state-only Plugin SDK permission/API design issue, not cleanup.

## What This Changes

Add a state-only permission or event so plugins can observe an agent turn ending without granting full conversation access.

## Best Possible Solution

Expose an additive sanitized lifecycle or turn-ended SDK contract carrying only run/session/state metadata, while keeping raw `agent_end` messages behind `allowConversationAccess`.

## Reproduction Assessment

Yes. Load a non-bundled plugin that calls `api.on("agent_end", ...)` without `plugins.entries.<id>.hooks.allowConversationAccess=true`; `registerTypedHook` drops it and `src/plugins/loader.test.ts` asserts the same behavior.

## Solution Assessment

Unclear. `allowStateAccess` could work only with a split or sanitized payload; reusing the existing `agent_end` event is not the best solution because it carries raw `messages`.
