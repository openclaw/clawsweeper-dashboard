## Summary

Keep open. Current main still gates `agent_end` as a raw conversation hook, has no `allowStateAccess` config or Plugin SDK contract, and the only direct implementation PR was closed unmerged; the active `agent:turn:end` PR is related lifecycle work but does not resolve this external plugin permission boundary.

## What This Changes

Request a state-only Plugin SDK permission or lightweight turn-ended signal so non-bundled plugins can observe agent turn completion without full raw conversation access.

## Best Possible Solution

Define an additive, documented Plugin SDK state or turn-ended contract that exposes only session/run status metadata while keeping raw `agent_end.messages` behind `allowConversationAccess`.

## Reproduction Assessment

Yes for the current limitation. Source inspection shows a non-bundled plugin registering `agent_end` is still blocked unless `allowConversationAccess=true`, and the loader test asserts that behavior.

## Solution Assessment

Unclear. `allowStateAccess` is plausible, but maintainers should choose the permission/API shape before changing a public Plugin SDK security boundary.
