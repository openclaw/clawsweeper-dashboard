## Summary

Keep open: current main and the latest release still require `allowConversationAccess` for non-bundled `agent_end`, and the issue is now paired with an open implementation candidate at https://github.com/openclaw/openclaw/pull/96556. The safe cleanup path is to review, land, or explicitly reject that PR before closing this issue.

## What This Changes

Request a granular plugin hook permission, or equivalent documented lifecycle contract, so state-only `agent_end` tracking does not require raw conversation access.

## Best Possible Solution

Resolve the open implementation candidate at https://github.com/openclaw/openclaw/pull/96556 with a maintainer-approved state-only `agent_end` contract, or choose a documented lightweight lifecycle alternative that keeps raw messages behind `allowConversationAccess`.

## Reproduction Assessment

Yes, by source inspection: `agent_end` is currently a conversation hook, non-bundled conversation hooks require `allowConversationAccess=true`, and no `allowStateAccess` surface exists on current main. I did not run a live external plugin repro in this read-only review.

## Solution Assessment

Not yet; the state-only permission is a plausible focused solution, but the best fix must be settled in the linked PR with security review and full config/schema/docs/startup/status coverage. Closing the issue before that PR lands would be premature.
