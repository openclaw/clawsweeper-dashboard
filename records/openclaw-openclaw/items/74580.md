## Summary

Keep open. Current main still lacks a dedicated state-only Plugin SDK turn-ended contract: raw `agent_end` remains conversation-gated because it carries `messages`, while the available lifecycle event subscription is only a generic sanitized monitor path and the issue discussion raises unresolved cadence/API-design concerns.

## What This Changes

Request to add a state-only Plugin SDK hook permission or equivalent lightweight lifecycle signal so external plugins can detect agent turn completion without full conversation access.

## Best Possible Solution

Define an additive, documented Plugin SDK lifecycle or turn-ended contract that exposes only minimal run/session/phase/timing metadata while keeping raw `agent_end.messages` behind `allowConversationAccess`.

## Reproduction Assessment

Yes. Source inspection on current main shows a non-bundled plugin registering `api.on("agent_end", ...)` is still blocked without `plugins.entries.<id>.hooks.allowConversationAccess=true`, and the loader test asserts that gate.

## Solution Assessment

Unclear. `allowStateAccess` is plausible, but the safer maintainable solution is likely a sanitized turn-ended/lifecycle event or an explicitly blessed lifecycle subscription recipe rather than weakening access to the existing `agent_end` payload.
