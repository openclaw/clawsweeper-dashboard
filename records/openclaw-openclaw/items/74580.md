## Summary

Keep open. Current main still treats `agent_end` as a raw-conversation hook, and the available lifecycle event subscription is only a partial workaround rather than a decided state-only Plugin SDK permission contract.

## What This Changes

Request a state-only agent lifecycle or turn-end plugin permission/API so external display/status plugins can observe agent state without granting full conversation access.

## Best Possible Solution

Keep raw-content hooks behind `allowConversationAccess` and add or clearly document a supported state-only lifecycle/turn-ended contract exposing only run/session identity, phase/status, timing, and terminal state.

## Reproduction Assessment

Yes. A non-bundled plugin that registers `api.on("agent_end", ...)` without `plugins.entries.<id>.hooks.allowConversationAccess=true` is blocked by `registerTypedHook`, and `src/plugins/loader.test.ts` locks that behavior.

## Solution Assessment

Unclear. `allowStateAccess` is plausible, but the safer design is a sanitized event or lifecycle subscription contract rather than relaxing the existing `agent_end` hook that carries `messages`.
