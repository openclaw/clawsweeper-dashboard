## Summary

Keep open. Current main still lacks a state-only `agent_end` permission or sanitized turn-ended Plugin SDK contract; the existing conversation gate is source-verified and covered by tests/docs, while the OpenClaw Interaction Bridge use case is a real external plugin workflow that needs maintainer API direction.

## What This Changes

Add a state-only hook permission or equivalent state-only lifecycle contract so non-bundled plugins can observe agent turn completion without full conversation access.

## Best Possible Solution

Define an additive, documented Plugin SDK turn/lifecycle signal that exposes only run/session/status metadata while keeping raw `agent_end.messages` behind `allowConversationAccess`.

## Reproduction Assessment

Yes. Source inspection on current main shows a non-bundled plugin registering `api.on("agent_end", ...)` is still blocked unless `allowConversationAccess=true`, and the loader regression test asserts that behavior.

## Solution Assessment

Unclear. `allowStateAccess` is plausible, but this changes a public Plugin SDK permission boundary, so maintainers should first choose between a sanitized `agent_end`, a new turn-ended event, or lifecycle subscription guidance.
