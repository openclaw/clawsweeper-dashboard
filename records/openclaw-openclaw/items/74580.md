## Summary

Keep open. Current main still treats `agent_end` as a raw conversation hook, has no `allowStateAccess` config surface, and the related allowStateAccess PR was closed unmerged; the open `agent:turn:end` PR is related lifecycle work but does not solve this external Plugin SDK permission boundary.

## What This Changes

Add a state-only Plugin SDK permission or turn-ended contract so non-bundled plugins can observe agent lifecycle state without full conversation access.

## Best Possible Solution

Define an additive, documented Plugin SDK state or turn-ended contract that exposes only session/run status metadata while keeping raw `agent_end.messages` behind `allowConversationAccess`.

## Reproduction Assessment

Yes for the current limitation. Source inspection shows a non-bundled plugin registering `agent_end` is still blocked unless `allowConversationAccess=true`, and the loader test asserts that behavior.

## Solution Assessment

Unclear. `allowStateAccess` is plausible, but maintainers should choose between a sanitized `agent_end`, a new lifecycle/turn-ended event, or documented event-subscription guidance before changing the public permission contract.
