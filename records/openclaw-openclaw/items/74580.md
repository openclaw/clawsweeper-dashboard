## Summary

Keep open. Current main still requires `allowConversationAccess` for non-bundled `agent_end`, no `allowStateAccess` config surface exists, the direct implementation PR was closed unmerged, and the open internal-hook PR is adjacent but does not settle this external Plugin SDK permission boundary.

## What This Changes

Request a state-only Plugin SDK hook permission or equivalent lifecycle signal so external plugins can observe turn completion without granting raw conversation access.

## Best Possible Solution

Define an additive, documented Plugin SDK state or turn-ended contract that exposes only session/run lifecycle metadata while keeping raw messages behind `allowConversationAccess`.

## Reproduction Assessment

Yes, for the current limitation. Source inspection shows non-bundled `agent_end` registration is still blocked unless `allowConversationAccess=true`, and the loader test asserts that behavior.

## Solution Assessment

Unclear. `allowStateAccess` is plausible, but maintainers should choose between a sanitized `agent_end`, a new plugin-visible turn-ended event, or documented lifecycle subscription guidance before changing the public permission contract.
