## Summary

Keep open. Current main still requires `allowConversationAccess` for non-bundled `agent_end`, the latest release lacks `allowStateAccess`, the related https://github.com/openclaw/openclaw/pull/78242 was closed unmerged, and the open https://github.com/openclaw/openclaw/pull/71399 is related turn-lifecycle work but does not solve this external Plugin SDK permission boundary.

## What This Changes

Request a granular state-only Plugin SDK hook permission or equivalent turn-ended state signal so external plugins can observe `agent_end` without raw conversation access.

## Best Possible Solution

Define an additive, documented Plugin SDK state or turn-ended contract that exposes only session/run status metadata while keeping raw `agent_end.messages` behind `allowConversationAccess`.

## Reproduction Assessment

Yes, for the current limitation. Source inspection shows non-bundled `agent_end` registration is still blocked unless `allowConversationAccess=true`, and the loader test asserts that behavior.

## Solution Assessment

Unclear. `allowStateAccess` is plausible, but maintainers should choose between a sanitized `agent_end`, a new lifecycle/turn-ended event, or documented event-subscription guidance before changing the public permission contract.
