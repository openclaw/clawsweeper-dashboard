## Summary

Keep open. Current main still has no state-only `agent_end` permission or documented lightweight turn-ended Plugin SDK contract; the existing sanitized agent-event subscription path is relevant, but the issue discussion narrows the remaining ask to a public state-only end-of-turn surface that does not require raw conversation access.

## What This Changes

Request a state-only Plugin SDK hook permission or turn-ended lifecycle signal so non-bundled plugins can track agent state without granting `allowConversationAccess`.

## Best Possible Solution

Define an additive, documented Plugin SDK lifecycle or turn-ended contract that exposes only minimal run/session/phase/status metadata while keeping raw `agent_end.messages` behind `allowConversationAccess`.

## Reproduction Assessment

Yes. Source inspection shows a non-bundled plugin registering `api.on("agent_end", ...)` is still blocked unless `plugins.entries.<id>.hooks.allowConversationAccess=true`, and the loader test asserts that behavior.

## Solution Assessment

Unclear. `allowStateAccess` is plausible, but the safer maintainable fix is probably a sanitized turn-ended/lifecycle event or filtered payload chosen by maintainers before changing the permission boundary.
