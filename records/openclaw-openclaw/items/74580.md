## Summary

Keep open. Current main still treats non-bundled `agent_end` as a full conversation hook, the payload still includes raw `messages`, and there is no state-only permission or documented turn-ended contract for the linked hardware/display plugin use case.

## What This Changes

Request to add a state-only way for non-bundled plugins to observe agent turn completion, narrowed in discussion to `agent_end`, without granting full conversation access.

## Best Possible Solution

Keep raw-content hooks behind `allowConversationAccess` and add or document an additive, sanitized turn-ended/lifecycle contract that exposes only run/session/state/timing fields, with contract tests proving no conversation text reaches state-only consumers.

## Reproduction Assessment

Yes. A non-bundled plugin that registers `api.on("agent_end", ...)` without `plugins.entries.<id>.hooks.allowConversationAccess=true` is rejected on current main; `src/plugins/loader.test.ts:6795` already codifies that path.

## Solution Assessment

Unclear. A new `allowStateAccess` flag is plausible, but because the current `agent_end` payload includes raw `messages`, a sanitized turn-ended event or an explicitly documented agent-event subscription contract is the safer maintainable direction.
