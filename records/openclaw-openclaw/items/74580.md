## Summary

Keep open: current main still forces non-bundled `agent_end` through the raw-conversation `allowConversationAccess` gate, while the only direct implementation attempt is closed unmerged and the existing sanitized lifecycle event surface needs an explicit maintainer decision before it can replace the requested state-only contract.

## What This Changes

The issue requests a state-only hook permission or equivalent lifecycle signal so plugins can observe agent turn end without full conversation access.

## Best Possible Solution

Keep this issue open as the canonical state-only hook permission decision and have maintainers choose between a documented lifecycle subscription contract, a state-only `agent_end` payload, or a new permission flag with normalization, docs, tests, and proof.

## Reproduction Assessment

Yes for the current-state check. Source inspection shows `agent_end` is still in `CONVERSATION_HOOK_NAMES`, non-bundled conversation hooks are still blocked unless `allowConversationAccess` is true, and there is no `allowStateAccess` schema/config surface on current main.

## Solution Assessment

Unclear as an implementation direction. The problem is real, but the best fix needs maintainer approval because current main already exposes sanitized lifecycle events while the proposed flag would add a new security-sensitive config and hook contract.
