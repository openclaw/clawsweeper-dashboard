## Summary

Keep open. Current main still treats non-bundled `agent_end` as a raw conversation hook, and there is no `allowStateAccess` or state-only `agent_end` payload. The existing agent-event subscription path is a plausible workaround, but the reporter's follow-up narrows a real public plugin API gap that needs maintainer design before changing the permission boundary.

## What This Changes

The issue requests a granular state-only way for non-bundled plugins to observe agent turn completion without granting full conversation access.

## Best Possible Solution

Keep raw-content hooks behind `allowConversationAccess`, then add or explicitly document an additive state-only turn-ended or lifecycle contract that exposes only sanitized run, session, state, and timing fields, with contract tests proving no conversation text reaches state-only consumers.

## Reproduction Assessment

Yes. A non-bundled plugin that registers `api.on("agent_end", ...)` without `plugins.entries.<id>.hooks.allowConversationAccess=true` is rejected on current main, and the loader test coverage codifies that behavior.

## Solution Assessment

Unclear. `allowStateAccess` is plausible, but exposing the current `agent_end` event under that flag would be unsafe because it carries `messages`; a sanitized turn-ended event or documented lifecycle subscription contract is the safer maintainable direction.
