## Summary

Keep open: current main and v2026.6.8 still require full `allowConversationAccess` for external `agent_end`, while `allowStateAccess` does not exist and the existing lifecycle event subscription path is not documented or settled as the lightweight turn-ended state contract.

## What This Changes

The issue requests a state-only plugin hook permission or equivalent lightweight lifecycle signal so external state/display plugins can observe turn completion without requesting raw conversation access.

## Best Possible Solution

Maintainers should choose the durable contract: either document lifecycle event subscriptions as the supported state-only turn signal, or add a state-only hook/event permission that keeps raw messages behind conversation access.

## Reproduction Assessment

Yes, source-based. Current main and v2026.6.8 still classify `agent_end` as a conversation hook, require `allowConversationAccess` for non-bundled registration, and provide no `allowStateAccess` config or SDK contract.

## Solution Assessment

Unclear. The proposed `allowStateAccess` split is plausible, but the best fix needs maintainer direction between a new state-only permission, a filtered `agent_end` payload, a new lightweight event, or documenting the existing sanitized lifecycle subscription path.
