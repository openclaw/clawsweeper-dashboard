## Summary

Keep open. Current main still gates non-bundled `agent_end` behind full conversation access, and the payload still includes raw `messages`; the existing lifecycle event subscription is a relevant workaround, but the reporter narrowed the ask to a plugin API and permission-boundary decision rather than cleanup.

## What This Changes

Request a fine-grained plugin hook permission or sanitized lifecycle signal so non-bundled plugins can observe agent turn-end state without full conversation access.

## Best Possible Solution

Keep raw-content hooks behind `allowConversationAccess` and either add a minimal permission-free turn-ended/lifecycle contract carrying only run, session, state, timing, and failure metadata, or explicitly bless `registerAgentEventSubscription` for this use case with docs and contract tests proving no conversation text reaches that path.

## Reproduction Assessment

Yes. A non-bundled plugin registering `api.on("agent_end", ...)` without `plugins.entries.<id>.hooks.allowConversationAccess=true` reproduces the block; the current loader test already codifies that behavior.

## Solution Assessment

Unclear. `allowStateAccess` is plausible, but not on the current `agent_end` payload as-is; a sanitized event or documented lifecycle subscription path is the safer maintainable direction.
