## Summary

Keep open. Current main still lacks a state-only Plugin SDK turn-ended contract; raw `agent_end` remains conversation-gated because it carries `messages`, and the existing lifecycle event subscription path is adjacent but not the documented lightweight per-turn API requested here.

## What This Changes

Request a granular state-only plugin permission or sanitized turn-ended Plugin SDK event so hardware/status plugins can observe agent turn completion without full conversation access.

## Best Possible Solution

Add an additive, documented Plugin SDK lifecycle or turn-ended contract that exposes only minimal run/session/phase/status metadata while keeping raw `agent_end.messages` behind `allowConversationAccess`.

## Reproduction Assessment

Yes. A non-bundled plugin that registers `api.on("agent_end", ...)` without `plugins.entries.<id>.hooks.allowConversationAccess=true` is still rejected by `registerTypedHook`, and the loader regression test locks in that behavior.

## Solution Assessment

Unclear. `allowStateAccess` is plausible, but the safer maintainable shape is likely a sanitized turn-ended event or explicitly documented lifecycle subscription rather than weakening access to the existing `agent_end` payload.
