## Summary

Keep open. Current main and v2026.6.6 still gate non-bundled `agent_end` behind `allowConversationAccess`, no `allowStateAccess` config surface exists, the direct implementation PR is closed unmerged, and the open split hook PRs are adjacent internal ACP hook work rather than this external Plugin SDK permission contract.

## What This Changes

Request a fine-grained plugin hook permission or equivalent state-only turn-ended signal so external plugins can observe agent lifecycle state without raw conversation access.

## Best Possible Solution

Define an additive, documented Plugin SDK state or turn-ended contract that exposes only session/run lifecycle metadata while keeping raw messages behind `allowConversationAccess`.

## Reproduction Assessment

Not applicable as a bug repro, but the current limitation is source-reproducible: `agent_end` is gated as a conversation hook and no state-only config key exists on current main or v2026.6.6.

## Solution Assessment

Unclear until maintainer direction is set. `allowStateAccess` is plausible, but maintainers should choose between a sanitized `agent_end`, a new plugin-visible turn-ended event, or formally documenting lifecycle subscriptions as the supported path.
