## Summary

Keep open. Current main and v2026.6.8 still gate non-bundled `agent_end` behind `allowConversationAccess`, no `allowStateAccess` config or schema surface exists, and the related ACP hook PRs are adjacent internal lifecycle work rather than this external Plugin SDK permission contract.

## What This Changes

Request a state-only hook permission or equivalent Plugin SDK contract so external plugins can observe agent turn/lifecycle state without raw conversation access.

## Best Possible Solution

Define an additive, documented Plugin SDK state or turn-ended contract that exposes only lifecycle metadata while keeping raw messages behind `allowConversationAccess`.

## Reproduction Assessment

Yes. Source inspection shows `agent_end` is still in `CONVERSATION_HOOK_NAMES`, non-bundled typed hook registration still requires `allowConversationAccess`, and no `allowStateAccess` schema field exists.

## Solution Assessment

Unclear until maintainer direction is set. `allowStateAccess` is plausible, but maintainers should choose between a sanitized `agent_end`, a new plugin-visible turn-ended event, or a documented lifecycle subscription contract.
