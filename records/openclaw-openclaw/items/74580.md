## Summary

Keep open. Current main still lacks a state-only Plugin SDK turn-ended contract; raw `agent_end` remains conversation-gated because it carries `messages`, and the issue discussion gives a real external plugin use case that should not require full conversation access.

## What This Changes

Request a granular state-only plugin permission or sanitized turn-ended Plugin SDK event so hardware/status plugins can observe agent turn completion without full conversation access.

## Best Possible Solution

Define an additive, documented Plugin SDK lifecycle or turn-ended contract that exposes only minimal run/session/phase/status metadata while keeping raw `agent_end.messages` behind `allowConversationAccess`.

## Reproduction Assessment

Yes. Source inspection shows a non-bundled plugin registering `api.on("agent_end", ...)` is still blocked unless `plugins.entries.<id>.hooks.allowConversationAccess=true`, and the loader regression test asserts that behavior.

## Solution Assessment

Unclear. `allowStateAccess` is plausible, but the safer maintainable shape is likely a sanitized turn-ended event or explicitly documented lifecycle subscription rather than weakening access to the existing `agent_end` payload.
