## Summary

Keep open. Current main still has no state-only `agent_end` permission or documented turn-ended Plugin SDK contract; the existing agent-event subscription path is relevant but does not fully settle the requested lightweight public API or permission boundary.

## What This Changes

Request a state-only hook permission or API so non-bundled plugins can observe agent turn completion without granting raw conversation access.

## Best Possible Solution

Add an additive, documented state-only lifecycle or turn-ended SDK contract that exposes only minimal run/session/phase/timing metadata while keeping raw `agent_end.messages` behind `allowConversationAccess`.

## Reproduction Assessment

Yes. A non-bundled plugin that registers `api.on("agent_end", ...)` without `plugins.entries.<id>.hooks.allowConversationAccess=true` is still rejected by `registerTypedHook`, and `src/plugins/loader.test.ts` codifies that behavior.

## Solution Assessment

Unclear. `allowStateAccess` is plausible, but the safer maintainable solution is likely a sanitized turn-ended event or documented lifecycle subscription rather than granting a new permission to the current `agent_end` payload.
