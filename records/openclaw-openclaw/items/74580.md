## Summary

Keep open. Current main still lacks a state-only Plugin SDK turn-ended contract: raw `agent_end` remains conversation-gated, the payload still carries `messages`, and the external plugin use case is concrete enough to warrant maintainer design follow-up.

## What This Changes

Request a state-only hook permission or lightweight lifecycle signal so plugins can detect agent state/turn completion without full conversation access.

## Best Possible Solution

Define an additive, documented Plugin SDK state-only turn/lifecycle contract that exposes minimal run/session/phase/timing metadata while keeping raw `agent_end.messages` behind `allowConversationAccess`.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a non-bundled plugin registering `api.on("agent_end", ...)` is rejected unless `plugins.entries.<id>.hooks.allowConversationAccess=true`, and the loader test asserts that behavior.

## Solution Assessment

Unclear. `allowStateAccess` is plausible, but the safer maintainable solution may be a sanitized turn-ended or lifecycle event rather than weakening access to the existing raw `agent_end` payload.
