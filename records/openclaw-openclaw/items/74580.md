## Summary

Keep open. Current main still gates non-bundled `agent_end` behind full conversation access because that payload includes raw `messages`; the existing lifecycle event subscription path is relevant, but the requested state-only turn-end contract remains an SDK permission/security design decision.

## What This Changes

The issue requests a state-only plugin permission or equivalent lightweight lifecycle signal so non-bundled plugins can observe agent turn completion without full conversation access.

## Best Possible Solution

Keep raw-content hooks behind `allowConversationAccess` and add or document an additive state-only lifecycle contract that exposes only run/session identity, phase, timing, and terminal status, with tests proving no prompts, responses, history, tool payloads, or `messages` reach state-only consumers.

## Reproduction Assessment

Yes. A non-bundled plugin that registers `api.on("agent_end", ...)` without `plugins.entries.<id>.hooks.allowConversationAccess=true` is blocked by `registerTypedHook`, and `src/plugins/loader.test.ts:5293` codifies that behavior.

## Solution Assessment

Unclear. `allowStateAccess` may be viable only with a sanitized payload; the safer direction is a dedicated turn-ended lifecycle event or documented `registerAgentEventSubscription({ streams: ["lifecycle"] })` contract rather than loosening the current raw `agent_end` gate.
