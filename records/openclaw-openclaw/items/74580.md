## Summary

Keep open. Current main still has no state-only `agent_end` permission or lightweight turn-ended SDK contract; the available lifecycle event subscription is relevant but does not fully settle the requested public Plugin SDK permission/API design, and the existing raw `agent_end` hook remains correctly gated because it carries conversation messages.

## What This Changes

Request a granular state-only plugin hook permission or sanitized turn-ended lifecycle contract so display/state plugins can know when an agent turn ends without full conversation access.

## Best Possible Solution

Add an additive, documented state-only lifecycle or turn-ended Plugin SDK contract that exposes only minimal run/session/phase/status metadata while keeping raw `agent_end.messages` behind `allowConversationAccess`.

## Reproduction Assessment

Yes. A non-bundled plugin that registers `api.on("agent_end", ...)` without `plugins.entries.<id>.hooks.allowConversationAccess=true` is still rejected by `registerTypedHook`, and the loader regression test locks in that behavior.

## Solution Assessment

Unclear. `allowStateAccess` is plausible, but the safer maintainable shape is likely a sanitized turn-ended event or explicitly documented lifecycle subscription rather than weakening access to the existing `agent_end` payload.
