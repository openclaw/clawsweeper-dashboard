## Summary

Keep open. Current main still gates non-bundled `agent_end` behind full conversation access, and that is still warranted because the payload includes raw `messages`; the remaining request is a public Plugin SDK permission/security design decision rather than stale cleanup.

## What This Changes

Add a state-only hook permission or equivalent lightweight lifecycle signal so external plugins can observe agent turn end without granting full conversation access.

## Best Possible Solution

Keep raw-content hooks behind `allowConversationAccess` and add or document an additive state-only turn-ended/lifecycle SDK contract that exposes only `runId`, `sessionKey`, phase, timing, and terminal status, with tests proving no prompts, responses, history, tool payloads, or `messages` array reach state-only consumers.

## Reproduction Assessment

Yes. A non-bundled plugin that registers `api.on("agent_end", ...)` without `plugins.entries.<id>.hooks.allowConversationAccess=true` is blocked by `registerTypedHook`, and `src/plugins/loader.test.ts:5293` codifies that behavior.

## Solution Assessment

Unclear. `allowStateAccess` is plausible, but reusing the current `agent_end` payload would be unsafe because it includes `messages`; a sanitized event or explicitly documented lifecycle subscription contract is the safer direction.

## Security Review

Status: needs_attention

Summary: This issue changes a plugin permission boundary, so any fix must preserve the raw conversation gate while exposing only sanitized lifecycle state.

Concerns:

- **[medium] Keep state access away from agent_end messages:** `src/plugins/hook-types.ts:272`
  - body: `agent_end` currently includes `messages`, and the runner passes the conversation snapshot into that hook. A state-only permission must not expose this payload to non-bundled plugins without full conversation access.
  - confidence: 0.97
- **[medium] Constrain any event-subscription workaround to lifecycle data:** `src/plugins/host-hook-runtime.ts:233`
  - body: `registerAgentEventSubscription` can dispatch cloned agent events by stream. A state-only contract should either require the `lifecycle` stream or add a dedicated turn-ended event so assistant/tool/content streams do not become a conversation-access bypass.
  - confidence: 0.86
