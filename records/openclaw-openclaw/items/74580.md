## Summary

Keep open: current main and v2026.6.10 still gate `agent_end` behind `allowConversationAccess`, and no `allowStateAccess` surface exists. The open linked PR is a candidate fix, not a reason to close this issue before merge.

## What This Changes

Request to add a state-only plugin hook permission or equivalent lightweight lifecycle contract for `agent_end` so non-bundled plugins can observe turn completion without full conversation access.

## Best Possible Solution

Keep this issue open as the canonical permission-design request, then review or revise the linked PR so maintainers explicitly choose the state-only `agent_end` contract versus the existing lifecycle subscription path.

## Reproduction Assessment

Yes. Source inspection on current main shows `agent_end` remains a conversation hook, non-bundled registration is blocked without `allowConversationAccess`, and there is no `allowStateAccess` schema/config surface.

## Solution Assessment

Unclear as a final product direction. A state-only `agent_end` permission is plausible, but maintainers still need to choose it over documenting or tightening the existing sanitized lifecycle event subscription contract.
