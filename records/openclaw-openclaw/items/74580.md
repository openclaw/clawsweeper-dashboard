## Summary

Keep open. Current main still treats `agent_end` as a raw conversation hook with `messages`, has no `allowStateAccess` or state-only turn-ended SDK contract, and the linked OpenClaw Interaction Bridge discussion shows a real plugin use case that should not require full conversation access.

## What This Changes

Request a state-only permission or equivalent SDK event so non-bundled plugins can observe agent turn completion without granting raw conversation-hook access.

## Best Possible Solution

Define an additive Plugin SDK turn-ended/state lifecycle contract that emits only sanitized metadata such as run/session identity, success, error, and duration while keeping raw `agent_end.messages` behind `allowConversationAccess`.

## Reproduction Assessment

Yes. Source inspection on current main shows a non-bundled plugin registering `api.on("agent_end", ...)` is classified as a conversation hook and blocked unless `allowConversationAccess=true`, and loader coverage asserts that behavior.

## Solution Assessment

Unclear. `allowStateAccess` is plausible, but maintainers should choose whether to add that policy knob, split/sanitize `agent_end`, or expose a documented turn-ended lifecycle event without weakening the raw conversation gate.
