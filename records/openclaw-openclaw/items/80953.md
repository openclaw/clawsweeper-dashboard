## Summary

Keep open because current main still has the source-level fallback-loss path for string-form default model config, and an open unmerged closing PR already owns the implementation work.

## What This Changes

The issue reports that string-form `agents.defaults.model` can silently drop configured model fallbacks, especially when config writers replace object-form model config with a string.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80961 or a replacement that preserves fallback-bearing object config at write boundaries and adds sanitized diagnostics for string-form fallback loss.

## Reproduction Assessment

Yes. Current main returns `[]` for string-form model config, and fallback candidate construction consumes that helper directly; this is source-reproducible without needing a live provider overload.

## Solution Assessment

Yes for the direction. Preserving object-form fallback config and adding targeted sanitized diagnostics is the narrow maintainable fix; the linked PR or a replacement should resolve its review blockers before landing.
