## What This Changes

The issue asks OpenClaw to stop silently losing model fallbacks when `agents.defaults.model` is represented or rewritten as a string.

## Best Possible Solution

Review and land the linked PR or a replacement that preserves object-form fallback config at write boundaries and adds sanitized diagnostics for string-form default model configs.

## Reproduction Assessment

Yes, source-level: current main returns `[]` for string model input and the runtime fallback path consumes that helper directly. I did not run a live overload scenario in this read-only review.

## Solution Assessment

Yes for the direction: preserve fallback-bearing object configs and add targeted diagnostics without changing the documented string-primary/object-fallback contract. The active linked PR should be reviewed or repaired before this issue closes.
