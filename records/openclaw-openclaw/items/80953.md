Keep open: current main and the latest release still have the source-level path where string-shaped default model config yields no fallback candidates, and the linked fixing PR is still open with closing syntax. ## What This Changes

The issue asks OpenClaw to stop silently losing model fallbacks when `agents.defaults.model` is represented or rewritten as a string.

## Best Possible Solution

Land or replace the linked fix with a sanitized diagnostic plus focused coverage, and preserve fallback-bearing object configs when only the primary model changes.

## Reproduction Assessment

Yes, at source level: a string model config returns `[]` from `resolveAgentModelFallbackValues()`, and default fallback candidate construction uses that result directly. I did not run a live overload scenario because this review is read-only.

## Solution Assessment

Yes, keeping this issue open behind the linked fix PR is the right path. The maintainable fix is to preserve or guard fallback-clobbering writes and add a clear diagnostic, not to make string configs inherit unknown previous fallbacks at read time.
