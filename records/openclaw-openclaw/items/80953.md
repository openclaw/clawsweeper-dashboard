## Summary

Keep open because current main still has the source-level string-model fallback-loss path, and an open unmerged closing PR already owns the proposed fix.

## What This Changes

The issue reports that string-form `agents.defaults.model` values cause fallback model lists to resolve to an empty array, silently disabling configured failover behavior.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80961 or a replacement that preserves object-form fallback config and adds sanitized diagnostics for string-form fallback loss.

## Reproduction Assessment

Yes, at source level: current main returns `[]` for string model input and the runtime/gateway fallback paths consume that helper directly. I did not run a live overload scenario in this read-only review.

## Solution Assessment

Yes for the direction: keep the documented string-primary/object-fallback semantics, preserve fallback-bearing object configs at write boundaries, and add targeted sanitized diagnostics. The active linked PR should be reviewed or replaced rather than opening a separate issue repair lane.
