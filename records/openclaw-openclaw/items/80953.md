## What This Changes

The issue reports that string-shaped `agents.defaults.model` config disables fallback candidates because `resolveAgentModelFallbackValues()` returns an empty array for non-object model config.

## Best Possible Solution

Land a narrow fix through the linked PR or a replacement that preserves fallback-bearing object config on primary-only changes and adds a clear doctor/log guard for string-shaped defaults.

## Reproduction Assessment

Yes. Source inspection shows string model input returns `[]`, and current runtime paths consume that helper for default fallback selection; no live overload run was executed in this read-only review.

## Solution Assessment

Yes for the narrow direction: preserve or warn at config write/diagnostic boundaries while keeping the documented string-primary/object-fallback semantics unchanged.
