## Summary

Keep open: current main still has a source-level fallback-loss path for string-shaped model config, while existing adjacent fixes only cover some write paths and per-agent doctor warnings.

## What This Changes

The issue asks OpenClaw to stop silently losing configured model fallbacks when a model config is represented or rewritten as a bare string instead of an object with `primary` and `fallbacks`.

## Best Possible Solution

Preserve fallback-bearing object model configs when changing only the primary model, and add a targeted guard or warning for writes that would replace `{ primary, fallbacks }` with a string or partial object without `fallbacks`.

## Reproduction Assessment

Yes, at source level: a string model config returns `[]` from `resolveAgentModelFallbackValues()`, and the fallback runner uses that result directly. I did not run a live overload scenario because this review is read-only.

## Solution Assessment

Yes, the maintainable fix is to preserve or reject clobbering writes at model-config write boundaries, not to make string configs magically inherit unknown previous fallbacks at read time.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/80953. Current main returns [] for string model configs in `resolveAgentModelFallbackValues()`, and fallback candidate resolution consumes that directly. Keep the existing contract that explicit `fallbacks: []` disables fallbacks, and avoid a broad config migration. Implement a narrow guard/preservation path so writes that only change a model primary do not turn an existing fallback-bearing `agents.defaults.model` or `agents.list[].model` object into a bare string or `{ primary }` without `fallbacks`; cover the agent-facing gateway `config.patch`/`config.apply` path and any model-switch/default-model helper that can perform this rewrite. Prefer preserving existing `fallbacks` when changing `primary`; otherwise reject or warn with a targeted message telling callers to patch `.primary` or include `fallbacks`. Add focused regression tests and a changelog Fixes entry. Do not change `fallbacksOverride: []` semantics or unrelated model-selection policy.
