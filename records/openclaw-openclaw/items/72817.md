Keep open for normal maintainer review. Current main does not contain this PR's source-artifact fallback, and the proposed diff looks narrow, covered, mergeable, and consistent with the plugin loader boundary; I found no blocking functional or security finding.

## What This Changes

This PR hardens bundled plugin public-surface loading by retrying selected source-artifact require failures through jiti, refactors bundled entry load profiling, and adds focused loader/runtime-deps test coverage plus minor test cleanups.

## Best Possible Solution

Land this PR after maintainer review if the branch remains current and CI stays green; it is a focused plugin-loader hardening with regression coverage and does not need a replacement branch.

## Reproduction Assessment

Yes. The PR includes a focused fixture where a source public artifact resolved as TypeScript hits a module-not-found error for an emitted-style `.js` import and then succeeds through jiti, plus a syntax-error fixture proving unrelated failures still throw.

## Solution Assessment

Yes. The patch preserves existing boundary validation and fast paths, adds a narrow fallback only for the source-artifact `.js` specifier mismatch, and avoids expanding the public plugin API.
