## Summary

Keep open. Current main still takes the source public-artifact `require()` fast path without a jiti retry, so the PR is not implemented on main; the inspected diff is a narrow loader hardening with focused regression coverage and no blocking review findings.

## What This Changes

The PR adds a guarded source-artifact `require()` to jiti fallback for bundled public surfaces, adds public-surface loader regression tests and test mock hygiene, and extracts bundled channel-entry load profiling into a helper.

## Best Possible Solution

Land the narrow loader hardening after maintainer review and current-base gates, keeping the existing boundary validation, public-surface cache behavior, and plugin API surface unchanged.

## Reproduction Assessment

Yes. The high-confidence reproduction path is a bundled source public artifact whose `.ts` file contains emitted-style `.js` sibling imports: current main returns from `sourceArtifactRequire` directly, while the PR's focused test simulates that module-not-found failure and expects a jiti retry.

## Solution Assessment

Yes. The patch fixes the failure at the loader seam, preserves the fast path for successful source requires, limits the fallback by error code and `.js`-specifier message, and tests that syntax errors are not masked.
