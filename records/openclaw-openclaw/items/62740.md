## Best Possible Solution

## Reproduction Assessment

Yes, by source inspection. A fetch returning a non-OK HTML/plain-text response reaches `response.json()` on current main before any `response.ok` guard, so the reported SyntaxError path is clear.

## Solution Assessment

No. The status guard and `httpStatus` split are the right direction, but the PR should bound the body read, avoid remote body text in `Error.message`, add focused coverage, and include real after-fix proof before merge.
