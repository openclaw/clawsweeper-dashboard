## Summary

Keep open. Current main still has fixed built-in media-tool local roots and no `tools.mediaLocalRoots` or `tools.fs.localRoots` config surface, but the request is a new security-sensitive configuration feature rather than a broken existing contract.

## What This Changes

Request configurable extra local roots for the built-in PDF and image tools so admins can opt in user directories without copying files into the agent workspace.

## Best Possible Solution

Define an explicit opt-in media-root config contract with schema, docs, tests, and per-agent/global precedence, then route only those validated roots through `resolveMediaToolLocalRoots` while preserving current default denial behavior.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main confirms the requested config key is absent and built-in PDF/image local roots remain fixed to defaults plus workspace scope.

## Solution Assessment

Unclear until maintainers choose the policy surface. The proposed opt-in root list is plausible, but it should be designed alongside `tools.fs.workspaceOnly`, per-agent overrides, docs, and security tests rather than treated as a small bug fix.
