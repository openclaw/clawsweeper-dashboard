## Summary

Keep open for maintainer product/security review: current main intentionally renders uploaded/structured image blocks, but a bare remote image URL in chat remains text/link content and the documented Control UI CSP blocks remote image loads.

## What This Changes

Request automatic inline image rendering when a user sends a plain image URL in the chat window.

## Best Possible Solution

If accepted, add a deliberate Control UI preview path for plain image URLs that preserves the current remote-content threat model, likely by converting only well-scoped image URLs into managed same-origin image blocks with tests and docs.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new chat-rendering feature request. Source inspection shows current main keeps typed URLs as text/link content and only renders explicit image blocks, attachments, transcript media paths, or data-image markdown as images.

## Solution Assessment

No, the suggested regex-to-img conversion is not the best standalone fix because current docs and CSP intentionally prevent remote image loads. A maintainable solution needs a product/security decision and should use an existing managed/same-origin image path if remote URL previews are allowed.
