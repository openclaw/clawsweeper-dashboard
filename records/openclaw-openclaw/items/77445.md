## Summary

Keep open: the PR fixes the earlier usage-accounting concern and the transcript reconciliation direction is right, but the updated duplicate check can still drop short final replies when earlier same-attempt assistant text happens to contain the same word.

## What This Changes

The PR adds an embedded-runner helper and focused tests to append missing delivered `assistantTexts` to the durable transcript after successful non-compacted attempts, plus docs and changelog updates.

## Best Possible Solution

Keep the reconciliation helper, but make coverage detection precise enough to skip true persisted duplicates while still appending short independent final answers, with a regression test for that same-attempt ordering.

## Reproduction Assessment

Yes. Current main source shows delivered `assistantTexts` can be used for outbound payloads while the successful attempt path only snapshots persisted session messages; the PR edge case is source-reproducible when an earlier current-attempt assistant text contains a short final candidate such as `Done.`.

## Solution Assessment

No, not as written. Appending a durable assistant-text mirror after successful non-compacted attempts is the narrow solution, but duplicate detection should not treat arbitrary short substrings in earlier assistant messages as proof the final reply is already persisted.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.78

Full review comments:

- **[P2] Preserve short final replies:** `src/agents/pi-embedded-runner/run/assistant-text-persistence.ts:21`
  - body: The new coverage check treats any candidate as already persisted when it appears anywhere inside any assistant text from the current attempt. In a tool turn that already persisted a pre-tool/commentary assistant message such as `I'll get this done.` and later only delivers `assistantTexts: ['Done.']`, this line drops the final reply as covered, so the next turn still lacks the visible answer. Keep substring coverage for true full-message duplicates, but add a length/ratio/order guard or a regression that proves short independent final replies are appended.
  - confidence: 0.78
