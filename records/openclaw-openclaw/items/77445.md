## Summary

Keep this PR open: it addresses a real transcript-persistence gap, but the current head still drops independent short final replies when they appear as incidental substrings of an earlier persisted assistant message.

## What This Changes

The PR adds an embedded-runner reconciliation helper, focused tests, docs, and a changelog entry to persist delivered `assistantTexts` into the durable transcript after successful non-compacted attempts.

## Best Possible Solution

Keep the reconciliation helper, but make coverage detection skip true persisted duplicates while still appending independent short final replies, with regression coverage for the incidental-substring case.

## Reproduction Assessment

Yes, source-reproducible. A current-attempt persisted assistant text such as `I'll get this done.` plus delivered `assistantTexts: ['Done.']` hits the PR head's substring check and prevents the visible final reply from being appended.

## Solution Assessment

No, not as written. Appending a durable assistant-text mirror after successful non-compacted attempts is the right narrow boundary, but the duplicate check needs to distinguish full-message or streamed-chunk coverage from unrelated short final answers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve independent short final replies:** `src/agents/pi-embedded-runner/run/assistant-text-persistence.ts:21`
  - body: The helper treats a delivered candidate as already persisted whenever it appears anywhere inside a current-attempt assistant text. If the tool-call assistant stored a pre-tool note like `I'll get this done.` and the delivered final answer is `Done.`, this returns no unpersisted text, so the durable transcript still misses the answer the user saw. Limit substring coverage to true full-message or streamed-chunk duplicates, and add a regression for this independent short-final case.
  - confidence: 0.86
