## Summary

Keep open. The bug is source-reproducible and the PR is aimed at the right embedded-runner boundary, but the latest head still persists synthetic mirrors with the wrong model API metadata and lacks external real behavior proof before merge.

## What This Changes

The PR appends delivered `assistantTexts` to the durable embedded-runner transcript after successful non-compacted attempts, with focused tests plus transcript-hygiene docs and changelog updates.

## Best Possible Solution

Land a rebased version that keeps the narrow zero-usage transcript mirror but records an accurate model API or explicit mirror discriminator, adds non-OpenAI/Codex coverage, and includes redacted real runtime proof.

## Reproduction Assessment

Yes, source-reproducible. Current main can deliver final text through `assistantTexts` while the successful attempt path only snapshots persisted session messages and has no append step for missing visible assistant text.

## Solution Assessment

No, not yet. The reconciliation boundary is a narrow fix, but persisting every mirror as `openai-responses` is not the best durable contract for non-OpenAI or OpenAI-Codex turns.
