## Summary

Keep open: current main still lacks the durable-text reconciliation path, and the PR is a narrow valid fix with no blocking review findings from this read-only pass.

## What This Changes

The PR adds embedded-runner transcript reconciliation so delivered assistant text is persisted after successful non-compacted attempts, with focused tests, transcript-hygiene docs, and a changelog entry.

## Best Possible Solution

Land this embedded-runner reconciliation after exact-head CI and maintainer review, while leaving broader memory hygiene and outbound artifact delivery policy to #77447 and #77448.

## Reproduction Assessment

Yes, source-reproducible. Current main can collect delivered `assistantTexts` and use them for outbound payloads, but the successful-attempt path only snapshots session messages and has no source path that writes missing visible assistant text back to durable transcript storage.

## Solution Assessment

Yes. Appending a zero-usage assistant-text mirror only after successful non-compacted attempts is a narrow maintainable fix, and the PR avoids provider replay, outbound delivery, tool-result repair, and compaction behavior changes.
