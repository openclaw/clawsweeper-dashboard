## Summary

Keep this open. Current main still has SSRF/private-network controls, trusted proxy routing, and provider-specific domain filters, but it does not expose or enforce the requested global/per-agent allowedDomains plus denyDomains egress firewall across OpenClaw-managed web tools; the prior urlAllowlist PR was reverted as an accidental merge, and the related open PR is only a partial draft for per-agent web_fetch SSRF overrides.

## What This Changes

Request a configurable allowedDomains/denyDomains network egress policy for web_fetch and web_search, with global and per-agent config and enforcement through the existing SSRF guard path.

## Best Possible Solution

Add a documented first-class web-tool egress policy with typed global and per-agent config, strict schema/help/docs, deny-before-allow enforcement, explicit web_fetch and web_search behavior, cache/redirect/audit coverage, and focused regression tests, while coordinating with the broader Security Profile umbrella.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a missing security/config feature, not broken existing behavior. Source inspection on current main shows the requested config shape and enforcement hook are still absent.

## Solution Assessment

Unclear for a one-shot implementation. Reusing SsrFPolicy allowlist primitives is plausible and the reverted urlAllowlist PR is useful prior art, but the maintainable fix needs owner review across deny semantics, provider coverage, redirects, cache isolation, and audit behavior.
