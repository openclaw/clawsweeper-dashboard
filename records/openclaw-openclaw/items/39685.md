## Summary

Keep open: current main and v2026.6.10 still lack the requested generic `allowedDomains`/`denyDomains` egress firewall for web tools. The closest current pieces are a lower-level SSRF hostname allowlist primitive, narrow `web_fetch` fake-IP opt-ins, Codex-native-search-only `allowedDomains`, and open partial PRs that still need maintainer product/security direction.

## What This Changes

Request to add configurable outbound domain allow/deny controls for OpenClaw web tools, especially `web_fetch` and `web_search`.

## Best Possible Solution

Keep this as the canonical web-tool egress-policy tracker until maintainers choose one public contract and implement it with schema, docs, audit, cache, redirect, provider-fallback, managed-search, and native-search coverage or explicitly narrow the scope.

## Reproduction Assessment

Yes for the missing capability: source inspection shows current main rejects the requested `web_fetch` domain policy keys and only threads fake-IP SSRF opt-ins. This is a feature/security-policy request rather than a broken existing behavior repro.

## Solution Assessment

Unclear as proposed: reusing `SsrFPolicy.hostnameAllowlist` is plausible for direct `web_fetch`, but the best solution needs a maintainer-approved boundary across fetch, search, redirects, cache isolation, provider fallback, native search, and plugin/proxy ownership.
