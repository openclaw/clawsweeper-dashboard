## Summary

Keep open. Current main still lacks the requested global/per-agent `allowedDomains` plus `denyDomains` egress policy for OpenClaw-managed web tools; https://github.com/openclaw/openclaw/issues/53268 was closed into this tracker, https://github.com/openclaw/openclaw/pull/67421 is only an open draft for adjacent per-agent `web_fetch` SSRF overrides, and https://github.com/openclaw/openclaw/issues/8719 is a broader security-profile umbrella rather than a replacement for this narrower network-policy request.

## What This Changes

The issue requests configurable global and per-agent domain allow/deny controls for OpenClaw-managed `web_fetch` and `web_search` egress.

## Best Possible Solution

Define and implement a first-class, documented web-tool egress policy with typed global and per-agent config, strict schema/help/docs, deny-before-allow matching, explicit `web_fetch` and `web_search` coverage, redirect/cache/provider-fallback/audit tests, and coordination with the broader Security Profile work.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a missing security/config feature, not broken existing documented behavior. Source inspection gives high confidence that the requested config shape and enforcement hook are absent on current main.

## Solution Assessment

Unclear for a direct one-shot implementation. Reusing existing `SsrFPolicy` hostname primitives is plausible, but the maintainable fix needs owner review for deny semantics, redirects, cache boundaries, provider coverage, native search behavior, and audit output.
