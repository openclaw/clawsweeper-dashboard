## Summary

Keep this open. Current main still has SSRF guard primitives, process-level proxy routing, and a narrow Codex-native search domain filter, but it does not expose or enforce the requested global/per-agent allow plus deny domain egress firewall across OpenClaw-managed web_fetch and web_search.

## What This Changes

The issue requests a global and per-agent web-tool egress firewall with allow/deny domain policy for OpenClaw-managed web fetch/search.

## Best Possible Solution

Keep this open until OpenClaw has a first-class web-tool network policy: typed config/schema/help/docs, global and per-agent resolution, deny-before-allow domain matching, web_fetch enforcement including redirects and cache keys, explicit behavior for managed versus provider-native web_search, and security audit coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: the proposed config shape is absent from strict runtime schemas, and the current web_fetch/web_search code paths do not receive or enforce a resolved allow/deny domain policy.

## Solution Assessment

Unclear. The requested direction is a reasonable core security feature, but the current implementation surfaces are only partial; #67421 may be a useful building block for web_fetch, not a complete solution for this issue.
