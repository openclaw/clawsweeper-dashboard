## Summary

Keep open. Current main still lacks a shared `allowedDomains`/`denyDomains` egress firewall for OpenClaw-managed web tools; existing support is limited to narrow `web_fetch` SSRF opt-ins, browser-only SSRF settings, operator-managed proxy guidance, and a Codex-native search-only `allowedDomains` filter.

## What This Changes

Add global and per-agent network access control for web tools using `allowedDomains` and `denyDomains`.

## Best Possible Solution

Define a documented first-class web-tool egress policy with typed global and per-agent config, deny-before-allow matching, explicit `web_fetch` plus managed/native `web_search` coverage, redirect/cache/audit tests, and alignment with the broader security-profile work.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that current main lacks a shared `allowedDomains`/`denyDomains` policy in config, schema, `web_fetch` runtime, and managed `web_search` enforcement.

## Solution Assessment

Unclear as a concrete implementation path. Reusing `SsrFPolicy` hostname primitives is plausible, but maintainers need to settle public config shape, cross-tool scope, precedence, redirect/cache behavior, provider boundaries, and audit semantics first.
