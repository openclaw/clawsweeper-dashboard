## Summary

Keep open: current main and v2026.6.6 still do not provide a generic web-tool egress firewall; the existing pieces are narrow SSRF primitives, browser-only allowlist exceptions, and Codex-native search filtering rather than the requested `allowedDomains`/`denyDomains` policy.

## What This Changes

The issue requests configurable allowed and denied outbound domains for `web_fetch` and `web_search` as an OpenClaw web-tool egress firewall.

## Best Possible Solution

Keep this issue open as the canonical web-tool egress-policy tracker until maintainers define one public contract and then implement schema/help/docs/audit, cache, redirect, fallback, managed-search, and native-search coverage around it.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a requested security/config capability. Source and release inspection confirm the requested generic domain allow/deny surface is absent on current main and v2026.6.6.

## Solution Assessment

Unclear as proposed. Reusing `SsrFPolicy.hostnameAllowlist` is plausible for direct `web_fetch`, but the best solution needs a maintainer-approved contract across fetch, search, redirects, cache isolation, provider fallback, and plugin boundaries.
