## Summary

Keep this open. Current main still has SSRF/private-network protections, an external proxy path, and a Codex-native search domain filter, but it does not expose or enforce the requested global/per-agent allow plus deny domain egress firewall across OpenClaw-managed web tools.

## What This Changes

The issue requests global and per-agent network egress controls for OpenClaw web tools using allowed and denied domains.

## Best Possible Solution

Add a documented first-class web-tool network policy with typed config/schema/help, global and per-agent resolution, deny-before-allow enforcement, explicit web_fetch and web_search behavior, cache/redirect/audit coverage, and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a missing security/config feature, not broken existing behavior. Source inspection on current main shows the requested config shape and enforcement hook are still absent.

## Solution Assessment

Unclear. The proposed direction is plausible because SsrFPolicy already has hostname allowlist primitives, but the best maintainable solution needs maintainer design across deny semantics, precedence, managed search providers, redirects, caching, and audit behavior.
