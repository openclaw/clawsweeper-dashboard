## Summary

Keep open: current main and v2026.6.6 still do not provide a native web-tool domain allow/deny egress firewall, and the related work is either a closed duplicate, a browser-specific tracker, a broader security-profile umbrella, or a draft partial SSRF override PR that does not implement this request.

## What This Changes

The issue requests configurable network egress controls for OpenClaw web tools through domain allow and deny policies.

## Best Possible Solution

Define and implement a native web-tool egress policy with explicit schema, deny-before-allow semantics, redirect/cache/provider coverage, docs, audit behavior, and security review before exposing it as a trusted firewall.

Not applicable as a bug reproduction: this is a new security/config capability request. Source and release inspection do confirm the requested web_fetch domain allow/deny policy is absent today.

## Solution Assessment

No, the issue is not ready as a direct implementation plan.
