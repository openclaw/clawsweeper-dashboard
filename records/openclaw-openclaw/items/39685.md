## Summary

Keep this open. Current main still has SSRF/private-network primitives and a Codex-native search allow-domain filter, but it does not expose or enforce the requested global/per-agent allow plus deny domain egress firewall across OpenClaw-managed web tools.

## What This Changes

Add configurable global and per-agent network access control for web tools using `allowedDomains` and `denyDomains`.

## Best Possible Solution

OpenClaw should add a documented first-class web-tool network policy with typed config/schema/help, global and per-agent resolution, web_fetch enforcement, explicit web_search behavior, cache and redirect handling, audit coverage, and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a missing security feature, not a broken existing contract. Source inspection gives high confidence that the requested config and enforcement path are absent on current main.

## Solution Assessment

Unclear. The requested direction is plausible, but the best maintainable solution needs maintainer design across config precedence, provider-native search behavior, redirects, cache keys, and audit semantics; #67421 may be a useful building block but is not complete.
