## Summary

Keep open. Current main still lacks the requested shared global/per-agent `allowedDomains` plus `denyDomains` egress policy for OpenClaw-managed `web_fetch` and `web_search`; the existing SSRF and Codex-native search knobs are narrower and the related open PR is only a draft partial implementation.

## What This Changes

The issue requests a global and per-agent domain allow/deny egress firewall for OpenClaw-managed `web_fetch` and `web_search`.

## Best Possible Solution

Define a documented first-class web-tool egress policy with typed global and per-agent config, deny-before-allow matching, explicit managed/native `web_search` and `web_fetch` coverage, cache/redirect/provider-fallback/audit tests, and coordination with the broader security-profile work.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a missing security/config feature, not broken existing documented behavior. Source inspection gives high confidence that the shared allow/deny-domain config and enforcement hook are absent on current main.

## Solution Assessment

No for a one-shot automated fix. Reusing existing `SsrFPolicy` hostname primitives is plausible, but the best solution needs maintainer/security decisions for public config shape, policy precedence, provider coverage, redirects, cache boundaries, and audit output.
