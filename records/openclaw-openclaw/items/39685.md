## Summary

Keep open. Current main still lacks a shared allowedDomains/denyDomains egress firewall for OpenClaw-managed web tools; the existing code only has narrow web_fetch SSRF opt-ins and provider-specific Codex native search filters, while the related security-profile issue and draft SSRF PR do not implement the requested policy.

## What This Changes

Request to add global and per-agent network egress policy, with allowedDomains and denyDomains enforcement for web_fetch and web_search using the existing SSRF infrastructure.

## Best Possible Solution

Define a documented first-class web-tool egress policy with typed global and per-agent config, deny-before-allow matching, explicit web_fetch plus managed/native web_search coverage, redirect/cache/audit tests, and alignment with the broader security-profile work.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that the requested shared allow/deny-domain egress policy is absent from current main.

## Solution Assessment

Unclear as a concrete implementation path. Reusing SsrFPolicy hostname primitives is plausible, but maintainers need to settle public config shape, scope across web_fetch and web_search, precedence, redirects, cache boundaries, and audit behavior first.
