## Summary

Keep open. Current main still does not provide a shared allowedDomains/denyDomains egress firewall for OpenClaw-managed web tools; the existing support is limited to narrow web_fetch SSRF opt-ins, browser-only SSRF settings, operator-managed proxy guidance, and a Codex-native search-only allowedDomains filter.

## What This Changes

The issue requests global and per-agent allowedDomains/denyDomains egress controls for web_fetch and web_search by reusing existing SSRF hostname policy primitives.

## Best Possible Solution

Define a documented first-class web-tool egress policy with typed global and per-agent config, deny-before-allow matching, explicit web_fetch plus managed/native web_search coverage, redirect/cache/audit tests, and alignment with the broader security-profile work.

## Reproduction Assessment

Not applicable as a bug reproduction. Yes for source-level confirmation: current main lacks a shared allowedDomains/denyDomains policy in config, schema, web_fetch runtime, and managed web_search enforcement.

## Solution Assessment

Unclear as a concrete implementation path. Reusing SsrFPolicy hostname primitives is plausible, but maintainers need to settle the public config shape, cross-tool scope, precedence, redirects, cache boundaries, provider behavior, and audit semantics first.
