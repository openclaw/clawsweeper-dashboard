## Summary

Keep open. Current main still lacks a shared allowedDomains/denyDomains egress firewall for OpenClaw-managed web tools; the existing code only exposes narrow web_fetch SSRF opt-ins and a Codex-native search-only allowedDomains filter, while the related open PRs are partial security-sensitive work rather than a complete implementation.

## What This Changes

The issue requests first-class allowedDomains/denyDomains network access control for OpenClaw web tools such as web_fetch and web_search.

## Best Possible Solution

Define a documented first-class web-tool egress policy with typed global and per-agent config, deny-before-allow matching, web_fetch plus managed/native web_search coverage, redirect/provider/cache/audit tests, and alignment with the broader security-profile work.

## Reproduction Assessment

Yes for source-level confirmation, not as a bug reproduction. Current main has no shared allowedDomains/denyDomains policy in tools config/schema, web_fetch runtime, or managed web_search enforcement.

## Solution Assessment

Unclear as a concrete implementation path. Reusing SsrFPolicy hostname primitives is plausible, but maintainers need to settle the public config shape, cross-tool scope, precedence, redirects, cache boundaries, provider behavior, and audit semantics first.
