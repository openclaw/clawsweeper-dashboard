## Summary

Keep open. Current main still lacks a shared allowedDomains/denyDomains egress firewall for OpenClaw-managed web tools; existing support is limited to narrow web_fetch SSRF opt-ins and a Codex-native search-only allowedDomains filter, while the related open PR is partial and security-sensitive rather than a complete implementation.

## What This Changes

The issue requests configurable allowedDomains/denyDomains network egress controls for OpenClaw web_fetch and web_search tools.

## Best Possible Solution

Define a documented first-class web-tool egress policy with typed global and per-agent config, deny-before-allow matching, explicit web_fetch plus managed/native web_search coverage, redirect/cache/audit tests, and alignment with the broader security-profile work.

## Reproduction Assessment

Not applicable as a bug reproduction. Yes for source-level confirmation: current main lacks a shared allowedDomains/denyDomains policy in config, schema, web_fetch runtime, and managed web_search enforcement.

## Solution Assessment

Unclear as a concrete implementation path. Reusing SsrFPolicy hostname primitives is plausible, but maintainers need to settle public config shape, cross-tool scope, precedence, redirects, cache boundaries, provider behavior, and audit semantics first.
