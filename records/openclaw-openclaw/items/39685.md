## Summary

Keep open: current main and v2026.6.8 still do not provide the requested generic `allowedDomains`/`denyDomains` egress firewall for OpenClaw web tools. The existing pieces are a lower-level SSRF allowlist primitive, narrow `web_fetch` fake-IP opt-ins, and a Codex-native-search-only `allowedDomains` setting.

## What This Changes

Request adds operator-configurable allowed and denied domain egress policy for `web_fetch` and `web_search`.

## Best Possible Solution

Keep this as the canonical web-tool egress-policy tracker until maintainers define one public contract and implement schema, docs, audit, cache, redirect, provider-fallback, managed-search, and native-search coverage around it.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a requested security/config capability. Source and release inspection show current main and v2026.6.8 still lack a generic web-tool domain allow/deny surface.

## Solution Assessment

Unclear as proposed. Reusing `SsrFPolicy.hostnameAllowlist` is plausible for direct `web_fetch`, but the best solution needs a maintainer-approved contract across fetch, search, redirects, cache isolation, provider fallback, and plugin boundaries.
