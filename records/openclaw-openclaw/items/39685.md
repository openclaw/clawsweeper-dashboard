## Summary

Keep this open. Current main still lacks the requested global/per-agent allowedDomains plus denyDomains egress firewall for OpenClaw-managed web tools; existing code only provides private-network SSRF controls, narrow web_fetch SSRF opt-ins, and provider-specific search filters.

## What This Changes

Add configurable network egress policy with allowedDomains and denyDomains for OpenClaw web_fetch and web_search access.

## Best Possible Solution

Define a first-class, documented web-tool egress policy with typed global and per-agent config, deny-before-allow enforcement, explicit web_fetch and web_search coverage, cache/redirect/provider/audit behavior, and focused regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a missing security/config feature, not broken behavior under an existing contract. Source inspection on current main shows the requested config shape and enforcement hook are absent.

## Solution Assessment

Unclear as a one-shot implementation. Reusing SsrFPolicy hostname allowlist primitives is plausible, but the maintainable solution needs maintainer/security signoff on deny semantics, per-agent precedence, web_search coverage, redirects, cache isolation, and audit behavior.
