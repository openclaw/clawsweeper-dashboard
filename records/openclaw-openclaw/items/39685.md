## Summary

Keep open: current main and v2026.6.10 still lack a generic domain allow/deny egress policy for OpenClaw web tools. The current code exposes only narrow `web_fetch` fake-IP SSRF opt-ins plus Codex-native search `allowedDomains`, and the linked implementation PR is still a draft/conflicting `web_fetch`-only slice.

## What This Changes

The issue requests `allowedDomains`/`denyDomains` network access control for OpenClaw web tools by wiring public config into the existing SSRF policy layer.

## Best Possible Solution

Keep this as the canonical web-tool egress-policy tracker until maintainers choose one public contract and land schema, runtime, cache, fallback, docs, audit, and search/fetch coverage around that contract.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a requested security/config capability. Source and release inspection show current main and v2026.6.10 still lack a generic web-tool domain allow/deny surface.

## Solution Assessment

Unclear as proposed: reusing `SsrFPolicy.hostnameAllowlist` is plausible for direct `web_fetch`, but the best final solution needs maintainer-approved scope across fetch, search, redirects, cache isolation, provider fallback, docs, and audit behavior.
