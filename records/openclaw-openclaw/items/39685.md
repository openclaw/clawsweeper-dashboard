## Summary

Keep open. Current main still lacks the requested global/per-agent `allowedDomains` plus `denyDomains` egress policy for OpenClaw-managed web tools; existing SSRF flags, trusted proxy routing, and provider-specific filters are narrower, while https://github.com/openclaw/openclaw/pull/67421 is only an open draft for adjacent per-agent `web_fetch` SSRF overrides.

## What This Changes

The issue asks to add configurable `allowedDomains` and `denyDomains` network egress controls for `web_fetch` and `web_search`.

## Best Possible Solution

Define and implement a first-class, documented web-tool egress policy with typed global and per-agent config, strict schema/help/docs, deny-before-allow enforcement, explicit `web_fetch` and `web_search` coverage, redirect/cache/provider-fallback/audit tests, and coordination with the broader Security Profile work.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a missing security/config feature, not broken existing documented behavior. Source inspection gives high confidence that the requested config shape and enforcement hook are absent on current main.

## Solution Assessment

Unclear for a one-shot implementation. Reusing the existing `SsrFPolicy` hostname allowlist primitive is plausible, but the maintainable fix needs owner review for deny semantics, redirects, cache boundaries, provider coverage, native search behavior, and audit output.
