## Summary

Keep open. Current main still lacks a shared allowedDomains/denyDomains egress firewall for OpenClaw-managed web tools; existing support is limited to narrow web_fetch SSRF opt-ins and a provider-specific Codex native-search allowlist, while the related open PRs are partial or security-review work rather than a complete implementation.

## What This Changes

The issue requests configurable global and per-agent web-tool egress policy with allowedDomains and denyDomains for web_fetch and web_search.

## Best Possible Solution

Define a documented first-class web-tool egress policy with typed global and per-agent config, deny-before-allow matching, explicit web_fetch plus managed/native web_search coverage, redirect/cache/audit tests, and alignment with the broader security-profile work.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that current main lacks the requested shared allow/deny-domain egress policy for web_fetch and web_search.

## Solution Assessment

Unclear as a concrete implementation path. Reusing SsrFPolicy hostname primitives is plausible, but maintainers need to settle public config shape, scope across managed/native search and fetch, precedence, redirects, cache boundaries, and audit behavior first.
