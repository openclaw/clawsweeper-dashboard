## Summary

Keep open: current main and the latest release still do not expose the requested user-visible low-confidence memory warning. The related merged Codex PR restored Memory Recall prompt registration, but it did not add confidence metadata or display annotations to memory retrieval output.

## What This Changes

The issue asks OpenClaw to show a user-visible warning when retrieved memory data comes from low-confidence entries.

## Best Possible Solution

Define a small memory confidence/provenance display contract at the memory plugin/API boundary, then render authoritative low-confidence or stale metadata in memory_search/wiki_search snippets or prompt assembly with focused tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this requests a new user-visible confidence signal. Source inspection does confirm current memory_search, citations, and result types do not add that warning.

## Solution Assessment

Unclear until maintainers choose the UX and metadata contract. A narrow implementation should extend the memory-core/wiki retrieval surfaces rather than relying on the model to self-assess confidence after retrieval.
