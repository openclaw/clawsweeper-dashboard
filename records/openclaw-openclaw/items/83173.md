## Summary

Keep open: current main has memory citations and wiki claim-confidence metadata, but it does not provide the requested user-visible low-confidence warning when retrieved memory is presented. This is a product/UX feature request rather than a broken documented contract.

## What This Changes

Request to add user-visible low-confidence annotations when AI retrieves or cites memory entries with weak confidence, recency, provenance, or accuracy metadata.

## Best Possible Solution

Define a small confidence/provenance display contract for memory and wiki retrieval, then surface authoritative low-confidence or stale metadata in memory_search/wiki_search content with focused citation-mode tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this requests a new user-visible confidence signal. Source inspection shows current main can cite memory sources and expose some wiki claim confidence metadata, but does not implement the requested warning annotation.

## Solution Assessment

Unclear until maintainers choose the UX and metadata contract. The safer implementation path is to extend retrieval/tool output from authoritative memory/wiki metadata rather than relying on model self-assessment after retrieval.
