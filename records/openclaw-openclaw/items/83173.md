## Summary

Keep open: current main and the latest release have memory source citations and wiki claim-confidence metadata, but neither provides the requested user-visible low-confidence warning at retrieval time. This remains a memory UX/product feature request rather than a broken documented contract.

## What This Changes

The issue requests a user-visible warning or inline tag when memory retrieval uses low-confidence memory entries.

## Best Possible Solution

Define a small memory confidence/provenance display contract, then surface authoritative low-confidence or stale wiki/memory metadata in memory_search or wiki_search results with focused tests and docs, coordinated with https://github.com/openclaw/openclaw/issues/71728.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this asks for a new visibility behavior. Source inspection gives high confidence that current main lacks the requested annotation, while the production examples are not replayable from the issue alone.

## Solution Assessment

Unclear until maintainers choose the UX and metadata contract. The narrow maintainable path is to extend the memory-core/wiki retrieval surfaces rather than relying on the model to self-rate confidence after retrieval.
