## Summary

Keep open: current main has memory citations, retrieval scores, and partial wiki claim-confidence metadata, but it does not have a general user-visible low-confidence annotation for ordinary memory retrieval. The request is useful, but it needs a memory API/product decision before a small display-only patch is safe.

## What This Changes

Add a user-visible low-confidence annotation when AI output relies on low-confidence memory entries.

## Best Possible Solution

Define a typed confidence/freshness/provenance contract for memory retrieval results where sources actually provide it, then render a compact warning without reusing retrieval score as factual confidence.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-main state check: ordinary memory retrieval lacks a typed low-confidence signal to render.

## Solution Assessment

Unclear as a display-only patch. The direction is reasonable, but the maintainable fix is to define confidence semantics across memory-core, sessions, wiki, and dreaming before changing reply/tool output.
