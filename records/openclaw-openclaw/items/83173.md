## Summary

Keep open: current main and the latest shipped release expose memory citations and some wiki claim-confidence metadata, but they do not provide the requested user-visible low-confidence warning at retrieval time. This remains a product/UX feature request rather than a broken documented contract.

## What This Changes

The issue requests a user-visible warning when retrieved memory content is backed by low-confidence memory metadata.

## Best Possible Solution

Define a small memory provenance/confidence display contract, then surface authoritative low-confidence or stale wiki/memory metadata in shared retrieval results with focused tests for citation and wiki claim modes.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this requests new visibility behavior. Source and shipped-release inspection show current memory retrieval has citations and wiki confidence details, but no low-confidence warning annotation.

## Solution Assessment

Unclear until maintainers choose the UX and metadata contract. The narrow maintainable path is to extend the memory-core/wiki retrieval surfaces, not rely on model self-assessment after retrieval.
