## Summary

Keep open: current main has memory citations and wiki claim-confidence metadata, but it does not provide the requested user-visible low-confidence warning at the point of retrieval. This is a product/UX feature request, not a broken documented contract.

## What This Changes

The issue requests an inline user-visible confidence warning when an answer uses memory entries whose metadata indicates low confidence, uncertain freshness, or weak source reliability.

## Best Possible Solution

Define a small confidence/provenance display contract for memory retrieval, then surface authoritative low-confidence or stale wiki/memory metadata in memory_search/wiki_search snippets or prompt assembly with focused tests for citation modes.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this is a requested new visibility behavior. Source inspection shows current main can cite memory sources and expose some wiki claim confidence metadata, but it does not implement the requested warning annotation.

## Solution Assessment

Unclear until maintainers choose the UX and metadata contract. The narrow maintainable path would be to extend the memory-core/wiki retrieval surfaces rather than relying on the model to self-assess confidence after retrieval.
