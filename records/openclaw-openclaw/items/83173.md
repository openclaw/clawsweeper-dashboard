## Summary

Keep open: current main and the latest release still do not expose a user-visible warning when low-confidence memory/wiki facts are retrieved, and the remaining work needs a memory plugin/API and rendering product decision rather than conservative cleanup.

## What This Changes

The issue requests an inline user-visible confidence annotation when an AI answer relies on retrieved memory entries marked with low confidence, uncertain recency, or uncertain accuracy.

## Best Possible Solution

Define a memory-plugin confidence metadata contract and render a concise user-visible warning in the memory/wiki result path when low-confidence facts are used, with coverage across memory-core, memory-wiki, and Codex/context-engine prompt consumers.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this is a feature request for a new visible confidence signal. Source inspection does show current main lacks the requested annotation, but no existing documented behavior is failing.

## Solution Assessment

Unclear until maintainers choose the product/API shape. The requested inline tag is plausible, but the maintainable fix should decide the confidence threshold, metadata contract, and render location across memory-core, memory-wiki, and harness consumers.
