## Summary

Keep open: current main can cite memory sources and expose wiki claim-confidence metadata, but it still does not render a user-visible low-confidence warning when retrieved memory is used. This is a feature/product decision, not a broken documented contract.

## What This Changes

Request to show a visible low-confidence or verify warning when an agent uses retrieved memory entries with low confidence, stale, or uncertain metadata.

## Best Possible Solution

Define a small memory confidence/provenance display contract, then surface authoritative low-confidence or stale wiki/memory metadata in shared retrieval output with focused tests for citation modes.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this is a requested new user-visible annotation behavior. Source inspection gives a high-confidence current-state check that memory retrieval has citations and some wiki confidence metadata but no warning annotation.

## Solution Assessment

Unclear until maintainers choose the UX and metadata contract. The narrow maintainable path is to extend shared memory/wiki retrieval surfaces with authoritative metadata rather than asking the model to self-assess confidence after retrieval.
