## Summary

Keep open: this is a plausible scanner-layer fix for the linked doctor false-positive, but the patch is not merge-ready because it only gates the shared default warning path, can suppress valid top-level diagnostics, and has mock-only proof.

## What This Changes

The PR changes `scanEmptyAllowlistPolicyWarnings` to pass a top-level skip hook when every configured account has a populated `groupAllowFrom` or `allowFrom`.

## Best Possible Solution

Land one canonical scanner fix that suppresses both shared and plugin parent-scope group warnings only when every real active account scope has an effective group allowlist, while preserving disabled-only and active root/default diagnostics with focused regression tests and real terminal proof.

## Reproduction Assessment

Yes. Current main scans the top-level channel record before account records, and the linked issue provides the config shape that makes that parent-scope warning false; this review did not run a live doctor command because the checkout is read-only.

## Solution Assessment

No. The scanner is the right layer, but this implementation is too narrow for plugin warning hooks and too broad for disabled-only or active root/default account cases.
