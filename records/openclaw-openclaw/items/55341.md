## Summary

Keep open. Current main does not yet persist refreshed Twitch tokens and still has the OpenProse fall-through behavior, but this PR is not merge-ready because its writeback path can turn authored Twitch token env references into literal OAuth tokens in the config file.

## What This Changes

The PR adds Twitch OAuth refresh token writeback with tests, wires it into the Twitch refresh callback, and changes an OpenProse fast-loop example so terminal output branches are exclusive.

## Best Possible Solution

Keep the Twitch persistence feature direction, but preserve or skip authored env-reference token fields during refresh writeback, add regression coverage for base-level, accounts.default, and named-account ${...} accessToken configs, then rebase and revalidate the Twitch and OpenProse changes together.

## Reproduction Assessment

Yes. A config-authored Twitch access token such as channels.twitch.accessToken: "${OPENCLAW_TWITCH_ACCESS_TOKEN}" is resolved into snapshot.config, treated as tokenSource "config", changed to the refreshed token, and then no longer matches the authored env reference restoration checks.

## Solution Assessment

No. Persisting refreshed config-backed Twitch tokens is the right product direction, but this implementation writes through the resolved config shape and needs an authored-config/env-reference guard before it is the narrowest maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve env-referenced Twitch tokens:** `extensions/twitch/src/token-writeback.ts:105-106`
  - body: This clones the env-resolved config snapshot and later mutates `accessToken`. If the user authored `channels.twitch.accessToken: "${OPENCLAW_TWITCH_ACCESS_TOKEN}"`, the refreshed token differs from the resolved env value, so the config writer will keep the new literal OAuth token instead of restoring the env reference. Preserve or skip env-referenced token fields before writeback.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The PR introduces a concrete secret-handling regression for Twitch configs that use env references for access tokens.

Concerns:

- **[medium] Env token refs can become literal OAuth tokens:** `extensions/twitch/src/token-writeback.ts:106`
  - body: The writeback helper mutates a resolved config snapshot before calling `writeConfigFile`; once the refreshed token differs from the env-resolved value, the existing env-ref restoration paths keep the refreshed OAuth token as plaintext config.
