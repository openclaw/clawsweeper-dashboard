## Summary

Keep open: current main does not implement the central Twitch writeback or OpenProse control-flow changes, but this PR is not merge-ready because the writeback path can persist env-referenced OAuth tokens as literals and uses config APIs that current guards reject.

## What This Changes

The branch adds a Twitch token writeback helper and tests, wires it into the Twurple refresh callback, and wraps OpenProse terminal output branches in exclusive else blocks.

## Best Possible Solution

Revise the PR to preserve or skip env-authored Twitch token fields, use `openclaw/plugin-sdk/config-mutation` with `replaceConfigFile` or `mutateConfigFile` and explicit `afterWrite`, keep the OpenProse else-chain fix, add env-ref regression coverage, and include redacted real behavior proof.

## Reproduction Assessment

Yes for the PR-introduced blocker via source inspection: a config-authored `${...}` token resolves as a config token, then the PR mutates the resolved snapshot and the current env-ref restoration keeps changed paths literal. I did not run a live Twitch refresh.

## Solution Assessment

No. The feature direction is useful, but the current implementation is not the narrowest maintainable fix until it uses the focused config-mutation API and preserves env-authored token references.

The PR needs to be rebased and moved to that boundary before it can pass the current checks.
  - confidence: 0.88
