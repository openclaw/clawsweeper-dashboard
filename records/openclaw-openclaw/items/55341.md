## Review Findings

Full review comments:

- **[P1] Preserve env-authored Twitch token refs:** `extensions/twitch/src/token-writeback.ts:105-113`
  - body: This writeback starts from `snapshot.config`, which has already had `${OPENCLAW_TWITCH_ACCESS_TOKEN}` resolved. If that authored env ref is refreshed to a different token, `writeConfigFile` cannot restore the `${...}` reference and will serialize the OAuth token into config, breaking env-managed credentials and exposing the secret in `openclaw.json`.
  - confidence: 0.92
- **[P2] Use the config mutation SDK seam for writeback:** `extensions/twitch/src/token-writeback.ts:1-5`
  - body: Production extension code is importing the deprecated broad `openclaw/plugin-sdk/config-runtime` entry and calling `writeConfigFile` directly. The current config boundary guard blocks that pattern and expects extension writes to go through the narrow mutation seam so ownership and reload semantics stay centralized.
  - confidence: 0.88
