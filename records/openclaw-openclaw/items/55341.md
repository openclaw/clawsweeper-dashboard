## Summary

Keep open. Current main still lacks the Twitch refresh-token persistence and OpenProse fast-loop branch fix, but this PR needs revision before merge because it introduces a secret writeback regression and conflicts with the current config SDK boundary guard.

## What This Changes

The PR adds Twitch OAuth refresh-token writeback for config-backed accounts, wires it into Twitch refresh callbacks with tests, and changes an OpenProse fast-loop example so terminal branches are exclusive.

## Best Possible Solution

Revise the PR to preserve or skip authored env-reference token fields, use config-mutation/replaceConfigFile with an explicit afterWrite policy, skip absent optional token fields, add regression coverage, and then rebase and validate the Twitch/OpenProse changes together.

## Reproduction Assessment

Yes. A config such as channels.twitch.accessToken: "${OPENCLAW_TWITCH_ACCESS_TOKEN}" with a refreshed token value exercises the leak path: the PR mutates the resolved snapshot, and current env-ref restoration keeps changed paths literal.

## Solution Assessment

No. The feature direction is useful, but this implementation is not the narrowest maintainable fix because it bypasses the authored env-reference contract and uses a blocked broad config SDK barrel.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve env-referenced Twitch tokens:** `extensions/twitch/src/token-writeback.ts:105-106`
  - body: This writes refreshed tokens from a resolved config snapshot. If the user authored `channels.twitch.accessToken: "${OPENCLAW_TWITCH_ACCESS_TOKEN}"`, the refreshed value no longer matches the env-resolved original, so the config writer keeps the new literal OAuth token instead of restoring the env reference. Preserve or skip env-referenced token fields before writeback.
  - confidence: 0.92
- **[P2] Use the focused config mutation subpath:** `extensions/twitch/src/token-writeback.ts:1-4`
  - body: Current main's config-boundary guard rejects bundled plugin imports and mocks of `openclaw/plugin-sdk/config-runtime`. This new module imports read/write helpers from that broad barrel, so the patch will fail the deprecated-internal-config-api check after rebase; use `config-types` and `config-mutation` with `replaceConfigFile` instead.
  - confidence: 0.88
- **[P2] Skip expiry writes when the refresh omits it:** `extensions/twitch/src/token-writeback.ts:55-59`
  - body: When `token.expiresIn` is absent, this converts it to `null` and overwrites a previously stored expiry. Treat missing expiry like missing refreshToken and leave the stored value unchanged unless the refresh payload provides a number.
  - confidence: 0.84
- **[P3] Skip timestamp writes when the refresh omits it:** `extensions/twitch/src/token-writeback.ts:61-66`
  - body: When `token.obtainmentTimestamp` is absent, `Date.now()` makes the value differ on every callback, causing a config rewrite even when token data did not change. Only update this field when the refresh payload carries an explicit timestamp.
  - confidence: 0.82
