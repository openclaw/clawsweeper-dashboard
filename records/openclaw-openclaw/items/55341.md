## Summary

Keep open: the PR is useful, but it is not mergeable as-is because the token writeback can persist env-authored OAuth tokens as literal config values, uses a blocked broad config SDK import, and still lacks real behavior proof.

## What This Changes

The PR adds Twitch OAuth refresh token writeback and tests, wires Twitch refresh callbacks to persist config-backed tokens, and wraps one OpenProse fast-loop example’s terminal branches in else blocks.

## Best Possible Solution

Revise token persistence to preserve or skip env-authored token fields through the focused config-mutation surface, keep the OpenProse else-chain fix, add focused regression coverage, and include redacted runtime proof.

## Reproduction Assessment

Yes for the PR-introduced blocker by source inspection: the PR mutates an env-resolved config snapshot, and current env-ref restoration skips changed paths. I did not run a live Twitch refresh in this read-only review.

## Solution Assessment

No. The direction is useful, but the writeback must avoid leaking env-authored secrets and use the focused config mutation API before this is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Preserve authored env token references:** `extensions/twitch/src/token-writeback.ts:104-112`
  - body: This clones snapshot.config, which is already env-substituted, and then writes a refreshed access token back through the normal config writer. If the user authored channels.twitch.accessToken as an env reference, the changed token path is not restored and can be persisted as a literal OAuth token.
  - confidence: 0.93
- **[P2] Use focused config mutation subpaths:** `extensions/twitch/src/token-writeback.ts:1-5`
  - body: This new bundled plugin production file imports openclaw/plugin-sdk/config-runtime, but current main marks that broad barrel deprecated and the config boundary guard rejects it for extension code. Use config-contracts plus config-mutation so the write goes through the supported mutation surface with explicit after-write behavior.
  - confidence: 0.88
- **[P3] Guard optional refresh metadata updates:** `extensions/twitch/src/token-writeback.ts:55-66`
  - body: The helper type allows missing expiresIn and obtainmentTimestamp, but absence clears expiresIn to null and synthesizes a new timestamp, making direct helper calls rewrite config even when no metadata actually changed. Either require the Twurple AccessToken shape or skip absent optional fields consistently.
  - confidence: 0.74
