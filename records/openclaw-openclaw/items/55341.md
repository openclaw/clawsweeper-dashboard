## Summary

Keep open: current main still lacks both the Twitch refresh writeback and the OpenProse exclusive-output flow, but this branch is not merge-ready because its token writeback can persist env-authored secrets as literal OAuth tokens, uses deprecated config-runtime APIs that current guardrails reject, and has no real behavior proof.

## What This Changes

The branch adds Twitch OAuth refresh writeback for config-backed accounts with tests and changes an OpenProse fast-loop example so terminal output branches are exclusive.

## Best Possible Solution

Revise the branch to preserve or skip env-authored Twitch token fields, use config-mutation with explicit afterWrite behavior, keep the OpenProse else-chain fix, add env-ref regression coverage, and include redacted real behavior proof.

## Reproduction Assessment

Yes for the PR-introduced blocker by source inspection: a config-authored env token resolves as a config token, then the PR mutates snapshot.config and writes a changed token path that env restoration will not re-template. I did not run a live Twitch refresh in this read-only review.

## Solution Assessment

No. The feature direction is useful, but this implementation is not the narrowest maintainable solution until it preserves env-authored token references and uses the focused config-mutation API.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve authored env token references:** `extensions/twitch/src/token-writeback.ts:106`
  - body: This clones `snapshot.config`, which is already env-substituted, and then writes a changed access token path. If the user authored `channels.twitch.accessToken` as `${OPENCLAW_TWITCH_ACCESS_TOKEN}` or another env ref, the refreshed token will be persisted as a literal OAuth token instead of preserving the authored reference.
  - confidence: 0.92
- **[P2] Use the focused config-mutation API:** `extensions/twitch/src/token-writeback.ts:1-5`
  - body: This new production plugin file imports `openclaw/plugin-sdk/config-runtime`, but current main marks that broad barrel deprecated for bundled extension production imports and the config-boundary guard flags it. Move the writeback through `openclaw/plugin-sdk/config-mutation` with `mutateConfigFile` or `replaceConfigFile` and an explicit `afterWrite` policy.
  - confidence: 0.88
