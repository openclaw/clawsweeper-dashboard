## Summary

Keep open: the PR addresses real Twitch refresh persistence and OpenProse flow bugs, but the current diff introduces a token secret-handling regression and conflicts with the current config mutation boundary.

## What This Changes

The PR adds config-backed Twitch OAuth refresh-token writeback with tests and changes an OpenProse fast-loop example so terminal outputs are guarded by exclusive branches.

## Best Possible Solution

Revise the PR to use source-aware config-mutation writeback, skip or preserve authored env-referenced token fields, treat omitted optional token metadata as no-ops, keep the OpenProse else-flow fix, and add the required changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a config-authored Twitch token env reference resolves into snapshot.config, the PR writes a different refreshed literal, and current writeback treats that path as changed rather than restoring the env ref.

## Solution Assessment

No. The feature direction is useful, but this PR is not the best implementation until it uses the focused config mutation API, preserves env-authored token fields, and skips absent optional token metadata.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Preserve env-referenced Twitch token fields:** `extensions/twitch/src/token-writeback.ts:106`
  - body: This helper mutates `snapshot.config`, which has already resolved `${...}` env refs. When the refreshed token differs from the env snapshot, `writeConfigFile` marks the token path as changed and keeps the new literal OAuth token instead of restoring the authored env ref, leaking a secret into config and breaking env-managed token workflows.
  - confidence: 0.93
- **[P2] Use config-mutation instead of config-runtime writes:** `extensions/twitch/src/token-writeback.ts:1-5`
  - body: Production bundled plugin code now imports `openclaw/plugin-sdk/config-runtime` and calls `writeConfigFile`. Current main's config boundary guard rejects that broad subpath and direct write API, so this needs `openclaw/plugin-sdk/config-mutation` with `mutateConfigFile` or `replaceConfigFile` and an explicit `afterWrite` policy.
  - confidence: 0.88
- **[P2] Skip expiry writes when refresh omits it:** `extensions/twitch/src/token-writeback.ts:55-59`
  - body: When `token.expiresIn` is `undefined` or `null`, this converts the stored expiry to `null` and marks the config changed. Missing optional expiry metadata should leave the existing stored value alone, matching the refresh-token guard below.
  - confidence: 0.86
- **[P3] Skip timestamp writes when refresh omits it:** `extensions/twitch/src/token-writeback.ts:61-66`
  - body: When `token.obtainmentTimestamp` is absent, `Date.now()` makes the holder differ on every refresh callback, causing repeated config rewrites even when token data did not otherwise change. Only update this field when the refresh payload carries an explicit timestamp.
  - confidence: 0.83
- **[P3] Add the required changelog entry:** `extensions/twitch/src/twitch-client.ts:54`
  - body: This PR changes user-facing Twitch token persistence and OpenProse behavior, but the file list does not include `CHANGELOG.md`. OpenClaw policy requires a changelog entry for user-facing fix changes.
  - confidence: 0.8
