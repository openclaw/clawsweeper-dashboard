## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve env-authored token refs before writeback:** `extensions/twitch/src/token-writeback.ts:105-113`
  - body: The new helper mutates the resolved config snapshot and writes it whenever `tokenSource` is `"config"`. A config value authored as `${OPENCLAW_TWITCH_ACCESS_TOKEN}` is resolved before `resolveTwitchToken` classifies it, and the refreshed token will no longer match the env-ref restoration value, so this can persist the OAuth token literally. Gate writeback on the authored source/SecretRef state, or skip those holders.
  - confidence: 0.91
- **[P2] Use the config-mutation subpath for bundled writes:** `extensions/twitch/src/token-writeback.ts:1-5`
  - body: Bundled plugin production code cannot import the broad `openclaw/plugin-sdk/config-runtime` barrel; the boundary guard flags this pattern, and docs require `config-mutation` writes with an explicit `afterWrite` policy. Import the focused subpaths and use `replaceConfigFile` or `mutateConfigFile` so the runtime reload behavior is explicit.
  - confidence: 0.94
