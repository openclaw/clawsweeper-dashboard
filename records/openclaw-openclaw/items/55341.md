## Summary

Keep open: current main has not adopted the Twitch token persistence or the OpenProse branch fix, but this PR remains incorrect because the Twitch writeback path can persist env-authored OAuth tokens, uses a forbidden broad config-runtime import, and lacks after-fix real behavior proof.

## What This Changes

The branch adds Twitch OAuth refresh-token writeback with tests and changes one OpenProse fast-loop example so terminal output branches are exclusive.

## Best Possible Solution

Keep the OpenProse else-chain fix, rework Twitch writeback through source-preserving focused config mutation APIs, add env-ref and absent-metadata regression coverage, and include redacted runtime proof.

## Reproduction Assessment

Yes. The PR-introduced blocker is source-reproducible: a config-authored ${ENV} token is resolved as a config token, the PR clones the resolved snapshot, changes accessToken, and current writeback skips env-ref restoration for changed paths.

## Solution Assessment

No. The OpenProse change is fine, but the Twitch writeback implementation should preserve authored credential references and use focused config-mutation APIs before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Preserve authored Twitch token refs:** `extensions/twitch/src/token-writeback.ts:104-112`
  - body: This clones the resolved config snapshot and writes the refreshed access token back through the normal writer. If the user authored the token as an env reference, the changed token leaf is not restored and can be persisted as a literal OAuth token.
  - confidence: 0.93
- **[P2] Use focused config mutation subpaths:** `extensions/twitch/src/token-writeback.ts:1-5`
  - body: Bundled plugin code now imports the broad config-runtime compatibility barrel, which current docs and boundary guards reject. Use config-contracts plus config-mutation, and update the tests to mock the focused subpaths.
  - confidence: 0.88
- **[P3] Guard absent refresh metadata:** `extensions/twitch/src/token-writeback.ts:55-66`
  - body: The helper type allows expiresIn and obtainmentTimestamp to be absent, but absence clears expiresIn to null and synthesizes a new timestamp, causing unnecessary or lossy writes for partial helper calls.
  - confidence: 0.74
