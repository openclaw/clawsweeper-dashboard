Full review comments:

- **[P1] Preserve env-managed tokens before writing refreshes:** `extensions/twitch/src/token-writeback.ts:103-111`
  - body: This path writes `snapshot.config` after env substitution. A config like `accessToken: "${OPENCLAW_TWITCH_ACCESS_TOKEN}"` is resolved as `tokenSource: "config"`; when Twitch returns a different refreshed token, `restoreEnvVarRefs` cannot restore the placeholder because the incoming value no longer matches the env snapshot, so the secret is serialized into `openclaw.json`. Detect authored env/SecretRef-backed token fields and skip or update only literal config-backed tokens.
  - confidence: 0.9
- **[P2] Use the narrow config mutation SDK subpath:** `extensions/twitch/src/token-writeback.ts:1-5`
  - body: Production extension code is guarded against `openclaw/plugin-sdk/config-runtime`; the config boundary guard reports that broad barrel and sibling writeback code imports `readConfigFileSnapshotForWrite`/`replaceConfigFile` from `openclaw/plugin-sdk/config-mutation`. This new import will trip the boundary guard until it moves to the narrow config mutation/type subpaths.
  - confidence: 0.86
