## Summary

Keep this PR open, but it is not merge-ready: current main still lacks both the Twitch token persistence and the OpenProse branch fix, while the PR head introduces a concrete env-referenced token leak, current config-boundary violations, and optional-field writeback bugs.

## What This Changes

The PR adds Twitch OAuth refresh-token writeback for config-backed accounts with tests and changes the OpenProse fast-loop example so terminal outcomes are guarded by exclusive else branches.

## Best Possible Solution

Revise the PR to preserve or skip env-referenced token fields, use config-mutation/replaceConfigFile with an explicit afterWrite policy, skip absent optional token metadata, add regression coverage, and rebase on current main.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a config-authored env reference resolves as a config token, then the PR clones snapshot.config and writes a refreshed literal while current env-ref restoration only restores values that still match the env snapshot.

## Solution Assessment

No. The feature direction is useful, but this PR is not the best implementation until it uses the focused config mutation API, preserves authored env-token references, and treats omitted optional token fields as no-ops.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve env-referenced Twitch tokens:** `extensions/twitch/src/token-writeback.ts:106`
  - body: This clones and mutates snapshot.config, which is already env-substituted. If the user authored channels.twitch.accessToken as ${OPENCLAW_TWITCH_ACCESS_TOKEN}, the refreshed value differs from the env snapshot, so the write path keeps the new literal OAuth token instead of restoring the env reference.
  - confidence: 0.92
- **[P2] Use focused config mutation subpaths:** `extensions/twitch/src/token-writeback.ts:1-5`
  - body: Current main rejects bundled plugin imports from openclaw/plugin-sdk/config-runtime, and this new production module imports read/write helpers from that broad barrel. Rework this to use config-types plus config-mutation helpers with an explicit afterWrite policy so the config-boundary guard can pass after rebase.
  - confidence: 0.88
- **[P2] Skip expiry writes when refresh omits it:** `extensions/twitch/src/token-writeback.ts:55-59`
  - body: When token.expiresIn is undefined or null, this converts the stored expiresIn to null and marks the config changed. Missing expiry should be handled like missing refreshToken: leave the existing stored value alone unless the refresh payload provides a number.
  - confidence: 0.84
- **[P3] Skip timestamp writes when refresh omits it:** `extensions/twitch/src/token-writeback.ts:61-66`
  - body: When token.obtainmentTimestamp is absent, Date.now() makes the holder differ on every refresh callback, causing repeated config rewrites even when token data did not otherwise change. Only update this field when the payload carries an explicit timestamp.
  - confidence: 0.82
