## Summary

Keep open. The report is concrete and recent, and current `main` still has config read/recovery paths that can write `.clobbered.*` snapshots without a doctor-scoped suppression or a regression test for `doctor --fix` across subprocess-style reads.

## What This Changes

Prevent `openclaw doctor --fix` from producing redundant identical `openclaw.json.clobbered.*` snapshots during sanctioned config migrations.

## Best Possible Solution

Keep normal `.clobbered.*` recovery for direct edits/startup corruption, but make the sanctioned `doctor --fix` migration path dedupe or suppress redundant identical snapshots for the duration of the repair flow.
