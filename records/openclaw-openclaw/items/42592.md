## Summary

Keep open: current main and v2026.5.12 still only provide partial npm diagnostics and broad fallback retries, not the requested structured installer error taxonomy or selective transient retry policy. This is a product-scoped installer UX feature, not a narrow broken existing contract.

## What This Changes

The issue requests structured installer error-code handling, clearer npm remediation messages, and selective retry behavior for transient install failures.

## Best Possible Solution

Define a scoped installer error policy with tested npm-code diagnostics and transient retries only around safe install steps, while preserving intentional best-effort probes and keeping Unix, Windows, Docker, update, and docs behavior aligned.

## Solution Assessment
