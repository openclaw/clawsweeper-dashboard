## Summary

This should stay open: current main and the latest release have useful partial installer diagnostics and targeted recovery, but they still do not implement a structured npm/installer error policy with retryable network classification and per-code guidance. The remaining work is product-scoped because automatic permission repair and replacing optional probe fallbacks need maintainer decisions.

## What This Changes

Request to add structured installer error classification, actionable npm failure guidance, and smart retries for transient install failures.

## Best Possible Solution

Define a narrow maintainer-approved installer error policy, then add focused helpers and regression tests for transient npm/network retry and per-code guidance without broad auto-repair or blanket optional-probe hard failures.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this is an enhancement request. Source inspection does confirm that current main only has partial diagnostics and targeted retries, not the requested structured retry/error policy.

## Solution Assessment

No, the broad Bash `eval` retry wrapper and blanket `|| true` replacement are not the best final shape. A safer solution is a narrow, tested installer policy around known npm failures and explicitly optional config probes.
