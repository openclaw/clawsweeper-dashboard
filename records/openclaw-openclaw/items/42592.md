## Summary

Keep open: current main and v2026.6.8 still only provide partial npm diagnostics plus a few targeted installer retries, while the requested structured per-code npm guidance, transient network retry policy, and broader permission-recovery behavior remain unresolved product decisions.

## What This Changes

Request structured installer error handling, clearer npm failure guidance, and retry behavior for transient installation failures.

## Best Possible Solution

Define a narrow installer error policy, then add focused helpers and regression coverage for transient npm/network retry and per-code guidance while leaving optional setup probes explicitly optional.

## Reproduction Assessment

No live failing installer run was reproduced for this review. Source inspection does show current main lacks the requested structured npm error policy and transient network retry classification.

## Solution Assessment

No: the broad sample Bash wrapper is not the best final shape because it could retry permanent failures or hard-fail optional probes. A safer solution is a narrow, tested installer policy around known npm failures and explicitly optional setup probes.
