## Summary

This remains useful only as a narrower installer-policy item. Current main already provides structured JSON failures, bounded download retries, explicit npm-prefix remediation, and troubleshooting guidance, but a failed `openclaw@latest` installation still falls back to `openclaw@next` without classifying the error or preserving the requested release channel.

## What This Changes

Request structured installer error classification, transient retry handling, actionable remediation, and replacement of silently ignored failures.

## Best Possible Solution

Define and test one version-preserving installer failure contract, rely on supported npm fetch retries where adequate, add focused remediation for demonstrated fatal classes, and keep npm or shell configuration repair opt-in.

## Reproduction Assessment

Not applicable as a single bug reproduction: this is a broad improvement proposal, although source inspection clearly establishes the remaining unconditional `latest` to `next` fallback.

## Solution Assessment

No. The proposed generic `eval` retry wrapper and automatic permission repair are broader and riskier than a narrow, version-preserving installer contract with explicit configuration changes.
