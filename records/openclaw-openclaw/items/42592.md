The issue requests structured installer error classification, clearer remediation guidance, and retry behavior for transient install failures.

## Best Possible Solution

Define a narrow installer error policy first, then implement classified npm failure guidance and bounded retries in Bash/PowerShell installers while keeping optional setup probes explicitly best-effort.

## Solution Assessment

No, not as proposed verbatim. The broad Bash `eval` retry wrapper and blanket `|| true` removal are riskier than a narrow, tested installer policy for known npm failures and explicitly optional probes.
