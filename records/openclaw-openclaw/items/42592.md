The issue requests structured installer error handling, clearer npm diagnostics, and retry behavior for transient installer failures.

## Best Possible Solution

Define a maintainer-approved installer error policy with code-aware diagnostics and transient retries only around safe npm/setup operations, then align Unix, Windows, Docker setup, update behavior, docs, and focused tests.
