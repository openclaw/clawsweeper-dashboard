## What This Changes

The issue requests structured installer error handling, per-code npm guidance, and smarter retry or repair behavior for installation failures.

## Best Possible Solution

Define a narrow installer error policy, then add focused helpers and regression coverage for classified npm failures while keeping optional setup probes explicitly best-effort.

## Reproduction Assessment

Yes at source level: current main lacks the requested structured retry/error policy, and macOS EACCES is not pre-repaired because `fix_npm_permissions` returns outside Linux. No live install run was attempted in this read-only review.

## Solution Assessment

No: the broad Bash `eval` retry wrapper and blanket `|| true` replacement are not the best final shape. A safer solution is a narrow, tested policy around known npm failures and explicitly optional config probes.
