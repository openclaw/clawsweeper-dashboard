## Summary

Current main still separates bearer-authenticated Control UI HTTP reads from WebSocket connect-frame authentication, so a reverse-proxy-injected header cannot bootstrap the browser session. The request remains useful, but the prior direct upgrade-header proposal was closed and the credential lifetime, trusted-header source, origin checks, and rate-limit behavior require an explicit security/product decision.

## What This Changes

Allow a reverse proxy to bootstrap Control UI shared-secret authentication from an HTTP header instead of requiring URL-fragment or manually stored credentials.

## Best Possible Solution

Define one hardened reverse-proxy bootstrap contract, preferably validating a purpose-specific header from configured trusted proxies on a same-origin endpoint and exchanging it for a narrowly scoped, short-lived browser session credential, while retaining existing fragment/manual-token and trusted-proxy identity modes.

## Reproduction Assessment

Yes, at source level with high confidence: a bearer header can authorize protected Control UI HTTP reads, but the browser WebSocket independently requires credentials in its connect frame and cannot read the proxy-injected request header.

## Solution Assessment

No final implementation direction is approved yet. A short-lived, purpose-specific credential exchange is safer and more maintainable than exposing the long-lived proxy bearer token to browser state or broadly treating every forwarded Authorization header as a gateway token.

## Risks / Open Questions

- Trusting a generic forwarded Authorization header can confuse an upstream OIDC bearer credential with the OpenClaw gateway secret and consume shared-secret rate-limit attempts.
- Returning or persisting the proxy's long-lived gateway secret in browser-readable bootstrap data would enlarge the credential exposure boundary.
- Authenticating directly during WebSocket upgrade must preserve current origin validation, trusted-proxy source validation, pre-authentication connection limits, and shared-secret rate limiting.
