## Summary

Keep open. The PR targets a real shared `/status` bug, but the current patch can display an older compaction checkpoint as current context and the external contributor still has not provided after-fix real behavior proof.

## What This Changes

The PR adds a status formatter fallback from stale session totals to compaction checkpoint `tokensAfter` and adds a regression test for post-compaction context output.

## Best Possible Solution

Land a corrected shared formatter fix that selects the latest checkpoint first, uses its finite non-negative `tokensAfter` only when totals are stale or unavailable, adds the missing regression coverage, and includes real `/status` proof.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `buildStatusMessage()` on current main with `totalTokensFresh: false`, stale totals, and a latest compaction checkpoint containing `tokensAfter`; current main renders unknown context instead of the checkpoint post-compaction size.

## Solution Assessment

No. The PR direction is narrow, but the helper validates `tokensAfter` before selecting the latest checkpoint; the maintainable fix is latest-checkpoint-first, then validate that latest checkpoint's token count.
