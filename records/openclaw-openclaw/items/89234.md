## Summary

Keep open: the browser credential-placeholder feature is focused and useful, but it is not merge-ready because source inspection confirms env SecretRef allowlists are optional for the implicit default provider, resolved secret redaction is process-global, and the external PR still has no inspectable real browser proof.

## What This Changes

The PR adds `{{env:KEY}}` substitution for browser type/fill/select/batch action values, redacts resolved values from snapshots, adds unit tests, and documents the syntax in the browser automation skill.

## Best Possible Solution

Keep the capability only after explicit configured allowlist gating, scoped redaction lifecycle, unsafe-default regression tests, redacted real browser proof, and maintainer security review.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence for the PR-introduced security issue: with no explicit env provider allowlist, current main resolves the implicit default env provider and the PR would pass `{{env:KEY}}` through that path before browser dispatch.

## Solution Assessment

No. The feature direction is plausible, but the current patch is not the best merge shape because it relies on optional allowlist behavior and process-global redaction state; explicit allowlist gating and scoped redaction are safer.

Full review comments:

- **[P1] Require explicit env allowlist before substituting:** `extensions/browser/src/browser/env-secrets.ts:97-99`
  - body: This relies on `resolveSecretRefValues` to enforce the allowlist, but current main supplies `{ source: "env" }` for the implicit default provider and only checks `providerConfig.allowlist` when it exists. In configs without `secrets.providers.default.allowlist`, a browser-capable agent can resolve and type any set process env var by name despite the PR's allowlist-only contract.
  - confidence: 0.94

  - body: `knownSecretValues` is module-global and never cleared or tied to an agent, profile, tab, request, or browser lifecycle. After one action resolves a secret, unrelated later snapshots in the same process can be altered and callers can infer guessed-secret presence by watching for redaction.
  - confidence: 0.84
