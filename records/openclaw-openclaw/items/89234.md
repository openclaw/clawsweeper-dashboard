## Summary

Keep open: the feature is useful, but the PR is not merge-ready because the new placeholder resolver relies on the existing implicit env SecretRef behavior, which is unrestricted when no env provider allowlist is configured, and it stores resolved secrets in process-wide redaction state. The external PR also lacks after-fix real browser proof.

## What This Changes

The PR adds browser action support for resolving `{{env:KEY}}` placeholders in type/fill/select/batch actions, redacts resolved values from snapshots, and documents the syntax in the browser automation skill.

## Best Possible Solution

Keep the browser capability only after explicit allowlist-only placeholder gating, scoped or lifecycle-bound redaction state, regression coverage for unsafe defaults, and redacted real browser proof.

Yes, source-level reproduction is clear: with normal empty or unrestricted env SecretRef config, `{{env:GITHUB_TOKEN}}` resolves through the implicit default env provider and can be typed by the browser action path. I did not run the browser, but current source and tests prove that resolver behavior.

## Solution Assessment

No. The feature direction is useful, but this patch is not the best shape because it depends on SecretRef's unrestricted implicit default env provider and uses process-global redaction state; a safer implementation would require an explicit allowlist and scoped redaction.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Require an explicit env allowlist before substituting:** `extensions/browser/src/browser/env-secrets.ts:94-99`
  - body: This call uses the default env SecretRef provider, but current OpenClaw treats a missing default env provider as `{ source: "env" }` with no allowlist. In a normal config with no `secrets.providers.default.allowlist`, `{{env:GITHUB_TOKEN}}` will resolve and be typed into whatever page the agent controls, which breaks the stated security contract. Please reject placeholders unless the selected env provider is explicitly configured with an allowlist containing the requested key.
  - confidence: 0.94
- **[P2] Scope resolved secret redaction state:** `extensions/browser/src/browser/env-secrets.ts:28-35`
  - body: `knownSecretValues` is a module-global set that is never cleared or tied to an agent, profile, tab, or request. Once one action resolves a secret, unrelated later snapshots in the same process will redact that value, which can leak information as a guessed-secret oracle and can corrupt unrelated browser output. Please keep redaction material scoped to the browser/session context that typed it, or avoid retaining raw secret values globally.
  - confidence: 0.82
