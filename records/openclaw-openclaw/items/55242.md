## Summary

Keep open. Current main still passes raw `identity.emoji` text into Feishu card headers in the reply, streaming, presentation, and direct outbound card paths, and the adjacent header-hide work at https://github.com/openclaw/openclaw/pull/54979 does not sanitize descriptive identity text.

## What This Changes

The issue requests that Feishu card headers stop rendering descriptive prose from `identity.emoji` and show only the agent name or an emoji-only prefix.

## Best Possible Solution

Add one Feishu-local card-header title resolver shared by all identity-derived card header paths, preserving explicit presentation titles while dropping non-emoji prose from `identity.emoji`.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure `identity.emoji` with descriptive text plus emoji, trigger Feishu card rendering, and the current card header code passes that raw value into the serialized card title.

## Solution Assessment

Yes. A narrow Feishu-local sanitizer is the best maintainable fix; changing the global identity parser or adding a new config toggle would be broader than the leak being reported.
