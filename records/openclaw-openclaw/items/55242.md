## Summary

Keep open. Current `main` and `v2026.5.22` still compose Feishu card headers from raw `identity.emoji` in reply/streaming, payload/presentation, and direct outbound card paths, so mixed prose-plus-emoji identity values can still leak into visible card titles.

## What This Changes

Feishu card mode should avoid displaying non-emoji prose from agent `identity.emoji` in card header titles.

## Best Possible Solution

Add a Feishu-local card-header title resolver shared by reply, streaming, payload/presentation, and direct outbound card paths, preserving explicit titles and names while dropping non-emoji prose from `identity.emoji`.

## Reproduction Assessment

Yes, at source level. Configure `identity.emoji` with descriptive text plus emoji, trigger Feishu card rendering, and current `main` passes that raw value into serialized card titles; no live Feishu send was run.

## Solution Assessment

Yes. A narrow Feishu-local sanitizer shared across identity-derived card header paths is the maintainable fix; changing global identity parsing or adding a new config option would be broader than this leak.
