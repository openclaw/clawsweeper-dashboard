## Summary

Keep open. Current main and the latest release still pass raw `identity.emoji` text into Feishu card headers, so a mixed prose-plus-emoji value can be serialized into the visible card title; the related header-hide PR does not sanitize identity text.

## What This Changes

The issue asks Feishu card headers to stop displaying non-emoji prose from `identity.emoji` when card rendering is used.

## Best Possible Solution

Add one Feishu-local card-header title resolver shared by all identity-derived header paths, preserving explicit presentation titles while dropping non-emoji prose from `identity.emoji`.

## Reproduction Assessment

Yes, at source level. Configure `identity.emoji` with descriptive text plus emoji, trigger Feishu card rendering, and current reply/outbound code passes that raw value into the serialized card title.

## Solution Assessment

Yes. A narrow Feishu-local sanitizer shared across reply, streaming, presentation, and outbound card header construction is the maintainable fix; changing global identity parsing or adding a config option would be broader than this leak.
