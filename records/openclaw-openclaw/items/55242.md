## Summary

Keep open. Current main and v2026.5.20 still compose Feishu structured-card headers from raw `identity.emoji` in reply, payload, and direct outbound card paths, and no closing PR is attached; this remains a narrow source-reproducible Feishu plugin bug.

## What This Changes

The issue asks Feishu card headers to avoid showing descriptive prose from `identity.emoji` when long or card-format messages render a structured card.

## Best Possible Solution

Add a Feishu-local card-header title helper shared by reply, streaming, payload, and direct outbound card paths that preserves explicit presentation titles and agent names while including `identity.emoji` only when it is a pure emoji sequence.

## Reproduction Assessment

Yes, at source level. Configure `identity.emoji` with prose plus emoji and trigger Feishu card rendering; current main passes that raw value into `plain_text` header content, although no live Feishu send was run.

## Solution Assessment

Yes. The narrowest maintainable fix is a shared Feishu-local sanitizer for identity-derived card titles; changing global identity parsing or adding a new config option would be broader than this leak.
