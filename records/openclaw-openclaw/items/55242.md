## Summary

Keep open. Current main still builds Feishu card headers by concatenating raw `identity.emoji` with the agent name in the streaming/static reply path and outbound card paths, and the latest release tag still contains the same behavior.

## What This Changes

The issue reports that Feishu long-message cards include non-emoji descriptive text from `identity.emoji` in the card header instead of showing only the agent name or a pure emoji prefix.

## Best Possible Solution

Fix Feishu’s shared card-header title construction so raw descriptive `identity.emoji` text cannot enter headers, and cover the reply-dispatcher plus outbound card call sites with focused regression tests.

## Reproduction Assessment

Yes, from source inspection: a configured `identity.emoji` containing descriptive text is interpolated directly into Feishu card headers before sending. I did not run a live Feishu card send in this read-only review.

## Solution Assessment

Yes. The best fix is a narrow Feishu plugin formatter/sanitizer shared by the current card-header call sites; adding a new config option would be broader than the reported leak requires.
