## Summary

Keep open. Current main and v2026.5.18 still pass raw `identity.emoji` text into Feishu structured-card headers in reply and outbound paths, so mixed prose-plus-emoji values can be rendered visibly; the adjacent header-hide PR does not sanitize identity text.

## What This Changes

The issue reports Feishu structured-card headers displaying the full `identity.emoji` value, including descriptive prose, when card rendering is used.

## Best Possible Solution

Add a Feishu-local card-header title resolver shared by identity-derived header paths, preserving explicit presentation/native titles while dropping non-emoji prose from `identity.emoji`.

## Reproduction Assessment

Yes, at source level. Configure `identity.emoji` with descriptive text plus emoji, trigger Feishu card rendering, and current main passes that raw value into serialized card titles; no live Feishu send was run.

## Solution Assessment

Yes. A narrow Feishu-local sanitizer shared across reply, streaming, presentation, and outbound card header construction is the maintainable fix; changing global identity parsing or adding a config option would be broader than this leak.
