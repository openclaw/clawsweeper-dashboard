## Summary

Keep open. Current main and the latest release still build Feishu card headers from the raw `identity.emoji` string, so descriptive non-emoji text can appear in card headers; the adjacent open card-header toggle PR is related but does not sanitize identity-derived titles.

## What This Changes

Feishu card headers should avoid displaying descriptive prose from `identity.emoji`, using the agent name alone or an emoji-only prefix when card/long-message rendering is used.

## Best Possible Solution

Add a Feishu-local shared card-header title resolver used by reply, streaming, presentation, and direct outbound card paths. It should preserve explicit presentation titles and valid identity names, include `identity.emoji` only when it is an emoji-only sequence, drop descriptive prose from identity-derived headers, keep the existing generic-`main` omission behavior, and add focused Feishu regression tests.

## Reproduction Assessment

Yes. The current source gives a high-confidence reproduction path: configure an agent with `identity.emoji` containing prose plus emoji, then send a Feishu message that uses card rendering; the header title is constructed from that raw string and serialized verbatim.

## Solution Assessment

Yes. The narrowest maintainable fix is Feishu-local header-title sanitization rather than changing the global identity parser or requiring users to hide all card headers.
