## Summary

Keep open. Current `main` and the latest release still interpolate raw `identity.emoji` into Feishu structured-card headers in reply and outbound paths, so prose-plus-emoji identity values can be shown visibly in card titles.

## What This Changes

The issue asks Feishu card headers to stop exposing non-emoji prose from `identity.emoji` when card-format messages are rendered.

## Best Possible Solution

Add a Feishu-local card-header title resolver shared by reply, streaming, presentation, and outbound card paths, preserving explicit titles/names while dropping non-emoji prose from `identity.emoji`.

## Reproduction Assessment

Yes, at source level. Configure `identity.emoji` with descriptive text plus emoji, trigger Feishu card rendering, and current `main` passes that raw value into serialized card titles; no live Feishu send was run.

## Solution Assessment

Yes. A narrow Feishu-local sanitizer shared across identity-derived card header paths is the maintainable fix; changing global identity parsing or adding a config option would be broader than this leak.
