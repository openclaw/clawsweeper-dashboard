## Summary

Keep open. Current main and the latest release still build Feishu structured-card headers from raw `identity.emoji` in reply, direct outbound, and payload/presentation card paths, so mixed prose-plus-emoji identity values can be displayed as card titles.

## What This Changes

The issue reports Feishu structured-card headers exposing descriptive prose from `identity.emoji` instead of showing only the agent name or a pure emoji prefix.

## Best Possible Solution

Add a Feishu-local card-header title resolver shared by reply, streaming, presentation/payload, and direct outbound card paths, preserving explicit titles/names while dropping non-emoji prose from `identity.emoji`.

## Reproduction Assessment

Yes, at source level. Configure `identity.emoji` with descriptive text plus emoji, trigger Feishu card rendering, and current main passes that raw value into serialized card titles; no live Feishu send was run.

## Solution Assessment

Yes. A narrow Feishu-local sanitizer shared across identity-derived card header paths is the maintainable fix; changing global identity parsing or adding a new config option would be broader than this leak.
