## Label Justifications

- `P2`: This is a valid Feishu plugin bug with user-visible card header leakage, but the blast radius is limited to Feishu card rendering.

## Summary

Keep open. Current main and the latest release still pass raw `identity.emoji` text into Feishu card headers, so mixed prose-plus-emoji identity values can become visible card titles; the adjacent header-hide PR does not sanitize this value.

## What This Changes

The issue requests that Feishu card headers avoid rendering non-emoji prose from `identity.emoji` and show only the agent name or a pure emoji prefix.

## Best Possible Solution

Add a Feishu-local shared card-header title resolver used by reply, streaming, presentation, and outbound card paths, preserving explicit presentation titles while dropping non-emoji prose from `identity.emoji`.

## Reproduction Assessment

Yes, at source level. Configure `identity.emoji` with descriptive text plus emoji, trigger Feishu card rendering, and current main passes that raw value into serialized card titles; no live Feishu send was run.

## Solution Assessment

Yes. A narrow Feishu-local sanitizer shared across identity-derived card header paths is the maintainable fix; changing global identity parsing or adding a config option would be broader than this leak.
