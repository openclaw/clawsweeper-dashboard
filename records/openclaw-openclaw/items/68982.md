## Summary

Keep open. The PR adds useful Control UI rendering support, but the branch is still UI-only while current main’s live chat pipeline emits and stores text-only deltas, so it does not fix reasoning streaming end to end.

## What This Changes

The PR changes Control UI chat item construction and streaming rendering to accept structured stream messages, pass reasoning visibility into streaming groups, render reasoning-only assistant content, and add grouped-render coverage.

## Best Possible Solution

Land one consolidated end-to-end reasoning-stream implementation across gateway chat delta/final payloads, Control UI stream state, item building/rendering, tests, and changelog while preserving the `/reasoning` visibility contract.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: current main emits text-only chat deltas and `handleChatEvent` stores `extractText(payload.message)` in a string stream field, so a thinking-only live payload cannot reach the new object-rendering path.

## Solution Assessment

No. Renderer support alone is not the best fix; the maintainable solution needs structured payload plumbing through gateway and controller state, with reasoning visibility kept separate from `/think` effort.
