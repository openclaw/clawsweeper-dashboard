## Summary

Keep open. The PR targets useful reasoning-stream work, but this branch is not ready to merge because it only updates the Control UI render side, lacks after-fix real behavior proof, and can widen reasoning visibility beyond the documented `/reasoning` setting.

## What This Changes

The PR updates Control UI chat item construction, streaming rendering, chat view reasoning gating, and grouped-render tests to accept structured stream messages and render reasoning-only live bubbles.

## Best Possible Solution

Land one coordinated gateway/controller/UI change, likely reconciled with https://github.com/openclaw/openclaw/pull/59086, that carries structured thinking/text blocks end to end while preserving `/reasoning` visibility gates.

## Reproduction Assessment

Yes. Source inspection shows current main emits and stores live chat streams as text-only payloads, so this PR's object-stream renderer path cannot receive thinking-only live payloads in real runs.

## Solution Assessment

No. Renderer support is useful, but the safer solution must also update gateway emission, controller/app stream state, segment handoff, visibility gating, and tests together.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Plumb structured streams through the live chat path:** `ui/src/ui/chat/build-chat-items.ts:17-18`
  - body: `BuildChatItemsProps.stream` accepts object payloads, but the real live path still types `chatStream` as `string | null`, stores `extractText(payload.message)`, and receives text-only gateway chat messages. Reasoning-only deltas therefore never reach this renderer in real runs.
  - confidence: 0.94
- **[P2] Preserve the reading indicator when reasoning is hidden:** `ui/src/ui/chat/build-chat-items.ts:301-306`
  - body: Object streams are emitted as renderable stream items even when their only content is thinking. With `showReasoning=false`, the renderer suppresses the bubble, so users can see empty assistant chrome instead of the existing reading indicator until visible text arrives.
  - confidence: 0.9
- **[P2] Keep thinking effort out of reasoning visibility:** `ui/src/ui/views/chat.ts:199`
  - body: `resolveStreamingReasoningLevel` falls back to `props.thinkingLevel` when session reasoning is unset. `/think` controls model effort while `/reasoning` controls display, so sessions with thinking enabled can reveal reasoning without explicitly enabling reasoning display.
  - confidence: 0.91
