## Summary

Keep open. The latest PR head is useful Control UI renderer work, but it is not merge-ready because the live chat path on current main still emits and stores text-only deltas, while the patch also widens reasoning visibility and can render hidden reasoning as an empty assistant row.

## What This Changes

The PR updates Control UI chat item building and grouped streaming rendering to accept optional structured stream payloads, pass reasoning visibility into streaming groups, render reasoning-only assistant bubbles, and add grouped-render tests.

## Best Possible Solution

Land one consolidated end-to-end reasoning-stream implementation across gateway chat delta/final payloads, Control UI controller state, stream segment preservation, grouped rendering, tests, and changelog while keeping `/think` effort separate from `/reasoning` visibility.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: current main emits text-only chat deltas and stores `extractText(payload.message)` in string `chatStream`, so thinking-only or structured deltas cannot reach this PR's object-stream rendering branch in real live chat.

## Solution Assessment

No. The renderer changes are useful, but the branch is not the best complete fix until structured payloads are plumbed through gateway and controller state and the reasoning visibility fallback follows the documented `/reasoning` contract.

## Review Findings

Thinking-only deltas still never reach this branch in real runs, so this PR does not fix live reasoning streaming end to end.
  - confidence: 0.94
- **[P2] Keep hidden reasoning streams on the reading indicator:** `ui/src/ui/chat/build-chat-items.ts:299-300`
  - body: The object stream path treats any object as renderable even when it has no visible text. If the payload is reasoning-only and `showReasoning` is false, `renderGroupedMessage` returns `nothing`, so the prior reading indicator is replaced by an empty assistant group until answer text arrives.
  - confidence: 0.88
- **[P2] Keep thinking effort out of reasoning visibility:** `ui/src/ui/views/chat.ts:195-200`
  - body: `/think` controls model effort, while `/reasoning` controls whether thinking is shown and falls back to `off`. Falling back from a missing `sessionReasoningLevel` to `props.thinkingLevel` can show reasoning for sessions that never enabled reasoning visibility.
  - confidence: 0.9
- **[P3] Add the required changelog entry:** `ui/src/ui/chat/grouped-render.ts:342-343`
  - body: This changes user-facing Control UI live chat behavior, but the diff does not update `CHANGELOG.md`. OpenClaw policy requires a single-line active release entry for user-facing fix changes before merge.
