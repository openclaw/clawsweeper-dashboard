## Summary

Keep open. The latest PR still does not carry structured reasoning payloads through the live gateway/controller path, and it changes reasoning visibility so `/think` effort can expose reasoning when `/reasoning` is unset.

## What This Changes

This PR updates Control UI chat item construction and grouped rendering so structured assistant stream payloads can carry reasoning-only or reasoning-plus-text content into streaming bubbles, with grouped-render regression tests.

## Best Possible Solution

Land one consolidated end-to-end reasoning-stream implementation that preserves structured assistant payloads from gateway deltas through Control UI controller state into rendering, keeps `/reasoning` visibility independent from `/think` effort, and adds the required user-facing changelog entry.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: current main emits text-only chat deltas and stores live delta payloads as `extractText(payload.message)` in a string `chatStream`, so a reasoning-only live payload cannot reach this PR's object-stream renderer path.

## Solution Assessment

No. Renderer support is useful, but this is not the best standalone fix until structured payloads are plumbed through gateway and controller state and the documented `/reasoning` fallback remains `off`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Plumb structured stream payloads before rendering:** `ui/src/ui/chat/build-chat-items.ts:15`
  - body: `BuildChatItemsProps.stream` now accepts object payloads, but the live chat pipeline still cannot produce them: current `main` emits text-only deltas and `handleChatEvent` stores `extractText(payload.message)` in `chatStream`. Reasoning-only live deltas never reach this object path, so the PR does not fix live reasoning streaming end to end.
  - confidence: 0.94
- **[P2] Keep hidden reasoning on the reading indicator:** `ui/src/ui/chat/build-chat-items.ts:299-300`
  - body: An object stream is now always treated as renderable, even when it only contains hidden reasoning. With `showReasoning=false`, `renderGroupedMessage` suppresses that content, so the previous reading indicator is skipped and users can see an empty assistant row until visible text arrives.
  - confidence: 0.87
- **[P2] Keep thinking effort out of reasoning visibility:** `ui/src/ui/views/chat.ts:195-200`
  - body: `/think` controls model effort, while `/reasoning` controls whether reasoning is displayed and currently defaults to `off`. Falling back from a missing session reasoning level to `props.thinkingLevel` can show reasoning for sessions that never enabled reasoning visibility.
  - confidence: 0.9
- **[P3] Add the required changelog entry:** `ui/src/ui/chat/grouped-render.ts:344-356`
  - body: This changes user-facing Control UI live chat rendering, but the PR file list has no `CHANGELOG.md` update. OpenClaw policy requires a single-line active release entry for user-facing fix changes before merge.
  - confidence: 0.82
