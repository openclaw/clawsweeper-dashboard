## Summary

Keep open. The renderer changes are useful, but the latest patch is still UI-only while current main emits and stores live chat deltas as text-only strings, so reasoning-only live payloads cannot reach the new structured rendering path; it also widens reasoning visibility via the thinking-level fallback.

## What This Changes

This PR updates Control UI chat item construction and grouped rendering so structured assistant stream payloads can carry reasoning-only or reasoning-plus-text content into streaming bubbles, with grouped-render regression tests.

## Best Possible Solution

Land one consolidated end-to-end reasoning-stream implementation that preserves structured assistant payloads from gateway deltas through Control UI state and rendering, keeps /reasoning visibility independent from /think effort, and includes the required changelog and regression coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: current main emits only text chat.delta payloads and stores delta content as extractText(payload.message) in string chatStream, so a reasoning-only live delta cannot reach this PR's object-stream renderer path.

## Solution Assessment

No. Renderer support is useful, but the narrow maintainable fix must plumb structured payloads through gateway and controller state and preserve the documented reasoning-visibility fallback of off.
