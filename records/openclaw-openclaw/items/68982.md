## Summary

Keep open: the PR still does not carry structured reasoning payloads through the real live gateway/controller path, lacks real behavior proof, and is currently conflicting; the useful renderer work should be coordinated with the broader stream-reasoning work.

## What This Changes

The PR changes Control UI chat item building, streaming rendering, chat view reasoning gating, and grouped-render tests to render structured assistant stream messages with thinking blocks.

## Best Possible Solution

Reconcile this with the broader reasoning-stream design by carrying structured thinking/text blocks through gateway emission, controller/app state, segment handoff, and rendering while keeping /reasoning visibility separate from /think effort.

## Reproduction Assessment

Yes for the PR's claimed fix gap. Source inspection shows current main stores live chat streams as text and emits text-only gateway chat messages, so the PR's object-stream renderer path cannot receive thinking-only live payloads in real runs.

## Solution Assessment

No. Renderer support is useful, but the maintainable solution needs end-to-end structured payload plumbing and visibility gating, likely coordinated with https://github.com/openclaw/openclaw/pull/59086.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Plumb structured streams through the live chat path:** `ui/src/ui/chat/build-chat-items.ts:17-18`
  - body: `BuildChatItemsProps.stream` accepts object payloads, but the real live path still keeps `chatStream` as `string | null`, assigns `extractText(payload.message)`, and receives text-only gateway chat messages. Reasoning-only deltas therefore never reach this renderer in real runs, so live reasoning still will not stream.
  - confidence: 0.94
- **[P2] Preserve the reading indicator when reasoning is hidden:** `ui/src/ui/chat/build-chat-items.ts:301-306`
  - body: Object streams are emitted as renderable stream items even when their only content is thinking. With `showReasoning=false`, the grouped renderer suppresses the bubble content while `renderStreamingGroup` still renders assistant chrome, replacing the reading indicator with an empty row until visible text arrives.
  - confidence: 0.9
- **[P2] Keep thinking effort out of reasoning visibility:** `ui/src/ui/views/chat.ts:199`
  - body: `resolveStreamingReasoningLevel` falls back to `props.thinkingLevel` when the session reasoning level is unset. `/think` controls model effort while `/reasoning` controls display, so sessions with thinking enabled can reveal reasoning without explicitly enabling reasoning display.
  - confidence: 0.91
