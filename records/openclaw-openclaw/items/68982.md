## Summary

Keep open. The PR is active and targets a real live-chat reasoning gap, but the latest diff is still not correct as-is: it adds renderer-side support for structured stream objects while current main still emits and stores live chat streams as plain text, and it can widen reasoning visibility by falling back from `/reasoning` to `/think`.

## What This Changes

The PR updates Control UI chat item construction and grouped streaming rendering to accept optional structured stream message payloads, render reasoning-only streaming bubbles, pass reasoning visibility into stream rendering, and add grouped-render regression coverage.

## Best Possible Solution

Land one consolidated end-to-end implementation that carries structured assistant thinking/text blocks from agent events through gateway chat delta/final payloads, UI controller state, buildChatItems, and renderStreamingGroup, while keeping `/reasoning` fallback at off, preserving the reading indicator when reasoning is hidden, and adding focused gateway/UI regression tests plus the required user-facing changelog entry.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: current main emits live chat deltas only from assistant text events and the Control UI stores `extractText(payload.message)` into a string stream, so reasoning-only live payloads cannot reach the new renderer object path.

## Solution Assessment

No. Renderer-side object support alone is not the best fix because the live payload is still collapsed before rendering; the safer solution is an end-to-end structured stream contract coordinated with the broader #59086 work.

Full review comments:

- **[P1] Plumb structured stream payloads end to end:** `ui/src/ui/chat/build-chat-items.ts:15`
  - body: `BuildChatItemsProps.stream` now accepts object payloads, but real live chat still cannot supply one: gateway chat delta/final messages are text-only, and the Control UI controller stores `extractText(payload.message)` into a `string | null` stream. Reasoning-only deltas are dropped before this renderer branch can run, so the stated live reasoning fix remains incomplete.
  - confidence: 0.94
- **[P2] Keep /reasoning fallback independent of /think:** `ui/src/ui/views/chat.ts:197`
  - body: `resolveStreamingReasoningLevel` falls back from a missing session reasoning level to `props.thinkingLevel`, but `/think` controls model effort while `/reasoning` controls display and falls back to `off`. A session with thinking enabled but no reasoning visibility override can show reasoning unexpectedly.
  - confidence: 0.9
- **[P2] Preserve the reading indicator when reasoning is hidden:** `ui/src/ui/chat/build-chat-items.ts:297-301`
  - body: The object-stream path emits a stream item for any structured message, even when it has no visible text. If reasoning is hidden, `renderGroupedMessage` suppresses the thinking block, replacing the previous reading indicator with an empty assistant row until answer text arrives.
  - confidence: 0.87
