## Summary

Keep open. The author has narrowed the PR and fixed the earlier streaming `showReasoning` hardcode noted in review, but the latest head is still a Control UI renderer slice: current main and the PR diff do not plumb structured thinking/text payloads through gateway chat deltas or Control UI stream state, so the stated live reasoning fix remains incomplete. The related #59086 is still open and covers the broader gateway/controller/rendering path, so this needs maintainer direction rather than cleanup closure.

## What This Changes

The PR updates Control UI chat item building and grouped streaming rendering to accept optional structured stream payloads, pass reasoning visibility into streaming rendering, add grouped-render tests, and derive streaming reasoning visibility from session reasoning or thinking level.

## Best Possible Solution

Land one consolidated end-to-end implementation that carries structured thinking and text from agent events through gateway chat delta/final payloads, Control UI stream state, chat item building, and grouped rendering, while keeping `/think` effort separate from `/reasoning` visibility and preserving the hidden-reasoning reading indicator behavior.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: on current main, thinking-only agent events do not become chat delta messages, and received delta messages are collapsed through `extractText` into a string `chatStream`, so the PR's object-stream rendering branch cannot be exercised by live chat.

## Solution Assessment

No. The renderer additions are useful, but they are not the best complete fix because the structured payload has to be carried through gateway and controller state first; the safer direction is the broader #59086-style pipeline with explicit reasoning visibility policy.
