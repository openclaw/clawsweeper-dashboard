## Summary

Close as superseded: this PR is now a partial Control UI renderer change that does not carry structured thinking payloads through the live gateway/controller path, while the same end-to-end reasoning-stream work is already tracked by https://github.com/openclaw/openclaw/pull/59086 and related open issues.

## What This Changes

The branch changes Control UI chat item building, grouped streaming rendering, chat view reasoning gating, and grouped-render tests so structured assistant stream messages can carry thinking blocks.

## Best Possible Solution

Keep one coordinated end-to-end reasoning stream path that carries structured thinking/text through gateway emission, controller/app state, stream segments, and rendering while preserving the separate /reasoning visibility contract.

## Reproduction Assessment

Yes at source level. Current main stores live chat streams as text and emits text-only gateway chat messages, so a thinking-only payload has no current path to the renderer branch added here.

## Solution Assessment

No. Renderer support is useful, but this PR is not the narrowest maintainable standalone fix because the live stream contract needs to be plumbed end to end and coordinated with https://github.com/openclaw/openclaw/pull/59086.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Plumb structured streams through the live chat path:** `ui/src/ui/chat/build-chat-items.ts:17-18`
  - body: `BuildChatItemsProps.stream` now accepts object payloads, but the real live path still keeps `chatStream` as `string | null`, assigns `extractText(payload.message)`, and receives text-only gateway chat messages. Reasoning-only deltas therefore never reach this renderer in real runs.
  - confidence: 0.94
- **[P2] Preserve the reading indicator when reasoning is hidden:** `ui/src/ui/chat/build-chat-items.ts:301-306`
  - body: Object streams are emitted as renderable stream items even when their only content is thinking. With `showReasoning=false`, the grouped renderer suppresses the bubble content while `renderStreamingGroup` still renders assistant chrome, replacing the reading indicator with an empty row until visible text arrives.
  - confidence: 0.9
- **[P2] Keep thinking effort out of reasoning visibility:** `ui/src/ui/views/chat.ts:199`
  - body: `resolveStreamingReasoningLevel` falls back to `props.thinkingLevel` when the session reasoning level is unset. `/think` controls model effort while `/reasoning` controls display, so sessions with thinking enabled can reveal reasoning without explicitly enabling reasoning display.
  - confidence: 0.91
