## Summary

Keep open. The PR still only changes the Control UI rendering side while current main keeps live chat gateway/controller payloads text-only, and one added fallback can widen reasoning visibility beyond the documented `/reasoning` setting.

## What This Changes

The PR updates Control UI chat item construction, streaming rendering, chat view reasoning gating, and grouped-render tests so live stream items can carry structured assistant messages with thinking/text blocks.

## Best Possible Solution

Land one coordinated gateway/controller/UI fix, likely reconciled with #59086, that carries structured thinking and text end to end while preserving `/reasoning` visibility semantics and adding the required changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: gateway chat events are text-only and Control UI stores live deltas through extractText into a string chatStream, so thinking-only live content cannot reach this PR's object-stream renderer.

## Solution Assessment

No. Renderer support is useful, but the patch is incomplete until gateway emission, controller/app stream state, segment handoff, visibility gating, tests, and changelog are handled together.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Plumb structured stream payloads through chat state:** `ui/src/ui/chat/build-chat-items.ts:15`
  - body: `BuildChatItemsProps.stream` now accepts object payloads, but the live path still cannot produce them: current main emits text-only chat messages and `handleChatEvent` stores only `extractText(payload.message)` in string `chatStream`. Reasoning-only deltas therefore never reach this object branch in real runs.
  - confidence: 0.94
- **[P2] Keep hidden reasoning streams on the reading indicator:** `ui/src/ui/chat/build-chat-items.ts:301-306`
  - body: When `props.stream` is an object, this branch emits a stream item even if its only content is thinking. With `showReasoning=false`, the renderer suppresses the bubble, so the existing reading indicator is skipped and users get an empty assistant row until visible text arrives.
  - confidence: 0.9
- **[P2] Keep thinking effort separate from reasoning visibility:** `ui/src/ui/views/chat.ts:199`
  - body: `resolveStreamingReasoningLevel` falls back to `props.thinkingLevel` when the session has no reasoning setting. `/think` controls model effort while `/reasoning` controls display and defaults to off, so this can show reasoning for sessions that did not enable reasoning display.
  - confidence: 0.91
- **[P3] Add the required changelog entry:** `ui/src/ui/chat/grouped-render.ts:339-356`
  - body: This is a user-facing Control UI live chat fix, but `CHANGELOG.md` is not updated. OpenClaw policy requires a single-line active-release changelog entry for user-facing fix changes before merge.
  - confidence: 0.82
