## Summary

Keep open. The branch adds useful Control UI rendering pieces, but it still does not make live reasoning stream end to end because current gateway deltas and Control UI controller state remain text-only, and the new reasoning fallback can widen visibility beyond the documented `/reasoning` setting.

## What This Changes

The branch adds Control UI support for structured stream payloads, passes reasoning visibility into streaming rendering, allows reasoning-only assistant bubbles, and adds grouped-render regression coverage.

## Best Possible Solution

Land one coherent end-to-end reasoning-stream change across gateway chat delta/final payloads, Control UI controller stream state, rendering, tests, and changelog, or fold the useful UI pieces into #59086.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: current main emits text-only chat deltas and stores extracted text in a string `chatStream`, so structured or thinking-only deltas cannot reach this branch's object-stream rendering path in real live chat.

## Solution Assessment

No. Renderer support alone is not the best complete fix; the maintainable path is end-to-end structured payload plumbing while keeping `/think` effort separate from `/reasoning` visibility.

Thinking-only deltas still never reach this branch in real runs, so the PR does not fix live reasoning streaming end to end.
  - confidence: 0.94
- **[P2] Keep hidden reasoning streams on the reading indicator:** `ui/src/ui/chat/build-chat-items.ts:299-300`
  - body: The object stream path treats any object as renderable even when it has no visible text. If the payload is reasoning-only and `showReasoning` is false, grouped rendering returns no bubble content, so the prior reading indicator is replaced by an empty assistant group until answer text arrives.
  - confidence: 0.88
- **[P2] Keep thinking effort out of reasoning visibility:** `ui/src/ui/views/chat.ts:199`
  - body: `/think` controls model effort, while `/reasoning` controls whether thinking is shown and falls back to `off`. Falling back from a missing session `reasoningLevel` to `props.thinkingLevel` can show reasoning for sessions that never enabled reasoning visibility.
  - confidence: 0.9

  - body: This changes user-facing Control UI live chat behavior, but the diff does not update `CHANGELOG.md`. OpenClaw policy requires a single-line active-version entry for user-facing fix changes before merge.
