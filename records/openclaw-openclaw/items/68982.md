## Summary

Keep open. The latest branch is still not merge-ready because it adds renderer-side structured stream handling without the gateway/controller plumbing that would deliver structured reasoning deltas in live chat, and it widens reasoning visibility by falling back from `/reasoning` to `/think`; #59086 remains the broader overlapping implementation candidate.

## What This Changes

The branch extends Control UI chat item building and grouped streaming rendering to accept structured stream payloads, pass reasoning visibility into streaming groups, render reasoning-only assistant bubbles, and add grouped-render regression tests.

## Best Possible Solution

Land one consolidated end-to-end reasoning-stream implementation across gateway chat delta/final payloads, Control UI controller state, stream segment preservation, grouped rendering, tests, and changelog while keeping `/think` effort separate from `/reasoning` visibility.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: current main emits text-only chat deltas/finals and stores extracted text in string `chatStream`, so thinking-only or structured deltas cannot reach the PR's object-stream rendering branch in real live chat.

## Solution Assessment

No. The renderer changes are useful, but the branch is not the best complete fix until structured payloads are plumbed through gateway and controller state and the visibility fallback follows the documented `/reasoning` contract.
