## Summary

Keep open. The latest patch is not merge-ready because it only adds Control UI renderer support for object-valued streams while current main still emits and stores live chat deltas as text-only payloads; it also widens reasoning visibility by falling back from `/reasoning` to `/think`.

## What This Changes

This PR extends Control UI chat item construction, streaming rendering, chat view wiring, and grouped-render tests so structured streaming assistant message objects can carry thinking/text blocks into live stream rendering.

## Best Possible Solution

Land one coordinated gateway/controller/UI implementation, likely reconciled with #59086, that preserves structured thinking and text blocks end to end while keeping `/think` effort independent from `/reasoning` visibility and documenting the user-facing fix.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: current main emits text-only chat payloads and stores live deltas through `extractText(payload.message)` in a string `chatStream`, so a thinking-only live payload cannot reach this PR's object-stream renderer path.

## Solution Assessment

No. Renderer support is useful, but the maintainable fix must also plumb structured payloads through gateway emission and Control UI state, preserve hidden-reasoning loading behavior, and keep reasoning visibility defaulting to `off`.
