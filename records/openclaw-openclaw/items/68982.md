## Summary

Keep open. The PR targets a real live-chat reasoning gap, and the author addressed an earlier streaming visibility review thread, but the latest provided head is still incomplete: the changed files only add renderer-side structured stream handling while current main still emits chat deltas as text-only payloads and the UI controller still collapses incoming messages to a string before rendering. The diff also risks widening reasoning visibility by falling back from `/reasoning` to `/think`, contrary to the documented fallback.

## What This Changes

The PR changes Control UI chat item construction, grouped streaming rendering, chat view reasoning selection, and grouped-render tests to accept optional structured stream payloads and render thinking-only streaming bubbles.

## Best Possible Solution

Land one consolidated end-to-end implementation that carries structured thinking/text payloads from agent events through gateway chat delta/final events, Control UI stream state, item building, and grouped rendering, while keeping `/reasoning` visibility fallback at `off` and preserving the reading indicator when reasoning is hidden.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: emit a thinking-only agent event on current main and it will not become a chat delta because the gateway only converts assistant text events; structured thinking also cannot survive the current UI controller because `chatStream` is string-only and assigned from `extractText`.

## Solution Assessment

No. The current PR is not the best complete fix because renderer-side object support does not plumb structured payloads through the gateway or controller; the safer path is an end-to-end change coordinated with the broader #59086 work.

  - confidence: 0.94
- **[P2] Preserve the reading indicator when reasoning is hidden:** `ui/src/ui/chat/build-chat-items.ts:296-301`
  - body: The object-stream path emits a stream item for any structured message, even if it has no visible text. When reasoning is hidden, `renderGroupedMessage` suppresses the thinking block, so the previous reading indicator is replaced by an empty assistant row until answer text arrives.
  - confidence: 0.88
- **[P2] Keep thinking effort out of reasoning visibility:** `ui/src/ui/views/chat.ts:197`
  - body: `resolveStreamingReasoningLevel` falls back from a missing session reasoning level to `props.thinkingLevel`, but `/think` controls model effort while `/reasoning` controls whether reasoning is shown and falls back to `off`. A session with thinking enabled but no reasoning visibility override can show reasoning unexpectedly.
  - confidence: 0.9
