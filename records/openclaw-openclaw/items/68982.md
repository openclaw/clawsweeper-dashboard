## Summary

Keep open. The latest PR still does not implement live reasoning streaming end to end: the current main gateway only emits text chat delta/final payloads and the Control UI controller still stores live chat streams as text before the new renderer path could see structured thinking blocks.

## What This Changes

The PR updates Control UI chat item construction and grouped streaming rendering to accept optional structured stream message payloads, render reasoning-only streaming bubbles, pass reasoning visibility into stream rendering, and add grouped-render regression tests.

## Best Possible Solution

Land one consolidated end-to-end implementation that preserves structured thinking/text payloads from agent events through gateway chat delta/final payloads, Control UI stream state, item building, and grouped rendering while keeping /reasoning visibility fallback at off and preserving the reading indicator when reasoning is hidden.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: current main emits dedicated thinking agent events, but the chat delta path only consumes assistant text events and the UI controller stores extractText(payload.message) into a string stream.

## Solution Assessment

No. Renderer-side object support alone is not the best fix because the live payload is still collapsed before rendering; the safer path is an end-to-end structured stream change coordinated with the broader #59086 work.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Plumb structured payloads before rendering:** `ui/src/ui/chat/build-chat-items.ts:15`
  - body: `BuildChatItemsProps.stream` now accepts object payloads, but real live chat cannot supply one: gateway chat delta/final payloads are still text-only, and `ChatState.chatStream` is still populated from `extractText(payload.message)`. Reasoning-only deltas are dropped before this branch can render them, so the stated live reasoning fix remains incomplete.
  - confidence: 0.94
- **[P2] Preserve the reading indicator when reasoning is hidden:** `ui/src/ui/chat/build-chat-items.ts:296-301`
  - body: The object-stream path emits a stream item for any structured message, even if it has no visible text. When reasoning is hidden, `renderGroupedMessage` suppresses the thinking block, so the previous reading indicator is replaced by an empty assistant row until answer text arrives.
  - confidence: 0.88
- **[P2] Keep thinking effort out of reasoning visibility:** `ui/src/ui/views/chat.ts:197`
  - body: `resolveStreamingReasoningLevel` falls back from a missing session reasoning level to `props.thinkingLevel`, but `/think` controls model effort while `/reasoning` controls whether reasoning is shown and falls back to `off`. A session with thinking enabled but no reasoning visibility override can show reasoning unexpectedly.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The diff has no dependency, workflow, or supply-chain changes, but it can widen reasoning visibility by treating thinking effort as a display setting.

Concerns:

- **[medium] Reasoning visibility can be widened:** `ui/src/ui/views/chat.ts:197`
  - body: `resolveStreamingReasoningLevel` uses `thinkingLevel` when no session reasoning level is set. The documented /reasoning fallback is off, so sessions with thinking effort enabled could display reasoning blocks contrary to the visibility setting.
