## Summary

Keep open. Current main still drops structured thinking before live Control UI rendering, and this PR's latest UI-only patch does not plumb structured payloads through gateway/controller state while also risking reasoning visibility widening.

## What This Changes

This PR changes Control UI chat item construction, streaming rendering, chat view wiring, and grouped-render tests so stream items can carry structured assistant message payloads with thinking/text blocks.

## Best Possible Solution

Land one coordinated gateway/controller/UI implementation, likely reconciled with #59086, that preserves structured thinking and text blocks end to end while keeping /think effort independent from /reasoning visibility and documenting the user-facing fix.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: current main emits and stores live chat deltas as text-only payloads, so a thinking-only live payload cannot reach this PR's object-stream renderer path.

## Solution Assessment

No. Renderer support is useful, but the maintainable fix must also plumb structured payloads through gateway emission and Control UI state, preserve hidden-reasoning loading behavior, and keep reasoning visibility defaulting to off.

  - body: BuildChatItemsProps.stream now accepts object payloads, but the real live path still cannot produce them: current main emits text-only chat deltas and handleChatEvent stores extractText(payload.message) in chatStream.   - confidence: 0.94

  - body: When props.stream is an object, this branch emits a stream item even if its only content is thinking. With showReasoning=false, renderGroupedMessage returns no bubble, so the existing reading indicator is skipped and users see an empty assistant row until visible text arrives.
  - confidence: 0.9

  - body: resolveStreamingReasoningLevel falls back to props.thinkingLevel when the session has no reasoning setting. /think controls model effort while /reasoning controls visibility, so this can show reasoning for sessions where display should still default to off.
  - confidence: 0.91
