## Summary

Keep open. The branch still only teaches the Control UI renderer to accept structured stream objects, while current main emits and stores live chat streams as text-only payloads; it also changes the reasoning visibility fallback in a way that can expose thinking when `/reasoning` is unset.

## What This Changes

The PR updates Control UI chat item construction, streaming rendering, chat view wiring, and grouped-render tests so live stream objects with thinking blocks can be rendered.

## Best Possible Solution

Land one coordinated gateway/controller/UI implementation, likely reconciled with #59086, that preserves structured thinking and text blocks end to end while keeping `/think` effort independent from `/reasoning` visibility.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: current main emits text-only chat payloads and stores live deltas through `extractText(payload.message)` in a string `chatStream`, so a thinking-only live payload cannot reach this PR's object-stream renderer path.

## Solution Assessment

No. Renderer support is useful, but the maintainable fix must also plumb structured payloads through gateway emission and Control UI state, preserve hidden-reasoning loading behavior, and keep reasoning visibility defaulting to `off`.
