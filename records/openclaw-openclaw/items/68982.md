## Summary

Keep open. The latest branch is now a UI-only renderer change, while current main still emits and stores live chat deltas as text-only payloads, so the intended live reasoning stream is not fixed end to end and the reasoning visibility fallback can widen exposure.

## What This Changes

The PR updates Control UI chat item building and grouped streaming rendering to accept structured stream payloads, render reasoning-only assistant bubbles, pass reasoning visibility into streaming groups, and add grouped-render tests.

## Best Possible Solution

Land one consolidated end-to-end reasoning-stream implementation across gateway chat delta/final payloads, Control UI controller state, stream segment preservation, grouped rendering, tests, docs/changelog, and the existing /reasoning visibility contract.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: current main emits text-only chat deltas and stores extractText(payload.message) in string chatStream, so thinking-only or structured deltas cannot reach this PR's object-stream rendering branch in real live chat.

## Solution Assessment

No. The renderer changes are useful, but this is not the best complete fix until structured payloads are plumbed through gateway and controller state and reasoning visibility remains separate from /think effort.

  - confidence: 0.88

  - body: `/think` controls model effort, while `/reasoning` controls whether thinking is shown and falls back to `off`. Falling back from a missing `sessionReasoningLevel` to `props.thinkingLevel` can show reasoning for sessions that never enabled reasoning visibility.
  - confidence: 0.9
