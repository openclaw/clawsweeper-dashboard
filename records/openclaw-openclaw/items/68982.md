## Summary

Keep open. The latest branch is still UI-only while current main's live chat path emits and stores text-only deltas, so reasoning-only streaming payloads cannot reach the new renderer path; it also risks widening reasoning visibility beyond the documented `/reasoning` setting.

## What This Changes

The PR changes Control UI chat item construction and grouped stream rendering to accept structured assistant stream payloads, pass reasoning visibility into streaming bubbles, render reasoning-only messages, and add grouped-render tests.

## Best Possible Solution

Land one consolidated end-to-end implementation that carries structured reasoning/text blocks from gateway chat events through Control UI state and rendering, keeps `/reasoning` as the display gate, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: current main emits text-only chat deltas and stores `extractText(payload.message)` in a string `chatStream`, so a thinking-only live payload cannot reach this PR's object-stream rendering path.

## Solution Assessment

No. Renderer support is useful, but the proposed patch is not the best standalone solution until structured payloads are plumbed through gateway/controller state and thinking effort remains separate from reasoning visibility.

## Review Findings

  - body: `BuildChatItemsProps.stream` now accepts object payloads, but the live chat pipeline still cannot produce them: gateway deltas/finals are text-only and `handleChatEvent` stores `extractText(payload.message)` in string `chatStream`. Reasoning-only live deltas never reach this object path, so the PR does not fix live reasoning streaming end to end.
  - confidence: 0.94
- **[P2] Keep hidden reasoning on the reading indicator:** `ui/src/ui/chat/build-chat-items.ts:295-296`
  - body: Treating any object stream as renderable means a reasoning-only payload with `showReasoning=false` becomes a stream item even though rendering suppresses all bubble content. That skips the previous reading indicator and can leave an empty assistant row until visible text arrives.
  - confidence: 0.88
- **[P2] Keep thinking effort out of reasoning visibility:** `ui/src/ui/views/chat.ts:195-200`
  - body: `/think` controls model effort, while `/reasoning` controls whether reasoning is displayed and falls back to `off`. Falling back from a missing session reasoning level to `props.thinkingLevel` can show reasoning for sessions that never enabled reasoning visibility.
  - confidence: 0.9

  - body: This is a user-facing Control UI live chat rendering fix, but the PR file list has no `CHANGELOG.md` update. OpenClaw policy requires a single-line active release entry for user-facing fixes before merge.
