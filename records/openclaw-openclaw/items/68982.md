## Summary

Keep open. Current main still has the live-reasoning gap, and the latest branch does not solve it end to end: the PR diff is UI-only while gateway chat deltas and Control UI stream state remain text-only. The branch also introduces a reasoning-visibility regression by treating thinking effort as a fallback for `/reasoning` visibility.

## What This Changes

The PR branch updates Control UI chat item building and grouped streaming rendering to accept optional structured stream payloads, pass reasoning visibility into streaming rendering, add grouped-render tests, and derive streaming reasoning visibility from session reasoning or thinking level.

## Best Possible Solution

Land one consolidated end-to-end implementation that carries structured thinking/text payloads from agent events through gateway chat delta/final events, Control UI stream state and segments, item building, and grouped rendering, while preserving `/reasoning` fallback as `off` and keeping the reading indicator when hidden reasoning has no visible text. Coordinate that with the broader #59086 work rather than merging this renderer-only slice as-is.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: on current main, a thinking-only agent event does not become a chat delta because the gateway only converts assistant text events, and the UI controller collapses received delta messages through `extractText` into a string stream.

## Solution Assessment

No. This is not the best complete fix because renderer-side object support cannot be exercised by the current live chat pipeline; the safer solution is to plumb structured payloads through gateway and controller state first, then render them under the existing reasoning visibility policy.
