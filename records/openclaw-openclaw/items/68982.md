## Summary

Keep open: the latest patch still only changes the Control UI render/build path, while current main's gateway and controller live-chat path remains text-only; it also risks widening reasoning visibility by treating thinking effort as display permission.

## What This Changes

The PR changes Control UI chat item construction, streaming rendering, chat view reasoning gating, and grouped-render tests to accept structured stream messages and render reasoning-only live bubbles.

## Best Possible Solution

Land one coordinated gateway/controller/UI fix, likely reconciled with #59086, that carries structured thinking and text blocks end to end while preserving `/reasoning` visibility semantics and adding the required changelog entry.

## Reproduction Assessment

Yes. Source inspection of current main shows gateway chat events are text-only and Control UI stores live deltas through `extractText(payload.message)` in a string `chatStream`, so a thinking-only live payload cannot reach this PR's object-stream renderer path.

## Solution Assessment

No. Renderer support is useful, but the maintainable fix must also update gateway emission, controller/app stream state, segment handoff, visibility gating, tests, and changelog together.

  - body: When `props.stream` is an object, this branch emits a stream item even if its only content is thinking. With `showReasoning=false`, the renderer suppresses the bubble, so the existing reading indicator is skipped and users get an empty assistant row until visible text arrives.
  - confidence: 0.9

  - confidence: 0.91

  - body: This is a user-facing Control UI live chat fix, but `CHANGELOG.md` is not updated. OpenClaw policy requires a single-line active-release changelog entry for user-facing fix changes before merge.
