## Summary

Keep open. The branch adds useful rendering support, but the latest supplied head still cannot make live reasoning appear in real runs because current main collapses gateway chat events to text before the new object-stream renderer path can see them.

## What This Changes

The PR updates Control UI chat item construction, grouped streaming rendering, chat view wiring, and grouped-render tests so live chat can render structured assistant stream objects containing reasoning blocks.

## Best Possible Solution

Land one coordinated gateway/controller/UI implementation, likely reconciled with #59086, that preserves structured reasoning and text blocks end to end while keeping `/think` effort independent from `/reasoning` visibility.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: current main emits text-only chat payloads and stores live deltas as `extractText(payload.message)` in a string `chatStream`, so a thinking-only live payload cannot reach this PR's object-stream renderer path.

## Solution Assessment

No. Renderer support is useful, but the maintainable fix must also carry structured payloads through gateway emission and Control UI state, preserve hidden-reasoning loading behavior, and keep reasoning visibility defaulting to `off`.
