## Summary

Keep open. The latest head is still a UI-only renderer change, while current main still emits text-only chat deltas and stores live stream state as strings, so this does not fix live reasoning streaming end to end and also risks widening reasoning visibility.

## What This Changes

This PR changes Control UI chat item construction and grouped streaming rendering to carry structured stream message payloads, pass reasoning visibility into streaming bubbles, render reasoning-only bubbles, and add grouped-render tests.

## Best Possible Solution

Land one consolidated end-to-end reasoning-stream implementation across gateway chat delta/final payloads, Control UI stream state/types, grouped rendering, tests, and changelog while preserving /reasoning as the visibility gate.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: current main emits text-only chat deltas and handleChatEvent stores extractText(payload.message) in string chatStream, so a thinking-only payload cannot reach this PR's structured object-rendering path in real live chat.

## Solution Assessment

No. The renderer support is useful, but the fix is incomplete until structured reasoning payloads are plumbed through gateway and controller state without conflating /think effort with /reasoning visibility.
