## Summary

Keep open. The latest head still does not complete the live reasoning path because it adds renderer-side object handling without carrying structured thinking/text through gateway deltas or Control UI state, and it can widen reasoning visibility by treating `/think` effort as a `/reasoning` fallback. The open broader PR #59086 already covers the end-to-end shape, so maintainer direction is needed before more work lands here.

## What This Changes

The PR updates Control UI chat item building and grouped streaming rendering to accept optional structured stream payloads, pass reasoning visibility into streaming groups, render reasoning-only assistant bubbles, and add grouped-render tests.

## Best Possible Solution

Land one end-to-end reasoning-stream implementation across gateway chat delta/final payloads, Control UI controller state, stream segment preservation, grouped rendering, and changelog, while keeping `/think` effort separate from `/reasoning` visibility.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: current main emits text-only chat deltas and stores `extractText(payload.message)` in string `chatStream`, so thinking-only or structured deltas cannot reach this PR's object-stream rendering branch.

## Solution Assessment

No. The renderer additions are useful, but this is not the best complete fix until structured payloads are plumbed through gateway and controller state and the reasoning visibility fallback is kept aligned with the documented `/reasoning` contract.
