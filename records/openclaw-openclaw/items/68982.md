## Summary

Keep open. The latest branch adds useful renderer-side support, but it still does not preserve structured reasoning payloads through the live gateway/controller path and it can widen reasoning visibility beyond the documented `/reasoning` setting.

## What This Changes

This PR updates Control UI chat item construction and grouped rendering so structured assistant stream payloads can render reasoning-only or reasoning-plus-text streaming bubbles, with grouped-render regression coverage.

## Best Possible Solution

Land one end-to-end implementation that preserves structured reasoning/text blocks through gateway emission, UI controller state, and rendering while keeping `/think` effort independent from `/reasoning` visibility.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: current main emits text-only chat deltas and stores live deltas as `extractText(payload.message)` in a string `chatStream`, so a thinking-only live payload cannot reach this PR's object-stream renderer path.

## Solution Assessment

No. Renderer support is useful, but it is not the best standalone fix until structured payloads are plumbed through the gateway and controller state, hidden reasoning keeps the reading indicator, and reasoning visibility continues to default to `off`.
