## Summary

Keep open. The latest branch is a useful Control UI renderer slice, but it is still not mergeable as the live-chat reasoning fix because the gateway/controller path remains text-only and the new fallback can widen reasoning visibility beyond the documented `/reasoning` setting.

## What This Changes

The PR changes Control UI chat item construction and grouped stream rendering to accept structured assistant stream payloads, pass reasoning visibility into streaming bubbles, render reasoning-only messages, and add grouped-render tests.

## Best Possible Solution

Land one consolidated end-to-end reasoning-stream implementation across gateway chat delta/final payloads, Control UI stream state, item building/rendering, tests, and changelog while preserving the `/reasoning` visibility contract.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: current main emits text-only chat deltas and the Control UI stores `extractText(payload.message)` in a string `chatStream`, so a thinking-only payload cannot reach this PR's object-stream rendering path in real live chat.

## Solution Assessment

No. Renderer support is useful, but the proposed branch is not the best standalone fix until structured payloads are plumbed through gateway and controller state and `/reasoning` visibility remains separate from `/think` effort.

## Review Findings

Full review comments:

- **[P1] Plumb structured stream payloads before rendering:** `ui/src/ui/chat/build-chat-items.ts:15`
  - body: `BuildChatItemsProps.stream` now accepts object payloads, but the live chat pipeline still cannot produce them: gateway deltas are text-only and `handleChatEvent` stores `extractText(payload.message)` in string `chatStream`. Reasoning-only live deltas never reach this object path, so the PR does not fix live reasoning streaming end to end.
  - confidence: 0.94
- **[P2] Keep hidden reasoning on the reading indicator:** `ui/src/ui/chat/build-chat-items.ts:299-300`
  - body: Treating any object stream as renderable means a reasoning-only payload with `showReasoning=false` becomes a stream item even though `renderGroupedMessage` returns no bubble content. The previous reading indicator is skipped and the UI can show an empty assistant row until visible text arrives.
  - confidence: 0.88
- **[P2] Keep thinking effort out of reasoning visibility:** `ui/src/ui/views/chat.ts:195-200`
  - body: `/think` controls model effort, while `/reasoning` controls whether reasoning is displayed and falls back to `off`. Falling back from a missing session reasoning level to `props.thinkingLevel` can show reasoning for sessions that never enabled reasoning visibility.
  - confidence: 0.9
- **[P3] Add the required changelog entry:** `ui/src/ui/chat/grouped-render.ts:344-356`
  - body: This changes user-facing Control UI live chat rendering, but the PR file list has no `CHANGELOG.md` update. OpenClaw policy requires a single-line active release entry for user-facing fix changes before merge.
  - confidence: 0.82
