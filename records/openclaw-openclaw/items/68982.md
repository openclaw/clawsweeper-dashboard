## Summary

Keep open. The branch adds useful Control UI rendering support, but the current patch is not an end-to-end fix for live reasoning streams and introduces a reasoning-visibility concern that should block merge until resolved.

## What This Changes

This PR updates Control UI chat item construction, grouped rendering, and grouped-render tests so structured assistant stream payloads with thinking/text blocks can render as live streaming bubbles.

## Best Possible Solution

Land one end-to-end implementation that preserves structured reasoning/text blocks through gateway emission, UI controller state, and rendering while keeping /think effort separate from /reasoning visibility.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: current main emits text-only chat deltas and stores live deltas as extractText(payload.message) in a string chatStream, so a thinking-only live payload cannot reach this PR's object-stream renderer path.

## Solution Assessment

No. Renderer support is useful, but it is not the best standalone fix until structured payloads are plumbed through the gateway and controller state and reasoning visibility continues to default to off unless /reasoning enables it.
