## Summary

Keep open. The branch adds useful renderer-side support, but it still does not solve live reasoning streaming end-to-end against current main and it introduces reasoning-visibility regressions that should block merge.

## What This Changes

This PR updates Control UI chat item construction and streaming rendering to accept structured stream message payloads, with grouped-render coverage for reasoning-only streaming.

## Best Possible Solution

Land one coordinated gateway/controller/UI implementation, likely reconciled with #59086, that preserves structured reasoning/text/tool stream blocks while keeping `/think` effort independent from `/reasoning` visibility.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: current main emits text-only chat deltas and stores live deltas as `extractText(payload.message)` in a string `chatStream`, so a thinking-only live payload cannot reach this PR's object-stream renderer path.

## Solution Assessment

No. Renderer support is useful, but it is not the best standalone fix until structured payloads are carried through gateway emission and Control UI state, hidden reasoning keeps the reading indicator, and reasoning visibility continues to default to `off`.
