## Summary

Keep open. The latest head is still a UI-only renderer change, while current main still emits and stores live chat deltas as text-only payloads; the patch also risks widening reasoning visibility beyond the documented `/reasoning` setting.

## What This Changes

This PR changes Control UI chat item construction and grouped streaming rendering to accept structured stream messages, pass reasoning visibility into streaming bubbles, render reasoning-only bubbles, and add grouped-render tests.

## Best Possible Solution

Land one consolidated end-to-end reasoning-stream implementation across gateway chat delta/final payloads, Control UI stream state, item building/rendering, tests, and changelog while preserving the `/reasoning` visibility contract.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: current main emits text-only chat deltas and `handleChatEvent` stores `extractText(payload.message)` in string `chatStream`, so a thinking-only payload cannot reach this PR's object-rendering path in real live chat.

## Solution Assessment

No. The renderer support is useful, but the fix is incomplete until structured payloads are plumbed through gateway and controller state and reasoning visibility remains separate from `/think` effort.

  - body: `/think` controls model effort, while `/reasoning` controls whether thinking is shown and falls back to `off`. Falling back from a missing session reasoning level to `props.thinkingLevel` can show reasoning for sessions that never enabled reasoning visibility.
  - confidence: 0.9
