## Summary

Keep open. The branch adds useful renderer-side support, but it is not merge-ready because current main still emits and stores live chat streams as text-only payloads, and this PR can widen reasoning visibility by treating `/think` effort as `/reasoning` display permission.

## What This Changes

This PR changes Control UI chat item construction, streaming rendering, and grouped-render tests to accept structured stream message payloads and pass reasoning visibility into live streaming bubbles.

## Best Possible Solution

Land one coordinated gateway/controller/UI fix, likely reconciled with #59086, that carries structured thinking and text blocks end to end while preserving hidden-reasoning loading behavior and keeping `/think` effort separate from `/reasoning` visibility.

## Reproduction Assessment

Yes. Source inspection of current main shows gateway chat events are text-only and Control UI stores live deltas through `extractText(payload.message)` in a string `chatStream`, so a thinking-only live payload cannot reach this PR's object-stream renderer path.

## Solution Assessment

No. Renderer support is useful, but the maintainable fix must also update gateway emission, controller/app stream state, segment handoff, visibility gating, tests, and changelog together.

  - confidence: 0.91

  - body: This is a user-facing Control UI live chat fix, but `CHANGELOG.md` is not updated. OpenClaw policy requires a single-line active-release changelog entry for user-facing fix changes before merge.
