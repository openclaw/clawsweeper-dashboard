## Summary

Keep open. The latest PR still only changes the Control UI renderer/item-building side, while current main still emits live chat deltas/finals as text-only payloads and the Control UI controller still stores deltas in a string stream. The patch therefore does not yet make live reasoning reach the new structured stream renderer, and it also widens reasoning visibility by falling back from `/reasoning` to `/think`.

## What This Changes

The PR updates Control UI chat item building and grouped streaming rendering to accept optional structured stream payloads, pass reasoning visibility into streaming groups, render reasoning-only assistant bubbles, and add grouped-render tests.

## Best Possible Solution

Land one end-to-end implementation that carries structured thinking and text through gateway chat delta/final payloads, Control UI controller state, chat item construction, and grouped rendering, while keeping `/think` effort separate from `/reasoning` visibility and preserving a visible reading indicator when reasoning is hidden.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: current main emits text-only chat deltas/finals and the Control UI stores delta text in `chatStream`, so thinking-only payloads cannot reach the PR's new object-stream branch in live chat.

## Solution Assessment

No. The renderer additions are useful, but the current PR is not the best complete fix because the structured payload needs to be plumbed through gateway and controller state first, with explicit reasoning visibility policy.

## Review Findings

Full review comments:

- **[P1] Plumb structured deltas before rendering:** `ui/src/ui/chat/build-chat-items.ts:15`
  - body: `BuildChatItemsProps.stream` now accepts object payloads, but live chat still cannot provide one: current gateway chat deltas are text-only and `handleChatEvent` stores `extractText(payload.message)` into a string `chatStream`. Thinking-only deltas still never reach this branch in real runs, so the live reasoning fix remains incomplete.
  - confidence: 0.94
- **[P2] Preserve the reading indicator when reasoning is hidden:** `ui/src/ui/chat/build-chat-items.ts:296-301`
  - body: The object-stream path emits a stream item for any structured message, including reasoning-only messages with no visible text. When reasoning is hidden, `renderGroupedMessage` suppresses the thinking block, so the previous reading indicator can be replaced by an empty assistant group until answer text arrives.
  - confidence: 0.88
- **[P2] Keep thinking effort out of reasoning visibility:** `ui/src/ui/views/chat.ts:199`
  - body: `resolveStreamingReasoningLevel` falls back from a missing session reasoning level to `props.thinkingLevel`, but `/think` controls model effort while `/reasoning` controls whether reasoning is shown and falls back to `off`. A session with thinking enabled but no reasoning visibility override can show reasoning unexpectedly.
  - confidence: 0.9
- **[P3] Add the required changelog entry:** `ui/src/ui/chat/grouped-render.ts:339`
  - body: This changes user-visible Control UI live chat behavior, but the diff does not update `CHANGELOG.md`. OpenClaw policy requires user-facing `fix` changes to add a single-line Unreleased entry with contributor credit before merge.
  - confidence: 0.82
