## Summary

Keep open: the optimization is plausible, but the current PR head still has a correctness bug because the cache ignores `manifestRegistry`, which current main uses for both candidate detection and materialization. The supplied proof is also unit/timing output only, not a real gateway/dashboard fanout run.

## What This Changes

The PR adds a `WeakMap` cache around `applyPluginAutoEnable` keyed by config/env object identity and adds cache behavior/timing tests.

## Best Possible Solution

Keep the optimization only after the cache identity includes the explicit or resolved manifest registry, or caching is bypassed when that identity cannot be keyed; add registry-varying regression coverage and redacted gateway/dashboard fanout proof.

## Reproduction Assessment

Yes for the blocking PR regression: source inspection gives a high-confidence path by calling the patched helper twice with the same config/env references but different `manifestRegistry` objects. I did not live-reproduce the full dashboard fanout.

## Solution Assessment

No; caching is a reasonable direction, but config/env-only memoization is not the narrowest correct solution because the helper contract also depends on `manifestRegistry`. The safer fix is to include registry identity or skip caching when it cannot be keyed correctly.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Key the cache by manifest registry too:** `src/config/plugin-auto-enable.apply.ts:52-57`
  - body: `applyPluginAutoEnable` still accepts `manifestRegistry`, and both detection and materialization use it to decide which plugin owns configured channels/providers and how entries are registered. This cache returns the first result for a given `config`/`env` pair even when a later call passes a different registry, so callers can get stale auto-enable output for the wrong plugin set. Include the explicit or resolved registry identity in the key, or bypass caching when that identity is unavailable.
  - confidence: 0.91
