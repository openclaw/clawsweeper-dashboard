## Summary

Keep open. Current main still persists only mtime/size for daily ingestion and skips unchanged daily notes before signal recording, so the reported promotion stall remains possible; #76359 is an open implementation candidate, not a merged fix.

## What This Changes

The issue reports that dreaming deep promotion remains stuck at `Promoted 0` because unchanged daily memory files stop accumulating daily signals across scheduled sweeps.

## Best Possible Solution

Land a narrow checkpoint fix that records the dreaming calendar day per file while preserving same-day dedupe, then close after it is merged and shipped.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: run dreaming once for a daily note, keep `memory/.dreams/daily-ingestion.json`, then run on a later day without changing the note; current main skips the file and daily signals do not grow.

## Solution Assessment

Yes. The #76359 direction is the narrowest maintainable fix because it changes checkpoint semantics for unchanged daily notes across dreaming days without changing promotion thresholds or adding configuration.
