## Summary

Keep open. Current main still skips unchanged daily memory files once `daily-ingestion.json` has the same mtime/size fingerprint, so daily signals do not accumulate across later dreaming days; the linked open PR #76359 is the implementation candidate.

## What This Changes

The issue reports that automatic memory-core dreaming deep promotion remains stuck at `Promoted 0` on v2026.4.29 despite eligible daily-memory candidates.

## Best Possible Solution

Review and land #76359 or an equivalent checkpoint-day fix so unchanged daily notes are re-ingested once per dreaming calendar day while same-day dedupe remains intact, then close this issue after merge.

## Reproduction Assessment

Yes. A high-confidence source path is to ingest a daily note once, keep `daily-ingestion.json`, run dreaming on the next calendar day without changing the file, and observe `collectDailyIngestionBatches` skip the note before `recordShortTermRecalls` can increment daily signals.

## Solution Assessment

Yes. The narrow maintainable solution is to fix daily-ingestion checkpoint semantics; changing thresholds or normalization would miss the current-main cause because normalization already preserves counters.
