## Summary

Keep this open. Current main preserves recall counters during normalization, but the daily dreaming ingestion checkpoint still skips unchanged daily memory files after their first pass, which can leave entries below the default promotion signal gate and matches the reported repeated `Promoted 0` behavior.

## What This Changes

The issue asks to fix a memory-core dreaming promotion regression where scheduled deep sweeps rank candidates but promote zero because short-term signals do not accumulate across daily runs.

## Best Possible Solution

Make dreaming re-ingest eligible unchanged daily memory files at most once per dreaming day, preserve same-day dedupe, and add regression coverage for successive daily sweeps without deleting the checkpoint.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to create a daily memory note, run dreaming on day one, then run it on day two without changing the file or deleting `daily-ingestion.json`; current main skips the unchanged file, so the daily signal does not increase and default deep promotion can remain blocked.

## Solution Assessment

No, the issue's normalization-reset theory is not the best current fix target because current normalization preserves counters. The narrower maintainable fix is the daily ingestion checkpoint semantics plus a regression test that does not clear the checkpoint.
