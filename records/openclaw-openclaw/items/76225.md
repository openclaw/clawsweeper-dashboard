## Summary

Keep open. Current main still skips unchanged daily memory files solely by persisted mtime/size, so later dreaming days cannot accumulate daily signals; open PR #76359 is the closing implementation candidate and should be reviewed or landed before this issue closes.

## What This Changes

The report asks OpenClaw to restore memory-core dreaming promotions by preserving daily recall signal accumulation across scheduled sweeps for unchanged memory notes.

## Best Possible Solution

Land #76359 or an equivalent day-aware daily-ingestion checkpoint fix with persisted-state regression coverage while preserving same-day dedupe.

## Reproduction Assessment

Yes, from source inspection rather than live execution. Run a dreaming sweep that writes `memory/.dreams/daily-ingestion.json`, leave a daily note unchanged, then sweep on a later calendar day; current main skips the file by mtime/size before `recordShortTermRecalls` can increment `dailyCount`.

## Solution Assessment

Yes for the linked PR direction. Making the daily-ingestion checkpoint calendar-day aware is narrower than changing thresholds or promotion policy because it fixes the missing signal accumulation path directly.
