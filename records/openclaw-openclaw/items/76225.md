## Summary

Keep this open because current main still skips unchanged daily memory files after their first daily-ingestion pass, which can prevent promotion signals from accumulating. The open linked PR #76359 is the active fix candidate and should be reviewed or landed before this issue is closed.

## What This Changes

Fix memory-core dreaming so repeated daily sweeps accumulate promotion signals and stop producing `Promoted 0` when eligible daily memory exists.

## Best Possible Solution

Land or revise #76359 so daily-ingestion checkpoints record the last dreaming calendar day per file, preserve same-day dedupe, and add regression coverage for successive daily sweeps without deleting the checkpoint.

## Reproduction Assessment

Yes. Source inspection shows a stable `memory/YYYY-MM-DD.md` file is skipped on later dreaming days because the checkpoint only compares `mtimeMs` and `size`, so daily signals do not accumulate under the default promotion gates.

## Solution Assessment

Yes, the linked PR direction is the narrow maintainable fix. The issue's normalization-reset theory is not supported by current main, but per-dreaming-day daily-ingestion checkpoint semantics directly target the remaining failure path.
