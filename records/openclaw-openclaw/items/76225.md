## Summary

Keep open. Current main still source-reproduces the daily-ingestion checkpoint bug, and the linked open PR is the implementation candidate rather than proof that the issue is fixed.

## What This Changes

The issue reports that memory-core dreaming promotion remains stuck at zero promotions because daily recall signals do not accumulate across scheduled dreaming cycles.

## Best Possible Solution

Review and land PR #76359, or an equivalent memory-core fix that records the last dreaming calendar day per daily note while preserving same-day dedupe.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: ingest a daily note once, keep `memory/.dreams/daily-ingestion.json`, then run dreaming on a later calendar day without changing the note; current main skips the file and does not add another daily signal.

## Solution Assessment

Yes. The linked PR direction targets the current defect narrowly by making the checkpoint day-aware, which is safer than lowering thresholds or treating normalization as the current reset point.
