## Summary

Keep open. Current main still skips unchanged daily memory files using only mtime/size checkpoint state before recording another daily signal, and the linked open PR is the active implementation candidate rather than proof of a shipped fix.

## What This Changes

The issue asks to fix memory-core dreaming promotion so stable daily memory files keep adding daily signals across calendar days and deep promotion no longer stays at Promoted 0.

## Best Possible Solution

Review and land PR #76359, or an equivalent memory-core fix that records the last dreaming calendar day per daily note while preserving same-day dedupe.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: run dreaming once to write `memory/.dreams/daily-ingestion.json`, then run on a later calendar day without modifying the daily note; current main sees the same mtime/size and emits no daily batch.

## Solution Assessment

Yes for the linked PR direction. Adding day-scoped checkpoint state targets the broken accumulation path more directly than lowering thresholds or treating normalization as the current reset point.
