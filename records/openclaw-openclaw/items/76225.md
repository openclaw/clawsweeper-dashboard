## Summary

Keep open. Current main still has the persisted daily-ingestion checkpoint defect: unchanged daily notes are skipped by mtime/size alone on later dreaming days, so daily signals cannot accumulate; open PR #76359 is the active closing implementation candidate and this issue should remain open until it lands.

## What This Changes

Fix memory-core dreaming promotion so stable daily memory notes keep accumulating cross-day promotion signals instead of producing repeated `Promoted 0` runs.

## Best Possible Solution

Review and land #76359, or an equivalent memory-core patch that records the last dreaming calendar day per daily note while preserving same-day dedupe and adding persisted-checkpoint regression coverage.

## Reproduction Assessment

Yes, from source inspection. Ingest a daily note once, keep `memory/.dreams/daily-ingestion.json`, then run dreaming on a later calendar day without changing the note; current main skips the file and records no additional daily signal.

## Solution Assessment

Yes for the linked PR direction. Making the checkpoint calendar-day aware is narrower than lowering thresholds or changing promotion policy, and it targets the remaining defect without broad product changes.
