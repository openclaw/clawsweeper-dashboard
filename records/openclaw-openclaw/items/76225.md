## Summary

Keep this open. Current main still skips unchanged daily memory files once `daily-ingestion.json` records the same `mtimeMs` and `size`, so daily dreaming signals can stop accumulating; the linked open PR #76359 is the implementation candidate rather than proof that the issue is fixed.

## What This Changes

The issue reports that dreaming deep promotion remains stuck at `Promoted 0` because short-term memory signals do not accumulate across daily dreaming runs.

## Best Possible Solution

Land #76359 or an equivalent memory-core checkpoint fix with regression coverage for next-day re-ingestion while preserving same-day dedupe.

## Reproduction Assessment

Yes. By source inspection, create a daily memory note, run dreaming once to write `daily-ingestion.json`, then run on a later dreaming day without changing file `mtimeMs` or `size`; current main skips the file, so `dailyCount` does not grow and the default deep gate can keep promotion at zero.

## Solution Assessment

No, the issue should not be closed yet. The reporter's normalization-reset theory is not the current source-level cause, but the linked daily-ingestion checkpoint fix is the narrow maintainable direction.
