## Summary

Keep open. Current main still skips unchanged daily memory files solely by mtime/size checkpoint, and the linked open PR is an implementation candidate rather than proof that the issue is fixed.

## What This Changes

The issue reports that daily dreaming promotion remains stuck at zero because unchanged daily notes stop accumulating promotion signals across calendar days.

## Best Possible Solution

Review and land #76359, or an equivalent memory-core fix that records calendar-day ingestion state while preserving same-day dedupe.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: ingest a daily note once, keep memory/.dreams/daily-ingestion.json, then run dreaming on a later calendar day without changing the note; current main skips the file and does not add another daily signal.

## Solution Assessment

Yes. The linked PR direction is the narrowest maintainable fix because it changes checkpoint bookkeeping, not promotion thresholds or broader dreaming policy.
