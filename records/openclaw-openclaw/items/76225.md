## Summary

Keep open. Current main still skips unchanged daily memory files using only mtime and size, so later dreaming days cannot add another daily signal; open PR #76359 is the active closing fix and should not be bypassed before merge.

## What This Changes

The issue asks for memory-core dreaming promotion to accumulate daily/recall signals across scheduled runs so high-signal entries stop stalling at Promoted 0.

## Best Possible Solution

Land #76359 or an equivalent memory-core fix that records the last dreaming calendar day per daily note while preserving same-day dedupe and adding persisted-checkpoint regression coverage.

## Reproduction Assessment

Yes, from source inspection. Process a daily note once, keep `memory/.dreams/daily-ingestion.json`, then run dreaming on a later calendar day without changing the note; current main skips the file and does not add another daily signal.

## Solution Assessment

Yes for the linked PR direction. Making the daily-ingestion checkpoint day-aware is narrower than lowering thresholds or changing promotion policy, and it targets the current defect rather than the older normalization-reset hypothesis.
