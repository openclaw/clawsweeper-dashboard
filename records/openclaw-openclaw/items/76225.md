## Summary

Keep open. Current main still skips unchanged daily memory files using only the persisted mtime/size checkpoint, and open PR #76359 is the active closing implementation candidate rather than a reason to close this issue before merge.

## What This Changes

The issue asks for dreaming promotion to accumulate daily and recall signals across scheduled cycles again so high-scoring memory candidates can pass the deep promotion gates.

## Best Possible Solution

Land a day-aware daily-ingestion checkpoint fix with persisted-state regression coverage, preserving same-day dedupe and the existing promotion thresholds.

## Reproduction Assessment

Yes. From source inspection, ingest a daily note once, keep `memory/.dreams/daily-ingestion.json`, then run dreaming on a later calendar day without editing the note; current main sees the same mtime/size and emits no new daily signal batch.

## Solution Assessment

Yes for the linked PR direction. Making the checkpoint dreaming-day-aware is narrower than changing thresholds or normalization behavior, and it targets the source-backed defect that still blocks signal accumulation.
