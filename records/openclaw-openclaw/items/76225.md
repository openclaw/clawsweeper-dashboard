## Summary

Keep this open. Current main still has the daily-ingestion checkpoint behavior that can skip unchanged daily memory files across later dreaming days, and the linked open PR is the active fix candidate rather than proof that the issue is resolved.

## What This Changes

The issue reports a memory-core dreaming promotion regression where daily recall signal stops accumulating and repeated deep sweeps rank candidates but promote zero entries.

## Best Possible Solution

Review and land or replace the linked daily-ingestion checkpoint fix so unchanged daily memory files are re-ingested once per dreaming calendar day while preserving same-day dedupe.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to run dreaming for a daily note, keep `memory/.dreams/daily-ingestion.json`, run a later dreaming day without changing the file, and observe current main skip that file before daily signal can increase.

## Solution Assessment

Yes. The narrow maintainable solution is to fix daily-ingestion checkpoint semantics, not to change promotion thresholds or reopen the older normalization-reset theory.
