## Summary

Keep open. Current main still skips unchanged daily memory files using only `mtimeMs` and `size`, so later dreaming days cannot accumulate daily signals; linked PR #76359 is the active implementation candidate and should be reviewed or landed before this issue closes.

## What This Changes

The issue reports a regression where memory-core dreaming promotion stays at `Promoted 0` because daily recall signals do not accumulate across dreaming cycles.

## Best Possible Solution

Review and land #76359, or an equivalent memory-core fix that makes the daily-ingestion checkpoint day-aware while preserving same-day dedupe and adding persisted-checkpoint regression coverage.

## Reproduction Assessment

Yes, from source inspection. Ingest a daily note once so `memory/.dreams/daily-ingestion.json` records unchanged `mtimeMs` and `size`, then run dreaming on a later calendar day without changing the note; current main skips the file and cannot add another daily signal.

## Solution Assessment

Yes for the linked PR direction. Making the daily-ingestion checkpoint calendar-day-aware is narrower than changing promotion thresholds or policy, and it targets the current skipped-ingestion defect.
