## Summary

Keep open. Current main still has the daily-ingestion checkpoint defect: unchanged daily files are skipped solely by mtime/size, so later dreaming days do not add another daily signal; open PR #76359 is the implementation candidate and should not close this issue before merge.

## What This Changes

The issue reports that memory-core dreaming promotion remains stuck at zero because daily signals do not accumulate across daily dreaming runs.

## Best Possible Solution

Review and land #76359, or an equivalent memory-core fix that records the last dreaming calendar day per daily note while preserving same-day dedupe and adding persisted-checkpoint regression coverage.

## Reproduction Assessment

Yes from source inspection. Ingest a daily note once, keep `memory/.dreams/daily-ingestion.json`, then run dreaming on a later calendar day without changing the note; current main skips the file and does not add another daily signal.

## Solution Assessment

Yes for the linked PR direction. Making the daily-ingestion checkpoint day-aware is narrower than lowering thresholds or changing promotion policy, and it targets the current defect rather than the older normalization-reset hypothesis.
