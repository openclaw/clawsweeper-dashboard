## Summary

Keep open. Current main still has the daily-ingestion checkpoint defect: unchanged daily memory files are skipped by mtime/size alone on later dreaming days, and open PR #76359 is the active closing implementation candidate.

## What This Changes

The issue asks to fix a memory-core dreaming regression where daily promotion stays at "Promoted 0" because short-term signals do not accumulate across daily runs.

## Best Possible Solution

Review and land #76359, or an equivalent memory-core fix that records the last dreaming calendar day per daily note while preserving same-day dedupe and adding persisted-checkpoint coverage.

## Reproduction Assessment

Yes from source inspection. Ingest a daily note once, keep memory/.dreams/daily-ingestion.json, then run dreaming on a later calendar day without changing the note; current main skips the file and does not add another daily signal.

## Solution Assessment

Yes for the linked PR direction. Making the daily-ingestion checkpoint day-aware is narrower than changing thresholds or promotion policy, and it targets the current defect rather than the older normalization-reset hypothesis.
