## Summary

Keep open. Current main still skips unchanged daily memory files after the persisted daily-ingestion checkpoint matches mtime and size, and the linked open PR is the implementation candidate rather than proof that this issue is fixed.

## What This Changes

The issue reports that memory-core dreaming promotion keeps producing zero promotions because stable daily memory files do not keep accruing promotion signals across scheduled dreaming days.

## Best Possible Solution

Review and land #76359, or an equivalent memory-core fix that records the last dreaming calendar day per daily note while preserving same-day dedupe.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: ingest a daily note once, keep `memory/.dreams/daily-ingestion.json`, then run dreaming on a later calendar day without changing the note; current main skips the file and does not add another daily signal.

## Solution Assessment

Yes for the linked PR direction. Adding day-scoped checkpoint state targets the current defect more narrowly than lowering thresholds or treating normalization as the current reset point.
