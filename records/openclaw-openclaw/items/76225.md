## Summary

Keep open. Current main still skips unchanged daily memory files after the daily-ingestion checkpoint has recorded matching mtime/size, and the open linked PR #76359 is the implementation candidate rather than proof that this issue is fixed.

## What This Changes

The issue reports that dreaming deep promotion keeps returning `Promoted 0` on v2026.4.29 because promotion signals stop accumulating across scheduled sweeps.

## Best Possible Solution

Review and land #76359 if its checkpoint schema and regression coverage preserve same-day dedupe while re-ingesting unchanged daily notes once per dreaming day.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: run dreaming once for a daily note, keep `memory/.dreams/daily-ingestion.json`, then run on a later day without changing the note; current main skips the file and daily signals do not increase.

## Solution Assessment

Yes, the linked PR direction is the narrowest maintainable fix. Changing daily-ingestion checkpoint semantics targets the remaining current-main defect, while normalization already preserves counters and threshold tuning is a separate product request.
