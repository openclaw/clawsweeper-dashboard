## Summary

Keep open. Current main still skips unchanged daily notes using only mtime/size checkpoint state, so later dreaming days can fail to add daily promotion signals; open PR #76359 is the active closing implementation candidate and should be reviewed or landed before this issue is closed.

## What This Changes

The issue reports a memory-core dreaming regression where unchanged daily notes stop accumulating promotion signals, leaving deep promotion at Promoted 0 after v2026.4.29.

## Best Possible Solution

Review and land #76359, or an equivalent day-aware daily-ingestion checkpoint fix with persisted-checkpoint regression coverage, then close this issue after merge.

## Reproduction Assessment

Yes, from source inspection. Run a dreaming sweep with a daily note, keep memory/.dreams/daily-ingestion.json and the daily file unchanged, then run on a later calendar day; current main skips the file before recordShortTermRecalls can add that day's daily signal.

## Solution Assessment

Yes for the linked PR direction. Making the daily-ingestion checkpoint day-aware is narrower than lowering promotion thresholds or changing promotion policy, and it preserves same-day dedupe while fixing cross-day accumulation.
