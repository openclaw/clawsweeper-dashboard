## Summary

Keep open. Current main still skips unchanged daily notes solely by mtime/size, so cross-day daily signals do not accumulate; open PR #76359 is the active closing implementation candidate and should be reviewed or landed before this issue closes.

## What This Changes

Fix memory-core dreaming promotion so daily signals accumulate across scheduled sweeps and high-signal short-term entries promote to MEMORY.md instead of repeatedly reporting Promoted 0.

## Best Possible Solution

Land #76359 or an equivalent memory-core fix that records the last dreaming calendar day per daily note while preserving same-day dedupe and adding persisted-checkpoint regression coverage.

## Reproduction Assessment

Yes, from source inspection. With an existing memory/.dreams/daily-ingestion.json entry for a daily note, a later dreaming day with the same file mtime/size hits the unchanged branch and emits no batch, so dailyCount cannot grow toward the promotion gates.

## Solution Assessment

Yes for the linked PR direction. Making the daily-ingestion checkpoint calendar-day-aware is narrower than changing thresholds or promotion policy, and it targets the remaining current-main defect.
