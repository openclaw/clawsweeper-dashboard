## Summary

Keep this open. Current main still skips unchanged daily memory files after their first `daily-ingestion.json` checkpoint, and the open closing PR #76359 is the right implementation candidate rather than a reason to close before merge.

## What This Changes

The issue requests fixing memory-core dreaming so unchanged daily memory notes continue accruing daily promotion signals across scheduled sweeps instead of repeatedly reporting zero promotions.

## Best Possible Solution

Review and land #76359, or an equivalent memory-core fix that records the last dreaming calendar day per daily note while preserving same-day dedupe.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: ingest a daily note once, leave it unchanged, then run dreaming on the next calendar day with the persisted daily-ingestion checkpoint; current main skips the file and does not add another daily signal.

## Solution Assessment

Yes for the linked PR direction. Adding day-scoped checkpoint state is narrower and more maintainable than lowering thresholds or treating normalization as the current reset point.
