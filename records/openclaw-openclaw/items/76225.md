## Summary

Keep open. Current main still has the persisted daily-ingestion checkpoint path that skips unchanged daily memory files across later dreaming days, and open PR #76359 is the active implementation candidate rather than proof of a shipped fix.

## What This Changes

The report asks to fix memory-core dreaming promotion so unchanged daily memory notes keep accumulating daily promotion signals instead of producing repeated `Promoted 0` runs.

## Best Possible Solution

Review and land #76359, or an equivalent memory-core change that records the last dreaming calendar day per daily note while preserving same-day dedupe.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: ingest a daily note once, keep `memory/.dreams/daily-ingestion.json`, then run dreaming on a later calendar day without changing the note; current main skips the file and records no new daily signal.

## Solution Assessment

Yes for the linked PR direction. Adding day-scoped checkpoint state targets the current defect more narrowly than lowering thresholds or changing promotion policy.
