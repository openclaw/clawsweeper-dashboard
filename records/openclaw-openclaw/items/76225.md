## Summary

Keep this open. Current main no longer supports the older normalization-reset theory, but it still skips unchanged daily memory files after the daily-ingestion checkpoint is written, which can prevent daily signals from accumulating and matches the repeated `Promoted 0` report.

## What This Changes

The issue reports that dreaming promotion on v2026.4.29 keeps ranking candidates but promotes zero because accumulated recall signals do not cross the default promotion gate.

## Best Possible Solution

Make daily dreaming ingestion track the last ingestion day per daily file, re-read eligible unchanged files once per dreaming day, and cover the normal persisted-checkpoint two-day sweep in regression tests.

## Reproduction Assessment

Yes. Create a daily note, run light/dreaming once, keep `memory/.dreams/daily-ingestion.json`, then run on the next day without touching the note; current main skips the unchanged file, so `dailyCount` does not increase.

## Solution Assessment

No, the issue's normalization-reset theory is not the best current fix target because normalization now preserves counters. The narrow maintainable fix is the daily ingestion checkpoint semantics plus regression coverage that does not delete the checkpoint.
