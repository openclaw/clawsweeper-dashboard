## Summary

Keep open: this is a focused fix PR for an open webchat session bug, and current main does not yet implement it. The Gateway path and after-fix live output support the approach, but the docs still describe the old unconditional dashboard-session behavior.

## What This Changes

The PR adds a `session.dmScope: "main"` branch in Gateway `sessions.create` that resets the canonical parent session in place for bare webchat `/new`, with regression coverage and a changelog entry.

## Best Possible Solution

Land the scoped Gateway reset branch after documenting the Control UI `/new` `dmScope: "main"` exception while preserving dashboard-child creation for other create shapes.

## Reproduction Assessment

Yes. Current main is source-reproducible: the Control UI can call `sessions.create` with `parentSessionKey` and `emitCommandHooks`, and the server then creates a `dashboard:` child when no requested key is supplied; the PR body also supplies after-fix live Gateway output.

## Solution Assessment

Yes, with a docs follow-up. Reusing `performGatewaySessionReset` only for the bare canonical-main `dmScope: "main"` case is narrower than adding API surface and preserves existing dashboard-child behavior elsewhere.
