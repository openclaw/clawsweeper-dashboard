## Summary

Keep open: the PR targets a source-reproducible gateway/webchat bug and the code direction looks narrow, but this external PR still needs real behavior proof before merge and should update the user-facing Control UI slash-command docs for the new `dmScope: "main"` exception.

## What This Changes

The PR adds a `session.dmScope: "main"` short-circuit in `sessions.create` so bare webchat `/new` resets the parent main session in place, plus a regression test and changelog entry.

## Best Possible Solution

Land the scoped gateway reset path through `performGatewaySessionReset`, keep the existing dashboard-child behavior for other create shapes, update the Control UI/slash-command docs, and require redacted real behavior proof before merge.

## Reproduction Assessment

Yes, source-reproducible. Current main's Control UI passes `parentSessionKey` plus `emitCommandHooks` into `sessions.create`, and the server currently falls through to `buildDashboardSessionKey(agentId)` when no explicit key is requested.

## Solution Assessment

Yes, with follow-up. Reusing `performGatewaySessionReset` for only the bare canonical-main `dmScope: "main"` case is narrower than adding new API surface and preserves the existing dashboard-child path for other calls.
