## Summary

Keep open: current main still carries `message(action=send)` text through the WebChat internal source-reply path and into markdown rendering without any handling for raw citation control markers, and I found no canonical issue or merged fix that covers this exact formatting leak.

## What This Changes

The issue reports that WebChat shows raw citation tokens from message-tool-delivered assistant replies instead of rendering or sanitizing them.

## Best Possible Solution

Strip or degrade unsupported citation control markers in the WebChat/message-tool visible text path now, and only render them as links later if the delivery payload carries resolvable citation metadata.

## Reproduction Assessment

Yes at source level: current main preserves the raw message-tool text through `sourceReply` and renders it as markdown without a citation-marker transform. I did not run a live WebChat session during this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is a display/source-reply sanitizer for unsupported internal citation markers, with tests around the WebChat message-tool path. Rendering clickable citations would require additional source metadata that the raw marker alone does not provide.
