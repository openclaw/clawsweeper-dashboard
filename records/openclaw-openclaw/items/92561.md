## Label Justifications

- `P1`: The report describes a current shipped prompt/context security boundary issue that can affect real agent turns on shared hosts.
- `impact:security`: The issue is about untrusted files outside the workspace influencing model-visible instructions across a trust boundary.
- `impact:session-state`: The injected ancestor files become session prompt context and can alter agent behavior for the session.

## Summary

Keep open: current main and latest release still let `loadProjectContextFiles` scan ancestor `AGENTS.md`/`CLAUDE.md` files up to the filesystem root, and the linked fix PRs are all closed unmerged.

## What This Changes

This issue reports that automatic project context loading should be bounded to the workspace or project trust boundary so files outside the workspace cannot be injected into agent prompt context.

## Best Possible Solution

Patch session context discovery to use an explicit workspace or project-root boundary, keep legitimate nested workspace files, reject outside-ancestor files, and add focused regression tests for both paths.

## Reproduction Assessment

Yes at source level: current main has a direct cwd-to-root loop that loads matching files before stopping at `/`. I did not create files in the checkout or run a filesystem repro because this review was read-only.

## Solution Assessment

Unclear for the exact implementation: bounding discovery is the right direction, but home-directory fixes are too broad for the reported workspace-boundary problem. Maintainers should choose the explicit trust boundary before landing a security fix.
