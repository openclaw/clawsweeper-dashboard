## Summary

Keep this issue open: current main and the latest release still have the unbounded ancestor walk, and the only open closing PR has already been flagged as a cwd-only fix that can drop trusted workspace-root context.

## What This Changes

The issue asks OpenClaw to stop auto-loading `AGENTS.md` or `CLAUDE.md` from ancestor directories outside the trusted workspace boundary.

## Best Possible Solution

Land a corrected fix that passes or derives the trusted workspace boundary, walks only within that boundary, preserves root-to-cwd ordering, and adds focused regression coverage.
