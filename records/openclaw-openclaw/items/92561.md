## Summary

Keep open: the reported unbounded ancestor context-file loading is still present on current `main` and in the latest release, and all linked fix PR attempts are closed unmerged. The issue is security-sensitive, so the remaining decision is the exact trusted boundary, not whether the bug exists.

## What This Changes

The issue requests bounding automatic AGENTS.md/CLAUDE.md ancestor loading so files outside the workspace boundary cannot be injected into an agent prompt.

## Best Possible Solution

Keep this issue open and fix the loader around an explicit trusted workspace or project boundary, with regression coverage proving parent/outside-boundary AGENTS.md and CLAUDE.md files are not injected.
