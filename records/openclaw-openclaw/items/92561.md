## Label Justifications

- `P1`: The issue reports untrusted prompt-context injection in an agent/session workflow with current-source support.
- `impact:security`: Files outside the trusted workspace can currently influence system-prompt context in the affected loader path.
- `impact:session-state`: The injected AGENTS.md/CLAUDE.md content changes session prompt state and agent behavior.

## Summary

Keep open: current main and the latest release still walk `loadProjectContextFiles()` to the filesystem root, and all related fix PRs are closed unmerged. This remains a source-reproducible security-boundary issue needing a workspace-root-bounded fix.

## What This Changes

The issue asks OpenClaw to prevent AGENTS.md/CLAUDE.md files outside the trusted workspace from being auto-injected into agent context.

## Best Possible Solution

Bound this resource-loader path to the resolved workspace root, preferably by reusing the existing workspace bootstrap guard model or passing an explicit workspace boundary into DefaultResourceLoader, with traversal/symlink/hardlink regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows `loadProjectContextFiles()` reads AGENTS.md/CLAUDE.md from each ancestor until `/`, and `DefaultResourceLoader` can feed those files into the session system prompt.

## Solution Assessment

Yes on direction, but not as a home-directory-only fix. The maintainable fix should use the configured workspace root and existing boundary-safe file-read patterns rather than accepting parent directories outside the workspace.
