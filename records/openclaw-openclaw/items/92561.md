## Label Justifications

- `P1`: The report describes a security-boundary failure that can inject untrusted ancestor files into agent prompt context on shared hosts.
- `impact:security`: The affected behavior automatically trusts instruction files outside the intended workspace boundary.
- `impact:session-state`: Loaded ancestor files become prompt context and can change the session’s effective instructions before the user turn runs.

## Summary

Keep open: current `main` and latest release still let the regular session resource loader walk from `cwd` to filesystem root, and the linked fix PRs were closed unmerged after home-directory caps were found to leave the workspace-boundary issue unresolved.

## What This Changes

The issue requests bounding automatic `AGENTS.md`/`CLAUDE.md` ancestor loading so files outside the workspace boundary cannot be injected into agent context.

## Best Possible Solution

Implement a maintainer-reviewed workspace or project-root boundary for regular session context discovery, preserve intended in-workspace context loading, exclude parent/home/system files, and add focused regression coverage.

## Reproduction Assessment

Yes. Source inspection shows `loadProjectContextFiles()` starts at `cwd`, calls `loadContextFileFromDir()` for each ancestor, and stops only at filesystem root; no live filesystem repro was run because this review is read-only.

## Solution Assessment

Mostly yes for the issue’s workspace-boundary goal, but the previous home-directory PR variants are not the best fix. The safer solution is to pass or derive the actual trusted workspace/project root and test parent/home/system exclusions.
