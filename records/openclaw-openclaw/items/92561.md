## Summary

Keep this open: current main and the latest release still have the reported unbounded ancestor walk in `loadProjectContextFiles`, and the related fix PRs found for this issue were closed unmerged.

## What This Changes

The issue reports that automatic AGENTS.md/CLAUDE.md loading can walk above the workspace and inject instructions from untrusted ancestor directories.

## Best Possible Solution

Thread an explicit workspace boundary into the session resource-loader path, stop AGENTS.md/CLAUDE.md discovery at that boundary, and cover outside-workspace ancestors with focused regression tests while keeping broader directory-relative loading as separate product work.

## Reproduction Assessment

Yes. Source inspection shows the current loader walks from `cwd` to filesystem root and feeds the discovered files into the session system prompt; I did not run a live exploit because this review is read-only.

## Solution Assessment

Yes in direction, but it needs maintainer-owned boundary design. The workspace-root boundary requested here is safer than the closed home-directory variants because a shared home directory remains outside the intended workspace trust boundary.
