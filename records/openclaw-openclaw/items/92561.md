## Label Justifications

- `impact:security`: The affected behavior can load untrusted instruction files from outside the workspace into system prompt context.
- `impact:session-state`: The injected ancestor instructions alter the session prompt/context state that the agent uses for subsequent behavior.

## Summary

Keep this open: current main and the latest release still let `loadProjectContextFiles()` walk from `cwd` to the filesystem root, and the related fix PRs were closed unmerged after review found the home-directory boundary incomplete.

## What This Changes

The issue requests bounding automatic `AGENTS.md` / `CLAUDE.md` ancestor loading so files outside the workspace cannot enter agent prompt context.

## Best Possible Solution

Pass or derive the trusted workspace/project root for this session context path, stop ancestor discovery at that boundary, use guarded workspace reads where possible, and add regression coverage for parent/home files above the workspace.

## Reproduction Assessment

Yes, from source inspection: the loader checks every ancestor directory and only stops at filesystem root, then the result is passed into prompt construction. I did not run a live session because this review is read-only.

## Solution Assessment

No current fix is present on main, and the closed home-boundary PRs are not the best solution because they still trust `~/AGENTS.md` above a workspace under home. The maintainable fix is a workspace/project-root boundary with focused regression tests.
