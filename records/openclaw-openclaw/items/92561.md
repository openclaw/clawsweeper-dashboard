## Label Justifications

- `P1`: Current main and the latest release still expose a prompt-context injection path through normal session resource loading.
- `impact:security`: The issue is about untrusted ancestor files crossing a workspace trust boundary into system prompt context.
- `impact:session-state`: The affected files become session bootstrap context and can change agent behavior across turns.

## Summary

Keep open: current main and the latest release still load AGENTS.md/CLAUDE.md from every ancestor of the session cwd up to filesystem root, and all issue-linked fix PRs are closed unmerged. The report is a valid core session security-boundary bug, not a stale or duplicate cleanup candidate.

## What This Changes

The issue asks OpenClaw to prevent automatic AGENTS.md/CLAUDE.md discovery from loading files outside the trusted workspace boundary into agent prompt context.

## Best Possible Solution

Define the trusted workspace/root contract for normal session resource loading, then bound loadProjectContextFiles to that boundary and add regression coverage proving outside-boundary ancestor files are excluded while in-boundary context files still load in deterministic order.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: create a normal session with cwd below an ancestor containing AGENTS.md/CLAUDE.md, and the loader will read that ancestor before stopping only at filesystem root; I did not run a live temp-file repro because this review is read-only.

## Solution Assessment

Yes on the fix location: the resource loader is the right choke point because it is the component that reads context files before AgentSession injects them into buildSystemPrompt. The exact boundary parameter still needs maintainer/security decision, and the home-directory-only approach in closed PR attempts is likely too broad for the reported workspace boundary.
