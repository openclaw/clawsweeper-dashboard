## Summary

Keep open. Current `main` and the latest release still have the ancestor walk that stops only at filesystem root, and the open linked fix at https://github.com/openclaw/openclaw/pull/92787 has not merged yet.

## What This Changes

The issue reports that `loadProjectContextFiles()` can load ancestor `AGENTS.md` / `CLAUDE.md` files above the workspace boundary, allowing untrusted prompt-context injection from outside the workspace.

## Best Possible Solution

Review and land a fix through https://github.com/openclaw/openclaw/pull/92787 or a replacement PR that bounds context-file discovery to the intended workspace trust boundary and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: with a `cwd` under a parent containing `AGENTS.md`, the loop reads each ancestor and stops only at filesystem root.

## Solution Assessment

Yes on direction, with one boundary caveat. Bounding the walk is the narrow maintainable fix, but the final PR should either use the configured workspace root or explicitly justify a different boundary such as the home directory.
