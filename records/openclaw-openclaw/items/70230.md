## Summary

Keep open. Current main still lacks the requested per-connection health telemetry, and the open implementation PR is the right place to review or resolve the new public API surface before closing this issue.

## What This Changes

The issue requests connection-level gateway reachability telemetry, such as `connected`, `rttMs`, and `lastHeartbeatAt`, on `health` or an equivalent lightweight ping surface.

## Best Possible Solution

Keep this issue paired with the open implementation PR and, if maintainers accept the API, land an additive per-client `health.connection` overlay that reuses the authenticated WebSocket ping owner without mutating cached service health.

## Reproduction Assessment

Yes for source-level verification: current main lacks the requested fields in the health type, handler response, snapshot builder, method list, tests, and docs. This is a feature request rather than a broken existing behavior reproduction.

## Solution Assessment

Yes, keeping it open behind https://github.com/openclaw/openclaw/pull/74783 is the best current handling. The maintainable implementation direction is an additive per-client health overlay coordinated with the existing authenticated WebSocket ping loop.
