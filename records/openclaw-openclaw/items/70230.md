## Summary

Keep open. Current main still lacks the requested per-connection gateway health telemetry, and open PR #74783 is already linked as the implementation candidate, so this issue should remain open until that PR lands or maintainers choose a different protocol shape.

## What This Changes

The issue requests connection-level gateway health fields such as `connected`, `rttMs`, and `lastHeartbeatAt` for UI reachability indicators.

## Best Possible Solution

Review and land or revise #74783 as the focused implementation: add a documented per-client `health.connection` overlay, keep the cached service snapshot global, and coordinate the WebSocket ping owner with #56668.

## Reproduction Assessment

Yes for the missing behavior: current-main source and docs inspection show no `health.connection` fields and no `gateway.ping` method. This is a feature request, so a runtime failure reproduction is not applicable.

## Solution Assessment

Unclear until #74783 is reviewed, but the per-response `health.connection` overlay appears to be the narrowest compatible direction because the authenticated health handler already has client context while the cached service snapshot can remain shared.
