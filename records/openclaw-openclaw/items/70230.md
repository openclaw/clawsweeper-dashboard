## Summary

Keep open. Current main still lacks the requested connection telemetry, and open PR #74783 is already linked as the implementation candidate, so this issue should stay open until that PR lands or maintainers choose a different protocol shape.

## What This Changes

Add live connection-level gateway reachability telemetry, such as `connected`, `rttMs`, and `lastHeartbeatAt`, to the existing `health` RPC or an equivalent lightweight ping surface.

## Best Possible Solution

Review #74783 as the focused implementation candidate. The desired end state is an additive, documented per-client connection telemetry surface for authenticated gateway clients, with tests proving cached snapshot isolation, fresh and cached health responses, stale pongs, socket close behavior, and multi-client separation.

## Reproduction Assessment

Yes for the current missing behavior: source inspection and targeted search on current main show no `health.connection` fields and no `gateway.ping` method. This is a feature request rather than a runtime bug reproduction.

## Solution Assessment

Unclear until #74783 is reviewed, but the proposed per-response `health.connection` overlay is the narrowest compatible direction because the authenticated health handler already has client context and a cached global snapshot to preserve.
