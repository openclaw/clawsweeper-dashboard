## Summary

Keep open. The PR is a useful fix for the linked Azure compaction fallback bug, but the current head still misses the overflow recovery runtime-context path, so one important fallback-override invariant can regress before merge.

## What This Changes

The PR wraps implicit embedded compaction in the session model fallback chain, threads per-run fallback overrides through agent execution paths, preserves compaction failure metadata, and updates docs, changelog, and focused tests.

## Best Possible Solution

Update the branch so every implicit compaction runtime-context path, especially overflow recovery in run.ts, carries the effective modelFallbacksOverride. Add a runner-level overflow regression that proves a user model switch override such as [] is preserved, while keeping explicit agents.defaults.compaction.model exact and fallback selection ephemeral.

## Reproduction Assessment

Yes. A source-level reproduction is clear: resolveEffectiveModelFallbacks returns [] for a user model switch, but overflow recovery currently builds runtimeContext without that field, so compactEmbeddedPiSessionDirect can fall back to configured defaults instead of preserving the user override.

## Solution Assessment

No. The fallback wrapper, metadata preservation, docs, and focused direct tests are the right direction, but the current PR is incomplete until the overflow runner path mirrors the timeout runtime-context field and covers that behavior in a runner test.
