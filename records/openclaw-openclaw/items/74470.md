## Summary

Keep open. The latest head fixes the serialized failure metadata problem, but the fallback override is still only wired into timeout compaction; the separate overflow-recovery context, which is the path described by #64960, still omits the per-run override. The protected `maintainer` label also means this PR should stay open for explicit maintainer handling.

## What This Changes

The PR wraps implicit embedded compaction in the session model fallback chain, threads fallback override data through agent run parameters, preserves compaction failure metadata, updates compaction docs/changelog, and adds focused regression tests.

## Best Possible Solution

Update the branch so both timeout and overflow recovery pass the effective `modelFallbacksOverride` into embedded compaction, and add an overflow-runner regression that proves a user model switch override such as `[]` is preserved instead of falling back to configured defaults.

## Reproduction Assessment

Yes. A source-level reproduction is clear: with a user model switch, `resolveEffectiveModelFallbacks` returns `[]`, but overflow recovery still builds a runtime context without that field, so compaction can resolve fallback defaults as if no override existed.

## Solution Assessment

No. The fallback wrapper and metadata preservation are the right direction, but the current patch misses the overflow runtime-context path; the narrower maintainable fix is to mirror the timeout field into the overflow context and test that runner path.
