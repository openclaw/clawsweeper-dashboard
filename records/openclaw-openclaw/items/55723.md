## Summary

Keep open: current main and the latest release still derive ACP heartbeat eligibility from the parent session key, while this PR carries the remaining narrow fix to honor the effective requester override. The branch is not merge-ready because it is conflicted against main and still has only mock/unit proof, not real behavior proof.

## What This Changes

The branch forwards `requesterAgentIdOverride` through ACP `sessions_spawn`, applies it to ACP requester heartbeat/requester-state resolution, and adds focused regression coverage.

## Best Possible Solution

Rebase on current main and update ACP heartbeat eligibility to use the effective requester agent id already derived from `requesterAgentIdOverride`, with regression coverage and real behavior proof.

## Reproduction Assessment

Yes from source inspection: current main passes the override into ACP spawn but then calculates `heartbeatEnabled` from the parent session key. A subagent parent key for one agent plus `requesterAgentIdOverride` for another agent exercises the mismatch.

## Solution Assessment

Yes in direction: the narrow maintainable fix is to feed the effective requester agent into heartbeat eligibility instead of re-parsing the parent session key. The submitted branch still needs a current-main rebase and real behavior proof before it is the best merge candidate.
