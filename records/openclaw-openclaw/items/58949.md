## Summary

Keep this PR open, but do not merge the current patch: it targets a real source-visible timing asymmetry, yet the implementation switches a shared credential helper back to a CodeQL-flagged HMAC/hash comparison path.

## What This Changes

The PR changes `safeEqualSecret` to normalize null/undefined inputs, compare fixed-key HMAC digests with `timingSafeEqual`, and add null/undefined/empty-string regression cases.

## Best Possible Solution

Keep the padded timing-safe byte comparison, add reviewed comparable work for invalid inputs before returning false, retain the new edge-case tests, and require a green CodeQL/security scan on the final head.

## Reproduction Assessment

Yes, source-reproducible: current `main` returns early for non-string inputs while valid strings continue through buffer allocation and `timingSafeEqual`. I did not run a timing benchmark in the read-only checkout.

## Solution Assessment

No. The added edge-case tests are useful, but the production fix should harden the existing padded-buffer primitive instead of switching shared credential comparison to fixed-key HMAC digests.
