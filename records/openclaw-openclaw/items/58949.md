## Summary

This PR should stay open because it targets a real source-reproducible timing asymmetry, but the current patch changes the shared secret comparison primitive to fixed-key HMAC digests and the live PR head is still CodeQL-blocked. The prior review comments and current checks point to a security-reviewed replacement approach plus contributor real behavior proof before merge.

## What This Changes

The PR changes `safeEqualSecret` from padded `timingSafeEqual` buffer comparison to fixed-key HMAC digest comparison and adds null/undefined/empty-string edge-case tests.

## Best Possible Solution

Keep the useful invalid-input edge tests, but implement the production fix on top of the existing padded `timingSafeEqual` buffer comparison and require green security review on the final head.

## Reproduction Assessment

Yes, source-reproducible: current main returns before buffer allocation and `timingSafeEqual` when either input is not a string, while string inputs go through padded comparison. I did not run timing benchmarks in the read-only checkout.

## Solution Assessment

No. The edge-case tests are useful, but changing shared credential comparison to fixed-key HMAC digests is not the narrowest maintainable fix while CodeQL flags that exact path.
