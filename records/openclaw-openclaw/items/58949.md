## Summary

Keep open: current main does have a source-visible fast-fail path for invalid inputs, but this PR’s production fix switches shared credential comparison to a fixed-key HMAC path that is currently failing CodeQL and needs security-owner review/rework.

## What This Changes

The PR replaces `safeEqualSecret`'s padded buffer comparison with fixed-key HMAC digest comparison and adds null/undefined/empty-string edge tests.

## Best Possible Solution

Rework `safeEqualSecret` on top of the existing padded `timingSafeEqual` buffer comparison, add comparable invalid-input work plus edge-case tests, and require green CodeQL/security review on the final head.

## Reproduction Assessment

Yes, source-reproducible: current main returns before buffer allocation/comparison when either value is not a string, while valid string pairs go through padding and `timingSafeEqual`. I did not run a timing benchmark in the read-only checkout.

## Solution Assessment

No. The new edge-case tests are useful, but the production fix should harden the existing padded byte comparison instead of switching shared credential comparison to fixed-key HMAC digests that CodeQL flags.
