## Summary

Keep open. The PR targets a real source-level timing-shape concern, but the current patch should not merge because it replaces the existing padded byte comparison with fixed-key HMAC hashing on candidate credentials, which is exactly the kind of path current main recently moved away from and CodeQL is now flagging.

## What This Changes

The PR changes `safeEqualSecret` to normalize invalid inputs, compare fixed-key HMAC digests, and add null/undefined/empty-string regression cases.

## Best Possible Solution

Keep the padded timing-safe byte comparison, add reviewed comparable dummy work for invalid inputs, and retain the new null/undefined-versus-empty regression coverage.

## Reproduction Assessment

Yes, source-reproducible. Current main returns early for non-string inputs at `src/security/secret-equal.ts:16`, while valid string inputs run the padded timing-safe comparison path at `src/security/secret-equal.ts:25`.

## Solution Assessment

No. The regression tests are useful, but the production fix should harden the existing padded-buffer primitive instead of switching candidate credentials back to fixed-key HMAC digests.
