## Summary

Keep open. Current main still lacks an iMessage-owned per-turn tool-activity watchdog: the monitor dispatches replies without progress callbacks, while the shared reply runner exposes those callbacks and Telegram already uses the same seam for tool progress.

## What This Changes

Add an iMessage monitor watchdog that sends one progress ping during a silent turn after tool activity has occurred.

## Best Possible Solution

Implement the watchdog in the iMessage monitor using the existing GetReplyOptions progress callbacks, reset it per inbound turn and visible output, and avoid new core SDK contracts unless #75846 exposes a real missing event seam.

## Reproduction Assessment

Yes. Source inspection on current main shows iMessage dispatch does not register progress callbacks, while the shared runner emits the tool/item/command callbacks that a channel watchdog would need.

## Solution Assessment

Yes. A channel-local watchdog using the existing reply callback seam is the narrowest maintainable approach; it should be sequenced with #75846 before claiming claude-cli coverage.
