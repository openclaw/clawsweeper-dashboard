## Summary

Keep open. Current main still lacks an iMessage-owned per-turn watchdog or visible progress ping for tool activity during agent silence; the shared reply callback seam exists and Telegram uses it, but iMessage only wires typing lifecycle options into its dispatch path.

## What This Changes

Request adding an iMessage-local per-turn watchdog that sends one progress ping when tool events occur during an otherwise silent agent turn.

## Best Possible Solution

Add an iMessage-local one-shot progress watchdog on the existing reply callback seam, reset it per inbound turn and visible output, and coordinate Claude CLI coverage with the linked event-bridge issue.

## Reproduction Assessment

Yes. Source inspection is high-confidence: iMessage does not register tool/progress callbacks or a watchdog, while shared reply execution emits those callbacks when a channel registers them.

## Solution Assessment

Yes, with maintainer UX confirmation. A channel-local watchdog using existing `GetReplyOptions` callbacks is the narrowest maintainable direction, but the exact timeout/message policy and the linked Claude CLI dependency should be reviewed together.
