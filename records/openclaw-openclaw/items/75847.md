## Summary

Keep open. Current main still lacks an iMessage-owned per-turn tool-activity watchdog: the iMessage monitor does not register the shared tool/progress callbacks, while shared reply execution already forwards those events and Telegram consumes the same seam. The linked #75846 remains a separate prerequisite for claude-cli tool_use coverage, not a replacement for this channel fix.

## What This Changes

Add an iMessage monitor per-turn watchdog that sends one progress ping after agent silence when tool activity has occurred in that turn.

## Best Possible Solution

Implement an iMessage-local per-turn watchdog using the existing GetReplyOptions progress callbacks, reset it on each inbound turn and visible output, and sequence the #75846 bridge before claiming claude-cli coverage.

## Reproduction Assessment

Yes. Static current-main verification is high-confidence: iMessage does not register the progress callbacks, while shared reply execution emits them and Telegram consumes them.

## Solution Assessment

Yes. The requested channel-local watchdog is the narrowest maintainable fix because it reuses the existing reply callback seam and avoids a new global event subscription path.
