## Summary

Keep open. Current main still has no iMessage progress callbacks or per-turn tool-activity watchdog; the shared reply callback seam and Telegram implementation show a maintainable path, but this is new iMessage-visible progress behavior and full claude-cli coverage still depends on #75846.

## What This Changes

Request an iMessage monitor watchdog that sends one progress ping when a turn is silent but tool activity has occurred.

## Best Possible Solution

Implement an iMessage-local single-ping watchdog wired to existing GetReplyOptions progress callbacks, reset it per inbound turn and visible output, and sequence it with #75846 before claiming claude-cli coverage.

## Reproduction Assessment

Yes, by source inspection. Current main’s iMessage monitor does not register the progress callbacks, while shared reply execution emits them and Telegram consumes the same seam.

## Solution Assessment

Yes, with one sequencing caveat. A channel-local callback/watchdog is the narrowest maintainable solution, but it should be paired with #75846 for claude-cli tool events.
