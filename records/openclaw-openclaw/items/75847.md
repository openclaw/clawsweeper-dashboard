## Summary

Keep open. Current main still has the iMessage-specific gap described here: the monitor dispatches replies without tool/progress callbacks, while the shared reply runner and other channel monitors already expose and consume those signals; the linked claude-cli bridge gap in #75846 remains relevant for claude-cli-backed turns.

## What This Changes

Add an iMessage monitor per-turn watchdog that sends one progress ping when a turn is silent but tool activity has occurred.

## Best Possible Solution

Add an iMessage-owned, per-turn, single-shot watchdog over existing reply progress callbacks, reset it on each turn and visible output, and coordinate claude-cli tool-event delivery through #75846.

## Solution Assessment

Yes. The narrowest maintainable fix is to use existing GetReplyOptions callbacks inside the iMessage monitor instead of adding a broad global event subscription or core-owned channel policy.

## Repair Work Prompt

Implement a narrow fix for this issue, coordinating with #75846 as the related prerequisite. In the iMessage plugin, add a per-turn, single-shot watchdog that uses existing reply progress callbacks such as onToolStart, onItemEvent, onCommandOutput, onPlanUpdate, onApprovalEvent, and onPatchSummary to detect tool activity during agent silence. The watchdog should track last visible output per turn, send one concise progress ping after the configured/internal silence threshold only when tool activity occurred, reset on each turn and visible reply output, and avoid spam. Keep the behavior extension-owned in extensions/imessage; do not add a broad core/global event subscription or hard-code bundled plugin policy in core. Preserve existing delivery sanitization, echo-cache/self-chat protections, and block-streaming behavior. If #75846 is fixed in the same PR, bridge Claude CLI tool_use JSONL events into the existing onAgentEvent/stream:tool shape used by embedded pi. Add focused tests for the iMessage watchdog and callback wiring; add a user-facing CHANGELOG.md fix entry unless maintainers classify the change as internal-only.
