## Summary

Keep open. Current main still lacks an iMessage-owned per-turn watchdog: the iMessage monitor does not pass tool/progress callbacks into reply dispatch, while shared reply execution and other channel monitors already expose and consume those events; the paired #75846 prerequisite remains separate for claude-cli turns.

## What This Changes

Add an iMessage per-turn watchdog that sends one progress ping when a turn is silent but has tool activity.

## Best Possible Solution

Implement a channel-local watchdog using the existing GetReplyOptions progress callbacks, reset it per inbound turn and visible output, and handle #75846 in the same or prerequisite PR for claude-cli tool events.

## Reproduction Assessment

Yes. Static current-main verification is high-confidence: the iMessage dispatch path has no progress callbacks or watchdog, while shared reply execution and other channels already expose and consume those signals.

## Solution Assessment

Yes. The proposed channel-local watchdog is the narrowest maintainable fix; it should avoid global event subscriptions and reuse the established reply callback seam.
