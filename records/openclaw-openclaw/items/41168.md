## Summary

Keep open. Current main and v2026.5.12 still do not choose the latest active non-heartbeat user session on TUI startup; they resume an exact remembered session or fall back to the agent main session, while the storage-bloat half is covered by the open canonical session-store tracker.

## What This Changes

The issue asks the TUI to reopen the latest active non-heartbeat user session after restart and to stop duplicated skill context from bloating `sessions.json`.

## Best Possible Solution

Define the TUI resume policy explicitly, add latest-active non-heartbeat selection only with a reliable user-activity signal, and keep the static session-store payload work centralized in https://github.com/openclaw/openclaw/issues/55334.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence. Current main defaults to the agent main session with no explicit session and validates only one remembered key, so it never selects the newest non-heartbeat user session by activity time.

## Solution Assessment

No, not as a direct `updatedAt` sort patch. The maintainable fix needs an explicit TUI resume policy and a user-activity freshness signal, while storage bloat should continue through the canonical session-store schema tracker.
