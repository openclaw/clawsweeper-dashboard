## Summary

Keep open. Current main and v2026.5.22 still restore only an exact remembered TUI session rather than the latest active non-heartbeat user session, and the session-store bloat half is only partially mitigated while prompt/report compaction remains tracked elsewhere.

## What This Changes

The issue asks for TUI startup to resume the latest active user session instead of a heartbeat/main session and for saved sessions to stop repeatedly carrying large skill-context metadata.

## Best Possible Solution

Define the TUI resume contract around a durable user-active signal that excludes heartbeat/system sessions, and sequence prompt/report compaction through the canonical session-store bloat work with migration and cold-resume coverage.

## Reproduction Assessment

Yes, at source level. Current main restores only an exact remembered TUI session, falls back to main/global without latest-active discovery, and still preserves skillsSnapshot.prompt plus systemPromptReport in the session entry type.

## Solution Assessment

No, not as a cleanup close or a simple latest-by-updatedAt sort. The TUI half needs a maintainer-approved user-active resume contract, and the storage half should follow the canonical session-store design rather than broad metadata deletion.
