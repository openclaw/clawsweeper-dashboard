## Summary

Close as implemented on current main: the TUI now restores a remembered user-selected session instead of blindly loading the default main/heartbeat-shaped session, and session persistence strips the heavy runtime skill cache that caused sessions.json bloat.

## What This Changes

The issue requests TUI reconnects to resume the recent active user session instead of the heartbeat/default session, and asks for session metadata persistence to stop accumulating full skill context per saved session.

## Best Possible Solution

Keep the current main implementation and ship it in the next release so TUI reconnects use remembered sessions and sessions.json avoids persisting runtime skill bodies.

## Reproduction Assessment

No current-main failure path was established. Source and regression-test inspection show the historical symptom is fixed on main rather than still reproducible.

## Solution Assessment

Yes. Remembering the explicit TUI-selected session is a narrower and more stable fix than sorting by heartbeat-mutated session timestamps, and stripping resolvedSkills removes the large runtime cache while preserving the lightweight prompt snapshot needed for resume.
