## Summary

Keep open. Current main fixed the older TUI relaunch continuity slice by remembering the last selected TUI session, but it still does not implement latest-active non-heartbeat startup selection, and the storage-bloat half remains open under the narrower session-bloat issue.

## What This Changes

The issue asks TUI startup to choose the latest active non-heartbeat user session and asks session persistence to stop accumulating full skill/system-prompt context per session.

## Best Possible Solution

Define the TUI resume contract explicitly, then implement latest-active non-heartbeat selection only with a reliable user-activity signal while resolving the remaining storage payload policy through https://github.com/openclaw/openclaw/issues/45718.

## Reproduction Assessment

Yes. Source inspection shows current main restores an exact remembered TUI key or falls back to main/global, and session persistence still retains prompt/report metadata beyond the stripped resolvedSkills cache.

## Solution Assessment

No. A direct updatedAt sort patch would not be the best fix because maintainers need to define user-active session semantics, and the storage portion should follow the existing session-bloat contract work instead of deleting fields broadly.
