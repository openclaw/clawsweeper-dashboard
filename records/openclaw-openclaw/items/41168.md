## Summary

Keep open. Current main partially reduces the reported pain, but it still does not implement latest-active user-session selection, and the broader skill-snapshot serialization cleanup is represented by open PR #48425 rather than a merged fix.

## What This Changes

The issue asks the TUI to reopen the latest active user session instead of a heartbeat/main session and asks session persistence to stop duplicating skill-context data per saved session.

## Best Possible Solution

Define the TUI startup policy explicitly, expose user-interaction freshness if latest-active selection is desired, filter heartbeat/system-only rows where appropriate, and handle the remaining skill-snapshot serialization cleanup through #48425 or an equivalent focused change.

## Reproduction Assessment

Yes for source-level reproduction: current main defaults an empty TUI session to the agent main key, restores only a remembered key, and sorts/list-filters sessions by updatedAt rather than user interaction freshness. The storage half is partially fixed because resolvedSkills is stripped, but the broader skillsSnapshot exclusion is not on main.

## Solution Assessment

No. Current main is only a partial solution: last-selected resume and resolvedSkills stripping help, but they do not implement latest-active user-session selection or settle the remaining transient snapshot serialization policy.
