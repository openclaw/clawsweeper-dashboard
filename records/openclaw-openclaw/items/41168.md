## What This Changes

Fix TUI startup selecting heartbeat state and prevent full skill context from bloating each persisted session entry.

## Best Possible Solution

Keep the shipped heartbeat-filtered session restoration and deduplicated skill-prompt persistence; request a new focused report only if a current release still produces a specific oversized artifact.

## Solution Assessment

Yes. Remembering the last selected non-heartbeat session preserves user intent, while stripping and content-addressing heavy skill data fixes growth at the persistence boundary without adding a competing runtime path.
