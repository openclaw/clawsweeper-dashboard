## Summary

Keep open. Current main has shipped partial relief through last-selected TUI resume and `resolvedSkills` stripping, but it still does not implement latest-active user-session startup and the remaining `skillsSnapshot` serialization cleanup is represented by open PR #48425.

## What This Changes

The issue requests that TUI startup select the most recently active user session instead of the heartbeat/main session and that `sessions.json` stop accumulating full per-session skill context.

## Best Possible Solution

Define the TUI startup policy explicitly, add any needed freshness signal if latest-active selection is desired, and finish the transient `skillsSnapshot` serialization cleanup through #48425 or an equivalent focused change.

## Reproduction Assessment

Yes, for source-level reproduction. Current main defaults an empty per-sender TUI launch to the agent main session, restores only a remembered last-selected key, and still serializes lightweight `skillsSnapshot` fields while stripping only `resolvedSkills`.

## Solution Assessment

No. The current implementation is a partial mitigation, not the requested latest-active selection policy or complete transient snapshot serialization cleanup.
