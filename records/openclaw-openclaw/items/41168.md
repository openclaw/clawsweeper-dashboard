## Summary

Keep open. Current main partially reduces the pain with last-selected TUI resume and resolvedSkills stripping, but it still does not implement latest-active user-session selection and the broader skillsSnapshot serialization cleanup remains represented by open PR #48425.

## What This Changes

The issue requests that TUI relaunch attach to the latest active user session rather than the heartbeat/main session and that sessions.json avoid repeated full skill context snapshots.

## Best Possible Solution

Define the TUI startup policy explicitly, add a real user-interaction freshness signal if latest-active selection is desired, and handle transient skillsSnapshot serialization through #48425 or an equivalent focused storage change.

## Reproduction Assessment

Yes for source-level reproduction. Current main still defaults an empty TUI session to the agent main key, restores only a remembered last-selected key, and serializes a normalized store that preserves lightweight skillsSnapshot fields.

## Solution Assessment

No. Current main is only a partial solution: last-selected resume and resolvedSkills stripping help, but they do not provide latest-active user-session selection or settle the remaining transient snapshot serialization policy.
