## What This Changes

The issue reports that `openclaw doctor --fix` adds `codex` to plugin config and enables it when `openai` routes require Codex runtime, even when the user never configured that plugin.

## Best Possible Solution

Keep this doctor-specific tracker open until maintainers decide the OpenAI-through-Codex dependency contract, then update doctor repair, tests, and docs to either preserve explicit plugin intent or clearly require Codex with explicit diagnostics.

## Reproduction Assessment

Yes, at source level: current `main` has a doctor repair test matching the reported `plugins.allow: ["openai"]` and missing-`codex` entry shape, and it expects `codex` to be enabled and allowlisted. I did not run `doctor --fix` because this review is read-only.
