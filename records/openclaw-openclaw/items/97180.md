## Summary

Keep open: current main and the latest stable release still intentionally add and enable `codex` during doctor repair for OpenAI routes that resolve to the Codex runtime, and the broader related issue does not fully cover this doctor-specific config mutation report.

## What This Changes

The issue asks OpenClaw to stop `openclaw doctor --fix` from adding or enabling an unconfigured or uninstalled `codex` plugin when `openai` is enabled.

## Best Possible Solution

Keep this as the doctor-specific tracker and align doctor repair with explicit plugin/operator intent, coordinating with #95131 for the broader OpenAI/Codex default contract.

## Reproduction Assessment

Yes at source level: current main's doctor repair path and tests cover a config with `plugins.allow: ["openai"]`, enabled `openai`, no codex entry, and an OpenAI default model, and they expect doctor repair to enable and allowlist `codex`.

## Solution Assessment
