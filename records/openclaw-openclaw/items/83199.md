The issue asks OpenClaw to preserve Codex commandExecution outputDelta text as fallback final command output when aggregatedOutput is missing or empty.

## Best Possible Solution

Yes, from source inspection: send item/commandExecution/outputDelta for a command item, then complete the turn with commandExecution.aggregatedOutput null or empty. Current main streams the delta for verbose output but final transcript and trajectory output read only aggregatedOutput.

## Solution Assessment

Yes. The narrow maintainable fix is to retain per-item streamed command output in the Codex projector and use it only as a fallback when aggregatedOutput is empty; the linked PR is the active path for that fix.
