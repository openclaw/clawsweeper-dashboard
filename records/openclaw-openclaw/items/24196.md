## Summary

Keep open. Current main still carries multiple media references upstream, but Discord delivery remains scalar or sequences each file as a separate send instead of creating one native Discord message with multiple attachments.

## What This Changes

Allow Discord outbound messages to send multiple file attachments in one native Discord message up to Discord's per-message attachment limit.

## Best Possible Solution

Add a Discord-owned batching path that loads each attachment through existing media helpers, sends files together up to the verified Discord cap, splits overflow into follow-up messages, and keeps scalar inputs compatible.

## Reproduction Assessment

Yes, at source level. Current main can carry multiple media references upstream, but Discord action/send paths still accept one `mediaUrl` and shared helpers sequence multiple media URLs as separate sends.

## Solution Assessment

Yes. A Discord-owned batching implementation is the narrowest maintainable path; avoid adding a new config knob or broadly changing shared outbound behavior unless the plugin cannot express the batching locally.
