## Summary

Keep open. Current main still treats Discord outbound uploads as one `mediaUrl`/`filename` at a time and fans multi-media payloads into repeated one-file sends, so native multi-attachment Discord messages remain unimplemented.

## What This Changes

Request native Discord outbound messages that can attach up to the Discord per-message file limit instead of exposing only one file per message.

## Best Possible Solution

Add Discord-owned multi-file batching that accepts ordered attachments, sends up to the verified Discord per-message limit in one `files` array, splits overflow into follow-up messages, and keeps scalar send behavior compatible.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: multi-media payloads become multiple `sendMedia` calls, while the Discord action/runtime/send stack only accepts one `mediaUrl`.

## Solution Assessment

Yes. A Discord-owned native multi-attachment path is the narrowest maintainable direction; shared/core changes should be limited to an additive typed seam only if Discord needs to receive ordered batches.
