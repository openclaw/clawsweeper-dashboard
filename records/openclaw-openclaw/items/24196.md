## Summary

Keep open: current main and v2026.6.8 still expose Discord outbound uploads through scalar `mediaUrl`/`filename` paths, and https://github.com/openclaw/openclaw/pull/94122 is an open, unmerged candidate rather than an implemented fix.

## What This Changes

Request native Discord outbound sends that can attach the requested multi-file batch in one message instead of exposing or delivering only one file at a time.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/94122; the desired end state is a Discord-owned batch-send path with scalar compatibility, media safety checks, and overflow splitting.

## Reproduction Assessment

Yes, at source level: current main resolves one `mediaUrl`, the Discord sender only accepts one media URL, and shared planning maps each `mediaUrls` entry to a separate media unit. I did not run a live Discord send in this read-only review.

## Solution Assessment

Yes. The narrow maintainable solution is Discord plugin-owned native batching while preserving scalar compatibility; a new config option or generic all-channel batching behavior is not needed for this request.
