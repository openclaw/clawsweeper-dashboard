## Summary

Keep open. Current main and the latest release still route Discord outbound uploads through scalar `mediaUrl`/`filename` fields and one-file sends, while shared `mediaUrls` delivery fans out into repeated media sends instead of one native Discord multi-file message.

## What This Changes

Allow Discord outbound messages to send up to Discord's native multi-file attachment limit in one message instead of one file per send.

## Best Possible Solution

Add a Discord-owned multi-attachment send path that preserves scalar `media`/`path`/`filePath`, batches ordered files up to Discord's per-message cap, splits overflow, and updates focused tests, docs, and changelog.

## Reproduction Assessment

Yes for the current limitation: source inspection shows `mediaUrls` are planned as separate media sends and Discord upload code accepts one `mediaUrl` per request. I did not run a live Discord send in this read-only review.

## Solution Assessment

Yes. A Discord-owned batching path is the narrowest maintainable direction; shared/core changes should be minimal and additive only if the plugin cannot otherwise receive ordered batches.
