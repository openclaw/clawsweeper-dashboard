## Summary

Keep open. Current main still exposes Discord outbound uploads through scalar `mediaUrl`/`filename` paths, and shared outbound delivery still fans multi-media payloads into repeated one-file sends, so native multi-attachment Discord delivery remains unimplemented.

## What This Changes

The issue requests native Discord support for sending multiple file attachments in a single message up to Discord's per-message limit.

## Best Possible Solution

Add native multi-file sending in the bundled Discord plugin, preserving existing scalar behavior and batching ordered attachments through the existing guarded media loading path with overflow split into follow-up messages.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a payload with two `mediaUrls` is planned as two media units and delivered through two `sendMedia` calls, while the Discord action/runtime/send stack has no multi-attachment input shape.

## Solution Assessment

Yes. A Discord-owned multi-attachment send path is the narrowest maintainable solution; shared/core changes should be limited to a typed outbound adapter seam only if needed to avoid changing other channels' behavior.
