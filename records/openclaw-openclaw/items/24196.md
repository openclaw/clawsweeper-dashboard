## Summary

Keep open. Current main still accepts Discord uploads through scalar `mediaUrl`/`filename` paths and delivers multiple shared `mediaUrls` as repeated one-file sends, so the requested native Discord multi-file message behavior remains unimplemented.

## What This Changes

Request to let the bundled Discord plugin send multiple file attachments in one Discord message instead of requiring one message per file.

## Best Possible Solution

Implement Discord-owned multi-attachment batching that keeps existing scalar sends compatible, batches ordered attachments up to the verified Discord limit, and splits overflow into follow-up messages.

## Reproduction Assessment

Yes for the current limitation: source inspection on current main shows the Discord action/runtime/send stack accepts one `mediaUrl`, while shared outbound iterates multi-media payloads into separate sends. I did not run a live Discord send in this read-only review.

## Solution Assessment

Yes. A Discord-owned native multi-attachment path is the narrowest maintainable direction; shared/core changes should be limited to an additive typed seam only if the plugin cannot otherwise receive ordered batches.
