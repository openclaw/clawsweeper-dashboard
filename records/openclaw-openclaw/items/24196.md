## Summary

Keep open. Current main and v2026.6.5 still sequence multi-media payloads into separate Discord sends and build native upload requests with one file, so native Discord attachment batching remains unimplemented.

## What This Changes

The issue requests allowing Discord messages to attach up to Discord's native per-message file limit instead of exposing only one file attachment.

## Best Possible Solution

Add Discord-plugin-owned native batching that treats existing single-file inputs as the one-item case, batches ordered attachments up to the verified Discord limit, and splits overflow without adding configuration.

## Reproduction Assessment

Yes, at source level with high confidence: current main accepts and sequences multi-media payloads, but Discord send still consumes one `mediaUrl` and constructs a one-file native upload request. I did not run a live Discord account test in this read-only review.

## Solution Assessment

Yes. A Discord-plugin-owned batching seam is the narrowest maintainable direction; adding new global `paths` schema/config or changing generic outbound fan-out would create unnecessary cross-channel surface.
