## Summary

Keep open: current main and the latest release still route Discord outbound media through scalar `mediaUrl`/single-file send paths, while https://github.com/openclaw/openclaw/pull/94122 is an open, unmerged implementation candidate that explicitly closes this issue.

## What This Changes

Request native Discord outbound support for sending multiple file attachments in one message instead of requiring one message per file.

## Best Possible Solution

Review and either land or replace https://github.com/openclaw/openclaw/pull/94122; after a merged fix proves ordered Discord attachment batching and overflow splitting, close this issue as implemented.

## Reproduction Assessment

Yes, at source level: `mediaUrls` are planned as separate media units and the Discord sender itself accepts only one `mediaUrl` and builds one uploaded file. I did not run a live Discord send in this read-only review.

## Solution Assessment

Yes, the best direction is a Discord-owned batching path that preserves existing scalar inputs and media safety checks. A new config knob or generic gateway-wide batching behavior is not needed for this request.
