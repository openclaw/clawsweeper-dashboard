## Summary

Keep open. Current main and v2026.6.6 still send Discord multi-media payloads as repeated scalar uploads and build one native upload request with one file, and no open or merged canonical PR now owns the requested batching behavior.

## What This Changes

Allow Discord outbound messages to include the platform's native maximum file attachments in one message instead of exposing only single-file uploads.

## Best Possible Solution

Add a Discord-owned ordered attachment batch send path that posts up to the verified Discord per-message limit per request, splits overflow into follow-up messages, and preserves existing scalar callers and media safety checks.

## Reproduction Assessment

Yes at source level. A Discord payload with two mediaUrls reaches sendPayloadMediaSequenceOrFallback, whose callback passes one mediaUrl at a time to scalar sendMessageDiscord, and sendDiscordMedia creates a one-element files array.

## Solution Assessment

Yes. Native batching inside the Discord plugin is the narrowest maintainable direction; a new config knob or broad shared-outbound behavior change is not needed for the requested capability.
