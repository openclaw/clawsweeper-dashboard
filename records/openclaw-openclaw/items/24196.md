## Summary

Keep open. Current main still has scalar Discord upload paths and one-file send loops; the newer component/media-gallery surface is not the requested native multi-file attachment batching for ordinary Discord messages.

## What This Changes

Allow Discord outbound messages to attach multiple files in one native Discord message instead of requiring one file per send.

## Best Possible Solution

Implement Discord-owned multi-attachment batching that keeps scalar sends compatible, batches ordered attachments up to Discord's verified limit, and splits overflow into follow-up messages.

## Reproduction Assessment

Yes for the current limitation: source inspection on current main shows the Discord action/runtime/send stack accepts one media reference, while shared outbound iterates `mediaUrls` into separate sends. I did not run a live Discord send in this read-only review.

## Solution Assessment

Yes. A Discord-owned native multi-attachment path is the narrowest maintainable direction; shared/core changes should be limited to an additive typed seam only if the plugin cannot otherwise receive ordered batches.
