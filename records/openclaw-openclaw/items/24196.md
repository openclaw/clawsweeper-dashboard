## Summary

Current main still does not implement native multi-file Discord message uploads. The ordinary Discord send action and runtime accept one `mediaUrl`/`filename`, `sendDiscordMedia` builds a `files` array with exactly one upload, and shared outbound `mediaUrls` are still planned and delivered as repeated one-file sends. The April 26 maintainer review in the issue remains accurate after checking the current tree.

## What This Changes

Allow Discord outbound messages to attach multiple files to one Discord message, up to Discord's supported per-message attachment limit, instead of exposing only a single file path.

## Best Possible Solution

Add Discord-owned native multi-attachment sending while keeping the existing single `media`/`path`/`filePath`/`filename` behavior compatible. The Discord plugin should accept ordered multi-attachment input, load each file through the current outbound media helpers, batch up to the verified Discord per-message limit in one `files` array, split overflow into follow-up messages, and document the user-facing shape.
