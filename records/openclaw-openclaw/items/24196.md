## Summary

Keep open. Current main and the latest release still route Discord media through scalar `mediaUrl` sends or one-file-at-a-time media sequences, so the requested native multi-file Discord message behavior remains unimplemented.

## What This Changes

The issue requests Discord outbound delivery that can attach multiple files to one message up to Discord's native per-message limit.

## Best Possible Solution

Add a Discord-owned batching path that loads each attachment through the existing media helpers, sends up to the verified Discord per-message file limit together, splits overflow into follow-up messages, and preserves current scalar inputs.

## Reproduction Assessment

Yes, at source level. Current main preserves multiple media hints upstream, but Discord send options and delivery helpers still pass one `mediaUrl` at a time and tests assert separate sends for multiple URLs.

## Solution Assessment

Yes. A Discord-owned batching implementation is the narrowest maintainable solution; avoid a new config knob or generic outbound rewrite unless the plugin cannot express batching locally.
