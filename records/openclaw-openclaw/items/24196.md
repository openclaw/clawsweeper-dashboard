## Summary

Keep open. Current main and v2026.6.6 still sequence multiple media URLs into separate scalar Discord sends and construct each native upload request with exactly one file, so the requested multi-attachment message behavior remains missing; no canonical implementation PR was found.

## What This Changes

Allow Discord outbound messages to include up to 10 ordered file attachments in one native message and split overflow into follow-up messages.

## Best Possible Solution

Extend the Discord plugin send boundary with an ordered attachment-batch shape, post chunks of at most 10 files through native Discord requests, and retain scalar callers plus existing access, size, voice, component, reply, retry, text-chunk, and receipt semantics.

## Reproduction Assessment

Yes at source level: providing two `mediaUrls` to the current Discord outbound payload path deterministically invokes the one-file sender twice rather than building one two-file request.

## Solution Assessment

Yes. Native batching inside the Discord plugin is the narrowest maintainable direction; neither a new configuration option nor a cross-channel outbound policy change is needed.
