## Summary

Keep open. Current main and the latest release still accept multiple media hints upstream, but Discord outbound delivery remains scalar or sends them as separate one-file messages; no open or merged PR appears to supersede this Discord-specific request.

## What This Changes

Allow Discord messages to send multiple file attachments in one native Discord message, up to Discord's per-message attachment cap.

## Best Possible Solution

Add a Discord-owned batching path that loads each attachment through existing media helpers, sends up to 10 files together when Discord allows it, splits overflow into follow-up messages, and keeps scalar inputs compatible.

## Reproduction Assessment

Yes, at source level. Current main preserves multiple attachment hints upstream, but Discord send options and delivery helpers still pass one `mediaUrl` at a time and tests assert separate sends for multiple URLs.

## Solution Assessment

Yes. A Discord-owned batching implementation is the narrowest maintainable path; avoid a new config knob or generic gateway rewrite unless the plugin cannot express batching locally.
