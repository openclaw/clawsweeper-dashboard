## Summary

Keep open. Current main still does not implement native multi-file Discord message delivery: upstream reply/send payloads can carry multiple media references, but the Discord action/runtime/upload path remains scalar and the shared fallback sends media one file at a time.

## What This Changes

Request to let the Discord plugin attach multiple files to one Discord message up to the platform limit instead of exposing only single-file sends.

## Best Possible Solution

Add a Discord-owned multi-attachment send path that preserves scalar inputs, batches ordered media up to the verified Discord per-message cap, splits overflow, and updates focused tests, docs, and changelog.

## Reproduction Assessment

Yes, at source level. Current main can carry multiple media references into outbound payloads, but the Discord action/runtime/send helper paths reduce normal upload delivery to one `mediaUrl` or sequential one-file sends.

## Solution Assessment

Yes. A Discord-owned batching path is the narrowest maintainable solution; avoid a broad core fan-out rewrite or a new config option unless maintainers explicitly want a tunable cap.
