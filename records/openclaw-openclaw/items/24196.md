## Summary

Keep open. Current main still models ordinary Discord outbound uploads as a single `mediaUrl`/`filename` and splits multi-media payloads into one upload call per file, so native multi-attachment Discord messages remain unimplemented.

## What This Changes

The issue requests Discord message sending support for up to 10 file attachments in one message instead of one attachment per message.

## Best Possible Solution

Implement Discord-owned multi-file batching for ordered attachments, splitting overflow into follow-up messages while keeping existing scalar send behavior compatible.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a payload with two `mediaUrls` is planned as two media units and delivered through two media sends, while the normal Discord action/runtime/send stack has no multi-attachment input shape.

## Solution Assessment

Yes. A Discord-owned native multi-attachment send path is the narrowest maintainable direction; shared/core changes should be limited to an additive typed seam if Discord needs to receive an ordered batch without changing other channels.
