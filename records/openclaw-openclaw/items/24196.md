## Summary

Keep open. Current main still supports multiple media references upstream, but the Discord action/runtime/upload path remains scalar and shared outbound delivery still fans out multiple media URLs into separate one-file sends.

## What This Changes

The issue requests native Discord support for sending up to ten file attachments in one message instead of requiring one message per file.

## Best Possible Solution

Add a Discord-owned multi-attachment send path that preserves scalar inputs, batches ordered media up to the verified Discord per-message cap, splits overflow, and updates focused tests, docs, and changelog.

## Reproduction Assessment

Yes, at source level. Current main can carry multiple media references into outbound payloads, but the Discord action/runtime/send helper paths reduce normal upload delivery to one mediaUrl or sequential one-file sends.

## Solution Assessment

Yes. A Discord-owned batching path is the narrowest maintainable solution; avoid a broad shared outbound fan-out rewrite or a new config option unless maintainers explicitly want a tunable cap.
