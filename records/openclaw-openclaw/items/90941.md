## What This Changes

The issue reports that doctor sandbox image repair cannot find scripts/sandbox-setup.sh when the openclaw launcher path is a symlink.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/90942 if its realpath candidate handling is acceptable, while keeping the package-only sandbox script distribution question in the separate follow-up issue.

## Reproduction Assessment

Yes. Source inspection shows current main derives launcher roots from path.resolve(process.argv[1]) without following symlinks, and the linked PR supplies before/after resolver proof for that path.

## Solution Assessment

Yes for the scoped symlink bug. Adding realpath-derived launcher candidates while preserving cwd and literal launcher candidates is the narrow fix; package-only script shipping remains a separate product decision.
