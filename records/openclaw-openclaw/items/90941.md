## What This Changes

The issue requests that `openclaw doctor` locate sandbox setup scripts correctly when the CLI launcher is a symlink to the real checkout or package entrypoint.

## Best Possible Solution

Review and land an equivalent shared-root-resolver fix through https://github.com/openclaw/openclaw/pull/90942, then close this issue while keeping package-only script availability tracked separately.

## Reproduction Assessment

Yes, as a source-level reproduction. Current main resolves `process.argv[1]` with `path.resolve` and does not add a realpath candidate, so a symlinked launcher searches beside the symlink rather than the real checkout.

## Solution Assessment

Yes for keeping this issue open until the linked fix lands. The best implementation should fix sandbox script root lookup through the existing package-root resolution boundary and leave the npm package script-shipping decision to the separate follow-up.
