## What This Changes

The issue asks doctor sandbox image repair to follow a symlinked openclaw launcher when locating bundled sandbox setup scripts.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/90942 for the symlinked source/linked-install resolver fix, while keeping package-only script distribution scoped to https://github.com/openclaw/openclaw/issues/90951.

## Reproduction Assessment

Yes. A linked/source install where process.argv[1] is a symlink to the real openclaw.mjs and the default sandbox image is missing will run the current path.resolve-only resolver and search beside the symlink.

## Solution Assessment

Yes. Adding realpath-derived launcher candidates while preserving the cwd and literal argv1 candidates is the narrowest maintainable fix for this issue; package-only script shipping belongs to the separate follow-up.
