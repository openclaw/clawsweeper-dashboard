## Summary

Keep open: this PR targets a real silent model-selection bug, but the patch currently drops provider-selected `agentModelOverride` for every deferred setup path, which would break no-global-default agent creation. It also still lacks after-fix real `openclaw agents add` proof and is conflicting with current main.

## What This Changes

The branch changes provider auth-choice default-model handling and related tests so `openclaw agents add` does not persist a provider default model over inherited agent defaults.

## Best Possible Solution

Land a narrow conditional fix that omits per-agent overrides only when an existing global primary should be inherited, preserves provider-selected overrides when no global default exists, and includes real CLI proof for both setups.

## Reproduction Assessment

Yes, source-reproducible. `agents add` calls `applyAuthChoice` with `setDefaultModel: false` and only writes the per-agent model when `agentModelOverride` is returned; current provider paths return that override, while the PR removes it.

## Solution Assessment

No. The PR targets the right seam, but the best fix is conditional: inherit an existing global primary while preserving the selected provider model when no global primary exists.
