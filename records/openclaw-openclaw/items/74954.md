## What This Changes

The PR changes provider auth/default-model selection and related tests so provider default models are not written as per-agent overrides when agents should inherit `agents.defaults.model`.

## Best Possible Solution

Rebase or replace this with a narrow current-main fix in `src/plugins/provider-auth-choice.ts`: suppress per-agent overrides only when an inherited global primary exists, and preserve the selected provider model when no global default exists.

## Reproduction Assessment

Yes, from source inspection: `openclaw agents add` writes returned `agentModelOverride` values into `agents.list[].model`, and current provider auth paths still return the provider default in the inherited-default case. I did not run the CLI because this review is read-only.

## Solution Assessment

No. The PR targets the right seam, but the best fix is conditional: inherit an existing global primary while preserving the selected provider model when no global default exists; fixing only the `agents add` caller would miss sibling provider-auth callers.
