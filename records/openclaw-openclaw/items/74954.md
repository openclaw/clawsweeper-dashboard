## Summary

Keep open. The reported agents/default-model bug is valid and this PR moves in the right direction, but the patch suppresses provider-selected agent overrides even when no inherited default model exists, which would drop the model the user just configured.

## What This Changes

The PR removes `agentModelOverride` returns from provider-auth/default-model flows when `setDefaultModel` is false and rewrites related tests so new agents inherit `agents.defaults.model` instead of baking in provider defaults.

## Best Possible Solution

Keep the branch open and narrow the fix so new agents inherit an existing global primary, while agents with no global primary still keep the provider model selected during auth; add focused regression coverage and the required changelog entry.

## Reproduction Assessment

Yes. Source inspection is enough: current main persists `agentModelOverride` from deferred provider auth into `agents.list[].model`, and the PR removes that override even when the resolver has no inherited `agents.defaults.model.primary` to use.

## Solution Assessment

No. The requested direction is right for the inherited-default case, but the PR should conditionalize override suppression on an existing inherited primary and preserve the selected provider model otherwise.
