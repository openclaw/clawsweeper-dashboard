## Summary

Keep open. The targeted bug is valid, but this PR removes deferred provider model overrides unconditionally, which fixes the configured-global-default case while regressing agents that have no inherited `agents.defaults.model.primary`. The patch needs a narrower conditional fix before merge.

## What This Changes

The PR removes `agentModelOverride` returns from deferred provider/default-model auth-choice paths and updates auth-choice tests so new agents inherit `agents.defaults.model` instead of baking in provider defaults.

## Best Possible Solution

Implement conditional deferred override behavior: when `setDefaultModel` is false and a string or object `agents.defaults.model.primary` exists, omit `agentModelOverride` so the new agent inherits it; when no inherited primary exists, preserve the selected provider default as the agent override. Keep provider config patches and allowlist entries intact, cover loaded plugin, plugin-provider, shared default-model helper, and `agents add` behavior, and add the required changelog entry.

## Reproduction Assessment

Yes. The current-main source path is high-confidence: `agents add` calls auth setup with `setDefaultModel: false`, provider/default-model helpers return `agentModelOverride`, and the agent entry writes it. The PR regression is also source-reproducible by following a deferred provider setup with no global primary, where the PR no longer preserves the selected provider model.

## Solution Assessment

No. The direction is right for the configured-global-primary case, but removing every deferred override is broader than the product contract. The safer fix is to suppress overrides only when an inherited primary exists and keep explicit provider defaults for no-default configs.
