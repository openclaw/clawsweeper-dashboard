## Summary

Keep this PR open, but it is not merge-ready: it targets a valid silent agent model-selection bug, while the current branch is conflicting, lacks real behavior proof, and unconditionally removes the provider-selected per-agent override in cases where no global default model exists.

## What This Changes

The PR changes provider auth-choice default-model handling and related tests so `setDefaultModel: false` no longer returns a provider default `agentModelOverride` when adding agents.

## Best Possible Solution

Rebase or replace this with a narrow current-main fix in `src/plugins/provider-auth-choice.ts` that suppresses per-agent overrides only when an inherited global primary exists, preserves the no-global-default provider selection, and proves the `agents add` config write path.

## Reproduction Assessment

Yes, source-reproducible: current `agents add` writes any returned `agentModelOverride`, and current provider-auth paths return one when `setDefaultModel` is false. I did not run the CLI because this review was read-only, but the linked issue provides concrete CLI steps.

## Solution Assessment

No. The PR targets the right seam, but the best fix is conditional: inherit an existing global primary while preserving the selected provider model when no global default exists, then rebase and prove the real CLI flow.
