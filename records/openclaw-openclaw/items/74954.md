## Summary

Keep open: current main still has the reported agent model inheritance bug, and this PR targets the right surface, but the patch removes the selected provider model even when no inherited default exists and lacks after-fix real behavior proof.

## What This Changes

The PR changes provider auth/default-model handling and related tests so `setDefaultModel: false` stops returning provider default models as per-agent overrides during agent auth setup.

## Best Possible Solution

Rebase the branch and conditionally suppress `agentModelOverride` only when the original config already has an inherited primary, with coverage for both inherited-default and no-default `agents add` paths.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main returns `agentModelOverride` under `setDefaultModel: false`, and `agents add` writes that value into the agent entry.

## Solution Assessment

No. The production seam is the right place to fix this, but the PR should keep the provider-selected override when no inherited primary exists instead of removing the override for every deferred provider-auth flow.
