## Summary

Keep open. Current main still has the linked write-time model override path, but this PR is not merge-ready because it suppresses deferred provider model overrides unconditionally; that fixes the configured-global-default case while breaking setups that have no inherited `agents.defaults.model.primary`.

## What This Changes

The PR branch changes provider auth-choice/default-model handling and related tests so `setDefaultModel: false` stops returning provider defaults as per-agent model overrides during agent auth setup.

## Best Possible Solution

Keep the PR open and narrow the implementation so provider defaults are not baked into new agents when an inherited `agents.defaults.model.primary` exists, while still storing the selected provider model when no inherited primary is available. The corrected patch should cover string-form and object-form global defaults plus the no-global-default case.

## Reproduction Assessment

Yes. The linked issue gives concrete configured-global-primary steps, and current main shows the source path from `applyAuthChoice(... setDefaultModel: false)` to `agents.list[].model`; the PR regression is reproducible by following the same deferred provider setup with no inherited primary configured.

## Solution Assessment

No. The PR direction is right for the reported inherited-default case, but removing every deferred override is broader than the product contract; conditional suppression based on whether an inherited primary exists is the safer solution.
