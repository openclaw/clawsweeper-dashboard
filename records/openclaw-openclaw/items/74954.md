## Summary

Keep open. Current main still has the reported write path, and the PR fixes the inherited-default case but removes the deferred provider override even when there is no inherited `agents.defaults.model.primary`, which would make a newly configured agent fall back to the generic default instead of the selected provider model.

## What This Changes

The PR changes provider auth/default-model handling and related tests so `setDefaultModel: false` stops returning provider defaults as per-agent model overrides.

## Best Possible Solution

Keep the PR open and narrow the fix so provider defaults are suppressed only when an inherited `agents.defaults.model.primary` exists, while preserving the selected provider model when no inherited primary is available.

## Reproduction Assessment

Yes. The linked issue gives concrete configured-global-primary steps, and current main shows `agents add` persisting deferred auth overrides into `agents.list[].model`; the PR regression is source-verifiable from the no-global-default test now expecting no override.

## Solution Assessment

No. The intended direction is right for agents that already inherit a default primary, but unconditional removal is broader than the model-selection contract; the safer solution is a conditional deferred override plus focused tests for string defaults, object defaults, and no global default.
