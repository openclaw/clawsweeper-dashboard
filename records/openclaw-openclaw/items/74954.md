## Summary

Keep open: this PR targets a real `openclaw agents add` model-selection bug that still exists on current main and v2026.6.6, but the branch is not merge-ready because it drops the provider-selected model override even when there is no inherited default model to use.

## What This Changes

The PR removes deferred provider default-model overrides from auth-choice paths and rewrites related tests so new agents should inherit `agents.defaults.model`.

## Best Possible Solution

Keep the PR open only as a fix candidate: rebase it, make `agentModelOverride` suppression conditional on an existing inherited default primary, preserve the provider-selected override for no-default configs, and prove both CLI cases with focused tests plus redacted real `openclaw agents add` output.

## Reproduction Assessment

Yes. Source inspection shows current main returns a provider `agentModelOverride` under `setDefaultModel: false`, and `openclaw agents add` writes that value into `agents.list[].model`; I did not run the CLI because this review was constrained to read-only inspection.

## Solution Assessment

No. The PR targets the right seam, but the best fix is conditional: keep inheritance when a global primary exists, and keep the selected provider default as the per-agent override when there is no inherited default.
