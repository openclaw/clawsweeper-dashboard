## Summary

Keep open. The PR targets a real write-time model-inheritance bug, but its current implementation drops provider-selected agent overrides even when there is no inherited global primary, which would make newly configured agents fall back to the generic default instead of the provider model the user just set up.

## What This Changes

The PR changes provider auth/default-model handling and tests so `setDefaultModel: false` no longer returns provider defaults as per-agent model overrides.

## Best Possible Solution

Keep the PR open and narrow the fix so provider defaults are suppressed only when a string-form or object-form inherited `agents.defaults.model.primary` exists, while preserving the selected provider model when no inherited primary is available.

## Reproduction Assessment

Yes. The linked issue gives concrete configured-global-primary steps, and the current-main source shows the write path from deferred provider auth to `agents.list[].model`; the PR regression is source-verifiable by the no-global-default provider setup returning only `{ config }`.

## Solution Assessment

No. The direction is right for the inherited-default case, but unconditional removal is broader than the product contract; the safer fix is a conditional deferred override plus focused tests for string defaults, object defaults, and no global default.
