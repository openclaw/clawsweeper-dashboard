## Summary

Keep open. The linked bug is valid and current main still has the write path, but this PR suppresses provider model overrides too broadly and would drop the selected provider model when there is no inherited `agents.defaults.model.primary`.

## What This Changes

The PR changes provider auth/default-model handling and tests so `setDefaultModel: false` stops returning provider defaults as per-agent model overrides during agent creation.

## Best Possible Solution

Land a conditional fix: `agents add` should omit a per-agent model only when an inherited default primary exists, while preserving the selected provider model when no inherited primary is available.

## Reproduction Assessment

Yes. The linked issue gives concrete `agents.defaults.model.primary` steps, and the source path is direct: deferred provider auth returns `agentModelOverride`, then `agents add` writes it into `agents.list[].model`.

## Solution Assessment

No. The direction is right for the inherited-default case, but unconditional removal is broader than the product contract; the safer implementation is a conditional deferred override with focused coverage for string defaults, object defaults, and no global default.

## Review Findings

Overall correctness: patch is incorrect

Return `agentModelOverride` only when no inherited primary exists, and apply the same condition to the explicit-provider path.
