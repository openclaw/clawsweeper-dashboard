## Summary

Keep open: this is a plausible fix for the linked `openclaw agents add` model-inheritance bug, but the current branch is not merge-ready because it suppresses provider-selected per-agent model overrides even when no global primary exists and it lacks real CLI proof.

## What This Changes

The PR changes provider auth-choice default-model handling and related tests so `setDefaultModel: false` restores prior defaults and stops returning `agentModelOverride` during agent setup.

## Best Possible Solution

Land a narrow conditional fix that omits per-agent overrides only when an existing global primary should be inherited, preserves provider-selected overrides when no global default exists, and includes real CLI proof for both setups.

## Reproduction Assessment

Yes, source-reproducible. Current main’s `agents add` path calls `applyAuthChoice` with `setDefaultModel: false` and persists only `authResult.agentModelOverride`, while the PR removes that return from provider-auth paths.

## Solution Assessment

No. The PR targets the right seam, but the best fix is conditional: inherit an existing global primary while preserving the selected provider model when no global primary exists.
