## Summary

Keep open. The PR targets a real agents-add model inheritance bug, but its deferred-auth change removes provider model overrides unconditionally, which would break the current no-global-default path that still needs the selected provider model to become the new agent's explicit model.

## What This Changes

The PR changes provider auth/default-model handling and related tests so `setDefaultModel: false` stops returning provider default models as per-agent overrides.

## Best Possible Solution

Keep the branch open and narrow the fix so new agents inherit an existing global primary, while agents with no global primary still keep the provider model selected during auth.

## Reproduction Assessment

Yes. Source inspection is enough: current main persists `agentModelOverride` from deferred auth, and the PR removes that override even when `resolveAgentEffectiveModelPrimary` would find no inherited primary.

## Solution Assessment

No. The requested direction is right for the inherited-default case, but the PR should conditionalize override suppression on an existing inherited primary rather than removing the override for every deferred provider-auth flow.
