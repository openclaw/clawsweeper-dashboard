## Summary

Keep open, but this PR is not merge-ready: it addresses a real agent model-inheritance bug, yet the patch drops the selected provider model when no global default exists, the head conflicts with current main, and no real behavior proof is attached.

## What This Changes

The PR changes provider auth-choice/default-model handling and related tests so provider `defaultModel` no longer becomes a per-agent override when creating agents with `setDefaultModel: false`.

## Best Possible Solution

Implement the conditional behavior in the current `src/plugins/provider-auth-choice.ts` path: inherit an existing global primary, but keep the provider-selected model as the new agent override when no inherited default exists.

## Reproduction Assessment

Yes, from source inspection: `openclaw agents add` applies `authResult.agentModelOverride` into `agents.list[].model`, and current main's provider auth-choice path still returns the selected provider default model when `setDefaultModel` is false. I did not run the CLI because this review is read-only.

## Solution Assessment

No. The PR targets the right owner seam, but the fix is too broad; the maintainable solution is conditional so agent setup inherits an existing global default without losing the selected provider model when no global default is configured.
