## Summary

Keep open. The linked bug remains valid, but this PR removes deferred provider-model overrides too broadly and would break new-agent setup when no inherited `agents.defaults.model.primary` exists.

## What This Changes

The PR changes provider auth/default-model handling and tests so `setDefaultModel: false` no longer returns provider defaults as per-agent model overrides.

## Best Possible Solution

Merge a narrowed fix that suppresses provider overrides only when a string-form or object-form inherited default primary exists, while preserving the selected provider model for agents with no inherited primary.

## Reproduction Assessment

Yes. #24170 gives concrete configured-primary/fallback `openclaw agents add` steps, and current main shows the returned deferred override being written into the new agent entry; the PR regression is source-verifiable from the no-default provider setup returning only `{ config }`.

## Solution Assessment

No. The direction is right for inherited defaults, but unconditional removal is broader than the model-selection contract; the safer fix is conditional suppression plus focused tests for string defaults, object defaults, explicit plugin auth, and no global default.
