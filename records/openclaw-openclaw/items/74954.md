Keep this PR open, but it is not merge-ready: it targets the right `agents add` model-inheritance bug, yet the current diff drops the provider-selected model for no-global-default setups and still lacks real CLI proof.

## What This Changes

The PR changes provider auth-choice default-model handling so `setDefaultModel: false` no longer returns per-agent model overrides during agent setup.

## Best Possible Solution

Land a narrow conditional fix that omits per-agent overrides only when an existing global primary should be inherited, preserves provider-selected overrides when no global default exists, and includes real CLI proof for both setups.

## Solution Assessment

No. The PR targets the right seam, but the best fix is conditional: inherit an existing global primary while preserving the selected provider model when no global primary exists.
