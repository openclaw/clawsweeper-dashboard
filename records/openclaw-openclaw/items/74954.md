## Summary

Keep open. The underlying `agents add` bug is still valid on current `main`, but this PR removes deferred provider overrides unconditionally, which would regress the no-global-default case, and the branch is stale against current provider-auth code.

## What This Changes

The PR changes provider auth/default-model handling and focused tests so `setDefaultModel: false` stops returning provider defaults as per-agent model overrides during agent auth setup.

## Best Possible Solution

Implement conditional suppression in the current provider-auth paths: if `agents.defaults.model.primary` already exists, do not bake the provider default into the new agent; if no inherited primary exists, keep returning the selected provider model so the new agent remains runnable with the provider the user just configured.

## Reproduction Assessment

Yes. The linked issue gives concrete configured-global-primary steps, and the current source path verifies the write-time failure from `applyAuthChoice(... setDefaultModel: false)` to `agents.list[].model`.

## Solution Assessment

No. The PR’s direction is right for the inherited-default case, but suppressing every deferred override is broader than the product contract; the safer fix is conditional on whether an inherited primary exists, covering string and object default forms.

## Review Findings
