## Summary

Keep open. Current main still lacks an Accessible built-in Control UI theme; the custom tweakcn import slot helps theme customization but is not equivalent to a one-click, shipped accessibility preset, and the approved built-in-theme boundary needs maintainer product/design judgment.

## What This Changes

Request adds an Accessible Control UI theme preset with high-contrast, colour-vision-deficiency-safe dark CSS variable overrides.

## Best Possible Solution

Maintainers should decide whether accessibility warrants a built-in theme-family exception; if approved, implement it consistently through the theme contract, bootstrap resolver, Appearance and Quick Settings lists, CSS tokens, docs, changelog, and focused UI tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs checks on current main show Claw, Knot, Dash, and one browser-local tweakcn import slot, not an Accessible preset.

## Solution Assessment

Unclear until maintainers choose the product direction. The existing theme-family seam is the right technical path if a built-in preset is approved; otherwise a tweakcn import recipe or the broader theming discussion is the safer direction.
