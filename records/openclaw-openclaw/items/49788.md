## Summary

Keep open. Current main still lacks an Accessible Control UI theme preset; the broader theming issue and the shipped tweakcn custom slot are related context, but they do not settle whether OpenClaw should ship an accessibility-focused built-in theme.

## What This Changes

Request adding a high-contrast, color-vision-deficiency-safe Accessible preset to the Control UI theme picker using CSS variable overrides.

## Best Possible Solution

Maintainers should decide whether accessibility warrants a built-in theme exception; if approved, implement it consistently through the theme contract, first-paint bootstrap, Appearance and Quick Settings pickers, CSS token block, docs, changelog, and focused UI tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source checks on current main show the theme contract, first-paint bootstrap, pickers, CSS, docs, and tests omit an Accessible preset.

## Solution Assessment

Unclear until maintainers choose the product direction. The existing theme-family seam is the right technical path if approved, but the current tweakcn import slot is not the same as a reusable one-click accessibility preset.
