## Summary

Keep open: this PR addresses a real Slack direct-message route contamination bug on current main, but the proposed dispatch change still writes threaded non-main DM route updates to the base peer session instead of the prepared turn session. The patch also needs the required user-facing changelog entry before merge.

## What This Changes

The PR changes Slack DM prepare/dispatch last-route persistence so non-main `dmScope` DMs avoid `agent:main:main`, and adds focused prepare/dispatch dmScope regression tests.

## Best Possible Solution

Revise the dispatch path so non-main DM last-route updates target the prepared turn session key, preserve main-DM owner pinning only for main-session updates, add a threaded non-main DM regression, and document the user-visible Slack routing fix in the changelog.

## Reproduction Assessment

Yes. A high-confidence seam reproduction is a prepared Slack DM with non-main `lastRoutePolicy: "session"`, base `route.sessionKey`, thread-suffixed `ctxPayload.SessionKey`, and `MessageThreadId`; the PR dispatch code writes the update to the base key.

## Solution Assessment

No. The prepare-side approach is the right direction, but dispatch should use `prepared.ctxPayload.SessionKey ?? route.sessionKey` for non-main last-route updates and attach `ctx` only when updating that prepared turn session.
