## Summary

Keep this PR open; it addresses a live Slack DM routing bug, but the current head still writes non-main threaded DM route updates to the base peer session instead of the prepared turn session.

## What This Changes

The PR changes Slack DM prepare/dispatch last-route persistence to avoid updating the main session for non-main `dmScope` values and adds focused dmScope regression tests.

## Best Possible Solution

Land a corrected Slack plugin patch that updates the main route only for main-DM mode, writes non-main or threaded DM route metadata to the prepared turn session, and includes regression plus changelog coverage.

## Reproduction Assessment

Yes. A high-confidence seam reproduction is a prepared Slack DM with `lastRoutePolicy: "session"`, base `route.sessionKey`, thread-suffixed `ctxPayload.SessionKey`, and `MessageThreadId` set; the PR dispatch code writes the update to the base key.

## Solution Assessment

No. The prepare-side direction is correct, but dispatch should resolve the non-main target from `prepared.ctxPayload.SessionKey ?? route.sessionKey`, pass ctx only for that prepared turn session, and add a threaded regression plus changelog entry.
