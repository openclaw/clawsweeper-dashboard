## Summary

Keep this PR open. Current main still has the Slack DM last-route bug, and the PR is pointed at the right seam, but the dispatch-side change still targets the base route session for non-main threaded DMs and the user-facing Slack routing fix needs a changelog entry before merge.

## What This Changes

The PR gates Slack DM last-route updates in prepare and dispatch on the resolved route policy, adds dmScope regression tests, and preserves shared main-DM owner pin behavior.

## Best Possible Solution

Land a corrected Slack plugin patch that updates the main route only for true main-DM mode, writes non-main and threaded DM route metadata to the prepared turn session, and includes focused regression plus changelog coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence seam reproduction: a Slack DM thread reply under a non-main `dmScope` has a base `route.sessionKey` and a thread-suffixed `ctxPayload.SessionKey`, while the PR dispatch path writes the route update to the base key.

## Solution Assessment

No. The prepare-side direction is correct, but dispatch should resolve the non-main target from `prepared.ctxPayload.SessionKey ?? route.sessionKey`, pass ctx only for that prepared turn session, and add the missing changelog entry.
