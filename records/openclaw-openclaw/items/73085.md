## Summary

Keep open. Current main still has the Slack DM main-route contamination, and this PR is close, but its dispatch path still targets the base peer session for non-main threaded DMs and the user-facing Slack routing fix is missing a changelog entry.

## What This Changes

The PR changes Slack DM prepare/dispatch last-route persistence to use non-main route policy for isolated DM scopes and adds focused dmScope regression tests.

## Best Possible Solution

Land a corrected Slack plugin fix that updates the main route only for main-DM mode and writes non-main or threaded DM route metadata to the actual prepared turn session, with changelog coverage.

## Reproduction Assessment

Yes. A high-confidence seam reproduction is a prepared Slack DM with lastRoutePolicy: "session", base route.sessionKey, thread-suffixed ctxPayload.SessionKey, and MessageThreadId set; the PR dispatch code writes the update to the base key.

## Solution Assessment

No. The prepare-side approach is the right direction, but dispatch should resolve the non-main target from prepared.ctxPayload.SessionKey ?? route.sessionKey, pass ctx only for that prepared turn session, and add a threaded regression.
