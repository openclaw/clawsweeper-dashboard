## Summary

Keep open: the idea is useful, but the patch is not merge-ready because it exposes a no-auth `dummy/dummy` model without teaching setup/auth diagnostics that this provider intentionally needs no credentials, and the core-vs-plugin product boundary still needs maintainer judgment.

## What This Changes

The PR adds a core `dummy/dummy` deterministic gateway model, exposes it in the default-model picker, accepts it as `agents.defaults.model.primary`, and returns a fixed reply before entering the agent harness.

## Best Possible Solution

If maintainers want `dummy/dummy` in core, make it a first-class no-auth model contract across picker, setup/auth checks, and follow-on diagnostics; otherwise move this to an official plugin or plugin-API design tracked by the gateway-lite issue.
