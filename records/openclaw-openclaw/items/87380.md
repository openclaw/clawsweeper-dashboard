## Summary

## What This Changes

The PR adds `gateway.tls.extraCaCerts` as a durable gateway TLS config key, injects it into managed gateway service environment generation as `NODE_EXTRA_CA_CERTS`, and updates schema help, labels, placeholders, docs, and a focused install-plan test.

## Best Possible Solution

Land a documented opt-in CA trust path that affects only the generated managed gateway service environment, with regression coverage proving the caller env is not mutated and maintainer acceptance of the security boundary.

## Reproduction Assessment

Yes, by source inspection: current main filters existing `NODE_EXTRA_CA_CERTS` through host-env security and regenerates the macOS service env from defaults, so forced reinstall drops manual CA edits. I did not run the failing macOS update cycle live.

## Solution Assessment

Mostly, but not yet as patched. A durable config key matches one expected fix path and preserves generated service-env ownership, but the injection should not mutate the caller env and maintainers need to accept the process-wide CA trust boundary.
