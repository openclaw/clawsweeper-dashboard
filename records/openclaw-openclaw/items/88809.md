## Summary

Keep open: current main and the latest release still carry the risky `status` shell variable in macOS update/restart handoff templates, and the linked closing PR remains open rather than merged.

## What This Changes

The issue reports that macOS web UI Update Now can leave the gateway stopped because a zsh-interpreted update handoff script assigns to readonly `status`.

## Best Possible Solution

Keep this issue as the canonical bug until the linked PR or a narrow replacement removes the `status` variable from the generated macOS handoff templates and proves the live macOS restart flow.

## Reproduction Assessment

Yes at source level: current main still generates macOS handoff shell templates that assign to `status`, and zsh documents `status` as a shell-set read-only parameter. I did not run a live macOS Control UI Update Now flow in this checkout.

## Solution Assessment

Yes: the narrow maintainable fix is to rename the internal shell variable in the macOS update/restart handoff templates while preserving exit-code logging and adding regression assertions. The open linked PR is the right candidate path, but it still needs real macOS proof before the issue can close.
