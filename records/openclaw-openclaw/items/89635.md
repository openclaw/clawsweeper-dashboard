## Summary

Keep open: the state-directory permission bug is real and the terminal proof is sufficient, but the patch changes shared JSON/text helper defaults that are still reachable through the deprecated public plugin SDK compatibility surface, so it needs scoped private-state hardening or explicit maintainer SDK compatibility approval before merge.

## What This Changes

The PR changes `src/infra/json-files.ts` so `writeTextAtomic` and a new `writeJson` wrapper default parent directory creation to `0o700`, with regression tests for defaults and overrides.

## Best Possible Solution

Preserve shared helper compatibility by applying `dirMode: 0o700` only to private OpenClaw state-root writes, or explicitly approve and document the SDK-wide default tightening with focused compatibility tests.

## Reproduction Assessment

Yes. Source inspection shows the startup update-check write calls `writeJson` under `resolveStateDir()` without `dirMode`, fs-safe derives the parent mode from umask, and `replaceFileAtomic` chmods that parent directory.

## Solution Assessment

No. The PR proves the state-root problem but solves it too broadly by changing shared helper and SDK-visible defaults; a narrower private-state write fix is the safer maintainable path unless maintainers explicitly approve the compatibility break.
