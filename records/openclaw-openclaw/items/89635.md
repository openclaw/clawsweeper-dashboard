## Best Possible Solution

Preserve shared helper compatibility by applying dirMode: 0o700 only to private OpenClaw state-root writes, or explicitly approve and document the SDK-wide default tightening with focused compatibility tests.

## Reproduction Assessment

Yes. Source inspection and the fs-safe 0.3.0 contract show the update-check path writes under resolveStateDir() through writeJson without dirMode, then the dependency chmods the parent directory to the umask-derived mode.

## Solution Assessment
