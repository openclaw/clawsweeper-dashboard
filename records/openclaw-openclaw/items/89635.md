## Summary

Keep open: the PR fixes the reported private state-root permission symptom, but it does so by changing shared JSON/text helper defaults that remain reachable through a compatibility SDK subpath.

## What This Changes

The PR changes OpenClaw's shared JSON/text atomic write helpers to default parent directories to 0700 and adds regression tests for default and override behavior.

## Best Possible Solution

Preserve compatibility by applying `dirMode: 0o700` only to private OpenClaw state-root writes, or explicitly approve, document, and test the SDK-wide permission default change before merge.

## Reproduction Assessment

Yes. Source inspection shows the update-check write calls `writeJson` under `resolveStateDir()` without `dirMode`, and the pinned fs-safe dependency derives the parent mode from umask before chmodding that parent.

## Solution Assessment

No. The PR proves the state-root bug but fixes it too broadly; a narrower private-state write fix is the safer default unless maintainers explicitly approve the SDK-wide compatibility change.
