## What This Changes

The PR changes OpenClaw's shared JSON/text write helpers to default parent directories to 0700 and adds regression tests for default and explicit `dirMode` behavior.

## Best Possible Solution

Preserve shared helper compatibility by applying `dirMode: 0o700` only on private OpenClaw state-root writes, or explicitly approve and document the SDK-wide permission change with focused compatibility tests and real path proof.

## Reproduction Assessment

Yes. Source inspection shows `update-startup` writes through `writeJson`, current `writeTextAtomic` supplies a umask-derived `dirMode`, and the dependency chmods the parent directory to that supplied mode.

## Solution Assessment

No. The patch is a plausible hardening fix, but it is too broad as submitted because the safer maintainable path is to harden private state-root writes without silently changing shared helper and plugin SDK compatibility behavior.
