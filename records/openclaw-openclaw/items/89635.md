## Summary

Keep this PR open, but it is not merge-ready: the reported state-directory permission regression is source-reproducible, while the patch still changes shared writer defaults used outside the private state-root path and lacks real update-check/stat proof.

## What This Changes

The PR changes OpenClaw's JSON/text atomic write helpers to default parent directory creation to 0700, wraps writeJson, adds dirMode regression tests, and adds an Unreleased changelog entry.

## Best Possible Solution

Keep private state-root/update-check parents at 0700 via a scoped state writer or explicit private-state dirMode while preserving current shared helper behavior for non-private package, skill, and install paths.

## Reproduction Assessment

Yes for source-level reproduction: current main writes update-check.json through writeJson, and the dependency-backed atomic writer chmods the parent directory using a umask-derived default. I did not run the end-to-end stat repro in this read-only review.

## Solution Assessment

No. The private-state 0700 outcome is right, but changing shared JSON/text writer defaults is broader than the reported state-root bug; a scoped private-state fix is the narrower maintainable solution.
