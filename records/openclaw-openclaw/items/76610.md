## Summary

Keep open. Current main still writes the Matrix crypto IDB snapshot directly to the final JSON path, so the unclean-shutdown corruption risk remains valid and narrowly repairable.

## What This Changes

The issue requests atomic persistence for Matrix crypto IDB snapshots instead of direct final-path JSON writes.

## Best Possible Solution

Use the existing plugin SDK atomic JSON writer inside the current Matrix IDB file lock, add focused failure-preservation coverage, and include a user-facing changelog entry.

## Reproduction Assessment

Yes, at source level: current main routes Matrix crypto init, periodic persistence, and shutdown persistence through `persistIdbToDisk()`, which still writes directly to the target snapshot file. I did not establish a live crash-timing repro.

## Solution Assessment

Yes. Reusing `writeJsonFileAtomically()` under the existing advisory lock is the narrowest maintainable fix because the helper already provides temp-file replacement, fsync, permissions, and cleanup semantics used by nearby Matrix state files.

## Repair Work Prompt

Fix Matrix crypto IDB snapshot persistence so `persistIdbToDisk()` no longer writes JSON directly to the final snapshot path. In `extensions/matrix/src/matrix/sdk/idb-persistence.ts`, reuse `writeJsonFileAtomically()` from `openclaw/plugin-sdk/json-store` inside the existing `withFileLock()` callback, remove the redundant final-path chmod/write sequence, and preserve current lock ordering and empty-snapshot behavior. Add focused regression coverage in `extensions/matrix/src/matrix/sdk/idb-persistence.test.ts` or a narrow companion test to prove successful writes remain parseable with `0o600` permissions and that a simulated atomic-write/replacement failure leaves an existing valid snapshot intact and cleans temp files where the helper exposes that behavior. Do not change Matrix snapshot format, persistence cadence, lock options, account storage layout, or plugin/core boundaries. Add a single-line user-facing `CHANGELOG.md` fix entry for Matrix E2EE crypto-store durability. Validate with targeted Matrix IDB tests, targeted formatting, `git diff --check`, and then the changed gate in Testbox per repo policy.
