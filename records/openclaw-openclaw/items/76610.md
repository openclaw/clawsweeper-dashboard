## Summary

Keep open. Current main and v2026.5.7 still write the Matrix crypto IDB snapshot directly to the final JSON path, so the reported corruption window remains; the open related PR is scoped to awaited shutdown helpers, not atomic replacement.

## What This Changes

Request atomic replacement for Matrix crypto IndexedDB snapshot writes in `persistIdbToDisk` to avoid corrupting `crypto-idb-snapshot.json` on unclean shutdown.

## Best Possible Solution

Use the existing plugin SDK atomic JSON writer inside the existing Matrix IDB file lock, with regression coverage proving failed replacement preserves the prior valid snapshot.

## Reproduction Assessment

Yes, at source level. Current main still writes the final snapshot path directly, and a focused test can simulate replacement failure without relying on live crash timing.

## Solution Assessment

Yes. Reusing `writeJsonFileAtomically()` under the existing `withFileLock()` boundary is the narrowest maintainable fix; the sibling timer/shutdown lifecycle issue should stay separate.

## Repair Work Prompt

Fix the Matrix crypto IDB snapshot atomicity bug reported at https://github.com/openclaw/openclaw/issues/76610. In `extensions/matrix/src/matrix/sdk/idb-persistence.ts`, replace the direct final-path `fs.writeFileSync(snapshotPath, JSON.stringify(snapshot))` plus `fs.chmodSync(snapshotPath, 0o600)` inside the existing `withFileLock()` callback with the existing plugin SDK `writeJsonFileAtomically(snapshotPath, snapshot)` helper from `openclaw/plugin-sdk/json-store`. Keep the existing lock boundary and do not broaden the sibling timer/shutdown lifecycle work tracked at https://github.com/openclaw/openclaw/issues/76611 or the open async stop-helper PR. Add focused regression coverage in `extensions/matrix/src/matrix/sdk/idb-persistence.test.ts` proving the success path still restores the input data, the resulting file remains private on POSIX and includes the helper newline, and a simulated failed atomic replacement leaves the previous snapshot parseable and restorable. Add or update only narrow Matrix persistence tests; do not change snapshot schema, storage paths, config, Matrix SDK behavior, or core plugin contracts. Add a single `CHANGELOG.md` fix entry crediting the human reporter if consistent with the current changelog style.
