## Summary

Current main, and the shipped v2026.6.6 release, no longer use the non-atomic Matrix IDB snapshot JSON write path described here; Matrix crypto snapshots now persist through SQLite-backed plugin state, with legacy JSON files imported and archived.

## What This Changes

The issue asks for Matrix IDB crypto snapshots to avoid non-atomic direct JSON writes during `persistIdbToDisk`.

## Best Possible Solution

Keep the v2026.6.6 SQLite-backed Matrix crypto sidecar implementation; handle any distinct shutdown-timing concern through the separate persistence/shutdown report if it still applies.

## Reproduction Assessment

No current-main reproduction remains: the direct JSON write path described in the report is gone, and current code persists Matrix IDB snapshots through SQLite-backed plugin state. The original report was source-valid for older code, but current main and v2026.6.6 no longer match that failure path.

## Solution Assessment

Yes: the current solution is stronger than swapping one JSON write to an atomic helper because steady-state Matrix crypto snapshots no longer depend on the legacy JSON file at all. Legacy JSON snapshots are still imported and archived for upgrade continuity.
