## Summary

Keep open. Current main and the latest shipped release still write the Matrix crypto IndexedDB snapshot directly to the final JSON path, while the related locking PR only serialized access and did not make replacement atomic.

## What This Changes

The issue asks Matrix crypto IndexedDB snapshot persistence to use atomic replacement instead of writing `crypto-idb-snapshot.json` directly.

## Best Possible Solution

Use the existing plugin SDK atomic JSON writer inside the existing Matrix IDB file lock, with regression coverage proving failed replacement preserves the prior valid snapshot.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.5.7` still write the JSON snapshot directly to the target path, and a focused test can simulate an interrupted write or failed replacement without live crash timing.

## Solution Assessment

Yes. Reusing `writeJsonFileAtomically()` under the existing `withFileLock()` boundary is the narrowest maintainable fix; the sibling timer/shutdown lifecycle work should stay separate.

In `extensions/matrix/src/matrix/sdk/idb-persistence.ts`, replace the direct final-path `fs.writeFileSync(snapshotPath, JSON.stringify(snapshot)); fs.chmodSync(snapshotPath, 0o600);` sequence inside the existing `withFileLock()` callback with `writeJsonFileAtomically(snapshotPath, snapshot)` from `openclaw/plugin-sdk/json-store`. Preserve the existing lock ordering, `databasePrefix` filtering, restore behavior, no-snapshot no-op behavior, and warning semantics. Add focused Vitest coverage in the Matrix IDB persistence tests for successful atomic persistence, restrictive mode/trailing newline where practical, and failure during replacement preserving the previous valid parseable snapshot; avoid broad mocks that hide the real write contract. Do not expand this PR into event-driven crypto flushing or awaited shared-client shutdown work tracked by https://github.com/openclaw/openclaw/issues/76611. Add a single `CHANGELOG.md` Fixes entry for the Matrix crypto snapshot durability fix.
