## Summary

Keep open. The underlying stale-provider/context-window bug is still visible on current main, but this PR is not ready to merge because it changes the merge-mode provider retention contract, carries blocking accounting regressions, and mixes unrelated embedded-runner/cron terminal-error behavior into the context fix.

## What This Changes

The PR removes merge-mode preservation of orphaned generated model providers, switches several context-token lookups to provider-qualified resolution, adds focused regressions, and also changes embedded-runner and cron terminal error payload handling.

## Best Possible Solution

Rebase and narrow the branch to the context-window bug: decide the merge-mode provider deletion contract, keep provider-qualified context resolution in gateway/session/CLI/cron paths, treat stored session context as fallback rather than override, and split embedded-runner/cron terminal-error payload changes into separate work.

## Reproduction Assessment

Yes for source-level reproduction. Current main still preserves absent existing providers in merge mode and still has bare context lookups in Gateway defaults, CLI sessions, and cron accounting; I did not run tests because this review is read-only.

## Solution Assessment

No, not as currently proposed. Provider-qualified context lookup is the right direction, but this diff keeps stale persisted context authoritative in two paths and bundles unrelated empty-turn behavior changes that should be separated or reviewed independently.
