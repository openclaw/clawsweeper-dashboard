## Best Possible Solution

Land a narrowed, rebased fix that uses provider-qualified context lookup where provider and model are known, treats stored session context only as fallback, and preserves merge-mode provider retention unless maintainers approve a documented pruning contract.

No, not as currently proposed. Provider-qualified lookup is the right direction, but the branch needs to preserve merge-mode compatibility, treat persisted context as fallback, avoid unrelated failover regressions, and add real after-fix proof.
