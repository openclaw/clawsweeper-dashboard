## Summary

Keep open: current main already records fallback attribution and shows it in verbose/status/WebUI surfaces, but ordinary non-verbose chat replies still suppress the fallback marker that this issue asks users to see.

## What This Changes

The issue requests visible model attribution when a fallback model answers through the same agent/session identity on chat surfaces.

## Best Possible Solution

Define a maintainer-approved fallback attribution policy, then implement it in the reply delivery layer with channel-friendly metadata where possible and a minimal inline fallback for surfaces that only support text.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a fallback transition with `resolvedVerboseLevel: "off"` records and emits lifecycle metadata but does not prepend `Model Fallback:`, and the existing e2e test asserts that behavior.

## Solution Assessment

Unclear. The issue identifies a real UX gap, but the exact remedy needs maintainer policy on default visibility, opt-out/config shape, and whether channel metadata should be preferred over inline text.
