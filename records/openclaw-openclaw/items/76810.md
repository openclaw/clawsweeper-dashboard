## Summary

Keep open: current main still knows the winning fallback model internally, but the visible chat notice is only prepended in verbose mode, while normal default verbose-off replies remain untagged. Changing that default visibility is a UX/product decision rather than a narrow bug fix.

## What This Changes

Request to visibly tag chat replies produced by model fallback with the fallback candidate model name, via an inline prefix, configurable template, or structured channel metadata.

## Best Possible Solution

Define and implement a minimal fallback-attribution surface that uses structured channel metadata where supported and a documented inline fallback where metadata is unavailable.

## Reproduction Assessment

Yes. Source inspection on current main shows a fallback response with resolvedVerboseLevel set to off reaches final delivery without Model Fallback text, and the existing e2e test asserts that behavior.

## Solution Assessment

Unclear. The proposed inline prefix would address the reported confusion, but the repository currently treats fallback notices as verbose-only operational output, so maintainers should decide the default visibility, opt-out shape, and channel rendering policy before implementation.
