## Summary

Keep open: current main still suppresses the fallback identity marker in normal non-verbose replies, and the related open tracker/PR cover an opt-in notice rather than this issue’s default-visible or structured attribution policy.

## What This Changes

The issue requests visible attribution of fallback responses with the candidate model name so chat users can tell when a non-primary model answered.

## Best Possible Solution

Choose one canonical UX policy for fallback attribution, then implement it in the shared reply path with channel-aware rendering, docs, config schema/help updates, and regression coverage for verbose-off delivery.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a fallback transition with `resolvedVerboseLevel: "off"` records state and emits lifecycle metadata, but no fallback notice is prepended to the delivered payload; current e2e coverage asserts that behavior.

## Solution Assessment

Unclear. Inline tagging would address the confusion, but default-on versus opt-in and inline text versus structured channel metadata needs maintainer UX/privacy direction; #69399 is the narrow opt-in candidate if that policy is accepted.
