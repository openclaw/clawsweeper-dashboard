## Summary

Keep open: current main still records fallback attribution internally and exposes it in verbose/status surfaces, but normal chat replies with verbose off do not show which fallback model answered. The remaining decision is user-facing UX/policy, not a cleanup-close candidate.

## What This Changes

The issue requests visible attribution when a fallback model, rather than the session primary model, produces a chat response.

## Best Possible Solution

Add a maintainer-approved fallback identity surface, likely structured channel metadata with a minimal inline fallback for channels that cannot render metadata, and document the default plus opt-out behavior.

## Reproduction Assessment

Yes. Source inspection shows a fallback response with `resolvedVerboseLevel: "off"` reaches final delivery without `Model Fallback:` text, and the existing e2e test asserts that normal-mode behavior.

## Solution Assessment

No. The current verbose-only notice is useful diagnostics, but it does not solve the normal chat identity confusion; the best fix needs a maintainer decision on visibility, opt-out/config shape, and channel rendering.
