## Summary

Keep open: current main still suppresses fallback attribution on normal chat replies. The code records the fallback winner and emits lifecycle metadata, but the user-visible `Model Fallback:` notice is only prepended when verbose mode is enabled, while docs default verbose off and advise keeping it off in public rooms.

## What This Changes

The issue requests visible model attribution when a fallback model, rather than the originally selected model, produces a chat response.

## Best Possible Solution

Add maintainer-approved fallback identity attribution for ordinary chat replies, preferably structured channel metadata with a minimal inline fallback for channels that cannot render metadata, plus documented default and opt-out behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a fallback response with `resolvedVerboseLevel: "off"` emits fallback lifecycle metadata but reaches final delivery without `Model Fallback:` text, and the existing e2e test asserts that behavior.

## Solution Assessment

No. The current verbose-only diagnostic notice is useful plumbing, but it is not the best solution for normal chat identity attribution; the fix needs a maintained policy for visibility, opt-out/config, and channel-specific rendering.
