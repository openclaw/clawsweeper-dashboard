## Summary

Keep open: current main has internal fallback attribution and verbose/status visibility, but normal non-verbose chat replies still suppress the model-fallback marker described in the report.

## What This Changes

Request visible attribution for model-fallback replies so chat users can tell when a non-primary candidate model answered under the same agent identity.

## Best Possible Solution

Adopt a channel-aware fallback attribution policy with structured model identity, a minimal visible fallback where approved, an opt-out/config story, and docs covering privacy/noise defaults.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a fallback transition with `resolvedVerboseLevel: "off"` reaches final delivery without `Model Fallback:` text, and the existing e2e test asserts that behavior.

## Solution Assessment

No. The current verbose-only notice is useful diagnostic plumbing but does not solve normal chat identity confusion; an unconditional prefix also needs maintainer policy because verbose diagnostics are intentionally discouraged in public rooms.
