## Summary

Keep open. This is a useful focused PR for the open gateway config include-watch bug, but the current head is draft and still has a blocking startup include-watch race that can drop newer watched include paths.

## What This Changes

The PR adds dynamic `$include` file tracking to the gateway config reloader, passes the startup snapshot through managed startup, and adds regression tests for include reload behavior.

## Best Possible Solution

Keep the include-watch design, but serialize or generation-guard include watch synchronization so stale startup results cannot replace a newer validated watched set.

## Reproduction Assessment

Yes. Source inspection shows current main only watches the root config path, and the PR defect is reproducible by interleaving the unawaited startup recursive include scan with an early valid reload.

## Solution Assessment

No, not yet. The PR’s direction is the narrow maintainable fix, but the startup recursive sync needs serialization or stale-result rejection before merge.
