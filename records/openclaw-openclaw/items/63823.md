## Summary

Keep open. Current main still lacks the MiniMax manifest auth-alias mapping while auth lookup is manifest-backed, and https://github.com/openclaw/openclaw/pull/72798 is the active implementation path.

## What This Changes

Add MiniMax manifest `providerAuthAliases` for `minimax-cn` and `minimax-portal-cn` so CN endpoint auth resolves to the existing MiniMax auth providers.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/72798 or apply an equivalent maintainer patch that adds the MiniMax manifest aliases with focused regression coverage while keeping the generic auth resolver manifest-backed.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: current main has `minimax-cn` and `minimax-portal-cn` runtime aliases but no MiniMax manifest `providerAuthAliases`, so auth lookup falls back to those alias ids before credential lookup.

## Solution Assessment

Yes. The manifest-level alias map is the narrowest maintainable fix because core source and public plugin docs already define `providerAuthAliases` as the contract for provider variants sharing auth.
