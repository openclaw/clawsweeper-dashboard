## Summary

Keep open: current main has reusable channel probe surfaces, but `openclaw onboard` does not yet provide the requested interactive per-channel reachability check before finalizing setup. This is a new onboarding UX feature rather than a broken existing contract.

## What This Changes

Request to add an onboarding preflight step or flag that checks configured channel credentials, provider reachability, and gateway port reachability before setup completes.

## Best Possible Solution

Add a generic channel preflight design that reuses plugin `probeAccount`/`auditAccount` and gateway reachability checks, with clear skip/opt-in behavior and no provider-specific logic in core onboarding.

## Reproduction Assessment

Not applicable; this is a feature request, not a report of a failing current contract. Source and docs show onboard has gateway health checks and separate channel probes, but no interactive channel reachability preflight.

## Solution Assessment

Unclear; reusing the existing channel probe contracts is likely the narrowest technical path, but maintainers should decide whether this belongs in onboard, `channels add`, doctor, or an explicit opt-in prompt/flag.
