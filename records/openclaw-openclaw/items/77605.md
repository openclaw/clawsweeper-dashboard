## Summary

Keep open: current main and the latest release still have post-setup gateway/channel diagnostics, but not an interactive per-channel connection preflight or `--test-connection`-style onboarding path before config is finalized. This is aligned setup UX work, but it needs a maintainer decision on timing, opt-in behavior, and credential/gateway lifecycle before repair automation should start.

## What This Changes

Request to add an onboarding channel reachability preflight that validates channel credentials and gateway reachability during `openclaw onboard`.

## Best Possible Solution

Design a generic onboarding channel preflight that reuses plugin `probeAccount`/`auditAccount` and gateway reachability contracts, with clear skip/timeout behavior and no Telegram- or Discord-specific core logic.
