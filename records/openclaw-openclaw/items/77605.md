## Summary

Keep open: current main has reusable gateway and channel probe plumbing, but interactive `openclaw onboard` still does not expose a per-channel reachability preflight during channel setup. This is a new onboarding UX feature that needs maintainer direction.

## What This Changes

Request adds an onboard connection preflight that checks configured channel credentials and gateway reachability before setup completes.

## Best Possible Solution

Add a generic onboard channel preflight that reuses `probeAccount`/`auditAccount` and gateway reachability checks, with explicit skip/opt-in behavior and no provider-specific handshakes in core onboarding.

## Reproduction Assessment

Not applicable; this is a feature request, not a failing current contract. Source and docs show existing diagnostics but no requested interactive onboard channel preflight.

## Solution Assessment

Unclear; reusing existing channel probe contracts is likely the narrowest technical path, but maintainers should decide the UX boundary between onboard, `channels add`, `channels status --probe`, and doctor/status flows.
