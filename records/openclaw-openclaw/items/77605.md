## Summary

Keep open: current main has gateway health checks and reusable channel probe surfaces, but `openclaw onboard` still writes channel config without an interactive per-channel reachability preflight. This remains a new onboarding UX feature, not a broken existing contract.

## What This Changes

Add an onboarding preflight that tests configured channel credentials and gateway reachability before finalizing setup.

## Best Possible Solution

Add a generic onboarding channel preflight design that reuses plugin `probeAccount`/`auditAccount` and gateway reachability plumbing, with clear skip/opt-in behavior and no provider-specific Telegram or Discord logic in core.

## Reproduction Assessment

Not applicable; this is a feature request, not a failing current contract. Source and docs show live diagnostics through `openclaw status --deep`, `openclaw health --verbose`, and `openclaw channels status --probe`, but not an onboarding preflight.

## Solution Assessment

Unclear; the requested direction is technically plausible, but maintainers should decide the onboarding UX and credential/gateway lifecycle first. If accepted, the narrow path is to reuse existing plugin probe contracts rather than add provider-specific core handshakes.
