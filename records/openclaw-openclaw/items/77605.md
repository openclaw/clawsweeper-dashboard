## Summary

Keep open: current main has reusable gateway and per-channel probe surfaces, but classic `openclaw onboard` still does not offer the requested interactive channel reachability preflight before setup completes. This is a new onboarding UX/API feature, not a broken existing contract.

## What This Changes

The issue requests an interactive `openclaw onboard` preflight that checks channel credential and network reachability before setup completes.

## Best Possible Solution

Add a generic optional onboarding preflight that reuses plugin `probeAccount`/`auditAccount` and gateway reachability checks, keeps provider logic in plugins, and documents skip/failure behavior.

## Reproduction Assessment

Not applicable; this is a feature request rather than a failing current contract. Source and docs show onboarding has gateway health checks and separate channel probe commands, but no interactive channel reachability preflight.

## Solution Assessment

Unclear; reusing existing channel probe contracts is the narrowest technical path, but maintainers should decide whether the UX belongs in onboarding, `channels add`, doctor, or an explicit opt-in flag/prompt.
