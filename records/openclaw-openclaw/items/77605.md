## Summary

Keep open: current main has gateway health checks and reusable channel probe APIs, but `openclaw onboard` does not yet run an interactive per-channel reachability preflight before finishing setup. This is a new onboarding UX feature and needs maintainer direction.

## What This Changes

Add an interactive `openclaw onboard` channel reachability preflight before finalizing setup.

## Best Possible Solution

Add a generic channel preflight design that reuses plugin `probeAccount`/`auditAccount` and gateway reachability checks, with clear skip/opt-in behavior and no provider-specific logic in core onboarding.

## Reproduction Assessment

Not applicable; this is a feature request, not a report of broken current behavior. Source and docs show gateway health checks and separate channel probes, but no interactive onboarding channel reachability preflight.

## Solution Assessment

Unclear; reusing existing channel probe contracts is likely the narrow technical path, but maintainers should decide whether this belongs in onboard, `channels add`, doctor, or an explicit opt-in prompt/flag.
