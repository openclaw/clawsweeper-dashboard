## Summary

Keep open: current main has reusable live channel probe surfaces, but `openclaw onboard` still does not provide the requested interactive per-channel reachability preflight before completing setup. This is a new onboarding UX feature rather than a broken existing contract.

## What This Changes

Request to add an onboarding Connection Pre-flight step or flag that checks channel credentials and gateway reachability before setup completes.

## Best Possible Solution

Add a generic onboarding preflight that reuses existing channel `probeAccount`/`auditAccount` and gateway reachability contracts, with clear timeout, skip, and credential-handling behavior while keeping provider-specific checks inside plugins.

## Reproduction Assessment

Not applicable; this is a feature request, not a report of a failing current contract. Source and docs show current onboard has channel configuration and gateway health, but no interactive channel reachability preflight before setup completes.

## Solution Assessment

Unclear; reusing existing channel probe contracts is the narrowest technical direction, but maintainers should decide whether this belongs in onboard, `channels add`, `channels status`, `doctor`, or an explicit prompt/flag.
