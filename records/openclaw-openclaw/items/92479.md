## Summary

Current main still lacks a Zen catalog even though OpenCode Zen previously shipped curated model metadata and the live Zen endpoint now exposes discoverable IDs; the issue is valid, but maintainers should choose the supported offline seed and live-discovery policy before implementation.

## What This Changes

Restore a bundled static model catalog and add live model discovery for the OpenCode Zen provider.

## Best Possible Solution

Restore a provider-owned Zen catalog through the existing plugin SDK, retain an approved curated offline seed, enrich current IDs through authenticated live discovery, and preserve correct Zen API/base-URL routing when discovery is unavailable.

## Reproduction Assessment

Yes from source with high confidence: current main supplies no Zen catalog hooks, while the model picker consumes registered provider catalogs and the live Zen endpoint exposes 48 usable model IDs. A full runtime request was not executed during this read-only review.

## Solution Assessment

The proposed plugin-local implementation is the narrowest maintainable direction, but the exact static seed and default live-discovery policy require provider-owner approval rather than blindly copying all 48 upstream rows.
