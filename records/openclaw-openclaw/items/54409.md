## Summary

Keep open: current main and the latest release still route ordinary Feishu same-chat messages through the full sequential handler, and the linked closing PR is still open and conflicting, so the central user problem is not implemented on main yet.

## What This Changes

The issue reports that Feishu same-chat serialization prevents `messages.queue.mode = "collect"` and related active-run follow-ups from reaching shared queue handling while a prior turn is active.

## Best Possible Solution

Review, rebase, prove, and land https://github.com/openclaw/openclaw/pull/92962 or an equivalent Feishu dispatch-boundary fix that preserves first-message ordering plus `:control` and `/btw` serialization, then close this issue after the fix reaches main.
