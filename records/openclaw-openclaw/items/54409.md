## Summary

Keep open: current main and the latest release still route ordinary Feishu same-chat messages through a sequential queue around the full handler, and the only linked closing PR is still open and conflicting.

## What This Changes

Fix Feishu so `messages.queue.mode = "collect"` can batch same-chat follow-ups that arrive while an agent run is active.

## Best Possible Solution

Review, make mergeable, and land https://github.com/openclaw/openclaw/pull/92962 or an equivalent bounded Feishu plugin fix that lets ordinary active follow-ups reach the shared queue while preserving control and `/btw` ordering.

## Reproduction Assessment

Yes, source-reproducible: current Feishu dispatch awaits the same-chat sequential queue around the full handler, while core collect behavior only batches when the follow-up reaches it during an active run. I did not run a live Feishu tenant repro in this read-only review.

## Solution Assessment

Yes for keep-open routing: the best path is to review and land or replace the linked Feishu PR, not close this issue before a fix merges. A plugin-side queue-boundary repair is the right ownership boundary because Feishu blocks before the shared queue policy can act.
