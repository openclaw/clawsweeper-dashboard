- `impact:session-state`: The failure changes whether follow-up messages join an active session queue or become separate turns, affecting conversation and context continuity.

## Summary

Keep open. Current main and the latest release still serialize ordinary same-chat Feishu messages through the plugin queue before the shared active-run queue can see them, and the direct closing PR is still open and conflicting rather than merged.

## What This Changes

Fix Feishu so ordinary same-chat follow-up messages can reach the shared active-run collect/steer queue while an earlier Feishu turn is still running.

## Best Possible Solution

Land or replace the linked Feishu-owned fix so ordinary same-chat active follow-ups reach the shared queue while the first ordinary task, same-tick queued work, :control, and /btw lanes keep their ordering guarantees.

## Reproduction Assessment

Yes, at source level. Current main still awaits the same-key Feishu task around the full message handler, while the shared queue only enqueues follow-ups if a message reaches it while the run is active; I did not run a live Feishu tenant reproduction in this read-only review.

## Solution Assessment

Yes. The maintainable fix belongs at the Feishu dispatch boundary so ordinary active follow-ups can reach the existing shared queue policy, instead of adding a new core queue mode or changing global active-run semantics.
