## Summary

Keep open. Current main still keeps ordinary same-chat Feishu messages behind the per-chat sequential queue until the prior agent turn finishes, so gateway collect mode cannot see them while the session is active; the merged timeout fix for #70133 does not address this collect-mode behavior.

## What This Changes

Make the Feishu plugin honor `messages.queue.mode = "collect"` by letting same-chat follow-up messages reach the gateway while an earlier Feishu turn is active.

## Best Possible Solution

Fix the Feishu inbound dispatch boundary so normal same-chat follow-up messages can reach `dispatchReplyFromConfig` during an active run, allowing the existing gateway collect queue to batch them while preserving Feishu-specific ordering and control lanes.

## Reproduction Assessment

Yes, as a source-level reproduction. Hold the first same-key Feishu queued task open, enqueue later normal same-chat messages, and current main keeps them behind `previous.then(...)` until the first task settles, so the gateway sees `isActive=false` and chooses `run-now`.

## Solution Assessment

Yes. The narrowest maintainable direction is to adjust the Feishu inbound dispatch boundary so collect-mode follow-ups reach the existing gateway queue while preserving Feishu dedupe, ordering, `:control`, and `:btw`; a broad core change should wait for evidence that multiple channels share this gap.
