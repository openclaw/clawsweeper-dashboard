## Summary

Keep open. Current main and latest release still route ordinary Feishu same-chat messages through a sequential queue that awaits the full handler, and the open fixing pull request has not merged yet, so this issue remains the canonical active-run collect/steer bug.

## What This Changes

Feishu should let same-chat messages sent during an active run reach the shared queue so `messages.queue.mode = "collect"` can batch them instead of starting separate turns.

## Best Possible Solution

Review, prove, and land or replace https://github.com/openclaw/openclaw/pull/92962; once a Feishu active-followup fix merges, close this issue with the commit and release provenance.

## Reproduction Assessment

Yes, source-reproducible: current main still awaits the Feishu same-key sequential queue around the full handler, while shared queue policy only collects followups that arrive while the session is active. I did not run live Feishu transport proof in this read-only review.

## Solution Assessment

Yes, the best current path is a scoped Feishu plugin fix that preserves first-message ordering and control/`/btw` lanes while letting ordinary active followups reach shared queue policy. The open linked PR appears to target that boundary, but it still needs maintainer review and real behavior proof before this issue can close.
