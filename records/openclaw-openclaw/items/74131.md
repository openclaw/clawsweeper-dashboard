## Summary

Keep this PR open: the refreshed retry hardening is useful and the earlier undefined-helper/read-scope blockers are addressed, but the latest spawn preflight now runs before local target/sandbox guards and regresses the no-gateway-contact contract for forbidden spawns.

## What This Changes

The PR adds bounded retry/readiness handling for subagent spawn, announce finalization, registry sweep cleanup, focused tests, and changelog entries.

## Best Possible Solution

Keep the retry hardening, but move the readiness probe after local allowlist, sandbox, and validation guards so denied spawns return their intended local errors without contacting the gateway.

## Solution Assessment

No. The retry direction is appropriate, but the spawn preflight placement should be after local validation and immediately before the first gateway lifecycle operation.
