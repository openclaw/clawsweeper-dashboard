## Summary

Keep open. The PR targets a real subagent lifecycle resilience gap on main, but the latest patch still has merge-blocking correctness issues in the spawn readiness probe and sweep-delete retry implementation, plus a required changelog gap.

## What This Changes

This PR exports the announce transient-error classifier, adds finalize retries, adds a pre-spawn readiness probe, wraps sweep deletion in a helper, and expands subagent lifecycle tests.

## Best Possible Solution

Revise the branch so readiness handling is compatible with the lifecycle call scopes, sweep cleanup performs real bounded transient delete retries, and the fix is documented in the active changelog before targeted agents validation and the changed gate pass.

## Reproduction Assessment

Yes. The remaining PR defects are reproducible by source inspection of the PR diff against current main: the preflight uses `sessions.list` before higher-scope lifecycle calls, and the sweep helper contains only one delete attempt.

## Solution Assessment

No. The bug-fix direction is narrow and appropriate, but this implementation is not mergeable until the readiness strategy and sweep retry helper match the gateway scope and retry contracts.
