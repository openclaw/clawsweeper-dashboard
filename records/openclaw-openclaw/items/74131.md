## Summary

Keep open: the PR targets a real subagent lifecycle resilience gap, but the current patch is not merge-ready because two new retry paths use a narrower delivery-error classifier than the lifecycle readiness errors this fix is meant to absorb. The supplied live output proves a real OpenClaw runtime and parent agent turn, but it still does not exercise a real subagent lifecycle or transient lifecycle failure after the patch.

## What This Changes

The PR adds bounded readiness and lifecycle retries for subagent spawn, announce finalize, and archive sweep cleanup, with focused tests and changelog entries.

## Best Possible Solution

Use one shared gateway lifecycle retry classifier for spawn readiness, announce finalize, and sweep delete; add focused tests for a non-1006 readiness error in finalize and sweep cleanup; then rerun the changed agents checks and provide redacted real subagent-lifecycle proof.

## Reproduction Assessment

No high-confidence live reproduction was established in this review. The issue is source-reproducible: current main has single-shot spawn/finalize/sweep gateway calls, and the PR tests simulate only one transient gateway-close variant.

## Solution Assessment

No: the bounded localized retry direction is maintainable, but this implementation should use the same lifecycle retry classifier across all three changed paths before it can be considered the best fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Use lifecycle retry errors for announce finalize:** `src/agents/subagent-announce.ts:174-177`
  - body: `callGatewayForAnnounceFinalize` retries only when `isTransientAnnounceDeliveryError` matches, but that classifier omits lifecycle readiness failures that this PR's spawn preflight treats as transient, such as `handshake timeout`, `closed before connect`, `not yet ready to accept connections`, and plain `gateway closed`. Those `sessions.patch`/`sessions.delete` finalize failures still get one attempt and are then swallowed, so the announce finalize hardening is incomplete.
  - confidence: 0.84
- **[P2] Use lifecycle retry errors for sweep deletes:** `src/agents/subagent-registry-helpers.ts:107-110`
  - body: `deleteSubagentSessionWithRetry` also uses the announce-delivery classifier, so archive cleanup retries only a narrower delivery-error subset. A transient `sessions.delete` failure like `Gateway not yet ready to accept connections` or `closed before connect` immediately rethrows, the sweep keeps the run for a later pass, and the claimed immediate retry hardening is skipped.
  - confidence: 0.82
