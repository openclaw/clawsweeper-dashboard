## Summary

Keep open: the latest PR refresh addresses the earlier missing helper wiring, but it still introduces a blocking ordering regression by running an admin-scoped gateway readiness probe before local spawn authorization and sandbox guards.

## What This Changes

Adds bounded gateway readiness and retry handling for subagent spawn, announce finalization, and sweep cleanup, with focused regression tests and changelog entries.

## Best Possible Solution

Keep the retry hardening, but move the readiness probe after local allowlist, sandbox, and validation guards and immediately before the first gateway lifecycle operation, with tests proving denied spawns still make no gateway calls.

## Reproduction Assessment

Yes. Source inspection of the PR head shows the admin-scoped readiness probe runs before target-policy and sandbox guards, while current guard tests assert those forbidden paths do not call the gateway.

## Solution Assessment

No. The retry direction is appropriate, but the preflight placement is not the narrow maintainable fix because local authorization and sandbox checks should complete before any gateway handshake.
