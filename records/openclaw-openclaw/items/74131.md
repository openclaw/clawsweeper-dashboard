## Summary

Keep open: the lifecycle retry hardening is useful, but one merge-blocking ordering bug remains because the new readiness probe can contact the gateway before local spawn authorization rejects a request.

## What This Changes

The branch adds subagent gateway readiness, announce-finalize, and sweep-cleanup retry handling with focused tests and changelog entries for transient lifecycle failures.

## Best Possible Solution

Keep the retry hardening, but move the readiness probe after local allowlist, sandbox, depth, and validation guards, then lock that order with denied-spawn no-gateway regression coverage.

## Reproduction Assessment

Yes. Source inspection of the PR diff shows the new `sessions.list` readiness call before target policy and sandbox guards, while current tests assert those denied paths must not call the gateway.

## Solution Assessment

No. The retry direction is appropriate, but the readiness preflight should run only after local spawn authorization and immediately before the first gateway lifecycle operation.
