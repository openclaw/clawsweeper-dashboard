## Summary

Keep open. The PR is aimed at a real current-main subagent lifecycle weakness, but the latest head still has merge-blocking source-visible issues: a duplicate admin readiness probe before all local denial guards, lifecycle retry paths wired to a delivery classifier that misses documented readiness variants, a committed changelog conflict marker, and real behavior proof that does not exercise the changed subagent retry paths.

## What This Changes

The PR adds bounded gateway readiness and transient retry handling for subagent spawn, announce finalization, and registry sweep cleanup, with focused regression tests and changelog entries.

## Best Possible Solution

Revise this PR to keep a single post-guard readiness probe, use a shared lifecycle transient classifier for all lifecycle retry wrappers, remove the changelog marker, and add redacted live proof for a real subagent lifecycle or forced transient retry path.

## Reproduction Assessment

Yes for the PR blockers: the latest head has source-visible duplicate readiness probing, classifier gaps, and a raw changelog marker. The original transient gateway failure is source-reproducible from current main's single-shot lifecycle calls, but I did not force it live.

## Solution Assessment

No. Bounded retries are the right direction, but this branch needs the duplicate privileged probe removed, lifecycle retry classification fixed for announce and sweep cleanup, the changelog cleaned, and stronger real behavior proof before merge.
