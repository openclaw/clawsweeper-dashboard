## Summary

Keep open. Current main still lacks the requested subagent lifecycle hardening, but the latest PR head is not mergeable: the new spawn readiness probe can create the documented lower-scope-first gateway pairing failure, and the sweep cleanup helper still performs only one delete call instead of bounded immediate retry. The earlier missing-helper Greptile finding was addressed by later commits, but the remaining branch defects and missing changelog entry need fixes before merge.

## What This Changes

The PR exports the announce transient-error classifier, retries announce finalize calls, adds a pre-spawn gateway readiness probe, wraps registry sweep deletion in a helper, and expands focused subagent lifecycle tests.

## Best Possible Solution

Revise the branch so spawn readiness uses a lifecycle-safe scope strategy compatible with `callSubagentGateway`, registry sweep cleanup performs bounded immediate retries for known transient delete failures, and the user-facing fix has a changelog entry before the focused agents tests and changed gate pass.

## Reproduction Assessment

Yes. The remaining PR defects are reproducible by source inspection: the new preflight calls read-scoped `sessions.list` before later write/admin lifecycle calls despite the wrapper's scope-upgrade warning, and the sweep helper contains only one `sessions.delete` call.

## Solution Assessment

No. The bug-fix direction is narrow and appropriate, but the current patch is not the best mergeable solution until the readiness probe avoids the documented scope-upgrade hazard and the sweep helper implements real bounded retry.
