## Summary

Keep open. Current main does not already implement this lifecycle hardening, and the PR head still has merge-blocking correctness gaps: the spawn preflight uses a lower-scope readiness call before higher-scope lifecycle calls, sweep cleanup is wrapped in a helper that does not actually retry, and the user-facing fix lacks the required changelog entry. Later PR commits do appear to address the earlier Greptile undefined-helper and restored-assertion concerns, so the remaining blockers are narrower.

## What This Changes

This PR exports the announce transient-error classifier, adds announce finalize retries, adds a pre-spawn gateway readiness probe, wraps registry sweep deletion in a helper, and expands focused subagent lifecycle tests.

## Best Possible Solution

Revise the branch so readiness handling uses the same lifecycle scope it protects, sweep cleanup performs bounded immediate transient delete retries, and the active changelog records the user-facing fix.

## Reproduction Assessment

Yes. Source inspection against current main and the PR diff reproduces the defects: `sessions.list` is read-scoped while later lifecycle calls require write/admin, and the new sweep helper has no retry loop.

## Solution Assessment

No. The bug-fix direction is narrow and appropriate, but this implementation should be adjusted to match the gateway scope contract and the advertised bounded retry behavior before merge.
