## Summary

Keep open. The PR is a useful focused fix attempt, but the latest branch is still not mergeable because the new spawn readiness preflight can seed the wrong gateway scope before the real lifecycle call, the sweep delete helper still performs only one delete, and the user-facing fix is missing the required changelog entry.

## What This Changes

The PR exports the announce transient classifier, retries announce finalize calls, adds a pre-spawn gateway readiness probe, wraps registry sweep deletion in a helper, and expands focused subagent lifecycle tests.

## Best Possible Solution

Revise the branch so spawn readiness retries the idempotent `agent` start or otherwise uses a scope-safe strategy, make sweep cleanup perform real bounded immediate retries for known transient delete failures, add the required changelog entry, and then rerun the focused agents tests plus the changed gate.

## Reproduction Assessment

Yes. The remaining PR defects are reproducible by source inspection: the new readiness preflight is a `sessions.list` call before higher-scope lifecycle methods, and the new sweep helper has exactly one `sessions.delete` call.

## Solution Assessment

No. The bug-fix direction is sound, but the current patch is not the best mergeable solution until readiness avoids the gateway scope-upgrade trap and sweep cleanup actually retries transient delete failures.

## Review Findings

Full review comments:

- **[P2] Retry readiness without a read-scoped preflight:** `src/agents/subagent-spawn.ts:560-564`
  - body: The new preflight opens `sessions.list` before the real `agent` lifecycle call. `sessions.list` is read-scoped, while the later spawn/delete calls need write/admin scopes, and `callSubagentGateway` already documents that a lower-tier first connection can make later scope upgrades fail with pairing errors.
