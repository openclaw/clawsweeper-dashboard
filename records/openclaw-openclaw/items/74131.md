Keep open. The current main code still lacks the requested lifecycle hardening, and the PR has addressed the earlier missing-helper/test-assertion comments, but the latest head still has blocking correctness issues plus insufficient real subagent-lifecycle proof.

## What This Changes

The PR adds bounded gateway readiness probing and transient retry handling for subagent spawn, announce finalization, and registry sweep cleanup, with focused tests and changelog entries.

## Best Possible Solution

Revise this PR to remove the early duplicate readiness probe, use a shared lifecycle retry classifier for finalize and sweep cleanup, clean the changelog, and add redacted live proof for a real subagent lifecycle or forced transient retry path.

## Reproduction Assessment

Yes for the PR blockers: the latest head has source-visible duplicate readiness probing, a raw changelog conflict marker, and a retry-classifier mismatch. The original transient gateway failure is source-reproducible from current main's single-shot lifecycle calls, but I did not force it live.

## Solution Assessment

No. Bounded retries are the right direction, but this branch needs the duplicate privileged probe removed, lifecycle retry classification fixed, changelog debris cleaned, and stronger real subagent-lifecycle proof before merge.

  - body: This admin-scoped readiness probe runs before target-policy and sandbox checks, while the later post-guard probe remains.
