## Summary

Keep open: current main and the latest release still use `zh-CN` and `zh-TW` as canonical Control UI and wizard locale IDs, while script-based values are only partially recognized as inputs in the wizard and docs publish metadata. The request remains a compatibility and product-contract migration, not stale cleanup or an already-implemented change.

## What This Changes

Rename Chinese locale identifiers from `zh-CN`/`zh-TW` to `zh-Hans`/`zh-Hant` across UI, wizard, detection, docs, tests, and stored preference compatibility.

## Best Possible Solution

If maintainers approve the contract change, land one coordinated compatibility migration that makes `zh-Hans`/`zh-Hant` canonical while preserving legacy aliases for stored UI values, wizard env inputs, docs routing, generator assets, and tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a locale-code contract enhancement. Source, tests, docs, related PR discussion, and release inspection give high-confidence proof that current main still treats `zh-CN` and `zh-TW` as canonical IDs.

## Solution Assessment

Unclear until maintainers set compatibility policy. A direct rename is not enough; the maintainable solution needs legacy aliases, stored preference migration, wizard env handling, docs routing, and generator/test alignment.
