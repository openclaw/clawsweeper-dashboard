## Summary

Keep open: current main and the latest shipped release still use `zh-CN` and `zh-TW` as canonical Control UI locale IDs, while the current wizard intentionally accepts script-code inputs only as aliases back to the region-coded IDs. This is a compatibility/product migration, not a stale cleanup item.

## What This Changes

Rename Chinese locale identifiers from `zh-CN`/`zh-TW` to script-based `zh-Hans`/`zh-Hant` across OpenClaw i18n surfaces.

## Best Possible Solution

If maintainers approve the contract change, migrate canonical IDs in one coordinated compatibility change across Control UI, wizard env handling, docs sync, docs, tests, workflows, and storage aliases while preserving `zh-CN`/`zh-TW` legacy compatibility.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a locale-contract enhancement. Source, docs, tests, release inspection, and related closed PRs give high-confidence proof that current main still treats `zh-CN` and `zh-TW` as canonical IDs.

## Solution Assessment

Unclear until maintainers set compatibility policy. The direction is reasonable, but the maintainable solution needs legacy aliases, stored preference migration, env handling, docs routing, and redirect policy rather than a direct rename.
