## Summary

Keep open: current main still uses `zh-CN` and `zh-TW` as the canonical Chinese locale IDs across the Control UI, wizard, generator, tests, and user-facing docs. ## What This Changes

Rename Chinese locale identifiers from `zh-CN`/`zh-TW` to `zh-Hans`/`zh-Hant` across runtime locale contracts, generated locale files, docs, browser detection, and saved preferences.

## Best Possible Solution

If maintainers approve the contract change, make `zh-Hans` and `zh-Hant` canonical across the UI, wizard, generators, tests, docs, and published docs metadata while preserving legacy `zh-CN`/`zh-TW` aliases and storage migration.

## Solution Assessment
