## Summary

Keep open: current main and release v2026.6.5 still use `zh-CN` and `zh-TW` as canonical Control UI and wizard locale IDs. Script tags are accepted only as wizard input aliases and appear in docs publishing metadata, while prior direct rename work closed unmerged; changing the canonical identifiers therefore remains a compatibility and product-policy decision.

## What This Changes

Adopt `zh-Hans` and `zh-Hant` as the canonical Simplified and Traditional Chinese locale identifiers across OpenClaw i18n surfaces.

## Best Possible Solution

Set one script-tag canonical policy across UI, wizard, and generated localization tooling, migrate persisted Control UI values, and retain region tags only as deliberate legacy input aliases or redirects where shipped contracts require them.

## Reproduction Assessment

Not applicable as a bug reproduction: current main consistently implements its existing region-based locale contract. Source, tests, history, and release inspection provide high-confidence evidence of the present behavior.

## Solution Assessment

No, a mechanical repository-wide rename is not the best solution by itself. The maintainable approach needs coordinated canonicalization, stored-preference migration, legacy input normalization, generated-artifact continuity, and focused upgrade tests.
