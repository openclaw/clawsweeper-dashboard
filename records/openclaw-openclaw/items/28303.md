## Summary

Keep open: current main and the latest release still use `zh-CN` and `zh-TW` as canonical Control UI and wizard locale IDs, while script-code handling is limited to wizard aliases and docs publish metadata. The exact rename attempts were closed unmerged, so the remaining work is a compatibility and product-policy decision rather than completed cleanup.

## What This Changes

Rename OpenClaw Chinese locale identifiers from region-based `zh-CN`/`zh-TW` to script-based `zh-Hans`/`zh-Hant` across UI, wizard, docs, detection, and persisted locale storage.

## Best Possible Solution

If maintainers approve the contract change, migrate canonical IDs in one compatibility pass while preserving legacy storage, env inputs, docs publish behavior, redirects, translation-memory continuity, and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a locale-code contract enhancement. Source, docs, tests, release-tag inspection, and related closed PRs give high-confidence proof that current main still treats `zh-CN` and `zh-TW` as canonical IDs.

## Solution Assessment

Unclear until maintainers set compatibility policy. The direction is reasonable, but a direct rename is not enough; the maintainable fix needs aliases, stored-preference migration, docs publish and redirect policy, and regression coverage.
