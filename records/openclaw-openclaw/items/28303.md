## Summary

Keep open: current main still uses `zh-CN` and `zh-TW` as the Control UI locale contract, generator input, persisted settings boundary, tests, workflow metadata, and docs copy. The exact implementation PR at https://github.com/openclaw/openclaw/pull/28354 was closed unmerged, and the docs-related PRs/search hits do not implement this runtime rename.

## What This Changes

Rename Chinese locale identifiers from region-based `zh-CN`/`zh-TW` to script-based `zh-Hans`/`zh-Hant`, with migration for existing saved preferences.

## Best Possible Solution

Make `zh-Hans` and `zh-Hant` canonical for the Control UI runtime, picker, generator, workflow, tests, and docs copy while normalizing legacy `zh-CN`/`zh-TW` at browser-detection and storage boundaries.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a locale-contract enhancement. Source inspection gives high confidence that current main still exposes and accepts `zh-CN`/`zh-TW`, not `zh-Hans`/`zh-Hant`, at the supported-locale boundary.

## Solution Assessment

Yes directionally: a canonical rename with backwards-compatible alias normalization is the maintainable technical path. Maintainer approval is still needed because it changes a persisted/public locale contract and intersects the docs publish route shape.
