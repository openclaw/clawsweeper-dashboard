## Summary

Keep open: current main and the latest release still use `zh-CN` and `zh-TW` as canonical Control UI and wizard locale IDs, while docs publishing already mixes script-language metadata with region-coded generated directories. This is a real compatibility/product migration question, not an implemented or stale cleanup item.

## What This Changes

The issue requests changing canonical Chinese locale identifiers from region-coded `zh-CN`/`zh-TW` to script-coded `zh-Hans`/`zh-Hant`.

## Best Possible Solution

If maintainers approve the contract change, migrate canonical IDs in one coordinated compatibility change across Control UI, wizard env handling, docs sync, docs, tests, workflows, and storage aliases while preserving `zh-CN`/`zh-TW` legacy compatibility.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a locale-contract enhancement. Source, tests, docs, release inspection, and related closed PRs give high-confidence proof that current main still treats `zh-CN` and `zh-TW` as canonical IDs.

## Solution Assessment

Unclear until maintainers set compatibility policy. The direction is reasonable, but the maintainable solution needs legacy aliases, stored preference migration, env handling, docs routing, and redirect policy rather than a direct rename.
