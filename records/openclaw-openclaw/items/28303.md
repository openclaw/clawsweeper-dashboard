Rename Chinese i18n locale identifiers from `zh-CN`/`zh-TW` to `zh-Hans`/`zh-Hant` across Control UI, wizard, docs, storage, and tests.

## Best Possible Solution

If maintainers approve the contract change, migrate canonical IDs in one coordinated compatibility pass across Control UI, wizard env handling, docs sync, docs, tests, and storage aliases while preserving `zh-CN`/`zh-TW` legacy inputs.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a locale-code contract enhancement. Source, tests, docs, and release inspection give high-confidence proof that current main still treats `zh-CN` and `zh-TW` as canonical IDs.

## Solution Assessment

Unclear until maintainers set compatibility policy. The direction is reasonable, but the maintainable fix needs legacy aliases, stored preference migration, wizard env handling, docs routing, and redirect policy rather than a direct rename.
