Keep open: current main and the latest release still use `zh-CN` and `zh-TW` as canonical locale IDs across Control UI runtime, generation, wizard, docs, and workflow surfaces. ## Best Possible Solution

If maintainers approve the contract change, make `zh-Hans` and `zh-Hant` canonical across Control UI runtime, generator metadata, wizard APIs, workflow matrices, docs publish routing, and tests while preserving `zh-CN`/`zh-TW` as backward-compatible aliases with storage migration.
