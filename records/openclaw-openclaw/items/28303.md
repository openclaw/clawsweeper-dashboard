## Summary

Keep open: current main and the latest release still use `zh-CN` and `zh-TW` as canonical Control UI and documented locale IDs, while the docs publish pipeline already has a separate script-language-versus-directory mapping. The linked docs rename PR was closed unmerged, and the open zh-TW terminology PR is about translation quality, not renaming the public locale contract.

## What This Changes

The issue requests changing Chinese locale identifiers from region-coded `zh-CN`/`zh-TW` to script-coded `zh-Hans`/`zh-Hant`, including migration for saved preferences and docs/runtime wiring.

## Best Possible Solution

If maintainers approve the contract change, make `zh-Hans` and `zh-Hant` canonical across Control UI, wizard, generated metadata, docs publishing, and tests while preserving `zh-CN`/`zh-TW` aliases with storage and URL migration.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a locale-contract enhancement. Source and release inspection provide high-confidence proof that `zh-CN` and `zh-TW` remain canonical today.

## Solution Assessment

Yes directionally, but not as an autonomous cleanup. The maintainable path is a coordinated canonical-ID migration with backward-compatible aliases and docs routing policy after maintainer approval.
