## Summary

Keep this issue open. Current main still uses `zh-CN` and `zh-TW` as the Control UI runtime locale IDs across the type contract, lazy registry/imports, browser-language mapping, saved-settings validation, generator metadata, workflow matrix, docs, and focused tests. The latest release still has the same contract, and the related PRs in the provided context are closed and unmerged.

## What This Changes

Rename the Control UI Chinese locale identifiers from `zh-CN`/`zh-TW` to canonical script-based `zh-Hans`/`zh-Hant`, including detection, persistence migration, generator, docs, workflow, selector, and tests.

## Best Possible Solution

Make `zh-Hans` and `zh-Hant` the canonical Control UI runtime IDs, preserve existing translated content, normalize legacy saved values, and update the generator, workflow, docs, and focused tests. If maintainers intentionally want region-coded runtime IDs to remain stable, record that product/API decision here instead.
