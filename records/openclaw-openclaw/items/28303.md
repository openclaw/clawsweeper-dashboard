## Best Possible Solution

If maintainers approve the contract change, migrate canonical IDs in one coordinated compatibility pass across UI registry/types, wizard env handling, docs sync, docs, tests, and storage aliases while preserving `zh-CN`/`zh-TW` as legacy inputs.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a locale-code contract enhancement. Source, docs, tests, release inspection, and prior closed PRs give high-confidence proof that current main still treats `zh-CN` and `zh-TW` as canonical IDs.

## Solution Assessment

Unclear until maintainers set compatibility policy. The requested direction is reasonable, but the maintainable fix needs legacy aliases, stored preference migration, wizard env handling, docs routing, and redirect policy rather than a direct rename.
