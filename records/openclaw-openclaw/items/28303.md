Keep open: current main and v2026.5.20 still use `zh-CN` and `zh-TW` as canonical Control UI and wizard locale IDs. Script-code handling exists only as wizard aliases and docs publish metadata, and the exact rename PRs were closed unmerged, so this remains a compatibility/product decision.

## What This Changes

The issue requests changing OpenClaw’s Chinese locale contract from region-coded identifiers to script-coded identifiers across UI, wizard, docs, locale files, browser detection, and persisted preferences.

## Best Possible Solution

If maintainers approve the contract change, migrate canonical IDs in one compatibility pass while preserving legacy `zh-CN`/`zh-TW` storage, env inputs, docs redirects, generated paths, and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a locale-code contract enhancement. Source, tests, docs, release-tag inspection, and related closed PRs give high-confidence proof that current main still treats `zh-CN` and `zh-TW` as canonical IDs.

## Solution Assessment

Unclear until maintainers set compatibility policy. The direction is reasonable, but a direct rename is not enough; the maintainable fix needs aliases, stored-preference migration, docs publish and redirect policy, and regression coverage.
