## Summary

Keep open: current main still uses `zh-CN` and `zh-TW` as canonical Control UI and wizard locale IDs, while docs publishing already separates Mintlify script-code metadata from generated region-coded directories. This is a compatibility/product migration, not stale cleanup.

## What This Changes

Request to make `zh-Hans` and `zh-Hant` the canonical Chinese locale IDs instead of `zh-CN` and `zh-TW`, including loaders, detection, stored preferences, docs, and display names.

## Best Possible Solution

If maintainers approve the contract change, migrate canonical IDs in one coordinated compatibility change across Control UI, wizard, docs sync, docs, tests, workflows, and storage/env aliases while preserving `zh-CN`/`zh-TW` legacy compatibility.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a locale-contract enhancement. Source and release inspection give high-confidence proof that current main still treats `zh-CN` and `zh-TW` as canonical IDs.

## Solution Assessment

Unclear until maintainers set compatibility policy. The direction is reasonable, but the maintainable solution must define legacy aliases, stored preference migration, env handling, and docs redirects rather than applying a direct rename.
