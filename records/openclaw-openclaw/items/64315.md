## Summary

Keep open. Current main still lacks an opt-in overflow-scrolling or marquee-style status display in the sampled Control UI and TUI status surfaces, and the related topbar status issue is complementary rather than a replacement.

## What This Changes

Request support for accessible horizontal overflow scrolling for long status text in constrained OpenClaw UI/status surfaces.

## Best Possible Solution

Define a small accessible overflow-status helper, then opt it into one maintainer-selected UI or TUI status surface before broadening it.

## Reproduction Assessment

Not applicable: this is a feature request rather than a broken existing behavior. Source inspection on current main gives high confidence that sampled status surfaces do not provide overflow-scrolling behavior.

## Solution Assessment

Unclear until maintainers choose the first surface and motion/accessibility policy. A reusable opt-in helper is the maintainable direction; a broad always-moving marquee across all status text would not be a good first implementation.
