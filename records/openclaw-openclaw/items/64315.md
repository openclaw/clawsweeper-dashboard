## Summary

Keep open. Current main still lacks an opt-in overflow-scrolling or marquee-style display for long status text in the sampled Control UI and TUI status surfaces, and the related topbar status issue remains complementary rather than a superseding fix.

## What This Changes

Add an optional accessible scrolling or marquee-style overflow mode for long status information in constrained OpenClaw UI, TUI, plugin status, or banner surfaces.

## Best Possible Solution

Define a small accessible opt-in overflow-status helper, then wire it into one maintainer-selected status surface before broadening it; coordinate with https://github.com/openclaw/openclaw/issues/13670 if the topbar becomes the first target.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection of the Control UI topbar/header/context notice and TUI footer gives high confidence that current main still uses static, wrapped, clipped, or ellipsized text rather than opt-in overflow scrolling.

## Solution Assessment

Unclear until maintainers choose the first surface and motion/accessibility policy. A reusable opt-in helper that honors reduced motion and pauses on hover/focus is the maintainable direction; a broad always-moving marquee would not be a good first implementation.
