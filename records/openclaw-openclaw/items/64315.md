## Summary

Keep open. Current main still does not implement optional overflow-scrolling or marquee-style status text for constrained Control UI or TUI status surfaces, and the related topbar model/token request is complementary content work rather than a replacement for this overflow behavior.

## What This Changes

Request optional auto-scrolling or marquee-style overflow behavior for long status/info text in constrained OpenClaw UI and TUI status surfaces.

## Best Possible Solution

Define a small accessible, opt-in overflow-status text helper for the chosen surface, then apply it first to one high-value constrained status area. Coordinate with #13670 if model/token content is added to the topbar, but keep the overflow behavior as a reusable UI/TUI capability rather than bundling it into that content request.

## Reproduction Assessment

Not applicable as a feature request rather than a failure report. The current-main verification path is high-confidence source/docs inspection showing the requested overflow-scrolling behavior is absent from the relevant Control UI and TUI status surfaces.

## Solution Assessment

Unclear until maintainers choose target surfaces and motion policy. The maintainable direction is a reusable opt-in helper with reduced-motion and pause behavior, not a broad automated marquee rollout across every status-like element.
