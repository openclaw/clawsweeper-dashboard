## Summary

Keep open. Current main still lacks an opt-in overflow-scrolling or marquee-style status display for constrained Control UI or TUI status surfaces, and the related open topbar model/token request is complementary content work rather than a replacement for the overflow behavior.

## What This Changes

Requests an accessible opt-in horizontal scrolling display for long status text in constrained Control UI, WebChat, TUI, or plugin status surfaces.

## Best Possible Solution

Define a small reusable accessible overflow-status text helper, then opt it into one or two maintainer-chosen high-value status surfaces.

## Reproduction Assessment

Not applicable as a feature request rather than a reported failure. The current-main verification path is source inspection of the Control UI topbar, compact text CSS, chat controls, and TUI footer, which still lack optional horizontal overflow scrolling.

## Solution Assessment

Unclear until maintainers choose the target surfaces and motion policy. A reusable opt-in helper with reduced-motion and pause behavior is the maintainable direction; a broad marquee rollout across all status text would not be the best first fix.
