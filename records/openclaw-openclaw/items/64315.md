## Summary

Keep open: current main still lacks an opt-in overflow-scrolling status display, while sampled Control UI, WebChat, and TUI status surfaces remain static, wrapped, ellipsized, or single-line text. The related topbar model/token request is complementary, not a superseding implementation.

## What This Changes

The issue requests an optional accessible horizontal scrolling or marquee-style display for overflowing status information in constrained OpenClaw UI surfaces.

## Best Possible Solution

Define a small accessible opt-in overflow-status helper, then wire it into one maintainer-selected status surface before broadening it to other UI or TUI regions.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main lacks opt-in marquee or horizontal overflow scrolling for the sampled status surfaces.

## Solution Assessment

Unclear until maintainers choose the first target surface and accessibility policy. A reusable opt-in helper that respects reduced motion and pauses on hover/focus is preferable to a broad always-moving marquee.
