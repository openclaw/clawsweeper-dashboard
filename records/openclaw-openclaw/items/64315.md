## Summary

Keep open. Current main still lacks an opt-in overflow-scrolling or marquee-style display for long status text in the Control UI topbar and TUI footer, and the related open topbar model/token request is complementary content work rather than a replacement for this overflow behavior.

## What This Changes

Add optional marquee-style horizontal scrolling for long status or info text in constrained OpenClaw UI surfaces.

## Best Possible Solution

Define a small reusable accessible overflow-status text helper, then opt it into one or two maintainer-chosen high-value Control UI or TUI status surfaces.

## Reproduction Assessment

Not applicable: this is a feature request rather than broken existing behavior. Source inspection on current main shows static/truncating status/header/footer paths and no status-marquee helper.

## Solution Assessment

Unclear until maintainers choose the first surfaces and motion policy. A reusable opt-in helper with reduced-motion and hover/focus pause behavior is the maintainable direction; a broad marquee rollout across all status text is not the best first fix.
