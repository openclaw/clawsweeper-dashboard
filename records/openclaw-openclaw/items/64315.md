## Summary

Keep open. Current main still lacks an opt-in scrolling or marquee-style overflow behavior for long status text in the sampled Control UI, chat, and TUI status surfaces, and the related topbar status request is complementary rather than a superseding implementation.

## What This Changes

Request support for an accessible opt-in scrolling display mode when long status information overflows constrained UI surfaces.

## Best Possible Solution

Define a small accessible opt-in overflow-status helper, then wire it into one maintainer-selected status surface before broadening it; coordinate with https://github.com/openclaw/openclaw/issues/13670 if the topbar becomes the first target.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection of the Control UI topbar, chat context notice, and TUI footer shows current main still uses static, wrapped, or ellipsized status text rather than opt-in overflow scrolling.

## Solution Assessment

Unclear until maintainers choose the first surface and accessibility policy. A reusable opt-in helper that respects reduced motion and pauses on hover/focus is the maintainable direction; a broad always-moving marquee would not be the best first implementation.
