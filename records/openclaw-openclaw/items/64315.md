## What This Changes

Request to add optional accessible horizontal scrolling or marquee behavior for long status/info text in constrained UI, TUI, and plugin-capable status regions.

## Best Possible Solution

Define a small accessible opt-in overflow-status helper, then wire it into one maintainer-selected status surface before broadening it to other UI or TUI regions.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection of current main gives high confidence that sampled status surfaces do not yet have opt-in horizontal overflow scrolling.

## Solution Assessment

Unclear until maintainers choose the first target surface and accessibility policy. A reusable opt-in helper that respects reduced motion and pauses on hover/focus is preferable to a broad always-moving marquee.
