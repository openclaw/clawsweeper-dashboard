## Summary

Keep open. Current main still renders the relevant Control UI topbar/header text and TUI footer as static, hidden, or ellipsized content rather than offering an opt-in overflow-scrolling status display, and the related open topbar request at https://github.com/openclaw/openclaw/issues/13670 is complementary content work rather than a replacement for this behavior.

## What This Changes

Request an optional marquee-style overflow display for long status or information text in constrained OpenClaw UI surfaces.

## Best Possible Solution

Define a small reusable accessible overflow-status helper, then opt it into one or two maintainer-selected Control UI or TUI status surfaces and coordinate any topbar content with https://github.com/openclaw/openclaw/issues/13670.

## Reproduction Assessment

Not applicable: this is a feature request rather than a broken existing behavior. Source inspection on current main confirms the sampled Control UI and TUI status surfaces do not have a status-marquee or overflow-scrolling helper.

## Solution Assessment

Unclear until maintainers choose the first surface and motion policy. A reusable opt-in helper with reduced-motion and hover/focus pause behavior is the maintainable direction; a broad marquee rollout across all status text would not be the best first fix.
