## Summary

Keep open. Current main still does not provide an opt-in overflow-scrolling or marquee-style display for long status text in the sampled Control UI and TUI status surfaces, and the related topbar status issue remains complementary rather than a superseding fix.

## What This Changes

Request to add an optional scrolling or marquee-style overflow display for long status/info text in constrained OpenClaw UI surfaces.

## Best Possible Solution

Define a small accessible opt-in overflow-status helper, then wire it into one maintainer-selected status surface before broadening it to other UI or TUI regions.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection on current main gives high confidence that sampled status surfaces still use static, clipped, or hidden text rather than opt-in overflow scrolling.

## Solution Assessment

Unclear until maintainers choose the first surface and motion/accessibility policy. A reusable opt-in helper is the maintainable direction; a broad always-moving marquee across status text would not be a good first implementation.
