## Summary

Keep open. Current main still lacks an opt-in overflow-scrolling or marquee-style status display in the Control UI topbar or TUI footer, and the related open topbar model/token request is complementary content work rather than a replacement for this overflow behavior. Because the request spans multiple UI surfaces and motion/accessibility policy, it needs maintainer product scoping before implementation.

## What This Changes

Request adding opt-in horizontal overflow scrolling or marquee behavior for long status/info text in constrained OpenClaw UI surfaces.

## Best Possible Solution

Define a small reusable accessible overflow-status text helper, then opt it into one or two maintainer-chosen high-value Control UI or TUI status surfaces.

## Reproduction Assessment

Not applicable: this is a feature request rather than broken existing behavior. Current-main source inspection confirms the relevant topbar and TUI footer paths remain static/truncating and no status-marquee helper exists.

## Solution Assessment

Unclear until maintainers choose the first surfaces and motion policy. A reusable opt-in helper with reduced-motion and hover/focus pause behavior is the maintainable direction; a broad marquee rollout across all status text is not the best first fix.
