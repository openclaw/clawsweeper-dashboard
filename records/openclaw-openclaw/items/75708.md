## Summary

Keep open because current main still relists the full task registry inside the per-task maintenance loop, and PR #75709 is an open same-author implementation candidate with closing syntax.

## What This Changes

The issue requests reusing the existing task snapshot during task registry maintenance instead of relisting every task record per task, with regression coverage.

## Best Possible Solution

Review and land PR #75709, or an equivalent narrow patch, while keeping the final fresh snapshot for orphaned parent-owned ACP session cleanup.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to seed many active task records, run runTaskRegistryMaintenance with a counting listTaskRecords runtime, and observe one initial listing plus one listing per processed task before the final orphan cleanup listing.

## Solution Assessment

Yes. Reusing the already captured tasks snapshot for cleanupTerminalAcpSession(current, tasks) is the narrowest maintainable fix because that helper only needs the supplied task array for child-session activity checks.
