## Summary

Keep open because current main still relists the full task registry inside the per-task maintenance loop, and open PR #75709 targets this exact fix but is not merged.

## What This Changes

The issue requests eliminating quadratic task registry maintenance by reusing the existing maintenance task snapshot during per-task ACP cleanup.

## Best Possible Solution

Review and land PR #75709, or an equivalent narrow patch, keeping the final fresh snapshot for orphaned ACP cleanup and adding the counting regression test.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to inject a counting `listTaskRecords` runtime with many current task records and run `runTaskRegistryMaintenance`; current main calls the full listing once initially, once per processed task, and once for final orphan cleanup.

## Solution Assessment

Yes. Passing the already captured `tasks` snapshot to `cleanupTerminalAcpSession(current, tasks)` is the narrowest maintainable fix because the helper only needs that array for active child-session checks, while the final orphan cleanup can still use a fresh snapshot.
