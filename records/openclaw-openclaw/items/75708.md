## Summary

Keep open. Current main still has the reported quadratic maintenance path, and an open same-author PR explicitly targets this issue, so cleanup should wait for that PR or an equivalent fix to land.

## What This Changes

Request to avoid O(n²) task registry maintenance by reusing the maintenance task snapshot for terminal ACP cleanup instead of relisting all task records per task.

## Best Possible Solution

Review and land PR #75709, or an equivalent narrow patch, so terminal ACP cleanup reuses the initial maintenance snapshot while keeping the final fresh snapshot for orphaned parent-owned ACP sessions and adding the counting regression test.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to configure many task records with a counting listTaskRecords maintenance runtime and run runTaskRegistryMaintenance; current main lists once for the initial sweep, once per task before terminal ACP cleanup, and once again for final orphan cleanup.

## Solution Assessment

Yes. Reusing the already captured tasks snapshot for cleanupTerminalAcpSession(current, tasks) is the narrowest maintainable fix because the helper only needs the supplied task array for child-session activity checks.
