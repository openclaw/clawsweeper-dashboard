## Summary

Keep open. Current main still has the reported O(n^2) maintenance pattern, and the linked same-author PR remains open and unmerged, so this issue should stay as the canonical tracker until a corrected patch lands.

## What This Changes

The issue requests that task registry maintenance avoid relisting the full task registry once per task during terminal ACP cleanup.

## Best Possible Solution

Land a revised PR #75709, or equivalent narrow patch, that removes the full per-task registry relist while keeping terminal ACP cleanup based on a current same-child-session guard, with focused regression coverage and the user-facing changelog entry.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to inject a counting listTaskRecords runtime, seed many task records, and run runTaskRegistryMaintenance(); current main will list once for the sweep, once per eligible task, and once for final orphan cleanup.

## Solution Assessment

The requested direction is right, but the safest implementation should not make the active-task guard stale. A current indexed lookup for the child session, or another current narrow view, is a better fix than blindly reusing only the sweep-start array.
