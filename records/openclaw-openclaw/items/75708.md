## Summary

Keep open. Current main still calls `listTaskRecords()` inside the per-task maintenance loop, and that helper clones, sorts, and maps the whole registry each time. The same-author PR #75709 is open, unmerged, and explicitly targets this issue with closing syntax, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Avoid repeated full task-registry relisting during terminal ACP session cleanup in `runTaskRegistryMaintenance` and add regression coverage for the maintenance pass.

## Best Possible Solution

Review and land PR #75709, or an equivalent narrow patch, so `runTaskRegistryMaintenance` reuses the initial task snapshot for terminal ACP cleanup while preserving the final orphan-cleanup snapshot and adding the regression test.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to configure many terminal ACP task records with a counting `listTaskRecords` runtime and run `runTaskRegistryMaintenance`; current main calls the whole-registry listing once for the initial sweep, once per terminal task, and once for orphan cleanup.

## Solution Assessment

Yes. Reusing the already captured `tasks` snapshot for `cleanupTerminalAcpSession(current, tasks)` is the narrowest maintainable fix because the helper only needs same-child-session activity checks and does not require a fresh global list for each task.
