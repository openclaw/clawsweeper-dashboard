## Summary

Keep open. Current main still has the reported O(n^2) source pattern: the maintenance pass takes an initial task snapshot, but then relists the full task registry inside the per-task terminal ACP cleanup path. A same-author PR is open, unmerged, and explicitly targets this issue, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Avoid quadratic task registry maintenance by reusing one task snapshot during per-task terminal ACP session cleanup and covering it with a regression test.

## Best Possible Solution

Review and land PR #75709, or an equivalent narrow patch, so terminal ACP cleanup reuses the initial maintenance snapshot while retaining the final fresh snapshot for orphaned parent-owned ACP sessions and adding the counting regression test.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to configure many terminal ACP task records with a counting listTaskRecords runtime and run runTaskRegistryMaintenance; current main calls the full-registry listing once for the initial sweep, once per task, and once for final orphan cleanup.

## Solution Assessment

Yes. Reusing the already captured tasks snapshot for cleanupTerminalAcpSession(current, tasks) is the narrowest maintainable solution because that helper only needs the supplied task array for active child-session checks and does not need a fresh global listing per task.
