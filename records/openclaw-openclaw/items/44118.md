## Summary

Keep open. Current main still does not implement the unmentioned Feishu slash-command bypass, and this PR is useful but not merge-ready because it uses the coarse command-auth probe as the mention-bypass condition rather than an authorized standalone control-command decision. That can route unmentioned Feishu group messages, including broadcast active-agent dispatch, past an operator-configured mention gate for inline slash-like text or unauthorized senders.

## What This Changes

The PR changes Feishu group message handling to detect command-like messages before the `requireMention` gate and reuse that flag when selecting the active broadcast agent.

## Best Possible Solution

Implement Feishu's bypass through the shared mention-gating contract: normalize the Feishu command probe, compute real `hasControlCommand` plus sender command authorization before the mention gate, bypass only authorized standalone control commands, carry the same decision into broadcast active-agent selection, and add focused Feishu single-agent and broadcast tests for authorized commands, unauthorized commands, and inline slash-like non-commands.

## Reproduction Assessment

Yes. Static source inspection gives a high-confidence path: current main drops unmentioned Feishu group messages before command probing, while the PR-specific regression follows directly from `shouldComputeCommandAuthorized` returning true for inline slash-like tokens such as `please check /foo`.

## Solution Assessment

No. The direction is useful, but this patch uses a coarse preauthorization detector as the bypass condition; the narrower maintainable solution is to bypass only after the message is known to be an authorized control command, preferably via `resolveMentionGatingWithBypass` semantics.
