## Summary

Keep open. Current main still enforces Feishu group `requireMention` before command probing, so this PR targets a real remaining no-mention slash-command gap, but the current patch should not merge as-is because it uses a coarse command-token detector as the mention-bypass flag and can route unmentioned unauthorized or slash-like group messages past the mention gate.

## What This Changes

The PR changes Feishu bot message handling to detect command-like group messages before the mention gate and use that flag to bypass `requireMention` and select the active broadcast agent.

## Best Possible Solution

Implement the Feishu group bypass using the shared mention-gating contract: normalize the command probe, compute whether the message is an actual control command and whether the sender is command-authorized before the mention gate, bypass only for authorized control commands, and carry that same decision into broadcast active-agent selection while preserving normal pending-history behavior for other unmentioned messages.

## Reproduction Assessment

Yes. On current main, an allowed Feishu group with `requireMention: true` will return before command probing for an unmentioned `/status` message; for the PR-specific regression, `shouldComputeCommandAuthorized` also returns true for inline slash-like tokens, so a message such as `please check /foo` would bypass the mention gate under the proposed patch.

## Solution Assessment

No. The direction is useful, but the patch uses a broad command-auth computation trigger as the bypass condition; the safer solution is to bypass only after actual authorized control-command gating is computed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Require authorized control commands for bypass:** `extensions/feishu/src/bot.ts:425-428`
  - body: `shouldComputeCommandAuthorized` is only a coarse trigger and returns true for inline slash-like tokens, not just recognized or authorized commands. With this as the mention-bypass flag, an unmentioned group message such as `please check /foo` or an unauthorized `/status` can pass `requireMention` and enter the single/broadcast dispatch path instead of remaining non-mentioned context. Compute the real authorized control-command decision before using it to bypass the mention gate.
  - confidence: 0.87
