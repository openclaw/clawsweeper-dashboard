## Summary

Keep open: current main already fixed the linked mention-prefixed Feishu command problem, but this PR proposes a different no-mention command bypass and is not mergeable as-is because the bypass is syntax-only, security-sensitive, and lacks real Feishu runtime proof.

## What This Changes

The PR changes Feishu group handling so recognized slash-command text can bypass the requireMention gate and select an active broadcast agent without an explicit bot mention.

## Best Possible Solution

Use an authorization-aware Feishu command activation path, preferably through the shared mention decision or equivalent Feishu ingress decision, with regression tests for authorized, unauthorized, single-agent, and broadcast group cases plus redacted runtime proof.

## Reproduction Assessment

Yes, source-level: current main resolves Feishu group activation with requireMention and mentionedBot, then returns before command probing when an unmentioned group message is not admitted. I did not establish a live Feishu runtime reproduction.

## Solution Assessment

No. The branch is not the best way to solve this because it bypasses requireMention on command syntax rather than an authorized command decision; the safer path is to compute command access before activation or route through the shared mention-gating contract.
