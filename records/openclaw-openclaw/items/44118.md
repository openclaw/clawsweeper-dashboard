## Summary

Keep open. The PR targets a real Feishu gap on current main, but it should not merge as-is because it uses a coarse command-probe helper as the mention-bypass condition even though that helper intentionally returns false positives; shared mention gating requires a real authorized control command before bypassing `requireMention`.

## What This Changes

The PR changes Feishu group message handling to detect command-like text before the `requireMention` gate and reuse that flag when selecting the active broadcast agent.

## Best Possible Solution

Implement Feishu's bypass through the shared mention-gating semantics: normalize the Feishu command probe, compute real control-command presence plus sender authorization before the gate, carry that same decision into broadcast activation, and cover authorized, unauthorized, and inline slash-like group cases.

## Reproduction Assessment

Yes. Static control flow is enough: current main returns before command probing for unmentioned Feishu group messages with `requireMention=true`, while the PR path lets inline slash-like text such as `please check /foo` set the bypass flag before command authorization is known.

## Solution Assessment

No. The direction is useful, but the proposed bypass condition is broader than the shared command authorization contract; the narrower maintainable fix is to bypass only for authorized control commands.
