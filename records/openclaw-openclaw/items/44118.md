## What This Changes

The PR modifies Feishu group handling to detect command-shaped slash text before the `requireMention` gate and to select the active broadcast agent without an explicit bot mention.

## Best Possible Solution

Use an authorization-aware Feishu mention-bypass path through the shared ingress or mention decision, with explicit policy approval and regression coverage for authorized and unauthorized single-agent and broadcast group cases.

Yes, source-level: current main resolves Feishu group activation with `requireMention` and `ctx.mentionedBot`, then returns before command probing for unmentioned group messages. I did not establish a live Feishu workspace reproduction.

## Solution Assessment

No. The PR's direction may be useful, but the implementation bypasses the mention gate on command-shaped text before `CommandAuthorized` is known; the safer solution is authorization-aware and aligned with the shared mention-gating contract.
