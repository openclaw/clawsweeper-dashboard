## Summary

Keep this PR open, but it still needs changes before merge: current main still drops unmentioned Feishu group messages before command probing, while the PR bypasses the operator mention gate using a coarse detector that intentionally has false positives.

## What This Changes

The PR adds a pre-gate Feishu group command flag from `shouldComputeCommandAuthorized` and uses it to bypass `requireMention` and broadcast active-agent selection for slash-like messages.

## Best Possible Solution

Implement the Feishu bypass through the shared mention-gating semantics: normalize the Feishu probe body, require an actual control command plus authorized sender before skipping the mention gate, reuse that decision for broadcast activation, and add false-positive, unauthorized, and broadcast regression coverage.

## Reproduction Assessment

Yes. Source inspection is enough: current main returns at the Feishu group `requireMention` gate before command probing, and the PR path would let `shouldComputeCommandAuthorized` false positives bypass that gate.

## Solution Assessment

No. The direction is useful, but the proposed bypass flag is broader than the shared command-authorization contract; the maintainable fix is to bypass only for authorized control commands.
