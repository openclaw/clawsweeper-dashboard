## Summary

Keep open, but this PR is not merge-ready: the source path is real, yet the diff bypasses Feishu `requireMention` on command syntax detection rather than authorized command access, lacks after-fix Feishu runtime proof, and is currently conflicting with main.

## What This Changes

The PR changes Feishu group handling so recognized slash-command text can bypass the group mention gate and selects an active broadcast agent when that bypass fires.

## Best Possible Solution

Use an authorization-aware Feishu command path, preferably through the shared mention decision or an equivalent Feishu ingress decision, with regression tests for authorized, unauthorized, single-agent, and broadcast group cases plus redacted runtime proof.

## Reproduction Assessment

Yes, source-level: current main resolves Feishu group activation before command probing and returns when `requireMention=true` and `ctx.mentionedBot=false`. I did not establish a live Feishu runtime reproduction.

## Solution Assessment

No. The branch is not the best solution because it bypasses on `shouldComputeCommandAuthorized` syntax detection rather than an authorized command decision; the safer path is to compute command access first or reuse the shared mention-gating contract.
