## Summary

Keep open; this PR is not safe to merge as written. Current main still gates unmentioned Feishu group messages before command probing, but the PR bypasses the operator mention gate with a coarse false-positive probe and only cites tests/CI, not real Feishu behavior proof.

## What This Changes

The PR changes Feishu group-message handling so slash-like commands can bypass `requireMention` and activate broadcast routing without an explicit bot mention.

## Best Possible Solution

If maintainers want Feishu text commands to bypass `requireMention`, implement it through shared mention-gating semantics: normalize the Feishu body, identify a real control command, resolve sender command authorization, reuse that decision for single-agent and broadcast activation, and cover false-positive, unauthorized, and broadcast cases.

## Reproduction Assessment

Yes, by source inspection rather than live Feishu: current main returns at the Feishu group mention gate before command probing, and the PR's proposed bypass path is visible in the diff. I did not run a live Feishu group reproduction in this read-only review.

## Solution Assessment

No. The direction may be useful if maintainers want this Feishu policy, but the patch should bypass only on an actual authorized control command and share that resolved decision with broadcast routing instead of using the coarse probe.
