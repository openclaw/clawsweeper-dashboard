## Summary

This PR should stay open for human review rather than cleanup closure: the source-level problem is understandable, but the patch admits unmentioned Feishu group messages based on command-shaped text before sender command authorization, and the contributor has not provided real Feishu runtime proof.

## What This Changes

The PR changes Feishu group handling so command-shaped slash text can bypass `requireMention` before the mention gate and can select the active broadcast agent without an explicit bot mention.

## Best Possible Solution

Use an authorization-aware Feishu command activation path through the shared ingress or mention decision, with regression tests for authorized and unauthorized no-mention single-agent and broadcast group cases plus redacted runtime proof.

## Reproduction Assessment

Yes, source-level: current main resolves Feishu group activation with `requireMention` and `ctx.mentionedBot`, then returns before command probing when an unmentioned group message is not admitted. I did not establish a live Feishu runtime reproduction.

## Solution Assessment

No. The PR bypasses `requireMention` on command-shaped text before `CommandAuthorized` is known; the safer solution is an authorization-aware Feishu ingress or shared mention-gating path.
