## Summary

The PR should stay open but is not merge-ready: current main still has the Feishu gate-ordering bug, while the patch bypasses `requireMention` with a coarse probe that intentionally has false positives before sender authorization.

## What This Changes

The PR changes Feishu group handling to compute a pre-gate command-like flag and use it to bypass `requireMention`, including broadcast active-agent selection.

## Best Possible Solution

Implement Feishu's bypass through the shared mention-gating semantics: normalize the probe, compute real control-command presence plus sender authorization before the skip decision, reuse that decision for broadcast activation, and cover authorized, unauthorized, inline false-positive, and broadcast cases.

## Reproduction Assessment

Yes. Source inspection is enough: current main returns before command probing for unmentioned Feishu group messages with `requireMention=true`, and the PR path would let `shouldComputeCommandAuthorized` false positives bypass that gate.

## Solution Assessment

No. The direction is useful, but the proposed bypass condition is broader than the shared command-authorization contract; the narrower maintainable fix is to bypass only for authorized control commands.
