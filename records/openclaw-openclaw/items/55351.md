  - body: Returning a bare UUID here drops the owning agent while per-agent stores are merged. A UUID session from a previous or non-default agent can later be resolved against the current default agent store, returning empty history or mutating the wrong session.
  - confidence: 0.9

  - body: This helper only strips `agent:main:<uuid>`, but legacy Control UI links can be generated under the configured default agent, such as `agent:ops:<uuid>`. Those links keep the prefixed key and can continue the split identity the PR is trying to remove.
  - confidence: 0.88
- **[P2] Canonicalize UUID aliases before subscribing:** `src/gateway/session-utils.ts:668-669`
  - body: The UUID fast path leaves bare UUIDs and `agent:<id>:<uuid>` as different canonical strings. Session-message subscriptions and fan-out use exact keys, so a legacy client subscribed on the prefixed form can miss live messages emitted under the bare UUID key.
