Keep open: the Open WebUI UUID-session split is still a valid current-main bug and this PR is the canonical open implementation path, but the branch is not mergeable as-is because it still has session-key edge regressions, lacks real behavior proof, is merge-conflicting, and bundles unrelated credentialed Codex proxy/auth/routing work.

## What This Changes

The branch preserves Open WebUI UUID session keys across gateway, transcript, and Control UI URL paths, adds regression tests, and also adds Codex proxy/auth, compaction-auth, and account-routed send changes.

## Best Possible Solution

Land a narrow revised PR or replacement that fixes UUID identity through one shared, agent-aware alias/canonicalization contract, preserves transcript lookup, and splits Codex proxy/auth/routing changes into separately reviewed work.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve agent scope for UUID store rows:** `src/gateway/session-utils.ts:643-645`
  - body: Returning bare UUIDs from canonicalizeSessionKeyForAgent also affects per-agent store merging: a UUID row from another agent store can be listed as unscoped, so later get/patch/delete resolves through the current default agent and can miss or mutate the wrong history.
  - confidence: 0.87
- **[P2] Normalize UUID links for configured default agents:** `ui/src/ui/app-settings.ts:51-52`
  - body: This only strips agent:main:<uuid>, but legacy links were generated with whichever agent was configured as default. Non-main defaults keep the prefixed key and can continue the split identity this PR is trying to converge.
  - confidence: 0.86
- **[P2] Resolve aliases before UUID fast paths:** `src/gateway/session-utils.ts:668-669`
  - body: The UUID early return runs before main-key alias handling and leaves bare/prefixed UUID forms as different canonical keys. A UUID-shaped session.mainKey or a legacy prefixed subscriber can split history or miss live message events that are emitted under the other form.
  - confidence: 0.8
- **[P2] Send the system prompt only once:** `src/agents/pi-embedded-runner/codex-proxy-transport.ts:722-723`
  - body: buildRequestBody sets instructions from context.systemPrompt while convertResponsesMessages already injects the same prompt into the input messages, duplicating system instructions on every Codex proxy request.
  - confidence: 0.88
