- **[P1] Split the Codex proxy transport from this fix:** `src/agents/pi-embedded-runner/extra-params.ts:305`
  - body: This wraps every embedded agent stream with a new Codex proxy transport from inside a PR whose stated bug is Open WebUI UUID session identity. That changes provider auth/transport behavior for `openai-codex-responses` without the review, proof, or ownership needed for this session fix, so the session repair needs to be split from this auth-provider change.
  - confidence: 0.9
- **[P1] Preserve agent scope for UUID store keys:** `src/gateway/session-utils.ts:643-644`
  - body: Returning a bare UUID here drops the owning agent while combined per-agent stores are merged. A UUID session from a previous or non-default agent can list as unscoped, and later get/patch/delete operations can resolve it against the current default agent store, returning empty history or mutating the wrong place.
  - confidence: 0.9
- **[P2] Normalize UUID links for configured defaults:** `ui/src/ui/app-settings.ts:51-52`
  - body: This only strips `agent:main:<uuid>`, but legacy links can be generated under the configured default agent, such as `agent:ops:<uuid>`. Those URLs keep the prefixed key and can continue the split identity the PR is supposed to remove.
  - confidence: 0.88
- **[P2] Unify UUID aliases for live subscriptions:** `src/gateway/session-utils.ts:668-669`
  - body: The new UUID fast path makes bare UUIDs and `agent:main:<uuid>` remain different canonical strings. Session-message subscriptions and fan-out use exact keys, so legacy clients subscribed on the prefixed form can miss live messages emitted under the bare UUID key.
  - confidence: 0.84
- **[P2] Send the system prompt once:** `src/agents/pi-embedded-runner/codex-proxy-transport.ts:722-723`
  - body: `buildRequestBody` sends the system prompt in the Responses `instructions` field, but `convertResponsesMessages` already inserts the same text as a system/developer input message. That duplicates prompt content on each Codex proxy call, increasing tokens and risking changed model behavior.
  - confidence: 0.86
