## Summary

Keep open: the Open WebUI UUID split-session bug is still source-reproducible on current main, but this PR should not merge as-is because the branch has blocking session alias/agent-scope regressions and mixes in unrelated credentialed agent transport/auth changes.

## What This Changes

The PR preserves bare Open WebUI UUID session keys across gateway storage, transcript lookup, and Control UI URL handling, adds regression tests, and also changes Codex proxy transport, compaction auth, and outbound send routing paths.

## Best Possible Solution

Land a narrow revised or replacement PR against current main that preserves Open WebUI UUID identity without losing agent ownership or alias/subscription convergence, then split the proxy/auth/routing work into separate owner-reviewed changes.

## Reproduction Assessment

Yes. Source inspection on current main shows a bare UUID-shaped session key is still treated as a generic bare key and prefixed with the default agent, while the Control UI accepts and writes URL session keys unchanged.

## Solution Assessment

No. The intended UUID-preservation direction is right, but this PR is not the best implementation because it introduces alias/agent-scope regressions, duplicates Codex system prompts, and bundles unrelated credentialed transport/auth changes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve agent scope for UUID store rows:** `src/gateway/session-utils.ts:643-644`
  - body: Returning bare UUIDs here also affects per-agent store merging. A UUID row from another agent store can be listed as unscoped, so later get/patch/delete resolves through the current default agent and can miss or mutate the wrong history.
  - confidence: 0.87
- **[P2] Normalize UUID aliases for any default agent:** `ui/src/ui/app-settings.ts:51-52`
  - body: This only strips agent:main:<uuid>, but legacy UUID links were generated with whichever agent was default. Installs with a non-main default keep the prefixed key and can continue the split identity this PR is meant to converge.
  - confidence: 0.86
- **[P2] Resolve main-key aliases before UUID fast path:** `src/gateway/session-utils.ts:668-669`
  - body: The UUID early return runs before session.mainKey alias handling. If an operator configured a UUID-shaped main key, it now resolves to the bare UUID instead of the canonical main session key, splitting existing clients.
  - confidence: 0.8
- **[P2] Canonicalize UUID aliases for subscriptions:** `src/gateway/session-utils.ts:668-669`
  - body: Bare UUID and agent:<id>:<uuid> remain different canonical keys, while message subscriptions and transcript fan-out use exact keys. A legacy client subscribed on the prefixed form can miss live messages emitted under the bare form.
  - confidence: 0.78
- **[P2] Send the system prompt only once:** `src/agents/pi-embedded-runner/codex-proxy-transport.ts:722-723`
  - body: buildRequestBody sets instructions from context.systemPrompt, while convertResponsesMessages also injects the same prompt into input messages. That duplicates system instructions on every Codex proxy call.
  - confidence: 0.88
