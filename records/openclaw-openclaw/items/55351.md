## What This Changes

The branch preserves bare Open WebUI UUID session keys across gateway/session lookup and Control UI URLs, adds transcript lookup coverage, and also adds Codex proxy/auth, compaction-auth, and account-routed send changes.

## Best Possible Solution

Land a focused, rebased session fix that uses one alias-aware raw-session contract, preserves agent scope and main-key compatibility, keeps only the necessary Control UI URL normalization, and includes redacted real Open WebUI proof.

Yes at source level. Current main still canonicalizes bare non-main session keys through the default-agent path, while the Control UI preserves session URL keys verbatim, matching the reported UUID versus agent-prefixed split path.

## Solution Assessment

No. The reported bug is real, but this branch is broader than the fix and its raw UUID fast path is less maintainable than a shared alias-aware contract with upgrade coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Keep UUID keys scoped in combined stores:** `src/gateway/session-utils.ts:643-645`
  - body: Returning bare UUIDs here means `loadCombinedSessionStoreForGateway` later merges per-agent stores under the same unscoped key. A UUID session from a previous or non-default agent can then round-trip through the current default agent and show empty or wrong history, so the fix needs to preserve enough agent/alias ownership for per-agent stores.
  - confidence: 0.86
- **[P2] Normalize legacy UUID links for the configured default agent:** `ui/src/ui/app-settings.ts:51-53`
  - body: This only strips `agent:main:<uuid>`, but older Control UI links were generated with whatever agent was the default when the link was created, such as `agent:ops:<uuid>`. Those installs would keep the prefixed URL/session key and continue the alias split instead of converging on the bare Open WebUI UUID.
  - confidence: 0.82
- **[P2] Resolve main-key aliases before the UUID fast path:** `src/gateway/session-utils.ts:668-670`
  - body: The new early return runs before the configured `session.mainKey` alias check, so a UUID-shaped mainKey now resolves to the raw UUID instead of the canonical main session. That breaks existing configs that intentionally use a UUID-like main key.
  - confidence: 0.79
- **[P2] Avoid duplicating the system prompt in Codex requests:** `src/agents/pi-embedded-runner/codex-proxy-transport.ts:722-723`
  - body: `buildRequestBody` sets `instructions` to the system prompt while `convertResponsesMessages` also injects the same prompt as a system/developer input message. That doubles prompt content for proxy Codex calls, increasing token use and potentially changing model behavior.
  - confidence: 0.76
