## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Preserve agent ownership for UUID rows:** `src/gateway/session-utils.ts:643-644`
  - body: Returning a bare UUID here also affects per-agent store rows when `loadCombinedSessionStoreForGateway` canonicalizes them. A UUID row from another agent store can be listed as an unscoped key and later fetched or mutated against the current default agent, so keep enough agent ownership or use an explicit alias contract instead of making UUIDs globally bare.
  - confidence: 0.91
- **[P2] Normalize legacy UUID links for any default agent:** `ui/src/ui/app-settings.ts:51-52`
  - body: Legacy UUID Control UI links were generated with the configured default agent, not always `main`. With a default like `ops`, `agent:ops:<uuid>` stays prefixed and the URL/settings path does not converge on the bare Open WebUI key this PR is trying to preserve.
  - confidence: 0.87
- **[P2] Resolve main-key aliases before UUID preservation:** `src/gateway/session-utils.ts:668-669`
  - body: This UUID fast path runs before the configured `session.mainKey` alias check. If an existing install has a UUID-shaped main key, `resolveSessionStoreKey` now returns the raw UUID instead of the canonical main session key, splitting configured main-session history.
  - confidence: 0.84
