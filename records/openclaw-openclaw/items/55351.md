## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Preserve agent scope for UUID keys:** `src/gateway/session-utils.ts:643-644`
  - body: Returning the bare UUID here drops the owning agent while combined per-agent stores are merged. A UUID session from a previous or non-default agent can list as unscoped, and later get/patch/delete operations can resolve it against the current default agent store, returning empty history or mutating the wrong place.
  - confidence: 0.9
- **[P2] Normalize UUID links for configured defaults:** `ui/src/ui/app-settings.ts:51-52`
  - body: This only strips `agent:main:<uuid>`, but legacy links can be generated under the configured default agent, such as `agent:ops:<uuid>`. Those URLs keep the prefixed key and can continue the split identity the PR is supposed to remove.
  - confidence: 0.88
- **[P2] Canonicalize UUID aliases before subscriptions:** `src/gateway/session-utils.ts:668-669`
  - body: The new UUID fast-path makes bare UUIDs and `agent:main:<uuid>` remain different canonical strings. Session-message subscriptions and fan-out use exact keys, so legacy clients subscribed on the prefixed form can miss live messages emitted under the bare UUID key.
  - confidence: 0.84
- **[P1] Use a WebSocket client that can send auth headers:** `src/agents/pi-embedded-runner/codex-proxy-transport.ts:811`
  - body: This constructs `globalThis.WebSocket` with a headers object, but upstream Codex uses a header-aware client/request path for the websocket upgrade. The PR path can drop `Authorization` and `chatgpt-account-id`, breaking proxied Codex auth or tenant routing.
  - confidence: 0.86
- **[P2] Stop duplicating the Codex system prompt:** `src/agents/pi-embedded-runner/codex-proxy-transport.ts:724-725`
  - body: The request body sends `instructions: context.systemPrompt` while `convertResponsesMessages(...)` also injects the same prompt into `input`. That duplicates prompt content on every proxied Codex request and can change behavior or waste context.
  - confidence: 0.82
