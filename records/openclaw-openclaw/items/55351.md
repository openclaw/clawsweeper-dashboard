## Summary

Keep open. The Open WebUI UUID split is still real on current main, but this PR is not mergeable as-is because it mixes the valid session fix with unrelated credentialed Codex proxy/auth/routing changes and still introduces session-key alias regressions.

## What This Changes

The PR preserves Open WebUI UUID chat ids across gateway session resolution, transcript lookup, and Control UI URL sync, adds regression tests, and also adds Codex proxy/auth/routing changes.

## Best Possible Solution

Land a narrow split or replacement fix that preserves Open WebUI UUID session identity while retaining agent ownership, alias precedence, transcript fallback, URL normalization, and subscription fan-out; route Codex proxy/auth/routing work separately.

## Reproduction Assessment

Yes. The linked bug gives concrete Open WebUI and Control UI steps, and current main still has the two source conditions: bare UUID-like keys canonicalize through the default agent and Control UI session URLs pass through unchanged.

## Solution Assessment

No. The intended direction is right, but this PR is not the best implementation because it bundles unrelated security-sensitive transport work and leaves alias/scoping regressions in the session fix.

Full review comments:

- **[P1] Preserve agent scope for UUID store rows:** `src/gateway/session-utils.ts:640-642`
  - body: Returning bare UUIDs from canonicalizeSessionKeyForAgent also affects per-agent store merging. A UUID row from another agent store can be listed as unscoped, so later get/patch/delete resolves through the current default agent and can miss or mutate the wrong history.
  - confidence: 0.87
- **[P2] Normalize UUID aliases for any default agent:** `ui/src/ui/app-settings.ts:51-52`
  - body: This only strips agent:main:<uuid>, but legacy UUID links were generated with whichever agent was default. Installs with a non-main default keep the prefixed key and can continue the split identity this PR is meant to converge.
  - confidence: 0.86
- **[P2] Resolve main-key aliases before UUID fast path:** `src/gateway/session-utils.ts:665-670`
  - body: The UUID early return runs before session.mainKey alias handling. If an operator configured a UUID-shaped main key, it now resolves to the bare UUID instead of the canonical main session key, splitting existing clients.
  - confidence: 0.8
- **[P2] Canonicalize UUID aliases for subscriptions:** `src/gateway/session-utils.ts:665-670`
  - body: Bare UUID and agent:<id>:<uuid> forms remain different canonical keys, while message subscriptions and transcript fan-out use exact keys. A legacy client subscribed on the prefixed form can miss live messages emitted under the bare form.
  - confidence: 0.78
- **[P1] Use a WebSocket client that sends headers:** `src/agents/pi-embedded-runner/codex-proxy-transport.ts:763-766`
  - body: connectWebSocket passes headers to globalThis.WebSocket, but the standard Node/Bun constructor does not send custom headers there. Authorization and chatgpt-account-id can be dropped for proxied Codex calls, breaking auth or tenant routing.
  - confidence: 0.74
- **[P2] Send the system prompt only once:** `src/agents/pi-embedded-runner/codex-proxy-transport.ts:722-723`
  - body: buildRequestBody sets instructions from context.systemPrompt while convertResponsesMessages also injects the same prompt into input messages. That duplicates system instructions on every Codex proxy call.
  - confidence: 0.88
