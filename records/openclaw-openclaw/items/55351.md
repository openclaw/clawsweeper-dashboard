Overall confidence: 0.86

Full review comments:

- **[P1] Use a WebSocket client that sends auth headers:** `src/agents/pi-embedded-runner/codex-proxy-transport.ts:809`
  - body: `globalThis.WebSocket` uses the standard `new WebSocket(url, protocols?)` shape in Node/browser-compatible runtimes, so this `{ headers }` object is not sent as handshake headers. That drops `Authorization` and `chatgpt-account-id` for proxied `openai-codex-responses` calls, breaking auth or tenant routing for this transport.
  - confidence: 0.91
- **[P2] Normalize UUID legacy links for non-main agents:** `ui/src/ui/app-settings.ts:51-52`
  - body: The gateway alias fallback is agent-specific, but this UI helper only strips `agent:main:<uuid>`. Deployments whose default agent was `ops` or another id can keep `agent:<id>:<uuid>` in Control UI URLs, so those legacy links do not converge on the bare UUID form the PR is trying to preserve.
  - confidence: 0.84
