Full review comments:

- **[P1] Preserve agent scope for UUID session rows:** `src/gateway/session-utils.ts:643-645`
  - body: Returning a bare UUID here makes `loadCombinedSessionStoreForGateway()` collapse UUID rows from per-agent stores into the same unscoped key. Later lookup derives the owning agent from that bare key as the current default agent, so sessions from a different/default-at-the-time agent can show empty history or be mutated in the wrong store.
  - confidence: 0.9
- **[P2] Normalize UUID links for the configured default agent:** `ui/src/ui/app-settings.ts:51-53`
  - body: This strips only `agent:main:<uuid>`, but the backend canonicalizes bare keys under the configured default agent. Existing links like `agent:ops:<uuid>` from an ops default deployment stay prefixed, so those installs do not converge on the intended bare Open WebUI UUID identity.
  - confidence: 0.84
