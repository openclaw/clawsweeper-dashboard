- **[P1] Remove the unrelated Codex proxy wrapper:** `src/agents/pi-embedded-runner/extra-params.ts:305`
  - body: `applyExtraParamsToAgent` now wraps every agent stream function with the new Codex proxy transport, even though this PR is supposed to fix Open WebUI UUID session identity.   - confidence: 0.9
- **[P1] Preserve agent ownership for UUID keys:** `src/gateway/session-utils.ts:643-644`
  - body: Returning the bare UUID from `canonicalizeSessionKeyForAgent` drops the owning agent in per-agent or combined stores. A UUID session belonging to a non-default agent can later resolve through the current default agent path, producing empty history or mutating the wrong session entry.
  - confidence: 0.88
- **[P2] Normalize configured default-agent UUID links:** `ui/src/ui/app-settings.ts:51-52`
  - body: This only strips `agent:main:<uuid>`, but legacy links and stored settings can be generated under the configured default agent, such as `agent:ops:<uuid>`. Those URLs remain prefixed and can keep the split identity the PR is meant to remove.
  - confidence: 0.86
- **[P2] Keep UUID subscription aliases together:** `src/gateway/session-utils.ts:668-669`
  - body: The UUID fast path leaves bare UUIDs and `agent:main:<uuid>` as different canonical strings. Live session subscriptions fan out by exact session key, so clients subscribed through the prefixed legacy form can miss messages emitted under the bare UUID form.
  - confidence: 0.82
- **[P2] Avoid duplicating the Codex system prompt:** `src/agents/pi-embedded-runner/codex-proxy-transport.ts:722-723`
  - body: The new transport sends `context.systemPrompt` as `instructions` while also converting the same context into input messages. Proxied Codex requests can receive duplicated system/developer instructions, changing model behavior and token use.
  - confidence: 0.8
