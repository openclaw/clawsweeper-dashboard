## Summary

Keep open: the Open WebUI UUID split is a real session-state bug on current main, but this PR is not merge-ready because it introduces session alias regressions, lacks after-fix real behavior proof, and bundles unrelated credentialed agent/proxy/auth runtime changes into a session fix.

## What This Changes

The branch preserves Open WebUI UUID session keys in gateway/store and Control UI URL paths, threads agent context into transcript lookup, adds regression tests, and also adds embedded Codex proxy/auth, compaction-auth, and account-routed send changes.

## Best Possible Solution

Land a narrow rebased replacement that defines one agent-aware alias contract for UUID-backed Open WebUI chats, preserves upgrade/read/write behavior, and splits proxy/auth/routing work into separate reviewed PRs.

## Reproduction Assessment

Yes, at source level. Current main still prefixes bare UUID-like session keys through the default agent and applies Control UI URL session values unchanged, matching the linked Open WebUI split-session path.

## Solution Assessment

No. UUID preservation is the right direction, but this branch is not the best mergeable solution because it introduces alias/scope regressions and mixes unrelated credentialed runtime changes into the bug fix.
