## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use a WebSocket client that can send headers:** `src/agents/pi-embedded-runner/codex-proxy-transport.ts:809`
  - body: This transport builds Authorization and chatgpt-account-id headers, but passes them through a non-standard headers option to global WebSocket. In Node/Bun-style WebSocket implementations those headers are not sent, so proxied Codex calls can fail authentication or tenant routing; use a handshake path that supports custom headers like upstream Codex does.
  - confidence: 0.89
- **[P1] Keep UUID sessions scoped when merging agent stores:** `src/gateway/session-utils.ts:643-644`
  - body: Returning a bare UUID for every agent drops the owner store identity while combined stores are merged. With multiple agents or a changed default agent, later get/patch/delete calls can resolve the same visible UUID against the wrong store or show empty history, so the fix needs an alias/owner-preserving round trip.
  - confidence: 0.82
- **[P2] Normalize UUID links for non-main default agents:** `ui/src/ui/app-settings.ts:51-52`
  - body: normalizeChatUrlSessionKey only strips agent:main:<uuid>, but legacy links can use the configured default agent, for example agent:ops:<uuid>. Those installs keep the prefixed form and continue the session split the PR is trying to eliminate.
  - confidence: 0.84
- **[P2] Resolve main-key aliases before UUID fast paths:** `src/gateway/session-utils.ts:668-669`
  - body: The UUID early return runs before the existing main-key alias handling. If session.mainKey is UUID-shaped, resolveSessionStoreKey now returns the raw UUID instead of the configured main session key, splitting clients that rely on that alias.
  - confidence: 0.8
- **[P2] Avoid duplicating the system prompt:** `src/agents/pi-embedded-runner/codex-proxy-transport.ts:722-723`
  - body: buildRequestBody sets instructions to context.systemPrompt while convertResponsesMessages also prepends the same system prompt as a system/developer input message. Every Codex proxy request can therefore send duplicate prompt content, increasing tokens and changing model behavior.
  - confidence: 0.86
