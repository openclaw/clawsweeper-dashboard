## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve agent scope for UUID keys:** `src/gateway/session-utils.ts:643-644`
  - body: Returning a bare UUID here drops the owning agent while combined per-agent stores are being merged. A UUID session from a previous or non-default agent can then list as unscoped, and later get/patch/delete operations resolve it against the current default agent store, returning empty history or mutating the wrong place.
  - confidence: 0.9
- **[P2] Normalize UUID links for configured defaults:** `ui/src/ui/app-settings.ts:51-52`
  - body: This only strips `agent:main:<uuid>`, but legacy links can be generated under the configured default agent, such as `agent:ops:<uuid>`. Those URLs keep the prefixed key and can continue the split identity the PR is supposed to remove.
  - confidence: 0.88
- **[P2] Canonicalize UUID aliases before subscriptions:** `src/gateway/session-utils.ts:668-669`
  - body: The new UUID fast-path makes bare UUIDs and `agent:main:<uuid>` remain different canonical strings. Session-message subscriptions and fan-out use exact keys, so legacy clients subscribed on the prefixed form can miss live messages emitted under the bare UUID key.
  - confidence: 0.84
- **[P2] Avoid duplicating the Codex system prompt:** `src/agents/pi-embedded-runner/codex-proxy-transport.ts:722-723`
  - body: This request body sends `instructions: context.systemPrompt` while `convertResponsesMessages(...)` is called with its default `includeSystemPrompt` behavior, so the same prompt is also inserted into `input`. Current main avoids that for Codex Responses; duplicating it can change behavior and waste context on every proxied Codex request.
  - confidence: 0.83
