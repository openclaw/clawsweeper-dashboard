## Summary

Keep open. Current main still has the Open WebUI UUID session split at source level, but this PR is not mergeable as-is because it mixes the valid UUID-session fix with unrelated credentialed Codex proxy/auth/routing work and introduces session alias regressions.

## What This Changes

The PR changes gateway session-key canonicalization, transcript lookup, Control UI URL normalization, and regression tests for Open WebUI UUID chats, while also adding Codex proxy transport/auth and outbound routing changes.

## Best Possible Solution

Land a narrow revised PR or replacement that fixes UUID/raw session identity through a shared alias contract, keeps transcript lookup agent-aware, and routes Codex proxy/auth/routing work through separate owner-reviewed changes.

## Reproduction Assessment

Yes, at source level. Current main still canonicalizes a bare UUID-like key through the default agent and the Control UI still applies URL session keys unchanged, matching the split-session path described in the linked report.

## Solution Assessment

No. The UUID-preservation direction is valid, but this branch is not the best implementation because it loses agent/alias semantics and bundles unrelated credentialed proxy/auth/routing changes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve agent scope for UUID store rows:** `src/gateway/session-utils.ts:643-645`
  - body: Returning bare UUIDs from canonicalizeSessionKeyForAgent also affects per-agent store merging. A UUID row from another agent store can be listed as unscoped, so later get/patch/delete resolves through the current default agent and can miss or mutate the wrong history.
  - confidence: 0.87
- **[P2] Normalize UUID aliases for every default agent:** `ui/src/ui/app-settings.ts:51-52`
  - body: This only strips agent:main:<uuid>, but legacy UUID links were generated with whichever agent was configured as default. Non-main defaults keep the prefixed key and can continue the split identity this PR is trying to converge.
  - confidence: 0.86
- **[P2] Resolve main-key aliases before UUID fast paths:** `src/gateway/session-utils.ts:668-669`
  - body: The UUID early return runs before session.mainKey alias handling. If an operator configured a UUID-shaped main key, it now resolves to the bare UUID instead of the canonical main session key, splitting existing clients.
  - confidence: 0.8
- **[P2] Use one canonical key for UUID subscriptions:** `src/gateway/session-utils.ts:668-669`
  - body: Bare UUID and agent:<id>:<uuid> forms remain different canonical keys, while sessions.messages.subscribe uses the exact canonical key. A legacy client subscribed on the prefixed form can miss live messages emitted under the bare form.
  - confidence: 0.78
- **[P2] Send the system prompt only once:** `src/agents/pi-embedded-runner/codex-proxy-transport.ts:722-723`
  - body: buildRequestBody sets instructions from context.systemPrompt while convertResponsesMessages also injects the same prompt into the input messages. That duplicates system instructions on every Codex proxy call.
  - confidence: 0.88
- **[P1] Use a WebSocket client that can send headers:** `src/agents/pi-embedded-runner/codex-proxy-transport.ts:809`
  - body: connectWebSocket passes { headers } to globalThis.WebSocket, but the runtime WebSocket constructor does not expose a custom-header init contract. Proxied Codex calls can silently drop Authorization and chatgpt-account-id headers.
  - confidence: 0.82
