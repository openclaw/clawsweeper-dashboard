Because `extra-params.ts` imports this file, the branch can fail module resolution unless the imports are switched to the installed packages or an approved dependency change is added.
  - confidence: 0.96
- **[P1] Keep UUID sessions scoped to the owning agent store:** `src/gateway/session-utils.ts:643-645`
  - body: Returning the bare UUID here drops the agent store identity while per-agent stores are merged and looked up by canonical key. If two agent stores contain the same UUID, or the default agent changes, follow-up `sessions.list`/`sessions.get`/patch/delete flows can target the wrong store; the alias contract needs to preserve the owning agent while still allowing Open WebUI's bare UUID key.
  - confidence: 0.88
- **[P2] Normalize UUID links for the configured default agent:** `ui/src/ui/app-settings.ts:51-53`
  - body: This only strips `agent:main:<uuid>`, but legacy UUID links can be generated with whatever agent was configured as default, such as `agent:ops:<uuid>`. Those links stay prefixed and can keep the split identity alive for non-main default-agent setups.
  - confidence: 0.86
