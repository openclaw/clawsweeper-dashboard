## Summary

Keep open. The PR is the right narrow docs/skill direction for the linked 1Password bug, but it still has a blocking service-account example that uses an unsupported probe and a smaller Windows standalone-signin gap now that the skill points that mode at tmux.

## What This Changes

This PR rewrites the bundled 1Password skill and get-started reference to choose direct `op` execution or tmux by authentication mode, and adds an Unreleased changelog fix entry for #52540.

## Best Possible Solution

Land the auth-mode-aware skill/reference/changelog update after replacing the service-account probe with a documented service-account-safe check and adding a Windows note for standalone signin, keeping the change limited to bundled skill guidance.

## Reproduction Assessment

Yes. Current main provides a source-level reproduction of the tmux-only guidance, #52540 provides direct-versus-tmux desktop-app runtime steps, and the PR blockers are reproducible by comparing the changed lines against the 1Password docs and the bundled tmux skill metadata.

## Solution Assessment

No, not as written. The overall approach is the narrowest maintainable fix, but the service-account example and Windows standalone-signin caveat need small markdown corrections before merge.
