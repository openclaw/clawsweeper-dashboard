## Summary

Keep this PR open. The auth-mode-aware skill rewrite is the right direction for the linked 1Password desktop-app bug, but the latest patch still includes a service-account `op vault list` probe that is not listed in 1Password's service-account supported-command docs, so it can fail in the headless setup the section is meant to guide.

## What This Changes

This PR rewrites the bundled 1Password skill and get-started reference to choose direct `op` execution or tmux by auth mode, and adds an Unreleased changelog fix entry for #52540.

## Best Possible Solution

Land the auth-mode-aware bundled skill update after replacing the unsupported service-account `op vault list` probe with a documented `op whoami` or `op read` check, keeping the change limited to the skill, reference note, and changelog.

## Reproduction Assessment

Yes. Current main plainly contains the tmux-only instruction, #52540 provides a direct-versus-tmux desktop-app reproduction, and the PR finding is reproducible by comparing the new service-account example against 1Password's documented service-account command support.

## Solution Assessment

No, not as written. The overall PR direction is the narrowest maintainable fix for the reported bug, but the service-account example should use a documented service-account-safe command before merge.
