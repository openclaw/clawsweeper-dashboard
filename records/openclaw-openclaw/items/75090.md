## Summary

This PR should stay open with changes requested. The overall direction matches the linked 1Password desktop-app bug, but the latest head still uses `op vault list` as the service-account probe even though current 1Password service-account docs do not list that command as supported and do document safer checks such as `op read` and `op whoami`. ([developer.1password.com](https://developer.1password.com/docs/cli/reference/commands/whoami/))

## What This Changes

This PR rewrites the bundled 1Password skill and get-started reference to choose direct `op` execution or tmux by auth mode, and adds an Unreleased changelog fix entry for #52540.

## Best Possible Solution

Land the auth-mode-aware skill/reference update after replacing the service-account `op vault list` smoke check with a documented service-account-safe probe such as `op whoami` or the first intended `op read`, keeping the change limited to markdown and the existing changelog entry.
