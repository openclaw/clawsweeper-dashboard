## Summary

Keep open: the PR targets a real bundled-skill bug and the direction is right, but latest head still contains a service-account example that can fail before the intended secret read and a standalone tmux flow that queues verification before interactive sign-in completes.

## What This Changes

This PR rewrites the bundled 1Password skill and reference note so `op` execution depends on service-account, desktop-app, or standalone-signin auth mode, and adds an Unreleased changelog entry.

## Best Possible Solution

Land the auth-mode-aware skill update after replacing the service-account probe and making the standalone tmux sample an explicit attach/wait/manual verification flow.

## Reproduction Assessment

Yes. The underlying bug is reproducible from current main's tmux-only skill text plus the linked #52540 direct-vs-tmux report, and the PR defects are reproducible by inspecting latest head at `skills/1password/SKILL.md:54` and `skills/1password/SKILL.md:82`.

## Solution Assessment

No, not as written. The requested direction is the narrow maintainable fix, but the examples must use a service-account-supported operation and avoid queuing commands before interactive sign-in completes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Use a service-account-supported probe:** `skills/1password/SKILL.md:54`
  - body: The service-account example still runs `op vault list` before the intended `op read`. 1Password's service-account docs list supported commands such as `op read`, `op inject`, `op run`, and limited vault management, not `op vault list`, so a generic headless setup can fail before reaching the real secret-read path.
  - confidence: 0.87
- **[P2] Wait for standalone sign-in before verification:** `skills/1password/SKILL.md:83-85`
  - body: The snippet sends interactive sign-in and immediately queues `op whoami`, `op vault list`, and `capture-pane`. Because sign-in can block for password or app authentication before the session export exists, split this into an attach/manual wait before sending verification commands.
  - confidence: 0.9
