## Summary

Keep open: the PR is aimed at a real bundled-skill guidance bug, but the latest patch still contains a narrow blocking service-account example that can send agents down an unsupported 1Password CLI path before they read a secret.

## What This Changes

This PR rewrites the bundled 1Password skill and get-started reference to choose direct `op` execution or tmux by auth mode, and adds an Unreleased changelog fix entry for #52540.

## Best Possible Solution

Land the auth-mode-aware bundled skill update after replacing the service-account `op vault list` probe with a documented `op whoami` or real `op read` check, preserving the narrow docs/skill/changelog scope and keeping tmux only for standalone interactive signin where a persistent shell is needed.

## Reproduction Assessment

Yes. Current main plainly contains the tmux-only instruction, #52540 gives a direct-versus-tmux desktop-app reproduction, and the PR finding is reproducible by comparing the new service-account example with 1Password's documented service-account command support.

## Solution Assessment

No, not as written. The overall PR direction is the right narrow fix for the reported bug, but the service-account probe should use a documented service-account-safe command before merge.
