## Summary

Keep open: this PR targets a real bundled-skill guidance bug, but the current PR head still makes the standalone tmux signin example unreliable by queueing verification and capture immediately after interactive `op signin`. Reviewed PR: https://github.com/openclaw/openclaw/pull/75090

## What This Changes

The PR rewrites the bundled 1Password skill, reference note, and changelog so service-account and desktop-app auth use direct `op` execution while tmux is reserved for standalone interactive signin.

## Best Possible Solution

Land the auth-mode-aware skill/reference/changelog update after making standalone tmux signin an explicit interactive flow that waits for signin to complete before verification.

## Reproduction Assessment

Yes. Current main and v2026.5.2 are source-reproducible for the tmux-only guidance, and PR head is source-reproducible at lines 82-85 because it queues verification/capture immediately after interactive signin.

## Solution Assessment

No, not as written. The PR's auth-mode split is the right scope, but the standalone tmux example should wait for `op signin` to complete before verification commands or capture run.
