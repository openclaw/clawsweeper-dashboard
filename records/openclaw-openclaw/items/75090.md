## Summary

Keep open: current main still has the tmux-only 1Password guidance, and the PR is the right direction, but the latest head still documents a service-account probe that can fail and a standalone tmux signin example that is not reliable.

## What This Changes

The PR rewrites bundled 1Password skill/reference guidance and the changelog so agents choose direct `op` or tmux by auth mode instead of requiring tmux for every command.

## Best Possible Solution

Land the markdown-only auth-mode update after repairing the service-account and standalone-signin examples, preserving direct exec for desktop app integration and tmux only for standalone interactive signin.

## Reproduction Assessment

Yes. The underlying bug reproduces by inspecting current main's tmux-only skill text plus the linked direct-vs-tmux issue steps; the PR-specific defects reproduce by reading latest head lines 54 and 82-86 against 1Password service-account/manual-signin docs and tmux pane behavior.

## Solution Assessment

No, not as written. The auth-mode-aware direction is the narrow maintainable fix, but the examples need to avoid unsupported service-account probing and make the interactive tmux signin flow wait and preserve its authenticated shell.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Use a supported service-account probe:** `skills/1password/SKILL.md:54`
  - body: The service-account example exports `OP_SERVICE_ACCOUNT_TOKEN` and then runs `op vault list` before the intended secret read. For a generic headless service-account setup, use a documented setup check such as `op whoami` or `op service-account ratelimit`, or go straight to the intended `op read`, so the example does not fail before the real operation.
  - confidence: 0.87
- **[P2] Make the tmux signin example wait and preserve the session:** `skills/1password/SKILL.md:82-86`
  - body: The standalone-signin block queues `op whoami` and `op vault list` immediately after interactive `op signin`, then captures and kills the pane. Since signin can block for input before emitting the session export, this can run too early or discard the authenticated shell; split the signin step from follow-up commands and keep the tmux session alive for later `op` use.
  - confidence: 0.9
