## Summary

Keep open: the PR is a focused fix for a valid bundled-skill guidance bug, but the current head still queues standalone-signin verification commands before the interactive signin can complete.

## What This Changes

The PR updates the bundled 1Password skill, reference note, and changelog to use direct `op` execution for service-account and desktop-app auth while reserving tmux for standalone signin.

## Best Possible Solution

Land the auth-mode-aware skill/reference/changelog update after rewriting standalone signin as an explicit interactive wait/attach flow before verification.

Yes. Current main and `v2026.5.2` still mandate tmux for every `op` command, the linked issue provides direct-vs-tmux runtime steps, and the PR-head defect is visible in `skills/1password/SKILL.md:82-85`.

## Solution Assessment

No, not yet. The auth-mode split is the right narrow solution, but the standalone tmux example must wait for interactive signin before sending verification commands.
