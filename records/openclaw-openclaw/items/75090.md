## Summary

Keep open: the PR direction is the right narrow fix for the linked 1Password skill bug, but the current head still queues verification commands immediately after interactive standalone `op signin`, so that documented fallback can fail before merge.

## What This Changes

The PR rewrites the bundled 1Password skill, reference note, and changelog so agents use direct `op` execution for service-account and desktop-app auth while reserving tmux for standalone interactive signin.

## Best Possible Solution

Land the auth-mode-aware skill/reference/changelog update after making the standalone tmux example an explicit interactive flow that waits for sign-in to finish before verification and keeps the authenticated session available for later commands.

## Reproduction Assessment

Yes. Current main and `v2026.5.2` still require tmux for every `op` command, the linked issue gives direct-vs-tmux runtime steps, and the remaining PR defect is source-reproducible in the head snippet at lines 82-85.

## Solution Assessment

No, not as written. The auth-mode split is the right narrow fix, but the standalone signin snippet should wait for interactive signin completion before verification commands are sent in the same pane.
