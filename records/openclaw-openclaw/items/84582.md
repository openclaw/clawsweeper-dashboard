## What This Changes

The branch changes `src/infra/net/undici-runtime.ts` so `createHttp1ProxyAgent` enables the `proxyTls` target only when the proxy URI starts with `https:`.

## Best Possible Solution

Fix shared proxy scheme handling so SOCKS avoids truthy `proxyTls`, HTTP/HTTPS proxy endpoint policy stays intact, and explicit plus env proxy modes have regression tests and redacted live Telegram proof.

## Reproduction Assessment

Yes, by source inspection for the core failure: Telegram SOCKS proxy support reaches Undici proxy helpers, and Undici 8.5.0 treats truthy SOCKS `proxyTls` as an HTTPS proxy connection. I did not run a live Telegram/SOCKS repro in this read-only review.

## Solution Assessment

No. The shared proxy helper is the right area, but this patch is too narrow because it misses env SOCKS proxies and can drop HTTP proxy endpoint connector policy.
