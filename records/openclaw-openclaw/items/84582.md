Keep open: the bug target is real, but this PR is not merge-ready because it only changes the explicit proxy helper path, leaves the documented env SOCKS path exposed, can alter HTTP proxy endpoint behavior, and still lacks inspectable real behavior proof.

## What This Changes

The PR changes `createHttp1ProxyAgent` so OpenClaw's `proxyTls` target is enabled only when the proxy URI string starts with `https:`.

## Best Possible Solution

Fix shared proxy scheme handling so SOCKS avoids truthy `proxyTls`, HTTP/HTTPS proxy endpoint policy remains intact, and explicit plus env proxy modes have regression tests and redacted real Telegram proxy proof.

## Reproduction Assessment

Yes, by source inspection for the core failure mode: Telegram SOCKS proxy support reaches Undici proxy helpers, and Undici 8.3.0 treats truthy `proxyTls` as HTTPS-to-SOCKS. I did not run a live Telegram/SOCKS repro in this read-only review.

## Solution Assessment

No. The fix belongs in shared proxy scheme handling, but this patch is too narrow because it misses env SOCKS proxies and can drop HTTP proxy endpoint connector policy.
