## What This Changes

The PR changes `src/infra/net/undici-runtime.ts` so `createHttp1ProxyAgent` only applies proxy-endpoint `proxyTls` defaults when the explicit proxy URI starts with `https:`.

## Best Possible Solution

Fix shared proxy-scheme handling so explicit and env SOCKS avoid TLS-to-SOCKS while HTTP/HTTPS proxy endpoint connector policy remains intact, then add focused regression tests and redacted live Telegram proxy proof.

## Reproduction Assessment

Yes for the core failure from source: current code can pass truthy `proxyTls` into Undici SOCKS proxy construction, and Undici treats that as HTTPS-to-proxy. A live Telegram getMe/sendMessage repro was not run in this read-only review.

## Solution Assessment

No, this is only a plausible partial fix. The better fix covers explicit and env SOCKS while preserving HTTP proxy connector options for existing plain HTTP proxies.
