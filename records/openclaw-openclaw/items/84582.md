## What This Changes

The PR changes `src/infra/net/undici-runtime.ts` so `createHttp1ProxyAgent` sets Undici `proxyTls` only when the proxy URI starts with `https:`.

## Best Possible Solution

Fix shared proxy scheme handling so explicit and env SOCKS avoid TLS-to-SOCKS while HTTP/HTTPS proxy endpoint connector policy remains intact, then add focused regression tests and redacted live Telegram proxy proof.

## Reproduction Assessment

Yes, the core failure is source-reproducible: current main sends truthy `proxyTls` into Undici SOCKS proxy construction, and Undici treats truthy `proxyTls` as HTTPS-to-proxy. A live Telegram/SOCKS getMe/sendMessage repro was not run in this read-only review.

## Solution Assessment

No, this is only a plausible partial fix. The better fix belongs in shared scheme handling that covers explicit and env SOCKS while preserving HTTP proxy connector options.
