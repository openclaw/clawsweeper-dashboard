## Summary

Keep open: the patch targets a real Telegram SOCKS proxy failure, but it is not merge-ready because it misses env SOCKS proxy paths, can drop HTTP proxy connector policy, and lacks inspectable real behavior proof.

## What This Changes

This PR changes `createHttp1ProxyAgent` so OpenClaw only applies its `proxyTls` target options when the explicit proxy URI starts with `https:`.

## Best Possible Solution

Fix shared proxy scheme handling so SOCKS avoids truthy proxy TLS, HTTP and HTTPS proxy endpoint policy remains intact, and explicit plus env proxy modes have focused tests and redacted live Telegram proof.

## Reproduction Assessment

Yes, source-reproducible for the core failure mode: Telegram explicit SOCKS proxy support reaches `createHttp1ProxyAgent`, and Undici 8.5.0 treats truthy `proxyTls` as HTTPS-to-SOCKS. I did not run a live Telegram/SOCKS repro in this read-only review.

## Solution Assessment

No. The fix belongs in shared proxy scheme handling, but this patch is too narrow because it misses env SOCKS proxies and can drop HTTP proxy endpoint connector policy.
