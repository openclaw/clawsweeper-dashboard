## What This Changes

The PR updates `src/infra/net/undici-runtime.ts` so `createHttp1ProxyAgent` only applies OpenClaw's `proxyTls` target when the proxy URI appears to use `https:`.

## Best Possible Solution

Fix scheme handling in the shared proxy helper layer so SOCKS avoids truthy `proxyTls`, HTTP/HTTPS proxy endpoint policy remains intact, and both explicit and env proxy modes have regression tests plus redacted real Telegram proxy proof.

## Reproduction Assessment

Yes by source inspection: documented Telegram SOCKS proxy support reaches the shared Undici proxy helpers, and Undici 8.3.0 treats truthy `proxyTls` as HTTPS-to-SOCKS in its SOCKS agent. I did not run a live SOCKS proxy repro in this read-only review.

## Solution Assessment

No. The patch addresses only the explicit proxy helper path and should be revised to cover env SOCKS proxies while preserving HTTP proxy endpoint connector policy.
