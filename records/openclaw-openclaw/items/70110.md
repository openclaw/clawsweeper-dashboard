## Summary

Keep open: current main still has the reported Nextcloud Talk reaction wiring gap. The plugin advertises reactions and has a sender helper, but it does not register a message action adapter, so shared `react` dispatch cannot reach the sender; two open PRs already track the narrow fix with closing syntax.

## What This Changes

The issue requests that Nextcloud Talk emoji reactions be wired so agent `message` tool `react` actions actually send a reaction.

## Best Possible Solution

Keep the issue open and review, land, or supersede one of the linked PRs with a plugin-owned Nextcloud Talk `ChannelMessageActionAdapter` that dispatches `react` to `sendReactionNextcloudTalk` and adds regression coverage for discovery and dispatch.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: `nextcloud-talk` lacks `actions.handleAction`, so `dispatchChannelMessageAction` returns `null` for `react` before `sendReactionNextcloudTalk` can run.

## Solution Assessment

Yes. The narrow maintainable fix is plugin-owned action wiring, not a generic core inference from `capabilities.reactions`; closing this issue should wait until one linked PR or an equivalent fix merges.
