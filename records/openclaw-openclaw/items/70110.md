## Summary

Keep open: current main still advertises Nextcloud Talk reactions and contains the guarded sender, but no Nextcloud Talk message-action adapter is registered, so `react` dispatch cannot reach the sender. Two open fixing PRs already close this issue, so the issue should remain open until one canonical implementation lands or supersedes the other.

## What This Changes

Nextcloud Talk should send emoji reactions when an agent uses the shared message tool's `react` action.

## Best Possible Solution

Land or supersede one canonical PR that registers a Nextcloud Talk-owned `ChannelMessageActionAdapter` for `react`, covers dispatch and sender behavior, and then close this issue after merge.

## Reproduction Assessment

Yes. Source inspection on current main shows `react` dispatch for `nextcloud-talk` has no `actions.handleAction` path and therefore cannot reach `sendReactionNextcloudTalk`.

## Solution Assessment

Yes. A plugin-owned action adapter is the narrow maintainable fix because reactions are channel-specific behavior and the shared chat builder does not infer handlers from capability flags.
