## What This Changes

Nextcloud Talk advertises reaction support, but agent `react` actions cannot send emoji reactions because the existing sender is not wired into channel message actions.

## Best Possible Solution

Review and land one canonical Nextcloud Talk-owned `ChannelMessageActionAdapter` for `react`, with dispatch and sender coverage, then close the issue after merge.

## Reproduction Assessment

Yes. Source inspection on current main shows Nextcloud Talk advertises reactions and has a sender, but no `actions.handleAction` registration exists, so shared `react` dispatch returns no handler.

## Solution Assessment

Yes. A plugin-owned action adapter is the narrow maintainable fix because reactions are channel-specific behavior and the shared chat builder does not infer handlers from capability flags.
