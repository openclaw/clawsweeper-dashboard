## Summary

Keep this open because current main still advertises Nextcloud Talk reactions without a registered message action adapter, and two active PRs already reference this issue as the implementation path.

## What This Changes

Nextcloud Talk should send emoji reactions through the shared message action surface instead of leaving `sendReactionNextcloudTalk` unreachable.

## Best Possible Solution

Land or supersede #72348 and #72679 with one narrow Nextcloud Talk-owned `ChannelMessageActionAdapter` for `react`, plus focused discovery/dispatch coverage.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: dispatching `react` for `nextcloud-talk` reaches the shared dispatcher, finds no `plugin.actions.handleAction`, and returns unsupported before `sendReactionNextcloudTalk` can run.

## Solution Assessment

Yes. A plugin-owned action adapter is the narrow maintainable fix because reactions are channel-specific behavior and the shared chat builder does not infer action handlers from capability flags.
