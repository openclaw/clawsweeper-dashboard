## Summary

Keep this open: current main and the latest shipped release still advertise Nextcloud Talk reaction support, but the channel plugin does not register an action adapter, so `react` dispatch cannot reach the existing reaction sender. Two open PRs already reference this issue with closing syntax, so the issue should remain open until one canonical implementation lands or supersedes the other.

## What This Changes

The issue requests that Nextcloud Talk message reactions be dispatchable through OpenClaw instead of leaving the existing reaction sender unreachable.

## Best Possible Solution

Land or supersede one canonical PR that registers a Nextcloud Talk-owned `ChannelMessageActionAdapter` for `react`, covers dispatch and sender request shape, and then close this issue after merge.

## Reproduction Assessment

Yes. Source inspection on current main shows `nextcloud-talk` advertises reaction support and has `sendReactionNextcloudTalk`, but the plugin lacks `actions.handleAction`, so the shared dispatcher cannot call it.

## Solution Assessment

Yes. A plugin-owned action adapter is the narrow maintainable fix because reactions are channel-specific behavior and the shared chat builder does not infer handlers from capability flags; the remaining step is to consolidate and land one linked PR.
