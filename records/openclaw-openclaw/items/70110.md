## Summary

Keep open: current main still advertises Nextcloud Talk reactions but does not register a channel action handler, so the shared `react` dispatch path still cannot reach the existing sender. The linked open implementation PR is the right closure path once it lands.

## What This Changes

The issue requests wiring Nextcloud Talk emoji reactions so agent `react` actions actually call the existing reaction sender.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/72348 or an equivalent plugin-owned action adapter, then close this issue after the fix merges.

## Reproduction Assessment

Yes. Source inspection on current main shows Nextcloud Talk advertises reactions and has a guarded sender, but no `actions.handleAction` registration exists, so shared `react` dispatch returns `null`.

## Solution Assessment

Yes. A Nextcloud Talk-owned `ChannelMessageActionAdapter` is the narrow maintainable fix because reactions are channel-specific behavior and the shared dispatcher does not infer handlers from capability flags.
