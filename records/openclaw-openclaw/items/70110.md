## Summary

Keep this open: current main still advertises Nextcloud Talk reaction support, but no Nextcloud Talk message-action adapter is registered, so `react` dispatch cannot reach the existing reaction sender. Two open fixing PRs already reference this issue with closing syntax, so the issue should remain open until one canonical implementation lands or supersedes the other.

## What This Changes

The issue asks OpenClaw to make the advertised Nextcloud Talk emoji reaction capability actually send reactions to messages.

## Best Possible Solution

Land or supersede one canonical PR that registers a Nextcloud Talk-owned `ChannelMessageActionAdapter` for `react`, covers dispatch and sender behavior, then close the issue after merge.

## Reproduction Assessment

Yes. Source inspection on current main shows `react` dispatch for `nextcloud-talk` has no `actions.handleAction` path and therefore cannot reach `sendReactionNextcloudTalk`.

## Solution Assessment

Yes. A plugin-owned action adapter is the narrow maintainable fix because reactions are channel-specific behavior and the shared chat builder does not infer handlers from capability flags.
