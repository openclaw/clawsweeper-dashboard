## Summary

Keep open: current main and the latest shipped release still advertise Nextcloud Talk reactions without registering a channel action handler, and the open linked implementation PR is the right place to resolve it before closing this issue.

## What This Changes

The issue requests wiring Nextcloud Talk emoji reactions so agent `react` actions call the existing Nextcloud Talk reaction sender.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/72348 or an equivalent Nextcloud Talk-owned action adapter, then close this issue after the fix merges.

## Reproduction Assessment

Yes. Source inspection on current main shows Nextcloud Talk advertises reactions and has a guarded sender, but no `actions.handleAction` registration exists, so shared `react` dispatch returns no handler.

## Solution Assessment

Yes. A plugin-owned `ChannelMessageActionAdapter` is the narrow maintainable fix because reactions are channel-specific behavior and the shared chat builder does not infer handlers from capability flags.
