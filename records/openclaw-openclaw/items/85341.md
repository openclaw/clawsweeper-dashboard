Full review comments:

- **[P1] Use manifest env vars for provider API-key lookup:** `src/llm/env-api-keys.ts:131-163`
  - body: `getApiKeyEnvVars` hardcodes a central provider map instead of using provider-owned manifest/setup metadata. It misses the canonical `moonshot` provider and `KIMI_API_KEY`, and maps `cloudflare-ai-gateway` to `CLOUDFLARE_API_KEY` even though that plugin declares `CLOUDFLARE_AI_GATEWAY_API_KEY`; because `AuthStorage.hasAuth`/`getAuthStatus` call this resolver, env-only installs for those providers are reported as unauthenticated.
  - confidence: 0.9
- **[P2] Require the Harmony `to=` marker before buffering text:** `src/plugin-sdk/provider-stream-shared.ts:142-147`
  - body: This branch returns `true` for bare `analysis`, `final`, or `commentary` prefixes before seeing a `to=` tool-call target. Since `wrapPlainTextToolCallStream` buffers text while this predicate is true, ordinary prose starting with those words stops streaming incrementally for providers using this wrapper, instead of flushing once it is clearly not a tool call as current main does.
  - confidence: 0.86
- **[P2] Preserve CRLF bracketed tool-call parsing:** `src/plugin-sdk/provider-stream-shared.ts:115-116`
  - body: The bracketed tool-call precheck now treats only `\n` after `]` as the JSON payload separator. Current main also accepts `\r\n`; losing that path means Windows-newline tool-call text can be classified as plain text and never promoted into a tool call on providers whose final `done` payload cannot reconstruct it.
  - confidence: 0.82
- **[P3] Remove the direct changelog edit:** `CHANGELOG.md:228`
  - body: `CHANGELOG.md` is release-owned in this repository; normal PRs should carry release-note context in the PR body or commit message and let release generation update the changelog. Please drop this entry from the branch.
