Full review comments:

- **[P1] Honor explicit OpenClaw runtime pins before CLI routing:** `src/auto-reply/reply/agent-runner-execution.ts:1681-1692`
  - body: `sessionRuntimeOverride` now returns `"openclaw"` for the built-in runtime, but this block only short-circuits CLI values before calling `resolveCliRuntimeExecutionProvider`. A session explicitly pinned to the built-in runtime can therefore be rerouted to a CLI backend by auth-profile inference, whereas current main stops the old built-in `pi` override at the provider path. Add a non-CLI `openclaw` branch before CLI inference.
  - confidence: 0.88
- **[P1] Use the shared graceful process-tree killer:** `src/agents/utils/shell.ts:200-215`
  - body: This new helper hard-kills the whole process tree immediately with `taskkill /F` or SIGKILL. The repository already has `src/process/kill-tree.ts`, which sends graceful termination first and only force-kills after a bounded grace period; duplicating the hard-kill path reopens that safety boundary.
  - confidence: 0.9
- **[P2] Normalize legacy `pi` model directive runtime values:** `src/auto-reply/reply/directive-handling.persist.ts:55-57`
  - body: `/model ... --runtime pi` was a shipped way to pin the built-in runtime, but this parser now only accepts `openclaw` and treats `pi` as invalid. Existing chats and scripts using the old token will silently lose the runtime override instead of migrating to the renamed built-in runtime.
  - confidence: 0.86
- **[P2] Preserve legacy Discord runtime picker submissions:** `extensions/discord/src/monitor/native-command-model-picker-interaction.ts:135-139`
  - body: When there are no provider runtime choices, this gate keeps only `openclaw`; a current session still carrying raw `agentRuntimeOverride: "pi"` is read by `resolveDiscordModelPickerCurrentRuntime` and then dropped here. Model picker submissions can therefore switch old Discord sessions back to provider defaults instead of preserving the built-in runtime.
  - confidence: 0.82
- **[P2] Accept legacy `pi` runtime-pair replay inputs:** `extensions/qa-lab/src/jsonl-replay.ts:169-172`
  - body: Saved QA replay inputs and older automation can still contain `runtimePair: ["pi", "codex"]`; this type/assertion now only accepts `openclaw,codex` and throws before replaying. Normalize `pi` to `openclaw` at the replay boundary so historical evidence remains usable during the rename.
  - confidence: 0.8
- **[P2] Keep `OPENCLAW_QA_FORCE_RUNTIME=pi` as a built-in alias:** `src/agents/model-runtime-policy.ts:183-187`
  - body: Private QA environments that still export the shipped `OPENCLAW_QA_FORCE_RUNTIME=pi` no longer force the built-in runtime because this branch accepts only `openclaw` or `codex`.
