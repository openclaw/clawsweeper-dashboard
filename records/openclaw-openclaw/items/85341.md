- **[P1] Restore manifest-backed env auth lookup:** `src/llm/env-api-keys.ts:131-163`
  - body: The new resolver only checks this hardcoded map, but bundled provider manifests still declare envVars that are missing or incomplete here, such as Arcee, Qwen, MiniMax token-plan, OpenCode Zen, and Z.ai secondary keys.   - body: This public SDK subpath no longer exports resolveLegacyPiAgentAccessToken, which current main exposed and bundled Z.ai imported from openclaw/plugin-sdk/provider-usage. Existing provider plugins using that shipped export will fail to type-check or load after upgrade, so keep a deprecated alias until the SDK removal window is explicit.
  - confidence: 0.93
- **[P1] Normalize legacy pi runtime ids:** `src/agents/agent-runtime-id.ts:11-20`
  - body: normalizeEmbeddedAgentRuntime maps openclaw/auto/codex-app-server but lets the shipped legacy value pi fall through as an arbitrary runtime id. Persisted session/config state carrying pi can then be treated as a missing plugin harness instead of the built-in runtime, which is an upgrade regression.
  - confidence: 0.92
- **[P1] Keep providerDiscoveryEntry as a manifest fallback:** `src/plugins/manifest-registry.ts:469-475`
  - body: The record builder now ignores the deprecated providerDiscoveryEntry field entirely, even though current main used it as the fallback when providerCatalogEntry was absent.   - confidence: 0.94
- **[P1] Reject nested paths for official install trust:** `src/plugins/manifest-registry.ts:783-785`
  - body: Adding isPathInside(candidatePath, trackedPath) means a config or global candidate rooted inside a recorded official plugin install can be marked as trustedOfficialInstall even though it is not the recorded package root.   - confidence: 0.9
- **[P1] Map legacy pi middleware runtimes:** `src/plugins/agent-tool-result-middleware.ts:17-19`
  - body: The middleware normalizer only maps codex-app-server as a legacy runtime; shipped plugin configs or typed constants that still use runtimes/harnesses: ["pi"] now normalize to no built-in runtime match. Tool-result middleware can silently stop running on OpenClaw runtime turns.
  - confidence: 0.88
- **[P2] Keep PI_CODING_AGENT_DIR in the SDK resolver:** `src/plugin-sdk/agent-dir-compat.ts:8-10`
  - body: resolveOpenClawAgentDir is explicitly a deprecated third-party compatibility helper, but it now ignores PI_CODING_AGENT_DIR. Upgraded plugin/runtime code using this public helper will fall back to the default agent directory instead of the operator-selected legacy agent root.
  - confidence: 0.9

  - confidence: 0.88
