  - body: This schema remains `.strict()` but removes the shipped `plugins.bundledDiscovery` key.   - confidence: 0.94
- **[P1] Normalize legacy pi runtime ids before selection:** `src/agents/agent-runtime-id.ts:20`
  - body: `normalizeEmbeddedAgentRuntime` returns unknown values unchanged and harness selection only treats `openclaw` as built-in, so persisted or configured `agentRuntime.id: "pi"` can fall through to plugin-harness lookup and hard-fail as missing. Map `pi` to `openclaw` at this compatibility boundary and avoid lowercasing arbitrary custom ids.
  - confidence: 0.91
- **[P2] Keep PI_CODING_AGENT_DIR for compatibility lookups:** `src/plugin-sdk/agent-dir-compat.ts:9-10`
  - body: The deprecated SDK helper now ignores the shipped legacy `PI_CODING_AGENT_DIR` alias, and changed secret-scan/models/migration paths use the same new-only behavior. Existing env-based installs can read, probe, or audit the default agent directory instead of the active agent directory.
  - confidence: 0.9
- **[P2] Keep PI_BASH aliases for exec controls:** `src/agents/bash-tools.exec-runtime.ts:117`
  - body: The exec runtime now reads only `OPENCLAW_BASH_MAX_OUTPUT_CHARS` here, with the same alias removal pattern for the bash env family. Deployments still exporting `PI_BASH_*` controls will silently fall back to defaults, changing output caps, pacing, or TTL after upgrade.
  - confidence: 0.86
- **[P2] Keep PI_CACHE_RETENTION as a cache fallback:** `src/agents/openai-transport-stream.ts:1834`
  - body: This cache-retention resolver now checks only `OPENCLAW_CACHE_RETENTION`, so installs still using `PI_CACHE_RETENTION=long` lose long-retention behavior on OpenAI transport paths and fall back to short caching without a migration warning.
  - confidence: 0.86
- **[P1] Use provider-declared env vars in key lookup:** `src/llm/env-api-keys.ts:146-155`
  - body: The new env-key map has no `moonshot` entry and maps `cloudflare-ai-gateway` to `CLOUDFLARE_API_KEY`, while the provider manifests declare `moonshot` with `MOONSHOT_API_KEY`/`KIMI_API_KEY` and Cloudflare AI Gateway with `CLOUDFLARE_AI_GATEWAY_API_KEY`. Env-only auth for those providers can be reported missing at runtime.
