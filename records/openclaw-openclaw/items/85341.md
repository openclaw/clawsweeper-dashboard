Full review comments:

- **[P1] Honor explicit OpenClaw pins before CLI inference:** `src/auto-reply/reply/agent-runner-execution.ts:1686-1697`
  - body: When `sessionRuntimeOverride` is `openclaw`, this path still calls `resolveCliRuntimeExecutionProvider`, so an auth profile can route a session explicitly pinned to the built-in runtime onto a CLI backend. Preserve the non-CLI short-circuit before CLI/auth inference.
  - confidence: 0.92
- **[P1] Keep followup turns on the pinned runtime:** `src/auto-reply/reply/followup-runner.ts:652-663`
  - body: The followup runner repeats the same fallthrough, so queued followup turns can ignore an explicit OpenClaw runtime pin and execute on a CLI harness selected from auth-profile inference. Apply the same pinned-built-in short-circuit here.
  - confidence: 0.9
- **[P2] Preserve case for custom runtime ids:** `src/agents/agent-runtime-id.ts:30-31`
  - body: `normalizeOptionalAgentRuntimeId` lowercases arbitrary ids before harness selection, but registered harness ids are stored and looked up case-sensitively. Previously working mixed-case plugin runtime ids can now fail as missing harnesses.
  - confidence: 0.88
- **[P2] Normalize legacy QA runtime aliases:** `src/agents/model-runtime-policy.ts:184-186`
  - body: `OPENCLAW_QA_FORCE_RUNTIME=pi` is no longer accepted here, while other runtime paths still map `pi` to the built-in runtime. Existing private QA environments can silently lose their forced runtime and fall back to normal selection.
  - confidence: 0.86
- **[P2] Treat legacy pi history as embedded runtime context:** `src/agents/embedded-agent-runner/run/helpers.ts:104-105`
  - body: Pre-rename transcripts can still contain assistant messages tagged with provider `pi`. Recognizing only `openclaw` here makes error-context resolution treat those embedded messages as a real provider, which can misclassify failover and cooldown handling.
  - confidence: 0.84
- **[P1] Generate OAuth state separately from PKCE verifier:** `src/llm/utils/oauth/anthropic.ts:281`
  - body: The Anthropic OAuth URL uses the PKCE verifier as `state`. OAuth state should be independent nonce material so a leaked or replayed state value does not also expose verifier material.
  - confidence: 0.86
- **[P2] Use graceful process-tree termination:** `packages/agent-core/src/harness/env/nodejs.ts:241`
  - body: The internalized node harness kills the whole process group with immediate `SIGKILL`. Use the existing graceful kill-tree helper pattern so child processes get SIGTERM and a bounded grace period before SIGKILL.
  - confidence: 0.82
- **[P2] Accept legacy QA runtime aliases:** `extensions/qa-lab/src/cli.runtime.ts:160-164`
  - body: `normalizeQaRuntimeId` now rejects `pi`, so existing QA invocations such as `--runtime-pair pi,codex` and saved replay inputs fail instead of mapping the shipped alias to `openclaw` during migration.
  - confidence: 0.84

  - body: Pre-rename session messages can still be tagged with provider `pi`; after this change `resolveReportedModelRef` treats that as a real provider instead of using the active model provider, which can misclassify failover or cooldown handling before a new assistant message is emitted.
  - confidence: 0.78
