  - body: Existing plugins can still declare `runtimes: ["pi"]` or deprecated `harnesses: ["pi"]`. This new normalizer only maps `codex-app-server`, so those manifests normalize to an empty/nonmatching runtime set and their tool-result middleware stops running on the built-in OpenClaw runtime.
  - confidence: 0.94

  - body: Changing the public middleware runtime type from `"pi" | "codex"` to `"openclaw" | "codex"` breaks third-party plugins that still compile against the shipped `pi` token. Keep `pi` as a deprecated accepted type and normalize it to `openclaw` at runtime.
  - confidence: 0.9

  - body: `openclaw/plugin-sdk/provider-usage` is a public SDK subpath, and existing plugins importing `resolveLegacyPiAgentAccessToken` will fail to build after this export is removed. Re-export it as a deprecated alias to the new helper until the SDK removal window is documented.
  - confidence: 0.92
