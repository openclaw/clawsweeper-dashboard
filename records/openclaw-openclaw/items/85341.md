## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep all documented provider env aliases:** `src/llm/env-api-keys.ts:131`
  - body: The new shared lookup can only return one env var per provider, but shipped plugin contracts expose multiple aliases and provider IDs. For example, Google still documents `GOOGLE_API_KEY`, Copilot documents `GH_TOKEN`/`GITHUB_TOKEN`, Cloudflare AI Gateway documents `CLOUDFLARE_AI_GATEWAY_API_KEY`, and Kimi/OpenCode have alternate keys; env-only setups using those aliases will be reported as unauthenticated and may skip runtime credential fallback.
  - confidence: 0.92
- **[P2] Require a Harmony tool target before buffering prose:** `src/plugin-sdk/provider-stream-shared.ts:143-147`
  - body: This branch now treats untagged text beginning with `analysis`, `final`, or `commentary` as a possible Harmony tool call even when no ` to=` tool target follows. Providers using `createPlainTextToolCallCompatWrapper` will buffer normal prose such as `final answer ...` until `done`, losing incremental streaming for ordinary responses.
  - confidence: 0.88
- **[P2] Do not match arbitrary tool-name prefixes as tool calls:** `src/plugin-sdk/provider-stream-shared.ts:61-62`
  - body: `text.startsWith(toolName)` means ordinary bracketed prose that merely starts with a tool name, such as `[readme ...]` when `read` is enabled, is kept in the possible-tool-call state. That suppresses streaming until later disambiguation or the final event; the check should only accept prefixes that can still become a syntactically valid tool call name.
  - confidence: 0.84
- **[P2] Preserve the configured Bedrock region fallback:** `extensions/amazon-bedrock/register.sync.runtime.ts:465`
  - body: When no provider baseUrl yields a region, this now returns `undefined`; the Bedrock runtime then falls back to `us-east-1` when no AWS profile/env region is present. Upgraded setups that still rely on the configured Bedrock discovery region can silently sign/send requests to the wrong region, so the old fallback needs either preservation or a tested migration path.
  - confidence: 0.86
