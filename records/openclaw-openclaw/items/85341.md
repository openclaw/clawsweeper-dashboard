Full review comments:

- **[P2] Preserve case-sensitive harness ids:** `src/agents/agent-runtime-id.ts:30-31`
  - body: Lowercasing every runtime override changes arbitrary plugin harness ids before selection. The registry stores and looks up harness ids with only `trim()`, so a previously valid mixed-case id can no longer be selected via `agentRuntime.id` or a runtime override. Keep only known built-in aliases case-normalized and preserve custom ids as registered.
  - confidence: 0.88
- **[P2] Keep the legacy QA force-runtime alias:** `src/agents/model-runtime-policy.ts:184-186`
  - body: This path still ignores `OPENCLAW_QA_FORCE_RUNTIME=pi` even though the QA CLI parser now maps `pi` to `openclaw`. Existing private QA environments with the shipped legacy value will silently fall back to normal runtime policy here, which can route provider/auth checks differently from the forced built-in runtime they requested.
  - confidence: 0.84
- **[P1] Use graceful process-tree termination in agent core:** `packages/agent-core/src/harness/env/nodejs.ts:226-241`
  - body: The internalized agent-core harness introduces a local process-tree killer that immediately sends SIGKILL/taskkill on timeouts and aborts. That bypasses the shared `src/process/kill-tree.ts` grace window and detached-process safeguards, reintroducing the exact process-boundary class that current process supervisor code now avoids.
  - confidence: 0.9
- **[P2] Treat legacy pi as an embedded provider marker:** `src/agents/embedded-agent-runner/run/helpers.ts:104-105`
  - body: Pre-rename transcripts can still contain assistant messages tagged with provider `pi`. Since this helper now recognizes only `openclaw` as the embedded marker, error/failover context can treat that legacy marker as a real provider instead of falling back to the active model provider.
  - confidence: 0.82
