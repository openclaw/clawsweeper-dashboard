  - body: `normalizeEmbeddedAgentRuntime` still returns unknown values unchanged, so stored `agentHarnessId: "pi"` or runtime overrides from existing sessions become a plugin runtime id instead of the built-in OpenClaw runtime.   - body: The new resolver only checks the hardcoded env map and returns undefined for manifest-declared provider IDs that are not listed. Providers such as `arcee` still declare env auth in their manifests, and `AuthStorage.hasAuth`/`getAuthStatus` rely on this path, so env-only setups can be reported unauthenticated and miss fallback keys.
  - confidence: 0.93

  - body: The manifest record builder now only considers `providerCatalogEntry`; current main still accepts the deprecated `providerDiscoveryEntry`.   - confidence: 0.9
- **[P1] Keep the provider-usage compatibility export:** `src/plugin-sdk/provider-usage.ts:17-20`
  - body: The public SDK subpath no longer re-exports `resolveLegacyPiAgentAccessToken`, even though current main exports it and provider plugins import this symbol. Removing the named export without a deprecated alias breaks existing plugins at build/runtime on upgrade.
  - confidence: 0.92

  - body: Tool-result middleware normalization now only remaps `codex-app-server`; manifests or plugin code that still pass `runtimes: ["pi"]` or legacy harnesses normalize to no built-in runtime. Existing middleware can silently stop running on OpenClaw turns, so the deprecated `pi` token needs to map to `openclaw`.
  - confidence: 0.9

  - body: This matcher treats a candidate path inside the recorded official install root as a trusted official install.
