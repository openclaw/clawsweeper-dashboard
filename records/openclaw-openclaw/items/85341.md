Full review comments:

- **[P1] Restore manifest-backed env auth lookup:** `src/llm/env-api-keys.ts:131-163`
  - body: `getApiKeyEnvVars` now returns values only from this hardcoded map, and callers like `AuthStorage.hasAuth`/`getAuthStatus` depend on `getEnvApiKey` and `findEnvKeys`. Manifest-owned providers that still declare env credentials, such as `arcee` with `ARCEEAI_API_KEY`, are not in this map and can be reported unauthenticated or lose env fallback until every provider is manually duplicated here; keep the manifest/setup-provider fallback or generate this contract from metadata.
  - confidence: 0.91
- **[P1] Keep the legacy provider-usage SDK export:** `src/plugin-sdk/provider-usage.ts:16`
  - body: This public SDK subpath drops `resolveLegacyPiAgentAccessToken` instead of keeping it as a deprecated alias. Existing provider plugins importing the shipped helper from `openclaw/plugin-sdk/provider-usage` will fail at build/runtime immediately after upgrade, so preserve the old export while the renamed API goes through a documented deprecation window.
  - confidence: 0.89
- **[P1] Restore cross-provider Mistral compat contribution:** `extensions/mistral/index.ts:46`
  - body: The Mistral provider now only normalizes models when the matched provider is `mistral`, while the deleted hook/test covered OpenAI-compatible Mistral-family routes such as OpenRouter or custom Mistral endpoints.
