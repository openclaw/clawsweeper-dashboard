## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve case for plugin runtime IDs:** `src/agents/agent-runtime-id.ts:30`
  - body: `normalizeOptionalAgentRuntimeId` lowercases every runtime id before selection, but plugin harness ids are registered and looked up case-sensitively. A mixed-case custom harness id that previously matched its registration will be changed to lowercase and then fail as missing.
  - confidence: 0.89
- **[P1] Honor OpenClaw runtime pins before CLI inference:** `src/auto-reply/reply/agent-runner-execution.ts:1678-1689`
  - body: An explicit persisted `agentRuntimeOverride: "openclaw"` now falls through to `resolveCliRuntimeExecutionProvider`. If auth/profile inference picks a CLI backend, the turn can run on the wrong runtime despite the session pin; keep the embedded-runtime short-circuit that existed for `pi`.
  - confidence: 0.9
- **[P1] Honor follow-up runtime pins before CLI inference:** `src/auto-reply/reply/followup-runner.ts:579-590`
  - body: The same CLI fall-through exists in the queued follow-up runner, so follow-up turns from a session pinned to the built-in runtime can be rerouted to a CLI backend. This needs the same explicit OpenClaw short-circuit as the main agent-runner path.
  - confidence: 0.88
- **[P1] Reuse graceful process-tree termination:** `packages/agent-core/src/harness/env/nodejs.ts:241-245`
  - body: The new agent-core helper sends immediate SIGKILL/taskkill `/F` for timeouts and aborts. OpenClaw already has a shared `src/process/kill-tree.ts` helper that sends SIGTERM first and only force-kills after a grace period; duplicating the hard-kill path regresses cleanup and process-boundary behavior.
  - confidence: 0.86
