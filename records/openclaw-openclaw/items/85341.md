## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Map legacy `pi` before selecting harnesses:** `src/agents/agent-runtime-id.ts:27-28`
  - body: `normalizeOptionalAgentRuntimeId` lowercases every runtime id and does not translate the shipped `pi` token. Legacy configs or session pins with `agentRuntime.id: "pi"` then fall through to plugin-harness lookup as `pi` and can throw `MissingAgentHarnessError`; the same lowercasing also breaks case-sensitive custom harness ids. Map `pi` to `openclaw` at this compatibility boundary and preserve casing for non-built-in ids.
  - confidence: 0.93
- **[P1] Preserve bundled discovery compatibility:** `src/plugins/bundled-compat.ts:22-23`
  - body: This path now filters bundled plugins through `plugins.allow` unconditionally, while the PR removes `plugins.bundledDiscovery: "compat"`. Existing configs that doctor migrated to preserve bundled provider discovery under a restrictive allowlist will silently lose provider/web capabilities unless this compat behavior is kept or replaced by a tested migration.
  - confidence: 0.88
