- **[P1] Restore manifest-driven env auth lookup:** `src/llm/env-api-keys.ts:131-163`
  - body: `getApiKeyEnvVars` now resolves credentials from a fixed table, so manifest-declared providers that are absent or use different env names are treated as unauthenticated. For example, the Cloudflare AI Gateway manifest declares `CLOUDFLARE_AI_GATEWAY_API_KEY`, but this map checks `CLOUDFLARE_API_KEY`; provider auth should keep using manifest/setup metadata as the source of truth instead of requiring every plugin id to be hard-coded here.
  - confidence: 0.94
- **[P1] Preserve the provider-usage SDK token helper:** `src/plugin-sdk/provider-usage.ts:16`
  - body: This public SDK subpath removes `resolveLegacyPiAgentAccessToken` without a deprecated alias, so existing provider plugins importing the shipped symbol fail to build or load after upgrade. Keep the old export pointing at the new implementation until the public plugin API has a documented removal window.
  - confidence: 0.91
- **[P1] Keep providerDiscoveryEntry as a manifest fallback:** `src/plugins/manifest-registry.ts:469-475`
  - body: `providerSourceEntry` now ignores the legacy `providerDiscoveryEntry` spelling even though current docs and tests say it still works for existing plugins. External plugins that have not renamed their manifest key will lose provider discovery/catalog loading, so this should continue to prefer `providerCatalogEntry` but fall back to `providerDiscoveryEntry`.
  - confidence: 0.93
- **[P1] Normalize legacy pi runtime ids:** `src/agents/agent-runtime-id.ts:17-20`
  - body: Legacy persisted session/config state can still contain `pi` as the built-in runtime id, but this normalizer returns unknown values unchanged. That lets `pi` flow as a non-default plugin runtime instead of mapping to `openclaw`, which can fail closed or select the wrong runtime during upgrade.
  - confidence: 0.89
- **[P2] Keep PI_CODING_AGENT_DIR in the SDK resolver:** `src/plugin-sdk/agent-dir-compat.ts:8-10`
  - body: `resolveOpenClawAgentDir` is explicitly retained for third-party SDK compatibility, but it now honors only `OPENCLAW_AGENT_DIR`. Upgraded plugin/runtime callers in environments that still export `PI_CODING_AGENT_DIR` will silently read the default agent directory instead of the configured one.
  - confidence: 0.86
- **[P2] Preserve legacy cache-retention env fallback:** `src/agents/anthropic-payload-policy.ts:59-60`
  - body: The cache-retention resolver now checks only `OPENCLAW_CACHE_RETENTION`, so upgraded Anthropic/OpenAI-family environments that still set `PI_CACHE_RETENTION=long` fall back to short cache retention. Keep the legacy env alias during the migration window or add a doctor-backed migration before removing it.
  - confidence: 0.84
