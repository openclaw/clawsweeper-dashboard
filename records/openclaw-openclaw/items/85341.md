## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Honor explicit OpenClaw pins before CLI inference:** `src/auto-reply/reply/followup-runner.ts:579-583`
  - body: `resolveSessionRuntimeOverrideForProvider` now normalizes legacy `pi` pins to `openclaw`, but this path only short-circuits CLI providers before falling through to `resolveCliRuntimeExecutionProvider`. A session explicitly pinned to the built-in runtime can therefore be rerouted to an inferred CLI backend such as `claude-cli`, so keep the non-CLI built-in short-circuit before CLI inference in both normal and follow-up routing.
  - confidence: 0.88
- **[P2] Keep pi as a QA force-runtime alias:** `src/agents/model-runtime-policy.ts:185-186`
  - body: Current main treats `OPENCLAW_QA_FORCE_RUNTIME=pi` as the private-QA built-in runtime override. This PR only accepts `openclaw` or `codex`, so existing QA environments with the shipped legacy value silently fall back to normal model policy instead of forcing the built-in runtime.
  - confidence: 0.86
- **[P1] Use graceful process-tree termination:** `packages/agent-core/src/harness/env/nodejs.ts:226-248`
  - body: This new helper kills Windows trees with `/F` and Unix process groups with immediate `SIGKILL`. OpenClaw already has a shared graceful process-tree contract that sends `SIGTERM`, waits a bounded grace period, and only then force-kills; duplicating the hard-kill behavior reintroduces the process-boundary risk flagged by security review.
  - confidence: 0.84
- **[P2] Normalize legacy runtime pairs in JSONL replay:** `extensions/qa-lab/src/jsonl-replay.ts:170-171`
  - body: Saved or scripted QA replay inputs that still contain `runtimePair: ["pi", "codex"]` worked on current main, but this check now rejects them as unsupported instead of normalizing `pi` to `openclaw`. That makes historical replay automation fail during the runtime rename migration.
  - confidence: 0.8
