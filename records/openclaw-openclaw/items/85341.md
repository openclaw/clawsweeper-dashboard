Full review comments:

- **[P1] Preserve providerDiscoveryEntry manifest fallback:** `src/plugins/manifest-registry.ts:475`
  - body: This now ignores the deprecated `providerDiscoveryEntry` key when building `providerDiscoverySource`. External plugins that have not renamed their manifest field will be treated as having no discovery entry, losing provider discovery/auth catalog loading until manually edited; keep the fallback or add a tested migration.
  - confidence: 0.94
- **[P1] Do not trust parent paths as official installs:** `src/plugins/manifest-registry.ts:785`
  - body: Adding `isPathInside(candidatePath, trackedPath)` lets a configured candidate directory that merely contains the recorded official install be marked `trustedOfficialInstall`. That broadens the official-plugin trust boundary beyond the installed package root, so this should stay exact or child-of-recorded-root only.
  - confidence: 0.91
- **[P1] Restore manifest-backed env auth lookup:** `src/llm/env-api-keys.ts:121`
  - body: `getApiKeyEnvVars` is now a fixed provider map, so manifest-declared providers not listed here, such as Arcee's `ARCEEAI_API_KEY`, return no env key. Since auth status and env fallback paths depend on this lookup, env-only provider setups can become falsely unauthenticated; keep a manifest/setup-provider fallback.
  - confidence: 0.9
- **[P1] Keep the deprecated provider-usage token export:** `src/plugin-sdk/provider-usage.ts:16`
  - body: Removing `resolveLegacyPiAgentAccessToken` from the public `openclaw/plugin-sdk/provider-usage` barrel breaks existing third-party plugins that still import the shipped helper. The SDK boundary needs a deprecated alias until a documented removal window passes.
  - confidence: 0.93
- **[P1] Map legacy pi middleware runtime to OpenClaw:** `src/plugins/agent-tool-result-middleware.ts:17-19`
  - body: The middleware normalizer now preserves only `codex-app-server`, so manifests or typed plugin code that still use `runtimes: ["pi"]` or deprecated harness values normalize to no built-in runtime. Preserve `pi -> openclaw` during the compatibility window so existing middleware still runs on upgraded installs.
  - confidence: 0.89
- **[P2] Require Harmony tool marker before buffering prose:** `src/plugin-sdk/provider-stream-shared.ts:142-147`
  - body: This branch now returns `true` for untagged text that starts with `analysis`, `final`, or `commentary` before requiring a `to=` tool marker. In wrapped providers, normal prose such as `final answer...` stays buffered until `done` instead of streaming incrementally; require the Harmony tool-call marker before keeping those prefixes buffered.
  - confidence: 0.88
