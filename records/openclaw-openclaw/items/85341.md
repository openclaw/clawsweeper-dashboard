  - confidence: 0.91

  - body: The queued followup path has the same fallthrough: a persisted `openclaw` runtime override skips the CLI-provider branch but still lets `resolveCliRuntimeExecutionProvider` choose a CLI backend. Followup turns should respect the explicit built-in runtime pin before auth-profile inference.
  - confidence: 0.9
- **[P2] Preserve casing for custom runtime ids:** `src/agents/agent-runtime-id.ts:30-31`
  - body: Lowercasing every runtime id makes mixed-case plugin harness ids unselectable, because `registerAgentHarness` and `getRegisteredAgentHarness` preserve case and look up the trimmed id exactly. Only normalize known built-in aliases such as `pi`, `openclaw`, `auto`, and `codex-app-server`.
  - confidence: 0.88

  - body: The new Anthropic OAuth flow sends the PKCE verifier as `state`, tying CSRF state to verifier material. Generate an independent random state token and keep the verifier only for the PKCE challenge/exchange path.
  - confidence: 0.86

  - body: This new helper force-kills the process tree immediately on both Windows and Unix, bypassing the existing `src/process/kill-tree.ts` behavior that sends SIGTERM first and escalates only after a grace period. Reuse the shared helper so shell timeouts and aborts do not skip cleanup or ownership safeguards.
  - confidence: 0.83
