Overall confidence: 0.88

Full review comments:

- **[P1] Honor embedded session pins before CLI inference:** `src/auto-reply/reply/agent-runner-execution.ts:1679-1682`
  - body: `resolveSessionRuntimeOverrideForProvider` returns `"openclaw"` for explicit embedded pins, including legacy `pi`, but this branch only short-circuits CLI runtime ids before calling `resolveCliRuntimeExecutionProvider`. That lets an explicitly embedded session reroute to a CLI provider based on auth/profile inference, so preserve the OpenClaw pin by using the model provider directly before CLI inference.
  - confidence: 0.91
- **[P1] Honor embedded pins in queued followups:** `src/auto-reply/reply/followup-runner.ts:579-583`
  - body: The same CLI-inference fallback is present in the followup runner, so queued follow-up turns can ignore an explicit OpenClaw/legacy-pi session runtime pin and run on a CLI harness. Add the same non-CLI embedded short-circuit here before resolving a CLI execution provider.
  - confidence: 0.9
- **[P1] Keep the deprecated provider-usage export:** `src/plugin-sdk/provider-usage.ts:34-37`
  - body: This public SDK barrel used to export `resolveLegacyPiAgentAccessToken`, but the PR only exports the renamed helper. Existing plugins importing the shipped symbol from `openclaw/plugin-sdk/provider-usage` will fail to build, so re-export the old name as a deprecated alias until a documented removal window passes.
  - confidence: 0.9
- **[P2] Accept the legacy QA runtime token:** `src/agents/model-runtime-policy.ts:184-186`
  - body: Private QA environments that still export `OPENCLAW_QA_FORCE_RUNTIME=pi` now miss the force-runtime override and fall back to normal policy selection. Since the PR keeps pi-to-openclaw compatibility elsewhere, normalize `pi` to `openclaw` here instead of dropping it.
  - confidence: 0.92
- **[P2] Use graceful process-tree termination:** `packages/agent-core/src/harness/env/nodejs.ts:226-241`
  - body: This new helper immediately sends `SIGKILL` or `taskkill /F`, bypassing the existing `src/process/kill-tree.ts` behavior that first sends a graceful termination signal and force-kills only after a bounded grace period. Reuse or mirror the shared helper so aborted commands can clean up sockets, temp files, and children before hard kill.
  - confidence: 0.88
- **[P2] Normalize legacy replay runtime pairs:** `extensions/qa-lab/src/jsonl-replay.ts:169-171`
  - body: Saved QA replay inputs that still carry `runtimePair: ["pi", "codex"]` now fail with `unsupported jsonl replay runtime pair` instead of replaying through the renamed embedded runtime. Normalize `pi` to `openclaw` at this replay boundary before enforcing the pair.
  - confidence: 0.86
- **[P2] Keep legacy runtime in Discord picker submissions:** `extensions/discord/src/monitor/native-command-model-picker-interaction.ts:135-138`
  - body: When a persisted Discord session still has `agentRuntimeOverride: "pi"` and no provider runtime choices exist, this check drops the runtime instead of preserving the embedded route. Normalize legacy `pi` to `openclaw` here so model changes do not silently switch old sessions back to provider defaults.
  - confidence: 0.84
- **[P2] Preserve runtime id casing for custom harnesses:** `src/agents/agent-runtime-id.ts:30-31`
  - body: `normalizeOptionalAgentRuntimeId` lowercases every runtime id before selection, but harness registration and lookup use the trimmed id case-sensitively. Mixed-case custom plugin harness ids that previously matched exactly can now be selected as a lowercased id and fail as missing.
  - confidence: 0.82
