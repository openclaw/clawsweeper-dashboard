  - body: The secret scan now derives the active agent directory from `OPENCLAW_AGENT_DIR` only. Installs still using `PI_CODING_AGENT_DIR` can have their real `models.json` omitted from credential leakage checks, producing false-negative audit results.

  - body: Switching the exec runtime to only the `OPENCLAW_BASH_*` names drops shipped `PI_BASH_*` controls for output caps, yield timing, and job TTL. Existing deployments then silently fall back to defaults, changing runtime behavior and resource limits on upgrade.

  - body: The provider cache policy now checks only `OPENCLAW_CACHE_RETENTION`, so installs that still export `PI_CACHE_RETENTION=long` silently lose long-retention prompt caching and revert to short cache behavior on Anthropic/OpenAI paths.
  - confidence: 0.87
