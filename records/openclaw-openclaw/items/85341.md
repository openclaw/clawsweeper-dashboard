Overall confidence: 0.87

Full review comments:

- **[P1] Honor OpenClaw runtime pins before CLI inference:** `src/auto-reply/reply/agent-runner-execution.ts:1686-1697`
  - body: When `resolveSessionRuntimeOverrideForProvider` returns `openclaw`, this block still calls `resolveCliRuntimeExecutionProvider`, so a session explicitly pinned to the built-in runtime can be rerouted to a CLI provider through auth/profile inference. Add a built-in-runtime short-circuit before CLI resolution, matching the shipped `pi` behavior.
  - confidence: 0.9
- **[P1] Honor OpenClaw pins in queued followups too:** `src/auto-reply/reply/followup-runner.ts:652-663`
  - body: Queued followups repeat the same fallthrough: `sessionRuntimeOverride === "openclaw"` is not treated as a terminal built-in runtime choice before CLI inference. This can make queued turns ignore the user's persisted runtime pin and run on the wrong backend.
  - confidence: 0.88
- **[P2] Preserve runtime id casing for plugin harnesses:** `src/agents/agent-runtime-id.ts:30-31`
  - body: `normalizeOptionalAgentRuntimeId` lowercases every runtime id, but harness ids are registered and looked up with their original trimmed casing. Mixed-case custom harness ids that worked before will now be looked up under a different id and fail as missing.
  - confidence: 0.86
- **[P2] Keep legacy pi assistant metadata embedded:** `src/agents/embedded-agent-runner/run/helpers.ts:104-105`
  - body: Pre-rename transcripts can still contain assistant metadata with `provider: "pi"`. After this change `resolveReportedModelRef` treats that as a real provider instead of the embedded runtime marker, so error and failover context can be attributed to the wrong provider.
  - confidence: 0.84
- **[P1] Use graceful process-tree termination:** `packages/agent-core/src/harness/env/nodejs.ts:226-245`
  - body: The new agent-core helper hard-kills process trees with `/F` or `SIGKILL` on timeout and abort. OpenClaw already uses a bounded graceful termination pattern that lets children clean up before escalation; mirror or inject that behavior here instead of reintroducing immediate hard kill semantics.
  - confidence: 0.87
