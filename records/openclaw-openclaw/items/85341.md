## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Map legacy `pi` runtime to OpenClaw:** `src/agents/agent-runtime-id.ts:20`
  - body: `normalizeEmbeddedAgentRuntime("pi")` now returns `"pi"`, but the built-in harness is selected only for `"openclaw"`; explicit non-auto runtimes that do not match a plugin fall through to `MissingAgentHarnessError`. Upgraded sessions or configs carrying the previously shipped `pi` runtime id can therefore fail closed instead of using the built-in runtime, so this normalizer should treat `pi` as a deprecated alias for `openclaw`.
  - confidence: 0.93
- **[P1] Restore manifest-backed env auth lookup:** `src/llm/env-api-keys.ts:121-164`
  - body: The new resolver hardcodes a small provider-to-env map, while provider manifests still declare supported env vars such as `ARCEEAI_API_KEY`, `HUGGINGFACE_HUB_TOKEN`, `GH_TOKEN`/`GITHUB_TOKEN`, Qwen aliases, `OPENCODE_ZEN_API_KEY`, and MiniMax token-plan keys. Because `AuthStorage.hasAuth()` and `getAuthStatus()` now rely on `getEnvApiKey()`/`findEnvKeys()`, env-only installs for those providers can be reported unauthenticated or miss credential fallback until users rename already-supported variables.
  - confidence: 0.94
- **[P1] Do not over-trust parent plugin paths:** `src/plugins/manifest-registry.ts:780`
  - body: The added reverse containment branch makes an install record match when the recorded official install lives under a candidate path, not only when the candidate is the recorded package root. That can let a config/global plugin source that wraps an official install be marked `trustedOfficialInstall`, which weakens the trust boundary used for official-only plugin behavior.
  - confidence: 0.88
