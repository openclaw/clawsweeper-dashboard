  - body: Existing plugins can still declare `runtimes: ["pi"]` or deprecated `harnesses: ["pi"]`. With this runtime set, those declarations normalize to nothing and their tool-result middleware stops running on the built-in runtime, so accept `pi` as a deprecated alias for `openclaw`.
  - confidence: 0.96

  - confidence: 0.9

  - body: Existing private-QA jobs that export `OPENCLAW_QA_FORCE_RUNTIME=pi` now lose the override and fall back to normal runtime selection, even though the runtime stack still normalizes `pi` to OpenClaw elsewhere. Treat `pi` as the legacy OpenClaw alias here too.
