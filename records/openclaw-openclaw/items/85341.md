  - confidence: 0.9

  - body: The follow-up runner has the same routing regression: a persisted `openclaw` runtime override falls through to CLI provider inference because only CLI runtime IDs are honored here. Queued follow-up turns can therefore run on a CLI harness even though the session was pinned to the built-in runtime.
  - confidence: 0.9
- **[P2] Preserve the legacy QA runtime alias:** `src/agents/model-runtime-policy.ts:183-187`
  - body: Changing this gate to accept only `openclaw` or `codex` drops the shipped `OPENCLAW_QA_FORCE_RUNTIME=pi` value, while other compatibility paths still treat `pi` as the built-in runtime alias. Normalize `pi` to `openclaw` here so private QA automation does not silently change runtime behavior.
  - confidence: 0.88
- **[P2] Accept `pi` in model runtime directives:** `src/auto-reply/reply/directive-handling.persist.ts:55-56`
  - body: `resolveModelRuntimeOverride` now accepts `openclaw` but no longer accepts `/model ... --runtime pi`, so existing chat commands or scripts using the shipped legacy token lose the runtime override. Map `pi` to `openclaw` here instead of treating it as invalid.
  - confidence: 0.82
- **[P2] Normalize legacy replay runtime pairs:** `extensions/qa-lab/src/jsonl-replay.ts:169-171`
  - body: Saved JSONL replay inputs that still contain `runtimePair: ["pi", "codex"]` now fail with `unsupported jsonl replay runtime pair`. This is a migration path, so normalize `pi` to `openclaw` before enforcing the supported pair.
  - confidence: 0.82
- **[P2] Use graceful process-tree termination:** `packages/agent-core/src/harness/env/nodejs.ts:226-249`
  - body: This helper kills the process group with immediate `SIGKILL`, bypassing the shared behavior that sends `SIGTERM`, waits a bounded grace period, and only then escalates.   - confidence: 0.87
