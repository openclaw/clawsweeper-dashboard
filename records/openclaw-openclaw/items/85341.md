Full review comments:

- **[P1] Remove the nested trusted-install match:** `src/plugins/manifest-registry.ts:780`
  - body: `isPathInside(candidatePath, trackedPath)` treats a config/global plugin candidate inside a recorded official install path as the recorded install itself. That can mark an unverified subtree as `trustedOfficialInstall`; keep the match exact or require the recorded install to be inside the candidate root, not the reverse.
  - confidence: 0.9
- **[P1] Restore manifest-backed env auth lookup:** `src/llm/env-api-keys.ts:121-164`
  - body: `getApiKeyEnvVars` is now a fixed map, so provider ids or alternate env vars declared in plugin manifests are invisible to `findEnvKeys` and `getEnvApiKey`. Env-only setups such as GitHub Copilot `GH_TOKEN`, Hugging Face `HUGGINGFACE_HUB_TOKEN`, Qwen-family keys, and Arcee now report unauthenticated even though the manifests still advertise those credentials.
  - confidence: 0.92
- **[P1] Keep the legacy provider-usage SDK export:** `src/plugin-sdk/provider-usage.ts:16-21`
  - body: The public `openclaw/plugin-sdk/provider-usage` subpath no longer exports `resolveLegacyPiAgentAccessToken`, which current main still exposes. Third-party plugins importing the shipped symbol will fail to build or load immediately after upgrade; keep a deprecated alias until a documented removal window.
  - confidence: 0.88
- **[P1] Preserve bundledDiscovery config compatibility:** `src/config/zod-schema.ts:1162-1182`
  - body: The strict `plugins` schema no longer accepts `plugins.bundledDiscovery`, and the provider legacy migration list no longer repairs that key. Existing configs written by earlier doctor flows can fail validation on upgrade until users manually edit config, so restore schema/migration compatibility for the retired key.
  - confidence: 0.86
- **[P1] Restore cross-provider Mistral compat contribution:** `extensions/mistral/index.ts:46-52`
  - body: The native Mistral provider still normalizes its own models, but the deleted provider compat contributor handled Mistral-shaped models routed through OpenRouter or custom OpenAI-compatible endpoints.   - confidence: 0.82
- **[P2] Keep PI_CACHE_RETENTION during migration:** `src/agents/anthropic-payload-policy.ts:59-60`
  - body: `resolveAnthropicEphemeralCacheControl` now reads only `OPENCLAW_CACHE_RETENTION`, so upgraded Anthropic-family deployments that still set `PI_CACHE_RETENTION=long` silently fall back to short cache retention.   - confidence: 0.8
- **[P2] Keep the legacy agent-dir env fallback in the SDK helper:** `src/plugin-sdk/agent-dir-compat.ts:8-10`
  - body: `resolveOpenClawAgentDir` is explicitly kept for third-party SDK compatibility, but it now ignores `PI_CODING_AGENT_DIR`. Plugins or upgraded environments still using only the legacy override can resolve the default agent directory and read the wrong auth/session/model state.
