Full review comments:

- **[P1] Honor OpenClaw runtime pins before CLI inference:** `src/auto-reply/reply/agent-runner-execution.ts:1678-1689`
  - body: When `resolveSessionRuntimeOverrideForProvider` returns `"openclaw"`, this block still calls `resolveCliRuntimeExecutionProvider`, so a session explicitly pinned to the built-in runtime can be rerouted to a CLI provider through auth/profile inference. Add the same built-in-runtime short-circuit that current main has for legacy `pi` before CLI resolution.
  - confidence: 0.9
- **[P1] Honor OpenClaw pins in queued followups too:** `src/auto-reply/reply/followup-runner.ts:579-590`
  - body: Queued followups repeat the same fallthrough: `sessionRuntimeOverride === "openclaw"` is not treated as a terminal built-in runtime choice before CLI inference. This can make queued turns ignore the user's persisted runtime pin and run on the wrong backend.
  - confidence: 0.88
- **[P2] Preserve runtime id casing for plugin harnesses:** `src/agents/agent-runtime-id.ts:30-31`
  - body: `normalizeOptionalAgentRuntimeId` lowercases every runtime id, but harness ids are registered and looked up with their original trimmed casing. Mixed-case custom harness ids that worked before will now be looked up under a different id and fail as missing.
  - confidence: 0.86
- **[P2] Keep legacy pi assistant metadata embedded:** `src/agents/embedded-agent-runner/run/helpers.ts:104-106`
  - body: Pre-rename transcripts can still contain assistant metadata with `provider: "pi"`. After this change `resolveReportedModelRef` treats that as a real provider instead of the embedded runtime marker, so error/failover context can be attributed to the wrong provider.
  - confidence: 0.84
- **[P1] Use the shared graceful process-tree terminator:** `packages/agent-core/src/harness/env/nodejs.ts:226-249`
  - body: The new agent-core helper hard-kills process trees with `/F` or `SIGKILL` on timeout/abort. OpenClaw already has a shared helper that sends `SIGTERM`, waits a grace window, and only then escalates; bypassing it reintroduces the cleanup and ownership risks that helper was added to avoid.
  - confidence: 0.87
