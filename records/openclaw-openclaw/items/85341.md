  - confidence: 0.91

  - body: The same CLI-inference fallback is present in the followup runner, so queued follow-up turns can ignore an explicit OpenClaw/legacy-pi session runtime pin and run on a CLI harness. Add the same non-CLI embedded short-circuit here before resolving a CLI execution provider.

  - body: Private QA environments that still export `OPENCLAW_QA_FORCE_RUNTIME=pi` now miss the force-runtime override and fall back to normal policy selection. Since the PR keeps pi-to-openclaw compatibility elsewhere, normalize `pi` to `openclaw` here instead of dropping it.
