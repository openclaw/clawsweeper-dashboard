- **[P1] Normalize legacy pi runtime IDs:** `src/agents/agent-runtime-id.ts:20`
  - body: `normalizeEmbeddedAgentRuntime` returns unknown values unchanged, so stored `agentHarnessId: "pi"` or runtime overrides from existing sessions become a plugin runtime id instead of the built-in OpenClaw runtime. Map `pi` to `openclaw` during the compatibility window.
  - confidence: 0.91
- **[P1] Restore manifest-backed env key lookup:** `src/llm/env-api-keys.ts:162-163`
  - body: The resolver now returns `undefined` for provider ids missing from its hardcoded map. Manifest-declared providers such as `arcee` still advertise env auth, and auth status depends on this path, so env-only setups can be reported unauthenticated and miss fallback keys.
  - confidence: 0.93
- **[P1] Preserve legacy providerDiscoveryEntry manifests:** `src/plugins/manifest-registry.ts:469-475`
  - body: The manifest record builder now only considers `providerCatalogEntry`. External plugins using the previously supported `providerDiscoveryEntry` key lose their discovery source after upgrade, dropping provider catalog/auth discovery until manifests are manually renamed.
  - confidence: 0.9
- **[P1] Keep the provider-usage compatibility export:** `src/plugin-sdk/provider-usage.ts:17-20`
  - body: The public SDK subpath no longer re-exports `resolveLegacyPiAgentAccessToken`, even though current main exports it and provider plugins import this symbol. Removing the named export without a deprecated alias breaks existing plugins at build/runtime on upgrade.
  - confidence: 0.92
- **[P1] Map pi middleware runtimes during normalization:** `src/plugins/agent-tool-result-middleware.ts:17-19`
  - body: Tool-result middleware normalization now only remaps `codex-app-server`. Manifests or plugin code that still pass `runtimes: ["pi"]` or legacy harnesses normalize to no built-in runtime, so existing middleware can silently stop running on OpenClaw turns.
  - confidence: 0.9
- **[P1] Do not trust nested plugin paths as official installs:** `src/plugins/manifest-registry.ts:780-786`
  - body: This matcher treats a candidate path inside the recorded official install root as a trusted official install. A config-scoped plugin path can point at a nested subtree and inherit official-only trust even though it is not the installed package root.
  - confidence: 0.86
- **[P2] Keep PI_CODING_AGENT_DIR in the SDK helper:** `src/plugin-sdk/agent-dir-compat.ts:9-10`
  - body: This helper is explicitly kept for third-party SDK compatibility but now ignores the legacy `PI_CODING_AGENT_DIR` override. Plugins using it in upgraded environments with only the legacy variable resolve the default agent directory and can read or write the wrong auth/session state.
  - confidence: 0.84
- **[P2] Require Harmony tool-call markers for analysis/final prefixes:** `src/plugin-sdk/provider-stream-shared.ts:141-148`
  - body: The new branch treats untagged text starting with `analysis` or `final` as a possible tool call without requiring the `to=` marker. Providers using this wrapper can buffer normal prose until `done`, regressing incremental text streaming.
  - confidence: 0.82
