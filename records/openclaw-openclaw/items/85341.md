  - body: The new `envMap` hardcodes provider IDs and returns `undefined` for manifest-declared providers or secondary env vars such as Qwen, Moonshot, MiniMax token-plan keys, and OpenCode Zen.   - confidence: 0.96

  - body: `normalizeEmbeddedAgentRuntime()` now returns unknown values unchanged, so persisted `agentRuntime.id`/session overrides using the shipped `pi` built-in runtime token are treated as custom runtimes instead of OpenClaw.   - confidence: 0.94
- **[P1] Keep `providerDiscoveryEntry` as a manifest fallback:** `src/plugins/manifest-registry.ts:469-475`
  - body: The PR drops the current-main fallback from deprecated `providerDiscoveryEntry` to `providerCatalogEntry`. External plugins still using the shipped legacy key will lose `providerDiscoverySource` and provider discovery/auth catalog loading until their manifests are manually rewritten.
  - confidence: 0.95

  - body: `openclaw/plugin-sdk/provider-usage` no longer exports the shipped `resolveLegacyPiAgentAccessToken` helper.   - confidence: 0.93
- **[P1] Reject nested paths for official install trust:** `src/plugins/manifest-registry.ts:785`
  - body: Adding `isPathInside(candidatePath, trackedPath)` makes a candidate under a recorded official plugin root count as the trusted official install. A config-scoped nested subtree should not inherit official-install trust unless it matches the recorded package root boundary.

  - confidence: 0.86
