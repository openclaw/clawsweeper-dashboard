- **[P1] Restore manifest-backed env auth lookup:** `src/llm/env-api-keys.ts:131-164`
  - body: The new hardcoded env map misses manifest-declared provider credentials such as Qwen, Hugging Face, OpenCode, MiniMax, and Z.ai aliases. - **[P1] Preserve providerDiscoveryEntry manifests:** `src/plugins/manifest-registry.ts:469-475`
  - body: This selection now ignores `providerDiscoveryEntry`, so external plugins that still use the previously supported manifest key lose their lightweight provider discovery/catalog module. Keep the old key as a deprecated alias or migrate it through doctor before removing it.
  - confidence: 0.9
- **[P1] Map legacy pi middleware runtimes to openclaw:** `src/plugins/agent-tool-result-middleware.ts:17-19`
  - body: Existing plugin manifests can still declare `runtimes: ["pi"]` or deprecated `harnesses: ["pi"]`; with only `codex-app-server` in the legacy map, those entries normalize to no middleware and stop running on the built-in runtime. Preserve `pi -> openclaw` in both runtime normalization and the deprecated type surface during the migration window.
  - confidence: 0.92
- **[P1] Keep the deprecated provider usage export:** `src/plugin-sdk/provider-usage.ts:16`
  - body: Removing `resolveLegacyPiAgentAccessToken` from `openclaw/plugin-sdk/provider-usage` breaks existing provider plugins that import the shipped symbol. Re-export it as a deprecated alias to the renamed helper until a documented removal window passes.
  - confidence: 0.91
- **[P1] Tighten official install path matching:** `src/plugins/manifest-registry.ts:785`
  - body: Adding `isPathInside(candidatePath, trackedPath)` makes any candidate nested under an installed official plugin path count as a trusted official install. The trust check should require the recorded package root, or the previous one-way containment semantics, so unverified subtrees do not inherit official-only behavior.
  - confidence: 0.88
- **[P2] Require the Harmony tool-call marker before buffering prose:** `src/plugin-sdk/provider-stream-shared.ts:143-147`
  - body: This branch treats untagged text beginning with `analysis`, `final`, or `commentary` as a possible Harmony tool call even without a `to=` marker. Providers using `wrapPlainTextToolCallStream` can now buffer normal prose like `final answer...` until `done`, losing incremental assistant delivery; only keep buffering once the tool-call marker is still possible.
  - confidence: 0.84
- **[P2] Keep PI_CODING_AGENT_DIR in the compatibility helper:** `src/plugin-sdk/agent-dir-compat.ts:9-10`
  - body: `resolveOpenClawAgentDir` is explicitly kept for third-party SDK compatibility, but it now ignores the legacy `PI_CODING_AGENT_DIR` override. Existing plugin/runtime callers using only that shipped env var resolve the default agent directory and can read the wrong auth/session/model state.
  - confidence: 0.86
