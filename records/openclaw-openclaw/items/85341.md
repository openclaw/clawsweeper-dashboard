  - confidence: 0.9

  - body: `normalizeProviderId` now only lowercases, dropping shipped aliases such as `bedrock`/`aws-bedrock` to `amazon-bedrock`, `z.ai` to `zai`, and `moonshotai` to `moonshot`. Existing configs and provider-owned lookups that depend on canonical IDs can stop matching.
  - confidence: 0.9
- **[P1] Honor provider-declared env API key names:** `src/llm/env-api-keys.ts:131-155`
  - body: The new env resolver maps only a subset or wrong keys: Google omits `GOOGLE_API_KEY`, Cloudflare AI Gateway uses `CLOUDFLARE_API_KEY` instead of `CLOUDFLARE_AI_GATEWAY_API_KEY`, Moonshot lacks canonical `moonshot`, and Kimi misses `kimi`/`KIMICODE_API_KEY`.   - confidence: 0.9
- **[P1] Tighten trusted official install path matching:** `src/plugins/manifest-registry.ts:783-785`
  - body: The added reverse `isPathInside(candidatePath, trackedPath)` branch marks a candidate nested under the recorded official install as trusted. Official-only behavior should not apply to an unverified subtree unless that path is the recorded package root.
  - confidence: 0.86
- **[P2] Trim env keys before marking providers configured:** `src/llm/env-api-keys.ts:179-191`
  - body: `findEnvKeys` treats any truthy raw env value as configured and `getEnvApiKey` returns it verbatim. Whitespace or newline-padded secrets can now show as configured while downstream requests receive an invalid token; trim and reject empty-after-trim values before reporting auth.
  - confidence: 0.82
