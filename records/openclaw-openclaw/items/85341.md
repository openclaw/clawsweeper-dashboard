Overall confidence: 0.88

Full review comments:

- **[P1] Derive env auth keys from plugin contracts:** `src/llm/env-api-keys.ts:135-155`
  - body: The new hard-coded map drops supported keys that provider manifests still declare, such as `GOOGLE_API_KEY`, `OPENCODE_ZEN_API_KEY`, and `HUGGINGFACE_HUB_TOKEN`. Since OpenAI-compatible requests call `getEnvApiKey(model.provider)`, documented env-only installs for those providers will be treated as unauthenticated or fall through to the wrong credential path; this should reuse the manifest/provider env-var resolver or include every declared key.
  - confidence: 0.9
- **[P1] Restore provider compat contributions for proxy transports:** `extensions/mistral/index.ts:46-51`
  - body: Removing the `contributeResolvedModelCompat` path leaves Mistral request-shape fixes applied only when the selected provider is `mistral`. Mistral models routed through OpenRouter or custom OpenAI-compatible endpoints can now use OpenAI defaults like `max_completion_tokens`/store support, producing invalid requests that the removed hook and test were covering.
  - confidence: 0.84
- **[P1] Keep the public legacy provider-usage export linked:** `src/plugin-sdk/provider-usage.ts:16`
  - body: `openclaw/plugin-sdk/provider-usage` no longer exports `resolveLegacyPiAgentAccessToken`, but current main ships that public SDK symbol and external provider plugins can still import it. The plugin SDK guide treats public API compatibility as the exception, so keep a deprecated alias or an explicit compatibility stub until a documented removal window passes.
  - confidence: 0.87
