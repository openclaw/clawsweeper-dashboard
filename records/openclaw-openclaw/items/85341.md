  - body: Removing `contributeResolvedModelCompat` from the public `ProviderPlugin` surface breaks external provider plugins that contribute compat flags for vendor models behind another OpenAI-compatible transport.   - body: The loader now ignores manifests that still use the deprecated `providerDiscoveryEntry` key, so external provider plugins can lose their provider discovery source after upgrade. Preserve the fallback while warning through the deprecation path.
  - confidence: 0.9
- **[P1] Map legacy pi middleware runtimes to openclaw:** `src/plugins/agent-tool-result-middleware.ts:17-19`
  - body: `normalizeAgentToolResultMiddlewareRuntime` no longer maps `pi` to the built-in runtime, so existing plugins declaring `runtimes: ["pi"]` or legacy harnesses silently stop applying tool-result middleware on OpenClaw runs. Keep `pi` as a deprecated alias of `openclaw`.
  - confidence: 0.88
- **[P1] Restore provider alias normalization:** `src/agents/provider-id.ts:3-4`
  - body: `normalizeProviderId` now lowercases only, dropping shipped aliases such as `z.ai`, `moonshotai`, and `aws-bedrock`. That can split stored config/auth/profile IDs from canonical provider IDs and make status, OAuth, usage, or model lookup treat valid setups as different providers.
  - confidence: 0.87
- **[P1] Derive env API keys from plugin metadata:** `src/llm/env-api-keys.ts:131-155`
  - body: The new hard-coded env map omits canonical provider IDs and documented env vars from provider manifests, including Google, Kimi, and Cloudflare AI Gateway variants. Build this from plugin auth/setup metadata or add complete manifest-backed coverage so env-only users are not treated as unauthenticated.
  - confidence: 0.86
