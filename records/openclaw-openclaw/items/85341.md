## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Use plugin-declared env credential aliases:** `src/llm/env-api-keys.ts:148-180`
  - body: `getApiKeyEnvVars` is now a hard-coded single-env map for many providers, but `AuthStorage` uses `findEnvKeys(provider)` for auth status. Bundled provider manifests still declare aliases such as `GOOGLE_API_KEY`, `CLOUDFLARE_AI_GATEWAY_API_KEY`, `GH_TOKEN`/`GITHUB_TOKEN`, `KIMICODE_API_KEY`, and `OPENCODE_ZEN_API_KEY`, so env-only installs following those plugin contracts can be reported unauthenticated. Build this from provider metadata or preserve every declared alias.
  - confidence: 0.9
- **[P2] Preserve Bedrock discovery region fallback:** `extensions/amazon-bedrock/register.sync.runtime.ts:465`
  - body: `resolveBedrockRegion` now returns `undefined` unless a provider `baseUrl` carries a region, while current main falls back to the configured Bedrock discovery region after those checks. Since this PR migrates `models.bedrockDiscovery` into plugin-owned discovery config, installs that configured only that region can fall through to `stream.runtime.ts`'s default `us-east-1` path and send Bedrock requests to the wrong region. Keep the migrated discovery-region fallback or pass it into stream options explicitly.
  - confidence: 0.86
