  - body: `normalizeProviderId` now only lowercases input, dropping shipped aliases such as `z.ai`/`z-ai` to `zai`, `moonshotai` to `moonshot`, and `aws-bedrock` to `amazon-bedrock`.   - confidence: 0.93
- **[P1] Derive env auth keys from provider metadata:** `src/llm/env-api-keys.ts:131-160`
  - body: The new LLM env-key resolver hardcodes a partial provider map instead of using manifest-declared setup env vars. Providers such as `nvidia` and `microsoft-foundry` declare `NVIDIA_API_KEY`/`AZURE_OPENAI_API_KEY`, but `getEnvApiKey(model.provider)` will return no key for them and can make env-only runtime calls fail as unauthenticated.
  - confidence: 0.9
- **[P1] Keep the legacy providerDiscoveryEntry fallback:** `src/plugins/manifest-registry.ts:464-470`
  - body: `buildRecord` now ignores `providerDiscoveryEntry` when `providerCatalogEntry` is absent. Current main and the manifest docs keep that field as a deprecated spelling, so already-shipped third-party plugins using it lose provider discovery after upgrade.
  - confidence: 0.92
- **[P1] Restore the provider-usage SDK alias:** `src/plugin-sdk/provider-usage.ts:16-20`
  - body: `openclaw/plugin-sdk/provider-usage` no longer exports `resolveLegacyPiAgentAccessToken`, but current main exposes it as a public SDK helper and bundled Z.ai still imports it. Existing plugins importing the shipped symbol will fail to compile or load immediately after upgrade.
  - confidence: 0.9
- **[P1] Preserve resolved-model compat hooks:** `src/plugins/types.ts:1319-1321`
  - body: Removing `contributeResolvedModelCompat` from `ProviderPlugin` is a public SDK break: TypeScript plugins using the hook fail excess-property checks and JavaScript plugins silently stop contributing proxy-routed model compat. Keep a deprecated hook or a tested equivalent migration path before removing it.
  - confidence: 0.86
- **[P2] Keep legacy agent-dir env fallback:** `src/plugin-sdk/agent-dir-compat.ts:8-10`
  - body: `resolveOpenClawAgentDir` is documented as a third-party compatibility helper, but the PR head only honors `OPENCLAW_AGENT_DIR`. Plugins or hosts that still export `PI_CODING_AGENT_DIR` resolve the default agent dir instead of the intended legacy state directory.
