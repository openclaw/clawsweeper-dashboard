Full review comments:

- **[P1] Honor explicit OpenClaw runtime pins before CLI inference:** `src/auto-reply/reply/followup-runner.ts:579-590`
  - body: This removes the current-main built-in-runtime short-circuit. After legacy `pi` values normalize to `openclaw`, a pinned session falls through to `resolveCliRuntimeExecutionProvider`, so queued follow-up turns can execute on a CLI runtime despite the explicit session runtime override.
  - confidence: 0.88
- **[P2] Preserve case for custom harness runtime ids:** `src/agents/agent-runtime-id.ts:30-31`
  - body: `normalizeOptionalAgentRuntimeId` lowercases every runtime id before selection, but the harness registry stores and looks up ids with only `trim()`. A plugin harness registered as `MyHarness` can no longer be selected via `agentRuntime.id` because the lookup becomes `myharness`; only known aliases should be case-normalized.
  - confidence: 0.87
- **[P2] Treat legacy pi assistant providers as embedded metadata:** `src/agents/embedded-agent-runner/run/helpers.ts:104-105`
  - body: Persisted histories can still contain assistant messages tagged with `provider: "pi"`. This check now only recognizes `openclaw`, so `resolveReportedModelRef` returns `pi` as a real provider instead of the active model provider, which can misclassify embedded error/failover context.
  - confidence: 0.85
- **[P2] Reuse the shared graceful process-tree killer:** `src/agents/utils/shell.ts:200-215`
  - body: This new helper hard-kills with `taskkill /F` or SIGKILL immediately, bypassing `src/process/kill-tree.ts`'s SIGTERM and grace-period contract. Timed-out or aborted shell/LSP processes lose their cleanup window and duplicate a security-sensitive lifecycle helper.
  - confidence: 0.9
