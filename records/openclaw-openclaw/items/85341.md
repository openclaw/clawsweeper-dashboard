Full review comments:

- **[P1] Remove the parent-directory trusted install match:** `src/plugins/manifest-registry.ts:780`
  - body: The added inverse containment check lets a candidate path that is an ancestor of a recorded official install path be treated as `trustedOfficialInstall`. That weakens the official-plugin trust boundary because a config-scoped parent directory can inherit trust that belongs only to the recorded package root.
  - confidence: 0.86
- **[P1] Restore manifest-backed env auth lookup:** `src/llm/env-api-keys.ts:121-164`
  - body: `getApiKeyEnvVars` is now a fixed map, so manifest-declared provider env vars that are not listed here, such as `ARCEEAI_API_KEY`, `HUGGINGFACE_HUB_TOKEN`, `GH_TOKEN`, and `GITHUB_TOKEN`, are invisible to `findEnvKeys` and `getEnvApiKey`. Env-only provider installs can be reported unauthenticated or lose fallback credentials.
  - confidence: 0.9
- **[P1] Keep the legacy provider-usage SDK export:** `src/plugin-sdk/provider-usage.ts:16-21`
  - body: This public SDK subpath no longer exports `resolveLegacyPiAgentAccessToken`, but existing provider plugins can still import that shipped symbol. Removing it without a deprecated alias breaks plugin builds and runtime named imports during upgrade.
  - confidence: 0.88
- **[P1] Preserve bundledDiscovery config compatibility:** `src/config/zod-schema.ts:1162-1182`
  - body: The strict `plugins` schema no longer accepts `plugins.bundledDiscovery`, and the matching doctor migration is gone. Existing configs written by earlier migration flows can fail validation or require manual edits instead of being repaired by `openclaw doctor --fix`.
  - confidence: 0.84
- **[P1] Keep legacy agent-dir env fallbacks:** `src/plugin-sdk/agent-dir-compat.ts:8-10`
  - body: `resolveOpenClawAgentDir` is explicitly a deprecated compatibility helper, but it now ignores `PI_CODING_AGENT_DIR`.
