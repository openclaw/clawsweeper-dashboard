Full review comments:

- **[P1] Derive API-key env lookup from provider metadata:** `src/llm/env-api-keys.ts:131-160`
  - body: The new hard-coded `envMap` misses provider contracts declared in this same PR, such as Google `GOOGLE_API_KEY`, Hugging Face `HUGGINGFACE_HUB_TOKEN`, and Qwen `QWEN_API_KEY`/`MODELSTUDIO_API_KEY`/`DASHSCOPE_API_KEY`. Because request paths call `getEnvApiKey(model.provider)`, env-only users can be treated as unauthenticated or fail calls despite valid plugin-declared keys; keep plugin metadata or generated metadata as the source of truth.
  - confidence: 0.93
- **[P1] Normalize legacy pi runtime ids during migration:** `src/agents/agent-runtime-id.ts:20`
  - body: `normalizeEmbeddedAgentRuntime` returns legacy `"pi"` unchanged, so persisted configs and sessions that still carry `agentRuntime.id: "pi"` can be classified as custom plugin runtimes instead of the built-in OpenClaw runtime. Map `pi` to `openclaw` at this central boundary while doctor/migration removes old values.
  - confidence: 0.9
- **[P1] Restore Groq resolved-model compat registration:** `extensions/groq/index.ts:15`
  - body: The PR leaves Groq's reasoning compat helper detached from provider registration, so Groq models that need reasoning-effort normalization can receive unsupported OpenAI-compatible request parameters. Rewire the compat contributor through the new provider contract or add an equivalent replacement.
  - confidence: 0.86
- **[P1] Restore the legacy provider-usage SDK alias:** `src/plugin-sdk/provider-usage.ts:16`
  - body: `openclaw/plugin-sdk/provider-usage` is a public Plugin SDK subpath, and this change removes `resolveLegacyPiAgentAccessToken` without a deprecated re-export. Existing plugins importing the shipped helper will fail to compile or load after upgrade; keep the old symbol as an alias until a documented removal window.
  - confidence: 0.9
- **[P1] Restore X.ai compat for routed Grok models:** `extensions/xai/index.ts:210`
  - body: The X.ai provider now only applies compat through `normalizeResolvedModel` on direct `xai` models, but current main also contributed X.ai compat for downstream-routed models such as `openrouter/x-ai/...`. Keep a cross-provider compat contribution path so Grok-family proxy routes still get X.ai tool schema and tool-call argument handling.
  - confidence: 0.84
- **[P2] Keep the legacy cache-retention env fallback:** `src/llm/providers/openai-completions.ts:111-114`
  - body: `resolveCacheRetention` now reads only `OPENCLAW_CACHE_RETENTION`, so installs still using the previously supported `PI_CACHE_RETENTION=long` fallback silently switch to short prompt-cache retention on OpenAI-compatible completions. Preserve the deprecated env alias through the migration window.
  - confidence: 0.78
