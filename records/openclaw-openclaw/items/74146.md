The PR changes Feishu streaming-card block delivery from delta append to snapshot merging, adds dispatcher regression coverage for overlapping and independent block payloads, and updates `CHANGELOG.md`.

## Best Possible Solution

Land this narrow Feishu dispatcher/test/changelog fix after maintainer review, while leaving broader final/stale delivery variants on https://github.com/openclaw/openclaw/issues/77685.

## Solution Assessment

Yes. Switching only the Feishu block fallback update to snapshot mode is the narrow maintainable fix because it reuses `mergeStreamingText`, preserves the existing block-only fallback path, and adds coverage for both overlap and non-overlap behavior.
