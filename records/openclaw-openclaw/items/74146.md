## Summary

Keep open. Current main still routes Feishu streaming-card block fallback payloads through delta concatenation and has a test that expects duplicated overlap, while this PR directly changes that path, adds focused regression coverage, and includes the required changelog entry.

## What This Changes

The PR changes Feishu streaming-card block fallback updates from delta concatenation to snapshot merging, adds Feishu regression tests, and documents the user-facing fix in CHANGELOG.

## Best Possible Solution

Land the narrow Feishu dispatcher, regression-test, and changelog update after normal exact-head checks and maintainer review so the linked bug report closes through the PR flow.

## Reproduction Assessment

Yes. Current main has a source-level reproduction: partial snapshot `hello` followed by block payload `lo world` closes as `hellolo world` because the block fallback still uses delta concatenation.

## Solution Assessment

Yes. Switching only the block fallback update to snapshot mode is the narrow maintainable fix because it reuses the existing `mergeStreamingText` behavior and the PR covers overlap, non-overlap, and cumulative block cases.
