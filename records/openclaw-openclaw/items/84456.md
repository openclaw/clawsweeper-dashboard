## Summary

Current main fixes the reported manual channel `/approve` resolver bug: shared text approvals now use the approval-runtime resolver instead of a plain gateway client, with authorization and gateway-scope checks still in front of the trusted resolver path.

## What This Changes

The issue requests that WhatsApp text `/approve <id> allow-once` resolve pending exec approvals the same way Telegram approval actions resolve them.

## Best Possible Solution

Keep the merged resolver-based implementation and track any remaining WhatsApp prompt rendering or duplicate-delivery behavior separately if it reproduces on a build containing the fix.

## Reproduction Assessment

No current-main failing reproduction remains from source inspection: the old parent used a plain gateway call, while current main routes `/approve` through the approval-runtime resolver. I did not run a live WhatsApp plus Telegram replay in this read-only review.

## Solution Assessment

Yes. Reusing `resolveApprovalOverGateway` after the existing sender authorization and gateway-scope checks is the narrow maintainable fix; it avoids broadening generic gateway visibility rules.
