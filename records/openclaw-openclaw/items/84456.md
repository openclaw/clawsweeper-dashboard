## Summary

Current main implements the requested fix: shared manual chat /approve now resolves through the approval-runtime gateway resolver after the existing authorization checks, matching the trusted client path used by native approval actions.

## What This Changes

The issue reports that WhatsApp text /approve cannot resolve a pending exec approval that Telegram can resolve with the same approval ID.

## Best Possible Solution

Keep the shared resolver-based implementation on main, ship it in the next release, and track any remaining WhatsApp prompt duplication or rendering defects separately if they still reproduce.

## Reproduction Assessment

No current-main failure was reproduced; current main has the resolver fix in the shared /approve path. The old source path and reporter logs explain the failure, and the merged PR includes live Gateway after-fix output for exec/plugin approval resolution.

## Solution Assessment

Yes. Reusing resolveApprovalOverGateway after the existing sender authorization and gateway-scope checks is the narrow maintainable fix, because it aligns generic text approvals with the established approval-runtime visibility contract instead of widening approval lookup rules.
