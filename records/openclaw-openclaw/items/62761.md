## Summary

Keep open. Current main has addressed the later normal-DM `is_from_me` echo-loop subcase, but the legacy iMessage monitor still does not persist a rowid cursor or pass a resume point into `watch.subscribe`, and related replay reports have already been consolidated here.

## What This Changes

Persist the legacy iMessage rowid cursor and resume `imsg watch` after restarts so old messages are not replayed as new inbound traffic.

## Best Possible Solution

Add an iMessage-owned per-account high-water cursor with an explicit startup fence, pass the dependency-supported `since_rowid` value to `watch.subscribe`, and keep the existing from-me/self-chat guards with restart replay regression coverage.

## Reproduction Assessment

Yes. The source-level reproduction is to inspect or mock `monitorIMessageProvider` startup and observe that the `watch.subscribe` request sends only `attachments` despite the reporter and related issues showing restart replay from missing rowid persistence.

## Solution Assessment

Yes. A per-account persisted rowid plus startup fence is the narrowest maintainable fix for restart replay; the already-landed `is_from_me` filtering is necessary but only covers the outbound echo subcase.
