## Summary

Keep open. Current main fixed the normal `is_from_me` echo-loop subcase and shipped opt-in iMessage catchup, but the default live `watch.subscribe` path still starts without a `since_rowid`/`start` cursor and catchup remains disabled by default.

## What This Changes

Persist and use an iMessage rowid cursor or startup fence so restarts do not replay old messages, while preserving outbound `is_from_me` filtering.

## Best Possible Solution

Add a maintainer-approved default stale-replay guard in the iMessage plugin, likely by persisting a per-account high-water rowid or applying a startup fence to the live watch path while keeping broad catchup replay explicitly bounded.

## Reproduction Assessment

Yes, at source level but not with a live macOS run. Current main still subscribes live with no cursor/fence and catchup is disabled by default, so a mocked restart or iCloud-sync notification path can exercise the remaining replay gap.

## Solution Assessment

Unclear until maintainer review. The shipped opt-in catchup is a good partial fix, but the narrow remaining solution should prevent stale default live-watch replay without unexpectedly enabling broad historical replay.
