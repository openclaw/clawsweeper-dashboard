## Summary

Keep open. Current main fixes the normal `is_from_me` echo-loop subcase and adds opt-in iMessage catchup, but it still does not pass a persisted cursor or startup fence into the default live `watch.subscribe` path; the remaining work is actively represented by https://github.com/openclaw/openclaw/pull/79085 and is not safe to close yet.

## What This Changes

Persist an iMessage rowid/high-water cursor and resume or fence the watcher after gateway restarts so old chat history is not processed as new inbound messages.

## Best Possible Solution

Review and repair https://github.com/openclaw/openclaw/pull/79085, or land an equivalent iMessage-owned cursor/resume implementation that preserves safe first-run behavior and has live macOS/imsg restart proof.

## Reproduction Assessment

Yes at source level, but not with a live macOS run. Current main still subscribes to iMessage watch without a persisted resume cursor or startup fence by default, while upstream imsg exposes the needed exclusive `since_rowid` contract.

## Solution Assessment

Unclear as a final product choice. The open PR is a plausible implementation path, but the default replay window, cursor-store concurrency, and required live proof need maintainer review before this should close.
