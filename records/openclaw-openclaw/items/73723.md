## What This Changes

The issue requests a fix or clearer health signal for Codex native hook relay registrations that remain unavailable after gateway restart while ordinary internal hooks report ready.

## Best Possible Solution

Review, land, or replace PR #73950 with a narrow fix that makes Codex native relay registration shared/recoverable across duplicated runtime module instances and stale session resumes, while keeping hook readiness distinct from active relay health. A user-facing opt-out should only be added with explicit security semantics and documentation.
