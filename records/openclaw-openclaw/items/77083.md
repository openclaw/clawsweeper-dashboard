## Summary

Keep open: current main has the right packaging-side fixes, but the npm registry still exposes `@openclaw/brave-plugin@2026.5.2` as `latest` with no compiled runtime output or `runtimeExtensions`, and no fixed `2026.5.3`, `2026.5.3-beta.3`, or `2026.5.3-beta.4` brave plugin package is published yet.

## What This Changes

The issue asks for published `@openclaw/brave-plugin` tarballs to include validator-compatible runtime artifacts and metadata instead of source-only or mismatched package contents.

## Best Possible Solution

Publish a new verified `@openclaw/brave-plugin` package with package-local `dist/` output and `runtimeExtensions`, and keep #77085's prerelease resolver fix so beta hosts do not reinstall the broken stable artifact.

## Reproduction Assessment

Yes. The stable npm tarball plus current validator source gives a high-confidence reproduction: `@2026.5.2` declares `./index.ts` but lacks both `dist/index.js` and `runtimeExtensions`, which current main rejects for installed package validation.

## Solution Assessment

No, this should not close on source changes alone. The current implementation direction is narrow and maintainable, but the user-facing fix needs a maintainer-controlled npm plugin publish or release follow-through because the broken stable tarball is immutable.
