## Summary

Current main already has the packaging-side fix: publishable plugins now build package-local `dist/` output, overlay `openclaw.runtimeExtensions` during npm pack/publish, and validate explicit runtime entries without requiring the TypeScript source entry to ship.

## What This Changes

The issue reports broken published `@openclaw/brave-plugin` tarballs where stable lacks compiled output and beta lacks the declared TypeScript entry, causing installed plugin validation to fail.

## Best Possible Solution

Keep the current package-local runtime build and `runtimeExtensions` packaging path, then publish a new Brave plugin package from current main; leave beta-channel resolver selection to #77085.

## Reproduction Assessment

Yes. Source inspection shows installed packages with TypeScript entries and no compiled runtime are rejected, and the reported stable tarball shape matches that failure path; the current fix path can be verified by the runtime build, manifest overlay, and published-runtime verifier code.

## Solution Assessment

Yes. Building package-local runtime output and declaring `runtimeExtensions` is the narrow maintainable fix because it preserves strict installed-package validation without requiring npm packages to ship TypeScript source files.
