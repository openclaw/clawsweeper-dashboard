## Summary

Keep open: the docs clarification is the right direction for the linked install issue, but the patch currently overstates `install.sh` behavior and the new changelog entry is missing the repo-required human credit.

## What This Changes

This PR adds a changelog entry and updates the install docs to warn that `install-cli.sh` uses a managed Node 22.22.0 runtime by default, with `--node-version` as the Node 24 opt-in path.

## Best Possible Solution

Keep the docs-only fix, but phrase the warning as `install.sh` defaulting to Node 24 only when it needs to install Node, and credit the known human contributors in the changelog.

## Reproduction Assessment

Yes. Source inspection shows the docs mismatch on current main and the PR diff is enough to reproduce the introduced wording problem against `scripts/install.sh` behavior.

## Solution Assessment

No, not as written. The docs-warning approach is the narrow maintainable solution, but it should avoid implying `install.sh` always installs Node 24 and should satisfy changelog credit policy.
