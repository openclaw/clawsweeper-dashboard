## What This Changes

The issue asks OpenClaw to resume automatically posting generated Discord guild-channel replies after an upgrade to 2026.5.7.

## Best Possible Solution

Keep the current tool-only guild-channel default and direct affected users to set messages.groupChat.visibleReplies: "automatic" or use a model that reliably calls message(action=send).

Yes for the current configured behavior: source inspection shows Discord guild/channel turns default to message_tool_only and suppress automatic final reply delivery unless the config is set to automatic or the model calls the message tool.
