## Summary

Keep open. Current main still treats unset or `full` tool policy as unrestricted and preserves permissive embedded-run behavior when `toolsAllow` is missing or empty; the Active Memory example is now narrowed, but the broader least-privilege default request is not implemented.

## What This Changes

Make agent tool permissions least-privilege by default, with tools enabled explicitly per agent or runtime context.

## Best Possible Solution

Define default-deny semantics intentionally across agent, sub-agent, embedded-run, plugin, and MCP tool policy; keep a named unrestricted profile for compatibility; add migration or doctor warnings; and update docs/schema/tests together.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main by inspecting profile resolution, policy matching, embedded-run `toolsAllow` handling, docs, and tests that preserve broad defaults.

## Solution Assessment

No complete solution is present yet. The Active Memory allowlist is a useful targeted mitigation, but the requested least-privilege default requires a coordinated policy change rather than another caller-specific allowlist.
