## Summary

Keep open. Current main still leaves unset tool profiles and missing or empty runtime allowlists permissive; Active Memory is now narrowed with a memory-only allowlist, but the broader least-privilege default request remains unresolved policy work.

## What This Changes

Make agent tool availability least-privilege by default so agents and embedded runs only receive explicitly enabled tools.

## Best Possible Solution

Define intentional default-deny semantics across agent, sub-agent, embedded-run, plugin, and MCP tool policy, while preserving an explicit unrestricted escape hatch with migration warnings, docs, schema/help updates, and regression coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main by inspecting profile resolution, policy matching, embedded-run `toolsAllow` handling, docs, and tests that preserve broad defaults.

## Solution Assessment

No complete solution is present yet. The Active Memory allowlist is a useful targeted mitigation, but the requested least-privilege default needs a coordinated compatibility and security-policy design rather than another caller-specific allowlist.
