## Summary

Keep open. Current main still has the reported split-state behavior: the shared state resolver honors `OPENCLAW_STATE_DIR`, but exec approval file/socket defaults, effective-policy reporting, tests, and docs still point at `~/.openclaw`; the three related fix PRs that referenced this issue were closed unmerged.

## What This Changes

The issue asks for `exec-approvals.json` to be stored under the configured `OPENCLAW_STATE_DIR` instead of creating `~/.openclaw`.

## Best Possible Solution

Route the default exec approval file and default socket path through `resolveStateDir()` while preserving explicit `socket.path` values, define legacy read or migration behavior for existing home-based approval files, and update focused tests plus approvals docs so doctor, CLI, gateway, and node-host behavior agree.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: with `OPENCLAW_STATE_DIR=~/openclaw`, `resolveStateDir()` resolves the override, while the exec approvals resolvers still expand `~/.openclaw/exec-approvals.json` and `.sock`.

## Solution Assessment

No current solution is in place. The narrow maintainable fix is state-dir-backed defaults plus explicit legacy policy preservation; changing only the constants is likely incomplete for existing installs.
