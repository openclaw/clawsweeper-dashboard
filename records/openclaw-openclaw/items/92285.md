## Summary

Current main still lacks reconciliation for this full-agent mirror chain: task maintenance can mark the backing row lost, but TaskFlow maintenance does not derive a terminal state for a running task-mirrored flow from its linked tasks. The issue is distinct from the shipped Codex-native childless-mirror fix and remains necessary.

## What This Changes

Reconcile a mirrored parent subagent task and task-mirrored TaskFlow after the linked backing CLI child becomes lost or disappears.

## Best Possible Solution

Add one canonical mirrored-state reconciliation rule that terminalizes the parent task from its linked backing task and recomputes the task-mirrored flow, used both by live lifecycle updates and maintenance for persisted stale rows.

## Reproduction Assessment

Yes at source level: current main has no maintenance transition for a running task-mirrored flow whose linked work is terminal, and the reporter supplied internally consistent parent, child, flow, audit, and cancellation records. The intermittent child-loss trigger itself was not rerun.

## Solution Assessment

Yes in principle: the narrowest maintainable fix is canonical terminal propagation and flow-summary recomputation, not a new status, config option, or provider-specific workaround.
