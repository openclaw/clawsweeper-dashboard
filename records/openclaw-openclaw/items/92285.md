## What This Changes

Parent subagent task and its task_mirrored TaskFlow should reconcile to a terminal state when the backing CLI child sharing the runId becomes terminal or lost.

## Best Possible Solution

Review, repair, and either land or replace https://github.com/openclaw/openclaw/pull/92454 with a narrow task-registry maintenance fix that terminalizes the parent task and lets existing task_mirrored TaskFlow sync follow the task mutation.

## Reproduction Assessment

Yes at source level: current main keeps a childless non-Codex subagent task as backed, so maintenance cannot reconcile the reported parent from a terminal CLI sibling. I did not run the intermittent live Windows orchestration path in this read-only review.

## Solution Assessment

Unclear whether the current linked PR head is the final best patch, but closing the issue before that PR lands would be wrong. The maintainable end state is a bounded task-registry maintenance fix with focused regression coverage and no duplicate TaskFlow reconciliation path.
