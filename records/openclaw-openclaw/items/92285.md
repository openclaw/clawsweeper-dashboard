## What This Changes

The issue reports that a mirrored parent subagent task and its task-mirrored TaskFlow remain `running` after the backing child CLI task becomes terminal as `lost`.

## Best Possible Solution

Land a reviewed fix that reconciles stale childless subagent parent tasks from terminal scoped CLI children and then relies on the existing task-mirrored flow sync to terminalize the TaskFlow.

## Reproduction Assessment

Yes at source level: the reporter supplied consistent parent, backing child, TaskFlow, audit, and cancellation evidence, and current main has no reconciliation path for the childless parent subagent whose terminal backing CLI child shares the run id. I did not run a live repro because this review is read-only and the trigger is intermittent.

## Solution Assessment

Yes, the best fix direction is to propagate the backing child terminal state into the mirrored parent task and let the existing `syncFlowFromTaskAfterTaskMutation` path update the task-mirrored flow. A new status, config option, or manual-only cleanup path would be broader than needed.
