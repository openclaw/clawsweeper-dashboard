## What This Changes

The PR moves follow-up tool event forwarding ahead of the follow-up verbose progress gate and updates four follow-up runner assertions to expect `onToolStart` while verbose is off.

## Best Possible Solution

Fix the dispatch/channel-owned progress callback gate so status reactions receive tool-start lifecycle events while verbose text summaries stay suppressed, then verify the Discord reaction sequence in a real transport run.

## Reproduction Assessment

Yes at source level, but not from a live Discord run in this read-only review. With verbose off, `dispatch-from-config.ts` can still drop `onToolStart` before Discord's `statusReactions.setTool` path, matching the linked report's missing intermediate reactions.

## Solution Assessment

No. Moving the callback in `followup-runner.ts` is a plausible helper-level mitigation, but the actual Discord dispatch path still wraps the callback behind `requiresToolSummaryVisibility`, so the reported default-config path remains gated.
