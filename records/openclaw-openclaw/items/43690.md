## Summary

Keep open. Current main still lacks a Feishu card table-limit classifier or fallback, and this issue is already paired with open implementation PRs that should be reviewed or consolidated rather than replaced.

## What This Changes

The issue asks Feishu to recover from card table-count errors by falling back to plain text/table conversion or by rendering tables outside the limited card path.

## Best Possible Solution

Land a consolidated Feishu plugin fix that classifies 230099/11310 table-limit failures and retries affected reply, streaming-card, and outbound card delivery through the existing text/table-conversion path while rethrowing unrelated card errors.

## Reproduction Assessment

Yes. The issue includes concrete Feishu 230099/11310 logs and current-main source inspection shows table-heavy auto/card replies and outbound sends still enter card delivery without a table-limit fallback.

## Solution Assessment

Yes for the fallback direction, but it is not implemented on current main. The narrow maintainable fix is to classify the Feishu table-limit error and reuse sendMessageFeishu's existing markdown-table conversion; image rendering can remain a separate enhancement.
