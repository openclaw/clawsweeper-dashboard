## Summary

Keep open. Current main still sends Feishu table-heavy auto/card replies and outbound card sends through interactive-card delivery without a 230099/11310 table-limit classifier or text retry, and open PR #50429 directly closes this issue while #73651 covers the related outbound surface.

## What This Changes

The issue asks Feishu card delivery to avoid silently dropping table-heavy replies by falling back to text or otherwise avoiding Feishu card table-count limits.

## Best Possible Solution

Land a coordinated Feishu plugin fix that classifies 230099/11310 table-limit failures and retries affected reply, streaming-card fallback, and outbound card delivery through the existing text/table-conversion path; table-to-image rendering can remain a separate enhancement.

## Reproduction Assessment

Yes. The issue includes concrete Feishu 230099/11310 logs, and current main still routes table-heavy auto/card replies and outbound sends into card delivery without a table-limit fallback.

## Solution Assessment

Yes for the fallback direction, but it is not implemented on current main. The narrow maintainable solution is to classify the Feishu table-limit error and reuse the existing sendMessageFeishu markdown-table conversion path rather than making image rendering part of the bug fix.
