## Summary

Keep open: current main still has a Feishu card-send path that can propagate table-limit card failures without degrading to text, and an open closing PR already tracks the narrow fix.

## What This Changes

The issue asks Feishu replies that hit the card table-count limit to fall back to text or otherwise avoid dropping messages with many markdown tables.

## Best Possible Solution

Land or consolidate the narrow Feishu reply-dispatcher fallback from #50429, with focused regression coverage for table-limit card failures and existing text-table conversion.

## Reproduction Assessment

Yes. The provided logs match a current-main source path where card-mode Feishu replies call sendStructuredCardFeishu and failures are logged rather than retried as text; a live Feishu account would be needed only to confirm the platform threshold.

## Solution Assessment

Yes. Falling back to the existing sendMessageFeishu path with markdown table conversion is the narrowest maintainable fix; native table/image rendering is a larger enhancement and should not block delivery recovery.
