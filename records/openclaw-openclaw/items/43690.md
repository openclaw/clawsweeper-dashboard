## Summary

Keep open. Current main still routes Feishu table-heavy auto/card replies and outbound sends into card delivery without a 230099/11310 table-limit classifier or text fallback, and open implementation PRs already reference the remaining work.

## What This Changes

The issue asks Feishu to deliver markdown-table-heavy responses by falling back to text or another safe rendering when interactive cards hit the platform table limit.

## Best Possible Solution

Land a consolidated Feishu plugin fix that classifies 230099/11310 table-limit failures and retries affected reply, streaming-card, and outbound card delivery through the existing text/table-conversion path while leaving image rendering as a separate enhancement.

## Reproduction Assessment

Yes, source-reproducible with high confidence. The issue includes concrete Feishu 230099/11310 logs, and current-main code still routes table-heavy auto/card replies and outbound sends into card delivery without a table-limit fallback.

## Solution Assessment

Yes for the fallback direction, but it is not implemented on current main. The narrow maintainable fix is to classify the Feishu table-limit error and reuse the existing sendMessageFeishu markdown-table conversion path; image rendering can remain separate.
