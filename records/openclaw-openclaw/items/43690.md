Issue requests a Feishu fallback or table-safe rendering path when interactive cards exceed the platform table-component limit.

## Best Possible Solution

Land or replace the narrow Feishu static-card table-limit fallback so messages with more than five markdown tables use the existing post/text conversion path, while keeping streaming CardKit behavior tracked separately unless proven to share the same limit.

## Solution Assessment
