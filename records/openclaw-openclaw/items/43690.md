## What This Changes

Add a Feishu fallback for replies whose interactive card creation fails on markdown-table limits so users still receive the generated response.

## Best Possible Solution

Land or replace a coordinated Feishu fix that classifies the 230099/11310 card table-limit failure and retries affected reply-dispatcher card delivery through the existing plain-text table-conversion path, while keeping outbound-only work coordinated separately.

## Solution Assessment

Yes for a narrow table-limit-specific plain-text fallback in the reply dispatcher; no for making table-image rendering the required bug fix. The maintainable path is to reuse sendMessageFeishu and existing markdown-table conversion for the affected failure only.
