## Summary

Keep open: current main has provider-level request timeout support but no per-agent model timeout field, and adding that config surface needs a maintainer product decision rather than cleanup closure.

## What This Changes

Request a per-agent `model.requestTimeoutSeconds` config knob so complex isolated cron agent turns can raise per-provider LLM request timeouts without changing broad provider defaults.

## Best Possible Solution

Maintainers should decide the timeout ownership first; if per-agent tuning is approved, add it through the canonical agent model schema and resolution path with docs and regression coverage, otherwise route users to provider-level timeout plus the canonical fallback-timeout issues.

## Reproduction Assessment

Partly at source level: current main has no per-agent request timeout field and only wires provider-level timeout into model requests, while the linked Aegis issue supplies production timing evidence. I did not reproduce a live cron run in this read-only review.

## Solution Assessment

Unclear: the per-agent override is plausible, but not proven best because OpenClaw already documents `models.providers.*.timeoutSeconds` and related open issues suggest the fallback or cron timeout hierarchy may be the deeper fix.
