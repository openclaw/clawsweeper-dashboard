## Summary

Keep open: current main still lacks the requested per-agent model request-timeout config, and the remaining choice is product/config ownership rather than cleanup closure. A merged main-only cron idle-stall fix overlaps the fallback symptom, but it does not add the requested per-agent knob or prove the Aegis cron workload is fixed.

## What This Changes

Request adding per-agent `model.requestTimeoutSeconds` so specific agents with heavy cron payloads can tune model request ceilings without changing provider or global defaults.

## Best Possible Solution

Maintainers should decide timeout ownership first; if per-agent tuning is approved, add it through the canonical agent model schema/resolution path with docs and regression coverage, otherwise document provider timeout plus run/cron timeout as the supported answer.

## Reproduction Assessment
