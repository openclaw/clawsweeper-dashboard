## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Match overrides against stripped profile IDs:** `extensions/amazon-bedrock/discovery.ts:174`
  - body: The override fallback tests only `modelId`. When a system inference profile has no `models[]` ARN, discovery passes the region-prefixed profile ID, so a pattern like `^example\.` from the added test never matches and the model falls back to the default context window. Reuse the same full, stripped, and version-normalized candidate list for the regex pass.
  - confidence: 0.94
- **[P3] Use an override example that can take effect:** `docs/providers/bedrock.md:175`
  - body: The sample sets `claude-sonnet-4-6` to 200000, but that model is already in `KNOWN_CONTEXT_WINDOWS` at 1000000 and built-in entries take precedence. Copying this example will be ignored, so use an unlisted model pattern or a value that matches the built-in behavior.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `extensions/amazon-bedrock/discovery.ts:115`
  - body: This adds user-facing Bedrock discovery config and runtime behavior, but the diff does not update `CHANGELOG.md`. OpenClaw policy requires a single-line changelog entry with contributor credit for user-facing fix, feat, and perf changes.
  - confidence: 0.9
