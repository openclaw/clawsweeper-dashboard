## Summary

Keep this PR open: current main still lacks the requested Bedrock regex override config, but the PR head is not merge-ready because override matching still misses normalized profile IDs and the generated schema diff regresses current-main schema data.

## What This Changes

The PR adds an Amazon Bedrock `modelContextWindowOverrides` regex map across discovery, tests, plugin manifest metadata, config types, docs, and generated schema/API hash files.

## Best Possible Solution

Land one canonical Bedrock context-window fix; if this PR is chosen, make override matching use normalized candidate IDs, clean generated schema drift, correct docs and changelog, then validate the plugin/config surface.

## Reproduction Assessment

Yes. Static review of the PR head reproduces the main defect: a regional profile ID can be stripped for known lookups, but regex overrides are tested only against the raw ID, so the added stripped-ID scenario would still fall back to 32K.

## Solution Assessment

No, not as written. The override config is a plausible fix, but the implementation needs normalized matching and generated/docs/changelog cleanup before maintainers compare it against #73382's smaller built-in fallback approach.
