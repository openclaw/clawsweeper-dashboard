## Summary

Keep this PR open. It is still relevant because current main does not expose `modelContextWindowOverrides`, but the latest PR head still has a functional mismatch in the stripped inference-profile path and also needs small docs/changelog cleanup before it is merge-ready.

## What This Changes

The PR adds an Amazon Bedrock discovery `modelContextWindowOverrides` regex config with runtime lookup, cache-key handling, tests, manifest metadata, docs, config typing, and plugin SDK API hash updates.

## Best Possible Solution

Land one canonical Bedrock context-window fix. If this PR is the chosen path, make regex overrides use the same full, stripped, and version-normalized candidate IDs as the built-in lookup, keep deterministic cache keys, fix the docs example, and add the required changelog entry before merge.

## Reproduction Assessment

Yes. The PR's added stripped-ID inference-profile test is a high-confidence reproduction path: with no base ARN, `us.example...` plus `^example\.` cannot match because the implementation tests only the prefixed `modelId`.

## Solution Assessment

No, not as written. The configurable override design is plausible, but the implementation misses the documented/tested stripped-ID path; maintainers should also decide whether this PR or #73382 is the preferred product policy.

  - body: The override fallback currently tests only `modelId`.
