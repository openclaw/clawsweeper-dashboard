## Summary

Keep open: current main and the latest release have native Gateway cron and ACP `sessions_spawn`, but they still do not expose a declarative cron target for `sessions_spawn(runtime="acp")`; the same author has an open paired implementation PR that needs maintainer/security review before this issue can close.

## What This Changes

Request native scheduled cron jobs that can invoke ACP runtime sessions, either through cron-aware `agentTurn` runtime fields or an explicit cron job target for `sessions_spawn(acp)`.

## Best Possible Solution

Define a narrow, documented ACP cron contract that reuses the existing cron task ledger and ACP permission gates, likely by reshaping or replacing https://github.com/openclaw/openclaw/pull/85335 after maintainer review.

## Reproduction Assessment

Not applicable: this is a feature/API request, not a bug report. Source inspection confirms current main has native cron and ACP `sessions_spawn`, but no cron payload/config field that directly targets ACP runtime spawning.

## Solution Assessment

Unclear: the requested direction is plausible, but the best implementation needs maintainer choice on whether ACP scheduling belongs in cron payloads, agent runtime config, or a documented composition of existing cron plus ACP sessions.
