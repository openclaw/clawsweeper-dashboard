## Summary

Keep open: current main and v2026.5.22 have native cron and ACP `sessions_spawn` separately, but neither exposes a cron payload, CLI flag, or protocol field for directly scheduling ACP runs. The same author has an open implementation attempt at https://github.com/openclaw/openclaw/pull/85335, and that path still needs maintainer product/security review.

## What This Changes

The issue asks for native cron scheduling that can trigger ACP `sessions_spawn`/`agentTurn` jobs through declarative config or cron expressions.

## Best Possible Solution

Define a narrow, documented cron ACP payload/CLI/UI/protocol contract that reuses existing ACP permission gates and task-ledger behavior, then land it through the paired PR or a replacement after real-run proof.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug. Source/docs inspection shows cron and ACP `sessions_spawn` exist separately, but there is no direct cron ACP runtime target on current main or v2026.5.22.

## Solution Assessment

Unclear: the proposed YAML shapes are plausible, but the best path needs a maintainer decision on cron payload kind, permission defaults, CLI/UI/protocol surface, and how much should compose existing ACP sessions versus add a new scheduler contract.
