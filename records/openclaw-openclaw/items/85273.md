## Summary

Keep open as a feature/API design request: current OpenClaw already has native Gateway cron and ACP spawn surfaces, but the cron contract does not expose a direct declarative ACP `sessions_spawn`/`runtime: "acp"` target.

## What This Changes

The issue requests first-class scheduled cron configuration that can periodically trigger ACP runtime work via `agentTurn` or `sessions_spawn(acp)`.

## Best Possible Solution

Define a narrow, documented scheduler contract for ACP-targeted jobs, likely a new cron payload kind or explicit runtime/action field that reuses the existing task ledger and ACP policy gates without changing existing cron payload semantics.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report. Source inspection shows native cron and ACP spawn exist, but no direct cron payload/config field for ACP `sessions_spawn` scheduling.

## Solution Assessment

Unclear: the proposed YAML shapes are plausible but would add new public scheduler/protocol surface. A maintainable solution should first decide whether ACP scheduling belongs in cron as a first-class payload, in agent runtime config, or as documented composition of existing cron plus ACP sessions.
