## Summary

Keep open as a feature/API design request. Current main has Gateway cron and ACP `sessions_spawn` separately, but the cron payload, Gateway protocol schema, and CLI only accept message-driven `agentTurn` jobs or `systemEvent` jobs, not a direct `runtime`/`action`/`target` ACP scheduling contract.

## What This Changes

Request native declarative cron scheduling that can trigger ACP runtime `sessions_spawn` or `agentTurn` jobs without external crontab or script wrappers.

## Best Possible Solution

Define a narrow documented cron payload/API for ACP-targeted jobs that reuses existing ACP `sessions_spawn` policy gates and task ledger semantics, aligned with the adjacent direct-spawn, delivery, and affinity designs.

## Reproduction Assessment

Not applicable: this is a feature/API request, not a bug report. Source inspection shows native cron and ACP `sessions_spawn` exist, but no accepted cron payload, CLI, or protocol field for direct ACP `runtime`/`action`/`target` scheduling.

## Solution Assessment

Unclear: the proposed YAML captures the user need, but it would expand public scheduler and ACP spawn contracts. The maintainable path is a maintainer design decision across cron, Gateway protocol, direct spawn, and ACP policy rather than an autonomous patch.
