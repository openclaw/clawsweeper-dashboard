## Summary

Keep open: current main has native cron and ACP `sessions_spawn`, but cron payloads, protocol schema, and CLI only schedule system events or agent-turn prompts; direct scheduled ACP spawn is still a new scheduler/API/security design.

## What This Changes

Request native declarative scheduling that can trigger ACP `sessions_spawn` from cron, either through cron `runtime`/`action` fields or a `cron_jobs` target syntax.

## Best Possible Solution

Define an additive, documented scheduler contract for ACP-targeted jobs that reuses the current cron task ledger and `sessions_spawn` policy gates, with explicit owner/session scoping and protocol, CLI, docs, and regression coverage.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report. Source inspection confirms current main lacks direct cron `runtime`/`action`/`target` fields for scheduled ACP `sessions_spawn`.

## Solution Assessment

Unclear: the proposed YAML shapes are plausible, but this is new public scheduler/protocol surface. The maintainable path should first choose whether ACP scheduling belongs as a new cron payload kind, an `agentTurn` extension, or documented composition of cron plus existing `sessions_spawn`.
