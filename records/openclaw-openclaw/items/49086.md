## Summary

Keep open: current main still ships the unsafe default AGENTS heartbeat guidance with raw `memory/heartbeat-state.json` epoch values and no instruction to obtain timestamps from a tool call. The safer `HEARTBEAT.md` task-state path exists, but the template that triggers the reported footgun has not been aligned yet.

## What This Changes

Request safer heartbeat template guidance so agents do not invent Unix epoch values when updating `memory/heartbeat-state.json`.

## Best Possible Solution

Align the default workspace AGENTS template with `HEARTBEAT.md` `tasks:` for interval checks; if manual JSON remains documented, require tool-produced timestamps such as `date +%s`.

## Reproduction Assessment

Yes for the unsafe source condition: current main and `v2026.5.12` still show raw epoch examples without timestamp-generation instructions. No for deterministic reproduction of a specific LLM hallucination, which the issue describes as probabilistic.

## Solution Assessment

Yes for a narrow docs/template repair. A new CLI command or runtime clamp would be separate follow-up work; the maintainable first fix is to stop the default template from inviting manual epoch calculation.
