## Summary

Keep open: current main and the latest release still show raw `memory/heartbeat-state.json` epoch values in the default heartbeat template without telling agents to obtain them from a tool call, so the reported template footgun remains. Existing `HEARTBEAT.md` task state is the safer path, but the default template has not been aligned with it yet.

## What This Changes

The issue asks OpenClaw to stop the default heartbeat guidance from encouraging model-authored Unix epoch values in `memory/heartbeat-state.json`.

## Best Possible Solution

Align the default workspace template and heartbeat docs around `HEARTBEAT.md` `tasks:` for interval checks; if manual JSON remains documented, require tool-produced timestamps such as `date +%s`.

## Reproduction Assessment

No for the probabilistic model hallucination itself; yes for the unsafe source condition. Current main and `v2026.5.12` still show raw `heartbeat-state.json` epoch values without timestamp-generation instructions.

## Solution Assessment

Yes for a narrow docs/template repair. A new CLI command or runtime clamp would be separate follow-up work; the maintainable first fix is to stop the default template from inviting manual epoch calculation.
