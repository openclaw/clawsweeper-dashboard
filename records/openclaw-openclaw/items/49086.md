## Summary

Keep open: current main and the latest release still ship heartbeat template guidance that shows raw `memory/heartbeat-state.json` epoch values without telling the agent to obtain timestamps from a tool call, so the reported template footgun remains. The safer `HEARTBEAT.md` task-state path exists, but the default template has not been aligned with it yet.

## What This Changes

The issue asks OpenClaw to remove or guard raw Unix timestamp bookkeeping in default heartbeat guidance so models do not author guessed `heartbeat-state.json` epochs.

## Best Possible Solution

Update default AGENTS/HEARTBEAT guidance to steer interval checks to `HEARTBEAT.md` `tasks:` and, if manual JSON remains documented, require tool-produced timestamps such as `date +%s`.

## Reproduction Assessment

No for the probabilistic model hallucination itself; yes for the unsafe source condition. Current main and `v2026.5.7` still show raw `heartbeat-state.json` epochs without timestamp-generation instructions.

## Solution Assessment

Yes. The narrowest maintainable fix is a docs/template repair that prefers existing `HEARTBEAT.md` task state or requires tool-generated epoch values; a new CLI command or runtime clamp can be separate follow-up work.
