## Summary

Keep open: current main still ships the default heartbeat template with raw `memory/heartbeat-state.json` epoch examples and no instruction to obtain timestamps from a tool call, matching the reporter’s and follow-up commenter’s concern.

## What This Changes

The issue asks OpenClaw to stop default heartbeat guidance from steering agents to write raw Unix timestamps to `memory/heartbeat-state.json` without a tool-backed timestamp source.

## Best Possible Solution

Make `HEARTBEAT.md` `tasks:` blocks the default interval-check recommendation in the workspace template, and only mention manual JSON state with an explicit tool-call timestamp warning if it remains documented.

## Reproduction Assessment

No high-confidence live reproduction exists for the probabilistic LLM hallucination itself. The unsafe source condition is reproducible on current main: the default template still shows raw epoch heartbeat state without a tool-call instruction, and the current-time prompt does not provide epoch seconds.

## Solution Assessment

Yes, the narrowest maintainable first fix is a template/docs repair that steers interval checks to existing `HEARTBEAT.md` task state or requires `date +%s` for any manual epoch values. A new CLI command or runtime timestamp clamp can be considered separately, but is not required to address this report.
