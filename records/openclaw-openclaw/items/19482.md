## Summary

Current `main` still has the bug: `HEARTBEAT.md` is written whenever optional bootstrap allows it, not only for brand-new workspaces. The PR’s direction is right, but the branch is stale against current `workspace.ts` because a newer optional-bootstrap guard must be preserved and the tests now belong in `src/agents/workspace.test.ts`.

## What This Changes

The PR gates `HEARTBEAT.md` workspace seeding behind brand-new workspace detection and adds fresh/existing workspace regression tests.

## Best Possible Solution

Land a rebased or replacement fix that writes `HEARTBEAT.md` only when the workspace is brand-new and `shouldWriteBootstrapFile(DEFAULT_HEARTBEAT_FILENAME)` is true, with focused coverage in `src/agents/workspace.test.ts`.

## Reproduction Assessment

Yes. Source inspection on current `main` shows an existing workspace missing `HEARTBEAT.md` still reaches `writeFileIfMissing()`, and the May 9 PR comment supplies live v2026.5.7 reproduction plus after-fix hotpatch proof.

## Solution Assessment

No as currently written, but yes in direction. The narrow maintainable solution is to add `isBrandNewWorkspace` to the current `shouldWriteBootstrapFile(DEFAULT_HEARTBEAT_FILENAME)` condition and move the regression cases into the maintained workspace test file.
