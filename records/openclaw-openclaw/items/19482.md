## Summary

Keep open: current main and v2026.5.7 still recreate a missing `HEARTBEAT.md` in existing workspaces, while the docs and heartbeat runtime treat a missing file as an active heartbeat path. The PR's central fix is still useful, but the branch needs a current-main repair because it targets an obsolete test path and its negative fixture is under-specified.

The PR gates `HEARTBEAT.md` seeding behind brand-new workspace detection and adds fresh/existing workspace regression coverage.

Land a current-main repair that guards the heartbeat write with both `shouldWriteBootstrapFile(DEFAULT_HEARTBEAT_FILENAME)` and `isBrandNewWorkspace`, with regression coverage in `src/agents/workspace.test.ts` for fresh, deleted-heartbeat, and skip-optional paths.

## Solution Assessment

Yes, with current-main adaptation. The narrow maintainable fix is to add `isBrandNewWorkspace` to the existing heartbeat write guard while preserving optional-bootstrap skips and moving the tests into the maintained workspace suite.
