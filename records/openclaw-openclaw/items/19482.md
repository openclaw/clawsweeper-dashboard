Keep this PR open: current main still recreates a missing HEARTBEAT.md in existing workspaces, and the real-user May 9 proof confirms the bug on v2026.5.7. ## Best Possible Solution

Land a rebased or replacement fix that writes HEARTBEAT.md only when the workspace is brand-new and the optional-bootstrap allowlist permits it, with focused regression coverage in src/agents/workspace.test.ts.

Yes, with current-main adaptation. The narrow maintainable fix is to require isBrandNewWorkspace && shouldWriteBootstrapFile(DEFAULT_HEARTBEAT_FILENAME) for the heartbeat write and to cover the initialized-workspace deletion case in the maintained workspace test suite.
