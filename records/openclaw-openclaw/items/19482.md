## Summary

Keep open: current main still recreates a deleted HEARTBEAT.md even though the docs and heartbeat runtime treat a missing file as active. The PR is useful and live-proven, but the branch is stale/conflicting and needs a rebased repair that preserves optional-bootstrap skipping and moves coverage into the maintained workspace test.

## What This Changes

The PR gates HEARTBEAT.md seeding in ensureAgentWorkspace() behind brand-new workspace detection and adds fresh/existing workspace regression tests.

## Best Possible Solution

Land a rebased or replacement fix that writes HEARTBEAT.md only when the workspace is brand-new and the current optional-bootstrap allowlist permits it, with focused regression coverage in src/agents/workspace.test.ts.

## Reproduction Assessment

Yes. Source inspection shows current main still reaches writeFileIfMissing() for a missing HEARTBEAT.md in an existing workspace, and the May 9 live-output comment verifies the same behavior on v2026.5.7 with an after-fix hotpatch result.

## Solution Assessment

Yes, with current-main adaptation. The narrow maintainable fix is to require isBrandNewWorkspace && shouldWriteBootstrapFile(DEFAULT_HEARTBEAT_FILENAME) for the heartbeat write and to cover the initialized-workspace deletion case in the maintained workspace test suite.
