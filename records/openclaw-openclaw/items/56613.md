## Summary

Keep open. Current main has implemented the per-agent TTS half through `agents.list[].tts`, and iOS has a gateway Bot picker that feeds Chat and Talk routing, but Android still has no Voice-tab session picker and both manual mic and Talk Mode continue to route through the resolved main session key. The maintainer discussion explicitly narrowed the remaining scope to Talk/Voice session selection, so this is still actionable rather than cleanup-closeable.

## What This Changes

The issue requests Talk/Voice agent/session switching plus per-agent TTS voice configuration for multi-agent voice conversations.

## Best Possible Solution

Keep `agents.list[].tts` as the canonical per-agent voice surface and implement the remaining Android Voice/Talk session-selection path by adding a Voice/Talk agent/session selector, routing manual mic and Talk Mode through that selected session key, and documenting the Android behavior change; iOS/macOS should only be changed if a parity audit finds a concrete gap beyond their current selected-agent or active-session routing.
