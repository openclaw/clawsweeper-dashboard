## Summary

Keep open. Current main has the per-agent TTS half through the canonical `agents.list[].tts` surface, but Android Voice/Talk still has no agent/session selector and both manual mic and Talk Mode continue to send through the resolved main session key.

## What This Changes

Request to let Voice/Talk choose the active agent or session and to give agents distinct TTS voices.

## Best Possible Solution

Keep `agents.list[].tts` as the canonical voice identity surface, then add selected agent/session routing to Voice/Talk through a maintainer-approved session or agent selector contract with Android and platform-specific UX proof.

## Reproduction Assessment

Yes. Source inspection on current main shows the Android Voice tab lacks selector state, while manual mic and Talk Mode still build `chat.send` around the resolved main session key.

## Solution Assessment

Yes for the remaining direction, with UX scoping. Reusing the existing session or agent-selector contract is the maintainable path; adding the requested `talk.voiceId` config shape would duplicate the canonical `agents.list[].tts` surface.
