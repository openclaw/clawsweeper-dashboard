## Summary

Keep open. Current main and the latest release implement the per-agent TTS half through `agents.list[].tts`, but the Android Voice/Talk path still has no user-selectable agent/session target and continues sending manual mic and Talk Mode turns through the resolved main session key; the related open PR only covers Android Chat selection.

## What This Changes

The issue requests Voice/Talk agent or session switching and per-agent TTS voice configuration for multi-agent voice workflows.

## Best Possible Solution

Keep `agents.list[].tts` as the canonical voice identity surface, then add a maintainer-approved Voice/Talk target-selection contract that routes manual mic and Talk Mode turns to the selected agent/session with platform proof.

## Reproduction Assessment

Yes. Source inspection on current main shows Android Voice has no selector state, and manual mic plus Talk Mode still build `chat.send` around the resolved main session key.

## Solution Assessment

Yes for the implemented half, but unclear for the remaining UX contract. `agents.list[].tts` is the right TTS surface; the remaining work should reuse a shared agent/session selection model rather than adding the proposed `talk.voiceId` config shape.
