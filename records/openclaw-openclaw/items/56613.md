## Summary

Keep open. Current main implements the per-agent TTS half as `agents.list[].tts`, but the Android Voice/Talk path still has no session or agent selector and still routes manual mic and Talk Mode sends through the resolved main session; the related open PR addresses Android Chat selection only.

## What This Changes

The issue requests Talk/Voice agent/session switching and per-agent TTS voice configuration across Android, iOS, and macOS voice surfaces.

## Best Possible Solution

Keep `agents.list[].tts` as the voice identity surface, then add selected session/agent routing to Voice/Talk using the existing session or agent-selector contract and validate platform parity separately.

## Reproduction Assessment

Yes. Source inspection on current main shows Android Chat selection is isolated in `ChatController.switchSession`, while manual mic and Talk Mode still build `chat.send` around `resolveMainSessionKey()` or `mainSessionKey`.

## Solution Assessment

Yes for the direction, but it needs maintainer coordination. Reusing the existing session-selection or agent-selector contract is narrower than adding the requested `talk.voiceId` shape or a second agent voice identity model.
