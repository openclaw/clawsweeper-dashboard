## Summary

Keep open. Current main and the latest release have the per-agent TTS half through the canonical `agents.list[].tts` surface, but Android Voice/Talk still lacks target selection and continues routing manual mic and Talk Mode through the resolved main session key; the related Android Chat selector PR does not cover Voice/Talk.

## What This Changes

Request agent/session selection in mobile/macOS Voice/Talk mode and per-agent TTS voice identity for multi-agent setups.

## Best Possible Solution

Keep `agents.list[].tts` as the voice identity surface, then add a maintainer-approved shared Voice/Talk target-selection contract with Android/iOS/macOS behavior proof.

## Reproduction Assessment

Yes. Source inspection on current main shows the Android Voice tab lacks selector state, manual mic uses `resolveMainSessionKey()` for `chat.send`, and Talk Mode sends and filters around `mainSessionKey`.

## Solution Assessment

Partly. `agents.list[].tts` is the best current solution for per-agent voice identity; the remaining routing work should reuse or extend a shared target-selection contract rather than add `talk.voiceId` or another ad hoc session source.
