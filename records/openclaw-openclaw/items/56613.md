## Summary

Keep open. Current main implements the per-agent TTS half through `agents.list[].tts`, but the Android Voice/Talk path still has no session picker and still routes mic/Talk sends through the resolved main session key; the maintainer discussion also explicitly preserved that remaining scope.

## What This Changes

Request to let Talk/Voice users switch the active agent/session and to configure distinct TTS voices per agent.

## Best Possible Solution

Keep `agents.list[].tts` as the voice identity surface, then route Android manual mic and Talk Mode through a selected session/agent using the existing Chat session or agent-selector contract; coordinate with https://github.com/openclaw/openclaw/pull/80422 and validate iOS/macOS parity separately.

## Reproduction Assessment

Yes. A source-level trace on current main shows Android Chat selection is isolated to `ChatController.switchSession`, while Voice manual mic and Talk Mode still build `chat.send` around `resolveMainSessionKey()` or `mainSessionKey`.

## Solution Assessment

Yes for the implementation direction, but it needs maintainer coordination. Reusing the existing session-selection contract is narrower than adding a new `talk.voiceId` config shape or a second agent identity model.
