## Summary

Keep open: current main and v2026.6.8 still use a single shared Gateway token/password operator credential, while this issue asks for a new scoped Gateway principal model plus token-bound agent dispatch restrictions that require maintainer security and product design.

## What This Changes

The issue requests `gateway.auth.tokens[]` with per-token scopes and an optional per-token allowed-agent dispatch lane.

## Best Possible Solution

Keep this as the focused scoped-service-principal design thread until maintainers define the Gateway principal schema, scope vocabulary, dispatch constraints, docs, migration/doctor behavior, and security test plan.

## Reproduction Assessment

Yes, as a source-level capability check rather than a runtime bug repro. Current main and v2026.6.8 support singular shared token/password auth plus trusted-proxy/device identity paths, with no scoped token list or token-bound allowedAgents lane.

## Solution Assessment

Unclear as proposed. Gateway core is the plausible owner boundary for bearer-token authority, but the exact schema, scope names, wildcard semantics, dispatch-lane behavior, and upgrade path need maintainer security/product direction first.
