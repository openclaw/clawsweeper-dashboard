## Summary

Keep open: current main and the latest release still model Gateway bearer auth as one shared operator credential, while this asks for a new scoped Gateway principal model plus token-bound agent dispatch restrictions that need maintainer security/product design.

## What This Changes

Request scope-bound `gateway.auth.tokens[]` credentials and a per-token primitive limiting which agents a dispatch-capable credential may target.

## Best Possible Solution

Keep this as the focused scoped-service-principal design thread until maintainers define the Gateway principal schema, scope vocabulary, dispatch constraints, docs, migration/doctor behavior, and test plan.

## Reproduction Assessment

No runtime bug reproduction is applicable, but there is a high-confidence source capability check. Current main and v2026.6.8 only support singular shared token/password auth plus trusted-proxy/device identity paths, with no scoped token list or token-bound allowedAgents lane.

## Solution Assessment

Unclear as proposed. Gateway core is the plausible owner boundary for bearer-token authority, but the exact schema, scope names, wildcard semantics, dispatch-lane behavior, and upgrade path need maintainer security/product direction first.
