## Summary

Keep open: current main and v2026.6.9 still expose Gateway shared-secret auth as one broad operator credential, while this issue asks for a new scoped Gateway principal model plus token-bound agent dispatch restrictions that require maintainer security/product design.

## What This Changes

The issue requests native scope-bound gateway bearer tokens plus optional per-token agent dispatch restrictions.

## Best Possible Solution

Define one Gateway-owned principal model that binds bearer credentials to current operator scopes and optional dispatch target restrictions, with docs, tests, SecretRef support, revocation/rotation semantics, and migration guidance.

## Reproduction Assessment

Yes, as a source-level capability check: current main documents and implements shared-secret bearer auth as a full operator surface, not a bearer-bound scope/principal model.

## Solution Assessment

Unclear; the gateway is the right owner boundary, but the proposed `tokens[]` and dispatch-lane shape needs maintainer security/product agreement before code.
