## Summary

Keep open. Current main still has the reported `/model status` behavior, and this PR is useful source work, but the branch is not merge-ready because it can hide legitimate native providers, misses alias-prefix matches, regresses defensive registry coercion, and lacks the required changelog entry.

## What This Changes

The PR adds a `loadModelCatalog()` post-filter and regression tests for provider rows inferred from slashed aggregator model IDs.

## Best Possible Solution

Land one corrected model-catalog/status fix that suppresses only proven duplicate aggregator-prefix artifacts, preserves real native-provider rows from explicit config and implicit discovery, normalizes provider aliases consistently, keeps defensive registry coercion, and includes focused tests plus a changelog entry.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to provide an `openrouter` row with `google/gemini-3-flash-preview` plus a registry `google`/`gemini-3-flash-preview` row; current main will include both and `/model status` will group both providers.

## Solution Assessment

No, not as written. Filtering in `loadModelCatalog()` is the right boundary, but this implementation should preserve implicitly discovered native providers, normalize split prefixes before matching, keep defensive coercion, and add the required user-facing changelog entry.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve implicitly discovered native providers:** `src/agents/model-catalog.ts:170-172`
  - body: The filter treats only `cfg.models.providers` keys as real providers. If OpenRouter is configured with `google/gemini-*` and Google is discovered through auth/env/profile or built-in provider discovery, the legitimate `google` row matches the split aggregator key and is dropped from `/model status`.
  - confidence: 0.88
- **[P2] Normalize split prefixes before matching:** `src/agents/model-catalog.ts:147`
  - body: `aggregatorModelKeys` stores the raw prefix from the slashed model ID, but lookup later uses `normalizeProviderId(provider)`. Aliased prefixes such as `z-ai` or `bytedance` will miss the key and leave phantom rows visible.
  - confidence: 0.84
- **[P2] Keep defensive registry field coercion:** `src/agents/model-catalog.ts:155`
  - body: The new loop calls `.trim()` directly on registry `id`, `provider`, and `name` values. A malformed non-string registry row can throw and abort catalog loading instead of being skipped or normalized defensively as current main does.
  - confidence: 0.82
- **[P3] Add the required changelog entry:** `CHANGELOG.md:20-25`
  - body: This is a user-facing `/model status` fix, but the PR does not update `CHANGELOG.md`. OpenClaw policy requires a single-line changelog entry for user-facing fixes with appropriate human contributor credit.
  - confidence: 0.9
