## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve implicitly discovered native providers:** `src/agents/model-catalog.ts:170-172`
  - body: The filter treats only `cfg.models.providers` keys as real providers. If OpenRouter is configured with `google/gemini-*` and Google is discovered through auth/env/profile, the legitimate direct Google row matches the split aggregator key and is dropped from `/model status`.
  - confidence: 0.88
- **[P2] Normalize split prefixes before matching:** `src/agents/model-catalog.ts:147`
  - body: `aggregatorModelKeys` stores the raw prefix from the slashed model ID, but lookup later uses `normalizeProviderId(provider)`. Aliased prefixes such as `z-ai` or `bytedance` will miss the key and leave phantom rows visible.
  - confidence: 0.84
- **[P2] Coerce registry fields before trimming:** `src/agents/model-catalog.ts:155`
  - body: The new loop calls `.trim()` directly on registry `id`, `provider`, and `name` values. A malformed non-string registry row can throw and abort catalog loading instead of being skipped or normalized defensively as current main does.
  - confidence: 0.82
