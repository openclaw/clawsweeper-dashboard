## Summary

Keep open. Current main still lacks a phantom-provider filter in the model catalog, and the paired bug report remains open, so this PR is not obsolete. The proposed branch is pointed at the right surface, but it has blocking correctness issues: it can hide real implicitly discovered native provider entries, misses aliased provider prefixes, and weakens defensive registry-field handling.

## What This Changes

The PR adds a `loadModelCatalog` post-filter for registry entries that look like provider/model splits from configured slashed aggregator model IDs, plus regression tests for the OpenRouter/Google case and explicit-provider preservation.

## Best Possible Solution

Land one current-main model-catalog/status fix that filters only proven duplicate aggregator-prefix artifacts, preserves real native providers from explicit config, auth/env/profile discovery, and generated `models.json`, normalizes provider aliases consistently, and keeps defensive registry value handling. That canonical fix should close #62317 and supersede duplicate implementation attempts.

## Reproduction Assessment

Yes. A high-confidence reproduction is to stub `ModelRegistry.getAll()` with `openrouter` + `google/gemini-3-flash-preview` and `google` + `gemini-3-flash-preview`; current main returns both, and `/model status` groups them by provider.

## Solution Assessment

No, not as written. Filtering in `loadModelCatalog()` is the right boundary, but this implementation needs to preserve implicitly discovered native providers, normalize split prefixes before matching, and keep robust string coercion.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve implicitly discovered native providers:** `src/agents/model-catalog.ts:170-171`
  - body: The new filter treats only `cfg.models.providers` keys as real providers. If a user has direct Google auth/profile/env discovery but only configures OpenRouter in `models.providers`, a legitimate `google/gemini-3-flash-preview` catalog row matches the split OpenRouter key and is dropped from `/model status`.
  - confidence: 0.88
- **[P2] Normalize split prefixes before matching:** `src/agents/model-catalog.ts:147`
  - body: `aggregatorModelKeys` stores the raw prefix from a slashed model ID, but lookup uses `normalizeProviderId(provider)`. Aliased prefixes such as `z-ai` or `bytedance` therefore do not match their normalized provider rows, so those phantom entries remain visible.
  - confidence: 0.84
- **[P2] Keep defensive registry field coercion:** `src/agents/model-catalog.ts:155`
  - body: The loop now calls `.trim()` directly on registry `id`, `provider`, and `name` values. A malformed non-string registry entry can throw and abort the catalog load path instead of being skipped or normalized defensively as current main does.
  - confidence: 0.82
