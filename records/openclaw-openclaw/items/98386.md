The issue reports that Claude Sonnet 5 requests through Anthropic Messages can still send legacy thinking, sampling, and service-tier payload fields that Anthropic rejects.

## Best Possible Solution

Keep this issue open until the open Sonnet 5 support PR is repaired and merged, or land an equivalent narrow fix that adds Sonnet 5 to the shared Anthropic contract and final payload normalization without an unsafe default flip.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main does not classify `claude-sonnet-5` as adaptive/max/xhigh, so Sonnet 5 reasoning can fall through to manual `budget_tokens` and configured request fields can still reach the Anthropic payload path. I did not run a live Anthropic request because this read-only review has no credentials.

Yes, the maintainable fix is to extend the shared Claude model contract and Anthropic Messages final payload sanitizer, then cover direct and canonical-alias Sonnet 5 cases in regression tests. The existing open PR is the right review target but needs repair and proof before it can close this issue.
