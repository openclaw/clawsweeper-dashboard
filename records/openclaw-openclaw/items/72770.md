Full review comments:

- **[P1] Validate share authority before minting tokens:** `src/gateway/server-methods/session-share-tokens.ts:27-31`
  - body: This handler turns any valid sessionKey string into a bearer token without checking that the session exists or that the caller is allowed to share it. Once token auth is wired, a write-scoped caller could pre-mint read tokens for sessions they should not expose.
  - confidence: 0.9
- **[P1] Store bearer tokens in the shared SQLite state path:** `src/infra/session-share-tokens.ts:90`
  - body: The new store appends full bearer-token entries to session-share-tokens.jsonl under the state dir. Runtime state should use the shared SQLite path, and this token store needs transactional revoke/prune behavior plus schema/upgrade review instead of a sidecar file.
  - confidence: 0.88
- **[P2] Keep token methods private until auth consumes them:** `src/gateway/server-methods-list.ts:99-100`
  - body: The PR advertises create/revoke share-token methods, but the token-auth middleware is explicitly left as follow-up work. That creates a public API that returns tokens users cannot use for the promised read-only session access.
  - confidence: 0.84
