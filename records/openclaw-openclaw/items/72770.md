## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Validate session ownership before minting share tokens:** `src/gateway/server-methods/session-share-tokens.ts:27-31`
  - body: This handler turns any validated `sessionKey` string into a bearer token without checking that the session exists or that the caller is allowed to share it. Once token auth is wired, a write-scoped caller could mint read tokens for other sessions, so resolve and authorize the session before creating the token.
  - confidence: 0.9
- **[P1] Store share tokens in SQLite-owned state:** `src/infra/session-share-tokens.ts:90`
  - body: This writes full bearer-token records to `session-share-tokens.jsonl` under the state dir. OpenClaw runtime state is SQLite-owned, and a token store needs durable revoke/prune/upgrade semantics; the sidecar creates a separate secret lifecycle.
  - confidence: 0.88
- **[P2] Keep token methods private until read-only auth is wired:** `src/gateway/server-methods-list.ts:99-100`
  - body: The PR advertises the create/revoke methods, but the PR body says the auth middleware that would consume these tokens is a follow-up. That publishes a token API users cannot actually use for read-only access and commits the protocol before the security contract is reviewed.
  - confidence: 0.84

## Security Review
