## Summary

Close as implemented: current main no longer masks Telegram DM-topic rejection as a successful plain-DM delivery. The merged Telegram fix makes private-topic sends fail closed, so cron delivery can surface failure instead of recording the fallback as ordinary delivered.

## What This Changes

The issue requests that Telegram DM-topic threadless fallback be distinguishable from clean topic delivery in cron state and status output.

## Best Possible Solution

Keep the fail-closed Telegram DM-topic behavior: rejected private-topic sends should surface as failed delivery rather than being converted into a misleading plain-DM success.

## Reproduction Assessment

No for current main: source inspection shows the original fallback-to-main path no longer runs for Telegram DM-topic message thread not found failures. The old v2026.5.7 behavior is backed by the issue and the merged fix PR's before/after live proof.

## Solution Assessment

Yes. Failing private-topic sends loudly is narrower and safer than adding a new delivered-fallback cron status while still allowing wrong-surface delivery.
