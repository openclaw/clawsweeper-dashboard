## Summary

Close as implemented: current main no longer converts a rejected Telegram DM-topic send into a successful plain-DM fallback, so the cron can no longer record that fallback-to-main delivery as ordinary `delivered`.

## What This Changes

The issue asks cron Telegram delivery state to distinguish topic-routed delivery from fallback-to-main DM delivery after Telegram rejects a private DM topic id.

## Best Possible Solution

Keep the fail-closed Telegram DM-topic behavior: rejected private-topic sends should surface as delivery failure rather than being converted into a misleading plain-DM success.

## Reproduction Assessment

No for current main: source inspection and regression tests show the fallback-to-main path no longer runs for Telegram DM-topic `message thread not found` failures. The old v2026.5.7 behavior is supported by the reporter's logs and the merged fix PR history.

## Solution Assessment

Yes. Failing private-topic sends loudly is narrower and safer than adding a new delivered-fallback cron status while still allowing wrong-surface delivery.
