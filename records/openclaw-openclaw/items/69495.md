## Summary

Keep open. The feature is still needed on current main, but this external PR needs semantic fixes for heartbeat fallback overrides, a rebase against the current fallback-helper structure, and real after-fix behavior proof before merge.

## What This Changes

The PR extends `agents.*.heartbeat.model` from a string-only override to a `{ primary, fallbacks }` model config and threads heartbeat fallback data through runtime, pricing, docs, schema, tests, and changelog surfaces.

## Best Possible Solution

Rebase the feature onto current main, preserve `undefined` versus explicit `[]` heartbeat fallbacks end to end, derive fallbacks for both runner-driven and direct heartbeat reply paths, update generated config/docs/tests, and attach redacted real runtime proof before merge.

## Reproduction Assessment

Yes. Current main is source-reproducible as string-only: the schema/type reject object-form heartbeat models and the runner only forwards one `heartbeatModelOverride`; the PR defects are also source-visible against the fallback contract that treats `[]` differently from `undefined`.

## Solution Assessment

No. The product direction is the narrow maintainable solution, but this patch is not merge-ready until it preserves explicit empty fallback overrides, forwards direct heartbeat fallbacks, and adapts the override plumbing to current main.
