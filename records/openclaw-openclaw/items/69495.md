## Summary

Keep open. Current main still rejects object-form `agents.*.heartbeat.model` and has no heartbeat fallback plumbing, so the PR addresses a real feature gap, but the patch is not merge-ready because it drops explicit empty fallback overrides, misses direct heartbeat reply fallback forwarding, needs current-main rebase/adaptation, and lacks the required changelog attribution.

## What This Changes

The PR changes heartbeat model configuration to accept `{ primary, fallbacks }`, threads heartbeat fallback overrides through reply execution, updates pricing-cache/config-schema/help/docs/changelog surfaces, and adds heartbeat model override tests.

## Best Possible Solution

Land a rebased implementation that reuses the shared model schema/type for heartbeat config, preserves `undefined` versus explicit empty fallback arrays end to end, derives fallbacks for both runner-driven and direct heartbeat reply paths, updates pricing/docs/generated schema/help/tests, and keeps changelog credit aligned with repository policy.

Yes. ## Solution Assessment

No. The product direction is the right narrow solution, but the current patch is not the best implementation until it preserves explicit empty fallback overrides, forwards fallbacks for direct heartbeat calls, and is rebased onto current fallback-helper structure.
