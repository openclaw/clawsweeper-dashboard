## What This Changes

The PR extends `agents.*.heartbeat.model` to accept `{ primary, fallbacks }` and threads heartbeat fallback models through schema, runtime reply plumbing, pricing metadata, docs, tests, and changelog.

## Best Possible Solution

Rebase onto current main and implement heartbeat object-form model config through the current schema and per-run fallback seams, preserving `undefined` versus explicit `[]`, with regenerated docs/schema outputs and redacted real fallback proof.

## Reproduction Assessment

Yes, from source inspection rather than a live run. Current main still rejects object-form heartbeat models and forwards only a single heartbeat model string; the PR defects are visible against the existing `fallbacksOverride` contract for `undefined` versus `[]`.

## Solution Assessment

No. The requested direction is maintainable and consistent with existing model-list config, but this patch should not merge until it preserves explicit empty fallback overrides, forwards direct heartbeat fallback lists, rebases cleanly, and includes real behavior proof.
