## Summary

Keep open: current main still lacks the requested heartbeat `{ primary, fallbacks }` support, and this PR is the active implementation thread. The branch is not merge-ready because it drops explicit empty fallback semantics, misses heartbeat fallback forwarding for direct calls, lacks real behavior proof, and is currently merge-conflicting.

## What This Changes

The branch widens `agents.*.heartbeat.model` to string-or-object form, threads heartbeat fallback overrides through reply/model-fallback execution, and updates pricing metadata, docs/help/generated schema, tests, and changelog.

## Best Possible Solution

Land a rebased implementation that uses the shared model config shape for heartbeat, preserves explicit `fallbacks: []`, covers direct heartbeat calls, regenerates docs/schema outputs, and includes redacted live or terminal proof of fallback behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows the schema/type/runtime path is still string-only, and the PR defects are visible from the existing `undefined` versus `[]` fallback contract without needing a live provider outage.

## Solution Assessment

No, not as currently written. The direction matches existing model-list config, but the patch needs to preserve explicit empty fallback overrides, forward fallback lists for direct heartbeat calls, rebase onto current main, and add real behavior proof before merge.

  - confidence: 0.86
