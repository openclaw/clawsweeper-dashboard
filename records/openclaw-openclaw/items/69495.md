## Summary

Current main still lacks heartbeat `{ primary, fallbacks }` config/runtime support, so this PR contains unique useful work and should stay open. It should not merge as-is because the patch drops explicit empty heartbeat fallback overrides, omits fallback derivation for direct heartbeat reply calls, is outdated against current main, and still lacks after-fix real behavior proof.

## What This Changes

The PR changes heartbeat config, runtime model-selection plumbing, pricing metadata, docs/help/generated schema, tests, and changelog to support `{ primary, fallbacks }` for `agents.*.heartbeat.model`.

## Best Possible Solution

Rebase onto current main and implement the heartbeat object-form model through the current schema and per-run fallback seams, preserving `undefined` versus explicit `[]`, deriving fallback lists for direct heartbeat calls, regenerating schema/docs outputs, and adding redacted real fallback proof.

## Reproduction Assessment

Yes, from source inspection rather than a live run. Current main still accepts only a string heartbeat model and forwards only one heartbeat model override, while the PR defects are visible against the existing `fallbacksOverride` contract for `undefined` versus `[]`.

## Solution Assessment

No. The requested direction is consistent with existing model-list config, but this patch should not merge until it preserves explicit empty fallback overrides, forwards direct heartbeat fallback lists, rebases cleanly, and includes real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve explicit empty heartbeat fallbacks:** `src/infra/heartbeat-runner.ts:1035`
  - body: The new `heartbeatModelFallbacks` option is emitted only when the sanitized list is non-empty. Downstream fallback resolution treats `undefined` as inherit and `[]` as an explicit no-fallback override, so `{ primary, fallbacks: [] }` still inherits agent-level fallbacks despite the documented disable semantics.
  - confidence: 0.95
- **[P2] Forward fallback lists for direct heartbeat calls:** `src/auto-reply/reply/get-reply.ts:203`
  - body: This fallback path resolves the object-form primary when `heartbeatModelOverride` is absent, but it never derives `heartbeatModelFallbacks`. Direct `getReplyFromConfig(..., { isHeartbeat: true })` callers that rely on `heartbeat.model` fallbacks will run only the primary and lose failover.
  - confidence: 0.86
