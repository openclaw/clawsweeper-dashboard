## Summary

Keep open. The PR targets a real heartbeat fallback gap, but the patch still drops explicit empty fallback overrides, misses direct heartbeat callers, lacks after-fix real behavior proof, and is now conflicting with current main.

## What This Changes

Adds object-form `agents.*.heartbeat.model` support, heartbeat fallback plumbing, pricing/docs/help/schema updates, tests, and a changelog entry.

## Best Possible Solution

Land a rebased implementation that widens heartbeat model config, preserves explicit `[]` fallback semantics, forwards fallback overrides through all heartbeat reply paths, updates docs/schema/pricing, and includes redacted real behavior proof.

## Reproduction Assessment

Yes, at source level. Current main has a string-only heartbeat schema/type and forwards only one heartbeat model override; I did not run a live provider 429 fallback.

## Solution Assessment

No, not as currently written. The direction matches the existing model-list contract, but the patch needs empty-array preservation, direct-call fallback forwarding, a rebase, and real behavior proof before merge.
