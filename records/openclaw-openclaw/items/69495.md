## Summary

Keep open: current main still lacks object-form `agents.*.heartbeat.model` support and heartbeat-specific fallback plumbing, so this PR is not obsolete. The proposed patch still has correctness blockers around explicit empty fallback overrides and direct heartbeat callers, and the external contributor has not posted after-fix real behavior proof.

## What This Changes

The PR widens heartbeat model config to accept `{ primary, fallbacks }` and threads heartbeat fallback choices through reply execution, pricing collection, generated schema/help, docs, tests, and changelog.

## Best Possible Solution

Land a rebased implementation that uses the existing `AgentModelConfig` object form for heartbeat models, preserves explicit per-run fallback overrides including `[]`, updates generated schema/docs/pricing/tests, and includes redacted real heartbeat fallback proof.

## Reproduction Assessment

Yes, at source level: current main still rejects object-form heartbeat model config and forwards only a single heartbeat model override. I did not reproduce the live provider-429 failover path.

## Solution Assessment

No, not as currently written. The object-form direction matches the existing model config contract, but the patch needs empty-array preservation, direct-call fallback forwarding, a rebase onto current fallback plumbing, and real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve explicit empty heartbeat fallbacks:** `src/infra/heartbeat-runner.ts:1038`
  - body: The PR documents `fallbacks: []` as disabling heartbeat fallbacks, and downstream fallback resolution treats `[]` differently from `undefined`. Because this conditional only passes non-empty arrays, an explicit empty list is dropped and heartbeat ticks can still inherit configured fallbacks.
  - confidence: 0.95
- **[P2] Forward heartbeat fallbacks in the direct reply path:** `src/auto-reply/reply/get-reply.ts:207`
  - body: This fallback path now extracts the object-form heartbeat primary from config when no runner override is supplied, but it never derives the paired fallback list. Direct heartbeat callers using object-form config would run only the primary and lose failover.
  - confidence: 0.86
