## Summary

Keep open: current main still has the single-pass Claude CLI transcript probe, and false probe results still invalidate reusable Claude CLI session state. The issue is also paired with the open closing PR https://github.com/openclaw/openclaw/pull/81048, which is unmerged and currently conflicting, so this should remain open until a reviewed fix or replacement lands.

## What This Changes

The issue reports a Claude CLI transcript probe race where a transcript can exist before an assistant record is flushed, causing `transcript-missing` invalidation and cold session starts.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/81048 with a reviewed fix that uses the canonical Claude project path rules, preserves safe invalidation for genuinely missing transcripts, handles delayed transcript/session-id availability, and adds regression coverage for both resume call sites.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No for a live Claude CLI timing repro in this read-only pass; yes at source level, because a missing or header-only Claude project transcript returns false and both resume paths turn that into `transcript-missing` invalidation.

## Solution Assessment

Is this the best way to solve the issue? Keeping this issue open is the right path while the linked closing PR or a replacement is reviewed; the final repair should stay narrow around transcript/session-id verification, canonical path handling, diagnostics, and tests.
