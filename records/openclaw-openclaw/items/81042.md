## Summary

Keep open: current main still has the single-pass Claude CLI transcript probe, and an open PR with closing syntax is already the implementation path for this issue.

## What This Changes

Fix the Claude CLI transcript probe so an asynchronously flushed existing transcript is not misclassified as missing and does not force a cold session restart.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81048 if its retry budget and proof are acceptable, then let the merge close this issue.

## Reproduction Assessment

Yes, at source level: a matching Claude JSONL without an assistant record makes the helper return false, and both resume paths convert that false result into `transcript-missing`. Live timing proof is supplied in the linked PR discussion, but was not rerun here.

## Solution Assessment

Yes, the requested direction is the narrow fix: keep the Claude-only transcript validation but retry boundedly across the flush window and log redacted negative diagnostics. The open PR is the right vehicle to review that implementation.
