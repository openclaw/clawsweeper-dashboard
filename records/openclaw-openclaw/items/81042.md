## Summary

Keep open: current main still has the single-pass Claude CLI transcript probe described by the report, and both resume callers can discard a saved binding when that probe returns false. This is narrow enough for a focused repair, but the live Claude CLI flush timing was not reproduced during this read-only sweep.

## What This Changes

The issue reports a race where `claudeCliSessionTranscriptHasContent` can treat an existing Claude CLI transcript as missing before an assistant message is flushed, causing a cold session start.

## Best Possible Solution

Add a bounded retry and structured negative diagnostics inside the Claude CLI transcript probe while preserving the existing Claude-only resume policy and path-safety checks.

## Reproduction Assessment

No high-confidence live timing reproduction was established in this read-only sweep. Source inspection does show the deterministic false-negative shape: a matching JSONL without an assistant record at probe time returns false and triggers missing-transcript invalidation.

## Solution Assessment

Yes, the suggested direction is likely the narrowest maintainable fix: retry once only when a matching transcript file exists, then log redacted scan diagnostics on the remaining negative path.
