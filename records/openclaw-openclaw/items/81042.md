## Summary

Keep open: current main still has the single-pass Claude CLI transcript probe, and false results still invalidate resume state in both command execution and CLI preparation. The issue is paired with the open closing PR https://github.com/openclaw/openclaw/pull/81048, which is still unmerged, conflicting, and under review, so the issue should stay open until a reviewed fix lands.

## What This Changes

The issue requests a bounded retry and negative-path diagnostics for Claude CLI transcript probing so transient JSONL flush gaps do not clear reusable resume bindings.

## Best Possible Solution

Land a reviewed fix that makes Claude transcript verification tolerant of flush or creation latency, uses the canonical workspace path rules, preserves safe invalidation for genuinely missing transcripts, and covers both resume call sites with regression tests.
