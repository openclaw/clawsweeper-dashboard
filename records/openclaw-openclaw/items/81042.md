## Summary

Keep open: current main still has the single-pass Claude CLI transcript probe, and an open same-author PR already targets this issue with a bounded retry and diagnostics. The issue should stay open until https://github.com/openclaw/openclaw/pull/81048 is reviewed and either merged or closed.

## What This Changes

The issue reports that the Claude CLI transcript probe can return a false negative while the project JSONL exists but has not yet flushed an assistant message, causing resume state to be cleared as missing.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81048 if it remains correct, so the linked issue closes through the normal merge path.

## Reproduction Assessment

No high-confidence live timing reproduction was established in this read-only review. Source inspection does show the deterministic false-negative shape: a matching JSONL without an assistant record makes the helper return false, and both resume callers treat that as missing transcript.

## Solution Assessment

Yes, the linked PR direction is the narrowest maintainable fix: retry once only when a matching transcript file exists, then log structured negative diagnostics if the rescan still fails.
