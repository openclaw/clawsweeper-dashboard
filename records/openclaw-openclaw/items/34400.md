## Summary

Keep open. Current main still has source-visible recursive memory gaps: nested dated files miss temporal decay, dirty searches can query before refresh completes, and the QMD watcher still builds Chokidar watch targets from path-plus-glob patterns while Chokidar 5 has no glob support.

## What This Changes

The issue asks for memory_search to recursively support markdown files under nested memory/ subdirectories such as memory/YYYY-MM/YYYY-MM-DD.md.

## Best Possible Solution

Review and land or supersede the memory-core follow-up so recursive collection, nested dated-path temporal decay, dirty first-search freshness, QMD watcher behavior, public wording, and focused regression tests all agree.

## Reproduction Assessment

Yes, source-level. Current main recursively collects memory files, but nested dated paths miss temporal decay, dirty builtin searches launch sync in the background before querying, and QMD watcher setup still joins roots with glob patterns.

## Solution Assessment

Yes. The maintainable path is to fix the existing recursive memory contract across date parsing, freshness, watcher roots, wording, and tests rather than add a new config mode or separate collection feature.
