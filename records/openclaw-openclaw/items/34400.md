## Summary

Keep open. Current main still only partially satisfies the request: recursive memory file collection exists, but nested dated paths still miss temporal decay, built-in dirty searches still sync in the background, QMD watcher targets still include glob-shaped paths, and remaining user-facing copy still says `memory/*.md`; the reporter also has paired open PR #72748 for the same follow-up.

## What This Changes

Request recursive `memory/**/*.md` support for `memory_search`, including nested daily memory files such as `memory/YYYY-MM/YYYY-MM-DD.md`.

## Best Possible Solution

Land #72748 or an equivalent focused memory-core fix that recognizes nested dated memory paths, awaits dirty search synchronization before querying, fixes QMD watch filtering under chokidar 5, updates flat-path copy, and adds targeted regression coverage.
