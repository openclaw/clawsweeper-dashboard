## What This Changes

memory_search should support Markdown files nested under memory subdirectories such as memory/YYYY-MM/YYYY-MM-DD.md without losing freshness, watcher updates, or temporal ranking.

## Best Possible Solution

Review and land or supersede the memory-core follow-up so recursive collection, nested dated-path temporal decay, dirty first-search freshness, QMD watcher behavior, public wording, and focused regression tests all agree.

## Reproduction Assessment

Yes, source-level. Current main recursively collects memory files, but nested dated paths miss temporal decay, dirty builtin searches launch sync in the background before querying, and QMD watcher setup still joins roots with glob patterns.

## Solution Assessment

Yes. The maintainable fix is to repair the existing recursive memory contract across date parsing, search freshness, watcher roots, wording, and regression tests rather than add a new config mode or separate collection feature.
