## Summary

Keep open. Current main partially supports recursive memory collection and watcher syncing, but dated nested memory files still miss temporal decay, dirty built-in searches still start an unawaited sync before searching, and several user-facing strings still advertise the flat `memory/*.md` pattern; the issue is already paired with open follow-up PR #72748.

## What This Changes

Make `memory_search` and `memory_get` consistently support recursive `memory/**/*.md` notes instead of only flat `memory/*.md` files.

## Best Possible Solution

Finish the linked recursive-memory follow-up so collection, watching, temporal decay, first dirty search, tests, and tool/docs copy all agree on `memory/**/*.md`.

## Reproduction Assessment

Yes from source inspection: a dated nested note such as `memory/2026-02/2026-02-15.md` is collected recursively, but the dated-path regex cannot match it and dirty built-in searches still start sync asynchronously. I did not run a live embedding-backed turn because this review is read-only.

## Solution Assessment

Yes. Completing the memory-core follow-up with regression coverage is the narrow maintainable fix, and the open same-author PR is the implementation candidate rather than closing this issue early.
