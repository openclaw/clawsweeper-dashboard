## Summary

Keep open. Current main still only partially satisfies the request: recursive memory file collection exists, but nested dated memory paths still miss temporal decay, built-in dirty searches still run background sync without awaiting it, QMD watcher targets still combine path plus glob under chokidar 5, and user-facing copy still advertises flat `memory/*.md` files. The reporter also has paired open PR #72748 for these remaining gaps, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Request recursive `memory/**/*.md` support for `memory_search`, including nested memory subdirectories such as monthly folders.

## Best Possible Solution

Land #72748 or an equivalent focused memory-core fix that treats nested daily memory paths as dated memory, synchronizes dirty search state before querying, watches QMD collection roots while filtering patterns safely under chokidar 5, updates remaining flat-path copy to `memory/**/*.md`, and adds targeted regression coverage.

## Reproduction Assessment

Yes. The issue discussion provides concrete runtime steps, and current main has code-level reproduction evidence: the flat dated-path regex cannot match nested daily files, dirty search sync is fire-and-forget, and QMD watch targets still include glob-shaped paths.

## Solution Assessment

Yes, the intended solution direction is narrow and maintainable: keep the fix in memory-core/QMD runtime plus user-facing memory copy and tests. Closing before #72748 or an equivalent merged fix would split the issue from its paired implementation path.
