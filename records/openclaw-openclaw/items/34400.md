## Summary

Keep open. Current main still only partially satisfies recursive memory search: file collection is recursive, but nested dated memory paths still miss temporal decay, dirty built-in searches still launch background sync without awaiting it, QMD watcher targets still include glob-shaped paths, and user-facing copy still advertises flat memory files; the reporter also has paired open PR #72748 for these remaining gaps.

## What This Changes

Support recursive `memory/**/*.md` subdirectory search and indexing behavior for `memory_search`.

## Best Possible Solution

Land #72748 or an equivalent focused memory-core fix covering nested dated memory paths, first-search freshness, QMD watch target filtering, flat-path copy, and regression tests.

## Reproduction Assessment

Yes. The issue discussion gives concrete runtime steps, and current-main source provides a code-level reproduction path for nested dated paths, fire-and-forget dirty search sync, QMD glob-shaped watcher targets, and stale flat-path descriptions.

## Solution Assessment

Yes. The narrowest maintainable solution is in memory-core/QMD runtime plus user-facing memory copy and targeted tests; closing before #72748 or an equivalent merged fix would split the issue from its active implementation path.
