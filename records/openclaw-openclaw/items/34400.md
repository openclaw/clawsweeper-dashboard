## Summary

Current main still has evidence of incomplete recursive memory support: file collection and the built-in watcher are recursive, but dated memory paths under subdirectories still miss temporal decay and some user-facing memory guidance still says `memory/*.md`; the open same-author follow-up PR should remain the implementation path.

## What This Changes

The issue requests recursive `memory/**/*.md` support for `memory_search` so markdown memory files under subdirectories remain searchable.

## Best Possible Solution

Finish the existing follow-up by making recursive memory files consistently indexed, watched, date-classified, searched on the first dirty search, and described accurately in docs and tool guidance.

## Reproduction Assessment

Yes from source inspection: `memory/2026-02/2026-02-15.md` is collected recursively, but `DATED_MEMORY_PATH_RE` cannot match it, and dirty built-in searches still start sync asynchronously before searching. I did not run a live embedding-backed reproduction because this review is read-only.

## Solution Assessment

Yes, completing the recursive-memory follow-up in the memory-core implementation and its tests is the narrowest maintainable fix; the open same-author PR is the current path to review rather than closing this issue early.
