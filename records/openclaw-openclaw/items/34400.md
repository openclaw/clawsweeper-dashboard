## What This Changes

The issue requests recursive `memory/**/*.md` support for `memory_search` so markdown memory files inside subdirectories are indexed and searchable.

## Best Possible Solution

Review and land or supersede the existing recursive-memory follow-up so collection, watching, temporal decay, first dirty search, tests, and user-facing copy all agree on `memory/**/*.md`.

## Reproduction Assessment

Yes from source inspection: nested markdown files are collected recursively, but nested dated paths miss the dated-memory regex and dirty builtin searches still run sync asynchronously before querying. I did not run a live embedding-backed search in this read-only review.

## Solution Assessment

Yes. Completing the open recursive-memory follow-up is the narrow maintainable fix; closing this issue before that PR lands would lose the remaining runtime and copy fixes.
