## Summary

Keep open. Current main still treats billing-classified failures as auth-profile-wide disabled state, and the LiteLLM provider path has no proxy-aware exception for upstream model billing failures sharing one proxy key.

## What This Changes

The issue requests model-scoped or configurable billing cooldown behavior so one LiteLLM upstream model billing error does not disable all models behind the same provider auth profile.

## Best Possible Solution

Define and implement a LiteLLM/proxy-aware billing cooldown scope that limits upstream-model billing failures without weakening direct-provider account exhaustion behavior, then document it and add regression coverage.

## Reproduction Assessment

Yes, from source inspection. Seed one LiteLLM auth profile with active disabledUntil and disabledReason: billing, configure sibling LiteLLM fallback models, and current fallback logic skips those same-provider candidates.

## Solution Assessment

No, the current profile-wide billing disable is not the best fit for proxy aggregators. The safer fix needs an explicit proxy-aware policy rather than a global change to all billing cooldowns.
