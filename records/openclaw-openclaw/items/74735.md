## Summary

Keep open: current main does not support session-scoped `bootstrap-extra-files`, and the latest PR head addresses the prior changelog gap while keeping arbitrary basenames scoped to exact session entries. I found no blocking correctness or security findings in the current diff.

## What This Changes

The PR adds exact `bootstrap-extra-files.sessions[sessionKey]` path lists, permits custom basenames only for matching session entries, preserves global extra-file basename filtering, and updates tests, docs, and changelog.

## Best Possible Solution

Land the narrow exact-session hook contract after maintainer approval, while leaving broader session-type and tiered bootstrap policy to https://github.com/openclaw/openclaw/issues/61001 and https://github.com/openclaw/openclaw/issues/22438.

## Reproduction Assessment

Not applicable as a feature PR. The before/after is still high-confidence from source inspection: current main is global-only with a basename allowlist, while the PR head adds exact session-key handling and focused tests.

## Solution Assessment

Yes. Exact `sessionKey` entries plus the existing workspace path, realpath, and budget guards are a narrow maintainable approach, and preserving the global allowlist avoids broad arbitrary-file injection.
