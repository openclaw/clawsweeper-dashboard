## Summary

Keep open: this is a useful feature PR with sufficient real behavior proof, but the diff regresses existing literal extra-file paths containing square brackets by treating every `[` as a glob character. The exact-session config surface also still needs maintainer acceptance alongside related bootstrap policy work.

## What This Changes

The PR adds exact `bootstrap-extra-files.sessions[sessionKey]` path lists, scoped arbitrary-basename loading for those sessions, deterministic extra-file ordering and dedupe, docs/changelog updates, and focused tests.

## Best Possible Solution

Preserve backward-compatible literal path handling, land exact-session extra files only if maintainers accept the config contract, and leave broader arbitrary/global/session-type bootstrap policy to the canonical related threads.

## Reproduction Assessment

Not applicable to the requested feature itself. The introduced bracket-path regression is reproducible with a temporary workspace containing `docs/[draft]/TOOLS.md`, where Node `fs.glob("docs/[draft]/TOOLS.md")` returns no matches.

## Solution Assessment

No, not until literal bracket paths remain backward-compatible. After that repair, the exact-session approach is a maintainable narrow solution if maintainers accept this hook-owned config surface.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve literal bracket paths before globbing:** `src/agents/workspace.ts:737`
  - body: Treating any `[` as a glob changes previously valid literal paths like `docs/[draft]/TOOLS.md` into a character-class pattern. Node `fs.glob` returns no match for a literal `[draft]` directory, and this code only falls back when globbing throws, so configured extra files in bracketed directories silently stop loading; preserve the literal path path or fall back to it when the glob has no matches.
  - confidence: 0.88
- **[P3] Align the short docs with the basename allowlist:** `docs/cli/hooks.md:291`
  - body: This summary now uses `PROJECT.md` as the example extra file, but global `paths` still reject non-bootstrap basenames unless the file is added through an exact `sessions` entry. Readers following this CLI page can configure a global `PROJECT.md` path that is skipped, so use a recognized global filename or mention the session-specific requirement here and in the bundled hook README.
  - confidence: 0.78
