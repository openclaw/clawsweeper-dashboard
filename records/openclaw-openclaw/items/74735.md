## Summary

Keep open; the PR is useful but not ready because the session list is still passed through glob expansion while arbitrary basenames are enabled, and the user-facing feature still lacks an active changelog entry.

## What This Changes

The branch adds exact `bootstrap-extra-files.sessions[sessionKey]` support, scoped custom-basename loading, ordering/dedupe tests, and hook documentation updates.

## Best Possible Solution

Keep the exact-session feature, but make session entries literal-only or explicitly document/test wildcard arbitrary-file injection, then add the active changelog entry before merge.

## Reproduction Assessment

Not applicable as a feature PR, but the before/after is high-confidence from current-main source and the PR diff. The main review concern is reproducible from the PR code path with a configured `sessions[sessionKey]` value containing a glob such as `dir/*`.

## Solution Assessment

No, not yet. Exact session keys plus workspace guards are a narrow direction, but allowing glob expansion under `allowArbitraryBasenames` undercuts the stated exact-path boundary; a literal-path session loader or an explicit documented wildcard contract would be safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep session entries literal before allowing custom names:** `src/hooks/bundled/bootstrap-extra-files/handler.ts:62-65`
  - body: The PR describes `sessions[sessionKey]` as exact path lists and says custom filenames are explicitly configured, but these entries are passed into the glob-capable loader with `allowArbitraryBasenames`. A config like `sessions[key]: ["secrets/*"]` will inject every matching custom-basename file for that session, bypassing the global basename allowlist. Please reject glob metacharacters for session entries, or document and test wildcard arbitrary-file injection as intentional.
  - confidence: 0.84
- **[P2] Add a changelog entry for the new hook config:** `docs/automation/hooks.md:193-196`
  - body: This introduces the user-facing `bootstrap-extra-files.sessions` config surface and scoped custom-basename behavior, but the PR file list still leaves `CHANGELOG.md` unchanged. OpenClaw policy requires user-facing `feat` changes in the active changelog, so add a single-line `### Changes` entry before merge.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: Session-specific entries can expand globs while arbitrary basenames are enabled, broadening configured prompt data selection beyond the exact file paths described by the PR.

Concerns:

- **[medium] Wildcard session entries can inject arbitrary workspace files:** `src/hooks/bundled/bootstrap-extra-files/handler.ts:62`
  - body: Because session entries reuse the glob-capable extra-file loader with `allowArbitraryBasenames: true`, a wildcard session value can include many non-bootstrap files in model context for that session. Workspace path and size guards still apply, but this is a data-exposure boundary that should be made explicit or constrained before merge.
  - confidence: 0.82
