## Summary

Keep open. Current main and v2026.6.5 still intentionally export OTLP log bodies as "log" by default and only attach trace context for trusted diagnostic log records; changing that default needs maintainer product and security direction.

## What This Changes

The issue requests diagnostics-otel OTLP LogRecord bodies to contain the rendered log message and trace/span IDs so centralized logs are readable and trace-correlatable.

## Best Possible Solution

Keep the privacy default, then add an explicit redaction-bounded log-body opt-in and a trusted-internal trace correlation path only after maintainers approve the security boundary.

## Reproduction Assessment

Yes at source level. Current exporter code and tests show default OTLP log bodies remain "log" and untrusted log-record trace context is omitted; I did not run a live collector.

## Solution Assessment

No, not as a direct default change. The maintainable path is a maintainer-approved privacy/security design, likely an explicit log-body opt-in plus trusted-only trace correlation rather than exporting all rendered log messages by default.
