## What This Changes

The issue reports live-process session `.jsonl.lock` files persisting after embedded-run timeout or subagent completion, causing later requests to fail with `SessionWriteLockTimeoutError`.

## Best Possible Solution

Consolidate the generic live-process session-lock cleanup work on https://github.com/openclaw/openclaw/issues/84193 and https://github.com/openclaw/openclaw/pull/84353, while tracking the subagent completion variant on https://github.com/openclaw/openclaw/issues/85953.
