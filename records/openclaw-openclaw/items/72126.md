## Summary

Keep open. Current main still has the reported unresolved OpenAI Realtime env fallback, and this PR is useful source work, but the patch is not merge-ready because the new synchronous Keychain lookup is unbounded, the branch is dirty against main, and the user-facing fix has no changelog entry.

## What This Changes

The PR adds an OpenAI Realtime API-key resolver/cache for `keychain:<service>:<account>` values and uses it for browser session creation and backend voice bridge creation in `extensions/openai/realtime-voice-provider.ts`.

## Best Possible Solution

Land a provider-local OpenAI Realtime API-key resolver that resolves `keychain:<service>:<account>` for both browser client-secret sessions and backend bridges, bounds the synchronous Keychain call, caches only safe successful results, covers the env/config paths with focused tests, and records the user-facing fix in the active changelog.

## Reproduction Assessment

Yes. On current main, setting `OPENAI_API_KEY` to `keychain:openclaw:OPENAI_API_KEY` and invoking the OpenAI realtime browser-session or bridge path reaches the literal env value in the auth path; linked #72120 includes the resulting OpenAI 401 for the browser-session path.

## Solution Assessment

No, not as-is. The provider-local resolver/cache is the right narrow direction for this PR, but the unbounded synchronous Keychain call, missing changelog entry, stale branch state, and missing focused tests need to be addressed first.
