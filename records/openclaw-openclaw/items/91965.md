## Label Justifications

- `P1`: The reported regression blocks semantic memory retrieval in active agent workflows and can initiate an expensive index recovery.
- `impact:session-state`: The failure prevents the agent from retrieving persisted memory context through `memory_search`.

## Summary

Current main already handles the reported missing-metadata state: search refreshes identity from SQLite, gateway sync rebuilds missing metadata when the configured embedding provider is available, and focused coverage proves search works after repair. The fix is also included in the `v2026.6.6-beta.1` prerelease.

## What This Changes

Investigate and fix a reported state where gateway `memory_search` remains unavailable after index metadata is rebuilt externally.

## Best Possible Solution

Keep the current fresh metadata checks and provider-gated gateway recovery as the canonical behavior; users on `v2026.6.1` should upgrade to `v2026.6.6-beta.1` or a later release containing the fix.

## Reproduction Assessment

Yes at source level: current regression coverage recreates missing metadata with existing indexed chunks and verifies gateway repair plus successful subsequent search. This read-only review did not rerun the reporter's live Ollama environment.

## Solution Assessment

Yes. Fresh SQLite identity reads plus provider-gated missing-metadata recovery solve the actual failure state without introducing another cache or invalidation path.
