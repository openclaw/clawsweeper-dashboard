## Summary

Current main already fixes the reported destructive missing-metadata recovery loop: memory_search refreshes index identity from SQLite before search, missing metadata with existing chunks is repaired only through the safe reindex path when the embedding provider is available, and the repair is included in v2026.6.6-beta.1.

## What This Changes

The issue reports that gateway memory_search can stay paused with "index metadata is missing" and delete repaired memory index metadata after a failed reindex.

## Best Possible Solution

Keep the current main and v2026.6.6-beta.1 memory-core recovery behavior; reopen or file a new focused report only if the same Ollama CLI-rebuild to gateway-search flow still reproduces there.

## Reproduction Assessment

Yes, for the reported released behavior at source level: the issue and the linked prior report describe missing metadata with existing chunks causing paused search. I did not run the macOS/Ollama repro locally, but current source and regression tests cover the repaired state.

## Solution Assessment

Yes. The durable fix belongs in memory-core's identity refresh and safe reindex decisions, not in a gateway-specific cache workaround or new configuration flag.
