## Summary

Keep open. Current main still has only global, chat-type, and channel-level reset policy; agent entries and route-binding session config do not expose reset policy, and the related per-group tracker is narrower rather than a canonical replacement.

## What This Changes

Add per-agent or per-binding session reset configuration so selected workflows can reset sessions independently of global or channel-wide policy.

## Best Possible Solution

Define one additive reset-policy override contract with explicit precedence, then update strict schemas, generated config metadata/help, public docs, and focused runtime tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main does not accept `agents.list[].session.reset` or `bindings[].session.reset`, and runtime reset selection cannot consume such overrides.

## Solution Assessment

Unclear until maintainers choose the public config surface. The requested direction is reasonable, but the maintainable fix should define a single precedence model rather than adding separate ad hoc shortcuts.
