## Summary

Keep this open. Current main and the latest release still only support global, type, and channel reset policy; the existing binding session override is only `dmScope`, and ACP `oneshot` is an adjacent ACP-only mode rather than generic per-agent or per-binding reset policy.

## What This Changes

Add configuration for per-agent or per-route-binding session reset behavior so some workflows can use fresh sessions without changing global or channel-wide policy.

## Best Possible Solution

Define an additive routed reset-policy override contract after effective agent and binding resolution, then update types, strict schemas, generated metadata, docs, and precedence tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence verification path: the requested agent and binding fields are absent from strict schemas, and the runtime reset resolver cannot select policy by agent or binding.

## Solution Assessment

Unclear until maintainers choose the public API and precedence rules. The maintainable direction is a generic routed reset-policy resolver rather than an ACP-only workaround or a channel-specific shortcut.
