## Summary

Keep open. Current main still lacks a generic per-agent or per-route-binding reset policy; existing support is limited to global/type/channel reset policy, binding `dmScope`, and narrower ACP/cron isolated-session paths.

## What This Changes

Request configurable session reset policies per agent or route binding so one-shot workflow agents can reset sessions without changing global or channel-wide behavior.

## Best Possible Solution

Define an additive routed reset-policy override contract after agent and binding resolution, document explicit precedence, and cover config schemas, generated metadata, runtime resolution, and session/routing tests while preserving current defaults.

## Reproduction Assessment

Not applicable as a feature request. There is a high-confidence source verification path: strict config schemas lack the requested fields, and runtime reset resolution has no effective agent or binding input.

## Solution Assessment

Unclear until maintainers choose the public API shape. The maintainable direction is a generic routed reset-policy resolver with explicit precedence, not an ACP-only workaround or channel-specific patch.
