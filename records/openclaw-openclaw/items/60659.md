## Summary

Keep this issue open. Current main still lacks generic per-agent or per-route-binding reset policy; reset configuration remains global/type/channel-scoped, while the existing binding override only changes DM session key scoping and ACP oneshot mode is an adjacent ACP-only behavior.

## What This Changes

The issue requests configurable session reset behavior per agent or per binding so one-shot workflow agents can start fresh without changing global or channel-wide session behavior.

## Best Possible Solution

Define an additive reset-policy override contract after routing resolves the effective agent and binding, then update config types, strict schemas, generated metadata, docs, and precedence tests while preserving existing defaults.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence source verification path: the requested agent and binding fields are absent from strict schemas, and the runtime reset resolver cannot select policy by agent or binding.

## Solution Assessment

Unclear until maintainers choose the public API shape. The maintainable direction is a generic routed reset-policy resolver with explicit precedence rather than an ACP-only workaround or channel-specific patch.
