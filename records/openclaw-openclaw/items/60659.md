## Summary

Keep open. Current main still lacks per-agent or per-route-binding session reset policy: reset config is global/type/channel scoped, route binding session overrides only expose DM scoping, and runtime reset resolution does not receive agent or binding reset input; the related per-group request is narrower, still open, and not a canonical replacement.

## What This Changes

Request adding per-agent or per-binding session reset policies so different agents or routed workflows can use independent session lifecycles.

## Best Possible Solution

Define an additive route-aware reset-policy override contract after agent and binding resolution, then update schemas, generated metadata/help, public docs, and precedence tests around the runtime resolver.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that the requested config shapes are not accepted and that runtime reset resolution has no agent or binding reset input today.

## Solution Assessment

Unclear until maintainers choose the public API shape. The maintainable direction is a generic routed reset-policy resolver with explicit precedence, not an ACP-only workaround or channel-specific patch.
