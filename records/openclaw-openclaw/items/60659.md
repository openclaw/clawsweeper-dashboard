## Summary

Keep open. Current main still lacks per-agent or per-route-binding session reset policy: the public config surface remains global/type/channel scoped, route bindings only expose `dmScope`, and the runtime reset resolver receives no effective agent or binding reset input.

## What This Changes

Request per-agent or per-binding session reset configuration so one-shot workflow agents can reset on each message or short idle windows without changing other agents.

## Best Possible Solution

Define an additive route-aware reset-policy override contract after agent and binding resolution, document explicit precedence, and cover config schemas, generated metadata, runtime resolution, routing, and session tests while preserving current defaults.

## Reproduction Assessment

Not applicable as a feature request. Source verification is high-confidence: strict schemas do not accept `agents.list[].session.reset` or `bindings[].session.reset`, and runtime reset resolution has no effective agent or binding input.

## Solution Assessment

Unclear until maintainers choose the public API shape. The maintainable direction is a generic routed reset-policy resolver with explicit precedence, not an ACP-only workaround or channel-specific patch.
