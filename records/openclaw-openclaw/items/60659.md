## Summary

Keep open. Current main still lacks a generic per-agent or per-route-binding reset policy; the only accepted binding session override is `dmScope`, and the reset resolver still works from global/type/channel policy rather than effective agent or binding context.

## What This Changes

The issue requests configurable session reset behavior per agent or route binding instead of only global, type, or channel-level reset policy.

## Best Possible Solution

Define an additive routed reset-policy override contract after agent and binding resolution, document explicit precedence, and cover config schemas, generated metadata, runtime resolution, and focused session/routing tests while preserving current defaults.

## Reproduction Assessment

Not applicable as a feature request. There is a high-confidence source verification path: strict config schemas reject the requested shapes and runtime reset resolution has no effective agent or binding input.

## Solution Assessment

Unclear until maintainers choose the public API shape. The maintainable direction is a generic routed reset-policy resolver with explicit precedence, not an ACP-only, cron-only, or channel-specific workaround.
