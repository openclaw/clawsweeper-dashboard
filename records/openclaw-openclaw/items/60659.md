## Summary

Keep open. Current main still lacks per-agent or per-route-binding session reset policy: reset config is global/type/channel scoped, binding session overrides only support DM scoping, and runtime reset resolution has no effective agent or binding reset input.

## What This Changes

Request to allow session reset policies at agent or route-binding scope so one-shot agents can reset independently of conversational agents.

## Best Possible Solution

Define an additive route-aware reset-policy override contract after agent and binding resolution, with explicit precedence and coverage for schemas, generated metadata, runtime resolution, docs, and session/routing tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection is high-confidence: strict config schemas do not accept agents.list[].session.reset or bindings[].session.reset, and runtime reset resolution receives no agent or binding reset input.

## Solution Assessment

Unclear until maintainers choose the public API shape. The maintainable direction is a generic routed reset-policy resolver with explicit precedence, not an ACP-only workaround or channel-specific patch.
