## Summary

Keep open. Current main still only supports session reset policy at global, chat-type, and channel scopes; neither agent entries nor route bindings accept reset policy, and the related per-group tracker is narrower than this broader agent/binding request.

## What This Changes

Request per-agent or per-route-binding session reset policy so one-shot workflows can reset on every message or after short idle periods without changing other agents.

## Best Possible Solution

Add an additive reset-policy override contract for agents and/or route bindings, with explicit precedence over channel/type/global policy and coordination with https://github.com/openclaw/openclaw/issues/8122.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high-confidence verification that current main does not accept the proposed agent or binding reset-policy shapes and cannot select reset policy from those scopes.

## Solution Assessment

Unclear until maintainers choose the public config surface. The requested direction is reasonable, but the maintainable fix should define one precedence model instead of adding an isolated shortcut.
