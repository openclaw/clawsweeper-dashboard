## Summary

Keep open. Current main still lacks a generic per-agent or per-route-binding session reset policy; the implemented surface remains global/type/channel reset plus binding-level DM session-key scoping and narrower ACP/cron isolated-session paths.

## What This Changes

Request to add per-agent or per-route-binding session reset configuration so one-shot workflow agents can reset on every or idle inbound message without affecting conversational agents.

## Best Possible Solution

Define an additive routed reset-policy override contract after agent and binding resolution, document explicit precedence, and cover config types, strict schemas, runtime resolution, docs, and focused session/routing tests while preserving current defaults.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence verification path: strict schemas and runtime reset resolution do not accept or use per-agent or per-binding reset policy today.

## Solution Assessment

Unclear until maintainers choose the public API shape. The maintainable direction is a generic routed reset-policy resolver with explicit precedence, not a channel-specific or ACP-only workaround.
