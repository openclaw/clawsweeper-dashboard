## Summary

Keep open: current main still lacks per-agent or normal route-binding reset policy, and the related per-group issue is narrower rather than a complete duplicate.

## What This Changes

The issue requests configurable session reset behavior at the agent or binding level so one-shot workflows can avoid retaining conversation history across inbound messages.

## Best Possible Solution

Define one additive scoped reset policy model, wire it through the shared resolver with explicit precedence, and coordinate it with the narrower Telegram group/topic request instead of adding channel-specific shortcuts first.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high-confidence verification that the requested per-agent and normal binding reset fields are not accepted or consulted on current main.

## Solution Assessment

Unclear until maintainers choose the public API and precedence. The maintainable direction is a shared scoped override passed into the reset resolver, not scattered per-channel reset checks.
