## Summary

Keep open. Current main and v2026.6.8 still only resolve session reset policy at global, chat-type, and channel scopes; per-agent or per-binding reset policy remains a real public config/API decision.

## What This Changes

Requests per-agent or per-binding session reset configuration so one-shot agents can reset without changing global or channel-wide policy.

## Best Possible Solution

Define one additive scoped reset-policy model with explicit precedence, aligned TypeScript types, strict schemas, config help/labels, public docs, and focused runtime tests.

## Reproduction Assessment

Yes, at source level: current main's strict schemas, public types, docs, and runtime resolver show no accepted or applied per-agent or per-binding reset-policy shape. This is a missing feature rather than a broken existing contract.

## Solution Assessment

Unclear until maintainers choose the public API shape. The requested direction is reasonable, but the maintainable fix should be one scoped precedence model rather than parallel agent and binding shortcuts.
