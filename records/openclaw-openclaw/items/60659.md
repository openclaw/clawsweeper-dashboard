## Summary

Keep open. Current main still lacks per-agent or per-route-binding session reset policy; the implemented surface remains global/type/channel scoped, with binding support limited to `dmScope` and adjacent ACP/cron isolated-session workarounds.

## What This Changes

Request adding per-agent or per-binding session reset overrides for workflows that need fresh or short-lived sessions without changing global/channel policy.

## Best Possible Solution

Define an additive route-aware reset-policy override contract after agent and binding resolution, document precedence, and cover schemas, generated metadata, runtime resolution, and session/routing tests while preserving current defaults.

## Reproduction Assessment

Not applicable as a feature request. Source verification is high-confidence: strict config schemas do not accept `agents.list[].session.reset` or `bindings[].session.reset`, and runtime reset resolution has no effective agent or binding input.

## Solution Assessment

Unclear until maintainers choose the public API shape. The maintainable direction is a generic routed reset-policy resolver with explicit precedence, not an ACP-only workaround or channel-specific patch.
