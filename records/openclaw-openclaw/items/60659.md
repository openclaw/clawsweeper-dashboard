## Summary

Keep this issue open. Current main still does not provide per-agent or per-route-binding session reset policy; the implementation remains limited to global, chat-type, and channel-level reset policy, and the existing binding session override only controls DM session scoping.

## What This Changes

The issue requests configurable session reset behavior per agent or per binding so one-shot workflow agents can start fresh without changing global or channel-wide session behavior.

## Best Possible Solution

Define an additive core reset-policy override contract after routing has identified the effective agent and binding, then update types, strict schemas, generated config metadata, docs, and focused precedence tests while preserving existing global/type/channel defaults.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence source verification path: proposed fields such as agents.list[].session.reset and bindings[].session.reset are absent from strict schemas, and the runtime reset resolver cannot select policy by agent or binding.

## Solution Assessment

Unclear until maintainers choose the public API shape. The maintainable direction is a generic core reset-policy resolver with explicit precedence rather than an ACP-only workaround or a channel-specific patch.
