## Summary

Keep open. Current main still supports session reset policy only at global, chat-type, and channel scopes; route bindings accept only `dmScope`, agent entries have no `session` reset field, and the related per-group tracker is narrower rather than a duplicate.

## What This Changes

The issue requests per-agent or per-route-binding session reset policy so one-shot workflows can reset independently from conversational agents.

## Best Possible Solution

Add an additive reset-policy override contract for agents and/or route bindings, with explicit precedence over channel/type/global policy and coordinated semantics with the per-group tracker.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high-confidence verification that current main does not accept the proposed agent or binding reset-policy shapes and cannot select reset policy from those scopes.

## Solution Assessment

Unclear until maintainers choose the public config surface. The requested direction is reasonable, but the maintainable fix should define one precedence model instead of adding an isolated shortcut.
