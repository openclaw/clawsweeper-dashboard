## Summary

Keep open. Current main still lacks a generic per-agent, per-binding, or per-group session reset policy; the implemented contract remains global/type/channel scoped, and binding `session` config only affects direct-message key scoping. The related per-group request at https://github.com/openclaw/openclaw/issues/8122 is narrower and still open, so it does not supersede this broader request.

## What This Changes

Request adding session reset overrides at the agent or route-binding level so one-shot workflows can reset history without changing global/channel behavior.

## Best Possible Solution

Define an additive routed reset-policy contract after agent and binding resolution, document explicit precedence, and update schemas, generated metadata/help, runtime reset resolution, routing/session tests, and docs while preserving current defaults.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The limitation is source-reproducible because strict config contracts lack the requested fields and the runtime reset resolver has no effective agent or binding input.

## Solution Assessment

Unclear until maintainers choose the public API shape. The maintainable direction is a generic routed reset-policy resolver with explicit precedence, not an ACP-only, cron-only, or channel-specific workaround.
