## Summary

Keep this open. Current main still only supports global, chat-type, and channel-level reset policy plus a binding-level `dmScope` override; the broader per-agent/per-binding reset policy is not implemented and is not fully superseded by the narrower open Telegram/group tracker at https://github.com/openclaw/openclaw/issues/8122.

## What This Changes

Request per-agent or per-binding session reset policies so selected workflows can reset or isolate sessions without changing global or channel-wide defaults.

## Best Possible Solution

Define an additive scoped reset contract with explicit precedence, then update config types, strict schemas, docs/help metadata, and focused reset-policy tests across agent and binding routes.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high-confidence verification that current main lacks per-agent and per-binding reset policy fields and runtime inputs.

## Solution Assessment

Unclear until maintainers choose the public API shape and precedence. The maintainable path should add one scoped reset resolver rather than ad hoc per-channel shortcuts.
