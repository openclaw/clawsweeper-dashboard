## Summary

Keep open. Current main still limits session reset policy to global, chat-type, and channel scope; agent entries and route bindings do not expose reset policy, and the open per-group tracker is narrower than this broader per-agent/per-binding request.

## What This Changes

The issue requests per-agent or per-binding session reset configuration so one-shot workflow agents can reset independently from conversational agents.

## Best Possible Solution

Add an additive reset-policy override contract for agent and/or route binding with explicit precedence over channel/type/global policy, coordinated with the conversation-scoped reset work tracked in https://github.com/openclaw/openclaw/issues/8122.

## Reproduction Assessment

Not applicable as a feature request. Source inspection verifies current main only supports global, type, and channel reset policy, while binding session config only supports `dmScope`.

## Solution Assessment

Unclear until maintainers choose the config surface. The requested direction is reasonable, but the maintainable fix should define one precedence model rather than adding an isolated shortcut.
