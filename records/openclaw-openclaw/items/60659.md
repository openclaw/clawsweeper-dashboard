## Summary

Keep open. Current main still exposes session reset policy only globally, by chat type, or by channel; agent entries and route-binding session blocks remain strict and do not carry reset policy. The related per-group tracker at https://github.com/openclaw/openclaw/issues/8122 overlaps, but it does not replace this agent/binding API request.

## What This Changes

Request per-agent or per-route-binding session reset configuration so one-shot agents can reset without changing global, chat-type, or channel-wide policy.

## Best Possible Solution

Define one additive scoped reset-policy model with explicit precedence, aligned TypeScript types, strict Zod schemas, generated config metadata/help, public docs, and focused runtime tests, coordinated with the per-group reset tracker.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that current main does not accept or apply the proposed per-agent or per-binding reset-policy shapes.

## Solution Assessment

Unclear until maintainers choose the public API shape. The requested direction is reasonable, but the maintainable fix should define one scoped precedence model rather than adding an isolated shortcut.
