## Summary

Keep open: current main still has the source-level two-Enter `/status` path, and this PR is the active linked implementation candidate, but the current PR head is conflicting, lacks real behavior proof, and routes status through a reduced browser-side formatter with unresolved correctness gaps.

## What This Changes

The PR marks web UI `/status` as a local slash command, adds a browser-side status executor/default plumbing, and registers focused slash-command tests.

## Best Possible Solution

Fix slash-menu selection so selected no-argument `/status` submits with one Enter while preserving or delegating to the canonical status implementation, with focused regression coverage for that menu-selection path.

## Reproduction Assessment

Yes. Source inspection on current main shows `/status` is present in the slash menu but is not `executeLocal`, and the Enter handler only submits no-argument commands when that flag is true.

## Solution Assessment

No. The PR targets the right user bug, but replacing the canonical status path with a partial browser-side formatter is not the narrowest maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep `/status` on the canonical status path:** `ui/src/ui/chat/slash-command-executor.ts:73-74`
  - body: Adding this local case routes web UI `/status` to `executeStatus` instead of the canonical status implementation. That omits broader status details such as usage, queue, tasks, subagents, auth, uptime, and plugin/session context, so the one-Enter fix should preserve or call the existing status path.
  - confidence: 0.88
- **[P2] Compute fast mode for the active session model:** `src/gateway/session-utils.ts:750-757`
  - body: `fastModeDefault` is derived from the globally resolved default model and then consumed for whichever session is active. In multi-model setups with different per-model `params.fastMode`, a non-default session can display the opposite of runtime behavior in `/status` or `/fast status`.
  - confidence: 0.86
- **[P2] Honor per-model thinking defaults in local status:** `ui/src/ui/chat/slash-command-executor.ts:307-308`
  - body: The local status path only receives a global `thinkingDefault` plus catalog metadata, so it cannot apply per-model configured thinking defaults for the active session. Sessions without a persisted override can therefore show the wrong effective thinking level after this routing change.
  - confidence: 0.84
