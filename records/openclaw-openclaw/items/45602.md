## Summary

Keep open. Current main still has the one-Enter `/status` regression, but this PR should not merge as-is because it moves `/status` into a duplicate local UI status executor with correctness gaps against the canonical status path, and the external PR only provides test output rather than real web UI behavior proof.

## What This Changes

The PR marks `/status` as a local web UI slash command, adds a local status executor plus session-default fields for thinking and fast-mode display, and registers regression tests.

## Best Possible Solution

Fix the slash-menu selection/send path so selecting no-argument agent commands like `/status` submits on the first Enter while preserving the canonical status builder, then add focused regression coverage for that menu flow.

## Reproduction Assessment

Yes. Source inspection on current main shows `/status` is not local and the menu only sends commands marked `executeLocal` with no args, so the first Enter completes the draft and the second Enter sends it.

## Solution Assessment

No. Routing `/status` to a new local executor fixes the keypress symptom but duplicates status formatting and misses gateway/runtime defaults; a menu-submit fix or canonical status RPC path is narrower and safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep `/status` on the canonical status path:** `ui/src/ui/chat/slash-command-executor.ts:72-73`
  - body: Routing `/status` to this new local executor changes more than the Enter behavior: it returns a minimal session summary instead of the existing `buildStatusText` status output, dropping runtime, queue, task, plugin, auth, and other status context. Fix the slash-menu submission behavior or call a canonical status path instead of rebuilding `/status` in the UI.
  - confidence: 0.88
- **[P2] Compute fast mode for the active session model:** `src/gateway/session-utils.ts:750-757`
  - body: `fastModeDefault` is computed from the globally resolved default model and then reused by `/status` and `/fast status` for the active session. If the session uses a different model with a different per-model `params.fastMode`, the UI can display the opposite of runtime behavior.
  - confidence: 0.91
- **[P2] Honor per-model thinking defaults in local status:** `ui/src/ui/chat/slash-command-executor.ts:304-307`
  - body: The local status path only receives a global `thinkingDefault` plus catalog data, so it cannot apply `agents.defaults.models.<provider>/<model>.params.thinking` when no session override is stored. That can make `/status` report a catalog/default level while runtime uses the configured per-model thinking level.
  - confidence: 0.87
