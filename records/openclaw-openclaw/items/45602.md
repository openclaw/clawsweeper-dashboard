## Summary

Keep open. Current main still has the source-level two-Enter path for Control UI `/status`, and this PR targets the right bug, but the proposed branch moves `/status` onto a browser-side formatter that bypasses the canonical status implementation and still lacks real behavior proof.

## What This Changes

The PR marks Control UI `/status` as a local slash command, adds browser-side status/default formatting, and registers slash-command regression tests.

## Best Possible Solution

Fix slash-menu selection so selecting `/status` submits once while preserving or calling the canonical status implementation, with focused regression coverage for the menu-selection flow.

## Reproduction Assessment

Yes. By source inspection on current main, `/status` is present in the menu but is not `executeLocal`, and the Enter handler only sends no-arg commands when that flag is true.

## Solution Assessment

No. The PR targets the right user-visible bug, but replacing `/status` with a browser-side formatter is not the narrowest maintainable fix; it should preserve the canonical status path while fixing one-Enter selection.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Keep `/status` on the canonical status path:** `ui/src/ui/chat/slash-command-executor.ts:73-74`
  - body: Adding this local case sends `/status` to `executeStatus` instead of the canonical status implementation. That replacement omits broader runtime status details such as usage, queue, task, subagent, auth, and plugin/session context, so the one-Enter fix should reuse the canonical path rather than replace it.
  - confidence: 0.88
- **[P2] Compute fast mode for the active session model:** `src/gateway/session-utils.ts:750-757`
  - body: `fastModeDefault` is derived from the globally resolved default model and then consumed for whichever session is active. In multi-model setups with different per-model `params.fastMode`, a non-default session can display the opposite of runtime behavior in `/status` or `/fast status`.
  - confidence: 0.86
- **[P2] Honor per-model thinking defaults in local status:** `ui/src/ui/chat/slash-command-executor.ts:307-308`
  - body: The local status path only receives a global `thinkingDefault` plus catalog metadata, so it cannot apply per-model configured thinking defaults for the active session. Sessions without a persisted override can therefore show the wrong effective thinking level after this routing change.
  - confidence: 0.84
