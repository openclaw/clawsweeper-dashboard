## Summary

Keep open. The linked web UI bug is source-reproducible on current main, but this PR changes `/status` onto a duplicate local-status path, still has blocking correctness findings, and has no real web UI behavior proof before merge.

## What This Changes

The PR marks `/status` as a local web UI slash command, adds a local status executor plus session-default fields, and registers slash-command regression tests.

## Best Possible Solution

Preserve canonical `/status` behavior and fix the slash-menu selection/send path so selecting the no-argument status command submits once, with focused regression coverage for the menu flow.

## Reproduction Assessment

Yes, by source inspection. Current main builds `/status` into the menu but does not mark it local, and the menu sends immediately only for no-arg commands with `executeLocal`.

## Solution Assessment

No. The PR fixes the keypress symptom by moving `/status` to a new local executor, but the narrower maintainable fix is to submit the selected no-arg agent command once while preserving the canonical status path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep `/status` on the canonical status path:** `ui/src/ui/chat/slash-command-executor.ts:73-74`
  - body: Routing `/status` to this local executor changes more than the Enter behavior: it bypasses `buildStatusText`/`buildStatusReply` and returns a reduced Session Status block instead of the canonical status output with runtime, queue, task, plugin, and auth context. Fix the menu selection path or call a canonical status path rather than rebuilding it in the UI.
  - confidence: 0.88
- **[P2] Compute fast mode for the active session model:** `src/gateway/session-utils.ts:750-756`
  - body: `fastModeDefault` is computed from the globally resolved default model and then reused for whichever session is open. A session using a non-default model with different `agents.defaults.models.*.params.fastMode` can show the opposite fast-mode state from runtime behavior.
  - confidence: 0.91
- **[P2] Honor per-model thinking defaults in local status:** `ui/src/ui/chat/slash-command-executor.ts:308`
  - body: The local `/status` path only receives a global `thinkingDefault` plus catalog data, so it cannot apply `agents.defaults.models.<provider>/<model>.params.thinking` when no session override is stored. That can make `/status` report a lower/default level than runtime uses.
  - confidence: 0.87
