## Summary

Keep open. Current main still has the reported one-Enter Control UI bug, but this PR’s latest head is not merge-ready because it fixes the menu behavior by replacing the shared `/status` path with a partial browser-side formatter that can misreport runtime status and configured defaults.

## What This Changes

The PR marks Control UI `/status` as locally executable, adds a client-side status executor using `sessions.list`/`models.list`, extends session defaults with thinking/fast defaults, adjusts `/think` and `/fast status` plumbing, and registers slash-command regression tests.

## Best Possible Solution

Keep the shared runtime `/status` builder as the source of truth and change the Control UI slash-menu flow so selecting `/status` submits it once through the existing agent/gateway command path, or expose a gateway-backed shared status RPC if the command must remain local. Add focused regression coverage for the menu send path and resolved-default status output.

## Reproduction Assessment

Yes. Current main proves the reproduction path: `/status` is not local, and the menu selection handler only sends immediately for no-arg `executeLocal` commands, so selecting `/status` fills the draft and needs a second Enter.

## Solution Assessment

No. The PR targets the correct user-facing bug, but the local browser formatter is not the narrowest maintainable fix because it forks `/status` semantics and default resolution from the shared runtime path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve shared /status output semantics:** `ui/src/ui/chat/slash-command-executor.ts:296-312`
  - body: After `/status` is marked local, this new formatter replaces the existing status command path with a short session summary. The shared status path includes runtime/execution state, usage/context, task/subagent status, trace lines, and resolved runtime defaults, so Web UI `/status` would lose current user-visible status information. Route the selected command through the existing status path or expose a shared gateway-backed status builder instead.
  - confidence: 0.88
- **[P2] Honor per-model thinking defaults:** `ui/src/ui/chat/slash-command-executor.ts:307-309`
  - body: The local `/status` path can only see the persisted session value, `defaults.thinkingDefault`, and catalog metadata. It cannot apply configured per-model thinking defaults from `agents.defaults.models.<provider>/<model>.params.thinking`, which runtime status and current session rows resolve, so sessions without a stored override can display the wrong thinking level.
  - confidence: 0.84
- **[P2] Resolve fast mode for the active session model:** `src/gateway/session-utils.ts:751-757`
  - body: `fastModeDefault` is computed once from the globally resolved default model and then reused for whichever session is open. A session using a different model or owning-agent default can therefore show the opposite fast-mode state from runtime behavior in `/status` and `/fast status`.
  - confidence: 0.9
