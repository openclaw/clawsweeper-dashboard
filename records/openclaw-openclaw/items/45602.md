## Summary

Keep open: the PR targets a real source-reproducible Control UI bug, but the current branch changes `/status` semantics by duplicating status formatting in the UI and still lacks after-fix real behavior proof.

## What This Changes

The PR makes `/status` execute through the local web UI slash-command path, adds local status formatting/defaults plumbing, and registers unit tests.

## Best Possible Solution

Preserve canonical `/status` output and adjust the slash-menu selection flow so the no-argument status command submits on the first Enter, with focused chat/slash regression coverage.

## Reproduction Assessment

Yes, by source inspection. On current main, `/status` is in the menu but not `executeLocal`, and the selection handler only sends no-arg commands when that flag is true, so first Enter fills the draft and a later Enter sends it.

## Solution Assessment

No. The PR fixes the keypress symptom by moving `/status` into a local UI formatter, but the narrower maintainable fix is to submit the selected command while preserving or invoking the canonical status path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Keep `/status` on the canonical status path:** `ui/src/ui/chat/slash-command-executor.ts:70-73`
  - body: Adding this local `status` case routes the command to a new UI-built summary instead of the canonical `buildStatusText`/`buildStatusReply` path. That drops runtime, usage/auth, queue, task, subagent, plugin, and other status context, so fix the menu selection/send behavior or call the canonical status path instead of rebuilding status in the UI.
  - confidence: 0.88
- **[P2] Compute fast mode for the active session model:** `src/gateway/session-utils.ts:750-757`
  - body: `fastModeDefault` is derived from the globally resolved default model and then reused by `/status` and `/fast status` for whichever session is active. In multi-model setups with different per-model `params.fastMode`, a session using a non-default model can display the opposite of runtime behavior.
  - confidence: 0.86
- **[P2] Honor per-model thinking defaults in local status:** `ui/src/ui/chat/slash-command-executor.ts:304-307`
  - body: The local status path only receives a global `thinkingDefault` plus catalog metadata, so it cannot apply `agents.defaults.models.<provider>/<model>.params.thinking` for the active session. That can make `/status` report the wrong effective thinking level after this routing change.
  - confidence: 0.86
