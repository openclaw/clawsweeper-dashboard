## Summary

Keep open: the underlying one-Enter `/status` bug is still source-reproducible on current main, but this PR is stale and its local status implementation is not merge-ready because it can misreport effective per-model thinking and fast-mode defaults.

## What This Changes

The PR marks `/status` for local Control UI execution, adds a local status executor with tests, and extends session defaults with thinking and fast-mode defaults.

## Best Possible Solution

Rework the fix on current main so selecting `/status` from the slash menu submits with one Enter while preserving the existing effective status semantics, preferably by sharing or forwarding the agent-backed status path instead of duplicating partial defaults in the UI.

## Reproduction Assessment

Yes, by source inspection: current main excludes `status` from `LOCAL_COMMANDS`, derives `executeLocal` from that set, and the slash menu only auto-sends no-arg commands with `executeLocal`. I did not run a live UI session because this was a read-only review.

## Solution Assessment

No. The one-Enter direction is correct, but the PR's local status formatter should not merge until it handles active-session per-model thinking and fast-mode defaults or uses a shared agent-backed status implementation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Resolve thinking defaults from the active session:** `src/gateway/session-utils.ts:750`
  - body: The PR only serializes `cfg.agents.defaults.thinkingDefault` into `sessions.defaults`, then the local `/status` path uses that value plus catalog metadata. Runtime thinking resolution prefers per-model `agents.defaults.models.<provider>/<model>.params.thinking`, so a session with a per-model override and no persisted `thinkingLevel` can display the wrong effective level.
  - confidence: 0.87
- **[P2] Compute fast mode for the active session:** `src/gateway/session-utils.ts:751-758`
  - body: The added `fastModeDefault` is computed from the globally resolved default model and reused for whatever session is open. In a multi-model or per-agent setup, a session using a different active model can show `Fast mode` opposite to runtime behavior when there is no persisted session override.
  - confidence: 0.88
