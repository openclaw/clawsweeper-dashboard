## Summary

Keep open. The underlying one-Enter `/status` bug is still present on current main, but this PR is not merge-ready because it moves web `/status` to a reduced local formatter, has acknowledged default-resolution gaps, is currently conflicting, and lacks contributor real-environment proof.

## What This Changes

The PR marks web UI `/status` as an `executeLocal` slash command, adds browser-side status formatting/default plumbing, and registers focused slash-command tests.

## Best Possible Solution

Fix the slash-menu selection so no-argument `/status` submits with one Enter while preserving or calling the canonical status implementation, with focused regression coverage for that menu-selection path.

## Reproduction Assessment

Yes. By source inspection on current main, `/status` is present in the slash menu but is not local, and the Enter handler only sends no-argument commands when `executeLocal` is true.

## Solution Assessment

No. The PR targets the right bug, but moving `/status` to a partial browser-side formatter is not the narrowest maintainable fix; it should preserve or delegate to the canonical status path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep `/status` on the canonical status path:** `ui/src/ui/chat/slash-command-executor.ts:73-74`
  - body: Adding this local case routes web UI `/status` to `executeStatus` instead of the canonical status implementation. That omits broader status details such as usage, queue, tasks, subagents, auth, uptime, and plugin/session context, so the one-Enter fix should preserve or call the existing status path.
  - confidence: 0.88
- **[P2] Compute fast mode for the active session model:** `src/gateway/session-utils.ts:750-757`
  - body: `fastModeDefault` is derived from the globally resolved default model and then consumed for whichever session is active. In multi-model setups with different per-model `params.fastMode`, a non-default session can display the opposite of runtime behavior in `/status` or `/fast status`.
  - confidence: 0.86
- **[P2] Honor per-model thinking defaults in local status:** `ui/src/ui/chat/slash-command-executor.ts:307-308`
  - body: The local status path only receives a global `thinkingDefault` plus catalog metadata, so it cannot apply per-model configured thinking defaults for the active session. Sessions without a persisted override can therefore show the wrong effective thinking level after this routing change.
  - confidence: 0.84
