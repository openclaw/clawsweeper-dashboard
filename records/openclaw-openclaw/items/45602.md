## Summary

This PR should stay open for now: current main still has the source-level two-Enter `/status` path and this PR is the linked fix candidate for the open bug, but the branch is not correct to merge because it routes `/status` through a reduced local formatter, still has default-resolution gaps, conflicts with main, and lacks required real web UI proof.

## What This Changes

This PR marks `/status` as a local web UI slash command, implements client-side `/status` formatting/default plumbing, and registers slash-command regression tests.

## Best Possible Solution

Preserve the canonical `/status` output path and fix slash-menu selection so selecting the no-argument status command submits once, with focused regression coverage and real web UI proof.

## Reproduction Assessment

Yes. By source inspection on current main, `/status` is present in the menu but not `executeLocal`, and the Enter handler only sends no-arg commands when that flag is true, so the first Enter fills the draft and the next Enter sends it.

## Solution Assessment

No. The PR targets the right user-visible bug, but moving `/status` to a new UI formatter is not the narrowest maintainable fix; preserve or call the canonical status route while fixing the menu selection flow.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Keep `/status` on the canonical status path:** `ui/src/ui/chat/slash-command-executor.ts:73-74`
  - body: Adding this local `status` case routes `/status` to a new UI-built summary instead of the canonical `buildStatusText`/`buildStatusReply` path. That drops runtime, usage/auth, queue, task, subagent, plugin, and other status context, so preserve or call the canonical status path while fixing the one-Enter menu behavior.
  - confidence: 0.88
- **[P2] Compute fast mode for the active session model:** `src/gateway/session-utils.ts:750-757`
  - body: `fastModeDefault` is derived from the globally resolved default model and then reused by `/status` and `/fast status` for whichever session is active. In multi-model setups with different per-model `params.fastMode`, a session using a non-default model can display the opposite of runtime behavior.
  - confidence: 0.86
- **[P2] Honor per-model thinking defaults in local status:** `ui/src/ui/chat/slash-command-executor.ts:307-308`
  - body: The local status path only receives a global `thinkingDefault` plus catalog metadata, so it cannot apply `agents.defaults.models.<provider>/<model>.params.thinking` for the active session. That can make `/status` report the wrong effective thinking level after this routing change.
  - confidence: 0.84
