## Summary

Keep open. Current main still preserves the reported two-Enter Control UI `/status` behavior, and this PR is still a plausible implementation candidate, but the current patch is not safe to merge because it moves `/status` onto a partial local formatter that can lose shared status semantics and misreport configured thinking/fast-mode defaults.

## What This Changes

The PR marks Control UI `/status` as locally executable, adds a local status executor with session/default plumbing, and registers slash-command regression tests.

## Best Possible Solution

Make the Control UI slash-menu selection submit `/status` with one Enter while preserving the shared runtime status semantics. The maintainable path is either to route the selected no-arg command through the existing agent-side status path or expose a gateway-backed status RPC/shared builder that resolves thinking and fast mode for the active session model, then cover the slash-menu flow and resolved-default cases in focused UI/gateway tests.

## Reproduction Assessment

Yes. On current main, `/status` is present in the command registry but is not in `LOCAL_COMMANDS`, and the slash-menu selection handler only sends immediately for no-arg `executeLocal` commands, so selecting `/status` fills the draft and requires a second Enter.

## Solution Assessment

No. The PR targets the right user-facing bug, but duplicating a partial local status formatter is not the best fix because it forks `/status` semantics and misses active-session default resolution; reusing the shared status path or adding a gateway-backed status endpoint would be safer.

Full review comments:

- **[P2] Preserve shared /status output semantics:** `ui/src/ui/chat/slash-command-executor.ts:296-312`
  - body: After `/status` is marked local, this formatter replaces the existing status command path with a short session summary. The shared status path includes runtime/execution state, usage, task/subagent, and resolved default details, so Web UI `/status` would lose current user-visible status information unless this reuses the shared status builder or a gateway RPC.
  - confidence: 0.86
- **[P2] Honor per-model thinking defaults:** `ui/src/ui/chat/slash-command-executor.ts:307-309`
  - body: The new local status path resolves thinking from only the persisted session value, the global defaults object, and catalog metadata. It cannot see per-agent or per-model config defaults that runtime status already honors, so sessions without a stored override can display the wrong thinking level.
  - confidence: 0.84
- **[P2] Resolve fast mode for the active session model:** `src/gateway/session-utils.ts:751-757`
  - body: `fastModeDefault` is computed from the globally resolved default model and then reused for whichever session is open. A session using a different model with different per-model fast-mode config can therefore show the opposite state from runtime behavior in `/status` and `/fast status`.
  - confidence: 0.9
