## Summary

Keep open. The reported Web UI one-Enter bug is still source-reproducible on current main, but this PR should not merge as-is because it moves `/status` to a local browser formatter that diverges from the shared status path and still has default-resolution gaps.

## What This Changes

The PR marks Control UI `/status` as locally executable, adds a browser-side status formatter and session-default fields, updates `/think` and `/fast status` helpers, and registers slash-command tests.

## Best Possible Solution

Preserve the shared runtime `/status` builder as the source of truth, change the Control UI slash-menu flow so selecting `/status` submits it once through the existing agent/gateway command path, and cover that menu-send behavior with focused UI tests.

## Reproduction Assessment

Yes. Source inspection on current main shows `/status` is not local while `selectSlashCommand()` only submits immediately for no-arg local commands, so selecting `/status` fills the draft and requires a second Enter.

## Solution Assessment

No. The PR fixes the selection symptom by making `/status` local, but the narrower maintainable fix is to keep `/status` on the shared agent/gateway status path and only change the menu submission behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve shared /status output semantics:** `ui/src/ui/chat/slash-command-executor.ts:293-316`
  - body: After `/status` is marked local, this formatter replaces the existing shared status command path with a short session summary. The shared path includes runtime/execution state, usage/context, task and subagent status, queue details, auth/runtime labels, and resolved defaults, so Web UI `/status` would lose current user-visible status information. Route the selected command through the existing status path or expose a shared gateway-backed status builder instead.
  - confidence: 0.9
- **[P2] Honor per-model thinking defaults:** `ui/src/ui/chat/slash-command-executor.ts:307-309`
  - body: The local `/status` path can only use the persisted session field, `defaults.thinkingDefault`, and catalog metadata. It cannot apply configured per-model thinking defaults from `agents.defaults.models.<provider>/<model>.params.thinking`, which runtime status and current session rows resolve, so sessions without a stored override can display the wrong thinking level.
  - confidence: 0.84
- **[P2] Resolve fast mode for the active session model:** `src/gateway/session-utils.ts:750-756`
  - body: `fastModeDefault` is computed once from the globally resolved default model and reused for whichever session is open. A session using a different model or owning-agent default can therefore show the opposite fast-mode state from runtime behavior in `/status` and `/fast status`.
  - confidence: 0.9
- **[P3] Add the required changelog entry:** `ui/src/ui/chat/slash-commands.ts:114`
  - body: This is a user-facing Web UI fix, but the PR does not update `CHANGELOG.md`. Add a single-line fix entry for the active version before this lands.
  - confidence: 0.78
