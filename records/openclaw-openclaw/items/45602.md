## Summary

Keep open. Current main still has the source-level two-Enter path for Control UI `/status`, but this PR needs revision because it moves `/status` into a smaller local UI formatter instead of preserving the canonical status path, and it still lacks real web UI proof.

## What This Changes

The PR marks `/status` as `executeLocal`, adds local web UI status execution/defaults plumbing, and registers slash-command regression tests.

## Best Possible Solution

Preserve canonical `/status` output and fix slash-menu selection so selecting the no-argument status command submits once, with focused regression coverage.

## Reproduction Assessment

Yes, by source inspection. On current main, `/status` is listed but not local, and `selectSlashCommand` only sends no-arg commands when `executeLocal` is true, so the first Enter fills the draft and the second sends it.

## Solution Assessment

No. The narrower maintainable fix is to adjust the menu selection/send flow or invoke the canonical status route instead of rebuilding status in the UI.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Keep `/status` on the canonical status path:** `ui/src/ui/chat/slash-command-executor.ts:70-73`
  - body: Adding this local `status` case routes `/status` to a new UI-built summary instead of the canonical `buildStatusText`/`buildStatusReply` path. That drops runtime, usage/auth, queue, task, subagent, plugin, and other status context, so preserve or call the canonical status path while fixing the one-Enter menu behavior.
  - confidence: 0.88
- **[P2] Compute fast mode for the active session model:** `src/gateway/session-utils.ts:750-757`
  - body: `fastModeDefault` is derived from the globally resolved default model and then reused by `/status` and `/fast status` for whichever session is active. In multi-model setups with different per-model `params.fastMode`, a session using a non-default model can display the opposite of runtime behavior.
  - confidence: 0.86
- **[P2] Honor per-model thinking defaults in local status:** `ui/src/ui/chat/slash-command-executor.ts:304-307`
  - body: The local status path only receives a global `thinkingDefault` plus catalog metadata, so it cannot apply `agents.defaults.models.<provider>/<model>.params.thinking` for the active session. That can make `/status` report the wrong effective thinking level after this routing change.
  - confidence: 0.86
