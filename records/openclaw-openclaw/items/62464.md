## Summary

Keep open. Current main still treats MiniMax timeout failures as generic model fallback and can exhaust the configured candidates without a bounded retry after the chain fails.

## What This Changes

The issue asks for retry/backoff when MiniMax timeout failures exhaust the configured fallback chain.

## Best Possible Solution

Add a bounded, replay-safe timeout retry path for configured fallback chains while keeping MiniMax catalog/auth behavior and broad provider SDK retry policy unchanged.

## Reproduction Assessment

Yes for the core behavior with mocked fallback paths: current source disables same-model idle-timeout retry when fallbacks are configured, and existing tests cover timeout-to-fallback but not retry after fallback exhaustion. Live MiniMax outage reproduction is still unverified.

## Solution Assessment

Unclear as written: generic exponential backoff for every timeout could duplicate side effects or stall requests. The safer solution is a narrow retry using the existing idle-timeout/replay-safety gates plus focused regression tests.

Fix https://github.com/openclaw/openclaw/issues/62464. Current main treats MiniMax timeouts as generic model fallback, but configured fallback runs disable the existing same-model idle-timeout retry and `runWithModelFallback` tries each candidate once before throwing `FallbackSummaryError`. Implement a bounded, replay-safe timeout retry for configured fallback chains. Scope the change to the generic fallback/embedded runner path, not MiniMax catalog/auth/model defaults. Reuse or extend the existing idle-timeout safety gates so retries occur only when the failed attempt had no assistant text, tool calls, outbound message sends, deterministic approval prompts, cron additions, or other potential side effects. Add focused regression coverage for timeout exhaustion retry and for side-effectful/partial attempts not being retried. Update `CHANGELOG.md` if the fix is user-facing. Do not introduce broad provider SDK retry policy changes or additional MiniMax fallback models.
