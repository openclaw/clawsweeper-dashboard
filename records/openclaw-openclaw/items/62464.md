The request is for retry/backoff when MiniMax timeout-shaped model failures exhaust configured fallbacks instead of failing the whole request.

## Solution Assessment

Unclear as suggested: generic exponential backoff around whole runs is risky, while an idempotency-gated same-model transient retry policy is the safer solution direction.
