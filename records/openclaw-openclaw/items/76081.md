## What This Changes

The issue reports that Discord DM replies on OpenClaw 2026.4.29 became much slower, with repeated 20-45 second reply delays and longer startup/warmup before the assistant responds.

## Best Possible Solution

Keep the shared runtime/prep cache fixes on main and ship them; any remaining Discord DM delay after that should be reopened with fresh `trace:embedded-run` prep-stage timings from the fixed build.
