## What This Changes

## Best Possible Solution

Revise or split the branch so exec-policy security/UX review is separate from TUI copy/rendering work, render timestamps as TUI chrome rather than mutating Markdown content, preserve configured session defaults, and add focused regression tests plus real CLI/TUI proof.

- **[P1] Render timestamps outside assistant Markdown:** `src/tui/components/assistant-message.ts:13`
  - body: Prepending the timestamp inside the Markdown payload changes parsing for replies that begin with line-start Markdown syntax such as code fences, headings, or lists. Render the timestamp as separate TUI chrome instead of modifying the assistant message text.
  - confidence: 0.93

  - body: Comparing only the parsed rest key lets a `sessions.changed` event for another agent with the same rest key update the active session and route later sends to the wrong agent. Require exact agent-qualified identity for this event path.
  - confidence: 0.91
- **[P2] Preserve configured TUI session defaults:** `src/tui/tui.ts:225-226`
  - body: When `--session` is omitted, generating a project key before `resolveTuiSessionKey` runs prevents configured global or main defaults from applying, silently moving users into a per-directory session.
  - confidence: 0.88
- **[P2] Keep pending user timestamps on updates:** `src/tui/components/user-message.ts:7`
  - body: `UserMessageComponent` prefixes only constructor text, but `ChatLog.addPendingUser` updates existing pending rows through the inherited raw `setText`. Updated pending-user entries lose the timestamp format added by this PR.
  - confidence: 0.84
- **[P3] Format timestamps deterministically:** `src/tui/components/timestamp.ts:2-6`
  - body: `toLocaleTimeString` inherits the host hour cycle, so the intended `HH:MM:SS` prefix can become locale-dependent output with AM/PM. Set an explicit hour cycle or format manually.
  - confidence: 0.78

  - body: The template hardcodes `working` while one default phrase is `working on the request`, producing duplicated wording in the waiting status. Use an elapsed label that cannot collide with phrase text.
  - confidence: 0.72
