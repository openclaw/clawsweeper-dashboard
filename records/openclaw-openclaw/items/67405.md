## Summary

Keep open: current main still lacks the proposed preset, copy, and timestamp changes, but this PR is not merge-ready because the latest head still has TUI correctness blockers and changes exec approval posture.

## What This Changes

The PR adds a hands-free exec-policy preset and docs, neutralizes CLI/TUI copy, prefixes TUI rows with timestamps, and adds TUI session activity/project-scoping behavior.

## Best Possible Solution

Land this only after revision or maintainer review separates the exec-policy approval decision from TUI behavior and fixes timestamp/session handling without changing existing defaults.

## Reproduction Assessment

Yes for the PR findings: the current PR head shows timestamps inserted into Markdown text, rest-only session matching, and generated project sessions before default resolution. No local tests were run because this review was read-only.

## Solution Assessment

No. The intent is useful, but the implementation should render timestamps out-of-band, match session events by full agent-qualified key, preserve configured TUI defaults, and get explicit approval for the no-prompt exec preset.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Render assistant timestamps outside Markdown:** `src/tui/components/assistant-message.ts:13`
  - body: Prepending the timestamp into the assistant Markdown payload changes parsing for replies that begin with Markdown-at-line-start syntax such as code fences, headings, or lists. Render the timestamp as separate TUI chrome instead of modifying the Markdown text.
  - confidence: 0.93
- **[P1] Match session events on agent-qualified keys:** `src/tui/tui-session-actions.ts:146`
  - body: The new event matcher compares only the parsed rest key, so an event for another agent with the same rest key can update the active session and route later sends to the wrong agent. Require exact agent-qualified identity for this path.
  - confidence: 0.9
- **[P2] Preserve configured TUI session defaults:** `src/tui/tui.ts:225-226`
  - body: When no --session is passed, this now forces a generated project key before resolveTuiSessionKey can apply configured global or main defaults. Users with global/default-main workflows would silently land in a per-directory session.
  - confidence: 0.86
- **[P2] Keep timestamps when pending user rows update:** `src/tui/components/user-message.ts:7`
  - body: UserMessageComponent timestamps only the constructor text, but ChatLog.addPendingUser later updates existing pending rows through the inherited raw setText. Those updated pending-user entries lose the timestamp format added by this PR.
  - confidence: 0.84
- **[P3] Format TUI timestamps deterministically:** `src/tui/components/timestamp.ts:2-6`
  - body: toLocaleTimeString inherits the host hour cycle, so the intended HH:MM:SS prefix can become locale-dependent output such as a 12-hour timestamp with AM/PM. Set an explicit hour cycle or format manually.
  - confidence: 0.78
- **[P3] Avoid duplicate waiting-status wording:** `src/tui/tui-waiting.ts:46`
  - body: The template hardcodes 'working' while one default phrase is 'working on the request', producing duplicated wording in the waiting status. Use a neutral elapsed label that cannot collide with phrase text.
  - confidence: 0.72
