## Summary

Keep open. Current main does not implement the PR's main requested behavior, and the current PR head still has unresolved correctness issues in TUI Markdown rendering/session routing plus a security-sensitive no-prompt exec preset that needs maintainer review before merge.

## What This Changes

The PR adds a `hands-free` exec-policy preset and docs, replaces informal CLI/TUI copy with neutral text, prefixes TUI rows with timestamps, and changes TUI session activity/project-scoping behavior with tests.

## Best Possible Solution

Land only after maintainer/security review or contributor revision: split or narrow the exec-policy/docs and copy cleanup from TUI session behavior, render timestamps outside Markdown content, preserve configured session defaults, require exact agent-qualified session matching, and add focused regression coverage.

## Reproduction Assessment

Yes for the PR findings: the provided diff and comments show the changed lines that prepend timestamps into Markdown, generate project sessions when `--session` is omitted, and compare session keys by rest value. No tests were run because this review was read-only.

## Solution Assessment

No. The direction has useful pieces, but the current branch mixes security policy, copy cleanup, timestamp rendering, and session-routing changes, and the implementation is not yet the narrowest maintainable fix.

Full review comments:

- **[P1] Render assistant timestamps outside Markdown:** `src/tui/components/assistant-message.ts:13`
  - body: Prepending `[time]` into the assistant Markdown payload changes parsing for replies that start with code fences, headings, or lists. Render the timestamp as separate TUI chrome so the original Markdown remains unchanged.
  - confidence: 0.93
- **[P1] Match session events by the full agent key:** `src/tui/tui-session-actions.ts:149`
  - body: The new matcher normalizes session keys down to their rest value, so an event for another agent's `main` session can be treated as the current session and switch active state. Require exact agent-qualified identity for event/update paths.
  - confidence: 0.9
- **[P2] Respect configured defaults when no TUI session is passed:** `src/tui/tui.ts:313-315`
  - body: Generating a project session whenever `--session` is omitted prevents the empty-input path from applying `global` scope or the configured agent main key. Keep omitted session input flowing through the existing default resolver.
  - confidence: 0.86
- **[P2] Keep pending user rows timestamped after updates:** `src/tui/components/user-message.ts:7`
  - body: `UserMessageComponent` timestamps only constructor text, but pending rows are later updated through inherited `setText`, which replaces the row with raw text. Override updates or store the timestamp so pending rows keep the new format.
  - confidence: 0.84
- **[P3] Use a deterministic hour cycle for timestamps:** `src/tui/components/timestamp.ts:2-6`
  - body: `toLocaleTimeString` inherits the host locale hour cycle, so the intended `[HH:MM:SS]` prefix can become `02:30:45 PM` and vary by environment. Force a 24-hour format or format the fields directly.
  - confidence: 0.78
