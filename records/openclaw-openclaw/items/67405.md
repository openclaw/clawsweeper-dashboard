## Summary

Keep open. Current main still lacks the requested hands-free preset, TUI timestamping, and neutral-copy changes, while this PR has source-proven correctness issues, missing real behavior proof, and a security-sensitive no-prompt exec preset that needs maintainer approval.

## What This Changes

The branch adds a hands-free exec-policy preset and docs, neutralizes banner/tagline/waiting copy, prefixes TUI rows with timestamps, and adds TUI session activity/project-scoping behavior with tests.

## Best Possible Solution

Revise or split the branch so the exec-policy approval decision is reviewed separately, timestamps render as TUI chrome, session defaults and agent-qualified identity are preserved, and focused tests plus real behavior proof cover the accepted behavior.

## Reproduction Assessment

Yes for the PR review findings: source inspection of the diff shows timestamps inserted into Markdown, rest-only session matching, generated project sessions before default resolution, and pending-user raw setText updates. No local tests were run because this review was read-only.

## Solution Assessment

No. The requested UX direction is plausible, but this implementation should be split or revised before merge because it mixes a no-prompt exec-policy decision with TUI rendering/session changes and introduces avoidable regressions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Render timestamps outside assistant Markdown:** `src/tui/components/assistant-message.ts:13`
  - body: Prepending the timestamp into the assistant Markdown payload changes parsing for replies that begin with line-start Markdown syntax such as code fences, headings, or lists. Render the timestamp as separate TUI chrome instead of modifying the Markdown text.
  - confidence: 0.93
- **[P1] Match session events by full agent key:** `src/tui/tui-session-actions.ts:146`
  - body: Comparing only the parsed rest key lets an event for another agent with the same rest key update the active session and route later sends to the wrong agent. Require exact agent-qualified identity for this event path.
  - confidence: 0.9
- **[P2] Preserve configured TUI session defaults:** `src/tui/tui.ts:225-226`
  - body: When --session is omitted, this generates a project key before resolveTuiSessionKey can apply configured global or main defaults. Users with default-session workflows would silently land in a per-directory session.
  - confidence: 0.86
- **[P2] Keep pending user timestamps on updates:** `src/tui/components/user-message.ts:7`
  - body: UserMessageComponent prefixes only the constructor text, but ChatLog.addPendingUser updates existing pending rows through the inherited raw setText. Updated pending-user entries lose the timestamp format added by this PR.
  - confidence: 0.84
- **[P3] Format TUI timestamps deterministically:** `src/tui/components/timestamp.ts:2-6`
  - body: toLocaleTimeString inherits the host hour cycle, so the intended HH:MM:SS prefix can become locale-dependent output with AM/PM. Set an explicit hour cycle or format manually.
  - confidence: 0.78
- **[P3] Avoid duplicate waiting-status wording:** `src/tui/tui-waiting.ts:46`
  - body: The template hardcodes working while one default phrase is working on the request, producing duplicated wording in the waiting status. Use an elapsed label that cannot collide with phrase text.
  - confidence: 0.72
