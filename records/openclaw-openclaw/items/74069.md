## What This Changes

The issue reports that an npm-global OpenClaw 2026.4.26 install fails to downgrade to 2026.4.25 via `openclaw update --tag 2026.4.25`, ending with `global install verify` and the version unchanged.

## Best Possible Solution

Fix the npm global downgrade path so an explicit older target installs into the staged prefix, verifies as that exact version, swaps into the live global root, and preserves doctor, plugin sync, completion refresh, and restart behavior.

Yes for triage: the issue provides a concrete Ubuntu npm-global command and failure output, and current main still contains the matching `global install verify` failure path. Final validation still needs a live npm-global downgrade proof for the reported release pair.
