## What This Changes

Allow an npm-installed OpenClaw 2026.4.26 install to downgrade to 2026.4.25 via `openclaw update --tag 2026.4.25` instead of failing `global install verify`.

## Best Possible Solution

Fix the npm global downgrade path so an explicit older target installs into the staged prefix, verifies as that exact version, swaps into the live global root, and preserves post-update doctor, plugin sync, completion refresh, and restart behavior. Add focused regression coverage for the 2026.4.26-to-2026.4.25 case plus Linux npm-global package proof and a user-facing changelog entry.
