## What This Changes

The issue reports that `openclaw update --tag 2026.4.25` fails to downgrade an npm global install from 2026.4.26 because verification still sees 2026.4.26.

## Best Possible Solution

Fix the npm global downgrade path so an explicit older target installs into the staged prefix, verifies as that exact version, swaps into the live global root, and preserves doctor, plugin sync, completion refresh, and restart behavior with regression coverage for the reported version pair.
