## Summary

Keep open: the Codex rate-limit display bug is real, but this PR is not mergeable as-is because it scales already-percent values by 100, leaves Codex command tests stale, and bundles unrelated SDK, Telegram, and changelog regressions.

## What This Changes

The branch replaces Codex rate-limit array summarization with a new formatter and also edits Telegram native-command behavior/tests, the session-store SDK runtime export, and CHANGELOG.md.

## Best Possible Solution

Land a narrow Codex-only fix that parses GetAccountRateLimitsResponse, uses usedPercent as a 0-100 value, updates Codex command tests and one changelog line, and leaves Telegram and SDK behavior unchanged.

## Reproduction Assessment

Yes for source-level reproduction: current main passes the object-shaped GetAccountRateLimitsResponse into summarizeArrayLike, which cannot extract primary or secondary RateLimitWindow data and falls back to the old count/none path. I did not run executable tests in this read-only review.

## Solution Assessment

No. Formatting the Codex limits is the right direction, but this branch is not the best fix until the percent scale is corrected, Codex tests are updated, and unrelated Telegram, SDK, and changelog changes are removed.
