Keep open. Current main still lacks the requested doctor detector/docs, and this PR remains the active closing implementation, but it needs a small repair-mode fix plus contributor real behavior proof before merge.

## What This Changes

The PR adds a doctor health contribution that detects stale absolute agent workspace paths, proposes `~`-relative rewrites, registers it in doctor, and updates tests, migration docs, and changelog.

## Best Possible Solution

Land a rebased doctor-owned detector that repairs stale home-shaped agent workspace paths in repair mode, documents the migration pitfall, keeps focused detector and repair-mode tests, and is backed by redacted real behavior proof.

## Reproduction Assessment

Yes. Configure `agents.defaults.workspace` or an agent `workspace` with a stale `/home/<user>/.openclaw/workspace` value on a macOS host; current main resolves that value directly and passes it to `fs.mkdir`. I did not run the live mutation path because this was a read-only review, but the source path and linked logs are high-confidence.

## Solution Assessment

No, not yet. A doctor-layer repair plus migration-guide warning is the right maintainable direction, but this PR should honor explicit headless `--fix` repair mode and still needs after-fix real behavior proof.
