## Summary

Keep open: current main and v2026.6.9 still preserve configured default fallbacks for cron payload model overrides, so the reported pinned-model bypass is not solved; the empty-compaction subcase is partly addressed by a current no-real-conversation guard, and the fence side overlaps existing canonical trackers.

## What This Changes

The issue reports an isolated cron run where a pinned MiniMax model fell back to Ollama after provider overload, then empty-session compaction wrote to the transcript and tripped the embedded session takeover fence.

## Best Possible Solution

Decide the cron payload model contract explicitly, keep empty-compaction suppression, and route the remaining session-fence false-takeover work through the existing same-lane and storage-migration trackers.

## Reproduction Assessment

Yes for source-level fallback behavior: current cron tests and fallback policy show payload model overrides can still inherit default fallbacks. No fresh live MiniMax overload reproduction was run.

## Solution Assessment

Unclear: making payload models strict is a narrow code change but reverses tested current behavior, so the best fix needs a maintainer decision before implementation.
