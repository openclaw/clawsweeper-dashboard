## Summary

Keep open: the `/status` UX change is focused and proof-backed, but the patch should not merge until its new elapsed-runtime rows preserve accumulated subagent runtime for restarted or continued runs.

## What This Changes

This PR adds multiline `/status` subagent detail rows with labels and elapsed runtime, updates focused status tests, and adds a changelog entry.

## Best Possible Solution

Keep the localized `/status` UX if maintainers want it, but compute elapsed runtime through the existing subagent runtime contract and cover restarted or continued subagent records.

## Reproduction Assessment

Yes at source level: current main only emits a count in non-verbose `/status`, and the proposed duration bug follows from a valid run record with `accumulatedRuntimeMs` plus a fresh `startedAt`. Existing steer-restart tests establish that record shape.

## Solution Assessment

No as-is. The formatter is the right narrow surface, but it should reuse `getSubagentSessionRuntimeMs` or the same contract instead of manually subtracting `startedAt` from `now`.
