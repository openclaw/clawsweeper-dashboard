## Summary

Keep open: current main has partial edit recovery and mismatch diagnostics, but it does not implement whitespace-tolerant matching or forced fallback after repeated edit failures, and those remaining behaviors need maintainer product judgment.

## What This Changes

Request to make the `edit` tool tolerate near-miss whitespace mismatches or steer repeated failures toward `apply_patch` or shell fallback, with mismatch context hints.

## Best Possible Solution

Keep the existing exact edit semantics until maintainers choose a safe policy: likely improve mismatch diagnostics and loop guidance first, land the adjacent no-op classifier separately, and make any fuzzy matching opt-in or heavily constrained.
