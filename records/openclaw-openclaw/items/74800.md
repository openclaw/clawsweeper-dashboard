## Summary

Keep open: current `main` still has the unsafe raw plugin-command result path, and PR #74989 is an open closing implementation candidate that should land or be replaced before this issue is closed.

## What This Changes

The issue reports that Telegram plugin slash commands can throw when a plugin command handler returns no result because callers read `continueAgent` from `undefined`.

## Best Possible Solution

Land or replace PR #74989 with a narrow fix that normalizes nullish/non-object plugin command returns at the executor boundary, keeps defensive caller guards, adds regression tests, and includes the required changelog entry.

## Reproduction Assessment

Yes. A plugin command handler that completes without returning a payload can flow through `executePluginCommand` unchanged and then hit `result.continueAgent` in `handlePluginCommand`; the Telegram native path has the same raw-result consumption.

## Solution Assessment

Yes. The linked PR direction is the narrow maintainable fix because executor-boundary normalization protects all plugin command callers while preserving existing `continueAgent` behavior.
