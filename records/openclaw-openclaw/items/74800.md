## Summary

Keep open: current main and the latest release still return raw plugin command handler results and dereference result fields, while #74989 remains the open closing PR for the focused fix.

## What This Changes

Plugin command handlers that return `undefined` can cause Telegram command dispatch to throw instead of treating the command as a no-op reply or producing a clear handler error.

## Best Possible Solution

Land or close #74989; the final fix should normalize nullish/non-object command returns at `executePluginCommand`, keep defensive caller guards, and add focused regression tests.

## Reproduction Assessment

Yes, source-reproducible: a plugin command handler that returns `undefined` flows through `executePluginCommand` unchanged and is then dereferenced by the generic and Telegram native command paths. I did not run a live Telegram command in this read-only sweep.

## Solution Assessment

Yes. The open PR direction is the narrow maintainable fix because it protects the shared executor boundary while preserving existing `continueAgent` behavior and adding caller defenses.
