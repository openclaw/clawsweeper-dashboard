## Summary

Keep open. Current main still has the reported undefined-result crash path, and the issue is paired with the open, unmerged closing PR #74989, so cleanup should wait for that PR to land or be closed. The reporter's diagnosis matches the current executor and Telegram command plumbing.

## What This Changes

The issue requests that plugin/custom command handlers returning undefined or no reply payload stop crashing Telegram command dispatch and instead be treated as a no-op or clear plugin-handler error.

## Best Possible Solution

Review and land #74989, or an equivalent narrow fix, that normalizes nullish/non-object plugin command returns at executePluginCommand(), preserves continueAgent behavior, guards the generic and Telegram native consumers, and adds focused regression tests.

## Reproduction Assessment

Yes. A registered plugin command whose handler completes without returning a payload follows the current source path from executePluginCommand() to unguarded result.continueAgent or Telegram native result.text reads, which is enough to reproduce the TypeError without the reporter's private plugin.

## Solution Assessment

Yes. Normalizing at the shared plugin command executor boundary is the narrowest maintainable fix, with caller-side normalization in the generic and Telegram native paths as useful defense-in-depth.
