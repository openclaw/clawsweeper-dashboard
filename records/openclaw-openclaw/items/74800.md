## Summary

Keep open: current main still returns raw plugin command handler results and dereferences them without nullish guards, while #74989 is an open closing PR rather than merged or shipped code.

## What This Changes

Guard Telegram/plugin command dispatch so handlers that return undefined or no reply payload do not surface Telegram’s generic failure.

## Best Possible Solution

Review and land or close #74989; the final fix should normalize nullish or non-object command returns at `executePluginCommand`, keep caller defenses, and add focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible: a plugin command handler that returns `undefined` flows through `executePluginCommand` unchanged and is then dereferenced by the generic and Telegram native command paths. I did not run a live Telegram command in this read-only review.

## Solution Assessment

Yes. Normalizing at the shared executor boundary with defensive caller guards is the narrow maintainable fix, and #74989 already targets that path.
