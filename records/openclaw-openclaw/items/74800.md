## Summary

Keep open: current main and release v2026.5.2 still return raw plugin command handler results and dereference `result` fields, while #74989 remains the open closing PR for the focused fix.

## What This Changes

Normalize undefined or invalid plugin command handler results so Telegram custom commands do not crash into a generic error reply.

## Best Possible Solution

Land one canonical fix that normalizes plugin command results at the executor boundary, keeps defensive generic and Telegram caller tests, and preserves the existing `continueAgent` contract.

## Reproduction Assessment

Yes. A plugin command handler that returns `undefined` flows through `executePluginCommand` unchanged and is then dereferenced by generic and Telegram native command paths on current main.

## Solution Assessment

Yes. Normalizing invalid command returns at the shared executor boundary with defensive caller guards is the narrow maintainable solution, and #74989 already follows that direction.
