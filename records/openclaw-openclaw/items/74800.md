## Summary

Keep open: current main and the latest release still let an undefined plugin command handler result escape and dereference it in the shared and Telegram command paths. PR #74989 is the focused open closing candidate, but it has not merged.

## What This Changes

Treat undefined or non-object plugin command handler returns as no-op replies instead of crashing Telegram command dispatch.

## Best Possible Solution

Land PR #74989, or an equivalent narrow fix, that normalizes nullish and non-object plugin command returns at the shared executor boundary, keeps defensive caller guards, adds regression tests, and includes the required changelog entry.

## Reproduction Assessment

Yes. A plugin command handler that completes without returning a payload follows the current source path from `executePluginCommand` to `result.continueAgent`, and Telegram dispatch converts the resulting exception into the reported generic error text.

## Solution Assessment

Yes. Normalizing at the shared executor boundary with defensive guards in the shared and Telegram native consumers is the narrow maintainable fix because it protects all plugin command callers while preserving `continueAgent` behavior.
