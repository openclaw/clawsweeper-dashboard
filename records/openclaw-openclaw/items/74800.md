## Summary

Keep open: current main still allows a plugin command handler to return undefined and then dereferences that result in the generic command path, and open PR #74989 is the unmerged closing implementation candidate.

## What This Changes

The issue asks OpenClaw to treat undefined or no-payload plugin command handler returns as no-op replies instead of failing Telegram dispatch with a generic error.

## Best Possible Solution

Land PR #74989, or an equivalent narrow fix, that normalizes nullish and non-object plugin command returns at executePluginCommand, keeps caller-side guards, adds regression tests, and includes the required changelog entry.

## Reproduction Assessment

Yes. A plugin command handler that completes without returning a payload follows the current path from executePluginCommand to result.continueAgent in handlePluginCommand; the Telegram native path has the same unsafe raw-result consumption.

## Solution Assessment

Yes. Normalizing at the shared executor boundary, with defensive guards in the generic and Telegram native consumers, is the narrow maintainable fix because it protects all plugin command callers while preserving continueAgent behavior.
