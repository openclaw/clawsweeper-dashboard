## Summary

Keep open: current main still lets an undefined plugin command handler result flow into unchecked property reads, and the linked closing PR #74989 is open and unmerged.

## What This Changes

The issue requests normalization of undefined plugin command handler results so Telegram plugin commands do not fail with a generic dispatch error.

## Best Possible Solution

Land or close #74989; the final fix should normalize nullish/non-object returns at the executor boundary, keep defensive caller guards, and include regression tests plus any required changelog entry.

## Reproduction Assessment

Yes. A plugin command handler that completes without returning a payload can flow through `executePluginCommand` unchanged and then hit `result.continueAgent` in the generic path or `result.text`/Telegram channel-data reads in the native Telegram path.

## Solution Assessment

Yes. The linked PR direction is the narrow maintainable fix because it protects all plugin command callers at the shared executor boundary while preserving existing `continueAgent` behavior.
