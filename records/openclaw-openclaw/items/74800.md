## Summary

Keep open: current main and the latest release still let an undefined plugin command handler result reach raw result consumers, and open PR #74989 is the focused unmerged implementation candidate.

## What This Changes

Guard Telegram/plugin command handling so command handlers that return undefined do not crash and surface a generic Telegram failure.

## Best Possible Solution

Land PR #74989, or an equivalent narrow fix, that normalizes nullish/non-object command returns at the shared executor boundary and keeps defensive caller-side handling with regression tests.

## Reproduction Assessment

Yes. A plugin command handler that completes without returning a payload follows the current path from executePluginCommand to result.continueAgent in handlePluginCommand, and the Telegram native path has the same unsafe raw-result consumption.

## Solution Assessment

Yes. Normalizing at executePluginCommand with defensive guards in the generic and Telegram native consumers is the narrowest maintainable fix because it protects all plugin command callers while preserving continueAgent behavior.
