## Summary

Keep open: current main and the latest release still return raw plugin command handler results and dereference them without a nullish guard, while #74989 is an open closing PR rather than merged or shipped code.

## What This Changes

Telegram/plugin command handlers that return undefined should be treated as no-op replies or clear handler errors instead of causing generic Telegram dispatch failures.

## Best Possible Solution

Review and land or replace #74989; the final fix should normalize nullish or non-object command returns at `executePluginCommand`, keep caller defenses, and add focused regression coverage.

## Reproduction Assessment

Yes. A plugin command handler that returns undefined flows through `executePluginCommand` unchanged and reaches direct `result.continueAgent` and Telegram reply-field reads; I did not run a live Telegram command.

## Solution Assessment

Yes. Normalizing at the shared executor boundary with defensive caller guards is the narrow maintainable fix, and #74989 already targets that path.
