Overall confidence: 0.88

Full review comments:

- **[P1] Route voice generation through the existing TTS surface:** `extensions/telegram/src/speakeasy-voice.ts:386-388`
  - body: `generateSpeakeasyVoiceNote` runs a workspace script from the Telegram callback path, even though OpenClaw already has `messages.tts` providers, credential handling, status, output-format rules, and channel voice-note delivery. This creates a second TTS contract with different security/setup behavior; use the existing TTS path or get an explicit documented maintainer-approved contract before merging.
  - confidence: 0.88
- **[P2] Preserve existing reply buttons before adding Speakeasy:** `extensions/telegram/src/bot/delivery.replies.ts:851-860`
  - body: This call adds explicit Telegram buttons before the delivery path resolves portable `presentation` or `interactive` actions. Because `resolveTelegramInlineButtons` prefers explicit `buttons`, eligible replies that already had action/select buttons will be delivered with only the voice-note row.
  - confidence: 0.93
- **[P2] Do not let cache locks block text delivery:** `extensions/telegram/src/speakeasy-voice.ts:355-359`
  - body: `withSpeakeasyVoiceButton` creates the cache entry synchronously before the Telegram text is sent, and lock acquisition errors bubble out of this helper. A stale lock or slow writer can therefore prevent a normal DM reply from being delivered just because the optional voice button could not be cached.
  - confidence: 0.84
- **[P2] Roll back failed generation quota reservations:** `extensions/telegram/src/bot-handlers.runtime.ts:2199-2202`
  - body: The callback path records a daily generation before the TTS script succeeds. If the script is missing, misconfigured, or returns a bad path, repeated failed taps can exhaust the daily cap even though no voice note was produced.
  - confidence: 0.82
- **[P2] Use the canonical agent workspace resolver:** `extensions/telegram/src/speakeasy-voice.ts:82-89`
  - body: This new resolver duplicates workspace discovery and probes `OPENCLAW_WORKSPACE` instead of the documented `OPENCLAW_WORKSPACE_DIR`/profile-aware default used by `resolveAgentWorkspaceDir`. Service/profile installs can miss a valid Speakeasy config unless they add a second Speakeasy-specific override.
  - confidence: 0.76
- **[P3] Use generic expired-callback wording:** `extensions/telegram/src/bot-handlers.runtime.ts:2186-2189`
  - body: These user-visible Telegram toasts hard-code “Nox”, so OpenClaw deployments with a different bot or assistant name will give incorrect recovery instructions when a Speakeasy button expires.
  - confidence: 0.86
