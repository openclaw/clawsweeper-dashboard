## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Move TTS execution out of the writable workspace:** `extensions/telegram/src/speakeasy-voice.ts:458`
  - body: `resolveSpeakeasyWorkspaceDir` can select the agent workspace, then this line executes `scripts/tts_elevenlabs_v2.py` from that same workspace when a Telegram user taps the button. If an agent or tool can write there, the tap becomes gateway-side code execution; use the trusted TTS/provider path or an explicit trusted command location.
  - confidence: 0.91
- **[P1] Write Speakeasy cache files with private permissions:** `extensions/telegram/src/speakeasy-voice.ts:153`
  - body: The cache stores exact assistant reply text for callback replay, but this write uses the process default mode. On a typical shared host that can create world-readable `state/speakeasy-cache.json`; write with private permissions like the Telegram ingress spool.
  - confidence: 0.88
- **[P1] Handle stdin write errors from the TTS child process:** `extensions/telegram/src/speakeasy-voice.ts:536`
  - body: If the Python wrapper exits before consuming stdin, `child.stdin.end(params.text)` can emit an `EPIPE` error on the stdin stream. There is no stdin error listener here, so a failed TTS script can crash the gateway instead of rejecting this voice generation.
  - confidence: 0.82
- **[P2] Preserve script-directory imports in the wrapper:** `extensions/telegram/src/speakeasy-voice.ts:34`
  - body: Running Python with `-c` leaves the script directory out of `sys.path[0]`, unlike executing `scripts/tts_elevenlabs_v2.py` directly. Sidecar scripts that import sibling helpers under `scripts/` will fail unless the wrapper inserts the script directory before `runpy.run_path`.
  - confidence: 0.78
- **[P2] Recover stale Speakeasy cache locks:** `extensions/telegram/src/speakeasy-voice.ts:167`
  - body: The lock is a plain sidecar opened with `wx` and no stale-owner or mtime reclaim. A gateway crash after acquiring it can leave voice buttons skipped or callbacks failing until the lock file is removed manually.
  - confidence: 0.77
