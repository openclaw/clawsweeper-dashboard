## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Do not run workspace TTS scripts from Telegram taps:** `extensions/telegram/src/speakeasy-voice.ts:522`
  - body: This spawns Python for scripts/tts_elevenlabs_v2.py under the resolved workspace, and that workspace can come from agents.defaults.workspace, OPENCLAW_WORKSPACE_DIR, or cwd. In a Speakeasy-enabled DM, an agent or plugin that can write workspace files can turn a user tap into gateway-side code execution; use a trusted plugin/provider path or an explicit operator-owned executable outside the mutable workspace.
  - confidence: 0.92
- **[P2] Answer reservation failures before leaving the callback:** `extensions/telegram/src/bot-handlers.runtime.ts:2175-2179`
  - body: reserveSpeakeasyVoiceGeneration can still throw on lock/path errors before any Speakeasy-specific callback answer is sent, and the outer catch only logs non-retryable errors. A contended or unwritable cache leaves the Telegram client spinning instead of showing a retry/disabled message, so wrap the reservation and answer the callback on failure.
  - confidence: 0.84
- **[P2] Chmod existing cache files before writing reply text:** `extensions/telegram/src/speakeasy-voice.ts:157-161`
  - body: writeFileSync with mode 0600 does not change permissions on an existing cache inode, so an existing world-readable speakeasy-cache.json receives fresh assistant reply text before the later chmod runs. Chmod before writing, or write a private temp file and rename it into place.
  - confidence: 0.78
- **[P2] Make stale lock recovery ownership-safe:** `extensions/telegram/src/speakeasy-voice.ts:202-209`
  - body: This removes a stale lock path only by age. If two processes see the old lock, one can remove it and another can create a fresh lock before the second unlink runs, allowing the second process to delete the fresh lock and break the cache/quota critical section; use lock contents or an atomic claim before unlinking.
  - confidence: 0.76

## Security Review

Status: needs_attention

Summary: The diff introduces Telegram-triggered workspace Python execution and private reply-text cache handling that need security attention before merge.

Concerns:

- **[high] Workspace TTS execution from Telegram callbacks:** `extensions/telegram/src/speakeasy-voice.ts:522`
  - body: The callback path can spawn python3 for a script under the resolved workspace, which may be writable by agents or project code and can execute with gateway credentials after a user taps the button.
  - confidence: 0.92
- **[medium] Private reply text can be written before permissions are fixed:** `extensions/telegram/src/speakeasy-voice.ts:157`
  - body: Existing cache files keep their mode during writeFileSync, so private assistant reply text can be written into a readable inode before chmod runs.
  - confidence: 0.78

  - body: mtime-only unlink recovery can race between processes and remove a newly acquired lock, weakening cache/quota serialization for private text and generation counts.
  - confidence: 0.76
