## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Route voice generation through canonical TTS:** `extensions/telegram/src/speakeasy-voice.ts:313`
  - body: This executes `scripts/tts_elevenlabs_v2.py` from the workspace when an authorized Telegram user taps the button. That bypasses the documented `messages.tts` provider config, SecretRef/provider routing, channel TTS overrides, and existing `/tts` surfaces, creating a parallel Telegram-triggered code-execution path. Please route this through the existing TTS runtime/delivery contract or make the owner-specific behavior an explicitly approved plugin path.
  - confidence: 0.9
- **[P2] Reserve the daily generation slot before awaiting TTS:** `extensions/telegram/src/bot-handlers.runtime.ts:2250-2257`
  - body: The cap check runs before `generateSpeakeasyVoiceNote`, but the cache count is only updated after that async generation returns. Concurrent taps can all observe the same count, generate audio in parallel, and then write back a single increment, bypassing the advertised per-chat/day cap. Reserve or serialize the generation slot before awaiting external work and cover the concurrent tap case.
  - confidence: 0.86
- **[P2] Reuse Telegram voice-send fallback handling:** `extensions/telegram/src/bot-handlers.runtime.ts:2267`
  - body: This callback sends the generated file with a direct `bot.api.sendVoice` call, so it skips the existing Telegram voice delivery fallback for recipients that reject voice messages. If Telegram returns `VOICE_MESSAGES_FORBIDDEN`, the tap fails instead of falling back to a useful text/audio path. Reuse the existing voice delivery helper or mirror its failure handling for button-generated voice notes.
  - confidence: 0.84
- **[P2] Drop source-root fingerprints from the memo key:** `src/plugins/plugin-registry-snapshot.ts:153-157`
  - body: The Speakeasy PR also changes plugin registry memoization to stat the global, stock, and workspace source roots on memo-key creation. The plugin registry guidance treats metadata as process-stable and explicitly avoids per-call stat/read/hash freshness for source roots; this unrelated change can alter plugin cache behavior outside the Telegram feature and should be removed or handled in a focused registry PR.
  - confidence: 0.82
