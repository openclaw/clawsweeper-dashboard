  - body: This executes `<workspace>/scripts/tts_elevenlabs_v2.py` whenever an authorized Speakeasy callback is tapped, bypassing documented `messages.tts` provider/SecretRef/status controls and adding a new remote-triggered local execution path.   - confidence: 0.86
- **[P2] Make Speakeasy cache writes non-blocking for text delivery:** `extensions/telegram/src/speakeasy-voice.ts:270`
  - body: If a stale lock file remains or another writer holds the lock for more than one second, `acquireSpeakeasyCacheLock` throws out of `withSpeakeasyVoiceButton` before the Telegram text reply is sent. Voice-button cache failures should skip the optional button instead of failing normal message delivery.
  - confidence: 0.91
- **[P2] Lock callback cache writes before persisting generation counts:** `extensions/telegram/src/bot-handlers.runtime.ts:2184`
  - body: The callback path loads the cache, rewrites it, then later rewrites it again without the lock used for new button entries. A concurrently delivered reply can append a visible callback entry between those reads and writes and then lose it, making the rendered button expire immediately.
  - confidence: 0.87
- **[P2] Merge existing reply buttons before adding Speakeasy:** `extensions/telegram/src/bot/delivery.replies.ts:851`
  - body: In reply delivery, `withSpeakeasyVoiceButton` runs before portable `interactive` or `presentation` controls are converted into Telegram buttons, so the helper creates explicit buttons containing only the voice row. Because explicit Telegram buttons take precedence, existing action buttons disappear for eligible Speakeasy DMs.
  - confidence: 0.9
- **[P2] Use the canonical OpenClaw workspace:** `extensions/telegram/src/speakeasy-voice.ts:83-89`
  - body: This candidate list checks a new Speakeasy env var and `OPENCLAW_WORKSPACE`, but documented default workspaces use `OPENCLAW_WORKSPACE_DIR`, `OPENCLAW_HOME`, and `OPENCLAW_PROFILE`. Service/profile installs with `config/speakeasy-chats.json` in the real default workspace will be treated as disabled unless operators add a second override.
  - confidence: 0.88
- **[P3] Avoid the hard-coded Nox recovery text:** `extensions/telegram/src/bot-handlers.runtime.ts:2189`
  - body: Expired or missing callbacks tell users to “Ask Nox” even when the Telegram bot or assistant is not named Nox, so non-Nox installs get misleading recovery text. Use a generic assistant label or derive the configured name.
  - confidence: 0.82
