## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Do not execute TTS scripts from the agent workspace:** `extensions/telegram/src/speakeasy-voice.ts:386`
  - body: `resolveSpeakeasyWorkspaceDir` can select `agents.defaults.workspace`, then this path executes `scripts/tts_elevenlabs_v2.py` from that writable workspace. In a Speakeasy-enabled DM, an agent or workspace writer could replace that script and turn a button tap into gateway-side code execution; use the existing TTS provider path or a trusted plugin-owned/configured executable outside the agent workspace.
  - confidence: 0.93
- **[P1] Keep cache failures from blocking text sends:** `extensions/telegram/src/speakeasy-voice.ts:355-359`
  - body: `withSpeakeasyVoiceButton` creates and persists the cache entry before the normal Telegram text send. If the optional cache lock or write fails, the helper throws and the user loses the original reply; catch cache failures and return the unmodified reply without the voice button.
  - confidence: 0.9
- **[P2] Preserve existing presentation buttons first:** `extensions/telegram/src/bot/delivery.replies.ts:851-860`
  - body: This delivery path adds Speakeasy before resolving `presentation` and legacy `interactive` actions. Because explicit Telegram `buttons` take precedence later, an enabled DM reply that already had action/select controls can be delivered with only the voice-note row.
  - confidence: 0.88
- **[P2] Honor the canonical workspace environment variable:** `extensions/telegram/src/speakeasy-voice.ts:83-86`
  - body: OpenClaw's workspace resolver and docs use `OPENCLAW_WORKSPACE_DIR` plus profile handling, but this candidate list checks unsupported `OPENCLAW_WORKSPACE`. Container or profile installs can miss `config/speakeasy-chats.json` and the TTS script unless operators add a second, undocumented env var.
  - confidence: 0.87
- **[P2] Resolve relative TTS outputs against the workspace:** `extensions/telegram/src/speakeasy-voice.ts:390-399`
  - body: The script runs with `cwd: workspaceDir`, so relative paths it prints are workspace-relative. Returning the raw string means `InputFile` later resolves paths from the gateway cwd, making valid outputs such as `state/speakeasy/out.ogg` fail to send.
  - confidence: 0.86
- **[P2] Bound the Speakeasy TTS child process:** `extensions/telegram/src/speakeasy-voice.ts:386-389`
  - body: This `execFile` call has no timeout or abort signal. A hung provider script can leave the callback handler and a child process stuck per tap; reuse the existing TTS timeout contract or set a finite timeout for this execution path.
  - confidence: 0.84
- **[P2] Serialize callback quota updates:** `extensions/telegram/src/bot-handlers.runtime.ts:2177-2200`
  - body: The callback path loads and writes the shared Speakeasy cache without the lock used by cache-entry creation. Concurrent taps can both pass the same stale daily count and can also overwrite entries appended while TTS generation is running.
  - confidence: 0.82
- **[P3] Use generic expired-callback wording:** `extensions/telegram/src/bot-handlers.runtime.ts:2186-2189`
  - body: Expired or missing Speakeasy callbacks tell every deployment to ask `Nox` to resend, even when the bot or agent is named something else. Use generic assistant wording or derive the configured identity so Telegram users get accurate recovery text.
  - confidence: 0.8
