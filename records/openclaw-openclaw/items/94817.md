## Summary

Keep open, but this PR is not merge-ready: the linked Ollama `/think` menu bug is real, yet the patch treats missing Ollama reasoning metadata as support instead of using the live capability-derived fact, and the contributor proof is unit-test only.

## What This Changes

The PR changes Ollama's provider-policy `resolveThinkingProfile` so `reasoning: undefined` returns the full Ollama thinking profile and adds unit coverage for empty/undefined reasoning.

## Best Possible Solution

Route `/think` choices through runtime or capability-derived Ollama `reasoning` metadata for the selected model, keep explicit `reasoning: false` off-only, and close narrower fallback branches after one canonical fix lands.

## Reproduction Assessment

Yes from source, though I did not run a live Telegram/Ollama setup in this read-only review. Current main can call the Ollama profile resolver with `reasoning: undefined`, which returns off-only, while Ollama discovery maps `/api/show` `thinking` capability to `reasoning: true` when that metadata is available.

## Solution Assessment

No. The proposed condition is a plausible mitigation for the reported model, but the best fix is to pass or consume live capability-derived metadata rather than treating every absent Ollama reasoning fact as support.
