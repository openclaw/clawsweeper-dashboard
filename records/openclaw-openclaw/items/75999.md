Keep open. ## What This Changes

The issue reports a v2026.4.29 regression where embedded agent dispatch prep spends tens of seconds of synchronous CPU before streaming, especially in `core-plugin-tools`, `system-prompt`, and `stream-setup`.

## Best Possible Solution

Amortize stable embedded-run preparation inputs behind explicit invalidation keys while keeping volatile channel, session, provider, model, and prompt-context state dynamic.

## Reproduction Assessment

Yes. The issue provides concrete v2026.4.29 versus v2026.4.27 steps, real stage timings, and independent corroborating logs; current main still contains the same named stage boundaries and hot-path calls, though this review did not run the live gateway scenario.

## Solution Assessment

Unclear. Caching stable prompt/tool/provider inputs is plausible, but the implementation should first measure current main after the latest plugin cache commits and avoid carrying dynamic runtime state across turns.
