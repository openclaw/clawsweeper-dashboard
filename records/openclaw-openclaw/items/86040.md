Add a Control UI-only timestamp fallback for unlabeled dashboard sessions, using browser-local time while preserving explicit labels, display names, and future generated-title behavior.

## Solution Assessment

Yes, if implemented as a display fallback rather than a persisted overriding label. That keeps the timestamp useful for fresh tabs without conflicting with manual labels or future generated titles.
