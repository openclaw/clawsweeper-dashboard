## Summary

Keep open: current main still has the reported reminder-guard bug and this PR is a plausible focused fix, but the latest head can over-suppress warnings when the session key is absent or malformed, and the external PR still lacks real behavior proof.

## What This Changes

The branch preserves `successfulCronAdds` across compaction retries, expands the reminder guard to match enabled isolated same-agent cron jobs, adds guard tests, and narrows a doctor e2e fixture `git add` command.

## Best Possible Solution

Land this PR or a narrow replacement after deriving `agentId` only from a canonical parsed agent session key, keeping the compaction-counter and isolated-cron fixes with regression coverage and real behavior proof.

## Reproduction Assessment

Yes, at source level. Current main resets `successfulCronAdds` during compaction and only matches existing cron jobs by `sessionKey`, while the PR head regression follows from defaulting a missing or malformed session key to agent `main`.

## Solution Assessment

No, not as currently proposed. The fix direction is narrow, but the runner should parse the session key first and pass an agent id only when the current session key is canonical and agent-scoped.
