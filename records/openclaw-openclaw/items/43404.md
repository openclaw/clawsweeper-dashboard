## Summary

Keep open: current main still has the reported reminder-guard bug and this PR is a plausible implementation path, but the PR head still has a source-level correctness blocker, lacks after-fix real behavior proof, and is currently dirty against the base branch.

## What This Changes

The PR updates the reminder guard to recognize isolated cron jobs by agent id, preserves the cron-add counter across compaction retries, adds focused reminder-guard tests, and narrows one doctor fixture git command.

## Best Possible Solution

Land an updated version or narrow replacement that preserves cron-add evidence across compaction and matches isolated crons only when same-agent evidence comes from a canonical session key, with focused regression tests and redacted runtime proof.

## Reproduction Assessment

Yes, at source level. Current main resets `successfulCronAdds` during compaction and only matches existing cron rows by `sessionKey`, so an enabled isolated cron row cannot suppress the note after the counter is lost.

## Solution Assessment

No, not as currently proposed. The fix direction is narrow, but the caller should pass `agentId` only when the session key parses as canonical so missing or malformed sessions remain fail-closed.
