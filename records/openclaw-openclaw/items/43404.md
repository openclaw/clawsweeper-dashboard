## Summary

Keep open: this is a real bug-fix PR for a still-broken current-main path, but the branch still has a P2 fail-closed regression from deriving `agentId` with the defaulting helper and lacks required real behavior proof.

## What This Changes

The PR preserves `successfulCronAdds` across compaction retries, expands reminder-guard matching for enabled isolated same-agent cron jobs, adds guard tests, and changes a doctor e2e fixture `git add` command.

## Best Possible Solution

Land this PR or a narrow replacement after deriving `agentId` only from a canonical parsed agent session key, keeping the compaction-counter and isolated-cron fixes with regression coverage and real behavior proof.

## Reproduction Assessment

Yes, at source level. Current main resets `successfulCronAdds` during compaction and only matches existing cron jobs by `sessionKey`; the PR-head regression is also source-reproducible because missing or malformed session keys resolve to agent `main`.

## Solution Assessment

No, not as currently proposed. The fix direction is narrow, but the runner should parse the session key first and pass an agent id only when the current session key is canonical and agent-scoped.
