## Summary

Keep open. Current main still has both reported reminder-guard gaps, and this PR is the right narrow direction, but the branch still has a P2 regression from defaulting missing or malformed session identity to the main agent before suppressing the guard.

## What This Changes

The PR preserves `successfulCronAdds` across compaction retry, expands the reminder guard to match enabled isolated cron jobs by agent id, adds focused guard tests, and narrows one doctor e2e fixture `git add` command.

## Best Possible Solution

Land this contribution or a narrow replacement after deriving an optional agent id only from a parsed canonical agent session key, preserving the new isolated-cron and compaction-counter coverage, and adding the required changelog entry.

## Reproduction Assessment

Yes. Source inspection of current main shows isolated cron jobs cannot match the reminder guard and compaction clears the same-turn cron-add counter; the remaining PR regression is source-reproducible from a `runReplyAgent` call without a canonical session key plus an enabled isolated `main` cron.

## Solution Assessment

No, not as currently proposed. Matching isolated cron jobs by trusted agent id and preserving `successfulCronAdds` is the right narrow fix, but the runner must omit `agentId` unless the session key parses as an agent-scoped key.
