## Summary

Keep open: current main still has the reported reminder-guard bug, and this PR contains useful narrow fixes, but the latest diff still has a blocking regression where an absent or malformed session key can suppress the guard through the default `main` agent id; it also lacks contributor-provided real behavior proof.

## What This Changes

The PR updates the reminder guard to match isolated cron jobs by agent id, preserves `successfulCronAdds` across compaction retries, adds reminder-guard tests, and tweaks a doctor e2e git fixture command.

## Best Possible Solution

Land this contribution or a narrow replacement after passing an agent id only when the session key parses as a canonical agent session key, while preserving the isolated-cron and compaction-counter fixes with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection of current main shows both reported bug paths: isolated cron jobs cannot match the reminder guard, and compaction retry clears the same-turn cron-add counter; the remaining PR regression is also source-reproducible from the missing-session runner path plus an enabled isolated `main` cron.

## Solution Assessment

No, not as currently proposed. Matching isolated cron jobs by a trusted agent id and preserving `successfulCronAdds` is the narrow maintainable fix, but the runner must omit `agentId` unless the session key is parsed and trusted.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Derive agent id only from canonical session keys:** `src/auto-reply/reply/agent-runner.ts:1449`
  - body: `resolveAgentIdFromSessionKey(sessionKey)` returns `main` when `sessionKey` is missing or malformed. Since `runReplyAgent` can run without a session key and existing coverage expects the guard note to remain in that case, this can match an unrelated enabled isolated `main` cron and incorrectly suppress the warning; parse the session key first and pass `agentId` only for canonical agent session keys.
