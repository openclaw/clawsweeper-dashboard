## Summary

Keep open. Current main still shows the reported reminder-guard gaps, and this PR is useful but not merge-ready because the latest head still references an unimported session-key resolver and derives a default `main` agent id from missing or malformed session keys, which can suppress the guard for unrelated isolated main-agent jobs.

## What This Changes

The branch preserves `successfulCronAdds` across compaction retry, expands the reminder guard to match enabled isolated cron jobs by `agentId`, passes an agent id from the reply runner, and adds focused reminder-guard tests.

## Best Possible Solution

Land this contribution or a narrow replacement after the runner derives an optional agent id only from a trusted parsed agent session key, the resolver reference is imported or replaced with an existing scoped helper, the compaction counter remains preserved, focused runner and helper regressions cover the edge cases, and the required changelog entry is added.

## Reproduction Assessment

Yes. Current main has a source-level reproduction for the original report: isolated cron jobs cannot match the guard, and compaction clears `successfulCronAdds`; the PR regression is reproducible from the diff because `runReplyAgent` can be called without `sessionKey` while `resolveAgentIdFromSessionKey(undefined)` returns `main`.

## Solution Assessment

No. Matching isolated cron jobs by agent id and preserving the counter is the right narrow direction, but the current PR is not the best mergeable solution until it imports the resolver and avoids passing a defaulted agent id when session identity is missing or malformed.
