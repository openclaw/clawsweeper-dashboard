## Summary

Keep open: this is a useful implementation candidate for a source-reproducible reminder-guard bug, but the PR still has a P2 correctness blocker, lacks required real behavior proof, and is currently conflicting.

## What This Changes

The PR updates the reminder guard to recognize isolated cron jobs by agentId, preserves successfulCronAdds across compaction retries, adds reminder-guard tests, and narrows a doctor E2E fixture git add command.

## Best Possible Solution

Land an updated version or narrow replacement that preserves cron-add evidence across compaction and matches isolated crons only when same-agent evidence comes from a reliable session key, with focused regression tests and redacted runtime proof.

## Reproduction Assessment

Yes, at source level. Current main resets successfulCronAdds during compaction and only matches existing cron rows by sessionKey, so an enabled isolated cron row cannot suppress the note after the counter is lost.

## Solution Assessment

No, not as currently proposed. The fix direction is narrow, but the caller should pass agentId only when the session key parses reliably so missing or malformed sessions remain fail-closed.
