## Summary

Keep open: the central bug is real and current main still has both source gaps, but this PR still has a P2 missing-session regression and lacks required after-fix real behavior proof.

## What This Changes

The PR updates the reminder guard to match enabled isolated same-agent cron jobs, preserves `successfulCronAdds` across compaction retries, adds guard tests, and narrows a doctor e2e fixture git add command.

## Best Possible Solution

Land this contribution or a narrow replacement after passing an agent id only from a canonical parsed agent session key, while preserving the isolated-cron and compaction-counter fixes with focused regression coverage and real behavior proof.

## Reproduction Assessment

Yes. Source inspection of current main shows both reported bug paths, and the remaining PR regression is source-reproducible from unconditional defaulting of a missing session key to the `main` agent id.

## Solution Assessment

No, not as currently proposed. The core fixes are narrow and appropriate, but the runner must pass `agentId` only when the session key parses as a canonical agent session key.
