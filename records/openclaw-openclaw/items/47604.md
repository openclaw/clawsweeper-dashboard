Full review comments:

- **[P1] Exclude Wear auth identity from backups:** `apps/android/wear/src/main/AndroidManifest.xml:10-12`
  - body: The Wear app enables backup/data extraction while the rules include the whole file domain. The shared identity store writes `openclaw/identity/device.json` with `privateKeyPkcs8Base64`, so cloud backup or device transfer can copy the device-auth private key. Disable backup for the Wear app or exclude identity and other sensitive auth files from both backup rule files.
  - confidence: 0.94
- **[P1] Wire Wear into shared Android release gates:** `apps/android/wear/build.gradle.kts:17-18`
  - body: The new Wear module has hard-coded `versionCode` and `versionName`, while current Android version sync, CI, and release artifact scripts only build and version `:app`. A merged Wear module needs the same version source and explicit CI/release tasks, otherwise maintainers can ship phone builds without validating or packaging the companion.
  - confidence: 0.86
- **[P2] Remove the deprecated broad Wear listener action:** `apps/android/app/src/main/AndroidManifest.xml:97`
  - body: This service already has a filtered `MESSAGE_RECEIVED` listener for `/openclaw`, but the manifest also registers deprecated `BIND_LISTENER`, which Google/Android lint flags because it wakes apps for all Wear events instead of the filtered path. Drop the broad action and keep the specific listener registration. ([android-developers.googleblog.com](https://android-developers.googleblog.com/2016/04/deprecation-of-bindlistener.html?utm_source=openai))
