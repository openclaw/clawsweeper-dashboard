- **[P1] Exclude Wear identity files from backup:** `apps/android/wear/src/main/res/xml/data_extraction_rules.xml:4-7`
  - body: The Wear app enables backup and includes the whole file domain for cloud backup/device transfer, while the shared identity store writes `openclaw/identity/device.json` with `privateKeyPkcs8Base64`.   - body: Adding `:wear` creates a second Android application with its own `versionCode` and `versionName`, but the existing release helper still bumps and builds only the phone `:app` bundles.   - confidence: 0.88
- **[P2] Clear bootstrap auth when editing direct credentials:** `apps/android/wear/src/main/java/ai/openclaw/wear/ui/ConnectionSettingsScreen.kt:50-51`
  - body: Changing the direct password preserves any existing `bootstrapToken`, and `buildWearConnectParams` selects bootstrap auth before password when no token is set. A user switching from fallback/bootstrap sync to password auth can keep sending the stale bootstrap token and fail direct auth until config is reset elsewhere.
  - confidence: 0.86
