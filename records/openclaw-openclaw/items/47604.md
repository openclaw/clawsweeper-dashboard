Full review comments:

- **[P1] Preserve private-LAN gateway setup:** `apps/android/app/src/main/java/ai/openclaw/app/ui/GatewayConfigResolver.kt:147-148`
  - body: This switches endpoint validation from `isLocalCleartextGatewayHost` to `isLoopbackGatewayHost`, so Android setups that current main accepts, such as `ws://192.168.x.x:18789`, now fail as insecure. The branch also updates tests to expect that rejection, but current main's tests and user-facing copy keep private-LAN `ws://` supported, so the Wear rebase should preserve that compatibility unless maintainers choose a separate breaking change.
  - confidence: 0.91
- **[P1] Exclude Wear auth identity from backup:** `apps/android/wear/src/main/res/xml/backup_rules.xml:3`
  - body: `WearGatewayClient` creates a `GatewayDeviceIdentityStore`, which writes the Ed25519 private key to `context.filesDir/openclaw/identity/device.json`, and this backup rule includes the whole `file` domain while the Wear manifest enables backup/transfer. A restored or transferred watch can inherit the auth identity used in `connect` signatures, unlike the phone app's current `allowBackup=false` and root exclusions; disable backup or exclude the identity/auth paths.
  - confidence: 0.87
