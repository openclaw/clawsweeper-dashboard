## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Disable backup for Wear auth files:** `apps/android/wear/src/main/AndroidManifest.xml:10-12`
  - body: The Wear app enables backup and includes the full files domain, while the shared identity store writes `openclaw/identity/device.json` with `privateKeyPkcs8Base64` under `context.filesDir`. Android backs up filesDir data unless excluded, so cloud backup or device transfer can copy the watch gateway signing key; disable backup or explicitly exclude the identity/auth paths from both backup rule files.
  - confidence: 0.92
- **[P2] Drop the deprecated BIND_LISTENER action:** `apps/android/app/src/main/AndroidManifest.xml:97`
  - body: This service already has the fine-grained `MESSAGE_RECEIVED` filter scoped to `/openclaw`, but adding `BIND_LISTENER` reintroduces the deprecated broad listener action. Google’s Wear guidance says to migrate away from `BIND_LISTENER`, and the deprecation is lint-fatal, so remove it unless there is a measured compatibility reason.
  - confidence: 0.84
