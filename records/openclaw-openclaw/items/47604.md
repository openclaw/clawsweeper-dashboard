## Summary

Keep open: current main still has no Wear OS app, and this PR remains a large in-tree Android/Wear feature with a concrete security blocker in its backup rules plus maintainer triage signal to keep it active.

## What This Changes

The PR adds a Wear OS companion app, a shared Android gateway module, phone-side Wear proxying, direct watch gateway access, tile/complication services, background reply notifications, tests, and Android docs.

## Best Possible Solution

Keep the PR open for maintainer product/security review; if Wear OS belongs in tree, land staged Android/Wear slices after backup/identity exclusion and fresh device validation.

## Reproduction Assessment

Yes for the blocking review finding: PR head source shows Wear backup enabled with include-all file rules, and the identity store writes the private signing key under the app files directory. Full Wear behavior still needs paired phone/watch validation rather than source-only proof.

## Solution Assessment

No; merging this PR as-is is not the best way to solve the Wear OS feature because the backup/private-key exposure needs to be fixed first. The safer path is staged maintainer review or a replacement branch that preserves the useful work while narrowing security and product review scope.
