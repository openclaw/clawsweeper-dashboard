## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Exclude Wear identity and auth state from backup:** `apps/android/wear/src/main/res/xml/backup_rules.xml:3`
  - body: The Wear app enables backup and the rules include the whole files directory, but the shared identity store writes `openclaw/identity/device.json` with `privateKeyPkcs8Base64`. A cloud restore or device transfer can clone the watch's gateway device identity, so the Wear app should disable backup or explicitly exclude identity/auth/pin paths like the phone app does.
  - confidence: 0.93
- **[P2] Clear bootstrap auth when direct credentials change:** `apps/android/wear/src/main/java/ai/openclaw/wear/ui/ConnectionSettingsScreen.kt:47-52`
  - body: Editing the direct token or password preserves any existing `bootstrapToken`, and `buildWearConnectParams` prefers bootstrap auth before password when no token is set. A user switching to password auth after phone fallback sync can keep sending the stale bootstrap token and fail direct auth until the config is reset elsewhere.
  - confidence: 0.91
- **[P2] Remove the blanket Wear listener binding:** `apps/android/app/src/main/AndroidManifest.xml:97`
  - body: The phone manifest now registers both the path-scoped `MESSAGE_RECEIVED` action and the broad `BIND_LISTENER` action. The service only needs the filtered message listener for `/openclaw` messages; keeping the bind listener can wake the phone app for unrelated Wear Data Layer traffic and undercuts the path-filtered design discussed earlier in the PR.
  - confidence: 0.79
- **[P2] Wire the Wear module into Android CI and release paths:** `apps/android/settings.gradle.kts:20-21`
  - body: Adding `:wear` makes this a new application module, but current Android CI only runs `:app` unit/build tasks and the release helper only archives `:app` Play/third-party artifacts. The Wear app can drift or fail to ship without a build/test/signing decision, so this include needs matching CI and release coverage or an explicit unshipped gate.
  - confidence: 0.88
