## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Exclude Wear identity files from backup:** `apps/android/wear/src/main/res/xml/data_extraction_rules.xml:4`
  - body: `GatewayDeviceIdentityStore` writes the direct-auth private key into `context.filesDir/openclaw/identity/device.json`, but the new Wear data-extraction rules include the entire file domain for cloud backup and device transfer. That can copy watch device private key material into Google backup or a transferred device, so the backup/data-extraction rules need to exclude the identity path before this direct-auth client ships.
  - confidence: 0.86
- **[P1] Wire the Wear module into Android gates:** `apps/android/settings.gradle.kts:20`
  - body: Adding `:wear` makes this a new shippable Android app module, but the current Android CI lane still runs only `:app:testPlayDebugUnitTest`, `:app:testThirdPartyDebugUnitTest`, and `:app:assemblePlayDebug`, and Android CodeQL builds only `:app:assemblePlayDebug`. Green checks would not compile, test, or analyze the Wear app directly, so the PR needs Wear build/test/CodeQL/release coverage before merge.
  - confidence: 0.84
- **[P2] Clear bootstrap auth when password changes:** `apps/android/wear/src/main/java/ai/openclaw/wear/ui/ConnectionSettingsScreen.kt:51`
  - body: The password editor saves only `password`, but `buildWearConnectParams` prefers any stored `bootstrapToken` over `password`. If the watch received a bootstrap token from phone fallback sync and the user later switches to password auth, direct connect keeps sending the stale bootstrap token and the password entry cannot take effect until some other reset clears it.
  - confidence: 0.8
- **[P2] Use shared Android versioning for Wear:** `apps/android/wear/build.gradle.kts:17-18`
  - body: The phone app reads the generated Android version properties, but the new Wear module hard-codes `versionCode = 2026042600` and `versionName = "2026.4.26"` while current Android version metadata is 2026.6.10. Unless Wear uses the same version source and release sync checks, future Android releases can ship stale Wear metadata or fail update ordering independently of the phone app.
  - confidence: 0.78
