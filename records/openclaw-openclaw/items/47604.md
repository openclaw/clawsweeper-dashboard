## Summary

Keep this PR open: current main still has no Wear OS companion, and the timeline has an explicit no-stale maintainer signal, but the current PR head has a blocking Wear backup/private-key exposure before it can merge.

## What This Changes

The PR adds a Wear OS companion module, shared Android gateway helpers, phone-side Wear proxying, direct watch gateway access, tile/complication services, background reply notifications, tests, and Android docs.

## Best Possible Solution

Keep the PR open while maintainers decide whether Wear OS belongs in-tree; if it does, land it in staged Android/Wear slices with backup exclusions, focused tests, docs, and fresh paired-device security proof.

## Reproduction Assessment

Yes for the blocking review finding: PR head source shows Wear backup enabled with include-all file rules, and the shared identity store writes the device-auth private key under app files. Full Wear behavior still needs paired phone/watch validation rather than source-only proof.

## Solution Assessment

No; merging this PR as-is is not the best way to solve the Wear OS feature because the backup/private-key exposure must be fixed first. The safer path is staged maintainer review or a replacement branch that preserves the useful work while narrowing security and product review scope.
