## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Disable backup for Wear auth files:** `apps/android/wear/src/main/AndroidManifest.xml:10-12`
  - body: The Wear app enables backup and includes the file domain while the shared identity store writes `privateKeyPkcs8Base64` under `context.filesDir/openclaw/identity/device.json`. Android backs up `getFilesDir()` data unless excluded, so cloud backup or device transfer can copy the watch gateway signing key; set `allowBackup=false` like the phone app or exclude auth material from both rules. ([developer.android.com](https://developer.android.com/identity/data/autobackup))
  - confidence: 0.92
- **[P2] Drop the deprecated Wear listener binding:** `apps/android/app/src/main/AndroidManifest.xml:97`
  - body: This service already has the fine-grained `MESSAGE_RECEIVED` filter scoped to `/openclaw`, but adding `BIND_LISTENER` reintroduces the deprecated broad listener path. Android Wear guidance says to migrate away from `BIND_LISTENER` and use path-scoped filters to avoid unnecessary service wakeups, so remove it unless there is a measured compatibility reason. ([android-developers.googleblog.com](https://android-developers.googleblog.com/2016/04/deprecation-of-bindlistener.html))
  - confidence: 0.84
