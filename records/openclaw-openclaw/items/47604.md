  - body: The Wear app stores its gateway identity private key under the app files directory, but this manifest enables backup and points to rules that include the full file domain.   - confidence: 0.92

  - body: Current main disables Android phone app backup and excludes root backup/data extraction, but this PR head enables backup and include-all rules for the phone app. Rebasing this branch needs to keep the current hardening so auth-bearing phone app data is not backed up.
  - confidence: 0.9
- **[P2] Clear bootstrap auth when editing direct password:** `apps/android/wear/src/main/java/ai/openclaw/wear/ui/ConnectionSettingsScreen.kt:51`
  - body: The password editor keeps any existing bootstrapToken, but buildWearConnectParams sends bootstrap auth before password when no token is set. After fallback sync, a user-entered password can be ignored until config is reset, so clear bootstrap auth when explicit direct password auth is edited.
  - confidence: 0.84
