Full review comments:

- **[P1] Disable backup for Wear auth material:** `apps/android/wear/src/main/AndroidManifest.xml:10-12`
  - body: The Wear app opts into backup while both backup rule files include the full file domain. Because the shared identity store writes a private signing key under `filesDir`, cloud backup or device transfer can copy gateway identity material; match the phone app's backup hardening or narrowly exclude the identity/auth paths before shipping.
  - confidence: 0.91
- **[P2] Add Wear to the Android release path:** `apps/android/wear/build.gradle.kts:17-18`
  - body: The new `:wear` app has its own `versionCode` and `versionName`, but the existing release helper still bumps only `app/build.gradle.kts` and builds only `:app` AAB variants. As-is, maintainers do not have a proven version/sign/publish path for the companion app introduced here.
  - confidence: 0.86
- **[P2] Clear bootstrap auth when editing direct credentials:** `apps/android/wear/src/main/java/ai/openclaw/wear/ui/ConnectionSettingsScreen.kt:47-51`
  - body: Editing the direct token or password preserves `bootstrapToken`, but `buildWearConnectParams` tries bootstrap auth before password when no token is set. A stale fallback bootstrap token can therefore keep shadowing a newly entered password and make direct re-auth fail until hidden state is cleared elsewhere.
  - confidence: 0.84
