## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Disable backup for Wear auth files:** `apps/android/wear/src/main/AndroidManifest.xml:10-12`
  - body: The Wear app enables backup while the shared identity store writes `privateKeyPkcs8Base64` under `context.filesDir/openclaw/identity/device.json`, and the new backup/data-extraction rules include the whole file domain. Cloud backup or device transfer can copy the watch gateway signing key, so set `allowBackup=false` like the phone app or exclude `openclaw/identity` and other auth material from both rules.
  - confidence: 0.9
- **[P2] Remove the broad Wear listener binding:** `apps/android/app/src/main/AndroidManifest.xml:97`
  - body: This service already has a `MESSAGE_RECEIVED` filter scoped to `/openclaw`. Adding `BIND_LISTENER` reintroduces the deprecated broad listener path that wakes/binds the proxy service for unrelated Data Layer events, so keep the fine-grained message filter unless there is a measured need for the old binding.
  - confidence: 0.83
