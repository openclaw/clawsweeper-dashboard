## Summary

Keep open. Current main still has the reported subagent fallback mismatch: isolated cron model selection can use a `subagents.model` primary, but the fallback override and embedded `fallbackConfigured` paths only consult payload fallbacks or normal `agents.*.model` fallback policy; open PR #72984 is adjacent primary-resolution work, not a complete fix for this cron fallback-policy path.

## What This Changes

The issue requests that isolated embedded-agent cron runs fail over from a timed-out `subagents.model` primary to the fallbacks configured with that subagent model.

## Best Possible Solution

Add generic subagent-aware fallback-policy resolution so the same `subagents.model` object that supplies an isolated cron primary also supplies its fallback chain, while preserving payload fallback precedence and explicit empty arrays.

## Reproduction Assessment

Yes. Configure `agents.defaults.subagents.model` or `agents.list[].subagents.model` as `{ primary: "kimi/kimi-code", fallbacks: [...] }`, run an isolated cron `agentTurn` without payload fallbacks, and current source selects that primary while fallback resolution ignores that fallback list.

## Solution Assessment

Yes. The narrow maintainable fix is shared subagent-aware fallback-policy plumbing, not Kimi-specific timeout handling or provider-specific failover behavior.

## Repair Work Prompt

Fix the isolated cron embedded-agent fallback mismatch reported in https://github.com/openclaw/openclaw/issues/74985. When `agents.defaults.subagents.model` or `agents.list[].subagents.model` is object-form `{ primary, fallbacks }`, isolated cron `agentTurn` runs should use that primary and the matching fallback chain. Preserve existing precedence: payload `fallbacks` wins, explicit `fallbacks: []` disables fallback, payload `model` remains a job primary, and normal `agents.*.model` fallback behavior must not regress. Prefer a generic subagent-aware helper rather than Kimi/provider-specific handling. Coordinate with open PR #72984 and related issue #44353, but do not treat either as complete coverage unless the cron fallback-policy and embedded `fallbackConfigured` paths are actually fixed. Add focused regression coverage for cron fallback override resolution and embedded fallback observation as needed. Add a single-line user-facing `CHANGELOG.md` fix entry unless the final patch is tests-only.
