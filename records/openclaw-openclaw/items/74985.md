## Summary

Keep open. Current main still has a source-level mismatch: isolated cron model selection can choose a `subagents.model` primary, while the cron fallback override and embedded-runner `fallbackConfigured` paths still consult only payload fallbacks or normal `agents.*.model` fallback policy.

## What This Changes

The issue asks for isolated embedded-agent timeouts on a configured subagent primary to fail over through the configured subagent fallback chain.

## Best Possible Solution

Add shared subagent-aware fallback-policy resolution so the same selected `subagents.model` object supplies both the primary and fallback chain, while preserving payload fallback precedence and explicit empty arrays.

## Reproduction Assessment

Yes. Source inspection is enough: configure a subagent model object with `primary` and `fallbacks`, run an isolated cron `agentTurn` without payload fallbacks, and current main selects the subagent primary while the fallback resolver and embedded `fallbackConfigured` flag ignore that subagent fallback list.

## Solution Assessment

Yes. The narrow maintainable fix is generic subagent-aware fallback-policy plumbing, not Kimi-specific timeout handling or a new config option.

## Repair Work Prompt

Fix the isolated embedded-agent subagent fallback mismatch from https://github.com/openclaw/openclaw/issues/74985. Current main can select a primary from `agents.defaults.subagents.model` or `agents.list[].subagents.model`, but `resolveCronFallbacksOverride`, `resolveEffectiveModelFallbacks`, and `hasConfiguredModelFallbacks` only read payload fallbacks or normal `agents.*.model` policy. Add shared subagent-aware fallback-policy plumbing so the selected subagent model object supplies its `fallbacks` to cron isolated runs and the embedded runner `fallbackConfigured` decision. Preserve payload `fallbacks` precedence, preserve explicit empty arrays as strict/no-fallback overrides, and do not add a new config option or provider/Kimi-specific behavior. Coordinate with, but do not duplicate, the adjacent open primary-resolution work in https://github.com/openclaw/openclaw/pull/72984.
