## Summary

Keep open. Current main still has the reported source-level mismatch: isolated cron selection can run on a subagent model primary while cron fallback resolution and the embedded runner fallback flag ignore the fallback list attached to that same subagent model config.

## What This Changes

The issue requests that isolated embedded-agent cron runs fail over to configured subagent model fallbacks when the primary embedded model times out.

## Best Possible Solution

Centralize subagent-aware fallback-policy resolution so the selected `subagents.model` object supplies both the primary and fallback chain, while preserving payload fallback precedence and explicit empty arrays.

## Reproduction Assessment

Yes, at source level. Configure `subagents.model` as `{primary,fallbacks}`, run an isolated cron `agentTurn` without payload fallbacks, and current main selects the subagent primary while fallback policy and `fallbackConfigured` ignore that fallback list.

## Solution Assessment

Yes. The narrow maintainable fix is shared subagent-aware fallback-policy plumbing, not Kimi-specific timeout handling or a new configuration option.
