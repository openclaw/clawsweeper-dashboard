## Summary

Keep open. Current main decodes `WizardStep.sensitive` and the SwiftUI wizard honors it, but the bundled `openclaw-mac wizard` text prompt path ignores the flag, prints `[initial]`, and reads through echoing `readLine()`. Related #76693 is still open and explicitly defers this macOS CLI leak path.

## What This Changes

Request that the bundled macOS `openclaw-mac wizard` CLI mask `WizardStep.sensitive` text inputs and avoid displaying sensitive initial values.

## Best Possible Solution

Implement the existing sensitive-step contract in `WizardCommand.swift` with a termios-based no-echo reader, suppress sensitive initial-value display, preserve long credentials, and validate with the focused macOS Swift lint/format/build plus manual or scripted TTY proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: drive `openclaw-mac wizard` against a gateway step with `type: "text"`, `sensitive: true`, and an initial value, and the current CLI will build a prompt containing `[initial]` and read through echoing `readLine()`.

## Solution Assessment

Yes. The requested fix is the narrow maintainable solution because the protocol field already exists and the SwiftUI client already honors it; only the macOS CLI renderer needs no-echo handling for sensitive text steps.
