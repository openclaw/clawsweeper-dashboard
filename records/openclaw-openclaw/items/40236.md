## Summary

Keep open. Current main still lacks the requested persisted per-job cron lifecycle contract; it has payload-level agent timeouts, one-shot delete-after-run behavior, run/session retention, failure alerts, and observation hooks, but not job-level timeout, retry/cleanup policy, or per-job start/complete/failure hooks.

## What This Changes

Request adds cron job-level lifecycle configuration for timeout, retry/cleanup policy, and start/complete/failure hooks.

## Best Possible Solution

Design one additive cron lifecycle contract across Gateway schemas, stored job normalization, scheduler timeout/cleanup behavior, hook execution boundaries, CLI/UI/docs, and focused regression tests while coordinating with #45465.

## Reproduction Assessment

Not applicable as a feature/API request. The absence is directly verifiable by inspecting the current cron protocol schema, stored job types, scheduler timeout/cleanup logic, docs, and related open PR context.

## Solution Assessment

No. The API sketch is directionally useful, but the hook portion needs maintainer API/security design, and the open hook PR only covers part of the lifecycle surface while timeout and cleanup remain scheduler/store concerns.
