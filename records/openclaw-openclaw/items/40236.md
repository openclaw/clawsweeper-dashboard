## Risks / Open Questions

- The public API still needs to reconcile payload.timeoutSeconds, recurring backoff, one-shot retry, deleteAfterRun, failureAlert, retention, run logs, and cron_changed without duplicated semantics.

- Lifecycle hooks around scheduled jobs can become an unattended host-code or outbound-notification surface, so permissions, timeouts, logging, and secret handling need security review.
- Open related issues and PRs overlap but do not yet settle the unified lifecycle contract.
