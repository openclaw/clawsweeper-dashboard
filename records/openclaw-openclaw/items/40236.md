## Summary

Keep open. Current main has adjacent cron lifecycle pieces, but it still does not implement the requested per-job lifecycle policy/API for `timeoutMs`, job-level retry/cleanup rules, or job-configured start/complete/failure hooks; related issues cover only parts of the request and do not fully supersede it.

## Best Possible Solution

Design a backward-compatible cron lifecycle contract that reuses existing timeout, retry, failure-alert, retention, run-log, and `cron_changed` primitives where possible, then update schema, CLI, docs, and focused cron tests together.
