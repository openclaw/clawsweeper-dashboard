## Summary

Keep open. Current main has adjacent cron timeout, retry/backoff, retention, run-log, failure-alert, and `cron_changed` hook primitives, but it still lacks the requested per-job `lifecycle` contract for timeout, retry/cleanup policy, and job-configured start/complete/failure hooks; related open issues cover pieces of the design but do not fully supersede this combined API request.

## What This Changes

Request a cron job-level lifecycle API with timeout, max retry, auto-cleanup, and start/complete/failure hooks.

## Best Possible Solution

Design a backward-compatible cron lifecycle contract that reuses existing timeout, retry, failure-alert, retention, run-log, and plugin-hook primitives where possible, then update schema, CLI/docs, UI if applicable, and focused cron tests together.
