## Summary

Keep open. Current main still lacks the requested persisted per-job cron lifecycle contract; it has payload-level timeouts, one-shot cleanup, failure alerts, run-log/session retention, and plugin observation hooks, but not job-level timeout, retry/cleanup policy, or per-job start/complete/failure hooks.

## What This Changes

The issue requests a cron lifecycle API with per-job timeout, retry/cleanup policy, auto-cleanup rules, and job-specific start/complete/failure hooks.

## Best Possible Solution

Design one additive cron lifecycle contract across Gateway schemas, stored job normalization, scheduler timeout and cleanup behavior, hook execution boundaries, CLI/UI/docs, and focused regression tests while coordinating with #45465.

## Reproduction Assessment

Not applicable as a feature/API request. The absence is directly verifiable on current main by inspecting the cron protocol schema, stored job types, scheduler timeout/cleanup logic, docs, and related open PR context.

## Solution Assessment

No. The API sketch is directionally useful, but the hook portion needs maintainer API/security design, and #45465 only covers part of the requested lifecycle surface while timeout and cleanup remain scheduler/store concerns.
