## Summary

Keep open. Current main still does not provide the requested persisted per-job lifecycle contract for timeout, retry/cleanup policy, and job-specific start/complete/failure hooks; it only has adjacent payload-level timeouts, one-shot deletion, failure alerts, retention cleanup, and plugin-level cron observation. The related open PR #45465 overlaps the hook portion but is unmerged and does not cover the full timeout plus cleanup request.

## What This Changes

The issue requests cron job-level lifecycle configuration for maximum runtime, retry/cleanup policies, and start/complete/failure hooks.

## Best Possible Solution

Design one additive cron lifecycle contract across Gateway protocol schemas, stored job normalization, scheduler timeout semantics, cleanup policy, hook execution boundaries, CLI/UI/docs, and focused regression tests while preserving current payload timeout, one-shot deletion, failure alerts, run-log retention, and plugin cron_changed observation behavior.

## Reproduction Assessment

Not applicable as a feature/API request. The absence is directly verifiable on current main by inspecting the cron protocol schema, stored job types, scheduler timeout/cleanup logic, CLI flags, docs, and related open PR context.

## Solution Assessment

No. The API sketch is directionally useful, but the hook portion needs maintainer API/security design, and #45465 only covers part of the requested lifecycle surface while timeout and cleanup remain scheduler/store concerns.
