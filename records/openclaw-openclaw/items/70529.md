## Summary

Keep open. Current main still routes the packaged Desktop `user` browser profile through the Chrome MCP existing-session path that defaults to spawning `npx -y chrome-devtools-mcp@latest`; the EasyClaw relay symptom is obsolete, but the Desktop-specific `spawn npx ENOENT` failure remains valid and separately scoped by the discussion.

## What This Changes

The issue reports that Genspark Claw Desktop cannot attach to existing Chrome sessions because the retired EasyClaw relay sign-in fails and the built-in `user` profile tries to spawn unavailable `npx`.

## Best Possible Solution

Keep the EasyClaw relay retired and make packaged Desktop handle the built-in `user` profile without relying on ambient `npx`, either by resolving a supported Chrome MCP runner through the app boundary or by failing early with a structured Desktop diagnostic and documented override.

## Reproduction Assessment

Yes, at source level. Current main resolves the built-in `user` profile to existing-session, defaults Chrome MCP to `npx -y chrome-devtools-mcp@latest`, and passes that command to stdio; the issue supplies the packaged Desktop environment where `npx` is absent.

## Solution Assessment

Unclear because there is no patch here. The existing `mcpCommand` override is a useful workaround, but the maintainable product fix depends on Desktop packaging and Chrome MCP launch policy.
