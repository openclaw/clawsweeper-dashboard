## Summary

Keep open. Current main still defaults the built-in `user` existing-session browser profile to a Chrome MCP launch that spawns `npx`, while the maintainer discussion explicitly scoped this Desktop packaged-app failure as separate from the broader browser attach and login-handoff reports.

## What This Changes

The issue asks OpenClaw Desktop to attach to existing Chrome sessions without failing through the retired EasyClaw OAuth relay or the `user` profile's missing-`npx` Chrome MCP launch path.

## Best Possible Solution

Keep the retired EasyClaw relay out of the path, and make packaged Desktop existing-session attach work without assuming ambient `npx`, or fail before spawn with a clear Desktop diagnostic and documented `mcpCommand` workaround.

## Reproduction Assessment

Yes. Current main gives a high-confidence source-level reproduction: resolve the built-in `user` profile, observe the default Chrome MCP command is `npx -y chrome-devtools-mcp@latest`, and follow the stdio launch path that spawns that executable directly.

## Solution Assessment

Unclear because there is no patch here. The `mcpCommand` override is a useful escape hatch, but the maintainable Desktop fix needs a packaging/process-launch decision rather than another core fallback guess.
