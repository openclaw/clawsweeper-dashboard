Add shared Linux systemd hygiene collection for `KillMode`, `TasksCurrent`, `MemoryCurrent`, and `MainPID`, surface it in doctor/status human and JSON output, and warn conservatively while leaving default/migration policy separate.

## Solution Assessment

Yes: the observability-first path is the narrow maintainable fix because generated units already default to `control-group`; any `mixed` versus `control-group` migration policy should stay separate.
