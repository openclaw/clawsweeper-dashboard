## Summary

Keep open. Current main and v2026.5.7 already generate Linux gateway units with `KillMode=control-group`, but doctor/status still do not collect, model, or print `KillMode`, `TasksCurrent`, or `MemoryCurrent`, so the main observability request remains valid.

## What This Changes

Add Linux systemd gateway hygiene visibility to `openclaw doctor` and `openclaw gateway status --deep`, with consideration of safer gateway unit KillMode behavior.

## Best Possible Solution

Add read-only Linux systemd hygiene collection shared by gateway status and doctor, expose the raw fields, and warn conservatively while leaving KillMode default/migration policy as a separate follow-up if maintainers want it.

## Reproduction Assessment

Yes for the remaining diagnostic gap: current main requests/parses only active state, substate, main PID, and exit fields, so `KillMode`, `TasksCurrent`, and `MemoryCurrent` would be ignored. The original production cgroup growth was not live-reproduced.

## Solution Assessment

Yes for the next step: observability-first is the narrow maintainable path because generated units already default to `control-group`. Any `mixed` versus `control-group` migration policy should stay separate from the diagnostic PR.

## Repair Work Prompt

Implement a narrow read-only diagnostic PR for https://github.com/openclaw/openclaw/issues/50186. Scope: Linux local systemd gateway services only. Do not change generated `KillMode` defaults, silently migrate existing units, or add a config option. Extend the systemd runtime/query/parser to request and carry `KillMode`, `TasksCurrent`, `MemoryCurrent`, and `MainPID` as typed fields; expose them in `openclaw gateway status --deep` text/JSON and in doctor service diagnostics. Add a conservative non-fatal warning when `KillMode=process` and the service hygiene looks suspicious, and include concrete next-step commands such as `systemctl --user show openclaw-gateway.service -p KillMode -p TasksCurrent -p MemoryCurrent`, `systemd-cgls --user-unit openclaw-gateway.service`, and `systemctl --user restart openclaw-gateway.service`. Add focused unit tests for parser/runtime shaping, status output, and doctor warnings. Add a changelog entry because this is user-visible diagnostics.
