Implement the remaining diagnostic portion of https://github.com/openclaw/openclaw/issues/50186. Add read-only Linux user-systemd hygiene collection for the gateway service: request and parse `KillMode`, `TasksCurrent`, `MemoryCurrent`, and keep `MainPID`; carry those fields through `GatewayServiceRuntime`; expose them in `openclaw gateway status --deep` JSON/human output; and add conservative doctor/status warnings and next-step hints when `KillMode=process` plus high task or memory counts suggest cgroup contamination. Keep warnings low-noise, update focused tests, and add a changelog entry if this ships as user-visible diagnostics.
