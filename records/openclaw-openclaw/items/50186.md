Add Linux systemd gateway hygiene diagnostics to doctor/status and consider safer KillMode handling for generated or repaired units.

## Best Possible Solution

Keep the shipped `control-group` default, add read-only Linux systemd hygiene metrics and conservative warnings to doctor/status, and leave automatic existing-unit migration to explicit repair policy.

## Solution Assessment

Yes for the staged observability path: add read-only `systemctl show` diagnostics and focused warnings while preserving the current `control-group` default. Existing-unit migration should stay explicit rather than automatic.
