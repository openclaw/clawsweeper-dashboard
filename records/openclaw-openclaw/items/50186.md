## Best Possible Solution

Add shared Linux systemd hygiene collection for `KillMode`, `TasksCurrent`, `MemoryCurrent`, and `MainPID`, surface it in doctor/status human and JSON output, and warn conservatively while leaving process reaping and default migration policy separate.
