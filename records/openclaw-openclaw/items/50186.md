## Summary

Keep open. Current main and the latest release already use `KillMode=control-group` for generated Linux systemd gateway units, but doctor/status still do not collect, model, expose, or warn on `KillMode`, `TasksCurrent`, or `MemoryCurrent`, so the remaining observability request is still valid.

## What This Changes

Request to surface Linux systemd gateway hygiene metrics in doctor/status and reconsider safer generated-unit `KillMode` defaults.

## Best Possible Solution

Add a read-only Linux systemd hygiene collector shared by gateway status and doctor/service audit, expose the values in deep status/JSON, and warn on suspicious `KillMode=process` plus high cgroup task or memory counts while leaving restart-policy changes separate.

## Reproduction Assessment

Yes for the remaining diagnostic gap: source inspection shows mocked `systemctl show` output containing `KillMode=process`, `TasksCurrent`, and `MemoryCurrent` would be ignored because current main neither requests nor models those fields. The original production cgroup growth was not live-reproduced here.

## Solution Assessment

Yes for the next step: observability first is the narrow maintainable fix because generated units already default to `control-group`. Any `mixed` versus `control-group` policy change should be reviewed separately.

## Repair Work Prompt

Implement the diagnostics portion of https://github.com/openclaw/openclaw/issues/50186. Add a read-only Linux systemd hygiene collector that requests/parses `KillMode`, `TasksCurrent`, `MemoryCurrent`, and `MainPID` for the managed gateway service; thread those fields through `GatewayServiceRuntime` or an adjacent typed status/audit structure; expose them in `openclaw gateway status --deep --json` and human deep status; and have doctor/service audit emit a focused warning when `KillMode=process` combines with suspiciously high cgroup task or memory counts. Include concrete operator next steps such as `systemctl --user show openclaw-gateway.service -p KillMode -p TasksCurrent -p MemoryCurrent`, `systemd-cgls --user-unit openclaw-gateway.service`, and `openclaw gateway restart` or `systemctl --user restart openclaw-gateway.service`. Do not change the current generated-unit default from `KillMode=control-group`, do not silently migrate existing services, and do not add a config option. Add regression tests with mocked `systemctl show` output and a user-facing `CHANGELOG.md` fix/feature entry.
