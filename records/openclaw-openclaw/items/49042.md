## Summary

Keep open: current main does not expose `agent_end.finalLlmOutcome`, and the PR is useful but not merge-ready because it still misclassifies provider-call timeouts and is stale against the current hook contract split.

## What This Changes

The PR adds optional `agent_end.finalLlmOutcome` plumbing, docs, and tests for embedded runner terminal provider/prompt/runner outcomes, plus several test-stabilization edits.

## Best Possible Solution

Land a current-main version of the plugin hook contract in `src/plugins/hook-types.ts`, populate it from embedded `agent_end` without conflating provider timeouts with runner failures, and include docs, changelog, and focused regression tests.

## Reproduction Assessment

Yes for the review finding: source inspection of PR head `94c1f5e` shows provider-call `TimeoutError` values satisfy the runner-failure branch before the provider branch. Current main inspection also confirms the requested field is not already implemented.

## Solution Assessment

No. The direction may be useful, but this PR should be rebased to the current hook contract file and should classify provider-call timeouts as provider outcomes unless the timeout came from local runner abort/timeout handling.

- **[P2] Classify provider-call timeouts as provider outcomes:** `src/agents/pi-embedded-runner/run/attempt.ts:1422`
