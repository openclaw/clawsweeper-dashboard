## Summary

Keep this PR open: current main still lacks structured `agent_end.finalLlmOutcome`, and the PR is useful, but it is not merge-ready because it changes a public plugin contract on a stale branch and still has a provider-timeout classification bug plus missing changelog coverage.

## What This Changes

The PR adds optional structured `finalLlmOutcome` metadata to the `agent_end` plugin hook, wires it through the embedded runner, documents it, and adds runner/helper tests plus unrelated test-stabilization edits.

## Best Possible Solution

Rebase onto current main, add the public hook type in `src/plugins/hook-types.ts`, preserve legacy `success`, classify provider-started timeouts as provider outcomes, document the field in `docs/plugins/hooks.md`, add a changelog entry, and drop unrelated test edits.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the missing contract because both the hook type and embedded runner emit path omit `finalLlmOutcome`; static inspection of the PR diff reproduces the timeout classification bug.

## Solution Assessment

No, not as-is. The additive `agent_end.finalLlmOutcome` direction is plausible, but the current patch needs rebase, contract-location, timeout-classification, changelog, and scope cleanup before it is the narrow maintainable solution.
