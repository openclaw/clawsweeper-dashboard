## Summary

Keep open. Current main still lacks the requested structured `agent_end.finalLlmOutcome` contract, so the PR is not obsolete, but the branch is stale against the current hook contract split and still has a provider-timeout classification bug before it is merge-ready.

## What This Changes

The PR adds optional structured `finalLlmOutcome` metadata to the plugin `agent_end` hook, documents it, adds embedded-runner tests, and includes several unrelated test-stabilization edits.

## Best Possible Solution

Rebase the useful API change onto current main, put the public hook type in `src/plugins/hook-types.ts`, keep legacy `success` semantics unchanged, classify provider-started timeouts as provider outcomes unless a local runner abort is known, document `ok` versus `success`, add the changelog entry, and drop unrelated test-stabilization edits.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the missing contract because both the hook type and embedded runner emit path omit `finalLlmOutcome`; static inspection of the PR diff also reproduces the timeout classification bug.

## Solution Assessment

No, not as-is. The additive `agent_end.finalLlmOutcome` direction is plausible, but the current branch needs a rebase, a provider-timeout classification fix, changelog coverage, and a narrower diff before it is the maintainable solution.
