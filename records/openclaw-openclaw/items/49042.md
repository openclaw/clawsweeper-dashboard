## Summary

Keep this PR open, but do not merge it as-is. Current main still lacks the requested terminal `agent_end.finalLlmOutcome` contract, so the work is not obsolete; however the proposed helper still misclassifies provider-side timeouts as runner failures, and the branch is stale against the current hook type split.

## What This Changes

The PR adds an optional structured `finalLlmOutcome` object to the `agent_end` plugin hook in the embedded runner, updates plugin docs and tests, and includes several unrelated test-stabilization edits.

## Best Possible Solution

Rebase the useful API change onto current main, put the public hook type in `src/plugins/hook-types.ts`, keep `success` semantics unchanged, classify provider-started timeouts as provider outcomes unless a local abort/compaction runner failure is known, document `finalLlmOutcome.ok` versus `success`, and drop unrelated test stabilization from this PR.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the missing contract because `agent_end` has no `finalLlmOutcome`, and the PR helper bug is reproducible from the proposed `buildAgentEndFinalLlmOutcome` branch where a provider-started `TimeoutError` is classified before `providerCallStarted` is considered.

## Solution Assessment

No, not as-is. The additive `agent_end.finalLlmOutcome` direction is plausible, but the current branch needs rebase work, a provider-timeout classification fix, and a narrower diff before it is the maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Classify provider timeouts as provider outcomes:** `src/agents/pi-embedded-runner/run/attempt.ts:1423`
  - body: When `activeSession.prompt()` has already started and fails with an SDK or network `TimeoutError`, this predicate marks the final outcome as `source: "runner"` before `providerCallStarted` is considered. That breaks the new contract’s purpose because plugins cannot distinguish provider timeouts from local runner aborts/timeouts; local `aborted`, `timedOut`, runner-abort, and compaction failures should take precedence, but provider-started timeout errors should remain provider outcomes.
