## Best Possible Solution

If maintainers want `agent_end` to carry terminal LLM outcome data, implement it narrowly in current `src/plugins/hook-types.ts` and `docs/plugins/hooks.md`, classify provider-started failures accurately, preserve guardrail tests, and require redacted real plugin-output proof; otherwise keep provider outcome data on `model_call_*` telemetry.
