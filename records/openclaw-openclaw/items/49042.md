## Summary

Keep open: current main still lacks the requested `agent_end.finalLlmOutcome`, but this PR is not merge-ready because it targets stale hook/docs locations, misclassifies provider-started timeouts, and lacks real behavior proof for the new public plugin API.

## What This Changes

The PR adds optional structured `finalLlmOutcome` metadata to the `agent_end` hook, wires it through the embedded runner, documents it, and adds helper tests plus unrelated test-stabilization edits.

## Best Possible Solution

Define the additive hook field in `src/plugins/hook-types.ts`, document it in `docs/plugins/hooks.md`, classify provider-started failures consistently, keep unrelated test churn out, and prove it with focused tests plus a real plugin run.

## Reproduction Assessment

Yes for the code-path gap and PR defect: static inspection shows current main lacks the field in the hook contract and runner payload, and the PR diff shows the provider-timeout classification issue. This is primarily a new public hook capability rather than a strict existing-contract bug.

## Solution Assessment

No, not as-is. The additive direction is plausible, but the patch needs rebase to current hook/docs surfaces, a timeout-source fix, scope cleanup, maintainer API review, and real behavior proof before it is the narrow maintainable solution.
