## Summary

Keep open: current main still has the native sub-agent skill-visibility gap, but this PR is not mergeable as written because the added block reads `spawnedMetadata` before its declaration.

## What This Changes

The PR imports workspace skill helpers and appends the target agent's resolved workspace skills prompt to native sub-agent system prompts.

## Best Possible Solution

Move skill prompt resolution after normalized spawned metadata is available, keep the existing filtered skill loader and tool-policy boundary, and add focused regression tests for injection plus non-blocking loader failure.

## Reproduction Assessment

Yes. Source inspection on current main shows native sub-agent spawns build their own `extraSystemPrompt` without workspace skill prompt injection, and the contributor supplied before/after live output showing the workspace-skill tool wall.

## Solution Assessment

No, not as currently written. The repair direction is narrow and appropriate, but the block must stop reading `spawnedMetadata` before declaration and should add regression coverage for the prompt injection path.
