## Best Possible Solution

Move skill prompt injection to the point where normalized spawned workspace metadata is available, preserve the existing filtered skill loader contract, and add focused prompt-injection and failure-tolerance tests.

## Reproduction Assessment

## Solution Assessment

No, not as currently written. The approach is narrow, but the block must stop reading `spawnedMetadata` before initialization and should be covered by focused sub-agent prompt tests before merge.
