## Summary

Current main already includes eligible skills in the normal agent-run prompt path used by sub-agents, and this branch is both redundant and typecheck-breaking because it reads spawned metadata before initialization.

## What This Changes

The PR imports workspace skill helpers into `src/agents/subagent-spawn.ts` and appends a resolved skills prompt to the child sub-agent extra system prompt.

## Best Possible Solution

Close this branch and keep the shipped generic prompt-building path; any fresh skills gap should be handled with a targeted `sessions_spawn` regression test against that existing pipeline.

## Reproduction Assessment

Yes for the PR defect: source inspection shows the new code reads `spawnedMetadata` before the current declaration, which is a TypeScript use-before-declaration error. The originally reported tool-wall behavior was not live-reproduced, but current source and release provenance show the shipped sub-agent skills path already exists.

## Solution Assessment

No. The maintainable solution is the existing generic agent-run skills pipeline, not a second direct skill scan inside `subagent-spawn.ts`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.96

Full review comments:

- **[P1] Initialize spawned metadata before reading it:** `src/agents/subagent-spawn.ts:1004`
  - body: The new skills block runs before `spawnedMetadata` is declared later in this function. TypeScript will reject this use-before-declaration, and if the code were emitted the caught ReferenceError would make the new skills injection silently do nothing.
