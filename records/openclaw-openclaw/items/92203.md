## Summary

Keep open. The PR is not merge-ready because it removes CLI backend ids from startup owner metadata that current gateway startup code and tests intentionally use to preload configured CLI backend owner plugins.

## What This Changes

The PR removes CLI auth backends from installed-plugin startup agent harness lists, adds a regression assertion, adds optional Docker build args for Claude Code CLI installation, and edits the release changelog.

## Best Possible Solution

Keep CLI backend owner preload semantics intact, separate embedded harness ids from CLI backend owner metadata if the field name is misleading, and move release notes plus any optional Docker install surface through the repository-approved paths.

## Reproduction Assessment

Yes. Source inspection and existing startup tests show the patch-level regression: CLI backend ids are intentionally used as startup owner metadata so configured CLI runtime policies preload their plugins.

## Solution Assessment

No. Removing `cliBackends` from the installed-index startup list is too broad; the better fix is to keep owner preload behavior while separating or renaming metadata if embedded harness and CLI backend concepts need clearer boundaries.
