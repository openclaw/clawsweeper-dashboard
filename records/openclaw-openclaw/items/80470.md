## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Clear the connect timer when baseFn throws:** `src/agents/pi-embedded-runner/run/llm-idle-timeout.ts:234`
  - body: The connection watchdog is armed before calling `baseFn`, but only the async rejection and stream-wrapping paths clear it. If a provider stream function throws synchronously during setup, the original error is thrown while `connectTimer` remains live, so `onIdleTimeout` can fire later with a stale timeout and keep the process open after the real failure. Wrap the `baseFn(...)` call in a `try/catch` that clears the connect timer before rethrowing.
  - confidence: 0.87
