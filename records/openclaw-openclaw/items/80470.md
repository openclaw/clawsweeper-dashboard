## Summary

Keep open: the reported heartbeat failure is credible and source-backed, but the patch still has blocking correctness issues in the new timeout race and doctor warning, and the external real-behavior proof is descriptive rather than inspectable output.

## What This Changes

This PR changes heartbeat broadcasts to run per agent in parallel, scopes busy-lane checks per agent, appends HEARTBEAT.md directives more broadly, adds an initial LLM stream connection timeout, adds a doctor warning for missing heartbeat sessions, updates focused tests, and adds a changelog entry.

## Best Possible Solution

Land a focused heartbeat fix after clearing the timeout timer on every promise outcome, reusing heartbeat session canonicalization in doctor, aligning user-facing help for per-agent busy semantics, and attaching redacted live proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current main serializes broadcast `runOnce` calls and globally checks Subagent/Nested lanes, and the PR body gives a four-agent Linux gateway scenario. I did not run the live gateway in this read-only review.

## Solution Assessment

No, not as-is. The overall direction is plausible, but the timeout race needs rejection cleanup, doctor needs to use the runtime session-key resolver, and maintainers should explicitly accept the per-agent `skipWhenBusy` contract before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Clear the connection timer when stream setup rejects:** `src/agents/pi-embedded-runner/run/llm-idle-timeout.ts:275`
  - body: When `baseFn` returns a promise that rejects before the idle timeout, this race rejects immediately but `clearConnectTimer()` is never called because only `wrapStream()` clears it. The timer remains active, later invokes `onIdleTimeout`, and can report a stale timeout or keep the process open after the real provider error. Clear the timer on both resolve and reject, for example with a `finally` around the setup promise before racing.
  - confidence: 0.91
- **[P2] Canonicalize heartbeat sessions before warning:** `src/commands/doctor-heartbeat-session-target.ts:55`
  - body: The runtime resolves `heartbeat.session` through `toAgentStoreSessionKey(...)` and main-alias canonicalization before looking in the session store. This doctor check uses `store[configuredSession]` directly, so valid shorthand or alias values that runtime maps to `agent:<id>:...` can produce a false warning telling operators their heartbeat replies are dropped. Reuse the same resolver/canonical key before checking existence.
  - confidence: 0.86
- **[P3] Update skipWhenBusy help for agent-scoped lanes:** `src/infra/heartbeat-runner.ts:1265`
  - body: This line changes `skipWhenBusy` from broad global subagent/nested lane pressure to per-agent session-keyed lane pressure, but the docs and schema help still say it defers on subagent or nested command work broadly. Update the user-facing help/docs so operators do not expect the old global pause behavior.
  - confidence: 0.78
