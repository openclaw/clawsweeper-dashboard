## Summary

Keep open: current `main` still has a Matrix package-boundary mismatch. The issue has strong reporter evidence, and source inspection shows Matrix runtime code can still be present/loadable from the root package while `matrix-js-sdk` and the Matrix crypto packages are only plugin-local and explicitly absent from root dependencies; a later comment says someone intends to fix it, but the provided context does not show a merged fix PR.

## What This Changes

The issue reports that host npm updates to OpenClaw 2026.5.4 leave configured Matrix accounts unable to start because `matrix-js-sdk` and Matrix crypto packages are missing from the installed OpenClaw package root.

## Best Possible Solution

Make host-package updates consistently preserve Matrix availability: either repair/install the external `@openclaw/matrix` plugin before startup, or keep bundled Matrix runtime in the root package with resolvable runtime dependencies and package-acceptance coverage.

## Reproduction Assessment

Yes, at source level. A host npm install with Matrix configured can reach Matrix runtime imports for `matrix-js-sdk`, while the root package excludes extension node_modules and does not declare the Matrix SDK packages; I did not execute the live update path in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is package-boundary alignment, not runtime npm installs during gateway startup. Since current docs already call Matrix downloadable, completing externalized plugin repair/migration is likely cleaner unless maintainers choose to keep Matrix bundled in root.
