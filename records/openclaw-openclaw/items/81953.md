## Summary

Current main already implements the requested Telegram forum-topic isolation: the isolated polling spool now dispatches by Telegram sequential lane, and the merged fix directly addresses the pre-agent serialization described in the issue.

## What This Changes

The issue reports that Telegram forum-topic messages in different topics of one group appear to wait behind each other before entering the agent pipeline.

## Best Possible Solution

Keep the current lane-keyed isolated ingress implementation and ship it in the next release rather than adding a new Telegram config or changing session semantics.

## Reproduction Assessment

No, not as a current-main failure. The release logs and merged fix identify the old serial spool drain path, while current source and regression coverage show different Telegram topic lanes now interleave before agent dispatch.

## Solution Assessment

Yes. Reusing Telegram's existing sequential key at the isolated spool drain is the narrowest maintainable fix because it preserves same-topic order while allowing independent topics and control lanes to progress.
