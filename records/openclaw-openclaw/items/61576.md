## Security Review

Status: needs_attention

Summary: The diff adds a credentialed native gateway client with concrete TLS-verification and token-persistence concerns that need security review before merge.

- **[medium] Gate the TLS verification bypass:** `apps/linux-gtk/crates/gateway-client/src/client.rs:234`
  - body: When `OPENCLAW_TLS_ACCEPT_INVALID` is set, the Rust client installs a verifier that accepts any certificate for `wss://` URLs.   - body: The Settings view writes gateway tokens only into process environment variables while the PR marks token persistence as a known security gap. The merge path should use the established OpenClaw config/credential model instead of ad hoc process-local token handling.
