- **[P1] Advertise the current gateway protocol version:** `apps/linux-gtk/crates/gateway-client/src/protocol.rs:218`
  - body: This client advertises protocol 3, but current main requires client min/max protocol 4. The gateway will reject the app during connection negotiation until this native client uses the current protocol contract.
  - confidence: 0.94
- **[P1] Send raw config text to config.set:** `apps/linux-gtk/crates/app/src/views/config.rs:222-225`
  - body: `config.set` validates `{ raw, baseHash? }`, but this button sends `{ path, value }`. Users can edit the JSON view, but Save will be rejected by current gateway schema instead of persisting the config.
  - confidence: 0.92
- **[P1] Persist connection settings instead of process env only:** `apps/linux-gtk/crates/app/src/views/settings.rs:219-236`
  - body: The settings save path only mutates this process's environment and even notes the running client will not be rebuilt. After an app restart the new URL/token are gone, so the UI claims settings were saved when they were not durable.
  - confidence: 0.88
- **[P2] Use current skill update and install params:** `apps/linux-gtk/crates/app/src/views/skills.rs:123-163`
  - body: Current `skills.update` expects `skillKey`, and `skills.install` expects a legacy `{ name, installId }` or source-specific payload. Sending `skillId` makes enable/disable and install actions fail validation.
  - confidence: 0.9
- **[P2] Wrap cron enablement in a patch object:** `apps/linux-gtk/crates/app/src/views/cron.rs:327-331`
  - body: `cron.update` requires mutable fields under `patch`; this sends `enabled` at the top level. The switch updates optimistically, but current gateway validation rejects the request.
  - confidence: 0.9
- **[P2] Pass supported channel status probe params:** `apps/linux-gtk/crates/app/src/views/channels.rs:126-128`
  - body: `channels.status` accepts `probe`, `timeoutMs`, and `channel`, not `channelId`. This probe button will return an invalid-request error instead of checking the selected channel.
  - confidence: 0.86
- **[P2] Read pending node pairs from pending:** `apps/linux-gtk/crates/app/src/views/instances.rs:96-100`
  - body: `node.pair.list` returns `{ pending, paired }`, but this view reads `requests`. Real pending pair approvals will be hidden, so operators cannot approve or reject them from this screen.
  - confidence: 0.86
- **[P2] Call the supported restart request method:** `apps/linux-gtk/crates/app/src/views/control_room.rs:66`
  - body: Current main advertises `gateway.restart.request` and `gateway.restart.preflight`, not `gateway.restart`. The Control Room Restart action will fail with unknown-method instead of requesting a gateway restart.
  - confidence: 0.9
