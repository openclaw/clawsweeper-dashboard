Full review comments:

- **[P1] Advertise the current gateway protocol:** `apps/linux-gtk/crates/gateway-client/src/protocol.rs:218`
  - body: Current main only accepts clients whose min/max range includes protocol 4, but this client hard-codes `PROTOCOL_VERSION` to 3 and sends 3/3 in the init payload. The gateway will reject the connection before the Linux app can use any view.
  - confidence: 0.95
- **[P1] Send current config RPC payloads:** `apps/linux-gtk/crates/app/src/views/config.rs:137-138`
  - body: `config.get` no longer accepts a path payload, and `config.set` requires `{ raw, baseHash? }`; this view sends `{ path: "" }` and later `{ path: "", value }`. Loading or saving config will fail schema validation against current main.
  - confidence: 0.92
- **[P1] Use supported skills mutation shapes:** `apps/linux-gtk/crates/app/src/views/skills.rs:123-128`
  - body: The current gateway expects `skills.update` to receive `skillKey` and `skills.install` to receive one of the named install variants, but this view sends `skillId` for both paths. Enable/disable and install actions therefore use unsupported request shapes.
  - confidence: 0.9
- **[P1] Wrap cron mutations in the current schema:** `apps/linux-gtk/crates/app/src/views/cron.rs:147-152`
  - body: `cron.add` expects a structured schedule, session target, wake mode, and payload, and `cron.update` expects a `patch`; this view sends flat legacy fields. Creating or toggling cron jobs from the app will be rejected by the gateway.
  - confidence: 0.9
- **[P2] Probe channels with the supported field name:** `apps/linux-gtk/crates/app/src/views/channels.rs:126-128`
  - body: `channels.status` accepts `channel`, not `channelId`, with additional properties disabled. The probe action therefore sends an invalid payload instead of refreshing the selected channel.
  - confidence: 0.86
- **[P2] Call the supported gateway restart methods:** `apps/linux-gtk/crates/app/src/views/control_room.rs:65-67`
  - body: The gateway exposes `gateway.restart.preflight` and `gateway.restart.request`, not `gateway.restart`. The Control Room restart button is wired to a method current main does not advertise.
  - confidence: 0.86
- **[P1] Persist connection settings before showing a saved state:** `apps/linux-gtk/crates/app/src/views/settings.rs:219-236`
  - body: The settings UI says the gateway URL/token were saved, but it only mutates the current process environment. Those values disappear on restart and do not rebuild the active client, so users can be told their credential or endpoint change was saved when it was not.
  - confidence: 0.88
- **[P2] Add the new Linux app to label routing:** `apps/linux-gtk/Cargo.toml:1`
  - body: Root policy requires new app surfaces to update label routing, but this PR adds `apps/linux-gtk` without a corresponding `.github/labeler.yml` entry. Follow-up issues and fixes for the app will not receive the app label maintainers rely on for triage.
  - confidence: 0.82
