Full review comments:

- **[P1] Advertise The Current Gateway Protocol:** `apps/linux-gtk/crates/gateway-client/src/protocol.rs:218`
  - body: Current main requires protocol 4 for both the gateway and clients, but this Rust client advertises only protocol 3 for min and max. A user running this app against current main will fail the handshake before any UI workflow can work; update the Rust constants and add this client to the native protocol guard.
  - confidence: 0.94
- **[P1] Use The Current Config RPC Payloads:** `apps/linux-gtk/crates/app/src/views/config.rs:137-138`
  - body: `config.get` no longer accepts a `path`, and `config.set` expects `{ raw, baseHash? }`, not `{ path, value }`. As written, the Config view's load and save actions will be rejected by current gateway schema validation.
  - confidence: 0.93
- **[P1] Persist Connection Settings Before Saying They Are Saved:** `apps/linux-gtk/crates/app/src/views/settings.rs:221-236`
  - body: The Save & Reconnect button only mutates process environment variables even though the running `GatewayClient` was already built from the original config, then tells the user the settings were saved. This neither persists the new URL/token nor reliably reconnects the active client, so users can be left on the old gateway after a successful-looking save.
  - confidence: 0.9
- **[P1] Send Supported Skill RPC Payloads:** `apps/linux-gtk/crates/app/src/views/skills.rs:123-127`
  - body: The Skills view sends `skillId` to both `skills.update` and `skills.install`, but current main accepts `skillKey` for updates and `name`/`installId`, ClawHub fields, or upload fields for installs. The enable and install controls will therefore fail against the gateway.
  - confidence: 0.89
- **[P1] Wrap Cron Mutations In The Current Schema:** `apps/linux-gtk/crates/app/src/views/cron.rs:327-331`
  - body: `cron.update` requires `{ id|jobId, patch: { ... } }`, and `cron.add` requires structured schedule, target, wake mode, and payload fields. The flat `enabled`, `schedule`, `agentId`, and `message` payloads sent here will be rejected by current main.
  - confidence: 0.89
- **[P2] Call The Supported Restart RPC:** `apps/linux-gtk/crates/app/src/views/control_room.rs:65-67`
  - body: Current main exposes `gateway.restart.preflight` and `gateway.restart.request`; there is no `gateway.restart` method in the registered descriptors. The restart button will report failure instead of requesting a restart.
  - confidence: 0.86
