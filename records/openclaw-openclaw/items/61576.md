  - confidence: 0.94

  - confidence: 0.94
- **[P2] Read node pair requests from pending:** `apps/linux-gtk/crates/app/src/views/instances.rs:96-100`
  - body: `node.pair.list` returns `{ pending, paired }`, but this view reads `payload.requests`. Real pending node approvals will be hidden, so operators cannot approve or reject them from this screen.
  - confidence: 0.93

  - confidence: 0.9

  - confidence: 0.9
- **[P2] Persist connection settings durably:** `apps/linux-gtk/crates/app/src/views/settings.rs:219-236`
  - body: Save & Reconnect mutates only process environment variables and then tells users settings are saved for restart. Those values disappear when the app exits, so URL/token changes are not actually durable.
  - confidence: 0.88

  - body: The client waits indefinitely for `connect.challenge` after a WebSocket connection is established. A misrouted or stalled endpoint that accepts WebSockets but never sends the OpenClaw challenge can hang this connection attempt instead of returning to reconnect/backoff.
  - confidence: 0.84
