## Security Review

Status: needs_attention

Summary: Needs attention: the diff adds a credentialed native gateway client with TLS-verification bypass and token-persistence concerns that need owner security review before merge.

Concerns:

- **[medium] Gate the TLS verification bypass:** `apps/linux-gtk/crates/gateway-client/src/client.rs:234`
  - body: When `OPENCLAW_TLS_ACCEPT_INVALID` is set, the Rust client installs a verifier that accepts any certificate for `wss://` URLs. Before shipping a credentialed desktop client, this needs explicit dev-only UX, documentation, and tests so tokens are not sent through an unintended MITMable path.
  - confidence: 0.86

  - body: The Settings view writes gateway tokens only into process environment variables while the PR marks token persistence as a known security gap. The merge path should use the established OpenClaw config and credential model instead of ad hoc process-local token handling.
  - confidence: 0.82
