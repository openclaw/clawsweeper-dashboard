## Summary

Keep open. Full review comments:

- **[P2] Read the configured gateway auth token:** `apps/linux-gtk/crates/gateway-client/src/config.rs:21-25`
  - body: The config reader only deserializes `gateway.token`, while normal OpenClaw config and docs use `gateway.auth.token` or `gateway.auth.password`. Users with generated config will not authenticate unless they also set `OPENCLAW_GATEWAY_TOKEN`.
  - confidence: 0.93
- **[P2] Call config.get without extra params:** `apps/linux-gtk/crates/app/src/views/config.rs:137-138`
  - body: This sends `{ path: "" }`, but current main validates `config.get` against an empty object with no additional properties. The Config view load path will fail validation.
  - confidence: 0.95
- **[P2] Save config through raw and baseHash:** `apps/linux-gtk/crates/app/src/views/config.rs:222-225`
  - body: `config.set` currently accepts `{ raw, baseHash? }`, not `{ path, value }`. The Save button lets users edit JSON but then submits a payload current main rejects.
  - confidence: 0.95
- **[P2] Build cron.add with required fields:** `apps/linux-gtk/crates/app/src/views/cron.rs:147-151`
  - body: `cron.add` requires a name plus structured `schedule`, `sessionTarget`, `wakeMode`, and `payload`; this flat `{ schedule, agentId, message }` request will be rejected when creating jobs.
  - confidence: 0.94
- **[P2] Wrap cron enable updates in patch:** `apps/linux-gtk/crates/app/src/views/cron.rs:327-330`
  - body: `cron.update` requires `{ id|jobId, patch }`, but this toggle sends `enabled` at the top level. The optimistic UI update can show a job as disabled even though the gateway rejected the change.
  - confidence: 0.94
- **[P2] Use skillKey for skill updates:** `apps/linux-gtk/crates/app/src/views/skills.rs:123-127`
  - body: `skills.update` requires `skillKey` for installed skill changes, not `skillId`. Enable/disable toggles will fail validation instead of persisting.
  - confidence: 0.94
- **[P2] Send valid skills.install params:** `apps/linux-gtk/crates/app/src/views/skills.rs:160-163`
  - body: `skills.install` requires either `{ name, installId }`, a ClawHub source/slug, or upload source fields. Sending only `skillId` makes install actions fail validation.
  - confidence: 0.94
- **[P2] Read pending node pairs from pending:** `apps/linux-gtk/crates/app/src/views/instances.rs:96-100`
  - body: `node.pair.list` returns `{ pending, paired }`, but this view reads `payload.requests`. Real pending approvals will be hidden from operators.
  - confidence: 0.93
- **[P2] Call the registered restart RPCs:** `apps/linux-gtk/crates/app/src/views/control_room.rs:65-67`
  - body: Current main registers `gateway.restart.preflight` and `gateway.restart.request`, not `gateway.restart`. The Control Room restart button will return an unknown-method error.
  - confidence: 0.9
- **[P2] Probe channels with supported params:** `apps/linux-gtk/crates/app/src/views/channels.rs:126-128`
  - body: `channels.status` accepts `probe`, `timeoutMs`, and optional `channel`, not `channelId`. This button can be rejected before it runs a health probe.
  - confidence: 0.9
- **[P2] Persist connection settings durably:** `apps/linux-gtk/crates/app/src/views/settings.rs:219-236`
  - body: Save & Reconnect only mutates process environment variables, then tells users settings are saved for restart. Those values disappear when the app exits, so URL/token changes are not durable.
  - confidence: 0.88
- **[P2] Bound the handshake wait:** `apps/linux-gtk/crates/gateway-client/src/client.rs:258-260`
  - body: After a WebSocket connects, the client waits indefinitely for `connect.challenge`. A misrouted or stalled endpoint can hang this connection attempt instead of returning to reconnect/backoff.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The diff adds a credentialed native gateway client with concrete TLS verification bypass and token persistence concerns that need security review.

Concerns:

- **[medium] Gate the TLS verification bypass:** `apps/linux-gtk/crates/gateway-client/src/client.rs:234`
  - body: When `OPENCLAW_TLS_ACCEPT_INVALID` is set, the Rust client installs a verifier that accepts any certificate for `wss://` URLs. Before shipping a credentialed desktop client, this needs explicit dev-only UX, documentation, and tests so tokens are not sent through an unintended MITMable path.
  - confidence: 0.86
- **[medium] Align token persistence with OpenClaw credentials:** `apps/linux-gtk/crates/app/src/views/settings.rs:219`
  - body: The Settings view writes gateway tokens only into process environment variables while the PR marks token persistence as a known security gap. The merge path should use the established OpenClaw config/credential model instead of ad hoc app-local token handling.
  - confidence: 0.82
