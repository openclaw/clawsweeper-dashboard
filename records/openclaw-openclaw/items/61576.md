- **[P1] Advertise the current gateway protocol version:** `apps/linux-gtk/crates/gateway-client/src/protocol.rs:218-222`
  - body: Current main requires gateway protocol 4 and closes mismatches during connect, but this client sends min/max protocol 3. As written, the app cannot connect to a current gateway before any UI workflow can run.
  - confidence: 0.95
- **[P1] Send raw config text to config.set:** `apps/linux-gtk/crates/app/src/views/config.rs:222-225`
  - body: `config.set` accepts `{ raw, baseHash? }` with no extra properties, but this save path sends `{ path, value }`. The Config view will report a failed save against the current gateway schema.
  - confidence: 0.94
- **[P2] Use the current skills payload fields:** `apps/linux-gtk/crates/app/src/views/skills.rs:123-126`
  - body: The current skills schemas do not accept `skillId` for `skills.update` or `skills.install`; toggles need `skillKey`, and installs need the supported install source/name fields. Both UI actions will fail validation.
  - confidence: 0.9
- **[P2] Wrap cron enable changes in a patch object:** `apps/linux-gtk/crates/app/src/views/cron.rs:327-330`
  - body: `cron.update` requires the update data under `patch`, but this toggle sends top-level `enabled` and updates the UI optimistically before the invalid request returns. Users can see a changed state that the gateway rejected.
  - confidence: 0.9
- **[P2] Call the supported restart method:** `apps/linux-gtk/crates/app/src/views/control_room.rs:66`
  - body: There is no `gateway.restart` descriptor on current main; restart is split into `gateway.restart.preflight` and `gateway.restart.request`. This button will fail instead of requesting a restart.
  - confidence: 0.88
- **[P2] Read pending node pairs from the returned field:** `apps/linux-gtk/crates/app/src/views/instances.rs:96-100`
  - body: `node.pair.list` returns `{ pending, paired }`, but this view reads `requests`. Pending pair requests will be hidden and shown as empty even when the gateway returns them.
  - confidence: 0.86
