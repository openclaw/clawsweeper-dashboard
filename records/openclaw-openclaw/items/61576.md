Full review comments:

- **[P1] Advertise the current gateway protocol:** `apps/linux-gtk/crates/gateway-client/src/protocol.rs:218`
  - body: Current main requires protocol 4 and closes mismatched WebSocket handshakes, but this app sends protocol 3 for both `minProtocol` and `maxProtocol`. As written, the companion cannot connect to a current gateway at all.
  - confidence: 0.98
- **[P1] Send raw config text to config.set:** `apps/linux-gtk/crates/app/src/views/config.rs:222-226`
  - body: `config.set` validates only `{ raw, baseHash? }`, but the Save button sends `{ path, value }` after parsing the editor JSON. The request is rejected, so the Config view reports a save attempt but cannot persist edits.
  - confidence: 0.97
- **[P1] Use skillKey for skills.update:** `apps/linux-gtk/crates/app/src/views/skills.rs:123-127`
  - body: The gateway schema for installed skill updates requires `skillKey`, not `skillId`. This makes the enable/disable switch fail validation and leaves the UI out of sync with the stored skill state.
  - confidence: 0.95
- **[P1] Use valid install parameters for skills.install:** `apps/linux-gtk/crates/app/src/views/skills.rs:160-163`
  - body: `skills.install` accepts either `{ name, installId }` or a source-specific install shape, not a bare `skillId`. The Install button therefore cannot install missing skills through the current gateway contract.
  - confidence: 0.95
- **[P1] Wrap cron updates in patch:** `apps/linux-gtk/crates/app/src/views/cron.rs:327-331`
  - body: `cron.update` requires a `patch` object, but this toggle sends `enabled` at the top level. The backend rejects the request while the chip is updated optimistically, so users can think a schedule changed when it did not.
  - confidence: 0.96
- **[P2] Probe channels with supported parameters:** `apps/linux-gtk/crates/app/src/views/channels.rs:126-128`
  - body: `channels.status` allows `probe`, `timeoutMs`, and `channel`; it does not accept `channelId`. The Probe button therefore sends an invalid request instead of checking channel health.
  - confidence: 0.94
- **[P2] Read node pair requests from pending:** `apps/linux-gtk/crates/app/src/views/instances.rs:96-100`
  - body: `node.pair.list` returns `{ pending, paired }`, but the view reads `payload.requests`. Real pending pairing approvals are treated as empty, hiding operator actions that need approval or rejection.
  - confidence: 0.94
- **[P2] Call a supported restart path:** `apps/linux-gtk/crates/app/src/views/control_room.rs:65-67`
  - body: The Control Room invokes `gateway.restart`, which is not part of the advertised current gateway method surface. This primary recovery action will fail with an unknown-method error instead of restarting the gateway.
  - confidence: 0.86
