Full review comments:

- **[P1] Advertise the current gateway protocol:** `apps/linux-gtk/crates/gateway-client/src/protocol.rs:218`
  - body: Current main requires gateway protocol 4, but this client advertises protocol 3. As written, a current gateway can reject the companion during handshake before any UI workflow is usable.
  - confidence: 0.98
- **[P1] Send raw config text to config.set:** `apps/linux-gtk/crates/app/src/views/config.rs:222-226`
  - body: `config.set` validates `{ raw, baseHash? }` with no extra properties, but the Save button sends `{ path, value }`. Config edits will be rejected instead of persisted.
  - confidence: 0.97
- **[P1] Use supported skills RPC payloads:** `apps/linux-gtk/crates/app/src/views/skills.rs:123-127`
  - body: The current skills schemas do not accept `skillId` for either `skills.update` or `skills.install`; toggles and installs will fail validation and leave the UI out of sync.
  - confidence: 0.96
- **[P1] Wrap cron enablement inside patch:** `apps/linux-gtk/crates/app/src/views/cron.rs:327-331`
  - body: `cron.update` requires an id/jobId plus a `patch` object, but this sends `enabled` at the top level after optimistically flipping the chip. The gateway will reject the toggle while the UI shows the new state.
  - confidence: 0.95
- **[P2] Call channels.status with supported parameters:** `apps/linux-gtk/crates/app/src/views/channels.rs:126-128`
  - body: The current channel status schema accepts `probe`, `timeoutMs`, and optional `channel`, not `channelId`. The Probe button therefore sends an invalid request instead of running the requested health check.
  - confidence: 0.92
- **[P2] Read pending pair requests from pending:** `apps/linux-gtk/crates/app/src/views/instances.rs:96-100`
  - body: `node.pair.list` returns `{ pending, paired }`, but this view reads `requests`. Real pending approvals will render as empty, hiding actionable pairing requests from operators.
  - confidence: 0.9
- **[P2] Use the safe restart RPC name:** `apps/linux-gtk/crates/app/src/views/control_room.rs:65-67`
  - body: Current main exposes `gateway.restart.request` and `gateway.restart.preflight`, not `gateway.restart`. The Control Room Restart button will fail with an unknown method instead of requesting a safe restart.
  - confidence: 0.9
- **[P2] Do not default settings to TLS for local gateway:** `apps/linux-gtk/crates/app/src/views/settings.rs:55-58`
  - body: The Settings view still pre-fills `wss://127.0.0.1:18789` even though the local gateway default is plain `ws://127.0.0.1:18789`. Saving the untouched value can turn a working local setup into a TLS connection failure.
  - confidence: 0.88
