Full review comments:

- **[P1] Define a desktop client identity:** `apps/linux-gtk/crates/gateway-client/src/protocol.rs:221-228`
  - body: This bundled desktop app currently handshakes as `cli`/`cli` while requesting broad operator scopes. Current main has a closed client-id registry with app-specific ids but no Linux desktop id, so this would make policy, diagnostics, token approval, and auth treatment indistinguishable from the CLI; add an approved Linux client id/mode and gateway tests before merging.
  - confidence: 0.93
- **[P1] Send raw config text to config.set:** `apps/linux-gtk/crates/app/src/views/config.rs:222-226`
  - body: `config.set` validates `{ raw, baseHash? }`, but this save path sends `{ path, value }`. The gateway rejects that request as invalid, so users can edit JSON in the Config view but cannot persist it through Save.
  - confidence: 0.97
- **[P1] Use current skills request fields:** `apps/linux-gtk/crates/app/src/views/skills.rs:123-163`
  - body: The skills contract does not accept `skillId`: updates require `skillKey`, and installs require either `{ name, installId }`, ClawHub `{ source, slug }`, or upload fields. These toggle/install requests will be rejected, so skill management in the app will not apply.
  - confidence: 0.95
- **[P1] Wrap cron enablement in patch:** `apps/linux-gtk/crates/app/src/views/cron.rs:327-331`
  - body: `cron.update` requires changes under a `patch` object, but this toggle sends `enabled` at the top level. The request fails validation while the UI updates optimistically, leaving users with a displayed cron state that does not match the gateway.
  - confidence: 0.95
- **[P1] Persist connection settings instead of env-only state:** `apps/linux-gtk/crates/app/src/views/settings.rs:221-236`
  - body: The Save & Reconnect path only mutates process environment variables, so URL/token changes are lost on real restart even though the UI says settings were saved. Persist to the canonical config/settings path or make this an explicitly temporary test action.
  - confidence: 0.9
- **[P2] Pass supported channel status params:** `apps/linux-gtk/crates/app/src/views/channels.rs:126-128`
  - body: `channels.status` accepts `probe`, `timeoutMs`, and optional `channel`; it does not accept `channelId`. The Probe button therefore sends an invalid request instead of checking channel health.
  - confidence: 0.92
