Overall confidence: 0.96

Full review comments:

- **[P1] Advertise the current gateway protocol:** `apps/linux-gtk/crates/gateway-client/src/protocol.rs:218`
  - body: Current main requires protocol 4 and closes mismatched WebSocket handshakes, but this app sends protocol 3 for both `minProtocol` and `maxProtocol`. As written, the companion cannot connect to a current gateway.
  - confidence: 0.98
- **[P1] Send raw config text to config.set:** `apps/linux-gtk/crates/app/src/views/config.rs:222-226`
  - body: `config.set` validates only `{ raw, baseHash? }` with no extra properties, but this sends `{ path, value }`. The Save button will get an invalid-params response instead of persisting the edited config.
  - confidence: 0.97
- **[P1] Use skillKey for skills.update:** `apps/linux-gtk/crates/app/src/views/skills.rs:123-127`
  - body: The current `skills.update` schema accepts `skillKey` or a ClawHub update shape, not `skillId`. Toggling a skill will be rejected by the gateway and leave the UI out of sync.
  - confidence: 0.96
- **[P1] Send a supported skills.install payload:** `apps/linux-gtk/crates/app/src/views/skills.rs:160-163`
  - body: `skills.install` requires a legacy `{ name, installId }`, ClawHub `{ source: "clawhub", slug }`, or upload payload; a bare `skillId` is invalid. The Install button cannot install skills against current main.
  - confidence: 0.95
- **[P1] Wrap cron updates in a patch object:** `apps/linux-gtk/crates/app/src/views/cron.rs:327-331`
  - body: `cron.update` expects the job id plus `patch: { ... }`, but this sends `enabled` at the top level. Enabling or disabling a cron job will be rejected by current gateway validation.
  - confidence: 0.95
- **[P2] Use the channels.status probe contract:** `apps/linux-gtk/crates/app/src/views/channels.rs:126-128`
  - body: `channels.status` accepts `channel`, `probe`, and `timeoutMs`, with additional properties disabled; this sends `channelId`. The Probe action will be rejected instead of refreshing that channel.
  - confidence: 0.92
- **[P2] Read pending pair requests from pending:** `apps/linux-gtk/crates/app/src/views/instances.rs:96-100`
  - body: Current node pairing lists expose `pending` and `paired`, but this view reads `requests`. Pending pairing requests will render as empty even when the gateway returns them.
  - confidence: 0.9
- **[P2] Call the supported gateway restart method:** `apps/linux-gtk/crates/app/src/views/control_room.rs:65-67`
  - body: Current main registers `gateway.restart.request` and `gateway.restart.preflight`, not `gateway.restart`. The Control Room restart button will call an unknown method.
  - confidence: 0.93
