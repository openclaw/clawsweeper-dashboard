## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Read the nested gateway auth token:** `apps/linux-gtk/crates/gateway-client/src/config.rs:21-25`
  - body: The config reader only deserializes `gateway.token`, but current OpenClaw config stores shared gateway auth under `gateway.auth.token` or `gateway.auth.password`. Normal generated config will not provide the token this client expects unless users also set an env var.
  - confidence: 0.93
- **[P2] Send supported config RPC payloads:** `apps/linux-gtk/crates/app/src/views/config.rs:137-138`
  - body: `config.get` is sent with `{ path: "" }` and `config.set` with `{ path, value }`, but current validators accept empty get params and `{ raw, baseHash? }` for save. The Config view load/save path will fail validation against current main.
  - confidence: 0.95
- **[P2] Build cron mutations with current schemas:** `apps/linux-gtk/crates/app/src/views/cron.rs:147-151`
  - body: The create and enablement flows send older top-level cron fields, while current `cron.add` and `cron.update` require structured add fields and a `patch` object. These actions can fail while the UI has already updated optimistically.
  - confidence: 0.94
- **[P2] Use current skills RPC identifiers:** `apps/linux-gtk/crates/app/src/views/skills.rs:123-127`
  - body: The Skills view sends `skillId` to `skills.update` and `skills.install`, but current schemas require `skillKey` for updates and install identifiers or ClawHub/upload source params for installs. Skill toggles and installs will be rejected.
  - confidence: 0.94
- **[P2] Read node pair requests from pending:** `apps/linux-gtk/crates/app/src/views/instances.rs:96-100`
  - body: `node.pair.list` returns `{ pending, paired }`, but the Instances view reads `payload.requests`. Real pending node approvals will be hidden, so operators cannot approve or reject them from this screen.
  - confidence: 0.93
- **[P2] Call a registered restart method:** `apps/linux-gtk/crates/app/src/views/control_room.rs:65-67`
  - body: The Control Room restart button calls `gateway.restart`, but current main registers `gateway.restart.preflight` and `gateway.restart.request`. Clicking Restart will return an unknown-method error instead of requesting a restart.
  - confidence: 0.9
- **[P2] Persist connection settings durably:** `apps/linux-gtk/crates/app/src/views/settings.rs:219-236`
  - body: Save & Reconnect mutates only process environment variables and then tells users settings are saved for restart. Those values disappear when the app exits, so URL/token changes are not actually durable.
  - confidence: 0.88
- **[P2] Bound the gateway handshake wait:** `apps/linux-gtk/crates/gateway-client/src/client.rs:258-260`
  - body: The client waits indefinitely for `connect.challenge` after a WebSocket connection is established. A misrouted or stalled endpoint that accepts WebSockets but never sends the OpenClaw challenge can hang this connection attempt instead of returning to reconnect/backoff.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The patch adds a credentialed native gateway client and the PR itself leaves TLS, device identity, token persistence, and packaging review open.

Concerns:

- **[medium] Gate the TLS verification bypass:** `apps/linux-gtk/crates/gateway-client/src/client.rs:234`
  - body: When `OPENCLAW_TLS_ACCEPT_INVALID` is set, the Rust client installs a verifier that accepts any certificate for `wss://` URLs. Before shipping a credentialed desktop client, this needs explicit dev-only UX, documentation, and tests so tokens are not sent through an unintended MITMable path.
  - confidence: 0.86
- **[medium] Finish token persistence review:** `apps/linux-gtk/crates/app/src/views/settings.rs:219`
  - body: The Settings view writes gateway tokens only into process environment variables while the PR marks token persistence as a known security gap. The merge path should align with OpenClaw's credential/config semantics instead of leaving ad hoc app-local token handling.
  - confidence: 0.82
