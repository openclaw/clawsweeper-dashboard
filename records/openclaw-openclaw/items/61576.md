Overall confidence: 0.93

Full review comments:

- **[P1] Send raw config payloads to config.set:** `apps/linux-gtk/crates/app/src/views/config.rs:222-225`
  - body: `config.get` takes an empty object and `config.set` accepts `{ raw, baseHash? }`, but this view sends `{ path: "" }` for load and `{ path, value }` for save. With current main validation, the config editor cannot load or persist changes.
  - confidence: 0.96
- **[P1] Use skillKey and valid install params:** `apps/linux-gtk/crates/app/src/views/skills.rs:123-127`
  - body: `skills.update` requires `skillKey`, and `skills.install` requires either `{ name, installId }` or a source-specific install payload. Sending `skillId` makes skill enablement and install actions fail validation.
  - confidence: 0.95
- **[P1] Wrap cron enablement in the patch payload:** `apps/linux-gtk/crates/app/src/views/cron.rs:327-330`
  - body: `cron.update` validates `{ id|jobId, patch }`, but this toggle sends `enabled` at the top level. The request is rejected while the UI has already flipped the status chip, leaving users with misleading cron state.
  - confidence: 0.95
- **[P2] Call channels.status with supported probe params:** `apps/linux-gtk/crates/app/src/views/channels.rs:126-128`
  - body: `channels.status` allows `channel`, `probe`, and `timeoutMs`; this sends `channelId`, so the Probe button returns an invalid-request error instead of checking channel health.
  - confidence: 0.91
- **[P2] Read pending pair requests from pending:** `apps/linux-gtk/crates/app/src/views/instances.rs:96-100`
  - body: `node.pair.list` returns the node-pairing list with pending entries, but this code reads `payload.requests`. Real pending pair requests will be treated as empty, hiding approvals operators need to handle.
  - confidence: 0.88
- **[P2] Call the supported restart request method:** `apps/linux-gtk/crates/app/src/views/control_room.rs:66`
  - body: Current main exposes `gateway.restart.request` and `gateway.restart.preflight`, not `gateway.restart`. The Restart button will report an unknown-method failure instead of scheduling the safe restart path.
  - confidence: 0.9
