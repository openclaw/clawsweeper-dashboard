## What This Changes

The PR adds optional `filter`, `sort`, `field_names`, and `automatic_fields` parameters to `feishu_bitable_list_records` and forwards them to the Lark list API.

## Best Possible Solution

Land a corrected Feishu plugin update that uses the list API's sort string contract or translates structured input, adds focused forwarding coverage, and includes redacted real behavior proof.

## Reproduction Assessment

Yes for the review finding: source inspection of the PR schema and forwarding path, compared with the locked Lark SDK types, shows object-array `sort` belongs to `appTableRecord.search` while list-records accepts a sort string. The broader feature still lacks real Feishu behavior proof.

## Solution Assessment

No. Passing supported list query params through the Feishu plugin is the right narrow direction, but the sort schema must match the list API or translate structured input before forwarding.
