## Summary

Keep open. Current main still lacks Telegram custom emoji reaction support, but this branch remains incorrect for the advertised custom-only action path, targets files that have moved on main, is currently conflicting, and has no real Telegram proof.

## What This Changes

The PR adds Telegram custom emoji reaction handling for inbound `message_reaction` updates and outbound react action/send calls, with bot and send tests.

## Best Possible Solution

Port the reaction-only feature into current Telegram plugin surfaces: add `customEmojiId` to the shared reaction tool schema and runtime parser, send Telegram `custom_emoji` reaction payloads, emit inbound custom reaction events after existing gates, update docs/tests, and attach redacted real Telegram proof.

## Reproduction Assessment

Yes for the PR defect: source inspection shows `customEmojiId` is parsed after the required `emoji` parser, so a custom-only action fails before the new path can run. The requested behavior itself is a new feature, not a broken existing contract on current main.

## Solution Assessment

No. The feature direction is valid, but this branch is not the best mergeable solution until it is ported to current Telegram runtime/schema files, fixes custom-only validation, and supplies real Telegram proof.
