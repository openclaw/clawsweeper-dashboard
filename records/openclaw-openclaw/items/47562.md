## Summary

Keep open. Current main still lacks Telegram custom emoji reaction support, and the PR's feature direction is useful, but the branch is not mergeable as-is because custom-only action calls still fail validation, the runtime surfaces moved on current main, and real Telegram behavior proof is absent.

## What This Changes

The PR adds inbound and outbound Telegram custom emoji reaction handling plus bot and send tests.

## Best Possible Solution

Port the reaction-only feature into the current Telegram plugin surfaces, add customEmojiId to the shared message-tool schema and action runtime, preserve existing authorization/reaction gates, document the semantics, and include focused tests plus redacted real Telegram proof.

## Reproduction Assessment

Yes, source-level for the PR defect. On the PR diff, a custom-only react action reaches readReactionParams before customEmojiId is read, and that helper still requires the emoji key.

## Solution Assessment

No. The feature direction is useful, but this branch is not the best mergeable solution until it is ported to current main's action-runtime/bot-handlers surfaces, fixes custom-only validation, updates the shared schema/docs, and supplies real Telegram proof.
