## Summary

Keep open. Current main still lacks Telegram custom emoji reaction support, and this PR is useful source work but is not merge-ready because it targets legacy files and still blocks the advertised custom-only action call before `customEmojiId` is read.

## What This Changes

The PR adds Telegram custom emoji reactions for inbound `message_reaction` events and outbound reaction sends/actions, plus bot and send tests.

## Best Possible Solution

Port the feature into the current Telegram plugin surfaces with shared schema, docs, changelog, and focused tests, preserving existing authorization and reaction-notification gates while avoiding deleted legacy files.

## Reproduction Assessment

Yes. Static inspection of current main gives a high-confidence reproduction path: custom-only action input is not accepted, outbound sends cannot build `custom_emoji` payloads, and inbound updates with only `type: "custom_emoji"` are filtered out before enqueue.

## Solution Assessment

No. The feature direction is valid, but this PR is stale against current files and incomplete for the custom-only action/schema path; a narrow replacement or port is safer than merging this branch as-is.
