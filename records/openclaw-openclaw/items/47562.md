## Summary

Keep open. The feature is valid and current main still lacks Telegram custom emoji reaction support, but this PR is stale against renamed Telegram files, still has a custom-only action validation bug, and only includes test/build proof rather than real Telegram behavior proof.

## What This Changes

The branch adds Telegram custom emoji reaction support for inbound `message_reaction` events and outbound Telegram reaction sends/actions, plus bot and send tests.

## Best Possible Solution

Port the narrow reaction feature to the current Telegram plugin surfaces, add `customEmojiId` to the shared message-tool schema/action runtime, keep the existing authorization and reaction notification gates, document reaction semantics, and add focused send/action/inbound/schema coverage plus real Telegram proof.
