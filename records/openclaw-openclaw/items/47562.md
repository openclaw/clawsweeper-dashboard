Full review comments:

- **[P1] Permit customEmojiId without requiring emoji:** `src/agents/tools/telegram-actions.ts:147`
  - body: The new parameter is read only after `readReactionParams`, and that helper still requires the `emoji` key before `customEmojiId` can be used. That tells the agent a failed custom emoji add was a removal, so the warning result needs the same custom-add metadata as the success path.
  - confidence: 0.84
