## Summary

Keep open: the fix is directionally useful, but the gateway raw/canonical session-key matching remains asymmetric and the external PR still lacks proof of a real active turn being stopped.

## What This Changes

The PR adds a shared stop-abort helper, broadens gateway stop handling across raw/canonical session keys, persists stopped partials canonically, and adds focused regression tests.

## Best Possible Solution

Keep the shared active-run helper, make gateway stop matching symmetric across raw and canonical session keys without weakening requester authorization, and land with focused tests plus redacted live stop proof.

## Reproduction Assessment

Yes, at source level: current main registers active gateway chat abort controllers under the raw requested key and session abort matching is exact. The related Telegram report gives concrete `/stop` steps, but I did not run a live provider turn in this read-only review.

## Solution Assessment

No. The auto-reply active-run helper is a reasonable part of the fix, but the gateway alias handling should be symmetric and the PR still needs after-fix real behavior proof.
