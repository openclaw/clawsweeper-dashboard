## Summary

Keep open. Current `main` still indexes memory-wiki lint targets too narrowly, and open PR #73722 is already paired to this issue with closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Request memory-wiki lint to resolve Obsidian wikilinks by title, slug, fragment-stripped target, path/basename, and imported source-path suffix instead of flagging valid pages as broken.

## Best Possible Solution

Review and land PR #73722, or an equivalent focused memory-wiki lint resolver and regression-test change, while leaving broader markdown-aware extractor work to #70986.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: a page titled `Alpha Database` at `syntheses/alpha-db.md` linked as `[[Alpha Database#Evidence]]` reaches lint with the fragment intact, while current valid targets only include path and basename forms.

## Solution Assessment

Yes. The narrow maintainable fix is to normalize lint target comparison using existing page metadata and focused regression tests; broader markdown-aware extraction belongs in #70986.
