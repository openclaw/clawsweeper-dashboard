## Summary

Keep open. Current main still has the memory-wiki lint resolver gap, and open PR #73722 is already paired to this issue as the focused implementation candidate, so closing or queuing duplicate repair work would be premature.

## What This Changes

The issue asks for `openclaw wiki lint` to resolve existing memory-wiki pages when Obsidian links use page titles, slug variants, fragments, or imported source-path suffixes.

## Best Possible Solution

Land PR #73722 or an equivalent memory-wiki-owned change that normalizes lint lookup keys for titles, slugs, fragments, and imported source-path variants, with focused regression tests and without rewriting imported evidence; keep broader markdown-aware extraction work tracked separately under #70986.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: a page at `syntheses/alpha-db.md` with title `Alpha Database` linked as `[[Alpha Database#Evidence]]` will be extracted as `Alpha Database#Evidence` and compared against only `syntheses/alpha-db` and `alpha-db`, producing `broken-wikilink`.

## Solution Assessment

Yes. Normalizing targets inside the memory-wiki lint resolver is the narrowest maintainable fix for this report; rewriting imported evidence would damage provenance, and the broader Markdown parser false-positive work belongs in #70986.
