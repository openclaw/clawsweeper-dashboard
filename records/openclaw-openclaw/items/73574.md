## Summary

Keep open. Current main and the latest release still use the narrow memory-wiki lint resolver that compares raw link targets only against extensionless relative paths and basenames, and the issue already has open closing PR #73722 as the implementation candidate.

## What This Changes

Request to expand memory-wiki lint wikilink resolution so title, slugified, fragment-stripped, path/basename, and imported source-path suffix variants resolve without rewriting imported evidence.

## Best Possible Solution

Land PR #73722, or an equivalent memory-wiki-owned fix, that expands resolver lookup normalization, strips fragments and queries consistently, preserves imported provenance, and adds focused regression coverage while leaving broader markdown-aware extraction work to #70986.

## Reproduction Assessment

Yes. A source-level reproduction is clear: a page stored as `syntheses/alpha-db.md` with title `Alpha Database` and linked as `[[Alpha Database#Evidence]]` will not match the current `validTargets` entries `syntheses/alpha-db` and `alpha-db`.

## Solution Assessment

Yes. The narrow maintainable solution is resolver-side normalization with colocated lint tests; rewriting imported evidence would damage provenance and is not the right fix.
