## Summary

Keep open. Current main still reports valid memory-wiki Obsidian links as broken when they use titles, fragments, slug variants, or imported source-path forms, and open PR #73722 is already paired to this issue with closing syntax.

## What This Changes

The issue requests expanding memory-wiki lint wikilink resolution to accept title, slug, fragment-stripped, and source-path variants for imported Obsidian-style links.

## Best Possible Solution

Review and land PR #73722, or an equivalent focused memory-wiki lint resolver and regression-test change, while leaving broader markdown-aware extractor work to #70986.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: a page titled Alpha Database at syntheses/alpha-db.md linked as [[Alpha Database#Evidence]] reaches lint with the fragment intact, while validTargets only contains path and basename forms.

## Solution Assessment

Yes. The narrow maintainable fix is to normalize lint target comparison using existing page metadata and focused regression tests; broader markdown-aware extraction belongs in #70986.
