## Summary

Keep open. Current main still compares memory-wiki lint targets too narrowly, and open PR #73722 is already paired to this issue with closing syntax, so closing before that PR merges would be premature.

## What This Changes

The issue asks memory-wiki lint to resolve Obsidian wikilinks by titles, slug variants, fragment-stripped targets, and imported source-path suffixes without rewriting provenance-bearing source evidence.

## Best Possible Solution

Review and land PR #73722, or an equivalent focused memory-wiki change, while leaving broader markdown-aware extractor work to #70986.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: a page titled `Alpha Database` at `syntheses/alpha-db.md` linked as `[[Alpha Database#Evidence]]` is extracted with the fragment, while current lint only accepts `syntheses/alpha-db` and `alpha-db`.

## Solution Assessment

Yes. The narrow maintainable solution is to normalize lint target comparison using existing page metadata and regression tests; rewriting imported evidence would weaken provenance.
