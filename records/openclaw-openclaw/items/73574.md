## Summary

Keep open. Current main still only indexes path and basename link targets and keeps Obsidian fragments intact, while open PR #73722 is already the paired implementation candidate and should land or be replaced before this issue closes.

## What This Changes

Request that `openclaw wiki lint` normalize Obsidian wikilinks by page title, slug/path variants, stripped fragments, and imported source-path suffixes before reporting `broken-wikilink`.

## Best Possible Solution

Land or replace #73722 with focused memory-wiki lint resolver normalization and regression tests, while leaving broader markdown-aware extractor work to #70986.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: a page titled `Alpha Database` at `syntheses/alpha-db.md` linked as `[[Alpha Database#Evidence]]` reaches lint as `Alpha Database#Evidence`, while current main only accepts path and basename forms.

## Solution Assessment

Yes. The narrow maintainable fix is to normalize lint target lookup using existing page metadata and strip fragment/query suffixes before comparison, with focused coverage in `extensions/memory-wiki/src/lint.test.ts`.
