## What This Changes

Request to make memory-wiki lint resolve Obsidian links by title, slug/path/basename, fragment-stripped target, and imported source-path variants without rewriting imported evidence.

## Best Possible Solution

Land PR #73722 or an equivalent focused memory-wiki lint change that expands lookup keys from existing page metadata, strips query and fragment suffixes before comparison, preserves imported provenance, and adds regression coverage for the reported link variants while leaving broader markdown parsing work to #70986.
