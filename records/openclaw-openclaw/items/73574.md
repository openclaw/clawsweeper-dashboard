## Summary

Keep open. Current main still compares memory-wiki lint targets too narrowly, and open PR #73722 is already paired to this issue with closing syntax, so cleanup should wait for that PR or an equivalent fix to land.

## What This Changes

Request that memory-wiki lint resolve Obsidian links by page title, slug variants, fragment-stripped targets, path/basename variants, and imported source-path suffixes.

## Best Possible Solution

Review and land PR #73722, or an equivalent focused memory-wiki lint resolver change, while leaving broader markdown-aware extractor work to #70986.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: a page titled `Alpha Database` at `syntheses/alpha-db.md` linked as `[[Alpha Database#Evidence]]` reaches lint with the fragment intact, while current valid targets only include path and basename forms.

## Solution Assessment

Yes. The narrow maintainable fix is to normalize lint target comparison using existing page metadata and focused regression tests; rewriting imported evidence would weaken provenance.
