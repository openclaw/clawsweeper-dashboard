Keep open. Current main still compares raw memory-wiki lint targets against only path and basename forms, and the open paired PR has not landed yet.

## What This Changes

Expand memory-wiki lint so title, slug, fragment-stripped, path, basename, and imported source-path Obsidian link variants resolve to existing pages.

## Best Possible Solution

Land PR #73722, or an equivalent focused memory-wiki lint resolver and regression-test change, while leaving broader markdown-aware extraction work to #70986.

## Solution Assessment

Yes. The narrow maintainable fix is to normalize lint target comparison using existing page metadata and focused regression tests; broader markdown-aware extraction belongs in #70986.
