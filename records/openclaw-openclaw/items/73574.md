## Summary

Keep open. Current main still has the narrow memory-wiki lint resolver, and open PR #73722 already targets this issue with closing syntax, so cleanup should wait for that PR or an equivalent fix to land.

## What This Changes

The issue requests memory-wiki lint resolver normalization so title-style, slug, fragment-stripped, and source-path-derived Obsidian wikilinks resolve without rewriting imported evidence.

## Best Possible Solution

Review and land PR #73722, or an equivalent focused memory-wiki lint resolver change with regression coverage; leave broader markdown-aware extractor work to #70986.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: a page titled `Alpha Database` at `syntheses/alpha-db.md` linked as `[[Alpha Database#Evidence]]` reaches lint with the fragment intact while current valid targets only contain path and basename forms.

## Solution Assessment

Yes. The narrow maintainable solution is to normalize lint target comparison using existing page metadata and focused regression tests, rather than rewriting imported evidence or broadening this issue into the extractor work tracked by #70986.
