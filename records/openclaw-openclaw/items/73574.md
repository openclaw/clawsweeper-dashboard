## Summary

Keep open. Current main still has the memory-wiki lint resolver gap: it indexes only extensionless relative paths and basenames, while Obsidian link extraction preserves title-style targets and fragments. The issue is also paired with open closing PR #73722, so the right cleanup path is to review or land that PR rather than close this issue early.

## What This Changes

The issue requests that memory-wiki lint normalize Obsidian wikilink targets by title, slug, path/basename, fragment-stripped form, and imported source-path suffixes before reporting broken-wikilink warnings.

## Best Possible Solution

Land PR #73722 or an equivalent focused memory-wiki change that expands lint lookup keys, strips fragments and query strings before comparison, preserves imported provenance, and adds regression coverage for the reported link variants while leaving broader markdown parsing work to #70986.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: a page at syntheses/alpha-db.md titled Alpha Database linked as [[Alpha Database#Evidence]] will be extracted as Alpha Database#Evidence, but current lint only accepts syntheses/alpha-db and alpha-db.

## Solution Assessment

Yes. The narrow maintainable solution is to normalize target comparison inside the memory-wiki lint resolver using existing page metadata and focused tests; rewriting imported evidence would damage provenance and should not be the fix.
