Keep open. Current main still has the reported memory-wiki lint resolver gap, and open PR #73722 is already paired to this issue with closing syntax, so the maintainer path is to review, land, or close that PR rather than close this issue early.

## What This Changes

The issue requests that `openclaw wiki lint` resolve Obsidian wikilinks by page title, slug variants, fragment-stripped targets, path/basename variants, and imported source-path suffixes.

## Best Possible Solution

Land PR #73722 or an equivalent focused memory-wiki change that expands lint lookup keys, strips fragments and query strings before comparison, preserves imported provenance, and adds regression coverage for the reported link variants while leaving broader markdown-awareness work to #70986.
