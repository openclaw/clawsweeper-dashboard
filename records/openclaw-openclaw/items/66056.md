## Summary

Keep open. Current main still has English-only runtime failure copy in the sanitizer and reply-runner paths, no `messages` locale setting, and no structured error-key/locale contract for plugins to localize these messages safely.

## What This Changes

Request to add locale-aware user-facing runtime error messages with English fallback and configurable locale selection.

## Best Possible Solution

Define a stable runtime localization contract with keyed user-facing error identifiers, English fallback, documented locale resolution, sanitizer and reply-runner wiring, and focused locale/fallback tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/API request. Source inspection gives high-confidence proof that current main still emits English-only runtime failure text and lacks a runtime locale contract.

## Solution Assessment

Unclear. The proposed lightweight JSON catalog is plausible, but the best maintainable solution depends on maintainer choices around config naming, locale resolution, translation ownership, and plugin versus core responsibility.
