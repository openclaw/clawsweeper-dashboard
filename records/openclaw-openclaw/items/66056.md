Keep open. Current main still has English-only runtime failure text and no locale input or message-locale config; this is a real feature/API request that needs maintainer product direction, not conservative cleanup.

Add locale-aware user-facing error messages for runtime failures such as rate limits, context overflow, billing, timeouts, transport errors, and missing provider keys.

## Best Possible Solution

Define stable user-facing error keys with English fallback, document locale resolution, wire sanitizer and reply-runner failure paths through that contract, and add focused config, fallback, and non-English regression coverage.
