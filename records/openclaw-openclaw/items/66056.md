## Summary

Keep open. Current main still has English-only runtime failure text in the sanitizer and reply-runner paths, and solving this needs a new localization/config/API direction rather than conservative cleanup.

## What This Changes

Request locale-aware user-facing runtime error messages with English fallback and a configurable locale or message catalog.

## Best Possible Solution

Define stable runtime error keys with English fallback, document locale resolution, wire the sanitizer and reply-runner failure paths through it, and add focused translated-output plus fallback tests.

## Reproduction Assessment

Not applicable as a normal bug reproduction because this is a feature/API request. Source and test inspection provide high-confidence verification that current main still emits English-only runtime failure text and has no locale-aware config or sanitizer API.

## Solution Assessment

Unclear. The proposed lightweight JSON catalog is plausible, but maintainers first need to decide the stable key contract, locale source, config name, and core-versus-plugin ownership model.
