  - body: This treats the entire `thinkingSignature` string as opaque. Preserve the ciphertext/signature bytes without bypassing redaction for plaintext JSON siblings.
  - confidence: 0.88
- **[P2] Do not strip valid encrypted replay without proof:** `src/llm/providers/openai-chatgpt-responses.ts:485`
  - body: Passing `replayEncryptedReasoningContent: false` makes every native ChatGPT request discard prior encrypted reasoning before dispatch. The checked upstream Codex path still sends `store:false` with `reasoning.encrypted_content` while skipping item ids, and the linked issue evidence says clean blobs can replay; keep this to corrupted/invalid retry handling or land it only as an explicit, live-proven provider policy.
  - confidence: 0.78
