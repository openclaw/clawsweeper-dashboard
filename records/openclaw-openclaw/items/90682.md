## Summary

Keep open: the PR addresses a real OpenAI/ChatGPT Responses replay and transcript-corruption problem, but the submitted patch is not merge-ready because it can bypass transcript secret redaction inside whole JSON-stringified reasoning signatures and it makes an unproven provider-policy change by dropping native encrypted reasoning replay everywhere.

## What This Changes

The PR exempts provider reasoning fields from transcript redaction and adds an option to omit encrypted reasoning replay for native ChatGPT Responses request construction and websocket continuations.

## Best Possible Solution

Preserve opaque ciphertext/signature bytes without exempting plaintext JSON siblings from redaction, and land native replay stripping only as an explicit provider policy backed by live two-turn proof and sibling custom-backend coverage.

## Reproduction Assessment

Yes, with source-level confidence and linked live release reports: current main stores full reasoning items in `thinkingSignature`, redacts every string field, and sends `encrypted_content` on native store-disabled replay. I did not run a live current-main ChatGPT/Codex session in this read-only review.

## Solution Assessment

No. Preserving opaque bytes is the right direction, but exempting the whole signature string and blanket-disabling native encrypted replay is broader than the evidence currently supports.
