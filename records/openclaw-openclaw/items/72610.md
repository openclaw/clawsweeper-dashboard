  - body: Hermes YAML-derived provider ids, MCP names, and nested config keys are merged into plain objects without blocking `__proto__`, `constructor`, or `prototype`.   - confidence: 0.9

  - body: `mergeConfig` redaction runs regexes over a serialized action and `report.md` writes reasons verbatim, so nested `authorization`, `cookie`, arbitrary token, URL credential, or reason text can be persisted in reports. Redact by walking action/result objects with key-aware secret handling before JSON or Markdown output.
  - confidence: 0.91

  - body: Apply uses plan-provided targets and archive paths directly, and archive joins `action.archivePath` under the report dir without rejecting absolute or traversal paths. Validate all write destinations under the intended workspace/state/report roots and harden source/destination symlink handling before copying imported state.
  - confidence: 0.89

  - body: The importer copies `command`, `args`, `env`, and `cwd` from Hermes MCP config into live `mcp.servers`, which can make imported state auto-runnable later. Treat stdio MCP entries as manual/quarantined until the user explicitly reviews and enables the command.
  - confidence: 0.87
