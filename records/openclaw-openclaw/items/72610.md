Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use the shipped plugin migration path instead of core wiring:** `src/migrations/registry.ts:4`
  - body: Current main already ships a plugin-owned Hermes migration provider and documents `openclaw migrate` as provider-plugin based. Registering a hard-coded core Hermes provider here creates a second owner for the same import behavior and conflicts with the repository boundary that optional provider-style capability should live behind plugin seams when those seams suffice.
  - confidence: 0.92
- **[P1] Block untrusted config keys before merging:** `src/migrations/config-merge.ts:29`
  - body: Hermes config/provider/MCP names are untrusted import input, but the merge helper recursively assigns arbitrary keys into normal objects. A crafted import can write `__proto__`, `constructor`, or `prototype` keys while building OpenClaw config; reject dangerous keys or use a safer existing migration config path before merging.
  - confidence: 0.86
- **[P1] Require explicit confirmation for non-interactive apply:** `src/cli/program/register.migrate.ts:128`
  - body: This guard only refuses missing `--yes` when stdout is a TTY, so CI, cron, or piped runs can apply a real migration without explicit confirmation. Migration apply mutates config/workspace state, so non-dry-run apply must require `--yes` regardless of stdout.
  - confidence: 0.9
- **[P1] Quarantine executable MCP imports:** `src/migrations/providers/hermes.ts:336-344`
  - body: The Hermes importer copies `command`, `args`, `env`, and `cwd` from untrusted Hermes MCP definitions into `mcp.servers`. Those entries can later spawn local stdio MCP processes, so executable server definitions need manual review or an explicitly disabled/quarantined import path.
  - confidence: 0.84
- **[P2] Redact migration output structurally:** `src/migrations/report.ts:12`
  - body: `mergeConfig` actions are redacted by stringifying the whole action and applying a small set of token regexes, which misses arbitrary nested secret-looking keys and report reasons. Migration plans/reports are persisted to disk, so redact recursively by key and apply the same redaction to human-readable report fields.
  - confidence: 0.78
- **[P1] Confine migration file writes before copying:** `src/migrations/apply.ts:49`
  - body: The apply path copies source files/trees and archive entries using action-controlled paths without a containment check or symlink-safe copy routine. A migration provider or crafted source state can escape the intended report/workspace/state roots or copy linked files; resolve targets under explicit bases and reject symlinks before writing.
  - confidence: 0.8
