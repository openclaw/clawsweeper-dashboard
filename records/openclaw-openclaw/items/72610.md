  - body: Current main already resolves migration providers through the plugin registry and the Hermes importer is declared by the bundled `migrate-hermes` plugin.   - body: Hermes YAML-derived provider, MCP, and skill names flow into object assignment without rejecting `__proto__`, `constructor`, or `prototype`. Migration sources are untrusted input, so this helper must structurally reject prototype-polluting keys before writing config.
  - confidence: 0.88
- **[P1] Require explicit confirmation for non-interactive apply:** `src/cli/program/register.migrate.ts:129-130`
  - body: `migrate apply` only refuses a missing `--yes` when stdout is a TTY, so scripts and CI can run a non-dry-run migration without an explicit confirmation flag. Require `--yes` for every non-dry-run non-interactive apply, with any interactive prompt handled separately.
  - confidence: 0.86
- **[P1] Redact migration reports structurally:** `src/migrations/report.ts:13-16`
  - body: `mergeConfig` actions are redacted by stringifying the whole action and applying value regexes, while report reasons are written directly. Nested secret-looking keys such as authorization headers, cookies, or arbitrary token fields can remain in `plan.json` or `report.md`; redact recursively by key and apply the same redaction to human-readable report fields.
  - confidence: 0.82
- **[P1] Quarantine executable MCP server definitions:** `src/migrations/providers/hermes.ts:544-551`
  - body: The Hermes importer merges MCP server definitions, including `command`, `args`, `env`, and `cwd`, directly into live OpenClaw config. Imported stdio commands should be manual-review or disabled/quarantined until the user explicitly trusts them, otherwise an imported agent home can persist commands that run later.
  - confidence: 0.82
