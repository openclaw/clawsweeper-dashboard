  - body: This guard only refuses a missing `--yes` when stdout is a TTY, so scripts or CI can run a non-dry-run migration without explicit confirmation. Require `--yes` for all non-dry-run non-interactive applies, with any interactive prompt handled separately.

Nested secret-looking keys such as authorization headers, cookies, or arbitrary token fields can remain in `plan.json` or `report.md`; redact recursively by key and apply the same redaction to human-readable report fields.
