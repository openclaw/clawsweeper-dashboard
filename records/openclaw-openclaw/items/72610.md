## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Use the plugin-owned migration provider path:** `src/migrations/registry.ts:4`
  - body: Current main already resolves migration providers through plugin manifest contracts and includes the `migrate-hermes` plugin. Adding this hard-coded core registry creates a second source of truth for Hermes imports, so behavior, docs, safety checks, and future fixes can drift between core and the plugin-owned path.
  - confidence: 0.91
- **[P1] Require --yes outside dry runs:** `src/cli/program/register.migrate.ts:129`
  - body: This guard only refuses an apply when stdout is a TTY. In CI, pipes, cron, or other non-TTY contexts, `openclaw migrate apply` can proceed without `--yes` even though it copies files and writes config/secrets; require explicit confirmation for every non-dry-run apply.
  - confidence: 0.94
- **[P1] Reject unsafe config keys before merging:** `src/migrations/config-merge.ts:28-29`
  - body: Hermes YAML-derived provider IDs, MCP names, and config object keys flow into this merge helper, but the helper writes keys like `__proto__`, `constructor`, or `prototype` into plain objects. That can pollute prototypes or corrupt config; reject dangerous keys or use a hardened merge before accepting untrusted import data.
  - confidence: 0.88
- **[P1] Redact migration reports structurally:** `src/migrations/report.ts:13-15`
  - body: Stringifying a config action and running token regexes misses many secret-shaped nested fields, and `report.md` later writes item reasons verbatim. Migration plans and reports can persist credentials from imported provider, MCP, or plugin config unless redaction walks objects by key and redacts all report text fields.
  - confidence: 0.89
- **[P1] Constrain archive and copy targets before writing:** `src/migrations/apply.ts:50`
  - body: The apply path trusts action-controlled source, target, and archive paths and then writes with `copyFile`, `cp`, or `path.join(reportDir, archivePath)`. A malformed or hostile migration plan can write outside the intended workspace/report roots or follow symlinks; apply should enforce allowed base directories and reject symlink/traversal cases.
  - confidence: 0.87
- **[P1] Quarantine executable MCP imports:** `src/migrations/providers/hermes.ts:544-552`
  - body: The Hermes mapper copies MCP `command`, `args`, `env`, and `cwd` into runtime config as a normal merge action. Importing an untrusted Hermes home can persist executable stdio MCP definitions that later run locally; executable entries should become manual/quarantined review items unless the user explicitly enables them.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The proposed core importer handles untrusted local files, config patches, secrets, and executable MCP definitions, so the branch needs security hardening or retirement before merge.

Concerns:

- **[high] Prototype-polluting config keys are not rejected:** `src/migrations/config-merge.ts:28`
  - body: `mergeConfigObjectValue` recursively assigns untrusted keys from imported config into plain objects without blocking `__proto__`, `constructor`, or `prototype`, which can corrupt object prototypes or config state.
  - confidence: 0.88
- **[high] Migration reports can leak nested secrets:** `src/migrations/report.ts:13`
  - body: Redaction only regexes serialized config actions and does not structurally redact nested key names or markdown report reasons, so provider, MCP, plugin, or token values can be persisted in report artifacts.
  - confidence: 0.89
- **[high] File writes are not confined to safe roots:** `src/migrations/apply.ts:50`
  - body: The apply code trusts migration action paths for file copies and archive targets, so traversal, absolute archive paths, and symlink-following source or target paths can escape intended directories.
  - confidence: 0.87

  - body: Hermes MCP entries with `command`, `args`, `env`, or `cwd` are merged into OpenClaw config rather than quarantined for manual review, allowing imported state to become later local command execution.
