## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Use the plugin-owned migration provider path:** `src/migrations/registry.ts:4`
  - body: Current main already routes `openclaw migrate` through plugin-registered providers, and the shipped Hermes provider declares `migrationProviders: ["hermes"]` in the plugin manifest. Adding a separate core `builtInMigrationProviders` list creates competing CLI/docs/security behavior for the same import path instead of using the existing provider seam.
  - confidence: 0.93
- **[P1] Require --yes outside dry runs:** `src/cli/program/register.migrate.ts:129`
  - body: This guard only refuses non-dry-run apply when `stdout` is a TTY, so a script or CI job with non-TTY output can apply a migration without `--yes`. Current main’s migrate command requires `--yes` for non-interactive apply, so this branch would reintroduce a confirmation bypass on a command that writes files, config, and secrets.
  - confidence: 0.96
- **[P1] Reject unsafe config keys before merging:** `src/migrations/config-merge.ts:28-29`
  - body: Hermes YAML-derived provider ids, MCP server names, and nested config keys flow into `mergeConfigObjectValue`, but this loop writes every key into ordinary objects without blocking `__proto__`, `constructor`, or `prototype`. That lets imported config mutate object prototypes or poison later config lookups before the generated OpenClaw config is written.
  - confidence: 0.9
- **[P1] Redact migration reports structurally:** `src/migrations/report.ts:13-14`
  - body: Redacting a `mergeConfig` action by stringifying and applying token regexes misses key-based secrets such as `authorization`, `cookie`, `password`, or arbitrary nested API tokens; `report.md` also writes result reasons verbatim. Migration reports are meant to be shareable support artifacts, so this can persist imported credentials in `plan.json` or `report.md`.
  - confidence: 0.88
- **[P1] Constrain archive and copy targets before writing:** `src/migrations/apply.ts:222-223`
  - body: `archiveOnly` joins the plan-controlled archive path under `reportDir` and then copies the source after a `stat`, while copy actions write their target paths directly. The apply layer needs to resolve writes under approved state/workspace/report roots and reject symlinks/traversal, because migration plans are built from imported local state and can otherwise write or copy outside the intended report tree.
  - confidence: 0.86
- **[P1] Quarantine executable MCP imports:** `src/migrations/providers/hermes.ts:330-342`
  - body: This copies `command`, `args`, `env`, and `cwd` from Hermes MCP definitions into live OpenClaw `mcp.servers` config. Importing an untrusted agent home should archive or mark executable stdio servers for manual review instead of making them runnable by later MCP discovery.
  - confidence: 0.9
