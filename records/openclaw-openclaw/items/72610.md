Full review comments:

- **[P1] Use the plugin-owned migration provider path:** `src/migrations/registry.ts:4`
  - body: Current main already resolves migration providers through plugin registry contracts and ships Hermes under `extensions/migrate-hermes`. Adding a second hard-coded core Hermes registry would create competing CLI/docs/security behavior for the same import path, so this branch should be retired or narrowed to the plugin-owned path.
  - confidence: 0.93
- **[P1] Require --yes before every non-dry-run apply:** `src/cli/program/register.migrate.ts:129`
  - body: This guard only refuses missing `--yes` when stdout is a TTY, so `openclaw migrate apply` can mutate state in CI, cron, or piped output without explicit confirmation. Current main rejects non-interactive apply without `--yes`; this branch should keep that safety invariant.
  - confidence: 0.95
- **[P1] Reject unsafe config keys before merging:** `src/migrations/config-merge.ts:28-29`
  - body: Hermes YAML-derived provider IDs, MCP names, and nested config keys flow into these object assignments. Without blocking `__proto__`, `constructor`, and `prototype` or using null-prototype objects, a migration plan can prototype-pollute config objects during merge.
  - confidence: 0.88
- **[P1] Redact migration reports structurally:** `src/migrations/report.ts:13-15`
  - body: The plan redaction only regexes a stringified `mergeConfig` action and leaves other action details plus report reasons largely untouched. Credentials in nested headers/env/plugin config or non-matching token formats can be written to `plan.json` or `report.md`.
  - confidence: 0.86
- **[P1] Quarantine executable MCP imports:** `src/migrations/providers/hermes.ts:330-340`
  - body: The Hermes importer copies `command`, `args`, `env`, and `cwd` into live `mcp.servers`, which can later start local stdio processes. Imported executable MCP definitions should become manual/quarantined review items unless the user explicitly approves each command.
  - confidence: 0.87
- **[P1] Constrain migration copy and archive targets:** `src/migrations/apply.ts:221-229`
  - body: The apply path writes `action.target` and `action.archivePath` directly and uses `stat`/`copyFile`/`cp` without enforcing a safe base or rejecting symlinks. A malicious or compromised source plan can escape the intended report/workspace/state directories or copy linked files unexpectedly.
  - confidence: 0.84
