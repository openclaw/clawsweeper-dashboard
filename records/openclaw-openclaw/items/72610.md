Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use the plugin-owned migration provider path:** `src/migrations/registry.ts:1-5`
  - body: Current main already resolves migration providers through plugin manifest contracts and ships Hermes as `extensions/migrate-hermes`. Adding a separate core `src/migrations` registry would give operators two Hermes migration implementations with different CLI shape, docs, defaults, and safety behavior; keep Hermes-owned logic in the plugin path instead.
  - confidence: 0.93
- **[P1] Require `--yes` outside dry runs:** `src/cli/program/register.migrate.ts:129-133`
  - body: This guard only refuses when `process.stdout.isTTY` is true, so the same `openclaw migrate apply` invocation can mutate config, files, and `.env` values without `--yes` in CI, cron, or piped-output runs. Require `--yes` for every non-dry-run apply unless an actual interactive confirmation is completed.
  - confidence: 0.96
- **[P1] Reject unsafe config keys before merging:** `src/migrations/config-merge.ts:27-30`
  - body: Hermes provider IDs, MCP server names, and skill keys come from imported YAML and are merged into plain objects without blocking `__proto__`, `constructor`, or `prototype`. A malicious source can therefore pollute object prototypes during migration planning/apply; validate keys or use safe prototype-less merge targets before writing config.
  - confidence: 0.91
- **[P1] Redact migration reports structurally:** `src/migrations/report.ts:13-16`
  - body: `mergeConfig` redaction serializes the action and applies token regexes, while `report.md` writes `entry.reason` without key-aware redaction. Secrets in nested fields such as headers, cookies, arbitrary tokens, or provider config can remain in `plan.json` or reports; redact recursively by key and apply it to report text fields too.
  - confidence: 0.9
- **[P1] Constrain migration writes to owned targets:** `src/migrations/apply.ts:44-60`
  - body: The apply path trusts `action.target` and `action.archivePath`, then copies files or trees without confining writes to the migration report, state, or workspace roots and without symlink-safe copy semantics. Provider-controlled or raced paths can write or archive outside the intended migration area; resolve under an allowed base and reject symlinks/special files before copying.
  - confidence: 0.9
- **[P1] Quarantine executable MCP imports:** `src/migrations/providers/hermes.ts:330-343`
  - body: The importer copies `command`, `args`, `env`, and `cwd` from Hermes config directly into live `mcp.servers`. That can persist an executable stdio server from an untrusted source directory and have OpenClaw run it later, so executable MCP entries should be manual, disabled, or explicitly confirmed before becoming runtime config.
  - confidence: 0.88

## Security Review

Status: needs_attention
