  - body: Current main already resolves migration providers through plugin contracts and includes a bundled `migrate-hermes` provider.   - confidence: 0.95

  - body: This guard only blocks when `process.stdout.isTTY` is true, so a scripted or CI invocation can apply a migration without `--yes`. The current plugin-owned command rejects non-interactive apply without `--yes`; this branch should require explicit confirmation for every non-dry-run apply path.
  - confidence: 0.96
- **[P1] Reject unsafe config keys before merging:** `src/migrations/config-merge.ts:27-29`
  - body: Hermes YAML-derived provider, MCP, and skill keys flow into config merge objects, but this merge writes arbitrary `Object.entries()` keys into plain objects. Keys such as `__proto__`, `constructor`, or `prototype` need to be rejected or handled with null-prototype maps before any config write.
  - confidence: 0.9
- **[P1] Redact migration reports structurally:** `src/migrations/report.ts:13-15`
  - body: The report redaction only stringifies `mergeConfig` actions and applies token regexes, while markdown report reasons are written separately. That can leave nested secret-looking keys, headers, env values, or error text in persisted migration output; redact object structures and all report fields before writing.
  - confidence: 0.89
- **[P1] Quarantine executable MCP imports:** `src/migrations/providers/hermes.ts:330-343`
  - body: The importer copies MCP `command`, `args`, `env`, and `cwd` fields into live OpenClaw config. Imported agent homes are untrusted migration input, so stdio MCP definitions should become manual-review items or require explicit per-server opt-in before they can later execute local commands.
  - confidence: 0.9

## Security Review

Status: needs_attention

  - body: Untrusted Hermes-derived object keys are merged into plain objects without blocking `__proto__`, `constructor`, or `prototype`, which can corrupt the config object graph during migration.

  - body: The apply path copies plan-controlled sources and targets and archives with `path.join(reportDir, action.archivePath)` without a root/symlink confinement layer, which is risky for untrusted migration trees.
