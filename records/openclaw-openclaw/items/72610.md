## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use the plugin-owned migration provider path:** `src/migrations/registry.ts:4`
  - body: This hard-codes Hermes into a new core migration registry. Current main already resolves migration providers from plugin manifests/runtime and includes `extensions/migrate-hermes`, so merging this would create two Hermes import implementations with diverging CLI, docs, security, and maintenance behavior.
  - confidence: 0.94
- **[P1] Require --yes before every non-dry-run apply:** `src/cli/program/register.migrate.ts:129`
  - body: This guard only rejects missing `--yes` when stdout is a TTY. In non-interactive scripts or CI, `openclaw migrate apply` can mutate files, config, and env state without explicit confirmation; require `--yes` for all non-dry-run applies or perform a real interactive prompt.
  - confidence: 0.95
- **[P1] Reject unsafe config keys before merging:** `src/migrations/config-merge.ts:28-29`
  - body: Hermes YAML-derived provider IDs, MCP server names, and skill keys flow into plain object assignment without blocking `__proto__`, `constructor`, or `prototype`. That lets imported config data pollute object prototypes during migration planning or apply.
  - confidence: 0.9
- **[P1] Redact migration reports structurally:** `src/migrations/report.ts:13-15`
  - body: This only regex-redacts a JSON string for `mergeConfig` actions and writes report reasons without redaction. Short/custom tokens in nested MCP env, headers, plugin config, or reason text can be persisted in `plan.json` or `report.md`; use recursive key-aware redaction like the current migration SDK path.
  - confidence: 0.9
- **[P1] Quarantine executable MCP imports:** `src/migrations/providers/hermes.ts:330-347`
  - body: The importer copies `command`, `args`, `env`, and `cwd` from Hermes config into live `mcp.servers`. A compromised or untrusted source home can persist stdio commands that OpenClaw later executes, so executable MCP definitions should be manual/quarantined or explicitly approved.
  - confidence: 0.9
- **[P1] Constrain archive and copy targets before writing:** `src/migrations/apply.ts:221-229`
  - body: `archiveOnly` joins a plan-controlled archive path into the report directory and copy actions trust plan-controlled source/target paths. Apply should enforce base-directory containment and symlink-safe copies so a malicious or buggy provider cannot write or copy outside the intended migration targets.
  - confidence: 0.84
