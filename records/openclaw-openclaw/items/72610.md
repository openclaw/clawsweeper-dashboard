Overall confidence: 0.9

Full review comments:

- **[P1] Require --yes outside TTYs too:** `src/cli/program/register.migrate.ts:129`
  - body: This guard only blocks missing `--yes` when stdout is a TTY, so `openclaw migrate apply` can mutate config, env, workspace files, and reports from CI or piped scripts without explicit confirmation. The apply path should require `--yes` for any non-dry-run noninteractive apply, matching the current main behavior.
  - confidence: 0.93
- **[P1] Quarantine executable MCP imports:** `src/migrations/providers/hermes.ts:546-552`
  - body: The Hermes importer merges source `mcp_servers` directly into live `mcp.servers`, including `command`, `args`, `env`, and `cwd`. Current MCP projection treats command-bearing servers as stdio launch config, so an imported Hermes home can persist local commands that run later; command-bearing entries should be manual or disabled/quarantined until explicitly trusted.
  - confidence: 0.88
- **[P1] Redact migration output structurally:** `src/migrations/report.ts:13-14`
  - body: Redacting `mergeConfig` by regexing `JSON.stringify(action)` misses arbitrary token-shaped values under keys like headers/env and `writeMigrationReport` writes reasons without redaction. Migration reports and JSON plans are support/share artifacts, so action details and report fields need recursive key-aware redaction before writing.
  - confidence: 0.86
