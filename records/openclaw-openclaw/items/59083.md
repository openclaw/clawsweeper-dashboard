## Summary

Keep open. The underlying privacy bug is still real on current main for Windows-style path strings, but this PR patches the old local OpenAI-compatible audio upload path; current main now routes uploads through the shared form-data resolver, so the useful fix needs to move there before merge.

## What This Changes

The PR adds a local OpenAI-compatible audio upload filename resolver plus unit coverage for POSIX host paths and missing runtime filenames.

## Best Possible Solution

Update `resolveAudioTranscriptionUploadFileName` in `src/media-understanding/shared.ts` to strip POSIX and Windows path segments before preserving the existing missing-name fallback and AAC/M4A normalization.

## Reproduction Assessment

Yes for the remaining path-leak behavior: on current main, a Windows-style filename remains unchanged through the shared resolver on POSIX hosts. The missing-runtime-filename crash no longer reproduces because the current shared resolver already falls back to `audio`.

## Solution Assessment

No, not as currently placed. The basename algorithm is the right direction, but the maintainable fix belongs in the shared upload filename resolver that current main and SDK-facing callers use.
