Keep this PR open, but do not merge the current head: the layered input bounds and shape checks are useful, while the new limiter bypasses OpenClaw’s existing serialized-attempt contract and therefore allows concurrent forged handshakes to race past the configured threshold. The PR bounds device-handshake public keys and signatures, rejects implausible Ed25519 inputs before crypto, adds a per-IP pre-auth signature-verification limiter, and includes gateway tests plus a WebSocket flood PoC.

## Best Possible Solution

Wrap the complete device-signature admission, crypto verification, and final failure/reset accounting in the existing per-IP/per-scope serialized-attempt seam, add concurrent and shared-IP regression coverage, and demonstrate the current head against the real running-gateway WebSocket flood path.
