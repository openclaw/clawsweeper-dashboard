## Summary

Keep open. The PR targets a real pre-auth Gateway CPU-DoS path, but latest head still leaves unauthenticated paths that can force crypto work after the new bucket should be locked.

## What This Changes

The PR adds device handshake field bounds, device identity shape pre-checks, a device-signature auth rate-limit scope in WebSocket connect handling, regression tests, a PoC script, and a changelog entry.

## Best Possible Solution

Keep the layered hardening, but move the signature gate before public-key derivation and clear it only after the complete connect handshake is authorized.

## Reproduction Assessment

Yes. Source inspection on current main shows a pre-auth device connect can reach public-key derivation and v3/v2 signature verification before full auth; the PR body also includes a PoC, but I did not run it here.

## Solution Assessment

No. The mitigation direction is right, but the limiter is placed too late for PEM public keys and is reset before full authentication succeeds.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Move the signature limiter before PEM parsing:** `src/gateway/server/ws-connection/message-handler.ts:780-784`
  - body: This gate still runs after `deriveDeviceIdFromPublicKey(device.publicKey)`. For PEM inputs, that helper can reach `crypto.createPublicKey` before a locked-out IP is rejected, so an attacker can keep forcing key parsing even after the new bucket is closed. Move the check ahead of device-id derivation or make pre-derivation PEM handling non-crypto, and cover the PEM lockout case.
  - confidence: 0.9
- **[P2] Reset the signature bucket only after auth succeeds:** `src/gateway/server/ws-connection/message-handler.ts:824`
  - body: The new bucket is reset as soon as the device signature is valid, before shared auth, device-token auth, or pairing authorization has succeeded. A client can generate its own keypair, send valid signatures, still fail auth, and avoid accumulating failures while forcing pre-auth signature work. Clear this scope only after the whole connect handshake is authorized.
  - confidence: 0.84
