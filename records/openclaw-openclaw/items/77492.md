Full review comments:

- **[P1] Serialize device-signature accounting through auth success:** `src/gateway/server/ws-connection/message-handler.ts:1017-1020`
  - body: The new gate records a device-signature failure for every device-bearing handshake before the async auth decision, but resets only after `resolveConnectAuthDecision` returns `authOk`. With a low `maxAttempts`, a burst of valid device-token or bootstrap handshakes from the same IP can fill the bucket before the first success resets it, so later valid clients are rejected as rate-limited. Wrap the full device-signature attempt in the existing serializer or otherwise keep success/reset atomic with the pessimistic reservation.
