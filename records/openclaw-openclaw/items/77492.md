## Summary

Keep open: the mitigation is well-scoped and source-backed, but the PR still resets the new signature bucket before final device admission, and the supplied live proof does not exercise the real WS gateway path.

## What This Changes

The branch adds device handshake length caps, pre-crypto key/signature shape checks, a per-IP device-signature rate-limit gate, regression tests, a PoC script, and a changelog entry.

## Best Possible Solution

Keep the layered mitigation, move the signature bucket reset after successful device admission or registration, and add real WS/gateway after-fix proof plus regression coverage for shared-auth rejection paths.

## Reproduction Assessment

Yes, source-reproducible. Current main derives the submitted public key and runs v3/v2 signature verification before final auth/pairing admission; I did not run the gateway in this read-only review.

## Solution Assessment

No. The schema bounds, shape pre-checks, and pre-crypto gate are the right direction, but the reset should move after final device admission and the proof should cover the real WS path.
