## Summary

This PR should stay open: the hardening is useful, but the proposed rate-limit bucket is bypassable because any unauthenticated client can clear it with a valid self-signed device signature before the connect is actually authorized.

## What This Changes

The PR adds device handshake length caps, device-identity shape pre-checks, a new per-IP device-signature rate-limit scope, regression tests, a PoC script, and a changelog entry.

## Best Possible Solution

Keep the length caps and cheap shape checks, but make the device-signature limiter survive until a connect is actually authorized or paired, with a regression test for an attacker interleaving forged signatures with valid self-signed unpaired device signatures.

## Reproduction Assessment

Yes. Current main’s source shows unauthenticated connect frames can reach public-key derivation and v3/v2 signature verification before final authorization; the PR’s own tests cover the forged-signature path, and the reset bypass is source-reproducible by interleaving forged failures with a valid self-signed but unauthorized device signature.

## Solution Assessment

No. The caps and pre-checks are good pieces, but resetting the device-signature bucket on signature validity alone is not the safest fix because that proof is available to any remote client before authorization.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Do not reset the signature bucket before auth succeeds:** `src/gateway/server/ws-connection/message-handler.ts:821`
  - body: This reset runs as soon as the device signature verifies, before `resolveConnectAuthDecision` decides whether the client is actually authorized or paired. A remote client can generate its own Ed25519 device key, sign the challenge correctly, clear the per-IP `device-signature` failures, and then keep sending forged signatures without staying locked out. Move this reset until after the connect is accepted, or avoid resetting this pre-auth failure bucket.
  - confidence: 0.91
