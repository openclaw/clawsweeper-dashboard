## What This Changes

The issue reports that `chunkTextByBreakResolver` trims loop-emitted chunks but leaves trailing whitespace on the final residual chunk.

## Best Possible Solution

Review and land, revise, or close the linked fix PR; after it merges, close this issue as implemented by that PR.

## Reproduction Assessment

Yes. Source inspection shows over-limit text ending in whitespace can exit the loop with a trailing-space `remaining` tail, and current main pushes that tail without `trimEnd()`.

## Solution Assessment

Yes. The narrow maintainable fix is to trim and skip the final residual chunk consistently with the loop branch, and the linked PR also covers the Matrix sibling copy before merge.
