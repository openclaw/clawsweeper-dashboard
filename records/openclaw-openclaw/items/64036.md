## Summary

Keep this open: current main still has the untrimmed final residual chunk behavior, and open draft PR #72491 already tracks the exact fix with closing syntax, so the issue should remain open until that PR or an equivalent patch lands.

## What This Changes

Normalize `chunkTextByBreakResolver` so the final residual chunk is trimmed like loop-emitted chunks and skipped when empty.

## Best Possible Solution

Land PR #72491 or an equivalent narrow patch that trims the final residual chunk, skips it when empty, and keeps the shared helper, Matrix runtime copy, guardrail, and focused regression tests aligned.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes. The source path is deterministic: an over-limit input that leaves `remaining` as a whitespace-suffixed final chunk exits the loop and reaches `chunks.push(remaining)` without `trimEnd()`.

## Solution Assessment

Is this the best way to solve the issue? Yes. Normalizing only the final residual chunk to match loop-emitted chunks is the narrowest maintainable fix; broader channel or delivery changes are unnecessary.
