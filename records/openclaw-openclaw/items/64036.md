## Reproduction Assessment

Yes from source inspection: over-limit text can exit the loop with a trailing-space `remaining` tail, and current main pushes that tail without `trimEnd()`. I did not run tests because this review had to keep the checkout read-only.

## Solution Assessment

Yes: the narrow maintainable fix is to trim and skip the final residual chunk consistently with the in-loop branch, while keeping the Matrix sibling copy aligned before merge.
