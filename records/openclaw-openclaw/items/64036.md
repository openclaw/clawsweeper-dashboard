## What This Changes

The issue reports that `chunkTextByBreakResolver` leaves trailing whitespace on the final residual chunk even though loop-emitted chunks are trimmed.

## Best Possible Solution

Review, refresh, and land https://github.com/openclaw/openclaw/pull/72491 or an equivalent narrow patch, then close this issue after the fix reaches `main`.

## Reproduction Assessment

Yes. Source inspection on current `main` gives a deterministic path: over-limit input such as `a b ` with limit `2` can leave `remaining` as `b `, and the final branch pushes that value unchanged.

## Solution Assessment

Yes. Trimming and skipping only the final residual chunk is the narrow maintainable fix, but the issue should stay open while the closing PR remains unmerged.
