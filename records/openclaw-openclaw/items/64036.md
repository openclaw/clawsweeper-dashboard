## Summary

Keep open: current main and the latest release still push the final remaining text chunk unchanged, while https://github.com/openclaw/openclaw/pull/72491 is an open draft PR with closing syntax for this exact bug.

## What This Changes

The issue requests trimming and skipping the final residual chunk from `chunkTextByBreakResolver` so final chunks do not retain trailing whitespace.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/72491 or an equivalent narrow patch that trims the final residual chunk, skips it when empty, and preserves the documented Markdown and Matrix edge cases.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes. Source inspection gives a deterministic path: an over-limit input like `a b ` with limit 2 can leave `remaining` as `b `, and current main pushes that value without `trimEnd()`.

## Solution Assessment

Is this the best way to solve the issue? Yes. Trimming the final residual chunk before pushing, while skipping empty final chunks and preserving the Markdown newline exception covered by the active PR, is the narrow maintainable fix.
