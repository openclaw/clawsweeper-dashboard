## Summary

Keep open. Current main still routes stateful `openclaw googlemeet` CLI session commands through a CLI-process-local `GoogleMeetRuntime`; the #72441 gateway-owned fix applies to agent tool actions and gateway methods, so the reported empty registry after CLI exit remains source-backed.

## What This Changes

Ensure Google Meet realtime sessions started or joined from CLI commands remain gateway-owned after the command exits.

## Best Possible Solution

Route stateful Google Meet CLI session lifecycle commands through the registered gateway methods, while keeping non-session OAuth/API/export helpers local.

## Reproduction Assessment

Yes from source inspection, though not from a live Meet run here. Current main creates/listens to CLI sessions in an in-process `GoogleMeetRuntime` map, so `test-speech` or `join` can return a session that a later CLI process cannot list via `status`.

## Solution Assessment

No, the current CLI runtime path is not the best way to solve the issue. The maintainable fix is to reuse the already-registered `googlemeet.*` gateway methods for stateful CLI commands, matching the #72441 tool-action route.

## Repair Work Prompt

Fix the Google Meet CLI session ownership bug reported in openclaw/openclaw#76344. Stateful `openclaw googlemeet` commands that create, join, inspect, speak to, or leave realtime sessions should go through the gateway-owned `googlemeet.*` methods instead of a CLI-process-local `GoogleMeetRuntime`, so sessions survive after the initiating CLI command exits. Keep OAuth login/doctor, artifact/attendance/export, and other non-session API helpers local unless they truly need gateway runtime state. Prefer reusing the existing `callGatewayFromCli`/gateway-method mapping already used by the `google_meet` agent tool, including appropriate long join/test timeouts and structured error details. Add focused regression coverage proving CLI `join`/`test-speech`/`status`/`speak`/`leave` dispatch through gateway RPC and no longer call `ensureRuntime` directly for stateful session lifecycle operations. Add a single-line user-facing changelog fix entry; do not change bundled plugin boundaries or unrelated Google Meet OAuth/artifact behavior.
