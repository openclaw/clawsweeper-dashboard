## Summary

Keep open. Current main still routes stateful `openclaw googlemeet` CLI session commands through a CLI-process-local `GoogleMeetRuntime`, while the related #72441 fix routes only agent tool actions through the gateway-owned runtime.

## What This Changes

Requests that stateful `openclaw googlemeet` CLI session commands use the gateway-owned Google Meet runtime so sessions survive after the starting CLI process exits.

## Best Possible Solution

Move the stateful Google Meet CLI lifecycle commands onto the existing `googlemeet.*` gateway RPC methods while leaving OAuth, calendar, artifact, and other non-session helpers on their current local/API paths.

## Reproduction Assessment

Yes from source inspection, though not from a live Meet run here. A `join` or `test-speech` CLI process writes the session to its own `GoogleMeetRuntime` map, while a later `status` or `speak` CLI process constructs or queries a separate map.

## Solution Assessment

Yes. Reusing the already-registered `googlemeet.*` gateway methods is the narrow maintainable fix because the gateway runtime is the shared owner #72441 already uses for agent tool actions.

## Repair Work Prompt

Fix the Google Meet CLI session lifecycle bug from https://github.com/openclaw/openclaw/issues/76344. On current main, stateful `openclaw googlemeet` CLI commands (`join`, `test-speech`, `test-listen` if it creates/query live sessions, `status`, `doctor` for session status, `recover-tab`, `setup` if gateway-owned node/runtime state is required, `leave`, and `speak`) call a CLI-process-local `GoogleMeetRuntime`; move the session lifecycle/addressing commands to the existing `googlemeet.*` gateway RPC methods so CLI follow-up commands share the gateway-owned session registry, matching the #72441 agent tool path. Preserve local/API-only commands such as OAuth, resolve/preflight, calendar/artifacts/attendance/export, and Meet API conference ending unless they need live session state. Keep the fix inside the Google Meet plugin/SDK-supported gateway RPC surface; do not add core owner-specific behavior or a new config option. Add/update CLI regression tests that assert stateful commands call gateway RPC rather than an injected local runtime, keep existing output/error shapes as much as possible, and add a user-facing changelog fix entry without bot/maintainer-only attribution.
