## Summary

Keep open: the PR addresses a real credential-migration bug, but the config-last ordering creates an inverse failure window for plans whose auth-store refs depend on provider aliases introduced in the same config write.

## What This Changes

The branch reorders `runSecretsApply` write mode so satellite files and per-agent SQLite auth stores commit before `openclaw.json`, and adds a regression test for auth-store commit failure.

## Best Possible Solution

Use a staged design that completes fault-prone satellite/auth-store preparation before touching live state, commits config/provider declarations before live store promotion, then promotes stores with regression coverage for config-write failure and same-plan `providerUpserts`.

## Reproduction Assessment

Yes. Source inspection and the PR's terminal proof show current main can commit config before later auth-store writes; I did not rerun the fault harness in this read-only review.

## Solution Assessment

No. The patch fixes the reported auth-store-failure order but moves `replaceConfigFile` after live store commits, which is not the safest fix when same-plan provider aliases live in config.
