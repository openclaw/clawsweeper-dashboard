## Summary

Keep open: the PR addresses a real `secrets apply` half-migration bug and includes credible terminal before/after proof, but the config-last ordering is not merge-ready because auth-store SecretRefs can be committed before the config provider aliases they depend on. This is also a security-owned credential persistence path that still needs secops review.

## What This Changes

The PR reorders `runSecretsApply` so satellite files and per-agent auth profile stores are committed before `openclaw.json`, and adds a regression test for an auth-store commit fault.

## Best Possible Solution

Use staged satellite/auth-store preparation, commit `openclaw.json` before promoting live store changes, add a providerUpserts plus auth-profile config-failure regression, then get secops approval.

## Reproduction Assessment

Yes. Source inspection and the PR's terminal proof show current main can commit config before later auth-store writes; I did not rerun the fault harness in this read-only review.

## Solution Assessment

No. The patch fixes the reported auth-store-failure order but makes config the last live write, which is not the safest shape when same-plan provider aliases live only in the pending config.
