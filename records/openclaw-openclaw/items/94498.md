## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve config CAS for the final write:** `src/secrets/apply.ts:939-943`
  - body: In the provider-changed path, Phase B calls `replaceConfigFile` with only `nextConfig` and `io`, so it re-reads the current file and treats that snapshot as the base for a `nextConfig` projected before Phase 0. If another config writer changes `openclaw.json` after Phase 0 but before this call, the previous one-shot path would fail because it wrote against `projected.configSnapshot`; this new path can silently overwrite that concurrent edit.
  - confidence: 0.86
- **[P1] Do not delete providers before dependent refs move:** `src/secrets/apply.ts:915-924`
  - body: `providerMutationConfig` includes `providerDeletes`, and this Phase 0 write commits it before any auth-store or target migration. Because the code then re-snapshots that partial state for rollback, a later auth-store failure can leave existing persisted SecretRefs pointing at an alias that was already removed, even when config rollback itself succeeds.
  - confidence: 0.8
