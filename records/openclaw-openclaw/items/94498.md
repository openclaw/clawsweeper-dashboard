## Summary

Keep open: this PR is the current candidate fix for the linked `secrets apply` half-migration bug, and the code/proof now target the current per-agent SQLite auth-store path. I found no discrete line-level correctness bug in the latest patch, but the credential source-of-truth write-order semantics still need secops/maintainer acceptance before merge.

## What This Changes

This PR reorders `secrets apply` writes to precommit provider additions, write satellite/auth stores, then commit the final config, with regression tests for auth-store faults, provider alias ordering, provider deletes, and config compare-and-set.

## Best Possible Solution

Land this dependency-ordered mitigation after secops accepts the residual fault windows, or replace it with a broader journaled transaction design if owners require true cross-store atomicity.
