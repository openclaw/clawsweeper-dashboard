Full review comments:

- **[P1] Reject symlinked transcripts before opening:** `src/agents/cli-runner/session-history.ts:46-55`
  - body: The new helper uses `statSync()` and then opens the resolved transcript path, but `stat` follows symlinks. A symlink inside the sessions directory can redirect hook-history loading to another local file; preserve current main's `lstat` plus realpath containment hardening before any rebase.
  - confidence: 0.9
- **[P1] Gate conversation hooks before exposing raw prompts:** `src/agents/cli-runner.ts:64-73`
  - body: This path sends the system prompt, user prompt, transcript history, and assistant output to lifecycle hooks without current main's non-bundled plugin conversation-access gate. That would re-expose raw conversation contents to plugins that have not explicitly opted in.
  - confidence: 0.88
- **[P2] Honor the configured session store:** `src/agents/cli-runner/session-history.ts:32-34`
  - body: `resolveSessionFilePathOptions` receives only the agent id here, so deployments using `config.session.store` can silently lose prior CLI history in hook payloads. Keep current main's `storePath: params.config?.session?.store` behavior.
  - confidence: 0.84
- **[P2] Load transcript history only when hooks need it:** `src/agents/cli-runner.ts:57-63`
  - body: `runPreparedCliAgent` parses session history before checking active hooks, adding synchronous transcript I/O to the default CLI hot path. Current main avoids that by loading history only for hooks that consume it.
  - confidence: 0.8
- **[P2] Skip llm_output when there is no assistant text:** `src/agents/cli-runner.ts:96-109`
  - body: `executeCliAttempt` emits `llm_output` even when the CLI returns no assistant text. Current main and the embedded path avoid sending an output hook with no text content, which hook consumers may not expect.
  - confidence: 0.78
