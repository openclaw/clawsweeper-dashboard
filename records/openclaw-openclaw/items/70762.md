## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Reject symlinked transcripts before opening them:** `src/agents/cli-runner/session-history.ts:50-54`
  - body: `statSync()` follows symlinks, and the following `SessionManager.open(sessionFile)` then reads the symlink target. A user or local process that can write under the OpenClaw state sessions directory can point this at an arbitrary file or blocking special file, and the hook payload path will read or hang before returning history.
  - confidence: 0.87
- **[P1] Pass the configured session store into history resolution:** `src/agents/cli-runner/session-history.ts:32-34`
  - body: `resolveSessionFilePathOptions()` is called with only `agentId`, so configs that put `session.store` outside the default state sessions directory resolve the provided `sessionFile` as out of scope. Those users silently get empty hook history even though the transcript exists in their configured store.
  - confidence: 0.9
- **[P2] Skip transcript loading when no hook can consume it:** `src/agents/cli-runner.ts:57-63`
  - body: `loadCliSessionHistoryMessages()` runs before checking whether any lifecycle hooks are registered. That adds synchronous file I/O and transcript parsing to default CLI runs, which is exactly the hot agent path this scoped guide asks us to keep runtime-free when the data is not needed.
  - confidence: 0.82
- **[P2] Do not emit llm_output for empty CLI output:** `src/agents/cli-runner.ts:133-144`
  - body: This call fires even when `assistantText` is empty, so hook consumers receive `assistantTexts: []` and no `lastAssistant`. The embedded runner and current main only emit `llm_output` for real assistant content, so this branch would leave CLI hooks with a different empty-output contract.
  - confidence: 0.8
