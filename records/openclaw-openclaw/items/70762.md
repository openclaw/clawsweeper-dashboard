## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Block symlinked transcripts before opening history:** `src/agents/cli-runner/session-history.ts:50-54`
  - body: `fs.statSync()` follows symlinks, and the next `SessionManager.open(sessionFile)` reads that target. A symlink planted under the sessions directory can redirect hook-history loading to another local file or special file; reject symlinks and verify the real path stays inside the sessions directory before opening it.
  - confidence: 0.9
- **[P1] Honor custom session stores when resolving transcripts:** `src/agents/cli-runner/session-history.ts:32-34`
  - body: `resolveSessionFilePathOptions` is called with only the agent id, so deployments using `config.session.store` resolve against the default state sessions directory instead of the configured store. That makes hook history disappear for existing custom-store users; pass the configured store path through the resolver.
  - confidence: 0.88
- **[P1] Gate CLI conversation hooks before exposing prompts:** `src/agents/cli-runner.ts:64-72`
  - body: This new CLI hook path sends `systemPrompt`, the user prompt, and transcript history into lifecycle hooks. Without the current `allowConversationAccess` policy for non-bundled plugins, enabling a plugin hook can expose raw conversation content that was not previously sent from CLI runs.
  - confidence: 0.84
- **[P2] Skip history reads when hooks do not need them:** `src/agents/cli-runner.ts:57-63`
  - body: `runPreparedCliAgent` loads and parses transcript history before checking whether any lifecycle hook needs it. That adds synchronous file I/O and JSONL parsing to the default CLI hot path; only load history when `llm_input` or `agent_end` hooks are active.
  - confidence: 0.86
- **[P2] Suppress empty llm_output events:** `src/agents/cli-runner.ts:133-144`
  - body: `runAgentHarnessLlmOutputHook` fires even when the CLI returns only whitespace, producing an `llm_output` event with no assistant text and no `lastAssistant`. The embedded runner only emits this hook for real assistant output, so guard this call on non-empty text.
  - confidence: 0.8
