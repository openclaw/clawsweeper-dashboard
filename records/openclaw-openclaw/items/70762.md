## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Reject symlinked transcripts before opening them:** `src/agents/cli-runner/session-history.ts:50-54`
  - body: `fs.statSync()` follows symlinks and `SessionManager.open()` then reads the target path. A symlink planted under the sessions directory can make CLI hook history read a non-transcript file or blocking special file; use `lstat` plus realpath containment before opening the transcript.
  - confidence: 0.89
- **[P1] Honor custom session stores for hook history:** `src/agents/cli-runner/session-history.ts:32-34`
  - body: This resolver only passes `agentId`, so deployments with `config.session.store` outside the default state sessions directory resolve the provided transcript as out-of-scope and return empty hook history. Pass the configured store path into `resolveSessionFilePathOptions` so custom stores keep their conversation window.
  - confidence: 0.91
- **[P2] Skip transcript parsing when hooks do not need history:** `src/agents/cli-runner.ts:57-63`
  - body: `runPreparedCliAgent` loads and parses session history before checking active hooks, adding synchronous filesystem work to ordinary CLI runs that only have no hooks or output-only hooks. Gate the load on input/end style hooks that actually consume `historyMessages`.
  - confidence: 0.84
