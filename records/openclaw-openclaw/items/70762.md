- **[P1] Reject symlinked transcripts before opening:** `src/agents/cli-runner/session-history.ts:50-54`
  - body: This helper calls `statSync` and then opens the transcript by path, so a transcript under the sessions directory can still be a symlink to another local file. Preserve current main's `lstat` plus realpath containment hardening before any rebase.
  - confidence: 0.88
- **[P1] Gate conversation hooks before exposing raw prompts:** `src/agents/cli-runner.ts:64-73`
  - body: The CLI runner sends system prompt, user prompt, history, and assistant output to lifecycle hooks without the current non-bundled plugin trust gate. That makes conversation contents available to plugins that have not explicitly opted into trusted conversation access.
  - confidence: 0.86
- **[P2] Honor the configured session store:** `src/agents/cli-runner/session-history.ts:32-34`
  - body: `resolveSessionFilePathOptions` is called without the configured `session.store`, so deployments using a custom store can silently lose prior CLI history in hook payloads. Keep the current main store-aware resolver behavior.
  - confidence: 0.82
- **[P2] Load transcript history only when hooks need it:** `src/agents/cli-runner.ts:57-63`
  - body: `runPreparedCliAgent` parses transcript history before checking whether lifecycle hooks need that payload, adding synchronous file work to CLI runs that do not consume history. Current main avoids this by loading history only for hooks that require it.
  - confidence: 0.78
- **[P2] Skip llm_output when the CLI returns no text:** `src/agents/cli-runner.ts:133-144`
  - body: `llm_output` is dispatched even when `assistantText` is empty, while consumers expect this hook to carry actual assistant output. Current main guards the dispatch on non-empty assistant text.
  - confidence: 0.76
