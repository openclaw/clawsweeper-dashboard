Overall confidence: 0.9

Full review comments:

- **[P1] Reject symlinked transcripts before opening:** `src/agents/cli-runner/session-history.ts:50-54`
  - body: `statSync()` follows symlinks before `SessionManager.open()` reads the transcript, so a symlink inside the sessions directory can redirect hook-history loading to another local file. Preserve current main's `lstat` plus realpath containment hardening before any rebase.
  - confidence: 0.9
- **[P1] Gate conversation hooks before exposing raw prompts:** `src/agents/cli-runner.ts:225-228`
  - body: This path sends system prompt, user prompt, history, and assistant output to lifecycle hooks without current main's non-bundled plugin conversation-access gate. That would re-expose raw conversation contents to plugins that have not explicitly opted in.
  - confidence: 0.88
- **[P2] Honor the configured session store:** `src/agents/cli-runner/session-history.ts:32-34`
  - body: The resolver calls `resolveSessionFilePathOptions` with only the agent id, so deployments using `config.session.store` can silently lose prior CLI history in hook payloads. Keep current main's `storePath: params.config?.session?.store` behavior.
  - confidence: 0.84
- **[P2] Load transcript history only when hooks need it:** `src/agents/cli-runner.ts:57-63`
  - body: `runPreparedCliAgent` parses session history before checking active hooks, adding synchronous transcript I/O to the default CLI hot path. Current main avoids that by loading history only for hooks that consume it.
  - confidence: 0.8
