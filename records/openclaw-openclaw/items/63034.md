## Summary

Keep open. Current main still applies OpenRouter Anthropic cache markers only to system/developer messages, current tests and docs still encode that behavior, and an open implementation PR already uses closing syntax for this issue.

## What This Changes

Request that the OpenRouter Anthropic cache-control path also mark conversation messages, at least the trailing user turn, instead of only system/developer prompt blocks.

## Best Possible Solution

Review and land one focused OpenRouter Anthropic cache-marker policy that reuses the direct Anthropic trailing-user behavior while resolving breakpoint-limit and cacheable-block semantics raised by the overlapping PR.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is an OpenRouter Anthropic payload with a system message plus a user message: current main tags the system block and leaves the user content unchanged, and the current tests assert that behavior.

## Solution Assessment

Unclear as a final product decision. Reusing the existing trailing-user helper is the smallest technical fix, but https://github.com/openclaw/openclaw/pull/57987 raises additional breakpoint-limit and cacheable-block concerns that maintainers should weigh before choosing the final patch.
