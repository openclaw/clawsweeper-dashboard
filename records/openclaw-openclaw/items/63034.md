## Summary

Keep open. Current main still applies OpenRouter Anthropic cache markers only to system/developer messages, the focused tests and public docs still describe that system/developer-only behavior, and an open PR already uses closing syntax for this issue.

## What This Changes

The issue requests that OpenRouter Anthropic cache-control payload shaping mark conversation/user messages, not only system/developer prompts.

## Best Possible Solution

Review and land one canonical OpenRouter Anthropic cache-marker policy with regression tests, docs alignment, shared-helper impact accounted for, and redacted real OpenRouter proof.

## Reproduction Assessment

Yes. A high-confidence source-level path on current main is an OpenRouter Anthropic payload with system plus user messages: the wrapper calls the affected helper, the helper only tags system/developer content, and focused tests assert the user message remains unchanged.

## Solution Assessment

Unclear as the final merge path.
