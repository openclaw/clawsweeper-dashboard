## Summary

Keep open. Current main still applies OpenRouter Anthropic cache markers only to system/developer messages, while the direct Anthropic path still marks the trailing user message; the issue is also paired with open implementation PR #63062 using closing syntax, with overlapping PR #57987 still open.

## What This Changes

The issue requests that the OpenRouter Anthropic path apply `cache_control` to conversation messages, at minimum the trailing user message, instead of only system/developer prompt blocks.

## Best Possible Solution

Review the existing PRs as the implementation candidates, pick the desired OpenRouter Anthropic cache-marker policy, and land one focused change with tests and user-facing docs. The end state should either mirror the direct Anthropic trailing-user cache breakpoint on verified OpenRouter routes or document why OpenRouter intentionally remains system/developer-only.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is the current OpenRouter wrapper path with a system message plus a user message: current code tags the system block and leaves the user content unchanged, and the current test suite asserts that behavior.

## Solution Assessment

Unclear as a final product decision. Reusing the existing trailing-user helper is the smallest technical fix, but #57987 raises additional breakpoint-limit and cacheable-block concerns that maintainers should weigh before choosing the best patch.
