- `impact:session-state`: The issue is about memory-wiki context/index state omitting nested source pages from compiled/searchable memory.

## Summary

Keep open. Current main and the latest release still only enumerate direct Markdown children in memory-wiki page-group directories, and the open recursive/page-groups PR is not merged yet.

## What This Changes

Request support for recursive memory-wiki page discovery so Markdown under `sources/<subdir>/...` can be indexed without flattening existing vaults.

## Best Possible Solution

Define the nested-page and raw-source contract, then land an owner-reviewed implementation with one deterministic recursive Markdown collector shared by compile, query/get, status, lint, and affected docs/tests.

## Reproduction Assessment

Yes, at source level. Put Markdown under `sources/<subdir>/page.md`; compile, query/get, and status each inspect only direct page-group children, so the nested page is skipped on current main and `v2026.5.18`.

## Solution Assessment

Unclear for the final implementation. Recursive collection is the right direction, but maintainers still need to choose the page ID, raw-source, traversal safety, and configurability contract before the issue can close.
