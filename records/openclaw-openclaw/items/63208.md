## Summary

Keep open. Current `main` still skips nested memory-wiki pages in compile, local search/get discovery, and status counts, and the newest issue comment confirms the flat-only behavior still affects an Obsidian-style vault user.

## What This Changes

The issue asks memory-wiki to recursively discover Markdown pages under page-group directories so nested Obsidian-style vault layouts are indexed without flattening files.

## Best Possible Solution

Add a shared memory-wiki-local recursive Markdown collector for existing page-group directories, reuse it in compile, query/get, and status, document nested vault support, and cover nested pages with focused plugin tests.

## Reproduction Assessment

Yes, source-level. Place a valid page under `sources/topic/page.md`; current `main` reads only direct page-group children with `entry.isFile()`, so the directory is skipped before compile, search/get, or status can see the nested file.

## Solution Assessment

Yes. A shared plugin-local recursive collector is the narrowest maintainable direction; a new config flag is only needed if maintainers want compatibility gating for recursive indexing.
