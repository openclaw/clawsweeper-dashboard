## Summary

Keep open. Current main and the latest release still enumerate only direct Markdown files in memory-wiki page-group directories, and the open recursive-scanning PR is blocked rather than a merged or merge-ready fix.

## What This Changes

## Best Possible Solution

Add a shared memory-wiki-local recursive Markdown collector for existing page-group directories, reuse it in compile/query/status, and separately decide any path-derived ID or config option contract.

Yes. Source inspection on current main gives a high-confidence path: place Markdown under `sources/<subdir>/page.md`; compile, search/get, and status each enumerate only direct Markdown children, so the nested page is skipped.

## Solution Assessment

Unclear for the current open proposal. Recursive discovery is the right core fix, but path-derived IDs and configurable page groups need maintainer API/product direction, and the open broad PR is not merge-ready.
