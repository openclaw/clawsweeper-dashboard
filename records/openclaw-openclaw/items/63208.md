## Summary

Keep open. Current main and the latest release still enumerate only direct Markdown children in memory-wiki page-group directories, while the requested recursive Obsidian-style vault behavior remains undefined around page IDs and raw/frontmatter-free source files.

## What This Changes

Request recursive memory-wiki scanning of nested Markdown files under page-group directories, with preserved directory organization or an optional recursive ingest setting.

## Best Possible Solution

Add a shared memory-wiki-local recursive Markdown collector for page-group directories, reuse it in compile/search/get/status, and decide the nested page ID plus raw-source/frontmatter policy before landing.

## Reproduction Assessment

Yes, at source level. Put Markdown under sources/<subdir>/page.md; compile, query/get, and status each inspect only direct page-group children, so the nested page is skipped.

## Solution Assessment

Unclear for implementation. Recursive collection is the right direction, but path-derived IDs, optional config, raw Obsidian files, and lint/frontmatter semantics need maintainer API/product direction first.
