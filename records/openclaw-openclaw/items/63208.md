## Summary

Keep open. Current main still uses flat one-level Markdown discovery for memory-wiki compile, local search/get, and status counts, and a recent second user comment confirms the Obsidian-style nested vault problem is still active.

## What This Changes

Request support for memory-wiki to recursively discover Markdown pages in page-group subdirectories while preserving the user’s vault organization.

## Best Possible Solution

Add a shared plugin-local recursive Markdown collector for the existing page-group directories, reuse it in compile, query/get, and status, then document and test nested page discovery.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: put a valid Markdown page under `sources/<subdir>/page.md`; compile, search/get, and status each only enumerate direct Markdown files, so the nested page is skipped.

## Solution Assessment

Yes. A shared memory-wiki-local recursive collector is the narrowest maintainable solution; an opt-in config flag or generated path-id contract should be added only if maintainers want that compatibility policy.
