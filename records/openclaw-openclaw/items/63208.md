## Summary

Keep open. Current main and v2026.5.12 still scan only direct Markdown children in memory-wiki page-group directories, while the related recursive page-groups PR is open and blocked rather than a merged fix.

## What This Changes

Request memory-wiki to recursively index Markdown files in page-group subdirectories so existing nested vaults can be compiled and searched without flattening.

## Best Possible Solution

Use one memory-wiki-local recursive Markdown collector for page-group directories across compile/search/status, with explicit handling for nested relative paths, index.md, symlinks, and duplicate/collision behavior.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: Markdown under `sources/<subdir>/page.md` never reaches compile/search/status because the collectors include only direct file entries.

## Solution Assessment

Unclear. Recursive discovery is the right direction, but the path-derived id/config option semantics are not defined yet; the safer implementation is a narrow shared collector plus documented traversal rules.
