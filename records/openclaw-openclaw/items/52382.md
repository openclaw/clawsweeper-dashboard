Keep this issue open. Current main and the latest release still enforce the reported ceiling of 5 in runtime clamping, config validation, generated metadata, and docs, and open PR #52400 already targets this issue with closing syntax.

## What This Changes

The issue requests raising the `session.agentToAgent.maxPingPongTurns` ceiling above 5 or adding a trusted per-workflow override for longer multi-agent coordination chains.

## Best Possible Solution

Land PR #52400 or an equivalent focused change that raises the runtime clamp and schema ceiling together, updates generated metadata/docs, adds regression coverage for configured values above 5, and leaves the separate error-message UX work in #52500.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a value above 5 is rejected by the Zod schema, and the runtime helper would clamp any parsed value back to 5 before the A2A loop uses it.

## Solution Assessment

Unclear for the full requested design. Raising the existing ceiling consistently is the narrowest likely near-term fix, while a per-workflow trusted override needs maintainer product/API judgment.
