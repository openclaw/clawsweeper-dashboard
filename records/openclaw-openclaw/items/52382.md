## What This Changes

The issue requests raising the `session.agentToAgent.maxPingPongTurns` hard ceiling above 5 or adding a per-workflow override for trusted multi-agent workflows.

## Best Possible Solution

Land PR #52400 or an equivalent focused change that raises the runtime clamp and schema/metadata/docs together, adds regression coverage for configured values above 5, and leaves the separate error-message UX work in #52500.

## Reproduction Assessment

Yes. Source inspection is enough: `SessionSchema` rejects values over 5, and `resolvePingPongTurns` clamps any parsed value back down to 5 before the A2A reply loop uses it.

## Solution Assessment

Unclear. Raising the existing ceiling consistently is the narrowest near-term fix, but the exact ceiling and whether to add a trusted per-workflow override need maintainer product/API judgment.
