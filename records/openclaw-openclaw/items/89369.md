## Summary

Keep open for normal maintainer review: current main still has the forked-session context-loss path, and this PR addresses both the missing fork-file write and the child-run preparation reset with focused tests and sufficient live-output proof.

## What This Changes

The PR makes forked session transcript writing unconditional for assistant-less parent branches and preserves inherited fork entries during child-run preparation.

## Best Possible Solution

Land the two-boundary fix after normal maintainer review because it preserves the documented fork contract without adding config, API, dependency, or storage-schema surface.

## Reproduction Assessment

Yes. Source inspection shows an assistant-less parent branch can make the fork writer return a missing transcript path, and even after writing that file current run preparation would clear the inherited branch before the child run.

## Solution Assessment

Yes. The patch repairs the writer invariant and the child-run preparation invariant at their owner boundaries; caller-only handling would leave sibling fork surfaces exposed.
