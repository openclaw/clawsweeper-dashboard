## Summary

Keep open. Current main still sends WebChat final assistant output from the Gateway chat event path without invoking the existing `message_sending` hook, while the documented hook contract and shared outbound delivery path already support rewrite/cancel behavior.

## What This Changes

Request to invoke the existing `message_sending` plugin hook for WebChat/WebSocket final assistant delivery so hooks can rewrite or cancel the payload before broadcast.

## Best Possible Solution

Wire `message_sending` into WebChat final assistant delivery with `channelId: "webchat"`, session/run context, rewrite/cancel handling, hook-failure isolation, and focused regression coverage while leaving broader lifecycle unification to the umbrella issues.

## Reproduction Assessment

Yes, at source level. A WebChat run that emits assistant text and then lifecycle `end` reaches `emitChatFinal`, which broadcasts the final payload without any `message_sending` hook call.

## Solution Assessment

Yes. Reusing the existing `message_sending` hook in the WebChat final path is the narrowest maintainable fix; adding a WebChat-only hook or universal outbound boundary would be broader than this issue.
