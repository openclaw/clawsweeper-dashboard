## Summary

Keep open. Current main and latest release still bypass the existing `message_sending` hook on the WebChat final delivery path, while shared outbound and Telegram delivery paths invoke it; related open issues are broader hook-coverage umbrellas rather than replacements for this narrow WebChat parity gap.

## What This Changes

The issue requests firing the existing `message_sending` plugin hook for WebChat/WebSocket final message delivery, including rewrite/cancel support and WebChat session correlation.

## Best Possible Solution

Wire the existing `message_sending` hook into WebChat final assistant delivery with `channelId: "webchat"`, `sessionKey`, and run correlation, apply rewrites/cancel consistently to WebSocket and node-session payloads, and leave broader outbound-policy unification to the umbrella items.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/65792. On current main, WebChat final assistant delivery in `src/gateway/server-chat.ts` broadcasts the final `chat` payload without running the existing `message_sending` plugin hook. Reuse the current hook runner contract instead of adding a new hook or config option: obtain the global hook runner, gate on `hasHooks("message_sending")`, call `runMessageSending` for `jobState === "done"` with `channelId: "webchat"`, session/run correlation (`sessionKey`, `runId`), and the final assistant text, then apply rewritten content to both `broadcast("chat", ...)` and `nodeSendToSession(...)`. Honor `cancel: true` without letting stale streamed deltas become the persisted final UI message; update `ui/src/ui/controllers/chat.ts` only if needed for that cancel semantic. Keep hook failures isolated from message delivery, matching the outbound delivery path. Add focused regression tests in `src/gateway/server-chat.agent-events.test.ts` for rewrite, cancel, context fields, and hook failure isolation; add/update `ui/src/ui/controllers/chat.test.ts` if cancel changes final-event handling. Do not claim this resolves broader lifecycle-hook coverage issues such as post-send hooks or unbypassable outbound policy.
