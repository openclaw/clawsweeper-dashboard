## Summary

Keep open: current main and the latest release still pass subagent completion text into the parent-session internal event without a dedicated child-result cap, and the related fix PR was closed unmerged. The remaining decision is product/API shape, especially whether this should be a new public config key or fit the existing agent context-limits surface.

## What This Changes

Request adding a configurable cap to truncate subagent completion results before they are injected into the parent session context.

## Best Possible Solution

Keep this open for a maintainer API decision, then bound task-completion handoff at the agent context-limits or internal-events boundary while preserving prompt-data hardening and media/attachment formatting.

## Reproduction Assessment

Yes for source-level reproduction: current main passes full child completion text into `internalEvents[0].result`, and the formatter wraps that exact text without a child-result cap. I did not run a live large-subagent scenario in this read-only review.

## Solution Assessment

Unclear as proposed: bounding the handoff is the right area, but the requested `session.maxSubagentResultChars` key conflicts with the repository's high bar for new config surface and may fit better under existing agent context limits or as an internal default.
