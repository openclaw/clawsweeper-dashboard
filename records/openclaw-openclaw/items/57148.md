## Summary

Keep open: current main and the latest release still pass subagent task-completion text into the parent-session internal event without a dedicated result-size cap, and the focused implementation PR remains open rather than merged.

## What This Changes

The issue requests a `session.maxSubagentResultChars` configuration limit so subagent task-completion text is truncated before it is injected into the parent session context.

## Best Possible Solution

Add a focused cap at the internal task-completion result boundary, preserve the current prompt-data wrapping for untrusted child output, and document whichever `agents.*.contextLimits`-style config/default shape maintainers choose.

## Reproduction Assessment

Yes, at source level. A long `childCompletionFindings` or `reply` value still flows into `internalEvents[0].result`, through requester-agent delivery, and into prompt formatting without a dedicated subagent result cap.

## Solution Assessment

Yes in direction, but the exact proposed config key is not settled. The maintainable fix should cap task-completion payloads while preserving current prompt-data wrapping and aligning the knob with the existing agent context-limit config contract.
