## Summary

Keep open: current main and the latest release still lack a dedicated cap for subagent task-completion result text before it reaches requester-agent context, and the linked implementation PR is still open and unresolved.

## What This Changes

Request to add a bounded subagent task-completion result cap before child output is injected into the parent session context.

## Best Possible Solution

Land or replace the focused PR with a cap at the task-completion event or requester-agent delivery boundary, while preserving `wrapPromptDataBlock` prompt hardening and aligning any public knob with `agents.*.contextLimits`.

## Reproduction Assessment

Yes, at source level. A long `childCompletionFindings` or `reply` value can flow into `internalEvents[0].result`, through requester-agent delivery, and into prompt formatting without a dedicated subagent result cap.

## Solution Assessment

Yes in direction, but the exact proposed session-level config key is not settled. The maintainable solution should cap task-completion payloads while preserving current prompt-data wrapping and deciding whether the knob belongs under `agents.*.contextLimits`.
