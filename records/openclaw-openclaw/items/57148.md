## Summary

Keep open: current main and the latest release still pass subagent completion text into the parent-session internal event without a dedicated result cap, and the related implementation PR was closed unmerged with this issue left as the canonical tracker. The requested config knob needs maintainer product/config approval before an automated fix lane should choose the permanent API shape.

## What This Changes

Request to add a `session.maxSubagentResultChars` cap so oversized subagent completion text is truncated before it is injected into the parent session.

## Best Possible Solution

Add a bounded internal completion-result handoff in the agent runtime, reusing existing truncation semantics where appropriate and preserving prompt-data hardening; expose a public config/default only after maintainer approval.

## Reproduction Assessment

Yes, at source level: current main can place arbitrary-length subagent completion text into `internalEvents[0].result` and then render that result into the parent prompt without a task-completion cap. I did not run a live large-subagent flow because this review is read-only.

## Solution Assessment

Unclear as proposed: bounding the handoff is the right owner boundary, but adding `session.maxSubagentResultChars` directly needs maintainer approval under OpenClaw's high bar for new config surface. A fixed or context-derived cap may be the safer default if maintainers do not want another public knob.
