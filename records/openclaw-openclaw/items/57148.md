## Summary

Keep open: current main still lets subagent/task-completion result text flow into internal events, gateway delivery, and prompt formatting without a dedicated size cap, and the directly related implementation PR is still open and unmerged.

## What This Changes

The issue requests a configurable `maxSubagentResultChars` limit to truncate subagent completion results before they are injected into the parent session.

## Best Possible Solution

Land `https://github.com/openclaw/openclaw/pull/73320` or an equivalent focused patch that bounds task-completion event results before delivery and prompt rendering while preserving full child output in the child session history.

## Reproduction Assessment

Yes. Source inspection shows a long child completion result flows into `internalEvents[0].result`, through gateway delivery, and into parent prompt formatting without a current-main cap.

## Solution Assessment

Yes in direction, but unclear on the exact proposed config key. The safest maintainable fix is to cap the task-completion event at the event/delivery boundary, with any public knob aligned to the existing agent context-limits model.
