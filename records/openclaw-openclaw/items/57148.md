## What This Changes

Request adds a configurable maximum character cap for subagent result text before child completions are injected into parent-session context.

## Best Possible Solution

Review, rebase, land, or replace the linked PR with a cap at the task-completion event or delivery boundary while preserving `wrapPromptDataBlock` hardening and aligning any public knob with agent context limits.

## Reproduction Assessment

Yes, at source level. A long child completion can flow into `internalEvents[0].result`, through requester-agent delivery, and into prompt formatting without a dedicated subagent result cap.

## Solution Assessment

Unclear as proposed. The cap direction is right, but maintainers still need to decide whether the public knob belongs under the existing `agents.*.contextLimits` contract or a new session-level key.
