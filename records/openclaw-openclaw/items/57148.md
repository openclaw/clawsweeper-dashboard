## Summary

Keep open: current main and the latest release still pass raw child completion text into parent-session internal events, prompt rendering, and gateway request serialization without a dedicated task-completion cap, while the direct fix PR remains open and unmerged.

## What This Changes

Request a bounded maximum for subagent/task-completion result text injected back into parent sessions, proposed as `session.maxSubagentResultChars`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73320, or an equivalent focused patch, that caps task-completion event results before delivery and prompt rendering, preserves full child output in the child session history, and aligns any public knob with the existing agent context-limits model.

## Reproduction Assessment

Yes. A high-confidence source reproduction path is clear: a long child completion can populate `internalEvents[0].result` via `src/agents/subagent-announce.ts:472` and `:523`, then render and serialize without a current-main cap.

## Solution Assessment

Yes in direction, but the exact proposed `session.maxSubagentResultChars` key is not obviously the best public API. The narrow maintainable fix is to cap at the task-completion event/delivery boundary and only add a config knob if maintainers want it aligned with `agents.*.contextLimits`.
