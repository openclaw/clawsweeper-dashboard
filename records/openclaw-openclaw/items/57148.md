## Summary

Keep open: current main and v2026.6.8 still route selected subagent completion text into the parent task-completion internal event without a dedicated child-result injection cap, and the linked implementation PR was closed unmerged. The remaining blocker is maintainer product/API direction for the cap boundary, default, and config surface.

## What This Changes

Request to add a configurable maximum for subagent completion result text before it is injected into the parent session context.

## Best Possible Solution

Keep this open until maintainers choose the cap boundary and default, then bound child-result prompt injection while preserving the prompt-data wrapper and existing frozen-result recovery behavior.

## Reproduction Assessment

Yes. Current main source still passes selected child completion text into a parent task-completion internal event without a dedicated child-result injection cap; I did not run a live large-subagent scenario in this read-only review.

## Solution Assessment

Unclear as proposed. Bounding the handoff is the right product area, but the exact `session.maxSubagentResultChars` key needs maintainer approval because OpenClaw has a high bar for new config surface and already has an `agents.*.contextLimits` namespace.
