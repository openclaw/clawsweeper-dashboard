## Summary

Keep open: current main and v2026.5.18 still pass unbounded subagent/task-completion result text into the requester-agent prompt path, and the direct implementation PR remains open and conflicted rather than merged.

## What This Changes

Request adding a configurable cap for subagent/task-completion result text before it is injected into the parent session.

## Best Possible Solution

Land or replace the linked focused PR so task-completion event results are capped before requester-agent delivery while preserving prompt-data wrapping and documenting any accepted context-limits knob.

## Reproduction Assessment

Yes, at source level. A long childCompletionFindings or reply string flows into internalEvents[0].result and then prompt rendering without a current-main cap; I did not run a live large-subagent session.

## Solution Assessment

Yes in direction, but the exact proposed session.maxSubagentResultChars API is not settled. The maintainable path is to cap at the task-completion event or delivery boundary and align any public knob with the existing agents.*.contextLimits contract.
