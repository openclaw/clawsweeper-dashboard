## Summary

Keep open. Current main still passes native subagent completion text into the parent-session internal event and prompt formatter without a dedicated size cap, and the closest implementation is open PR #73320 rather than merged.

## What This Changes

Add a bounded default or configuration key so oversized subagent completion results are truncated before they are injected into the parent session prompt.

## Best Possible Solution

Review, fix, and land PR #73320 or an equivalent narrow patch that bounds task-completion internal event results before delivery and prompt injection, with formatter coverage for both runtime-context and plain-prompt paths. If maintainers want a public knob, it should align with the existing agent context-limits model.

## Reproduction Assessment

Yes. A source-level reproduction is clear: a long child assistant reply or completion findings value flows into `internalEvents[0].result`, then into `formatTaskCompletionEvent` and the parent prompt without a current-main cap.

## Solution Assessment

Yes in direction, but unclear on the exact proposed config key. A generic task-completion result cap is the narrow maintainable fix; a public knob should probably live under `agents.defaults/list[].contextLimits` if maintainers decide configurability is needed.
