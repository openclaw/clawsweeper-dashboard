## Summary

Keep open: current main and the latest release still do not provide a dedicated cap for subagent task-completion text before it reaches requester-agent context, and the focused implementation PR remains open and conflicted.

## What This Changes

The issue requests a configurable maximum for subagent result text injected into the parent session.

## Best Possible Solution

Land or replace the focused PR with a cap at the task-completion event or requester-agent delivery boundary, preserving wrapPromptDataBlock prompt hardening and aligning any public knob with agents.*.contextLimits.

## Reproduction Assessment

Yes, at source level. A long childCompletionFindings or reply value can pass into internalEvents[0].result and prompt formatting without a dedicated subagent result cap; I did not run a live large-subagent session in this read-only review.

## Solution Assessment

Yes in direction, but the exact proposed session-level config key is not settled. The maintainable solution should cap the task-completion payload while preserving current prompt-data wrapping and deciding whether the knob belongs under agents.*.contextLimits.
