## Summary

Keep open: current main and v2026.6.8 still route selected subagent completion text into the parent task-completion internal event without a dedicated parent-injection cap, and the linked fix PR was closed unmerged. The remaining blocker is maintainer product/API choice around the cap, default, and config surface.

## What This Changes

Request to add a configurable `maxSubagentResultChars` cap so oversized subagent completion text is truncated before injection into the parent session.

## Best Possible Solution

Keep this open for a maintainer API/default decision, then bound the subagent completion handoff at the agent context-limits or internal-events boundary while preserving prompt-data hardening and existing frozen-result recovery behavior.

## Reproduction Assessment

Yes for source-level reproduction: current main passes selected child completion text into a parent task-completion internal event without a dedicated child-result injection cap. I did not run a live large-subagent scenario in this read-only review.

## Solution Assessment

Unclear as proposed: bounding the handoff is the right product area, but the exact `session.maxSubagentResultChars` key needs maintainer approval because OpenClaw already has a high bar for new config surface and an existing `agents.*.contextLimits` namespace.
