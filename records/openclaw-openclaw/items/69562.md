Full review comments:

- **[P1] Keep cron validation off full channel plugins:** `src/gateway/server-methods/cron.ts:3`
  - body: `cron.add` and `cron.update` run on a gateway hot path, but this import lets the account validator call `getChannelPlugin()` and fall back to bundled full channel plugins. The scoped gateway/channel policy asks for lightweight public artifacts for static questions, and the PR's gateway watch check already failed with 5,692 added dist-runtime files and about 48 MB of apparent runtime growth. Please move account-id membership checks to a lightweight/read-only account metadata resolver instead of loading the full plugin registry here.
  - confidence: 0.94
- **[P2] Use configured account IDs in the cron tests:** `src/gateway/server.cron.test.ts:936`
  - body: These Telegram fixtures define only `channels.telegram.accounts.sentry`, with no root token or env-backed implicit default account. Current Telegram account selection therefore lists `sentry`, not `default`, so the baseline cron jobs using `accountId: "default"` are invalid and the expected error list should not require `default`. Use the configured `sentry` account or add real default-account credentials to the fixture.
  - confidence: 0.88
