The PR adds gateway cron.add/update validation for explicit announce delivery and failure-destination account IDs, plus server tests for invalid IDs, failure destinations, unlisted defaults, normalized IDs, and omitted-channel behavior.

Land one focused gateway-side validation change that rejects unknown explicit account IDs for cron.add and cron.update, including failure destinations, while preserving omitted account IDs, omitted-channel/last behavior, normalized IDs, and unlisted/default account compatibility through a lightweight configured-account resolver.
