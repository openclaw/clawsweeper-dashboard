## Summary

Keep open. Current main still supports Xiaomi MiMo preset voice/style TTS only; official Xiaomi docs show VoiceDesign uses a distinct `mimo-v2.5-tts-voicedesign` request shape that OpenClaw does not expose, document, or test.

## What This Changes

Add Xiaomi MiMo v2.5 TTS VoiceDesign configuration support while preserving existing preset `voice` behavior.

## Best Possible Solution

Implement a Xiaomi-plugin-local VoiceDesign config path that maps to Xiaomi's official `mimo-v2.5-tts-voicedesign` contract, omits preset `audio.voice` for that model, and updates focused tests, docs, and changelog.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the gap: `voiceDesign` search is empty, Xiaomi config has no such field, and synthesis always sends preset `audio.voice`.

## Solution Assessment

Yes. A provider-local Xiaomi change is the narrowest maintainable direction because the existing speech provider contract already permits provider-owned config without a core SDK change.
