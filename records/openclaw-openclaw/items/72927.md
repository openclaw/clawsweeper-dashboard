## Summary

Keep open. Current main still has Xiaomi MiMo preset TTS support only: there is no `voiceDesign` config, request mapping, documentation, or focused test coverage, while the plugin surface is already provider-owned enough for a narrow Xiaomi-local implementation.

## What This Changes

Add Xiaomi MiMo v2.5 TTS VoiceDesign support to the Xiaomi TTS provider configuration while preserving existing preset `voice` behavior.

## Best Possible Solution

Implement a Xiaomi-plugin-local `voiceDesign` config shape that maps to Xiaomi's official VoiceDesign request contract, keeps existing preset `voice` behavior backward-compatible, and documents the supported fields with focused unit coverage.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the gap: no `voiceDesign` config path is parsed or documented, and synthesis always sends `audio.voice`; live API proof is unavailable here because no Xiaomi API key is present.

## Solution Assessment

Yes. A provider-local Xiaomi implementation is the narrowest maintainable direction because speech provider config is plugin-owned; a shared core passthrough would be broader than this vendor-specific request requires unless Xiaomi's official contract proves otherwise.
