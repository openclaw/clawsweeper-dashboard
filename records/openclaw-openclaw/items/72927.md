## Summary

Keep open. Current main still supports only Xiaomi preset TTS voices/style guidance, while Xiaomi's current MiMo V2.5 TTS docs expose VoiceDesign as a separate model/request shape, so this remains a narrow bundled-plugin feature gap.

## What This Changes

Add Xiaomi MiMo V2.5 TTS VoiceDesign configuration support to the bundled Xiaomi speech provider.

## Best Possible Solution

Extend only the bundled Xiaomi plugin with a documented `voiceDesign` config that maps to Xiaomi's `mimo-v2.5-tts-voicedesign` contract while preserving existing preset voice/style behavior.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the gap: no VoiceDesign config/model is accepted, and the serialized Xiaomi request always includes preset `audio.voice`.

## Solution Assessment

Yes. A provider-local Xiaomi implementation is the narrowest maintainable path; a generic core passthrough would be broader than this provider-specific API shape requires.

Implement Xiaomi MiMo V2.5 TTS VoiceDesign support for this issue. Keep the change scoped to the bundled Xiaomi plugin and related docs/tests/changelog. Add a documented `voiceDesign` config shape, with at least a required/string `prompt`, and map it to Xiaomi's `mimo-v2.5-tts-voicedesign` chat-completions request: `messages` should include the voice-design prompt as the required `user` message and the synthesized text as the `assistant` message; `audio` should include `format` but omit preset `voice` for VoiceDesign. Add focused unit coverage in `extensions/xiaomi/speech-provider.test.ts` for config normalization and request body serialization; live test coverage is optional and should only run when Xiaomi credentials/entitlement are available.
