## Summary

Keep open. Current main still implements Xiaomi MiMo TTS only for built-in voices plus optional style text, while Xiaomi's official MiMo v2.5 TTS docs define VoiceDesign as a separate request mode that OpenClaw does not parse, serialize, test, or document.

## What This Changes

Request support for Xiaomi MiMo v2.5 TTS VoiceDesign configuration in the bundled Xiaomi TTS provider.

## Best Possible Solution

Add Xiaomi-plugin-local VoiceDesign support using the official `mimo-v2.5-tts-voicedesign` contract, preserving the existing built-in `voice` path and documenting/testing the new config shape.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, tests, and upstream API docs give high-confidence proof that current main lacks the requested VoiceDesign config and request path.

## Solution Assessment

Yes directionally. The narrow maintainable solution is provider-local config parsing and request serialization in `extensions/xiaomi`, not a core TTS API change.
