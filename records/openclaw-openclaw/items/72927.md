## Summary

Keep open. Current main still lacks Xiaomi MiMo VoiceDesign support: the bundled Xiaomi provider only normalizes preset voice/style configuration and always sends `audio.voice`, while Xiaomi documents VoiceDesign as the separate `mimo-v2.5-tts-voicedesign` request shape.

## What This Changes

Add Xiaomi MiMo v2.5 TTS VoiceDesign configuration support to OpenClaw's bundled Xiaomi speech provider.

## Best Possible Solution

Extend only the bundled Xiaomi plugin with a documented `voiceDesign` config shape that maps to Xiaomi's `mimo-v2.5-tts-voicedesign` contract, while preserving existing preset `voice` and `style` behavior.

## Reproduction Assessment

Yes. Static inspection reproduces the gap: a `voiceDesign` config value would be ignored by the normalizer, and synthesis still builds a preset-voice request body.

## Solution Assessment

Yes. A provider-local Xiaomi implementation is the narrowest maintainable path; a generic TTS `extraBody` passthrough or new core SDK seam would be broader than the documented provider-specific API requires.
