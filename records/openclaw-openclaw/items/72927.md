## Summary

Keep open. Current main still lacks Xiaomi MiMo VoiceDesign support: the Xiaomi speech provider advertises only preset TTS models, normalizes only preset voice/style config, and always serializes `audio.voice`; Xiaomi's current public docs describe VoiceDesign as a separate `mimo-v2.5-tts-voicedesign` request shape, so this remains a real provider-local feature gap.

## What This Changes

Add Xiaomi MiMo v2.5 TTS VoiceDesign support to OpenClaw's Xiaomi TTS provider configuration while preserving existing preset `voice` behavior.

## Best Possible Solution

Extend only the bundled Xiaomi plugin with a documented `voiceDesign` config shape that maps to Xiaomi's `mimo-v2.5-tts-voicedesign` contract, using the prompt as the required `user` message, keeping synthesized text as the `assistant` message, omitting preset `audio.voice` for VoiceDesign, and preserving existing preset voice/style behavior.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the gap: there is no VoiceDesign config key or model metadata, and the Xiaomi request body always includes preset `audio.voice`.

## Solution Assessment

Yes. A provider-local Xiaomi implementation is the narrowest maintainable solution; a generic core `extraBody` passthrough or new SDK seam would be broader than this provider-specific API shape requires.
