## Summary

Keep open. Current main still supports Xiaomi MiMo TTS through built-in voices plus optional style text, but it does not parse, serialize, test, or document the separate MiMo v2.5 VoiceDesign request mode.

## What This Changes

Add Xiaomi MiMo v2.5 TTS VoiceDesign configuration and request support to the built-in Xiaomi TTS provider.

## Best Possible Solution

Add Xiaomi-plugin-local VoiceDesign support for `mimo-v2.5-tts-voicedesign`, preserving the existing built-in voice path and documenting/testing the new config shape.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, tests, and Xiaomi's official API guide give a high-confidence current-main gap for VoiceDesign support.

## Solution Assessment

Yes directionally. The narrow maintainable solution is provider-local config parsing and request serialization in `extensions/xiaomi`, with maintainer approval for the public config schema.
