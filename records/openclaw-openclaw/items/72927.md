## Summary

Keep open. Current main still lacks Xiaomi MiMo VoiceDesign config, model metadata, request serialization, tests, and docs, while Xiaomi's public docs now describe VoiceDesign as a supported MiMo v2.5 TTS variant.

## What This Changes

Add Xiaomi MiMo v2.5 TTS VoiceDesign support to the bundled Xiaomi TTS provider so users can describe a custom voice instead of only selecting preset voices.

## Best Possible Solution

Extend only the bundled Xiaomi plugin with a documented `voiceDesign` config shape that maps to Xiaomi's `mimo-v2.5-tts-voicedesign` contract while preserving existing preset `voice` and `style` behavior.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the gap: the provider drops any `voiceDesign` config and always sends preset `audio.voice`; live API reproduction was blocked by missing Xiaomi credentials.

## Solution Assessment

Yes. A Xiaomi provider-local implementation is the narrowest maintainable path; a generic core `extraBody` passthrough or new SDK seam would be broader than this provider-specific API shape requires.
