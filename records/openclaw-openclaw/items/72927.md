## Summary

Keep open. Current main still implements Xiaomi MiMo TTS as preset voice plus optional style guidance only; there is no VoiceDesign config key, directive token, docs entry, or request serialization branch. The gap is provider-local and suitable for a focused fix after verifying Xiaomi's official VoiceDesign request contract.

## What This Changes

Support Xiaomi MiMo v2.5 TTS VoiceDesign/custom voice design parameters in the bundled Xiaomi TTS provider configuration while preserving existing preset voice behavior.

## Best Possible Solution

Extend only the bundled Xiaomi speech provider with a documented VoiceDesign config shape mapped to Xiaomi's verified API contract, while preserving existing `voice`, `style`, `format`, `mimo-v2.5-tts`, and `mimo-v2-tts` behavior. Add focused provider tests, Xiaomi/TTS docs, and a user-facing changelog entry.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the feature gap: `voiceDesign` has no schema/normalization path, no directive handling, and synthesis always sends a preset `audio.voice` request body.

## Solution Assessment

Yes. A provider-local Xiaomi change is the narrowest maintainable direction; a generic core `extraBody` passthrough or new SDK seam would be broader than this provider-specific request currently requires.
