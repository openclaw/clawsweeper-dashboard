## Summary

Keep open. Current main still lacks Xiaomi MiMo VoiceDesign support: the bundled Xiaomi speech provider has no VoiceDesign config path, does not advertise the VoiceDesign model, and always serializes preset-voice `audio.voice` requests. The issue is narrow, provider-local, and not superseded by the related original Xiaomi TTS PR context.

## What This Changes

Request support for Xiaomi MiMo v2.5 TTS VoiceDesign configuration so OpenClaw can send a custom voice-description prompt instead of only preset voice IDs.

## Best Possible Solution

Add provider-local Xiaomi VoiceDesign support in `extensions/xiaomi` with a documented config shape, preserve existing preset `voice` and `style` behavior, and add focused unit/docs coverage for the new request serialization path.

## Reproduction Assessment

Yes. Static inspection of current main is enough to reproduce the gap: a `voiceDesign` config field is neither normalized nor serialized, and every Xiaomi TTS request still sends a preset `audio.voice`.

## Solution Assessment

Yes. A narrow Xiaomi plugin change is the best fit; a generic core passthrough or new SDK seam would be broader than this provider-specific VoiceDesign request requires.
