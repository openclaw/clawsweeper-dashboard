## Summary

Keep open. Current main still exposes Xiaomi MiMo TTS as preset voice/style configuration only, while Xiaomi’s public MiMo page describes Voice Design as a supported TTS-series capability, so this remains a real bundled-provider feature gap. ([mimo.mi.com](https://mimo.mi.com/))

## What This Changes

Add Xiaomi MiMo v2.5 TTS VoiceDesign configuration support so users can describe a custom voice instead of choosing only preset voice IDs.

## Best Possible Solution

Extend the bundled Xiaomi plugin with documented `voiceDesign` config and provider-local request serialization while preserving existing preset `voice` and `style` behavior.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the gap: there is no VoiceDesign config key or model metadata, and Xiaomi synthesis always sends preset `audio.voice`.

## Solution Assessment

Yes. A Xiaomi-provider-local change is the narrowest maintainable direction; a generic core passthrough would be broader than this provider-specific API shape appears to require.
