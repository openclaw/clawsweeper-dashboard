## Summary

Keep open. Current main still has Xiaomi MiMo TTS preset voice/style support only: there is no VoiceDesign config key, no VoiceDesign model advertised by the Xiaomi speech provider, and synthesis still serializes `audio: { format, voice }`. The prior ClawSweeper comment already routed the provider-local gap to @zoujiejun and @leonchui, and I found no current-main implementation or superseding open/merged fix in the provided context or local checkout.

## What This Changes

The issue requests Xiaomi MiMo v2.5 TTS VoiceDesign configuration support in the bundled Xiaomi TTS provider.

## Best Possible Solution

Extend only the bundled Xiaomi speech provider with a documented VoiceDesign config shape mapped to Xiaomi's official VoiceDesign TTS request contract, while preserving existing `voice`, `style`, `format`, `mimo-v2.5-tts`, and `mimo-v2-tts` behavior. Add focused provider tests, Xiaomi/TTS docs, and a user-facing changelog entry.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the gap: `voiceDesign` has no schema or normalization path, the provider advertises only the base MiMo TTS model IDs, and synthesis always sends a preset `audio.voice`.

## Solution Assessment

Yes. A provider-local Xiaomi change is the narrowest maintainable direction; a generic core `extraBody` passthrough or new SDK seam would be broader than this provider-specific request currently requires.
