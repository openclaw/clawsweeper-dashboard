## Summary

Keep open. Current main still exposes Xiaomi MiMo TTS as preset `voice` plus optional `style` only; there is no `voiceDesign` config, request mapping, documentation, or regression coverage, while Xiaomi's current public MiMo page confirms Voice Design exists but does not expose the request-field contract in the crawled content. ([mimo.mi.com](https://mimo.mi.com/))

## What This Changes

Add Xiaomi MiMo v2.5 TTS VoiceDesign configuration so users can describe or configure a custom TTS voice instead of only selecting a preset voice.

## Best Possible Solution

Add a Xiaomi-plugin-local `voiceDesign` config shape only after confirming Xiaomi's official VoiceDesign request contract, preserve existing `voice` behavior, and document the supported fields with focused unit coverage.

## Reproduction Assessment

Yes for the feature gap: source inspection shows no `voiceDesign` parsing, docs, tests, or request serialization path on current main. This is a new feature request rather than a failing existing-behavior bug.

## Solution Assessment

Yes directionally: a Xiaomi-plugin-local implementation is the narrow maintainable path because the bundled Xiaomi plugin already owns this provider behavior. The exact public config schema and payload mapping need maintainer/provider confirmation before implementation.
