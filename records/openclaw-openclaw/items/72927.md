## Summary

Keep open. Current main still supports Xiaomi MiMo TTS through preset `voice` plus optional `style`, while the official VoiceDesign path uses the separate `mimo-v2.5-tts-voicedesign` model and omits `audio.voice`; this is now a narrow Xiaomi-plugin feature rather than a stale or duplicate request.

## What This Changes

Add Xiaomi MiMo v2.5 TTS VoiceDesign support so users can describe generated voices instead of selecting only preset voice IDs.

## Best Possible Solution

Add provider-local support for `mimo-v2.5-tts-voicedesign`: expose the model, use the existing user-message style/prompt path as the voice description, omit preset `audio.voice` for that model, and cover it in Xiaomi tests and docs.

## Reproduction Assessment

Yes for the feature gap: source inspection shows current main has no VoiceDesign model metadata, docs, or request serialization path, and it always sends `audio.voice`. This is not a broken existing contract; it is a missing provider capability.

## Solution Assessment

Yes. The narrow maintainable fix belongs inside the bundled Xiaomi speech provider, preserving existing preset-voice behavior while special-casing the official VoiceDesign model request shape.
