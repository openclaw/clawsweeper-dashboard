## Summary

Keep open. Current main still supports Xiaomi MiMo TTS through preset `voice` plus `style`, but it does not parse, serialize, test, or document a VoiceDesign config path; Xiaomi's public MiMo page also describes Voice Design as part of the V2.5 TTS series. 

## What This Changes

Add Xiaomi MiMo v2.5 TTS VoiceDesign support to the Xiaomi TTS provider configuration while preserving existing preset voice behavior.

## Best Possible Solution

Implement a Xiaomi-plugin-local `voiceDesign` config shape that maps to Xiaomi's official VoiceDesign request contract, keeps existing `voice` behavior backward-compatible, and updates focused tests plus TTS docs.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the gap: no `voiceDesign` config path is read, serialized, tested, or documented; live API verification was not feasible because no Xiaomi API key is available locally.

## Solution Assessment

Yes. A Xiaomi provider-local implementation is the narrowest maintainable direction; a generic core passthrough would be broader than this vendor-specific feature unless the official contract proves a shared seam is required.
