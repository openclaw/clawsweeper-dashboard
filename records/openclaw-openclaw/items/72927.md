## Summary

Keep open. Current main still supports Xiaomi MiMo TTS through preset voice/style fields only: there is no VoiceDesign config key, docs entry, test coverage, or request serialization path, while Xiaomi's public MiMo page advertises Voice Design as part of the MiMo-V2.5-TTS series. The prior ClawSweeper review and issue timeline already routed the provider-local gap to the relevant Xiaomi/TTS maintainers, and the issue has no protected labels or linked implementation PR.

## What This Changes

Add Xiaomi MiMo v2.5 TTS VoiceDesign configuration passthrough to the bundled Xiaomi TTS provider.

## Best Possible Solution

Extend only the bundled Xiaomi speech provider with a documented `voiceDesign` config shape mapped to Xiaomi's official VoiceDesign TTS request contract, preserving existing `voice`, `style`, `format`, `mimo-v2.5-tts`, and `mimo-v2-tts` behavior. Add focused provider tests, Xiaomi/TTS docs, and a user-facing changelog entry credited to the reporter.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the gap: `voiceDesign` has no schema or normalization path, and synthesis always sends a preset `audio.voice`.

## Solution Assessment

Yes. A provider-local Xiaomi change is the narrowest maintainable direction; a generic core `extraBody` passthrough or new SDK seam would be broader than this provider-specific request currently requires.
