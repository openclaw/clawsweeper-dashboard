## Summary

Keep open. Current main still lacks Xiaomi MiMo VoiceDesign support: the bundled Xiaomi speech provider exposes only preset voice/style fields, docs describe only built-in voices, and synthesis always sends a preset `audio.voice`. Xiaomi's current public TTS docs define `mimo-v2.5-tts-voicedesign` as a separate request shape where the user message is the required voice description and `audio` only needs the output format, so this remains a real provider-local gap rather than a cleanup close.

## What This Changes

Add Xiaomi MiMo v2.5 TTS VoiceDesign support to the Xiaomi TTS provider configuration while preserving existing preset `voice`, `style`, and format behavior.

## Best Possible Solution

Extend only `extensions/xiaomi` with a documented VoiceDesign config shape that maps to Xiaomi's official `mimo-v2.5-tts-voicedesign` contract: use the configured prompt as the required `user` message, keep the target text as the `assistant` message, omit preset `audio.voice` for VoiceDesign, and preserve existing `voice`, `style`, `format`, `mimo-v2.5-tts`, and `mimo-v2-tts` behavior.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the gap: there is no VoiceDesign config key, the provider does not advertise the VoiceDesign model, and the request body always includes preset `audio.voice`.

## Solution Assessment

Yes. A provider-local Xiaomi implementation is the narrowest maintainable path; a generic core `extraBody` passthrough or new SDK seam would be broader than this provider-specific request requires.
