## Summary

Keep open. Current main still has Xiaomi MiMo TTS wired only for built-in voices plus style text, while Xiaomi's current public docs define VoiceDesign as a separate `mimo-v2.5-tts-voicedesign` request path that OpenClaw does not parse, serialize, test, or document.

## What This Changes

Request support for Xiaomi MiMo v2.5 TTS VoiceDesign configuration and request mapping in the Xiaomi speech provider.

## Best Possible Solution

Add Xiaomi-plugin-local VoiceDesign support using the official `mimo-v2.5-tts-voicedesign` contract, preserve existing built-in voice behavior, and document/test the new config shape.

## Reproduction Assessment

Yes for the feature gap: current-main source, docs, and tests contain no VoiceDesign config or request path, while upstream docs define one. This is not a broken existing behavior bug; it is a missing provider capability.

## Solution Assessment

Yes directionally: the Xiaomi plugin already owns this TTS provider, so the narrow solution is provider-local config parsing and request serialization rather than a core TTS API change. The implementation should follow the official `mimo-v2.5-tts-voicedesign` model and message/audio contract.
