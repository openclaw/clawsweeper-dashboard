## Summary

Keep open. Current main still only supports preset Xiaomi TTS voice/style configuration: the Xiaomi plugin has no `voiceDesign` config path, does not advertise a VoiceDesign model, always serializes `audio.voice`, and the Xiaomi/TTS docs and focused tests cover only preset voices.

## What This Changes

Support Xiaomi MiMo v2.5 TTS VoiceDesign parameters in the bundled Xiaomi TTS provider configuration.

## Best Possible Solution

Add a Xiaomi-plugin-local `voiceDesign` config shape, map it to the official MiMo v2.5 VoiceDesign request contract, preserve existing preset `voice` behavior, and document the supported fields with focused tests and a changelog entry.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the gap: no `voiceDesign` key is read or documented, the provider metadata omits the VoiceDesign model, and synthesis always sends a preset `audio.voice`.

## Solution Assessment

Yes. A Xiaomi provider-local implementation is the narrowest maintainable direction because the speech-provider SDK already accepts provider-owned config records and synthesis hooks; a generic core passthrough would be broader than this vendor-specific request shape requires.
