## Summary

Keep open. Current main still lacks Xiaomi MiMo VoiceDesign support: the Xiaomi speech provider only normalizes preset voice/model/format/style settings, does not advertise the VoiceDesign model ID, and always serializes a preset `audio.voice` request shape. The prior ClawSweeper comment correctly routed this as provider-local work, and a bounded API/search check did not find an open PR superseding the issue.

## What This Changes

Request to add Xiaomi MiMo v2.5 TTS VoiceDesign configuration support so users can pass a voice-description prompt instead of only selecting preset Xiaomi voices.

## Best Possible Solution

Extend only the Xiaomi speech provider with a documented `voiceDesign` config shape mapped to Xiaomi's `mimo-v2.5-tts-voicedesign` chat-completions contract, keep existing preset `voice`, `style`, `format`, `mimo-v2.5-tts`, and `mimo-v2-tts` behavior unchanged, and add focused provider tests, docs, and a user-facing changelog entry.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the gap: a `voiceDesign` value has no schema or normalization path, the VoiceDesign model is not advertised, and synthesis always serializes a preset `audio.voice`.

## Solution Assessment

Yes. A provider-local Xiaomi change is the narrowest maintainable fix; a generic core `extraBody` passthrough or new SDK seam would be broader than this provider-specific request contract requires.
