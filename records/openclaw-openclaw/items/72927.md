## Summary

Keep open. Current main still has Xiaomi MiMo TTS preset voice/style support only: there is no VoiceDesign config key, no request-body mapping, and no test or docs coverage for the requested capability. The request is recent, unprotected, not paired with an open fix PR in the provided context, and narrow enough to route as a focused Xiaomi provider follow-up.

## What This Changes

The issue requests Xiaomi MiMo v2.5 TTS VoiceDesign configuration support in the bundled Xiaomi speech provider.

## Best Possible Solution

Extend only the bundled Xiaomi speech provider with an official, documented VoiceDesign config shape mapped to Xiaomi's MiMo v2.5 TTS request contract, while preserving existing `voice`, `style`, `format`, `mimo-v2.5-tts`, and `mimo-v2-tts` behavior. Add focused provider tests, Xiaomi/TTS docs, and a user-facing changelog entry.

## Reproduction Assessment

Yes. Static inspection of current main is enough to reproduce the gap: a `voiceDesign` config value has no normalization path and synthesis still sends only `audio.format` plus `audio.voice`.

## Solution Assessment

Yes. A provider-local Xiaomi change is the narrowest maintainable solution; a generic core `extraBody` passthrough or new SDK seam would be broader than this provider-specific request currently requires.
