## Summary

Keep open. Current main still supports Xiaomi MiMo TTS through fixed `voice`, `format`, and optional `style` only; VoiceDesign exists upstream, but the official request field/model contract is not established enough for a conservative automated implementation.

## What This Changes

Request Xiaomi MiMo v2.5 TTS VoiceDesign config support so users can describe or configure a custom voice instead of selecting only preset voice IDs.

## Best Possible Solution

Add a Xiaomi-plugin-local VoiceDesign config and request mapping only after confirming Xiaomi's official API contract, while preserving existing `voice` behavior and documenting the supported schema with focused tests.

## Reproduction Assessment

Yes for the feature gap: source inspection on current main shows no `voiceDesign` parsing, docs, tests, or request serialization path. This is a new provider capability request, not a broken existing behavior bug.

## Solution Assessment

Unclear as a concrete implementation until the Xiaomi VoiceDesign API contract is confirmed. Directionally, the narrow maintainable path is to implement it inside the bundled Xiaomi plugin rather than in core.
