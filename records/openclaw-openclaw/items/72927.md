## Summary

Keep open. Current main still supports Xiaomi built-in voice/style TTS only; official Xiaomi docs show VoiceDesign needs the `mimo-v2.5-tts-voicedesign` request shape, which OpenClaw does not document, expose, or test today.

## What This Changes

Add Xiaomi MiMo v2.5 TTS VoiceDesign configuration support while preserving existing preset `voice` behavior.

## Best Possible Solution

Implement a Xiaomi-plugin-local VoiceDesign config path that maps to Xiaomi's official `mimo-v2.5-tts-voicedesign` contract, omits preset `audio.voice` for that model, and documents the supported config.

## Reproduction Assessment

Yes. Static inspection reproduces the gap: the requested `voiceDesign` field is ignored, docs do not list the VoiceDesign model, and synthesis always sends a preset `audio.voice`.

## Solution Assessment

Yes. A provider-local Xiaomi change is the narrowest maintainable solution because the existing speech provider contract already allows provider-owned config without a core SDK change.
