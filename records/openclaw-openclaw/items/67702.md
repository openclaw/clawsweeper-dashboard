The PR adds a git-mode update preflight that detects a same-install running Gateway service, auto-stops it before in-place dist replacement, blocks `--no-restart`, and adds update CLI tests.

## Best Possible Solution

Land a rebased git-mode guard integrated with current managed-service update helpers: detect same-install services including wrappers, stop only with guaranteed recovery, restart on every failure/skip path, use the service-control stream contract, and prove the real git-mode path.

## Solution Assessment

No. The intended guard belongs in the update lifecycle, but this branch should reuse current managed-service contracts and handle wrappers, recovery, stream output, and real git-mode proof before it is the best fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Handle wrapper-backed gateway commands:** `src/cli/update-cli/update-command.ts:135`
  - body: `findGatewayServiceEntrypoint` assumes the token before `gateway` is the dist entrypoint. Supported wrapper installs use commands like `[wrapper, "gateway", "run"]`, so this compares the wrapper path to `dist/index.js` and allows the same running install to update.
  - confidence: 0.91
- **[P1] Restart the gateway after failed or skipped git updates:** `src/cli/update-cli/update-command.ts:1128`
  - body: After this stop, the `result.status === "error"` and `result.status === "skipped"` branches return before any restart. A dirty checkout, failed fetch, or failed rebase can leave the previously running Gateway down even though no update was applied.
  - confidence: 0.9
- **[P1] Pass a writable stream to service.stop:** `src/cli/update-cli/update-command.ts:1128`
  - body: `GatewayServiceControlArgs.stdout` is a `NodeJS.WritableStream`, and launchd/systemd/schtasks stop implementations call `stdout.write`. `defaultRuntime` has logging helpers but no `write`, so the real auto-stop path can throw instead of continuing.
  - confidence: 0.89
- **[P2] Remove restart from the pre-update block guidance:** `src/cli/update-cli/update-command.ts:1117`
  - body: In the `--no-restart` block, telling users to run `openclaw gateway restart` before retrying leaves the service running and triggers the same block again. The guidance should be stop, update, then start or restart after the update.
  - confidence: 0.86
