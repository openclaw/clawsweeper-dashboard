## Summary

Keep open. Current main and v2026.6.6 still provide only the generic Agents > Files editor for allowlisted bootstrap files plus root MEMORY.md; there is no dedicated Control UI memory panel, no /memory route, and no editable daily memory note surface.

## Best Possible Solution

Design a scoped memory-files Control UI and Gateway contract that explicitly covers USER.md, MEMORY.md, and validated daily notes with safe read/write authorization, visible save semantics, tests, and docs.
