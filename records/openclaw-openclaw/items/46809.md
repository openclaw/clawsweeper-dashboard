## Summary

Keep open. Current main and v2026.6.8 only provide a generic Core Files editor for bootstrap files plus root MEMORY.md; the requested dedicated Memory panel, editable daily memory note, and product/security model remain missing. ## What This Changes

The issue requests a Control UI panel for reading and editing MEMORY.md, USER.md, and daily memory notes such as memory/YYYY-MM-DD.md.

## Best Possible Solution

Design a scoped memory-specific Gateway/API and Control UI surface for USER.md, root MEMORY.md, and date-validated daily notes, with explicit trust boundaries and save semantics before implementation.
