## Summary

Keep open. Current main only partially covers the request through the generic Agents > Files editor for allowlisted bootstrap files and root MEMORY.md; it still lacks a dedicated Memory panel or /memory route for editing USER.md, MEMORY.md, and today's daily memory note, while related workspace and pinned-context work remains open and the closest memory-view PR is closed unmerged.

## What This Changes

Requests a dedicated Control UI Memory panel/sidebar to view and edit USER.md, MEMORY.md, and today's memory/YYYY-MM-DD.md.

## Best Possible Solution

Add a scoped, security-reviewed memory-file API and discoverable Control UI memory UX, or intentionally fold USER.md, MEMORY.md, and daily-note shortcuts into the canonical workspace file manager after maintainers settle the product shape.
