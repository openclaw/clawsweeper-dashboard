## Summary

Keep open. Current main partially covers the request through the generic Agents > Files editor for USER.md and root MEMORY.md, but it still has no dedicated Memory panel or /memory route, and no editable memory/YYYY-MM-DD.md daily-note surface; the related implementation PR at https://github.com/openclaw/openclaw/pull/73773 was closed unmerged.

## What This Changes

Add a Control UI Memory panel that displays and edits USER.md, MEMORY.md, and today's daily memory note.

## Best Possible Solution

Add a scoped Gateway memory-files API and Control UI Memory surface for USER.md, root MEMORY.md, and today's memory/YYYY-MM-DD.md, with strict path/date validation, UI coverage, docs, and changelog follow-through.
