## Summary

Keep open: the report is valid on current main and is already paired with focused open fix PRs, so the issue should stay open until one canonical fix lands.

## What This Changes

The issue requests that heartbeat delivery sanitize or suppress Codex protocol scaffolding before posting to visible chat channels.

## Best Possible Solution

Land a single focused sanitizer and heartbeat delivery fix that suppresses protocol-only heartbeat text while preserving real visible text and media, then close this issue after merge.

## Reproduction Assessment

Yes, at source level. Current main normalizes heartbeat replies from raw payload text without the canonical assistant-visible sanitizer, so a payload containing only `<tool_calls>` or `<file_contents ...>` scaffolding can pass through to delivery.

## Solution Assessment

Yes. The maintainable direction is to reuse the canonical assistant-visible sanitizer at the heartbeat delivery boundary and extend it for Codex scaffolding, preferably by landing one focused PR such as https://github.com/openclaw/openclaw/pull/81434.
