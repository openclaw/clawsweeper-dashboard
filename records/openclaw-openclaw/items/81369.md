## Summary

Keep open: current main still routes heartbeat reply text to visible delivery without the canonical assistant-visible sanitizer, and https://github.com/openclaw/openclaw/pull/81434 is an open main-targeted fix with closing syntax for this issue.

## What This Changes

The issue asks OpenClaw to suppress raw Codex protocol/tool scaffolding from heartbeat delivery while preserving real assistant-visible text and media.

## Best Possible Solution

Land one focused main-targeted fix, preferably https://github.com/openclaw/openclaw/pull/81434 or an equivalent canonical patch, that applies the shared assistant-visible sanitizer to heartbeat delivery and covers `<file_contents>` and marker-only payloads with regression tests.

Yes, at source level: inject a heartbeat `getReplyFromConfig` result containing only `<tool_calls>` or `<file_contents ...>` text and current `normalizeHeartbeatReply` will pass normalized raw text to `sendDurableMessageBatch`. I did not execute a failing test because this sweep is read-only.

## Solution Assessment

Yes, the best direction is to reuse and extend the canonical assistant-visible sanitizer at the heartbeat delivery boundary, while preserving media-bearing payload behavior. The open main-targeted fix PR is the right place to review and land that implementation.
