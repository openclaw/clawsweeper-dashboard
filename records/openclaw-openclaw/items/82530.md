## Summary

Keep this PR open: the updated real Feishu proof resolves the earlier proof gate, but the functional blocker remains because the patch still marks coarse authorization probes as text commands.

## What This Changes

The PR adds `CommandSource: "text"` to Feishu inbound contexts when command authorization probing runs and updates Feishu command-authorization tests.

## Best Possible Solution

Mark only actual Feishu text slash commands as explicit command turns, while leaving ordinary chat and inline slash-like text under the existing visible-reply defaults.

## Reproduction Assessment

Yes, source-level: current main can authorize Feishu command-looking messages without `CommandSource`, while reply delivery only forces automatic source replies for explicit command turns. The contributor also supplied before/after Feishu DM logs for the `/models` path.

## Solution Assessment

No, not as written: the PR uses the coarse authorization-probe predicate as the command-source predicate. The safer repair is to derive `CommandSource: "text"` from actual text command detection, such as the existing `isControlCommandMessage`/`hasControlCommand` path, and keep the broad probe only for authorization checks.
