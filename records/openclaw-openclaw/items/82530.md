## Summary

Keep open: the reported Feishu text-command reply bug is plausible and the PR touches the right seam, but the patch still marks broad authorization-probe matches as explicit text commands and lacks after-fix real Feishu proof.

## What This Changes

The PR adds `CommandSource: "text"` to Feishu finalized inbound contexts when command authorization probing runs and updates Feishu command-authorization tests.

## Best Possible Solution

Set text-command metadata only for actual explicit slash-command turns, preserve ordinary chat delivery defaults, and add redacted after-fix Feishu proof before merge.

## Reproduction Assessment

Yes, source-level: current main can finalize Feishu turns with `CommandAuthorized` but no `CommandSource`, while reply delivery only forces automatic source replies for explicit command turns. I did not run a live Feishu DM.

## Solution Assessment

No, not as written: using `shouldComputeCommandAuthorized` as the `CommandSource` guard is broader than the command-turn contract. The safer fix should derive `CommandSource: "text"` from an actual explicit text command, not from the coarse probe predicate.

## Review Findings
