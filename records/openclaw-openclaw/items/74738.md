## Summary

Keep open. Current main still has the OpenAI Completions transport bug, and this PR contains the right core guard for that path, but the submitted patch also mixes in an unrelated Pi retry settings change that can persist user settings and has a fallback regression test that does not exercise the real tool-history shape. It also needs the required changelog entry before merge.

## What This Changes

The PR adds a `supportsModelTools(model)` guard around OpenAI Completions `tools`/`tool_choice` payload emission and the `tools: []` tool-history fallback, adds regression tests, and also changes embedded Pi settings preparation to call `setRetryEnabled(false)`.

## Best Possible Solution

Land a narrow provider transport fix for #74664: gate OpenAI Completions tool payloads and the tool-history fallback with the existing model tool-support helper, cover both real payload paths with accurate fixtures, and record the user-facing fix in the changelog. Keep #73781 as separate runtime-retry work unless maintainers approve a non-persistent Pi retry override design.
