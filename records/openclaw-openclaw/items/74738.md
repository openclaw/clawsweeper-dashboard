## Summary

Keep open. Current main still has the `openai-completions` payload bug and this PR contains the right core guard, but the branch is not ready as-is because it mixes in a separate Pi retry behavior change, lacks the required user-facing changelog entry, and one regression test does not exercise the fallback path it names.

## What This Changes

The PR adds a `supportsModelTools(model)` guard and regression tests for OpenAI Completions tool payloads, and also disables embedded Pi retry during prepared settings creation.

## Best Possible Solution

Land a rebased, narrow #74664 fix that guards OpenAI Completions tool payload emission at the transport layer, uses real OpenClaw tool-history fixtures for coverage, includes the required changelog entry, and leaves the Pi retry behavior to a separate maintainer-reviewed change.

## Solution Assessment
