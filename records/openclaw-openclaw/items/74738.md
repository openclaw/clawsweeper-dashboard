## Summary

Keep open. Current main still has the exported OpenAI Completions builder emitting tool payload fields without the existing `compat.supportsTools` guard; the linked issue was closed by a runtime-wrapper fix, but this PR still covers the lower builder/SDK helper surface and only lacks real behavior proof before merge.

## What This Changes

The branch imports `supportsModelTools`, wraps OpenAI Completions builder tool emission and tool-history fallback with that guard, adds two regression tests, and adds a changelog entry.

## Best Possible Solution

Land this builder-level guard, or an equivalent narrow patch, after redacted real payload proof is posted while keeping the embedded-runner wrapper as defense-in-depth.

## Reproduction Assessment

Yes. Source inspection on current main and the linked payload-capture report show `buildOpenAICompletionsParams()` reaches unguarded `params.tools` assignments when `compat.supportsTools` is false and either tools or tool history are present.

## Solution Assessment

Yes. Reusing `supportsModelTools(model)` inside the builder is the narrow maintainable fix because the existing runtime wrapper covers common agent runs but not the exported builder/direct-caller surface.
