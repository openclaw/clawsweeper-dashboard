The PR wraps OpenAI Completions tool payload emission and the tool-history fallback in `supportsModelTools(model)`, adds regression tests, and updates the changelog.

## Best Possible Solution

Land this builder-level guard, or an equivalent narrow patch, so both the exported payload builder and the existing runtime wrapper honor `compat.supportsTools: false`.

## Reproduction Assessment

Yes. Source inspection on current main shows the builder still reaches unguarded `params.tools`, `params.tool_choice`, and `tools: []` assignments for `compat.supportsTools: false`; the linked report and latest PR proof cover the same payload path.

## Solution Assessment

Yes. Reusing `supportsModelTools(model)` inside `buildOpenAICompletionsParams()` is the narrow maintainable fix because the existing wrapper only protects wrapped stream execution paths, not direct exported builder callers.
