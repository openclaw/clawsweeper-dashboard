Define the `enabled: true` contract first, then land either a narrow host-binary override with hard gates preserved or a documented blanket operator override with upgrade proof.

No, not as written: the patch fixes the host-binary case but also bypasses OS, env, and config gates without an established docs contract. The safer path is to preserve hard gates unless maintainers explicitly accept and document blanket override semantics.
