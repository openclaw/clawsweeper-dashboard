## Summary

Keep open. Current main still lacks SDK-facing task ledger RPCs, and this PR is the active implementation for the #74707 feature, but the branch is not mergeable as-is because release notes, public docs, and generated Swift protocol models are still inconsistent with the new API.

## What This Changes

The PR adds `tasks.list`, `tasks.get`, and `tasks.cancel` Gateway RPCs with SDK helpers/types, protocol schemas/scopes, generated Swift models, docs, and tests over the existing task registry.

## Best Possible Solution

Land one additive Gateway task-ledger contract after making release notes, public protocol/API docs, and generated native models agree with the new SDK surface.

## Solution Assessment

Yes in direction, but not as-is. Exposing the existing durable task registry through additive authenticated Gateway RPCs is the narrow maintainable solution, but the branch must repair the stale docs, changelog, and generated Swift output before merge.
