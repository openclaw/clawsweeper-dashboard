## Summary

Keep open. Current main still lacks SDK-facing task ledger RPCs, and this PR is the active implementation for #74707, but it remains blocked by a missing changelog entry, incomplete Gateway protocol docs, and a generated Swift model mismatch.

## What This Changes

This PR adds `tasks.list`, `tasks.get`, and `tasks.cancel` Gateway RPCs plus SDK methods/types, protocol schemas/scopes, generated Swift models, docs, and regression tests.

## Best Possible Solution

Land one additive Gateway task-ledger contract after repairing release notes, public protocol docs, and generated native models; then #74707 can close through the merged PR.

## Reproduction Assessment

Yes. Static verification on `origin/main` is enough: `oc.tasks.list/get/cancel` still throw unsupported errors, SDK docs still list them as unsupported, and Gateway method discovery lacks `tasks.*`.

## Solution Assessment

Yes in direction, but not as-is. Exposing the existing durable task registry through additive Gateway RPCs and SDK helpers is the narrow maintainable fix, but the public API/documentation artifacts must be consistent before merge.
