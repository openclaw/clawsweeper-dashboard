## Summary

Keep open. Current main still lacks SDK-facing task ledger RPCs, and this PR is the active implementation for #74707, but it needs focused repairs before merge: release notes, Gateway protocol docs, and generated Swift model typing are not yet consistent with the new public contract.

## What This Changes

This PR adds `tasks.list`, `tasks.get`, and `tasks.cancel` Gateway RPCs plus SDK methods/types, protocol schemas/scopes, generated Swift models, docs, and regression tests over the existing task registry.

## Best Possible Solution

Land one additive task-ledger contract after syncing the changelog, public Gateway protocol docs, and generated native models with the schema.

## Reproduction Assessment

Yes. Static verification on current main is enough: `oc.tasks.list/get/cancel` still throw unsupported SDK errors, and Gateway method discovery lacks `tasks.*`.

## Solution Assessment

Yes in direction, but not as-is. Exposing the existing durable task registry through additive Gateway RPCs and SDK helpers is the narrow maintainable solution, once the public artifacts are consistent.
