This PR adds Gateway `tasks.list`, `tasks.get`, and `tasks.cancel` RPCs, SDK task methods/types, protocol schemas/scopes, generated Swift models, docs, and regression tests over the existing task registry.

## Best Possible Solution

## Solution Assessment

Yes in direction, but not as-is. Exposing the existing durable task registry through additive Gateway RPCs and SDK helpers is the narrow maintainable fix, but the public API artifacts must be consistent before merge.
