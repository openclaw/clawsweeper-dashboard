## Summary

Keep open. Current main still lacks the SDK-facing task ledger RPC surface, and this PR is the active implementation for #74707, but it remains blocked by changelog, public docs, and generated Swift model drift before merge.

## What This Changes

The PR adds `tasks.list`, `tasks.get`, and `tasks.cancel` Gateway RPCs, SDK methods/types, protocol schemas/scopes, generated Swift models, docs, and tests over the existing task registry.

## Best Possible Solution

Land one additive Gateway task-ledger contract after release notes, public protocol/API docs, and generated native models all agree with the new SDK surface.

## Reproduction Assessment

Yes. Source inspection on current main shows `oc.tasks.list/get/cancel` still throw unsupported SDK errors and Gateway discovery/scopes do not expose `tasks.*`; this is a feature gap rather than a broken existing contract.

## Solution Assessment

Yes in direction, but not as-is. Exposing the existing durable task registry through additive authenticated Gateway RPCs is the narrow maintainable approach, but this branch must repair the stale docs, changelog, and generated Swift output before merge.
