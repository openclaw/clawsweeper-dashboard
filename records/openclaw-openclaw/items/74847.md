Keep open. Current main still lacks the SDK-facing task ledger RPCs, and this non-draft PR is the active implementation for the open member-authored API request; it should be reviewed and rebased rather than closed as implemented or duplicate.

The PR adds Gateway `tasks.list`, `tasks.get`, and `tasks.cancel` RPCs, SDK task methods/types, protocol schemas, operator scopes, docs, generated Swift models, and regression coverage over the existing task registry.

## Best Possible Solution

Land one polished additive Gateway task-ledger contract on the canonical branch after maintainer review confirms the metadata boundary, status vocabulary, operator scopes, cancel semantics, generated native model shape, docs, and rebase result.

## Reproduction Assessment

Yes. Current main can be verified by static source and docs inspection: SDK task helpers still throw unsupported errors, tests expect that behavior, and Gateway discovery/handler registration lacks `tasks.*`.

## Solution Assessment

Yes overall, with low-priority caveats. Exposing the existing durable task registry through additive, operator-gated Gateway RPCs and SDK methods is the narrow maintainable direction; the native model/doc inconsistencies and dirty branch should be fixed before landing.
