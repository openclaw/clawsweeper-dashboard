## Summary

Current main still lacks the requested switch-time warning, but #58957 now tracks the same remaining user problem with broader reproduction data and proposed fixes, so keeping both open would split the same implementation work.

## What This Changes

The issue requests a `/model` switch-time context-size check that warns before moving a long-running session to a target model whose effective context or rate-limit budget may be unsafe.

## Best Possible Solution

Keep #58957 as the canonical implementation thread and preserve this issue’s OpenAI Codex TPM/rate-limit example there. The eventual fix should add a shared model-switch context-risk warning for WebChat and messaging directives, while leaving provider/account-specific TPM prediction and automatic compaction as explicit policy decisions.

Yes for the observable missing warning: inspect or exercise `/model` in WebChat or directive-only chat with a large-session token snapshot, and current main persists plus acknowledges the switch without a target-budget comparison.
