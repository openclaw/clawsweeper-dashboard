## Summary

Keep open: the report targets real gateway reasoning gaps, but this branch is not mergeable and current main does not fully supersede it. A newer focused PR covers the callback-less WebSocket stream half, while the remaining `agent.create` session reasoning forwarding gap still appears untracked by a merged or canonical follow-up.

## What This Changes

This PR forwards stored session reasoning into embedded agent runs, loosens embedded reasoning-stream callback gating for WebSocket broadcasts, and adds scheduled-reset/session-memory metadata handling with tests.

## Best Possible Solution

Land the focused callback-decoupling PR for WebSocket thinking streams, handle daily/idle reset hooks in the canonical reset PR, and address any remaining `agent.create` reasoning forwarding gap with a small owner-module patch and regression coverage.

## Reproduction Assessment

Yes. Source tracing on current main shows the callback-less WebSocket stream path is still gated by `onReasoningStream`, and the `agent.create` path still reaches `runEmbeddedPiAgent()` without forwarding a stored `reasoningLevel`; I did not run a live gateway repro in this read-only review.

## Solution Assessment

No. The branch targets real bugs, but the copied command implementation, unrelated reset/session-memory changes, missing proof, and weakened authorization/visibility gates make the focused callback PR plus a separate narrow attempt-execution fix safer.
