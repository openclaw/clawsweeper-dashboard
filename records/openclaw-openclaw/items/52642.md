## Summary

Keep open: the reported gateway reasoning bug is still source-reproducible on current main, but this PR branch is not mergeable as-is because it mixes unrelated reset/session-memory work, copies a drifted command implementation, weakens security/visibility gates, and lacks real behavior proof.

## What This Changes

This PR forwards stored session reasoning into embedded Pi agent runs, loosens embedded reasoning-stream callback gating for WebSocket broadcasts, and adds scheduled-reset/session-memory metadata changes with tests.

## Best Possible Solution

Land a focused gateway reasoning fix in the owner modules: forward stored `reasoningLevel` into `runEmbeddedPiAgent()`, decouple only the channel callback dependency for explicit stream visibility, preserve silent/visibility/authorization gates, and keep reset/session-memory work separate.

## Reproduction Assessment

Yes. Source tracing on current main shows `sessions.patch` persists `reasoningLevel`, `runAgentAttempt()` does not pass it to `runEmbeddedPiAgent()`, and callback-less gateway runs cannot pass the current `onReasoningStream` gate before `emitAgentEvent()`.

## Solution Assessment

No. The branch targets real bugs, but the copied command implementation, unrelated reset/session-memory changes, and weakened authorization/visibility gates make a focused replacement or split safer.
