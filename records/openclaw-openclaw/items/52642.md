## Summary

Keep open. The reported gateway reasoning bug is real on current main, but this PR is not mergeable as-is because it mixes unrelated session-memory work and introduces correctness, authorization, and reasoning-visibility regressions already visible in the proposed diff.

## What This Changes

This PR forwards stored session reasoning into embedded Pi runs, loosens embedded reasoning stream callback gating, and adds scheduled-reset/session-memory metadata handling with tests.

## Best Possible Solution

Land a focused gateway reasoning fix in the owner modules: pass stored `sessionEntry.reasoningLevel` into `runEmbeddedPiAgent()`, allow callback-less `thinking` agent events for gateway clients while preserving `canShowReasoning` and silent-reply gates, and add targeted embedded-subscribe plus gateway coverage. Keep scheduled-reset/session-memory behavior in a separate reviewed change.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to set `reasoningLevel: "stream"` with `sessions.patch` and then send via WebSocket `chat.send`; current main persists the setting but neither forwards it into the embedded run nor supplies the callback required before thinking events are emitted.

## Solution Assessment

No. The proposed branch addresses the right user-visible gap, but it is not the best fix because it copies a large command implementation, mixes unrelated session-memory changes, and removes existing authorization and visibility safeguards; a narrow split or replacement is safer.
