## Summary

Keep open: current main still has the reported gateway reasoning gap, but this PR is not mergeable as-is because it mixes the focused WebSocket reasoning fix with unrelated scheduled-reset/session-memory work and introduces blocking correctness and security regressions.

## What This Changes

The branch forwards stored session reasoning into embedded Pi runs, decouples reasoning stream broadcasts from channel callbacks, and adds scheduled-reset/session-memory metadata handling with tests.

## Best Possible Solution

Land a focused gateway reasoning fix in the owner modules: pass stored `reasoningLevel` into `runEmbeddedPiAgent()`, broadcast WebSocket thinking events without a channel callback while preserving visibility and silent-reply gates, and keep scheduled-reset/session-memory cleanup in a separate reviewed change.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `sessions.patch` persists `reasoningLevel`, the embedded attempt path does not forward it, and callback-less WebSocket runs cannot pass the current `onReasoningStream` gate before `emitAgentEvent()`.

## Solution Assessment

No. The branch targets the right user-visible gap, but the copied command implementation, unrelated session-memory changes, and weakened authorization/visibility gates make a narrower replacement or split safer.
