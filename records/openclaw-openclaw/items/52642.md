## Summary

Keep open: current main still has a source-reproducible gateway reasoning gap and this PR has useful intent, but the branch is unsafe to merge because it introduces command/runtime regressions, security-sensitive gate changes, unrelated session-memory work, and lacks external real behavior proof.

## What This Changes

The branch forwards stored session reasoning into embedded Pi agent runs, loosens reasoning-stream callback gating for WebSocket broadcasts, and adds scheduled-reset/session-memory metadata handling with tests.

## Best Possible Solution

Land one focused gateway reasoning fix that forwards stored `reasoningLevel`, removes only the channel-callback dependency while preserving visibility/silent gates, and keep daily/idle reset hook work on the canonical reset-hooks PR.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `sessions.patch` persists `reasoningLevel`, the embedded attempt path does not forward it, and callback-less WebSocket runs cannot pass the current `onReasoningStream` gate before `emitAgentEvent()`.

## Solution Assessment

No. The branch targets real bugs, but the large copied command implementation, unrelated reset/session-memory changes, and weakened authorization/visibility gates make a focused replacement or split safer.
