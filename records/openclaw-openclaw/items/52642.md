## Summary

Keep this PR open for maintainer handling: it targets real gateway reasoning bugs, but the branch is not mergeable as-is because it mixes unrelated session-memory work, duplicates large command code, lacks real behavior proof, and introduces blocking regressions already called out in review discussion.

## What This Changes

The branch forwards stored session reasoning into embedded agent runs, loosens embedded reasoning-stream callback gating for WebSocket broadcasts, and adds scheduled-reset/session-memory metadata handling with tests.

## Best Possible Solution

Land the focused no-callback reasoning stream PR, handle daily/idle reset hooks in the dedicated reset-hook PR, and split any remaining `agent.create` stored-reasoning forwarding into a small patch in `src/agents/command/attempt-execution.ts` with targeted regression coverage.

## Reproduction Assessment

Yes. Source tracing on current main shows the callback-less WebSocket stream path is still gated by `onReasoningStream`, and the embedded attempt path still reaches `runEmbeddedPiAgent()` without forwarding stored `reasoningLevel`; I did not run a live gateway repro in this read-only review.

## Solution Assessment

No. The branch addresses real bugs, but the copied command implementation, unrelated reset/session-memory changes, missing proof, and weakened authorization/visibility gates make focused replacement/split fixes safer.
