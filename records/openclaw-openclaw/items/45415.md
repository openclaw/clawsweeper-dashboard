## Summary

The shipped runtime already warns after bootstrap truncation, and current main adds detailed `/context` size reporting, but the requested pre-limit Control UI indicator remains absent. An open linked PR proposes the remaining visibility work, so this issue should stay open until maintainers review or close that implementation and decide whether any enforcement belongs in scope.

## What This Changes

Add MEMORY.md capacity visibility, threshold warnings, save-limit enforcement, startup alerts, and optional archival before bootstrap context is truncated.

## Best Possible Solution

Reuse the existing bootstrap-budget analysis as the single source of truth for a non-blocking capacity indicator in the workspace editor or status surface, preserve file saves, and handle hard enforcement or archival only through separately approved, reversible designs.

## Reproduction Assessment

## Solution Assessment

No as one bundled proposal: advisory capacity visibility is the narrowest maintainable addition, while save refusal and auto-archive are separate persistent-state product decisions.
