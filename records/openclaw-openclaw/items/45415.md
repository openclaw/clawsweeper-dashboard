- `P2`: This is a normal-priority product safety and ergonomics improvement for one bootstrap-memory workflow with bounded runtime impact.

## Summary

Keep open. Current main has shared bootstrap-budget analysis and doctor diagnostics, but it still lacks the requested Control UI capacity indicator, save refusal, near-limit startup warning, and MEMORY.md archive flow; the linked closing PR at https://github.com/openclaw/openclaw/pull/64874 is still open, draft, and conflicting.

## What This Changes

Add MEMORY.md/bootstrap size visibility, warnings, save enforcement, startup notices, and optional archival to prevent silent truncation.

## Best Possible Solution

Extend the shared bootstrap-budget analyzer into Control UI/status/startup warnings, then choose a compatibility-safe save-enforcement and archive policy before rewriting durable memory.

## Reproduction Assessment

Yes, source-reproducible. Current main analyzes near-limit bootstrap files and doctor can report them, but the startup prompt warning path is truncation-only and the Control UI/gateway save path writes MEMORY.md without budget enforcement.

## Solution Assessment

Unclear as a complete solution. The shared analyzer is the right base, but save refusal and archival need maintainer UX, compatibility, and data-safety decisions beyond a narrow bug fix.
