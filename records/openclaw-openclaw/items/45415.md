## Summary

Keep open because current main only partially addresses MEMORY.md bootstrap truncation visibility: it has budget analysis, doctor diagnostics, docs, and truncation-only prompt/context warnings, but it still lacks the requested Control UI near-limit indicator, save-time refusal/confirmation, and archive workflow; open draft PR #64874 is already linked as the implementation lane.

## What This Changes

The issue requests MEMORY.md size visibility, warning thresholds, save-time limit enforcement, startup warnings, and optional archive support before bootstrap truncation loses context.

## Best Possible Solution

Keep this issue open and use #64874, or a maintainer-owned successor, to expose generic bootstrap-file budget metadata through the gateway and Control UI, add clear threshold/save behavior, and split auto-archive into a deliberate follow-up if it is larger than one focused change.

## Reproduction Assessment

Yes. A source-level reproduction is to save a MEMORY.md larger than agents.defaults.bootstrapMaxChars through agents.files.set or the Control UI Files editor; current main accepts the write, and bootstrap injection later truncates it while near-limit-only UI/save warnings remain absent.

## Solution Assessment

No for current main as a complete fix. The narrow maintainable direction is a generic bootstrap-budget status/save-policy path shared by MEMORY.md and other bootstrap files, with archive behavior decided separately if needed.
