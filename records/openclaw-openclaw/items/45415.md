## Summary

Keep open. Current main has useful bootstrap-budget analysis and doctor diagnostics, but it still lacks the requested Control UI capacity indicator, save refusal, near-limit startup warning, and MEMORY.md archive flow; the linked closing PR is still open and draft/conflicting.

## What This Changes

Request warning and enforcement for oversized MEMORY.md files, including Control UI size display, threshold warnings, startup warning, save blocking at the limit, and optional archival.

## Best Possible Solution

Extend the shared bootstrap-budget analyzer into coherent Control UI/status/startup warnings, then make an explicit maintainer decision on opt-in save enforcement and any archival workflow that preserves durable memory safely.

## Reproduction Assessment

Yes, source-reproducible. Current main can analyze near-limit bootstrap files and doctor can report them, but prompt warnings only fire after truncation and the Control UI/gateway save path does not enforce bootstrap budgets.

## Solution Assessment

Unclear as a complete solution. The shared analyzer is the right base, but save refusal and archival need maintainer UX, compatibility, and data-safety decisions beyond a narrow bug fix.
