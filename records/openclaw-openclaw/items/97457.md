## What This Changes

The issue asks OpenClaw to keep the active WebChat session after a Stop abort races with compaction, instead of remapping the session key to a new session and losing context.

## Best Possible Solution

Add focused coverage and a fix for the RPC Stop plus compaction path so user aborts do not leave restartRecoveryRuns or remap the active WebChat session, then validate with live WebChat proof.
