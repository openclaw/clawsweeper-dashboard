## What This Changes

The issue reports that WebChat Stop during compaction can remap the active chat to a new session and lose the prior conversation context.

## Best Possible Solution

Land or replace a narrow fix that terminalizes confirmed WebChat RPC aborts, preserves the selected session through the next send, and proves the Stop plus compaction/restartRecoveryRuns path without weakening the existing compaction and restart-recovery safeguards.
