## What This Changes

The PR adds Windows Scheduled Task enable-state helpers and update-flow wiring to disable enabled Scheduled Task autostart during package replacement, then resume only when this update disabled it.

## Best Possible Solution

Rebase the update-scoped suspension onto current main's mutable-update lifecycle, make the package-versus-git scope explicit, restore only tasks this invocation suspended, and validate the final Windows update lifecycle before merge.

## Reproduction Assessment

## Solution Assessment

Yes in design, but not yet as a merge result. Update-scoped suspension is narrower than changing generic gateway stop semantics, and the branch needs rebase onto current main's mutable-update lifecycle before it can be landable.
