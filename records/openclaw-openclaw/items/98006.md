## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Keep device codes out of group chats:** `extensions/telegram/src/bot-native-commands.ts:215`
  - body: The new Telegram prompter sends the OpenAI device-code URL/code back to the invoking `chatId` and `threadParams`, so `/login codex` in a group/topic posts a redeemable pairing code where other members can use it first. The Telegram maintainer note says pairing is DM-only, and the Mantis candidate screenshot confirms the code appears in `# General`; restrict this to a private owner DM or a DM-only flow before starting the device-code poll.
  - confidence: 0.96
- **[P2] Scope reauth hints to OpenAI/Codex failures:** `src/auto-reply/reply/agent-runner-execution.ts:981`
  - body: `buildExternalRunFailureReply` formats every OAuth refresh failure with `Send /login codex`, regardless of the failed provider. For providers such as xAI or OpenRouter OAuth, that command repairs OpenAI instead of the failed profile, so keep the channel login hint only for OpenAI/Codex-capable failures and leave the provider-specific terminal command otherwise.
  - confidence: 0.9
- **[P2] Re-auth the profile pinned to the target session:** `extensions/telegram/src/bot-native-commands.ts:1354-1358`
  - body: This early Telegram `/login` branch starts `loginFlow` with provider/method/agent only, before the normal command path resolves `targetSessionEntry?.authProfileOverride`. A user whose session is pinned to an expired OpenAI profile can complete `/login codex` and still retry against the unchanged pinned profile; resolve the target session first and pass its profile id into the login flow.
  - confidence: 0.88
- **[P2] Avoid a global /login plugin-command break:** `src/plugins/command-registration.ts:43`
  - body: Adding `login` to the global reserved command set makes existing external plugins with a `login` command fail validation even on channels where this built-in flow is not the right owner. Either keep the reservation scoped to the built-in command path with compatibility coverage or make the plugin API break an explicit maintainer-approved contract change.
  - confidence: 0.8
