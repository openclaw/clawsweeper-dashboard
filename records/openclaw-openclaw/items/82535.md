## Summary

Keep open: the PR targets a real source-reproducible audit false positive, but the patch currently weakens deep code-safety coverage for runtime-reachable test-named modules and still lacks real CLI audit proof.

## What This Changes

The PR adds `excludeTestFiles: true` to the shared audit code-safety helper and adds two regression tests for plugin test-file false positives.

## Best Possible Solution

Exclude non-runtime test fixtures only after preserving manifest-entry runtime graph coverage, then add focused coverage for both the false positive and a reachable test-named runtime import.

## Reproduction Assessment

Yes, source inspection shows current main calls the scanner without `excludeTestFiles`, and the scanner default includes test files. I did not run the audit command because this review is read-only.

## Solution Assessment

No, not as written: reusing the existing exclusion option is the right direction, but deep audit must preserve runtime-reachable file coverage before skipping test-like paths.
