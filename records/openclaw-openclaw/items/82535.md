## Summary

Keep open: the reported deep-audit false positive is still present on current main, but this PR needs real CLI/audit proof and has a security scan-coverage regression before it is merge-ready.

## What This Changes

The PR passes `excludeTestFiles: true` through `getCodeSafetySummary` and adds two `audit-extra.async` regression tests for plugin code-safety scans.

## Best Possible Solution

Land one canonical fix that excludes non-runtime test/spec/mock fixtures only after preserving manifest-entry runtime graph coverage, with focused regression coverage and redacted real deep-audit proof.

## Reproduction Assessment

Yes. Source inspection is high confidence: current main calls the scanner without `excludeTestFiles`, the scanner default includes tests, and https://github.com/openclaw/openclaw/issues/82469 provides concrete Discord audit false-positive examples.

## Solution Assessment

No, not as written. Reusing `excludeTestFiles` is the right direction, but the patch should preserve runtime graph coverage for manifest entries before skipping test-like files and still needs real CLI/audit proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve runtime graph coverage before skipping tests:** `src/security/audit-extra.async.ts:260`
  - body: This passes `excludeTestFiles: true` to the deep audit scan, but that audit only force-includes manifest entries. A plugin runtime entry can import `./payload.test.ts`, and current main would scan that walked file; after this change it is skipped, so dangerous runtime code can disappear from `openclaw security audit --deep`. Add runtime graph coverage or another force-include step before excluding test fixtures.
  - confidence: 0.86
