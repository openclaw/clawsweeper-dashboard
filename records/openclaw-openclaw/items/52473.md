## Summary

Current main and the latest release still handle normal explicit skill slash commands by rewriting the turn into a model prompt instead of loading the selected SKILL.md, so there is no conservative cleanup close reason. The issue remains a clear, bounded feature gap with source-level reproduction and a reasonable repair path.

## What This Changes

The issue requests deterministic SKILL.md injection when a user explicitly invokes a skill through `/skill-name` or `/skill <name>` slash commands.

## Best Possible Solution

Add an explicit-invocation-only path that resolves the selected eligible skill, loads its SKILL.md through the existing skill loader/snapshot contracts, injects it as model-only context before the user args, and leaves natural-language matching plus tool-dispatch skills unchanged.

## Reproduction Assessment

Yes. Source inspection on current main and `v2026.5.19` shows explicit non-tool skill commands resolve to command metadata and args, then rewrite the turn into a model prompt without loading SKILL.md.

## Solution Assessment

Yes. The slash-command-only injection path is the narrow maintainable direction; it should reuse the existing eligible skill snapshot/loader contracts rather than changing natural-language skill matching.
