## Summary

Keep open: current main and the latest release still handle explicit channel skill slash commands for ordinary non-tool skills by rewriting the turn into advisory model text, not by deterministically injecting SKILL.md content. The remaining work is valid, but it needs a maintainer product/security decision because deterministic skill injection changes the instruction-boundary contract.

## What This Changes

Request deterministic SKILL.md injection for explicit slash-invoked non-tool skills so the model receives the skill instructions before handling the user input.

## Best Possible Solution

Define a maintainer-approved explicit skill invocation and trust-boundary contract, then implement one core path that deterministically loads SKILL.md for non-tool slash-invoked skills across channel and session entry points with focused security, prompt, command, docs, and regression coverage.

Yes, at source level: current main recognizes explicit channel skill slash commands, but ordinary non-tool skills are rewritten into advisory model text instead of having SKILL.md injected. I did not run a live model transcript reproduction in this read-only review.

## Solution Assessment

Unclear as a final implementation path: deterministic injection is a plausible fix for explicit user intent, but the best solution must first settle the instruction isolation, security precedence, and shared channel/session contract.
