## Summary

Keep open: current main and v2026.6.8 still handle explicit non-tool skill slash commands by rewriting the user message into advisory model text, not by deterministically injecting SKILL.md content, and the remaining fix needs a maintainer product/security decision.

## What This Changes

Request deterministic SKILL.md loading or context injection for explicit non-tool skill slash commands before the model handles the user's arguments.

## Best Possible Solution

Define the explicit slash-invoked skill loading contract and trust boundary, then implement one core path that deterministically loads SKILL.md for non-tool skill commands with focused command, prompt, security, test, and docs coverage.

## Reproduction Assessment

Yes, at source level: explicit skill slash commands are recognized, but ordinary non-tool skills are rewritten into advisory model text instead of having SKILL.md injected. I did not run a live model transcript reproduction.

## Solution Assessment

Unclear as a final implementation path: deterministic injection is a plausible fix, but maintainers first need to choose the instruction boundary, security semantics, and compatibility contract.
