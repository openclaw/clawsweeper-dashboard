## Summary

Current main and the latest release still use model-mediated handling for normal skill slash commands, so there is no cleanup close reason. The requested deterministic SKILL.md injection path remains a clear, source-provable feature gap.

## What This Changes

Add deterministic SKILL.md loading for explicit `/skill <name>` and direct per-skill slash-command invocation.

## Best Possible Solution

Add an explicit-invocation-only path that resolves the selected eligible skill, loads its SKILL.md, injects those instructions before the user args, and leaves natural-language matching plus tool-dispatch skills unchanged.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.5.18 shows explicit skill commands resolve to command metadata and args, then non-tool skills rewrite the message to a model prompt without reading SKILL.md.

## Solution Assessment

Yes. The slash-command-only injection path is the narrow maintainable direction; it should reuse the existing eligible skill snapshot/loader contracts rather than broadening natural-language matching.
