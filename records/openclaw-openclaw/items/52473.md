## Summary

Keep open. Current main still treats normal skill slash commands as a rewritten model request and relies on prompt guidance for the model to read `SKILL.md`, while only `command-dispatch: tool` has deterministic bypass behavior.

## What This Changes

Request deterministic `SKILL.md` loading and context injection when a user explicitly invokes a skill with `/skill-name` or `/skill <name>`.

## Best Possible Solution

Add an explicit-invocation-only path that loads the selected eligible normal skill and injects its instructions before the user args, while leaving natural-language matching and tool-dispatch skills unchanged.

## Reproduction Assessment

Yes. Source inspection on current main shows `/skill` and direct skill commands resolve to command metadata plus args, then normal commands are rewritten into a model prompt without reading or injecting the selected `SKILL.md`.

## Solution Assessment

Yes. The requested slash-command-only injection is the narrow maintainable direction; it should reuse the eligible skill snapshot and an existing context/message mechanism rather than broadening natural-language skill matching.
