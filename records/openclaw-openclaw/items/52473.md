## Summary

Keep open. Current main and v2026.6.8 still route explicit non-tool skill slash invocations through model-facing prompt text rather than deterministic SKILL.md injection, and the remaining work needs a maintainer product/security decision before implementation.

## What This Changes

The issue requests deterministic SKILL.md context injection when a user explicitly invokes a skill through a slash command.

## Best Possible Solution

Define the explicit slash-invoked skill loading contract, including instruction isolation and trust semantics, then implement the chosen context-injection path with focused skill command, prompt, and docs coverage.

## Reproduction Assessment

Yes, source-level. Current main recognizes explicit skill slash invocations, but non-tool skills are rewritten into advisory model text rather than having SKILL.md content injected.

## Solution Assessment

Unclear as a final implementation path. Deterministic injection is a plausible solution, but the best fix needs a maintainer-approved instruction boundary and compatibility contract first.
