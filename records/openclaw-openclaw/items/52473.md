## Summary

Keep open: current main and v2026.5.22 still rewrite explicit non-tool skill slash commands into advisory prompt text, so SKILL.md loading remains model-mediated and no canonical fix supersedes this request.

## What This Changes

Guarantee that explicit `/skill-name` or `/skill <name>` invocations load and inject the selected skill's SKILL.md before the model handles the user arguments.

## Best Possible Solution

Add an explicit-invocation-only path that resolves the selected eligible skill through existing skill loader or snapshot contracts, injects its SKILL.md as model-only context before user args, and leaves natural-language matching unchanged.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.5.22 shows explicit non-tool skill commands resolve to metadata and args, then rewrite the turn into a model prompt without loading SKILL.md.

## Solution Assessment

Yes. Slash-command-only injection is the narrow maintainable direction; it should reuse existing eligible skill loading contracts rather than changing natural-language skill matching.
