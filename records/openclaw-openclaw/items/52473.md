## Summary

Keep open. Current main still treats normal SKILL.md-backed slash invocations as prompt rewrites, while deterministic behavior exists only for `command-dispatch: tool`; this leaves the requested explicit-invocation injection behavior unimplemented.

## What This Changes

The issue requests deterministic loading and context injection of the selected SKILL.md for explicit `/skill <name>` and direct skill slash command invocations.

## Best Possible Solution

Add an explicit-invocation-only activation path that loads the selected eligible normal skill and injects its SKILL.md instructions into the agent run context before the user args, while leaving natural-language matching model-mediated and tool-dispatch skills unchanged.

## Reproduction Assessment

Yes. Source inspection on current main shows a normal `SkillCommandSpec` without `dispatch` is resolved and rewritten into a generic model prompt, with no code path that reads or injects the selected SKILL.md.

## Solution Assessment

Yes. The requested explicit slash-command-only injection is the narrow maintainable direction; the safer implementation is to reuse the visible/eligible skills snapshot and an existing transcript-safe context mechanism rather than broadening natural-language autoloading.
