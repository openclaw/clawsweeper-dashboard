## Summary

Keep open. Current main and the latest release still convert explicit skill slash invocations into a normal model request that asks the model to use the skill, while the system prompt still relies on the model voluntarily reading SKILL.md; no merged or viable related item currently owns this exact deterministic slash-injection behavior.

## What This Changes

The issue requests deterministic loading or injection of the selected SKILL.md content when a user explicitly invokes a skill slash command.

## Best Possible Solution

Define one canonical explicit skill-command activation contract, then implement it across text and native command paths so non-tool skill invocations load the selected SKILL.md before the model turn while preserving command-dispatch: tool.

## Reproduction Assessment

Yes, at source level: current main rewrites explicit skill slash commands into a normal model request and leaves SKILL.md loading to prompt guidance. I did not run a live model transcript, but the missing deterministic injection path is clear in source and docs.

## Solution Assessment

Unclear. The requested direction is plausible, but maintainers should first choose the hidden-message/tool-result/system-context contract and whether this applies to direct skill aliases, generic `/skill`, and native command surfaces.
