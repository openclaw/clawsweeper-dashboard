## Summary

Keep open: current main and v2026.5.22 still turn explicit non-tool skill slash commands into advisory model text, so SKILL.md loading remains model-mediated and no merged/open canonical item supersedes this explicit-invocation request.

## What This Changes

The issue asks OpenClaw to deterministically inject SKILL.md content when a user explicitly invokes a user-invocable skill through `/skill-name` or `/skill <name>`.

## Best Possible Solution

Implement deterministic context injection for explicit non-tool skill slash commands using the already-authorized resolved skill entry, while preserving `command-dispatch: tool`, bundle prompt templates, and natural-language skill matching behavior.

## Reproduction Assessment

Yes from source inspection, not live execution: current main resolves explicit skill slash commands but only rewrites the prompt to ask the model to use the skill. This read-only review did not run tests or a live gateway session.

## Solution Assessment

Yes: for explicit slash invocation, deterministic SKILL.md injection is the narrowest maintainable direction, provided it reuses existing skill eligibility/authorization and leaves tool-dispatched skills unchanged.
