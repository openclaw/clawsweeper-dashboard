## Summary

Keep open. Current main and v2026.6.6 still route explicit non-tool skill slash invocations as a normal model-backed request, so SKILL.md loading remains advisory rather than deterministic; this needs a maintainer product/API decision, not stale cleanup.

## What This Changes

Request deterministic injection of the invoked skill's SKILL.md content for explicit `/skill` or direct skill slash commands before the model turn.

## Best Possible Solution

Define one canonical explicit skill-command activation contract, inject the selected skill content through that chosen context channel for text and native non-tool invocations, preserve `command-dispatch: tool`, and update focused tests and docs.

## Reproduction Assessment

Yes, at source level: current main rewrites explicit non-tool skill slash commands into a normal model request and leaves SKILL.md loading to advisory prompt guidance. I did not run a live model transcript, but the missing deterministic injection path is clear in source and docs.

## Solution Assessment

Unclear. The requested direction is plausible, but maintainers should first choose the hidden-message, system-context, or synthetic-tool-result contract and exactly which slash/native command surfaces it covers.
