## Summary

Keep open. Current main still rewrites normal skill slash invocations into a model request and only handles `command-dispatch: tool` deterministically, so the requested automatic `SKILL.md` injection remains unimplemented.

## What This Changes

Request deterministic `SKILL.md` content injection when a user explicitly invokes a normal skill slash command.

## Best Possible Solution

Add an explicit-invocation-only activation path that loads the selected eligible normal skill and injects its instructions before the user args, while leaving natural-language matching and tool-dispatch skills unchanged.

## Reproduction Assessment

Yes. Source inspection on current main shows `/skill` and direct skill commands resolve to `{ command, args }`, then normal commands are rewritten to a model prompt without reading or injecting `SKILL.md`.

## Solution Assessment

Yes. The slash-command-only injection direction is the narrow maintainable fix; it should reuse the visible/eligible skill snapshot and an existing context mechanism rather than broadening natural-language skill matching.
