## Summary

Keep open: current main and the latest release still route explicit non-tool skill slash commands into an advisory model prompt instead of loading the selected SKILL.md, and the remaining work is not superseded by a canonical fix.

## What This Changes

The issue requests deterministic SKILL.md loading when a user explicitly invokes a skill with `/skill-name` or `/skill <name>`.

## Best Possible Solution

Add an explicit-invocation-only path that resolves the selected eligible skill through existing skill loader/snapshot contracts, injects its SKILL.md as model-only context before user args, and leaves natural-language matching unchanged.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.5.20 shows explicit non-tool skill commands resolve to command metadata and args, then rewrite the turn into a model prompt without loading SKILL.md.

## Solution Assessment

Yes. Slash-command-only injection is the narrow maintainable direction; it should reuse the existing eligible skill loading contracts rather than changing natural-language skill matching.
