## Summary

Keep open. Current main and the latest shipped release still route explicitly invoked normal skill commands through a model-mediated rewrite instead of loading and injecting the selected SKILL.md, so the requested deterministic activation path remains unimplemented.

## What This Changes

Request deterministic SKILL.md injection for explicitly invoked normal skill slash commands instead of relying on the model to read the skill.

## Best Possible Solution

Add an explicit-invocation-only activation path that loads the selected eligible normal skill and injects its instructions before the user args, while leaving natural-language matching and tool-dispatch skills unchanged.

## Reproduction Assessment

Yes. Source inspection on current main shows `/skill` and direct skill commands resolve to command metadata and args, then normal commands are rewritten to a model prompt without reading or injecting SKILL.md.

## Solution Assessment

Yes. The slash-command-only injection direction is the narrow maintainable fix; it should reuse the eligible skill snapshot and an existing context mechanism rather than broaden natural-language skill matching.
