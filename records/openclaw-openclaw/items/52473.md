## Summary

Keep open: current main and v2026.5.19 still forward normal skill slash commands through model-mediated prompt guidance, so there is no cleanup close reason. The issue remains source-reproducible and narrow enough for a focused repair lane.

## What This Changes

Request deterministic injection of the selected skill's SKILL.md instructions when invoked via `/skill-name` or `/skill <name>` slash command.

## Best Possible Solution

Add an explicit-invocation-only path that resolves the selected eligible skill, loads its SKILL.md, injects those instructions before the user args, and leaves natural-language matching plus tool-dispatch skills unchanged.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.5.19 shows explicit skill commands resolve to command metadata and args, then non-tool skills rewrite the message to a model prompt without reading SKILL.md.

## Solution Assessment

Yes. The slash-command-only injection path is the narrow maintainable direction; it should reuse existing eligible skill snapshot/loader contracts rather than broaden natural-language matching.
