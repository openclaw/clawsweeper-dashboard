The related deterministic tool-dispatch work solves a different path and does not supersede this request.

## What This Changes

Request deterministic SKILL.md context injection when a user explicitly invokes a normal skill with `/skill-name` or `/skill <name>`.

## Best Possible Solution

Implement deterministic context injection for explicit non-tool skill slash commands using the already-resolved eligible skill metadata, while preserving `command-dispatch: tool`, bundle prompt templates, and natural-language skill matching behavior.

## Reproduction Assessment

Yes, from source inspection: explicit non-tool skill slash commands resolve to a skill invocation but only rewrite the user body to an advisory model request. I did not run a live gateway session in this read-only review.

## Solution Assessment

Yes, deterministic injection for explicit slash invocation is the narrowest maintainable direction, as long as it reuses existing skill eligibility/snapshot data and leaves tool-dispatched skills unchanged.
