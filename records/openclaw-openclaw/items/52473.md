## Best Possible Solution

Implement deterministic SKILL.md injection only for authorized explicit non-tool skill slash invocations, while preserving `command-dispatch: tool`, bundle prompt templates, and natural-language skill matching as separate paths.

Yes. Deterministic injection for explicit slash invocation is the narrowest maintainable direction as long as it reuses existing skill eligibility and authorization and leaves tool-dispatched skills unchanged.

Preserve `command-dispatch: tool`, Claude bundle promptTemplate commands, authorization behavior, skill eligibility/filtering, natural-language skill matching, and command docs.
