## Summary

Keep open: current main still does not implement deterministic SKILL.md injection for explicit non-tool skill slash invocations. The request remains valid, but the fix needs maintainer product/security direction because it changes how untrusted skill instructions enter agent context.

## What This Changes

The issue requests deterministic injection of a skill's SKILL.md content when a user explicitly invokes that skill via slash command.

## Best Possible Solution

Define a maintainer-approved explicit skill invocation and trust-boundary contract, then implement one core path that deterministically loads SKILL.md for non-tool slash-invoked skills across channel and session entry points with focused security, prompt, command, docs, and regression coverage.

## Reproduction Assessment

Yes. At source level, current main recognizes explicit channel skill slash commands, but ordinary non-tool skills are rewritten into advisory model text instead of having SKILL.md injected.

## Solution Assessment

Unclear as a final implementation path. Deterministic injection is a plausible fix for explicit user intent, but the best solution must first settle instruction isolation, security precedence, and the shared channel/session contract.
