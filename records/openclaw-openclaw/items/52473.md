## Summary

Current `main` still has the model-mediated skill slash-command behavior described by the issue, so there is no cleanup close reason yet; the requested deterministic SKILL.md injection path remains useful and source-provable.

## What This Changes

Request deterministic loading/injection of a selected skill's `SKILL.md` when a user explicitly invokes that skill via `/skill-name` or `/skill <name>`.

## Best Possible Solution

Add an explicit-invocation-only path that resolves the selected eligible skill, loads its `SKILL.md`, and injects those instructions before the user args while leaving natural-language matching and tool-dispatch skills unchanged.

## Reproduction Assessment

Yes. Source inspection on current `main` shows `/skill` and direct skill commands resolve to command metadata and args, then normal commands are rewritten into a model prompt without reading or injecting `SKILL.md`.

## Solution Assessment

Yes. The requested slash-command-only injection path is the narrow maintainable direction; it should reuse the existing eligible skill snapshot/loader contracts instead of broadening natural-language skill matching.
