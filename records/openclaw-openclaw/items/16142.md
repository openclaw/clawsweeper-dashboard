## Summary

Keep open. Current main and the latest release still route main-session cron systemEvent reminder text through heartbeat cron-event prompting and the model-backed reply path, while related static/direct cron trackers remain open and do not replace dynamic watcher-generated reminders.

## What This Changes

Request a low-cost deterministic path for simple cron reminders so OpenClaw does not use the model as a string formatter for repeated watcher alerts.

## Best Possible Solution

Define a maintainer-approved low-cost cron delivery path for simple main-session systemEvent reminders, while preserving the model-backed path for complex or reasoning-dependent cron events.

## Reproduction Assessment

Yes, at source level. Main-session cron enqueues systemEvent text with a cron context key, heartbeat selects buildCronEventPrompt, and getReplyFromConfig sends that prompt through the model-backed reply path.

## Solution Assessment

Unclear. Deterministic delivery is plausible for simple reminders, but maintainers need to define which cron events may bypass the model and how that relates to the open static-message and direct-exec cron contracts.
