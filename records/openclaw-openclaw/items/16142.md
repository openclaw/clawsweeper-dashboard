## Summary

Keep open. Current main still routes tagged main-session cron reminder text through the heartbeat cron-event prompt and model-backed reply path, while the requested low-cost deterministic delivery path remains a product/API boundary decision.

## What This Changes

Requests a low-cost deterministic path for simple cron reminders instead of using the OpenAI/ChatGPT model as a reminder formatter.

## Best Possible Solution

Define a narrow low-cost path for simple main-session reminders, preserve the model-backed cron-event path for complex cases, and coordinate docs/tests with https://github.com/openclaw/openclaw/pull/48883.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a main-session cron systemEvent is enqueued with a cron context key, heartbeat selects `buildCronEventPrompt`, and `getReplyFromConfig` handles the model-backed reply.

## Solution Assessment

Unclear. Deterministic formatting is reasonable for simple reminders, but maintainers need to define which cron events may bypass the model and coordinate that boundary with https://github.com/openclaw/openclaw/pull/48883.
