## Summary

Close as implemented: current main and v2026.6.5 now provide command cron jobs for deterministic watcher scripts, so users can run and deliver preformatted Slack/email-style checks without an agent/model call while the legacy systemEvent reminder path remains model-mediated by design.

## What This Changes

The issue asks OpenClaw to stop using the OpenAI model as a formatter for repeated simple cron reminders because that can create high API usage and throttling.

## Best Possible Solution

Keep the shipped command cron path as the core no-token solution for deterministic watcher output, and leave legacy systemEvent cron reminders model-mediated when they need reasoning or rewriting.

## Reproduction Assessment

Yes, source-level: the legacy main-session systemEvent path still queues cron text into heartbeat and can call the model-backed reply path, while current main also exposes a command cron path that avoids that model call for deterministic watcher output.

## Solution Assessment

Yes for the central cost problem: command cron is a narrower maintainable solution than bypassing the model for every systemEvent reminder because it preserves existing reminder semantics and gives deterministic scripts a direct delivery path.
