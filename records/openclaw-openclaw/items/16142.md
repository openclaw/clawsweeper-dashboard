## Summary

Keep open. Current main still routes main-session cron `systemEvent` reminder text through the heartbeat cron-event prompt and model-backed reply path; the related remediation PRs and static-message tracker are open rather than shipped fixes.

## What This Changes

The issue requests avoiding model-backed formatting for simple scheduled reminder and cron messages that can be delivered deterministically.

## Best Possible Solution

Define and land a maintainer-approved low-cost path for simple main-session cron reminders while preserving the model-backed path for complex, internal, tool-using, and response-tool cron events.

## Reproduction Assessment

Yes, at source level. Main-session cron enqueues `systemEvent` text with a `cron:` context key, heartbeat selects `buildCronEventPrompt`, and `getReplyFromConfig` sends that prompt through the model-backed reply path.

## Solution Assessment

Unclear. Deterministic formatting is plausible for simple reminders, but maintainers need to define which cron events may bypass the model and coordinate with the open remediation/static-message work.
