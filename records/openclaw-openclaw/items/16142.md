## Summary

Keep open. Current main still routes tagged main-session cron reminders through the heartbeat cron-event prompt and model-backed reply path, while the requested deterministic low-cost delivery path remains a product/API decision and overlaps the open cron-wrapper remediation in #48883.

## What This Changes

The issue requests replacing model-backed formatting of simple cron reminders with deterministic formatting to reduce OpenAI/ChatGPT usage.

## Best Possible Solution

Define a narrow low-cost path for simple main-session reminders, preserve the model-backed cron-event path for complex cases, and align the behavior with #48883 in docs and focused tests.

## Reproduction Assessment

Yes. A high-confidence source reproduction path exists: a main-session cron systemEvent is enqueued with a cron context key, heartbeat selects buildCronEventPrompt, and getReplyFromConfig handles the reply path.

## Solution Assessment

Unclear. Deterministic formatting is reasonable for simple reminders, but the maintainable fix needs a maintainer-defined boundary for which cron events may bypass the model and should account for #48883.
