## Summary

Keep open. Current main and the latest release still route main-session cron reminder text through the heartbeat cron-event prompt and model-backed reply path, so the cost/throttling concern remains valid but needs a maintainer product decision before implementation.

## What This Changes

Request a deterministic low-cost path so simple cron reminders are delivered without using ChatGPT/OpenAI as a string formatter.

## Best Possible Solution

Add an explicit literal-notification cron payload or delivery mode for trusted simple reminders, while keeping systemEvent on the existing model-backed heartbeat path for reasoning-dependent events.

## Reproduction Assessment

Yes, at source level: a main-session cron job with non-empty systemEvent text enters heartbeat, becomes a cron-event prompt, and calls getReplyFromConfig. I did not repeat the reporter's live API logging or measure token volume.

## Solution Assessment

No, the requested blanket string-formatting bypass is not the safest general solution because system events can require agent semantics. A separate explicit literal-delivery payload or mode is the more maintainable direction.
