## Summary

Keep open. Current main and v2026.5.19 still route main-session cron systemEvent reminders through the heartbeat cron-event prompt and model-backed reply path, while the related no-LLM/static cron and remediation work is open rather than a shipped exact fix.

## What This Changes

Request a deterministic or no-LLM path for simple cron reminder formatting and delivery so OpenAI is not used as a repeated string formatter.

## Best Possible Solution

Define and land a maintainer-approved low-cost path for simple cron reminders, likely as an additive direct/static delivery contract or conservative simple-payload mode, while preserving the existing model-backed cron-event path for complex, internal, tool-using, and reasoning-dependent events.

## Reproduction Assessment

Yes, at source level. Main-session cron enqueues systemEvent text with a cron context key, heartbeat selects buildCronEventPrompt, and getReplyFromConfig sends that prompt through the model-backed reply path.

## Solution Assessment

Unclear. Deterministic delivery is plausible for simple reminders, but maintainers need to define which cron events may bypass the model and how that relates to static-message, direct-exec, and direct-tool cron contracts.
