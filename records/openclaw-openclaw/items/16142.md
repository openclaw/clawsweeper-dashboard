## Summary

Keep open. Current main and the latest release still route main-session cron reminder text through the heartbeat cron-event prompt and model-backed reply path, so the cost/throttling concern remains valid and needs a maintainer product decision before implementation.

## What This Changes

Issue asks OpenClaw to avoid using ChatGPT/OpenAI as a formatter for simple scheduled Slack/email reminder text that fires repeatedly.

## Best Possible Solution

Add an explicit literal-notification cron payload or delivery mode for trusted simple reminders, while preserving `systemEvent` as the model-backed heartbeat path for reasoning-dependent events.

## Reproduction Assessment

Yes, at source level: a main-session cron job with non-empty `systemEvent` text enters heartbeat, becomes a cron-event prompt, and calls `getReplyFromConfig`. I did not repeat the reporter's live OpenAI logging or measure exact token volume.

## Solution Assessment

No, the requested blanket string-formatting bypass is not the safest general solution because `systemEvent` can require agent semantics. A separate explicit literal-delivery payload or mode is the more maintainable direction.
