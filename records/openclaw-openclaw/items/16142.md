## Summary

Keep open. Current main and v2026.6.10 still route main-session cron systemEvent reminders through heartbeat's cron-event prompt and `getReplyFromConfig`; the merged de-duplication work removed duplicate raw `System:` surfacing but did not address the remaining model-backed formatting/cost path.

## What This Changes

Request a lower-cost deterministic path for simple cron reminder formatting so recurring main-session reminders do not call the OpenAI/ChatGPT model every tick.

## Best Possible Solution

Add an explicit literal-notification cron payload or delivery mode for trusted simple reminders, while preserving `systemEvent` as the model-backed heartbeat path for reasoning-dependent events.

## Reproduction Assessment

Yes. At source level, a main-session cron job with non-empty `systemEvent` text enters heartbeat, becomes a cron-event prompt, and calls `getReplyFromConfig`; I did not repeat the reporter's live OpenAI logging.

## Solution Assessment

No, not as a blanket string-formatting bypass. The maintainable solution is a narrow explicit literal-notification mode or conservative simple-reminder boundary, leaving complex cron events on the existing model-backed heartbeat path.
