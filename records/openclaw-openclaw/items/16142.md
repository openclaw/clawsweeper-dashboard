## Summary

Keep open. Current main still routes main-session cron systemEvent text into the heartbeat cron-event model prompt and `getReplyFromConfig`, and the overlapping remediation at https://github.com/openclaw/openclaw/pull/48883 is open with review blockers rather than merged.

## What This Changes

The issue requests a deterministic low-cost delivery path for simple cron reminders instead of sending reminder formatting through the OpenAI model path every few minutes.

## Best Possible Solution

Define and document a narrow low-cost main-session cron reminder path while preserving the existing model-backed cron-event behavior for complex, internal, tool-using, or reasoning-dependent events.

## Reproduction Assessment

Yes. Source inspection shows a main-session cron `systemEvent` is enqueued with a `cron:` context key, selected by heartbeat as a cron event, wrapped by `buildCronEventPrompt`, and sent through `getReplyFromConfig`.

## Solution Assessment

Unclear. Deterministic formatting is reasonable for simple reminders, but the maintainable fix needs a maintainer-defined boundary and should coordinate with https://github.com/openclaw/openclaw/pull/48883.
