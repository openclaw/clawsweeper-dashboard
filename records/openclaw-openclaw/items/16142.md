## Summary

Current main and v2026.6.5 now provide command cron jobs that run deterministic watcher scripts without an agent/model call and can announce captured output, so the central token-spend problem is implemented even though legacy systemEvent reminders still keep their model-mediated semantics.

## What This Changes

The issue requests a lower-cost deterministic path so simple scheduled reminders do not route through ChatGPT/OpenAI as a string formatter.

## Best Possible Solution

Keep the shipped command cron path as the core no-token solution for deterministic watcher output, while leaving legacy systemEvent reminders model-mediated when they need rewriting, reasoning, or existing heartbeat semantics.

## Reproduction Assessment

Yes, source-level. The legacy systemEvent cron path still goes through the heartbeat cron-event prompt and model-backed reply path, while current main now provides a separate command cron path that avoids that call for deterministic watcher output.

## Solution Assessment

Yes. The shipped command cron solution is narrower than bypassing the model for every systemEvent reminder because it preserves existing reminder semantics while giving deterministic scripts a direct delivery path.
