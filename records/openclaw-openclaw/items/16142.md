## Summary

Keep open. Current main and the latest shipped tag still route main-session cron reminder text through heartbeat cron-event prompting and the model-backed reply path, and the open static-message/direct-exec/direct-invoke tracks do not yet ship a replacement for dynamic watcher-generated reminders.

## What This Changes

The issue requests a low-cost deterministic formatting or delivery path for simple cron reminders instead of repeatedly routing reminder text through the OpenAI model-backed heartbeat reply path.

## Best Possible Solution

Define a maintainer-approved low-cost cron delivery path for simple main-session systemEvent reminders, likely through an explicit direct/static delivery option or conservative simple-payload boundary, while preserving the model-backed path for complex or reasoning-dependent cron events.

## Reproduction Assessment

Yes, at source level. Main-session cron enqueues systemEvent text with a cron context key, heartbeat selects buildCronEventPrompt, and getReplyFromConfig sends that prompt through the model-backed reply path.

## Solution Assessment

Unclear. Deterministic delivery is plausible for simple reminders, but maintainers need to define which cron events may bypass the model and how that relates to the open static-message, direct-exec, and direct-invoke cron contracts.
