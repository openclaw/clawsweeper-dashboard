## Summary

Keep open. Current main still routes main-session cron systemEvent/reminder text into the heartbeat cron-event prompt and model-backed reply path, so the reported OpenAI overuse/cost concern remains real; the related remediation at https://github.com/openclaw/openclaw/pull/48883 is still open rather than merged.

## What This Changes

The issue asks for a low-cost deterministic path so simple cron/reminder text is not repeatedly sent through ChatGPT as a string formatter.

## Best Possible Solution

Define and land a maintainer-approved low-cost path for simple main-session cron reminders while preserving the existing model-backed cron-event path for complex, internal, tool-using, and response-tool cases.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: main-session cron enqueues systemEvent text, heartbeat selects buildCronEventPrompt, and getReplyFromConfig sends the cron-event prompt through the model-backed reply path.

## Solution Assessment

Unclear. Deterministic formatting is reasonable for simple reminders, but a blanket bypass is not safe until maintainers define which cron events may bypass the model and how that coordinates with the open remediation PR.
