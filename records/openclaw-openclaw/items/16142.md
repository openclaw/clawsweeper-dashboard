## Summary

Keep open. Current main still routes main-session cron reminder text through the heartbeat cron-event prompt and model-backed reply path, so the cost/throttle concern remains valid and needs a maintainer product boundary rather than cleanup closure.

## What This Changes

The issue requests a lower-cost deterministic delivery path for simple cron reminders instead of repeatedly using OpenAI model calls as a formatter.

## Best Possible Solution

Define a narrow low-cost delivery mode for simple main-session cron reminders while preserving the model-backed cron-event path for complex, internal, tool-using, or reasoning-dependent events, and reconcile the boundary with #48883.

## Reproduction Assessment

Yes. A high-confidence source reproduction path exists: enqueue a tagged main-session cron `systemEvent`, run heartbeat, and observe `buildCronEventPrompt` plus `getReplyFromConfig`; current tests assert this behavior.

## Solution Assessment

Unclear. Deterministic formatting is a good direction for simple reminders, but the safest implementation needs a maintainer-defined boundary or explicit mode and must account for the related open cron-wrapper remediation PR.
