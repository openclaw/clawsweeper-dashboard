## Summary

Keep open. Current main still routes main-session cron reminder text through the heartbeat cron-event prompt and model-backed reply path, while the related remediation at https://github.com/openclaw/openclaw/pull/48883 remains open and not landed.

## What This Changes

Request a low-cost deterministic delivery path so simple cron/heartbeat reminder text is not repeatedly sent through the OpenAI model as formatting work.

## Best Possible Solution

Define and land a maintainer-approved low-cost path for simple main-session cron reminders while preserving the model-backed cron-event path for complex, internal, tool-using, and response-tool cases.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: main-session cron enqueues systemEvent text, heartbeat selects buildCronEventPrompt, and getReplyFromConfig sends the cron-event prompt through the model-backed reply path.

## Solution Assessment

Unclear. Deterministic formatting is reasonable for simple reminders, but a blanket model bypass is not safe until maintainers define which cron events may bypass the model and how that coordinates with the open remediation PR.
