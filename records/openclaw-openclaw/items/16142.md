## Summary

Keep open. Current main still routes main-session cron reminder text through a cron-tagged heartbeat prompt and model-backed reply call, so the reported API cost/throttle problem remains real rather than cleanup-closeable.

## What This Changes

The issue asks OpenClaw to stop using ChatGPT/OpenAI as a formatter for simple recurring cron reminder messages and use deterministic formatting instead.

## Best Possible Solution

Define a narrow low-cost delivery mode for simple main-session cron reminders, preferably through an explicit cron option or conservative simple-payload detector, while preserving the model-backed cron-event path for complex or reasoning-dependent events.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: run a main-session cron systemEvent with wake now, observe executeMainSessionCronJob enqueue a cron-tagged system event, then heartbeat builds buildCronEventPrompt and calls getReplyFromConfig; current tests assert that behavior.

## Solution Assessment

Unclear. Deterministic formatting is likely the right direction for simple reminders, but replacing the cron-event model path broadly is not safe until maintainers decide which cron events may bypass model interpretation and how this interacts with #48883.
