## Summary

Keep open. Current main and the latest release still route main-session cron `systemEvent` reminder text through the heartbeat cron-event prompt and model-backed reply path; related no-token/static cron work remains open rather than a shipped replacement.

## What This Changes

The issue requests a lower-cost deterministic path for simple cron reminder delivery instead of using ChatGPT/OpenAI as a formatter for repeated reminder text.

## Best Possible Solution

Define and land a maintainer-approved low-cost path for simple cron reminders, likely as an additive static/direct payload or explicit deterministic option, while preserving the existing model-backed path for complex, internal, tool-using, and reasoning-dependent cron events.

## Reproduction Assessment

Yes, at source level. Main-session cron enqueues `systemEvent` text with a `cron:` context key, heartbeat selects `buildCronEventPrompt`, and `getReplyFromConfig` sends that prompt through the model-backed reply path.

## Solution Assessment

Unclear. Deterministic formatting or direct delivery is plausible for simple reminders, but maintainers need to define which cron events may bypass the model and how that relates to the open static-message cron contract.
