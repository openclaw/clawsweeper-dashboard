## Summary

Keep open. Current main still routes main-session cron `systemEvent` reminder text through the heartbeat cron-event prompt and model-backed reply path; the overlapping remediation at https://github.com/openclaw/openclaw/pull/48883 is open, and the no-token static-message tracker at https://github.com/openclaw/openclaw/issues/11473 is adjacent rather than a shipped fix for dynamic cron reminder text.

## What This Changes

The issue requests a low-cost deterministic path for recurring cron reminder text so OpenAI/ChatGPT is not used as a formatter every few minutes.

## Best Possible Solution

Define and land a maintainer-approved low-cost path for simple main-session cron reminders while preserving the model-backed path for complex, internal, tool-using, and response-tool cron events.

## Reproduction Assessment

Yes, at source level. Main-session cron enqueues `systemEvent` text with a `cron:` context key, heartbeat selects `buildCronEventPrompt`, and `getReplyFromConfig` sends that prompt through the model-backed reply path.

## Solution Assessment

Unclear. Deterministic formatting is plausible for simple reminders, but the maintainable solution needs an explicit boundary for which cron events may bypass the model and must coordinate with the open remediation PR.
