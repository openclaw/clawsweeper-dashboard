## Summary

Keep open. Current main and v2026.5.20 still route main-session cron `systemEvent` reminder text through the heartbeat cron-event prompt and `getReplyFromConfig`; no merged deterministic/no-model delivery path exists, and the related static-message/direct-exec cron tracks do not fully replace dynamic watcher-generated reminder text.

## What This Changes

Request a low-cost deterministic path for simple cron-generated reminder text so OpenClaw does not call ChatGPT/OpenAI just to reformat and relay recurring messages.

## Best Possible Solution

Define a maintainer-approved low-cost cron delivery path for simple main-session `systemEvent` reminders, likely through an explicit direct/static option or conservative simple-payload boundary, while preserving the model-backed path for complex cron events.

## Reproduction Assessment

Yes, at source level. Main-session cron enqueues `systemEvent` text with a `cron:` context key, heartbeat selects `buildCronEventPrompt`, and `getReplyFromConfig` sends that prompt through the model-backed reply path.

## Solution Assessment

Unclear. Deterministic formatting or direct delivery is plausible for simple reminders, but maintainers need to define which cron events may bypass the model and how that relates to the open static-message and direct-exec cron contracts.
