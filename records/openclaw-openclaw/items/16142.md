## Summary

Keep open. Current main still routes main-session cron system-event text through the heartbeat cron-event prompt and model-backed reply path; the requested low-cost deterministic delivery path remains a product/API boundary and overlaps, but is not solved by, the open cron-wrapper remediation PR.

## What This Changes

The issue requests a lower-cost deterministic delivery path so simple cron reminders do not use the ChatGPT/OpenAI model as a string formatter every few minutes.

## Best Possible Solution

Define a narrow low-cost path for simple main-session reminders, preserve the model-backed cron-event path for complex cases, and align docs/tests with the chosen boundary and https://github.com/openclaw/openclaw/pull/48883.

## Reproduction Assessment

Yes. Source inspection provides a high-confidence path: main-session cron enqueues systemEvent text with a cron context key, heartbeat selects buildCronEventPrompt, and getReplyFromConfig handles the model-backed reply.

## Solution Assessment

Unclear. Deterministic formatting is reasonable for simple reminders, but a blanket model bypass is not the best maintainable fix until maintainers define which cron events may bypass the model and how that coordinates with https://github.com/openclaw/openclaw/pull/48883.
