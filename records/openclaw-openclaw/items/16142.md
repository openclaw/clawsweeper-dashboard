## Summary

Keep open. Current main still routes main-session cron systemEvent reminder text through the heartbeat cron-event prompt and model-backed reply path, and the overlapping remediation at https://github.com/openclaw/openclaw/pull/48883 remains open with proof/check gaps, so there is no conservative close reason yet.

## What This Changes

Request a deterministic, lower-cost delivery path for simple cron reminder/systemEvent text instead of repeatedly using ChatGPT/OpenAI for formatting.

## Best Possible Solution

Define and document a narrow low-cost path for simple main-session reminders while preserving the model-backed cron-event path for complex, internal, tool-using, or reasoning-dependent events.

## Reproduction Assessment

Yes, at source level. Main-session cron enqueues systemEvent text with a cron context key, heartbeat builds the cron-event prompt, and getReplyFromConfig sends it through the model-backed reply path.

## Solution Assessment

Unclear. Deterministic formatting is reasonable for simple reminders, but a blanket bypass is not the best maintainable fix until maintainers define which cron events may bypass the model and how that coordinates with https://github.com/openclaw/openclaw/pull/48883.
