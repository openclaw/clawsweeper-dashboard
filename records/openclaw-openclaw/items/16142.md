## Summary

Keep open. Current main still routes main-session cron systemEvent text into the heartbeat cron-event prompt and model-backed reply path, while the overlapping remediation in https://github.com/openclaw/openclaw/pull/48883 remains open and has not settled the low-cost deterministic reminder boundary.

## What This Changes

Request a lower-cost deterministic delivery path for simple main-session cron reminders instead of sending reminder prose through ChatGPT/OpenAI every interval.

## Best Possible Solution

Define a narrow, documented low-cost path for simple main-session reminders, while preserving the model-backed cron-event path for complex, internal, tool-using, or reasoning-dependent cron events.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: main-session cron enqueues systemEvent text, heartbeat selects buildCronEventPrompt, and getReplyFromConfig sends the prompt through the model-backed reply path.

## Solution Assessment

Unclear. Deterministic formatting is reasonable for simple reminders, but a blanket bypass is not the best maintainable fix until maintainers define which cron events may bypass the model and how that coordinates with https://github.com/openclaw/openclaw/pull/48883.
