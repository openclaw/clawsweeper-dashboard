## Summary

Keep open. Current main and v2026.6.10 still send main-session cron reminder text through heartbeat's cron-event prompt and `getReplyFromConfig`; the merged de-duplication work only removed duplicate raw `System:` surfacing, not the remaining model-backed formatting/cost path.

## What This Changes

The issue requests deterministic formatting or delivery for simple cron reminders so repeated watcher notifications do not consume OpenAI/ChatGPT API calls as a string formatter.

## Best Possible Solution

Add an explicit literal-notification cron payload or delivery mode for trusted simple reminders, while preserving `systemEvent` as the model-backed heartbeat path for reasoning-dependent events.

## Reproduction Assessment

Yes. At source level, a main-session cron job with non-empty `systemEvent` text enters heartbeat, becomes a cron-event prompt, and calls `getReplyFromConfig`; I did not repeat the reporter's live OpenAI logging.

## Solution Assessment

No, not as a blanket string-formatting bypass. The maintainable path is a narrow explicit literal-notification mode or payload boundary, leaving complex cron events on the existing model-backed heartbeat path.
