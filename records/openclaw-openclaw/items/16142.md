## Summary

Keep open. Current main and v2026.5.12 still route main-session cron systemEvent reminder text through the heartbeat cron-event prompt and model-backed reply path, while the overlapping remediation at https://github.com/openclaw/openclaw/pull/48883 is still open and does not settle the deterministic low-cost reminder boundary.

## What This Changes

Request to stop using ChatGPT/OpenAI as the formatter for simple cron reminder text and add a lower-cost deterministic delivery path.

## Best Possible Solution

Define and document a narrow low-cost path for simple main-session reminders while preserving the model-backed cron-event path for complex, internal, tool-using, or reasoning-dependent events.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: main-session cron enqueues systemEvent text, heartbeat selects buildCronEventPrompt, and getReplyFromConfig sends the prompt through the model-backed reply path.

## Solution Assessment

Unclear. Deterministic formatting is reasonable for simple reminders, but a blanket bypass is not the best maintainable fix until maintainers define which cron events may bypass the model and how that coordinates with https://github.com/openclaw/openclaw/pull/48883.
