## Summary

Keep open. Current main still routes main-session cron reminder text through the heartbeat cron-event prompt and model-backed reply path, so the API-cost concern remains valid and should be resolved with a maintainer-defined product boundary rather than cleanup closure.

## What This Changes

Request a low-cost deterministic delivery or formatting path for simple cron/main-session reminders so OpenAI is not repeatedly invoked as a string formatter.

## Best Possible Solution

Define a narrow low-cost path for simple main-session reminders, while preserving the existing model-backed cron-event path for complex, internal, tool-using, or reasoning-dependent events and reconciling the design with #48883.

## Reproduction Assessment

Yes. A high-confidence source reproduction path exists: enqueue a tagged main-session cron `systemEvent`, wake heartbeat with `target: "last"`, and observe `buildCronEventPrompt` followed by `getReplyFromConfig`; current tests assert this path.

## Solution Assessment

Unclear. Deterministic formatting is a reasonable direction for simple reminders, but the maintainable fix needs a product/API boundary for which cron events may bypass the model and should be reconciled with #48883.
