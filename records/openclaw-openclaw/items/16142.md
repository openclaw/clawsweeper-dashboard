## Summary

Keep open. Current main and the latest release still route main-session cron reminder text through the heartbeat cron-event prompt and model-backed reply path, so the reported cost/throttling concern remains real and needs a maintainer product decision rather than cleanup closure.

## What This Changes

The issue requests a lower-cost cron reminder delivery path so simple Slack/email watcher reminders are not repeatedly routed through ChatGPT only to reformat text.

## Best Possible Solution

Add an explicit literal-notification cron payload or delivery mode that sends trusted text through normal channel delivery without an agent turn, while retaining systemEvent for reasoning, tools, filtering, and summarization.

## Reproduction Assessment

Yes at source level: a main-session cron job with non-empty systemEvent text enters heartbeat, becomes a cron-event prompt, and calls getReplyFromConfig. I did not repeat the reporter's live API logging or measure token volume.

## Solution Assessment

No, the proposed blanket string-formatting bypass is not the safest general fix because system events can require agent semantics. A separate explicit literal-delivery payload or mode is the more maintainable direction.
