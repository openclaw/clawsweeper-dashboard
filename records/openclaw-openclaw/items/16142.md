## Summary

Keep open. Current main and the latest release v2026.6.10 still route main-session cron reminder text through heartbeat's cron-event prompt and `getReplyFromConfig`, so the cost/throttling concern remains and needs a maintainer product decision before changing cron delivery semantics.

## What This Changes

Request to reduce avoidable OpenAI/ChatGPT usage by delivering simple recurring cron reminder text without using the model as a formatter.

## Best Possible Solution

Add an explicit literal-notification cron payload or delivery mode, or a conservative simple-reminder boundary, while preserving `systemEvent` as the model-backed heartbeat path for reasoning-dependent events.

## Reproduction Assessment

Yes. At source level, a main-session cron job with non-empty `systemEvent` text enters heartbeat, becomes a cron-event prompt, and calls `getReplyFromConfig`; I did not repeat the reporter's live OpenAI logging or measure exact token volume.

## Solution Assessment

No, not as a blanket string-formatting bypass. The better maintainable solution is an explicit literal-notification cron payload or delivery mode, leaving the existing model-backed `systemEvent` path for events that need agent semantics.
