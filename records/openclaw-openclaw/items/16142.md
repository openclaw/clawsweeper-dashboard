## Summary

Keep open. Current main still sends main-session cron systemEvent reminder text through the heartbeat cron-event prompt and model-backed reply path, and the overlapping remediation at https://github.com/openclaw/openclaw/pull/48883 is open, conflicting, and not yet a landed fix.

## What This Changes

The issue requests a lower-cost deterministic path for simple cron reminder delivery instead of repeatedly using the OpenAI model as a formatter.

## Best Possible Solution

Define and document a narrow low-cost path for simple main-session reminders while preserving the model-backed cron-event path for complex, internal, tool-using, response-tool, or reasoning-dependent events.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: main-session cron enqueues systemEvent text, heartbeat selects buildCronEventPrompt, and getReplyFromConfig sends the cron-event prompt through the model-backed reply path.

## Solution Assessment

Unclear. Deterministic formatting is reasonable for simple reminders, but maintainers still need to define which cron events may bypass the model and how that coordinates with the open remediation PR.
