## Summary

Keep open. Current main still routes main-session cron reminder text through the heartbeat cron-event prompt and model-backed reply path, while the requested deterministic low-cost delivery path needs a maintainer-defined product/API boundary rather than cleanup closure.

## What This Changes

Request to stop using ChatGPT/OpenAI as a formatter for recurring main-session cron reminder text by adding a cheaper deterministic formatting or delivery path.

## Best Possible Solution

Define an explicit low-cost path for simple main-session reminders, preserve the model-backed cron-event path for complex cases, and reconcile the boundary with #48883 in docs and focused tests.

## Reproduction Assessment

Yes. A high-confidence source reproduction path exists: enqueue a tagged main-session cron systemEvent, wake heartbeat with target last, then observe buildCronEventPrompt followed by getReplyFromConfig; current tests assert this path.

## Solution Assessment

Unclear. Deterministic formatting is a reasonable direction for simple reminders, but the maintainable fix needs a product/API boundary for which cron events may bypass the model and should account for #48883.
