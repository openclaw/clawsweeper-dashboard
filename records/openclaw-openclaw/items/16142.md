## Summary

Keep open. Current main still queues main-session cron reminder text as a system event and routes cron-tagged events through the heartbeat cron-event prompt plus model-backed reply path, so the reported cost/throttle problem remains valid and needs maintainer product/API judgment rather than cleanup closure.

## What This Changes

Request to reduce paid ChatGPT/OpenAI usage by formatting or delivering simple cron reminder text deterministically instead of using the model as a string formatter.

## Best Possible Solution

Define a narrow low-cost delivery mode for simple main-session cron reminders, preferably through an explicit cron option or conservative simple-payload detector, while preserving the model-backed cron-event path for complex or reasoning-dependent events.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: enqueue a tagged main-session cron system event, run heartbeat, and observe buildCronEventPrompt plus getReplyFromConfig; current tests assert this behavior.

## Solution Assessment

Unclear. Deterministic formatting is a good direction for simple reminders, but the broad requested replacement is not yet the safest fix until maintainers define which cron events may bypass model interpretation and reconcile #48883.
