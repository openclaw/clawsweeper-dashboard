## Summary

Keep open. Current main and the latest release still route main-session cron `systemEvent` reminder text through the heartbeat cron-event prompt and model-backed reply path, while the related static/no-LLM cron work and remediation PR are open rather than shipped fixes.

## What This Changes

The issue asks for simple cron reminder text to be delivered without repeatedly using OpenAI/ChatGPT as a formatter and consuming API tokens.

## Best Possible Solution

Define and land a maintainer-approved low-cost path for simple cron reminders, likely as an additive static/direct payload or explicit deterministic option, while preserving the model-backed path for complex, internal, tool-using, and reasoning-dependent cron events.

## Reproduction Assessment

Yes, at source level. Main-session cron enqueues `systemEvent` text with a `cron:` context key, heartbeat selects `buildCronEventPrompt`, and `getReplyFromConfig` sends that prompt through the model-backed reply path.

## Solution Assessment

Unclear. Deterministic formatting or direct delivery is plausible for simple reminders, but maintainers need to define which cron events may bypass the model and how that relates to the open static-message cron contract.
