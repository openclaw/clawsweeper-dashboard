## Summary

Keep open. Current main still routes main-session cron systemEvent/reminder text into the heartbeat cron-event prompt and model-backed reply path, so the reported OpenAI overuse remains source-reproducible; the overlapping remediation at https://github.com/openclaw/openclaw/pull/48883 is still open and needs maintainer/product review before this issue can close.

## What This Changes

The issue asks for a lower-cost cron reminder delivery path because simple Slack/email reminder text is repeatedly sent through the OpenAI-backed model reply path.

## Best Possible Solution

Define and land a maintainer-approved low-cost path for simple main-session cron reminders while preserving the existing model-backed path for complex, internal, tool-using, and response-tool cron events.

## Reproduction Assessment

Yes, at source level. Main-session cron enqueues systemEvent text with a cron context key, heartbeat selects buildCronEventPrompt, and getReplyFromConfig sends that cron-event prompt through the model-backed reply path.

## Solution Assessment

Unclear. Deterministic formatting is plausible for simple reminders, but the maintainable solution needs an explicit boundary for which cron events may bypass the model and must coordinate with the open remediation PR.
