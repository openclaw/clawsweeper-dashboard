## Summary

Keep open. Current main still routes main-session cron system-event text through the heartbeat cron-event prompt and model-backed reply path, and the open remediation PR has not settled the low-cost deterministic-delivery boundary.

## What This Changes

Request a low-cost deterministic formatting/delivery path for simple cron reminders instead of using ChatGPT/OpenAI as the formatter on every interval.

## Best Possible Solution

Define a narrow low-cost path for simple main-session reminders, preserve the model-backed cron-event path for complex/internal/no-relay cases, and align docs/tests with the chosen boundary and https://github.com/openclaw/openclaw/pull/48883.

## Reproduction Assessment

Yes. Source inspection on current main shows a main-session cron systemEvent is enqueued, selected as a cron event, wrapped by buildCronEventPrompt, and passed to getReplyFromConfig as Provider cron-event.

## Solution Assessment

Unclear. Deterministic formatting is reasonable for simple reminders, but a blanket model bypass is not the best maintainable fix until maintainers define which cron events may bypass the model and how that coordinates with https://github.com/openclaw/openclaw/pull/48883.
