## Summary

Keep open. Current main still routes main-session cron system-event text through the heartbeat cron-event prompt and model-backed reply path, while the overlapping remediation PR remains open and does not yet settle the low-cost deterministic delivery boundary.

## What This Changes

The issue requests reducing repeated OpenAI/ChatGPT calls used to relay simple cron reminder text.

## Best Possible Solution

Define a narrow low-cost path for simple main-session reminders, preserve the model-backed cron-event path for complex/internal/no-relay cases, and align docs/tests with the chosen boundary and https://github.com/openclaw/openclaw/pull/48883.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: main-session cron enqueues systemEvent text, heartbeat selects buildCronEventPrompt for cron events, and getReplyFromConfig handles the model-backed reply.

## Solution Assessment

Unclear. Deterministic formatting is reasonable for simple reminders, but a blanket model bypass is not the best maintainable fix until maintainers define which cron events may bypass the model and how that coordinates with https://github.com/openclaw/openclaw/pull/48883.
