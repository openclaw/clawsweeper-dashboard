## Summary

Keep open: current main still has the MS Teams `$search` bug and this PR is the canonical fix thread, but PR head still sends an unsupported sender `$filter` on the replacement list path and lacks real app-only Graph proof.

## What This Changes

The PR changes MS Teams message search to stop using Graph `$search`, page recent chat/channel messages with `$top=50`, filter message bodies locally after HTML stripping, update tests, and add a changelog note.

## Best Possible Solution

Land a corrected MS Teams search fix that uses only supported list parameters, applies query and sender matching locally over the bounded paginated window, updates the changelog wording, and includes redacted real app-only Graph output or logs.

## Reproduction Assessment

Yes, source-reproducible at the URL-construction seam. Current main builds `/messages?$search=...`, and PR head still builds a sender `$filter` on the list request when `from` is passed.

## Solution Assessment

No, not yet. Removing `$search`, paging with `$top=50`, and filtering bodies locally is the right direction, but sender filtering also needs to be local and the contributor still needs real after-fix Graph proof.
