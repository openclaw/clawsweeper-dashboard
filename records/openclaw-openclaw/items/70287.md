## Summary

Keep open: the PR addresses a real current-main MS Teams search bug, but the proposed replacement still sends an unsupported sender `$filter` on the Graph message-list endpoint and the external PR only provides mocked/unit proof rather than a real after-fix Graph run.

## What This Changes

The branch changes MS Teams message search to list a recent paginated Graph message window, filter body text locally after HTML stripping, update search regression tests, and add a changelog entry.

## Best Possible Solution

Land a corrected MS Teams search fix that uses only app-only-supported message-list parameters, applies text and sender matching locally over the bounded paginated window, and includes redacted real Graph output or logs showing the after-fix path.

## Reproduction Assessment

Yes, at the source and URL-construction seam. Current main still builds `/messages?$search=...`, and the PR-specific remaining defect is source-reproducible by passing `from`, which appends an unsupported sender `$filter` to the message-list request.

## Solution Assessment

No, not yet. Removing `$search`, paging with `$top=50`, and filtering message bodies locally is the right direction, but sender filtering also needs to be local and the contributor still needs real after-fix behavior proof.

## Review Findings

Overall correctness: patch is incorrect
