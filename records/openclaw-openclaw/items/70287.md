## Summary

Keep open. Current main still uses the unsupported Graph `$search` path for `msteams:search`, and this PR is the active implementation path, but the latest diff still leaves a sender-filter request shape that conflicts with Microsoft Graph’s documented chat/channel message-list contracts and needs a changelog credit before merge.

## What This Changes

The PR updates MS Teams message search to list recent chat/channel messages, page through `@odata.nextLink`, strip HTML for local text matching, add regression tests, and add a changelog entry.

## Best Possible Solution

Land a corrected version that uses only app-only-supported chat/channel message listing plus pagination, applies both text matching and sender matching locally over the bounded recent window, keeps focused regression tests for the URL contract and local filters, and credits the reporter/contributor in the changelog.

## Reproduction Assessment

Yes. The original bug is reproducible at the URL-construction seam on current main because the helper and tests still build `$search`; the remaining PR defect is reproducible by passing `from`, which still appends a sender `$filter` to the message-list endpoint.

## Solution Assessment

No, not yet. Removing `$search` and paging recent messages is the right direction, but keeping Graph sender `$filter` is not the narrowest maintainable app-only fix because it can preserve 400s for filtered searches; sender matching should be local after listing.
