## What This Changes

The PR replaces MS Teams Graph `$search` with paged recent message listing and local body filtering while still pushing sender filtering to Graph, updates search tests, and adds a changelog note.

## Best Possible Solution

Land a corrected MS Teams search fix that lists messages with only supported Graph parameters, applies body and sender matching locally over the bounded paginated window, and includes redacted app-only Graph output or logs.

## Reproduction Assessment

Yes. The failure is source-reproducible at the URL-construction seam: current main builds `/messages?$search=...`, and PR head still builds an unsupported sender `$filter` when `from` is passed.

## Solution Assessment

No, not yet. Removing `$search`, paging with `$top=50`, and filtering message bodies locally is the right direction, but sender filtering must also move local and the PR still needs real after-fix Graph proof.
