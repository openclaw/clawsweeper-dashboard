## Summary

Keep open: the PR targets a real MS Teams search bug on current main, but the branch still pushes an unsupported sender `$filter` to Graph and still lacks after-fix proof from a real app-only Microsoft Graph setup.

## What This Changes

The PR replaces MS Teams Graph `$search` message search with paged recent-message listing plus local body filtering, updates MS Teams search tests, and adds a changelog entry.

## Best Possible Solution

Land a corrected MS Teams search fix that lists messages with only supported Graph parameters, applies body and sender matching locally over the bounded paginated window, and includes redacted app-only Graph output or logs.

## Reproduction Assessment

Yes. The failure is source-reproducible: current main builds `/messages?$search=...`, and PR head still builds an unsupported sender `$filter` when `from` is passed.

## Solution Assessment

No, not yet. Removing `$search`, paging with `$top=50`, and filtering bodies locally is the right direction, but sender filtering must also move local and the PR still needs real after-fix Graph proof.
