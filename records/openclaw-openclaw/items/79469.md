## Summary

Keep open: current main still intentionally excludes reasoning-only WebChat payloads from completed transcript replay, and the remaining ask is a Control UI/WebChat reasoning retention and disclosure decision rather than conservative cleanup.

## What This Changes

The issue asks for a Control UI/WebChat affordance to keep completed reasoning visible or manually expandable after an assistant response finishes.

## Best Possible Solution

Define a WebChat reasoning visibility contract that decides which reasoning artifacts may be retained after completion, then add an accessible disclosure/toggle UI reusing the existing `reasoningLevel` and `showThinking` surfaces with focused gateway and UI tests.

## Reproduction Assessment

Yes, at source level. Current docs and tests assert WebChat suppression of `isReasoning` payloads, while the UI only renders persisted thinking when session/display state already allows it; I did not run a live browser session.

## Solution Assessment

Unclear. A Control UI disclosure and retention design is likely the right direction, but exposing completed reasoning needs maintainer UX and privacy policy before a narrow code fix is chosen.
