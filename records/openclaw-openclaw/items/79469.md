## Summary

Keep open: current main still lacks completed WebChat reasoning review, and the newer duplicate request was closed in favor of this issue. The related open PRs mostly address live reasoning stream delivery, not the retention and disclosure contract needed after completion.

## What This Changes

The issue requests completed WebChat reasoning text to remain reviewable through an always-visible or manually expandable Control UI/WebChat control.

## Best Possible Solution

Define one completed WebChat reasoning disclosure and retention contract that preserves existing `/reasoning` and Show Thinking gates, then update gateway replay, Control UI disclosure, docs, and focused tests together.

## Reproduction Assessment

Yes, at source level: current docs, gateway code, and tests show reasoning payloads are excluded from completed WebChat replay, while the UI only renders thinking already present on stored assistant messages. I did not run a live browser reproduction.

## Solution Assessment

Unclear until maintainers decide the privacy and UX contract. A UI-only always-show toggle would not recover reasoning payloads that are deliberately excluded from completed WebChat history.
