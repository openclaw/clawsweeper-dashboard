## Summary

Keep open: current main still does not provide durable completed WebChat reasoning review. The docs, gateway code, and tests intentionally suppress reasoning-flagged payloads from completed replay, while the Control UI can only render reasoning that is already stored on the message object; the later collapsible-think request was closed into this item as the canonical tracker.

## What This Changes

Add completed WebChat/Control UI reasoning review, either by keeping reasoning visible after response completion or by allowing manual expansion afterward.

## Best Possible Solution

Define one WebChat reasoning retention and disclosure contract, then update gateway replay, Control UI rendering, docs, and focused tests together while preserving the existing `/reasoning` and Show Thinking gates.

## Reproduction Assessment

Yes, at source level: current docs, gateway code, and tests deliberately exclude `isReasoning` WebChat payloads from completed replay, while the UI only renders reasoning already present in stored message content. I did not run a live browser reproduction.

## Solution Assessment

Unclear until maintainers define the UX and privacy contract. A blanket always-show replay would be riskier than a scoped design that explicitly decides what reasoning, if any, is retained after completion.
