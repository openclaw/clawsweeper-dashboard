## Summary

Keep open: current main still treats WebChat reasoning as gated or transient data, and the requested post-completion review/expansion behavior is not implemented. This needs a maintainer UX/privacy decision before a code fix should decide what reasoning is retained, replayed, or exposed.

## What This Changes

The issue requests persistent or expandable reasoning display in Control UI/WebChat after streaming completes.

## Best Possible Solution

Define a WebChat reasoning disclosure and retention contract, then update gateway replay, live event delivery, Control UI rendering, and tests around the existing `/reasoning` and Show Thinking gates.

## Reproduction Assessment

Yes, at source level: current docs, gateway code, and tests exclude `isReasoning` payloads from completed WebChat replay, while the UI can only render thinking already present in the message it receives. I did not run a live browser reproduction.

## Solution Assessment

Unclear until maintainers define the retention and disclosure policy. A blanket always-show toggle is riskier than a scoped design that reuses existing reasoning/session gates and explicitly decides what reasoning is durable after completion.
