## Summary

Keep open: current main and v2026.6.8 still render completed reasoning only as a static `.chat-thinking` block behind broad reasoning visibility gates, with no completed-block expand/collapse or always-show UX. The open live-render regression path at https://github.com/openclaw/openclaw/issues/91727 and https://github.com/openclaw/openclaw/pull/91810 may reduce one symptom, but it does not settle this issue's completed-reasoning disclosure decision.

## What This Changes

Request that WebChat keep completed reasoning reviewable after a response finishes, preferably with an always-show toggle or at least a manual expand/collapse control.

## Best Possible Solution

Keep this as the canonical completed WebChat reasoning disclosure tracker, finish the separate live-render regression path, and implement a focused per-block review pattern only after maintainers choose the intended default.

## Reproduction Assessment

Yes at source level: current WebChat extracts reasoning and renders it as a static `.chat-thinking` block only when broad reasoning visibility gates are on, with no completed-block expansion state. I did not run a live browser session in this read-only review.

## Solution Assessment

Unclear until maintainers choose the desired UX. The safest path is to handle the live-render regression separately, then add completed-reasoning disclosure using the existing reasoning gates unless a new persistent setting is explicitly approved.
