## Summary

Keep open: current main has general reasoning visibility controls, but I do not see a completed-reasoning disclosure/manual-review pattern that fully satisfies this UX request. The newer live-render regression at https://github.com/openclaw/openclaw/issues/91727 is related and more concrete, but it does not fully subsume the product choice about completed reasoning review/collapse behavior.

## What This Changes

The issue requests a WebChat/Control UI control or expandable completed state so users can review reasoning after a response finishes.

## Best Possible Solution

Keep one product decision for completed WebChat reasoning disclosure: reuse the existing reasoning extraction and visibility controls, fix the live-render regression in the canonical regression issue, and only add per-block completed expand/collapse if maintainers want that UX.

## Reproduction Assessment

Yes, at source level for the missing completed-disclosure behavior: current rendering emits a static `.chat-thinking` block only when reasoning is visible, with no per-block expand/collapse state. I did not run a live browser session in this read-only review.

## Solution Assessment

Unclear as an implementation direction until maintainers decide the intended UX. The safest path is to avoid a new config knob, repair the live-render regression under https://github.com/openclaw/openclaw/issues/91727, then add a focused completed-reasoning disclosure only if that remains desired.
