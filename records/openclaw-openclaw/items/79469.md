## Summary

Keep open: current main has partial reasoning-visibility plumbing, but WebChat still deliberately drops reasoning-only reply payloads from completed transcript replay, so this is a real UX feature request rather than conservative cleanup.

## What This Changes

The issue requests a Control UI/WebChat affordance to keep reasoning visible, or at least review it after a response completes.

## Best Possible Solution

Add a clearly scoped Control UI/WebChat reasoning visibility affordance that reuses the existing `reasoningLevel` session contract and defines exactly which completed reasoning artifacts may be retained or replayed.

## Reproduction Assessment

Yes, from source inspection: current WebChat docs and tests explicitly exclude `isReasoning` payloads from completed transcript replay, while the UI only renders persisted thinking blocks behind existing session/display state. I did not run a live browser scenario.

## Solution Assessment

Unclear: a chat-level control over the existing session `reasoningLevel` is likely the narrowest implementation, but retaining transient reasoning after completion needs maintainer UX/privacy policy first.
