## Summary

Keep open: current main still does not provide completed WebChat reasoning review. The later collapsible-`<think>` request was closed as a duplicate of this issue, and open reasoning-stream PRs address parts of live delivery but do not settle the completed reasoning retention/disclosure contract.

## What This Changes

Request a Control UI/WebChat control that keeps completed reasoning visible, or at least lets users manually expand it after the response finishes.

## Best Possible Solution

Define one WebChat reasoning disclosure and retention contract, preserve the existing `/reasoning` and Show Thinking gates, then update Gateway replay/live deltas, Control UI collapse/expand behavior, docs, and focused tests together.

## Reproduction Assessment

Yes, at source level: current main excludes reasoning payloads from completed WebChat replay, keeps live chat deltas text-only, and renders reasoning only when it is already present on the stored message. I did not run a live browser reproduction in this read-only review.

## Solution Assessment

Unclear until maintainers choose the UX and privacy contract. A blanket always-show replay is riskier than a scoped design that decides what reasoning is retained and how existing reasoning visibility gates apply after completion.
