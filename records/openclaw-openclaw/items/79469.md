## Summary

Keep open: current main still lacks a completed WebChat reasoning review/disclosure contract, and the newer collapsible-thinking request was closed as a duplicate of this issue rather than superseding it.

## What This Changes

The issue asks Control UI/WebChat to keep reasoning reviewable after a response completes, preferably with an always-show control or at least manual expansion.

## Best Possible Solution

Define one WebChat completed-reasoning retention and disclosure contract, then update gateway replay behavior, Control UI disclosure rendering, docs, and focused tests around that contract.

## Reproduction Assessment

Yes, at source level: docs, gateway code, and tests deliberately exclude isReasoning payloads from completed WebChat replay, while the UI can only render thinking already present on a stored assistant message. I did not run a live browser reproduction.

## Solution Assessment

Unclear until maintainers decide the UX and privacy contract. A narrow implementation should reuse the existing /reasoning and Show Thinking gates and explicitly define what reasoning is durable versus transient.
