## Summary

Keep open: current main still does not provide completed WebChat reasoning review. Reasoning-flagged delivery payloads are intentionally excluded from transcript replay, live chat deltas are text-only, and the Control UI only renders thinking when it is already present on the message object; the open related PRs address pieces of live streaming but do not establish the retention/disclosure contract requested here.

## What This Changes

Request that Control UI/WebChat keep reasoning visible or manually expandable after a response completes.

## Best Possible Solution

Define a WebChat reasoning disclosure and retention contract that preserves the existing `/reasoning` and Show Thinking gates, distinguishes transient delivery payloads from durable transcript content, and updates gateway plus Control UI tests together.

## Reproduction Assessment

Yes, at source level: current docs, gateway code, and tests deliberately exclude `isReasoning` WebChat payloads from completed replay, while the UI only renders thinking already present in stored message content. I did not run a live browser reproduction.

## Solution Assessment

Unclear until maintainers define the UX and privacy contract. A blanket always-show replay would be riskier than a scoped design that reuses existing reasoning/session gates and explicitly decides what reasoning, if any, is retained after completion.
