## Summary

Current main still shows this as the canonical open WebChat/Control UI reasoning-disclosure request: reasoning payload replay is intentionally suppressed, live chat deltas remain text-oriented, and open PRs only address pieces of the stream/renderer path rather than a landed retention or expansion contract.

## What This Changes

Issue requests a WebChat/Control UI option or completed-message affordance to keep reasoning visible or make it manually expandable after the final answer.

## Best Possible Solution

Define one WebChat reasoning disclosure and retention contract that preserves the existing /reasoning and Show Thinking gates, then update gateway replay, live deltas, Control UI disclosure, docs, and focused tests together.

## Reproduction Assessment

Yes, at source level: current docs, gateway code, and tests deliberately exclude isReasoning payloads from completed WebChat replay, and the Control UI only renders reasoning that is already present on the message object. I did not run a live browser reproduction.

## Solution Assessment

Unclear until maintainers decide the UX and privacy contract. A blanket always-show replay is riskier than a scoped design that reuses /reasoning and Show Thinking gates and explicitly decides what reasoning is retained after completion.
