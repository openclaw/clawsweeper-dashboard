## Summary

Keep open: current main still excludes delivery-only reasoning from WebChat transcript replay and only renders reasoning that already exists in persisted messages, so completed reasoning review is still unsolved. This is a UX/privacy feature decision rather than a narrow bug-fix cleanup item.

## What This Changes

Requests a Control UI/WebChat control or expansion affordance so reasoning remains reviewable after a response completes.

## Best Possible Solution

Define a WebChat reasoning disclosure and retention contract that reuses the existing `/reasoning` and Show Thinking gates, then update gateway replay, Control UI disclosure/toggle behavior, docs, and focused tests together.

## Reproduction Assessment

Yes, source-level: current main docs, gateway replay code, and tests deliberately exclude `isReasoning` WebChat payloads from completed replay, while the UI only renders thinking already present on stored messages. I did not run a live browser reproduction.

## Solution Assessment

Unclear until maintainers define the UX and privacy contract. A UI-only always-show toggle would be incomplete unless it also decides which reasoning is retained, replayed, and exposed after completion.
