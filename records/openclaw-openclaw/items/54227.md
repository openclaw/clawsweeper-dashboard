## Summary

Current main still consumes only the clicked Discord component entry, so a sibling button in the same non-reusable panel remains active. The prior ClawSweeper comment and the closed related #54233 are useful context, but they do not supersede the remaining Discord plugin bug.

## What This Changes

The issue requests that Discord `reusable: false` component panels expire as a whole after the first authorized component click instead of expiring only the clicked button entry.

## Best Possible Solution

Implement Discord plugin-level panel-wide consumption for non-reusable component messages, with regression coverage for sibling buttons and no change to `reusable: true`, unauthorized-click, expiry, or modal lifecycle behavior.

In the bundled Discord plugin, non-reusable component messages/panels should be consumed as a group after the first authorized click, so clicking a sibling button/select afterward returns the expired ephemeral response and does not dispatch another callback. Keep the fix inside the Discord plugin unless a tiny generic helper is absolutely necessary. Likely approach: add a shared consumption group/panel id to entries created from the same component message/panel, or otherwise consume sibling non-reusable entries that belong to the same registered message after authorization succeeds.
