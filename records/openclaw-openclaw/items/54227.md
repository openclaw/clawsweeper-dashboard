## Summary

Keep this open. Current main still stores and consumes Discord component interactions per component id, while the Discord docs describe non-reusable components as single-use by default; the related closed report mixed this same bug with another issue and does not supersede the remaining panel-wide consumption problem.

## What This Changes

The issue requests that a Discord `reusable: false` interactive component panel expire all sibling buttons/selects after the first authorized click instead of consuming only the clicked control.

## Best Possible Solution

Implement Discord plugin-owned panel-wide consumption for non-reusable component messages, likely with a shared group key assigned when building/registering entries from the same component message and a registry operation that invalidates sibling entries after the first authorized click. Preserve `reusable: true`, unauthorized-click behavior, modal trigger/submission lifecycle, and persistent registry behavior.

## Reproduction Assessment

Yes. A focused repro can register two non-reusable button entries with the same `messageId`, click one id, then click the other; current source deletes only the first id, so the second still resolves and dispatches.

## Solution Assessment

Yes, the requested direction is the narrowest maintainable fix. This should stay in the Discord plugin registry/handler path rather than adding core channel behavior or changing unrelated component APIs.
