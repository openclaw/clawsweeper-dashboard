## Summary

## What This Changes

The PR changes source-managed provider secret enforcement so plaintext source provider `apiKey` values are emitted as `secretref-managed`, and adds a runtime source snapshot regression test.

## Best Possible Solution

Land the redaction fix after real generated-config/auth proof shows `models.json` omits plaintext while provider auth still works, with the broader secrets roadmap remaining in https://github.com/openclaw/openclaw/issues/11829.

## Solution Assessment

Yes, the PR's fallback to the existing `secretref-managed` marker is the narrowest maintainable fix for this leak. Merge should wait on real command proof and explicit compatibility acceptance for generated-file auth behavior.
