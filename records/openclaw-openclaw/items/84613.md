## Label Justifications

- `P1`: The PR addresses a provider credential leak into generated model config, which can expose secrets to agent/prompt surfaces.
- `merge-risk: 🚨 compatibility`: Generated `models.json` will stop carrying plaintext provider `apiKey` values for this source path, which can break users treating that file as the credential source.

## Summary

Keep open: source inspection supports this as a focused credential-leak fix, but the external PR still needs real OpenClaw command proof and explicit maintainer acceptance of the generated `models.json` auth compatibility change before merge.

## What This Changes

The PR replaces plaintext source provider `apiKey` strings with the existing `secretref-managed` marker when generating `models.json`, and adds a runtime source snapshot regression test.

## Best Possible Solution

Land the redaction after redacted generated-config/auth proof shows plaintext is absent from `models.json` while provider auth still works, keeping the broader secrets roadmap in https://github.com/openclaw/openclaw/issues/11829.

## Reproduction Assessment

Yes, from source inspection: current main returns no source marker for a non-SecretRef plaintext source `apiKey`, so the runtime plaintext value can pass through the generated providers path. I did not run the regression because this review is read-only.

## Solution Assessment

Yes, the PR's use of the existing `secretref-managed` marker is the narrowest code fix for this leak. Merge should wait on real command proof and maintainer acceptance of the generated-file auth compatibility change.
