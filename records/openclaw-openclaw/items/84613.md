- `P1`: The PR addresses provider credential exposure in generated model config, a security-sensitive agent/auth workflow issue.
- `merge-risk: 🚨 compatibility`: Generated `models.json` can stop carrying plaintext provider credentials for this source path, which may break existing standalone generated-config usage.

## Summary

Keep open: current main still lacks this PR's narrow plaintext `apiKey` redaction, but the branch is not merge-ready because proof is test-only and the generated-auth compatibility decision is now tied to the broader, currently conflicting maintainer replacement path in https://github.com/openclaw/openclaw/pull/87166.

## What This Changes

The PR changes source-managed provider snapshot planning so plaintext provider `apiKey` values are replaced with the existing non-secret marker before generated `models.json` is written, and adds a regression test for that path.

## Best Possible Solution

Land the redaction through a current maintainer-approved path that proves real generated `models.json` output and provider auth after redaction, while keeping the broader secrets roadmap in https://github.com/openclaw/openclaw/issues/11829 and the replacement-path discussion in https://github.com/openclaw/openclaw/pull/87166 aligned.

## Reproduction Assessment

Yes, source-reproducible: current main returns no source marker for a non-SecretRef plaintext source `apiKey`, so the resolved runtime value can reach generated providers. I did not run the regression because this review is read-only.

## Solution Assessment

Not as-is: using a non-secret marker at the generated-config boundary is the right mitigation shape, but this branch still needs real command/auth proof and maintainer alignment with the broader replacement path in https://github.com/openclaw/openclaw/pull/87166.
