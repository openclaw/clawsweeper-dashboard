## Summary

Keep open: current main still has the leak-shaped plaintext source-provider path, but this PR is not merge-ready because it can turn a plaintext config credential into an unbacked marker and only supplies planner-test proof. ## What This Changes

This PR replaces plaintext source provider `apiKey` values with the existing `secretref-managed` marker during generated models config planning and adds a planner regression test.

## Best Possible Solution

Land a current maintainer-approved fix that removes plaintext from prompt-visible generated catalogs while preserving or migrating request auth for plaintext config, SecretRef, and env cases, with redacted generated-file and provider-auth proof.

Yes, source-reproducible: current main returns no marker for a plaintext non-SecretRef source `apiKey`, and the planner writes enforced providers into generated model catalog output. I did not run a mutating real setup because this review is read-only.

## Solution Assessment

No, not as this branch stands. Redacting generated output is the right security direction, but mapping plaintext config credentials to an unbacked SecretRef marker is not the narrowest maintainable fix without request-auth migration or proof.
