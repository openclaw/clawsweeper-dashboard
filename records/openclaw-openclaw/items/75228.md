## Summary

Keep open: this PR still targets a real macOS stale-pin recovery gap, but it has a blocking config-contract defect, missing after-fix real behavior proof, and a security-sensitive TLS policy decision.

## What This Changes

The PR adds a default-off macOS `gateway.remote.allowSystemTrustedPinRepair` option that lets stale gateway TLS pins be replaced on system-trusted remote WSS hosts, plus Swift tests and a changelog entry.

## Best Possible Solution

Register and document the opt-in through the canonical config contract, preserve strict defaults, and merge only after maintainer/security acceptance plus redacted real macOS proof.

## Reproduction Assessment

Yes for source-level reproduction: current `main` has a trusted public WSS pin-mismatch test that is not auto-repaired, and the TLS delegate cancels stored-pin mismatches. A live reproduction would need a macOS app install with a stale stored pin and a rotated public certificate.

## Solution Assessment

No. A default-off opt-in is a plausible direction, but this patch is incomplete until the config key is registered across the canonical contract and maintainers accept the TLS trust boundary.
