## Summary

Keep this PR open: it addresses a real macOS remote-gateway TLS pin rotation problem, but the latest patch is still not merge-ready because the new opt-in is read from raw config without being added to the strict shared config contract.

## What This Changes

The PR adds a default-off macOS companion config flag for system-trusted public `wss://` gateway TLS pin auto-repair, plus Swift tests and a changelog entry.

## Best Possible Solution

Keep the default loopback/`.ts.net` boundary, and make `gateway.remote.allowSystemTrustedPinRepair` a documented, schema-valid, default-false config key before considering merge.

## Reproduction Assessment

Yes. Current main source shows trusted arbitrary public hosts are not auto-repaired, and the PR-specific blocker is source-reproducible by comparing the new raw config read with the strict `gateway.remote` schema.

## Solution Assessment

No. The default-off opt-in is safer than broad default repair, but this implementation is incomplete until the new key is first-class in the shared config contract and docs.
