## Summary

Keep open: the PR fixes a real macOS stale TLS pin gap with a safer default-off opt-in, but the new config key is only read by the macOS helper and is not registered in the strict canonical config contract.

## What This Changes

This PR adds a macOS companion opt-in to auto-repair stale TLS pins for arbitrary system-trusted `wss://` remote gateways, with Swift tests and a changelog entry.

## Best Possible Solution

Keep the default loopback/`.ts.net` policy, and make the opt-in a first-class config key with schema, exported type, help/label metadata, generated baseline, docs, and validation coverage before merge.

## Reproduction Assessment

Yes. Current main's helper rejects `wss://gateway.example.com` even when `systemTrustOk` is true, and the PR-specific blocker is source-reproducible because the branch reads a key omitted from the strict `gateway.remote` schema.

## Solution Assessment

No. A default-off opt-in is safer than broad default repair, but this implementation is incomplete until the new key is registered and documented in the canonical config contract.
