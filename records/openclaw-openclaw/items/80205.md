## Summary

Keep open: current main already has install/update/uninstall peer-link repair, but it still lacks the issue's requested `plugins doctor` peer-link audit and startup repair/diagnostic path. The remaining startup behavior needs maintainer judgment because current docs say runtime loading does not repair plugin dependency trees.

## What This Changes

The issue asks OpenClaw to audit and repair npm-installed plugin `openclaw` peer links so Discord plugin contract imports do not degrade configured tokens to `configured_unavailable`.

## Best Possible Solution

Add a maintainer-approved peer-link repair model: at minimum audit/report broken managed npm peer links in doctor/plugin diagnostics, and only add bounded gateway startup repair if owners accept that dependency-policy change.

## Reproduction Assessment

Yes, at source level. A managed npm plugin declaring `peerDependencies.openclaw` with a missing plugin-local `node_modules/openclaw` link has no current `plugins doctor` audit and no gateway startup repair before runtime loading, although install/update/uninstall paths can relink it.

## Solution Assessment

Unclear. The doctor audit and focused regression tests look like the right direction, but automatic gateway-startup repair needs maintainer approval because current docs keep dependency repair out of runtime loading.
