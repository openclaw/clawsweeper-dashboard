## Summary

Keep open: current main repairs managed npm `openclaw` peer links in install/update/uninstall paths, but it still lacks the requested `openclaw plugins doctor` audit, gateway pre-load repair, and wrong-target link detection. The overlapping open doctor PR covers only part of the remaining issue and needs coordination.

## What This Changes

The issue asks OpenClaw to detect and repair npm-managed plugin `node_modules/openclaw` peer-link drift so Discord plugin secret contracts do not degrade into `configured_unavailable` token status.

## Best Possible Solution

Add one canonical peer-link audit/repair helper, wire diagnostics into plugin doctor and doctor repair, and make a maintainer decision before adding gateway startup-time mutation.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: create a managed npm plugin declaring `peerDependencies.openclaw` with a missing or wrong `node_modules/openclaw` link, then current plugin doctor has no direct audit and gateway startup loads runtime plugins without repairing it; I did not run a live reproduction because this review is read-only.

## Solution Assessment

Unclear. The doctor audit/repair helper is the right direction, but startup-time repair needs maintainer confirmation because it changes the documented startup dependency-repair boundary.
