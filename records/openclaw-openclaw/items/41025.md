## Summary

Keep open. The PR targets a real requested Control UI behavior, but current main intentionally blocks remote image fetches through renderer tests, CSP, and docs; this branch still needs a remote-media policy decision, coordinated CSP/sanitizer changes, and real browser proof before merge.

## What This Changes

The PR adds a remote http/https image branch to the Control UI markdown renderer, updates markdown tests to expect inline remote images with `referrerpolicy`, and adds a changelog entry claiming CSP support for `https:` image loads.

## Best Possible Solution

Make an explicit Control UI remote-media policy decision; if approved, land coordinated renderer, CSP, sanitizer allowlist, docs, tests, privacy mitigation, and real browser proof changes, otherwise keep the no-remote-fetch policy.

## Reproduction Assessment

Yes, at source level. Current main returns escaped alt text for non-data markdown image URLs, tests assert that behavior, and the served Control UI CSP rejects remote image fetches.

## Solution Assessment

No. A renderer/test/changelog-only patch is not the best way to solve this because it conflicts with current CSP/docs/privacy policy and omits the sanitizer and CSP changes needed for the claimed behavior.
