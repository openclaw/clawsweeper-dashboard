## Summary

Keep open, but the patch is not mergeable as-is: current main intentionally blocks remote image fetches in the renderer, CSP, docs, and tests, while this PR enables that behavior without the required product/security policy decision or served-CSP follow-through.

## What This Changes

The PR changes Control UI markdown rendering and tests so http/https markdown image URLs emit inline img elements instead of alt text.

## Best Possible Solution

Decide the Control UI remote-media policy first; if approved, implement it consistently across renderer, CSP, docs, changelog, and tests with an explicit HTTPS/proxy/allowlist/referrer strategy, otherwise keep the shipped no-remote-fetch behavior and track broader media UX in #64607.

## Reproduction Assessment

Yes. Source inspection of current main shows `toSanitizedMarkdownHtml("![Alt text](https://example.com/img.png)")` is expected to flatten to alt text, and the served Control UI CSP would block the remote image load even if the renderer emitted it.

## Solution Assessment

No. A renderer-only change that permits both http and https images conflicts with current CSP, docs, and tests, and it changes operator-browser privacy behavior without an approved policy.

## Review Findings

  - body: This path emits img elements for http/https URLs, but the Gateway-served Control UI still sends `img-src 'self' data: blob:` and tests assert that https is not allowed. In the real app, browsers will block these remote image loads unless the CSP and tests are intentionally updated too.

  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The diff has no dependency, workflow, lockfile, or publishing change, but it introduces browser remote-fetch behavior in chat and needs privacy/security review.

Concerns:

- **[medium] Arbitrary remote image fetches leak viewer metadata:** `ui/src/ui/markdown.ts:149`
  - body: The new http/https image path would let untrusted chat or model content make each viewer's browser request attacker-controlled image URLs, exposing network metadata and read timing to the image host.
