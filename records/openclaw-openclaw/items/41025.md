## Summary

Keep open. Current main and v2026.5.7 intentionally flatten remote Control UI markdown images and block remote image fetches, while this PR still misses the CSP, sanitizer, policy, and proof pieces needed to enable them safely.

## What This Changes

The PR changes Control UI markdown rendering/tests and the changelog so remote http/https markdown image URLs render inline with `referrerpolicy="no-referrer"`.

## Best Possible Solution

Make an explicit Control UI remote-media policy decision; if approved, implement renderer, CSP, sanitizer allowlist, docs, tests, privacy mitigations, and real browser proof together, otherwise keep the no-remote-fetch policy.

## Reproduction Assessment

Yes, at source level. Current main returns escaped alt text for non-data markdown image URLs, tests assert that behavior, and current plus shipped CSP block remote image fetches.

## Solution Assessment

No. A renderer/test/changelog-only change is not the best solution because it conflicts with the current Control UI remote-fetch policy and omits the CSP and sanitizer changes needed for the claimed behavior.
