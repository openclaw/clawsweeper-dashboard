## Summary

Keep open. The request is still not implemented on current main, but this PR remains blocked by missing real browser proof plus an unresolved Control UI remote-media policy/security decision.

## What This Changes

The PR changes Control UI markdown rendering and tests to render remote `http://` and `https://` Markdown images inline, and adds a changelog note claiming CSP support.

## Best Possible Solution

Make an explicit Control UI remote-media policy decision; if approved, port the change to the current markdown-it renderer and coordinate sanitizer, CSP, docs, tests, privacy mitigation, and browser proof, otherwise keep the documented no-remote-fetch behavior.

## Reproduction Assessment

Yes, at source level. Current main returns escaped alt text for non-data Markdown image URLs, tests assert that behavior, and the Control UI CSP rejects remote image loads.

## Solution Assessment

No. A renderer/test/changelog-only patch is not the best way to solve this because it conflicts with the current CSP, docs, sanitizer allowlist, and privacy policy boundary.
