## Summary

Keep open. The PR targets a real requested Control UI behavior, but current main intentionally blocks remote image fetches in the markdown renderer, CSP, tests, and docs, while this branch is conflicting and still needs product/security approval plus real browser proof.

## What This Changes

The PR changes Control UI markdown rendering, markdown tests, and changelog text so http/https markdown image URLs render inline with a referrer-policy attribute instead of falling back to alt text.

## Best Possible Solution

Decide the Control UI remote-media policy first; if approved, land a coordinated change that updates the renderer, sanitizer allowlist, CSP, docs, tests, changelog, privacy mitigations, and real browser proof together.

## Reproduction Assessment

Yes, at source level. Current main returns escaped alt text for non-data markdown image URLs, tests assert that behavior, and the served CSP would block remote loads even if an img tag were emitted.

## Solution Assessment

No. A renderer-only http/https allowance is not the best way to solve this because current main documents and enforces a no-remote-fetch policy across renderer, CSP, tests, and docs, and this PR misses sanitizer/CSP alignment.

## Security Review
