## Summary

Keep open. The PR addresses a real requested Control UI behavior, but current main intentionally blocks remote image fetches in the renderer, CSP, tests, and docs; this branch still needs maintainer product/security judgment, CSP/sanitizer alignment, and contributor real browser proof before it can merge.

## What This Changes

The PR changes Control UI markdown rendering/tests and the changelog so http/https markdown image URLs render inline with `referrerpolicy="no-referrer"`.

## Best Possible Solution

Make an explicit Control UI remote-media policy decision; if approved, implement renderer, CSP, sanitizer allowlist, docs, tests, privacy mitigations, and real browser proof together, otherwise keep the no-remote-fetch policy.

## Reproduction Assessment

Yes, at source level. Current main returns escaped alt text for non-data markdown image URLs, tests assert that behavior, and CSP/docs block remote image fetches.

## Solution Assessment

No. A renderer/test/changelog-only change is not the best solution because it conflicts with the current Control UI remote-fetch policy and omits the CSP and sanitizer changes needed for the claimed behavior.

## Security Review
