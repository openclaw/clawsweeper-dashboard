## Summary

Keep open for maintainer product/security judgment: the PR addresses real user demand, but it conflicts with current main's intentional no-remote-image-fetch policy and does not fully work with the served CSP.

## What This Changes

The PR updates Control UI markdown rendering and tests so http/https markdown image URLs emit inline img elements instead of alt text.

## Best Possible Solution

Decide the Control UI remote-media policy first; if approved, implement it across renderer, CSP, docs, changelog, and tests with an explicit HTTPS/proxy/allowlist/referrer strategy, otherwise keep the current no-remote-fetch behavior and track broader media UX in #64607.

## Reproduction Assessment

Yes. Source inspection shows current main's renderer returns alt text for non-data-image markdown sources, the tests assert no img for https URLs, and the served CSP would block remote image loads even if emitted.

## Solution Assessment

No. A renderer-only http/https change conflicts with the served CSP and current docs/tests, and it changes remote-fetch privacy behavior without an approved policy or mitigation.

## Review Findings

  - body: This path emits img elements for http/https URLs, but the Gateway-served Control UI still sends `img-src 'self' data: blob:` and its test asserts that https is not allowed. Current main's renderer, CSP, docs, and tests intentionally avoid that fetch class, so this needs approved policy and mitigation before enabling.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The diff has no workflow, dependency, lockfile, package-publishing, or supply-chain change, but it intentionally enables browser remote-fetch behavior in chat.

Concerns:

- **[medium] Arbitrary remote image fetches leak viewer metadata:** `ui/src/ui/markdown.ts:149`
  - body: The new http/https image path would let untrusted chat or model content make each viewer's browser request attacker-controlled image URLs, exposing network metadata and read timing to the image host. Current main's renderer, CSP, and docs prevent that fetch class.
  - confidence: 0.9
- **[low] HTTP image sources create mixed-content risk:** `ui/src/ui/markdown.ts:149`
  - body: The proposed remote-image regex allows plain http sources as well as https. On HTTPS Control UI deployments this can be blocked as mixed content or weaken the remote-media policy unless maintainers explicitly allow it.
  - confidence: 0.8
