## Summary

Keep open for maintainer decision, but the current patch should not merge as-is: it enables remote Control UI markdown image loads without aligning CSP, sanitizer attributes, docs, and the remote-media privacy policy, and the external PR still lacks after-fix real browser proof.

## What This Changes

The PR changes Control UI markdown rendering/tests and the changelog so http/https markdown image URLs render as inline images with `referrerpolicy` instead of falling back to alt text.

## Best Possible Solution

Make an explicit Control UI remote-media policy decision; if approved, implement renderer, CSP, sanitizer attributes, docs, tests, and privacy mitigations together with real browser proof.

## Reproduction Assessment

Yes, at source level. Current main returns escaped alt text for non-data markdown image URLs, tests assert that behavior, and CSP/docs block remote image fetches.

## Solution Assessment

No. A renderer-only allowance is not the best solution because current main documents and enforces a no-remote-fetch policy, and the proposed `referrerpolicy` mitigation is stripped by the sanitizer allowlist.
