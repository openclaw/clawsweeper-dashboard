## Summary

Keep this PR open, but it is not ready to merge: current main intentionally blocks remote chat image fetches, and this renderer-only change conflicts with the served CSP, docs, and privacy/security policy.

## What This Changes

The PR changes Control UI markdown rendering and tests so http/https markdown image URLs render as inline `<img>` elements instead of alt text.

## Best Possible Solution

Decide the Control UI remote-media policy first; if approved, implement it across the current markdown-it renderer, Gateway CSP, docs, changelog, and tests, otherwise keep the shipped no-remote-fetch behavior and track broader media UX in #64607.

## Reproduction Assessment

Yes, at source level. Current main's `toSanitizedMarkdownHtml("![Alt text](https://example.com/img.png)")` path returns alt text, and the served CSP would block remote image loads even if an img tag were emitted.

## Solution Assessment

No. A renderer-only http/https allowance is not the best solution because current CSP, docs, and tests intentionally prevent remote image fetches; the maintainable path needs an approved remote-media policy and matching surfaces.

## Security Review

Status: needs_attention

Summary: The diff has no workflow, dependency, lockfile, package-publishing, or supply-chain change, but it introduces browser remote-fetch behavior in chat and needs privacy/security review.

Concerns:

- **[medium] Arbitrary remote image fetches leak viewer metadata:** `ui/src/ui/markdown.ts:149`
  - body: The new http/https image path would let untrusted chat or model content make each viewer's browser request attacker-controlled image URLs, exposing network metadata and read timing to the image host. Current main's renderer, CSP, and docs prevent that fetch class.
  - confidence: 0.9
- **[low] HTTP image sources create mixed-content risk:** `ui/src/ui/markdown.ts:45`
  - body: The proposed remote-image regex allows plain http sources as well as https. On HTTPS Control UI deployments this can be blocked as mixed content or weaken the remote-media policy unless maintainers explicitly allow it.
  - confidence: 0.8
