## Summary

Keep open for maintainer review: the requested Control UI behavior is real, but current main intentionally blocks remote image fetches through renderer, CSP, tests, and docs, so this PR needs a product/security decision, a rebase to the active renderer, matching policy updates, and real browser proof.

## What This Changes

The branch changes Control UI markdown image rendering, tests, and changelog so http/https markdown images render as inline `<img>` tags with `referrerpolicy="no-referrer"`.

## Best Possible Solution

Decide the Control UI remote-media policy first; if approved, land a coordinated update to the active renderer, sanitizer attributes, CSP, docs, tests, and real browser proof.

## Reproduction Assessment

Yes, at source level: current main returns escaped alt text for non-data markdown image URLs, tests assert that behavior, and the served Control UI CSP rejects remote image loads. I did not run live browser proof in this read-only review.

## Solution Assessment

No. The PR is not the best solution as written because it changes an old renderer path and does not align the current sanitizer attributes, CSP, docs, tests, browser proof, or remote-fetch privacy policy.
