## Summary

Keep open. Current main intentionally blocks remote Control UI image fetches, and this PR still needs product/security approval, coordinated CSP/sanitizer changes, and real browser proof before it can merge.

## What This Changes

The PR renders remote http/https markdown images as Control UI `<img>` elements, updates markdown tests for that behavior, and adds a changelog note claiming CSP/referrer-policy support.

## Best Possible Solution

Make an explicit Control UI remote-media policy decision; if approved, land coordinated renderer, sanitizer allowlist, CSP, docs, tests, privacy mitigation, and browser proof, otherwise keep the no-remote-fetch policy.

## Reproduction Assessment

Yes, at source level. Current main flattens non-data markdown image URLs to alt text, tests assert that behavior, and the served CSP rejects remote image loads.

## Solution Assessment

No. A renderer/test/changelog-only patch is not the best solution because it conflicts with current CSP/docs/privacy policy and omits the sanitizer and CSP changes needed for the claimed behavior.

  - confidence: 0.9

## Security Review
