## Summary

Keep open: the PR targets a requested UI behavior, but current main intentionally blocks remote markdown image fetches through renderer behavior, CSP, docs, and tests, so this needs maintainer product/security approval plus real browser proof before merge.

## What This Changes

The PR changes Control UI markdown rendering and tests so http/https markdown image URLs emit inline `<img>` tags with `referrerpolicy`, and adds a changelog entry claiming CSP support.

## Best Possible Solution

Decide the Control UI remote-media policy first; if approved, update renderer, CSP, docs, changelog, tests, and privacy mitigations together, otherwise keep the shipped no-remote-fetch behavior and continue the discussion in https://github.com/openclaw/openclaw/issues/64607.

## Reproduction Assessment

Yes, at source level. Current main flattens non-data markdown image sources to alt text, tests assert that behavior, and the served CSP would block remote image loads even if the renderer emitted `<img>` tags.

## Solution Assessment

No. The proposed renderer-only allowance is outdated against current main and does not align CSP, docs, tests, or the remote-fetch privacy policy needed for a maintainable change.

## Security Review

Status: needs_attention

Summary: The patch enables untrusted remote image fetches from rendered chat content while current main intentionally blocks that fetch class.

Concerns:

- **[medium] Remote image fetches leak viewer metadata:** `ui/src/ui/markdown.ts:149`
  - body: Rendering arbitrary remote image URLs in chat/model content lets the image host observe viewer IP address, user agent, and read timing; `referrerpolicy="no-referrer"` only removes the Referer header.
  - confidence: 0.91

  - body: The proposed regex allows `http://` as well as `https://`; on secure Control UI deployments this can be blocked as mixed content or create a weaker remote-media policy than maintainers may intend.
  - confidence: 0.82

  - body: The changelog states that the PR allows `https:` in the Control UI CSP, but the PR does not modify the CSP helper, which could mislead users and reviewers about the security behavior.
