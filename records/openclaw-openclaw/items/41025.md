## Summary

Keep open. The PR targets a real Control UI limitation, but the proposed patch still conflicts with current CSP/docs policy, drops its own referrer mitigation through the sanitizer, lacks real browser proof, and changes a remote-media privacy policy that needs maintainer judgment.

## What This Changes

The PR changes Control UI markdown image rendering to emit inline `<img>` tags for remote `http://` and `https://` image URLs, updates markdown tests, and adds a changelog entry.

## Best Possible Solution

Make an explicit Control UI remote-media policy decision; if approved, land coordinated renderer, sanitizer allowlist, CSP, docs, tests, privacy mitigation, and browser proof changes, otherwise keep the current no-remote-fetch policy.

## Reproduction Assessment

Yes, at source level. Current main returns escaped alt text for non-data markdown image URLs, tests assert that behavior, and the served Control UI CSP rejects remote image loads.

## Solution Assessment

No. A renderer/test/changelog-only patch is not the best way to solve this because it conflicts with current CSP/docs/privacy policy and omits the sanitizer and CSP changes needed for the claimed behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Align CSP before enabling remote images:** `ui/src/ui/markdown.ts:149`
  - body: This branch emits remote `<img>` elements, but `src/gateway/control-ui-csp.ts` still serves `img-src 'self' data: blob:`, so browsers would block the remote image loads this PR is meant to enable.
  - confidence: 0.94
- **[P2] Allow referrerpolicy through the sanitizer:** `ui/src/ui/markdown.ts:149`
  - body: The remote-image branch adds `referrerpolicy="no-referrer"`, but `sanitizeOptions` uses an explicit allowed-attribute list that does not include `referrerpolicy`; DOMPurify will strip the attribute, so the mitigation and updated tests will not hold.
  - confidence: 0.9
- **[P2] Resolve remote image policy before default-on loads:** `ui/src/ui/markdown.ts:149`
  - body: Allowing arbitrary chat content to load remote images lets third parties observe viewer IP, user agent, and read timing; `referrerpolicy` only removes the Referer header, so this needs an approved remote-media policy before default-on rendering.
  - confidence: 0.9
- **[P3] Correct the CSP claim in the changelog:** `CHANGELOG.md:15`
  - body: The changelog says this PR allows `https:` in the Control UI CSP `img-src` directive, but the diff does not modify the CSP helper, so the release note overstates the actual behavior.
  - confidence: 0.92

## Security Review

Status: needs_attention

Summary: The diff enables untrusted remote browser image fetches without an approved remote-media policy, matching CSP, or surviving referrer mitigation.

Concerns:

- **[medium] Remote image fetches leak viewer metadata:** `ui/src/ui/markdown.ts:149`
  - body: Rendered chat/model content could include tracking pixels that make the browser fetch a third-party URL, exposing viewer IP address, user agent, and read timing; `referrerpolicy` does not prevent the request itself.
