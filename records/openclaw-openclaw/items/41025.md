## Summary

Keep open, but this PR is not ready to merge: it proposes a user-visible remote-image capability that current main still blocks in the renderer, CSP, tests, and docs, and the branch still needs product/security approval plus real browser proof. The related request at https://github.com/openclaw/openclaw/issues/40945 remains open, and the user comment asking not to close this PR is consistent with keeping the discussion centralized here.

## What This Changes

The PR changes Control UI markdown rendering/tests and the changelog so http/https markdown image URLs render as inline images with `referrerpolicy` instead of falling back to alt text.

## Best Possible Solution

Make an explicit Control UI remote-media policy decision; if approved, port the change to the current markdown-it renderer and align CSP, sanitizer attributes, tests, docs, changelog, privacy mitigations, and real browser proof.

## Reproduction Assessment

Yes, at source level. Current main returns escaped alt text for non-data markdown image URLs, tests assert that behavior, and the served CSP blocks remote http/https image loads.

## Solution Assessment

No. A renderer-only http/https allowance is not the best solution because current main documents and enforces a no-remote-fetch policy through renderer behavior, CSP, tests, and docs.

## Security Review

Status: needs_attention

Summary: The PR enables untrusted remote image fetches from rendered chat content without an approved remote-media policy or matching CSP change.

Concerns:

- **[medium] Remote image fetches leak viewer metadata:** `ui/src/ui/markdown.ts:150`
  - body: Rendered chat/model content could include tracking pixels that make the browser fetch a third-party URL, exposing viewer IP address, user agent, and read timing; `referrerpolicy` does not prevent the request itself.
  - confidence: 0.91

  - body: The proposed regex allows `http://` as well as `https://`; on HTTPS Control UI deployments this can be mixed-content blocked and is weaker than a secure remote-media allowlist or proxy policy.
