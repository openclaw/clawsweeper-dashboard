## Summary

Keep open, but this PR is not merge-ready: current main, docs, and tests intentionally prevent remote image fetches, while the PR only changes the renderer and needs product/security approval before that policy changes.

## What This Changes

The PR updates Control UI markdown rendering and tests so http/https markdown image URLs emit inline img elements instead of being flattened to alt text.

## Best Possible Solution

Decide the Control UI remote-media policy first; if approved, implement remote images end-to-end with an explicit https/proxy/allowlist/referrer strategy across renderer, CSP, docs, changelog, and tests, otherwise keep the shipped no-remote-fetch behavior and track broader media UX in #64607.

## Reproduction Assessment

Yes, at source level. Current main has a direct path where toSanitizedMarkdownHtml("![Alt text](https://example.com/img.png)") returns alt text, and the served CSP would block remote image loads even if an img tag were emitted.

## Solution Assessment

No. A renderer-only http/https allowance conflicts with current CSP, docs, and tests and changes operator-browser privacy behavior without an approved policy or mitigation.

## Security Review

Status: needs_attention

Summary: The diff has no workflow, dependency, lockfile, package-publishing, or supply-chain change, but it introduces browser remote-fetch behavior in chat and needs privacy/security review.

Concerns:

- **[medium] Arbitrary remote image fetches leak viewer metadata:** `ui/src/ui/markdown.ts:149`
  - body: The new http/https image path would let untrusted chat or model content make each viewer's browser request attacker-controlled image URLs, exposing network metadata and read timing to the image host. Current main's renderer, CSP, and docs prevent that fetch class.
  - confidence: 0.9

  - body: The proposed remote-image regex allows plain http sources as well as https. On HTTPS Control UI deployments this can be blocked as mixed content or weaken the remote-media policy unless maintainers explicitly allow it.
  - confidence: 0.8
