## Summary

Keep this PR open, but it should not merge yet: the dependency patch has a state-retention regression, the generic port helper gets a Manifest-specific hint, the required changelog entry is missing, and the external contributor has not provided real behavior proof.

## What This Changes

This PR adds a pnpm patch for `@mariozechner/pi-coding-agent` to prune in-memory `SessionManager` entries after compaction, adds a regression test, and updates self-owned port-conflict diagnostics/tests.

## Best Possible Solution

Land a maintainer-approved runtime fix that prunes stale transcript entries while preserving latest session state and extension-owned metadata, keep generic port diagnostics extension-agnostic, and require real long-session/gateway proof before merge.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source-level yes for the memory-retention class: upstream `SessionManager` keeps all loaded entries in `fileEntries`/`byId`, but I did not establish a live current-main gateway memory reproduction.

## Solution Assessment

Is this the best way to solve the issue? No; patching the dependency may be acceptable with maintainer approval, but this implementation drops current session state before the compaction boundary and puts extension-specific guidance into a shared core helper.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve state entries when pruning the active branch:** `patches/@mariozechner__pi-coding-agent@0.73.0.patch:51`
  - body: Starting the retained set at `firstKeptEntryId` drops state entries that occurred immediately before the kept window, including the latest `model_change`, `thinking_level_change`, and `session_info`. Current compaction-rotation code explicitly preserves those entries, so long sessions can lose the selected model/thinking level or session title as soon as this memory cap trips.
  - confidence: 0.84
- **[P2] Keep self-owned port hints generic:** `src/infra/ports.ts:65-70`
  - body: `handlePortError()` is shared infra, but any current-process listener will now suppress normal port details and tell users to use a Manifest/clawrouter-specific environment variable. For non-Manifest self-collisions this gives the wrong repair path; keep the self-PID diagnostic generic or let the plugin provide a plugin-specific hint.
  - confidence: 0.78
- **[P3] Add the required changelog entry:** `package.json:1846`
  - body: This changes user-visible runtime memory behavior and port diagnostics, so OpenClaw policy requires a `CHANGELOG.md` entry under the active version. Please add a concise user-facing fix entry before merge.
  - confidence: 0.9
