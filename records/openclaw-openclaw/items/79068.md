## Summary

Keep open: the updated wrapper still replaces Pi `SessionManager.fileEntries`, and Pi rewrite paths serialize that same array, so an existing rewrite after pruning can truncate the persisted JSONL transcript that the PR promises to preserve.

## What This Changes

Adds an OpenClaw wrapper around Pi `SessionManager` to prune stale post-compaction in-memory entries, routes CLI and embedded session-manager paths through it, adds regression and port-diagnostic tests, and updates the changelog.

## Best Possible Solution

Keep the bounded live-memory behavior, but decouple live-context pruning from the full persistence backing data or wrap rewrite paths so every rewrite preserves the complete append-only transcript, with a regression test that prunes and then rewrites.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: prune the wrapped manager, then hit an existing CLI or embedded cleanup/flush path that calls `_rewriteFile()` on the pruned `fileEntries` array.

## Solution Assessment

No. The wrapper direction is narrow and plausible, but it is not the best mergeable solution until persistence rewrites are protected from the pruned live-memory view.
