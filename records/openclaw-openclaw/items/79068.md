## Summary

Keep open: the SessionManager memory fix is plausible and has supplied real terminal proof, but the PR currently introduces a registry regression by dropping plugin-scope wrapping for deprecated runtime config helpers. The dependency patch is also supply-chain-sensitive and needs explicit maintainer approval before merge.

## What This Changes

The branch patches `@earendil-works/pi-coding-agent@0.74.0` to prune stale in-memory SessionManager entries after compaction, adds regression coverage, and adjusts related gateway/plugin tests and dependency patch metadata.

## Best Possible Solution

Restore plugin-scope wrapping for both current and deprecated config helpers, then land only after maintainers approve this exact Pi patch or replace it with an upstream package release.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create or open a persisted Pi SessionManager, append many entries through compaction, and observe current 0.74.0 retaining all loaded/appended entries in `fileEntries` and `byId`.

## Solution Assessment

No, not as submitted. The SessionManager boundary is a narrow place to bound live memory, but the PR should preserve existing runtime config helper attribution and still receive explicit dependency-patch approval.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve deprecated config helper scoping:** `src/plugins/registry.ts:2402-2405`
  - body: This replacement scopes only `current`/`mutateConfigFile`/`replaceConfigFile` and leaves `loadConfig` and `writeConfigFile` coming straight from the spread `config` object. Those deprecated helpers are still part of `PluginRuntime`, and their warnings rely on `getPluginRuntimeGatewayRequestScope()` to include the calling plugin id/source and de-dupe per plugin. External/global plugins that still call them would now emit anonymous deprecation warnings again, regressing the current-main attribution behavior. Keep the old wrappers while adding wrappers for the new helpers, and keep coverage for both paths.
  - confidence: 0.87
