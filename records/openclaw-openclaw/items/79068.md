## Summary

Keep open: this is useful active bug-fix work, but the current PR head still risks truncating persisted JSONL history because it bounds Pi's `fileEntries` array, which is also Pi's rewrite backing store, while OpenClaw still calls `_rewriteFile()` on live session managers.

## What This Changes

The PR adds a bounded Pi `SessionManager` wrapper for gateway and CLI runtime paths, updates related tests and diagnostics, and includes changelog plus macOS formatting changes.

## Best Possible Solution

Keep the bounded-memory direction only after separating live retained state from full persisted transcript history, or making every rewrite path operate from complete JSONL state, then require current-head regression and real behavior proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: prune a persisted manager, invoke a later `_rewriteFile()` path, then reread the JSONL transcript and stale pre-compaction entries can be lost.

## Solution Assessment

No. The wrapper idea is plausible, but using Pi's `fileEntries` persistence array as the bounded live-memory cache is not rewrite-safe; persistence and live-memory retention need separate state or a complete-history rewrite path.
