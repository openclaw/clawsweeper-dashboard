## Summary

Keep this PR open, but it still needs changes before merge: the bounded SessionManager wrapper solves the live-memory direction, while the current diff can still truncate persisted JSONL history when an existing rewrite path runs after pruning.

## What This Changes

The PR adds an OpenClaw wrapper around Pi SessionManager to prune stale pre-compaction live memory, routes gateway/CLI session opens through it, adjusts port diagnostics, adds tests, and updates the changelog.

## Best Possible Solution

Keep the bounded live-memory approach, but separate live-context pruning from the full persistence rewrite state or make every rewrite path preserve the complete append-only JSONL history.

## Reproduction Assessment

Yes. Source inspection shows Pi 0.74.0 keeps SessionManager fileEntries/byId in memory, and the PR regression is source-reproducible because current OpenClaw rewrite paths can serialize the wrapper's pruned fileEntries array.

## Solution Assessment

No. The wrapper is a plausible narrow fix direction, but it must protect _rewriteFile and cleanup/rewrite paths before it is the best maintainable solution.
