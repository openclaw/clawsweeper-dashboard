## Summary

Keep this PR open, but it is not mergeable yet: the memory-bound wrapper still prunes Pi's persistence backing array, so later rewrite hooks can truncate the JSONL transcript the PR promises to preserve.

## What This Changes

The PR adds an OpenClaw wrapper around Pi SessionManager to bound live in-memory transcript entries after compaction, reroutes embedded/CLI SessionManager openings through it, and updates related tests, diagnostics, macOS formatting, and changelog.

## Best Possible Solution

Separate bounded live-memory state from the full persisted transcript, or otherwise make every rewrite path operate on the complete JSONL history, then add a regression that prunes and rewrites before reopening the transcript.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the PR replaces `fileEntries` with a pruned subset, Pi `_rewriteFile()` serializes `fileEntries`, and current OpenClaw paths still invoke `_rewriteFile()` after session mutations.

## Solution Assessment

No. The wrapper approach is plausible, but this implementation uses the persistence backing array as the bounded cache; the maintainable fix needs a separate live-memory view or rewrite-safe persistence handling.
