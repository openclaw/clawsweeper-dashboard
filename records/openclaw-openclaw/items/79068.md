## Summary

Keep open: the memory-bounding direction is useful, but the patch is not merge-ready because the wrapper prunes Pi's `fileEntries` array even though upstream rewrites persisted JSONL from that same array, so later rewrite paths can truncate the transcript the PR promises to preserve. The PR body's real proof is also stale for the current head/Pi package and does not cover prune -> rewrite -> reopen preservation.

## What This Changes

The PR adds a bounded wrapper around Pi `SessionManager` for gateway and CLI compaction paths, plus focused tests, port/origin diagnostics changes, source-delivery target matching changes, macOS channel config rendering changes, and a changelog entry.

## Best Possible Solution

Make the bounded SessionManager design rewrite-safe by separating retained live context from full append-only persistence, then land only the memory fix and any explicitly accepted companion changes with current-head proof.

## Reproduction Assessment

Yes at source level: upstream `_rewriteFile()` serializes `fileEntries`, the PR replaces `fileEntries` with a pruned subset, and OpenClaw still calls `_rewriteFile()` from patched runtime paths. I did not run a live gateway soak in this read-only review.

## Solution Assessment

No: bounding live memory is the right goal, but pruning the persistence backing array is not the safe boundary. The safer solution keeps full JSONL persistence separate from the retained context view or moves this into an upstream SessionManager contract.
