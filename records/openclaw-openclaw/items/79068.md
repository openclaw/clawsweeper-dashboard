## Summary

Keep open: the PR is a focused fix for a real SessionManager retention path, but the current patch still has a blocking persistence regression because the pruned live-memory array is also the array Pi rewrites to disk.

## What This Changes

The PR adds a bounded Pi SessionManager wrapper for gateway/CLI session managers, regression coverage, clearer same-process port diagnostics, and a changelog entry.

## Best Possible Solution

Keep the bounded live-memory behavior, but separate live-context pruning from persisted transcript rewrite state or make every rewrite path preserve the full append-only JSONL history.

## Reproduction Assessment

Yes. Source inspection against Pi 0.74.0 shows the original unbounded `fileEntries` retention and the PR regression path where a prune followed by `_rewriteFile()` can persist only the shortened in-memory view.

## Solution Assessment

No. The wrapper is a plausible narrow direction, but this implementation conflates the live-memory branch with the persistence rewrite source and needs a rewrite-after-prune guard before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve disk history when pruned managers rewrite:** `src/agents/pi-session-manager-memory-bound.ts:197`

Existing cleanup and flush paths still call `_rewriteFile()` after appending or removing entries, so any rewrite after pruning can truncate the historical JSONL transcript despite the PR's append-only guarantee.
