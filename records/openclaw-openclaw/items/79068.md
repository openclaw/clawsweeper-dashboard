## Summary

Keep this PR open: the current head still has a P1 transcript data-loss blocker because the bounded live-memory subset is assigned back to Pi's persistence backing array, while runtime rewrite paths can later serialize that subset over the JSONL file.

## What This Changes

The PR adds an OpenClaw wrapper around Pi SessionManager, routes embedded/CLI/compaction paths through it, adds a regression test plus port/origin diagnostic adjustments, and updates changelog/macOS formatting.

## Best Possible Solution

Keep the bounded-memory direction only after live retained state is separated from full persisted transcript history, or after every rewrite path operates from complete JSONL state, with current-head regression and real behavior proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: prune a persisted SessionManager, invoke a later _rewriteFile path, then reopen/read the JSONL transcript and stale pre-compaction entries can be lost.

## Solution Assessment

No. The wrapper goal is plausible, but using Pi's fileEntries persistence array as the bounded live-memory cache is not rewrite-safe; persistence and memory retention need separate state or a complete-history rewrite path.
