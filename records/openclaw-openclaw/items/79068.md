## Summary

Keep open: the PR has a useful target and now includes sufficient real behavior proof for the narrowed SessionManager path, but the diff still has correctness blockers, a missing changelog entry, and a runtime dependency patch that needs maintainer approval.

## What This Changes

Adds a pnpm patch to `@mariozechner/pi-coding-agent` to prune live `SessionManager` entries after compaction, adds regression coverage, and changes self-owned port diagnostics.

## Best Possible Solution

Land a maintainer-approved runtime fix that prunes stale in-memory transcript entries after compaction while preserving latest state/custom metadata, keeps persisted JSONL append-only, and leaves shared port diagnostics generic or plugin-owned.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source-level yes for the narrowed `SessionManager` retention behavior: upstream 0.73.0 keeps all loaded entries in memory for the manager lifetime, and the PR body exercises the patched path with real persisted JSONL; I do not have a high-confidence current-main gateway soak reproduction for the broader Paperclip memory-growth claim.

## Solution Assessment

Is this the best way to solve the issue? No; pruning after compaction is the right shape, but this implementation must preserve latest state/custom entries, keep shared port diagnostics generic, and get maintainer approval for the runtime dependency patch.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve state entries when pruning live memory:** `patches/@mariozechner__pi-coding-agent@0.73.0.patch:52`
  - body: Starting the retained set at `firstKeptEntryId` drops pre-boundary state entries that still define the current session, including the latest model, thinking level, session title, and extension custom entries. Current successor transcript rotation keeps those entries while dropping stale transcript messages, so this cap can silently lose session state as soon as pruning runs.
  - confidence: 0.86

  - body: `handlePortError()` is shared infra, but any current-process listener will now suppress normal port details and tell users to use Manifest/clawrouter-specific advice from https://github.com/openclaw/openclaw/issues/73655. For non-Manifest self-collisions this points users at the wrong repair path; keep the shared diagnostic generic or let the owning plugin add a plugin-specific hint.
  - confidence: 0.8
- **[P3] Add the required changelog entry:** `package.json:1845`
  - body: This PR changes user-visible runtime memory behavior and port diagnostics, so OpenClaw policy requires a concise `CHANGELOG.md` entry under `Unreleased` before merge.
  - confidence: 0.9
