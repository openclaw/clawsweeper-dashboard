## Summary

Keep open. The SessionManager memory-bound fix is not on current main and has sufficient focused proof, but the latest head still regresses deprecated plugin config helper scoping and the runtime dependency patch needs explicit maintainer approval.

## What This Changes

This PR patches `@earendil-works/pi-coding-agent@0.74.0` to prune stale pre-compaction SessionManager entries from live memory, adds regression coverage, updates port diagnostics, and adjusts related tests, snapshots, changelog, and dependency metadata.

## Best Possible Solution

Keep the SessionManager memory bound only if maintainers approve this local dependency patch or an equivalent upstream Pi release, while preserving existing plugin runtime config attribution for every helper method.

## Reproduction Assessment

Yes. Source inspection of published `@earendil-works/pi-coding-agent@0.74.0` shows live `fileEntries` and `byId` grow without a pruning path, and the PR body supplies after-fix terminal proof against a persisted JSONL SessionManager; I did not run the write-heavy repro in this read-only review.

## Solution Assessment

No, not as submitted. The SessionManager pruning boundary is plausible, but the PR should preserve deprecated config helper scoping and still needs maintainer approval for the dependency patch.
