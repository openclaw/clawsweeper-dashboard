## Summary

Keep this PR open. It targets a real gateway/session memory-retention bug and includes useful terminal proof, but the current patch can still turn live-memory pruning into a destructive persisted-transcript rewrite.

## What This Changes

The branch adds a bounded Pi SessionManager wrapper for CLI, embedded runner, and CLI compaction paths, plus focused session-memory and port-diagnostic tests.

## Best Possible Solution

Keep the bounded live-memory behavior, but separate the pruned live context/index view from the full persistence backing transcript or wrap rewrite paths so JSONL rewrites always preserve complete history.

## Reproduction Assessment

Yes for the merge blocker by source inspection. Open a wrapped persisted SessionManager, prune after compaction, then trigger an existing CLI or embedded `_rewriteFile()` path; Pi will serialize the pruned `fileEntries` array.

## Solution Assessment

No. The wrapper direction is narrow and plausible, but mutating `fileEntries` violates Pi's rewrite contract until persistence rewrites are protected and covered by a regression test.
