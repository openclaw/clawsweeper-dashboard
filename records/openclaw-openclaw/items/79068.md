## Summary

Keep open. The refreshed proof is now adequate for the wrapper smoke path, but the patch still risks losing the persisted transcript history it promises to preserve because later `_rewriteFile()` calls can serialize the pruned in-memory `fileEntries` view.

## What This Changes

The PR adds a bounded Pi SessionManager wrapper, routes embedded/CLI session opens through it, adds regression coverage, updates port diagnostics, and adds a changelog entry.

## Best Possible Solution

Keep the runtime memory bound, but make the disk persistence contract explicit so every rewrite path preserves the full JSONL transcript or avoids rewriting from a pruned live view.

## Reproduction Assessment

Yes. The original retention problem is source-reproducible from Pi 0.74.0 keeping every appended entry in `fileEntries` and `byId`, and the PR regression is source-reproducible because Pi `_rewriteFile()` serializes the shortened array after pruning.

## Solution Assessment

No. The wrapper is a plausible narrow fix, but it is not the best final form until it preserves the append-only persisted transcript across private rewrite paths.
