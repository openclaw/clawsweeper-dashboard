## Summary

Keep this PR open because the latest wrapper still has a concrete persistence-contract blocker: pruning mutates the same `fileEntries` array that Pi rewrites to disk.

## What This Changes

The PR adds a bounded OpenClaw wrapper around Pi `SessionManager`, routes gateway/CLI session opens through it, adds memory/port tests, updates port diagnostics, and adds a changelog entry.

## Best Possible Solution

Keep the bounded live-memory approach, but make rewrite/cleanup paths preserve the full append-only JSONL history and cover rewrite-after-prune with a regression test.

## Reproduction Assessment

Yes. Source inspection shows current main uses Pi 0.74.0, whose SessionManager retains entries in memory, and the PR regression is source-reproducible because current cleanup paths can call `_rewriteFile()` after the wrapper prunes `fileEntries`.

## Solution Assessment

No. The wrapper is a narrow plausible direction, but it is not safe until persisted rewrite state is separated from the bounded live view or every rewrite path is made history-preserving.
