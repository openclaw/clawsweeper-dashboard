## Summary

Keep open: the current wrapper still stores the pruned live view back into `fileEntries`, so existing rewrite cleanup paths can rewrite the JSONL without the historical entries the PR promises to preserve. The updated real-behavior proof now matches the wrapper branch and the security pass is clear, but this correctness blocker remains.

## What This Changes

The PR adds a bounded Pi SessionManager wrapper, routes active gateway and CLI SessionManager opens through it, adds memory and port tests, updates port diagnostics, and adds a changelog entry.

## Best Possible Solution

Keep the bounded live-memory behavior, but separate live-context pruning from persisted transcript rewrite state or make every rewrite path preserve the full append-only JSONL history.

## Reproduction Assessment

Yes. Source inspection reproduces the original retention path in Pi 0.74.0 and the PR regression path where `_rewriteFile()` serializes the pruned `fileEntries` array after cleanup or flush code runs.

## Solution Assessment

No. The wrapper is a plausible narrow direction, but it must protect `_rewriteFile()` and cleanup/rewrite paths before merge.
