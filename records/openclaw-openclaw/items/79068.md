## Summary

Keep this PR open, but do not merge the current head: the branch addresses a real gateway/session memory problem, yet the current wrapper can still turn live-memory pruning into persisted transcript truncation when OpenClaw later calls Pi's rewrite hook. The earlier lockfile drift appears resolved, but the PR still needs a code fix and current-head proof that covers a post-prune rewrite path.

## What This Changes

The PR adds a bounded Pi SessionManager wrapper, reroutes embedded and CLI SessionManager usage through it, adds regression coverage, adjusts port/origin diagnostics and macOS formatting, and updates the changelog.

## Best Possible Solution

Land a focused memory-bound SessionManager change that bounds live context without mutating the persisted rewrite source, adds regression coverage for post-prune `_rewriteFile()`, and updates current-head real behavior proof.

## Reproduction Assessment

Yes. Source inspection shows current main opens Pi SessionManager in gateway/CLI paths, upstream retains all entries in `fileEntries`, and the PR-specific data-loss bug follows from `_rewriteFile()` serializing a pruned `fileEntries` array.

## Solution Assessment

No. The wrapper approach is plausible, but this implementation must decouple live-memory pruning from the persistence-backed rewrite array before it is the maintainable fix.
