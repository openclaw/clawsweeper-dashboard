## Summary

Keep open: the memory-bound direction targets a real gateway/session problem, but this PR is not merge-ready because it can rewrite pruned session entries back to disk, references a missing pnpm patch file, and replaces plugin-backed delivery normalization with a partial local copy.

## What This Changes

The PR adds a bounded Pi SessionManager wrapper for gateway/CLI session paths, plus tests and additional changes to source-delivery matching, port diagnostics, browser-origin handling, macOS channel config rendering, dependency patch metadata, and process-test behavior.

## Best Possible Solution

Land a narrower branch that keeps full durable JSONL separate from bounded live context, preserves plugin target normalization or splits that change, fixes dependency metadata, and updates current-head proof.

## Reproduction Assessment

Yes at source level: current main opens Pi SessionManager directly in active gateway/CLI paths, and Pi 0.75.4 keeps live `fileEntries` while `_rewriteFile()` serializes that same array. I did not run a long gateway soak or PR tests in this read-only review.

## Solution Assessment

No: bounding live memory is the right problem, but mutating Pi's durable `fileEntries` array is not a safe implementation while OpenClaw still calls `_rewriteFile()`. A safe fix should separate live retained context from persisted transcript state or use an upstream API designed for this.
