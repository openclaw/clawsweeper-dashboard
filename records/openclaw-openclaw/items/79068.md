## Summary

Keep open: the branch is still a plausible fix vehicle, but it is not merge-ready because the memory wrapper mutates the same Pi `fileEntries` array that rewrite paths persist, source-delivery matching bypasses plugin target normalizers, and the supplied proof is stale for the current head/package context.

## What This Changes

The branch adds an OpenClaw runtime wrapper around Pi `SessionManager`, routes gateway and CLI session managers through it, adjusts source-delivery/port/origin behavior, changes macOS channel config UI, and adds tests plus a changelog entry.

## Best Possible Solution

Keep the memory fix only after separating bounded live context from append-only persisted history, preserving plugin-backed target normalization or an equivalent lightweight contract, splitting the unrelated macOS UI rewrite, and adding current-head proof for prune, rewrite, reopen, and delivery matching.

## Reproduction Assessment

Yes for the blocking findings from source inspection: the PR assigns a pruned subset to `fileEntries`, upstream `_rewriteFile()` writes exactly `fileEntries`, and OpenClaw still invokes `_rewriteFile()` on active managers. I did not run a failing runtime repro in this read-only review.

## Solution Assessment

No. A bounded wrapper can be the right direction, but the implementation must not make the persisted transcript depend on the pruned live cache, and source-delivery matching should keep the plugin normalizer contract instead of duplicating partial rules.
