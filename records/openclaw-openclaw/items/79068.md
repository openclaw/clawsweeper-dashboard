Full review comments:

- **[P1] Keep pruning separate from persisted fileEntries:** `src/agents/pi-session-manager-memory-bound.ts:197`
  - body: Assigning the retained subset back to `fileEntries` makes the pruned list the source used by upstream `_rewriteFile()`. The PR routes active embedded/CLI managers through this wrapper while existing paths still call `_rewriteFile()`, so a later rewrite can replace the persisted JSONL with only retained entries and lose pre-compaction disk history.
  - confidence: 0.94
- **[P1] Preserve provider target normalizers:** `src/infra/outbound/source-delivery-plan.ts:112`
  - body: This fallback strips only the channel prefix for most providers instead of calling `normalizeTargetForProvider`. Provider-owned forms such as LINE `line:group:<id>` normalize to `<id>` today; after this change they can be marked unverified, causing duplicate fallback delivery or wrong delivery accounting.
  - confidence: 0.87

Current CI now fails cron delivery-target tests because context keys gain `telegram:telegram:...`, the normalizer spy is not called, and a thread id is kept when normalization should fail, which can route topic delivery incorrectly.
  - confidence: 0.93
