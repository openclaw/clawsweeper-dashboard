## Summary

Keep open. The memory-bound approach targets a real session-retention path, but the current patch makes the pruned `fileEntries` array the persistence source, so existing rewrite/flush paths can truncate the full JSONL transcript.

## What This Changes

Adds a bounded Pi `SessionManager` wrapper, reroutes embedded/CLI/compaction session-manager usage through it, and includes related regression tests plus port/origin/pairing/changelog adjustments.

## Best Possible Solution

Keep the bounded-memory design only after separating live retention from the complete persisted transcript, so all rewrites and first-flush paths preserve the full JSONL history; split or prove unrelated origin/pairing changes if they remain.

## Reproduction Assessment

Yes. A source-reproducible path is: open a large compacted session through the wrapped manager, let pruning replace `fileEntries`, then hit the CLI before-agent-run blocked path so `_rewriteFile()` rewrites the persisted JSONL from the pruned array.

## Solution Assessment

No. The wrapper is a plausible narrow direction for live memory, but it must not let a pruned in-memory view become the disk rewrite source; the safer fix is to keep a full persisted-entry source or remove rewrite/first-flush dependence on pruned `fileEntries`.
