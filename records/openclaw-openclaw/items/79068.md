## Summary

Keep this PR open: the bounded live-memory direction is useful, but the current head still prunes `SessionManager.fileEntries` while Pi's rewrite hook serializes that same array back to disk, so a later rewrite can truncate persisted JSONL history despite the PR's append-only contract.

## What This Changes

The PR adds a bounded Pi SessionManager wrapper for embedded, CLI, and CLI-compaction paths, adds memory-retention and port-diagnostic coverage, updates the changelog, and includes some formatting-only churn.

## Best Possible Solution

Keep the bounded live-memory approach, but decouple the live-memory view from the persistence backing data or wrap rewrite paths so full JSONL history is preserved, then add a rewrite-after-prune regression test.

## Reproduction Assessment

Yes for the blocking review finding by source inspection: the PR shortens `fileEntries`, Pi `_rewriteFile()` writes `fileEntries`, and wrapped embedded/CLI paths can still call `_rewriteFile()` after pruning.

## Solution Assessment

No. The wrapper direction is plausible, but it is not mergeable until live-memory pruning is separated from persisted-transcript rewrite semantics.
