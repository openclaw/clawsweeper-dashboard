## Summary

Keep this PR open: the current patch still has a P1 transcript data-loss risk because it prunes Pi's persistence backing array while OpenClaw and Pi rewrite paths can later flush that same array back to JSONL.

## What This Changes

The PR adds a bounded wrapper around Pi SessionManager for gateway and CLI runtime paths, adjusts port/origin diagnostics, updates tests, applies macOS formatting, and adds a changelog entry.

## Best Possible Solution

Separate bounded live-memory state from the full persisted transcript, or make every rewrite path operate on complete JSONL history, then add a prune-plus-rewrite regression and current-head real behavior proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the PR assigns a pruned subset to `fileEntries`, upstream `_rewriteFile()` serializes `fileEntries`, and current OpenClaw runtime paths still invoke `_rewriteFile()` after mutations.

## Solution Assessment

No. The wrapper direction is plausible, but this implementation uses Pi's persistence backing array as the bounded cache; the safer solution needs rewrite-safe persistence handling or a separate live-memory view.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93
