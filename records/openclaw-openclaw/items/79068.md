## What This Changes

The PR adds a bounded Pi SessionManager wrapper for gateway/CLI session paths, updates source-delivery matching, port/origin tests, macOS config rendering, pnpm patch metadata, changelog text, and test stability code.

## Best Possible Solution

Keep the memory-bounding goal, but implement it as a rewrite-safe live-retention layer that preserves full transcript persistence, restore the plugin target-normalizer source of truth, and split or explicitly approve unrelated UI/origin/dependency changes before merge.

## Reproduction Assessment

Yes, at source level: current main opens Pi SessionManager in active gateway/CLI paths, and upstream Pi 0.75.4 keeps entries in `fileEntries` while `_rewriteFile()` serializes that same array. I did not run a gateway soak in this read-only review.

## Solution Assessment

No. The proposed wrapper bounds live memory by mutating `fileEntries`, which is upstream's full persistence rewrite source; the safer solution separates live retained context from the full transcript rewrite source and keeps plugin target normalization centralized.
