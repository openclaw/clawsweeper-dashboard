## Summary

Keep this PR open: current main still has the unbounded Pi SessionManager usage in the gateway/CLI paths, but the current patch is not mergeable because it can later rewrite the persisted JSONL from a pruned in-memory entry list, and it also carries macOS SwiftUI drift that likely breaks the build.

## What This Changes

The PR adds a bounded Pi SessionManager wrapper, routes embedded/CLI SessionManager opens through it, adds regression/port coverage and changelog text, and carries additional gateway auth, macOS, and formatting drift.

## Best Possible Solution

Land a focused SessionManager memory-bound change that keeps live memory bounded without rewriting disk from pruned entries, drops unrelated drift, restores the macOS helper, and includes current-head persisted-session proof plus targeted tests.

## Reproduction Assessment

Yes. The memory issue is source-reproducible from upstream SessionManager retaining every append in fileEntries and current main opening that manager directly; the patch-specific data-loss bug is also source-reproducible by pruning and then invoking an existing _rewriteFile flush path.

## Solution Assessment

No. The wrapper approach is plausible, but the current implementation must preserve full persisted transcript history across rewrite/flush paths and remove or fix unrelated macOS drift before it is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Protect persisted transcripts from pruned rewrites:** `src/agents/pi-session-manager-memory-bound.ts:197-199`
  - body: This replaces `manager.fileEntries` with only the retained post-compaction branch. OpenClaw still calls `_rewriteFile()` on active managers after append/cleanup paths, and upstream `_rewriteFile()` writes exactly `fileEntries` back to JSONL, so once pruning has happened a later flush can silently drop the historical transcript the PR says remains append-only. Keep a full persisted buffer or override/rework rewrite paths so disk history is never rebuilt from the pruned live cache.
  - confidence: 0.94
- **[P1] Restore the ViewBuilder on the SwiftUI helper:** `apps/macos/Sources/OpenClaw/ChannelConfigForm.swift:339-342`
  - body: `renderAdditionalProperties` returns `some View` but now has conditional and no-view paths without `@ViewBuilder`. When `additionalProperties` is absent there is no returned view, and the nested SwiftUI branches need the result builder, so this unrelated drift is likely to break the macOS target build. Restore the attribute/formatting or drop this change from the gateway memory PR.
  - confidence: 0.87
