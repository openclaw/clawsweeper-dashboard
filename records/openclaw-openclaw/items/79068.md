## Best Possible Solution

Keep the bounded-memory fix only after it separates live retention from persisted transcript rewrites, restores provider-aware source-delivery matching through a lightweight shared path, removes stale package-patch metadata, and supplies current-head redacted runtime proof.

## Reproduction Assessment

Yes for the blocking findings from source inspection: upstream _rewriteFile writes fileEntries, the PR assigns prunedEntries to fileEntries, and current OpenClaw still calls _rewriteFile on active managers. I did not run a live gateway soak in this read-only review.

## Solution Assessment

No. The right fix should bound live memory without making pruned entries the persistence source of truth, and source-delivery should keep plugin-owned normalization instead of reimplementing per-provider rules in this branch.
