## Summary

Keep open: the bounded-memory direction is useful, but the current PR is not merge-safe because it prunes Pi's persistence backing array while current runtime paths can still rewrite that array back to disk. The branch also carries unrelated macOS channel-form churn and needs current-head real behavior proof that covers the rewrite path.

## What This Changes

The PR adds a bounded Pi SessionManager wrapper for gateway and CLI session paths, regression coverage, port/origin diagnostics, a changelog entry, and macOS ChannelConfigForm changes.

## Best Possible Solution

Keep the bounded-memory approach only after separating live retained state from full persisted transcript history, remove the unrelated macOS UI drift, and require a current-head regression plus real behavior proof for rewrite-safe persistence.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: prune a persisted manager, trigger an existing _rewriteFile() path, then reopen the JSONL and stale pre-compaction history can be missing. I did not execute that repro in this read-only review.

## Solution Assessment

No. The bounded wrapper is plausible, but using Pi's fileEntries array as both the live cache and rewrite source is not the safest fix; persistence and bounded context state need separate contracts or a complete-history rewrite source.
