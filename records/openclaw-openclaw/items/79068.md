## Summary

Keep this PR open; it is not merge-ready. The useful memory-bounding goal is real, but source review found a blocking transcript-preservation risk, plus unrelated delivery/UI changes and stale proof against an older head and Pi package version.

## What This Changes

The PR adds a bounded OpenClaw wrapper around Pi SessionManager, routes gateway/CLI SessionManager opens through it, and also changes source-delivery matching, macOS channel config rendering, port/origin tests, package scripts, and changelog text.

## Best Possible Solution

Keep the memory fix, but separate durable transcript state from the bounded live view or use an upstream-safe SessionManager API, preserve plugin-backed target normalization, rebase away unrelated UI churn, and refresh current-head proof that includes rewrite/reopen behavior.

## Reproduction Assessment

Yes, at source level: the PR prunes `fileEntries`, Pi 0.75.4 `_rewriteFile()` writes `fileEntries`, and current OpenClaw runtime paths still call `_rewriteFile()`. I did not run a live gateway soak or tests because this was a read-only review.

## Solution Assessment

No. A bounded live-memory view is a reasonable direction, but this implementation conflates live and durable state, changes delivery matching, and carries stale macOS UI churn that should be removed or split.
