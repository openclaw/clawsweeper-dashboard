## Summary

Keep open. The SessionManager memory fix is still useful and has convincing proof, but the current PR regresses deprecated plugin runtime config helper scoping and the runtime dependency patch still needs explicit maintainer approval before merge.

## What This Changes

This PR patches `@earendil-works/pi-coding-agent@0.74.0` to prune live SessionManager entries after compaction, adds focused regression and port tests, updates port diagnostics, and adjusts related dependency metadata/test imports.

## Best Possible Solution

Preserve runtime config scope for all config helpers, then land bounded SessionManager pruning after maintainer approval of the pnpm patch or an upstream `pi-coding-agent` release.

## Reproduction Assessment

Yes, from source inspection. The published `@earendil-works/pi-coding-agent@0.74.0` package loads and appends every JSONL entry into `fileEntries`/`byId` without pruning, so a long persisted session with compaction can reproduce the unbounded live-memory path.

## Solution Assessment

No, not as submitted. The pruning target is focused, but the branch must keep both deprecated and current config helpers scoped and still needs maintainer approval or an upstream release for the dependency patch.

## Review Findings
