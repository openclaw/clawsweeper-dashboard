Keep open: the gateway memory direction is plausible and tied to real session-retention reports, but this PR is not merge-ready because the bounded wrapper can turn a later SessionManager rewrite into persisted transcript truncation, it removes release-soak automation, and the supplied real behavior proof is stale and misses that rewrite path.

## Best Possible Solution

Keep the SessionManager memory fix, but separate bounded live context from full persisted rewrite state, restore release-soak automation, and require current-head proof that pruning, rewrite, reopen, and visible source delivery all preserve expected behavior.

## Solution Assessment

No. The wrapper direction is maintainable only if full JSONL persistence is kept separate from bounded live context or rewrite paths reload/merge full history; the current PR mutates the rewrite source of truth and also bundles unrelated automation/UI/delivery changes.
