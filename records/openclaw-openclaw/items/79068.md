## Summary

Keep this PR open: it targets a real SessionManager memory-retention problem, but the latest wrapper can still turn an in-memory prune into persisted transcript loss when existing rewrite paths run, and the real-behavior proof is stale after the branch moved away from a patched Pi package.

## What This Changes

The PR adds an OpenClaw SessionManager wrapper that prunes stale pre-compaction live entries, routes several runner open paths through it, adds regression coverage, adjusts same-process port diagnostics, and adds a changelog entry.

## Best Possible Solution

Keep the runtime memory bound or upstream an equivalent Pi fix, but preserve the full persisted JSONL across every rewrite path and refresh real-environment proof against the current wrapper branch.

## Reproduction Assessment

Yes. The original memory symptom is source-reproducible from Pi 0.74.0 retaining every entry in `fileEntries`/`byId`, and the PR regression is source-reproducible because existing OpenClaw `_rewriteFile()` callers serialize the pruned `fileEntries` array.

## Solution Assessment

No. The wrapper is a plausible narrow fix, but it must make Pi rewrite paths preserve disk history or avoid replacing `fileEntries` in a way that later rewrites can persist.
