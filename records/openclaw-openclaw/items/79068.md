## Best Possible Solution

Make the bounded SessionManager rewrite-safe by separating retained live context from full persistence, then rebase and split or explicitly accept the companion transport/UI/package changes with current-head proof.

## Reproduction Assessment

Yes, at source level: the PR prunes `manager.fileEntries`, upstream `_rewriteFile()` serializes that array, and patched runtime paths still call `_rewriteFile()`. I did not run a live gateway soak in this read-only review.

## Solution Assessment

No: bounding live memory is the right direction, but pruning the persistence backing array is not a safe boundary. The safer approach keeps full persistence separate from the retained context view or moves the contract upstream.
