## Summary

Keep open: the PR still mutates Pi's `fileEntries` to the pruned live-memory view, and Pi plus OpenClaw rewrite paths can later serialize that shortened array back to disk, dropping historical JSONL entries the PR promises to preserve.

## What This Changes

The PR adds a bounded Pi SessionManager wrapper, routes embedded and CLI session-manager openings through it, adds memory-retention and port-diagnostic tests, and updates the changelog.

## Best Possible Solution

Keep the bounded live-memory behavior, but decouple live-context pruning from persistence or wrap rewrite paths so every rewrite preserves the full append-only transcript, with a regression test that prunes and then rewrites.

## Reproduction Assessment

Yes. Source inspection shows the failure path: the wrapper prunes `fileEntries`, Pi `_rewriteFile()` serializes `fileEntries`, and current CLI/embedded cleanup paths can call `_rewriteFile()` after the wrapped manager has pruned.

## Solution Assessment

No. The wrapper direction is narrow and plausible, but it is not mergeable until the persistence rewrite contract is protected and covered by a rewrite-after-prune regression test.
