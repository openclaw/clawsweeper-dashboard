## Summary

Close: current main fixes the observed non-thinking Ollama 400 by omitting truthy native `think` for models marked `reasoning: false`; the remaining fallback-next-none behavior matches the documented policy for exact user `/model` selections.

## What This Changes

The issue asks OpenClaw to avoid a silent turn when a session model override selects a non-thinking Ollama model and the next request would include unsupported thinking parameters.

## Best Possible Solution

Keep the capability-aware Ollama request shaping on main and leave strict user override fallback policy unchanged unless maintainers separately decide to change model-selection semantics.

## Reproduction Assessment

No for a current-main failure. The original 2026.5.5 path is credible from the logs, but current main source and regression tests omit `think` for non-reasoning Ollama models, so I did not establish the reported failure on main.

## Solution Assessment

Yes for the observed failure: omitting unsupported Ollama thinking is the narrowest maintainable fix. Changing user-selected model overrides to fall through to unrelated defaults would be a separate product-policy change and conflicts with current docs.
