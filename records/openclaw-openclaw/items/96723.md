## Summary

Keep open: current main still leaves the legacy Anthropic provider SSE success-body read unbounded, while this PR applies the existing shared guard cleanly. The remaining question is maintainer acceptance of the fixed 16 MiB fail-closed boundary for Anthropic-compatible streaming responses.

## What This Changes

This PR wraps the legacy Anthropic Messages provider SSE parser with the shared streaming byte guard at a 16 MiB cap and adds direct regression coverage.

## Best Possible Solution

Land this provider-path cap if maintainers accept the 16 MiB boundary; otherwise tune the cap and matching regression test before merge.

## Reproduction Assessment

## Solution Assessment

Yes, with a compatibility caveat: reusing the existing core guard is the narrowest maintainable fix for this parser. A new SDK export or config knob would be broader than this bugfix unless maintainers reject the fixed threshold.
