The branch adds an internal streaming byte guard, wraps `src/llm/providers/anthropic.ts` SSE reads with a 16 MiB cap, exports the iterator for tests, and adds a regression test.

## Best Possible Solution

Land the narrow provider-path SSE guard after maintainers accept or tune the 16 MiB boundary; keep SDK promotion and sibling parser hardening in separate focused PRs.

## Reproduction Assessment

## Solution Assessment

Yes, if maintainers accept the 16 MiB boundary. Guarding the reader preserves the existing SSE parser and avoids public SDK surface, while sibling parser coverage is already split into separate PRs.
