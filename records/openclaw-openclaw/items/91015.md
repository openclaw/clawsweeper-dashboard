## Summary

This PR should stay open: the source-backed hang is real and the patch is plausible, but it adds a public sandbox Docker timeout/default without aligned help/docs and still lacks redacted real Docker/OrbStack proof for the wedged-engine path.

## What This Changes

The PR adds a per-call Docker sandbox-init deadline, a new `agents.defaults.sandbox.docker.initTimeoutMs` config/default, doctor messaging, and mock regression tests for hung Docker children.

## Best Possible Solution

Land one canonical bounded sandbox-init timeout with config help, labels, docs, upgrade/default rationale, and redacted real wedged-Docker proof.

## Reproduction Assessment

Yes, source-reproducible: a Docker child that never emits `close` or `error` leaves current `execDockerRaw` pending, and sandbox init calls it without a deadline; no live OrbStack/Docker repro was run or attached.

## Solution Assessment

No, not yet best/merge-ready: the per-call deadline is a plausible repair, but the public config surface lacks help/docs/labels, real behavior proof is missing, and maintainers still need to choose it over the broader timeout design.
