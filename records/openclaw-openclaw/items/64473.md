## Summary

Keep open for maintainer/security handling, but the patch should not merge as-is: it widens a shared root-boundary primitive beyond the workspace-bootstrap symptom, conflicts with current `main`, and overlaps the narrower open fix path at https://github.com/openclaw/openclaw/pull/73401.

## What This Changes

The PR adds hard-coded `~/.openclaw` containment fallbacks to boundary path checks so workspace bootstrap symlinks can target sibling paths under the OpenClaw state tree.

## Best Possible Solution

Use a scoped fixed-bootstrap reader policy, or an explicit maintainer decision to reject external bootstrap symlinks, while keeping generic fs-safe/root-boundary helpers strict by default.

## Reproduction Assessment

Yes. Source inspection shows a fixed bootstrap symlink that resolves outside `workspaceDir` goes through `openRootFile`, returns a non-ok result, becomes `missing: true`, and renders as `[MISSING] Expected at: ...`; no runtime fixture was run in this read-only review.

## Solution Assessment

No. A global `~/.openclaw` fallback in shared boundary checks is broader than the symptom, while the narrower maintainable path is a fixed-bootstrap reader exception or an explicit unsupported-policy decision.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Restrict the fallback to symlink traversal:** `src/infra/boundary-path.ts:642-646`
  - body: `assertLexicalBoundaryOrCanonicalAlias` runs for any absolute path outside the root, not only followed symlinks. Returning for every canonical path under `~/.openclaw` can let direct state paths pass the workspace boundary, so this exception must not live in the generic guard.
  - confidence: 0.9
- **[P1] Scope the exception to bootstrap readers:** `src/infra/boundary-path.ts:313-317`
  - body: `applyResolvedSymlinkHop` is a shared boundary primitive. Any sandbox, hook, install, media, or generic safe-file root containing a symlink to `~/.openclaw` would inherit a broader target boundary than the caller requested, so this needs a bootstrap-specific mode or reader-layer fallback.
  - confidence: 0.88
- **[P2] Use the configured canonical state root:** `src/infra/boundary-path.ts:316-317`
  - body: The fallback computes `path.join(os.homedir(), '.openclaw')`, which ignores `OPENCLAW_STATE_DIR` and `OPENCLAW_HOME` and compares a lexical path with realpath-derived values. That makes the trust decision wrong for overridden or symlinked state roots.
  - confidence: 0.82
