## Summary

Keep open. The underlying bootstrap-symlink bug is still present on current main and the latest shipped tag, but this branch is not merge-ready because it broadens shared boundary checks, conflicts with the current fs-safe extraction, and lacks inspectable real behavior proof.

## What This Changes

The PR adds `~/.openclaw` containment fallbacks to `src/infra/boundary-path.ts` so workspace bootstrap symlinks can resolve to sibling directories under the OpenClaw state tree.

## Best Possible Solution

Keep shared fs-safe/root-boundary helpers strict by default and solve this through a scoped fixed-bootstrap reader policy, or explicitly document that external bootstrap symlinks are unsupported.

## Reproduction Assessment

Yes, by source inspection. A fixed bootstrap symlink resolving outside `workspaceDir` reaches `openRootFile`, returns non-ok under the fs-safe root-boundary contract, becomes `missing: true`, and renders as `[MISSING] Expected at: ...`; no filesystem fixture was run in this read-only review.

## Solution Assessment

No. A global `~/.openclaw` fallback in shared boundary checks is broader than the symptom; the narrower maintainable path is a fixed-bootstrap reader exception with explicit security boundaries, or an explicit unsupported-policy decision.
