## Summary

Keep open, but the proposed patch is not mergeable as-is: it addresses a real symlinked-bootstrap symptom by relaxing a shared security boundary, permits broader OpenClaw state-tree access than the PR describes, ignores the configured state root, and lacks inspectable real behavior proof.

## What This Changes

The PR adds hard-coded `~/.openclaw` containment fallbacks to three boundary path checks so workspace symlink targets outside the workspace root can pass.

## Best Possible Solution

Keep the generic fs-safe boundary root-confined by default, then land or explicitly supersede a scoped fixed-bootstrap reader implementation with negative security coverage.

## Reproduction Assessment

Yes, by source inspection. A fixed bootstrap symlink resolving outside the workspace root reaches `openRootFile`, fails the workspace-root boundary contract, is recorded as missing, and renders as `[MISSING] Expected at: ...`; no runtime fixture was run in this read-only review.

## Solution Assessment

No. A global `~/.openclaw` fallback in shared boundary checks is broader than the workspace-bootstrap symptom; a scoped fixed-bootstrap reader exception or explicit boundary mode is the narrower maintainable path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Restrict the fallback to symlink traversal:** `src/infra/boundary-path.ts:642-646`
  - body: `assertLexicalBoundaryOrCanonicalAlias` runs for any absolute path outside the root, not only followed symlinks. Returning here for every canonical path under `~/.openclaw` can let direct state paths pass the workspace boundary, so this exception must not live in the generic guard.
  - confidence: 0.9
- **[P1] Scope the exception to bootstrap readers:** `src/infra/boundary-path.ts:313-317`
  - body: `applyResolvedSymlinkHop` is a shared boundary primitive. Any sandbox, hook, install, media, or generic safe-file root containing a symlink to `~/.openclaw` would inherit a broader target boundary than the caller requested, so this needs a bootstrap-specific mode or reader-layer fallback.
  - confidence: 0.88
- **[P2] Use the configured canonical state root:** `src/infra/boundary-path.ts:316-317`
  - body: The fallback computes `path.join(os.homedir(), '.openclaw')`, which ignores `OPENCLAW_STATE_DIR` and `OPENCLAW_HOME` and compares a lexical path with realpath-derived values. That makes the trust decision wrong for overridden or symlinked state roots.
  - confidence: 0.82
