## Summary

Keep open: the reported bootstrap symlink problem is source-reproducible, but this branch is not safe as-is because it widens a shared filesystem boundary, lacks inspectable real behavior proof, and overlaps an open scoped replacement PR while the linked issue remains open.

## What This Changes

The PR adds `~/.openclaw` fallback containment checks to `src/infra/boundary-path.ts` so workspace symlinks can resolve to sibling OpenClaw state directories.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73401 or an equivalent fixed-bootstrap-only symlink reader with negative security coverage, while preserving the shared fs-safe boundary invariant.

## Reproduction Assessment

Yes. Source inspection shows fixed workspace bootstrap files are opened with the workspace root as the boundary and guarded-read failures are recorded as missing entries; I did not run a filesystem fixture because this is a read-only review.

## Solution Assessment

No. A broad `~/.openclaw` fallback in shared boundary checks is not the narrowest maintainable solution; a scoped fixed-bootstrap exception or explicit boundary mode is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Restrict the fallback to symlink traversal:** `src/infra/boundary-path.ts:642-646`
  - body: `assertLexicalBoundaryOrCanonicalAlias` runs for any absolute path that is lexically outside the root, not only followed symlinks. With this fallback, a direct path under `~/.openclaw/credentials` can pass the workspace boundary without symlink traversal, so the exception must not live in the generic outside-lexical guard.
  - confidence: 0.9
- **[P1] Gate the exception to workspace bootstrap reads:** `src/infra/boundary-path.ts:316-317`
  - body: `applyResolvedSymlinkHop` is shared by sandbox, managed-media, install, hook, and generic safe-file callers. As written, any such root can contain a symlink to `~/.openclaw/...` and inherit a broader trust boundary than the caller requested, so the exception needs a workspace-bootstrap-specific mode or should be moved to the bootstrap reader layer.
  - confidence: 0.88
- **[P2] Use the configured canonical state root:** `src/infra/boundary-path.ts:316-317`
  - body: The fallback computes `path.join(os.homedir(), ".openclaw")`, which ignores configured OpenClaw home/state locations and compares a lexical path with realpath-derived values. That makes the trust decision wrong for overridden or symlinked state roots, so the effective state root must be resolved canonically before use.
  - confidence: 0.83
