- **[P1] Restrict the fallback to symlink traversal:** `src/infra/boundary-path.ts:642-646`
  - body: `assertLexicalBoundaryOrCanonicalAlias` runs for any absolute path outside the root, not only followed symlinks. Returning for every canonical path under `~/.openclaw` can let direct state paths pass the workspace boundary, so this exception must not live in the generic guard.
  - confidence: 0.9
- **[P1] Scope the exception to bootstrap readers:** `src/infra/boundary-path.ts:316-317`
  - body: `applyResolvedSymlinkHop` is a shared boundary primitive. Any sandbox, hook, media, plugin, or generic safe-file root containing a symlink to `~/.openclaw` would inherit a broader target boundary than the caller requested, so this needs a bootstrap-specific mode or reader-layer fallback.
  - confidence: 0.88
- **[P2] Use the configured canonical state root:** `src/infra/boundary-path.ts:316-317`
  - body: The fallback computes `path.join(os.homedir(), '.openclaw')`, which ignores configured state roots and compares a lexical path with realpath-derived values. That makes the trust decision wrong for overridden or symlinked state locations.
  - confidence: 0.82
