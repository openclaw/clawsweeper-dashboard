## Summary

Keep open for maintainer review. The reported bootstrap-symlink problem is source-reproducible, but this PR widens a shared path-boundary primitive and is already paired with a scoped maintainer-labeled replacement path.

## What This Changes

The PR adds `~/.openclaw` containment fallbacks to three `src/infra/boundary-path.ts` checks so workspace symlink targets outside the workspace root can pass when they land under the OpenClaw state tree.

## Best Possible Solution

Keep the generic fs-safe boundary root-confined by default, then land or explicitly supersede a scoped fixed-bootstrap reader implementation with negative security coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a fixed bootstrap symlink resolving outside the workspace root reaches `openRootFile`, fails the root-boundary contract, is recorded as missing, and is rendered as `[MISSING] Expected at: ...`.

## Solution Assessment

No. Relaxing `src/infra/boundary-path.ts` globally is wider than the reported workspace-bootstrap use case; a reader-scoped exception or explicit boundary mode is the narrower maintainable path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Restrict the fallback to symlink traversal:** `src/infra/boundary-path.ts:642-646`
  - body: `assertLexicalBoundaryOrCanonicalAlias` runs for any absolute path outside the root, not only followed symlinks. Returning here for every canonical path under `~/.openclaw` can let direct state paths pass the workspace boundary, so this exception must not live in the generic guard.
  - confidence: 0.9
- **[P1] Scope the exception to bootstrap readers:** `src/infra/boundary-path.ts:313-317`
  - body: `applyResolvedSymlinkHop` is a shared boundary primitive. Any sandbox, hook, install, media, or generic safe-file root containing a symlink to `~/.openclaw` would inherit a broader target boundary than the caller requested, so this needs a bootstrap-specific mode or reader-layer fallback.
  - confidence: 0.88
- **[P2] Use the configured canonical state root:** `src/infra/boundary-path.ts:316-317`
  - body: The fallback computes `path.join(os.homedir(), '.openclaw')`, which ignores `OPENCLAW_STATE_DIR`/`OPENCLAW_HOME` and compares a lexical path with realpath-derived values. That makes the trust decision wrong for overridden or symlinked state roots.
  - confidence: 0.82
