## Summary

Keep open: the use case is real, but this PR widens a shared boundary primitive and has security findings; https://github.com/openclaw/openclaw/pull/73401 tracks a safer scoped direction but is still open.

## What This Changes

The branch adds `~/.openclaw` fallbacks to three `src/infra/boundary-path.ts` containment checks so symlinked workspace bootstrap files can resolve to sibling OpenClaw state directories.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73401 or an equivalent workspace-bootstrap-only symlink reader with negative security coverage, while preserving the shared fs-safe boundary invariant.

## Reproduction Assessment

Yes. Source inspection shows fixed bootstrap files are opened with the workspace root as the boundary, and guarded-read failures are recorded as missing entries; I did not run tests because this is a read-only review.

## Solution Assessment

No. A broad `~/.openclaw` fallback in shared boundary checks is not the narrowest maintainable solution; a scoped fixed-bootstrap exception or explicit boundary mode is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Restrict the fallback to symlink traversal:** `src/infra/boundary-path.ts:642-646`
  - body: `assertLexicalBoundaryOrCanonicalAlias` runs for any absolute path that is lexically outside the root, not only followed symlinks. With this fallback, a direct path under `~/.openclaw/credentials` can pass the workspace boundary without symlink traversal, so the exception must not live in the generic outside-lexical guard.
  - confidence: 0.9
- **[P1] Gate the exception to the intended boundary mode:** `src/infra/boundary-path.ts:316-317`
  - body: `applyResolvedSymlinkHop` is shared by sandbox, managed-media, and generic safe-file callers. As written, any such root can contain a symlink to `~/.openclaw/...` and inherit a broader trust boundary than the caller requested, so the exception needs an explicit workspace-bootstrap mode or equivalent guard.
  - confidence: 0.88
- **[P2] Use the configured canonical state root:** `src/infra/boundary-path.ts:316-317`
  - body: The fallback computes `path.join(os.homedir(), ".openclaw")`, which ignores configured state locations and compares a lexical path with realpath-derived values. That makes the trust decision wrong for overridden or symlinked state roots, so the effective state root must be resolved canonically before use.
  - confidence: 0.83

## Security Review
