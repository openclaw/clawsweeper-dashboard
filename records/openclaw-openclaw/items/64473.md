## Summary

Keep open. Current main still enforces root-only boundary checks, so the linked workspace-symlink problem remains real, but this PR broadens a shared security boundary too far by accepting generic `~/.openclaw` targets in multiple enforcement points.

## What This Changes

The PR changes `src/infra/boundary-path.ts` so boundary resolution can accept canonical targets under `os.homedir()/.openclaw` even when they are outside the configured root.

## Best Possible Solution

Keep the PR and linked issue open, but redesign the change as an explicit workspace-bootstrap symlink policy: resolve the effective state root canonically, apply the exception only when the boundary root is an OpenClaw workspace under that state root, preserve direct-path and non-workspace rejection, and add regression tests for allowed bootstrap symlinks plus blocked credentials, sessions, sandbox, tmp, and managed-media escapes.

## Reproduction Assessment

Yes. The current-main source path is clear: workspace bootstrap calls `openBoundaryFile`, `resolveBoundaryPath` follows symlink parents, and `applyResolvedSymlinkHop` rejects canonical targets outside the workspace root; existing boundary tests also assert symlink escapes are rejected.

## Solution Assessment

No. The PR solves the user-visible missing-bootstrap symptom by widening generic path containment, but the safer maintainable fix is a narrow workspace-bootstrap symlink mode tied to the effective canonical OpenClaw state root.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Restrict the fallback to symlink traversal:** `src/infra/boundary-path.ts:645-649`
  - body: `assertLexicalBoundaryOrCanonicalAlias` runs for any absolute path that is lexically outside the root, not only followed symlinks. With this fallback, a direct path under `~/.openclaw/credentials` can pass the workspace boundary without any symlink traversal.
  - confidence: 0.9
- **[P1] Scope the exception to workspace roots:** `src/infra/boundary-path.ts:316-317`
  - body: `applyResolvedSymlinkHop` is shared by sandbox, tmp, managed-media, and other boundary checks. As written, any such root can contain a symlink to `~/.openclaw/...` and inherit a broader trust boundary than the caller requested.
  - confidence: 0.87
- **[P2] Use the effective canonical state root:** `src/infra/boundary-path.ts:316-317`
  - body: The fallback computes `path.join(os.homedir(), ".openclaw")`, which ignores `OPENCLAW_STATE_DIR`, `OPENCLAW_CONFIG_PATH`, and `OPENCLAW_HOME`, and it compares a lexical root to canonical targets. Resolve the effective state root canonically before using it for a trust decision.
  - confidence: 0.82
