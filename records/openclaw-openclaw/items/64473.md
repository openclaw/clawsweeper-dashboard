## What This Changes

The PR adds `~/.openclaw` containment fallbacks to three shared boundary-path checks so symlinked workspace bootstrap files can resolve to sibling OpenClaw state directories.

## Best Possible Solution

Keep the shared fs-safe boundary root-confined by default, then land or explicitly supersede a scoped fixed-bootstrap reader implementation such as https://github.com/openclaw/openclaw/pull/73401 with negative security coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a fixed bootstrap symlink resolving outside the workspace root reaches `openRootFile`, fails the fs-safe boundary contract, is recorded as missing, and is rendered as `[MISSING] Expected at: ...`; I did not create a filesystem fixture because this review is read-only.

## Solution Assessment

No. Relaxing `src/infra/boundary-path.ts` globally is wider than the reported workspace-bootstrap use case; a reader-scoped exception or explicit boundary mode is the narrower maintainable path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Restrict the fallback to symlink traversal:** `src/infra/boundary-path.ts:645-648`
  - body: `assertLexicalBoundaryOrCanonicalAlias` runs for any absolute path outside the root, not only followed symlinks. This return can let direct `~/.openclaw` state paths pass the workspace boundary without symlink traversal, so the exception must not live in this generic guard.
  - confidence: 0.9
- **[P1] Scope the exception to bootstrap readers:** `src/infra/boundary-path.ts:316-317`
  - body: `applyResolvedSymlinkHop` is a shared boundary primitive. Any sandbox, hook, install, media, or generic safe-file root containing a symlink to `~/.openclaw` would inherit a broader target boundary than the caller requested, so this needs a bootstrap-specific mode or reader-layer fallback.
  - confidence: 0.88
- **[P2] Use the configured canonical state root:** `src/infra/boundary-path.ts:316-317`
  - body: The fallback computes `path.join(os.homedir(), '.openclaw')`, which ignores configured state locations and compares a lexical path with realpath-derived values. That makes the trust decision wrong for overridden or symlinked state roots.
  - confidence: 0.82
