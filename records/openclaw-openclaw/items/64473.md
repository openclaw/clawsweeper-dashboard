## Summary

Keep open: the reported user problem is plausible, but this branch is not safe to merge because it broadens a shared path-boundary primitive and a narrower open replacement already tracks the same workspace-bootstrap behavior.

## What This Changes

The branch adds `os.homedir()/.openclaw` fallbacks to three `src/infra/boundary-path.ts` containment checks so symlinked workspace bootstrap files under sibling state directories can load.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73401 or an equivalent narrow workspace-bootstrap reader fix while preserving the shared fs-safe boundary invariant.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: fixed workspace bootstrap files are opened with the workspace as the boundary root, while current tests still reject symlinks that resolve outside that root. I did not run tests because this was a read-only review.

## Solution Assessment

No. Widening the generic boundary primitive is not the narrowest maintainable solution; the safer path is a scoped workspace-bootstrap exception or explicit boundary mode with negative security coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Restrict the fallback to symlink traversal:** `src/infra/boundary-path.ts:642-646`
  - body: `assertLexicalBoundaryOrCanonicalAlias` runs for any absolute path that is lexically outside the root, not only followed symlinks. With this fallback, a direct path under `~/.openclaw/credentials` can pass the workspace boundary without symlink traversal, so this exception must not live in the generic outside-lexical guard.
  - confidence: 0.9
- **[P1] Gate the exception to the intended boundary mode:** `src/infra/boundary-path.ts:316-317`
  - body: `applyResolvedSymlinkHop` is shared by sandbox, tmp, managed-media, SSH upload, and generic safe-file callers. As written, any such root can contain a symlink to `~/.openclaw/...` and inherit a broader trust boundary than the caller requested, so the exception needs an explicit workspace-bootstrap mode or equivalent guard.
  - confidence: 0.88
- **[P2] Use the configured canonical state root:** `src/infra/boundary-path.ts:316-317`
  - body: The fallback computes `path.join(os.homedir(), ".openclaw")`, which ignores configured state locations and compares a lexical path with realpath-derived values. That makes the trust decision wrong for overridden or symlinked state roots, so the effective state root must be resolved canonically before use.
  - confidence: 0.83
