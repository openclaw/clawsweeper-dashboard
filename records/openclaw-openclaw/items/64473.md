## Summary

This PR should stay open for maintainer/security handling: it addresses a real workspace-bootstrap symlink problem, but the patch relaxes a shared path-boundary primitive too broadly and an open maintainer-scoped replacement already tracks the safer bootstrap-layer approach.

## What This Changes

The PR adds `~/.openclaw` fallback containment checks to `src/infra/boundary-path.ts` so workspace symlink targets under sibling OpenClaw state directories can pass boundary validation.

## Best Possible Solution

Land the scoped bootstrap-reader fix in https://github.com/openclaw/openclaw/pull/73401 or an equivalent implementation with negative security coverage, keeping the shared boundary-path primitive root-confined by default.

## Reproduction Assessment

Yes. Source inspection shows fixed workspace bootstrap files are opened with the workspace directory as their boundary, so symlinks to sibling `~/.openclaw` directories fail the current guard and are recorded as missing bootstrap entries.

## Solution Assessment

No. Relaxing `src/infra/boundary-path.ts` globally is not the narrowest maintainable fix; the safer path is a workspace-bootstrap-specific exception or dedicated boundary mode with regression and negative security tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Restrict the fallback to symlink traversal:** `src/infra/boundary-path.ts:642-646`
  - body: `assertLexicalBoundaryOrCanonicalAlias` runs for any absolute path that is lexically outside the root, not only for followed symlinks. With this fallback, a direct path under `~/.openclaw/credentials` can pass the workspace boundary without symlink traversal, so the exception must not live in this generic outside-lexical guard.
  - confidence: 0.9
- **[P1] Keep the exception out of generic boundary callers:** `src/infra/boundary-path.ts:316-317`
  - body: `applyResolvedSymlinkHop` is shared by sandbox, install, hook, managed-media, and generic safe-file callers. As written, any such root can contain a symlink to `~/.openclaw/...` and inherit a broader trust boundary than the caller requested, so the exception needs a workspace-bootstrap-specific mode or reader-layer handling.
  - confidence: 0.88
- **[P2] Use the configured canonical state root:** `src/infra/boundary-path.ts:316-317`
  - body: The fallback computes `path.join(os.homedir(), ".openclaw")`, which ignores configured OpenClaw state locations and compares a lexical path with realpath-derived values. That makes the trust decision wrong for overridden or symlinked state roots, so the effective state root must be resolved canonically before use.
  - confidence: 0.82
