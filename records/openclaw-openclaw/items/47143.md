## What This Changes

The issue reports that `memoryFlush` is incorrectly skipped for never-compacted sessions when both `compactionCount` and `memoryFlushCompactionCount` are `0`.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/83015; after a fix merges, this issue can close as implemented.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: a row with `compactionCount: 0`, `memoryFlushCompactionCount: 0`, fresh tokens above threshold, and no `memoryFlushAt` reaches the equality guard and is skipped as already flushed. I did not run tests because this review is read-only.

## Solution Assessment

Yes, the best fix is a narrow state-contract disambiguation such as the linked pull request's `memoryFlushAt` guard, because it allows legacy or never-flushed 0/0 rows through while preserving real cycle-0 flush deduplication.
