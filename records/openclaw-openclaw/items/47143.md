## Summary

Keep open. Current main still treats the reported 0/0 memory-flush counter state as already flushed, and the merged adjacent counter fix does not change that helper or the focused test expectation.

## What This Changes

The issue reports that memoryFlush skips the first eligible flush for persisted sessions with both compaction counters at 0.

## Best Possible Solution

Define a backward-compatible memory-flush state contract that treats 0/0 rows without a flush marker as not flushed while preserving deduplication for real cycle-0 flush records, with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an above-threshold entry with `compactionCount: 0` and `memoryFlushCompactionCount: 0` reaches the helper, which returns true and causes `shouldRunMemoryFlush` to skip.

## Solution Assessment

No current fix is present. The narrow maintainable solution is to distinguish never-flushed legacy rows from recorded cycle-0 flushes, not to add a broad pre-request guard or remove once-per-cycle deduplication.
