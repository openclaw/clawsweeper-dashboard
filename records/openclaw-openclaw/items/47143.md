## Summary

## What This Changes

The issue reports that memoryFlush is skipped for never-flushed sessions whose compaction and memory-flush counters are both zero.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83015 or an equivalent narrow guard fix, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes. Source inspection on current main shows the reported 0/0 state returns true from `hasAlreadyFlushedForCurrentCompaction`, and the adjacent test still asserts that behavior; I did not run tests because this review is read-only.

## Solution Assessment

Yes. The best current path is the narrow linked PR approach: disambiguate real cycle-0 flushes from never-flushed 0/0 rows while keeping broader pre-request context-budget work separate.
