## What This Changes

The issue reports that `memoryFlush` skips never-compacted sessions when `compactionCount` and `memoryFlushCompactionCount` are both `0`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83015 or a narrow equivalent, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes, at source level: current main's helper returns true for a numeric `0` matching the defaulted compaction count of `0`, and the runtime uses that helper to skip memory flush. I did not run tests because this review is read-only.

## Solution Assessment

Yes, the linked PR's `memoryFlushAt` discriminator is the narrowest existing-state fix for distinguishing never-flushed 0/0 rows from real cycle-zero flushes. The issue itself should stay open until that PR or an equivalent lands.
