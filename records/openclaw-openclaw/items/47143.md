## What This Changes

The issue reports that `memoryFlush` is skipped for above-threshold sessions when both `compactionCount` and `memoryFlushCompactionCount` are `0`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83015, or an equivalent `memoryFlushAt`-aware fix, while keeping broader preflight context-budget work separate.

## Reproduction Assessment

Yes, at source level. An above-threshold entry with `compactionCount: 0` and `memoryFlushCompactionCount: 0` reaches the equality helper, returns true for already flushed, and suppresses the memory-flush path.

## Solution Assessment

Yes for the linked `memoryFlushAt`-aware direction. It is narrower than changing all cycle-0 rows to unflushed because it preserves dedupe for real cycle-0 flush records while allowing never-flushed legacy rows through.
