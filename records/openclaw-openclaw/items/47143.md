## What This Changes

The issue reports that memoryFlush is skipped for never-compacted sessions when both `compactionCount` and `memoryFlushCompactionCount` are 0.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83015 or an equivalent narrow fix that uses `memoryFlushAt` to distinguish real cycle-0 flushes from legacy/no-flush 0/0 rows, then close this issue via the merge.

## Reproduction Assessment

## Solution Assessment

Yes. The best fix is the narrow state-contract disambiguation used by the linked PR, because `memoryFlushAt` already separates real recorded flushes from legacy or never-flushed 0/0 rows without broadening the issue into pre-request context-budget work.
