## Summary

Keep open. Current main and the latest release still treat the reported 0/0 memory-flush counter state as already flushed, and the open closing PR has not merged yet.

## What This Changes

The request is to let memoryFlush run for never-flushed sessions whose compaction counters are both 0 instead of treating them as already flushed.

## Best Possible Solution

Review, rebase, and land https://github.com/openclaw/openclaw/pull/83015 or an equivalent `memoryFlushAt`-aware fix while keeping broader pre-request context-budget work separate.

## Reproduction Assessment

Yes, at source level. An above-threshold entry with `compactionCount: 0` and `memoryFlushCompactionCount: 0` reaches the equality helper, returns true for already flushed, and suppresses the memory-flush path.

## Solution Assessment

Yes for the linked `memoryFlushAt`-aware direction; it is narrower than treating every cycle-0 row as unflushed because it preserves dedupe for real cycle-0 flush records. Current main is not fixed yet.
