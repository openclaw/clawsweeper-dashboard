## Summary

Keep open. Current main and v2026.5.12 still treat an explicit 0/0 memory-flush counter state as already flushed, but an open closing PR now owns the focused fix path.

## What This Changes

The issue reports that memoryFlush is skipped when both compactionCount and memoryFlushCompactionCount are 0, even for never-flushed above-threshold sessions.

## Best Possible Solution

Review and land or replace the linked fix so legacy or never-flushed 0/0 rows can flush while real cycle-0 flushes remain deduped with memoryFlushAt or an explicit sentinel.

## Reproduction Assessment

Yes, at source level. An above-threshold entry with compactionCount: 0 and memoryFlushCompactionCount: 0 reaches the equality helper, returns true, and makes shouldRunMemoryFlush skip the flush.

## Solution Assessment

Yes, the current best path is the linked memoryFlushAt-based disambiguation rather than a blanket compactionCount === 0 special case, because a real cycle-0 flush can also store 0.
