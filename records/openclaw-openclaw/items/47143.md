## Summary

Keep open. Current main and the latest release still treat a persisted `compactionCount: 0` plus `memoryFlushCompactionCount: 0` row as already flushed, so the reported source-backed skip remains possible.

## What This Changes

The issue asks OpenClaw to allow `memoryFlush` for never-flushed sessions whose persisted flush and compaction counters are both zero.

## Best Possible Solution

Define a backward-compatible memory-flush state contract that distinguishes legacy or never-flushed 0/0 rows from recorded cycle-0 flushes, likely using `memoryFlushAt` or an explicit sentinel, with focused regression coverage.

## Reproduction Assessment

Yes, at source level. An above-threshold entry with `totalTokensFresh: true`, `compactionCount: 0`, and `memoryFlushCompactionCount: 0` reaches the helper, returns true, and causes `shouldRunMemoryFlush` to skip.

## Solution Assessment

No current fix is present. The safe repair should not special-case all `compactionCount: 0` rows; it needs to preserve duplicate-flush prevention for a real recorded cycle-0 flush.
