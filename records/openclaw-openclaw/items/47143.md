## What This Changes

The issue reports that memoryFlush is skipped for never-compacted sessions when `compactionCount` and `memoryFlushCompactionCount` are both `0`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83015 or a narrow equivalent, then close this issue after the fix reaches main while tracking broader context-budget bypasses separately.

## Reproduction Assessment

Yes. Current main's helper returns true for a numeric `0` matching the defaulted compaction count of `0`, and the runtime uses that helper to skip memory flush; I did not run tests because this review is read-only.

## Solution Assessment

Yes. The linked PR's `memoryFlushAt` discriminator is the narrowest visible fix because it preserves real cycle-zero flush deduplication while letting legacy or never-flushed 0/0 rows proceed.
