## What This Changes

The issue asks for memoryFlush to run for never-compacted, never-flushed sessions where both counters are 0 instead of treating them as already flushed.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83015, or an equivalent memoryFlushAt-aware fix that preserves once-per-cycle dedupe while allowing legacy or never-flushed 0/0 rows to flush.

## Reproduction Assessment

Yes, at source level. An above-threshold entry with compactionCount: 0 and memoryFlushCompactionCount: 0 reaches the equality helper, returns true for already-flushed, and suppresses memoryFlush.

## Solution Assessment

Yes. The narrow maintainable fix is to distinguish legacy or never-flushed 0/0 rows from real cycle-0 flush records, likely using memoryFlushAt as in the linked PR, while leaving broader preflight budget work separate.
