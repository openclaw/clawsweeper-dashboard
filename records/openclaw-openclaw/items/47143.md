## What This Changes

The issue requests that memoryFlush allow a first flush for never-flushed sessions whose `compactionCount` and `memoryFlushCompactionCount` are both 0.

## Best Possible Solution

Review and land or close the linked implementation PR; this issue should close only after the guard fix reaches main or maintainers choose another canonical resolution.

## Reproduction Assessment

Yes, source-level: current main's helper still defaults missing compaction count to 0 and compares it directly to `memoryFlushCompactionCount`, so a 0/0 entry is treated as already flushed. I did not run tests because this review is read-only.

## Solution Assessment

Yes. The linked PR's `memoryFlushAt` discriminator is the narrowest maintainable repair because it distinguishes real cycle-0 flushes from never-flushed 0/0 rows while preserving once-per-cycle dedupe.
