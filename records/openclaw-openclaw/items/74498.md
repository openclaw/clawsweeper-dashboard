## What This Changes

The PR adds Control UI committed-stream prefix tracking and tests so post-tool live chat deltas do not replay text already committed above tool cards.

## Best Possible Solution

Land the UI-side projection after maintainer validation, preserving monotonic full-snapshot chat deltas for gateway, ACP, TUI, and generic subscribers.

## Reproduction Assessment

Yes. A Control UI run with assistant text, then a tool call, then more assistant text exercises the issue because current main commits the first text into `chatStreamSegments` and then renders the next full-snapshot delta as the active stream.

## Solution Assessment

Yes. The UI-only offset is the narrowest maintainable fix I found because the earlier gateway-side direction in #54374 would have changed the full-snapshot contract used by ACP and TUI subscribers.
