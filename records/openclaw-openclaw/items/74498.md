## Summary

Keep open: current main still has the Control UI duplicate-render path, and this PR's UI-only offset/slicing approach matches the existing full-snapshot gateway contract without an identified blocking correctness or security issue.

## What This Changes

The branch adds Control UI committed-stream prefix tracking, reset coverage, regression tests, and a changelog entry so post-tool live chat text is sliced from full-snapshot gateway deltas.

## Best Possible Solution

Land the UI-side projection after normal maintainer validation, preserving monotonic gateway chat deltas for ACP/TUI while fixing Control UI duplicate live rendering.

## Reproduction Assessment

Yes. On current main, a Control UI run with assistant text, then a tool start, then a full-snapshot post-tool delta commits the pre-tool text into `chatStreamSegments` and then renders the same prefix again from `chatStream`.

## Solution Assessment

Yes. The UI-only offset is the narrowest maintainable fix I found because the earlier gateway-side #54374 direction would change the full-snapshot contract used by ACP, TUI, and generic WebSocket consumers.
