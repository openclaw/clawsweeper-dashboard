## Summary

Keep open: current main still lacks the broad-find timeout cap, and this PR is not superseded, but the proposed branch has parser bugs, no after-fix real behavior proof, and a maintainer policy question around the hardcoded 20s exec cap.

## What This Changes

The PR adds a 20s timeout cap for broad `find` scans rooted at `$HOME` or `/`, parser tests, and prompt guidance to prefer workspace-scoped discovery.

## Best Possible Solution

If maintainers want this policy, land a corrected guard that evaluates every `find` start path, resolves common shell home forms, exempts only truly scoped workspace searches, documents the behavior, and includes redacted real exec proof.

## Reproduction Assessment

Yes at source level: current main lacks the guard and broad `find $HOME` or `find /` commands without `params.timeout` inherit the normal exec timeout. I did not run a live broad home/root scan in this read-only review.

## Solution Assessment

No, not as currently proposed. A targeted cap is plausible, but this implementation needs parser fixes, real after-fix proof, and maintainer acceptance of the timeout policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Inspect every find start path:** `src/agents/bash-tools.exec.ts:344-345`
  - body: `find` accepts multiple start paths, but the parser records only the first path-like token. From a workspace, `find . / -type f` records `.` and can return uncapped as workspace-scoped while still traversing `/`; collect all start paths and cap when any resolved root is broad.
  - confidence: 0.9
- **[P2] Resolve braced HOME roots:** `src/agents/bash-tools.exec.ts:283-289`
  - body: The resolver handles `$HOME` and `$HOME/...` but not `${HOME}` or `${HOME}/...`, which the shell expands to the same directory before `find` runs. As written, `find ${HOME} ...` bypasses the intended cap.
  - confidence: 0.86
- **[P2] Classify home relative finds before workspace exemption:** `src/agents/bash-tools.exec.ts:355-359`
  - body: When `workdir` is the user's home directory, `find . ...` resolves to home and returns uncapped through the workspace exemption before the broad-root check. That misses the home scan this guard is meant to limit.
  - confidence: 0.91
