## Summary

Keep open: the central guard is not on current main, and this PR still has merge conflicts, missing real behavior proof, and parser false negatives that would let broad scans bypass the proposed cap.

## What This Changes

Adds a 20s timeout cap and warning for broad exec `find` scans over `$HOME` or `/`, plus system-prompt guidance and focused tests.

## Best Possible Solution

Land a rebased exec guard that evaluates all `find` start paths, handles common shell home forms, preserves explicit operator intent, and includes redacted real exec proof.

## Reproduction Assessment

Yes at source level: current main has no broad-find classifier and still derives exec timeout from the explicit/default timeout path. I did not run a live `$HOME` or `/` scan in this read-only review.

## Solution Assessment

No, not as currently proposed. A targeted cap is plausible, but this branch needs parser fixes, a rebase, real after-fix proof, and maintainer agreement on the 20s timeout policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Inspect every find start path:** `src/agents/bash-tools.exec.ts:344-345`
  - body: `find` accepts multiple starting paths, but this parser stores only the first path-like token. From a workspace, `find . / -type f` records `.` and returns uncapped through the workspace exemption while still traversing `/`; collect all start paths and cap when any resolved root is broad.
  - confidence: 0.9
- **[P2] Handle braced HOME roots:** `src/agents/bash-tools.exec.ts:283-289`
  - body: The resolver handles `$HOME` and `$HOME/...` but not `${HOME}` or `${HOME}/...`, which the shell expands to the same home directory before `find` runs. As written, `find ${HOME} ...` is resolved as a relative literal and bypasses the intended cap.
  - confidence: 0.86
- **[P2] Classify broad roots before workspace exemptions:** `src/agents/bash-tools.exec.ts:355-359`
  - body: When `workdir` is the user's home directory, `find . ...` resolves to home and returns uncapped through the workspace exemption before broad-root classification. That misses the home scan this guard is meant to limit.
  - confidence: 0.91
