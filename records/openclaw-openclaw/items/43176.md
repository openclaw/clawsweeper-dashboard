## Summary

Keep open. Current main still lacks a broad-find timeout guard, and the PR branch remains useful but incorrect because the classifier misses common broad-root forms and is applied after some exec host paths have already planned their timeouts.

## What This Changes

The PR adds a 20-second timeout classifier for unexplicit broad shell `find` scans, a new guard test file, and system-prompt guidance to prefer scoped workspace discovery.

## Best Possible Solution

Keep the PR open and request changes: the end state should be a shared broad-`find` timeout classifier applied before local, node, and gateway approval-follow-up timeout planning, with coverage for all supported root forms and traversal-limiting escape hatches.

## Reproduction Assessment

Yes. Source inspection against the PR head gives high-confidence reproductions: `find .` from the home directory, `find . / -type f` from a workspace, `find ${HOME} ...`, and node or gateway approval-follow-up execution can bypass the intended cap.

## Solution Assessment

No. The direction is reasonable, but this implementation is too narrow and too late in the execution flow; the classifier should feed all exec timeout planning before host-specific dispatch.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Check broad roots before exempting the workdir:** `src/agents/bash-tools.exec.ts:355-360`
  - body: The workspace exemption runs before the broad-root check, so `find . ...` with `workdir` set to the home directory resolves to the workdir and returns uncapped. That is still a broad home-directory scan, which is the main case this guard is meant to limit.
  - confidence: 0.9
- **[P2] Inspect every find start path before skipping the cap:** `src/agents/bash-tools.exec.ts:344-345`
  - body: `find` accepts multiple start paths, but the parser stores only the first path-like token. A command such as `find . / -type f` from a workspace can be treated as workspace-scoped even though the same invocation also walks `/` with the full default timeout.
  - confidence: 0.91
- **[P2] Apply the guard to all exec host paths:** `src/agents/bash-tools.exec.ts:682-686`
  - body: This call site only affects the later local `runExecProcess` path. `host=node` returns before this block, and gateway approval follow-up execution is planned before this cap, so broad scans on those surfaces can still inherit the default timeout.
  - confidence: 0.88
- **[P2] Treat braced HOME as a broad find root:** `src/agents/bash-tools.exec.ts:283-287`
  - body: The resolver handles `$HOME` but not `${HOME}`. In a shell command, `find ${HOME} ...` expands to the home directory, but this classifier resolves it as a literal relative path and skips the 20-second cap.
  - confidence: 0.86
