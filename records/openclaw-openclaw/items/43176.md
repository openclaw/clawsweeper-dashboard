- **[P2] Evaluate every find start path:** `src/agents/bash-tools.exec.ts:344-345`
  - body: The parser stores only the first path-like token. For `find . / -type f` from a workspace, `rootToken` stays `.` and the workspace exemption returns before `/` is considered, so a broad root still inherits the full timeout. Collect all start paths and cap when any resolves to `$HOME` or `/`.
  - confidence: 0.91
- **[P2] Handle braced HOME roots:** `src/agents/bash-tools.exec.ts:283-287`
  - body: `resolveFindRootToken` recognizes `$HOME` but not `${HOME}`. Bash expands `find ${HOME} ...` to the same broad root, but this guard treats it as a relative literal under `workdir` and skips the cap. Add `${HOME}` and `${HOME}/...` coverage with tests.
  - confidence: 0.9
