## Summary

Keep this PR open. Current main still lacks the broad-find timeout cap, and the PR is useful but incomplete: its classifier is only applied to the local exec path and still misses common broad traversal forms that the proposed guard is supposed to catch.

## What This Changes

The PR adds a local exec broad-find timeout helper, regression tests, and system-prompt guidance to cap unexplicit broad `find` scans at 20 seconds.

## Best Possible Solution

Keep the PR open and request changes. The maintainable end state is a shared broad-`find` timeout classifier applied before all exec host paths, parsing every start path, resolving relative roots against the command workdir, supporting `~`, `$HOME`, and `${HOME}`, preserving explicit timeout intent, honoring only real traversal-limiting escape hatches, and covering the behavior with focused local, gateway, node, prompt, and docs tests.

## Reproduction Assessment

Yes. Code inspection gives a high-confidence reproduction path: examples like `find .` with workdir set to `$HOME`, `find . / -type f` from a workspace, `find ${HOME} ...`, and broad node or gateway approval-follow-up runs all bypass the proposed cap.

## Solution Assessment

No. The direction is good, but this implementation is not the best fix yet because it applies the guard too late and too narrowly; a shared classifier needs to run consistently before local, gateway follow-up, and node execution timeout planning.

Full review comments:

- **[P2] Check broad roots before exempting the workdir:** `src/agents/bash-tools.exec.ts:355-360`
  - body: The workspace exemption runs before the broad-root check, so `find . ...` from a `$HOME` workdir resolves to the workdir and returns uncapped. That is still a broad home-directory scan, which is the main case this guard is meant to limit.
  - confidence: 0.9
- **[P2] Inspect every find start path before skipping the cap:** `src/agents/bash-tools.exec.ts:344-345`
  - body: The parser stores only the first path-like token. A command such as `find . / -type f` from a workspace records `.` and returns as workspace-scoped, even though the same invocation also walks `/` with the full default timeout.
  - confidence: 0.91
- **[P2] Apply the guard to all exec host paths:** `src/agents/bash-tools.exec.ts:682-686`
  - body: This call site only affects the local `runExecProcess` path. `host=node` returns before this block, and approval follow-up execution runs later in the gateway helper, so broad scans on those surfaces still inherit the default timeout.
  - confidence: 0.88
- **[P2] Treat braced HOME as a broad find root:** `src/agents/bash-tools.exec.ts:296-304`
  - body: The resolver handles `$HOME` but not `${HOME}`. In a shell command, `find ${HOME} ...` expands to the home directory, but this classifier treats it as a relative token and skips the 20s cap.
  - confidence: 0.86
