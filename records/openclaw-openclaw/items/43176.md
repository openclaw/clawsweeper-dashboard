## Summary

Keep open: current main still has no broad-find timeout cap, the related heavy-command work is complementary rather than a replacement, and this PR remains useful but has concrete parser/dispatch blockers plus missing real behavior proof before merge.

## What This Changes

The PR adds a 20-second cap for broad `$HOME` or `/` shell `find` scans without explicit exec timeouts, tests for the classifier, and system-prompt guidance to prefer scoped discovery.

## Best Possible Solution

Keep the broad-find protection direction, but implement timeout classification once in shared exec timeout planning before local, node, and gateway approval-follow-up dispatch, with parser coverage for every start path and shell HOME form.

## Reproduction Assessment

Yes. Source inspection of the PR head and current main gives high-confidence bypass paths: `find .` from `$HOME`, `find . / -type f`, `${HOME}` roots, and node or gateway approval-follow-up executions.

## Solution Assessment

No. The direction is useful, but this implementation is too narrow; the maintainable fix is a shared classifier that evaluates all `find` start paths before every exec host timeout is planned.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Check broad roots before exempting the workdir:** `src/agents/bash-tools.exec.ts:355-360`
  - body: The workspace exemption runs before checking whether the resolved root is `$HOME`, so `find . ...` with `workdir` equal to the home directory returns uncapped even though it is the broad home scan this guard is meant to limit.
  - confidence: 0.9
- **[P2] Inspect every find start path before skipping the cap:** `src/agents/bash-tools.exec.ts:344-345`
  - body: `find` accepts multiple start paths, but the parser keeps only the first path-like token. A command like `find . / -type f` from a workspace can be treated as scoped while it also walks `/` with the default timeout.
  - confidence: 0.91
- **[P2] Plan the capped timeout before host dispatch:** `src/agents/bash-tools.exec.ts:682-686`
  - body: The classifier is called only after the `host=node` return and after gateway approval processing can schedule a follow-up run, so node and approval follow-up executions can still inherit the uncapped default timeout.
  - confidence: 0.88
- **[P2] Treat braced HOME forms as home roots:** `src/agents/bash-tools.exec.ts:283-287`
  - body: The resolver handles `$HOME` but not `${HOME}`. Bash expands both forms to the same directory, so `find ${HOME} ...` skips the cap even though it scans the home directory.
  - confidence: 0.86
