## Summary

Keep open. Current main still lacks any broad-find timeout cap, and the PR’s direction is useful, but the proposed guard is not merge-ready because common broad roots and host execution paths can still bypass it.

## What This Changes

The PR adds a 20-second timeout classifier for unexplicit broad `find` commands, a new broad-find guard test file, and system-prompt guidance to prefer scoped workspace discovery.

## Best Possible Solution

Keep the PR open and request changes. The maintainable end state is a shared broad-`find` timeout classifier applied before local, gateway approval-followup, and node execution timeout planning; it should parse every start path, resolve relative roots against the command workdir, support `~`, `$HOME`, and `${HOME}`, preserve explicit timeout intent, honor only real traversal-limiting escape hatches, and add focused coverage for all host paths.

## Reproduction Assessment

Yes. Code inspection gives high-confidence reproductions against the PR head: `find .` from `$HOME`, `find . / -type f` from a workspace, `find ${HOME} ...`, and `host=node` or gateway approval-followup runs all bypass the intended cap.

## Solution Assessment

No. The product direction is reasonable, but this implementation is too narrow and too late in the execution flow; a shared classifier needs to feed all exec timeout planning before host-specific dispatch.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Check broad roots before exempting the workdir:** `src/agents/bash-tools.exec.ts:355-360`
  - body: The workspace exemption runs before the broad-root check, so `find . ...` with `workdir` set to `$HOME` resolves to the workdir and returns uncapped. That is still a broad home-directory scan, which is the main case this guard is meant to limit.
  - confidence: 0.9
- **[P2] Inspect every find start path before skipping the cap:** `src/agents/bash-tools.exec.ts:344-345`
  - body: The parser stores only the first path-like token. A command such as `find . / -type f` from a workspace records `.` and can return as workspace-scoped even though the same invocation also walks `/` with the full default timeout.
  - confidence: 0.91
- **[P2] Apply the guard to all exec host paths:** `src/agents/bash-tools.exec.ts:682-686`
  - body: This call site only affects the later local `runExecProcess` path. `host=node` returns before this block, and gateway approval follow-up execution is planned before this cap, so broad scans on those surfaces can still inherit the default timeout.
  - confidence: 0.88
- **[P2] Treat braced HOME as a broad find root:** `src/agents/bash-tools.exec.ts:296-304`
  - body: The resolver handles `$HOME` but not `${HOME}`. In a shell command, `find ${HOME} ...` expands to the home directory, but this classifier treats it as a non-special token and skips the 20s cap.
  - confidence: 0.86
