## Summary

Keep open: current main still lacks a broad-find exec cap, but this PR is not merge-ready because the PR head misses important broad-root cases and the external real-behavior proof gate is still missing. The related heavy-exec issue and closed unmerged PR are complementary, not a replacement for this narrower find-scan guard.

## What This Changes

The PR adds a 20s cap and warning for broad home/root `find` exec commands without explicit timeout, associated parser tests, and system-prompt guidance to prefer scoped searches.

## Best Possible Solution

Land a corrected, policy-approved broad-find guard that evaluates every start path, handles common home-expansion forms, applies workspace exemptions after broad-root classification, and includes after-fix real exec proof.

## Reproduction Assessment

Yes at source level: current main derives exec timeout from explicit/default timeout only and has no broad-find classifier. I did not run a live `$HOME` or `/` scan in this read-only review.

## Solution Assessment

No, not as currently proposed. A targeted cap is plausible, but this implementation needs parser fixes, real after-fix proof, and maintainer acceptance of the timeout policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Inspect every find start path:** `src/agents/bash-tools.exec.ts:344-345`
  - body: `find` accepts multiple start paths, but the parser records only the first path-like token. From a workspace, `find . / -type f` records `.` and can return uncapped as workspace-scoped while still traversing `/`; collect all start paths and cap when any resolved root is broad.
  - confidence: 0.9
- **[P2] Resolve braced HOME roots:** `src/agents/bash-tools.exec.ts:283-289`
  - body: The resolver handles `$HOME` and `$HOME/...` but not `${HOME}` or `${HOME}/...`, which the shell expands to the same directory before `find` runs. As written, `find ${HOME} ...` bypasses the intended cap.
  - confidence: 0.86
- **[P2] Classify home-relative roots before workspace exemption:** `src/agents/bash-tools.exec.ts:355-359`
  - body: When `workdir` is the user's home directory, `find . ...` resolves to home and returns uncapped through the workspace exemption before broad-root classification. That misses the home scan this guard is meant to limit.
  - confidence: 0.91
