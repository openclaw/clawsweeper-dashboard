## Summary

Keep open: current main still has no broad-find cap, and the PR head leaves concrete bypasses in parser classification and host dispatch that should block merge.

## What This Changes

The PR adds a 20-second cap for broad home/root shell `find` scans without an explicit timeout, classifier tests, and system-prompt guidance to prefer scoped searches.

## Best Possible Solution

Keep the PR open, rebase it onto current main, and move the broad-find classifier into shared exec timeout planning used before local, node, and gateway approval-follow-up dispatch.

## Reproduction Assessment

Yes. Source inspection of the PR head gives high-confidence bypass cases: `find .` from the home directory, `find . / -type f`, `find ${HOME} ...`, and host=node or gateway approval-follow-up execution.

## Solution Assessment

No. The direction is good, but this implementation is too narrow; the safer fix is a shared classifier that evaluates every start path before all host-specific timeout planning.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Check broad roots before exempting the workdir:** `src/agents/bash-tools.exec.ts:355-360`
  - body: The workspace exemption runs before the broad-root check, so `find . ...` with `workdir` set to the home directory returns uncapped. That is still a broad home-directory scan, which is the main case this guard is meant to limit.
  - confidence: 0.9
- **[P2] Inspect every find start path before skipping the cap:** `src/agents/bash-tools.exec.ts:344-345`
  - body: `find` accepts multiple start paths, but this parser stores only the first path-like token. `find . / -type f` from a workspace can therefore be treated as scoped while it also walks `/` with the full default timeout.
  - confidence: 0.91
- **[P2] Apply the guard before all exec host dispatch paths:** `src/agents/bash-tools.exec.ts:682-686`
  - body: The classifier is called after the `host=node` return and after gateway approval processing starts, so node runs and approval follow-up runs can still inherit the uncapped default timeout.
  - confidence: 0.88
- **[P2] Treat braced HOME as a broad find root:** `src/agents/bash-tools.exec.ts:283-287`
  - body: The resolver handles `$HOME` but not `${HOME}`. A shell expands `find ${HOME} ...` to the home directory, while this classifier treats it as a literal relative path and skips the cap.
  - confidence: 0.86
