## Summary

Keep open: current main still lacks a broad-find cap, so the central idea is not implemented, but the PR cannot merge as-is because its parser misses or over-exempts broad scans and no after-fix real behavior proof is supplied.

## What This Changes

The PR adds a 20s timeout cap for broad home/root `find` exec scans without explicit timeout, plus guard tests and system-prompt guidance.

## Best Possible Solution

Land a corrected, policy-approved broad-find guard that evaluates every start path, handles common shell home forms, applies workspace exemptions only after broad-root classification, and includes real exec proof.

## Reproduction Assessment

Yes at source level: current main computes exec timeout from only an explicit/default timeout and has no broad-find classifier. I did not run a live broad home/root scan in this read-only review.

## Solution Assessment

No, not as currently proposed. A targeted cap is plausible, but this implementation needs parser fixes, real after-fix proof, and maintainer acceptance of the timeout policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Inspect every find start path:** `src/agents/bash-tools.exec.ts:344-345`
  - body: `find` accepts multiple start paths, but the parser records only the first path-like token. From a workspace, `find . / -type f` records `.` and can return uncapped as workspace-scoped while still traversing `/`; collect all start paths and cap when any resolved root is broad.
  - confidence: 0.9
- **[P2] Resolve braced HOME roots:** `src/agents/bash-tools.exec.ts:283-289`
  - body: The resolver handles `$HOME` and `$HOME/...` but not `${HOME}` or `${HOME}/...`, which the shell expands to the same directory before `find` runs. As written, `find ${HOME} ...` bypasses the intended cap.
  - confidence: 0.86
- **[P2] Classify home-relative roots before workspace exemption:** `src/agents/bash-tools.exec.ts:355-359`
  - body: When `workdir` is the user's home directory, `find . ...` resolves to home and returns uncapped through the workspace exemption before the broad-root check. That misses the home scan this guard is meant to limit.
  - confidence: 0.91
