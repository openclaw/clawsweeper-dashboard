## Summary

Keep open: current main still has no broad-find timeout guard, and the PR head leaves concrete bypasses in parser coverage and host dispatch paths that would keep broad scans on the default exec budget.

## What This Changes

The PR adds a 20-second timeout cap for broad `$HOME` or `/` shell `find` scans without explicit timeouts, plus classifier tests and system-prompt guidance for scoped discovery.

## Best Possible Solution

Keep the broad-find protection, but move timeout classification into shared exec timeout planning before local, node, and gateway approval-follow-up dispatch, with parser coverage for every start path and shell HOME form.

## Reproduction Assessment

Yes. Source inspection of the PR head gives high-confidence bypass cases: `find .` from the home directory, `find . / -type f` from a workspace, `find ${HOME} ...`, and node or gateway approval-follow-up execution paths.

## Solution Assessment

No. The direction is useful, but this implementation is too narrow; the maintainable fix is a shared classifier that evaluates all start paths before every exec host timeout is planned.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Check broad roots before exempting the workdir:** `src/agents/bash-tools.exec.ts:355-360`
  - body: The workspace exemption runs before checking whether the resolved root is `$HOME`, so `find . ...` with `workdir` equal to the home directory returns uncapped. That is the broad home scan this guard is meant to limit; check home/root before treating a workdir root as safe.
  - confidence: 0.9
- **[P2] Inspect every find start path before skipping the cap:** `src/agents/bash-tools.exec.ts:344-345`
  - body: `find` accepts multiple start paths, but this parser keeps only the first path-like token. `find . / -type f` from a workspace can therefore be treated as scoped while it also walks `/` with the default timeout.
  - confidence: 0.91
- **[P2] Apply the guard before all exec host dispatch paths:** `src/agents/bash-tools.exec.ts:682-686`
  - body: The classifier is called after the `host=node` return and after gateway approval processing can schedule a follow-up run, so node and approval follow-up executions can still inherit the uncapped default timeout. Plan the effective timeout before dispatching into each host path.
  - confidence: 0.88
- **[P2] Treat braced HOME as a broad find root:** `src/agents/bash-tools.exec.ts:283-287`
  - body: The resolver handles `$HOME` but not `${HOME}`. A shell expands `find ${HOME} ...` to the home directory, while this classifier treats it as a relative token and skips the cap.
  - confidence: 0.86
