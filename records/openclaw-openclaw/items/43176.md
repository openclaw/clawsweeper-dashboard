## Summary

Keep open: current main still lacks a broad-find exec cap, but this PR has blocking parser gaps, no contributor real behavior proof, and needs maintainer acceptance of the new hardcoded exec timeout policy.

## What This Changes

The branch adds a 20s cap for broad home/root `find` exec commands without an explicit tool timeout, plus parser tests and system-prompt guidance to prefer scoped discovery.

## Best Possible Solution

If maintainers want this policy, land a focused exec guard that parses every traversal root, handles common shell home forms, respects real traversal constraints, documents the timeout exception, and includes redacted real exec proof.

## Reproduction Assessment

Yes at source level: current main passes `explicitTimeoutSec ?? defaultTimeoutSec` into `runExecProcess`, so a broad `find $HOME` command without a tool timeout inherits the normal run budget. I did not run a live broad home/root scan in this read-only review.

## Solution Assessment

No, not as currently proposed. A targeted broad-find cap is plausible, but this implementation needs parser fixes, real after-fix proof, and maintainer acceptance of the exec timeout policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Cap home-rooted relative finds before workspace exemption:** `src/agents/bash-tools.exec.ts:355-359`
  - body: When `workdir` is the user's home directory, `find . ...` resolves to the workdir and returns uncapped before broad-root classification. That misses the home scan this guard is meant to limit; classify home/root before the workspace-scoped exemption, or avoid exempting home.
  - confidence: 0.91
- **[P2] Inspect every find start path:** `src/agents/bash-tools.exec.ts:344-345`
  - body: `find` accepts multiple start paths, but this parser keeps only the first path-like token. From a workspace, `find . / -type f` records `.` and returns uncapped as workspace-scoped while still traversing `/`; collect all start paths and cap when any resolved root is broad.
  - confidence: 0.9
- **[P2] Resolve ${HOME} as a home root:** `src/agents/bash-tools.exec.ts:283-289`
  - body: The root checks handle `$HOME` and `$HOME/...` but not `${HOME}` or `${HOME}/...`, which the shell expands to the same directory before `find` runs. As written, `find ${HOME} ...` is treated as a literal relative path and bypasses the intended cap.
  - confidence: 0.86
