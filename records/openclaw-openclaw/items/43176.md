## Summary

Keep open. Current main still lacks the requested broad-`find` cap, and the PR’s latest guard has source-verifiable parser gaps plus missing after-fix real behavior proof.

## What This Changes

The PR adds an exec-timeout guard for broad home/root `find` commands, focused tests, and system-prompt guidance to prefer scoped workspace searches.

## Best Possible Solution

Keep the narrow guard direction, but fix the parser to evaluate all start paths, classify home/root before workspace exemptions, support `${HOME}` forms, and add redacted terminal or log proof from a real capped exec run.

## Reproduction Assessment

Yes at source level: current main has no `find`-specific timeout cap, and PR head’s guard logic can be traced to miss `find .` from home, `find . / ...`, and `${HOME}` forms. I did not run a live broad home/root scan in this read-only review.

## Solution Assessment

No. The cap is a plausible direction, but this implementation is not yet the narrow maintainable solution because common shell forms still bypass it and the runtime policy needs maintainer acceptance plus real proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Cap home-rooted relative finds before workspace exemption:** `src/agents/bash-tools.exec.ts:355-359`
  - body: When `workdir` is the user’s home directory, `find . ...` and `find $HOME ...` resolve to the workdir and return uncapped before `isBroadRoot` is checked. That misses the broad home scan this guard is meant to limit; classify home/root before exempting workspace-scoped paths, or avoid exempting home.
  - confidence: 0.91
- **[P2] Inspect every find start path:** `src/agents/bash-tools.exec.ts:344-345`
  - body: `find` accepts multiple start paths, but this parser keeps only the first path-like token. From a workspace, `find . / -type f` records `.` and returns uncapped as workspace-scoped while still traversing `/`; collect all start paths and cap when any resolved root is broad.
  - confidence: 0.9
- **[P2] Resolve ${HOME} as a home root:** `src/agents/bash-tools.exec.ts:296-304`
  - body: The resolver handles `$HOME` and `$HOME/...` but not `${HOME}` or `${HOME}/...`, which the shell expands to the same directory before `find` runs. As written, `find ${HOME} ...` is treated as a literal relative path and bypasses the intended cap.
  - confidence: 0.86
