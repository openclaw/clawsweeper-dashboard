## Summary

Keep open: the broad-find guard is a useful direction, but the PR is not ready because the patch leaves common timeout bypasses, is now merge-conflicting with current main, and has no contributor-supplied real behavior proof.

## What This Changes

The PR adds a 20-second timeout cap for broad shell `find` scans over `$HOME` or `/`, classifier tests, and system-prompt guidance to prefer scoped workspace discovery.

## Best Possible Solution

Implement shared exec timeout planning before local, gateway approval follow-up, and node dispatch; evaluate every `find` start root; handle common HOME shell forms; preserve explicit operator intent; and add focused tests plus redacted runtime proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main has no broad-find timeout planner, and PR head can be bypassed by multi-root, `${HOME}`, node, and gateway follow-up exec paths. No live capped-run proof was supplied.

## Solution Assessment

No. The guard direction is useful, but the maintainable fix is shared timeout planning before all exec dispatch paths plus more complete `find` root parsing, not the current late local-only classifier.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Inspect every find start path before skipping the cap:** `src/agents/bash-tools.exec.ts:344-345`
  - body: `find` accepts multiple start paths, but this parser records only the first path-like token. A command such as `find . / -type f` from a workspace can be treated as scoped while it also walks `/` with the default timeout.
  - confidence: 0.91
- **[P2] Check broad roots before exempting the workdir:** `src/agents/bash-tools.exec.ts:355-360`
  - body: The workspace exemption runs before checking whether the resolved root is `$HOME`, so `find . ...` with `workdir` equal to the home directory returns uncapped. That is the broad home scan this guard is meant to limit.
  - confidence: 0.9
- **[P2] Apply the guard before all exec host dispatch paths:** `src/agents/bash-tools.exec.ts:681-686`
  - body: The classifier runs after the `host === "node"` return and after gateway approval processing can schedule a follow-up run, so those executions can still inherit the uncapped default timeout.
  - confidence: 0.88
- **[P2] Treat braced HOME as a broad find root:** `src/agents/bash-tools.exec.ts:283-287`
  - body: The resolver handles `$HOME` but not `${HOME}`. A shell expands `find ${HOME} ...` to the home directory, while this classifier treats it as a relative token and skips the cap.
  - confidence: 0.86
