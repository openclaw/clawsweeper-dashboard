## Summary

Keep open: current main still lacks a broad-find timeout cap, and this PR is a useful direction, but the PR head leaves common timeout bypasses and still lacks contributor-supplied real behavior proof.

## What This Changes

The PR adds a 20-second timeout cap for broad shell `find` scans over `$HOME` or `/`, classifier tests, and system-prompt guidance for scoped discovery.

## Best Possible Solution

Keep the broad-find protection direction, but implement shared timeout planning before all exec dispatch paths, evaluate every `find` start root, handle common HOME shell forms, and require redacted runtime proof.

## Reproduction Assessment

Yes. Source inspection of the PR head gives high-confidence bypass paths for `find .` from `$HOME`, multi-root `find . / -type f`, `${HOME}` roots, and node or gateway approval-follow-up execution paths.

## Solution Assessment

No. The guard is useful, but this patch applies timeout classification too late for some dispatch paths and parses too little of `find`; shared timeout planning before dispatch is the narrower maintainable approach.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Check broad roots before exempting the workdir:** `src/agents/bash-tools.exec.ts:355-360`
  - body: The workspace exemption runs before checking whether the resolved root is `$HOME`, so `find . ...` with `workdir` equal to the home directory returns uncapped. That is the broad home scan this guard is meant to limit.
  - confidence: 0.9
- **[P2] Inspect every find start path before skipping the cap:** `src/agents/bash-tools.exec.ts:344-345`
  - body: `find` accepts multiple start paths, but this parser keeps only the first path-like token. `find . / -type f` from a workspace can therefore be treated as scoped while it also walks `/` with the default timeout.
  - confidence: 0.91
- **[P2] Apply the guard before all exec host dispatch paths:** `src/agents/bash-tools.exec.ts:682-686`
  - body: The classifier is called after the `host === "node"` return and after gateway approval processing can schedule a follow-up run, so those executions can still inherit the uncapped default timeout.
  - confidence: 0.88
- **[P2] Treat braced HOME as a broad find root:** `src/agents/bash-tools.exec.ts:283-287`
  - body: The resolver handles `$HOME` but not `${HOME}`. A shell expands `find ${HOME} ...` to the home directory, while this classifier treats it as a relative token and skips the cap.
  - confidence: 0.86
