## Summary

Keep open: current main and the latest release still lack the proposed broad-find guard, but this PR is not ready because the parser still misses supported broad `find` forms and the external PR lacks real behavior proof.

## What This Changes

The branch adds a broad `find` timeout classifier in `src/agents/bash-tools.exec.ts`, focused tests for it, and system-prompt guidance to avoid unscoped home/root `find` scans.

## Best Possible Solution

Land a rebased exec guard that parses every `find` start path, handles common shell home expansions, classifies broad roots before workspace exemptions, preserves explicit controls, and includes redacted real exec proof.

## Reproduction Assessment

Yes at source level: current main still sends exec runs through the explicit/default timeout path with no broad-find classifier. I did not run a live broad `$HOME` or `/` scan in this read-only review.

## Solution Assessment

No, not as currently proposed. A targeted cap is plausible, but this branch needs parser fixes, maintainer agreement on the runtime policy, and real after-fix proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Inspect every find start path:** `src/agents/bash-tools.exec.ts:344-345`
  - body: `find` accepts multiple starting paths, but this parser records only the first path-like token. From a workspace, `find . / -type f` records `.` and can return uncapped through the workspace exemption while still traversing `/`; collect all start paths and cap when any resolved root is broad.
  - confidence: 0.9
- **[P2] Handle braced HOME roots:** `src/agents/bash-tools.exec.ts:283-289`
  - body: The resolver handles `$HOME` and `$HOME/...` but not `${HOME}` or `${HOME}/...`, which the shell expands to the same home directory before `find` runs. As written, `find ${HOME} ...` is treated as a relative literal and bypasses the intended cap.
  - confidence: 0.86
- **[P2] Classify broad roots before workspace exemptions:** `src/agents/bash-tools.exec.ts:355-359`
  - body: When `workdir` is the user's home directory, `find . ...` resolves to home and returns uncapped through the workspace exemption before broad-root classification. That misses the home scan this guard is meant to limit.
  - confidence: 0.91
