## Summary

Keep open: current main still lacks the broad-find timeout guard, but this PR should not merge as-is because the parser misses valid broad `find` forms and the external contributor has not supplied real after-fix behavior proof.

## What This Changes

The PR adds a 20-second timeout cap for broad exec `find` scans over home/root paths, prompt guidance to prefer scoped searches, and targeted tests.

## Best Possible Solution

Land a rebased guard that evaluates every `find` start path, recognizes common home expansions, treats home/root as broad before workspace exemptions, preserves explicit operator controls, and includes redacted real exec proof.

## Reproduction Assessment

Yes at source level: current main has no broad-find classifier and still sends exec runs through the normal explicit/default timeout path. I did not run a live broad `$HOME` or `/` scan in this read-only review.

## Solution Assessment

No, not as currently proposed. A targeted cap is plausible, but this branch still misses valid broad `find` forms and needs maintainer agreement on the timeout policy plus real after-fix proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Inspect every find start path:** `src/agents/bash-tools.exec.ts:344-345`
  - body: `find` accepts multiple starting paths, but this parser records only the first path-like token. From a workspace, `find . / -type f` records `.` and returns uncapped through the workspace exemption while still traversing `/`; collect all start paths and cap when any resolved root is broad.
  - confidence: 0.9
- **[P2] Handle braced HOME roots:** `src/agents/bash-tools.exec.ts:283-289`
  - body: The resolver handles `$HOME` and `$HOME/...` but not `${HOME}` or `${HOME}/...`, which the shell expands to the same home directory before `find` runs. As written, `find ${HOME} ...` resolves as a relative literal and bypasses the intended cap.
  - confidence: 0.86
- **[P2] Classify broad roots before workspace exemptions:** `src/agents/bash-tools.exec.ts:355-359`
  - body: When `workdir` is the user's home directory, `find . ...` resolves to home and returns uncapped through the workspace exemption before broad-root classification. That misses the home scan this guard is meant to limit.
  - confidence: 0.91
