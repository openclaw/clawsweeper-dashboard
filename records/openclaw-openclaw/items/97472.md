## Summary

## What This Changes

The PR switches Mattermost draft preview truncation from raw `slice` to the SDK surrogate-safe slicer and adds a regression test for an emoji straddling the preview limit.

## Best Possible Solution

Land the focused Mattermost plugin fix after normal CI and maintainer checks, keeping surrogate-safe truncation localized to the draft preview normalizer and its regression test.

## Reproduction Assessment

Yes. Source inspection shows current main raw-slices at `maxChars - 3`; routing `"a".repeat(8) + "\u{1F600}" + "b".repeat(5)` through `createMattermostDraftStream` with `maxChars: 12` reaches that path and would keep a lone high surrogate before the ellipsis.

## Solution Assessment

Yes. Using the existing `sliceUtf16Safe` SDK helper at the existing Mattermost truncation point is the narrowest maintainable fix and preserves the current cap and ellipsis behavior for non-boundary input.
