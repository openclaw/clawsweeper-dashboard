## Summary

Keep open: current `main` still has the reported merge bug. Source inspection shows decoded request fields are spread after explicit CLI options, and the decoded object always includes `sessionKey` as `""` when the encoded payload omits it, so a direct `--session-key` can be erased before the required-session check.

## What This Changes

The issue asks OpenClaw to preserve explicitly supplied export-trajectory CLI options when an encoded request omits those fields.

## Best Possible Solution

Update the decoder or merge step so absent or blank encoded fields are not materialized as overriding properties, while preserving non-empty encoded values, then add a focused regression test for direct `sessionKey` plus a partial encoded request.

## Reproduction Assessment

Yes. A high-confidence source reproduction is `exportTrajectoryCommand({ sessionKey, requestJsonBase64: base64url({ output }) })`: the decoder produces `sessionKey: ""`, the later spread overwrites the explicit key, and the command follows the `--session-key is required` path.

## Solution Assessment

Yes. The narrow maintainable fix is to return only present non-empty decoded fields or to merge decoded values without overwriting defined direct options with `undefined` or empty placeholders.
