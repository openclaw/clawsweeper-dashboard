## Summary

Keep open: current main still has the reported option-merge bug. The decoder materializes absent encoded fields as own properties, including `sessionKey: ""`, and then spreads the decoded object after explicit CLI options, so a partial encoded request can erase a direct session key before the required-session check.

## What This Changes

The issue reports that a partial base64url export-trajectory request can overwrite an explicit CLI `sessionKey` with an empty decoded placeholder.

## Best Possible Solution

Update the decoder or merge step so absent or blank decoded fields do not overwrite already-defined direct CLI options, while preserving non-empty encoded values and adding a focused regression test.

## Reproduction Assessment

Yes. A high-confidence source reproduction is `exportTrajectoryCommand({ sessionKey, requestJsonBase64: base64url({ output }) })`: the decoded object contains `sessionKey: ""`, the later spread overwrites the direct key, and the command takes the `--session-key is required` path.

## Solution Assessment

Yes. The narrow maintainable fix is to return only present non-empty decoded fields or otherwise merge decoded values without replacing defined direct options with empty placeholders.
