## What This Changes

Prevent encoded trajectory export requests with absent or blank fields from overwriting explicit direct options such as `sessionKey`.

## Best Possible Solution

Review and land one focused linked PR that omits absent or blank decoded fields while preserving non-empty encoded-value precedence.

## Reproduction Assessment

Yes. A source-level reproduction is `exportTrajectoryCommand({ sessionKey, requestJsonBase64: base64url({ output }) })`, where decoding creates `sessionKey: ""` and the later spread overwrites the explicit key.

## Solution Assessment

Yes. The narrow maintainable fix is to omit absent or blank decoded fields, or merge decoded values without replacing already-defined direct options with empty placeholders.
