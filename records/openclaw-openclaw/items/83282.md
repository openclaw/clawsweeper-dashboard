## What This Changes

The issue reports that an encoded trajectory export request can overwrite explicit direct options such as `sessionKey` with empty decoded placeholders.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83308 or an equivalent focused fix that omits absent or blank decoded request fields while preserving intentional non-empty encoded-value precedence.

## Reproduction Assessment

Yes. A source-level reproduction is `exportTrajectoryCommand({ sessionKey, requestJsonBase64: base64url({ output }) })`, where the decoded object carries `sessionKey: ""` and the later spread overwrites the explicit key before the required-session check.

## Solution Assessment

Yes. The narrow maintainable fix is to omit absent or blank decoded fields, or otherwise merge decoded values without replacing already-defined direct options with empty placeholders.
