## What This Changes

Prevent partial encoded trajectory export requests from erasing directly supplied sessionKey/store options during option resolution.

## Best Possible Solution

Review and land one focused fix PR that preserves non-empty encoded-value precedence while preventing absent or blank decoded fields from overwriting direct options.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: call exportTrajectoryCommand with a direct sessionKey and requestJsonBase64 for JSON that only has output; decoding returns sessionKey: "" and the spread overwrites the direct key before the required-session check.

## Solution Assessment

Yes. The narrow maintainable fix is to omit absent or blank decoded fields, or otherwise merge decoded values without replacing already-defined direct options with empty placeholders.
