## What This Changes

The issue asks to preserve explicit CLI export-trajectory options when an encoded request omits those fields.

## Best Possible Solution

Land one focused fix PR that omits absent or blank decoded fields, preserves non-empty encoded overrides, and adds a regression test for direct `sessionKey` plus a partial encoded request.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: a direct `sessionKey` plus base64url JSON containing only another option decodes to `sessionKey: ""`, then the decoded spread overwrites the direct key before the required-session check.

## Solution Assessment

Yes. The narrow maintainable solution is to stop materializing absent or blank decoded fields, or otherwise merge decoded values without replacing already-defined direct options with placeholders.
