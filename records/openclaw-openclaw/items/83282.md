## Summary

Keep open because current `main` still has the reported source-level bug and live GitHub state shows an open closing PR for the same issue. The issue should close only after the linked fix PR lands or maintainers deliberately choose another resolution.

## What This Changes

The issue reports that a partial encoded `sessions export-trajectory` request can overwrite an explicitly supplied direct `sessionKey` with an empty placeholder.

## Best Possible Solution

Review and land the existing focused fix PR, preserving non-empty encoded-value precedence while preventing absent or blank decoded fields from overwriting explicit direct options.

## Reproduction Assessment

Yes. A high-confidence source reproduction is `exportTrajectoryCommand({ sessionKey, requestJsonBase64: base64url({ output }) })`, where the decoder returns `sessionKey: ""` and the later spread overwrites the explicit value before the required-session check.

## Solution Assessment

Yes. The narrow maintainable fix is to omit absent or blank decoded fields, or otherwise merge decoded values without replacing already-defined direct options with empty placeholders; the existing open PR is the right implementation thread.
