## What This Changes

The issue reports that a partial encoded trajectory export request can erase an explicitly supplied direct session key during option resolution.

## Best Possible Solution

Review and land one existing focused fix PR, preserving non-empty encoded-value precedence while preventing absent or blank decoded fields from overwriting explicit direct options.

## Reproduction Assessment

Yes. A high-confidence source reproduction is `exportTrajectoryCommand({ sessionKey, requestJsonBase64: base64url({ output }) })`, where decoding produces `sessionKey: ""` and the later spread overwrites the explicit value before the required-session check.

## Solution Assessment

Yes. The narrow maintainable solution is to omit absent or blank decoded fields, or merge decoded values without replacing already-defined direct options with empty placeholders.
