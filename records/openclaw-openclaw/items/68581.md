## Summary

Keep open. Current main and v2026.6.10 still implement Android node location as foreground-only, and the prior Always/background-location PR was closed because the permission and privacy model needs product/security approval before code review.

## What This Changes

The issue requests Android `location.enabledMode: always` support so background `location.get` can run through Android background location permission and a foreground service.

## Best Possible Solution

Keep this as the canonical Android Always/background location request, decide the privacy and product policy first, then implement the approved foreground-service path with flavor-aware permissions, focused Android tests, docs, and real-device proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set the Android `LocationHandler` foreground callback to false and call `handleLocationGet`; the current test asserts `LOCATION_BACKGROUND_UNAVAILABLE`.

## Solution Assessment

Unclear. A foreground-service-backed Always mode is plausible, but the best permanent shape depends on maintainer approval for Android background-location policy, build flavor handling, notification UX, and permission fallback.
