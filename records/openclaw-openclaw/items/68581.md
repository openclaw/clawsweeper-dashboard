## Summary

Keep open. Current `main` and latest release still implement Android node location as foreground-only, duplicate search points back to this issue as the canonical remaining request, and a prior implementation PR was closed pending explicit product/privacy/security approval.

## What This Changes

Request Android node support for `location.enabledMode: always` so background `location.get` can run through a location foreground service.

## Best Possible Solution

Make an explicit Android product/privacy/security decision first; if approved, implement Always mode with permission reconciliation, location foreground-service typing, notification UX, preserved While Using denial, focused Android tests, live-device proof, docs, and changelog coverage.

## Reproduction Assessment

Yes for the current limitation: source inspection and unit coverage show backgrounded Android `location.get` returns `LOCATION_BACKGROUND_UNAVAILABLE`, with no Always mode, background permission, or location foreground-service type on current `main`. I did not run a live Android device reproduction in this read-only review.

## Solution Assessment

Unclear until owners make the product/security decision. The foreground-service/background-permission direction matches Android's platform model, but the repo intentionally removed this capability for Play-policy scope and later closed a replacement PR pending explicit approval.
