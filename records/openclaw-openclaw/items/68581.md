Keep open. ## What This Changes

Request to add Android `location.enabledMode: always` support with background-location permission, a location foreground service, settings UI, tests, and docs.

## Best Possible Solution

Android product/security owners should explicitly approve or reject Always/background location, then implement the approved path with opt-in UI, OS permission handling, a location foreground service, docs, tests, and real Android proof.

## Reproduction Assessment

Yes, as a source-level reproduction: when the Android runtime foreground flag is false, `LocationHandler.handleLocationGet` returns `LOCATION_BACKGROUND_UNAVAILABLE`, and the current unit test asserts that behavior. I did not run a live Pixel repro in this read-only review.

## Solution Assessment

Unclear until maintainers decide the product/security posture. A foreground-service-backed Always mode is a plausible implementation shape, but the prior attempt was closed for owner approval, so automation should not pick the policy for them.
