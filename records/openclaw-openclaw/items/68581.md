## Summary

Keep open. Current main still implements Android node location as foreground-only, so the requested Always/background-location behavior is neither implemented nor superseded by the related metadata issue.

## What This Changes

Request Android node support for `location.enabledMode: always` by adding an Always settings mode, Android background-location permission flow, and foreground-service-backed background `location.get`.

## Best Possible Solution

Make an explicit Android product/privacy decision, then either document that Android remains foreground-only or implement `Always` as a first-class Android location mode with background-location permission handling, `foregroundServiceType="location"`, short-lived background fixes, preserved While Using denial, tests, and updated public docs.

## Reproduction Assessment

Yes. Source and tests provide a high-confidence reproduction path: with `isForeground = false`, `LocationHandler.handleLocationGet` returns `LOCATION_BACKGROUND_UNAVAILABLE`, and the settings/model code has no reachable Always state.

## Solution Assessment

Unclear. A foreground-service-backed episodic fix is Android-plausible, but it is not a routine cleanup because it expands background location collection and Google Play policy surface.
