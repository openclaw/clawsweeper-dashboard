Keep open. Current main and v2026.5.7 still implement Android node location as foreground-only, related duplicate reports point back to this issue as the canonical request, and a prior Always/background-location PR was closed pending explicit product and security approval.

## What This Changes

Request Android `location.enabledMode: always` support backed by background location permission and a location foreground service so background `nodes location get` can return a fix.

## Best Possible Solution
