## Summary

Keep open. Current main still implements Android node location as foreground-only: the Android model has no `Always` mode, legacy `always` values are migrated to `whileUsing`, the settings UI exposes only Off/While Using, and background `location.get` still returns `LOCATION_BACKGROUND_UNAVAILABLE`. The related closed #71015 covered location metadata delivery to agents and does not supersede this Android background-permission/foreground-service request.

## What This Changes

The issue requests Android support for `location.enabledMode: always` using background location permission and a foreground service so scheduled node location requests can succeed while the app is backgrounded.

## Best Possible Solution

Make an explicit Android product/privacy decision, then implement `Always` as a first-class Android location mode with background permission handling, a `foregroundServiceType="location"` foreground service for short-lived fixes, preserved `whileUsing` background denial, focused Android tests, docs, and changelog coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows the Android handler returns `LOCATION_BACKGROUND_UNAVAILABLE` whenever the app is backgrounded, and the model/UI/tests still only support foreground `whileUsing` behavior.

## Solution Assessment

Unclear. The proposed foreground-service direction is plausible for Android, but no implementation exists on main and the product/privacy tradeoff needs maintainer approval before a code path is selected.
