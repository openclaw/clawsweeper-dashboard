## Summary

Keep open. Current main and the latest release still implement Android node location as foreground-only, related duplicate reports point back to this issue as the canonical remaining request, and the prior implementation PR was closed pending explicit product/security approval for background location.

## What This Changes

Request Android node support for an `Always` location mode so background `location.get` can use Android background location permission plus a foreground service.

## Best Possible Solution

Make an explicit Android product/privacy/security decision first; if approved, implement Always mode with real OS grant reconciliation, location foreground-service typing, notification UX, preserved While Using denial, focused tests, live Android proof, docs, and changelog coverage.

## Reproduction Assessment

Yes for the current limitation: current-main source and unit coverage show backgrounded Android `location.get` returns `LOCATION_BACKGROUND_UNAVAILABLE`, with no Always mode or background-location service support. I did not run live Android device proof in this read-only review.

## Solution Assessment

Unclear until owners make the product/security decision. The foreground-service/background-permission direction matches Android’s platform model and the prior implementation attempt, but it must be approved and validated before a fix PR is safe.
