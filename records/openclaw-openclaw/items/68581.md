## Summary

Keep open. Current main still implements Android node location as foreground-only: there is no `Always` location mode, legacy `always` settings are migrated to `whileUsing`, the settings UI exposes only Off/While Using, and background `location.get` still returns `LOCATION_BACKGROUND_UNAVAILABLE`. The related closed #71015 requested location metadata delivery to agents and does not supersede this foreground-service/background-permission feature request.

## What This Changes

Add Android support for `location.enabledMode: always` so background `location.get` can use an opt-in background-location grant and foreground service instead of failing while the app is backgrounded.

## Best Possible Solution

Make an explicit Android product/privacy decision, then implement `Always` as a first-class Android location mode with background permission handling, a `foregroundServiceType="location"` foreground service for short-lived fixes, preserved `whileUsing` background denial, focused Android tests, and updated public docs.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: with `isForeground = false`, `LocationHandler.handleLocationGet` returns `LOCATION_BACKGROUND_UNAVAILABLE`, and `LocationHandlerTest` asserts that behavior even when fine/coarse location permissions are granted.

## Solution Assessment

No implemented solution exists on main. The proposed foreground-service path is plausible for Android, but the best next step is maintainer product/privacy approval before code changes because the feature introduces background location collection and Play-policy implications.
