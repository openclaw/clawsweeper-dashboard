## Summary

Keep open. Current main and the latest release still implement Android node location as foreground-only, the duplicate search points back to this issue as the canonical remaining request, and a prior implementation attempt was closed because background location needs explicit product/privacy/security approval before code review.

## What This Changes

The issue requests Android `location.enabledMode: always` support with background location permission, foreground-service location typing, settings UI, runtime handling, tests, and docs for background `location.get`.

## Best Possible Solution

Make an explicit Android product/privacy/security decision first; if approved, implement Always mode with permission reconciliation, location foreground-service typing, notification UX, preserved While Using denial, focused Android tests, live-device proof, docs, and changelog coverage.

## Reproduction Assessment

Yes for the current limitation: source inspection and unit coverage show backgrounded Android `location.get` returns `LOCATION_BACKGROUND_UNAVAILABLE`, with no Always mode, background permission, or location foreground-service type on current main. I did not run a live Android device reproduction in this read-only review.

## Solution Assessment

Unclear until owners make the product/security decision. The foreground-service/background-permission approach matches Android’s platform shape, but the repo intentionally removed this capability for Play-policy scope and later closed a replacement PR pending explicit approval.
