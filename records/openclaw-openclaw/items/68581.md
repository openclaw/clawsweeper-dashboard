## Summary

Keep open. Current main and the latest release still implement Android node location as foreground-only, the duplicate search points back to this issue as the canonical remaining request, and the prior implementation attempt was closed because background location needs explicit product/privacy/security approval before code review.

## What This Changes

Request Android node support for an `Always` location mode that can satisfy background `location.get` calls through Android background location permission and a location foreground service.

## Best Possible Solution

Make an explicit Android product/privacy/security decision first; if approved, implement Always mode with permission reconciliation, location foreground-service typing, notification UX, preserved While Using denial, focused Android tests, live-device proof, docs, and changelog coverage.

## Reproduction Assessment

Yes for the current limitation: source and unit coverage show backgrounded Android `location.get` returns `LOCATION_BACKGROUND_UNAVAILABLE`, with no Always mode, background permission, or location foreground-service type on current main. I did not run a live Android device reproduction in this read-only review.

## Solution Assessment

Unclear until owners make the product/security decision. The foreground-service/background-permission approach matches Android’s platform shape and the prior implementation attempt, but it must preserve Play flavor boundaries, actual OS grant checks, notification behavior, tests, and live proof.
