## Summary

Keep open. Current main and the latest release still make Android node location foreground-only, and this request is a security-sensitive product feature rather than a stale or already-implemented cleanup item.

## What This Changes

Request Android node support for `location.enabledMode: always` so `location.get` can return a fix while the app is backgrounded via background permission and a foreground service.

## Best Possible Solution

Android product and security owners should decide whether Always background location is an approved core capability, then implement it with explicit opt-in UI, OS permission handling, a location foreground service, docs, tests, and real Android proof.

## Reproduction Assessment

Yes from source inspection: when the Android runtime foreground flag is false, `LocationHandler` returns `LOCATION_BACKGROUND_UNAVAILABLE` before fetching location. I did not run a live Pixel repro in this read-only review.

## Solution Assessment

Unclear: a foreground-service-backed Always mode is plausible, but Android background location is privacy and policy sensitive, so the best fix needs owner approval before implementation.
