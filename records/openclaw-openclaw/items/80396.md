## Summary

Keep open. The warn-only goal is plausible, but this patch introduces a blocking browser-safety regression by importing the Node logger into the shared media parser used by Control UI, and external real behavior proof is still missing.

## What This Changes

The PR imports logger support into `src/media/parse.ts` to warn when fenced `MEDIA:` lines are skipped and adds parser tests for the warn-only behavior.

## Best Possible Solution

Keep `src/media/parse.ts` browser-safe, for example by injecting an optional warning callback from Node-only call sites or logging from a server/runtime wrapper, then require redacted real behavior proof before normal maintainer review; leave extraction-from-fences policy to https://github.com/openclaw/openclaw/issues/41966.

## Reproduction Assessment

Yes. Current-main source shows fenced lines are skipped before `MEDIA:` matching, and the parser test asserts a fenced `MEDIA:` line remains visible text; the PR regression is also source-reproducible because Control UI imports the parser and the new logger import pulls Node-only modules into that browser graph.

## Solution Assessment

No. Warn-only is a reasonable narrow direction, but the current implementation is not the best way because it adds a static Node logger dependency to a shared parser used by browser UI code; keep the diagnostic behind a browser-safe seam.

## Review Findings
