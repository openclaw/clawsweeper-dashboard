## What This Changes

The branch adds a log warning from `splitMediaFromOutput()` when a line-start `MEDIA:` directive appears inside a fenced code block, plus parser tests for warning and no-warning cases.

## Best Possible Solution

Keep the warning path browser-safe, for example with an optional warning callback or Node-only caller logging, then add redacted runtime proof that a fenced `MEDIA:` line emits the warning while extraction remains unchanged.

## Reproduction Assessment

Yes. Current source preserves fenced `MEDIA:` lines as text before media matching and the docs describe those wrapped directives as silently dropped, so a focused `splitMediaFromOutput()` call with a fenced `MEDIA:` line source-reproduces the missing warning path.

## Solution Assessment

No. The warning-only direction is narrow, but the current implementation puts a Node logger import in a shared parser used by browser UI; a browser-safe callback or Node-boundary logging seam is the safer fix.
