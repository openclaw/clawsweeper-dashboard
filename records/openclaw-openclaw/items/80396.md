## Summary

Keep open. The warn-only direction is plausible and matches the linked discussion, but this PR pulls the Node logger into a parser that is also imported by the browser Control UI, and the external contributor proof gate is still unmet.

## What This Changes

The PR adds a `logWarn()` when `splitMediaFromOutput()` skips fenced `MEDIA:` lines and adds parser tests for that warn-only behavior.

## Best Possible Solution

Emit the skipped-`MEDIA:` diagnostic through a browser-safe seam, such as an optional parser warning callback or server-only caller wrapper, then land it with parser/UI regression coverage and contributor real-run proof.

## Reproduction Assessment

Yes. Source inspection shows current main skips fenced `MEDIA:` lines before token matching, and the PR diff shows the proposed warning would add a Node logger dependency to a parser that is also imported by the Control UI; no tests were run in this read-only review.

## Solution Assessment

No. Warn-only is a reasonable direction, but importing `../logger.js` into the shared parser is not the narrowest maintainable implementation because that parser sits in a browser import graph; use a browser-safe warning seam or server-only wrapper instead.
