## Summary

Keep open. The PR addresses a real source-reproducible silent skip path, but the current implementation introduces a browser-facing dependency problem and the external contributor proof gate is still unmet.

## What This Changes

The PR adds a `logWarn()` call when `splitMediaFromOutput()` sees a fenced `MEDIA:` line and adds parser tests for the warn-only behavior.

## Best Possible Solution

Keep the warn-only diagnostic, but implement it through an injected callback or Node-owned call site so `src/media/parse.ts` remains browser-safe, then add focused parser/UI coverage and real behavior proof.

## Reproduction Assessment

Yes. Source inspection of current main shows fenced lines are skipped before `MEDIA:` matching, and existing parser coverage asserts that fenced `MEDIA:` remains visible text; I did not run tests because this is a read-only review.

## Solution Assessment

No. The warn-only direction is reasonable, but this implementation puts the Node logger dependency in a shared parser that the browser Control UI imports; the safer fix keeps logging injected or in Node-owned call sites.
