## Summary

Keep open: current main still skips fenced MEDIA directives silently, and this PR is the active linked fix for https://github.com/openclaw/openclaw/issues/41966. The final branch shape looks correct: it preserves inert fenced code examples, avoids a new config surface, keeps the browser-shared parser free of Node logger imports, and has sufficient real output proof.

## What This Changes

This PR adds a callback for fenced MEDIA-token skips in the shared media parser, wires it to logWarn() from reply directive parsing, and adds parser tests for fenced and unfenced MEDIA handling.

## Best Possible Solution

Land the callback-based warn-only implementation after maintainer approval and current-head checks, keeping fenced code inert and the shared parser browser-safe.

## Reproduction Assessment

Yes, at source level: current main keeps fenced MEDIA lines before directive extraction, and the existing parser test encodes that fenced MEDIA remains visible text rather than delivered media. I did not run tests because this is a read-only review.

## Solution Assessment

Yes. Warn-only through an optional callback is the narrowest maintainable fix I found because extracting from fences would mutate code examples, direct parser logging would violate the browser boundary, and docs-only would not make the failure observable in logs.
