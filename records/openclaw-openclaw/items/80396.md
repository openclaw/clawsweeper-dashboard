## What This Changes

The PR adds a `logWarn()` when `splitMediaFromOutput()` sees a line-start `MEDIA:` token inside a fenced code block and adds parser tests for the warning plus unchanged extraction behavior.

## Best Possible Solution

Expose skipped-fenced-media diagnostics without a static logger dependency in the browser-shared parser, then log from a Node-only boundary and include real redacted runtime proof.

## Reproduction Assessment

Yes. Current main visibly skips fenced `MEDIA:` lines in `splitMediaFromOutput()` without a warning path; a direct parser call with a fenced `MEDIA:` line exercises the source path.

## Solution Assessment

No. The warn-only behavior is a reasonable fix shape, but importing `../logger.js` from the browser-shared parser is not the safest implementation.
