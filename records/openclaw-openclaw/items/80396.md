## Summary

Keep open. The diagnostic is useful, but this patch imports the Node logger into a browser-shared parser, and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR adds a warning and parser tests for `MEDIA:` lines skipped inside fenced code blocks by `splitMediaFromOutput()`.

## Best Possible Solution

Keep `src/media/parse.ts` browser-safe by injecting a Node/runtime warning callback from server call sites, preserve the fenced-code extraction contract in tests, and merge only after redacted real-run proof is added.

## Reproduction Assessment

Yes. Current main source and tests show fenced `MEDIA:` lines are skipped before extraction, and the PR regression is source-reproducible from the Control UI import graph; I did not run tests because this review is read-only.

## Solution Assessment

No. Emitting a warning is a reasonable narrow fix, but importing the Node logger from a parser used by browser UI is not the maintainable way to do it.
