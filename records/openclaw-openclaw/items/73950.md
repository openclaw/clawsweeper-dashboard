## What This Changes

## Best Possible Solution

Land the shared native relay state fix after rebasing onto current main and including every relay state collection, especially `permissionAllowAlwaysApprovals`, with focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible: current main keeps relay state module-local and still throws `native hook relay not found` when the lookup misses the registered relay. The linked issue and PR comments add live logs, live-smoke evidence, and production-backport confirmation, but I did not run a fresh live Telegram/Codex scenario in this read-only review.

## Solution Assessment

No, not as-is. The process-global shared-state direction is the narrow maintainable fix, but the branch must be rebased and must include the current allow-always approval cache in the shared state before merge.
