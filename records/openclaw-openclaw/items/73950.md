## Summary

Keep open. This is a focused bug-fix PR for a source- and production-confirmed Codex native hook relay split-state failure, but it is currently conflicting with main and the shared-state bag omits the newer allow-always approval cache that now lives beside the relay maps.

## What This Changes

The PR moves native hook relay registration, bridge, invocation, and approval-window state to a process-global Symbol.for state bag and adds a regression test for global relay registrations and bridges.

## Best Possible Solution

Land the process-global native relay state fix after rebasing it to include every current relay state collection, especially permissionAllowAlwaysApprovals, with focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible: duplicate module instances can split registerNativeHookRelay from nativeHook.invoke's module-local maps, and current main still throws native hook relay not found on a missed lookup. The PR comments add live smoke and production backport confirmation.

## Solution Assessment

No, not quite as proposed. The globalThis shared-state direction is the narrow maintainable fix, but the branch must include the newer allow-always approval cache and rebase onto current main before it is safe to merge.
