## Summary

Keep open. This is a focused bug-fix PR for a real Codex native hook relay split-state failure, but the branch now needs a current-main refresh because new allow-always permission cache state would otherwise remain module-local while the rest of the relay state is shared.

## What This Changes

The PR moves Codex/native hook relay registry, bridge, invocation, and permission-window state into a process-wide global symbol and adds a regression test for global relay and bridge storage.

## Best Possible Solution

Keep the process-global relay state approach, refresh it against current main, include every current in-memory relay and permission cache in the shared bag, and land it with focused relay tests.

## Reproduction Assessment

Yes, source-reproducible. Load duplicate instances of `native-hook-relay.ts` so registration lands in one module while gateway `nativeHook.invoke` uses another; current main then misses `relays.get(relayId)` and throws the reported `native hook relay not found` error, and the PR comments include live production confirmation.

## Solution Assessment

No, not quite on current main. Sharing relay state through a process-global symbol is the right narrow fix, but the patch should also share the newer `permissionAllowAlwaysApprovals` map added since the branch was written.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82
