## Summary

Keep open: current main still keeps native hook relay state in module-local collections, while this PR narrowly shares the relay and bridge state needed for duplicate module instances; I found no blocking correctness or security finding in the patch.

## What This Changes

The PR moves native hook relay registry, bridge, invocation, and permission-window state from module-local collections to a `Symbol.for` process-global state bag and adds a regression test for global relay and bridge storage.

## Best Possible Solution

Land this narrow shared-state fix, then handle broader native relay diagnostics or health-output improvements as separate follow-up work if maintainers still want them.

## Reproduction Assessment

No deterministic local reproduction was established here. The current-main source path is clear: duplicate module instances can split `relays`/`relayBridges`, and `nativeHook.invoke` throws `native hook relay not found` when lookup hits the empty copy.

## Solution Assessment

Yes. Sharing the in-process relay state through a process-global symbol is the narrowest maintainable fix for duplicate package copies, and the latest patch includes the bridge state identified in review discussion.
