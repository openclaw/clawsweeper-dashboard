## Summary

Keep open: current main still keeps native hook relay state module-local, while this PR shares relays, bridge registrations, invocations, and permission state through a process-global symbol. I found no blocking correctness or security issue, and exact-head checks are green, so this should proceed through normal maintainer merge approval rather than a repair lane.

## What This Changes

The PR moves native hook relay registry, bridge, invocation, and permission-window state into a process-wide `Symbol.for` global and adds a regression test asserting relay and bridge storage.

## Best Possible Solution

Land this narrow shared-state fix, then handle relay health diagnostics or a supported opt-out as separate follow-up work if maintainers still want those broader safeguards.
