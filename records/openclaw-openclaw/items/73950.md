## What This Changes

The PR moves Codex native hook relay in-memory registries, bridge state, invocation history, and permission approval state into a process-global `Symbol.for` state bag and adds regression coverage for relay and bridge storage.

## Best Possible Solution

Land this shared-state fix with a concise `CHANGELOG.md` entry, and leave broader relay health diagnostics or fallback policy changes for separate follow-up work.

## Reproduction Assessment

Yes, at the source level: load two copies of `src/agents/harness/native-hook-relay.ts`, register in one copy, then invoke through the other copy and `relays.get(relayId)` cannot see the registration. I did not run that duplicate-module harness locally, but the current code path and production comments make the failure mode high-confidence.

## Solution Assessment

Yes, with one release-note gap. Sharing `relays`, `relayBridges`, invocation history, and permission approval state through a process-global symbol is the narrowest maintainable fix for duplicate package copies, and it follows an existing OpenClaw global-state pattern used by Codex app-server shared-client state.
