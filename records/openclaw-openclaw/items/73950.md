## Summary

Keep open with changes requested. The PR targets a real Codex native-hook relay bug that is still present on current main, but the branch predates the newer allow-always permission cache and would leave that cache split across duplicate module instances after a rebase.

## What This Changes

The PR moves native hook relay registries, bridge registrations, invocation history, and permission approval state into a process-global symbol-backed state bag and adds a focused regression test.

## Best Possible Solution

Keep the process-global relay state approach, refresh it against current main, include every current relay and permission cache in the shared bag, and land it with focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible: load duplicate instances of `native-hook-relay.ts` so registration lands in one module while gateway `nativeHook.invoke` uses another, and current main still throws `native hook relay not found` on the missed `relays.get(relayId)`. The PR comments also include live smoke and production backport confirmation.

## Solution Assessment

No, not quite on current main. Sharing relay state through a process-global symbol is the narrow fix, but the patch must also share the newer `permissionAllowAlwaysApprovals` map added after the branch was written.
