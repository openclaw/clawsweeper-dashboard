## Summary

Keep open: this is a valid focused bug fix for the Codex native hook relay split-state failure, but the branch is not merge-ready against current main because it omits newer relay permission state and is currently conflicting.

## What This Changes

The PR moves the native hook relay registry, bridge registry, invocation history, and approval-window state into process-global state so duplicate module instances share relay lookup data.

## Best Possible Solution

Keep the process-global relay state approach, refresh it against current main, include every current in-memory relay and permission cache in the shared state bag, and land it with focused relay coverage.

## Reproduction Assessment

Yes, source-reproducible. Duplicate module instances can split `registerNativeHookRelay` from `invokeNativeHookRelay`, and current main still throws `native hook relay not found` when the lookup misses.

## Solution Assessment

No, not quite on current main. Sharing relay state through a process-global symbol is the right narrow fix, but the patch must also share the newer allow-always permission cache before it is merge-ready.
