## Summary

Keep open. Current main and v2026.6.10 still expose only static Docker bind arrays; the requested per-session pre-container bind policy is not implemented and is security/product-sensitive rather than cleanup-closeable.

## What This Changes

Request a host-side pre-container hook or resolver that can vary Docker sandbox bind mounts per session context.

## Best Possible Solution

Design a maintainer-approved pre-container sandbox policy seam that resolves per-session bind decisions before Docker creation, reuses existing bind validation, and documents the security and upgrade contract.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that current main only accepts static Docker bind arrays, so the requested dynamic pre-container capability is absent by design.

## Solution Assessment

Unclear whether the proposed executable `resolveBinds` field is the best final API. A pre-container bind decision point is plausible, but host execution, config, plugin boundary, timeout, and audit contracts need maintainer security/product review.
