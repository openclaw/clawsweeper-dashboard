## Summary

Keep open. Current main and the latest release still expose only static Docker bind arrays, with no pre-container, session-aware bind resolver; the related volume-strategy PR and external-bind policy issue do not supersede this security-sensitive API request.

## What This Changes

The issue requests a host-side pre-container hook that resolves Docker sandbox bind mounts dynamically from channel, user, agent, or session metadata.

## Best Possible Solution

Design a security-reviewed pre-create sandbox lifecycle contract that resolves per-session mounts before Docker create, validates them through the existing bind policy, and includes the resolved mount set in container reuse identity.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that the requested capability is absent: schema/types expose static binds, resolution has no session metadata, and Docker create consumes `cfg.binds` before `setupCommand`.

## Solution Assessment

Unclear. The user problem is valid, but the proposed `resolveBinds` executable should be weighed against a trusted plugin or lifecycle API with explicit trust, validation, timeout, audit, and reuse semantics.
