## Summary

Keep open. Current main still exposes static Docker bind arrays only, and the requested pre-create, session-aware bind resolver remains a security-sensitive sandbox lifecycle/API design item rather than an implemented or superseded cleanup candidate.

## What This Changes

Add a host-side pre-container-creation hook or config resolver that computes Docker sandbox bind mounts from session, channel, user, or metadata context.

## Best Possible Solution

Design a security-reviewed pre-create sandbox lifecycle contract that resolves per-session mounts before Docker create, validates the final mount set through existing bind policy, and includes the resolved set in container reuse identity.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence the requested capability is absent: schema/types expose static binds, resolution has no session metadata, and Docker create consumes cfg.binds before setupCommand.

## Solution Assessment

Unclear. The user problem is valid, but the proposed resolveBinds executable should be weighed against a trusted plugin or lifecycle API with explicit trust, validation, timeout, audit, and reuse semantics.
