## Summary

Keep open. Current main and the latest release still support only static Docker sandbox bind arrays, while the requested per-session pre-container bind resolver remains a security-sensitive sandbox/config/plugin API decision with no merged canonical fix or superseding tracker.

## What This Changes

Add a host-side pre-container hook or resolver that computes Docker sandbox bind mounts per session from channel, user, agent, or metadata context.

## Best Possible Solution

Design a maintainer-approved pre-container sandbox policy seam that resolves per-session bind decisions before Docker creation, reuses existing bind validation and config-hash/recreate behavior, and documents the security and upgrade contract.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection confirms current main only accepts static Docker bind arrays, so the requested dynamic pre-container capability is absent by design rather than a broken existing contract.

## Solution Assessment

Unclear whether the proposed `resolveBinds` executable is the best final API. A pre-container bind decision point is plausibly the right capability, but the final shape should be a maintainer security/product decision because it changes host execution, config, sandbox policy, and plugin extension boundaries.
