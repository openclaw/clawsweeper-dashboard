## Summary

Keep open. Current main still has only static Docker bind configuration and no pre-create, session-aware bind resolver; this is a security-sensitive sandbox/API feature that needs maintainer design rather than cleanup closure.

## What This Changes

Request to add a host-side pre-container bind-mount resolver so Docker sandbox binds can vary by session context.

## Best Possible Solution

Define a security-reviewed pre-create sandbox lifecycle contract that resolves binds before Docker create, validates output through existing bind policy, and makes dynamic bind identity part of container reuse decisions.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. Source inspection gives high confidence that the requested capability is absent: schema/types accept only static `binds`, resolution only merges arrays, and Docker create uses `cfg.binds`.

## Solution Assessment

Unclear. The user problem is valid, but the proposed `resolveBinds` executable should not be accepted without maintainer/security design against a trusted plugin or lifecycle API alternative.
