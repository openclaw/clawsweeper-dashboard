## What This Changes

Request a host-side pre-container hook or resolver that can add Docker sandbox bind mounts per session from channel, user, or metadata context.

## Best Possible Solution

Design a maintainer-approved pre-container sandbox policy seam that can compute or deny per-session binds with typed context, deterministic validation, and clear container lifecycle semantics.

## Risks / Open Questions

- A host-side bind resolver would be a privileged sandbox-boundary decision point and needs explicit trust, timeout, validation, audit, and failure-mode design.
- Per-session dynamic binds interact with sandbox `scope`, container reuse, and config hashes; shared or agent-scoped containers may not safely support user-specific mounts without lifecycle semantics.
- The API shape is unresolved: an executable config field is plausible, but a typed plugin hook or sandbox policy seam may be safer and easier to maintain.
