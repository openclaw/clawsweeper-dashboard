## Summary

Keep open: current main and the latest release still model Docker sandbox binds as static config, and the requested dynamic per-session bind resolver is a security-sensitive feature/config decision rather than an already-implemented cleanup item.

## What This Changes

Add a pre-container host-side resolver for Docker sandbox bind mounts so per-session, channel, or user context can choose mounts dynamically.

## Best Possible Solution

Maintainers should first choose the product/security boundary for per-session mount resolution, then implement it before backend creation with existing bind validation, explicit lifecycle/hash semantics, docs, and focused sandbox tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high-confidence evidence that current main lacks a pre-container dynamic bind resolver and only supports static configured bind arrays.

## Solution Assessment

Unclear whether the proposed host executable is the best final shape. The underlying least-privilege problem is valid, but a typed plugin/backend resolver API or narrower config surface may be safer than an arbitrary script hook.
