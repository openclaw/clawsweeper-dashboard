## Summary

Keep open. Current main and the latest release still expose static Docker sandbox bind arrays only, and this remains a security-sensitive sandbox lifecycle/config/plugin API product decision rather than an implemented or stale cleanup item.

## What This Changes

Request a host-side pre-container hook or callback that can resolve Docker sandbox bind mounts dynamically from session/channel/user context.

## Best Possible Solution

Design a maintainer-approved pre-container sandbox policy seam, likely in core or a documented plugin SDK boundary, that resolves per-session binds before Docker creation, reuses existing bind validation, records config/recreate behavior safely, and documents the security contract.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection shows current main only supports static configured Docker bind arrays, so the request is for a missing feature rather than broken existing behavior.

## Solution Assessment

Unclear whether the proposed `resolveBinds` executable is the best final shape. A pre-container bind decision point is plausible, but the permanent API should be a maintainer-approved sandbox policy or plugin SDK seam because it affects host execution, config surface, and sandbox authorization.
