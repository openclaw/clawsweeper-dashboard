## Summary

Keep open. Current `main` and `v2026.6.8` still only support static Docker bind arrays, while the existing plugin gate hooks run too late to decide container mounts; the request is valid but needs maintainer product and security design before implementation.

## What This Changes

Request a pre-container host-side hook or resolver that can vary Docker sandbox bind mounts per session using channel, user, or metadata context.

## Best Possible Solution

Design a maintainer-approved pre-container sandbox policy seam that can compute or deny per-session binds with typed context, deterministic validation, and explicit container lifecycle semantics.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection give high confidence that current main lacks a pre-container per-session bind resolver.

## Solution Assessment

Unclear. The requested script field is plausible, but the best maintainable solution likely needs maintainer choice between a config script, a typed plugin hook, or a broader sandbox policy API.
