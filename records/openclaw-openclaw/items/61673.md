## Summary

Keep open. Current main and the latest release still expose only static Docker sandbox bind arrays, and the requested per-session pre-container bind resolver remains a security-sensitive config/plugin API design decision rather than a stale or already-implemented cleanup item.

## What This Changes

Requests a host-side pre-container hook or config field to resolve Docker sandbox bind mounts dynamically from session context.

## Best Possible Solution

Design a maintainer-approved pre-container sandbox policy seam that resolves per-session binds before Docker creation, reuses existing bind validation, records/recreates container config safely, and documents the security and upgrade contract.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that current main only supports static configured bind arrays, so the requested dynamic pre-container behavior is absent by design rather than a failing existing contract.

## Solution Assessment

Unclear whether the proposed `resolveBinds` executable is the best final shape. A pre-container bind decision point is plausibly the right capability, but the permanent API should be chosen by maintainers because it changes config surface, host code execution, sandbox policy, and plugin extension boundaries.
