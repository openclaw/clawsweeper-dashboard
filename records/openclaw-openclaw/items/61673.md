## Summary

Keep open. Current main and the latest release still only support static Docker sandbox bind arrays, and there is no merged resolver, closing PR, or canonical replacement for the per-session pre-container bind policy this issue requests.

## What This Changes

The issue requests a host-side pre-container hook that can resolve Docker sandbox bind mounts dynamically from channel, user, agent, or session metadata.

## Best Possible Solution

Design a maintainer-approved pre-container sandbox policy seam that resolves per-session bind decisions before Docker creation, reuses existing bind validation, and documents the security and upgrade contract.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that current main only accepts static Docker bind arrays, so the requested dynamic pre-container capability is absent by design rather than a broken existing contract.

## Solution Assessment

Unclear whether the proposed `resolveBinds` executable is the best final API. A pre-container bind decision point is plausible, but the final shape needs maintainer security/product approval because it changes host execution, config, sandbox policy, and plugin boundaries.
