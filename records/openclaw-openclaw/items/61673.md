## Summary

Keep open. Current main and the latest release still support only static Docker sandbox bind arrays, and the requested per-session pre-container bind resolver remains a security-sensitive sandbox/config/plugin API design decision with no merged canonical replacement.

## What This Changes

Request a host-side pre-container resolver that derives Docker sandbox bind mounts from per-session context before container creation.

## Best Possible Solution

Keep this issue open until maintainers choose a pre-container sandbox policy seam that can resolve per-session mounts safely, reuse existing bind validation, and define container reuse plus audit semantics.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that current main only accepts static `sandbox.docker.binds` arrays and has no pre-container session-context bind resolver.

## Solution Assessment

Unclear whether an executable `resolveBinds` field is the best final API. The maintainable solution should be a maintainer-approved sandbox policy or plugin seam that runs before backend creation, reuses bind validation, and makes container reuse and audit semantics explicit.
