## Summary

Keep open: the PR improves the env/file SecretRef path, but it preserves the misleading unavailable warning for the linked exec SecretRef reproduction and still lacks after-fix real `openclaw doctor` proof.

## What This Changes

The PR updates `runGatewayAuthHealth()` to resolve non-exec `gateway.auth.token` SecretRefs before emitting the Gateway auth warning.

## Best Possible Solution

Use a doctor-specific SecretRef check that resolves env/file refs, avoids executing exec providers unless explicitly opted in, and reports skipped exec refs as unverified rather than unavailable.

## Reproduction Assessment

Yes, source-reproducible: configure local token auth with `gateway.auth.token` as a resolvable exec SecretRef and run `openclaw doctor`; current main detects the SecretRef but `resolveGatewayAuth()` ignores SecretRef object values and reaches the unavailable warning branch.

## Solution Assessment

No. Resolving env/file refs before warning is the right direction, but this patch deliberately skips exec refs and emits the same unavailable diagnostic for the exact linked reproduction; the safer solution is a policy-aware exec path or an explicit unverified diagnostic.
