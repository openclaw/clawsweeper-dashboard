## Summary

Keep open: the PR helps env/file SecretRefs, but it still reports skipped exec SecretRefs as unavailable, so the exact linked exec-SecretRef bug remains unresolved and the external PR still lacks after-fix real behavior proof.

## What This Changes

This PR updates `runGatewayAuthHealth()` to resolve non-exec `gateway.auth.token` SecretRefs before warning and to include unresolved reasons for env/file failures.

## Best Possible Solution

Resolve safe env/file Gateway token SecretRefs after token auth is known to require a token, avoid implicit exec execution, and report skipped exec refs as unverified or explicitly opt-in rather than unavailable.

## Reproduction Assessment

Yes. Source inspection plus the linked report gives a high-confidence path: configure local token auth with a resolvable `gateway.auth.token` exec SecretRef, run `openclaw secrets audit --allow-exec`, then run `openclaw doctor`; current main reaches the unavailable warning before materializing the SecretRef.

## Solution Assessment

No. Resolving env/file refs before warning is the right direction, but preserving the generic unavailable diagnostic for skipped exec refs leaves the linked exec-SecretRef report unresolved.
