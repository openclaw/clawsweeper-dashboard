## Summary

Keep open: this PR fixes the env/file SecretRef half of a real doctor false-warning path, but it still leaves exec SecretRefs on the old “currently unavailable” diagnostic, lacks after-fix real `openclaw doctor` proof, and overlaps another open implementation PR.

## What This Changes

The PR changes `runGatewayAuthHealth()` to resolve non-exec `gateway.auth.token` SecretRefs before warning and to report unresolved env/file reasons.

## Best Possible Solution

Resolve env/file Gateway token SecretRefs in doctor, avoid implicit exec execution, report skipped exec refs as unverified or unavailable in this command path, and require redacted real `openclaw doctor` proof.

## Reproduction Assessment

Yes, source-reproducible: configure local token auth with a resolvable `gateway.auth.token` SecretRef and run `openclaw doctor`; current main detects the SecretRef, `resolveGatewayAuth()` ignores object values, and the warning branch remains reachable. I did not run a live exec provider in this read-only review.

## Solution Assessment

No. Resolving env/file refs before warning is the right direction, but this patch skips exec refs and then emits the same unavailable wording for the linked exec reproduction; a safer fix should make the skipped verification explicit or add an opt-in policy.
