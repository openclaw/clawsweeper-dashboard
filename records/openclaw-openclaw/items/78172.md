## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Resolve filtering with channel context:** `extensions/speech-core/src/tts.ts:1106-1109`
  - body: This resolves `skipEmojiSymbols` with only agent and account IDs, while the current TTS config contract also applies `channels.<id>.tts` and channel account overrides through `channelId`. Channel or account policy for the new option would be ignored on automatic and command TTS paths.
  - confidence: 0.94
- **[P2] Remove the tool-local prefilter:** `src/agents/tools/tts-tool.ts:97-103`
  - body: The tool strips text before calling the shared TTS path using only `opts?.agentId`. Once the text is stripped, a channel or account override that disables `skipEmojiSymbols` cannot restore it, so this can violate the same precedence used elsewhere in TTS.
  - confidence: 0.93
- **[P3] Update the config docs baseline:** `src/config/zod-schema.core.ts:566`
  - body: Adding a strict `messages.tts` key changes the public config surface, but the PR does not update the TTS docs or regenerated config baseline hash. Users would not discover the field, and the config-doc baseline check expects intentional schema drift to be committed.
  - confidence: 0.86
