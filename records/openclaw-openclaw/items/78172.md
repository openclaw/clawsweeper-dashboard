## Summary

Keep open: the PR has useful direction for https://github.com/openclaw/openclaw/issues/77969, but the current patch still breaks documented TTS config precedence and only supplies mocked test proof, so it should not merge yet.

## What This Changes

The PR adds a `messages.tts.skipEmojiSymbols` TTS config field, schema/type wiring, shared and tool-local emoji/symbol filtering, and mocked agent-tool tests.

## Best Possible Solution

Make filtering a single shared TTS text-preparation step that uses agent, channel, and account context; document and release-note the new option after maintainers confirm the scope.

## Reproduction Assessment

Not applicable as a current-main bug reproduction because this PR adds a new config option. The PR defects are source-reproducible from the diff against the current TTS config precedence contract.

## Solution Assessment

No. The maintainable fix is shared-path filtering with full effective config context, plus docs/changelog and real runtime proof; the current tool-local prepass and partial shared context are not the right boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Remove the tool-local emoji prefilter:** `src/agents/tools/tts-tool.ts:94-99`
  - body: The tool resolves `skipEmojiSymbols` with only `opts?.agentId` and strips before calling `textToSpeech`. That bypasses the documented channel/account override precedence, so a channel or account that disables the option still receives already-stripped text.
  - confidence: 0.94
- **[P2] Resolve the filter with channel context:** `extensions/speech-core/src/tts.ts:1104-1107`
  - body: The shared filter calls `resolveEffectiveTtsConfig` with `agentId` and `accountId` but not `channelId`. Current TTS config resolution needs `params.channel` to apply `channels.<id>.tts` and account overrides, so shared callers such as auto-TTS and `/tts audio` ignore channel policy for this option.
  - confidence: 0.96
- **[P3] Document the new TTS option:** `src/config/types.tts.ts:81`
  - body: This adds user-facing `messages.tts.skipEmojiSymbols`, but `docs/tools/tts.md` and `CHANGELOG.md` are unchanged. Users would not find the option in the TTS field reference, and repo policy requires a changelog entry for user-facing feature config.
  - confidence: 0.88
