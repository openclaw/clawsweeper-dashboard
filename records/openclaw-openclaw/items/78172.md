## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Keep filtering out of the tool-local prepass:** `src/agents/tools/tts-tool.ts:94-99`
  - body: The tool still strips text before calling `textToSpeech` using `resolveEffectiveTtsConfig(cfg, opts?.agentId)`, which ignores the selected channel and account. If global config enables the option but a channel/account override disables it, the text has already been stripped before the shared runtime can honor that override.
  - confidence: 0.94
- **[P2] Pass channel context to TTS config resolution:** `extensions/speech-core/src/tts.ts:1104-1107`
  - body: The shared filter resolves `skipEmojiSymbols` with only `agentId` and `accountId`. Current TTS config resolution needs `channelId` to find both channel and account overrides, so `channels.<id>.tts.skipEmojiSymbols` would be ignored in auto-TTS, `/tts audio`, gateway conversion, and other shared callers.
  - confidence: 0.96
- **[P3] Document the new TTS option:** `src/config/types.tts.ts:81`
  - body: This adds a user-facing `messages.tts` config field, but the PR does not update `docs/tools/tts.md` or `CHANGELOG.md`. OpenClaw policy requires user-facing feature config to be documented and release-noted.
  - confidence: 0.88
