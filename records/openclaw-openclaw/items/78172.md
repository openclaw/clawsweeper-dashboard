## Summary

Keep open: the requested TTS filtering feature is not implemented on current main, and this PR is a plausible but currently incorrect implementation because it bypasses established TTS config precedence and only supplies mocked test proof.

## What This Changes

The PR adds a `messages.tts.skipEmojiSymbols` config field, schema/type wiring, emoji/symbol stripping before TTS synthesis, and focused agent-tool tests.

## Best Possible Solution

Implement one shared TTS text-preparation step after effective config resolution with full agent, channel, and account context, document the setting, and leave file-path or system-message suppression to a separate scoped decision if needed.

## Reproduction Assessment

No high-confidence current-main bug reproduction is applicable because this PR adds a new opt-in config option. The PR defects are source-reproducible from the diff against the current TTS config precedence path.

## Solution Assessment

No. The shared TTS path is the right place for this behavior, but this branch resolves the option without full channel context, duplicates filtering in the agent tool, and omits user-facing docs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Resolve filtering with channel context:** `extensions/speech-core/src/tts.ts:1104-1107`
  - body: The new shared filter calls `resolveEffectiveTtsConfig` with agent/account IDs but not `params.channel`. Current TTS config resolution applies channel overrides and channel-scoped account overrides, so `channels.<id>.tts.skipEmojiSymbols` cannot control synthesis on those paths.
  - confidence: 0.95
- **[P2] Keep filtering out of the tool wrapper:** `src/agents/tools/tts-tool.ts:94-99`
  - body: The tool strips text before calling `textToSpeech` using only the agent ID. Once the text is mutated, a channel argument or account override that disables the option cannot restore it, so the wrapper can violate the same effective-config precedence used by the rest of TTS.
  - confidence: 0.93
- **[P3] Document the new TTS option:** `src/config/types.tts.ts:81-82`
  - body: This adds a user-facing `messages.tts.skipEmojiSymbols` setting, but the TTS docs do not list the field or explain how it participates in global, agent, channel, and account precedence. Users would not be able to discover or configure it reliably.
  - confidence: 0.84
