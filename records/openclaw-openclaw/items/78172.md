## Summary

Keep open: the shared speech-core direction and real-provider proof are strong, but the current stripper removes emoji/symbol runs with an empty string, so enabled TTS can collapse words when emoji or symbols are used as separators.

## What This Changes

Adds a default-off `messages.tts.skipEmojiSymbols` option, applies shared emoji/symbol stripping across file, stream, and telephony TTS paths, updates config docs/metadata, and adds focused tests.

## Best Possible Solution

Keep the centralized speech-core preprocessing design, but strip emoji/symbol runs as whitespace, normalize the result, cover a no-space separator regression, and leave long path/lifecycle elision to the linked broader issue.
