## Summary

I found one remaining correctness blocker: the opt-in stripping path removes emoji/symbol runs with no separator, so symbols used between words change the spoken text by merging adjacent words. The shared speech-core placement, config/schema/docs wiring, and real-provider proof are otherwise strong enough to keep reviewing rather than close this PR.

## What This Changes

The branch adds a default-off `messages.tts.skipEmojiSymbols` config option, applies shared TTS text preprocessing before file, stream, and telephony synthesis, updates config metadata/docs, and adds focused TTS tests.

## Best Possible Solution

Keep the shared speech-core preprocessing design, replace stripped emoji/symbol runs with normalized whitespace, add a no-space separator regression, and leave long path/file elision to https://github.com/openclaw/openclaw/issues/77969.
