## What This Changes

The PR adds a default-off `messages.tts.skipEmojiSymbols` option, config/docs metadata, and shared speech-core preprocessing so TTS file, stream, and telephony synthesis can strip emoji/decorative symbols and reject empty-after-strip input.

## Best Possible Solution

Keep the default-off shared speech-core preprocessing design, replace stripped emoji/symbol runs with normalized whitespace, add an adjacent-word regression, and leave broader file/path elision to the existing follow-up issue.

## Solution Assessment

No as written; centralizing the default-off filter in speech-core is the maintainable direction, but the helper needs to preserve word boundaries before this public option merges.
