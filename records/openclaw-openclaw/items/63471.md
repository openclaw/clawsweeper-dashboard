## Summary

Keeping this open: current main still requires a setup/config step before raw Docker Compose can start the gateway from an empty mounted config, and choosing a one-paste Dockge-style bootstrap affects auth, bind mode, and Control UI origin policy.

## What This Changes

The issue asks for a Dockge/raw Docker Compose flow that starts OpenClaw from a pasted Compose YAML and .env without separate CLI setup commands.

## Best Possible Solution

Add a first-class Docker/Compose bootstrap path that idempotently creates minimal persisted local gateway config, provisions durable gateway auth, seeds safe Control UI origins for the published port, and documents the raw Compose/Dockge workflow.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: start the bundled Compose gateway with an empty mounted config directory, and the command reaches the missing-config startup guard before the server starts.

## Solution Assessment

No current solution is complete. The setup script is the supported guided path, but the requested raw Compose flow needs a maintainer-approved bootstrap/auth/origin contract rather than only bypassing the guard or changing a bind default.
