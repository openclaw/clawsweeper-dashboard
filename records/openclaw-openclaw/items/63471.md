## Summary

Current main still does not provide the requested Dockge-style one-paste Docker Compose bootstrap: the setup script and manual docs can seed the needed config, but raw Compose still starts the gateway directly and the startup guard still exits on a fresh config mount.

## What This Changes

The issue asks for Docker/Dockge-style Compose startup that can be pasted with env defaults and start without a separate pre-start OpenClaw setup step.

## Best Possible Solution

Add a first-class Docker/Compose bootstrap path that idempotently creates minimal persisted local gateway config, provisions or requires durable gateway auth, seeds safe localhost Control UI origins for the published port, and documents the Dockge/raw Compose workflow.

## Reproduction Assessment

Yes. A high-confidence static reproduction path remains: start the bundled Compose gateway with an empty mounted config directory, and the Compose command reaches the gateway startup guard without `--allow-unconfigured` or `gateway.mode=local`, producing the reported missing-config exit.

## Solution Assessment

No current solution is complete. The setup script is useful for guided installs, but the reported workflow needs a raw Compose bootstrap contract rather than only documenting pre-start commands or passing `--allow-unconfigured`.
