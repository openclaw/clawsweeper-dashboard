## Summary

Keeping this open: current main still lacks a Dockge-style paste-and-start Compose bootstrap. The bundled Compose service starts the gateway directly, while the startup guard still exits on a fresh config mount unless setup has already written `gateway.mode=local` or the user explicitly opts into `--allow-unconfigured`.

## What This Changes

Request a Docker Compose/Dockge flow where a pasted Compose file plus `.env` can start OpenClaw without pre-running setup/config commands.

## Best Possible Solution

Add a first-class Docker/Compose bootstrap path that idempotently persists minimal local gateway config, durable gateway auth, safe Control UI origins for the published port, and Dockge/raw Compose documentation.

## Reproduction Assessment

Yes. Source inspection against current main shows `docker-compose.yml` starts the gateway directly, and with an empty config mount the startup guard returns the reported missing-config error before server startup.

## Solution Assessment

No current implementation fully solves it. The narrow maintainable direction is a maintainer-approved Docker bootstrap/auth/origin contract rather than blindly passing `--allow-unconfigured` or documenting extra pre-start commands only.
