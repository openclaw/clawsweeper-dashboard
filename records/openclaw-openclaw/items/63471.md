## Summary

Keeping this open: current main still lacks the requested Dockge-style one-paste Docker Compose bootstrap. The related Docker work improved CLI networking, setup-script automation, and mount defaults, but raw Compose still starts the gateway directly and the startup guard still exits on a fresh config mount.

## What This Changes

Request for Docker/Dockge-style startup where a pasted Compose file plus environment can bring up the gateway without running extra setup commands first.

## Best Possible Solution

Add a first-class Docker/Compose bootstrap path that idempotently creates minimal persisted local gateway config, provisions or requires durable gateway auth, seeds safe localhost Control UI origins for the published port, and documents the Dockge/raw Compose workflow.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: start the bundled Compose gateway with an empty mounted config directory, and the command reaches the startup guard without `--allow-unconfigured` or `gateway.mode=local`, producing the reported missing-config exit.

## Solution Assessment

No current solution is complete. The setup script is a good guided path, but the requested Dockge/raw Compose flow needs a maintainer-approved bootstrap/auth/origin contract rather than only documenting pre-start commands or bypassing the guard.
