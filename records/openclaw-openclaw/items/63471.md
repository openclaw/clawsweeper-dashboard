## Summary

Keep open: current main and v2026.6.10 still require setup or explicit gateway config before raw Dockge-style Compose startup from an empty config, and the linked allowed-origins work is only a partial overlap.

## What This Changes

The issue requests a Dockge/Portainer-friendly Docker Compose startup path that can run from pasted YAML and `.env` without extra CLI setup steps.

## Best Possible Solution

Add a maintainer-approved Docker-manager bootstrap contract that persists auth, `gateway.mode`, bind, and Control UI origin policy before the gateway serves a published port.

## Reproduction Assessment

Yes, source-level: a raw Compose gateway start with an empty mounted config reaches the missing-config guard before server startup. I did not run Docker because this review is read-only.

## Solution Assessment

No, not as proposed: letting the container start unconfigured is not the safest fix for a published Docker gateway. The safer path is a defined bootstrap contract that establishes auth, mode, bind, persistence, and origins before serving.
