- `impact:security`: The request changes when an unconfigured Docker gateway may serve before durable auth, bind, and Control UI origin policy are established.

## Summary

Keep open: current main and v2026.6.8 still require setup or explicit config before a raw Dockge-style Docker Compose startup from an empty config, and making that flow safe needs a maintainer product/security decision rather than cleanup closure.

## What This Changes

Request a Dockge/manager-friendly Docker Compose flow where a pasted compose stack plus `.env` can start OpenClaw without extra CLI setup steps.

## Best Possible Solution

Add a maintainer-approved Docker/Compose first-run bootstrap that works in manager UIs, persists safe gateway config and durable auth, handles Control UI origins for the published URL, and never serves an exposed gateway before those facts exist.

## Reproduction Assessment

Yes, source-level: a raw Compose gateway start with an empty mounted config reaches the missing-config guard and exits before `startGatewayServer`. I did not run Docker because this review is read-only.

## Solution Assessment

No, not as proposed: simply letting the container start without pre-start config would weaken the fail-closed gateway boundary. The safer direction is a defined Docker bootstrap contract that establishes auth, mode, bind, and origins before serving.
