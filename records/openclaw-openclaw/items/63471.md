## Summary

Keeping this open: current main still does not provide a Dockge-style raw Compose paste-and-start path from an empty config, and the supported Docker path still relies on setup or manual pre-start config writes. Changing that startup behavior crosses gateway auth, bind, and Control UI origin policy, so it needs maintainer product/security direction first.

## What This Changes

Request a Dockge/raw Docker Compose startup path that can start the gateway from a pasted Compose YAML and .env without running setup or pre-start CLI config commands.

## Best Possible Solution

Add a maintainer-approved Docker/Compose first-run bootstrap that persists local gateway config, establishes durable auth, seeds safe Control UI origins for the published port, and documents the Dockge/raw Compose workflow.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: start the bundled Compose gateway with an empty mounted config directory and no `--allow-unconfigured`, and the command reaches the missing-config guard before the server starts.

## Solution Assessment

No. Letting the container start or changing bind defaults alone is not the best fix; the maintainable solution needs an explicit Docker bootstrap/auth/origin contract before serving.
