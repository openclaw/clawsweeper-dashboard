## Summary

Current main still does not implement the requested Dockge-style one-paste Docker Compose bootstrap. The helper script and manual docs can seed the required config, but the bundled Compose service still starts the gateway directly and the startup guard still exits on a fresh config mount.

## What This Changes

The issue requests a Dockge-friendly Docker Compose flow that can start OpenClaw from pasted Compose YAML and an .env file without separate pre-start CLI setup.

## Best Possible Solution

Add a first-class Docker Compose bootstrap path for Dockge-style installs that idempotently creates minimal persisted local gateway config, provisions or requires durable gateway auth, seeds safe Control UI origins for the published local port, preserves non-loopback auth/origin guardrails, and updates the Docker and ClawDock docs.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: fresh Compose with no mounted `openclaw.json` runs the Compose-defined gateway command without `--allow-unconfigured`, reaches `getGatewayStartGuardErrors`, and exits with the reported missing-config message before server startup.

## Solution Assessment

No current solution is complete. The setup script path is useful for scripted installs, but it still requires pre-start CLI work; simply adding `--allow-unconfigured` to Compose would be a weaker fix than a durable bootstrap path that handles auth and Control UI origins.
