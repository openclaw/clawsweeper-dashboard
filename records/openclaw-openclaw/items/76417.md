## Summary

Keep open: current main still starts Discord typing from the queued reply pipeline, while Telegram already has the earlier best-effort cue requested here.

## What This Changes

Discord should send a native typing indicator immediately after an inbound message is accepted, before slow pre-dispatch or queued agent work begins.

## Best Possible Solution

Add a Discord-owned best-effort early typing cue at the accepted inbound/queue boundary, preserving existing typing modes, ack reactions, refresh cadence, and cleanup semantics.

## Reproduction Assessment

Yes from source inspection: accepted Discord messages are queued before processDiscordMessage runs, and typing is only started later through the reply-start callback. I did not run a live Discord bot in this read-only sweep.

## Solution Assessment

Yes: mirroring Telegram's fire-and-forget early cue inside the Discord channel plugin is the narrow maintainable fix, without changing global typing-mode semantics or adding config.
