## Summary

Keep open: current main still starts Discord native typing from the queued reply pipeline, so the requested inbound-acceptance cue remains missing while Telegram already has that earlier best-effort signal.

## What This Changes

Request to send a best-effort Discord native typing indicator immediately after an inbound DM is accepted, before queueing and pre-dispatch work.

## Best Possible Solution

Add the early best-effort cue inside the Discord channel adapter at the accepted inbound/queue boundary, reusing the existing typing helper and preserving current typing-mode and ack-reaction semantics.

## Reproduction Assessment

Yes from source inspection: accepted Discord messages are queued before processDiscordMessage runs, and native typing is only started later through the reply-start callback. I did not run a live Discord bot in this read-only review.

## Solution Assessment

Yes: mirroring Telegram's channel-owned fire-and-forget early cue is the narrowest maintainable fix, without changing global typing configuration or adding new product policy.
