## Summary

Keep open: current main still queues accepted Discord messages before invoking native typing, so the requested early liveness cue remains missing; draft PR #76091 appears to be the active implementation path rather than a reason to close before merge.

## What This Changes

Request to send a best-effort Discord native typing indicator immediately after inbound message acceptance, before queueing and pre-dispatch delays, matching the Telegram fix.

## Best Possible Solution

Review and land #76091, or an equivalent Discord-owned fix, so accepted Discord inbound messages start native typing before queued pre-dispatch work and then reuse the existing refresh/cleanup lifecycle.

## Reproduction Assessment

Yes from source inspection. Accepted Discord messages are queued before processDiscordMessage runs, and native typing is only started later through the reply-start callback; I did not run a live Discord bot in this read-only review.

## Solution Assessment

Yes. A Discord-owned early cue carried into the existing typing lifecycle is the narrow maintainable direction; the open draft PR appears to implement that shape and should be reviewed rather than duplicating work.
