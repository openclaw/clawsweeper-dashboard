## What This Changes

Report that cross-OS migrations can leave Linux/macOS absolute agent workspace paths in `openclaw.json`, which doctor does not flag and the migration guide does not mention.

## Best Possible Solution

Review and either land or replace PR #63803 so stale home-shaped agent workspace paths are handled by doctor-owned repair plus migration documentation.

## Reproduction Assessment

Yes. The reporter supplied concrete WSL/Linux-to-macOS migration steps and a stack trace, and current main still routes configured workspace strings directly into `fs.mkdir`.

## Solution Assessment

Yes. The narrow maintainable direction is a doctor/docs repair path for legacy config, rather than silent startup mutation or a generic rewrite of every absolute path.
