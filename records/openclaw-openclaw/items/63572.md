## Best Possible Solution

Land or replace the linked doctor-owned repair so safe stale home-shaped agent workspace paths are detected and rewritable to portable ~ paths, with the migration guide documenting the cross-OS path update requirement.

## Solution Assessment

Yes. A doctor-layer repair plus migration-guide warning is the right boundary because it handles migrated legacy config explicitly without adding startup-time config mutation to the runtime path.
