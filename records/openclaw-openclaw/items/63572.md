## Summary

Keep open. Current main still lacks the doctor repair and migration-guide warning, and the live closing PR is still open and conflicting rather than merged.

## What This Changes

The issue asks OpenClaw to detect or document stale absolute agent workspace paths when users migrate state across operating systems.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/63803 so doctor handles stale home-shaped workspace paths and the migration guide documents the cross-OS path update requirement.

## Reproduction Assessment

Yes. Configure `agents.defaults.workspace` or an agent `workspace` with a stale `/home/<user>/.openclaw/workspace` value on a macOS host; current main resolves that value directly and `ensureAgentWorkspace` passes it to `fs.mkdir`.

## Solution Assessment

Yes. A doctor-owned repair plus migration-guide warning is the narrow maintainable fix because it targets legacy config without silently mutating startup/runtime workspace creation.
