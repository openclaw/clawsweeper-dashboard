## What This Changes

The issue requests either migration documentation or `openclaw doctor` detection/repair for stale absolute agent workspace paths after cross-OS migration.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/63803 so doctor handles stale home-shaped agent workspace paths and the migration guide documents the cross-OS path update requirement.

## Reproduction Assessment

Yes. Configure `agents.defaults.workspace` or an agent `workspace` with a stale `/home/<user>/.openclaw/workspace` value on a macOS host; current main resolves that value directly and `ensureAgentWorkspace` passes it to `fs.mkdir`.

## Solution Assessment

Yes. A doctor-owned repair plus migration-guide warning is the narrow maintainable solution because it targets legacy config without silently mutating startup/runtime workspace creation.
