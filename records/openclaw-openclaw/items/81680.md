## Summary

Keep this PR open: the Signal hook gap is source-proven and the fix is in the right area, but the branch still drops media metadata from the hook event and only supplies unit-test proof instead of a real Signal run.

## What This Changes

The PR imports the global plugin hook runner into Signal monitor reply delivery and runs `message_sending` before sending each resolved reply, honoring cancellation and text rewrites.

## Best Possible Solution

Have Signal reply delivery match the existing `message_sending` semantics with media URL metadata and focused regression coverage, then validate it with a real Signal account and registered hook.

## Reproduction Assessment

Yes, by source inspection on current main: inbound Signal replies call `deps.deliverReplies`, and `deliverReplies` sends text/media through `sendMessageSignal` without a hook runner. I did not run a live Signal reproduction in this read-only review.

## Solution Assessment

Partially. Adding the hook in `extensions/signal/src/monitor.ts` is the narrow maintainable fix, but this branch should also pass media URLs in the hook metadata and provide real Signal proof before merge.
