The issue reports that aborted embedded runs can leave partial assistant text and a transient aborted marker in the next-turn recovery context, causing unstable session state.

## Best Possible Solution

Keep this as the canonical recovery-state tracker, then fix active replay/history assembly and abort-state handling so aborted incomplete assistant fragments become structured recovery facts without losing valid completed content.

No high-confidence live reproduction was established in this read-only pass. The source path is reproducible: current main and v2026.5.18 still have the abort hint, transient abort-marker write, test-only incomplete-text prefill helper behavior, and active replay paths that bypass that helper.
