## What This Changes

The PR removes Telegram thread-not-found threadless retries from draft previews and outbound text, media, sticker, and poll sends, then updates regressions to expect one topic-qualified failure.

## Best Possible Solution

Land the narrow fallback removal after redacted live Telegram forum-topic proof and a green rebased check run confirm topic-qualified failures do not send into the parent chat.

## Reproduction Assessment

Yes, from source inspection. Current main retries `message thread not found` sends without `message_thread_id` in `send.ts` and `draft-stream.ts`, which can produce a visible reply outside the intended forum topic.

## Solution Assessment

Yes for the code direction. Removing threadless retry from topic-qualified sends is the narrow maintainable fix while preserving the existing General-topic omission; the remaining blockers are live Telegram proof and green checks.
