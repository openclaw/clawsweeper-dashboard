## Summary

Keep open. Current main and the latest release still clear the Overview Gateway Token when the WebSocket URL changes, and the live fix path is the still-open, conflicting PR at https://github.com/openclaw/openclaw/pull/73923.

## What This Changes

Editing the Overview Gateway Access WebSocket URL clears the Gateway Token field and causes Connect to fail unless the token is reentered.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/73923 with the scoped Control UI resolver that preserves tokens for same-normalized gateway edits and reloads or clears tokens when the gateway scope changes.

## Reproduction Assessment

Yes. Source inspection shows the URL input clears token on every non-identical trimmed URL edit, and current browser coverage asserts the token is empty after a gateway URL change.

## Solution Assessment

Yes. The normalized-scope resolver in the open fix path is the narrow maintainable solution because it preserves harmless same-endpoint edits without reusing a token for an unrelated gateway.
