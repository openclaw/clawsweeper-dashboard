## Best Possible Solution

Refresh and land or replace https://github.com/openclaw/openclaw/pull/73923 with a focused fix that preserves tokens for same-normalized gateway edits while clearing or reloading credentials when the gateway scope changes.

## Solution Assessment

Yes. The linked scoped-fix direction is the maintainable solution because it preserves credentials for harmless same-scope edits without reusing a token across unrelated gateways.
