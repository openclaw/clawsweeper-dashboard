- `impact:auth-provider`: The cleared gateway token causes Connect to fail until credentials are re-entered.

## Summary

Keep open because current main and v2026.5.20 still clear the Gateway Token from the Overview WebSocket URL input path, and an open closing PR already owns the scoped fix path.

## What This Changes

The issue reports that editing the Overview Gateway Access WebSocket URL immediately clears the in-memory Gateway Token and makes Connect fail unless the token is re-entered.

## Best Possible Solution

Refresh and land or replace https://github.com/openclaw/openclaw/pull/73923 with a focused fix that preserves tokens for same-normalized gateway edits while loading or clearing credentials when the gateway scope changes.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes. Source inspection on current main shows the URL input handler clears the token when the trimmed URL changes, and current browser coverage asserts that cleared-token state after editing the gateway URL.

## Solution Assessment

Is this the best way to solve the issue? Yes. The linked scoped-fix direction is the maintainable solution because it preserves credentials for harmless same-scope edits without reusing a token across unrelated gateways.
