## Summary

Keep open. Current main and v2026.5.22 still contain the Overview URL-edit handler that clears the Gateway Token, and the open linked fix PR is the right path to finish before this issue closes.

## What This Changes

The issue reports that editing the Overview Gateway Access WebSocket URL immediately clears the Gateway Token and makes Connect fail unless the token is re-entered.

## Best Possible Solution

Refresh and land or replace https://github.com/openclaw/openclaw/pull/73923 with a focused Control UI fix that preserves tokens for same-normalized gateway edits while reloading or clearing tokens when the gateway scope changes.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes: source inspection on current main shows the URL input clears settings.token for any non-identical trimmed URL, and current browser coverage asserts the token is empty after editing the gateway URL.

## Solution Assessment

Is this the best way to solve the issue? Yes: the linked scoped-fix direction is the maintainable solution because it preserves harmless same-endpoint edits without carrying a scoped credential to an unrelated endpoint.
