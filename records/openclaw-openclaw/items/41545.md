## Summary

Keep open. Current main and the latest release still clear the Overview Gateway Token when the WebSocket URL input changes, and open PR #73923 references this issue as the current fix path, so the issue should remain open until that PR or another canonical fix lands.

## What This Changes

Preserve the Gateway Token in Overview > Gateway Access while users edit the WebSocket URL, without carrying the token across unrelated gateway endpoints.

## Best Possible Solution

Land a scoped Control UI fix that preserves the in-memory token for same-normalized WebSocket endpoint edits, reloads an existing scoped token or clears it when the normalized endpoint changes, and updates browser/node coverage plus the changelog credit.
