## Summary

Keep open. Current main and the latest release still clear the Overview Gateway Token during WebSocket URL input, and open PR https://github.com/openclaw/openclaw/pull/73923 already references this issue with closing syntax as the canonical fix path.

## What This Changes

The issue asks for Overview Gateway Access to preserve the Gateway Token when editing the WebSocket URL so Connect does not fail from an unexpected cleared token.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/73923 with a scoped Control UI fix that preserves tokens for same-normalized gateway endpoint edits and clears or reloads tokens when the gateway scope changes.

## Reproduction Assessment

Yes. Source inspection on current main shows the input handler writes `token: ""` when the edited URL is not trimmed-string identical, and existing browser coverage asserts the token is cleared after a gateway URL edit.

## Solution Assessment

Yes. The open PR’s normalized-scope approach is the narrow maintainable direction because it preserves harmless edits without reusing a gateway-scoped credential for unrelated endpoints.
