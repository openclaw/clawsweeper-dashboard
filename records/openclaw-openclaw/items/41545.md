## Summary

Keep open. Current main and the latest release still clear the Overview Gateway Token when the WebSocket URL changes, and the active fix path is the open, currently conflicting PR at https://github.com/openclaw/openclaw/pull/73923.

## What This Changes

The issue asks for the Control UI Overview WebSocket URL input to preserve the Gateway Token during safe URL edits instead of clearing it immediately.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/73923 with the scoped Control UI resolver that preserves tokens for same-normalized gateway edits and reloads or clears tokens when the gateway scope changes.

## Reproduction Assessment

Yes. Source inspection shows the URL input clears token on every non-identical trimmed URL edit, and current browser coverage asserts the token is empty after a gateway URL change.

## Solution Assessment

Yes. The linked PR's normalized-scope resolver is the narrow maintainable fix because it preserves harmless same-endpoint edits without reusing a token for an unrelated gateway.
