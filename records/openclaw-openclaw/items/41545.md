## Summary

Keep open. Current main and the latest release still contain the Overview URL-edit path that clears the Gateway Token, and the open closing PR is the right path to finish before this issue closes.

## What This Changes

Editing the Control UI Overview WebSocket URL clears the Gateway Token field and can make Connect fail unless the token is re-entered.

## Best Possible Solution

Refresh and land or replace https://github.com/openclaw/openclaw/pull/73923 with a focused Control UI fix that preserves tokens for same-normalized gateway edits and reloads or clears tokens when the gateway scope changes.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes: source inspection on current main shows the URL input clears settings.token for any non-identical trimmed URL, and current browser coverage asserts the token is empty after editing the gateway URL.

## Solution Assessment

Is this the best way to solve the issue? Yes: the linked scoped-fix direction is the maintainable solution because it preserves harmless same-endpoint edits without carrying a scoped credential to an unrelated endpoint.
