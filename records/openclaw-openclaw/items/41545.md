## Summary

Keep open. Current main and the latest release still clear the Overview Gateway Token when the WebSocket URL changes, and the active fix path is the open, unmerged PR at https://github.com/openclaw/openclaw/pull/73923.

## What This Changes

The issue requests that editing the Overview Gateway Access WebSocket URL preserve the Gateway Token unless the user explicitly edits or clears it.

## Best Possible Solution

Refresh and land or replace https://github.com/openclaw/openclaw/pull/73923 with the scoped Control UI resolver that preserves tokens for same-normalized gateway edits and reloads or clears tokens when the gateway scope changes.

## Reproduction Assessment

Yes. Current-main source inspection shows the URL input clears token on every non-identical trimmed URL edit, and current browser coverage asserts the token is empty after a gateway URL change.

## Solution Assessment

Yes. The linked PR's normalized-scope resolver is the narrow maintainable fix because it preserves harmless same-endpoint edits without reusing a token for an unrelated gateway.
