## Summary

Keep open. Current main still has the Overview WebSocket URL input handler clearing the in-memory Gateway Token on ordinary URL edits, current browser coverage still asserts that clearing behavior, and an open unmerged PR is already linked as the likely fix path.

## What This Changes

The issue reports that editing the Overview Gateway Access WebSocket URL immediately clears the Gateway Token field and causes Connect to fail unless the token is re-entered.

## Best Possible Solution

Review and land one canonical fix, preferably PR #73923 if it passes maintainer review: preserve the Gateway Token during same-normalized-endpoint URL edits, clear or reload it when the normalized gateway endpoint changes, and update the browser/node regression coverage plus changelog for the user-visible fix.

## Reproduction Assessment

Yes. Static current-main reproduction is high confidence: the Overview handler clears token on any typed URL value that differs from the prior trimmed value, and the issue already reports a frontend-only reproduction with no backend required.

## Solution Assessment

Yes for the scoped direction described by the open fix path. The safest maintainable solution is not unconditional token preservation, but preservation only within the same normalized gateway token scope, with different endpoints clearing or loading their own scoped token.
