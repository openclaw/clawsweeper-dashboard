## Summary

Keep open. Current main and v2026.5.3 still clear the Overview Gateway Token on WebSocket URL input, and open PR #73923 already tracks the scoped fix path.

## What This Changes

Editing the Overview Gateway Access WebSocket URL clears the Gateway Token field and can make Connect fail unless the token is re-entered.

## Best Possible Solution

Land or replace PR #73923 with a scoped fix that preserves tokens for same-normalized gateway endpoint edits and clears or reloads tokens when the gateway scope changes, with regression coverage and a changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows the WebSocket URL input immediately writes `token: ""` when the typed value differs from the current URL, and existing browser coverage asserts that clearing behavior; I did not run the UI in this read-only sweep.

## Solution Assessment

Yes. The scoped direction in PR #73923 is the narrowest maintainable fix because storage and docs make gateway tokens selected-gateway scoped rather than globally reusable across arbitrary URL changes.
