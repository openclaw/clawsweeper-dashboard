## Summary

Current main and the latest release still contain the source-level token-clearing behavior, and an open conflicting PR already references this issue as the fix path, so this issue should remain open rather than being closed or queued into another parallel repair.

## What This Changes

The issue reports that editing the Overview Gateway Access WebSocket URL clears the in-memory Gateway Token before Connect is clicked.

## Best Possible Solution

Refresh or replace the linked PR with an endpoint-aware fix that preserves the token during same-gateway URL edits while still clearing or reloading credentials when the normalized gateway endpoint truly changes.

## Reproduction Assessment

Yes by source inspection: the Overview URL input handler clears `settings.token` whenever the typed URL differs from the previous value. I did not run a browser repro because this review is read-only and tests can create artifacts in this checkout.

## Solution Assessment

No, current main is not the best state because it conflates every URL keystroke with a gateway endpoint change. The best fix should compare normalized endpoint scope or defer credential clearing until an actual gateway change is confirmed.
