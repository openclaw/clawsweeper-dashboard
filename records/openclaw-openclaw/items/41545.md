## Summary

Keep open. Current main and v2026.5.12 still clear the Overview Gateway Token while editing the WebSocket URL, and the open closing fix path is https://github.com/openclaw/openclaw/pull/73923.

## What This Changes

The issue requests that Overview preserve the Gateway Token while the WebSocket URL field is edited instead of blanking it immediately.

## Best Possible Solution

Refresh and land or replace https://github.com/openclaw/openclaw/pull/73923 with a scoped Control UI fix that preserves tokens for same-normalized endpoint edits and reloads or clears tokens when the gateway scope changes.

## Reproduction Assessment

Yes. Source inspection on current main shows the URL input handler assigns `token: ""` for any non-identical trimmed URL, and the current browser test asserts the token is empty after a gateway URL edit.

## Solution Assessment

Yes. The normalized-scope approach in the open fix path is the narrow maintainable solution because it preserves harmless same-endpoint edits without reusing a token for an unrelated gateway.
