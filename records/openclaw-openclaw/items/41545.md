## Summary

Keep open. Current main and v2026.5.2 still clear the Overview Gateway Token on WebSocket URL input, existing browser coverage still asserts that clearing behavior, and open PR #73923 is the unmerged canonical fix path.

## What This Changes

The report asks for the Overview Gateway Access WebSocket URL editor to stop clearing an already-entered Gateway Token unless the token scope actually changes.

## Best Possible Solution

Review and land PR #73923 or an equivalent canonical fix that preserves tokens for same-normalized gateway edits, clears or reloads tokens for different gateway scopes, and updates regression coverage plus changelog.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: entering a token and then typing a different WebSocket URL runs the Overview input handler that immediately writes token: "", and current browser coverage asserts that state transition.

## Solution Assessment

Yes for the current direction. The narrow maintainable fix is scoped preservation within the same normalized gateway endpoint, not unconditional token preservation across unrelated gateway URLs.
