## Summary

Keep open. Current main and v2026.5.19 still clear the Overview Gateway Token during WebSocket URL edits, and the live closing fix path at https://github.com/openclaw/openclaw/pull/73923 remains open but conflicting.

## What This Changes

The issue reports that editing the Overview Gateway Access WebSocket URL clears the Gateway Token and causes Connect to fail unless the token is re-entered.

## Best Possible Solution

Refresh and land or replace https://github.com/openclaw/openclaw/pull/73923 with a scoped resolver that preserves tokens for same-normalized gateway edits and loads or clears tokens when the gateway scope changes.

## Reproduction Assessment

Yes. Source inspection on current main shows the URL input handler clears the token for non-identical trimmed URL edits, and current browser coverage asserts that cleared-token state after editing the gateway URL.

## Solution Assessment

Yes. The normalized-scope direction in the linked PR is the maintainable fix because it avoids both accidental token loss on harmless edits and credential reuse across unrelated gateways.
