## Summary

Keep open. Current main and the latest release still contain the Overview token-clearing behavior, and an open closing PR at https://github.com/openclaw/openclaw/pull/73923 is the existing fix path but is currently conflicting.

## What This Changes

Editing the WebSocket URL in Overview Gateway Access should not clear the Gateway Token field unless the gateway scope actually changes.

## Best Possible Solution

Refresh and land or replace https://github.com/openclaw/openclaw/pull/73923 with a scoped resolver that preserves tokens for same-normalized gateway edits and loads or clears tokens when the gateway scope changes.

## Reproduction Assessment

Yes. Source inspection on current main shows the input handler clears token on non-identical trimmed URL edits, and current browser coverage asserts the token is empty after a gateway URL input event.

## Solution Assessment

Yes. The linked PR's normalized-scope direction is the narrow maintainable fix because it avoids both accidental token loss and credential reuse across unrelated gateways.
