## Summary

Keep open: the startup gate is a plausible fix direction, but the loader change introduces a stale global hook-runner path for ordinary default-mode plugin loads, and the external PR still needs stronger real behavior proof before merge.

## What This Changes

The PR gates RPC and HTTP `tools.invoke` during startup readiness and changes plugin loader activation so a gateway startup hook runner can survive later default/scoped plugin loads.

## Best Possible Solution

Keep the `tools.invoke` startup-unavailable gate, but constrain hook-runner preservation to a proven gateway-bindable registry path or track the runner origin explicitly, with regression coverage for ordinary default-mode reloads and the HTTP startup gate.

## Reproduction Assessment

Yes, by source inspection rather than a live timing probe: current main binds the HTTP server before post-attach sidecars are ready, and direct `/tools/invoke` currently lacks the unavailable-method check. A request in that window can reach `invokeGatewayTool` before startup hook runtime is attached.

## Solution Assessment

No. The `tools.invoke` gate is the right direction, but the count-based hook-runner preservation is too broad; it should preserve only a known gateway-bindable startup runner or reinitialize for ordinary default-mode plugin loads.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Do not preserve stale default hook runners:** `src/plugins/loader.ts:1502-1506`
  - body: The new count-based preservation path keeps the existing global hook runner whenever a later default-mode registry has fewer hooks, even if the existing runner also came from an ordinary default-mode load rather than gateway startup. Loading plugin A with a `before_tool_call` hook and then loading a different/default registry with no hooks would leave plugin A's hooks active while the active registry points at the new registry, so disabled or replaced hooks can keep gating tool calls. Restrict this preservation to a recorded gateway-bindable runner or reinitialize for normal default loads.
  - confidence: 0.89
