## Summary

Keep open. Current main and v2026.6.8 still have the Skill Workshop approval-wait versus Codex dynamic-tool watchdog mismatch, and the live comments tie this report to canonical issue #91266 while two open PRs need maintainer choice before the item can close.

## What This Changes

`skill_workshop action=apply` times out while waiting for owner approval on large Skill Workshop proposals, leaving the proposal pending and the live skill unchanged.

## Best Possible Solution

Land one focused timeout-boundary fix that lets Skill Workshop lifecycle approval finish before the Codex dynamic-tool watchdog while preserving default pending approval, then close this issue and the canonical issue with merged-commit and release provenance.

## Reproduction Assessment

Yes at source level: default `skill_workshop apply` requires pending approval, plugin approval can wait 120s plus gateway grace, and Codex dynamic-tool calls still default to 90s; live comments reproduce it with 25-28 KB proposals, but I did not run a live Gateway/chat approval session.

## Solution Assessment

The best fix appears to be aligning the Codex `skill_workshop` watchdog with the approval ceiling, and maintainers still need to choose how that interacts with the direct apply-timeout PR for large proposal review time.
