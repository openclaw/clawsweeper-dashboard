## What This Changes

The issue reports that `skill_workshop action=apply` times out while approving a roughly 28 KB skill proposal, leaving the proposal pending and the live skill unchanged.

## Best Possible Solution

Review and either land https://github.com/openclaw/openclaw/pull/94315 or replace it with an equivalent bounded fix that gives large Skill Workshop applies enough approval time without changing proposal storage semantics.

## Reproduction Assessment

Yes, at source level: current `main` requires approval for `skill_workshop action=apply`, and the approval runtime falls back to a 120 second timeout when no custom timeout is supplied. I did not execute the live 28 KB approval UI flow.

## Solution Assessment

Unclear for the issue alone: a per-apply timeout increase is a plausible narrow fix, but the open PR should be reviewed for real behavior proof and best-fix fit before this issue closes.
