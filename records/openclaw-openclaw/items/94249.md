## What This Changes

The issue reports that `skill_workshop action=apply` times out on large Skill Workshop proposals while leaving the proposal pending and the live skill unchanged.

## Best Possible Solution

Keep the issue open until maintainers land or replace the linked timeout fix path, preserving pending approval semantics and then closing with the merged commit and release provenance.

Yes at source level: current main still sends `skill_workshop` lifecycle apply through pending plugin approval without an apply-specific timeout, and the approval path returns `Approval timed out` when the wait resolves without a decision. I did not run a live large-proposal Gateway approval flow.

Unclear: the issue has two viable linked timeout-boundary fixes, and the newer comments argue the true root is the Codex watchdog waiting on approval rather than disk writes. Maintainers should choose whether to align the Codex watchdog, the Skill Workshop approval timeout, or both without weakening pending approval.
