## Summary

Keep open. Current main and v2026.6.8 still have the default Skill Workshop lifecycle approval timeout path, while the two candidate fixes are still open and partly overlap on whether the right fix is a longer approval window, a Codex watchdog alignment, or both.

## What This Changes

The issue reports that `skill_workshop action=apply` times out on delayed or large approval proposals instead of applying after owner approval.

## Best Possible Solution

Choose one coherent timeout contract that preserves pending Skill Workshop approval, align any Codex `skill_workshop` watchdogs with that contract, add focused regression coverage and live delayed-approval proof, then close this issue after the fix merges.

## Reproduction Assessment

Yes, at source level: current main routes `skill_workshop` lifecycle actions through pending approval with a 120s default and can return `Approval timed out`; live comments also report repeated delayed-approval failures. I did not run a live Gateway/chat approval flow in this read-only review.

## Solution Assessment

Unclear until maintainers choose the timeout contract: raising only the apply approval TTL is plausible, but it must be coordinated with Codex dynamic-tool watchdogs and the security boundary. The narrowest maintainable fix should preserve pending approval and avoid leaving overlapping timeout ceilings inconsistent.
