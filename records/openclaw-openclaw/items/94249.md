## Summary

Keep open. Current main and the latest release still route `skill_workshop apply` through the Skill Workshop lifecycle approval gate without an apply-specific timeout, so it inherits the 120-second plugin approval expiry; an open linked PR currently owns the direct fix path, and the broader related approval-watchdog issue remains open.

## What This Changes

`skill_workshop action=apply` times out through the agent-tool approval path when owner approval does not arrive before the current approval window expires.

## Best Possible Solution

Land one reviewed fix path that keeps Skill Workshop’s secure approval default while giving legitimate apply reviews enough time, then close this issue and preserve the broader timeout discussion on the canonical related issue if anything remains.

## Reproduction Assessment

Yes for source-level reproduction: current main routes `skill_workshop apply` through a plugin approval request with no custom timeout, so it falls back to the 120-second approval window. I did not run a live chat approval expiry in this read-only review.

## Solution Assessment

Unclear whether the open PR’s fixed 5-minute apply timeout is the best final shape; it addresses the observed expiry, but maintainers still need to choose the approval-window security posture for Skill Workshop lifecycle actions.
