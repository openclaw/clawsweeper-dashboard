## Summary

Keep open: the PR targets a real embedded-agent fallback regression, but the early abort runs before provider failover metadata is preserved, so it can suppress configured fallback for provider errors that wrap a session-lock cause; after-fix real behavior proof is also missing.

## What This Changes

The PR adds a failover-error predicate for session write-lock timeout and embedded session takeover errors, rethrows matching errors from `runWithModelFallback`, and adds unit coverage.

## Best Possible Solution

Land a narrowed guard that fails fast only for true local session coordination errors while preserving explicit `FailoverError` and provider status/code fallback semantics, with mixed-wrapper coverage and redacted real setup proof.

## Reproduction Assessment

Yes, source-level: current main throws `EmbeddedAttemptSessionTakeoverError` on post-prompt session-file mutation and `runWithModelFallback` continues non-last unknown errors to later candidates. I did not run a live gateway reproduction in this read-only review.

## Solution Assessment

No, not as proposed. The intended fail-fast direction is right, but the guard is too broad because it runs before the existing explicit provider metadata exception; the safer fix should mirror that precedence.
