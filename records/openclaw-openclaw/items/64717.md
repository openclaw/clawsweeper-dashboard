## What This Changes

The PR updates Telegram text chunk planning to use the configured `textChunkLimit` capped at 4096 and adds a focused 4096-limit send test.

## Best Possible Solution

Keep the fix in the Telegram plugin: resolve `channels.telegram.textChunkLimit`, clamp it to Telegram's 4096 maximum, use it consistently in the current chunk planner, and preserve rendered HTML plus plain-text fallback behavior.

## Reproduction Assessment

Yes. Configure `channels.telegram.textChunkLimit = 4096` and send a 4001-4096 character Telegram text; source inspection shows current main still plans chunks with literal `4000`, below the configured limit.

## Solution Assessment

Yes in direction, but not as submitted against current main. Reusing `resolveTextChunkLimit` is the narrow maintainable fix, but the branch must be rebased to preserve rendered HTML as planner input and pass `cfg` in the regression test.
