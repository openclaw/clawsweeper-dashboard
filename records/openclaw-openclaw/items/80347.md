## Summary

Keep open. Current main still has the scoped allowlist omission, and https://github.com/openclaw/openclaw/pull/80366 is an open closing PR for the exact fix, so this issue should remain open until that implementation lands or is superseded.

## What This Changes

The configure wizard should expose or prune stale provider model allowlist entries instead of leaving them in `agents.defaults.models` indefinitely.

## Best Possible Solution

Land or supersede https://github.com/openclaw/openclaw/pull/80366 with the narrow model-picker fix that includes stale same-provider configured keys in scoped cleanup while preserving unrelated providers.

## Reproduction Assessment

Yes, by source inspection. A provider-scoped configure run with `openrouter/elephant-alpha` in `agents.defaults.models` but absent from the catalog filters that key out of `scopeKeys`, so unselecting visible OpenRouter models cannot prune it.

## Solution Assessment

Yes. The linked PR's approach is the narrow maintainable fix: include same-provider configured keys in the existing picker scope instead of adding a broad cleanup flag or silently pruning config.
