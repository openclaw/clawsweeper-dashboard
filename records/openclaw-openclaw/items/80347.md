## Summary

Keep open. Current main still has a source-reproducible stale allowlist path, and the linked open fix PR is the active implementation candidate rather than a reason to close this issue before merge.

## What This Changes

The issue requests that `openclaw configure` let operators remove stale or delisted provider model entries from `agents.defaults.models` instead of preserving them indefinitely.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80366 or an equivalent narrow fix that includes stale same-provider allowlist keys in the picker and scoped write set while preserving unrelated providers.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: stale same-provider keys under `agents.defaults.models` are filtered out before the picker and are not included in `scopeKeys`, so scoped cleanup cannot delete them.

## Solution Assessment

Yes. The narrow maintainable solution is to show same-provider configured keys missing from the current catalog as removable configured choices and prune them only when unselected, rather than adding a broad cleanup flag or silently deleting config.
