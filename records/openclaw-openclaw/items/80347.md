## What This Changes

The issue asks for `openclaw configure` to let operators prune or clearly flag configured model entries that no longer appear in the selected provider's current catalog.

## Best Possible Solution

Land a narrow provider-scoped picker fix that includes stale same-provider configured keys in the visible/write scope while preserving unrelated provider entries.

## Reproduction Assessment

Yes, by source inspection. On current main, a stale same-provider key missing from the catalog is filtered out of provider-scoped `scopeKeys`, and the apply path only prunes keys inside that scope.

## Solution Assessment

Yes. The linked PR's approach of adding stale same-provider configured keys to the existing picker and scoped write set is narrower than a new cleanup mode and keeps unrelated providers protected.
