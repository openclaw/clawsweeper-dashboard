## Summary

Keep open: current `main` still has a source-level configure gap where stale same-provider `agents.defaults.models` entries missing from the provider catalog fall outside the picker scope, and an open linked PR already targets this issue with closing syntax.

## What This Changes

The issue asks `openclaw configure` to let operators remove stale or delisted model allowlist entries instead of leaving them in `agents.defaults.models`.

## Best Possible Solution

Resolve the linked PR: the desired end state is narrow provider-scoped cleanup that shows stale same-provider configured entries as removable while preserving unrelated providers.

## Reproduction Assessment

Yes, source-level. A focused `promptModelAllowlist` call with `preferredProvider: "openrouter"`, a catalog missing `openrouter/elephant-alpha`, and `agents.defaults.models` containing that key will exclude it from `scopeKeys`, so `applyModelAllowlist` cannot prune it.

## Solution Assessment

Yes. The narrow maintainable fix is to include same-provider configured keys missing from the catalog in both the picker options and scoped write set, then reuse the existing scoped pruning behavior.
