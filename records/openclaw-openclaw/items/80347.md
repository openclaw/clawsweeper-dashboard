## What This Changes

The issue asks for `openclaw configure` to surface or prune stale same-provider `agents.defaults.models` entries that disappeared from the provider catalog.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80366 if it remains correct; after it merges, close this issue as implemented by that PR.

## Reproduction Assessment

Yes. Source inspection shows that current main filters stale same-provider allowlist keys out of `scopeKeys`, and the scoped writer then preserves those out-of-scope keys.

## Solution Assessment

Yes. Adding same-provider configured keys to the provider-scoped picker scope is the narrowest maintainable fix, because it lets operators remove stale entries while preserving unrelated providers.
