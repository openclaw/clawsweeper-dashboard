## Summary

Keep open. Current main still has a source-reproducible path where stale same-provider model allowlist entries are omitted from the provider-scoped picker and write scope, and the linked fix PR is still open rather than merged.

## What This Changes

The issue asks `openclaw configure` to prune or visibly surface stale/delisted provider model allowlist entries instead of leaving `agents.defaults.models` append-only.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80366 if CI and maintainer review stay green; otherwise keep the fix focused in the existing model-picker scoped allowlist path.

## Reproduction Assessment

Yes. Source inspection on current main shows a preferred-provider configure run scopes selections to catalog keys, so a configured same-provider key that disappeared from the catalog is omitted from both the picker and deletion scope.

## Solution Assessment

Yes. The narrow maintainable fix is to show same-provider configured keys that are missing from the catalog as removable configured choices and include them in `scopeKeys`, instead of adding a broad cleanup flag or silently pruning all unknown models.
