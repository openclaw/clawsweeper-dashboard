## Summary

Keep open. Current main still leaves a source-reproducible stale same-provider allowlist path, and an open linked fix PR with closing syntax is the right item to review before this issue can close.

## What This Changes

The issue asks for `openclaw configure` to surface or prune stale same-provider model allowlist entries that are no longer present in the provider catalog.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/80366 so provider-scoped configure shows stale same-provider configured keys and prunes only the unselected keys in that scoped provider set.

## Reproduction Assessment

Yes. Source inspection shows the current-main path: provider-scoped configure builds `scopeKeys` from catalog entries, omits same-provider allowlist keys missing from the catalog, and `applyModelAllowlist` can only delete keys included in that scope.

## Solution Assessment

Yes. The best fix is the linked narrow picker-scope change: include same-provider configured keys that are missing from the catalog as removable choices, instead of silently pruning custom refs or adding a separate cleanup flag.
