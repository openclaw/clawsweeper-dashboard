## Summary

Keep open. Current main still has a source-reproducible configure/model-picker path that omits stale same-provider allowlist keys from the scoped write set, and the issue is already paired with the open fix PR at https://github.com/openclaw/openclaw/pull/80366.

## What This Changes

The issue asks `openclaw configure` to let operators prune or explicitly remove stale delisted model entries from `agents.defaults.models`.

## Best Possible Solution

Review and land the linked PR if CI stays green, keeping cleanup in the existing provider-scoped picker path so stale same-provider entries are visible and removable while unrelated providers are preserved.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a provider-scoped configure run builds `scopeKeys` from current catalog rows, so a configured same-provider model that disappeared from the catalog is neither offered as removable nor deleted by the scoped write.

## Solution Assessment

Yes. Extending the existing picker scope to include stale same-provider configured keys is narrower and safer than adding a separate cleanup flag or silently pruning all unknown models.
