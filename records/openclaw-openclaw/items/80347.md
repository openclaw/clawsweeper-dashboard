## Summary

Keep open: current main still has a provider-scoped configure path where stale same-provider model allowlist entries can be excluded from the picker and scope set, and an open linked PR is already targeting that exact bug.

## What This Changes

The issue asks for `openclaw configure` to surface and prune provider-scoped `agents.defaults.models` entries that disappeared from the current provider catalog instead of leaving them append-only.

## Best Possible Solution

Land or reject the linked provider-scoped picker fix; the end state should show same-provider configured-but-missing catalog entries as removable options and include them in `scopeKeys` without touching unrelated providers.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a preferred-provider configure run with an `agents.defaults.models` key absent from the current catalog filters that key out of `scopeKeys`, so it cannot be unselected or pruned.

## Solution Assessment

Yes. The narrow solution is to include same-provider configured-but-not-in-catalog keys in the provider-scoped picker/scope set; a new global cleanup flag would be broader than needed for the reported wizard bug.
