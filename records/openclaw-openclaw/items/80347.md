## Summary

Keep open. Current main still has the source-reproducible stale same-provider allowlist path, and the linked open fix PR at https://github.com/openclaw/openclaw/pull/80366 references this issue with closing syntax, so this issue should stay open until that implementation lands or is closed.

## What This Changes

The issue asks for `openclaw configure` to prune or visibly flag stale/delisted model allowlist entries instead of preserving them indefinitely.

## Best Possible Solution

Land the linked narrow picker-scope fix so stale same-provider configured keys appear as removable configured choices while unrelated provider allowlist entries remain preserved.

## Reproduction Assessment

Yes. Source inspection against current main shows a preferred-provider configure run builds `scopeKeys` from catalog entries only, then deletes only those scoped keys, so a configured same-provider key missing from the catalog remains unpruned.

## Solution Assessment

Yes. The narrow maintainable solution is to include same-provider configured keys missing from the catalog in the provider-scoped picker/write set, then prune them only when the operator unselects them.
