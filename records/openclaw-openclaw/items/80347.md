## What This Changes

The issue asks the configure wizard to prune or expose stale provider model allowlist entries that are no longer present in the upstream catalog.

## Best Possible Solution

Land the focused picker-scope fix if it stays green, so stale same-provider configured models are visible and removable without silently deleting unrelated provider entries.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: a preferred-provider configure run scopes the picker to catalog keys, so a configured same-provider key absent from the catalog is excluded from both the picker and deletion scope.

## Solution Assessment

Yes. The narrow maintainable direction is to include same-provider configured keys missing from the catalog as explicit stale/configured options and prune them only when the operator unselects them.
