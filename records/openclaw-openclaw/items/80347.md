## Summary

Keep open: current main still excludes stale same-provider configured model keys from provider-scoped cleanup, and the linked open fix PR is the active implementation path for this issue.

## What This Changes

The issue asks for `openclaw configure` to prune or visibly flag stale/delisted model entries instead of leaving `agents.defaults.models` append-only.

## Best Possible Solution

Land or update https://github.com/openclaw/openclaw/pull/80366 so provider-scoped configure cleanup includes same-provider configured keys that are missing from the current catalog while preserving unrelated providers.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure with a preferred provider, an active catalog model selected, and a stale same-provider `agents.defaults.models` key absent from the catalog; current main omits the stale key from `scopeKeys`, so it is not pruned.

## Solution Assessment

Yes. The linked PR’s approach is the narrow maintainable fix: add same-provider configured keys to the existing provider-scoped picker/write set instead of adding a separate cleanup mode.
