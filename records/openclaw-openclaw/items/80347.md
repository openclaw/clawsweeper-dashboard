Keep open. Current main still has the source-reproducible stale same-provider allowlist path, and an open linked fix PR uses closing syntax for this issue, so the issue should remain open until that PR or an equivalent fix merges.

## What This Changes

The issue asks configure's model allowlist flow to expose or prune stale provider model entries that disappeared from the provider catalog instead of preserving them indefinitely.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80366 or an equivalent narrow picker-scope fix, then close this issue after the merge.

## Reproduction Assessment

Yes. Source inspection on current main shows a high-confidence path: stale same-provider keys absent from the catalog are omitted from provider-scoped scopeKeys, and scoped writes only delete keys in that set.

## Solution Assessment

Yes. The best fix is the linked narrow configure/model-picker change that includes same-provider configured missing-catalog keys as removable scoped options, rather than adding a separate cleanup flag or silently pruning.
