## Summary

Keep open. Current main still has a source-reproducible path where provider-scoped configure runs omit same-provider allowlist keys that disappeared from the current catalog, so those stale entries cannot be unselected and survive the scoped write.

## What This Changes

The issue asks the configure wizard to prune or visibly flag stale/delisted model allowlist entries instead of preserving them indefinitely.

## Best Possible Solution

Make provider-scoped model allowlist prompts include stale same-provider configured allowlist keys as selected `configured (not in catalog)` options, preserve unrelated providers, and prune stale entries only when the operator unselects them.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: a provider-scoped configure run derives scope from catalog rows, filters configured same-provider keys missing from that catalog out of the picker, and then deletes only keys included in that scope.

## Solution Assessment

Yes. Extending the existing configured/not-in-catalog option path for same-provider stale keys is the narrowest maintainable fix; silent pruning or a new cleanup flag would add more policy surface than needed.
