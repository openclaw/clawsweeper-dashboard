## Summary

Keep open. Current main still has a source-reproducible path where provider-scoped configure selections omit configured same-provider keys that are absent from the current catalog, then preserve those omitted keys during the scoped allowlist write.

## What This Changes

The issue asks for `openclaw configure` to remove or surface stale/delisted entries in `agents.defaults.models` instead of preserving them indefinitely.

## Best Possible Solution

Make provider-scoped model allowlist prompts include same-provider configured keys that are not in the current catalog as stale/configured options, include them in the scoped write set, and prune only when the operator unselects them.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a preferred-provider configure run scopes the multi-select to catalog keys, so a configured same-provider key that disappeared from the catalog is excluded from both the picker and deletion scope.

## Solution Assessment

Yes. The narrow maintainable fix is to extend the existing scoped picker behavior rather than add a new cleanup flag or silently prune config entries.
