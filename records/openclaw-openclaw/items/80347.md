## What This Changes

The issue asks for `openclaw configure` to stop preserving stale provider model entries indefinitely after a provider catalog no longer offers them.

## Best Possible Solution

Land the narrow linked configure/model-picker fix so stale same-provider allowlist entries are shown as removable configured choices and pruned only when the operator unselects them.

## Reproduction Assessment

Yes. Source inspection shows a preferred-provider configure run can omit stale same-provider `agents.defaults.models` keys from both the picker options and `scopeKeys`, so `applyModelAllowlist` preserves them.

## Solution Assessment

Yes. The best path is the existing linked PR, which extends the current provider-scoped picker/write-set behavior instead of adding a broad cleanup flag or silently pruning config.
