## Summary

Keep open. Current main still has the source-reproducible stale same-provider allowlist path, and an open linked PR already targets this issue with the narrow fix, so the issue should remain open until that PR is reviewed and merged or closed.

## What This Changes

The issue asks `openclaw configure` to stop preserving stale provider model entries that are no longer present in the current upstream catalog.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/80366; the desired end state is for provider-scoped configure to show stale same-provider configured entries and prune them only when the operator unselects them.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: a provider-scoped configure run derives the picker and deletion scope from current catalog entries, so a configured same-provider model missing from that catalog is neither shown nor deleted.

## Solution Assessment

Yes. The narrow maintainable solution is to extend the existing provider-scoped picker/write scope to include stale same-provider configured keys, rather than adding a broad cleanup flag or silently pruning every non-catalog model.
