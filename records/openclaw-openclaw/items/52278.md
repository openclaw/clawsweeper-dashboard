## Summary

Keep open: current main still only repairs one trailing user leaf, so the central bug is not implemented on main. The PR is useful, but it still has a blocking image-dedupe regression, lacks contributor real behavior proof, and is currently conflicting against the newer helper structure.

## What This Changes

The PR changes embedded-runner orphan repair to walk multiple trailing user leaves, carry recovered text and images into the next prompt, and add focused helper tests.

## Best Possible Solution

Land a current-main port of multi-leaf orphan recovery in the embedded message-merge/helper boundary, with post-sanitize image dedupe, focused regressions, and contributor-supplied real behavior proof.

## Reproduction Assessment

Yes, source-reproducible. Current main only reads and repairs one leafEntry, so a chain with two trailing user leaves cannot have both orphaned turns carried into the next prompt.

## Solution Assessment

No. The recovery direction is useful, but this branch should be ported to current main and should dedupe recovered/current images after sanitization before merge.
