## Summary

Keep open. Current main has merged the narrower #65478 single-leaf orphan prompt preservation, but it does not implement this PR's full chain-walk recovery or repeated orphan-turn handling; the PR branch is useful but currently dirty and blocked by a concrete image dedupe defect.

## What This Changes

The PR modifies embedded-runner attempt logic and tests to recover trailing orphaned user turns, carry their text and images into the next prompt, and dedupe retry echoes while preserving repeated real turns.

## Best Possible Solution

Port the useful multi-leaf orphan recovery onto current main's message-merge helper boundary, keep #65478's single-leaf behavior as the baseline, and add canonical post-sanitization image dedupe with tests for multi-leaf ordering, repeated text, image order, and sanitized duplicate retries.

## Reproduction Assessment

Yes for the blocking PR defect: a vision retry can combine a recovered sanitized orphan image with the current unsanitized params.images attachment, and the PR's raw mimeType+data dedupe cannot recognize them before sanitization. For the broader feature gap, current main's one-leaf repair path is visible in source and does not walk a trailing user chain.

## Solution Assessment

No. The recovery direction is maintainable, but this branch is not the best mergeable fix until it is adapted to current main's helper split and image dedupe uses the same canonical post-sanitization representation that will actually be sent to the provider.

## Review Findings

Full review comments:

- **[P2] Deduplicate recovered images after sanitization:** `src/agents/pi-embedded-runner/run/attempt.ts:363`
  - body: `mergeRecoveredPromptImages` dedupes recovered/current attachments by raw `mimeType` and `data` before `detectAndLoadPromptImages` sanitizes or resizes them. On retry, the recovered orphan can already contain sanitized bytes from the previous attempt while `params.images` still has the original attachment, so both survive the raw-key check and are sent as duplicate logical images. Move the duplicate check to the sanitized image list or use the sanitizer's canonical key.
