## Summary

Keep open: the latest head fixes the prior custom-token regression and the patch looks code-correct, but it still needs maintainer acceptance because it broadens an exported silent-reply helper that affects channel delivery and overlaps with another open matcher PR.

## Best Possible Solution

Land or fold one shared classifier policy that preserves exact custom-token matches, suppresses maintainer-approved edge-punctuated token-only replies, and keeps the substantive-reply guards from the earlier strict matcher fix.

## Solution Assessment
