## Summary

Keep open. Current main still does not implement a SQLite session-store backend, but the PR is not merge-ready because the latest diff still has data-integrity defects around SQLite fallback and migration failure handling, plus user-facing docs/changelog follow-through is missing.

## What This Changes

The PR adds opt-in `session.storeType: "sqlite"` support with a SQLite session metadata store, JSON-to-SQLite migration helpers, `openclaw sessions migrate/store-info` commands, config schema/help updates, SQLite store tests, and a repository PR description file.

## Best Possible Solution

A mergeable solution should keep one authoritative source of truth in SQLite mode, surface migration failures explicitly, normalize legacy JSON rows before import, retry migrations only after successful completion, and update docs plus the active changelog for the new config and CLI surface.

## Reproduction Assessment

Yes for the blocking PR defects: configure sqlite mode and force `saveSessionStoreSqlite` or JSON-to-SQLite migration to fail; the diff shows writes can land in JSON while reads continue to prefer SQLite, and failed migrations can be reported as zero migrated rows. I did not run tests because this review was read-only.

## Solution Assessment

No, not as-is. The high-level performance goal is valid, but this patch should not merge until SQLite mode has strict source-of-truth semantics, migration failures are observable, legacy rows are normalized, and the branch is reconciled with current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Avoid fallback JSON writes after SQLite save failures:** `src/config/sessions/store-facade.ts:152-154`
  - body: In sqlite mode this catch writes updates to `sessions.json` when SQLite throws, but future reads still prefer SQLite as soon as it is readable again. A transient `SQLITE_BUSY` or filesystem error can therefore acknowledge a save that disappears on the next load because the SQLite DB remains stale. Propagate the error or make the source-of-truth switch explicit and durable.
  - confidence: 0.93
- **[P1] Surface JSON-to-SQLite migration failures:** `src/config/sessions/store-sqlite.ts:603-605`
  - body: `migrateJsonToSqlite` rolls back and then converts every migration failure into `0`, which callers treat like a normal empty store. On a failed first migration, the facade can proceed to read SQLite and show an empty or stale store instead of falling back to JSON or failing the CLI migration. Let the error propagate or return an explicit failure result.
  - confidence: 0.91
- **[P2] Mark migration attempted only after success:** `src/config/sessions/store-facade.ts:83`
  - body: `migrationAttempted` is set before the migration runs. If the first attempt fails or imports zero rows because of a transient read/SQLite problem, later loads in the same process skip the JSON-to-SQLite import and can keep serving an empty or stale database until restart. Record the memo only after a verified successful or unnecessary migration.
  - confidence: 0.86
- **[P2] Apply session-store migrations before importing rows:** `src/config/sessions/store-sqlite.ts:574-589`
  - body: The existing JSON loader runs `applySessionStoreMigrations` and normalization before callers use entries, but this migration inserts raw parsed JSON directly. Older stores with fields like `provider` or `lastProvider` will get null indexed `channel`/`last_channel` values, so routing/list metadata can be wrong until each session is rewritten. Reuse the current migration/normalization path before binding rows.
  - confidence: 0.85
- **[P3] Add the required changelog and docs:** `src/cli/program/register.status-health-sessions.ts:226`
  - body: This user-facing feature adds a config key and new `openclaw sessions` commands, but the diff does not update `CHANGELOG.md` or the session CLI/config docs. OpenClaw policy requires user-facing `feat` changes to be documented and recorded under the active version with contributor credit.
  - confidence: 0.94
