## Summary

Keep open. Current main still exposes brew-only skill installs in Docker/Linux environments without providing Linuxbrew, an apt/native fallback for brew-only specs, or container-aware onboarding filtering, and multiple open PRs still target this same failure mode.

## What This Changes

The issue asks OpenClaw to make Docker onboarding handle brew-based skill dependencies instead of failing when Homebrew is absent in the Linux container.

## Best Possible Solution

Keep this issue open while maintainers review and either land or close the active fix candidates, then align the chosen path with Docker skill-dependency regression coverage and updates to https://docs.openclaw.ai/install/docker and the skills docs.

## Reproduction Assessment

Yes. The high-confidence path is Docker onboarding selecting a brew-only skill such as openai-whisper when brew is absent; current main still offers the generic Homebrew prompt and installSkill still returns the brew-missing failure for that spec.

## Solution Assessment

No. Closing is not the best way to solve this now because current main still lacks the requested Docker/Linux path and open PRs with closing syntax are already under review; the narrowest maintainable fix needs maintainer selection among those candidates.
