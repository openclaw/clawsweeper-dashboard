## What This Changes

The issue reports that Docker onboarding can offer brew-based skill installs even though the official Linux container lacks Homebrew, causing selected skills such as openai-whisper to fail.

## Best Possible Solution

Keep this as the canonical report while maintainers choose one Docker skill-dependency strategy, land a focused fix with regression coverage, and update the Docker and skills docs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Docker lacks brew, onboarding can select a brew-only skill such as openai-whisper, and installSkill returns the missing-brew failure when no brew executable resolves.
