## Summary

Keep open. Current main still has the Docker brew-missing path for brew-only skill installs, and multiple open implementation PRs still target this issue, so it is not cleanup-ready.

## What This Changes

Docker onboarding and skill installation should handle brew-based skill dependencies in Linux containers instead of failing with `brew not installed`.

## Best Possible Solution

Keep this issue as the canonical report while maintainers select one Docker skill-dependency strategy, land a focused fix with regression coverage, and update the Docker and skills docs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the Docker image lacks Linuxbrew, onboarding can select the brew-only `openai-whisper` installer, and `installSkill` returns the missing-brew failure when no brew executable resolves.

## Solution Assessment

No for cleanup or closure. Current main still has the failure path, and the best fix requires maintainer selection among open PRs with different Docker, package-manager, and supply-chain tradeoffs.
