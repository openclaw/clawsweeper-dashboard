## Summary

Keep open. Current main and the latest release still expose the WSL2 installer path, run a global npm install, and accept Node 22.14+, so the reported Node 22.22.x V8 crash is not obsolete. The Node 24 default helps clean installs where Node is missing, but it does not prove a fix for systems where Node 22.22.x is already active or isolate whether the npm lifecycle path triggers the crash.
