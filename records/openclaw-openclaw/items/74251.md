## Summary

Keep open. Current main and the latest release still expose the WSL/Linux installer path that runs global npm install, still accept Node 22.14+ including the reported Node 22.22.x range, and do not contain a targeted WSL2/V8 crash workaround. The Node 24 default may improve clean installs where Node is missing, but it does not prove the reported crash is fixed for systems with Node 22.22.x already active.

## What This Changes

The issue reports that the official WSL2 Ubuntu 24.04 installer path fails during `npm install -g openclaw@latest` with V8 fatal `unreachable code` and Turboshaft crashes under Node 22.22.2.
