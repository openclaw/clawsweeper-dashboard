## Summary

Keep open. Current main still supports the reported Node 22.22.x runtime, still routes the Linux/WSL installer through a global npm install, and still runs OpenClaw package postinstall lifecycle code during that install. The Node 24 default helps only when the installer has to install Node itself; it does not prove a fix for already-active Node 22.22.x environments, and duplicate #74277 was closed back to this issue as the canonical report.

## What This Changes

The issue reports that the one-line Linux/WSL2 installer crashes during `npm install -g openclaw@latest` on WSL2 Ubuntu 24.04 with Node 22.22.x V8/Turboshaft fatal errors.
