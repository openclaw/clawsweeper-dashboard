## What This Changes

The issue requests that `/v1/chat/completions` accept inbound `max_completion_tokens` and legacy `max_tokens` and forward the chosen value as `streamParams.maxTokens`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80867 with real behavior proof, then close this issue after the fix reaches `main`.

Yes, source inspection gives a high-confidence reproduction path: send `max_completion_tokens` or `max_tokens` to `/v1/chat/completions` and inspect the mocked `agentCommandFromIngress` input, which currently has no `streamParams.maxTokens`.
