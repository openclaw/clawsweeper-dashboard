## Summary

Current main still lacks the requested cron CLI warning, and the linked same-author fixing PR remains open with closing syntax, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Warn users when main-session cron `systemEvent` payload text looks like a shell command because that text is enqueued as a system event rather than executed.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/63112, or an equivalent narrow pre-RPC CLI warning that preserves current main-session `systemEvent` runtime semantics.

## Reproduction Assessment

Yes. Source inspection shows `cron add --system-event "python3 foo.py" --session main` reaches `cron.add` without a warning, while main-session execution enqueues the text as a system event and can persist an ok run.

## Solution Assessment

Yes. A non-blocking pre-RPC warning is the narrow maintainable solution because executing shell-looking `systemEvent` text would change cron safety and runtime semantics.
