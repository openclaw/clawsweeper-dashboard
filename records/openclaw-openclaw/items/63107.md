## Summary

Keep open. Current main still lacks the requested pre-RPC cron CLI warning, and the linked same-author fix PR #63112 is still open and unmerged with closing syntax for this issue.

## What This Changes

Add a non-blocking stderr warning for cron add/edit when a main-session systemEvent payload looks like a shell command.

## Best Possible Solution

Keep main-session systemEvent delivery non-executable and add a shared cron add/edit warning helper before `cron.add` or `cron.update` when shell-looking text targets the main session.

## Reproduction Assessment

Yes. The source path is clear: `cron add --system-event "python3 foo.py" --session main` resolves to a main systemEvent payload and reaches `cron.add` without a warning, while runtime dispatch enqueues the text as a system event.

## Solution Assessment

Yes. A warning-before-RPC fix is the narrow maintainable solution; executing shell-looking systemEvent text from the main session would change the safety model.
