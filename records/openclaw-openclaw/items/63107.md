## Summary

Keep open: current main and v2026.6.10 have first-class command cron jobs, but the old main-session systemEvent CLI path still accepts shell-looking text without the requested warning; the open linked PR remains the implementation candidate.

## What This Changes

The issue asks for a non-blocking cron add/edit warning when main-session systemEvent payload text looks like a shell command.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/63112 or a successor lands a narrow add/edit warning that points shell-script users at --command or --command-argv without changing main-session runtime semantics.
