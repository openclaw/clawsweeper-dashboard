## Summary

Keep open. Current main still lacks the requested cron CLI warning, and the same-author fixing PR is open, unmerged, and explicitly linked as the implementation path.

## What This Changes

The issue requests a non-blocking `cron add`/`cron edit` warning when a main-session `systemEvent` payload looks like a shell command.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/63112, or an equivalent narrow CLI warning that preserves main-session `systemEvent` semantics.

## Reproduction Assessment

Yes. Source inspection shows `cron add --system-event "python3 foo.py" --session main` reaches `cron.add` without a warning, while main-session cron execution enqueues the text as a system event and can record an ok run.

## Solution Assessment

Yes. A pre-RPC warning is the narrow maintainable solution because executing shell-looking `systemEvent` text would change the cron safety and runtime model.
