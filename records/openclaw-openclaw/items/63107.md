## Summary

Keep open. Current main still lacks the requested pre-RPC cron CLI warning, and #63112 remains an open, unmerged same-author implementation candidate that links this issue with closing syntax.

## What This Changes

Add a non-blocking stderr warning when cron add/edit resolves to a main-session systemEvent payload whose text looks like a shell command.

## Best Possible Solution

Cron add/edit should share a small helper that warns to stderr before cron.add/cron.update when the resolved target is main, the payload is systemEvent, and the text looks like a shell command; main-session systemEvent semantics should remain event delivery, not shell execution.

## Reproduction Assessment

Yes. A source-level reproduction path is clear: `cron add --system-event "python3 foo.py" --session main` resolves to a main systemEvent payload and reaches `cron.add` without any warning logic.

## Solution Assessment

Yes, the proposed warning-before-RPC direction is the narrow maintainable fix. Executing shell text from main-session systemEvent payloads would change the safety model; a focused CLI warning with tests is safer.
