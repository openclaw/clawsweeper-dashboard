## Summary

Current main still lacks the requested cron CLI warning, and the same-author fixing PR is still open with closing syntax, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Request a non-blocking cron CLI warning when a main-session `systemEvent` payload text looks like a shell command because the text is enqueued as a system event rather than executed.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/63112, or an equivalent narrow pre-RPC cron add/edit warning that preserves current main-session `systemEvent` runtime semantics.

## Reproduction Assessment

Yes, at source level. Current main sends `cron add/edit` main-session shell-looking `systemEvent` payloads to the gateway without warning, while the runtime enqueues the text as a system event and can record an ok handoff.

## Solution Assessment

Yes. A non-blocking pre-RPC warning is the narrow maintainable solution because executing shell-looking `systemEvent` text would change cron safety and runtime semantics.
