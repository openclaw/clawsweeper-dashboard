## Summary

Keep open. Current main still lacks the requested pre-RPC cron CLI warning, and the linked same-author implementation PR #63112 remains open and unmerged with closing syntax for this issue.

## What This Changes

The issue requests a non-blocking stderr warning when cron add/edit targets the main session with a systemEvent payload whose text looks like a shell command.

## Best Possible Solution

Cron add/edit should share a focused helper that warns to stderr before `cron.add` or `cron.update` when the resolved target is main, the payload is systemEvent, and the text matches shell-command heuristics; main-session systemEvent behavior should remain event delivery rather than shell execution.

## Reproduction Assessment

Yes. The source-level path is clear: `cron add --system-event "python3 foo.py" --session main` resolves to a main systemEvent payload and reaches `cron.add` without warning, while runtime dispatch enqueues the text as a system event.

## Solution Assessment

Yes. A warning-before-RPC fix is the narrow maintainable solution; executing shell-looking text from main-session systemEvent payloads would change the safety model and should not be used to solve this report.
