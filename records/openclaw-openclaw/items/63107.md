Keep open. Current main still lacks the requested cron CLI warning, and https://github.com/openclaw/openclaw/pull/63112 remains open with closing syntax as the active implementation path.

## What This Changes

Request a non-blocking cron add/edit warning when a main-session systemEvent payload looks like a shell command.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/63112, or an equivalent narrow pre-RPC CLI warning that preserves current main-session systemEvent semantics.

## Reproduction Assessment

Yes. Source inspection shows `cron add --system-event "python3 foo.py" --session main` reaches `cron.add` without a warning, while main-session execution enqueues the text as a system event and can record an ok run.

## Solution Assessment

Yes. A non-blocking pre-RPC warning is the narrow maintainable solution because executing shell-looking systemEvent text would change cron safety and runtime semantics.
