## Summary

OpenClaw currently defines configured model prices, calculated usage, persisted session estimates, and displayed totals as USD, so the requested capability is not implemented and has no canonical replacement. This should remain open for an explicit product decision, but the proposed provider-level symbol switch is not safe because it would relabel USD accounting and make cross-provider aggregates invalid.

## What This Changes

Add a configuration option that lets model cost outputs display a user-selected currency instead of always using USD.

## Best Possible Solution

Either retain USD as the sole canonical accounting currency and require operators to enter converted USD rates, or approve a broader design that attaches ISO 4217 currency to model pricing and preserves correctness through conversion or currency-separated totals across transcripts, sessions, CLI output, gateway APIs, and Control UI.

## Reproduction Assessment

Not applicable: the dollar display is consistent with the documented USD pricing contract, so this is a request for new multi-currency behavior rather than a reproducible bug.

## Solution Assessment

No. A symbol-only provider or global option would misstate the existing USD values and break aggregate meaning; currency must be addressed at the canonical pricing and accounting boundary.
