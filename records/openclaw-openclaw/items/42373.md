## Summary

Current main still lacks currency metadata for model pricing and cost display. The pricing schema, runtime cost config, public/session fields, status/usage output, UI/macOS formatters, metrics, and docs remain explicitly USD or dollar-formatted, so this is still valid but needs maintainer design before a repair PR.

## What This Changes

The issue requests a configurable cost display currency, such as a provider-level `costCurrency`, so locally configured model costs can be shown as CNY/EUR/USD instead of always dollars.

## Best Possible Solution

Keep this open for a maintainer-led pricing-currency design. The end state should either explicitly keep OpenClaw USD-only or add backward-compatible currency metadata through config/schema/types, pricing lookup and aggregation, session/API payloads, CLI/chat/status output, Control UI/macOS formatting, diagnostics metrics, docs, and focused tests for mixed-currency behavior.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence verification path for the current gap: source inspection shows no accepted currency config key and all cost display/metrics paths remain USD-shaped.

## Solution Assessment

Unclear that the proposed per-provider display symbol alone is the best fix. A maintainable solution needs a product decision about per-provider versus per-model currency, global display currency, conversion, and mixed-currency summaries before implementation.
