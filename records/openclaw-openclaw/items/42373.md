## Summary

## What This Changes

The issue requests a `costCurrency` option so model cost displays can use non-USD symbols such as CNY or EUR.

## Best Possible Solution

Choose one canonical currency contract first; if accepted, update schema, docs, tests, status, `/usage`, CLI summaries, Gateway usage surfaces, and Control UI formatting while preserving USD as the default.

## Reproduction Assessment

Yes, at source level: current schema and docs still define model cost as USD, and cost renderers still emit dollar-prefixed labels. No live run was needed because the relevant config and formatting paths are explicit in source.

## Solution Assessment

Unclear. A per-provider `costCurrency` field is plausible, but maintainers need to decide whether the permanent contract is provider pricing metadata, user display preference, or conversion-aware accounting.
