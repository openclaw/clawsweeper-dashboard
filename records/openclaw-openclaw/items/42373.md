## Summary

Keep open: current main still has a USD-only pricing contract and dollar-only display paths, and the requested currency option is still missing. This is useful, but it needs maintainer product/API direction because it changes public config and mixed-currency cost semantics.

## What This Changes

The issue requests a new `costCurrency` configuration option so OpenClaw can display model cost estimates in currencies such as CNY or EUR instead of always USD.

## Best Possible Solution

Define a backward-compatible currency contract across model config, provider/plugin metadata, usage aggregation, SDK/API fields, CLI/status output, UI formatting, and docs, or explicitly keep model pricing USD-only.

## Reproduction Assessment

Not applicable: this is a feature/config request, not a broken existing behavior report. Source inspection verifies current main has no currency config key and still documents and formats costs as USD.

## Solution Assessment

No: the proposed per-provider display symbol is not enough until maintainers decide currency scope, mixed-currency aggregation behavior, and `costUsd` compatibility. The maintainable fix needs a typed currency contract or an explicit decision to keep costs USD-only.
