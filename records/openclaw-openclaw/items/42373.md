## What This Changes

The issue requests a new `costCurrency` configuration option so cost displays can show currencies such as CNY or EUR instead of the current dollar sign.

## Best Possible Solution

Maintainers should choose one canonical currency/accounting model first; if accepted, implement it across config schema/types/help/docs, usage estimation fields, status/CLI output, and Control UI formatters with USD as the default.

## Reproduction Assessment

Yes. Current source shows no `costCurrency` key, strict numeric USD pricing in model config, and dollar/USD formatting in status and Control UI paths.

## Solution Assessment

Unclear. A per-provider currency field is plausible, but maintainers need to choose whether the permanent contract is pricing metadata, user display preference, or conversion-aware accounting before implementation.
