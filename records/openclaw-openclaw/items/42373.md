## Summary

Keep open: current main still has USD-shaped pricing contracts and dollar-only display paths, and adding `costCurrency` needs a maintained config/API decision rather than conservative cleanup.

## What This Changes

Requests a `costCurrency` config option so configured model costs can display as CNY, EUR, USD, or another currency symbol instead of always dollars.

## Best Possible Solution

Define a backward-compatible pricing currency contract across model config, plugin manifest metadata, usage totals, SDK/diagnostic fields, and display surfaces, or explicitly document model pricing as USD-only.
