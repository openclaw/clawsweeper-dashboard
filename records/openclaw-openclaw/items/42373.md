## Summary

Keep open: current main still has USD-only pricing contracts, dollar-only cost display, and no `costCurrency` surface; this is a feature/config design request that needs maintainer API and product judgment before implementation.

## What This Changes

Add a `costCurrency` configuration option so configured model costs can display as currencies such as CNY or EUR instead of always using USD/dollar formatting.

## Best Possible Solution

Design a backward-compatible currency model for pricing metadata, totals, SDK/API fields, metrics, docs, and display surfaces, or explicitly document configured model pricing as USD-only.

## Solution Assessment

Unclear: a per-provider display symbol alone is likely not enough because usage totals and public fields aggregate costs as USD. Maintainers should decide currency scope, mixed-currency semantics, and compatibility before implementation.
