## Summary

Keep open: current main still has no currency-aware pricing config and continues to expose USD/dollar-shaped cost contracts. This is a user-facing config/API feature that needs maintainer product direction, not a cleanup close.

## What This Changes

The issue requests a configurable cost display currency, such as CNY or EUR, for model pricing values that are not USD.

## Best Possible Solution

Define a backward-compatible pricing currency contract across model config, provider/plugin metadata, usage aggregation, diagnostics, SDK/metrics, and display surfaces, or explicitly document model pricing as USD-only.

## Solution Assessment
