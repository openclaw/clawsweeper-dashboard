## Summary

Keep open: current main still has no currency-aware pricing config and the cost contract is explicitly USD-shaped across config, display, SDK, and telemetry surfaces. This is a user-facing config/API feature that needs maintainer product direction before implementation.

## What This Changes

The issue requests a `costCurrency` config option so model costs can display currencies such as CNY or EUR instead of always showing USD-style dollar amounts.

## Best Possible Solution

Define a backward-compatible pricing currency contract across model config, plugin/provider metadata, usage aggregation, diagnostics, SDK/metrics, and display surfaces, or explicitly document model pricing as USD-only.

## Reproduction Assessment

Not applicable: this is a feature/config request, not a bug. Source inspection on current main shows no currency config key and multiple dollar/USD-shaped display and API paths.

## Solution Assessment

Unclear: a per-provider display symbol is not enough until maintainers decide currency scope, mixed-currency aggregation behavior, and compatibility for existing `costUsd` surfaces.
