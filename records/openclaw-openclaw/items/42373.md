## Summary

Keep open: current main and v2026.6.6 still treat configured model costs as USD and render cost summaries with dollar-specific formatters. The request remains a low-priority product/config feature because maintainers need to choose whether currency is display-only, pricing-unit metadata, or something broader.

## What This Changes

The issue requests a `costCurrency` configuration option so model cost displays can use non-USD symbols such as CNY or EUR.

## Best Possible Solution

Decide one canonical currency contract first; if accepted, add schema, docs, tests, and shared formatting across status, `/usage`, CLI, Gateway usage summaries, and Control UI while preserving USD as the default.

## Reproduction Assessment

Yes. Source inspection reproduces the limitation: the config schema has no currency field, docs define configured pricing as USD, and status/CLI/UI formatters hardcode dollar output.

## Solution Assessment

Unclear. A per-provider `costCurrency` field is plausible, but maintainers should first choose whether this is display-only metadata or a canonical pricing-unit contract that flows through all cost aggregation surfaces.
