## Summary

Keep open: current main still has no currency-aware model pricing config, while config schemas, docs, SDK usage fields, CLI/status output, and Control UI formatting remain USD/dollar-oriented. ## What This Changes

The issue requests a new `costCurrency` model-provider configuration option so cost displays can use symbols such as `¥` or `€` instead of always showing dollars.

## Best Possible Solution

Define a backward-compatible currency contract across model config, provider/plugin metadata, usage aggregation, SDK/API fields, CLI/status output, UI formatting, and docs, or explicitly keep model pricing USD-only.
