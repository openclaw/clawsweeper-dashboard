## Summary

Keep open: current source and the beta package have the right packaging guardrails, but npm `latest` still serves the broken `@openclaw/discord@2026.5.2` artifact, so stable `openclaw update` users can still hit the Discord load failure.

## What This Changes

The issue asks for the Discord plugin npm package/update path to install a built runtime package so Discord loads after update and appears in channel status.

## Best Possible Solution

Publish or promote a fixed stable `@openclaw/discord` package with built `dist` runtime entries and `runtimeExtensions`, verify npm `latest`, then close after the user-visible update path is fixed.

## Reproduction Assessment

Yes for artifact-level reproduction: current npm `latest` is still `@openclaw/discord@2026.5.2` with TypeScript-only entries and no built runtime entry. I did not run a live gateway update, but registry metadata, tarball contents, and current install validation establish the failure path with high confidence.

## Solution Assessment

Yes, the best fix is to use the existing package-local runtime build plus `runtimeExtensions` publish path and get a fixed stable artifact onto npm `latest`. Replacing source `extensions` with dist paths in source metadata is not necessary under the documented contract and would not by itself fix already-published stable artifacts.
