Keep open: current main and the latest release still freeze the file-log hostname at module load, while this PR fixes the same write path with focused regression coverage. The older related PR is still open but has blocking review feedback, so this PR remains the stronger landing candidate.

## What This Changes

The PR replaces the module-level file-log hostname constant with lazy recovery/caching during JSONL writes, aligns `_meta.hostname`, and adds an empty-then-real hostname regression test.

## Best Possible Solution

Land this narrower PR after required checks and maintainer review, then close the linked bug and retire the older blocked duplicate PR if this path is chosen.

## Reproduction Assessment

Yes for source-level reproduction. Current main freezes `os.hostname()` into a module-level `HOSTNAME`, and the PR test models the failing empty-first, real-later sequence without needing to reproduce the intermittent macOS timing live.

## Solution Assessment

Yes. Resolving in the file-log write path, caching only the first real hostname, and copying that value into `_meta.hostname` is narrower and more maintainable than import-order or environment bootstrap workarounds.
