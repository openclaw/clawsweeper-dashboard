## Best Possible Solution

Land the lazy per-write hostname resolution after required checks and maintainer review, preserving the `unknown` fallback and existing log format.

## Reproduction Assessment

Yes. Current main freezes `os.hostname()` at module load, and the PR regression test models an empty first lookup followed by a real hostname without needing the intermittent macOS timing live.

## Solution Assessment

Yes. Resolving the hostname inside the file-log write path and reusing that value for `_meta.hostname` is the narrowest maintainable fix.
