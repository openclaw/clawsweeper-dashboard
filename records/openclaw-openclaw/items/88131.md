## What This Changes

The PR updates core file logging to resolve hostnames for each JSONL write, align `_meta.hostname` with top-level `hostname`, reset the test resolver, and add an empty-then-real hostname regression test.

## Best Possible Solution

Land this focused logging fix after required checks are acceptable, then let it close https://github.com/openclaw/openclaw/issues/87258.

## Reproduction Assessment

Yes for the source-level failure: current main computes `HOSTNAME` at module load and reuses it for file-log records, so an empty first lookup stays `unknown`. The intermittent macOS startup timing itself was not independently live-reproduced here.

## Solution Assessment

Yes. Resolving hostname inside the file-log write path and aligning `_meta.hostname` from the same value is narrow, preserves the existing JSONL shape, and avoids new config or compatibility surfaces.
