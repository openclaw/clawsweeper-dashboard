## Summary

Keep open for maintainer review and normal merge gating. The PR is a focused fix for a real current and shipped logging bug: current code freezes `os.hostname()` at module load, while the branch resolves it for each JSONL write and adds targeted regression coverage.

## What This Changes

The branch changes core file logging to resolve hostnames per JSONL write, align `_meta.hostname` with the top-level hostname, and add an empty-then-real hostname regression test.

## Best Possible Solution

Land the lazy per-write hostname resolution after required checks and maintainer review, preserving the existing `unknown` fallback while allowing later log writes to recover.

## Reproduction Assessment

Yes. Current source freezes `os.hostname()` at module load, and the PR's regression test models an empty first lookup followed by a real hostname to prove the failure mode without needing the intermittent macOS timing live.

## Solution Assessment

Yes. Resolving the hostname at file-log write time and reusing that value for `_meta.hostname` is the narrowest maintainable fix; it does not add config, change user preferences, or introduce a new compatibility path.
