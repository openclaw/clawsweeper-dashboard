## Summary

Keep open. Current `main` still reports doctor model-auth health from an implicit/default auth profile store with generic note titles, and the already-linked open PR #71923 directly targets this bug but has not merged.

## What This Changes

Request `openclaw doctor` to identify which agent auth store produced model-auth status when multiple agent stores exist, or at least report divergent auth state across stores.

## Best Possible Solution

Keep this issue open until #71923 or an equivalent focused fix lands. The desired end state is doctor enumerating relevant agent auth stores, labeling multi-agent auth diagnostics by agent, preserving stable single-agent output, scoping refresh attempts to the inspected store, and adding divergent-store regression coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: configure multiple agents with divergent `auth-profiles.json` stores, then current `main` loads doctor auth health through `ensureAuthProfileStore(undefined, ...)` and emits generic auth note titles.

## Solution Assessment

Yes. The narrow maintainable solution is a doctor/auth reporting fix that reuses existing agent-scope helpers and read-only auth-store loading, without changing auth inheritance, credential selection, or provider runtime semantics.
