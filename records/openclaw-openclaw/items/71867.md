## Summary

Keep open. Current main still checks doctor model-auth health through one implicit auth profile store and emits generic auth note titles, while the linked open PR #71923 directly targets the missing per-agent labeling and divergent-store coverage.

## What This Changes

Request that `openclaw doctor` identify which agent auth store produced model-auth diagnostics when multiple per-agent stores exist.

## Best Possible Solution

Land #71923 or an equivalent focused fix that enumerates relevant agent stores, labels multi-agent auth diagnostics by agent, preserves stable single-agent output, scopes refresh attempts to the inspected store, and adds divergent-store regression coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: configure multiple agents with divergent `auth-profiles.json` stores, then current main loads auth health through `ensureAuthProfileStore(undefined, ...)` and reports generic note titles.

## Solution Assessment

Yes. The best path is a narrow doctor/auth reporting fix that reuses existing agent-scope helpers and read-only auth-store loading, without changing auth inheritance, credential selection, or provider runtime semantics.
