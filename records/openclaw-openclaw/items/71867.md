## Summary

Keep open. Current main still reports doctor model-auth health through the implicit default auth store and emits generic auth notes, while the linked PR #71923 is still open as the focused implementation candidate.

## What This Changes

`openclaw doctor` should identify which agent auth store produced model-auth warnings when multiple agent stores exist.

## Best Possible Solution

Land PR #71923 or an equivalent focused fix that enumerates relevant agent auth stores, labels multi-agent auth diagnostics by agent, preserves stable single-agent output, and adds divergent-store regression coverage.

Yes. A high-confidence static reproduction path exists: configure multiple agent auth stores with divergent token state, then current main loads doctor auth health through `ensureAuthProfileStore(undefined, ...)` and emits unlabeled `Model auth` output.

## Solution Assessment

Yes. The narrow maintainable fix belongs at the doctor/auth diagnostic boundary, using existing agent-dir resolution and read-only auth-store loading without changing provider runtime credential selection.
