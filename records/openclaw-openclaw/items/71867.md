## Summary

Keep open. Current main still reports doctor model-auth health through an implicit/default auth profile store and emits unlabeled auth notes, while the linked implementation candidate remains open and unmerged.

## What This Changes

Request that `openclaw doctor` identify which agent auth store produced model-auth diagnostics, or at least warn when per-agent auth state diverges.

## Best Possible Solution

Land #71923 or an equivalent focused fix that enumerates the relevant agent auth stores, labels multi-agent auth diagnostics by agent, preserves stable single-agent output, scopes refresh attempts to the inspected store, and adds divergent-store regression coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: create multiple per-agent auth stores with divergent token state, then current main loads doctor auth health through `ensureAuthProfileStore(undefined, ...)` and emits generic auth note titles.

## Solution Assessment

Yes. The narrow maintainable solution is to fix doctor/auth reporting at the diagnostic boundary using existing agent-dir resolution and read-only auth-store loading, without changing credential selection or provider runtime semantics.
