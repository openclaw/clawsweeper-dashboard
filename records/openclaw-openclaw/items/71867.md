## Summary

Keep open. Current main still reports doctor auth health through an implicit auth store and emits generic auth note titles, while the linked PR #71923 remains open as the focused implementation candidate.

## What This Changes

Request that `openclaw doctor` identify which per-agent auth store it is reporting when multiple agent stores exist.

## Best Possible Solution

Land PR #71923 or an equivalent focused doctor/auth diagnostic fix that enumerates relevant agent auth stores, labels multi-agent diagnostics by agent, preserves stable single-agent output, and adds divergent-store regression coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: configure multiple agent auth stores with divergent auth state, then `openclaw doctor` reaches `noteAuthProfileHealth`, loads `ensureAuthProfileStore(undefined, ...)`, and emits unlabeled `Model auth` or related auth notes.

## Solution Assessment

Yes. The narrow maintainable fix belongs at the doctor/auth diagnostic boundary, using existing agent-dir resolution and read-only auth-store loading without changing provider runtime credential selection.
