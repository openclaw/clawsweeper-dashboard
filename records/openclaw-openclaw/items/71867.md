## Summary

Keep open. Current `main` still reports doctor model-auth health through the implicit default auth store and unlabeled note titles, and the provided context shows open PR #71923 as the focused implementation candidate rather than a merged fix.

## What This Changes

The issue asks for `openclaw doctor` auth diagnostics to identify which per-agent auth store produced each model-auth status when multiple agent stores exist.

## Best Possible Solution

Doctor auth health should enumerate relevant agent auth stores, label multi-agent diagnostics by agent, preserve stable single-agent output, scope refresh attempts to the inspected store, and cover divergent stores in regression tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: create multiple per-agent auth stores with divergent token state, then current `main` loads health through `ensureAuthProfileStore(undefined, ...)` and emits unlabeled `Model auth` output.

## Solution Assessment

Yes. The narrow maintainable solution is to fix doctor/auth reporting at the diagnostic boundary using existing agent-dir resolution and read-only auth-store loading, which is the direction described by PR #71923.
