## Summary

Keep open. Current main still checks doctor model-auth health through an implicit/default auth store and emits generic auth note titles, while the related open PR #71923 targets this exact bug but is not merged.

## What This Changes

Request that `openclaw doctor` identify which agent store produced model-auth diagnostics when multiple agent auth stores exist.

## Best Possible Solution

Land #71923 or an equivalent focused fix so doctor enumerates the relevant agent stores, labels multi-agent auth diagnostics by agent, scopes refresh attempts to the same inspected store, preserves stable single-agent output, and adds divergent-store regression coverage plus any required changelog entry.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: configure multiple agents with divergent `auth-profiles.json` stores, then current main loads doctor auth health through `ensureAuthProfileStore(undefined, ...)` and reports generic note titles.

## Solution Assessment

Yes. The narrow maintainable solution is a doctor/auth reporting fix that reuses existing agent-scope helpers and read-only auth-store loading, without changing auth inheritance, credential selection, or provider runtime semantics.
