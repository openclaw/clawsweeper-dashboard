## Summary

Keep open. Current main still reports doctor model-auth health from an implicit/default auth profile store with generic note titles, and the open implementation candidate has not merged yet.

## What This Changes

Request doctor diagnostics to identify the agent store behind model-auth status, or at least flag divergent auth state across multiple agent stores.

## Best Possible Solution

Land #71923 or an equivalent focused fix that enumerates relevant agent stores, labels multi-agent auth diagnostics by agent, preserves stable single-agent output, scopes refresh attempts to the inspected store, and adds divergent-store regression coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: create multiple agent auth stores with divergent token state, then current main loads doctor auth health through `ensureAuthProfileStore(undefined, ...)` and emits generic auth note titles.

## Solution Assessment

Yes. The narrow maintainable direction is to fix doctor/auth reporting at the diagnostic boundary using existing agent-dir resolution and read-only auth-store loading, without changing credential selection or provider runtime semantics.
