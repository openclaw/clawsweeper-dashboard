## Summary

Keep open. Current main still runs doctor model-auth health against the implicit default auth store and emits generic auth notes, while open PR #71923 already targets the exact diagnostic gap.

## What This Changes

Request that `openclaw doctor` identify which agent auth store its model-auth diagnostics come from when multiple agent stores exist.

## Best Possible Solution

Land #71923 or an equivalent focused fix that enumerates relevant agent auth stores, labels multi-agent auth diagnostics by agent, preserves stable single-agent output, scopes refresh attempts to the inspected store, and adds divergent-store regression coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: configure multiple agent auth stores with divergent token state, then current main loads doctor auth health through `ensureAuthProfileStore(undefined, ...)` and emits generic note titles.

## Solution Assessment

Yes. Fixing doctor/auth reporting at the diagnostic boundary is the narrowest maintainable solution; changing credential selection or provider runtime auth would be broader than this bug requires.
