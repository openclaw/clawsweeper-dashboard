## Summary

Keep open. Current main still checks doctor model-auth health through the implicit default auth store and emits generic auth notes, while per-agent auth is documented and PR #71923 remains open as the focused implementation candidate.

## What This Changes

Request that `openclaw doctor` identify which agent auth store it is reporting, or at least report divergent auth state across multiple agent stores.

## Best Possible Solution

Land PR #71923 or an equivalent focused fix that enumerates relevant agent auth stores, labels multi-agent auth diagnostics by agent, preserves stable single-agent output, scopes refresh attempts to the inspected store, and adds divergent-store regression coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: create multiple per-agent auth stores with divergent token state, then current main loads doctor auth health through `ensureAuthProfileStore(undefined, ...)` and emits unlabeled `Model auth` output.

## Solution Assessment

Yes. The narrow maintainable solution is to fix doctor/auth reporting at the diagnostic boundary using existing agent-dir resolution and read-only auth-store loading, without changing credential selection or provider runtime semantics.
