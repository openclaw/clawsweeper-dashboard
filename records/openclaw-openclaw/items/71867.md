Keep open. Current main still reports doctor auth health through the implicit default auth profile store and emits generic auth notes, while open PR #71923 remains the focused implementation candidate for this report.

## What This Changes

The issue asks `openclaw doctor` to identify which agent auth store produced model-auth diagnostics when multiple per-agent stores exist.

## Best Possible Solution

Land PR #71923 or an equivalent focused fix that enumerates relevant agent auth stores, labels multi-agent auth diagnostics by agent, preserves stable single-agent output, and adds divergent-store regression coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: configure multiple agent auth stores with divergent token state, then current main loads doctor auth health through the implicit store and emits unlabeled `Model auth` diagnostics.

## Solution Assessment

Yes. The narrow maintainable solution belongs at the doctor/auth diagnostic boundary, using existing agent-dir resolution and read-only auth-store loading without changing runtime credential selection.
