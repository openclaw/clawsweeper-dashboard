## Summary

Keep this open: current main and the latest release still force non-internal Slack `room_event` turns to `message_tool_only` before `visibleReplies` can apply, while Slack still wires typing/status callbacks on that path. The current docs and tests make strict room-event delivery intentional, so the remaining work is a maintainer UX/privacy decision rather than a safe cleanup close.

## What This Changes

The issue asks whether Slack room-event turns should honor `messages.groupChat.visibleReplies: "automatic"` or suppress/align Slack typing/status when final source delivery is tool-only.

## Best Possible Solution

Preserve strict ambient-room privacy unless maintainers intentionally revise it, and make a focused decision to suppress, alter, or explicitly bless Slack typing/status feedback during tool-only room events.
