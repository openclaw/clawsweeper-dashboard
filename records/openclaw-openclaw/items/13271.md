## Summary

Keep open. Current main still lacks a channel-scoped max-per-turn tool-call budget and the requested warn, block, and auto-delegate overflow behaviors; related #9912 is only a global limit request, and the prior maintainer-facing discussion also kept this design open.

## What This Changes

The issue requests per-channel tool-call budgets with warn, block, and auto-delegate behavior after a configured per-turn limit.

## Best Possible Solution

Define a generic scoped tool-call budget resolver that composes with the existing effective tool-policy pipeline; ship deterministic run-scoped warn/block enforcement first, then design auto-delegation through existing `sessions_spawn` authority and delivery guardrails.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The high-confidence current-main check is source, schema, docs, and related-discussion inspection showing the requested budget config and overflow enforcement are absent.

## Solution Assessment

Unclear. Per-channel budgeting is plausible, but combining hard enforcement with automatic subagent creation in the first change is not the narrowest maintainable path; warn/block enforcement should be designed before auto-delegation.
