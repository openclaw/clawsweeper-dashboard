## Summary

Keep open. Current main still has adjacent loop detection, channel/group allow-deny tool policy, plugin before-tool hooks, and explicit `sessions_spawn`, but it does not implement a channel-scoped per-turn tool-call budget or the requested `warn`, `block`, and `auto-delegate` overflow behavior; maintainer discussion in this issue also treats the design as unresolved.

## What This Changes

Request a channel-scoped config that limits tool calls per turn and chooses warn, block, or auto-delegate behavior when the limit is exceeded.

## Best Possible Solution

Define a generic scoped tool-call budget model that composes with the effective tool-policy pipeline; ship deterministic run-scoped warn/block enforcement first, then design auto-delegation separately through `sessions_spawn` authority and delivery guardrails.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The high-confidence check is source, docs, schema, and issue-discussion inspection showing the requested channel budget config and overflow enforcement are absent on current main.

## Solution Assessment

Unclear. Per-channel budgeting is plausible, but combining hard enforcement with automatic subagent creation in the first implementation is not the narrowest maintainable path; warn/block should be designed before auto-delegation.
