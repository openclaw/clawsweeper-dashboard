## Summary

Keep open. Current main and the latest release still lack channel-scoped per-turn tool-call budgets and the requested warn, block, or automatic delegation overflow behavior; the open global budget tracker at https://github.com/openclaw/openclaw/issues/9912 only partially overlaps and does not supersede this channel-scoped design request.

## What This Changes

Request a channel-scoped tool-call budget config that warns, blocks, or delegates overflow tool calls to a sub-agent.

## Best Possible Solution

Define a generic scoped tool-budget model with explicit precedence and turn accounting, implement deterministic warn/block enforcement first in the trusted before-tool path, then design automatic delegation as a separate opt-in policy around existing spawn, sandbox, child-run, and completion-routing contracts.

## Reproduction Assessment

Yes: source-level inspection of current main and v2026.6.6 shows adjacent loop detection, channel tool policy, and explicit spawning, but no channel-scoped per-turn budget or warn/block/auto-delegate overflow behavior. This is a missing capability rather than a failing existing contract.

## Solution Assessment

No: the full proposal should not land as one gateway interception feature without design. The maintainable path is generic scoped budget enforcement for warn/block first, with automatic delegation approved separately because it crosses security and routing boundaries.
