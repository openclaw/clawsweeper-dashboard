## Summary

Keep open: current main still has adjacent loop detection, channel/group tool allow-deny policy, and explicit `sessions_spawn`, but it does not implement the requested channel-scoped per-turn tool-call budget or the `warn`/`block`/`auto-delegate` overflow behavior. The issue is valid, but it needs maintainer product and security design before implementation.

## What This Changes

Request a per-channel tool-call budget config that enforces max calls per turn and can warn, block, or auto-delegate overflow work to a sub-agent.

## Best Possible Solution

Define a generic scoped tool-budget model with explicit precedence and per-turn accounting; ship deterministic warn/block enforcement first in the trusted before-tool path, then separately design auto-delegation around `sessions_spawn`, sandbox/tool inheritance, child-run tracking, and completion handoff.

## Reproduction Assessment

Not applicable as a broken-behavior reproduction: this is a missing feature request. Source and release inspection give high confidence that current main and v2026.6.8 lack the requested channel-scoped per-turn budget and overflow behavior.

## Solution Assessment

No as a single-step config plus auto-delegate implementation; the combined gateway interception and automatic child-session behavior is too security- and routing-sensitive to land without design. The maintainable path is staged generic budget enforcement first, then separately approved auto-delegation semantics.
