## Summary

Keep open. Current main and the latest release still have adjacent loop detection, channel tool allow/deny policy, and explicit `sessions_spawn`, but not the requested channel-scoped per-turn tool-call budget or `warn`/`block`/`auto-delegate` overflow behavior.

## What This Changes

Request a per-channel tool-call budget config that can warn, block, or automatically delegate overflow work to a subagent.

## Best Possible Solution

Define a generic scoped tool-budget model with explicit precedence and per-turn accounting; ship deterministic warn/block enforcement first, then separately approve automatic delegation around existing spawn, sandbox, registry, and completion-routing contracts.

## Reproduction Assessment

Not applicable as a broken-behavior reproduction: this is a missing feature request. Source and release inspection give high confidence that current main and v2026.6.8 lack the requested channel-scoped per-turn budget and overflow behavior.

## Solution Assessment

No as a single-step config plus auto-delegate implementation; the combined gateway interception and automatic child-session behavior is too security- and routing-sensitive to land without design. The maintainable path is staged generic budget enforcement first, then separately approved auto-delegation semantics.
