## Summary

Keep open. Current main has loop detection, scoped tool allow/deny policy, and explicit subagent spawning, but it still lacks channel-scoped per-turn budgets and the requested overflow behaviors; this remains a valid design item requiring maintainer product and security decisions.

## What This Changes

Requests channel-scoped per-turn tool-call limits with warn, block, and automatic subagent delegation overflow policies.

## Best Possible Solution

Define one generic scoped tool-call budget model with deterministic policy precedence; implement warn and block in the trusted before-tool path first, then evaluate auto-delegation as a separately approved operator-controlled capability using the existing subagent lifecycle.

## Reproduction Assessment

Yes at source level. Current main passes channel identity and loop-detection settings through the trusted tool hook, but exposes no scoped counter or warn, block, or auto-delegate overflow contract.

## Solution Assessment

No, the proposed channel-specific setting is not yet the best complete design. A generic scoped budget with explicit precedence is the maintainable enforcement layer, and automatic delegation should be considered separately because it crosses security and subagent-lifecycle boundaries.
