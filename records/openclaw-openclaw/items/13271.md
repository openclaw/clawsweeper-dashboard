- `P2`: This is a normal-priority enhancement with real multi-channel stability impact, but it requires public config and runtime semantics rather than an urgent regression fix.

## Summary

Keep open. Current main and the latest shipped tag still lack a channel-scoped max-per-turn tool-call budget and the requested warn, block, or auto-delegate overflow behavior; related searches leave this as the canonical per-channel request rather than a duplicate of the global budget issue.

## What This Changes

The issue requests per-channel tool-call budgets with configurable warn, block, and auto-delegate behavior when a turn exceeds the channel limit.

## Best Possible Solution

Define an additive scoped budget contract, ship deterministic warn/block enforcement in the trusted before-tool path first, then design auto-delegation around existing sessions_spawn guardrails and child-run tracking.

## Reproduction Assessment

Not applicable as a bug reproduction. Source, schema, docs, shipped tag inspection, and live related-item checks give a high-confidence absence check for channel-scoped budgets and overflow actions on current main.

## Solution Assessment

Unclear. A channel-scoped hard budget is plausible, but auto-delegation is not the narrowest safe first step until maintainers define policy precedence, warning injection, block result shape, and delegation authority.
