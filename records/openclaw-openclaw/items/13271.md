## Summary

Keep open. Current main still lacks a channel-scoped hard tool-call budget and the requested `warn`, `block`, or `auto-delegate` overflow behavior; existing loop detection, channel tool allow/deny policy, and explicit subagent spawning are adjacent but not a substitute.

## What This Changes

Request a per-channel configuration surface that limits tool calls per turn and can warn, block, or delegate overflow work to a subagent.

## Best Possible Solution

Define a generic scoped tool-call budget contract in the trusted before-tool path, ship deterministic warn/block enforcement first, then design auto-delegation around existing `sessions_spawn` guardrails and child-run tracking.

## Reproduction Assessment

Not applicable as a bug reproduction. This is a feature/config request, and source, schema, docs, related-issue, and prior-comment inspection give a high-confidence check that the requested channel budget and overflow behavior are absent on current main.

## Solution Assessment

Unclear. A per-channel hard budget is plausible, but automatic subagent creation is not the narrowest safe first step until maintainers define precedence, warning injection, block result shape, and delegation authority.
