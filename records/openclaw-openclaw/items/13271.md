## Summary

Keep open. Current main still has loop detection, channel allow/deny tool policy, and explicit `sessions_spawn`, but it does not implement a channel-scoped per-turn tool budget or the requested `warn`, `block`, and `auto-delegate` overflow modes.

## What This Changes

Request a configurable per-channel tool-call budget that can warn, block, or auto-delegate overflow work to a sub-agent.

## Best Possible Solution

Define a generic scoped budget resolver in the trusted tool-policy path, ship deterministic `warn` and `block` behavior first, then design `auto-delegate` around existing `sessions_spawn` guardrails.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The high-confidence check is source, schema, docs, and related-comment inspection showing the requested budget config and overflow behavior are absent on current main.

## Solution Assessment

Unclear. A per-channel budget is plausible, but tying first-pass enforcement to automatic subagent creation is not yet the narrowest or safest solution without maintainer design on policy precedence and delegation authority.
