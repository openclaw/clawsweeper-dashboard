## Summary

Keep open. Current main and the latest release still have adjacent loop detection, channel tool allow/deny policy, and explicit `sessions_spawn`, but no channel-scoped tool-call budget or `warn`/`block`/`auto-delegate` overflow behavior; live related context keeps this as the canonical per-channel design request rather than a duplicate of https://github.com/openclaw/openclaw/issues/9912.

## What This Changes

The issue requests per-channel tool-call limits with configurable warn, block, and automatic sub-agent delegation overflow behavior.

## Best Possible Solution

Define a generic scoped tool-call budget contract in the trusted before-tool path, implement deterministic warn/block enforcement first, then design auto-delegation around existing `sessions_spawn` guardrails and child-run tracking.

## Reproduction Assessment

Not applicable as a bug reproduction. This is a feature/config request, and source, schema, docs, shipped-tag, and related-issue inspection give a high-confidence absence check for channel budgets and overflow actions.

## Solution Assessment

Unclear. A channel-scoped hard budget is plausible, but automatic subagent creation is not the narrowest safe first step until maintainers define policy precedence, warning injection, block result shape, and delegation authority.
