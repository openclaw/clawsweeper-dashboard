## Summary

Keep open. Current main still has adjacent loop detection, channel allow/deny tool policy, and explicit subagent spawning, but it lacks channel-scoped max-per-turn tool-call budgets and the requested warn/block/auto-delegate overflow modes.

## What This Changes

Request a per-channel tool-call budget configuration with warn, block, and auto-delegate overflow behavior for multi-channel agent sessions.

## Best Possible Solution

Define a generic scoped tool-call budget contract that composes with existing policy layers, ship deterministic warn/block enforcement first in the trusted before-tool path, then separately design auto-delegation around sessions_spawn guardrails and child-run tracking.

## Reproduction Assessment

Not applicable as a bug reproduction. Source, schema, docs, live issue context, and related-report inspection give a high-confidence absence check for channel-scoped budgets and overflow actions on current main.

## Solution Assessment

Unclear. A channel-scoped hard budget is plausible, but auto-delegation is not the narrowest safe first step until maintainers define policy precedence, warning injection, block result shape, and delegation authority.
