## Label Justifications

- `P2`: This is a concrete multi-channel stability improvement with credible production impact, but it needs product and security design rather than an urgent regression fix.
- `impact:session-state`: A scoped budget would alter per-turn tool state, transcript continuation, and subagent handoff behavior.
- `impact:security`: Auto-delegation would create child agent runs from an enforcement path and must preserve sandbox, target-agent, and inherited-tool authority boundaries.
- `impact:message-loss`: The reported failure mode is runaway inline tool use delaying or preventing channel replies, especially on Discord.

## Summary

Keep open. Current main and v2026.6.5 still lack a per-channel max-per-turn tool-call budget and the requested warn, block, or auto-delegate overflow behavior, while the remaining design crosses public config, session state, channel delivery, and subagent security boundaries.

## What This Changes

Request a channel-scoped hard tool-call budget, especially for Discord, with warn, block, and automatic subagent delegation overflow modes.

## Best Possible Solution

Design an additive scoped budget contract, enforce deterministic warn/block behavior in the trusted before-tool path first, then handle auto-delegation only after sessions_spawn authority, child tracking, and completion handoff rules are explicit.

## Reproduction Assessment

Not applicable as a bug replay. Source, schema, docs, release, and related-item checks give a high-confidence absence check for channel-scoped budgets and overflow actions on current main.

## Solution Assessment

Unclear. A scoped hard budget is plausible, but auto-delegation is not the narrowest safe first step until maintainers define policy precedence, warning injection, block result shape, and delegation authority.
