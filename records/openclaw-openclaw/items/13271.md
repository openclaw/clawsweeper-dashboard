## Label Justifications

- `P2`: This is a normal-priority enhancement with real multi-channel stability impact, but it requires public config and runtime semantics rather than an urgent regression fix.
- `impact:session-state`: The requested guard would alter per-turn tool state, transcript continuation, and subagent handoff behavior.

## Summary

Keep open. Current main still lacks the requested per-channel max-per-turn tool-call budget and warn/block/auto-delegate overflow behavior, and the remaining work needs product and security design rather than conservative cleanup.

## What This Changes

Request a per-channel tool-call budget config that can warn, block, or auto-delegate after a configured number of tool calls in one turn.

## Best Possible Solution

Define an additive scoped budget contract, enforce deterministic warn/block behavior in the trusted before-tool path first, then design auto-delegation around sessions_spawn guardrails and child-run handoff.

## Reproduction Assessment

Not applicable as a bug reproduction. Source, schema, docs, release, and related-item checks give a high-confidence absence check for channel-scoped budgets and overflow actions on current main.

## Solution Assessment

Unclear. A channel-scoped hard budget is plausible, but auto-delegation is not the narrowest safe first step until maintainers define policy precedence, warning injection, block result shape, and delegation authority.
