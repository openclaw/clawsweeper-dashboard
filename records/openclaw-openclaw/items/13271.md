- `P2`: This is a normal-priority enhancement with real multi-channel stability impact, but it requires public config and runtime semantics rather than an urgent regression fix.
- `impact:session-state`: The requested guard would alter per-turn tool state, transcript continuation, and subagent handoff behavior.

## Summary

Keep open. Current main still lacks a channel-scoped max-per-turn tool-call budget and the requested warn, block, or auto-delegate overflow behavior; the related global budget request remains separate, and this issue is still the canonical per-channel request.

## What This Changes

The issue requests per-channel tool-call limits with warn, block, and auto-delegate overflow behavior.

## Best Possible Solution

Define an additive scoped budget contract, ship deterministic warn/block enforcement in the trusted before-tool path first, then design auto-delegation around existing sessions_spawn guardrails and child-run tracking.
