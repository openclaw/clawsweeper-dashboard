## What This Changes

Requests channel-scoped tool-call limits with warn, block, and automatic subagent delegation overflow behavior.

## Best Possible Solution

Define one generic scoped tool-budget model with explicit precedence and turn accounting, implement deterministic warn and block enforcement in the trusted before-tool path, and design automatic delegation separately as an opt-in policy using existing spawn, sandbox, child-run, and completion-routing contracts.

## Reproduction Assessment

Yes at the source level: current main clearly has adjacent loop detection, channel tool policy, and explicit spawning, but no channel-scoped per-turn budget or requested overflow actions. This is a missing capability rather than a failing existing contract.

## Solution Assessment

No, the complete proposal should not be implemented as one gateway-level interception feature. The narrower maintainable path is generic warn/block enforcement first, followed by a separately approved opt-in delegation policy with explicit security and routing semantics.
