## Summary

Keep open. Current main still lacks a channel-scoped per-turn tool-call budget and the requested `warn`, `block`, or `auto-delegate` overflow behavior; the existing code only covers loop detection, allow/deny tool policy, and explicit `sessions_spawn`.

## What This Changes

Request a per-channel tool-call budget config that can warn, block, or automatically delegate overflow work to a subagent.

## Best Possible Solution

Define a generic scoped tool-call budget resolver in the trusted tool-policy path, ship deterministic `warn` and `block` behavior first, then design `auto-delegate` around existing `sessions_spawn` guardrails.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The high-confidence check is source and schema inspection showing the requested budget config and overflow enforcement are absent on current main.

## Solution Assessment

Unclear. Per-channel budgeting is plausible, but coupling first-pass enforcement to automatic subagent creation is not yet the narrowest maintainable or safest design.
