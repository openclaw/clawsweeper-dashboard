## Summary

Keep open. Current main still lacks a channel-scoped per-turn tool-call budget and the requested `warn`, `block`, and `auto-delegate` overflow behaviors; the recent maintainer-facing comments also kept this as unresolved design work.

## What This Changes

The issue requests per-channel max-per-turn tool-call budgets with `warn`, `block`, and `auto-delegate` overflow behaviors.

## Best Possible Solution

Define a generic scoped tool-call budget model in the trusted tool-policy path, ship deterministic `warn` and `block` enforcement first, then design `auto-delegate` around the existing `sessions_spawn` guardrails and child-run tracking.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The high-confidence check is source/schema inspection showing the requested budget config and overflow enforcement are absent on current main.

## Solution Assessment

Unclear. Per-channel budgeting is plausible, but coupling first-pass enforcement to automatic subagent creation is not the narrowest or safest design until policy precedence and authority boundaries are specified.
