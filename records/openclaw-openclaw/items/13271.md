## Summary

Current main still lacks the requested channel-scoped deterministic tool-call budget and overflow actions. This remains a valid feature/design item, but combining hard limits with transparent subagent spawning requires maintainer decisions about policy precedence, warning/block semantics, authorization, inherited tools and sandboxing, session ownership, and completion delivery; the open global-limit request does not cover those channel and delegation semantics.

## What This Changes

Requests channel-scoped per-turn tool-call limits with warn, block, and automatic subagent delegation overflow modes.

## Best Possible Solution

Design one generic scoped tool-run budget resolved before the hot path, ship deterministic warn/block behavior with explicit transcript and error contracts first, and treat automatic delegation as a separate opt-in capability built on existing spawn ownership, security, depth, lifecycle, and completion-delivery contracts.

## Reproduction Assessment

Yes at the source-capability level: current main has no channel-scoped deterministic tool-call budget or warn/block/auto-delegate overflow action in its config and before-tool enforcement surfaces. The specific Discord heartbeat and 241K-token production failure was not independently reproduced.

## Solution Assessment

No, not as one combined implementation. A generic deterministic budget is the narrow maintainable foundation, while automatic delegation needs a separate product and security design rather than being an implicit overflow side effect.
