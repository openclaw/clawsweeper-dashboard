## Summary

Keep open. Current main has adjacent loop detection, channel/group allow-deny tool policy, and explicit subagent spawning, but it still lacks the requested channel-scoped per-turn tool-call budget and the warn/block/auto-delegate overflow contract.

## What This Changes

The issue requests a per-channel tool-call budget configuration with warn, block, or automatic subagent delegation when a turn exceeds the configured tool-call limit.

## Best Possible Solution

Define a generic scoped tool-call budget contract in the trusted before-tool path, ship deterministic warn/block enforcement first, then design auto-delegation around existing sessions_spawn guardrails and child-run tracking.

## Reproduction Assessment

Not applicable as a bug reproduction. This is a feature/config request, and source, schema, docs, and related-issue inspection give a high-confidence absence check for channel-scoped budgets and overflow actions.

## Solution Assessment

Unclear. A scoped hard budget is plausible, but automatic subagent creation is not the narrowest safe first step until maintainers define policy precedence, warning injection, block result shape, and delegation authority.
