## Summary

Keep open. Current main still has adjacent loop detection, channel/group allow-deny policy, plugin before-tool hooks, and explicit `sessions_spawn`, but not the requested channel-scoped max-per-turn tool-call budget or the `warn`, `block`, and `auto-delegate` overflow modes.

## What This Changes

Requests a channel-scoped config that caps tool calls per turn and can warn, block, or auto-delegate overflow work to a subagent.

## Best Possible Solution

Define a generic scoped tool-call budget resolver in the trusted tool-policy path, ship deterministic warn/block enforcement first, then design automatic delegation around existing `sessions_spawn` guardrails and child-run tracking.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. The high-confidence check is source, schema, docs, related-issue, and prior-comment inspection showing the requested budget config and overflow behavior are absent on current main.

## Solution Assessment

Unclear. A per-channel budget is plausible, but automatic subagent creation is not the narrowest safe first step until maintainers define policy precedence, warning injection, block result shape, and delegation authority.
