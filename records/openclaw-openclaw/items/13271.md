## Summary

Keep open. Current main still lacks a channel-scoped per-turn tool-call budget and the requested `warn`, `block`, and `auto-delegate` overflow behavior; recent maintainer discussion also kept this as unresolved design work.

## What This Changes

Request channel-scoped tool-call budgets with max-per-turn limits and `warn`, `block`, or `auto-delegate` overflow behavior.

## Best Possible Solution

Define a generic scoped budget resolver in the tool-policy path, ship deterministic `warn` and `block` enforcement first, then design `auto-delegate` through `sessions_spawn` guardrails.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The high-confidence check is source/schema inspection showing the requested budget config and overflow enforcement are absent on current main.

## Solution Assessment

Unclear. Per-channel budgeting is plausible, but coupling first-pass enforcement to automatic subagent creation is not the narrowest maintainable or safest design.
