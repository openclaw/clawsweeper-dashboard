## Summary

Keep open. Current main still has adjacent loop detection, channel/group allow-deny tool policy, prompt-only subagent delegation guidance, and explicit `sessions_spawn`, but it does not implement a channel-scoped hard max-per-turn tool-call budget or the requested `warn`/`block`/`auto-delegate` overflow behavior.

## What This Changes

Request a typed per-channel tool-call budget config with warn, block, and auto-delegate overflow behavior for channel-scoped agent runs.

## Best Possible Solution

Define a generic scoped tool-call budget resolver in the trusted before-tool path, ship deterministic warn/block enforcement first, then design automatic delegation around existing `sessions_spawn` guardrails and child-run tracking.

## Reproduction Assessment

Not applicable as a bug reproduction. This is a feature/config request, and the high-confidence check is source, schema, docs, related-issue, and prior-comment inspection showing the requested channel budget and overflow behavior are absent on current main.

## Solution Assessment

Unclear. A per-channel hard budget is plausible, but automatic subagent creation is not the narrowest safe first step until maintainers define policy precedence, warning injection, block result shape, and delegation authority.
