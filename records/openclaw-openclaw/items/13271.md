## Summary

Current main still lacks the requested channel-scoped per-turn budget and warn, block, or automatic-delegation overflow contract. Existing loop detection, channel access policies, request-level OpenResponses field, and explicit subagent tools are adjacent primitives, not an implementation of the requested behavior.

## What This Changes

Add channel-scoped per-turn tool-call budgets with warn, block, or automatic subagent delegation when the limit is exceeded.

## Best Possible Solution

Add one generic run-level hard-budget primitive shared with the global limit work, resolve scoped policy before entering the tool hot path, ship deterministic warn and block behavior first, and design automatic delegation as a separately authorized lifecycle feature.

## Reproduction Assessment

Yes for the capability gap: current main has no channel-scoped per-turn budget or requested overflow modes, which is directly verifiable in config and before-tool source. The reported token growth and Discord heartbeat causal chain was not independently reproduced in this read-only review.

## Solution Assessment

No, not as one combined gateway feature. The narrow maintainable approach is a shared hard-limit primitive with scoped resolution and deterministic warn or block semantics, followed by a separate product and security design for automatic delegation.
