## Summary

Keep open. Current main and the latest release still lack a channel-scoped hard tool-call budget and the requested warn, block, or auto-delegate overflow behavior; related global-budget work remains open separately, while the older inline-delegation duplicate points back to this issue.

## What This Changes

The issue requests per-channel tool-call budgets with configurable warn, block, or auto-delegation behavior after a per-turn limit is exceeded.

## Best Possible Solution

Define a generic scoped tool-call budget model that composes with existing tool policy, channel scoping, loop detection, and sandbox rules; land deterministic warn/block enforcement first, then design auto-delegation around `sessions_spawn` as a separate security-reviewed behavior.

## Reproduction Assessment

Yes, at source level: current main lacks the proposed config fields and the before-tool path has no channel-scoped budget state or overflow action. I did not run a live Discord/Telegram multi-channel scenario because the missing capability is visible in strict source/config surfaces.

## Solution Assessment

Unclear as specified. Gateway/before-tool enforcement is the right general boundary, but auto-delegation should not be merged until maintainers define precedence, security inheritance, and child-run handoff semantics.
