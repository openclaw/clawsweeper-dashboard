## Summary

Keep open. Current main still lacks the requested channel-scoped hard tool-call budget and overflow actions, and the related issue context keeps this as the canonical per-channel design tracker rather than a duplicate of the global budget request.

## What This Changes

Request channel-specific tool-call budgets with `warn`, `block`, and `auto-delegate` behavior when a turn exceeds the configured limit.

## Best Possible Solution

Define a generic scoped tool-call budget contract in the trusted before-tool path, ship deterministic warn/block enforcement first, then design auto-delegation around existing `sessions_spawn` guardrails and child-run tracking.

## Reproduction Assessment

Not applicable as a bug reproduction. This is a feature/config request, and source, schema, docs, related-issue, and prior-comment inspection give a high-confidence check that the requested channel budget and overflow behavior are absent on current main.

## Solution Assessment

Unclear. A channel-scoped hard budget is plausible, but automatic subagent creation is not the narrowest safe first step until maintainers define precedence, warning injection, block result shape, and delegation authority.
