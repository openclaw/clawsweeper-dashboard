## Summary

Keep open. Current main still has loop detection, channel/group allow-deny policy, explicit `sessions_spawn`, and plugin hooks, but not a channel-scoped max-per-turn tool budget or the requested `warn`, `block`, and `auto-delegate` overflow behavior; the prior maintainer-facing comments also kept this open for design follow-up.

## What This Changes

Request a configurable per-channel tool-call budget with warn, block, and automatic subagent-delegation overflow modes.

## Best Possible Solution

Define a generic scoped tool-call budget resolver in the trusted tool-policy path, ship deterministic warn/block enforcement first, then design auto-delegation around existing `sessions_spawn` guardrails.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The high-confidence check is source, schema, docs, and related-comment inspection showing the requested budget config and overflow behavior are absent on current main.

## Solution Assessment

Unclear. A per-channel budget is plausible, but automatic subagent creation is not yet the narrowest safe first step without maintainer decisions on policy precedence, warning delivery, block result shape, and delegation authority.
