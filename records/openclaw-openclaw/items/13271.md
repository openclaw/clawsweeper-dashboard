## Summary

Keep open: current main still has loop detection, channel/group allow-deny tool policy, and explicit `sessions_spawn`, but not the requested per-channel max-per-turn tool-call budget or `warn`/`block`/`auto-delegate` overflow behavior. The issue remains valid and security-sensitive, but it needs maintainer product design before implementation.

## What This Changes

The issue requests channel-scoped tool-call budgets with overflow modes that warn, block, or auto-delegate remaining work to a subagent.

## Best Possible Solution

Define a generic scoped tool-budget model with explicit precedence and per-turn accounting; ship deterministic warn/block enforcement first in the trusted before-tool path, then separately design auto-delegation around `sessions_spawn`, sandbox/tool inheritance, child-run tracking, and completion handoff.

## Reproduction Assessment

Yes for the source-level feature gap, not as a broken-behavior repro: current strict config and before-tool/subagent code show no channel-scoped per-turn budget or overflow handling. The reported Discord stability motivation remains production-context evidence rather than a live repro in this review.

## Solution Assessment

No for the literal single-step config plus auto-delegate implementation: the combined gateway interception and child-session behavior is too security- and routing-sensitive to land without design. The maintainable path is staged generic budget enforcement first, then separately approved auto-delegation semantics.
