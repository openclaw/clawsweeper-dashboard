## Summary

Keep open. Current main has adjacent loop detection, channel/group tool allow/deny policy, plugin tool hooks, and `sessions_spawn`, but it still does not implement a channel-scoped per-turn tool-call budget or the requested `warn`, `block`, and `auto-delegate` overflow behavior; recent maintainer context also keeps this as unresolved design work.

## What This Changes

Add channel-scoped tool-call budgets that warn, block, or auto-delegate after a per-turn limit.

## Best Possible Solution

Define a generic scoped tool-call budget resolver that composes with the existing effective tool-policy pipeline; ship deterministic run-scoped `warn` and `block` enforcement first, then design `auto-delegate` through the existing `sessions_spawn` authority and delivery guardrails.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The high-confidence current-main check is source, schema, docs, and discussion inspection showing the requested budget config and overflow enforcement are absent.

## Solution Assessment

Unclear. Per-channel budgeting is a plausible direction, but combining hard enforcement with automatic subagent creation in one first change is not the narrowest maintainable solution.
