## Summary

Keep open. Current main still has adjacent loop detection, channel/group tool allow/deny policy, plugin before-tool hooks, and explicit subagent spawning, but it does not implement a channel-scoped max-per-turn tool-call budget or the requested warn, block, and auto-delegate overflow behaviors; recent maintainer discussion also treated this as unresolved design work.

## What This Changes

The issue requests channel-scoped tool-call budgets with warn, block, and auto-delegate overflow behavior.

## Best Possible Solution

Define a generic scoped tool-call budget resolver that composes with the existing effective tool-policy pipeline, ship deterministic run-scoped warn/block enforcement first, and design auto-delegation separately around existing subagent authority and completion handoff rules.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The high-confidence current-main check is source, schema, docs, and test-surface inspection showing the requested config keys and enforcement path are absent.

## Solution Assessment

Unclear. Per-channel budgeting is a plausible direction, but combining enforcement and automatic subagent creation in the first change is not the narrowest maintainable solution; warn/block budget enforcement should land before auto-delegation.
