## Summary

Keep open. Current main still has adjacent loop detection, channel tool allow/deny policy, and explicit sessions_spawn, but it does not implement the requested channel-scoped max-per-turn tool-call budget or warn/block/auto-delegate overflow behavior; this remains maintainer design work.

## What This Changes

Request per-channel hard tool-call budgets with warn, block, and automatic sub-agent delegation overflow behavior.

## Best Possible Solution

Define a generic scoped tool-budget model with explicit precedence and per-turn accounting; ship deterministic warn/block enforcement first in the trusted before-tool path, then separately approve automatic delegation around existing spawn, sandbox, registry, and completion-routing contracts.

## Reproduction Assessment

Yes for the missing capability: source inspection of current main and v2026.6.8 shows adjacent loop detection, channel tool policy, prompt-only delegation guidance, and explicit spawning, but no channel-scoped per-turn budget or warn/block/auto-delegate overflow behavior. This is a missing feature rather than a failing current contract.

## Solution Assessment

No as a single-step config plus auto-delegate implementation; the combined gateway interception and automatic child-session behavior is too security- and routing-sensitive to land without design. The maintainable path is staged generic budget enforcement first, then separately approved auto-delegation semantics.
