## Summary

Keep open. Current main and v2026.5.20 still lack a channel-scoped max-per-turn tool-call budget and the requested warn, block, or auto-delegate overflow behavior; this is useful but needs product and security design before implementation.

## What This Changes

The issue requests per-channel tool-call budgets with configurable warn, block, and auto-delegate overflow actions.

## Best Possible Solution

Define an additive scoped budget contract, enforce deterministic warn/block behavior in the trusted before-tool path first, then design auto-delegation around sessions_spawn guardrails and child-run handoff.
