## Summary

The feature remains absent on current main, this issue is the canonical request for scoped auto-delegation, and the design still requires maintainer decisions across tool-policy precedence, model-visible overflow behavior, child-session security, and completion delivery.

## What This Changes

Add a configurable per-channel, per-turn tool-call budget with warn, block, and automatic subagent-delegation overflow modes.

## Best Possible Solution

Define one generic resolved tool-budget policy at the canonical agent/tool boundary, coordinate shared counting and terminal semantics with the global-limit issue, ship deterministic block or warning behavior first, and add automatic delegation only after child authority, lifecycle, and completion-delivery contracts are approved.

## Risks / Open Questions

- A channel-local config shape could create a competing policy source unless precedence with global, agent, provider, sender/group, sandbox, and plugin policies is explicitly defined.
- The warn mode needs a durable model-visible injection contract, while block needs a stable terminal tool-result shape that does not accidentally continue the run.
- Automatic delegation changes tool and sandbox inheritance, child-run ownership, concurrency limits, cleanup, cancellation, transcript state, and completion routing; an unsafe design could widen authority or lose the originating channel response.
- The related global limit work at https://github.com/openclaw/openclaw/issues/9912 should share counter and terminal semantics without absorbing this issue's distinct scoped-delegation requirements.
