## Summary

Keep open. Current main still has loop detection, channel/group allow-deny policy, and explicit `sessions_spawn`, but it does not implement the requested channel-scoped per-turn tool budget or `warn`/`block`/`auto-delegate` overflow behavior.

## What This Changes

Requests a per-channel tool-call budget config with warn, block, and auto-delegate overflow behavior.

## Best Possible Solution

Define a generic scoped tool-call budget contract in the trusted before-tool path, ship deterministic warn/block enforcement first, then design auto-delegation around existing `sessions_spawn` guardrails and child-run tracking.

## Reproduction Assessment

Not applicable as a bug reproduction. This is a feature/config request, and source, schema, docs, and related-issue inspection give a high-confidence absence check for channel budgets and overflow actions.

## Solution Assessment

Unclear. A channel-scoped hard budget is plausible, but automatic subagent creation is not the narrowest safe first step until maintainers define policy precedence, warning injection, block result shape, and delegation authority.
