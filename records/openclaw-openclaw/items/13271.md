## Summary

Keep open. Current main still lacks the requested channel-scoped max-per-turn tool-call budget and `warn`/`block`/`auto-delegate` overflow behavior, while prior review context and related issue state keep this as the canonical per-channel design request rather than a duplicate of the global budget request.

## What This Changes

The issue requests per-channel tool-call budgets with configurable overflow handling, including automatic delegation to a sub-agent after a channel-specific per-turn limit.

## Best Possible Solution

Define a generic scoped budget resolver in the trusted before-tool path, ship deterministic warn/block behavior first, then layer auto-delegation on existing `sessions_spawn` guardrails and child-run tracking.

## Reproduction Assessment

Not applicable as a bug reproduction. Source, schema, docs, and related-issue inspection give a high-confidence absence check for channel budgets and overflow actions on current main.

## Solution Assessment

Unclear until maintainers choose the public API and semantics. A channel-scoped hard budget is plausible, but automatic subagent creation is not the narrowest safe first step without policy precedence and authority-boundary design.
