## Summary

Keep open. Current main still has loop detection, channel allow/deny policy, and explicit subagent spawning, but it does not have a channel-scoped tool-call budget or the requested warn/block/auto-delegate overflow behavior; related live discussion keeps this as the canonical per-channel design request.

## What This Changes

The issue requests a per-channel tool-call budget with warn, block, and auto-delegate overflow behavior.

## Best Possible Solution

Define a generic scoped tool-call budget contract in the trusted before-tool path, implement deterministic warn/block enforcement first, then design auto-delegation around existing sessions_spawn guardrails and child-run tracking.

## Reproduction Assessment

Not applicable as a bug reproduction. This is a feature/config request, and source, schema, docs, and related-issue inspection give a high-confidence absence check for channel budgets and overflow actions.

## Solution Assessment

Unclear. A channel-scoped hard budget is plausible, but automatic subagent creation is not the narrowest safe first step until maintainers define policy precedence, warning injection, block result shape, and delegation authority.
