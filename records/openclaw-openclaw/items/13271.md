## Summary

Keep open. Current main and v2026.6.8 still have loop detection, channel/sender tool allow/deny policy, prompt-only delegation guidance, and explicit `sessions_spawn`, but not the requested channel-scoped per-turn tool-call budget or warn/block/auto-delegate overflow behavior.

## What This Changes

The issue requests per-channel tool-call budgets with `warn`, `block`, and `auto-delegate` overflow modes for multi-channel agents.

## Best Possible Solution

Define a generic scoped tool-budget model with explicit precedence and per-turn accounting; ship deterministic warn/block enforcement first in the trusted before-tool path, then design automatic delegation as a separate opt-in policy around existing spawn, sandbox, registry, and completion-routing contracts.

## Reproduction Assessment

Yes for the missing capability: source inspection of current main and v2026.6.8 shows adjacent loop detection, channel tool policy, prompt-only delegation guidance, and explicit spawning, but no channel-scoped per-turn budget or warn/block/auto-delegate overflow behavior. This is a missing feature rather than a failing current contract.

## Solution Assessment

No as proposed in one step: the combined gateway interception plus automatic delegation behavior is too security- and routing-sensitive to land without design. The maintainable path is staged generic budget enforcement first, then separately approved auto-delegation semantics.
