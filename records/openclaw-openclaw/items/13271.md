## Summary

Keep open for maintainer design: current main still lacks channel-scoped per-turn budgets and overflow handling, this issue is the canonical auto-delegation request, and the combined proposal crosses configuration, policy-precedence, session-state, and subagent security boundaries.

## What This Changes

Requests channel-scoped per-turn tool-call limits with warn, block, and automatic subagent-delegation overflow modes.

## Best Possible Solution

Define one generic scoped budget model and deterministic block semantics first; handle warning injection and automatic delegation separately, with delegation reusing explicit spawn authorization, inheritance, tracking, cancellation, and completion-delivery contracts.

## Reproduction Assessment

Yes for the capability gap: current main has channel-aware before-tool enforcement and explicit subagent spawning, but no per-turn budget, warning injection, or automatic delegation mode. The claimed Discord event-loop failure chain was not independently reproduced.

## Solution Assessment

No, not as one combined gateway feature; deterministic budget enforcement is a narrower maintainable change, while automatic delegation requires a separate product and security design around existing spawn contracts.
