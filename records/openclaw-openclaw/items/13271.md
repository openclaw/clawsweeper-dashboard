## Summary

Keep open. Current main and latest release still have adjacent loop detection, channel tool allow/deny policy, and explicit `sessions_spawn`, but not the requested channel-scoped per-turn budget or warn/block/auto-delegate overflow behavior; the prior review discussion also kept this as maintainer design work.

## What This Changes

Request a per-channel tool-call budget with configurable warn, block, and automatic subagent-delegation behavior when a turn exceeds the channel limit.

## Best Possible Solution

Define a generic scoped tool-budget model with explicit precedence and per-turn accounting; ship deterministic warn/block enforcement first in the trusted before-tool path, then separately approve automatic delegation around existing spawn, sandbox, registry, and completion-routing contracts.

## Reproduction Assessment

Yes for the missing capability: source inspection of current main and the latest release shows adjacent loop detection, channel tool policy, prompt-only delegation guidance, and explicit spawning, but no channel-scoped per-turn budget or warn/block/auto-delegate overflow behavior. This is a missing capability rather than a failing current contract.

## Solution Assessment

No as a single-step config plus auto-delegate implementation; the combined gateway interception and automatic child-session behavior is too security- and routing-sensitive to land without design. The maintainable path is staged generic budget enforcement first, then separately approved auto-delegation semantics.
