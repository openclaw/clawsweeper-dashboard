## Summary

Current main still lacks the requested scoped tool-call budget and overflow modes, and this issue remains the canonical request after the narrower auto-delegation report was closed as its duplicate. The capability is useful, but combining channel-specific enforcement with automatic child creation requires maintainer decisions on policy precedence, transcript semantics, completion delivery, and security boundaries, so this should remain open rather than enter an automated repair lane.

## What This Changes

Requests channel-scoped per-turn tool-call limits with warn, block, or automatic subagent delegation overflow behavior.

## Best Possible Solution

Define a generic prepared tool-budget policy at the agent-run boundary, compose it explicitly with existing policy layers, ship deterministic warn/block behavior through the trusted before-tool path first, and consider automatic delegation only as a separately approved reuse of the canonical subagent lifecycle.

## Reproduction Assessment

Yes for the capability gap: current main has loop detection, channel identity in the before-tool context, and explicit sessions_spawn, but no scoped per-turn budget or warn/block/auto-delegate overflow behavior. The reported Discord heartbeat causality itself was not reproduced.

## Solution Assessment

No, not as one channel-local gateway feature. Warn/block enforcement fits the generic before-tool policy boundary, while automatic delegation needs a separate product and security design around the existing subagent lifecycle.
