## Summary

Keep open. Current main adds prompt-only subagent delegation guidance and still has loop detection, channel allow/deny tool policy, plugin before-tool hooks, and explicit `sessions_spawn`, but it does not implement a channel-scoped hard max-per-turn tool-call budget or `warn`/`block`/`auto-delegate` overflow enforcement.

## What This Changes

Request a per-channel tool-call budget config that enforces max tool calls per turn and handles overflow by warning, blocking, or automatically delegating remaining work to a subagent.

## Best Possible Solution

Define a generic scoped tool-call budget resolver in the trusted before-tool path, ship deterministic warn/block enforcement first, then design automatic delegation around existing `sessions_spawn` guardrails and child-run tracking.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. The high-confidence check is source, schema, docs, related-issue, and prior-comment inspection showing the requested budget config and overflow behavior are absent on current main.

## Solution Assessment

Unclear. A per-channel budget is plausible, but automatic subagent creation is not the narrowest safe first step until maintainers define policy precedence, warning injection, block result shape, and delegation authority.
