## Summary

Current main still lacks a channel-scoped tool-call budget and the requested warn, block, or auto-delegate overflow behavior, so this remains a product and security design item rather than conservative cleanup.

## What This Changes

Request a per-channel tool-call budget configuration that can warn, block, or auto-delegate when a session exceeds a configured tool-call count.

## Best Possible Solution

Define a generic scoped tool-budget policy that composes with existing tool, channel, provider, sandbox, and subagent controls; ship deterministic warn/block enforcement first, then design auto-delegation around sessions_spawn with explicit child-run tracking and inheritance rules.

Unclear. A channel-scoped hard budget is plausible, but auto-delegation is not the narrowest safe first step until maintainers define policy precedence, warning injection, block result shape, and delegation authority.
