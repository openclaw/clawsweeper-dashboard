## Summary

Current main still lacks the requested channel-scoped per-turn tool-call budget and all three overflow modes. This issue should remain open because it is the canonical home for the unique channel-scoping and automatic-delegation design, while the broader global-limit request remains open and does not cover the full behavior.

## What This Changes

Add channel-scoped per-turn tool-call limits with warn, block, and automatic subagent delegation overflow modes.

## Best Possible Solution

Define one generic resolved scoped-budget contract shared with the global-limit work, implement deterministic warn and block semantics in the trusted before-tool path, and design automatic delegation separately as an explicit opt-in that reuses existing spawn authorization, sandbox, requester-origin, child tracking, and completion-handoff contracts.

## Reproduction Assessment

Yes for the capability gap: current-main source and schema inspection clearly show no channel-scoped per-turn tool budget or warn, block, and auto-delegate overflow contract. The claimed Discord heartbeat causal chain was not reproduced.

## Solution Assessment

No, not as one combined implementation. A generic deterministic budget primitive coordinated with https://github.com/openclaw/openclaw/issues/9912 is the maintainable first step; automatic delegation needs a separate product and security decision.
