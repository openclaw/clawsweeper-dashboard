## Summary

Keep open. The PR addresses a real first-page-only live catalog bug and now includes sufficient local stub proof, but the final-URL pagination path can re-add provider auth headers after a cross-origin redirect where the guarded fetch deliberately stripped them.

## What This Changes

The branch changes the plugin SDK live provider catalog helper to follow same-origin pagination hints, reject incomplete catalogs, share a timeout budget, and adds focused regression tests.

## Best Possible Solution

Keep pagination in the shared helper, but make follow-up requests preserve the guarded fetch redirect-auth contract by not re-adding provider auth to redirected cross-origin URLs unless explicit retention is supported.
