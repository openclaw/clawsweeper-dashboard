## Summary

Keep this PR open: the Feishu bug is source-reproducible and the proposed direction is useful, but the current patch still has blocking correctness issues and lacks inspectable after-fix real behavior proof.

## What This Changes

The PR adds Feishu full-registration deduplication, Feishu named-account promotion keys/fallbacks, and regression tests for duplicate registration plus preserving top-level `groupPolicy`.

## Best Possible Solution

Repair the PR by making Feishu dedupe per active registry/API, exposing Feishu named-account promotion through current package/setup metadata, updating the test helper imports, and merging only after focused tests plus redacted real Feishu proof.

## Reproduction Assessment

Yes, at source level. Current main reruns Feishu full registration hooks/tools each time `registerFull` runs, and the legacy normalizer can still move Feishu top-level policy keys for named-account configs when Feishu promotion metadata is not visible.

## Solution Assessment

No. The PR targets the right symptoms, but the module-level registration guard is not registry-safe, the current manifest-gated promotion path needs Feishu package metadata, and the new test imports need current SDK helper subpaths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Scope Feishu full registration to the active registry:** `extensions/feishu/index.ts:80-84`
  - body: The module-level flag makes `registerFull` a process-lifetime one-shot. Since activating plugin reloads clear registered runtime state before rebuilding, a later registry/API can skip all Feishu hook/tool registration and end up missing Feishu tools; key the dedupe to the active API or registry instead.
  - confidence: 0.93
- **[P2] Advertise Feishu config promotion in package metadata:** `extensions/feishu/src/setup-core.ts:39-40`
  - body: Adding `namedAccountPromotionKeys` to the setup adapter is not enough on current main because early bundled setup loading is gated by `openclaw.setupFeatures.configPromotion`. Add the Feishu package/setup contract hint so the legacy normalizer can see the filter before plugin runtime is loaded.
  - confidence: 0.88
- **[P2] Use the current SDK test helper subpaths:** `extensions/feishu/index.test.ts:2-3`
  - body: These new repo-relative test helper imports no longer resolve and are blocked for extension tests on current main. Import `assertBundledChannelEntries` from `openclaw/plugin-sdk/channel-test-helpers` and `createTestPluginApi` from `openclaw/plugin-sdk/plugin-test-api`.
  - confidence: 0.9
