## Summary

Keep this PR open: it targets a real Feishu bug, but the current branch is not mergeable as-is because the registration guard is process-lifetime state and the setup-promotion change does not advertise the current manifest-gated config-promotion contract.

## What This Changes

The branch adds Feishu full-registration idempotency, Feishu named-account promotion keys, a core early-promotion fallback, and regression tests for duplicate registration and top-level `groupPolicy` preservation.

## Best Possible Solution

Preserve the Feishu named-account `groupPolicy` fix, but implement idempotency in a registry/reload-safe way and declare Feishu config-promotion support through package metadata with focused regression coverage.

## Reproduction Assessment

Yes. The config bug is source-reproducible from current main because static `groupPolicy` is promoted without a Feishu named-account filter, and the registration path is source-reproducible by rerunning Feishu full registration across an activating plugin reload that clears runtime state.

## Solution Assessment

No. The PR fixes the right user-visible behavior in principle, but the process-lifetime guard is unsafe across active registry reloads and the promotion metadata needs to use the current manifest-gated setup contract.
