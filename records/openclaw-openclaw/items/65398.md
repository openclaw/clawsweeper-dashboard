## Summary

Keep open: the underlying Feishu bug is still source-reproducible on current main, but this PR is not merge-ready because the idempotency guard is process-lifetime-scoped, promotion support is not advertised through current package metadata, the new test imports are stale, and external real behavior proof is still missing.

## What This Changes

The PR adds Feishu registration idempotency, named-account promotion metadata/fallbacks, and regression tests for preserving top-level groupPolicy with named accounts.

## Best Possible Solution

Keep the Feishu fix direction, but make registration dedupe per active API/registry, expose Feishu promotion through package metadata/setup contracts, update tests to current SDK helper subpaths, and merge only after focused tests plus redacted real behavior proof.

## Reproduction Assessment

Yes, at source level. Current main still registers Feishu hooks/tools on each full registration, and the legacy normalizer can move top-level groupPolicy when Feishu has named accounts but no visible named-account promotion filter.

## Solution Assessment

No. The PR targets the right behavior, but the module-level guard is not registry-safe, the promotion metadata is incomplete for current main's manifest-gated path, and the test imports need the current plugin-sdk helper subpaths.
