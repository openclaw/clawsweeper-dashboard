## What This Changes

The PR adds internal memory-core and memory-wiki fallbacks that synthesize same-agent session keys for owner-scoped live `sessions/<agent>/<uuid>.jsonl` hits when the combined session store has no entry.

## Best Possible Solution

Land the internal fallback only after current-head proof demonstrates same-agent live orphan visibility plus cross-agent and legacy basename-only fail-closed behavior, and after maintainers accept the visibility change.

## Reproduction Assessment

Yes, source-reproducible: current main emits owner-qualified `sessions/<agent>/<basename>` paths and parses owner metadata, but only synthesizes missing-store keys for archived owner metadata. The same-agent live orphan path therefore resolves to no key on main.

## Solution Assessment

Yes for code shape: keeping the fallback internal to memory-core and memory-wiki is the narrow owner-boundary fix and avoids SDK expansion. It is not merge-ready until current-head proof and maintainer acceptance cover the visibility behavior.
