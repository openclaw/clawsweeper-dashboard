## What This Changes

The PR lets same-agent owner-scoped live transcript hits synthesize session keys when the combined session store has no entry, applies that path in memory-core and memory-wiki, and adds focused regression tests.

## Best Possible Solution

Land the owner-scoped fallback after either keeping the fallback internal to memory filters or explicitly approving and documenting the SDK contract, while preserving the cross-agent and legacy-orphan fail-closed behavior.

## Reproduction Assessment

Yes, at source level. Current main emits owner-scoped session paths and parses owner metadata, but the resolver only falls back for archived-owner metadata, so a live same-agent orphan hit with an empty store resolves to no key.

## Solution Assessment

Not fully as submitted. The fallback belongs in the right runtime path, but the public SDK parameter shape needs maintainer approval, documentation, or an internal-only repair before merge.
