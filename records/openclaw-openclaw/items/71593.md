Request plugin-defined SecretRef providers and broader core/plugin schema coverage so sensitive plugin, skill, and channel config fields can use SecretRef objects without repeated core PRs.

## Best Possible Solution

Keep the shipped providerIntegration and configContracts design, then define a bounded maintainer-approved SecretInput coverage contract for skill env or specific sensitive paths with active-surface, redaction, and audit coverage.

## Reproduction Assessment

## Solution Assessment
