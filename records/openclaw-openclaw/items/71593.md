## Summary

Keep open: current main and v2026.6.10 ship the pluginIntegration provider seam and manifest-declared plugin config SecretInput paths, but the broader skill env and core schema coverage request remains unresolved and security-sensitive.

## What This Changes

Request to let plugins provide SecretRef providers and mark additional OpenClaw config fields, including channel credentials, skill env values, and plugin config paths, as SecretRef-capable without repeated core edits.

## Best Possible Solution

Keep the shipped providerIntegration and manifest configContracts design, then add a bounded maintainer-approved SecretInput coverage contract for skill env or specific sensitive paths with active-surface and audit coverage.

## Reproduction Assessment

Not applicable as a bug reproduction, but yes as a source-level capability check: current main still makes skills.entries.*.env string-only and only resolves manifest-declared plugin config SecretInput paths.

## Solution Assessment

Unclear until maintainers choose the remaining contract; the shipped providerIntegration plus configContracts path is the right partial seam, but arbitrary all-field SecretRef expansion would be too broad without an allowlisted trust model.
