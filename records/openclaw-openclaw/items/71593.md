## Summary

Keep open. Current main still has a closed env/file/exec SecretRef provider model and path-specific plugin config coverage, while open PR #72548 is the explicit closing implementation candidate, so this issue should remain open until that PR is reviewed, landed, replaced, or closed. https://github.com/openclaw/openclaw/issues/71593

## What This Changes

The issue requests a plugin-based SecretRef provider mechanism so users can add secret-reference backends and broader config field coverage without repeated core schema changes.

## Best Possible Solution

Land or replace #72548 with a versioned, documented plugin SecretRef provider contract that preserves strict built-in env/file/exec behavior and plugin activation boundaries, then track any remaining broad field-coverage gaps as explicit follow-up work.

## Reproduction Assessment

Not applicable as a feature/API request. The gap is high-confidence by static inspection of current main: the config types, zod schemas, gateway wire schemas, docs, and resolver dispatch remain closed to env/file/exec provider sources.

## Solution Assessment

Unclear until #72548 is reviewed. A plugin-defined provider seam matches the plugin-first architecture, but the final shape needs maintainer security/API review and a deliberate decision on whether broad config field coverage is included or split out.
