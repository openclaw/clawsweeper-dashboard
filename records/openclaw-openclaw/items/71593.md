## Summary

Keep open: current main and v2026.6.10 implement the plugin-owned exec provider integration and manifest-declared plugin config SecretInput paths, but the broader requested coverage for skill env values and arbitrary sensitive plugin/config fields remains missing and security-sensitive.

## What This Changes

The issue requests a plugin-based SecretRef provider model plus broader SecretRef-capable schema coverage for channel credentials, skill env values, and plugin config secrets.

## Best Possible Solution

Keep the shipped providerIntegration/configContracts seam, then design a bounded maintainer-approved SecretInput coverage contract for specific skill env and sensitive plugin config paths with redaction, audit, activation, and docs coverage.

## Reproduction Assessment

Yes, as a source-level capability check: current main and v2026.6.10 still make skills.entries.*.env string-only and only resolve manifest-declared plugin config SecretInput paths.

## Solution Assessment

Unclear until maintainers choose the remaining contract; the shipped providerIntegration plus configContracts path is a good partial seam, but arbitrary all-field SecretRef expansion would be too broad without an allowlisted trust model.
