## Summary

Keep open: current main and v2026.5.18 still expose only env/file/exec SecretRef provider sources, while manifest `secretInputs` only partially covers plugin config fields. The open implementation candidates at https://github.com/openclaw/openclaw/pull/72548 and https://github.com/openclaw/openclaw/pull/82326 need maintainer API/security selection before this feature can close.

## What This Changes

The issue requests a plugin-based SecretRef provider mechanism that can cover sensitive OpenClaw config fields without repeated core schema changes.

## Best Possible Solution

Keep this issue open until maintainers choose and land a security-reviewed provider contract, then track remaining field widening through focused follow-ups such as env vars, plugin config paths, and channel credentials.

Not applicable as a bug reproduction; this is a feature/API request. Source and docs checks give a high-confidence current-main status: provider sources remain env/file/exec-only while selected manifest-declared plugin config SecretInput paths already exist.

## Solution Assessment

Unclear until maintainers choose the contract. A plugin or manifest seam fits OpenClaw's plugin architecture, but secret resolution through plugins is security-sensitive and should be separated from broader field-coverage expansion.
