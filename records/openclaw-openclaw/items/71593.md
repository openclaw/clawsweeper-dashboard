## Best Possible Solution

Keep the shipped providerIntegration/configContracts design, then add a bounded maintainer-approved SecretInput coverage contract for specific skill env or sensitive plugin paths with active-surface, redaction, audit, and docs coverage.

## Reproduction Assessment

Not applicable as a runtime bug reproduction, but yes as a source-level capability check: current main and v2026.6.10 still make env.vars string-only and only resolve manifest-declared plugin config SecretInput paths.

## Solution Assessment
