## Summary

Keep open. Current main still has a closed `env`/`file`/`exec` SecretRef provider model, #72548 is the explicit open closing implementation candidate, and the engram authToken comment adds a concrete plugin-config case that remains useful for reviewing the broader field-coverage part.

## What This Changes

Request a plugin-defined SecretRef provider mechanism that can cover core config schemas such as channel credentials, skill env values, and plugin config sensitive paths.

## Best Possible Solution

Review, land, replace, or close #72548 with a versioned documented plugin SecretRef provider contract, then explicitly resolve the remaining field-coverage scope for skill env and plugin config paths.

## Reproduction Assessment

Not applicable as a feature/API request. Static inspection of current main gives a high-confidence verification path: types, zod schemas, gateway schema, docs, and resolver dispatch remain closed to env/file/exec provider sources.

## Solution Assessment

Unclear until #72548 is reviewed. A manifest-first plugin seam fits OpenClaw's plugin direction, but the final design needs maintainer API/security review and a decision on the deferred broad schema coverage.
