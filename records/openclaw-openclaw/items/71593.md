## Summary

Keep open. Current main still has a closed `env`/`file`/`exec` SecretRef provider model, and open PR #72548 is the explicit closing implementation candidate, so this issue should stay open until that PR is reviewed, landed, replaced, or closed; the engram authToken comment adds a concrete user-visible plugin-config data point.

## What This Changes

The issue requests a plugin-defined SecretRef provider mechanism that can cover OpenClaw config schema fields without repeated core schema edits.

## Best Possible Solution

Land or replace #72548 with a versioned, documented plugin SecretRef provider contract that preserves strict built-in behavior, then explicitly resolve or split the remaining core schema field-coverage work.

## Reproduction Assessment

Not applicable as a feature/API request. Static inspection of current main is high-confidence: the config types, zod schemas, gateway wire schema, and resolver dispatch remain closed to env/file/exec provider sources.

## Solution Assessment

Unclear until #72548 is reviewed. A plugin-defined provider seam matches OpenClaw's plugin-first direction, but the final shape needs maintainer API/security review and a decision on the deferred broad field coverage.
