## What This Changes

The issue reports that `openclaw memory index --force` can send oversized OpenAI-compatible embedding batches to a Qianfan bge-large-zh endpoint and fail the whole index instead of splitting or skipping irreducible file failures.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/80241 with a memory-core fix that adaptively splits provider size failures, preserves true provider failures, adds regression coverage, and includes real low-limit endpoint proof.

Yes. ## Solution Assessment
