## Summary

Keep this PR open: current main still lacks a tool-activity heartbeat for the embedded-agent LLM idle watchdog, and this PR has useful proof and a plausible fix direction, but the current patch drops existing tool metadata when it wraps tool objects, which is a blocking session/replay-safety defect.

## What This Changes

The PR adds a run-scoped tool-activity heartbeat, wires tool completions to notify it, and extends embedded-agent idle-timeout tests for heartbeat resets.

## Best Possible Solution

Keep the run-scoped heartbeat approach, preserve all existing runtime tool metadata on heartbeat wrappers, add focused metadata/replay regression coverage, then choose this PR as the canonical fix or close it in favor of the competing heartbeat PR.

## Reproduction Assessment

Yes. Source inspection shows current main has no tool-activity heartbeat in streamWithIdleTimeout, and the PR defect is reproducible from source because spreading a metadata-bearing tool creates a new object that lacks the WeakMap metadata used by replay-safety checks.

## Solution Assessment

No, not in its current form. The run-scoped heartbeat is the right direction and avoids a new config knob, but the wrapper must preserve existing tool metadata before it is the best mergeable fix.
