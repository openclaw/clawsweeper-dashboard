- **[P1] Consume stale tool activity before arming later waits:** `src/agents/embedded-agent-runner/run/llm-idle-timeout.ts:297-300`
  - body: After any tool activity, `runLastActivityMs` stays set for the whole run and `firstArmPending` is reset to `true` for every later `next()`. Once that timestamp is older than `timeoutMs`, this branch clamps `effectiveTimeout` to 1ms, so a model stream that already yielded output after the tool can still abort almost immediately on a subsequent chunk. Track a current last-activity value or consume the pre-stream tool timestamp after the first bridge so old tool activity cannot shorten later LLM waits.
