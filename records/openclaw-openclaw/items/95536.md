## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Use the subscribed signal for notifications:** `src/agents/embedded-agent-runner/run/attempt.ts:3687`
  - body: `streamWithIdleTimeout` subscribes tool activity on the stream `sourceSignal`, which is normally the agent-core active-run signal, but this notification uses the embedded attempt's outer `runAbortController.signal`. Because the heartbeat map is keyed by `AbortSignal`, the new notification can miss the listener and leave the provider wait timing out.
  - confidence: 0.94
- **[P1] Record activity across provider waits:** `src/agents/embedded-agent-runner/run/llm-idle-timeout.ts:303`
  - body: Leaving old stream callbacks registered does not preserve tool progress: once a provider stream has finished, that callback has `waitingForProvider = false`, so a tool completion between model turns calls `armTimer()` as a no-op. Store run-scoped last activity, or otherwise carry the activity into the next provider wait, instead of relying on stale per-stream callbacks.
  - confidence: 0.9
- **[P1] Emit heartbeats from shared tool completion:** `src/agents/embedded-agent-runner/run/attempt.ts:3687`
  - body: This hook only runs for Tool Search catalog calls. Normal direct tools such as `read` and `exec` execute through agent-core's shared `executePreparedToolCall` / `tool_execution_end` path, so those reported workflows can still bypass the heartbeat unless that shared completion boundary is wired or proved irrelevant.
  - confidence: 0.88
