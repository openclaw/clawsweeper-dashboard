## Summary

Keep open: the PR is directionally useful and the PR body now has real terminal-style after-fix proof, but the new heartbeat wrapper is applied after directory-mode tool catalog registration, so hidden tools hydrated later can still bypass `notifyToolActivity` and hit the same availability failure.

## What This Changes

The PR adds a run-scoped tool-activity heartbeat around embedded-runner effective tool execution, wires that activity into the LLM idle watchdog, clears run state, preserves tool metadata, and adds focused tests.

## Best Possible Solution

Apply the heartbeat at a shared tool-object boundary before tool-search catalog registration, or reuse the same wrapper when hydrating deferred directory tools, then keep the real subagent proof and add a regression for that hidden-tool path.
