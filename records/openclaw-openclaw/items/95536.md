- **[P1] Consume stale tool activity before arming later waits:** `src/agents/embedded-agent-runner/run/llm-idle-timeout.ts:296-300`
  - body: After any tool activity, `runLastActivityMs` stays set for the whole run and `firstArmPending` is reset before every later `next()`. Track a current last-activity value or consume the pre-stream tool timestamp after the first bridge so old tool activity cannot shorten later LLM waits.
