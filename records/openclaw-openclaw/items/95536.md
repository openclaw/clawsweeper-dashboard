## Summary

Keep this PR open, but it needs changes before merge: the heartbeat direction and proof are strong, yet the current wrapper replaces tool objects without preserving WeakMap-backed plugin/channel/terminal metadata, which can change downstream ownership and replay-safety behavior.

## What This Changes

The PR adds a run-scoped tool-activity heartbeat for embedded-agent runs, wires tool completions into the LLM idle watchdog, clears heartbeat state at attempt teardown, and adds heartbeat/idle-timeout tests.

## Best Possible Solution

Preserve all existing tool metadata when adding the heartbeat wrapper, keep this as the canonical fix only after that repair and focused regression coverage, then supersede the older competing PR.

## Reproduction Assessment

Yes for the PR defect: source inspection shows the new spread wrapper replaces metadata-keyed tool objects without calling the existing metadata copy helpers. I did not run a live current-main reproduction of the original subagent timeout in this read-only review, but the PR body includes after-fix terminal proof.

## Solution Assessment

No, not in its current form. A run-scoped heartbeat is the right fix direction, but the implementation must preserve existing tool metadata when wrapping execute before it is the safest maintainable solution.
