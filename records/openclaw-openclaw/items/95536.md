## Summary

Keep open: current main and the latest release do not implement the requested tool-activity heartbeat, but this PR is still not merge-ready because the heartbeat is keyed to the wrong run signal, records activity only as stale per-stream callbacks, misses the shared direct-tool path, and lacks real after-fix subagent proof.

## Best Possible Solution

Rework the fix around one run-scoped activity state at shared tool completion, cover both direct and cataloged tools with regressions, and require redacted real subagent proof before merge.

Yes for the PR defects from source inspection: agent-core passes an internal active-run signal to the stream wrapper, direct tools complete through agent-core, and the PR notifies only a catalog path using the outer run signal. ## Solution Assessment

No. The heartbeat direction is plausible, but this implementation is not the best fix because it relies on per-stream callbacks and a catalog-specific notification instead of durable run-scoped activity at shared tool completion.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Notify the signal the stream wrapper receives:** `src/agents/embedded-agent-runner/run/attempt.ts:3687`
  - body: `streamWithIdleTimeout` registers tool activity on the stream `sourceSignal`, which is the agent-core active-run signal passed as `options.signal`; this new call notifies `runAbortController.signal` instead. Because the listener map is keyed by `AbortSignal` identity, cataloged tool completion can miss the registered listener and leave the idle watchdog armed for the reported path.
  - confidence: 0.94
- **[P1] Persist activity instead of stale stream callbacks:** `src/agents/embedded-agent-runner/run/llm-idle-timeout.ts:305`
  - body: Leaving the per-stream listener registered does not persist tool progress: after the provider stream yields the tool call, `waitingForProvider` is false, so a later tool completion calls `armTimer()` as a no-op before the next provider wait exists. Store run-scoped last activity or otherwise carry the event into the next provider wait instead of relying on stale stream callbacks.
  - confidence: 0.9
- **[P1] Emit heartbeats from shared tool completion:** `src/agents/embedded-agent-runner/run/attempt.ts:3687`
  - body: This hook only runs for Tool Search catalog execution. The reported direct tools such as `read` and `exec` execute through agent-core's shared `executePreparedToolCall` path and emit `tool_execution_end` outside this executor, so those workflows can still hit the LLM idle timeout unless the shared completion path is wired or proved irrelevant.
  - confidence: 0.88
