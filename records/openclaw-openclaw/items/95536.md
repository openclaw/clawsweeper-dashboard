- **[P1] Scope tool activity to the active run:** `src/shared/tool-activity-heartbeat.ts:1-5`
  - body: `notifyToolActivity()` broadcasts through one process-global listener set, while every idle-timeout wrapper subscribes to the same bus. A tool finishing in one embedded run can therefore re-arm the idle watchdog for an unrelated run whose provider stream is genuinely stalled, so the fix can mask the availability failure it is supposed to detect. Keep the heartbeat keyed to the run/session/signal lifecycle instead of broadcasting globally.
  - confidence: 0.91
