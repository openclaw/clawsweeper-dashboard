## Summary

Keep open: this is a focused Codex migration bug-fix PR, but the current head introduces a failure-classification regression and still lacks real after-fix migration proof.

## What This Changes

The PR adds polling around target Codex `plugin/list` for `openai-curated`, converts missing curated plugin activation into manual-review skipped items with warnings/next steps, and adds provider tests.

## Best Possible Solution

Keep the retry behavior, but only classify the timeout/no-marketplace case as manual review, preserve true missing-plugin errors, and merge after redacted real migration proof.

## Reproduction Assessment

No; I do not have a high-confidence live reproduction. Current-main source shows one isolated target `plugin/list` activation path with no wait, and the PR tests simulate the late marketplace case.

## Solution Assessment

No as written; polling is a plausible narrow fix, but the current branch also hides true missing-plugin errors behind the new manual-review warning. The safer fix is to distinguish marketplace-not-loaded timeout from an actual missing plugin.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve real missing-plugin errors:** `extensions/codex/src/migration/apply.ts:172`
  - body: `plugin_missing` is returned when `openai-curated` is present but the selected plugin is not in that marketplace, not only when the marketplace failed to load. This branch turns both cases into a manual-review skip with a retry-after-onboarding warning, so a renamed or removed curated plugin no longer surfaces as an install error and users can rerun without any fix being possible. Gate the manual-review path on the final `plugin/list` actually missing the marketplace, or add a distinct activation reason for marketplace-not-loaded.
