## Summary

Current main still applies one global Microsoft Teams streaming mode to every personal conversation, so Copilot-only streaming is a genuine missing capability rather than an already-implemented fix. Keep the issue open for a Teams-owner product decision because implementation requires both a new compatibility-sensitive config surface and verified Microsoft metadata that reliably distinguishes inbound Copilot custom-engine turns; the open Microsoft tracker only catalogs this work and does not supersede it.

## What This Changes

Add Microsoft Teams plugin configuration that enables partial or progress streaming for Copilot custom-engine conversations while leaving ordinary Teams personal, group, and channel replies non-streaming.

## Best Possible Solution

Establish a stable Copilot surface fact from a redacted inbound activity or authoritative Microsoft SDK contract, then add one narrow plugin-owned Copilot streaming override whose absence preserves the existing global-mode behavior and whose unknown-surface path always falls back to that global setting.

## Reproduction Assessment

Yes for the reported limitation: current main exposes one global Teams streaming decision and treats every personal conversation alike. No high-confidence live reproduction yet proves which inbound metadata reliably distinguishes a Copilot custom-engine turn, so the detector portion remains unverified.

## Solution Assessment

The requested outcome is valid, but the general per-surface configuration example is not yet the best proven implementation. A narrow Copilot-only override with unchanged global fallback is the safer shape after Microsoft metadata is verified and maintainers approve the new public config contract.
