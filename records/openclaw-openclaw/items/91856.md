## Summary

Keep this open: current main and the latest release still expose only one global Microsoft Teams streaming mode, so Copilot-only streaming remains missing and depends on a maintainer-approved Copilot surface detector and config shape.

## What This Changes

Request support for enabling Microsoft Teams native streaming only for Microsoft 365 Copilot custom-engine conversations while keeping normal Teams DMs, group chats, and channels non-streaming by default.

## Best Possible Solution

Keep the current global default, then add a plugin-owned per-surface streaming config only after maintainers approve a reliable Copilot detector and backward-compatible precedence.

## Reproduction Assessment

Yes for the source-level limitation: current main still resolves streaming from one global MS Teams config and `conversationType === "personal"`. No live Copilot activity metadata is captured yet, so the exact detector is not reproducible with high confidence.

## Solution Assessment

Unclear. A plugin-owned config knob is plausible, but the best fix depends on maintainer approval of the Copilot detector and whether the API should be `streamingBySurface`, `copilotStreaming`, or a narrower documented shape.
