## Summary

Keep open: current `main` and the latest release still expose one global msteams streaming mode gated by `conversationType === "personal"`, and no Copilot-specific surface detector or config key exists yet. This is a valid feature request, but it needs maintainer product/API review because it adds config surface and depends on reliable Microsoft Teams Copilot activity metadata.

## What This Changes

The issue requests a Microsoft Teams channel configuration path that enables streaming for Copilot custom-engine conversations while keeping ordinary Teams DMs non-streaming.

## Best Possible Solution

Add an msteams-owned Copilot surface classifier based on safe Bot Framework metadata, then introduce an additive streaming override that preserves existing `channels.msteams.streaming` behavior by default and documents detector limits.

## Reproduction Assessment

Partly yes: current source and docs reproduce the limitation by showing one global msteams streaming config and a personal-conversation native stream gate. No live Copilot custom-engine activity payload was verified in this review.

## Solution Assessment

Unclear: the examples in the issue are plausible, but the best fix depends on maintainers choosing a supported config shape and proving reliable Copilot activity detection. A narrow implementation should avoid changing default Teams DM behavior.
