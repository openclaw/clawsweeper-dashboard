## Summary

Keep this open: current main still has only one `channels.msteams.streaming` mode and the native Teams stream path is gated by `conversationType === "personal"`, so there is no supported Copilot-only streaming mode yet.

## What This Changes

Add a Microsoft Teams plugin-owned way to enable streaming for Microsoft 365 Copilot custom-engine conversations without enabling Teams DM/group/channel streaming.

## Best Possible Solution

Implement this inside the MS Teams plugin with an additive surface detector, a backward-compatible streaming override, focused tests for Teams DM versus Copilot activity, and docs that name any detection limits.

## Reproduction Assessment

Partly. Source inspection shows the current limitation on main, but a high-confidence live reproduction still needs a real Copilot custom-engine activity sample to prove the correct detector metadata.

## Solution Assessment

Unclear until maintainers choose the config shape and validate Copilot activity metadata. The right implementation layer appears to be the MS Teams plugin, not core streaming.
