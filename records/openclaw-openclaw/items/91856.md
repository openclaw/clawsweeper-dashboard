## Summary

Keep this open: current main and v2026.6.8 still expose one global Microsoft Teams streaming mode, so Copilot-only streaming remains missing and needs maintainer-approved surface detection plus config shape.

## What This Changes

Request a plugin-owned way to enable Microsoft Teams native streaming for Microsoft 365 Copilot custom-engine conversations while leaving normal Teams DMs, group chats, and channels non-streaming.

## Best Possible Solution

Keep the existing global default, then add a plugin-owned per-surface streaming config only after maintainers approve a reliable Copilot detector and backward-compatible precedence.

## Reproduction Assessment

Yes for the source-level limitation: current main still resolves MS Teams streaming from one global config and `conversationType === "personal"`. The exact live Copilot detector is not high-confidence without captured Copilot activity metadata.

## Solution Assessment

Unclear. A plugin-owned config knob is plausible, but the best fix depends on maintainer approval of the Copilot detector and whether the API should be surface-based or a narrower Copilot-only setting.
