Keep open: current main still has only the global msteams streaming config and the native Teams stream gate is still based on personal conversations, so Copilot-only streaming is not implemented or superseded.

## What This Changes

The issue requests a Microsoft Teams plugin configuration path that enables streaming only for Microsoft 365 Copilot custom-engine conversations while keeping normal Teams chats non-streaming.

## Best Possible Solution

Implement plugin-local Copilot surface detection and one backward-compatible msteams streaming override only after maintainers settle the detector metadata and config precedence.

## Reproduction Assessment

No, not as a live Copilot reproduction. Source inspection does confirm the reported current limitation: native streaming is controlled by global msteams streaming config plus conversationType=personal, with no Copilot-specific detector or config key.

## Solution Assessment

Unclear whether the exact proposed config shape is the best one. The maintainable direction is msteams-plugin-owned surface detection plus a single canonical override that preserves existing defaults.
