## Summary

Keep this open: current main and the latest release still have only one global Microsoft Teams streaming mode, so the requested Copilot-only streaming behavior remains missing and needs a maintainer product/config decision.

## What This Changes

The issue requests a supported Microsoft Teams channel configuration path that enables streaming for Copilot custom-engine conversations while keeping normal Teams DMs and other Teams surfaces non-streaming.

## Best Possible Solution

Keep this open until maintainers choose a stable Copilot surface detector and implement a plugin-owned per-surface streaming config that preserves the current global default.

## Reproduction Assessment

Yes for the source-level limitation: current msteams streaming is resolved from one global config and `conversationType === "personal"`. No live Copilot activity metadata has been captured yet, so the exact detector is not reproducible with high confidence.

## Solution Assessment

Unclear. A plugin-owned config knob is plausible, but the best fix depends on maintainer approval of the Copilot detector and whether the API should be `streamingBySurface`, `copilotStreaming`, or a narrower documented shape.
