## Summary

Keep open: current main and the latest release still use one global `channels.msteams.streaming` mode and gate native Teams streaming by `conversationType === "personal"`; the requested Copilot-only split is a new msteams config/runtime capability that needs maintainer API judgment.

## What This Changes

The issue requests a Microsoft Teams channel option to enable partial/progress streaming for Copilot custom-engine conversations while keeping normal Teams DMs, group chats, and channels non-streaming.

## Best Possible Solution

Add a narrow, plugin-owned msteams surface detector and backward-compatible streaming override after maintainers choose the config shape and verify the Copilot activity metadata contract.

## Reproduction Assessment

Yes for the current limitation: source inspection shows the msteams native streaming decision only uses `conversationType` and global stream mode. A live Copilot activity sample is still needed before the detector itself can be reproduced or validated with high confidence.

## Solution Assessment

No implementation exists yet. The best fix is likely a plugin-owned, backward-compatible config/runtime addition rather than a core streaming change, but maintainers need to decide the permanent config contract first.
