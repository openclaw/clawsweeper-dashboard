## Summary

Keep open: current main still has one Microsoft Teams preview streaming config and still enables native streaminfo from `conversationType === "personal"`, so it cannot distinguish Copilot custom-engine personal conversations from ordinary Teams personal DMs. The request remains a feature/config/API decision that needs a reliable Copilot activity sample before implementation.

## What This Changes

Request support for Microsoft Teams Copilot-only streaming so Copilot custom-engine conversations can stream while normal Teams personal DMs remain non-streaming.

## Best Possible Solution

Add a plugin-owned Copilot surface detector plus an additive, backward-compatible msteams streaming override after maintainers choose the config shape; preserve the existing global `channels.msteams.streaming` fallback and document detection limits.

## Reproduction Assessment

Yes for the current limitation: source inspection shows msteams native streaming is controlled by one global config mode and `conversationType`. A live Copilot activity sample is still missing, so the future surface detector is not yet high-confidence reproducible.

## Solution Assessment

No implementation exists yet. The requested direction is plausible, but the best fix should be a plugin-owned, backward-compatible config and detector after maintainers choose the permanent schema and verify Copilot metadata.
