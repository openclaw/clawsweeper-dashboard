## Summary

Keep open: current main and v2026.6.6 still expose only global `channels.msteams.streaming` and gate native Teams streaming by `conversationType === "personal"`, so Copilot-only streaming remains an unresolved product/config request.

## What This Changes

Request a Microsoft Teams plugin configuration and detection path that enables Copilot custom-engine streaming while leaving ordinary Teams DM, group, and channel replies non-streaming.

## Best Possible Solution

Implement this inside the `msteams` plugin after live metadata proof: detect Copilot safely, add the narrowest opt-in surface-specific streaming config with global fallback, and document detection limits.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection verifies the current limitation, but choosing the right detector needs a live or redacted Copilot custom-engine activity sample.

## Solution Assessment

Unclear until maintainers choose the config shape and Copilot detector. The best direction is plugin-owned detection plus a narrow opt-in override with global fallback, not a core streaming change.
