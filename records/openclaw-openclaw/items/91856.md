## Summary

Keep this open: current main and the latest release still use one Microsoft Teams streaming config and gate native streaming on personal conversations, so the requested Copilot-only streaming behavior is not implemented.

## What This Changes

Request a Microsoft Teams channel option to enable native partial/progress streaming for Microsoft 365 Copilot custom-engine conversations while leaving normal Teams DMs, groups, and channels non-streaming by default.

## Best Possible Solution

Add an msteams-owned surface-aware streaming resolver only after validating safe Copilot activity metadata, with existing channels.msteams.streaming remaining the default fallback.

## Reproduction Assessment

No high-confidence live reproduction is available for Copilot activity metadata. Source inspection does confirm the current limitation: native streaming uses only conversationType plus the global msteams streaming mode.

## Solution Assessment

Unclear until maintainers choose the config shape and detector contract. The best direction is a narrow plugin-owned surface resolver that keeps current global streaming behavior backward compatible.
