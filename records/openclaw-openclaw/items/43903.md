## Summary

The requested capability remains absent on current main and has concrete multi-user demand, but it crosses gateway authentication, operator authorization, attribution, and session ownership; adjacent open issues do not replace it, so it should stay open for an explicit security/product design decision.

## What This Changes

Add multiple gateway credentials mapped to identities and scopes so shared Control UI users can receive isolated, attributable sessions.

## Best Possible Solution

Define one first-class authenticated gateway principal and authorization-grant contract, let explicitly approved credential modes populate it, and derive Control UI attribution and session ownership from that principal with fresh-install, upgrade, revocation, and cross-user isolation proof.

## Reproduction Assessment

Yes. Current source has one shared gateway token, token auth has no user principal, and the Control UI uses shared caller-selected session keys; this directly reproduces the reported design limitation from source.

## Solution Assessment

No, not as specified yet. A raw `tokens` map prematurely combines credential lookup, trusted identity, authorization scopes, and session partitioning; the narrowest maintainable solution starts with a canonical principal contract and then chooses its credential representation.

## Risks / Open Questions

- Treating bearer-token labels as trusted users needs defined secret storage, comparison, rotation, revocation, audit, and identity-normalization semantics.
- Per-token scopes could duplicate or conflict with the existing paired-device/operator scope model.
- Authentication changes alone do not guarantee isolation unless the authenticated principal is propagated through every Control UI session-key, chat-history, attachment-ownership, and authorization path.
- Adding `gateway.auth.tokens` is a new compatibility-sensitive secret/config surface and needs a deliberate relationship with the existing `token`, environment variable, SecretRef, CLI, and device-pairing behavior.
