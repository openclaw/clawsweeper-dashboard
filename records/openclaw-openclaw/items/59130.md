## Summary

Current main and the latest release still use semantic-only LanceDB auto-recall with no freshness-aware reranking or assistant-visible timestamp/provenance, and the linked implementation PR that closes this issue remains open and conflicting.

## What This Changes

Request memory-lancedb recall to handle freshness-sensitive prompts so latest/current queries do not answer from older semantic matches without provenance.

## Best Possible Solution

Land a refreshed memory-lancedb fix via https://github.com/openclaw/openclaw/pull/59141 or an equivalent replacement that preserves before_prompt_build, reranks freshness-sensitive recall by timestamp, and surfaces provenance/caveats.

## Reproduction Assessment

Yes, at source level. Seed two matching memories where the older item has stronger vector similarity, ask a latest/current prompt, and current main will call db.search(vector, 3, 0.3) and inject that semantic order without recency reranking or provenance.

## Solution Assessment

Yes, the requested direction is the maintainable fix for this plugin. The implementation should stay on the current before_prompt_build path, overfetch/rerank freshness-sensitive candidates, and include timestamp/provenance context without adding a new config knob unless maintainers want tunability.
