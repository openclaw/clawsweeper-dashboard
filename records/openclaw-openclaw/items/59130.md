## Summary

Keep open: current main and v2026.5.18 still use semantic-only LanceDB auto-recall without freshness-aware reranking or assistant-visible timestamp/provenance, and the linked implementation PR that closes this issue remains open and conflicting.

## What This Changes

The issue requests freshness-aware and provenance-aware LanceDB memory recall so latest or most-recent prompts do not answer from stale semantic matches.

## Best Possible Solution

Land or replace the linked memory-lancedb fix on the current before_prompt_build path, overfetch and rerank freshness-sensitive recall by timestamp, and surface timestamp/provenance caveats in injected memory context.

## Reproduction Assessment

Yes, at source level. Seed two matching memories where the older item has stronger vector similarity, ask a latest/current prompt, and current main will call db.search(vector, 3, 0.3) and inject that semantic order without recency reranking or provenance.

## Solution Assessment

Yes, the requested direction is the right fix for this plugin. The implementation should stay on before_prompt_build, preserve current timeout/input bounds, rerank freshness-sensitive candidates, and add provenance context without requiring a new config option.
