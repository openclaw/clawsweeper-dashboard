## Summary

Keep open: current main still performs semantic-only memory-lancedb recall without freshness reranking or assistant-visible timestamp/provenance, and the open implementation PR that closes this issue has not merged.

## What This Changes

The issue requests freshness-aware memory-lancedb recall and provenance/caveat context so latest/most-recent prompts do not confidently answer from stale recalled memories.

## Best Possible Solution

Land or replace the linked memory-lancedb fix on the current before_prompt_build path with freshness detection, wider candidate recall, timestamp-aware reranking, provenance/caveat context, and regression coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to seed two matching LanceDB memories where the older item has stronger vector similarity, ask a latest/current-style prompt, and observe current main injecting db.search(vector, 3, 0.3) results without recency reranking or timestamp/provenance hints.

## Solution Assessment

Yes. The requested direction is the narrow maintainable fix for this plugin, but this issue should stay open until https://github.com/openclaw/openclaw/pull/59141 lands, is refreshed, or is replaced by an equivalent implementation.
