The issue requests freshness-aware and provenance-aware LanceDB memory recall so latest/most-recent prompts do not answer from stale recalled memories without indicating source freshness.

## Best Possible Solution

## Solution Assessment

Unclear. Recency reranking with timestamp/provenance hints is a plausible mitigation, but maintainers still need to choose the boundary between memory recall, structured source metadata, and live/history lookup.
