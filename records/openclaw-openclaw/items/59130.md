## Summary

Current main still has semantic-only LanceDB auto-recall without freshness-aware ranking or assistant-visible timestamp/provenance, and the open implementation path at https://github.com/openclaw/openclaw/pull/59141 references this issue with closing syntax.

## What This Changes

The issue requests freshness-aware LanceDB memory recall and assistant-visible provenance so latest/current questions do not answer from stale recalled memories.

## Best Possible Solution

Land a refreshed memory-lancedb fix that preserves the current before_prompt_build path while adding freshness detection, wider candidate recall, timestamp-aware reranking, provenance/caveat context, and regression coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to seed two matching LanceDB memories where the older item has stronger vector similarity, ask a freshness-sensitive prompt, and observe current main injecting db.search(vector, 3, 0.3) results without recency reranking or timestamp/provenance hints.

## Solution Assessment

Yes, the requested direction is the narrow maintainable fix for this plugin. The issue should stay open until https://github.com/openclaw/openclaw/pull/59141 lands, is refreshed, or is replaced by an equivalent memory-lancedb implementation.
