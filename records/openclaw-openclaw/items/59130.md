## Summary

Current main still has semantic-only LanceDB auto-recall without freshness intent handling or provenance, and an open PR with closing syntax is already the implementation path, so this issue should stay open until that PR lands or is replaced.

## What This Changes

The issue requests recency-aware and provenance-aware LanceDB memory recall for freshness-sensitive prompts.

## Best Possible Solution

Land a refreshed memory-lancedb fix that preserves current before_prompt_build behavior while adding freshness detection, widened candidate recall, timestamp-aware reranking, recalled-memory provenance hints, and regression coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to seed two semantically matching LanceDB memories where the older one has stronger vector similarity, then ask a freshness-sensitive prompt; current main calls db.search(vector, 3, 0.3) and injects the semantic order without recency or provenance handling.

## Solution Assessment

Yes. The requested direction is the right narrow boundary for memory-lancedb: improve the plugin’s recall ranking/context first, and only add a broader generic memory-provider seam later if multiple backends need the same contract.
