## Summary

Keep open. Current main still requires explicit opt-in for generic non-loopback private model-provider hosts, and the active draft fix in https://github.com/openclaw/openclaw/pull/80751 is the right place for maintainer/security review rather than closing this issue early.

## What This Changes

The issue asks OpenClaw to trust operator-configured OpenAI-compatible model provider base URLs on private or local network hosts without requiring broad manual SSRF opt-in.

## Best Possible Solution

Review the exact-origin approach in https://github.com/openclaw/openclaw/pull/80751, decide the SSRF boundary explicitly, then land a scoped implementation with regression tests and docs; track the bundled `codex` provider-id collision separately if maintainers agree it is distinct.

## Reproduction Assessment

Yes, by source inspection: generic non-loopback private provider hosts without `request.allowPrivateNetwork` resolve to a policy with fake-IP hostname scoping but no `allowedHostnames` or private-network bypass. I did not run a live LM Studio/vLLM server in this read-only review.

## Solution Assessment

No, not as a cleanup one-liner. The safest path is the draft PR’s origin-scoped policy review, because the current tests and docs intentionally keep generic LAN/tailnet/private-DNS hosts opt-in-only.
