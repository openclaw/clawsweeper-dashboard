## What This Changes

The issue asks OpenClaw to skip disabled PDF/plugin work, diagnose severe Gateway event-loop stalls, and avoid sessions staying pinned to failed fallback models after config changes.

## Best Possible Solution

Split the report into focused fixes: clarify or validate PDF tool versus `document-extract` disablement, capture diagnostics that identify the event-loop blocker, and decide the model-selection policy for invalidating auto fallback pins when config changes.

## Reproduction Assessment

No high-confidence full reproduction path is available for all three claims. Source inspection does show the auto-failover override path can stay sticky, but the PDF example targets a non-plugin id and the event-loop stall lacks a captured blocker.
