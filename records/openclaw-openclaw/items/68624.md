## Summary

Keep open. Current main does not already implement the cancel seam, and the latest PR head is still not merge-ready: sticky terminal cancel replay is retained without production cleanup, minimal test gateways still wire the process-global requester, and the public SDK subpath lacks the required user-facing release/docs follow-through.

## What This Changes

The PR adds a plugin-neutral `session_run` cancel seam, wires gateway chat aborts to request and emit that seam, exposes a new Plugin SDK runtime subpath, and adds focused gateway/session/SDK tests, while also carrying the stacked `sessions_send` hook work from #68622.

## Best Possible Solution

Land the neutral cancel seam only after folding the remaining lifecycle and public-surface cleanup into the branch: keep the emitter core-internal, bound or explicitly sweep sticky terminal replay state, skip global requester wiring for minimal gateways, add the changelog and SDK subpath docs entry, and rebase so the sessions_send hook stack is handled by #68622 rather than duplicated here.

## Reproduction Assessment

Yes. Static inspection of the latest PR head reproduces the blockers: `TERMINAL_CANCELS` is populated without a production cleanup path, and `startGatewayEarlyRuntime` wires the process-global requester even when `minimalTestGateway` is true.

## Solution Assessment

No. The plugin-neutral seam is the right architecture direction, and the latest head fixes the earlier public emitter leak, but this is not the safest implementation until replay state is bounded/cleaned, minimal gateway wiring is guarded, and the public SDK addition is documented and changeloged.
