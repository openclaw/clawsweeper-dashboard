## Summary

Keep open. Current main still lacks the session-run cancel seam, so the PR is not obsolete, but the latest head is not merge-ready: it still leaves terminal replay state unbounded, wires the global cancel requester in minimal gateway startup, and publishes a new Plugin SDK subpath without the required changelog entry.

## What This Changes

The PR adds a plugin-neutral session-run cancel SDK/runtime seam, gateway chat-abort fan-out/request wiring, lifecycle handles, stacked sessions_send hook support, generated SDK metadata, and focused tests.

## Best Possible Solution

Keep the neutral cancel seam, but land it only after rebasing away the duplicate #68622 sessions_send stack, bounding or lifecycle-cleaning terminal cancel replay state, guarding minimal gateway requester wiring, adding the changelog/API proof, and rerunning the targeted gateway/session/plugin SDK tests plus build.

## Reproduction Assessment

Yes. Static inspection of the latest PR head reproduces the blockers: TERMINAL_CANCELS is populated without a production cleanup path, and startGatewayEarlyRuntime still wires the process-global requester when minimalTestGateway is true.

## Solution Assessment

No. The plugin-neutral cancel seam is the right architectural direction and the latest head fixed the emitter exposure issue, but the current implementation is not the safest maintainable solution until replay state cleanup, minimal-gateway isolation, changelog coverage, and stack/rebase cleanup are handled.
