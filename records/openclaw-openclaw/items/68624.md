## Summary

Keep open. Current main still lacks the requested session-run cancel seam, and the PR head is useful but not merge-ready: the prior public-emitter SDK boundary issue is fixed, but sticky terminal cancels are not cleaned up in production, minimal test gateways still wire the process-global requester, the new public SDK subpath lacks a changelog entry, and related patch work is still split across open PRs.

## What This Changes

The PR adds a typed sessions_send plugin hook, a session-run cancel module and Plugin SDK runtime subpath, gateway chat-abort requester/fan-out wiring, generated SDK export metadata, and focused tests.

## Best Possible Solution

Land the neutral cancel seam after integrating the separate guard/changelog patch, adding bounded production cleanup for sticky terminal cancels, and unstacking or rebasing after #68622 so the final PR contains only the intended cancel-seam delta. The final head should keep emitSessionRunCancel core-internal and preserve plugin-safe on/request APIs.

## Reproduction Assessment

Yes. The remaining blockers are reproducible by static inspection of the PR head: emitSessionRunCancel stores terminal state without any production cleanup call, startGatewayEarlyRuntime wires the requester outside the minimalTestGateway guard, and the package export has no matching changelog entry.

## Solution Assessment

No. The plugin-neutral seam is the right direction, but the current patch is not the narrowest safe implementation until sticky cancel lifetime, minimal-gateway isolation, changelog/API documentation, and stacked-PR integration are resolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Clear sticky cancels after run cleanup:** `src/sessions/session-run-cancel.ts:123-128`
  - body: `emitSessionRunCancel` now stores every aborted target in `TERMINAL_CANCELS`, and the comment says that state lives until `clearSessionRunCancelTarget` runs. No production path calls that cleanup when aborted runs are pruned, so a long-running gateway accumulates terminal cancel entries and can replay stale cancellation if a run id is reused. Wire cleanup into the aborted-run TTL/session-run teardown path or bound this store.
  - confidence: 0.84
- **[P2] Skip requester wiring for minimal gateways:** `src/gateway/server-startup-early.ts:124-134`
  - body: `startGatewayEarlyRuntime` still calls `wireSessionRunCancelRequester` even when `minimalTestGateway` is true. That registers a process-global requester against test-local chat state while the other side-effectful startup hooks are skipped, so later cancellation requests can route to the wrong gateway state in tests.
  - confidence: 0.82
- **[P3] Add the SDK changelog entry:** `package.json:700-703`
  - body: The patch publishes a new `openclaw/plugin-sdk/session-run-cancel-runtime` subpath for plugin authors but does not add a changelog entry. Repo policy requires user-facing feature/API changes to be recorded before merge.
  - confidence: 0.78
