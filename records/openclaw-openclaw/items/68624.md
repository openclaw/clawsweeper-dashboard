## Summary

Keep open. Current main does not implement the requested session-run cancel seam, and the latest PR head fixed the earlier public-emitter and sticky-replay blockers, but it is still not merge-ready because terminal cancel replay state is unbounded, minimal test gateways still wire the process-global requester, and the new public SDK subpath lacks a changelog entry. The branch is also dirty/stacked on the still-open sessions_send hook PR, with #75371 covering only part of the remaining cleanup.

## What This Changes

The PR adds a plugin-neutral session_run cancel seam, gateway chat-abort wiring in both directions, a public Plugin SDK runtime subpath, stacked sessions_send delegation-hook support, generated SDK export metadata, and focused tests.

## Best Possible Solution

Land the neutral cancel seam only after folding the remaining lifecycle and boundary cleanup into the branch: keep emitSessionRunCancel core-internal, bound or explicitly clean terminal replay state, skip global requester wiring for minimal gateways, add the required changelog/API proof, and rebase so the sessions_send hook stack is handled by #68622 instead of duplicated here.

## Reproduction Assessment

Yes. Static inspection of the latest PR head reproduces the blockers: TERMINAL_CANCELS is populated without a production cleanup path, and startGatewayEarlyRuntime wires the process-global requester even when minimalTestGateway is true.

## Solution Assessment

No. The plugin-neutral seam is the right direction, and the latest head fixes two earlier blockers, but this is not yet the safest maintainable implementation until terminal replay state is bounded, minimal gateway wiring is guarded, and the public API change is documented/changeloged.
