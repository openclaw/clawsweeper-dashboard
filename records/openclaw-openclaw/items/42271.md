## Summary

Keep open. The reported Feishu quoted/replied media bug is still present on current main, and the PR targets the right behavior, but the branch is stale against the current Feishu split and is not merge-ready without adapting the raw-content type, current media helper location, audio-preflight ordering, focused regression tests, and changelog coverage.

## What This Changes

The PR preserves raw fetched Feishu message content, resolves media from quoted/replied Feishu messages before building the agent media payload, and updates reaction-test mocks for the expanded message-info shape.

## Best Possible Solution

Land an updated Feishu plugin fix that preserves raw fetched parent-message content or structured media keys, resolves quoted media only after the existing fetched-context visibility gate passes, builds the final agent media payload after all permitted media is collected, keeps audio preflight coherent for any collected audio media, and adds focused quoted image/file regression tests plus the required changelog entry.

## Reproduction Assessment

Yes. A high-confidence reproduction is a Feishu group chat where a user sends an image or file, then replies to it while mentioning the bot; current main fetches only the parent placeholder text and has no code path that can attach the parent media to `MediaPaths`/`MediaTypes`.

## Solution Assessment

No, not as-is. The approach is directionally correct, but the current branch should be rebased or replaced so the implementation lands in the current `types.ts`/`bot-content.ts` structure, preserves group-context visibility, and adds regression coverage and changelog proof.
