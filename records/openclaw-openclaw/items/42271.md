## Summary

Keep open. Current main still drops media from quoted Feishu messages, but this PR is not merge-ready because it targets pre-refactor type/resolver locations and lacks focused regression and changelog coverage.

## What This Changes

This PR preserves raw Feishu parent-message JSON and resolves media from quoted/replied messages so attachments can be included in the agent media payload.

## Best Possible Solution

Land a refreshed Feishu plugin fix that preserves raw parent-message content, resolves quoted media after existing visibility checks pass, builds the final media payload from the combined list, and adds focused tests plus changelog coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: in a Feishu group, send an image or file, reply to it while mentioning the bot, and current main can fetch only the parent placeholder because it never resolves media from the fetched parent.

## Solution Assessment

No, not as-is. The approach is the right shape, but the branch needs to be refreshed for the current `types.ts`/`bot-content.ts` split and must add regression and changelog coverage before merge.
