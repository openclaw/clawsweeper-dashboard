## Summary

Keep open. Current main still resolves Feishu media only from the triggering message, then fetches quoted message text after the media payload has already been built; fetched parent messages also do not retain the raw JSON media keys needed by the resolver. The PR addresses a real Feishu plugin gap supported by prior reports and live verification, but it needs an update for current main’s file split and focused regression coverage before it is landable.

## What This Changes

The PR preserves raw fetched Feishu message content and resolves media from quoted/replied messages before building the agent media payload, with reaction-test mock updates for the expanded message-info shape.

## Best Possible Solution

Land an updated Feishu plugin implementation that preserves raw fetched parent-message content or structured media keys, downloads media only for quoted messages that pass the existing group-context visibility policy, builds the final agent media payload after all permitted media is collected, and adds regression tests for quoted image/file media plus blocked quoted-context senders.

## Reproduction Assessment

Yes. The high-confidence reproduction is a Feishu group chat where a user sends an image or file, then replies to it while mentioning the bot; current main fetches only the quoted placeholder text and has no code path that can attach the parent media to `MediaPaths`/`MediaTypes`.

## Solution Assessment

No, not as-is. The approach is directionally right, but the branch is stale against current main and should be rebased or replaced with the same behavior adapted to the moved types/media helper split, current audio-preflight ordering, and new regression coverage.
