## Summary

Keep open. Current main still lacks public session write-lock configuration and does not fully narrow the embedded lock lifecycle, although lock-timeout failover masking is partly fixed; recent comments add fresh operator reproductions.

## What This Changes

Request configurable session write-lock timeout/stale/hold behavior, narrower lock scope, and clearer lock-timeout handling instead of model-failover errors.

## Best Possible Solution

Add a supported session write-lock config contract with schema, exported types, config help/generated schema, and docs; wire it into embedded attempts and session-store mutation paths while preserving lock-timeout failover suppression and staging deeper lock-scope changes behind focused runner/session tests.

## Reproduction Assessment

Yes. The discussion provides recent v2026.4.26 and busy-gateway reports, and current-main source inspection shows same-session embedded runs can still wait on the 10s acquire default while another attempt holds the transcript lock through prompt execution.

## Solution Assessment

Unclear. A config-first fix is narrow and useful, but the exact public config/env contract and safe lock-lifecycle boundary need maintainer design; PR #69159 is relevant but incomplete for this issue.
