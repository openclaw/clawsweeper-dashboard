## Summary

Keep open. Current main has shipped useful acquire-timeout and failover-classification mitigations, but the requested stale/max-hold/env configurability and a lock lifecycle that does not span model/tool execution remain unresolved.

## What This Changes

Expose configurable session write-lock policy and narrow the embedded lock lifecycle so lock contention does not surface as model/provider failure.

## Best Possible Solution

Keep the shipped acquire-timeout and failover fixes, then land an agreed public lock policy plus a designed narrower or single-writer transcript lifecycle.
