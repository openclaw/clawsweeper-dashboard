## Summary

Keep open. Current main still has only partial mitigations: session lock timeouts are typed and excluded from model failover, but the requested public timeout/stale configuration and narrower embedded lock lifecycle are still missing; the discussion also contains a fresh v2026.4.26 operator reproduction and an open overlapping PR that covers only part of the path.

## What This Changes

Request to make session write-lock timeout and stale settings configurable, narrow the embedded session lock scope, and report lock contention separately from model/provider failures.

## Best Possible Solution

Add a supported session write-lock configuration contract with schema, exported types, config help/metadata, generated schema coverage, and public docs; wire it into embedded attempts and transcript/session-store mutation paths; preserve typed lock-timeout handling; then stage the deeper lock-scope or single-writer redesign with focused runner/session tests.

## Solution Assessment

Unclear. A config-first fix looks useful and narrow, but the exact public config/env contract and safe lock-lifecycle boundary need maintainer design; PR #69159 is helpful overlap but not a complete solution for this issue.
