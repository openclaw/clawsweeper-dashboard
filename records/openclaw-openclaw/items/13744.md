## Summary

Keep open. Current main has shipped useful acquire-timeout and failover-classification mitigations, but the requested stale/max-hold/env configurability and a lock lifecycle that does not span model/tool execution remain unresolved.

## What This Changes

Request to expose session write-lock timeout/stale policy and reduce the embedded run lock lifetime so contention no longer appears as model failure.

## Best Possible Solution

Keep the shipped acquire-timeout and failover fixes, then land an agreed public lock policy and a designed narrower or single-writer transcript lifecycle.

## Reproduction Assessment

## Solution Assessment

No for closing this issue. Current main is a useful partial mitigation, but it does not finish stale/max-hold/env configurability or the requested narrower lock scope.
