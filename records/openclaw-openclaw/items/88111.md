## Summary

Keep open: current main still has the stale plugin approval wait gap, and https://github.com/openclaw/openclaw/pull/88203 is an open linked fix with closing syntax rather than a merged implementation.

## What This Changes

The issue requests typed fail-closed handling for stale or missing plugin approval wait decisions so agent turns do not fail as generic runtime errors.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/88203 so the shared stale-approval classifier and all reported wait callers are fixed together.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: current main returns `approval expired or not found` from the gateway wait path while the shared classifier and affected wait callers do not consistently map that into the typed stale/unavailable denial outcome.

## Solution Assessment

Yes. The maintainable fix is to extend the shared stale approval classifier and reuse it in sibling approval wait callers, while preserving throw behavior for unrelated gateway errors.
