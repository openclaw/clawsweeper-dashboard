## Summary

Keep open: current main and the latest release still have the stale plugin approval wait gap, and https://github.com/openclaw/openclaw/pull/88203 is an open linked fix with closing syntax rather than a merged implementation.

## What This Changes

The issue asks for stale or missing `plugin.approval.waitDecision` ids to fail closed as typed unavailable/declined approval outcomes instead of surfacing as generic agent-turn failures.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/88203 so the shared stale-approval classifier and all reported wait callers are fixed together.

## Reproduction Assessment

Yes. Source inspection shows the gateway returns `approval expired or not found` while the shared classifier does not match that message and affected wait callers directly await the failing method.

## Solution Assessment

Yes. The maintainable fix is to extend the shared stale-approval classifier and reuse it in sibling waitDecision callers while preserving throw behavior for unrelated gateway errors.
