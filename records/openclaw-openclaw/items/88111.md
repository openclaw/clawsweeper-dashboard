## Best Possible Solution

## Reproduction Assessment

Yes. Source inspection shows `plugin.approval.waitDecision` can return `approval expired or not found` while the shared not-found classifier does not match that message, and affected wait callers directly await the failing method.

## Solution Assessment

Yes. The best fix is the linked PR's shared-classifier repair plus consistent sibling caller mapping; this issue should remain open until that PR lands or is replaced.
