## Summary

Keep open: current main still has the stale plugin approval wait gap, and https://github.com/openclaw/openclaw/pull/88203 is an open linked fix with closing syntax rather than a merged implementation.

## What This Changes

The issue asks for stale or missing `plugin.approval.waitDecision` ids to fail closed as typed unavailable/declined approval outcomes instead of surfacing as generic agent-turn failures.

## Best Possible Solution

Review, refresh if needed, and land or replace https://github.com/openclaw/openclaw/pull/88203 so the stale wait classifier and all three reported callers are fixed together.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: mock `plugin.approval.waitDecision` to reject with `approval expired or not found` in the before_tool_call, Codex app-server, or native hook relay wait paths and current main does not map it consistently to the typed fail-closed outcome.

## Solution Assessment

Yes. The maintainable fix is to extend the shared stale approval classifier and reuse it in the sibling approval wait callers, while preserving exceptional behavior for unrelated gateway errors.
