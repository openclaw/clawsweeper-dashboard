The issue asks OpenClaw to preserve Google Chat thread/reply routing metadata across streamed block replies and queued follow-up replies.

## Best Possible Solution

## Solution Assessment

Yes in direction, but it is not shipped yet. The maintainable fix is to keep Google Chat resource-shape normalization inside the plugin boundary while carrying a generic thread fact through reply, follow-up, and message-tool plumbing.
