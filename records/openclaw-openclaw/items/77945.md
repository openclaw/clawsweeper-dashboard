## Summary

Keep open: the PR targets a real current-main backup-growth bug, but it is still draft, lacks after-fix real behavior proof, and is incomplete against current main because docs and newer tests still expect retained backups.

## What This Changes

The branch deletes successful session-file repair `.bak-*` snapshots, reports `backupPath` only when cleanup fails, and updates some repair tests for no retained backup.

## Best Possible Solution

Prevent unbounded session repair backups while preserving the current atomic replacement path, then align all repair tests and transcript-hygiene docs with the chosen delete-or-rotate contract.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: repeatedly trigger `repairSessionFileIfNeeded` on a session that needs repair and current main writes a fresh unique `.bak-*` each time; the linked field report confirms the same growth in a real setup.

## Solution Assessment

No, not as submitted. Deleting successful transient backups is a plausible narrow fix, but this branch still needs current-main test/doc alignment, rebase onto the atomic replacement helper, contributor real behavior proof, and maintainer agreement on the retention contract.
