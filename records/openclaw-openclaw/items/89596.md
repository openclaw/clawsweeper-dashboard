## What This Changes

The branch adds declared plugin and MCP allowlist context to agent, gateway, embedded-runner, and skill tool-policy resolution, with tests for unavailable, denied, duplicate-safe, and plugin-group MCP declarations.

## Best Possible Solution

Keep the declared-context approach, but distinguish exact MCP tool denies from server-wide plugin/MCP denies, then require focused regression coverage and redacted real behavior proof before maintainer merge review.

## Reproduction Assessment

Yes, source-reproducible: at PR head, configure mcp.servers.paperless, allow paperless__*, and deny paperless__delete. The new startsWith declaration filter drops the paperless namespace while runtime glob matching treats paperless__delete as an exact deny.

## Solution Assessment

No, not yet. The declared-context layer is the right boundary, but deny handling must match runtime exact-vs-glob semantics and proof must come from a real OpenClaw warning path.

- **[P2] Preserve MCP declarations for exact tool denies:** `src/agents/tool-policy-declared-context.ts:36-39`
