## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Filter declarations to materializable tools:** `src/agents/tool-policy-declared-context.ts:31`
  - body: This loop adds every manifest-declared plugin id/tool to the known allowlist context, but plugin tool materialization later skips unavailable plugins, plugins disabled with `plugins.entries.<id>.enabled=false`, and plugins denied by config. After this change, an allowlist entry for a blocked plugin tool will stop warning even though no tool can appear, so the diagnostic becomes silently wrong for blocked plugins. Build this set from the same enabled/selected plugin context used by tool loading, or keep those entries warnable.
  - confidence: 0.84
- **[P3] Ignore disabled MCP servers in declared prefixes:** `src/agents/tool-policy-declared-context.ts:11-13`
  - body: `collectConfiguredMcpServerNames` treats every object under `config.mcp.servers` as a declared namespace, including entries with `enabled: false`. Runtime MCP merging filters disabled configured servers out before materialization, so `disabled__*` would no longer warn even though it cannot match any MCP tool. Filter through the same enabled-server rule before adding prefixes.
  - confidence: 0.78
