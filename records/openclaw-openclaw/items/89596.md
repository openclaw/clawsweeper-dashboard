## What This Changes

The PR threads declared plugin and MCP allowlist context into agent, gateway, embedded-runner, and skill tool-policy analysis, with regressions for unavailable, denied, duplicate-safe, and plugin-group MCP declarations.

## Best Possible Solution

## Reproduction Assessment

Yes, at source level: configure an MCP server such as paperless, allow paperless__*, and deny an exact tool such as paperless__delete. The PR drops the entire paperless declaration via startsWith, while runtime deny matching treats the exact deny as one tool only.

## Solution Assessment

No, not yet. The declared-context layer is the right boundary, but this implementation must distinguish exact MCP tool denies from server-wide deny entries and still needs real OpenClaw warning proof.
