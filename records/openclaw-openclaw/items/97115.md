## Summary

Keep open: current main still has the reported cron fallback gaps, and an open closing PR is still only a candidate fix. Source shows the isolated cron path calls `runWithModelFallback` without the embedded result classifier/merge helper and keeps reusing the cron abort signal, so the issue is still necessary until a canonical fix lands.

## What This Changes

The issue requests fixing isolated cron `agentTurn` fallback chains so empty/zero-token embedded results and cron timeouts do not stop later fallback models from running or poison cron session state.

## Best Possible Solution

Review and land one canonical cron fix that handles both returned empty/incomplete embedded results and cron timeout abort inheritance, with focused regression coverage, then close this issue after the fix reaches main.
