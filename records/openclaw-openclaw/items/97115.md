## What This Changes

The issue reports that isolated cron agentTurn fallback chains can stop after timeout-aborted attempts or clean-looking zero-token provider results, causing lost cron answers and possible stale model selection state.

## Best Possible Solution

Keep https://github.com/openclaw/openclaw/issues/96525 as the canonical tracking issue, land one narrow classifier/merge-helper fix there, and use this issue's Google/Gemini and session-state details as linked cluster evidence.
