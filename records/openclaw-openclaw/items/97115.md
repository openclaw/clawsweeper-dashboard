## What This Changes

The issue reports isolated cron agentTurn fallback chains accepting zero-token or empty embedded results, or aborting later fallback attempts, which can drop scheduled output and poison cron session model state.

## Best Possible Solution

Land or consolidate the embedded-result classifier fix for isolated cron, then keep or split the timeout AbortController claim only if focused proof shows it still fails after that fix.

## Reproduction Assessment
