## Summary

Keep open. Current main still contains the reported restart chain, and open PR #75545 is the linked implementation candidate that should be reviewed or landed before this issue closes.

## What This Changes

The issue reports that host `exec` can run a login-shell PATH probe that sources startup files, reruns `openclaw gateway start`, and restarts the managed Gateway.

## Best Possible Solution

Make managed `gateway start` an ensure-running operation for an already-running service, while keeping `gateway restart` as the explicit forced restart path.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: host exec still performs a login-shell PATH probe, and managed start still maps to a platform restart for installed or loaded services.

## Solution Assessment

Yes. Making `startGatewayService()` idempotent for `state.running` is the narrowest maintainable fix for the reported startup-file restart loop; PATH-probe hardening can remain a separate defense-in-depth discussion.
