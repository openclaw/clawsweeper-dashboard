## Summary

Keep open. Current main still contains the reported failure chain, and the linked open PR is an implementation candidate that should be reviewed or landed before this issue closes.

## What This Changes

The issue requests fixing a managed-Gateway restart loop where host exec's login-shell PATH probe can source `~/.profile`, run `openclaw gateway start`, and restart the active systemd service.

## Best Possible Solution

Land PR #75545 or an equivalent narrow change that makes managed `gateway start` mean ensure-running for an already-running service, while preserving `gateway restart` for forced cycles and adding regression coverage.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: host exec still performs a login-shell PATH probe, and current managed start still calls the systemd restart path for an installed or loaded service.

## Solution Assessment

Yes. Making managed `gateway start` idempotent is the narrowest maintainable fix for the reported `.profile` restart loop; a separate side-effect-safe PATH-probe hardening can be considered as defense in depth.
