## Summary

Keep open. Current main still contains the reported restart chain, and open PR #75545 is the focused implementation candidate linked with closing syntax.

## What This Changes

The issue reports that exec's login-shell PATH probe can re-run `openclaw gateway start` from shell startup files, causing a managed gateway restart.

## Best Possible Solution

Review and land PR #75545 or an equivalent narrow change that makes `gateway start` ensure-running for an already-running service while preserving explicit `gateway restart` semantics.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: a host exec command can run the login-shell PATH probe, which can source startup files and reach `startGatewayService()`'s unconditional restart path.

## Solution Assessment

Yes. Making managed `gateway start` idempotent for `state.running` is the narrowest maintainable fix for the reported `.profile` restart loop; PATH-probe hardening would be defense in depth.
