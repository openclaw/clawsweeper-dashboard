## Summary

Keep open. Current main and the latest release still route managed `gateway start` through the restart path, and the open closing PR has not merged.

## What This Changes

The issue requests that exec tool invocations stop restarting a managed gateway when login-shell startup files invoke `openclaw gateway start`.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/75545 with a narrow fix that makes managed `gateway start` an ensure-running no-op after repair checks, while preserving explicit `gateway restart`.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: gateway exec invokes a login-shell PATH probe, `gateway start` reaches `service.restart()`, and systemd activation performs a restart; I did not run a live systemd/profile reproduction.

## Solution Assessment

Yes as a direction. Making managed `gateway start` idempotent after repair-required checks is the narrow maintainable fix; changing exec away from login-shell PATH discovery would be broader and could break PATH behavior.
