## Summary

Keep open. Current main still contains the reported source-level failure chain, and the open closing PR is the right implementation path to review or land before closing this issue.

## What This Changes

The issue reports that `openclaw gateway start` can restart the running gateway when exec PATH discovery sources login shell startup files.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/75545 or an equivalent narrow fix that makes managed `gateway start` ensure-running for healthy already-running services while preserving repair-required behavior and forced `gateway restart`.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: exec performs a login-shell PATH probe, current `gateway start` still reaches `service.restart()`, and the Linux service restart path uses `systemctl --user restart`.

## Solution Assessment

Yes. Making managed `gateway start` idempotent for healthy already-running services is the narrowest maintainable fix; hardening the login-shell PATH probe would be defense in depth, not the required primary repair.
