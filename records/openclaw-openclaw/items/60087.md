## Summary

Keep open. Current main and the latest release still restart the managed gateway on `gateway start`, and the linked open closing PR is the implementation path rather than a reason to close early.

## What This Changes

The issue asks for exec tool invocations not to restart the gateway when a login shell startup file runs `openclaw gateway start`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/75545, or an equivalent narrow fix that makes managed `gateway start` ensure-running after preserving repair-required service checks.

## Reproduction Assessment

Yes, source-level. Host exec still performs a login-shell PATH probe, and current managed start still calls the platform restart path for an installed service even when runtime state is already running.

## Solution Assessment

Yes. Making managed `gateway start` idempotent for healthy already-running services is the narrowest maintainable fix; changing the login-shell probe would be useful defense in depth but would not address the non-idempotent lifecycle command itself.
