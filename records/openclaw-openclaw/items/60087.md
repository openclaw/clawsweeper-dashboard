## Summary

Keep open. Current main still contains the source-level failure chain, and https://github.com/openclaw/openclaw/pull/75545 is an open, unmerged closing PR rather than a landed fix.

## What This Changes

The issue reports that exec login-shell startup can rerun `openclaw gateway start`, causing the managed gateway service to restart and drop sessions.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/75545 or an equivalent narrow fix that makes managed `gateway start` ensure-running for healthy already-running services while preserving repair-required behavior and explicit `gateway restart`.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: gateway exec calls the login-shell PATH probe, current `gateway start` still reaches `service.restart()`, and the Linux service adapter maps restart to `systemctl --user restart`.

## Solution Assessment

Yes. Making managed `gateway start` idempotent for healthy already-running services is the narrowest maintainable fix; changing exec away from login-shell probing would be broader and risks PATH behavior.
