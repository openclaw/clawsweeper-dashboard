## Summary

Keep open: this is a broad draft feature PR with missing real behavior proof and a source-proven regression in the shared plugin TaskFlow DTO mapper. Current main explicitly tests that plugin task-flow DTOs do not expose `revision`, `controllerId`, or `syncMode`, but the PR adds those fields in the shared mapper instead of a Gateway-only view.

## What This Changes

Adds task metadata, pending-decision, phone-probe, and Run Harness supervision CLI surfaces, task and assistant Gateway/SDK methods, generated Swift protocol models, and governance/domain-pack docs.

## Best Possible Solution

Keep the public plugin TaskFlow DTO sanitized, add any internal task-flow metadata through a Gateway-specific DTO or mapper, and require real runtime proof before considering the broader control-plane feature for merge.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature PR. For the review finding, source inspection is sufficient: current main asserts the fields are absent from plugin DTOs, and the PR adds them to the shared mapper.

## Solution Assessment

No. The feature direction needs maintainer API/product review, and the concrete patch should not expose task-flow registry internals through the shared plugin DTO mapper; a Gateway-specific view is the safer boundary.
