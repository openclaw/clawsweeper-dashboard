## Summary

Keep open: current main and v2026.6.6 still expose `sessions.compact` only as a raw Gateway RPC, while the requested CLI/docs surface and explicit CLI `/compact` rejection are owned by an open closing PR.

## What This Changes

The issue requests a documented `openclaw sessions compact <session-key>` command backed by the existing `sessions.compact` RPC and a non-zero diagnostic for CLI-originated `/compact` messages.

## Best Possible Solution

Use https://github.com/openclaw/openclaw/pull/91378 as the canonical implementation path, then close this issue only after that PR lands or maintainers choose a replacement design.

## Reproduction Assessment

Yes. Current source still has the `sessions.compact` Gateway method but no registered `sessions compact` CLI or docs entry, and the CLI `/compact` path is only protected from timeout fallback, not rejected before normal agent dispatch.

## Solution Assessment

Yes. A thin documented CLI wrapper over the existing RPC plus an explicit CLI `/compact` rejection is the narrowest maintainable fix; executing slash commands through `agent --message` would blur the chat-command authorization boundary.
