## Summary

Keep open: the PR addresses a real Google model-resolution bug, but the patch is incomplete and unsafe as written because it only changes the synchronous resolver and can strip provider-local nested IDs.

## What This Changes

The PR changes `resolveModel` to strip same-provider model prefixes before registry lookup and adds resolver tests for Google configured models and OpenRouter pass-through IDs.

## Best Possible Solution

Land one shared resolver normalization path that strips only redundant outer same-provider refs, preserves provider-native nested IDs, covers sync and async resolution, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. Source inspection shows provider `google` plus model id `google/gemini-2.5-pro` can still miss a bare configured `gemini-2.5-pro`, and the normal runner path reaches `resolveModelAsync`.

## Solution Assessment

No. The direction is right, but the patch is not the best solution because it is sync-only and strips provider-local nested IDs that current docs and NVIDIA catalog behavior intentionally preserve.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep provider-local nested IDs intact:** `src/agents/pi-embedded-runner/model.ts:257-263`
  - body: The new rewrite strips any model ID whose first segment equals the provider. For providers with namespaced local IDs, such as NVIDIA after selecting `nvidia/nvidia/...`, this turns the provider-local `nvidia/...` id into a bare id, misses catalog/config entries, and can send the wrong model. Scope the normalization to redundant outer refs without stripping provider-native local IDs.
  - confidence: 0.9
- **[P1] Normalize the async resolver too:** `src/agents/pi-embedded-runner/model.ts:257-263`
  - body: The fix only runs in `resolveModel`, but normal embedded agent turns call `resolveModelAsync` from `run.ts`. That path can still carry `google/gemini-2.5-pro`, miss a bare configured Google model, and fail before the intended normalization is applied.
  - confidence: 0.91
- **[P3] Add the required changelog entry:** `src/agents/pi-embedded-runner/model.ts:257-263`
  - body: This is a user-facing model-resolution fix, and repo policy requires a `CHANGELOG.md` entry for user-visible fixes. The branch currently changes only resolver code and tests.
  - confidence: 0.86
