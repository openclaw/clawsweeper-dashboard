## Summary

Keep open. The Google prefix regression is real and the PR has useful focused tests, but the patch is not merge-ready because it changes only the sync resolver while current embedded turns resolve through the async path, and its generic same-provider strip can break valid provider-local namespaced model IDs.

## What This Changes

The PR strips same-provider model prefixes inside `resolveModel` before lookup and adds resolver tests for Google prefix stripping plus OpenRouter cross-provider pass-through IDs.

## Best Possible Solution

Move Google-owned redundant prefix stripping into the shared manifest/static model-id normalization path used by both sync and async resolution, preserve OpenRouter pass-through and provider-native namespaced IDs, add regression coverage, and include the required changelog entry.

## Reproduction Assessment

## Solution Assessment

No. The PR points at the right regression, but the local sync-only parser is not the narrowest maintainable fix; provider-owned stripping belongs in the shared manifest/static normalization seam with tests for async resolution and provider-local IDs.
