## Summary

Keep open: current main still runs update-mode doctor in fix mode, reaches the gateway service staging path, and Linux staging backs up then rewrites the live systemd unit from the canonical template containing the reported timeout and EnvironmentFile defaults. The paired fix PR is still open and draft at https://github.com/openclaw/openclaw/pull/80485, so this issue should stay open while maintainers decide the update-time systemd ownership policy.

## What This Changes

Prevent update-mode `openclaw doctor` from silently rewriting customized Linux systemd gateway service units during updates.

## Best Possible Solution

Define the update-time systemd unit ownership contract, then land the paired PR or an equivalent implementation that preserves operator-owned directives while keeping OpenClaw-managed service repair explicit and tested.
