## Summary

Keep open: current main still lets update-mode noninteractive doctor stage gateway service repairs, and staging rewrites the live systemd unit from the canonical template. That matches the reported timeout, launcher, and environment-file clobbering, and I found no linked merged fix or canonical open replacement for the remaining behavior.

## What This Changes

The issue reports that update-time `openclaw doctor` rewrites the live systemd user unit and overwrites operator customizations such as timeout, launcher, and environment-file settings.

## Best Possible Solution

Define systemd unit ownership so update-mode doctor changes only OpenClaw-owned service state or emits a suggested diff, while preserving operator-owned directives such as timeouts, post-start hooks, launcher overrides, and required environment files.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: update runs noninteractive doctor fix mode, doctor enters update repair mode, staged service repair writes a fresh systemd template, and the template carries the reported replacement defaults.

## Solution Assessment

No. The current whole-unit rewrite is not the best fix because it treats backup as preservation; the safer solution is a targeted managed-field update or explicit suggested diff for operator review.
