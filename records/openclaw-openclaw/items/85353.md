## What This Changes

The PR changes the plugin module loader so compiled JavaScript files with OpenClaw plugin-SDK imports try native loading before falling back to alias-aware jiti transformation, with focused tests for the native-first and fallback paths.

## Best Possible Solution

Land the native-first loader path after the contributor adds redacted installed-runtime proof showing plugin discovery/transcription still loads correctly and the temporary test fixture is cleaned up.

## Reproduction Assessment

Yes from source for the loader-order issue: current main forces jiti before native when a compiled JS file contains plugin-SDK imports. The reported 35-38s runtime cost was not independently reproduced in this read-only review.

## Solution Assessment

Yes, the proposed native-first path with an alias-aware jiti fallback is the narrow maintainable fix for the loader ordering. It still needs real installed-runtime proof because the change sits on the plugin-SDK resolution boundary.
