## Summary

Keep open: current main still persists `gateway.controlUi.allowInsecureAuth=true` for QuickStart loopback setup, while runtime auth, docs, audit, and startup warnings limit and flag the behavior. Changing the setup default or adding setup-specific warnings is a security/product policy decision, not a cleanup close.

## What This Changes

Request to stop setup from leaving `gateway.controlUi.allowInsecureAuth=true` enabled, or warn users prominently when it does.

## Best Possible Solution

Gateway/security owners should decide the QuickStart policy, then update setup behavior, warnings/docs, security audit, and regression coverage consistently.

## Solution Assessment

Unclear. Defaulting the flag to false, adding setup warnings, or relying on existing audit/startup warnings are security and UX policy choices, so the narrow maintainable path needs owner approval first.
