## Summary

Keep open. Current main still lacks edit-time feedback for disallowed cron model overrides, but this PR’s warning can disagree with the gateway/runtime validation path and it still lacks after-fix real CLI proof.

## What This Changes

The PR adds a non-blocking `cron edit --model` warning by loading local config/catalog data and checking the requested model against an allowlist before updating the cron job.

## Best Possible Solution

Compute the warning through a target-gateway/runtime validation path that shares cron execution’s config, catalog, aliases, defaults, and allowlist resolver.

## Reproduction Assessment

Yes. Source inspection shows current main stores `cron edit --model` into the cron payload without edit-time feedback, while cron execution later rejects invalid payload models through `resolveAllowedModelRef`; I did not run a live CLI command in this read-only review.

## Solution Assessment

No. The warning is useful, but this implementation duplicates a subset of runtime validation against local CLI state instead of using the selected gateway/runtime resolver.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Validate against the selected gateway runtime:** `src/cli/cron-cli/register.cron-edit.ts:218-219`
  - body: This warning reads `loadConfig()` and `loadModelCatalog()` in the CLI process, but `cron edit` sends `cron.update` to the selected gateway and supports `--url`/`--token`. In remote-gateway setups, the local allowlist may not match the gateway that will execute the job, so the warning can be wrong or absent.
  - confidence: 0.93
- **[P2] Reuse the runtime model resolver for the warning:** `src/cli/cron-cli/register.cron-edit.ts:220-228`
  - body: The warning builds `allowedKeys` with hard-coded defaults and then checks `parseModelRef(model, DEFAULT_PROVIDER)` directly. Cron execution uses `resolveAllowedModelRef` with configured defaults, aliases, provider inference, auth-profile suffix stripping, and compat aliases, so valid models can still warn and invalid ones can be missed.
  - confidence: 0.94
