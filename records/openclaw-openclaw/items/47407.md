## Summary

Keep this PR open, but it is not mergeable as-is: the reported cron-model warning is a real source-reproducible DX problem, while the patch checks local CLI config instead of the selected gateway runtime and duplicates only part of execution-time model resolution. It also still lacks contributor-supplied real behavior proof.

## What This Changes

The PR adds a non-blocking `cron edit --model` allowlist warning in `src/cli/cron-cli/register.cron-edit.ts` by loading local config and model catalog data before `cron.update`.

## Best Possible Solution

Add a gateway-owned cron model warning or preflight that uses the same runtime config, catalog, defaults, and `resolveAllowedModelRef` path as cron execution, with focused regression coverage for remote-gateway and alias/default-provider cases.

## Reproduction Assessment

Yes, source-reproducible. Current main accepts `cron edit --model` into `patch.payload.model`, while cron execution later rejects disallowed payload models through `resolveAllowedModelRef`; I did not run a live CLI repro in this read-only review.

## Solution Assessment

No. The warning is useful, but this implementation reads local CLI config and partially reimplements model validation instead of using the gateway/runtime resolver that will execute the cron.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Validate against the selected gateway runtime:** `src/cli/cron-cli/register.cron-edit.ts:218-219`
  - body: This warning reads local CLI config/catalog data, but `cron edit` can target a remote gateway via `--url`/`--token`. If local and gateway allowlists differ, the command can miss the warning for a cron that will fail or warn for one that will run.
  - confidence: 0.93
- **[P2] Reuse the runtime model resolver:** `src/cli/cron-cli/register.cron-edit.ts:220-231`

Cron execution uses `resolveAllowedModelRef` with resolved defaults, aliases, provider inference, auth-profile suffix stripping, and compatibility aliases, so valid refs can warn and invalid refs can be missed.
  - confidence: 0.94
