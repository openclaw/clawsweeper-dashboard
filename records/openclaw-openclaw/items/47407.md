## Summary

Keep open: the requested warning is still useful, but this PR should not merge as-is because it predicts runtime model policy from local CLI state and partial parsing, and contributor proof is still missing.

## What This Changes

The PR adds a non-blocking `cron edit --model` warning by loading local config/catalog data, building an allowed model set, parsing the model ref, and warning when the ref is outside that set.

## Best Possible Solution

Add a gateway-backed or shared runtime validation path for `cron edit --model` that uses the target gateway config and resolver, with coverage for aliases, bare IDs, auth-profile suffixes, default providers, and remote gateway edits.

## Reproduction Assessment

Yes. Source inspection shows current main stores `--model` in `cron.update` without warning, while cron execution later rejects invalid `payload.model` values through `resolveAllowedModelRef`; I did not run a live command in this read-only review.

## Solution Assessment

No. The warning is useful, but this PR predicts runtime behavior from local CLI config and `parseModelRef` instead of using the target gateway's runtime resolver.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Validate against the target gateway:** `src/cli/cron-cli/register.cron-edit.ts:218-219`
  - body: This warning reads local CLI config/catalog state before `cron.update`, but `cron edit` can target a remote gateway with `--url`/`--token`. If local and remote allowlists differ, the command can warn incorrectly or miss the warning for the job that will actually run.
  - confidence: 0.93
- **[P2] Reuse the cron runtime resolver:** `src/cli/cron-cli/register.cron-edit.ts:223-228`
  - body: This builds the allowlist with hard-coded defaults and checks `parseModelRef(model, DEFAULT_PROVIDER)`, while cron execution uses `resolveAllowedModelRef` with runtime defaults, aliases, bare-id provider inference, auth-profile suffix stripping, and compat aliases. Valid cron models can warn incorrectly, and the warning can still disagree with runtime validation.
  - confidence: 0.94
