## Summary

Keep open: the CLI footgun is still present on current main, but this PR should not merge as-is because the warning can diverge from the gateway/runtime model validation path and lacks after-fix real behavior proof.

## What This Changes

The PR adds a non-blocking `cron edit --model` warning in `src/cli/cron-cli/register.cron-edit.ts` by loading config/catalog data and checking the requested model against the allowed model set before `cron.update`.

## Best Possible Solution

Add a gateway-owned cron model warning or preflight that reuses `resolveAllowedModelRef` with the gateway runtime config, catalog, and defaults, with focused coverage for remote-gateway and resolver edge cases.

## Reproduction Assessment

Yes, source-reproducible. Current main writes `cron edit --model` into `patch.payload.model`, while cron execution later rejects disallowed payload models through `resolveAllowedModelRef`; I did not run a live CLI repro in this read-only review.

## Solution Assessment

No. The warning is useful direction, but this implementation reads local CLI config and partially reimplements model validation instead of using the gateway/runtime resolver that will execute the cron.
