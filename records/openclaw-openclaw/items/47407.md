## Summary

Keep open: the user problem is valid, but this PR still warns from local CLI config instead of the selected gateway runtime and duplicates only part of the execution-time model resolver.

## What This Changes

The PR adds a non-blocking `cron edit --model` warning when the requested model appears outside the configured allowed model set.

## Best Possible Solution

Add a gateway-owned cron model warning or preflight that reuses the execution resolver, runtime config, catalog, defaults, and alias handling, with focused regression coverage.

## Reproduction Assessment

Yes, by source inspection: current main accepts `cron edit --model` into `patch.payload.model` without an edit-time warning, while cron execution later rejects disallowed payload models through `resolveAllowedModelRef`. I did not run a live CLI repro in this read-only review.

## Solution Assessment

No. The warning is useful, but this implementation validates from local duplicated CLI logic instead of the selected gateway/runtime resolver that will execute the cron.

## Review Findings

Overall correctness: patch is incorrect

Valid overrides can warn and invalid ones can be missed.
