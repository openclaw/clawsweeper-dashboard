## What This Changes

The PR adds a non-blocking CLI warning when `cron edit --model` appears outside the configured model allowlist.

## Best Possible Solution

Put the warning or preflight behind a gateway/runtime seam that uses the executing gateway's config, resolved defaults, model catalog, and `resolveAllowedModelRef` path.

## Reproduction Assessment

Yes, at source level: current main accepts `cron edit --model` into `patch.payload.model`, while cron execution later rejects disallowed payload models through `resolveAllowedModelRef`. I did not run a live CLI repro in this read-only review.

## Solution Assessment

No. The warning is useful, but this implementation reads local CLI config and partially reimplements model validation instead of using the gateway/runtime resolver that will execute the cron.
