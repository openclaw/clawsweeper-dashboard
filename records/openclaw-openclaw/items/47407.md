## Summary

Keep open: current main still accepts `cron edit --model` without an edit-time allowlist warning, but this PR’s local CLI check can disagree with the gateway/runtime resolver that will actually execute the cron job.

## What This Changes

The PR adds a non-blocking `cron edit --model` warning by checking the requested model against a locally loaded allowed model set before sending `cron.update`.

## Best Possible Solution

Add a gateway-owned cron model warning or preflight that reuses the same runtime resolver, config, catalog, defaults, aliases, and tests as cron execution.

## Reproduction Assessment

Yes by source inspection. Current main accepts `cron edit --model` into `patch.payload.model`, while cron execution later rejects disallowed payload models through `resolveAllowedModelRef`; I did not run a live CLI repro in this read-only review.

## Solution Assessment

No. The warning direction is useful, but this implementation duplicates a partial local resolver instead of validating against the selected gateway/runtime path that will execute the cron job.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Validate against the selected gateway runtime:** `src/cli/cron-cli/register.cron-edit.ts:218-219`
  - body: This warning reads local CLI config/catalog data, but `cron edit` sends `cron.update` to the selected gateway and supports `--url`/`--token`. In remote-gateway setups, local and gateway allowlists can differ, so this can miss the warning for a job that will fail or warn for one that will run.
  - confidence: 0.93
- **[P2] Reuse the cron execution resolver for this warning:** `src/cli/cron-cli/register.cron-edit.ts:220-228`
  - body: The warning builds `allowedKeys` with hard-coded defaults and checks `parseModelRef(model, DEFAULT_PROVIDER)`. Cron execution uses `resolveAllowedModelRef` with configured defaults, aliases, provider inference, auth-profile suffix stripping, and compatibility aliases, so valid models can still warn and invalid ones can be missed.
  - confidence: 0.94
