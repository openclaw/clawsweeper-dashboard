## Summary

Keep open. The requested warning is source-reproducible, but this PR validates local CLI config with a partial resolver instead of the selected gateway/runtime model-selection path, and it still lacks after-fix real behavior proof.

## What This Changes

The PR adds a non-blocking `cron edit --model` warning when the selected model is not in the configured allowlist.

## Best Possible Solution

Move the warning or preflight into a gateway/runtime-owned path that reuses `resolveAllowedModelRef` with the executing gateway's config, catalog, defaults, and focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible. Current main stores `cron edit --model` into the job payload, while cron execution later rejects invalid payload models through `resolveAllowedModelRef`; I did not run a live CLI repro in this read-only review.

## Solution Assessment

No. A useful warning should be gateway/runtime-owned and resolver-backed, not a local CLI reimplementation of allowlist checks.

## Review Findings

Overall correctness: patch is incorrect
