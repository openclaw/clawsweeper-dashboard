## Summary

Keep open. Current main still has the doctor SecretRef false positive, while this PR contains a narrow doctor fix and the task timestamp half is already covered by merged #71871; the remaining blocker is maintainer review and exact-head CI triage.

## What This Changes

This PR lazily resolves SecretRef-backed `gateway.auth.token` in doctor before warning, adds doctor e2e coverage for resolved SecretRefs and non-token auth modes, and adds a changelog entry.

## Best Possible Solution

Review and land the doctor-only SecretRef fix once exact-head CI is understood, while leaving task timestamp behavior anchored to the already shipped #71871 implementation.

## Reproduction Assessment

Yes. Configure local token auth with `gateway.auth.token` as a resolvable SecretRef and no plaintext token, then run `openclaw doctor` on current main; the source path warns before invoking the SecretRef-aware resolver.

## Solution Assessment

Yes. The current PR uses the existing resolver only after token auth is actually required, passes the result through `authOverride`, preserves lazy loading, and adds regression coverage for both resolved SecretRefs and non-token auth modes.
