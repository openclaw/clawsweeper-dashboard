## Summary

Keep open. Current main still has the doctor SecretRef false-positive path, while this PR now contains a narrow doctor fix, regression coverage, and the required changelog entry; the task timestamp portion is already covered by #71871.

## What This Changes

This PR lazily resolves SecretRef-backed gateway auth tokens in `openclaw doctor`, adds doctor e2e coverage for resolved SecretRefs and non-token modes, and adds a changelog entry.

## Best Possible Solution

Land the doctor-only SecretRef fix after branch mergeability and targeted/changed-gate validation; leave task timestamp normalization with the already-merged #71871 implementation.

## Reproduction Assessment

Yes. On current main, a local token-auth config with `gateway.auth.token` as a resolvable SecretRef reaches the token-needed path, but `resolveGatewayAuth` omits SecretRef values and doctor emits the unavailable warning.

## Solution Assessment

Yes. The patch uses the existing SecretRef-aware resolver only after token auth is required, feeds the resolved value through `authOverride`, and adds coverage for both resolved SecretRefs and non-token auth modes.
