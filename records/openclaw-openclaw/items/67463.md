## Summary

Keep open. Current main still reports a SecretRef-managed Gateway token as unavailable before using the SecretRef-aware service resolver, while this PR contains a focused doctor fix; the task timestamp half is already covered by merged #71871.

## What This Changes

This PR lazily resolves a SecretRef-backed `gateway.auth.token` during doctor Gateway auth health, adds e2e coverage for resolved SecretRefs and non-token modes, and records the doctor fix in the changelog.

## Best Possible Solution

Land the doctor-only SecretRef fix after exact-head CI is green or the failures are proven unrelated, and keep task timestamp normalization anchored to #71871.

## Reproduction Assessment

Yes. Configure local token auth with `gateway.auth.token` as a resolvable env SecretRef and no plaintext token, then run `openclaw doctor`; current main reaches the SecretRef warning because `resolveGatewayAuth` ignores configured SecretRef values before `needsToken` is evaluated.

## Solution Assessment

Yes for the remaining doctor bug. The PR reuses the existing SecretRef-aware resolver only after token auth is required and passes the resolved token through `authOverride`; the task timestamp behavior is better left to the already merged #71871 implementation.
