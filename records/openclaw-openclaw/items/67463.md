## Summary

Keep open. Current main still emits the doctor warning before resolving a configured SecretRef token, and this PR is the active narrow fix for that behavior while the task timestamp half is already covered by #71871.

## What This Changes

This PR lazily resolves a SecretRef-backed `gateway.auth.token` in doctor auth health, adds doctor e2e coverage for resolved SecretRefs and non-token modes, and adds a changelog entry.

## Best Possible Solution

Land the doctor-only SecretRef fix after CI is clean, and keep task timestamp behavior anchored to the already merged #71871 implementation.

## Reproduction Assessment

Yes. Configure local token auth with `gateway.auth.token` as a resolvable SecretRef and no plaintext token, then run `openclaw doctor` on current main; the source path warns before invoking the SecretRef-aware resolver.

## Solution Assessment

Yes for the doctor behavior. The PR uses the existing resolver only after token auth is required, passes the resolved token via `authOverride`, and preserves cheap non-token auth paths.
