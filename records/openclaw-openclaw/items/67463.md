## Summary

Keep open: current main still has the doctor false-warning path and this PR is a focused fix candidate, but it should not merge as-is because the new doctor path can execute exec-backed SecretRef providers and the PR still lacks after-fix real behavior proof. The overlapping https://github.com/openclaw/openclaw/pull/77698 and https://github.com/openclaw/openclaw/issues/77687 should be considered during maintainer consolidation.

## What This Changes

This PR changes Gateway doctor auth health to resolve a SecretRef-backed `gateway.auth.token` before warning, adds doctor e2e coverage for resolved and non-token SecretRef cases, and adds a changelog entry.

## Best Possible Solution

Keep the doctor fix narrow: resolve only safe SecretRef-backed Gateway tokens after token auth is required, avoid exec provider execution without an explicit policy or opt-in, keep the regression coverage, and require redacted real `openclaw doctor` proof before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure local token auth with a resolvable `gateway.auth.token` SecretRef and run `openclaw doctor`; current main checks raw auth before SecretRef materialization and reaches the warning branch.

## Solution Assessment

No, not as written. Resolving after the `needsToken` check is the right shape for env/file SecretRefs, but using `resolveGatewayAuthTokenForService` here also executes exec-backed SecretRef providers; doctor should skip or explicitly gate that path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Skip exec SecretRefs during doctor:** `src/flows/doctor-health-contributions.ts:179`
  - body: This calls `resolveGatewayAuthTokenForService` for any token SecretRef once token auth is needed. That helper flows through `resolveSecretRefValues`, and exec providers run their configured command; the secrets docs intentionally make even audit use `--allow-exec` to avoid command side effects. Gate exec refs here or report them as unverified so `openclaw doctor` does not start executing exec-backed token resolvers during a health check.
  - confidence: 0.86
