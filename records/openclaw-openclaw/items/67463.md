## Summary

Keep open: current main still has the doctor SecretRef false positive, and this PR's latest head is a narrow implementation candidate for that remaining bug while the task timestamp half is already covered by merged #71871.

## What This Changes

This PR updates doctor Gateway auth health to resolve SecretRef-backed tokens lazily after token auth is required, adds doctor e2e coverage, and adds a changelog entry.

## Best Possible Solution

Land the doctor-only SecretRef fix after exact-head CI and normal maintainer landing gates are clean, leaving task timestamp normalization anchored to #71871.

## Reproduction Assessment

Yes. On current main, a local token-auth config with a resolvable gateway.auth.token SecretRef reaches the token-needed path, but resolveGatewayAuth omits SecretRef values and doctor emits the unavailable warning.

## Solution Assessment

Yes. The patch uses the existing SecretRef-aware resolver only after token auth is required, passes the resolved value through authOverride, and adds coverage for both resolved SecretRefs and non-token modes.
