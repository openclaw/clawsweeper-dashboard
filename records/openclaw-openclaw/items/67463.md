## Summary

Keep open. The doctor SecretRef false-positive is still present on current main and this PR contains a focused fix plus useful regression coverage, but the current patch would implicitly execute exec-backed SecretRef providers during `openclaw doctor` and still needs contributor real behavior proof and maintainer consolidation with the overlapping implementation.

## What This Changes

The PR updates Gateway doctor auth health to resolve a SecretRef-backed `gateway.auth.token` before warning, adds doctor E2E coverage for resolved and non-token cases, and adds a changelog entry.

## Best Possible Solution

Consolidate into one doctor fix that resolves env/file Gateway token SecretRefs only after token auth is required, avoids implicit exec provider execution, preserves the useful regression coverage, and leaves task timestamp behavior anchored to the shipped https://github.com/openclaw/openclaw/pull/71871 implementation.

## Reproduction Assessment

Yes. Source inspection plus linked reporter evidence gives a high-confidence path: configure local token auth with a resolvable `gateway.auth.token` SecretRef and run `openclaw doctor`; current main detects the SecretRef but never materializes it before warning.

## Solution Assessment

No, not as written. Resolving after the token-needed check is the right shape for env/file SecretRefs, but this PR’s helper path also executes exec-backed providers during a health check, so it needs an exec gate or unverified diagnostic before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Skip exec SecretRefs during doctor:** `src/flows/doctor-health-contributions.ts:179`
  - body: This new `resolveGatewayAuthTokenForService` call runs for any token SecretRef once token auth is needed. That helper reaches `resolveSecretRefValues()`, whose exec provider path invokes `runExecResolver()`; because secrets audit intentionally requires `--allow-exec` to run exec providers, doctor should gate exec refs or report them unverified instead of executing provider commands during a health check.
  - confidence: 0.88
