## Summary

Keep open. Current main still has the doctor false-positive path for resolvable SecretRef gateway tokens, and this PR is a focused implementation candidate; the task timestamp portion is already covered separately by merged PR #71871.

## What This Changes

The PR lazily resolves SecretRef-backed `gateway.auth.token` values in doctor Gateway auth health, adds e2e coverage for resolved SecretRefs and non-token modes, and records the doctor fix in the changelog.

## Best Possible Solution

Review and land the doctor-only SecretRef fix once exact-head CI is green or the unrelated failures are resolved, while leaving task timestamp normalization anchored to #71871.

## Reproduction Assessment

Yes. Configure local token auth with `gateway.auth.token` as a resolvable SecretRef and run `openclaw doctor`; current main uses the synchronous resolver that ignores SecretRef token values and then emits the unavailable warning.

## Solution Assessment

Yes. The PR reuses the existing SecretRef-aware token resolver only after token auth is known to need a token, which is narrower than changing synchronous auth resolution or resolving secrets for non-token modes.
