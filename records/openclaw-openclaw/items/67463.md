## Summary

Keep open. Current main still emits the Gateway SecretRef doctor warning before attempting the SecretRef-aware token resolver, while this PR contains a narrow doctor-path fix and regression coverage; the task timestamp half is already covered by merged #71871.

## What This Changes

The PR lazily resolves SecretRef-backed `gateway.auth.token` values in Gateway doctor auth health, adds doctor e2e coverage for resolved SecretRefs and non-token modes, and records the doctor fix in the changelog.

## Best Possible Solution

Review and land the doctor-only SecretRef fix once exact-head CI is green or the unrelated check failure is resolved, while keeping task timestamp normalization anchored to #71871.

## Reproduction Assessment

Yes. Configure local token auth with `gateway.auth.token` as a resolvable SecretRef and run `openclaw doctor`; current main reaches the SecretRef-unavailable warning because the doctor path never calls the async SecretRef-aware resolver.

## Solution Assessment

Yes. The PR reuses the existing Gateway token resolver only after token auth is known to require a token, which is narrower than changing synchronous auth resolution or resolving SecretRefs for non-token modes.
