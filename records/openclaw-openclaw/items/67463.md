## What This Changes

This PR changes Gateway doctor auth health to resolve a SecretRef-backed `gateway.auth.token` after token auth is required, adds doctor e2e coverage for resolved SecretRefs and non-token auth modes, and records the user-facing fix in `CHANGELOG.md`.

## Best Possible Solution

Land a doctor-only SecretRef fix that preserves lazy resolution after the token-needed decision, with task timestamp normalization left to the already merged #71871 implementation.

## Reproduction Assessment

Yes. Configure local Gateway token auth with `gateway.auth.token` as a resolvable SecretRef and run `openclaw doctor`; current main strips the SecretRef before the doctor token-needed check and can warn even though Gateway startup can resolve the token.

## Solution Assessment

Yes at the code-review level. The PR reuses the existing SecretRef-aware Gateway token resolver only after token auth is required, which is narrower than changing synchronous auth resolution or resolving secrets for non-token modes; merge should still wait for exact-head CI triage.
