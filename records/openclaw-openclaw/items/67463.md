## Summary

Keep open. Current main still has the doctor SecretRef false-positive path, while this PR contains a narrow doctor fix and focused coverage; the task timestamp portion is already covered by #71871.

## What This Changes

This PR changes Gateway doctor auth health to lazily resolve a configured `gateway.auth.token` SecretRef after token auth is required, adds e2e coverage, and records the fix in the changelog.

## Best Possible Solution

Review and land the doctor-only SecretRef fix once exact-head CI is green, while leaving task timestamp normalization anchored to the already shipped #71871 implementation.

## Reproduction Assessment

Yes. Configure local token auth with `gateway.auth.token` as a resolvable SecretRef and run `openclaw doctor`; current main checks `needsToken` before the SecretRef-aware resolver and reaches the false warning path.

## Solution Assessment

Yes. The PR uses the existing async doctor token resolver only after token auth is required, which is narrower than changing the synchronous auth resolver or resolving secrets for non-token modes.
