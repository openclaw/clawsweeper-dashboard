## Summary

Keep open. Current main still exposes the shared `read` message action, but the bundled Telegram plugin does not advertise, map, gate, dispatch, test, or document Telegram message-history reads.

## What This Changes

Add Telegram support for the shared `message read` action so agents can read recent Telegram chat or group messages.

## Best Possible Solution

Implement Telegram-owned observed-message reads with bounded runtime storage, explicit retention/privacy limits, no secondary `getUpdates` polling, and aligned adapter, discovery, runtime, config, docs, and tests.

## Reproduction Assessment

Yes. The source path is high-confidence: the CLI dispatches `read`, shared target plumbing maps `--target` to `to`, and the Telegram adapter/runtime omit `read`, producing an unsupported action path.

## Solution Assessment

Unclear until maintainers approve Telegram read semantics. The requested `message read` surface is appropriate, but a bounded observed-message cache is safer than trying to fetch chat history through Telegram Bot API polling.
