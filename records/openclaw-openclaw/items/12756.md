## Summary

Keep open: current main and v2026.6.6 still expose `read` as a shared action, but Telegram does not advertise, map, dispatch, gate, or document Telegram message-history reads. The related duplicate and closed unmerged PR are useful context, but they do not implement the remaining behavior on main.

## What This Changes

Request adding Telegram support for the `message` tool's `read` action so agents can inspect recent Telegram group messages similarly to Slack or Discord.

## Best Possible Solution

Implement Telegram `message read` inside the Telegram plugin with a bounded runtime-owned history surface, explicit action gating, account/chat/topic scoping, CLI target/thread semantics, public docs, focused regression tests, and maintainer-approved retention and privacy behavior.

## Reproduction Assessment

Yes, at source level. Current main includes `read` as a shared action, but the Telegram plugin does not advertise, map, alias, or handle it, so Telegram `read` reaches the unsupported-action path.

## Solution Assessment

No approved best implementation exists yet. The narrow product direction is clear, but the safest implementation needs maintainer approval for observed-history semantics, action gating, retention, and Telegram access boundaries before coding.
