## Summary

Keep open: current main still exposes `read` as a shared message action, but Telegram does not advertise, map, gate, dispatch, test, or document Telegram message-history reads; the related duplicate and candidate PR did not land a fix.

## What This Changes

The issue requests adding Telegram support for the shared `message read` action so agents can read recent Telegram group messages.

## Best Possible Solution

Implement Telegram `message read` inside the Telegram plugin after maintainer approval of bounded observed-history semantics, storage/retention behavior, access gating, target/topic semantics, docs, focused tests, and live Telegram validation.

## Reproduction Assessment

Yes. At source level, current main has the shared `read` action but Telegram does not advertise, map, or dispatch it, and unsupported Telegram actions throw before execution.

## Solution Assessment

No final implementation is approved yet. The safest direction is a Telegram-plugin-owned bounded observed-message store or equivalent cache, not a second `getUpdates` reader or a core-owned one-off path.
