## Summary

Keep open. Current main still has shared `read` CLI/plumbing, but the bundled Telegram plugin does not advertise, map, gate, dispatch, test, or document Telegram message-history reads; related PR #14367 was closed unmerged.

## What This Changes

Add Telegram support for the shared `openclaw message read` action so agents can read recent observed Telegram chat and group messages.

## Best Possible Solution

Implement Telegram-owned observed-message reads with bounded runtime storage, explicit retention/privacy limits, no secondary `getUpdates` polling, and aligned adapter, discovery, runtime, config, docs, and tests.

## Reproduction Assessment

Yes. The source path is high-confidence: the CLI dispatches `read`, shared target plumbing maps `--target` to `to`, and Telegram adapter/runtime omit `read`, causing the unsupported-action path.

## Solution Assessment

Unclear until maintainers approve Telegram read semantics. The requested user-facing `message read` surface is right, but a bounded observed-message cache is safer than trying to fetch chat history through Telegram Bot API polling.
