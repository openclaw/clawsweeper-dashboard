## Summary

Keep open. Current main still has shared/CLI `read` plumbing, but the Telegram plugin does not advertise, map, gate, dispatch, test, or document Telegram message-history reads; the related inbound-store PR was closed unmerged.

## What This Changes

Add Telegram support for `openclaw message read` so agents can read recent Telegram chat or group messages.

## Best Possible Solution

Implement Telegram-owned observed-message reads with bounded runtime storage, explicit retention/privacy limits, and end-to-end adapter, runtime, config, docs, and test coverage.

## Reproduction Assessment

Yes. The static current-main path is high-confidence: the CLI dispatches `read`, shared target plumbing maps it to `to`, and the Telegram adapter/runtime omit `read`, causing an unsupported action path.

## Solution Assessment

Unclear until maintainers approve Telegram read semantics. The requested user-facing `message read` surface is right, but a direct Bot API history fetch is not the maintainable path; an observed-only bounded cache is safer.
