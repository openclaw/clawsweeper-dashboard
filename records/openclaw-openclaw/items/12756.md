## Summary

Keep open. Current main still has the shared `message read` action and a Telegram observed-message cache, but Telegram does not advertise, map, gate, dispatch, or document `read` as a channel action.

## What This Changes

The issue requests Telegram support for the shared `message read` action so agents can read recent Telegram group messages.

## Best Possible Solution

Add a Telegram-owned observed-message read action backed by bounded message-cache state, with explicit platform-limit docs, target/topic semantics, and read-access gating.

## Reproduction Assessment

Yes for the current limitation: current main has shared/CLI `read` plumbing, while Telegram discovery, mapping, config, docs, and runtime dispatch still omit it. This is a feature gap rather than broken documented Telegram behavior.

## Solution Assessment

Unclear as originally phrased. The maintainable direction is not a direct Bot API history read or a second `getUpdates` poller; it should expose bounded observed-message reads from Telegram-owned runtime state after maintainer decisions on privacy and retention.
