## Summary

Keep open. Current main still has shared and CLI `message read` plumbing plus a Telegram observed-message cache, but the bundled Telegram plugin still does not expose, map, gate, dispatch, test, or document `read` as a Telegram message action.

## What This Changes

Add Telegram support for the shared `message read` action so agents can read recent observed Telegram group messages.

## Best Possible Solution

Add a Telegram-owned observed-message read action backed by bounded cache state, explicit platform-limit docs, target/topic semantics, and read-access gating.

## Reproduction Assessment

Yes for the current limitation: current main has shared/CLI `read` plumbing, while Telegram discovery, mapping, config, docs, and runtime dispatch still omit it. This is a source-reproducible feature gap rather than broken documented Telegram behavior.

## Solution Assessment

Unclear as originally phrased. The maintainable solution is not direct Bot API history access; it should expose bounded observed-message reads from Telegram-owned runtime state after maintainers settle privacy, persistence, and gating.
