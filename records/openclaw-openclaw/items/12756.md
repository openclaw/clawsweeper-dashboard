## Summary

Keep open. Current main still lacks Telegram support for the shared `message read` action, and prior maintainer/ClawSweeper review already identified the remaining product, privacy, and authorization decisions.

## What This Changes

The issue requests Telegram support for `openclaw message read` so agents can inspect recent Telegram group messages.

## Best Possible Solution

Add a Telegram-owned observed-message `read` action backed by the bounded message cache, with explicit platform-limit docs, target/topic semantics, retention policy, access gating, and focused tests.

## Reproduction Assessment

Yes, by source inspection. The shared CLI `read` path exists while Telegram discovery, mapping, config, docs, and runtime dispatch still omit a Telegram read action.

## Solution Assessment

Unclear as originally phrased. The maintainable solution is not a full Telegram Bot API history read or a secondary `getUpdates` poller; it should expose bounded observed-message reads from Telegram-owned runtime state after privacy and retention decisions.
