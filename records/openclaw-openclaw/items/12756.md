## Summary

Keep open. Current main still exposes the shared `read` action and CLI command, but Telegram does not advertise, map, gate, dispatch, test, or document a Telegram `message read` action; the related implementation PR closed unmerged and the duplicate issue was closed back to this one.

## What This Changes

The issue requests Telegram support for the shared `message read` action so agents can read recent Telegram group messages.

## Best Possible Solution

Add a Telegram-owned observed-message read action backed by bounded cache state, with explicit gating, authorization, retention, target/topic semantics, docs, and tests while avoiding a secondary `getUpdates` history reader.

## Reproduction Assessment

Yes. Source inspection on current main shows the shared `read` action and CLI command exist while Telegram discovery, mapping, runtime dispatch, config gates, tests, and docs still omit Telegram read support.

## Solution Assessment

Unclear as originally phrased. The maintainable path is a bounded observed-message read from Telegram-owned cache state after privacy, retention, authorization, and topic semantics are decided; a full history reader or secondary polling path is not the safe default.
