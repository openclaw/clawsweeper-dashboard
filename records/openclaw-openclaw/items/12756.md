## Summary

Keep open. Current main still exposes the shared `read` action, but the Telegram plugin does not advertise, map, gate, dispatch, test, or document Telegram message reads, and the prior inbound-store PR was closed unmerged.

## What This Changes

Add Telegram support for the shared `message read` action so agents can read recent Telegram chat or group messages.

## Best Possible Solution

Implement Telegram-owned observed-message reads with bounded storage, explicit retention/privacy limits, no secondary `getUpdates` polling, and aligned adapter, runtime, config, docs, and tests.

## Reproduction Assessment

Yes. Source inspection on current main shows the CLI dispatches shared `read`, while Telegram discovery, adapter mapping, runtime aliases, config gates, and docs omit `read`, leading to the unsupported-action path.

## Solution Assessment

Unclear. The requested surface is reasonable, but the maintainable solution should avoid direct `getUpdates` history reads and needs maintainer-approved observed-message retention and privacy semantics first.
