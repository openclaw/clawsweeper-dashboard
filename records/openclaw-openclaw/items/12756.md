## Summary

Keep open. Current main still has shared `message read` plumbing, but Telegram does not advertise, map, gate, dispatch, test, or document a Telegram read action; the prior focused implementation PR was closed unmerged and the duplicate issue was closed in favor of this one.

## What This Changes

Request to add Telegram support for `openclaw message read` so agents can read recent Telegram chat messages.

## Best Possible Solution

Add a Telegram-owned observed-message `read` action backed by the existing cache, with explicit platform-limit docs, target/topic semantics, retention policy, action gating, and focused tests.

## Reproduction Assessment

Yes for the current limitation by source inspection: shared/CLI `read` exists, while Telegram discovery, mapping, config, docs, and runtime dispatch still omit it. I did not run the CLI because this review is read-only.

## Solution Assessment

Unclear as originally phrased. The maintainable solution is not direct full-history parity with Discord/Slack, but an observed-message Telegram read backed by OpenClaw runtime state after privacy and retention decisions.
