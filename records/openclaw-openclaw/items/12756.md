Implement Telegram-owned observed-message reads using the existing message cache or a bounded successor, with explicit action gating, authorization and retention policy, target/topic semantics, docs, and focused tests, without secondary `getUpdates` polling.

## Risks / Open Questions

- A Telegram read action would expose observed or persisted Telegram message content, so maintainers need explicit authorization, retention, persistence, and group/topic boundary decisions before implementation.

- Bot API 10.0 added some new message-reading surface for user personal chats, but it does not replace a plugin-owned design for configured Telegram groups/chats.
