## Summary

Keep open. Current main still lacks Telegram support for the shared message read action, and the remaining work needs explicit Telegram retention, authorization, and platform-scope decisions before implementation.

## What This Changes

The issue requests Telegram support for the shared message read action so automation can inspect recent Telegram group messages.

## Best Possible Solution

Add a Telegram-owned observed-message read action backed by the existing persisted message cache, with explicit platform-limit docs, retention/authorization gating, target/topic semantics, and focused tests.

## Reproduction Assessment

Yes, by source inspection. The shared CLI read path exists while Telegram discovery, mapping, config, docs, and runtime dispatch still omit a Telegram read action.

## Solution Assessment

Unclear as originally phrased. The maintainable path is not a full Telegram group-history reader or secondary getUpdates poller; it should expose bounded observed-message reads from Telegram-owned runtime state after privacy and retention decisions.
