## Summary

Keep open. Current main still lacks the requested subagent operating-principle and max-failure-attempt config, and the compaction request is only partially covered by existing reserve-token based controls rather than the proposed ratio/message-count API. The linked implementation PR was closed unmerged, so there is no merged fix to close against.

## What This Changes

Request to add config under `agents.defaults.subagents` and `agents.defaults.compaction` for subagent prompt operating principles, failure retry attempts, proactive compaction ratio, and recent-message retention.

## Best Possible Solution

Keep this open for a narrowed design decision: either add a conservative, validated core config surface for subagent prompt policy and retry budgets with docs/tests, or explicitly choose a plugin/API extension path for prompt customization while documenting how existing reserve-token compaction controls cover the threshold use case.
