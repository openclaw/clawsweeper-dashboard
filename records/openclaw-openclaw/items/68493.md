## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/68493 as a narrow Windows gateway restart bug. The observable problem is that ordinary non-update config/SIGUSR1 restarts under Windows Scheduled Task markers still go through `restartGatewayProcessWithFreshPid` -> `triggerOpenClawRestart` -> supervised full-process handoff and exit the running gateway, which can leave stale locks or race the port bind. Keep update restarts that require a fresh process on the existing hard-respawn/scheduled-task path. Prefer making ordinary config/SIGUSR1 restarts deterministic on Windows, likely by using the in-process restart path or adding a health-gated scheduled-task handoff that cannot exit before a replacement is reachable. Preserve existing duplicate schtasks `/Run` protection, stale listener cleanup, and supervised lock recovery behavior. Add/update focused regression tests around `src/infra/process-respawn.ts`, `src/cli/gateway-cli/run-loop.ts`, `src/infra/supervisor-markers.ts`, `src/infra/windows-task-restart.ts`, `src/infra/gateway-lock.ts`, and adjacent restart-health tests as needed. Add a changelog entry because the fix is user-visible. Do not add a new config option or change Linux/macOS restart policy unless necessary for the Windows bug.
