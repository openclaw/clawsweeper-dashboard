## Summary

Keep open. Current main still has the import-heavy embedded attempt runner and Discord monitor boundaries described in the report, and the related #70743/#70772 work added targeted runtime seams without completing this broader split.

## What This Changes

The issue asks to reduce import coupling in `src/agents/pi-embedded-runner/run/attempt.ts` and Discord monitor files by introducing prepared context or service bundles instead of broad direct imports.

## Best Possible Solution

Keep this as the canonical staged refactor: introduce behavior-preserving prepared service/context bundles around `runEmbeddedAttempt`, then narrow Discord preflight/process/provider responsibilities behind local typed helpers and tests.

## Reproduction Assessment

Not applicable as a runtime bug, but yes for the static architecture signal. Current-main import counts, file sizes, `runEmbeddedAttempt` inline setup, harness param forwarding, and Discord broad context/process signatures reproduce the reported coupling.

## Solution Assessment

Yes directionally, but not as one broad DI rewrite. A staged runner context plus narrower Discord helper/context types is the safer maintainable solution.
