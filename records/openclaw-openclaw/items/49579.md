## Summary

Keep open. The branch is not safe as-is because it suppresses every WhatsApp disconnected system event, including the generic line current main queues for terminal and retry close decisions, and it still lacks after-fix real WhatsApp runtime proof.

## What This Changes

The PR adds a shared prompt-assembly filter for WhatsApp gateway connected/disconnected system events and focused drainFormattedSystemEvents tests.

## Best Possible Solution

Replace the shared prefix filter with a WhatsApp-owned lifecycle notification policy that suppresses only short routine flaps while preserving operator-actionable channel-health context, coordinated with https://github.com/openclaw/openclaw/issues/64624.

## Reproduction Assessment

Yes by source inspection. The PR branch filters all `WhatsApp gateway disconnected` lines, while current main queues that line for every WhatsApp close decision before the detailed runtime-error/log branches.

## Solution Assessment

No. A shared prefix filter is too broad; the safer solution should classify routine WhatsApp lifecycle flaps near the WhatsApp source, or introduce a channel-owned policy, while preserving terminal disconnect context.
