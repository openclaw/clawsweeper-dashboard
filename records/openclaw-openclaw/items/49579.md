## Summary

Keep open. Current main still exposes WhatsApp gateway connect/disconnect system events in prompt assembly, and the PR’s blanket formatter filter would also remove terminal disconnect context that the WhatsApp monitor currently only queues as the generic disconnect banner.

## What This Changes

The PR filters WhatsApp gateway connected/disconnected banners during `drainFormattedSystemEvents` and adds focused formatting tests for suppression and preservation behavior.

## Best Possible Solution

Suppress routine connect/retry transport flaps at the WhatsApp event source or with classified event metadata, while preserving logged-out, conflict, and max-attempt-stop prompt-visible health context.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the PR drops every `WhatsApp gateway disconnected` line, while current main queues that generic line before terminal close handling and does not enqueue the detailed terminal guidance as a system event.

## Solution Assessment

No. A blanket formatter-level disconnect filter is not the best fix because the formatter lacks the WhatsApp close decision; the safer solution is to filter only routine classified flaps or enqueue distinct terminal system events first.
