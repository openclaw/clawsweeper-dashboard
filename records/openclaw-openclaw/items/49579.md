## Summary

Keep this PR open, but it should not merge as-is: the blanket formatter-level disconnect filter can hide terminal WhatsApp channel-health context that current main only queues as the generic disconnect system event.

## What This Changes

The PR filters WhatsApp gateway connected/disconnected system-event banners during auto-reply prompt assembly and adds focused tests for suppression and preservation behavior.

## Best Possible Solution

Classify WhatsApp gateway events at the source, or otherwise filter only retryable connect/retry flaps while preserving logged-out, conflict, and max-attempt-stop health context in prompt-visible system events.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the PR drops every `WhatsApp gateway disconnected` line, while current main queues that generic line before terminal close handling and does not queue the detailed terminal guidance as a system event.

## Solution Assessment

No. A blanket formatter-level prefix filter is not the best fix because the formatter lacks the WhatsApp close decision; the narrower maintainable fix is to suppress only routine classified flaps or enqueue distinct terminal system events first.
