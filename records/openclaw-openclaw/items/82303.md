## Summary

Keep open: this PR proposes a new opt-in Telegram streaming feature that is not on current main, lacks real Telegram behavior proof, and needs owner judgment because it changes progress mode from the documented single status draft model to a two-lane visible preview model.

## What This Changes

The PR adds an opt-in `streaming.progress.assistantPreview` setting for Telegram progress mode, routes assistant partials to a separate transient preview lane, updates config/schema metadata, and adds focused tests.

## Best Possible Solution

If Telegram owners approve the two-lane UX, keep it opt-in, document it in the public Telegram/streaming docs and config UI hints, and land only with real Telegram proof; otherwise preserve progress mode's single status draft model.

## Reproduction Assessment

Not applicable: this is a new opt-in Telegram streaming feature, not a broken existing behavior report. The required observable check is a real Telegram run with `streaming.mode: "progress"` and `streaming.progress.assistantPreview: true`.

## Solution Assessment

No, not as submitted: the code path is narrow, but the public docs/config hints and real Telegram proof are missing, and the two-visible-preview behavior needs maintainer agreement before it is the right product direction.
