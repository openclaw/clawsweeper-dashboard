## Summary

## What This Changes

The PR adds an opt-in `streaming.progress.assistantPreview` config, routes Telegram progress-mode assistant partials to a separate transient assistant draft lane, updates generated config metadata, and adds focused tests.

## Best Possible Solution

Revise the branch to rearm or recreate the assistant preview stream after clears, align the new config/docs surface if maintainers accept the UX, and land only after redacted live Telegram before/after proof shows the progress draft plus assistant preview lane.

## Reproduction Assessment

Yes for the PR-introduced bug by source inspection: PR head clears the assistant stream on an assistant-message boundary and resets only lane bookkeeping, while current draft-stream controls ignore updates once stopped. The requested assistant preview lane itself is a new feature, not a current-main bug reproduction.

## Solution Assessment

No. The additive config direction is plausible, but this implementation must honor the draft-stream reset contract and prove the actual Telegram chat behavior before it is a maintainable solution.
