## What This Changes

This PR adds opt-in Telegram `streaming.progress.assistantPreview`, routes progress-mode assistant partials to a separate transient draft lane, updates config/schema metadata, and adds focused tests.

## Best Possible Solution

Keep the opt-in design only if the Telegram UX is accepted, reset or recreate the assistant draft stream after clearing it, document the config surface, and require redacted live Telegram delivery proof before merge.

## Reproduction Assessment

Yes for the PR regression by source inspection: `clear()` stops the draft stream and later `update()` ignores stopped streams, while the added boundary path only resets lane state. The requested capability itself is a new feature, not a current-main bug reproduction.

## Solution Assessment

No. The feature direction is narrow, but this implementation needs a stream reset/recreation after assistant-lane clears and real Telegram proof before it is a maintainable merge candidate.
