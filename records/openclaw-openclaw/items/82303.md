## Summary

Keep open: this is an active Telegram feature PR, but the patch should not merge yet because the assistant preview lane can be left in a stopped draft-stream lifecycle and the visible Telegram behavior still lacks real delivery proof.

## What This Changes

The PR adds `streaming.progress.assistantPreview` config/schema support and a Telegram assistant draft lane for answer partials during progress mode, plus tests and regenerated channel metadata.

## Best Possible Solution

Revise the branch to reset or recreate the assistant draft stream after every clear, align the user-facing config/docs surface, and land only after redacted live Telegram proof shows the progress draft and assistant preview lane together.

## Reproduction Assessment

Yes for the PR-introduced regression by source inspection: the branch clears the assistant stream and only resets lane state, while current draft-stream controls ignore updates after `stopForClear`. The underlying assistant preview lane is a new feature rather than a current-main bug reproduction.

## Solution Assessment

No. The additive config direction is plausible, but this implementation must honor the draft-stream lifecycle and provide real Telegram delivery proof before it is the maintainable solution.
