The issue requests an opt-in runtime mechanism that detects durable user preference signals, runs a reflection sub-turn, writes vetted rules into SOUL.md through a structured tool, notifies the user, and supports undo.

Define an explicit opt-in SOUL/persona persistence contract with user-visible notification, undo, provenance/audit, hostile-input coverage, and a decided core-versus-plugin owner boundary before implementation.
