## What This Changes

Request an opt-in runtime mechanism that detects durable user preferences, runs a separate reflection sub-turn, and persists approved rules into SOUL.md with notification and undo support.

## Best Possible Solution

Design this as an explicit opt-in persona or memory lifecycle contract with notification, undo, auditability, and security tests, reusing plugin/API seams where they are sufficient and adding core runtime only for missing extension points.
