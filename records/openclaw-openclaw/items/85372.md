This PR adds a generic `isStatusNotice` reply payload marker, sends the first runtime plan update as a transient status notice, and excludes status notices from final-content, TTS, commitment, and reply-threading accounting.

## Best Possible Solution

Keep the status-notice accounting abstraction, but preserve `/verbose off` suppression by default unless maintainers explicitly approve an always-on plan-notice policy with docs and upgrade proof.
