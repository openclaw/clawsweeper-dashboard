## Summary

Keep this PR open, but it should not merge as-is: the LAN display URL bug is still source-reproducible on current main, while the branch is conflicting, misses an active status rendering path, lacks real behavior proof, and bundles unrelated security/privacy-sensitive changes.

## What This Changes

The PR normalizes several Control UI display URL callers for `gateway.bind: "lan"`, changes embedded-agent error formatting/context propagation, and adds a Drop Box document-intake skill.

## Best Possible Solution

Land a narrow current-main fix that centralizes display-only LAN-to-loopback normalization for Control UI URLs across dashboard, configure completion, status overview, daemon status, and setup finalization, while keeping probe/runtime bind behavior unchanged and splitting agent error-copy plus Drop Box skill work into separate reviews.

## Solution Assessment

No. LAN-to-loopback display normalization is the right direction, but this PR is not the best implementation because it targets stale status code, is conflicting, and bundles unrelated security/privacy-sensitive changes.
