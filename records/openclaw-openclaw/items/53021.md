## Summary

Keep open: the underlying LAN display-URL bug still exists on current main, but this PR should not merge as-is because it targets stale status code and bundles unrelated user-facing error-copy plus Drop Box skill changes with security/privacy concerns.

## What This Changes

The branch normalizes some Control UI LAN display links, adds embedded-agent error-message context/workaround copy, and adds a Drop Box document-intake skill with a PDF renderer.

## Best Possible Solution

Land a narrow current-main fix that centralizes display-only LAN-to-loopback normalization across dashboard, configure completion, status overview, daemon status, and setup finalization while leaving probe/runtime bind behavior unchanged; split agent error copy and Drop Box skill work into separate reviews.

## Reproduction Assessment

Yes. Source inspection is high-confidence: raw `gateway.bind: "lan"` resolves to a private LAN host, and current status/configure/daemon/setup display callers still pass the raw bind while the browser origin check rejects unallowlisted LAN origins.

## Solution Assessment

No. The LAN-to-loopback direction is correct, but this diff is not the best fix because it targets stale status code and bundles unrelated security-sensitive changes; a shared display resolver or normalization helper is safer.
