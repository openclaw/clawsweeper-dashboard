## What This Changes

The PR normalizes LAN Control UI display URLs in configure/status callers, adds embedded-agent error-message context/workaround copy, and adds a bundled Drop Box document-intake skill with a PDF renderer.

## Best Possible Solution

Land a narrow current-main fix that centralizes display-only LAN-to-loopback normalization across all Control UI display URL surfaces, while splitting or removing the agent diagnostics and Drop Box skill work.

Yes, by source inspection: raw `lan` bind resolves to a LAN IPv4 host, and active configure/status/daemon/setup display callers still pass that raw bind while dashboard normalizes it for browser display.

## Solution Assessment

No. LAN-to-loopback display normalization is the right direction, but this PR patches stale status code and bundles unrelated agent error-copy and skill changes; a current-main helper or formatter fix is narrower.
