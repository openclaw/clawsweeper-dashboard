## Summary

Keep open. Current main still has the Control UI LAN display URL problem, but this PR should not merge as-is because the status fix is stale against the active formatter and the branch includes unrelated embedded-agent error-copy and Drop Box intake changes with privacy/security implications.

## What This Changes

The PR normalizes some Control UI display URL call sites from LAN bind to loopback, changes embedded-agent transient and transport error text/payload handling, and adds a Drop Box document-intake skill plus PDF renderer.

## Best Possible Solution

Land a narrow current-main fix that centralizes display-only LAN-to-loopback normalization for browser Control UI URLs across dashboard, configure completion, status overview/status-all, daemon status, and setup finalization while preserving probe/runtime bind, custom bind, tailnet, basePath, and TLS semantics. Split embedded-agent error-copy and Drop Box intake work into separate reviews.

## Reproduction Assessment

Yes. Configure gateway.bind to "lan" and inspect the current display call sites: resolveControlUiLinks returns a LAN IP for raw LAN binds, while configure/status/daemon/setup still pass raw bind values into user-facing URL rendering.

## Solution Assessment

No. The LAN-to-loopback display direction is right, but this branch is not the best fix because it misses the active status formatter and bundles unrelated security/privacy-sensitive changes; a small shared display resolver or normalization helper is safer.
