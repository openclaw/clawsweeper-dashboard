Keep open: the LAN display URL bug is still source-reproducible on current main, but this branch is not suitable to merge as-is because it targets a stale status path, bundles unrelated agent error-copy and skill work, and lacks after-fix real behavior proof.

The PR normalizes selected Control UI display URL callers from `lan` to `loopback`, modifies embedded-agent error copy/context propagation, and adds a Drop Box document-intake skill.

## Best Possible Solution

Land a narrow current-main fix that centralizes display-only LAN-to-loopback normalization for user-facing Control UI/Dashboard links across status, configure, daemon status, and setup finalization while leaving runtime bind and probe semantics unchanged.

## Solution Assessment

No. LAN-to-loopback display normalization is the right direction, but this PR targets a stale status renderer and bundles unrelated security/privacy-sensitive changes that should be split or dropped.
