The PR branch normalizes some Control UI display URL callers for `gateway.bind: "lan"`, changes embedded-agent error formatting/context propagation, and adds a Drop Box document-intake skill with a PDF renderer.

## Best Possible Solution

Land a narrow current-main fix that centralizes display-only LAN-to-loopback normalization for Control UI URLs across dashboard, configure completion, status overview, daemon status, and setup finalization, while leaving probe/runtime bind behavior unchanged and splitting the agent error-copy and Drop Box skill work into separate reviews.

No. LAN-to-loopback display normalization is the right direction, but this PR is not the best implementation because it targets stale status code and bundles unrelated security/privacy-sensitive changes.
