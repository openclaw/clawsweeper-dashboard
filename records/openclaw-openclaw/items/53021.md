## Summary

Keep open. The LAN-to-loopback display direction is valid, but this PR is not mergeable as-is because it misses current-main status rendering and bundles unrelated security/privacy-sensitive agent error-copy and Drop Box skill changes.

## What This Changes

The PR normalizes some Control UI LAN display-link callers and also changes embedded-agent error-message context handling plus adds a Drop Box document-intake skill.

## Best Possible Solution

Land a narrow current-main fix that centralizes display-only LAN-to-loopback normalization for Control UI URLs across dashboard, configure completion, status overview/status-all, daemon status, and setup finalization while leaving probe/runtime bind behavior unchanged and splitting unrelated agent error-copy and Drop Box skill work into separate reviews.

## Reproduction Assessment

Yes. Set `gateway.bind` to `lan` or inspect the call graph: `resolveControlUiLinks` returns a LAN IP for raw LAN binds, while current configure/status/daemon/setup display call sites still pass raw bind values.

## Solution Assessment

No. The normalization direction is right, but this branch is not the best fix because it misses the active formatter and bundles unrelated security-sensitive changes; a small shared display resolver or normalization helper is safer.
