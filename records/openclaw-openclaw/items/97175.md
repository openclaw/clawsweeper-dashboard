## Summary

Keep open: the patch is a credible fix with sufficient runtime proof, but merge still needs maintainer acceptance of the new default timeout semantics and one config-doc alignment fix.

## What This Changes

This PR adds a configurable 120000ms default timeout for deferred context-engine turn maintenance, fences late transcript/task side effects after timeout, throttles repeated stalled-session diagnostics, and updates tests, config schema/help, and docs.

## Best Possible Solution

Land the timeout-and-fence approach after maintainers explicitly accept or adjust the default/disable semantics and the canonical agent config guide documents the new key.

## Reproduction Assessment

Yes at source level: current main and v2026.6.10 enqueue deferred turn maintenance without taskTimeoutMs, and the PR body includes redacted real-gateway logs showing the fixed timeout/fence path. I did not rerun the live container reproduction in this read-only review.

## Solution Assessment

Mostly yes: using the existing command-lane task timeout plus a synchronous late-side-effect fence is the narrowest owner-boundary fix I found. The unresolved part is the product decision around the 120s default and upgrade behavior, not the core mechanism.
