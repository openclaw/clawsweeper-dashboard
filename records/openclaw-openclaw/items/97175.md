## Summary

Keep open: the PR addresses the deferred turn-maintenance stall with the right core seam, but the new defaulted timeout changes upgrade behavior for background context-engine maintenance and the public agent config guide is still missing the new key.

## What This Changes

The PR adds a configurable per-task timeout and late-side-effect fence for deferred context-engine turn maintenance, throttles repeated `session.stalled` diagnostics, updates config metadata/docs, and adds regression coverage.

## Best Possible Solution

Land the timeout and fence only after maintainers explicitly accept or adjust the default upgrade semantics and the canonical agent config guide documents the new key.

## Reproduction Assessment

Yes. Current main and `v2026.6.10` enqueue deferred turn maintenance without `taskTimeoutMs` even though the command queue supports task timeouts, and the linked issue supplies production stall logs; I did not rerun the live lock-contention scenario.

## Solution Assessment

Mostly yes. Using the existing command-lane timeout seam plus a late-side-effect fence is the narrow owner-boundary fix, but the timeout default remains a maintainer compatibility decision and the public config docs need alignment.
