## Summary

Keep open: current main and the latest release still lack a deferred turn-maintenance timeout, and this PR is a viable candidate fix, but the new default timeout remains compatibility-sensitive and the public config guide still needs the new key documented before maintainer acceptance.

## What This Changes

Adds a configurable per-task timeout and timeout fence for deferred context-engine turn maintenance, throttles repeated session.stalled diagnostics, and updates config metadata, docs, and tests.

## Best Possible Solution

Land the timeout-and-fence mechanism after maintainers explicitly accept or adjust the default/disable semantics and the canonical agent config guide documents the new key.
