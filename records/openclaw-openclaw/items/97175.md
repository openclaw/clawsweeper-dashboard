## Best Possible Solution

Land the queue timeout and fence after maintainers explicitly approve or adjust the default semantics and the canonical agent config guide documents the new key.

## Solution Assessment

Mostly yes. Using the existing command-lane timeout seam plus a late-side-effect fence is the narrow owner-boundary fix, but the timeout default remains a maintainer compatibility decision and the public config docs need alignment.
