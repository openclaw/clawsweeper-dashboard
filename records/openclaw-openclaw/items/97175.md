## Summary

Keep open: this is a focused candidate fix for a real source-visible session stall, but it is not merge-ready because the timeout path frees the lane while the maintenance worker can still run, the new default/config surface needs docs and upgrade acceptance, and the external PR has no after-fix real behavior proof.

## What This Changes

The branch adds a configurable 120s per-task timeout to deferred context-engine turn maintenance, cancels the task record on timeout, throttles repeated `session.stalled` diagnostics, and adds focused tests plus config schema metadata.

## Best Possible Solution

Keep the command-lane timeout only with cooperative abort or generation fencing for late maintenance effects, document the new timeout knob and default, and require redacted real setup proof before merge.

## Reproduction Assessment

Yes at source level: current main enqueues deferred turn maintenance without task-timeout options while the command queue supports them. I did not run a live lossless-claw lock-contention reproduction, so the exact production stall remains issue-log-backed rather than locally reproduced here.

## Solution Assessment

No as submitted: using the existing command-lane timeout is a plausible repair direction, but releasing the lane without aborting or fencing the worker is not the safest fix. The maintainable path is timeout plus late-side-effect fencing, public docs for the config surface, and real runtime proof.
