## Best Possible Solution

Land a segment-aware filename-only guard that rejects separators and `.`/`..` path segments, preserves legal dotted filenames and the full-path override, and includes redacted Windows proof.

## Solution Assessment

No, not as written: the filename-only direction is right, but substring matching `..` is broader than the security boundary requires. A segment-aware guard plus proof for the unchanged `OPENCLAW_TASK_SCRIPT` override is safer.
