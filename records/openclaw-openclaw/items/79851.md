## Summary

Keep open: this PR targets a real Feishu quoted-media bug that is still present on current main, but it is not merge-ready because quote-only replies can return before quoted media is fetched and the external PR still lacks after-fix live Feishu proof.

## What This Changes

The PR preserves raw Feishu fetched-message content, resolves quoted parent media through the existing Feishu media resolver, merges it into the agent media payload, and adds focused Feishu tests.

## Best Possible Solution

Land one current-main Feishu fix that preserves raw fetched parent content, resolves quoted media before quote-only messages can be skipped, keeps visibility checks, and includes focused regression coverage plus redacted real Feishu proof.

## Reproduction Assessment

Yes, at source level: current main fetches quoted parent messages as parsed placeholder text and builds MediaPaths before the parent fetch, so quoted file/image media cannot reach the agent context. I did not run a live Feishu account in this read-only review.

## Solution Assessment

No, not as patched yet. Preserving rawContent and reusing the existing media resolver is the right direction, but quoted-media lookup must happen before the empty-message skip or the skip must account for quoted media, and merge should wait for real Feishu proof.
