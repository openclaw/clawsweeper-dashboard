## Summary

Keep open. The PR targets a real source-reproducible Feishu quoted-media bug, but the patch still misses quote-only replies where the bot mention is stripped before the empty-message guard, and external real Feishu proof is still absent.

## What This Changes

The PR preserves raw Feishu fetched-message content, resolves media from quoted parent messages, attaches that media to the agent context, and adds focused Feishu tests.

## Best Possible Solution

Land one current-main Feishu fix that preserves raw fetched parent content, resolves quoted media before empty quote-only messages are skipped, keeps visibility checks, and includes focused regression coverage plus redacted real Feishu proof.

## Reproduction Assessment

Yes at source level: current main fetches only parsed placeholder quoted content and builds MediaPaths before fetching the parent, so quoted file/image media cannot reach the agent. I did not run a live Feishu account.

## Solution Assessment

No, not as patched yet. Preserving rawContent and reusing the existing media resolver is the right direction, but the quoted-media fetch must happen before the empty-message skip and merge should wait for real Feishu proof.
