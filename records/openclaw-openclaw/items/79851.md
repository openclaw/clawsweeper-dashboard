## Summary

Keep open: the PR targets a real source-reproducible Feishu bug, but the patch still misses empty current-message replies such as mention-only quoted media, and it lacks the required after-fix real Feishu proof.

## What This Changes

The branch preserves raw Feishu quoted-message content, resolves quoted media through the existing Feishu media path, and adds regression coverage for quoted file media in agent context.

## Best Possible Solution

Resolve quoted parent media before deciding the inbound turn is empty, keep the existing visibility and max-byte guards, add coverage for mention-only quoted media, and land one non-duplicative Feishu fix with live proof.

## Reproduction Assessment

Yes, from source inspection: current main fetches quoted parent text only after resolving current-message media and after the empty-message return. Since fetched file/image parents expose only parsed placeholder content, their raw media keys cannot reach MediaPaths.

## Solution Assessment

No, not yet: preserving rawContent and reusing resolveFeishuMediaList is the right owner-boundary direction, but the new block is placed too late for empty current-message replies and the PR lacks real Feishu proof.
