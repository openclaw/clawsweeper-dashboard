## Summary

Keep open. The PR targets a real Feishu quoted-media gap on current main, but the patch still misses quote-only bot-mention replies because quoted-media resolution runs after the empty-message guard, and the external PR still lacks redacted real Feishu behavior proof.

## What This Changes

The PR preserves raw Feishu fetched-message content, resolves media from allowed quoted parent messages, attaches those media paths to the agent context, and adds focused Feishu tests.

## Best Possible Solution

Land one current Feishu fix that preserves raw fetched parent content, resolves quoted media before empty quote-only messages are skipped, keeps existing visibility checks, includes focused regression coverage, and is backed by redacted real Feishu proof.

## Reproduction Assessment

Yes, from source inspection: current main fetches only parsed parent placeholder text and builds MediaPaths before the parent fetch, so a Feishu reply to a file/image parent cannot pass that media to the agent. I did not run a live Feishu account.

## Solution Assessment

No, not yet. Preserving rawContent and reusing the existing media resolver is the right direction, but the quoted-media resolution needs to happen before or be considered by the empty-message skip, and merge should wait for redacted real Feishu proof.
