## Summary

Keep open: current main still has the Feishu quoted-media gap, and this PR is the current focused replacement for the older superseded attempts. The direction is right, but the patch still skips mention-only replies before quoted media can be resolved and it lacks required real Feishu behavior proof.

## What This Changes

The PR preserves raw fetched Feishu message content, resolves media from quoted Feishu parent messages, and extends Feishu bot/send regression tests.

## Best Possible Solution

Update the Feishu fix so visible quoted media is resolved before empty-message filtering, keep raw parent content in the fetched-message contract, preserve visibility checks, and land with focused tests plus redacted live Feishu proof.

## Reproduction Assessment

Yes, from source inspection: current main fetches parent text after building current-message media and returns only placeholder parent content without raw media keys. A Feishu reply to a file or image parent can therefore reach the agent with `ReplyToBody` only and no `MediaPaths`.

## Solution Assessment

No as submitted: the core approach is the right owner-boundary fix, but the quoted-media work still runs after the empty-message return, so mention-only replies can be skipped before parent media is attached. Move or defer that guard, then require live Feishu proof before merge.

## Review Findings

- **[P2] Resolve quoted media before empty-message filtering:** `extensions/feishu/src/bot.ts:1020-1027`

Move the quoted-media lookup ahead of that guard or defer the guard until after visible parent media is merged.
