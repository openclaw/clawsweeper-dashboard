## Summary

Keep open. The Feishu quoted-media bug is valid and the PR targets the right surfaces, but the patch still skips empty quote-only replies before resolving parent media and the external PR has only test/lint proof rather than redacted live Feishu behavior proof.

## What This Changes

The PR preserves raw Feishu fetched-message content, resolves media from quoted parent messages, merges it into agent MediaPath/MediaPaths/MediaTypes, and adds focused Feishu tests.

## Best Possible Solution

Land one current-main Feishu fix that preserves raw fetched parent content, resolves quoted media before empty-message skipping, keeps existing visibility checks, includes focused regression coverage, and is backed by redacted real Feishu proof.

## Reproduction Assessment

Yes, from source inspection: on current main a Feishu reply to a file/image parent fetches only parsed placeholder text because raw media keys are not retained and MediaPaths is built before the quote fetch. I did not run a live Feishu account, so this is source-reproducible rather than live reproduced.

## Solution Assessment

No, not quite yet. Preserving rawContent and reusing the existing media resolver is the right direction, but quoted-media resolution needs to run before the empty-message guard and merge should wait for redacted real Feishu proof.
