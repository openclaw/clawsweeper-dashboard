## What This Changes

The PR preserves raw Feishu fetched-message content, resolves media from quoted parent messages, attaches the downloaded media to agent context, and adds focused Feishu tests.

## Best Possible Solution

Land one current Feishu fix that resolves quoted parent media before empty-message skipping, preserves existing visibility and media size guards, includes focused regression coverage, and is backed by redacted real Feishu proof.

## Solution Assessment

No, not yet. Preserving rawContent and reusing the existing media resolver is the right direction, but the added quoted-media fetch needs to run before the empty-message guard or be accounted for by that guard, and merge should wait for real Feishu proof.
