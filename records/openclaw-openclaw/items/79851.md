## Summary

Keep open. The PR is aimed at the right Feishu code path, but the quoted-media merge still runs after the empty-message guard, so a quote reply that only mentions the bot can return before the parent media is fetched; the external PR also lacks required real Feishu proof.

## What This Changes

The PR preserves raw Feishu fetched-message JSON and resolves media from quoted parent messages into the agent `MediaPath`, `MediaPaths`, and `MediaTypes` context fields with focused Feishu tests.

## Best Possible Solution

Land one current-main Feishu fix that fetches and merges quoted media before the empty-message skip, preserves visibility checks, covers quote-only mention replies, and includes redacted live Feishu proof.

## Reproduction Assessment

Yes, from source inspection: current main reduces fetched quoted file/image messages to placeholder text and builds `MediaPaths` before fetching the parent. I did not run a live Feishu account, so this is source-reproducible rather than live reproduced.

## Solution Assessment

No, not quite yet. Preserving `rawContent` and reusing the existing media resolver is the right direction, but the quoted-media fetch must happen before the empty-message guard or the guard must account for quoted media.
