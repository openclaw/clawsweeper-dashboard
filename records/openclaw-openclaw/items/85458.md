## What This Changes

The issue reports a possible Codex app-server conversation-thread subscription leak when missing-thread recovery creates a replacement thread and the retry then rejects without cleanup.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/85496, or close that PR and keep this issue as the canonical source-level bug until an equivalent bound-turn unsubscribe fix is merged.

## Reproduction Assessment

Yes, at source level: a targeted mock can make the first `turn/start` throw `thread not found`, recovery create `thread-new`, and the retry reject while current main has no bound-turn `thread/unsubscribe` cleanup. I did not run a live Codex hosted-session reproduction during this read-only review.

## Solution Assessment

Yes: adding best-effort `thread/unsubscribe` cleanup to the bound-turn finalizer, with retry-failure regression coverage, is the narrow maintainable fix and should preserve the persisted replacement binding on generic retry errors.
