## Summary

Keep open. Current main still has the startup-task rejection path this PR targets, but the patch broadens graceful failure handling through the shared runtime registry loader and still needs focused tests, better rejection logging, and a changelog entry.

## What This Changes

The PR changes gateway startup maintenance to log rejected startup tasks after `Promise.allSettled` and changes shared runtime plugin registry loading from strict plugin-load failures to graceful error records.

## Best Possible Solution

Keep the gateway startup resilience, but scope non-strict plugin-load handling to gateway bootstrap or an explicit option while preserving strict shared registry behavior, then add focused tests, stack-preserving logs, and a single-line changelog entry.

## Reproduction Assessment

Yes for the review findings. Current source shows `Promise.all(startupTasks)` can reject gateway bootstrap, and existing tests show the strict registry-loader contract the PR changes globally.

## Solution Assessment

No as written. `Promise.allSettled` is a reasonable narrow startup hardening, but flipping the shared registry loader to `throwOnLoadError: false` is broader than the reported gateway startup failure mode.
