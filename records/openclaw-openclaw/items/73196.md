## Summary

Keep open. The availability goal is valid, but this branch is not merge-ready because it broadly changes a shared strict plugin-loader contract, leaves existing tests asserting the old behavior, and lacks the required changelog entry for a user-facing gateway fix.

## What This Changes

The PR changes gateway startup maintenance to await startup tasks with `Promise.allSettled` and changes runtime plugin registry ensures to continue after plugin load errors.

## Best Possible Solution

Keep the gateway resilience behavior only where it is intended, add focused tests for startup-task rejection and plugin-load failure behavior, preserve strict shared loader errors unless a narrower graceful option is introduced, and add the required changelog entry.

## Reproduction Assessment

Yes for the review findings. The current tests explicitly assert `throwOnLoadError: true`, and the PR diff changes that value without touching those tests; applying the patch should be reproducible with the runtime registry and CLI plugin-registry test files.

## Solution Assessment

No as written. `Promise.allSettled` can be a reasonable narrow startup hardening, but changing the shared runtime registry ensure path to non-strict is broader than the stated gateway fix and conflicts with the existing strict-loader contract.
