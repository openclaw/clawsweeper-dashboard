## Summary

Keep open: the gateway startup isolation is plausible, but the PR still changes a shared runtime registry contract that current tests and non-gateway callers expect to stay strict, and it lacks after-fix real behavior proof.

## What This Changes

The PR replaces gateway startup `Promise.all` with `Promise.allSettled` logging and changes `ensurePluginRegistryLoaded` to pass `throwOnLoadError: false`.

## Best Possible Solution

Keep the startup failure isolation, but scope non-strict plugin-load handling to gateway bootstrap or an explicit internal option while preserving strict shared registry behavior, stack-preserving diagnostics, and focused regression coverage.

## Reproduction Assessment

Yes, by source inspection for the review findings: current main still uses `Promise.all(startupTasks)`, and tests plus source assert strict shared runtime loading. I did not run tests because this was a read-only review.

## Solution Assessment

No as written. `Promise.allSettled` is a reasonable narrow startup hardening, but flipping the shared runtime registry loader to `throwOnLoadError: false` is broader than the reported gateway startup failure mode.
