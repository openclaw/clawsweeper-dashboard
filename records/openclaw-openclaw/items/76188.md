## Summary

Keep open. Current main contains several same-day mitigations for plugin-tool and session-list latency, but the low-power CPU/event-loop saturation report still spans unresolved plugin activation, runtime dependency staging, session-store discovery, Discord startup behavior, and fixed concurrency defaults.

## What This Changes

The issue requests low-power-host performance fixes for default plugin loading, runtime dependency staging, workspace/session scans, startup network stalls, and concurrency defaults.

## Best Possible Solution

Keep this as an umbrella until maintainers split or land bounded fixes for credential-aware/lazy tool loading, idempotent runtime-deps staging, bounded session-store discovery, startup network skip paths, and host-aware low-power concurrency.

## Reproduction Assessment

Unclear. The full saturation chain needs the reporter’s low-power host, slow storage, network timeouts, plugin set, and large local state, but the static ingredients are source-backed on current main.

## Solution Assessment

Unclear as one fix. The maintainable path is to split the work by subsystem and preserve existing compatibility, especially disk-only session visibility and third-party plugin contracts.
