## Summary

Keep open. Current main still contains the generic registry/config paths that can emit the missing `channelConfigs` warning for disabled installed external channel plugins and replay it for same-path duplicates, and open PR #76110 already targets this issue with closing syntax.

## What This Changes

Suppress duplicate missing-`channelConfigs` startup warnings for disabled external channel plugins such as `openclaw-weixin`.

## Best Possible Solution

Review and land or close PR #76110; the durable end state is a generic registry diagnostic fix that suppresses inactive external-plugin warning noise and dedupes same-path replay while preserving the `channelConfigs` contract.

## Reproduction Assessment

Yes. The source-level reproduction path is high-confidence: disabled installed plugins can be included in cold metadata, the missing-`channelConfigs` diagnostic has no enabled-state guard, and validation surfaces registry diagnostics as startup warnings.

## Solution Assessment

Yes. A generic core fix like PR #76110 is the narrow maintainable direction; a Weixin special case or removing the metadata warning would weaken the manifest contract for third-party channel plugins.
