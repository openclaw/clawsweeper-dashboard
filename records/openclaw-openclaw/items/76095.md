## Summary

Keep open. Current main still has the generic diagnostic paths that can warn for a disabled external channel plugin and replay the same warning, and open PR #76110 explicitly targets this issue so it should stay open until that PR lands or is closed.

## What This Changes

Suppress duplicate and disabled-plugin `channelConfigs` startup warnings for the external `openclaw-weixin` channel plugin while preserving useful active-plugin metadata diagnostics.

## Best Possible Solution

Review and land or close PR #76110; the durable end state is a generic core diagnostic fix that suppresses inactive or replayed warning noise while keeping the active external-plugin metadata contract.

## Reproduction Assessment

Yes. The source-level path is high-confidence: disabled installed plugins are included in cold metadata, missing `channelConfigs` diagnostics are emitted without enabled-state filtering, the same-plugin path can replay them, and config validation surfaces them as startup warnings.

## Solution Assessment

Yes. A generic manifest-registry/installed-index diagnostic fix like PR #76110 is the narrow maintainable solution; a WeChat-specific exception would leave the same core warning behavior for other external channel plugins.
