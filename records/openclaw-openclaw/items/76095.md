## Summary

Keep open. Current main still has the generic core path that emits the missing `channelConfigs` warning for inactive external channel plugins and can replay the same diagnostic for same-physical-plugin replacement; the linked fix PR is closed unmerged.

## What This Changes

Request to suppress or dedupe startup warnings for the external `openclaw-weixin` channel plugin when it is disabled or excluded by plugin allow rules.

## Best Possible Solution

Add a generic plugin-registry diagnostic fix that skips missing `channelConfigs` warnings for plugins disabled by effective plugin policy, dedupes same-physical-plugin diagnostic replay, preserves active external-plugin metadata warnings, and covers both cases with regression tests.

## Reproduction Assessment

Yes, by source inspection. Disabled installed plugins are included in cold metadata, the missing `channelConfigs` diagnostic does not check effective plugin enablement, same-path replacement can replay the diagnostic, and config validation surfaces it as a startup warning.

## Solution Assessment

Yes, the narrow maintainable fix belongs in the generic manifest registry/config diagnostic path. A WeChat-specific exception or only updating the external plugin metadata would not address disabled-plugin warning noise or same-path duplicate replay for other channel plugins.
