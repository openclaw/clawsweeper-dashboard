## Summary

Keep open. Current main still emits the missing `channelConfigs` diagnostic for non-bundled channel manifests without checking explicit plugin disablement, so a disabled installed `openclaw-weixin` plugin without manifest metadata can still surface startup config noise; the related fix PR is closed unmerged.

## What This Changes

Suppress missing `channelConfigs` startup warnings for disabled `openclaw-weixin` and avoid duplicate copies of the same warning.

## Best Possible Solution

Teach manifest-registry compatibility diagnostics to skip explicitly disabled plugin records while preserving active-plugin metadata warnings and existing diagnostic dedupe.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: disabled installed plugins are included in cold metadata, and the missing-`channelConfigs` diagnostic ignores explicit disablement before config validation surfaces it as a warning.

## Solution Assessment

Yes. A generic fix in `src/plugins/manifest-registry.ts` is the narrow maintainable solution; a Weixin-specific exception would leave the same warning behavior for other disabled external channel plugins.
