## Summary

Keep open. Current main still contains the generic warning path that reports missing `channelConfigs` for non-bundled channel manifests before plugin enablement is considered, and this issue is already paired with open PR #76110 using closing syntax, so it should remain open until that PR lands or is replaced.

## What This Changes

Issue reports duplicate startup/config warnings for an external Weixin channel plugin that lacks `channelConfigs` metadata even when the plugin is disabled or removed from the allow list.

## Best Possible Solution

Review and land or supersede PR #76110 with a generic core fix that suppresses this diagnostic for effectively disabled external plugins and dedupes replayed/recomputed diagnostics while preserving the active-plugin contract warning.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: an installed external channel plugin without `channelConfigs` can enter cold metadata even when disabled, and registry diagnostics are surfaced as config warnings; the reporter also provided exact duplicate startup output.

## Solution Assessment

Yes. A generic plugin registry/config validation fix is the narrowest maintainable solution; it should not special-case `openclaw-weixin` or remove the documented warning for active non-bundled channel plugins.
