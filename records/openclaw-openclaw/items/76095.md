## Summary

Keep open. Current `main` still has the generic manifest/config path that can surface the missing `channelConfigs` warning for inactive external channel plugins, and PR #76110 is an open unmerged closing PR for the same bug.

## What This Changes

Suppress duplicate and inactive-plugin `channelConfigs` startup warnings for the external `openclaw-weixin` channel plugin.

## Best Possible Solution

Keep the active external-channel metadata warning, suppress it for effectively disabled plugins, dedupe replayed diagnostics, and let PR #76110 close this issue only after it lands.

## Reproduction Assessment

Yes. The source-level path is high-confidence: disabled installed plugins are included in cold metadata, the missing-`channelConfigs` diagnostic is generated without checking activation state, and config validation prints registry warnings.

## Solution Assessment

Yes. A generic core fix in manifest registry and installed-index diagnostic handling is the narrowest maintainable direction; this should not special-case `openclaw-weixin` or remove the useful active-plugin warning.
