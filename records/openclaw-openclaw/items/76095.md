## Summary

Keep open. Current main still has a core startup/config warning path that can surface the external channel `channelConfigs` diagnostic even when the plugin is inactive, and the installed-index rebuild path can plausibly duplicate that same diagnostic.

## What This Changes

The issue requests suppressing or deduplicating the `openclaw-weixin` missing `channelConfigs` startup warning when the external plugin is disabled or excluded from `plugins.allow`.

## Best Possible Solution

Keep the active-plugin `channelConfigs` contract warning, but make startup/config validation suppress diagnostics for effectively inactive external plugins and dedupe diagnostics replayed from the installed index.

## Reproduction Assessment

Yes. The source-level path is high-confidence: an installed external channel plugin with `channels` but no `channelConfigs` enters cold metadata even when disabled, its registry diagnostic is surfaced as a config warning, and prior index diagnostics can be replayed alongside recomputed diagnostics.

## Solution Assessment

Yes. A generic core fix is the narrowest maintainable path: do not special-case `openclaw-weixin`, do not remove the active-plugin warning, and keep the external plugin metadata fix as separate ClawHub/package work.
