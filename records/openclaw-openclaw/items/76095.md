Keep open. Current main still has the generic core paths that can surface the missing `channelConfigs` warning for an inactive external channel plugin, and an open linked PR already targets this issue so it should remain open until that PR lands or is closed.

## What This Changes

The issue asks OpenClaw to suppress duplicate startup `channelConfigs` metadata warnings for disabled or allowlist-removed `openclaw-weixin` channel plugins.

## Best Possible Solution

Review and land or close PR #76110; the durable end state is a generic core diagnostic fix that suppresses inactive external-plugin warnings and dedupes replayed diagnostics while preserving the active-plugin metadata contract.

## Reproduction Assessment

Yes. The source-level reproduction path is high-confidence: installed disabled plugins enter cold metadata, missing `channelConfigs` diagnostics are emitted without consulting enabled state, and config validation surfaces those diagnostics as startup warnings.

## Solution Assessment
