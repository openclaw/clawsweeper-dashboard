Full review comments:

- **[P1] Keep agent listing scoped to the caller state root:** `src/agents/agent-scope-config.ts:77`
  - body: `listAgentIds({})` now falls through to `resolveDefaultAgentId(cfg)`, which reads ambient `process.env.OPENCLAW_DEFAULT_AGENT_ID`. Existing auth/model scans accept a target `stateDir` but call `listAgentIds(config)` and then resolve agent dirs from ambient env, so this can pull credential/model paths from outside the scan root. Thread the caller env/state root through this path or keep `listAgentIds` env-free.
  - confidence: 0.86
- **[P1] Do not build persisted main keys from ambient env:** `src/config/sessions/main-session.ts:34-36`
  - body: `resolveMainSessionKey(cfg)` is called by gateway/heartbeat-style paths without an explicit env. After this change, the same config can resolve to `agent:main:main` or `agent:<env>:main` depending on process environment, which can split session history across gateway, CLI, and worker processes. Persisted session identity should come from a config-resolved agent id or an explicit caller parameter, not ambient env.
  - confidence: 0.82
- **[P1] Avoid adding a second default-agent config source:** `src/config/zod-schema.agents.ts:12`
  - body: The current doctor/docs path already migrates legacy `routing.defaultAgentId` into `agents.list` plus `agents.list[].default`. Accepting `agents.defaultAgentId` creates a competing source that is ignored once `agents.list` is populated, which can create drift and confusing upgrades unless maintainers explicitly sponsor and document the new contract.
  - confidence: 0.84
- **[P1] Preserve default-agent auth stores before switching dirs:** `src/plugins/provider-auth-choice.ts:275-279`
  - body: When the new fallback changes the effective default agent, provider auth setup writes profiles under `resolveDefaultAgentDir(...)`, which can be a new empty directory instead of the existing `agents/main/agent` store. Existing auth profiles and model credentials can appear missing unless the PR adds a doctor migration, copy/repair path, or redacted upgrade proof for existing state.
  - confidence: 0.79
