## Summary

Keep open. Current main still lacks the requested configurable default-agent fallback, and the latest PR head is useful but not merge-ready: it addresses several earlier schema/session/provider-auth gaps, but model-state paths still use the bootstrap-only default agent directory, and current CI fails on a missed test mock for the new helper.

## What This Changes

The PR adds `agents.defaultAgentId` and `OPENCLAW_DEFAULT_AGENT_ID` fallback handling across agent resolution, bootstrap agent-dir resolution, session-key helpers, provider auth setup, config schema/generated baseline, changelog, and focused tests.

## Best Possible Solution

Finish the default-agent contract as one coherent path: keep the normalized bootstrap env fallback, route config-loaded default-agent model/auth state callers through the config-aware default-agent dir helper, update the test mocks and precedence wording, then rerun the focused agent/session/model/auth tests plus the changed gate.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with only `agents.defaultAgentId: "ops"` and no `OPENCLAW_DEFAULT_AGENT_ID`, model config/list/status/probe code still resolves state through `resolveOpenClawAgentDir()`, which points at `agents/main/agent`. CI on the latest PR head also reproduces the missing mock export failure in `src/commands/auth-choice.test.ts`.

## Solution Assessment

No. The PR direction is appropriate, but the current patch is not the narrowest complete solution because it leaves config-aware default routing split from common model-state storage paths and has a focused test failure. The safer fix is to use `resolveDefaultAgentDir(cfg, env)` or `resolveAgentDir(cfg, resolveDefaultAgentId(cfg), env)` anywhere a config-loaded default-agent state path is needed.
