- **[P1] Keep agent listing scoped to the caller state root:** `src/agents/agent-scope-config.ts:77`
  - body: `listAgentIds({})` now falls through to `resolveDefaultAgentId(cfg)`, which reads ambient `process.env.OPENCLAW_DEFAULT_AGENT_ID`. Existing secrets/model scans accept a target `stateDir` but enumerate `listAgentIds(config)` and resolve non-main agent dirs from ambient env, so this can pull credentials/models from a different state root into a scan intended for the supplied root. Thread the caller env/state root through this path or keep `listAgentIds` env-free.
  - confidence: 0.86
- **[P1] Do not derive persisted session keys from ambient env:** `src/config/sessions/session-key.ts:53`
  - body: This makes any caller that omits `agentId` persist the same direct chat under either `agent:main:main` or `agent:<env>:main` depending on the current process environment. Gateway, service-manager, CLI, and worker env drift would split session history, so persisted session keys should come from a config-resolved agent id passed by callers rather than from `process.env`.
  - confidence: 0.84
- **[P1] Avoid adding a second default-agent config source:** `src/config/zod-schema.agents.ts:12`
  - body: Current docs and doctor migration already direct default-agent selection into `agents.list[].default`; accepting `agents.defaultAgentId` creates a competing source that is ignored whenever `agents.list` is populated. Either repair hardcoded-main paths through the existing list/default contract or add an explicit maintainer-sponsored migration/docs story for the new contract.
  - confidence: 0.84
- **[P1] Preserve default-agent auth profiles before switching dirs:** `src/plugins/provider-auth-choice.ts:275-279`
  - body: When the new fallback changes the effective default agent, provider auth setup writes to `resolveDefaultAgentDir(...)`, which can be a new empty directory instead of the existing `agents/main/agent` store. Existing auth profiles and model credentials can appear missing unless the PR adds a doctor migration, copy/repair path, or upgrade proof for existing state.
