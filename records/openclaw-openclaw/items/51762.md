## Summary

Keep open. The PR is useful and current main does not implement configurable fallback agent IDs, but the latest patch still has a blocking state-routing gap: config-loaded default-agent routing can use `agents.defaultAgentId` while model/auth state paths still default through the bootstrap-only agent-dir helper and can remain under `agents/main/agent`.

## What This Changes

The PR adds `agents.defaultAgentId` and `OPENCLAW_DEFAULT_AGENT_ID` fallback support across agent resolution, session-key helpers, provider auth setup, config schema/baseline, changelog, and focused tests.

## Best Possible Solution

Land one coherent default-agent contract: config and env fallback should select the same default routing, session keys, workspace, auth/model state, and generated config documentation, while keeping bootstrap-only behavior available for pre-config startup paths.

## Reproduction Assessment

Yes. On PR head, configure only `agents.defaultAgentId: "ops"`; the new resolver/tests make the default agent `ops`, while `ensureOpenClawModelsJson()` and model probes still default their `agentDir` through the bootstrap helper that targets `main` unless the env var or explicit agent dir is also set.

## Solution Assessment

No. The PR direction is the right narrow solution, but the current patch is incomplete; the safer fix is to use `resolveDefaultAgentDir(cfg, env)` or an explicit resolved agent dir at config-loaded state callers and then align the precedence comment/generated docs.
