## Summary

Keep this PR open: current main still lacks the requested configurable default-agent fallback, but PR head still has source-reproducible env-threading bugs that can route provider setup/workspace discovery through `main` while using the env-selected agent elsewhere.

## What This Changes

Adds `agents.defaultAgentId` and `OPENCLAW_DEFAULT_AGENT_ID` fallback handling across config schema, agent/session resolution, provider auth paths, generated config metadata, tests, and changelog.

## Best Possible Solution

Land one consistent default-agent precedence contract across IDs, dirs, workspaces, provider setup, and session keys, with schema/docs comments matching the implemented order.

## Reproduction Assessment

Yes. Source inspection at PR head gives a high-confidence path: supply only `OPENCLAW_DEFAULT_AGENT_ID` through the env bag, then follow workspace/provider-auth calls that compute the agent with that env but resolve workspace without it.

## Solution Assessment

No. The feature direction is reasonable, but this patch should thread the supplied env consistently through workspace/provider setup resolution and align the precedence comment before merge.
