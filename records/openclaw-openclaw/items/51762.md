## Summary

Keep open. Current main still lacks configurable fallback-agent support, and the latest PR head is useful but incomplete because config-only default agents still leave common model/auth state paths on the bootstrap-only default agent directory.

## What This Changes

The PR adds `agents.defaultAgentId` and `OPENCLAW_DEFAULT_AGENT_ID` support across default-agent resolution, bootstrap agent-dir resolution, session-key helpers, provider auth setup, config schema/generated baseline metadata, changelog, and focused tests.

## Best Possible Solution

Finish the feature as one coherent default-agent contract: keep the env bootstrap fallback, route config-loaded default-agent model/auth state through a config-aware agent-dir resolver, and make comments/generated config docs match the tested precedence.

## Reproduction Assessment

Yes. A high-confidence source reproduction is `agents.defaultAgentId: "ops"` with no `OPENCLAW_DEFAULT_AGENT_ID`: the PR would route `resolveDefaultAgentId(cfg)` to `ops`, while model config/list/status/probe paths still default their state directory through `resolveOpenClawAgentDir()` and land on `agents/main/agent`.

## Solution Assessment

No. The PR direction is appropriate, but the patch is not yet the best solution because it splits default routing from model/auth state storage; the safer fix is to use `resolveDefaultAgentDir(cfg, env)` or `resolveAgentDir(cfg, resolveDefaultAgentId(cfg, env), env)` wherever config-loaded default-agent state is needed.
