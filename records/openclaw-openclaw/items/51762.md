## Summary

Keep open; current main still hardcodes the bootstrap fallback to `main`, and this PR is useful but still incomplete because config-loaded model/auth state paths can keep using the config-blind default agent directory while routing uses the configured default.

## What This Changes

The PR adds `agents.defaultAgentId` and `OPENCLAW_DEFAULT_AGENT_ID` fallback support, schema/generated metadata, session/auth helper updates, tests, and a changelog entry.

## Best Possible Solution

Land one consistent default-agent contract: after config is loaded, routing, session keys, model/auth state, status/list/probe paths, schema docs, and tests should use the same precedence chain while `resolveOpenClawAgentDir()` remains the pre-config bootstrap fallback.

## Reproduction Assessment

Yes. Source inspection of PR head shows that with only `agents.defaultAgentId: "ops"`, default routing resolves to `ops` while model config/status/list/probe paths can still default their agent directory through `resolveOpenClawAgentDir()`, which has no config input.

## Solution Assessment

No. The PR direction is the narrow maintainable fix, but the patch should also route config-loaded model/auth callers through `resolveDefaultAgentDir(cfg, env)` or an explicit resolved `agentDir`, and align the precedence comment with the resolver/tests.
