## Summary

Keep open. Current main still hardcodes the bootstrap default agent as `main`, and the PR is directionally right but still leaves config-loaded model/auth state paths using the bootstrap-only default agent directory instead of the configured default agent.

## What This Changes

The PR adds config and environment fallbacks for the default agent ID across agent scope, session keys, provider auth routing, schema metadata, changelog, and focused tests.

## Best Possible Solution

Use one config-aware default-agent contract after config load: keep `resolveOpenClawAgentDir()` as the pre-config bootstrap fallback, but route models/auth/status/probe paths through `resolveDefaultAgentDir(cfg, env)` or an explicit resolved `agentDir`.

## Reproduction Assessment

Yes. On the PR head, configure only `agents.defaultAgentId: "ops"`; static inspection shows routing/workspace helpers resolve `ops` while models config, status/list, and probe paths still default agent state through `resolveOpenClawAgentDir()`.

## Solution Assessment

No. The PR is the right narrow direction, but it is incomplete until all config-loaded default-agent state paths use the same resolved default agent directory and the precedence comment matches the resolver.
