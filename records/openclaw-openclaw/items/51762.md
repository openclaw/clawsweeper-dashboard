## Summary

Keep open: the PR addresses several earlier default-agent routing blockers, but it still leaves config-loaded model/auth state paths on the config-blind bootstrap agent directory, and one new config comment misstates precedence.

## What This Changes

The PR adds an `agents.defaultAgentId` config key and `OPENCLAW_DEFAULT_AGENT_ID` env fallback for default-agent routing/bootstrap, updates session-key/provider-auth paths, schema metadata, changelog, and focused tests.

## Best Possible Solution

Land one consistent default-agent contract: after config is loaded, routing, sessions, model/auth state, status/list/probe paths, schema docs, and tests should use the same precedence chain, while pre-config bootstrap remains env/main fallback only.

## Reproduction Assessment

Yes. On PR head, configure only `agents.defaultAgentId: "ops"`; static inspection shows default ids/session keys move to `ops`, while models JSON/status/list/probe state still defaults through `resolveOpenClawAgentDir()` and can read/write `agents/main/agent`.

## Solution Assessment

No. The PR direction is narrow and maintainable, but config-loaded model/auth callers should use `resolveDefaultAgentDir(cfg, env)` or receive an explicit resolved `agentDir`, and the precedence comment should match the resolver.
