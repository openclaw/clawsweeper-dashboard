## Summary

Keep open: current main still hardcodes the fallback agent ID, and this PR is useful but not yet merge-ready because model auth/status/list/probe paths can still read or write default-agent state under `main` after `agents.defaultAgentId` changes routing.

## What This Changes

The PR adds `agents.defaultAgentId` and `OPENCLAW_DEFAULT_AGENT_ID` default-agent fallbacks, schema/generated metadata, session/auth helper updates, tests, and a changelog entry.

## Best Possible Solution

Land one consistent default-agent contract so routing, session keys, model/auth state, status/list/probe paths, generated config docs, and tests all use the same precedence after config is loaded, while `resolveOpenClawAgentDir()` remains the pre-config bootstrap fallback.

## Reproduction Assessment

Yes. Source inspection of PR head plus current call sites shows that with only `agents.defaultAgentId: "ops"`, routing can resolve to `ops` while model config/status/list/probe default their `agentDir` through config-blind `resolveOpenClawAgentDir()`.

## Solution Assessment

No. The feature direction is narrow and maintainable, but the patch should also route config-loaded model/auth callers through `resolveDefaultAgentDir(cfg, env)` or an explicit resolved `agentDir`, and align the precedence comment with the resolver/tests.
