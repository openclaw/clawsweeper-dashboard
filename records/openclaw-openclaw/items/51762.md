Adds `agents.defaultAgentId` and `OPENCLAW_DEFAULT_AGENT_ID` handling across agent/session/provider-auth resolution, config metadata, tests, and changelog.

## Best Possible Solution

Land one default-agent precedence contract that carries the same cfg/env through ID, directory, workspace, provider setup, session keys, schema, generated docs metadata, and focused regressions.

Yes. Source inspection of PR head shows a high-confidence path: supply only `OPENCLAW_DEFAULT_AGENT_ID` through an env bag, then follow `resolveAgentWorkspaceDir()` or provider-auth setup paths that compute agent ID/dir with that env but resolve workspace without it.

## Solution Assessment
