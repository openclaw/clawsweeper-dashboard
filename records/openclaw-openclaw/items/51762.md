## Summary

Keep open. Current main does not implement configurable default-agent fallback, and the PR is still useful, but the latest patch remains incomplete: model/auth state paths still default through the bootstrap-only agent-dir helper, so a configured non-main default can route sessions/workspaces to one agent while models/auth status and probes read or write `agents/main/agent`.

## What This Changes

The PR adds config and environment fallbacks for the default agent ID, updates schema/generated metadata, session-key/default-agent helpers, provider auth-choice routing, changelog, and focused tests.

## Best Possible Solution

Land one consistent default-agent contract: after config is loaded, routing, workspaces, session keys, model/auth state, status/list/probe paths, schema docs, and tests should use the same precedence chain, while `resolveOpenClawAgentDir()` remains the pre-config bootstrap fallback.

## Reproduction Assessment

Yes. On PR head, configure only `agents.defaultAgentId: "ops"`; static inspection shows default routing helpers return `ops`, while model config/status/list/probe paths still default agent state through `resolveOpenClawAgentDir()`, which targets `main` unless an env or explicit dir override is set.

## Solution Assessment

No. The PR direction is the right narrow solution, but the patch is incomplete; config-loaded model/auth callers should use `resolveDefaultAgentDir(cfg, env)` or receive an explicit resolved `agentDir`, and the precedence copy should match the resolver.
