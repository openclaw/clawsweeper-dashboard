## Summary

Keep open. The latest head is useful and addresses several earlier blockers, but it still leaves config-loaded model/auth state paths using the bootstrap-only default agent directory, so `agents.defaultAgentId` can route workspaces and sessions to the configured default while `models.json`, auth status, and probes still read or write under `agents/main/agent`.

## What This Changes

The PR adds configurable default agent ID fallback support through `agents.defaultAgentId` and `OPENCLAW_DEFAULT_AGENT_ID`, updates session/default-agent helpers, provider auth choice routing, schema/generated baseline metadata, changelog, and focused tests.

## Best Possible Solution

Land one coherent default-agent contract: after config is loaded, default routing, workspaces, session keys, model/auth state, status/list/probe paths, schema docs, and tests should all use the same precedence chain while keeping `resolveOpenClawAgentDir()` as the pre-config bootstrap fallback.

## Reproduction Assessment

Yes. On PR head, configure only `agents.defaultAgentId: "ops"`; static inspection shows `resolveDefaultAgentId()` returns `ops`, while `ensureOpenClawModelsJson()` and model probes still default their `agentDir` through `resolveOpenClawAgentDir()`, which targets `main` unless an env/dir override is set.

## Solution Assessment

No. The PR direction is the right narrow solution, but the current patch is incomplete; config-loaded model/auth callers should use `resolveDefaultAgentDir(cfg, env)` or an explicitly resolved `agentDir`, and the precedence comment/generated docs should match the resolver.
