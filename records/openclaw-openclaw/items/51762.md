## Summary

Keep open. The PR addresses a real gap in default-agent configurability, but the latest head still splits config-loaded model state from the configured default agent directory, and one new config comment documents a different precedence than the resolver implements. The remaining fixes are narrow and suitable for an automated repair pass.

## What This Changes

The PR adds an `agents.defaultAgentId` config key and `OPENCLAW_DEFAULT_AGENT_ID` fallback across agent resolution, session-key helpers, provider auth setup, config schema/baseline, changelog, and focused tests.

## Best Possible Solution

Finish the feature as one coherent default-agent contract: keep `resolveOpenClawAgentDir()` for pre-config bootstrap/env fallback, but route config-loaded model/auth/probe/default-state callers through `resolveDefaultAgentDir(cfg, env)` or an explicitly resolved agent directory, then align config comments/generated docs with the chosen precedence.

## Reproduction Assessment

Yes. On PR head, configure only `agents.defaultAgentId: "ops"` with no `OPENCLAW_DEFAULT_AGENT_ID` or explicit agent-dir override; `resolveDefaultAgentId(cfg)` returns `ops`, while `ensureOpenClawModelsJson()` and model probes still default `agentDir` through the bootstrap helper that targets `main`.

## Solution Assessment

No. The PR direction is the right narrow fix, but the current patch is incomplete because config-aware routing and bootstrap-state paths still diverge; the safer solution is to use the new config-aware default-agent dir helper at config-loaded state call sites and correct the precedence docs.
