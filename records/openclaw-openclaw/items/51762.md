## Summary

Keep open: current main still lacks the requested configurable default-agent fallback, and the current PR head still has env-threading gaps that can route provider setup/workspaces differently from the selected default agent.

## What This Changes

Adds `agents.defaultAgentId` and `OPENCLAW_DEFAULT_AGENT_ID` fallback handling across config schema, agent/session resolution, provider auth paths, generated config metadata, tests, and changelog.

## Best Possible Solution

Land one consistent default-agent precedence contract that threads the supplied env through IDs, dirs, workspaces, provider setup, and session keys, with docs/types matching `default:true`, first list entry, config, env, then `main`.

## Reproduction Assessment

Yes for the remaining PR defects. Source inspection at PR head shows env-only defaults can select `agentId` from `params.env` while workspace resolution calls omit that same env bag.

## Solution Assessment

No. The feature direction is plausible, but the patch should thread `env` consistently through workspace/provider setup resolution and align the precedence comment before merge.
