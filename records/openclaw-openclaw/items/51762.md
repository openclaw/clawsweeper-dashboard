## Summary

Keep open. Current main still lacks configurable fallback default-agent IDs, and this active PR is the implementation candidate, but the branch is not merge-ready because the new config fallback still leaves several default-agent state paths on `main` and the new env-derived path segment is not normalized before filesystem use.

## What This Changes

The PR adds `agents.defaultAgentId` and `OPENCLAW_DEFAULT_AGENT_ID` fallback handling across agent resolution, session key helpers, bootstrap agent-dir resolution, schema/types, generated config schema, and focused tests.

## Best Possible Solution

Keep the feature in this PR, but make the default-agent contract single and normalized: config-aware callers should derive default-agent state paths from `resolveDefaultAgentId(cfg)` and `resolveAgentDir`, bootstrap should use a validated env-only fallback before config exists, docs/schema metadata should describe the supported precedence, and the focused agent/session tests plus the changed gate should pass before landing.

## Reproduction Assessment

Yes. Current main can be verified by source inspection: with no `agents.list`, `resolveOpenClawAgentDir`, `resolveDefaultAgentId`, `resolveMainSessionKey`, and `resolveSessionKey` all fall back to `main`, and the config schema has no `agents.defaultAgentId` key.

## Solution Assessment

No. The PR is on the right path, but the best maintainable solution needs config-aware default-agent directory resolution and normalized ID handling before filesystem joins; otherwise auth/model/session state can split between `main` and the configured default agent.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Route default-agent dir lookups through config:** `src/agents/agent-paths.ts:11-12`
  - body: With only `agents.defaultAgentId` set, the new resolvers switch runtime/session defaults to the custom agent, but default-agent auth/model callers that use `resolveOpenClawAgentDir()` still land on the env-or-`main` bootstrap path. Config-aware call sites such as provider auth and model status will keep reading and writing default-agent state under `main` unless they derive the directory from `resolveDefaultAgentId(cfg)`/`resolveAgentDir(cfg, id)` or a config-aware helper.
  - confidence: 0.9
- **[P2] Normalize the env agent id before joining paths:** `src/agents/agent-paths.ts:11-12`
  - body: `OPENCLAW_DEFAULT_AGENT_ID` is an agent ID, but this inserts the raw env text into `path.join`. Values that normalize elsewhere, such as `Ops Team`, create a different directory from runtime routing, and path segments like `../ops` escape the `agents` directory; normalize or reject the ID before building the path.
  - confidence: 0.88
