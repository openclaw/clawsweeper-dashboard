## Summary

Keep open: the PR addresses a real current-main default-agent gap, but it is not merge-ready because it adds a compatibility-sensitive config/env contract without enough startup/upgrade proof and still duplicates default-agent precedence outside the central resolver.

## What This Changes

## Best Possible Solution

Centralize default-agent precedence in one resolver, document the config/env contract, remove the release-owned changelog edit, and require redacted startup plus upgrade proof before accepting the new surface.
