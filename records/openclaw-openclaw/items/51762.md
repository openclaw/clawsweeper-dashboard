## Summary

Keep open. Current main does not implement configurable fallback agent IDs, and the latest PR direction is useful but still incomplete because config-aware default routing can continue using `agents/main/agent` for model state and some provider-auth paths have inconsistent env injection.

## What This Changes

The PR adds `agents.defaultAgentId` and `OPENCLAW_DEFAULT_AGENT_ID` handling across default-agent resolution, bootstrap paths, session-key helpers, provider auth setup, generated config schema, changelog, and focused tests.

## Best Possible Solution

Land a complete config-aware default-agent implementation that keeps unconfigured users on `main`, treats early bootstrap env fallback and config-loaded default-agent state as coherent paths, and updates generated/config comments to match the actual precedence contract.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: set `agents.defaultAgentId: "ops"` without `OPENCLAW_DEFAULT_AGENT_ID`, then default model config/list/status/probe paths still resolve their agent state directory through `resolveOpenClawAgentDir()`, which points at `agents/main/agent`.

## Solution Assessment

No. The PR is the right direction, but the current patch is not the best complete solution until config-loaded model/auth state paths use `resolveDefaultAgentDir(cfg, env)` or `resolveAgentDir(cfg, resolveDefaultAgentId(cfg, env), env)` consistently and the precedence wording is aligned.

## Review Findings

With only `agents.defaultAgentId: "ops"`, routing/workspace resolve to `ops` while `models.json`, model list/status, and probes still read or write under `agents/main/agent`; route those config-loaded default-state callers through the new config-aware default-agent dir helper.

When callers supply an injected env containing `OPENCLAW_DEFAULT_AGENT_ID`, auth setup can run with `agentId === "main"` while the agent dir points at the env-selected default; pass the same env through the default-id and workspace resolution calls.
  - confidence: 0.85
