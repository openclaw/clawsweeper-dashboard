## Summary

Keep open: current main still lacks the new config/env fallback, but this PR is not merge-ready because it has a concrete env-injection bug in provider auth, no real behavior proof, and conflicts with current main's agent-dir refactor.

## What This Changes

The PR adds `agents.defaultAgentId` and `OPENCLAW_DEFAULT_AGENT_ID` support across default-agent resolution, session-key helpers, provider auth directory selection, schema/generated config, tests, and changelog.

## Best Possible Solution

Rebase onto current main's agent-dir compatibility refactor and route every default-agent/session/auth path through one env-aware resolver, with docs/schema/tests and real gateway/auth proof.

## Reproduction Assessment

Yes for the PR finding: source inspection shows provider auth compares agent IDs resolved without `params.env` against directories resolved with `params.env`. No runtime repro was run because this was a read-only review.

## Solution Assessment

No. The direction is useful, but the mergeable solution should centralize the priority chain in one env-aware resolver and adapt it to the current-main agent-dir compatibility refactor instead of duplicating partial resolution logic.
