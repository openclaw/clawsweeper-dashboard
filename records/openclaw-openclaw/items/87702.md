## Summary

Keep this PR open: the fix direction is right, but the patch clears `CLAUDE_CODE_SESSION` while Claude Code's current environment-variable contract uses `CLAUDE_CODE_SESSION_ID`, so nested Claude workers can still inherit the real session marker and silently fail. ## What This Changes

This PR adds Claude nesting guard env-var clearing to the Anthropic Claude CLI backend and coding-agent skill launch recipe, with parity tests for the clear-env list.

## Best Possible Solution

Update the production clearEnv list, CLI backend fixture/tests, and both coding-agent recipe lines to strip `CLAUDE_CODE_SESSION_ID` while optionally retaining `CLAUDE_CODE_SESSION`, then refresh the child-env proof against that marker before merge.

## Reproduction Assessment

No live nested Claude Code run was reproduced in this review. Source inspection shows current main does not clear the current documented `CLAUDE_CODE_SESSION_ID` marker, and the linked report describes 6/6 nested worker dispatches producing no output.

## Solution Assessment

No; the ownership boundary and clear-env approach are appropriate, but the implementation must use Claude Code's current `CLAUDE_CODE_SESSION_ID` marker and refresh proof before this is the best fix.
