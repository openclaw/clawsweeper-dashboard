The PR updates the ACP event mapper to forward validated image blocks from tool call result content and adds unit coverage for text, image, mixed, malformed, empty, whitespace-only, and string-input cases.

## Best Possible Solution

Forward image content in the shared ACP tool-call mapper, keep the current validation, and require one redacted real ACP client proof before merge.

## Reproduction Assessment

Yes at source level: current main routes ACP partial and final tool results through `extractToolCallContent()`, and that function only forwards text blocks. A live ACP bridge/client reproduction is still missing.

## Solution Assessment

Yes: the shared event mapper is the right layer because both partial and final ACP tool-call updates use it, and the pinned ACP SDK already allows image content in `ToolCallContent`. The remaining blocker is proof, not a different implementation path.
