## Summary

Keep open: current main still drops ACP tool-result image blocks, and this PR is a narrow protocol-aligned mapper/test change. It is not merge-ready yet because the external contributor has not provided real ACP client proof after the fix.

## What This Changes

This PR extends the ACP event mapper to forward non-empty image blocks from tool result content in tool-call updates and adds unit coverage for text, image, mixed, malformed, and fallback cases.

## Best Possible Solution

Land the narrow mapper/test change after redacted real ACP client proof shows an image-producing tool result appears as image content in the client.

## Reproduction Assessment

Yes from source inspection: current main routes tool update/result payloads through extractToolCallContent(), and that function only emits text content blocks. I did not run a live ACP client in this read-only review.

## Solution Assessment

Yes: the ACP translator already centralizes partial and final tool output conversion in extractToolCallContent(), and the SDK's ToolCallContent accepts standard content including images. The remaining gap is real client proof, not a better code layer.
