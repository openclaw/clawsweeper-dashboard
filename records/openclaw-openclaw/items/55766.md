## Summary

Keep open. The PR targets a real ACP gap on current main, where tool-result image blocks are still dropped from ACP tool call content, but the branch is not merge-ready because whitespace-only image fields still pass validation and the exact head has failing CI.

## What This Changes

The PR adds ACP image block extraction in `extractToolCallContent()` and expands `src/acp/event-mapper.test.ts` coverage for text, image, mixed, malformed, empty, and string-result cases.

## Best Possible Solution

Land this PR or an equivalent narrow patch after trimming and validating both image fields, adding whitespace-only regression coverage, keeping text and fallback behavior unchanged, and documenting/release-noting the user-visible ACP image output support.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: on current main, an object like `{ content: [{ type: "image", data: "iVBORw0KGgo=", mimeType: "image/png" }] }` produces no ACP content, while on the PR head `{ data: "   ", mimeType: "image/png" }` is accepted as image content.

## Solution Assessment

No, not yet. The PR changes the right helper and test file, but the validation should reject trimmed-empty `data` and `mimeType` before emitting image content, and the user-facing change needs the repo-required release note.
