Keep open. Current main and v2026.6.10 still forward only text blocks from ACP tool-call result content, and this focused PR matches the ACP SDK content contract; the remaining blocker is missing real ACP bridge/client behavior proof before merge.

## What This Changes

The PR extends ACP tool-result content mapping to forward image blocks and adds unit coverage for text, image, mixed, malformed, empty, and string fallback cases.

## Best Possible Solution

Land this mapper change after redacted real ACP bridge/client proof or maintainer proof override, keeping broader inline media and Control UI rendering work in their separate items.

## Reproduction Assessment

Yes at source level: current main routes ACP partial and final tool results through extractToolCallContent(), and that function only forwards text blocks. I did not run a live ACP client in this read-only review.

## Solution Assessment

Yes, the shared event mapper is the right layer because it feeds both partial and final ACP tool-call updates and the pinned ACP SDK already supports image content. The implementation still needs real client proof before merge.
