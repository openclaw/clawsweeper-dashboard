## Summary

This PR should stay open because the ACP image-forwarding goal is valid, but the current mapper-only patch still does not carry image payloads through the sanitized production tool-result path, and after-fix real ACP client proof is still missing.

## What This Changes

The PR adds image block extraction to `extractToolCallContent()` in the ACP event mapper and unit tests for text, image, mixed, malformed, empty, whitespace-only, and string fallback tool results.

## Best Possible Solution

Carry an ACP-safe image payload or media reference through the sanitized tool-event pipeline, cover the handler-to-translator path, and verify rendering in a real ACP client.

## Reproduction Assessment

Yes, from source inspection. Image-capable raw tool results can exist and ACP supports image content, but current production handlers strip image `data` before the ACP translator sees the result.

## Solution Assessment

No. Adding raw image extraction to `extractToolCallContent()` is not enough; the maintainable fix needs an ACP-safe sanitized payload or media reference that survives the real tool-event path.
