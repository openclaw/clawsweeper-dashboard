The PR updates the ACP event mapper to emit non-empty image content blocks from tool partial/result payloads and adds unit coverage for text, image, mixed, empty, whitespace-only, and malformed content cases.

## Best Possible Solution

## Reproduction Assessment

Yes at source level: current main routes ACP partial and final tool outputs through `extractToolCallContent()`, and that function only emits text content blocks from `record.content`. I did not run a live ACP client reproduction in this read-only review.

## Solution Assessment

Yes, the patch is at the best layer: the ACP event mapper is shared by partial and final tool updates, and the pinned ACP SDK already supports image content inside tool-call content. Client-specific handling would duplicate the protocol invariant.
