## Summary

Keep open for changes. The PR targets a real ACP image-forwarding gap, but the latest branch still maps only raw image blocks in the helper while production tool events are sanitized before ACP translation, so tool-result images can still be omitted; after-fix real behavior proof is also missing.

## What This Changes

The PR updates `src/acp/event-mapper.ts` to emit ACP image content from tool result content blocks and adds unit tests for text, image, mixed, malformed, empty, whitespace-only, and string fallback cases.

## Best Possible Solution

Carry ACP-safe image data or media references through the sanitized tool-event pipeline, add handler-to-translator coverage, and require redacted ACP client proof before merge.

## Reproduction Assessment

Yes, from source inspection. Image-capable tool results can exist and ACP supports image content, but production update/end handlers sanitize those results before ACP mapping and strip `data`, so the PR's raw image branch does not exercise the real path.

## Solution Assessment

No. Mapping raw image blocks in `extractToolCallContent()` is not enough; the maintainable fix needs a safe payload or reference that survives the sanitizer boundary without weakening redaction.
