## Summary

Keep open: current main still has a source-reproducible ACP spawn task-history gap, and this PR is the canonical open fix path, but the branch is not mergeable as-is because it lacks real behavior proof and mixes the ACP fix with stale imports, transcript persistence bypasses, and PR-controlled workflow changes.

## What This Changes

The PR seeds ACP `sessions_spawn` transcripts with the initial task, forwards input provenance, dedupes the first replayed prompt, adds ACP regression tests, updates the changelog, and changes several CI/labeler workflows.

## Best Possible Solution

Land a narrowed ACP-only fix on current main that persists the initial task through the current transcript append, locking, parent-link, and redaction contract before accepted child sessions can look empty, with first-replay dedupe and no workflow churn.

## Reproduction Assessment

Yes, at source level. Current main accepts the ACP child run before async dispatch and only persists ACP transcript content after `runTurn` completes, while the linked reports document live empty-history failures.

## Solution Assessment

No, not as-is. Seeding before dispatch is the right direction, but the branch should be rebased onto current imports, use `appendSessionTranscriptMessage` and its redaction/locking semantics, report delivery from actual seed success, remove workflow churn, and include real ACP proof.
