## Summary

This PR is still useful but not mergeable as-is: current main still lacks ACP spawn transcript seeding, while the branch mixes the ACP fix with unrelated workflow/auth changes and reports `taskDelivered: true` even when seed persistence is skipped or fails.

## What This Changes

The PR adds ACP prompt transcript seeding and dedupe coverage for `sessions_spawn(runtime: "acp")`, refactors ACP transcript persistence, updates a changelog entry, and changes several CI/labeler workflows.

## Best Possible Solution

Replace or rebase this as a narrow ACP-only fix that persists the initial spawn task before claiming delivery, preserves seeded-replay dedupe, keeps focused regression coverage and changelog credit, and leaves workflow changes to a separate reviewed PR.

## Reproduction Assessment

Yes. Source inspection on current main shows ACP spawn sends the task to the agent call without pre-seeding transcript history, while history reads transcript-backed messages; the linked production validation also reported frequent empty histories immediately after ACP spawn.

## Solution Assessment

No as-is. The seed-and-dedupe direction is the right fix, but the PR needs corrected delivery reporting and removal or separate review of unrelated workflow/auth changes before it is the best maintainable solution.
