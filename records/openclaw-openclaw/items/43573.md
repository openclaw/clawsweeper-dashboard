## Summary

Keep open: current main still lacks durable spawn-time ACP task seeding, and this PR remains the canonical implementation path for the linked ACP empty-history reports, but the branch is not mergeable as-is because it is stale against current transcript/dependency code, reports task delivery after a failed seed write, lacks real behavior proof, and mixes in workflow/security-sensitive churn.

## What This Changes

The PR seeds ACP session transcripts with the initial `sessions_spawn` task, forwards provenance, dedupes the first ACP replay, adds tests/changelog, and changes several CI/labeler workflows.

## Best Possible Solution

Rebase and rescope this into an ACP-only fix that appends the initial task through the current lock-backed transcript append path before dispatch, dedupes only the first replay, removes workflow churn, and includes real ACP spawn/history proof.

## Reproduction Assessment

Yes, at source level: current main accepts ACP spawn before async child dispatch and only persists ACP transcript content after `runTurn`, so an immediate `sessions_history` read can race ahead of the initial user-message write or remain empty if the turn stalls.

## Solution Assessment

No, not as-is. The seeding direction is right, but the branch should use current transcript helpers, make delivery reporting depend on seed success, remove unrelated workflow changes, and provide real behavior proof before merge.
