## Summary

Keep this PR open as the canonical ACP initial-task transcript fix path, but it is not mergeable as-is: current main still has the accepted-before-durable-history behavior, while this branch is conflicting, lacks required real behavior proof, and has concrete code/security review blockers.

## What This Changes

The PR seeds ACP spawned-session transcripts with the initial task, forwards provenance, dedupes the later replay, adds ACP regression tests, updates the changelog, and changes several CI workflows.

## Best Possible Solution

Land a narrowed ACP-only fix on current main that appends the initial task through the current transcript append, locking, redaction, and provenance contract before an accepted child session can appear empty, with replay dedupe and no workflow churn.

## Reproduction Assessment

Yes, at source level. Current main accepts and dispatches the ACP child run before durable seed history exists, and ACP transcript persistence only happens after `runTurn` completes while linked reports document immediate empty-history failures.

## Solution Assessment

No, not as-is. Seed-before-dispatch is the right direction, but the branch should be rebased to current package imports and transcript helpers, remove workflow churn, report delivery from actual seed success, and include real ACP proof.
