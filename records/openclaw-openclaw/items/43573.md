## Summary

Keep open: the ACP empty-history bug is still real and this PR is the canonical implementation attempt, but the patch is not merge-ready because it lacks real after-fix behavior proof, unconditionally reports delivery success, and bundles security-sensitive PR workflow changes.

## What This Changes

The branch adds ACP transcript seeding and dedupe for initial `sessions_spawn` tasks, passes input provenance through the first ACP dispatch, adjusts prompt-only transcript normalization, adds tests and a changelog entry, and changes several CI/labeler workflows.

## Best Possible Solution

Port the ACP seed-and-dedupe behavior onto current main using the current transcript append/write-lock path, make delivery reporting truthful, add focused regression coverage, and keep workflow changes separate or owner-reviewed.

## Reproduction Assessment

Yes, with source-level confidence: call `sessions_spawn(runtime: "acp", task: ...)`, then inspect `sessions_history` immediately. Current main acknowledges the Gateway agent request before async dispatch and does not pre-seed the transcript, matching the linked production reports; I did not run live ACP validation in this read-only pass.

## Solution Assessment

No, not as currently packaged. The ACP seeding direction is the right fix, but the patch should be rebased/ported to current transcript APIs, make `taskDelivered` reflect actual seed success, and split or redesign the workflow changes.
