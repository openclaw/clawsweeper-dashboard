## Summary

This PR still targets a real ACP `sessions_spawn` history bug, but it should stay open because the branch is not mergeable as-is: current main still lacks spawn-time task seeding, while the PR mixes the ACP fix with unrelated workflow/labeler changes and can falsely report task delivery after seed persistence fails.

## What This Changes

The PR seeds ACP transcripts with the initial `sessions_spawn` task, forwards input provenance into the first ACP turn, deduplicates the replayed prompt, adds tests/changelog, and changes several GitHub workflows/actions.

## Best Possible Solution

Rebase and rescope the ACP fix onto current main, seed the initial task through the current transcript append path before async dispatch, make delivery confirmation depend on seed success, remove unrelated workflow changes, and attach redacted real runtime proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: spawn an ACP session with a task, then read `sessions_history` immediately after `sessions_spawn` returns; current main accepts before async dispatch and persists ACP transcript content only after `runTurn`.

## Solution Assessment

No, not as-is. The ACP direction is right, but the branch should use current main's transcript append path, make `taskDelivered` reflect actual seed persistence, and drop unrelated workflow/labeler changes.
