## Summary

Keep open: current main still has the ACP spawn/history gap, and the PR's ACP seed-and-dedupe direction is useful, but it should not merge as-is because delivery can be reported as successful after seed persistence failed and unrelated workflow auth/checkout changes are mixed into the bug fix.

## What This Changes

The PR seeds ACP `sessions_spawn` tasks into transcripts, forwards provenance into the first ACP dispatch, adds seed/replay dedupe coverage and a changelog entry, and changes several GitHub workflow checkout/auth paths.

## Best Possible Solution

Rebase or replace this with a narrow ACP-only fix that guarantees or accurately reports initial task transcript seeding, preserves one-shot replay dedupe, updates focused regression tests and changelog credit, and leaves workflow changes to a separate owner-reviewed PR.

Yes. Source inspection on current main shows `sessions_spawn(runtime="acp")` dispatches the task without pre-seeding transcript history, while `sessions_history` is transcript-backed; the PR discussion also includes empirical empty-history validation.

No as-is. The seed-and-dedupe approach is the right direction, but the patch should make delivery confirmation reflect seed persistence, add required contributor credit, and split unrelated workflow/auth changes.
