## Summary

Keep open: current main still lacks immediate ACP spawn task transcript seeding, and this PR targets a real bug, but the branch is dirty and the patch still mixes the ACP fix with correctness and workflow-auth concerns.

## What This Changes

The PR seeds ACP `sessions_spawn` transcripts with the initial task, forwards provenance into the first dispatch, dedupes the later replay, adjusts prompt-only SessionManager handling, adds tests/changelog, and changes several CI/labeler workflows.

## Best Possible Solution

Land a narrow ACP-only fix on current main that seeds the initial task before reporting delivery, dedupes only the seeded replay, preserves prompt-only transcript normalization, adds focused regression tests, and leaves workflow/auth changes to a separate reviewed PR.

## Reproduction Assessment

Yes. Spawn an ACP session with `sessions_spawn(runtime: "acp", task: ...)`, inspect `sessions_history` or gateway `chat.history` immediately, and current main shows the task is dispatched while history remains transcript-file-backed with no pre-dispatch seed.

## Solution Assessment

No as-is. The seed-and-dedupe direction is appropriate, but the PR should only claim delivery when the seed actually persisted, restore required changelog attribution, rebase on current main, and split or remove unrelated CI/token changes.
