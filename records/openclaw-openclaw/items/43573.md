## Summary

Keep open: current main still lacks spawn-time ACP task seeding and the linked bug was closed in favor of this PR, but the branch is not mergeable as-is because real behavior proof is missing, `taskDelivered` can be false-positive after seed persistence failure, and unrelated workflow/security changes are bundled.

## What This Changes

The PR seeds ACP `sessions_spawn` task text into ACP transcripts before dispatch, carries input provenance into the first ACP turn, dedupes the seeded replay, adds tests/changelog, and changes CI/labeler workflows.

## Best Possible Solution

Rebase and rescope the ACP fix onto current main, seed the initial task through the current locked transcript append path before asynchronous dispatch, make delivery confirmation depend on seed success, drop unrelated workflow changes, and add focused tests plus redacted real runtime proof.

## Reproduction Assessment

Yes, with source-backed confidence rather than a live run: spawn an ACP session with a task and read `sessions_history` immediately after `sessions_spawn` returns. Current main accepts the Gateway request before asynchronous dispatch and only mirrors ACP transcript content after the turn completes.

## Solution Assessment

No, not as-is. The ACP direction is right, but the patch should use current main's locked transcript append path, make `taskDelivered` reflect actual seed persistence, and remove unrelated CI/labeler workflow changes.
