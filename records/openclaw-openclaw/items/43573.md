## Summary

Keep open: current main still has the accepted-before-durable-history ACP spawn path, and this PR is the canonical tracker for closed duplicate reports, but the branch is not mergeable as-is because it needs real ACP proof, a rebase/rescope, and fixes for workflow, dependency-import, and delivery-confirmation problems.

## What This Changes

The PR seeds ACP `sessions_spawn` prompts into transcripts, dedupes the replayed first turn, threads input provenance through dispatch, adjusts prompt-only session-manager initialization, adds tests, and changes CI/labeler workflows.

## Best Possible Solution

Rebase and rescope this into an ACP-only fix that writes the initial task through the current transcript append path before dispatch, returns delivery status only after persistence succeeds, drops workflow churn, and includes real ACP spawn/history proof.

## Reproduction Assessment

Yes, at source level. Current main accepts the ACP child dispatch before async runner execution and only persists ACP transcript content after `runTurn`, while the linked reports and PR comments document live empty-history failures.

## Solution Assessment

No, not as-is. The seeding direction is appropriate, but the branch should use current Pi imports and transcript append helpers, avoid false `taskDelivered` acknowledgements, remove unrelated workflow changes, and provide real ACP proof.
