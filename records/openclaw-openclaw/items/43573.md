## Summary

Keep open. The ACP `sessions_spawn` bug is still source-reproducible on current main, but this branch should not merge as-is because it reports delivery even when seed persistence fails, mixes unrelated workflow/auth changes into the runtime fix, and lacks after-fix real behavior proof.

## What This Changes

The branch seeds ACP `sessions_spawn` tasks into transcript history, forwards input provenance, adds seed/dedupe tests and a changelog entry, and changes several CI/labeler workflows.

## Best Possible Solution

Rebase or replace this with a narrow current-main ACP-only fix that seeds the initial task before delivery confirmation, dedupes only the seeded replay, preserves provenance, and leaves workflow changes to a separate owner-reviewed PR.

## Reproduction Assessment

Yes. Source inspection shows current main accepts the ACP spawn before post-turn transcript persistence, and the linked issue/PR discussion reports frequent empty immediate histories after ACP spawns.

## Solution Assessment

No as-is. The seed-and-dedupe direction is the right repair, but delivery confirmation must reflect seed success and the unrelated workflow/auth changes should be split out.
