## Summary

Keep open: the underlying ACP `sessions_spawn` history bug is still source-reproducible on current main, and this PR is the canonical fix path, but the branch is currently dirty and has blocking correctness/security issues plus missing real ACP runtime proof.

## What This Changes

The PR seeds ACP transcripts with the initial spawn task, forwards provenance, dedupes the later replay, adds transcript tests, changes session-manager init, and edits CI/labeler workflows.

## Best Possible Solution

Rebase and rescope this into an ACP-only fix that writes the initial task through the current lock-backed transcript append path before dispatch, reports delivery accurately, removes workflow churn, and includes real ACP spawn/history proof.

## Reproduction Assessment

Yes, at source level. Current main accepts the ACP child dispatch before async runner execution and only persists ACP transcript content after `runTurn` completes; related issue comments also report live empty-history failures.

## Solution Assessment

No, not as-is. The seeding direction is right, but this branch should use current transcript helpers/imports, report `taskDelivered` only when the seed write succeeds, remove unrelated workflow changes, and provide real ACP proof before merge.
