## Summary

Keep this PR open because it targets a real ACP sessions_spawn task-history bug, but the current branch is not mergeable as-is: it is outdated against current main, lacks real behavior proof, mixes in workflow/security-sensitive churn, and bypasses the current transcript append contract.

## What This Changes

The PR seeds ACP spawned-session transcripts with the initial task, dedupes the later replayed prompt, forwards input provenance, adds ACP regression tests, updates a changelog entry, and changes several CI/labeler workflows.

## Best Possible Solution

Land a narrowed ACP-only fix on current main that appends the initial task through the current transcript append, locking, and redaction contract before accepted child sessions can appear empty, with first-replay dedupe and no workflow churn.

## Reproduction Assessment

Yes, at source level. Current main accepts the ACP child run before async dispatch and only persists ACP transcript content after runTurn completes, while linked reports document live empty-history failures.

## Solution Assessment

No, not as-is. Seed-before-dispatch is the right direction, but the branch should be rebased onto current imports, use appendSessionTranscriptMessage and its redaction/locking semantics, report delivery from actual seed success, remove workflow churn, and include real ACP proof.
