## Summary

Keep open: the ACP `sessions_spawn` transcript bug still appears valid on current main, but this PR should not merge as-is because it reports delivery after best-effort transcript seeding and mixes unrelated workflow/auth changes into a runtime fix.

## What This Changes

The PR seeds ACP `sessions_spawn` tasks into transcript/history, adds provenance/dedupe tests and a changelog entry, and also changes CI/labeler checkout and token handling workflows.

## Best Possible Solution

Replace or rebase this as a narrow ACP-only fix on current main that seeds the initial task before reporting delivery, dedupes the replayed first turn, preserves provenance, and leaves workflow changes out.

## Reproduction Assessment

Yes. Source inspection shows current main sends the task to the ACP turn but only writes transcript history after the turn completes, and the linked validation comment reports frequent empty immediate histories after ACP spawn.

## Solution Assessment

No as-is. The seed-and-dedupe direction is the right repair, but delivery reporting must reflect seed success and the unrelated workflow/auth changes should be split from the ACP runtime fix.
