## Summary

Keep open: the underlying ACP initial-task history bug is still source-reproducible on current main and related issues were consolidated here, but this branch is not mergeable as-is because review found correctness, proof, and workflow-security blockers.

## What This Changes

The PR seeds ACP session transcripts with the initial `sessions_spawn` task, forwards provenance, deduplicates the first replay, adds tests/changelog, and changes PR/labeler workflows.

## Best Possible Solution

Rebase and rescope to an ACP-only fix that appends the initial task through the current lock-backed transcript append path before dispatch, dedupes only the first replay, removes workflow churn, and includes real ACP spawn/history proof.

## Reproduction Assessment

Yes. Source inspection shows ACP spawn accepts before async child dispatch and current ACP transcript persistence runs only after `runTurn`, so an immediate `sessions_history` read can see empty history or remain empty if the turn stalls.

## Solution Assessment

No, not as-is. The ACP seeding direction is right, but the branch should use current transcript helpers, make `taskDelivered` depend on seed success, drop unrelated workflow changes, and provide real behavior proof.
