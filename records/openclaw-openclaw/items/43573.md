## Summary

Keep open: current main still defers ACP task transcript persistence until after the ACP turn completes, so the reported bug remains valid, but this PR is not mergeable as-is because it lacks real runtime proof and includes stale imports plus unrelated workflow/security churn.

## What This Changes

The branch seeds ACP transcripts with the initial `sessions_spawn` task, forwards input provenance, dedupes the later replay, adjusts prompt-only session-manager initialization, adds tests, and changes CI/labeler workflows.

## Best Possible Solution

Rebase and rescope this into an ACP-only fix that writes the initial task through the current transcript append path before dispatch, reports delivery accurately, drops workflow churn, and includes real ACP spawn/history proof.

## Reproduction Assessment

Yes, at source level. Current main accepts the ACP child dispatch before async runner execution and persists ACP transcript content only after `runTurn`, while related comments report live empty-history failures.

## Solution Assessment

No, not as-is. The seeding direction is right, but the branch should use current Pi imports/transcript helpers, return `taskDelivered` only on successful seeding, remove unrelated workflow changes, and provide real ACP proof.
