## Summary

Keep this PR open because it is still the canonical implementation path for a real ACP sessions_spawn history bug, but it should not merge as-is: current main still lacks spawn-time task seeding, the PR reports `taskDelivered: true` even after seed persistence fails, and the external PR still needs real behavior proof.

## What This Changes

The PR adds ACP transcript seeding/deduplication for initial `sessions_spawn` tasks, extracts ACP transcript persistence helpers and tests, updates prompt-only session-manager handling, adds a changelog entry, and changes several CI/labeler workflows.

## Best Possible Solution

Land a narrowed current-main ACP fix that seeds the initial task before dispatch, dedupes only the first replay, reports delivery status truthfully, and includes regression plus real ACP spawn/history proof.

## Reproduction Assessment

Yes, at source level: current main creates and dispatches the ACP session task but does not pre-seed transcript history, while the gateway accepted ack can return before the async runner writes history. I did not run a live ACP repro in this read-only review.

## Solution Assessment

No, not as submitted. Seeding before dispatch is the right direction, but `taskDelivered` must reflect whether the seed actually persisted, and the PR should be rebased or narrowed against current main before merge.
