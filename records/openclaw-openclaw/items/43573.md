## Summary

Keep open: current main still has a source-reproducible ACP spawned-session history bug, and this PR is the canonical implementation path, but it is not merge-ready because it lacks real behavior proof, reports task delivery even when seed persistence fails, and mixes unrelated workflow/security-sensitive changes into the ACP fix.

## What This Changes

The PR seeds ACP `sessions_spawn` task text into ACP transcripts before dispatch, carries input provenance into the first ACP turn, dedupes the seeded replay, adds transcript tests/changelog, and modifies CI/labeler workflows.

## Best Possible Solution

Rebase and rescope the ACP fix onto current main, seed the initial task through the current locked transcript append path before dispatch, report delivery only after seed persistence succeeds, drop unrelated workflow changes, and add focused tests plus redacted real runtime proof.

## Reproduction Assessment

Yes, with source-backed confidence rather than a live run: spawn an ACP session with a task and read `sessions_history` immediately after `sessions_spawn` returns. Current main accepts the Gateway agent request before asynchronous dispatch and only persists ACP transcript content after the turn completes.

## Solution Assessment

No, not as-is. The ACP direction is right, but the patch should use current main's transcript append/locking path, make `taskDelivered` reflect actual seed success, and remove unrelated CI/labeler workflow changes.
