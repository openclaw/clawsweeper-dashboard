## Summary

Keep open. The bug remains valid on current main and the branch aims at the right ACP boundary, but it is not mergeable as-is: it is stale against renamed Pi packages and current transcript append helpers, can report `taskDelivered: true` after seed persistence failure, mixes in security-sensitive workflow changes, and lacks after-fix real behavior proof.

## What This Changes

The PR adds ACP transcript pre-seeding and dedupe for `sessions_spawn` initial tasks, forwards provenance into the first dispatch, adjusts prompt-only session initialization, adds regression tests, updates the changelog, and modifies CI/labeler workflows.

## Best Possible Solution

Land a narrowed ACP-only fix that pre-seeds the initial task through the current transcript append and lock path before accepted history reads, reports delivery only when the seed write succeeds, and drops unrelated workflow changes.

## Reproduction Assessment

Yes, at source level. Current main accepts the ACP child run before async dispatch and persists ACP transcript content only after `runTurn`, while linked reports document live empty-history failures.

## Solution Assessment

No, not as-is. The seed-before-dispatch direction is appropriate, but the PR should be rebased onto current imports and transcript helpers, avoid false delivery acknowledgements, remove workflow churn, and include real ACP proof.
