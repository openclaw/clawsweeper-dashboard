## Summary

Keep open: current main still has the ACP accepted-before-durable-history path, and this PR is the canonical tracker for the reported empty-history bug, but the branch is not mergeable as-is because it is stale, conflicting, missing real behavior proof, and mixes ACP transcript work with CI/labeler security-sensitive changes.

## What This Changes

The PR seeds ACP `sessions_spawn` tasks into transcripts before dispatch, forwards provenance into the first ACP turn, dedupes the replayed prompt, adjusts prompt-only session initialization, adds tests, and changes CI/labeler workflows.

## Best Possible Solution

Rebase and rescope this into an ACP-only fix that writes the initial task through the current transcript append path before dispatch, reports delivery only after persistence succeeds, drops workflow churn, and includes real ACP spawn/history proof.

## Reproduction Assessment

Yes, at source level. Current main accepts the ACP child dispatch before async runner execution and persists ACP transcript content only after `runTurn`, while linked reports document live empty-history failures.

## Solution Assessment

No, not as-is. The seeding direction is appropriate, but the branch should be rebased onto current Pi imports and transcript helpers, avoid false `taskDelivered` acknowledgements, remove unrelated workflow changes, and provide real ACP proof.
