## Summary

Keep open: the ACP task-history bug is still present on current main, but this PR should not merge as-is because it can report delivery after a skipped or failed seed write and it mixes unrelated workflow/auth changes into the ACP fix.

## What This Changes

This PR seeds ACP `sessions_spawn` transcripts with the initial task, adds ACP transcript dedupe/provenance handling and tests, updates the changelog, and changes several CI/labeler workflows.

## Best Possible Solution

Land a narrow ACP-only fix on current main that seeds the initial task before returning, reports delivery only when that seed succeeds, dedupes only the seeded replay, credits the contributor, and leaves workflow changes to a separate reviewed PR.

## Reproduction Assessment

Yes. Spawn an ACP session with `sessions_spawn(runtime: "acp", task: ...)` and immediately call `sessions_history`; current main dispatches the task without pre-seeding the transcript, while history is transcript-backed.

## Solution Assessment

No as-is. The seed-and-dedupe direction is appropriate, but the branch needs corrected delivery semantics, contributor changelog credit, a rebase, and removal or separate review of unrelated workflow/auth changes.
