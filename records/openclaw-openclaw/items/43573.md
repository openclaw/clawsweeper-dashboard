## Summary

Keep open: current main still has the ACP empty-history window and this PR is the canonical fix path, but the branch is not mergeable as-is because it lacks after-fix real ACP proof, is stale against current Pi imports, returns a false delivery guarantee on seed failure, and mixes unrelated workflow/security-sensitive changes into the ACP fix.

## What This Changes

The PR seeds ACP transcripts with the initial `sessions_spawn` task, forwards input provenance, deduplicates the first replayed prompt, adds ACP regression tests and a changelog entry, and changes several CI/labeler workflows.

## Best Possible Solution

Land a narrowed ACP-only fix on current main that writes the initial task through the current transcript append, locking, and redaction contract before reporting delivery, with no workflow churn and with real ACP proof.

## Reproduction Assessment

Yes, at source level. Current main accepts the ACP child run before async dispatch and persists ACP transcript content only after `runTurn`, while linked reports document live empty-history failures.

## Solution Assessment

No, not as-is. The seed-before-dispatch direction is right, but the PR should be rebased onto current imports and transcript append helpers, report delivery only after a successful seed write, remove workflow churn, and include real ACP proof.
