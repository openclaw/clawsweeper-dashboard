## Summary

Keep open: current main still lacks spawn-time ACP transcript seeding, and the related discussion supports a real ACP `sessions_spawn` history bug. The PR is useful but not mergeable as-is because it is outdated/conflicting, lacks after-fix real behavior proof, and mixes security-sensitive workflow changes into the ACP fix.

## What This Changes

The PR seeds ACP session transcripts with the initial `sessions_spawn` task, forwards input provenance into the first ACP dispatch, dedupes the seeded replay, adds ACP transcript tests, and changes CI/labeler workflows.

## Best Possible Solution

Rebase and rescope this into an ACP-only fix that writes the initial task through the current lock-backed transcript append path before dispatch, dedupes only the first replay, removes workflow churn, and includes real ACP spawn/history proof.

## Reproduction Assessment

Yes, at source level: current main accepts ACP spawns before async child dispatch and persists ACP transcript content only after `runTurn` completes. I did not run a live ACP runtime, so this is source-reproducible rather than live-reproduced.

## Solution Assessment

No, not as-is. The seeding direction is right, but the branch should use current transcript helpers, report delivery only after seed persistence succeeds, remove unrelated workflow changes, and add real behavior proof before merge.
