## What This Changes

The PR changes `truncateLine` in `src/shared/subagents-format.ts` to call `truncateUtf16Safe` and adds surrogate-pair tests for emoji and CJK Extension B truncation.

## Best Possible Solution

Land this focused helper swap with its regression test after ordinary maintainer review and required checks finish; keep unrelated truncation surfaces in separate targeted fixes.

## Reproduction Assessment

Yes. Current main's `truncateLine` uses raw `slice(0, maxLength)`, and a focused Node check of that expression produces an unpaired surrogate for a boundary emoji input.

## Solution Assessment

Yes. Reusing the existing dependency-free `truncateUtf16Safe` helper at the shared `truncateLine` boundary is the narrow maintainable fix; caller-level or bespoke string logic would duplicate the existing helper and miss one of the label/task call sites.
