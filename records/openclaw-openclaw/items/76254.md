## Summary

Keep open. Current main still truncates stale-gateway ancestor exclusion to `process.ppid` on macOS, so a nested caller can still leave a gateway grandparent in the stale PID list; the reporter's follow-up makes this a latent correctness bug rather than the confirmed source of the live SIGTERM loop now tracked by #76259.

## What This Changes

Extend stale-gateway ancestor exclusion so macOS callers nested more than one level under the Gateway do not classify a gateway grandparent as stale.

## Best Possible Solution

Add a bounded Darwin/BSD parent-PID walk inside `getSelfAndAncestorPidsSync`, keep the existing Linux `/proc` path, and cover the macOS grandparent case with targeted regression tests plus a changelog entry.

## Reproduction Assessment

Yes, source-level. Stubbing `process.platform` to `darwin`, `process.ppid` to a plugin host, and lsof output containing a gateway grandparent would leave that grandparent in the returned stale PID list because the helper returns before any transitive walk.

## Solution Assessment

Yes. Extending the existing ancestor-exclusion helper with a bounded macOS/BSD parent lookup is the narrowest maintainable fix and avoids changing cleanup call-site policy.

## Repair Work Prompt

Fix openclaw/openclaw#76254. Current `getSelfAndAncestorPidsSync` in `src/infra/restart-stale-pids.ts` returns self plus direct parent on macOS because only Linux walks transitive parents through `/proc`; a nested macOS caller can therefore leave a gateway grandparent in the stale PID list used by `findGatewayPidsOnPortSync` and `cleanStaleGatewayProcessesSync`. Add a bounded Darwin/BSD parent-PID walk, likely using the existing `ps -p <pid> -o ppid=` pattern from `src/infra/ports-inspect.ts`, while preserving the Linux `/proc` path and existing Windows/direct-parent behavior unless the tests justify a small shared helper. Add targeted regression coverage in `src/infra/restart-stale-pids.test.ts` for a darwin `gateway -> plugin-host -> sidecar` chain where the gateway grandparent is excluded and an unrelated stale gateway PID is still returned. Keep the lookup timeout bounded, avoid new dependencies and runtime override surfaces unless necessary, do not change #70664 lsof command verification or #76259 kill-tree behavior in this PR, and add a user-facing `CHANGELOG.md` fix entry.
