## Summary

## What This Changes

## Best Possible Solution

## Reproduction Assessment

Yes. Source inspection on current main shows cron payload `toolsAllow` reaches prompt/delivery logic and the embedded runner but is omitted from the CLI runner call, while the CLI preparation layer fails closed when it receives `toolsAllow`.

## Solution Assessment

No. Forwarding the existing `toolsAllow` field into the CLI path is the narrow fix, but unconditional `senderIsOwner: true` is a separate trust-model change that needs removal or explicit maintainer approval.
