## Summary

Keep open: current main and v2026.6.8 still implement selected Telegram quote replies as a documented trade-off that disables the answer draft stream, so temporary progress plus final native quote delivery remains an unresolved Telegram product change.

## What This Changes

The issue requests temporary Telegram tool-progress previews for selected/native quote-reply turns while preserving final native quote replies.

## Best Possible Solution

Have the Telegram owner define the visible-message contract, then implement a Telegram-owned temporary progress lane with cleanup/error semantics, focused dispatch tests, docs/doctor updates, and live Telegram proof.

## Reproduction Assessment

Yes, for the current limitation by source inspection: selected quote context keeps answerLane.stream absent, so tool-progress preview callbacks cannot render. It is not a strict bug reproduction because current docs and doctor now document this as a trade-off.

## Solution Assessment

Unclear until Telegram product review. A separate temporary progress lane is plausible, but it changes visible message lifecycle semantics that the prior guardrail explicitly left as separate product work.
