## Summary

Keep open: current main and the latest release still implement selected Telegram quote replies as a documented trade-off that skips the answer draft stream, so the requested temporary progress message remains a product change rather than implemented cleanup.

## What This Changes

The issue requests temporary Telegram tool-progress/status previews during selected/native quote-reply turns while keeping the final answer as a native Telegram quote reply.

## Best Possible Solution

Have the Telegram owner decide the visible-message contract, then implement the approved Telegram-owned temporary progress lane with cleanup/error semantics, docs/doctor updates, focused tests, and live Telegram proof.

## Reproduction Assessment

Yes for the current limitation by source inspection: selected quote context keeps the answer draft stream absent, so tool-progress preview lines cannot render. It is not a strict bug reproduction because the issue asks for new behavior beyond current docs.

## Solution Assessment

Unclear until Telegram product review. A temporary second progress message is plausible, but it changes the current Telegram streaming contract that one visible preview message owns the stream unless final edit fails.
