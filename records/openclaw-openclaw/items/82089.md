## Summary

Keep open: current main still implements and documents selected Telegram quote replies as a trade-off that skips answer-preview/tool-progress streaming, while the requested separate temporary progress message is new Telegram runtime behavior that needs owner/product review.

## What This Changes

The issue requests temporary Telegram tool-progress previews for selected native quote-reply turns, followed by a final native quote reply and clean preview removal.

## Best Possible Solution

Have the Telegram owner decide whether selected-quote turns may use a second temporary progress message; if approved, implement it in the Telegram plugin with cleanup/failure semantics, docs/doctor updates, focused tests, and live Telegram proof.

## Reproduction Assessment

Not applicable as a bug reproduction because this requests new Telegram behavior. Source inspection gives high confidence that current main still skips the answer draft stream for selected quote replies, so preview tool-progress cannot render for those turns.

## Solution Assessment

Unclear until product review. The temporary progress lane is plausible, but it intentionally adds a second visible Telegram message around a path whose current contract is one preview message unless final edit fails.
