## Summary

Keep open: current main still treats selected Telegram quote replies as a documented trade-off that skips answer-preview/tool-progress streaming, while the requested temporary progress message is new Telegram runtime behavior that prior maintainer discussion left as a separate product decision.

## What This Changes

Request selected Telegram native-quote turns to show a temporary tool-progress preview during execution, then deliver the final answer as a native quote reply and clean up the temporary message.

## Best Possible Solution

Have the Telegram owner decide whether selected-quote turns may use a separate temporary progress draft; if approved, implement it in the Telegram plugin with cleanup/failure semantics, updated docs/doctor text, focused tests, and live Telegram proof.

## Reproduction Assessment

Not applicable as a bug reproduction because this requests new behavior. Source inspection confirms current main’s selected-quote path skips answer draft streaming when replyToMode is enabled, so the requested progress preview is absent by design today.

## Solution Assessment

Unclear until product review. The temporary progress lane is plausible, but it changes Telegram’s current preview/final delivery contract and needs explicit cleanup, failure, and live-proof semantics.
