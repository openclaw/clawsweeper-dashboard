## Summary

Keep open: current main still documents and implements selected Telegram quote replies as a trade-off that skips answer-preview/tool-progress streaming, while the requested temporary progress message is a new Telegram runtime behavior that prior maintainer discussion explicitly left for a separate product decision.

## What This Changes

Requests a Telegram runtime mode where selected quote turns show a temporary tool-progress message while final answers still use native quote replies.

## Best Possible Solution

Have the Telegram owner decide whether selected-quote turns may use a second temporary progress message; if approved, implement it in the Telegram plugin with cleanup/failure semantics, updated docs/doctor text, focused tests, and live Telegram proof.

## Reproduction Assessment

Not applicable as a bug reproduction because this requests new behavior. Source inspection does confirm current main’s documented limitation: selected quote replies skip answer draft streaming, so preview tool-progress cannot render for those turns.

## Solution Assessment

Unclear until product review. The proposed temporary progress lane is plausible, but it conflicts with current Telegram streaming guidance that one visible preview message owns the stream unless final edit fails.
