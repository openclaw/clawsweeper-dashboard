## Summary

Keep open: current main still treats Telegram selected native quote replies as a documented trade-off that skips answer-preview/tool-progress streaming, and the requested temporary progress message is a new Telegram-visible behavior requiring owner/product review. The adjacent open progress-lane PR does not close or solve selected-quote turns because its new lane is still gated on the answer draft stream that selected quotes disable.

## What This Changes

Request support for visible temporary Telegram tool-progress messages during selected native quote-reply turns, while preserving the final answer as a native quote reply and cleaning up the temporary preview.

## Best Possible Solution

Have Telegram owners decide whether selected-quote turns may create a second temporary progress message; if approved, implement it in the Telegram plugin with cleanup/failure semantics, docs/doctor updates, focused tests, and live Telegram proof.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection on current main confirms the existing limitation: selected quote replies do not create answerLane.stream, so preview tool-progress cannot render.

## Solution Assessment

Unclear until product review. The requested temporary progress lane is plausible, but it conflicts with current Telegram streaming guidance that treats preview streaming as a single visible message unless final edit fails.
