## Summary

Keep open: current main still implements and documents selected Telegram quote replies as a trade-off that skips the answer draft stream, while the requested temporary progress message is a new Telegram runtime behavior that needs owner/product review.

## What This Changes

Request Telegram selected-quote turns to show a temporary tool-progress preview while still sending the final answer as a native quote reply.

## Best Possible Solution

Have the Telegram owner decide whether selected-quote turns may use a second temporary progress message; if approved, implement it in the Telegram plugin with cleanup/failure semantics, docs/doctor updates, focused tests, and live Telegram proof.

## Reproduction Assessment

Not applicable as a bug reproduction because this requests new behavior. Source inspection does confirm current main’s documented limitation: selected quote replies skip the draft stream that powers tool-progress previews.

## Solution Assessment

Unclear until product review. The proposed temporary progress lane is plausible, but it changes Telegram streaming semantics around the current single visible preview-message guidance.
