## Summary

Keep open: current main still implements and documents selected Telegram quote replies as a trade-off that skips answer-preview/tool-progress streaming, and the requested temporary progress message is new visible Telegram runtime behavior that prior review explicitly left for product/owner decision.

## What This Changes

The issue asks for Telegram selected-quote turns to show temporary tool-progress previews while still sending the final answer as a native Telegram quote reply.

## Best Possible Solution

Have the Telegram owner decide whether selected-quote turns may use a second temporary progress message; if approved, implement it in the Telegram plugin with cleanup/error semantics, docs/doctor updates, focused tests, and live Telegram proof.

## Reproduction Assessment

Not applicable as a bug reproduction because this requests new behavior. Source inspection confirms the current-main limitation: selected quote replies skip answer draft streaming, so preview tool-progress cannot render for those turns.

## Solution Assessment

Unclear until product review. The proposed temporary progress lane is plausible, but it conflicts with the current Telegram streaming guidance that one visible preview message owns the stream unless final edit fails.
