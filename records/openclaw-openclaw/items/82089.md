## Summary

Keep open: current `main` and latest release still implement selected Telegram quote replies as a documented trade-off that disables the answer draft stream, so temporary progress plus final native quote delivery remains unresolved. The related merged work documents and warns about the caveat or fixes adjacent quote anchoring, while the open long-DM progress PR explicitly leaves native quote progress to this issue.

## What This Changes

The issue requests Telegram temporary tool-progress/status previews during selected native quote-reply turns, followed by a final native Telegram quote reply and cleanup of the temporary preview.

## Best Possible Solution

Keep this as the canonical Telegram product request until maintainers approve the temporary progress-lane UX, then implement it inside the Telegram plugin with focused cleanup/error tests, docs/doctor updates, and live Telegram proof.

## Reproduction Assessment

Yes. Set `channels.telegram.replyToMode` to a non-off mode and send a Telegram message with selected quote text; current source disables the answer draft stream for that turn, though I did not run live Telegram proof in this read-only pass.

## Solution Assessment

Unclear. A separate temporary progress lane is plausible, but the best fix depends on maintainer decisions about Telegram message lifecycle, cleanup, failures, approvals, and media behavior.
