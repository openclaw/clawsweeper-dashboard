## Summary

Keep open: current main still treats Telegram selected/native quote replies as a documented trade-off that skips the answer-preview stream, while this asks for a new second temporary progress message and cleanup behavior that needs Telegram product/owner review.

## What This Changes

The issue requests temporary Telegram tool-progress previews during selected native quote-reply turns, followed by a final native quote reply and cleanup of the temporary preview.

## Best Possible Solution

Have the Telegram owner decide whether selected-quote turns may use a second temporary progress message; if approved, implement it in the Telegram plugin with cleanup/error semantics, docs/doctor updates, focused tests, and live Telegram proof.

## Reproduction Assessment

Not applicable as a bug reproduction because this requests new behavior. Source inspection does confirm the current limitation: selected quote replies skip answer draft streaming, so preview tool-progress cannot render for those turns.

## Solution Assessment

Unclear until product review. The proposed temporary progress lane is plausible, but it changes the current Telegram streaming contract that one visible preview message owns the stream unless final edit fails.
