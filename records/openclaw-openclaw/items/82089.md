## Summary

Keep open: current main and v2026.5.18 still intentionally disable Telegram answer/progress draft streams for selected native quote replies, and prior review left a separate temporary-progress lane as future product work rather than cleanup closure.

## What This Changes

The issue requests temporary Telegram tool-progress previews during selected native quote-reply turns, with the final answer still delivered as a native Telegram quote reply.

## Best Possible Solution

Have the Telegram owner decide whether selected-quote turns may create a separate temporary progress lane; if approved, implement it in the Telegram plugin with cleanup/failure rules, docs/doctor updates, focused dispatch tests, and live Telegram proof.

## Reproduction Assessment

Not applicable as a bug reproduction because this requests new Telegram behavior. Source inspection gives high confidence in the current limitation: selected quote replies disable the answer draft stream, and progress rendering returns when that stream is absent.

## Solution Assessment

Unclear until Telegram owner/product review. The proposed temporary progress lane is plausible, but it must define cleanup and failure behavior without regressing final native quote delivery.
