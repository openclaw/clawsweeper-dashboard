## Summary

Keep open: current main still implements and documents selected Telegram quote replies as a trade-off that skips the answer draft stream, so the requested temporary progress message is still a new Telegram runtime behavior that needs product/owner approval.

## What This Changes

The issue requests temporary Telegram tool-progress previews for selected quote-reply turns while keeping the final answer as a native Telegram quote reply.

## Best Possible Solution

Have the Telegram owner decide whether selected-quote turns may use a second temporary progress message; if approved, implement it in the Telegram plugin with cleanup/failure semantics, docs/doctor updates, focused tests, and live Telegram proof.

## Reproduction Assessment

Not applicable as a bug reproduction because this requests new behavior. Source inspection gives high confidence in the current limitation: selected quote context disables the answer draft stream used by preview tool-progress.

## Solution Assessment

Unclear until product review. The proposed temporary progress lane is plausible, but it changes the current Telegram streaming contract that one visible preview message owns the stream unless final edit/final delivery requires cleanup.
