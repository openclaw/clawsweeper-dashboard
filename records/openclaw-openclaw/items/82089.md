## Summary

Keep open: current main still intentionally treats selected Telegram native quote replies as a trade-off that skips the answer draft stream and therefore skips preview tool-progress, while the requested temporary progress message is new Telegram runtime behavior needing owner/product review.

## What This Changes

The issue requests Telegram to show a temporary tool-progress preview during selected native quote-reply turns, then deliver the final answer as a native Telegram quote reply and clean up the preview.

## Best Possible Solution

Have the Telegram owner decide whether selected-quote turns may use a separate temporary progress message; if approved, implement it inside the Telegram plugin with cleanup/failure semantics, docs/doctor updates, focused tests, and live Telegram proof.

## Reproduction Assessment

Not applicable as a bug reproduction because this asks for new Telegram behavior. Source inspection does confirm the current-main path: selected quote text with replyToMode enabled prevents the answer draft stream, so preview tool-progress cannot render for that turn.

## Solution Assessment

Unclear until product review. The temporary progress lane is plausible, but it changes Telegram's current one-preview-message streaming model and needs owner decisions on cleanup, errors, approvals, and final delivery semantics.
