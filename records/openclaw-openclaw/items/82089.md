## Summary

Keep open. Current main still intentionally skips Telegram preview/tool-progress streaming for selected native quote replies, and this asks for a new temporary progress-message path plus cleanup semantics rather than a missing already-defined bug fix.

## What This Changes

The issue requests temporary Telegram tool-progress previews during selected-quote turns, followed by final delivery through Telegram native quote replies and cleanup of the temporary preview.

## Best Possible Solution

Design a Telegram-owned temporary progress lane for selected native quote turns only if maintainers accept the extra transient message semantics, then update docs/doctor and prove the flow with live Telegram evidence.

## Reproduction Assessment

Yes for the current limitation: source and tests show that non-off `replyToMode` with selected quote text skips the answer draft stream that renders preview tool progress. This is not a broken existing contract, though; the requested behavior is a new feature on top of the documented trade-off.

## Solution Assessment

Unclear. The requested direction is plausible, but it adds a separate temporary Telegram message path with deletion/error semantics, so maintainers should approve the product shape before implementation.
