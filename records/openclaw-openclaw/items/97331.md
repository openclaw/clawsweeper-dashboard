## Summary

Keep open: current `main` still copies parent `contextTokens` into dashboard child sessions, the issue has the protected `maintainer` label, and https://github.com/openclaw/openclaw/pull/97332 is the open candidate fix rather than a merged implementation.

## What This Changes

The issue reports that new Control UI dashboard child sessions inherit stale cached parent context-token metadata instead of deriving the current model context budget.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/97332 if it preserves inherited runtime model/auth settings while dropping cached context and usage metadata from newly created child sessions.

## Reproduction Assessment

Yes from source inspection: `sessions.create` inherits the parent runtime selection and current main includes `contextTokens`, so a stale parent entry can seed a child row with stale context metadata. I did not run a live dashboard repro because this review is read-only.

## Solution Assessment

Yes, the linked PR is the right narrow fix shape: keep inherited provider/model/auth/runtime settings but stop copying cached context and usage metadata. Starting a separate ClawSweeper repair branch would duplicate the open candidate fix.
