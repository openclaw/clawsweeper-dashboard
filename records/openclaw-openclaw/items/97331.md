## What This Changes

The issue reports that dashboard child sessions can inherit stale parent context-budget metadata instead of deriving the current model budget.

## Best Possible Solution

Review and land the linked candidate fix if it preserves inherited provider/model/auth/runtime settings while dropping cached context and usage metadata from newly created child sessions.

## Reproduction Assessment

Yes. Source inspection shows `sessions.create` inherits parent runtime selection and current main includes `contextTokens`, so a stale parent entry can seed a child row with stale context metadata; no live dashboard repro was run in this read-only review.

## Solution Assessment

Yes, the linked PR appears to be the narrow owner-boundary fix shape: keep inherited provider/model/auth/runtime settings but stop copying cached context and usage metadata. Starting a separate ClawSweeper repair branch would duplicate the open candidate fix.
