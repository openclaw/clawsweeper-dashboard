## Summary

Keep this PR open: it is recent, maintainer-labeled, and adds a user-facing feature, but the current patch has a blocking privacy/security issue in the generated PNG temp-file path.

## What This Changes

The PR adds `/context map`, a generated PNG treemap for the current context report, and updates the slash-command docs, changelog, and focused command test.

## Best Possible Solution

Keep the feature shape if maintainers want it, but store generated context-map media in OpenClaw's private temp/media area with restrictive permissions before landing.

## Reproduction Assessment

Yes for the review finding: source inspection shows `/context map` writes its sensitive PNG to `os.tmpdir()` with default `writeFile` permissions. For the feature itself, bug-style reproduction is not applicable, and the PR body provides live Telegram output showing the command returns a photo.

## Solution Assessment

No, not as currently written: reusing the existing context report and marking the payload sensitive is a good direction, but the generated file must not be left as a default-mode file in the shared OS temp directory.
