## Summary

Keep this PR open. Current main does not already implement the strict tool mode/report contract, but the submitted patch has blocking integration gaps in the generated config surface and the real Google provider-owned transport path.

## What This Changes

The PR adds opt-in strict/warn/off tool-call strictness handling and diagnostics across embedded agent runs, provider transport replay, config, OpenAI-compatible gateway responses, tests, and docs.

## Best Possible Solution

Make the PR internally consistent before merge: regenerate the config schema/docs artifacts, carry strictness through the real Google provider-owned stream path with an appropriate plugin/provider seam, and preserve default permissive behavior.

## Reproduction Assessment

Yes for the review findings. The config drift is reproducible from the missing generated artifact changes, and the Google gap is reproducible by following embedded stream resolution into provider registration, where the strictness option is never passed to the Google factory.

## Solution Assessment

No as submitted. The feature direction is reasonable, but this is not the best mergeable form until all public config surfaces and real provider runtime paths reflect the new strictness contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Regenerate the base config schema:** `src/config/zod-schema.ts:350-355`
  - body: Adding `toolStrictness` to `OpenClawSchema` is not enough because `openclaw config schema` is served from `GENERATED_BASE_CONFIG_SCHEMA`. Without regenerating and committing `src/config/schema.base.generated.ts` and the config docs baseline, schema/UI/docs consumers will not expose the new key even though the parser accepts it.
  - confidence: 0.91

  - body: The Google transport factory now accepts `strictness`, but the provider-owned runtime path still resolves the Google stream without passing that option. Embedded runs using Google can therefore continue repairing replayed JSON-string tool args even when strict mode is requested.
  - confidence: 0.89
