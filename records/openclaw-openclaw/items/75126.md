  - body: The new early return runs before `emitJsonEnvelope()` and before the existing no-visible-payload deliveryStatus path. In `--json` mode, an empty agent reply now logs text and returns without writing the structured envelope callers currently rely on.

  - body: The helper still types the options as `mode?: "off" | "strict"`, but the new tests pass `strictToolMode: true`. TypeScript will reject those object literals, so the added test suite cannot typecheck.
  - confidence: 0.95

  - body: The PR deletes both tracked `.sha256` baseline files, but `docs/.generated/README.md` says those hashes are the committed drift-detection artifacts.   - confidence: 0.9
- **[P2] Thread strict mode into argument repair:** `src/agents/pi-embedded-runner/run/attempt.ts:2379-2381`
  - body: The runner resolves `strictToolMode` and passes it to the name/transcript wrappers, but provider-specific malformed argument repair is still wrapped without that mode. Kimi, OpenAI completions, and Responses-style repair paths can still silently repair arguments in strict mode.
  - confidence: 0.86
