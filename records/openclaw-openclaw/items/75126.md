Full review comments:

- **[P2] Preserve JSON envelopes for empty replies:** `src/agents/command/delivery.ts:506-511`
  - body: The new early return runs before `emitJsonEnvelope()` and before the existing no-visible-payload delivery status path. In `--json` mode, an empty agent reply now logs text and returns without writing the structured envelope callers currently rely on.
  - confidence: 0.91

  - body: The helper still types the options as `mode?: "off" | "strict"`, but the new tests pass `strictToolMode: true`. TypeScript will reject those object literals, so the added test suite cannot typecheck.
  - confidence: 0.95
- **[P2] Regenerate generated baseline hashes:** `docs/.generated/config-baseline.sha256:1-4`
  - body: The PR deletes both tracked `.sha256` baseline files, but `docs/.generated/README.md` says those hashes are committed drift-detection artifacts. A config schema change should regenerate `config-baseline.sha256`; deleting the Plugin SDK baseline is unrelated and breaks the generated-artifact contract.
  - confidence: 0.9
- **[P2] Gate malformed argument repair in strict mode:** `src/agents/pi-embedded-runner/run/attempt.ts:2379-2381`
  - body: The runner resolves `strictToolMode` and passes it to the name/transcript wrappers, but provider-specific malformed argument repair is still wrapped without that mode. Kimi, OpenAI completions, and Responses-style repair paths can still silently repair arguments in strict mode.
  - confidence: 0.86
