## Summary

Keep open. Current main still uses a 30-second TUI streaming watchdog by default, and only the internal event-handler harness can override or disable it; the public CLI, TUI command surface, config schema, and docs do not expose the requested control.

## What This Changes

The issue requests a user-facing way to configure or disable the TUI streaming watchdog timeout for long silent reasoning phases.

## Best Possible Solution

Add a validated TUI-owned watchdog control, plumb it from the chosen public surface into `createEventHandlers`, document it, and cover default, configured, disabled, and warning behavior with focused tests.

## Reproduction Assessment

Yes. Source inspection and existing fake-timer tests show that a run with no stream delta for the watchdog window triggers the warning/reset path, and normal `runTui` does not override the 30-second default.

## Solution Assessment

Unclear on the exact UX. A TUI-owned setting or command is the maintainable direction, while provider/model timeout changes would not solve this client-side watchdog and #69026's fixed threshold bump would not satisfy configurability.
