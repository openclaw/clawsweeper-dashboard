## Summary

Keep open. Current main still has only an internal TUI streaming watchdog override, while normal TUI startup, CLI options, config schema, and public docs expose no user-facing timeout or disable control; open implementation work still needs maintainer review rather than cleanup closure.

## What This Changes

The issue requests a configurable TUI streaming watchdog timeout or disable control for long silent reasoning/model phases.

## Best Possible Solution

Land a focused, validated TUI-owned watchdog control that plumbs the chosen public surface into `createEventHandlers`, documents default and disabled behavior, and keeps the related quiet-run/dedup work coordinated with https://github.com/openclaw/openclaw/issues/69978 and https://github.com/openclaw/openclaw/issues/78360.

## Reproduction Assessment

Yes. Source inspection and existing fake-timer tests show the 30-second watchdog still fires unless an internal `streamingWatchdogMs` value is passed, and normal TUI startup does not pass one.

## Solution Assessment

Unclear on the exact UX. A TUI-owned config/command/flag surface is the maintainable direction, but maintainers still need to choose the public contract and decide whether the broader CLI watchdog default changes in the open PR belong with this issue.
