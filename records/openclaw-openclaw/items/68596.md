## Summary

Keep open. Current main still leaves the TUI watchdog user-facing behavior fixed at the 30-second default: the handler has only an internal `streamingWatchdogMs` option, while normal TUI startup, CLI options, slash commands, config schema, and docs expose no threshold or disable control; related #69026 and #69978 remain open rather than shipped fixes.

## What This Changes

The issue requests a configurable TUI streaming watchdog timeout or disable switch for long-silence reasoning/model runs.

## Best Possible Solution

Add a validated TUI-owned watchdog control, plumb it into `createEventHandlers`, document it, and cover default, configured, disabled, and warning behavior with focused tests.

## Reproduction Assessment

Yes. Source inspection and fake-timer tests give a high-confidence path: a run with no stream delta for the watchdog window triggers the warning/reset, and normal `runTui` does not pass an override.

## Solution Assessment

Unclear. The internal handler option is a useful lower-level seam, but #69026's hardcoded threshold bump would not satisfy configurability and the final UX/API surface needs maintainer choice.
