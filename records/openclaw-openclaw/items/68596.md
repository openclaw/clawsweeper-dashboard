## Summary

Keep open: current main and the latest release still only expose an internal TUI watchdog override, while users cannot configure or disable the 30-second threshold through config, CLI, docs, or the normal TUI startup path; https://github.com/openclaw/openclaw/pull/79997 is an open attempt but is not merged or proof-positive yet.

## What This Changes

Allow users to configure or disable the TUI streaming watchdog threshold instead of being stuck with the 30-second default during quiet long-running model turns.

## Best Possible Solution

Choose one canonical TUI watchdog control surface, validate and document it, pass the value into `runTui`/`createEventHandlers`, and keep the current quiet-active-run behavior intact with focused TUI and config tests.

## Reproduction Assessment

Yes. Source inspection on current main shows the handler default remains 30,000 ms, normal `runTui` startup does not pass `streamingWatchdogMs`, and the config schema has no public watchdog key.

## Solution Assessment

Unclear on the exact public surface. The existing handler seam is the right low-level hook, but maintainers still need to choose the durable config, CLI, or TUI command contract before implementation.
