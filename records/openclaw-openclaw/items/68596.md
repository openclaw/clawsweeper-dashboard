## Summary

Keep open: current main and v2026.6.10 still expose only an internal TUI handler override, not a user-facing watchdog config path, and the open implementation PR is still unmerged and review-blocked. The issue remains the canonical tracker for the TUI streaming-watchdog configuration decision.

## What This Changes

The issue requests a user-facing configuration knob to tune or disable the TUI streaming watchdog timeout for long quiet reasoning or tool-call phases.

## Best Possible Solution

Keep this canonical issue open while maintainers review or replace the open PR with a top-level TUI config path, preserved pending-notice semantics, docs/schema metadata, and proof for default, configured, and disabled behavior.

## Solution Assessment

Unclear as proposed: the need is real, but the issue's suggested gateway or acp.stream locations are probably not the cleanest owner for a terminal UI watchdog. The best path is a maintainer-approved TUI namespace and default policy, then implementation through the open PR or a narrow replacement.
