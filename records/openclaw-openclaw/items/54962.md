## Summary

Keep this PR open because the reported Telegram callback bug is valid on current main, but the proposed one-line patch is too broad and would break existing slash-command callback routing before merge.

## What This Changes

The PR changes the Telegram callback-query fallback to send `callback_data: ${data}` to the agent instead of the raw callback payload.

## Best Possible Solution

Update or replace this PR with a conditional Telegram callback fallback that prefixes only generic agent callbacks, preserves slash/native/internal command routes, and adds focused regression coverage plus a changelog entry.

## Reproduction Assessment

Yes by source inspection: current main sends generic callback data raw while the Telegram docs require `callback_data: <value>`. I did not run a live Telegram/model webhook reproduction in this read-only review.

## Solution Assessment

No. The intended fix is right, but this exact patch prefixes every fallback payload and needs to preserve slash-command callbacks such as `/models add <provider>`.
