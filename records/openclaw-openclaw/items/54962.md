## Summary

Keep open: the underlying Telegram callback text bug still exists on current main and the latest release, but this PR is stale against current callback routing and lacks required real Telegram proof before merge.

## What This Changes

The PR changes Telegram callback-query fallback routing to forward `callback_data: ${data}` to the agent instead of the raw callback payload.

## Best Possible Solution

Rebase or replace this with a targeted main-line fix that prefixes only generic fallback callback text, preserves native/internal callback routes, adds regression coverage, and attaches redacted real Telegram proof before merge.

## Reproduction Assessment

Yes for the source-level routing contract: current main and `v2026.5.7` forward generic callback fallback text as raw data while the docs promise `callback_data: <value>`. I did not establish a live Telegram/webhook run in this read-only review.

## Solution Assessment

No. Prefixing generic fallback text is the right layer, but this branch must preserve `nativeCallbackCommand` and raw slash-command callback paths before it is mergeable.
