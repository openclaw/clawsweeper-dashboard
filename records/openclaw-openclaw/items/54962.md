## Summary

Keep open: the underlying Telegram callback bug is still present on current main and the latest release, but this PR is not merge-ready because it lacks inspectable real behavior proof and a current-main rebase must preserve native and slash callback routing.

## What This Changes

The PR changes the Telegram callback-query fallback synthetic message to send `callback_data: ${data}` instead of the raw callback payload.

## Best Possible Solution

Rebase or replace this with a targeted main-line fix that prefixes only generic fallback callbacks, preserves internal/native/slash callback paths, adds regression coverage, and includes redacted live Telegram proof.

## Reproduction Assessment

Yes for the source-level routing contract: current main and `v2026.5.7` route generic callback fallback text as raw data while the docs promise `callback_data: <value>`. I did not establish a live Telegram/webhook run in this read-only review.

## Solution Assessment

No. Prefixing generic fallback text is the right layer, but this stale branch needs a targeted rebase so native `tgcmd:` and slash-command callback paths stay structured.
