## Summary

Keep open: the documented Telegram callback text-shape bug still exists on current main, but this PR cannot merge as-is because it lacks inspectable real Telegram proof and needs a rebase that preserves native/slash callback routing.

## What This Changes

The PR changes Telegram callback-query fallback text from raw callback data to `callback_data: <data>` in `extensions/telegram/src/bot-handlers.runtime.ts`.

## Best Possible Solution

Land a rebased equivalent that prefixes only generic callback fallback text, preserves internal/native/slash callback paths, adds regression coverage, and includes redacted real Telegram proof.

## Reproduction Assessment

Yes for the source-level routing contract: current main sends generic Telegram callback fallback text as raw data while the docs require `callback_data: <value>`. I did not establish a live Telegram/webhook run in this read-only review.

## Solution Assessment

No, not as-is. Prefixing generic fallback text is the right layer, but the PR branch needs a targeted rebase so the prefix applies only after native and internal callback paths are excluded.
