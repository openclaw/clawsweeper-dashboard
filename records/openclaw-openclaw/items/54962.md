## Summary

Keep this PR open. It targets a real Telegram callback-routing bug, but the one-line branch is not merge-ready against current main because callback fallback data also carries command/native callback payloads that must stay routable.

## What This Changes

The branch changes the Telegram callback-query fallback to forward inline button callback data as `callback_data: ${data}` instead of the raw callback value.

## Best Possible Solution

Rebase the Telegram handler on current main so `callback_data:` is used only for generic user callbacks, while specialized, native, and slash-command callback paths stay on their existing command routes with focused tests and a changelog entry.

## Reproduction Assessment

Yes. The linked bug gives concrete Telegram inline-button steps, and source inspection confirms current main still forwards generic callback data raw while the docs require prefixed agent text.

## Solution Assessment

No. The one-line prefix is aimed at the right bug, but the maintainable fix must preserve current-main command/native callback routing and add regression coverage for both prefixed generic callbacks and command callbacks.

## Review Findings
