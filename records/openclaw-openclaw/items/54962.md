## Summary

Keep this PR open, but not merge-ready: it targets a real Telegram callback/docs mismatch, while the one-line prefix change is too broad for current command callback routing.

## What This Changes

The PR changes the Telegram callback-query fallback to forward inline button callback data as `callback_data: ${data}` instead of the raw callback value.

## Best Possible Solution

Rebase the Telegram handler so only generic user callbacks are prefixed, while plugin, approval, pagination, model, native, and slash-command callback routes keep their existing behavior with focused tests and a changelog entry.

## Reproduction Assessment

Yes. The linked bug gives concrete Telegram inline-button steps, and current main source inspection confirms generic fallback callbacks still reach the agent as raw data while the docs require prefixed text.

## Solution Assessment

No. The PR is aimed at the right bug, but the current one-line replacement is broader than the maintainable fix because fallback callback payloads can intentionally be commands.
