## Summary

Keep this PR open, but do not merge it as-is: the underlying Telegram callback bug is valid, while the one-line patch is too broad and would break current slash-command callback routing.

## What This Changes

This PR changes the Telegram callback-query fallback to send `callback_data: ${data}` to the agent instead of the raw callback payload.

## Best Possible Solution

Rebase or replace this PR with a narrow Telegram fix that prefixes only generic agent fallback callbacks, preserves native and slash-command callback routes, and adds regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. Current main source shows generic Telegram callbacks are forwarded as raw callback data while the docs require `callback_data: <value>`; live Telegram webhook proof was not run in this read-only pass.

## Solution Assessment

No. The intent is right, but prefixing every fallback payload is not the narrowest maintainable fix because current slash-command callback payloads still need to reach command parsing with a leading `/`.
