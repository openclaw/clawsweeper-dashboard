## Summary

Keep open: the PR targets a real Telegram callback contract bug, but it lacks inspectable real behavior proof and the stale one-line patch must be rebased to preserve current native/slash callback routing.

## What This Changes

The PR changes Telegram callback-query fallback routing to forward `callback_data: ${data}` to the agent instead of the raw callback payload.

## Best Possible Solution

Rebase or replace this with a targeted main-line fix that prefixes only generic fallback text, preserves native/internal and raw slash callback routes, adds regression coverage, and attaches real Telegram proof before merge.

## Reproduction Assessment

Yes for the source-level routing contract: current main and v2026.5.7 still route generic Telegram callback fallback text as raw data while the docs promise `callback_data: <value>`. I did not establish a live Telegram/webhook run in this read-only review.

## Solution Assessment

No. Prefixing generic callbacks is the right layer, but this branch is stale against current callback handling and must preserve `nativeCallbackCommand` plus raw slash-command callback payloads before merge.
