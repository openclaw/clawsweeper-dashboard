## Summary

Current main no longer has the reported active-agent auth-store collision: omitted auth-store paths now resolve through the default agent directory instead of the active `OPENCLAW_AGENT_DIR`, while `models status --agent` and MiniMax catalog resolution pass the selected agent directory into auth-profile loading.

## What This Changes

The issue reports that MiniMax OAuth for a selected agent such as `kate` is effectively read from `agents/main/agent/auth-profiles.json` instead of that agent's `auth-profiles.json`.

## Best Possible Solution

Keep the current main agent-dir/auth-store resolution and let the next release carry it to packaged users.

## Reproduction Assessment

No current-main failing reproduction remains by source inspection. `models status --agent kate` now resolves `kate`'s agent directory explicitly, and omitted auth-store paths no longer follow `OPENCLAW_AGENT_DIR`, so the prior snapshot collision path is gone.

## Solution Assessment

Yes. The current main fix is the maintainable shared auth-path solution and avoids a MiniMax-specific workaround or credential-copy behavior.
