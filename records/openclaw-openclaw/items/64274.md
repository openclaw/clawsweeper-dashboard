## Summary

Keep open: current main and v2026.5.18 still have the reported main-first auth-profile precedence, and the linked closing PR remains open rather than merged.

## What This Changes

Ensure MiniMax auth for a selected agent resolves that agent's local auth-profiles.json before inherited main-agent profiles.

## Best Possible Solution

Land a focused shared auth-profile precedence fix that tries selected-agent same-provider profiles before inherited main profiles while preserving main fallback and explicit auth-order exclusions.

## Reproduction Assessment

Yes, by source inspection. A main store with a MiniMax portal profile plus a selected-agent store with a different local MiniMax portal profile can leave the inherited main profile first in current-main resolution.

## Solution Assessment

Yes: the narrow maintainable fix belongs in shared auth-profile merge/order semantics, not in MiniMax-specific token copying or provider special cases. The linked PR is the right path after its auth-order and unrelated-change findings are resolved.
