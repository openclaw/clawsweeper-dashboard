## Summary

Keep open: current main and the latest shipped tag still have the reported main-first auth-profile precedence, and the focused closing PR remains open rather than merged.

## What This Changes

The issue reports that a non-main agent's MiniMax OAuth profile can be bypassed because shared auth-profile resolution tries an inherited main-agent profile first.

## Best Possible Solution

Land a focused shared auth-profile precedence fix that tries selected-agent same-provider profiles before inherited main profiles while preserving main fallback for providers with no local profile.

## Reproduction Assessment

Yes, by source inspection. Current main merges main and selected-agent auth stores in a way that can preserve inherited main MiniMax OAuth order ahead of a selected-agent MiniMax OAuth profile when the profile IDs differ.

## Solution Assessment

Yes. The narrow maintainable fix belongs in shared auth-profile merge/order precedence, and the open linked PR is the right implementation path to review rather than opening a duplicate repair lane.
