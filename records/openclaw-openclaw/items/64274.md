## Summary

Keep open: current main and the latest release still preserve inherited main-agent auth profiles ahead of selected-agent same-provider profiles in this source path, and the open closing PR has not merged yet.

## What This Changes

The issue reports that a selected agent's MiniMax OAuth profile can lose precedence to an inherited main-agent auth profile even though status points at the selected agent's auth store.

## Best Possible Solution

Land a focused shared auth-profile precedence fix that tries selected-agent same-provider profiles before inherited main profiles while preserving main fallback and explicit auth-order exclusions.

## Reproduction Assessment

Yes, by source inspection. A main store with a same-provider MiniMax profile plus a selected-agent store with a different MiniMax profile id can merge with inherited main order first on current main.

## Solution Assessment

Yes. The narrow maintainable solution belongs in shared auth-profile merge/order semantics, not in MiniMax-specific token copying or provider special cases.
