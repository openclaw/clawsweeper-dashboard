## Summary

Keep open: current main and the latest release still have the main-first merge/order behavior, and the linked closing PR is open and not merged yet.

## What This Changes

The issue requests agent-specific MiniMax OAuth to resolve from the selected agent's `auth-profiles.json` before inherited main-agent credentials.

## Best Possible Solution

Land a focused shared auth-profile precedence fix that tries selected-agent same-provider profiles before inherited main profiles while preserving main fallback and explicit auth-order filtering.

## Reproduction Assessment

Yes, by source inspection rather than a live MiniMax OAuth run. A main store with `minimax-portal:cli` plus a selected-agent store with `minimax-portal:default` and no selected-agent order can resolve the inherited main profile first on current main.

## Solution Assessment

Yes, the right fix boundary is shared auth-profile merge/order semantics. The issue should remain open until the linked PR or an equivalent focused fix lands and preserves fallback plus explicit order behavior.
