## Summary

Keep open: current main removed the legacy env-based agent-dir resolver from runtime auth paths, but selected-agent MiniMax auth can still lose to an inherited main MiniMax profile when the profile IDs differ, because the merged store and profile ordering keep main-store same-provider profiles ahead of local ones.

## What This Changes

The issue asks for MiniMax auth in a selected agent session to resolve from that agent's `auth-profiles.json` rather than from `agents/main`.

## Best Possible Solution

Land a focused auth-profile precedence fix so selected-agent profiles for a provider override inherited main profiles for that provider, while retaining main fallback for providers absent from the selected agent.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create main and selected-agent stores with different `minimax-portal` OAuth profile IDs, then the base-first merge plus stable same-type ordering can try the main profile before the selected-agent profile.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the shared auth-profile merge/order semantics with regression coverage, not in a MiniMax-specific special case or credential copying.

## Repair Work Prompt

Current main already removed the legacy env-based default agent-dir resolver from runtime paths, so do not reintroduce `resolveOpenClawAgentDir` into core runtime/auth paths. The remaining bug is selected-agent provider precedence: when a non-main agent has any local `minimax-portal` profile, inherited main `minimax-portal` profiles with different IDs must not be tried before the local agent profile. Preserve main-store fallback for providers that are absent from the selected agent, preserve exact profile-id override behavior, and avoid a MiniMax-only special case if a shared auth-profile merge/order helper is cleaner. Add regression coverage for main `minimax-portal:minimax-cli` plus selected-agent `minimax-portal:default` with the selected agent winning, including the runtime snapshot path when feasible.
