## Summary

Keep open. Current main still implements WebChat agent/session switching as a grouped native dropdown in the chat header, not as a persistent sidebar-style agent conversation list, and the related dropdown/session issues do not supersede this distinct UX request.

## What This Changes

The issue requests replacing or augmenting the WebChat multi-agent dropdown with a persistent sidebar list where configured agents appear as clickable conversations.

## Best Possible Solution

Add a focused Control UI WebChat sidebar/list that uses existing agents.list and sessions.list data, optionally sessions.preview or includeLastMessage for previews, routes clicks through the existing session-switch path, preserves model/thinking controls and a compact dropdown fallback, and documents/tests the multi-agent behavior.

## Reproduction Assessment

Not applicable as a feature request. The source-level verification is high-confidence: current main renders a native header select for switching and has no persistent agent conversation list in the chat layout.

## Solution Assessment

Yes, if scoped conservatively. The best first solution is a UI-only Control UI change over existing Gateway session/agent RPCs, with broader team hierarchy and new-session creation handled only where they naturally fit or remain in their separate issues.
