## Summary

Keep open. Current main has a documented WebChat agent filter plus session picker, but it still does not provide the requested sidebar-style agent conversation list; related Control UI issues overlap but do not supersede this exact WebChat layout request.

## What This Changes

Request to replace the WebChat multi-agent dropdown/filter workflow with a persistent sidebar-style agent conversation list for faster agent switching.

## Best Possible Solution

Choose the Control UI multi-agent chat model first, then implement a desktop sidebar or visible agent list that reuses existing agents.list and sessions.create/session-switching contracts with responsive fallback and focused UI tests.

## Reproduction Assessment

Not applicable as a feature request. Source, docs, and tests give a high-confidence verification path that current main still uses an agent filter/select plus session picker rather than a sidebar agent conversation list.

## Solution Assessment

Unclear. The requested sidebar is plausible, but the best maintainable solution needs a maintainer UX decision on whether this is only a more visible switcher, a per-agent session-history sidebar, or part of the broader New Chat/agent-selection work.
