- **[P1] Guard the redeploy target before deleting it:** `scripts/redeploy-control-ui.sh:28`
  - body: The script lets `OPENCLAW_INSTALLED_DIR` point anywhere and then runs `rm -rf` before any sanity check. A typo can delete an arbitrary current-user directory before the service restart, so validate the resolved target or require an explicit force mode before removing it.
  - confidence: 0.91
- **[P2] Fail verification when deployed assets do not match:** `scripts/redeploy-control-ui.sh:45-54`
  - body: The Python block only prints existence and `same_index`; missing files or a false comparison still fall through to `Done.` with exit 0. Make the verification raise or exit non-zero so stale Control UI assets are not reported as a successful redeploy.
  - confidence: 0.94
- **[P2] Remove the personal install path from the docs:** `docs/web/control-ui-redeploy.md:29-31`
  - body: The runbook includes the author's local `/home/gonzo/...` install path, which is not reusable and conflicts with the docs rule to keep local paths generic. Replace it with a dynamic `$(npm root -g)` example or a placeholder.
  - confidence: 0.96
- **[P2] Localize the new static shell copy:** `ui/index.html:270-286`
  - body: The added `index.html` shell renders English headings and notes outside the Control UI i18n system, so non-English users still see English onboarding copy. Move this copy into the app/i18n path or remove the static shell text.
  - confidence: 0.84
- **[P3] Use python3 in the manual redeploy check:** `docs/web/control-ui-redeploy.md:69`
  - body: The docs' quick manual check still invokes `python`, while the script now uses `python3`; hosts without a `python` shim will fail this verification snippet. Use `python3` consistently.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The PR adds a local redeploy script with destructive filesystem mutation and a temporary PATH shim that need guardrails before merge.

Concerns:

- **[medium] Unbounded target deletion:** `scripts/redeploy-control-ui.sh:28`
  - body: `rm -rf "$INSTALLED_DIR"` runs on an overrideable path without checking that it is the intended OpenClaw Control UI install directory, so a mistaken or malicious environment value can delete unrelated files as the current user.
  - confidence: 0.91
- **[low] Predictable temporary PATH shim:** `scripts/redeploy-control-ui.sh:9`
  - body: The script writes a `pnpm` shim under a fixed `/tmp/openclaw-pnpm-shim` directory and prepends it to PATH for the build. Use a private `mktemp -d` directory with cleanup to avoid shared-system clobbering or unexpected executable resolution.
  - confidence: 0.72
