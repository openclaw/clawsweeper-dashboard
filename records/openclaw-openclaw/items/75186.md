## Summary

Keep open. Current main still lacks the typed Gateway plugin-management RPCs, the merged Crestodian work covers CLI/Crestodian plugin operations rather than this Gateway protocol surface, and maintainer discussion explicitly says the PR is useful but not ready to land as-is.

## What This Changes

This PR adds Gateway `plugins.*` management RPC schemas, handlers, admin scopes, write rate limits, shared plugin-management services, Swift protocol models, docs, and tests.

## Best Possible Solution

Keep the service-layer direction only if maintainers accept the remote admin boundary, then fix the install error classification, add a single-line changelog entry, rebase, and run targeted Gateway/plugin tests plus the changed gate before merge.

## Reproduction Assessment

Yes for the blocking behavior: call `plugins.install` with `source: "path"` and a missing Gateway-host path. Current install helpers return an uncoded `directory not found` or `file not found` failure, and PR head maps that to `UNAVAILABLE`.

## Solution Assessment

No as-is. The shared service layer is a plausible maintainable path, but the branch should preserve invalid-request semantics for bad local paths and needs maintainer approval for the new remote management boundary.
