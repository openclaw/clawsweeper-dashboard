## Summary

Keep open. Current main still lacks Gateway plugin-management RPCs, so this PR is not obsolete, but the patch should not merge as-is because one install error path still reports deterministic bad input as a transient outage and the required user-facing changelog entry is missing.

## What This Changes

This PR adds typed Gateway `plugins.*` management RPC schemas and handlers, shared plugin-management services reused by CLI flows, admin scopes and write rate limits, generated Swift protocol models, docs, and tests.

## Best Possible Solution

Land the feature after correcting install error classification, adding a single-line `CHANGELOG.md` entry with appropriate contributor credit, and getting green protocol/generated-client and changed-gate validation for the Gateway and plugin-management surfaces.

## Reproduction Assessment

Yes for the blocking review finding: call `plugins.install` with `source: "path"` and a missing path on the Gateway host. Current path resolution returns an uncoded `path not found` failure, and the PR maps that undefined code to `UNAVAILABLE`.

## Solution Assessment

No as-is. The shared service layer plus admin-scoped, rate-limited Gateway handlers is a maintainable direction, but the PR should preserve invalid-request semantics for deterministic local install input failures and satisfy the changelog policy before merge.
