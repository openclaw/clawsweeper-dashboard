## Best Possible Solution

## Solution Assessment

No as-is. The shared service-layer approach is plausible, but the branch should preserve client-error semantics for bad host paths and wait for maintainer acceptance of the remote plugin-management boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Classify missing path installs as invalid requests:** `src/plugins/management-core.ts:75-78`
