## What This Changes

The PR adds admin-scoped Gateway `plugins.*` RPC schemas, handlers, shared plugin-management services, Swift protocol models, docs, and focused Gateway/plugin tests.

## Best Possible Solution

Keep the service-layer approach only if maintainers accept the admin-scoped remote management boundary, then fix the error mapping and changelog, rebase, and validate the Gateway protocol/plugin lanes before merge.
