## Summary

The issue remains open and actionable: current main has watchdog and cleanup machinery for this path, but no source/history proof shows the reported scheduled `model-call-started` stall and persistent queue poisoning are fixed.

## What This Changes

## Best Possible Solution

Make every scheduled isolated attempt deterministically release its model-call, session queue, and cron lane state on abort or stall, then prove repeated automatic runs and a manual run after failure without restarting the gateway.
