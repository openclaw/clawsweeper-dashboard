## Best Possible Solution

Land a rebased test-scoped patch that keeps the replay regression, fixes the copied-config type, preserves supported `bundledDiscovery` enum values, and coordinates coverage with https://github.com/openclaw/openclaw/pull/76783.

## Reproduction Assessment

Yes. Source inspection shows the PR's `parsed.plugins` access conflicts with the current `sanitizeLiveConfig` type, and current main's config/provider code proves `plugins.bundledDiscovery` has supported values that the patch would strip.

## Solution Assessment

No, not as-is. The regression-test direction is narrow, but the branch should widen the copied-config type and only remove invalid legacy `bundledDiscovery` values while preserving supported `"compat"` and `"allowlist"` settings.
