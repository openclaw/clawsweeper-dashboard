## Summary

Keep open. The normal current-main and release-era gateway paths appear to honor `gateway.tailscale.mode: "off"`, and an independent Docker check against `openclaw@2026.5.7` plus the Hostinger image could not reproduce repeated Tailscale calls, but the reported systemd/LXC environment is still concrete enough that closing without the exact service unit, image digest, and journal evidence would be premature.

## What This Changes

The issue reports that setting `gateway.tailscale.mode: "off"` still allows repeated `sudo tailscale serve` invocations from the gateway service.

## Best Possible Solution

Capture the effective service command and runtime status, then fix the service/doctor/wrapper path if it is pinning Serve despite config, or document the external supervisor setup if OpenClaw is not making the calls.
