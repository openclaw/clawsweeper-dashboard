## Summary

Close as cannot reproduce. Current main’s normal Gateway startup path treats `gateway.tailscale.mode: "off"` as no Tailscale automation, source search found no 3-second Serve reassert loop, and a follow-up Docker check against both `openclaw@2026.5.7` and the Hostinger image captured no `tailscale` or `sudo` calls.

## What This Changes

The issue reports that `gateway.tailscale.mode: "off"` still causes repeated `sudo tailscale serve` calls from the managed Gateway service.

## Best Possible Solution

Keep the existing off-mode guards; if the symptom reappears, diagnose the specific service command or image wrapper that is invoking Tailscale outside the normal Gateway startup path.

## Reproduction Assessment

No. Current-main source inspection and an independent Docker reproduction against the reported release/image path both show no `tailscale` or `sudo` invocations when the effective mode is `off`.

## Solution Assessment

Yes. Adding another generic off-mode skip is not the maintainable fix because the normal config, startup, and helper paths already have that guard; any future fix should target a proven wrapper, override, or service-unit path.
