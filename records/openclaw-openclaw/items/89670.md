## What This Changes

The branch changes the Control UI skills list to render skill rows with Lit keyed `repeat()` and adds a jsdom regression test for stale checkbox state after a disabled skill leaves the filtered tab.

## Best Possible Solution

Land the keyed-row rendering change with its regression test once normal CI and maintainer review are green.

## Reproduction Assessment

Yes, by source and supplied recording rather than a locally run repro: current main uses positional row rendering around a mutable checkbox, and the before video shows stale visual state transfer after a skill leaves the Disabled tab.

## Solution Assessment

Yes. Keying the row list by `skill.skillKey` is the narrow maintainable fix because Lit's keyed `repeat()` contract prevents DOM reuse across different skills while the existing controller already reloads canonical skill status.
