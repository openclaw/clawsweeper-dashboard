## Summary

Keep open; the underlying legacy timestamp problem is still source-provable, but this branch is not mergeable because it patches the wrong cron load boundary for current main and mixes in unrelated secret-handling changes.

## What This Changes

The PR adds legacy cron timestamp normalization during service store load, expands a cron regression test, and refreshes iOS Fastlane secret-scanner metadata.

## Best Possible Solution

Rework this as a narrow cron-store fix in src/cron/store.ts that synthesizes finite non-negative missing timestamps through the split-store loader, verifies sidecar-aware persistence, and drops unrelated secret-handling edits.

## Reproduction Assessment

Yes. By source inspection, a legacy jobs.json entry missing createdAtMs can pass through loadCronStore(), cron.list returns loaded jobs, and both the protocol schema and Swift model require numeric timestamp fields.

## Solution Assessment

No. Load-time repair is the right direction, but it should live in the canonical split-store loader with sidecar-aware tests and no unrelated Fastlane or secret baseline changes.
