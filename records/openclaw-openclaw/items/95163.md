## What This Changes

The PR adds `/status` and `/compact` cases to the TUI command handler so local mode prints an unsupported-command system message while gateway mode still forwards the raw command.

## Best Possible Solution

Align local TUI autocomplete, help, and dispatch so advertised slash commands are handled, explicitly rejected, or safely routed before embedded send; if maintainers accept the narrow mitigation, keep the broader command-boundary follow-up explicit.

## Reproduction Assessment

Yes from source, though I did not run the local PTY repro in this read-only pass. Slash input enters `handleCommand`, advertised shared commands without local cases hit `sendMessage(raw)`, and local embedded send passes the text to `agentCommandFromIngress`.

## Solution Assessment

No. The two-command guard is a plausible mitigation for the named commands, but the best fix is a registry-aware local command boundary or an explicit maintainer decision to keep the remaining advertised-command invariant as separate work.
