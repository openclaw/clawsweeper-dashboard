## Best Possible Solution

Align local TUI autocomplete, help, and dispatch so every advertised slash command is handled, safely routed, or explicitly rejected before embedded send, then prove it with focused unit coverage and local TUI proof.

## Solution Assessment

No. The two-command guard is a plausible mitigation for the named commands, but the best fix is a registry-aware local command boundary or a maintainer-approved narrower scope with explicit follow-up.
