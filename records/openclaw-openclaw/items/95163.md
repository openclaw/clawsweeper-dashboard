## What This Changes

The PR adds `/status` and `/compact` cases to the TUI command handler so local mode shows an unsupported-command system line while non-local mode forwards the raw command.

## Best Possible Solution

Make local TUI help, autocomplete, and dispatch agree so every advertised local slash command is handled, safely routed, or explicitly rejected before embedded send, then prove representative commands in the local PTY lane.

## Reproduction Assessment

Yes from source: slash input enters `handleCommand`, advertised unhandled command names hit the default branch, and local embedded send passes the raw text to `agentCommandFromIngress`. I did not run the local PTY repro in this read-only review.

## Solution Assessment

No. The two-command guard is a plausible mitigation for the named symptoms, but the best fix is a registry-aware local command boundary or an explicit maintainer decision to keep the broader invariant tracked separately.
