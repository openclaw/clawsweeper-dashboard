## What This Changes

The PR adds `/status` and `/compact` cases to `src/tui/tui-command-handlers.ts` so local mode shows an unsupported-command system message while gateway mode still forwards the raw command.

## Best Possible Solution

Align local TUI autocomplete, help, and dispatch so every advertised slash command is handled, explicitly rejected, or safely routed before embedded send; if maintainers accept the narrow mitigation, keep the broader command-boundary follow-up explicit.

## Reproduction Assessment

## Solution Assessment

No. The two-command guard is a plausible mitigation for the named commands, but the best fix is a registry-aware local command boundary or a maintainer-approved narrower scope with explicit follow-up.
