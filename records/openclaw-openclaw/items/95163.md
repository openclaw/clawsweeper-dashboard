## What This Changes

This PR adds `/status` and `/compact` cases to `src/tui/tui-command-handlers.ts` so local mode prints an unsupported-command message and gateway mode forwards the raw command.

## Best Possible Solution

Make local TUI help, autocomplete, and dispatch agree so every advertised local slash command is handled, safely routed, or explicitly rejected before embedded send, then prove the local TUI path.

## Reproduction Assessment

Yes from source: slash input enters `handleCommand`, shared commands are advertised in TUI discovery/help, unhandled command names call `sendMessage(raw)`, and the embedded backend passes the message into `agentCommandFromIngress`. I did not run the local PTY repro in this read-only review.

## Solution Assessment

No. The two-command guard is a plausible mitigation for the named commands, but the best fix is a registry-aware local command boundary or an explicit maintainer decision to keep the broader invariant tracked separately.
