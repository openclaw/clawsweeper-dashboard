## Summary

Keep open: current main still has the Feishu/Lark plain-text slash-command delivery gap. Feishu builds authorized command context, but does not mark the turn as a text command before Codex/GPT message-tool-only delivery defaults can suppress visible source replies.

## What This Changes

Authorized Feishu/Lark slash commands in Codex/GPT-backed sessions dispatch with `delivered=false` and no visible response.

## Best Possible Solution

Represent authorized Feishu/Lark plain-text slash commands as text command turns in the Feishu inbound context, with focused regression coverage, while preserving Codex message-tool defaults for normal turns.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No live credentialed Feishu run was performed, but source inspection shows the reported path on current main: authorized Feishu slash commands can be finalized as normal message turns without `CommandSource: "text"`, matching the delivered=false logs.

## Solution Assessment

Is this the best way to solve the issue? Yes, the narrow maintainable fix is to set the existing text-command contract in Feishu inbound command context; changing Codex defaults or global reply policy would be broader than needed.
