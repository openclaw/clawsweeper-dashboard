## Summary

The broader native Codex Feishu direct-session final-reply/HTTP 400 problem remains tracked separately in #78689.

## What This Changes

The issue reports Feishu/Lark slash commands in Codex/GPT-backed sessions being authorized but completing with no visible delivered reply.

## Best Possible Solution

Close this issue against the shipped #86874 fix, and keep any remaining native Codex Feishu direct-session final-reply failures on #78689.

## Reproduction Assessment

Yes, source-reproducible: the reported Feishu path supplies CommandAuthorized and CommandBody while Codex normal source replies default to message_tool, and current tests cover the formerly missing CommandSource case.

## Solution Assessment

Yes, the shipped fix is the better layer because it derives explicit command status in shared command-turn detection instead of adding a Feishu-only fast path or weakening Codex defaults.
