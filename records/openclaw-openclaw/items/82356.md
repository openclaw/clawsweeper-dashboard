## Summary

Keep open: the report is plausible and current main still has a real boundary between native command fast-path handling and plain-text command dispatch, but the exact Feishu/Lark external-plugin plus Codex/GPT delivered=false path was not reproduced locally and is not safe to close or auto-fix from core alone.

## What This Changes

Authorized Feishu/Lark slash commands in Codex/GPT-backed sessions should deliver visible command responses instead of completing with delivered=false.

## Best Possible Solution

Align Feishu/Lark slash-command dispatch with the existing native command contract, or add a narrow SDK/core compatibility path that preserves visible command acknowledgements for Codex/GPT sessions without widening command authorization.

## Reproduction Assessment

No, not end-to-end. The source shows why plain-text dispatch does not enter the native command fast path, and the reporter supplied strong runtime logs, but the external Lark plugin and live Feishu/Codex path were not reproduced here.

## Solution Assessment

Unclear. Forcing affected commands onto the native fast path matches the workaround, but maintainers should decide whether the durable fix belongs in the external Lark plugin, the plugin SDK bridge, or core dispatch semantics.
