## Summary

The report should stay open because current source still has a plausible Feishu/Lark plain-text slash-command delivery gap: Feishu records authorization and command body, but it does not mark the turn as a text command before the shared source-reply policy applies Codex/GPT message-tool-only suppression.

## What This Changes

The issue requests restoration of visible Feishu/Lark slash-command replies for Codex/GPT-backed sessions after authorized commands dispatch with `delivered=false`.

## Best Possible Solution

Represent authorized Feishu/Lark plain-text slash commands as text command turns so their acknowledgements bypass Codex message-tool-only suppression while preserving Codex defaults for normal turns.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No live Feishu credentialed reproduction was run, but the source path is clear: an authorized Feishu slash message can reach dispatch without `CommandSource: "text"`, so Codex/GPT message-tool-only defaults can suppress visible delivery.

## Solution Assessment

Is this the best way to solve the issue? Yes, the narrowest maintainable fix is to set the existing text-command contract in the Feishu inbound context instead of changing Codex defaults or global source-reply policy.
