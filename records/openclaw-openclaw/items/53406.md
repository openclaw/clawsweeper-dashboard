## Summary

Keep open: current main and v2026.6.10 still have streaming-only active steering gates and same-session admission/Discord queue paths that can keep chat input from reaching a running ACP or sub-agent session, while the related fixes are open or closed-unmerged rather than implemented.

## What This Changes

Request interactive Discord and other chat-thread messages to steer running ACP or sub-agent sessions instead of waiting until the current turn finishes.

## Best Possible Solution

Keep this issue open as the ACP/sub-agent chat-surface tracker while maintainers land or replace a coordinated lifecycle-aware steering fix plus same-session admission path.

## Reproduction Assessment

Yes, at source level: current main still gates active steering on token streaming and rejects active non-streaming handles before chat input can reliably steer a running session. I did not run a live Discord ACP session in this read-only review.

## Solution Assessment

Unclear as a final implementation path. Lifecycle-aware active/stopped steering plus same-session admission is the right shape, but maintainers still need to combine those fixes across ACP, sub-agent, and channel surfaces without changing ordering guarantees unexpectedly.
