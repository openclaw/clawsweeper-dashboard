## Label Justifications

- `P2`: The issue affects important active agent steering workflows, but it is a bounded feature/product gap rather than an emergency outage.
- `impact:session-state`: The requested behavior changes how input enters already-running ACP and sub-agent sessions without losing their progress.

## Summary

Keep open: current main still has adjacent normal-session steering, but it does not provide same-turn live input into running ACP turns, and controlled sub-agent steering remains abort-and-restart rather than in-place input.

## What This Changes

The issue requests live input steering for running ACP and sub-agent sessions from Discord threads and other chat surfaces.

## Best Possible Solution

Define a generic, permission-aware active-run input contract for ACP and controlled sub-agents, then wire authorized chat-thread messages into it without bypassing queue policy or ownership checks.

## Reproduction Assessment

Yes, by source inspection: Discord messages remain per-session queued before processing, ACP steer uses the serialized ACP turn path, and controlled sub-agent steering restarts the run. I did not run a live Discord or ACP harness scenario in this read-only review.

## Solution Assessment

Not applicable as a patch. The best path is a maintainer-approved live-input contract across ACP manager, channel ingress, and sub-agent control rather than a one-off Discord queue bypass.
