## Summary

Keep open: current main and the latest release still lack an ACP active-turn input primitive, while controlled subagent steering is explicitly abort-and-restart rather than same-turn input. The open main-session steering work at https://github.com/openclaw/openclaw/pull/73365 is adjacent but does not settle the ACP runtime contract or controlled subagent semantics requested here.

## What This Changes

The issue requests live same-turn input for running ACP and controlled subagent sessions from Discord and other chat surfaces.

## Best Possible Solution

Add a backend-advertised ACP live-input or turn-steer capability with documented fallback behavior, then explicitly decide whether controlled subagent steering should remain restart-based or become same-turn input.

## Reproduction Assessment

Yes, source-level. Start one ACP runTurn and submit a bound reply or /acp steer for the same session; current main serializes that second runTurn behind the active turn, while controlled subagent steer is implemented as abort-and-restart rather than same-turn input.

## Solution Assessment

Unclear until maintainers choose the product/API boundary. The maintainable path is capability-gated ACP live input with documented fallback semantics, plus an explicit decision on controlled subagent live input versus restart steering.
