## Summary

Keep open: current main still lacks an ACP active-turn input primitive and still implements sub-agent steering as abort-and-restart, while the related open PRs cover adjacent queueing or restart details rather than the full live-input contract requested here.

## What This Changes

The issue requests live input/steering for running ACP and sub-agent sessions from Discord and other chat surfaces.

## Best Possible Solution

Define a backend-advertised ACP live-input capability with documented fallback behavior, then explicitly decide whether native sub-agent chat steering should become same-turn input or remain restart-based.

## Reproduction Assessment

Yes, source-level: start one ACP runTurn and submit a bound reply or /acp steer for the same session; current main serializes that second turn until the active one exits. Native sub-agent steering is also source-visible as abort-and-restart rather than same-turn input.

## Solution Assessment

Unclear until maintainers choose the product/API boundary. The maintainable path is capability-gated ACP live input with documented fallback semantics, plus an explicit decision on sub-agent live input versus restart steering.
