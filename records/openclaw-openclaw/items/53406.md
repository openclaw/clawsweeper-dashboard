## Summary

Keep open: current main still lacks an ACP active-turn input primitive, and controlled subagent steering is implemented as abort-and-restart rather than same-turn input. The already-closed ACP action issue covers basic `/acp steer` and control commands, but this request is specifically about live input while the run is still active.

## What This Changes

Request ACP and controlled subagent messages from chat threads reach the running session mid-turn instead of waiting for a later turn or restarting the child run.

## Best Possible Solution

Define a backend-advertised ACP live-input or turn-steer capability with documented fallback behavior, then explicitly decide whether controlled subagent steering should remain restart-based or become same-turn input.

## Reproduction Assessment

Yes, source-level. Start one ACP runTurn and submit a bound reply or `/acp steer` for the same session; current main serializes that second turn behind the active turn, while controlled subagent steer aborts and replaces the run.

## Solution Assessment

Unclear until maintainers choose the product/API boundary. The maintainable path is capability-gated ACP live input with fallback semantics, plus an explicit decision on controlled subagent live input versus restart steering.
