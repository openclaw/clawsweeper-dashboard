## Summary

Keep open: current main has the shared steering runtime and docs, but Discord still serializes same-session inbound jobs before the second message can reach that runtime, so the central Discord ACP/sub-agent steering gap remains.

## What This Changes

The issue requests live input steering for running ACP and sub-agent sessions from Discord threads and other chat surfaces.

## Best Possible Solution

Move same-session Discord steer-mode admission ahead of serialization, or into a shared channel queue seam, while preserving non-steer serialization, replay dedupe, typing cleanup, and delivery guarantees.

## Reproduction Assessment

Yes. Source inspection shows Discord same-session inbound jobs are serialized by session key before the shared active-run steering path can process the second message.

## Solution Assessment

Unclear. Live steering is the right end state, but the best implementation needs maintainer choice between a Discord steer bypass and a shared channel/runtime admission seam.
