## Summary

Keep open for maintainer review: current main still lacks the proposed pricing/context split and config toggle, but this PR head is stale and would regress current bootstrap auth, fresh context handling, CSS wrapping, and config-doc baseline workflow. The closed duplicate #48898 and merged #71297 context-freshness work are useful context, but neither implements this threshold behavior on main.

## What This Changes

The PR adds `gateway.controlUi.chat.showPricingThreshold`, wires it through Gateway bootstrap and Control UI state, and changes chat context notices/styles/tests to show pricing-threshold hints separately from model context limits.

## Best Possible Solution

Refresh or replace the branch on current main, add the chat setting inside the authorized bootstrap response, implement the threshold split in the current `ui/src/ui/chat/context-notice.ts` model, use a fresh request-context source for pricing hints, and update only the current config baseline hash artifact.

## Reproduction Assessment

Yes. The auth regression is reproducible by comparing PR-head `src/gateway/control-ui.ts:347-376` with current main `src/gateway/control-ui.ts:759-808`; the pricing false positive follows from a session with high cumulative `inputTokens` and lower fresh `totalTokens`.

## Solution Assessment

No. The direction is useful, but this branch is not the best current solution because it applies the behavior to stale Gateway/UI paths, bases a billing-threshold claim on aggregate session input, and carries obsolete generated docs artifacts.
