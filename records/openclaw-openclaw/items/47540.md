## Summary

Keep open: current main still does not implement the pricing-threshold split, and https://github.com/openclaw/openclaw/pull/71297 explicitly left this threshold behavior out of scope. The PR is not mergeable as-is because it is stale against current Control UI auth/context code, uses aggregate input tokens for a per-context warning, has a CSS cascade regression, and lacks real behavior proof.

## What This Changes

The PR adds `gateway.controlUi.chat.showPricingThreshold`, gateway/bootstrap/UI state plumbing, and Control UI context-notice rendering/tests for separate pricing-threshold and model-context messages.

## Best Possible Solution

Rebase or replace the branch on current main, extend the current `ui/src/ui/chat/context-notice.ts` model, preserve https://github.com/openclaw/openclaw/pull/70247 and https://github.com/openclaw/openclaw/pull/71297 behavior, and define a provider-owned or fresh-context source for pricing threshold notices.

Yes for the review findings: source inspection of PR head against current main shows the unauthenticated bootstrap path, aggregate-token pricing check, and CSS cascade conflict. I did not run tests because this was a read-only review.

## Solution Assessment

No, not as-is. The requested UX may be useful, but the implementation needs a current-main rebase, a clearer pricing-threshold source of truth, and real Control UI proof before it is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve bootstrap auth when adding chat settings:** `src/gateway/control-ui.ts:370-375`
  - body: Current main requires `authorizeControlUiReadRequest` before serving `/__openclaw/control-ui-config.json`, but this PR adds the chat setting on a stale handler that sends bootstrap JSON without that gate. Rebase the field into the authorized response path so auth-enabled gateways do not expose bootstrap metadata again.
  - confidence: 0.9
- **[P2] Base threshold warnings on fresh request context:** `ui/src/ui/views/chat.ts:353-356`
  - body: `shouldShowPricing` is driven by cumulative `session.inputTokens`, which can include cached input, retries, or tool-loop calls across a run. That can show a higher-rate billing warning even when no fresh prompt/context snapshot crossed the provider tier, so use a fresh context/request usage value or provider-owned billing signal.
  - confidence: 0.84
- **[P2] Prevent stacked notice CSS from being overridden:** `ui/src/styles/components.css:1401-1408`
  - body: The new stacked rule is loaded from `components.css`, but PR-head `styles.css` imports `chat.css` afterward and `chat/layout.css` still has a later plain `.context-notice` rule with equal specificity. The later rule can restore single-line pill layout, so scope the legacy rule or raise/move the stacked selector.
  - confidence: 0.82
