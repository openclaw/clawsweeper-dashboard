## What This Changes

The PR adds `gateway.controlUi.chat.showPricingThreshold`, wires it through Gateway bootstrap and Control UI state, and changes chat context notice rendering/styles/tests to separate pricing-threshold hints from model context-limit notices.

## Best Possible Solution

Rebuild the threshold split on the current `ui/src/ui/chat/context-notice.ts` model and authenticated bootstrap contract, keep pricing hints behind a clearly defined fresh/provider-owned usage signal, and attach redacted real Control UI proof before merge.

## Solution Assessment

Scope the later legacy rule or raise the stacked selector specificity so long notices can wrap.
