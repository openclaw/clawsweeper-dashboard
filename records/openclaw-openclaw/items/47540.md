## Summary

Keep open: current main still does not implement the pricing-threshold split or `gateway.controlUi.chat.showPricingThreshold`, and the merged freshness work explicitly left this threshold PR outside its scope. The current PR head is not mergeable as-is because it is stale against the authenticated bootstrap contract, uses cumulative token usage for pricing decisions, has a CSS cascade regression, and still lacks real behavior proof.

## What This Changes

The PR adds a Control UI pricing-threshold notice setting, wires it through gateway/bootstrap/UI state, rewrites chat context notice rendering/styles/tests, and updates config schema baselines.

## Best Possible Solution

Rebuild the threshold split on current main's `ui/src/ui/chat/context-notice.ts` and authenticated bootstrap contract, preserve existing bootstrap fields, use a fresh/provider-owned usage signal for pricing hints, and attach redacted real Control UI proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: PR head serves bootstrap config without the current auth gate, uses cumulative `inputTokens` for pricing-threshold decisions, and has a later CSS rule that overrides the stacked notice. I did not run tests because this review is read-only.

## Solution Assessment

No, not as-is. The direction may be useful, but the patch should be rebased or rebuilt on the current bootstrap-auth and context-freshness contracts before maintainers decide the pricing-threshold semantics.
