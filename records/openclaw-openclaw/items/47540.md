- **[P1] Preserve bootstrap auth when adding chat settings:** `src/gateway/control-ui.ts:370-375`
  - body: Current main authorizes `/__openclaw/control-ui-config.json` before returning bootstrap metadata, but this branch adds the chat setting on the older unauthenticated handler shape.   - confidence: 0.9
- **[P2] Base threshold notices on fresh request context:** `ui/src/ui/views/chat.ts:353-356`
  - body: `shouldShowPricing` is driven by cumulative `session.inputTokens`, while the notice says the higher-rate billing threshold was crossed. That can warn after aggregate/tool-loop usage even when no fresh request context crossed the tier, so use fresh context/request usage or a provider-owned billing signal.
  - confidence: 0.84
- **[P2] Prevent stacked notice CSS from being overridden:** `ui/src/styles/components.css:1401-1408`
  - body: The new stacked rule has the same specificity as the later `.context-notice` rule from `chat/layout.css` on this branch, so `display: inline-flex` and `white-space: nowrap` win in the cascade. Exclude the stacked variant, raise specificity, or move the rule so long notices wrap as intended.
  - confidence: 0.86
- **[P3] Drop obsolete generated config baselines:** `docs/.generated/config-baseline.jsonl:1`
  - body: Current main tracks config docs drift with `docs/.generated/config-baseline.sha256`, but this branch still modifies the old JSON/JSONL baseline files. Refresh against the current hash-baseline workflow instead of reintroducing generated artifacts that will drift and conflict.
  - confidence: 0.78
