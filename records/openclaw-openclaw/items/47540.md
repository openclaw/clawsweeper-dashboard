- **[P1] Preserve bootstrap auth when adding chat settings:** `src/gateway/control-ui.ts:370-375`
  - body: Current main authorizes `/__openclaw/control-ui-config.json` before returning bootstrap metadata, but this branch adds the chat setting on a stale handler that returns the payload without that gate.   - confidence: 0.9
- **[P2] Base threshold warnings on fresh context:** `ui/src/ui/views/chat.ts:353-356`
  - body: `shouldShowPricing` is driven by cumulative `session.inputTokens`, while current session state separates fresh context totals with `totalTokens` and `totalTokensFresh`. This can show a higher-rate billing warning from aggregate run usage rather than a fresh request/context or provider-owned billing signal.
  - confidence: 0.84

  - body: The new stacked rule is in `components.css`, but `chat.css` is imported later and still defines a plain `.context-notice` rule with equal specificity, including `display: inline-flex` and `white-space: nowrap`. Scope the later legacy rule or increase the stacked selector specificity so long stacked notices can wrap.
  - confidence: 0.78
