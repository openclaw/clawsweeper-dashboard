## Summary

Keep open: current main still lacks the pricing-threshold split and toggle, but this PR is stale against current Control UI/Gateway behavior and should not merge as-is because it reopens the bootstrap auth boundary, uses aggregate token data for billing hints, has a CSS cascade regression, and lacks real behavior proof.

## What This Changes

The PR adds `gateway.controlUi.chat.showPricingThreshold`, passes it through the Gateway bootstrap payload, and changes Control UI chat context notices/styles/tests to show pricing-threshold hints separately from model context usage.

## Best Possible Solution

Rebuild this on current main by extending the existing `ui/src/ui/chat/context-notice.ts` model, adding the bootstrap/config field inside the authorized Gateway path, sourcing threshold truth from a maintained provider/request contract, refreshing only current generated artifacts, and attaching real Control UI proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve bootstrap auth when adding chat settings:** `src/gateway/control-ui.ts:370-375`
  - body: Current main requires Control UI read auth before serving `/__openclaw/control-ui-config.json`, but this branch adds the chat setting on a stale handler that returns the bootstrap payload without that gate. Rebase onto the current handler and add the field inside the already-authorized response path, otherwise auth-enabled gateways expose bootstrap metadata again.
  - confidence: 0.9
- **[P2] Base threshold warnings on fresh request context:** `ui/src/ui/views/chat.ts:353-356`
  - body: `shouldShowPricing` is driven by `inputUsed`, which comes from cumulative `session.inputTokens` and can include multiple calls, retries, and tool-loop usage. That can show “Higher-rate billing threshold crossed” when no fresh request/context snapshot crossed the provider tier, so use a fresh request/context value or provider-owned billing signal instead.
  - confidence: 0.84
- **[P2] Keep stacked notice styles out of the later pill rule:** `ui/src/styles/components.css:1401-1408`
  - body: The new stacked notice rule is loaded from `components.css`, but `chat/layout.css` is imported later and still has a plain `.context-notice` rule with `display: inline-flex` and `white-space: nowrap`. Since these selectors have equal specificity for the overlapping properties, long stacked notices can collapse back into the old single-line pill; exclude the stacked variant or move/increase the stacked rule.
  - confidence: 0.86
- **[P3] Drop obsolete generated config baselines:** `src/config/schema.base.generated.ts:9931-9938`
  - body: Current main tracks the config docs baseline by SHA file only and no longer tracks the full JSON/JSONL baseline artifacts or `src/config/schema.base.generated.ts`. Refresh the config docs output on current main instead of reintroducing the old generated baseline surfaces.
  - confidence: 0.78
