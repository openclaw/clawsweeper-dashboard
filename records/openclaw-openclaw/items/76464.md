## Summary

Keep this issue open. Current main still omits `systemPrompt` from the local `model.run` simple-completion context and still collapses empty provider error results into a generic no-text error; two open closing PRs already target the bug, so the issue should wait for one to land.

## What This Changes

Make local `openclaw infer model run --model openai-codex/gpt-5.5` send instructions and surface provider error details instead of returning a generic empty-output failure.

## Best Possible Solution

Review and land one of the open fix PRs, or an equivalent narrow CLI patch, adding a default local model-run `systemPrompt` and propagating provider error details when no text is returned.

## Reproduction Assessment

Yes. Current main still sends only `context.messages` through the local simple-completion path while Codex Responses derives `instructions` from `context.systemPrompt`; the reporter supplied a live command that reaches the same backend rejection.

## Solution Assessment

Yes. The narrow maintainable fix is in the CLI local model-run context and empty-output error handling, while #76470 and #76488 are implementation candidates that should be reviewed rather than closing the issue early.
