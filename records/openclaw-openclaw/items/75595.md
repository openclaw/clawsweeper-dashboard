## Summary

Keep this PR open: it targets real current-main Gateway event-loop blockers and includes useful sessions-list and checkpoint-copy improvements, but it is not merge-ready because the queued engine-owned compaction path still performs a full synchronous transcript open before the new async checkpoint helper can run.

## What This Changes

The PR adds an async/yielded `sessions.list` store listing path, an async compaction checkpoint snapshot helper, switches Gateway and compaction callers toward those paths, and adds tests plus a changelog entry.

## Best Possible Solution

Keep the sessions-list yielding and async checkpoint-copy direction, then revise queued engine-owned compaction so checkpoint metadata is obtained without `SessionManager.open()` or another full synchronous transcript read before the async work starts.

## Reproduction Assessment

Yes. A high-confidence reproduction path is a Gateway `sessions.list` request with derived titles/previews over a large session store, plus engine-owned compaction over a large transcript; source inspection and the pinned dependency implementation show the synchronous calls that block the event loop.

## Solution Assessment

No. The proposed direction is mostly right, but the current PR is not the best complete fix because one changed compaction caller still performs the same full synchronous transcript open before reaching the async checkpoint helper.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Avoid synchronous session opens in queued checkpoint capture:** `src/agents/pi-embedded-runner/compact.queued.ts:118-120`
  - body: The queued engine-owned compaction path now awaits the async checkpoint helper, but it still builds the argument with `SessionManager.open(params.sessionFile)`. In `@mariozechner/pi-coding-agent@0.70.6`, `open()` synchronously reads the whole JSONL transcript and construction reads it again, so large transcripts can still block the Gateway event loop before the async copy starts. Pass/read only bounded checkpoint metadata asynchronously instead.
  - confidence: 0.89
