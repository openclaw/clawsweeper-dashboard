## Summary

Keep this PR open. It is protected by the `maintainer` label and is currently dirty against the base, and the latest patch still leaves one compaction hot path doing a full synchronous transcript open before the new async checkpoint helper can run.

## What This Changes

The PR adds a yielded async `sessions.list` path, an async compaction checkpoint snapshot helper, tests for those paths, and a changelog entry for Gateway event-loop responsiveness.

## Best Possible Solution

Keep the useful sessions-list yielding changes, tests, and changelog entry, then revise the compaction checkpoint path so queued engine-owned compaction obtains the pre-compaction session id and leaf id without `SessionManager.open()` or another full synchronous transcript read. After that, rebase and run the focused Gateway/compaction tests plus the changed gate.

## Reproduction Assessment

Yes. A high-confidence reproduction path is a Gateway `sessions.list` request with derived titles/previews over a large store, and an engine-owned compaction over a large transcript; current main and PR-head code show the synchronous calls that match the linked reports.

## Solution Assessment

No. The sessions-list change and async copy helper are directionally good, but this is not yet the best fix because one compaction caller still performs the same full synchronous transcript open the PR is trying to remove from the hot path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Avoid synchronous session opens in queued checkpoint capture:** `src/agents/pi-embedded-runner/compact.queued.ts:118-120`
  - body: The queued engine-owned compaction path now awaits the async checkpoint helper, but it still builds the `sessionManager` argument with `SessionManager.open(params.sessionFile)`. In the pinned `@mariozechner/pi-coding-agent@0.70.6`, `open()` synchronously reads the whole JSONL transcript and construction reads it again, so a large transcript can still block the Gateway event loop before the async copy starts. Read only the needed checkpoint metadata with bounded async I/O, or add/use a dependency API that avoids a full synchronous open here.
  - confidence: 0.89
