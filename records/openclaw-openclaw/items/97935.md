## Summary

Keep open: current main and v2026.6.10 still route Skill Workshop proposal discovery and lifecycle through the resolved agent workspace, while fleet-wide cross-agent proposal visibility remains a product and security-boundary decision.

## What This Changes

The issue requests shared, session-agnostic Skill Workshop proposal visibility so one agent can create a proposal and another agent can list, inspect, review, or apply it.

## Best Possible Solution

Maintainers should decide the canonical Skill Workshop visibility and authorization model, likely alongside https://github.com/openclaw/openclaw/issues/74601, then update service filtering, Gateway RPCs, model-tool behavior, docs, and tests around that model.
