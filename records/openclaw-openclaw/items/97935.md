## Summary

Keep open: current main still scopes Skill Workshop proposal list/inspect/lifecycle calls to the resolved workspace, while this report asks for cross-agent proposal visibility that remains a product and security boundary decision.

## What This Changes

The issue requests making `skill_workshop` proposal discovery and inspection session-agnostic across agent sessions on the same Gateway host.

## Best Possible Solution

Decide whether Skill Workshop governance should be host-wide or workspace-scoped, ideally alongside https://github.com/openclaw/openclaw/issues/74601, then update service filtering, Gateway RPCs, tool behavior, docs, and tests around one canonical visibility model.

Yes, source-level: current main filters proposal list and inspect results by resolved workspace, and tests assert cross-workspace proposals are hidden. I did not run a live multi-agent repro because this review is read-only.

## Solution Assessment

Unclear. Removing the workspace filter is plausible, but it would reverse the current selected-workspace contract and needs maintainer direction on host-wide proposal governance.
