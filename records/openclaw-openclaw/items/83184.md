Keep open: current main still has the source-reproducible heartbeat success gap, and the issue is already linked to an open closing PR that should be reviewed or landed before this issue closes. ## What This Changes

The issue asks OpenClaw to clear pending final-delivery state after a successful substantive heartbeat send so later heartbeats invoke the agent normally.

## Best Possible Solution

Review and land the narrow cleanup in https://github.com/openclaw/openclaw/pull/83187, with regression coverage and real behavior proof or an explicit maintainer waiver, then close this issue through the merged PR.

## Reproduction Assessment

Yes. Source inspection shows a substantive heartbeat reply can be saved as pending delivery, successfully sent, and then left uncleared so the next heartbeat enters the pending redelivery path before normal agent work.

## Solution Assessment

Yes. Clearing the existing pending-final-delivery fields in the heartbeat send-success branch mirrors the established dispatch success contract and is narrower than changing pending capture or heartbeat replay policy.
