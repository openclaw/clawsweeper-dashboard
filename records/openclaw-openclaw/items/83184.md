## What This Changes

The issue reports heartbeat-driven substantive replies leaving `pendingFinalDelivery*` state uncleared after successful delivery, which can block later heartbeat runs.

## Best Possible Solution

Review and land the focused linked fix PR, or close that PR with a clear alternative before closing this issue.

## Reproduction Assessment

Yes. Source inspection shows a substantive heartbeat reply can be persisted as pending delivery, the heartbeat success path sends it while recording only last-heartbeat metadata, and the next heartbeat can hit the pending-delivery replay path instead of normal agent work.

## Solution Assessment

Yes. Clearing the existing pending-final-delivery fields in the heartbeat send-success branch mirrors the established final-dispatch cleanup and keeps failed-send retry behavior intact.
