Full review comments:

- **[P1] Keep fallback diary output free of raw snippets:** `extensions/memory-core/src/dreaming-narrative.ts:149-150`
  - body: This fallback now builds diary prose from `data.snippets` and promotions. Current main intentionally writes a generic placeholder on fallback because those raw fragments can contain session metadata, summaries, or operational logs, so this would reintroduce a Dream Diary leak on timeout/empty/request-scoped fallback paths.
  - confidence: 0.9
- **[P1] Do not let transcript text suppress the whole session:** `src/memory-host-sdk/host/session-files.ts:431-435`
  - body: Setting `generatedBySystemAutomation` from a heartbeat-looking record causes the rest of the transcript to be skipped. A user can type the same heartbeat prompt/token shape in an ordinary session and suppress later real content from memory ingestion; current main sanitizes generated prompt/ack messages individually instead.
  - confidence: 0.86
- **[P2] Preserve italic diary body lines in fingerprints:** `extensions/memory-core/src/dreaming-narrative.ts:390-391`
  - body: The fingerprint loop now skips every line wrapped in `*...*`, not only the date line. A diary entry with an italic standalone sentence can collapse to the same fingerprint as another entry, so append/dedupe can silently skip or delete valid diary content.
  - confidence: 0.82
