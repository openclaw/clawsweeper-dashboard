## Summary

Keep open: the branch addresses real memory-core Dream Diary problems that are still present on current main, but it should not merge as-is because the new body-only fingerprint drops valid italic prose and the external PR lacks required after-fix real behavior proof.

## What This Changes

The branch changes memory-core dreaming narrative and phase code/tests to strip nested managed daily dreaming blocks, render request-scoped fallback snippets as diary prose, and dedupe Dream Diary entries by body.

## Best Possible Solution

Refresh or replace the branch with the useful strip/fallback/dedupe hardening, make body dedupe ignore only the diary timestamp while preserving italic prose, and coordinate the fallback split with https://github.com/openclaw/openclaw/pull/70523.

## Reproduction Assessment

Yes for the blocking PR regression from source inspection: PR head skips every standalone italic line before fingerprint comparison, so entries that differ only by italic prose can share a fingerprint. I did not execute tests because this review had to keep the checkout read-only.

## Solution Assessment

No, not as currently written. The heading-aware strip and fallback-prose direction are useful, but the dedupe implementation should ignore only the timestamp line, keep all body prose in the fingerprint, and add a regression for italic body text.
