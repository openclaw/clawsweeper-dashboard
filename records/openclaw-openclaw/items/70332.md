## Summary

Keep this PR open: it still targets a real memory-core Dream Diary bug family and is the canonical lane for the nested-heading stripping report, but the current branch has blocking correctness issues, missing real behavior proof, and conflicts with current main’s moved memory-host path.

## What This Changes

The PR changes memory-core Dream Diary stripping, request-scoped fallback prose, diary dedupe, contaminated-snippet filtering, tests, and memory-host session parsing.

## Best Possible Solution

Refresh or replace this with a narrow memory-core fix that keeps the useful strip/fallback/dedupe hardening, ports any session filtering to the current package path, preserves italic diary prose, normalizes real REM theme fragments, and includes redacted real behavior proof.

## Reproduction Assessment

Yes, for the PR regressions by source inspection: the duplicate fingerprint drops italic body lines, the REM fallback helper receives caller-shaped theme strings it does not normalize, and the heartbeat classifier can be triggered by user-authored sentinel text. I did not run tests because the checkout had to remain read-only.

## Solution Assessment

No, not as currently written. The direction is maintainable, but the patch should ignore only the generated timestamp line, normalize REM theme fragments from the real caller shape, avoid spoofable whole-transcript heartbeat classification, port session filtering to the current package path, and add real behavior proof.

- **[P2] Preserve italic diary body lines:** `extensions/memory-core/src/dreaming-narrative.ts:390-392`
  - body: The new fingerprint skips every standalone `*...*` line, not just the generated diary timestamp. If a generated entry contains an italic standalone sentence, that text disappears from the fingerprint, so distinct diary entries can be skipped or removed as duplicates; only exclude the timestamp line and add a regression.
  - confidence: 0.93

Request-scoped REM fallbacks can still expose raw `Theme:` and backticked metadata; pass plain tags or strip that wrapper and cover the real caller shape.
  - confidence: 0.82
- **[P3] Do not classify user heartbeat text as system automation:** `src/memory-host-sdk/host/session-files.ts:428-433`
  - body: `isHeartbeatPollRecord` treats any user message exactly equal to `[OpenClaw heartbeat poll]` as proof that the whole transcript is system-managed, after which later normal content is skipped. A real user can quote that sentinel; use session-store/runtime-context provenance for whole-transcript classification or drop only the generated heartbeat line.
  - confidence: 0.76
