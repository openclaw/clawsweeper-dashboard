## What This Changes

The branch changes memory-core Dream Diary stripping, request-scoped fallback prose, diary dedupe, contaminated-snippet filtering, memory-host session automation filtering, and related regression tests.

## Best Possible Solution

Refresh or replace this with a narrow memory-core fix that keeps the useful strip/fallback/dedupe hardening, ports session filtering to the active package path, preserves user-authored diary/session content, and includes redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection of the PR head shows the duplicate fingerprint drops italic body lines, the fallback theme helper receives caller-shaped REM theme strings it does not normalize, and the heartbeat classifier can be triggered by user-authored sentinel text.

## Solution Assessment

No, not as currently written. The right direction is a refreshed patch that ignores only generated timestamp lines, normalizes the real REM theme shape, avoids whole-transcript heartbeat classification from user text, and ports session filtering to the current package path.
