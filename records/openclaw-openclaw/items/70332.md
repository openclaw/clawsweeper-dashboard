## Summary

Keep open: the PR targets valid memory-core Dream Diary regressions that are still present on current main, but the proposed fingerprint change can drop italic diary body prose and the external PR lacks after-fix real behavior proof.

## What This Changes

The branch changes memory-core dreaming narrative/phases code and tests to strip nested or legacy managed dreaming sections, render request-scoped fallback snippets as diary prose, and dedupe Dream Diary entries by body.

## Best Possible Solution

Land a refreshed branch or replacement that keeps the useful diary hardening, preserves italic body prose while ignoring only the timestamp for body dedupe, coordinates fallback behavior with the open fallback-suppression PR, and adds maintainer-owned release notes during landing.

## Reproduction Assessment

Yes for the blocking PR regression: on PR head, the fingerprint loop skips every standalone italic line before append/dedupe compares bodies, so entries that differ only by italic body prose can collapse. I did not execute the test suite because this review was required to keep the checkout read-only.

## Solution Assessment

No, not as currently written. The heading-aware strip and fallback-prose direction are useful, but the dedupe implementation should ignore only the timestamp line and keep all body prose in the fingerprint.
