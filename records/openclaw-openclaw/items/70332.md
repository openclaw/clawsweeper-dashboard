## Summary

Keep open: the PR still addresses real memory-core dreaming diary bugs on current main, but the current head introduces an over-broad duplicate fingerprint regression and the external contributor has not supplied after-fix real behavior proof.

## What This Changes

The PR changes memory-core dreaming narrative and phase logic/tests to strip nested managed daily dreaming blocks, format request-scoped fallback fragments as diary prose, and dedupe Dream Diary entries by body.

## Best Possible Solution

Refresh or replace the branch with a narrow memory-core fix that keeps the useful strip, fallback, and body-dedupe hardening while preserving italic body prose and coordinating the fallback split with the related open PRs.

## Reproduction Assessment

Yes for the blocking PR regression by source inspection: PR head skips every standalone italic line before fingerprint comparison, so distinct entries that differ only by italic diary prose can collapse. I did not run tests because this review had to keep the checkout read-only.

## Solution Assessment

No, not as currently written. The heading-aware strip and fallback-prose direction are useful, but duplicate fingerprints should ignore only the diary timestamp while preserving all body prose, and the fallback work should be reconciled with the related open PRs.
