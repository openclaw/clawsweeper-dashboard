## Summary

Keep this PR open. Current main has not implemented the useful Dream Diary hardening, but the proposed fingerprint change would incorrectly collapse distinct entries that contain standalone italic body prose; the branch also overlaps the still-open fallback cleanup in #70523 and needs the required changelog entry before merge.

## What This Changes

The branch changes memory-core dreaming narrative/phases code and tests to strip nested or legacy managed dreaming sections, render request-scoped fallback snippets as diary prose, dedupe Dream Diary entries by body, skip duplicate appends, and cover those paths with regressions.

## Best Possible Solution

Land a revised branch or narrow replacement that keeps the useful diary hardening, changes fingerprinting to ignore only the timestamp line while preserving italic body content, adds regression coverage for italic diary body prose, includes the required changelog entry, and coordinates fallback behavior with #70523 rather than duplicating or contradicting it.

## Reproduction Assessment

Yes. Create two Dream Diary blocks with the same non-italic body but different standalone italic body lines, then run the PR's fingerprint/dedupe path; the italic body lines are skipped and the entries can be treated as duplicates.

## Solution Assessment

No, not as currently written. The direction is narrow and maintainable, but the fingerprint implementation should preserve body prose and exclude only the date/timestamp line; the related fallback suppression in #70523 also needs to be reconciled before merge.
