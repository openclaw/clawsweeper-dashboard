## Summary

Keep open: the PR targets real memory-core Dream Diary bugs that are still present on current main, but the current branch should not merge because its body-dedupe fingerprint drops every standalone italic diary line and the external PR has no after-fix real behavior proof.

## What This Changes

This PR changes memory-core dreaming narrative/phases code and tests to strip nested or legacy managed dreaming sections, render request-scoped fallback snippets as diary prose, and dedupe Dream Diary entries by body before appending.

## Best Possible Solution

Refresh or replace the branch so it keeps the useful diary hardening, fixes fingerprinting to ignore only the timestamp line, coordinates with the related fallback PRs, and includes focused tests plus redacted real behavior proof.

## Reproduction Assessment

Yes for the blocking PR regression: source inspection shows the PR skips every standalone italic line before comparing diary fingerprints, so distinct entries that differ only by italic body prose can collapse. I did not run the test suite because this review was required to keep the checkout read-only.

## Solution Assessment

No, not as currently written. The heading-aware strip and fallback-prose direction are useful, but the dedupe implementation should ignore only the timestamp line and preserve all body prose in the fingerprint.
