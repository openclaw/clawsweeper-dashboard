## Summary

Keep this PR open: it targets real memory-core Dream Diary bugs that are still present on current `main`, but the branch is not merge-ready because it lacks required real behavior proof and introduces a duplicate-fingerprinting regression.

## What This Changes

The PR changes memory-core dreaming narrative and phase logic/tests to strip nested managed daily dreaming blocks, rewrite request-scoped fallback fragments as diary prose, and dedupe Dream Diary entries by body.

## Best Possible Solution

Refresh this branch or replace it with a narrower memory-core fix that keeps the useful strip, fallback, and body-dedupe hardening while preserving all diary body prose and coordinating with the related fallback PRs.

## Reproduction Assessment

Yes for the blocking PR regression by source inspection: the proposed fingerprint skips every standalone italic line before comparing diary bodies. I did not run tests because this review had to keep the checkout read-only.

## Solution Assessment

No, not as currently written. The direction is maintainable, but the duplicate fingerprint should ignore only the generated timestamp line while preserving italic body prose, and the fallback cleanup should be reconciled with the narrower related PRs.
