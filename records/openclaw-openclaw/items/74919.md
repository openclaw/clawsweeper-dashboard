## Summary

Keep open. Current main still takes the normal idle-wait path before clearing pending tool results, while this PR's functional change is a narrow abort-only fast path that appears appropriate; the branch needs small process cleanup before merge.

## What This Changes

The PR makes embedded agent abort cleanup pass an immediate timeout to pending tool-result flushing and adds unit coverage that nonpositive timeouts bypass `waitForIdle`.

## Best Possible Solution

Merge the abort-only fast path after adding the required single-line changelog entry, removing the trailing whitespace in the new test, and rerunning the targeted wait-before-flush test plus the changed gate while preserving the normal non-abort idle wait.

## Reproduction Assessment

Yes. On current main, a guarded session manager plus an agent whose `waitForIdle` never resolves reproduces the delay because `flushPendingToolResultsAfterIdle({ clearPendingOnTimeout: true })` waits for the timeout before clearing pending results.

## Solution Assessment

Yes, with process fixes. The proposed immediate-timeout path is the narrow maintainable solution for aborted cleanup because it reuses existing clear-on-timeout behavior only when the run is already interrupted.
