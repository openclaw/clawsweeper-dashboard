## Summary

Keep open. Current main still waits for the idle timeout before clearing pending tool results during cleanup, and the PR is a narrow, plausible fix for abort-time session-lock latency. The code path looks correct, but the branch is missing the required user-facing changelog entry before merge.

## What This Changes

The PR changes the embedded agent cleanup helper and abort cleanup callers to treat abort cleanup as an immediate timeout, plus adds unit coverage that `waitForIdle` is skipped for nonpositive timeouts.

## Best Possible Solution

Merge this abort-only fast path after adding the required changelog entry and confirming the exact branch still passes the targeted wait-before-flush test plus the changed gate; keep the normal non-abort idle wait intact.

Yes, with one process fix.
