## Summary

Keep open. Current main still waits for embedded-agent idle cleanup before releasing the session lock on abort paths, and this PR is a useful narrow fix, but it still needs the required changelog entry and trailing-whitespace cleanup before merge.

## What This Changes

The PR adds an immediate-timeout path for pending tool-result cleanup, passes it during aborted embedded attempts, and covers zero/negative timeout behavior in a helper test.

## Best Possible Solution

Merge the abort cleanup fast path after adding the required Unreleased changelog entry and trimming the whitespace-only test lines, while keeping broader session-lock latency investigation separate.

## Reproduction Assessment

Yes. On current main, a guarded session manager plus an agent whose `waitForIdle` never resolves exercises the wait path before pending results can clear and before the cleanup `finally` releases the session lock.

## Solution Assessment

Yes, with small process fixes. The `timeoutMs <= 0` fast path reuses existing clear-on-timeout semantics for aborted cleanup while preserving the normal idle wait that protects successful tool-result completion.
