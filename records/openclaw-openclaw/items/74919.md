## Summary

Keep open. Current main still waits for the embedded agent idle timeout before clearing pending tool results during abort cleanup, so this PR addresses a real session-lock latency path; the branch is directionally correct but still needs narrow changelog and whitespace cleanup before merge.

## What This Changes

The PR adds an immediate-timeout path for embedded-runner pending-tool-result cleanup, passes that path during aborted attempts, and tests the zero/negative timeout bypass.

## Best Possible Solution

Merge the abort cleanup fast path after adding the required Unreleased changelog entry and trimming the whitespace-only test lines, while keeping any broader session-lock latency work separate.

## Reproduction Assessment

Yes. On current main, a guarded session manager plus an agent whose `waitForIdle` never resolves reproduces the wait path: cleanup cannot clear pending tool results or release the session lock until the timeout path finishes.

## Solution Assessment

Yes, with small process fixes. The proposed `timeoutMs <= 0` fast path reuses existing clear-on-timeout semantics for aborted cleanup while preserving the normal idle wait that protects successful tool-result completion.
