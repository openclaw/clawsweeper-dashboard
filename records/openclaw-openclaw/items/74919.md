## Summary

Keep open. Current main still waits for agent idle before clearing pending tool results during teardown, which can keep the session lock held on interrupted runs. The PR is a narrow abort fast path with targeted coverage, and I did not find a blocking correctness issue in the provided diff.

## What This Changes

The PR adds an abort-only immediate-timeout path for pending tool-result cleanup, wires interrupted embedded-runner teardown to pass `timeoutMs: 0`, and extends the wait-before-flush unit test coverage.

## Best Possible Solution

Merge this abort fast path, or an equivalent maintainer patch, after required CI confirms aborted/interrupted runs skip the idle wait while successful teardown still waits for real tool results to land.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: use a guarded session manager with an agent whose `waitForIdle` never resolves, then call `flushPendingToolResultsAfterIdle` with `clearPendingOnTimeout: true`; teardown waits for the timeout before clearing and releasing the session lock.

## Solution Assessment

Yes. The proposed approach is the narrowest maintainable fix because it reuses the existing timeout cleanup behavior and scopes the zero-timeout path to aborted or interrupted teardown, preserving the normal idle wait.
