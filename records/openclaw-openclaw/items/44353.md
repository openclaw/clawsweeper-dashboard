## Summary

Keep open: current main can continue fallback after unrecognized provider errors when fallback candidates exist, but the reported subagent configuration still loses its `fallbacks` array before runtime fallback resolution.

## What This Changes

The issue requests automatic fallback to configured backup models when a primary provider returns terminal provider errors, including Bedrock inference-profile failures in subagent model configs.

## Best Possible Solution

Preserve subagent-specific fallback arrays through spawn/session resolution and effective fallback selection, while keeping explicit user model overrides strict.

## Reproduction Assessment

Yes, at source level. A config using `agents.defaults.subagents.model: { primary, fallbacks }` selects the primary for the spawned subagent, but the fallback resolver later reads only agent/default model fallbacks, leaving the subagent fallback list unavailable.

## Solution Assessment

Yes. The narrow maintainable fix is to propagate subagent fallback arrays into the automatic subagent run fallback options; broadening provider-error classification would not fix the missing candidate list.
