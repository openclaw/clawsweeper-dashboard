## Summary

Keep open: current main still resolves auto-selected spawned-subagent fallback candidates from normal agent/default model policy instead of the selected `subagents.model.fallbacks` chain; related Bedrock/generic-error and cron fixes do not cover the latest normal spawned-subagent reproduction.

## What This Changes

The issue asks OpenClaw to trigger configured model fallbacks when a primary provider fails, with the remaining path involving `subagents.model.fallbacks` being ignored for normal spawned subagents.

## Best Possible Solution

Thread the selected subagent fallback policy into normal spawned-subagent execution so `runWithModelFallback` receives the same fallback chain that selected the child primary.

## Reproduction Assessment

Yes, at source level. Configure `agents.defaults.subagents.model` or per-agent `subagents.model` with `{ primary, fallbacks }`, spawn a normal subagent without an explicit model override, and current fallback resolution does not read that selected subagent fallback chain.

## Solution Assessment

Yes. The narrow maintainable fix is to carry the already-supported selected subagent fallback list into normal spawned-subagent execution, not to add another config option or broadly change provider-error classification.
