## Summary

Keep open: current main has Bedrock inference-profile discovery and broad fallback improvements, but the exact Bedrock on-demand-throughput assistant error in this report is still not classified as failover-eligible, so the assistant-error path can still terminate without advancing to configured fallbacks.

## What This Changes

The issue asks OpenClaw to try configured fallback models when AWS Bedrock rejects a raw model ID and requires an inference profile ID.

## Best Possible Solution

Add a narrow provider-owned Bedrock classifier for the inference-profile-required rejection as `model_not_found`, then cover both the classifier and fallback advancement with focused regression tests.

## Reproduction Assessment

Yes, source-level: the reported assistant error text is not matched by the current classifiers, and assistant fallback only rotates on classified failover reasons, failover assistant errors, or timeouts. I did not run a live Bedrock request.

## Solution Assessment

No for closing as already implemented. The prior PR’s idea is directionally right, but the current best fix should live in the Bedrock provider failover hook unless a test proves the failing path needs a shared fallback classifier too.
