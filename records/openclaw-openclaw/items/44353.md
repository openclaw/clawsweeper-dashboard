## Summary

Keep this open: current main covers the Bedrock inference-profile discovery and generic fallback-loop parts, but the exact `agents.defaults.subagents.model.fallbacks` config shown in the report still appears not to be carried into spawned subagent runs.

## What This Changes

Fallback should advance to configured backup models when a Bedrock primary fails with provider-level inference-profile errors, including subagent model fallback config.

## Best Possible Solution

Propagate `subagents.model` fallback lists into spawned subagent auto model selections and the fallback runner, while keeping the existing Bedrock discovery and generic fallback behavior.

## Reproduction Assessment

Yes, source-level. The report's `agents.defaults.subagents.model: { primary, fallbacks }` shape is accepted by config types, but current subagent model selection keeps only the primary and `resolveEffectiveModelFallbacks` does not consult `subagents.model.fallbacks`.

## Solution Assessment

No, closing as fully implemented is not the best outcome. The already-shipped Bedrock discovery work should stay, but the remaining fix should narrowly preserve subagent fallback lists and add regression coverage for that config path.
