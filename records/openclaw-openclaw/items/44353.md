## Summary

Keep this open: current main has the generic provider-error fallback loop and Bedrock inference-profile discovery, but the subagent-specific fallback list reported here is still not carried into spawned subagent runs.

## What This Changes

The issue asks for configured model fallbacks to run when a primary provider or subagent model fails, especially for `agents.defaults.subagents.model.fallbacks`.

## Best Possible Solution

Preserve subagent fallback lists through subagent model selection and effective fallback resolution, then add focused regression coverage for default and per-agent subagent fallback config.

## Reproduction Assessment

Yes, source-level: the reported config shape is accepted, but current subagent selection and effective fallback resolution do not carry `subagents.model.fallbacks` into the fallback runner. I did not execute a live provider failure in this read-only review.

## Solution Assessment

No, closing as fully implemented is not the best solution because the shipped Bedrock and generic fallback fixes do not cover subagent fallback propagation. The narrow maintainable fix is to pass the existing subagent fallback config through the current fallback machinery without adding a new config mode.
