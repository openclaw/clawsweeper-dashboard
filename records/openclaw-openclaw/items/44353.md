## Summary

Keep open. Current main has a generic fallback loop that can continue after provider errors when fallback candidates exist, and Bedrock inference-profile discovery is present, but the reported subagent config path still does not propagate `agents.defaults.subagents.model.fallbacks` or `agents.list[].subagents.model.fallbacks` into the runtime fallback candidate list.

## What This Changes

The issue asks for provider-level failures, especially Bedrock model ID or inference-profile errors, to advance through configured model fallbacks instead of failing the session or subagent run.

## Best Possible Solution

Add a subagent-aware fallback resolution path that preserves fallback arrays from `agents.defaults.subagents.model` and per-agent `subagents.model` for automatic native subagent runs, passes them into `runWithModelFallback()`, and adds focused tests for defaults, per-agent overrides, explicit empty fallback arrays, and explicit user model overrides.
