## Summary

Keep open: current main still drops selected `subagents.model.fallbacks` for normal spawned subagent runs, and the Bedrock hook still does not classify the exact inference-profile-required error; the recent cron fix covers a narrower isolated-cron path.

## What This Changes

The issue asks OpenClaw to trigger configured fallback models when a primary provider fails with provider-level errors, including Bedrock model-ID changes and subagent model fallback configuration.

## Best Possible Solution

Carry the selected `subagents.model.fallbacks` into `runWithModelFallback` for normal spawned subagent runs and classify the exact Bedrock inference-profile-required error, without adding a new config surface or changing cron preflight behavior here.

## Reproduction Assessment

Yes, at source level. Configure `agents.defaults.subagents.model` or `agents.list[].subagents.model` with `{ primary, fallbacks }`, spawn a subagent without an explicit user model override, and current `agent-command` resolves fallback candidates from regular agent/default model config instead of the selected subagent model object.

## Solution Assessment

Yes. The narrow maintainable fix is to preserve the already-supported subagent fallback chain for normal spawned subagent runs and add exact provider-error classification for Bedrock, rather than broadening fallback policy or adding new settings.
