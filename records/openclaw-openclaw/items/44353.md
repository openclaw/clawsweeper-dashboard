## Summary

Keep open: current main has the Bedrock inference-profile and generic provider-error fallback improvements, but the reported subagent fallback list is still dropped before spawned subagent runs reach the fallback runner.

## What This Changes

The issue asks for configured fallback model chains to be used when subagent primary models fail with provider-level or runtime errors.

## Best Possible Solution

Preserve the selected subagent model's fallback chain through spawn planning or fallback resolution, with regression coverage for default and per-agent `subagents.model` configs.

## Reproduction Assessment

Yes, source-level. Current main accepts `subagents.model: { primary, fallbacks }`, but the spawn/model-selection path keeps only `primary` and `resolveEffectiveModelFallbacks` never reads the subagent fallback list.

## Solution Assessment

Yes, the best fix is narrow: carry the already-supported subagent fallback config into `fallbacksOverride` for auto-selected spawned subagent runs while keeping explicit user model overrides strict.
