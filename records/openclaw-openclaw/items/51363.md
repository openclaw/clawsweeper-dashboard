## Summary

Keep open. Current main and v2026.5.18 still derive Docker and browser sandbox container names from the sandbox scope key only, and the related fix PR was closed unmerged, so co-hosted HOME-isolated instances using the default agent can still collide.

## What This Changes

The issue reports Docker sandbox container-name collisions between multiple HOME-isolated OpenClaw instances using the default agent key.

## Best Possible Solution

Add a stable per-instance discriminator to Docker and sandbox-browser container-name inputs for shared and agent scopes, preserve session-scope naming, and cover the migration behavior in tests and release notes.

## Reproduction Assessment

Yes from source inspection, though not from a live Docker run in this read-only review. Default agent scope resolves to agent:main, slugifies to agent-main-f331f052, and the Docker/browser container names ignore workspaceDir.

## Solution Assessment

Yes on direction, but not as an unconditional cleanup-only change. The maintainable fix is to include a stable per-instance discriminator for agent/shared scopes, keep session-scope names stable, and make the upgrade behavior explicit.
