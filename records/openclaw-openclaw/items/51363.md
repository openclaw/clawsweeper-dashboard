## Summary

Keep open: current main and the latest release still derive Docker and sandbox-browser container names from the sandbox scope key before instance or workspace identity participates, while the direct fix PR closed unmerged and the broader multi-instance issue only partially overlaps.

## What This Changes

The issue reports Docker sandbox container-name collisions across multiple OpenClaw instances that use the default agent key on the same host.

## Best Possible Solution

Patch Docker and browser sandbox naming so shared and agent scopes include stable instance or workspace identity before Docker lookup, preserve session-scope stability, and add regression plus real Docker proof for two HOME-isolated default-agent instances.

## Reproduction Assessment

Yes, source-reproducible. Current main defaults sandbox scope to agent and derives Docker/browser container names from shared or agent:main before workspace identity participates, so two HOME-isolated default-agent instances still converge on the same container name.

## Solution Assessment

Unclear until maintainer review. Including stable workspace or instance identity in shared and agent sandbox names is the narrow likely fix, but maintainers should confirm the identity source and upgrade/orphan-container behavior before repair.
