## Summary

Keep open. Current main and v2026.6.6 still derive Docker and browser sandbox container names from the sandbox scope slug only, and the closed candidate PR that addressed this collision was not merged.

## What This Changes

The issue requests unique Docker sandbox container names for co-hosted OpenClaw instances that share the default agent key.

## Best Possible Solution

Patch the core sandbox naming contract so shared and agent-scoped Docker and browser container names include deterministic instance identity, keep session-scoped names stable, and add focused regression coverage plus release-note context.

## Reproduction Assessment

Yes, source-level: with agent or shared scope, two instances with different workspaces but the same default agent key compute the same scope slug and Docker name. I did not run the live Docker repro because this review is read-only.

## Solution Assessment

Yes on direction, but the exact identity and migration choice needs maintainer review. The fix should cover both `ensureSandboxContainer` and `ensureSandboxBrowser`, and should avoid renaming session-scoped containers that were already unique.
