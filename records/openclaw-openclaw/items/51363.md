## Summary

Keep open. Current main still derives Docker sandbox and sandbox-browser container names from the sandbox scope key only, so two HOME-isolated instances using the default main agent still collide; the related fix PR is closed unmerged.

## What This Changes

The issue requests unique Docker sandbox container names per OpenClaw instance so co-hosted instances using the default main agent do not share one sandbox container.

## Best Possible Solution

Add a stable per-instance discriminator, preferably derived from the resolved OpenClaw instance state/workspace root rather than a dynamic port, to both Docker and sandbox-browser container-name slug inputs while preserving per-scope reuse inside one instance and documenting the one-time container rename behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: two HOME-isolated gateways with default agent scope both resolve to agent:main, and current main slugifies only that value into agent-main-f331f052.

## Solution Assessment

Yes, the requested direction is the narrow maintainable fix, but it should cover both src/agents/sandbox/docker.ts and src/agents/sandbox/browser.ts and avoid dynamic discriminators like gateway port.
