## Summary

Keep open. Current main and the latest release still derive Docker sandbox and sandbox-browser container names from the sandbox scope key only, and the related fix PR was closed unmerged, so HOME-isolated instances using the default agent can still collide.

## What This Changes

The issue requests unique Docker sandbox container names for multiple HOME-isolated OpenClaw instances sharing a host and default agent key.

## Best Possible Solution

Add a stable per-instance discriminator to Docker and sandbox-browser container-name inputs for shared and agent scopes, preserve session-scope naming, and document the migration behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows two HOME-isolated gateways using default agent scope both resolve to `agent:main`, which slugifies to the reported `agent-main-f331f052` name.

## Solution Assessment

Yes on direction, but the exact identity input and migration policy need maintainer approval. The closed PR at https://github.com/openclaw/openclaw/pull/51380 is a useful reference because it covers both Docker and browser containers and avoids changing session scope after review feedback.
