## Summary

Keep open. Current main still names Docker sandbox and sandbox-browser containers from the sandbox scope key only, so HOME-isolated instances using the default agent can collide; the related fix PR at https://github.com/openclaw/openclaw/pull/51380 was closed unmerged.

## What This Changes

The issue requests unique Docker sandbox container names for multiple HOME-isolated OpenClaw instances on the same host.

## Best Possible Solution

Add a stable per-instance discriminator, preferably from the resolved OpenClaw instance state or workspace root, to Docker and sandbox-browser container-name inputs for shared and agent scopes while preserving session-scope names and documenting the one-time container rename behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: two HOME-isolated gateways with default agent scope both resolve to agent:main, and current main slugifies only that value into agent-main-f331f052.

## Solution Assessment

Yes on direction, but it needs maintainer approval for the instance identity input and migration behavior. The closed PR at https://github.com/openclaw/openclaw/pull/51380 is a useful reference because it covers both Docker and browser containers and narrows the change away from session scope.
