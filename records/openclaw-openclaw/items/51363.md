## Summary

Keep open. Current main and the latest release still derive Docker and sandbox-browser container names from the sandbox scope key only, and the related fix PR was closed unmerged, so co-hosted default-agent instances can still reuse one Docker runtime.

## What This Changes

The issue reports that multiple Docker-sandboxed OpenClaw instances on one host can share the same container name and therefore the wrong workspace when they use the default agent key.

## Best Possible Solution

Add a stable per-instance discriminator to Docker and browser runtime-name inputs for shared and agent scopes, preserve session-scope names, and cover upgrade cleanup or release-note behavior with focused tests.

## Reproduction Assessment

Yes from source inspection, though not from a live Docker run. Default agent scope resolves to agent:main, slugifies to agent-main-f331f052, and Docker/browser container names ignore workspaceDir.

## Solution Assessment

Yes on direction, but the exact discriminator and migration behavior need maintainer approval. The narrow maintainable fix is to include stable per-instance identity for shared and agent scopes while preserving session-scope names.
