## Summary

Keep open. Current main and v2026.5.22 still build Docker and sandbox-browser runtime names from the sandbox scope key only, while the related fix PR was closed unmerged, so co-hosted instances with the same default agent can still collide.

## What This Changes

The issue requests instance-unique Docker sandbox container names so multiple OpenClaw instances on one host cannot share one exec/browser sandbox runtime by accident.

## Best Possible Solution

Add a stable per-instance discriminator to Docker and browser runtime-name inputs for shared and agent scopes, preserve session-scope names, and cover upgrade cleanup or release-note behavior with focused tests.

## Reproduction Assessment

Yes from source inspection, though not from a live Docker run. Default agent scope resolves to agent:main, slugifies to agent-main-f331f052, and Docker/browser container names ignore workspaceDir.

## Solution Assessment

Yes on direction, but the exact discriminator and migration behavior need maintainer approval. The narrow maintainable fix is to include stable per-instance identity for shared/agent scopes while preserving session-scope names.
