## Summary

Keep open. Current main and v2026.5.20 still derive Docker and sandbox-browser runtime names from the sandbox scope key only, and the related fix PR was closed unmerged, so co-hosted instances with the same default agent can still collide.

## What This Changes

The issue reports that multiple HOME-isolated OpenClaw instances using Docker sandbox and the default agent key derive the same sandbox container name.

## Best Possible Solution

Add a stable per-instance discriminator to Docker and browser container-name inputs for agent/shared scopes, preserve session-scope names, and cover upgrade cleanup in tests and release notes.

## Reproduction Assessment

Yes from source inspection, though not from a live Docker run. Default agent scope resolves to agent:main, slugifies to agent-main-f331f052, and Docker/browser container names ignore workspaceDir.

## Solution Assessment

Yes on direction, but the exact discriminator and migration behavior need maintainer approval. The narrow maintainable fix is to include stable per-instance identity for agent/shared scopes while preserving session-scope names.
