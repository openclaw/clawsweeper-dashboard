## Summary

Keep open. Current main and the latest release still derive Docker and browser sandbox container names from shared or agent scope keys without stable instance/workspace identity, while https://github.com/openclaw/openclaw/pull/96866 is the active open fix candidate and has not merged yet.

## What This Changes

The issue reports that co-hosted OpenClaw instances using the same default agent key can collide on one Docker sandbox container and cross-read or cross-write workspaces.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/96866 after maintainer/security acceptance of the documented one-time non-shared runtime reset.

## Reproduction Assessment

Yes. Current main is source-reproducible: shared and agent sandbox names omit workspace or HOME identity, and the Docker reuse path can keep a same-named running container; I did not run Docker because this was a read-only cleanup review.

## Solution Assessment

Yes, the active fix path is the best current route. Centralizing workspace identity in sandbox scope keys covers Docker, browser, SSH, skills workspace, and recreate behavior, but maintainers still need to accept the upgrade reset before merge.
