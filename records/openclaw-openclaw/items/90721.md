## Summary

Keep open: current main and v2026.6.8 still expose tool-policy `group:*` terms next to channel `groups`/`groupPolicy`/`groupAllowFrom`, and the central terminology/design issue is not resolved by current docs, doctor checks, or config schema.

## What This Changes

The issue requests disambiguating OpenClaw config terminology for tool-policy `group:*` entries versus channel group authorization fields through a rename, docs callouts, and/or a doctor warning.

## Best Possible Solution

Maintainers should choose a compatibility-safe terminology plan: add docs cross-links and a doctor diagnostic first, and approve any config rename only with schema, docs, and doctor migration coverage.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.8 still expose `group:*` tool-policy entries beside channel `groups` and `groupAllowFrom` terminology. This is a feature/design request rather than a single failing runtime path, so no live repro was needed.

## Solution Assessment

Unclear as a final product direction. A broad rename is not automatically the best first fix because these are shipped config surfaces; docs plus a doctor warning is the safer narrow path unless maintainers approve a schema rename and migration.
