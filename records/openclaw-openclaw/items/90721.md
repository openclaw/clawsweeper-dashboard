## Best Possible Solution

Make an explicit maintainer-scoped product decision: add docs cross-links and a doctor warning as the safest first step, or sponsor a migration-backed config terminology change if a canonical rename is worth the compatibility cost.

Yes. Current main and v2026.6.8 both show `group:*` tool policy shorthands and channel `groups`/`groupAllowFrom` controls in the same public config surface; this is source-reproducible terminology and product-scope evidence, not a runtime failure path.

## Solution Assessment

Unclear. Docs and doctor clarification look like the safest first step, while renaming `group:` or `groupAllowFrom` would be compatibility-sensitive and should only happen with a maintainer-approved migration plan.
