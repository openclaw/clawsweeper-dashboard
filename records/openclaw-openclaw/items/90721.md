## Summary

Keep open: current main and v2026.6.8 still expose `group:*` tool-policy shorthands beside channel `groups`/`groupPolicy`/`groupAllowFrom` terminology, and the Feishu-only `groupSenderAllowFrom` work does not resolve the broader cross-config ambiguity.

## What This Changes

Request to disambiguate tool-policy `group:*` shorthands from channel group configuration by renaming terms and/or adding docs and doctor guidance.

## Best Possible Solution

Maintainers should choose a compatibility-safe terminology plan: add docs cross-links and a doctor diagnostic first, then approve any config rename only with schema, docs, and doctor migration coverage.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.8 still expose tool-policy `group:*` entries beside channel `groups` and `groupAllowFrom` terminology. This is a terminology/product request rather than a single failing runtime path.

## Solution Assessment

Unclear as a final product direction. A broad rename is not automatically the best first fix because these are shipped config surfaces; docs plus a doctor warning is the safer narrow path unless maintainers approve a schema rename and migration.
