## What This Changes

## Best Possible Solution

Land the reference-based removal fix, ideally with a focused regression test for the helper-level overflow race if maintainers want CI coverage for this specific failure mode.

## Reproduction Assessment

Yes. Source inspection shows the shared queue array can be spliced by overflow admission while a drain awaits, and the PR body supplies before/after helper output for that race.

## Solution Assessment

Yes. Removing successfully processed items by object reference covers both single-item and collect-group drains while preserving the existing retry/deferred behavior that depends on leaving items queued until the awaited run succeeds.
