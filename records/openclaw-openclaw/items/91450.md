## Summary

Keep this PR open for normal maintainer merge review: current main has a source-reproducible reply-queue race that can silently remove an undelivered survivor, and the branch applies a focused identity-based removal with terminal before/after proof and a committed regression test.

## What This Changes

The PR changes reply queue draining so processed queue entries are removed by object identity instead of by current front-of-array position, and adds regression coverage for overflow mutation during an awaited drain.

## Best Possible Solution

Land the focused identity-removal queue change with its regression coverage after normal CI and maintainer review.

## Reproduction Assessment

Yes from source inspection and the PR's terminal proof: current main keeps the in-flight item in queue while applyQueueDropPolicy can splice the same array, then drainNextQueueItem removes the current front after await. I did not rerun tests in this read-only review.

## Solution Assessment

Yes. Removing processed entries by reference is the narrowest maintainable fix that preserves post-await deferred retry semantics better than pre-removing items before run completion.
