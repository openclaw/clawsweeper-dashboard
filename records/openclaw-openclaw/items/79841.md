## What This Changes

The issue asks OpenClaw to keep subagent workspace metadata internal so `workspaceDir` is not sent in Gateway `method: "agent"` params and rejected by schema validation.

## Best Possible Solution

Land or explicitly decline the focused regression PR, keeping `workspaceDir` as internal spawn/session metadata rather than expanding the public Gateway schema.

## Solution Assessment

Yes. The narrow maintainable fix is to keep workspace inheritance in internal session/run metadata, and the linked PR adds the missing regression guard without making `workspaceDir` public API.
