## Summary

Keep open: current main keeps `workspaceDir` out of Gateway `agent` params, but https://github.com/openclaw/openclaw/pull/79821 is still an open closing PR for the exact regression guard, so this issue should remain paired with that PR until it lands or is explicitly closed.

## What This Changes

The issue asks OpenClaw to keep `workspaceDir` as internal subagent/session metadata and prevent it from being forwarded to Gateway `agent` RPC params.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79821 after its proof and mergeability gates are satisfied, while keeping `workspaceDir` internal instead of expanding public Gateway params.

## Reproduction Assessment

Yes, at source level: `AgentParamsSchema` has no `workspaceDir` and rejects extra properties, and current Gateway tests cover rejection when a public `agent` request carries that field. I did not run the live same-workspace canary, and current main already strips the field before the Gateway call.

## Solution Assessment

Yes. Keeping workspace metadata internal and adding a spawn-assembly regression guard is the narrow maintainable fix; expanding the public Gateway schema would expose an internal runtime detail.
