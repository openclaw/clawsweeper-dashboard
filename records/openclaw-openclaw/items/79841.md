## Summary

Keep open: current main keeps `workspaceDir` internal before the Gateway `agent` request, but this issue is explicitly paired with open closing PR https://github.com/openclaw/openclaw/pull/79821 and should not be closed before that PR path is resolved.

## What This Changes

The issue reports that internal `workspaceDir` metadata can leak into Gateway `method: "agent"` params during subagent spawn and be rejected by schema validation.

## Best Possible Solution

Resolve the linked PR path, likely by landing or superseding the focused guard while keeping `workspaceDir` internal rather than expanding public Gateway params.

## Reproduction Assessment

No high-confidence current-main end-to-end failure was reproduced; current main strips `workspaceDir` before the Gateway call. Source inspection does confirm the reported failure mode if the field leaks, because the Gateway schema disallows it.

## Solution Assessment

Yes. Keeping workspace metadata in session/run bookkeeping and out of public Gateway `agent` params is the narrow maintainable fix; expanding the public schema would expose an internal runtime detail.
