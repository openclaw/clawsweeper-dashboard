## What This Changes

Ensure subagent workspace metadata stays internal and is not forwarded as a public Gateway `method="agent"` `workspaceDir` param.

## Best Possible Solution

Land or close the linked regression-guard PR while preserving the current internal-only workspace metadata boundary rather than expanding the public Gateway schema.

## Reproduction Assessment

No, not as a failing current-main path: source inspection shows current main strips `workspaceDir` before the Gateway call. The schema rejection is clear if the field leaks, and the reporter supplied a 2026.5.6 canary plus an open PR adding the missing guard.

## Solution Assessment

Yes. Keeping workspace metadata in `spawnedWorkspaceDir` and internal run/session metadata is the narrowest maintainable fix; adding `workspaceDir` to public Gateway `agent` params would unnecessarily publish an internal runtime detail.
