## Summary

Keep open: https://github.com/openclaw/openclaw/pull/79821 is an open closing PR for this issue, and current main already keeps `workspaceDir` internal before the Gateway `agent` call. Cleanup should wait for that PR to merge, close, or be folded into the adjacent cwd fix path.

## What This Changes

The issue reports that internal `workspaceDir` metadata can leak into Gateway `method="agent"` params during native subagent spawn, causing strict `AgentParams` validation to reject the run.

## Best Possible Solution

Resolve https://github.com/openclaw/openclaw/pull/79821, or fold its assertion into the active cwd fix path, while preserving internal-only workspace metadata and schema-clean public Gateway `agent` params.

## Reproduction Assessment

Yes, at source level: `AgentParamsSchema` rejects `workspaceDir`, and the Gateway test verifies `invalid agent params` when it is present. I did not reproduce a current-main leak path because current main strips `workspaceDir` before the `method="agent"` call.

## Solution Assessment

Yes. Keeping `workspaceDir` as internal spawn/session metadata and adding a focused regression guard is the narrow maintainable fix; expanding the public Gateway `agent` schema would weaken the boundary.
