## Summary

Current main already keeps `workspaceDir` internal for native subagent spawns: the spawn path stores it as `spawnedWorkspaceDir`, strips it before the Gateway `agent` request, and the Gateway schema remains strict.

## What This Changes

The issue asks for same-workspace subagent spawn metadata to stay out of public Gateway `method="agent"` params while preserving internal workspace bookkeeping.

## Best Possible Solution

Keep the current main design: persist inherited workspace as `spawnedWorkspaceDir` session metadata and keep `workspaceDir` out of the public Gateway `AgentParams` contract.

## Reproduction Assessment

No high-confidence current-main failure path remains. Source inspection shows the old rejection condition is real if `workspaceDir` is sent publicly, but current `sessions_spawn` strips it before the Gateway `agent` call and preserves it through session metadata.

## Solution Assessment

Yes. Keeping `workspaceDir` out of the public schema while forwarding only stored `spawnedWorkspaceDir` into the runner is the narrowest maintainable fix.
