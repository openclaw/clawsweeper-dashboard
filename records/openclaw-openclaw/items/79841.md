## What This Changes

The issue asks to prevent internal `workspaceDir` metadata from leaking into Gateway `method="agent"` params during subagent spawn while preserving it for internal run/session bookkeeping.

## Best Possible Solution

Review and either land or close https://github.com/openclaw/openclaw/pull/79821; after it lands, this issue can close with the internal metadata boundary and regression guard in place.

## Reproduction Assessment

Yes, source inspection gives a high-confidence boundary reproduction: `AgentParamsSchema` disallows `workspaceDir` and the handler rejects schema failures with `invalid agent params`. I did not establish a live failing current-main path because current main strips `workspaceDir` before the Gateway call.

## Solution Assessment

Yes, keeping `workspaceDir` as internal spawn/session metadata is the narrow maintainable fix. Expanding the public Gateway schema would expose an internal runtime detail, while the linked PR adds the missing regression guard without changing the protocol.
