## Summary

Keep open: this issue is specifically paired with an open closing PR, and current main still benefits from that PR's focused regression guard even though the runtime boundary now keeps `workspaceDir` out of public Gateway `agent` params.

## What This Changes

The issue reports a Gateway schema rejection when internal `workspaceDir` metadata leaks into public `method="agent"` params during native subagent spawn.

## Best Possible Solution

Review and either land or close https://github.com/openclaw/openclaw/pull/79821 while preserving the current internal-only workspace metadata boundary instead of expanding public Gateway `agent` params.

## Reproduction Assessment

Yes, at source level: `AgentParamsSchema` rejects unknown fields, so any leaked `workspaceDir` in public `agent` params fails validation. Current main strips that field before the Gateway call, and the open PR adds the missing regression guard.

## Solution Assessment

Yes, the narrow solution is to keep workspace/cwd metadata internal through `spawnedWorkspaceDir` and run/session metadata, not to add `workspaceDir` to the public Gateway schema. The immediate maintainer path is normal review of the linked PR.
