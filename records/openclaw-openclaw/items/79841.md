## What This Changes

The issue requests keeping inherited subagent `workspaceDir` as internal session/run metadata instead of forwarding it in Gateway `method: "agent"` params.

## Best Possible Solution

Review and either land or close https://github.com/openclaw/openclaw/pull/79821 while keeping `workspaceDir` internal to session/run metadata rather than expanding the public Gateway agent schema.

## Reproduction Assessment

Yes, from source and reporter evidence. The Gateway schema rejects `workspaceDir` on public `agent` params, and the reported affected path is specifically a leak of that internal field during subagent spawn.

## Solution Assessment

Yes. Stripping `workspaceDir` before the validated Gateway call is the narrow maintainable fix; adding it to the public schema would unnecessarily widen the protocol surface.
