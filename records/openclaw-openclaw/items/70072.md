## Summary

Keep open. Current main still lacks the requested deep-dreaming introspection surfaces: the scheduled report remains aggregate-only, verbose scoring is logger-only, rejected entries are filtered before callers can explain them, and the memory CLI still has only single-candidate `promote-explain` rather than an aggregate dreaming diagnostic.

## What This Changes

Request to add user-readable introspection for deep dreaming candidate ranking, rejection reasons, verbose scoring artifacts, and an aggregate dreaming diagnostic command.

## Best Possible Solution

Add one shared deep-promotion evaluation/explain surface that returns bounded considered candidates, score components, thresholds, pass/fail state, promotion status, and rejection reasons, then reuse it for reports, verbose JSONL, and the CLI diagnostic without changing promotion policy.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is an additive diagnostics request. Current-main source inspection is a high-confidence verification path because the relevant report, logging, ranking, and CLI registration paths are visible and still lack the requested surfaces.

## Solution Assessment

Yes, the requested direction is best solved by sharing the actual promotion evaluation logic across report, JSONL, and doctor outputs. Duplicating scoring or gate logic in separate callers would risk drift from real promotion behavior.
