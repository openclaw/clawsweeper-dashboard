## Summary

Keep open. Current main still lacks the requested deep-dreaming introspection surfaces: deep reports stay aggregate-only, verbose scoring is logger-only, rejected candidates are filtered before callers can explain them, and the memory CLI has single-candidate `promote-explain` but no aggregate dreaming diagnostic.

## What This Changes

Expose per-candidate deep-dreaming ranking, promotion/rejection reasons, structured verbose scoring artifacts, and an aggregate memory dreaming doctor command.

## Best Possible Solution

Add one shared deep-promotion evaluation/explain surface that reports bounded considered candidates, score components, thresholds, pass/fail state, promotion status, and rejection reasons, then reuse it for the deep report, verbose JSONL, and the CLI diagnostic without changing promotion policy.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is an additive diagnostics request. Current-main source inspection is a high-confidence verification path because the report, verbose logging, ranking, CLI, and docs paths are visible and still lack the requested surfaces.

## Solution Assessment

Yes. The maintainable path is a shared evaluator/explainer used by scheduled reports, structured verbose output, and CLI diagnostics; duplicating scoring or gate logic in separate callers would risk drift from the real promotion behavior.
