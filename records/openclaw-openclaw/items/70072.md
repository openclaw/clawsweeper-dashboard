## Summary

Keep open. Current main has some adjacent status and `promote-explain` surfaces, but it still does not write per-candidate deep-run rejection reasons into the deep report, does not persist verbose scoring as a workspace artifact, and does not expose an aggregate `memory dreaming doctor` diagnostic.

## What This Changes

The issue requests more introspection for deep dreaming: per-candidate report details, a persisted verbose scoring trace, and a CLI diagnostic for promotion gate reachability.

## Best Possible Solution

Add one shared deep-promotion evaluation/explain API and reuse it from the deep report, verbose artifact writer, CLI diagnostic, and doctor/UI status surfaces without changing ranking thresholds or promotion policy.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an additive diagnostics request. Source inspection gives a high-confidence verification path because the report writer, verbose path, ranking API, docs, and CLI registration still lack the requested surfaces.

## Solution Assessment

Yes. The maintainable direction is to centralize real promotion gate evaluation and rejection reasons, then render that same data in Markdown, JSONL, and CLI output so the diagnostics cannot drift from the scorer.
