## Summary

Keep open. Current main and release v2026.5.18 still expose aggregate deep-dreaming counts plus logger-only verbose scoring; the existing `promote-explain` and REM harness paths do not provide per-run rejection reasons, persisted scoring traces, or an aggregate dreaming doctor diagnostic.

## What This Changes

Request to add deep dreaming diagnostics: per-candidate report reasons, persisted verbose scoring, and an `openclaw memory dreaming doctor` command.

## Best Possible Solution

Centralize deep-promotion evaluation and rejection reasons, then render the same bounded data in the deep Markdown report, optional JSONL trace, and CLI diagnostic without changing ranking policy.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an additive diagnostics feature. Source inspection gives a high-confidence verification path because current main still lacks persisted per-candidate reasoning, verbose scoring artifacts, and the requested diagnostic command.

## Solution Assessment

Yes. A shared evaluation/explain data path is the maintainable solution because the report, JSONL trace, and CLI output should not duplicate or drift from the scorer’s gate logic.
