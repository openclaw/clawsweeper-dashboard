## Summary

Keep open. Current main still lacks the requested introspection: scheduled deep reports only contain aggregate counts, verbose scoring is emitted to the logger, filtered-out candidates do not carry rejection reasons to callers, and the memory CLI has single-candidate `promote-explain` but no aggregate dreaming doctor.

## What This Changes

The issue requests memory-core dreaming diagnostics that expose ranked/rejected candidate details in deep reports, write structured verbose scoring artifacts, and add a dreaming doctor command for aggregate gate reachability.

## Best Possible Solution

Add one shared deep-promotion evaluation surface that returns bounded considered candidates with score components, thresholds, pass/fail state, promotion status, and rejection reasons, then reuse it for reports, verbose JSONL, and the CLI diagnostic without changing promotion policy.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is an additive diagnostics request. Current-main source inspection is a high-confidence verification path because the report, logging, ranking, and CLI registration paths are visible and lack the requested surfaces.

## Solution Assessment

Yes, the requested direction is best solved by sharing the actual promotion evaluation logic across report, JSONL, and doctor outputs. Duplicating scoring or gate logic in separate callers would risk drift from real promotion behavior.
