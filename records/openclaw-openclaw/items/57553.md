## Best Possible Solution

## Solution Assessment

Yes for this PR's scoped mitigation: hiding brew-only choices in Linux containers avoids steering users into a broken Homebrew path without adding package-manager execution. It is still a product choice versus the open Linuxbrew-in-Docker and apt-fallback PRs.
