## Best Possible Solution

Keep the generated-docs approach, but land it only after both new package scripts invoke the TypeScript generator through Node so the preflight works on Windows, macOS, and Linux.

## Reproduction Assessment

Yes at source level: `package.json` passes `tsx` to `run-with-env`, while `run-with-env` only special-cases `node` and spawns without a shell. A Windows run would be the live confirmation, but the code path is clear.

## Solution Assessment

No for the branch as-is; generating the marker blocks from the matrix is the right layer, but the package-script invocation should use the existing `node --import tsx` pattern before merge.
