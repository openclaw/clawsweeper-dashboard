## Summary

Keep open: the linked Telegram stale-reply bug is valid, but this branch keys suppression to the abort fence, which does not advance for ordinary newer Telegram messages and therefore misses the reported interleaving.

## What This Changes

The PR adds a generation field to Telegram buffered final-answer state and passes the dispatch abort-fence generation when buffering and flushing final answer segments.

## Best Possible Solution

Add turn-bound Telegram delivery semantics so late work is suppressed or explicitly bound to its original inbound message, with regression coverage for ordinary newer-message and sub-agent completion interleavings.

## Reproduction Assessment

Yes for source-level reproduction: current main only advances the checked generation for authorized abort commands, while the linked report's ordinary newer-message interleaving leaves that generation unchanged. I did not run a live Telegram E2E.

## Solution Assessment

No. The branch uses `abortFenceGeneration` as the stale-turn discriminator, but the reported supersession is a normal newer inbound message; a per-inbound turn generation or equivalent delivery binding is the narrower maintainable fix.
