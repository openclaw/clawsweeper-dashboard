## Summary

Keep open. The patch is a plausible and mostly well-placed fix for the sandbox collision, but it still needs real Docker or SSH behavior proof and upgrade-facing docs or PR-body wording because the new non-shared scope key changes persistent sandbox identity beyond Docker names.

## What This Changes

The PR appends a workspace hash to non-shared sandbox scope keys and passes the resolved scope key into Docker sandbox creation, labels, registry entries, and regression tests.

## Best Possible Solution

Keep the context-level scope-key fix, but merge only after redacted real Docker or SSH proof and explicit docs or PR-body upgrade wording for the changed sandbox identity are in place.

## Reproduction Assessment

Yes. Current main and v2026.6.10 derive agent/session sandbox identity without workspace identity, and the linked canonical report gives a same-agent multi-HOME Docker collision path; I did not run Docker in this read-only review.

## Solution Assessment

Mostly yes, but not merge-ready. Computing the workspace-qualified key once in sandbox context is a better owner-boundary fix than Docker-only slug patching, but the upgrade/docs gap and missing real runtime proof still need resolution.
