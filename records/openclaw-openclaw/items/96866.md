  - body: The workspace slug can be 52 chars, so with the default `openclaw-sbx-browser-` prefix the final `name.slice(0, 63)` drops the full-key hash.   - confidence: 0.9

Existing shared SSH sandboxes such as the old `openclaw-ssh-shared-8198076c` root would be bypassed by a new hash-derived root after upgrade, which is especially risky for remote-canonical workspaces. Keep the legacy shared id or add explicit migration/docs for that broader reset.
  - confidence: 0.86
