## Summary

Keep open. The patch appears to address a source-proven sandbox isolation bug and the submitted Docker proof is sufficient, but it intentionally changes persistent non-shared sandbox identity, so maintainers still need to accept the one-time upgrade/runtime recreation behavior before merge.

## What This Changes

The PR adds a workspace hash to non-shared sandbox scope keys, threads that key through Docker/SSH/skills/recreate paths, documents upgrade behavior, and adds regression coverage.

## Best Possible Solution

Land the workspace-qualified non-shared sandbox identity after maintainers accept the documented one-time runtime recreation and cleanup behavior.

## Reproduction Assessment

## Solution Assessment

Yes, subject to maintainer acceptance of the upgrade reset. Computing the workspace-qualified key once in sandbox context and carrying it through Docker, browser, SSH, registry, and skills workspace paths is cleaner than patching only Docker/browser slugs.
