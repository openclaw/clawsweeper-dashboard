## Summary

Keep open: current main still fabricates a visible Telegram DM silent-reply fallback for successful no-visible-response turns, while open PRs #70630 and #75185 already target the narrow fix.

## What This Changes

Telegram direct-message no-visible-response turns should remain silent instead of synthesizing a `NO_REPLY` payload that direct silent-reply policy rewrites into visible filler text.

## Best Possible Solution

Land one canonical Telegram adapter fix that treats successful no-visible direct turns as handled silent no-ops while preserving real dispatch-error, non-silent delivery-failure, and group fallback behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the current Telegram test constructs a direct session with `queuedFinal: false`, direct silence disallowed, and rewrite enabled, then asserts a visible rewritten reply is delivered.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Telegram adapter/status fallback path; changing global direct silent-reply defaults would alter broader documented policy.
