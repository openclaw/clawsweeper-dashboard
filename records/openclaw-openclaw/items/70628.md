Keep open for maintainer follow-up. Current main, via https://github.com/openclaw/openclaw/pull/78228, fixes the visible Telegram DM filler message, but the original acceptance also asked for quiet no-response turns to complete as handled status; current main still routes that quiet DM path through the no-final-response error status branch, and https://github.com/openclaw/openclaw/pull/70630 remains open with that unique remainder.

## What This Changes

Telegram direct-message turns with no visible final reply should stay silent instead of synthesizing a NO_REPLY fallback that can be rewritten into visible filler text.

## Best Possible Solution

## Solution Assessment

Not fully. The merged fix is the right narrow boundary for the visible Telegram DM chatter, but the remaining handled-status semantics need maintainer judgment or a small current-main follow-up rather than merging the stale paired branch as-is.
