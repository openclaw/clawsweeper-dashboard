## Summary

Keep open. Current main still contains the Telegram no-visible-response branch that synthesizes `NO_REPLY` and sends the projected payload, the regression test still asserts the visible rewritten fallback, and open PRs #70630 and #75185 already target the narrow fix.

## What This Changes

Telegram direct-message turns that end without a visible final reply should stay silent instead of fabricating visible silent-reply filler text.

## Best Possible Solution

Land one canonical Telegram adapter fix that treats successful no-visible direct turns as handled silent no-ops while preserving dispatch-error, non-silent delivery-failure, and group fallback behavior.

## Reproduction Assessment

Yes, from source and regression coverage. Current main has a direct-session test that sets `queuedFinal: false`, disallows direct silent replies, enables direct rewrite, and asserts a visible rewritten reply is delivered.

## Solution Assessment

Yes. The narrow maintainable fix is in the Telegram adapter/status path: suppress the synthetic fallback for successful no-visible DM turns rather than changing global direct silent-reply policy.
