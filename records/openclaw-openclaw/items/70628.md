## Summary

Keep open. Current main still contains the Telegram DM no-visible-response path that synthesizes a transport-visible `NO_REPLY`, the regression test still asserts the rewritten filler reply, and open PRs #70630 and #75185 already target the narrow fix.

## What This Changes

Telegram direct-message turns that finish without a visible final reply should stay silent instead of sending a rewritten `NO_REPLY` fallback bubble.

## Best Possible Solution

Land one canonical Telegram adapter fix that treats successful no-visible direct turns as handled silent no-ops while preserving real dispatch-error, non-silent delivery-failure, and group fallback behavior.

## Reproduction Assessment

Yes. Current main has a source-level reproduction path in the Telegram dispatch test: a direct session with `queuedFinal: false`, direct silence disallowed, and direct rewrite enabled is expected to deliver one visible rewritten fallback.

## Solution Assessment

Yes. The narrowest maintainable fix is in the Telegram adapter/status path, not a global change to direct silent-reply defaults that would alter broader product policy.
