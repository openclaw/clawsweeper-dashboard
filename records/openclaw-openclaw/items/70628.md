## Summary

Keep open. Current main still sends a synthesized Telegram `NO_REPLY` fallback through direct-chat rewrite for successful no-visible DM turns, and open PR #70630 explicitly closes this issue, so the issue should stay open until a canonical fix merges.

## What This Changes

Telegram direct-message turns that finish with no visible final reply should remain silent instead of sending rewritten `NO_REPLY` filler text.

## Best Possible Solution

Land one canonical Telegram adapter fix that treats successful no-visible direct turns as handled silent no-ops while preserving real error, non-silent delivery-failure, and group fallback paths.

## Reproduction Assessment

Yes. The existing source test constructs a Telegram direct session with `queuedFinal: false`, direct silent replies disallowed, and direct rewrite enabled, then asserts that a visible rewritten reply is delivered.

## Solution Assessment

Yes. The narrowest maintainable fix belongs in the Telegram adapter/status path, not in the global direct silent-reply defaults that cover broader product policy.
