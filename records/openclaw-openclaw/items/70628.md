## Summary

Keep open. Current main still contains the Telegram direct-message no-visible-response path that synthesizes `NO_REPLY`, the regression test still expects visible rewritten chatter, and open PR #70630 already has closing syntax for this issue.

## What This Changes

The issue asks Telegram direct-message turns with no visible final reply and no actual dispatch or delivery error to complete silently instead of sending a synthetic silent-reply fallback bubble.

## Best Possible Solution

Land one canonical Telegram adapter fix that treats successful no-visible direct-message turns as handled silent no-ops while preserving real dispatch-error, non-silent delivery-failure, and group fallback behavior.

Yes. Source inspection shows the current branch still synthesizes and delivers `NO_REPLY`, and the current Telegram dispatch regression test constructs the direct-session case and expects a rewritten visible fallback.

## Solution Assessment
