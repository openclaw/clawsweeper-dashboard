## Summary

Keep open. Current main has swept through the visible Telegram DM filler path, but a successful quiet DM turn can still be treated as having no final response for status-reaction finalization, and the issue is paired with an open closing PR for that remaining behavior.

## What This Changes

Telegram DM no-visible-response turns should stay silent and finalize as handled instead of emitting rewritten `NO_REPLY` filler or error status.

## Best Possible Solution

Represent successful quiet Telegram DM no-response turns as handled silence in dispatcher/status state, with regression coverage proving no delivery and `setDone` rather than `setError`.

## Reproduction Assessment

Yes, at source level. A no-visible Telegram DM turn with no dispatch error and no delivery now skips fallback delivery, but `hasFinalResponse` remains false, so a status reaction controller finalizes as error.

## Solution Assessment

No, current main is only a partial solution. The best path is to refresh, land, or replace https://github.com/openclaw/openclaw/pull/70630 with the remaining handled-silent status fix and proof.
