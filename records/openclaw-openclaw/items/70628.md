## Summary

Keep open. Current main still has the Telegram no-visible-response branch that synthesizes `NO_REPLY`, the direct silent-reply projection can rewrite that into visible filler, and an open PR with closing syntax already targets this issue.

## What This Changes

The issue requests that Telegram DMs stay silent when a successful turn has no visible final reply instead of synthesizing `NO_REPLY` and rewriting it into filler text.

## Best Possible Solution

Review and land one canonical Telegram adapter patch that treats successful no-visible direct turns as handled silent no-ops while preserving real error and group fallback paths.

## Reproduction Assessment

Yes, from source and regression coverage. I did not run tests in this read-only review, but current main has a direct-session fixture that sets `queuedFinal: false`, disallows direct silence, enables direct rewrite, and asserts a visible fallback delivery.

## Solution Assessment

Yes. The narrow maintainable fix is in the Telegram adapter/status path: suppress the synthetic fallback for successful no-visible DM turns rather than changing global direct silent-reply policy.
