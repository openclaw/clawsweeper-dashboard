## Summary

Keep open. Current main still has the Telegram DM no-visible-response path that synthesizes `NO_REPLY`, the regression test still expects visible rewritten chatter, and open PRs #70630 and #75185 already target the narrow fix.

## What This Changes

Telegram direct-message turns should stay silent when a successful turn has no visible final reply instead of sending rewritten silent-reply filler.

## Best Possible Solution

Land one canonical Telegram adapter fix that treats successful no-visible direct turns as handled silent no-ops while preserving real dispatch-error, non-silent delivery-failure, and group fallback behavior.

## Reproduction Assessment

Yes. The current test fixture constructs a Telegram direct session with `queuedFinal: false`, direct silent replies disallowed, and direct rewrite enabled, then asserts that a rewritten visible reply is delivered.

## Solution Assessment

Yes. The narrow fix belongs in the Telegram adapter/status path rather than changing global direct silent-reply defaults, which remain broader product policy.
