## Summary

Keep open. Current main still has the Telegram DM no-visible-response path that synthesizes `NO_REPLY`, shared outbound policy rewrites that sentinel into visible filler for direct conversations, and open PRs #70630 and #75185 already target the narrow fix.

## What This Changes

The issue requests that Telegram DM turns with no visible final reply and no dispatch or delivery failure complete silently instead of sending rewritten silent-reply filler.

## Best Possible Solution

Land one canonical Telegram adapter fix that treats successful no-visible direct turns as handled silent no-ops, while preserving real error fallback and group fallback behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the current Telegram dispatch test constructs a direct session with `queuedFinal: false`, direct silence disallowed, and rewrite enabled, then asserts a visible rewritten delivery is sent.

## Solution Assessment

Yes. The narrow maintainable solution is in the Telegram adapter/status path, not a global change to direct silent-reply defaults that affect other surfaces.
