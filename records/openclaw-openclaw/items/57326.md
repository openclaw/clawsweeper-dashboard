## Summary

Keep open: current main fixed some original paths, but source inspection still shows CLI-backed diagnostic, slug, and subagent-dispatch gaps, and recent 2026.6.5 comments plus https://github.com/openclaw/openclaw/issues/92405 preserve active repro context.

## What This Changes

The issue asks for helper and diagnostic paths to use the same CLI runtime dispatch as the main agent runner for CLI-backed model configurations.

## Best Possible Solution

Keep this umbrella open until the remaining diagnostic/slug helper paths and the subagent dispatcher boundary are fixed or deliberately split, while preserving canonical model identity and carrying CLI execution runtime as a prepared dispatch fact.

## Reproduction Assessment

Yes at source level: current main still routes list.probe and llm-slug-generator through runEmbeddedAgent and persists raw providers in subagent spawn state. I did not run the live CLI-backed reproduction in this read-only review.

## Solution Assessment

No, not as a single raw-provider-to-modelProvider persistence patch. The safer direction is to preserve canonical provider/model session state and resolve CLI execution providers at the runner/dispatch boundary, matching the accepted followup and cron patterns.
