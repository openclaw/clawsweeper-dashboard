## Summary

Keep open: current main and the latest release fixed some original paths, but source inspection still shows `models status --probe`, LLM slug generation, and subagent spawn/session persistence do not share the accepted CLI-runtime dispatch boundary; the subagent subset is also preserved in open related issue https://github.com/openclaw/openclaw/issues/92405.

## What This Changes

The issue reports that CLI-backed helper paths can bypass `runCliAgent()` and lose the configured runtime/auth/session routing context.

## Best Possible Solution

Keep canonical model/session identity intact, carry a prepared execution-runtime decision at the owner boundary, route probe/slug/subagent helper execution through the same CLI dispatch invariant as main/followup/cron paths, and use https://github.com/openclaw/openclaw/issues/92405 for the subagent-specific follow-up.

## Reproduction Assessment

Yes, at source level: current main still routes `list.probe` and `llm-slug-generator` through `runEmbeddedAgent()` and persists raw providers in subagent spawn state while sibling paths now resolve CLI runtimes first. I did not run the live CLI-backed reproduction in this read-only review.

## Solution Assessment

No, the broad issue should not be auto-fixed with a one-off helper patch until maintainers choose the execution-runtime boundary. The maintainable direction is to preserve canonical model identity and pass resolved execution runtime explicitly where runner selection or registration needs it.
