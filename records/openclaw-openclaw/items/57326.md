## Summary

Keep open: current main and the latest release still leave meaningful helper-path gaps. Followups and cron now use the shared CLI runtime resolver, but `models status --probe`, LLM slug generation, and subagent/gateway registration still have source-visible divergence from CLI dispatch.

## What This Changes

The issue requests routing all CLI-backed helper paths through the same CLI dispatch decision as normal agent runs while preserving runtime, auth, session, and routing context.

## Best Possible Solution

Keep this issue open until diagnostic, slug, and subagent/gateway helper paths share the CLI execution-provider decision without corrupting canonical model/session state, backed by focused regression coverage and live proof for the nested subagent path.

## Reproduction Assessment

Yes for source-level reproduction: current main still has direct embedded auth-probe and slug helper paths, plus raw-provider subagent persistence, under the CLI-backed configuration described by reporters. I did not run a live CLI account reproduction in this read-only review.

## Solution Assessment

Unclear as the final fix shape: the shared `resolveCliRuntimeExecutionProvider` pattern is the right direction, but the subagent boundary needs maintainer review to avoid mixing canonical model state with execution runtime identity.
