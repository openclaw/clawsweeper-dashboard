## Summary

Keep open. Current main still has source-level helper paths for queued followups, model auth probes, LLM slug generation, and subagent model persistence that write or execute raw provider/model data without the CLI-runtime dispatch step; the issue is also paired with the open closing PR https://github.com/openclaw/openclaw/pull/72499.

## What This Changes

The issue asks OpenClaw helper execution paths to resolve CLI-backed runtimes and route through the CLI runner instead of the embedded/API runner.

## Best Possible Solution

Review, rebase, or replace https://github.com/openclaw/openclaw/pull/72499 so followups, probes, and slug generation use the shared CLI dispatch pattern, while splitting subagent/session persistence hardening if needed.

## Reproduction Assessment

Yes, at source level. Configure a canonical provider with a CLI runtime and trigger queued followup execution, openclaw models status --probe, LLM slug generation, or subagent model persistence; current main still has helper paths that use embedded execution or raw provider persistence without resolving the CLI runtime first.

## Solution Assessment

Yes for the direction, not yet for the open implementation. The maintainable fix is to reuse resolveCliRuntimeExecutionProvider, isCliProvider, and runCliAgent while preserving helper-specific auth, session, timeout, cancellation, and tool-policy semantics.
