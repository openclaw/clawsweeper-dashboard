## Summary

Keep open. Current main still has source-level helper paths that call the embedded runner directly for queued followups, model auth probes, and LLM slug generation, and the issue is paired with the open closing PR https://github.com/openclaw/openclaw/pull/72499.

## What This Changes

Route CLI-backed helper paths through the same CLI runtime dispatch used by normal agent turns.

## Best Possible Solution

Review, rebase, or replace https://github.com/openclaw/openclaw/pull/72499 so queued followups, probes, and slug generation share the main CLI dispatch pattern, while splitting subagent persistence hardening if needed.

## Reproduction Assessment

Yes, at source level. Configure a canonical provider with a CLI runtime and trigger queued followup execution, openclaw models status --probe, or LLM slug generation; current main still reaches runEmbeddedPiAgent directly in those helpers without resolving the CLI execution provider first.

## Solution Assessment

Yes. Reusing resolveCliRuntimeExecutionProvider plus isCliProvider plus runCliAgent is the narrow maintainable direction, provided the helper-specific auth, session, timeout, cancellation, and tool-policy semantics are preserved.
