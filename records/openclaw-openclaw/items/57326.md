## Summary

Keep open: current main has fixed some earlier CLI-backed helper routing paths, but source inspection still shows `models status --probe`, LLM slug generation, and subagent dispatch/state paths that do not consistently use the main runner's CLI-runtime decision. The newer open subagent report preserves one narrowed slice, but it does not cover the diagnostic and slug-helper remainder.

## What This Changes

The issue asks for CLI-backed helper and diagnostic paths to use the same CLI dispatch decision as normal agent runs while preserving auth, session, and routing context.

## Best Possible Solution

Use a maintainer-reviewed execution-runtime boundary that resolves CLI runtimes before helper dispatch while preserving canonical model/auth/session identity, and let https://github.com/openclaw/openclaw/issues/92405 own the subagent-specific narrowing.

## Reproduction Assessment

Yes for source-level reproduction: current main still has direct embedded/probe and slug-helper paths plus raw subagent provider persistence that bypass the accepted CLI-runtime resolver pattern. I did not independently run the live CLI-backed scenarios during this read-only review.

## Solution Assessment

No single submitted patch is currently the proven best fix. The accepted followup/cron pattern is a strong template, but probe behavior and subagent session identity need maintainer review so execution runtime does not overwrite canonical model/provider state.
