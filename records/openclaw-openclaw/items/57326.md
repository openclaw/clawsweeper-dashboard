## Summary

Keep open. Current main has the shared CLI-routing pattern for queued followups and cron, but model auth probes, LLM slug generation, and initial child-session runtime persistence still have source-level paths that bypass the configured CLI runtime; the issue also has an open closing PR that should be reviewed rather than closing the issue first.

## What This Changes

The issue asks for helper execution paths to resolve CLI-backed runtimes and dispatch through the CLI runner instead of embedded/API paths.

## Best Possible Solution

Review and land a rebased narrow fix or replacement for the remaining probe, slug, and child-session persistence paths, while splitting distinct stale-pin or compaction hardening into their own linked items such as https://github.com/openclaw/openclaw/pull/84966.

## Reproduction Assessment

Yes, at source level. Configure a canonical provider model with a CLI runtime such as claude-cli, then trigger models status --probe, LLM slug generation, or child-session model persistence; current main still uses embedded execution or raw provider persistence in those paths.

## Solution Assessment

Yes for the direction, but current main is incomplete. The maintainable fix is to reuse resolveCliRuntimeExecutionProvider and the existing CLI dispatch contract at each remaining helper boundary without broadening the runtime namespaces.
