## Best Possible Solution

Carry the effective ask or a typed Allow Always unavailable reason through the canonical request/detail/view models and update every approval renderer/caller with tests for both one-shot and ask=always prompts.

## Reproduction Assessment

## Solution Assessment

No. Branching in a shared renderer on optional `ask` is only safe if every production caller and structured detail path supplies it; the safer fix is to preserve the effective ask or a typed unavailable reason through the approval model.
