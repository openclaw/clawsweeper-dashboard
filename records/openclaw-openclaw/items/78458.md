## Summary

Keep this open: current main still has a source-level path where Dreaming narrative sessions rely on in-process cleanup, and a Gateway restart can leave persisted session entries behind.

## What This Changes

The issue asks for Dreaming child sessions to be cleaned up durably when Gateway restarts interrupt the normal `deleteSession` cleanup path.

## Best Possible Solution

Update or replace the open memory-core cleanup PR so Dreaming uses bounded/durable transient session cleanup and removes completed plugin-owned Dreaming sessions left by restarts, with regression coverage for persisted `sessions.json` entries.

## Reproduction Assessment

Yes. A high-confidence source path exists: current main creates timestamped Dreaming narrative sessions and relies on a `finally`-block `deleteSession`, while the scrubber keeps referenced transcript entries that would still appear in `sessions.json`.

## Solution Assessment

No complete fix is on main yet. The open PR's stable-key and pre-cleanup direction is relevant, but the final solution should preserve the generic plugin-owned deletion contract and add restart-orphan coverage.
