## Summary

Keep open: current main still has a source-level path where Dreaming narrative sessions use timestamped keys and rely on process-local cleanup, so a Gateway restart can leave visible child sessions behind.

## What This Changes

The issue reports that Dreaming child sessions can survive a Gateway restart because cleanup depends on in-process `deleteSession` calls.

## Best Possible Solution

Keep this as restart-specific evidence while maintainers review, land, or replace https://github.com/openclaw/openclaw/pull/70464, then add regression coverage for restart-left Dreaming session rows.

## Reproduction Assessment

Yes, at source level. Current main still creates timestamped Dreaming narrative session keys, relies on a `finally`-block `deleteSession`, and preserves live transcript-backed Dreaming rows in the scrubber.

## Solution Assessment

No complete fix is on main yet. The stable workspace/phase key plus timestamped idempotency direction in https://github.com/openclaw/openclaw/pull/70464 is the right narrow path, but restart-orphan coverage still needs to be proven.
