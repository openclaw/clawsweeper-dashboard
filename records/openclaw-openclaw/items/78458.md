## Summary

Keep this open: current main still has a source-level path where Dreaming narrative sessions use timestamped keys and rely on in-process `finally` cleanup, so a Gateway restart can still leave child sessions visible.

## What This Changes

The issue asks for Dreaming-spawned child sessions to be durably cleaned up when Gateway restarts during a Dreaming cycle.

## Best Possible Solution

Keep this issue as restart-specific evidence while maintainers land or replace https://github.com/openclaw/openclaw/pull/70464 and add regression coverage for restart-left `sessions.json` rows.

## Reproduction Assessment

Yes, at source level. Current main still creates timestamped Dreaming narrative session keys, relies on a `finally`-block `deleteSession`, and preserves live transcript-backed Dreaming rows in the scrubber.

## Solution Assessment

No complete fix is on main yet. Stable workspace/phase keys plus timestamped idempotency in https://github.com/openclaw/openclaw/pull/70464 is the right partial direction, but the restart-orphan path still needs durable cleanup coverage.
