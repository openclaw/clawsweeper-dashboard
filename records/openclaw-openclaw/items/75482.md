## Summary

This PR should close as a ClawHub/external-plugin item: it adds an optional OpenPaths model-provider integration, and current main already has documented provider-plugin, manifest, API-key auth, model-catalog, and ClawHub/npm install surfaces for this shape without a missing core API.

## What This Changes

The PR adds a bundled OpenPaths provider plugin with `OPENPATHS_API_KEY` onboarding, static auto/easy/medium/hard/autothink model catalog entries, OpenAI-compatible transport metadata, `/think` support, docs, labeler coverage, changelog coverage, and package metadata.

## Best Possible Solution

Keep OpenClaw core unchanged and have OpenPaths ship as an external provider plugin on ClawHub/npm using the existing provider-plugin contract, including ClawHub compatibility metadata; open a focused SDK issue only if implementation finds a concrete missing generic seam.
