## Summary

Keep open: the health-hook idea is useful, but this branch is not merge-ready because it emits a new `whatsapp` internal hook type through a typed helper whose current contract excludes that type, and the posted proof does not show one of the new health events actually firing.

## What This Changes

The PR adds background internal hook dispatch for WhatsApp Web watchdog timeout, scheduled reconnect, and terminal disconnect events, plus mocked e2e assertions.

## Best Possible Solution

Add a typed and documented WhatsApp health hook event contract or reuse an existing supported event family, keep dispatch bounded, cover the contract with focused tests, and provide redacted real hook logs for a triggered health event.

## Reproduction Assessment

Not applicable as a user bug, but the PR blocker is source-reproducible: the branch passes `"whatsapp"` to a helper whose current event-type union does not include it.

## Solution Assessment

No. The maintainable path is to define and document the hook event contract first, or choose an existing supported hook family, before emitting these events from the WhatsApp monitor.
