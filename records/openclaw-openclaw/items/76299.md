## Summary

Close as implemented on current main. OpenClaw now has global/per-agent `thinkingDefault` config plus per-model `agents.defaults.models[ref].params.thinking`, and the selected model path resolves those defaults at runtime; the proof commit is on `main` but not in `v2026.4.29`.

## What This Changes

Request a global and per-model `openclaw.json` mechanism for default reasoning/thinking levels when switching models.

## Best Possible Solution

Keep the current `thinkingDefault` plus per-model `params.thinking` contract as the supported config surface, with no additional duplicate reasoning-level key.

## Reproduction Assessment

Not applicable as a feature request. The current-main verification path is source/schema/docs/tests showing the requested global and per-model defaults are already wired into model selection.

## Solution Assessment

Yes. The existing thinking-default mechanism is the narrow maintainable solution because `reasoningDefault` is already used for reasoning visibility, while `thinkingDefault` controls provider reasoning effort.
