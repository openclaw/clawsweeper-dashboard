## Summary

Keep open: current main still does not make unmentioned Telegram group media available to a later mentioned turn. Main now has a shared history-media seam, but Telegram’s unaddressed group-media path returns before download and the no-mention history recorder remains text-only; the direct fix PRs are closed unmerged.

## What This Changes

The issue requests preserving actual Telegram group media from unmentioned messages in later mention-triggered history context instead of retaining only placeholder text.

## Best Possible Solution

Define and implement a bounded Telegram history-media policy that reuses HistoryEntry.media, preserves the no-error skip behavior, and only exposes recent local images to later mentioned turns under clear retention and visibility rules.

Yes, source-level reproduction is high confidence: current main returns before media download for unmentioned requireMention group media, and the no-mention history branch records only text fields. I did not run live Telegram proof in this read-only review.

## Solution Assessment

No, the original mediaPaths/mediaTypes sketch is not the best current shape because main now has HistoryEntry.media and an intentional skip-before-download behavior. The maintainable fix is a bounded Telegram policy using the shared history-media seam.
