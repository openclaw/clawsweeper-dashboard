## Summary

Keep open. Current main now has generic history-media support from https://github.com/openclaw/openclaw/pull/82068, but Telegram still either skips unmentioned `requireMention` group media before download or records no-mention history as text-only, so this Telegram-specific behavior is not implemented on main.

## What This Changes

The issue requests Telegram group history to preserve usable media references for unmentioned group media so a later bot mention can inspect earlier images.

## Best Possible Solution

Have the Telegram owner decide the privacy/retention policy; if accepted, wire Telegram dropped-message history into the existing `HistoryEntry.media`/`recordWithMedia` contract with bounded local image retention, stale filtering, focused tests, and live Telegram proof.

## Reproduction Assessment

Yes, at source level. Current main skips unmentioned `requireMention` Telegram group media before download, and the downstream no-mention history path records only text fields rather than `HistoryEntry.media`.

## Solution Assessment

Unclear until Telegram policy is decided. The older `mediaPaths`/temp-file touch approach is not the best shape now that current main has a bounded generic history-media contract, but Telegram still needs an approved retention policy before using it.
