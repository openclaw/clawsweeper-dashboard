## Summary

Keep open: current main and the latest release still do not make unmentioned Telegram group images available to later history context; the media path now skips download before mention handling, and the text-history branch still records no media metadata.

## What This Changes

Report that Telegram group images sent without mentioning the bot are preserved only as placeholder history text, so a later mentioned turn cannot inspect the image.

## Best Possible Solution

Define a maintainer-approved Telegram history-media policy, then reuse HistoryEntry.media and recordWithMedia with bounded retention, visibility filtering, path redaction, focused tests, and live Telegram proof while preserving the no-spam default behavior.

## Reproduction Assessment

Yes, at source level: current main returns before media download for unmentioned requireMention group images, and the downstream no-mention history branch records only text fields. I did not run live Telegram proof in this read-only review.

## Solution Assessment

No, the old mediaPaths/mediaTypes proposal is not the best current shape because main now has a shared HistoryEntry.media contract. The maintainable fix is to use that seam after maintainers decide the retention and visibility policy.
