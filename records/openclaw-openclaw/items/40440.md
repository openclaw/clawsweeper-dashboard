## Summary

Keep open: current main and v2026.6.10 still record unmentioned Telegram group media history as text-only, so the later mentioned turn can see the placeholder but not the image. Main now has a shared history-media seam, but Telegram does not populate it for this path and the direct fix PRs are closed unmerged.

## What This Changes

The issue requests preserving media from unmentioned Telegram group messages in recent group history so a later mentioned turn can inspect the image rather than only seeing a placeholder.

## Best Possible Solution

Define a bounded Telegram history-media policy, then populate HistoryEntry.media through the shared history window without regressing the current silent skip behavior for unaddressed group media.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main returns before media download for unmentioned requireMention group media, and the no-mention history branch records only text fields. I did not run live Telegram proof in this read-only review.

## Solution Assessment

No, the original mediaPaths/mediaTypes sketch is not the best current shape because main now has HistoryEntry.media and an intentional skip-before-download behavior. The maintainable fix is a bounded Telegram policy using the shared history-media seam.
