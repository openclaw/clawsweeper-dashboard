## Summary

Keep open. Current main has shared history-media support, but the Telegram path still skips unmentioned `requireMention` group media before download and records skipped group history as text-only, so the reported Telegram behavior remains unresolved and needs a privacy/retention policy decision before implementation.

## What This Changes

The issue requests preserving Telegram group media from unmentioned messages so a later mention can access the image data from recent chat history.

## Best Possible Solution

Have the Telegram area reviewer decide the unmentioned-media retention policy; if accepted, wire Telegram into the existing `HistoryEntry.media` and `recordWithMedia` contract with bounded local image retention, focused tests, and live Telegram proof.

## Reproduction Assessment

Yes, at source level. Current main skips unmentioned `requireMention` Telegram group media before download, and the downstream skipped-history path records only text fields rather than `HistoryEntry.media`.

## Solution Assessment

Unclear until Telegram policy is decided. The older `mediaPaths` and temp-file touch approach is not the best shape now that main has a generic bounded history-media contract, but Telegram still needs approval before using it for unmentioned group media.
