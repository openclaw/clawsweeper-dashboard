## Summary

Keep open. Current main and the latest shipped release still record skipped Telegram group media history as text-only entries and later replay only current-message plus reply media, while the two related fix attempts are closed unmerged.

## What This Changes

Telegram group messages with media sent without mentioning the bot should retain usable media references in recent group history so a later mention can inspect them.

## Best Possible Solution

Add an additive media-bearing pending-history contract, preserve skipped Telegram media for a bounded lifetime, filter surviving refs, and merge those refs into later mention context on the current extensions/telegram paths.

## Reproduction Assessment

Yes, source-level. In a Telegram requireMention group, downloaded allMedia reaches resolveTelegramInboundBody, the no-mention skip branch records only rawBody history, and later context construction builds MediaPaths only from current or replied-to media.

## Solution Assessment

Yes for a narrow repair. The maintainable path is optional mediaPaths/mediaTypes on pending history plus bounded retention and surviving-path filtering, implemented against current extensions/telegram code rather than reviving either closed PR branch verbatim.

In Telegram requireMention groups, media-only or media-bearing messages sent without mentioning the bot are downloaded but recorded in pending group history as placeholder text only; a later mentioned message receives no usable media refs for that history entry. Do not add a broad core Telegram policy, do not introduce unbounded cache retention or unbounded fs.utimes work, and do not inject stale paths. Add focused regression tests and a CHANGELOG.md fix entry if the resulting PR is user-visible.
