## Summary

Keep open. Current main still cannot preserve Telegram history media for unmentioned `requireMention` groups: the Telegram media handler skips downloads before media resolution, and the skipped-history branch records only text, while the current shared history-media contract is available but not wired into this Telegram path; resolving it needs a Telegram/security policy decision for retaining unmentioned group media.

## What This Changes

The issue requests preserving media from unmentioned Telegram group-history messages so a later mentioned turn can inspect the image instead of only seeing a placeholder.

## Best Possible Solution

Have a Telegram/security area reviewer approve the unmentioned-media retention policy; if approved, implement bounded Telegram-owned history-media recording through `HistoryEntry.media`/`recordWithMedia`, likely behind an explicit retention opt-in, with stale-file filtering and live Telegram proof.

## Reproduction Assessment

Yes, at source level. Current main skips unmentioned `requireMention` Telegram group media before `resolveMedia`, and the skipped-history branch records a text-only entry without `HistoryEntry.media`.

## Solution Assessment

Unclear until the Telegram retention policy is decided. The old `mediaPaths`/`mediaTypes` approach is superseded by the current `HistoryEntry.media` and `recordWithMedia` contract, but applying it to unmentioned group media needs owner/security approval.
