## Summary

Keep open. Current main still records skipped Telegram group media messages as text-only pending history, and later @mention replay only carries current/replied-to media; the related fix PRs are useful context but are closed unmerged.

## What This Changes

Preserve Telegram group media references from non-mentioned history messages so a later @mention can include those media files.

## Best Possible Solution

Add an additive media-bearing pending-history contract, preserve bounded safe local media refs for skipped Telegram group messages, and merge those refs into later mention context.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: in a requireMention Telegram group, a media message without a mention records pending history without media fields, and a later mention builds `MediaPaths` only from current/replied-to media.

## Solution Assessment

Yes. The narrow maintainable direction is optional `mediaPaths`/`mediaTypes` on pending history plus bounded retention/replay merging; current main does not solve the user-visible problem.
