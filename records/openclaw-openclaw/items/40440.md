## Summary

Keep open: current main and the latest release still record Telegram mention-skipped group history as text-only entries, while the shared history-media pipeline exists but is not wired into this Telegram skip path. The prior Telegram fix PRs are closed unmerged and surfaced retention/path-exposure concerns, so this needs maintainer policy review before a new automated fix.

## What This Changes

The issue reports that Telegram group media sent without mentioning the bot is stored in later group history only as placeholder text, leaving the later mentioned turn without usable media attachments.

## Best Possible Solution

Keep this issue open and implement a bounded, security-reviewed Telegram fix that records skipped group image media through the existing `HistoryEntry.media`/`recordWithMedia` contract and preserves safe retention/path redaction boundaries.

## Reproduction Assessment

Yes, at source level. With Telegram `requireMention: true` and recent group-history context enabled, the current no-mention branch records only sender/body/timestamp/messageId even when `allMedia` was downloaded, so the later mentioned turn has only placeholder history text.

## Solution Assessment

No, the issue's original `mediaPaths`/`mediaTypes` sketch is no longer the best shape. Current main has a safer shared `HistoryEntry.media` and recent-history attachment pipeline, but maintainers still need to decide the retention and path-redaction policy before wiring Telegram into it.
