## Summary

Keep open. Current main still has a source-reproducible Telegram gap: unmentioned `requireMention` group media is skipped before download and the skipped-history entry is text-only, while the shared history-media contract now exists but is not wired into this Telegram path.

## What This Changes

The issue asks Telegram group history to preserve media from unmentioned group messages so a later mentioned turn can inspect the files.

## Best Possible Solution

Have Telegram/security ownership decide the unmentioned-media retention policy; if approved, implement bounded Telegram-owned history-media recording through `HistoryEntry.media`/`recordWithMedia`, likely as an explicit opt-in with stale-file filtering and live Telegram proof.

## Reproduction Assessment

Yes, at source level. Current main skips unmentioned `requireMention` Telegram group media before `resolveMedia`, and the skipped-history branch records only a text history entry without `HistoryEntry.media`.

## Solution Assessment

Unclear until the Telegram retention policy is decided. The maintainable implementation should reuse the current `HistoryEntry.media` and `recordWithMedia` contract, not the older `mediaPaths`/`mediaTypes` branch shape.
