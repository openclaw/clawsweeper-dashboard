## Summary

Keep open. Current main still has a source-reproducible Telegram gap: unmentioned `requireMention` group media is skipped before download, the skipped-history entry is text-only, and the newer shared history-media contract is not wired into this Telegram path.

## What This Changes

The issue requests preserving Telegram group media from unmentioned history messages so a later mentioned turn can access the image data instead of only `<media:image>` placeholder text.

## Best Possible Solution

Have a Telegram area reviewer decide the unmentioned-media retention policy; if approved, wire Telegram into the existing `HistoryEntry.media` and `recordWithMedia` contract with bounded local image retention, stale-file filtering, and live Telegram proof.

## Reproduction Assessment

Yes, at source level. Current main skips unmentioned `requireMention` Telegram group media before download, then records the skipped group history as a text-only entry without `HistoryEntry.media`.

## Solution Assessment

Unclear until Telegram policy is decided. The old `mediaPaths`/`mediaTypes` approach is no longer the best shape because current main has a generic bounded history-media contract, but Telegram still needs approval before using it for unmentioned group media.
