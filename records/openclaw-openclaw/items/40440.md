## Summary

Keep open: current main and v2026.6.6 still leave unmentioned Telegram group media unavailable to later history context, and the behavior now intersects an intentional skip-before-download path that needs maintainer security/product direction.

## What This Changes

Preserve media from unmentioned Telegram group messages in pending group history so a later mentioned turn can inspect the images.

## Best Possible Solution

Define an opt-in Telegram history-media policy that reuses the shared HistoryEntry.media contract with bounded retention, visibility filtering, path redaction, regression tests, and live Telegram proof.

Yes, at source level: current main skips unmentioned requireMention Telegram group media before download and the skipped-message history entry records only text fields. I did not run a live Telegram group reproduction in this read-only pass.

## Solution Assessment

No. The original mediaPaths/mediaTypes sketch is not the best shape now that current main has a shared HistoryEntry.media contract; the right fix needs a maintainer-approved retention and visibility policy around that contract.
