## Summary

Keep open: current main and the latest release still do not make unmentioned Telegram group media available to later history context. The remaining work needs maintainer product/security direction because current Telegram code intentionally skips unaddressed requireMention media before download while the no-mention history branch records only text fields.

## What This Changes

Preserve enough Telegram group-history media metadata so a later mentioned turn can inspect media sent earlier without mentioning the bot.

## Best Possible Solution

Define a maintainer-approved Telegram history-media policy, then reuse HistoryEntry.media/recordWithMedia with bounded local-image retention, visibility filtering, path redaction, regression tests, and live Telegram proof.

## Reproduction Assessment

Yes, at source level: current main skips unmentioned requireMention group media before download, while the no-mention history branch records only text fields. I did not run live Telegram proof in this read-only review.

## Solution Assessment

No, the original mediaPaths/mediaTypes sketch is no longer the best current shape because main now has a shared HistoryEntry.media contract. A maintainable fix should reuse that seam after maintainers decide the retention and visibility policy.
