## Summary

Keep open. Current main still skips unmentioned Telegram `requireMention` group media before download and records the skipped history entry as text only, so the reported later-mentioned media context is still unavailable; the remaining blocker is a Telegram/security policy decision about retaining unmentioned group media.

## What This Changes

Preserve media from unmentioned Telegram group messages in recent group history so a later mentioned turn can use the attachments.

## Best Possible Solution

Have a Telegram/security area reviewer decide the unmentioned-media retention policy; if approved, implement it through `HistoryEntry.media` and `recordWithMedia` with bounded retention, stale-file filtering, focused tests, and live Telegram proof.

## Reproduction Assessment

Yes, at source level. Current main skips unmentioned `requireMention` Telegram group media before `resolveMedia`, and the skipped-history branch records text-only entries without `HistoryEntry.media`.

## Solution Assessment

Unclear until Telegram policy is decided. The old `mediaPaths`/`mediaTypes` approach is superseded by the current generic `HistoryEntry.media` contract, but using it for unmentioned group media needs owner and security review.
