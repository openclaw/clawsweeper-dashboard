## Summary

Keep open. Current main still records skipped Telegram group history as text-only entries even though Telegram media has already been downloaded, and the two related fix attempts are closed unmerged with no canonical replacement found.

## What This Changes

Telegram group messages with media that are skipped for missing bot mention should preserve media references in recent group history for later mentioned turns.

## Best Possible Solution

Add an additive media-bearing pending-history contract with bounded retention and surviving-file filtering, then replay surviving history media into later Telegram mention context.

## Reproduction Assessment

Yes, source-level with high confidence. The current Telegram requireMention/no-mention path receives downloaded allMedia, records text-only history, and later builds MediaPaths only from current or replied-to media.

## Solution Assessment

Yes for a narrow repair. The maintainable path is optional mediaPaths/mediaTypes on pending history, bounded media lifetime handling, stale-path filtering, and replay merging on the current extensions/telegram paths.
