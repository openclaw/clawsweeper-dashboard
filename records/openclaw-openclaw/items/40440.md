## Summary

Keep open. Current main still records mention-skipped Telegram group history as text-only while downloaded media refs remain only on the skipped turn, and both prior fix PRs are closed unmerged; the fix needs maintainer review because automatic history-media injection overlaps the open stale-media/privacy boundary.

## What This Changes

The issue asks Telegram group history to retain media file references for no-mention messages so a later @mention can inspect those images.

## Best Possible Solution

Implement a Telegram-owned history-media contract that preserves observed media references for mention-skipped group messages, filters stale files, and exposes them only under a bounded context policy aligned with current-turn media privacy.

## Reproduction Assessment

Yes, at source level. Current main downloads Telegram media into `allMedia`, records a mention-skipped group history entry without those refs, and later builds `MediaPaths` only from current/reply media; this review did not run a live Telegram group probe.

## Solution Assessment

Partly. Preserving media on the Telegram history/context path is the right owner boundary, but the final fix should not blindly activate all history media without stale-file filtering, retention bounds, and a policy that accounts for non-current image contamination.
