## Summary

Keep open. Current main still downloads Telegram media for inbound group messages, but the requireMention/no-mention skip path stores only text history and later replay builds MediaPaths only from the current turn or replied-to media; the related fix PRs #40470 and #40490 are closed unmerged.

## What This Changes

Preserve Telegram media paths and types from no-mention group messages in pending history so a later mention can inspect that media.

## Best Possible Solution

Add an additive media-bearing pending-history contract, retain skipped Telegram media for a bounded safe lifetime, and merge those refs into later mention context and structured history.

## Reproduction Assessment

Yes, source-level. In a Telegram requireMention group, a media message without a bot mention reaches resolveTelegramInboundBody with allMedia, then the skip branch records only rawBody before returning null; later payload construction omits history media from MediaPaths.

## Solution Assessment

Yes, for a narrow repair. The maintainable direction is optional mediaPaths/mediaTypes on pending history plus bounded retention or touching of downloaded refs and replay merging; current main does not implement that path.
