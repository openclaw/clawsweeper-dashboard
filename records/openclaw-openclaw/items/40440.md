## Summary

Keep open. Current main still records Telegram no-mention group history as text-only entries, while the related fix attempts at https://github.com/openclaw/openclaw/pull/40470 and https://github.com/openclaw/openclaw/pull/40490 are closed unmerged.

## What This Changes

Preserve downloaded Telegram media refs from unmentioned group messages in pending history so a later mention can supply those images to the agent.

## Best Possible Solution

Add an additive media-bearing pending-history contract, keep skipped Telegram media refs alive for a bounded safe lifetime, filter missing refs, and merge surviving history media into later mention context on current plugin paths.

## Reproduction Assessment

Yes, source-level. In a Telegram requireMention group, downloaded allMedia reaches resolveTelegramInboundBody, the no-mention skip branch records only rawBody history, and later context construction builds MediaPaths only from current or replied-to media.

## Solution Assessment

Yes for a narrow repair. The maintainable path is optional mediaPaths/mediaTypes on pending history plus bounded retention/surviving-path filtering and replay merging, implemented against current extensions/telegram paths rather than the closed legacy src/telegram PR branches.
