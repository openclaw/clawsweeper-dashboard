## Summary

Keep open. Current main still has no media-bearing Telegram history contract, and the newer requireMention behavior intentionally skips unmentioned non-audio group media before download, so the remaining request needs maintainer product/privacy direction rather than cleanup closure.

## What This Changes

The issue asks Telegram group history to retain media references from messages sent without mentioning the bot so a later mention can use those images or files.

## Best Possible Solution

Decide the Telegram policy first; if maintainers want this behavior, add a bounded Telegram-owned observed-history media contract that stores durable references only under the approved policy, filters missing or stale files before injection, and avoids treating unrelated older media as current-turn input.

## Reproduction Assessment

Yes, at source level. Current main either skips unmentioned non-audio group media before download under requireMention, or records downstream no-mention history as text-only and later builds MediaPaths only from current and reply media.

## Solution Assessment

Unclear. Preserving history media is technically feasible, but the suggested mediaPaths/touch-temp-files approach is not sufficient without a maintainer decision on the requireMention privacy boundary and bounded retention semantics.
