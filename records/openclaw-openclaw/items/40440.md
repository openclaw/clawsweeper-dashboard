## Summary

Keep open. Current main and v2026.5.22 still skip unmentioned Telegram `requireMention` group media before download and record skipped history as text-only; the shared `HistoryEntry.media` contract exists, but wiring it into this Telegram path needs an explicit security/product decision.

## What This Changes

Request to preserve media from unmentioned Telegram group messages so later mentioned turns can inspect recent group-history images.

## Best Possible Solution

Have a Telegram/security area owner approve the unmentioned-media retention policy; if approved, implement bounded Telegram-owned history-media recording through `HistoryEntry.media`, preferably as an explicit opt-in that preserves current no-download defaults.

## Reproduction Assessment

Yes, at source level. Current main skips unmentioned `requireMention` Telegram group media before `resolveMedia`, and the skipped-history branch records a text-only `HistoryEntry` without media.

## Solution Assessment

Unclear until the Telegram retention policy is decided. The old `mediaPaths`/`mediaTypes` approach is superseded by the current `HistoryEntry.media` contract, but applying it here needs owner/security approval and likely an opt-in retention setting.
