## Summary

Keep open. Current main and v2026.5.22 still skip unmentioned Telegram requireMention group media before download and record skipped history as text-only; the shared history-media contract exists, but Telegram needs an explicit retention/security policy before wiring it in.

## What This Changes

The issue asks Telegram group history to preserve media attachments from unmentioned requireMention messages so a later mentioned turn can inspect them.

## Best Possible Solution

Have a Telegram/security owner approve the unmentioned-media retention policy; if approved, implement bounded Telegram-owned history-media recording through HistoryEntry.media, preferably behind an explicit opt-in that preserves current no-download defaults.

## Reproduction Assessment

Yes, at source level. Current main skips unmentioned requireMention Telegram group media before resolveMedia, and the skipped-history branch records only text fields without media.

## Solution Assessment

Unclear until the Telegram retention policy is decided. The prior mediaPaths/mediaTypes approach is superseded by the current HistoryEntry.media contract, but applying it here needs owner/security approval and likely an opt-in retention setting.
