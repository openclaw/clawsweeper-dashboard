## Summary

Keep open. Current main still records mention-skipped Telegram group media as text-only history and later builds `MediaPaths` only from current-turn and reply media; both linked fix PRs are closed unmerged, so the remaining work needs maintainer review for a retention/privacy-safe history-media design.

## What This Changes

Telegram group messages sent without mentioning the bot should preserve downloaded media references so a later mention can let the agent inspect those recent images.

## Best Possible Solution

Implement a Telegram-owned observed-history media contract that preserves recent skipped-message media only within bounded retention, filters stale files before prompt/media injection, and avoids treating unrelated old images as current-turn input.

## Reproduction Assessment

Yes, at source level. Current main downloads Telegram media into `allMedia`, records the no-mention group history entry without those refs, and later builds `MediaPaths` from only current and reply media; this review did not run a live Telegram group probe.

## Solution Assessment

Partly. Preserving media on the Telegram history path is the right direction, but the fix should include stale-file filtering, bounded retention, and a policy that prevents non-current image contamination.
