## Summary

Keep open. Current main and v2026.6.6 still do not preserve Telegram unmentioned group media for later mention: one path skips unaddressed media before download, and the mention-skip history record path writes only text metadata; the closed candidate PRs did not merge and their comments show retention/path exposure needs owner direction.

## What This Changes

The issue requests that Telegram group history preserve media metadata from unmentioned group messages so a later mention can let the agent inspect those images.

## Best Possible Solution

Land a maintainer-approved Telegram fix that uses the existing HistoryEntry.media contract, keeps retention bounded, avoids prompt-facing local path leakage, and proves both unmentioned-media recording and later mentioned-turn replay.

## Reproduction Assessment

Yes at source level: current main either skips unaddressed requireMention media before download or records the skipped history entry without media. I did not run a live Telegram group reproduction in this read-only review.

## Solution Assessment

No implemented fix is on main. The best path is not the old parallel mediaPaths/mediaTypes shape verbatim, but a bounded use of the current HistoryEntry.media contract with explicit retention and prompt-redaction rules.
