## Summary

Keep open. Current `main` still records Telegram no-mention group history as text-only entries and later builds media context only from current or replied-to media, while the two linked fix attempts are closed unmerged: https://github.com/openclaw/openclaw/pull/40470 and https://github.com/openclaw/openclaw/pull/40490.

## What This Changes

Telegram group history should preserve media references for non-mentioned media messages so a later mentioned message can include the actual files.

## Best Possible Solution

Implement an additive media-bearing pending-history path for Telegram group skips, keep skipped media alive only within a bounded safe lifetime, filter missing refs, and replay surviving refs into later mention context.

## Reproduction Assessment

Yes, source-level. In a Telegram requireMention group, downloaded `allMedia` reaches `resolveTelegramInboundBody`, the no-mention skip branch records only `rawBody` history, and later context construction builds `MediaPaths` only from current or replied-to media.

## Solution Assessment

Yes for a narrow repair, but not by merging either old branch verbatim. The maintainable fix is optional media refs on pending history plus bounded retention, stale-path filtering, and replay merging on the current `extensions/telegram` paths.
