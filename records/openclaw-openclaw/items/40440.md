## Summary

Keep open. Current main still has a source-reproducible Telegram gap: unmentioned `requireMention` group media is skipped before download and the skipped-history entry is text-only, while the newer shared history-media contract exists but is not wired into this Telegram path.

## What This Changes

The issue asks Telegram group history to preserve media from unmentioned group messages so a later mention can attach the recent image data to the agent turn.

## Best Possible Solution

Have a Telegram area reviewer decide the unmentioned-media retention policy; if approved, wire Telegram into the existing `HistoryEntry.media`/`recordWithMedia` or preflight-media contract with bounded local image retention, stale-file filtering, and live Telegram proof.

## Reproduction Assessment

Yes, at source level. Current main skips unmentioned `requireMention` Telegram group media before download, then records skipped group history as text-only entries without `HistoryEntry.media`.

## Solution Assessment

Unclear until Telegram policy is decided. The older `mediaPaths` and temp-file touch approach is not the best shape now that main has a generic bounded history-media contract, but Telegram still needs approval before using it for unmentioned group media.
