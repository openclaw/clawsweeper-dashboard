## Summary

Keep open. Current main still records skipped Telegram group media messages as text-only pending history, and later mention replay only carries current-turn or replied-to media; the related fix PRs are closed unmerged.

## What This Changes

Telegram group messages with media that are skipped by mention gating should preserve media references in pending history so a later bot mention can inspect them.

## Best Possible Solution

Add an additive media-bearing pending-history contract, preserve bounded local media refs for skipped Telegram group messages, and merge those refs into later mention context with focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: in a requireMention Telegram group, a media message without a mention records pending history without media fields, and a later mention builds MediaPaths only from current/replied-to media.

## Solution Assessment

Yes. The narrow maintainable direction is optional mediaPaths/mediaTypes on pending history plus bounded retention/replay merging; current main does not solve the user-visible problem.
