## Summary

Keep open. Current main still stores mention-skipped Telegram group media as text-only pending history and later builds media context only from current-turn and reply media; the linked fixes are closed unmerged, so the remaining work needs a retention/privacy-safe Telegram design.

## What This Changes

The issue asks for Telegram no-mention group history entries with media to preserve usable media references so a later mentioned request can inspect the earlier image or file.

## Best Possible Solution

Add a Telegram-owned observed-history media contract that stores bounded durable references for recently skipped group media, filters missing or stale files before prompt/media injection, and keeps unrelated old media out of current-turn context.

## Reproduction Assessment

Yes, at source level. Current main downloads Telegram media before mention gating, records skipped group history without those media refs, and later builds media context only from the current message and replied-to message.

## Solution Assessment

Partly. Preserving media on the Telegram history path is the right direction, but the fix should include bounded retention, stale-file filtering, and safeguards against treating old group media as current-turn input.
