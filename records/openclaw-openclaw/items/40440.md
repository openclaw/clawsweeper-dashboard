## Summary

Keep open. Current main still records mention-skipped Telegram group media history as sender/body/timestamp/messageId only, and later context builds MediaPaths only from current/reply media, so the reported placeholder-only history path is still present; the two linked fix PRs are closed unmerged.

## What This Changes

The issue asks Telegram group history to preserve media file paths/types for media sent without mentioning the bot so a later mention can use those attachments.

## Best Possible Solution

Fix Telegram group history to carry bounded, validated media references for mention-skipped messages and merge only surviving refs into later mentioned-turn context, with focused regression coverage.

## Reproduction Assessment

Yes for source-level reproduction: send a Telegram group media message that is skipped by mention gating, and current main records only body metadata while later context reads only current/reply media. This was not live-reproduced against Telegram in this read-only pass.

## Solution Assessment

Yes, the mediaPaths/mediaTypes direction is the narrowest maintainable fix, but the new implementation should not copy the closed PRs verbatim if that would extend TTL indefinitely or create unbounded cache growth.
