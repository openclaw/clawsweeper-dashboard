## Summary

Keep open: current main still records Telegram mention-skipped group history as sender/body/timestamp/messageId only, while the shared media-aware history path exists but is not wired into this Telegram branch. The latest release tag also has the same text-only skip path, and the closed unmerged fix attempts raised retention/path-exposure concerns that need maintainer security/product review before a new fix lane.

## What This Changes

Telegram group messages with media sent without mentioning the bot should preserve usable media metadata in recent group history for a later mentioned turn.

## Best Possible Solution

Keep this issue open and implement a bounded, security-reviewed Telegram fix that records skipped group image media through the existing HistoryEntry.media and recordWithMedia contract without leaking local paths or extending retention indefinitely.

## Reproduction Assessment

Yes, at source level. With Telegram requireMention enabled and recent group-history context, the current no-mention branch records only text fields even when allMedia contains downloaded image media, so the later mentioned turn cannot resolve entry.media for that image.

## Solution Assessment

No, the original parallel mediaPaths/mediaTypes sketch is no longer the best shape. Current main has a safer shared HistoryEntry.media and recordWithMedia contract, but maintainers still need to decide the retention and path-redaction policy before wiring Telegram into it.
