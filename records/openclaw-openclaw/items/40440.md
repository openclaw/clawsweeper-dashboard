## Label Justifications

- `P2`: The issue affects Telegram group media context for a real user flow, but it is scoped to one channel and requires policy-sensitive review before implementation.
- `impact:message-loss`: The later mentioned Telegram turn receives only a placeholder and cannot access the historical image attachment.
- `impact:session-state`: The missing media belongs to recent group history that is intended to inform later agent context.

## Summary

Keep open. Current main and the latest release still do not preserve Telegram media from unmentioned `requireMention` group messages: the media path is skipped before download and the pending history entry remains text-only, while the shared history-media contract now exists but is not wired into Telegram; fixing it needs a Telegram privacy/retention decision first.

## What This Changes

The issue asks Telegram to preserve media metadata/files from unmentioned `requireMention` group messages so a later mentioned turn can inspect recent images.

## Best Possible Solution

Have Telegram/security owners decide the unmentioned-media retention policy; if approved, add a bounded Telegram-owned opt-in path that records recent local image media through `HistoryEntry.media`/`recordWithMedia` and proves the live supergroup flow.

## Reproduction Assessment

Yes, at source level. Current main skips unmentioned `requireMention` Telegram group media before `resolveMedia`, and the skipped-history branch records only text without `HistoryEntry.media`.

## Solution Assessment

Unclear until the Telegram retention policy is decided. The maintainable implementation should use the existing `HistoryEntry.media`/`recordWithMedia` contract, likely behind an explicit opt-in, rather than reviving the older `mediaPaths`/`mediaTypes` shape.
