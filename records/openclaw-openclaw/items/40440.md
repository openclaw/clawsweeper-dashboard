## Summary

Keep open: current main still leaves unmentioned Telegram group media unavailable to later history context, and the remaining fix needs maintainer product/security direction because current code intentionally skips those media downloads before mention gating.

## What This Changes

The issue reports that unmentioned Telegram group media is reduced to placeholder text in later chat history, leaving agents unable to inspect the media when mentioned afterward.

## Best Possible Solution

Define a maintainer-approved Telegram history-media policy that can attach bounded, local recent images through the shared `HistoryEntry.media` seam without leaking paths or undoing no-mention media-download suppression.

## Reproduction Assessment

Yes at source level: current main returns before media download for unmentioned `requireMention` group media, and the no-mention history branch records only text fields. I did not run live Telegram proof in this read-only review.

## Solution Assessment

No, the original `mediaPaths`/`mediaTypes` sketch is not the best current shape because main now has a shared `HistoryEntry.media` contract and an intentional skip-before-download behavior. A maintainable fix should reuse that seam after maintainers decide the retention and visibility policy.
