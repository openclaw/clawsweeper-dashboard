## Summary

Keep open. Current main still records skipped Telegram group media messages as text-only pending history, and later mention replay only includes current-turn or replied-to media; the related fix PRs are closed unmerged, so the reported behavior remains valid.

## What This Changes

Preserve Telegram media file paths and content types for mention-gated group history entries so a later @mention can access recently sent images.

## Best Possible Solution

Add an additive media-bearing pending-history contract, retain skipped Telegram media with bounded safe local lifetime, and merge those refs into later mention context and structured history.

Yes. A high-confidence source-level reproduction exists: in a requireMention Telegram group, send media without mentioning the bot, then later mention the bot; the skip path records only placeholder text, and replay builds `MediaPaths` without history media.

## Solution Assessment

Yes. The narrow maintainable direction is optional `mediaPaths`/`mediaTypes` on pending history plus bounded retention/replay merging; current main does not solve the user-visible bug.
