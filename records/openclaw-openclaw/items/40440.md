- `impact:message-loss`: Unmentioned Telegram group media is not available to the later mentioned turn even when recent group history is the relevant context.

## Summary

Keep open: current main and the latest release still do not preserve usable media for unmentioned Telegram group-history entries. Current main has shared history-media plumbing, but the Telegram unmentioned-media path either records text-only history or skips media download before any history/media attachment can be built, so the central user problem remains.

## What This Changes

The issue asks OpenClaw to preserve media references from unmentioned Telegram group messages so a later mentioned Telegram turn can inspect the prior image or file.

## Best Possible Solution

Wire Telegram recent group-history media into the shared HistoryEntry.media and agent-turn attachment pipeline only under an explicit trusted/opt-in policy, with bounded retention, path redaction, and focused tests for pre-download skip, history replay, and later mentioned turns.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: with Telegram requireMention group media and a later mentioned turn, the current unmentioned media path does not put any usable media entry into InboundHistory; I did not run live Telegram proof in this read-only review.

## Solution Assessment

No. The original mediaPaths/mediaTypes sketch is not the best current shape because main now has a shared HistoryEntry.media pipeline; the safer solution is to wire Telegram into that pipeline with an explicit retention and privacy policy.
