## Summary

Current main provides a generic, bounded recent-history media pipeline, but Telegram’s no-mention branch still omits resolved media when recording history. The reported behavior therefore remains actionable, while the privacy implications of retaining unmentioned group content require explicit maintainer approval.

## What This Changes

Preserve image attachments from unmentioned Telegram group messages so a later mentioned turn can inspect them through recent inbound history.

## Best Possible Solution

Populate `HistoryEntry.media` from the already-resolved Telegram image facts in the no-mention record call, rely on the existing bounded recent-history attachment pipeline, and add a focused skip-then-mention regression test without adding configuration or new storage paths.

## Reproduction Assessment

Yes—source inspection gives a high-confidence path on current main: resolved Telegram media reaches the no-mention branch, but the recorded history entry omits `media`. A live Telegram reproduction was not performed.

## Solution Assessment

Yes, with a narrower shape than the earlier PRs: use the existing generic `HistoryEntry.media` and bounded recent-history image pipeline rather than parallel fields or a persistent cache directory.
