## Summary

Keep open: current main still does not let a later Telegram mention inspect media from an earlier unmentioned group message; current code now skips unaddressed `requireMention` media before download and the group-history recorders that do run remain text-only, so this needs a maintainer security/product decision rather than cleanup.

## What This Changes

The issue requests preserving unmentioned Telegram group media as recent group-history context so a later mention can inspect the image instead of receiving only text or no attachment.

## Best Possible Solution

Maintainers should decide whether Telegram recent group history may retain bounded unmentioned images; if yes, wire the approved Telegram path through the existing `HistoryEntry.media` and `recordWithMedia()` seam with retention, visibility, and live Telegram proof.

## Reproduction Assessment

Yes, at source level. Current main either skips unaddressed `requireMention` media before download or records group history with text-only `record(...)`, so a later mention has no historical image attachment to inspect.

## Solution Assessment

Unclear until maintainers settle the security/product policy. If retaining unmentioned media is approved, the existing shared `HistoryEntry.media` and `recordWithMedia()` contract is the narrowest implementation shape; blindly restoring media downloads would conflict with the current no-download fix for unaddressed groups.
