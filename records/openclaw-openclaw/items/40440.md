## Summary

Keep this open. Current main still records skipped Telegram group media messages as text-only history entries and later rebuilds `InboundHistory`/`MediaPaths` without those prior media files; the two linked fix PRs were closed unmerged, and their review discussion confirms the useful direction but also flagged media-retention concerns that need a careful follow-up.

## What This Changes

The issue asks Telegram group history entries to retain downloaded media paths/types so a later bot mention can access prior images.

## Best Possible Solution

Implement bounded Telegram history media preservation: carry safe media path/type metadata through `HistoryEntry`, replay surviving recent media into the agent media context, clean or expire retained files predictably, and cover the no-mention-then-mention flow with focused tests plus real Telegram proof.

## Reproduction Assessment

Yes, by source inspection. A no-mention Telegram group message with media records only text history, and a later mentioned message only replays text history plus current/reply media; I did not run a live Telegram E2E.

## Solution Assessment

No, current main is not the right end state. The linked mediaPaths/mediaTypes direction is sensible, but the best fix needs a bounded retention and cleanup design rather than an unbounded cache or blind TTL refresh.
