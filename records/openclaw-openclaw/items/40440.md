## Summary

Keep open. Current main still stores skipped Telegram group media messages as text-only pending history, so a later @mention can see the placeholder but cannot receive the earlier image path; the linked fix PRs are useful context but both are closed unmerged.

## What This Changes

Preserve downloaded media references from unmentioned Telegram group messages so later mentioned turns can include those media files in agent context.

## Best Possible Solution

Add a backward-compatible media-bearing pending-history contract, preserve bounded local media references for skipped Telegram group messages, merge those references into later `MediaPaths`/`MediaTypes` during mention replay, and cover the image-without-mention followed by @mention flow with focused Telegram and shared-history tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: in a requireMention Telegram group, a media message without a mention records pending history without media fields, and a later mention builds `MediaPaths` only from current/replied-to media.

## Solution Assessment

Yes. The narrow maintainable solution is an additive `mediaPaths`/`mediaTypes` history extension plus bounded retention/replay merging; current main does not solve the user-visible problem.
