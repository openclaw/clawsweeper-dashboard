## Summary

Keep open. Current main still drops usable media refs when storing no-mention Telegram group messages for history, and both related fix PRs are closed unmerged, so the reported behavior remains a valid narrow bug.

## What This Changes

Preserve Telegram group media references in pending history so later @mentions can access images or files sent without mentioning the bot.

## Best Possible Solution

Add an additive media-bearing pending-history contract, retain skipped Telegram media with bounded safe local lifetime, and merge those refs into later mention context and structured history with focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence source-level path exists: in a requireMention Telegram group, a media message without a bot mention enters the skip branch, stores only rawBody in groupHistories, and later replay builds MediaPaths only from current or replied-to media.

## Solution Assessment

Yes. The narrow maintainable solution is optional mediaPaths/mediaTypes on pending history plus bounded retention or touching of downloaded refs and replay merging; current main does not implement that path.
