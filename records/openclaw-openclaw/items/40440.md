## Summary

Keep open. Current main still does not make unmentioned Telegram group media available to later mentioned turns; the clean implementation shape exists now, but the fix changes current privacy and retention behavior for unmentioned group media.

## What This Changes

The issue asks Telegram group history to preserve media data from unmentioned group messages so a later mentioned turn can inspect those files.

## Best Possible Solution

Have a Telegram area reviewer approve the unmentioned-media retention policy; if approved, wire this path into `HistoryEntry.media` and `recordWithMedia` with bounded retention, stale-file filtering, focused tests, and live Telegram proof.

## Reproduction Assessment

Yes, at source level. Current main skips unmentioned `requireMention` Telegram group media before `resolveMedia`, and the skipped-history branch records text-only entries without `HistoryEntry.media`.

## Solution Assessment

Unclear until Telegram policy is decided. The old `mediaPaths`/`mediaTypes` approach is superseded by the current generic `HistoryEntry.media` contract, but using it for unmentioned group media needs owner and security review.
