## Summary

Keep open: current main and the latest release still do not preserve Telegram media from unmentioned group messages for a later mentioned turn, and both related Telegram fix PRs closed unmerged. ## What This Changes

The issue asks Telegram group history to preserve media data from unmentioned group messages so a later mentioned turn can inspect recent images.

## Best Possible Solution

Use the existing `HistoryEntry.media` and `recordWithMedia` seam in Telegram after maintainers define bounded retention, cleanup, redaction, and resource limits for unmentioned group media.

## Reproduction Assessment

Yes from source inspection: the Telegram no-mention branch stores a text-only history entry, and ordinary unaddressed group media can be skipped before `resolveMedia`. I did not run a live Telegram group repro in this read-only review.

## Solution Assessment

No, not as originally suggested with new parallel `mediaPaths`/`mediaTypes` fields; current main already has a better shared `HistoryEntry.media` and `recordWithMedia` seam. The implementation should wait for a maintainer decision on the retention/security policy before wiring Telegram into that seam.
