## Summary

Keep open. Current main still drops downloaded Telegram media from no-mention group history, and the newer local conversation context only preserves textual `telegram:file/...` refs rather than replaying files into `MediaPaths`; the related fix attempts are closed unmerged.

## What This Changes

Preserve downloaded Telegram media from no-mention group history so a later mention can give the agent access to the recent files.

## Best Possible Solution

Add a bounded media-bearing pending-history contract for skipped Telegram group messages, filter missing refs, and merge surviving history media into later mention context without broadening core channel policy.

## Reproduction Assessment

Yes, source-level. In a Telegram requireMention group, downloaded `allMedia` reaches the no-mention branch, but only text fields are stored and later `MediaPaths` are built from current or reply media.

## Solution Assessment

Yes for a narrow repair. Optional media metadata on pending history plus bounded retention/filtering and replay into `MediaPaths` is the maintainable fix; the current chat-window `media_ref` text is not enough to process the image.
