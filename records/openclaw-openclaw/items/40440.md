## Summary

Keep open: current main still does not make unmentioned Telegram group media available to a later mentioned turn. The current Telegram skip path either avoids downloading unaddressed group media before context construction or records a text-only history entry, while the later context builder only forwards current-turn media to `MediaPaths`.

## What This Changes

The issue asks Telegram group history to preserve media from unmentioned group messages so a later mentioned turn can inspect those images.

## Best Possible Solution

Use the existing `HistoryEntry.media` and `recordWithMedia()` seam for Telegram only after maintainers approve bounded local-image retention and visibility rules; avoid reviving the old unbounded parallel `mediaPaths`/`mediaTypes` approach.

## Reproduction Assessment

Yes at source level: current Telegram code skips unaddressed group media before download or records only text history, and later builds model media from current-turn media only. I did not run live Telegram proof because this review is read-only.

## Solution Assessment

No for the original suggested shape: adding parallel `mediaPaths`/`mediaTypes` or cache mtime refresh repeats concerns from the closed PRs. The better fix is the existing shared `HistoryEntry.media` and `recordWithMedia()` path with explicit retention, visibility, and regression tests.
