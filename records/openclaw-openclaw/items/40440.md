## Summary

Keep open. Current main still cannot attach unmentioned Telegram group media from history: Telegram now skips those media messages before download under `requireMention`, and the skipped-history path records text-only entries, while the newer shared history-media contract is not wired into this Telegram path.

## What This Changes

The issue asks for Telegram group history to preserve usable media from unmentioned group messages so a later mentioned turn can inspect it.

## Best Possible Solution

Have the Telegram area reviewer decide the unmentioned-media retention policy; if approved, wire Telegram into the existing `HistoryEntry.media` and `recordWithMedia` contract with bounded local image retention, focused tests, and live Telegram proof.

## Reproduction Assessment

Yes, at source level. Current main skips unmentioned `requireMention` Telegram group media before download, then records skipped group history as text-only entries without `HistoryEntry.media`.

## Solution Assessment

Unclear until Telegram policy is decided. The older `mediaPaths` and temp-file touch approach is not the best shape now that main has a generic bounded history-media contract, but Telegram still needs approval before using it for unmentioned group media.
