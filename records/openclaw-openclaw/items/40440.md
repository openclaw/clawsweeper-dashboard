## Summary

Keep open: current main still does not preserve Telegram media on the unmentioned group-history path, and the two linked fix PRs were closed unmerged. The shared history-media contract now exists, but Telegram still records the skipped text entry without using it and can also skip many unmentioned media messages before download.

## What This Changes

Preserve media metadata from unmentioned Telegram group messages in pending group history so a later mentioned turn can inspect recent images.

## Best Possible Solution

Reuse the existing HistoryEntry.media and recordWithMedia contract in the Telegram group skip path, with bounded local-image retention, redacted prompt metadata, and focused Telegram regression coverage.

## Reproduction Assessment

Yes, at source level: current main records the Telegram no-mention group history entry without media, and many unmentioned media messages can return before media download or history recording. I did not run live Telegram proof in this read-only review.

## Solution Assessment

No, not exactly as originally suggested: parallel mediaPaths/mediaTypes plus a new cache directory repeats older risky PR shapes. The safer fix is to reuse the existing shared HistoryEntry.media and recordWithMedia seam with explicit security and retention bounds.
