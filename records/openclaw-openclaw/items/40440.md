## Summary

Keep open. Current main still skips unmentioned Telegram `requireMention` group media before download and records the skipped history entry as text only; the shared history-media contract exists, but wiring Telegram into it needs an explicit privacy/retention policy.

## What This Changes

The issue asks Telegram group history to preserve media metadata from unmentioned messages so a later mentioned turn can inspect the media.

## Best Possible Solution

Have a Telegram/security owner approve an opt-in unmentioned-media retention policy, then implement bounded Telegram-owned history-media recording through `HistoryEntry.media` while preserving the current no-download default.

## Reproduction Assessment

Yes, at source level. Current main skips unmentioned `requireMention` Telegram group media before `resolveMedia`, and the skipped-history branch records only text fields without media.

## Solution Assessment

Unclear until the Telegram retention policy is decided. The current shared `HistoryEntry.media` contract is the right technical seam, but using it here should be opt-in or otherwise explicitly approved because it changes the no-download default.
