## Summary

Close as ClawHub/external plugin work: current OpenClaw policy keeps the bar high for optional bundled plugins, and this Codex Desktop bridge can be built with the existing native plugin CLI and bundle surfaces outside core. The proposed patch also has blocking setup and hook-resolution defects, so it should not be merged as-is.

## What This Changes

The branch adds a bundled `codex-claw` plugin with `openclaw codex-claw` CLI commands, a generated Codex Desktop marketplace/hooks payload, docs, inventory entries, and tests.

## Best Possible Solution

Keep core unchanged and move the bridge to an external ClawHub/native plugin, using the existing CLI registration and Codex bundle surfaces after fixing the hook and config-key issues there.

## Reproduction Assessment

Not applicable: this is a feature PR adding a new bundled plugin rather than a bug report. Source review of the PR diff and upstream Codex plugin contracts was enough to identify the setup and hook-resolution failures without running write commands.

## Solution Assessment

No: adding this optional Codex Desktop bridge as a bundled OpenClaw plugin is not the narrowest maintainable path under current plugin policy. External ClawHub/native plugin distribution can provide the CLI and payload with current APIs, and the proposed patch also has blocking correctness and security issues.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Anchor hook execution to the plugin root:** `extensions/codex-claw/src/codex-desktop-bridge.ts:74-75`
  - body: The generated hook scans all of `$HOME/.codex/plugins/cache` for any `*/codex-claw/*/scripts/load-context.mjs` and runs the lexicographically last match. If another marketplace has a same-named plugin, this hook can execute that plugin's script while the user has configured private AGENTS/SOUL paths; use Codex's plugin hook env such as `PLUGIN_ROOT` to execute this plugin's own script instead.
  - confidence: 0.88
- **[P2] Use the generated marketplace name in Codex config:** `docs/plugins/codex-claw.md:50-51`
  - body: The docs tell users to enable `[plugins."codex-claw@codex-claw"]`, but the generated marketplace manifest is named `openclaw-codex-claw`. Codex plugin keys are `<plugin>@<marketplace>`, so this config will not enable or refresh the generated plugin; either align the marketplace name or document `codex-claw@openclaw-codex-claw`.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The patch introduces a concrete plugin-cache script-resolution risk in the generated Codex hook.

Concerns:

- **[medium] Hook can execute a same-named plugin from another marketplace:** `extensions/codex-claw/src/codex-desktop-bridge.ts:75`
  - body: The generated hook locates `load-context.mjs` by scanning the global Codex plugin cache for `*/codex-claw/*` instead of anchoring to this plugin's installed root. A same-named plugin from another marketplace could be selected and run in a context where private AGENTS/SOUL paths are configured.
  - confidence: 0.86
