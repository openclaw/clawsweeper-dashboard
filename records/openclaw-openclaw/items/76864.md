## What This Changes

Adds a bundled `codex-claw` plugin with CLI commands, generated Codex Desktop marketplace payload, docs, inventory updates, tests, and a lockfile entry.

## Best Possible Solution

Keep Codex Claw as an external Codex/ClawHub plugin and consider a narrow OpenClaw core PR only if that implementation proves a missing plugin or bundle API.

## Reproduction Assessment

Not applicable: this is a feature PR, not a report of broken current behavior. Source and docs inspection show current main does not ship `codex-claw`, and the linked request is a scope/product question rather than a failing reproduction path.

## Solution Assessment

No: bundling this optional bridge in OpenClaw core is not the narrowest maintainable solution under the current plugin policy. The external Codex/ClawHub plugin path already exists, and the proposed diff also has a privacy-sensitive mode handling bug.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Honor mode=off in generated hooks:** `extensions/codex-claw/src/codex-desktop-bridge.ts:75`
  - body: The generated hook command always passes `--mode ${CODEX_CLAW_MODE:-full}`. Since the script reads CLI args before `userConfig.mode`, a user who runs `openclaw codex-claw install --mode off` still gets full AGENTS/SOUL injection unless they also set an environment variable, defeating the documented safety knob.
  - confidence: 0.93

## Security Review

Status: needs_attention

Summary: The diff introduces a privacy-sensitive hook bug: configured `mode: off` is ignored, so private context can still be emitted.

Concerns:

- **[high] Generated hook ignores configured off mode:** `extensions/codex-claw/src/codex-desktop-bridge.ts:75`
  - body: The hook runner passes `--mode full` by default before the script can read `~/.codex/codex-claw.json`, so an explicit `mode: "off"` written by install does not stop AGENTS.md and SOUL.md from being injected.
  - confidence: 0.93
