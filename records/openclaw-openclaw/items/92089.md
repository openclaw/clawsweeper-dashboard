## Summary

This PR addresses a real current-main limitation, but it is not mergeable as written: it lets the model delete active goals, can clear state after a failed archive write, creates a non-canonical JSONL state store, and omits central tool metadata and mutation registration. The desired authority change and implicit replacement behavior also need explicit maintainer judgment before implementation is repaired.

## What This Changes

The branch adds a model-facing `clear_goal` tool, makes `create_goal` automatically replace terminal goals, and appends removed goal records to `memory/goal-archive.jsonl`.

## Best Possible Solution

Add one canonical, atomic terminal-goal rotation operation under the session-state owner, keep active goals protected, store any audit history in canonical SQLite state, and register the approved operation across all tool catalog, display, construction, mutation, and policy surfaces.

## Reproduction Assessment

Yes. Current main deterministically rejects `create_goal` whenever a goal already exists, while clearing remains available only through user-facing `/goal` controls.

## Solution Assessment

No. A canonical terminal-only rotation operation is narrower and safer than a general model clear tool plus non-transactional workspace archival, and maintainers must first approve the authority and replacement semantics.
