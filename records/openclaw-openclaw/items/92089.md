## Summary

This PR should remain open for explicit maintainer judgment, but it is not merge-ready: the proposed lifecycle conflicts with the documented operator-only authority boundary and introduces blocking state-integrity and integration defects.

## What This Changes

The PR adds a model-facing `clear_goal` tool, makes `create_goal` automatically replace terminal goals, and archives cleared goals in `memory/goal-archive.jsonl`.

## Best Possible Solution

Keep the operator-only boundary unless maintainers explicitly approve model-controlled rotation; if approved, implement one terminal-only canonical session transition with atomic database-backed history and complete metadata, documentation, and upgrade coverage.

## Reproduction Assessment

Yes, at source level: current main deterministically rejects a second goal and explicitly reserves clearing and replacement to operator/session controls. No live execution was performed because this was a read-only review.

## Solution Assessment

No. The patch conflicts with the established authority contract and couples recovery to unrestricted deletion, non-atomic archival, and a parallel persistent state store; a narrow canonical terminal-rotation operation would be safer if maintainers approve the capability.
