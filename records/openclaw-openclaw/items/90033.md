The PR restores opt-in OpenAI-compatible session-affinity headers, forwards embedded-agent session ids into wrapped stream paths, and enables the Fireworks catalog entries to use that compat behavior.

## Best Possible Solution

Land the narrow Fireworks transport/catalog fix after maintainers explicitly accept the default affinity-header behavior; keep generic dynamic headers and config-schema compatibility in their separate tracks.

## Solution Assessment

Yes for the narrow transport fix: it reuses the existing typed compat seam and provider request-policy merge instead of adding Fireworks-specific transport branches. The remaining decision is whether bundled Fireworks should opt in by default, with broader dynamic headers and config-schema support handled separately.
