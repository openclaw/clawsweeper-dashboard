## What This Changes

## Best Possible Solution

Forward the trusted sender fields through the channel factory API while preserving the existing construction-plan gates, then cover sender forwarding and allowlist-driven tool construction before merge.

## Reproduction Assessment

Yes for source-level behavior: current main only passes `cfg` to channel agent-tool factories while adjacent OpenClaw tools receive trusted sender context. The PR regression is also source-reproducible from the diff against the construction-plan tests; no live channel proof was provided.

## Solution Assessment

No. The API direction is reasonable, but the patch should preserve `includeChannelTools` and `includeOpenClawTools` instead of replacing both with `includeCoreTools`.
