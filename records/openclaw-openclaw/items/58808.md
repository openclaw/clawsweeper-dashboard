## What This Changes

The branch extends `ChannelAgentToolFactory` and `listChannelAgentTools` with optional `requesterSenderId`/`senderIsOwner` fields and forwards sender context from `createOpenClawCodingTools`.

## Best Possible Solution

Land this or an equivalent narrow patch only after preserving `includeChannelTools`/`includeOpenClawTools`, adding focused regression coverage for sender-context propagation, and completing maintainer review of the public SDK surface.

## Reproduction Assessment

Yes for the PR-introduced regression by source inspection: current main computes `includeChannelTools` and `includeOpenClawTools` separately, while the PR gates both factory calls on `includeCoreTools`. For the requested feature, current-main type and call-site inspection confirms the context is not yet forwarded.

## Solution Assessment

No as written. Passing trusted sender context into the factory is the right narrow direction, but the `pi-tools.ts` hunk must keep `includeChannelTools` for channel tools and `includeOpenClawTools` for OpenClaw tools.
