## Summary

Keep this PR open, but do not merge it as written: current main still has the requested sender-context gap for channel agent-tool factories, while the latest patch regresses existing tool-family gates and still lacks real behavior proof.

## What This Changes

The PR extends the channel agent-tool factory parameters and forwards sender identity plus owner status from `createOpenClawCodingTools` into channel-owned agent tool factories.

## Best Possible Solution

Land the additive sender-context API after preserving the existing channel/OpenClaw tool gates, adding focused regression coverage, and collecting redacted real setup proof.

## Reproduction Assessment

Yes for the PR-introduced regression by source inspection: current main supports `includeCoreTools: false` with `includeChannelTools: true`, while this patch gates channel tools on `includeCoreTools`. I did not run tests because this review is read-only.

## Solution Assessment

No, not as written. The narrow maintainable fix is to forward `options?.senderId` and `options?.senderIsOwner` into `listChannelAgentTools()` while keeping `includeChannelTools` and `includeOpenClawTools` separate.
