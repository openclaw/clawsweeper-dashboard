## Summary

Keep open: the requested SDK seam is real and current main still lacks it, but the latest PR patch breaks the per-family tool construction gates and still lacks required real behavior proof and changelog coverage.

## What This Changes

This PR extends channel agent-tool factory params with trusted sender context and forwards coding-tool sender options into `listChannelAgentTools`, while changing the channel/OpenClaw tool construction guards in `pi-tools.ts`.

## Best Possible Solution

Add the optional trusted sender fields through the existing `includeChannelTools` path, leave `createOpenClawTools` behind `includeOpenClawTools`, add changelog/regression coverage, and merge only after real sender-aware channel-tool proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an additive public plugin SDK feature. Source inspection confirms current main only passes `cfg` to channel tool factories, and current tests define the construction-gate behavior the PR must preserve.

## Solution Assessment

No. The API direction is plausible, but the current patch should preserve the per-family construction gates and add changelog coverage before maintainer review can consider it the right solution.
