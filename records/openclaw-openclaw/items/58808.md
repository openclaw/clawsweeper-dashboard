## Summary

Keep open; this is a plausible public plugin SDK feature, but the current head is not mergeable because it changes the channel/OpenClaw tool guards to `includeCoreTools`, lacks changelog coverage, and has no real behavior proof.

## What This Changes

The branch adds optional `requesterSenderId` and `senderIsOwner` parameters to `ChannelAgentToolFactory`/`listChannelAgentTools` and forwards sender context from `createOpenClawCodingTools` into channel agent-tool factories.

## Best Possible Solution

Thread the trusted sender fields through `listChannelAgentTools` under the existing `includeChannelTools` path, leave `createOpenClawTools` behind `includeOpenClawTools`, add changelog/regression coverage, and merge only after real sender-aware channel-tool proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an additive public plugin SDK feature. Source inspection confirms current `main` only passes `cfg` to channel agent-tool factories, and the PR behavior is reviewable from the diff plus construction-plan tests.

## Solution Assessment

No. The API direction is narrow and plausible, but the current implementation should preserve `includeChannelTools` and `includeOpenClawTools` instead of switching those call sites to `includeCoreTools`.
