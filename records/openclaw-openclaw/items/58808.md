## Summary

Keep open: the sender-context API addition is useful and still not present on current main, but the latest PR head regresses tool-family gating and lacks required real behavior proof for an external contributor PR.

## What This Changes

The PR widens channel agent-tool factory parameters with requesterSenderId/senderIsOwner and forwards sender context from createOpenClawCodingTools, while also changing the channel and OpenClaw tool gates to includeCoreTools.

## Best Possible Solution

Thread the trusted sender fields through the channel agent-tool factory while preserving the existing includeChannelTools and includeOpenClawTools gates, then add focused regression coverage and contributor-provided real behavior proof.

## Reproduction Assessment

Yes, for the PR regression: current main supports includeCoreTools false with includeChannelTools true, while this PR gates channel tools on includeCoreTools. CI also reports includeChannelTools as unused on the PR head; I did not run local tests because this review is read-only.

## Solution Assessment

No, not as written. The additive sender-context plumbing is the right narrow direction, but the call site must keep the existing separate channel and OpenClaw tool-family gates and add proof/coverage.
