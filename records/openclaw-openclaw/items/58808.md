## Best Possible Solution

Thread optional sender context through the channel agent-tool factory while preserving the existing per-family gates, then land with focused regression tests and redacted real setup proof.

## Reproduction Assessment

Yes. Source inspection and current tests show current main supports `includeCoreTools: false` with `includeChannelTools: true`, and `includeCoreTools: true` with `includeOpenClawTools: false`; the PR guard replacement violates those paths.

## Solution Assessment

No. Passing sender context into the channel agent-tool factory is the right seam, but the patch should preserve `includeChannelTools` and `includeOpenClawTools` instead of using `includeCoreTools` for both.
