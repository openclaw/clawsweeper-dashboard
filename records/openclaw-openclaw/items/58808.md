## What This Changes

The PR widens ChannelAgentToolFactory/listChannelAgentTools with optional requesterSenderId and senderIsOwner fields and forwards createOpenClawCodingTools sender context into channel agent-tool factories.

## Best Possible Solution

Land this PR or an equivalent narrow patch after adding the API baseline, changelog entry, and regression coverage, while keeping DMWork-specific authorization policy in the channel plugin.

## Reproduction Assessment

Yes. On current main, createOpenClawCodingTools({ senderId, senderIsOwner }) reaches listChannelAgentTools({ cfg }) only, so a function-valued channel agentTools factory cannot observe trusted sender context.

## Solution Assessment

Yes, directionally. Optional server-injected fields on the existing factory params are a narrow backward-compatible seam, but this branch needs release/API/test follow-through before merge.
