## Summary

Keep this PR open: the sender-context API addition is useful and still missing on current main, but the latest patch regresses tool-family gating and currently fails production typecheck.

## What This Changes

The PR adds optional `requesterSenderId` and `senderIsOwner` fields to channel agent-tool factories and forwards sender context from `createOpenClawCodingTools`.

## Best Possible Solution

Land this API addition or an equivalent narrow patch after preserving the existing tool-family gates, adding focused sender-context regression coverage, and collecting real behavior proof.

## Reproduction Assessment

Yes for the PR-introduced regression: source inspection and CI show the patch gates channel tools on `includeCoreTools`, while current main intentionally supports `includeCoreTools: false` with `includeChannelTools: true`. I did not run local tests because this review is read-only.

## Solution Assessment

No, not as written. The maintainable fix is to keep `includeChannelTools` and `includeOpenClawTools` separate, add only the sender fields to the channel-tools call, and prove the real plugin path works.
