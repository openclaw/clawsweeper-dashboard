## Summary

Keep open: the requested sender-context seam is still missing on current main, but the latest PR head has a blocking tool-gate regression, failing proof/type gates, and no after-fix real behavior proof.

## What This Changes

The PR widens `ChannelAgentToolFactory` and `listChannelAgentTools` params with `requesterSenderId`/`senderIsOwner` and forwards sender context from `createOpenClawCodingTools` into channel agent tool factories.

## Best Possible Solution

Land this PR or an equivalent narrow patch after restoring the family-specific gates, adding propagation coverage, and providing maintainer-approved real behavior proof or override.

## Reproduction Assessment

Yes. The PR diff source-reproduces the blocker by replacing `includeChannelTools` and `includeOpenClawTools` with `includeCoreTools`, and live CI reports `check-prod-types` failing on the latest head.

## Solution Assessment

No, not as written. Passing trusted sender params is the right seam, but `pi-tools` must keep channel tools under `includeChannelTools` and OpenClaw tools under `includeOpenClawTools`, with focused propagation coverage.
