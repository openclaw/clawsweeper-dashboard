## Summary

## What This Changes

The PR extends channel agent-tool factory params with trusted sender identity/owner fields and forwards `createOpenClawCodingTools` sender context into `listChannelAgentTools`.

## Best Possible Solution

Keep the additive sender-context API direction, restore the family-specific gates, add focused propagation coverage, and merge only after contributor proof or a maintainer proof override.

## Reproduction Assessment

Yes for the review finding: the supplied PR diff source-reproduces the gate regression by switching channel and OpenClaw construction to `includeCoreTools` while current tests require family-specific construction plans. The requested sender-context API itself is a new capability, not a current-main bug reproduction.

## Solution Assessment

No, not as written. Passing trusted sender params is a reasonable narrow API direction, but the patch must preserve `includeChannelTools`/`includeOpenClawTools` and prove the runtime propagation path.
