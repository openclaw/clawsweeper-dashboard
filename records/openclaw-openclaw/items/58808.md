## Summary

Keep this PR open: current main still lacks the requested sender-context factory seam, but the current PR head regresses tool-family gating and still lacks required real behavior proof.

The PR adds optional trusted sender identity fields to channel agent tool factories and forwards sender context from tool construction.

## Best Possible Solution

Forward options?.senderId and options?.senderIsOwner into listChannelAgentTools() while preserving the existing includeChannelTools and includeOpenClawTools gates, then add focused regression coverage and real setup proof.

Yes for the PR-introduced regression by source inspection: current main supports includeCoreTools false with includeChannelTools true, while the PR gates channel tools on includeCoreTools and leaves includeChannelTools unused under noUnusedLocals. I did not run tests because this review is read-only.

## Solution Assessment

No, not as written. The narrow maintainable solution is the additive sender-context propagation without changing the existing channel/OpenClaw tool-family gates.
