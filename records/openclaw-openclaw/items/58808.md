## Summary

Keep this PR open: the sender-context seam is still useful and not implemented on current main, but the current patch regresses tool-family construction gates and still lacks real behavior proof.

## What This Changes

Adds optional `requesterSenderId`/`senderIsOwner` fields to `ChannelAgentToolFactory` and forwards sender context into channel agent-tool factories.

## Best Possible Solution

Land a narrow additive propagation that preserves the separate construction gates, adds focused regression coverage, and includes redacted proof from a real channel plugin.

## Reproduction Assessment

Yes by source inspection: current construction plans can set `includeCoreTools: false` while keeping `includeChannelTools: true`, and the PR's guard would skip those channel tools. I did not run tests in this read-only review.

## Solution Assessment

No, not as written. The sender fields are a reasonable additive seam, but the implementation should preserve `includeChannelTools` and `includeOpenClawTools` independently and prove the runtime behavior.
