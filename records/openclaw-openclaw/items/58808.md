## What This Changes

The PR extends channel agent-tool factory params with trusted sender identity and owner fields, then forwards them from `createOpenClawCodingTools`.

## Best Possible Solution

Land the additive sender-context seam after preserving the existing tool-family guards, adding focused contract/regression coverage, and documenting or changelogging the public plugin behavior if maintainers accept the SDK addition.

## Reproduction Assessment

Yes from source inspection, not a live run. Current construction-plan coverage shows channel/plugin-only allowlists can have `includeCoreTools: false` with `includeChannelTools: true`, while the PR gates channel tools on `includeCoreTools`.

## Solution Assessment

No as written. Passing trusted sender context into the factory is the right narrow direction, but the patch must keep `includeChannelTools` and `includeOpenClawTools` as separate guards before it is maintainable.
