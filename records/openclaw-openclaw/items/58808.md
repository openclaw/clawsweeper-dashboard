## Summary

Keep this PR open. It addresses a real current-main gap for channel-owned agent-tool factories, but it should not merge until the public SDK API baseline and changelog are updated and a focused propagation test is added.

## What This Changes

The PR adds optional `requesterSenderId` and `senderIsOwner` fields to `ChannelAgentToolFactory`/`listChannelAgentTools` and forwards `createOpenClawCodingTools` sender context into channel-owned agent-tool factories.

## Best Possible Solution

Land this PR or an equivalent narrow additive patch after updating the generated SDK API baseline, adding release-note coverage, and proving sender-context propagation with a focused test while keeping DMWork-specific authorization policy in the external channel plugin.

## Reproduction Assessment

Yes. A source-level reproduction is to register a function-valued `ChannelPlugin.agentTools` factory and call `createOpenClawCodingTools({ senderId, senderIsOwner })`; current main invokes `listChannelAgentTools` with only `cfg`, so the factory cannot see the trusted sender context.

## Solution Assessment

Yes, directionally. Adding optional trusted fields to the existing factory params and forwarding already-available server-injected context is the narrowest backward-compatible core seam; the proposed patch still needs generated API, release-note, and regression-test follow-through.
