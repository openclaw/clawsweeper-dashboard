## Best Possible Solution

Land this seam or an equivalent narrow patch after preserving the existing family-specific guards, adding regression coverage for sender-context propagation into ChannelAgentToolFactory, and updating SDK docs/changelog if maintainers treat the fields as public.

## Reproduction Assessment

Yes, from source inspection. Current main lacks the sender fields on ChannelAgentToolFactory/listChannelAgentTools, and PR head can be checked against construction-plan coverage where whatsapp_login keeps channel tools enabled while includeCoreTools is false.

## Solution Assessment

No as written. Passing trusted sender context into the factory is the right narrow direction, but the implementation must keep includeChannelTools and includeOpenClawTools as separate guards.
