## Summary

Keep open. Current main still drops trusted sender context before channel-owned agent tools, and the proposed branch is the right narrow direction, but it should not merge until the public plugin-contract change has release-note coverage and focused propagation coverage.

## What This Changes

The PR adds optional `requesterSenderId` and `senderIsOwner` fields to channel agent-tool factory params and forwards the existing `createOpenClawCodingTools` sender context into `listChannelAgentTools`.

## Best Possible Solution

Land this PR or an equivalent small patch after adding the required changelog entry and a regression test for sender-context propagation, while keeping DMWork-specific authorization policy inside the channel plugin rather than hardcoding it in core.

## Reproduction Assessment

Yes. On current main, a function-valued `ChannelPlugin.agentTools` factory reached through `createOpenClawCodingTools({ senderId, senderIsOwner })` receives only `cfg`, because the caller passes only `cfg` into `listChannelAgentTools`.

## Solution Assessment

Yes, directionally. Optional server-injected fields on the existing factory params are the narrowest backward-compatible fix; the safer completion is to keep the contract shape and add release-note plus test coverage before merge.
