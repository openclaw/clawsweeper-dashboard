## Summary

Keep open: the sender-context seam is a plausible feature, but the current patch regresses existing tool-family guards and still lacks required after-fix real behavior proof from a channel-plugin setup.

## What This Changes

The PR extends `ChannelAgentToolFactory` and `listChannelAgentTools` with optional trusted sender fields and forwards existing sender context from `createOpenClawCodingTools`.

## Best Possible Solution

Land this seam, or an equivalent narrow patch, after preserving the existing per-family guards, adding focused propagation and channel-only allowlist coverage, and posting real behavior proof.

## Reproduction Assessment

Yes. Source inspection shows current main supports channel-only plans with `includeCoreTools: false` and `includeChannelTools: true`, while the PR gates channel tools on `includeCoreTools`.

## Solution Assessment

No as written. Passing trusted sender context into the factory is the right narrow direction, but the patch must keep `includeChannelTools` and `includeOpenClawTools` as the controlling guards.
