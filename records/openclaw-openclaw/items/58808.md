The PR widens `ChannelAgentToolFactory`/`listChannelAgentTools` params with optional `requesterSenderId` and `senderIsOwner`, then forwards `createOpenClawCodingTools` sender context into channel agent-tool factories.

## Best Possible Solution

Keep the sender-context contract addition, but restore `listChannelAgentTools` to the `includeChannelTools` gate and `createOpenClawTools` to the `includeOpenClawTools` gate, then add focused regression proof and required real behavior evidence.
