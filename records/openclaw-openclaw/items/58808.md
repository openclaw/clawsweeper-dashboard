## Summary

Keep open: current main still drops trusted sender context on the channel-owned agent-tool factory path, and this PR is a narrow, directionally correct fix. ## What This Changes

The PR adds optional `requesterSenderId` and `senderIsOwner` fields to `ChannelAgentToolFactory` and `listChannelAgentTools`, then forwards `createOpenClawCodingTools` sender context into channel-owned agent-tool factories.

## Best Possible Solution

Land this PR or an equivalent narrow patch after adding release-note coverage and a regression test for trusted sender-context propagation, while keeping DMWork-specific authorization policy inside the channel plugin rather than hardcoding it in core.

## Reproduction Assessment

Yes. A source-level reproduction on current main is to register a function-valued `ChannelPlugin.agentTools` factory and call `createOpenClawCodingTools({ senderId, senderIsOwner })`; the factory receives only `cfg` today.

## Solution Assessment

Yes, directionally. Optional server-injected fields on the existing factory params are the narrowest backward-compatible fix, but the branch should add changelog and contract coverage before merge.
