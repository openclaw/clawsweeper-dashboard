## Summary

Keep open: current main still lacks the requested plugin-facing sender context, but this PR head is not mergeable because it replaces family-specific tool gates with `includeCoreTools` and still lacks required real behavior proof.

## What This Changes

The PR widens `ChannelAgentToolFactory`/`listChannelAgentTools` params with optional sender context and forwards `createOpenClawCodingTools` sender fields to channel agent-tool factories.

## Best Possible Solution

Forward sender context through `ChannelAgentToolFactory` while preserving `includeChannelTools` and `includeOpenClawTools`, add focused propagation coverage, and update the PR body with redacted real channel-plugin proof.

## Reproduction Assessment

Yes for the PR-introduced regression by source inspection: the diff gates channel and OpenClaw tool construction on `includeCoreTools`, while current construction-plan code and tests intentionally allow those gates to diverge.

## Solution Assessment

No, not as written. The sender-context seam is a reasonable narrow direction, but the patch must preserve the existing family-specific gates before it is maintainable.
