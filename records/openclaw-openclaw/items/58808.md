## Summary

Keep open: the sender-aware channel tool seam is still absent on current main, but this PR is not mergeable as written because the live diff replaces family-specific tool gates with `includeCoreTools` and still lacks required real behavior proof.

## What This Changes

This PR extends `ChannelAgentToolFactory`/`listChannelAgentTools` with optional `requesterSenderId` and `senderIsOwner` fields and forwards sender context from `createOpenClawCodingTools`.

## Best Possible Solution

Merge a revised narrow patch that adds the optional sender fields while preserving existing family gates, plus focused propagation/gate coverage and real plugin proof or a maintainer proof override.

## Reproduction Assessment

Yes. Source review of the live PR diff plus current-main tests shows channel-only allowlists can have `includeCoreTools=false` and `includeChannelTools=true`, so the PR's `includeCoreTools` guard drops allowed channel tools.

## Solution Assessment

No, not as written. Adding trusted sender params is a reasonable narrow direction, but the patch should preserve `includeChannelTools` and `includeOpenClawTools` and only add the sender context to the channel factory call.
