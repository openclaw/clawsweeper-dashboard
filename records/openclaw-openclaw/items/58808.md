## What This Changes

The PR adds optional trusted sender fields to channel-agent-tool factory params and forwards sender context from `createOpenClawCodingTools`, while changing the channel/OpenClaw tool construction gates at the call site.

## Best Possible Solution

Forward trusted sender context through `ChannelAgentToolFactory` while preserving the existing family-specific gates, then add focused propagation coverage and SDK docs/changelog follow-through if maintainers accept the contract.

## Reproduction Assessment

Yes for the PR-introduced regression by source inspection: current main uses separate `includeChannelTools` and `includeOpenClawTools` gates, while the PR gates both families on `includeCoreTools`. I did not run tests because this review is read-only, but the existing construction-plan tests encode the mismatched gate states.

## Solution Assessment
