## What This Changes

The PR extends channel agent-tool factory parameters with trusted sender context and forwards createOpenClawCodingTools sender options into listChannelAgentTools.

## Best Possible Solution

Thread the trusted sender fields through ChannelAgentToolFactory/listChannelAgentTools while preserving the existing tool-family gates, then add focused regression coverage and SDK docs if maintainers accept the public contract change.

Yes for the PR regression by source inspection: current main permits includeCoreTools false with includeChannelTools true for plugin-only allowlists, while the PR gates channel tools on includeCoreTools. I did not run tests because this is a read-only review.

## Solution Assessment

No, not as written. The sender-context plumbing is the right narrow direction, but replacing includeChannelTools/includeOpenClawTools with includeCoreTools is not behavior-preserving.
