## Best Possible Solution

Yes, for the PR regression by source inspection: current main supports `includeCoreTools: false` with `includeChannelTools: true`, while the PR gates channel tools and OpenClaw tools on `includeCoreTools`. I did not run tests because this is a read-only review.

## Solution Assessment

No, not as written. The sender-context plumbing is the narrow direction, but the call site must keep `includeChannelTools` and `includeOpenClawTools` as separate gates.
