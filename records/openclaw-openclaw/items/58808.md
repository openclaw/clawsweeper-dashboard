## What This Changes

The PR adds optional `requesterSenderId` and `senderIsOwner` fields to `ChannelAgentToolFactory`/`listChannelAgentTools` and forwards sender context from `createOpenClawCodingTools` into channel-owned agent-tool factories.

## Best Possible Solution

Forward the trusted sender context through `ChannelAgentToolFactory` while preserving `includeChannelTools` and `includeOpenClawTools`, add focused propagation coverage, and require redacted real setup proof before merge.

## Solution Assessment

No, not as written. The sender-context seam is narrow and plausible, but the implementation should preserve the existing family-specific tool gates before the public SDK addition is accepted.
