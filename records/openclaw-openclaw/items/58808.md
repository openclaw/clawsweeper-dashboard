## Summary

Keep this PR open: the requested sender-context seam is still useful, but the current patch regresses tool-family construction gates and lacks required real behavior proof.

## What This Changes

Adds optional requester sender context to ChannelAgentToolFactory/listChannelAgentTools and forwards createOpenClawCodingTools sender context into channel-owned agent tool factories.

## Best Possible Solution

Land this PR or an equivalent narrow patch after preserving the per-family gates, adding focused regression coverage, and providing redacted runtime proof from a real channel plugin.

## Reproduction Assessment

Yes for the PR-introduced regression by source inspection: a channel-only allowlist can have includeCoreTools false while includeChannelTools is true, and the PR head would skip those channel tools. I did not run tests in this read-only review.

## Solution Assessment

No, not as written. The sender fields are a reasonable additive seam, but the implementation should preserve includeChannelTools and includeOpenClawTools separately and prove the runtime behavior.
