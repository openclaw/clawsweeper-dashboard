## Summary

## What This Changes

The PR extends channel agent-tool factory params with trusted sender context and forwards `senderId`/`senderIsOwner` from `createOpenClawCodingTools` into `listChannelAgentTools()`.

## Best Possible Solution

Keep the additive sender-context API, restore the separate channel/OpenClaw tool-family guards, add focused propagation coverage, and land only after SDK review plus redacted real plugin proof.

## Reproduction Assessment

Yes, source-reproducible. Current main lacks sender context on `ChannelAgentToolFactory`, and the PR diff's `includeCoreTools` guard would skip channel-only allowlists that existing construction-plan tests keep available.

## Solution Assessment

No as written. Passing trusted sender context into the channel factory is the narrow maintainable direction, but the patch should preserve `includeChannelTools`/`includeOpenClawTools` and add focused propagation coverage.
