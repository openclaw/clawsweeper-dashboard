## Summary

Keep open: the requested sender context is still absent from current main and is useful, but the latest branch regresses tool-family gating by replacing the separate channel/OpenClaw gates with `includeCoreTools`; external real behavior proof is also still missing.

## What This Changes

The PR adds optional trusted sender fields to channel agent-tool factory params and forwards `createOpenClawCodingTools` sender context through `listChannelAgentTools`.

## Best Possible Solution

Thread the trusted sender fields through `ChannelAgentToolFactory` while preserving the existing tool-family gates and adding focused regression coverage for propagation plus plugin/channel-only allowlists.

## Reproduction Assessment

Yes for the PR-introduced regression: current-main source and tests show channel/plugin-only allowlists can set `includeCoreTools: false` while `includeChannelTools: true`, and the latest diff would skip those channel tools. I did not run tests because this review is read-only.

## Solution Assessment

No, not as written. The narrow maintainable solution is to add the optional sender fields and forward them through `listChannelAgentTools()` while keeping `includeChannelTools` and `includeOpenClawTools` separate.
