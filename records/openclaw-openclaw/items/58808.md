## Summary

Keep open: the requested sender-aware channel-tool seam is still absent from current main, but the PR as written regresses tool-family materialization by replacing the existing channel/OpenClaw gates with `includeCoreTools`.

## What This Changes

The PR adds optional `requesterSenderId` and `senderIsOwner` fields to `ChannelAgentToolFactory`/`listChannelAgentTools` and forwards sender context from `createOpenClawCodingTools`.

## Best Possible Solution

Forward trusted sender context through `ChannelAgentToolFactory` while preserving `includeChannelTools` and `includeOpenClawTools`, then add focused regression coverage and SDK docs/changelog if maintainers accept the public contract.

## Reproduction Assessment

Yes for the PR-introduced regression by source inspection: current main uses separate channel/OpenClaw construction gates and tests expect channel/plugin tools with `includeCoreTools: false`, while the PR gates both families on `includeCoreTools`. I did not run tests because this review is read-only.

## Solution Assessment

No, not as written. The optional sender fields are a narrow plausible SDK seam, but the patch should preserve the existing per-family gates and add focused coverage before merge.
