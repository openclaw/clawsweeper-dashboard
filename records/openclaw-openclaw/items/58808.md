## Summary

Keep this PR open: the requested SDK/context addition is plausible, but the current diff has blocking tool-construction gate regressions and still lacks real behavior proof.

## What This Changes

The PR adds requesterSenderId and senderIsOwner to ChannelAgentToolFactory/listChannelAgentTools and forwards createOpenClawCodingTools sender context into channel-owned agent-tool factories.

## Best Possible Solution

Keep the additive sender-context contract, but preserve includeChannelTools/includeOpenClawTools gating and add focused propagation coverage before maintainer review and merge.

## Reproduction Assessment

Yes. Source inspection shows the PR changes two independent tool-family gates to includeCoreTools, and existing construction-plan tests define cases where those booleans intentionally differ.

## Solution Assessment

No. The sender-context fields are a reasonable seam, but the implementation should only widen the factory params and call-site object while leaving includeChannelTools and includeOpenClawTools as the guards.
