## What This Changes

The branch extends `ChannelAgentToolFactory`/`listChannelAgentTools` with optional `requesterSenderId` and `senderIsOwner` params, forwards sender context from `createOpenClawCodingTools`, and currently replaces the channel/OpenClaw tool guards with `includeCoreTools`.

## Best Possible Solution

Thread the optional sender context through the channel agent-tool factory while preserving `includeChannelTools` and `includeOpenClawTools` as the construction guards, then land with focused regression tests and redacted real-environment proof.

## Reproduction Assessment

Yes. Source inspection against current `main` shows the separate construction gates, and existing tests document `includeCoreTools: false` with channel tools enabled plus `includeCoreTools: true` with OpenClaw tools disabled.

## Solution Assessment

No. Passing sender context into the channel agent-tool factory is the right direction, but replacing the per-family construction gates with `includeCoreTools` is not the narrow maintainable fix.
