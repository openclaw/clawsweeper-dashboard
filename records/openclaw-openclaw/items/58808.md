## Best Possible Solution

Forward the trusted sender context through `ChannelAgentToolFactory` while preserving the existing per-family construction gates, then add focused regression coverage and SDK docs if maintainers accept the public contract.

Yes for the PR-introduced regression by source inspection: current main has narrow plans where `includeCoreTools` can be false while `includeChannelTools` remains true, but the PR gates channel tools on `includeCoreTools`. I did not run tests because this review is read-only.

## Solution Assessment

No, not as written. The new optional sender fields are a narrow seam, but the implementation should keep `includeChannelTools` and `includeOpenClawTools` as separate gates.
