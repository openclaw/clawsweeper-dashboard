## Best Possible Solution

Land the SDK addition only after preserving `includeChannelTools` and `includeOpenClawTools`, adding focused regression coverage, and supplying redacted proof from a real sender-aware channel tool path.

## Reproduction Assessment

Yes, for the PR-introduced regression by source inspection: current construction plans can produce `includeCoreTools: false` with `includeChannelTools: true`, and this PR would skip channel tools in that case. I did not run tests because this review is read-only.

## Solution Assessment

No. The maintainable fix is to add the optional sender fields while preserving the existing `includeChannelTools` and `includeOpenClawTools` gates, then cover that with focused tests and real behavior proof.
