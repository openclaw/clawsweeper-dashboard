## What This Changes

This PR adds optional requesterSenderId and senderIsOwner parameters to ChannelAgentToolFactory/listChannelAgentTools and forwards sender context from createOpenClawCodingTools.

## Best Possible Solution

Keep the additive sender-context API only if maintainers accept the public SDK contract, preserve includeChannelTools/includeOpenClawTools exactly, and add focused regression coverage plus real channel-plugin proof.

## Reproduction Assessment

Yes for the PR-introduced regression by source inspection: current main allows plugin/channel-only plans with includeCoreTools false and includeChannelTools true, while the PR gates channel tools on includeCoreTools. I did not run tests because this is a read-only review.

## Solution Assessment

No, not as written. Passing trusted sender fields through the channel tool factory is the narrow direction, but replacing the separate tool-family gates with includeCoreTools is not behavior-preserving.
