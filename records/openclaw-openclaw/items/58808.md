The branch adds optional requesterSenderId/senderIsOwner parameters to ChannelAgentToolFactory/listChannelAgentTools, forwards createOpenClawCodingTools sender context into channel agent-tool factories, and replaces two family-specific tool-construction guards with includeCoreTools.

## Best Possible Solution

Land a narrow SDK addition that preserves includeChannelTools/includeOpenClawTools guards, adds regression coverage for sender params and narrow construction plans, and includes redacted proof from a real sender-aware channel tool path.

## Reproduction Assessment

Yes, for the PR-introduced regression by source inspection: current construction plans can produce includeCoreTools false with includeChannelTools true, and the PR would skip channel tools in that case. I did not run tests because this review is read-only.

## Solution Assessment

No. The maintainable fix is to add the optional sender fields while preserving the includeChannelTools and includeOpenClawTools guards, then cover that with focused tests and real behavior proof.
