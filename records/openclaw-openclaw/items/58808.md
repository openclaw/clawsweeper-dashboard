## What This Changes

The PR extends `ChannelAgentToolFactory`/`listChannelAgentTools` with optional `requesterSenderId` and `senderIsOwner`, forwards those values from `createOpenClawCodingTools`, and also changes channel/OpenClaw tool construction to use `includeCoreTools`.

## Best Possible Solution

Thread the optional sender context through the channel agent-tool factory while preserving `includeChannelTools` and `includeOpenClawTools` as the construction guards, then land with focused regression tests and redacted real-environment proof.

## Reproduction Assessment

Yes. Source inspection against current `main` shows the separate construction gates, and existing tests document `includeCoreTools: false` with channel tools enabled plus `includeCoreTools: true` with OpenClaw tools disabled.

## Solution Assessment

No. Passing sender context into the channel agent-tool factory is reasonable, but the best fix preserves the existing per-family gates and adds focused tests plus real sender-aware proof.
