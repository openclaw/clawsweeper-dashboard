The PR widens `ChannelAgentToolFactory`/`listChannelAgentTools` params with optional `requesterSenderId` and `senderIsOwner`, then forwards sender context from `createOpenClawCodingTools`.

## Best Possible Solution

Keep the additive sender-context API, restore the separate channel/OpenClaw tool-family guards, add focused propagation coverage, and land after SDK review plus redacted real plugin proof.

## Reproduction Assessment

Yes, source-reproducible. Current main lacks sender context on `ChannelAgentToolFactory`, and the PR diff would skip channel-only allowlists that existing construction-plan tests keep available.

## Solution Assessment

No as written. Passing trusted sender context into the channel factory is the narrow direction, but the patch should preserve `includeChannelTools`/`includeOpenClawTools` and add focused propagation coverage.
