## Summary

Keep open: current main still lacks the requested sender context for channel agent-tool factories, but this PR is not merge-ready because the latest head still replaces the separate channel/OpenClaw tool-family gates and lacks real behavior proof.

## What This Changes

The PR widens `ChannelAgentToolFactory` and `listChannelAgentTools` with trusted requester sender fields and forwards `createOpenClawCodingTools` sender context into channel agent-tool factories.

## Best Possible Solution

Thread the trusted sender fields through `ChannelAgentToolFactory` while preserving the existing `includeChannelTools` and `includeOpenClawTools` gates, then add focused regression coverage and SDK docs if maintainers want the new fields documented as public contract.

## Reproduction Assessment
