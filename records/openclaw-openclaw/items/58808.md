## Best Possible Solution

Preserve the existing per-family construction guards, forward the trusted sender fields only through the channel-agent-tool factory path, add focused regression coverage, and require maintainer acceptance plus redacted runtime proof before merge.

## Reproduction Assessment

Yes for the PR-introduced regression: source inspection shows construction plans can set `includeCoreTools:false` with `includeChannelTools:true`, and the PR would gate channel tools on `includeCoreTools`. I did not run tests because this is a read-only review.

## Solution Assessment

No. Forwarding trusted sender fields is the right direction, but the maintainable fix must preserve `includeChannelTools` and `includeOpenClawTools` rather than replacing them with `includeCoreTools`.
