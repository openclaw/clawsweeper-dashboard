## Summary

Keep this PR open, but do not merge it as written: the requested sender-context SDK feature is still absent from current main, while the latest diff changes existing tool-family gates and lacks contributor real-behavior proof.

## What This Changes

The PR adds optional requesterSenderId and senderIsOwner fields to ChannelAgentToolFactory/listChannelAgentTools, forwards them from createOpenClawCodingTools, and changes the channel/OpenClaw factory guards in pi-tools.ts.

## Best Possible Solution

Thread the trusted sender fields through ChannelAgentToolFactory/listChannelAgentTools while preserving includeChannelTools and includeOpenClawTools, then add focused regression coverage and public SDK docs if maintainers accept the contract change.

## Reproduction Assessment

Yes for the PR regression by source inspection: current tests and pi-tools.ts show plugin/channel-only construction can have includeCoreTools false while includeChannelTools is true. I did not run tests because this was a read-only review.

## Solution Assessment

No, not as written. The sender-context plumbing is the right narrow direction, but replacing the separate channel/OpenClaw tool-family guards with includeCoreTools is not behavior-preserving.
