## Summary

Keep open: the requested sender-context seam is still useful and not present on current main, but this PR is not mergeable as written because it collapses separate channel/OpenClaw tool-family gates and currently fails `check-prod-types`.

## What This Changes

This PR extends `ChannelAgentToolFactory`/`listChannelAgentTools` with optional sender context fields and forwards `createOpenClawCodingTools` sender context into channel-owned agent-tool factories.

## Best Possible Solution

Thread the trusted sender fields through `ChannelAgentToolFactory` while preserving the existing tool-family gates and adding focused regression coverage for both propagation and plugin/channel-only allowlists.

## Reproduction Assessment

Yes for the PR-introduced regression: source and existing tests show plugin/channel-only plans can have `includeCoreTools` false while `includeChannelTools` remains true, and CI also fails because the PR leaves `includeChannelTools` unused.

## Solution Assessment

No, not as written. Passing trusted sender fields to channel tool factories is the narrow direction, but the patch should keep `includeChannelTools` for `listChannelAgentTools()` and `includeOpenClawTools` for `createOpenClawTools()`.
