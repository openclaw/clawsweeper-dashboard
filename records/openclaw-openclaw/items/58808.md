## What This Changes

The PR extends `ChannelAgentToolFactory` and `listChannelAgentTools` with `requesterSenderId`/`senderIsOwner` and forwards sender context from `createOpenClawCodingTools`.

## Best Possible Solution

Thread trusted sender context through the channel agent tool factory contract while preserving tool-family guards, with regression coverage, changelog, and redacted real plugin proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an additive plugin SDK feature. Source inspection shows current main gives channel agent tool factories only `cfg`, while adjacent OpenClaw tools already receive trusted sender context.

## Solution Assessment

No. Passing the fields is the right seam, but this patch should preserve `includeChannelTools`/`includeOpenClawTools`, add changelog and regression coverage, and include real behavior proof before merge.
