## What This Changes

The branch extends `ChannelAgentToolFactory` params with optional `requesterSenderId` and `senderIsOwner`, then forwards sender context from `createOpenClawCodingTools` while changing the channel/OpenClaw tool guards to `includeCoreTools`.

## Best Possible Solution

Thread the optional sender context through the channel agent-tool factory while preserving the existing per-family gates, then land with focused regression coverage and redacted real-environment proof.

## Reproduction Assessment

Yes. Source inspection and existing tests show current main supports `includeCoreTools: false` with `includeChannelTools: true`, and `includeCoreTools: true` with `includeOpenClawTools: false`; the PR's guard replacement violates those paths.

## Solution Assessment

No. Passing sender context into the channel agent-tool factory is the right seam, but the current patch should preserve the existing `includeChannelTools` and `includeOpenClawTools` gates instead of using `includeCoreTools` for both.
