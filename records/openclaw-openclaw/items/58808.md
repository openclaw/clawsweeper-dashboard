The PR widens ChannelAgentToolFactory/listChannelAgentTools params with trusted sender context and forwards createOpenClawCodingTools sender/owner fields into channel agent tool factories.

## Best Possible Solution

Land this seam or an equivalent narrow patch only after preserving the existing per-family guards, adding focused regression coverage, and posting redacted real behavior proof from a channel-plugin setup.

No as written. Passing trusted sender context into the factory is the right direction, but the patch must keep includeChannelTools and includeOpenClawTools as separate guards.
