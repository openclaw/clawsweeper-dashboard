Keep open: the requested SDK capability is still absent from current main and this PR remains the canonical implementation candidate, but the current diff has a P2 tool-family gating regression and lacks real behavior proof.

## What This Changes

The PR widens channel agent-tool factory params with trusted sender fields and changes the `pi-tools` call site to forward sender context while gating channel and OpenClaw tool factories on `includeCoreTools`.

## Best Possible Solution

Thread the trusted sender fields through the channel agent-tool factory while preserving `includeChannelTools` and `includeOpenClawTools`, then add focused regression coverage and SDK docs/changelog updates if maintainers accept the public contract change.

## Reproduction Assessment

Yes, for the PR regression by source inspection: current main supports channel/plugin-only construction with `includeCoreTools: false`, while the PR gates channel tools and OpenClaw tools on `includeCoreTools`. I did not run tests because this is a read-only review.

## Solution Assessment

No, not as written. The additive sender-context plumbing is the right narrow direction, but the call site must keep the separate channel and OpenClaw tool-family gates.
