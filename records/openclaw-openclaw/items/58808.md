Keep open: the requested additive sender context is still useful and not on current main, but the PR head currently regresses existing tool-construction guards and still lacks required real behavior proof for an external contributor PR.

## What This Changes

The branch extends `ChannelAgentToolFactory` and `listChannelAgentTools` to accept trusted sender context, then forwards `options.senderId` and `options.senderIsOwner` from `createOpenClawCodingTools`.

## Best Possible Solution

Preserve the current `includeChannelTools` and `includeOpenClawTools` guards, pass the new sender fields through `listChannelAgentTools`, add focused coverage for channel-only allowlists plus factory params, and align public SDK docs before landing.

## Reproduction Assessment

Yes, by source inspection of the PR diff against current main. ## Solution Assessment
