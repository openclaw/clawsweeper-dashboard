## What This Changes

The PR widens `ChannelAgentToolFactory` and `listChannelAgentTools` params with trusted sender context and forwards `senderId`/`senderIsOwner` from `createOpenClawCodingTools`.

## Best Possible Solution

Preserve `includeChannelTools` as the channel-agent-tool gate, add a focused regression test that captures the new sender params and construction-plan behavior, then land the additive SDK contract with maintainer-approved docs/changelog handling.

## Reproduction Assessment

## Solution Assessment

No, not as currently written. The sender-context threading is the right narrow direction, but the call site should keep the existing channel-specific gate rather than replacing it with the broader core gate.
