## Best Possible Solution

Keep the additive sender-context API, restore `includeChannelTools` and `includeOpenClawTools`, add focused propagation coverage, and merge only after real behavior proof or a maintainer proof override.

The underlying sender-context request is a feature gap, not a broken current-main behavior reproduction.
