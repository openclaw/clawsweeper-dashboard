## Summary

Keep this PR open: the sender-context seam is useful and still missing on current main, but the latest patch regresses tool-family guards and lacks real behavior proof from a channel-plugin setup.

## What This Changes

The PR adds optional trusted sender fields to ChannelAgentToolFactory/listChannelAgentTools and forwards createOpenClawCodingTools sender context into channel-owned agent-tool factories.

## Best Possible Solution

Land this seam or an equivalent narrow patch after preserving the family-specific tool guards, adding regression coverage for sender-context propagation and allowlist behavior, and updating SDK docs/changelog if maintainers treat it as public.

## Reproduction Assessment

Yes, from source inspection. Current main lacks the sender fields, and the PR's latest patch can be checked against construction-plan coverage where whatsapp_login keeps channel tools enabled while includeCoreTools is false.

## Solution Assessment

No as written. Passing trusted sender context into the factory is the right narrow direction, but the implementation must keep includeChannelTools and includeOpenClawTools as separate guards.
