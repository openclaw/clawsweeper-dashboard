## Summary

Keep this PR open: the requested sender-context SDK seam is still useful, but the current patch regresses the existing tool-family construction gates and lacks required after-fix real behavior proof.

## What This Changes

The branch widens `ChannelAgentToolFactory`/`listChannelAgentTools` params with `requesterSenderId` and `senderIsOwner`, then forwards `createOpenClawCodingTools` sender context into channel-owned agent-tool factories.

## Best Possible Solution

Revise the PR to pass sender fields through while preserving `includeChannelTools` and `includeOpenClawTools`, add focused propagation coverage, and provide redacted real channel-plugin proof before maintainer review/merge.

## Reproduction Assessment

Yes. The regression is source-reproducible from the PR diff because it replaces two family-specific gates with `includeCoreTools`, and CI now reports the formerly-used `includeChannelTools` variable as unused.

## Solution Assessment

No. The SDK extension is a reasonable narrow direction, but the implementation should only add the sender params and must preserve the existing channel/OpenClaw construction gates.
