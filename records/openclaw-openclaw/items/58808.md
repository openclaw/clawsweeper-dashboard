## Summary

The PR extends `ChannelAgentToolFactory`/`listChannelAgentTools` with trusted sender context and forwards `senderId`/`senderIsOwner` from `createOpenClawCodingTools`.

## Best Possible Solution

Forward trusted sender context through `ChannelAgentToolFactory` while preserving the existing family-specific gates, then add focused propagation coverage and real channel-plugin proof.

I did not run tests because this review is read-only.

## Solution Assessment

No, not as written. The sender-context addition is a narrow and plausible SDK seam, but the patch should preserve `includeChannelTools` and `includeOpenClawTools` and include focused coverage before acceptance.
