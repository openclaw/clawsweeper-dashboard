## Summary

Keep open. Current main still embeds BOOT.md as ordinary prompt text for boot-md runs, preserves that prompt body across model fallback retries, and has no boot-specific literal-echo guard before message-tool delivery. The closely related PR #55016 is closed unmerged and addressed outbound boot-md delivery, not prompt-echo suppression.

## What This Changes

Fallback models can echo BOOT.md startup instructions to users instead of executing them silently during boot-md fallback runs.

## Best Possible Solution

Add a narrow internal boot-run context that carries a prompt fingerprint or comparable suppression signal through agent execution and tool/final-delivery surfaces, rejects substantial literal echoes of the generated BOOT.md prompt before user-visible delivery, and preserves intentional BOOT.md-triggered outbound notifications with focused regression tests.

## Security Review
