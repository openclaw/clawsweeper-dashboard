## Summary

Keep open. Current main still embeds BOOT.md as ordinary prompt text, preserves that body across fallback attempts, and only strips reasoning tags from message-tool text fields; the issue is also paired with open closing PR #75128, so closing before that lands would be premature.

## What This Changes

The issue asks OpenClaw to prevent fallback-model boot runs from echoing BOOT.md startup instructions to users.

## Best Possible Solution

Land #75128 or an equivalent narrow fix that marks BOOT.md content as internal runtime context and strips that context before final replies or message-tool sends while preserving legitimate boot-directed outbound messages.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence mock path: BOOT.md content enters `buildBootPrompt`, fallback attempts preserve the body, and message-tool text delivery lacks internal-context stripping.
