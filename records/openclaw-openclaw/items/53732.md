## Summary

Keep open. Current main and the latest release still expose the source-level BOOT.md fallback echo path, and the linked closing PR at https://github.com/openclaw/openclaw/pull/75128 is still open rather than merged.

## What This Changes

The issue asks OpenClaw to prevent fallback models from echoing BOOT.md startup instructions to users instead of executing them silently.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/75128, or an equivalent layered fix that wraps BOOT.md as internal runtime context and suppresses substantial boot echoes before final replies and message-tool sends.

## Reproduction Assessment

Yes. Source inspection on current main shows BOOT.md content is embedded in a normal prompt, reused during model fallback, and not filtered in the message-tool send path before delivery.

## Solution Assessment

Yes in direction. The linked layered approach is the maintainable fix: mark BOOT.md as internal runtime context and add a narrow boot-echo guard at final reply and message-tool delivery boundaries.
