## What This Changes

The issue reports that fallback models can echo BOOT.md startup instructions to users instead of executing them silently, exposing internal prompt text.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/75128 or an equivalent layered boot-context and message-tool sanitizer fix, then close this issue as implemented.

## Reproduction Assessment

Yes, at source level. Current main embeds BOOT.md in the prompt body, preserves that body for fallback attempts, and forwards message-tool sends without a boot/internal-context echo guard.

## Solution Assessment

Yes. The maintainable solution is the linked layered fix or equivalent: wrap BOOT.md as internal runtime context and suppress substantial boot echoes before final replies or message-tool sends.
