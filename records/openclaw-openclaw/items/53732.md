## Summary

Keep open. Current main still exposes the reported path: BOOT.md content is embedded as ordinary model prompt text, fallback retries preserve that body, and the message tool only strips reasoning tags before outbound delivery. The issue is also now paired with open closing PR #75128, so it should remain open until that PR lands or is closed.

## What This Changes

The issue reports that model fallback during BOOT.md startup can echo BOOT.md startup instructions verbatim to users instead of executing them silently.

## Best Possible Solution

Review and land #75128, or an equivalent narrow fix, so BOOT.md startup context is marked internal and stripped before user-visible final replies or message-tool sends while preserving legitimate BOOT.md-directed outbound notifications.

## Reproduction Assessment

Yes. A high-confidence mock reproduction can pass BOOT.md content through `buildBootPrompt`, preserve it through fallback retry handling, and observe that message-tool text fields currently have no BOOT.md/internal-context suppression before delivery.

## Solution Assessment

Yes. Keeping this issue open while the active closing PR is reviewed is the right path; closing now would be premature because the fix is not on current main.
