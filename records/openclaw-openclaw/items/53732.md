Keep open. The issue requests preventing fallback boot runs from exposing BOOT.md startup instructions, especially literal echoes delivered through user-visible replies or the message tool.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/75128, or an equivalent layered fix that wraps BOOT.md as internal runtime context and suppresses substantial boot echoes at final-reply and message-tool delivery boundaries.

## Solution Assessment

Yes in direction. The linked repair path is the narrow maintainable fix: treat BOOT.md as internal runtime context and add a boot-echo guard at user-visible delivery boundaries.
