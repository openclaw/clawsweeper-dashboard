## Summary

Keep open for maintainer/security decision: current main already handles the linked local-media failure through scoped roots, but this PR adds a broader Feishu-owned localRoots config with an unrestricted readFile bypass that is not yet approved or fully surfaced.

## What This Changes

The branch adds Feishu top-level/account `localRoots` config, resolves it during outbound media loading, injects raw `readFile` for `"any"`, and adds media tests.

## Best Possible Solution

Keep scoped `mediaLocalRoots` as the supported fix unless maintainers approve a documented, narrower Feishu local-roots config surface with manifest, docs, changelog, and security coverage.

## Reproduction Assessment

Partly. Source inspection confirms current main does not implement the proposed Feishu `localRoots` config, while current tests and code show the original local-media path is handled through scoped `mediaLocalRoots`.

## Solution Assessment

No. The PR is broader than the linked bug fix because it adds a new config option and an unrestricted read bypass; the safer path is maintainer-approved scoped configuration or keeping the existing scoped roots.
