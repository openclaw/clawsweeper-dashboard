## Summary

Keep open; current main already fixes the linked local-media send failure through scoped `mediaLocalRoots`, while this PR adds a new Feishu config option and an unrestricted `"any"` filesystem-read bypass that needs maintainer/security judgment before merge.

## What This Changes

The PR adds `channels.feishu.localRoots`/account `localRoots` support for Feishu media sends, injects a raw `readFile` override for `"any"`, and adds tests for `"any"`, explicit arrays, and empty arrays.

## Best Possible Solution

Keep the scoped `mediaLocalRoots` behavior as the supported fix unless maintainers explicitly approve a documented, narrower Feishu local-roots config surface with security review.

## Reproduction Assessment

Partly. Source inspection confirms current main does not support the PR's new `channels.feishu.localRoots` key, but the original linked local-media failure is already addressed by the scoped `mediaLocalRoots` pipeline.

## Solution Assessment

No. The PR is broader than the linked bug fix because it exposes an unrestricted Feishu read bypass; a maintainer-approved scoped config design, with docs/manifest/changelog updates, would be safer if this capability is still desired.
