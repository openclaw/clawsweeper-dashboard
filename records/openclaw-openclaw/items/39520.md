## Summary

Keep open. Current main still lacks Feishu-owned `channels.feishu.localRoots` and account-scoped local media roots, and the PR remains a focused implementation candidate, but it is not merge-ready because the new config surface is missing generated config metadata and public documentation for the security-sensitive `"any"` mode.

## What This Changes

The PR adds Feishu `localRoots` config support for media uploads, resolves roots from merged account config, injects a raw `readFile` override for explicit `"any"`, and adds focused media handling tests.

## Best Possible Solution

Land this PR or an equivalent Feishu-owned patch after adding generated config metadata, config baseline updates, public Feishu docs for top-level and account-scoped roots, and explicit security review for `"any"`; coordinate with #51375 so reply-path scoped roots and channel/account override roots work together without widening access unexpectedly.

## Reproduction Assessment

Yes. A focused current-main check is to configure `channels.feishu.localRoots: "any"` and call `sendMediaFeishu` with a local path: current main never reads the merged Feishu config and core would require a `readFile` override for the `"any"` bypass.

## Solution Assessment

Not yet. The PR’s owner-boundary approach is the right shape for Feishu-specific behavior, but the mergeable solution also needs generated config metadata, public docs/security warning, and coordination with the adjacent reply-path forwarding PR.
