## Summary

Keep open: current main does not implement the proposed Feishu `localRoots` config, and the PR still needs maintainer/security approval because it adds an unrestricted local-file read bypass rather than only relying on the shipped scoped media-root path.

## What This Changes

The PR adds Feishu top-level/account `localRoots` config, uses it during media uploads including `"any"`, and adds Feishu media handling tests.

## Best Possible Solution

Keep the shipped scoped `mediaLocalRoots` behavior for the linked failure, and add Feishu `localRoots` only with a maintainer-approved config design and complete public docs/schema coverage.

## Reproduction Assessment

Partly. Source inspection confirms current main does not accept `channels.feishu.localRoots`, while the linked `LocalMediaAccessError` path is already covered by scoped `mediaLocalRoots` propagation and a regression test.

## Solution Assessment

No. The PR is broader than the linked fix because it exposes an unrestricted read bypass; a narrower documented design, or explicit security approval for that bypass, is needed first.

## Review Findings

Overall correctness: patch is incorrect
