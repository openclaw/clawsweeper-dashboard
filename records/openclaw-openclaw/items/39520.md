## Summary

Keep open. Current main still lacks Feishu `localRoots` support, and this PR addresses the linked Feishu local-media failure, but the patch is not merge-ready because the new public config surface has stale generated metadata/docs, no changelog entry, and an intentionally security-sensitive `"any"` local-read mode that needs user-facing documentation and maintainer review.

## What This Changes

The PR adds a Feishu `localRoots` config field, resolves merged account config roots for `sendMediaFeishu` including an `"any"` readFile bypass, and adds media handling tests for `"any"`, array override, and empty-array behavior.

## Best Possible Solution

Land this PR or an equivalent Feishu-owned patch after adding generated channel config metadata, config docs baseline updates, a public Feishu docs/config-reference section at https://docs.openclaw.ai/channels/feishu, a user-facing changelog entry, and explicit maintainer signoff for the `"any"` local-read escape hatch; coordinate with #51375 so account/channel override roots and reply-path media roots compose without widening access unexpectedly.

## Reproduction Assessment

Yes. A high-confidence focused check is to configure `channels.feishu.localRoots: "any"` or an account-scoped `localRoots` value and call `sendMediaFeishu` with a local path; current main has no Feishu schema field for it and only forwards caller-provided `mediaLocalRoots` to `loadWebMedia`.

## Solution Assessment

Not yet. The PR's Feishu-owned implementation shape is the right direction because it reads the merged account config and satisfies core's `"any"` readFile requirement, but the mergeable solution must also update generated config surfaces, public docs, changelog, and get security review for the unrestricted read mode.
