## Summary

Keep open. Current main still lacks Feishu `localRoots` support, and this PR addresses the linked local-media failure, but it is not merge-ready because the new public config surface needs generated metadata/docs/changelog follow-through and maintainer review for the unrestricted `"any"` local-read mode.

## What This Changes

The PR adds a Feishu `localRoots` config field, resolves merged account roots for `sendMediaFeishu` including an `"any"` readFile bypass, and adds media handling tests for `"any"`, array override, and empty-array behavior.

## Best Possible Solution

Land this PR or an equivalent Feishu-owned patch after adding generated channel config metadata, config docs baseline updates, a public Feishu docs/config-reference section at https://docs.openclaw.ai/channels/feishu, a user-facing changelog entry, and explicit maintainer signoff for the `"any"` local-read escape hatch.

## Reproduction Assessment

Yes, from source inspection. Configure `channels.feishu.localRoots: "any"` or an account-scoped `localRoots` value and call `sendMediaFeishu` with a local path; current main has no Feishu schema field for it and only forwards caller-provided `mediaLocalRoots` to `loadWebMedia`.

## Solution Assessment

Not yet. The Feishu-owned implementation shape is directionally right because it reads merged account config and satisfies core's `"any"` readFile requirement, but the mergeable solution must also update generated config surfaces, public docs, changelog, and get maintainer security review.
