## Summary

Keep open: the PR addresses a real explicit-targeting problem, but the current diff is not mergeable because it changes the shared `sessions_spawn` contract and blocks documented ACP default-agent spawns. Current `main` already has a config-backed strict mode for requiring explicit targets without making the global schema mandatory.

## What This Changes

The branch makes `sessions_spawn.agentId` required in the shared tool schema and execution reader, then updates spawn-related tests and mocks to pass explicit agent IDs.

## Best Possible Solution

Preserve ACP `acp.defaultAgent` compatibility while improving native subagent target reliability through the existing `requireAgentId` strict mode, clearer prompt/schema guidance, or an additive runtime-specific schema path.

## Reproduction Assessment

Yes. A high-confidence static reproduction is `sessions_spawn({ runtime: "acp", task: "..." })` with `acp.defaultAgent` configured: current `main` forwards missing `agentId` to ACP resolution, while the PR rejects it in the shared reader first.

## Solution Assessment

No. A global shared-schema requirement is not the narrowest maintainable fix because ACP intentionally supports omitted `agentId`; the safer direction is config-backed strictness or runtime-specific schema behavior.
