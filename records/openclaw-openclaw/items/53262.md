## What This Changes

The branch makes `sessions_spawn.agentId` required in the shared tool schema/runtime reader and updates spawn-related tests and mocks to pass explicit agent IDs.

## Best Possible Solution

Preserve ACP `acp.defaultAgent` compatibility while improving explicit native subagent targeting through the existing `requireAgentId` path or runtime-specific validation/schema guidance.

Yes for the PR-introduced regression: source inspection shows the required shared reader would reject before ACP can resolve `acp.defaultAgent`. The original model-omission improvement is not independently reproduced from attached real behavior proof.

## Solution Assessment

No. A global required field is not the narrowest maintainable fix because the shared tool intentionally supports ACP default fallback and current main already has config-backed strict mode for native subagent omission.
