## Summary

Keep open: the PR is not obsolete, but the proposed global `agentId` requirement regresses the documented ACP default-agent contract and still needs real behavior proof plus maintainer API-contract direction before merge.

## What This Changes

The PR makes `sessions_spawn.agentId` required in the shared tool schema/runtime reader and updates spawn-related tests and mocks to pass explicit agent IDs.

## Best Possible Solution

Preserve ACP `acp.defaultAgent` compatibility while improving explicit native subagent targeting through the existing `requireAgentId` config path, stronger prompt/schema guidance, or runtime-scoped validation.

## Reproduction Assessment

Yes for the PR-introduced regression: source inspection shows the required reader executes before ACP dispatch, while current main still lets ACP resolve `acp.defaultAgent`. The original model-omission improvement is not independently reproduced from attached real behavior proof.

## Solution Assessment

No. A global shared required parameter is not the narrowest maintainable fix because current source/docs intentionally allow ACP default fallback and native explicit targeting already has config-backed `requireAgentId`.
