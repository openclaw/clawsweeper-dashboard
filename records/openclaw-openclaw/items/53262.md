## Summary

Keep this PR open for maintainer API-contract direction, but it should not merge as written: current main deliberately keeps `sessions_spawn.agentId` optional so ACP can use `acp.defaultAgent`, while the related native strict-mode guardrail has already landed through a narrower config-backed path.

## What This Changes

The PR makes `sessions_spawn.agentId` required in the shared tool schema/runtime reader and updates spawn-related tests and mocks to pass explicit agent IDs.

## Best Possible Solution

Preserve ACP `acp.defaultAgent` compatibility while improving native explicit-target reliability through the existing `requireAgentId` config path, stronger prompt/schema guidance, or a runtime-scoped schema contract.

## Reproduction Assessment

Yes. A high-confidence source reproduction for the PR regression is `sessions_spawn({ runtime: "acp", task: "..." })` with `acp.defaultAgent` configured: current main reaches the ACP fallback, while the PR's shared required reader rejects before ACP dispatch.

## Solution Assessment

No. A global shared-schema requirement is not the narrowest maintainable fix because current docs and code intentionally allow omitted ACP `agentId`; the safer direction is config-backed or runtime-scoped enforcement.
