## Summary

Keep open: current main still intentionally accepts omitted `agentId`, and the PR's global schema/reader requirement would regress ACP callers that rely on `acp.defaultAgent`. The branch needs maintainer direction on the shared ACP/native contract plus real behavior proof before merge.

## What This Changes

The PR makes `sessions_spawn.agentId` required in the shared tool schema/runtime reader and updates spawn-related tests to pass explicit agent IDs.

## Best Possible Solution

Preserve ACP `acp.defaultAgent` compatibility while improving explicit native subagent targeting through the existing `requireAgentId` config path, stronger prompt/schema guidance, or runtime-scoped validation.

## Reproduction Assessment

Yes for the PR-introduced regression: source inspection shows the required reader executes before ACP dispatch, while current main lets ACP resolve `acp.defaultAgent`. The original Haiku omission-rate claim is not independently reproduced from attached proof.

## Solution Assessment

No. A global shared schema/reader requirement is not the narrowest maintainable fix because current source, docs, and prompts intentionally allow ACP default fallback; config-backed or runtime-scoped enforcement is safer.
