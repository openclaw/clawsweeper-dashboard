## Summary

Keep open. The PR targets a real explicit-agent reliability problem, but the current patch is not mergeable as-is because it makes `agentId` globally required and regresses the documented ACP `acp.defaultAgent` fallback; current main already has a config-backed strict mode for omitted `agentId`, and maintainer API-contract direction plus real behavior proof are still missing.

## What This Changes

The branch makes `sessions_spawn.agentId` required in the shared tool schema/runtime reader and updates spawn-related tests and mocks to pass explicit agent IDs.

## Best Possible Solution

Preserve ACP `acp.defaultAgent` compatibility while improving explicit native subagent targeting through the existing `requireAgentId` config path, stronger schema/prompt guidance, or runtime-scoped validation.

## Reproduction Assessment

Yes for the PR-introduced regression: source inspection shows the required shared reader would run before ACP dispatch, while current main still lets ACP resolve `acp.defaultAgent`. The original model-omission improvement is not independently reproduced from attached real behavior proof.

## Solution Assessment

No. A global required field is not the narrowest maintainable fix because the shared tool intentionally supports ACP default fallback and current main already has config-backed strict mode for omitted native subagent targets.
