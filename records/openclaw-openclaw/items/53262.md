## Summary

Keep open. Current main does not implement the PR's global required-schema change, and the proposed patch still breaks the documented ACP `acp.defaultAgent` omission contract; the related native strict-mode path already exists through `requireAgentId`, so this needs maintainer contract direction before merge.

## What This Changes

The PR makes `sessions_spawn.agentId` required in the shared tool schema and runtime reader, then updates spawn-related tests and mocks to pass explicit agent IDs.

## Best Possible Solution

Preserve the shared `sessions_spawn` contract for ACP defaults, and improve native subagent target reliability through the existing `requireAgentId` strict mode, clearer prompt/schema guidance, or an additive runtime-specific schema path that does not block ACP `acp.defaultAgent`.

## Reproduction Assessment

Yes. A high-confidence static reproduction is `sessions_spawn({ runtime: "acp", task: "..." })` with `acp.defaultAgent` configured: current main forwards the missing `agentId` to ACP resolution, while the PR's required reader rejects it before `spawnAcpDirect` can use the default.

## Solution Assessment

No. A global shared-schema requirement is not the narrowest maintainable fix because ACP intentionally supports omitted `agentId`; a runtime-specific or config-backed strict path is safer.
