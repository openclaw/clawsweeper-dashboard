## Summary

Keep open: the PR conflicts with the current ACP default-agent contract and needs maintainer API-contract direction before it can merge or be replaced.

## What This Changes

The PR makes `sessions_spawn.agentId` required in the shared tool schema/runtime reader and updates spawn-related tests to pass explicit agent IDs.

## Best Possible Solution

Preserve ACP `acp.defaultAgent` compatibility while improving explicit target reliability for native subagents through a scoped contract change, prompt/schema guidance, or the existing strict config path.

## Reproduction Assessment

Yes. A high-confidence source reproduction is `sessions_spawn({ runtime: "acp", task: "..." })` with `acp.defaultAgent` configured: current main reaches the ACP fallback, while the PR rejects it in the shared reader before ACP dispatch.

## Solution Assessment

No. A global shared-schema requirement is not the narrowest maintainable fix because ACP intentionally supports omitted `agentId`; the safer direction is runtime-scoped schema behavior, clearer prompt guidance, or config-backed strictness.
