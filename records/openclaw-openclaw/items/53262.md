## Summary

Keep this PR open, but it should not merge as-is: the patch makes a shared `sessions_spawn` field required and would break the documented ACP `acp.defaultAgent` fallback while current main already has config-backed strict mode for native subagent spawns.

## What This Changes

The PR makes `sessions_spawn.agentId` required in the shared tool schema/runtime reader and updates spawn-related tests to pass explicit agent IDs.

## Best Possible Solution

Preserve ACP `acp.defaultAgent` compatibility while improving explicit native subagent targeting through the existing `requireAgentId` config path, stronger prompt/schema guidance, or runtime-specific validation that does not block documented ACP omission.

## Reproduction Assessment

Yes for the PR-introduced regression: source inspection shows the required shared reader would run before ACP dispatch, while current main still lets ACP resolve `acp.defaultAgent`. The original model-omission improvement is not independently reproduced from attached real behavior proof.

## Solution Assessment

No. A global required field is not the narrowest maintainable fix because the shared tool intentionally supports ACP default fallback and current main already has config-backed strict mode for omitted native subagent targets.
