## Summary

Keep open: current main deliberately preserves optional `agentId` for ACP default-agent fallback and already has config-backed strict native subagent enforcement; the branch needs maintainer contract direction plus real behavior proof before merge.

## What This Changes

The PR makes `sessions_spawn.agentId` required in the shared tool schema/runtime reader and updates spawn-related tests and mocks to pass explicit agent IDs.

## Best Possible Solution

Preserve ACP `acp.defaultAgent` compatibility while improving native explicit-target reliability through the existing `requireAgentId` path, stronger guidance, or a runtime-scoped schema contract.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction for the introduced regression: `sessions_spawn({ runtime: "acp", task: "..." })` with `acp.defaultAgent` configured reaches the ACP fallback on current main, while the PR's shared required reader rejects before ACP dispatch.

## Solution Assessment

No. A global shared-schema requirement is not the narrowest maintainable fix because current docs and code intentionally allow omitted ACP `agentId`; config-backed or runtime-scoped enforcement is safer.
