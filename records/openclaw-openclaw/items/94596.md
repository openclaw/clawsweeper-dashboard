## Best Possible Solution

Keep this issue open and fix the SSH tunnel caller to scope the preferred-port probe to `127.0.0.1` while preserving the existing `EADDRINUSE` ephemeral-port fallback.

## Reproduction Assessment

## Solution Assessment

Yes. Passing `127.0.0.1` at the SSH caller is the narrowest maintainable fix because it matches the caller's own bind/connect surface without changing the global helper default.
