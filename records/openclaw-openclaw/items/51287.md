## Summary

Keep open: current main increased the default exec approval expiration to 30 minutes, but the requested user-configurable approval timeout is still not implemented or documented, and the related implementation PRs are closed unmerged.

## What This Changes

Allow users to configure how long exec approval requests remain pending instead of relying on the built-in default.

## Best Possible Solution

Add a validated shared exec approval expiration setting, preserve the current 30-minute default, thread it through gateway/node/CLI approval request and wait budgets, and document the security tradeoff.

## Reproduction Assessment

Yes: source inspection gives a high-confidence reproduction path for the feature gap. Adding the proposed timeout fields has no supported schema/resolver path, while exec approval requests continue to use the default constants.

## Solution Assessment

Yes: a shared config-backed timeout is the narrowest maintainable direction. Channel-specific or RPC-only tuning would leave gateway, node, CLI, and native approval clients inconsistent.
