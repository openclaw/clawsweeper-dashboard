## Summary

Keep open: current main and v2026.5.3 still use a fixed exec approval expiration, with no supported config field or resolver path for overriding it.

## What This Changes

The issue requests an `openclaw.json` setting to override the fixed exec approval request timeout.

## Best Possible Solution

Add a validated shared exec approval expiration setting, keep the 30-minute default, thread it through request registration and wait budgets, and document the operational tradeoffs.

## Reproduction Assessment

Yes: source inspection gives a high-confidence reproduction path for the missing capability. Adding a proposed timeout field today has no supported schema/resolver path, while approval requests continue to send the default constants.

## Solution Assessment

Yes: a shared config-backed timeout is the narrowest maintainable direction. Channel-specific tuning would leave gateway, node, CLI, and native approval clients inconsistent.

Prefer the shared `tools.exec` config surface alongside `tools.exec.timeoutSec` and `tools.exec.approvalRunningNoticeMs` unless maintainers specify a different name; keep the current 30-minute default. Add typed config/schema/help/labels, resolve global and per-agent values, pass the value through `ExecToolDefaults`, `resolveExecConfig`, approval request context creation, `registerExecApprovalRequest`, wait/request transport budgets, and gateway/node approval paths that depend on the same timeout.
