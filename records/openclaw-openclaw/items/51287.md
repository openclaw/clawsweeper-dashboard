## Summary

Keep open. Current main and the latest release still expose only a fixed 30-minute exec approval default plus an internal RPC timeout parameter; there is no supported user config field, and the related implementation PRs were closed unmerged.

## What This Changes

Allow users to configure exec approval expiration timeouts instead of relying on the fixed default.

## Best Possible Solution

Expose a shared exec approval expiration setting, preferably alongside the existing `tools.exec` timing knobs, validate it in the config schema, resolve global and per-agent values, thread it through `ExecToolDefaults` into the approval request payload and wait budgets, and document the default plus security implications.

## Reproduction Assessment

Yes: static inspection reproduces the gap. Adding either proposed config key has no supported schema path, and the agent request path always sends the default constant to `exec.approval.request`.

## Solution Assessment

Yes: a shared config-backed timeout is the narrowest maintainable solution. A channel-specific or RPC-only timeout would leave gateway, node, CLI, and native approval clients inconsistent.

## Repair Work Prompt

Fix the configurable exec approval timeout feature requested in https://github.com/openclaw/openclaw/issues/51287. Implement a supported user config field for exec approval expiration, preferably `tools.exec.approvalTimeoutMs` because `tools.exec` already owns exec timing/policy and `approvals` currently owns forwarding. Preserve the current default of `DEFAULT_EXEC_APPROVAL_TIMEOUT_MS = 1_800_000`. Add types, Zod validation, schema labels/help, docs, and a changelog entry with reporter credit. Resolve global and per-agent values in `resolveExecConfig`, add the value to `ExecToolDefaults`, thread it into `RequestExecApprovalDecisionParams`, `buildExecApprovalRequestToolParams`, `registerExecApprovalRequest`, `waitForExecApprovalDecision`, and any CLI/node approval transport budget that must be at least approval timeout plus buffer. Do not add a channel-specific setting and do not change approval policy defaults. Cover gateway/agent config plumbing with focused tests, including that a configured value changes the request payload and wait RPC timeout while the default remains 30 minutes.
