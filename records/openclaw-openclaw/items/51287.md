## Summary

Keep open: current main and v2026.6.10 still lack a supported config or environment override for the exec approval wait timeout. This remains the canonical request, but it needs maintainer product/security judgment because it changes how long host-exec approvals remain actionable.

## What This Changes

The issue requests a supported configuration override for the exec approval wait timeout instead of relying on the built-in default.

## Best Possible Solution

Define one stable, bounded approval-timeout contract after maintainer product/security agreement, preserve the current default unless intentionally changed, and thread the resolved value consistently through registration, expiry, wait budgets, docs, and tests.

## Reproduction Assessment

Yes, at source level: current main has no public approval-timeout config or env key while the runtime uses fixed timeout constants. No live approval flow is required to prove the missing public config surface.

## Solution Assessment

Unclear as proposed: configurability is plausible, but exposing internal constant names is not the right public contract. Maintainers still need to choose the key, bounds, precedence, and security semantics.
