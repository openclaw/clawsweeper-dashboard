## Summary

Keep open: current main and the latest release have raised the default exec approval window to 30 minutes, but they still do not provide a supported configurable timeout surface. The remaining work is a feature/API and security policy decision, not conservative cleanup.

## What This Changes

Allow users to configure the exec approval timeout through OpenClaw configuration instead of relying on a fixed default.

## Best Possible Solution

Add a bounded, documented maintainer-chosen approval-timeout config surface, preserve the current 30-minute default, thread it through all approval registration/wait/transport paths, and cover schema, propagation, and expiry behavior with tests.

## Reproduction Assessment

Yes, at source level. Current main and v2026.5.12 have no accepted config key for approval timeout, and the runtime paths still use fixed constants rather than a resolved configuration value.

## Solution Assessment

Unclear as proposed. A configurable timeout is plausible, but exposing internal constant names is not a stable public API; maintainers should choose the key, precedence, bounds, and security contract first.
