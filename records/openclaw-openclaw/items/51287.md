## Summary

Keep open: current main and the latest release still use a fixed 30-minute exec approval timeout with no supported config key, while the related implementation PRs are closed unmerged after incomplete wiring and unresolved API/security concerns.

## What This Changes

The issue requests a user-configurable exec approval timeout for openclaw.json instead of the fixed approval-expiration window.

## Best Possible Solution

Add a bounded, documented maintainer-chosen approval-timeout config surface, preserve the current 30-minute default, thread the resolved value through registration/wait/transport paths, and cover schema, docs, propagation, and expiry behavior with focused tests.

## Reproduction Assessment

Yes, at source level: current main and v2026.5.18 have no accepted approval-timeout config key, and the agent/gateway paths still use fixed timeout constants unless an internal caller passes timeoutMs directly. No live approval flow was needed to prove the missing config surface.

## Solution Assessment

Unclear as proposed. A configurable timeout is plausible, but exposing internal constant names is not a stable public API; maintainers should choose the key, precedence, maximum, and security contract first.
