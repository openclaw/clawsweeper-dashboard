## Summary

Keep open: current main fixed the original two-minute pain by using a 30-minute default, but the requested user-configurable exec approval timeout is still not implemented, documented, or accepted by the strict config schema.

## What This Changes

Allow users to configure the exec approval expiration window in OpenClaw config instead of relying on a fixed default.

## Best Possible Solution

Add one maintainer-approved, bounded approval-timeout config surface, preserve the current 30-minute default, thread the resolved value through agent, gateway, node, and CLI waits, and document the security tradeoff.

## Reproduction Assessment

Yes. Source inspection shows proposed timeout fields have no supported schema/resolver path, while exec approval registration and wait paths still use fixed default constants.

## Solution Assessment

Partly. A config-backed timeout is the right direction, but exposing internal constant names is not the best public API; maintainers should choose a stable key, bounds, and propagation contract first.
