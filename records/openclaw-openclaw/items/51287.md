## Summary

Keep open: current main and v2026.5.20 still do not expose a supported exec-approval timeout setting. The original two-minute pain is mitigated by the shipped 30-minute default, and the related heartbeat fail-fast bug is now closed separately, but the remaining request is a security-sensitive public config/API decision.

## What This Changes

The issue requests a supported way to configure how long exec approval requests remain pending before timing out.

## Best Possible Solution

Define one bounded public approval-timeout contract, preserve the shipped 30-minute default, enforce conservative limits, and thread the resolved value consistently through approval registration and wait paths with focused tests and docs.

## Reproduction Assessment

Yes, at source level: current main and v2026.5.20 expose a fixed 30-minute approval timeout but no accepted config key for changing it. No live approval flow is required to prove the missing public config surface.

## Solution Assessment

Unclear as proposed. Configurability is plausible, but exposing internal constant names is not the right public contract; maintainers should choose the key, precedence, bounds, and security semantics first.
