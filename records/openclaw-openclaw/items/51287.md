## Summary

Keep open: current main and the latest release still have a fixed exec approval expiration with no supported user config field, and the related implementation PRs were closed unmerged.

## What This Changes

Allow users to override the exec approval expiration/request timeout from OpenClaw configuration instead of relying on an internal default.

## Best Possible Solution

Expose a validated shared exec approval expiration setting alongside the existing `tools.exec` timing knobs, preserve the current 30-minute default, resolve global/per-agent values, pass it through approval request and wait budgets, and document the security implications.

## Reproduction Assessment

Yes: source inspection gives a high-confidence reproduction path. Adding the proposed timeout fields has no supported schema/resolver path, while exec approval requests continue to send the default constants.

## Solution Assessment

Yes: a shared config-backed timeout is the narrowest maintainable direction. Channel-specific or RPC-only tuning would leave gateway, node, CLI, and native approval clients inconsistent.
