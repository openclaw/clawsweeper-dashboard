## Summary

Keep open: current main and v2026.5.19 still use a fixed 30-minute exec approval window with no supported config key, while the related implementation PRs were closed unmerged after incomplete wiring and unresolved API/security concerns.

## What This Changes

Allow users to configure how long exec approval requests remain pending before they expire.

## Best Possible Solution

Choose one bounded public approval-timeout config contract, preserve the current 30-minute default, thread the resolved value through agent, gateway, node, CLI, registration, wait, and transport paths, then update schema/help/docs with focused tests.

## Reproduction Assessment

Yes, at source level: current main and v2026.5.19 have no accepted approval-timeout config key, and the runtime paths still use fixed timeout constants unless an internal caller supplies timeoutMs directly. No live approval flow is required to prove the missing config surface.

## Solution Assessment

Unclear as proposed. Configurability is plausible, but exposing internal constant names is not a stable API; maintainers should choose the key, precedence, upper bound, and host-exec security contract first.
