## Summary

Keep open. Current main still routes image-capable Kimi models through the Anthropic messages defaults, and this PR is the right provider-owned direction, but two small cleanup findings remain before merge.

## What This Changes

The PR switches the bundled Kimi provider defaults, onboarding, tests, docs, and changelog from Anthropic messages at `https://api.kimi.com/coding/` to OpenAI chat completions at `https://api.kimi.com/coding/v1`, and adds Kimi endpoint metadata.

## Best Possible Solution

Keep the Kimi fix provider-owned and merge after the endpoint metadata and public docs are aligned while preserving explicit user overrides.

## Reproduction Assessment

Yes. The linked report gives a concrete Kimi Code Plan image request failure and curl comparison, and current main source shows image-capable Kimi models still routed through Anthropic messages defaults.

## Solution Assessment

Mostly yes. Switching the bundled Kimi defaults to OpenAI completions at Kimi's `/coding/v1` base URL is the narrow maintainable fix, but the PR should also repair ineffective endpoint metadata and stale public docs.
