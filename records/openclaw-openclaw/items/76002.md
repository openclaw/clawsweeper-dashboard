## Summary

Keep open. Current main still routes image-capable Kimi Coding models through the Anthropic Messages defaults, and this PR is the right provider-owned direction, but it still has two narrow cleanup items before merge: ineffective endpoint metadata and contradictory public docs.

## What This Changes

The PR switches the bundled Kimi Coding provider defaults, tests, docs, and changelog from Anthropic Messages at `https://api.kimi.com/coding/` to OpenAI completions at `https://api.kimi.com/coding/v1` for image support.

## Best Possible Solution

Land the provider-owned Kimi default switch after making or removing endpoint metadata intentionally and aligning all public Kimi Coding docs.

## Reproduction Assessment

Yes, source-backed rather than live. Current main advertises image input for `kimi-code`/`k2p5` but routes the default provider through `anthropic-messages`, and the linked issue gives a concrete failing image request plus working OpenAI-compatible curl.

## Solution Assessment

Mostly yes. Switching the Kimi plugin defaults to OpenAI completions at Kimi's `/coding/v1` base URL is the narrow maintainable fix, but the PR should repair the no-op endpoint metadata and update the remaining docs first.
