## Summary

Keep open: current main still routes the image-capable Kimi Coding provider through the Anthropic messages default, and this PR is a plausible provider-owned fix, but two narrow docs/metadata follow-ups remain before merge.

## What This Changes

The PR switches bundled Kimi Coding provider defaults, onboarding/tests, docs, and changelog from Anthropic messages at `https://api.kimi.com/coding/` to OpenAI completions at `https://api.kimi.com/coding/v1`, and adds Kimi endpoint metadata.

## Best Possible Solution

Land the Kimi plugin default switch after making endpoint metadata effective and aligning all public Kimi Coding docs, then let the linked issue close after merge.

## Reproduction Assessment

Yes. The linked #75959 report gives a concrete curl comparison, and current main source shows Kimi advertises image input while routing through the Anthropic messages provider config.

## Solution Assessment

Mostly yes. A provider-owned switch to OpenAI completions at Kimi's coding `/v1` base URL is the narrow maintainable fix, but the PR should also remove contradictory docs and make the new endpoint metadata recognizable to current core.
