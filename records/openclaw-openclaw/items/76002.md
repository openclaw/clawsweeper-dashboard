## Summary

Keep open: this PR addresses a real current-main Kimi image-routing bug, and the provider-owned endpoint switch is the right direction, but the branch still leaves ineffective endpoint metadata and contradictory public docs.

## What This Changes

The PR changes the bundled Kimi Coding provider defaults, onboarding/tests, docs, manifest metadata, and changelog to use OpenAI chat completions at `https://api.kimi.com/coding/v1` for image-capable Kimi requests.

## Best Possible Solution

Land the Kimi plugin default switch after making endpoint metadata effective or removing it, aligning all public Kimi Coding docs, and preserving explicit user overrides.

## Reproduction Assessment

Yes. The linked issue gives a concrete Kimi key plus image request path and curl comparison, and current main source shows image-capable Kimi models still routed through the Anthropic messages defaults.

## Solution Assessment

Mostly yes. A provider-owned switch to OpenAI completions at Kimi's `/coding/v1` base URL is the narrow maintainable fix, but the PR should also repair the ineffective endpoint metadata and remaining docs.
