## Summary

Keep open: the PR addresses a real Kimi image-routing bug, but the latest diff still leaves ineffective endpoint metadata and contradictory public Kimi Coding docs that should be fixed before merge.

## What This Changes

The PR switches the bundled Kimi Coding provider defaults, onboarding assertions, stream tests, provider docs, and changelog from Anthropic Messages at `https://api.kimi.com/coding/` to OpenAI completions at `https://api.kimi.com/coding/v1`, and adds Kimi endpoint metadata.

## Best Possible Solution

Land the provider-owned route switch after making the endpoint metadata effective or removing it, and align all public Kimi Coding docs with the new OpenAI-compatible default.

## Reproduction Assessment

Yes. The linked report gives a concrete curl comparison, and current main source shows Kimi advertises image input while routing the default provider through Anthropic Messages.

## Solution Assessment

Mostly yes. A Kimi-plugin default switch to OpenAI completions at `https://api.kimi.com/coding/v1` is the narrow maintainable fix and matches Kimi's documented OpenAI-compatible route, but the metadata/docs follow-through is incomplete.
