## Summary

Keep open. Current main still routes image-capable Kimi Code defaults through the Anthropic Messages path, and this PR is a plausible narrow fix, but it still needs real after-fix behavior proof plus small patch cleanup before merge.

## What This Changes

The PR switches bundled Kimi Coding provider defaults, onboarding/tests, docs, and changelog from Anthropic Messages at `https://api.kimi.com/coding/` to OpenAI chat completions at `https://api.kimi.com/coding/v1`, and adds Kimi endpoint metadata.

## Best Possible Solution

Land a rebased Kimi-plugin-owned endpoint switch that preserves `kimi-for-coding`, makes endpoint metadata effective or removes it, aligns all Kimi docs, and includes redacted real Kimi image proof.

## Reproduction Assessment

Yes. Current main source still advertises image-capable Kimi models while routing default Kimi requests through the Anthropic Messages base URL, and the linked issue provides a concrete failing/working curl comparison.

## Solution Assessment

Mostly yes. Switching the provider-owned Kimi defaults to OpenAI completions at `/coding/v1` is the narrow maintainable fix, but this PR should first fix ineffective endpoint metadata, align remaining docs, preserve current model IDs, and add real after-fix proof.
