Keep open: current main still defaults Kimi Coding to the Anthropic-compatible transport, while the PR addresses the linked image failure in the right owner plugin and matches Kimi's current OpenAI-compatible endpoint docs. ## What This Changes

The PR changes the bundled Kimi Coding provider defaults, onboarding assertions, docs, tests, and changelog from Anthropic Messages at `https://api.kimi.com/coding/` to OpenAI Chat Completions at `https://api.kimi.com/coding/v1`.

## Best Possible Solution

Land the provider-owned transport switch with endpoint metadata and docs kept consistent, then close the linked image-support issue after merge if targeted Kimi tests and any available live smoke pass.

## Reproduction Assessment

Yes. The linked issue gives a concrete curl comparison and current main source shows the mismatch: Kimi advertises image input but routes through the Anthropic-compatible provider config; live reproduction was not attempted because the needed API keys are unset.

## Solution Assessment

Mostly yes. Moving this in `extensions/kimi-coding` is the right owner boundary and the dependency transport emits OpenAI `image_url` payloads, but the best version should also align the remaining docs and register the new Kimi Code endpoint metadata.
