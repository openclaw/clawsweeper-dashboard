## Summary

Keep open: current main still accepts and resolves `memorySearch.outputDimensionality`, but the OpenAI memory embedding adapter drops it before building inline requests, batch JSONL requests, and provider cache identity. The issue is also paired with open, unmerged PR #55507, so it should stay open until that PR or a replacement lands.

## What This Changes

The issue requests that `memorySearch.outputDimensionality` be passed to OpenAI embedding requests as the OpenAI `dimensions` parameter.

## Best Possible Solution

Land an updated #55507 or replacement patch that threads `memorySearch.outputDimensionality` into the OpenAI embedding client, sends OpenAI `dimensions` for inline and batch `/v1/embeddings` requests when configured, includes the dimension in provider cache/reindex identity, and adds regression coverage for inline bodies, batch JSONL bodies, and cache-key behavior.
