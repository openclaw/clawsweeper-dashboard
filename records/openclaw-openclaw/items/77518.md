## Summary

Keep open: current main still has the CLI-only post-turn persistence gate, while WebChat intentionally does not mirror normal agent-run final replies because Pi is expected to persist them. The report’s incidents match the source shape, but this read-only pass did not establish a clean live reproduction.

## What This Changes

Ensure WebChat and other embedded-runtime agent turns persist assistant transcript entries so chat history survives reloads.

## Best Possible Solution

Add a guarded embedded-run assistant persistence fallback that writes the missing final assistant turn only when Pi did not already persist it, with regression coverage for history reload and duplicate prevention.

## Reproduction Assessment

No high-confidence live reproduction was established in this pass. Source inspection does show the CLI-only gate, embedded runner trace value, WebChat’s no-mirror agent-run branch, and two reported real-world incidents.

## Solution Assessment

Yes, the best direction is a narrow embedded-run persistence fallback with duplicate detection. Relabeling embedded runs as "cli" would blur runner semantics and is less maintainable.
