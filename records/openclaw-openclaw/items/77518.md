## Summary

Keep open: current main still has the CLI-only post-turn persistence gate, and WebChat intentionally avoids mirroring agent-run final replies because it expects Pi to persist assistant turns. The report names two live incidents and matches the current source shape, but I did not establish a clean live reproduction in this read-only pass.

## What This Changes

Investigate a reported regression where embedded-runtime WebChat assistant turns are streamed to the UI but not persisted to the session JSONL transcript.

## Best Possible Solution

Persist final assistant turns for embedded WebChat/main-session runs when Pi did not write them, using a guarded existing transcript path and regression coverage that proves reload history keeps the assistant turn without duplicating normal message_end entries.

## Reproduction Assessment

No high-confidence live reproduction was established in this pass. Source inspection does show the current CLI-only persistence gate, embedded runner trace value, WebChat’s no-mirror agent-run branch, and two reported real-world incidents.

## Solution Assessment

Yes, the best direction is a narrow embedded-run persistence fallback or repair path with duplicate detection. Relabeling embedded runs as "cli" would blur runner semantics and is less maintainable.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/77518. Current main has WebChat agent-run final replies treated as live delivery only, while agent-command only calls persistCliTurnTranscript for executionTrace.runner === "cli"; embedded results set runner: "embedded". Add a narrow persistence repair so embedded WebChat/main-session assistant turns are durable when Pi did not already append the assistant message. Do not change embedded runs to report runner "cli". Do not blindly mirror every final payload; first detect whether the current turn already has a Pi-persisted assistant message to avoid duplicate chat.history entries. Preserve provider/model/usage metadata where available, reuse existing transcript/session locking helpers, and keep compaction semantics unchanged unless a maintainer explicitly broadens the scope. Add regression coverage for an embedded agent-run final reply that reloads through chat.history/session JSONL with the assistant turn present, and coverage that normal Pi-persisted assistant turns are not duplicated. Add a CHANGELOG.md fix entry because this is user-facing transcript persistence behavior.
