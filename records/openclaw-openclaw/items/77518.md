## Summary

Keep open: current main still has the CLI-only post-turn transcript persistence gate, embedded Pi results still report `runner: "embedded"`, and WebChat intentionally avoids mirroring normal agent-run final replies into JSONL. The issue is also paired with open PR #77665, so it should remain open until that fix or a replacement lands.

## What This Changes

WebChat embedded-runtime assistant turns should be durably written to the session JSONL so reloaded history preserves assistant replies across turns and compaction.

## Best Possible Solution

Land a guarded embedded-run transcript persistence fallback, likely via #77665 or a replacement, that repairs missing assistant turns only when Pi did not already persist them.

## Reproduction Assessment

No: I did not establish a high-confidence live minimal reproduction in this read-only pass. Source inspection on current main shows the embedded runner tag, the CLI-only persistence gate, and WebChat's no-mirror agent-run branch, which matches the reported failure mode.

## Solution Assessment

Yes: the narrow maintainable direction is an embedded-run persistence fallback with duplicate detection. Relabeling embedded runs as `cli` would blur runner semantics and risk compaction/lifecycle side effects.
