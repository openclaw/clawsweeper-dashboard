## Summary

Keep open: current main still has the CLI-only post-turn transcript fallback, while WebChat deliberately avoids mirroring normal agent-run final payloads because Pi is expected to persist them. The report includes two concrete incidents and matches the current source shape, but I did not establish a clean live reproduction in this read-only pass.

## What This Changes

The issue reports that embedded-runtime WebChat turns stream assistant replies but do not persist those assistant turns to the session transcript JSONL.

## Best Possible Solution

Add guarded fallback persistence for embedded WebChat/main-session final assistant turns only when the transcript lacks the Pi assistant message, with regression coverage and a changelog entry.

## Reproduction Assessment

No live current-main reproduction was established in this pass. Source inspection shows the embedded runner value, the CLI-only persistence gate, WebChat’s no-mirror agent-run branch, and the issue documents two real sessions with assistant turns missing from JSONL.

## Solution Assessment

Yes, the best direction is a narrow embedded-run persistence fallback or repair path with duplicate detection. Retagging embedded runs as "cli" would blur runner semantics and is less maintainable.

## Repair Work Prompt

Fix the embedded-runtime WebChat assistant-turn persistence gap. Current main still gates persistCliTurnTranscript on executionTrace.runner === "cli", while runEmbeddedPiAgent reports runner: "embedded" and WebChat intentionally does not mirror normal agent-run final payloads. Add a narrow fallback/repair path for embedded WebChat or main-session runs that persists the final assistant turn only when Pi did not already write it, preserving runner semantics and avoiding duplicate assistant entries. Add regression coverage proving chat.history reload keeps the assistant turn for an embedded agent run and does not duplicate normal Pi-owned message_end entries. Add a user-facing CHANGELOG.md fix entry. Do not add a config option, do not relabel embedded runs as cli, and do not change unrelated channel delivery behavior.
