## Summary

Keep open: current main still has the CLI-only post-turn persistence gate, embedded Pi runs identify as `runner: "embedded"`, and WebChat deliberately avoids mirroring normal agent-run final delivery payloads. The source shape matches the reported assistant-turn drop, though I did not establish a clean live reproduction in this read-only pass.

## What This Changes

The issue requests durable persistence of assistant turns for WebChat and embedded-runtime sessions whose final replies stream to the UI but do not land in the session JSONL.

## Best Possible Solution

Add a narrow embedded-run assistant persistence fallback for WebChat/main-session turns when Pi did not already write the assistant message, preserving the transcript parent chain and duplicate guards.

## Reproduction Assessment

No high-confidence live reproduction was established in this pass. Source inspection does show the CLI-only persistence gate, embedded runner trace value, WebChat's no-mirror agent-run branch, and the reported real-world user-only JSONL pattern.

## Solution Assessment

Yes, the best direction is a narrow embedded-run persistence fallback with duplicate detection. Relabeling embedded runs as `cli` would blur runner semantics and is less maintainable.

## Repair Work Prompt

Fix the embedded WebChat assistant-turn persistence regression reported in https://github.com/openclaw/openclaw/issues/77518. Current main only calls `persistCliTurnTranscript` for `runner === "cli"`; embedded Pi runs set `executionTrace.runner` to `"embedded"`; WebChat intentionally does not mirror normal agent-run final delivery payloads because Pi normally owns the assistant `message_end`. Add a narrow fallback for embedded WebChat/main-session runs that persists the final assistant turn only when Pi has not already written it. Use a SessionManager/`appendSessionTranscriptMessage`-compatible path so the parentId chain remains valid, and avoid duplicating normal Pi assistant turns, aborted partials, and WebChat media supplements. Do not relabel embedded runs as CLI, add config, or broaden delivery-mirror semantics. Add/update regression coverage and a user-facing `CHANGELOG.md` fix entry.
