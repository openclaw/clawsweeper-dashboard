Keep open. Current main still has a source-provable mismatch: normal replies apply `channels.modelByChannel` for unpinned channel sessions, but `/status` only labels a channel override when the selected/default model already matches that override.

## What This Changes

The issue requests that `/status` display the effective model from `channels.modelByChannel` for the current channel/session instead of only the session-selected/default model.

## Best Possible Solution

Have the shared status path resolve the same channel override used by normal replies and render the effective model, with the selected/default model shown separately when they differ.

## Reproduction Assessment

Yes, this is source-reproducible: configure a channel override that differs from the agent default, clear session model overrides, and `/status` follows the selected/default model path while normal replies apply the channel override path.

## Solution Assessment

Yes, the best fix is in the shared status rendering/status-text path, not in model execution or config, because normal reply execution already resolves the override correctly.
