## Summary

## What This Changes

This PR passes `manifest.contracts` into bundled capability `PluginRecord`s, seeds and deduplicates contract-backed provider id fields, and adds regression coverage for bundled contract preservation.

## Best Possible Solution

Land this focused manifest-contract propagation after maintainer acceptance and green or explained current-head checks, or explicitly fold the same code and regression coverage into the broader hardening PR before closing this PR.

## Reproduction Assessment

Yes, source-level. Current main builds bundled capability `PluginRecord`s without `contracts` and then validates captured tools against `record.contracts`, while the affected bundled manifests declare `contracts.tools`.

## Solution Assessment

Yes, with a sequencing caveat. Preserving `manifest.contracts` on the synthesized bundled `PluginRecord` is the narrow manifest-first fix and aligns the bundled runtime with the normal loader record path; maintainers still need to choose sequencing with the broader hardening PR.
