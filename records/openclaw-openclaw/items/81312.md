## What This Changes

The PR preserves bundled capability plugin `manifest.contracts` on synthesized plugin records, seeds contract-backed provider id lists, dedupes runtime-captured ids, and adds focused regression coverage.

## Best Possible Solution

Land the focused manifest-contract propagation after current-head CI finishes and maintainer review accepts bundled-runtime parity with the normal loader path, then let it close the linked Feishu/Lark tool-registration bug.

## Reproduction Assessment

Yes. Source inspection shows current bundled records are built without `contracts`, while the same runtime validates captured tools against `record.contracts`; a Feishu manifest declares those tools, so the failure path is source-reproducible.

## Solution Assessment

Yes. Passing `manifest.contracts` into the bundled `PluginRecord` is the narrow manifest-first fix and aligns the bundled capability path with `createPluginRecord` in the normal loader; reading contracts from module exports would be the wrong layer for this manifest-owned contract.
