## Summary

Keep open for PR handling. The central bug fix is valid: current main drops bundled manifest contracts before bundled runtime tool validation, and this PR restores that contract metadata, but it still introduces duplicate capability provider IDs when a contract-seeded ID is also captured at runtime.

## What This Changes

The PR passes bundled manifest contracts into synthesized capability plugin records, seeds capability provider ID arrays from those contracts, and adds a memory-core regression test.

## Best Possible Solution

Land the manifest-contract propagation fix after deduplicating contract-seeded provider ID arrays, then let the merged PR close the linked bug and supersede the older overlapping PR at https://github.com/openclaw/openclaw/pull/78012.

## Reproduction Assessment

Yes. Source inspection shows current main drops manifest.contracts while creating bundled capability records, then validates captured runtime tools against record.contracts in the same bundled runtime path.

## Solution Assessment

Yes. Carrying manifest.contracts into the synthesized bundled record matches the normal loader contract and is the narrow maintainable fix; the remaining refinement is to keep seeded provider-like ID lists unique when captured registrations repeat the same IDs.
