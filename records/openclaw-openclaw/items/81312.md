## What This Changes

The PR passes manifest contracts into bundled capability plugin records, seeds contract-backed provider id arrays, and adds a bundled runtime regression test for `memory-core`.

## Best Possible Solution

Land the narrow manifest-contract propagation fix after either deduping contract-seeded provider ids or deciding that duplicate record ids are acceptable, then close the linked bug and supersede the overlapping PR as appropriate.

## Reproduction Assessment

Yes, from source inspection: current `main` drops manifest contracts when synthesizing bundled capability records, then validates captured tools against `record.contracts`. I did not run a live Feishu/Lark flow in this read-only review.

## Solution Assessment

Yes, the PR uses the narrow maintainable fix by carrying `manifest.contracts` into the bundled record like the normal loader path. The only refinement I see is to dedupe provider id arrays when seeded ids are later captured from runtime registration.
