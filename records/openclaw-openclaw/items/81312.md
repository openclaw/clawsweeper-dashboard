## Summary

Keep open: current main still drops manifest contracts in the bundled capability runtime, and this PR is a focused fix with sufficient after-fix proof. The remaining issue is a narrow PR-introduced duplicate provider-id array entry when manifest contracts and captured runtime registration name the same capability.

## What This Changes

The PR carries manifest contracts into synthesized bundled capability plugin records, seeds provider-like ID lists from those contracts, and adds a memory-core regression test.

## Best Possible Solution

Land this PR or an equivalent patch after deduplicating contract-seeded provider ID arrays; then it should close https://github.com/openclaw/openclaw/issues/77982 and supersede https://github.com/openclaw/openclaw/pull/78012.

## Reproduction Assessment

Yes from source inspection. Current main creates bundled capability records without contracts while the same runtime validates captured tools against record.contracts; the PR body supplies after-fix registry output, but this read-only review did not run a live plugin load.

## Solution Assessment

Yes. Carrying manifest.contracts into the synthesized bundled record matches the normal loader contract and is the narrow maintainable fix; the remaining refinement is to keep provider-like ID lists unique when captured registrations repeat manifest-declared IDs.
