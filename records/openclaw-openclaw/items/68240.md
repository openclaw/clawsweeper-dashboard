## What This Changes

The PR adds a repeatable `--daemon-env KEY=VALUE` option to `openclaw onboard --install-daemon` and merges parsed entries into gateway daemon install environments for non-interactive and wizard paths.

## Best Possible Solution

Add a reviewed daemon-env mechanism that normalizes portable keys, rejects dangerous host-env and override names through existing policy helpers, preserves service env provenance, documents the bootstrap use case, and includes focused tests plus real generated-unit or install proof.

## Solution Assessment

No, not as written. The bootstrap capability is useful, but the maintainable solution needs policy-backed key validation, current service-env provenance handling, and proof before user input becomes persistent daemon metadata.
