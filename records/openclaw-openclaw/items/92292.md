## Summary

The patch is useful and narrowly scoped, but the doctor warning currently conflates missing `models.providers` registration with missing provider configuration, which can misdirect users of bundled/plugin providers; the PR should remain open for that focused correction.

## What This Changes

The PR adds a doctor diagnostic for configured default models absent from the catalog and enriches `openclaw infer model inspect` errors with same-provider model suggestions.

## Best Possible Solution

Keep the generic catalog diagnostic, but describe the checked condition precisely as missing custom model registration and avoid implying that plugin authentication or provider setup is absent; track the Google catalog or migration repair separately.
