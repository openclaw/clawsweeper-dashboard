## Summary

The PR addresses a real released regression and includes credible live proof, but the new doctor check equates catalog absence with runtime unavailability even though current inference supports dynamically resolved models outside that catalog. This can produce persistent false warnings and tell users to replace a valid default model, so the patch needs a narrow correctness repair before merge.

## What This Changes

The PR adds a doctor warning when the configured default model is absent from the loaded catalog and extends `infer model inspect` errors with same-provider model suggestions.

## Best Possible Solution

Introduce one shared diagnostic that distinguishes a genuinely unrunnable configured default from a model merely absent from the catalog, reuse it in both doctor presentations, and add focused tests for removed catalog models, valid dynamic models, no configured default, and same-provider suggestions.

## Reproduction Assessment

Yes. The linked issue and PR give exact released and after-fix command paths, and current source clearly reproduces the catalog-miss decision without requiring speculative state.

## Solution Assessment

No. Better diagnostics are the right direction, but catalog membership is not the runtime execution contract; the narrow maintainable fix is a shared runtime-aware diagnostic with focused coverage.
