## Summary

Keep open: current main still uses direct Discord outbound delivery, so the gateway-routing fix remains useful, but the branch also changes the shared message CLI to make every CLI message action non-owner, which contradicts current tests and can break owner-authorized commands.

This PR changes the Discord outbound adapter to advertise gateway delivery, adds a delivery-mode assertion, and changes the shared message CLI to pass `senderIsOwner` as false for all message actions.

## Best Possible Solution
