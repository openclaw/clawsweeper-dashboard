## Summary

Keep open: the alias feature is focused, and the PR already fixed Greptile's `/ms` docs ambiguity, but the branch is not correct as implemented because `/m` does not reach model directive parsing and `/c` drops compact instructions. It also still lacks required after-fix real behavior proof.

## What This Changes

Adds short built-in slash-command aliases in the shared command registry and documents them in the slash-command docs.

## Best Possible Solution

Accept the alias feature only after directive-backed and raw-body-sensitive aliases are wired through the same execution paths as their full commands, with current docs and real channel or terminal proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `/m gpt-5` is not parsed by `extractModelDirective` and has no `/model` handler fallback, while `/c focus` dispatches compaction but `extractCompactInstructions` strips only raw `/compact`.

## Solution Assessment

No. Registry-only aliases are not the best complete solution for commands with separate directive or raw-body paths; those aliases need to be wired into the underlying parsers or omitted.
