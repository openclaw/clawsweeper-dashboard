## Summary

Keep this PR open, but it is not mergeable as-is: the alias feature is not on current main, and the PR still leaves `/m` and `/c` non-equivalent to their advertised long commands while lacking real behavior proof.

## What This Changes

The PR adds nine short built-in slash-command aliases in the shared command registry and updates slash-command documentation to list them.

## Best Possible Solution

Accept the alias feature only after aliases that map to directive-backed or raw-body-sensitive commands are wired through those execution paths, with focused regression coverage and real behavior proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with the PR applied, `/m gpt-5` is not seen by raw model directive parsing and has no `/model` handler fallback, while `/c focus` dispatches compaction but loses custom instructions because extraction strips only raw `/compact`.

## Solution Assessment

No. Registry-only aliases are not the best complete solution because model selection and compact instructions have separate parsing paths; the PR should wire those paths or omit aliases that cannot be equivalent.

## Review Findings

Overall correctness: patch is incorrect
