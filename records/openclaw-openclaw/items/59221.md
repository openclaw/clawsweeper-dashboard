## Summary

Keep open. The alias idea is focused, but the current patch still has runtime defects: `/m` is registered only as a command alias while model switching is parsed from raw inline directives, and `/c instructions` would trigger compaction without forwarding the custom instructions. The external PR also still lacks after-fix real behavior proof.

## What This Changes

The PR adds nine built-in text aliases to the chat command registry and updates slash-command documentation to list them.

## Best Possible Solution

Accept the alias set only after each alias is wired through its real execution path; `/m` needs directive support and regression coverage, and `/c` must forward compaction instructions.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `/m gpt-5` would be normalized only in the command context while model switching parses raw text, and `/c instructions` reaches compaction but the instruction extractor only recognizes raw `/compact`.

## Solution Assessment

No. Registry-only aliases are enough for ordinary command handlers, but not for inline directives or handlers that re-read raw command text; the safer path is to wire those aliases through the underlying parser paths or omit the affected aliases.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Wire `/m` through model directive parsing:** `src/auto-reply/commands-registry.shared.ts:831`
  - body: Registering `/m` as a chat-command alias only normalizes `command.commandBodyNormalized`, but model switching is parsed earlier from raw command text by `extractModelDirective`, and there is no `/model` command handler. As a result, `/m gpt-5` falls through instead of applying the model override.
  - confidence: 0.92
- **[P2] Forward `/c` instructions to compaction:** `src/auto-reply/commands-registry.shared.ts:827`
  - body: The dispatcher will normalize `/c instructions` to `/compact instructions`, but `handleCompactCommand` extracts custom instructions from the raw message and only accepts a `/compact` prefix. This means the new alias runs compaction while silently dropping the user's instructions.
  - confidence: 0.88
