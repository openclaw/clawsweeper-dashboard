The PR adds nine built-in text aliases for existing slash commands and updates the slash-command docs to list them.

## Best Possible Solution

## Reproduction Assessment

Yes from source inspection for the blocking PR behavior: `/m <model>` becomes a command alias, but model switching is parsed by `extractModelDirective()` from raw directive text and only recognizes `/model` or configured aliases.

## Solution Assessment

No. Adding `/m` only through `registerAlias()` is not the best solution because `/model` is a directive-owned model switch, not a normal command handler path.
