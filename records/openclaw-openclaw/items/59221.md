## Summary

Keep open: the alias feature is plausible and the `/ms` documentation ambiguity was addressed, but the PR head still makes `/m` a registry alias without wiring the model directive path that actually implements `/model` switching.

## What This Changes

The PR adds nine built-in text aliases for existing slash commands and updates slash-command documentation.

## Best Possible Solution

Keep the focused alias idea only if each alias is wired through the real execution path; `/m` should either be parsed as a model directive with regression coverage and live proof, or be omitted.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: with this PR, `/m gpt-5` normalizes as a command alias, but model switching is parsed earlier from raw text by `extractModelDirective`, which does not recognize `/m`.

## Solution Assessment

No. Registry-only aliases are fine for normal command handlers, but `/model` is an inline directive, so `/m` needs explicit directive support or should be left out.
