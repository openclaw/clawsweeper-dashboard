## Summary

Keep open: this focused feature PR still has a blocking command-routing defect and compatibility risk. The `/ms` docs issue was fixed, but `/m` is registered only as a text-command alias while model switching is owned by the raw inline directive parser, and the PR also expands the reserved slash-command namespace without upgrade proof for existing configured model aliases.

## What This Changes

Adds nine built-in slash-command aliases in `src/auto-reply/commands-registry.shared.ts` and documents them in `docs/tools/slash-commands.md`.

## Best Possible Solution

Repair or narrow the alias set so directive-owned commands and configured model aliases keep a single source of truth, then provide redacted real chat/gateway proof before merge.

## Reproduction Assessment

Yes. Source inspection shows `/m` would be normalized for command handlers, while model switching is parsed earlier from raw command text by `extractModelDirective()` and no `/model` handler exists afterward.

## Solution Assessment

No. Registering every shortcut in the command registry is only a plausible partial fix; directive-owned model switching and existing configured aliases need explicit handling or the risky aliases should be dropped.
