## Summary

Keep open: the feature is plausible, but the current PR still has blocking alias behavior gaps and only test/build claims rather than real after-fix behavior proof.

## What This Changes

The PR registers nine short built-in slash-command aliases in the shared command registry and updates slash-command docs to list them.

## Best Possible Solution

Keep the alias feature only after every advertised alias is wired through its real execution path, with focused regression coverage and docs that describe only equivalent aliases.

## Reproduction Assessment

Yes. Source inspection gives high-confidence paths: /m model selection goes through raw directive parsing that does not consult registry aliases, and /c custom instructions reach compact dispatch while instruction extraction only strips a raw /compact prefix.

## Solution Assessment

No. Registry-only aliases are not the best complete solution because model selection and compact instructions have separate raw-text parsing paths; the patch should wire those paths or omit aliases that cannot behave like their long forms.
