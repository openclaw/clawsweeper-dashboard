## Summary

Keep this PR open: the general alias approach is plausible, but the proposed `/m` alias is not wired into the model directive path that actually performs model switching, and external real behavior proof is still missing before merge.

## What This Changes

The PR adds short text aliases for existing built-in slash commands and updates the slash-command docs.

## Best Possible Solution

Keep the focused alias idea, but either wire `/m` through model directive parsing with regression coverage or omit `/m` until the command/directive boundary is designed deliberately.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path for the `/m` defect: with this PR, `/m gpt-5` would normalize as a command alias, but model switching is parsed earlier from raw text by `extractModelDirective`, which does not recognize `/m`. I did not run a live gateway test because this review is read-only.

## Solution Assessment

No. A registry-only alias is the narrowest correct path for normal command handlers, but `/model` is an inline directive, so `/m` needs explicit directive support or should be left out.
