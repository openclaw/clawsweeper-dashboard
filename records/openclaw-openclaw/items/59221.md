## Summary

Keep open. The ergonomics change is useful and the Greptile `/ms` documentation concern appears addressed by the follow-up docs commit, but the PR is not merge-ready because `/m` is only added to the command-registry alias map while model switching is parsed earlier as an inline directive. The branch also lacks the required changelog entry for a user-facing feature.

## What This Changes

The PR branch registers `/c`, `/ctx`, `/s`, `/n`, `/m`, `/ms`, `/h`, `/f`, and `/u` as built-in chat command aliases and updates the slash-command docs, including a follow-up split of `/model` and `/models` documentation.

## Best Possible Solution

Keep the alias feature only after the model alias path is made consistent with `/model` directive handling, with focused coverage proving `/m <model>` switches models and `/ms` still lists models. The feature should also land with the required changelog entry and contributor credit.

## Reproduction Assessment

Yes. Apply the PR and send `/m openai/gpt-4o`: `parseInlineDirectives` will not set `hasModelDirective` because it only sees literal `/model` or configured aliases, and the later command-handler loop has no `/model` handler to recover the switch.

## Solution Assessment

No. Registry-only aliases are the narrowest solution for handler-backed commands such as `/status` or `/compact`, but `/model` is directive-backed; `/m` needs to be recognized by the directive parser/alias collection, or the `/m` alias should be dropped from this PR.
