## Summary

## What This Changes

The PR registers nine short built-in text aliases for existing slash commands and documents them in the slash commands guide.

## Best Possible Solution

Keep the non-conflicting aliases only after command-specific tests and real proof, and either leave `/m` to configured model aliases or add a maintainer-approved, compatibility-tested model directive path.

## Reproduction Assessment

Yes for the review findings from source inspection: `/m` is parsed from raw command text by the model directive path, and `/c` instructions are read from raw command text by the compaction helper. I did not run tests because this review is read-only.

## Solution Assessment

No; registry aliases are fine for ordinary command handlers, but `/model` is directive-owned and `/compact` has raw-body instruction parsing. The safer path is command-specific wiring, tests, proof, and a maintainer decision on whether `/m` should be reserved at all.
