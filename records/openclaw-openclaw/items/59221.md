## Summary

Keep open: the alias feature is still useful, but the branch remains incomplete because registry-only aliases do not cover the raw parsing paths for `/m` and `/c`; the docs ambiguity noted earlier was fixed, and no duplicate or current-main implementation was found.

## What This Changes

The PR adds nine short slash-command aliases in the shared command registry and updates the slash-command docs.

## Best Possible Solution

Keep the ergonomic alias direction only after wiring every advertised alias through its actual execution path, with focused regression coverage and docs that list only aliases that behave identically to their long forms.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `/m gpt-5` is parsed by directive code that does not consult registry aliases, and `/c instructions` reaches compact dispatch while instruction extraction only strips a raw `/compact` prefix.

## Solution Assessment

No. Registry-only aliases are not the best complete solution because model switching and compact instructions have separate raw-text parsing paths; the patch should wire those paths or omit aliases that cannot be made equivalent.

## Review Findings

Overall correctness: patch is incorrect

Model selection is parsed from raw command text by `extractModelDirective`, configured aliases are filtered against built-in command tokens, and there is no `/model` command handler fallback after normalization.
