## Summary

The PR should stay open: the docs follow-up addressed the `/ms` ambiguity, but the runtime patch still leaves `/m <model>` nonfunctional because model switching is directive-backed rather than command-handler-backed. A required changelog entry is also still missing for this user-visible feature.

## What This Changes

The branch adds nine built-in chat command aliases and updates the slash-command docs, including a split between `/model` and `/models`.

## Best Possible Solution

Wire `/m` into the model directive path with focused coverage, or drop that alias, while keeping `/ms` routed to model listing and adding the required changelog entry.

## Reproduction Assessment

Yes. On the PR branch, send `/m openai/gpt-4o`: directive parsing sees raw `/m`, registry aliases are not passed to `extractModelDirective`, and there is no `/model` command handler later to apply the switch.

## Solution Assessment

No. Registry-only aliases are the narrow path for handler-backed commands, but model switching must be recognized by directive parsing or omitted from this alias set.
