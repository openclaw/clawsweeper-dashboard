## Summary

Keep open, but the patch is not merge-ready: the alias idea is useful, yet `/m` is registered as a normal text command alias even though model switching is parsed by the directive path and no `/model` command handler exists.

## What This Changes

This PR adds nine short built-in slash command aliases in the shared chat command registry and updates slash command docs to list them.

## Best Possible Solution

Land only aliases that preserve existing directive and configured-alias behavior; handle any `/m` shortcut through the model directive owner path with compatibility tests and real channel proof.

## Reproduction Assessment

Yes. Source inspection shows applying the PR would reserve `/m` in the command registry, filter a configured `m` model alias out of directive parsing, and still leave no `/model` setter handler after command normalization.

## Solution Assessment

No. Registering `/m` as a normal command alias is the wrong layer for model switching; the safer path is to drop it or make it a directive-aware shortcut with compatibility coverage.
