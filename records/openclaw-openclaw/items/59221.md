## Summary

Keep open: the alias idea is plausible, but the current diff only registers aliases and leaves two runtime paths broken: `/m` is not wired into model directive parsing, and `/c` drops compact instructions. The external PR also still needs real behavior proof before merge.

## What This Changes

The PR adds nine built-in text aliases to the shared chat command registry and updates slash-command documentation to list them.

## Best Possible Solution

Wire aliases through the actual execution paths or drop aliases whose handlers are directive-only, then add focused regression coverage and require redacted real chat or terminal proof before merge.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: `/m gpt-5` is parsed from raw text by `extractModelDirective`, not registry normalization, and `/c instructions` dispatches via the normalized body while instruction extraction rejects the raw `/c` prefix.

## Solution Assessment

No. Registry-only aliases are enough for ordinary command handlers, but not for directive-backed commands or handlers that re-read raw command text; `/m` needs directive support and `/c` needs alias-aware instruction extraction.
