## Summary

## What This Changes

Registers nine short text aliases for existing chat slash commands and updates the slash-command documentation.

## Best Possible Solution

Accept the alias feature only after directive-backed and raw-body-sensitive aliases are wired through the same execution paths as their full commands, risky aliases are omitted if needed, docs are rebased to current main, and redacted real behavior proof is added.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `/m` is only a registry alias while model switching is parsed from raw `/model` or configured aliases, and `/c` dispatches compaction after normalization while instruction extraction reads only raw `/compact` text.

## Solution Assessment

No. Registry-only aliases are not the best complete solution for directive-backed or raw-body-sensitive commands; those aliases need parser support, handler support, or omission until maintainers accept the compatibility tradeoff.
