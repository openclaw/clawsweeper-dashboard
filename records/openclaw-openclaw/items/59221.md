## Summary

Keep open: the alias feature is not on current main, and the PR still has blocking alias-routing defects plus missing real behavior proof for an external contribution.

## What This Changes

The PR adds short text aliases for existing built-in slash commands and updates the slash-command documentation.

## Best Possible Solution

Keep the alias idea, but either route `/m` and `/c` through the directive/raw-argument paths with regression coverage or omit those aliases until their ownership rules are settled.

## Reproduction Assessment

Yes from source inspection. The proposed `/m` alias is normalized only in command-handler context while model switching is parsed from raw directive text, and `/c` instructions are extracted from raw text that only recognizes `/compact`.

## Solution Assessment

No. Registering aliases centrally is fine for handler-backed commands, but it is not sufficient for `/model` directive behavior or `/compact` raw instruction parsing.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

- **[P1] Route `/m` through model directive parsing:** `src/auto-reply/commands-registry.shared.ts:834`
  - body: Registering `/m` as a text alias does not make it switch models.   - confidence: 0.91

  - confidence: 0.84
