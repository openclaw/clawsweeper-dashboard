## Summary

This PR should stay open because the proposed registry-only aliases still leave meaningful runtime defects in the command paths that do not rely solely on registry normalization, and the contributor has not provided after-fix real behavior proof. The current review evidence is strong enough to block merge, but not to close the PR because the feature remains coherent and repairable.

## What This Changes

The PR adds nine built-in slash-command aliases in the shared command registry and updates the slash-command docs to list them.

## Best Possible Solution

Keep the ergonomic alias idea, but wire each alias through the actual execution path it targets and document only aliases that work identically to their long command forms.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: /m gpt-5 is parsed by directive code that does not consult registry aliases, and /c instructions reach compact dispatch while instruction extraction still looks for the raw /compact prefix.

## Solution Assessment

No. Adding only registry aliases is not the best complete solution because /model and compact instructions have separate raw-text parsing paths; the patch should either wire those paths or omit aliases that cannot be made equivalent.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Wire `/m` through model directive parsing:** `src/auto-reply/commands-registry.shared.ts:831`
  - body: Adding the registry alias does not make `/m gpt-5` switch models because model selection is parsed from raw command text by `extractModelDirective`, which only recognizes `/model` plus configured model aliases; there is also no normal `/model` command handler to catch this after normalization.
  - confidence: 0.92
- **[P2] Forward `/c` instructions to compaction:** `src/auto-reply/commands-registry.shared.ts:827`
  - body: The normalized body can dispatch to `/compact`, but `handleCompactCommand` extracts custom instructions from the raw message and `extractCompactInstructions` only strips a raw `/compact` prefix. `/c focus on decisions` would trigger compaction while dropping the requested instructions.
  - confidence: 0.88
