## Best Possible Solution

Land the narrow row-level fix if maintainers accept the documented provider-level availability semantics; use a separate live-readiness field or probe if they want exact endpoint/model execution status.

## Reproduction Assessment

## Solution Assessment

Yes, this is the narrowest maintainable fix for the documented provider-level Auth semantics because it changes the row normalization boundary shared by configured, catalog, and discovered rows. A live readiness probe would be a separate product/API change, not a smaller fix for this display bug.
