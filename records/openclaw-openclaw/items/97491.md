## What This Changes

The PR changes `toModelRow` so local-baseUrl model rows report `available: true` when registry availability omits the exact model key, and adds a regression test for that case.

## Best Possible Solution

Land this narrow row-helper fix if maintainers accept local-marker availability semantics; if exact live readiness is desired, add a separate probe or field instead of overloading `available`.

## Reproduction Assessment

## Solution Assessment
