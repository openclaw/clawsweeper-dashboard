## Summary

Keep open. The feature is still missing on current main and this PR is the active implementation path, but the branch is not mergeable as-is because it writes Gemini safety settings into the wrong current-main request shape and lacks after-fix real behavior proof.

## What This Changes

The branch adds `models.providers.*.safetySettings`, config/schema/help/docs/changelog updates, and extra-params wrappers/tests intended to inject Gemini safety thresholds for `google-generative-ai` providers.

## Best Possible Solution

Rebase and rework this as a Google transport/provider-aware implementation that keeps the strict config/schema/docs surface, emits top-level REST `safetySettings` for direct Gemini requests, and preserves explicit request-level settings.

## Reproduction Assessment

Yes for the PR findings by source inspection: current main calls `onPayload` with a REST `GenerateContentRequest`, while the branch writes the settings under `config.safetySettings`. The underlying request is a feature/config addition rather than a broken existing contract on main.

## Solution Assessment

No. The provider-scoped config direction is good, but this branch should not solve it by writing an SDK-shaped `config.safetySettings` object from core extra-params; the safer fix belongs in the current Google transport/provider wrapper path with top-level REST payload coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Write safety settings to the REST request body:** `src/agents/pi-embedded-runner/extra-params.ts:319-320`
  - body: Current main's Google transport calls `onPayload` with the REST `GenerateContentRequest` and then sends that object directly. This code writes `safetySettings` under `payload.config`, but the REST API expects `safetySettings` as a top-level field next to `contents`, so the configured OpenClaw setting will not affect direct Gemini requests on current main. Move the injection to the current Google transport/provider path and cover the top-level request body.
  - confidence: 0.91
- **[P2] Strip nested safety wrappers before reapplying:** `src/agents/pi-embedded-runner/extra-params.ts:590`
  - body: The branch only removes the Google safety wrapper when it is the top-level `agent.streamFn`. In the real runner, cache tracing, transcript repair, tool-call cleanup, timeout, and diagnostic wrappers are added after `applyExtraParamsToAgent`, so a later call can leave the old safety wrapper nested inside and stale thresholds can still win when config changes or is removed.
  - confidence: 0.8
- **[P2] Keep Z.AI tool_stream out of non-Z.AI requests:** `src/agents/pi-embedded-runner/extra-params.ts:468`
  - body: This core wrapper writes `tool_stream=true` to every payload once installed and is never stripped on later `applyExtraParamsToAgent` calls. If the same agent stream is reused for a different provider, strict OpenAI-compatible backends can receive an unexpected top-level field; keep this behavior provider-owned or guard the wrapper by provider/model before mutating payloads.
  - confidence: 0.74
