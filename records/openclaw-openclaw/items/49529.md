## Summary

## What This Changes

This PR adds a Gemini safety-settings provider config field with schema/docs/test coverage and runner wrappers intended to inject thresholds for `google-generative-ai` providers.

## Best Possible Solution

Revise or replace this branch with a Google-transport-owned implementation that validates strict provider config, writes top-level REST `safetySettings`, preserves explicit request settings, updates docs/schema/tests, and includes redacted live Gemini proof.

## Reproduction Assessment

## Solution Assessment

No. The config/schema direction is useful, but runtime injection should happen at the Google transport boundary and write the documented top-level request field instead of adding generic runner payload wrappers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Write safetySettings to the Gemini request body:** `src/agents/pi-embedded-runner/extra-params.ts:313-320`
  - body: This wrapper creates `payload.config.safetySettings`, but OpenClaw's Google transport sends a REST `GenerateContentRequest` where request controls are top-level fields. The new config can validate while failing to populate the native Gemini request body that is actually sent.
  - confidence: 0.9
- **[P2] Scope Z.AI tool_stream injection to Z.AI payloads:** `src/agents/pi-embedded-runner/extra-params.ts:468`
  - body: The branch installs the Z.AI `tool_stream` wrapper from the generic extra-params path, and that helper writes `tool_stream=true` to every payload it sees. If the same retained stream function later serves another provider, strict OpenAI-compatible backends can receive an unexpected top-level field and reject the request.
  - confidence: 0.82
- **[P2] Strip nested safety wrappers before reapplying:** `src/agents/pi-embedded-runner/extra-params.ts:333-338`
  - body: `stripGoogleSafetySettingsWrapper` only removes a top-level safety wrapper. When later wrappers sit outside it, an older safety wrapper can remain nested and populate stale thresholds before the new wrapper can update or clear them.
  - confidence: 0.78
