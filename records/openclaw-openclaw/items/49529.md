## Summary

Keep open. The requested Gemini safety-settings config is still useful and not implemented on current main, but this PR still needs rework because the runtime change writes the wrong request shape and adds wrapper state that can leak across later provider runs.

## What This Changes

The PR adds `models.providers.*.safetySettings` config/schema/docs/tests and generic runner wrappers intended to inject Gemini safety thresholds for `google-generative-ai` providers.

## Best Possible Solution

Land or replace this PR with a provider-owned Google transport implementation that validates strict provider config, writes top-level REST `safetySettings`, preserves explicit request settings, updates docs/schema/tests, and includes redacted live Gemini proof.

## Reproduction Assessment

Not applicable as a normal bug reproduction because this is a new provider config feature. The PR blockers are source-reproducible from current main's Google REST request builder and the branch's wrapper composition.

## Solution Assessment

No. The config/schema direction is useful, but the runtime implementation should live in the Google provider transport and write the documented request-body field instead of adding generic runner wrappers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Write safetySettings to the Gemini request body:** `src/agents/pi-embedded-runner/extra-params.ts:313-320`
  - body: This wrapper creates `payload.config.safetySettings`, but OpenClaw's current Google transport sends a REST `GenerateContentRequest` where request controls are top-level fields. As written, the new config can validate while failing to populate the native Gemini request that is actually sent.
  - confidence: 0.9
- **[P1] Keep Z.AI tool_stream from leaking into later providers:** `src/agents/pi-embedded-runner/extra-params.ts:468`
  - body: The branch installs the Z.AI `tool_stream` wrapper from the generic extra-params path, and the helper writes `tool_stream=true` to every payload it sees. If the same agent later runs another provider through the retained wrapper, strict OpenAI-compatible backends can receive an unexpected top-level field and reject the request.
  - confidence: 0.86
- **[P2] Strip nested safety wrappers before reapplying:** `src/agents/pi-embedded-runner/extra-params.ts:333-337`
  - body: `stripGoogleSafetySettingsWrapper` only removes a top-level safety wrapper. Since later setup composes more wrappers around the stream function, an older safety wrapper can remain nested and populate stale thresholds before the new wrapper can update or clear them.
  - confidence: 0.82
