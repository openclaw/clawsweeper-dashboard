## Summary

Keep open: the feature is still useful and not implemented on current main, but this PR should not merge as-is because the runtime injection is at the generic runner layer, writes the wrong Gemini REST request shape, carries unrelated Z.AI wrapper churn, and still lacks real Gemini proof.

## What This Changes

This PR adds `models.providers.*.safetySettings` schema, docs, generated config baselines, tests, and runner payload wrappers for Gemini safety threshold defaults.

## Best Possible Solution

Implement the safety mapping at the Google transport boundary, write top-level REST `safetySettings`, preserve explicit request-level values, remove unrelated generic runner wrapper churn, update docs/schema/tests, and include redacted live Gemini proof.

## Reproduction Assessment

Yes for the blocking patch defects: source inspection shows the PR writes `payload.config.safetySettings` while the current Google transport serializes a REST `GenerateContentRequest`. This is not a current-main bug reproduction because the PR is adding a new config feature.

## Solution Assessment

No. The config/schema direction is useful, but the maintainable fix is a Google-transport-owned top-level REST mapping plus proof, not generic runner payload wrappers.
