## Summary

Keep open: the requested Gemini safety-settings feature is still missing from current main, but this PR needs rework before merge because its runtime injection targets a stale wrapper shape instead of the current Google REST transport and it introduces an unrelated Z.AI wrapper leak.

## What This Changes

The branch adds `models.providers.*.safetySettings` config/schema/docs/generated baselines/tests and runner wrappers intended to inject Gemini safety thresholds for Google Generative AI providers.

## Best Possible Solution

Rework or replace the branch with a provider-owned Google transport implementation that adds strict config, writes top-level REST `safetySettings`, preserves explicit request settings, updates docs/schema/tests, and includes redacted live Gemini proof.

## Reproduction Assessment

Not applicable as a normal bug reproduction because this is a new provider config feature. The PR blockers are source-reproducible from current main's REST request shape and the branch's stream-wrapper composition.

## Solution Assessment

No. The config/schema direction is useful, but the runtime change should live in the Google provider/native transport path and write the REST request shape directly instead of adding stale generic runner wrappers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Write safetySettings to the Gemini request body:** `src/agents/pi-embedded-runner/extra-params.ts:313-320`
  - body: This wrapper creates `payload.config.safetySettings`, but OpenClaw's current Google transport sends a REST `GenerateContentRequest` where `safetySettings` is a top-level field. The new config can validate successfully without changing the native Gemini request that is actually sent.
  - confidence: 0.9
- **[P1] Keep Z.AI tool_stream from leaking into later providers:** `src/agents/pi-embedded-runner/extra-params.ts:468`
  - body: The branch installs the Z.AI `tool_stream` wrapper from the generic extra-params path, and the helper writes `tool_stream=true` to every payload it sees. If the same agent later runs another provider through the retained wrapper, strict OpenAI-compatible backends can receive an unexpected top-level field and reject the request.
  - confidence: 0.86
- **[P2] Strip nested safety wrappers before reapplying:** `src/agents/pi-embedded-runner/extra-params.ts:333-337`
  - body: `stripGoogleSafetySettingsWrapper` only removes a top-level safety wrapper. Current run setup adds wrappers after `applyExtraParamsToAgent`, so an older safety wrapper can remain nested and populate stale thresholds before the new wrapper can update or clear them.
  - confidence: 0.82
- **[P3] Support the civic-integrity safety category:** `src/config/types.models.ts:59-64`
  - body: The new config type is strict but exposes only four categories, while the current Gemini API reference lists `HARM_CATEGORY_CIVIC_INTEGRITY` as supported. Users who need that threshold cannot configure it through the new field.
  - confidence: 0.76
