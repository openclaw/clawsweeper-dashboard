Overall confidence: 0.91

Full review comments:

- **[P1] Remove the duplicate generic Z.AI tool_stream wrapper:** `src/agents/pi-embedded-runner/extra-params.ts:468`
  - body: Current main already lets the Z.AI plugin own `tool_stream` wrapping, but this branch adds another core runner wrapper that unconditionally writes `tool_stream=true` and is not stripped on later provider switches. A stale retained stream function can leak this Z.AI-only field to strict non-Z.AI backends.
  - confidence: 0.86
- **[P2] Write safetySettings to the Gemini request body:** `src/agents/pi-embedded-runner/extra-params.ts:319-320`
  - body: This wrapper adds `payload.config.safetySettings`, but OpenClaw's Google transport serializes the direct REST `GenerateContentRequest`, where `safetySettings` is a top-level request field. The new config can validate while failing to set the safety settings that Gemini actually consumes.
  - confidence: 0.92
- **[P2] Strip nested safety wrappers before reapplying:** `src/agents/pi-embedded-runner/extra-params.ts:333-338`
  - body: `stripGoogleSafetySettingsWrapper` only removes a top-level safety wrapper, while the runner adds several wrappers after `applyExtraParamsToAgent` returns. On a later apply, the old safety wrapper can be nested and still populate stale thresholds before the new wrapper runs.
  - confidence: 0.8
- **[P2] Include all supported Gemini safety categories:** `src/config/types.models.ts:59-64`
  - body: The config type and strict schema only allow four category keys, but the current Google GenerateContent API reference also supports civic integrity. Users would be unable to configure that supported category through the new field.
  - confidence: 0.76
