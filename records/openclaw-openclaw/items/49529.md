## Summary

Keep open: the Gemini safety-settings request is still useful and current main does not expose this config, but this PR is not ready to merge because the runtime injection targets the wrong request shape and the PR still lacks after-fix live Gemini proof.

## What This Changes

The PR adds `models.providers.*.safetySettings` config/schema/docs/generated baselines/tests and runner payload wrappers for Gemini safety thresholds, while also adjusting Z.AI `tool_stream` wrapper handling.

## Best Possible Solution

Implement the feature at the Google transport boundary with strict config validation, top-level REST `safetySettings`, explicit request precedence, updated docs/schema/tests, and redacted live Gemini proof.

## Reproduction Assessment

Not applicable as a normal user bug because this is a new provider config feature. The PR blockers are source-reproducible: the branch writes nested `payload.config.safetySettings` while current main serializes a top-level Google `GenerateContentRequest`.

## Solution Assessment

No. The config/schema direction is useful, but the mergeable implementation should live in the Google transport/request-shaping path and should drop the unrelated generic Z.AI wrapper churn.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Write safetySettings at the Google request boundary:** `src/agents/pi-embedded-runner/extra-params.ts:313-320`
  - body: This wrapper puts defaults under `payload.config.safetySettings`, but the current Google transport serializes a `GenerateContentRequest` whose request controls are top-level fields. The config can validate while still failing to send `safetySettings` to Gemini.
  - confidence: 0.92
- **[P2] Keep Z.AI tool_stream out of the generic runner wrapper:** `src/agents/pi-embedded-runner/extra-params.ts:468`
  - body: The branch installs `createZaiToolStreamWrapper` from `applyExtraParamsToAgent`, and that helper unconditionally writes `tool_stream=true` to every payload once retained. If the same agent later serves a non-Z.AI provider, strict OpenAI-compatible backends can receive an unexpected field; current main keeps this default in the Z.AI plugin path.
  - confidence: 0.84
- **[P2] Remove nested stale safety wrappers before reapplying:** `src/agents/pi-embedded-runner/extra-params.ts:333-338`
  - body: `stripGoogleSafetySettingsWrapper` only peels a top-level wrapper, but run setup wraps `streamFn` again after `applyExtraParamsToAgent`. On a later apply call, an older nested safety wrapper can set stale thresholds before the new config can update or clear them.
  - confidence: 0.8
