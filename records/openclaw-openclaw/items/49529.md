## Summary

Keep open. Current main still lacks configurable Gemini safety settings, and related issue #12008 remains open, but this PR should not merge as-is because the diff has blocking wrapper-lifetime bugs and a changelog policy miss.

## What This Changes

The PR adds `models.providers.*.safetySettings` config/schema/docs/generated baselines/tests and runtime injection for `google-generative-ai` payloads, while also touching Z.AI `tool_stream` handling and scoped Vitest config setup.

## Best Possible Solution

Land this as a rebased, narrowed provider-config change: keep a strict documented `models.providers.*.safetySettings` contract for active `google-generative-ai` providers, preserve explicit payload-level `config.safetySettings`, avoid `google-gemini-cli`, remove stale nested wrappers on repeated runs, and split unrelated Z.AI/Vitest changes unless maintainers explicitly want them here.

## Reproduction Assessment

Yes. The current-main gap is high-confidence because the provider schema is strict and lacks `safetySettings`, and source search found no runtime injector. The PR-specific wrapper bug is reproducible by reusing an agent across runs after attempt-level wrappers have nested the older safety wrapper.

## Solution Assessment

No, not as currently shaped. Provider-scoped config with explicit-payload precedence is the right direction, but the patch needs wrapper lifetime fixes, a changelog credit, and either removal or separate validation of unrelated Z.AI/Vitest changes.
