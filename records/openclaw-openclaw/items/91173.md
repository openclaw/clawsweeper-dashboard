## Summary

Keep open for maintainer review. ## What This Changes

Adds a memory-core sync self-heal path that treats missing index identity with existing chunks as a full in-process reindex and adds unit tests for the guard and identity-state behavior.

## Best Possible Solution

Land the narrow missing-identity self-heal after maintainer acceptance of the automatic reindex behavior, and keep the valid-identity stale-handle follow-up separate.

## Reproduction Assessment

Yes by source inspection, though I did not run the live gateway repro in this read-only review. Current main and v2026.6.1 return before full reindex when identity is missing with existing chunks, and search gates invalid identity to empty results.

## Solution Assessment

Yes, this appears to be the best narrow fix for Defect A: the sync decision already owns when to call `runSafeReindex`, and limiting self-heal to `missing` avoids automatically rebuilding deliberate provider/model mismatches.
