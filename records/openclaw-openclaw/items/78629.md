## Summary

Keep open: the PR targets a valid Telegram ordering bug and supplies usable Telegram screenshot proof, but the patch is not mergeable because it changes shared clear/discard semantics to flush unsent drafts before stopping.

## What This Changes

The PR adds a `stopAndDrain()` draft stream helper, calls it from shared `stopForClear()` with a 50 ms delay, and exposes a Telegram `waitForDrain` hook.

## Best Possible Solution

Localize the final-ordering wait to Telegram final delivery or a delivery-callback drain seam while preserving shared clear/discard semantics, then cover the race and non-Telegram discard behavior with tests.

## Reproduction Assessment

No live reproduction of the intermittent Telegram race was run in this read-only review. Source inspection gives a high-confidence reproduction path for the PR-introduced regression: pending draft text is flushed from shared clear/discard before the stream is marked stopped.

## Solution Assessment

No; changing shared `stopForClear()` plus a fixed 50 ms sleep is broader than the Telegram final-send ordering bug. The safer path is a targeted Telegram drain or callback-ack boundary that does not flush discarded drafts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
