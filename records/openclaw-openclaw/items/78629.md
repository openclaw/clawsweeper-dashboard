## Summary

This PR should stay open but is not mergeable as-is: the proposed clear path can still race because it marks the stream stopped before a flush-based drain, and the contributor has not supplied after-fix real Telegram behavior proof.

## What This Changes

The PR adds a shared draft-stream `stopAndDrain()` helper, calls it from Telegram draft clearing with a 50 ms delay, and exposes an optional Telegram draft drain hook.

## Best Possible Solution

Repair the lifecycle so clearing a progress draft deterministically drains or discards pending work while awaiting in-flight Telegram API calls before the final durable send, then add focused regression coverage and real Telegram proof.

## Reproduction Assessment

No for the linked live Telegram race; the PR body has only an unchecked manual test plan. The blocking patch defect is source-reproducible because `stopForClear()` marks the stream stopped before a flush-based drain whose loop exits while stopped.

## Solution Assessment

No; the proposed helper is called after the stopped flag is set and is implemented as `flush()` then `stop()`, so it skips the wait this fix needs. A safer repair should drain before marking stopped or make the stopped-drain helper explicitly await in-flight work.
