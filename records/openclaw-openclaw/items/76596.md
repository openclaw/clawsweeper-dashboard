## Summary

Keep open. Current main still has the reported Telegram dependency gap, and open PR #76599 is explicitly linked with closing syntax, so the issue should remain open until that fix is reviewed and merged.

## What This Changes

The issue reports that the bundled Telegram channel in `openclaw@2026.5.2` fails at gateway startup because `@grammyjs/transformer-throttler` cannot resolve `bottleneck` after a global npm install.

## Best Possible Solution

Review and land #76599 or an equivalent narrow dependency-manifest fix that adds `bottleneck` to the packaged runtime dependency set, updates ownership metadata and lockfile state, and keeps a regression guard.

## Reproduction Assessment

Yes for source-level reproduction: current main loads `apiThrottler` during Telegram bot creation, the upstream throttler requires `bottleneck`, and the root and Telegram manifests do not directly declare it. I did not perform the Windows global install live reproduction in this read-only review.

## Solution Assessment

Yes. Adding `bottleneck` to the packaged runtime dependency declarations and ownership metadata is the narrow maintainable fix; the linked open PR follows that shape and should be handled before closing the issue.
