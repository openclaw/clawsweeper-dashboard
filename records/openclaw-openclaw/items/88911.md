- **[P1] Preserve DM pairing sends that include chat ids:** `extensions/imessage/src/send.ts:1065-1070`
  - body: The pairing challenge still calls `sendMessageIMessage(sender, ..., { chatId })` for direct-message notifications that include a chat id. Keep pairing on the sender/direct target path or avoid group-target gating for that pairing-only chat id.
  - confidence: 0.9

  - body: This passes the trusted requester whenever raw `opts.replyToId` is truthy, but `sanitizeReplyToId` runs later and can strip that value to undefined.
