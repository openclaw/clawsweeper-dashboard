## What This Changes

The issue requests that Signal monitor reply delivery invoke the `message_sending` hook before sending generated Signal replies.

## Best Possible Solution

Land or replace the focused Signal monitor hook fix so replies honor `message_sending` cancel, rewrite, and media metadata semantics, with targeted regression coverage, then close the issue through the merged PR.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: inbound Signal handling dispatches generated replies into `deliverReplies`, and that function sends through `sendMessageSignal` without invoking `message_sending`.

## Solution Assessment

Yes. The narrow maintainable solution is to add the hook gate in the Signal reply delivery path through the plugin SDK hook surface, matching the shared outbound and Telegram semantics.
