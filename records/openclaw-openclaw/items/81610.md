## Summary

Keep open: current main and the latest release still send Signal monitor replies directly through `sendMessageSignal` without the documented `message_sending` hook, and the linked closing PR remains open rather than merged.

## What This Changes

Signal monitor reply delivery should invoke the `message_sending` hook before each Signal reply send so plugins can rewrite or cancel outbound content.

## Best Possible Solution

Land a focused Signal monitor fix that applies `message_sending` cancel/rewrite/media metadata before `sendMessageSignal`, preserves empty-reply suppression, and adds targeted regression coverage.

## Reproduction Assessment

Yes. By source inspection, current main routes inbound Signal replies through `deps.deliverReplies`, and that function sends text/media via `sendMessageSignal` without invoking `message_sending`.

## Solution Assessment

Yes. The narrow maintainable fix is to wire the existing hook contract into Signal monitor reply delivery, then land the linked PR or an equivalent patch after its review gaps are addressed.
