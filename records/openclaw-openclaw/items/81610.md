## Summary

Keep open: current main still has the Signal reply hook bypass, and the linked closing PR is open rather than merged, so this issue should stay open until that fix lands or is replaced.

## What This Changes

Signal monitor reply delivery should run `message_sending` hooks before sending agent replies to inbound Signal messages.

## Best Possible Solution

Land a focused Signal monitor fix that applies `message_sending` cancel and rewrite semantics before text and media reply sends, with regression coverage for the reply path.

## Reproduction Assessment

Yes, by source inspection: inbound Signal replies flow from `createReplyDispatcherWithTyping` into `deliverReplies`, which sends directly through `sendMessageSignal` without the documented `message_sending` hook. I did not run a live Signal DM reproduction in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is to add the hook gate in Signal reply delivery using the plugin SDK runtime hook surface, matching the existing Telegram and shared outbound semantics. The open linked PR is the current implementation candidate and should be reviewed or replaced rather than opening a duplicate repair job.
