## Summary

Keep open: the central precedence fix is valid and current main still has the wrong order in the agent-scope helper, but the external PR still needs real behavior proof and a maintainer decision on the bundled cron-normalization and transport-diagnostic changes.

## What This Changes

The PR reorders agent-scope subagent model selection to prefer per-agent subagent model, then the global subagent default, then the agent primary model, while also adding cron string-null normalization and transport stream error propagation changes.

## Best Possible Solution

Land a focused precedence fix that aligns agent-scope and cron selection with the documented subagent-default order, backed by redacted real setup proof; carry the cron-null and transport diagnostic fixes separately unless maintainers deliberately batch them.

Yes at source level: current main checks `agentConfig.model` before `agents.defaults.subagents.model` in `resolveSubagentModelConfigSelectionResult`, and cron consumes that helper. I did not run a live repro in this read-only review.

## Solution Assessment
