## Summary

Keep open. Current main still has the low-opacity selection styling, and the PR remains useful, but the latest patch needs changes before merge: it can break selected-text readability for supported custom themes and still lacks the required changelog entry.

## What This Changes

This PR changes `ui/src/styles/base.css` to use selection-specific CSS variables and theme-scoped foreground/background overrides for higher-contrast Control UI/WebChat text selection.

## Best Possible Solution

Keep the PR open and update the selection-color approach so built-in and imported custom themes both get readable foreground/background pairs, then add the required changelog entry and land the narrow accessibility fix so the linked issue closes through the merge.

## Reproduction Assessment

Yes. Current main is reproducible by selecting text in Control UI/WebChat, where the global `::selection` rule uses the low-alpha `--accent-subtle`; the PR-specific custom-theme regression is reproducible at source level with the existing custom light theme fixture that has a near-white accent.

## Solution Assessment

Mostly yes, but not as written. A selection-specific CSS fix is the right narrow surface, but it should respect custom theme foreground tokens or generate custom selection tokens instead of hardcoding the light-mode foreground fallback.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve custom theme selection foregrounds:** `ui/src/styles/base.css:556`
  - body: The new light-mode `--selection-fg: #fff` combines with `background: var(--selection-bg, var(--accent))`, but imported custom themes do not generate `--selection-fg` or `--selection-bg`. Supported custom light themes can have a very light `--accent` (the existing test fixture uses `oklch(0.98 0.02 155)`), so selected text becomes white on a near-white highlight. Generate selection tokens for custom themes or fall back to the theme's `--accent-foreground`.
  - confidence: 0.88

  - confidence: 0.93
