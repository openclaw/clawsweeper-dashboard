## Summary

Keep open. Current main still lets explicit model-supplied `sessions_spawn.runTimeoutSeconds` override the configured default, no `minRunTimeoutSeconds` or lock config exists, and the direct fix PRs were closed unmerged.

## What This Changes

Add a subagent timeout floor or lock so model-supplied `sessions_spawn.runTimeoutSeconds` cannot undercut configured policy.

## Best Possible Solution

Add a backward-compatible optional `agents.defaults.subagents.minRunTimeoutSeconds` floor for native subagent spawns, with schema/types/docs/changelog alignment and regression coverage for explicit, omitted, and zero timeout semantics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: with configured default `120` and explicit `runTimeoutSeconds: 2`, the resolver returns `2`, and the current unit test asserts that precedence.

## Solution Assessment

Yes. An optional minimum floor is the narrowest backward-compatible solution; making the existing default authoritative would change released semantics more broadly, while a lock option can remain a later policy feature if needed.
