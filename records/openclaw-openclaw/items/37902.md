## Summary

Keep open. Current main still treats `agents.defaults.subagents.runTimeoutSeconds` as a fallback-only default, explicit `sessions_spawn.runTimeoutSeconds` still wins, and the prior `minRunTimeoutSeconds` fix PRs are closed unmerged.

## What This Changes

The issue requests a subagent timeout floor or lock so model-supplied `sessions_spawn.runTimeoutSeconds` values cannot undercut configured defaults.

## Best Possible Solution

Add a backward-compatible optional `agents.defaults.subagents.minRunTimeoutSeconds` floor for native subagent spawns, with explicit zero-timeout semantics, aligned config/schema/docs/changelog updates, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: with configured default `120` and explicit `runTimeoutSeconds: 2`, the resolver returns `2`, and the current test suite asserts that precedence.

## Solution Assessment

Yes. An optional minimum floor is the narrowest maintainable fix because it preserves current fallback behavior when unset while giving operators an enforceable lower bound.

## Repair Work Prompt

Implement a backward-compatible optional `agents.defaults.subagents.minRunTimeoutSeconds` floor for native `sessions_spawn` subagent run timeouts. Update the timeout resolver/spawn path so model-provided or default `runTimeoutSeconds` values below the configured minimum are clamped when the minimum is set; preserve existing behavior when the minimum is absent or `0`; deliberately handle explicit `runTimeoutSeconds: 0` so the chosen semantics cannot silently bypass the operator floor. Update TypeScript config types, Zod schema, schema help/labels or generated config docs as required, public docs, and CHANGELOG. Add regression coverage for explicit lower values being clamped, values above the floor passing through, no-minimum preserving current behavior, omitted timeout/default behavior, and explicit zero semantics. Do not change ACP runtime timeout semantics or the separate model-fallback hard-stop behavior.
