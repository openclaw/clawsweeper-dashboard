## Summary

Keep open. Current main still treats `agents.defaults.subagents.runTimeoutSeconds` as a fallback only, has no floor or lock config key, and the related fix PRs in the provided context are closed unmerged.

## What This Changes

The issue requests a configurable minimum or locked subagent run timeout so model-supplied `sessions_spawn.runTimeoutSeconds` values cannot undercut the configured default.

## Best Possible Solution

Add a backward-compatible optional `agents.defaults.subagents.minRunTimeoutSeconds` floor in the native subagent timeout resolver, with aligned config types, zod/generated schema, docs, changelog, and regression tests.

## Reproduction Assessment

Yes. Source inspection provides a high-confidence current-main reproduction: with configured default `120` and explicit `runTimeoutSeconds: 2`, the resolver returns `2`, and the current test suite asserts that precedence.

## Solution Assessment

Yes. The optional minimum floor is the narrowest maintainable direction supported by the report and prior PRs; making the existing default fully authoritative would be more disruptive.
