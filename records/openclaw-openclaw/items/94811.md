## Summary

## What This Changes

The PR carries Ollama memory `outputDimensionality` into the provider client, truncates embeddings before normalization, adds the dimension to runtime cache identity, and adds focused regression tests.

## Best Possible Solution

Land the narrow Ollama provider and adapter fix after real Ollama proof demonstrates reduced vector length and maintainers accept the reindex-on-identity-change behavior.

## Reproduction Assessment

Yes. Current-main source shows `outputDimensionality` reaches the Ollama memory provider options but is dropped before normalization and cache identity; the PR's mock test is a focused reproduction, though not live Ollama proof.

## Solution Assessment

Yes, this is the best bounded memory-path fix: it changes the Ollama provider boundary and identity key instead of adding a downstream workaround. Native Ollama `dimensions` support remains worth live-proofing, but local truncation avoids adding a new request-parameter compatibility risk in this PR.
