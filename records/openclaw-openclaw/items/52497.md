## Summary

Keep open. Current main still has a source-proven path where `sessions_send` can only return a bare timeout because the gateway `agent.wait` path can observe a lifecycle/model error but time out before surfacing that error to the tool layer.

## What This Changes

Request that `sessions_send` timeout responses include the target session's recent model or lifecycle error reason instead of an opaque timeout.

## Best Possible Solution

Carry bounded, sanitized pending lifecycle/model error context through `waitForAgentJob` or `agent.wait` so `sessions_send` can forward it via the existing timeout `error` field, while keeping the broader failure taxonomy separate.

## Reproduction Assessment

Yes, at source level. Current gateway coverage emits a lifecycle error during a short `agent.wait` and still expects a timeout, and source inspection shows `waitForAgentJob` can resolve `null` before the deferred error snapshot becomes visible.

## Solution Assessment

Yes for the narrow repair. The maintainable path is to carry sanitized lifecycle/model error context through the existing wait response path, not to add a new config option or implement the full failure taxonomy here.
