## Summary

Keep open. Current main still can collapse a pending lifecycle/model error into a bare `agent.wait` timeout, and `sessions_send` only forwards an error when the wait layer supplies one; related #50981 and #43095 do not close this exact diagnostic gap.

## What This Changes

The issue asks for `sessions_send` timeout responses to carry the target run's recent model or lifecycle error reason.

## Best Possible Solution

Preserve already-available lifecycle/model error context in the gateway wait path, then rely on the existing `sessions_send` timeout `error` field without adding a new taxonomy or config option.

## Reproduction Assessment

Yes, at source level. Current tests already model a lifecycle error during a short `agent.wait` and expect a timeout, and the `waitForAgentJob` timer path explains why the error is lost.

## Solution Assessment

Yes for the narrow repair. The maintainable fix is to carry the pending lifecycle/model error through `waitForAgentJob` or `agent.wait`; a complete `sessions_send` failure taxonomy should be a separate product decision.
