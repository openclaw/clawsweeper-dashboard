## Summary

Keep open. Current main still can collapse an observed lifecycle/model error into a bare `agent.wait` timeout before `sessions_send` receives an `error` field to forward; adjacent timeout-delivery work does not close this diagnostic gap.

## What This Changes

Request that `sessions_send` timeout responses include target-session model error context instead of only an opaque timeout.

## Best Possible Solution

Preserve already-observed lifecycle/model error context through `waitForAgentJob` or `agent.wait` so the existing `sessions_send` `error` field can carry it, while deferring a full failure taxonomy to a separate design item.

## Reproduction Assessment

Yes, at source level. Current main has a lifecycle-error wait path that still expects timeout, and the source shows the pending error can be delayed past the caller wait and then collapse to a bare timeout.

## Solution Assessment

Yes for the narrow repair. The maintainable fix is to carry bounded lifecycle/model error context through the existing wait response path, not to add a new config option or implement the broader failure taxonomy here.
