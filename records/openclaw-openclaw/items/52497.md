## Summary

Keep open. Current main still has a source-proven path where a lifecycle/model error is observed during `agent.wait`, but `waitForAgentJob` holds it behind retry grace and the gateway can return a bare timeout before `sessions_send` has an error string to forward; the related soft-timeout work in https://github.com/openclaw/openclaw/issues/43095 does not cover this model-error reason gap.

## What This Changes

The issue asks for `sessions_send` timeout results to include the target session's model/provider error reason when that context exists.

## Best Possible Solution

Carry bounded, sanitized pending lifecycle/model error context through `waitForAgentJob` or `agent.wait` so `sessions_send` can forward it through the existing `error` field.

## Reproduction Assessment

Yes, at source level. Current gateway coverage emits a lifecycle error during a short `agent.wait` and still expects timeout, and `waitForAgentJob` can resolve `null` before the deferred error snapshot becomes visible.

## Solution Assessment

Yes for the narrow repair. The maintainable path is to carry sanitized lifecycle/model error context through the existing wait response path, not to add a new config option or implement the complete failure taxonomy here.
