## Summary

Keep open. Current main and the latest release still have a source-proven path where `sessions_send` can return a timeout without the target run's lifecycle/model error, because `agent.wait` can observe a lifecycle error but let the wait deadline win before the deferred error snapshot is returned.

## What This Changes

The issue requests that `sessions_send` timeout responses include the target session's recent model/provider error reason instead of an opaque timeout.

## Best Possible Solution

Carry bounded, sanitized pending lifecycle/model error context through `waitForAgentJob` or `agent.wait` so `sessions_send` can forward it through the existing timeout `error` field.

## Reproduction Assessment

Yes, at source level. Current gateway coverage emits a lifecycle error during a short `agent.wait` and still expects timeout, while source inspection shows the deferred error snapshot can lose to the wait deadline.

## Solution Assessment

Yes for the narrow repair. The maintainable fix is to preserve sanitized lifecycle/model error context through the existing wait response path, not to add a new config option or implement the full failure taxonomy here.
