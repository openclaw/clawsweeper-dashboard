## Summary

Keep open. Current main still can collapse an already-observed lifecycle/model error into a bare `agent.wait` timeout before `sessions_send` sees an error to forward; https://github.com/openclaw/openclaw/issues/43095 and https://github.com/openclaw/openclaw/pull/50981 are adjacent but do not close this exact diagnostic gap.

## What This Changes

Request that `sessions_send` timeout responses include the target session's recent model error reason instead of only an opaque timeout.

## Best Possible Solution

Preserve bounded lifecycle/model error context in the gateway wait path and let `sessions_send` forward the existing `error` field; defer any broader taxonomy to a separate product decision.

## Reproduction Assessment

Yes, at source level. Current main has a test where a lifecycle `error` during a short `agent.wait` still expects timeout, and `waitForAgentJob` can resolve `null` before the delayed error snapshot is recorded.

## Solution Assessment

Yes for the narrow repair. The maintainable path is to carry already-available lifecycle/model error context through `waitForAgentJob` or `agent.wait`, not to add a new config option or full failure taxonomy here.
