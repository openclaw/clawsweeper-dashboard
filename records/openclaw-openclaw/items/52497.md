## Summary

Keep open. Current main still has a source-proven gateway wait path that can delay a lifecycle/model error past the caller wait and return a bare timeout before `sessions_send` has an error string to forward.

## What This Changes

Request that `sessions_send` timeout results include the target session's recent model error reason instead of only an opaque timeout.

## Best Possible Solution

Preserve sanitized last-known lifecycle/model error context through `waitForAgentJob` or `agent.wait` so the existing `sessions_send` `error` field can carry it, while leaving the broader failure taxonomy to separate design work.

## Reproduction Assessment

Yes, at source level. Current gateway coverage emits a lifecycle error during a short `agent.wait` and still expects timeout, and `waitForAgentJob` can resolve `null` before the deferred error snapshot becomes visible.

## Solution Assessment

Yes for the narrow repair. The maintainable path is to carry bounded lifecycle/model error context through the existing wait response path, not to add a new config option or implement the complete failure taxonomy here.
