The issue asks for long-running Discord/ACP tasks to resurface terminal and waiting status back to the originating channel/thread.

## Best Possible Solution

Keep the durable task-delivery implementation on main and let the next release carry it; open a new report only if a build containing the proof commit still leaves delivery status stuck.
