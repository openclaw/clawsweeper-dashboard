## Summary

Keep open. Current main still uses a recursive Chokidar watcher over skill roots without a depth limit or custom ignore contract, v2026.4.29 has the same relevant watcher shape, and #75518 is an open unmerged implementation candidate linked with closing syntax.

## What This Changes

The issue reports Skills Watcher EMFILE failures on macOS 15.4 with OpenClaw 2026.4.29 when a large workspace contains directories such as venv, node_modules, logs, caches, daily_stock_analysis, and quant_system.

## Best Possible Solution

Review or replace #75518 with a focused skills watcher fix that matches the bounded skill-discovery contract, adds regression coverage, and closes this issue only after the fix merges.

## Reproduction Assessment

Unclear. The report gives a concrete macOS v2026.4.29 symptom and large-workspace shape, and current source shows a plausible recursive watcher path, but this review did not run a high-file-count macOS reproduction and the exact watched directory layout is missing.

## Solution Assessment

No. The linked PR’s refresh.ts change only reorders two false-return branches and does not bound traversal or add the missing ignore/config coverage; a safer fix should limit watching to skill-definition surfaces or use explicit depth/ignore behavior aligned with discovery.
