## Summary

Keep this open. Current main still starts Chokidar on skill-root directories without a traversal depth or exact SKILL.md target list, while the reported release has the same relevant watcher shape and an open, unmerged PR already references this issue with closing syntax.

## What This Changes

The issue requests that the skills watcher stop exhausting file descriptors in very large macOS workspaces by pruning heavy directories or supporting watcher excludes.

## Best Possible Solution

Review #75518 as the current implementation candidate, but prefer a fix that aligns the watcher with skill discovery: watch root SKILL.md files plus immediate and one grouped skill-folder SKILL.md surfaces, or set an equivalent traversal bound, while keeping default heavy-directory pruning and adding regression coverage for large non-skill subtrees.

## Reproduction Assessment

Unclear. The issue gives a concrete macOS v2026.4.29 symptom and large-workspace shape, and source plus Chokidar docs show a plausible recursive watcher path, but this review did not run a live high-file-count EMFILE reproduction.

## Solution Assessment

No. The linked PR's stated ignore-order fix is not sufficient by itself because the reported release and current main already check ignore patterns before the directory early-return; the narrower maintainable fix is to bound watcher traversal or targets to the documented skill discovery surface.
