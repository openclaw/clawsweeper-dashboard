## Summary

Keep open. Current main has the fixed default ignore ordering, but the skills watcher still watches directory roots recursively through Chokidar while skill discovery is bounded to immediate skill folders and one grouped level; the linked open PR only covers the ignore-order part and is not a merged terminal fix.

## What This Changes

The issue reports a Skills Watcher EMFILE regression on macOS after upgrade and asks for default excludes or a configurable ignore mechanism for large workspace trees.

## Best Possible Solution

Keep this issue open until the watcher is bounded to the same skill layouts discovery supports, or a documented ignore mechanism is added with regression coverage; then close or revise #75518 accordingly.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection gives a plausible current-main path when a watched skills root or configured `extraDirs` contains a large non-ignored tree, because current main still watches directory roots recursively and filters after handing those roots to Chokidar.

## Solution Assessment

No. The linked PR's ignore-order fix is already present on current main, so the narrower maintainable solution is to bound watcher targets/depth to skill discovery or add a documented ignore contract if maintainers want user-controlled excludes.
