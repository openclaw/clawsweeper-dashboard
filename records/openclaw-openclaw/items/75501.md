## Summary

Keep open. Current main has a default ignored list and applies it before allowing directories, but the skills watcher still hands whole skill roots and configured extraDirs to Chokidar; the open closing PR #75518 is the active implementation thread rather than a reason to close this issue before merge.

## What This Changes

Prevent the skills watcher from exhausting file descriptors on large macOS workspaces by ignoring or otherwise avoiding traversal of large non-skill directories.

## Best Possible Solution

Keep this issue open while maintainers revise, land, or replace #75518 with a watcher that mirrors bounded skill discovery or a documented ignore mechanism with regression coverage.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection gives a plausible current-main path when a large unignored tree sits under a watched skill root or configured extraDir, because directories not matched by the fixed ignore list remain eligible for Chokidar traversal.

## Solution Assessment

No, an ignore-list-only fix is only partial. The narrower maintainable fix is to bound watcher targets to the same skill layouts discovery supports, or add a documented ignore setting if maintainers want user control.
