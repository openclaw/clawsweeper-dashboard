## Summary

Keep open. Current main and v2026.5.3 include the direct ignored-directory ordering fix, but the watcher still hands whole skill roots to Chokidar without a depth limit or bounded target set, so unignored large trees under a watched skill root or configured extraDir can still hit the reported failure mode.

## What This Changes

The issue asks for the skills watcher to avoid EMFILE failures when large non-skill directories such as virtualenvs, node_modules, logs, caches, or project trees are present near watched skills.

## Best Possible Solution

Keep the issue open until maintainers land or replace the linked PR with a bounded skills watcher that mirrors skill discovery depth/caps, or explicitly document and implement a user ignore mechanism if that is the desired product surface.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection gives a plausible current-main path when an unignored large directory sits under a watched skill root or configured extraDir, because those roots are still watched recursively.

## Solution Assessment

No. The linked PR's ignore-order fix is already present on current main, while the narrower maintainable fix for the remaining risk is to bound Chokidar traversal to the same skill layouts discovery supports or add a documented ignore setting if maintainers choose that route.
