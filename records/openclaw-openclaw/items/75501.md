## Summary

Keep open. Current main still starts Chokidar on skill-root directories without a traversal depth or exact `SKILL.md` target list, the reported v2026.4.29 release has the same relevant watcher shape, and open PR #75518 already references this report with closing syntax.

## What This Changes

The issue asks the skills watcher to stop exhausting macOS file descriptors in very large workspaces by pruning heavy directories or supporting watcher excludes.

## Best Possible Solution

Review #75518 as the current implementation candidate, but prefer a fix that aligns the watcher with skill discovery by bounding Chokidar traversal or watching only the equivalent `SKILL.md` surfaces, with focused regression coverage for large non-skill subtrees.

## Reproduction Assessment

Unclear. The report gives a concrete macOS v2026.4.29 symptom and large-workspace shape, and current source plus Chokidar docs show a plausible recursive watcher path, but this review did not run a live high-file-count EMFILE reproduction.

## Solution Assessment

No. Reordering or adding common ignore patterns alone is not enough because the reported release and current main already check ignore patterns before the directory early-return; the maintainable fix is to bound watcher traversal to the documented skill discovery contract.
