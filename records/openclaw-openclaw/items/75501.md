## Summary

Keep this open. Current main and the latest release already have common ignore patterns, but the watcher still recursively watches each skill root without a traversal depth bound, while the skills loader only discovers immediate skill directories plus one grouped level. That leaves the reported EMFILE failure plausible for large watched skill roots or extraDirs, so there is not enough evidence for implemented-on-main or cannot-reproduce cleanup.

## What This Changes

The issue asks OpenClaw to prevent the skills watcher from exhausting file descriptors on large macOS workspaces by ignoring large non-source directories or allowing custom watch excludes.

## Best Possible Solution

Harden the skills watcher so its filesystem traversal matches the bounded skills discovery contract: watch only skill roots deeply enough for root skills, direct child skills, and one grouped level, keep the existing default ignores, and add a regression test that large non-skill subtrees under a watched root are not traversed.

## Reproduction Assessment

No high-confidence live reproduction was performed. The report gives a concrete EMFILE startup symptom and file-count context, and current source provides a focused static reproduction target: `ensureSkillsWatcher` configures recursive Chokidar watching without a depth cap.

## Solution Assessment

No, adding only more default directory names is not the best fix. The safer maintainable fix is to bound watcher traversal to the same depth the loader actually discovers, with an optional follow-up only if users still need custom ignore patterns.
