## Summary

Keep open. Current main still uses a recursive Chokidar watcher over resolved skills roots without a traversal bound or custom ignore list, and the linked open #75518 uses closing syntax for this issue, so this should remain open until that PR is reviewed, revised, merged, replaced, or closed.

## What This Changes

The issue requests a skills watcher fix to avoid macOS EMFILE failures in large workspaces by ignoring bulky directories or supporting custom ignore paths.

## Best Possible Solution

Keep this issue open and resolve it through a reviewed watcher change that mirrors bounded skill discovery or adds a documented ignore mechanism, with regression coverage and a changelog entry if user-facing behavior changes.

## Reproduction Assessment

Yes from source, but not from a live macOS run. A large non-ignored directory under a watched skills root can still be traversed because directories not matching the fixed ignore list are allowed through to Chokidar.

## Solution Assessment

No for closing now. The linked PR is still open, and its stated ignore-order fix appears incomplete because current main already checks default ignores before the directory early return; the safer direction is bounded traversal or a documented ignore list.
