## Summary

Keep this open. The report is current, plausible from the skills watcher code path, and an open unmerged PR already references it with closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that OpenClaw 2026.4.29’s skills watcher can exhaust file descriptors on macOS by recursively watching large workspace trees containing directories such as `venv`, `node_modules`, caches, and logs.

## Best Possible Solution

Review #75518 against current main, then land either that PR or a narrower replacement that proves ignored directories are not descended into and, if needed, bounds watcher traversal to the same immediate-plus-one-grouped-level contract used by skill discovery.

## Reproduction Assessment

No high-confidence live reproduction was performed. The issue gives a concrete startup symptom and file-count context, and current source provides a focused static reproduction path around the skills watcher watching large skill roots or `skills.load.extraDirs`.

## Solution Assessment

Unclear that the linked PR is the complete fix, because the current release already has the ignore-order shape described by the PR while the watcher still lacks an explicit traversal-depth bound. The safer path is maintainer review of #75518 plus focused regression coverage for directory-level ignores and large non-skill subtrees.
