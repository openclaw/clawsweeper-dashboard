## Summary

Keep open. Current main still uses an unbounded Chokidar watcher over skill-root directories, the reported v2026.4.29 release has the same relevant watcher shape, and open PR #75518 still references this issue with closing syntax but has not merged.

## What This Changes

The issue reports a macOS v2026.4.29 EMFILE regression where the Skills Watcher recursively watches a very large workspace and exhausts file descriptors.

## Best Possible Solution

Revise, land, or replace #75518 with a narrow fix that bounds Skills Watcher traversal to the same immediate and one-grouped-level skill discovery contract, with regression tests and a changelog entry.

## Reproduction Assessment

Unclear. The report gives a concrete macOS 15.4, v2026.4.29, large-workspace symptom and current source shows a plausible recursive Chokidar path, but this read-only review did not run a live high-file-count macOS EMFILE reproduction.

## Solution Assessment

No for an ignore-list-only or check-reordering fix. The maintainable fix is to bound the watcher’s traversal or watched targets to supported skill discovery depth while retaining default ignored-directory pruning.
