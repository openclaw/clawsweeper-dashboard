## Summary

Keep open. Current main and the reported v2026.4.29 release still start Chokidar on whole skill-root directories without a traversal bound, and open PR #75518 still references this report with closing syntax.

## What This Changes

The issue reports a macOS Skills Watcher EMFILE regression in v2026.4.29 when large workspace skill roots contain hundreds of thousands of files.

## Best Possible Solution

Revise, land, or replace #75518 with a narrow watcher fix that bounds traversal to the supported skill discovery shape, with focused regression coverage and a changelog entry.

## Reproduction Assessment

Unclear. The report gives a concrete macOS v2026.4.29 symptom and large-file workspace shape, and source plus Chokidar docs show a plausible recursive watcher path, but no live high-file-count EMFILE run was performed.

## Solution Assessment

No for an ignore-pattern-only or stats-reordering fix. The maintainable fix is to bound watcher traversal or watch only the effective `SKILL.md` surfaces that skill discovery supports.
