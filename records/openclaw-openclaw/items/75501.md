## Summary

Keep this open. The report is current and plausible: the shipped 2026.4.29 code already has the ignore-order change described by the linked PR, while the watcher still gives Chokidar recursive skill-root targets without a depth limit, so large non-skill subtrees can still exhaust file descriptors.

## What This Changes

The issue asks OpenClaw to prevent the Skills Watcher from exhausting file descriptors on large macOS workspaces by ignoring heavy directories or exposing configurable watcher excludes.

## Best Possible Solution

Align the watcher with skill discovery: either constrain Chokidar traversal to the immediate-plus-one-grouped-level skill layout or explicitly watch only the relevant `SKILL.md` discovery surface, while preserving default heavy-directory ignores and adding focused regression coverage for large non-skill subtrees.

## Reproduction Assessment

Unclear. The issue gives a concrete macOS startup symptom and large-workspace shape, and the static code path is focused, but this review did not live-reproduce the EMFILE failure with a 300k-file workspace.

## Solution Assessment

No. The linked PR's stated ignore-order fix is not enough by itself because that order is already in `v2026.4.29`; a better fix should bound watcher traversal to the skill discovery contract or add an equivalent narrow watch-target strategy.
