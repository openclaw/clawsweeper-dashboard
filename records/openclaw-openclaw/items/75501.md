## Summary

Keep open. Current main and v2026.5.2 still pass skill roots to Chokidar with filters rather than a bounded target set, and the linked open PR is an implementation candidate rather than a merged fix.

## What This Changes

The issue requests preventing the skills watcher from exhausting file descriptors on large macOS workspaces by ignoring or bounding non-skill directories.

## Best Possible Solution

Keep this issue open while maintainers revise, land, or replace #75518 with a watcher that mirrors bounded skill discovery or a documented ignore mechanism with regression coverage.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection gives a plausible current-main path when large trees are under a watched skills root or configured extraDir, because Chokidar recursively watches directory targets and the skills watcher still uses those targets.

## Solution Assessment

No, the current linked PR is not enough as-is because current main already applies the fixed ignore list before allowing directories. The narrower maintainable fix is to bound watcher targets/depth to the same skill layouts discovery supports, or add a documented ignore setting if maintainers want user control.
