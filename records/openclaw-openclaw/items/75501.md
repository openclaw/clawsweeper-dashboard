## Summary

Keep open. Current main still starts Chokidar on skill-root directories without a traversal depth or exact `SKILL.md` target list; the shipped v2026.4.29 watcher has the same relevant shape, and the linked open PR appears to address only an ignore-order premise that current source already satisfies.

## What This Changes

Prevent the Skills Watcher from exhausting file descriptors on large macOS workspaces by ignoring or bounding large non-skill directories.

## Best Possible Solution

Review or replace #75518 with a watcher fix that matches skill discovery bounds, such as watching only skill-definition surfaces or setting a safe traversal depth, with regression coverage and a changelog entry.

## Reproduction Assessment

Unclear. The report gives a concrete macOS v2026.4.29 symptom and large-workspace shape, and current source exposes a plausible recursive watcher path, but this review did not run a live high-file-count reproduction.

## Solution Assessment

No. The linked PR’s ignore-order fix is not the best solution because current main and v2026.4.29 already use that ordering; the maintainable fix should bound traversal to the skill discovery contract and cover large unlisted directories.
