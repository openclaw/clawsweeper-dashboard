## Summary

Keep open. The issue is still paired with open #75518, and current `main` still hands root skill directories and `skills.load.extraDirs` to Chokidar as recursive watch targets; the default ignore list covers common `venv`/`node_modules` paths, but there is not yet a bounded watcher or documented custom ignore mechanism for large non-ignored trees.

## What This Changes

The issue reports a macOS EMFILE startup regression from the skills watcher traversing very large workspace or skill trees and requests default excludes or user-configurable ignores.

## Best Possible Solution

Land or replace #75518 with a regression-tested watcher that mirrors bounded skill discovery or supports documented ignores, then close this issue after the fix ships.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection gives a plausible current-main path when a very large non-ignored directory exists under a watched skills root or `skills.load.extraDirs`, because Chokidar receives those roots as recursive directory targets.

## Solution Assessment

No, closing this now is not the best outcome. The narrow maintainable fix is to make the watcher follow the same bounded skill layouts discovery supports, or add a documented ignore mechanism if maintainers want user-controlled exclusions.
