## Summary

Keep open. Current main and the reported v2026.4.29 release still pass skill-root directories to Chokidar without a traversal depth or concrete SKILL.md target list, and the open #75518 closing PR is the current implementation candidate but does not fully address arbitrary large non-ignored subtrees.

## What This Changes

Prevent the skills watcher from exhausting macOS file descriptors on large skill/workspace trees by ignoring heavy directories or adding configurable excludes.

## Best Possible Solution

Align watcher traversal with skill discovery by bounding Chokidar depth or watching concrete SKILL.md surfaces, keep common heavy-path ignores, and add focused regression coverage for large non-skill subtrees.

## Reproduction Assessment

Unclear. The report gives a concrete macOS 15.4 v2026.4.29 symptom and large tree shape, and current source plus Chokidar's contract show a plausible recursive watcher path, but this review did not run a live high-file-count macOS reproduction.

## Solution Assessment

No. Expanding or reordering common ignore patterns is only a partial fix because current main and v2026.4.29 already check those names before directory traversal; the maintainable fix is to bound traversal to the documented skill discovery contract.
