## Summary

Keep open. Current main and the reported v2026.4.29 release still create a recursive Chokidar watcher over skill-root directories without a depth limit or exact SKILL.md target list, and open PR #75518 references this issue with closing syntax, so the issue should remain open until a fix lands or that PR is closed/replaced.

## What This Changes

Prevent the skills watcher from exhausting file descriptors in large workspaces by ignoring or avoiding large non-skill directories while watching skill roots.

## Best Possible Solution

Review #75518 as the current implementation candidate, but prefer a bounded watcher that matches skill discovery's immediate/one-level grouped SKILL.md contract instead of relying only on more ignore names.

## Reproduction Assessment

Unclear. The report gives a concrete macOS v2026.4.29 symptom, EMFILE/EBADF logs, and a 308k-file workspace shape, and current source shows a plausible recursive watcher path, but this review did not run a live high-file-count reproduction.

## Solution Assessment

No. Adding or reordering common ignore patterns alone is not the narrowest maintainable fix because current main already ignores several named heavy paths; the safer fix is to avoid recursively watching non-skill subtrees in the first place.
