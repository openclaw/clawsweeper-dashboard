## Summary

Keep open. Current main and v2026.5.2 have common ignore-order coverage, but the skills watcher still watches whole skill roots recursively, allows non-default large directories to be traversed, has no documented custom ignore list, and the linked #75518 PR is still open and only targets behavior already present on main.

## What This Changes

The issue requests preventing Skills Watcher EMFILE crashes on large macOS workspaces by ignoring heavy directories or adding configurable watcher excludes.

## Best Possible Solution

Constrain skills watcher traversal to the same bounded `SKILL.md` discovery shape, or add a documented ignore mechanism, then resolve #75518 as revised, replaced, or obsolete.

## Reproduction Assessment

Unclear for a high-confidence live reproduction. The report gives a concrete macOS 2026.4.29 symptom and large-workspace shape, and source shows a plausible current path if those directories are under watched skill roots, but no live high-file-count reproduction was run.

## Solution Assessment

No. The linked PR's ignore-order fix is already present on current main; the narrower maintainable direction is bounding watcher traversal or adding documented excludes for non-default large directories.
