## Summary

Keep this open. The report is current against v2026.4.29, an open unmerged PR already references it as the closing fix, and current main still leaves the skills watcher with broad Chokidar traversal over skill roots while the loader itself only discovers immediate and one grouped nested level.

## What This Changes

The issue reports macOS startup EMFILE and cascading spawn EBADF failures from the Skills Watcher traversing very large workspace or skill-root directories, and requests default ignores or configurable exclude paths.

## Best Possible Solution

Revise or replace #75518 with a watcher fix that prevents Chokidar from descending into large non-skill subtrees, ideally by matching the loader’s immediate-plus-one-grouped-level discovery boundary and expanding or configuring heavyweight default ignores, then add focused regression coverage for directory-level ignores and traversal bounds.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No live reproduction was performed, but the static path is clear: start a session with the skills watcher enabled and a configured skill root containing a very large non-ignored directory tree; current main does not bound Chokidar traversal to the loader discovery depth.

## Solution Assessment

Is this the best way to solve the issue? No for the currently linked PR as described, because current main and v2026.4.29 already have the ignore-order behavior it claims to add. The safer fix is to bound watcher descent and cover large non-skill subtrees, not only reorder existing ignore checks.
