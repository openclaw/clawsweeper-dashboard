## Summary

Keep open. Current main still passes skills roots to Chokidar as recursive directory targets, and the linked open PR is an implementation candidate rather than a merged fix; source inspection supports the reported EMFILE path for large watched skill roots even though I did not run a live 300k-file macOS reproduction.

## What This Changes

The issue asks for the skills watcher to avoid exhausting file descriptors on large workspaces by ignoring or bounding traversal of common large non-skill directories.

## Best Possible Solution

Keep this issue open while maintainers land, revise, or replace the linked PR with a watcher that mirrors bounded skill discovery or a documented ignore mechanism with regression coverage.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection gives a plausible current-main path: configured skill roots are watched recursively, so a large skills root or extraDir can still drive Chokidar into enough directories to hit EMFILE.

## Solution Assessment

No, current main is not the best end state because it still watches directory roots rather than bounded skill file targets. A narrow maintainable fix would bound watcher targets or depth to supported skill layouts, with user ignore configuration only if maintainers want that policy surface.
