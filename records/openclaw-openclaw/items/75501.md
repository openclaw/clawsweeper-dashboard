## Summary

Keep open. Current main still starts Chokidar on skill-root directories without a traversal depth or exact SKILL.md target list, v2026.4.29 has the same relevant watcher shape, and #75518 is an open unmerged implementation candidate linked from the discussion.

## What This Changes

The issue reports a macOS EMFILE/spawn EBADF regression where the Skills Watcher watches very large skill/workspace trees in v2026.4.29.

## Best Possible Solution

Review, land, or replace #75518 with a bounded watcher that matches skill discovery, ideally by limiting traversal or watching only skill-definition surfaces with focused regression coverage.

## Reproduction Assessment

Unclear. The report gives a concrete macOS v2026.4.29 symptom and large-workspace shape, and current source shows a plausible recursive watcher path, but this review did not run a live high-file-count EMFILE reproduction.

## Solution Assessment

No. The linked PR’s described ignore-order fix handles common names, but the maintainable fix should bound traversal to the skill discovery contract so unknown large subtrees cannot exhaust file descriptors.
