## Summary

Keep open. Current main and v2026.5.2 include default ignores for common large directories, but the issue is still paired with open, unmerged #75518 and source inspection still shows Chokidar watching skills roots recursively rather than a bounded `SKILL.md` target set.

## What This Changes

The issue reports that the skills watcher can exhaust file descriptors on macOS when a large workspace is watched after upgrading to OpenClaw 2026.4.29.

## Best Possible Solution

Keep this issue open until maintainers review, land, close, or replace #75518 with a watcher that matches bounded skill discovery or a documented ignore mechanism with regression coverage.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection gives a plausible current-main path when large non-ignored trees live under a watched skills root or configured `extraDirs`, because Chokidar recursively watches directory targets.

## Solution Assessment

No, the currently linked PR is not clearly the complete fix as described because current main already has the fixed default-ignore ordering. The narrower maintainable direction is to bound watcher targets/depth to supported skill layouts, or add a documented ignore setting if maintainers choose that policy.
