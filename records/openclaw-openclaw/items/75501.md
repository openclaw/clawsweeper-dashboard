## Summary

Keep this open. The report is current and plausible: current main and the reported v2026.4.29 release already include the linked PR's ignore-order shape, but the skills watcher still hands recursive skill-root directory targets to Chokidar without a traversal bound, so large non-skill subtrees remain a credible FD-exhaustion path.

## What This Changes

The issue asks OpenClaw to prevent the Skills Watcher from exhausting file descriptors on large macOS workspaces by ignoring heavy directories or supporting configurable watcher excludes.

## Best Possible Solution

Align the watcher with skill discovery by constraining Chokidar traversal to root `SKILL.md`, immediate skill folders, and one grouped level, or by watching only that equivalent `SKILL.md` surface, while preserving default heavy-directory pruning and adding focused regression coverage for large non-skill subtrees.

## Reproduction Assessment

Unclear. The issue gives a concrete macOS 2026.4.29 symptom and large-workspace shape, and current source shows a focused recursive watcher path, but this review did not run a live 300k-file EMFILE reproduction and the exact watched root/config is missing.

## Solution Assessment

No. The linked PR's stated ignore-order fix is not the best solution because that order already exists in the reporter's release; the safer fix is to bound watcher traversal or targets to the documented skill discovery contract.
