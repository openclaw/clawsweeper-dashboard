## Summary

Keep open. Current main and the reported v2026.4.29 release still start Chokidar on whole skill-root directories without a depth limit or exact `SKILL.md` target list, and open PR #75518 already references this issue with closing syntax but is not merged.

## What This Changes

The issue reports that the Skills watcher can exhaust file descriptors on macOS by recursively watching a very large workspace with dependency/cache directories.

## Best Possible Solution

Review #75518 as the current implementation candidate, but prefer a fix that bounds watcher traversal to the documented skill discovery surface or watches only equivalent `SKILL.md` targets, with regression coverage for large non-skill subtrees.

## Reproduction Assessment

Unclear. The report gives a concrete macOS v2026.4.29 symptom and large-workspace shape, and current source plus Chokidar docs show a plausible recursive watcher path, but this review did not run a live high-file-count EMFILE reproduction.

## Solution Assessment

No. The current linked PR only reorders two post-ignore checks and carries unrelated auto-reply changes; the maintainable fix should bound traversal or target only the skill files OpenClaw actually discovers.
