The PR adds timeout-specific external-run failure copy, regression tests, and a changelog entry for Claude CLI no-output and overall timeout errors when verbose failures are off.

## Best Possible Solution

Keep the actionable timeout copy, but make it backend-aware or generic when Claude cannot be proven, with regression coverage for both Claude and non-Claude CLI timeout summaries.

No. The proposed approach is close, but matching only the generic CLI timeout text and hardcoding Claude is not the narrowest maintainable fix for a system with Codex, Gemini, and custom CLI backends.
