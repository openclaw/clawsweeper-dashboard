## Best Possible Solution

Make a maintainer-approved Active Memory timeout policy change in the plugin surface: keep safe defaults, allow a higher advanced budget if accepted, and update the manifest, docs, and focused cap tests together.

## Solution Assessment

Unclear. The request belongs in the Active Memory plugin config surface, but maintainers should decide whether the best policy is a higher fixed ceiling or an advanced opt-in because this budget blocks the reply path.
