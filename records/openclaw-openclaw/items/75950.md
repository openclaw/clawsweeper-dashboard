## Summary

Keep open. Current main still hard-blocks Google Meet Chrome realtime audio on non-macOS hosts, the docs and config remain BlackHole/macOS-centered, and the maintainer comment confirms this is a real plugin-owned gap rather than cleanup-close material.

## What This Changes

The issue requests Linux support or clearer guidance for Google Meet local Chrome realtime joins that currently fail on the macOS-only BlackHole backend.

## Best Possible Solution

Keep this issue open until Google Meet has plugin-owned platform-aware behavior: either native Linux Chrome realtime audio with setup/docs/tests or, short term, non-macOS agent-tool hiding plus explicit Linux alternatives.
