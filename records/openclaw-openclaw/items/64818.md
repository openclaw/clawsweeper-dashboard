## Summary

Keep open. Current main still has the reported failure shape: the A2UI hash is tracked while ignored, the build path can rewrite it, and the dev update preflight walkback still reuses one worktree with plain checkout attempts after candidate failures.

## What This Changes

The issue asks for `openclaw update` preflight walkback to survive a generated tracked A2UI hash file becoming dirty between fallback checkouts.

## Best Possible Solution

Harden the disposable dev preflight worktree so each candidate starts from a clean state before checkout, then add a focused regression test; re-untracking the A2UI hash can be separate cleanup but should not be the only updater fix.

## Reproduction Assessment

Yes. Dirtying a tracked `src/canvas-host/a2ui/.bundle.hash` and then checking out another commit that tracks a different blob reproduces the same Git refusal, and current main still performs that plain checkout sequence inside the reused preflight worktree.

## Solution Assessment

Yes. Resetting or cleaning the disposable preflight worktree before each candidate checkout is the narrow maintainable fix because it covers this A2UI case and the broader class of generated tracked-file dirtiness.

## Repair Work Prompt

Fix the updater walkback bug described in https://github.com/openclaw/openclaw/issues/64818. Current main can leave the disposable dev preflight worktree dirty when a candidate build or lint fails after `scripts/bundle-a2ui.mjs` rewrites the tracked `src/canvas-host/a2ui/.bundle.hash`; the next `git checkout --detach <sha>` can then fail with local changes and the update returns `preflight-no-good-commit`. Implement the fix in `src/infra/update-runner.ts` by ensuring each preflight candidate starts from a clean disposable worktree before checkout, using a narrow reset/clean or equivalent safe cleanup inside the preflight worktree only. Add a regression test in `src/infra/update-runner.test.ts` that simulates a failed candidate leaving tracked generated output dirty and verifies the next candidate can still be checked out and selected. Do not rely solely on untracking `.bundle.hash`, do not change unrelated A2UI bundle reproducibility behavior from #67547, and do not touch dependency patches or generated baselines except as required by the focused fix. Add a user-facing `CHANGELOG.md` fix entry unless maintainers classify the change as internal only.
