## Summary

Keep open. Current main still has the reported failure shape: the A2UI hash is both tracked and ignored, the build path can rewrite it, and the dev update preflight walkback still checks out fallback commits in one reused worktree without cleaning it first.

## What This Changes

Fix `openclaw update` so failed dev preflight candidates that dirty generated A2UI hash artifacts do not block checkout of fallback commits.

## Best Possible Solution

Harden the disposable dev preflight worktree so each candidate starts clean before checkout, then add focused regression coverage; re-untracking the A2UI hash can be separate cleanup but should not be the only guard.

## Reproduction Assessment

Yes. Current source gives a high-confidence path: a tracked ignored hash can be rewritten by the build, and the next plain fallback checkout in the reused preflight worktree can fail with Git's local-changes refusal.

## Solution Assessment

Yes. Resetting or cleaning the disposable preflight worktree before each candidate checkout is the narrow maintainable fix because it covers this A2UI case and the broader class of generated tracked-file dirtiness.
