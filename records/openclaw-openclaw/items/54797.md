## Summary

Keep this open. Current main still renders completed subagent sessions in the Control UI chat session selectors without an on-demand clear or dismiss affordance; the related #52803 is a broader orchestration umbrella, not a merged fix for this focused dropdown cleanup.

## What This Changes

Add a Control UI affordance to clear or dismiss completed subagent sessions from the chat session dropdown.

## Best Possible Solution

Add a focused desktop and mobile Control UI action beside the chat session selector that clears only completed subagent rows, derives eligibility from existing lifecycle fields, calls the existing session deletion RPC with explicit transcript handling, refreshes sessions.list, and covers active/subagent/main/channel/cron edge cases in UI/controller tests.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: add a sessions.list row with a key containing :subagent:, status: done, and endedAt, then resolveSessionOptionGroups will still add it to the desktop and mobile selectors with no clear/dismiss control available.

## Solution Assessment

Yes. The narrowest maintainable solution is to add a dropdown-adjacent Clear completed flow using the existing session row metadata and sessions.delete RPC; the separate Sessions tab bulk delete workflow does not solve on-demand cleanup from the chat selector.
