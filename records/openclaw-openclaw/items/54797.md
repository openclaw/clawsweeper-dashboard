## Summary

Keep open: current main still has no on-demand Clear completed or per-entry dismiss affordance for completed subagent rows in the Control UI chat session selector. The related orchestration, session-kind filter, and visual-grouping items are useful adjacent work, but they do not replace this focused cleanup action.

## What This Changes

Add a Control UI affordance to clear or dismiss completed subagent sessions from the chat session dropdown.

## Best Possible Solution

Add a focused desktop and mobile Control UI action near the chat session selector that clears only terminal completed subagent rows, refreshes sessions.list, and preserves active sessions plus broader retention/filtering behavior.

Yes, as a source-level feature-gap check: a listed subagent row with status "done" still flows through resolveSessionOptionGroups into desktop and mobile selectors, and no clear or dismiss control is rendered. I did not run a live browser scenario in this read-only review.

## Solution Assessment

Yes. A selector-adjacent Clear completed flow is the narrow maintainable solution; broader orchestration, kind filtering, and lifecycle retention changes should stay in their existing separate trackers.
