## Summary

Keep this open: current main still shows completed subagent rows in the chat session selectors without an on-demand clear or dismiss affordance, and the related orchestration umbrella remains open rather than superseding this focused UI cleanup.

## What This Changes

Add an on-demand Control UI affordance to clear or dismiss completed subagent sessions from the chat session selector.

## Best Possible Solution

Add a focused desktop and mobile Control UI action beside the chat session selector that clears only terminal completed subagent rows, refreshes sessions.list, and preserves active sessions, main/channel sessions, cron visibility behavior, and transcript history unless explicitly intended.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a sessions.list row for `agent:main:subagent:*` with `status: "done"` and `endedAt` still flows through resolveSessionOptionGroups into the desktop and mobile selectors, with no clear/dismiss control rendered.

## Solution Assessment

Yes. The narrow maintainable path is a selector-adjacent Clear completed flow using existing session row lifecycle fields and the existing sessions.delete RPC; the open #52803 orchestration umbrella is broader than needed for this cleanup.

## Repair Work Prompt

Implement this issue as a focused Control UI cleanup. Add a desktop and mobile chat-session-selector affordance, preferably Clear completed, that removes only terminal completed subagent session rows from the selector. Derive eligibility from existing session row metadata, not key shape alone: require a subagent session plus terminal status or historical subagent state/endedAt and no active subagent run. Do not clear main sessions, channel/group sessions, global/unknown rows, cron rows, or active subagents. Use the existing sessions.delete RPC with an explicit deleteTranscript decision; for a selector dismissal, prefer preserving transcripts unless maintainer guidance says to archive them. Refresh sessions.list after deletion, surface partial failures, and keep the existing Sessions tab bulk-delete behavior intact. Add focused regression coverage for desktop selector rendering, mobile controls, eligibility edge cases, and the controller delete/refresh path. Update CHANGELOG.md because this is a user-facing Control UI change. Related broader context: #52803.
