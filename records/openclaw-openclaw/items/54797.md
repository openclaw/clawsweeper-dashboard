## Summary

Close: current main now keeps inactive or completed subagent rows out of the normal Control UI chat session picker, which addresses the reported dropdown clutter without requiring automatic session deletion.

## What This Changes

The issue requests an on-demand clear, dismiss, or separation affordance for completed subagent rows in the Control UI session dropdown.

## Best Possible Solution

Keep the current picker filtering for the narrow dropdown-clutter problem; broader optional kind filters can stay tracked in https://github.com/openclaw/openclaw/issues/73222.

## Reproduction Assessment

No. Against current main, source inspection and regression coverage show inactive or spawned subagent rows are skipped unless they are the selected session, so the reported dropdown clutter should no longer reproduce.

## Solution Assessment

Yes. Hiding inactive subagent rows from normal picker choices solves the noisy dropdown without deleting session records automatically; explicit deletion remains available through the Sessions management path and gateway API.
