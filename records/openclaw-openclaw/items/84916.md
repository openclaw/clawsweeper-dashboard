## Summary

Keep open: this PR has useful bug signal, but it is not merge-ready because it removes ACP timeout config emission globally while current main still has targeted timeout tolerance/suppression, tests, docs, and `/acp timeout` behavior that treat backend-advertised timeout config as part of the runtime contract.

## What This Changes

The branch removes `timeoutSeconds` from ACP runtime session config option emission and from ACP spawn session initialization while keeping dispatch-level run timeout handling.

## Best Possible Solution

Land a contract-aligned timeout fix: either keep current targeted best-effort/adapter-specific behavior, or deliberately switch to watchdog-only ACP timeouts across manager tests, `/acp timeout`, docs, and live proof.

## Reproduction Assessment

## Solution Assessment

No, not as-is. Removing all implicit timeout config emission is plausible but broader than the current targeted fixes, so the best solution needs an explicit timeout-contract decision plus aligned tests/docs/proof.
