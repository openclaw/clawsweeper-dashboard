## Summary

Keep open: the PR targets a real ACP timeout-config failure class, but it changes the documented ACP timeout contract broadly while leaving related manager tests, `/acp timeout`, docs, and after-fix live proof unresolved.

## What This Changes

The PR removes implicit ACP timeout config emission from runtime option pairs and stops `sessions_spawn` from persisting `runTimeoutSeconds` as ACP runtime options while keeping gateway dispatch timeouts.

## Best Possible Solution

Land a contract-aligned ACP timeout fix that treats spawn run timeouts as OpenClaw watchdog state, updates manager tests/docs and `/acp timeout` semantics, preserves an explicit backend-specific timeout path if maintainers want one, and includes redacted after-fix live ACP proof.

## Reproduction Assessment

Yes, source-reproducible with high confidence: current main can still translate ACP runtime `timeoutSeconds` into a `timeout` config option, and the PR body includes a redacted real failure for `Unknown config option: timeout`. I did not run a live ACP adapter in this read-only review.

## Solution Assessment

No, not as-is: the implementation direction is plausible for `sessions_spawn` run timeouts, but the patch needs the remaining timeout-control paths, tests, docs, and proof aligned before it is the maintainable fix.
