## Summary

Keep open: this protected security/maintainer PR is still not merge-ready because the current head has source-reproducible blockers in auth refresh locking, scoped doctor migrations, and security-sensitive legacy cleanup.

## What This Changes

The PR moves OpenClaw runtime state from JSON/JSONL/sidecar files into global and per-agent SQLite databases, with doctor migrations, app/plugin call-site updates, docs, and checks around that storage boundary.

## Best Possible Solution

Land the SQLite storage refactor only after fixing the remaining scoped-state and credential regressions, adding representative redacted runtime/migration proof, and getting explicit storage/security maintainer review.

## Reproduction Assessment

Yes. The blocking PR defects are source-reproducible on the current head by reading the auth refresh, doctor contribution, and legacy migration paths; no runtime tests were run in this read-only review.

## Solution Assessment

No. The SQLite storage direction may be intended, but this patch is not yet the best way to solve it because it can mis-scope migrations, drop legacy metadata, and lose refreshed OAuth state under concurrency.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Serialize refresh writes by auth store:** `src/agents/auth-profiles/oauth-manager.ts:333-338`
  - body: This now locks only by `provider+profileId`, but the code still does a read-modify-write of the entire auth profile store. Concurrent refreshes for different profiles in one store can overwrite each other's refreshed token snapshots.
  - confidence: 0.9
- **[P1] Pass scoped env into session transcript migration:** `src/flows/doctor-health-contributions.ts:347`
  - body: `runSessionTranscriptsHealth` calls the legacy transcript migration without `ctx.env`. Doctor runs targeting a non-default state dir can migrate/delete the wrong transcript files while leaving the requested scope unmigrated.
  - confidence: 0.86
- **[P1] Thread scoped env through legacy cron migration:** `src/flows/doctor-health-contributions.ts:354-358`
  - body: The cron doctor path still calls `maybeRepairLegacyCronStore` without the caller env, so scoped repairs can read or write through process defaults and put migrated cron state in the wrong database.
  - confidence: 0.85
- **[P1] Restore config-audit scrub in doctor:** `src/flows/doctor-health-contributions.ts:727-730`
  - body: The doctor contribution list no longer invokes config-audit scrubbing. That stops `doctor --fix` from warning/fixing historical `config-audit.jsonl` entries that may contain pre-redactor plaintext argv secrets.
  - confidence: 0.88
- **[P1] Pass scoped env into auth-profile state imports:** `src/commands/doctor/legacy/runtime-state.ts:453`
  - body: `maybeRepairLegacyRuntimeStateFiles` discovers legacy auth state under the caller env, then imports each file through a helper called without that env. A scoped doctor run can delete targeted legacy state after writing rows to the default SQLite state.
  - confidence: 0.88
- **[P2] Resolve default agent model path with scoped env:** `src/commands/doctor/legacy/models-config.ts:14`
  - body: `resolveCandidateModelConfigPaths` uses `resolveDefaultAgentDir(params.cfg ?? {})` without the caller env, so scoped doctor runs can import and delete `models.json` from the user's real default agent directory.
  - confidence: 0.86
- **[P2] Pass caller env when resolving legacy approvals:** `src/commands/doctor/legacy/plugin-conversation-binding.ts:12-16`
  - body: `resolveLegacyApprovalsPath()` hard-codes `process.env`, and the import helper has no env parameter. Scoped doctor runs can skip or import/delete plugin binding approvals from the wrong profile.
  - confidence: 0.9
- **[P2] Route legacy media imports to the scoped database:** `src/commands/doctor/legacy/media.ts:133-138`
  - body: `importLegacyMediaFilesToSqlite` accepts an env for the legacy directory, but writes each blob through `saveMediaBufferWithId` without that scope. Scoped migrations can delete target media after storing blobs in another DB.
  - confidence: 0.86
- **[P1] Keep managed-image records until import succeeds:** `src/commands/doctor/legacy/managed-image-attachments.ts:101-105`
  - body: This loop removes each legacy record file even when JSON parsing fails or `importLegacyManagedImageRecord` returns false. That can drop the only metadata needed to retry migration and orphan managed outgoing attachments.
  - confidence: 0.9
