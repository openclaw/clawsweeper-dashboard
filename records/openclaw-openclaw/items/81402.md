## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Do not replace current device identity from stale legacy JSON:** `src/commands/doctor/legacy/device-identity.ts:156`
  - body: `importLegacyDeviceIdentityFileToSqlite` writes the legacy keypair into the single default SQLite identity row before deleting the source file. If SQLite already contains the current identity and an old `identity/device.json` remains, `doctor --fix` can roll back the node/device identity and break existing pairings or signatures without leaving an easy recovery source.
  - confidence: 0.89
- **[P1] Reject empty plugin approval imports before writing:** `src/commands/doctor/legacy/plugin-conversation-binding.ts:45`
  - body: A parseable legacy file without valid approvals normalizes to an empty approval list, and the SQLite writer deletes all `plugin_binding_approvals` rows for that snapshot. A stale or malformed file can therefore revoke current persistent plugin approvals during doctor migration.
  - confidence: 0.9
- **[P1] Reject malformed Voice Wake imports before resetting triggers:** `src/commands/doctor/legacy/voicewake.ts:41`
  - body: `normalizeVoiceWakeConfigSnapshot({})` falls back to default triggers, then `writeVoiceWakeConfigSnapshot` deletes and rewrites the existing trigger rows. A stale or shape-drifted legacy file can reset a user's current Voice Wake configuration during `doctor --fix`.
  - confidence: 0.87
- **[P2] Preserve config health rows on empty legacy imports:** `src/commands/doctor/legacy/runtime-state.ts:166`
  - body: This path treats any object as a successful config-health import, but `writeConfigHealthStateToSqlite` normalizes objects without `entries` to zero rows and deletes the existing table contents. That can wipe current config recovery fingerprints when an old `logs/config-health.json` contains `{}` or a drifted shape.
  - confidence: 0.84
- **[P1] Skip stale plugin index imports when SQLite already exists:** `src/commands/doctor/legacy/installed-plugin-index.ts:46`
  - body: The legacy installed-plugin index importer overwrites the persisted SQLite index unconditionally and then removes the source file. In a delayed-upgrade state, an old `installed-plugin-index.json` can roll back current plugin install records and metadata used by startup/location lookups.
  - confidence: 0.85
- **[P2] Keep expiry saves inside the commitments writer lock:** `src/commitments/store.ts:457`
  - body: `loadCommitmentStoreWithExpiredMarked` can save an expired snapshot without the exclusive writer gate used by mutation paths. A listing/heartbeat read that expires stale rows can race a concurrent upsert/status update and rewrite the commitments table from an older snapshot.
  - confidence: 0.78
