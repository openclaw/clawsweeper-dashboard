Full review comments:

- **[P1] Avoid overwriting newer exec approvals from legacy files:** `src/commands/doctor/legacy/exec-approvals.ts:34-35`
  - body: `importLegacyExecApprovalsFileToSqlite` writes the legacy JSON into the single SQLite config row and then removes the source file. If a user already has newer SQLite approvals plus a stale `exec-approvals.json`, `doctor --fix` rolls the approval policy back and deletes the rollback artifact.
  - confidence: 0.93
- **[P1] Merge legacy device-auth tokens before deleting the source:** `src/commands/doctor/legacy/device-auth-store.ts:39-41`
  - body: This import treats the legacy device-auth snapshot as authoritative, but the SQLite writer deletes token rows outside that snapshot. A stale `identity/device-auth.json` can therefore roll back newer device tokens and then remove the source file.
  - confidence: 0.91
- **[P1] Reject malformed web-push snapshots before import:** `src/commands/doctor/legacy/push-web.ts:47-50`
  - body: The import accepts any parsed object, including `{}` or arrays, and passes it to `writeWebPushRegistrationStateSnapshot`; that persistence path deletes all subscription rows when the snapshot has no entries. A stale or malformed legacy file can wipe newer web-push registrations before being removed.
  - confidence: 0.9
- **[P1] Validate node-host config before saving SQLite state:** `src/commands/doctor/legacy/node-host-config.ts:42-43`
  - body: Malformed-but-parseable payloads are coerced to `{}` and saved as a successful import, which normalizes to a fresh node id and upserts the current SQLite row. That can overwrite an existing node-host config during `doctor --fix` and then remove the legacy file.
  - confidence: 0.9
- **[P1] Merge pairing snapshots instead of replacing rows:** `src/commands/doctor/legacy/pairing-files.ts:52-56`
  - body: `writePairingStateRecord` uses replacement semantics, so importing a stale or reduced `pending.json` or `paired.json` can delete newer device/node pairing rows not present in the legacy file. Migration should merge or skip older rows before deleting the source files.
  - confidence: 0.89
