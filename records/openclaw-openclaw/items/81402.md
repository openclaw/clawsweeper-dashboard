## Summary

Keep open: this is an active protected `security`/`maintainer` SQLite storage refactor, and source review of PR head still shows blocking auth, doctor migration, secret-scrub, and Node 22 compatibility regressions.

## What This Changes

The PR moves active runtime state from JSON/JSONL/sidecar files to SQLite global/per-agent databases and updates doctor migrations, app/plugin state, docs, CI, and package metadata around that boundary.

## Best Possible Solution

Land only after fixing scoped-state and auth locking regressions, restoring config-audit and Node 22 support, and validating real gateway/runtime plus legacy migration paths under maintainer review.

Yes. The blocking PR defects are source-reproducible by reading PR head against current main in the auth refresh, doctor migration, config-audit, media/model migration, and CI paths; no runtime tests were run because this was a read-only review.

## Solution Assessment

No. The SQLite storage direction may be intended, but this patch is not yet the narrow maintainable solution because it regresses scoped state writes, auth-store serialization, secret-scrub coverage, and Node 22 compatibility proof.

Overall confidence: 0.88

Full review comments:

- **[P1] Serialize refresh writes by auth store:** `src/agents/auth-profiles/oauth-manager.ts:328-335`
  - body: Current main takes the auth-store lock before reading and saving the whole profile store, but this patch only locks by provider/profile. Concurrent refreshes for different profiles in the same store can save stale snapshots over each other and drop a refreshed token.
  - confidence: 0.9
- **[P1] Pass scoped env into auth-profile state imports:** `src/commands/doctor/legacy/runtime-state.ts:453`
  - body: `maybeRepairLegacyRuntimeStateFiles` discovers legacy auth state under the caller env, then imports each file through a helper called without that env. A scoped doctor run can delete the targeted legacy file after writing its rows to the default SQLite state.
  - confidence: 0.88
- **[P1] Restore config-audit scrub in doctor:** `src/flows/doctor-health-contributions.ts:717-727`
  - body: The doctor contribution list no longer invokes the config-audit scrub path, leaving `maybeScrubConfigAuditLog` orphaned. That stops `doctor --fix` from warning/fixing historical `config-audit.jsonl` entries that can contain pre-redactor plaintext argv secrets.
  - confidence: 0.88
- **[P1] Pass scoped env into session transcript migration:** `src/flows/doctor-health-contributions.ts:344-347`
  - body: This doctor contribution calls the new legacy transcript migration without `ctx.env`; that helper resolves state from process defaults and imports/deletes legacy transcripts. Scoped repairs can migrate the wrong state directory while leaving the requested target unmigrated.
  - confidence: 0.86
- **[P1] Thread scoped env through legacy cron migration:** `src/flows/doctor-health-contributions.ts:350-358`
  - body: The cron doctor path still calls the SQLite migration without the caller env, so store keys and legacy paths fall back to process-wide state. Scoped repairs can remove targeted cron files while persisting rows in another database.
  - confidence: 0.85
- **[P2] Resolve default agent model path with scoped env:** `src/commands/doctor/legacy/models-config.ts:8-15`
  - body: `resolveCandidateModelConfigPaths` uses `resolveDefaultAgentDir(params.cfg ?? {})` without the caller env. A scoped doctor run can import and delete `models.json` from the user's real default agent directory instead of the requested state scope.
  - confidence: 0.86
- **[P2] Pass caller env when resolving legacy approvals:** `src/commands/doctor/legacy/plugin-conversation-binding.ts:12-16`
  - body: `resolveLegacyApprovalsPath()` hard-codes `process.env`, and the import function has no env parameter. Doctor runs targeting a scoped state dir can skip or import/delete plugin binding approvals from the wrong profile.
  - confidence: 0.9
- **[P2] Route legacy media imports to the scoped database:** `src/commands/doctor/legacy/media.ts:133-138`
  - body: `importLegacyMediaFilesToSqlite` resolves the legacy directory from the caller env, but writes every blob through `saveMediaBufferWithId` without passing that scope. Scoped migrations can delete media from the target state after storing blobs in the default database.
