- **[P1] Reject empty APNS snapshots before replacing registrations:** `src/commands/doctor/legacy/push-apns.ts:40-41`

  - confidence: 0.91
- **[P1] Do not clear plugin approvals from malformed legacy input:** `src/commands/doctor/legacy/plugin-conversation-binding.ts:42-45`
  - body: A malformed-but-parseable `plugin-binding-approvals.json` normalizes to `{ approvals: [] }`, then `writePluginBindingApprovalsSnapshot` deletes all persisted approvals and the importer unlinks the source. That can revoke current plugin/channel approvals during upgrade; treat invalid or empty legacy snapshots as a failed import unless maintainers explicitly choose a destructive reset.
  - confidence: 0.92
- **[P1] Merge bootstrap tokens instead of replacing current rows:** `src/commands/doctor/legacy/device-bootstrap.ts:35-41`
  - body: The bootstrap importer passes the legacy snapshot to `writePairingStateRecord`, whose `devices/bootstrap` branch deletes SQLite token rows not present in the incoming file. A stale `devices/bootstrap.json` can roll back or wipe newer bootstrap/onboarding tokens, so the migration should merge or skip stale rows rather than treat the file as authoritative.
  - confidence: 0.9
- **[P1] Validate Voice Wake routing before replacing routes:** `src/commands/doctor/legacy/voicewake-routing.ts:40-42`
  - body: This import normalizes any object-shaped legacy routing payload and immediately writes it, while the writer deletes existing `voicewake_routing_routes` before inserting normalized routes.
