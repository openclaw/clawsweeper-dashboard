Full review comments:

- **[P1] Reject empty APNS imports before clearing registrations:** `src/commands/doctor/legacy/push-apns.ts:40-41`
  - body: `normalizeApnsRegistrationStateSnapshot({})` returns an empty snapshot, and `writeApnsRegistrationStateSnapshot` deletes all current APNS rows when the snapshot has no registrations. A stale or shape-drifted legacy file can therefore wipe existing SQLite APNS registrations during doctor repair before the source file is removed.
  - confidence: 0.91
- **[P1] Skip stale identity imports when SQLite already has one:** `src/commands/doctor/legacy/device-identity.ts:156-158`
  - body: This import writes any valid legacy identity directly into the default SQLite identity row. If an installation already has a newer SQLite device identity but the old `identity/device.json` remains, doctor can roll back the identity and break pairings or signatures without leaving the source artifact for recovery.
  - confidence: 0.9
- **[P1] Merge plugin approvals instead of replacing current rows:** `src/commands/doctor/legacy/plugin-conversation-binding.ts:45-47`
  - body: The legacy approval importer passes the normalized snapshot to a writer that deletes all approval rows for an empty snapshot and removes rows missing from a nonempty snapshot. A stale or malformed legacy file can revoke current persistent plugin binding approvals during migration.
  - confidence: 0.88
- **[P1] Reject empty Voice Wake imports before resetting triggers:** `src/commands/doctor/legacy/voicewake.ts:40-42`
  - body: Malformed or empty `voicewake.json` normalizes to default triggers, and the writer deletes existing `voicewake_triggers` rows before inserting the normalized set. Running doctor after SQLite already has current Voice Wake settings can silently reset user routing state to defaults.
  - confidence: 0.86
