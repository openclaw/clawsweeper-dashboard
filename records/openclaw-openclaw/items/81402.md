  - confidence: 0.9
- **[P1] Merge plugin-binding approvals instead of replacing rows:** `src/commands/doctor/legacy/plugin-conversation-binding.ts:45`
  - body: Importing a legacy approval file calls the snapshot writer directly. If SQLite already has newer approvals, a stale legacy snapshot can replace the live set before the legacy file is removed, rolling back plugin binding approvals.
  - confidence: 0.86
- **[P1] Avoid replacing live transcript events from legacy JSONL:** `src/commands/doctor/legacy/session-transcript-health.ts:233-237`
  - body: The legacy transcript migration uses `replaceSqliteSessionTranscriptEvents`, whose implementation deletes existing events for the session. If a user has newer SQLite transcript events and an old JSONL file still exists, doctor can roll the session history back and then remove the source.
  - confidence: 0.9

  - confidence: 0.88
