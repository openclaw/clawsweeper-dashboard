Full review comments:

- **[P2] Honor env-requested Node versions:** `scripts/install-cli.sh:56`
  - body: `NODE_VERSION` still reads `OPENCLAW_NODE_VERSION`, but the PR removed the initialization that sets `NODE_VERSION_REQUESTED=1` for that env var. `required_node_version()` now falls back to `22.19.0`, so `OPENCLAW_NODE_VERSION=24.x` is not enforced in the system/Alpine linking paths unless the user passed `--node-version`.
  - confidence: 0.91
- **[P2] Export the sharp libvips override:** `scripts/install-cli.sh:59`
  - body: This default is only a shell-local variable, while the later npm installs are launched via `env ... npm install`. Unless the caller already exported `SHARP_IGNORE_GLOBAL_LIBVIPS`, npm and sharp do not inherit the intended default and can still pick a system libvips on affected hosts.
  - confidence: 0.88
- **[P2] Decode the cron status field the gateway still sends:** `apps/macos/Sources/OpenClaw/GatewayConnection.swift:769`
  - body: The macOS client now requires `storeKey`, but current `cron.status` still returns `storePath`. Decoding can fail and `CronJobsStore.refreshJobs` catches that with `try?`, silently dropping scheduler enabled and next-wake status in the UI.
  - confidence: 0.9
- **[P2] Keep sessions.list compatible with the path field:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatSessions.swift:156`
  - body: The shared Swift response renamed `path` to `databasePath` without `CodingKeys` fallback, while the current sessions JSON response still emits `path`. Live gateway responses will decode with `databasePath == nil`, so WebChatSwiftUI loses the session store/database location.
  - confidence: 0.87
- **[P2] Normalize health sessions before exposing databasePath:** `ui/src/ui/types.ts:771`
  - body: The UI type now says `health.sessions.databasePath`, but `loadHealth()` returns the gateway result directly and the current health command still serializes `sessions.path`. Successful health responses therefore expose `undefined` for the renamed field instead of the live session database path.
  - confidence: 0.82
