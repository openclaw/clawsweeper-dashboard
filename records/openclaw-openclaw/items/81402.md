## Summary

Keep this PR open. It has protected `security` and `maintainer` labels, and the latest head still has source-reproducible P1/P2 correctness and security-sensitive migration blockers that need contributor fixes plus maintainer storage/security review.

## What This Changes

The PR moves active OpenClaw runtime state into global and per-agent SQLite databases and updates apps, plugins, docs, workflows, package metadata, and doctor migration paths around that storage boundary.

## Best Possible Solution

Keep this as an active maintainer-reviewed SQLite storage refactor, fix the scoped-env, serialization, config-audit, media import, and Node contract blockers, then require representative runtime and legacy-migration proof before merge.

## Reproduction Assessment

Yes. The blocking findings are source-reproducible on PR head 069ce49973ffaa377d800571dfeee97bc7b23328 from the changed auth, doctor, media, cron, transcript, and CI paths; no tests were run because this was a read-only sweep.

## Solution Assessment

No, not yet. The SQLite direction may be intended, but this patch needs scoped state writes, auth-store serialization, restored secret-scrub coverage, safer migration parsing/import behavior, and support-policy alignment before it is the maintainable solution.

Full review comments:

- **[P1] Serialize refresh writes by auth store:** `src/agents/auth-profiles/oauth-manager.ts:328-335`
  - body: The lock is keyed by provider/profile, but the critical section still reads and saves the whole auth-profile store. Concurrent refreshes for different profiles in the same store can take different locks and overwrite the earlier refreshed token with a stale snapshot.
  - confidence: 0.9
- **[P1] Pass scoped env into auth-profile state migration writes:** `src/commands/doctor/legacy/runtime-state.ts:453`
  - body: `maybeRepairLegacyRuntimeStateFiles` discovers legacy auth state under the caller env, then calls an importer that writes through default state DB options. A scoped doctor run can delete the requested legacy file after writing its state to the wrong SQLite database.
  - confidence: 0.88
- **[P1] Restore config-audit scrub in doctor:** `src/flows/doctor-health-contributions.ts:718-726`
  - body: The doctor contribution list no longer invokes the config-audit scrub path, so `doctor --fix` stops warning/fixing historical `config-audit.jsonl` entries that may contain pre-redactor plaintext argv secrets.
  - confidence: 0.86
- **[P1] Pass scoped env into session transcript migration:** `src/flows/doctor-health-contributions.ts:347`
  - body: `runSessionTranscriptsHealth` drops `ctx.env`, while `noteSessionTranscriptHealth` resolves legacy session dirs from `process.env`. Scoped doctor repairs can migrate/delete transcripts from the default state instead of the requested target.
  - confidence: 0.88
- **[P1] Thread scoped env through legacy cron migration:** `src/flows/doctor-health-contributions.ts:354-358`
  - body: The legacy cron doctor path is invoked without `ctx.env`, so cron SQLite writes and default legacy-path resolution fall back to process-wide state. Scoped repairs can remove targeted cron files while persisting rows in another database.
  - confidence: 0.86
- **[P1] Resolve default agent model path with scoped env:** `src/commands/doctor/legacy/models-config.ts:14`
  - body: `resolveCandidateModelConfigPaths` uses `resolveDefaultAgentDir(params.cfg ?? {})` without the caller env. A scoped doctor run can import and delete `models.json` from the caller's real default state instead of the requested state dir.
  - confidence: 0.86
- **[P2] Pass caller env when resolving legacy approvals:** `src/commands/doctor/legacy/plugin-conversation-binding.ts:12-16`
  - body: `resolveLegacyApprovalsPath()` hard-codes `process.env`, so doctor flows using an explicit scoped env can skip plugin-binding approvals or import/delete them from the wrong profile.
  - confidence: 0.9
- **[P2] Preserve session id extraction with corrupt transcript lines:** `src/commands/doctor/legacy/session-transcript-health.ts:308-310`
  - body: The sidecar migration parses the whole transcript to recover the session id, but `parseTranscriptEntries` returns empty when any line is malformed. A valid header followed by one bad line can import/delete the sidecar under the filename-derived wrong session key.
  - confidence: 0.84
- **[P2] Route legacy media imports to scoped state database:** `src/commands/doctor/legacy/media.ts:133-138`
  - body: `importLegacyMediaFilesToSqlite` accepts an explicit env, but each blob is saved through `saveMediaBufferWithId`, which opens the process-level state DB. Scoped migration can delete media from the target state while storing blobs elsewhere.
  - confidence: 0.86
- **[P2] Keep managed image blobs in the target state scope:** `src/commands/doctor/legacy/managed-image-attachments.ts:46-51`
  - body: Managed outgoing records are written to the caller-provided `stateDir`, but original blobs go through `saveMediaBufferWithId` and can land in the default state DB. The migrated record can then point at media missing from the target state.
  - confidence: 0.85
- **[P2] Keep the Node 22 compatibility contract covered:** `.github/instructions/copilot.instructions.md:7`
  - body: This changes guidance to Node 24+ and removes the Node 22 compatibility CI lane while package metadata and root policy still promise Node 22.16+ support. Either keep coverage or explicitly update the support contract across the repo.
  - confidence: 0.82
