Full review comments:

- **[P1] Do not replace current device identity from legacy files:** `src/commands/doctor/legacy/device-identity.ts:156`
  - body: `importLegacyDeviceIdentityFileToSqlite` writes any valid legacy `identity/device.json` into SQLite, and the writer upserts the singleton identity row. If a user already has a newer SQLite device identity, a delayed `doctor --fix` with a stale legacy file can roll back the keypair and break existing pairings or signatures; seed only when SQLite has no current identity, or preserve the legacy file for manual recovery.
  - confidence: 0.9
- **[P1] Reject empty approval imports before clearing rows:** `src/commands/doctor/legacy/plugin-conversation-binding.ts:45`
  - body: A malformed or shape-drifted `plugin-binding-approvals.json` normalizes to an empty approvals list, and `writePluginBindingApprovalsSnapshot` deletes all current approval rows for an empty snapshot. That lets stale legacy input revoke current plugin binding approvals during upgrade; reject empty malformed snapshots or merge without clobbering existing SQLite approvals.
  - confidence: 0.87
- **[P1] Preserve APNS registrations on empty legacy snapshots:** `src/commands/doctor/legacy/push-apns.ts:40`
  - body: `normalizeApnsRegistrationStateSnapshot({})` produces an empty snapshot, and this write path persists it before deleting the legacy file.   - confidence: 0.86
