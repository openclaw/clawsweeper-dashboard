## Summary

Keep this PR open. It is a recent, protected `security`/`maintainer` XL storage refactor, and the latest head still has source-reproducible correctness and security regressions in auth refresh locking, scoped doctor migrations, and config-audit scrubbing.

## What This Changes

The PR moves active runtime state into global and per-agent SQLite databases and updates doctor migrations, apps, plugins, docs, CI, and package metadata around that storage boundary.

## Best Possible Solution

Keep this as an active maintainer-reviewed SQLite storage refactor, fix the scoped-env, auth serialization, config-audit, media import, and Node contract blockers, then require representative runtime and migration proof before merge.

## Reproduction Assessment

Yes. The blocking PR defects are source-reproducible on PR head 2ae2698a7d100ed0c4a1bb2554a890e397ead504 from the changed auth, doctor, media, session, cron, and CI paths; no tests were run because this was a read-only review.

## Solution Assessment

No, not yet. The SQLite storage direction may be intended, but the current patch needs scoped state writes, auth-store serialization, restored secret-scrub coverage, safer migration behavior, and support-policy alignment before it is the maintainable solution.

  - confidence: 0.88
