  - body: For a stale or shape-drifted APNS legacy file, normalization returns an empty registration snapshot instead of rejecting it, and the writer deletes all `apns_registrations`.   - confidence: 0.9
- **[P1] Skip stale device identity when SQLite already has one:** `src/commands/doctor/legacy/device-identity.ts:156`
  - body: This imports any valid legacy identity by upserting the singleton SQLite identity row and then removing the file. If SQLite already contains the current device identity, a stale `identity/device.json` can roll it back and break pairings or signatures; only import when no current SQLite identity exists, or require an explicit migration decision.
  - confidence: 0.89
