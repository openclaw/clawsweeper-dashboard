## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Reject malformed approvals before clearing rows:** `src/commands/doctor/legacy/plugin-conversation-binding.ts:45`
  - body: This importer normalizes whatever JSON it reads and immediately writes the snapshot. `normalizePluginBindingApprovalsSnapshot({})` returns an empty approval list, and `writePluginBindingApprovalsSnapshot` deletes all `plugin_binding_approvals` rows for empty input, so a stale or malformed legacy file can revoke current plugin approvals before the source is unlinked. Reject malformed/empty legacy input or merge only missing approvals instead of treating the file as authoritative.
  - confidence: 0.89
- **[P1] Avoid wiping APNS registrations from empty imports:** `src/commands/doctor/legacy/push-apns.ts:40-41`
  - body: `normalizeApnsRegistrationStateSnapshot` accepts an object without a valid registrations map as an empty snapshot, and the write path deletes every `apns_registrations` row when the normalized set is empty. In a delayed upgrade where SQLite already has live APNS registrations, `doctor --fix` can erase them from a stale or shape-drifted JSON file; reject empty/malformed snapshots or merge without clobbering existing rows.
  - confidence: 0.87
- **[P1] Do not replace current device identity from stale JSON:** `src/commands/doctor/legacy/device-identity.ts:156-158`
  - body: After parsing a valid legacy identity, this path unconditionally writes it into SQLite and removes the source file. If the SQLite store already has the current device identity and the old JSON is stale, this can roll back the node/device identity and break pairings or signatures; skip the import when a current identity exists, or require an explicit merge rule that preserves newer SQLite state.
  - confidence: 0.86
- **[P1] Preserve shipped plugin SDK subpaths or deprecate them:** `package.json:170-176`
  - body: The root package export map removes existing plugin SDK subpaths while adding new SQLite/provider subpaths. Because these exports are the public plugin contract, removing `./plugin-sdk/provider-oauth-runtime`, `./plugin-sdk/inline-image-data-url-runtime`, `./plugin-sdk/qa-live-transport-scenarios`, and `./plugin-sdk/secret-provider-integration` needs either compatibility aliases with tests or explicit maintainer approval and a documented deprecation path.
  - confidence: 0.78
