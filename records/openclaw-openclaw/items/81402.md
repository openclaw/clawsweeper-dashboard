Full review comments:

- **[P1] Keep malformed auth state until import succeeds:** `src/commands/doctor/legacy/auth-profile-state.ts:30-33`
  - body: `loadJsonFile` can return `undefined` for malformed or unreadable JSON, but this path coerces that to an empty auth state, saves it, and deletes the legacy file. During `doctor --fix`, a bad `auth-state.json` can clear auth-profile runtime state and remove the only retry source.
  - confidence: 0.91
- **[P1] Keep unreadable pairing files for retry:** `src/commands/doctor/legacy/channel-pairing.ts:263-271`
  - body: `readLegacyPairingStore` returns `null` on read or parse failure, but the caller still removes the pairing file and counts it as processed. A malformed pairing snapshot can therefore drop pending requests without importing anything.
  - confidence: 0.9
- **[P1] Reject empty web-push snapshots before import:** `src/commands/doctor/legacy/push-web.ts:46-50`
  - body: Any parsed object, including `{}`, is written as the web-push registration snapshot before the source file is removed. A stale or shape-drifted legacy file can wipe newer SQLite registrations and prevent retry from the original file.
  - confidence: 0.87
- **[P1] Do not overwrite model catalogs from stale legacy JSON:** `src/commands/doctor/legacy/models-config.ts:58-62`
  - body: This importer blindly writes every legacy `models.json` payload into SQLite and then deletes the source. If SQLite already has newer model catalog state after upgrade, a delayed doctor repair can roll it back from stale JSON.
  - confidence: 0.84
- **[P2] Preserve cron run logs on read failure:** `src/commands/doctor/legacy/cron-run-log.ts:37-42`
  - body: A failed read is coerced to an empty string, then the `.jsonl` file is removed after importing zero entries. Transient read or permission failures can silently discard run-log history instead of leaving it for a later doctor pass.
  - confidence: 0.86
