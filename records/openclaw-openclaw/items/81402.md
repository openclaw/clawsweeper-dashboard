Keep this as an active maintainer-reviewed storage refactor, fix the blocking correctness/security findings, align the Node support policy, review dependency deltas, and require representative migration/runtime proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Serialize refresh writes by auth store:** `src/agents/auth-profiles/oauth-manager.ts:328-335`
  - body: The refresh critical section locks only by provider/profile while reading and saving the whole auth-profile store. Concurrent refreshes for different profiles in the same store can read the same old snapshot and the later save can overwrite the earlier refreshed token.
  - confidence: 0.9
- **[P1] Pass scoped env into auth-profile state migration writes:** `src/commands/doctor/legacy/runtime-state.ts:453`
  - body: `maybeRepairLegacyRuntimeStateFiles` discovers legacy auth state under the caller env, then calls an importer that writes through default state DB options. A doctor run targeting `OPENCLAW_STATE_DIR` can delete the requested legacy file after writing its state to the wrong SQLite database.
  - confidence: 0.88
- **[P1] Restore config-audit scrub in doctor:** `src/flows/doctor-health-contributions.ts:718-726`
  - body: The doctor contribution list no longer invokes the config-audit scrub path, so `doctor --fix` stops warning/fixing historical `config-audit.jsonl` entries that may contain pre-redactor plaintext argv secrets.
  - confidence: 0.86
- **[P2] Pass caller env when resolving legacy approvals:** `src/commands/doctor/legacy/plugin-conversation-binding.ts:12-16`
  - body: The legacy plugin-binding approval importer resolves its file path from global `process.env` instead of the doctor caller env. Runs with an explicit state dir can skip approvals or import them from the wrong profile.
  - confidence: 0.9
- **[P2] Preserve session-id extraction with corrupt transcript lines:** `src/commands/doctor/legacy/session-transcript-health.ts:308-310`
  - body: The sidecar migration parses the entire transcript to recover the session id, but `parseTranscriptEntries` returns an empty list when any line is malformed. A valid header followed by one corrupt line falls back to the filename and can import/delete the sidecar under the wrong session key.
  - confidence: 0.84
- **[P2] Keep the Node 22 compatibility contract covered:** `.github/instructions/copilot.instructions.md:7`
  - body: This changes repo guidance to Node 24+ and removes the Node 22 compatibility CI lane while package metadata and public docs still promise Node 22.16+ support. Either keep coverage or make the support-policy change explicit across docs, engines, and release validation.
  - confidence: 0.82
