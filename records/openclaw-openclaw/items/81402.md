## What This Changes

The branch moves runtime state from JSON/JSONL/sidecar files into global and per-agent SQLite databases across core stores, apps, plugins, migrations, docs, workflows, and package manifests.

## Best Possible Solution

Keep this as an active maintainer-reviewed storage refactor, fix the blocking correctness/security findings, align the Node support decision, complete dependency review, and require representative migration/runtime proof before merge.

## Reproduction Assessment

Yes, source-reproducible for the blocking PR findings. The OAuth race follows from provider/profile-only locking around whole-store saves, the doctor migration bug follows from global env path resolution, and the config-audit regression follows from removing the doctor contribution.

## Solution Assessment

No, not yet. The SQLite direction may be intended, but this branch needs store-level auth serialization, caller-env doctor imports, restored config-audit scrub registration, and a coherent Node support policy before it is the maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Serialize refresh writes by auth store:** `src/agents/auth-profiles/oauth-manager.ts:328-335`
  - body: The refresh critical section now locks only by provider/profile, but it still reads and saves the whole auth-profile store. Concurrent refreshes for different profiles in the same store can each read the same old snapshot and the later save can overwrite the earlier refreshed token.
  - confidence: 0.9

  - body: The legacy plugin-binding approval migration resolves paths from global `process.env` instead of the doctor caller env. Runs with an explicit `OPENCLAW_STATE_DIR` or `OPENCLAW_HOME` can skip approvals or import them from the wrong profile.
  - confidence: 0.9
- **[P1] Restore config-audit scrub in doctor:** `src/flows/doctor-health-contributions.ts:718-726`
  - body: The doctor contribution list no longer invokes the config-audit scrub path, while the scrub implementation remains otherwise orphaned. That removes the `doctor --fix` cleanup for historical plaintext credential entries in config audit logs.
