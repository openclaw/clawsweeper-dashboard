## Summary

Keep this PR open. It is a recent, protected `security`/`maintainer` XL storage refactor with current-head source-reproducible correctness and security blockers, plus insufficient representative real behavior proof for the SQLite runtime cutover.

## What This Changes

This PR moves OpenClaw runtime state from file-backed JSON/JSONL/sidecar stores into global and per-agent SQLite databases across core, apps, plugins, docs, workflows, and package metadata.

## Best Possible Solution

Keep this as an active maintainer-reviewed SQLite storage refactor, fix the blocking correctness/security findings, align or explicitly change the Node support contract, and require representative runtime plus migration proof before merge.

## Reproduction Assessment

Yes. The blocking findings are source-reproducible on PR head 98459363885b114b36cac6141e5e5c01b0283ad5 from the OAuth lock/save path, env-less doctor migration calls, removed config-audit contribution, transcript parsing fallback, media import scope, and Node 22 contract mismatch; no tests were run because this sweep is read-only.

## Solution Assessment

No, not yet. The SQLite direction may be intended, but the current patch needs scoped state writes, auth-store serialization, restored secret-scrub coverage, safer legacy parsing/import behavior, and support-policy alignment before it is the maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Serialize refresh writes by auth store:** `src/agents/auth-profiles/oauth-manager.ts:328-335`
  - body: The refresh lock is keyed by provider/profile, but the code reads and saves the whole auth-profile store. Concurrent refreshes for different profiles in the same store can take different locks and clobber the earlier refreshed token with a stale snapshot.
  - confidence: 0.9
- **[P1] Pass scoped env into auth-profile migration writes:** `src/commands/doctor/legacy/runtime-state.ts:453`
  - body: `maybeRepairLegacyRuntimeStateFiles` discovers legacy auth state under the caller env, then calls an importer that writes through default state DB options. A doctor run targeting a non-default state dir can delete the requested legacy file after writing its state to the wrong SQLite database.
  - confidence: 0.88
- **[P1] Restore config-audit scrub in doctor:** `src/flows/doctor-health-contributions.ts:718-726`
  - body: The doctor contribution list no longer invokes the config-audit scrub path, so `doctor --fix` stops warning/fixing historical `config-audit.jsonl` entries that may contain pre-redactor plaintext argv secrets.
  - confidence: 0.86
- **[P2] Pass caller env when resolving legacy approvals:** `src/commands/doctor/legacy/plugin-conversation-binding.ts:12-16`
  - body: The legacy plugin-binding approval importer resolves its file path from global `process.env` instead of the doctor caller env. Runs with an explicit state dir can skip approvals or import them from the wrong profile.
  - confidence: 0.9
- **[P2] Preserve session-id extraction with corrupt transcript lines:** `src/commands/doctor/legacy/session-transcript-health.ts:308-310`
  - body: The sidecar migration parses the entire transcript to recover the session id, but `parseTranscriptEntries` returns an empty list when any line is malformed. A valid header followed by one corrupt line falls back to the filename and can import/delete the sidecar under the wrong session key.
  - confidence: 0.84
- **[P2] Route legacy media imports to scoped state database:** `src/commands/doctor/legacy/media.ts:133-138`
  - body: `importLegacyMediaFilesToSqlite` accepts an explicit env, but each imported blob is written through `saveMediaBufferWithId`, which writes to the process-level state DB. Scoped doctor migration can delete media from the requested state dir while persisting blobs elsewhere.
  - confidence: 0.86
- **[P2] Keep managed image blobs in the target state scope:** `src/commands/doctor/legacy/managed-image-attachments.ts:46-51`
  - body: Managed outgoing records are written to the caller-provided `stateDir`, but their original blobs go through `saveMediaBufferWithId` and can land in the default state DB. The migrated record can then point at media missing from the target state.
  - confidence: 0.85
- **[P2] Keep the Node 22 compatibility contract covered:** `.github/instructions/copilot.instructions.md:7`
  - body: This changes guidance to Node 24+ and removes the Node 22 compatibility CI lane while package metadata and root policy still promise Node 22.16+ support. Either keep coverage or make the support-policy change explicit across docs, engines, and release validation.
  - confidence: 0.82
