Full review comments:

- **[P1] Reject empty config-health imports before deleting rows:** `src/commands/doctor/legacy/runtime-state.ts:166-167`

  - confidence: 0.92
- **[P1] Reject malformed Voice Wake snapshots before rewriting triggers:** `src/commands/doctor/legacy/voicewake.ts:40-42`
  - body: This importer normalizes any parsed legacy value and writes it into SQLite. A stale `{}` becomes default triggers, and `writeVoiceWakeConfigSnapshot` deletes/replaces the existing trigger rows, so upgrade repair can reset a user's current Voice Wake config and remove the legacy source.
  - confidence: 0.91
- **[P1] Do not overwrite current device identity from stale JSON:** `src/commands/doctor/legacy/device-identity.ts:156-158`
  - body: A valid legacy `identity/device.json` is written to SQLite without checking whether the default `device_identities` row already exists. In a delayed doctor run, an old file can roll back the current device keypair and break existing pairings or signatures before the source is removed.
  - confidence: 0.9
- **[P2] Skip stale plugin indexes when SQLite already has one:** `src/commands/doctor/legacy/installed-plugin-index.ts:46-49`
  - body: The runtime-state doctor path writes the legacy installed-plugin index unconditionally. That bypasses the shared registry migration's existing-row guard, so a stale `installed-plugin-index.json` can roll back plugin install metadata used by startup and plugin location lookups.
  - confidence: 0.86
