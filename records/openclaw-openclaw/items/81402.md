  - confidence: 0.93
- **[P1] Merge device-auth imports instead of replacing rows:** `src/commands/doctor/legacy/device-auth-store.ts:39`
  - body: This migration writes the legacy snapshot through `writeDeviceAuthStoreSnapshot`, whose writer deletes rows for other device IDs and roles missing from the imported snapshot. A stale legacy file can roll back or remove newer SQLite device tokens before the source is deleted.
  - confidence: 0.91
- **[P1] Validate web-push snapshots before replacement writes:** `src/commands/doctor/legacy/push-web.ts:47-50`
  - body: The importer accepts any parsed object and sends it to `writeWebPushRegistrationStateSnapshot`; the writer deletes all subscriptions when the coerced snapshot has no rows. A malformed or shape-drifted file can wipe existing SQLite registrations and then be removed.
  - confidence: 0.9
- **[P2] Reject malformed node-host config before saving:** `src/commands/doctor/legacy/node-host-config.ts:42-43`
  - body: This casts a coerced partial value, falling back to `{}`, then saves it as node-host config. Malformed-but-parseable legacy JSON can generate a fresh node id, overwrite current SQLite node-host state, and remove the source file, making rollback difficult.
  - confidence: 0.86
- **[P1] Merge pairing snapshots instead of replacing rows:** `src/commands/doctor/legacy/pairing-files.ts:52-68`
