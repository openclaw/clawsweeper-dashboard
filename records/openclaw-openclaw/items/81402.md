  - confidence: 0.91
- **[P1] Reject empty APNS snapshots before replacing rows:** `src/commands/doctor/legacy/push-apns.ts:40`
  - body: normalizeApnsRegistrationStateSnapshot({}) returns an empty snapshot, and writeApnsRegistrationStateSnapshot deletes all apns_registrations rows for an empty state. A stale or shape-drifted legacy file can therefore wipe newer APNS registrations during doctor --fix, so this path should reject empty/malformed input or merge without treating the legacy file as authoritative.
  - confidence: 0.9
- **[P1] Merge bootstrap tokens instead of replacing current rows:** `src/commands/doctor/legacy/device-bootstrap.ts:35-40`
  - body: This passes legacy bootstrap.json directly to writePairingStateRecord, whose devices/bootstrap branch deletes device_bootstrap_tokens rows not present in the snapshot. If SQLite already has newer bootstrap tokens and the old file remains, doctor --fix can roll back or remove current onboarding tokens, so import should merge or skip stale rows rather than replace the table.
  - confidence: 0.9
- **[P1] Serialize expiry saves through the commitments writer:** `src/commitments/store.ts:453`
  - body: loadCommitmentStoreWithExpiredMarked now saves expired commitments without the commitments writer queue that protects other read-modify-write paths.
