## Summary

Keep open: current main still retries API-key rotation only for caller-provided predicates or rate-limit-shaped errors, while the media and Google embedding call sites do not opt into 5xx, timeout, or network retries. The linked implementation candidate remains open at https://github.com/openclaw/openclaw/pull/75219, so the issue should stay open until that PR is reviewed, landed, replaced, or closed.

## What This Changes

Request bounded transient retries for provider execution failures such as 5xx responses, timeouts, and network errors instead of only rate-limit key rotation.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/75219 with bounded opt-in same-key transient retries for idempotent provider operations while preserving 429 alternate-key rotation.

## Reproduction Assessment

Yes, source-reproducible. On current main, call executeWithApiKeyRotation with one key and an execute callback that throws Error("HTTP 500") or an AbortError; the helper classifies only rate-limit errors by default, so it calls once and rethrows.

## Solution Assessment

Yes, pending PR review and proof. The narrow maintainable direction is opt-in same-key transient retries for selected safe provider operations, not global retries or rotating API keys on 5xx failures.
