## Summary

Keep open: current main still checks only the resolved `/steer` target lane for an active embedded run, and https://github.com/openclaw/openclaw/pull/81613 is an open implementation candidate that should be reviewed before this issue closes.

## What This Changes

The issue requests `/steer` to fall back from an idle slash-command lane to an active sibling direct/DM lane for the same peer.

## Best Possible Solution

Review and land a focused fix through https://github.com/openclaw/openclaw/pull/81613, with regression coverage for slash-lane fallback and maintainer confirmation of the sibling-key scope.

## Reproduction Assessment

Yes. A focused source reproduction can call `handleSteerCommand` with a slash-lane `sessionKey` and an active embedded run registered on the sibling direct lane, then observe that current main checks only the slash lane.

## Solution Assessment

Yes. The narrow solution belongs in `/steer` session-id resolution: prefer the requested key when it is active, then fall back only to same-peer direct/DM siblings when the slash lane is idle.
