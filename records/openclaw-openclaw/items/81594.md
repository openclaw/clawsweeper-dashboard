## Summary

Keep open: current main still has the reported `/steer` target-selection gap, and an open linked fix PR already references this issue with closing syntax, so the issue should remain open until that PR is reviewed, proved, and merged or closed.

## What This Changes

The issue reports that text `/steer` can resolve to a `:slash:` command lane and miss an active same-peer direct/DM lane.

## Best Possible Solution

Review and land the linked narrow `/steer` fix with regression coverage for slash-to-direct/DM fallback, then close this issue from the merged PR.

## Reproduction Assessment

Yes, from source inspection. A focused harness can set `sessionKey` to `agent:main:telegram:slash:<peer>`, register the active run under `agent:main:telegram:direct:<peer>`, and observe that current main only checks the slash key before falling back as idle.

## Solution Assessment

Yes. The narrowest maintainable fix is in `/steer` session-id resolution: keep native `CommandTargetSessionKey` precedence, but when an idle slash lane is selected, probe same-peer direct/DM siblings before using the existing normal-prompt fallback.
