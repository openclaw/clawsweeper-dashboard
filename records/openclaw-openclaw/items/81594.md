## Summary

Keep open: current main still resolves `/steer` to one target key and only checks that lane for an active embedded run, while the open paired fix PR already references this issue with closing syntax. The right cleanup is to keep this issue open until that implementation is reviewed, proven, and either merged or replaced.

## What This Changes

The issue reports that text `/steer` can resolve to a slash-command lane and miss an active sibling direct/DM run.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81613, or replace it with the same narrow fallback plus regression coverage for slash-lane to sibling direct/DM active-run selection.

## Reproduction Assessment

Yes. A focused harness can call `handleSteerCommand` with a text-command `:slash:` session key, no active run for that key, and an active embedded run registered on the same-peer `:direct:` or `:dm:` key; current main checks only the slash key.

## Solution Assessment

Yes. The narrowest maintainable fix is in `/steer` session-id resolution: preserve the requested lane when active, and only for an idle slash lane probe same-peer direct/DM siblings before existing fallback behavior.
