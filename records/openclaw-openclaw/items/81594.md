## Summary

Keep open: current main still resolves `/steer` against one target lane and the open paired fix PR references this issue with closing syntax, so this issue should remain open until that PR is reviewed, proven, and merged or replaced.

## What This Changes

The issue asks for text `/steer` to fall back from an inactive slash-command lane to the active same-peer direct/DM lane.

## Best Possible Solution

Review and land the open paired fix PR, or replace it with the same tightly scoped slash-lane sibling fallback plus regression coverage for direct/DM active-run selection.

## Reproduction Assessment

Yes. A focused current-main harness can call `handleSteerCommand` with a `:slash:` session key that has no active run and a same-peer `:direct:` or `:dm:` active embedded run; source inspection shows current main checks only the slash key.

## Solution Assessment

Yes. The narrowest maintainable fix is in `/steer` session-id resolution: keep the exact requested key when active, then only for inactive `:slash:` lanes probe same-peer `:direct:` and `:dm:` siblings before preserving the existing fallback behavior.
