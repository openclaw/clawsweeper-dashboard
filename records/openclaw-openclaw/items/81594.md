## Summary

Keep open: current main and v2026.6.8 still lack the slash-lane to direct/DM active-run fallback, and the linked closing PR remains open and conflicting rather than merged.

## What This Changes

The issue requests that text `/steer` fall back from an inactive slash-command lane to an active sibling direct/DM lane for the same peer.

## Best Possible Solution

Land or replace the linked PR with a current-main fallback in `resolveSteerSessionId()` that tries active direct/DM sibling lanes only after a slash-lane target has no active run, with focused regression coverage and real behavior proof.

## Reproduction Assessment

Yes for a source-level reproduction: make a text `/steer` command run with `sessionKey` on a `:slash:` lane while the active embedded run exists only on a sibling `:direct:` or `:dm:` lane; current main checks the slash key and then falls through as a normal prompt.

## Solution Assessment

Yes, the best fix shape is narrow: add slash-lane sibling active-run fallback where `/steer` already has the target key and active-run lookup. The open linked PR is the current candidate, but it needs refresh, tests, and proof before this issue can close.
