## Summary

Keep open: current main still checks only the resolved `/steer` target lane for an active embedded run, and https://github.com/openclaw/openclaw/pull/81613 is an open implementation PR that references this issue with closing syntax.

## What This Changes

The issue reports that text `/steer` can resolve to a slash-command session lane and miss an active embedded run on the sibling direct/DM lane for the same peer.

## Best Possible Solution

Review and either land or replace https://github.com/openclaw/openclaw/pull/81613 with a narrow same-peer slash-to-direct/DM fallback plus regression coverage, then close this issue after the fix merges.

## Reproduction Assessment

Yes. A focused current-main source check can pass `/steer` through a slash `sessionKey` while an active embedded run is stored under the sibling direct/DM key, and current code only checks the slash key.

## Solution Assessment

Yes. The narrowest maintainable solution is to preserve existing target precedence, then only when an idle slash lane is selected probe same-peer direct/DM siblings before falling back to the normal prompt path.
