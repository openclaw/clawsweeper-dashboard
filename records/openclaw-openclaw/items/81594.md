- `impact:message-loss`: A control message can be routed to the wrong lane and fail to reach the active run.
- `impact:session-state`: The failure is caused by session-lane state mismatch between slash and direct/DM keys.

## Summary

Keep open: current main still checks only the resolved `/steer` lane for an active run, and an open paired PR references this issue with closing syntax. The issue should stay open until that PR is reviewed, proven, and merged or replaced.

## What This Changes

The issue asks text `/steer` to fall back from an idle slash-command lane to a same-peer direct/DM lane with an active embedded run.

## Best Possible Solution

Review and land the open paired fix PR, or replace it with the same narrowly scoped slash-lane fallback plus regression coverage for slash-to-direct/DM active-run selection.

## Reproduction Assessment

Yes. A focused harness can call `handleSteerCommand` with a `:slash:` session key that has no active run while a same-peer `:direct:` or `:dm:` key has one; current main only checks the slash key.

## Solution Assessment

Yes. The narrowest maintainable fix is in `/steer` target/session-id resolution: preserve an active requested key, then only for idle slash lanes probe same-peer direct/DM siblings before using the existing fallback behavior.
