Keep open: current main still checks only one resolved `/steer` target key for an active run, while the linked open fix PR references this issue with closing syntax. The issue should remain open until https://github.com/openclaw/openclaw/pull/81613 is reviewed, proven, and either merged or replaced.

## What This Changes

The issue reports that text `/steer` can resolve to a slash-command lane instead of the active same-peer direct/DM lane, causing a false no-active-run result.

## Best Possible Solution

Review and land the open paired fix PR, or replace it with the same narrowly scoped sibling-lane fallback plus regression coverage for slash-lane to same-peer direct/DM active-run selection.

## Reproduction Assessment

Yes. A focused harness can call `handleSteerCommand` with a `:slash:` session key that has no active run and a same-peer `:direct:` or `:dm:` key with an active run; current main checks only the slash key.

## Solution Assessment

Yes. The narrow fix belongs in `/steer` target/session-id resolution: preserve active runs on the requested key, then only for idle `:slash:` lanes probe same-peer `:direct:` and `:dm:` siblings before existing fallback behavior.
