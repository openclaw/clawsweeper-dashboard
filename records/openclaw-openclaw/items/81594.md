## Summary

Keep open: current main and latest release v2026.6.10 still resolve text `/steer` to the slash-lane session first and stop after checking that one lane. The linked fix PR is open, unmerged, and conflicting, so this issue remains the canonical tracker.

## What This Changes

The issue requests that text `/steer` fall back from an inactive slash-command lane to active sibling direct/DM lanes for the same peer.

## Best Possible Solution

Refresh and land, or replace, the linked resolver fix with focused regression coverage for slash-lane fallback to active direct/DM siblings.

## Reproduction Assessment

Yes, at source level: text command turns prefer the slash-lane `sessionKey`, and the steer resolver checks only that selected lane before returning undefined. I did not run a live channel replay.

## Solution Assessment

Yes, the narrow maintainable solution is to adjust steer active-run resolution or target selection where the exact-key lookup happens. The linked PR follows that direction but still needs refresh, tests, and proof before merge.
