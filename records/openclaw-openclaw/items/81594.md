- `impact:message-loss`: The steering message can be routed away from the active run and continue as a normal prompt instead of reaching the intended session.
- `impact:session-state`: The bug is about choosing the wrong session lane when slash and direct/DM lanes represent the same peer.

## Summary

Keep open: current main and the latest release still check only the resolved `/steer` target lane, and the open paired fix PR at https://github.com/openclaw/openclaw/pull/81613 references this issue with closing syntax.

## What This Changes

The report asks `/steer` to fall back from an inactive slash-command lane to the same peer's active direct/DM lane.

## Best Possible Solution

Review and land the paired fix PR, or replace it with the same narrow fallback plus regression coverage for slash-lane to same-peer direct/DM active-run selection.

## Reproduction Assessment

Yes. A focused harness can call `handleSteerCommand` with a `:slash:` session key, no active run for that key, and an active embedded run registered for the same-peer `:direct:` or `:dm:` key; current main checks only the slash key.

## Solution Assessment

Yes. The narrow maintainable fix belongs in `/steer` target/session-id resolution: keep an active run on the requested key, then only for idle slash lanes probe same-peer direct/DM siblings before preserving fallback behavior.
