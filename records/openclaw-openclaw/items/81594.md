## Summary

Keep open: current main and v2026.6.8 still lack the slash-lane to direct/DM active-run fallback, and https://github.com/openclaw/openclaw/pull/81613 is open rather than merged, so this issue remains the canonical bug until that PR lands or is replaced.

## What This Changes

Text /steer can target a slash-command session lane instead of the active sibling direct/DM lane, causing the command to miss an active embedded run.

## Best Possible Solution

Land or replace the linked steer resolver fix with focused regression coverage that proves slash-lane /steer falls back only to active direct/DM siblings when the slash lane has no active run.

## Reproduction Assessment

Yes, source-reproducible from current main: a text /steer turn with a :slash: session key checks that key and its stored entry only, not an active direct/DM sibling. I did not run a live Discord or Telegram replay in this read-only review.

## Solution Assessment

Yes, the best fix shape is a narrow resolver fallback plus regression coverage; the linked PR is the active implementation candidate but is not merged.
