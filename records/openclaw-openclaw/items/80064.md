## Summary

Keep open: current main still matches the report. The voice-call plugin generates and persists raw `voice:*` keys while Gateway startup migration canonicalizes bare keys into `agent:<agentId>:...`, and the docs promise per-phone caller memory.

## What This Changes

Voice Call should keep per-phone and per-call sessions stable across Gateway restarts by aligning generated voice session keys with Gateway canonical session keys.

## Best Possible Solution

Make generated voice-call per-phone and per-call keys agent-scoped in the voice-call-owned resolver/call-record paths, with compatibility for existing raw voice-call records where needed.

## Reproduction Assessment

Yes, from source inspection: current main writes raw `voice:*` session keys and startup migration rewrites bare keys to canonical agent-scoped keys. A live restart repro was not run in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix belongs in the voice-call plugin’s generated session-key resolver because `sessionScope` is plugin-owned; the broader external raw-key policy should remain with https://github.com/openclaw/openclaw/issues/63005.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/80064. Current `@openclaw/voice-call` generated session keys are raw `voice:*` / `voice:call:*`, while Gateway startup migration canonicalizes bare session keys to `agent:<agentId>:...`, so per-phone memory can split after restart. Keep the fix in the voice-call plugin unless a tiny SDK/helper seam is clearly necessary. Update `resolveVoiceCallSessionKey` and its inbound/outbound/response-generator call sites so generated per-phone and per-call keys use the effective voice config `agentId` or `main` as `agent:<agentId>:voice:<phone>` and `agent:<agentId>:voice:call:<callId>`. Preserve documented explicit `sessionKey` behavior unless adding a small compatibility fallback/rewrite for legacy raw voice-call call records is required to avoid one-time history loss. Do not import core `src/**` internals from the plugin. Add focused regression tests for resolver output, call-record creation, response-generator store lookup/write, and a canonicalized store/restart-shaped case that reuses the same session id. Add a user-facing changelog entry because this fixes voice-call continuity.
