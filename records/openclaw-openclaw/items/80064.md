## What This Changes

The issue asks for the voice-call plugin’s per-phone and per-call session keys to use the gateway’s canonical agent-scoped session-key format so restart migration does not split caller history.

## Best Possible Solution

Review and land or reject the linked voice-call fix PR, keeping the broader raw external session-key behavior tracked separately in https://github.com/openclaw/openclaw/issues/63005.

## Reproduction Assessment

Yes, from source inspection: current main still writes raw `voice:*` session keys and startup migration canonicalizes bare keys to `agent:<agentId>:...`. I did not run a live gateway restart repro in this read-only pass.

## Solution Assessment
