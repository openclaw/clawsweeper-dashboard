## Summary

Keep open. Current main still has the reported raw `voice:*` Voice Call keys, Gateway startup still canonicalizes raw keys into agent-scoped keys, and an open linked fix PR already references this issue with closing syntax, so this should stay open until that PR or an equivalent fix lands.

## What This Changes

Voice Call per-phone/per-call session keys should use the same canonical agent-scoped format that survives Gateway startup session migration.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/80069 or an equivalent Voice Call-owned fix that emits normalized agent-scoped generated keys, updates affected call-path tests, and includes redacted restart proof.

## Reproduction Assessment

Yes. Source inspection reproduces the mismatch: Voice Call generates raw `voice:*` keys, while Gateway startup migration canonicalizes raw keys to `agent:<agentId>:voice:*`, so repeat caller lookup can miss after restart.

## Solution Assessment

Yes. Fixing the Voice Call-owned session resolver to emit canonical agent-scoped keys is the narrowest maintainable direction; the linked PR is the right path but needs its existing PR review issues and real behavior proof addressed before merge.
