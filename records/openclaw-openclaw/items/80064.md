## Summary

Keep open. Current main still has the reported raw-vs-canonical Voice Call session-key mismatch, and the related open fix PR has not merged, so there is no safe close reason yet.

## What This Changes

Voice Call generated per-phone and per-call session keys should use canonical agent-scoped keys so caller continuity survives Gateway restarts.

## Best Possible Solution

Land a Voice Call-owned fix that emits canonical agent-scoped generated keys from the configured agentId while preserving explicit session keys; leave the broader raw external key behavior to https://github.com/openclaw/openclaw/issues/63005.

## Reproduction Assessment

Yes, source-reproducible. The resolver emits raw `voice:*` keys, response generation reads/writes those keys, and Gateway migration canonicalizes raw keys to `agent:<agentId>:...`, so restart can split lookup from storage without a live telephony run.

## Solution Assessment

Yes. The narrow maintainable fix is in the Voice Call resolver/use sites: generated keys should be agent-scoped using the configured agentId, while explicit user-supplied session keys keep their current behavior.
