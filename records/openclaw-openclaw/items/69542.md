## Summary

Keep this PR open, but it is not mergeable as-is: current main does not already provide ACP backend failover, and the branch introduces blocking regressions in ACP stream completion, session backend resolution, cached handle cleanup, and retry safety while also mixing in unrelated plugin runtime-dependency staging work.

## What This Changes

The branch adds an `acp.fallbacks` config surface and wraps ACP `runTurn` with backend failover for unavailable-like turn failures, plus install-smoke parsing and plugin runtime-dependency staging changes.

## Best Possible Solution

Replace this with a narrower ACP-only failover design that preserves terminal stream validation, session metadata precedence, handle lifecycle cleanup, side-effect safety, and a precise retry classifier while leaving runtime-deps repair in its existing plugin-staging path.

## Reproduction Assessment

Yes for the PR defects via source inspection: the PR head removes current main's terminal `done` validation, computes candidates before session metadata resolution, clears cached handles without close, and retries without checking partial output. I did not run tests because this review is read-only.

## Solution Assessment

No. Backend failover may be useful, but this implementation is not the narrow maintainable solution because it weakens current ACP stream/session guarantees and mixes in unrelated plugin runtime-dependency staging work.

Full review comments:

- **[P1] Restore terminal done-event validation:** `src/acp/control-plane/manager.core.ts:868-889`
  - body: Current main rejects ACP streams that end without a terminal `done` event, and there is a regression test for that behavior. This branch records success after `awaitTurnWithTimeout` without an equivalent `sawTerminalEvent` check, so a backend that silently exits after output is marked succeeded instead of failed.
  - confidence: 0.94
- **[P1] Derive backend candidates after session resolution:** `src/acp/control-plane/manager.core.ts:724-736`
  - body: The new candidate list and missing-backend guard run before `resolveSession()`. Bound sessions whose backend comes from persisted metadata can now throw `ACP_BACKEND_MISSING` before the valid metadata backend is used, regressing the current `meta.backend || cfg.acp?.backend` precedence.
  - confidence: 0.91
- **[P1] Close the old handle before switching backends:** `src/acp/control-plane/manager.core.ts:749-750`
  - body: On fallback attempts this clears the runtime cache without first closing the previous persistent handle. Once the cache entry is discarded, idle eviction and local session accounting can no longer reclaim that old backend session.
  - confidence: 0.88
- **[P1] Block failover after output or tool calls:** `src/acp/control-plane/manager.core.ts:1022-1028`
  - body: The failover gate ignores whether the first backend already emitted text or tool calls. If an unavailable-looking error arrives after partial output or side effects, callers can receive output from backend A followed by a second full attempt on backend B.
  - confidence: 0.86
- **[P2] Tighten the failover error classifier:** `src/acp/control-plane/manager.core.ts:1022-1027`
  - body: Matching broad substrings like `rate` and `limit` can classify unrelated `ACP_TURN_FAILED` messages as failover-worthy. That can switch backends and mask the original error instead of surfacing it immediately.
  - confidence: 0.82
- **[P2] Route missing-backend errors through runTurn failure handling:** `src/acp/control-plane/manager.core.ts:732-736`
  - body: The new early `ACP_BACKEND_MISSING` throw happens after a prompt background task may have been created but before the existing `runTurn` catch path. That can leave prompt task state without the normal terminal failure bookkeeping.
  - confidence: 0.8
- **[P2] Split runtime-deps staging from ACP failover:** `scripts/stage-bundled-plugin-runtime-deps.mjs:921-940`
  - body: This branch materializes a host `openclaw` package under staged plugin runtime deps, changing package resolution on a plugin execution path. That is unrelated to ACP failover and overlaps the separately merged bundled runtime-deps staging fix path.
  - confidence: 0.84
