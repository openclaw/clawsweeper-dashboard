## Summary

Keep open, but this PR should not merge as-is: current main still lacks ACP backend fallbacks, while the branch regresses ACP stream/session guarantees, mixes in stale runtime-dependency staging work, and has no real behavior proof.

## What This Changes

Adds an `acp.fallbacks` config option and wraps ACP `runTurn` in backend failover, while also changing install-smoke parsing and bundled plugin runtime-dependency staging.

## Best Possible Solution

Replace this with a narrower ACP-only failover design that preserves terminal stream validation, session metadata precedence, handle lifecycle cleanup, side-effect safety, and a precise retry classifier; keep runtime-deps repair in its canonical plugin-staging path.

## Reproduction Assessment

Yes for the PR defects: comparing PR head to current main shows terminal done validation removed, backend candidates computed before session metadata resolution, and cached handles cleared without close on failover. I did not run tests because this review was read-only.

## Solution Assessment

No. Backend failover may be useful, but this implementation is not the narrow maintainable solution because it weakens existing ACP stream/session guarantees and mixes in unrelated runtime-dependency staging work.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Restore terminal done-event validation:** `src/acp/control-plane/manager.core.ts:868-888`
  - body: Current main rejects ACP streams that end without a terminal `done` event. This patch records success whenever `streamError` is empty, so a backend that silently exits after output is marked succeeded instead of failed.
  - confidence: 0.94
- **[P1] Derive backend candidates after session resolution:** `src/acp/control-plane/manager.core.ts:724-736`
  - body: The new candidate list and missing-backend guard run before `resolveSession()`. Bound sessions whose backend comes from binding metadata, runtime defaults, or registry selection can now throw `ACP_BACKEND_MISSING` before the valid backend is used.
  - confidence: 0.91
- **[P1] Close the old handle before switching backends:** `src/acp/control-plane/manager.core.ts:749-750`
  - body: On fallback attempts this clears the runtime cache without closing the previous persistent handle. Once the cache entry is discarded, idle eviction and local session accounting can no longer reclaim that old backend session.
  - confidence: 0.87
- **[P1] Block failover after output or tool calls:** `src/acp/control-plane/manager.core.ts:1022-1028`
  - body: The failover gate ignores whether the first backend already emitted text or tool calls. If an unavailable-looking error arrives after partial output or side effects, callers can receive output from backend A followed by a second full attempt on backend B.
  - confidence: 0.86
- **[P2] Tighten the failover error classifier:** `src/acp/control-plane/manager.core.ts:1022-1027`
  - body: Matching broad substrings like `rate` and `limit` can classify unrelated `ACP_TURN_FAILED` messages as failover-worthy. That can switch backends and mask the original error instead of surfacing it.
  - confidence: 0.82
- **[P2] Split runtime-deps staging from ACP failover:** `scripts/stage-bundled-plugin-runtime-deps.mjs:921-940`
  - body: This branch materializes a host `openclaw` package under staged plugin runtime deps, changing package resolution on a plugin execution path. That work is unrelated to ACP failover and belongs in the already separated runtime-deps staging path.
  - confidence: 0.84
