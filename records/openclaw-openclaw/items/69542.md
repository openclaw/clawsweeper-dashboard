## Summary

Keep open: current main does not implement `acp.fallbacks`, and this PR is not safe to merge as-is because the branch weakens ACP turn/session guarantees, lacks real ACP failover proof, and includes unrelated plugin runtime-dependency staging work already covered by a separate merged path.

## What This Changes

Adds `acp.fallbacks` config/schema/help entries, wraps ACP `runTurn` in backend failover, adjusts install-smoke JSON parsing, and stages an `openclaw` package root in plugin runtime deps.

## Best Possible Solution

Use a narrower ACP-only failover design that preserves terminal stream validation, session metadata precedence, handle cleanup, side-effect safety, and a precise retry classifier, with runtime-deps work kept in the existing plugin staging path.

## Reproduction Assessment

Yes for the PR defects by source inspection: comparing PR head to current main shows the removed terminal `done` validation, pre-resolution backend candidate selection, and cache clearing without handle close. I did not run tests because this review is read-only.

## Solution Assessment

No. Backend failover may be useful, but this implementation is not the narrow maintainable path because it weakens existing ACP stream/session guarantees and mixes unrelated package-resolution staging work.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Restore terminal done-event validation:** `src/acp/control-plane/manager.core.ts:868-888`
  - body: Current main rejects ACP streams that finish without a terminal `done` event. This patch records success whenever `streamError` is empty, so a backend that silently exits after output is marked succeeded instead of failed.
  - confidence: 0.94
- **[P1] Derive backend candidates after session resolution:** `src/acp/control-plane/manager.core.ts:724-736`
  - body: The new candidate list and missing-backend guard run before `resolveSession()`. Bound sessions whose backend comes from stored metadata can now throw `ACP_BACKEND_MISSING` before the valid backend is used.
  - confidence: 0.91
- **[P1] Close the old handle before switching backends:** `src/acp/control-plane/manager.core.ts:749-750`
  - body: On fallback attempts this clears the runtime cache without closing the previous persistent handle. Once discarded, idle eviction and local session accounting can no longer reclaim that backend session.
  - confidence: 0.87
- **[P1] Block failover after output or tool calls:** `src/acp/control-plane/manager.core.ts:1022-1028`
  - body: The failover gate ignores whether the first backend already emitted text or tool calls. An unavailable-looking error after partial output can make callers see output from backend A followed by a second full attempt on backend B.
  - confidence: 0.86
- **[P2] Tighten the failover error classifier:** `src/acp/control-plane/manager.core.ts:1022-1027`
  - body: Matching broad substrings like `rate` and `limit` can classify unrelated `ACP_TURN_FAILED` messages as failover-worthy, switching backends and masking the original error.
  - confidence: 0.82
- **[P2] Split runtime-deps staging from ACP failover:** `scripts/stage-bundled-plugin-runtime-deps.mjs:921-940`
  - body: This branch materializes a host `openclaw` package under staged plugin runtime deps, changing package resolution on a plugin execution path. That work is unrelated to ACP failover and belongs in the separately maintained runtime-deps staging path.
  - confidence: 0.84
