## Summary

Keep open: the ACP fallback feature is not on current main, and the supplied terminal proof is useful, but the latest patch still misses advertised failover cases in the classifier.

## What This Changes

The PR adds `acp.fallbacks` config wiring and an ACP `runTurn` backend loop that retries pre-output availability failures across fallback ACP runtime backends, with tests and a changelog entry.

## Best Possible Solution

Keep the feature in ACP manager/config, but land it only after the classifier and tests cover registry-level backend-unavailable failures and common rate-limit messages.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to configure a primary backend with `acp.fallbacks`, have `requireAcpRuntimeBackend` throw `ACP_BACKEND_UNAVAILABLE` before output, and observe that the PR classifier records it as terminal instead of trying the fallback.

## Solution Assessment

No, not yet. The overall ACP-manager approach is plausible, but the narrow maintainable repair is to include the registry unavailable code and common rate-limit wording in the pre-output failover classifier with focused tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Fail over on unhealthy backend errors:** `src/acp/control-plane/manager.core.ts:770-773`
  - body: When the configured primary backend is registered but unhealthy, `requireRuntimeBackend` throws `ACP_BACKEND_UNAVAILABLE` before a turn starts. This classifier only accepts `ACP_TURN_FAILED` and `ACP_SESSION_INIT_FAILED`, so `acp.fallbacks` are not tried for that UNAVAILABLE path; include `ACP_BACKEND_UNAVAILABLE` and add a regression test.
  - confidence: 0.91
- **[P2] Match rate-limit wording before failover:** `src/acp/control-plane/manager.core.ts:773`
  - body: The advertised condition includes rate limiting, but this regex matches `rate limited` while missing common messages like `rate limit`, `rate limiting`, and `rate limit exceeded`. Those primary errors will surface instead of trying configured fallbacks, so broaden the matcher and cover the wording in a focused test.
  - confidence: 0.86
