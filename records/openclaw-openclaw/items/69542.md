## Summary

Keep this PR open. Current main does not implement ACP backend fallbacks, and the branch adds useful config/runtime/test work, but the latest head still misses an UNAVAILABLE registry path and common rate-limit wording in the failover classifier, so it should not merge as-is.

## What This Changes

This PR adds `acp.fallbacks` config wiring and an ACP manager backend loop that retries pre-output availability failures on configured fallback runtime backends, with tests and a changelog entry.

## Best Possible Solution

Land the feature only after the classifier covers all pre-output backend availability failures and common rate-limit wording, with focused regression tests and maintainer approval for the new config surface.

## Reproduction Assessment

Yes. A source-level reproduction is to configure a primary backend plus `acp.fallbacks`, make `requireAcpRuntimeBackend` throw `ACP_BACKEND_UNAVAILABLE` for the primary before output, and observe that the PR classifier excludes that code before trying the fallback.

## Solution Assessment

No, not yet. The design is close, but the narrow maintainable repair is to include registry-level backend-unavailable failures and common rate-limit wording in the pre-output failover classifier, then cover both with manager tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Fail over on unhealthy backend errors:** `src/acp/control-plane/manager.core.ts:770-773`
  - body: When the configured primary backend is registered but unhealthy, `requireAcpRuntimeBackend` throws `ACP_BACKEND_UNAVAILABLE` before any turn starts. This classifier only accepts `ACP_TURN_FAILED` and `ACP_SESSION_INIT_FAILED`, so `acp.fallbacks` are never tried for that UNAVAILABLE path; include `ACP_BACKEND_UNAVAILABLE` here and add a manager regression test.
  - confidence: 0.91
- **[P2] Match rate-limit wording before failover:** `src/acp/control-plane/manager.core.ts:773-775`
  - body: The advertised failover condition includes rate limiting, but this regex matches `rate limited` while missing common messages like `rate limit`, `rate limiting`, and `rate limit exceeded`. Those errors will surface from the primary instead of trying configured fallbacks, so broaden the matcher and cover the wording in a focused test.
  - confidence: 0.86
