Full review comments:

- **[P1] Keep tokens out of dashboard logs:** `src/commands/dashboard.ts:25`
  - body: This logs `dashboardUrl`, which includes `#token=` for plain gateway tokens. Current main intentionally logs only the base URL and has CVE regression coverage that treats token fragments in console-captured logs as a security bug; keep copy/open tokenized but human logs token-free.
  - confidence: 0.95
- **[P1] Keep tokens out of SSH hints:** `src/commands/dashboard.ts:49-53`
  - body: Passing the gateway token into `formatControlUiSshHint` can render it into terminal/log output when browser launch fails. Current main calls the SSH hint without the token and uses separate manual-auth guidance, so preserve that boundary.
  - confidence: 0.94
- **[P1] Preserve TLS when resolving control UI links:** `src/commands/control-ui-access-url.ts:39-44`
  - body: The new resolver calls `resolveControlUiLinks` without `tlsEnabled`, so TLS-enabled gateways get `http`/`ws` dashboard URLs even though current main emits `https`/`wss`. Carry the gateway TLS setting through this shared path.
  - confidence: 0.91
- **[P2] Skip token resolution when Control UI is disabled:** `src/cli/daemon-cli/status.gather.ts:333-337`
  - body: `gatherDaemonStatus` resolves the dashboard access URL unconditionally, even when status output will show `Dashboard: disabled`. Guard this call so disabled-Control-UI deployments do not invoke SecretRef/token providers for a URL that is never displayed.
  - confidence: 0.86
