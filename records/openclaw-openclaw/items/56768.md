## Summary

Keep open. Current main still lacks the requested token-aware `gateway status` dashboard URL, so the PR is not obsolete, but the patch is not mergeable as-is because it regresses existing token-log safety, TLS link rendering, and disabled-Control-UI behavior.

## What This Changes

The PR adds a shared Control UI access URL resolver, wires it into `openclaw gateway status` and `openclaw dashboard`, redacts token fragments in JSON status output, adds SecretRef hints/tests, and adds a changelog entry.

## Best Possible Solution

Centralize the dashboard/status access URL logic only after it preserves the current clean-log dashboard contract, passes TLS state through link resolution, skips token work when Control UI is disabled, redacts machine-readable output, and has explicit maintainer/secops approval for any token-bearing human status output.

## Reproduction Assessment

Yes. Current main's status print path always renders `links.httpUrl`, and the PR branch makes plain-token auth produce `#token=` URLs; the dashboard token-log regressions are covered by the existing dashboard link tests.

## Solution Assessment

No. The shared-resolver direction is reasonable, but this implementation is not the narrowest maintainable fix until it preserves token-log, SSH-hint, TLS, and disabled-dashboard contracts and gets security approval for human status token exposure.

  - body: Current main logs only the clean Control UI URL while sending the tokenized URL to clipboard/browser, and the regression test rejects any `#token=` in `runtime.log`. This refactor logs `access.dashboardUrl`, so plain/env token auth would be emitted to CLI output and captured logs.

  - confidence: 0.9
- **[P2] Pass TLS state into access URL resolution:** `src/commands/control-ui-access-url.ts:35-41`
  - body: Current main threads `gateway.tls.enabled` into dashboard/status link rendering so TLS gateways get `https://` Control UI URLs. The new resolver calls `resolveControlUiLinks` without `tlsEnabled`, so shared dashboard/status URLs fall back to `http://` even when the gateway is serving TLS.
  - confidence: 0.9
- **[P2] Skip token resolution when Control UI is disabled:** `src/cli/daemon-cli/status.gather.ts:330-334`
  - body: `gateway status` already prints `Dashboard: disabled` when `gateway.controlUi.enabled === false`, but this code resolves dashboard access data unconditionally before printing. That can invoke SecretRef/token providers and add latency or failures for a URL that will not be used.
  - confidence: 0.82
