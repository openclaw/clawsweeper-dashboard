The PR extracts Control UI access URL generation into a shared resolver, uses it from `openclaw gateway status` and `openclaw dashboard`, redacts token fragments in status JSON, and adds tests plus a changelog entry.

## Best Possible Solution

Rebase onto current main and implement status URL parity through the existing dashboard/auth/link contracts while preserving token-free dashboard logs and SSH hints, TLS-aware URLs, readiness gating, disabled-Control-UI behavior, JSON redaction, and adding redacted real CLI proof.

## Solution Assessment

No, not as currently proposed: sharing URL resolution is a reasonable direction, but this branch must preserve token-free logs and SSH hints, TLS handling, readiness checks, and disabled-Control-UI behavior before it is the right fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Do not log tokenized dashboard URLs:** `src/commands/dashboard.ts:25`
  - body: Current main intentionally logs only the base Dashboard URL while sending the tokenized URL to clipboard/browser, and the adjacent CVE regression test asserts logs must never contain `#token=`. This branch logs `access.dashboardUrl`, so a locally resolved gateway token can land in terminal output or captured logs.
  - confidence: 0.93
- **[P1] Do not pass gateway tokens to SSH hints:** `src/commands/dashboard.ts:49-53`
  - body: Current main's SSH fallback test verifies `formatControlUiSshHint` never receives the token because that hint is written to the same captured log path. Passing `token` here reintroduces that secret exposure when browser launch fails.
  - confidence: 0.9
- **[P1] Preserve TLS when resolving access URLs:** `src/commands/control-ui-access-url.ts:39-44`
  - body: `resolveControlUiLinks` only emits `https`/`wss` when `tlsEnabled` is passed, but the new resolver never forwards `cfg.gateway.tls.enabled`. TLS-enabled gateways would get plain `http`/`ws` dashboard/status URLs, regressing current TLS-aware behavior.
  - confidence: 0.88
- **[P2] Skip token resolution when Control UI is disabled:** `src/cli/daemon-cli/status.gather.ts:333-337`
  - body: `status.print` hides the Dashboard URL when Control UI is disabled, but `gatherDaemonStatus` resolves the access URL unconditionally. For SecretRef-backed tokens this can invoke secret-provider resolution, latency, or failures for a URL the command will not print.
  - confidence: 0.78
