Full review comments:

- **[P1] Keep tokenized dashboard URLs out of logs:** `src/commands/dashboard.ts:25`
  - body: Current main intentionally logs only the base Dashboard URL while sending the tokenized URL to clipboard/browser, and regression tests assert logs must never contain `#token=`.   - confidence: 0.93
- **[P1] Keep gateway tokens out of SSH hints:** `src/commands/dashboard.ts:49-53`
  - body: `formatControlUiSshHint` returns text that is written to `runtime.log`; current main deliberately calls it without the token. Passing the token here reintroduces the same secret exposure when browser launch fails.
  - confidence: 0.9
- **[P1] Forward TLS state when resolving Control UI links:** `src/commands/control-ui-access-url.ts:39-44`
  - body: `resolveControlUiLinks` emits `https`/`wss` only when `tlsEnabled` is passed, but this helper never forwards `cfg.gateway.tls.enabled`. TLS-enabled gateways would get plain `http`/`ws` dashboard/status URLs.
  - confidence: 0.88
- **[P2] Skip access URL resolution when Control UI is disabled:** `src/cli/daemon-cli/status.gather.ts:333-337`
  - body: `status.print` hides the Dashboard URL when Control UI is disabled, but `gatherDaemonStatus` resolves token access data unconditionally. For SecretRef-backed tokens this can invoke secret-provider resolution, latency, or failures for a URL the command will not print.
  - confidence: 0.78
