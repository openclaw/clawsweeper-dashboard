## Summary

Keep open. The status/dashboard parity gap still exists on current main and the linked issue was closed in favor of this PR, but this branch is not mergeable as-is because it reintroduces token exposure paths, misses current TLS-aware links, and lacks real behavior proof.

## What This Changes

The PR extracts Control UI URL resolution, uses it in `openclaw gateway status`, redacts token fragments from JSON status, adds SecretRef hints/tests, and updates the changelog.

## Best Possible Solution

Rebuild the fix on current auth and Control UI link helpers so status can offer an auth-capable handoff without logging secrets, while preserving TLS, disabled-Control-UI, SecretRef, JSON redaction, and focused tests.

## Reproduction Assessment

Yes, by source inspection: current main's `gateway status` prints the bare Control UI URL while `dashboard` delivers a tokenized browser/clipboard URL under token auth. I did not run the CLI in this read-only review.

## Solution Assessment

No. A shared resolver is a reasonable direction, but this branch must be rebuilt on current token-log, TLS, disabled-Control-UI, SecretRef, and JSON-redaction behavior before it is the best mergeable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Keep token fragments out of dashboard logs:** `src/commands/dashboard.ts:25`
  - body: `dashboardUrl` can contain `#token=...` for plain token auth, and this line writes it to runtime logs. Current main deliberately logs only the clean URL and has regression coverage forbidding both the token value and `#token=` in dashboard log output.
  - confidence: 0.94
- **[P1] Keep tokens out of SSH fallback hints:** `src/commands/dashboard.ts:49-53`
  - body: Passing the token into `formatControlUiSshHint` lets the returned fallback hint include a tokenized URL, and that hint is logged when browser opening fails. Current main deliberately calls the helper without a token for this logged path.
  - confidence: 0.9
- **[P2] Preserve TLS when resolving Control UI links:** `src/commands/control-ui-access-url.ts:39-44`
  - body: The new resolver calls `resolveControlUiLinks` without the gateway TLS flag. TLS-enabled gateways can therefore get `http://` dashboard links even though current main's link contract supports `https://` and `wss://`.
  - confidence: 0.9
- **[P2] Skip token resolution when Control UI is disabled:** `src/cli/daemon-cli/status.gather.ts:333-337`
  - body: `gateway status` will not show a dashboard URL when Control UI is disabled, but this code still resolves token and SecretRef access data. Guard the resolver so disabled UI setups avoid unnecessary provider latency or failures for a URL that is never used.
  - confidence: 0.84
