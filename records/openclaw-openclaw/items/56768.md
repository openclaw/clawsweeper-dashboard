## Summary

Keep open: current main still has the status/dashboard token URL parity gap, but this PR head is not mergeable as-is because it would leak tokenized dashboard URLs in logged dashboard paths, drops TLS when resolving the new access URL, and still lacks real after-fix CLI proof. The linked issue is already closed as superseded by this PR, so this should remain the canonical PR until it is fixed or replaced.

## What This Changes

The PR adds a shared Control UI access URL resolver, uses it from `openclaw gateway status`, redacts token fragments from JSON status output, adds SecretRef hints, and updates focused tests and changelog text.

## Best Possible Solution

Keep the canonical fix focused on status/dashboard parity, but implement it on current helpers with separate clean display/log URLs and tokenized delivery URLs, TLS propagation, disabled-Control-UI short-circuiting, SecretRef-safe hints, JSON redaction, focused tests, and redacted real CLI proof.

## Reproduction Assessment

Yes, by source inspection: current `gateway status` prints a bare Control UI URL while current `dashboard` builds a tokenized browser/clipboard URL under token auth. I did not run the CLI in this read-only review.

## Solution Assessment

No. Sharing URL resolution is a reasonable direction, but this branch must be rebuilt on current token-log, TLS, disabled-Control-UI, SecretRef, and JSON-redaction behavior before it is the narrowest maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Keep token fragments out of dashboard logs:** `src/commands/dashboard.ts:25`
  - body: `dashboardUrl` can contain `#token=...` for plain token auth, and this logs it directly. Current main deliberately logs only the clean URL and has regression coverage forbidding both the token value and `#token=` in dashboard log output.
  - confidence: 0.94
- **[P1] Keep tokens out of SSH fallback hints:** `src/commands/dashboard.ts:49-53`
  - body: Passing the resolved token into `formatControlUiSshHint` lets the returned fallback hint include a tokenized URL, and that hint is logged when browser launch is unavailable. Current main calls the helper without a token for this logged path.
  - confidence: 0.9
- **[P2] Preserve TLS when resolving Control UI links:** `src/commands/control-ui-access-url.ts:39-44`
  - body: The new resolver calls `resolveControlUiLinks` without the gateway TLS flag. TLS-enabled gateways can therefore get `http://` dashboard links even though current main's link contract supports `https://` and `wss://`.
  - confidence: 0.9
- **[P2] Skip token resolution when Control UI is disabled:** `src/cli/daemon-cli/status.gather.ts:333-337`
  - body: `gateway status` will not show a dashboard URL when Control UI is disabled, but this code still resolves token and SecretRef access data. Guard the resolver so disabled UI setups avoid unnecessary provider latency or failures for a URL that is never used.
  - confidence: 0.84
