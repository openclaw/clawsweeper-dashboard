Overall confidence: 0.9

Full review comments:

- **[P1] Keep bearer tokens out of status output:** `src/cli/daemon-cli/status.print.ts:190`
  - body: `dashboardUrl` can include `#token=...` from `resolveControlUiAccessUrl`, so this line makes normal `openclaw gateway status` output expose the gateway bearer token in terminals, logs, and issue pastes. Current `openclaw dashboard` deliberately logs only the base URL while sending the tokenized URL to clipboard/browser, with tests asserting logged output never contains `#token=`; status needs the same token-free output or an explicit safe delivery path.
  - confidence: 0.93
- **[P2] Skip UI token resolution when the UI is disabled:** `src/cli/daemon-cli/status.gather.ts:333-337`
  - body: This call runs before the print path checks `gateway.controlUi.enabled`; when Control UI is disabled, status prints `Dashboard: disabled`, so any SecretRef/provider work done here is unused. Guard the resolver behind the same enabled check to avoid latency, provider failures, or external secret resolution on a status path that will not display the URL.
  - confidence: 0.87
- **[P2] Reuse the canonical gateway token resolver:** `src/commands/control-ui-access-url.ts:46-52`
  - body: Current main already centralizes gateway token precedence and SecretRef fallback policy in `resolveGatewayAuthToken`; this helper calls lower-level secret fallback plumbing directly and recreates dashboard auth semantics in a second place. Rebase this helper onto the canonical resolver so status and dashboard stay on one token-resolution contract.
  - confidence: 0.84
- **[P3] Remove the release-owned changelog entry:** `CHANGELOG.md:106`
  - body: `CHANGELOG.md` is release-generation owned for normal OpenClaw PRs. Keep the user-visible release-note context in the PR body or commit message instead of editing the changelog here.
  - confidence: 0.92
