## Summary

Keep this PR open as the canonical fix path for the gateway-status dashboard URL parity gap, but it is not merge-ready: the branch is stale/conflicting against current main, lacks real behavior proof, and would regress current token-log, SSH-hint, TLS, and dashboard-readiness safeguards.

## What This Changes

The branch adds a shared Control UI access URL resolver, wires it into `openclaw gateway status`, redacts token fragments from JSON status output, adds SecretRef hints/tests, refactors `openclaw dashboard`, and updates the changelog.

## Best Possible Solution

Rebuild the status URL parity fix on current dashboard auth/readiness/link helpers so status can offer an authenticated handoff without leaking shared secrets, while preserving TLS, disabled-Control-UI, SecretRef, JSON-redaction, and current dashboard behavior.

## Reproduction Assessment

Yes, by source inspection. Current main `gateway status` prints the bare Control UI URL while `openclaw dashboard` can deliver a tokenized URL under token auth, and the linked issue discussion shows this status output in a failing auth context.

## Solution Assessment

No. A shared resolver is a reasonable direction, but the PR head must be rebuilt on current token-log, TLS, disabled-Control-UI, SecretRef, JSON-redaction, and dashboard-readiness behavior before it is the best mergeable fix.

Full review comments:

- **[P1] Keep token fragments out of dashboard logs:** `src/commands/dashboard.ts:25`
  - body: `dashboardUrl` can contain `#token=...` for plain token auth, and this line writes that URL to runtime logs. Current main deliberately logs only the clean base URL and has regression coverage forbidding both the token value and `#token=` in dashboard log output.
  - confidence: 0.94
- **[P1] Keep tokens out of SSH fallback hints:** `src/commands/dashboard.ts:49-53`
  - body: Passing the token into `formatControlUiSshHint` lets the returned fallback hint include a tokenized URL, and that hint is logged when browser opening fails. Current main deliberately calls the helper without a token for this logged path.
  - confidence: 0.9
- **[P2] Preserve TLS when resolving Control UI links:** `src/commands/control-ui-access-url.ts:39-44`
  - body: The new resolver calls `resolveControlUiLinks` without the gateway TLS flag. TLS-enabled gateways can therefore get `http://` dashboard links even though the current link contract supports `https://` and `wss://`.
  - confidence: 0.9
- **[P2] Restore the dashboard readiness gate:** `src/commands/dashboard.ts:20`
  - body: Current main checks `ensureGatewayReadyForOperation` and returns before copy/open when the gateway is not ready. This branch starts URL resolution and delivery directly, so rebasing it without restoring that gate can copy or open stale dashboard URLs.
  - confidence: 0.86
- **[P2] Skip token resolution when Control UI is disabled:** `src/cli/daemon-cli/status.gather.ts:333-337`
  - body: `gateway status` will not show a dashboard URL when Control UI is disabled, but this code still resolves token and SecretRef access data. Guard the resolver so disabled UI setups avoid unnecessary provider latency or failures for a URL that is not used.
  - confidence: 0.84
