## Summary

Keep this PR open, but it is not merge-ready: the underlying status/dashboard token parity bug is still source-reproducible on current main, while this branch regresses token-log safety, TLS-aware Control UI links, disabled-Control-UI behavior, and still lacks real behavior proof.

## What This Changes

The PR adds a shared Control UI access URL resolver, wires token-aware dashboard URLs into `openclaw gateway status`, redacts JSON status output, adds SecretRef hints/tests, and updates the changelog.

## Best Possible Solution

Rebuild the fix on current `resolveGatewayAuthToken` and `resolveControlUiLinks` paths so status can provide an auth-capable dashboard handoff without logging secrets, while preserving TLS, disabled-Control-UI, SecretRef, JSON redaction, and focused regression coverage.

## Reproduction Assessment

Yes, by source inspection: current main `gateway status` prints only the bare Control UI URL while `dashboard` can deliver a tokenized browser/clipboard URL. The PR-introduced regressions are also source-reproducible from the PR head and current token-log guardrail tests.

## Solution Assessment

No. A shared resolver is a reasonable direction, but this branch is not the best mergeable solution until it preserves token-log safety, TLS-aware links, disabled-Control-UI behavior, SecretRef handling, JSON redaction, and has real behavior proof.

Full review comments:

- **[P1] Keep token fragments out of dashboard logs:** `src/commands/dashboard.ts:25`
  - body: `dashboardUrl` can contain `#token=...` for plain token auth, and this line writes it to runtime logs. Current main logs the clean URL and has regression coverage forbidding both the token value and `#token=` in dashboard log output.
  - confidence: 0.94

  - body: Passing the token into `formatControlUiSshHint` lets the returned fallback hint include a tokenized URL, and that hint is logged when browser opening fails. Current main deliberately calls the helper without a token for this logged path.
  - confidence: 0.9
- **[P2] Preserve TLS-aware Control UI links:** `src/commands/control-ui-access-url.ts:39-44`
  - body: The new resolver calls `resolveControlUiLinks` without the gateway TLS flag. TLS-enabled gateways can therefore get `http://` dashboard links even though current main's link contract supports `https://` and `wss://`.
  - confidence: 0.9
- **[P2] Skip token resolution when Control UI is disabled:** `src/cli/daemon-cli/status.gather.ts:333-337`
  - body: `gateway status` will not show a dashboard URL when Control UI is disabled, but this code still resolves token and SecretRef access data. Guard the resolver so disabled UI setups avoid unnecessary provider latency or failures.
  - confidence: 0.84
