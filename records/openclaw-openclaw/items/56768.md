## Review Findings

Full review comments:

- **[P1] Keep token fragments out of dashboard logs:** `src/commands/dashboard.ts:25`
  - body: Current main intentionally logs only the base Dashboard URL while sending the tokenized URL to clipboard/browser, and the regression test says logged output must never contain `#token=` because console-captured logs can be readable by operator-scoped devices. This branch feeds `access.dashboardUrl` into the Dashboard URL path, so a locally resolved token can be printed instead of only delivered to the browser/clipboard.
  - confidence: 0.91
- **[P1] Pass TLS state into access URL resolution:** `src/commands/control-ui-access-url.ts:38-43`
  - body: `resolveControlUiLinks` chooses `https`/`wss` only when `tlsEnabled` is passed, but the new resolver never forwards `cfg.gateway.tls.enabled`. TLS-enabled gateways would get `http`/`ws` dashboard/status URLs, regressing the current TLS-aware status/dashboard behavior.
  - confidence: 0.88
- **[P2] Skip token resolution when Control UI is disabled:** `src/cli/daemon-cli/status.gather.ts:333-337`
  - body: `status.print` hides the Dashboard URL when Control UI is disabled, but `gatherDaemonStatus` now resolves the access URL unconditionally. For SecretRef-backed tokens this can invoke secret-provider resolution, latency, or failures for a URL the command will not print.
  - confidence: 0.78
