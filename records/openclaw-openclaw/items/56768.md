## Summary

Keep open: the underlying status/dashboard parity bug is still source-reproducible, but this PR is not mergeable as-is because it leaks tokenized dashboard URLs through logged output paths, misses current TLS-aware link handling, resolves token data when Control UI is disabled, and lacks real behavior proof.

## What This Changes

The PR adds a shared Control UI access URL resolver, uses it from dashboard and gateway status to produce token-aware dashboard URLs, redacts JSON status output, adds SecretRef hints/tests, and updates the changelog.

## Best Possible Solution

Rebuild the fix on current resolveGatewayAuthToken and resolveControlUiLinks paths so status can provide an auth-capable dashboard handoff without logging secrets, while preserving TLS, disabled-Control-UI, SecretRef, JSON redaction, and focused regression coverage.

## Reproduction Assessment

Yes. By source inspection, current main gateway status prints only the bare Control UI httpUrl while dashboard can deliver a tokenized browser/clipboard URL; the PR-introduced regressions are also visible from the PR head and current token-log guardrail tests.

## Solution Assessment

No. A shared resolver is a reasonable direction, but this branch should not be merged until it preserves token-log safety, TLS-aware links, disabled-Control-UI behavior, SecretRef handling, JSON redaction, and real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Log only the clean dashboard URL:** `src/commands/dashboard.ts:25`
  - body: Current main deliberately logs the clean Control UI URL and has regression coverage forbidding both the token value and `#token=` in runtime logs. Logging `dashboardUrl` here exposes gateway tokens in terminal output and captured logs when token auth is configured.
  - confidence: 0.94
- **[P1] Keep tokens out of SSH fallback hints:** `src/commands/dashboard.ts:49-53`
  - body: `formatControlUiSshHint` renders `#token=` when a token is supplied, and the returned hint is logged when browser opening fails. Passing the resolved token here reverses the current token-free fallback behavior.
  - confidence: 0.9
- **[P2] Preserve TLS-aware dashboard links:** `src/commands/control-ui-access-url.ts:39-44`
  - body: The shared resolver calls `resolveControlUiLinks` without the gateway TLS flag. TLS-enabled gateways can therefore get `http://` dashboard URLs even though the gateway serves HTTPS/WSS.
  - confidence: 0.9
- **[P2] Skip token resolution when Control UI is disabled:** `src/cli/daemon-cli/status.gather.ts:333-337`
  - body: `gateway status` suppresses the dashboard URL when Control UI is disabled, but this code resolves token and SecretRef access data unconditionally. That can add avoidable provider latency or failures for a URL status will not show.
  - confidence: 0.84
