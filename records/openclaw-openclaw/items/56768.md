## Summary

Keep this PR open because current main still lacks the requested `gateway status` token-aware dashboard URL parity, and the closed linked issue explicitly left this PR as the canonical follow-up. It is not merge-ready: the branch reintroduces token logging paths, loses TLS-aware dashboard links, resolves token data when Control UI is disabled, lacks real behavior proof, and is currently conflicting with main.

## What This Changes

The PR adds a shared Control UI access URL resolver, wires it into `gateway status`, redacts token fragments from JSON output, adds SecretRef hints/tests, and updates the changelog.

## Best Possible Solution

Rework the shared URL path so status can surface an authenticated access path without logging secrets, while preserving TLS-aware links, disabled-dashboard guards, SecretRef guidance, JSON redaction, and focused regression tests.

## Reproduction Assessment

Yes for the PR-introduced regressions by source inspection: the diff logs `access.dashboardUrl`, forwards token data into a logged SSH hint, omits TLS state, and resolves access data without checking whether Control UI is disabled. The original browser flow was not live-reproduced in this read-only review.

## Solution Assessment

No. A shared resolver is a reasonable direction, but this branch is not the best fix until it preserves existing token-log and TLS guardrails and gets maintainer policy approval for any token-bearing human status output.

Full review comments:

- **[P1] Log only the clean dashboard URL:** `src/commands/dashboard.ts:25`
  - body: Current main sends the tokenized dashboard URL only to clipboard/browser and has a regression test forbidding both the token value and `#token=` in logs. Logging `access.dashboardUrl` here can expose config or env gateway tokens in terminal output and captured runtime logs.

  - body: `formatControlUiSshHint` includes a `#token=` URL when a token is passed, and the returned hint is logged when browser launch fails. Passing `access.authToken` here reverses current main's token-free fallback behavior.
  - confidence: 0.9
- **[P2] Pass TLS state into access URL links:** `src/commands/control-ui-access-url.ts:39-44`
  - body: The shared resolver calls `resolveControlUiLinks` without the gateway TLS flag. TLS-enabled gateways can therefore get `http://` dashboard URLs even though the gateway serves HTTPS/WSS.
  - confidence: 0.9
- **[P2] Skip access resolution when Control UI is disabled:** `src/cli/daemon-cli/status.gather.ts:333-337`
  - body: `gateway status` already suppresses the dashboard URL when Control UI is disabled, but this code resolves token and SecretRef access data unconditionally. That can add avoidable provider latency or failures for a URL status should not show.
  - confidence: 0.84
