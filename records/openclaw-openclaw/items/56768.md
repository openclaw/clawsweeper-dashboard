## Summary

Keep open: the requested status/dashboard parity gap is still real on current main, but this PR regresses existing token-log guardrails, misses current TLS link handling, lacks after-fix real CLI proof, and is conflicting with main.

## What This Changes

The PR adds a shared Control UI access URL resolver so `openclaw gateway status` can print a token-aware dashboard URL, redacts JSON status output, adds SecretRef hints/tests, and updates the changelog.

## Best Possible Solution

Refactor onto the current `resolveGatewayAuthToken` and `resolveControlUiLinks` paths so status can expose an auth-capable dashboard path without logging secrets, while preserving TLS, disabled-Control-UI, SecretRef, JSON redaction, and focused tests.

## Reproduction Assessment

Yes, by source inspection: current `gateway status` prints only the bare `links.httpUrl`, while `openclaw dashboard` can deliver a tokenized URL to browser/clipboard. The PR-introduced regressions are also source-reproducible from the PR head and current guardrail tests.

## Solution Assessment

No. A shared resolver is a reasonable direction, but this branch is not the best fix until it preserves token-log safety, TLS-aware links, disabled-Control-UI behavior, and gets maintainer approval for any token-bearing human status output.

  - body: Current main deliberately logs the clean Control UI URL and has regression coverage forbidding both the token value and `#token=` in runtime logs. Logging `access.dashboardUrl` here exposes gateway tokens in terminal output and captured logs when token auth is configured.

  - confidence: 0.9
- **[P2] Pass TLS state into access URL links:** `src/commands/control-ui-access-url.ts:39-44`
  - body: The shared resolver calls `resolveControlUiLinks` without the gateway TLS flag. TLS-enabled gateways can therefore get `http://` dashboard URLs even though the gateway serves HTTPS/WSS.
  - confidence: 0.9
- **[P2] Skip token resolution when Control UI is disabled:** `src/cli/daemon-cli/status.gather.ts:333-337`
  - body: `gateway status` suppresses the dashboard URL when Control UI is disabled, but this code resolves token and SecretRef access data unconditionally. That can add avoidable provider latency or failures for a URL status will not show.
  - confidence: 0.84
