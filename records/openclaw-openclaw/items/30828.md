## Summary

Current main still lacks a configurable preflight model rate limiter; adjacent 429 handling, auth-profile cooldowns, WHAM cooldown probing, model fallback, and webhook fixed-window limiters do not satisfy manual token/request budgets.

## What This Changes

The issue requests user-configurable model throttling by tokens per interval and requests per interval.

## Best Possible Solution

Add a documented core model request budget system that can enforce per-provider/per-model, and likely per-auth-profile, request and token windows before shared model calls and reconcile actual usage after responses/streams.

## Reproduction Assessment

Not applicable as a feature request. Current-main verification is source/docs based: there is no config/schema/docs surface or model-path scheduler that can express and enforce the requested budgets.

## Solution Assessment

Unclear until maintainers settle the config contract and scheduling semantics. A shared model-path scheduler integrated with fallback, streaming, auth-profile cooldowns, and optional provider header feedback is the maintainable direction.
