## Summary

Keep open: current main still has reactive rate-limit cooldown, fallback, and quota-suspension behavior, but it does not expose a configurable token/request interval budget or preflight scheduler.

## What This Changes

The issue asks for manually configurable model throttling by tokens per interval and requests per interval.

## Best Possible Solution

Add a documented shared model-request budget layer that can key limits by provider/model/auth profile, preflight queue or defer requests, and reconcile observed token/request usage after responses or streams.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection gives a high-confidence absence check: current config and docs expose provider connection/context/timeout and reactive cooldown/fallback fields, not token/request interval budgets.

## Solution Assessment

Unclear until maintainers choose the public config contract and scheduling semantics. The maintainable direction is a shared budget/scheduler integrated with model request and failover paths, with optional provider-header feedback rather than a one-provider header-only fix.
