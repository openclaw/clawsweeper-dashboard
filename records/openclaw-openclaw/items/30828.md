## Summary

Current main still does not implement the requested configurable model rate limiter. The codebase has adjacent reactive handling for 429s, Retry-After, auth-profile cooldowns, key/profile rotation, and model fallback, but I found no public config/schema/docs surface or shared preflight scheduler for tokens-per-interval or requests-per-interval budgets.

## What This Changes

The issue requests user-configurable model throttling by tokens per interval and requests per interval.

## Best Possible Solution

Add a documented core model request-path budget system with per-provider/per-model, and likely per-auth-profile, request and token budgets over interval windows; enforce preflight scheduling before shared model calls; reconcile actual usage after responses and streams; and optionally ingest provider headers such as OpenAI x-ratelimit-* as adaptive input.

## Reproduction Assessment

Not applicable as a feature request. The current-main verification is source/docs based: there is no schema, docs entry, or scheduler path that can express and enforce the requested budgets.

## Solution Assessment

Unclear until maintainers settle the public config contract and scheduling semantics. A shared core scheduler integrated with model calls, fallback, streaming, and auth-profile behavior would be more maintainable than OpenAI-header handling alone.
