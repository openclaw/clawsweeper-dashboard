## Summary

Keep this open because current main still derives the `memory status` dreaming line from the deep/short-term promotion resolver, so a light-enabled and deep-disabled setup can still render as `Dreaming: off`. The issue is also paired with open, unmerged PR #67910 using closing syntax, so it should wait for that PR or a focused replacement fix to land.

## What This Changes

Make `openclaw memory status` report active dreaming phase state instead of treating the deep phase as the only source for the `Dreaming` summary.

## Best Possible Solution

Land or replace #67910 with a focused memory-core change that derives the `memory status` summary from the full dreaming config or the light/deep/REM phase resolvers, reports `off` only when no phase is active, and adds a regression test for global dreaming enabled with light enabled and deep disabled.

## Reproduction Assessment

Yes. The source path is high confidence: `memory status` calls `formatDreamingSummary`, which reads the short-term promotion resolver, which delegates to the deep phase resolver; with global dreaming enabled, light enabled, and deep disabled, that path can return `off` without consulting light state.

## Solution Assessment

Yes. A phase-aware status summary is the narrow maintainable fix, and because open PR #67910 already targets this with closing syntax, this issue should remain open until that PR or a replacement merges.
