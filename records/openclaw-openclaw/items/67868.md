## What This Changes

The issue reports that `openclaw memory status` says `Dreaming: off` for a valid memory-core dreaming configuration with light enabled and deep disabled.

## Best Possible Solution

Review, repair, or replace https://github.com/openclaw/openclaw/pull/67910 with a narrow phase-aware status fix that reports `off` only when no effective dreaming phase is active and keeps useful deep-promotion diagnostics covered by regression tests.
