## Summary

Keep open: current main and the latest release still derive the memory `Dreaming` status line from the deep-only promotion config, while https://github.com/openclaw/openclaw/pull/93113 is the open closing PR for the same bug.

## What This Changes

The issue reports that `openclaw memory status` can say `Dreaming: off` for a valid light-only dreaming configuration because the status formatter consults only deep dreaming state.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/93113, or close that PR with a maintainer-backed replacement, then close this issue only after the fix reaches main.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: with global dreaming enabled, light enabled, and deep disabled, `formatDreamingSummary` reads only the deep-derived config and can return `Dreaming: off`.

## Solution Assessment

Yes. The best current path is the narrow linked PR that reuses existing phase resolvers in the memory-core status formatter and adds focused CLI regression coverage, rather than starting a second fix lane.
