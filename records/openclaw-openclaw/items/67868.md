## Summary

Keep this open because current main still derives the `memory status` `Dreaming` line from the deep/short-term promotion resolver, and the linked fix PR #67910 remains open and unmerged.

## What This Changes

`openclaw memory status` should report active light/REM/deep dreaming phases instead of treating a deep-disabled light-only config as fully off.

## Best Possible Solution

Land or replace #67910 with a focused phase-aware status fix that reports `off` only when no dreaming phase is active and adds a regression for global dreaming enabled with light enabled and deep disabled.

## Reproduction Assessment

Yes. The source path is direct: `memory status` calls `formatDreamingSummary`, which reads the short-term promotion resolver, which delegates to the deep phase; with global dreaming enabled, light enabled, and deep disabled, that path can still return `off`.

## Solution Assessment

Yes. A phase-aware summary derived from the full dreaming config or the individual light/deep/REM resolvers is the narrow maintainable fix; the issue should stay open until the linked PR or a replacement lands.
