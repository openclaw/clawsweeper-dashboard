## Summary

Keep this open because current main still renders `memory status` through the deep-derived dreaming resolver, so a valid light-enabled/deep-disabled setup can still be reported as `Dreaming: off`. The issue is also paired with the open, unmerged implementation PR at https://github.com/openclaw/openclaw/pull/67910, which uses closing syntax for this issue.

## What This Changes

The issue reports that `openclaw memory status` can show `Dreaming: off` when global dreaming and the light phase are enabled but deep dreaming is disabled.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/67910 with a focused phase-aware status fix that reports `off` only when no effective dreaming phase is active and adds light-enabled/deep-disabled regression coverage.

## Reproduction Assessment

Yes. Current-main source inspection gives a high-confidence path: `memory status` prints `formatDreamingSummary`, which reads a deep-derived resolver while light dreaming resolves and runs independently.

## Solution Assessment

Yes for the requested direction. A narrow phase-aware status summary with regression coverage is the maintainable fix, while the open linked PR remains the active implementation lane until it is landed, narrowed, replaced, or closed.
