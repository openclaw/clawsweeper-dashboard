## Summary

Keep this open because current main still builds the `memory status` `Dreaming` line from the deep/short-term promotion resolver, so a light-enabled and deep-disabled configuration can still render as `Dreaming: off`. The issue is also paired with open, unmerged PR #67910 using closing syntax, so it should wait for that PR or a focused replacement to land.

## What This Changes

The issue asks for `openclaw memory status` to report light/deep/REM dreaming phase state instead of treating deep dreaming as the only enabled signal.

## Best Possible Solution

Review and either land or replace #67910 with a focused phase-aware status fix that reports `off` only when no dreaming phase is active and adds a light-enabled/deep-disabled regression test.

## Reproduction Assessment

Yes. The source path is high confidence: `memory status` calls `formatDreamingSummary`, which reads the short-term promotion resolver, which delegates to the deep phase resolver without consulting light state.

## Solution Assessment

Yes. A phase-aware status summary is the narrow maintainable fix, and the open linked PR is already the appropriate implementation path until it merges or is replaced.
