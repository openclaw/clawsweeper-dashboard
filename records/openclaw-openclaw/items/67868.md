## Summary

Keep this open because current main and the latest release still render `openclaw memory status` through the deep/short-term promotion resolver, so a light-enabled and deep-disabled config can still show `Dreaming: off`. The issue is also paired with the open, unmerged implementation PR at https://github.com/openclaw/openclaw/pull/67910, which uses closing syntax for this issue.

## What This Changes

The issue asks for `openclaw memory status` to report active light dreaming instead of collapsing light-only configs to `Dreaming: off`.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/67910 with a focused phase-aware memory status fix that reports `off` only when no dreaming phase is active and adds light-enabled/deep-disabled regression coverage.

## Reproduction Assessment

Yes. The current-main source path is high confidence: `memory status` formats `Dreaming` through `formatDreamingSummary`, which reads the deep-derived short-term resolver while light dreaming resolves and runs independently.

## Solution Assessment

Yes. A narrow phase-aware status summary is the maintainable fix; the open linked PR is the right implementation lane until it merges, is narrowed, or is replaced.
