## Summary

Current main still has the reported false-negative path, and an open unmerged implementation PR already targets the issue with closing syntax, so this should remain open until that PR or a replacement lands.

## What This Changes

The issue asks for `openclaw memory status` to report active light/deep/REM dreaming phases instead of showing `Dreaming: off` for a valid light-only dreaming configuration.

## Best Possible Solution

Land or replace #67910 with a focused memory-core change that derives the status summary from the full dreaming config or the light/deep/REM resolvers, reports `off` only when no phase is active, and adds a regression for global dreaming enabled with light enabled and deep disabled.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: with global dreaming enabled, light enabled, and deep disabled, the status formatter reads the deep-derived short-term promotion config and returns `off` without consulting the light phase.

## Solution Assessment

Yes. A phase-aware `memory status` summary is the narrow maintainable fix; the safest path is normal review of #67910 or a replacement with the same behavioral boundary.
