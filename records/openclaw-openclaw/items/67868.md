## Summary

Keep this open because current `main` still formats `openclaw memory status` through the deep/short-term promotion resolver, so a light-enabled and deep-disabled config can still render as `Dreaming: off`. The issue is also paired with the open, unmerged implementation PR at https://github.com/openclaw/openclaw/pull/67910, which uses closing syntax for this issue.

## What This Changes

The issue asks for `openclaw memory status` to report light-only dreaming as enabled instead of collapsing the status to `Dreaming: off`.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/67910 with a focused phase-aware `memory status` fix that reports `off` only when no effective dreaming phase is active and adds light-enabled/deep-disabled regression coverage.

## Reproduction Assessment

Yes. Current-main source inspection gives a high-confidence reproduction path: `memory status` reads a deep-derived resolver while light dreaming resolves and runs independently.

## Solution Assessment

Yes. A narrow phase-aware status summary is the maintainable fix; the open linked PR is the active implementation lane until it is landed, narrowed, replaced, or closed.
