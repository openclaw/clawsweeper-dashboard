## Summary

Keep open: current `main` and the latest release still derive the `memory status` `Dreaming` line from the deep-promotion resolver, so a globally enabled light-only dreaming setup can still render as `off`; the remaining work is now represented by the open fix PR at https://github.com/openclaw/openclaw/pull/92979.

## What This Changes

The issue asks for `openclaw memory status` to report active light dreaming instead of `Dreaming: off` when global dreaming and the light phase are enabled while the deep phase is disabled.

## Best Possible Solution

Land a narrow memory-core CLI formatter and regression-test change that reports light, REM, and deep accurately, preserves useful deep-promotion diagnostics, and returns `off` only when no phase is active.

## Reproduction Assessment

Yes. Source inspection on current `main` shows a light-enabled, deep-disabled config can resolve light dreaming as active while `formatDreamingSummary` still returns `off` from the deep-only resolver.

## Solution Assessment

Yes for the requested direction. The best fix is a narrow status-rendering change in `extensions/memory-core/src/cli.runtime.ts` with focused CLI regression coverage; the open PR should cover REM and preserve existing deep diagnostics before landing.
