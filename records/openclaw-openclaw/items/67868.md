Keep open: current `main` and the latest release still render the memory status `Dreaming` line through the deep-only short-term promotion resolver, so a valid light-enabled, deep-disabled memory-core dreaming config can still report `off`; the prior implementation PRs are now closed unmerged.

## What This Changes

The issue reports that `openclaw memory status` shows `Dreaming: off` for a valid light-enabled, deep-disabled memory-core dreaming configuration.

## Best Possible Solution

Land a narrow memory-core CLI fix that derives the `Dreaming` status line from the full dreaming config or phase resolvers, reports light/deep/REM state accurately, keeps `off` only when no phase is active, and adds focused CLI regression coverage.

## Reproduction Assessment

Yes. Source inspection on current `main` shows a light-enabled, deep-disabled config can resolve light dreaming as active while `formatDreamingSummary` still returns `off` from the deep-only resolver.

## Solution Assessment

Yes. The best fix is a narrow status-rendering change in `extensions/memory-core/src/cli.runtime.ts` with regression coverage, not a config-schema, scheduler, or dreaming execution change.

## Repair Work Prompt

Fix the memory-core CLI status false negative reported in https://github.com/openclaw/openclaw/issues/67868. On current main, `formatDreamingSummary` in `extensions/memory-core/src/cli.runtime.ts` calls the deep-only `resolveShortTermPromotionDreamingConfig`, so global dreaming enabled with light enabled and deep disabled can still print `Dreaming: off`. Implement a narrow status-rendering fix that derives the summary from `resolveMemoryDreamingConfig` or the phase resolvers, reports light/deep/REM state accurately, and only prints `off` when global dreaming is disabled or no phase is effectively active. Preserve or intentionally replace useful deep-promotion diagnostics; do not change dreaming execution, cron registration, config schema, migrations, plugin ownership, or CHANGELOG.md. Add focused regression coverage in `extensions/memory-core/src/cli.test.ts` for global dreaming enabled with light enabled and deep disabled, plus any low-cost off/default case needed to prove no regression.
