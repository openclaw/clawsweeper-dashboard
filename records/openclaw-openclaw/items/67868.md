## Summary

Keep open: current `main` and the latest release still render `openclaw memory status` through the deep-only short-term promotion resolver, so a light-enabled/deep-disabled dreaming config can still report `Dreaming: off`; the open linked PR is the implementation path and should be reviewed or replaced before closing this issue.

## What This Changes

The issue asks for `openclaw memory status` to report active light dreaming instead of collapsing valid light-only dreaming setups to `Dreaming: off`.

## Best Possible Solution

Land a narrow memory-core CLI fix that derives the status line from the overall dreaming config and active phases, with a regression test for global dreaming enabled, light enabled, and deep disabled.

## Reproduction Assessment

Yes. Source inspection shows that a config with global dreaming enabled, light enabled, and deep disabled resolves the deep-only summary to disabled, so `formatDreamingSummary` returns `off` without consulting the active light phase.

## Solution Assessment

Yes. The best fix is a narrow status-rendering change in memory-core that reads the full dreaming config and reports active phase state; the linked open PR targets that direction but has not merged.
