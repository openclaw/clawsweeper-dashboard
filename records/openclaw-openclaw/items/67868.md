## Summary

Keep this open because current main still renders `openclaw memory status` through the deep/short-term promotion resolver, so a light-enabled and deep-disabled config can still appear as `Dreaming: off`. The issue is also paired with the open, unmerged implementation PR at https://github.com/openclaw/openclaw/pull/67910, which uses closing syntax for this issue.

## What This Changes

Request that `openclaw memory status` report active light dreaming instead of `Dreaming: off` when global dreaming is enabled and the deep phase is disabled.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/67910 with a focused phase-aware memory status fix that reports `off` only when no effective dreaming phase is active and adds light-enabled/deep-disabled regression coverage.

## Reproduction Assessment

Yes. The current-main source path is high confidence: `memory status` formats `Dreaming` through `formatDreamingSummary`, which reads the deep-derived short-term resolver while light dreaming resolves and runs independently.

## Solution Assessment

Yes. A narrow phase-aware status summary is the maintainable fix; the linked open PR is the active implementation lane until it merges, is narrowed, or is replaced.
