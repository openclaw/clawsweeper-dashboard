## Summary

Keep open: current main still has the reported source-level auto-scroll bug, but the latest provided PR diff still suppresses every scroll event while the programmatic flag is active, so the prior P2 concern remains. The linked #76966 bug should stay paired with this PR until a corrected guard lands.

## What This Changes

The PR adds a `chatIsProgrammaticScroll` flag around Control UI chat auto-scroll writes, updates focused scroll tests, and adds a changelog fix entry.

## Best Possible Solution

Revise the guard so only OpenClaw’s own scroll write is suppressed while immediate user scroll-away detection and scroll direction state remain intact, then keep the focused regression tests and changelog entry.

## Reproduction Assessment

Yes, source-level: on current main, a `handleChatScroll` event with grown `scrollHeight`, old `scrollTop`, and distance from bottom above 450 px sets `chatUserNearBottom` false with no programmatic-scroll distinction. I did not run tests because this review is read-only.

## Solution Assessment

No; the proposed boolean early return is not yet the best fix because it ignores all scroll events while active. A narrower implementation should suppress only the programmatic scroll event and preserve user-scroll pause behavior.
