## Summary

Keep open: the PR targets a real source-level Control UI streaming-scroll bug, but the latest diff has one scroll bookkeeping regression and still lacks after-fix real browser behavior proof.

## What This Changes

The branch adds a programmatic chat-scroll guard in Control UI, focused app-scroll tests, and a changelog entry for streaming webchat auto-scroll behavior.

## Best Possible Solution

Keep the focused app-scroll guard, preserve scroll-position bookkeeping while suppressing only near-bottom state changes, add the focused regression, and require real streaming browser proof before merge.

## Reproduction Assessment

Yes, source-level. Current main can reproduce the state-corruption path because programmatic scroll writes are followed by handleChatScroll recalculating chatUserNearBottom from every scroll event after streaming content can grow.

## Solution Assessment

No, not as written. The guard belongs in app-scroll and is the right boundary, but the early return should still keep chatLastScrollTop in sync so the documented chat-header scroll direction behavior does not regress.

Overall correctness: patch is incorrect
