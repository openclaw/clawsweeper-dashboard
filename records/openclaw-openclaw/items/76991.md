## Summary

Keep open: this PR targets a valid Control UI streaming-scroll bug, but the current diff still skips scroll-position bookkeeping for guarded events and the external PR still lacks after-fix real browser streaming proof.

## What This Changes

The PR adds a Control UI programmatic chat-scroll guard, focused app-scroll tests, and a changelog entry for webchat streaming auto-scroll.

## Best Possible Solution

Keep the app-scroll guard, update scroll-position bookkeeping before suppressing only near-bottom/new-message mutations, add a regression for the header-control direction case, then provide redacted real Control UI streaming proof.

## Reproduction Assessment

Yes, source-level. Current main's `scheduleChatScroll` writes programmatic bottom scroll positions while `handleChatScroll` treats every scroll event as user state and can flip `chatUserNearBottom` false after streamed content grows.

## Solution Assessment

No, not as written. The guard belongs in `app-scroll`, but returning before updating `chatLastScrollTop` can regress documented header-control scroll direction behavior.
