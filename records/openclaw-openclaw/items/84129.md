## What This Changes

The issue reports that Telegram direct plain-text replies from a Codex-backed agent stop appearing while the agent still records assistant responses and other Telegram paths continue working.

## Best Possible Solution

Restore visible no-config Codex direct Telegram replies through a maintainer-approved default path, while keeping `messages.visibleReplies: "message_tool"` as the documented explicit tool-only override.

## Reproduction Assessment

Yes at source/log level, not from a live current-main run: the bundle shows Telegram inbound turns and Codex assistant transcript entries after restarts, while current main suppresses normal Codex direct final replies unless the message tool sends.

## Solution Assessment

Yes for the existing PR path if maintainers accept the default flip; a separate automatic fix PR is not the best path because https://github.com/openclaw/openclaw/pull/83814 already targets the likely root cause and needs upgrade-behavior review.
