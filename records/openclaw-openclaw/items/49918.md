## Summary

Keep this issue open: current main and the latest release still route TUI `/new` through local session switching only, while the documented hook contract and gateway reset path make `/new` hook-visible. The open linked implementation PR at https://github.com/openclaw/openclaw/pull/49945 still owns the fix path and has not merged.

## What This Changes

The issue reports a regression where TUI `/new` creates an isolated `tui-*` session without emitting hook-visible `command:new` behavior.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/49945 with a narrow TUI fix that preserves isolated `tui-*` sessions while emitting `/new` hooks against the session being left.

## Reproduction Assessment

Yes, source-level with high confidence. Current main’s TUI `/new` only switches local session state, while docs and reset-service code show `command:new` hooks are emitted through the gateway reset path.

## Solution Assessment

Yes, keeping this issue open is the right current outcome. The maintainable fix should restore hook observability without losing isolated TUI sessions or broadening reset RPC behavior unnecessarily.
