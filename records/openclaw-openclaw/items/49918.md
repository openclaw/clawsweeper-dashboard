Keep this issue open: current main and v2026.5.12 still route TUI `/new` through local session switching only, while the docs and gateway reset path make `/new` hook-visible. ## What This Changes

The issue reports that TUI `/new` switched to a local `setSession(tui-<uuid>)` path and no longer emits the documented `command:new` hook-visible new-session behavior.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/49945 with a narrow fix that preserves isolated `tui-*` sessions and restores hook emission against the session being left, without broadening the public reset RPC unnecessarily.
