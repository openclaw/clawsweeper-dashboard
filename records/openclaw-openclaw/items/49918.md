## What This Changes

The issue reports that TUI `/new` switches to an isolated local `tui-*` session without emitting the hook-visible `command:new` behavior that previously fired through the gateway reset path.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/49945 with a narrow fix that preserves isolated TUI sessions and emits `command:new` hooks for the session being left before switching.
