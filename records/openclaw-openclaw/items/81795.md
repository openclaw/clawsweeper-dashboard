## What This Changes

The PR adds resolved thinking metadata to gateway agent rows, renders it in the Agents overview, and prevents chat and /think fallbacks from applying global thinking defaults to sessions on different models.

## Best Possible Solution

Land this focused PR after normal CI and maintainer checks so Control UI uses gateway-resolved agent/session thinking metadata as the source of truth.

## Reproduction Assessment

Yes, with source-level confidence. Current main omits thinking metadata from agents.list rows, and the chat plus /think fallback paths can return defaults.thinkingDefault before resolving the active session model.

## Solution Assessment

Yes. Exposing per-agent resolved thinking fields from the gateway and guarding global-default fallback by model match is the narrow maintainable fix for the reported UI behavior.
