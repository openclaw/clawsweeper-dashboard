The PR adds resolved thinking metadata to gateway agent rows, renders Thinking Default in the Agents overview, and changes chat and /think fallbacks to avoid global defaults when the active session model differs.

## Solution Assessment

Yes. Exposing gateway-resolved per-agent thinking fields and guarding global-default fallbacks by model match is the narrow maintainable fix; changing the global defaults path would be broader and less accurate.
