## Summary

Keep open: the linked Control UI bug is valid and the PR is directionally right, but it is not merge-ready because it lacks real behavior proof and misses the gateway protocol schema update for the new `agents.list` response fields.

## What This Changes

The PR adds per-agent thinking metadata to `agents.list`, displays the resolved thinking default in the Agents overview, and changes chat `/think` fallback logic so sessions on a different model do not inherit global thinking defaults.

## Best Possible Solution

Add the new agent thinking fields to the gateway protocol schema/contract tests, then land the narrow gateway/UI fix after contributor-supplied live Control UI or gateway proof confirms the corrected default display.

## Reproduction Assessment

Yes from source inspection: current main omits per-agent thinking metadata from `agents.list`, and chat `/think` falls back to global defaults when the session row lacks a thinking default. I did not execute a live UI repro in this read-only pass.

## Solution Assessment

Mostly yes, but not as-is. Exposing resolved agent thinking metadata at the gateway boundary is the narrow maintainable direction, but the protocol schema must be updated and real behavior proof must be added before merge.

## Review Findings

Overall correctness: patch is incorrect
