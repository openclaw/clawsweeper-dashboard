The PR adds per-agent thinking metadata to gateway `agents.list` rows, displays the thinking default in the Agents overview, and changes chat and `/think` fallback logic plus tests to avoid cross-model global defaults.

## Best Possible Solution

Yes, source-reproducible. Current main omits per-agent thinking metadata from `agents.list` and lets chat plus `/think` fall back to global thinking defaults when the active session can be on another model.

## Solution Assessment

Mostly yes, but incomplete. Gateway-resolved per-agent thinking metadata plus model-match fallback guards are the right fix, but the new `agents.list` fields need to be added to the Gateway protocol schema before merge.
