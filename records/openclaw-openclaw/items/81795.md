## Summary

## What This Changes

The PR adds resolved thinking level/default metadata to `agents.list`, renders the selected agent’s thinking default in Control UI, and prevents chat and `/think` fallbacks from applying global defaults to sessions on a different model.

## Best Possible Solution

Update the strict gateway protocol schema alongside the shared row type, then land the focused gateway/UI fix with the existing tests and live Control UI proof.

## Reproduction Assessment

Yes, source-level. Current main omits thinking metadata from `agents.list` and has chat plus `/think` fallback paths that can use global defaults for a different model; I did not run a live current-main reproduction in this read-only sweep.

## Solution Assessment

Not yet. The runtime/UI boundary is the right place to solve the bug, but the patch should also update the gateway protocol schema so the new `agents.list` response remains a valid documented contract.
