## Summary

Keep open. Current main and v2026.5.7 still validate and expose `ui.seamColor` on backend config paths, but the Control UI bootstrap contract, HTTP payload, frontend loader, and theme application path still do not carry or apply that value; the only linked implementation PR is closed unmerged.

## What This Changes

Make `ui.seamColor` from `openclaw.json` affect Control UI accent styling instead of being ignored.

## Best Possible Solution

Wire `ui.seamColor` through the authenticated Control UI bootstrap payload, validate/normalize it in the browser, derive related accent CSS variables only when configured, and add focused regression coverage plus docs/changelog alignment.

## Reproduction Assessment

Yes. The issue gives concrete config-and-restart steps, and source inspection on current main confirms the backend accepts/exposes the value while the browser Control UI path never receives or applies it.

## Solution Assessment

Yes. The narrowest maintainable fix is to use the existing authenticated bootstrap endpoint and frontend theme-token application path; adding a browser `talk.config` dependency is unnecessary for initial theme state.
