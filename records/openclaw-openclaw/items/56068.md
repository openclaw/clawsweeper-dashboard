## What This Changes

The issue requests that the Control UI apply the existing `ui.seamColor` config setting to frontend accent, focus, button, and sidebar styling.

## Best Possible Solution

Wire `ui.seamColor` through the authenticated Control UI bootstrap payload and apply validated, scoped accent CSS variables only when configured, while preserving current default theme behavior when unset.

## Reproduction Assessment

Yes, at source level. Current main accepts and exposes `ui.seamColor` through backend config paths, but the Control UI bootstrap contract, endpoint payload, frontend state, and theme-token path have no field or application logic for it.

## Solution Assessment

Yes. The narrow maintainable fix is to extend the existing authenticated Control UI bootstrap and theme-token application path rather than making the browser depend on a separate `talk.config` fetch for initial theme state.
