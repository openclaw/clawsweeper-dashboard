## Summary

Keep open. Current main still accepts and exposes `ui.seamColor` on backend config paths, but the Control UI bootstrap contract, bootstrap response, frontend bootstrap state, and theme token application path still do not carry or apply that value; the only linked fixing PR was closed unmerged.

## What This Changes

Make `ui.seamColor` from `openclaw.json` affect Control UI accent styling instead of leaving the frontend on hardcoded accent tokens.

## Best Possible Solution

Wire `ui.seamColor` through the authenticated Control UI bootstrap payload, validate and normalize it in the browser, derive the related accent CSS variables only when configured, and update focused regression coverage plus docs/changelog.

## Reproduction Assessment

Yes, at source level. The issue gives concrete config-and-restart steps, and current main shows the backend accepts/exposes the value while the browser bootstrap path never receives or applies it.

## Solution Assessment

Yes. The narrowest maintainable fix is to use the existing authenticated Control UI bootstrap endpoint and frontend theme-token application path; adding a browser dependency on `talk.config` is not needed for initial theme state.
