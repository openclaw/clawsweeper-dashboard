Wire the existing `ui.seamColor` config value into the Control UI frontend so web UI accent CSS variables reflect the configured color.

## Best Possible Solution

Add `seamColor` to the existing Control UI bootstrap contract and payload, then apply and remove validated accent CSS variables in the browser loader with focused regression coverage.

## Solution Assessment

Yes. The narrow maintainable fix is to wire the existing field through the existing Control UI bootstrap path and CSS variables; adding a separate frontend `talk.config` fetch would duplicate config plumbing.
