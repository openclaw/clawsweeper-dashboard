Keep open. ## What This Changes

The issue asks for the existing `ui.seamColor` configuration setting to affect Control UI accent styling instead of being ignored by the frontend.

## Best Possible Solution

Wire `ui.seamColor` through the authenticated Control UI bootstrap payload and apply validated, scoped accent CSS variables only when configured, while preserving current default theme behavior when unset.

## Reproduction Assessment

Yes, at source level. The current backend validates and exposes `ui.seamColor`, but the Control UI bootstrap contract, endpoint payload, and frontend bootstrap/theme path have no field or application logic for it.

## Solution Assessment

Yes. The narrow maintainable fix is to extend the existing authenticated Control UI bootstrap and theme-token application path rather than making the browser depend on `talk.config` for initial theme state.
