## Summary

Keep open. Current main still validates and exposes `ui.seamColor` on backend config paths, but the Control UI bootstrap contract, gateway bootstrap response, frontend bootstrap state, and theme application path do not carry or apply the value; the only linked fixing PR is closed unmerged.

## What This Changes

The issue asks for `ui.seamColor` from `openclaw.json` to update the Control UI accent color instead of leaving frontend accent tokens hardcoded.

## Best Possible Solution

Wire `ui.seamColor` through the authenticated Control UI bootstrap payload, validate and normalize it in the browser, apply derived accent CSS variables only when configured, and align tests/docs/changelog with that contract.

## Reproduction Assessment

Yes, at source level. The reported config-and-restart path maps to current main: backend config accepts/exposes the value, but the browser bootstrap path never receives or applies it.

## Solution Assessment

Yes. The narrowest maintainable fix is to use the existing authenticated Control UI bootstrap endpoint and theme-token application path; adding a browser dependency on `talk.config` is unnecessary for initial theme state.
