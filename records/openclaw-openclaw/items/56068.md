## Summary

Keep open. Current main still accepts and exposes `ui.seamColor` on backend config paths, but the Control UI bootstrap contract, endpoint payload, frontend bootstrap state, and theme application path still do not carry or apply it; the linked fixing PR is closed unmerged.

## What This Changes

The issue asks for `ui.seamColor` in `openclaw.json` to update Control UI accent, focus, sidebar, and button colors instead of being ignored by the frontend.

## Best Possible Solution

Wire `ui.seamColor` through the authenticated Control UI bootstrap payload, validate and normalize it in the browser, apply derived accent CSS variables only when configured, and update tests/docs/changelog for that contract.

## Reproduction Assessment

Yes, at source level. Adding `ui.seamColor` reaches validation and `talk.config`, but the authenticated Control UI bootstrap and theme application path has no field or CSS-variable update, so the browser cannot observe it.

## Solution Assessment

Yes. The narrowest maintainable fix is to use the existing authenticated Control UI bootstrap endpoint and theme-token application path; adding a browser dependency on `talk.config` is unnecessary for initial theme state.
