## Summary

Keep open. Current main still validates and exposes `ui.seamColor` in backend config paths, but the Control UI bootstrap contract, endpoint payload, frontend bootstrap state, and theme application path still omit it; the only linked fixing PR is closed unmerged.

## What This Changes

The issue requests that the existing `ui.seamColor` config setting change Control UI accent, focus, sidebar, and button colors instead of being silently ignored.

## Best Possible Solution

Wire `ui.seamColor` through the authenticated Control UI bootstrap payload, apply validated derived accent CSS variables only when configured, and preserve the current default theme behavior when it is unset.

## Reproduction Assessment

Yes, at source level. Adding `ui.seamColor` reaches validation and `talk.config`, but current main has no field in the authenticated Control UI bootstrap contract or frontend CSS application path, so the browser cannot observe it through the shipped Control UI bootstrap flow.

## Solution Assessment

Yes. The narrow maintainable fix is to extend the existing authenticated Control UI bootstrap and theme-token application path rather than making the browser depend on `talk.config` for initial theme state.
