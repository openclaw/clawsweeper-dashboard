## Summary

Keep open. Current main has partial Anthropic signed-thinking safeguards, including stale/invalid signature cleanup after compaction, but the reported Claude Sonnet 4.6/Opus 4.8 long-session hard-failure path is not proven fixed because current replay policy still intentionally preserves Claude 4-family signed thinking and the default pre-prompt estimate does not visibly account for `thinkingSignature` bytes.

## What This Changes

The issue reports that `thinking:adaptive` and `thinking:high` Anthropic sessions accumulate signed thinking blocks until sessions hard-fail and require manual transcript deletion.

## Best Possible Solution

Keep Claude 4.x provider-required signed thinking replay intact, but make the default compaction, precheck, and recovery paths account for signature-block pressure so long sessions degrade without manual transcript deletion.

## Reproduction Assessment

No live long-session repro was run. Source inspection gives a medium-confidence path: Claude 4.x signatures are persisted and replayed, cleanup is conditional, and the default pre-prompt estimate counts thinking text but not known `thinkingSignature` bytes.

## Solution Assessment

Unclear as a complete current-main fix: the stale-signature stripping on main is the right class of repair for invalid-signature replay, but it does not prove the accumulation and context-saturation failure is fixed. The best fix should live in provider replay and compaction accounting, not in globally dropping signed thinking.
