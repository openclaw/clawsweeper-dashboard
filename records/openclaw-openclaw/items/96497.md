## Summary

Keep open: current main and v2026.6.10 still have the JSON-to-SSE synthesis path that can rewrap an already-SSE `application/json` response as `data: data: ...`, and the direct candidate fix is still open and unmerged.

## What This Changes

OpenAI-compatible streaming endpoints that return standard SSE with an `application/json` content type can be double-prefixed before OpenAI SDK parsing, failing the agent run before a reply.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/96503 or an equivalent narrow transport fix lands with proof, preserving true-JSON fallback and avoiding a new config knob.

## Reproduction Assessment

Yes, at source level: a `stream: true` request with an upstream `application/json` response body beginning `data: {...}` can pass through JSON-to-SSE synthesis and become `data: data: {...}`. I did not run a live gateway in this read-only review.

## Solution Assessment

Yes, the narrow best fix is to sniff JSON-labeled streaming bodies and relabel already-SSE responses as `text/event-stream` before synthesis. A new `sse.strictMode` config option is not needed for the core repair.
