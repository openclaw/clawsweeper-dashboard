## What This Changes

The issue requests fixing OpenAI-compatible streaming parsing so standard SSE frames mislabeled as JSON are not wrapped into `data: data: ...` before SDK JSON parsing.

## Best Possible Solution

Land or replace the narrow provider-transport fix that content-sniffs JSON-labeled streaming bodies, relabels true SSE before the JSON wrap fallback, preserves true-JSON fallback behavior, and avoids adding a new strict-mode config option.

## Reproduction Assessment

Yes, there is a high-confidence source-level reproduction path: an `application/json` streaming response whose body starts with `data: ...` reaches the JSON synthesis branch and can be wrapped as `data: data: ...`. I did not run a live gateway reproduction in this read-only review.

## Solution Assessment

Yes, the best fix shape is the linked narrow provider-transport repair: sniff JSON-labeled stream bodies and relabel real SSE before the JSON wrap path. The requested strict-mode config option is not needed for this regression.
