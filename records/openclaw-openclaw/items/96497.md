## What This Changes

Fix a regression where custom OpenAI-compatible `openai-completions` streaming responses can be wrapped as `data: data: ...` and fail OpenAI SDK JSON parsing.

## Best Possible Solution

Review and land the linked transport fix, or an equivalent change, that sniffs JSON-labeled stream bodies before JSON-to-SSE synthesis while preserving true JSON fallback behavior.

## Reproduction Assessment

Yes. A `stream: true` request receiving an `application/json` body that is already SSE will skip relabeling, then the sanitizer wraps it as `data: data: ...`, which the OpenAI SDK parses as invalid JSON.

## Solution Assessment

Yes. The best fix is the narrow provider transport change to sniff and relabel mislabeled SSE before JSON fallback synthesis, not a new `sse.strictMode` config option.
