## Summary

Keep open: current main and the latest release still contain the JSON-labeled streaming path that can rewrap an already-SSE body as `data: data: ...`, and the linked fix PR is open rather than merged.

## What This Changes

Issue reports an `openai-completions` streaming regression where SSE bodies mislabeled as JSON can be double-prefixed and fail JSON parsing before an agent reply.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/96503, or replace it with an equivalent maintainer fix that sniffs JSON-labeled SSE without adding a new config option.

## Reproduction Assessment

Yes, at source level. Current main can wrap an already-SSE `application/json` streaming body as `data: data: ...`, and the OpenAI SDK then parses `data: {...}` as JSON and throws.

## Solution Assessment

Yes, the narrow best fix is in `src/agents/provider-transport-fetch.ts`: sniff JSON-labeled streamed bodies and relabel genuine SSE before JSON-to-SSE synthesis. A new strict-mode config option would add surface area without solving the core regression better.
