## Summary

Keep open: current `main` and latest release still contain the JSON-labeled streaming fallback that can wrap an already-SSE body as `data: data: ...`; the linked fix PR is open and unmerged, so this issue should remain open until that candidate lands or is replaced.

## What This Changes

The issue reports a regression where OpenAI-compatible SSE responses mislabeled as JSON are rewrapped and fail JSON parsing before an agent reply is produced.

## Best Possible Solution

Land or replace the linked provider-transport fix so JSON-labeled bodies are classified by content before wrapping, preserving the true-JSON fallback while passing already-SSE frames through the SSE parser.

## Reproduction Assessment

Yes. Current `main` routes JSON-labeled streaming responses through the fallback wrapper without checking whether the body is already SSE, matching the reported `data: data: ...` parser failure; I did not run a live custom gateway in this read-only review.

## Solution Assessment

Yes. The best fix is the narrow transport classification repair in the linked candidate PR or an equivalent patch; adding a new `sse.strictMode` config option is not needed for this regression.
