## What This Changes

Issue requests fixing an `openai-completions` streaming regression where already-SSE responses mislabeled as JSON are wrapped as SSE again and fail JSON parsing.

## Best Possible Solution

Land one narrow provider-transport fix, preferably the canonical open PR path, that detects JSON-labeled bodies that are already SSE without adding a new config option.

## Reproduction Assessment

Yes, at source level. Current main wraps any JSON-labeled streaming body as `data: ${data}`, so a standard SSE body beginning with `data: {...}` can become `data: data: {...}` before the SDK parser consumes it.

## Solution Assessment

Yes. The narrow maintainable fix is in `src/agents/provider-transport-fetch.ts`, not a new strict-mode config option; review and land the strongest linked PR or fold its approach into a single canonical fix.
