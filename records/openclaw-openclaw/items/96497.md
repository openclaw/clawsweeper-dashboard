## What This Changes

The issue reports a shipped `openai-completions` streaming regression where JSON-labeled upstream SSE can be rewrapped as `data: data: ...` and fail OpenAI SDK JSON parsing.

## Best Possible Solution

Review and land a narrow provider transport fix like https://github.com/openclaw/openclaw/pull/96503 after proof, preserving true-JSON fallback and avoiding a new strict-mode config option.

## Reproduction Assessment

Yes, at source level. A streaming request that receives an already-SSE body with `application/json` content type can return through the JSON content-type gate and then be wrapped as `data: data: ...`; I did not run a live gateway in this read-only review.

## Solution Assessment

Yes, the best fix is the narrow provider transport normalization already proposed by the linked PR. Sniffing and relabeling mislabeled SSE is more maintainable than adding a new `sse.strictMode` config surface.
