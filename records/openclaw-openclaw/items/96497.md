## What This Changes

The issue reports that OpenAI-compatible streaming responses labeled as JSON can be wrapped into `data: data: ...`, causing JSON parse failures before an agent reply.

## Best Possible Solution

Land the narrow provider-transport fix in https://github.com/openclaw/openclaw/pull/96503 after proof and validation, without adding a new strict-mode config option.

## Reproduction Assessment

Yes at source level. Current main wraps JSON-labeled streaming bodies as SSE, and the issue plus linked PR evidence show that an upstream `data: {...}` body can become `data: data: {...}` before the OpenAI SDK parses it.

## Solution Assessment

Yes, the narrow transport normalization fix is the right shape. It should distinguish true JSON fallback bodies from already-SSE bodies instead of adding a new `sse.strictMode` config option.
