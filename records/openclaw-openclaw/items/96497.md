## What This Changes

The issue reports a shipped `openai-completions` regression where JSON-labeled but already-SSE upstream responses can be wrapped into `data: data: ...` and fail JSON parsing before an agent reply.

## Best Possible Solution

Review and land the canonical narrow provider-transport fix, preserving true JSON fallback behavior and adding focused regression coverage for mislabeled SSE.

## Reproduction Assessment

Yes, at source level: current main still wraps any streaming JSON response body as `data: ${data}`, which turns an already-SSE `data: {...}` body into the reported `data: data: ...` shape.

## Solution Assessment

Yes, the best path is a narrow `src/agents/provider-transport-fetch.ts` fix that distinguishes true JSON from already-SSE bodies; adding a new strict-mode config option is not necessary for the regression.
