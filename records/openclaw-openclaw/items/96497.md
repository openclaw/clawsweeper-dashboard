## Summary

Keep open: current main and the latest release still have a source-reproducible OpenAI-compatible streaming regression for SSE bodies mislabeled as JSON, and the linked fix PR is open but not merged.

## What This Changes

The issue requests fixing `openai-completions` streaming so standard SSE `data:` frames from OpenAI-compatible gateways are not re-prefixed and fed to JSON parsing as `data: {...}`.

## Best Possible Solution

Review and land `https://github.com/openclaw/openclaw/pull/96503` or an equivalent narrow transport fix, without adding a new strict-mode config option, then close this issue after the fix is on main.

## Reproduction Assessment

Yes, at source level. A streamed request that receives an already-SSE body with an `application/json` content type can bypass body sniffing and then be wrapped as `data: data: ...` before the OpenAI SDK parses it.

## Solution Assessment

Yes, the best fix is a narrow transport change that classifies JSON-labeled stream bodies before JSON-to-SSE synthesis. The requested strict-mode config option is not needed for the central bug and would add unnecessary config surface.
