## Summary

Keep open: current main and the shipped v2026.5.20 code still contain the Telegram same-session supersede path that can skip outbound delivery, and the focused linked fix PR is open rather than merged.

## What This Changes

Telegram DM turns sharing the main/webchat session should deliver generated replies back to the originating Telegram DM instead of leaving them transcript-only in webchat.

## Best Possible Solution

Review and land or reject the linked Telegram-owned fix; the end state should ensure each direct Telegram turn either delivers its final reply to the triggering DM or records/surfaces supersession instead of looking delivered only in webchat.

## Reproduction Assessment

Yes, at source level: current main and v2026.5.20 let a newer normal same-session Telegram turn supersede earlier reply work, and superseded dispatch paths return before outbound delivery. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

Unclear until maintainers accept the product semantics: the linked PR is a narrow implementation if direct DMs should deliver every overlapping final reply. If latest-wins is desired, the better fix is to surface/record superseded transcript replies rather than silently leaving them webchat-only.
