## Summary

Keep open: the legacy gateway.env compatibility gap has been repaired and the real CLI proof is sufficient, but the fast-path precheck still misses the config-dir `.env` that current config loading can read from an explicit `OPENCLAW_CONFIG_PATH` directory.

## What This Changes

The branch adds a root-help no-config fast path that probes config and dotenv files before importing config, plus regression coverage for default and legacy gateway.env cases.

## Best Possible Solution

Keep the startup fast path, but derive or mirror the dotenv probes from the same config-dir and state-dir contract as `loadDotEnv` before skipping the config import.

## Reproduction Assessment

Yes, by source inspection: current main can load `dirname(OPENCLAW_CONFIG_PATH)/.env` before deciding whether root help is plugin-sensitive, while the PR precheck does not probe that path. I did not run a runtime repro because this review was read-only.

## Solution Assessment

No. The fast-path shape is maintainable, but it needs to mirror the existing dotenv source contract before returning null and printing precomputed help.
