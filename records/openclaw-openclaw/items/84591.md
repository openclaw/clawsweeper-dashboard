## Summary

Keep open. Current main has a partial reasoning-only recovery path, but the reported DeepSeek/Feishu shape can still slip through when the same attempt already collected visible pre-tool text, so the final thinking-only `stop` turn is not treated as a missing visible answer.

## What This Changes

The issue reports that DeepSeek V4 high-thinking final responses can contain only `reasoning_content`, leaving no channel-visible final answer after a tool-use turn.

## Best Possible Solution

Keep reasoning content internal and fix the embedded-runner terminal-turn recovery so a final signed thinking-only `stop` message is retried or surfaced as an explicit incomplete-turn error even when earlier pre-tool text exists in the same attempt.

## Reproduction Assessment

Yes, at source level. A focused test can construct an OpenAI-compatible attempt with earlier visible `assistantTexts` plus a terminal signed thinking-only `stop` assistant message and observe that current main does not request the reasoning-only continuation.

## Solution Assessment

No, promoting `reasoning_content` directly to visible text is not the best default fix. The safer path is to extend the existing bounded retry/incomplete-turn guard so OpenClaw asks for a visible answer without leaking internal thinking.
