## Summary

Keep open. The report has concrete DeepSeek/Feishu evidence for a thinking-only terminal turn, but current main already has a reasoning-only retry guard, so the remaining work is to explain why that guard did not protect this path without exposing raw reasoning content.

## What This Changes

The issue reports that DeepSeek V4 high-thinking completions can finish with only `reasoning_content`, leaving Feishu users with no visible assistant reply.

## Best Possible Solution

Keep raw thinking hidden by default, make reasoning-only terminal turns retry for visible text when replay-safe, and after retry exhaustion surface a clear user-visible warning instead of counting a blank delivery as success.

## Reproduction Assessment

Partially. The raw session evidence plus source inspection show a thinking-only assistant turn has no visible text, but I did not establish the exact current-main Feishu path that bypasses the existing reasoning-only retry guard.

## Solution Assessment

Unclear. Retrying for visible text is safer than promoting raw `reasoning_content`; maintainers should decide whether the open signed-thinking retry PR covers this path or whether DeepSeek needs its own direct-channel regression and policy hook.
