## Summary

Keep open: the report matches a real non-visible DeepSeek V4 final-turn shape, and current main has generic retry mitigation but not a proven DeepSeek/Feishu fix for the post-tool, reasoning-only final-answer case.

## What This Changes

Handle DeepSeek V4 high-thinking turns where the final answer lands only in `reasoning_content`/thinking and no visible text is delivered to the channel.

## Best Possible Solution

Add a provider-owned DeepSeek V4 handling path that prevents blank final delivery while keeping hidden reasoning conservative: prefer visible-answer retry or thinking-off fallback first, and only promote `reasoning_content` for DeepSeek if maintainers explicitly decide that model output is safe to surface.

## Reproduction Assessment

Yes at source level, but not as a live current-main repro. The issue comment gives a concrete DeepSeek/Feishu session shape, and current source shows signed thinking-only turns are non-visible with only generic retry mitigation, but no live DeepSeek V4 run was performed here.

## Solution Assessment

No, not as the proposed blanket promotion. The safest fix needs a maintainer decision: retry or disable thinking for the continuation by default, or explicitly allow DeepSeek V4 to promote terminal `reasoning_content` only with provider-specific tests and docs.
