## Summary

Keep open: the reporter supplied corrected raw-session evidence for a real DeepSeek V4 high-thinking Feishu DM that ended with reasoning-only content and no visible answer, while current main has generic recovery code that should have caught this class of turn, so the remaining work is to find the gap rather than close it as already fixed.

## What This Changes

DeepSeek V4 high-thinking final turns can place the whole answer in `reasoning_content`/thinking, leaving direct-channel users with no visible reply.

## Best Possible Solution

Fix the direct-chat DeepSeek/openai-compatible reasoning-only path so a final turn with no text either retries for a visible answer or fails visibly, while keeping hidden thinking out of normal channel output.

## Reproduction Assessment

No live end-to-end reproduction was established here. The reporter's sanitized JSONL and current source give a clear source-level path: a DeepSeek V4 `stop` turn with only a signed thinking block has no visible text, and direct chats should not treat that as silent success.

## Solution Assessment

No, the proposed raw promotion of thinking content is not the best default because it risks exposing internal reasoning. The narrower fix is to make the existing reasoning-only continuation/error path cover this DeepSeek direct-channel case and add regression coverage.
