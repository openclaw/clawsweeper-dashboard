## Summary

Close as implemented on current main: the agent runner now treats reasoning-only or empty OpenAI-compatible terminal turns as non-visible output, retries with a visible-answer continuation, and surfaces an explicit error if recovery is exhausted instead of queuing a blank channel reply.

## What This Changes

The issue asks OpenClaw to detect DeepSeek V4 high-thinking stop turns with no visible text and avoid dispatching a blank channel message, preferably by retrying for visible text.

## Best Possible Solution

Keep the current-main non-visible-turn recovery path and ship it in the next release; avoid exposing hidden reasoning content as user-visible fallback text.

## Reproduction Assessment

Yes. The issue includes sanitized Feishu and trajectory logs for a stop turn with only empty thinking content, and current source shows the matching non-visible-turn path; I did not run a live DeepSeek/Feishu repro.

## Solution Assessment

Yes. Retrying or surfacing an explicit visible error before channel dispatch is the safest maintained fix, and it avoids leaking hidden reasoning content into user-visible channels.
