## Summary

This PR is a focused bug fix for the gateway restart continuation race. Source inspection confirms current main can return a coalesced restart after a signal has already emitted without ever running the sentinel write hook, and the PR surfaces that state as explicit queue metadata with targeted scheduler and tool coverage.

## What This Changes

The branch adds `emitHooksQueued` to scheduled restart results, returns `continuationQueued` from the gateway restart tool when a continuation exists, and covers already-emitted coalescing in scheduler/tool tests.

## Best Possible Solution

Land the explicit scheduler/tool result contract if CI stays green, and keep broader continuation merging or auto-resume guarantees separate from this focused fix.

## Reproduction Assessment

Yes. Current-main source shows the already-emitted coalescing branch returns before hook registration, and the PR body supplies after-fix terminal proof for that scheduler/tool path.

## Solution Assessment

Yes. Returning explicit queue state and mapping it to `continuationQueued` is the narrow maintainable fix; merging or replaying continuations would be broader restart/session design work.
