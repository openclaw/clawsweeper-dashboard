## Summary

Current main still has the stale cached-session writeback path, and the branch's focused fix appears valid; keep it open for maintainer review rather than close or replace.

## What This Changes

The PR changes the gateway agent session writeback patch to stop carrying cached override/session fields such as modelOverride and providerOverride into the store merge, and adds regression coverage for stale-cache/fresh-store races.

## Best Possible Solution

Land this focused modelOverride preservation fix, or an equivalent gateway agent-handler change with the same stale-cache/fresh-store regression tests, while keeping the delivery-field follow-up separate in #70575.

## Reproduction Assessment

Yes, source-level. Current main reads a cached session entry, writes modelOverride/providerOverride from that cached object into the patch, then merges that patch over the fresh store row where sessions.patch writes the override.

## Solution Assessment

Yes for the scoped bug. Removing cached override/session writebacks from nextEntryPatch lets mergeSessionEntry preserve the fresh store row, which is the narrowest maintainable fix for the reported modelOverride race.
