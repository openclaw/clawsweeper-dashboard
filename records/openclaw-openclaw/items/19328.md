## Summary

Keep open: current main still has the stale-cache writeback path that can clobber a freshly patched subagent modelOverride, and the proposed patch addresses that path without a clear functional or security blocker.

## What This Changes

The PR narrows gateway agent session writeback to request-derived fields and adds stale-cache/fresh-store regression tests for preserving modelOverride/providerOverride and related session metadata.

## Best Possible Solution

Land this focused change or an equivalent gateway fix that removes stale cached-entry fields from the session writeback, and keep the delivery-field residual tracked separately in #70575.

## Reproduction Assessment

Yes from source inspection, though not by executing tests: sessions.patch can write modelOverride, the agent handler builds nextEntryPatch from a cached entry, and mergeSessionEntry lets that stale patch override the fresh store row.

## Solution Assessment

Yes. Dropping cached-entry carryover from nextEntryPatch is the narrowest maintainable fix for this race, while the remaining delivery-field stale fallback belongs in the separate #70575 follow-up.
