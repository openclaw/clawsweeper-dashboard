## Summary

Keep open. Current main still builds the gateway agent session-store patch from a cached `entry`, includes stale model/provider override values, and merges that patch over the fresh store row; this PR is a focused fix with regression coverage and sufficient terminal proof.

## What This Changes

The PR removes cached `entry` metadata writebacks from the gateway agent session patch and adds regression tests for stale cached entries versus fresh session-store rows.

## Best Possible Solution

Land this focused gateway fix, or an equivalent patch, after maintainer validation while keeping the delivery/channel follow-up in https://github.com/openclaw/openclaw/issues/70575 separate.

## Solution Assessment
