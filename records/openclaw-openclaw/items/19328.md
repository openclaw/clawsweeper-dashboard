## Summary

Keep open. Current main still builds the gateway agent session patch from a pre-lock cached `entry`, including model/provider override fields that can overwrite a fresher store row, while this PR is a focused fix with regression coverage and sufficient terminal proof.

## What This Changes

This PR removes cached session-entry metadata carryovers from the gateway agent writeback patch and adds regression tests for stale-cache versus fresh-store races.

## Best Possible Solution

Land this focused gateway fix, or an equivalent patch, while keeping the residual delivery/channel stale-writeback work in https://github.com/openclaw/openclaw/issues/70575 separate.

## Reproduction Assessment

Yes. Current main has a high-confidence source-level reproduction path: `sessions.patch` writes model/provider overrides, the agent handler builds `nextEntryPatch` from stale `loadSessionEntry` data, and `mergeSessionEntry(store[primaryKey], nextEntryPatch)` lets the stale patch overwrite the fresh store row.

## Solution Assessment

Yes. Dropping cached-entry override and metadata carryovers from the writeback patch is the narrow maintainable fix for the modelOverride race; the lower-priority delivery/channel writeback race is separately tracked in https://github.com/openclaw/openclaw/issues/70575.
