## Summary

Keep open for maintainer review. Current main still has the stale cached-entry writeback pattern that can overwrite a fresh session-store model override, and the PR is a focused implementation candidate rather than obsolete or implemented on main.

## What This Changes

The PR removes cached session-entry metadata from the gateway agent session writeback patch and adds regression tests for stale-cache/fresh-store preservation of model override and related session fields.

## Best Possible Solution

Land this PR or an equivalent focused change that keeps cached override/session metadata out of the agent writeback patch while preserving fresh store values; handle broader stale delivery or lineage writebacks separately unless maintainers decide to expand the scope now.

## Reproduction Assessment

Yes. The high-confidence reproduction path is a deterministic stale-cache/fresh-store unit shape: make `loadSessionEntry` return a cached row without overrides, make `updateSessionStore` receive a fresh row with overrides, and observe current main merging the stale patch over the fresh row.

## Solution Assessment

Yes for the reported modelOverride/providerOverride race. The narrow maintainable fix is to stop putting cached override fields into the writeback patch so `mergeSessionEntry` preserves the fresh store row; broader stale metadata cleanup should stay in a follow-up unless maintainers intentionally broaden this PR.
