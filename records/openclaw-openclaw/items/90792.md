Full review comments:

- **[P1] Resample Nova Sonic audio at the bridge boundary:** `extensions/amazon/nova-sonic/bridge.ts:74-75`
  - body: `createBridge` receives the caller's `audioFormat`, but this bridge always treats input as 8 kHz mu-law and declares it to Nova Sonic as 16 kHz PCM; it also converts Nova Sonic PCM output straight to mu-law without resampling. Voice-call and gateway-relay paths can send or play audio at the wrong rate, so honor the requested format and use the existing realtime audio codec/resampling helpers.
  - confidence: 0.92
- **[P1] Do not stage dependencies from a bundled plugin:** `extensions/amazon/package.json:13-14`
  - body: This bundled plugin asks OpenClaw to stage runtime dependencies even though current dependency docs say bundled plugin manifests must not request dependency staging. Move the AWS SDK dependency tree to an approved external plugin/package shape or get an explicit packaging exception before bundling.
  - confidence: 0.91
- **[P2] Declare Nova Sonic realtime capabilities:** `extensions/amazon/nova-sonic/realtime-voice-provider.ts:42-45`
  - body: `talk.catalog` and clients discover valid realtime transports and audio formats from `provider.capabilities`, but this provider returns none. Add the capability object so browser and native Talk clients can select the Amazon Nova Sonic transport/audio contract safely.
  - confidence: 0.88
- **[P2] Use setup provider env metadata:** `extensions/amazon/openclaw.plugin.json:3-6`
  - body: `providerAuthEnvVars` is deprecated compatibility metadata for new plugins; starting a new AWS provider on that field creates immediate setup/auth migration debt.   - confidence: 0.87

  - confidence: 0.94
