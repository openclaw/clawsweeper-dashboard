## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Resample Nova Sonic audio at the bridge boundary:** `extensions/amazon/nova-sonic/bridge.ts:74-82`
  - body: The bridge always expands incoming chunks as 8 kHz mu-law, then declares them to Nova Sonic as 16 kHz PCM, and always returns mu-law output. Voice-call sends/expects 8 kHz mu-law while gateway relay requests 24 kHz PCM, so this can feed mislabeled audio upstream and play responses in the wrong format. Honor `req.audioFormat` and use the existing codec/resampling helpers at the boundary.
  - confidence: 0.9
- **[P1] Do not stage dependencies from a bundled plugin:** `extensions/amazon/package.json:13-14`
  - body: `openclaw.bundle.stageRuntimeDependencies` is added for a bundled plugin, but dependency-resolution docs say bundled plugin manifests must not request dependency staging. With three AWS SDK runtime dependencies, this needs an external plugin package shape or explicit maintainer packaging exception before bundling.
  - confidence: 0.92
- **[P2] Use setup provider env metadata:** `extensions/amazon/openclaw.plugin.json:3-6`
  - body: `providerAuthEnvVars` is deprecated compatibility metadata; manifest docs say new plugins should put setup/status env metadata on `setup.providers[].envVars`. Starting a new AWS provider on the deprecated surface creates immediate auth/status migration debt.
  - confidence: 0.87
- **[P2] Declare realtime voice capabilities:** `extensions/amazon/nova-sonic/realtime-voice-provider.ts:42-45`
  - body: The provider omits `capabilities`, while current realtime surfaces use those fields to expose transports, audio formats, browser support, barge-in, and tool-call behavior. Without explicit capabilities, clients cannot tell whether Nova Sonic supports gateway relay, browser sessions, and which audio formats are valid.
  - confidence: 0.84
- **[P2] Remove or use the Nova Sonic model list:** `extensions/amazon/nova-sonic/realtime-voice-provider.ts:10`
  - body: `SUPPORTED_MODELS` is declared but never read. `tsconfig.extensions.json` enables `noUnusedLocals` for `extensions/**/*`, so extension typecheck fails until this constant is removed or used for validation.
  - confidence: 0.94
- **[P3] Link the Amazon provider page from docs navigation:** `docs/providers/amazon.md:1`
  - body: The PR adds `docs/providers/amazon.md`, but the docs navigation and provider index still have no Amazon provider entry. Without those links, the public page is orphaned from the provider docs users browse.
  - confidence: 0.86
