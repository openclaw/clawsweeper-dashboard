## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Resample Nova Sonic audio at the bridge boundary:** `extensions/amazon/nova-sonic/bridge.ts:74-82`
  - body: The bridge always expands incoming chunks as 8 kHz mu-law, then declares them to Nova Sonic as 16 kHz PCM; it also converts upstream PCM output directly to mu-law for OpenClaw sinks. Voice-call sends/expects 8 kHz mu-law and the gateway relay requests 24 kHz PCM, so this can feed mislabeled audio upstream and play responses at the wrong rate/pitch. Honor `req.audioFormat` and use the existing resampling helpers before encoding or decoding.
  - confidence: 0.9
- **[P1] Do not stage dependencies from a bundled plugin:** `extensions/amazon/package.json:13-14`
  - body: `openclaw.bundle.stageRuntimeDependencies` is added for a bundled plugin, but current dependency-resolution docs say bundled plugin manifests must not request dependency staging. With three AWS SDK runtime dependencies, this needs an external npm/ClawHub package shape or an explicit maintainer packaging exception before bundling.
  - confidence: 0.92
- **[P2] Use setup provider env metadata for the new plugin:** `extensions/amazon/openclaw.plugin.json:3-6`
  - body: `providerAuthEnvVars` is deprecated compatibility metadata; manifest docs say new plugins should put setup/status env metadata on `setup.providers[].envVars`. Starting a new AWS provider on the deprecated surface creates immediate auth/status migration debt.
  - confidence: 0.87
- **[P2] Remove or use the Nova Sonic model list:** `extensions/amazon/nova-sonic/realtime-voice-provider.ts:10`
  - body: `SUPPORTED_MODELS` is declared but never read. `tsconfig.extensions.json` enables `noUnusedLocals` for `extensions/**/*`, so extension typecheck fails until this constant is removed or used for validation.
  - confidence: 0.94
- **[P3] Add the Amazon provider page to docs navigation:** `docs/providers/amazon.md:1`
  - body: The PR adds `docs/providers/amazon.md`, but the docs navigation and provider index still have no Amazon provider entry. Without those links, the public page is orphaned from the provider docs users browse.
  - confidence: 0.86
