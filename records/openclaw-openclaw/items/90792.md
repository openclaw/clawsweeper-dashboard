## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Honor the requested realtime audio format:** `extensions/amazon/nova-sonic/bridge.ts:74-82`
  - body: The bridge treats every inbound chunk as 8 kHz mu-law and emits mu-law output, but the gateway relay creates bridge sessions with 24 kHz PCM audio. This can feed Nova Sonic mislabeled input and send relay clients audio in the wrong format, so the bridge needs to honor `req.audioFormat` and resample/convert at the boundary.
  - confidence: 0.91
- **[P1] Remove bundled dependency staging:** `extensions/amazon/package.json:13-14`
  - body: This new bundled plugin requests `openclaw.bundle.stageRuntimeDependencies`, but current packaging checks keep bundled plugins out of separate dependency-staging graphs. Use the supported external plugin package path or get explicit maintainer approval for a packaging model change before bundling.
  - confidence: 0.9
- **[P2] Move auth env metadata to setup providers:** `extensions/amazon/openclaw.plugin.json:3-6`
  - body: `providerAuthEnvVars` is deprecated compatibility metadata, and the manifest docs direct new plugins to `setup.providers[].envVars`. Starting a new AWS provider on the deprecated field creates immediate setup/status migration debt.
  - confidence: 0.87
- **[P2] Declare realtime voice capabilities:** `extensions/amazon/nova-sonic/realtime-voice-provider.ts:42-45`
  - body: The Nova Sonic provider registers without `capabilities`, so `talk.catalog` cannot report its supported transport, input/output audio formats, browser-session support, barge-in, or tool-call behavior. Add capabilities that match the bridge contract so clients do not have to guess.
  - confidence: 0.84
- **[P2] Remove or use the Nova Sonic model list:** `extensions/amazon/nova-sonic/realtime-voice-provider.ts:9-10`
  - body: `SUPPORTED_MODELS` is declared and never read, while the extension TypeScript config enables `noUnusedLocals`. Either validate config against it or remove it so the extension typecheck lane can pass.
  - confidence: 0.94
- **[P3] Link the provider docs into navigation:** `docs/providers/amazon.md:1`
  - body: The PR adds a provider page but does not add it to the provider navigation, so the public docs page is orphaned from the provider list users browse.
  - confidence: 0.83
- **[P3] Remove the release-owned changelog edit:** `CHANGELOG.md:7-9`
  - body: Normal PRs should not edit `CHANGELOG.md`; release generation owns that file. Keep release-note context in the PR body or final commit message instead.
  - confidence: 0.86
