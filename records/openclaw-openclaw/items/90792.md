Full review comments:

- **[P1] Honor the requested realtime audio format:** `extensions/amazon/nova-sonic/bridge.ts:76-77`
  - body: `createRealtimeVoiceBridgeSession` passes the selected `audioFormat` into the provider, and gateway relay/Discord use PCM16 24 kHz, but this bridge still decodes every input chunk as 8 kHz mu-law and emits mu-law output. That corrupts gateway-relay and Discord voice sessions whose browser contract advertises PCM16 24 kHz, so branch on `req.audioFormat` and return the matching sink format.
  - confidence: 0.93
- **[P1] Do not stage dependencies from a bundled plugin:** `extensions/amazon/package.json:13-14`
  - body: Current dependency docs say bundled plugin manifests must not request dependency staging; this new bundled plugin sets `openclaw.bundle.stageRuntimeDependencies`, so packaged installs would depend on a path the repo is retiring. Move this to an approved external plugin/package shape or get an explicit packaging exception before bundling.
  - confidence: 0.91
- **[P2] Declare Nova Sonic realtime capabilities:** `extensions/amazon/nova-sonic/realtime-voice-provider.ts:48-51`
  - body: `talk.catalog` and clients discover transports, audio formats, and feature flags from `provider.capabilities`, but this provider returns none while exposing `createBridge`. Add explicit capabilities for supported transports, audio formats, barge-in, and tool calls so clients do not select an undefined contract.
  - confidence: 0.88
