## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Resample Nova Sonic audio at the bridge boundary:** `extensions/amazon/nova-sonic/bridge.ts:73-75`
  - body: `createBridge` receives the caller's `audioFormat`, and gateway relay sends 24 kHz PCM, but this bridge always decodes input as 8 kHz mu-law and declares it to AWS as 16 kHz PCM; it also returns AWS 24 kHz PCM as mu-law without resampling. Honor the requested format and convert/resample at the boundary so voice sessions do not send or play audio at the wrong rate.
  - confidence: 0.92
- **[P1] Do not stage dependencies from a bundled plugin:** `extensions/amazon/package.json:13-14`
  - body: This bundled plugin asks OpenClaw to stage runtime dependencies even though current dependency docs say bundled plugin manifests must not request dependency staging. Move the AWS SDK dependency tree to an approved external plugin/package shape or get an explicit packaging exception before bundling.
  - confidence: 0.91
- **[P2] Declare Nova Sonic realtime capabilities:** `extensions/amazon/nova-sonic/realtime-voice-provider.ts:41-43`
  - body: `talk.catalog` and clients discover valid realtime transports and audio formats from `provider.capabilities`, but this provider returns none. Add the capability object so browser and native Talk clients can select the Amazon Nova Sonic transport and audio contract safely.
  - confidence: 0.88
- **[P2] Use setup provider env metadata:** `extensions/amazon/openclaw.plugin.json:3-6`
  - body: `providerAuthEnvVars` is deprecated compatibility metadata for new plugins; starting a new AWS provider on that field creates immediate setup/auth migration debt. Put the AWS credential environment variables under `setup.providers[].envVars` so setup and status surfaces use the canonical manifest path.
  - confidence: 0.87
- **[P2] Remove or validate the Nova Sonic model list:** `extensions/amazon/nova-sonic/realtime-voice-provider.ts:9-10`
  - body: `SUPPORTED_MODELS` is declared but never used, and `tsconfig.extensions.json` enables `noUnusedLocals` for extension sources. Either use it to validate config or remove it so the extension typecheck lane can pass.
  - confidence: 0.94
