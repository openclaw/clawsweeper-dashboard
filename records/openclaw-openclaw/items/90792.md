## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Resample Nova Sonic audio at the bridge boundary:** `extensions/amazon/nova-sonic/bridge.ts:74-82`
  - body: `sendAudio` always treats incoming chunks as 8 kHz mu-law and output always converts 24 kHz PCM directly to mu-law, but callers pass an explicit `audioFormat`; gateway relay requests 24 kHz PCM and telephony uses 8 kHz mu-law. Honor `req.audioFormat` and resample/encode at the bridge boundary so voice-call and relay audio are not mislabeled or played at the wrong rate.
  - confidence: 0.91
- **[P1] Emit Nova Sonic tool results with the TOOL event shape:** `extensions/amazon/nova-sonic/bridge.ts:123-137`
  - body: `submitToolResult` starts a `TEXT` content block and sends the JSON through `textInput`, but Nova Sonic tool results use `type: "TOOL"` with `toolResultInputConfiguration` and a `toolResult` event. As written, tool-call voice sessions can fail when the agent returns a tool result. ([docs.aws.amazon.com](https://docs.aws.amazon.com/nova/latest/nova2-userguide/sonic-input-events.html))
  - confidence: 0.88
- **[P1] Parse Nova Sonic tool-use events with AWS field names:** `extensions/amazon/nova-sonic/bridge.ts:527-534`
  - body: The AWS output event uses `toolName`, `toolUseId`, and `content`, but this parser reads `name` and `input`, so valid tool-use events have no name and are dropped before OpenClaw can execute the tool. ([docs.aws.amazon.com](https://docs.aws.amazon.com/nova/latest/nova2-userguide/sonic-output-events.html))
  - confidence: 0.9
- **[P1] Do not stage dependencies from a bundled plugin:** `extensions/amazon/package.json:13-14`
  - body: `openclaw.bundle.stageRuntimeDependencies` is added for a bundled plugin, but current dependency-resolution docs say bundled plugin manifests must not request dependency staging. With three AWS SDK runtime dependencies, this needs an external plugin/package shape or an explicit maintainer packaging exception before bundling.
  - confidence: 0.93
- **[P2] Use setup provider env metadata for the new plugin:** `extensions/amazon/openclaw.plugin.json:3-6`
  - body: `providerAuthEnvVars` is deprecated compatibility metadata; manifest docs say new plugins should put setup/status env metadata on `setup.providers[].envVars`. Starting a new AWS provider only on the deprecated surface creates immediate auth/status migration debt.
  - confidence: 0.88
- **[P2] Remove or use the Nova Sonic model list:** `extensions/amazon/nova-sonic/realtime-voice-provider.ts:10`
  - body: `SUPPORTED_MODELS` is declared but never read, and `tsconfig.extensions.json` enables `noUnusedLocals` for extension code. The extension typecheck will fail until this constant is removed or used for validation.
  - confidence: 0.94
- **[P3] Add the Amazon provider page to docs navigation:** `docs/providers/amazon.md:1`
