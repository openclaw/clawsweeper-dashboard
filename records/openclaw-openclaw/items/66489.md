## Summary

Keep open. Current main still does not classify the reported local-media allowlist rejection as permanent, and there is an open linked fix PR that directly targets this issue, so the right cleanup path is to review and land or close that PR first.

## What This Changes

Classify deterministic outbound delivery failures such as `Local media path is not under an allowed directory` and overlong Telegram messages as permanent delivery errors instead of retrying them.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80929 if the narrow classifier and regression-test change is acceptable, while keeping broader enqueue validation, queue metadata, idempotency, and Telegram splitting as separate work.

## Reproduction Assessment

Yes. By source inspection, a relative Telegram media path can reach `loadWebMedia`, throw the current local-media allowlist error, miss `isPermanentDeliveryError`, and remain in the retry path.

## Solution Assessment

Yes. The narrow maintainable fix is to add precise permanent-error classifier coverage and regression tests; the open linked PR is the current implementation path and the issue should stay open until that PR is resolved.
