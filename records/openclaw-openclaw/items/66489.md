## Summary

Keep open. Current main and the latest release still miss the requested permanent-error classification, so the reported local-media allowlist rejection continues through retry/backoff; the related broader issue and PR are closed without a merged fix.

## What This Changes

Request to classify local-media allowlist rejections and related deterministic payload errors as permanent delivery failures so queued deliveries move to failed/ without burning retry budget.

## Best Possible Solution

Extend the existing delivery-queue permanent-error classifier with precise deterministic rejection patterns and focused policy/recovery tests, leaving enqueue-time validation and new queue metadata fields as separate follow-ups.

## Reproduction Assessment

Yes, by source inspection. A queued Telegram payload with a relative `mediaUrls` entry reaches `loadWebMedia` and `assertLocalMediaAllowed`, which throws the reported message; current recovery classification misses that message and routes it through retry/backoff.

## Solution Assessment

Yes. The narrow maintainable fix is to add precise patterns to the existing permanent-error classifier with regression coverage; earlier enqueue validation and a `lastErrorClass` field are broader than the retry-budget bug.
