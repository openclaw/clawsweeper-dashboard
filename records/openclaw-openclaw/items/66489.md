## Summary

Keep open. Current main still does not classify the reported local-media allowlist rejection or Telegram overlong-message rejection as permanent delivery errors, and an open PR with closing syntax already implements the narrow classifier and regression-test fix.

## What This Changes

Classify deterministic delivery failures for disallowed local media paths, and related overlong Telegram messages, as permanent delivery errors instead of retrying them through the delivery queue.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/80929 for the narrow classifier/test fix, while tracking broader deterministic-provider errors separately.

## Reproduction Assessment

Yes. Source inspection on current main shows Telegram media delivery can throw the current local-media allowlist error, that error misses `isPermanentDeliveryError`, and the recovery path then remains on the retry/backoff path.

## Solution Assessment

Yes. Adding precise permanent-error classifier patterns plus regression coverage is the narrowest maintainable fix for this issue; broader enqueue validation and additional provider-specific permanent errors should stay separate.
