## Summary

Keep open. Current main still has the exact local-media allowlist rejection, but the delivery-queue permanent-error classifier does not match it or the overlong Telegram message string, so these failures still consume retry/backoff instead of moving directly to failed/.

## What This Changes

Classify deterministic delivery failures such as local-media allowlist rejection and overlong Telegram messages as permanent queue errors.

## Best Possible Solution

Add focused permanent-error classification and regression coverage for the local-media allowlist rejection and overlong-message case, leaving enqueue-time validation and new queue metadata for separate follow-ups.

## Reproduction Assessment

Yes. A queued Telegram payload with a relative mediaUrls entry reaches loadWebMedia and assertLocalMediaAllowed, which throws the reported message; the current classifier then misses that message and routes recovery through retry/backoff.

## Solution Assessment

Yes. Extending the existing classifier with targeted tests is the narrowest maintainable fix for the retry-budget bug; synchronous enqueue validation would be useful but is a broader product change.
