## Summary

Keep open. Current main still treats the reported local-media allowlist rejection and overlong-message rejection as retryable, and the linked open PR with closing syntax is the active implementation candidate rather than a reason to close this issue before merge.

## What This Changes

The issue requests classifying deterministic delivery failures for disallowed local media paths and overlong Telegram messages as permanent delivery errors.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/80929 for the focused classifier and regression-test fix, while tracking broader deterministic provider failures separately.

## Reproduction Assessment

Yes, by source inspection. A Telegram delivery with a relative local media path can reach the local-media allowlist error, the current classifier does not match that string or `message is too long`, and recovery leaves nonmatches on the retry path.

## Solution Assessment

Yes. Adding precise permanent-error classifier patterns with regression coverage is the narrowest maintainable fix for this issue; enqueue-time validation and broader provider-specific errors should remain separate follow-up work.
