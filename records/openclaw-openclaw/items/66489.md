## Summary

Keep open. Current main still misses the requested permanent-error classification, so the local-media allowlist rejection continues into the retry/backoff path; the related implementation PR is closed unmerged, and the broader open queue issue does not preserve this local-media string tightly enough to close this issue as a duplicate.

## What This Changes

The issue requests classifying deterministic delivery failures such as local media allowlist rejection and overlong Telegram messages as permanent delivery errors instead of consuming retry backoff.

## Best Possible Solution

Extend the existing delivery permanent-error classifier, and preferably the first-send cleanup path that writes retry state, with precise deterministic rejection handling plus focused regression tests; leave enqueue-time validation, new `lastErrorClass` metadata, queue idempotency, and Telegram splitting to separate work.
