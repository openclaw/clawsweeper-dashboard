## Summary

Keep open: current main still has the unfiltered bundled channel setup enumeration described in the report, and the latest release contains the same behavior. The exact Windows `openclaw doctor` warning path still needs an executable repro, but the source path shows disabled config is not available before broad setup-entry loading.

## What This Changes

Disabled bundled channel/plugin setup entries should be skipped before missing generated bundled channel modules are loaded and warned about.

## Best Possible Solution

Add a config-aware setup enumeration path that filters disabled plugin/channel entries before loading generated setup modules, while preserving unconfigured onboarding discovery for channels the user can choose to enable.

## Reproduction Assessment

Yes for a source-level repro: current main enumerates all bundled setup ids without config and then attempts setup-entry loading, while the disabled-config filter is only used by feature-specific helpers. I did not run the reported Windows `openclaw doctor` scenario in this read-only pass.

## Solution Assessment

Unclear on the exact suggested patch: filtering inside the root id helper may be too broad for onboarding, but threading config into diagnostic/status setup enumeration and filtering before generated module loads is the narrow maintainable direction.
