## Summary

Keep this PR open: the performance direction and real-world proof are useful, but the latest head still introduces model-ref normalization regressions in the new fast paths.

## What This Changes

The PR adds request-local `sessions.list` model-identity memoization, a fast path for qualified selected defaults, lightweight-row thinking skips, focused gateway tests, a changelog entry, and a test typing adjustment.

## Best Possible Solution

Keep the request-local cache design, but route selected-default and complete-override fast paths through the same normalized model-ref contract as current main, with alias regression coverage.

## Reproduction Assessment

Yes. Source inspection of PR head shows rows with no runtime metadata and `primary: "anthropic/sonnet-4.6"`, or complete override aliases, bypass the current normalized resolver path.

## Solution Assessment

No. The cache direction is maintainable, but hand-splitting selected defaults and directly returning override tuples is not the narrowest safe fix because it drops existing model alias normalization.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve default-model normalization in the fast path:** `src/gateway/session-utils.ts:551-553`
  - body: The selected-default shortcut returns the hand-parsed suffix for any qualified agent primary before `resolveDefaultModelForAgent()` and the shared parser run. A row with no runtime metadata and `primary: "anthropic/sonnet-4.6"` will now show `sonnet-4.6` instead of the existing normalized `claude-sonnet-4-6`; cache the normalized resolver result or parse through the shared model-ref resolver.
  - confidence: 0.91
- **[P2] Normalize complete override refs before returning them:** `src/gateway/session-utils.ts:645-649`
  - body: This early return bypasses `resolveSessionModelRef()` when both override fields are present. Existing behavior sends `providerOverride: "anthropic", modelOverride: "sonnet-4.6"` through `resolvePersistedSelectedModelRef()`/`parseModelRef()`, so the PR would display the raw alias in `sessions.list`; reuse the shared selected-model resolver or normalize the tuple before returning it.
  - confidence: 0.88
