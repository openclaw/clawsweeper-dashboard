## Summary

Keep this PR open: the cache direction is useful, but the current head still introduces a session-list correctness regression by bypassing established model-ref normalization for provider-qualified default models.

## What This Changes

The PR adds session-list model identity caching and qualified default-model fast paths, plus a gateway regression test, changelog entry, and a TypeScript test annotation.

## Best Possible Solution

Keep the cache design, but make the default-model fast path preserve the existing model-selection normalization contract and add session-list coverage for provider-qualified aliases without runtime metadata.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: list a session row with no runtime model metadata and an agent primary like `anthropic/sonnet-4.6` or `vercel-ai-gateway/opus-4.6`; the PR fast path returns the raw suffix instead of the normalized model ref.

## Solution Assessment

No. The broader caching approach is maintainable, but this fast path should reuse or cache the existing normalized model resolver rather than hand-splitting qualified strings.
