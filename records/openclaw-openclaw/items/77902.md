## Summary

Keep open: the latest head has useful session-list performance work and sufficient real gateway log proof, but it still drops the existing lightweight `sessions.list` thinking-options contract and bypasses model-ref normalization for complete override rows.

## What This Changes

The PR adds request-local gateway session-list model identity memoization, selected-default fast paths, gateway/model fallback tests, and a changelog entry.

## Best Possible Solution

Keep the request-local cache design, but preserve lightweight rows' per-session thinking options and route complete overrides through the shared normalized selected-model resolver.

## Reproduction Assessment

Yes. Source inspection of PR head shows the lightweight row path now omits thinkingOptions, and current main has e2e coverage that expects those options for Control UI; a focused alias regression can reproduce the complete-override normalization gap.

## Solution Assessment

No. The cache direction is maintainable, but the current implementation is not the best fix until it preserves the existing thinking-options row contract and normalized model-ref contract.
