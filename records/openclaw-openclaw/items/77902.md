## Summary

Keep open; the latest head still bypasses established model-ref normalization in the selected-default fast path, so qualified aliases can be displayed incorrectly despite the useful performance direction and supplied real-world proof.

## What This Changes

The PR adds a request-local `sessions.list` model-identity cache, a qualified default-model fast path, focused gateway regression coverage, a changelog entry, and a test typing adjustment.

## Best Possible Solution

Keep the cache design, but make the selected-default fast path preserve the existing normalized model-ref contract and add session-list coverage for provider-qualified aliases without runtime metadata.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `listSessionsFromStore` builds rows with a row context, and a row with no runtime metadata/overrides plus agent primary `anthropic/sonnet-4.6` returns the hand-parsed raw suffix instead of the normalized model ref.

## Solution Assessment

No. The broader cache approach is maintainable, but the qualified-default shortcut should reuse or cache the existing normalized resolver result rather than hand-splitting provider/model strings.
