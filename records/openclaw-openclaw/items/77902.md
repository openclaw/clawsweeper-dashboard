## Summary

Keep open: the PR targets a real gateway session-list hot path and includes credible real-world latency proof, but the current head would regress established model-ref normalization for qualified aliases.

## What This Changes

The PR adds a request-local `sessions.list` model-identity cache, a fast path for already-qualified selected default model refs, focused gateway tests, a changelog entry, and a test typing adjustment.

## Best Possible Solution

Keep the cache design, but make the selected-default fast path preserve the existing normalized model-ref contract and add session-list coverage for provider-qualified aliases without runtime metadata.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `listSessionsFromStore` builds rows with a row context, and a row with no runtime metadata or overrides plus agent primary `anthropic/sonnet-4.6` returns the hand-parsed raw suffix instead of the normalized model ref.

## Solution Assessment

No. The request-local cache direction is maintainable, but the qualified-default shortcut should reuse or cache the existing normalized resolver result instead of hand-splitting provider/model strings.
