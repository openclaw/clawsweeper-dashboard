## Summary

Current main fixes the reported legacy `claude-cli/*` isolated-cron allowlist rejection by preserving legacy runtime model keys during canonical runtime migration, with regression coverage and a changelog entry; the fix first shipped in `v2026.5.2-beta.2`, while `v2026.4.29` predates it.

## What This Changes

Allow isolated cron jobs with `payload.model: "claude-cli/*"` to continue passing model allowlist validation after the Claude CLI runtime migration.

## Best Possible Solution

Keep the compatibility fix for legacy `claude-cli/*` cron payloads; handle the separate canonical `anthropic/*` plus `agentRuntime.id: "claude-cli"` transport bug through #76265 and #75840.

## Reproduction Assessment

Yes. The original reproduction is source-backed by the prior fail-closed cron validation path, and current main has targeted tests showing the legacy `claude-cli/*` key is preserved and accepted.

## Solution Assessment

Yes. Preserving the legacy allowlist key while also adding the canonical provider key is the narrowest compatible fix because it keeps documented legacy refs working without undoing the runtime migration.
