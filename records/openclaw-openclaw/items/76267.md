## Summary

Current main and the shipped v2026.6.6 release already cover the reported cron/Claude CLI regression: cron execution now resolves canonical Anthropic attempts to the Claude CLI execution provider, legacy `claude-cli/...` allowlist refs are accepted, and older Claude CLI config is migrated to the canonical runtime-policy shape.

## What This Changes

The issue requests restoring Claude CLI-backed model selection for isolated cron jobs that rejected `claude-cli/...` or alias payload models after v2026.4.29.

## Best Possible Solution

Keep the shipped v2026.6.6 canonical Anthropic plus `agentRuntime.id: "claude-cli"` path and legacy compatibility tests; ask for a new report only if the latest release still rejects the same cron payload shape.
