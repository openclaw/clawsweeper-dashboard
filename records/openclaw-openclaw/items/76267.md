## Summary

Keep open. Current main still has a credible regression path: the Anthropic CLI migration rewrites `claude-cli/*` allowlist/default entries to canonical `anthropic/*`, while isolated cron still validates a raw `payload.model` through the generic allowlist resolver and can reject the legacy `claude-cli/*` ref before execution.

## What This Changes

The issue asks to restore support for `claude-cli/*` model references in isolated cron job payloads after v2026.4.29's Anthropic CLI migration and strict cron model validation.

## Best Possible Solution

Add a narrow compatibility path so isolated cron accepts legacy runtime model refs against the migrated canonical allowlist and still executes through the selected CLI backend.

## Reproduction Assessment

Yes. A focused unit test can call `resolveCronModelSelection` with `agents.defaults.models` containing `anthropic/claude-sonnet-4-6`, `agentRuntime.id: "claude-cli"`, and `payload.model: "claude-cli/claude-sonnet-4-6"`; the current source path explains the reported pre-execution rejection.

## Solution Assessment

Yes. The maintainable fix is to normalize legacy runtime refs at the model-selection/cron boundary while preserving runtime routing, not to relax cron's fail-closed model validation.
