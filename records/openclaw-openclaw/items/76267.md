## Summary

Keep open. Current main still has a source-reproducible compatibility regression: isolated cron validates `payload.model` against the exact resolved allowlist key, while Claude CLI setup now migrates allowlists to canonical `anthropic/*` refs and docs still promise legacy `claude-cli/*` refs work.

## What This Changes

The issue asks to restore isolated cron support for legacy `claude-cli/*` model refs after Claude CLI configs were migrated to canonical `anthropic/*` allowlist entries with `agentRuntime.id: "claude-cli"`.

## Best Possible Solution

Restore legacy runtime-ref compatibility by accepting `claude-cli/<claude-model>` when the matching canonical `anthropic/<model>` is allowed and `claude-cli` runtime execution is selected, while keeping genuinely unallowed models fail-closed.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a migrated Claude CLI config contains canonical `anthropic/claude-sonnet-4-6` in `agents.defaults.models`, but an isolated `agentTurn` cron payload with `model: "claude-cli/claude-sonnet-4-6"` resolves to the legacy key and is rejected before execution.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing legacy runtime alias seam during cron allowlist resolution, not to weaken the allowlist or add a new fallback execution policy.

## Repair Work Prompt

Fix the isolated cron Claude CLI legacy model compatibility regression from https://github.com/openclaw/openclaw/issues/76267. In current main, migrated Claude CLI configs allow canonical `anthropic/*` refs with `agents.defaults.agentRuntime.id: "claude-cli"`, but isolated cron rejects legacy payload refs like `claude-cli/claude-sonnet-4-6` because allowlist resolution checks the exact legacy key. Restore compatibility by accepting legacy runtime refs only when the corresponding canonical provider/model is allowed and the selected runtime/backend matches the legacy runtime alias. Preserve fail-closed behavior for genuinely unallowed providers/models and unrelated runtime aliases. Prefer the existing `src/agents/model-runtime-aliases.ts` seam over hard-coding Claude-only policy in cron. Add focused regression coverage for `resolveAllowedModelRef` and isolated cron payload model selection/run behavior. Add a one-line user-facing `CHANGELOG.md` fix entry. Do not relax the global model allowlist and do not add a new config option.
