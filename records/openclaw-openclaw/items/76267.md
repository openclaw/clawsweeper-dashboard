## Summary

Current main already implements the compatibility fix for the reported regression: legacy `claude-cli/*` model refs are preserved in the configured allowlist during canonical runtime migration, the cron selector continues to accept them when present, and the changelog records the matching #75753 fix. The proof is on current main only; no release tag contains the proof SHA yet.

## What This Changes

The issue asks for isolated cron jobs using `claude-cli/...` model refs to keep running after the v2026.4.29 cron model allowlist fail-closed change.

## Best Possible Solution

Keep the current compatibility approach: preserve legacy `claude-cli/*` allowlist keys during migration while steering new configuration toward canonical `anthropic/*` refs plus `agentRuntime.id`.

## Reproduction Assessment

Yes. The report gives a concrete failing cron payload and current main has a direct static verification path through the allowlist normalizer, cron model selector, and regression test; I did not run tests because this review was constrained to read-only inspection.

## Solution Assessment

Yes. Preserving legacy allowlist entries is the narrowest maintainable fix because it restores scheduled Claude CLI jobs without weakening cron's fail-closed model validation or forcing a direct-API fallback.
