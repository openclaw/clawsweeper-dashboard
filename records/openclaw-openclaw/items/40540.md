## What This Changes

The issue reports that `openclaw update` fails on Windows npm global installs with an `EBUSY` rename error because the active OpenClaw install directory is locked during self-update.

## Best Possible Solution

Land a focused Windows update fix that covers the npm-global CLI path by releasing the active package tree before the verified swap, while preserving the staged verification and restart-result contracts already in current main.

No live current-main Windows run was performed here. The issue has concrete Windows npm-global EBUSY logs, and current source still exposes an in-process CLI package-tree swap path without a detached Windows handoff.
