## What This Changes

`openclaw update` should complete npm-global self-updates on Windows instead of failing with an `EBUSY` rename error.

## Best Possible Solution

Land a focused detached Windows npm-global update path that stages and verifies the package, releases the invoking process before swapping the locked package tree, preserves restart/status behavior, and includes redacted Windows proof.

## Reproduction Assessment

Yes. The report provides concrete Windows npm-global `EBUSY` rename logs, and current main still swaps the active npm package root from the updater path without first detaching the invoking CLI process.

## Solution Assessment

No. Current main's gateway-stop and control-plane handoff safeguards are partial; the maintainable fix is to finish https://github.com/openclaw/openclaw/pull/75649 or an equivalent detached staged-swap design with Windows proof.
