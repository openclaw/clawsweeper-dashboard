## Summary

Keep open. Current main still cannot satisfy the requested hook behavior because the public model-resolve result type and hook merger only carry provider/model overrides, but this PR is not ready to merge: the queued reply path still drops the legacy auth-profile compaction marker needed to distinguish migrated automatic profiles from user-pinned profiles, and the user-facing plugin API change needs a changelog entry.

## What This Changes

The PR adds `authProfileOverride` to `before_model_resolve` and legacy `before_agent_start` hook results, threads it into embedded-runner auth-profile selection and live-switch comparison, and updates related plugin docs and tests.

## Best Possible Solution

Keep the hook/API direction, but carry the legacy auth-profile compaction marker through queued reply run state into embedded-runner params and preserve the existing distinction between source-less user-pinned profiles and compaction-marked automatic profiles. Add a regression for the queued path plus the required changelog entry before merge.

## Reproduction Assessment

Yes. A high-confidence static reproduction is a migrated session entry with `authProfileOverride`, numeric `authProfileOverrideCompactionCount`, and no `authProfileOverrideSource` entering a queued/follow-up reply; the PR helper then receives no compaction count from `FollowupRun.run` and classifies the profile as locked user state.

## Solution Assessment

No. The proposed API is the right narrow product direction, but the implementation is incomplete until every runner entry path that can carry session auth-profile state forwards or normalizes the legacy compaction marker before `resolvePreferredRunAuthProfile` runs.
