## Summary

Keep open for maintainer review: current main still writes raw memory snippets/promotions into `DREAMS.md` on narrative fallback, and this PR directly replaces that with a generic placeholder plus focused tests and sufficient terminal proof. The only remaining concern is an intentional user-visible fallback behavior change that maintainers should accept before merge.

## What This Changes

This PR makes memory-core Dream Diary fallback entries use a generic placeholder instead of raw snippets/promotions and updates narrative tests for timeout and request-scoped fallback paths.

## Best Possible Solution

Land a placeholder-only fallback unless maintainers prefer a separate sanitized summary design; raw staging fragments should not remain the fallback diary prose.

## Reproduction Assessment

Yes. Current main's fallback helper returns raw `snippets`/`promotions`, and the orchestrator writes that value on timeout or empty output; the PR body also includes before/after terminal output for the same production write path.

## Solution Assessment

Yes. Centralizing the change in the fallback helper is the narrowest maintainable fix because all relevant fallback write paths already call that helper, and successful generated narratives are left unchanged.
