## Summary

Keep open: current main still uses the broad lowercase `startsWith("http")` heuristic, while this PR applies a focused shared-helper fix with targeted coverage and no obvious compatibility or security regression.

## What This Changes

The branch changes `formatDocsLink()` to recognize only case-insensitive HTTP(S) absolute URLs and adds regression tests for uppercase schemes and http-prefixed relative docs paths.

## Best Possible Solution

Land the shared terminal-core helper fix after normal maintainer validation, keeping docs URL normalization centralized with its regression tests.

## Reproduction Assessment

Yes. A deterministic helper call on current main shows the path: `formatDocsLink("http-status")` follows the absolute-URL branch, while `formatDocsLink("HTTPS://example.com/page")` is treated as relative; I did not run code in this read-only review.

## Solution Assessment

Yes. The shared helper is the narrowest maintainable layer because plugin SDK and setup callers already route through `formatDocsLink`, and caller-side special cases would duplicate URL policy.
