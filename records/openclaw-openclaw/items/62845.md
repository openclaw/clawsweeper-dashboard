## Summary

Keep open. The latest head fixes the previously reported `/activate` regex problem, but the WebChat lowercase silent-reply change is still incomplete because the final-event reload helper remains case-sensitive, so a deferred history reload can be cleared without replay after a lowercase `no_reply` final.

## What This Changes

The PR updates WebChat silent-reply filtering/tests for lowercase and streamed `NO_REPLY` fragments, adds a `/activate` group-command alias, and adds changelog bullets.

## Best Possible Solution

Align the controller and final-event reload helper around the same case-insensitive exact silent-token contract, keep prefix-fragment suppression in the stream/fallback path, and land with contributor changelog credit.

## Reproduction Assessment

Yes. A high-confidence static reproduction is: defer a `session.message` reload while `chatRunId` is active, receive a lowercase `no_reply` final that the PR controller drops, then have the unchanged reload helper return false so `app-gateway` clears the pending reload without replaying history.

## Solution Assessment

No. The browser-side suppression direction is correct, and the `/activate` alias regex is now fixed, but the patch is not complete until `ui/src/ui/chat-event-reload.ts` and its tests share the same lowercase exact-token behavior.
