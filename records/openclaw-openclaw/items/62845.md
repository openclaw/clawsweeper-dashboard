## What This Changes

The PR changes WebChat silent-reply stream filtering, adds controller tests, adds a `/activate` activation-command parser alias, and adds changelog entries.

## Best Possible Solution

Make the WebChat client defense consistent across the controller and reload helper, and finish `/activate` through the dedicated activation-alias path with registry and WhatsApp coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path for the review blockers: `/activate` is parser-only and absent from registry detection, and lowercase silent finals diverge between the controller and reload helper.

## Solution Assessment

No. The browser-side suppression direction is reasonable, but the current PR is not the narrow complete fix because it leaves related helper and command-detection paths inconsistent and can drop a legitimate one-character fallback reply.

## Review Findings

Overall confidence: 0.89

Full review comments:

- **[P1] Register `/activate` in command detection:** `src/auto-reply/group-activation.ts:31`
  - body: Changing `parseActivationCommand()` alone leaves `/activate mention` outside the registry-backed detector. WhatsApp group gating uses `hasControlCommand(commandBody, params.cfg)` for the owner no-mention bypass, so the alias can still be skipped before the activation handler runs unless the registry and coverage are updated too.
  - confidence: 0.91
- **[P2] Align lowercase silent finals with history reload:** `ui/src/ui/controllers/chat.ts:18`
  - body: The controller now drops lowercase `no_reply`, but `ui/src/ui/chat-event-reload.ts` still treats the same final payload as renderable. When a deferred `session.message` reload is pending, `app-gateway` can clear it without replaying history, so the reload helper and tests need the same case-insensitive exact-token contract.
  - confidence: 0.86
- **[P2] Avoid dropping legitimate single-letter fallback replies:** `ui/src/ui/controllers/chat.ts:77`
  - body: `isSilentReplyTerminalStream()` suppresses any terminal fallback text equal to `N` even when the final or abort event has no silent `NO_REPLY` payload proving it was a sentinel fragment. That can make a real one-character assistant reply disappear after streaming, so scope `N` suppression to confirmed silent finals or remove it.
  - confidence: 0.78
