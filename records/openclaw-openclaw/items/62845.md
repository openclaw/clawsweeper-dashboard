## Summary

Keep open: the latest head fixes the earlier broken `/activate` regex, but the PR still has blocking correctness gaps and lacks real after-fix behavior proof. The WebChat direction is useful, but lowercase finals, reload behavior, terminal fallback handling, and the bundled activation alias need to be made consistent before merge.

## What This Changes

The PR changes WebChat silent-reply stream filtering and tests, adds `/activate` parsing coverage, and adds changelog entries.

## Best Possible Solution

Finish one coherent fix: share the silent-reply exact/prefix contract across WebChat controller and reload paths, remove or scope the bare-N fallback suppression, and either split `/activate` to the dedicated alias PR or complete it with registry and WhatsApp group-gating coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: lowercase silent finals diverge between controller and reload helper, parser-only `/activate` misses registry-backed WhatsApp group detection, and the new terminal helper would drop a real fallback text of `N`.

## Solution Assessment

No. The browser-side suppression direction is reasonable, but this patch is not the narrow complete fix because it leaves related helpers out of sync, adds an overbroad terminal fallback, and mixes in incomplete activation-alias work.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Register `/activate` with command detection:** `src/auto-reply/group-activation.ts:31`
  - body: Changing `parseActivationCommand()` alone leaves `/activate mention` outside the registry-backed control-command detector. WhatsApp group gating uses `hasControlCommand(commandBody, params.cfg)` for the owner no-mention bypass, so this alias can still be skipped before the activation handler runs unless the registry and group-gating coverage are updated too.
  - confidence: 0.91
- **[P2] Update reload suppression for lowercase finals:** `ui/src/ui/controllers/chat.ts:21`
  - body: The controller now drops lowercase `no_reply`, but `ui/src/ui/chat-event-reload.ts` still treats that same final payload as renderable. When a deferred session-message reload is waiting, `app-gateway` can clear it without replaying history, so the reload helper and tests need the same case-insensitive exact-token contract.
  - confidence: 0.86
- **[P2] Scope single-letter fallback suppression:** `ui/src/ui/controllers/chat.ts:78`
  - body: `isSilentReplyTerminalStream()` treats any terminal fallback text equal to `N` as silent even when the final/abort event has no `NO_REPLY` payload proving it was a sentinel fragment. That can make a legitimate one-character assistant reply disappear after streaming, so scope it to confirmed silent finals or remove it.
  - confidence: 0.78
