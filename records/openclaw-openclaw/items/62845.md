## Summary

Keep open: the latest head fixes the earlier `/activate` regex problem, but it still misses the companion Control UI reload predicate, so lowercase silent finals can be dropped by the controller while the deferred history reload path treats them as renderable.

## What This Changes

The PR updates Control UI/WebChat silent-reply filtering and tests for lowercase and streamed `NO_REPLY` fragments, adds `/activate` as a group `/activation` alias, and adds changelog entries.

## Best Possible Solution

Align the controller and final-event reload helper around the same case-insensitive exact silent-token contract, keep prefix-fragment suppression limited to the WebChat stream/fallback path, and preserve the tested `/activate` alias if maintainers accept that extra scope.

## Reproduction Assessment

Yes. A high-confidence static reproduction is: defer a `session.message` reload while `chatRunId` is active, receive a final assistant payload containing lowercase `no_reply`, have `handleChatEvent` drop it, then have `shouldReloadHistoryForFinalEvent` return false so `app-gateway` clears the pending reload without calling `loadChatHistory`.

## Solution Assessment

No. The browser-side fix is the right direction, but it is incomplete until the companion final-event reload predicate and tests treat lowercase silent replies the same way as the controller.
