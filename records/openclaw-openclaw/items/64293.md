## Summary

Keep open. Current main still treats an explicit empty heartbeat object as enabled, resolves it to the default cadence, and the open closing PR for this issue remains unmerged.

## What This Changes

Treat `agents.defaults.heartbeat: {}` as disabled so scheduled heartbeat polls do not run or burn tokens during idle periods.

## Best Possible Solution

Review and land or replace PR #64540 with a narrow shared resolver/defaults fix, regression coverage, and docs/changelog for the chosen disable contract.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: configure `agents.defaults.heartbeat: {}`, then the resolver treats it as enabled and resolves a non-null default interval.

## Solution Assessment

No for current main. The narrow maintainable fix should make empty heartbeat objects an explicit disable signal consistently across summary, runner scheduling, and Anthropic default backfill, or document a different supported contract.
