## Summary

Keep open. Current main and v2026.5.18 still accept `agents.defaults.heartbeat: {}`, treat it as enabled, and resolve the missing cadence to the default interval; the open closing PR is still unresolved and conflicting.

## What This Changes

The issue asks OpenClaw to treat `agents.defaults.heartbeat: {}` as disabling automatic heartbeat polling so idle installs do not burn provider tokens.

## Best Possible Solution

Choose and document the empty-object heartbeat contract, then implement one shared resolver used by scheduler selection, interval/status, prompt/bootstrap, provider defaults, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure `agents.defaults.heartbeat: {}` and current main treats the object as enabled while resolving the missing cadence to the default interval; I did not run a live gateway or remeasure provider usage.

## Solution Assessment

Unclear until maintainers choose the empty-object config contract. The maintainable path is one shared heartbeat policy resolver, not another isolated check in only one helper.
