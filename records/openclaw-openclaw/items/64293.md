## Summary

Keep this issue open. Current main still accepts `agents.defaults.heartbeat: {}` and treats it as enabled, then falls through to the default heartbeat interval; Anthropic config defaults can also populate `every` onto an explicit empty heartbeat object. The linked closing PR remains open and unmerged, so it is a candidate implementation path rather than a reason to close the issue.

## What This Changes

The issue requests that `agents.defaults.heartbeat: {}` disable scheduled heartbeat polls instead of allowing the default 30-minute cadence and token usage.

## Best Possible Solution

Keep this issue open and review or replace PR #64540 with a maintainer patch that defines `{}` as a disable signal, applies it consistently in default and per-agent heartbeat resolution, prevents provider default backfill from overriding it, and adds focused regression tests plus docs for the chosen contract.

## Reproduction Assessment

Yes. Code inspection gives a high-confidence reproduction path on current main: set `agents.defaults.heartbeat: {}`, let config resolution run, and the heartbeat resolver still returns an enabled 30-minute interval; live API usage is only needed to measure cost, not to prove the bug.

## Solution Assessment

The best maintainable solution is a narrow contract-level fix with regression coverage for `{}`, `every: "0m"`, explicit non-empty configs, and provider defaults.
