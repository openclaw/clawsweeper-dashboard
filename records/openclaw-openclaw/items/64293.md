## Summary

Keep open. Current main still treats an explicit empty heartbeat object as enabled and resolves it to the default cadence, while the linked closing PR remains open and conflicting; docs also still name `0m` as the disable contract, so this needs maintainer follow-up rather than cleanup close.

## What This Changes

Treat an explicit empty heartbeat config as disabling automatic heartbeat polling instead of falling back to the default cadence.

## Best Possible Solution

Make one shared heartbeat disable policy cover summary, scheduler selection, Anthropic default backfill, prompt/spawn gates, docs, and regression tests, or document that `0m` is the only supported disable path.
