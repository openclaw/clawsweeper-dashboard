## Summary

Keep open. Current `main` and the latest tagged release still treat an explicit empty heartbeat config as enabled and resolve it to the default cadence, while https://github.com/openclaw/openclaw/pull/64540 remains an open/conflicting closing PR and related heartbeat work disagrees on the empty-object contract.

## What This Changes

Treat an explicit empty heartbeat config object as disabling automatic heartbeat polling instead of scheduling the default cadence.

## Best Possible Solution

Define one heartbeat disable/absence contract, then implement it through a shared resolver used by config/default backfill, scheduler, status, prompt/bootstrap, ACP spawn, docs, and focused regression tests.

## Reproduction Assessment

Yes, by source inspection: configure `agents.defaults.heartbeat: {}` and current `main` treats the object as truthy enablement while resolving the missing `every` field to the default cadence. I did not run the live gateway or measure provider usage in this read-only review.

## Solution Assessment

Unclear until maintainers choose the config contract. The narrow maintainable path is a shared heartbeat policy resolver, not another isolated empty-object check in only one helper.
