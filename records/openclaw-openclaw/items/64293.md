## Summary

Keep open. Current main and the latest release still treat `agents.defaults.heartbeat: {}` as an enabled heartbeat config that falls through to the default cadence, and the open closing PR remains unmerged and conflicting.

## What This Changes

The issue asks for an explicit empty heartbeat config object to disable automatic heartbeat polling instead of falling back to the default cadence.

## Best Possible Solution

Choose and document one heartbeat disable contract, then apply it through a shared resolver used by scheduler, status, prompt/bootstrap, ACP spawn, Anthropic defaults, and focused regression tests.

## Reproduction Assessment

Yes, by source inspection: configure `agents.defaults.heartbeat: {}`, and current main treats it as truthy enablement while resolving the interval to the default cadence. I did not run the gateway or tests because this review is read-only.

## Solution Assessment

Unclear until maintainers choose the config contract. The narrow maintainable solution is a shared heartbeat enablement/interval policy, not another one-off empty-object check in only one surface.
