## What This Changes

The issue reports that `agents.defaults.heartbeat: {}` still schedules heartbeat model calls every 30 minutes, causing idle token spend.

## Best Possible Solution

Define one heartbeat absence/disable resolver and apply it consistently across scheduler selection, interval resolution, status, system prompt, ACP spawn, provider defaulting, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure `agents.defaults.heartbeat: {}` and current main treats the object as enabled while resolving the missing cadence to the default interval; I did not run a live gateway or remeasure provider usage.

## Solution Assessment

Unclear. Treating `{}` as disabled addresses the reported symptom, but the maintainable fix needs a shared resolver plus an explicit maintainer decision because docs currently define `every: "0m"` as the disable path and adjacent PRs choose different empty-object semantics.
