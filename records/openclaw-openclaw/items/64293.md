## Summary

Keep open. Current main and the latest release still treat `agents.defaults.heartbeat: {}` as a truthy enabled heartbeat config that falls through to a default cadence, and an open conflicting PR already uses closing syntax for this issue.

## What This Changes

The issue requests that `agents.defaults.heartbeat: {}` disable automatic heartbeat polling instead of scheduling provider calls at the default cadence.

## Best Possible Solution

Define one heartbeat disable/absence contract, then apply it through a shared resolver used by scheduler selection, interval resolution, status, prompt/bootstrap, ACP spawn, provider defaulting, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure `agents.defaults.heartbeat: {}` and current main treats the object as enabled while resolving the missing cadence to the default interval; I did not run a live gateway or remeasure provider usage.

## Solution Assessment

Unclear until maintainers choose the empty-object config contract. The maintainable solution is one shared heartbeat policy resolver rather than another isolated check in only one helper.
