## Summary

Keep open. Current main and the latest release still treat an explicit empty heartbeat object as enabled/default cadence, while docs only define `0m` as the disable path and an open closing PR remains unresolved.

## What This Changes

The issue requests that `agents.defaults.heartbeat: {}` disable heartbeat polling so idle gateways stop making periodic model calls.

## Best Possible Solution

Choose one heartbeat disable contract, then implement it through a shared resolver covering summary, scheduler selection, Anthropic default backfill, prompt/bootstrap/ACP gates, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: configure `agents.defaults.heartbeat: {}`, and current main accepts the object, treats it as enabled, and resolves a non-null default heartbeat interval.

## Solution Assessment

Unclear. Treating `{}` as disabled is not the documented contract today, and the open narrow PR does not cover all heartbeat gates; the safer solution is a maintainer-approved shared policy plus docs and tests.
