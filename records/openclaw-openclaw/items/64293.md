## Summary

Keep open. Current main and v2026.5.12 still treat an explicit empty heartbeat config as enabled and resolve it to a default cadence, while https://github.com/openclaw/openclaw/pull/64540 remains open/conflicting and related heartbeat work disagrees on whether empty objects mean disabled or absent.

## What This Changes

The issue asks for `agents.defaults.heartbeat: {}` to disable automatic heartbeat polling so idle installs stop making model calls.

## Best Possible Solution

Define one heartbeat disable contract, then implement it through a shared resolver used by config/default backfill, scheduler, status, prompt/bootstrap, ACP spawn, docs, and focused regression tests.

## Reproduction Assessment

Yes, by source inspection: configure `agents.defaults.heartbeat: {}`, and current main treats it as truthy enablement while resolving the missing `every` field to the default cadence. I did not run gateway/tests because this review is read-only.

## Solution Assessment

Unclear until maintainers choose the config contract. The narrow maintainable solution is a shared heartbeat policy resolver, not another one-off empty-object check in only one surface.
