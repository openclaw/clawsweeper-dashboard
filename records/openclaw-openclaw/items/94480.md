## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Use the quota target before warning:** `extensions/line/src/send.ts:275-277`
  - body: `getMessageQuotaConsumption()` only returns `totalUsage`; the SDK and LINE docs expose the actual limit and unlimited/limited state through `getMessageQuota()`. With the current hard-coded `180/200` check, non-200 or unlimited accounts get misleading quota warnings and the advisory still does not know whether exhaustion is near. Fetch/cache the quota target, or remove the hard-coded warning.
  - confidence: 0.9
- **[P2] Gate loading animation to one-on-one user IDs:** `extensions/line/src/bot-handlers.ts:505-508`
  - body: LINE's loading animation endpoint accepts a user ID for one-on-one chats, and current main's monitor keepalive already gates it to `ctx.userId && !ctx.isGroup`. This new code prefers `groupId` or `roomId`, so processed group/room messages make invalid swallowed API calls instead of providing useful feedback. Keep this user-only or rely on the existing monitor keepalive.
  - confidence: 0.88
