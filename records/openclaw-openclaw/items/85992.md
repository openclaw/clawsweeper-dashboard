  - body: The new Telegram-specific guard is only checked in `onToolResult`. With `message_tool_only` and verbose mode, `onPlanUpdate`, `onApprovalEvent`, and `onPatchSummary` still fall through to `sendPlanUpdate`/`maybeSendWorkingStatus`, so planning, approval, or patch status text can still leak into Telegram after this fix.   - confidence: 0.88

Reply payload metadata is WeakMap-backed, so when TTS returns a cloned payload the delivered payload loses source-reply transcript/suppression metadata; the PR also deletes the regression test that covered this. Keep the metadata copy and the test.
