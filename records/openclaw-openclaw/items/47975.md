## Summary

Keep open. Current main still retains default run-mode subagent sessions and keeps terminal non-archived rows visible in normal session-list/UI paths; newer liveness, task-maintenance, and sessions-list performance work mitigates some stale-active and responsiveness symptoms but does not settle the cleanup/visibility contract reported here.

## What This Changes

The issue asks for completed one-shot subagent sessions to stop persisting as stale dashboard/status rows and for the parent session to remain responsive after subagent completion.

## Best Possible Solution

Define one terminal one-shot subagent lifecycle contract, then implement it consistently across subagent registry cleanup, sessions.list defaults, Control UI filtering/status, task reconciliation, attachment retention, docs, and focused regression tests.

## Reproduction Assessment

Yes for the retained completed-session symptom: current source and tests show keep-mode run subagents remain stored and visible as terminal non-archived rows. The full parent-session unresponsiveness symptom remains only partially reproducible without a live Gateway/Control UI scenario.

## Solution Assessment

Unclear. A blanket flip from keep to delete or hiding every completed child row is not clearly safe because it changes history, debugging, task status, and attachment semantics; the narrow maintainable path is a maintainer-approved lifecycle/visibility contract first.
