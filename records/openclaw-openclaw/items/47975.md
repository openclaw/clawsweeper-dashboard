## Summary

Keep open. Current main still preserves default one-shot subagent sessions after completion and can surface terminal non-archived rows, while the parent-session responsiveness part remains covered by related open yield/completion reports rather than a merged fix.

## What This Changes

The issue asks OpenClaw to clean up completed one-shot subagent sessions and keep the parent session responsive after subagent completion.

## Best Possible Solution

Define one lifecycle contract for one-shot subagent history and yielded parent wake behavior, then implement it consistently across registry cleanup, sessions.list defaults, task reconciliation, attachment retention, Control UI visibility, docs, and regression tests.

## Reproduction Assessment

Yes at source level for retained completed-session rows: default run-mode cleanup is keep, keep runs have no archive timer, the sweeper skips them, and UI tests keep terminal non-archived rows visible. I did not establish a live current-main reproduction for the transient Dashboard unresponsiveness.

## Solution Assessment

No current main behavior fully solves the issue. The chat picker has a partial mitigation, but the maintainable fix needs a maintainer-approved lifecycle contract rather than a one-file cleanup change.
