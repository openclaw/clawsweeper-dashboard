## Summary

Keep open: current main and the latest release still retain default run-mode subagent sessions, can expose completed child rows through broad session-list paths, and do not fully settle the yielded-parent unresponsiveness reported in the thread. The issue has fresh user confirmation and related open reports for narrower slot-leak and yielded-parent variants, so it is not safe to close as implemented or duplicate.

## What This Changes

The issue asks OpenClaw to clean up or hide completed run-mode subagent sessions and let the parent session resume promptly after subagent completion.

## Best Possible Solution

Define one terminal subagent contract, then align registry cleanup, sessions.list and Control UI visibility, task reconciliation, attachment retention, docs, and yielded-parent wake behavior around that contract while leaving narrower variants to their dedicated reports.

## Reproduction Assessment

Partially: source inspection gives a high-confidence path for retained completed subagent rows and stale list/status surfaces, but no fresh live current-main run reproduced the parent-session unresponsiveness. The related live issue discussion supports that wake/resume symptoms still occur in production paths.

## Solution Assessment

Unclear: the narrow code paths are identifiable, but the best fix depends on the maintainer-chosen retention/history and yielded-parent wake contract. A cleanup-only patch is likely too narrow until that product decision is made.
