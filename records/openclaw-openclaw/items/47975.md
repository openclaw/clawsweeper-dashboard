## Summary

Keep open: current `main` and the latest release still retain default one-shot subagent sessions and preserve terminal non-archived rows in normal session surfaces, so the central lifecycle/parent-responsiveness problem is not fully fixed.

## What This Changes

The issue asks OpenClaw to clean up completed one-shot subagent sessions and keep the parent session responsive after subagent completion.

## Best Possible Solution

Define one lifecycle contract for one-shot subagent history, archive/retention, task reconciliation, attachment cleanup, Control UI visibility, and yielded-parent wake behavior, then implement it with focused registry, gateway, UI, docs, and regression-test updates.

## Reproduction Assessment

Yes at source level for retained completed subagent sessions: default run-mode cleanup is `keep`, keep-mode runs receive no archive timer, the sweeper skips those records, and UI tests keep terminal non-archived rows visible. I did not establish a live WSL2/webchat reproduction for the transient parent-session delay.

## Solution Assessment

No current implementation fully solves the issue. The picker clutter has a partial shipped fix, but the durable solution needs a maintainer-approved lifecycle and retention policy plus consistent parent wake behavior.
