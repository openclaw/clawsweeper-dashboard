## Summary

Keep open. Current main has partial mitigations for stale child-link display and stale-running status, but ordinary one-shot subagent runs still default to retained sessions with no archive timer, and Control UI still keeps terminal non-archived rows visible.

## What This Changes

The issue requests cleanup or hiding of completed one-shot subagent sessions so stale child state does not make the main session or session lists appear stuck.

## Best Possible Solution

Define one terminal lifecycle contract for one-shot subagents, then align registry archive timers, sessions.list defaults, Control UI filtering, task/status surfaces, attachment retention, docs, and regression tests to that contract.

## Reproduction Assessment

Yes for the retained completed-session symptom by source inspection: default one-shot spawns keep child sessions, keep-mode runs get no archiveAtMs, and unfiltered session-list/UI paths can still show terminal non-archived rows. No live current-main reproduction was established for the transient main-session unresponsive delay.

## Solution Assessment

Unclear. A narrow fix could honor the documented auto-archive contract, but current tests also preserve terminal non-archived rows, so maintainers should first decide how explicit keep, recent history, and UI visibility should work.
