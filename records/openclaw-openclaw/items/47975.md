## Summary

Keep open: current main and v2026.6.6 still preserve default run-mode subagent sessions, keep-mode runs do not receive archive timers, and broad session-list paths can still return retained completed rows. The parent-session unresponsiveness path has fresh user confirmation but still needs live proof and a maintainer decision on retention versus visibility semantics.

## What This Changes

The issue asks OpenClaw to clean up or hide completed subagent sessions so parent sessions remain responsive and session/status surfaces stop showing stale children as active work.

## Best Possible Solution

Define the one-shot subagent history contract first, then make run-mode cleanup/archive, sessions.list defaults, Control UI visibility, task reconciliation, attachment cleanup, and docs/tests all enforce that same contract.

## Reproduction Assessment

No high-confidence live reproduction of the parent-session unresponsiveness was run. Source inspection does reproduce the retained completed subagent rows and stale-status preconditions, but the channel/WebChat stall still needs live proof.

## Solution Assessment

Unclear. The implicated code paths are clear, but the best fix depends on a maintainer decision for whether terminal run-mode subagents should be deleted, archived, hidden from default lists, or retained as inspectable history.
