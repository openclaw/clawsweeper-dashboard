## Summary

Keep open: current main and the latest release still default one-shot subagent runs to retained session rows, skip session deletion for keep-mode runs, and allow broad session-list paths to enumerate those rows. Recent user confirmation and open related reports show the parent wake/status contract is still unresolved.

## What This Changes

The issue asks OpenClaw to clean up completed one-shot subagent sessions and keep parent sessions responsive after subagent completion.

## Best Possible Solution

Define one subagent completion contract covering retention, active/history visibility, task reconciliation, attachment cleanup, and yielded-parent wake semantics, then implement it consistently across registry, sessions.list, Control UI, docs, and focused tests.

## Reproduction Assessment

Partly: current source reproduces the retained one-shot subagent row and broad listing path, but I did not establish a live current-main reproduction of the full parent-session unresponsive symptom. A real WebChat or Telegram run with subagent completion and post-completion user input is still needed.

## Solution Assessment

Unclear: deleting completed rows alone is not the best fix because it would not settle cleanup=keep history, task lost-state reconciliation, attachment retention, or sessions_yield parent wake behavior. The best path is a maintainer-chosen completion contract followed by one coordinated implementation.
