## Summary

Keep open. Current main still retains the core reported behavior: one-shot subagent cleanup defaults to keeping the child session, keep-mode runs do not get archive timers, and the ordinary Control UI session-list path can still return stored completed child sessions. Recent liveness and task-reconciliation paths reduce stale active display in some surfaces, but they do not settle the cleanup, visibility, attachment-retention, and responsiveness contract reported here.

## What This Changes

The issue asks for completed one-shot subagent sessions to be cleaned up or hidden promptly so the session list, Control UI, and main session remain responsive after failed or attachment-backed subagent runs.

## Best Possible Solution

Define the intended contract for terminal one-shot subagent history: whether completed run-mode child sessions are hidden, archived, or pruned by default, how cleanup=keep remains inspectable, and how attachments are retained. Then implement that consistently across subagent registry cleanup, sessions.list defaults, Control UI filters/status, task reconciliation, docs, and focused regression tests.

## Reproduction Assessment

Yes for the retained completed-session symptom: current source, docs, and tests show default run-mode subagents use cleanup=keep, do not get archive timers, and can remain visible in unfiltered session lists. The main-session unresponsiveness symptom still needs a live Gateway/Control UI reproduction to separate it from related lane-starvation issues.

## Solution Assessment

Unclear. A narrow default flip to cleanup=delete or a blanket UI hide would be risky because current docs/tests treat keep-mode history as intentional; the safer solution is a maintainer-approved retention and visibility contract followed by a targeted implementation.
