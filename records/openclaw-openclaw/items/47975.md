## Summary

Keep open. Current main still retains ordinary one-shot subagent sessions by default and keeps terminal non-archived rows visible in session-list/Control UI paths; newer task reconciliation and session-list performance work are partial mitigations, not a complete fix for the reported lifecycle and responsiveness problem.

## What This Changes

The issue requests cleanup or hiding of completed one-shot subagent sessions and immediate parent-session responsiveness after subagent completion.

## Best Possible Solution

Define one terminal lifecycle contract for one-shot subagents, then align archive timers, durable retention, sessions.list defaults, Control UI filtering, task/status reconciliation, attachment cleanup, docs, and focused regression tests to that contract.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the retained completed-session symptom by source inspection: default run-mode cleanup is keep, keep-mode runs get no archiveAtMs, and unfiltered session-list/Control UI paths still show terminal non-archived rows. No live current-main reproduction was established for the transient parent-session unresponsive delay.

## Solution Assessment

Is this the best way to solve the issue? Unclear; there is no patch under review, and current docs, tests, and implementation disagree about default auto-archive versus keep-mode history. The safer path is a maintainer lifecycle-policy decision followed by a focused implementation and regression tests.
