## Summary

Keep open. Current main still retains ordinary run-mode subagent sessions by default, keep-mode runs are not scheduled for archive/delete, unfiltered session-list paths can still return completed child rows, and the Control UI explicitly keeps terminal non-archived rows visible; this issue is also the canonical tracker for two closed duplicate reports.

## What This Changes

The issue requests that completed one-shot subagent sessions stop lingering in session/status surfaces and that parent sessions remain responsive after subagent completion.

## Best Possible Solution

Define one subagent lifecycle contract for terminal one-shot history, active/status visibility, attachment retention, and task reconciliation, then implement it consistently across the registry, sessions.list, Control UI, docs, and focused regression tests while keeping lane-starvation work on narrower related trackers.

## Reproduction Assessment

Yes for the retained completed-session symptom: current docs, source, and tests show cleanup=keep is the default, keep-mode runs are not archived, and terminal non-archived rows remain visible. No high-confidence live reproduction was established for the full parent-session unresponsive behavior against current main.

## Solution Assessment

Unclear. A narrow flip from keep to delete or blanket hiding of completed rows is not clearly the best fix because it changes history, debugging, task status, and attachment semantics; maintainers should choose the lifecycle/visibility contract first.
