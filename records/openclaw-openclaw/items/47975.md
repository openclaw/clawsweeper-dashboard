## Summary

Keep open. Current main still retains ordinary one-shot subagent sessions by default, preserves keep-mode runs without archive timers, and only hides stale child links rather than deleting or excluding terminal stored rows; the remaining fix needs a maintainer lifecycle/retention decision before changing user-visible history semantics.

## What This Changes

The issue asks OpenClaw to clean up completed one-shot subagent sessions and keep parent sessions responsive after subagent completion.

## Best Possible Solution

Define one terminal lifecycle contract for one-shot subagents, then align archive timers, durable retention, sessions.list defaults, Control UI/status filtering, task reconciliation, attachment cleanup, docs, and focused regression tests to that contract.

## Reproduction Assessment

Yes for the retained completed-session symptom by source inspection: default run-mode cleanup is keep, keep-mode runs get no archiveAtMs, and unfiltered sessions.list can still return terminal stored subagent rows. No live current-main reproduction was established for the transient parent-session unresponsive delay.

## Solution Assessment

Unclear; there is no patch under review, and current docs, tests, and implementation disagree about the default auto-archive versus keep-mode history contract. The safer path is a maintainer lifecycle-policy decision followed by a focused implementation and regression tests.
