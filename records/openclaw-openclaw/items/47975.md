## Summary

Keep this open: current main and v2026.6.6 still retain ordinary run-mode subagent sessions by default, keep-mode runs are not scheduled for archive deletion, and yielded-parent/session delivery symptoms are still represented by open related reports plus a recent user update.

## What This Changes

The issue asks OpenClaw to clean up completed one-shot subagent sessions and keep the parent session responsive immediately after subagent completion.

## Best Possible Solution

Define one explicit contract for one-shot subagent retention, archive timing, child visibility, and yielded-parent wake delivery, then make registry cleanup, sessions.list, Control UI/status, task reconciliation, attachments, and docs follow it consistently.

## Reproduction Assessment

Yes for the cleanup/listing half by source inspection: run-mode subagents still default to `cleanup: "keep"`, keep-mode runs are not archived, and unfiltered session lists still enumerate retained rows. The full WebChat/Telegram parent-unresponsive symptom has strong issue evidence but was not live-reproduced here.

## Solution Assessment

Unclear. Immediate deletion of every completed run-mode child would satisfy part of the request, but the best fix needs a maintainer decision on retention/history visibility and yielded-parent wake semantics so completion delivery and inspectability are not regressed.
