## Summary

Keep open: the current main tree has cleanup and maintenance paths, but source inspection still shows completed one-shot subagent sessions are retained by default, keep-mode subagents bypass the archive timer, and stale running task reconciliation is conservative when a child session row still exists; the recent comments also report the behavior still occurs.

## What This Changes

The issue reports stale completed or failed subagent sessions that remain visible after one-shot runs and can leave the parent session or dashboard unresponsive.

## Best Possible Solution

Keep this issue open as the umbrella for subagent cleanup plus parent responsiveness, and land a maintainer-approved fix that hides or archives terminal one-shot subagents without breaking explicit persistent session history.

## Reproduction Assessment

Yes, from source there is a high-confidence path for retained terminal subagent sessions: one-shot spawn defaults to keep cleanup mode, terminal registry updates do not archive keep-mode runs, and cleanup only prunes them through separate retention paths. I did not run a live multi-subagent WebChat reproduction, so the parent unresponsiveness part remains medium confidence.

## Solution Assessment

Unclear: immediate cleanup would match the report, but the best fix needs maintainer direction on whether completed one-shot subagent history should be hidden, archived after a retention window, or retained but excluded from active/status and quota paths.
