## Summary

Keep open. Current main still preserves default one-shot subagent sessions, leaves keep-mode runs without archive timers, and exposes terminal non-archived child rows through normal session-list and Control UI paths; the parent-session `sessions_yield` paused-state fix is still in an open PR rather than on main.

## What This Changes

The issue reports stale completed or failed subagent sessions that remain visible after completion and can leave the parent session or Dashboard channel appearing unresponsive.

## Best Possible Solution

Define the lifecycle contract for one-shot subagent history and yielded parent sessions, then implement it consistently across registry cleanup, sessions.list defaults, Control UI filtering, task reconciliation, attachment retention, docs, and regression tests.

## Reproduction Assessment

Yes for the retained completed-session symptom at source level: default run-mode cleanup is `keep`, keep-mode runs have no archive timer, and list/UI paths still include terminal non-archived rows. No live current-main reproduction was established for the transient Dashboard delay.

## Solution Assessment

No current main behavior fully solves it. The maintainable fix needs a maintainer-approved lifecycle and retention policy, plus the top-level yielded-session paused-state fix or equivalent.
