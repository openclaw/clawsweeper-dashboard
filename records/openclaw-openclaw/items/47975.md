## Summary

Keep open. Current main still retains run-mode subagent sessions by default and ordinary session-list/Control UI paths can enumerate completed child rows; recent liveness and task reconciliation work mitigates some stale-active display but does not settle the cleanup, visibility, attachment, or responsiveness contract reported here.

## What This Changes

The issue requests that completed run-mode subagent sessions stop persisting as stale active-looking sessions and stop delaying the parent session after completion.

## Best Possible Solution

Define one contract for terminal one-shot subagent history, then implement it consistently across subagent cleanup, sessions.list defaults, Control UI status/filtering, task reconciliation, attachment retention, docs, and focused regression tests.

## Reproduction Assessment

Yes for the retained completed-session symptom: current docs, source, and tests show keep-mode run subagents remain stored and listable as historical rows. The parent-session unresponsiveness symptom is only partially reproducible from static evidence and needs a live gateway/Control UI scenario to separate it from related lane-starvation reports.

## Solution Assessment

Unclear. A narrow flip from `keep` to `delete` or blanket hiding of all completed child rows is not clearly safe because it changes history, debugging, task status, and attachment semantics; the maintainable path is a maintainer-approved retention/visibility contract first.
