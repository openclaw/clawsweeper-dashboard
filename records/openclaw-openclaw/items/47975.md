## Summary

Keep open. Current main still defaults one-shot subagent runs to retained session rows, keep-mode runs do not receive archive timers, and the unfiltered session-list path still includes stored completed child sessions; related duplicate reports were closed into this issue as the canonical lifecycle/performance tracker.

## What This Changes

Completed one-shot subagent sessions should be cleaned up or hidden after completion so stale child sessions do not accumulate or delay parent-session responsiveness.

## Best Possible Solution

Define one terminal one-shot subagent lifecycle contract for archive/delete/hide behavior, retained history, attachments, task status, and Control UI visibility, then align code, docs, and regression tests to that contract.

## Reproduction Assessment

Yes for the retained completed-session symptom by source inspection: current main defaults run-mode cleanup to keep, gives keep-mode runs no archive timer, and keeps terminal non-archived rows visible. No live reproduction was established for the full parent-session unresponsive behavior against current main.

## Solution Assessment

Unclear. A blanket flip from keep to delete or hiding every completed child row may fix the visible symptom, but it changes history, attachment, task-status, and Control UI semantics; the safer path is to settle the lifecycle policy and implement it consistently.
