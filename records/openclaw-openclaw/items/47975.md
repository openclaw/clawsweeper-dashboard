## Summary

Keep open. Current main still retains default one-shot subagent session rows and normal session-list/Control UI paths can still show completed non-archived child sessions; recent work mitigates pieces of status/task/list behavior but does not settle the cleanup, archive, or visibility contract requested here.

## What This Changes

The issue asks for completed or failed one-shot subagent sessions to stop lingering in session lists and delaying or blocking the parent session after completion.

## Best Possible Solution

Define and document one terminal lifecycle contract for one-shot subagents, then align registry cleanup, sessions.list defaults, task/status surfaces, attachments, Control UI filtering, and regression tests to that contract.

## Reproduction Assessment

Yes for the retained completed-session symptom by source inspection: current main defaults one-shot run cleanup to keep, gives keep-mode runs no archiveAtMs, and leaves non-archived terminal rows visible. No live current-main reproduction was established for the full parent-session unresponsive delay.

## Solution Assessment

Unclear. A blanket switch to delete or hiding all completed child rows would address the visible symptom, but it changes retention/history semantics; maintainers should first choose the lifecycle policy and then implement it consistently.
