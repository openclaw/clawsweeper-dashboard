The full WebChat stall was not live-reproduced here, but the retained-row/session-state path remains source-reproducible and the issue is still the canonical cleanup/unresponsiveness tracker.

The issue requests cleanup and reconciliation for completed subagent sessions so stale child rows do not leave the main session unresponsive or status surfaces misleading.
