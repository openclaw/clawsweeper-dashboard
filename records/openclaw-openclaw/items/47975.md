## Label Justifications

- `P1`: The report describes a broken agent/session workflow where completed subagents can leave the parent session misleadingly busy or temporarily unresponsive.
- `impact:session-state`: The central failure is stale or mismatched subagent lifecycle state persisting after completion.
- `impact:message-loss`: The reporter observed follow-up user messages being queued or ignored while the parent session appeared stuck after subagent completion.

## Summary

Keep open: current main and the latest release still retain default one-shot subagent sessions and do not archive/prune keep-mode run rows, so the central stale-session lifecycle problem is not fully fixed.

## What This Changes

The issue asks OpenClaw to clean up completed one-shot subagent sessions and keep the parent session responsive immediately after subagent completion.

## Best Possible Solution

Define one lifecycle contract for one-shot subagent history, archive/retention, task reconciliation, attachment cleanup, Control UI visibility, and yielded-parent wake behavior, then implement it consistently with focused tests and docs.

## Reproduction Assessment

Yes at source level for retained completed subagent sessions: default run-mode cleanup is keep, keep-mode runs receive no archive timer, and the sweeper skips them. I did not establish a live WSL2/webchat reproduction for the transient parent-session delay.

## Solution Assessment

No current implementation fully solves the issue. The maintainable fix needs a maintainer-approved lifecycle and retention contract rather than a narrow cleanup-only change.
