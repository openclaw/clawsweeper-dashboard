## Summary

## What This Changes

Review issue about completed subagent sessions remaining visible and parent sessions becoming temporarily unresponsive after subagent completion.

## Best Possible Solution

Maintainers should choose one subagent retention and yielded-parent resumption contract, then make registry, session-store, sessions.list, Control UI/WebChat, and completion delivery all follow it consistently.

## Reproduction Assessment

Yes for the cleanup/listing half by source inspection: run-mode subagents still default to cleanup keep, and session-list filtering preserves recent completed child links. ## Solution Assessment

Unclear. Immediate deletion of every run-mode child would satisfy part of the request, but the best fix needs a maintainer decision on retention/history visibility and a unified yielded-parent wake contract so completion delivery is not regressed.
