## Summary

Keep open. Current main still retains ordinary run-mode subagent sessions by default and the normal session-list/Control UI path still shows terminal non-archived child rows; current code has partial mitigations for stale child links and lost task projection, but not the requested lifecycle cleanup or the full responsiveness symptom.

## What This Changes

The issue requests cleanup or hiding of completed run-mode subagent sessions and immediate parent-session responsiveness after subagent completion.

## Best Possible Solution

Define one subagent lifecycle contract for terminal one-shot history, active/status visibility, attachment retention, and task reconciliation, then align registry cleanup, sessions.list defaults, Control UI filtering, docs, and focused regression tests with that contract.

## Reproduction Assessment

Yes for the retained completed-session symptom by source inspection: current main defaults run-mode cleanup to keep, gives keep-mode runs no archive timer, and keeps terminal non-archived rows visible. No live reproduction was established for the full parent-session unresponsive behavior against current main.

## Solution Assessment

Unclear. A blanket flip from keep to delete or hiding every completed child row is not clearly the best fix because it changes history, debugging, attachment, task status, and Control UI semantics; maintainers should choose the lifecycle policy first.
