## Summary

Keep open: current main still defaults one-shot `sessions_spawn` cleanup to `keep`, keep-mode runs get no archive timer, the sweeper skips keep-mode run records without `archiveAtMs`, and unfiltered `sessions.list` still starts from stored rows. The discussion also has a recent user confirmation and #86745 was closed as a duplicate pointing back to this issue as the canonical cleanup/unresponsiveness tracker.

## What This Changes

Issue #47975 requests cleanup and responsiveness fixes for completed subagent sessions that persist after completion and can leave the main session unresponsive.

## Best Possible Solution

Keep #47975 open as the umbrella, define the terminal one-shot subagent retention and visibility contract, then implement it consistently across registry cleanup, sessions.list, Control UI/status surfaces, task reconciliation, docs, and delivery-adjacent trackers.
