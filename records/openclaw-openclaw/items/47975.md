## Label Justifications

- `impact:session-state`: Subagent lifecycle state can diverge from completion state, leaving child rows, task records, or parent yield state stale.

## Summary

Keep open: current main and v2026.6.6 still retain ordinary run-mode subagent sessions by default, do not archive keep-mode runs, and only partially age out child links; the parent-yield responsiveness half is still covered by live related reports.

## What This Changes

The issue reports completed or failed run-mode subagent sessions lingering in session/task state and making the parent session or Dashboard/WebChat lane appear unresponsive.

## Best Possible Solution

Define one subagent lifecycle contract for terminal run-mode children, then apply it consistently across registry cleanup, session-store listing, task reconciliation, attachment retention, Control UI/WebChat display, docs, and yielded-parent wake delivery.

## Reproduction Assessment

Yes for the cleanup/listing half by source inspection: run-mode subagents default to `cleanup: keep`, keep-mode runs get no archive timer, and normal sessions.list paths can still return retained rows. The full parent unresponsive symptom has recent production evidence but was not live-reproduced in this read-only review.

## Solution Assessment

Unclear: immediate deletion of all run-mode children would satisfy part of the report but may be the wrong retention contract. The best fix needs a maintainer decision that unifies retention, visibility, task reconciliation, and yielded-parent wake behavior.
