## Summary

Keep open. Current main still retains normal one-shot subagent sessions by default, keeps terminal non-archived session rows visible in list/UI paths, and lacks the main-session `sessions_yield` paused-state handling that would address the parent-session unresponsive symptom; task reconciliation and recent retention work are only partial mitigations.

## What This Changes

The issue reports completed or failed one-shot subagent sessions lingering in session/task surfaces and causing the parent session or Dashboard channel to appear unresponsive.

## Best Possible Solution

Define the terminal lifecycle contract for one-shot subagents and yielded parent sessions, then align archive timers, persisted session status, sessions.list/UI filtering, task reconciliation, attachment cleanup, docs, and regression tests to that contract.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the retained completed-session symptom by source inspection: default run-mode cleanup is keep, keep-mode runs get no archive timer, and unfiltered session-list/Control UI paths keep terminal non-archived rows visible. No live current-main reproduction was established for the transient Dashboard delay.

## Solution Assessment

Is this the best way to solve the issue? Unclear because there is no patch under review and related work only covers slices of the problem; https://github.com/openclaw/openclaw/pull/75662 may address the yielded main-session state, but the subagent retention/UI contract still needs maintainer policy before implementation.
