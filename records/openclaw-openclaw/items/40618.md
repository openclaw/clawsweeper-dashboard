## What This Changes

The PR adds an optional `hooks.internal.entries.session-memory.outputDir` setting, tests, and docs so bundled session-memory snapshots can be written outside the default `memory/` directory.

## Best Possible Solution

Reimplement the output-directory option on the current handler with root-safe parent creation, preserve current defaults and background/local-time/collision/`llmSlug` semantics, and align the config surface with https://github.com/openclaw/openclaw/issues/60034.

## Reproduction Assessment

Yes, at source level. Current main still hardcodes session-memory output under `<workspace>/memory`, and the PR body plus a live-user comment describe the custom-directory need clearly; I did not run a live workspace reproduction.

## Solution Assessment

No, not as the branch currently stands. A narrow `outputDir` may be acceptable, but it needs to be rebased onto current main, made root-safe before directory creation, and reconciled with the broader artifact-template design issue.
