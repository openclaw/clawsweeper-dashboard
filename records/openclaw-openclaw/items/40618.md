## Summary

Keep this PR open because current `main` still hardcodes session-memory output to `memory/`, while the PR addresses a real reported workflow but needs concrete correctness, docs-policy, and changelog fixes before merge.

## What This Changes

This PR adds optional `hooks.internal.entries.session-memory.outputDir` support to the bundled session-memory hook, extends handler tests, and updates English plus localized hook documentation and metadata.

## Best Possible Solution

Land the optional output directory feature after routing directory creation through root-aware filesystem helpers or the existing writer, removing localized docs changes, adding a changelog entry, and keeping the default `memory` behavior unchanged.

## Reproduction Assessment

Yes. On current main, configure `hooks.internal.entries.session-memory.outputDir`, trigger `/new` or `/reset`, and the hook still writes under `<workspace>/memory`; for the PR-introduced bug, use an in-workspace symlink prefix in `outputDir` and observe raw directory creation before the safe writer rejects the final write.

## Solution Assessment

No, not in the current patch shape. The feature boundary is narrow and maintainable, but the implementation should avoid raw pre-write directory creation, follow docs policy, and include the required changelog entry before merge.
