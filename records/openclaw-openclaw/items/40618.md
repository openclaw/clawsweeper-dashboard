## What This Changes

The PR adds an optional `hooks.internal.entries.session-memory.outputDir` setting plus docs and tests so session-memory snapshots can be written to a workspace-relative directory other than `memory`.

## Best Possible Solution

Rebase the optional output-directory feature onto the current handler, use root-aware filesystem helpers before any configured-path side effect, remove localized docs edits, keep English docs and hook metadata aligned, and add redacted real workspace proof.

## Reproduction Assessment

Yes, at source level. Current main always writes session-memory artifacts under `<workspace>/memory`, and the PR body gives a concrete `outputDir` config path that is not supported today.

## Solution Assessment

No, not as submitted. The `outputDir` direction is plausible, but the branch should be rebased onto current session-memory behavior, avoid raw filesystem side effects before root-bound validation, and keep localized docs out of this repo.
