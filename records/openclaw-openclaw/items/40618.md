## Summary

Keep this PR open. The requested configurable session-memory output directory is still missing from current main and the discussion shows it solves a real workspace-memory pollution problem, but the current patch has a filesystem-boundary regression, edits generated localized docs, and lacks the required changelog entry.

## What This Changes

This PR adds optional `hooks.internal.entries.session-memory.outputDir` support to the bundled session-memory hook, updates handler tests, and updates English plus localized hook documentation and bundled hook metadata.

## Best Possible Solution

Land the optional `outputDir` feature after routing all directory creation through root-aware helpers or the existing writer parent creation, adding a symlink-prefix regression test, removing localized docs edits, and adding a changelog entry. Keep the default `memory` path and leave broader memory-search duplicate-indexing guidance to #54524.

## Reproduction Assessment

Yes. On current main, configure `hooks.internal.entries.session-memory.outputDir`, trigger `/new` or `/reset`, and the hook still writes under `<workspace>/memory`; for the PR-introduced bug, use an in-workspace symlink prefix in `outputDir` and observe the raw mkdir side effect before the safe writer rejects the final write.

## Solution Assessment

No, not in the current patch shape. The feature boundary is narrow and maintainable, but the implementation should avoid raw pre-write directory creation, drop generated localized docs, and include the required changelog entry before merge.

## Review Findings

Full review comments:

- **[P2] Guard output directory creation with root-aware helpers:** `src/hooks/bundled/session-memory/handler.ts:213`
