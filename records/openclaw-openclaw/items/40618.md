## Summary

Keep this PR open. The requested output-directory feature is useful and has real-world support in the discussion, but current main still writes session-memory snapshots to `memory/`, and the PR branch needs revision before merge because its filesystem write path creates a directory before the root-aware safety helper runs.

## What This Changes

This PR adds optional `hooks.internal.entries.session-memory.outputDir` support for the bundled session-memory hook, updates tests for custom and invalid output paths, and updates hook documentation and metadata.

## Best Possible Solution

Land the optional `outputDir` feature after revising directory creation to use the existing root-aware filesystem helpers or the writer's parent creation, adding a symlink-alias regression test, keeping the default `memory` behavior, and limiting docs changes to English source docs plus bundled hook metadata. Leave broader duplicate-indexing guidance to #54524.

## Reproduction Assessment

Yes. Configure `hooks.internal.entries.session-memory.outputDir`, trigger `/new` or `/reset`, and current main still writes under `<workspace>/memory/`; for the PR-introduced bug, use an in-workspace symlink prefix such as `link/archive` and observe the raw mkdir side effect before the safe writer rejects the file.

## Solution Assessment

No, not in the current patch shape. The feature boundary is narrow and maintainable, but directory creation must stay inside the root-aware helper path and localized docs edits should be removed before merge.
