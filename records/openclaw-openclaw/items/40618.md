## Summary

Keep open. Current main still lacks configurable `session-memory` output paths and the live user comment supports the use case, but this PR is not mergeable as submitted because it is stale/conflicting, regresses newer handler behavior, has a filesystem-safety issue, edits localized docs against policy, and lacks after-fix real behavior proof.

## What This Changes

The PR adds an optional `hooks.internal.entries.session-memory.outputDir` config with tests and docs so session-memory snapshots can be written outside the default workspace `memory/` directory.

## Best Possible Solution

Rebase `outputDir` support onto the current handler, create configured paths only through alias-safe workspace-root helpers, drop localized docs edits, and coordinate scope with https://github.com/openclaw/openclaw/issues/60034 before landing.

## Reproduction Assessment

Yes, at source level. Current main hardcodes `<workspace>/memory` in `saveSessionMemoryNow`, and the PR body plus the live user comment describe the custom-directory scenario clearly.

## Solution Assessment

No, not as submitted. The `outputDir` direction is plausible, but this branch needs a rebase, root-safe directory creation/write handling, localized-doc removal, maintainer scope review, and real workspace proof.
