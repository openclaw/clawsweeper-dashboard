## Summary

## What This Changes

The PR adds an optional `hooks.internal.entries.session-memory.outputDir` setting, tests for custom and invalid directories, and docs for the new session-memory output path behavior.

## Best Possible Solution

Reapply the output-directory idea onto the current handler with root-safe workspace-relative creation and writes, preserve current defaults and filename/reset behavior, remove localized docs edits, and coordinate the broader path-template shape with the open feature request.

## Reproduction Assessment

Yes, at source level. Current main and the latest release still hardcode `<workspace>/memory` and expose no `outputDir` option, while the PR body and live user comment describe the custom-directory need clearly.

## Solution Assessment

No, not as submitted. The direction is plausible, but this branch needs a rebase onto current handler behavior, alias-safe directory handling before any filesystem side effect, localized-doc removal, maintainer config-surface review, and real workspace proof.
