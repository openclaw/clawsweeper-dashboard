## Summary

Keep open. The requested output-directory feature is useful and prior discussion includes a live-workspace workaround that supports the direction, but this branch is not mergeable as submitted because it is behind current session-memory behavior, has a symlink-prefix filesystem side effect, edits localized docs that repo policy excludes, and lacks external real behavior proof.

## What This Changes

The branch adds optional `hooks.internal.entries.session-memory.outputDir` handling, tests, and docs for writing session-memory snapshots under a workspace-relative directory.

## Best Possible Solution

Rebase output-dir support onto the current handler, preserve existing session-memory behavior, use root-aware directory/write handling before any configured-path side effect, drop localized docs edits, and land only after config review plus real workspace proof.

## Reproduction Assessment

Yes. Current-main source shows session-memory always writes under `<workspace>/memory`, and the PR body plus discussion give a concrete custom-directory scenario; I did not run a live gateway workspace.

## Solution Assessment

No, not as submitted. The `outputDir` direction is plausible, but the patch should be rebased onto current behavior, avoid raw mkdir before root-bound validation, and keep localized docs out of this repo.
