## What This Changes

The PR adds an optional `hooks.internal.entries.session-memory.outputDir` setting with docs and unit tests so session-memory snapshots can be written to a workspace-relative directory other than `memory/`.

## Best Possible Solution

Reimplement the output-directory option on the current handler with root-safe directory creation, preserve default output plus current background/local-time/collision semantics, and align the public config surface with the broader path/filename design thread.

## Reproduction Assessment

Yes, at source level. Current main still hardcodes session-memory output under `<workspace>/memory`, and the PR body plus a live-user comment describe the custom-directory need clearly.

## Solution Assessment

No, not as the branch currently stands. A narrow `outputDir` may be acceptable, but it needs to be rebased onto current main, made root-safe before directory creation, and reconciled with the broader artifact-template design issue.
