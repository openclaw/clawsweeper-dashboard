## Summary

Keep open: the requested configurable session-memory output directory is still useful and not implemented on current main, but this PR is conflicting and has blocking correctness, filesystem-safety, docs-policy, and real-behavior-proof issues.

## What This Changes

The PR adds optional `hooks.internal.entries.session-memory.outputDir`, updates hook docs, and adds unit tests for custom and invalid session-memory snapshot directories.

## Best Possible Solution

Reapply output-directory support onto the current handler with root-safe workspace-relative writes, preserve current defaults/background/local-time/collision/`llmSlug` behavior, remove localized docs edits, and keep broader filename-template design coordinated with the related feature request.

## Reproduction Assessment

Yes, at source level. Current main still writes session-memory artifacts under `<workspace>/memory` with no `outputDir`, and the PR body plus a live user comment describe the custom-directory need clearly.

## Solution Assessment

No, not as submitted. The direction is plausible, but this branch needs a rebase onto current handler behavior, alias-safe directory handling before any filesystem side effect, localized-doc removal, and real workspace proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Rebase onto current session-memory behavior:** `src/hooks/bundled/session-memory/handler.ts:207-213`
  - body: This branch computes `workspaceDir` from config only and keeps the old synchronous/UTC/default-LLM/fixed-filename path around the new `outputRoot`, so it would drop current `context.workspaceDir` routing, background writes, local-time collision-safe filenames, and opt-in `llmSlug` behavior. Reapply `outputDir` on top of current main.
  - confidence: 0.91

  - body: Creating `outputRoot` with raw recursive `fs.mkdir` can follow an in-workspace symlink prefix and create directories outside the workspace before the root-bound writer rejects the final file write. Validate alias safety before mkdir, or let the root-scoped writer create parents.
  - confidence: 0.92

  - body: The scoped docs guide says foreign-language docs are maintained outside this repo, so the `docs/zh-CN/**` edits should be removed and the English docs plus hook metadata kept as the source of truth.
  - confidence: 0.9
