## Summary

Keep open. The output-directory direction addresses a real session-memory/memory-search pain point, but the submitted branch is stale against current session-memory behavior, has a filesystem side effect before root-bound validation, edits localized docs that this repo does not maintain, and still lacks external real-behavior proof.

## What This Changes

The PR adds optional `hooks.internal.entries.session-memory.outputDir` handling plus docs and tests so session-memory snapshots can be written to a workspace-relative directory instead of always `memory/`.

## Best Possible Solution

Rebase the optional output-directory feature onto the current handler, preserve existing session-memory semantics, validate/create output paths through root-aware filesystem helpers before any side effect, remove localized docs edits, and add real workspace proof.

## Reproduction Assessment

Yes, at source level. Current main always writes session-memory snapshots under `<workspace>/memory`, and the PR body gives a concrete `outputDir` config path for the desired alternate directory behavior.

## Solution Assessment

No, not as submitted. The `outputDir` direction is useful, but this branch needs a rebase onto current main, safer root-aware filesystem handling, docs-policy cleanup, and real behavior proof before merge.
