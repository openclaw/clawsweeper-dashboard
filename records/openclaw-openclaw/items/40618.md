## Summary

Keep open. The configurable session-memory output directory is still missing from current main and the discussion shows a real workspace-noise problem, but the PR head has a blocking root-boundary filesystem issue plus docs/changelog cleanup before it should merge.

## What This Changes

The PR adds optional `hooks.internal.entries.session-memory.outputDir` handling to the bundled session-memory hook, extends handler tests, and updates English plus localized hook documentation and metadata.

## Best Possible Solution

Land the optional output directory support after routing all directory/file creation through root-aware helpers, keeping English docs and hook metadata as source of truth, adding focused regression coverage, and documenting the feature in `CHANGELOG.md`.

## Reproduction Assessment

Yes, by source inspection. Current main ignores `hooks.internal.entries.session-memory.outputDir` and always writes under `<workspace>/memory`, while the PR-introduced symlink-prefix side effect follows from raw `fs.mkdir` running before `writeFileWithinRoot`.

## Solution Assessment

No, not in the current patch shape. The feature boundary is narrow and maintainable, but the implementation should avoid raw pre-write directory creation, remove localized docs edits, and add the required changelog entry before merge.
