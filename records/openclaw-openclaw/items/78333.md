## What This Changes

## Best Possible Solution

Land or replace the focused status fix so the rendered count uses the freshest safe compaction source, preferably the max of persisted and transcript-observed counts, with focused regression coverage and concrete real-output proof.

## Reproduction Assessment

No, not as a live high-confidence current-main reproduction in this read-only review. Source inspection shows the stale path clearly: current main renders persisted `entry.compactionCount` while transcript fallback hydrates usage only, and the reporter supplied concrete release evidence.

## Solution Assessment
