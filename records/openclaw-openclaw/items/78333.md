## Summary

Keep open: current source still renders the compaction line from persisted session state, and the transcript-backed status fallback only updates token/model usage. The reported release already includes the compaction-end store reconcile path, so that existing path is not enough evidence to close this as implemented.

## What This Changes

`session_status` should report a live or clearly labeled compaction count when an active session transcript already contains compaction records.

## Best Possible Solution

Make the status card use the freshest safe compaction source, preferably the max of persisted count and active/transcript-observed count, while preserving the existing store reconcile behavior.
