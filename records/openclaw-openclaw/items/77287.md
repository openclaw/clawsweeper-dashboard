## Summary

Keep open: the replay-normalization chokepoint is appropriate, but the patch currently treats any trailing single text block equal to the stream-error sentinel as droppable without checking synthetic/error/zero-usage provenance, and the user-facing fix is missing its required changelog entry.

## What This Changes

This PR changes replay history normalization to drop trailing synthetic assistant error/sentinel turns and adds focused replay-history tests for the new trimming behavior.

## Best Possible Solution

Keep the shared normalization fix, but narrow persisted-sentinel trimming to synthetic/error/zero-usage turns, add regression coverage for a real sentinel-text assistant reply, and update changelog/docs before merge.

Yes, at source level. Current main rewrites a trailing empty assistant error turn to the fallback sentinel, session-file repair can persist the same sentinel shape, and the PR's added tests exercise the intended drop behavior without requiring a live provider call.
