## Summary

Keep open: the feature direction is related to the open Telegram processing-indicator request, but this PR is stale against current main and has concrete correctness issues that should block merge.

## What This Changes

The PR adds a waiting-tip controller, config schema/labels/help, tests, and Telegram dispatch wiring to send and delete bilingual tips while replies are pending.

## Best Possible Solution

Implement the accepted processing or waiting indicator inside the current Telegram plugin flow using SDK-facing feedback and delivery helpers, expose only implemented config, and cover the lifecycle with focused Telegram tests plus a changelog entry.

## Reproduction Assessment

Yes. Source inspection reproduces the stale integration because current main no longer has src/telegram and active dispatch lives under extensions/telegram/src; the cleanup race is also clear from the PR path where cleanup cancels before an in-flight sendTip resolves.

## Solution Assessment

No. This is not the best current implementation because it targets the removed Telegram path, bypasses current thread-aware send helpers, and exposes unimplemented config; a plugin-owned implementation against #6946 would be narrower.
