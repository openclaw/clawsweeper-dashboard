## Summary

Keep open. The feature is not implemented on current main and the PR still needs a product decision plus a rebase or replacement because it targets the removed Telegram dispatcher and has concrete lifecycle/config defects.

## What This Changes

The PR adds a configurable Telegram waiting-tip controller with bilingual built-in tips, config schema/labels/help, tests, and dispatch wiring to send and delete a tip while replies are pending.

## Best Possible Solution

Implement the accepted Telegram waiting or processing indicator inside extensions/telegram using SDK-facing delivery helpers, expose only implemented config, and cover send/delete/error lifecycle with focused Telegram tests plus a changelog entry.

## Reproduction Assessment

Yes. Source inspection reproduces the merge blockers: the PR wires a removed src/telegram dispatcher while current main uses extensions/telegram, and the waiting-tip controller can lose the sent handle when cleanup races an in-flight send.

## Solution Assessment

No. This PR is not the best current implementation because it targets stale structure, bypasses current Telegram delivery semantics, and exposes unimplemented custom-tip config; a plugin-owned implementation aligned with #6946 would be narrower.
