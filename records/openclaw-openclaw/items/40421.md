## Summary

Keep open for maintainer decision and rework: current main does not implement bilingual waiting tips, and this PR is stale against Telegram's plugin-owned dispatch path with concrete correctness and config defects.

## What This Changes

The PR adds a core waiting-tip controller, `messages.waitingTips` config metadata, tests, and Telegram dispatch wiring to send and later delete bilingual tips while replies are pending.

## Best Possible Solution

Settle the UX against #6946, then implement the accepted waiting or processing indicator inside the current Telegram plugin flow using SDK-facing feedback/delivery helpers, exposing only implemented config and adding focused tests plus a changelog entry.

## Reproduction Assessment

Yes. The stale integration is reproducible by comparing the PR diff against current main: `src/telegram/bot-message-dispatch.ts` is gone and Telegram dispatch now lives under `extensions/telegram/src`. The cleanup race is also reproducible by letting the tip timer fire, keeping `sendTip` in flight, and calling `cleanup()` before the send resolves.

## Solution Assessment

No. The proposed branch is not the best current solution because it targets the old Telegram path, bypasses current thread-aware delivery helpers, and exposes unimplemented config; a plugin-owned implementation would be narrower and more maintainable.
