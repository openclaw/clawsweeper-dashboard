## Summary

Keep open. Current main does not implement waiting tips, but this PR is not merge-ready: it targets the pre-plugin Telegram path, has a confirmed cleanup race, advertises an unimplemented `customTipsPath`, and misses generated config artifacts.

## What This Changes

The PR adds a waiting-tip controller, Telegram dispatch integration, config/schema/help labels, and tests to show then delete bilingual tips while a Telegram reply is pending.

## Best Possible Solution

If maintainers want this direction, rebase it into the Telegram plugin, align the UX with #6946, fix cleanup and custom-tip semantics, regenerate config/docs artifacts, and add focused lifecycle tests plus a changelog entry.

## Reproduction Assessment

Yes. The cleanup race can be reproduced with fake timers by letting `minWaitMs` fire, keeping `sendTip` pending, calling `cleanup()` or `cancel()`, then resolving `sendTip`; the proposed code discards the handle and never calls `deleteTip`. Current-main inspection also confirms the requested waiting-tip surface is absent.

## Solution Assessment

No. The current PR is not the best merge path because it targets the old Telegram location, exposes a no-op custom-tip config, and has a temporary-message cleanup race; a safer implementation would live in `extensions/telegram` and either implement or omit custom tips until they work.
