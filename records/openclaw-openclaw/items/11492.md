## Summary

Keep open. Current main still lacks Slack `channel-info` and `channel-list` as agent-facing message actions, and the linked #53199 implementation candidate is open and appears incomplete for the requested `channel-info` half.

## What This Changes

The issue requests Slack `channel-info` and `channel-list` message actions so agents can resolve Slack channel IDs to names and metadata.

## Best Possible Solution

Expose both requested Slack actions in the Slack plugin, with discovery, dispatch, runtime helpers around Slack conversations APIs, docs, and focused tests; review #53199 as a partial candidate before closing this issue.

## Reproduction Assessment

Yes. A static current-main check of Slack action discovery, dispatch, runtime helpers, config, and docs shows the requested agent-facing actions are absent, so no live Slack workspace is needed to confirm the gap.

## Solution Assessment

Yes. Implementing this inside `extensions/slack` is the narrowest maintainable path because Slack-owned message-tool behavior belongs in the Slack plugin and the generic action names already exist in the channel action contract.
