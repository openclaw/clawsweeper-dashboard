## Summary

Keep open: current main still does not expose Slack `channel-info` or `channel-list` as agent-facing message actions. Slack has private `conversations.info`/`conversations.list` usage, but the requested discovery, dispatch, runtime helper, export, docs, and test surfaces are still missing; the open Slack parity PR is the likely path but has not merged.

## What This Changes

The issue requests Slack `channel-info` and `channel-list` message actions so agents can resolve Slack channel IDs and enumerate workspace conversations.

## Best Possible Solution

Land or split the Slack plugin implementation so `extensions/slack` exposes both actions through discovery, dispatch, Slack API helpers, focused tests, and Slack docs under the agreed action gate.
