## Summary

Current main and latest release still omit Slack `channel-info` and `channel-list` from agent-facing Slack message actions, so the requested capability remains open; the linked open fork PR is an implementation candidate, not an implemented fix.

## What This Changes

The issue requests Slack `channel-info` and `channel-list` message actions so agents can resolve Slack channel IDs to names and channel metadata.

## Best Possible Solution

Keep this issue open until a reviewed Slack plugin change exposes both `channel-info` and `channel-list` through discovery, dispatch, runtime helpers, docs, and focused tests, preferably using the existing `channelInfo` gate unless maintainers approve a separate config surface.

## Reproduction Assessment

Yes. Source inspection against current main shows Slack discovery, dispatch, runtime helpers, tests, and docs omit `channel-info` and `channel-list`; this proves the feature gap without needing a live Slack workspace.

## Solution Assessment

Yes, the maintainable direction is to add the actions inside `extensions/slack` rather than relying on internal context resolvers or direct `curl` calls. A separate `channelList` config flag should be a maintainer decision because the existing Slack `channelInfo` gate already exists.
