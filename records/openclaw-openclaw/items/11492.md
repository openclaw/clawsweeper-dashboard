## Summary

Keep this open: current main and the latest release still do not expose Slack `channel-info` or `channel-list` as agent-facing message actions, and the related same-repo PRs are either closed unmerged or an open broad human-review candidate rather than a merged replacement.

## What This Changes

Request to add Slack `channel-info` and `channel-list` message actions so agents can resolve Slack channel IDs to human-readable channel metadata and workspace channel lists.

## Best Possible Solution

Keep this issue open until the Slack plugin lands both `channel-info` and `channel-list`, either by repairing the broad parity PR or by splitting a narrower channel-discovery PR with focused tests, docs, and real Slack proof.

## Reproduction Assessment

Yes, at source level. Current main's Slack discovery omits `channel-info` and `channel-list`, and the dispatcher would fall through to the unsupported-action error for those action names.

## Solution Assessment

Yes on owner boundary, but not yet on merge path. The maintainable solution belongs in `extensions/slack` using the existing message-action contract and Slack action gates; the current broad PR still needs maintainer review, fixes, or a narrower split.
