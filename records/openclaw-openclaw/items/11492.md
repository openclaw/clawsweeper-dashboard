## Summary

Keep open: current main still lacks Slack `channel-info` and `channel-list` as agent-facing message actions, and the linked #53199 implementation candidate is open, unmerged, and appears partial because it covers `channel-list` plus `search` rather than both requested actions.

## What This Changes

Request Slack message-tool actions `channel-info` and `channel-list` so agents can resolve Slack channel IDs and list workspace conversations.

## Best Possible Solution

Complete this in `extensions/slack` by wiring both `channel-info` and `channel-list` through discovery, dispatch, runtime helpers for Slack `conversations.info`/paginated `conversations.list`, focused tests, and Slack docs under the existing `channelInfo` gate unless maintainers intentionally choose a different gate.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main verifies the requested Slack actions are absent from discovery, dispatch, runtime helpers, tests, and docs.

## Solution Assessment

No, current main is not a solution yet, and the linked #53199 candidate is not sufficient if it only adds `channel-list` and `search`. The narrow maintainable path is Slack-owned wiring for both requested actions with contract/runtime coverage.
