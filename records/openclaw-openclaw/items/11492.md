## Summary

Keep this issue open: current main still does not expose either requested Slack action, and no merged or viable complete pull request owns the work. The remaining fork PR is conflicting, omits `channel-info`, and bundles a separate search feature.

## What This Changes

Add Slack `channel-info` and `channel-list` message actions so agents can resolve channel IDs and enumerate workspace conversations.

## Best Possible Solution

Implement both actions through the existing Slack plugin action registry and dispatcher, use a bounded paginated list contract, preserve one intentional action gate, and cover discovery, dispatch, Slack API response normalization, documentation, and a redacted live Slack call.

## Reproduction Assessment

Yes at source level. Current main's Slack registry, dispatcher, action helpers, tests, and documentation omit both actions, so agents cannot invoke them; no live Slack workspace is needed to establish that absence.

## Solution Assessment

Yes, the existing Slack plugin message-action boundary is the narrowest maintainable implementation path. The current channel-list/search fork is not the best complete solution because it omits channel-info, mixes another feature, and is conflicting.
