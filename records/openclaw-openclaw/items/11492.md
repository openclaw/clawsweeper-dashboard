## Summary

Current `main` still lacks both Slack discovery actions, and neither closed in-repo PR landed the requested behavior. The feature fits the bundled Slack plugin, but maintainers should settle its read-action gate and credential semantics before adopting a focused implementation; the remaining open fork PR is dirty and implements only `channel-list` plus a separate search feature.

## What This Changes

Add Slack message-tool actions that let agents inspect a conversation and enumerate workspace conversations so channel IDs can be resolved to human-readable metadata.

## Best Possible Solution

Add both actions within `extensions/slack`, using shared paginated conversation helpers and the active Slack account credential contract, then cover action discovery, dispatch, runtime results, pagination, permissions/errors, and docs; reuse an existing action group unless maintainers explicitly approve a new config gate.

## Reproduction Assessment

Yes at source level: current Slack action discovery, dispatch, runtime exports, tests, and docs all omit both requested actions. This establishes the missing capability without requiring a live failure reproduction.

## Solution Assessment

The plugin-local implementation direction is the best fit, but the exact proposal is not ready as written because current `main` has no existing Slack `channelInfo` gate and credential behavior is not specified. A focused implementation should settle those contracts rather than copy either closed PR unchanged.
