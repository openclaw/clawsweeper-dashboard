## Summary

Keep this issue open: current main and the latest release still omit Slack `channel-info` and `channel-list` from agent-facing Slack message actions, and the linked open fork PR is conflicting and appears to cover only `channel-list` plus `search`, not the full requested parity.

## What This Changes

Request to add Slack message-tool `channel-info` and `channel-list` actions so agents can resolve Slack channel IDs and list visible conversations.

## Best Possible Solution

Implement both Slack actions inside `extensions/slack`, reuse the existing `channelInfo` gate unless maintainers choose a separate config contract, and update focused action/runtime tests plus Slack docs before closing this issue.

## Reproduction Assessment

Yes. Source inspection is high-confidence: current Slack discovery, dispatch, runtime helpers, public exports, tests, and docs omit `channel-info` and `channel-list` even though generic message action names and Discord support them.

## Solution Assessment

Yes, the narrow maintainable direction is plugin-local Slack action support rather than agents calling Slack directly; the current linked PR is only a partial candidate until it includes `channel-info` and resolves the action-gating choice.
