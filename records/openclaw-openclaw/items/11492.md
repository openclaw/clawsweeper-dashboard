## Summary

Keep this issue open: current main still does not expose Slack `channel-info` or `channel-list` as agent-facing message actions, and the only current implementation path is a broad open PR that is conflicting and under human review rather than merged or safely superseding the issue.

## What This Changes

The issue requests Slack `channel-info` and `channel-list` message actions so agents can resolve Slack channel IDs into human-readable channel names and metadata.

## Best Possible Solution

Implement both requested actions inside the Slack plugin action surface, or land a reviewed Slack action parity PR that includes channel metadata lookup and paginated channel listing with docs, tests, and explicit token/read-boundary behavior.

## Reproduction Assessment

Yes, at source level: current main’s Slack message action discovery, dispatch, runtime helpers, public exports, docs, and tests omit `channel-info` and `channel-list` while Discord exposes both behind `channelInfo`. No live Slack workspace is needed to prove the missing action surface.

## Solution Assessment

Yes on owner boundary, but not yet on implementation path. The best fix belongs in `extensions/slack`; the broad open parity PR may be a candidate only after conflict refresh, proof, and security/product blockers are resolved.
