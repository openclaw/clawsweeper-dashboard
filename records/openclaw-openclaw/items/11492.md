## Summary

Keep open: current main and v2026.5.20 still do not expose Slack `channel-info` or `channel-list` as agent-facing message actions. The active Slack parity PR at https://github.com/openclaw/openclaw/pull/79818 covers this surface, but it is unmerged and maintainer-labeled, so this issue should remain open until both requested actions land or the channel-discovery subset is split into a narrower canonical PR.

## What This Changes

The issue requests Slack message-tool actions that let agents resolve Slack channel IDs to names and list Slack conversations through Slack conversation APIs.

## Best Possible Solution

Finish or split https://github.com/openclaw/openclaw/pull/79818 so the Slack plugin advertises and dispatches both requested actions, implements Slack SDK helpers with pagination where needed, and updates config metadata, docs, and focused tests within the plugin boundary.
