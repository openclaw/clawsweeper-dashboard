## Summary

Keep open. Current main still does not expose the requested Slack `channel-info` or `channel-list` message actions, and the linked #53199 implementation candidate is open/unmerged and appears partial for this issue because it clearly covers `channel-list` plus `search`, not `channel-info`.

## What This Changes

The issue requests Slack `channel-info` and `channel-list` message actions so agents can resolve Slack channel IDs to human-readable metadata and list available conversations.

## Best Possible Solution

Expose both requested actions in the Slack plugin: add `channel-info` and `channel-list` to Slack message action discovery, map them in `handleSlackMessageAction`, add focused runtime helpers around `conversations.info` and paginated `conversations.list`, gate them consistently with `channelInfo` unless maintainers choose a new documented gate, and update Slack docs plus focused tests. Review #53199 as a candidate, but do not close until `channel-info` is included or a narrower canonical follow-up tracks that missing half.

## Reproduction Assessment

Yes. This is a feature gap rather than a runtime-only bug: a static current-main check of Slack action discovery, dispatch, runtime helpers, tests, and docs shows the requested actions are absent, so no live Slack workspace is needed to reproduce the missing surface.

## Solution Assessment

Yes, the requested plugin-level implementation is the narrowest maintainable solution because Slack-owned behavior belongs in `extensions/slack` and the generic channel action names already exist. The linked implementation candidate should be treated as incomplete unless it adds `channel-info` or maintainers explicitly split that work into a separate canonical issue.
