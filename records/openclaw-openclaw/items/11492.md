Add Slack `channel-info` and `channel-list` message actions so agents can resolve channel IDs and discover workspace channels.

## Best Possible Solution

Implement both actions in `extensions/slack` through the existing discovery, dispatcher, runtime, and API seams; paginate `conversations.list`, add focused contract/runtime coverage and Slack documentation, and avoid a new config key unless maintainers intentionally require separate gating.

## Solution Assessment
