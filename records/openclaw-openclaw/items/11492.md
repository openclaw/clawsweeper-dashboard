## Summary

Keep open: current main still does not expose Slack `channel-info` or `channel-list` as agent-facing message actions, and the linked open implementation candidate appears partial because it covers `channel-list` plus `search` rather than both requested actions.

## What This Changes

Request adds Slack message-tool `channel-info` and `channel-list` actions so agents can resolve Slack channel IDs and enumerate workspace channels.

## Best Possible Solution

Complete this in `extensions/slack` by wiring both `channel-info` and `channel-list` through discovery, dispatch, runtime helpers around Slack `conversations.info`/paginated `conversations.list`, focused tests, and Slack docs under the existing `channelInfo` gate unless maintainers intentionally choose a separate gate.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main verifies the requested Slack actions are absent from discovery, dispatch, runtime helpers, tests, and docs.

## Solution Assessment

No, current main is not a solution yet, and the linked open PR appears incomplete if it only adds `channel-list` and `search`. The narrow maintainable fix is Slack-owned wiring for both requested actions with contract/runtime coverage.
