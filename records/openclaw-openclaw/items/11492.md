## Summary

Keep this open: current main and v2026.6.10 still omit Slack `channel-info` and `channel-list` as agent-facing message actions, while the related implementation attempts are closed unmerged or an open broad PR that is dirty and human-review blocked.

## What This Changes

The issue asks the Slack plugin to expose agent-facing `channel-info` and `channel-list` message actions so agents can resolve Slack channel IDs to human-readable channel metadata.

## Best Possible Solution

Land or split a focused Slack plugin change that exposes both `channel-info` and `channel-list` through discovery, dispatch, runtime helpers, docs, focused tests, and redacted real Slack proof while preserving Slack read-target and token boundaries.

## Reproduction Assessment

Yes, at source level. Current main's Slack discovery omits `channel-info` and `channel-list`, and the dispatcher would reject those actions as unsupported.

## Solution Assessment

Yes on owner boundary, but not yet on merge path. The implementation belongs in `extensions/slack`, while the current broad related PR needs maintainer review or a narrower split before it can be the best landing path.
