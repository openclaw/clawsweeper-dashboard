Keep open: current main still does not expose Slack `channel-info` or `channel-list` as message-tool actions, and the open implementation candidate only covers `channel-list` plus `search`, not the requested `channel-info` half.

The issue requests Slack message-tool actions for resolving channel IDs and listing channels via Slack `conversations.info` and `conversations.list`, gated through the existing Slack `channelInfo` action flag.

## Best Possible Solution
