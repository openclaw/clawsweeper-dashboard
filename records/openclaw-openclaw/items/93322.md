## Summary

Keep open: current main still lets the shared status path choose provider usage from the active runtime model, so stale DeepSeek runtime fields can still disagree with a session-selected OpenAI model, and an open linked PR now owns the focused fix path.

## What This Changes

The issue asks for `/status` and `session_status` usage to follow the session-selected model/provider after `/model` switches instead of showing the configured or stale runtime provider's usage.

## Best Possible Solution

Review and land one focused selected-provider status fix, preferably the resolver-level path in https://github.com/openclaw/openclaw/pull/93384, while keeping the broader session model/auth refactor in https://github.com/openclaw/openclaw/issues/87957 separate.

## Solution Assessment
