## Summary

Keep open. Current main still has only partial cwd/workspace breadcrumbs, not the requested durable per-session working-context capsule across compaction/new flows; the status surface and docs also do not expose or explain the requested task-repo/runtime-cwd/branch/temp-clone/push-target distinction. The issue discussion adds concrete WSL/cloud/Docker and external STATE.json examples, and the related PR #71135 is still open, draft, and described as storage-only rather than a shipped behavioral fix.

## What This Changes

The issue requests a persistent per-session working-context capsule that survives compaction/new-session flows, is updated from runtime activity, appears in status, and is documented separately from semantic conversation continuity.

## Best Possible Solution

Keep this issue as the product/API tracker. Land a versioned session working-context shape only with follow-up runtime wiring that updates it from exec/tool/subagent/ACP/compaction/new-session paths, exposes the current values in status, and updates the workspace plus compaction docs to separate semantic continuity from working-state continuity.

## Reproduction Assessment

Yes. The current-main gap is reproducible by source inspection: the requested field names have no implementation, SessionEntry only has partial cwd/workspace breadcrumbs, and status/docs do not expose the requested working-context state.

## Solution Assessment

No current-main solution exists yet. The related draft PR looks like a possible first storage step, but the maintainable fix still needs runtime updates, status rendering, and docs terminology changes before the reported problem is solved.
