## Summary

Keep open. Current main still has only initial cwd/session breadcrumbs and spawned-workspace metadata; it does not persist or expose the requested per-session working context, and the linked implementation path remains an open draft with unresolved schema, privacy, and integration scope.

## What This Changes

Request persistent per-session working context across `/compact`, `/new`, subagent, and temp-clone workflows, with `/status` and docs updates.

## Best Possible Solution

Design a private, versioned working-context model and wire it through runtime updates, `/status`, `/new`, compaction successor creation, subagent metadata, and workspace/compaction docs.

## Reproduction Assessment

Yes. The gap is reproducible by source inspection: the requested capsule symbols are absent, `SessionEntry` lacks a working-context field, `/status` does not render the requested state, and reset/compaction paths carry only existing transcript/session breadcrumbs.

## Solution Assessment

No current-main solution exists. The draft PR is a plausible storage primitive, but the maintainable solution still needs maintainer decisions on private schema, retention/redaction, update lifecycle, and where status, compaction, and subagent wiring belong.
