## Summary

Keep open. Current main still lacks a durable per-session working-context capsule across compact/new/subagent flows, `/status` exposes only the session key/activity and compaction count, and the only matching implementation candidate is still an open draft PR that explicitly adds storage only without runtime, status, compaction, or subagent wiring.

## What This Changes

The issue requests a persisted per-session working-context object, status display, compaction/new-session preservation, and clearer workspace/task-repo/runtime-cwd terminology.

## Best Possible Solution

Design a private, versioned working-context model and wire it through runtime updates, reset/new, compaction rotation, subagent spawn metadata, `/status`, and the workspace/compaction docs.

## Reproduction Assessment

Yes. The gap is reproducible by source inspection: the requested symbols are absent, SessionEntry only carries partial breadcrumbs, the transcript header stores only initial cwd, and `/status` does not expose repo/cwd/branch/temp-clone/push-target state.

## Solution Assessment

No current-main solution exists. The linked draft PR is a plausible storage primitive, but it is intentionally incomplete and still needs schema, privacy, lifecycle, status, and docs decisions before it solves the issue.
