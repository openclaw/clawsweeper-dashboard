## Summary

Keep open. Current main still does not implement a durable per-session working-context capsule or expose cwd/repo/branch/temp-clone/push-target state in `/status`, and the only direct implementation candidate remains an open draft PR that intentionally adds storage only without runtime, status, compaction, or subagent wiring.

## What This Changes

Request to persist session-level working context across compact/new/subagent flows and expose or clarify it in `/status` and docs.

## Best Possible Solution

Design a private, versioned working-context model and integrate it across runtime updates, `/new`, compaction rotation, subagent metadata, `/status`, and the workspace/compaction docs.

## Reproduction Assessment

Yes. The gap is reproducible by source inspection: current main has no working-context capsule fields, `/status` does not render the requested state, and reset/compaction paths only preserve existing session/transcript metadata.

## Solution Assessment

No current-main solution exists. The linked draft PR is a plausible storage primitive, but the maintainable solution still needs schema, privacy, lifecycle, status, subagent, compaction, and docs decisions.
