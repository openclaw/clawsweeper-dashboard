## Summary

Keep open. Current main still lacks the requested durable per-session working-context capsule across compact/new/subagent flows; `/status` and docs only expose partial cwd/workspace breadcrumbs, and the directly related implementation PR is still an open draft storage-only proposal. The supporting comments add real-world confirmation that this is a valid product pain point.

## What This Changes

The issue requests a durable per-session working-context capsule that preserves repo/cwd/branch/temp-clone/push-target state across compact, new-session, and subagent-heavy workflows, with status and docs updates.

## Best Possible Solution

Keep this as the product/API tracker and land a versioned working-context model only when it is wired through runtime updates, status rendering, compact/new preservation, subagent metadata, and terminology docs.

## Reproduction Assessment

Yes. The current-main gap is reproducible by source inspection: the requested fields and capsule are absent, SessionEntry only has partial cwd/workspace breadcrumbs, and status/docs do not expose the requested working-context state.

## Solution Assessment

No current-main solution exists yet. The linked draft PR is a plausible storage primitive, but the maintainable fix still needs schema/privacy approval plus lifecycle wiring, status output, and docs before it solves the reported problem.
