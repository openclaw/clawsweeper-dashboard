## Summary

Keep open. Current main still has only partial cwd/workspace breadcrumbs, not the durable per-session working-context capsule requested across compact/new/subagent flows; /status and the docs also do not expose the task-repo/runtime-cwd/branch/temp-clone/push-target distinction, and related PR #71135 is still an open draft storage-only MVP.

## What This Changes

Persist a structured per-session working-context capsule across compact/new/subagent/temp-clone flows, expose it in /status, and clarify workspace/task-repo/runtime-cwd terminology in docs.

## Best Possible Solution

Keep this as the product/API tracker and land a versioned workingContext model only when it is wired through runtime updates, status rendering, compact/new preservation, subagent metadata, and terminology docs.

## Reproduction Assessment

Yes. The current-main gap is reproducible by source inspection: the requested fields have no implementation, SessionEntry only has partial cwd/workspace breadcrumbs, and status/docs do not expose the requested working-context state.

## Solution Assessment

Unclear. The requested direction is sound, but a storage-only primitive is not enough; the maintainable fix needs an approved schema plus lifecycle wiring before it solves the reported problem.
