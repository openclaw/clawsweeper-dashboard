## Summary

Keep open. Current main still lacks a durable structured working-context model and does not expose the requested cwd/repo/branch/temp-clone state in `/status`, while the related draft PR at https://github.com/openclaw/openclaw/pull/71135 remains open and explicitly limits itself to an unwired storage primitive; confirming user comments support that this is still a real workflow gap.

## What This Changes

The issue requests durable per-session working context across `/compact`, `/new`, subagent, and temp-clone workflows, plus `/status` and docs updates to distinguish workspace, task repo, and runtime cwd.

## Best Possible Solution

Design a private, versioned working-context model and wire it through runtime updates, `/status`, `/new`, compaction successor creation, subagent metadata, and the workspace/compaction docs, preferably coordinated with the existing draft PR.

## Reproduction Assessment

Yes for the feature gap: current-main source inspection shows no structured working-context capsule, `/status` lacks the requested cwd/repo/branch state, and `/new`/compaction/subagent paths only carry transcript cwd or spawned-workspace breadcrumbs. This is not a strict bug reproduction because the request adds new durable behavior.

## Solution Assessment

No complete current-main solution exists. The related draft PR is a plausible storage primitive, but the maintainable solution still needs maintainer decisions on private schema, storage location, update lifecycle, retention/redaction, and integration points.
