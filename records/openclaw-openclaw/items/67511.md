## Summary

Keep open. Current main still has only one-time cwd/session breadcrumbs and spawned-workspace metadata; it does not persist or expose the requested structured working context, and the linked draft implementation remains open and intentionally incomplete.

## What This Changes

Request to persist a structured per-session working context across `/compact`, `/new`, subagent, and temp-clone flows and expose or clarify it in `/status` and docs.

## Best Possible Solution

Design a private, versioned working-context model and wire it through runtime updates, `/status`, `/new`, compaction successor creation, subagent metadata, and the workspace/compaction docs, preferably coordinated with the existing draft PR.

## Reproduction Assessment

Yes. Source inspection on current main shows `SessionEntry` lacks a working-context field, `/status` does not render the requested state, and reset/compaction/subagent paths carry only existing transcript or spawned-workspace breadcrumbs.

## Solution Assessment

No complete current-main solution exists. The linked draft PR is a plausible storage primitive, but the maintainable solution still needs maintainer decisions on private schema, retention/redaction, update lifecycle, and integration points.
