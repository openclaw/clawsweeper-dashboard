## Summary

Keep open. Current main still calls `decodeURIComponent` directly on Matrix `geo_uri` parameter values, so the reported malformed percent-encoding path remains valid; the PR patch is narrow and functionally sound, but it is missing the required changelog entry for a user-facing fix.

## What This Changes

The PR wraps Matrix `geo:` URI parameter decoding so malformed percent-encoding returns an invalid location, and adds focused Matrix location parser tests.

## Best Possible Solution

Keep the narrow parser guard and regression tests, add a single Unreleased Matrix changelog entry crediting the contributor, then run the targeted Matrix test plus formatter/lint and the normal changed gate before merge.

## Reproduction Assessment

Yes. Current main can be reproduced by calling the inbound Matrix location parser path with `geo_uri: "geo:45.5,9.2;u=%"`; `decodeURIComponent('%')` throws `URIError`, and the monitor calls this parser for inbound Matrix location events.

## Solution Assessment

Yes, with one process fix. Returning `null` when decoding a Matrix `geo:` parameter fails is the narrowest maintainable behavior for invalid inbound location data, and the added tests cover both valid encoded accuracy and malformed encoding; the missing changelog entry should be added before merge.
