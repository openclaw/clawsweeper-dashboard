## Summary

Keep open: current main and v2026.6.6 still omit event-time text from the cron failure alert bodies, and the linked implementation PR is open, draft, conflicting, and unmerged.

## What This Changes

The issue requests adding the actual cron event time to cron failure alert bodies so delayed delivery does not look like a fresh failure.

## Best Possible Solution

Refresh, review, and land or replace https://github.com/openclaw/openclaw/pull/80246 so the timestamp change reaches the existing cron alert paths without adding job config schema.

## Reproduction Assessment

Yes for source-level behavior: current main’s single-failure, repeated-failure, and auto-disable alert text paths omit event-time text. I did not run a live delayed-delivery scenario, and this is an additive alert-content feature rather than a strict broken-contract bug.

## Solution Assessment

Yes in direction: adding a formatter and plumbing event time at the existing cron alert boundaries is the maintainable shape. The current open PR needs refresh and maintainer compatibility review before this issue can close.
