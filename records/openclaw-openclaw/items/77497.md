## Summary

Keep open. Current main still omits the cron event time from channel failure-alert text, while the cron event and webhook paths already carry `runAtMs`, so this is a narrow additive enhancement rather than an already-implemented cleanup.

## What This Changes

Add the cron event timestamp to single-failure, repeated-failure, skipped-run, and auto-disable cron failure alert bodies.

## Best Possible Solution

Add a shared, deterministic timestamp formatter for cron alert bodies and thread the existing run timestamp into the affected alert construction paths without changing job schema or routing semantics.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high confidence that current alert bodies omit event time while cron events already carry `runAtMs`.

## Solution Assessment

Yes, the requested additive message change is the narrowest maintainable direction. It should reuse existing run timestamps and current delivery paths rather than introduce a new alert schema or config option.
