## Summary

Keep open: current main and the latest shipped release still omit event-time text from the cron failure alert bodies named in the report, and live GitHub shows an open draft implementation PR that should be resolved before this issue closes.

## What This Changes

The issue requests adding the cron event timestamp to failure and auto-disable alert bodies so delayed channel delivery is not mistaken for a fresh failure.

## Best Possible Solution

Land or replace the open draft PR with a narrow formatter and plumbing change that uses existing cron timezone semantics, updates focused cron/gateway coverage, and leaves routing and job schema unchanged.

## Reproduction Assessment

Yes for the source-level gap: current main still builds the single-failure, repeated-failure, and auto-disable text without event-time wording, while the single-failure payload already carries `runAtMs`. It is not a strict bug reproduction because this is an additive alert-content request.

## Solution Assessment

Yes, keeping this issue open behind the linked implementation path is the best current resolution. The maintainable fix is timestamp formatting and plumbing at the existing cron alert boundary, using the scheduler's timezone contract rather than adding a new config surface.
