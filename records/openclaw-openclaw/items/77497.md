## Summary

Keep open: current main and the latest release still omit event-time text from the cron failure alert bodies named in the report, and https://github.com/openclaw/openclaw/pull/80246 is the still-open candidate fix. Because that PR uses closing syntax but remains draft/conflicting, this issue should stay open until that implementation lands, is replaced, or is explicitly closed by maintainers.

## What This Changes

The issue requests adding the cron event timestamp to single-failure, repeated-failure, skipped-run, and auto-disable alert bodies so delayed delivery is not mistaken for a fresh failure.

## Best Possible Solution

Refresh, land, or replace https://github.com/openclaw/openclaw/pull/80246 with a bounded formatter/plumbing change at the current cron and gateway alert boundaries, with explicit maintainer handling for webhook message compatibility.

## Reproduction Assessment

Yes for source-level behavior: current main's single-failure, repeated-failure, skipped-run, and auto-disable message templates omit event-time text. I did not run a live delayed-delivery channel scenario in this read-only review.

## Solution Assessment

Yes in direction, but not complete on main. The maintainable solution is a bounded timestamp formatter and plumbing at the existing cron/gateway alert boundaries, with webhook message compatibility reviewed before merge.
