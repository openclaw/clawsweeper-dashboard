The issue requests adding the cron run/event timestamp to single failure, repeated failure, and auto-disable notification bodies, preferably using existing run metadata and the cron schedule timezone.

## Best Possible Solution

Land or replace #77560 with a narrow implementation that formats existing event time metadata at the cron alert boundary and updates the focused alert tests.
