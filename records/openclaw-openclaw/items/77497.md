## Summary

Keep open: current main still emits cron failure and auto-disable notifications without event-time text, and open PR #77560 is the active closing implementation candidate rather than a reason to close this issue before merge.

## What This Changes

Add event timestamps to cron single-failure, repeated-failure, and auto-disable alert bodies so delayed channel delivery is not mistaken for failure time.

## Best Possible Solution

Land a reviewed implementation such as #77560 that formats run timestamps at the cron alert boundary using existing run metadata and timezone helpers, with regression coverage for all affected alert bodies.

## Reproduction Assessment

Not applicable as a feature request. Source inspection is high-confidence: the current alert builders omit event-time text while cron events and run logs already carry `runAtMs`.

## Solution Assessment

Yes, the narrowest maintainable direction is additive timestamp formatting and timestamp plumbing in the existing cron alert path, without changing routing or job schema.
