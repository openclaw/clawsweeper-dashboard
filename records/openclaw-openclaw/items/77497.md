## Summary

Keep open: current main and v2026.6.8 still omit event-time text from the cron failure alert bodies named in the report, and https://github.com/openclaw/openclaw/pull/80246 is the still-open candidate fix.

## What This Changes

Request to include the cron event timestamp in single-failure, repeated-failure, and schedule-error auto-disable alert bodies so delayed channel delivery is not mistaken for event time.

## Best Possible Solution

Refresh and land or replace https://github.com/openclaw/openclaw/pull/80246 with a bounded formatter/plumbing change that uses existing cron event time and timezone semantics, updates focused cron/gateway coverage, and avoids new job config schema.

## Reproduction Assessment

Yes for the source-level behavior: current main's single-failure, repeated-failure, and auto-disable message templates omit event-time text. This is an additive alert-content request rather than a strict broken-contract bug.

## Solution Assessment

Yes, the maintainable solution is to add timestamp formatting and plumbing at the existing cron alert boundaries. The open PR is the current implementation path, but it needs refresh and maintainer compatibility review before this issue can close.
