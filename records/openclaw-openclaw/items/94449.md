## Summary

## What This Changes

This PR updates the local exec workdir resolver to expand leading `~` against the OS home before directory validation and adds a regression test for `HOME` versus `OPENCLAW_HOME`.

## Best Possible Solution

Land one canonical fix for https://github.com/openclaw/openclaw/issues/94434; if maintainers choose expansion-only behavior, this OS-home implementation is the strongest current candidate, otherwise continue with fail-fast or combine both behaviors.

## Reproduction Assessment

## Solution Assessment

Yes as an expansion-only fix, but unclear as the final product choice. The PR uses the shared home-path helper and focused test coverage; the main alternative is the fail-fast behavior proposed in https://github.com/openclaw/openclaw/pull/94441.
