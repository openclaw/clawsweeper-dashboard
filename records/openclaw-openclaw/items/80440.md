## Summary

Keep open: the PR addresses a real OpenRouter music-generation gap, but the current patch is not mergeable because it uses an unguarded provider HTTP path, has chunk-unsafe SSE parsing, and lacks real after-fix proof.

## What This Changes

The PR adds an OpenRouter music-generation provider, registers it in the OpenRouter plugin and manifest, and updates the bundled media provider contract test.

## Best Possible Solution

Keep the OpenRouter music provider work on this PR path, but make the streaming POST use the shared guarded HTTP helper, make SSE parsing chunk-safe, add focused coverage, and require redacted real-run proof before merge.

## Reproduction Assessment

Not applicable as a feature PR. Source inspection on current main confirms OpenRouter has image, video, media-understanding, and speech registration but no music-generation provider registration.

## Solution Assessment

No. The direction is reasonable, but the implementation should use OpenClaw's guarded provider HTTP path and preserve pending SSE event state across reads before it is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Use the guarded provider HTTP path:** `extensions/openrouter/music-generation-provider.ts:105-109`
  - body: This streaming POST goes through bare fetch after resolving an operator-configurable OpenRouter base URL, so provider request policy, dispatcher/proxy handling, and SSRF/private-network controls are not enforced. Route the request through fetchWithTimeoutGuarded or an equivalent shared helper that forwards allowPrivateNetwork, dispatcherPolicy, and audit context.
  - confidence: 0.91
- **[P2] Keep SSE data lines across reads:** `extensions/openrouter/music-generation-provider.ts:54`
  - body: dataLines is recreated on each reader.read() iteration, so an event whose data line and blank delimiter arrive in different chunks is dropped. Stream chunk boundaries are arbitrary, so this can lose audio deltas or produce missing/truncated music output.
  - confidence: 0.86
