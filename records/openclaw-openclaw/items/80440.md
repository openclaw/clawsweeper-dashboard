## Summary

Keep open: current main still lacks OpenRouter music generation, but the latest PR head is not mergeable because it still uses an unguarded provider POST, has chunk-unsafe SSE parsing, and lacks contributor-provided real behavior proof.

## What This Changes

The branch adds an OpenRouter music-generation provider, registers it in the OpenRouter plugin entry and manifest, and updates the bundled music provider contract expectation.

## Best Possible Solution

Keep the PR open, route OpenRouter music submission through the guarded provider HTTP path, make SSE parsing chunk-safe, add focused provider coverage, and require redacted live music_generate proof before merge.

## Reproduction Assessment

Not applicable as a feature PR. Source inspection confirms current main lacks OpenRouter music registration in both the plugin entrypoint and manifest, but no current-main bug reproduction was attempted or needed.

## Solution Assessment

No. The feature direction is plausible, but this implementation should use the shared guarded provider HTTP path and robust SSE event parsing before it is the maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Route the POST through guarded provider HTTP:** `extensions/openrouter/music-generation-provider.ts:105-110`
  - body: The provider resolves an operator-configurable OpenRouter base URL, but streamOpenRouterMusic submits it with bare fetch. That bypasses the shared provider request guard, dispatcher/proxy policy, and private-network controls used by the adjacent OpenRouter image/video providers; submit the streaming POST through postJsonRequest/fetchWithTimeoutGuarded or an equivalent guarded helper that preserves the response body until it is consumed.
  - confidence: 0.92
- **[P2] Keep SSE data lines across reads:** `extensions/openrouter/music-generation-provider.ts:54`
  - body: dataLines is recreated on every reader.read() loop, so if a chunk ends after a data: line and the blank SSE event delimiter arrives in the next chunk, the event is discarded. Stream chunk boundaries are arbitrary, so this can drop audio deltas; keep pending event state outside the read loop and cover a split-chunk event.
  - confidence: 0.87
