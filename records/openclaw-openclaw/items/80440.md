## Summary

Keep this PR open: current main does not have OpenRouter music generation, so the feature is useful, but the latest patch is not merge-ready because it bypasses guarded provider HTTP, can drop split SSE events, and lacks contributor-provided real behavior proof.

## What This Changes

The PR adds an OpenRouter music-generation provider, registers it in the OpenRouter plugin manifest and entrypoint, and updates the bundled music provider contract expectation.

## Best Possible Solution

Keep this PR path, but route the streaming POST through guarded provider HTTP, make SSE parsing chunk-safe, add focused coverage, and require redacted live music_generate proof before merge.

## Reproduction Assessment

Not applicable as a feature PR. Source inspection confirms current main lacks OpenRouter music registration in both the plugin entrypoint and manifest.

## Solution Assessment

No. The feature direction is reasonable, but the implementation should use the shared guarded provider HTTP path and preserve SSE event state across reads before it is the maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Route the POST through guarded provider HTTP:** `extensions/openrouter/music-generation-provider.ts:105-110`
  - body: The provider resolves an operator-configurable OpenRouter base URL, but streamOpenRouterMusic submits it with bare fetch. That bypasses the shared provider request guard, dispatcher/proxy policy, and private-network controls used by adjacent OpenRouter providers; route the streaming POST through fetchWithTimeoutGuarded or an equivalent guarded helper.
  - confidence: 0.92
- **[P2] Keep SSE event state across reads:** `extensions/openrouter/music-generation-provider.ts:54`
  - body: dataLines is recreated on every reader.read() loop, so if a chunk ends after a data: line and the blank SSE event delimiter arrives in the next chunk, the event is discarded. Stream chunk boundaries are arbitrary, so this can drop audio deltas; keep pending event state outside the read loop and cover a split-chunk event.
  - confidence: 0.87
