## Summary

Keep open: this is a fresh, scoped release-automation feature request, and current main does not implement a `pr-release` draft-release flow. Existing release code publishes or updates already-tagged releases from changelog-backed notes, while the issue asks for a new post-PR-merge workflow that creates draft releases/tags from recorded merge commits with generated notes, AI highlight caching, concurrency, and retry behavior.

## What This Changes

The issue requests a new `pr-release` phase that creates draft GitHub releases and tags from recorded PR merge commit SHAs with repo-specific previous-tag discovery, generated notes, AI highlight caching, bounded concurrency, retry cache rules, and tests.

## Best Possible Solution
