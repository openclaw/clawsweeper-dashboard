## Summary

Keep open: current main still routes group/channel text-only finals through the documented `message_tool_only` suppression path without a non-verbose diagnostic or fallback, and recent comments add Slack, Telegram, and Discord live evidence. The open diagnostic PR is the closest implementation path, but it is not merged and broader fallback behavior still needs maintainer judgment.

## What This Changes

The issue asks for a cross-channel diagnostic or fallback when a group/channel turn in message-tool-only visible-reply mode finishes with non-empty assistant text but no message-tool send.

## Best Possible Solution

Keep this as the canonical cross-channel tracker, land a shared dispatch diagnostic that is visible enough for operators, and treat auto-promotion or a fallback mode as a separate privacy/product decision.

## Reproduction Assessment

Yes. Source inspection shows group/channel finals are suppressed under `message_tool_only`, and multiple recent comments provide live Slack, Telegram, and Discord reproductions; I did not run a live Slack/claude-cli repro in this read-only review.

## Solution Assessment

Unclear. A shared hard diagnostic is the narrowest maintainable first step, while auto-promotion or a new fallback mode needs explicit privacy and UX judgment.
