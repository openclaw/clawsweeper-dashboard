## Summary

Keep open: current main still intentionally suppresses ordinary group/channel final replies under message-tool-only delivery, and the remaining ask is a cross-channel diagnostic or fallback policy that is not implemented yet.

## What This Changes

Add cross-channel observability or fallback handling for group/channel turns that finish with non-empty assistant text but no message-tool send under `messages.groupChat.visibleReplies = "message_tool"`.

## Best Possible Solution

Preserve the tool-only privacy contract, add a default-visible cross-channel diagnostic for suppressed substantive finals with no committed message-tool delivery, and handle any auto-promote fallback as a separate opt-in product decision.

## Reproduction Assessment

Yes. Source inspection and tests show the current-main suppression path, and the issue discussion adds live Slack, Telegram, and Discord examples; I did not run a live transport reproduction in this read-only review.

## Solution Assessment

Unclear. A non-verbose diagnostic is the narrowest maintainable first fix, while auto-promote, reactions, or a new fallback mode require maintainer policy judgment because they can change private-by-default group behavior.
