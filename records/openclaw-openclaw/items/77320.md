## Summary

Keep open: current main still suppresses ordinary group/channel final text in `message_tool_only` mode and only documents the behavior; the requested non-verbose diagnostic, guard, or fallback is not implemented. The related open Telegram report and Slack receipt PR are narrower, so they do not supersede this broader Slack/cross-channel policy issue.

## What This Changes

Request a Slack/cross-channel diagnostic or safety net when `visibleReplies="message_tool"` turns finish with non-empty assistant text and no successful message-tool send.

## Best Possible Solution

Add a shared non-verbose diagnostic or trajectory signal for suppressed non-empty finals without successful messaging-tool delivery, then decide separately whether any auto-promotion or fallback mode belongs in the product contract.

## Reproduction Assessment

Yes, at source level. Current main resolves group/channel turns to `message_tool_only` and skips ordinary final delivery, and the issue/comments provide live Slack and Telegram observations; I did not run a live channel reproduction in this read-only review.

## Solution Assessment

Unclear as a final product fix. A shared diagnostic is the narrowest maintainable first step, while auto-promotion or a new fallback mode needs maintainer privacy and UX judgment.
