## Summary

Keep open: current main still intentionally suppresses automatic group/channel final replies in `message_tool` mode, and I found no current Slack terminal diagnostic or fallback for the reported final-text-without-message-tool turn shape. The remaining work is a product/UX decision about observability versus private-by-default room semantics.

## What This Changes

The issue asks for Slack group/channel tool-only turns with final text but no message-tool call to produce a diagnostic, auto-promoted reply, or opt-in fallback instead of appearing silent.

## Best Possible Solution

Add an agreed, scoped terminal diagnostic for suppressed non-empty final text with no successful message-tool delivery, and only add auto-promotion or fallback delivery behind explicit maintainer-approved policy.

## Reproduction Assessment

Yes, source-reproducible: the resolver, Slack dispatch path, and tests show group/channel `message_tool` mode processes the turn but suppresses automatic final/block replies. I did not run a live Slack/claude-cli reproduction because the model turn shape is nondeterministic and this review is read-only.

## Solution Assessment

Unclear: the hard diagnostic is the narrowest maintainable improvement, but auto-promote or fallback delivery would change documented private-by-default semantics. Maintainers should choose the warning level and whether any fallback is acceptable before implementation.
