## Summary

Keep open: current main still preserves the tool-only group/channel contract by suppressing ordinary final payloads, and it does not yet provide the default-visible diagnostic or fallback requested here. The issue also has fresh Slack, Telegram, and Discord live evidence plus an open diagnostic PR that has not merged.

## What This Changes

Make group/channel turns under `visibleReplies="message_tool"` stop failing silently when the assistant emits final text without a message-tool send.

## Best Possible Solution

Keep this as the canonical cross-channel tracker, land a default-visible shared diagnostic for suppressed non-empty finals, and decide any auto-promotion or fallback mode separately as a privacy/UX change.

## Reproduction Assessment

Yes, at source level. Current main resolves group/channel replies to `message_tool_only` and skips ordinary final payload delivery, while live comments provide Slack, Telegram, and Discord reproductions; I did not run a live Slack/claude-cli repro.

## Solution Assessment

Unclear. A shared non-verbose diagnostic is the narrowest maintainable first step, while auto-promotion, reaction signals, or a new fallback mode require explicit maintainer privacy and UX judgment.
