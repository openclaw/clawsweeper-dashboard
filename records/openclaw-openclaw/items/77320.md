## Summary

Keep open: current main intentionally suppresses automatic group/channel final replies in `message_tool` mode, and the report asks for new observability or fallback semantics rather than a broken Slack transport path.

## What This Changes

Requests a Slack/group-channel diagnostic or fallback when tool-only visible replies suppress a non-empty final assistant text because no message tool call occurred.

## Best Possible Solution

Decide the product semantics first; if observability is desired, add a scoped terminal diagnostic for suppressed non-empty final text with no successful message-tool delivery while preserving the default private-by-design behavior.

## Reproduction Assessment

Yes, source-reproducible: current tests and dispatch code show group/channel `message_tool` mode suppresses automatic final/block replies while still running the agent. I did not run a live Slack/claude-cli reproduction in this read-only review.

## Solution Assessment

Unclear: the proposed hard diagnostic is the narrowest likely improvement, but auto-promote or a new fallback mode would change the documented private-by-default room behavior. Maintainers should choose the log level and fallback policy before implementation.
