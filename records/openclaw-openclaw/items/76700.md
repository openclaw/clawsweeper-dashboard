## Summary

Keep open. Current main partially addresses the same-config restore loop by failing invalid Gateway startup and hot reload closed, but the Brave provider path is still a fatal validation failure and `openclaw gateway stop` still preflights invalid config before stopping.

## What This Changes

Prevent an installed Brave `web_search` configuration from crash-looping the Gateway and preserve usable recovery commands when the provider is unavailable.

## Best Possible Solution

Make Brave install and register its `web_search` provider reliably, keep invalid provider/config states fail-closed without self-restore loops, and allow `openclaw gateway stop` to bypass config validation as an unconditional recovery command.

## Reproduction Assessment

Yes, source-level. Current main still has a supported Brave config path, a fatal unavailable-provider validation branch, and a stop preflight that blocks on invalid config; I did not run a live Gateway crash-loop repro.

## Solution Assessment

No, current main is only a partial solution. The fail-closed Gateway change removes the auto-restore loop from startup/hot reload, but the provider-registration/packaging path and unconditional stop recovery path still need a focused fix.
