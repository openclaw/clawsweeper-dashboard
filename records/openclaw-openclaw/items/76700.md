## Summary

Keep open. Current main partially addressed the last-known-good auto-restore loop by moving recovery under doctor, but the reported failure cluster is not fully resolved: a known but inactive `brave` web_search provider is still a fatal config validation issue, `gateway stop` still preflights invalid config, and the doctor legacy migration path can still discard safe migrations when unrelated validation remains.

## What This Changes

The issue requests that Brave web search plugin install/provider failures stop bricking gateway startup, last-known-good restore loops, doctor repair, and gateway stop recovery.

## Best Possible Solution

Keep the current no-auto-restore behavior, make service stop an unconditional escape path, let stale or unavailable optional web_search provider state degrade or self-disable without blocking startup/doctor repair, and land the doctor migration fix through the existing related PR or an equivalent patch.

## Reproduction Assessment

Yes, source-level. Current main still contains the fatal `brave` provider validation branch, a test that preserves it, and a stop preflight that blocks on invalid config; I did not run a live gateway crash loop in this read-only review.

## Solution Assessment

No complete solution is present on main. The narrow maintainable direction is to keep typo validation strict but make recovery paths unconditional and make stale optional plugin/provider state repairable instead of fatal.
