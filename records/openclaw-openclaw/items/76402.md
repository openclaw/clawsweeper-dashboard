## Summary

Keep open. The reported sessions.list event-loop stall is credible and the PR has useful source work, but the patch is not merge-ready because it strips existing sessions.list row fields, overlaps stale current-main async transcript work, and lacks the required changelog entry.

## What This Changes

The PR adds oversized transcript-line guards and a lightweight buildGatewaySessionRow() mode that listSessionsFromStoreAsync() uses for bulk sessions.list rows.

## Best Possible Solution

Rework the performance fix so sessions.list avoids repeated expensive work while preserving the existing row contract, with targeted regression coverage and a changelog entry.

## Reproduction Assessment

Yes for the blocking regressions: existing current-main tests cover transcript usage/model fallback, thinking defaults, and async/sync list parity that this patch would break. The original production stall is supported by the PR measurements but was not independently benchmarked in this read-only pass.

## Solution Assessment

No, not as written. A safer fix should keep sessions.list fields intact or introduce an explicit versioned/parameterized lightweight response instead of silently stripping row metadata.
