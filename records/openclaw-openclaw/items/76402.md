## Summary

The performance problem is credible and current main does not implement the lightweight row path, but this branch is not merge-ready because the new sessions.list lightweight mode drops documented/tested per-row metadata and the head is dirty against current main, which already contains part of the transcript-read fix.

The PR adds oversized transcript-line guards and a lightweight sessions.list row mode that skips transcript usage fallback and several row enrichment steps for async bulk listings.

## Best Possible Solution

Rebase onto current main, keep the sessions.list hot path lightweight only for expensive transcript and recomputation work, and preserve the public row contract with regression coverage for async list rows.
