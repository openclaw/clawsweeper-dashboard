## Summary

Keep open. The narrowed report targets the model-facing `sessions_spawn` path, and current main still has a plausible accepted-before-start gap where a child run can be acknowledged before provider/thinking validation or embedded startup has actually succeeded.

## What This Changes

The issue reports silent fast-failure for ad-hoc `sessions_spawn` subagent runs on OpenAI/OpenAI-Codex GPT models whenever `thinking` is non-`off`.

## Best Possible Solution

Make model-facing `sessions_spawn` either start a real child run with the requested supported OpenAI/Codex thinking level or return a visible startup/unsupported-level error before the parent treats the spawn as accepted.

## Reproduction Assessment

No live high-confidence reproduction was run in this review. The source path is reproducible enough to show that `sessions_spawn` can return accepted before the child embedded run reaches provider/thinking validation, matching the reported silent failure shape.

## Solution Assessment

Unclear until the exact OpenAI/Codex rejection branch is reproduced. The best fix direction is narrow runtime error propagation or pre-ack validation around `sessions_spawn`, `spawnSubagentDirect`, and gateway agent startup, not another catalog-only change.
