## Summary

Keep open: current main still defaults Telegram to partial streaming with tool progress and raw command detail, although it now cleans up tool-only preview bubbles after assistant boundaries; changing the normal Telegram chat UX remains a product/backcompat decision.

## What This Changes

Request to make Telegram DMs quieter during tool-heavy turns by defaulting to final-only or suppressing partial tool/status progress unless explicitly enabled.

## Best Possible Solution

Make an explicit Telegram streaming UX policy, then align defaults, setup/doctor guidance, docs, changelog, and focused Telegram streaming tests around the chosen final-only, quiet-preview, or progress-draft behavior.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: unset Telegram streaming resolves to partial, partial mode records tool-progress lines, and current tests assert Exec progress appears unless disabled.

## Solution Assessment

Unclear until maintainers choose the intended Telegram chat default. The current implementation already supports final-only and quieter preview modes, so changing defaults is a UX/backcompat decision rather than a narrow bug fix.
