## Summary

Keep open: current main still documents and tests Telegram partial streaming with tool-progress visible by default, so a final-only or quieter default remains a maintainer UX/backcompat decision rather than completed cleanup.

## What This Changes

Make Telegram DMs quieter during tool-heavy turns by defaulting to final-only delivery or suppressing/compacting partial tool-progress updates unless explicitly enabled.

## Best Possible Solution

Make an explicit maintainer decision for Telegram chat UX, then update runtime defaults or setup guidance, config metadata, docs, focused streaming tests, and changelog consistently.

## Reproduction Assessment

Yes. Source inspection and focused tests show unset Telegram streaming resolves to partial, partial mode writes Exec/tool progress into the draft preview, and explicit config can disable or hide those details; I did not run a live Telegram turn.

## Solution Assessment

Unclear until maintainers choose the intended Telegram default. Current main already supports off, preview.toolProgress=false, commandText=status, and progress mode, so changing released defaults is the remaining policy choice rather than the narrowest bug fix.
