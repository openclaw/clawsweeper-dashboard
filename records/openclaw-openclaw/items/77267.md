## Summary

Keep open: current `main` still defaults Telegram streaming to `partial`, enables preview tool progress by default, and tests expect visible Exec/progress lines. The remaining work is a Telegram UX/backcompat policy decision rather than conservative cleanup.

## What This Changes

The issue asks OpenClaw to make normal Telegram chats quieter by default, suppressing partial tool/status noise unless explicitly enabled or documented as a power-user mode.

## Best Possible Solution

Maintainers should decide the Telegram default/progress disclosure policy, then align the resolver, docs, config UI hints, tests, changelog, and live Telegram proof around that choice.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: unset Telegram streaming resolves to `partial`, active preview streaming enables tool-progress drafts by default, and current tests expect visible Exec/progress output before the final answer. I did not run a live Telegram chat turn in this read-only review.

## Solution Assessment

Unclear until maintainers choose the default/backcompat policy. The closed default-off PR was too narrow because it changed only the fallback while tests, docs, UI hints, and real Telegram proof requirements remained unresolved.
