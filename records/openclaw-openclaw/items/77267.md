## Summary

Keep open: current main still documents and tests Telegram preview streaming and tool-progress as default behavior, while quiet/final-only delivery is available only through explicit config. Changing that default or setup guidance is a Telegram UX/backcompat product decision, not completed cleanup.

## What This Changes

Request to make Telegram direct-chat streaming quieter or final-only by default, with less tool/status progress noise during tool-heavy agent turns.

## Best Possible Solution

Maintainers should choose the intended Telegram chat UX, then align the resolver defaults, setup/doctor guidance, docs, and focused Telegram streaming tests around that policy.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: unset Telegram streaming resolves to partial, preview tool progress defaults to true/raw command text, and tests assert Exec progress appears by default. I did not run a live Telegram turn.

## Solution Assessment

Unclear until maintainers choose the desired default. Current main already supports final-only and quieter preview modes, so the remaining change is the default/recommendation policy rather than a narrow bug fix.
