## Summary

Keep open: current main still documents and tests Telegram partial preview streaming with tool-progress visible by default, so the requested quieter/final-only default remains an unresolved UX/backcompat decision.

## What This Changes

Request quieter Telegram chat behavior for tool-heavy turns by defaulting to final-only or less verbose progress instead of noisy partial preview updates.

## Best Possible Solution

Make an explicit maintainer UX decision for Telegram defaults, then update the resolver, setup/doctor/docs guidance, and focused Telegram streaming tests consistently.

## Reproduction Assessment

Yes for the reported behavior at source level: unset Telegram streaming resolves to partial, tool progress defaults on, and tests assert Exec progress appears unless config disables it. I did not run a live Telegram Bot API turn.

## Solution Assessment

Unclear until maintainers choose the intended Telegram chat UX. The current implementation already supports final-only and quieter preview modes, but making either the default is a product/backcompat decision rather than a narrow bug fix.
