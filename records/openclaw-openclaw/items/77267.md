## Summary

Keep open: current main and the latest release still resolve unset Telegram streaming to `partial`, and the only linked default-change PR was closed unmerged after maintainer discussion said changing defaults needs more review.

## What This Changes

The issue asks OpenClaw to reduce noisy Telegram partial-streaming during tool-heavy turns by using final-only defaults, suppressing tool/status previews, or replacing repeated updates with one concise editable progress message.

## Best Possible Solution

Keep a maintainer-owned decision open for Telegram chat UX: either change the default/setup guidance with aligned tests, docs, config hints, and live proof, or explicitly keep `partial` as the released default and document `off`, `progress`, and `preview.toolProgress: false` as the supported quieter modes.

## Reproduction Assessment

Yes at source level: unset Telegram streaming still resolves to `partial`, and the dispatcher can send tool/progress callbacks into Telegram drafts when streaming is active. I did not run a live Telegram tool-heavy turn.

## Solution Assessment

No for the closed default-only PR as submitted: changing the resolver alone left tests/docs/config hints and backcompat policy unresolved. The best fix needs a maintainer decision and then one consistent implementation across runtime, docs, hints, tests, and proof.
