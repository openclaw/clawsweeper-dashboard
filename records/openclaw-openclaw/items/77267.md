## Summary

Keep open: current main and the latest release still keep Telegram preview streaming defaulted to partial with tool-progress visible by default, while the attempted default-off PR was closed for maintainer discussion rather than merged.

## What This Changes

Request to make Telegram partial streaming quieter or final-only by default, or guide normal chat users toward quieter Telegram streaming settings.

## Best Possible Solution

Maintainers should explicitly choose the Telegram chat UX policy, then align the resolver default, docs, config UI hints, doctor/setup guidance, tests, and changelog around that decision.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: unset Telegram streaming resolves to partial, partial mode enables progress drafts by default, and current tests expect Exec progress before final delivery. I did not run a live Telegram chat turn.

## Solution Assessment

Unclear until maintainers decide the default/backcompat policy. Current main already offers `streaming.mode: "off"`, `streaming.preview.toolProgress: false`, `commandText: "status"`, and `progress` mode, but making quieter behavior the default is not a narrow bug fix.
