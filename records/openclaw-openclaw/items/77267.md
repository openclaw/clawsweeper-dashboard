## Summary

Keep open: current main still makes Telegram partial previews and tool-progress lines the documented default, so the requested quieter/final-only Telegram chat behavior remains a maintainer UX and backcompat decision.

## What This Changes

The issue requests quieter Telegram DM streaming during tool-heavy turns, preferably final-only by default or with less verbose progress/status preview behavior.

## Best Possible Solution

Make an explicit maintainer UX decision for Telegram chat defaults, then align the runtime default, setup or doctor guidance, docs, changelog, and focused Telegram streaming tests.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: unset Telegram streaming resolves to partial, tool progress is enabled by default, and tests assert raw Exec progress appears unless preview config disables or quiets it.

## Solution Assessment

Unclear until maintainers choose the intended Telegram chat UX. The existing implementation already supports final-only and quieter preview settings, so the unresolved part is whether those should become defaults or setup recommendations.
