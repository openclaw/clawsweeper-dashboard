## Summary

Keep open: current main and the latest release still default Telegram preview streaming to `partial` with tool progress enabled, while docs/tests treat that as released behavior. Changing the default or suppressing visible progress is a Telegram UX/privacy/backcompat policy decision, not conservative cleanup.

## What This Changes

The issue asks to make Telegram partial streaming less noisy for tool-heavy turns, especially by defaulting DMs to final-only or suppressing tool/status progress unless explicitly enabled.

## Best Possible Solution

Maintainers should decide Telegram's default progress-disclosure policy, then align the resolver, config hints, docs, tests, doctor/setup guidance, changelog, and live Telegram proof around that choice.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: unset Telegram streaming resolves to `partial`, preview tool progress defaults true/raw, and current Telegram dispatch/tests update a visible draft with Exec progress before final delivery; no live Telegram run was performed.

## Solution Assessment

Unclear. Existing `off`, `streaming.preview.toolProgress: false`, and `commandText: "status"` controls reduce noise, but changing the default or suppressing tool/status progress globally would alter documented released behavior and needs maintainer policy judgment.
