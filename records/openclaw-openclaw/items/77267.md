## Summary

Keep open: current main still defaults Telegram preview streaming to partial with tool-progress lines and raw command detail enabled, while the requested quieter/final-only default remains a Telegram UX and backcompat decision.

## What This Changes

Request quieter Telegram chat streaming by default, especially suppressing partial tool/status progress during tool-heavy direct-message turns.

## Best Possible Solution

Make an explicit maintainer UX decision for Telegram DMs, then align the default, setup/doctor guidance, docs, changelog, and focused Telegram streaming tests around that policy.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: unset Telegram streaming resolves to partial, partial mode records tool-progress lines, and tests assert Exec progress appears unless disabled. I did not run a live Telegram chat turn.

## Solution Assessment

Unclear until maintainers choose the desired Telegram chat UX. The current implementation already supports final-only and quieter preview configurations, so changing defaults is a product/backcompat choice rather than a narrow bug fix.
