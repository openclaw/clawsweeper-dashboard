## Summary

Keep open: current main still preserves Telegram `partial` streaming with tool-progress and raw command details enabled by default, so the report is a valid UX/defaults request rather than an obsolete or already-fixed bug.

## What This Changes

Make Telegram tool-heavy turns quieter by default or steer setup toward final-only/progress-style streaming instead of raw partial tool-status previews.

## Best Possible Solution

Choose the Telegram chat UX policy, then update the Telegram plugin defaults, setup/doctor guidance, docs, and regression tests consistently.

## Reproduction Assessment

Yes. Source and tests show that unset or `partial` Telegram streaming creates preview tool-progress updates by default, including command progress text, without needing a live Telegram token to establish the behavior.

## Solution Assessment

Unclear. The existing implementation already offers opt-out knobs, but switching defaults or setup recommendations is a maintainer UX/product decision because current docs and tests intentionally preserve the released behavior.
