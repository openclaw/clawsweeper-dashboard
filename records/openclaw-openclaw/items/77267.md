## Summary

Keep open: current main still defaults Telegram preview streaming to `partial` with tool-progress and raw command detail enabled unless the user opts out, so the requested quieter Telegram default remains a UX/backcompat product decision. Current main has a related cleanup for tool-only previews, but it does not change the default behavior reported here.

## What This Changes

The issue requests quieter Telegram chat behavior for tool-heavy turns by defaulting to final-only or suppressing verbose tool lifecycle updates unless explicitly enabled.

## Best Possible Solution

Make an explicit maintainer decision for Telegram chat UX, then update the resolver defaults, config help, docs, focused tests, and changelog consistently while keeping explicit `partial` and `progress` modes available.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: unset Telegram streaming resolves to `partial`, preview tool progress defaults on, and current tests assert Exec progress appears by default. I did not run a live Telegram transport turn.

## Solution Assessment

Unclear until maintainers choose the intended default. The current implementation already supports final-only and quieter preview modes, so changing defaults is a product/backcompat decision rather than a narrow bug fix.
