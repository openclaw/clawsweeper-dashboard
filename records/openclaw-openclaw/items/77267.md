## Summary

Keep open: current main still intentionally defaults Telegram preview streaming to partial with tool-progress visible by default, so the requested final-only/quieter default remains a product policy decision rather than completed cleanup.

## What This Changes

The issue asks for Telegram DMs to be quieter during tool-heavy turns, preferably final-only by default or with internal tool/status progress suppressed unless explicitly enabled.

## Best Possible Solution

Make an explicit maintainer decision for Telegram chat UX, then update the default, setup/doctor guidance, docs, and focused Telegram streaming tests consistently.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: unset Telegram streaming resolves to partial, partial mode records tool-progress lines, and tests assert Exec progress appears unless disabled. I did not run a live Telegram chat turn.

## Solution Assessment

Unclear until maintainers choose the intended default. The current implementation already supports quiet final-only and quieter preview modes, but making those defaults is a product/backcompat choice rather than a narrow bug fix.
