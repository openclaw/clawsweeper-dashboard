## Summary

Keep open: current main still defaults Telegram preview streaming to `partial` with tool progress enabled, and the closed default-off PR was deferred for maintainer discussion. This is a Telegram chat UX/backcompat policy decision, not conservative cleanup.

## What This Changes

The issue asks to make normal Telegram turns quieter by default, either final-only or with less noisy tool-progress handling unless explicitly enabled.

## Best Possible Solution

Maintainers should decide the Telegram progress/default disclosure policy, then align the resolver, docs, config UI hints, tests, doctor/setup guidance, changelog, and live Telegram proof around that choice.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: unset Telegram streaming resolves to `partial`, active preview streaming enables tool-progress drafts by default, and current tests expect visible Exec progress before the final answer.

## Solution Assessment

Unclear until maintainers choose the default/backcompat policy. The prior default-off PR was too narrow because the tested/docs/UI contract stayed `partial`, and maintainers explicitly asked for further discussion.
