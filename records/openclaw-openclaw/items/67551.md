## Summary

Keep open. Current main still lacks Unicode smart-quote normalization for tool-call argument JSON, and the merged adjacent work in #70294 only enabled the existing malformed-argument repair wrapper for openai-completions. ## What This Changes

The PR changes embedded runner tool-call argument repair to normalize Unicode smart quotes in malformed argument JSON and teach balanced-prefix extraction to treat smart quotes as string delimiters.

## Best Possible Solution

Ship a rebased, narrow parser repair on current main that adds delimiter-aware smart-quote normalization where the current runner/shared helper structure actually reads it, with focused tests for structural U+201C/U+201D delimiters, ASCII-delimited content smart quotes, trailing junk repair, and the existing #70294 openai-completions wrapper behavior.
