## Summary

Keep open. Current main still hard-caps isolated cron run-log summaries at 2000 UTF-16 code units before persistence, while `cron.runLog` only exposes file-size and retained-line pruning; the report is recent, concrete, and not superseded by a merged fix.

## What This Changes

Make cron run-log summaries configurable or larger so long isolated-agent outputs are not persisted with an unconditional 2000-character cap.

## Best Possible Solution

Add a bounded `cron.runLog.maxSummaryChars` option to the config contract and apply it wherever cron derives persisted summary text, leaving delivery payloads and full `outputText` unchanged.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: any isolated cron final assistant or payload text longer than 2000 UTF-16 code units flows through `pickSummaryFromOutput` and is reduced to 2000 characters plus an ellipsis before persistence.

## Solution Assessment

Yes. The requested `cron.runLog` option is the narrowest maintainable fix because it changes only persisted run-log readability while existing `maxBytes` and `keepLines` continue to bound disk usage.
