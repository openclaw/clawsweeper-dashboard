## Label Justifications

- `P1`: The paired report describes a currently broken WhatsApp plugin update workflow that can destroy linked sessions and force QR re-pairing.
- `merge-risk: 🚨 compatibility`: The PR changes established terminal logged-out/conflict cleanup behavior from clearing credentials to preserving restorable backups.
- `merge-risk: 🚨 session-state`: The patch can restore invalid logged-out/conflict session state and races restore against socket startup.
- `merge-risk: 🚨 security-boundary`: WhatsApp session credentials are sensitive credential material, and the PR retains them in backup directories without a cleanup policy.

## Summary

Keep open. The reworked PR fixes the earlier explicit-logout default, but it is still not merge-ready because terminal logged-out/conflict cleanup now preserves credentials that startup can restore, startup restore is raced against socket creation, and the supplied proof still does not show a real WhatsApp plugin update preserving a linked session.

## What This Changes

The PR adds WhatsApp auth-directory backup/restore helpers, opts reconnect cleanup into `backupOnClear`, starts restore from the WhatsApp monitor, exports the restore helper, and updates auth/logout tests.

## Best Possible Solution

Preserve WhatsApp credentials at a proved plugin-update or migration boundary, await any startup restore before socket creation, and keep explicit logout plus terminal logged-out/conflict cleanup destructive unless maintainers intentionally change that security contract.

## Reproduction Assessment

No for the original user report: this review did not establish a live `openclaw plugins update whatsapp` reproduction with linked status before and after. The PR-introduced blockers are source-reproducible from the changed cleanup and startup paths.

## Solution Assessment

No. Targeting terminal logged-out/conflict cleanup is not the best proven boundary; the safer fix should preserve or migrate credentials around plugin update/startup while keeping invalid-session cleanup and explicit logout clearing credentials.
