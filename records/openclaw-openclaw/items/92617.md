## Summary

Keep open; this PR is not merge-ready because it renames the active WhatsApp auth directory for the shared logout/terminal-cleanup path, so the reported update flow can still leave the account not linked, and it also retains sensitive credentials after ordinary logout.

## What This Changes

This PR changes WhatsApp Web logout cleanup to rename managed auth directories to timestamped `.bak` directories and updates logout/auth-store tests to expect backups.

## Best Possible Solution

Preserve valid WhatsApp credentials in the active auth directory across plugin updates while keeping explicit logout credential-clearing semantics, with focused tests and redacted real update proof.

## Reproduction Assessment

No, not for the original plugin-update report: the issue gives concrete steps, but this review did not run a real WhatsApp update. The PR-introduced behavior is source-reproducible because it renames the active auth directory while current readers only check active `creds.json`.

## Solution Assessment

No. The patch is only a backup mitigation, and it changes the shared logout primitive while still removing credentials from the active location; the fix should target update/migration behavior without weakening logout semantics.

- **[P1] Preserve the active auth directory:** `extensions/whatsapp/src/auth-store.ts:371`
  - body: This renames the managed auth directory to `.bak.<timestamp>` for every non-legacy logout/terminal cleanup.   - confidence: 0.91
