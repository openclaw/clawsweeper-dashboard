## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Do not delete credentials when backup preservation fails:** `extensions/whatsapp/src/auth-store.ts:430`
  - body: `backupOnClear` is the path meant to preserve a linked session during update cleanup, but if `fs.rename` cannot create the backup this fallback deletes the active auth directory. A timestamp collision, permission issue, or other rename failure would silently recreate the exact session wipe this PR is trying to prevent; leave the active directory intact or abort cleanup instead.
  - confidence: 0.9
- **[P1] Prevent stale backups from relinking after logout:** `extensions/whatsapp/src/auth-store.ts:379-380`
  - body: `restoreLatestBackup` restores any sibling `authDir.bak.*` whenever active creds are missing, while those backups live outside the active auth directory and are not invalidated by default logout. If a backup remains after an aborted update or failed restore, an operator can try to clear the account and the next monitor startup can relink the old sensitive session anyway.
  - confidence: 0.86
