## Label Justifications

- `P1`: The paired report describes a currently broken WhatsApp plugin update workflow that can destroy linked sessions and force QR re-pairing.
- `merge-risk: 🚨 compatibility`: The PR changes established logged-out/conflict cleanup behavior from clearing credentials to preserving restorable backups.

## Summary

Keep open: this PR addresses a high-impact WhatsApp update session-loss report, but the patch is not merge-ready because it preserves/restores credentials from logged-out/conflict cleanup, races startup restore against socket creation, and lacks real plugin-update proof.

## What This Changes

The PR adds WhatsApp auth-directory backup/restore support, opts reconnect cleanup into `backupOnClear`, runs startup restore from the web monitor, exports the helper, and updates auth/logout tests.

## Best Possible Solution

Preserve or migrate credentials at a proved plugin-update/startup boundary, await any restore before socket creation, and keep explicit logout plus terminal logged-out/conflict cleanup destructive unless maintainers deliberately define a credential-backup policy.

## Reproduction Assessment

No for the original user report: this review did not establish a live `openclaw plugins update whatsapp` reproduction with linked status before and after. The PR-introduced blockers are source-reproducible from the changed cleanup and startup paths.

## Solution Assessment

Full review comments:

- **[P1] Do not restore terminal logged-out credentials:** `extensions/whatsapp/src/auto-reply/monitor.ts:157`
  - body: `clearTerminalWebAuthState` is used after the monitor classifies the session as logged-out or conflict, and current tests/docs expect that path to clear auth so the operator can relink. Passing `backupOnClear: true` makes those invalid credentials restorable on the next startup instead.
  - confidence: 0.9
- **[P1] Await restore before opening the socket:** `extensions/whatsapp/src/auto-reply/monitor.ts:221`

A missing active dir can be recreated before the rename runs, so the update path may still open with fresh empty auth.
  - confidence: 0.88
