## Summary

Keep open: the PR targets a real WhatsApp credential-loss regression, but the patch is not merge-ready because it retains recoverable credentials after normal logout and does not restore the new timestamped backup during startup, so the reported update flow can still require QR re-pairing.

## What This Changes

The PR changes WhatsApp `logoutWeb()` to rename the managed auth directory to a timestamped `.bak.*` directory instead of recursively deleting it, with logout/auth-store tests updated to expect backup directories.

## Best Possible Solution

Preserve WhatsApp sessions across plugin update with a recovery path that startup actually reads, while keeping explicit logout/remove destructive unless maintainers intentionally approve and document credential retention.
