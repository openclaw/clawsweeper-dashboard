## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep terminal cleanup from restoring invalid sessions:** `extensions/whatsapp/src/auto-reply/monitor.ts:157`
  - body: `clearTerminalWebAuthState` runs for both logged-out and conflict terminal closes. Passing `backupOnClear: true` here moves those relink-required credentials into `.bak.*`, and the new startup restore can put them back on the next gateway start, so a real 401/440 cleanup can loop instead of staying cleared.
  - confidence: 0.9
- **[P1] Await restore before opening the WhatsApp socket:** `extensions/whatsapp/src/auto-reply/monitor.ts:221`
  - body: `restoreLatestBackup(account.authDir)` is started in the background and the monitor immediately continues toward `controller.openConnection`. `createWaSocket` ensures/reads the active auth directory, so startup can race ahead with an empty auth state before the backup is restored, leaving the reported update path not linked.
  - confidence: 0.87
- **[P2] Do not delete when preservation backup fails:** `extensions/whatsapp/src/auth-store.ts:430`
  - body: In `backupOnClear` mode, a failed rename falls through to `fs.rm`, so a backup path collision, permission error, or other rename failure still wipes the active credentials in the path meant to preserve them. This should fail closed or keep the active directory instead of recreating the data-loss failure mode.
  - confidence: 0.82
