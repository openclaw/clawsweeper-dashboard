## Summary

No blocking findings. The PR fixes a real auth-routing bug: current main resolves OpenAI media audio without a model API, which lets OpenAI OAuth profiles win over `OPENAI_API_KEY`; the patch supplies the OpenAI audio API contract so the existing auth resolver can skip OAuth and use the platform key.

## What This Changes

The PR passes an audio-specific OpenAI model API into media-understanding provider auth resolution and adds a regression test for OAuth-plus-`OPENAI_API_KEY` audio transcription.

## Best Possible Solution

Land the focused auth-routing fix after normal maintainer checks stay green; keep batch OpenAI audio transcription on platform API-key auth while preserving Codex OAuth for the ChatGPT/Codex routes.

## Reproduction Assessment

Yes, source-reproducible. On current main, media audio auth omits `modelApi`, and OpenAI auth ordering can select OAuth before an env API key unless the resolver knows the request is a direct OpenAI platform API; I did not run tests because this review is read-only.

## Solution Assessment

Yes. Passing the OpenAI audio transcription API into the existing auth resolver is the narrowest maintainable fix here; a new plugin API seam would be heavier than this regression requires.
