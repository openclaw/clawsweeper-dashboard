## Summary

Keep open: current main still always passes and renders the desktop Workspace Files rail in Control UI chat, although CSS hides it below 1120px; this is a product/settings request, not an already-fixed bug.

## What This Changes

Request to add a Control UI/WebChat option, either config-backed or a sidebar UI setting, to hide the Workspace Files panel.

## Best Possible Solution

If maintainers accept the UX direction, add a browser-local Control UI toggle that suppresses the rail and avoids loading workspace files while hidden; only add gateway/openclaw config if product owners explicitly want shared deployment policy.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection does confirm the current desktop rail is always wired into the chat view and only hidden by responsive CSS on narrower viewports.

## Solution Assessment

Unclear until maintainers choose the product shape. The requested UX is reasonable, but a browser-local setting fits the existing Control UI preference pattern better than adding a new `openclaw.json` field by default.
