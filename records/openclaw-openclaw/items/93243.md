Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Mark top-level thread starts as serviced:** `extensions/mattermost/src/mattermost/monitor.ts:1646-1655`
  - body: This backfill block only runs when `post.root_id` is present. A top-level Mattermost post can start the thread via `effectiveReplyToId === post.id`, then the kernel clears pending history after dispatch; the next same-session threaded reply now looks like recovery and fetches/injects the full server thread. Mark the effective thread session as serviced for top-level thread-starting turns without fetching.
  - confidence: 0.9
- **[P2] Trim thread posts before resolving senders:** `extensions/mattermost/src/mattermost/monitor.ts:545-552`
  - body: `GET /posts/{id}/thread` can return the full thread, but this loop awaits `resolveUserInfo` for every ordered post before `entries.slice(-historyLimit)`. Select the effective history-window posts first so long cold threads do not fan out into unnecessary user lookups before the inbound turn can continue.
  - confidence: 0.84
