## Summary

Keep open: this is a meaningful Mattermost recovery fix candidate, but PR head still returns before marking a non-empty active thread as serviced, so the next same-session empty-window follow-up can fetch and inject the full server thread outside the intended recovery case.

The PR adds a Mattermost thread fetch helper, backfills server thread history into empty in-memory thread windows, gates backfill by agent session id, wires it into inbound thread handling, and adds unit coverage.

## Best Possible Solution

Keep the Mattermost-owned server backfill, but mark same-session threads as serviced when existing pending history already covers the turn and add regression coverage for that active-thread steady state.

## Reproduction Assessment

## Solution Assessment

No, not as currently patched. The Mattermost-owned backfill boundary is the right layer, but the gate must also mark non-empty first sightings so active same-session follow-ups do not refetch after the kernel clears pending history.
