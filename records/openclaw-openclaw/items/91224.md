## Summary

Keep open: current main and v2026.6.8 still route resolved external final payloads through durable delivery without an inter-session control-ack suppression check, while the directly linked fix PR was closed unmerged. The broader strict-internal handoff issue overlaps, but it does not replace this narrower delivery-ack track.

## What This Changes

The issue reports that internal inter-session confirmation acknowledgements can inherit a parent session's external route and appear in channels such as Slack.

## Best Possible Solution

Define the structural control-reply contract, then suppress internal inter-session acknowledgement replies before external provider delivery while preserving genuine inter-session completions and covering sessions_send plus subagent_announce regressions.

## Reproduction Assessment

Yes, source-reproducible: current main resolves the parent session's external route and calls durable delivery for non-empty payloads, while no current-main or v2026.6.8 suppression predicate distinguishes the reported acknowledgement shape. I did not run a live Slack reproduction in this read-only review.

## Solution Assessment

No, current main is not the best end state because the leak remains. The issue's metadata-driven direction is safer long term, while the closed candidate PR shows a plausible narrower predicate if maintainers accept that contract.
