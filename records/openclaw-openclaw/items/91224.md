## What This Changes

The issue reports that internal inter-session acknowledgement text can be routed to an external user channel when the parent session has an external delivery route.

## Best Possible Solution

Review and either merge https://github.com/openclaw/openclaw/pull/91234 or replace it with the maintainer-chosen structural control-reply suppression design before closing this issue.

## Reproduction Assessment

Yes. Source inspection shows the reported inter-session provenance and external route can reach `sendDurableMessageBatch` on current main without a control-reply suppression guard; I did not run a live Slack reproduction.

## Solution Assessment

Yes, with a maintainer design choice still needed. A structural suppression at the final delivery boundary or an explicit internal-control marker is the maintainable path; text matching acknowledgement copy would be brittle.
