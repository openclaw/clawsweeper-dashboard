## Best Possible Solution

Keep this issue open until maintainers review and either land or deliberately reject the linked fix PR.

## Reproduction Assessment

Yes. Source inspection shows an inter-session webchat result can inherit a persisted external route and then send a text payload through `sendDurableMessageBatch` with no current-main control-ack suppression branch; I did not run a live Slack repro.

## Solution Assessment

Yes for keeping this open. The linked PR is the active concrete repair path, and the desired end state is a structural suppression that keeps genuine inter-session completions deliverable while hiding internal control acknowledgements.
