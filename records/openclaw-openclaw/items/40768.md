## Summary

Keep open: current main and v2026.6.8 still use open_id-only Feishu bot mention matching, and the linked closing PR is open, conflicting, and paused for human review rather than merged.

## What This Changes

The issue reports that Feishu group @mentions can be ignored when multiple bot apps share a group because the mention open_id can be app-scoped and differ from the receiving bot's own identity.

## Best Possible Solution

Repair or replace the linked Feishu PR on current main with a plugin-local fix that uses a proven stable mention identity, covers mention detection, mention stripping, debounce paths, and provides redacted live multi-bot Feishu proof.

## Reproduction Assessment

Yes at source level: if Feishu supplies a mention open_id that differs from the receiving botOpenId, current main's matcher returns false and the group gate can drop the message. I did not run a live Feishu multi-bot tenant, so live reproduction remains the missing proof.

## Solution Assessment

Unclear: the issue describes a real bug, but the proposed union_id fallback is not proven because the reporter and closed sibling PR note bot info does not return union_id. The best fix needs a maintainer-approved stable identity source and current-main wiring across the Feishu message path.
