## Summary

Keep open: current main still matches Feishu bot mentions only by the receiving account's stored open_id, and an open PR with closing syntax is already the canonical repair path for this issue.

## What This Changes

Fix Feishu group @mention detection when multiple Feishu bot apps share a group and app-scoped mention open_ids do not match the receiving bot's stored open_id.

## Best Possible Solution

Keep this issue open until the linked PR is repaired or replaced with a Feishu-validated identity matcher that preserves strict mention gating, command stripping, rich-text behavior, and focused regression coverage.

## Reproduction Assessment

Yes, at source level: a Feishu group event whose mention metadata contains a non-broadcast open_id different from botOpenId makes checkBotMentioned return false, and the documented requireMention group path then drops it. I did not perform live Feishu validation.

## Solution Assessment

No final fix is on main yet. The best path is to review, repair, and land the linked Feishu identity-matching PR or replace it with a verified implementation rather than close this issue early.
