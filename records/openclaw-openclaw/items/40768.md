## Summary

Keep open: current main still recognizes Feishu bot mentions by the receiving bot open_id only, while the open linked PR remains the active but blocked implementation path for this bug.

## What This Changes

The issue reports Feishu group @mention detection failing when multiple Feishu bot apps share one group because the mention open_id can be scoped to a different app than the receiving bot.

## Best Possible Solution

Repair or replace the linked Feishu PR with an API-proven identity-resolution fix that covers mention detection, mention stripping, debounce handling, and real multi-bot Feishu behavior.

## Reproduction Assessment

Yes from source, but not from a live run here. Current main compares mention open_id values to botOpenId and then drops mention-required group messages when mentionedBot is false.

## Solution Assessment

Unclear. The linked union_id fallback is plausible, but the reported API behavior and current probe contract mean maintainers need a verified Feishu identity source before this is the best fix.
