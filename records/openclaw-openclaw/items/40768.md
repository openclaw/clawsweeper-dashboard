## Summary

Keep open: current main and the latest release still use an open_id-only Feishu bot mention check, and the issue is paired with an open same-author implementation PR that is conflicting and under human review rather than merged.

## What This Changes

The issue reports Feishu group @mentions being ignored when multiple bot apps share a group because mention open_id values can be app-scoped and fail the receiving bot's self open_id check.

## Best Possible Solution

Keep this issue open and resolve it through a current-main Feishu identity fix that proves the stable mention identifier across probing, event parsing, mention stripping, debounce merging, and group dispatch.

## Reproduction Assessment

Yes at source level: when the event mention open_id differs from botOpenId, current main sets mentionedBot false and the group requireMention gate can drop the message. A live multi-bot Feishu tenant reproduction was not run in this read-only review.

## Solution Assessment

No landed solution exists yet. The open linked PR is a plausible implementation candidate, but the best fix needs a current-main rewrite and proof of a stable Feishu bot identity source before merge.
