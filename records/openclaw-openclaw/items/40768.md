Keep open: current main and the latest release still gate Feishu group messages on an open_id-only mention match, while the linked closing PR remains open with human-review and real-proof blockers.

## What This Changes

The issue reports Feishu group @mention drops when multiple bots share a group because mention open_id values can be app-scoped and fail to match the receiving bot identity.

## Best Possible Solution

Keep this as the canonical issue until the linked PR is repaired or replaced with a Feishu-validated identity matcher applied consistently across text, rich text, mention stripping, and debounce paths, with focused regression coverage and redacted live proof.

## Reproduction Assessment

Yes, at source level: a Feishu group event whose non-broadcast mention open_id differs from botOpenId makes checkBotMentioned return false, and the requireMention group path then drops it. I did not perform live Feishu validation.

## Solution Assessment

No final fix is on main yet. The linked PR targets the right surface, but its union_id source is disputed and it still needs human review plus real Feishu proof before this issue should close.
