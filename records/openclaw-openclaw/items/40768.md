## Summary

Keep open: current main still matches Feishu group mentions only by app-scoped `open_id`, and an open linked fix PR remains unmerged, so cleanup should not close this issue yet.

## What This Changes

Feishu should recognize an @mention of the intended bot in shared multi-bot group chats even when Feishu reports app-scoped `open_id` values that differ from the receiving bot's self identity.

## Best Possible Solution

Keep this open until #40782 is reviewed, updated, or replaced with a Feishu-validated identity matcher that handles app-scoped mention IDs while preserving existing `@all`, alias, and `requireMention` behavior.

## Reproduction Assessment

Yes at source level: construct a Feishu group message whose mention metadata has an `open_id` different from the receiving bot's `botOpenId` under `requireMention: true`, and current main will mark `mentionedBot=false` and drop the message. I did not live-test against Feishu.

## Solution Assessment

Unclear. A fallback identity matcher is the right narrow area, but the current open PR's `union_id` source needs Feishu API validation because prior discussion says `bot/v3/info` does not provide that field.
