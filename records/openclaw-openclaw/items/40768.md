## What This Changes

The issue reports Feishu group @mentions being ignored in multi-bot groups because the event mention open_id can be scoped to another app and not match the receiving bot's startup botOpenId.

## Best Possible Solution

Land or replace the linked Feishu PR with a current-main identity-resolution fix that covers parsing, stripping, forwarding, debounce, startup probing, focused tests, and redacted real Feishu proof.

## Reproduction Assessment

Partially: source inspection gives a clear current-main reproduction path where a mention open_id that differs from botOpenId makes ctx.mentionedBot false and the group gate drops the message. A high-confidence live reproduction still needs a real multi-bot Feishu group because the app-scoped event behavior is external to the checkout.

## Solution Assessment

No current solution is ready yet: the open union_id PR is a plausible direction but not proven, conflicts with current main, and leaves the identity source disputed. The maintainable fix should first choose a documented Feishu identity strategy and then update all mention/debounce surfaces together.
