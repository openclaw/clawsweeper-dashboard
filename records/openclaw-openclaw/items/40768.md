## Summary

Keep open. Current main still gates Feishu group replies on an exact `open_id` match against the startup-resolved bot ID, while the report describes Feishu delivering mention IDs from a different app scope. The issue is also paired with open PR #40782 using closing syntax, so it should stay open until that PR is reviewed, refreshed, replaced, or closed.

## What This Changes

Request that Feishu group @mention detection work when multiple Feishu bot apps share the same group and mention `open_id` values are scoped to a different app.

## Best Possible Solution

Implement the fix in the Feishu plugin by using a verified stable mention identity source for multi-account group messages, plumbing that identity through mention detection, mention stripping, mention-forward, and debounce preservation, then adding targeted regression tests and a user-facing changelog entry.
