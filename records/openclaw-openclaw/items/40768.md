## What This Changes

The issue reports Feishu group @mentions being ignored when multiple bot apps share a group because mention open_id values are app-scoped and do not match the receiving bot's own open_id.

## Best Possible Solution

Keep this issue open and resolve it through a current-main Feishu identity fix that proves a stable bot identity across text, post, mention-forward, debounce, and group dispatch before merge.

## Reproduction Assessment

Yes at source level: a Feishu event whose mention open_id differs from botOpenId makes checkBotMentioned return false, which can keep the group requireMention gate from dispatching. A live multi-bot Feishu tenant was not run in this read-only review.
