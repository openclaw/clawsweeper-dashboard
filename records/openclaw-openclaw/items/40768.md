## What This Changes

Feishu multi-bot group @mention matching can fail when Feishu delivers an app-scoped mention open_id that does not match the receiving bot's stored botOpenId.

## Best Possible Solution

Keep this as the canonical issue and land a current-main Feishu plugin fix, or a repaired replacement for the linked PR, based on a verified stable bot identity or cross-app open_id mapping with focused tests and redacted live multi-bot proof.

## Reproduction Assessment

Partially. Source inspection shows that if Feishu supplies a mention open_id different from the receiving botOpenId, current main marks mentionedBot false and can skip the group message; a live Feishu multi-bot reproduction was not run.

## Solution Assessment

No final fix is on main yet. The linked PR is the current implementation candidate, but the best solution still needs maintainer/API review around the stable Feishu identity source plus coverage for all affected message paths.
