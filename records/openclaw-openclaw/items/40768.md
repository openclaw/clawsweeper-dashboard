## Summary

Keep open: current main and the latest release still gate Feishu group @mention recognition on the receiving bot's stored open_id, while the open linked PR with closing syntax is still the canonical repair path and is not merged.

## What This Changes

Feishu group @mentions should reach the intended bot when multiple Feishu bot apps share one group and mention open_id values are app-scoped.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/40782 is repaired and landed, or replaced with a Feishu-validated identity matcher with focused regression coverage and redacted live proof.

## Reproduction Assessment

Yes, at source level: a Feishu group event whose non-broadcast mention open_id differs from botOpenId makes checkBotMentioned return false, and the requireMention group path then drops it. I did not perform live Feishu validation.

## Solution Assessment

No final fix is on main yet. The linked PR targets the right identity-mismatch surface but needs human review, verified identity-source design, and broader path coverage before this issue should close.
