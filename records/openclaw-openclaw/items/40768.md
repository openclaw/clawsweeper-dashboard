## Summary

Keep open: current main still gates Feishu mentions on exact `open_id` equality, and the issue is paired with an open implementation PR that references it with closing syntax.

## What This Changes

The issue asks Feishu mention gating to recognize the intended bot in multi-bot groups despite Feishu app-scoped `open_id` values.

## Best Possible Solution

Keep this open until #40782 is reviewed, repaired, or replaced with a Feishu-validated identity matcher that handles app-scoped mention IDs while preserving strict mention gating.

## Reproduction Assessment

Yes, at source level: a Feishu group message whose mention metadata contains an `open_id` different from the receiving account's `botOpenId` will produce `mentionedBot=false`, and `requireMention` then drops the message. I did not live-test against Feishu.

## Solution Assessment

Unclear. The narrow fix belongs in Feishu identity/mention matching, but the current open PR's proposed `union_id` source conflicts with the reporter's note that `bot/v3/info` does not provide that field.
