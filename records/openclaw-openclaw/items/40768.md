Keep open: current main and the latest release still use open_id-only Feishu bot mention matching, and the open linked implementation PR still needs maintainer/API review plus live proof before this issue can close.

## What This Changes

Feishu group @mentions can be dropped in multi-bot groups when the mention open_id delivered by Feishu does not match the receiving bot account's self botOpenId.

## Best Possible Solution

Repair or replace the linked Feishu PR around a verified stable identity source or cross-app mapping, carry that identity through mention detection, stripping, debounce, and tests, then close this issue after the fix lands with live Feishu proof.

## Reproduction Assessment

Yes at source level: if Feishu supplies a mention open_id different from the receiving account's botOpenId, current main sets mentionedBot=false and the requireMention gate drops the group message. I did not live-reproduce this in a Feishu tenant.

## Solution Assessment

No, not yet. The open union_id fallback PR is a plausible candidate, but the best fix needs a verified Feishu identity source and must propagate it through detection, stripping, debounce, tests, and live proof.
