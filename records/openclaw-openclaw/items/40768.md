## Summary

Keep open: current main still gates Feishu group mentions on the receiving bot's own open_id, and the open linked PR is still the implementation path rather than a merged fix.

## What This Changes

Feishu group messages that @mention a bot can be dropped in multi-bot groups when Feishu delivers mention open_id values scoped to a different app than the receiving bot.

## Best Possible Solution

Keep this as the canonical issue until the linked PR or a replacement implements a Feishu-validated identity matcher across text, rich-text mention detection, mention stripping, and debounce paths, with focused regression coverage and redacted live Feishu proof.
