## Summary

Keep open: current main and v2026.6.8 still rely on Feishu mention `open_id` equality, while the linked closing PR is still open, conflicting, and missing live proof. https://github.com/openclaw/openclaw/issues/42410 is already closed as a duplicate of this issue, and https://github.com/openclaw/openclaw/issues/72504 overlaps but tracks a different self-mention stripping bug.

## What This Changes

The issue reports Feishu group @mentions being silently ignored when several bot apps share a group because app-scoped mention `open_id` values do not match the receiving bot identity.

## Best Possible Solution

Rebuild or replace the linked Feishu plugin fix around a proven stable identity source, then cover mention detection, mention stripping, debounce handling, and redacted live multi-bot Feishu proof before closing this issue.

## Reproduction Assessment

Yes at source level, but not from a live tenant in this review: when a Feishu mention carries an `open_id` different from the receiving `botOpenId`, current main's matcher returns false and the group `requireMention` gate can drop the message.

## Solution Assessment

Unclear. The linked `union_id` PR is a plausible direction, but the best fix needs a maintainer-approved and live-proven identity source wired through current-main mention detection, stripping, and debounce paths.
