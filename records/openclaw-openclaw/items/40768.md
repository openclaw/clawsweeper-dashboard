## Summary

Keep open: current main still recognizes Feishu group @mentions by the receiving bot account's stored open_id only, and the linked open repair PR with closing syntax is still the canonical implementation path.

## What This Changes

The issue requests a Feishu mention matcher that still recognizes a directly mentioned bot when multiple Feishu bot apps share one group and mention open_id values are app-scoped.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/40782 is repaired and landed, or replaced with a Feishu-validated identity matcher with focused regression coverage.

## Reproduction Assessment

Yes, at source level: a Feishu group event whose mention metadata contains a non-broadcast open_id different from botOpenId makes checkBotMentioned return false, and the requireMention group path then drops it. I did not perform live Feishu validation.

## Solution Assessment

No final fix is on main yet. The best path is to review, repair, and land the linked Feishu identity-matching PR or replace it with a verified implementation rather than close this issue early.
