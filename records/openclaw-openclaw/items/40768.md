## Summary

Keep open: current main still drops the reported Feishu multi-bot mention shape because mention gating compares only the event mention `open_id` to the receiving account's `botOpenId`. There is also an open linked fix PR with closing syntax, so this issue should stay open until that PR or a replacement lands.

## What This Changes

Feishu group @mention detection should recognize the intended bot when multiple Feishu bot apps share a group and Feishu delivers app-scoped mention IDs.

## Best Possible Solution

Land #40782 or a narrow replacement that resolves a stable Feishu bot identity inside the Feishu plugin, preserves strict mention gating, and adds regression coverage for app-scoped mention IDs.

## Reproduction Assessment

Yes, at source level: construct a Feishu group event whose mention `open_id` differs from the receiving account's `botOpenId`, then `checkBotMentioned()` returns false and `requireMention` drops the message. I did not live-verify the external Feishu payload behavior in this review.

## Solution Assessment

Unclear whether the proposed union_id fallback is the best final fix because the issue context says the current bot-info endpoint may not provide union_id. The maintainable path is still a Feishu-plugin fix, but it should be based on a verified stable identity source or cross-app mapping.
