Reported Feishu group @mentions can be ignored when multiple bot apps share a group because event mention IDs do not match the receiving bot account identity.

## Best Possible Solution

Keep this as the canonical issue and resolve it through a reviewed Feishu-plugin identity fix or replacement for https://github.com/openclaw/openclaw/pull/40782, backed by live multi-bot Feishu proof.

No high-confidence live reproduction was established. The source path is clear: a Feishu event whose mention `open_id` differs from the cached `botOpenId` leaves `mentionedBot` false and the requireMention group path returns.
