## Summary

Keep open: current main and v2026.6.8 still compare Feishu mention `open_id` directly to the receiving bot's `botOpenId`, so the reported app-scoped `open_id` mismatch can still suppress explicitly mentioned group messages. The linked fix PR is an open implementation candidate, but it is conflicted and under human review rather than merged proof.

## What This Changes

Fix Feishu group mention detection so a bot recognizes @mentions when multiple Feishu bot apps share a group and mention `open_id` values are app-scoped.

## Best Possible Solution

Keep this issue as the canonical tracker and repair or replace the linked Feishu PR on current main with a proven stable-identity strategy covering mention detection, mention stripping, debounce merging, focused tests, and redacted live Feishu proof.

## Reproduction Assessment

Yes at source level: a Feishu event whose mention `open_id` differs from `botOpenId` makes `checkBotMentioned` return false, and the group mention gate then drops the message. I did not establish a live multi-bot Feishu tenant reproduction.
