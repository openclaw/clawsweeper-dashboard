## Summary

Keep open: current main and the latest release still match Feishu bot mentions by the receiving bot's `open_id` only, and the linked implementation PR is still open, conflicting, and under human/API review.

## What This Changes

The issue reports that Feishu group @mentions can be ignored when multiple bot accounts share one group because the mention payload uses app-scoped `open_id` values that do not match the receiving bot's startup identity.

## Best Possible Solution

Keep this issue open until the linked Feishu repair is rebased or replaced with a current-main fix that uses a verified identity source and proves detection, stripping, debounce, and live multi-bot behavior.

## Reproduction Assessment

No live high-confidence Feishu reproduction was run here. The issue is source-reproducible when Feishu supplies a mention `open_id` different from `botOpenId`: current main sets `mentionedBot=false` and the group mention gate can drop the message.

## Solution Assessment

No for the currently linked union-id-only proposal as written; the stable Feishu identity source still needs maintainer/API confirmation. The maintainable fix should choose that source first, then carry the same identity through all mention-related paths.
