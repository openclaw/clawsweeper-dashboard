## Summary

Keep open: current main still uses the receiving Feishu bot's own `botOpenId` as the admission identity, so the reported app-scoped `open_id` mismatch can still suppress a valid group mention; the linked closing PR is open and not merged.

## What This Changes

The issue reports Feishu group @mentions being dropped when multiple OpenClaw bot apps share a group and the mention `open_id` does not match the receiving bot's app-scoped `botOpenId`.

## Best Possible Solution

Keep this as the canonical issue until https://github.com/openclaw/openclaw/pull/40782 or a replacement lands a Feishu-validated identity matcher across admission, stripping, rich-text posts, mention-forward/debounce paths, with focused tests and redacted live proof.

## Reproduction Assessment

Yes, at source level: a Feishu group event whose mention `open_id` differs from `botOpenId` makes `checkBotMentioned` return false, and the require-mention group path returns before dispatch. I did not run live Feishu validation in this read-only review.

## Solution Assessment

Unclear: the affected path is narrow, but the durable Feishu identity source is not settled and the open PR's union-id direction still needs proof. The best fix should use a supported stable identifier or cross-app alias strategy and carry it through all mention consumers.
