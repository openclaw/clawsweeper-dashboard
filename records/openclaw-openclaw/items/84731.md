## What This Changes

The issue reports that `/models` is slow with a large LiteLLM catalog because the command appears to use the full model catalog refresh path instead of the read-only cached catalog path.

## Best Possible Solution

Land a compatible fix through https://github.com/openclaw/openclaw/pull/84735 after preserving full-catalog behavior where required and attaching real runtime proof, or keep this issue as the canonical bug if that PR is closed.

## Solution Assessment

Yes on direction: reusing the existing read-only catalog contract for default browse is the narrowest maintainable fix. The implementation should still preserve explicit full-catalog cases and provider-wildcard compatibility before the linked PR closes this issue.
