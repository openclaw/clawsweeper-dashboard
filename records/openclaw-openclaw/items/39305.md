## Summary

Keep open. Current main and the latest release still lack the requested per-subagent tool-inactivity stall state, nudge threshold, kill threshold, per-spawn overrides, and parent notification policy; the linked implementation PR was closed unmerged, while the related activity/watchdog issues remain open design work rather than a shipped replacement.

## What This Changes

The issue requests two-stage subagent stall recovery based on time since last tool call, with a configurable nudge followed by automatic kill and per-spawn overrides.

## Best Possible Solution

Define the shared session/subagent activity model first, then add an opt-in subagent inactivity policy that records last tool activity, nudges through existing steering/session-message paths, kills through existing run termination, notifies the parent, and documents config plus per-spawn semantics.

## Solution Assessment

Unclear. The nudge-to-kill design is plausible, but the maintainable path should reconcile it with https://github.com/openclaw/openclaw/issues/39127 and https://github.com/openclaw/openclaw/issues/39141 before adding subagent-specific timers.
