## Summary

Keep open. Current main and the latest release still provide wall-clock `runTimeoutSeconds`, manual subagent steering/kill, orphan cleanup, and adjacent watchdog/activity proposals, but not the requested per-subagent tool-inactivity nudge/kill policy; the direct implementation PR at https://github.com/openclaw/openclaw/pull/43017 was closed unmerged.

## What This Changes

The issue requests configurable two-stage subagent stall recovery based on time since the child agent's last tool call, with nudge and kill thresholds plus per-spawn overrides.

## Best Possible Solution

Settle the shared session activity model first, then add an opt-in subagent inactivity nudge/kill policy through existing steer/termination and parent-notification paths with schema, docs, and focused tests.
