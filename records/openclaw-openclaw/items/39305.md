## Label Justifications

- `P1`: The June 10 production report shows a stuck sessions_spawn call making a live group conversation unresponsive for hours.
- `impact:session-state`: Parent and child runs can remain nonterminal without a reliable recovery transition.
- `impact:message-loss`: Subsequent channel messages remain queued with no acknowledgement or reply while the session is blocked.

## Summary

Keep open: current main still lacks activity-based sub-agent stall recovery, and the June 10 production report shows that a stuck child spawn can block an entire chat session for hours.

## What This Changes

Add escalating idle recovery for stalled sub-agents, with an automated nudge followed by termination and parent notification.

## Best Possible Solution

Define one lifecycle-owned progress model that can identify a genuine stall, emit the canonical terminal child outcome, release the parent queue, and optionally nudge before termination without adding parallel session state.

## Reproduction Assessment

Yes at source level plus recent production evidence: current main has an absolute timeout and stale classification but no activity-based nudge/kill path. A fresh controlled current-main live reproduction was not run.

## Solution Assessment

No, not exactly as proposed: two-stage recovery fits the symptom, but the progress signal must cover model and tool activity, and recovery should reuse canonical lifecycle and queue-release ownership rather than a parallel sweeper.
