## Summary

Keep open. Current main still lacks per-subagent tool-inactivity nudge-to-kill recovery: the configured subagent timeout surface is still wall-clock `runTimeoutSeconds`, `sessions_spawn` has no stall overrides, and the subagent run record has no last-tool-call or stall-phase state; the linked implementation PR at https://github.com/openclaw/openclaw/pull/43017 was closed unmerged while the broader activity/watchdog design items remain open.

## What This Changes

The issue requests configurable two-stage subagent stall recovery based on tool-call inactivity, with a nudge before automatic kill and per-spawn overrides.

## Best Possible Solution

Define a canonical child/session activity model first, then add accepted opt-in nudge/kill thresholds through existing child messaging and kill paths or fold the behavior into the broader session activity watchdog.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing behavior report. Source inspection gives a high-confidence current-main check that the requested config fields, per-spawn overrides, and per-subagent tool-inactivity recovery state are absent.

## Solution Assessment

Unclear. The two-stage nudge-to-kill approach is plausible, but maintainers should first settle the canonical activity model, defaults, and whether this belongs in targeted subagent runtime behavior or the broader session activity/watchdog design.
