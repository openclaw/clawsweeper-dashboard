## Summary

Keep open. Current main still has wall-clock subagent run timeouts, manual kill/steer controls, stale-run cleanup, task progress timestamps, and diagnostics, but it does not implement the requested per-subagent tool-inactivity nudge followed by automatic kill escalation.

## What This Changes

Request to add configurable two-stage subagent stall recovery based on time since last child tool call, with nudge and kill thresholds plus per-spawn overrides.

## Best Possible Solution

Define a canonical child activity model, then either implement accepted subagent nudge/kill thresholds through the existing child messaging and kill paths or fold the capability into the broader session activity/watchdog design with docs and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection on current main gives high confidence that the requested two-stage stall recovery is not implemented.

## Solution Assessment

Unclear. The two-stage approach is plausible, but maintainers should first choose whether it belongs as targeted subagent runtime behavior or as part of the broader #39127/#39141 activity/watchdog model.
