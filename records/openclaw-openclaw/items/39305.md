## Summary

Keep open. Current main has wall-clock subagent timeouts, manual steer/kill controls, stale-run cleanup, and adjacent diagnostics liveness recovery, but it still lacks the requested per-subagent tool-inactivity nudge/kill policy, config fields, per-spawn overrides, and durable stall state; the linked implementation PR is closed unmerged.

## What This Changes

The issue requests escalating stall recovery for sub-agents based on tool-call inactivity, with a nudge threshold followed by automatic kill and parent notification.

## Best Possible Solution

Settle the shared session/subagent activity model first, then add an accepted opt-in subagent nudge/kill policy through existing steer/kill and parent-notification paths with schema, docs, and focused tests.

## Solution Assessment

Unclear. The nudge-to-kill design is plausible, but the maintainable path should reconcile it with diagnostics recovery and the open session activity/watchdog design before adding subagent-specific timers.
