## Summary

Keep open. Current main still has wall-clock subagent run timeouts, stale/orphan cleanup, manual kill/control paths, and broader session liveness diagnostics, but not the requested per-subagent tool-inactivity nudge-to-kill recovery; the linked implementation PR was closed unmerged and the related activity/watchdog design issues remain open.

## What This Changes

Request to add configurable two-stage sub-agent stall recovery based on per-child tool-call inactivity, with nudge and kill thresholds plus per-spawn overrides.

## Best Possible Solution

Define a canonical child activity model, then implement accepted opt-in nudge/kill thresholds through the existing child messaging and kill paths or fold them into the broader session activity/watchdog design with docs and regression coverage.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing behavior report. Source inspection on current main gives a high-confidence verification path: the requested config fields, per-spawn overrides, and per-subagent tool-inactivity recovery state are absent.

## Solution Assessment

Unclear. The two-stage nudge-to-kill approach is plausible, but maintainers should first settle the canonical activity model, defaults, and whether this belongs in targeted subagent runtime behavior or the broader session activity/watchdog design.
