## Summary

Keep open. Current main still has wall-clock subagent run timeout, manual kill/steer, and coarse stale execution-context cleanup, but not the requested per-subagent tool-call inactivity tracking, nudge phase, kill escalation, config keys, or sessions_spawn overrides. The direct implementation PR #43017 was self-closed unmerged, and #39127/#39141 remain broader open activity/watchdog design items rather than shipped behavior.

## What This Changes

Add configurable two-stage subagent stall recovery that tracks last tool-call activity, nudges an idle child run, then auto-kills it if inactivity continues, with global defaults and sessions_spawn overrides.

## Best Possible Solution

Keep this open until maintainers either implement a targeted activity-based subagent recovery path or explicitly fold it into the broader session activity/watchdog design. The end state should have strict config/schema/type/docs coverage, sessions_spawn override semantics, deterministic child activity tracking, auditable nudge injection, and termination through the existing subagent kill/cancellation path.
