## Summary

Keep open: current main has adjacent timeout and diagnostic recovery work, but it still does not implement the requested sub-agent idle nudge/kill policy, default-on protection, or parent/channel-visible stall outcome.

## What This Changes

The issue requests two-stage sub-agent stall recovery that nudges an idle child run and then kills it if tool-call activity does not resume.

## Best Possible Solution

Define one maintainer-approved recovery design that reuses the canonical session/subagent activity owners, keeps config/default growth justified, and emits a parent-visible terminal outcome when recovery kills a stalled child.

## Reproduction Assessment

Yes, at source level plus production evidence, but not as a fresh current-main live repro. Current source still lacks the requested stall nudge/kill config while the June 10 logs show sessions_spawn blocking a Telegram group session with recovery=none.

## Solution Assessment

Unclear as written. The two-stage recovery direction fits the failure mode, but per-spawn timeout overrides and default-on killing need maintainer policy review and should probably reuse existing diagnostics/session activity rather than add a parallel state model.
