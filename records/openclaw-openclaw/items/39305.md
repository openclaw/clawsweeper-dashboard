## Summary

Keep open. Current main still has wall-clock subagent run timeouts, stale/orphan cleanup, task status, and manual kill controls, but not the requested per-subagent tool-inactivity nudge-to-kill recovery; the linked implementation PR was closed unmerged for more live testing, and the broader activity/watchdog designs remain open.

## What This Changes

Request adds configurable per-subagent tool-inactivity monitoring that nudges a stalled child and then kills it if it remains inactive.

## Best Possible Solution

Define a canonical child activity model, then implement accepted opt-in nudge/kill thresholds through the existing child messaging and kill paths or fold them into the broader session activity/watchdog design with docs and regression coverage.

## Reproduction Assessment

Not applicable; this is a feature request rather than a bug reproduction. Source inspection on current main gives high confidence that the requested config fields and per-subagent tool-inactivity recovery path are absent.

## Solution Assessment

Unclear. The two-stage approach is plausible, but maintainers should first settle the canonical activity model, defaults, and whether this belongs in targeted subagent runtime behavior or the broader session activity/watchdog design.
