## Summary

Keep open. Current main still has wall-clock subagent run timeouts, stale-run cleanup, manual send/steer/kill controls, and task diagnostics, but not the requested per-subagent tool-inactivity nudge followed by automatic kill escalation; the related implementation PR was closed unmerged and the broader activity/watchdog issues remain open.

## What This Changes

The issue requests configurable per-subagent tool-inactivity recovery with a nudge stage followed by automatic kill escalation.

## Best Possible Solution

Define one canonical child activity model, then either implement accepted subagent nudge/kill thresholds through the existing child messaging and kill paths or fold the capability into the broader #39127/#39141 activity/watchdog design with docs and regression coverage.
