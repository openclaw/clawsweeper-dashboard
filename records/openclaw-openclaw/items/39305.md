## Summary

Keep open. Current main still has wall-clock subagent run timeouts, stale/orphan run cleanup, lifecycle diagnostics, and manual steer/kill controls, but it does not implement per-subagent tool-inactivity tracking or the requested nudge-to-kill escalation.

## What This Changes

Request an opt-in per-subagent inactivity monitor that nudges stalled child agents and then kills them after a second threshold, with defaults and per-spawn overrides.

## Best Possible Solution

Define a canonical child activity model, then implement accepted opt-in nudge/kill thresholds through the existing child messaging and kill paths or fold them into the broader session activity/watchdog design with docs and regression coverage.

## Reproduction Assessment

Not applicable; this is a feature request rather than a bug reproduction. Source inspection on current main gives high confidence that the requested config fields and per-subagent tool-inactivity recovery path are absent.

## Solution Assessment

Unclear. The two-stage approach is plausible, but maintainers should first settle the canonical activity model and whether this belongs in targeted subagent runtime behavior or the broader session activity/watchdog design.
