## Summary

Keep open. Current main still lacks per-subagent tool-inactivity nudge-to-kill recovery; it has wall-clock `runTimeoutSeconds`, stale/orphan cleanup, and manual kill paths, while the linked implementation PR is closed unmerged and the broader activity/watchdog design items remain open.

## What This Changes

Request adds configurable per-subagent tool-inactivity stall recovery that first nudges a silent child run and later kills it, with defaults and per-spawn overrides.

## Best Possible Solution

Define a canonical child activity model, then implement accepted opt-in nudge/kill thresholds through existing child messaging and kill paths or fold them into the broader session activity/watchdog design.

## Reproduction Assessment

## Solution Assessment
