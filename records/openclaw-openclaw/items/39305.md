## Summary

Keep open: current main still lacks the requested two-stage sub-agent idle recovery, and the June 10 production report shows a real channel-impacting stall pattern that is not fully covered by current timeout, task, or watchdog work.

## What This Changes

The issue requests automatic sub-agent stall recovery that nudges an idle child run and then kills it if tool-call activity does not resume.

## Best Possible Solution

Define one maintainer-approved sub-agent recovery contract that reuses existing runTimeoutSeconds, session activity, task cancellation, and delivery semantics before adding stall nudge/kill config.

## Reproduction Assessment

No high-confidence live reproduction was run against current main. Source inspection confirms the requested automatic tool-idle nudge/kill surface is absent, and the June 10 production report gives credible real-world symptoms that still need live validation.

## Solution Assessment

Unclear as proposed. Runtime-side stall intervention is plausible, but default-on termination, per-spawn/config knobs, and channel-visible failure behavior need a maintainer product decision against the existing timeout and task/watchdog surfaces.
