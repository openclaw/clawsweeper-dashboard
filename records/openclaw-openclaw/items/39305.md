## Summary

Keep open: current main and the latest stable release have shipped adjacent hard subagent run-timeout enforcement and diagnostic stuck-session abort recovery, but they do not implement the requested two-stage subagent tool-inactivity policy, nudge delivery, kill behavior, per-spawn overrides, or user-visible channel stall handling.

## What This Changes

The issue requests escalating subagent stall recovery that nudges after tool-call inactivity and kills or reports the child if inactivity continues.

## Best Possible Solution

Make a maintainer decision on one bounded recovery model that reuses existing diagnostic/run-timeout primitives where possible, defines defaults and channel-visible behavior, and proves both fresh-install and upgrade behavior.

Yes for the missing requested surfaces by source inspection: current main lacks the stall nudge/kill config and subagent actions, while the live production comment gives a plausible real-world blocked sessions_spawn path that was not independently live-reproduced.

## Solution Assessment
