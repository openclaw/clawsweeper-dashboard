## Summary

Keep open: current main has hard `runTimeoutSeconds` handling and diagnostic active-abort recovery for blocked tool calls, but the requested default-on two-stage sub-agent nudge-to-kill policy, stall-specific config, and parent/user notification semantics are still not implemented and need maintainer product direction.

## What This Changes

The issue requests default-on two-stage sub-agent stall recovery based on tool-call inactivity, with a nudge first and eventual kill plus parent/user notification.

## Best Possible Solution

Consolidate around one maintainer-approved recovery policy that reuses the existing sub-agent lifecycle and diagnostics paths, then add only the nudge/default/notification pieces maintainers explicitly want.

## Reproduction Assessment

Partially: source inspection confirms the requested nudge/config surface is absent, and the June 10 comment gives concrete production stall logs. I did not run a live current-main Telegram reproduction, and current diagnostics now have a blocked-tool active-abort path.

## Solution Assessment

Unclear as proposed. The recovery goal is valid, but the best fix should consolidate with existing `runTimeoutSeconds`, diagnostic recovery, and session activity/watchdog surfaces before adding parallel stall config or per-spawn overrides.
