## Summary

Keep open. The intended fix from the source PR is valid, but this replacement enables flat-DM draft previews before the Slack send/edit path guarantees a DM channel ID that `chat.update` can use.

## What This Changes

The PR broadens Slack draft-preview streaming eligibility to every non-`off` mode, updates Slack streaming docs/changelog, and revises tests for flat DM previews.

## Best Possible Solution

Enable flat-DM draft previews only after the first draft send records a Slack DM conversation ID suitable for later edit/delete calls, with a regression test covering a `user:U...` top-level DM preview update.

## Reproduction Assessment

Yes. A source-level reproduction is clear: with the PR, a flat DM enters `createSlackDraftStream()`, the first `sendMessageSlack()` call has no `threadTs` and returns `U...` as `channelId`, then the next preview update calls `chat.update` with that user ID.

## Solution Assessment

No. Relaxing the preview eligibility gate is the right direction, but the patch needs to repair the Slack send/edit channel-id contract before this can be a reliable flat-DM streaming fix.
