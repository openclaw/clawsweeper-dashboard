## Summary

This issue still looks necessary: current main still builds terminal exec/bash tool-error payloads from the generic command summary, and cron delivery still selects that fatal error payload when there is no final assistant message. The latest release also has the same warning construction in the pre-refactor runner path, matching the reporter's `2026.5.26` environment.

## What This Changes

Issue requests clearer cron announce failure summaries for bash/shell tool failures that end without a final assistant message.

## Best Possible Solution

Add a narrow shell/exec terminal-failure fallback for cron announce failures that uses exit/error context and a short command excerpt, while keeping existing higher-level tool summaries for non-shell tools.

## Repair Work Prompt

Fix cron announce failure summaries for terminal bash/exec failures with no final assistant message. Current main builds an error payload in src/agents/embedded-agent-runner/run/payloads.ts via formatToolAggregate, then src/cron/isolated-agent/helpers.ts selects that fatal text for cron delivery; for shell-like tools this can produce garbled step-list summaries. Add a narrow cron-facing fallback that emits clear failure copy with shell/exec, exit/error context when available, and a short command excerpt, while preserving existing higher-level tool summaries and avoiding new config. Add focused regression coverage. Do not change channel delivery targets, config schema, or generic non-shell tool summaries.
