## Summary

This issue should stay open because current main and the latest release still route terminal cron exec/bash failures through the generic tool-summary text, while the open linked fix PR has not merged yet.

## What This Changes

The issue requests clearer cron announce failure text for raw bash/shell tool failures that end without a final assistant message.

## Best Possible Solution

Review, prove, and land or revise https://github.com/openclaw/openclaw/pull/92603, then close this issue as implemented once the fix reaches main.

## Reproduction Assessment

Yes, source-level: current main builds cron terminal tool-error payloads from the generic exec summary and promotes the last error payload when no final assistant answer recovers the run. I did not run a live cron-to-Discord reproduction in this read-only review.

## Solution Assessment

Yes, the best direction is a narrow cron shell terminal-failure formatter that bypasses generic shell paraphrasing while preserving useful summaries for non-shell tools. The linked PR is the active implementation path, with proof and excerpt-boundary review still pending.
