## Summary

Keep open. This is a focused plugin-owned crash fix for the linked OTLPExporterError bug, current main still lacks the diagnostics-otel handler, and I found no blocking correctness or security finding in the diff.

## What This Changes

The PR adds a diagnostics-otel unhandled-rejection handler for nested OTLPExporterError reasons, unregisters it on stop/restart, and adds service tests for Error, plain-object, and array-wrapped shapes.

## Best Possible Solution

Land the plugin-owned handler after normal CI remains acceptable, then close the linked crash issue as fixed by this PR.

## Reproduction Assessment

Yes, source-reproducible: current main has no diagnostics-otel rejection handler, the core fatal fallback exits on unmatched rejections, and OpenTelemetry 0.217.0 emits OTLPExporterError for non-retryable HTTP export failures. I did not run a live collector crash reproduction in this read-only pass.

## Solution Assessment

Yes. A diagnostics-otel-owned runtime-env handler is the narrowest maintainable fix because the plugin owns the OTLP dependency and the SDK already exposes a plugin-safe unhandled-rejection registration seam.
