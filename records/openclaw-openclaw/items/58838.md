## Summary

Keep this open. Current main still gives compaction chunks, staged split summaries, and the final merge separate retry budgets; the only merged related fix removed a duplicate partial fallback, while the circuit-breaker attempts for this issue were closed unmerged.

## What This Changes

Request a bounded compaction failure budget so persistent summarizer/model failures stop later compaction calls and fall back instead of burning tokens.

## Best Possible Solution

Add a per-compaction-run failure budget in the built-in LLM compaction path, shared across split chunks, staged summaries, dropped-history summaries, split-turn prefix summaries, and final merge fallback.

## Reproduction Assessment

Yes, at source level: mock `generateSummary()` to reject with a non-timeout provider error, call `summarizeInStages()` with enough messages to split, and current main will spend retry budgets across split summaries and the merge. I did not run the test because this review was required to keep the checkout read-only.

## Solution Assessment

Yes. A scoped run-level failure budget is the narrow maintainable fix; a broad global circuit breaker or new config knob would be a larger product decision and is not needed to stop the reported token burn.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/58838. Current main still applies `retryAsync({ attempts: 3 })` inside `summarizeChunks()` for each chunk, and `summarizeInStages()` calls `summarizeWithFallback()` separately for split chunks and the merge. Add a scoped compaction-run failure budget shared across `summarizeInStages()`, `summarizeWithFallback()`, and `summarizeChunks()` so persistent non-timeout summarizer/model failures stop later LLM calls and return the existing previous-summary or structured fallback behavior where available. Preserve AbortError and TimeoutError propagation, preserve provider-first behavior and provider fallback semantics, and do not add a new user config option or a global model-wide circuit breaker unless maintainers explicitly ask. Add regression coverage that mocks `generateSummary()` to reject with a non-timeout provider error and verifies a multi-split staged compaction does not spend independent retry budgets across every split plus merge. Keep the PR focused on compaction runtime/tests, with a changelog entry if the behavior is user-visible.
