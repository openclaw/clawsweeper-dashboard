## Summary

Keep this open. Current main and v2026.5.18 still give each built-in compaction chunk and staged merge pass its own retry budget, while the related circuit-breaker PRs were closed unmerged and the merged partial-fallback fix only removed one duplicate retry path.

## What This Changes

The issue asks OpenClaw to stop built-in compaction from spending independent retry budgets across chunks when the summarizer model is persistently unavailable.

## Best Possible Solution

Add a per-compaction-run failure budget shared across split chunks, dropped-history summaries, split-turn prefix summaries, and final merge fallback, while keeping abort/timeout behavior and existing provider semantics intact.

## Reproduction Assessment

Yes, at source level: mock `generateSummary()` to reject with a non-timeout provider error, call `summarizeInStages()` with enough messages to split, and current main will spend retry budgets across split summaries and the merge. I did not run the test because this review was required to keep the checkout read-only.

## Solution Assessment

Yes. A scoped run-level failure budget is the narrow maintainable fix; a broad global circuit breaker or new config knob would be a larger product decision and is not needed to stop the reported token burn.
