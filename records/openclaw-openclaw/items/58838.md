## Summary

Keep open: current main and the latest release still lack a staged compaction breaker, while the open closing PR is the active but unmerged fix candidate.

## What This Changes

The issue requests that compaction stop repeated summarizer calls after consecutive failures so unavailable summarizer models do not burn tokens across staged chunks.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/86900; the desired end state is a typed staged-compaction fallback breaker that preserves partial-summary recovery while avoiding repeated unavailable-summarizer calls.

## Reproduction Assessment

Yes, from source inspection. Current main still retries each chunk summary up to three times and summarizeInStages still processes every planned split plus a merge without a consecutive-failure breaker.

## Solution Assessment

Yes as a direction, but not as a close condition yet. A typed fallback result plus staged breaker is the narrow maintainable shape; the exact cutoff remains a maintainer session-state decision on the linked PR.
