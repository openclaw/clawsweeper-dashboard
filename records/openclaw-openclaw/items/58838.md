## Summary

Keep this open. Current main still spends the built-in compaction retry budget separately across chunked/staged summarization and the final merge when the summarizer keeps failing, and the linked circuit-breaker attempts are closed unmerged.

## What This Changes

The issue asks for compaction to stop later summarizer calls after repeated persistent failures so unavailable summarizer models do not burn tokens across chunks or stages.

## Best Possible Solution

Add a focused compaction-run failure budget shared by staged splits and the final merge, stopping later non-timeout provider/model calls and using the existing prior-summary or structured fallback behavior.

## Reproduction Assessment

Yes, at source level. A focused unit test can mock `generateSummary()` to reject with a non-timeout provider error, call `summarizeInStages()` with enough messages to split, and observe that current main still spends retry budgets across split summaries and the merge.

## Solution Assessment

Yes. A scoped built-in compaction failure budget is the narrow maintainable fix; a broad global circuit breaker would be riskier unless explicitly keyed and cooled down per compaction target.
