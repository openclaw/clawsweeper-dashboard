## Summary

Keep this open. Current main still spends the built-in compaction retry budget separately across staged split summaries and the final merge when the summarizer keeps failing, and the two linked circuit-breaker PR attempts were closed unmerged.

## What This Changes

The issue requests a compaction failure guard so persistent summarizer failures stop later split/merge LLM attempts and fall back without burning repeated retry budgets.

## Best Possible Solution

Add a focused failure budget in the built-in compaction summarization flow that is shared across staged split summaries and the merge step, stops later LLM calls after persistent non-timeout failures, and returns the prior summary or existing structured fallback while preserving history.

## Reproduction Assessment

Yes. A unit test can mock generateSummary() to reject with a non-timeout provider error and call summarizeInStages() with enough messages and parts to split; current main will spend the retry budget for each staged split and then the merge instead of opening a guard.

## Solution Assessment

Yes. A scoped built-in compaction guard is the narrow maintainable fix; a broad global circuit breaker would be riskier unless explicitly keyed and cooled down per compaction target.
