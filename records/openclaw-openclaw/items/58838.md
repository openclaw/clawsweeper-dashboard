## Summary

Keep open: current main and the latest release still run every staged compaction split through the fallback path without a consecutive-failure breaker, and the direct fix candidate is still an open, unmerged PR.

## What This Changes

The issue asks compaction to stop repeated summarizer attempts and fall back when the summarizer is persistently unavailable.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/86900; the desired end state is a typed staged-compaction fallback breaker that preserves partial-summary recovery while avoiding repeated unavailable-summarizer calls.

## Reproduction Assessment

Yes, from source inspection. Current summarizeInStages still calls summarizeWithFallback for every planned split, and summarizeChunks can spend three retry attempts per chunk before returning fallback text.

## Solution Assessment

Yes as an implementation shape, but not as a close condition yet. A typed fallback result plus staged breaker is the narrow maintainable direction, while the exact cutoff remains a maintainer session-state decision on the linked PR.
