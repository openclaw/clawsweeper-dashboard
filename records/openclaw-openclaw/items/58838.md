## Summary

Keep open. Current main and the latest release preserve partial summaries after mid-chain chunk failures, but they still do not stop repeated generic fallback attempts across staged compaction splits; an open linked PR is the active fix path for that remaining bug.

## What This Changes

The issue asks OpenClaw to stop repeated compaction summarizer retries from burning tokens when the summarizer model is persistently unavailable.

## Best Possible Solution

Review and either land or explicitly reject https://github.com/openclaw/openclaw/pull/86900; after a merged fix proves the staged fallback breaker on main, close this issue as implemented.

## Reproduction Assessment

Yes from source inspection, though no live run was executed in this read-only pass. `summarizeInStages()` still calls `summarizeWithFallback()` for every split, and each split can still invoke `summarizeChunks()` with a three-attempt retry budget before returning a generic fallback string.

## Solution Assessment

No, current main is only a partial fix for the broader compaction failure family. The best remaining path is a focused staged breaker or equivalent typed fallback mechanism that stops repeated generic fallback work while preserving the partial-summary behavior already on main.
