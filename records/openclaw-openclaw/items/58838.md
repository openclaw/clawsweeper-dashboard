## Summary

Keep open. Current main and the latest release still give compaction chunk, split, and merge summaries independent retry budgets; the two circuit-breaker PRs closed unmerged, and the open partial-summary PR addresses adjacent context preservation rather than a shared failure budget.

## What This Changes

The issue asks for compaction to stop spending retry budgets across chunk and staged summaries when the summarizer model is persistently unavailable.

## Best Possible Solution

Add a scoped per-compaction-run failure budget shared across summarizeChunks, summarizeWithFallback, and summarizeInStages, while preserving abort/timeout behavior and integrating cleanly with the partial-summary fallback if that PR lands.

## Reproduction Assessment

Yes, at source level: mock generateSummary to reject with a non-timeout provider error, call summarizeInStages with enough messages to split, and current main will spend retry budgets across split summaries and the merge. I did not run it because this review is read-only.

## Solution Assessment

Yes. A scoped run-level failure budget is the narrow maintainable fix; a broad global circuit breaker or new config option would be larger than needed for the reported token burn.
