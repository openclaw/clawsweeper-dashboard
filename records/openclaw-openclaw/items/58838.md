## Summary

Keep open. Current main and the latest release still process every staged compaction split through the three-attempt summarizer retry path, and the open linked fix PR is the active path for the remaining token-burn bug.

## What This Changes

The issue requests a compaction safeguard that stops repeated summarizer retries and falls back once the summarizer is persistently unavailable.

## Best Possible Solution

Review and either land or explicitly reject https://github.com/openclaw/openclaw/pull/86900; after a merged fix proves the staged fallback breaker on main, close this issue as implemented.

## Reproduction Assessment

Yes. Source inspection shows current main still calls `summarizeWithFallback()` for every staged split, and each split can still enter `summarizeChunks()` with a three-attempt retry budget.

## Solution Assessment

No for current main; it only preserves partial summaries after mid-chain failures. The best remaining path is a typed staged fallback breaker or equivalent mechanism, with maintainers accepting the threshold/session-state tradeoff before merge.
