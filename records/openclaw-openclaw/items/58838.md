## Summary

Keep open. Current main and the latest release still give each built-in compaction chunk and staged merge pass an independent retry budget, while the two circuit-breaker PRs were closed unmerged and the merged fallback fix only removed one duplicate partial retry path.

## What This Changes

Request a compaction failure budget so persistent summarizer model failures stop burning tokens across staged chunks and fall back safely.

## Best Possible Solution

Add a scoped per-compaction-run failure budget shared across split chunks, dropped-history summaries, split-turn prefix summaries, and the final merge pass, without adding a public config knob.

## Reproduction Assessment

Yes, at source level: mock `generateSummary()` to reject with a non-timeout provider error, call `summarizeInStages()` with enough messages to split, and current main will spend retry budgets across split summaries and the merge. I did not run that test because the review had to keep the checkout read-only.

## Solution Assessment

Yes. A scoped run-level failure budget is the narrow maintainable fix; a broad global circuit breaker or new config option would be larger than needed for the reported token burn.
