## Summary

Keep open: current main still has a source-level Matrix path where explicit reasoning replies are emitted by the agent layer but then suppressed before Matrix delivery, despite the documented `/reasoning on` behavior.

## What This Changes

Matrix should display a separate reasoning message when `/reasoning on` is enabled, while continuing to hide reasoning when it is off.

## Best Possible Solution

Add a narrow Matrix-aware delivery path for explicit `isReasoning` replies generated by enabled reasoning visibility, while preserving suppression of raw reasoning leaks and off/default reasoning payloads.

## Solution Assessment

Yes, a focused Matrix delivery fix is the best direction. Removing generic reasoning suppression would be too broad because the prior inverse leak shows why default channel suppression still matters.
