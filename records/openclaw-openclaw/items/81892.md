## Summary

Keep open: current main still has a source-level path where `/reasoning on` emits explicit reasoning payloads but Matrix delivery suppresses them before they can reach the room.

## What This Changes

The issue requests restoring Matrix delivery of explicit reasoning payloads when `/reasoning on` is enabled.

## Best Possible Solution

Add a narrow Matrix delivery path for explicitly flagged `isReasoning` payloads produced by `/reasoning on`, while keeping raw reasoning-tag leak suppression and generic non-Matrix suppression intact.

## Reproduction Assessment

Yes. Source inspection shows `/reasoning on` can emit an `isReasoning` block reply, but generic dispatch and Matrix delivery both suppress that payload before Matrix can send it.

## Solution Assessment

Yes. A focused Matrix opt-in for explicit reasoning payloads is the narrow maintainable fix; removing generic or Matrix suppression wholesale would risk reopening the earlier reasoning leak bugs.
