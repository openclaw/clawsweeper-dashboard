## Summary

Keep open: current main still has a source-level path where Matrix acknowledges `/reasoning on` but suppresses the reasoning payload before delivery, while the docs say enabled reasoning should be sent as a separate message.

## What This Changes

The issue reports that Matrix confirms `/reasoning on` but does not deliver generated reasoning blocks, even though the dashboard shows them.

## Best Possible Solution

Add a narrow Matrix opt-in for explicit `isReasoning` delivery when `/reasoning on` is active, while continuing to suppress raw reasoning-tag leaks and off/default reasoning payloads.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: enabled reasoning emits an `isReasoning` block reply, but generic dispatch and Matrix delivery both suppress that payload. I did not perform a live Matrix run.

## Solution Assessment

Yes, the best direction is a focused Matrix delivery fix that handles explicit reasoning payloads without reopening the prior off-mode leak. A broad removal of generic reasoning suppression would be riskier and would affect unrelated channels.
