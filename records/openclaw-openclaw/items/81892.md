## Summary

Keep open: current main still creates explicit reasoning payloads for `/reasoning on`, but both generic reply dispatch and Matrix delivery suppress those payloads before any Matrix send. The latest release has the same suppression, and the only linked fix PR is closed unmerged.

## What This Changes

Matrix `/reasoning on` should deliver generated reasoning as a separate Matrix message instead of only acknowledging the setting.

## Best Possible Solution

Add a narrow, capability-gated Matrix reasoning delivery path for explicit `isReasoning` payloads, while keeping `/reasoning off` and raw reasoning-looking text suppressed.

## Reproduction Assessment

Yes, by source inspection: `/reasoning on` persists the override and the runner emits `isReasoning` payloads, but generic dispatch and Matrix delivery both skip them before send. I did not run a live Matrix client reproduction in this read-only review.

## Solution Assessment

Yes, the maintainable fix is a narrow Matrix or channel-capability reasoning lane. Broadly allowing all reasoning payloads through generic delivery would be unsafe because the existing inverse Matrix leak behavior must remain suppressed.
