## Summary

Keep open: current main still produces explicit reasoning payloads for `/reasoning on`, but generic dispatch and Matrix delivery both suppress those payloads before Matrix can send them. The latest release has the same suppression, and the two linked fix PRs are closed unmerged.

## What This Changes

The issue asks for `/reasoning on` in Matrix to deliver generated reasoning blocks as a separate Matrix message instead of showing no reasoning.

## Best Possible Solution

Add a narrow Matrix or channel-capability opt-in that lets explicit `isReasoning` payloads reach Matrix as notice messages while preserving `/reasoning off` and raw reasoning-text suppression.

## Reproduction Assessment

Yes, by source inspection: `/reasoning on` persists the override and the runner emits `isReasoning` payloads, but generic dispatch and Matrix delivery both skip them before send. I did not run a live Matrix homeserver reproduction in this read-only review.

## Solution Assessment

Yes, the best fix is a narrow capability-gated Matrix reasoning lane. Changing all generic channel dispatch to deliver reasoning would be too broad because current suppression protects channels without a dedicated reasoning presentation path.
