## Summary

Keep open: current main still has a source-level path where Matrix accepts `/reasoning on` but both generic reply dispatch and Matrix delivery suppress explicit reasoning payloads, despite the docs promising a separate `Reasoning:` message when enabled.

## What This Changes

The issue asks for Matrix to deliver the separate reasoning message when `/reasoning on` is enabled while continuing to hide reasoning when off.

## Best Possible Solution

Keep the documented `/reasoning` contract and add a narrow Matrix delivery path for explicit `isReasoning` payloads emitted under enabled reasoning, while continuing to suppress raw reasoning-tag leaks and off/default reasoning.

## Reproduction Assessment

Yes, as a high-confidence source-level reproduction: enabled reasoning emits an `isReasoning` reply, Matrix calls the generic dispatcher, and both generic dispatch and Matrix delivery skip that payload. I did not establish a live Matrix client reproduction in this read-only review.

## Solution Assessment

Yes, the best path is a focused Matrix opt-in for explicit enabled reasoning delivery. A broad deletion of generic reasoning suppression is not the best fix because it would reopen the prior Matrix leak risk and affect unrelated channels.
