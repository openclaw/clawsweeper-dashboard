## Summary

Keep open: current main still has a source-level Matrix path where `/reasoning on` can emit an explicit reasoning block but both generic dispatch and Matrix delivery suppress it before it reaches the room.

## What This Changes

The issue asks for Matrix to deliver a separate reasoning message when `/reasoning on` is enabled, matching the documented reasoning visibility behavior.

## Best Possible Solution

Add a narrow Matrix-aware path for explicit `isReasoning` block delivery when reasoning visibility is on, while keeping raw reasoning-tag and off/default leak suppression intact.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: enabled reasoning emits an `isReasoning` block reply, but current generic dispatch and Matrix delivery both suppress that payload before Matrix sends it. I did not perform a live Matrix/Z.AI run.

## Solution Assessment

Yes, the best fix is a focused Matrix delivery opt-in for explicit reasoning payloads. Removing the generic reasoning suppression would be too broad and could reopen the prior Matrix leak.
