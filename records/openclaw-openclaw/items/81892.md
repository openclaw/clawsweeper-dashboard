## Summary

Keep open: current main and the latest release still suppress explicit Matrix reasoning payloads, and two open PRs now reference this issue with closing syntax, so the issue should remain open until one repair lands or is rejected.

## What This Changes

The issue asks for Matrix `/reasoning on` to deliver generated reasoning blocks as a separate Matrix message instead of dropping them.

## Best Possible Solution

Land one narrow Matrix repair that capability-gates explicit `isReasoning` delivery as `m.notice` while preserving generic/raw reasoning suppression and `/reasoning off`.

## Reproduction Assessment

Yes, by source inspection: `/reasoning on` is persisted and the runner emits `isReasoning` replies, but generic dispatch and Matrix delivery both suppress them before send. I did not run a live Matrix homeserver reproduction in this read-only pass.

## Solution Assessment

Yes, the best fix is a narrow Matrix or channel-capability opt-in for explicit reasoning payloads. Removing generic reasoning suppression globally would be too broad because it protects channels without a reasoning presentation lane.
