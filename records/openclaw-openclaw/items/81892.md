## Summary

Keep open: current main still has a source-level path where `/reasoning on` can emit a reasoning payload, but the generic dispatch path and Matrix delivery both suppress it before it reaches the Matrix room.

## What This Changes

Investigate a Matrix regression where `/reasoning on` acknowledges successfully but reasoning blocks are not delivered to Matrix while the dashboard still shows them.

## Best Possible Solution

Add a Matrix-specific path for explicit `isReasoning` payloads produced by enabled reasoning visibility, while keeping raw reasoning-tag and off/default reasoning suppression intact.

## Reproduction Assessment

Yes. Source inspection on current main shows enabled reasoning emits an `isReasoning` block reply, then generic dispatch and Matrix delivery both suppress that payload before send; I did not perform a live Matrix run.

## Solution Assessment

Yes. The narrow maintainable fix is a Matrix opt-in for explicit reasoning delivery, not a broad removal of generic reasoning suppression across channels.
