## Summary

Keep open. Current main still has the reported heartbeat model bleed-through path, and the linked closing PR is open and unmerged, so this issue should remain open until that or an equivalent fix lands.

## What This Changes

The issue asks for `agents.defaults.heartbeat.model` to behave as a heartbeat-turn-only model override instead of persisting as the active shared-session model after the turn completes.

## Best Possible Solution

Land PR #75567 or an equivalent narrow core fix that preserves or restores the prior shared-session runtime model and context metadata after heartbeat runs, with regression coverage around heartbeat persistence and `session_status`.

## Reproduction Assessment

Yes. Configure a shared-session heartbeat with `heartbeat.model` different from the agent primary model, let the heartbeat run, then inspect the shared session store or `session_status`; current main writes the heartbeat run's runtime identity back to the shared session.

## Solution Assessment

Yes, directionally. A heartbeat-only preserve/restore path in session persistence is the narrowest maintainable fix; changing model selection semantics or requiring `isolatedSession` would not address the shared-session persistence bug as directly.
