## Summary

Keep open. Current main still applies `heartbeat.model` for the heartbeat run, then persists the run result's provider/model/context metadata back into the shared session store; the linked closing PR #75567 is still open, so this issue should remain open until that or an equivalent fix lands.

## What This Changes

The issue asks for `agents.defaults.heartbeat.model` to behave as a single-heartbeat-turn override without leaving subsequent shared-session user turns on the heartbeat model.

## Best Possible Solution

Land PR #75567 or an equivalent narrow core fix that preserves or restores the prior shared-session runtime model and context metadata after heartbeat runs, with regression coverage around heartbeat persistence and `session_status`.

## Reproduction Assessment

Yes. The code-level path is high confidence: configure a shared-session heartbeat with a different `heartbeat.model`, let the heartbeat run, then inspect the shared session store or `session_status`; current main writes the heartbeat run's runtime identity back to the shared session.

## Solution Assessment

Yes directionally. A heartbeat-only preserve/restore path in session persistence is narrower than changing model selection semantics or requiring `isolatedSession`, but the linked PR still needs review before this issue can close.
