## Summary

Keep open. Current main still scopes `heartbeat.model` to heartbeat model selection, then persists the heartbeat run's runtime model and context metadata back into the shared session; #75567 is open with closing syntax, so this should wait for that PR or an equivalent fix to land.

## What This Changes

The issue requests that `agents.defaults.heartbeat.model` apply only to the heartbeat turn and not become the active model for later shared-session user turns.

## Best Possible Solution

Land #75567 or an equivalent narrow core fix that preserves or restores shared-session runtime model and context metadata after heartbeat turns, with regression coverage for `session_status`.

## Reproduction Assessment

Yes. Configure a shared-session heartbeat with `heartbeat.model` different from the agent primary, let the heartbeat run, then inspect `session_status` or the session store; the source path still selects the heartbeat model and persists that runtime identity.

## Solution Assessment

Yes directionally. A heartbeat-specific preserve/restore path in session persistence is narrower than changing model selection semantics or requiring isolated heartbeat sessions for correctness.
