## Summary

Keep open. Current main still scopes `heartbeat.model` only during model selection, then persists the heartbeat run's runtime model and context metadata back into the shared session; the linked closing PR #75567 is open, so this issue should stay open until that or an equivalent fix lands.

## What This Changes

The issue asks for `agents.defaults.heartbeat.model` to apply only to heartbeat turns instead of persisting as the active shared-session model.

## Best Possible Solution

Land PR #75567 or an equivalent narrow core fix that preserves or restores the prior shared-session runtime model and context metadata after heartbeat runs, with regression coverage for heartbeat persistence and `session_status`.

## Reproduction Assessment

Yes. Configure a shared-session heartbeat with a different `heartbeat.model`, let the heartbeat run, then inspect `session_status` or the session store; current main selects the heartbeat model and then writes that runtime identity back to the shared session.

## Solution Assessment

Yes directionally. A heartbeat-specific preserve/restore path in session persistence is narrower than changing model selection semantics or requiring isolated sessions for correctness, and #75567 is the active implementation candidate.
