## Summary

Keep open. Current main still routes `heartbeat.model` as a run-scoped selection override, but the post-run session-store update persists the heartbeat run's runtime model and context metadata back onto the shared session. The open closing PR #75567 is the active implementation candidate, so this issue should remain open until that or an equivalent fix lands.

## What This Changes

The issue requests that `agents.defaults.heartbeat.model` apply only to the heartbeat turn and stop downgrading subsequent shared-session user turns to the heartbeat model.

## Best Possible Solution

Merge PR #75567 or an equivalent narrow core fix that preserves the prior shared-session runtime model/provider/context metadata after heartbeat turns while keeping normal user turns and explicit model changes persistent.

## Reproduction Assessment

Yes. Configure a shared-session heartbeat with `heartbeat.model` different from the primary model, let the heartbeat run, then inspect `session_status`; current-main source shows the heartbeat run's runtime model is persisted back to the shared session.

## Solution Assessment

Yes directionally. A heartbeat-scoped preservation option in session persistence is the narrowest maintainable fix; changing model-selection semantics would not address the post-run persistence overwrite.
