## Summary

Keep open. Current main still treats `heartbeat.model` as run-scoped during model selection but then persists the heartbeat run's runtime model and context metadata back into the shared session; the linked closing PR #75567 is open and should be reviewed or replaced before this issue is resolved.

## What This Changes

The issue requests that `agents.defaults.heartbeat.model` apply only to the heartbeat turn and not remain as the session's active runtime model afterward.

## Best Possible Solution

Land PR #75567 or an equivalent narrow core fix that preserves or restores the pre-heartbeat shared-session runtime model and context metadata after heartbeat runs, with regression coverage for `session_status` and session persistence.

## Reproduction Assessment

Yes. Configure a shared-session heartbeat with a different `heartbeat.model`, let the heartbeat run, then inspect `session_status` or the persisted session entry; the current code path selects the heartbeat model and then writes that runtime identity back to the shared session.

## Solution Assessment

Yes, directionally. Preserving the prior runtime model/context fields during heartbeat persistence is narrower than changing model-selection semantics or requiring isolated heartbeats for correctness.
