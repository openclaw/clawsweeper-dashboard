Keep open for normal maintainer review: current main still drops QQ Bot C2C/group media that only appears under non-quote `msg_elements`, while this PR is a narrow plugin-owned fix with focused coverage and no actionable review findings.

The PR updates the QQ Bot dispatcher to fall back from top-level attachments to non-quote `msg_elements` attachments for C2C and group messages, adds dispatcher regression tests, and records the fix in the changelog.

## Best Possible Solution

Land a narrow QQ Bot-owned dispatcher fix that surfaces current-message `msg_elements` media as queued inbound attachments while keeping quote `msg_elements` reserved for reply metadata.

## Solution Assessment

Yes. Resolving the fallback in the QQ Bot dispatcher is the narrowest maintainable boundary because downstream media handling already works once queued messages carry attachments, and the patch stays inside the owning plugin.
