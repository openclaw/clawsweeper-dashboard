## Summary

Keep open: the PR is a plausible QQ Bot bug fix, but a recent maintainer live C2C run did not exercise the proposed fallback and the current patch can still promote quote `msg_elements` media as current-message attachments when quote markers are carried in `message_scene.ext`.

## What This Changes

The PR changes QQ Bot C2C and group inbound dispatching to fall back from top-level attachments to media nested under `msg_elements`, adds dispatcher tests, and adds a changelog entry.

## Best Possible Solution

Confirm the real QQ payload shape first; if current-message `msg_elements` media is real, update the dispatcher to distinguish quote/ref metadata from current media and add that regression coverage, otherwise rescope this as defensive normalization and stop claiming the linked issue is fixed.

## Reproduction Assessment

No high-confidence live reproduction is available for the original QQ 2026.5.x symptom. There is a source-level reproduction with a constructed C2C payload that puts media only under `msg_elements`, but the latest live maintainer run delivered C2C image media through top-level `attachments`.

## Solution Assessment

No. Normalizing in the QQ Bot dispatcher is the right area, but this patch needs stronger quote/ref detection and a real or redacted payload that proves the fallback fixes the reported C2C image loss.

- **[P2] Exclude ext-identified replies from msg_elements fallback:** `extensions/qqbot/src/engine/gateway/event-dispatcher.ts:187-188`
