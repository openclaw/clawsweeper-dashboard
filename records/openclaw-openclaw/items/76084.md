## Summary

Keep this PR open: it is a focused QQ Bot fix candidate, but a later member review supplied live C2C evidence that current traffic uses top-level attachments and identified a concrete quote-detection gap in the new fallback.

## What This Changes

The PR changes QQ Bot C2C and group dispatching to populate queued inbound attachments from msg_elements when top-level attachments are absent, with a dispatcher test and changelog entry.

## Best Possible Solution

Keep the PR open, either rescope it as defensive normalization with a broader quote guard and regression test, or wait for a redacted live payload proving msg_elements carries current-message media on current 2026.5.x QQ Bot traffic.

## Reproduction Assessment

Yes for the source-level gap: a dispatcher payload with media only in msg_elements still drops queued attachments on current main. No for the reported live QQ Bot symptom on current 2026.5.x, because the latest member-run C2C image flow delivered media in top-level attachments instead.

## Solution Assessment

No as currently written. The dispatcher is the right place for a defensive fallback, but it should not flatten msg_elements when message_scene.ext marks the event as a quote/reference, and the linked bug claim needs a real payload or rescope.
