## Summary

Keep open: current main still drops QQ Bot media that arrives only under `msg_elements`, and the PR is a narrow dispatcher-layer fix with focused regression coverage and no blocking review finding found.

## What This Changes

The PR normalizes QQ Bot C2C and group `msg_elements` media into queued inbound attachments when top-level attachments are absent, adds dispatcher tests, and updates the changelog.

## Best Possible Solution

Land the dispatcher normalization with the added regression tests so QQ Bot private-chat and group media consistently reach the existing inbound media pipeline.

## Reproduction Assessment

Yes. On current main, a `C2C_MESSAGE_CREATE` payload with `message_type: 7` and only `msg_elements[0].attachments` produces a queued message with no `attachments`; the new test fixture directly covers that path.

## Solution Assessment

Yes. Resolving attachments at the dispatcher boundary is the narrowest maintainable fix because downstream processing already uses `QueuedMessage.attachments`, while quote-stage continues to own quoted `msg_elements` metadata.
