## Summary

Keep open: current main still returns early from both gateway startup and doctor repair based only on the runtime-deps scan missing list, and two open implementation PRs still reference this issue. The issue should close after a chosen fix lands.

## What This Changes

The issue requests that gateway startup and `openclaw doctor --repair` detect and recover from corrupt or partially staged bundled runtime dependency installs left by interrupted update staging.

## Best Possible Solution

Land one runtime-deps fix that makes startup and doctor distinguish a fully usable staged install from a partially written package tree, with regressions for both early-return paths, then close this issue through the merged PR.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to create a packaged install root with `node_modules/<dep>/package.json` satisfying the scan while generated materialization or package contents are incomplete, then observe gateway startup and doctor repair skip repair because `missing.length === 0`.

## Solution Assessment

Unclear until the open PRs are resolved. The right boundary is to gate the gateway and doctor fast paths on real staged-install completeness, but the safest implementation should avoid treating a pre-install generated manifest or package sentinel as a post-success marker.
