## Summary

Keep open: the core Discord interaction seam may be useful, but this PR currently exposes a Discord-only `/ask` feature as a global plugin command and needs maintainer product review on whether the bundled plugin belongs in core or should ship externally.

## What This Changes

Adds a bundled Ask plugin with a Discord `/ask` command, a new `plugin-sdk/discord-interactions` subpath, Discord native-command component reply handling, tests, docs, and workspace metadata.

## Best Possible Solution

Land a narrow, documented Discord interaction SDK/native-command component seam if maintainers want that API, and keep `/ask` Discord-scoped or publish it externally through ClawHub unless core bundling is explicitly approved.

## Reproduction Assessment

Yes for the main review finding: source inspection shows unscoped plugin commands support every channel and `nativeNames.default` applies cross-provider. Live `/ask` behavior itself has not been reproduced because the PR only provides build/dist proof.

## Solution Assessment

No as written; the maintainable path is to separate the reusable Discord interaction API from the optional Ask feature, scope any bundled command to Discord, and document the public SDK surface.
