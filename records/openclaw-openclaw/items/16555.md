## Summary

Keep open. Current main still replays otherwise eligible file-backed outbound delivery queue entries without a queued-age cutoff, and the strict messages config still has no TTL/expiry surface; the related SQLite/outbox PRs are closed and unmerged prior art rather than canonical fixes.

## What This Changes

Add an optional TTL/expiry policy for persisted outbound delivery queue entries so stale messages are not replayed after gateway restarts.

## Best Possible Solution

Implement a narrow optional expiry policy on the existing file-backed outbound delivery queue, enforced in both startup recovery and reconnect drains while preserving default no-expiry behavior.

## Reproduction Assessment

Yes. Seed a `delivery-queue/*.json` entry with an old `enqueuedAt` and otherwise eligible retry/backoff state, then invoke startup recovery or a matching reconnect drain; source inspection shows current main will still consider it for delivery because no age check exists.

## Solution Assessment

Yes. A narrow optional TTL/expiry policy on the current file-backed queue is the maintainable fix; the broader SQLite/outbox rewrites are closed and unmerged, so they should be treated as prior art rather than the active solution.
