## Summary

Keep open. Current main still replays old first-attempt outbound delivery-queue entries with no age cutoff, the requested `messages.delivery` TTL config surface is absent, and https://github.com/openclaw/openclaw/pull/54900 is the focused but unmerged implementation candidate.

## What This Changes

Add configurable TTL/expiry for outbound delivery queue messages so stale queued entries are skipped or quarantined during recovery.

## Best Possible Solution

Land or replace the focused shared outbound recovery fix so stale pending deliveries are skipped or quarantined while recent crash recovery remains intact, then decide whether an operator-facing config surface is needed.

## Reproduction Assessment

Yes, at source level. Seed an old pending delivery entry with `retryCount: 0` and no `lastAttemptAt`; current main marks it eligible and startup recovery calls the delivery path.

## Solution Assessment

Unclear as proposed. The shared outbound recovery boundary is the right place, but TTL defaults, expiry action semantics, and the open implementation candidate's review blockers need resolution before this can be considered solved.
