## Summary

Keep open: current main and the latest tagged release still lack an outbound delivery-queue age/TTL policy or accepted config surface, and the prior focused TTL PR is closed unmerged.

## What This Changes

Request adds configurable TTL/expiry for persisted outbound delivery queue messages so stale pending sends are not replayed after gateway restart.

## Best Possible Solution

Choose the outbound expiry policy first, then add a narrow current-schema implementation for the SQLite-backed delivery queue with startup recovery, reconnect drain, storage, config, logging, and release-note-ready PR context.

## Reproduction Assessment

Yes, at source level: seed an old pending outbound delivery row with retryCount 0 and no lastAttemptAt, then startup recovery remains replay-eligible because there is no age cutoff. I did not run a live gateway reproduction because this is a read-only review.

## Solution Assessment

Unclear until maintainers choose the policy: the owner boundary is narrow, but the requested config/default/expire semantics and whether inbound freshness is included remain product decisions.
