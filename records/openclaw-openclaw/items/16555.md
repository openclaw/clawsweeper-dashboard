## Summary

Keep open: current main and the latest release still lack an outbound delivery-queue age or expiry gate, while the focused open implementation candidate remains unmerged and needs policy review.

## What This Changes

Add operator-configurable expiry for outbound delivery queue entries so stale queued messages are skipped or quarantined during gateway restart recovery instead of being replayed.

## Best Possible Solution

Land or replace a focused outbound delivery expiry policy that quarantines stale queued replies while preserving recent crash recovery, reconnect drains, budget deferrals, and uncertain-send reconciliation.

## Reproduction Assessment

Yes, at source level. Seed an old pending delivery entry with retryCount: 0 and no lastAttemptAt, then run recoverPendingDeliveries; current main treats it as eligible and gateway startup calls that path.

## Solution Assessment

Unclear as proposed. The outbound recovery boundary is right, but the TTL duration, default, expire action, config location, and inbound-symmetric scope still need maintainer alignment before https://github.com/openclaw/openclaw/pull/54900 or a replacement can close this.
