## Summary

Keep open. Current main still replays old pending outbound delivery entries without a queued-age or expiry gate, and the focused implementation candidate is still open rather than merged.

## What This Changes

The issue requests configurable TTL/expiry handling for outbound delivery queue entries so stale queued messages are not replayed after gateway restarts.

## Best Possible Solution

Land or replace a focused outbound expiry policy that quarantines stale queued replies while preserving recent crash recovery, reconnect drains, budget deferrals, and uncertain-send reconciliation.

## Reproduction Assessment

## Solution Assessment

Unclear as proposed. The outbound recovery boundary is right, but the TTL duration, default, expire action, reconnect behavior, and uncertain-send reconciliation still need maintainer alignment.
