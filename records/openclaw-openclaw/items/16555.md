## Summary

Keep open: current main still replays old pending outbound delivery queue entries without an age gate, there is no accepted config surface for TTL/expiry, and the focused fix candidate remains open rather than merged.

## What This Changes

The issue requests configurable TTL or expiry for delivery queue messages so stale queued outbound messages are not replayed after gateway restart.

## Best Possible Solution

Keep this as the canonical policy issue, then land or replace the focused outbound expiry fix once maintainers decide the default, config shape, expire action, and inbound scope.

## Reproduction Assessment

Yes, at source level. Seed an old pending delivery entry with retryCount 0 and no lastAttemptAt, then run recoverPendingDeliveries; current main treats it as eligible and gateway startup calls that path.

## Solution Assessment

Unclear as proposed. The outbound recovery boundary is correct, but the requested TTL default, config shape, expire action, and inbound scope are product-policy decisions before the open candidate or a replacement can close this.
