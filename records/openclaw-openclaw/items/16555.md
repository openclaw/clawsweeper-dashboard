## Summary

Keep open: current main still replays eligible pending outbound delivery entries with retry/backoff but no queued-age expiry policy, strict messages config lacks `messages.delivery.*`, and related implementation PRs are closed unmerged while the live issue is marked for maintainer product review.

## What This Changes

The issue requests a configurable TTL/expiry policy for outbound delivery-queue entries so stale queued messages are skipped, deleted, or failed during gateway recovery.

## Best Possible Solution

Keep this as the canonical product/implementation issue, decide the TTL config/default/expire-action policy, then land a narrow current-main outbound expiry change that covers startup recovery and reconnect drain with config schema, docs, and focused tests.
