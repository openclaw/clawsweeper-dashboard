## What This Changes

Request adds a configurable TTL/expiry policy for stale outbound delivery queue messages replayed during gateway recovery, with follow-up discussion about symmetric inbound freshness handling.

## Best Possible Solution

Land or replace the focused outbound recovery expiry policy, then handle inbound freshness as an explicit scoped follow-up if maintainers want symmetry.
