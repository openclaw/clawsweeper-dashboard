## Summary

## What This Changes

The issue requests a `SessionSendPolicy` predicate that can compare an outbound destination peer to the inbound peer for the active agent turn.

## Best Possible Solution

Have maintainers approve a turn-scoped outbound-routing policy design, then implement the chosen predicate or alternative with schema/types/help/docs and focused tests that preserve intentional docking and define no-inbound behavior.

No high-confidence runtime reproduction of a wrong-peer send was established against current main. Source inspection does give a high-confidence reproduction of the capability gap: the schema/types reject the requested predicate and the evaluator has no inbound/outbound peer inputs.

## Solution Assessment

Unclear: a native relational predicate is plausible and better integrated than a userland hook, but the exact public API, no-inbound semantics, docking interaction, and group/thread scope need maintainer design approval.
