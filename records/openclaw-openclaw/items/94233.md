## What This Changes

The PR adds bounded in-memory coalescing for duplicate auth/auth_permanent model fallback decision warnings and regression coverage for suppression, scoping, and passthrough behavior.

## Best Possible Solution

Land this logger-layer coalescer if maintainers accept summarized duplicate auth diagnostics, while keeping broader provider quarantine/status policy in https://github.com/openclaw/openclaw/issues/47910.

## Reproduction Assessment

## Solution Assessment

Yes. For the narrow log-spam bug, coalescing at the fallback decision logger is the smallest maintainable fix because it leaves auth cooldown, provider selection, fallback routing, and fallback-step callbacks unchanged.
