Investigate a beta update audit where a migrated `openai/gpt-5.5` primary still failed on `openai-codex` auth before fallback won.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Unclear. The best fix is not a blanket removal of `openai-codex` auth lookup, because current docs and tests make that lookup intentional for some routes; maintainers should first separate stale migration state from expected Codex-auth routing and fallback-audit visibility.
