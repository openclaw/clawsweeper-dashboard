## What This Changes

This PR changes `warnWhenAllowlistIsOpen` to emit a specific warning when a non-empty `plugins.allow` has no discovered plugin-id match and adds loader tests for mismatch, partial-match, and bundled-id allowlist cases.

## Best Possible Solution

Land the narrow helper and regression tests after required checks are green or explicitly waived, while leaving broader empty-allowlist remediation to https://github.com/openclaw/openclaw/issues/68780 and https://github.com/openclaw/openclaw/pull/78105.

## Reproduction Assessment

Yes, source-reproducible: current main exits `warnWhenAllowlistIsOpen` for any non-empty `plugins.allow`, so a channel-style or otherwise unknown id cannot get mismatch-specific guidance. I did not run tests because this review is read-only.

## Solution Assessment

Yes for the narrow bug: comparing allow entries to discovered plugin ids in the warning helper preserves allow/deny semantics, keeps valid bundled allowlist entries quiet, and avoids absorbing the broader first-run warning UX work.
