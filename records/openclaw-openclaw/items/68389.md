The PR changes the plugin loader allowlist warning so populated allowlists with no discovered plugin-id match get a specific mismatch warning, with focused loader tests for mismatch, partial-match, and bundled-id cases.

## Best Possible Solution

Land the focused helper and regression tests once required checks are green or explicitly waived, while keeping the broader first-run allowlist remediation tracked in https://github.com/openclaw/openclaw/issues/68780 and https://github.com/openclaw/openclaw/pull/78105.

## Reproduction Assessment

Yes, source-reproducible: on current main, a full active `loadOpenClawPlugins` call with a workspace/global plugin and a nonmatching non-empty `plugins.allow` exits the helper before any mismatch-specific warning can be emitted. I did not run tests because this review is read-only.

## Solution Assessment

Yes for the helper-level bug: the PR compares allow entries to discovered plugin ids, preserves the existing empty-allowlist warning, and avoids changing allow/deny policy semantics. The broader gateway first-run warning UX is intentionally separate work.
