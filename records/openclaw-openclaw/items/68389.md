## Best Possible Solution

Land this narrow diagnostic after required checks are green or explained, preserving the legacy empty-allowlist behavior and leaving broader first-run remediation to https://github.com/openclaw/openclaw/pull/78105 or its successor.

## Reproduction Assessment

I did not run tests because this review is read-only, but the PR body includes built-loader proof for the changed path.

## Solution Assessment

Yes. The warning helper is the narrow owner boundary for this diagnostic; channel-id aliasing, auto-load policy changes, or new config would be broader product/security decisions tracked separately.
