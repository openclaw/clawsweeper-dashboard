## Summary

Keep open: current main still has the reported source-level divergence, where `sessions.get` reads only the OpenClaw transcript file while `chat.history` hydrates Claude CLI-backed history. The related open PR addresses transcript-only artifact filtering, not CLI transcript hydration or the misleading `sessionFile` contract.

## What This Changes

Request that Gateway transcript reads and session metadata clearly distinguish wrapper transcripts from hydrated CLI chat history, or align `sessions.get` with `chat.history` for CLI-backed sessions.

## Best Possible Solution

Define the Gateway transcript contract additively, then either share `chat.history` CLI hydration from `sessions.get` with an explicit wrapper-transcript escape hatch or document/expose wrapper and CLI transcript metadata as separate scopes.

## Reproduction Assessment

Yes. A session entry with `cliSessionBindings['claude-cli'].sessionId` reaches Claude CLI import through `chat.history`, while `sessions.get` reads only `entry.sessionFile` on current main.

## Solution Assessment

Unclear. Proxying `sessions.get` to `chat.history` would address the empty transcript symptom, but changing endpoint semantics or adding `wrapperOnly`/transcript metadata is a public Gateway API decision.
