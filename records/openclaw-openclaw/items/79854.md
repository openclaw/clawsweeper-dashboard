## Summary

Keep open: current main still exposes different transcript scopes for CLI-backed sessions, and the related open PR only filters internal transcript artifacts rather than resolving CLI hydration or `sessionFile` semantics.

## What This Changes

The issue requests a clearer Gateway transcript contract because `sessions.get` returns only the OpenClaw wrapper transcript while `chat.history` returns CLI-hydrated conversation history, and `sessionFile` implies a full transcript file.

## Best Possible Solution

Define an additive Gateway transcript contract that either shares CLI hydration with an explicit wrapper-only escape hatch, or clearly separates wrapper transcript metadata from the full conversation API exposed by `chat.history`.

## Reproduction Assessment

Yes for the observed divergence. Source inspection on current main shows `sessions.get` reads only `entry.sessionFile`, while `chat.history` additionally hydrates a bound Claude CLI session through `augmentChatHistoryWithCliSessionImports`.

## Solution Assessment

Unclear. Proxying `sessions.get` to `chat.history` would address the empty-conversation symptom, but endpoint semantics, wrapper-only compatibility, and persisted transcript metadata naming need a maintainer API-contract decision first.
