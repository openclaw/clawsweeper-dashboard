## Summary

Keep open: current main still has the reported Gateway contract mismatch. `sessions.get` reads the OpenClaw transcript target only, while `chat.history` imports bound Claude CLI project history before display projection, so this still needs a maintainer product/API decision.

## What This Changes

The issue asks OpenClaw to resolve or document the mismatch between `sessions.get` wrapper-only transcript reads and `chat.history` hydrated CLI conversation reads, including the misleading `sessionFile` contract.

## Best Possible Solution

Define the public Gateway transcript contract, then add a compatibility-aware implementation such as explicit transcript-scope metadata or an additive wrapper-only mode, with docs and tests for CLI-backed sessions.

## Reproduction Assessment

Yes. Source inspection on current main shows `sessions.get` reads only the OpenClaw transcript scope while `chat.history` imports bound Claude CLI project messages; I did not run the live macOS Claude CLI repro.

## Solution Assessment

Unclear. A direct proxy to `chat.history` fixes the empty-conversation symptom, but wrapper-only access and `sessionFile` naming are public Gateway contract choices that should be decided additively.
