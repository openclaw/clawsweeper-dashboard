## Summary

Keep open: current main still has the CLI-backed transcript split, and the remaining fix is a Gateway API contract decision rather than conservative cleanup.

## What This Changes

The issue asks for `sessions.get` and session metadata to stop misleading gateway consumers when CLI-backed sessions keep hydrated chat history in provider-owned JSONL files rather than the OpenClaw wrapper transcript.

## Best Possible Solution

Define an additive Gateway transcript contract that either hydrates CLI-backed `sessions.get` results with an explicit wrapper-only escape hatch or clearly exposes separate wrapper and CLI transcript metadata in docs and response fields.

## Reproduction Assessment

Yes. Source inspection shows `sessions.get` still reads `entry.sessionFile` directly, while `chat.history` imports Claude CLI history through `cliSessionBindings` before projecting messages.

## Solution Assessment

Unclear. Proxying `sessions.get` to `chat.history` would address the observed empty transcript, but the default endpoint semantics, wrapper-only escape hatch, and `sessionFile` naming are public protocol choices that need maintainer direction.
