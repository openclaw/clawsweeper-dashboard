## What This Changes

The PR makes cached plugin tool execution skip malformed runtime sibling entries and adds a regression test proving the healthy requested cached tool still runs.

## Best Possible Solution

Land the narrow cached-path alignment after routine maintainer and CI checks, keeping cached descriptor execution consistent with the non-cached plugin resolver.

## Reproduction Assessment

Yes. From source, current main can reproduce by resolving a plugin once to cache descriptors, resolving again from cache, then executing a cached requested tool whose factory returns a malformed sibling before the healthy tool.

## Solution Assessment

Yes. Skipping malformed entries inside the cached candidate loop is the narrow maintainable fix because it mirrors the existing non-cached resolver behavior and avoids a new API or config surface.
