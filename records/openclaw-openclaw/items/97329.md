## Summary

Keep open: current main and v2026.6.10 still inject inbound channel metadata with no supported opt-out, and the requested config/control shape remains a product decision rather than an already-implemented cleanup.

## What This Changes

Request to make non-webchat inbound metadata prompt injection configurable so users can reduce repeated channel metadata in agent context.

## Best Possible Solution

Choose one core inbound metadata policy that preserves routing data internally, reduces or hides model-visible metadata where safe, and adds a narrow documented opt-out only if maintainers accept the channel-behavior tradeoffs.

## Reproduction Assessment

Yes, at source level. Current main still builds the trusted inbound_meta.v2 system block and the non-webchat Conversation info user-context block with no supported config opt-out.

## Solution Assessment

Unclear. A config opt-out is plausible, but maintainers should first choose whether the best durable shape is hidden runtime context, compaction, per-channel policy, or a strict metadata visibility toggle.
