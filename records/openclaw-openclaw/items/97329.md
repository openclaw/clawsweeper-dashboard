## Summary

Keep open: current main and the latest release still inject trusted inbound metadata plus user-prefix conversation metadata with no supported opt-out, while the best product direction is still unsettled against the hidden-runtime-context alternatives.

## What This Changes

Request to add an openclaw.json-controlled way to reduce or disable non-webchat inbound channel metadata in the agent system prompt and user-message prefix.

## Best Possible Solution

Choose one inbound-context policy: either land the hidden/sidecar runtime-context direction or add a narrow opt-in config that preserves essential metadata, with schema, docs, and prompt/regression tests kept aligned.

## Reproduction Assessment

Yes at source level. Current main calls the inbound metadata builders on every reply path and the builders still emit inbound_meta.v2 plus Conversation info for non-webchat channel turns without a supported opt-out.

## Solution Assessment

Unclear. A config opt-out is plausible, but the hidden-runtime-context path may reduce prompt waste without adding a second prompt-format configuration surface or dropping useful metadata.
