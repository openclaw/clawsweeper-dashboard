## Label Justifications

- `P2`: The issue describes recurring token/context overhead across non-webchat channel turns, but it is a bounded product improvement rather than an urgent outage.
- `impact:session-state`: The affected metadata is injected into active prompt/session context and can consume context budget over long channel conversations.

## Summary

Keep open: current main still injects inbound channel metadata into the trusted system prompt and current-user context with no supported opt-out, and the linked WeChat plugin discussion supports that this is core-owned behavior. The requested fix is a feature/config/product decision, not an already-implemented cleanup target.

## What This Changes

The issue requests a configurable opt-out for non-webchat inbound channel metadata injection to reduce model-visible context token use.

## Best Possible Solution

Maintainers should choose one core inbound metadata policy that preserves runtime routing data, keeps current defaults, and adds only a narrow documented opt-out if they accept the model-context tradeoff.

## Reproduction Assessment

Yes, at source level. Current main still builds the trusted inbound metadata block and includes direct non-webchat Conversation info; I did not run a live channel token-measurement scenario.

## Solution Assessment

Unclear. A config opt-out is plausible, but maintainers should first choose whether hidden runtime context, compact metadata, per-channel policy, or a strict opt-out is the intended product shape.
