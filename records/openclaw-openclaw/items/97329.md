## Label Justifications

- `P2`: The issue describes recurring token/context overhead across non-webchat channel turns, but it is a bounded product improvement rather than an urgent outage.
- `impact:session-state`: The affected metadata is injected into active prompt/session context and can consume context budget over long channel conversations.
- `maturity:stable`: The generic non-webchat metadata path applies to Discord channel turns, and the scorecard marks Discord as M4 Stable.

## Summary

Keep open: current main and v2026.6.10 still inject trusted inbound metadata plus direct non-webchat Conversation info without a supported config opt-out, and the linked WeChat plugin discussion confirms this is core-owned behavior. The remaining work is a feature/config policy decision rather than an already-implemented cleanup item.

## What This Changes

Request adding an openclaw.json option to disable or scope inbound channel metadata injected into agent prompts for non-webchat channels.

## Best Possible Solution

Choose one core inbound metadata policy that preserves runtime routing facts and current defaults, then either land hidden runtime-context handling or a narrow documented opt-out after maintainers accept the metadata-loss tradeoff.

## Reproduction Assessment

Yes, at source level. Current main still unconditionally builds the trusted inbound metadata block and includes direct non-webchat Conversation info; I did not run a live channel token-measurement scenario.

## Solution Assessment

Unclear. A config opt-out is plausible, but maintainers should first choose whether hidden runtime context, compact metadata, per-channel policy, or a strict opt-out is the intended product shape.
