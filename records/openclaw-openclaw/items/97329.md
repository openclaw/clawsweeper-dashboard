## Summary

Keep open: current main and v2026.6.10 still inject model-visible inbound channel metadata without a supported opt-out, and the linked WeChat plugin discussion confirms this is core behavior. The request is a product/configuration decision rather than a strict bug fix.

## What This Changes

Request adding an openclaw.json configuration surface to disable or narrow model-visible inbound channel metadata in the system prompt and user-message prefix.

## Best Possible Solution

Settle one core inbound metadata policy that preserves current defaults, keeps core routing/dedupe data available internally, and adds a narrow documented compact or opt-out mode only if maintainers choose that product shape.

## Reproduction Assessment

Yes, at source level. Current main unconditionally builds trusted inbound metadata and includes direct non-webchat conversation info; I did not run a live channel token-measurement scenario.

## Solution Assessment

Unclear. A config opt-out is plausible, but maintainers should first choose whether the best shape is compact metadata, hidden runtime context, per-channel policy, or a strict opt-out.
