## Summary

Keep open: the patch is not merge-ready because it can pass both text and card to the Feishu edit runtime, and it still skips the empty-presentation payload described by the linked report; the added proof is only typecheck output, not a real Feishu edit result.

## What This Changes

The PR changes the Feishu channel edit action to normalize `ctx.params.presentation` and build a Feishu interactive card before calling `editMessageFeishu`.

## Best Possible Solution

Choose one Feishu edit mode before the runtime call: preserve the interactive card envelope for presentation-backed edits, omit text when sending a card, explicitly handle empty presentation with fallback text, and add focused regression plus real Feishu proof.

## Reproduction Assessment

Yes for the PR defect: source inspection shows a non-empty `presentation` plus `message` builds a card while still passing text to a helper that rejects both. The exact Feishu API failure still needs live credentials for end-to-end proof.

## Solution Assessment

No. The Feishu edit action is the right layer, but this patch is not the best fix because it neither clears text when building a card nor handles the linked empty-presentation payload explicitly.
