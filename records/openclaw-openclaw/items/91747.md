## Summary

Keep open for maintainer review: the branch appears necessary and correct because current main still returns discovered raw CDP WebSocket URLs without policy validation, while the PR reuses the existing CDP SSRF helper across the sibling raw fallback paths and includes redacted runtime proof. The remaining decision is accepting the intended fail-closed compatibility behavior in strict SSRF mode.

## What This Changes

The PR validates discovered CDP `webSocketDebuggerUrl` values in raw tab listing, raw tab creation, and best-effort Playwright termination paths using the existing CDP SSRF policy, with regression tests.

## Best Possible Solution

Land the targeted validation after maintainer security and compatibility approval, keeping `assertCdpEndpointAllowed()` as the single CDP endpoint gate and adding operator guidance only if maintainers want the fail-closed upgrade impact called out more explicitly.

## Reproduction Assessment

Yes, this is source-reproducible: current main maps raw `/json/list` and `/json/new` `webSocketDebuggerUrl` values into returned tabs without validating the discovered WebSocket endpoint. The contributor also supplied redacted terminal proof that strict policy blocks the advertised link-local WebSockets after the patch.

## Solution Assessment

Yes, this is the best fix shape: validating immediately after CDP discovery protects all downstream tab consumers and matches the existing `/json/version` WebSocket validation path. A downstream-only check would leave competing entry points and make the boundary easier to miss again.
