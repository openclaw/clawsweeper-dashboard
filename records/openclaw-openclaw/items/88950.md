## Summary

Keep open for maintainer review: the patch appears to be a coherent hardening fix, but it changes bundled provider-policy hook failures from hard errors to logged fail-open behavior across config, auth, defaults, and thinking-profile paths.

## What This Changes

The branch wraps bundled provider-policy public-artifact hooks with logged fail-open handling and lets provider-runtime fall through to runtime hooks only after a current bundled-hook failure.

## Best Possible Solution

Land the centralized wrapper after provider/plugin owners confirm fail-open semantics for all four hooks and build/changed-gate validation is complete; split auth/default handling if those hooks should remain fail-closed.

## Reproduction Assessment

Yes, from source inspection. Current main returns raw bundled provider-policy surfaces and provider-runtime invokes those hooks directly before runtime fallback, so a throwing hook propagates.

## Solution Assessment

Yes, conditionally. Central wrapping at the public-artifact seam is the narrowest maintainable repair if fail-open is the intended policy; if auth/default hooks must remain fail-closed, the PR should split those before merge.
