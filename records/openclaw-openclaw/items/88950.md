## What This Changes

The branch wraps bundled provider-policy public-artifact hooks to log and ignore thrown failures, then lets provider-runtime fall through to runtime hooks after current bundled-hook failures with regression coverage.

## Best Possible Solution

Land the central wrapper only if provider owners accept logged fail-open semantics for all affected hooks and build/changed-gate validation is completed; otherwise split stricter auth/default handling.

## Reproduction Assessment

Yes, from source inspection. Current main returns raw bundled provider-policy hooks and provider-runtime invokes them directly before fallback, so a throwing hook propagates.

## Solution Assessment

Yes, conditionally. Central wrapping at the public-artifact seam is the narrowest maintainable repair if fail-open is the intended policy; if auth/default hooks must remain fail-closed, those hooks should be split before merge.
