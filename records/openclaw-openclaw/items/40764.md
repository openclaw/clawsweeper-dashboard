## Summary

Keep this PR open: it targets a real missing Android talk-mode language override, and current main has not implemented it, but the branch still needs contributor-supplied real behavior proof plus two P2 correctness fixes before merge. Earlier bot feedback was partly addressed by removing debug logs and reapplying the cached TTS language, but the latest branch still has an offline refresh regression and misses the current-main fallback TTS path.

## What This Changes

The PR adds an Android settings language picker persisted in preferences and threads the selected locale into gateway metadata, device info, STT, and TTS behavior.

## Best Possible Solution

Land a rebased Android feature that stores a user language override and applies it consistently to STT, local fallback TTS, gateway locale metadata, and device info without false offline errors.

## Reproduction Assessment

Not applicable. This is a feature PR, and current-main source inspection confirms Android still derives STT/TTS/gateway locale from system or directive paths rather than a user override.

## Solution Assessment

No, not as-is. The feature direction is plausible, but the branch must avoid false offline errors, adapt to current main's fallback TTS engine, and include contributor-supplied real device proof.

  - confidence: 0.86
