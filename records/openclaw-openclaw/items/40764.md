## Summary

Keep open. The Android language override is still absent from current main and the feature is useful, but this PR is not mergeable as-is because current main changed the Android talk/TTS paths, the branch still has a user-visible offline refresh regression, and external real behavior proof is missing.

## What This Changes

The PR adds an Android app talk-mode language override setting and threads the selected locale into preferences, settings UI, gateway/device locale metadata, speech recognizer intents, and local TTS handling.

## Best Possible Solution

Rebase the feature onto current main, validate the Android language setting once, apply it consistently to STT, local TTS fallback, and gateway/device locale metadata, and avoid offline failure states.

## Reproduction Assessment

Yes for source-level review: current main lacks the setting, and the PR diff plus current main show the offline refresh and active TTS-path problems. I did not run a real Android-device reproduction in this read-only pass.

## Solution Assessment

No, not as-is. The feature direction is reasonable, but the branch needs a current-main rebase and narrow fixes so language changes do not create a false offline failure and the override reaches the active local TTS path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Avoid failing offline language changes:** `apps/android/app/src/main/java/ai/openclaw/app/NodeRuntime.kt:640`
  - body: `setLanguageOverride()` calls `refreshGatewayConnection()` unconditionally. On current main that refresh path sets the visible status to `Failed: no cached gateway endpoint` when no endpoint is cached, so changing a local language setting before connection or after disconnect can surface a false failure state even though no reconnect is possible.
  - confidence: 0.93
- **[P2] Apply the override to current local TTS:** `apps/android/app/src/main/java/ai/openclaw/app/voice/TalkModeManager.kt:1198-1201`
  - body: This branch applies the override through the older `ensureSystemTts()` path, but current main's local fallback speaks through `speakWithSystemTts()` and `ensureTextToSpeech()`. After rebase, the selected language must be applied to that active path or fallback TTS can still ignore the setting.
  - confidence: 0.86
