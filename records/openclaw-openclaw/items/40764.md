## Summary

Keep this PR open, but it is not merge-ready. Current main still lacks an Android talk-mode language override, the contribution is directionally useful, and the author responded to earlier review feedback, but the latest patch still has an offline status regression and is stale against the current Android talk/TTS implementation.

## What This Changes

This PR adds an Android settings language dropdown, persists a `node.languageOverride` preference, and threads that locale into gateway connect metadata, device info, speech recognition, voice wake, mic capture, and system TTS paths.

## Best Possible Solution

Land a rebased Android-only implementation that stores an explicit language override, applies it to current STT/TTS/gateway locale paths without false offline failures, adds focused unit coverage, and includes a user-facing changelog entry.

## Reproduction Assessment

Yes. Current main reproduces the missing feature by inspection: the settings/runtime code has no language override and voice/gateway paths use the system locale. The PR's offline regression is also traceable from `setLanguageOverride()` to `refreshGatewayConnection()` when no endpoint is cached.

## Solution Assessment

No, not as-is. The product direction is reasonable, but the patch needs to be rebased onto current main, the reconnect refresh should avoid surfacing an offline failure, and the override must be applied to the active `speakWithSystemTts`/`ensureTextToSpeech` path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Avoid failing offline language changes:** `apps/android/app/src/main/java/ai/openclaw/app/NodeRuntime.kt:640`
  - body: Changing the setting before any gateway endpoint is cached now calls `refreshGatewayConnection()` unconditionally. On current main that method sets status to `Failed: no cached gateway endpoint` and returns when `connectedEndpoint` is null, so a fresh or intentionally disconnected app shows a false failure just from choosing a language. Guard the refresh or update the pending connect options without surfacing an error.
  - confidence: 0.93
- **[P2] Apply the override to the current TTS path:** `apps/android/app/src/main/java/ai/openclaw/app/voice/TalkModeManager.kt:1201`
  - body: The added `updateSystemTtsLanguage()` targets the older `ensureSystemTts` flow, but current main's fallback/system TTS path uses `speakWithSystemTts()` and `ensureTextToSpeech()` instead. When this branch is brought up to date, the override needs to be wired into that active path or fallback TTS will keep ignoring the selected language.
  - confidence: 0.78
