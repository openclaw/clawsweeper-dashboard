## What This Changes

This PR adds an Android language override setting persisted as `node.languageOverride` and threads it into gateway/device locale metadata plus Android speech recognition and system TTS paths.

## Best Possible Solution

Land a rebased Android-only implementation that persists a validated BCP 47 override, applies it consistently to STT, local TTS fallback, and gateway/device locale metadata without false offline failures, and adds focused tests plus a changelog entry.

## Solution Assessment

No, not as-is. The feature direction is reasonable, but the submitted branch must be rebased and repaired so language changes do not create an offline failure state and the override is applied to current main's active local TTS fallback path.
