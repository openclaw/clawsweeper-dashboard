## Summary

Keep this PR open. Current main still does not provide an Android Talk language override, and the contributor addressed earlier TTS/debug-log feedback, but the latest patch is not merge-ready because changing the language while offline can put the app into a false gateway failure state and the PR is currently merge-conflicted.

## What This Changes

The PR adds an Android Talk language override preference and Settings dropdown, then threads the selected locale through Android recognizer intents, system TTS, gateway connect metadata, device info, and related tests.

## Best Possible Solution

Land an Android-owned language override that keeps auto as the default, applies a valid BCP 47 locale consistently to Android STT, system TTS, gateway locale metadata, and device info, avoids false reconnect failures while offline, documents the Android Talk setting, and includes focused tests plus the required changelog entry.

## Reproduction Assessment

Yes. On the PR branch, change the language setting before any gateway endpoint is cached or after disconnecting; setLanguageOverride() calls refreshGatewayConnection(), and the current refresh path sets the status to "Failed: no cached gateway endpoint".

## Solution Assessment

No. The feature direction is reasonable, but the current patch should persist the preference and refresh only when an active or cached endpoint exists, or otherwise update pending connect options without surfacing a gateway failure for a local Settings change.
