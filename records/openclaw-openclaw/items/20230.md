## Summary

Keep open. Current main implements active assistant name/avatar resolution and generic user-vs-assistant bubble styling, but it still lacks the central per-agent chat identity/style contract: message groups do not carry assistant attribution or style metadata, assistant groups render with one active identity, and there is no validated per-agent bubble color setting or render hook. The existing maintainer discussion also explicitly kept the issue open for this remaining Control UI contract work.

## What This Changes

The issue requests distinct Control UI chat bubble styling by user/agent, optional per-agent bubble colors, and configured agent-name display in chat messages.

## Best Possible Solution

Define a versioned Control UI chat attribution/style contract that can render active and historical mixed-agent sessions accurately, validates any optional per-agent color/style input, updates docs/config only if a public key is added, and covers user, active assistant, mixed-agent history, and subagent cases with focused tests.
