## Summary

Keep open: current main has active-session assistant name/avatar support and generic user-vs-assistant styling, but it still lacks durable per-agent message/group attribution and any validated per-agent bubble color/style contract for mixed-agent chat history.

## What This Changes

The issue requests distinct WebUI chat bubble styling per agent and rendering the configured agent name instead of a generic label in chat messages.

## Best Possible Solution

Define an additive Control UI sender identity/style contract that preserves assistant attribution per message or group, validates optional color metadata, renders it safely, and updates focused tests plus docs/config if public keys are added.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken behavior under an existing contract. Source inspection on current main gives high confidence that per-agent assistant identity/style fields and render hooks are still absent.

## Solution Assessment

Unclear until maintainers choose the public sender identity/style contract. The maintainable path is additive sender metadata plus validated style settings, not a one-off active-assistant UI color/name patch.
