## Summary

Keep open: current main still lacks a per-agent chat identity/style contract and configurable per-agent bubble colors. The active assistant name/avatar path and generic user-vs-assistant styling are present, but they do not solve mixed-agent history attribution or styling, and the related multi-role chat issue remains open rather than superseding this request.

## What This Changes

The issue requests distinct WebUI chat bubble styling per agent and rendering agent names instead of generic labels in chat bubbles.

## Best Possible Solution

Define an additive Control UI sender identity/style contract that preserves assistant attribution per message or group, validates optional color metadata, renders names/colors safely, and updates focused tests and docs.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken behavior under an existing contract. Source inspection on current main shows the requested per-agent assistant identity/style fields and render hooks are still absent.

## Solution Assessment

Unclear until maintainers choose the public sender identity/style contract. The maintainable path is additive sender metadata plus validated style settings, not a UI-only active-assistant label or color patch.
