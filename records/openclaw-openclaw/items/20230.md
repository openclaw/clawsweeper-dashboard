## Summary

Keep open. Current main has active assistant name/avatar loading and distinct user-vs-assistant bubble styling, but it still lacks the requested per-message/per-group assistant attribution and any validated per-agent chat bubble color contract.

## What This Changes

The issue requests WebUI chat support for distinct user/agent bubbles, optional per-agent bubble colors, and rendering each agent's configured name instead of a generic label.

## Best Possible Solution

Define a Control UI message/group-level assistant attribution and safe optional style contract, then implement rendering, config/docs, and focused mixed-agent history tests if maintainers accept the feature.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection confirms current main has only the partial active-assistant identity path and generic user-vs-assistant styling, not the requested per-agent chat style contract.

## Solution Assessment

Unclear until maintainers choose the product/API shape. The requested direction is reasonable, but the narrow maintainable path should first define validated assistant attribution and style metadata instead of adding ad hoc CSS or unvalidated color fields.
