## Summary

Keep open. Current main covers the active assistant name/avatar path and gives user bubbles a distinct style, but it still lacks the central requested Control UI contract for per-agent assistant attribution and configurable assistant bubble styling in chat history; the existing maintainer discussion also explicitly kept this open for that remaining work.

## What This Changes

Request per-agent WebUI chat bubble styling and agent-name display so user and multiple assistant agents are visually distinguishable in chat.

## Best Possible Solution

Define an additive Control UI chat identity/style contract, carry assistant attribution at message or group level, validate any optional per-agent color/style input, render assistant-specific names and bubble styling through safe CSS hooks, and update docs/config/tests for active-session, mixed-agent, and historical chat cases.

## Reproduction Assessment

Not applicable as a feature request; there is no failing runtime reproduction required. Static verification against current main is high-confidence: the relevant group type, grouping logic, renderer, CSS, config schema, and docs still lack per-agent assistant bubble style and historical assistant attribution support.

## Solution Assessment

No, the current partial implementation is not the best complete solution because it applies one active assistant identity to every assistant group. The maintainable path is an explicit additive identity/style contract rather than ad hoc CSS injection or prompt-based sender prefixes.
