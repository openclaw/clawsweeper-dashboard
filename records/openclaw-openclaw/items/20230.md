## Summary

Keep open: current main covers active-session assistant identity and generic user-vs-assistant styling, but it still lacks per-agent chat style metadata, historical assistant attribution, and a validated color/style contract; the existing maintainer review also explicitly kept this issue open for that remaining Control UI design work.

## What This Changes

Request per-agent Control UI chat bubble colors/styles and display of each agent's configured name in WebUI chat history.

## Best Possible Solution

Define an additive Control UI chat identity/style contract, then implement per-group assistant attribution, validated optional color/style settings, CSS rendering, docs/config updates, and mixed-agent tests.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken behavior under an existing contract. Source inspection does give a high-confidence current-main check that the requested per-agent style/name attribution is not implemented in the group, render, CSS, or config contracts.

## Solution Assessment

Unclear until maintainers define the public contract. The maintainable direction is additive assistant identity/style metadata plus validated styling hooks, not only changing the active-session label used by today’s renderer.
