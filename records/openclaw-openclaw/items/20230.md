## Summary

Keep open: current main still renders chat from one active assistant identity, preserves sender labels only for user groups, and has no validated per-agent bubble color/style contract. The related multi-role tracker at https://github.com/openclaw/openclaw/issues/22774 overlaps, but it is broader and open; prior maintainer/ClawSweeper comments also kept this issue open as remaining Control UI identity/style work.

## What This Changes

The issue requests Control UI support for distinct per-agent chat bubble colors and rendering each agent's configured name in WebUI chat bubbles.

## Best Possible Solution

Define an additive Control UI sender identity/style contract with per-message or per-group assistant attribution, validated optional color/style metadata, CSS rendering hooks, docs/config updates if public settings are added, and focused tests for active, mixed-agent, and historical chat rendering.

## Reproduction Assessment

Not applicable; this is an enhancement request rather than broken behavior under an existing contract. Source inspection gives a high-confidence current-main capability gap in chat group metadata, rendering, config, docs, and CSS.

## Solution Assessment

Unclear until maintainers choose the public Control UI identity/style contract. A UI-only active-assistant label or color patch would be too shallow; the maintainable path is additive sender attribution plus validated style metadata.
