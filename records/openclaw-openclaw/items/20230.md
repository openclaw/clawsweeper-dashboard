## Summary

Keep open: current main still lacks the requested per-agent chat identity/style contract, and recent issue discussion already kept this open because active assistant identity and generic user styling do not solve mixed-agent history attribution or per-agent bubble colors.

## What This Changes

The issue requests distinct WebUI chat bubble styling per agent and display of each agent's configured name instead of a generic label.

## Best Possible Solution

Define an additive Control UI sender identity/style contract that preserves assistant attribution per message or group, validates optional visual metadata, renders names/colors safely, and updates focused tests and docs.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken behavior under an existing contract. Source inspection on current main shows the requested per-agent assistant identity/style fields and render hooks are absent.

## Solution Assessment

Unclear until maintainers choose the public sender identity/style contract. The maintainable path is additive sender metadata plus validated color settings, not a UI-only active-assistant label or color patch.
