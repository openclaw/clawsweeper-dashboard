## Summary

Keep open. Current main covers active assistant name/avatar resolution and generic user-vs-assistant styling, but it still lacks the central requested Control UI contract for per-agent assistant attribution and configurable assistant bubble styling in chat history; the existing maintainer discussion also kept this open for that remaining work.

## What This Changes

Request distinct WebUI chat bubble styling for user and agents, configurable per-agent bubble colors, and display of each agent's configured name in chat bubbles.

## Best Possible Solution

Define an additive Control UI chat identity/style contract, carry assistant attribution at message or group level, validate optional per-agent color/style input, render assistant-specific bubbles through safe CSS hooks, and update docs/config/tests.

## Reproduction Assessment

Not applicable as a feature request; there is no failing runtime reproduction required. Static verification against current main is high-confidence: the group type, grouping logic, renderer, CSS, config schema, and docs still lack per-agent assistant bubble style and historical assistant attribution support.

## Solution Assessment

No, the current partial implementation is not the best complete solution because it applies one active assistant identity to every assistant group. The maintainable path is an explicit additive identity/style contract rather than CSS injection or prompt-based sender prefixes.
