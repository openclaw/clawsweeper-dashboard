## Summary

Keep open: current main has active assistant name/avatar support and generic user-vs-assistant styling, but it still lacks per-agent assistant attribution, per-agent bubble style metadata, and a validated public color/style contract.

## What This Changes

Request per-agent chat bubble colors and assistant agent-name display in the Control UI chat view.

## Best Possible Solution

Define an additive Control UI chat identity/style contract, then implement per-group assistant attribution, validated optional color/style settings, CSS rendering, docs/config updates, and mixed-agent tests.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken behavior under an existing contract. Source inspection gives a high-confidence current-main check that per-agent style/name attribution is not implemented in the group, render, CSS, or config contracts.

## Solution Assessment

Unclear until maintainers define the public Control UI identity/style contract. The maintainable direction is additive assistant attribution plus validated styling hooks, not only changing the active-session label used by the current renderer.
