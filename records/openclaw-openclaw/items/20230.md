## Summary

Keep open: current main still renders Control UI chat with one active assistant identity and no per-agent assistant attribution or validated bubble color/style contract; prior maintainer and ClawSweeper comments kept this open for the same remaining API work.

## What This Changes

Request per-agent WebUI chat styling, including distinct user/agent bubble colors, configurable per-agent bubble colors, and each agent's configured name in chat bubbles.

## Best Possible Solution

Define an additive Control UI chat sender identity/style contract, persist or project per-message/per-group assistant attribution, then render validated per-agent names/colors with focused regression coverage and docs.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken behavior under an existing contract. Source inspection of current main gives a high-confidence check that per-agent style/name attribution is absent from the group, render, bootstrap/config, and CSS surfaces.

## Solution Assessment

Unclear until maintainers choose the public Control UI identity/style contract. The maintainable direction is additive sender metadata plus validated color settings, not only changing the active assistant label used by the current renderer.
