## Summary

Keep open: current main has active assistant name/avatar rendering and user-vs-assistant bubble styling, but it still lacks durable per-agent assistant attribution and a validated per-agent bubble color/style contract for mixed-agent chat history.

## What This Changes

Request per-agent WebUI chat bubble colors and display of each agent's configured name instead of a single generic assistant/user label.

## Best Possible Solution

Define an additive Control UI sender identity/style contract that preserves assistant attribution per message or group, validates optional color metadata, renders it safely, and updates focused tests plus docs/config if public keys are added.

## Solution Assessment

Unclear until maintainers choose the public sender identity/style contract. The maintainable path is additive sender metadata plus validated style settings, not a one-off active-assistant UI color/name patch.
