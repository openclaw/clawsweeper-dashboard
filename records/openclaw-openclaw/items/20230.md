## Summary

Keep open: current main still lacks per-agent assistant attribution and validated bubble-color/style metadata in the Control UI chat pipeline. Prior issue comments already kept this open, and related open issues cover broader sender-identity work but do not make this UI styling/name-display request obsolete.

## What This Changes

Request per-agent WebUI chat bubble colors and agent-name display so multi-agent conversations are visually distinguishable.

## Best Possible Solution

Define an additive Control UI sender identity/style contract that preserves per-message or per-group assistant attribution, validates optional color metadata, and renders agent names/colors with focused tests and docs.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken behavior under an existing contract. Source inspection on current main verifies the requested per-agent style/name attribution is absent from the group, render, config/docs, and CSS surfaces.

## Solution Assessment

Unclear until maintainers choose the public Control UI identity/style contract. A UI-only active-assistant label or color patch would not reliably handle mixed-agent historical sessions; additive sender metadata plus validated style settings is the safer direction.
