## Summary

Keep open. Current main still matches the reported custom-session cron transcript gap: direct cron delivery sends to the requested channel without outbound mirror metadata, while transcript persistence only runs when mirror metadata is present. The existing discussion also points to open PR #74913 as a focused fix candidate, so this should remain open until that PR or an equivalent patch lands.

## What This Changes

Custom-session cron jobs should persist direct-delivered assistant responses into the destination session transcript so later turns in that session can see them.

## Best Possible Solution

Land or replace a focused cron direct-delivery fix that passes the existing outbound mirror contract for successful `session:<id>` deliveries, using the resolved destination session key and direct-delivery idempotency key, with regression coverage for the custom-session path.

## Reproduction Assessment

Yes. The reporter's steps map directly to current main: a `session:<id>` cron run reaches direct delivery, that call omits `mirror`, and the transcript append path only runs when `params.mirror` is present.

## Solution Assessment

Yes. Reusing the existing outbound mirror contract is the narrow maintainable solution; PR #74913 should be reviewed and landed, revised, or replaced with an equivalent focused patch.
