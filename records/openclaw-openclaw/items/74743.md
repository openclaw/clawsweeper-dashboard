## Summary

Keep open. Current main still matches the reported custom-session cron transcript gap: direct cron delivery sends to the requested channel but does not pass outbound mirror metadata, while transcript persistence only runs when mirror metadata is present. The issue is also paired with open PR #74913 using closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks for cron jobs created with `--session session:<id>` to persist delivered cron responses into the destination session transcript so later turns can see what the agent just sent.

## Best Possible Solution

Direct cron delivery should mirror successful delivered assistant text into the resolved destination session transcript through the existing outbound mirror contract, with an idempotency key and regression coverage for `session:<id>` jobs.

## Reproduction Assessment

Yes. The reporter's steps map directly to current main: a custom-session cron job reaches direct delivery, that call omits `mirror`, and transcript append only runs when `params.mirror` is present.

## Solution Assessment

Yes. Reusing the existing outbound mirror contract is the narrow maintainable fix; PR #74913 appears aimed at that path and should be reviewed, landed, or replaced with an equivalent focused patch.
