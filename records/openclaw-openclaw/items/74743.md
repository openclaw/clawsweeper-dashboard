## Summary

Keep open. Current main still omits a transcript mirror/append for direct custom-session cron delivery, and https://github.com/openclaw/openclaw/pull/74913 is the open implementation candidate linked with closing syntax.

## What This Changes

Persist direct cron responses delivered with `--session session:<id>` into the destination session transcript so later turns remember them.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/74913, or an equivalent focused patch, so direct cron delivery persists the same delivered payload into the target session transcript after successful delivery.

## Reproduction Assessment

Yes. Source inspection on current main shows `session:<id>` cron jobs enter a persistent session context, direct delivery sends payloads without a transcript mirror, and transcript append behavior is gated on a mirror parameter.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing transcript mirror/append contract after successful direct delivery rather than adding a new plugin API or mutating transcripts ad hoc.
