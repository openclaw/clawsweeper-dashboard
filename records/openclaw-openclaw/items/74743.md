## Summary

Keep open. Current main still has the cron delivery/transcript gap: direct cron delivery sends the outbound payload without the mirror metadata that triggers transcript persistence, and the issue is already paired with open PR #74913 using closing syntax.

## What This Changes

Cron jobs targeting `session:<id>` should persist delivered assistant replies into the destination session transcript so the channel agent remembers them.

## Best Possible Solution

Review and land PR #74913, revise it, or replace it with an equivalent focused patch that passes destination-session mirror metadata through the existing outbound transcript mirror contract with idempotency and regression coverage.

## Reproduction Assessment

Yes. The reporter's steps map directly to current main: `session:<id>` resolves to a persistent session key, direct cron delivery sends the payload without `mirror`, and the only transcript append path in `deliverOutboundPayloads` requires `params.mirror`.

## Solution Assessment

Yes. Reusing the existing outbound mirror contract is the narrowest maintainable fix because it preserves shared delivery-success gating and idempotent transcript writes instead of adding a cron-only transcript append path.
