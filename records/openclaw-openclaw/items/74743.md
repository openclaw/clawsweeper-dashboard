## Summary

Close: current main and the shipped v2026.6.8 release now mirror successful custom-session cron direct deliveries into the destination session transcript, with regression coverage for the reported path.

## What This Changes

The issue asks for cron output delivered with `--session session:<sessionId>` to persist in the destination session so later turns can see what the agent sent.

## Best Possible Solution

Keep the shipped destination-session transcript mirror for custom-session cron jobs, and leave broader proactive-message context design to the separate related issues and PRs.

## Reproduction Assessment

Yes, source-level: current main and v2026.6.8 include a custom-session cron mirror test that would fail if the delivered assistant text were not appended to the custom session. I did not run the live Discord reproduction in this read-only review, but the merged fix PR includes real destination-session transcript proof for the same direct announce mechanism.

## Solution Assessment

Yes: resolving the delivery session before sending and appending a best-effort transcript mirror only after successful delivery is the narrow maintainable fix for this issue. Broader ephemeral awareness for proactive messages is separate follow-up work.
