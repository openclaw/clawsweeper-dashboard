## Summary

Keep open. Current main still matches the reported cron transcript gap: custom cron sessions are documented and resolved as persistent session keys, but direct cron delivery calls the outbound delivery helper without the `mirror` payload that gates transcript persistence. The issue is also already paired with open PR #74913 using closing syntax, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

Cron jobs targeting `--session session:<id>` should persist their delivered assistant output into that destination session transcript so later channel turns can see the message.

## Best Possible Solution

Review and land PR #74913, revise it, or replace it with an equivalent focused patch that passes direct cron delivery mirror metadata into the existing outbound transcript mirror contract with idempotency and regression coverage for `session:<id>` jobs.

## Reproduction Assessment

Yes. The reporter's steps map directly to current main: `session:<id>` resolves to a stable session key, direct cron delivery sends the payload without `mirror`, and the only transcript append path in `deliverOutboundPayloads` requires `params.mirror`.

## Solution Assessment

Yes. Reusing the existing outbound mirror contract is the narrowest maintainable fix because it preserves shared delivery-success gating and idempotent transcript writes instead of adding a separate cron-only append path.
