## Summary

Keep open. Current main still has the reported transcript persistence gap for direct cron delivery, and open PR #74913 is already the canonical implementation candidate linked with closing syntax.

## What This Changes

Cron jobs created with `--session session:<id>` should persist their direct-delivered assistant replies into that session transcript so later turns can remember them.

## Best Possible Solution

Review and land PR #74913, or an equivalent focused patch, so direct cron deliveries pass destination-session mirror metadata through `deliverOutboundPayloads` with regression coverage.

## Reproduction Assessment

Yes. The reporter's CLI path maps to current main: `session:<id>` selects a persistent cron run session, direct delivery succeeds without `mirror`, and transcript append only occurs when `params.mirror` is present.

## Solution Assessment

Yes. Reusing the existing outbound mirror contract is the narrowest maintainable fix because it preserves delivery-success gating, idempotency, and transcript update behavior without adding a new core API.
