## Summary

Keep open. Current `main` still has the reported transcript persistence gap for direct cron delivery, and PR #74913 is an open implementation candidate that uses closing syntax for this issue.

## What This Changes

The issue requests that cron jobs targeting `--session session:<id>` persist the delivered assistant response into the destination session transcript.

## Best Possible Solution

Land PR #74913, revise it, or replace it with an equivalent focused patch that passes destination-session mirror metadata through the existing outbound transcript mirror contract with idempotency and regression coverage.

## Reproduction Assessment

Yes. The reporter's steps map directly to current `main`: `session:<id>` resolves to a persistent session key, direct cron delivery omits `mirror`, and transcript append only happens when `params.mirror` is present.

## Solution Assessment

Yes. The linked PR direction is the narrowest maintainable fix because it reuses the shared outbound mirror path, preserving delivery-success gating, idempotency, and transcript update events.
