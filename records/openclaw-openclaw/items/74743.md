## Summary

Keep open. Current main still omits the outbound mirror when direct cron delivery sends a `session:<id>` job result, so the reported destination-session transcript gap remains; #74913 is the open closing PR for the same fix.

## What This Changes

The issue asks for cron jobs targeting `--session session:<id>` to persist delivered assistant output into that destination session transcript.

## Best Possible Solution

Land #74913, or an equivalent focused patch, so direct cron delivery passes destination-session mirror metadata and an idempotency key through the existing outbound transcript mirror contract with regression coverage.

## Reproduction Assessment

Yes. The reporter's steps map to the current source path: `session:<id>` selects a persistent cron run session, direct delivery omits `mirror`, and transcript append only runs when `params.mirror` is present.

## Solution Assessment

Yes. The linked PR direction is the narrowest maintainable solution because it reuses the existing idempotent outbound mirror path instead of adding raw transcript writes or a second persistence mechanism.
