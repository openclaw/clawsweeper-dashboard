## Summary

Keep open. Current main still omits transcript mirror metadata from direct cron delivery, while transcript append is gated on that mirror parameter; PR #74913 is an open, unmerged implementation candidate with closing syntax, so the issue should remain open until that PR lands or is replaced.

## What This Changes

Ensure cron jobs targeting `--session session:<id>` persist direct delivery output into that destination session transcript so follow-up turns remember the cron message.

## Best Possible Solution

Review and land PR #74913, or replace it with an equivalent focused patch that passes destination-session mirror metadata through the existing outbound transcript mirror contract with idempotency and regression coverage.

## Reproduction Assessment

Yes. The reporter's CLI flow maps directly to current main: `session:<id>` becomes the cron run session key, direct delivery omits `mirror`, and transcript append only happens when `params.mirror` is present.

## Solution Assessment

Yes. Mirroring through the existing outbound `DeliveryMirror` path is the narrowest maintainable fix because it reuses idempotent transcript append behavior instead of adding a separate raw transcript write path.
