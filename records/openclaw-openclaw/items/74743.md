## Summary

Keep open. Current main still has the custom-session cron transcript gap: direct cron delivery sends the outbound payload without mirror metadata, and #74913 is already the canonical open fix path using closing syntax.

## What This Changes

The issue reports that cron jobs targeting `session:<id>` deliver their response to the channel but do not persist that assistant response in the destination session transcript.

## Best Possible Solution

Review and land #74913, revise it, or replace it with an equivalent focused patch that passes destination-session mirror metadata through the existing outbound transcript mirror contract with idempotency and regression coverage.

## Reproduction Assessment

Yes. The reporter's CLI path maps to current main: `session:<id>` is used as the cron run session, direct delivery omits `mirror`, and transcript append only runs when `params.mirror` is present.

## Solution Assessment

Yes. Reusing the existing outbound mirror contract is the narrowest maintainable fix because it preserves shared delivery-success gating and idempotent transcript writes instead of adding a cron-only transcript append path.
