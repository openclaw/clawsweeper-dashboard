## Summary

Keep open. Current main still matches the reported custom-session cron transcript gap: direct cron delivery sends to the destination channel without mirror metadata, while transcript persistence is only performed when `deliverOutboundPayloads` receives a `mirror` option. Open PR #74913 already uses closing syntax for this issue, with an external fork PR targeting the same root cause, so the issue should stay open until that fix path or an equivalent patch merges.

## What This Changes

Ensure cron jobs targeting `--session session:<id>` persist delivered assistant responses into the destination session transcript so later turns can see them.

## Best Possible Solution

Review and land PR #74913, revise it, or replace it with an equivalent focused patch that passes direct cron delivery mirror metadata into the existing outbound transcript mirror contract with idempotency and regression coverage for `session:<id>` jobs.

## Reproduction Assessment

Yes. The reporter's steps map directly to current main: `session:<id>` resolves to a stable session key, direct cron delivery sends the payload without `mirror`, and the only transcript append path in `deliverOutboundPayloads` requires `params.mirror`.

## Solution Assessment

Yes. Reusing the existing outbound mirror contract is the narrowest maintainable solution because it preserves the shared delivery-success gating and transcript idempotency behavior already used by other outbound sends.
