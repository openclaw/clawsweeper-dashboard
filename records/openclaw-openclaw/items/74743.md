## Summary

Keep open. Current main still has the reported cron direct-delivery transcript persistence gap, and PR #74913 remains the open implementation candidate that directly targets this issue.

## What This Changes

Custom session cron jobs using direct channel delivery should persist the delivered assistant response into the destination session transcript.

## Best Possible Solution

Direct cron announce delivery should write one idempotent mirror entry to the intended persistent destination session transcript while preserving the existing isolated-job behavior that avoids mirroring into the main session.

## Reproduction Assessment

Yes. Source inspection on current main shows `session:<id>` cron runs enter a persistent session, direct announce delivery omits `mirror`, and transcript appending only happens through the mirror path.

## Solution Assessment

Yes. Reusing the existing `deliverOutboundPayloads` mirror contract is the narrowest maintainable fix; raw transcript mutation or a new plugin runtime API is not needed for this core cron path.
