## Summary

Keep open. Current main still matches the reported cron transcript gap: custom cron session targets resolve to persistent session keys, but the direct cron delivery path calls `deliverOutboundPayloads` without the `mirror` payload that triggers transcript persistence. The issue is also paired with open fix PR #74913, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

Cron jobs created with `--session session:<id>` should persist direct-delivered assistant replies into the destination session transcript so later turns can remember them.

## Best Possible Solution

Review and land PR #74913, revise it, or replace it with an equivalent focused patch that passes direct cron delivery mirror metadata into the existing outbound transcript mirror contract with idempotency and regression coverage for `session:<id>` jobs.

## Reproduction Assessment

Yes. The reporter's steps map directly to current main: `session:<id>` resolves to a stable session key, direct cron delivery sends the payload without `mirror`, and the only transcript append path in `deliverOutboundPayloads` requires `params.mirror`.

## Solution Assessment

Yes. Reusing the existing outbound mirror contract is the narrowest maintainable fix because it preserves shared delivery-success gating and idempotent transcript writes instead of adding a cron-only transcript append path.
