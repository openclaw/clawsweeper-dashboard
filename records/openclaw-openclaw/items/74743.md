## Label Justifications

- `P2`: The original bug dropped visible cron replies from the durable destination session transcript, a normal-priority behavior bug with limited blast radius.
- `impact:data-loss`: The reported assistant response reached chat but was not persisted to the session transcript, making the delivered content effectively lost for later context.

## Summary

Close as implemented. Current main and `v2026.5.27` now mirror successful direct cron delivery text into the resolved or custom destination session transcript, with focused regression coverage for `session:<id>` cron targets.

## What This Changes

The issue asks for cron jobs targeting `--session session:<id>` to persist direct-delivery assistant output into that destination session transcript.

## Best Possible Solution

Keep the shipped direct cron transcript mirror behavior; the separate plugin runtime API request remains tracked at https://github.com/openclaw/openclaw/issues/73274.

## Reproduction Assessment

No for current main: source and regression tests show successful direct cron delivery now appends the delivered text to the custom or resolved destination session transcript. The original 2026.4.27 repro is clear, but the current-main path is fixed.

## Solution Assessment

Yes. The current implementation mirrors only after successful direct delivery, uses an idempotency key, keeps custom `session:<id>` targets on their custom session, and preserves delivery success if the best-effort transcript append fails.
