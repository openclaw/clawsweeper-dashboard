## Summary

Keep open. Current main still has the reported transcript persistence gap for direct cron announce delivery, and open PR #74913 is already the canonical implementation candidate linked with closing syntax.

## What This Changes

Cron jobs created with `--session session:<id>` should persist their direct-delivered assistant reply in that destination session transcript.

## Best Possible Solution

Review and land PR #74913, or an equivalent focused patch, so direct cron announce delivery uses the existing outbound mirror contract with idempotent regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows `session:<id>` cron runs enter a persistent session, direct delivery omits `mirror`, and transcript appending is only performed through the mirror path.

## Solution Assessment

Yes. Reusing the existing `deliverOutboundPayloads` mirror contract is the narrowest maintainable fix; raw transcript mutation or a new plugin API is not needed for this core cron bug.
