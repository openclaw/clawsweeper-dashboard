## Summary

Keep open. Current main still has the reported source-level gap for custom-session cron delivery, and https://github.com/openclaw/openclaw/pull/74913 is still the direct open implementation candidate linked with closing syntax.

## What This Changes

Ensure cron jobs created with `--session session:<id>` persist the delivered assistant response in that session's transcript.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/74913, or an equivalent focused patch, while coordinating with the broader destination-channel work in https://github.com/openclaw/openclaw/pull/80786.

## Reproduction Assessment

Yes. Source inspection shows custom cron sessions use a persistent session key, direct delivery sends without a transcript mirror, and the outbound transcript append path is only invoked when mirror parameters are supplied.

## Solution Assessment

Yes. Reusing the existing transcript append/mirror contract after successful direct delivery is the narrow maintainable fix; the broader channel-session mirror work should be coordinated but does not make this issue closeable yet.
