- **[P2] Store the normalized status provider id:** `src/plugins/registry.ts:460-462`
  - body: `registerStatusProvider` trims `provider.id` for validation and duplicate lookup, but then stores the original provider object. If the first plugin registers `" contextclaw "`, the registry stores the padded ID and a later `"contextclaw"` registration bypasses the duplicate check, so downstream consumers see a noncanonical ID. Store `{ ...provider, id }` and add a padded-then-canonical regression case.
