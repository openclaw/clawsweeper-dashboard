## Summary

Keep this PR open. The API-only seam is useful and the prior duplicate-id feedback has been addressed, but the latest head still has three merge blockers: rollback omits the new mutable registry array, the public SDK API baseline hash is not updated for the export change, and the changelog would close the broader footer-rendering issue before rendering exists.

The PR adds a public `PluginStatusProvider`/`registerStatusProvider` plugin API, stores status providers in the plugin registry, exports the type through the Plugin SDK, updates docs/changelog/test fixtures, and adds focused registry unit tests.

## Best Possible Solution

Land this as an API-only precursor after adding lifecycle rollback coverage for `statusProviders`, regenerating the Plugin SDK API hash, and changing the changelog to a non-closing reference unless the TUI consumer is included. Keep #64086 and #53774 open for the rendering/template UX decisions.
