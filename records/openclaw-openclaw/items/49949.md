- `P2`: This is a clear provider feature with known affected surfaces, but existing managed web-search providers still work and it is not an urgent regression.
- `impact:security`: Anthropic native search would add provider-hosted web access, domain policy, encrypted content, citations, and cost-bearing server-tool behavior.
- `impact:auth-provider`: The feature depends on Anthropic provider routing and reuse of Anthropic credentials rather than a separate search API key.
- `impact:session-state`: Server result, citation, encrypted-content, and prompt-cache blocks need transcript preservation so multi-turn Anthropic sessions do not drift.

## Summary

Keep open: current main and the latest release still lack Anthropic native server-side `web_search`, and https://github.com/openclaw/openclaw/pull/50692 explicitly closes this issue but is still open and unresolved.

## What This Changes

Add Anthropic native `web_search_20260209` / `web_search_20250305` as a selectable OpenClaw web search provider using Anthropic server-side tools.

## Best Possible Solution

Resolve this through https://github.com/openclaw/openclaw/pull/50692 or a focused successor in the Anthropic plugin with manifest-declared `webSearchProviders`, explicit selection gating, config/auth propagation, server result preservation, docs, tests, and redacted live Anthropic proof.

Status: needs_attention

Summary: Needs attention because enabling Anthropic-hosted search changes outbound web access, domain policy, cost, credentials, and server-result transcript handling.

  - body: Native Anthropic web search should inject server tools only when the user selected and enabled that provider, with domain, max-use, location, and cost controls propagated before any hosted search can run.
