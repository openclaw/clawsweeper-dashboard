## Summary

Keep open. The PR is a useful implementation of the requested Jina task-parameter support, but it is not merge-ready because Gateway embeddings still drop the new task fields and the new docs examples use non-v5 model IDs.

## What This Changes

The PR adds typed `queryTask` and `documentTask` memory-search config, forwards them through the OpenAI embedding provider and batch cache identity, and updates docs, changelog, and targeted tests.

## Best Possible Solution

Land this PR or a narrow replacement after Gateway provider creation passes task settings end-to-end, the docs use actual Jina v5 text model IDs, and the branch is rebased.

## Reproduction Assessment

Yes. Source inspection on current main shows no `queryTask`/`documentTask` config or OpenAI `task` request-body path, and PR head still has a source-visible Gateway pass-through gap.

## Solution Assessment

No, not yet. The typed query/document task config is the right maintainable direction, but this PR still misses the Gateway `/v1/embeddings` creation path and documents v5 behavior with v3/v4 model IDs.

Update this section and the memory-config example to v5 text IDs such as `jina-embeddings-v5-text-small` or `jina-embeddings-v5-text-nano`.
  - confidence: 0.9
