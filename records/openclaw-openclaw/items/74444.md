The PR adds optional `queryTask` and `documentTask` memory-search config fields and forwards them as OpenAI-compatible embedding `task` values through resolved memory settings, SDK/provider options, inline and batch OpenAI embedding requests, cache identity, docs, and changelog.

## Best Possible Solution

Implement the task configuration on current main as a reviewed OpenAI-compatible memory embedding feature: add typed `queryTask`/`documentTask` config metadata, resolve and forward the values through memory-core and SDK/provider options, include them in inline and batch request bodies and cache identity, correct the Jina v5 docs, update the changelog, and add focused regression coverage.

Yes. A high-confidence static reproduction exists on current main: `memorySearch` is strict without `queryTask`/`documentTask`, and the OpenAI inline and batch embedding request bodies omit `task`, so the requested configuration cannot affect embedding requests today.

## Solution Assessment

No, not as submitted. The direction is narrow and appropriate, but the PR needs a current-main rebase or replacement plus config metadata/generated baseline updates, corrected Jina v5 docs, and focused tests before it is the maintainable fix.
