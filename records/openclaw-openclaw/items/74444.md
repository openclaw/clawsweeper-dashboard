## Summary

Keep open. The requested feature is valid and still absent on current main, but this PR should not merge as submitted because it breaks the current memory-host SDK bridge layout, omits config metadata/generated baseline updates, and documents v5 with older model names.

## What This Changes

The PR adds optional `queryTask` and `documentTask` memory-search config fields and forwards them as OpenAI-compatible embedding `task` request values through config resolution, provider options, inline and batch OpenAI requests, cache identity, docs, and changelog.

## Best Possible Solution

Land a repaired version on current main that preserves the package-owned SDK bridge, adds complete config metadata/generated baseline updates, corrects the Jina v5 docs, adds focused regression tests, and keeps the changelog credit.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: strict `memorySearch` config rejects the proposed task keys, and the OpenAI inline plus batch embedding request bodies never include `task`.

## Solution Assessment

No, not as submitted. The typed query/document task direction is the narrow maintainable solution, but the patch needs bridge-layout, metadata, docs, and test repairs before merge.
