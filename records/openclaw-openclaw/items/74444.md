## Best Possible Solution

Land this PR or a narrow replacement after rebasing and repairing dimensions preservation, Gateway task propagation, and v5 model examples.

## Reproduction Assessment

Yes. Source inspection on current main shows no config, SDK option, provider request-body, cache-identity, or docs path for Jina task settings; PR proof shows the direct provider can emit the intended request body after the patch.

## Solution Assessment

No, not yet. Typed query/document task fields are the right maintainable direction, but this head needs the current dimensions behavior preserved, Gateway propagation added, and docs examples corrected to actual v5 model IDs.

  - confidence: 0.9

  - body: The new section is labeled for Jina v5 but names `jina-embeddings-v3`/`jina-embeddings-v4` and samples `jina-embeddings-v3`. Update the OpenAI provider and memory-config examples to v5 text model IDs such as `jina-embeddings-v5-text-small` or `jina-embeddings-v5-text-nano` so readers configure the documented v5 adapter setup.
  - confidence: 0.9
