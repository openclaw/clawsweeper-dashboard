## What This Changes

The PR changes `src/cli/node-cli/daemon.ts` so node-status missing-unit and stopped follow-up hint lines go to stdout while the terse status diagnostics stay on stderr.

## Best Possible Solution

Keep the narrow stream-routing fix, require redacted terminal proof of the actual CLI stream behavior before merge, and let maintainers explicitly accept the small stdout/stderr compatibility tradeoff.

## Reproduction Assessment

## Solution Assessment

Yes, if maintainers accept stdout as the node-status hint stream: the PR preserves the terse stderr diagnostics and changes only the follow-up hint lines. The solution would be stronger with real command proof, and a focused regression test would be useful but is not a blocking code finding by itself.
