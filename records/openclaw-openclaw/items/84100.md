The PR changes `runNodeDaemonStatus` so missing-unit and stopped node runtime hint lines are written to stdout while preserving the terse status diagnostics on stderr.

## Best Possible Solution

Land the narrow stream-routing fix after maintainers accept the stdout/stderr compatibility tradeoff, ideally with a focused regression test for missing-unit and stopped status output.
