## What This Changes

This PR switches the Android node foreground service from `dataSync` to `connectedDevice`, adds connected-device manifest permissions, and updates the runtime/unit-test foreground-service type expectations.

## Best Possible Solution

Land the focused Android foreground-service type fix after normal Android owner review and CI, then close https://github.com/openclaw/openclaw/issues/64903 with the merged commit and release proof.

## Reproduction Assessment

Yes, at source level with high confidence. The paired issue’s stack trace points to `startForegroundWithTypes`, and current main plus v2026.5.12 still use a targetSdk 36 `dataSync` foreground service on that path.

## Solution Assessment

Yes. Switching this long-lived node connection to `connectedDevice` at the Android manifest/runtime boundary is narrower than adding restart workarounds, and Android’s connected-device contract fits an external-device interaction over a network connection with `CHANGE_NETWORK_STATE` as a qualifying permission.
