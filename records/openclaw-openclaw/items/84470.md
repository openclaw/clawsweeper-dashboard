## Summary

Keep open. Current main still has a source-reproducible iMessage routing path where an anchorless group watch payload can be treated as a direct sender message, and the linked fixing PR is still open rather than merged.

## What This Changes

The issue asks OpenClaw to prevent malformed iMessage group link-preview watch payloads with missing conversation anchors from being routed as sender DMs.

## Best Possible Solution

Review and land a corrected version of https://github.com/openclaw/openclaw/pull/84503, or an equivalent iMessage monitor guard that repairs by stable GUID before routing and otherwise drops fail-closed.

## Reproduction Assessment

Yes. Source inspection shows the reported chat_id: 0, empty chat anchors, is_group: false payload parses, is classified as direct, and builds To as imessage:<sender>; I did not run tests because this review is read-only.

## Solution Assessment

Yes in direction. The maintainable fix belongs before iMessage inbound routing: recover the real conversation from a stable message GUID when possible, otherwise fail closed rather than letting downstream session routing infer a DM.
