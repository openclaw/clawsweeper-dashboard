## Summary

Keep open. Current main still has a source-reproducible iMessage path where an anchorless group watch payload can be parsed as a non-group message and fall back to `imessage:<sender>` routing, and the linked fix PR is still open rather than merged.

## What This Changes

The issue requests an iMessage monitor guard that recovers a malformed group watch payload's conversation anchor from a stable message identifier or drops the event instead of routing it as a sender DM.

## Best Possible Solution

Land a focused iMessage monitor fix that repairs anchorless watch payloads before debounce/routing when the message GUID resolves to a real conversation, otherwise drops with a clear diagnostic, then close this issue through the merged PR.

## Reproduction Assessment

Yes. Source inspection on current main shows the reported payload shape parses, is not classified as a group, routes with the sender as the direct peer, and builds `To: imessage:<sender>`; I did not run tests because this review is read-only.

## Solution Assessment

Yes for the issue direction: a channel-local pre-routing guard that repairs from a stable GUID or fails closed is the narrow maintainable fix. The linked PR still needs maintainer review and proof before it can be the closing implementation.
