## Summary

Keep open. Current main still has a source-reproducible wrong-recipient path for the reported anchorless iMessage group watch payload, and the linked implementation PR is still open rather than merged.

## What This Changes

The issue asks OpenClaw to recover or drop anchorless iMessage group watch payloads before routing so replies cannot fall back to the sender DM.

## Best Possible Solution

Review and land a focused version of https://github.com/openclaw/openclaw/pull/84503 that repairs anchorless watch payloads by GUID when possible and otherwise drops them before routing, then close this issue after merge proof.

## Reproduction Assessment

Yes. Source inspection shows the reported payload shape parses, is classified as direct when `is_group` is false, and builds `To: imessage:<sender>`; I did not run tests because this review is read-only.

## Solution Assessment

Yes. The maintainable fix is a pre-routing iMessage monitor guard that recovers the real conversation from a stable message GUID or fails closed before any sender-DM route is created.
