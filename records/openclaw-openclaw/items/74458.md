## What This Changes

The PR adds SYSTEMROOT/WINDIR to the host environment security policy and generated artifacts, exports the shared Windows system-root default, resolves Windows ACL helper commands through validated Windows install roots, and adds dotenv, host-env, and Windows ACL regression tests.

## Best Possible Solution

Land one maintainer-approved Windows hardening that blocks untrusted workspace dotenv Windows trust-root overrides through the shared host env policy, validates Windows system roots before resolving audit/fix helper binaries, keeps generated policy artifacts in sync, and coordinates the ACL, reg.exe, taskkill.exe, and shell-resolution surfaces into a coherent final result.
