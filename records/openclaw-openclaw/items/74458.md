## Summary

Keep open: current main only has the workspace `.env` blocklist portion, while Windows ACL audit/fix helper execution still uses unvalidated `SystemRoot`/`WINDIR` values and bare `icacls`; the PR is narrow and has no discrete review finding, but it needs maintainer/security review and validation.

## What This Changes

The PR validates Windows system-root environment values before resolving `icacls.exe`/`whoami.exe`, uses validated System32 paths for ACL reset execution/display, updates host-env policy artifacts, tests, and changelog.

## Best Possible Solution

Land one maintainer-approved Windows hardening that keeps workspace dotenv blocks, host-env policy artifacts, and ACL helper execution aligned around validated System32 paths.
