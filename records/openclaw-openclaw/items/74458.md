## Summary

Keep open. Current main still resolves Windows ACL audit helpers from unvalidated SystemRoot/WINDIR values and still executes the reset helper as a bare icacls command, while this PR is a focused hardening patch for that remaining command-execution surface. Because the change is security-sensitive and overlaps the related Windows root/env hardening thread in #74454, it should go through maintainer/security review rather than cleanup closure or an automated replacement.

## What This Changes

The PR adds SYSTEMROOT/WINDIR to host-env policy artifacts, exports the shared default Windows system root, resolves Windows ACL audit/reset helpers through validated System32 paths, updates source/tests/generated baselines, and adds a changelog entry.

## Best Possible Solution

Land one maintainer-approved Windows hardening that keeps workspace dotenv and host-env policy artifacts consistent, validates Windows system roots before resolving security audit/fix helper binaries, executes ACL helpers by validated System32 paths, and coordinates the related Windows env hardening PRs into one coherent state.

## Solution Assessment
