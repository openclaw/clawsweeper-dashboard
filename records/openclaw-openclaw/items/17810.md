## Summary

Keep open: current main and v2026.6.5 still lack the requested configurable per-file elevation and Project Context exclusion surface. Recent Codex-specific routing now gives SOUL.md, IDENTITY.md, and USER.md stronger treatment for the native Codex harness, but non-Codex prompt rendering and the public config/API boundary remain unresolved, and the direct implementation at https://github.com/openclaw/openclaw/pull/18110 is closed unmerged.

## What This Changes

The issue requests configurable mapping of SOUL.md, IDENTITY.md, USER.md, and MEMORY.md into dedicated prompt sections with optional exclusion from the flat Project Context list.

## Best Possible Solution

Choose the supported prompt/context customization boundary, then implement deterministic per-file section mapping and Project Context exclusion through that boundary with focused docs, schema/types if needed, and coverage for Codex and non-Codex prompt paths.
