## Summary

Keep open. Current main and the latest release still render the requested bootstrap files through the generic Project Context path, with only SOUL.md and MEMORY.md guidance lines added; there is no configurable per-file elevation, exclusion, or plugin/context-engine contract that implements the central request. The closed implementation at https://github.com/openclaw/openclaw/pull/18110 is useful context, but it was not merged and current adjacent open PRs do not own this exact prompt-section feature.

## What This Changes

The issue requests configurable elevation of SOUL.md, IDENTITY.md, USER.md, and MEMORY.md into dedicated prompt sections with optional exclusion from Project Context.

## Best Possible Solution

Define a narrow, documented bootstrap prompt-section contribution contract, then implement deterministic per-file elevation, ordering, and exclusion through the chosen core or plugin-owned seam with aligned tests and docs.

## Reproduction Assessment

Yes at the source level: current main demonstrably routes the requested bootstrap files through the generic Project Context renderer. The reported quality degradation over long sessions is subjective and not a deterministic failing test.

## Solution Assessment

No for the exact config-first proposal until maintainers choose the permanent API. A narrower bootstrap prompt contribution seam would likely be more maintainable because it can support either core policy or plugin ownership without replacing the whole system prompt.
