## Summary

Keep open: current main still renders these bootstrap files as ordered but flat Project Context entries, there is no `context.elevation` or equivalent config surface, and the direct implementation PR is closed unmerged. The remaining decision is the accepted prompt/context customization boundary, not whether current main already implements the request.

## What This Changes

The issue requests configurable elevation of persona and memory files such as `SOUL.md`, `IDENTITY.md`, `USER.md`, and `MEMORY.md` into dedicated prompt sections instead of flat Project Context entries.

## Best Possible Solution

Maintainers should choose the supported context customization surface, then implement a documented deterministic elevation/exclusion path either as typed agent config or as a plugin/context-engine API that can restructure bootstrap files without replacing the whole OpenClaw prompt.

## Reproduction Assessment

Not applicable as a feature request. Source and tests do give a high-confidence current-main check that bootstrap files still render as flat Project Context entries, which is the behavior the issue wants changed.

## Solution Assessment

Unclear until maintainers choose the product/API boundary. The proposed core config is plausible, but the repo direction also favors plugin/API surfaces for optional behavior, and the current context-engine hook is coarser than this request.
