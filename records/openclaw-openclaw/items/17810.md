## Summary

Keep open: current main still renders these bootstrap files as ordered Project Context entries and lacks the requested configurable elevation/exclusion surface; the remaining blocker is a prompt/context API product decision, not conservative cleanup.

## What This Changes

The issue requests configurable elevation of selected context files into dedicated prompt sections instead of rendering them as flat Project Context entries.

## Best Possible Solution

Choose and document the supported context customization contract, then implement deterministic per-file section mapping and Project Context exclusion through the accepted core config or plugin/context-engine API with focused docs and tests.

## Reproduction Assessment

Not applicable: this is a feature/config request rather than a broken existing contract. Source inspection gives a high-confidence check of current main's behavior: bootstrap files still render under Project Context and no `context.elevation` config surface exists.

## Solution Assessment

Unclear until maintainers choose the product/API boundary. A core config option is plausible, but a context-engine or plugin prompt contract may be more maintainable if this customization should not become fixed core prompt policy.
