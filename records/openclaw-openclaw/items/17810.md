## Summary

Keep open: current main and the latest release still render bootstrap files as flat Project Context entries, no configurable elevation/exclusion contract exists, and the remaining blocker is a prompt/context API product decision rather than conservative cleanup.

## What This Changes

The issue requests a configurable `context.elevation` surface that maps files such as `SOUL.md`, `IDENTITY.md`, `USER.md`, and `MEMORY.md` into dedicated prompt sections and optionally excludes them from flat Project Context rendering.

## Best Possible Solution

Choose and document the supported prompt/context customization contract, then implement deterministic per-file section mapping and Project Context exclusion through that contract with focused docs and tests.

## Reproduction Assessment

Not applicable: this is a feature/config request rather than a broken existing contract. Source inspection gives a high-confidence verification path because current main still renders the named files under Project Context and has no `context.elevation` surface.

## Solution Assessment

Unclear until maintainers choose the product/API boundary. A core config option is plausible, but a context-engine or plugin prompt contract may be more maintainable if this customization should not become fixed core prompt policy.
