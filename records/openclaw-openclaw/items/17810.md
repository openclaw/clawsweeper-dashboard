## Summary

Keep open. Current main and the latest release still render the requested SOUL/IDENTITY/USER/MEMORY-style files through the existing Project Context path with no configurable elevation or exclusion surface, and the direct implementation PR is closed unmerged and conflicting.

## What This Changes

Requests configurable elevation and optional exclusion of selected bootstrap context files into named prompt sections instead of rendering all files as flat Project Context entries.

## Best Possible Solution

Maintainers should choose one accepted prompt/context contract, then implement either documented typed core elevation/exclusion config or a plugin/context-engine API that can place selected bootstrap files into named sections without replacing the whole system prompt.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.10 render the relevant files as Project Context entries and expose no elevation/exclusion config. The long-session persona drift itself was not live-reproduced in this read-only review.

## Solution Assessment

Unclear as proposed. The requested `context.elevation` shape is plausible, but the best maintainable solution needs maintainer agreement on core config versus plugin/context-engine ownership.
