## Summary

Keep open: current main still renders SOUL/IDENTITY/USER/MEMORY-style bootstrap files as flat Project Context entries and has no documented configurable elevation/exclusion surface; the remaining blocker is a prompt/context API product decision, not conservative cleanup.

## What This Changes

The issue requests configurable per-file prompt elevation for bootstrap files such as SOUL.md, IDENTITY.md, USER.md, and MEMORY.md, including optional exclusion from Project Context.

## Best Possible Solution

Maintainers should choose the supported prompt/context customization contract, then implement deterministic per-file section mapping and exclusion with focused prompt, config, SDK or docs coverage.

## Reproduction Assessment

Not applicable: this is a feature/config request, not a broken existing contract. Source inspection on current main verifies the requested surface is absent and the files still render as flat Project Context entries.

## Solution Assessment

Unclear until maintainers choose the product/API boundary. The requested core config is plausible, but a plugin/context-engine contract may be more maintainable if OpenClaw wants per-file prompt transforms outside core prompt policy.
