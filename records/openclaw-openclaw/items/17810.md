## Summary

Keep open. Current main and v2026.6.8 still render SOUL/IDENTITY/USER/MEMORY-style files as ordered Project Context entries, with no configurable elevation or exclusion API; the remaining blocker is maintainer product/API direction, not lack of source evidence.

## What This Changes

Request to add configurable prompt-section elevation for selected bootstrap/context files such as SOUL.md, IDENTITY.md, USER.md, and MEMORY.md, with optional exclusion from flat Project Context.

## Best Possible Solution

Maintainers should choose one accepted prompt/context contract, then implement either a documented typed core elevation/exclusion config or a plugin/context-engine API that can place selected bootstrap files into named sections without replacing the whole system prompt.

## Reproduction Assessment

Yes, at source level: current main still renders the relevant files as flat Project Context entries and exposes no elevation or exclusion config. The reported long-session persona drift was not deterministically reproduced in this read-only review.

## Solution Assessment

Unclear as proposed. The requested `context.elevation` shape is plausible, but the best maintainable solution needs maintainer agreement on whether this belongs as core config or as a context-engine/plugin API extension.
