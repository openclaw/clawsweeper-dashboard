## Summary

Keep open: current main still renders the requested bootstrap files as flat Project Context entries and lacks a documented config or plugin API for per-file elevation/exclusion, while the direct implementation PR is closed unmerged.

## What This Changes

Request configurable elevation of files such as `SOUL.md`, `IDENTITY.md`, `USER.md`, and `MEMORY.md` into dedicated prompt sections instead of flat Project Context entries.

## Best Possible Solution

Define a deterministic, documented bootstrap elevation contract that maps configured files to named sections, can exclude elevated files from the flat list, and updates renderer, schema, help labels, docs, and regression tests together.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to pass `SOUL.md`, `IDENTITY.md`, `USER.md`, and `MEMORY.md` as `contextFiles` into `buildAgentSystemPrompt`; current main sorts them but renders them as `## <path>` entries under `# Project Context`, with only the fixed SOUL.md guidance sentence.

## Solution Assessment

Unclear. The proposed `context.elevation` shape is plausible, but maintainers should first decide core config versus plugin/context-engine contract because current public plugin surfaces cannot selectively rewrite or exclude injected bootstrap files.
