Keep open: current main still lacks the requested configurable `context.elevation` section/exclusion contract. ## What This Changes

The issue requests configurable elevation of selected workspace context files into dedicated prompt sections, with optional exclusion from flat Project Context.

## Best Possible Solution

Choose the supported prompt/context customization boundary, then implement deterministic per-file section mapping and Project Context exclusion through that boundary with focused docs and tests.

Not applicable: this is a feature/config request rather than a broken existing contract. Source inspection gives high confidence that current main lacks the requested configurable elevation surface.

## Solution Assessment

Unclear until maintainers choose the product/API boundary. A typed core config option is plausible, but a plugin/context-engine contract may be the better long-term surface if this should not become fixed core prompt policy.
