## Summary

Keep open: current main still flat-renders the relevant bootstrap files under Project Context, there is no documented/configured elevation surface, and the remaining question is the product/API boundary for prompt customization.

## What This Changes

Request configurable elevation of selected bootstrap context files such as SOUL.md, IDENTITY.md, USER.md, and MEMORY.md into dedicated prompt sections with optional exclusion from Project Context.

## Best Possible Solution

Maintainers should choose the supported customization contract, then implement documented deterministic per-file elevation and exclusion either in typed agent config or through a plugin/context-engine API that exposes the needed bootstrap-file transform.

## Reproduction Assessment

Not applicable as a feature request. Current main can still be verified from source, tests, and docs: the files render as flat Project Context entries and no elevation config surface exists.

## Solution Assessment

Unclear until maintainers choose the product/API boundary. The requested core config is plausible, but a plugin/context-engine contract may be the more maintainable surface if OpenClaw wants this outside core prompt policy.
