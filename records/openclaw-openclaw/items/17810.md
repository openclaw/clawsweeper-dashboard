## Summary

Keep open. Current main still sorts the requested bootstrap files but renders them as flat Project Context entries, exposes no `context.elevation`-style schema or prompt parameter, and the direct implementation PR is closed unmerged; the remaining blocker is the maintainer product/API boundary.

## What This Changes

Request configurable elevation of `SOUL.md`, `IDENTITY.md`, `USER.md`, and `MEMORY.md` into dedicated prompt sections with optional exclusion from flat Project Context.

## Best Possible Solution

Define a deterministic, documented per-file bootstrap elevation contract that preserves prompt-cache ordering, excludes elevated files from the flat list when configured, and keeps schema, help labels, tests, and public docs aligned.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to pass `SOUL.md`, `IDENTITY.md`, `USER.md`, and `MEMORY.md` as `contextFiles` into `buildAgentSystemPrompt`; current code and tests show they render as `## <path>` entries under `# Project Context`, with only a SOUL.md guidance sentence.

## Solution Assessment

Unclear. The proposed `context.elevation` config is plausible, but current extension surfaces do not cleanly provide per-file rewrite/exclusion, so maintainers should choose the public core or plugin/context-engine contract first.
