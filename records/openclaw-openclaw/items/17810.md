## Summary

Keep open. Current main still renders the requested bootstrap files as ordered but flat Project Context entries, has no `context.elevation`-style schema or prompt parameter, and the direct implementation PR is closed unmerged; the remaining blocker is maintainer product/API direction rather than stale cleanup.

## What This Changes

The issue requests configurable elevation of selected bootstrap/context files such as `SOUL.md`, `IDENTITY.md`, `USER.md`, and `MEMORY.md` into dedicated system-prompt sections instead of flat Project Context entries.

## Best Possible Solution

Define a deterministic, documented bootstrap elevation contract that maps configured files to named sections, can exclude elevated files from the flat list, and updates renderer, schema, help labels, docs, and regression tests together.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to pass `SOUL.md`, `IDENTITY.md`, `USER.md`, and `MEMORY.md` as `contextFiles` into `buildAgentSystemPrompt`; current source and tests show they render as `## <path>` entries under `# Project Context`, with only the fixed SOUL.md guidance sentence.

## Solution Assessment

Unclear. The proposed `context.elevation` shape is plausible, but maintainers should first decide core config versus plugin/context-engine contract because the current public plugin surfaces do not provide a clean per-file rewrite/exclusion contract.
