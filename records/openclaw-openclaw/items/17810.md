## Summary

Keep open: current main still renders SOUL/IDENTITY/USER/MEMORY-style files as flat Project Context entries and has no configurable elevation/exclusion surface; the remaining question is a prompt/context API decision, not conservative cleanup.

## What This Changes

Request configurable elevation of selected context files such as SOUL.md, IDENTITY.md, USER.md, and MEMORY.md into dedicated prompt sections instead of flat Project Context entries.

## Best Possible Solution

Maintainers should choose the supported customization contract, then implement documented deterministic per-file section mapping and exclusion through typed agent config or a plugin/context-engine API with focused prompt, config, and docs coverage.

## Reproduction Assessment

Not applicable: this is a feature/config request, not a broken existing contract. Source inspection on current main verifies the requested surface is absent and the files still render as flat Project Context entries.

## Solution Assessment

Unclear until maintainers choose the product/API boundary. The requested core config is plausible, but a plugin/context-engine contract may be more maintainable if OpenClaw wants per-file prompt transforms outside core prompt policy.
