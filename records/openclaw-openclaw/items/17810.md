## What This Changes

The issue requests configurable elevation of selected bootstrap/context files into dedicated system-prompt sections instead of rendering them as ordinary Project Context entries.

## Best Possible Solution

Choose and document the supported prompt/context customization contract, then implement deterministic per-file section mapping and Project Context exclusion through that contract with focused docs and tests.

## Reproduction Assessment

Not applicable: this is a feature/config request rather than a broken existing contract. Source inspection gives high-confidence verification that current main still renders the named files under Project Context and has no `context.elevation` surface.

## Solution Assessment

Unclear until maintainers choose the product/API boundary. A core config option is plausible, but a context-engine or plugin prompt contract may be more maintainable if this customization should not become fixed core prompt policy.
