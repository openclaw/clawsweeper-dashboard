## Summary

Keep open: current main still renders the bootstrap files as ordered but flat Project Context entries, there is no `context.elevation` or equivalent config surface, and the direct implementation PR is closed unmerged. The remaining blocker is the accepted prompt/context customization boundary, not stale evidence.

## What This Changes

The issue requests configurable elevation of selected context files such as `SOUL.md`, `IDENTITY.md`, `USER.md`, and `MEMORY.md` into dedicated prompt sections.

## Best Possible Solution

Maintainers should choose the supported customization surface, then implement a documented deterministic elevation/exclusion path either as typed agent config or as a plugin/context-engine API that can restructure bootstrap files without replacing the whole system prompt.

## Reproduction Assessment

Not applicable as a feature request. Source, docs, and tests give a high-confidence current-main check that these files still render as flat Project Context entries, which is the behavior the issue wants changed.

## Solution Assessment

Unclear until maintainers choose the product/API boundary. The proposed core config is plausible, but the current plugin/context-engine surface is too coarse for selective file elevation and exclusion without a new or changed contract.
