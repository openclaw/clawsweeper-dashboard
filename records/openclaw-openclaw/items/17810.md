## Summary

Keep open. Current main and v2026.6.5 improve SOUL.md and MEMORY.md priority guidance, but all requested bootstrap files still render through the generic Project Context path, and the available context-engine contract cannot selectively elevate or exclude them. The remaining work needs maintainer approval on the permanent core-config versus plugin/API boundary.

## What This Changes

Requests configurable elevation of selected bootstrap context files into dedicated system-prompt sections instead of rendering them only under Project Context.

## Best Possible Solution

Add a narrow typed bootstrap prompt-section transformation or contribution contract, then implement deterministic elevation, ordering, and exclusion through that seam with aligned tests and documentation.

## Reproduction Assessment

Yes at the source level: current main demonstrably routes the requested bootstrap files through the generic Project Context renderer. The reported model-quality degradation itself is subjective and is not a deterministic failing test.

## Solution Assessment

No for the config-first implementation as written. A narrow typed bootstrap prompt contribution seam is more maintainable because it avoids full prompt replacement while preserving the option for either core policy or plugin ownership.
