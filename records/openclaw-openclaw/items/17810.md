## What This Changes

Request adds a configurable `context.elevation` mapping that renders selected bootstrap files as dedicated prompt sections and can exclude them from flat Project Context.

## Best Possible Solution

Choose and document the supported context customization contract, then implement deterministic per-file section mapping and Project Context exclusion with focused prompt, config or SDK, docs, and tests.

## Reproduction Assessment

Not applicable: this is a feature/config request, not a broken existing contract. Source inspection shows current main still renders these files under Project Context and lacks the requested elevation surface.

## Solution Assessment

Unclear until maintainers choose the product/API boundary. A core config option is plausible, but a context-engine or plugin prompt contract may be more maintainable if this customization should not become fixed core prompt policy.
