## Summary

Keep open: current main still only orders and flat-renders bootstrap context files, the direct implementation PR is closed unmerged, and the remaining decision is the supported prompt customization surface.

## What This Changes

Add a configurable `context.elevation`-style mapping that can render selected context files as dedicated prompt sections and optionally exclude them from flat Project Context.

## Best Possible Solution

Maintainers should choose the supported customization contract, then implement documented deterministic per-file elevation and exclusion either in typed agent config or through a plugin/context-engine API that exposes the needed bootstrap-file transform.

## Reproduction Assessment

Not applicable as a feature request. Current main can still be checked with high confidence from source, docs, and tests: these files render as flat Project Context entries and no elevation config surface exists.

## Solution Assessment

Unclear until maintainers choose the product/API boundary. The requested core config is plausible, but the current plugin/context-engine surfaces do not expose enough structured bootstrap-file control for selective elevation and exclusion.
