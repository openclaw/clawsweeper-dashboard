## Summary

Keep open: current main still renders the named bootstrap files as flat Project Context entries with only fixed SOUL.md guidance, the requested configurable elevation surface is absent, and the linked implementation PR is closed unmerged.

## What This Changes

Request configurable elevation of selected bootstrap/context files into dedicated system prompt sections, with optional exclusion from the flat Project Context.

## Best Possible Solution

Define one documented prompt/context contract that lets users deterministically map bootstrap files to named sections while preserving prompt-cache stability and plugin extensibility.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main confirms the existing behavior is flat Project Context rendering rather than the requested elevation surface.
