## Summary

Keep open. Current main and v2026.6.6 still lack the requested configurable per-file elevation/exclusion surface for the default OpenClaw system-prompt path; native Codex has a partial harness-specific route for SOUL/IDENTITY/USER and MEMORY, but that does not solve the general product/API decision.

## What This Changes

The issue requests configurable elevation of selected context files such as SOUL.md, IDENTITY.md, USER.md, and MEMORY.md into dedicated prompt sections instead of flat Project Context entries.

## Best Possible Solution

Maintainers should choose one accepted prompt/context contract: either a documented typed core config for deterministic per-file elevation and exclusion, or a plugin/context-engine API that can own per-file placement without replacing the whole system prompt.

## Reproduction Assessment

Yes at source level: current main still renders injected files inside Project Context for the embedded OpenClaw system-prompt path and exposes no configurable elevation or exclusion surface. The reported long-session persona drift is model-dependent, so the deterministic reproduction is the missing prompt/config behavior rather than a runtime failure trace.

## Solution Assessment

No, not as an immediate unreviewed context.elevation patch. The maintainable path is to choose the core-versus-plugin/context-engine boundary first, then implement without duplicating or conflicting with native Codex bootstrap routing.
