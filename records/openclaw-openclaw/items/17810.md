## Summary

Keep open. Current main and the latest release still render the requested bootstrap files as flat Project Context entries and expose no configurable per-file elevation or exclusion surface; the direct implementation PR is closed unmerged, so the remaining blocker is maintainer product/API direction.

## What This Changes

Request configurable prompt elevation so files like SOUL.md, IDENTITY.md, USER.md, and MEMORY.md can render as dedicated prompt sections and optionally be excluded from flat Project Context.

## Best Possible Solution

Maintainers should choose one accepted prompt/context contract, then either add a documented typed core elevation/exclusion config or extend the plugin/context-engine API so external code can place bootstrap files into named sections without replacing the whole system prompt.

## Reproduction Assessment

Yes, at source level: current main still renders the relevant files as flat Project Context entries and exposes no elevation or exclusion config. The reported persona drift over long sessions was not deterministically reproduced here.

## Solution Assessment

No, not as an immediate unreviewed `context.elevation` patch. The best path is to settle the core-versus-plugin/context-engine boundary first, then implement the chosen contract with schema, docs, and prompt-shape coverage.
