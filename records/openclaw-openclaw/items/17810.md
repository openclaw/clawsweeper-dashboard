## Summary

Keep open. Current main and the latest release still lack the requested configurable per-file elevation and Project Context exclusion surface; the only direct implementation PR is closed unmerged, and the remaining choice is a prompt/config/API product decision.

## What This Changes

The issue requests configurable elevation of selected workspace context files such as SOUL.md, IDENTITY.md, USER.md, and MEMORY.md into dedicated prompt sections instead of flat Project Context entries.

## Best Possible Solution

Maintainers should choose one accepted prompt/context contract, then either add a documented typed core elevation/exclusion config or extend the plugin/context-engine API so external code can place bootstrap files into named sections without replacing the whole system prompt.

## Reproduction Assessment

Yes, at source level: current main still renders the relevant files as flat Project Context entries and exposes no elevation or exclusion config. The reported persona drift over long sessions is not deterministically reproduced here.

## Solution Assessment

No, not as an immediate unreviewed context.elevation patch. The best path is to settle the core-versus-plugin/context-engine boundary first, then implement the chosen contract with schema, docs, and prompt-shape coverage.
