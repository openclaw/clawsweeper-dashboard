## Summary

Keep open: current main and v2026.5.22 still lack a general configurable context-file elevation and Project Context exclusion surface. The closed implementation PR and newer Codex-specific prompt-routing work are useful context, but the remaining decision is the accepted prompt/config/plugin boundary rather than a narrow bug fix.

## What This Changes

The issue requests configurable elevation of selected context files into dedicated prompt sections instead of rendering them only inside the flat Project Context block.

## Best Possible Solution

Choose the supported prompt/context customization boundary, then implement deterministic per-file section mapping and Project Context exclusion through that boundary with focused docs and tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request rather than a broken documented contract. Source, tests, docs, related PR state, and release inspection give high confidence that the requested general configurable elevation surface is absent.

## Solution Assessment

Unclear until maintainers choose the product/API boundary. A typed core config option is plausible, but a plugin/context-engine prompt contract may be safer if this should not become fixed core prompt policy.
