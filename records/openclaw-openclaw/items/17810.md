## Summary

Keep open. Current main still renders the requested bootstrap files as ordered but flat Project Context entries, exposes no `context.elevation`-style config or prompt parameter, and the direct implementation PR is closed unmerged; the remaining blocker is maintainer prompt/API direction.

## What This Changes

Request configurable elevation of selected bootstrap files such as `SOUL.md`, `IDENTITY.md`, `USER.md`, and `MEMORY.md` into dedicated prompt sections before flat Project Context, with optional exclusion from the flat list.

## Best Possible Solution

Define a deterministic, documented per-file bootstrap elevation contract that preserves prompt-cache ordering and aligns schema, tests, public docs, and either core config or plugin/context-engine API behavior.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to call `buildAgentSystemPrompt` with `SOUL.md`, `IDENTITY.md`, `USER.md`, and `MEMORY.md` as `contextFiles`; current source and tests show they render as `## <path>` entries under `# Project Context`, with only a fixed SOUL.md guidance sentence.

## Solution Assessment

Unclear. The proposed `context.elevation` shape is plausible, but the maintainable fix first needs a maintainer decision on whether this belongs in typed core agent config or a file-level extension API.
