## Summary

Current main still renders the named bootstrap files as ordered Project Context entries, with fixed guidance for SOUL.md and MEMORY.md but no configurable section mapping or exclusion. The request remains useful, but the linked implementation PR closed unmerged and maintainers must decide whether the permanent surface belongs in typed core config or a narrower plugin/context extension contract.

## What This Changes

Add configurable mapping that elevates selected bootstrap files into dedicated system-prompt sections and optionally removes them from the ordinary Project Context block.

## Best Possible Solution

Define one typed, deterministic prompt-section contribution contract for bootstrap context, then implement elevation through that contract with stable ordering, cache-boundary preservation, schema/docs alignment, and focused fresh-session plus long-context proof.

## Reproduction Assessment

Yes at source level: current main deterministically renders these files within Project Context and exposes no configurable elevation surface. The claimed multi-hour persona collapse itself is model-dependent and is not a deterministic runtime reproduction.

## Solution Assessment

No, not as a direct `context.elevation` addition yet; the narrowest maintainable path is to choose and type the core-versus-plugin prompt contribution boundary first, then add the behavior once without parallel sources of truth.
