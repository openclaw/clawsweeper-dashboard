## Summary

Keep open: current main still renders SOUL/IDENTITY/USER/MEMORY-style bootstrap files as ordered but flat Project Context entries, lacks the requested configurable elevation/exclusion surface, and the prior implementation PR is closed unmerged. The remaining decision is the public prompt/context API boundary, not conservative cleanup.

## What This Changes

Request configurable elevation of selected bootstrap files such as SOUL.md, IDENTITY.md, USER.md, and MEMORY.md into dedicated prompt sections with optional exclusion from Project Context.

## Best Possible Solution

Choose and document the supported prompt/context customization contract, then implement deterministic per-file section mapping and Project Context exclusion through that contract with focused docs and tests.

## Reproduction Assessment

Not applicable: this is a feature/config request rather than a broken existing contract. Source inspection gives a high-confidence verification path: current main still renders the files under Project Context and has no `context.elevation` config surface.

## Solution Assessment

Unclear until maintainers choose the product/API boundary. A core config option is plausible, but a context-engine or plugin prompt contract may be more maintainable if this customization should not become fixed core prompt policy.
