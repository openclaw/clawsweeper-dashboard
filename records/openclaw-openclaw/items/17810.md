## Summary

Keep open. Current main and the latest release still render these bootstrap files as ordered but flat Project Context entries with fixed SOUL/MEMORY guidance, while the exact implementation PR is closed unmerged and the remaining decision is the accepted core config versus plugin/context-engine API boundary.

## What This Changes

The issue requests configurable elevation of selected bootstrap/context files such as SOUL.md, IDENTITY.md, USER.md, and MEMORY.md into dedicated prompt sections with optional exclusion from Project Context.

## Best Possible Solution

Maintainers should choose the accepted context surface, then either add documented typed per-file elevation/exclusion in core or expose a plugin/context-engine hook that can transform bootstrap files before Project Context rendering.

## Reproduction Assessment

Yes at source level: current main deterministically renders the named files inside Project Context and exposes no configurable elevation surface. The claimed multi-hour persona collapse itself is model-dependent and was not live-reproduced.

## Solution Assessment

Unclear as an immediate config addition. The maintainable path depends on a maintainer choice between typed core config and a plugin/context-engine API, so this should not be queued as an automatic patch yet.
