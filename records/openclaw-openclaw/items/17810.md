## Summary

Keep open: current main still renders SOUL/IDENTITY/USER/MEMORY-style bootstrap files as ordered but flat Project Context entries, has no configurable elevation/exclusion surface, and the related implementation PR was closed unmerged. Request to add configurable context-file elevation and exclusion so selected bootstrap files render as dedicated prompt sections instead of flat Project Context files.

## Best Possible Solution

Maintainers should choose the supported customization contract, then implement documented deterministic per-file section mapping and exclusion through typed agent config or a plugin/context-engine API with focused prompt/docs/config tests.
