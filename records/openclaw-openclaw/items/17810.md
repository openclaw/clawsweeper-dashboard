## Summary

Keep open: current main still renders the requested bootstrap files as ordered but flat Project Context entries, the requested elevation/exclusion config surface is absent, and the related implementation PR at https://github.com/openclaw/openclaw/pull/18110 was closed unmerged. This remains a feature/API boundary decision rather than conservative cleanup.

## What This Changes

Request configurable elevation of SOUL.md, IDENTITY.md, USER.md, and MEMORY.md into dedicated prompt sections with optional exclusion from flat Project Context.

## Best Possible Solution

Maintainers should choose the supported customization contract, then implement documented deterministic per-file elevation and exclusion either in typed agent config or through a plugin/context-engine API that exposes bootstrap-file transforms.

## Reproduction Assessment

Not applicable as a feature/config request. Source inspection on current main verifies the gap: the files still render as flat Project Context entries and no elevation config surface exists.

## Solution Assessment

Unclear until maintainers choose the product/API boundary. The requested core config is plausible, but a plugin/context-engine contract may be more maintainable if OpenClaw wants this outside core prompt policy.
