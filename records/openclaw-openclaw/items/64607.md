## Summary

Keep open: current main and v2026.6.8 have partial structured assistant media support, but the broad inline-media request remains unresolved because remote Markdown images are still flattened, Control UI CSP intentionally blocks remote image/media loads, web previews are not present, and related narrower items remain open for maintainer product/security decisions.

## What This Changes

The issue requests inline chat previews for generated or received images, audio, video, and web links instead of requiring users to open media externally.

## Best Possible Solution

Keep this open as a maintainer-owned design item, preserve the existing secured same-origin assistant-media path, and split approved work into narrower milestones for remote image policy, tool-returned image rendering, audio/video presentation, and web-link previews.

## Reproduction Assessment

Yes, at source level: current main still flattens remote Markdown images to alt text and the Control UI CSP blocks remote image/media sources, while only structured same-origin assistant media has inline rendering. I did not run a live browser repro because this review is read-only and the source/tests/docs are already decisive for the remaining gaps.

## Solution Assessment

Unclear as a single broad feature request: inline media is valuable, but the best fix is not one large renderer patch. The maintainable path is a product/security decision plus smaller changes that keep the existing assistant-media ticket route and explicitly decide remote fetch/link-preview policy.
