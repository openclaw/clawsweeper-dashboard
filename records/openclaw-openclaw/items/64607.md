## Summary

Keep open. Current main has meaningful partial support for inline Control UI media, but the umbrella request is still not complete: remote markdown images are intentionally flattened and blocked by CSP, automatic web/link previews are absent, and the remaining media behavior spans UI, Gateway persistence, tool-result delivery, and security policy.

## What This Changes

The issue requests inline chat display for generated or received images, audio/music, video, and web page previews instead of requiring external open commands.

## Best Possible Solution

Keep this as the umbrella unless maintainers split it. Preserve the shipped secure media directive, managed image, attachment-player, and explicit embed paths, then finish the remaining work as smaller reviewed slices for remote-image policy, web/link previews, durable audio/video references, and tool-result preview behavior.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as one feature umbrella, but yes for key remaining gaps: current tests/source show remote markdown images flatten to alt text, CSP blocks remote image fetches, video uploads are rejected, and a focused source search found no automatic link-preview implementation.

## Solution Assessment

Is this the best way to solve the issue? No single patch is the best solution for the whole umbrella. The maintainable path is to keep this open or split it while landing smaller security-reviewed PRs for each media/link-preview class.
