## Summary

Keep open. Current main has partial structured attachment rendering for trusted/local images and audio/video/document attachment cards, but the umbrella request remains unresolved: remote Markdown images are still intentionally flattened, web/link previews are not implemented, and video upload remains unsupported.

## What This Changes

The issue requests inline display for chat media: images, audio, video, and rich web/link previews instead of requiring external open commands.

## Best Possible Solution

Keep this as a product/security umbrella until maintainers choose the allowed media policy, with narrower fixes landing against the open remote-image, tool-result, and inbound-media preview reports.

## Reproduction Assessment

Yes for the missing remote-image slice: current source and tests show Markdown remote images are flattened to alt text, and docs state the CSP rejects remote http(s) images. The full umbrella feature is broader than a single repro because audio, video, generated images, inbound media, and web previews have separate paths.

## Solution Assessment

No single automatic fix is the best way to solve the issue. The maintainable path is staged product/security work: preserve the authenticated local/managed media route, decide remote fetch/link-preview policy explicitly, and land narrow issues with proof.
