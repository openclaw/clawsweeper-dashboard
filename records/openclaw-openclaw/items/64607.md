## Summary

Keep open. Current main has partial Control UI media support for structured assistant attachments, but this umbrella still covers unresolved remote Markdown image policy, automatic web/link previews, video upload/persistence behavior, and an open related remote-image issue/PR that needs product and security review.

## What This Changes

The issue requests inline display of generated or received images, audio, video, and web previews in chat instead of requiring external open commands.

## Best Possible Solution

Split this umbrella into focused maintainer-owned slices for remote-image policy, secure automatic link previews, durable audio/video media handling, and upload/render behavior while keeping the shipped structured media paths intact.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature umbrella. Source checks give high-confidence representative gaps: remote Markdown images are still flattened, CSP blocks remote image loads, video uploads are filtered, and no Control UI link-preview implementation was found.

## Solution Assessment

No single broad patch is the best way to solve this issue. The maintainable path is to split the work and make explicit product/security decisions for remote fetching, CSP, persistence, and per-media rendering behavior.
