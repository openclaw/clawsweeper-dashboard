## Summary

Keep open. Current main and v2026.6.6 only partially cover inline media: structured/managed image and trusted local audio/video attachment rendering exists, but remote Markdown images are still intentionally flattened, video chat uploads are filtered, web previews are not implemented, and the remaining scope needs product and security review.

## What This Changes

Request inline display for images, audio, video, and web previews in OpenClaw chat instead of requiring external open commands.

## Best Possible Solution

Define a secure Control UI media-preview policy, land the narrow image/tool/inbound fixes where appropriate, and add web/link previews only through authenticated, scoped, or explicitly gated fetch/proxy paths.

## Reproduction Assessment

Yes for concrete remaining pieces: current source and tests show remote Markdown images flatten to alt text, CSP blocks remote image loads, and video chat uploads are filtered. The full umbrella still needs product-scoped acceptance criteria for which media sources and preview types should be supported.

## Solution Assessment

No, the issue as written is too broad for one direct fix. The best path is to split source types and media kinds, preserve the existing managed attachment safety paths, and require product/security approval for remote web previews.
