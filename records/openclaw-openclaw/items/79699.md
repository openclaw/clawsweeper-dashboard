## Summary

Keep open: current main has partial contamination defenses, but the session transcript path still records raw session-corpus lines as low-scoring daily signals while default deep promotion thresholds make those entries unable to promote by themselves.

## What This Changes

The issue asks for memory dreaming promotion to extract and promote real work from session transcripts instead of filling the short-term store with low-scoring, dreaming-contaminated corpus lines.

## Best Possible Solution

Memory-core should define and implement a grounded, session-aware extraction path that filters dreaming/cron chatter and produces durable work candidates capable of satisfying default deep promotion thresholds without promoting raw low-confidence chat lines.

## Reproduction Assessment

Yes, source-reproducible: current main still records session transcript corpus entries as score-0.58 daily signals, while default deep promotion requires 0.8 score, 3 signals, and 3 contexts. The existing session-corpus ranking tests lower all those gates to zero, so they do not reproduce a successful default promotion path.

## Solution Assessment

Unclear; the current contamination guards are a useful partial fix, but they do not solve the session-only promotion ceiling. A simple score bump would be risky, so the maintainable fix likely needs owner judgment on structured extraction or grounded candidate staging.
