## Summary

Keep open: current main has partial contamination defenses, but the session-ingestion path still records real session lines as one low-scored daily signal that cannot satisfy normal deep-promotion gates.

## What This Changes

The issue reports that memory dreaming promotion ingests contaminated session-corpus lines with low uniform scores, so real work does not get promoted into durable memory.

## Best Possible Solution

Keep this open and define a bounded memory-core path that turns real session work into high-confidence grounded promotion candidates while continuing to exclude dreaming, cron, heartbeat, and diary artifacts.

## Reproduction Assessment

Yes. Source inspection shows session ingestion still creates one 0.58 daily signal per session-corpus line, while scheduled deep promotion requires multiple signals, multiple query/day contexts, and a much higher weighted score.

## Solution Assessment

No. Current main’s contamination filters are useful, but the remaining raw-session scoring path is still not the best fix; lowering thresholds broadly would risk noisy MEMORY.md entries, so the maintainable solution needs session-aware or grounded extraction policy.
