## What This Changes

The PR relaxes `messages.groupChat.historyLimit` validation from positive integers to nonnegative integers and adds focused Zod and Gateway `config.patch` regression tests.

## Best Possible Solution

Land the focused schema and regression-test fix with the documented zero-means-disabled behavior, and let the linked bug issue close only after the PR merges.

## Reproduction Assessment

Yes, source-level. Current main uses `.positive()` for `messages.groupChat.historyLimit`, Zod 4.4.3 defines that as `> 0`, and the docs say `0` disables history; I did not run tests because this review was read-only.

## Solution Assessment

Yes. Changing the single Zod constraint to `.min(0)` is the narrowest maintainable fix, and current main computes the public runtime schema from `OpenClawSchema`, so no generated schema artifact needs updating.
