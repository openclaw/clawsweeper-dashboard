The patch is a narrow active fix with sufficient live Gateway proof, but it still needs maintainer acceptance of the upgrade behavior before merge.

## What This Changes

The branch changes `GroupChatSchema.historyLimit` from positive-only to non-negative and adds direct Zod plus Gateway `config.patch` regression tests.

## Best Possible Solution

Land the focused schema and regression-test change after maintainers explicitly accept the documented zero-means-disabled upgrade behavior, then let the linked issue close from the merge.

## Reproduction Assessment

Yes. Source inspection shows current main rejects `0` through `GroupChatSchema.historyLimit: z.number().int().positive()`, and the Gateway `config.patch` path validates through that schema.

## Solution Assessment

Yes, with maintainer acceptance of the compatibility effect. The narrow fix is to change the outlier validator to the existing `.min(0)` convention and cover direct parsing plus the Gateway patch path; current schema generation means no separate generated base-schema file is needed.
