## Summary

Keep open: current main still rejects `messages.groupChat.historyLimit: 0`, while this PR contains the narrow code, generated-schema, and test changes needed to align group chat history with the documented zero-disables-history behavior. The paired bug remains open and should close only after this PR or an equivalent fix lands.

## What This Changes

This PR changes `GroupChatSchema.historyLimit` from `.positive()` to `.min(0)`, updates the generated base config schema from `exclusiveMinimum: 0` to `minimum: 0`, and adds boundary tests for zero, positive, and negative values.

## Best Possible Solution

Land this PR or an equivalent narrow fix that keeps the Zod schema, generated base schema, and regression coverage in sync; then close #65305 as implemented.

## Reproduction Assessment

Yes. The source-level reproduction is deterministic on current main: parsing `{ historyLimit: 0 }` through `GroupChatSchema.unwrap().safeParse(...)` fails because `.positive()` requires `> 0`, and the generated schema also encodes `exclusiveMinimum: 0`.

## Solution Assessment

Yes. The proposed `.min(0)` validator change plus regenerated schema and boundary tests is the narrowest maintainable solution and matches the existing DM/channel history-limit semantics.
