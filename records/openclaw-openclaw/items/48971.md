## What This Changes

The PR exports `sanitizeMimeType`, anchors MIME type parsing with a case-insensitive parameter-aware regex, and adds focused regression tests for parameterized and malformed MIME inputs.

## Best Possible Solution

Keep the shipped v2026.5.6 implementation and close this PR as obsolete; future MIME parsing changes should build on `src/media-understanding/apply.sanitize-mime.test.ts`.

## Reproduction Assessment

Yes from source inspection: the original prefix-only sanitizer described by the linked report would accept a valid MIME prefix before malformed suffixes, while current main’s tests encode the expected rejection cases. I did not run tests because this was a read-only cleanup review.

## Solution Assessment

Yes for current main, no for this PR. The shipped fix is the better solution because it uses a structured parameter grammar and broader regression tests, while this PR’s broad `;.*` tail leaves malformed parameter payloads accepted.
