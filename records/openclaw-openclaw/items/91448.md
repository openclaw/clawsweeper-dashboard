Full review comments:

- **[P1] Accept documented Telegram topic targets:** `ui/src/ui/controllers/cron.ts:173`
  - body: This rejects every Telegram announce target that is not only `-?\d+`, but current cron docs and the Telegram parser support forum-topic targets such as `-1001234567890:topic:123` and `-1001234567890:123`. Editing an existing cron job with a documented topic target would now be blocked in the dashboard before runtime can route it.
  - confidence: 0.94
- **[P2] Render the announce target validation error:** `ui/src/ui/controllers/cron.ts:174`
  - body: The new `errors.deliveryTo` value is set for announce mode, but the announce `To` input does not set `aria-invalid`/`aria-describedby` or render `renderFieldError`; only the webhook branch renders that error. Invalid Telegram input can therefore disable save with no visible field message.
  - confidence: 0.9
- **[P1] Regenerate locale output without duplicate keys:** `ui/src/i18n/locales/ar.ts:377`
  - body: The locale edits add `cwd` to generated non-English objects that already contain `cwd`, and the UI guide says those bundles and `.i18n` metadata should come from the i18n pipeline. This creates generated-output drift and likely check failures instead of a narrow cron string update.
  - confidence: 0.82
