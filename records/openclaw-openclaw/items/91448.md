Full review comments:

- **[P1] Accept Telegram topic targets:** `ui/src/ui/controllers/cron.ts:173-174`
  - body: This regex only accepts signed digits, so the dashboard would reject documented/runtime targets like `-1001234567890:topic:42` even though Telegram delivery parses them with a `messageThreadId`. Users editing valid topic-routed cron jobs would be blocked before save.
  - confidence: 0.94
- **[P2] Render announce target errors:** `ui/src/ui/controllers/cron.ts:174`
  - body: The new `errors.deliveryTo` value can be set for Telegram announce delivery, but the view still renders `deliveryTo` errors only for webhook mode. Invalid Telegram input can therefore disable saving without showing the user the field-level reason.
  - confidence: 0.9
- **[P2] Regenerate locale output without unrelated regressions:** `ui/src/i18n/locales/es.ts:988`
  - body: This generated locale change turns the existing Spanish `agent` label from `Agente` to `Agent`, and similar non-English bundle churn is unrelated to Telegram target validation. Regenerate from the English source or narrow the locale output so non-English UI text does not regress.
  - confidence: 0.82
