## Summary

The linked cron Telegram target bug remains real on current main, but this PR is not merge-ready because its UI validator is stricter than the runtime Telegram target grammar and the new announce-field error is not visibly rendered.

The PR adds Telegram-specific cron announce To validation in the Control UI, adds a new cron validation string across locale bundles, and changes some cron Command/CWD labels to use i18n strings.

## Best Possible Solution

Validate cron Telegram announce targets against the same runtime-supported target grammar, render announce To errors visibly, keep generated i18n output clean, and add focused proof that invalid names are rejected while valid topic targets still save.
