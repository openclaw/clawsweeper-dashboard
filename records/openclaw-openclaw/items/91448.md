Full review comments:

- **[P1] Accept documented Telegram topic targets:** `ui/src/ui/controllers/cron.ts:171-174`
  - body: The new Telegram validation rejects any announce target that is not a bare signed integer, but current cron docs and runtime tests accept topic-qualified forms such as `-1001234567890:topic:123` and the `-1001234567890:123` shorthand. Existing users with topic cron targets would be unable to save those jobs from the dashboard, so the validator needs to match the runtime target grammar instead of narrowing it to `/^-?\d+$/`.
  - confidence: 0.94
- **[P2] Render the announce To error:** `ui/src/ui/controllers/cron.ts:174`
  - body: This adds a `deliveryTo` validation error for announce mode, but the announce To input does not render `aria-invalid`, `aria-describedby`, or `renderFieldError`; only webhook mode does. Invalid Telegram input can therefore disable Add/Save without showing the new message beside the field.
  - confidence: 0.9
