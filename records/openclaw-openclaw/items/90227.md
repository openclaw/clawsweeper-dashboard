## Summary

Keep this PR open: the Windows-test direction is useful, but the patch still leaves the symlink test gated by platform instead of the capability probe, so it can fail in non-Windows environments where file symlinks are unavailable. Real Windows or equivalent after-fix proof is also missing.

## What This Changes

The PR changes `extensions/zalouser/src/zalo-js.credentials.test.ts` to run the Zalo credential permission test on Windows with file/directory existence assertions and to replace one Windows skip with a symlink capability probe.

## Best Possible Solution

Keep the PR open and repair the test so the symlink case is skipped whenever the capability probe fails, while preserving POSIX permission assertions and Windows existence/type coverage.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path for the PR-introduced test bug: when `canCreateFileSymlinks` is false on non-Windows, the skip condition remains false and the test still calls `symlink(...)`. I did not run tests because this review is read-only.

## Solution Assessment

No. The Windows permission-test split is a plausible direction, but the symlink test should use the capability probe directly instead of mixing platform and capability checks.
