## Summary

Keep open: this is a focused bug-fix PR for a shipped/current-main regression where non-matching heartbeat transcript prompts drop exec-event runtime context before the model turn. The patch restores the prior helper fallback, adds targeted coverage, and the PR body includes plausible live gateway log proof for the changed runtime path.

## What This Changes

The branch updates `resolveRuntimeContextPromptParts` and its unit test so non-matching `effectivePrompt` text becomes hidden `runtimeContext` when a `transcriptPrompt` is supplied.

## Best Possible Solution

Land the focused helper fallback with the regression test once normal CI and maintainer review confirm the runtime-context contract.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path on current main: call `resolveRuntimeContextPromptParts` with exec-event text as `effectivePrompt` and a heartbeat placeholder as `transcriptPrompt`; the current helper returns no runtime context because the fallback only runs for empty prompts.

## Solution Assessment

Yes. Restoring the fallback in the shared prompt-splitting helper is narrower and more maintainable than special-casing the heartbeat or exec caller, and the new test protects the exact non-matching transcript/effective prompt shape.
