## What This Changes

The issue requests that QMD-backed memory_search stop timing out at the hardcoded 15-second tool deadline when memory.qmd.limits.timeoutMs is configured for slower qmd query runs.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/95347 as the canonical QMD whole-search timeout fix; close this issue only after the accepted fix reaches main.

## Reproduction Assessment

Yes, at source level: current main wraps every memory_search call in a fixed 15-second deadline while QMD search uses memory.qmd.limits.timeoutMs separately. The reporter's 46.7-second qmd query trace exceeds that outer wrapper path.
