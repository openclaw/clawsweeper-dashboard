## Label Justifications

- `P2`: The bundled memory tool defeats documented QMD timeout configuration and reliably blocks slower local query-mode searches, with a backend-specific blast radius.
- `impact:other`: Configured memory retrieval becomes unavailable because of a conflicting tool deadline, outside the narrower owned impact categories.

## Summary

The fixed 15-second deadline was introduced on current `main` in the bundled memory plugin and conflicts with the documented QMD timeout control; the issue remains necessary, while the adjacent open cancellation PR addresses orphaned work rather than the configured deadline.

## What This Changes

Make the bundled `memory-core` `memory_search` tool deadline accommodate slow QMD query-mode searches instead of always cutting them off after 15 seconds.

## Best Possible Solution

Align the tool deadline with the resolved QMD backend timeout through the typed memory contract, retain a finite safe bound, and preserve cancellation propagation from the adjacent timeout work without adding another public config option.

## Reproduction Assessment

Yes. Current source and tests prove the 15-second cutoff, while the reporter's measured 46.7-second reranking necessarily exceeds it despite a 60000-millisecond backend setting.

## Solution Assessment

Yes in direction, but the best fix is deadline composition plus cancellation rather than a new setting or a simple unconditional constant increase.
