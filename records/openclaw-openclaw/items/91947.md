## Summary

Keep this issue open: current `main` adds a tool-wide 15-second deadline that overrides the documented QMD timeout setting, making slow but healthy query-mode searches fail despite a higher configured budget.

## What This Changes

Make QMD-backed `memory_search` honor the existing configured search timeout instead of failing at a fixed tool-wide 15-second deadline.

## Best Possible Solution

Keep a finite plugin-owned safety deadline, but derive the QMD path from the existing resolved QMD timeout with bounded setup overhead and focused coverage for slow successful searches and never-settling operations.

## Reproduction Assessment

Yes at source level with high confidence: configure QMD query mode with `memory.qmd.limits.timeoutMs` above 15 seconds and run a query completing after 15 seconds; the outer tool race returns first. The supplied 46.7-second reranking trace matches that path, though this review did not run a live QMD model.

## Solution Assessment

No to adding another timeout setting; the best fix is to honor the existing QMD timeout contract while retaining a finite wrapper for setup and hung-provider protection.
