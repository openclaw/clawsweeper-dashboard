## Summary

Keep this issue open: current main and the latest release still use a fixed 15s `memory_search` wrapper deadline, while an open linked PR already owns the intended fix path.

## What This Changes

The issue asks for `memory_search` to stop timing out QMD query-mode searches at a hardcoded 15s when `memory.qmd.limits.timeoutMs` is configured higher.

## Best Possible Solution

Review and land one canonical PR that derives the QMD `memory_search` wrapper budget from the resolved QMD timeout with bounded overhead while keeping the default 15s guard for non-QMD and default configurations.

## Reproduction Assessment

Yes, at source level: set QMD query mode with `memory.qmd.limits.timeoutMs` above 15s and run a search that completes after 15s; the current outer tool race still returns `memory_search timed out after 15s` first.

## Solution Assessment

Partly: adding a new timeout knob is not the best first fix, but reusing the existing resolved QMD timeout with bounded wrapper overhead is the narrow maintainable solution.
