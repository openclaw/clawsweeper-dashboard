## Summary

Keep this issue open: current `main` still wraps `memory_search` in a fixed 15-second outer deadline, while the existing QMD search timeout is only honored inside the QMD manager. The open linked PR is the current implementation candidate, so the issue should remain open until that fix or a replacement lands.

## What This Changes

The issue asks for `memory_search` to stop timing out slow QMD query-mode searches at a hardcoded 15 seconds and to honor the configured QMD timeout budget.

## Best Possible Solution

Land or replace a narrow memory-core fix that derives the outer QMD `memory_search` deadline from the resolved `memory.qmd.limits.timeoutMs` with bounded overhead, while preserving the default 15s guard for non-QMD and default paths.

## Reproduction Assessment

Yes, at source level: configure QMD with `memory.qmd.limits.timeoutMs` above 15s and run a search that completes after the fixed outer 15s wrapper. I did not run a live QMD reranker locally, but current code and the reporter's 46.7s trace match the failure path.

## Solution Assessment

No, closing is not the best path because the configured-timeout behavior remains missing on current main. The best implementation should reuse the existing QMD timeout contract rather than adding a second timeout knob for the same search budget.
