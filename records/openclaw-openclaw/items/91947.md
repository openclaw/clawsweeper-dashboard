## Summary

Keep this issue open: current main and v2026.6.8 still wrap `memory_search` in a fixed 15-second deadline, while QMD timeout config is documented separately and https://github.com/openclaw/openclaw/pull/95347 is the open candidate fix path.

## What This Changes

The issue asks OpenClaw to make the `memory_search` outer timeout configurable or reuse `memory.qmd.limits.timeoutMs` so slow QMD query-mode searches can finish.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/95347 as the canonical QMD whole-search timeout fix, then close this issue only after the accepted fix reaches main.

## Reproduction Assessment

Yes, at source level: current main wraps every `memory_search` call in a fixed 15-second deadline while QMD search uses `memory.qmd.limits.timeoutMs` separately. The reporter's 46.7-second QMD query trace exceeds that outer wrapper path.

## Solution Assessment

Yes for keeping this issue open: current main is not fixed, and the best current path is maintainer review of the open candidate PR rather than closing this issue or queueing another duplicate fix.
