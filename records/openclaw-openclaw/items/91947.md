## Summary

Keep this issue open: current main and the latest release still have a fixed 15-second `memory_search` wrapper deadline, while QMD search itself honors `memory.qmd.limits.timeoutMs`. The remaining work is already paired with an open closing PR, so the issue should stay open until that PR is landed, replaced, or closed.

## What This Changes

The issue asks for `memory_search` to stop enforcing an unconfigurable 15-second outer timeout when QMD query mode needs a higher configured timeout.

## Best Possible Solution

Land or replace the linked fix with a memory-core change that derives the QMD `memory_search` outer deadline from the existing QMD timeout while retaining a bounded wrapper for non-QMD and hung operations.

## Reproduction Assessment

Yes at source level: configure QMD query mode with `memory.qmd.limits.timeoutMs` above 15 seconds and the current outer tool wrapper can still time out at 15 seconds before QMD's configured search budget finishes. I did not run a live slow QMD reranker in this read-only review.

## Solution Assessment

Yes for reusing the existing QMD timeout contract; no for adding a separate timeout knob as the first fix. The narrowest maintainable solution is to derive the QMD path's outer deadline from the resolved QMD timeout with bounded overhead.
