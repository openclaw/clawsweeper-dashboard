## Summary

Keep this issue open: current main and v2026.6.8 still wrap `memory_search` in a fixed 15-second outer deadline, while https://github.com/openclaw/openclaw/pull/95347 is the open candidate fix and has not merged. ## What This Changes

QMD-backed `memory_search` should not hit a hardcoded 15-second tool timeout when the existing `memory.qmd.limits.timeoutMs` setting is raised for slower query-mode searches.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/95347 as the canonical configured-timeout fix, while tracking timeout classification and zero-hit sync policy through their narrower related items if they remain distinct.
