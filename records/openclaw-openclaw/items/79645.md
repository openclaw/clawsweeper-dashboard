## What This Changes

The branch replaces full-message transcript redaction at the append sink with a config-aware helper, threads OpenClawConfig through transcript write/update paths, adds redaction coverage, and updates the changelog.

## Best Possible Solution

Keep the append-layer redaction config-aware, but preserve current recursive structured coverage by reusing/generalizing the redactSecrets traversal with passed options or by recursively redacting all safe message/content fields before replacing the sink.

## Reproduction Assessment

Yes. A gateway-injected transcript message with a custom content block such as source.url or data containing a secret-shaped token is redacted by current main's recursive append sanitizer but would be returned unchanged by the PR helper unless it is one of the selected text/tool fields.

## Solution Assessment

No, not as-is. Centralizing config-aware transcript redaction is the right direction, but the implementation must preserve current main's recursive coverage before it is safer than the existing append sink.
