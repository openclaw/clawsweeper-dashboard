## Summary

Keep open: this is the active repair PR after the clean replacement was closed, and current main still sends unredacted inline transcript-update payloads, but the PR head leaves a source-reproducible redaction bypass for non-role append payloads and is paused for human review.

## What This Changes

The branch adds shared transcript redaction, returns sanitized appended messages to gateway and Telegram transcript-update emitters, widens append config typing, and updates tests, changelog, and the SDK API baseline.

## Best Possible Solution

Land a maintainer-reviewed append-layer fix that emits sanitized transcript updates while preserving redaction for every payload shape accepted by the exported append helper.

## Reproduction Assessment

Yes, by source inspection: call the PR-head exported append helper with a raw string or no-role object containing an `sk-` token, and the fallback at `src/config/sessions/transcript-append.ts:297` writes it unchanged. Current main redacts that same payload shape with `redactSecrets(params.message)`.

## Solution Assessment

No, not yet. Returning the sanitized append result to gateway and Telegram emitters is the right direction, but the append sink should apply redaction to all accepted payload shapes or narrow the public contract so unsupported shapes cannot reach it.

## Review Findings
