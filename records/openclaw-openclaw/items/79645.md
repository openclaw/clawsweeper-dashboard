## Summary

Keep open: the latest head is still a security-sensitive transcript persistence change, and the branch narrows current main's recursive redaction coverage for arbitrary gateway/custom content-block fields. A member also stopped ClawSweeper automation, so this should stay in human review until the branch is repaired and re-reviewed.

## What This Changes

This PR replaces the shared transcript append sink's unconditional redaction call with a config-aware transcript redaction helper, threads OpenClawConfig through transcript and gateway append paths, returns/emits redacted messages, adds tests, and updates the changelog.

## Best Possible Solution

Keep the centralized append-layer fix, but make it config-aware without losing full recursive message redaction, preferably by extending the existing structured redaction traversal to accept options and reusing it at the append sink.

## Reproduction Assessment

Yes. Source inspection shows a gateway-injected content block with a secret in an unrecognized field such as source.url or data is redacted by current main's full-message recursive sanitizer but would bypass the PR helper's selected-field traversal.

## Solution Assessment

No, not as-is. Centralizing config-aware transcript redaction is the right direction, but replacing current main's full-message sanitizer with a partial content-block helper is not the safest maintainable fix.
