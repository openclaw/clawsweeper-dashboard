## Summary

Keep open: the latest head still regresses current main's recursive transcript sanitizer for non-text/custom content-block fields, and a maintainer has paused ClawSweeper automation for human review.

## What This Changes

The PR replaces whole-message transcript redaction at appendSessionTranscriptMessage with a shared config-aware transcript redactor, threads OpenClawConfig through transcript write paths, updates events/tests, and adds a changelog entry.

## Best Possible Solution

Keep the central append-layer redaction, but implement it as a config-aware recursive sanitizer that preserves current main coverage for every plain message/content field before another human review.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a gateway-injected assistant content block with a secret-bearing url/source/data field is redacted by current main's whole-message redactSecrets path but bypasses the PR helper unless it is one of the selected text or tool payload fields.

## Solution Assessment

No, not as-is. Centralizing config-aware transcript redaction is the right direction, but the helper must recurse through all redactable message/content fields or delegate to a config-aware variant of the existing recursive sanitizer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve recursive content-block redaction:** `src/agents/transcript-redact.ts:67-105`
