This PR centralizes session transcript redaction in `appendSessionTranscriptMessage`, shares transcript redaction with guarded writes, threads `OpenClawConfig` through transcript append wrappers, and updates tests, API baseline hashes, and changelog.

## Best Possible Solution

Land the append-boundary redaction after extending it to every structured tool-call payload shape OpenClaw already persists and revalidating the touched transcript tests.

## Reproduction Assessment

Yes. Source inspection on current main shows the direct append path writes `params.message` to JSONL, and PR-head source inspection shows a remaining leak path for supported `toolUse`/`functionCall` payloads.

## Solution Assessment

Not yet. Redacting once at `appendSessionTranscriptMessage` is the narrow maintainable fix, but the helper needs to cover all existing structured tool-call variants before automerge.
