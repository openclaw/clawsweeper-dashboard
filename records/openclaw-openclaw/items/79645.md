## Summary

Keep open: the PR addresses a real transcript-redaction leak on current main, but the live branch is dirty and the patch still narrows append-layer redaction for non-role payloads exported through the SDK.

## What This Changes

The PR centralizes session transcript redaction in `appendSessionTranscriptMessage`, reuses the returned redacted message for gateway and Telegram transcript updates, expands redaction tests, refreshes the SDK API hash, and adds a changelog entry.

## Best Possible Solution

Land a rebased append-layer redaction fix that preserves current broad redaction for every appended payload shape while returning the sanitized message to inline transcript emitters.

## Reproduction Assessment

Yes, by source inspection: current main redacts the persisted JSONL entry but Telegram and gateway inline transcript updates still emit the original payload. The related source PR also supplied live JSONL proof for the patched on-disk behavior.

## Solution Assessment

No, not yet. Centralizing redaction at the append sink is the right direction, but this patch should preserve broad redaction semantics for non-role payloads and be rebased with exact-head validation before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve redaction for all append payloads:** `src/config/sessions/transcript-append.ts:297-300`
  - body: Current main redacts `params.message` with `redactSecrets` regardless of shape, but this branch only redacts values that look like an `AgentMessage` with a string `role`. Since `appendSessionTranscriptMessage` remains exported through the plugin SDK and still accepts a generic message value, raw strings or structured no-role payloads containing keys or tokens would now be written unredacted. Please keep the append sink redacting unknown payloads, or narrow the API contract so unsupported shapes cannot reach this path.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The patch improves the main transcript-redaction path but introduces a concrete redaction bypass for non-role payloads passed to the exported append helper.

Concerns:

- **[medium] Non-role append payloads bypass transcript redaction:** `src/config/sessions/transcript-append.ts:297`
  - body: The changed append sink skips redaction unless the payload has a string `role`, whereas current main redacts the whole message value before writing it. This can leak secrets from SDK or internal callers that pass raw strings or structured payloads without `role`.
  - confidence: 0.86
