## Summary

Keep open: the append-boundary redaction direction is right, but the current PR head still leaves supported `toolUse`/`functionCall` payloads and `input` fields outside the new redaction path, so this is not safe to merge yet.

## What This Changes

This PR moves configurable transcript redaction into `appendSessionTranscriptMessage`, reuses it from guarded writes, returns/emits the redacted message, and updates transcript tests, API baseline hashes, and changelog.

## Best Possible Solution

Keep the append-boundary design, but cover every supported assistant tool-call block type and both `input` and `arguments` payload keys before landing.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: append an assistant `toolUse` or `functionCall` block with a secret in `input`, and the PR helper will not recurse into that payload before JSONL persistence.

## Solution Assessment

No, not yet. Redacting once at `appendSessionTranscriptMessage` is the narrow maintainable solution, but this implementation must cover all existing tool-call payload shapes before merge.
