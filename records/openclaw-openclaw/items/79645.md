## Summary

This PR should stay open for the normal automerge path: current main still writes transcript messages without the documented redaction gate, while the PR centralizes redaction at the append boundary and I found no blocking correctness or security finding.

## What This Changes

The PR extracts transcript redaction into a shared helper, applies it in `appendSessionTranscriptMessage`, threads `OpenClawConfig` through transcript emitters, and updates tests, SDK metadata, and the changelog.

## Best Possible Solution

Merge the centralized append-boundary redaction after exact-head gates finish green, closing the focused transcript-persistence issue while leaving broader sensitive-data work at https://github.com/openclaw/openclaw/issues/64046 open.

## Reproduction Assessment

Yes, source-reproducible: current main documents transcript redaction but `appendSessionTranscriptMessage` writes `params.message` directly. A direct append containing an `sk-` token would persist raw JSONL on current main.

## Solution Assessment

Yes. Centralizing the existing redaction behavior in `appendSessionTranscriptMessage` is the narrowest maintainable fix because all direct transcript append paths share that sink, and returning the redacted message keeps event emitters consistent with disk.
