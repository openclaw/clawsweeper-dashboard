## What This Changes

This PR replaces the current full-message transcript sanitizer with a config-aware transcript redaction helper, threads OpenClawConfig through transcript write/update paths, adds redaction tests, and updates the changelog.

## Best Possible Solution

Keep the append-layer redaction config-aware, but preserve current recursive structured coverage by reusing or generalizing the existing full-message redaction traversal and adding a regression case for custom gateway content-block metadata.

## Reproduction Assessment

Yes, source-reproducible. A gateway-injected assistant content block can include arbitrary `Record<string, unknown>` fields, and the PR helper does not traverse those fields while current main's whole-message sanitizer does.

## Solution Assessment

No, not as-is. Centralizing config-aware transcript redaction is the right direction, but replacing full-message recursive sanitization with selected-field handling is not the narrow safest fix.
