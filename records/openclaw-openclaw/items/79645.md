## What This Changes

This PR centralizes session transcript redaction at `appendSessionTranscriptMessage`, shares the helper with guarded tool-result writes, threads `OpenClawConfig` through transcript append wrappers, and updates tests, API baseline hashes, and the changelog.

## Best Possible Solution

Land the append-boundary redaction after it covers every supported assistant tool-call block shape and replay payload key that OpenClaw already persists.

## Reproduction Assessment

Yes. Source inspection shows current main writes raw direct-append messages, and PR-head inspection shows supported `toolUse`/`functionCall` or `input` payloads still bypass structured redaction.

## Solution Assessment

Not yet. Redacting once at `appendSessionTranscriptMessage` is the narrow maintainable fix, but the helper must cover all existing structured tool-call variants before merge.
