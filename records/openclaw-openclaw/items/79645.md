## Summary

Keep open. This PR targets a real transcript redaction gap, but the current head is not merge-ready because custom redact patterns can disable built-in secret patterns, exact-head CI is failing, and ClawSweeper has been paused for human review.

This PR centralizes session transcript redaction in `appendSessionTranscriptMessage`, returns redacted append results to gateway and Telegram transcript emitters, adds recursive redaction coverage, refreshes SDK baselines, and adds a changelog entry.

## Best Possible Solution

Keep the append-layer redaction design, but repair the custom-pattern merge contract and refresh the branch under human review before landing.

## Reproduction Assessment

## Solution Assessment

No, not as the current exact head. The append-layer approach is the right maintainable direction, but the PR must preserve default redaction patterns when custom patterns are configured and clear exact-head checks first.
