## Summary

Keep open for human review. The change is a useful security direction and carries source PR context, but it is not merge-ready because it replaces current main's recursive full-message transcript redaction with a narrower helper that can leave secret-bearing content-block fields unredacted.

## What This Changes

The PR replaces whole-message transcript redaction at `appendSessionTranscriptMessage` with a shared config-aware `redactTranscriptMessage` helper, threads `OpenClawConfig` through transcript write paths, adds redaction tests, and adds a changelog entry.

## Best Possible Solution

Keep the central append-layer redaction and make it config-aware while preserving at least the current recursive structured coverage before another human review.

## Reproduction Assessment

Yes, source-reproducible. A transcript message with a non-text content-block field containing a secret-shaped URL or token is redacted by current main's full-message `redactSecrets` path but would bypass the PR helper unless it is `text`, `thinking`, `partialJson`, or tool-call `arguments`/`input`.

## Solution Assessment

No, not as-is. Centralizing config-aware transcript redaction is the right direction, but the replacement must preserve the existing recursive message-object coverage before it is safer than current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve full content-block redaction coverage:** `src/agents/transcript-redact.ts:67-105`
  - body: `appendSessionTranscriptMessage` currently persists `redactSecrets(params.message)`, which recursively sanitizes every plain-object field in the message. This helper only visits `text`, `thinking`, `partialJson`, and tool-call `arguments`/`input`, so gateway-injected or custom/image content blocks with secret-bearing fields such as a signed URL/token in `data`, `url`, or `source` would be written raw. Keep the config-aware behavior, but make the new helper traverse all redactable content-block fields before replacing the old sink.
  - confidence: 0.86
