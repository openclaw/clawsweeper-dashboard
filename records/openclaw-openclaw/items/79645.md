## Summary

Keep open: this automerge repair PR is useful and has sufficient source proof, but the exact head still leaves supported tool-call `input` payloads outside the new append-boundary redaction and is currently dirty/conflicting, so it needs a narrow repair/rebase before merge.

## What This Changes

The PR carries forward https://github.com/openclaw/openclaw/pull/73563 by moving transcript redaction into `appendSessionTranscriptMessage`, threading `OpenClawConfig` through transcript append paths, updating emitted messages, tests, API baseline hashes, and the changelog.

## Best Possible Solution

Keep the append-boundary redaction approach, but rebase onto current main and make the helper config-aware while preserving the full current structured redaction coverage for every supported tool-call payload shape.

## Reproduction Assessment

Yes, from source inspection. A direct append of an assistant `toolUse` or `functionCall` block with a secret in `input` reaches the PR helper, which only redacts `toolCall.arguments`.

## Solution Assessment

Not yet. Centralizing redaction in `appendSessionTranscriptMessage` is the right boundary, but the current PR helper is narrower than OpenClaw's existing structured tool-call contract and current-main `redactSecrets` coverage.
