## Summary

Keep open. The central transcript-redaction fix is useful, but the current PR head can drop built-in secret patterns when custom redaction patterns are configured, and live PR metadata also shows a conflicting branch with a failing exact-head type check after ClawSweeper was paused for human review.

## What This Changes

This PR centralizes transcript message redaction in `appendSessionTranscriptMessage`, returns the redacted message to gateway and Telegram transcript emitters, adds redaction coverage, refreshes plugin SDK baseline hashes, and adds a changelog entry.

## Best Possible Solution

Keep the append-layer redaction design, merge custom transcript patterns with built-in defaults, add a regression test for custom-plus-default redaction, resolve the stale branch/type-check state, then land under human review.

## Reproduction Assessment

Yes. Source inspection shows current main emits original inline transcript update payloads after appending, and the PR head's new custom-pattern path is source-reproducible because a non-empty custom pattern array replaces the built-in defaults.

## Solution Assessment

No as an exact merge candidate. The append-layer redaction contract is the right narrow direction, but the implementation must preserve default redaction patterns with custom patterns and clear the conflicting/failing exact-head state first.
