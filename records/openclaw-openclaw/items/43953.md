## Summary

Keep open: current main still loses Feishu card-action form/input/select fields, but this PR is not merge-ready because it only handles fields after they are already present on `event.action`, while the live parser still strips them before the handler. The overlapping canonical branch at https://github.com/openclaw/openclaw/pull/42908 covers part of the same parser path, but it does not fully cover this PR's `input_value`/`name` scope and the paired issue at https://github.com/openclaw/openclaw/issues/43202 remains open.

## What This Changes

The PR adds optional Feishu card-action fields and handler tests that serialize form/input/select callback data into a structured synthetic message while preserving plain button command extraction.

## Best Possible Solution

Land one canonical Feishu callback-field fix that preserves the fields at the parser boundary, carries them through decoder/fallback/handler behavior with regression coverage, includes `input_value`/`name` scope intentionally, and has redacted live Feishu proof.

## Reproduction Assessment

Yes, source-level. A raw `card.action.trigger` payload with sibling form/select/input fields is normalized into only `action.value` and `action.tag`, then the fallback path serializes only `action.value`.

## Solution Assessment

No. The direction is useful, but this handler-centered patch is not the best current-main fix because live callback parsing must preserve the sibling fields before the handler can use them.
