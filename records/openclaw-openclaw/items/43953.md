## Summary

Keep open: current main still drops Feishu card-action sibling fields, and this PR is a plausible but insufficient implementation because it only changes the handler/test layer while the live parser strips the fields first. The remaining work overlaps https://github.com/openclaw/openclaw/pull/42908 and https://github.com/openclaw/openclaw/issues/43202, so maintainers should choose one canonical payload contract rather than cleanup-close this branch.

## What This Changes

The PR adds Feishu card-action form/input/select fields to the event type and builds a structured JSON synthetic message for those fields while keeping plain button command extraction unchanged.

## Best Possible Solution

Select one canonical Feishu callback payload contract, preserve supported sibling fields at the parser boundary through decoder/fallback/handler dispatch, add regression coverage, and require redacted live Feishu proof before landing.

## Reproduction Assessment

Yes, source-level: a raw card.action.trigger payload with form/input/select sibling fields is normalized to only action.value and action.tag, then fallback dispatch serializes only action.value. I did not run a live Feishu replay in this read-only review.

## Solution Assessment

No. The handler-level branch is useful but not sufficient; the durable fix must preserve fields in parseFeishuCardActionEventPayload and align parser, decoder/fallback, handler behavior, and the selected payload contract.
