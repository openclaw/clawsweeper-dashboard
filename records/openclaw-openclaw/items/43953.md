## Summary

Keep open. The Feishu callback field-loss problem is still real on current main, but this PR is not merge-ready: it adds handler-level fields without preserving them at the live `card.action.trigger` parser boundary, changes form/select callbacks into JSON text in a way that can bypass existing command handling, is dirty against current main, and only has mocked/local test proof.

## What This Changes

The PR extends Feishu card-action handling and tests to include `form_value`, `input_value`, `name`, `option`, and `options` in a structured JSON synthetic message while preserving plain button extraction when no form/select fields are present.

## Best Possible Solution

Rebase onto the current Feishu `monitor.account.ts` and `card-interaction.ts` path, preserve raw callback fields before decoding, keep command dispatch compatible or document a maintainer-approved JSON contract, and prove it with a redacted live Feishu callback.
