## Summary

Keep open: the requested Feishu form/select callback support is still needed, but this PR is not merge-ready because it updates the handler path without carrying the fields through the live callback parser and it lacks real Feishu behavior proof.

## What This Changes

## Best Possible Solution

Move field preservation into `parseFeishuCardActionEventPayload` and the card-interaction decoder, preserve command/text dispatch for existing cards, then prove the parser-to-handler path with tests and redacted real Feishu evidence.

## Reproduction Assessment

Yes, from source: live `card.action.trigger` data is parsed in `monitor.account.ts`, and that parser currently returns only `action.value` and `action.tag`, so the requested fields cannot reach the handler.

## Solution Assessment

No. This is a plausible handler-level patch, but the best fix belongs at the parser/decoder contract and must preserve current command extraction semantics.
