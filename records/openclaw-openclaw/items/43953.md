## Summary

Keep open: the Feishu callback bug is valid, but this PR is not merge-ready because the handler-only patch does not preserve fields through the current main parser path and the PR still lacks real Feishu after-fix proof.

## What This Changes

The PR adds Feishu card-action fields to the handler and tests so form/input/select callback data is forwarded as structured JSON while simple button commands remain unchanged.

## Best Possible Solution

Preserve the Feishu callback fields at the raw parser boundary, carry them through decoder/fallback/handler behavior with regression coverage, and land one canonical branch with redacted live Feishu proof.

## Reproduction Assessment

Yes, source-level. Current main normalizes raw `card.action.trigger` data into an action object with only `value` and `tag`, then fallback dispatch serializes only `action.value`, so sibling form/select/input fields are lost before dispatch.

## Solution Assessment

No. The direction is useful, but this handler-centered patch is not the best current-main solution because the parser/decoder/fallback path must preserve callback fields before the handler can use them.
