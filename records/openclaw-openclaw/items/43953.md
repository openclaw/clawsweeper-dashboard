## Summary

Keep open: this PR addresses a real Feishu card-action data-loss path, but it is not merge-ready against current main because the current live callback parser/decoder path still needs to preserve the sibling fields before the handler can use them, and the PR has only mocked/local test proof.

## What This Changes

The PR extends Feishu card-action handling and tests to forward input/form/select fields as structured JSON while preserving simple button extraction.

## Best Possible Solution

Port the fix onto current main by preserving the fields at the Feishu parser boundary and carrying them through decoder/fallback/handler behavior with regression coverage and real Feishu callback proof.

## Reproduction Assessment

Yes, source-level. Current main normalizes raw card.action.trigger data into an action object with only value/tag and then fallback dispatch serializes only action.value, so sibling form/select/input fields are lost before dispatch.

## Solution Assessment

No. The direction is useful, but the PR's handler-centered patch is not the best current-main solution; the parser/decoder/fallback path must preserve callback fields so production callbacks and tests exercise the same route.
