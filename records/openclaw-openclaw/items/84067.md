## Summary

Keep open and do not merge as-is: the structured error-code path is useful, but the patch removes the documented final-output denial fallback that current docs and tests explicitly encode, so blocked cron runs without typed metadata could look successful. The PR also lacks accepted real behavior proof because the proof workflow rejected the body fields.

## What This Changes

The branch removes denial-token parsing from cron payload outcomes, carries structured tool error codes through `lastToolError`, and updates cron/embedded-agent tests around failure classification.

## Best Possible Solution

Settle the cron classifier policy first, then either preserve a narrower documented fallback with false-positive regression coverage or intentionally move to structured-only denials with docs, tests, and upgrade proof showing exec denials reliably emit `failureSignal`.

## Solution Assessment
