## What This Changes

The PR changes cron denial classification to rely on structured embedded tool error metadata, preserves node-host denial metadata through lifecycle errors, updates focused tests, and updates one cron docs page.

## Best Possible Solution

Land the structured-only classifier once the companion CLI docs are aligned and maintainers accept that prose-only denial text is ordinary cron output unless typed metadata marks it fatal.

## Reproduction Assessment

## Solution Assessment

Yes with a docs caveat: moving fatal denial detection to typed tool metadata is the cleaner boundary, but the companion CLI docs should be updated and maintainers should explicitly accept the structured-only compatibility change.
