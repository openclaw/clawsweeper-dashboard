## Security Review

Status: needs_attention

Summary: The issue is security-sensitive because it proposes silent, tool-capable channel lifecycle turns that can persist durable memory without visible channel output.

Concerns:

- **[medium] Bound silent channel-side tool execution:** `src/auto-reply/reply/agent-runner-memory.ts:1069`
  - body: A channel-enabled memory flush would call runEmbeddedPiAgent with silentExpected while retaining tool access, so timeout, cancellation, approval, logging, sandbox writability, and operator-visible failure behavior need to be part of the design.
  - confidence: 0.86
- **[medium] Bind memory writes to the correct runtime policy:** `src/auto-reply/reply/agent-runner-memory.ts:1073`
  - body: The flush run passes sandboxSessionKey from runtimePolicySessionKey while writing memory, so a channel opt-in must prove per-channel or per-tenant session keys cannot cross-write another workspace or policy scope.
  - confidence: 0.78
