## Security Review

Status: needs_attention

Summary: The requested behavior would add silent, tool-capable channel-side agent turns, so the design needs explicit security review before implementation.

Concerns:

- **[medium] Bound silent channel-side tool execution:** `src/auto-reply/reply/agent-runner-memory.ts:1021`
  - body: A channel-enabled memory flush would call the embedded agent with silentExpected and memory-trigger tool access; the design must preserve sandbox writability checks and define timeout, cancellation, logging, approval, and visible failure behavior.
  - confidence: 0.84
- **[medium] Preserve channel and sandbox session scope:** `src/auto-reply/reply/agent-runner-memory.ts:1025`
  - body: The current flush forwards sandboxSessionKey from runtimePolicySessionKey; a channel opt-in must prove that channel-specific sessions cannot write memory under the wrong runtime policy or tenant context.
  - confidence: 0.78

  - body: The existing provider skip is tied to separate CLI compaction and reseed handling; enabling channel-driven CLI-backed runs needs proof that external provider resume state cannot go stale after the flush/compaction lifecycle.
