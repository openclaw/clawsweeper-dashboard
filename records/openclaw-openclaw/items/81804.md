## Summary

Keep open: current main still skips preflight compaction and memory flush for CLI providers, including channel-triggered runs that use a CLI backend, and the requested channel opt-in would add new public config plus silent tool-capable lifecycle policy.

## What This Changes

Request to add opt-in channel support under `agents.defaults.compaction.memoryFlush`, optionally allowlisted by backend, so channel-driven sessions can run a silent memory-curation turn before compaction.

## Best Possible Solution

Keep this as the focused design thread, then implement only after maintainers define a channel-scoped opt-in contract with schema/help/docs, channel eligibility rules, and CLI resume-state safeguards.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main has the described CLI-provider gates and lacks the proposed channel opt-in config.

## Solution Assessment

Unclear for implementation. Reusing the existing memory-flush primitive is plausible, but the maintainable solution needs a design decision for channel eligibility, CLI session-state safety, and silent side-effect policy before code changes.
