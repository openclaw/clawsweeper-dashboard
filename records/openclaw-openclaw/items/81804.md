## Summary

Keep open: current main and v2026.5.18 still have the embedded-only/CLI-skip memoryFlush behavior and no channel opt-in config, while the requested behavior would add new public config plus silent, tool-capable channel lifecycle semantics that need maintainer product and security review.

## What This Changes

The issue requests an opt-in channel-side memoryFlush config path that lets selected channel-driven sessions run a silent memory write turn before compaction.

## Best Possible Solution

Keep this as a focused design thread until maintainers define a channel-scoped opt-in contract with schema/help/docs, channel eligibility, runtime policy binding, sandbox/failure semantics, and targeted runner/config tests.

## Reproduction Assessment

Not applicable as a feature request. Source and release inspection confirm current main and v2026.5.18 lack the requested channel opt-in config and still preserve the CLI provider skip.

## Solution Assessment

Unclear for implementation. Reusing the existing memoryFlush primitive is plausible, but a direct gate change is not enough without maintainer-selected channel eligibility, public config, sandbox, timeout, cancellation, and failure semantics.

## Security Review

Status: needs_attention

Summary: The requested feature is security-sensitive because it would introduce silent, tool-capable agent turns for channel-triggered sessions.

Concerns:

- **[medium] Bound silent channel-side tool execution:** `src/auto-reply/reply/agent-runner-memory.ts:1069`
  - body: A channel-enabled memory flush would call the embedded agent with tool access while suppressing replies, so an implementation must preserve sandbox writability checks and define timeout, cancellation, logging, and operator-visible failure behavior.
  - confidence: 0.86
- **[medium] Bind memory writes to the correct runtime policy:** `src/auto-reply/reply/agent-runner-memory.ts:1073`
  - body: Channel sessions can carry channel-specific session keys and workspace policy; the flush turn must use the runtime policy session key and avoid cross-channel or cross-tenant memory writes.
  - confidence: 0.8
- **[medium] Preserve CLI resume-state safety:** `src/auto-reply/reply/agent-runner-memory.ts:585`
  - body: The current CLI skip is tied to separate CLI compaction and reseed behavior, so a channel opt-in must not accidentally re-enable agentic flushes for provider-owned CLI resume state.
  - confidence: 0.78
