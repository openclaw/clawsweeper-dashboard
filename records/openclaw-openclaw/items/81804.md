## Summary

Keep open: current main already supports memory flush for non-CLI channel runs, but the requested opt-in path for channel sessions that use CLI execution providers is not implemented and needs maintainer design.

## What This Changes

Request an opt-in pre-compaction memoryFlush mode for channel-driven sessions, with an enableForChannels flag and optional per-channel allowlist.

## Best Possible Solution

Design a channel-aware memory-flush policy that distinguishes chat channel identity from execution-provider state, then add schema, docs, and regression coverage once maintainers approve the boundary.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main lacks enableForChannels/channelBackends and still skips memory flush for CLI execution providers regardless of channel.

## Solution Assessment

Unclear. The direction is plausible, but the proposed channel override must be designed around provider runtime state and silent write-capable execution rather than implemented as a narrow bug fix.
