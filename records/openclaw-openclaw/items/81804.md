## Risks / Open Questions

- The public config shape is undecided: a global enableForChannels boolean, a backend allowlist, or a more channel/session-policy-oriented contract would have different upgrade and support costs.
- A silent channel-side agentic turn with tool access needs explicit timeout, cancellation, logging, approval, sandbox, and operator-visible failure behavior before implementation.
- The design must bind memory writes to the correct runtimePolicySessionKey/channel tenancy so a channel flush cannot write into the wrong workspace or memory file.
- Any opt-in path must preserve the CLI resume-state safety that motivated the current provider skip and the separate CLI compaction/reseed lifecycle.
