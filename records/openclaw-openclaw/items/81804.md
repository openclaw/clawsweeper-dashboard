## Summary

Keep open: current main still intentionally excludes CLI-backed channel sessions from preflight compaction and agentic memory flush, while the requested channel opt-in would add new public config and silent lifecycle policy that needs maintainer design.

## What This Changes

Request an opt-in pre-compaction agentic memory flush for channel-driven sessions such as Google Chat, Slack, Discord, and Matrix, with optional backend allowlisting.

## Best Possible Solution

Keep this as the focused design thread and implement only after maintainers define a channel-scoped opt-in contract with schema/help/docs, CLI resume-state safeguards, and focused runner/config tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main has the described CLI-provider gates and lacks the proposed channel opt-in config.

## Solution Assessment

Unclear for the final implementation. Reusing the existing memory-flush primitive is plausible, but the safest solution needs maintainer design for channel eligibility, CLI session-state safety, and silent side-effect policy before code changes.

Status: needs_attention
