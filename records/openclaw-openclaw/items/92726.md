## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Avoid cmd parsing for task names:** `src/infra/windows-task-restart.ts:136`
  - body: This builds a cmd.exe /c command string with the raw task name, dropping the previous quoteCmdScriptArg protection. A custom OPENCLAW_WINDOWS_TASK_NAME or profile containing quotes or cmd metacharacters can make /End fail or parse extra commands; spawn schtasks directly with args or preserve the existing quoting contract.
  - confidence: 0.91
- **[P1] Verify the gateway PID before taskkill:** `src/infra/windows-task-restart.ts:83-89`
  - body: The fallback scans netstat for :18789 and force-kills the matched PID, but OpenClaw supports configured gateway ports and current cleanup verifies gateway argv before signaling. On a custom-port install this can leave the stuck gateway alive, and if another process owns 18789 it can kill the wrong process.
  - confidence: 0.94
- **[P2] Keep task-state detection locale independent:** `src/infra/windows-task-restart.ts:67-69`
  - body: The launcher treats the task as stopped whenever localized schtasks output lacks literal running. Current main already treats Task Scheduler status text as locale-dependent, so non-English Windows hosts can skip the wait and issue /Run while the old task is still running.
  - confidence: 0.87
- **[P2] Delete the detached launcher after it finishes:** `src/infra/windows-task-restart.ts:114`
  - body: Every restart now writes a random openclaw-node-restart-*.mjs file, but the generated script reaches the finished log line and exits without removing itself. The old cmd helper deleted itself, so repeated restarts can accumulate executable temp files.
  - confidence: 0.9
