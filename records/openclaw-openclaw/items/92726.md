## Summary

Keep this PR open because it targets a real Windows gateway restart failure, but the submitted patch is not merge-ready: it fails current type/build checks and adds an unsafe hard-coded taskkill fallback.

The PR replaces the Windows scheduled-task self-restart `.cmd` helper with a detached Node.js handoff subprocess and updates the focused restart tests.

## Best Possible Solution

Keep the Windows restart repair in this owner path, but make it compile, preserve task-name safety, and use configured-port verified gateway cleanup instead of raw `netstat`/`taskkill` before refreshing Windows proof.

Yes, with a native Windows Task Scheduler setup: current source shows the running-state guard before `/Run`, and the PR body provides copied Windows restart logs. I did not run a Windows reproduction in this checkout.

No. Replacing the fragile handoff may be a valid direction, but this implementation should reuse the typed supervisor contract and verified configured-port cleanup instead of adding an inline raw process killer.

Full review comments:

- **[P1] Return a declared restart method:** `src/infra/windows-task-restart.ts:130-136`
  - body: `RestartAttempt.method` is still a closed union of supervisor mechanisms, so returning `"node-handoff"` makes `check-prod-types`, `check-lint`, and the build fail on this head. Keep this as `"schtasks"`, or update the type and all downstream consumers deliberately.
  - confidence: 0.98
- **[P1] Verify the configured gateway PID before taskkill:** `src/infra/windows-task-restart.ts:102-104`
  - body: This fallback kills any listener found on hard-coded `:18789`, but the gateway port is configurable and current helpers verify argv before treating a listener as OpenClaw. On custom-port installs this can miss the stuck gateway and terminate an unrelated process on the default port.
  - confidence: 0.9
- **[P2] Avoid shell-parsing task names through cmd:** `src/infra/windows-task-restart.ts:66`
  - body: The `/End` call interpolates `taskName` into a `cmd.exe /c` string, dropping the previous cmd quoting coverage for profile or override task names containing quotes or metacharacters. Spawn `schtasks` directly with argument array entries, or restore equivalent quoting and tests.
  - confidence: 0.86
- **[P2] Remove dead declarations before the type check:** `src/infra/windows-task-restart.ts:2-16`
  - body: The PR leaves `fs`, `path`, two retry constants, `quotePowerShellSingleQuotedLiteral`, and `isScheduledTaskRunning` unused, and `check-prod-types` fails with TS6133. Remove the stale declarations or wire them into the actual launcher logic.
  - confidence: 0.97
