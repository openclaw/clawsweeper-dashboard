## Summary

Keep open: the PR addresses a real Windows gateway restart failure, but the latest head still introduces merge-blocking process-safety and locale regressions in the active self-restart path.

## What This Changes

The PR replaces the Windows scheduled-task restart cmd helper with a detached Node.js launcher and updates focused tests for that restart handoff.

## Best Possible Solution

Keep the Node handoff only if it reuses the existing configured-port, verified-PID, locale-independent task-state, and helper-cleanup contracts before merge.

## Reproduction Assessment

Source-level yes for the PR-introduced defects: the generated launcher hard-codes ':18789' before taskkill, parses localized schtasks text for 'running', and does not delete the helper it writes. I did not live-reproduce the original scheduler race on Windows in this read-only review.

## Solution Assessment

No: a Node handoff may be a plausible direction, but this implementation bypasses existing configured-port, verified-PID, locale-independent task-state, and helper-cleanup contracts. The safer fix is to reuse those contracts or move the repair into the scheduled-task owner path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Verify the gateway PID before taskkill:** `src/infra/windows-task-restart.ts:83-89`
  - body: The fallback scans netstat for ':18789' and force-kills the matched PID, but OpenClaw supports configured gateway ports and existing cleanup verifies gateway argv before signaling. On a custom-port install this can leave the stuck gateway alive, and if another process owns 18789 it can kill the wrong process.
  - confidence: 0.94
- **[P2] Keep task-state detection locale independent:** `src/infra/windows-task-restart.ts:67-69`
  - body: The launcher treats the task as stopped whenever localized schtasks output lacks literal 'running'. Current main already treats Task Scheduler status text as locale-dependent, so non-English Windows hosts can skip the wait and issue /Run while the old task is still running.
  - confidence: 0.87
- **[P2] Delete the detached launcher after it finishes:** `src/infra/windows-task-restart.ts:114`
  - body: Every restart now writes a random openclaw-node-restart-*.mjs file, but the generated script reaches the finished log line and exits without removing itself. The old cmd helper deleted itself, so repeated restarts can accumulate executable temp files.
  - confidence: 0.9
