## Label Justifications

- `P1`: The PR targets a Windows gateway restart failure that can leave the gateway offline, and the current patch still has merge-blocking restart and process-safety defects.
- `merge-risk: 🚨 compatibility`: The patch can break custom task names, custom profiles, and custom gateway ports that current Windows restart helpers account for.

## Summary

Keep open: the Windows restart problem is real and the branch is focused, but the current implementation is not merge-ready because it introduces unescaped cmd parsing, unsafe force-kill behavior, locale-dependent task-state parsing, and temp helper cleanup gaps.

## What This Changes

This PR replaces the Windows self-restart cmd helper with a detached Node.js launcher that stops, waits for, force-kills, and relaunches the Scheduled Task, with updated unit tests.

## Best Possible Solution

Keep the Node handoff only if it reuses the existing task-name quoting, configured-port, verified-PID, locale-independent task-state, and helper-cleanup contracts before merge.

## Reproduction Assessment

Yes for the PR-introduced defects: source inspection shows the generated launcher hard-codes :18789 before taskkill, parses localized schtasks text for running, and does not delete the helper it writes. I did not live-reproduce the original scheduler race on Windows in this read-only review.

## Solution Assessment

No: a Node handoff may be a plausible direction, but this implementation bypasses existing task-name quoting, configured-port, verified-PID, locale-independent task-state, and helper-cleanup contracts. The safer fix is to reuse those contracts in the scheduled-task owner path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Avoid cmd parsing for task names:** `src/infra/windows-task-restart.ts:136`
  - body: This builds a cmd.exe command string with the raw task name, dropping the previous quoteCmdScriptArg protection. A custom OPENCLAW_WINDOWS_TASK_NAME or profile containing quotes or cmd metacharacters can make /End fail or parse extra commands; spawn schtasks directly with args or preserve the existing quoting contract.
  - confidence: 0.91
- **[P1] Verify the gateway PID before taskkill:** `src/infra/windows-task-restart.ts:83-89`
  - body: The fallback scans netstat for ':18789' and force-kills the matched PID, but OpenClaw supports configured gateway ports and current cleanup verifies gateway argv before signaling. On a custom-port install this can leave the stuck gateway alive, and if another process owns 18789 it can kill the wrong process.
  - confidence: 0.94
- **[P2] Keep task-state detection locale independent:** `src/infra/windows-task-restart.ts:67-69`
  - body: The launcher treats the task as stopped whenever localized schtasks output lacks literal 'running'. Current main already treats Task Scheduler status text as locale-dependent, so non-English Windows hosts can skip the wait and issue /Run while the old task is still running.
  - confidence: 0.87
- **[P2] Delete the detached launcher after it finishes:** `src/infra/windows-task-restart.ts:114`
  - body: Every restart now writes a random openclaw-node-restart-*.mjs file, but the generated script reaches the finished log line and exits without removing itself. The old cmd helper deleted itself, so repeated restarts can accumulate executable temp files.
  - confidence: 0.9
