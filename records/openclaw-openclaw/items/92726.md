## Summary

Keep open: this is a useful Windows restart bug fix attempt, but the patch is not merge-ready because the new handoff hard-codes and force-kills port 18789, regresses locale-independent task-state detection, and leaves unused production code under strict unused checks.

## What This Changes

The PR replaces the Windows scheduled-task restart cmd helper with a detached Node.js handoff process and rewrites the focused tests.

## Best Possible Solution

Keep the Windows restart repair in the scheduler-owned path, reuse configured-port and verified-PID helpers, preserve locale-independent task-state detection, remove unused code, and require live Windows restart/no-popup proof before merge.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source-level yes for the active Windows self-restart path and the PR-introduced defects; I did not live-reproduce the original scheduler race on a Windows host in this read-only review.

## Solution Assessment

Is this the best way to solve the issue? No: a Node handoff may be a plausible direction, but this implementation duplicates and bypasses existing scheduler, configured-port, verified-PID, and locale-independent state helpers; the safer fix is a narrower repair around those contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Resolve the actual gateway port before killing listeners:** `src/infra/windows-task-restart.ts:102-104`
  - body: The fallback scans `netstat` for `:18789` and force-kills that PID, but OpenClaw supports `OPENCLAW_GATEWAY_PORT`/`gateway.port` and existing cleanup resolves the configured port and verifies gateway argv before signaling. On a custom-port Windows install this leaves the stuck gateway alive, and if another process owns 18789 it can be killed instead.
  - confidence: 0.93
- **[P1] Remove unused production helpers before merge:** `src/infra/windows-task-restart.ts:32`
  - body: This patch adds `isScheduledTaskRunning` plus retry constants that are never called, while `tsconfig.core.json` enables `noUnusedLocals` for production sources. The branch will fail the production type/lint gate until those helpers are either used or removed.
  - confidence: 0.91
- **[P2] Keep task-state detection locale independent:** `src/infra/windows-task-restart.ts:92`
  - body: The new handoff treats the task as stopped whenever `schtasks /FO LIST /V` stdout lacks literal `running`. The current helper uses `Get-ScheduledTask(...).State` for this decision; parsing localized `schtasks` text can skip the wait on non-English Windows and issue `/Run` while the old task is still running.
  - confidence: 0.84
