## Summary

Keep open: current main still animates the TUI busy status with elapsed time, a Loader spinner, and waiting shimmer, and there is no quiet recording mode exposed today.

## What This Changes

Add an optional TUI quiet status mode that disables live spinner/shimmer and elapsed status repainting during active requests.

## Best Possible Solution

Add a maintainer-approved TUI quiet status setting that switches busy states to stable text while preserving the animated default and documenting the new surface.

## Reproduction Assessment

Not applicable: this is a feature/config request, not a reported regression. Source inspection gives high confidence that current main still has the animated elapsed/spinner/shimmer behavior the request wants to make optional.

## Solution Assessment

Unclear until maintainers choose the public UX surface. The likely implementation boundary is narrow in the TUI status renderer, but naming and persistence are product/API decisions.
