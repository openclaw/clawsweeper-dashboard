## Summary

Keep open for maintainer review: current main still routes Linux memory directories through chokidar, and this PR is a focused fix with source/dependency proof and targeted tests, but it changes Linux watcher delivery and fallback semantics under a protected maintainer label.

## What This Changes

This PR adds a Linux-specific directory-only fs.watch tree for memory-core directories while keeping file paths on chokidar and expanding watcher configuration tests.

## Best Possible Solution

Land the Linux directory-only watcher after maintainers accept the targeted proof or add Linux gateway soak evidence, while preserving chokidar fallbacks and the existing macOS/Windows native path.

## Reproduction Assessment

Yes, source-reproducible: current main routes Linux memory directories through chokidar, and Node/chokidar source supports the per-entry watcher contract described in the linked issue. I did not run a live gateway repro during this read-only review.

## Solution Assessment

Yes, this appears to be the best bounded fix shape: it avoids Node recursive and chokidar per-file fan-out while keeping chokidar for file paths and degraded fallback. The broader alternative of changing sync.watch defaults remains a separate product decision, not a replacement for this Linux-specific bug fix.
