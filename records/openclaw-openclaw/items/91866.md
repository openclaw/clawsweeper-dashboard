## Summary

Keep open: this PR targets a real Codex cleanup problem, but the patch deletes the active sidecar after approved pending binds, which is the state `runBoundTurn` needs for subsequent bound messages. The linked leak still needs a lifecycle-owned fix and real behavior proof.

## What This Changes

This PR removes the denied-only guard so Codex app-server binding sidecars clear on both approved and denied plugin conversation binding resolutions, with tests for approved cleanup and no-sidecar no-op behavior.

## Best Possible Solution

Preserve approved bind sidecars, keep cleanup on denied/error/detach paths, and fix the yolo leak in the app-server session/disposal or stale-sweep lifecycle with live process/file proof.

## Reproduction Assessment

Yes, from source. A pending Codex bind writes the sidecar, the PR clears it on approved resolution, and the next inbound bound turn reads no `threadId` and throws.

## Solution Assessment

No. The approval callback is the wrong cleanup boundary for successful binds; the safer fix is lifecycle-owned cleanup that preserves valid sidecars and separately reaps stale app-server state.
