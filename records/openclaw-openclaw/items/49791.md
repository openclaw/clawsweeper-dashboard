## Summary

Keep open. Current main still preserves the strict Feishu group command authorization boundary and still returns no user-visible reply for unauthorized directive-only text commands; the reporter’s workaround explains the immediate failure, but the requested visible rejection is not implemented and the related fixing PR is closed unmerged.

## What This Changes

Feishu group `/think` commands should produce a useful authorization rejection when an explicitly mentioned group directive-only command is blocked.

## Best Possible Solution

Keep group command authorization strict, but add a narrow non-sensitive rejection for explicitly addressed Feishu group directive-only commands that are blocked by sender authorization, with docs and regression tests covering both blocked and authorized paths.

## Reproduction Assessment

Yes. Source inspection shows a Feishu group `/think medium` without matching sender command authorization reaches dispatch with CommandAuthorized false, then the directive-only branch returns an undefined reply, matching the logged queuedFinal=false, replies=0 symptom.

## Solution Assessment

Yes. The narrow maintainable solution is a generic visible rejection for explicitly addressed blocked group commands, not opening group command execution or changing allowlist semantics.

## Repair Work Prompt

Fix the Feishu group command UX tracked by https://github.com/openclaw/openclaw/issues/49791. Current main normalizes `@Bot /think medium`, computes `CommandAuthorized: false` when the group sender is not in the Feishu command allowlist, and the directive-only path returns no reply. Add a narrow generic rejection for explicitly addressed Feishu group command/directive-only messages blocked by sender command authorization. Preserve the existing authorization boundary: do not make group commands implicitly open, do not execute directives for unauthorized senders, and do not leak sender IDs, group IDs, or allowlist/config details in the reply. Update or add focused regression coverage for unauthorized group `/think` producing the generic rejection and authorized group `/think` still applying/acknowledging normally. Update Feishu/slash-command docs if the visible text-command behavior changes. The related PR https://github.com/openclaw/openclaw/pull/49821 is closed unmerged and should be treated only as context, not as a branch to revive blindly.
