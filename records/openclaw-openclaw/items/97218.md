The issue reports that `openclaw doctor --fix` moves Slack shared allowlist policy from `channels.slack` into `channels.slack.accounts.default`, breaking named-account authorization after upgrades.

## Best Possible Solution

Keep the current policy-copying migration and let the next stable release carry the already-merged fix forward.
