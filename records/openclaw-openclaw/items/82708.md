## Repair Work Prompt

Fix the Discord heartbeat/background failure classification reported in https://github.com/openclaw/openclaw/issues/82708. On current main, heartbeat runs pass `isHeartbeat: true` into the normal reply path, but generic agent-runner failures can still become the foreground `Something went wrong... /new` payload and then be sent by `runHeartbeatOnce`. Implement a bounded fix in `src/infra/heartbeat-runner.ts` and/or `src/auto-reply/reply/agent-runner-execution.ts` so heartbeat/background failures are suppressed or rewritten as heartbeat-specific diagnostics, without adding a new config option and without changing normal foreground Discord user-request failure behavior. Preserve existing `channels.defaults.heartbeat.showAlerts` semantics, legitimate heartbeat alert delivery, `HEARTBEAT_OK` suppression, response-prefix handling, and message-tool-only heartbeat behavior. Add focused regression coverage for the heartbeat generic-failure path and the agent-runner heartbeat failure mapping; include a single `CHANGELOG.md` fix entry if the runtime behavior changes user-visible copy.
