## Summary

Keep open: current main still has a credible native Pi/Anthropic `before_tool_call` dispatch gap because the pre-exec path consults only the mutable global hook runner, while the documented hook contract is meant to gate tool execution before native tools run.

## What This Changes

The issue reports that plugin `before_tool_call` handlers are skipped for Anthropic/Pi native `exec` on OpenClaw 2026.4.29 while the same plugin's `after_tool_call` handler fires.

## Best Possible Solution

Native tool wrapping should resolve the active gateway or run plugin hook runner for `before_tool_call`, with regression coverage proving an Anthropic/Pi native `exec` call can be observed or blocked before execution.

## Reproduction Assessment

Yes, source-reproducible with high confidence. The supplied plugin repro is concrete, and current main still routes wrapped native tools through `runBeforeToolCallHook()` while that helper consults only the global hook runner for plugin hooks.

## Solution Assessment

Unclear until maintainers confirm the hook-runner ownership contract. The narrow maintainable direction appears to be threading or resolving the active gateway/run hook runner for native tool wrapping rather than relying only on the mutable global singleton.
