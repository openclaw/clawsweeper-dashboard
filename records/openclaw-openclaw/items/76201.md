## Summary

Keep open: current main still has a credible native Anthropic/Pi `before_tool_call` dispatch gap, the reporter re-verified the same behavior on the latest listed release, and I found no merged fix or canonical replacement for this native `exec` path.

## What This Changes

The issue requests that plugin-registered `before_tool_call` handlers fire before native `exec` and other native tool calls in the Anthropic/Pi harness.

## Best Possible Solution

Native tool wrapping should resolve the active run or gateway plugin hook runner for `before_tool_call`, then add regression coverage proving a plugin can observe or block a gateway-bound Anthropic/Pi native `exec` before execution.

## Reproduction Assessment

Yes, with high-confidence source reproduction rather than a live current-main run. The supplied minimal plugin repro is concrete, the latest-release recheck confirms the shipped behavior, and current main still routes wrapped native tools through a global-only before-hook runner while the after path can use `ctx.hookRunner`.

## Solution Assessment

Unclear until the hook-runner ownership contract is confirmed. The narrow maintainable direction is to use the active run/gateway hook runner for native before-tool dispatch, rather than adding another scattered cache or relying only on the mutable global singleton.
