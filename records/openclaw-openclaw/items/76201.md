## Summary

Keep open: the report is still credible against current main because the native Pi tool path still reaches `before_tool_call` through the global hook runner only, while the observed `after_tool_call` path can use the embedded run hook runner.

## What This Changes

Ensure plugin `before_tool_call` handlers fire before native Anthropic/Pi `exec` tool execution instead of being skipped while `after_tool_call` works.

## Best Possible Solution

Native Pi tool execution should resolve the same active plugin hook runner for `before_tool_call` that the embedded subscription path uses for `after_tool_call`, with regression coverage for gateway-bindable Anthropic/Pi `exec`.

## Reproduction Assessment

Yes. The issue gives a concrete minimal plugin and native `exec` turn on `v2026.4.29`, and current main still shows a source-level before/after hook-runner asymmetry on the native Pi path.

## Solution Assessment

Unclear. The narrow maintainable direction appears to be threading active run hook context into native tool wrapping, but maintainers should first confirm the intended global versus run-scoped hook runner ownership contract.
