## Summary

Keep open: current main still has credible source evidence for the native Pi embedded `before_tool_call` dispatch gap, and no shipped/current-main fix was found for the native `exec` path described in the report.

## What This Changes

Ensure plugin-registered `before_tool_call` handlers fire before native OpenClaw tool execution in the Anthropic/Pi embedded harness.

## Best Possible Solution

Native tool definitions should resolve the same active plugin registry and run context for `before_tool_call` that the embedded subscription path uses for `after_tool_call`, with regression coverage for Anthropic/Pi native `exec`.

## Reproduction Assessment

Yes. The issue gives a concrete minimal plugin and native `exec` turn on v2026.4.29, and current main still shows the native adapter calling `runBeforeToolCallHook()` without run context while that hook reads only the global runner.

## Solution Assessment

Unclear until maintainers confirm the intended hook-runner ownership contract. The narrow maintainable direction appears to be threading active run hook context into native tool wrapping rather than relying only on the mutable global singleton.
