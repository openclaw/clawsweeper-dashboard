## Summary

Keep open: current main still has a credible native Pi/Anthropic `before_tool_call` dispatch gap, and the related loopback MCP PR did not cover this native `exec` path.

## What This Changes

The issue reports that external plugin `before_tool_call` handlers are skipped for native `exec` tool execution through the Anthropic/Pi harness, while `after_tool_call` fires.

## Best Possible Solution

Native tool wrapping should resolve the active gateway/run hook runner for `before_tool_call`, with regression coverage for an embedded Pi/Anthropic native `exec` call where a plugin observes or blocks before execution.

## Reproduction Assessment

Yes, source-reproducible with high confidence, though I did not live-run the gateway. The supplied minimal plugin repro is concrete, and current main still routes wrapped native tools through a global-only before-hook lookup while the after path can use a captured run hook runner.

## Solution Assessment

Unclear because there is no proposed patch and the hook-runner ownership contract needs maintainer confirmation. The narrow maintainable direction is to thread or preserve the active gateway/run hook runner for native tool wrapping rather than relying only on a mutable global singleton.
