## Summary

Keep open: current main still has a plausible before/after hook asymmetry for embedded native tools, and the report includes a concrete live repro against v2026.4.29.

## What This Changes

Ensure plugins' `before_tool_call` handlers fire before native exec tool execution in the Anthropic embedded harness.

## Best Possible Solution

Use one request- or registry-scoped hook runner for both pre- and post-tool hooks in embedded native tool execution, then add a regression that proves `before_tool_call` can observe, modify, and block `exec` before it runs.

## Reproduction Assessment

Yes. The issue provides detailed v2026.4.29 live steps and logs showing plugin registration plus `after_tool_call` firing while `before_tool_call` stays silent; current source has the same asymmetric runner lookup shape.

## Solution Assessment

Unclear because no patch is proposed. The maintainable fix should avoid relying only on mutable global runner state at native tool execution time and should preserve existing wrapped-tool de-duplication.
