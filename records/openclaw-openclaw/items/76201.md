## Summary

Keep open: current main still has a credible native Pi embedded `before_tool_call` dispatch gap, with the same before/after hook-runner asymmetry described in the report and no source evidence of a fix on the native `exec` path.

## What This Changes

Ensure plugins registered for `before_tool_call` fire before native `exec` and other OpenClaw-owned tools in the Anthropic/Pi embedded harness.

## Best Possible Solution

Native tool definitions should resolve the same active plugin registry for `before_tool_call` that the embedded subscription path uses for `after_tool_call`, with regression coverage for gateway-bindable Anthropic/Pi `exec`.

## Reproduction Assessment

Yes. The issue gives a concrete minimal plugin and native `exec` turn on `v2026.4.29`, and current main still shows the native adapter calling `runBeforeToolCallHook` without run context while that hook reads only the global runner.

## Solution Assessment

Unclear until the runner ownership contract is confirmed. The narrow maintainable direction appears to be threading active run hook context into native tool wrapping rather than relying only on the mutable global singleton.
