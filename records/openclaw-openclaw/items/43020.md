## Summary

Keep open. Current main still has the Mattermost draft-preview path that can reuse one editable post across assistant/tool boundaries while core text block delivery is disabled, and the open related PR is the right place to finish the fix before this issue closes.

## What This Changes

Fix Mattermost edit-in-place streaming so each assistant text segment in a multi-tool-call response is preserved in its own streamed post instead of being overwritten and delivered only at the end.

## Best Possible Solution

Review and land or replace the open Mattermost-local PR with a race-safe `forceNewMessage()` boundary fix plus focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current Mattermost preview streaming updates one draft post, assistant/reasoning/tool boundaries do not rotate it, and core text block delivery is disabled on that path.

## Solution Assessment

Yes. The narrow maintainable solution is Mattermost-local draft-boundary rotation using the existing `forceNewMessage()` seam, with serialization and tests, rather than changing core block-streaming semantics.
