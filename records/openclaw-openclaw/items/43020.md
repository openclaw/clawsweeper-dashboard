## Summary

Keep open. Current main still has a source-level Mattermost draft-preview path that updates the same post across assistant/tool boundaries while core block delivery is disabled; the open #70508 PR appears to target this exact remaining bug.

## What This Changes

Mattermost preview streaming should preserve each assistant text segment across tool-call boundaries instead of overwriting one draft post and batching visible messages at the end.

## Best Possible Solution

Review and land #70508 or an equivalent Mattermost-local fix that uses the existing draft-stream boundary seam, adds focused regression coverage, and updates docs if the visible preview model changes.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: with Mattermost preview streaming active, a partial reply updates the draft post, assistant/reasoning/tool boundaries do not force a new post, and core block delivery is disabled. I did not run a live Mattermost instance.

## Solution Assessment

Yes, the narrowest maintainable fix is in the Mattermost plugin: reuse `draftStream.forceNewMessage()` at draft boundaries rather than changing core block-streaming semantics. The open #70508 direction matches that approach and should be reviewed with tests/docs alignment.
