## Summary

Keep open. Current main still matches the reported Mattermost draft-preview bug: partial replies update one draft post, assistant/reasoning/tool boundaries do not seal it, and text-only block replies are suppressed by `disableBlockStreaming`; https://github.com/openclaw/openclaw/pull/70508 remains the active fix path.

## What This Changes

Mattermost preview streaming should preserve multi-turn assistant/tool output instead of overwriting one draft post and batching completed text at the end.

## Best Possible Solution

The desired end state is a Mattermost-local boundary handler that preserves streamed assistant/tool segments without changing core block-streaming semantics, with focused regression coverage and docs updated if the visible preview model changes.

## Reproduction Assessment

Yes, there is a high-confidence source-level reproduction path: enable Mattermost preview streaming, send a multi-turn/tool-call response, and the current callbacks keep updating the same draft stream while core text block delivery is disabled. I did not run a live Mattermost server.

## Solution Assessment

Yes, the narrowest maintainable solution is in the Mattermost plugin: seal or split the draft stream at assistant/reasoning/tool boundaries using the existing `forceNewMessage()` seam or an equivalent append-preserving approach. https://github.com/openclaw/openclaw/pull/70508 follows that direction but needs review and conflict handling.
