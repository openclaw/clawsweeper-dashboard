## Summary

Keep open: current main still rebuilds dirty session JSONL entries and clears/reinserts indexed rows, while the active incremental-sync PR is open, conflicting, and still carrying merge-review blockers.

## What This Changes

The issue asks OpenClaw memory session sync to avoid full delete-and-reinsert reindexing for growing session JSONL files by indexing appended or changed content incrementally.

## Best Possible Solution

Land or replace the active incremental-sync PR with a rebased memory-core fix that covers direct and source-wide session sync without accidental public SDK or storage-contract expansion.

## Reproduction Assessment

Yes at source level: a changed session JSONL file flows through buildSessionEntry into indexFile, and writeChunks clears then reinserts all indexed rows for that session path. I did not run a live long-session benchmark in this read-only review.

## Solution Assessment

Yes on direction, but not yet as a landed implementation. Incremental internal session indexing is the maintainable fix, provided it covers direct and source-wide batch sync and keeps SDK/storage contract changes either internal or explicitly approved.
