## Summary

Keep open: current main still rebuilds a full session-memory entry and clears/reinserts all indexed rows for a dirty session file once the current debounce and delta thresholds are met. ## What This Changes

The issue requests incremental session-memory indexing so growing session JSONL files do not trigger whole-file delete/reinsert and re-embedding on each qualifying sync.

## Best Possible Solution

Keep this issue as the canonical tracker until maintainers either land a compatibility-safe incremental session-sync design or explicitly decide that threshold-gated full reindex is acceptable for experimental session memory.
