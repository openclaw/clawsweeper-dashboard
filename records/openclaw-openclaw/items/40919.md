## Summary

Keep open: current main and the latest release still rebuild dirty session transcript entries by reading the whole JSONL file and clearing indexed rows before writing chunks, while https://github.com/openclaw/openclaw/pull/75179 is an open but conflicting implementation candidate that still needs maintainer review.

## What This Changes

The issue asks OpenClaw memory-core to replace full delete-and-reinsert session JSONL indexing with an incremental session transcript sync path.

## Best Possible Solution

Land or replace the open memory-core implementation with a narrow incremental session-indexing design that preserves full-reindex fallbacks, handles overlap and compaction safely, and includes focused upgrade and performance proof.

## Reproduction Assessment

Yes at source level: current main still routes dirty session files through whole-file `buildSessionEntry`, then `writeChunks` clears existing rows before inserting the rebuilt chunks. I did not run the 50-turn performance scenario live.

## Solution Assessment

Unclear as a final merge shape: incremental indexing is the right direction, but the branch now needs maintainer review for overlap handling, compaction/full-reindex fallbacks, schema compatibility, and proof.
