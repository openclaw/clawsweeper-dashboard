## Summary

Keep open. Current main still rebuilds dirty session transcripts from full JSONL content and writes them through the shared clear-and-reinsert index path; the active implementation candidate is still open at https://github.com/openclaw/openclaw/pull/75179.

## What This Changes

The issue requests incremental indexing for growing session JSONL memory files instead of repeated full delete-and-reinsert sync cycles.

## Best Possible Solution

Land a corrected incremental session indexing implementation that preserves unchanged chunks, falls back to full reindex when overlap or transcript rewrites make reuse unsafe, and keeps MEMORY.md on the existing full-reindex path.

## Reproduction Assessment

Yes, at source level. With session memory enabled and a dirty session crossing the configured thresholds, current main rebuilds the full session entry and sends it through indexFile/writeChunks, whose first step clears existing indexed rows for that path/source.

## Solution Assessment

Yes for the incremental direction, but not by closing this issue before the fix lands. The maintainable path is to review and land the open PR or a replacement that handles overlap, rewrites, durable metadata, and focused validation.
