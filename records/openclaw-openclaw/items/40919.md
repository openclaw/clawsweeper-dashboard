- `P2`: This is a normal-priority memory performance bug with limited blast radius and an active implementation candidate.
- `impact:session-state`: The report concerns session transcript indexing and memory sync state for growing JSONL session files.

## Summary

Keep open. Current main still rebuilds dirty session transcripts from the full JSONL content and sends them through the shared clear-and-reinsert writer, while the active implementation candidate remains open at https://github.com/openclaw/openclaw/pull/75179.

## What This Changes

The issue requests incremental session-memory indexing so growing JSONL transcripts do not repeatedly delete and reinsert all indexed chunks.

## Best Possible Solution

Land a corrected incremental session indexing implementation that preserves unchanged chunks, falls back to full reindex when overlap or transcript rewrites make reuse unsafe, and keeps MEMORY.md on the existing full-reindex path.

## Reproduction Assessment

Yes, at source level. With session memory enabled and a dirty session crossing the configured thresholds, current main rebuilds the full session entry and sends it through indexFile/writeChunks, whose first step clears existing indexed rows for that path/source.

## Solution Assessment

Yes for the incremental direction, but not by closing this issue before the fix lands. The maintainable path is to review and land the open PR or a corrected replacement that handles overlap, rewrites, durable metadata, and focused validation.
