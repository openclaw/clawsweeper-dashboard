## Summary

Keep open. Current main still rebuilds changed session transcripts from the full rendered JSONL content and sends them through the shared clear-and-reinsert writer; the active implementation path is the open PR at https://github.com/openclaw/openclaw/pull/75179.

## What This Changes

The issue requests incremental session-memory syncing so growing session JSONL transcripts are not fully deleted and reinserted on each sync.

## Best Possible Solution

Review, repair, and land https://github.com/openclaw/openclaw/pull/75179 or a replacement that preserves valid existing session chunks, records durable chunk metadata, and falls back to full reindex when transcript rewrites or overlap make incremental reuse unsafe.

## Reproduction Assessment

Yes, at source level. With session memory enabled and a transcript crossing the configured delta thresholds, current main reaches syncSessionFiles, rebuilds the full session entry, and writeChunks clears existing rows before rewriting chunks.

## Solution Assessment

The incremental direction is sound, but a raw lastSyncOffset is not enough by itself because chunk overlap and transcript rewrites need explicit handling. The maintainable solution is a session-specific incremental writer with durable chunk metadata and full-reindex fallbacks.
