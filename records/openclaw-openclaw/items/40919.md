## Summary

This issue should stay open: current main and the latest release still rebuild changed session JSONL entries through the full chunk delete/reinsert path, and the active fix path is the open but not merge-ready https://github.com/openclaw/openclaw/pull/75179.

## What This Changes

The issue requests incremental session-memory syncing so growing session JSONL files are not fully deleted and reinserted on every sync.

## Best Possible Solution

Land or replace the open incremental-session-sync PR with a rebased memory-core fix that covers direct and source-wide batch session sync, keeps new storage/API surface intentional, and avoids adding config unless maintainers explicitly choose it.

## Reproduction Assessment

Yes, at source level: a changed session JSONL file is rebuilt by buildSessionEntry, passed to indexFile, and writeChunks deletes existing rows before reinserting chunks. I did not run a live long-session benchmark in this read-only review.

## Solution Assessment

Yes on the requested incremental-indexing direction, but not by closing this issue yet. The best path is to review, repair, and land or replace the open implementation PR so both direct and source-wide provider paths stop doing full rewrites.
