## Summary

Keep open. Current main still rebuilds changed session transcripts from the full JSONL and sends them through the shared clear-and-reinsert writer, while https://github.com/openclaw/openclaw/pull/75179 remains the active but conflicting implementation candidate.

## What This Changes

The issue requests incremental session JSONL indexing so growing session transcripts do not repeatedly delete, re-embed, and reinsert unchanged chunks.

## Best Possible Solution

Land a corrected incremental session indexing implementation that preserves unchanged chunks, handles overlap and transcript rewrites safely, stores durable metadata, and keeps MEMORY.md on the existing full-reindex path.

## Reproduction Assessment

Yes, at source level. With session memory enabled and a dirty session crossing the configured thresholds, current main rebuilds the full session entry and sends it through indexFile/writeChunks, whose first step clears existing indexed rows for that path/source.

## Solution Assessment

Yes for the incremental direction, but not by closing this issue before the fix lands. The maintainable path is to review and land https://github.com/openclaw/openclaw/pull/75179 or a corrected replacement that handles overlap, rewrites, durable metadata, and focused validation.
