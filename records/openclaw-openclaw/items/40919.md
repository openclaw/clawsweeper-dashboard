## Summary

Keep open. Current main still rebuilds changed session transcripts from the full JSONL and routes them through the shared clear-and-reinsert index writer, while the active fix path remains the open, conflicting implementation PR at https://github.com/openclaw/openclaw/pull/75179.

## What This Changes

The issue requests incremental session memory sync for growing JSONL transcript files instead of repeatedly deleting and re-inserting all indexed chunks.

## Best Possible Solution

Land a corrected incremental session indexing implementation that preserves unchanged chunks, safely falls back for overlap or transcript rewrites, and keeps MEMORY.md on the existing full-reindex path.

## Reproduction Assessment

Yes, at source level. With session memory enabled and a dirty session crossing the configured thresholds, current main rebuilds the full session entry and sends it through indexFile/writeChunks, whose first step clears existing indexed rows for that path/source.

## Solution Assessment

Yes for the incremental direction, but not by closing this issue before the fix lands. The maintainable path is to review and land the open PR or a corrected replacement that handles overlap, rewrites, durable metadata, and focused validation.
