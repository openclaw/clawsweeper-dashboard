## Summary

Keep open. Current main still rebuilds changed session transcripts from full JSONL content and writes them through the shared clear-and-reinsert path; the active implementation candidate is still open at https://github.com/openclaw/openclaw/pull/75179.

## What This Changes

Request incremental session memory syncing for growing JSONL transcripts to avoid repeated full delete-and-reinsert indexing.

## Best Possible Solution

Review, repair, and land https://github.com/openclaw/openclaw/pull/75179 or a replacement that preserves valid existing chunks, records durable chunk metadata, and falls back to full reindex when transcript rewrites or overlap make incremental reuse unsafe.

## Reproduction Assessment

Yes, at source level. With session memory enabled and a transcript crossing the configured delta thresholds, current main reaches syncSessionFiles, rebuilds the full session entry, and writes it through indexFile/writeChunks.

## Solution Assessment

The incremental direction is appropriate, but a raw lastSyncOffset-only fix is not enough because chunk overlap and transcript rewrites need explicit handling. The maintainable solution is a session-specific incremental writer with durable chunk metadata and full-reindex fallbacks.
