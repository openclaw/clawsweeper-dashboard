## Summary

Keep open: current main and the latest release still rebuild changed session JSONL entries through the full index path, and the open implementation attempt at https://github.com/openclaw/openclaw/pull/75179 remains unmerged and conflicting with review blockers.

## What This Changes

Request incremental session memory indexing so growing session JSONL files do not require deleting and re-embedding all chunks on each dirty sync.

## Best Possible Solution

Land a rebased memory-core fix that preserves existing session chunks for appended transcript content across direct and source-wide session sync, with storage and API compatibility settled before merge.

## Reproduction Assessment

Yes, at source level. Current main builds a full session entry for dirty JSONL transcripts, routes it through indexFile, and writeChunks clears existing rows before reinserting; I did not run a long-session benchmark in this read-only review.

## Solution Assessment

Yes, incremental session indexing is the right end state, but the current open PR is not the best merge shape yet because it is conflicting and still needs source-wide provider plus storage/API compatibility resolution.
