- `impact:session-state`: The affected path synchronizes persisted session transcripts into searchable memory state.

## Summary

Current main still rebuilds changed session transcripts end to end: it reads and parses the full JSONL file, rechunks it, and replaces all index rows. The embedding cache can reduce repeated provider calls, but it does not remove the growing parsing, chunking, memory, and SQLite work. This issue should remain open while the paired implementation at https://github.com/openclaw/openclaw/pull/75179 is unresolved.

## What This Changes

Avoid full transcript delete-and-reinsert work when session JSONL files grow, while preserving correct indexing across chunk overlap and transcript rewrites.

## Best Possible Solution

Implement chunk-aware incremental replacement in memory-core: preserve stable prefix chunks and cached embeddings, recompute the overlap boundary and appended suffix, and atomically fall back to a full rebuild for rewrites, compaction, model changes, or chunking changes.

## Reproduction Assessment

Yes, from source with high confidence. When transcript growth crosses the configured session delta threshold, current main reconstructs the full session entry and replaces all index rows for the changed file; execution was not run because this review is read-only.

## Solution Assessment

No, a raw last-synced line or byte offset alone is not the best complete solution. Incremental work should be chunk-aware and retain explicit full-rebuild fallbacks for overlap boundaries, compaction, rewrites, and index-identity changes.
