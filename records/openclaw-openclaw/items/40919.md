## Summary

Keep this issue open: current main and the latest release still rebuild changed session JSONL files through the full session-entry and delete-before-write chunk path, and the active fix path is the open but not merge-ready https://github.com/openclaw/openclaw/pull/75179.

## What This Changes

The issue requests incremental session JSONL memory sync instead of periodically deleting and reinserting all indexed chunks for a growing session file.

## Best Possible Solution

Land or replace the active incremental-session-sync PR with a rebased memory-core fix that covers direct and source-wide batch session sync without unapproved public SDK or storage-contract drift.

## Reproduction Assessment

Yes, at source level: a changed session JSONL file is rebuilt through `buildSessionEntry`, then `indexFile`/`writeChunks` clears and rewrites indexed rows. I did not run a live long-session benchmark in this read-only review.

## Solution Assessment

Yes on the incremental-indexing direction, but not by closing this issue yet. The best path is to review, repair, and land or replace the open implementation PR that already targets this issue.
