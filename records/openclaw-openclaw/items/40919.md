## Summary

Keep open. Current main still rebuilds changed session transcripts from the full JSONL content and sends them through the shared clear-and-reinsert writer; the active fix path is the open implementation PR at https://github.com/openclaw/openclaw/pull/75179, which already has focused review findings to resolve.

## What This Changes

The issue requests incremental session memory indexing for growing JSONL transcripts instead of repeatedly deleting and re-inserting all session chunks.

## Best Possible Solution

Review, repair, and land https://github.com/openclaw/openclaw/pull/75179 or a replacement that preserves valid existing session chunks, records durable chunk metadata, and falls back to full reindex when transcript rewrites or overlap make incremental reuse unsafe.

## Reproduction Assessment

Yes, at source level. With session memory enabled and a transcript crossing the configured delta thresholds, current main reaches syncSessionFiles, rebuilds the full session entry, and writes it through indexFile/writeChunks.

## Solution Assessment

The incremental direction is sound, but the raw lastSyncOffset proposal is not enough by itself because chunk overlap and transcript rewrites need explicit handling. The maintainable fix is a session-specific incremental writer with durable chunk metadata and full-reindex fallbacks.
