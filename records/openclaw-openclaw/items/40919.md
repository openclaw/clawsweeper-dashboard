## Summary

Keep open. Current main still rebuilds dirty session transcripts from full JSONL content and writes them through the shared clear-and-reinsert path; the still-open implementation candidate is https://github.com/openclaw/openclaw/pull/75179, so this issue should remain open until that PR or a corrected replacement lands.

## What This Changes

The issue asks for incremental session-memory sync so growing session JSONL files are not fully deleted, re-embedded, and reinserted on each dirty sync.

## Best Possible Solution

Land a corrected incremental session indexing implementation from https://github.com/openclaw/openclaw/pull/75179 or a replacement that preserves valid chunks and falls back to full reindex when overlap, rewrites, or provider/chunking changes make reuse unsafe.

## Reproduction Assessment

Yes, at source level. With session memory enabled and a dirty session crossing the configured thresholds, current main rebuilds the full session entry and sends it through indexFile/writeChunks, whose first step clears existing indexed rows for that path/source.

## Solution Assessment

Yes for the incremental direction, but not by closing this issue before the fix lands. The best path is to review and land the open PR or a replacement that handles overlap, transcript rewrites, durable metadata, and focused validation.
