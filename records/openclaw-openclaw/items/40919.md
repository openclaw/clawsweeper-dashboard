## Summary

Keep open. Current main still rebuilds changed session transcripts from full JSONL content and writes them through the shared clear-and-reinsert path; the canonical fix path is the still-open https://github.com/openclaw/openclaw/pull/75179.

## What This Changes

Request incremental session memory sync so growing session JSONL files are not fully deleted, re-embedded, and reinserted on each delta sync.

## Best Possible Solution

Land a corrected incremental session indexing implementation from https://github.com/openclaw/openclaw/pull/75179 or a replacement that preserves valid chunks and falls back to full reindex when overlap or rewrites make reuse unsafe.

## Reproduction Assessment

Yes, at source level. With session memory enabled and a session file crossing the configured delta thresholds, current main reaches syncSessionFiles, rebuilds the full session entry, and writes it through indexFile/writeChunks.

## Solution Assessment

Yes for the incremental direction, but not by closing this issue before the fix lands. The open PR or a replacement should handle chunk overlap, transcript rewrites, durable metadata, and focused validation.
