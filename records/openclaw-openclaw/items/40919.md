## Summary

Keep open. Current main and the latest release still rebuild changed session transcripts from the full JSONL and route them through the shared clear-and-reinsert writer, while the active fix remains in an open conflicting PR with review blockers.

## What This Changes

The issue requests incremental session memory indexing for growing JSONL transcripts instead of repeatedly deleting and reinserting all session chunks.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/75179 with a corrected incremental session indexing implementation that preserves unchanged chunks, handles overlap and transcript rewrites, stores durable cursor/hash metadata, and leaves MEMORY.md on the existing full-reindex path.

## Reproduction Assessment

Yes, at source level. With session memory enabled and a dirty session crossing the configured thresholds, current main builds the full session entry and sends it through indexFile/writeChunks, whose first step clears existing rows for that session path/source.

## Solution Assessment

Yes for the incremental direction, but not by closing this issue before the fix lands. The maintainable path is to review and land the active PR or a corrected replacement that handles overlap, rewrites, durable metadata, and focused validation.
