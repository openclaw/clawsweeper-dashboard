## What This Changes

The issue requests incremental indexing for growing session JSONL memory files instead of deleting and reinserting all indexed chunks on each sync.

## Best Possible Solution

Land or replace the active memory-core fix so incremental session indexing covers direct and source-wide batch session sync while preserving storage and SDK compatibility.

## Reproduction Assessment

Yes, at source level. Current main builds full session entries for dirty session files and routes them through indexFile, and the write path clears existing rows before reinserting chunks.

## Solution Assessment

Yes, incremental session indexing is the right direction, but the implementation should cover both direct and source-wide batch providers and keep storage/SDK compatibility explicit.
