## Summary

Keep open. Current main still rebuilds dirty session transcripts from the full JSONL and writes them through the shared clear-and-reinsert path, while https://github.com/openclaw/openclaw/pull/75179 remains the active but conflicting implementation candidate.

## What This Changes

The issue requests incremental session transcript indexing instead of full delete-and-reinsert reindexing as session JSONL files grow.

## Best Possible Solution

Land a corrected incremental session indexing implementation that preserves unchanged chunks, safely falls back on overlap or transcript rewrites, and keeps MEMORY.md on the existing full-reindex path.

## Reproduction Assessment

Yes, at source level. With session memory enabled and a dirty session crossing the configured thresholds, current main rebuilds the full session entry and sends it through indexFile/writeChunks, whose first step clears existing indexed rows for that path/source.

## Solution Assessment

Yes for the incremental direction, but not by closing this issue before the fix lands. The maintainable path is to review and land the open PR or a corrected replacement that handles overlap, rewrites, durable metadata, and focused validation.
