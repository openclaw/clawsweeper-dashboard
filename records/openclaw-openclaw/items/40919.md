## What This Changes

The issue requests incremental session JSONL memory sync so growing transcripts are not repeatedly deleted, re-embedded, and reinserted on each sync.

## Best Possible Solution

Land a maintainer-approved memory-core fix that preserves existing session search semantics while incrementally updating changed session chunks across both direct and source-wide batch indexing paths.

## Reproduction Assessment

Yes at source level. Current main rebuilds dirty session entries and writeChunks clears existing rows before reinserting, although the embedding cache can avoid provider calls for unchanged chunk hashes.

## Solution Assessment

No for the offset-only proposal as written. Incremental sync is the right direction, but the best fix must account for chunk overlap, source-wide batch providers, and any persisted storage/API contract change.
