## Summary

Keep open. Current main still matches path-shaped exec allowlist entries against the PATH-resolved symlink path while approval-backed execution pins the real executable path, and https://github.com/openclaw/openclaw/pull/73543 remains open with closing syntax for this issue.

## What This Changes

Use one canonical executable identity for exec approval matching, persistence, safe-bin/skill trust, metadata, and execution when binaries resolve through symlinks.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73543 or a focused replacement that makes executable approval identity consistent across matching, persistence, trust checks, audit metadata, and regression coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction is an ExecutableResolution with resolvedPath=/opt/homebrew/bin/rg and resolvedRealPath=/opt/homebrew/Cellar/ripgrep/14.1.1/bin/rg: matchAllowlist() checks the former while approved execution pins the latter.

## Solution Assessment

Unclear as a complete solution. Consulting resolvedRealPath is necessary for the allowlist miss, but the full issue also needs a maintainer trust-model decision for symlink approvals and adjacent approval identity surfaces.
