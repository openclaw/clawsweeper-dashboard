## Summary

Keep open. Current main and the latest stable release still compare path-shaped exec allowlist entries against the PATH-resolved symlink path while approval-backed execution pins the real executable path, and an open fixing PR already uses closing syntax for this issue.

## What This Changes

Make exec approval identity canonical so symlink and realpath checks, persistence, trust decisions, and execution agree.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/73543 with a focused fix that uses one canonical executable identity across matching, trust, persistence, audit metadata, and execution, with symlink/realpath regression coverage.

## Reproduction Assessment

Yes. A source-level reproduction is an ExecutableResolution with resolvedPath=/opt/homebrew/bin/rg and resolvedRealPath=/opt/homebrew/Cellar/ripgrep/14.1.1/bin/rg: current main matches allowlist entries against the former while approved execution pins the latter.

## Solution Assessment

Unclear as a complete implementation. Canonicalizing the executable identity is the right direction, but the final fix needs maintainer security-policy signoff for symlink compatibility and the adjacent trust and persistence surfaces.
