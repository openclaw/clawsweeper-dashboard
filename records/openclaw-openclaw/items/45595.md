## What This Changes

The issue asks for exec approval matching, trust, persistence, and execution to use one canonical executable identity when PATH entries are symlinks.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/73543 with a focused fix that uses one canonical executable identity across matching, trust, persistence, audit metadata, and execution, with symlink/realpath regression coverage.

## Reproduction Assessment

Yes. A source-level reproduction is an ExecutableResolution with resolvedPath=/opt/homebrew/bin/rg and resolvedRealPath=/opt/homebrew/Cellar/ripgrep/14.1.1/bin/rg: current main matches allowlist entries against the former while approved execution pins the latter.

## Solution Assessment

Unclear as a complete implementation. Canonicalizing executable identity is the right direction, but the final fix should be reviewed as a security-policy change across matching, persistence, trust, and audit surfaces.
