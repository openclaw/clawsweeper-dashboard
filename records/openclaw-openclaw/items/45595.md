## Summary

Keep open: current main still has the reported approval/execution executable identity mismatch, and an open linked PR already targets this issue with closing syntax but has not merged.

## What This Changes

The issue asks OpenClaw to use one canonical executable identity for exec approval matching, trust and persistence metadata, and the path actually executed when commands resolve through symlinks.

## Best Possible Solution

Land or replace the linked PR with a maintainer-approved executable identity fix that aligns matching, trust checks, persistence/audit metadata, and execution, with symlink and realpath regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: an executable resolution with resolvedPath as a symlink and resolvedRealPath as the final binary is matched against the symlink path while approved execution pins the realpath.

## Solution Assessment

Unclear as a complete implementation. Canonicalizing executable identity is the right direction, but maintainers should decide whether symlink-form approvals remain valid and ensure all trust and persistence surfaces follow the chosen identity.
