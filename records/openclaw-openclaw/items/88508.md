## Summary

Keep open: current main and the latest release already support thinking levels in user/session/provider paths, but the plugin hook contract still does not expose a clean per-run thinking-level override for workflow plugins.

## What This Changes

The issue requests a plugin contract hook or parameter that lets plugins choose `low`, `medium`, `high`, or `xhigh` thinking dynamically per round.

## Best Possible Solution

Design one narrow, provider-profile-aware plugin hook or hook result field that validates requested thinking levels and fits the existing directive/session/config/provider precedence model.
