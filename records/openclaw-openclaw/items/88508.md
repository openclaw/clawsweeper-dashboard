## Summary

Keep open: current main and the latest release support thinking levels through user/session/provider paths and plugin-started embedded runs, but the inbound plugin hook contract still has no clean per-round thinking override for workflow plugins.

## What This Changes

The issue requests a plugin contract hook or result field that lets plugins choose a per-round thinking level such as low, medium, high, or xhigh.

## Best Possible Solution

Design one narrow, provider-profile-validated one-shot thinking override in the chosen plugin hook or turn-preparation seam, preserving the existing directive/session/config/provider precedence model and documenting the SDK contract.

## Reproduction Assessment

Not applicable; this is a feature/API request rather than a bug report with a failing current-main path. Source inspection confirms the requested hook field is absent, but there is no runtime failure to reproduce.

## Solution Assessment

Unclear; a `thinkingOverride` result field is plausible, but maintainers should first choose whether the API belongs on `before_model_resolve`, `agent_turn_prepare`, a dedicated thinking-routing hook, or another turn-preparation seam.
