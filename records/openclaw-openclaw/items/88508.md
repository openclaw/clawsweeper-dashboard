## Summary

Keep open: current main and v2026.6.9 still do not expose a Plugin SDK hook result for per-round thinking-level override, and the closest prior implementation PR was closed unmerged for redesign.

## What This Changes

The issue requests a Plugin SDK contract for per-round `thinkingOverride` so plugins can dynamically choose thinking intensity per turn.

## Best Possible Solution

Design a narrow provider-profile-aware Plugin SDK hook or result field with validation, precedence rules, docs, API baseline updates, and focused contract tests.

## Reproduction Assessment

Yes for the source-level capability gap: current main's plugin hook result and merge types do not include a thinking override field, while agent-core has an internal `thinkingLevel` update primitive.

## Solution Assessment

Unclear as an implementation; a `thinkingOverride`-style field may be viable, but maintainers need to choose the hook phase, precedence, validation behavior, and SDK compatibility path first.
