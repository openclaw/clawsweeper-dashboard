## Summary

Keep open: current main still has executable allowlist policy plus workspace-only filesystem guards, not a shared path-keyed RWX policy for exec and file tools. The related implementation PRs are closed unmerged, and the remaining work is a new security/config contract that needs maintainer product and security review.

## What This Changes

The issue requests a path-scoped read/write/execute permission map shared by exec, read, write, edit, and related file tools.

## Best Possible Solution

Define a maintainer-approved path-keyed RWX permission contract, then implement it consistently across config/schema/protocol, exec approvals/runtime, file tools, apply_patch, platform sandboxing, docs, and security regression tests.

## Reproduction Assessment

## Solution Assessment
