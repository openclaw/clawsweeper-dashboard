## Summary

Keep this open: current main still has binary/safe-bin exec approvals plus a coarse `workspaceOnly` filesystem guard, not the requested path-keyed RWX permission map across file tools and exec. The closed implementation pull requests are useful design history, and https://github.com/openclaw/openclaw/issues/68631 was already closed in favor of this issue as the canonical remaining path-policy thread.

## What This Changes

Request path-scoped RWX permissions that apply uniformly to file tools and exec, with backward-compatible integration with existing exec approval configuration.

## Best Possible Solution

Define a maintainer-approved path-keyed RWX permission contract, then implement it across exec, read/write/edit, apply_patch, schema/docs, security audit, and focused regression tests.

## Reproduction Assessment

Not applicable as a feature request. Current-main source and tests provide a high-confidence state check: exec is still binary/safe-bin approval based, and filesystem tools still use only `workspaceOnly`.

## Solution Assessment

No implemented solution is present on current main. The requested direction is plausible, but the best path is a maintainer-owned security/config design before another implementation PR.
