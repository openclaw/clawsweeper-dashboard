## Summary

Keep open: current main and v2026.6.8 still provide workspace-only filesystem guards, sandbox options, and exec approval allowlists, but not the requested path-scoped RWX policy shared across file and exec tools. The related implementation PRs are closed unmerged, and the remaining work is security/product design rather than conservative cleanup.

## What This Changes

Request path-keyed read/write/execute permission rules shared across file and exec tools, with deny precedence and backward compatibility for existing exec approvals.

## Best Possible Solution

Keep this as the owner-reviewed design thread for a path-scoped RWX policy that defines config contract, threat model, file/exec semantics, OS-enforcement role, migration behavior, docs, and focused tests before implementation.

## Solution Assessment

No as an automated one-shot fix. The RWX direction fits OpenClaw's security goals, but maintainers need to settle the config, migration, and enforcement design first because this changes security guarantees and tool semantics.
