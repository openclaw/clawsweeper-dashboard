## Summary

Keep open: current main still has binary/command allowlists for exec plus workspace-only file guards, not the requested path-keyed RWX policy. The linked implementation PRs are both closed unmerged, so this remains real but needs maintainer/secops design review rather than cleanup closure.

## What This Changes

Request path-scoped read/write/execute permissions for exec and file tools while preserving backward compatibility with existing exec allowlist configuration.

## Best Possible Solution

Land a backwards-compatible, opt-in path policy that applies consistently to file tools and exec, documents OS-layer limitations, and includes focused regression coverage for path matching, symlinks, delegation identity, and legacy allowlist coexistence.
