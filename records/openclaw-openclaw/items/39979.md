## Summary

Keep open. Current main still lacks a shared path-keyed RWX policy across filesystem tools and exec, and the linked implementation PRs were closed unmerged, so this remains the canonical design thread rather than something already implemented or superseded.

## What This Changes

Request path-keyed RWX filesystem permissions enforced across file tools and exec while preserving existing exec allowlist compatibility.

## Best Possible Solution

Keep this as the canonical thread and design a versioned filesystem permission contract that covers read, write, edit, apply_patch, and exec operands while documenting platform limits and preserving existing exec approval behavior.

## Reproduction Assessment

Yes for the current-state gap: source and regression-test inspection show filesystem tools are governed by workspaceOnly and exec approvals match commands/executables rather than target paths. I did not run tests because this review was read-only.

## Solution Assessment

Unclear as a concrete implementation plan, but keeping this open is the best triage outcome. The path-RWX direction targets the right boundary, while schema, OS enforcement, and delegation semantics need maintainer/security design first.
