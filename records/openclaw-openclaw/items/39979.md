## Summary

Keep open: current main and the latest release still expose binary-scoped exec approvals plus boolean workspace-only filesystem guards, not a shared path-keyed RWX policy for exec and file tools. The linked implementation attempts are closed unmerged, and the remaining work is a security-sensitive config/runtime contract that needs maintainer product and security review.

## What This Changes

The issue requests a path-keyed read/write/execute permission map that applies uniformly to exec, read, write, edit, and related file tools while staying compatible with existing exec allowlist settings.

## Best Possible Solution

Define an owner-reviewed path permission contract first, then implement it consistently across config/schema/protocol, exec approvals/runtime, file tools, apply_patch, platform sandboxing, docs, and security regression proof.
