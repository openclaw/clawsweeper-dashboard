## Summary

Keep this open: current main still has coarse `workspaceOnly` filesystem policy plus binary/safe-bin exec approvals, while the requested path-scoped RWX policy across file and exec tools is not implemented. The closed unmerged implementation PRs remain useful design context, not a cleanup close reason.

## What This Changes

The issue requests a path-keyed read/write/execute permission model enforced across file tools and exec instead of only binary-scoped exec approvals.

## Best Possible Solution

Define and document a core path-keyed RWX permission contract, including backward compatibility and platform fallback semantics, then implement it across exec, read/write/edit, apply_patch, config schema, docs, and focused security tests.

## Reproduction Assessment

Not applicable as a feature request. Current-main source still gives a high-confidence state check: exec approvals are binary/safe-bin based and filesystem tools only expose `workspaceOnly`, with tests documenting outside-workspace access when unset or false.
