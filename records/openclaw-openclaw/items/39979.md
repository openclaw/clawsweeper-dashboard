## Summary

Keep open: current main still exposes binary/safe-bin exec approvals plus a coarse `workspaceOnly` filesystem switch, not the requested path-scoped RWX policy across file tools and exec. The closed implementation PRs are useful design context, and https://github.com/openclaw/openclaw/issues/68631 was already closed in favor of this canonical thread rather than superseding it.

## What This Changes

Request a path-keyed read/write/execute permission map that applies uniformly to filesystem tools and exec while preserving existing exec allowlist compatibility.

## Best Possible Solution

Define and document a core path-keyed RWX permission contract, including compatibility and platform fallback semantics, then implement it consistently across exec, read/write/edit, apply_patch, config/schema/docs, and focused security tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-state check: exec approvals are still binary/safe-bin based and filesystem tools only expose `workspaceOnly`.

## Solution Assessment

No implemented solution is present on current main. The requested direction is plausible, but the best maintainable path is a maintainer-approved security/config design before another implementation PR.
