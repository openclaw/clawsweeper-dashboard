## Summary

Keep open: current main and the latest release still have binary-scoped exec approvals plus workspace-only filesystem guards, not the requested path-keyed RWX policy. The linked implementation attempts at https://github.com/openclaw/openclaw/pull/42653 and https://github.com/openclaw/openclaw/pull/42780 are closed unmerged after concrete security-review concerns, while https://github.com/openclaw/openclaw/issues/68631 was closed as a duplicate pointing back to this issue as the canonical path-scoped policy thread.

## What This Changes

Request a path-scoped read/write/execute permission map shared by exec and filesystem tools, with deny/default semantics and backward compatibility for existing exec allowlist config.

## Best Possible Solution

Define an owner-reviewed security design, then implement one canonical path policy contract across file tools and exec with compatibility, docs, doctor validation, canonical path handling, and regression tests for known bypass classes.
