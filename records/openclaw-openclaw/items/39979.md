Keep open: current main and v2026.6.6 still provide workspace-only file-tool scoping, sandbox backends, and exec approval guardrails, but not the requested path-scoped RWX permission map shared across file and exec tools. The closed implementation attempts and related discussions show this is security/product design work, not conservative cleanup.

## What This Changes

The issue requests path-scoped read/write/execute permission enforcement across file and exec tools, with backward compatibility for existing exec allowlist settings.

Maintain an owner-reviewed design thread for a path-scoped RWX policy that defines the config contract, threat model, tool and exec semantics, sandbox or OS-enforcement role, migration behavior, docs, and focused tests before implementation.
