## Summary

Keep open: current main still has executable-scoped exec approvals plus coarse `workspaceOnly` filesystem guards, not a shared path-keyed RWX policy for exec, read, write, edit, and apply_patch. The linked closed PRs remain useful design history, and the duplicate path-policy thread was already closed in favor of this issue, so this is still the canonical security/config design item.

## What This Changes

Request a backward-compatible path-keyed read/write/execute permission map enforced uniformly across exec and file tools.

## Best Possible Solution

Define a maintainer-approved path-keyed RWX permission contract, then implement it across config/schema/protocol, exec approvals, read/write/edit/apply_patch enforcement, platform sandbox behavior, docs, and focused security regression tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-state check: exec approvals remain executable scoped and file tools remain governed by `workspaceOnly`, not path-keyed RWX rules.

## Solution Assessment
