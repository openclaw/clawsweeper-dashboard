## Summary

Keep open: current main and v2026.6.6 still expose binary-scoped exec approvals plus workspace-only filesystem guards, not the requested path-keyed RWX policy; prior implementation PRs are closed unmerged after security review concerns, so this remains an owner-reviewed security/product design item.

## What This Changes

Add path-scoped read/write/execute filesystem permissions that apply consistently to exec and file tools while preserving existing exec allowlist compatibility.

## Best Possible Solution

Define an owner-reviewed security design, then implement one canonical path policy contract with compatibility, docs, doctor validation, canonical path handling, file-tool and exec enforcement, OS-backed coverage where available, and regression tests for known bypass classes.
