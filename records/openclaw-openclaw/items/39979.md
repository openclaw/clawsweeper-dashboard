## Summary

Keep open. Current main still lacks a shared path-keyed RWX policy across filesystem tools and exec, and the prior implementation attempts are closed unmerged while this remains the canonical design thread.

## What This Changes

The issue requests path-scoped RWX permissions for read/write/edit/apply_patch and exec so access follows target paths instead of only executable allowlists.

## Best Possible Solution

Keep this as the canonical thread and design a versioned filesystem permission contract that covers read, write, edit, apply_patch, and exec operands while preserving existing exec approval behavior and documenting platform limits.

## Reproduction Assessment

Yes for the current-state gap: source and test inspection show filesystem tools are governed by workspaceOnly and exec approvals match commands/executables rather than target paths. I did not run tests because this review was read-only.

## Solution Assessment

Unclear as a concrete implementation plan, but keeping this open is the best triage outcome. The path-RWX direction addresses the observed gap, while schema, OS enforcement, and delegation semantics need maintainer/security design first.
