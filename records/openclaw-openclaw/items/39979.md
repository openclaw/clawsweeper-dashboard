## Summary

Keep open: current main and the latest shipped release still expose binary-scoped exec approvals plus boolean workspace-only filesystem guards, not a shared path-keyed RWX policy for exec and file tools. The linked implementation PRs are closed unmerged, and the remaining work is a security-sensitive config/runtime contract that needs maintainer product and security review.

## What This Changes

The issue requests a path-keyed RWX permission map for exec and filesystem tools so access is enforced by target path instead of executable allowlists or a workspace-only boolean.

## Best Possible Solution

Define an owner-reviewed path permission contract first, then implement it consistently across config/schema/protocol, exec approvals/runtime, filesystem tools, apply_patch, platform sandboxing, docs, and security regression proof.
