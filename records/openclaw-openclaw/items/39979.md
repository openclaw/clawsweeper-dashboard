## Summary

Keep this open: current main still has workspace-only filesystem tooling plus binary/safe-bin exec approval controls, while the requested path-scoped RWX policy across file and exec tools is not implemented. The linked implementation attempts (#42653 and #42780) were closed unmerged, so they are useful design context but not a close reason.

## What This Changes

Request path-scoped read/write/execute permissions that apply uniformly to exec and file tools while remaining backward-compatible with existing exec allowlist configuration.

## Best Possible Solution

Define a core filesystem permission contract for path-keyed RWX policy, document backward compatibility and platform fallback semantics, then implement it across exec, read/write/edit, apply_patch, config schema, docs, and focused security tests.

## Reproduction Assessment

Yes, from source inspection: current main exposes `tools.fs.workspaceOnly` and per-agent exec binary allowlists, not the requested path-RWX map. I did not run live tool calls because this review was constrained to read-only inspection, but current tests and docs confirm the coarse existing behavior.

## Solution Assessment

No implemented solution is present on current main. The unmerged PRs show plausible approaches, but the best path is maintainer-approved security/config design before a new implementation PR.
