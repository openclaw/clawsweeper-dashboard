## Summary

Keep open: current main still prepends full fenced Conversation info and Sender JSON blocks for direct Telegram turns, and #72749 is an open unmerged closing PR rather than a shipped fix.

## What This Changes

The issue asks OpenClaw to reduce or move Telegram direct-message metadata out of verbose inline JSON in the model-facing user message.

## Best Possible Solution

Review and land #72749 or an equivalent narrow inbound prompt-builder change that compacts direct-chat metadata while preserving richer untrusted context for groups, replies, forwards, locations, structured context, and history.

## Reproduction Assessment

Yes. A direct Telegram inbound context sets ChatType direct with Provider and Surface telegram, then buildInboundUserContextPrefix emits the full metadata blocks and get-reply-run prepends them to the user body.

## Solution Assessment

Yes in direction, but not yet complete on main. A focused prompt-builder/config change is the narrow maintainable surface; the open closing PR should be reviewed or replaced before closing this issue.
