## Summary

Keep open: current main still emits the full inline Conversation info and Sender JSON metadata for direct Telegram-style inbound messages, and the linked fix PR is still open rather than merged.

## What This Changes

The issue asks for Telegram inbound user messages to avoid large repeated inline Conversation info and Sender JSON blocks before short user text.

## Best Possible Solution

Land PR #72749 or an equivalent narrow prompt-builder/config change that compacts direct Telegram metadata while preserving full untrusted context for groups, replies, forwards, locations, structured context, and history.

## Reproduction Assessment

Yes. A direct Telegram inbound context sets ChatType direct and Provider/Surface telegram, and current main then emits the full Conversation info and Sender metadata blocks before the user body.

## Solution Assessment

Unclear for final product shape. A compact prompt-builder/config mode appears narrower and more portable than provider-specific metadata fields, but the open PR still needs normal maintainer review.
