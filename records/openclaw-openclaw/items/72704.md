## Summary

Keep open: current main still prepends full fenced Conversation info and Sender metadata for Telegram direct messages, and #72749 is an open, unmerged implementation candidate rather than a shipped fix.

## What This Changes

The issue requests reducing Telegram direct-message prompt metadata by moving or compacting the repeated Conversation info and Sender JSON blocks prepended to user text.

## Best Possible Solution

Review and land #72749 or an equivalent narrow inbound prompt/config change that compacts direct-channel metadata while preserving full untrusted context for richer chat scenarios.

## Reproduction Assessment

Yes, at source level. Telegram DMs set direct chat context with Provider/Surface telegram, buildInboundUserContextPrefix emits full untrusted metadata blocks, and get-reply-run prepends them to the user body.

## Solution Assessment

Unclear. The compact metadata direction is plausible, but the active opt-in compact-direct approach still needs maintainer judgment versus a default prompt-shape change or provider-structured metadata.
