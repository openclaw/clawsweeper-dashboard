## Summary

Keep open: current main still prepends full fenced Conversation info and Sender JSON blocks for direct Telegram turns, and #72749 is the open, unmerged implementation candidate rather than a shipped fix.

## What This Changes

Reduce or externalize Telegram inbound metadata so direct user messages are not prefixed with large Conversation info and Sender JSON blocks.

## Best Possible Solution

Review and land #72749 or an equivalent narrow prompt-builder/config change that compacts direct-chat metadata while preserving full untrusted context for richer chat scenarios.

Yes. Source inspection on current main gives a high-confidence path: Telegram DMs set direct chat context with Provider/Surface telegram, buildInboundUserContextPrefix emits full untrusted metadata blocks, and get-reply-run prepends them to the user body.

## Solution Assessment

Unclear. The compact metadata direction is plausible, but the open PR's opt-in compact-direct mode still needs maintainer judgment versus a default prompt-shape change or provider-specific structured metadata.
