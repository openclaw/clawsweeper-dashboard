## What This Changes

The issue asks OpenClaw to stop putting verbose Telegram conversation and sender JSON metadata directly into user-message prompt text, or to move that metadata into a compact or structured context path.

## Best Possible Solution

Land the narrow runtime-context fix in https://github.com/openclaw/openclaw/pull/89428 if review confirms it keeps reply/event metadata available without prepending JSON metadata to active user prompt text; leave the broader minimal-DM configuration idea to https://github.com/openclaw/openclaw/issues/62077.

## Solution Assessment
