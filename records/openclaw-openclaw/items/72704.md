## Summary

Keep open: current main still builds full fenced Conversation info and Sender JSON blocks for direct Telegram turns, and #72749 is the open unmerged closing PR for this issue.

## What This Changes

Request compact or structured delivery of Telegram inbound metadata so short user messages are not dominated by inline JSON prompt context.

## Best Possible Solution

Review and land #72749 or an equivalent narrow inbound prompt-builder/config change that compacts direct-chat metadata while preserving full untrusted context for groups, replies, forwards, locations, and history.

## Reproduction Assessment

Yes. Source inspection on current main shows direct Telegram turns set ChatType direct with Provider/Surface telegram, and the prompt builder prepends full Conversation info and Sender blocks before the user body.

## Solution Assessment

Unclear. #72749 is the active implementation candidate, but maintainers still need to decide whether its optional compact-direct mode is the best product shape or whether direct-chat compaction should be default or handled through another metadata contract.
