## Summary

Keep open: current main still sends full fenced Conversation info and Sender metadata blocks for direct Telegram turns, and the linked compact-metadata implementation remains an open, conflicting PR rather than merged behavior.

## What This Changes

The issue requests reducing Telegram direct-message prompt bloat by moving or compacting inline JSON metadata that is prepended to user message content.

## Best Possible Solution

Review, repair, land, or close https://github.com/openclaw/openclaw/pull/72749; if that path is abandoned, consolidate the remaining product decision with https://github.com/openclaw/openclaw/issues/62077.

Yes. Source inspection shows Telegram DMs build direct Telegram context, `buildInboundUserContextPrefix` emits full fenced metadata for direct non-webchat channels, and the reply runner carries that context into the model turn.

## Solution Assessment

Unclear. A compact direct metadata mode is plausible, but the current issue should be solved through the linked implementation or a maintainer-chosen equivalent that preserves provenance, sender attribution, and richer chat context.
