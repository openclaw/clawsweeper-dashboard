## Summary

Keep open: current main still prepends full fenced Conversation info and Sender JSON blocks for direct Telegram turns, and the linked closing PR is open and unmerged rather than a shipped fix.

## What This Changes

Request compact or structured Telegram inbound metadata so short direct-chat user messages are not preceded by large inline JSON blocks.

## Best Possible Solution

Review and land #72749 or an equivalent narrow inbound prompt-builder/config change that compacts direct-chat metadata while preserving full untrusted context for richer chat scenarios.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: Telegram DMs set ChatType direct with Provider/Surface telegram, buildInboundUserContextPrefix emits full untrusted metadata blocks, and get-reply-run prepends them to the user body.

## Solution Assessment

Unclear. #72749 is the active implementation candidate, but maintainers still need to decide whether its opt-in compact-direct mode is the desired product shape or whether direct-chat compaction should be handled differently.
