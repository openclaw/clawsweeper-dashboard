## Summary

Keep open: current main still prepends full fenced Conversation info and Sender metadata blocks for direct Telegram-style external-channel turns, and the linked closing PR #72749 remains open and unmerged rather than a shipped fix.

## What This Changes

The issue requests reducing or restructuring verbose inline Telegram inbound metadata so short user messages are easier for models to understand.

## Best Possible Solution

Review and land #72749 or an equivalent narrow inbound prompt-builder change that compacts direct-chat metadata while preserving richer untrusted context for groups, replies, forwards, locations, structured context, and history.

## Reproduction Assessment

Yes. A direct Telegram inbound context sets ChatType direct with Provider/Surface telegram, then buildInboundUserContextPrefix emits the full Conversation info and Sender metadata blocks before get-reply-run prepends them to the user body.

## Solution Assessment

Unclear for final product shape. Provider-native metadata fields are not portable across all model backends, so the maintainable fix likely belongs in prompt construction/config, but #72749 still needs normal maintainer review.
