## Summary

Keep open: current main still prepends full fenced Conversation info and Sender JSON blocks for direct Telegram turns, and #72749 is open and unmerged rather than a shipped fix.

## What This Changes

The issue asks to reduce Telegram direct-message prompt bloat by moving or compacting inline Conversation info and Sender JSON metadata.

## Best Possible Solution

Review and land #72749 or an equivalent narrow inbound prompt-builder/config change that compacts direct-chat metadata while preserving full metadata for richer chat contexts.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: Telegram direct chats set ChatType direct with Provider/Surface telegram, buildInboundUserContextPrefix emits full untrusted JSON metadata blocks, and get-reply-run prepends them to the model-facing user body.

## Solution Assessment

Unclear. #72749 is the active implementation candidate, but maintainers still need to decide whether its opt-in compact-direct mode is the desired product shape or whether the prompt format should change differently.
