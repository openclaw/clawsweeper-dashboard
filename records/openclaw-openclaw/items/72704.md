## Summary

Keep open: current main still prepends full fenced Conversation info and Sender JSON blocks for Telegram direct messages, and #72749 is the open, unmerged implementation candidate rather than a shipped fix.

## What This Changes

The issue requests reducing or externalizing Telegram direct-message metadata that is currently prepended as full JSON blocks before user text.

## Best Possible Solution

Review and land #72749 or an equivalent narrow prompt-builder/config change that compacts direct-chat metadata while preserving full context for groups, replies, forwards, locations, and history.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: Telegram DMs set ChatType direct with Provider/Surface telegram, buildInboundUserContextPrefix emits full untrusted metadata blocks, and get-reply-run prepends them to the user body.

## Solution Assessment

Unclear. Compact direct metadata is plausible, but the final choice between default prompt-shape changes, opt-in config, or provider-structured metadata needs maintainer product judgment because current tests intentionally preserve full direct external-channel metadata.
