## What This Changes

The issue asks OpenClaw to reduce or restructure verbose inline Telegram inbound metadata that is currently prepended to short user messages.

## Best Possible Solution

Review and land #72749 or an equivalent narrow prompt-builder/config change, then use #62077 for any broader minimal inbound metadata policy.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: Telegram DMs pass direct telegram context, buildInboundUserContextPrefix emits full untrusted metadata blocks for non-webchat direct channels, and get-reply-run prepends them to the model-facing prompt.

## Solution Assessment

Unclear. #72749 is the active implementation candidate, but maintainers still need to decide whether its optional compact-direct mode is the desired product shape or whether the default prompt format should change.
