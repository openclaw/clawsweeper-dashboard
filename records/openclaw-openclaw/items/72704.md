- `impact:session-state`: Generated inbound metadata is currently carried into active prompt/session text, which can distort transcript and runtime-context state.

## Summary

Keep open: current main and the latest release still prepend generated inbound metadata into the embedded agent prompt/session path, and the issue is already paired with open fix candidates that should be reviewed or consolidated before closure.

## What This Changes

The issue asks OpenClaw to stop putting verbose Telegram/channel `Conversation info` and `Sender` JSON blocks directly in user-authored prompt text.

## Best Possible Solution

Land one canonical hidden-runtime-context route that keeps inbound metadata model-available but out of user-authored prompt/session text, then close this issue from the merged PR.

## Reproduction Assessment

Yes at source level. Current main emits the reported metadata blocks, stores them as `currentInboundContext`, and then re-prepends that context into both session and model prompt strings.

## Solution Assessment

Yes. The hidden-runtime-context direction is the narrowest maintainable fix because it preserves channel metadata for the model without treating generated metadata as user-authored prompt text.
