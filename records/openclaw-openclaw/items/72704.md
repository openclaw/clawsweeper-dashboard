## Summary

Keep open. Current main still prepends full fenced Conversation info and Sender JSON metadata to direct external-channel user messages, including Telegram DMs, and #72749 is already the open closing implementation candidate.

## What This Changes

The issue requests reducing or restructuring the inline JSON metadata prepended to Telegram user messages so short direct messages are easier for models to interpret.

## Best Possible Solution

Review and land #72749, or an equivalent narrow inbound prompt/config change, so direct Telegram-style chats can use compact metadata while group chats, replies, forwards, locations, and richer history contexts keep the full untrusted context where it is still useful.

## Reproduction Assessment

Yes. A static reproduction path is high confidence: Telegram DMs finalize context with ChatType direct plus telegram provider/surface/channel and sender/message identifiers, then buildInboundUserContextPrefix emits full fenced Conversation info and Sender blocks that get prepended to the user body.

## Solution Assessment

Yes for triage. Keeping this issue open while #72749 remains open is the right handling; the code solution should stay narrow to inbound prompt construction/config and avoid dropping richer metadata from group or contextual flows.
