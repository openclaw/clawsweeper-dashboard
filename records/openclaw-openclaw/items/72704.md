## Label Justifications

- `P2`: The issue affects model comprehension and prompt/session quality for Telegram/channel turns, but the affected surface is bounded and has open fix candidates.
- `impact:session-state`: The generated metadata is currently carried through prompt/session context as user-role text, which can distort transcript and runtime-context state.

## Summary

Keep open: current main still generates `Conversation info` / `Sender` metadata and prepends `currentInboundContext` into the active embedded-agent prompt/session text, while the explicit closing fix at https://github.com/openclaw/openclaw/pull/89428 is still open and currently conflicting.

## What This Changes

The issue requests moving Telegram/channel metadata out of user-message text and into structured or hidden runtime context so model prompts stay focused on the actual user request.

## Best Possible Solution

Land or refresh the hidden-runtime-context routing so current inbound metadata remains available to the model without becoming user-authored prompt or session text; treat compact inline formats as a secondary mitigation, not the core fix.

## Reproduction Assessment

Yes. Source inspection shows `buildInboundUserContextPrefix` emits the reported metadata blocks and `run/attempt.ts` re-prepends `currentInboundContext` into `promptForSession` and `promptForModel`; I did not run a live Telegram turn in this read-only review.

## Solution Assessment

Yes. The hidden-runtime-context direction is the narrowest maintainable solution because it removes generated metadata from user-authored prompt/session text while preserving model access to channel metadata.
