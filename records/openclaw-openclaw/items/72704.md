Keep open: current main still prepends full fenced Conversation info and Sender JSON blocks for direct Telegram turns, and the linked closing PR is open and unmerged rather than a shipped fix.

## What This Changes

The issue requests reducing Telegram direct-message inline JSON metadata in model-facing user content, preferably via structured fields or a compact header.

## Best Possible Solution

Review and land #72749 or an equivalent narrow inbound prompt/config change that compacts direct-chat metadata while preserving full metadata for group and rich-context scenarios.

Unclear. The source problem is clear, but maintainers still need to decide whether #72749's optional compact-direct config is the desired product shape or whether direct-chat compaction should be handled another way.
