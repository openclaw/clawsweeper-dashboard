## Summary

Keep open: current main still builds full Conversation info and Sender metadata for direct Telegram/current-turn prompts, and the linked closing implementation remains open and unmerged, so cleanup closure would be premature.

## What This Changes

The issue requests reducing or restructuring repeated inline Telegram direct-message metadata so model turns are less dominated by JSON context blocks.

## Best Possible Solution

Ship a backward-compatible direct-chat metadata design that reduces prompt noise or moves metadata out of user text while preserving sender attribution, reply/forward context, history sanitization, schema/docs, and regression coverage.

## Reproduction Assessment

Yes for the observable prompt-shape complaint: source inspection shows Telegram DMs produce direct Telegram context, current main emits full Conversation info and Sender blocks, and the embedded runner prepends current-turn context into the model prompt. The downstream model-comprehension impact was not live-measured in this read-only review.

## Solution Assessment

Unclear. The compact-direct direction in https://github.com/openclaw/openclaw/pull/72749 is plausible, but it overlaps with the out-of-band runtime-context direction in https://github.com/openclaw/openclaw/pull/81521 and needs maintainer judgment on default behavior, config, and attribution preservation.
