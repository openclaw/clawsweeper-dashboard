## Summary

Keep open: current main and the latest release still route generated inbound metadata back into active prompt/session text, and the issue already has an open closing PR that should be reviewed or repaired rather than duplicated.

## What This Changes

The issue asks to keep Telegram inbound JSON metadata out of user prompt text, or make it compact enough that models can reliably find the user-authored message.

## Best Possible Solution

Land one canonical hidden-runtime-context route that preserves inbound metadata for model use while keeping user-authored prompt/session text clean, then close this issue from the merged PR.

## Reproduction Assessment

Yes at source level. Current main emits the reported metadata blocks, stores them as `currentInboundContext`, and then re-prepends that context into both session and model prompt strings.

## Solution Assessment

Yes. The hidden-runtime-context direction is the narrow maintainable fix because it preserves channel metadata for model use without treating generated metadata as user-authored prompt text.
