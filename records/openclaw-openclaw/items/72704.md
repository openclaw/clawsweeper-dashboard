## Summary

Keep open: current main still prepends full fenced Conversation info and Sender JSON blocks for direct Telegram turns, and #72749 is the open, unmerged implementation candidate rather than a shipped fix.

## What This Changes

The issue requests reducing Telegram/direct inbound prompt noise by moving repeated Conversation/Sender metadata out of inline user message text or compacting it.

## Best Possible Solution

Review and land #72749 or an equivalent narrow, provider-agnostic prompt-builder change that compacts direct-chat metadata while preserving full untrusted context for richer chat scenarios.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: Telegram DMs set direct Telegram context, buildInboundUserContextPrefix emits full untrusted metadata blocks, and get-reply-run prepends them to the user body.

## Solution Assessment

Unclear. Compact direct metadata is a plausible direction, but the exact default versus opt-in config behavior still needs maintainer product judgment.
