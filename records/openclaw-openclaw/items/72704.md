## Summary

Keep open: current main still prepends full fenced Conversation info and Sender JSON blocks for direct Telegram-style inbound messages, and #72749 is the open implementation candidate linked to close this issue.

## What This Changes

The issue requests a less noisy inbound Telegram DM prompt format so message metadata no longer dominates short user messages.

## Best Possible Solution

Land #72749 or an equivalent narrow inbound prompt-builder change that compacts direct-chat metadata while preserving richer untrusted context for groups, replies, forwards, locations, structured context, and history.

## Reproduction Assessment

Yes. A direct Telegram inbound context reaches buildInboundUserContextPrefix with ChatType direct and Provider/Surface telegram, and current main emits the full Conversation info and Sender metadata blocks before the user body.

## Solution Assessment

Unclear for final product shape. Provider-native metadata fields are not portable across model backends, so the likely maintainable fix belongs in prompt construction/config, but #72749 still needs normal maintainer review.
