## Summary

Keep open: current main has watcher debounce, direct restart coalescing, active-work deferral, and a 30-second restart cooldown, but it does not implement the requested protected-path config-edit debounce window with merged restart-required edits and an ack ETA.

## What This Changes

Request a debounce or batch mechanism so multiple restart-required `config.patch` edits produce one gateway restart instead of one restart per edit.

## Best Possible Solution

Keep this open for maintainer design of one canonical restart coalescing path that batches restart-required config writes and reports pending restart timing without adding redundant knobs when existing restart settings can carry the behavior.

## Reproduction Assessment

Yes at source level: current main either restarts immediately when no active work is present or coalesces only while a direct/scheduled/active-work restart is already pending; I did not run a live gateway repro.

## Solution Assessment

Unclear: reducing restart churn is directionally right, but the exact API/config shape needs maintainer judgment because adjacent reload debounce, restartDelayMs, coalescing, cooldown, and deferral semantics already exist.
