## Summary

Keep open. Current main still lists only directly controlled subagents for both the model tool and slash command; nested workers are exposed only as descendant counts or child-session summaries, so the requested recursive visibility is still missing.

## What This Changes

Add depth/recursive options to `/subagents list` and the `subagents` model tool so a parent session can inspect nested descendant subagents.

## Best Possible Solution

Ship a bounded descendant traversal that preserves default direct-child behavior, supports finite depth and all-descendants modes, and returns flat rows with hierarchy metadata.

## Reproduction Assessment

Yes. A static reproduction is to seed main -> orchestrator and orchestrator -> worker runs; listing from main passes through the exact direct-controller filter, so only the orchestrator row is returned while the worker is reduced to descendant metadata.

## Solution Assessment

Yes. The requested depth/recursive list option is the narrowest maintainable fix if it reuses the existing registry snapshot traversal and does not change control permissions by default.
