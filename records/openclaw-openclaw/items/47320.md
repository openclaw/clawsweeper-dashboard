## Summary

Keep open. Current main still lists only directly controlled subagents for both the model tool and slash command, while nested workers are only summarized through descendant counts or child-session metadata; the request is recent, corroborated in discussion, and has a clear core subagent listing path.

## What This Changes

The issue asks for recursive or depth-limited subagent listing so a depth-0 controller can inspect nested workers spawned by intermediate subagents.

## Best Possible Solution

Add a bounded traversal mode that preserves the default direct-child behavior, supports finite `depth` and all-descendants `recursive`, and emits flat rows with `depth` and parent session metadata for both tool and slash-command consumers.

## Reproduction Assessment

Yes. A high-confidence source reproduction is main -> orchestrator -> worker: current list requests from main use an exact direct-controller filter, so only the orchestrator row is returned while the worker remains visible only as descendant summary metadata.

## Solution Assessment

Yes. The requested depth/recursive list option is the narrowest maintainable direction if it reuses existing registry/descendant traversal data and keeps control operations scoped to direct children.
