## Summary

Keep open. Current main and the latest release still make `subagents list` a direct-controller inspection surface: nested workers can be summarized under a parent, but they are not returned as their own rows with depth or parent metadata.

## What This Changes

Request recursive or depth-bounded `subagents list` output so a root orchestrator can inspect nested workers and their hierarchy.

## Best Possible Solution

Add bounded read-only descendant traversal for `subagents list`, preserving direct-only default behavior and returning flat rows with `depth` plus parent session metadata for tool and slash-command consumers.

## Reproduction Assessment

Yes, by source inspection. The root caller's list path filters runs by exact controller session key, so a depth-2 worker owned by a depth-1 orchestrator is omitted from root `subagents list` rows.

## Solution Assessment

Yes. A backward-compatible `depth` or `recursive` list option is the narrow maintainable path if it reuses existing descendant traversal and keeps control operations direct-scoped.
