## Summary

Keep open. Current main still lists only directly controlled subagents for both the model tool and slash command; descendant traversal exists for counts/cleanup, but not as depth-bounded or recursive list rows, and the discussion includes a corroborating orchestrator-pattern use case.

## What This Changes

The issue requests depth-bounded or recursive `subagents list` output so a parent agent can inspect grandchildren with hierarchy metadata.

## Best Possible Solution

Add a list-only bounded traversal mode that preserves the default direct-child behavior, supports finite `depth` and all-descendants `recursive`, and emits flat rows with `depth` plus parent session metadata for both tool and slash-command consumers.

## Reproduction Assessment

Yes. A high-confidence source reproduction is main -> orchestrator -> worker: current list requests use an exact direct-controller filter, so the worker is omitted from depth-0 list output and only summarized through descendant metadata.

## Solution Assessment

Yes. The requested depth/recursive list option is the narrowest maintainable path if it reuses existing descendant traversal helpers and leaves control operations direct-scoped.
