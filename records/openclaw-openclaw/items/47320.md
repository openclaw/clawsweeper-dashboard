## Summary

Keep open. Current main still lists only directly controlled subagents through both the model tool and slash command; descendant data is present only as summary metadata, so the requested depth-bounded or recursive listing remains useful and narrowly scoped.

## What This Changes

Request adding depth-bounded or recursive `/subagents list` and `subagents` tool output so top-level orchestrators can see nested subagents with parent metadata.

## Best Possible Solution

Add read-only bounded descendant traversal for list output, preserving default direct-child behavior and emitting flat rows with `depth` and parent session metadata for tool and slash-command consumers.

## Reproduction Assessment

Yes. A source reproduction is main -> orchestrator -> worker: current list requests call `listControlledSubagentRuns` with the main session key, so the worker owned by the orchestrator is omitted as a row.

## Solution Assessment

Yes. A read-only `depth`/`recursive` option is the narrowest maintainable direction if it reuses existing descendant indexes and leaves control operations direct-scoped.
