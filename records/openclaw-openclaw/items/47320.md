## Summary

Keep open. Current main still exposes `subagents list` as a direct-child inspection surface: nested workers can appear as descendant summaries, but they are not returned as list rows with their own status, runtime, model, task, depth, or parent metadata.

## What This Changes

Request adding recursive/depth options to `subagents list` so a root orchestrator can inspect depth-2 subagents with hierarchy metadata.

## Best Possible Solution

Add bounded read-only descendant traversal for `subagents list`, preserving the direct-only default and returning flat rows with `depth` plus parent session metadata for tool and slash-command consumers.

## Reproduction Assessment

Yes, by source inspection. The root caller's list path filters runs by exact controller session key, so a depth-2 worker owned by a depth-1 orchestrator is omitted from root `subagents list` rows.

## Solution Assessment

Yes. A backward-compatible `depth` or `recursive` list option is the narrow maintainable path if it reuses existing descendant traversal and keeps control operations direct-scoped.
