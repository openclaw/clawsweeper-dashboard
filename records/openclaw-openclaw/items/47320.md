## Summary

Keep open. Current main still exposes `subagents list` as a direct-child view, while the requested depth-aware descendant rows remain absent; the corroborating user comment makes this a concrete nested-orchestrator visibility gap rather than a one-off preference.

## What This Changes

Add `--depth`/`--recursive` and tool-call depth support to `subagents list` so root coordinators can inspect descendant subagents with parent metadata.

## Best Possible Solution

Add bounded read-only descendant traversal for list output, preserving direct-only defaults and returning flat rows with `depth` plus parent session metadata for tool and slash-command consumers.

## Reproduction Assessment

Yes, by source inspection for the feature gap. A main -> orchestrator -> worker setup currently calls `listControlledSubagentRuns` with the main session key, so the worker owned by the orchestrator is omitted as its own row.

## Solution Assessment

Yes. A backward-compatible `depth` or `recursive` list option is the narrow maintainable path if it reuses existing descendant traversal and keeps control operations direct-scoped.
