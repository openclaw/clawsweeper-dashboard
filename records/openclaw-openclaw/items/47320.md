## Summary

Keep open. Current main still exposes only directly controlled subagents in both the model tool and slash command, while descendant information is only summarized, so the requested recursive or depth-bounded listing remains valid and narrowly scoped.

## What This Changes

Add `--depth` or `--recursive` support to `subagents list` and the model tool so callers can inspect nested subagents with parent hierarchy metadata.

## Best Possible Solution

Add read-only bounded descendant traversal for list output, preserving default direct-child behavior and emitting flat rows with `depth` and parent session metadata for tool and slash-command consumers.

## Reproduction Assessment

Yes. A high-confidence source reproduction is main -> orchestrator -> worker: current list requests call `listControlledSubagentRuns` with the main session key, so the worker owned by the orchestrator is omitted as its own row.

## Solution Assessment

Yes. A read-only `depth` or `recursive` option is the narrowest maintainable direction if it reuses the existing descendant indexes and leaves control operations direct-scoped.
