## Summary

Current main and v2026.6.5 still list only the effective controller’s direct run set, so the requested descendant visibility remains missing. The broader open chain-tracking issue is related but does not supersede this narrower command/tool feature; it proposes canonical lineage fields and leaves traversal as later work.

## What This Changes

Add a backward-compatible depth-limited or recursive descendant view to `/subagents list` and the `subagents` tool, including hierarchy metadata for nested runs.

## Best Possible Solution

Implement one bounded descendant traversal over the existing controller/registry graph or the canonical lineage contract selected in the related issue, preserve direct children as the default, and return flat rows with parent session key and relative depth for both tool and command consumers.

## Reproduction Assessment

Yes, at source level. Current main exposes no depth/recursive input and passes only the effective controller’s direct run set into the list renderer; a live failure run is unnecessary to establish that the requested capability is absent.

## Solution Assessment

The proposed additive interface is directionally good, but it is not implementation-ready until maintainers choose the canonical lineage source and ancestor visibility rules. The narrowest maintainable solution reuses the registry/controller graph and shares any durable parent/depth contract with the related chain-tracking work.
