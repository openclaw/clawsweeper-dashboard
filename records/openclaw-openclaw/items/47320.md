## Summary

Keep open. Current main and the latest release still expose `subagents list` as a direct-controller listing surface: nested workers can be summarized under a parent, but they are not returned as depth-aware rows with parent metadata. The broader chain-tracking request at https://github.com/openclaw/openclaw/issues/11040 is related, but it does not obsolete this narrower list-command feature.

## What This Changes

Request adds `--depth`/`--recursive` support to `/subagents list` and equivalent tool parameters so a requester can inspect descendant subagents with hierarchy metadata.

## Best Possible Solution

Add an explicit read-only depth/recursive listing path that defaults to direct children, reuses existing descendant traversal/indexing, returns parent/depth metadata, and leaves spawn permissions and announce routing unchanged.

## Reproduction Assessment

Yes for source reproduction: current main filters `subagents list` to the effective controller's direct runs and only summarizes descendants under parent rows. I did not run a live multi-agent scenario in this read-only sweep.

## Solution Assessment

Yes, the best narrow solution is an additive `depth` or `recursive` option with default depth 1. It should not change spawn-depth policy, control permissions, or completion announcement routing.
