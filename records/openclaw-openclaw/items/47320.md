## Summary

Keep open. Current main still routes both model-tool and slash-command list requests through direct child-only subagent queries; nested workers surface only as descendant counts or child session summaries, not flattened rows with depth or parent metadata. The issue is recent, corroborated by another user, and PR #14447 added nested orchestration without adding recursive listing.

## What This Changes

Request support for `subagents list` depth/recursive options, including tool-call depth input, so a parent session can list nested subagents with hierarchy metadata.

## Best Possible Solution

Add a bounded traversal mode that preserves the current default direct-child list, supports finite `depth` plus all-descendants `recursive`, emits flat rows with `depth` and `parentSessionKey`, updates docs, and covers both model-tool and slash-command paths without broadening control permissions.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists by seeding a main-owned orchestrator run plus an orchestrator-owned worker run: the current list query returns only the orchestrator, while the worker is visible only through descendant summary metadata.

## Solution Assessment

Yes. The requested depth/recursive list option is the narrowest maintainable path if it reuses the existing descendant traversal helper and leaves non-list actions scoped to directly controlled children.
