## Summary

Keep open. Current main still exposes only direct subagent children through the model tool and slash command, while nested workers are only summarized as descendant counts or child-session metadata; the request is recent, corroborated, and maps to a narrow core subagent listing feature.

## What This Changes

Request adds `--depth`/`--recursive` recursive listing for `/subagents list` and the `subagents` tool so a parent can see nested descendants with hierarchy metadata.

## Best Possible Solution

Add a bounded traversal mode that preserves default direct-child output, supports finite `depth` and all-descendants `recursive`, emits flat rows with `depth` and `parentSessionKey`, and updates tool/slash docs, changelog, and focused tests.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to seed main -> orchestrator and orchestrator -> worker runs: current main list requests from the main session use an exact direct-controller filter, so only the orchestrator row is returned while the worker appears only through descendant summary metadata.

## Solution Assessment

Yes. The requested depth/recursive list option is the narrowest maintainable direction if it reuses existing descendant traversal, keeps default behavior unchanged, and does not extend control actions to descendants.
