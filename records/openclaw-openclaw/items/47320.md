## Summary

Keep open. Current main still lists only directly controlled subagents for both the model tool and slash command, while nested workers appear only through descendant counts or child-session summaries; the request is recent, corroborated, and has a clear core subagent seam.

## What This Changes

Request adding finite-depth or recursive `subagents list` support so a parent session can inspect nested worker labels, status, runtime, model, and parent hierarchy.

## Best Possible Solution

Add a bounded traversal mode that preserves the default direct-child behavior, supports finite `depth` and all-descendants `recursive`, emits flat rows with `depth` and `parentSessionKey`, and updates model-tool, slash-command, docs, changelog, and focused tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to seed main -> orchestrator and orchestrator -> worker runs: current main list requests from the main session use an exact direct-controller filter, so only the orchestrator row is returned while the worker is visible as descendant summary metadata.

## Solution Assessment

Yes. The requested depth/recursive list option is the narrowest maintainable fix if it reuses existing descendant traversal and keeps control permissions scoped to direct children unless a separate maintainer decision broadens them.
