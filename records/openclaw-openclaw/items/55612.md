## Summary

Current `main` still only emits the CRITICAL elevated/runtime-filesystem exposure findings for `groupPolicy="open"`; `dmPolicy="open"` is only covered by the broader multi-user heuristic and exec-surface checks, so this issue is not implemented on `main` and should stay open.

## Evidence

- **Exposure matrix collector still only scans open group policies:** `collectExposureMatrixFindings()` calls `listGroupPolicyOpen()` and returns early when no open groups exist. `listGroupPolicyOpen()` only records `channels.*.groupPolicy` and `channels.*.accounts.*.groupPolicy`; it does not inspect `dmPolicy`, so `security.exposure.open_groups_with_elevated` and `security.exposure.open_groups_with_runtime_or_fs` cannot fire for open DMs.
