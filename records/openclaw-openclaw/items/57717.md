## Summary

Keep open: this is useful work for the active Slack markdown-table Block Kit gap, but it still needs real Slack after-fix proof, a rebase against current main, and a small correctness fix before merge.

## What This Changes

The PR adds Slack markdown table extraction in the send path so one markdown table can be posted as a Slack Block Kit table attachment, with code rendering fallback for multi-table messages.

## Best Possible Solution

Rebase or replace this on current main, preserve the current registry-driven config and Slack send infrastructure, fix fallback truncation counts, then land with focused tests and redacted real Slack proof.

## Reproduction Assessment

Yes for the PR finding: a table with headers and 110 data rows renders with a 98-row data budget in the Block Kit path but fallback text reports only 10 hidden rows. The overall feature gap is also source-verifiable on current main because Slack still posts markdown table output as text chunks.

## Solution Assessment

Partly. The single-table attachment approach matches the existing table metadata seam, but the branch needs a rebase, the fallback count fix, and real Slack proof before it is the maintainable merge path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

- **[P2] Use the block row budget for fallback counts:** `extensions/slack/src/block-kit-tables.ts:168-178`
