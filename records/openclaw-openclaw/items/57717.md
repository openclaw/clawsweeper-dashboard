## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve registry-backed table defaults:** `src/config/markdown-tables.ts:17-22`
  - body: Current main derives table defaults from the active channel plugin registry and invalidates the cache when the registry changes. Replacing that with this static map would drop plugin-provided defaults and make future channel/plugin defaults drift from the resolver, so this branch needs to rebase and keep the registry-backed path.
  - confidence: 0.89
- **[P1] Scope block tables to attachment-capable Slack sends:** `src/config/markdown-tables.ts:18`
  - body: Making Slack's default `block` here affects slash-command replies too: `extensions/slack/src/monitor/slash.ts` passes this resolver result into `deliverSlackSlashReplies`, which still calls `markdownToSlackMrkdwnChunks` and never attaches collected tables. In block mode the IR collects table metadata instead of rendering table text, so markdown tables in slash replies can disappear unless block mode is limited to the attachment-capable send path or slash replies coerce back to code.
  - confidence: 0.88
- **[P2] Report row loss for column-only overflow:** `extensions/slack/src/block-kit-tables.ts:100-103`
  - body: When columns overflow and `rows.length` is already at the table cap, this branch pops a data row to make room for the column indicator, but `truncatedRows` remains zero and the indicator reports only missing columns. That silently drops a visible row for full-height tables; either avoid the row eviction or include the row loss in the indicator/fallback behavior.
  - confidence: 0.82
