## Summary

Keep open. Current main still has only the shared table-block IR seam, while the PR’s Slack send-path implementation is useful but not merge-ready because it can drop slash-command table replies and can hide a data-row eviction in truncated table blocks.

## What This Changes

The PR adds Slack table metadata extraction, Block Kit table attachments and fallbacks in `sendMessageSlack`, table-mode default changes, targeted tests, and some unrelated formatting-only churn.

## Best Possible Solution

Land a rebased Slack implementation that consumes block-mode table metadata only on Slack paths that emit Block Kit table attachments, preserves safe text rendering for slash/non-Slack paths, and covers table-only, multi-table, media, truncation, and fallback cases.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main lacks table attachments, and the PR’s Slack-wide block default would send slash replies through `markdownToSlackMrkdwnChunks` without consuming collected table metadata.

## Solution Assessment

No. The direction is maintainable, but the PR should scope block table mode to `sendMessageSlack` or teach slash replies to emit the metadata, and it should make truncation indicators accurately report any row evicted to fit Slack limits.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep Slack slash replies out of block table mode:** `src/config/markdown-tables.ts:16`
  - body: Adding Slack to the default block table modes makes every Slack caller resolve `"block"`, but slash-command replies still render with `markdownToSlackMrkdwnChunks(...)` and never send the collected table metadata. In block mode the IR removes table text, so slash replies containing tables can lose content; scope block mode to `sendMessageSlack` or coerce slash replies back to `code` until they attach tables.
  - confidence: 0.91
- **[P2] Count rows evicted for truncation indicators:** `extensions/slack/src/block-kit-tables.ts:96-99`
  - body: When column truncation is present and the table already fills Slack's row cap, this path pops the last data row before adding the indicator while `truncatedRows` remains zero. The output reports only missing columns even though a row was dropped; count the eviction as row truncation or skip the indicator when no row is available.
  - confidence: 0.86
