## Summary

Keep open: the Slack table send path is useful and not implemented on current main, but this PR still has correctness blockers and lacks inspectable real Slack behavior proof before merge.

## What This Changes

The branch wires Slack markdown-table metadata into `sendMessageSlack` for Block Kit table attachments, adjusts table-mode defaults/config metadata, adds Slack table tests, and carries unrelated import/formatting churn.

## Best Possible Solution

Land a rebased Slack implementation that scopes block table mode to metadata-consuming send paths or teaches slash replies to send table attachments, reports truncation accurately, and includes redacted real Slack proof.

## Reproduction Assessment

Yes. Source inspection of the PR head gives a high-confidence reproduction path for the blockers: Slack slash replies receive block table mode while still sending text-only responses, and the row eviction path is deterministic in the table helper.

## Solution Assessment

No. The direction is plausible, but the current diff should scope block mode or add metadata delivery for slash replies, fix truncation reporting, rebase over current main, and provide real Slack proof before it is the maintainable merge path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep slash replies out of block table mode:** `src/config/markdown-tables.ts:17-18`
  - body: Setting Slack's default table mode to `block` also feeds block mode into slash-command replies. That path still renders with `markdownToSlackMrkdwnChunks` and responds with text only, so table content can be dropped unless slash replies consume the table metadata or coerce back to `code`.
  - confidence: 0.91
- **[P2] Report row loss when adding column indicators:** `extensions/slack/src/block-kit-tables.ts:100-102`
  - body: For a table that already fills Slack's row cap and only exceeds the column cap, this `pop()` removes the last data row before adding the indicator while `truncatedRows` remains zero. Count that eviction, skip the indicator when full, or reserve capacity explicitly so row loss is not hidden.
  - confidence: 0.86
