Full review comments:

- **[P1] Scope block tables to attachment-capable sends:** `src/config/markdown-tables.ts:69-72`
  - body: This allows `block` mode for every Slack caller once the Slack default or config resolves to block, but slash-command replies still render through `markdownToSlackMrkdwnChunks` and send text-only response_url payloads. In block mode the IR collector removes table text for metadata consumption, so slash replies with markdown tables can lose content; keep text-only slash delivery on `code` until it can send the table attachments too.
  - confidence: 0.92

  - body: This branch replaces current main's plugin-registry-backed `DEFAULT_TABLE_MODES` with a static map. Current main loads defaults from registered channel plugin metadata, so landing this stale resolver would drop plugin-owned defaults and fight the channel ownership boundary; rebase and add Slack's behavior through the current metadata/send-path model.
  - confidence: 0.88
- **[P2] Report row eviction for column-only overflow:** `extensions/slack/src/block-kit-tables.ts:100-101`
  - body: For a table already at the 100-row cap with only column overflow, `rows.length` is 100, so this pops the last data row before appending `+N more columns` while `truncatedRows` remains zero. The Slack message loses a row without telling the user; either preserve the row or report that row eviction in the indicator.
  - confidence: 0.9
