## Summary

Keep open: this PR targets a real current-main cron cleanup bug, but the latest branch can still abandon `deleteAfterRun` cleanup after a fixed retry budget, and the external PR still lacks visible after-fix real behavior proof.

## What This Changes

The PR adds descendant-aware isolated cron direct-delivery `deleteAfterRun` cleanup, regression tests, cron docs, and a changelog entry.

## Best Possible Solution

Make direct-delivery cleanup lifecycle-owned: defer deletion while descendants are active and keep one deduplicated cleanup owner until the descendant tree drains, with regression coverage for long-running descendants.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: structured or threaded direct delivery with `deleteAfterRun: true` reaches direct cleanup, and current main deletes the parent session without checking active descendants.

## Solution Assessment

No. The PR moves the guard into the right cleanup helper, but the fixed three-retry timer can stop before descendants finish; a lifecycle-owned drain cleanup is the narrower maintainable fix.
