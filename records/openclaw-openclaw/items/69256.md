## Summary

Keep open: current main still deletes isolated direct-delivery `deleteAfterRun` sessions without checking active descendants, and this PR targets the right bug, but the latest head can stop retrying before long descendants drain and still lacks visible real behavior proof.

## What This Changes

The PR adds descendant-aware isolated cron direct-delivery cleanup, a capped deferred cleanup retry, regression tests, docs, and a changelog entry.

## Best Possible Solution

Make direct-delivery cleanup lifecycle-owned or drain-triggered so one deduplicated cleanup owner remains active until the descendant run tree drains, then require redacted real Feishu or threaded direct-delivery proof before merge.

## Reproduction Assessment

Yes. By source inspection, current main routes structured/threaded direct delivery through cleanup that deletes the parent session without a descendant guard, while the text path and docs already establish descendant-aware isolated delivery.

## Solution Assessment

No. The PR moves the guard into the right helper and handles short drains, but the capped retry path can stop before long descendants finish; cleanup should remain owned until the descendant tree actually drains.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86
