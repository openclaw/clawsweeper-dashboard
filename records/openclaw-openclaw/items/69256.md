## Summary

Keep open: current main still has a real direct-delivery cleanup bug, but the PR head is not mergeable as-is because its capped deferred retry can permanently skip `deleteAfterRun` after longer descendant runs, and external after-fix proof is still missing.

## What This Changes

The branch adds descendant-aware cleanup and deferred retrying for isolated cron direct-delivery `deleteAfterRun` sessions, plus regression tests, docs, and a changelog entry.

## Best Possible Solution

Make direct-delivery cleanup lifecycle-owned or drain-triggered so one deduplicated cleanup owner remains active until the descendant tree drains, then require redacted real Feishu/threaded proof before merge.

## Reproduction Assessment

Yes. By source inspection, current main routes structured/threaded direct delivery through cleanup that deletes the parent session without a descendant guard, while the text path and docs already establish descendant-aware behavior.

## Solution Assessment

No. The PR moves the guard into the right helper, but the capped retry path can stop before long descendants finish; a lifecycle-owned or drain-triggered cleanup path is the narrower maintainable fix.
