## Summary

Keep open: current main still has the direct-delivery descendant cleanup bug, and this PR targets the right surface, but PR head is still not merge-ready because the bounded deferred retry can give up before long descendants drain and external real-behavior proof is still missing.

## What This Changes

The PR adds descendant-aware cleanup and deferred retrying for isolated cron direct-delivery `deleteAfterRun` sessions, plus regression tests, docs, and a changelog entry.

## Best Possible Solution

Make direct-delivery cleanup lifecycle-owned or drain-triggered so one deduplicated cleanup owner remains active until the descendant run tree drains, then require redacted real Feishu/threaded proof before merge.

## Reproduction Assessment

Yes. By source inspection, current main routes structured/threaded direct delivery through cleanup that deletes the parent session without a descendant guard, while the text path and docs already establish descendant-aware behavior.

## Solution Assessment

No. The PR moves the guard into the right helper and handles short drains, but the capped retry path can stop before long descendants finish; a lifecycle-owned or drain-triggered cleanup owner is the narrower maintainable fix.
