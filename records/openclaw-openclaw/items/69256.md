## Summary

Keep this PR open: the reported current-main bug is source-reproducible, but the branch still has a blocking cleanup-lifecycle defect and lacks required real behavior proof from the contributor’s setup.

## What This Changes

Adds descendant-aware deferred cleanup for isolated cron direct-delivery `deleteAfterRun` sessions, plus docs, changelog, and dispatch tests.

## Best Possible Solution

Make direct-delivery cleanup lifecycle-owned so one deduplicated cleanup owner remains active until the descendant tree drains, then require redacted real Feishu/threaded proof before merge.

## Reproduction Assessment

Yes. By source inspection, current main routes structured/threaded direct delivery through cleanup that deletes the parent session without a descendant guard, while the text path and docs already establish descendant-aware behavior.

## Solution Assessment

No. The PR moves the guard into the right helper, but the capped retry path can stop before long descendants finish; a lifecycle-owned or drain-triggered cleanup path is the narrower maintainable fix.
