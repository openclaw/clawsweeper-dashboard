## Summary

Keep open. Current main still displays cache read/write tokens in `/status`, but the cost estimate still sends only input/output tokens to the cache-aware estimator; the active fix path remains the open, conflicting PR at https://github.com/openclaw/openclaw/pull/65381.

## What This Changes

Fix `/status` cost estimates so cache read and cache write token costs are included when cache counters are present.

## Best Possible Solution

Forward `cacheRead` and `cacheWrite` through the existing `estimateUsageCost` call in the status message path, with regression coverage for stored and transcript-derived cache usage.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: cache counters reach the visible `/status` cache line, but current main passes only input/output tokens into `estimateUsageCost`.

## Solution Assessment

Yes. Reusing the existing cache-aware estimator contract from the status caller is the narrow maintainable fix; the linked PR follows that direction but still needs conflict resolution and validation.
