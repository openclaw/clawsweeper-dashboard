## Summary

Keep open. Current main still has the raw Feishu streaming-card path, but the latest PR patch remains not merge-ready because the sanitizer can duplicate final text after escaping and still misses tag-shaped and multi-backtick cases.

## What This Changes

The PR adds Feishu Card Kit markdown sanitization and tests, strips reasoning tags from streamed partials, and routes markdown-card replies with reply targets through post-format messages.

## Best Possible Solution

Keep the Feishu plugin fix, but make sanitization canonical across update and close state, cover unsafe tags and multi-backtick spans, and record the user-facing fix in the changelog before merge.

## Reproduction Assessment

Yes for the PR blockers: a source-level sanitizer probe reproduces final-text duplication, unescaped `<thinking>` text, and multi-backtick code-span corruption. The underlying current-main bug is source-reproducible because Feishu streaming cards still send raw markdown content.

## Solution Assessment

No. The PR direction is right, but sanitizing only the update path before state merge is not the narrowest maintainable fix; the Feishu streaming state/output boundary should use one sanitized representation for both partial and final updates.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Sanitize final text before merging stream state:** `extensions/feishu/src/streaming-card.ts:399`
  - body: `update()` stores sanitized content in `state.currentText`, but `close(finalText)` still merges the raw final text. For `x < 10`, the partial state becomes `x \\< 10`, the raw final no longer overlaps, and close sends duplicated unsanitized content.
  - confidence: 0.92
- **[P2] Escape tag-shaped unsafe angle brackets:** `extensions/feishu/src/streaming-card.ts:74-77`
  - body: The angle-bracket regex only catches `<` runs without a later `>` on the same line. Tag-shaped text such as `<thinking>` is not whitelisted but remains unescaped, so the reported Card Kit truncation case can still parse as a tag.
  - confidence: 0.86
- **[P2] Preserve multi-backtick code spans before escaping:** `extensions/feishu/src/streaming-card.ts:69-72`
  - body: The code-span placeholder only handles single-backtick spans. Valid Markdown spans delimited by multiple backticks leave their contents exposed, so examples like ``x < y`` are rewritten with an inserted backslash.
  - confidence: 0.84
- **[P3] Add the Feishu fix to the changelog:** `extensions/feishu/src/streaming-card.ts:35`
  - body: This is a user-facing Feishu delivery fix, but the PR branch has no `CHANGELOG.md` entry for the streaming truncation behavior. Repository policy requires user-visible fixes to be recorded.
  - confidence: 0.87
