## Summary

Keep open: current main still sends raw Feishu Card Kit markdown and lacks the sanitizer, but this PR head still has blocking sanitizer/state bugs, no real Feishu proof, and drift from newer reply-fallback behavior on main.

## What This Changes

The PR adds a Feishu Card Kit markdown sanitizer for streaming cards, strips risky summary/reasoning text in streaming paths, falls markdown-card replies back to post messages, and adds sanitizer tests.

## Best Possible Solution

Keep raw streaming state canonical, sanitize only outbound Card Kit update/close/summary payloads, cover unsafe tag-shaped text and multi-backtick spans, preserve reply fallback flags, and then collect redacted Feishu proof.

## Reproduction Assessment

Yes, at source level. Current main sends raw Card Kit markdown and the linked report gives concrete backtick and `<thinking>` inputs, but I did not run a live Feishu reproduction in this read-only review.

## Solution Assessment

No. The PR direction is useful, but this branch sanitizes too early by mutating stream state, misses unsafe tag-shaped text and multi-backtick spans, and needs to preserve current reply fallback behavior when rebased.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep raw stream state before sanitizing output:** `extensions/feishu/src/streaming-card.ts:398-399`
  - body: `update()` sanitizes incoming text before storing it in `state.currentText`. `close(finalText)` later merges raw final text into that sanitized state, so an overlap like `x < 10` can become duplicated as sanitized-plus-raw text and reintroduce a raw `<` on the final update. Keep state raw and sanitize only outbound Card Kit payloads.
  - confidence: 0.92
- **[P2] Escape unsafe tag-shaped text:** `extensions/feishu/src/streaming-card.ts:73-77`
  - body: The angle-bracket regex only escapes `<` when there is no later `>` before the line end, so non-whitelisted tag-shaped text like `<thinking>` remains unescaped. That leaves the reported Card Kit truncation class still parseable as a tag.
  - confidence: 0.86
- **[P2] Preserve multi-backtick code spans before escaping:** `extensions/feishu/src/streaming-card.ts:69-72`
  - body: The code-span placeholder only matches single-backtick spans. Valid Markdown spans delimited by multiple backticks remain exposed, so inline code such as double-backtick-delimited `x < y` can be rewritten with a backslash inside the code sample.
  - confidence: 0.84
- **[P2] Forward the reply fallback flag through post replies:** `extensions/feishu/src/send.ts:716-724`
  - body: Current main only allows top-level fallback for missing thread replies when `allowTopLevelReplyFallback` is true. This new reply branch calls `sendMessageFeishu` without forwarding that flag, so rebasing it would regress markdown-card replies in recoverable missing-thread cases.
  - confidence: 0.78
