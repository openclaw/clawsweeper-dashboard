## Summary

Keep open: current main still lacks a Feishu Card Kit markdown sanitizer for the reported streaming truncation path, but this PR is conflicting, lacks required real Feishu proof, and still has source-level correctness blockers around stream state, tag escaping, multi-backtick spans, and reply fallback propagation.

## What This Changes

The PR adds a Feishu Card Kit markdown sanitizer and tests, strips reasoning text from streaming partials, and routes markdown-card replies through post messages.

## Best Possible Solution

Keep raw streaming state canonical, sanitize only outbound Card Kit update/close/summary payloads, handle unsafe tag-shaped text and multi-backtick spans, preserve reply fallback flags, and collect redacted Feishu proof before merge.

## Reproduction Assessment

Yes, at source level. Current main sends raw Card Kit markdown through `updateCardContent`, and the linked report gives concrete backtick and `<thinking>` inputs, but I did not run a live Feishu reproduction in this read-only review.

## Solution Assessment

No. The PR direction is useful, but the current branch sanitizes too early, misses reported tag-shaped text and valid multi-backtick spans, drops a reply fallback flag, and still needs real Feishu proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep raw stream state before sanitizing output:** `extensions/feishu/src/streaming-card.ts:398-399`
  - body: `update()` sanitizes before `mergeStreamingText()` stores pending/current state. `close(finalText)` later merges raw final text into that escaped state, so input like `x < 10` can be duplicated as escaped-plus-raw text and reintroduce a raw `<` on the final update. Keep state raw and sanitize only outbound Card Kit payloads.
  - confidence: 0.92
- **[P2] Escape unsafe tag-shaped text:** `extensions/feishu/src/streaming-card.ts:73-77`
  - body: The angle-bracket regex only escapes a non-whitelisted `<` when there is no later `>` before the line end, so tag-shaped text like `<thinking>` remains unescaped. That leaves the reported truncation class still parseable as a tag.
  - confidence: 0.86
- **[P2] Preserve multi-backtick code spans before escaping:** `extensions/feishu/src/streaming-card.ts:69-72`
  - body: The code-span placeholder only matches single-backtick spans. Valid Markdown spans delimited by multiple backticks remain exposed, so double-backtick inline code containing `x < y` can be rewritten with a backslash inside the code sample.
  - confidence: 0.84
- **[P2] Forward the reply fallback flag through post replies:** `extensions/feishu/src/send.ts:716-724`
  - body: The new reply branch calls `sendMessageFeishu` without forwarding `allowTopLevelReplyFallback`. Current main forwards that flag through markdown-card sends, so missing-thread recoveries that are currently allowed can start failing after this change.
  - confidence: 0.78
