## Summary

Keep open: this is a real Feishu streaming-card bug and current main still sends raw Card Kit markdown, but this PR still has sanitizer correctness blockers and lacks after-fix real Feishu behavior proof.

## What This Changes

The PR adds Feishu Card Kit markdown sanitization/tests, strips reasoning tags from streamed partials, and routes Feishu markdown-card replies through post-format messages when replying.

## Best Possible Solution

Keep the Feishu plugin fix, but preserve raw streaming state, sanitize only the outbound Card Kit update/close/summary payloads, cover unsafe tag-shaped text and valid multi-backtick spans, and collect redacted Feishu proof after the fix.

## Reproduction Assessment

Yes for the PR blockers: a read-only source probe reproduces duplicated final text for `x < 10`, unescaped `<thinking>` text, and multi-backtick code-span corruption. The underlying Feishu truncation bug is source-reproducible on current main because Card Kit updates still send raw markdown content.

## Solution Assessment

No. The PR direction is plausible, but sanitizing before stream merging makes synthetic escapes canonical; the safer fix is to keep raw stream state and sanitize consistently at the Feishu Card Kit output boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep raw stream state before sanitizing output:** `extensions/feishu/src/streaming-card.ts:399`
  - body: `update()` sanitizes snapshots before merging them into `state.currentText`, but `close(finalText)` later merges the raw final text. For input like `x < 10`, the displayed partial becomes `x \\< 10`, no longer overlaps the raw final, and close sends duplicated unsanitized content. Keep state raw and sanitize only outbound Card Kit payloads, or sanitize final text before the same canonical merge.
  - confidence: 0.92
- **[P2] Escape unsafe tag-shaped text:** `extensions/feishu/src/streaming-card.ts:74`
  - body: The angle-bracket regex only catches `<` runs without a later `>` on the same line. Tag-shaped text such as `<thinking>` is not whitelisted but remains unescaped, so the reported Card Kit truncation class can still be parsed as a tag.
  - confidence: 0.86
- **[P2] Preserve multi-backtick code spans before escaping:** `extensions/feishu/src/streaming-card.ts:69`
  - body: The code-span placeholder only handles single-backtick spans. Valid Markdown spans delimited by multiple backticks leave their contents exposed, so examples like ``x < y`` are rewritten with an inserted backslash.
  - confidence: 0.84
