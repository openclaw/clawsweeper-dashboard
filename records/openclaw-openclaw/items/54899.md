## Summary

Keep open: current main still sends raw Feishu Card Kit markdown, and this PR is useful but still has sanitizer correctness blockers plus the required changelog gap before merge.

## What This Changes

The PR adds Feishu Card Kit markdown sanitization/tests, strips reasoning tags from streaming partials, and routes markdown-card replies with reply targets through post-format messages.

## Best Possible Solution

Keep the Feishu plugin fix, but make the Card Kit output/state boundary use one canonical sanitized representation, handle unsafe tag-shaped text and valid multi-backtick spans, and add the user-facing changelog entry.

## Reproduction Assessment

Yes at source/probe level. Current main sends raw markdown to Card Kit, and the latest PR review probe reproduced the remaining sanitizer failures for duplicated final text, tag-shaped angle brackets, and multi-backtick spans.

## Solution Assessment

No. The PR direction is plausible, but sanitizing partial updates while final close can merge raw text is not the narrowest maintainable fix; the sanitizer should be applied consistently at the Feishu Card Kit output/state boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Sanitize final text before merging stream state:** `extensions/feishu/src/streaming-card.ts:399`
  - body: `update()` stores sanitized partial content, but `close(finalText)` still merges the raw final payload. When the sanitizer changes `x < 10` to `x \< 10`, the raw final no longer overlaps and the close update can send duplicated, unsanitized content.
  - confidence: 0.92
- **[P2] Escape tag-shaped unsafe angle brackets:** `extensions/feishu/src/streaming-card.ts:74-77`
  - body: The angle-bracket regex only catches `<` when there is no later `>` on the same line. Text like `<thinking>` is not whitelisted but remains unescaped, so the reported Card Kit parser truncation case can still parse as a tag.
  - confidence: 0.86
- **[P2] Preserve multi-backtick code spans before escaping:** `extensions/feishu/src/streaming-card.ts:69`
  - body: The code-span placeholder only handles single-backtick spans. Valid Markdown spans delimited by multiple backticks remain exposed, so examples like ``x < y`` are rewritten with an inserted backslash inside code.
  - confidence: 0.84
- **[P3] Add the Feishu fix to the changelog:** `extensions/feishu/src/streaming-card.ts:35`
  - body: This is a user-facing Feishu delivery fix, but the branch has no `CHANGELOG.md` entry for the streaming truncation behavior. Repository policy requires user-visible fixes to be recorded.
  - confidence: 0.87
