Full review comments:

- **[P1] Keep raw stream state before sanitizing output:** `extensions/feishu/src/streaming-card.ts:398-400`
  - body: `update()` sanitizes `text` before `mergeStreamingText()` stores it as pending/current state. A later raw final snapshot such as `x < 10` can fail to overlap with the escaped partial, causing duplicate text or reintroducing the raw Card Kit markdown this PR is trying to avoid; keep state raw and sanitize only outbound payloads.
  - confidence: 0.92
- **[P1] Forward fallback permission through post replies:** `extensions/feishu/src/send.ts:716-724`
  - body: Current main has an `allowTopLevelReplyFallback` contract for unavailable Feishu reply targets, but this new reply branch delegates to `sendMessageFeishu` without accepting or forwarding that flag. Rebasing this as-is would make previously allowed recovery paths fail at runtime.
  - confidence: 0.86
- **[P2] Escape tag-shaped text that Card Kit can parse:** `extensions/feishu/src/streaming-card.ts:73-77`
  - body: The angle-bracket regex only escapes a non-whitelisted `<` when no later `>` appears before line end, so the reported `<thinking>` shape is left untouched. That can preserve the truncation bug for the exact input class cited by the PR.
  - confidence: 0.86
- **[P2] Mask multi-backtick spans before escaping:** `extensions/feishu/src/streaming-card.ts:69-72`
  - body: The placeholder pass only protects single-backtick spans. Valid Markdown spans delimited with multiple backticks remain exposed, so inline code containing `x < y` can be rewritten with a backslash inside the code sample and visibly corrupt streamed output.
  - confidence: 0.84
