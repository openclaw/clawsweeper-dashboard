## Summary

Keep open. The Feishu truncation problem is still not implemented on current main, but this PR branch is not safe to merge as-is because the sanitizer creates mixed sanitized/raw stream state and still misses common Markdown/tag cases from the reported surface.

## What This Changes

The PR adds Feishu Card Kit markdown sanitization for streaming card updates, sanitizer tests, streaming-summary cleanup, reasoning-tag filtering for partial updates, and a post-format reply fallback for markdown-card replies.

## Best Possible Solution

Revise the Feishu plugin patch so streaming-card state uses one canonical representation across partial updates and final closeout, unsafe tag-shaped text is escaped without breaking allowed Feishu/HTML tags, all valid Markdown code span forms are protected, and the user-facing fix is covered by focused tests and the changelog.

## Reproduction Assessment

Yes. The PR sanitizer can be reproduced with a read-only Node probe: `x < 10` becomes sanitized during partial update and then duplicates with raw final text, `<thinking>` remains unescaped, and ``x < y`` is rewritten inside a valid multi-backtick span.

## Solution Assessment

No. The current PR direction is plausible, but this implementation is not the narrowest safe fix until it either keeps raw text for merge state and sanitizes only at the Card Kit output boundary, or sanitizes every state transition consistently with fuller Markdown span/tag handling.

## Review Findings

Full review comments:

- **[P1] Sanitize final text before merging stream state:** `extensions/feishu/src/streaming-card.ts:399`
  - body: `update()` now stores sanitized content in `state.currentText`, but `close(finalText)` still merges raw final text. For `x < 10`, the partial state becomes `x \\< 10`; the raw final no longer overlaps and gets appended, sending duplicated unsanitized content on close.
  - confidence: 0.92
- **[P2] Escape tag-shaped unsafe angle brackets:** `extensions/feishu/src/streaming-card.ts:74`
  - body: The angle-bracket regex only catches `<` runs with no closing `>` before the line ends. The linked truncation case includes tag-shaped text such as `<thinking>`, which is not whitelisted but remains unescaped and can still be parsed as a tag by Card Kit.
  - confidence: 0.86
- **[P2] Preserve multi-backtick code spans before escaping:** `extensions/feishu/src/streaming-card.ts:69`
  - body: The code-span placeholder only handles single-backtick delimiters. Valid Markdown spans delimited by multiple backticks leave their contents exposed, so examples like double-backtick `x < y` are rewritten with an inserted backslash.
  - confidence: 0.82
- **[P3] Add the Feishu fix to the changelog:** `extensions/feishu/src/streaming-card.ts:35`
  - body: This is a user-facing Feishu delivery fix, but the PR does not add a `CHANGELOG.md` entry. Repository policy requires fix/feat/perf changes that users can observe to be recorded with contributor credit.
