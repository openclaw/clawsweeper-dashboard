## Summary

Keep open. The Feishu truncation bug is still not fixed on current main, but this PR is not mergeable as-is because the branch has blocking sanitizer/fallback regressions, no after-fix Feishu/Card Kit proof, and conflicts with newer Feishu streaming changes.

## What This Changes

The PR adds Feishu streaming-card markdown sanitization, strips reasoning tags from streaming partials, adds sanitizer tests, and routes markdown-card replies through post messages when replying.

## Best Possible Solution

Rebase and narrow the PR so Card Kit sanitization happens only at outbound update/close/summary boundaries, preserves raw stream state and current reply fallback flags, adds focused regression tests, and includes redacted Feishu proof.

## Reproduction Assessment

Yes, at source level. Current main sends raw Card Kit markdown through the streaming update/close path, and the linked report gives concrete backtick and `<thinking>` inputs; I did not run a live Feishu session.

## Solution Assessment

No. The direction is useful, but the current branch sanitizes canonical stream state too early, misses reported tag-shaped and multi-backtick cases, drops a current fallback flag, and lacks real Feishu proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep raw stream state before sanitizing output:** `extensions/feishu/src/streaming-card.ts:398-400`
  - body: `update()` sanitizes `text` before `mergeStreamingText()` stores it as pending/current state. `close(finalText)` later merges raw final text into that escaped state, so a case like `x < 10` can be duplicated as escaped-plus-raw text and reintroduce raw Card Kit markdown on the final update; keep state raw and sanitize only outbound payloads.
  - confidence: 0.92
- **[P2] Escape unsafe tag-shaped text:** `extensions/feishu/src/streaming-card.ts:73-77`
  - body: The angle-bracket regex only escapes a non-whitelisted `<` when there is no later `>` before the line end, so tag-shaped text like `<thinking>` remains unescaped. That is one of the reported truncation inputs, so the sanitizer can still leave Card Kit parsing it as a tag.
  - confidence: 0.86
- **[P2] Preserve multi-backtick code spans before escaping:** `extensions/feishu/src/streaming-card.ts:69-72`
  - body: The code-span placeholder only matches single-backtick spans. Valid Markdown spans delimited by multiple backticks remain exposed, so inline samples containing `x < y` can be rewritten with a backslash inside the code text.
  - confidence: 0.84
- **[P2] Forward the reply fallback flag through post replies:** `extensions/feishu/src/send.ts:716-724`
  - body: The new reply branch calls `sendMessageFeishu` without forwarding `allowTopLevelReplyFallback`. Current main uses that flag to allow specific missing-thread fallbacks, so rebasing this PR as-is would make previously allowed recovery paths fail.
  - confidence: 0.78
