## Summary

Keep open. Current main still lacks the Feishu Card Kit markdown sanitizer, and this PR targets a real Feishu streaming truncation bug, but the current branch has blocking sanitizer/state bugs plus a missing user-facing changelog entry.

## What This Changes

The PR adds Feishu streaming-card markdown sanitization and sanitizer tests, strips reasoning tags from partial streaming updates, and falls markdown-card replies back to post format when replying.

## Best Possible Solution

Revise the Feishu plugin patch so streaming state uses one canonical representation, unsafe tag-shaped text is escaped without breaking allowed Feishu tags, all valid Markdown code span forms are protected, and the user-facing fix has focused tests plus a changelog entry.

## Reproduction Assessment

Yes. A read-only sanitizer probe against the PR logic reproduces the remaining failures for partial/final 'x < 10', '<thinking>' text, and multi-backtick code spans.

## Solution Assessment

No. The PR direction is plausible, but sanitizing only update() before state merge is not the safest implementation; keeping raw state and sanitizing at the Card Kit output boundary, or sanitizing all state transitions consistently, is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Sanitize final text before merging stream state:** `extensions/feishu/src/streaming-card.ts:399`
  - body: update() stores sanitized content in state.currentText, but close(finalText) still merges raw finalText. For x < 10, the partial state becomes x \\< 10, the raw final no longer overlaps, and close sends duplicated unsanitized content.
  - confidence: 0.92
- **[P2] Escape tag-shaped unsafe angle brackets:** `extensions/feishu/src/streaming-card.ts:74`
  - body: The angle-bracket regex only catches '<' runs without a later '>' on the same line. Tag-shaped text such as <thinking> is not whitelisted but remains unescaped, so the reported Card Kit truncation case can still parse as a tag.
  - confidence: 0.86
- **[P2] Preserve multi-backtick code spans before escaping:** `extensions/feishu/src/streaming-card.ts:69`
  - body: The code-span placeholder only handles single-backtick spans. Valid Markdown spans delimited by multiple backticks leave their contents exposed, so examples like ``x < y`` are rewritten with an inserted backslash.
  - confidence: 0.84
- **[P3] Add the Feishu fix to the changelog:** `extensions/feishu/src/streaming-card.ts:35`
  - body: This is a user-facing Feishu delivery fix, but the PR branch has no CHANGELOG.md entry for the streaming truncation behavior. Repository policy requires user-visible fixes to be recorded.
  - confidence: 0.87
