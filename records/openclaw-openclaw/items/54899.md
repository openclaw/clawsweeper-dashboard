Overall confidence: 0.88

Full review comments:

- **[P1] Forward reply fallback permission through post replies:** `extensions/feishu/src/send.ts:716-724`
  - body: The new reply branch calls `sendMessageFeishu` without forwarding `allowTopLevelReplyFallback`. Current main carries that flag through Feishu reply sends so withdrawn thread reply targets can recover when explicitly allowed; after rebase this path would fail closed unless the flag is preserved.
  - confidence: 0.9
- **[P1] Keep raw stream state separate from sanitized output:** `extensions/feishu/src/streaming-card.ts:398-400`
  - body: `update()` sanitizes `text` before it is merged into `state.currentText`, but `close(finalText)` later merges the raw final snapshot. For content like `x < 10`, the escaped partial and raw final can stop overlapping, causing duplicated text and a final raw `<` update.
  - confidence: 0.88
- **[P2] Escape unsupported tag-shaped text before Card Kit parsing:** `extensions/feishu/src/streaming-card.ts:73-77`
  - body: The angle-bracket regex only escapes a `<...` run when it reaches end-of-line without `>`, so tag-shaped text such as `<thinking>` is left raw outside code. The linked truncation report specifically cites this shape, so the sanitizer still misses a central case.
  - confidence: 0.84
- **[P2] Preserve multi-backtick inline code spans:** `extensions/feishu/src/streaming-card.ts:69-71`
  - body: Code-span masking only protects single-backtick spans. Valid Markdown spans delimited by multiple backticks can still have `<` escaped inside the code sample, visibly corrupting generated code in streaming cards.
  - confidence: 0.78
