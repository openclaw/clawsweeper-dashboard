## Summary

Keep open: current main still lacks Feishu Card Kit markdown sanitization, but the latest PR head has blocking sanitizer defects and no real Feishu/Card Kit after-fix proof.

## What This Changes

The PR adds Feishu Card Kit markdown sanitization/tests, strips reasoning tags from streamed partials, and routes Feishu markdown-card replies through post-format messages when replying.

## Best Possible Solution

Preserve raw streaming state, sanitize only outbound Card Kit update/close/summary payloads, cover unsafe tag-shaped text and multi-backtick spans, then collect redacted Feishu proof.

## Reproduction Assessment

Yes. Source inspection shows current main sends raw Card Kit markdown, and the PR blockers follow directly from the sanitizer regexes plus the update/close merge path.

## Solution Assessment

No. The PR direction is useful, but the maintainable fix is to keep canonical stream state raw and sanitize consistently at the Feishu Card Kit output boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep raw stream state before sanitizing output:** `extensions/feishu/src/streaming-card.ts:399`
  - body: `update()` sanitizes snapshots before merging them into `state.currentText`, but `close(finalText)` later merges the raw final text. For input like `x < 10`, the displayed partial can become `x \< 10`, stop overlapping the raw final, and then send duplicated or unsanitized final content. Keep stream state raw and sanitize only outbound Card Kit payloads, or apply the same canonicalization before both merges.
  - confidence: 0.92
- **[P2] Escape unsafe tag-shaped text:** `extensions/feishu/src/streaming-card.ts:74`
  - body: The angle-bracket regex only escapes `<` when there is no later `>` on the line. Tag-shaped text such as `<thinking>` is not whitelisted but remains unescaped, so the reported Card Kit truncation class can still be parsed as a tag.
  - confidence: 0.86
- **[P2] Preserve multi-backtick code spans before escaping:** `extensions/feishu/src/streaming-card.ts:69`
  - body: The code-span placeholder only matches single-backtick spans. Valid Markdown spans delimited by multiple backticks leave their contents exposed, so examples like ``x < y`` are rewritten with an inserted backslash inside code.
  - confidence: 0.84
