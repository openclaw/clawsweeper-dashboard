## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Forward fallback permission through post replies:** `extensions/feishu/src/send.ts:716-724`
  - body: In the new reply branch, `sendMessageFeishu` is called without `allowTopLevelReplyFallback`. Current main uses that flag to let normal quoted group replies recover with a top-level send when the reply target is missing or withdrawn, so this route can fail closed for setups that currently recover.
  - confidence: 0.9
- **[P1] Keep raw stream state before sanitizing output:** `extensions/feishu/src/streaming-card.ts:395-400`
  - body: Sanitizing before `mergeStreamingText` stores escaped text in `state.currentText`/`sentText`, but `close(finalText)` still receives raw final snapshots. For a stream like `x < 10`, the escaped partial and raw final no longer overlap cleanly, so the final close can append duplicate text and reintroduce the raw `<` that this PR is trying to protect.
  - confidence: 0.88
- **[P2] Escape tag-shaped Card Kit text:** `extensions/feishu/src/streaming-card.ts:73-77`
  - body: The angle-bracket regex only matches a `<...` run that reaches the end of the line without a closing `>`, so tag-shaped text such as `<thinking>` is left raw. The linked truncation report specifically involves this shape, so the sanitizer still misses a central failing case.
  - confidence: 0.84
- **[P2] Preserve multi-backtick inline code spans:** `extensions/feishu/src/streaming-card.ts:68-71`
  - body: Code-span masking only protects single-backtick spans. Valid Markdown spans delimited with multiple backticks can still have `<` escaped inside the code sample, which visibly corrupts generated code in streaming cards.
  - confidence: 0.78
