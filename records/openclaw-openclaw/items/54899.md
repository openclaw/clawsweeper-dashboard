## Summary

Keep this PR open, but do not merge it as-is. Current main still lacks a Feishu Card Kit markdown sanitizer, so the PR is not obsolete, but the branch still has concrete sanitizer and rebase regressions plus no real Feishu/Card Kit behavior proof.

## What This Changes

The PR adds Feishu Card Kit markdown sanitization for streaming updates, summary cleanup, a markdown-card reply fallback to post messages, and sanitizer tests.

## Best Possible Solution

Keep raw streaming state canonical, sanitize only outbound Feishu Card Kit update/close/summary payloads, cover unsafe tag-shaped text and multi-backtick spans, preserve current reply fallback flags, and then collect redacted Feishu proof.

## Reproduction Assessment

Yes. Source inspection shows current main sends raw Feishu Card Kit markdown, and the linked issue gives concrete backtick and `<thinking>` inputs that target that raw output path.

## Solution Assessment

No. The PR direction is useful, but the current branch sanitizes too early by mutating stream state and does not cover unsafe tag-shaped text or multi-backtick code spans.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep raw stream state before sanitizing output:** `extensions/feishu/src/streaming-card.ts:398-399`
  - body: `update()` sanitizes the incoming snapshot before the merged value is stored in `state.currentText`. Later `close(finalText)` merges raw final text into that sanitized state, so a partial like `x < 10` can become `x \< 10`, fail to overlap the raw final text, and resend duplicated or unsafe content. Keep state raw and sanitize only the outbound Card Kit payloads.
  - confidence: 0.92
- **[P2] Escape unsafe tag-shaped text:** `extensions/feishu/src/streaming-card.ts:73-77`
  - body: The angle-bracket regex only escapes `<` when there is no later `>` before the line end, so non-whitelisted tag-shaped text like `<thinking>` remains unescaped. That leaves the reported Card Kit truncation class still parseable as a tag.
  - confidence: 0.86
- **[P2] Preserve multi-backtick code spans before escaping:** `extensions/feishu/src/streaming-card.ts:68-72`
  - body: The code-span placeholder only matches single-backtick spans. Valid Markdown spans delimited by multiple backticks remain exposed, so inline code such as double-backtick-delimited `x < y` can be rewritten with a backslash inside the code sample.
  - confidence: 0.84
- **[P2] Forward the reply fallback flag through post replies:** `extensions/feishu/src/send.ts:716-724`
  - body: Current main lets Feishu thread replies fall back to a top-level send when `allowTopLevelReplyFallback` is true. The new reply branch routes through `sendMessageFeishu` but does not pass that flag, so rebasing this change as-is would regress markdown-card replies when the quoted thread target is unavailable.
  - confidence: 0.78
