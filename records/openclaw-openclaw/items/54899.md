## Summary

Keep this PR open, but it is not mergeable yet: current main still lacks Feishu Card Kit markdown sanitization, while the PR branch leaves concrete sanitizer regressions and a required changelog entry unresolved.

## What This Changes

The PR adds Feishu Card Kit markdown sanitization and tests, strips reasoning tags from Feishu partial updates, and routes markdown-card replies with a reply target through post-format messages.

## Best Possible Solution

Keep the Feishu plugin fix, but make Card Kit output and streaming state use one canonical sanitized representation, cover unsafe tag-shaped text and valid multi-backtick spans, and add the user-facing changelog entry.

## Reproduction Assessment

Yes for the PR blockers: a read-only sanitizer probe reproduces duplicated final text for x < 10, unescaped <thinking> text, and corruption of multi-backtick code spans. The underlying current-main truncation bug is source-reproducible because streaming card updates still send raw markdown content.

## Solution Assessment

No. The PR direction is plausible, but sanitizing only update input while close merges raw final text is not the narrowest maintainable fix; sanitization should be consistent at the Feishu Card Kit output/state boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Sanitize final text before merging stream state:** `extensions/feishu/src/streaming-card.ts:399`
  - body: update() stores sanitized content in state.currentText, but close(finalText) still merges the raw final text. For x < 10, the partial state becomes x \\< 10, the raw final no longer overlaps, and close sends duplicated unsanitized content.
  - confidence: 0.92
- **[P2] Escape tag-shaped unsafe angle brackets:** `extensions/feishu/src/streaming-card.ts:74`
  - body: The angle-bracket regex only catches < runs without a later > on the same line. Tag-shaped text such as <thinking> is not whitelisted but remains unescaped, so the reported Card Kit truncation case can still parse as a tag.
  - confidence: 0.86
- **[P2] Preserve multi-backtick code spans before escaping:** `extensions/feishu/src/streaming-card.ts:69`
  - body: The code-span placeholder only handles single-backtick spans. Valid Markdown spans delimited by multiple backticks leave their contents exposed, so examples like ``x < y`` are rewritten with an inserted backslash.
  - confidence: 0.84
- **[P3] Add the Feishu fix to the changelog:** `extensions/feishu/src/streaming-card.ts:35`
  - body: This is a user-facing Feishu delivery fix, but the PR branch has no CHANGELOG.md entry for the streaming truncation behavior. Repository policy requires user-visible fixes to be recorded.
  - confidence: 0.87
