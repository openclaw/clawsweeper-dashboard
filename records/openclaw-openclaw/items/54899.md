## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep raw stream state before sanitizing output:** `extensions/feishu/src/streaming-card.ts:398-400`
  - body: `update()` sanitizes `text` before it is merged into `state.currentText`, but `close(finalText)` later merges raw final text into that escaped state. A stream like `x < 10` can become escaped partial text plus raw final text, causing duplication or reintroducing the raw Card Kit markdown this PR is trying to avoid; keep state raw and sanitize only outbound payloads.
  - confidence: 0.92
- **[P2] Escape tag-shaped text that Card Kit can parse:** `extensions/feishu/src/streaming-card.ts:73-77`
  - body: The angle-bracket regex only escapes a non-whitelisted `<` when no later `>` appears before line end, so the reported `<thinking>` shape is left untouched. That still lets Card Kit parse the text as a tag and can preserve the truncation bug for the exact class of input this PR cites.
  - confidence: 0.86
- **[P2] Mask multi-backtick spans before escaping:** `extensions/feishu/src/streaming-card.ts:69-72`
  - body: The placeholder pass only protects single-backtick spans. Valid Markdown spans delimited with multiple backticks remain exposed, so inline code containing `x < y` can be rewritten with a backslash inside the code sample and visibly corrupt streamed output.
  - confidence: 0.84
- **[P1] Forward fallback permission through post replies:** `extensions/feishu/src/send.ts:716-724`
  - body: Current main has an opt-in `allowTopLevelReplyFallback` contract for missing or withdrawn Feishu thread targets, but this new reply branch calls `sendMessageFeishu` without forwarding that flag. Rebasing this as-is would make previously allowed recovery paths fail at runtime.
  - confidence: 0.82
