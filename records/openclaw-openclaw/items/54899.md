## What This Changes

The branch adds Feishu Card Kit markdown sanitization and tests, strips reasoning tags from streamed partials, and routes Feishu markdown-card replies through post-format messages when replying.

## Best Possible Solution

Keep raw streaming state canonical, sanitize only outbound Feishu Card Kit update/close/summary payloads, cover unsafe tag-shaped text and multi-backtick spans, preserve current reply fallback flags, and then collect redacted Feishu proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main sends raw Feishu Card Kit markdown, and the PR blockers follow from the sanitizer regexes plus the update/close merge path at PR head.

## Solution Assessment

No. The PR direction is useful, but the maintainable fix is to sanitize consistently at the Feishu Card Kit output boundary instead of mutating canonical stream state.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep raw stream state before sanitizing output:** `extensions/feishu/src/streaming-card.ts:399`
  - body: `update()` sanitizes partial snapshots before storing them in `state.currentText`, but `close(finalText)` later merges raw final text into that sanitized state. If a partial contains `x < 10`, the stored text can become `x \< 10`, no longer overlap the raw final text, and the close path can duplicate or resend unsafe content. Keep stream state raw and sanitize only the outbound Card Kit payloads.
  - confidence: 0.92
- **[P2] Escape unsafe tag-shaped text:** `extensions/feishu/src/streaming-card.ts:74`
  - body: The angle-bracket regex only escapes `<` when there is no later `>` on the line, so non-whitelisted tag-shaped text like `<thinking>` stays unescaped. That leaves the exact reported Card Kit truncation class still parseable as a tag.
  - confidence: 0.86
- **[P2] Preserve multi-backtick code spans before escaping:** `extensions/feishu/src/streaming-card.ts:69`
  - body: The code-span placeholder only matches single-backtick spans. Valid Markdown spans delimited by multiple backticks remain exposed, so inline code such as double-backtick-delimited `x < y` can be rewritten with an inserted backslash inside the code sample.
  - confidence: 0.84
- **[P2] Forward the reply fallback flag through post replies:** `extensions/feishu/src/send.ts:716-724`
  - body: Current main lets Feishu replies fall back to a top-level send when the quoted thread target is missing. This new reply branch routes through `sendMessageFeishu` but does not forward `allowTopLevelReplyFallback`, so rebasing this change as-is would regress that recovery path for markdown-card replies.
  - confidence: 0.78
