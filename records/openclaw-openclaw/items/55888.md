## Repair Work Prompt

Add a narrow documentation profile for high-memory local Ollama compaction tuning based on https://github.com/openclaw/openclaw/issues/55888. Keep runtime defaults unchanged. Explain that OpenClaw's preflight threshold is contextWindow - reserveTokensFloor - memoryFlush.softThresholdTokens, so a 32,768-token model defaults to 8,768 with the current 20,000 reserve and 4,000 soft threshold, while an opt-in high-memory profile such as reserveTokensFloor 2000 and keepRecentTokens 20000 yields about 26,768 before preflight. Document that the config belongs under agents.defaults.compaction, that low-RAM users should not copy it blindly, that larger local models may need a larger reserve, and that users must apply config through the supported config/restart path rather than hand-copying agent snapshot files. Prefer docs/reference/session-management-compaction.md plus a short pointer from docs/providers/ollama.md or docs/gateway/local-models.md if that reads naturally. Do not add new config keys, change runtime threshold defaults, or alter compaction behavior. No changelog is expected for docs-only guidance unless maintainers ask.
