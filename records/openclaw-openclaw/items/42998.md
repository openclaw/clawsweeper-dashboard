## Summary

Close as mostly implemented on current main: the duplicate raw-stdout/empty-payload rerun path was fixed and shipped by https://github.com/openclaw/openclaw/pull/41401, while this branch’s remaining transcript-scan output recovery is risky and not the right contract to merge.

## What This Changes

The PR adds raw-stdout prompt detection, transcript toolResult text recovery, retry suppression for those cron requests, and a regression test.

## Best Possible Solution

Keep the shipped no-rerun behavior on main; if raw toolResult output is still wanted, design it as typed current-attempt runner state instead of reviving this stale transcript-scanning branch.

## Reproduction Assessment

Yes, by source inspection: the original rerun condition was empty/NO_REPLY output being treated as interim, and current main now returns false for empty text before the retry gate. I did not run a live cron job because this review was read-only.

## Solution Assessment

No. The shipped empty/NO_REPLY fix is the narrower solution for the duplicate rerun, while this PR’s prompt keyword detection and transcript-file scanning are not the safest way to expose raw tool output.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Gate retry suppression on recovered tool output:** `src/cron/isolated-agent/run.ts:730`
  - body: The raw-stdout guard disables the interim-ack follow-up based on prompt text before a successful tool result has been recovered. A raw-stdout job that first replies with an acknowledgement and no captured tool result can now exit incomplete instead of running the continuation that completes the task.
  - confidence: 0.92
- **[P2] Reject tool results with unverified timestamps:** `src/cron/isolated-agent/run.ts:226-228`
  - body: The transcript reader only skips entries when a timestamp parses and is older than runStartedAt, so missing or unparsable timestamps are accepted. That can promote stale transcript output as this run's cron result; require verified current-run state or a valid timestamp inside the run window.
  - confidence: 0.83
- **[P3] Use an isolated current-run transcript fixture:** `src/cron/isolated-agent/run.interim-retry.test.ts:102-109`
  - body: The test writes a fixed /tmp/transcript.jsonl before the runner captures runStartedAt, so clock advancement can exclude the fixture and the path can collide across tests. Use a per-test path and a timestamp guaranteed to be in the run window.
  - confidence: 0.86
