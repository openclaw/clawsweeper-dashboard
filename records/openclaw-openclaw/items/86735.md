## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Escape approval IDs when replacing fallback commands:** `src/plugin-sdk/approval-reaction-runtime.ts:353`
  - body: This helper now builds the manual fallback payload used by migrated channels. The removed iMessage helper escaped `$` before `String.prototype.replace`; without that, IDs containing `$&`, `$$`, `$\`` or `$'` are rewritten by the replacement engine, so the `/approve` fallback text can point at the wrong approval id. Preserve the escaping and add a shared regression test.
  - confidence: 0.87
- **[P2] Sanitize untrusted approval prompt fields:** `src/plugin-sdk/approval-reaction-runtime.ts:256-258`
  - body: GitHub Advanced Security flagged this new prompt builder for unsanitized literal interpolation. The helper imports `sanitizeForPromptLiteral` but only applies it to `cwd`; plugin title/description and exec metadata are still copied into the approval prompt with control/newline characters intact. Before exporting this SDK subpath, sanitize or wrap the untrusted fields, or document why the flagged dataflow is unreachable.
  - confidence: 0.74
