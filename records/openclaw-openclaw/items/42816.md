## Summary

Keep open. Current main has durable gateway restart/update continuations and task/Task Flow persistence, but it still does not implement the requested generic checkpoint, expected-outcome verification, and proactive reporting workflow for arbitrary host reboots or process-killing commands.

## What This Changes

Request persistent task context, expected-outcome checks, and proactive user reporting after OpenClaw or the host restarts during configuration work.

## Best Possible Solution

Define an authorized durable continuation contract on top of restart sentinels, the session-delivery queue, task registry, and Task Flow that records route, restart kind, expected checks, expiry, audit data, and bounded verification semantics.

## Reproduction Assessment

Not applicable as a feature request. Current-main source and docs show partial gateway restart/update continuations and durable task/flow state, but no generic checkpoint plus expected-outcome verifier for arbitrary rebooting work.

## Solution Assessment

No. A standalone local file/database checkpoint would be too ad hoc; the maintainable path is to extend the existing restart sentinel, delivery queue, task registry, and Task Flow contracts with explicit authorization and verification rules.
