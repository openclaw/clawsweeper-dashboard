## What This Changes

The PR adds a default-off Gateway `/v1/responses` config flag that emits built-in tool-start events as audit-only `function_call` items in JSON/SSE output, with schema, docs, changelog, and regression tests.

## Best Possible Solution

Land this PR or a maintainer-approved equivalent once CI is green, preserving default-off, run-scoped, sanitized, start-event-only audit semantics.

## Reproduction Assessment

Yes. Source inspection shows current main only returns assistant messages or caller-delegated `pendingToolCalls` on `/v1/responses`, while run-scoped tool-start events are emitted elsewhere and ignored by the endpoint.

## Solution Assessment

Yes for the proposed direction. The default-off flag is a narrow additive API/config change; tool-result capture and caller-delegated built-ins remain separate product decisions.
