## Summary

Keep open: current main still lacks the opt-in built-in tool-call audit surface, and this PR implements the requested Gateway Responses feature with focused config, JSON/SSE handling, docs, changelog, and regression coverage; one low-severity naming mismatch remains for normalized shell calls.

## What This Changes

The PR adds a default-off `gateway.http.endpoints.responses.exposeBuiltInToolCalls` option that surfaces built-in agent tool-start events as audit-only `function_call` items in `/v1/responses` JSON and SSE responses.

## Best Possible Solution

Land this PR or an equivalent change after maintainer review confirms the new config/API contract and aligns the shell tool-name semantics.

## Reproduction Assessment

Yes, source-level: current main has no config flag and both JSON and SSE `/v1/responses` paths ignore run-scoped built-in `tool/start` events. I did not live-run the endpoint because this sweep is read-only.

## Solution Assessment

Yes for the implementation shape, with maintainer approval for the new surface. The default-off, run-scoped, sanitized, start-only forwarding is narrow; the remaining API question is the normalized shell tool name.

## Review Findings

Overall correctness: patch is correct

Overall confidence: 0.86

Full review comments:

- **[P3] Align documented shell names with emitted events:** `docs/gateway/openresponses-http-api.md:128`
