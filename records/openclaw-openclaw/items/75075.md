## What This Changes

The PR adds a default-off Gateway Responses config flag, schema/docs/changelog updates, and JSON/SSE handling/tests to append built-in tool-start events as `function_call` output items.

## Best Possible Solution

Land a default-off, authenticated, start-only audit surface after the Responses wire contract documents the actual emitted tool names and redaction expectations.

## Reproduction Assessment

Yes, at source level. Current main emits run-scoped sanitized `tool/start` events, but `/v1/responses` only serializes assistant output and caller-provided `pendingToolCalls`; I did not live-run the endpoint in this read-only review.

## Solution Assessment

Unclear until maintainer API review. The default-off `function_call` audit shape is narrow and plausible, but shell-name normalization and argument exposure should be settled before merge.
