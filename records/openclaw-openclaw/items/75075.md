## What This Changes

The PR adds a default-off Gateway responses flag to expose sanitized built-in tool-start events as `/v1/responses` JSON/SSE `function_call` output items, with config schema, docs, changelog, and e2e coverage.

## Best Possible Solution

Land a maintainer-approved version after exact-head checks are green, preserving default-off behavior, sanitized run-scoped tool-start capture, client-tool delegate dedupe, and the documented ordering contract.

## Reproduction Assessment

Yes. Emit a run-scoped `stream: "tool"`, `phase: "start"` event during a `/v1/responses` turn; current main has no output path for that event, while the PR tests cover default-off, JSON, SSE, and dedupe behavior.

## Solution Assessment

Mostly yes. Reusing the existing sanitized agent event stream behind an opt-in flag is the narrowest maintainable path; the remaining gate is maintainer API-semantics approval plus green exact-head CI.
