## Summary

This PR is still useful but not merge-ready: current main does not expose built-in agent tool calls through `/v1/responses`, but the patch represents already-executed built-in tools as ordinary `function_call` items that compatible clients can treat as executable tool requests, and the PR still lacks real gateway JSON/SSE proof.

## What This Changes

The branch adds `gateway.http.endpoints.responses.exposeBuiltInToolCalls`, captures built-in agent tool-start events, emits them as `/v1/responses` `function_call` output items in JSON and SSE, and updates config schema, docs, tests, generated config baselines, and `CHANGELOG.md`.

## Best Possible Solution

Keep the audit feature opt-in/default-off, but land it only after maintainers choose a non-actionable output contract or explicitly accept `function_call` semantics, define the security boundary, remove release-owned changelog churn, rebase, and add redacted real gateway JSON/SSE proof.
