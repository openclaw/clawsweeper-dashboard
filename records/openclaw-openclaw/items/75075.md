## Summary

Keep this PR open, but do not merge it as-is. Current main still has the built-in tool visibility gap, and the PR's opt-in audit path is directionally sound, but the new client-tool dedupe compares raw request names against normalized tool event names and can emit duplicate function_call items.

## What This Changes

The branch adds a default-off Gateway /v1/responses config flag that surfaces run-scoped built-in tool-start events as Responses function_call output items, with config schema, docs, changelog, and e2e coverage.

## Best Possible Solution

Land a maintainer-approved version after fixing normalized client-tool dedupe, preserving default-off behavior, sanitized run-scoped start-only capture, stable ordering, and the documented audit-only contract.

## Reproduction Assessment

Yes. A /v1/responses run that emits a run-scoped stream: "tool", phase: "start" event reproduces the current main visibility gap, and the PR's tests exercise default-off/on JSON and SSE behavior. The introduced dedupe bug is reproducible with a client tool name whose emitted normalized form differs from the raw request name, such as case differences.

## Solution Assessment

No, not as-is. The opt-in sanitized event capture is the narrow maintainable direction, but the dedupe filter should compare names normalized with the same tool-name semantics as the event source before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Normalize client tool names before deduping:** `src/gateway/openresponses-http.ts:744-750`
  - body: The tool-start event emitted by `handleToolExecutionStart` normalizes the tool name before publishing it, but this set stores raw client tool names from the request. For a valid client tool whose raw name differs from the emitted normalized form, such as by case, the audit branch will not recognize it as a delegated client tool and can emit both an audit `function_call` and the normal delegate `function_call`. Build the set with the same normalization used by the event source and add a regression test for a non-identical raw/emitted name.
  - confidence: 0.82
