## Summary

Keep open. The PR addresses a real current-main gap with a focused default-off `/v1/responses` feature and the latest head checks are green, but it adds a new config/API surface and still needs maintainer decision on the shell tool-name contract (`exec` vs documented `bash`).

## What This Changes

The PR adds a default-off Gateway `/v1/responses` config flag to include built-in agent tool-start events as audit-only `function_call` output items in JSON/SSE, with docs, config schema, tests, and changelog updates.

## Best Possible Solution

Land this PR or an equivalent change after maintainer review accepts the opt-in API and the tool-name contract is made internally consistent.

## Reproduction Assessment

Yes for the source-level gap: current main lacks the config flag and both JSON and SSE `/v1/responses` paths ignore run-scoped built-in `tool/start` events. I did not live-run the endpoint in this read-only sweep.

## Solution Assessment

Yes for the implementation shape, pending maintainer API approval. The default-off, run-scoped, sanitized, start-only forwarding is narrow; the shell naming mismatch should be resolved before merge.
