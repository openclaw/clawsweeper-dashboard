## Summary

Keep this PR open for maintainer review. Current main still lacks the requested /v1/responses built-in tool-call audit surface, and the branch appears to be a narrow opt-in implementation, but it adds public Gateway response semantics and exact-head CI was not fully clean at inspection.

## What This Changes

The PR adds a default-off Gateway responses config flag that captures sanitized run-scoped built-in tool-start events as JSON/SSE `function_call` output items, with schema, docs, changelog, and e2e coverage.

## Best Possible Solution

Land a maintainer-approved version after CI is green, keeping the behavior opt-in, sourced from sanitized run-scoped tool-start events, preserving caller-provided client-tool delegation, and documenting the JSON/SSE ordering contract.

## Reproduction Assessment

Yes. A focused reproduction is to emit a run-scoped `stream: "tool"`, `phase: "start"` event during a `/v1/responses` turn; current main has no output path for that event, while the PR tests model the JSON and SSE paths.

## Solution Assessment

Mostly yes. The proposed path is additive and narrow because it reuses the existing sanitized agent event stream instead of changing PI runtime or plugin SDK behavior; the remaining question is maintainer acceptance of the public API semantics and final CI.
