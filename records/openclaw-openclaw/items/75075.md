## Summary

Keep this PR open: current main still drops built-in `/v1/responses` tool-start events, and the branch is a useful implementation candidate with focused tests and docs. ## What This Changes

The PR adds a default-off Gateway Responses config flag that emits sanitized built-in tool-start events as audit-only `function_call` output items in `/v1/responses` JSON and SSE, with schema, docs, tests, and changelog updates.

## Best Possible Solution

Land this PR or an equivalent change after maintainer approval, rebasing on current main and aligning the documented shell tool name with the emitted normalized name.

## Reproduction Assessment

Yes, source-level: current main has no config flag and both JSON and SSE `/v1/responses` paths ignore run-scoped `tool/start` events. I did not live-run the endpoint in this read-only sweep, but the source path and PR tests give high confidence.

## Solution Assessment

Yes, subject to maintainer API approval and the shell-name doc alignment. The default-off, run-scoped, sanitized, start-only forwarding is a narrow maintainable shape; tool results and built-in delegation are correctly left out of scope.
