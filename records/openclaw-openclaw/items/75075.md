## What This Changes

The PR adds a default-off Gateway Responses config flag, schema/docs/changelog updates, and JSON/SSE tests to surface built-in agent tool-start events as `function_call` output items.

## Best Possible Solution

Land this PR or an equivalent after rebase and maintainer API review, keeping the feature default-off, run-scoped, sanitized, and start-only while documenting the emitted tool names accurately.

## Reproduction Assessment

Yes, source-level. Current main emits run-scoped built-in tool start events, but `/v1/responses` only serializes assistant output and caller-provided `pendingToolCalls`; I did not live-run the endpoint because this review is read-only.

## Solution Assessment

Unclear until maintainer API review. The default-off `function_call` audit shape is narrow and plausible, but maintainers should confirm shell-name normalization, argument exposure, and the new config contract before merge.
