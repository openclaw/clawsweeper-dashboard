## Summary

Keep open. Current main still lacks REST-native session reset/status/compact/model/reasoning endpoints for HTTP API integrations; https://github.com/openclaw/openclaw/pull/29985 only covered plugin SDK reset, and https://github.com/openclaw/openclaw/pull/23941 plus https://github.com/openclaw/openclaw/pull/52307 are closed unmerged.

## What This Changes

Request REST-native HTTP endpoints for session reset, status, compaction, model override, and reasoning control for clients using `/v1/chat/completions`.

## Best Possible Solution

Expose a documented authenticated `/v1/sessions` management surface over the existing session RPC/schema behavior while keeping `/v1/chat/completions` slash-command default-deny intact and coordinating read-only shape with https://github.com/openclaw/openclaw/issues/47826.

## Reproduction Assessment

Yes. Source inspection on current main shows no REST session-management route or handler, so `/v1/sessions/*` management paths fall through to the HTTP 404 tail while `/v1/chat/completions` still builds agent input without `CommandAuthorized`.

## Solution Assessment

Yes. A small authenticated HTTP transport over the existing session RPC/schema operations is the narrow maintainable direction; re-enabling command interception in `/v1/chat/completions` would cross the current security boundary and is not the requested fix.
