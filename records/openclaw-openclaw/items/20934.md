## Summary

Keep open: current main still lacks the requested REST-native session reset/status/compact/model/reasoning endpoints, while related plugin-SDK and broader REST-query work do not replace this specific HTTP API request.

## What This Changes

The issue requests authenticated REST session-management endpoints for HTTP integrations to reset, inspect, compact, and update model or reasoning state by session key.

## Best Possible Solution

Define a small documented authenticated Gateway HTTP session-management API over the existing session RPC/schema operations while preserving default-deny slash-command behavior for /v1/chat/completions.

## Reproduction Assessment

Yes. Source inspection on current main shows no matching /v1/sessions/* management handler or request stage, so those paths fall through to the HTTP 404 tail; I did not start the gateway in this read-only review.

## Solution Assessment

Yes directionally, but not as an automatic repair yet. The maintainable path is an authenticated REST wrapper over existing session RPC/schema behavior, not command interception in /v1/chat/completions, and maintainers should approve the public API/auth contract first.
