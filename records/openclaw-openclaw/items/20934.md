## Summary

Keep open. Current main still lacks the requested REST-native session control endpoints, and the related plugin-SDK reset work plus the closed unmerged REST PR do not satisfy HTTP API integrations.

## What This Changes

Expose authenticated REST endpoints for HTTP API clients to reset, inspect, compact, and adjust OpenClaw sessions without re-enabling slash-command dispatch through `/v1/chat/completions`.

## Best Possible Solution

Define and implement a small documented Gateway HTTP session-management surface over the existing session RPC/schema operations, preserving the `/v1/chat/completions` slash-command default-deny boundary.

## Reproduction Assessment

Yes. Source inspection on current main shows the requested `/v1/sessions/*` routes and handler are absent, while unmatched HTTP paths fall through to 404 and chat completions still do not set `CommandAuthorized`.

## Solution Assessment

Yes, directionally, but the exact public API shape still needs maintainer approval. The narrow maintainable path is an authenticated REST wrapper over existing session RPC/schema behavior, not command interception in `/v1/chat/completions`.
