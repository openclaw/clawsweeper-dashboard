## Repair Work Prompt

Implement issue #20934 in openclaw/openclaw. Add a focused Gateway HTTP session-management surface for HTTP integrations, covering reset, status, compact, model, and reasoning by session key. Reuse existing `sessions.reset`, `sessions.describe`/`sessions.list` as appropriate for status, `sessions.compact`, and `sessions.patch` semantics instead of duplicating session business logic. Authenticate through the existing Gateway HTTP auth helpers and operator scope model; mutations should require the same admin-level authority as the corresponding RPC methods, while read-only status should use the existing read/status scope pattern. Preserve the current default-deny slash-command behavior on `/v1/chat/completions`; do not make `/new`, `/status`, or `/compact` text dispatch as commands on that endpoint. Add route/auth/request/response tests, include regression coverage for missing/invalid session keys and auth failures, update `docs/gateway/openai-http-api.md`, and add a user-facing changelog entry. Coordinate with but do not collapse this into the broader #47826 read-only REST request. Treat closed PR #23941 as useful prior art only; avoid its reviewed pitfalls around strict `sessions.list` params, broad path prefix matching, and unvalidated reset reasons.
