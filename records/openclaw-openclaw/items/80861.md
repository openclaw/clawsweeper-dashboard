## Summary

Keep open: current main has provider and internal reasoning-stream plumbing, but the OpenAI-compatible HTTP/API surfaces still do not expose a settled raw or normalized reasoning stream contract, and the remaining work needs maintainer API/privacy judgment rather than a narrow cleanup close.

## What This Changes

The issue requests restoring provider raw streaming passthrough, or an opt-in raw streaming mode, so DeepSeek, Claude, and other provider reasoning/thinking content can be observed in-stream through API/terminal paths.

## Best Possible Solution

Define an explicit opt-in, normalized reasoning stream contract for the approved API and terminal surfaces, keeping reasoning hidden by default and avoiding broad raw provider JSON broadcast.

## Reproduction Assessment

Yes, by source inspection. Provider transports already parse reasoning into internal thinking events, while the OpenAI-compatible HTTP handlers and schema/docs expose only assistant text/lifecycle output for the relevant API streams.

## Solution Assessment

Unclear as requested: raw passthrough by default is not clearly the best solution. The safer maintainable path is an explicit opt-in normalized reasoning stream/output contract with privacy gates per public surface.
