## Summary

Keep open: current main still normalizes provider reasoning internally and can emit `thinking` agent events, but the OpenAI-compatible HTTP streaming endpoints only forward visible assistant text and lifecycle events. Restoring raw or visible reasoning passthrough is an API and exposure-policy decision, not a conservative cleanup close.

## What This Changes

Expose provider raw reasoning/thinking stream content, or an opt-in raw streaming mode, through terminal/API paths.

## Best Possible Solution

Define documented opt-in normalized reasoning stream contracts for the approved API/client surfaces, keep reasoning hidden by default, and add provider-specific coverage only where replay and exposure semantics are understood.

## Reproduction Assessment

Yes, by source inspection: provider transports can produce internal thinking deltas, while `/v1/chat/completions` and `/v1/responses` only serialize assistant/lifecycle SSE. I did not run a live provider because the issue does not pin one exact caller surface and model configuration.

## Solution Assessment

Unclear. Raw passthrough by default is not the safest maintainable solution; a documented opt-in normalized reasoning stream is safer than dumping provider-specific raw JSON into every compatible client.
