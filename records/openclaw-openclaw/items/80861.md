## Summary

Keep open: current main has internal reasoning-stream plumbing and SDK typing, but the OpenAI-compatible API endpoints still forward only visible assistant text and lifecycle events, so restoring raw/visible reasoning passthrough is a new API and product-policy decision.

## What This Changes

Request to restore or add opt-in raw provider streaming passthrough so reasoning/thinking fields like `reasoning_content` and Claude thinking blocks can stream to terminal/API clients.

## Best Possible Solution

Define a documented opt-in reasoning stream contract per API surface, keeping reasoning hidden by default and adding focused DeepSeek/Claude coverage for whichever endpoints are approved.

## Reproduction Assessment

Yes, by source inspection rather than a live run: provider transports can emit internal thinking deltas, while the OpenAI-compatible HTTP streaming handlers only forward assistant/lifecycle events. The exact user-facing reproduction still depends on the caller surface and provider/model configuration.

## Solution Assessment

Unclear. Raw passthrough by default is not clearly the best solution; a safer maintainable path is an explicit opt-in normalized reasoning stream contract rather than dumping provider-specific raw JSON into every client.
