## Summary

Keep open: current main parses provider reasoning into internal thinking events, but the OpenAI-compatible HTTP streaming endpoints still forward only visible assistant text and lifecycle events. Exposing reasoning/raw provider chunks through those public API surfaces needs an explicit API and privacy policy decision, not a narrow cleanup close.

## What This Changes

The issue requests restoring raw or opt-in provider reasoning/thinking streaming passthrough for API/terminal users, including fields such as DeepSeek reasoning_content and Claude thinking blocks.

## Best Possible Solution

Define a documented opt-in normalized reasoning stream contract for the approved API surfaces, keep reasoning hidden by default, and add focused DeepSeek/Claude coverage once the public contract is chosen.

## Reproduction Assessment

Yes, by source inspection: provider transports can emit internal thinking deltas, while the OpenAI-compatible HTTP streaming handlers only forward assistant text and lifecycle events. A live end-to-end repro still depends on the caller surface and provider/model configuration.

## Solution Assessment

No for raw passthrough by default; dumping provider-specific raw JSON into public streams is not the safest maintainable fix. The better path is an explicit opt-in normalized reasoning stream contract with endpoint-specific tests.
