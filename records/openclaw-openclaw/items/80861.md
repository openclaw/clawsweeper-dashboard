## Summary

Keep open: current main parses provider reasoning into internal thinking and can emit normalized thinking events, but the OpenAI-compatible HTTP endpoints still serialize only assistant text and lifecycle events. The remaining work needs an explicit opt-in API/wire-format and reasoning-exposure policy decision, not conservative cleanup.

## What This Changes

The issue requests restoring provider raw reasoning/thinking stream passthrough, or adding an opt-in raw stream mode, for API/terminal clients.

## Best Possible Solution

Define a documented opt-in reasoning stream contract for approved API surfaces, keep reasoning hidden by default, and add focused tests for the selected Chat Completions/Responses wire format.

## Reproduction Assessment

Yes, at source level: provider transports can produce internal thinking deltas, while the OpenAI-compatible streaming handlers only forward assistant text/lifecycle events. I did not run a live provider/API reproduction in this read-only review.

## Solution Assessment

Unclear. Restoring raw provider passthrough by default is not clearly the best solution; an explicit opt-in normalized reasoning stream contract is safer than dumping provider-specific raw JSON to every client.
