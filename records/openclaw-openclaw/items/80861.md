## Summary

Keep open: current main has provider-side reasoning parsing and internal thinking events, but the OpenAI-compatible HTTP/API surfaces still do not expose reasoning content, and the remaining work is an opt-in API/privacy contract rather than a narrow bug fix.

## What This Changes

Request to restore raw provider reasoning/thinking stream passthrough, or add an opt-in raw streaming mode, for API/terminal clients.

## Best Possible Solution

Define a documented opt-in normalized reasoning stream contract for each approved Gateway/OpenAI-compatible surface, keep reasoning hidden by default, and coordinate the open Responses and WebSocket PRs around that policy.

## Reproduction Assessment

Yes. Source inspection shows provider transports produce internal thinking deltas while the OpenAI-compatible HTTP handlers only serialize assistant text and lifecycle events; a live provider run would only be needed to pin one model-specific symptom.

## Solution Assessment

Unclear. Raw passthrough by default is not the safest maintainable solution; an explicit opt-in, normalized reasoning stream contract is safer than dumping provider-specific raw JSON into every client.
