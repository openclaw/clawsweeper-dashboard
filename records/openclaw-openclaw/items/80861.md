## Summary

Keep open: current main preserves provider reasoning as internal thinking events and SDK event types, but the OpenAI-compatible HTTP streaming surfaces still emit only visible assistant text and lifecycle events. Exposing raw or normalized reasoning over those API paths is a new product/API/security decision, not a narrow bug cleanup.

## What This Changes

Request to restore or add opt-in raw provider reasoning/thinking stream passthrough for terminal and API clients.

## Best Possible Solution

Define a documented opt-in reasoning stream contract per approved API surface, keep reasoning hidden by default, and coordinate any implementation with the existing reasoning-stream PRs rather than dumping provider-specific raw JSON into all clients.

## Reproduction Assessment

Yes, by source inspection for the API paths: provider transports can produce internal thinking deltas, while the OpenAI-compatible HTTP streaming handlers only forward assistant text and lifecycle events. A live provider/model run was not performed in this read-only review.

## Solution Assessment

Unclear as proposed. Restoring raw passthrough by default is not the safest maintainable path; an explicit opt-in normalized reasoning stream contract is the better direction.
