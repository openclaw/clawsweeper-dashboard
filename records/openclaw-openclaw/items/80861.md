## Summary

Close as duplicate/superseded: the same HTTP/API reasoning exposure gap is already tracked by https://github.com/openclaw/openclaw/issues/68374, with https://github.com/openclaw/openclaw/pull/56674 open as an implementation candidate; current main still confirms the gap, so cleanup should consolidate discussion there.

## What This Changes

The issue requests restoring or adding an opt-in raw provider streaming passthrough so API/terminal clients can see provider reasoning/thinking deltas such as DeepSeek `reasoning_content` and Claude thinking blocks.

## Best Possible Solution

Consolidate remaining work under https://github.com/openclaw/openclaw/issues/68374 and review or replace https://github.com/openclaw/openclaw/pull/56674 with a gated, documented reasoning stream contract for the approved HTTP surfaces.

## Reproduction Assessment

Yes. Source inspection shows provider transports can produce internal thinking deltas while the OpenAI-compatible HTTP handlers forward only assistant text/lifecycle events, and the canonical issue includes concrete live CLI/Gateway evidence.

## Solution Assessment

Yes for cleanup. Closing this satellite as a duplicate is the narrowest maintainer-safe outcome; the product/API solution should be designed and reviewed in the canonical issue and open PR rather than split across duplicate reports.
