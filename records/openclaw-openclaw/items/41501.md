## Summary

Keep open and do not merge as-is. The narrow cron/subagent delivery-confirmation problem is still source-plausible on current main, but this PR bundles that fix with a root Langfuse tracing integration that current project direction routes through plugins/ClawHub, and the proposed patch has concrete security and correctness blockers.

## What This Changes

This PR adds a root Langfuse dependency and env-driven core tracing layer, wires tracing into CLI/reply/model/tool/subagent paths, changes agent-command/subagent announce delivery metadata, and adds related tests.

## Best Possible Solution

Split the work. Land a narrow core fix that returns explicit outbound delivery confirmation from the gateway agent command and requires that confirmation for completion-mode cron/subagent announces, with regression tests. Keep Langfuse out of core as a ClawHub/plugin adapter on the existing diagnostics-otel and plugin hook surfaces, adding only generic plugin API seams if a real adapter proves one is missing.

## Reproduction Assessment

Yes. The PR body describes a live Telegram-backed repro, and current main has a source-level path: best-effort outbound delivery can fail while the gateway agent call still returns visible payloads, after which subagent completion announce marks direct delivery as successful.

## Solution Assessment

No. This PR is not the best way to solve the issue because it bundles a narrow delivery-confirmation bug with a backend-specific tracing dependency and introduces concrete payload/secret leak paths; a split core delivery fix plus plugin-based Langfuse adapter is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Redact tool error payloads before exporting them:** `src/observability/langfuse-agent-hooks.ts:220-224`
  - body: When a tool fails with a structured result, this branch serializes the raw result into the Langfuse error status without passing it through `redactPayload`. Tokens, authorization headers, or nested credentials in tool errors can be exported in clear text when tracing is enabled.
  - confidence: 0.93
- **[P1] Redact depth overflows instead of returning raw values:** `src/observability/langfuse-agent-hooks.ts:28-30`
  - body: The depth guard returns the original value after `MAX_OBJ_DEPTH`. That skips sensitive-key redaction and string truncation for deeper payloads, so secrets below the depth limit can be sent to Langfuse unmodified.
  - confidence: 0.9
- **[P1] Keep follow-up drains outside the request trace scope:** `src/auto-reply/reply/get-reply.ts:118-124`
  - body: Wrapping the whole turn in one `AsyncLocalStorage` request scope lets asynchronous follow-up queue work inherit the earlier trace. Later queued messages can attach model/tool spans to the wrong inbound request under load.
  - confidence: 0.82
- **[P2] Clear tool spans when agent turns throw:** `src/auto-reply/reply/agent-runner.ts:363-365`
  - body: `activeRunId` is assigned only after `runAgentTurnWithFallback` returns, but that call can emit tool-start events and then throw. The `finally` block then skips `clearRunToolSpans`, leaving open span records alive for the failed run.
  - confidence: 0.86
- **[P2] Preserve silent duplicate announce success:** `src/agents/subagent-announce.ts:840-845`
  - body: The new delivery check returns failure when an external direct announce response lacks `result.delivery`. Non-completion announce prompts can legitimately suppress duplicate output with a silent response, which would now trigger retry or deferred cleanup behavior.
  - confidence: 0.78
- **[P2] Stop exposing the raw Langfuse secret in config:** `src/observability/langfuse.ts:27-32`
  - body: `LangfuseConfig` stores `secretKey` and returns that config on the instrumentation object. Keeping the raw secret on an exported object increases the chance it is serialized in debug output or tests; expose only a configured boolean.
  - confidence: 0.8

## Security Review

Status: needs_attention

Summary: The diff adds a new tracing sink and core dependency while introducing concrete secret and payload leak paths that should block merging as-is.

Concerns:

- **[high] Tool error payloads bypass redaction:** `src/observability/langfuse-agent-hooks.ts:220`
  - body: Structured tool errors are JSON-stringified and sent to Langfuse through `captureError` without `redactPayload`, so sensitive values in failed tool results can leave the process unredacted.
  - confidence: 0.93
- **[high] Nested payloads can escape redaction:** `src/observability/langfuse-agent-hooks.ts:28`
  - body: The redaction helper returns the original object after the max-depth guard, preserving deeply nested secrets instead of replacing them with a safe placeholder.
  - confidence: 0.9
- **[medium] Secret key is retained on exported config:** `src/observability/langfuse.ts:31`
  - body: The raw `LANGFUSE_SECRET_KEY` is stored on `LangfuseConfig` and returned on the instrumentation object, increasing accidental serialization or logging exposure risk.
  - confidence: 0.8
- **[low] Optional backend SDK expands core supply chain:** `package.json:375`
  - body: The diff adds `langfuse` and its transitive dependency to the root runtime dependency graph for an optional backend-specific integration that current policy routes through plugins.
  - confidence: 0.74
