## Summary

## What This Changes

The PR adds core Langfuse instrumentation and env config, wires traces through inbound replies/model/tool/subagent flows, changes agent delivery confirmation, and adds tests plus lockfile updates.

Split the delivery-confirmation fix into a narrow current-main PR with regression coverage, and route Langfuse-specific tracing through the existing diagnostics/OTEL plugin or a plugin/API design discussion with explicit opt-in content capture.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.8

Full review comments:

- **[P1] Redact deep payloads before exporting traces:** `src/observability/langfuse-agent-hooks.ts:28-29`
  - body: When redactValue exceeds MAX_OBJ_DEPTH it returns the original nested value, so sensitive keys below that depth bypass the redaction guarantee and can be sent to Langfuse. Return a sanitized placeholder or keep traversing with cycle protection instead of handing back raw payloads.
  - confidence: 0.9
- **[P1] Redact tool error results before status messages:** `src/observability/langfuse-agent-hooks.ts:170-178`
  - body: The error branch in onToolResult serializes the raw result into statusMessage without passing it through redactPayload, so failed tools can leak tokens, authorization fields, or nested credentials to Langfuse. Sanitize the error payload before stringifying or exporting it.
  - confidence: 0.82
- **[P2] Detach follow-up drains from the request scope:** `src/auto-reply/reply/get-reply.ts:96-120`
  - body: Wrapping the full getReplyFromConfig turn in AsyncLocalStorage lets async follow-up queue drains inherit the previous request trace, so later queued messages can be attributed to the wrong inbound request under load. Start follow-up drains outside the request scope or create a fresh scope per drained turn.
  - confidence: 0.78
- **[P2] Preserve silent duplicate announce success:** `src/agents/subagent-announce.ts:830-836`
  - body: The new delivery confirmation check treats any gateway agent response without result.delivery as a failed direct announce, but non-completion duplicate announces can intentionally return a silent response. Keep the silent duplicate path successful so idempotent announces do not retry or defer unnecessarily.
  - confidence: 0.8
- **[P2] Avoid retaining the raw Langfuse secret in config:** `src/observability/langfuse.ts:27-33`
  - body: LangfuseConfig stores secretKey on the exported instrumentation object, so debug serialization or assertions can expose the credential. Keep the secret local to client construction and expose only a configured boolean.
  - confidence: 0.76

## Security Review

Status: needs_attention

Summary: The diff introduces external tracing and has concrete secret/payload leakage paths that should block merge until repaired.

Concerns:

- **[high] Depth overflow returns raw sensitive payloads:** `src/observability/langfuse-agent-hooks.ts:28`
  - body: The proposed redaction helper returns the original value after MAX_OBJ_DEPTH, bypassing key-based redaction and truncation for deeply nested tool args/results.
  - confidence: 0.9
- **[high] Tool errors bypass redaction:** `src/observability/langfuse-agent-hooks.ts:170`
  - body: Failed tool results are stringified into Langfuse status messages without redactPayload, so structured error payloads can expose credentials when tracing is enabled.
  - confidence: 0.82
- **[medium] Raw Langfuse secret is retained on exported config:** `src/observability/langfuse.ts:31`
  - body: The instrumentation config includes secretKey, increasing the chance of accidental disclosure through logs, diagnostics, or serialized test output.
  - confidence: 0.76
