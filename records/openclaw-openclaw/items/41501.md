## Summary

Keep open. The PR is not merge-ready and should not be cleanup-closed: the Langfuse portion belongs on the plugin/ClawHub side of the boundary, while the cron/subagent delivery-confirmation bug is a real core behavior that is only partially covered on current main and needs a narrow split fix.

## What This Changes

This PR adds a root Langfuse dependency and core tracing layer, wires request/model/tool/subagent instrumentation into CLI and reply paths, changes agent-command delivery metadata and subagent announce delivery checks, and adds related tests and env docs.

## Best Possible Solution

Split the work. Keep or create a narrow core delivery fix that returns explicit outbound delivery confirmation from the gateway agent command and requires that confirmation for completion-mode cron/subagent announces, with focused regression tests. Build Langfuse as a plugin adapter on diagnostics-otel/plugin hooks, adding only generic plugin API seams if that adapter proves one is missing.

## Reproduction Assessment

Yes. The PR body gives a live Telegram-backed repro, and current main has a source-level reproduction path: best-effort external delivery can fail while deliverAgentCommandResult still returns visible payloads that subagent completion delivery treats as delivered.

## Solution Assessment

No. This PR is not the best way to solve the issue because it bundles a narrow core delivery-confirmation bug with a backend-specific tracing dependency and introduces concrete security findings; a split core fix plus plugin-based Langfuse adapter is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Redact tool error payloads before exporting them:** `src/observability/langfuse-agent-hooks.ts:220-224`
  - body: When a tool fails with a structured result, this branch serializes the raw result into the Langfuse error status without passing it through redactPayload. Tokens, authorization headers, or nested credentials in tool errors can be exported in clear text when tracing is enabled.
  - confidence: 0.93
- **[P1] Redact depth overflows instead of returning raw objects:** `src/observability/langfuse-agent-hooks.ts:28-30`
  - body: The depth guard returns the original value after MAX_OBJ_DEPTH. That skips sensitive-key redaction and string truncation for deeper payloads, so secrets below the depth limit can be sent to Langfuse unmodified.
  - confidence: 0.9
- **[P1] Run followup drains outside the request scope:** `src/auto-reply/reply/get-reply.ts:118-124`
  - body: Wrapping the whole turn in one AsyncLocalStorage request scope lets asynchronous followup queue work inherit the earlier trace. Later queued messages can attach model/tool spans to the wrong inbound request under load.
  - confidence: 0.82
- **[P2] Clear tool spans when the agent turn throws:** `src/auto-reply/reply/agent-runner.ts:363-365`
  - body: activeRunId is assigned only after runAgentTurnWithFallback returns, but that call can emit tool-start events and then throw. The finally block then skips clearRunToolSpans, leaving open span records alive for the failed run.
  - confidence: 0.86
- **[P2] Treat silent duplicate announces as successful dispatches:** `src/agents/subagent-announce.ts:840-845`
  - body: The new delivery check returns failure when an external direct announce response lacks result.delivery. Non-completion announce prompts can legitimately suppress duplicate output with a silent response, which would now trigger retry/deferred cleanup behavior.
  - confidence: 0.78
- **[P2] Stop exposing the raw Langfuse secret in config:** `src/observability/langfuse.ts:27-32`
  - body: LangfuseConfig stores secretKey and returns that config on the instrumentation object. Keeping the raw secret on an exported object increases the chance it is serialized in debug output or tests; expose only a configured boolean.
  - confidence: 0.8

## Security Review

Status: needs_attention

Summary: The diff adds a new tracing sink and core dependency while introducing concrete secret and payload leak paths that should block merging as-is.

Concerns:

- **[high] Tool error payloads bypass redaction:** `src/observability/langfuse-agent-hooks.ts:220`
  - body: Structured tool errors are JSON-stringified and sent to Langfuse through captureError without redactPayload, so sensitive values in failed tool results can leave the process unredacted.
  - confidence: 0.93
- **[high] Nested payloads can escape redaction:** `src/observability/langfuse-agent-hooks.ts:28`
  - body: The redaction helper returns the original object after the max-depth guard, which preserves deeply nested secrets instead of replacing them with a safe placeholder.
  - confidence: 0.9
- **[medium] Secret key is retained on exported config:** `src/observability/langfuse.ts:31`
  - body: The raw LANGFUSE_SECRET_KEY is stored on LangfuseConfig and returned on the instrumentation object, increasing accidental serialization or logging exposure risk.
  - confidence: 0.8
- **[low] Optional backend SDK expands core supply chain:** `package.json:375`
  - body: The diff adds langfuse and transitive mustache to the root runtime dependency graph for an optional backend-specific integration that current policy routes through plugins.
  - confidence: 0.74
