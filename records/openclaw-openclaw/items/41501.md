## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Redact tool error payloads before capture:** `src/observability/langfuse-agent-hooks.ts:220-224`
  - body: The error branch stringifies the raw tool result and sends it as the Langfuse status message without passing through `redactPayload`. A failed tool that returns structured data with `token`, `authorization`, or nested credentials would export those values in clear text when tracing is enabled.
  - confidence: 0.92
- **[P1] Do not return raw nested payloads at the depth limit:** `src/observability/langfuse-agent-hooks.ts:28-30`
  - body: The redaction depth guard returns the original value when `depth > MAX_OBJ_DEPTH`, so deeper objects bypass both key redaction and string truncation. Nested tool args/results with sensitive fields below that depth can be sent to Langfuse unredacted.
  - confidence: 0.88
- **[P2] Run queued followups outside the trace scope:** `src/auto-reply/reply/get-reply.ts:118-124`
  - body: Wrapping the whole reply turn in `withLangfuseRequestScope` lets async followup drains spawned during finalization inherit the previous request's AsyncLocalStorage context. Under queued followups, later messages can attach model/tool spans to the earlier trace instead of their own request trace.
  - confidence: 0.78
- **[P2] Treat silent duplicate announces as delivered:** `src/agents/subagent-announce.ts:840-845`
  - body: The new direct-delivery check treats every external gateway response without `result.delivery` as a failed direct announce. Non-completion announce prompts still allow the agent to return the silent duplicate token with no outbound delivery metadata, so legitimate duplicate suppression would now retry or defer unnecessarily.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The PR introduces concrete tracing data-leak risks in the proposed Langfuse payload handling, so it should not merge as written.

Concerns:

- **[high] Tool error payloads bypass redaction:** `src/observability/langfuse-agent-hooks.ts:220`
  - body: `onToolResult` redacts successful outputs but sends error results through `JSON.stringify(...).slice(...)` into `captureError`, so structured failed-tool payloads can leak tokens or credentials to Langfuse.
  - confidence: 0.92
- **[high] Depth overflow returns unsanitized objects:** `src/observability/langfuse-agent-hooks.ts:29`
  - body: The redactor returns the original nested value after the maximum depth instead of a sanitized placeholder, which can expose deeper sensitive keys in traced tool/model payloads.
  - confidence: 0.88
- **[medium] Raw Langfuse secret key is retained on exported config:** `src/observability/langfuse.ts:27`
  - body: `LangfuseConfig` includes `secretKey` and the instrumentation object exposes `config`, making accidental serialization or diagnostic logging of the Langfuse secret easier than necessary.
  - confidence: 0.8
