## Summary

Keep open: current main does not implement the Langfuse tracing work, and the PR still contains useful cron-delivery context, but the branch is stale against current main and has blocking redaction/scope issues that need maintainer review before merge.

## What This Changes

The PR adds optional Langfuse tracing with new env configuration and dependency changes, instruments request/model/tool/subagent paths, and changes cron/subagent completion delivery confirmation.

## Best Possible Solution

Split the cron delivery confirmation fix from the Langfuse tracing proposal, then land a current-main delivery regression fix and separately decide whether Langfuse belongs in core or a plugin/API extension.

## Reproduction Assessment

Partially yes. The cron-delivery concern is source-reproducible from the direct announce and delivery-evidence paths, and the PR-specific redaction defects are source-reproducible from the changed files, but I did not run a live Telegram cron instance in this read-only review.

## Solution Assessment

No. The cron delivery fix direction is useful, but bundling it with a new core Langfuse dependency and tracing PoC is not the narrowest maintainable path, and the proposed tracing implementation needs security and attribution fixes first.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Redact tool error payloads before exporting them:** `src/observability/langfuse-agent-hooks.ts:220-224`
  - body: In the error branch, structured tool results are stringified and sent to Langfuse without `redactPayload`, while successful outputs are redacted. A failed tool result containing fields such as `token` or `authorization` would be exported in clear text when tracing is enabled.
  - confidence: 0.93
- **[P1] Sanitize depth overflow instead of returning raw payloads:** `src/observability/langfuse-agent-hooks.ts:29-30`
  - body: When nesting exceeds `MAX_OBJ_DEPTH`, the helper returns the original value, so deeper tool args/results bypass sensitive-key redaction and truncation. Return a sanitized placeholder or keep traversing with cycle protection instead of handing raw data to Langfuse.
  - confidence: 0.91
- **[P2] Keep followup drains out of the request scope:** `src/auto-reply/reply/get-reply.ts:118-124`
  - body: Wrapping the full reply turn in `withLangfuseRequestScope` lets async followup queue work spawned during the turn inherit the prior request trace. Under queued traffic, later generations/tools can attach to the wrong inbound trace, corrupting attribution.
  - confidence: 0.82
- **[P2] Capture the run id before awaiting the agent turn:** `src/auto-reply/reply/agent-runner.ts:363-385`
  - body: `activeRunId` is assigned only after `runAgentTurnWithFallback` resolves. If that call throws after opening tool spans, the `finally` path has no run id to clear, leaving span records in the module map.
  - confidence: 0.86
- **[P2] Treat silent duplicate announces as successful:** `src/agents/subagent-announce.ts:840-845`
  - body: The new delivery check fails any external direct announce response without `result.delivery`, but non-completion announce prompts can legitimately produce the silent duplicate token with no outbound delivery. That path should not trigger failed-delivery retry/cleanup behavior.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The PR introduces telemetry export paths with concrete secret-redaction gaps before Langfuse tracing can be safely enabled.

Concerns:

- **[high] Tool error payloads bypass redaction:** `src/observability/langfuse-agent-hooks.ts:220`
  - body: Structured tool errors are stringified and passed to Langfuse error capture without `redactPayload`, so failed tools can leak sensitive fields that the success path would redact.
  - confidence: 0.93
- **[high] Deeply nested payloads bypass redaction:** `src/observability/langfuse-agent-hooks.ts:29`
  - body: The redaction depth guard returns the original nested value after the max depth, which can expose secrets below that depth to the external collector.
  - confidence: 0.91
- **[medium] Raw Langfuse secret is retained on exported config:** `src/observability/langfuse.ts:27`
  - body: `LangfuseConfig` stores `secretKey` and the enabled instrumentation exposes `config`, increasing the chance that debug serialization or tests disclose the secret.
  - confidence: 0.8
