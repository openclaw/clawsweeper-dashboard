## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Redact tool error payloads before exporting them:** `src/observability/langfuse-agent-hooks.ts:220-224`
  - body: The error branch stringifies structured tool results directly into the Langfuse status message without running them through redactPayload. A failing tool that returns token, authorization, or nested credential fields would export those values when tracing is enabled.
  - confidence: 0.94
- **[P1] Drain followup work outside the request scope:** `src/auto-reply/reply/get-reply.ts:118-124`
  - body: Wrapping the full reply turn in withLangfuseRequestScope lets followup drains spawned during finalization inherit the previous inbound request scope. Queued work from later inbound messages can then attach generations or tool spans to the wrong trace under load.
  - confidence: 0.86
- **[P2] Sanitize depth overflow instead of returning raw payloads:** `src/observability/langfuse-agent-hooks.ts:29-30`
  - body: When depth exceeds MAX_OBJ_DEPTH, redactValue returns the original nested value unchanged. Secrets below that depth skip both key redaction and string truncation, breaking the privacy guarantee for deeply nested tool args or results.
  - confidence: 0.9
- **[P2] Keep the Langfuse secret out of exported config:** `src/observability/langfuse.ts:27-33`
  - body: LangfuseConfig stores secretKey and the instrumentation object retains that config. If the instrumentation object is logged, inspected, or serialized for diagnostics, the Langfuse secret can leak; keep only a configured boolean outside client construction.
  - confidence: 0.82
- **[P2] Capture the run id before exception cleanup:** `src/auto-reply/reply/agent-runner.ts:409`
  - body: activeRunId is assigned only after runAgentTurnWithFallback returns a non-final outcome. If the run throws after opening tool spans, or returns through the early final branch, finally cannot clear the per-run Langfuse span map.
  - confidence: 0.83
- **[P2] Treat silent duplicate announces as delivered:** `src/agents/subagent-announce.ts:840-845`
  - body: The new delivery check treats any externally delivered announce response without result.delivery as a failed direct delivery. Non-completion announce prompts still allow silent duplicate replies, so a legitimate duplicate suppression can now drive retry or deferred-cleanup behavior.
  - confidence: 0.8
