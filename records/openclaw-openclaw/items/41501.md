## Summary

Keep this PR open for maintainer follow-up, but do not merge it as-is. Current main still lacks the external delivery-confirmation path needed for cron/subagent completion announces, while the Langfuse half is better handled through the existing diagnostics/OpenTelemetry plugin surface and still has concrete security and correctness findings in the PR diff.

## What This Changes

This PR adds a core Langfuse instrumentation layer and dependency, traces inbound/model/tool/subagent paths, exposes gateway agent delivery metadata, and adds tests for tool lifecycle and cron/subagent completion delivery confirmation.

## Best Possible Solution

Split the work: land a narrow core fix that has the gateway agent command return explicit external delivery results and makes completion-mode cron/subagent announces require that confirmation or use fallback delivery, with focused regression tests. Keep Langfuse as a ClawHub/plugin adapter built on diagnostics-otel and plugin hooks, proposing only generic plugin API additions if that adapter proves a real missing seam.

## Reproduction Assessment

Yes. Current main has a source-level reproduction path: deliverAgentCommandResult drops deliverOutboundPayloads results, and the dormant completion test still treats a payload-only gateway response as delivered=true, matching the false-positive delivery bug described in the PR body.

## Solution Assessment

No. The PR is not the best solution as-is because it bundles a narrow delivery bug fix with a core Langfuse dependency and tracing implementation, and the tracing diff has concrete security/correctness findings; a split core delivery fix plus plugin-based Langfuse adapter is safer and more maintainable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Redact tool error payloads before exporting them:** `src/observability/langfuse-agent-hooks.ts:220-224`
  - body: When a tool fails with a structured result, this branch JSON-stringifies the raw object and sends it to Langfuse as an error status without passing through redactPayload. Any token, authorization header, or nested credential in a tool error can be exported in clear text when tracing is enabled.
  - confidence: 0.93
- **[P1] Redact depth overflows instead of returning raw objects:** `src/observability/langfuse-agent-hooks.ts:28-30`
  - body: The depth guard returns the original value once MAX_OBJ_DEPTH is exceeded. That skips both sensitive-key redaction and string truncation for deeper nested payloads, so secrets below depth 5 can be sent to the tracing backend unmodified.
  - confidence: 0.9
- **[P1] Run followup drains outside the request scope:** `src/auto-reply/reply/get-reply.ts:118-124`
  - body: Wrapping the whole turn in one AsyncLocalStorage request scope means asynchronous followup queue work spawned from this path can inherit the earlier trace. Later queued messages can then attach model/tool spans to the wrong inbound request under load, corrupting trace attribution.
  - confidence: 0.82
- **[P2] Clear tool spans when the agent turn throws:** `src/auto-reply/reply/agent-runner.ts:363-365`
  - body: activeRunId is assigned only after runAgentTurnWithFallback returns, but that call can emit tool-start events and then throw. In that case the finally block has no run id and skips clearRunToolSpans, leaving open span records alive for the failed run.
  - confidence: 0.86
- **[P2] Treat silent duplicate announces as successful dispatches:** `src/agents/subagent-announce.ts:840-845`
  - body: This returns a delivery failure whenever an external direct announce response lacks result.delivery. Non-completion announce prompts can legitimately suppress duplicate output with a silent response, so those successful duplicate-suppression paths would now drive retry/deferred cleanup behavior.
  - confidence: 0.78
- **[P2] Stop exposing the raw Langfuse secret in config:** `src/observability/langfuse.ts:27-32`
  - body: LangfuseConfig stores secretKey and the enabled instrumentation returns that config object. Keeping the raw secret on an exported object increases the chance it is serialized in debug output or tests; keep the secret local to client construction and expose only a configured boolean.
  - confidence: 0.8

## Security Review

Status: needs_attention

Summary: The PR adds a new tracing sink and core dependency while introducing concrete secret and payload-leak paths that should block merging as-is.

Concerns:

- **[high] Tool error payloads bypass redaction:** `src/observability/langfuse-agent-hooks.ts:220`
  - body: Structured tool errors are JSON-stringified and sent to Langfuse through captureError without redactPayload, so sensitive values in failed tool results can leave the process unredacted.
  - confidence: 0.93
- **[high] Nested payloads can escape redaction:** `src/observability/langfuse-agent-hooks.ts:28`
  - body: The redaction helper returns the original object after the max-depth guard, which means deeply nested secrets are preserved instead of replaced with a safe placeholder.
  - confidence: 0.9
- **[medium] Secret key is retained on exported instrumentation config:** `src/observability/langfuse.ts:31`
  - body: The raw LANGFUSE_SECRET_KEY is stored on LangfuseConfig and returned on the instrumentation object, increasing accidental serialization/logging exposure risk.
  - confidence: 0.8
- **[low] Optional backend SDK expands core supply chain:** `package.json:375`
  - body: The diff adds langfuse and mustache to the root runtime dependency graph for an optional backend-specific integration that current project policy routes through plugins, increasing core install and review surface.
  - confidence: 0.74
