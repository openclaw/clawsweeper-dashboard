## Summary

Keep open: the current head is not merge-ready because it still adds plugin installer side effects to the normal doctor health path, and the new upgrade-survivor assertion can fail when ClawHub is intentionally overridden.

## What This Changes

The PR adds a 2026.5.2 doctor migration for configured downloadable plugin installs, updates ClawHub/npm repair behavior, expands upgrade-survivor coverage, and updates doctor/testing docs plus the changelog.

## Best Possible Solution

Keep the migration, but make installer and stamp side effects repair-only while leaving normal doctor diagnostic, and make the upgrade-survivor fixture assertion conditional on using the local fixture.

## Reproduction Assessment

Yes. The issue is reproducible from the diff: current main executes doctor health contributions unconditionally, and the PR adds the installer call in that path without a repair-mode guard.

## Solution Assessment

No. The migration direction is reasonable, but the implementation should preserve the existing doctor boundary by running package-manager/plugin install side effects only under `openclaw doctor --fix` or an equivalent repair/update mode.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate release installs behind repair mode:** `src/flows/doctor-health-contributions.ts:279-283`
  - body: `openclaw doctor` runs health contributions even when the user did not pass `--fix`, but this new contribution invokes configured plugin install repair and can then stamp `meta.lastTouchedVersion`. That makes a diagnostic doctor run download/install plugin code and mutate config, so please skip this side-effect path unless `ctx.prompter.shouldRepair` is true or make non-repair mode preview-only.
  - confidence: 0.92
- **[P2] Skip ClawHub log assertion when using an override:** `scripts/e2e/lib/upgrade-survivor/run.sh:320`
  - body: `start_configured_plugin_installs_clawhub_fixture` returns early when `OPENCLAW_CLAWHUB_URL` or `CLAWHUB_URL` is already set, but this assertion still requires the local fixture request log. In override environments the file is never created, so the configured-plugin-installs lane fails even when the candidate correctly attempted ClawHub.
  - confidence: 0.9
