## Solution Assessment

No. Lease-backed ownership is a good direction, but this patch still leaves a wrong-target SIGKILL window, can skip safe legacy cleanup after a stale lease, and can lose concurrent lease updates.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Clear the orphan SIGKILL fallback before returning:** `extensions/acpx/src/codex-auth-bridge.ts:291-292`
  - body: When orphan cleanup has started, the child exit handler returns before clearing `forceKillTimer`. If the child exits after SIGTERM, the delayed callback can later send SIGKILL to `-child.pid`; after PID/PGID reuse that can kill an unrelated process group.
  - confidence: 0.88
- **[P2] Fall back when lease cleanup does not terminate the tree:** `extensions/acpx/src/runtime.ts:749`
  - body: For legacy or migrated records, the same-session lease lookup can select a stale open lease. If lease cleanup skips as unverifiable or not owned, this unconditional return prevents the record PID fallback from running, leaving the live ACPX tree behind.
  - confidence: 0.82
- **[P2] Serialize lease file updates:** `extensions/acpx/src/process-lease.ts:93-98`
  - body: The lease store updates the shared JSON file with an unsynchronized read-modify-write. Concurrent ACP launches or closes can both read the same state and the later write can drop the earlier lease row, weakening startup and close reaping.
  - confidence: 0.86
