## Summary

Keep open. The PR addresses real ACPX cleanup gaps and includes useful live validation, but the patch introduces a blocker: per-launch lease metadata becomes part of acpx's persisted `agentCommand`, so acpx 0.6.1 will treat each persistent ensure as a different command and stop reusing existing ACP sessions.

## What This Changes

The PR adds ACPX process leases, wrapper lease metadata, fail-closed process reaping, session-list row ownership checks, Codex trust config generation, docs, changelog entries, and tests.

## Best Possible Solution

Keep the lease identity available to the live wrapper/process validation path without making acpx's persisted command identity change on every `ensureSession`, and add a regression test for repeat persistent ACPX reuse.

## Reproduction Assessment

Yes for the PR-introduced regression from source inspection: acpx 0.6.1 reuses persistent sessions only when `record.agentCommand` exactly matches the current command, while this PR adds a fresh lease id to that command for each `ensureSession`. I did not run live tests because this review was constrained to read-only inspection.

## Solution Assessment

No. The lease-backed cleanup direction is sound, but the implementation should keep lease metadata out of acpx's stable persisted command identity or otherwise normalize it before acpx performs its reuse comparison.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Keep lease args out of the persisted acpx command:** `extensions/acpx/src/runtime.ts:616-620`
  - body: `runWithLaunchLease` creates a new lease id for every `ensureSession`, and this line appends that id to the command that acpx receives from the registry. acpx 0.6.1 persists `agentCommand` and only reuses a persistent record when the stored command exactly equals the current command, so the next ensure for the same ACP session will see a different command and create a fresh backend session instead of resuming/reusing the existing one. Keep the live lease identity out of the stable command acpx stores or strip/normalize OpenClaw lease args before acpx compares reuse keys.
  - confidence: 0.94
