## Summary

Keep open: the updated branch fixes the earlier same-origin `/.../ws` sibling case, but the new legacy-filter predicate still accepts a stale non-root sibling URL when the current Control UI is mounted at the root path.

## What This Changes

The branch changes Control UI browser settings load/save to gate legacy global settings by page-derived Gateway scope, add page-scoped persistence, remove the shared legacy key on save, and extend storage regression coverage.

## Best Possible Solution

Keep the storage-boundary fix, but treat an empty page-derived path as the root Gateway scope so same-host non-root legacy sibling records are rejected while root `/ws` custom endpoints still migrate.

## Solution Assessment
