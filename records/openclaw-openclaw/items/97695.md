## Summary

Needs changes before merge: the PR is on the right storage boundary and has strong live-output proof for the non-root base-path case, but the latest predicate still lets a root-mounted current Control UI accept a same-host non-root legacy sibling Gateway URL.

## What This Changes

The PR changes Control UI browser settings load/save behavior to gate legacy global settings by page-derived Gateway scope, add page-scoped persistence, remove the shared legacy key on save, and extend storage regression coverage.

## Best Possible Solution

Keep the storage-boundary approach, but fix the legacy predicate so a root current page rejects same-host non-root legacy sibling paths while preserving root /ws custom endpoints, then land with focused storage regression coverage.
