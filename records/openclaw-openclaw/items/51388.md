## Best Possible Solution

Yes. A static reproduction is clear on current main: configure only `tools.subagents.tools.alsoAllow` for an optional plugin tool, then the subagent policy erases `alsoAllow` and `collectExplicitAllowlist()` produces no entry for `resolvePluginTools()`.

## Solution Assessment

Yes. The proposed direction is narrow and maintainable: preserve the already-configured `alsoAllow` values until the plugin tool allowlist is built, without making `alsoAllow` a separate gating mechanism.
