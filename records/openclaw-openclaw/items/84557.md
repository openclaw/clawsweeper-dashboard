## Summary

Keep open: the redaction direction matches the linked security roadmap and the supplied terminal proof is sufficient, but the patch can break existing custom provider auth when agent models.json is the only credential source.

## What This Changes

The PR adds a models.json planner step that removes plaintext provider apiKey values while preserving SecretRef/env/non-secret markers, plus regression tests for configured and merged keys.

## Best Possible Solution

Redact prompt-visible models.json without losing runtime auth by moving existing credentials to a non-prompt-visible source or adding a maintainer-approved fail-closed migration/setup path.

## Reproduction Assessment

Yes, from source: current main preserves non-marker agent models.json apiKey values and ModelRegistry reads them as provider request auth. The PR strips those values after merge, so the auth-loss path is clear without running a write test in this read-only review.

## Solution Assessment

No, not as-is: stripping prompt-visible secrets is the right direction, but this patch is not upgrade-safe for models.json-only credentials. The safer solution needs runtime credential preservation or an explicit fail-closed migration/setup decision.
