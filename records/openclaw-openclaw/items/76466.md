## Summary

Keep open: current main still prunes legacy `plugin-runtime-deps` roots during packaged postinstall, while doctor cleanup does not yet detect or repair dangling global scoped package symlinks. The same-author open PR #76465 covers the first diagnostic path, so this issue should stay open while maintainers review that paired work and choose the repair/prevention policy.

## What This Changes

The issue asks OpenClaw to diagnose, repair, or prevent bundled extension ESM failures caused by dangling global `@scope/<pkg>` symlinks after upgrade cleanup removes legacy `plugin-runtime-deps` targets.

## Best Possible Solution

Land #76465 or an equivalent diagnostic, then add a scoped doctor `--fix` or postinstall prevention path that removes only symlinks targeting pruned OpenClaw `plugin-runtime-deps` roots.

## Reproduction Assessment

Yes for source reproducibility: the report gives a concrete pre-2026.5.x upgrade path, current postinstall removes the target roots, current doctor cleanup misses the global scoped symlink location, and Slack imports `@slack/web-api` by bare name. I did not run a live global install reproduction in this read-only review.

## Solution Assessment

Unclear: the linked read-only diagnostic is a narrow first step, but it does not repair or prevent the import failure. Maintainers should choose whether the complete fix belongs in doctor `--fix`, packaged postinstall, or both.
