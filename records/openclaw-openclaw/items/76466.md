## Summary

Keep open: current main still has the upgrade path that can prune legacy `plugin-runtime-deps` targets without detecting or repairing dangling global scoped package symlinks. A same-author PR, #76465, is already open for the read-only diagnostic path, so this should remain available for maintainer follow-up instead of being closed or duplicated.

## What This Changes

The issue asks OpenClaw to detect, repair, or prevent dangling global `@scope/<pkg>` symlinks left after `plugin-runtime-deps` pruning breaks bundled extension ESM imports.

## Best Possible Solution

Land #76465 or an equivalent diagnostic, then add a narrow repair or prevention path that removes only dangling global scoped links targeting pruned `plugin-runtime-deps` roots.

## Reproduction Assessment

Yes for source reproducibility: the reporter's upgrade path maps to current postinstall pruning, current doctor cleanup misses the global scoped symlink location, and Slack imports the affected package by bare specifier. I did not run a live global-install upgrade reproduction in this read-only review.

## Solution Assessment

Unclear: the linked read-only doctor diagnostic is a narrow and reasonable first step, but it does not by itself repair or prevent the import failure. Maintainers should choose whether the complete fix belongs in doctor `--fix`, packaged postinstall, or both.
