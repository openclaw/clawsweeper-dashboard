## Summary

Keep open: current main still has the packaged postinstall prune that can remove legacy `plugin-runtime-deps` targets, while the checked doctor cleanup path does not yet detect or repair dangling global scoped package symlinks. A same-author open PR, #76465, already covers the read-only diagnostic path, so this issue should stay open while that paired work is reviewed.

## What This Changes

The issue requests handling for dangling global `@scope/<pkg>` symlinks left after packaged upgrades prune legacy `plugin-runtime-deps` roots, causing bundled extension ESM imports to fail.

## Best Possible Solution

Land #76465 or an equivalent diagnostic, then add a safe repair or prevention path that removes only global symlinks targeting pruned `plugin-runtime-deps` roots while preserving unrelated package-manager state.

## Reproduction Assessment

Yes for source reproducibility: the reporter gives a concrete pre-2026.5.x upgrade path, current postinstall deletes the legacy target roots, doctor cleanup does not walk the global scoped symlink location, and Slack imports `@slack/web-api` by bare name. I did not run a live global install reproduction in this read-only pass.

## Solution Assessment

Unclear: the linked read-only diagnostic is a narrow and reasonable first step, but it does not itself repair or prevent the import failure. Maintainers should choose whether the complete fix belongs in doctor `--fix`, packaged postinstall, or both.
