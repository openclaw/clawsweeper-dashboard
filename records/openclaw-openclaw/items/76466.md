## Summary

Keep open: current main still prunes legacy `plugin-runtime-deps` roots during packaged postinstall, while doctor repair does not inspect the global scoped symlinks described in the report. Same-author PR #76465 is open for the read-only diagnostic path, so this issue should stay open until that paired work lands and maintainers choose the repair or prevention path.

## What This Changes

The issue asks OpenClaw to detect and repair or prevent global scoped package symlinks left dangling after the 2026.5.x postinstall prunes legacy `plugin-runtime-deps` directories.

## Best Possible Solution

Land #76465 or an equivalent diagnostic, then add a constrained repair or prevention path that removes only global symlinks targeting pruned `plugin-runtime-deps` roots while preserving unrelated package-manager state.

## Reproduction Assessment

Yes for source reproducibility: the issue gives a concrete pre-2026.5.x upgrade path, current postinstall deletes the legacy target roots, doctor cleanup does not walk the global scoped symlink location, and Slack imports `@slack/web-api` by bare name. I did not run a live global install reproduction in this read-only review.

## Solution Assessment

Unclear: the linked read-only doctor diagnostic is a narrow and reasonable first step, but it does not repair or prevent failed channel imports by itself. The eventual fix should safely target only stale symlinks whose targets contain the `plugin-runtime-deps` marker.
