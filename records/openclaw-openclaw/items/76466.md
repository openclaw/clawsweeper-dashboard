## Summary

Keep open: current main still contains the packaged postinstall cleanup that can delete legacy `plugin-runtime-deps` targets, while the existing doctor cleanup path does not detect or repair the reported dangling global scoped package symlinks. A same-author open PR, #76465, covers the first read-only diagnostic path, so this should not be closed or duplicated by a new ClawSweeper branch yet.

## What This Changes

The issue reports that upgrades pruning legacy `plugin-runtime-deps` can leave dangling global scoped package symlinks, causing bundled plugin bare ESM imports such as Slack's `@slack/web-api` to fail.

## Best Possible Solution

Land #76465 or an equivalent diagnostic, then add a narrowly scoped doctor `--fix` or postinstall prevention path that removes only global scoped symlinks pointing into pruned `plugin-runtime-deps` roots, with regression coverage.

## Reproduction Assessment

Yes for source-level reproduction: the reporter gives a concrete pre-2026.5.x upgrade path, current postinstall deletes the legacy target roots, doctor cleanup does not inspect global scoped links, and Slack imports `@slack/web-api` by bare specifier. I did not mutate a global npm install in this read-only review.

## Solution Assessment

Unclear: the linked read-only diagnostic is a narrow first step, but it does not repair or prevent the import failure. Maintainers should choose whether the final fix belongs in doctor `--fix`, packaged postinstall, or both.
