## Summary

Keep open: current main still prunes legacy `plugin-runtime-deps` roots, and the checked doctor/postinstall paths do not detect or repair dangling global `node_modules/@scope/<pkg>` symlinks. A same-author open PR, #76465, already covers the read-only diagnostic path, so this issue should stay open while that paired work and the repair/prevention choice are reviewed.

## What This Changes

The issue reports that upgrades can leave dangling global scoped package symlinks after legacy runtime-deps pruning, causing bundled plugin ESM imports such as Slack to fail with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

Land #76465 or an equivalent diagnostic, then add a narrow repair/prevention path that removes only global scoped links targeting pruned `plugin-runtime-deps` roots while preserving unrelated package-manager state.

## Reproduction Assessment

Yes for source reproducibility: the reported upgrade path maps to current postinstall root pruning, current doctor cleanup does not scan the global scoped-link location, and Slack imports `@slack/web-api` by bare specifier. I did not run a live global install reproduction because this review is read-only.

## Solution Assessment

Unclear: the linked read-only diagnostic is a reasonable first step, but it does not repair or prevent the import failure. Maintainers should decide whether the complete fix belongs in `doctor --fix`, packaged postinstall, or both.
