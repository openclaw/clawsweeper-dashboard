## Summary

Keep open: current main still has the packaged postinstall prune that can remove legacy `plugin-runtime-deps` targets, but it does not yet include the global scoped-package symlink diagnostic or repair described here. The same author has an open paired PR, #76465, for the read-only doctor diagnostic, so this issue should stay open while that contribution is reviewed.

## What This Changes

The issue reports that upgrades from pre-2026.5.x can leave global scoped package symlinks pointing at pruned `plugin-runtime-deps` trees, causing bundled extension ESM imports such as Slack to fail with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

Land the paired diagnostic or equivalent, then add a conservative repair/prevention path that touches only OpenClaw-owned dangling scoped links into pruned `plugin-runtime-deps` targets.

## Reproduction Assessment

Yes. The reporter provides a concrete pre-2026.5.x-to-current upgrade path, and current main source shows postinstall removes the legacy targets while doctor cleanup does not inspect the global scoped links that can remain dangling.

## Solution Assessment

Unclear. The linked read-only doctor diagnostic is a narrow first step, but the complete fix likely needs a maintainer decision on whether repair belongs in doctor `--fix`, packaged postinstall, or both.
