## Summary

Keep open: current main still has the packaged postinstall prune path that can remove legacy `plugin-runtime-deps` targets, and I do not see a merged diagnostic or repair for dangling global scoped-package symlinks. The same author has open PR #76465 for the read-only doctor diagnostic, so this issue should remain open while that paired work is reviewed.

## What This Changes

Report an upgrade regression where packaged postinstall removes legacy bundled-plugin runtime dependency targets while old global `node_modules/@scope/<pkg>` symlinks keep pointing at those removed paths, breaking bundled extension ESM imports.

## Best Possible Solution

Add a focused diagnostic and repair/prevention path that handles only dangling global scoped package symlinks pointing into pruned `plugin-runtime-deps` trees, without touching unrelated `node_modules` links.
