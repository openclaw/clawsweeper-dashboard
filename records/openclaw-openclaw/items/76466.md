## Summary

Keep open: current `main` still has the packaged postinstall prune that can remove legacy `plugin-runtime-deps` targets, but it does not include a diagnostic or repair for dangling global scoped package symlinks. The same author has open PR #76465 for the read-only doctor diagnostic, so this issue should remain open while that paired fix is reviewed.

## What This Changes

The issue reports that upgrading after legacy `plugin-runtime-deps` pruning can leave global `node_modules/@scope/<pkg>` symlinks dangling, causing bundled plugin ESM imports such as Slack to fail with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

Land the focused diagnostic from #76465 or an equivalent, then add a safe repair/prevention path for global scoped symlinks that target pruned `plugin-runtime-deps` trees while preserving unrelated package-manager state.

## Reproduction Assessment

Yes: the reporter provides a concrete pre-2026.5.x-to-current upgrade path, and current source shows postinstall deletes the legacy target roots while doctor cleanup does not inspect global scoped symlinks. I did not run a live global install reproduction in this read-only review.

## Solution Assessment

Unclear: the linked read-only doctor diagnostic is a narrow and reasonable first step, but it does not repair or prevent the import failure. Maintainers should choose whether the complete fix belongs in doctor `--fix`, packaged postinstall, or both.
