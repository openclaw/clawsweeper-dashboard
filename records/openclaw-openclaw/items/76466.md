## Summary

Keep open: current main contains the postinstall prune that can remove legacy `plugin-runtime-deps` targets, but it does not yet include a merged diagnostic or repair for dangling scoped package symlinks in the global `node_modules` prefix. The same author has an open PR, #76465, that targets the read-only doctor diagnostic path, so this issue should remain open while that PR is reviewed.

## What This Changes

Report bundled plugin ESM import failures after upgrade when packaged postinstall prunes legacy runtime-dependency trees but old global package symlinks still point at them.

## Best Possible Solution

Land a focused diagnostic or repair that detects stale global `node_modules` symlinks targeting pruned `plugin-runtime-deps` trees, then decide whether `doctor --fix` or postinstall should safely remove them.

## Reproduction Assessment

Yes: the reporter provides a concrete upgrade path and current main source shows postinstall deletes the legacy target roots while existing doctor cleanup does not inspect global scoped symlinks. I did not run a live global-install reproduction in this read-only review.

## Solution Assessment

Unclear: the linked read-only doctor diagnostic is a narrow and reasonable first step, but it does not repair or prevent the import failure. Maintainers still need to choose whether the final fix belongs in doctor `--fix`, packaged postinstall, or both.
