## Summary

Keep open: current main has subagent list/info/log views, task summaries, and task cancellation, but it still lacks the durable ordered subagent lifecycle receipt, artifact/failure timeline fields, and public idempotent steer/resume supervision contract requested here.

## What This Changes

The issue requests deterministic subagent lifecycle observability and async supervision controls, including lifecycle events, persisted run summaries, artifact pointers, failure signatures, and idempotent cancel/steer/resume actions.

## Best Possible Solution

Keep this as the core design issue, define one canonical lifecycle receipt and supervision API with persistence, authorization, redaction, and audit semantics, then split implementation into protocol, state, and operator-UX increments that reuse the existing subagent and task registries.

## Reproduction Assessment

No, not as a bug reproduction: this is a feature/design request. Source and docs inspection gives high-confidence evidence that the requested lifecycle receipt and supervision surfaces are still not fully present on current main.

## Solution Assessment

Unclear until maintainers choose the permanent contract. Current main has useful partial pieces, but the best solution needs a bounded design across registry persistence, gateway protocol, redaction, audit, authorization, and operator UX rather than a one-off patch.
