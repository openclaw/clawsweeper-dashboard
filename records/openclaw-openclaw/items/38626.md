## Summary

Keep open: current main has scalar task/subagent tracking and useful controls, but it still does not implement the requested durable lifecycle event ledger, artifact-pointer completion payloads, first-failure diagnostics, or audited idempotent cancel/steer/resume supervision surface.

## What This Changes

Request deterministic operator observability and supervision for async subagent workflows, including lifecycle events, artifacts, failure diagnostics, and cancel/steer/resume controls.

## Best Possible Solution

Design a generic task/subagent event ledger with persisted lifecycle, tool, warning/error, completion, artifact, and supervision-action events, then expose it consistently through task/subagent inspection and control surfaces with migration/docs coverage.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature/design request. Source and docs inspection gives a high-confidence verification path because the current types, schema, listeners, and commands remain scalar/transient rather than a durable event/audit ledger.

## Solution Assessment

Unclear for a one-shot implementation PR. The maintainable direction is a generic durable task/subagent ledger and audited supervision API, not more ad hoc fields on `/subagents info` alone.
