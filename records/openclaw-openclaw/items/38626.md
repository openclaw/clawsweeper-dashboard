## Summary

Keep open. Current main has subagent summaries, control commands, task records, artifact lookup, diagnostics, and trajectory support, but it still lacks the requested durable per-subagent lifecycle receipt with ordered transition events, first-failure/next-action terminal data, artifact pointers in the completion payload, and audit-logged idempotent cancel/steer/resume semantics.

## What This Changes

The issue requests deterministic subagent lifecycle observability and async supervision controls for listing runs, inspecting state transitions, surfacing artifacts and failures, and supervising child runs.

## Best Possible Solution

Define a maintainer-approved subagent lifecycle receipt model that reuses the registry, task ledger, artifacts, agent-event, and trajectory pieces while specifying storage, redaction, retention, authorization, and audited supervision semantics before implementation.

## Reproduction Assessment

Not applicable as a bug reproduction, but source inspection gives a high-confidence feature-gap reproduction: current summaries, task records, and docs do not expose the requested lifecycle receipt or audited supervision controls.

## Solution Assessment
