## Summary

Current main already provides durable subagent run records, completion delivery, status/log/info views, and stop/steer controls, but the central request remains unimplemented: there is no canonical transition timeline, general resume contract, first-failure recovery guidance, or unified artifact receipt. No merged or viable open PR owns that remainder, and the product/security decisions require explicit maintainer review.

## What This Changes

Add durable timestamped subagent lifecycle events, artifact-aware terminal receipts, and idempotent cancel, steer, and resume supervision controls.

## Best Possible Solution

Extend the existing SQLite-backed subagent registry and canonical terminal-outcome model into one privacy-bounded lifecycle receipt, then expose authorized diagnostics, CLI, gateway, and UI projections with explicitly defined idempotent supervision transitions.

## Solution Assessment

Yes at the abstraction level: a canonical lifecycle receipt is the narrowest maintainable direction. It should extend the existing registry and terminal-outcome owners, not introduce a parallel timeline subsystem or UI-owned state model.
