## Summary

Keep open. Current main has background task tracking plus subagent list/kill/steer and gateway task summaries, but it still lacks the requested durable per-subagent lifecycle timeline, artifact pointers, first-failure next-action payload, resume action, and audited idempotent supervision contract.

## What This Changes

Request durable observability and deterministic supervision controls for async subagent runs, including lifecycle events, artifact-linked summaries, failure guidance, and cancel/steer/resume actions.

## Best Possible Solution

Extend the existing task/subagent registries with a versioned durable event timeline, artifact-reference metadata, terminal diagnostic/next-action fields, and audited idempotent supervision operations after maintainers settle the public API and retention model.

## Solution Assessment

Unclear. The current task ledger and subagent control tool are the right integration points, but the event timeline, artifact payload, first-failure diagnostic, resume semantics, and audit retention require maintainer API/product decisions.
