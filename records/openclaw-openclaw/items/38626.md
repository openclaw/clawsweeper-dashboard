## Summary

Keep open. Current main has a durable background task ledger and subagent control commands, but it only partially covers the requested lifecycle observability and does not yet provide a persisted per-subagent event timeline, artifact-pointer payloads, first-failure next-action diagnostics, or a clearly audited resume/control contract.

## What This Changes

The issue requests deterministic subagent lifecycle observability, persisted run summaries with artifact pointers, and idempotent async supervision controls for cancel, steer, and resume.

## Best Possible Solution

Extend the existing background task and subagent registries with a versioned event timeline, artifact-pointer metadata, failure diagnostic/next-action fields, and audited idempotent supervision actions after maintainers settle the operator-facing API and retention model.

## Reproduction Assessment

Not applicable. This is a feature/observability request, and source inspection shows current main has partial task/subagent tracking rather than a broken existing contract.

## Solution Assessment

Unclear. The existing task ledger is the right integration point, but the requested event schema, artifact metadata, diagnostics, and audit/resume semantics need maintainer product/API decisions before a narrow implementation can be judged best.
