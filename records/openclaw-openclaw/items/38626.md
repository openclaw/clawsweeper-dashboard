## Summary

Keep open: current main has background task records, Task Flow state, and subagent list/kill/steer controls, but it still lacks the requested durable per-subagent event timeline, artifact-pointer completion payloads, first-failure diagnostics, and audited idempotent cancel/steer/resume supervision model.

## What This Changes

Request durable lifecycle observability and async supervision controls for subagent workflows, including event timelines, artifact pointers, first-failure diagnostics, and idempotent audited cancel/steer/resume actions.

## Best Possible Solution

Design a generic task/subagent event ledger with persisted lifecycle, tool, warning/error, completion, artifact, and supervision-action events, then expose it consistently through task/subagent inspection and control surfaces with migration and docs coverage.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature/design request. Source inspection gives a high-confidence verification path because current main remains scalar/transient rather than a durable event and supervision audit ledger.

## Solution Assessment

No for a one-shot cleanup fix. The maintainable path is a designed generic durable task/subagent ledger and audited supervision API, not ad hoc fields on `/subagents info` or task rows alone.
