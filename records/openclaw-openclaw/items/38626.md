## Summary

Keep open. Current main has useful background task and subagent control pieces, but it still does not provide the requested persisted per-subagent lifecycle timeline, artifact-pointer completion payload, first-failure next-action diagnostic, or audited idempotent resume/control contract.

## What This Changes

Request deterministic operator observability and supervision controls for async subagent workflows, including lifecycle events, run summaries, artifact pointers, failure diagnostics, and cancel/steer/resume actions.

## Best Possible Solution

Extend the existing task/subagent registries with a versioned durable event timeline, artifact-reference metadata, failure diagnostic fields, and audited idempotent supervision operations after maintainers settle the public API and retention model.

## Reproduction Assessment

Not applicable. This is a feature/observability request, and source inspection shows current main has partial task/subagent tracking rather than a broken existing contract.

## Solution Assessment

Unclear. The existing task ledger is the right integration point, but the requested event schema, artifact payloads, diagnostics, and audit/resume semantics need maintainer product/API decisions before a specific patch can be judged best.
