## Summary

Keep open. Current main and the latest shipped release have useful task/subagent tracking and control primitives, but the requested durable per-subagent event timeline, artifact-pointer payloads, first-failure next-action diagnostic, and audited idempotent resume/control contract are still not present.

## What This Changes

Add deterministic lifecycle observability and supervision controls for async subagent runs, including persisted state transitions, artifacts, diagnostics, and cancel/steer/resume audit behavior.

## Best Possible Solution

Extend the existing task/subagent registries with a versioned durable event timeline, artifact-reference metadata, terminal diagnostic/next-action fields, and audited idempotent supervision operations after maintainers settle the API and retention model.

## Reproduction Assessment

Not applicable. This is a feature/observability request, and source inspection shows partial task/subagent tracking rather than a broken existing contract to reproduce.

## Solution Assessment

Unclear. The task ledger is the right integration point, but the public event schema, artifact payloads, diagnostics, resume semantics, and audit retention need maintainer product/API decisions before a concrete patch can be judged best.
