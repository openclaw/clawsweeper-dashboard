## Summary

Keep open. Current main has partial task/subagent observability and control, but it still lacks the requested persisted per-subagent lifecycle timeline, artifact-pointer completion payload, first-failure diagnostic, and audited idempotent supervision contract; the related background-spawn issue was closed as complementary, not superseding this work.

## What This Changes

Request durable async subagent lifecycle observability and deterministic supervisor controls for active/recent runs.

## Best Possible Solution

Extend the task/subagent registries with a versioned durable event timeline, artifact-reference metadata, failure diagnostic fields, and audited idempotent supervisor operations aligned across CLI, gateway/API, tests, and docs.
