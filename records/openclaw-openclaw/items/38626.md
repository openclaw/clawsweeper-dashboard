## Summary

Keep open. Current main has useful building blocks for subagent tracking, listing, task summaries, artifacts, and kill/steer controls, but it does not implement the requested durable per-subagent lifecycle timeline with tool/warning/artifact events, first-failure/recommended-action terminal data, or audit-logged cancel/steer/resume supervision. The remaining work defines a public observability and control contract with session-state and security implications, so it needs maintainer product/security review rather than closure or automated repair.

## What This Changes

Request deterministic subagent lifecycle observability, persisted run summaries with artifact pointers, and idempotent async supervision controls.

## Best Possible Solution

Define a maintainer-approved lifecycle receipt model that reuses the existing subagent registry, task ledger, artifact APIs, and diagnostics/hooks where appropriate, then add the missing durable timeline and supervision audit surface with explicit redaction, retention, authorization, and tests.
