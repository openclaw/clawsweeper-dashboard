## Summary

Keep open: current main has a task/subagent ledger plus cancel and steer controls, but it still exposes a coarser queued/running/terminal model rather than the full durable per-subagent lifecycle timeline, artifact-linked completion payload, first-failure guidance, idempotent action audit, and resume control requested here.

## What This Changes

Add deterministic subagent lifecycle visibility, persisted summaries and artifact references, and idempotent cancel, steer, and resume supervision controls.

## Best Possible Solution

Define an additive operator-facing contract for subagent task events, artifact references, supervised actions, and failure guidance, then implement it across the task registry, subagent control tool/CLI, gateway artifact/task APIs, and docs.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection shows current main has partial task/subagent tracking but not the complete requested observability and supervision contract.

## Solution Assessment

Unclear. The existing task ledger is a good base, but the best final shape needs maintainer decisions on durable event history, artifact pointer semantics, action audit responses, and how resume should differ from existing recovery/ACP resume paths.
