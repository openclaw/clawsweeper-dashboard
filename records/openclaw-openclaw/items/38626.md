## Summary

Keep open. Current main has useful subagent list/info/log surfaces, a task ledger, and kill/steer controls, but it does not fully satisfy the requested deterministic lifecycle timeline, artifact-backed terminal summaries, first-failure next-action payload, or audited idempotent cancel/steer/resume supervision surface.

## What This Changes

The issue requests persistent lifecycle observability and deterministic async supervision controls for subagent runs.

## Best Possible Solution

Define one generic background-run observability contract, then wire subagents to persist lifecycle events, artifact references, terminal diagnostics, and audited supervision actions through the CLI and Gateway/SDK surfaces.

## Reproduction Assessment

Not applicable. This is a feature/control-plane request, and source inspection shows current main has partial observability and control surfaces rather than a broken existing contract to reproduce.

## Solution Assessment

Unclear. The requested direction is reasonable, but maintainers should first decide whether the task ledger, subagent registry, or a new Gateway run API owns the durable event timeline and audited supervision semantics.
