## Summary

Keep open. Current main provides durable task state, status/progress events, summaries, artifact lookup, cancellation, and subagent steering, but it still lacks the requested detailed lifecycle receipt, artifact-linked completion payload, first-failure guidance, resume control, and one audited idempotent supervision contract; the related trace and timeline items do not fully supersede those requirements.

## What This Changes

Request deterministic subagent lifecycle timelines, durable run summaries with artifacts and failure guidance, and audited idempotent cancel, steer, and resume controls.

## Best Possible Solution

Extend the existing SQLite task registry into the canonical bounded lifecycle ledger, align its event taxonomy with the unified trace work, attach provenance-safe artifact references and structured failure guidance, and define authorized idempotent supervision operations before expanding operator UI surfaces.
