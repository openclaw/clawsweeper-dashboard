## Summary

Keep open. Current main has a durable task ledger, subagent task records, artifact query APIs, and partial kill/steer controls, but it still does not satisfy the requested persisted per-subagent lifecycle timeline, artifact-pointer completion payloads, first-failure next-action diagnostics, or audited operator resume/control contract.

## What This Changes

The issue requests deterministic operator visibility and control for async subagent workflows, including lifecycle events, summaries with artifacts, cancel/steer/resume actions, and terminal diagnostics.

## Best Possible Solution

Extend the existing task/subagent registries with a versioned persisted lifecycle event model, artifact metadata, terminal failure diagnostics, and audited idempotent supervision API after maintainers decide the public contract and retention model.

## Reproduction Assessment

Not applicable. This is a feature/observability request, and source inspection shows partial task/subagent tracking rather than a broken existing contract.

## Solution Assessment

Unclear. The existing background task ledger and subagent registry are the likely integration points, but the requested event schema, artifact/diagnostic fields, retention, and audited control semantics need maintainer API decisions first.
