## Summary

Current main has a background task ledger and CLI cancellation, but the requested subagent lifecycle timeline, artifact pointers, terminal diagnostics, and public async supervision API are still only partially present, so this should stay open for maintainer design follow-up.

## What This Changes

Request deterministic operator visibility and control for async subagent workflows, including lifecycle events, persisted summaries, artifacts, idempotent supervision actions, and actionable failure payloads.

## Best Possible Solution

Define and implement a maintainer-approved operator contract that persists subagent/task event history, exposes artifacts and terminal diagnostics consistently, and wires idempotent cancel/steer/resume across CLI, Gateway, SDK, and plugin runtime surfaces.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection confirms current main has a partial task ledger but not the full lifecycle timeline, artifact, diagnostic, and supervision surface requested.

## Solution Assessment

Unclear until maintainers choose the public contract. The narrowest maintainable direction is likely to extend the existing task/subagent ledger instead of adding a separate subagent-only observability store.
