## Summary

Keep open. Current main has useful background task records, `/subagents` inspection/control, terminal state notifications, and cancellation plumbing, but it still lacks the requested durable per-subagent lifecycle event ledger, persisted tool/warning/error timeline, artifact-pointer completion payloads, first-failure/recommended-action diagnostics, and audit-logged cancel/steer/resume supervision contract. The related #62478 background-spawn work and closed-unmerged #24925 lifecycle-hook PR are complementary, not replacements.

## What This Changes

The issue requests deterministic operator visibility and control for async subagent workflows, including lifecycle events, persisted run summaries, artifact pointers, failure diagnostics, and idempotent supervision actions.

## Best Possible Solution

Keep this open and split it into a maintainer-designed implementation plan: add a durable per-run event ledger for subagent/task spawn, queued/running, tool start/end, warning/error, completion, and supervision actions; persist artifact pointers plus terminal first-failure/recommended-action diagnostics; expose the ledger through `/subagents info/log`, `openclaw tasks list/show --json`, gateway/control UI surfaces, and docs; and make cancel, steer, and resume deterministic, idempotent, and audit-logged.
