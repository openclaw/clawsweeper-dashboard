## Summary

Keep open. Current main has useful building blocks for background task state, artifact lookup, task cancellation, and subagent list/kill/steer controls, but it still lacks the requested durable per-subagent event timeline, artifact-pointer completion payload, first-failure next-action diagnostic, public resume action, and audited idempotent supervision contract.

## What This Changes

The issue requests deterministic lifecycle observability and supervision controls for async subagent workflows, including event timelines, artifact pointers, failure diagnostics, and cancel/steer/resume actions.

## Best Possible Solution

Design a versioned subagent/task observability contract with durable event history, artifact-reference metadata, terminal diagnostics, and audited idempotent cancel/steer/resume operations, coordinated with the open run-trace and activity-timeline trackers before implementation.

## Solution Assessment

Unclear as an immediate implementation. The task ledger, artifact RPCs, subagent control paths, and run-trace/activity discussions are plausible integration points, but public schema, storage, audit, resume semantics, authorization, and retention need maintainer decisions first.
