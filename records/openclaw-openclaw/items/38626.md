## Summary

Keep open. Current main has useful background task records, `/subagents` inspection/control commands, terminal notifications, and scalar task delivery state, but it still lacks the requested durable per-subagent lifecycle event ledger, artifact-pointer completion payloads, first-failure/recommended-action diagnostics, and audit-logged idempotent cancel/steer/resume supervision surface.

## What This Changes

Request durable lifecycle observability and deterministic supervision controls for async subagent workflows, including event timelines, artifacts, failure diagnostics, and cancel/steer/resume actions.

## Best Possible Solution

Design a generic task/subagent event ledger that persists lifecycle, tool, warning/error, completion, artifact, and supervision-action events; expose it through task and subagent inspection surfaces; and make cancel/steer/resume deterministic, idempotent, and audit logged with docs and migration coverage.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature/design request. Source and docs inspection against current main gives a high-confidence verification path because the relevant records, schema, listeners, and commands are still scalar/transient rather than durable event-ledger based.

## Solution Assessment

Unclear for a single implementation PR. The maintainable direction is a generic durable task/subagent ledger and audit surface, likely split into smaller design-backed issues, rather than more ad hoc fields on `/subagents info` alone.
