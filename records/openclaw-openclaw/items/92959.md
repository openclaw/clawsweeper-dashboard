## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Use a valid probe fallback reason:** `src/agents/model-fallback-observation.test.ts:152`
  - body: `reason: "probe"` is not part of `FailoverReason`, and the PR's `check-test-types` job fails at the two `logModelFallbackDecision(params)` calls in this test. Use a real fallback reason such as `auth` or `rate_limit`, or omit the field if the test is meant to cover an unknown reason.
  - confidence: 0.99
- **[P2] Enforce the coalescer entry cap after pruning:** `src/agents/model-fallback-observation.ts:74`
  - body: When the map is already at `MAX_COALESCE_ENTRIES` and every existing entry is newer than the 60s prune horizon, this path still inserts another key, so high-cardinality fallback decisions can grow the supposedly bounded map past the cap. Evict an entry or skip storing the new key when pruning does not bring the map below the limit.
  - confidence: 0.9
- **[P2] Assert log emission instead of return fields:** `src/agents/model-fallback-observation.test.ts:82-84`
  - body: These duplicate-suppression tests only check the returned fallback-step fields, but suppressed calls intentionally return the same fields as logged calls. If `decisionLog.warn` kept firing on every duplicate, the tests would still pass; capture `model-fallback/decision` records and assert the emitted count and `suppressedDuplicateCount`.
  - confidence: 0.94
