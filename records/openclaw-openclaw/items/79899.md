## Summary

Keep open: current main has a merged partial mitigation from https://github.com/openclaw/openclaw/pull/75773, but source inspection shows the embedded runner still awaits Pi's reload path and upstream Pi still runs synchronous package/resource auto-discovery before the new skip flags filter loaded resources.

## What This Changes

The issue asks OpenClaw to stop Pi `DefaultResourceLoader.reload()` from synchronously scanning large Windows resource trees on every embedded agent run.

## Best Possible Solution

Add an OpenClaw-owned embedded loader path or upstream Pi option that skips package-manager auto-discovery entirely for embedded runs while preserving inline extension factories and OpenClaw skills prompt behavior, then validate it on Windows with event-loop-delay proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Unclear: the issue includes concrete Windows timing logs and the source path is inspectable, but I did not live-reproduce the NTFS stall on current main.

## Solution Assessment

Is this the best way to solve the issue? No: the current skip-flags wrapper is useful but not sufficient by source contract, because upstream reload still runs synchronous package/resource discovery before those flags filter loaded resources.
