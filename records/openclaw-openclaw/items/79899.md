## Summary

Keep open: current main still awaits Pi's `DefaultResourceLoader.reload()` on embedded run and compaction startup, and the current Pi dependency still performs broad synchronous filesystem discovery before OpenClaw can filter resource types.

## What This Changes

The issue asks OpenClaw to avoid blocking the Gateway event loop while Pi resource discovery scans resource directories during embedded agent startup on Windows.

## Best Possible Solution

Design a narrow cached or async embedded resource-loader boundary, or coordinate the equivalent upstream Pi fix, while preserving inline extension factories, OpenClaw skills prompt behavior, and Pi compaction guards.

## Reproduction Assessment

Unclear for live reproduction: the issue includes concrete logs and the source path is clear, but I did not reproduce the Windows/NTFS stall on current main. Source inspection confirms current main still calls the dependency reload path and the dependency uses synchronous broad filesystem discovery.

## Solution Assessment

Unclear: exposing skip flags alone is probably not the best fix because Pi resolves packages and auto-discovers resources before those flags filter loaded resources. A cached/async Pi fix or an OpenClaw-owned embedded loader boundary is the safer maintainable direction.
