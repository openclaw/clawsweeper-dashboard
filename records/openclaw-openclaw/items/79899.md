## Summary

Keep open: current main and the latest release still route embedded attempts and compaction through `DefaultResourceLoader.reload()`, and the linked candidate PR is open, dirty, retargeted to retired paths, and missing real Windows or large-tree proof.

## What This Changes

The issue reports Windows event-loop stalls because embedded session runs repeatedly invoke synchronous resource discovery through `DefaultResourceLoader.reload()`.

## Best Possible Solution

Land a maintainer-approved loader-owned fast path for embedded attempts and compaction that avoids ambient discovery while preserving settings reload semantics, inline extension factories, diagnostics, source metadata, and Windows timing proof.

## Reproduction Assessment

No, not for the exact Windows/NTFS 12-15s timing. Source inspection gives a clear current-main path through `resourceLoader.reload()` and synchronous package discovery, but a live large-tree Windows benchmark is still missing.

## Solution Assessment

Unclear. Avoiding unnecessary embedded-run discovery is the right direction, but adding user-facing skip config is not the narrowest fix and the current candidate PR needs a supported loader contract rather than private-field shortcuts.
