## Summary

Keep open: current main still awaits Pi's resource-loader reload in embedded agent startup and compaction, and the pinned Pi dependency still performs synchronous package/resource discovery before the OpenClaw skip flags filter resources. Related cleanup has made this the canonical tracker rather than resolving it.

## What This Changes

The issue reports Windows event-loop stalls and gateway timeouts because embedded agent runs call `DefaultResourceLoader.reload()` over large resource trees.

## Best Possible Solution

Keep this as the canonical bug and design an OpenClaw-owned loader boundary or upstream Pi fix that avoids synchronous discovery while preserving required inline extension behavior, then prove it on Windows.

## Reproduction Assessment

No high-confidence live reproduction was established here. Source inspection does confirm current main still reaches the upstream reload path whose dependency code performs synchronous resource discovery before skip-flag filtering.

## Solution Assessment

Unclear: the current skip-flag wrapper is a useful mitigation but not a complete fix because upstream resolution still scans first. A safer solution should cache or bypass stable discovery work without reusing stale per-run settings or extension factories.
