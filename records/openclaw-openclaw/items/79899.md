## Summary

Keep open: current main still awaits Pi `DefaultResourceLoader.reload()` in embedded run and compaction startup, and the pinned Pi package performs broad synchronous filesystem discovery before OpenClaw can filter resource types.

## What This Changes

The issue requests eliminating 12-15s Windows event-loop stalls from Pi `DefaultResourceLoader.reload()` during OpenClaw agent startup.

## Best Possible Solution

Keep this open and design a narrow OpenClaw-owned embedded resource-loader boundary, or coordinate an upstream Pi cached/async discovery fix, with Windows proof that Gateway responsiveness and inline compaction/context-pruning behavior are preserved.

## Reproduction Assessment

No, not as a high-confidence live Windows timing reproduction. Source inspection does show a current-main path that awaits the pinned dependency’s synchronous broad resource discovery before each embedded session starts.

## Solution Assessment

Unclear because there is no patch yet and the proposed skip flags are probably not the best standalone fix. The maintainable direction is an OpenClaw loader boundary or upstream Pi cache/async change, validated against real Windows Gateway behavior.
