## Summary

Keep open: current main still awaits Pi `DefaultResourceLoader.reload()` in both embedded run paths, and fresh 2026.5.12 reporter data says the `session-resource-loader` stall remains proportional to Windows file count. The linked fix PR is open but unmerged and has review blockers, so it is an implementation path rather than a close reason.

## What This Changes

The issue reports Windows/NTFS event-loop stalls from synchronous Pi resource discovery during embedded agent startup and asks for caching, async discovery, or skip controls.

## Best Possible Solution

Keep the issue open until a merged fix removes or replaces the expensive Pi discovery path for embedded runs while preserving inline OpenClaw extensions and proving Windows large-tree timing before and after.

## Reproduction Assessment

No high-confidence live reproduction was run in this review. The issue includes current 2026.5.12 timing logs, and source plus dependency inspection confirms the current-main path still awaits synchronous broad resource discovery.

## Solution Assessment

Unclear: a config-only skip flag is not the best fix because Pi still resolves package resources before filtering. The maintainable solution is a public-contract fast path or upstream Pi change with Windows proof, not another private-internals shortcut.
