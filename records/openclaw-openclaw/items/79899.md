## Summary

Keep open: current main has a shipped mitigation that passes Pi skip flags to embedded runs, but `DefaultResourceLoader.reload()` still calls the dependency package resolver first, and the pinned dependency still performs synchronous auto-discovery before those flags filter resources.

## What This Changes

The issue reports Windows event-loop stalls from `DefaultResourceLoader.reload()` synchronously scanning Pi resource directories during embedded agent startup.

## Best Possible Solution

Keep this open until embedded runs either skip Pi package auto-discovery entirely or Pi provides cached/async discovery, with Windows proof on a large `.openclaw`/skills tree.

## Reproduction Assessment

No live high-confidence reproduction was established in this review. The reporter provided concrete Windows timing logs, and current source plus dependency inspection still show a synchronous reload/discovery path that can plausibly reproduce the stall.

## Solution Assessment

No: the current shipped skip-flag wrapper is a useful mitigation, but it is not proven to be the best or complete fix because the dependency still resolves auto-discovered resources before those flags filter them.
