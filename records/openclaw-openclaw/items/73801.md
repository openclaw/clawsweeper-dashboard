## Summary

Keep open. Current main and v2026.6.8 still route the documented Cerebras Active Memory model through the embedded recall path, and the generic timeout/cleanup mitigations do not prove that `cerebras/gpt-oss-120b` now releases promptly or preserves gateway health under the reported live conditions.

## What This Changes

The issue reports that Active Memory configured with `cerebras/gpt-oss-120b` can exceed its configured timeout and leave the local gateway CPU-bound or slow to answer health/admin probes.

## Best Possible Solution

Keep this as the focused Cerebras Active Memory reliability tracker until maintainers either prove the v2026.6.8/current-main path healthy with live gateway evidence or land a targeted Active Memory/embedded-runner repair with that proof.
