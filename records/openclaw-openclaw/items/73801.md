## Summary

Keep open. Current main still documents `cerebras/gpt-oss-120b` as a good Active Memory recall model and has generic timeout/circuit-breaker mitigations, but there is no source, test, or live-provider evidence that the Cerebras reasoning/tool stream aborts cleanly or that gateway health stays responsive after the timeout. ## What This Changes

Active Memory should fail fast and preserve gateway responsiveness when configured with `cerebras/gpt-oss-120b`, or the documented Cerebras recommendation should be qualified.

## Best Possible Solution

Keep this as a focused Active Memory/Cerebras regression until maintainers can reproduce or simulate the `gpt-oss-120b` reasoning/tool stream, measure elapsed time, CPU, `/health`, and websocket/admin responsiveness during and after timeout, then either fix the embedded runner/provider transport cleanup or qualify the Cerebras Active Memory recommendation.
