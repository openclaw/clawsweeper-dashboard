## Summary

Keep open. Current main still documents and ships the reported Active Memory plus Cerebras `gpt-oss-120b` setup, and the code only shows generic timeout, circuit-breaker, abort-signal, and abort-retention mitigation. ## What This Changes

The issue reports that Active Memory configured with `cerebras/gpt-oss-120b` can exceed its timeout budget and leave the local gateway CPU-bound even though direct Cerebras chat and tool-call requests work.

## Best Possible Solution

Keep this as a focused Active Memory/Cerebras reliability regression until maintainers can reproduce or simulate the gpt-oss reasoning/tool stream, measure elapsed time, gateway CPU, `/health`, and websocket/admin responsiveness, then either fix the provider/runtime cancellation path or qualify the Cerebras Active Memory recommendation.
