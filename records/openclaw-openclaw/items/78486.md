## Summary

Keep open: this is a plausible feature PR, but the current diff still changes fallback policy too broadly, persists only part of the session cleanup, lacks convincing real behavior proof, and needs maintainer product approval before merge.

## What This Changes

The branch adds an `agents.defaults.revertAfterImageModel` config option and post-reply logic intended to clear auto model overrides after image or PDF analysis.

## Best Possible Solution

If maintainers accept this policy, implement an opt-in media/PDF-scoped revert from an explicit analysis completion signal, persist the full cleared fallback/session state, and add focused docs, regression tests, and real runtime proof.

## Reproduction Assessment

Not applicable as a strict current-main bug reproduction: current docs define sticky auto-fallback behavior. The PR defects are source-reproducible from the diff and the session-store merge contract.

## Solution Assessment

No. The opt-in direction may be viable, but this patch is not the best implementation because it is not media-scoped and does not persist every field the revert helper clears.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Gate reverts on media analysis:** `src/auto-reply/reply/agent-runner.ts:1392-1399`
  - body: This condition is true after any successful turn that starts with an auto model override when the flag is enabled. A normal text follow-up after fallback would clear the auto override even though no image or PDF analysis just completed.
  - confidence: 0.88

  - confidence: 0.9
