The PR adds an opt-in `agents.defaults.revertAfterImageModel` config key plus post-run session logic intended to clear auto model overrides after image or PDF analysis.

## Best Possible Solution

If maintainers accept the feature, implement an explicit opt-in media/PDF completion signal that reuses current one-turn image override behavior where possible, clears the complete override state, and adds focused docs plus regression tests under the canonical feature issue.

## Reproduction Assessment

Not applicable as a strict current-main bug: current docs define sticky auto fallback behavior and current main already has one-turn image override protection. The PR defects are source-reproducible from the broad guard and partial session-store merge patch.

## Solution Assessment

No. The opt-in direction may be viable, but this patch is not the best implementation because it is not tied to actual media/PDF completion and does not persist every field the revert helper clears.
