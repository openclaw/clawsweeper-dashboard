## Summary

Keep open: this is a new config-driven feature, and the proposed patch has correctness blockers in the session-store mutation and trigger condition before it is safe to merge.

## What This Changes

The PR adds `agents.defaults.revertAfterImageModel` config/schema/help labels and clears auto session model overrides in `runReplyAgent` after the claimed image/PDF analysis path completes.

## Best Possible Solution

If maintainers want this feature, trigger it from an explicit image/PDF analysis signal, persist the whole session-entry mutation through the session-store writer, and add docs, changelog, and focused regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction: current docs and source define auto fallback persistence as designed behavior. Source review does show concrete ways to exercise the PR regressions with an auto fallback session followed by a text-only turn or a provider-switch fallback.

## Solution Assessment

No. The PR uses a broad post-run condition and a partial persisted patch; the safer solution is a media/PDF-specific completion signal plus reusing the session mutation helper inside the store writer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Persist the full reverted session state:** `src/auto-reply/reply/agent-runner.ts:1414-1418`
  - body: This writer patch only persists `providerOverride`, `modelOverride`, and `modelOverrideSource` after `applyModelOverrideToSessionEntry` has already deleted additional fields such as `authProfileOverride*`, runtime model metadata, context tokens, and fallback notices. Because `updateSessionStoreEntry` merges partial patches, the session file can keep stale fallback/auth state even though the in-memory entry was reverted.
  - confidence: 0.9

  - body: The new condition is true for any successful turn that starts with an auto model override when the config flag is enabled. That means a normal text follow-up after a transient model fallback will clear the auto override, contradicting the documented fallback behavior and the option's image/PDF scope.
  - confidence: 0.86

  - body: This adds a user-facing `agents.defaults` config option and runtime behavior, but the PR does not include a `CHANGELOG.md` entry. OpenClaw policy requires user-facing `feat` changes to be listed before merge.
  - confidence: 0.82
