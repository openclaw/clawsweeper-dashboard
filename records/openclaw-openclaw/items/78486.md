## Summary

Keep open: this is a plausible feature/config request, but the current PR is not mergeable as-is. The diff is broader than the claimed image/PDF path, can leave stale persisted session-auth state, and still lacks convincing after-fix real behavior proof.

## What This Changes

The PR adds an `agents.defaults.revertAfterImageModel` config option and post-run `runReplyAgent` logic to clear auto session model overrides after claimed image/PDF analysis.

## Best Possible Solution

If maintainers want this policy, implement it from an explicit image/PDF analysis completion signal, clear and persist the full owned session override state, and cover the accepted config behavior with focused docs and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction: current source and docs define sticky session and auto-fallback behavior. The PR's defects are source-reproducible from the diff and the session-store merge contract.

## Solution Assessment

No. The PR uses a broad post-run condition and a partial persisted patch; the safer approach is a media/PDF-specific completion signal plus a writer path that persists every field the session override helper clears.
