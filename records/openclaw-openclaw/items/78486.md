## Summary

Keep open for maintainer handling: this PR targets a real open feature request, but it adds a new config/session policy and the current patch is not media-scoped, does not persist all cleared session state, and still lacks convincing real behavior proof.

## What This Changes

The PR adds an opt-in `agents.defaults.revertAfterImageModel` config key plus post-run session logic intended to clear auto model overrides after image or PDF analysis.

## Best Possible Solution

If accepted, implement an opt-in media/PDF-scoped revert from an explicit analysis completion signal, persist the full cleared override state, and update docs plus focused regression tests under the canonical feature issue.

## Reproduction Assessment

Not applicable as a strict current-main bug: current docs and tests define sticky auto-fallback behavior. The PR defects are source-reproducible from the diff and session-store merge contract.

## Solution Assessment

No. The opt-in direction may be viable, but this patch is not the best implementation because it is not tied to actual media/PDF completion and does not persist every field the revert helper clears.
