## What This Changes

The issue requests Feishu REST and bot identity calls to work when OpenClaw managed network proxy mode is enabled.

## Best Possible Solution

Fix the Feishu plugin HTTP wrapper so Lark SDK REST, token, and probe calls cooperate with managed proxy routing while preserving core proxy policy, then add regression coverage and a changelog entry.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/77323. Feishu bot identity and other Lark SDK REST calls fail when `proxy.enabled=true` because OpenClaw sets proxy env plus forced global-agent, while the Lark SDK Axios `defaultHttpInstance` can also consume proxy env. In `extensions/feishu/src/client.ts`, adjust the Feishu HTTP instance wrapper/interceptor so all SDK REST/token/probe calls avoid Axios env-proxy handling while preserving OpenClaw managed proxy/global-agent routing, for example by injecting Axios `proxy: false` or an equivalent owner-local normalization. Do not change core proxy lifecycle, do not set `GLOBAL_AGENT_FORCE_GLOBAL_AGENT=false`, do not reintroduce `NO_PROXY`, and do not add a Feishu direct-bypass config. Add regression coverage in `extensions/feishu/src/client.test.ts` showing wrapped request/get/post/etc. keep timeout behavior and pass the proxy-normalization option without clobbering caller options; add probe coverage only if needed. Add a single-line user-facing `CHANGELOG.md` fix entry for Feishu/proxy. Validate with the targeted Feishu tests and the changed gate per repo policy.
