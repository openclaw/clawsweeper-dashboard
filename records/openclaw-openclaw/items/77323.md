## Summary

Keep open: the current Feishu REST/token/probe path still matches the report, with Lark SDK Axios requests left to ambient env-proxy handling while managed proxy mode forces process-wide proxy env and global-agent routing.

## What This Changes

Fix Feishu bot identity and REST token/API calls failing with `ECONNRESET` when `proxy.enabled=true` is used with a local HTTP proxy.

## Best Possible Solution

Fix Feishu’s owner-local HTTP wrapper so Lark REST/token/probe calls avoid Axios env-proxy double handling while preserving OpenClaw managed-proxy egress, then add regression coverage and a changelog entry.

## Reproduction Assessment

Yes, source-reproducible: current main sets managed proxy env and forced global-agent, while Feishu bot identity uses the Lark SDK Axios REST instance without REST-specific proxy normalization. I did not perform a live Feishu credential run through a local proxy.

## Solution Assessment

Yes for a narrow Feishu-owned HTTP wrapper fix that disables Axios env-proxy rewriting while still letting managed proxy routing handle egress. No for a hard-coded direct bypass that would weaken the documented proxy guardrail.

## Repair Work Prompt

Fix the Feishu REST/token/probe proxy bug tracked by https://github.com/openclaw/openclaw/issues/77323. Current main routes managed proxy mode through env/global-agent, but `extensions/feishu/src/client.ts` delegates REST calls to the Lark SDK Axios `defaultHttpInstance` with timeout only, so Axios env-proxy rewriting can fight the managed proxy path. Implement an owner-local Feishu HTTP wrapper change that prevents Axios env-proxy double handling while preserving OpenClaw managed-proxy egress; do not add a new channel config option, do not change core proxy policy, and do not hard-code a direct bypass that defeats `proxy.enabled=true`. Add focused tests in `extensions/feishu/src/client.test.ts` and/or `extensions/feishu/src/probe.test.ts` covering REST request options under proxy env and preserving existing timeout/User-Agent behavior. Add a user-facing `CHANGELOG.md` fix entry. Treat #48949 and #59806 as related token/proxy evidence and #72595 as a broader product request; stop for maintainer review if the fix requires a per-channel proxy mode decision.
