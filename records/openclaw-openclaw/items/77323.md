## Repair Work Prompt

Fix openclaw/openclaw#77323. Feishu bot identity and REST calls fail under `proxy.enabled=true` because `extensions/feishu/src/client.ts` delegates Lark REST/token/probe requests to `defaultHttpInstance` without REST-specific proxy normalization while OpenClaw managed proxy sets HTTP(S)_PROXY, clears NO_PROXY, and forces global-agent. Implement a narrow Feishu-owned HTTP wrapper so all Lark REST calls, including token and `/open-apis/bot/v1/openclaw_bot/ping`, avoid Axios/env-proxy double handling and still honor the managed proxy routing contract. Do not add a new config option, do not restore NO_PROXY, do not set `GLOBAL_AGENT_FORCE_GLOBAL_AGENT=false`, and do not introduce a broad direct-egress bypass unless a maintainer explicitly changes the proxy policy. Add regression coverage in Feishu client/probe tests for REST request options under proxy env and keep existing WS proxy behavior covered. Add a single-line `CHANGELOG.md` fix entry crediting @linbojin if a user-facing fix ships.
