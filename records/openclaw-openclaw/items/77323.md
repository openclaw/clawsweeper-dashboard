## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/77323 in the Feishu plugin only. The observable bug is that Feishu/Lark REST bot identity and token requests can fail under `proxy.enabled=true` because `createTimeoutHttpInstance()` delegates to the Lark SDK Axios `defaultHttpInstance` with only timeout/User-Agent handling, while managed proxy mode sets HTTP(S)_PROXY, forces global-agent, and clears NO_PROXY. Implement owner-local REST request normalization so Axios does not also consume env proxy settings and malformed/double-proxy the request; preserve OpenClaw managed proxy routing through global-agent rather than disabling `GLOBAL_AGENT_FORCE_GLOBAL_AGENT`, adding a config option, or routing Feishu REST direct by default. Add/update focused tests in `extensions/feishu/src/client.test.ts` for injected REST options under proxy env and keep existing timeout behavior; update `extensions/feishu/src/probe.test.ts` only if needed. Add a single-line user-facing fix entry to `CHANGELOG.md`. Do not change core proxy policy or unrelated channel/provider proxy behavior.
