## Summary

Keep open: current main still hard-fails an explicit missing `plugins.slots.memory` plugin and the Memory LanceDB docs still describe the plugin as bundled/configurable before showing any install requirement. There is already an open closing PR for this issue, so the right cleanup path is to review and land or close that PR rather than closing this issue first.

## What This Changes

The issue reports that setting `plugins.slots.memory` to `memory-lancedb` before installing `@openclaw/memory-lancedb` crash-loops the gateway and that the Memory LanceDB docs do not clearly lead with the required npm install step.

## Best Possible Solution

Land a maintainer-approved fix that makes missing official external memory slots produce an actionable install warning or explicitly preserves fail-closed startup with clearer recovery docs, and update the Memory LanceDB docs to lead with npm installation.

## Reproduction Assessment

Yes, source-reproducible: current main turns an explicit missing `plugins.slots.memory` value into a fatal validation issue, and gateway startup throws invalid snapshots. I did not run the Docker restart loop, but the validator and startup paths match the reported error.

## Solution Assessment

Unclear pending maintainer policy: the docs install section is the narrow correct fix, and https://github.com/openclaw/openclaw/pull/82438 looks like a focused implementation if maintainers accept warning for official external memory slots while preserving hard failure for unknown ids.
