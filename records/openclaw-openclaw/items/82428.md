## Summary

Keep open: current main still hard-fails an explicit missing memory slot, and the memory-lancedb docs/package metadata still support the reported install-step mismatch. The related memory issues cover post-install runtime/tool exposure paths, not this boot-time validation and docs gap.

## What This Changes

The issue requests graceful gateway behavior and install-first docs when `plugins.slots.memory` selects `memory-lancedb` before `@openclaw/memory-lancedb` is installed.

## Best Possible Solution

Treat known official external missing memory slot owners as recoverable with the existing install hint, boot without binding that missing slot, and document installing `@openclaw/memory-lancedb` before selecting it.

## Reproduction Assessment

Yes, source-reproducible. A config with explicit `plugins.slots.memory: "memory-lancedb"` and no discovered plugin hits the hard-error branch in `src/config/validation.ts`, then config loading throws the invalid config.

## Solution Assessment

Yes, if scoped to known official-external missing installs. Broadly softening every unknown memory slot would not be the best fix because it could hide typos.
