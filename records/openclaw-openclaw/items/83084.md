## Summary

Keep open. The footer gap is source-backed, but this patch bypasses the strict message-tool-only room-event suppression path and only provides unit-test proof for a Telegram-visible behavior.

## What This Changes

The PR extracts a `Usage: ...` line from suppressed message-tool-only final replies and sends it as a standalone final payload, with one dispatch regression test.

## Best Possible Solution

Either land the docs clarification, or implement a Telegram-proven message-tool-aware footer path that preserves sendPolicy and room-event quiet semantics.

## Reproduction Assessment

Yes. Source inspection shows usage footers are appended to final reply payloads while `message_tool_only` suppresses automatic final delivery; I did not run a live Telegram forum-topic reproduction.

## Solution Assessment

No. The patch is not the best fix as written because it bypasses the strict room-event/message-tool-only delivery contract; a docs clarification or a message-tool-aware runtime design is safer.
