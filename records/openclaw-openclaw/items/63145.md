## Summary

Keep open. Current main still lacks `openclaw models status --probe --all`, while the linked open PR #63259 remains the implementation candidate rather than a shipped fix.

## What This Changes

Add a `--all` mode to `openclaw models status --probe` that probes every configured primary, fallback, and image model and reports per-model health.

## Best Possible Solution

Land #63259 or an equivalent narrow implementation that adds the CLI and route flag, threads `probeAll` through status options, expands configured model probe targets, preserves existing `--probe` defaults and filters, and updates focused docs/tests.

## Reproduction Assessment

Yes. Static inspection against current main reproduces the missing capability: no status CLI/route option or `probeAll` field exists, and `buildProbeTargets()` still plans one selected model per provider.

## Solution Assessment

Yes. Extending the existing `models status --probe` machinery is the narrowest maintainable solution because it reuses OpenClaw config resolution, auth profiles, provider filters, timeouts, and JSON rendering instead of adding provider-specific curl logic.
