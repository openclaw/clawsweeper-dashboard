## Summary

Keep open. Current main still lacks `openclaw models status --probe --all`, and the linked open PR #63259 remains the implementation candidate rather than a shipped fix.

## What This Changes

The issue requests a `models status --probe --all` mode that probes every configured primary, fallback, and image model instead of only checking one representative model per provider.

## Best Possible Solution

Keep the issue open until #63259 or an equivalent successor lands. The desired end state is a narrow status-level `--all` flag in both CLI paths, a threaded `probeAll` option through `modelsStatusCommand` and `runAuthProbes`, target expansion across all configured candidates while preserving current default behavior and filters, plus focused docs and tests.

## Reproduction Assessment

Yes. The missing capability is reproducible by static inspection of current main: no CLI/status option or `probeAll` field exists, and the target planner still selects `direct[0]` once per provider.

## Solution Assessment

Yes. A narrow `--probe --all` extension to the existing live probe machinery is the maintainable direction; it avoids provider-specific curl loops while preserving the existing auth-probe default path.
