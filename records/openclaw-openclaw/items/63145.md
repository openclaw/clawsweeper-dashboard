## What This Changes

Request adds a `--all` mode to `openclaw models status --probe` to live-check every configured primary, fallback, and image model instead of one representative model per provider.

## Best Possible Solution

Land the linked PR or an equivalent narrow implementation that preserves default one-model-per-provider probing, expands configured candidates only behind `--probe --all`, updates docs/types/tests, and includes redacted real CLI/provider proof.

## Reproduction Assessment

Yes for the feature gap: current `main` has no status-level `--all` or `probeAll` path, and `buildProbeTargets()` still selects one probe model per provider through `selectProbeModel()`. I did not run live provider probes in this read-only review.

## Solution Assessment

Yes in direction: extending existing `models status --probe` is the maintainable surface because it already owns config resolution, auth profiles, provider filtering, timeout controls, probe execution, and JSON rendering. The open PR still needs maintainer review and proof before it should close the issue.
