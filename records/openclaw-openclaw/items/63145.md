## Summary

Keep open: current main and v2026.6.8 still expose `models status --probe` as a provider/auth-profile probe that selects one model per provider, while the requested all-configured-model health sweep remains a product decision with a closed unmerged implementation attempt.

## What This Changes

Add `openclaw models status --probe --all` so OpenClaw probes every configured primary, fallback, and image model and reports per-model health.

## Best Possible Solution

Define one canonical model preflight contract for configured-model health, including the status `--all` flag semantics, JSON schema, provider/profile filters, timeout/concurrency behavior, and relationship to fallback-chain testing before implementation.

## Reproduction Assessment

Yes, at source level: current main has no `models status --all` flag and the probe planner still chooses one model per provider via `direct[0]`. I did not run live provider probes because the missing behavior is visible in command registration and probe-target source.

## Solution Assessment

Unclear until maintainers choose the product contract. Extending `models status --probe` is plausible, but it should be reconciled with the open fallback-test and staged-migration issues before a fix PR is queued.
