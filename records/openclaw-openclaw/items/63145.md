## Summary

Keep open because current main still lacks the requested all-configured-model probe mode, and the open implementation PR is the active fix path rather than a shipped resolution.

## What This Changes

Add `openclaw models status --probe --all` so status probes can health-check every configured model instead of one representative model per provider.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/63259 or an equivalent narrow implementation that adds the status flag, threads `probeAll` through routing and probe options, expands configured model targets, preserves default `--probe` behavior, and updates focused docs/tests.

## Reproduction Assessment

Yes. Static inspection against current main shows no status-level `--all`/`probeAll` path, while `buildProbeTargets()` still plans one selected model per provider through `selectProbeModel()`.

## Solution Assessment

Yes. Extending the existing `models status --probe` machinery is the narrowest maintainable direction because it reuses OpenClaw config resolution, auth profiles, provider filtering, timeout controls, probe execution, and JSON rendering.
