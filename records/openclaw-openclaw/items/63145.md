## Summary

Keep open because current main and the latest release still lack a status-level all-configured-model probe mode; the open implementation PR is the active fix path, not a shipped resolution.

## What This Changes

Request adds `openclaw models status --probe --all` to probe every configured primary, fallback, and image model and report per-model health.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/63259 or an equivalent narrow implementation with status-level `--all` plumbing, all-configured-model target expansion, docs, focused tests, and redacted real CLI/provider proof.

## Reproduction Assessment

Yes. Static inspection on current main shows no status-level `--all` or `probeAll` path, while `buildProbeTargets()` still chooses one model per provider through `selectProbeModel()`.

## Solution Assessment

Yes. Extending the existing `models status --probe` machinery is the narrowest maintainable direction because it reuses config resolution, auth profiles, provider filtering, timeout controls, probe execution, and JSON rendering.
