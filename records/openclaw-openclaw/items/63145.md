## Summary

Keep open: current main and latest release still expose `models status --probe` as a provider/profile probe that selects one model per provider, while the requested all-configured-model health sweep remains unimplemented and product-scoped.

## What This Changes

Add `openclaw models status --probe --all` to send live minimal requests to every configured primary, fallback, and image model and report per-model health.

## Best Possible Solution

Define one canonical configured-model preflight contract for `models status --probe --all`, including JSON shape, provider/profile filtering, timeout/concurrency behavior, and its relationship to fallback-chain testing.

## Reproduction Assessment

Yes. At source level, current main has no status-level `--all` or `probeAll` path, and `buildProbeTargets()` still selects one probe model per provider.

## Solution Assessment

Unclear until maintainers choose the product contract. Extending `models status --probe` is plausible, but it should be reconciled with fallback-chain testing and staged model migration before implementation is queued.
