## What This Changes

Request to add `openclaw models status --probe --all` so every configured model can be probed individually instead of only checking provider-level health.

## Best Possible Solution

Provide one canonical core model-execution probe with explicit scopes for all-configured-model health and fallback-order validation, preserving today’s one-model-per-provider default unless users opt into broader probing.

## Reproduction Assessment

## Solution Assessment

Unclear. Extending `models status --probe` is the maintainable surface for configured-model reachability, but maintainers still need to decide the relationship to fallback-chain testing, billable live calls, and output contract.
