## Summary

Keep open: current main does not implement Feishu `update` card actions, and the draft branch still has source-level blockers around the patch target, broadcast routing, final text size handling, and current SDK subpath drift. The PR also lacks inspectable after-fix real behavior proof.

## What This Changes

The PR adds a Feishu `update` card-action flow with a pending update registry, in-place card patching, duplicate final reply suppression, tests/docs, and a Signal runtime API entrypoint adjustment.

## Best Possible Solution

Keep the feature in the Feishu plugin, rebase onto current main, use callback-provided card IDs, thread card-update context through all reply paths, preserve Feishu chunk/fallback behavior, and align Signal imports with current SDK subpaths.

## Reproduction Assessment

Yes, source-level. The PR head shows the metadata-derived patch target, missing broadcast `cardUpdate` wiring, final patch bypass of existing chunking, and `signal-core` entrypoint drift against current main contracts.

## Solution Assessment

No. The direction is plausible, but this is not the best mergeable solution until the Feishu patch target, broadcast path, final-size fallback, and SDK subpath contract are corrected.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Use callback message ids for card updates:** `extensions/feishu/src/card-action.ts:310-314`
  - body: The update path takes the patch target from `envelope.m.messageId` and rejects callbacks without it. Use the parsed callback `open_message_id` after validation so normal callbacks work and stale or crafted card metadata cannot redirect the patch.
  - confidence: 0.89
- **[P2] Pass card-update context through broadcast dispatch:** `extensions/feishu/src/bot.ts:1023-1037`
  - body: The broadcast active-agent dispatcher is created without `cardUpdate`, so update actions in broadcast-enabled groups still use normal reply delivery instead of patching the source card. Thread the same card-update context into this dispatcher path.
  - confidence: 0.93
- **[P2] Reuse size fallback before final card patches:** `extensions/feishu/src/reply-dispatcher.ts:404-410`
  - body: Card-update finals send the full model text through one `updateCardFeishu` payload, bypassing the existing Feishu chunk/fallback path. Long outputs can make the patch fail and leave the card stuck in the processing state.
  - confidence: 0.86
- **[P2] Drop the retired signal-core SDK entrypoint:** `scripts/lib/plugin-sdk-entrypoints.json:188`
  - body: Current main has removed the `signal-core` public SDK subpath and the contract denies it, while this PR adds it back to the curated entrypoint list. Keep Signal on supported SDK subpaths after rebase.
  - confidence: 0.84
