## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use the callback card id for patches:** `extensions/feishu/src/card-action.ts:310-341`
  - body: The update flow reads messageId from card envelope metadata and uses that value for updateCardFeishu. Because that metadata is part of the clicked card payload, stale or manipulated payloads can make OpenClaw patch a different Feishu card than the one in the callback; use the callback-provided card id for the patch target.
  - confidence: 0.88
- **[P1] Pass cardUpdate into broadcast dispatchers:** `extensions/feishu/src/bot.ts:1141`
  - body: cardUpdateContext is only wired into the single-agent dispatcher. In groups using broadcast dispatch, the active dispatcher is created without cardUpdate, so update actions fall back to normal replies on the synthetic card-action message id and can reintroduce duplicate follow-up messages instead of patching the original card.
  - confidence: 0.9
- **[P2] Preserve the long-reply fallback before patching cards:** `extensions/feishu/src/reply-dispatcher.ts:404-410`
  - body: The card-update final text path patches one card payload directly instead of going through the existing chunking and fallback path used for normal Feishu replies. Long model outputs can exceed patch/card limits, fail the update, and leave the user with a processing card and no final text.
  - confidence: 0.86
