## What This Changes

The PR adds a Feishu `update` card-action flow with pending update state, in-place card patching, duplicate final-text suppression, Feishu tests/docs, and a Signal SDK entrypoint/import adjustment.

## Best Possible Solution

Rebase or replace this with a focused Feishu plugin change that uses callback-provided card ids, threads card-update context through every reply path, preserves existing chunk/fallback behavior, keeps Signal imports on current supported SDK/local seams, and includes redacted live Feishu proof.

## Reproduction Assessment

Yes, source-level. The PR head shows metadata-derived patch targeting, missing broadcast `cardUpdate` wiring, direct final card patching before the chunk/fallback helper, and `signal-core` entrypoint drift against current-main contracts.

## Solution Assessment

No. The feature direction is plausible, but this branch is not the best mergeable solution until the patch target, broadcast path, final-size fallback, SDK subpath contract, rebase scope, and real behavior proof are corrected.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Use callback message ids for card updates:** `extensions/feishu/src/card-action.ts:310-314`
  - body: The update path takes the patch target from `envelope.m.messageId` and rejects callbacks without that metadata field. Current callbacks already expose the clicked card id through `open_message_id`; use that parsed callback id so stale or crafted card metadata cannot redirect the patch.
  - confidence: 0.89
- **[P2] Thread card-update context through broadcast dispatch:** `extensions/feishu/src/bot.ts:1023-1037`
  - body: The broadcast active-agent dispatcher is created without `cardUpdate`, so update actions in broadcast-enabled groups still use normal reply delivery instead of patching the source card. Pass the same card-update context into this dispatcher path.
  - confidence: 0.92
- **[P2] Preserve reply size fallback before final card patches:** `extensions/feishu/src/reply-dispatcher.ts:404-410`
  - body: Card-update finals send the full model text through one `updateCardFeishu` payload, bypassing the existing chunk/fallback path. Long outputs can make the patch fail and leave the card stuck in the processing state.
  - confidence: 0.86
- **[P2] Drop the unsupported signal-core SDK entrypoint:** `scripts/lib/plugin-sdk-entrypoints.json:188`
  - body: Current main explicitly denies `signal-core` in the public SDK subpath list and has no matching `src/plugin-sdk/signal-core.ts`, but this branch adds that entrypoint and imports it from Signal runtime code. Keep Signal on the supported current SDK/local subpaths after rebase.
  - confidence: 0.84
