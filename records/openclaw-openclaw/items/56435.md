- **[P2] Use callback message ids for card updates:** `extensions/feishu/src/card-action.ts:310`
  - body: The update flow patches `metadata.messageId` from the card payload, while current callback parsing already exposes the clicked card id through `open_message_id`. Use the callback-provided id so stale or crafted card metadata cannot redirect the update to a different card.
  - confidence: 0.89

  - body: The broadcast active-agent dispatcher is created without `cardUpdate`, so `update` actions in broadcast-enabled groups still fall back to normal reply delivery instead of patching the source card. Pass the same card-update context into this dispatcher path.
  - confidence: 0.92

  - body: Card-update finals send the full model text through one `updateCardFeishu` payload, bypassing the existing chunk/fallback path used for normal Feishu replies. Long outputs can make the patch fail and leave the card stuck in the processing state.
  - confidence: 0.86

  - confidence: 0.84
