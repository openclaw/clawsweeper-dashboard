## Summary

Keep open. Current main does not implement the Feishu card-action update flow, and this draft PR still has source-level blockers around the patch target, broadcast dispatch, oversized final replies, and current-main SDK contract drift.

## What This Changes

The PR adds a Feishu `update` card-action flow with a pending update registry, in-place processing/final card patches, tests/docs, and a Signal runtime API/plugin SDK entrypoint adjustment.

## Best Possible Solution

Keep the feature in the Feishu plugin after rebase, patch the clicked callback card id, thread update context through broadcast, preserve Feishu chunk/fallback behavior, and keep the current SDK subpath contract.

## Reproduction Assessment

Yes. A high-confidence source-level path exists on the PR head: metadata-only patch targeting, broadcast dispatch without `cardUpdate`, and final text patching before chunk/fallback handling are visible in the changed files.

## Solution Assessment

No. The direction is useful, but the mergeable solution should use the callback card id, cover broadcast and oversized replies, and avoid reintroducing the retired `signal-core` SDK subpath.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Use callback message ids for card updates:** `extensions/feishu/src/card-action.ts:310-314`
  - body: The update branch patches the message id stored in card metadata and rejects callbacks when that metadata is absent. Feishu already provides the clicked card's `open_message_id`; using that callback id avoids normal callback rejection and prevents stale or crafted metadata from redirecting the patch.
  - confidence: 0.89
- **[P2] Pass card-update context through broadcast dispatch:** `extensions/feishu/src/bot.ts:1023-1037`
  - body: The broadcast active-agent dispatcher is still created without `cardUpdate`, so update actions in broadcast-enabled groups fall back to normal reply delivery instead of patching the source card. Thread the same card-update context into this dispatcher path.
  - confidence: 0.93
- **[P2] Reuse size fallback before final card patches:** `extensions/feishu/src/reply-dispatcher.ts:404-410`
  - body: Card-update finals send the full model text through one `updateCardFeishu` payload, bypassing the existing Feishu text chunk limit and fallback path. Long outputs can make the patch fail and leave the interaction card stuck in the processing state.
  - confidence: 0.86
- **[P2] Drop the retired signal-core SDK entrypoint:** `scripts/lib/plugin-sdk-entrypoints.json:188`
  - body: Current main explicitly keeps `signal-core` out of public plugin SDK subpaths and Signal runtime no longer imports it. Re-adding it conflicts with the current SDK contract after rebase.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: Needs attention because the new update path uses card metadata as the patch target instead of the callback's Feishu message id.

Concerns:

- **[medium] Card patch target comes from action metadata:** `extensions/feishu/src/card-action.ts:310`
