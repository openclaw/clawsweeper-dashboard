## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Use the callback message id for card patches:** `extensions/feishu/src/card-action.ts:310-341`
  - body: This path reads `messageId` from `envelope.m` and then uses it for both the pending update and `updateCardFeishu`. The callback event already carries the clicked card's `open_message_id`, so stale or manipulated card metadata can patch a different message than the one the user clicked.
  - confidence: 0.89
- **[P1] Propagate cardUpdate through broadcast replies:** `extensions/feishu/src/bot.ts:1023-1037`
  - body: The single-agent dispatcher passes `cardUpdate`, but the broadcast active-agent dispatcher is built without it. Broadcast-enabled Feishu groups will still use the normal reply path, so the original card is not updated in place and duplicate follow-up replies can reappear.
  - confidence: 0.91
- **[P2] Keep final card updates on the fallback path:** `extensions/feishu/src/reply-dispatcher.ts:404-410`
  - body: The card-update final reply calls `updateCardFeishu` once with the full text instead of using the existing chunking and fallback path. Long outputs can exceed Feishu patch/card limits, leave the card in processing, and drop the final text.
  - confidence: 0.84
