## Summary

Keep open: current main does not implement the Feishu in-place card-update flow, and this PR is still not mergeable because source review shows correctness and security blockers plus missing inspectable real behavior proof. A newer same-author Feishu card-update PR overlaps part of the work at https://github.com/openclaw/openclaw/pull/64027, so maintainers should decide the canonical path rather than closing this branch automatically.

## What This Changes

The branch adds a Feishu `update` card-action flow with a pending update registry, in-place card patching/final reply suppression, tests/docs, and a Signal runtime SDK subpath adjustment.

## Best Possible Solution

Rebase or replace this with a focused Feishu plugin feature that uses callback-provided card ids, threads card-update context through every reply path, preserves existing chunk/fallback behavior, aligns Signal imports with current SDK contracts, and includes redacted live Feishu proof.

## Reproduction Assessment

Yes, source-level. The PR head shows metadata-derived patch targeting, missing broadcast `cardUpdate` wiring, direct final card patching before the chunk/fallback helper, and `signal-core` entrypoint drift against current-main contracts.

## Solution Assessment

No. The direction is plausible, but this branch is not the best mergeable solution until the Feishu patch target, broadcast path, final-size fallback, SDK subpath contract, rebase scope, and real behavior proof are corrected.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Use callback message ids for card updates:** `extensions/feishu/src/card-action.ts:310-314`
  - body: The update path takes the patch target from `envelope.m.messageId` and rejects callbacks without that metadata field. Use the parsed callback `open_message_id` after validation so normal callbacks work and stale or crafted card metadata cannot redirect the patch.
  - confidence: 0.89
- **[P2] Thread card-update context through broadcast dispatch:** `extensions/feishu/src/bot.ts:1023-1037`
  - body: The broadcast active-agent dispatcher is created without `cardUpdate`, so update actions in broadcast-enabled groups still use normal reply delivery instead of patching the source card. Pass the same card-update context into this dispatcher path.
  - confidence: 0.92
- **[P2] Preserve reply size fallback before final card patches:** `extensions/feishu/src/reply-dispatcher.ts:404-410`
  - body: Card-update finals send the full model text through one `updateCardFeishu` payload, bypassing the existing Feishu chunk/fallback path. Long outputs can make the patch fail and leave the card stuck in the processing state.
  - confidence: 0.86
- **[P2] Drop the retired signal-core SDK entrypoint:** `scripts/lib/plugin-sdk-entrypoints.json:188`
  - body: Current main explicitly denies `signal-core` in the public SDK subpath list, but this branch adds it back and moves Signal runtime imports onto that subpath. Keep Signal on the supported current SDK/local subpaths after rebase.
  - confidence: 0.84
