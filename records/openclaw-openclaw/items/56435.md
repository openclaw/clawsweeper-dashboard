## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Use the callback message id for card patches:** `extensions/feishu/src/card-action.ts:310-341`
  - body: This path reads `messageId` from `envelope.m` and then uses it for both the pending update and `updateCardFeishu`. The callback event already carries the clicked card's `open_message_id`, so stale or manipulated card metadata can patch a different message than the one the user clicked.
  - confidence: 0.89
- **[P1] Propagate cardUpdate through broadcast replies:** `extensions/feishu/src/bot.ts:1023-1037`
  - body: Only the single-agent dispatcher receives `cardUpdate`; the broadcast active-agent dispatcher still uses the normal reply target. Broadcast-enabled groups will therefore post normal replies instead of updating the original card, reintroducing the duplicate-reply behavior this PR is meant to remove.
  - confidence: 0.86
- **[P2] Keep final card updates on the existing fallback path:** `extensions/feishu/src/reply-dispatcher.ts:404-410`
  - body: The final card-update branch sends the full model output as one `updateCardFeishu` patch and marks the update complete. Long outputs skip the existing chunking/fallback logic, so a rejected patch can leave the card in the processing state and drop the final text.
  - confidence: 0.82
