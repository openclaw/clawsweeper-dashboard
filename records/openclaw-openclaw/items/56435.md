## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Use callback message ids for card updates:** `extensions/feishu/src/card-action.ts:310-314`
  - body: The update path trusts `envelope.m.messageId` and rejects callbacks without that metadata, even though current callback parsing already exposes the clicked card id as `open_message_id`. Use the callback-provided card id so stale or crafted card metadata cannot redirect the patch to a different card.
  - confidence: 0.89
- **[P2] Thread card-update context through broadcast dispatch:** `extensions/feishu/src/bot.ts:1023-1037`
  - body: The broadcast active-agent dispatcher is created without `cardUpdate`, while the single-agent dispatcher receives it. In broadcast-enabled groups, update actions will still use normal reply delivery instead of patching the source card, so duplicate follow-up replies can reappear.
  - confidence: 0.92
- **[P2] Preserve reply-size fallback before final card patches:** `extensions/feishu/src/reply-dispatcher.ts:404-410`
  - body: Card-update finals send the full model text through one `updateCardFeishu` payload, bypassing the existing chunk/fallback path used for long Feishu replies. Long outputs can make the patch fail and leave the card stuck in the processing state.
  - confidence: 0.86
- **[P2] Drop the stale Signal SDK subpath change:** `scripts/lib/plugin-sdk-entrypoints.json:188`
  - body: Current main no longer has a `signal-core` SDK subpath, but this branch re-adds it to the entrypoint list and imports it from Signal runtime API. Rebase onto the current supported Signal seams instead of restoring the removed public subpath.
  - confidence: 0.84
