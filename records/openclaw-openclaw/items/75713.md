  - body: This branch emits any stored tool-call thoughtSignature into native Gemini replay whenever it is non-empty, even when the signature came from a different Google model/API/provider. Google's contract requires returning signatures in the original part, with the skip-validator reserved for transferred traces without a usable signature, so this can turn mixed-route history into invalid replay instead of a safe fallback.
  - confidence: 0.86

Replaying the opaque signature across route boundaries can attach a mismatched model-issued value to a different request path, which is exactly the trust boundary the focused replacement PR preserves.
  - confidence: 0.82
