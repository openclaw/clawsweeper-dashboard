## What This Changes

The PR prevents cron startup replay and backoff deferral from treating slots at or before the job’s latest persisted write as missed, with three restart regression tests.

## Best Possible Solution

Keep the single shared replay predicate and merge only with explicit owner acceptance that a later persisted job write supersedes best-effort missed-slot replay, prioritizing prevention of unintended outbound execution.

## Reproduction Assessment

Yes. The PR supplies a real CLI-to-Gateway-to-SQLite reproduction where current main replaces the API-updated future run with the 120-second startup deferral, while the patched run preserves it.

## Solution Assessment

Yes, subject to the explicit delivery-policy choice. Centralizing the same replay invariant for startup and backoff paths is narrower and less drift-prone than special-casing the reported schedule or adding another fallback.
