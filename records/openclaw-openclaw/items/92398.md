## What This Changes

The PR adds `state.scheduleActivatedAtMs`, writes it on cron creation and schedule/enabled updates, and uses it to bound restart missed-slot replay with a legacy fallback to `updatedAtMs`.

## Best Possible Solution

Keep the activation-boundary fix, but update the Gateway protocol cron state schema and focused protocol tests so the newly returned state field is an explicit additive API contract before merge.
