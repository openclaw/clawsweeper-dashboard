## What This Changes

The PR refactors cron startup missed-slot detection to ignore inferred cron slots at or before the job's last write time and adds restart catch-up regression coverage for updated schedules.

## Best Possible Solution

Land this PR's existing-updatedAtMs replay boundary if maintainers accept fail-safe suppression over late outbound sends, then close the companion PR and linked issue after merge.

## Reproduction Assessment

## Solution Assessment

Yes, with maintainer sign-off on the tradeoff. Reusing updatedAtMs is narrower than the companion PR's new persisted scheduleUpdatedAtMs state, but maintainers should explicitly accept that metadata-only writes can suppress best-effort catch-up.
