## What This Changes

The PR prevents cron startup catch-up and error-backoff handling from replaying missed slots that predate the job’s latest persisted definition write, with three restart regression scenarios.

## Best Possible Solution

Keep the single shared replay predicate and land this patch if maintainers accept that any later definition write invalidates older best-effort replay; retain the explanatory comment and all three regression scenarios as the contract.

## Solution Assessment

Yes. Given the stated outbound-safety policy, centralizing replay eligibility and bounding it by the last definition write is narrower and less drift-prone than maintaining separate startup and backoff comparisons.
