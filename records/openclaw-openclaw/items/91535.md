## Summary

Keep open: this PR targets the right Skill Workshop UI/Gateway mismatch, but it should not merge yet because the new scope resolver can turn default/global sessions into explicit `main` agent requests, regressing configured non-main default agents; it also lacks after-fix live Control UI/Gateway proof.

## What This Changes

The PR threads a session-derived `agentId` through Control UI Skill Workshop proposal list, inspect, apply, reject, and related detail-loading paths.

## Best Possible Solution

Resolve Skill Workshop scope with the same configured-default/session-default rules the UI uses elsewhere, keep explicit `agentId` for parsed agent sessions, add focused coverage for non-default and default-session cases, then provide redacted live UI/Gateway proof.

## Reproduction Assessment

Yes. The linked report plus current source show the UI sends unscoped proposal requests while Gateway scopes unscoped requests to the default agent, so non-default agent proposals are source-reproducible without a new backend API.

## Solution Assessment

No. Threading agent scope is the right direction, but this implementation must preserve configured-default behavior for unparsed/default session keys and add live behavior proof before merge.
