## Summary

Keep open: current main and the latest release still contain the LINE resolver behavior that ignores `channels.line.accounts.default` and treats configured named LINE accounts without `enabled: true` as disabled. The open closing PR https://github.com/openclaw/openclaw/pull/81471 already targets the exact account-resolution bug with focused tests and live LINE proof, so this issue should stay open until that PR lands or is replaced.

## What This Changes

The issue requests a fix for LINE multi-account configs where default and named account webhook routes fail to register and return 404.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81471, or an equivalent LINE-owned resolver fix, with coverage for `accounts.default`, named-account enablement, and explicit `enabled: false`.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: `accounts.default` credentials are ignored, and a configured named account without explicit `enabled: true` resolves disabled before webhook startup.

## Solution Assessment

Yes. The best path is the narrow LINE resolver fix already proposed in the open linked PR, not a core route workaround or a duplicate fix PR while that PR remains open.
