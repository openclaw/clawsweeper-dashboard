## Summary

Keep open: current `main` and `v2026.6.6` still contain the LINE account resolver behavior that skips `accounts.default` credentials and leaves configured named accounts disabled unless `enabled: true` is set. The issue is already paired with open closing PR https://github.com/openclaw/openclaw/pull/81471, so it should stay open until that PR lands or is replaced.

## What This Changes

The issue reports that LINE multi-account configs using `channels.line.accounts` prevent default and named webhook routes from registering, causing webhook POSTs to return 404.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81471 or an equivalent LINE-owned resolver fix with regression coverage for `accounts.default`, named-account enablement, and explicit `enabled: false` opt-out.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: `accounts.default` credentials are ignored, and a named account without `enabled: true` resolves disabled before gateway startup can register the webhook route.

## Solution Assessment

Yes. The narrow LINE resolver fix in the open linked PR is the best path; a core route workaround or a second repair PR would duplicate the active fix path.
