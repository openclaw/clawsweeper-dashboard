## What This Changes

The issue reports that LINE multi-account config under `channels.line.accounts` prevents both default and named webhook routes from registering, causing 404 responses.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/81471 with focused LINE resolver regression tests, then close this issue when that merge reaches main.

## Reproduction Assessment

Yes. Source inspection on current main shows `accounts.default` is skipped for the default LINE account and named accounts without `enabled: true` resolve disabled before `startAccount`, which prevents route registration.

## Solution Assessment

Yes. The best fix is the narrow LINE resolver change in the linked PR with regression tests; webhook routing itself can stay unchanged.
