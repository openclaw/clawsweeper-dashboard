Keep open: current main and the latest release still have the LINE resolver behavior that explains the reported 404s, and https://github.com/openclaw/openclaw/pull/81471 is the active open fix candidate with closing syntax.

## What This Changes

The issue reports that configuring LINE with `channels.line.accounts` prevents default and named webhook routes from registering, causing POST requests to return 404.

## Best Possible Solution

Review and land or explicitly reject https://github.com/openclaw/openclaw/pull/81471; after it merges, close this issue with the merge commit and release provenance.

## Reproduction Assessment

Yes. Source inspection shows `resolveLineAccount` skips `accounts.default` for the default LINE account and resolves configured named accounts without explicit `enabled: true` as disabled, which makes gateway startup return before webhook registration.

## Solution Assessment

Yes. The linked PR fixes the LINE account resolver, which is the narrow owner boundary before gateway route registration; a new separate fix lane would duplicate the active candidate.
