## Summary

Keep open: current main and v2026.6.6 still contain the LINE account resolver behavior that ignores `channels.line.accounts.default` and disables configured named accounts unless `enabled: true` is set. The open closing PR https://github.com/openclaw/openclaw/pull/81471 targets the exact bug with focused tests and live LINE proof, so this issue should stay open until that PR lands or is replaced.

## What This Changes

The issue reports that LINE multi-account config under `channels.line.accounts` makes default and named webhook routes return 404 instead of registering.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81471, or an equivalent LINE-owned resolver fix, with coverage for `accounts.default`, named-account default enablement, and explicit `enabled: false`.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: `accounts.default` credentials are ignored, and a configured named account without explicit `enabled: true` resolves disabled before webhook startup.

## Solution Assessment

Yes. The best path is the narrow LINE resolver fix already proposed in the open linked PR, not a core route workaround or a duplicate fix PR while that PR remains open.
