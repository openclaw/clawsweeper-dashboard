## Summary

Current main still matches the reported LINE multi-account startup failure, so this should stay open. The default account can be listed from `channels.line.accounts.default` while the resolver ignores that entry for credentials, named accounts without `enabled: true` resolve disabled, and the closed #47528 fix was not merged.

## What This Changes

The issue reports that LINE multi-account configuration using `channels.line.accounts` prevents default and named webhook routes from registering, causing webhook requests to return 404.

## Best Possible Solution

Fix the LINE plugin account resolver so `accounts.default` supplies default credentials, configured named accounts are enabled unless explicitly disabled, top-level `channels.line.enabled: false` still disables all accounts, and focused regression tests cover startup or route registration.

## Reproduction Assessment

Yes, by source inspection on current main. The supplied config maps directly to the resolver skipping `accounts.default`, disabling `twgreen` by default, and returning before LINE route registration.

## Solution Assessment

Yes. The narrowest maintainable fix is in the LINE plugin account resolver and its tests; a core HTTP route or registry change would miss the upstream account-resolution failure.
