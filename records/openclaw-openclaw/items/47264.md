## What This Changes

The issue reports that LINE multi-account configs using `channels.line.accounts` make both default and named webhook endpoints return 404 because account credentials and enablement are not resolved as expected.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81471 or an equivalent LINE-owned resolver fix that covers `accounts.default`, configured named-account enablement, and explicit `enabled: false`.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: `accounts.default` credentials are ignored for the default account, and a configured named account without explicit `enabled: true` resolves disabled before gateway startup can register its webhook route.

## Solution Assessment

Yes. The best path is the narrow LINE resolver fix already proposed in the open linked PR, not a core route workaround or a duplicate fix PR while that PR remains open.
