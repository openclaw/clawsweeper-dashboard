## Summary

Keep open: current main and v2026.6.6 still contain the LINE resolver behavior that skips `accounts.default` credentials and disables named accounts unless `enabled: true` is set. The issue is paired with an open closing PR, https://github.com/openclaw/openclaw/pull/81471, so the right cleanup path is to land or replace that PR before closing.

## What This Changes

The issue reports that LINE multi-account config under `channels.line.accounts` leaves default and named webhook routes unregistered, returning 404.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81471, or an equivalent LINE-owned resolver fix, with regression coverage for `accounts.default`, named-account enablement, and explicit `enabled: false` opt-out.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main reproduction path: `accounts.default` credentials are skipped and named accounts without explicit `enabled` resolve disabled before webhook startup.

## Solution Assessment

Yes. The best fix is the narrow LINE resolver change already proposed in the open linked PR, not a core route workaround or a second repair PR while that landing path remains open.
