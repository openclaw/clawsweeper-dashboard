## Summary

Keep open: current main still source-reproduces the LINE multi-account resolver bug, and the open linked fix PR references this issue with closing syntax. The issue should close only after that PR lands or is replaced.

## What This Changes

The issue reports that LINE multi-account config makes both the default and custom webhook routes return 404 because the configured accounts do not start.

## Best Possible Solution

Land or replace the linked LINE resolver fix with focused regression tests, then close this issue as implemented by that merge.

## Reproduction Assessment

Yes. Source inspection on current main shows accounts.default is ignored for the default LINE account and configured named accounts default to disabled, so gateway startup returns before LINE startAccount can register webhook routes.

## Solution Assessment

Yes. The narrow maintainable solution is to fix the LINE account resolver in extensions/line/src/accounts.ts and cover the resolver cases in extensions/line/src/accounts.test.ts; the open linked PR follows that boundary.
