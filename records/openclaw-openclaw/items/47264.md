## Summary

Current main still matches the reported LINE multi-account startup failure, so this should stay open. The resolver lists `channels.line.accounts.default` but ignores that entry for default credentials, named accounts without `enabled: true` resolve disabled, and startup returns before the LINE monitor can register webhook routes.

## What This Changes

The issue requests fixing LINE multi-account startup so `channels.line.accounts.default` and configured named accounts register webhook routes instead of returning 404.

## Best Possible Solution

Fix the LINE plugin account resolver so `accounts.default` supplies default credentials, configured named accounts are enabled unless explicitly disabled, top-level `channels.line.enabled: false` still disables all accounts, and focused regression tests cover the startup path.

## Reproduction Assessment

Yes, by source inspection on current main. The supplied config maps to `accounts.default` being listed but ignored for default credentials, the named account resolving disabled, and gateway startup returning before LINE route registration.

## Solution Assessment

Yes. The narrowest maintainable fix is in the LINE plugin account resolver and its tests; changing core HTTP route registration would miss the upstream account-resolution failure.
