## Summary

Keep open: current main still has the LINE account-resolution behavior that explains the reported 404s, and the directly linked fix pull request was closed unmerged. The remaining work is narrow and belongs in the LINE plugin resolver, not in generic gateway route registration.

## What This Changes

The issue reports that LINE multi-account configuration using `channels.line.accounts` leaves LINE webhook routes unregistered and returning 404.

## Best Possible Solution

Update LINE account resolution so `accounts.default` participates in default-account config resolution and configured named LINE accounts start unless explicitly disabled, with focused resolver regression tests.

## Reproduction Assessment

Yes, by source inspection on current main: the default account never reads `channels.line.accounts.default`, and named accounts without explicit `enabled: true` are skipped before provider startup. I did not run a live gateway/LINE webhook in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is in the LINE plugin account resolver, with tests for `accounts.default`, implicit named-account enablement, and explicit disable behavior. The closed unmerged linked pull request confirms the same fix direction but does not resolve the issue on main.
