## Summary

Keep open: current main and the latest release still contain the LINE resolver behavior that can skip `channels.line.accounts.default` credentials and treat named accounts as disabled unless `enabled: true` is set. The issue is already paired with the open closing PR at https://github.com/openclaw/openclaw/pull/81471, so it should stay open until that PR lands or is replaced.

## What This Changes

The issue reports that LINE multi-account configuration under `channels.line.accounts` leaves both default and named webhook routes unregistered and returning 404.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81471 or an equivalent LINE-owned resolver fix with regression coverage for `accounts.default`, named-account enablement, and explicit `enabled: false` opt-out.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: `accounts.default` credentials are ignored, and a named account without `enabled: true` resolves disabled before webhook startup.

## Solution Assessment

Yes. The best fix is the narrow LINE resolver change already proposed in the open linked PR, not a new core route workaround or a separate fix PR while that PR remains open.
