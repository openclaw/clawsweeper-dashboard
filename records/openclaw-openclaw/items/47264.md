## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/47264 on current main. The observable bug is that LINE multi-account configs with `channels.line.accounts.default` and a configured named account can fail to start providers and therefore return 404 on webhook paths. Keep the fix confined to LINE account resolution unless a test proves another narrow helper is required. Update `extensions/line/src/accounts.ts` so the default account reads `channels.line.accounts.default`, named configured LINE accounts are enabled unless explicitly `enabled: false`, and existing single-account/env/file credential behavior remains intact. Add focused regressions in `extensions/line/src/accounts.test.ts` for `accounts.default` credentials and webhookPath, top-level fallback/override behavior, implicit named-account enablement, and explicit disable. Do not change generic gateway route registration, setup writer behavior, or unrelated channel semantics. Add a changelog entry if the repair lane policy requires one for this user-visible fix.
