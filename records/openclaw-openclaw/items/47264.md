## What This Changes

The issue asks for LINE multi-account configs to register both default and named account webhook routes when credentials live under `channels.line.accounts`.

## Best Possible Solution

Review and land the linked LINE resolver PR if it remains correct, then close this issue as implemented by that merge.

## Reproduction Assessment

Yes. The current-main resolver skips `accounts.default` for the default account and marks configured named LINE accounts disabled unless `enabled: true` is set, so gateway startup returns before webhook registration.

## Solution Assessment

Yes. The narrow maintainable solution is to fix `extensions/line/src/accounts.ts` and cover the resolver cases in `extensions/line/src/accounts.test.ts`; the open linked PR follows that boundary.
