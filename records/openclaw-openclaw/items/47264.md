## What This Changes

Fix LINE multi-account configuration so `accounts.default` and configured named accounts register their webhook routes instead of returning 404.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/81471 with focused LINE resolver regression tests, then close this issue when that fix reaches main.

## Reproduction Assessment

Yes. Source inspection on current main shows `accounts.default` is skipped for the default LINE account and configured named accounts without `enabled: true` resolve disabled before `startAccount`, which prevents route registration.

## Solution Assessment

Yes. The narrow maintainable fix is in the LINE account resolver: read `accounts.default` for the default account and default configured named accounts to enabled unless explicitly disabled; webhook route handling can stay unchanged.
