## Summary

Keep open: current main still source-reproduces the LINE multi-account resolver bug, and the open closing PR at https://github.com/openclaw/openclaw/pull/81471 should be reviewed or replaced before this issue closes.

## What This Changes

The issue reports that LINE multi-account config under `channels.line.accounts` causes default and named webhook routes to return 404 instead of registering both providers.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/81471 with the focused LINE resolver fix and regression tests, then close this issue as implemented.

## Reproduction Assessment

Yes. Source inspection on current main shows `accounts.default` is skipped for the default LINE account, configured named accounts without `enabled: true` resolve disabled, and gateway startup returns before webhook registration.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the LINE account resolver: read `accounts.default` for the default account and default configured named accounts to enabled unless explicitly disabled, while leaving route registration unchanged.
