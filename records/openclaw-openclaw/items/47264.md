- `impact:message-loss`: The 404 webhook route prevents inbound LINE events from reaching OpenClaw for affected accounts.

## Summary

Keep open: current main and the latest release still contain the LINE account-resolution behavior that skips `channels.line.accounts.default` and treats configured named LINE accounts without `enabled: true` as disabled. The open linked PR https://github.com/openclaw/openclaw/pull/81471 targets the exact bug with focused resolver tests and live LINE/ngrok proof, so this issue should stay open until that PR lands or is replaced.

## What This Changes

The issue asks OpenClaw to make LINE multi-account configs register both the default and named webhook routes instead of returning 404.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/81471, then close this issue after main contains the LINE-owned resolver fix and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main reproduction path: `accounts.default` credentials are ignored, named accounts without explicit `enabled: true` resolve disabled, and gateway startup skips disabled or unconfigured accounts before route registration.

## Solution Assessment

Yes. The linked PR is the best narrow fix because it changes the LINE-owned resolver, matches the sibling multi-account semantics, preserves explicit `enabled: false`, and avoids core or plugin API churn.
