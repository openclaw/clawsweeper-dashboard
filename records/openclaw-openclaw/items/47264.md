## Summary

Keep open: current main still matches the reported LINE multi-account failure by source inspection, and the linked fix PR was closed unmerged. The broken behavior is narrow and belongs in the LINE plugin account resolver.

## What This Changes

Allow LINE multi-account configs to start `accounts.default` and named accounts so their webhook routes register instead of returning 404.

## Best Possible Solution

Patch the LINE plugin resolver so `accounts.default` participates in default-account credential/config resolution and configured named accounts start unless explicitly disabled, then add focused regression tests and a changelog entry.

## Reproduction Assessment

Yes, by source inspection on current main: a config using `channels.line.accounts.default` yields no default account credentials, and a named account without `enabled: true` is skipped before webhook registration. I did not run a live gateway/LINE webhook in this read-only sweep.

## Solution Assessment

Yes, the narrow maintainable fix is in LINE account resolution semantics, not in the generic gateway route registration path. PR #47528 identified the same direction but is closed unmerged, so a fresh or revived patch is still needed.

Update the LINE account resolver so the default account reads `accounts.default` credentials/config while preserving existing top-level/env/file fallback precedence, and so configured named accounts are enabled unless `enabled: false` is set.
