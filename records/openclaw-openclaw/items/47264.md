## Summary

Keep open: current main and the latest shipped release still have the LINE account-resolution behavior described in the report, while the related fix PR was closed unmerged.

## What This Changes

The issue reports that LINE multi-account config prevents webhook routes from registering, returning 404 for default and named accounts.

## Best Possible Solution

Fix LINE account resolution so accounts.default supplies the default account credentials and configured named accounts start unless explicitly disabled, with focused regression tests around resolver and startup-visible account state.

## Reproduction Assessment

Yes, source-reproducible on current main. The resolver ignores accounts.default for the default account and marks named accounts without enabled=true as disabled before gateway startAccount can register routes.

## Solution Assessment

Yes. A narrow LINE plugin resolver fix, matching the core direction of the closed unmerged PR, is the maintainable path; core gateway route registration does not need a product change.
