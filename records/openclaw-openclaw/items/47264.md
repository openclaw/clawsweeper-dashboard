## Summary

Keep open: current main and the latest release still source-reproduce the LINE multi-account resolver bug, and the open linked PR with closing syntax is the right implementation path. The issue should close only after that PR lands or is replaced.

## What This Changes

The issue asks OpenClaw to register LINE webhook routes for both `channels.line.accounts.default` and configured named LINE accounts instead of returning 404.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81471, or a narrow replacement with the same resolver boundary and regression tests, then close this issue as implemented.

## Reproduction Assessment

Yes. Source inspection on current main shows `accounts.default` is ignored for the default LINE account and configured named accounts default to disabled, so gateway startup returns before LINE `startAccount` can register webhook routes.

## Solution Assessment

Yes. The linked PR fixes the bug at the LINE account resolver boundary, preserves the prior name-resolution concern from the earlier closed attempt, and adds focused resolver regression tests without changing webhook handling.
