## Summary

Keep open: current main and v2026.6.6 still contain the LINE resolver behavior that ignores `channels.line.accounts.default` and disables configured named accounts unless `enabled: true` is set. The issue is already paired with the open closing PR https://github.com/openclaw/openclaw/pull/81471, so it should remain open until that PR lands or is replaced.

## What This Changes

The issue requests that LINE multi-account configs under `channels.line.accounts` register the default and named account webhook routes instead of returning 404.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81471, or replace it with an equivalent LINE-owned resolver fix and regression coverage, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes. Source inspection against current main shows `resolveLineAccount` ignores `channels.line.accounts.default` and defaults named accounts without `enabled` to disabled, and gateway startup skips disabled accounts before route registration.

## Solution Assessment

Yes. The best path is the narrow LINE-owned resolver fix already proposed in https://github.com/openclaw/openclaw/pull/81471; a core HTTP route workaround would miss the account-resolution root cause.
