## Summary

This should stay open: the linked repair PR was closed without merging, and current main still has the LINE account-resolution behavior that makes the reported multi-account config unconfigured or disabled before webhook routes are registered.

## What This Changes

The issue asks for LINE multi-account configs under `channels.line.accounts` to start both the default and named accounts and register their webhook routes.

## Best Possible Solution

Update LINE account resolution to merge `channels.line.accounts.default` for the default account, treat configured named accounts as enabled unless explicitly disabled while respecting channel-level disable, and add focused regression tests for default and named webhook account startup inputs.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: the reporter's config leaves the default account without resolved credentials and the named account disabled, so gateway startup returns before LINE route registration. I did not run the live gateway in this read-only pass.

## Solution Assessment

Yes. The narrowest maintainable fix is in LINE account resolution and tests, aligning LINE with the existing Telegram/Discord/Slack multi-account merge and enabled semantics rather than changing webhook registration itself.

## Repair Work Prompt

Fix the LINE multi-account bug reported at https://github.com/openclaw/openclaw/issues/47264. In current main, `extensions/line/src/accounts.ts` ignores `channels.line.accounts.default` when resolving the default account and treats named accounts without `enabled: true` as disabled, so gateway startup skips accounts before webhook route registration. Implement the narrow resolver fix in the LINE plugin only: merge account-scoped config for `accounts.default`, make configured named accounts enabled unless explicitly `enabled: false`, preserve channel-level `enabled: false`, and align with existing Telegram/Discord/Slack account-resolution semantics where practical. Add regression coverage in `extensions/line/src/accounts.test.ts` for the reporter-style config with `accounts.default` plus a named account with `webhookPath`, default-account credential override, named-account enabled-by-default, and explicit disable. Do not change webhook handler logic, HTTP route registration, setup writer behavior beyond what is necessary for resolver consistency, or unrelated plugins. Add a user-facing bug-fix changelog entry.
