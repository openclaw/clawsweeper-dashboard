## Summary

Keep open: current main still matches the reported LINE multi-account resolver failure, and the only linked fix PR is closed unmerged. The bug is source-reproducible without a product decision: `accounts.default` is ignored for the default account, while configured named accounts without `enabled: true` resolve disabled before the gateway can call `startAccount` and register webhook routes.

## What This Changes

The issue reports that LINE multi-account configuration under `channels.line.accounts` breaks webhook route registration, causing default and named LINE webhook endpoints to return 404.

## Best Possible Solution

Fix LINE account resolution so `accounts.default` supplies default credentials and configured named accounts start unless explicitly disabled, with focused regression coverage around resolver output and gateway-startup-visible account state.

## Reproduction Assessment

Yes, source-reproducible on current main. A config using `channels.line.accounts.default` resolves the default account without account-scoped credentials, and a named configured account without `enabled: true` resolves disabled before route startup.

## Solution Assessment

Yes. The narrow maintainable fix is in `extensions/line/src/accounts.ts` plus focused tests; the core gateway route registration path does not need a product or API change.

Current main ignores `channels.line.accounts.default` when resolving the default account and treats configured named LINE accounts as disabled unless each account explicitly sets `enabled: true`; this prevents gateway startup from registering webhook routes.
