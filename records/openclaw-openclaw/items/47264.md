## Summary

Keep open: current main still source-reproduces the LINE multi-account resolver bug, and open PR https://github.com/openclaw/openclaw/pull/81471 references this issue with closing syntax. The issue should close after that PR lands or is replaced.

## What This Changes

The issue reports that configuring LINE with `channels.line.accounts` prevents both default and named webhook routes from registering, causing 404 responses.

## Best Possible Solution

Land or replace the focused LINE resolver fix with regression tests, then close this issue as implemented by that merge.

## Reproduction Assessment

Yes. Source inspection on current main shows `accounts.default` is ignored for the default LINE account and configured named accounts resolve disabled unless explicitly enabled, so gateway startup returns before LINE webhook registration.

## Solution Assessment

Yes. The narrow maintainable solution is to fix the LINE account resolver in `extensions/line/src/accounts.ts` and cover the resolver cases in `extensions/line/src/accounts.test.ts`; the open linked PR follows that boundary.
