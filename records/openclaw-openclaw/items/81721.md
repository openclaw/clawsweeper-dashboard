## Summary

Keep open: the feature is useful and has supplied real provider proof, but the PR still has a source-reproducible prompt-handling bug for the OpenAI Codex media provider before merge.

## What This Changes

The PR adds diarized JSON audio request fields, normalized transcript segment preservation, Plugin SDK type exports, and focused media-understanding tests.

## Best Possible Solution

Keep the additive segment contract, but generalize the diarized prompt omission so every OpenAI-compatible path that reaches the helper can use the default-prompt path safely before landing.

## Reproduction Assessment

Not applicable as a bug reproduction for the feature; the PR body supplies live provider output for the intended behavior. The remaining review finding is source-reproducible because openai-codex uses the same OpenAI-compatible helper, but the runner only omits the default prompt for providerId openai.

## Solution Assessment

Mostly yes, but not yet: reusing providerOptions and preserving normalized segments through existing result/output contracts is narrow. The safer fix is to base the prompt omission on the diarized model or response format for all OpenAI-compatible helper paths, not only the openai provider id.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Omit default prompts for all diarized OpenAI-compatible audio paths:** `src/media-understanding/runner.entries.ts:453-454`
  - body: `resolveAudioPromptForProvider` returns the default audio prompt for every provider except `openai`. The bundled `openai-codex` media provider also calls `transcribeOpenAiCompatibleAudio`, whose new guard rejects `gpt-4o-transcribe-diarize` whenever a prompt is present; with a diarized `openai-codex` config, the runner will inject the default `Transcribe the audio.` prompt and then fail before the provider call even though the user did not configure a prompt. Base this omission on the model/response format or include the OpenAI-compatible Codex path too.
  - confidence: 0.9
