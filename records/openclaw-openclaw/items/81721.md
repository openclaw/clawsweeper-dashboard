## Summary

Keep this PR open: the diarized audio segment seam is useful and not present on current main or the latest release, and the supplied real behavior proof is sufficient, but PR head still has a source-proven OpenAI Codex prompt-guard bug that should block merge.

## What This Changes

The PR adds diarized audio transcript segment types/output, forwards OpenAI-compatible `response_format` and `chunking_strategy` options, exports the segment type through the Plugin SDK, and adds focused tests.

## Best Possible Solution

Keep the provider-option and segment-preservation seam, but make the diarized prompt handling cover every OpenAI-backed audio provider or centralize it in the OpenAI-compatible helper with regression coverage.

## Reproduction Assessment

Yes for the review finding: source inspection shows `openai-codex` uses the same OpenAI-compatible audio helper, while the new runner guard only treats `openai` as diarized-prompt-unsupported. I did not run tests in this read-only sweep.

## Solution Assessment

No as-is: the main seam is maintainable, but the prompt guard is too provider-specific for a registered OpenAI-backed provider. The safer solution is to include `openai-codex` or centralize the implicit-prompt suppression where the helper already knows the model/options.
