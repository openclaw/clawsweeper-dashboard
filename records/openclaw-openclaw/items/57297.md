## Summary

Keep this issue open. Current main has WebChat audio embedding and inline playback improvements, but the requested structured TTS generation/route/delivery events, tool-to-delivery correlation id, and TTS-specific final delivery/playback status are still missing; the companion playback report #57296 and broader WebChat audio issue #65859 are also still open.

## What This Changes

Add structured WebChat TTS generation, routing, and delivery telemetry with correlation IDs, plus optional user-visible final delivery/playback status in the Control UI.

## Best Possible Solution

Implement sanitized structured telemetry around the existing TTS tool and WebChat media promotion path: generation start/success/failure, WebChat route selection, delivery/embed success or failure with reason codes, and a run/tool correlation id. Add a small Control UI feedback path only for final TTS delivery/playback state, leaving broader audio upload/player work to #65859.

## Reproduction Assessment

Yes. The missing observability is reproducible by static inspection on current main: the requested event names and correlation fields are absent, while #57296 supplies concrete runtime steps for the opaque WebChat TTS symptom this issue is meant to diagnose.

## Solution Assessment

Yes, the requested direction is the right narrow fix if scoped to telemetry and final status. The safer implementation is to instrument existing TTS/WebChat media seams rather than changing provider synthesis, local media root policy, or the broader WebChat audio upload feature set.
