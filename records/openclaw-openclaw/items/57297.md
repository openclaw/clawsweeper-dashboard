## Best Possible Solution

Define an additive WebChat TTS telemetry/status contract keyed to existing tool or session identifiers, then emit generation, route, and delivery/playback outcomes from the existing TTS and delivery paths.

## Solution Assessment

Unclear until maintainers choose the event and UI status contract. The narrow maintainable direction is additive diagnostics and status on existing TTS/WebChat delivery paths, not another provider synthesis or audio-player rewrite.
