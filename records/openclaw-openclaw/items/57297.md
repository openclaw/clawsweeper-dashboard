## Summary

Keep open. Current main has WebChat audio embedding, inline playback, and generation attempt state, but it still lacks the requested WebChat TTS route/delivery outcome telemetry, tool-to-delivery correlation id, and TTS-specific final UI status.

## What This Changes

Request adds structured WebChat TTS generation, routing, delivery, correlation, and optional UI status signals.

## Best Possible Solution

Define an additive WebChat TTS telemetry/status contract keyed by existing run or tool ids, then implement gateway events and focused UI display without changing provider synthesis or generic audio playback.

## Solution Assessment
