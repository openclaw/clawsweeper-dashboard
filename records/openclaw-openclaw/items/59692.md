## Summary

Keep open. This PR has a protected `maintainer` label and an explicit HOLD comment, while current main and v2026.5.22 only partially cover the idea through the generic Barnacle external-plugin candidate path, not this PR's separate `r: plugin-clawhub` / `core-plugin-approved` workflow.

## What This Changes

The PR adds GitHub Actions automation that detects newly added plugin manifests and packages under `extensions/`, applies an `r: plugin-clawhub` close label, and lets `core-plugin-approved` remove the close label and reopen for maintainer override.

## Best Possible Solution

Keep the shipped generic Barnacle external-plugin candidate path unless maintainers explicitly want a stricter core-plugin-specific policy, then port it into `scripts/github/barnacle-auto-response.mjs` with idempotent reopen handling and documented override semantics.
