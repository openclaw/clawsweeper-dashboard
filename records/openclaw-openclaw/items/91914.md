## Label Justifications

- `P1`: A routine same-session follow-up can interrupt or stall the active response delivered to an external channel.
- `impact:message-loss`: Later reply chunks or final text can be absent from the originating channel even though the session turn continues.

## Summary

Current main still creates the foreground delivery fence before per-session turn admission. An older delivery waits while a newer generation is active, while that newer same-session turn can simultaneously wait for the older reply run to finish. The closest regression test settles the newer mocked dispatch immediately and therefore does not cover this admission-ordering cycle, so the issue remains necessary.

## What This Changes

Prevent a queued same-session inbound from blocking or suppressing the active turnâ€™s later source-channel delivery before the queued turn is admitted.

## Best Possible Solution

Move foreground freshness participation to a point after same-session turn admission, or otherwise make queued generations non-blocking, and add an integrated regression proving the first turn continues channel delivery while the second waits and then runs normally.

## Reproduction Assessment

Yes at source level with high confidence: the older delivery can wait for a newer active fence while that newer visible turn waits for the older same-session run to become idle. This read-only review did not execute a failing runtime test.

## Solution Assessment

The suggested lazy-fence repair is a plausible bounded approach, but the best implementation should explicitly eliminate pre-admission freshness participation and exercise the real admission path while preserving existing visible-delivery winner semantics.
