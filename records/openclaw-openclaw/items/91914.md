## Summary

Keep open: current main and v2026.6.8 still create the foreground reply fence at buffered-dispatch entry before deeper same-session admission, and the linked fix PRs remain open rather than merged.

## What This Changes

Fix foreground reply fencing so a queued same-session inbound does not stop the active turn's originating channel/source delivery before the queued turn is admitted.

## Best Possible Solution

Land a corrected lazy foreground-fence fix through the existing candidate PR path, preserving newer-visible stale suppression and adding focused freshness coverage.

## Reproduction Assessment

Yes, at source level: current main creates the foreground fence before same-session reply admission and the guard can cancel older source delivery behind a newer generation. I did not run a live external-channel replay in this read-only review.

## Solution Assessment

Yes in direction: deferring foreground fence creation until visible/source delivery begins is the right layer. The best fix still has to preserve newer-visible stale suppression, so the open candidate PRs need review or repair rather than closing this issue.
