## Summary

Keep this PR open. Current main does not implement the hidden `chat.send` user-turn suppression or URL-driven WebChat autostart flow, and the PR discussion shows earlier auditability, stale-prompt, retry, and attachment edge cases were addressed in follow-up pushes; the remaining issue I found is a documentation gap for the new public protocol/WebChat behavior.

## What This Changes

The PR adds an optional admin-gated `hideUserMessage` flag to `chat.send`, wires Control UI `autostart` URL handling for one-shot hidden bootstrap turns on empty chat sessions, updates Swift gateway models, and adds UI/gateway tests for the hidden prompt flow.

## Best Possible Solution

Keep the implementation path as an additive Gateway protocol field plus Control UI-only URL bootstrap flow, but require a small docs update for the new parameter, scope requirement, unsupported combinations, and supported `autostart` values before landing.

## Reproduction Assessment

Yes. Current main can be checked by inspecting the schema/UI/gateway paths: there is no `hideUserMessage` field or autostart parser, while existing `sendChatMessage` and `chat.send` paths visibly echo and persist the user turn.

## Solution Assessment

Yes, mostly. The PR uses a narrow optional protocol field, admin-gates transcript hiding, rejects risky deliver/attachment combinations, and keeps normal sends wire-compatible; the safer remaining improvement is documenting the new public contract.
