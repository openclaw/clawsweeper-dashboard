## Summary

Keep open. Current main has partial macOS LaunchAgent restart/recovery hardening, but the missing-service recovery path and cron failure-notification accounting are still not fully resolved by the inspected source.

## What This Changes

The issue reports that scheduled auto-upgrade failures can be invisible or misreported and that macOS LaunchAgent gateway recovery may not restore a not-loaded or not-installed service.

## Best Possible Solution

Define the macOS service ownership contract, then add focused recovery and accounting behavior: installed-but-unloaded recovery, explicit missing-service outcomes, readiness waits, and persisted failure-notification attempt results.

## Reproduction Assessment

No. This review did not run a live current-main macOS cron/update reproduction, but source inspection still shows the missing-service update recovery gap and fire-and-forget failure-notification accounting path.

## Solution Assessment

Not applicable as a patch review because there is no proposed fix here. The narrow maintainable direction is to settle macOS service ownership first, then cover update recovery and cron notification accounting with focused tests.
