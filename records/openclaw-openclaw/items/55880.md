## Summary

Keep open. Current main has meaningful partial fixes for installed-but-unloaded LaunchAgent recovery, managed update handoff, and cron failure-notification accounting, but source and related open PRs still show unresolved policy/work around missing-service recovery and delivery-status semantics.

## What This Changes

Issue reports scheduled auto-upgrade failures that are not reliably visible to the user and macOS LaunchAgent gateway recovery that does not restore a missing or unloaded service.

## Best Possible Solution

Keep this as the umbrella until maintainers decide the macOS missing-service contract, then land narrow changes that distinguish installed-but-unloaded repair from service-not-installed guidance and preserve accurate cron notification delivery state.

## Reproduction Assessment

No, not as a live end-to-end macOS cron/update reproduction. Source inspection does give a high-signal path for the missing-service gap: package updates only prepare restarts for installed services, LaunchAgent recovery requires an existing plist, and doctor install still needs interactive confirmation.

## Solution Assessment

Not applicable as a patch review because no fix is proposed here. The maintainable direction is to settle the macOS service ownership policy first, then add focused recovery/status behavior and tests around update, doctor, and cron delivery accounting.
