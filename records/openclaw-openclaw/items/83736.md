## Summary

Keep this open: current main and the latest release still reject non-probe node clients whose protocol range excludes the gateway protocol, and the related Android issue only covers the Play Store artifact slice, not the broader subordinate-node maintenance path.

## What This Changes

The issue asks the gateway to tolerate minor protocol skew so older subordinate nodes can still be upgraded, restarted, or inspected after a gateway upgrade.

## Best Possible Solution

Define a supported gateway/node skew policy and implement a restricted degraded maintenance channel for already-paired older nodes, while leaving Android artifact publishing to the narrower open Android thread.

## Reproduction Assessment

Yes for source-level reproduction: a node advertising protocol range 3..3 against the current protocol-4 gateway reaches the PROTOCOL_MISMATCH close path before node registration or management RPC. I did not run the reported Windows two-node setup or install the Play Store Android app.

## Solution Assessment

Unclear as an implementation direction: broadly accepting old protocol traffic is not the best fix. The maintainable path is an explicit degraded maintenance contract with tests for what old nodes may and may not do.
