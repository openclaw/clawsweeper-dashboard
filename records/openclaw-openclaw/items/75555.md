## Summary

Keep open. Current main still rejects the documented Feishu `blockStreaming` key, and open PR #75573 already uses closing syntax for this issue, so the issue should remain open until that PR or a complete replacement lands.

## What This Changes

The issue requests that Feishu accept `channels.feishu.blockStreaming` in config validation so the documented block-level streaming toggle can be enabled.

## Best Possible Solution

Land or replace PR #75573 with a Feishu-owned source-schema change, regenerated channel metadata, focused validation tests, a changelog entry, and an explicit runtime decision for the dispatcher block-streaming guard.

## Reproduction Assessment

Yes. Static inspection gives a high-confidence reproduction path: the strict Feishu source schema and generated metadata omit `blockStreaming`, while config validation uses that metadata for `channels.feishu`.

## Solution Assessment

No for the issue’s suggested generated/dist-only edit. The maintainable fix belongs in the Feishu source schema plus regenerated metadata and tests, with runtime behavior handled deliberately if block replies are meant to work end to end.
