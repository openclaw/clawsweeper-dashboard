## Summary

Keep open. Current main still matches the reported Feishu config mismatch: the documented `channels.feishu.blockStreaming` key is absent from Feishu's strict source schema and generated channel metadata, validation still routes through that strict metadata, and #75573 is an open closing PR rather than a merged fix.

## What This Changes

The issue requests that Feishu channel config accept `channels.feishu.blockStreaming` so documented block-level streaming can be enabled.

## Best Possible Solution

Land a Feishu-owned fix that adds `blockStreaming` to the source schema for both top-level and per-account config, regenerates bundled channel metadata, adds focused config validation tests, and deliberately resolves how Feishu should map the setting to `replyOptions.disableBlockStreaming`.

## Reproduction Assessment

Yes. Static inspection gives a high-confidence reproduction path: `openclaw config set channels.feishu.blockStreaming true` is validated through strict Feishu metadata that lacks the key, so AJV rejects it as an additional property.

## Solution Assessment

No for the issue's suggested direct `dist` metadata edit. The maintainable solution belongs in Feishu source schema, generated metadata, tests, and a runtime decision about the current hard-disable guard.
