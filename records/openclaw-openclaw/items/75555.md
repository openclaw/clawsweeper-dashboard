## Summary

Keep open. Current main still matches the reported Feishu config/runtime mismatch: the documented `channels.feishu.blockStreaming` key is absent from the strict Feishu schema and generated metadata, and Feishu still hard-disables core block streaming at dispatch time. The linked #75573 PR is an open implementation candidate, so this issue should remain open until that PR or a complete replacement lands.

## What This Changes

The issue asks for Feishu to accept `channels.feishu.blockStreaming` in config schema and use it to enable block-level streaming.

## Best Possible Solution

Land a complete Feishu-owned fix that adds `blockStreaming` to the source schema for top-level and per-account config, regenerates bundled channel metadata, updates focused config/runtime tests, and makes an explicit boolean setting control `replyOptions.disableBlockStreaming` without regressing the silent-drop fallback guard.

## Reproduction Assessment

Yes. Static current-main inspection gives a high-confidence reproduction path: `channels.feishu.blockStreaming` is absent from the strict generated Feishu schema, so config validation rejects it before runtime; even if accepted, the dispatcher currently forces `disableBlockStreaming: true`.

## Solution Assessment

No for the issue’s suggested direct `dist/extensions/feishu/openclaw.plugin.json` edit; the maintainable fix belongs in Feishu source schema, generated metadata, runtime dispatch behavior, and tests. #75573 is the active candidate, but it should be reviewed for the runtime hard-disable path too.
