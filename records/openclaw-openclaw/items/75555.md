## Summary

Keep open. Current main still appears to reject the documented Feishu `blockStreaming` config because the strict Feishu source and generated schemas omit the key, and open PR #75573 is already a closing implementation candidate.

## What This Changes

Allow the documented `channels.feishu.blockStreaming` boolean to be accepted for Feishu channel config so block replies can be enabled.

## Best Possible Solution

Land or replace PR #75573 with a Feishu-owned source-schema change, regenerated metadata, focused validation tests, and an explicit decision on the Feishu runtime block-streaming guard.

## Reproduction Assessment

Yes. Static inspection gives a high-confidence reproduction path: docs advertise `channels.feishu.blockStreaming`, strict Feishu schema metadata omits it, and config validation reports invalid channel config from that metadata.

## Solution Assessment

No for the issue’s suggested generated/dist-only edit. The maintainable fix belongs in the Feishu source schema, generated metadata, focused tests, and possibly the dispatcher behavior if block replies are meant to work end to end.
