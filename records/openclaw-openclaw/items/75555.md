## Summary

Keep open. Current main still lacks `blockStreaming` in the strict Feishu source and generated channel schemas while the docs and shared streaming resolver recognize the key, and #75573 is already the open closing PR for this gap.

## What This Changes

The issue requests making documented `channels.feishu.blockStreaming` accepted by Feishu channel config validation so block-level streaming can be enabled.

## Best Possible Solution

Review and land or replace #75573 with a Feishu source-schema change, regenerated metadata, focused validation tests, and an explicit owner decision on the runtime block-streaming guard.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: Feishu docs advertise `channels.feishu.blockStreaming`, but the strict source and generated schemas still omit that key and validation applies those schemas under `channels.feishu`.

## Solution Assessment

No for the issue's generated/dist-only suggested edit. The maintainable fix belongs in `extensions/feishu/src/config-schema.ts` with regenerated metadata and tests, plus a clear decision on Feishu's current runtime block-streaming guard.
