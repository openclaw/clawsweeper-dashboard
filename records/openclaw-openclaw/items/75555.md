## Summary

Keep open. Current main still matches the reported Feishu config mismatch: the documented `channels.feishu.blockStreaming` key is absent from the strict Feishu schema and generated channel metadata, and the linked #75573 PR is still the open closing implementation candidate.

## What This Changes

The issue requests that the Feishu plugin accept and honor `channels.feishu.blockStreaming` so documented block-level streaming can be enabled.

## Best Possible Solution

Review and land #75573 or a replacement Feishu-owned fix that adds `blockStreaming` to the source schema for top-level and per-account config, regenerates bundled channel metadata, adds focused schema/config tests, and deliberately reconciles the `disableBlockStreaming` guard with the silent-drop protection it was added for.

## Reproduction Assessment

Yes. Static current-main inspection gives a high-confidence reproduction path: the strict Feishu schema and generated metadata omit `blockStreaming`, while validation routes `channels.feishu` through that metadata before applying config.

## Solution Assessment

No for the issue’s suggested direct generated/dist metadata edit. The maintainable fix should start in `extensions/feishu/src/config-schema.ts`, regenerate metadata, cover validation with tests, and confirm the runtime dispatch behavior matches the documented feature.
