## Summary

Keep open. Current main still matches the reported Feishu config mismatch: the documented `channels.feishu.blockStreaming` key is absent from the strict Feishu source schema and generated channel metadata, and Feishu still hard-disables core block streaming at dispatch time. The open closing PR #75573 is the implementation candidate to review rather than closing this issue before merge.

## What This Changes

The issue requests that the bundled Feishu channel accept `channels.feishu.blockStreaming` as a boolean config key so the documented block streaming toggle can be enabled.

## Best Possible Solution

Land or replace #75573 with a Feishu-owned fix that adds `blockStreaming` to the source schema for top-level and per-account config, regenerates bundled metadata, adds focused validation tests, and deliberately reconciles `replyOptions.disableBlockStreaming` with the silent-drop protection that motivated the current guard.

## Reproduction Assessment

Yes. Static current-main inspection gives a high-confidence reproduction path: strict Feishu schema and generated metadata reject `channels.feishu.blockStreaming`, and validation routes channel config through that metadata before runtime.

## Solution Assessment

No for editing generated `dist` or metadata directly. The maintainable solution belongs in Feishu source schema, generated metadata, focused tests, and a reviewed runtime streaming decision; #75573 is the active candidate but should be checked against the dispatcher guard.
