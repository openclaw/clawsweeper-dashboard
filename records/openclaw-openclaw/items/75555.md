## Summary

Keep open. Current main still matches the report: Feishu documentation advertises `channels.feishu.blockStreaming`, the generated/current channel schema does not accept that key, and the Feishu reply dispatcher still forces core block streaming off.

## What This Changes

Request to make `channels.feishu.blockStreaming` a valid and effective Feishu channel setting so explicit block replies can be enabled.

## Best Possible Solution

Add source-level Feishu support for an explicit block-streaming opt-in, align generated schema/metadata and docs with the chosen default, and update the dispatcher so the setting controls `disableBlockStreaming` without regressing streaming-card fallback delivery.

## Reproduction Assessment

Yes. Static current-main inspection gives a high-confidence reproduction path: adding `channels.feishu.blockStreaming` enters the generated Feishu channel schema, which has `additionalProperties: false` and no such property, so config validation rejects it before runtime.

## Solution Assessment

No for the issue’s suggested `dist/extensions/feishu/openclaw.plugin.json` edit. The maintainable fix belongs in the Feishu source schema, generated config metadata, docs, and reply-dispatcher behavior, with tests covering default disablement and explicit opt-in.
