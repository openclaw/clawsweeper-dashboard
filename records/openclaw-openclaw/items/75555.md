## Summary

Keep open. Current main still matches the reported Feishu config mismatch: the documented `channels.feishu.blockStreaming` key is absent from the strict Feishu source schema and generated channel metadata, and Feishu still forces core block streaming off at dispatch time. The linked #75573 PR is an open closing implementation candidate, so this issue should remain open until that PR or a complete replacement lands.

## What This Changes

Request to allow the documented `channels.feishu.blockStreaming` boolean config so Feishu block-level streaming can be enabled instead of being rejected by channel config validation.

## Best Possible Solution

Land a Feishu-owned fix that adds `blockStreaming` to the source schema for top-level and per-account config, regenerates bundled channel metadata, adds focused schema/config validation tests, and deliberately handles `replyOptions.disableBlockStreaming` without regressing the silent-drop protection that motivated the current guard.

## Reproduction Assessment

Yes. Static current-main inspection gives a high-confidence reproduction path: strict Feishu schema and generated metadata reject `channels.feishu.blockStreaming`, and validation routes channel config through that metadata before runtime.

## Solution Assessment

No for the issue’s suggested direct generated/dist metadata edit; the maintainable solution belongs in Feishu source schema, generated metadata, focused tests, and the runtime dispatcher decision. #75573 is the active candidate, but it should be reviewed against the runtime hard-disable path too.
