## Summary

Keep open. Current main still matches the report: the Feishu source schema and generated strict channel metadata omit the documented `channels.feishu.blockStreaming` key, config validation enforces that schema, and the Feishu dispatcher still sets `disableBlockStreaming: true`. The linked #75573 PR is an open implementation candidate with closing syntax, so this issue should remain open until that PR or a complete replacement lands.

## What This Changes

Allow the documented `channels.feishu.blockStreaming` boolean in Feishu channel config so Feishu block-level streaming can be enabled.

## Best Possible Solution

Land a Feishu-owned fix that adds `blockStreaming` to the source schema for top-level and per-account config, regenerates bundled metadata, adds focused schema/config tests, and makes an explicit boolean setting control Feishu block streaming without regressing the silent-drop guard.

## Reproduction Assessment

Yes. Static current-main inspection gives a high-confidence reproduction path: `channels.feishu.blockStreaming` is absent from the strict Feishu schema, so config validation rejects it before runtime; the dispatcher also currently forces core block streaming off.

## Solution Assessment

No for the suggested direct generated/dist metadata edit. The maintainable solution belongs in the Feishu source schema, generated metadata, focused tests, and runtime dispatch behavior; #75573 is the active candidate but should be reviewed for the hard-disable path too.
