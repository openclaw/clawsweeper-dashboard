## Summary

Keep open: the linked Linux node exec bug is real and not solved on current main, and the PR has sufficient terminal proof for the fallback idea, but the patch is not merge-ready because it can execute a fallback shell after approval and audit metadata have already been bound to the original argv.

## What This Changes

The PR adds a Linux node-host fallback from `/bin/sh -lc` to `/usr/bin/sh -lc` after an ENOENT spawn failure and adds focused regression tests.

## Best Possible Solution

Keep the fallback, but bind or revalidate the selected shell argv before approval, audit, allowlist-use, and `exec.finished` metadata are finalized.

## Reproduction Assessment

Yes. The linked issue includes live reporter evidence from Linux headless node hosts, and current main still builds Linux node exec as `/bin/sh -lc` and directly spawns that argv.

## Solution Assessment

No. The fallback idea is useful, but this implementation selects the fallback inside `runCommand` after approval/audit metadata is finalized; the safer fix binds or revalidates the selected argv before those records are emitted.
