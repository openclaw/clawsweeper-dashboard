## Summary

Keep open: the revised PR fixes the earlier approval/audit placement problem, but it now only selects `/usr/bin/sh` when `/bin/sh` is absent, while the linked Linux node reports show `spawn /bin/sh ENOENT` even though `/bin/sh` exists.

## What This Changes

The PR canonicalizes Linux node-host shell argv from `/bin/sh` to `/usr/bin/sh` before `system.run` approval/execution planning when `/bin/sh` is absent and adds focused regression coverage.

## Best Possible Solution

Keep the pre-approval binding shape, but make shell selection cover the reported ENOENT-with-`/bin/sh`-present case and prove commandText, approval, allowlist, and `exec.finished` parity.

## Reproduction Assessment

Yes for source-level triage: the linked issue has live Linux node evidence for `spawn /bin/sh ENOENT` with `/bin/sh` and `/usr/bin/sh` present, and current main/latest release still plan `/bin/sh -lc`. I did not run the exact paired-node setup locally.

## Solution Assessment

No: moving fallback selection before approval is the right layer, but the current predicate is too narrow for the reported failure. The fix should cover ENOENT when `/bin/sh` appears to exist while preserving approval/audit parity.
