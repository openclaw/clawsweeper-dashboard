## Summary

Keep this PR open: the Linux node-shell fallback is a real, well-scoped fix with useful runtime proof, but the current patch selects `/usr/bin/sh` after approval binding, durable trust, allowlist audit, and completion metadata have already been tied to `/bin/sh`. The branch also needs a current-main refresh for the required Dependency Guard check noted by a member comment.

## What This Changes

The PR adds a `/usr/bin/sh` retry when node-host `system.run` spawning `/bin/sh` fails with `ENOENT`, plus focused regression coverage.

## Best Possible Solution

Represent the actual fallback shell in the approved execution plan, allowlist/durable audit, and completion metadata before execution, then refresh the branch and rerun the focused node-host gates plus Dependency Guard.

## Reproduction Assessment

Yes. Current main and v2026.5.27 build Linux node exec commands with `/bin/sh` only, and the PR supplies chroot terminal proof for the missing-`/bin/sh` failure condition, although I did not rerun it in this read-only review.

## Solution Assessment

No. Retrying a trusted absolute shell is a plausible fix, but doing the swap inside `runCommand` is not the best shape because security approval and audit records have already been bound to the original argv.
