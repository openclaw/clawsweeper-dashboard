## What This Changes

The branch adds a node-host shell resolution helper that rewrites `/bin/sh` system.run plans to `/usr/bin/sh` only when `/bin/sh` is absent, updates approval/execution planning, exports `runCommand` for tests, and adds regression coverage.

## Best Possible Solution

Keep shell selection before approval/policy, but make it handle or clearly diagnose `ENOENT` even when `/bin/sh` exists, add focused coverage for that case, and refresh current-head real behavior proof before merge.

## Reproduction Assessment

Yes for source-level triage, but not as a fresh local end-to-end run. The linked issue has live Linux node evidence with `/bin/sh` present, and current main plus the latest release still route non-Windows node exec to `/bin/sh` and return spawn error messages.

## Solution Assessment

No. Moving shell selection before policy is the right layer, but the current predicate is too narrow for the reported hosts and the current-head real behavior proof has not been refreshed.
