## Summary

Keep open: this is still the canonical Control UI feature PR, but the latest head is not mergeable because it now has a parse-breaking `grouped-render.ts` edit, still bypasses main's expansion-state seam, has i18n output drift, and lacks real behavior proof.

## What This Changes

The PR adds a persisted `chatExpandToolCalls` Control UI setting, desktop/mobile toolbar toggles, rendering plumbing, English copy, and test fixture updates to expand tool-call output by default.

## Best Possible Solution

Wire the persisted preference into main's `autoExpandToolCalls` expansion-state path, preserve manual disclosure state, fix toolbar disabled copy/onboarding behavior, regenerate i18n outputs, and add redacted real webchat proof.

## Reproduction Assessment

Yes. Source inspection of the PR head gives a high-confidence path: `ui/src/ui/chat/grouped-render.ts:746` is malformed, and the direct `<details open>` binding would force manual disclosure state on rerender once parsing is fixed.

## Solution Assessment

No. The feature direction is reasonable, but the maintainable fix is to feed the new setting through the existing expansion-state map rather than binding `open` directly, then repair the current parse/i18n/control proof blockers.
