## Summary

Keep open: the feature request is valid and current main still lacks a user-facing Control UI preference, but this PR needs refresh before merge because it binds disclosure state directly, has toolbar disabled-copy issues, and misses required user-facing release/i18n follow-through.

## What This Changes

The PR adds a browser-local `chatExpandToolCalls` setting, desktop/mobile Control UI toolbar buttons, render plumbing, English copy, and test fixture updates so tool-call output can expand by default.

## Best Possible Solution

Refresh the feature on current main by adding a persisted browser preference that feeds the existing `autoExpandToolCalls` / `syncToolCardExpansionState` path, with accurate toolbar disabled copy, generated i18n updates, a changelog entry, and focused UI tests.

## Reproduction Assessment

Yes. A source-level reproduction is clear: on the PR branch, render a tool disclosure with the preference false, manually open it, then trigger a streaming/history rerender; Lit reapplies `?open=false` and collapses it again.

## Solution Assessment

No. The feature direction is right, but the implementation should seed the current expansion-state map instead of binding `open` directly, and it should align toolbar disabled states plus release/i18n policy before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve manual tool expansion state:** `ui/src/ui/chat/grouped-render.ts:565`
  - body: Binding `open` directly to the global preference makes every rerender force the DOM disclosure state. With the default off, a manually opened tool section can collapse again during streaming/history rerenders; with it on, manual collapse can be undone. Seed the expansion state instead of binding the `<details>` attribute directly.
  - confidence: 0.89

  - body: The expand button stays enabled during onboarding because `showToolCalls` is forced true there, and when tool calls are manually hidden the desktop title says setup is disabled. Align the disabled condition with the neighboring controls and use separate hidden-tools copy.
  - confidence: 0.8
