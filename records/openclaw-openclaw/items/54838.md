## Summary

Keep open: current main still lacks the persisted Control UI preference, and the PR needs rework because it bypasses main's expansion-state seam and lacks after-fix real behavior proof.

## What This Changes

The PR adds a persisted Control UI `chatExpandToolCalls` browser setting, desktop/mobile toolbar controls, render plumbing, English copy, and fixture updates to expand tool-call output by default.

## Best Possible Solution

Add the persisted preference and toolbar control on top of main's `autoExpandToolCalls` state seeding, regenerate i18n outputs, and include real webchat proof without overriding manual disclosure toggles.

## Reproduction Assessment

Yes. Source inspection of the PR diff gives a high-confidence path: manually open or collapse a tool disclosure, then trigger a streaming/history rerender and the direct `open` binding reapplies the global preference.

## Solution Assessment

No. The feature direction is reasonable, but the maintainable fix is to wire the new setting into current main's expansion-state map instead of binding `<details open>` directly.

## Review Findings

Full review comments:

- **[P2] Preserve manual tool expansion state:** `ui/src/ui/chat/grouped-render.ts:565`
  - body: Binding `open` directly to the new preference makes rerenders force each disclosure state. With the default off, a manually opened tool section can collapse again during streaming or history refresh; with it on, a manual collapse can be undone. Feed the preference into the existing expansion-state map instead.
  - confidence: 0.9

  - body: The expand toggle still reuses the setup-disabled title when tool calls are manually hidden, and the mobile counterpart is not locked during onboarding like the neighboring chat controls. Use hidden-tool-call copy for that case and share the onboarding-disabled condition across desktop and mobile.
  - confidence: 0.83

  - body: This adds an English UI string without the generated locale bundles and `.i18n` metadata required by the Control UI i18n pipeline, so the i18n check can drift after merge.
  - confidence: 0.8
