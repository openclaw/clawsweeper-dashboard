The issue requests making `sessions_send` and `message` tool validation tolerant of LLM-supplied conflicting optional parameters.

## Best Possible Solution

Land or replace both focused fixes so `sessions_send` accepts authoritative `sessionKey` precedence and `message(action: "send")` no longer fails solely because stray poll creation fields were supplied.
