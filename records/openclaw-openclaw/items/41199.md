## Summary

Keep open. Current main and the latest release still contain both reported rejection paths, and the linked fixes remain open, conflicting, partial, or waiting on maintainer consolidation/proof.

## What This Changes

The issue asks OpenClaw to stop rejecting tool calls when models fill conflicting optional params for `sessions_send` and `message` send/poll actions.

## Best Possible Solution

Land or replace the linked fixes with one consolidated contract: explicit session identifiers take safe precedence, and message sends tolerate model-filled no-op poll params while preserving real poll validation.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: call `sessions_send` with both `sessionKey` and `label`, or call `message` with `action: "send"` plus non-empty poll creation params, and current main still reaches the reported rejection branches.

## Solution Assessment

Unclear as a complete fix today. Preferring explicit session identifiers is likely the narrow sessions fix, but the message send/poll path still needs maintainers to choose and land one safe normalization contract.
