## Summary

Keep open. Current main still has the two deterministic rejection paths described here, and the focused implementation PRs for the `sessions_send` selector conflict and the `message(action: "send")` poll-param conflict are still open and unmerged.

## What This Changes

The issue asks OpenClaw to avoid tool-call failures when LLMs supply redundant `sessions_send` selector parameters or stray poll parameters on `message(action: "send")`.

## Best Possible Solution

Review and land or replace both focused fixes: `sessions_send` should handle redundant selector fields safely, and `message(action: "send")` should not fail solely because stray poll creation fields were supplied.

## Reproduction Assessment

Yes. Source inspection gives deterministic current-main paths: call `sessions_send` with non-empty `sessionKey` and `label`, or call `message` with `action: "send"` plus explicit poll creation params such as `pollQuestion` and `pollOption`.

## Solution Assessment

Yes for keeping this issue open. The narrow maintainable path is to review and land or replace https://github.com/openclaw/openclaw/pull/46904 and https://github.com/openclaw/openclaw/pull/74009 rather than close this issue or queue duplicate repair work.
