## Summary

Keep open. Current main and `v2026.5.20` still contain both reported rejection paths, and the related open PRs are partial, conflicting, proof-gated, or waiting on maintainer consolidation.

## What This Changes

Request to make `sessions_send` and `message(action="send")` tolerate model-filled conflicting optional selector or poll parameters so agent-to-agent and channel sends do not fail.

## Best Possible Solution

Land or replace the linked fixes with one consolidated contract: explicit session identifiers take precedence without unauthorized key disclosure, and send actions tolerate model-filled no-op poll defaults while preserving real poll validation.

## Reproduction Assessment

Yes. Source inspection on current main shows `sessions_send` returns the mixed-selector error before resolution, and `runMessageAction` throws the poll-action error for `action: "send"` plus detected poll params; I did not run a live multi-agent repro in this read-only review.

## Solution Assessment

Unclear as a complete fix today. Preferring explicit session identifiers is the narrow sessions fix, but poll/send normalization must be reconciled across open PRs before landing.
