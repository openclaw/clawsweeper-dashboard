## What This Changes

Prevent agent tools from exposing conflicting optional parameters that cause `sessions_send` or `message` calls to fail.

## Best Possible Solution

Keep the merged explicit session-selector precedence and consolidate all remaining multi-action message schema pollution work, reproductions, and product decisions in https://github.com/openclaw/openclaw/issues/43015.

## Reproduction Assessment

Yes. Current main source and focused tests show the message runner still rejects `action: "send"` when meaningful poll creation fields are present, while the sessions selector conflict is now fixed.

## Solution Assessment

No, this broad umbrella is no longer the best tracking unit: the sessions fix has landed, and https://github.com/openclaw/openclaw/issues/43015 is the narrower, better-evidenced canonical place to decide action-specific schema behavior without silently discarding genuine poll intent.
