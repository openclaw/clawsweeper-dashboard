## Summary

Keep open. Current main still contains both reported rejection paths, and the linked fixes are open, conflicting, proof-gated, or partial, so this is not safely implemented or superseded yet.

## What This Changes

The issue asks OpenClaw to prevent model-filled optional parameters from causing `sessions_send` and `message(action="send")` tool calls to fail.

## Best Possible Solution

Land or consolidate the open fixes into one contract: explicit session identifiers win safely, and message sends tolerate default-filled no-op poll fields while preserving real poll validation.

## Reproduction Assessment

Yes, at source level. Current main still has deterministic guards that reject `sessions_send` with both `sessionKey` and `label`, and reject `message` sends when detected poll params are present.

## Solution Assessment

No current complete fix is on main. The best solution is a consolidated maintainer-approved repair across the open PRs, not a new parallel fix or a premature close.
