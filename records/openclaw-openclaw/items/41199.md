## Summary

Keep this issue open. Current main and the latest release still contain both reported source-level failure paths, while the linked fixes remain open, conflicting, partial, or awaiting maintainer choice and proof rather than shipped resolution.

## What This Changes

The issue asks OpenClaw to stop agent-to-agent tools from failing when LLMs add conflicting optional parameters to `sessions_send` and `message` send calls.

## Best Possible Solution

Land or replace the linked fixes with one consolidated contract: explicit session identifiers take safe precedence without denied-key disclosure, and message send normalization handles model-filled poll params while preserving explicit poll validation and focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: call `sessions_send` with both `sessionKey` and `label`, or call `message` with `action: "send"` plus non-empty poll creation params, and current main still takes the reported rejection branches.

## Solution Assessment

Unclear as a complete fix today. Preferring explicit session identifiers is likely the narrow sessions fix, but the message send/poll path still needs maintainers to choose and land one safe normalization contract across the open PRs.
