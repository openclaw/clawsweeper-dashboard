## Summary

Keep open: the PR has useful security intent, but it is not mergeable as-is because it targets stale Docker and CI paths, introduces non-portable secure Compose defaults, lacks contributor real-behavior proof, and needs Docker exposure policy review.

## What This Changes

The branch masks token output in the old root Docker setup script, changes Compose port publishing to loopback, adds a secure Compose file, ignores env files, and pins selected CI actions to SHAs.

## Best Possible Solution

Rework the hardening against current main: mask token output in scripts/docker/setup.sh without breaking the health hint, settle Docker port exposure policy with docs, drop or make secure Compose portable, and pin only current workflow actions to valid commit SHAs if that policy is accepted.
