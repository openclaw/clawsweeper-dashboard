## What This Changes

The PR masks Docker setup token output, changes Compose port publishing to loopback, adds a secure Compose file, ignores additional env files, and pins selected GitHub Actions refs to full SHAs.

## Best Possible Solution

Land a focused rebased hardening change: mask token output in scripts/docker/setup.sh while keeping the health-check command self-contained, make an explicit Docker port-exposure decision with matching docs, pin the current workflow action refs if that is the chosen CI policy, and drop or repair docker-compose.secure.yml before merge.

Yes. Read-only inspection reproduces the concerns: current main still prints the token and publishes Compose ports without loopback binding, while the PR patch shows the hardcoded secure Compose paths/IP and literal post-script token command.

## Solution Assessment

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Remove user-specific compose mount defaults:** `docker-compose.secure.yml:16-17`
  - body: The added secure Compose file defaults both persistent mounts to /home/cmdshadow paths. On any other host, running this file without explicit env vars can mount or create an unintended path, so the tracked secure config is not portable; use generic defaults matching docker-compose.yml or require explicit variables.
  - confidence: 0.92
- **[P2] Make the tailnet bind address configurable:** `docker-compose.secure.yml:24`
  - body: This port mapping binds 10.8.0.1 unconditionally. Docker Compose fails on hosts without that address, and on hosts that do have it the gateway is exposed beyond localhost despite the file being introduced as secure; make this an explicit env-driven opt-in or remove it.
  - confidence: 0.9
- **[P2] Keep the health-check command usable:** `docker-setup.sh:288`
  - body: The script now prints $OPENCLAW_GATEWAY_TOKEN literally, but users normally execute the setup script rather than source it, so that variable is usually unavailable after the script exits. Copying the hint expands to an empty token and fails auth; print a command that safely loads the generated token instead.
  - confidence: 0.88
