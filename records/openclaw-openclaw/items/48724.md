- **[P1] Move token masking to the active Docker setup script:** `docker-setup.sh:259`
  - body: Current main's Docker onboarding uses scripts/docker/setup.sh, which still prints OPENCLAW_GATEWAY_TOKEN. This hunk masks the removed root setup path, so rebasing it as-is leaves the active token output unfixed.
  - confidence: 0.92

  - body: The new secure Compose file defaults config and workspace mounts to /home/cmdshadow paths. On other machines this can create or mount unintended secret/state locations, so the tracked secure config is not portable.
  - confidence: 0.92
- **[P2] Make tailnet exposure opt-in and configurable:** `docker-compose.secure.yml:24`
  - body: This port mapping binds 10.8.0.1 unconditionally. Compose fails on hosts without that address, and on hosts that do have it the gateway is exposed beyond loopback despite this file being introduced as a secure config.
  - confidence: 0.9
- **[P2] Pin the Gradle action to the peeled commit SHA:** `.github/workflows/ci.yml:691`
  - body: The Gradle pin uses the annotated v4 tag object SHA, not the peeled commit SHA. That does not satisfy full commit-SHA pinning and should be replaced with the target commit SHA for auditability.
  - confidence: 0.88
- **[P2] Keep the health-check hint runnable after setup:** `docker-setup.sh:288`
  - body: The script now prints $OPENCLAW_GATEWAY_TOKEN literally, but docker-setup.sh is normally executed rather than sourced. After it exits, the caller's shell usually lacks that variable, so the copied health command expands to an empty token and fails auth.
  - confidence: 0.86
