## Summary

Keep open: the security hardening is still relevant, but this PR is stale against current main and has blocking Docker Compose portability/security issues before it can be merged.

## What This Changes

The PR masks token output in the old root Docker setup script, changes Compose port publishing to loopback, adds a secure Compose file, broadens env-file ignores, and pins selected CI Actions to SHA refs.

## Best Possible Solution

Replace this with a rebased, maintainer-approved hardening patch against the active Docker setup script, current compose file, and current CI workflow, and either drop or make portable the secure Compose surface.

## Reproduction Assessment

Yes. Source inspection of current main shows the active Docker setup script still prints the gateway token and docker-compose.yml still publishes unqualified host ports; the PR diff also shows the hardcoded secure Compose paths/IP and literal token hint.

## Solution Assessment

No. This PR is not the best solution as-is because it edits stale Docker setup and CI surfaces and introduces a non-portable secure Compose file; a narrower rebased patch is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Move masking to the active Docker setup script:** `docker-setup.sh:259`
  - body: Current main no longer has the root docker-setup.sh body this PR edits; Docker onboarding now runs scripts/docker/setup.sh, which still prints the token. Rebasing this hunk without moving the change leaves the reported leak in the active path.
  - confidence: 0.9
- **[P1] Remove user-specific compose mount defaults:** `docker-compose.secure.yml:16-17`
  - body: The added secure Compose file defaults both persistent host mounts to /home/cmdshadow paths. On other machines, running it without explicit env vars can mount or create unintended host paths, and the openclaw-cli service repeats the same defaults.
  - confidence: 0.92
- **[P2] Make the tailnet bind address configurable:** `docker-compose.secure.yml:24`
  - body: This port mapping binds 10.8.0.1 unconditionally. Docker Compose fails on hosts without that address, and on hosts that do have it the gateway is exposed beyond loopback despite this being introduced as a secure config.
  - confidence: 0.9
- **[P2] Keep the health-check command runnable:** `docker-setup.sh:288`
  - body: The script now prints $OPENCLAW_GATEWAY_TOKEN literally, but users normally execute the setup script rather than source it. Copying the hint after the script exits expands to an empty token and fails auth.
  - confidence: 0.88
