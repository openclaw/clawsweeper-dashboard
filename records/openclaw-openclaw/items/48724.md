## Summary

Keep open. The hardening direction is useful, but current main has moved the active setup script and CI refs, the requested behavior is not implemented on main, and the PR still introduces concrete secure-compose portability and exposure bugs.

## What This Changes

The PR masks Docker setup token output, changes Compose gateway/bridge port publishing to loopback, adds a secure Compose file, broadens env-file ignores, and pins selected CI Actions to SHA refs.

## Best Possible Solution

Land a focused rebased hardening change that masks active scripts/docker/setup.sh output without breaking health hints, makes Docker exposure policy and docs explicit, updates current CI refs only if SHA pinning is accepted, and drops or repairs docker-compose.secure.yml.

## Reproduction Assessment

Yes. Source inspection of current main shows the active Docker setup script still emits the token and Compose ports remain unqualified, while the PR diff shows the hardcoded secure Compose paths/IP and literal post-script token command.

## Solution Assessment

No. This PR is not the best fix as-is because it targets stale setup-script and CI surfaces and adds a non-portable secure Compose file; a rebased focused patch is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Remove user-specific compose mount defaults:** `docker-compose.secure.yml:16-17`
  - body: The added secure Compose file defaults both persistent mounts to /home/cmdshadow paths. On any other host, running it without explicit env vars can mount or create unintended host paths, so the tracked secure config is not portable.
  - confidence: 0.92
- **[P2] Make the tailnet bind address configurable:** `docker-compose.secure.yml:24`
  - body: This port mapping binds 10.8.0.1 unconditionally. Docker Compose fails on hosts without that address, and on hosts that do have it the gateway is exposed beyond localhost despite the file being introduced as secure.
  - confidence: 0.9
- **[P2] Keep the health-check command usable:** `docker-setup.sh:288`
  - body: The script now prints $OPENCLAW_GATEWAY_TOKEN literally, but users normally execute the setup script rather than source it. Copying the hint after the script exits expands to an empty token and fails auth.
  - confidence: 0.88
