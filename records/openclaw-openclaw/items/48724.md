## What This Changes

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Remove user-specific compose mount defaults:** `docker-compose.secure.yml:16-17`
  - body: The added secure Compose file defaults both persistent mounts to `/home/cmdshadow/...`. On any other host, running this file without explicit env vars mounts or creates an unintended path, so the tracked secure config is not portable; use the same generic defaults as the main Compose file or require explicit variables.
  - confidence: 0.92
- **[P2] Make the tailnet bind address configurable:** `docker-compose.secure.yml:24`
  - body: This port mapping binds `10.8.0.1` unconditionally. Docker Compose fails on hosts without that address, and on hosts that do have it the gateway is exposed beyond localhost despite the file being introduced as secure; gate this behind an env var or documented opt-in.
  - confidence: 0.9
- **[P2] Keep the health-check command copyable:** `docker-setup.sh:288`
  - body: The script now prints `$OPENCLAW_GATEWAY_TOKEN` literally, but users execute the setup script rather than source it, so that variable is usually unavailable after the script exits. Copying the hint expands to an empty token and fails auth; print a command that safely loads the generated token instead.
  - confidence: 0.88
