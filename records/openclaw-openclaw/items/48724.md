## Summary

## What This Changes

The branch masks token output in a Docker setup script, changes Docker Compose host port bindings, adds a secure Compose file, broadens env ignores, and pins selected CI Actions to SHA refs.

## Best Possible Solution

Rebase into a current, maintainer-reviewed hardening patch that masks tokens in scripts/docker/setup.sh, uses portable opt-in Docker exposure defaults, and pins current workflow Actions only if that CI policy is accepted.

## Reproduction Assessment

Yes. Source inspection of current main shows the active Docker setup still prints the gateway token and Compose still publishes unqualified host ports, though I did not run a Docker runtime reproduction in this read-only review.

## Solution Assessment

No. The current PR is not the best solution as-is because it targets a stale setup path, changes Docker exposure policy without maintainer alignment, and adds a hardcoded secure Compose surface.

- **[P1] Move token masking to the active Docker setup script:** `docker-setup.sh:259`
  - body: Current main's Docker onboarding uses scripts/docker/setup.sh, which still prints OPENCLAW_GATEWAY_TOKEN. This hunk masks the retired root setup path, so rebasing it as-is leaves the active token output in place.
  - confidence: 0.9

  - body: The new secure Compose file defaults config and workspace mounts to /home/cmdshadow paths. On other machines this can create or mount unintended secret/state locations, so the tracked secure config is not portable.
  - confidence: 0.92
- **[P2] Make the tailnet bind address configurable:** `docker-compose.secure.yml:24`
  - body: This port mapping binds 10.8.0.1 unconditionally. Compose fails on hosts without that address, and on hosts that do have it the gateway is exposed beyond loopback despite this file being introduced as a secure config.
  - confidence: 0.9
- **[P2] Keep the health-check hint self-contained:** `docker-setup.sh:288`
  - body: The script now prints $OPENCLAW_GATEWAY_TOKEN literally, but setup is normally executed rather than sourced. After it exits, the caller's shell usually lacks that variable, so the copied health command expands to an empty token and fails auth.
  - confidence: 0.88
