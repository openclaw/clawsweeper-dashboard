- **[P1] Mask the active Docker setup token output:** `docker-setup.sh:259`
  - body: Current main uses `scripts/docker/setup.sh` and still echoes the gateway token during onboarding and final summary. This PR masks the removed root `docker-setup.sh`, so rebasing it as-is leaves the active token leak unfixed.

  - confidence: 0.92

  - body: This port mapping binds `10.8.0.1` unconditionally. Compose fails on hosts without that address, and on hosts that do have it the gateway is exposed beyond loopback despite this file being introduced as a secure config.
  - confidence: 0.9

  - body: The Gradle pin uses the annotated `v4` tag object SHA, not the peeled commit SHA. That does not satisfy full commit-SHA pinning and should be replaced with the target commit SHA for auditability.
  - confidence: 0.88
- **[P2] Keep the health-check hint runnable after setup:** `docker-setup.sh:288`
  - body: The script now prints `$OPENCLAW_GATEWAY_TOKEN` literally, but this setup script is normally executed rather than sourced. After it exits, the caller's shell usually lacks that variable, so the copied health command expands to an empty token and fails auth.
  - confidence: 0.86
