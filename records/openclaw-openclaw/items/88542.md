## Summary

Keep this PR open: the deterministic prompt direction is right and the live proof covers the common Discord channel/DM paths, but the branch still loses the original per-run `sourceReplyDeliveryMode` before detached media completion, so a forced message-tool-only channel run can still receive a MEDIA-line completion instruction.

## What This Changes

The PR changes generated-media completion prompts to emit deterministic message-tool versus MEDIA-line instructions and adds focused prompt assertions for image/video/music completion paths.

## Best Possible Solution

Keep the deterministic prompt shape, but preserve the original `sourceReplyDeliveryMode` through OpenClaw tool construction, each media generator, the scheduler, and both success/error completion wakes before merge.

## Reproduction Assessment

Yes from source inspection: start a channel media generation run whose original source reply mode is `message_tool_only`, then the detached scheduler reaches `wakeTaskCompletion` without that mode and line 526 recomputes from chat type/config. I did not run tests because this review is read-only.

## Solution Assessment

No. The deterministic instruction is the right fix direction, but the maintainable solution must carry the already-resolved delivery mode instead of reconstructing a narrower approximation later.
