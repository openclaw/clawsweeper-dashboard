## Summary

Keep open: the prompt split is the right direction, but the patch still does not preserve the original `sourceReplyDeliveryMode` through the actual image/video/music tool and scheduler path, so some message-tool-only completions can still get a MEDIA-line prompt.

## What This Changes

The PR changes generated-media completion prompts to choose deterministic message-tool versus MEDIA-line instructions and adds focused prompt tests.

## Best Possible Solution

Thread the resolved source reply delivery mode from the run into all media tool constructors and the shared scheduler, then build the deterministic prompt from that preserved value for image, video, and music completions.

## Reproduction Assessment

Yes, source-reproducible rather than locally executed: current main contains the ambiguous conditional prompt, and the PR body provides live redacted logs showing repeated ambiguous Discord recipient errors before the patch. The remaining review finding is also source-reproducible because the upstream mode reaches `createOpenClawTools` but not the media tool constructors or scheduler.

## Solution Assessment

No, not yet: deterministic prompt text is the right fix shape, but reconstructing the mode at wake time is weaker than preserving the run's already-resolved delivery mode. The safer solution is a narrow propagation patch plus a real tool/scheduler-path regression.
