## Summary

Keep open: the PR is a plausible narrow fix for a source-proven crash path, but it still needs contributor-owned real behavior proof from an affected filesystem class before merge.

## What This Changes

The PR replaces the session cost usage cache lock's temp-file hard-link publish step with direct exclusive `writeFile(..., { flag: "wx" })` in `src/infra/session-cost-usage.ts`.

## Best Possible Solution

Land a narrow portable lock fix in the session-cost cache path with accurate filesystem-contract wording, focused regression coverage, and real proof from an affected filesystem or equivalent shared-mount setup.

## Reproduction Assessment

No live unsupported filesystem was mounted in this read-only review, so I do not have a high-confidence runtime reproduction. The issue is source-reproducible because current main publishes the lock with `fs.promises.link` and rethrows non-`EEXIST` errors, matching the linked ENOTSUP report.

## Solution Assessment

Yes, replacing the hard-link publish dependency with exclusive create or an unsupported-link fallback is the narrowest maintainable direction. This patch should soften the network-filesystem guarantee and add proof before merge.

## Review Findings

Overall correctness: patch is correct

Overall confidence: 0.88

Full review comments:

- **[P3] Soften the O_EXCL filesystem guarantee:** `src/infra/session-cost-usage.ts:235-237`
