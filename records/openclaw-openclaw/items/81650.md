## What This Changes

The PR replaces usage-cost cache lock publication via temp-file hard link with direct exclusive lock-file creation using `writeFile(..., { flag: "wx" })`.

## Best Possible Solution

Land the narrow portability fix after affected-filesystem proof or explicit maintainer proof override, ideally with focused regression coverage while preserving existing `EEXIST` contention and stale-lock cleanup behavior.

## Reproduction Assessment

No for live reproduction in this review; I did not exercise an unsupported-hard-link filesystem. Yes for source reproduction: current main publishes the lock with `fs.promises.link` and rethrows non-`EEXIST` errors from usage-cost refresh lock acquisition.

## Solution Assessment

Yes, the proposed direct exclusive-create lock is a narrow maintainable fix direction because it removes the hard-link publish step while preserving `EEXIST` contention handling. The best version still needs real affected-filesystem proof or maintainer override, and focused regression coverage would make the failure mode durable.
