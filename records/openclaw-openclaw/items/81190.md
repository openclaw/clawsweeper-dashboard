## What This Changes

The PR changes embedded-agent overflow recovery to try existing tool-result truncation before explicit LLM compaction when oversized tool-result pressure is detected, and updates focused run-loop tests.

## Best Possible Solution

Merge the narrow truncation-first overflow recovery after maintainer review, preserving the existing compaction fallback and avoiding new timeout or config surface.

## Reproduction Assessment

Yes. Current main source and tests show explicit overflow compaction runs before fallback truncation, and the linked live log shows a 912840ms compaction timeout followed by immediate truncation of 281 tool results.

## Solution Assessment

Yes. Reusing the existing deterministic truncation helper before compaction only when tool-result pressure is detected, then falling back to compaction on a truncation miss, is the narrow maintainable fix.
