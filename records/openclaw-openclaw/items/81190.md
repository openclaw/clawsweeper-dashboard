Keep open for normal maintainer merge review: current main still compacts before fallback truncation, while this PR narrowly moves existing tool-result truncation ahead of compaction with focused regression coverage, sufficient terminal proof, and no blocking findings.

## What This Changes

The PR updates the embedded agent run loop and overflow-compaction tests so detected tool-result-heavy overflows try session tool-result truncation before explicit LLM compaction, while preserving compaction fallback.

## Best Possible Solution

Merge the narrow truncation-first recovery change after normal maintainer and CI gating, keeping the existing heuristic and compaction fallback rather than adding timeout or config surface.

## Solution Assessment
