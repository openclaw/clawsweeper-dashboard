## Summary

Keep this PR open for normal maintainer review and merge gating: it addresses a source-reproducible overflow recovery ordering bug, supplies adequate after-fix runtime proof, and I found no blocking correctness or security findings.

## What This Changes

The branch tries existing session tool-result truncation before explicit overflow compaction for tool-result-heavy embedded-agent overflows, then updates overflow-compaction tests for truncation-first retry, compaction fallback, and persisted-turn continuation.

## Best Possible Solution

Merge the narrow truncation-first recovery change after normal maintainer and CI gating, preserving the existing compaction fallback and avoiding new timeout or config behavior.

## Reproduction Assessment

Yes. Current main has a clear source and harness-testable path where overflow compaction runs before reducible tool-result truncation, and the linked live log shows a 912840ms compaction timeout followed immediately by truncation of 281 tool results.

## Solution Assessment

Yes. Reusing the existing deterministic truncation helper before compaction only when tool-result pressure is detected, then falling back to compaction on a truncation miss, is the narrow maintainable solution.
