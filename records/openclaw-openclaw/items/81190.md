## Summary

Keep open for normal PR review and merge gating: current main still has the compaction-first overflow recovery ordering, while this PR applies a narrow truncation-first fix with focused tests and sufficient Docker runtime proof. I found no blocking correctness findings in the patch.

## What This Changes

This PR moves embedded-agent tool-result truncation ahead of explicit overflow compaction for tool-result-heavy overflows and updates the overflow loop tests for truncation-first, fallback, and persisted-turn retry behavior.

## Best Possible Solution

Merge the narrow truncation-first overflow recovery after CI and maintainer gating, leaving broader compaction escalation and futile-truncation policy to the related open work.

## Reproduction Assessment

Yes. Current main source and tests show compaction runs before tool-result truncation, and the linked live log shows a 912840ms compaction timeout followed by truncation of 281 tool results; I did not force a live Telegram overflow.

## Solution Assessment

Yes. Reusing the existing deterministic truncation helper before explicit compaction only for tool-result-heavy snapshots, then falling back to compaction on a miss, is the narrow maintainable fix.
