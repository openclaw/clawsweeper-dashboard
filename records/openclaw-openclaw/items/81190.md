## Summary

Keep open: current main and the latest release still run explicit overflow compaction before the later fallback tool-result truncation path, while this PR is the focused open candidate fix. The patch looks correct, but merging it needs maintainer acceptance of the session-state tradeoff because it can truncate persisted tool-result text before a compaction summary is attempted.

## What This Changes

The PR changes embedded-agent overflow recovery so tool-result-heavy context overflows try deterministic tool-result truncation before explicit auto-compaction, with focused regression coverage.

## Best Possible Solution

Merge after maintainers explicitly accept truncation-first recovery as the default for tool-result-heavy overflows, preserving the compaction fallback and persisted-message retry coverage.

## Solution Assessment

Yes, with maintainer acceptance: the PR reuses the existing tool-result pressure heuristic and truncation helper before explicit overflow compaction while preserving compaction fallback. A new config knob or timeout policy would be broader than this focused bug fix.
