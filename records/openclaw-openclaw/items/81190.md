## Summary

Keep open for normal review and merge consideration: current main still has the compaction-before-truncation ordering, while this PR is a narrow fix with focused regression coverage, supplied runtime proof, green relevant checks, and no blocking review findings.

## What This Changes

The PR moves tool-result truncation ahead of explicit overflow compaction for tool-heavy embedded-agent overflows and updates the run-loop regression tests.

## Best Possible Solution

Merge the narrow truncation-first recovery change after normal maintainer gating, preserving the existing compaction fallback and preflight semantics.

## Reproduction Assessment

Yes. Current main source and tests show explicit overflow compaction runs before fallback truncation, and the linked live log shows a 912840ms compaction timeout followed by immediate truncation of 281 tool results; I did not force a live overflow locally.

## Solution Assessment

Yes. Reusing the existing deterministic tool-result truncation path before explicit compaction only when tool-result pressure is detected, then falling back to compaction on a miss, is the narrow maintainable fix.
