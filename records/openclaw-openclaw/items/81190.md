## Summary

Keep open: the fix direction is narrow and useful, but the PR introduces a replay regression in the new truncation-success path and the external real-behavior proof is still mock-only after the patch.

## What This Changes

The PR changes embedded agent overflow recovery so tool-result-heavy context overflows try deterministic tool-result truncation before LLM compaction, with regression tests for ordering and compaction fallback.

## Best Possible Solution

Keep the truncation-first recovery, but share the existing transcript-continuation retry guard for persisted current messages and add focused regression coverage before merge.

## Reproduction Assessment

Yes. The original timeout-before-truncation problem is source-reproducible from current main’s overflow ordering and supported by the linked live log; the PR regression is also source-reproducible from the new immediate `continue` path.

## Solution Assessment

No. Trying truncation before compaction is the right narrow direction, but this implementation needs to preserve the existing persisted-message replay guard and provide real after-fix behavior proof.
