## Summary

Keep open: current main and the latest release still default Codex direct/source chats to message-tool delivery, preserve final Codex assistant text, and suppress automatic delivery when the message tool was not used. The report is source-reproducible, but the fix changes documented message-tool-only delivery semantics and overlaps an open direct-chat default PR, so it needs maintainer product review rather than an automated repair.

## What This Changes

The issue requests a fallback that delivers Codex `last_agent_message` to the originating channel when a message-tool-only agent finishes without calling the message tool.

## Best Possible Solution

Maintainers should choose the Codex direct-chat visibility contract: either land the automatic-default approach from https://github.com/openclaw/openclaw/pull/83814 or approve a narrowly scoped fallback that excludes silent tokens, duplicate message-tool sends, room events, and privacy-sensitive tool-only configurations.

## Solution Assessment

Unclear: the requested fallback would heal the observed silence, but it changes documented message-tool-only privacy semantics. The maintainable path is a maintainer decision between the open automatic-default PR and a deliberately scoped fallback policy.
