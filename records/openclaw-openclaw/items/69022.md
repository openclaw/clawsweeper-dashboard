Keep this PR open. Current main still has the Telegram block-stream dispatcher path without the configured `humanDelay` wiring, while the shared dispatcher, resolver, and peer channels already support that contract; the PR is the scoped fix maintainer triage already preferred for #68945.

## Best Possible Solution

Land this PR, or an equivalent scoped patch, with the existing resolver wired through Telegram's local runtime barrel and into the block-stream dispatcher options. Keep the scope limited to configured reply pacing for #68945 and do not treat it as a fix for Telegram long-poll stalls.
