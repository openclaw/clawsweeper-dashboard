## Summary

Keep open: the Azure support is a useful direction, but this branch should not merge because it uses generic OpenAI request construction instead of Azure deployment routing/auth, hard-codes the API version, misses supported Azure host families, and has only mock/local test proof.

## What This Changes

The branch adds Azure OpenAI endpoint detection, `api-key`/`api-version` OpenAI constructor defaults, mocked memory-lancedb test coverage, and a changelog entry.

## Best Possible Solution

Implement Azure-aware memory-lancedb embeddings with parsed host suffixes, `AzureOpenAI` or explicit deployment-scoped requests, configurable API versioning in config/manifest/docs, request-shape tests, and redacted live Azure proof.

## Reproduction Assessment

Yes. Source inspection of PR head shows the generic OpenAI client still posts `"/embeddings"` and keeps bearer auth, while the OpenAI SDK Azure client rewrites embedding paths, adds `api-version`, and uses `api-key` auth.

## Solution Assessment

No. Constructor defaults on the generic OpenAI client are not enough; the safer solution is hostname-parsed Azure routing with deployment-scoped requests and configurable API-version ownership.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Build deployment-scoped Azure embedding requests:** `extensions/memory-lancedb/index.ts:357-363`
  - body: The PR only adds Azure defaults to the generic OpenAI client, while `embed()` still posts to `"/embeddings"`. For the resource-root workflow described by this PR, Azure needs `/openai/deployments/{deployment}/embeddings?api-version=...` or the SDK `AzureOpenAI` deployment rewrite, so plain Azure resource URLs still route incorrectly.
  - confidence: 0.92
- **[P2] Use Azure auth without bearer auth:** `extensions/memory-lancedb/index.ts:357-362`
  - body: Passing `apiKey` to the generic `OpenAI` client still emits `Authorization: Bearer ...`; adding `defaultHeaders: { "api-key": apiKey }` adds a second credential form rather than switching auth modes. Use `AzureOpenAI` or explicitly omit bearer auth for Azure requests.
  - confidence: 0.9
- **[P2] Expose the Azure API version in config:** `extensions/memory-lancedb/index.ts:363`
  - body: The patch hard-codes `api-version=2024-02-01`, while memory-lancedb config and manifest do not accept an API-version setting. Users needing a different supported Azure API version would have to fork the plugin.
  - confidence: 0.9
- **[P2] Parse Azure hosts from the URL hostname:** `extensions/memory-lancedb/index.ts:354`
  - body: Substring matching the full `baseUrl` can switch on non-Azure URLs whose path or query contains `.openai.azure.com`, and it misses Azure host families OpenClaw already recognizes such as `.services.ai.azure.com` and `.cognitiveservices.azure.com`. Parse `URL.hostname` and suffix-check known Azure hosts before changing request behavior.
  - confidence: 0.84
