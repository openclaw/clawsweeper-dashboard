## Summary

Keep open: the Azure embeddings feature is useful, but this PR still routes the Azure path through the generic OpenAI embeddings transport with bearer auth and a fixed API version, and it lacks redacted live Azure proof.

## What This Changes

The PR adds Azure OpenAI endpoint detection to memory-lancedb embeddings, injects `api-key` and `api-version` defaults, adds a mocked constructor-options test, and updates the changelog.

## Best Possible Solution

Implement Azure-aware memory-lancedb embeddings with parsed Azure host suffixes, deployment-scoped or `AzureOpenAI` routing, configurable API versioning in config/manifest/docs, focused request-shape tests, and redacted live Azure proof.

## Reproduction Assessment

Yes, by source inspection for the PR defects: the patched path still constructs the generic OpenAI client and posts `"/embeddings"`, while the SDK Azure client rewrites deployment paths, adds `api-version`, and uses `api-key` auth. No live after-fix Azure proof is present.

## Solution Assessment

No. Constructor-level defaults on the generic OpenAI client are not sufficient; use `AzureOpenAI` or an explicit Azure request path with config-controlled API versioning and hostname parsing.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Build deployment-scoped Azure embedding requests:** `extensions/memory-lancedb/index.ts:357-361`
  - body: The branch only adds Azure defaults to the generic OpenAI client; `embed()` still posts to `"/embeddings"`. For the resource-root workflow described by this PR, Azure needs `/openai/deployments/{deployment}/embeddings?api-version=...` or the SDK `AzureOpenAI` deployment rewrite, so plain Azure resource URLs still route incorrectly.
  - confidence: 0.92
- **[P2] Use Azure auth without bearer auth:** `extensions/memory-lancedb/index.ts:356-360`
  - body: Passing `apiKey` to the generic `OpenAI` client still emits `Authorization: Bearer ...`; adding `defaultHeaders: { "api-key": apiKey }` adds a second credential form rather than switching auth modes. Use `AzureOpenAI` or explicitly omit bearer auth for Azure requests.
  - confidence: 0.9
- **[P2] Expose the Azure API version in config:** `extensions/memory-lancedb/index.ts:361`
  - body: The patch hard-codes `api-version=2024-02-01`, while memory-lancedb config and manifest do not accept an API-version setting. Users needing a different supported Azure API version would have to fork the plugin.
  - confidence: 0.9
- **[P2] Parse Azure hosts from the URL hostname:** `extensions/memory-lancedb/index.ts:352`
  - body: Substring matching the full `baseUrl` can switch on non-Azure URLs whose path or query contains `.openai.azure.com`, and it misses Azure host families OpenClaw already recognizes such as `.services.ai.azure.com` and `.cognitiveservices.azure.com`. Parse `URL.hostname` and suffix-check known Azure hosts before changing request behavior.
  - confidence: 0.84
