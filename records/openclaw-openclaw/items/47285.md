## Summary

Keep open: the feature is useful, but the current diff is not merge-ready because native Azure embeddings still lack deployment-scoped URL handling, Azure-only auth, configurable API-version support, robust host parsing, and real Azure proof.

## What This Changes

Adds Azure OpenAI endpoint detection to memory-lancedb's direct OpenAI embedding client, injects api-key/api-version defaults, adds mocked constructor coverage, and updates CHANGELOG.md.

## Best Possible Solution

Implement Azure-aware memory-lancedb request shaping with parsed Azure host suffixes, deployment-scoped embeddings URLs or AzureOpenAI, configurable apiVersion in config/manifest/docs, focused tests, and redacted live Azure proof while preserving the generic OpenAI-compatible path.

## Reproduction Assessment

Yes for the PR defects by source inspection, but not by live Azure execution. Configure memory-lancedb with embedding.provider openai, an API key, an Azure base URL, and a deployment/model; the request construction shows the remaining URL/auth/config gaps.

## Solution Assessment

No. The current constructor-only patch is not the best solution because native Azure support needs endpoint normalization, Azure auth semantics, API-version configuration, robust hostname parsing, docs/tests, and runtime proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Build deployment-scoped Azure embedding URLs:** `extensions/memory-lancedb/index.ts:359`
  - body: For the resource-root workflow described by this PR, baseUrl still feeds the regular OpenAI client and embed() posts to /embeddings, which would target https://resource.openai.azure.com/embeddings instead of Azure's /openai/deployments/{deployment}/embeddings route. Normalize or validate Azure base URLs, or use the SDK's Azure deployment request path before treating this as native Azure support.
  - confidence: 0.92
- **[P2] Suppress bearer auth for Azure API-key calls:** `extensions/memory-lancedb/index.ts:358-362`
  - body: Passing apiKey to the normal OpenAI client still enables Authorization: Bearer ..., and the PR only adds api-key as an extra default header. Azure API-key requests should use the Azure key header without also sending an invalid bearer credential.
  - confidence: 0.89
- **[P2] Expose Azure API version in config:** `extensions/memory-lancedb/index.ts:363`
  - body: The patch hard-codes api-version=2024-02-01 while MemoryConfig, the strict config key guard, the manifest schema, and docs still expose no API-version setting. Users whose Azure tenant requires a different version would need to fork the plugin.
  - confidence: 0.9
- **[P2] Parse Azure hosts by hostname suffix:** `extensions/memory-lancedb/index.ts:354`
  - body: baseUrl?.includes(".openai.azure.com") can match path or query text and misses Azure host families already recognized elsewhere in OpenClaw, including *.services.ai.azure.com and *.cognitiveservices.azure.com. Parse the URL hostname and check known suffixes instead.
  - confidence: 0.84
