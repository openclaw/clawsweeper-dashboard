## Summary

Keep open for maintainer review: the Azure memory-lancedb feature is not implemented on current main, but this PR still has Azure request-shaping, auth, and API-version defects, and it lacks live Azure proof.

## What This Changes

The PR adds Azure OpenAI detection to memory-lancedb's direct OpenAI embedding client, injects `api-key` and `api-version` defaults, adds mocked coverage, and adds a changelog entry.

## Best Possible Solution

Implement Azure-aware memory-lancedb embeddings with parsed Azure host detection, deployment-scoped routing through `AzureOpenAI` or an explicit Azure request path, non-bearer API-key auth, configurable API versioning, focused tests, and redacted live Azure proof.

## Reproduction Assessment

Yes, source-reproducible. PR head still uses the generic OpenAI client, bearer-capable `apiKey`, and `/embeddings` routing, while the OpenAI SDK Azure source shows Azure-specific API-version, deployment rewrite, and `api-key` auth behavior.

## Solution Assessment

No. Constructor defaults on the generic OpenAI client are not the best native Azure support path; the maintainable fix should own Azure routing, auth, host parsing, and API-version configuration together.

Full review comments:

- **[P2] Build deployment-scoped Azure embedding requests:** `extensions/memory-lancedb/index.ts:357-364`
  - body: The patch only adds Azure defaults to the generic OpenAI client, but memory-lancedb still posts to `/embeddings`. Resource-root Azure URLs need deployment-scoped embedding routing, so this can advertise Azure support while still failing real Azure embedding calls.
  - confidence: 0.92
- **[P2] Switch Azure credentials away from bearer auth:** `extensions/memory-lancedb/index.ts:360`
  - body: Passing `apiKey` to the generic `OpenAI` client still creates bearer auth; adding `defaultHeaders: { "api-key": apiKey }` sends an Azure header without cleanly switching auth mode. Use `AzureOpenAI` or an explicit request path that does not emit bearer auth for API-key Azure requests.
  - confidence: 0.9
- **[P2] Expose the Azure API version in config:** `extensions/memory-lancedb/index.ts:361`
  - body: The PR hard-codes `api-version=2024-02-01`, but memory-lancedb config and manifest do not accept an API-version setting. Users needing another supported Azure version would have no supported override path.
  - confidence: 0.9
- **[P2] Parse Azure hosts before changing auth:** `extensions/memory-lancedb/index.ts:352`
  - body: Checking `baseUrl?.includes(".openai.azure.com")` can match non-host text and misses Azure host families OpenClaw already treats as Azure, such as `.services.ai.azure.com` and `.cognitiveservices.azure.com`. Parse `URL.hostname` and suffix-check known Azure hosts before changing request behavior.
  - confidence: 0.84
