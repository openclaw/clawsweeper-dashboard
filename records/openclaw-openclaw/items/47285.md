## Summary

Keep open: this is a useful memory-lancedb Azure embeddings feature, but current main does not implement it and the PR branch still uses the generic OpenAI request shape without live Azure proof.

## What This Changes

The branch adds Azure endpoint detection to memory-lancedb's direct OpenAI embeddings client, injects api-key/api-version constructor defaults, adds mocked coverage, and updates the changelog.

## Best Possible Solution

Implement Azure-aware memory-lancedb embeddings with parsed host suffixes, AzureOpenAI or explicit deployment-scoped requests, configurable API versioning in config/manifest/docs, request-shape tests, and redacted live Azure proof.

## Reproduction Assessment

Yes for the PR defect: source inspection of PR head shows the generic OpenAI client still posts to /embeddings and generic OpenAI auth still emits bearer auth, while the OpenAI SDK Azure client rewrites deployment endpoints and uses api-key auth. I did not run a live Azure tenant.

## Solution Assessment

No. Constructor defaults on the generic OpenAI client are not enough; the maintainable fix is Azure-aware routing with deployment-scoped requests, hostname-suffix parsing, and explicit API-version ownership.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Build deployment-scoped Azure embedding requests:** `extensions/memory-lancedb/index.ts:355-363`
  - body: The PR only adds Azure defaults to the generic OpenAI client, while embed() still posts to /embeddings. For Azure resource-root workflows, this needs /openai/deployments/{deployment}/embeddings?api-version=... or the SDK AzureOpenAI deployment rewrite.
  - confidence: 0.92
- **[P2] Switch Azure requests to api-key auth only:** `extensions/memory-lancedb/index.ts:360`
  - body: Passing apiKey to the generic OpenAI client still emits Authorization: Bearer; adding defaultHeaders with api-key sends a second credential form rather than switching auth modes. Use AzureOpenAI or explicitly omit bearer auth for Azure requests.
  - confidence: 0.9
- **[P2] Expose the Azure API version in config:** `extensions/memory-lancedb/index.ts:361`
  - body: The patch hard-codes api-version=2024-02-01, while memory-lancedb config and manifest do not accept an API-version setting. Users needing a different supported Azure API version would have to fork the plugin.
  - confidence: 0.9
- **[P2] Parse Azure endpoints by hostname suffix:** `extensions/memory-lancedb/index.ts:352`
  - body: Substring matching the full baseUrl can switch on non-Azure URLs whose path or query contains .openai.azure.com, and it misses Azure host families OpenClaw already recognizes such as .services.ai.azure.com and .cognitiveservices.azure.com.
  - confidence: 0.84
