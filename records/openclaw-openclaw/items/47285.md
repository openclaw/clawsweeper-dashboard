## Summary

Keep open: native Azure embedding support is useful, but current main does not implement it and this PR remains incomplete because it would still call the wrong Azure embeddings path, keep generic OpenAI bearer auth, and hard-code the API version without config/docs/tests.

## What This Changes

The PR changes memory-lancedb's OpenAI-compatible embeddings client to detect `.openai.azure.com` base URLs and add Azure `api-key` plus `api-version=2024-02-01` defaults.

## Best Possible Solution

Add Azure-aware memory-lancedb embedding support with parsed hostname detection, deployment-scoped embeddings URL construction, configurable API version, docs/changelog, and targeted tests while preserving existing OpenAI-compatible proxy behavior.

## Reproduction Assessment

Yes. Source inspection shows the current path and this PR's approach route a resource-root Azure base URL through a generic `/embeddings` request, while Azure requires a deployment-scoped embeddings URL with `api-version`.

## Solution Assessment

No. The constructor-only patch is not the best fix because Azure support needs endpoint normalization, auth-header selection, API-version configuration, tests, and docs rather than hard-coded defaults on the generic OpenAI client.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Build deployment-scoped Azure embedding URLs:** `extensions/memory-lancedb/index.ts:175-176`
  - body: For the resource-root workflow described by the PR, keeping `baseURL` as the Azure resource root still makes `embed()` post to `/embeddings`. Azure embeddings require `/openai/deployments/{deployment-id}/embeddings?api-version=...`, so this will call the wrong endpoint.
  - confidence: 0.93
- **[P2] Remove generic bearer auth for Azure requests:** `extensions/memory-lancedb/index.ts:173-178`
  - body: The Azure branch still constructs the generic OpenAI client with `apiKey`, and OpenAI SDK v6.35.0 adds `Authorization: Bearer <apiKey>`. Adding an `api-key` header does not remove that invalid bearer credential for Azure API-key auth.
  - confidence: 0.88
- **[P2] Expose the Azure API version in config:** `extensions/memory-lancedb/index.ts:179`
  - body: Hard-coding `api-version=2024-02-01` gives users no supported way to use a tenant-required or newer Azure API version because the memory-lancedb config and manifest reject any API-version field.
  - confidence: 0.9
- **[P2] Parse Azure hostnames instead of matching the full URL:** `extensions/memory-lancedb/index.ts:172`
  - body: Checking `baseUrl?.includes(".openai.azure.com")` can match path or query text and misses Azure host families already handled elsewhere, including `*.services.ai.azure.com` and `*.cognitiveservices.azure.com`.
  - confidence: 0.84
