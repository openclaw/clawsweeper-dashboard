## Summary

Keep open: current main still lacks native Azure OpenAI support for memory-lancedb, and this PR has blocking request-shaping, configuration, and real-behavior-proof gaps before it can merge.

## What This Changes

The PR modifies memory-lancedb’s OpenAI-compatible embedding client to detect `.openai.azure.com`, add `api-key` and fixed `api-version` defaults, and adds a mocked constructor-options test plus a changelog entry.

## Best Possible Solution

Implement Azure-aware memory-lancedb embeddings with hostname-suffix detection, deployment-scoped routing or `AzureOpenAI`, configurable API versioning, focused request-shape tests, docs/manifest follow-through, and redacted live Azure proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: current and PR code use the generic `/embeddings` path while the OpenAI SDK Azure client rewrites deployment endpoints and switches to `api-key` auth. No live Azure after-fix proof was supplied.

## Solution Assessment

No. Constructor defaults on the generic OpenAI client are not enough; the safer fix is an Azure-aware embeddings path with deployment routing, hostname parsing, API-version configuration, and runtime proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Build deployment-scoped Azure embedding requests:** `extensions/memory-lancedb/index.ts:358-363`
  - body: For the resource-root workflow described by this PR, setting constructor defaults still leaves `embed()` posting to `/embeddings`. Azure embeddings need deployment-scoped routing such as `/openai/deployments/{deployment}/embeddings?api-version=...`, or the SDK `AzureOpenAI` deployment rewrite, so a plain Azure resource URL will hit the wrong route.
  - confidence: 0.92
- **[P2] Use Azure API-key auth without bearer auth:** `extensions/memory-lancedb/index.ts:360`
  - body: Passing `apiKey` to the generic OpenAI client still makes the SDK send `Authorization: Bearer ...`; adding `defaultHeaders: { "api-key": apiKey }` sends both credentials instead of replacing bearer auth. Use `AzureOpenAI` or an Azure-specific transport path that sends only the expected Azure credential form.
  - confidence: 0.9
- **[P2] Expose the Azure API version in config:** `extensions/memory-lancedb/index.ts:361`
  - body: The PR hard-codes `api-version=2024-02-01`, while memory-lancedb config and manifest do not accept an API-version setting. Users who need a different supported Azure API version would have to fork the plugin.
  - confidence: 0.9
- **[P2] Parse Azure hosts from URL hostname:** `extensions/memory-lancedb/index.ts:352`
  - body: `baseUrl?.includes(".openai.azure.com")` can match path or query text on non-Azure hosts and misses Azure host families OpenClaw already recognizes elsewhere, including `.services.ai.azure.com` and `.cognitiveservices.azure.com`. Parse `URL.hostname` and check known suffixes before switching request behavior.
  - confidence: 0.84
