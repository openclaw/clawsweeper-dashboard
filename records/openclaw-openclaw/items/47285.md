## Summary

Keep open: the Azure memory-lancedb feature is still useful, but this patch does not yet implement the Azure embedding request shape/auth/version contract and it lacks real Azure behavior proof.

## What This Changes

The PR adds Azure endpoint detection to memory-lancedb OpenAI-compatible embeddings, injects api-key/api-version constructor defaults, adds mocked coverage, and updates the changelog.

## Best Possible Solution

Implement Azure-aware memory-lancedb embeddings with parsed Azure host suffixes, AzureOpenAI or explicit deployment-scoped requests, configurable API versioning in config/manifest/docs, request-shape tests, and redacted live Azure proof.

## Reproduction Assessment

Yes. Source inspection of PR head and OpenAI Node v6.37.0 shows the patch still uses the generic client/post path while AzureOpenAI owns api-key auth, api-version query injection, and deployment endpoint rewriting.

## Solution Assessment

No. Constructor defaults on the generic OpenAI client are not enough; the maintainable fix is Azure-specific request construction plus explicit API-version ownership in the memory-lancedb config contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Build deployment-scoped Azure embedding requests:** `extensions/memory-lancedb/index.ts:357-363`
  - body: The patch only adds Azure defaults to the generic OpenAI client, but the embedding call still posts to `/embeddings`. For the resource-root Azure workflow described here, requests need `/openai/deployments/{deployment}/embeddings?api-version=...` or the SDK `AzureOpenAI` deployment rewrite.
  - confidence: 0.93
- **[P2] Use Azure auth without bearer auth:** `extensions/memory-lancedb/index.ts:357-363`
  - body: Passing `apiKey` to the generic `OpenAI` client still emits `Authorization: Bearer ...`; this patch adds `api-key` as an extra header instead of switching auth modes. Use `AzureOpenAI` or an explicit Azure request path that omits bearer auth.
  - confidence: 0.9
- **[P2] Expose the Azure API version in config:** `extensions/memory-lancedb/index.ts:363`
  - body: The patch hard-codes `api-version=2024-02-01`, while memory-lancedb config and manifest do not accept an API-version setting. Users needing a different supported Azure API version would have to fork the plugin.
  - confidence: 0.9
- **[P2] Parse Azure hosts from the URL hostname:** `extensions/memory-lancedb/index.ts:354`
  - body: Substring matching the whole `baseUrl` can trigger on non-Azure paths or queries and misses Azure host families OpenClaw already recognizes, such as `*.services.ai.azure.com` and `*.cognitiveservices.azure.com`. Parse `URL.hostname` and suffix-check known Azure hosts before changing request behavior.
  - confidence: 0.85
