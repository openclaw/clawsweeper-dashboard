## Summary

Keep open: current main still lacks native Azure OpenAI support for memory-lancedb embeddings, but this PR has blocking Azure request-shaping/config defects and still lacks live Azure behavior proof.

## What This Changes

The PR adds Azure endpoint detection to memory-lancedb’s OpenAI-compatible embedding client, adds a mocked constructor-options test, and adds a changelog entry.

## Best Possible Solution

Implement Azure-aware memory-lancedb embeddings with hostname-suffix detection, deployment-scoped routing or AzureOpenAI, configurable API versioning, focused request-shape tests, and redacted live Azure proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: the patched Azure path still uses generic OpenAI request construction and posts `/embeddings`, while the OpenAI SDK Azure client handles deployment routing and api-key auth. No live Azure after-fix proof is present.

## Solution Assessment

No. Constructor-only Azure detection is not the best solution because native Azure support needs Azure request routing/auth semantics, hostname parsing, API-version configuration, and runtime proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Build deployment-scoped Azure embedding requests:** `extensions/memory-lancedb/index.ts:357-360`
  - body: For the resource-root workflow described by this PR, setting `baseURL` still leaves `embed()` posting to `/embeddings`. Azure embeddings need deployment-scoped routing such as `/openai/deployments/{deployment}/embeddings?api-version=...`, so `https://resource.openai.azure.com` will hit the wrong route.
  - confidence: 0.92
- **[P2] Avoid bearer auth on Azure API-key requests:** `extensions/memory-lancedb/index.ts:357-363`
  - body: Passing `apiKey` to the generic OpenAI client still makes the SDK add `Authorization: Bearer ...`; adding `defaultHeaders: { "api-key": apiKey }` does not suppress that header. Use `AzureOpenAI` or an Azure-specific transport path that sends API-key auth without the conflicting bearer header.
  - confidence: 0.89
- **[P2] Expose the Azure API version in plugin config:** `extensions/memory-lancedb/index.ts:363`
  - body: The PR hard-codes `api-version=2024-02-01`, while the memory-lancedb config parser and manifest do not accept an `apiVersion` setting. Users who need a different supported Azure API version would have to fork the plugin.
  - confidence: 0.9
- **[P2] Parse Azure hosts by hostname suffix:** `extensions/memory-lancedb/index.ts:354`
  - body: `baseUrl?.includes(".openai.azure.com")` can match path/query text on non-Azure hosts and misses Azure host families that OpenClaw already recognizes elsewhere, including `.services.ai.azure.com` and `.cognitiveservices.azure.com`. Parse `URL.hostname` and check known suffixes before switching auth/request behavior.
  - confidence: 0.84
