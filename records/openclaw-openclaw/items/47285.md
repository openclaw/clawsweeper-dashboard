## Summary

Keep open: current main still does not provide native Azure OpenAI support for memory-lancedb, and this PR is still not merge-ready because it adds Azure defaults to the generic OpenAI client without fixing the Azure deployment route, bearer-auth behavior, API-version ownership, or real Azure proof.

## What This Changes

The PR adds Azure endpoint detection to memory-lancedb OpenAI embeddings, injects `api-key` and `api-version` constructor defaults, adds mocked coverage, and updates the changelog.

## Best Possible Solution

Implement Azure-aware memory-lancedb embeddings with parsed host suffixes, `AzureOpenAI` or explicit deployment-scoped requests, configurable API-version ownership in config/manifest/docs, focused request-shape tests, and redacted live Azure proof.

## Reproduction Assessment

Yes, source-reproducible. PR head still builds a generic OpenAI client and posts to `/embeddings`, while the OpenAI SDK Azure client source shows deployment-path rewriting, `api-version`, and `api-key` auth are Azure-specific behavior.

## Solution Assessment

No. The current patch is not the best fix because constructor defaults on the generic OpenAI client do not provide the native Azure route, auth mode, or configurable API-version contract this feature needs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Build deployment-scoped Azure embedding requests:** `extensions/memory-lancedb/index.ts:357-363`
  - body: The patch only adds Azure defaults to the generic OpenAI client, but `embed()` still posts to `/embeddings`. For the resource-root Azure workflow described here, requests need `/openai/deployments/{deployment}/embeddings?api-version=...` or the SDK `AzureOpenAI` deployment rewrite, otherwise plain Azure resource URLs still route incorrectly.
  - confidence: 0.92
- **[P2] Use Azure auth without bearer auth:** `extensions/memory-lancedb/index.ts:357-362`
  - body: Passing `apiKey` to the generic `OpenAI` client still emits `Authorization: Bearer ...`; adding `defaultHeaders: { "api-key": apiKey }` adds a second credential form rather than switching auth modes. Use `AzureOpenAI` or explicitly build Azure requests without bearer auth.
  - confidence: 0.9
- **[P2] Expose the Azure API version in config:** `extensions/memory-lancedb/index.ts:363`
  - body: The patch hard-codes `api-version=2024-02-01`, while memory-lancedb config and manifest do not accept an API-version setting. Users needing a different supported Azure API version would have to fork the plugin.
  - confidence: 0.9
- **[P2] Parse Azure hosts before switching request behavior:** `extensions/memory-lancedb/index.ts:354`
  - body: Substring matching the full `baseUrl` can switch on non-Azure URLs whose path or query contains `.openai.azure.com`, and it misses Azure host families OpenClaw already recognizes such as `.services.ai.azure.com` and `.cognitiveservices.azure.com`. Parse `URL.hostname` and suffix-check known Azure hosts before changing request behavior.
  - confidence: 0.84
