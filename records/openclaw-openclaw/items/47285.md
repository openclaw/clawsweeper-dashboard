## Summary

Keep open: the Azure OpenAI memory-lancedb feature is useful, but this PR is not merge-ready because it adds generic OpenAI constructor defaults instead of complete Azure request shaping, leaves API-version ownership unconfigured, and still lacks real Azure behavior proof.

## What This Changes

The branch adds Azure endpoint detection to memory-lancedb's direct OpenAI-compatible embedding client, constructor-level `api-key`/`api-version` defaults, mocked coverage, and a changelog entry.

## Best Possible Solution

Implement Azure-aware memory-lancedb embeddings with parsed Azure host detection, `AzureOpenAI` or explicit deployment-scoped requests, configurable API versioning in config/manifest/docs, request-shape tests, and redacted live Azure proof.

## Reproduction Assessment

Yes. Source inspection of PR head and OpenAI SDK v6.37.0 shows the generic OpenAI client still uses bearer auth and only posts the existing `/embeddings` path, while `AzureOpenAI` owns deployment rewriting and `api-key` auth.

## Solution Assessment

No. Constructor defaults on the generic OpenAI client are not enough; the maintainable path is parsed Azure host handling with deployment-scoped requests and explicit API-version ownership.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Build deployment-scoped Azure embedding requests:** `extensions/memory-lancedb/index.ts:357-364`
  - body: The PR only adds Azure defaults to the generic OpenAI client, so a resource-root Azure URL still posts to `/embeddings` rather than `/openai/deployments/{deployment}/embeddings?api-version=...`. Use `AzureOpenAI` or explicitly build the deployment-scoped embedding request.
  - confidence: 0.92
- **[P2] Avoid sending bearer auth to Azure endpoints:** `extensions/memory-lancedb/index.ts:362`
  - body: Passing `apiKey` to the generic OpenAI client still emits `Authorization: Bearer ...`; adding `defaultHeaders: { "api-key": apiKey }` adds a second credential form rather than switching auth modes. Use Azure auth semantics or explicitly omit bearer auth for Azure requests.
  - confidence: 0.88
- **[P2] Expose the Azure API version in config:** `extensions/memory-lancedb/index.ts:363`
  - body: The patch hard-codes `api-version=2024-02-01`, while the memory-lancedb config parser and manifest reject any API-version setting. Users needing another supported Azure version would have to fork the plugin.
  - confidence: 0.9
- **[P2] Parse Azure hostnames before switching modes:** `extensions/memory-lancedb/index.ts:354`
  - body: Checking the full `baseUrl` string can trigger on a path or query containing `.openai.azure.com`, and it misses Azure AI Foundry hosts that current OpenClaw onboarding recognizes, such as `.services.ai.azure.com`. Parse `URL.hostname` and suffix-check supported Azure hosts before changing request behavior.
  - confidence: 0.86
