## Summary

Keep open: this is a useful memory-lancedb feature direction, but the current PR still has blocking Azure request-shaping defects and lacks the required real Azure runtime proof before merge.

## What This Changes

The PR adds Azure OpenAI detection to memory-lancedb's direct OpenAI embedding client, passes api-key/api-version SDK defaults, adds mocked constructor coverage, and updates the changelog.

## Best Possible Solution

Implement Azure-aware memory-lancedb request shaping with parsed Azure host suffixes, deployment-scoped embeddings URLs, configurable API version, docs/tests, and redacted live Azure proof while preserving the generic OpenAI-compatible path.

## Reproduction Assessment

Yes for the PR defects by source inspection, but not by live Azure execution. Configure memory-lancedb with `embedding.provider: "openai"`, an API key, an Azure base URL, and a deployment/model; the PR still leaves URL/auth/config gaps visible in the request construction.

## Solution Assessment

No. The current constructor patch is not the best fix because native Azure support needs URL normalization, bearer-auth suppression, API-version configuration, robust hostname parsing, docs, and runtime proof rather than hard-coded SDK defaults.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Build deployment-scoped Azure embedding URLs:** `extensions/memory-lancedb/index.ts:359`
  - body: For the resource-root workflow described by this PR, `baseUrl: "https://resource.openai.azure.com"` still leaves `embed()` posting to `/embeddings` instead of `/openai/deployments/{deployment}/embeddings`. Normalize Azure base URLs or require and validate a deployment-scoped base URL before treating this as native Azure support.
  - confidence: 0.92
- **[P2] Omit bearer auth for Azure API-key requests:** `extensions/memory-lancedb/index.ts:358-362`
  - body: Passing `apiKey` keeps openai-node bearer auth enabled, and adding `defaultHeaders.api-key` does not suppress the generated `Authorization: Bearer ...` header. Azure API-key requests should send the Azure key header without also sending an invalid bearer credential.
  - confidence: 0.88
- **[P2] Expose the Azure API version in config:** `extensions/memory-lancedb/index.ts:363`
  - body: The patch hard-codes `api-version=2024-02-01`, while memory-lancedb config and manifest still reject any API-version setting. Users whose Azure tenant requires a different version would have to fork the plugin to change this value.
  - confidence: 0.9
- **[P2] Parse Azure hostnames instead of searching the full URL:** `extensions/memory-lancedb/index.ts:354`
  - body: `baseUrl?.includes(".openai.azure.com")` can match path or query text and misses Azure host families OpenClaw already recognizes elsewhere, including `*.services.ai.azure.com` and `*.cognitiveservices.azure.com`. Parse the URL hostname and check known suffixes.
  - confidence: 0.84
