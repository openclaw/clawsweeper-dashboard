## Summary

Keep open: current main still has a clear telemetry gap where model fallback failures are copied to `agentMeta.fallbackAttempts` but not folded into `executionTrace`, while the reported OpenAI/Codex timeout still needs live auth-profile/runtime investigation.

## What This Changes

The issue reports that beta.3 reaches the Codex/OpenAI primary route but times out into Anthropic/Pi fallback, with `agentMeta.fallbackAttempts` and `executionTrace` disagreeing about whether fallback occurred.

## Best Possible Solution

Investigate the live OpenAI/Codex auth-profile timeout separately, and update the agent-runtime trace assembly so outer model-fallback attempts are reflected in `executionTrace.attempts` and `fallbackUsed`.

## Reproduction Assessment

Partially: the telemetry inconsistency is source-reproducible because current main stores outer fallback failures in `agentMeta.fallbackAttempts` without merging them into `executionTrace`. The primary OpenAI/Codex timeout itself was not live-reproduced in this review.

## Solution Assessment

Unclear for the timeout root cause; the telemetry fix should be a narrow agent-runtime change, but the primary Codex route needs live beta/auth-profile diagnosis before choosing the transport fix.
