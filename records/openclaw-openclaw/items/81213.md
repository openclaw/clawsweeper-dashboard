## Summary

Keep open: current main still has the reported fallback telemetry gap, and the primary OpenAI/Codex timeout needs a live auth-profile/runtime investigation before this can be closed or safely automated.

## What This Changes

The issue reports that beta.3 reaches the OpenAI/Codex runtime but times out to Anthropic/Pi fallback, while `executionTrace` omits the failed primary attempt and says fallback was not used.

## Best Possible Solution

Investigate the live OpenAI/Codex auth-profile timeout on a current package, and update agent runtime trace assembly so outer model-fallback attempts appear in `executionTrace.attempts` with `fallbackUsed: true`.

## Reproduction Assessment

Partially: the telemetry mismatch is source-reproducible from current main because outer fallback attempts are stored in `agentMeta` after the embedded runner has already built `executionTrace`. The primary OpenAI/Codex timeout itself was not live-reproduced in this review.

## Solution Assessment

Unclear overall: the telemetry fix should be a narrow agent-runtime change, but the timeout root cause needs a live current-main or current-beta smoke before selecting a transport/auth/profile fix.
