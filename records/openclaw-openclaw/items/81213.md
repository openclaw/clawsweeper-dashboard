## What This Changes

The issue reports that beta.3 reaches the Codex/OpenAI route but times out, succeeds only through Anthropic/Pi fallback, and returns inconsistent fallback telemetry.

## Best Possible Solution

Investigate the live OpenAI/Codex auth-profile timeout on a current packaged build, and independently update the agent runtime trace assembly so failed outer model-fallback attempts are reflected in `executionTrace.attempts` and `fallbackUsed`.

## Reproduction Assessment

Partially: the telemetry inconsistency is source-reproducible because current main stores outer fallback failures in `agentMeta.fallbackAttempts` without folding them into the returned `executionTrace`. The primary OpenAI/Codex timeout itself was not live-reproduced in this read-only review.

## Solution Assessment

Unclear for the timeout root cause; the telemetry fix should be a narrow agent-runtime change, but the primary Codex route needs live beta/auth-profile diagnosis before choosing the transport fix.
