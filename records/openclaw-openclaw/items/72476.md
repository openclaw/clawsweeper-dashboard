## Summary

Keep open: this is a valid bug-fix PR for the embedded stream option path, but it still needs changes because the new validator forwards invalid token counts and the external contributor has only mock/test proof, not real configured-run proof.

## What This Changes

The PR adds a `thinkingBudgets` resolver/forwarder in embedded extra params and regression coverage for passing MiniMax Anthropic-compatible budgets to pi-ai stream options.

## Best Possible Solution

Forward `thinkingBudgets` at the embedded stream-wrapper seam after filtering positive integer token budgets, keep focused regression coverage, and require redacted real configured-run proof before merge.

## Reproduction Assessment

Yes, from source inspection. Current main resolves configured per-model `params.thinkingBudgets` into effective extra params, but the embedded wrapper never copies that value into pi-ai `SimpleStreamOptions`.

## Solution Assessment

No, not yet. The forwarding seam is the right place, but the PR should reject zero, negative, and fractional token budgets and provide real behavior proof before it is merge-ready.
