## Summary

Keep open: current main still drops configured thinking budgets, but the PR is not merge-ready because its validator accepts invalid token counts and the external real-behavior proof gate is still unmet.

## What This Changes

The PR adds a `thinkingBudgets` resolver, forwards configured budgets into embedded pi-ai stream options, and adds MiniMax Anthropic-compatible regression coverage.

## Best Possible Solution

Forward `thinkingBudgets` at the embedded stream-wrapper seam after filtering to positive integer token counts, preserving provider defaults for omitted levels.

## Reproduction Assessment

Yes, from source inspection. Configure `agents.defaults.models["minimax/MiniMax-M2.7"].params.thinkingBudgets`; current main keeps that value in resolved extra params but never copies it into pi-ai stream options.

## Solution Assessment

No, not yet. The forwarding seam is the right place, but the PR should reject zero, negative, and fractional token budgets before assigning `streamParams.thinkingBudgets`.
