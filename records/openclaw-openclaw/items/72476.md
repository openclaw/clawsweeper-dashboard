## Summary

Keep open: the pass-through fix is still useful, but the current patch forwards invalid token counts and lacks required real behavior proof, so it should not merge as-is.

## What This Changes

The branch adds a `thinkingBudgets` resolver, forwards configured budgets into pi-ai stream options, and adds a MiniMax Anthropic-compatible regression test.

## Best Possible Solution

Forward `thinkingBudgets` at the embedded stream-wrapper boundary, filter to positive integer token budgets, keep the focused regression coverage, and require redacted real configured-run proof before merge.

## Reproduction Assessment

Yes, from source inspection. Current main resolves configured model `params` into effective extra params, but the embedded wrapper never copies `thinkingBudgets` into pi-ai `SimpleStreamOptions`.

## Solution Assessment

No, not yet. The forwarding seam is the right place, but the patch should reject zero, negative, and fractional token budgets before assigning `streamParams.thinkingBudgets`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Reject invalid thinking budget token counts:** `src/agents/pi-embedded-runner/extra-params.ts:145-147`
