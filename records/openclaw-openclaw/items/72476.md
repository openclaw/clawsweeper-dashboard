## Summary

Keep open: this PR addresses a real source-reproducible gap in the embedded runner, but the patch still forwards invalid token-budget values and the external contributor has not provided after-fix real behavior proof.

## What This Changes

The branch adds a `thinkingBudgets` resolver, forwards configured budgets into embedded pi-ai stream options, and adds a MiniMax-style Anthropic-compatible regression test.

## Best Possible Solution

Forward `thinkingBudgets` at the embedded stream-wrapper seam after accepting only positive integer token counts and preserving provider defaults for omitted values.

## Reproduction Assessment

Yes. Configure `agents.defaults.models["minimax/MiniMax-M2.7"].params.thinkingBudgets`; current main keeps that value in resolved extra params but never copies it into stream options before calling pi-ai.

## Solution Assessment

No, not yet. The chosen seam is maintainable, but the resolver should reject zero, negative, and fractional token counts before the option reaches Anthropic-style budget consumers.
