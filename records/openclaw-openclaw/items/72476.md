## What This Changes

The PR forwards configured `thinkingBudgets` from embedded runner extra params into pi-ai stream options and adds MiniMax Anthropic-compatible regression coverage.

## Best Possible Solution

Land a narrow version that forwards `thinkingBudgets` only after accepting positive integer token budgets, adds valid and invalid-value coverage, and includes a single-line changelog Fixes entry without changing provider defaults or config schema.

No, not yet. Forwarding in `src/agents/pi-embedded-runner/extra-params.ts` is the right seam, but the resolver should reject non-positive and fractional token counts and the user-facing behavior change needs changelog coverage.
