## What This Changes

Reported crash in v2026.6.6 where session SDK attribution headers dereference a missing `model.baseUrl` for `amazon-bedrock` models.

## Best Possible Solution

Guard or normalize `baseUrl` in the session SDK attribution helper and add a regression test proving a Bedrock model without `baseUrl` reaches the provider path without losing OpenRouter or Cloudflare attribution behavior.

## Reproduction Assessment

Yes from source inspection and the reporter's release steps: with install telemetry enabled, a Bedrock model object without `baseUrl` reaches `getAttributionHeaders()` and throws on `.includes()` before the provider request. I did not perform a live Bedrock call in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is to make the attribution helper tolerate absent `baseUrl` or delegate to the existing guarded attribution resolver. A broader Bedrock provider rewrite or new config option is unnecessary for this failure mode.
