## Summary

Keep open. The PR fixes a real current-main scanner miss, but the new state machine introduces two concrete parser regressions that can either false-fail or silently miss package dist validation.

## What This Changes

The PR replaces the package dist import literal heuristic with a forward scanner and adds focused tests for static imports, re-exports, dynamic imports, and negative import-like cases.

## Best Possible Solution

Keep the forward-scanner direction, but make import-state transitions syntax-aware enough to reject object/property names and inspect template interpolation expressions, with regression tests and package/dist validation before merge.

## Reproduction Assessment

Yes. A current-main node probe using `collectPackageDistImportErrors` with a missing destructured static import returned `[]`, and focused PR-head probes reproduce the two new scanner regressions.

## Solution Assessment

No, not yet. The forward scanner is a maintainable direction, but this patch still overmatches non-import `import` properties/methods and skips dynamic imports inside template interpolations.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Limit import state to real import syntax:** `scripts/lib/package-dist-imports.mjs:188-197`
  - body: This enters `import` state for any word-boundary `import` token. On the PR head, valid JavaScript like `const table = { import: "./not-imported.js" };` and `const table = { import() { return "./not-imported.js"; } };` is reported as a missing dist import, while current main returns `[]`. That can make package/release validation fail on ordinary data objects or methods named `import`, so the scanner needs a negative state/test for these non-import tokens.
  - confidence: 0.93
- **[P2] Preserve dynamic imports inside template expressions:** `scripts/lib/package-dist-imports.mjs:211-230`
  - body: The new template-literal branch skips the whole backtick body, including `${...}` expressions. A valid file containing ``const text = `${await import("./missing.js")}`;`` now returns `[]` on the PR head even though current main reports the missing file, so this can still silently omit real dynamic import edges from the dist closure.
  - confidence: 0.9
