## What This Changes

The PR replaces the package dist import literal scanner with a forward state machine and adds Vitest regression coverage for destructured imports, re-exports, dynamic literal imports, and negative import-like cases.

## Best Possible Solution

Merge a tightened state-machine scanner that preserves the new import/re-export coverage while rejecting non-import `import` tokens, then rerun the narrow test and dist-scanner proof.

## Reproduction Assessment

Yes. Current main’s reverse context scan is source-reproducible for destructured imports, and the PR body supplies terminal before/after output showing missing destructured and re-export specifiers are newly detected.

## Solution Assessment

No. The forward state machine is the right general direction, but the current implementation should verify real static import syntax before entering import state so valid non-import object properties do not become false package errors.
