## Summary

Keep open. Current main still misses the central destructured-import and re-export cases, and this PR is the right surface to fix that package-safety bug, but the new scanner still has dynamic-import edge cases that can produce both false positives and false negatives before merge.

## What This Changes

The PR replaces the package dist import scanner with a forward state machine and adds Vitest coverage for static imports, re-exports, dynamic imports, and negative import-like cases.

## Best Possible Solution

Land the scanner fix after separating static imports from dynamic import() parsing, accepting only literal dynamic arguments, and scanning template interpolation expressions while ignoring raw template text.

## Reproduction Assessment

Yes. A current-main node probe reproduces the central missed destructured import and re-export errors, and a raw PR-head probe reproduces the remaining dynamic-import false-positive and template-interpolation false-negative cases.

## Solution Assessment

No. The forward state machine is the right direction for the central bug, but it needs a narrower dynamic import() mode and template interpolation scanning before it is the best maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Require literal dynamic import arguments:** `scripts/lib/package-dist-imports.mjs:196-197`
  - body: After any `import` token that is not `import.meta`, this sets `stmtState = "import"` and records the next relative string anywhere before the statement ends. That makes valid code like `import(resolve("./chunk.js"))` fail the dist scanner even though `"./chunk.js"` is only an argument to `resolve`, not the import specifier; current main returns no error for that input. Please split dynamic `import(` into its own state and only accept a string literal when it is the first significant argument.
  - confidence: 0.91
- **[P3] Scan template interpolation imports:** `scripts/lib/package-dist-imports.mjs:211-236`
  - body: Skipping the whole template literal means a real dynamic import inside `${...}` is no longer checked; the PR head returns no error for `` `${await import("./missing.js")}` `` while current main reports the missing file. The scanner should ignore raw template text but still walk interpolation expressions, or add an explicit tested contract if maintainers want to stop supporting that shape.
  - confidence: 0.83
