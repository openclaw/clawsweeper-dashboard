## Summary

Keep open. Current `main` added a generic plugin `before_install` hook for skill dependency installs, but the separate ClawHub skill download/install path still bypasses that hook and any install-security scan. Because #18677 explicitly asks for a pre-activation hook when installing skills from ClawHub or other external sources, the request is only partially covered on `main`, not implemented well enough to close.

## Evidence

- **ClawHub skill install is still a separate flow:** The CLI docs explicitly say the plugin install safety flag/hook behavior applies to plugin installs and gateway-backed skill dependency installs, while `openclaw skills install` remains a separate ClawHub download/install flow.
  - file: [docs/cli/plugins.md:99](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/docs/cli/plugins.md#L99)

The hook-covered installer mode is not the same as ClawHub skill download/install.
