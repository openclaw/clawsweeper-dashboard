Full review comments:

- **[P2] Reject env keys with surrounding whitespace:** `extensions/openshell/src/config.ts:115-132`
  - body: The schema checks `trimmed` but returns the original object key, so a config like `{" FOO ": "bar"}` passes validation and later sends an invalid `--env  FOO =bar` argument. Validate the raw key or normalize the returned map before building create args.
  - confidence: 0.86
- **[P2] Align manifest env-key validation:** `extensions/openshell/openclaw.plugin.json:62-67`
  - body: The manifest schema for `env` accepts any property name while the runtime resolver rejects empty, `OPENSHELL_`, and non-identifier keys. Since loader validation uses the manifest JSON schema before plugin registration, add manifest-equivalent key constraints or a shared validation path plus negative coverage.
  - confidence: 0.87
- **[P3] Update the general sandbox recreate guide:** `docs/gateway/openshell.md:307`
  - body: This page now says env changes require sandbox recreation, but `docs/cli/sandbox.md` still lists only backend/from/mode/policy for the OpenShell recreate path. Add the new env key there too so users do not update config and keep running an old sandbox.
  - confidence: 0.78
