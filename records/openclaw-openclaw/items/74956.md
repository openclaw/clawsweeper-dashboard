## Summary

Keep open. Current main still omits `agentId` and `workspaceDir` from the internal `message:received` hook context even though dispatch has already resolved both values, so the reported per-agent hook routing bug remains actionable.

## What This Changes

The issue requests route identity on `message:received` hook events so WAL/session-state hooks can write to the correct agent workspace.

## Best Possible Solution

Add optional route identity to the internal inbound message hook context and keep the memory-writing policy in the hook/plugin layer.

## Reproduction Assessment

Yes. Source inspection on current main shows the dispatcher resolves `sessionAgentId` and `workspaceDir` before emitting `message:received`, while the hook type, mapper, docs, and tests omit those fields.

## Solution Assessment

Yes. Additive enrichment of the existing hook context is the narrowest maintainable fix because it preserves current hook consumers and avoids forcing WAL hooks to rediscover session routing.
