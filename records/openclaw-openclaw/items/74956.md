## Summary

Keep open. Current main still has a source-backed internal hook contract gap: `message:received` carries message/channel metadata but not the already-resolved agent/workspace routing data needed by workspace-scoped hooks.

## What This Changes

Request to add per-agent route context such as `workspaceDir` and/or `agentId` to the internal `message:received` hook so WAL/session-state hooks write to the correct agent workspace.

## Best Possible Solution

The maintainable end state is an additive internal hook contract where `message:received` carries optional route identity and workspace path wherever the dispatcher already has those values, with docs and tests updated while WAL-specific file-writing remains plugin-owned.

## Reproduction Assessment

Yes for the root cause: current source and docs show `message:received` lacks `agentId` and `workspaceDir`, while the main dispatcher resolves both before emitting the hook. The exact WAL file-writing symptom was not run because this was a read-only review and the reported dist hook is not in source.

## Solution Assessment

Yes. Additive context enrichment is the narrowest fix because it preserves existing hook consumers and avoids moving memory/WAL behavior into core.
