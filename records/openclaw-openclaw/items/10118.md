## Summary

Keep open: current main has dependency-backed Shift+Enter newline handling, but the requested user-facing docs/help coverage and configurable send/newline keybinding surface remain unresolved. ## What This Changes

The issue requests TUI multiline composition with Enter continuing to send and Shift+Enter, or a configurable binding, inserting a newline.

## Best Possible Solution

Keep pi-tui as the newline/submit implementation, document Shift+Enter and fallback behavior in public TUI docs and /help, and add an OpenClaw config/CLI bridge only after maintainers approve the keybinding surface.

## Reproduction Assessment

Not applicable as a bug reproduction: this is an enhancement request. Source and dependency inspection show Shift+Enter is partially available through pi-tui while docs/help and configurable keybinding support remain absent on current main.

## Solution Assessment

Unclear until maintainers decide the config surface. A direct OpenClaw Shift+Enter handler is not the narrowest fix because pi-tui already owns newline handling; the maintainable remainder is docs/help plus any approved keybinding bridge.
