## Summary

Keep open: current main already gets partial multiline composition from pi-tui, but the TUI docs and /help still omit newline composition, and configurable send/newline keybindings remain an unresolved product/API choice.

## What This Changes

The request asks for Enter to send while Shift+Enter or a configurable keybinding inserts a newline in the TUI input, with help and docs coverage.

## Best Possible Solution

Document Shift+Enter and the fallback newline behavior in TUI docs/help, then expose a supported keybinding preference only if maintainers choose a stable config or CLI contract.

## Reproduction Assessment

Not applicable as a feature request. Source inspection verifies dependency-backed partial support plus remaining docs/help/config gaps, so there is no failing current-main bug path to reproduce.

## Solution Assessment

Not yet. The dependency behavior covers part of the request, but the maintainable completion is docs/help plus a deliberate keybinding configuration design rather than a redundant CustomEditor handler.
