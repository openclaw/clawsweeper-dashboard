## Summary

Keep open: current main gets partial newline composition from pi-tui, but TUI docs and /help still omit the keybinding/fallback behavior and configurable send/newline keybindings remain an unresolved product/API choice.

## What This Changes

Request to support multiline TUI composition with Enter sending by default, Shift+Enter inserting a newline, optional configurable send behavior, and docs/help visibility.

## Best Possible Solution

Document Shift+Enter and fallback newline behavior in TUI docs/help, then expose a supported keybinding preference only if maintainers choose a stable config or CLI contract.
