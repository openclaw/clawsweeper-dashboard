## Summary

Keep this issue open. OpenClaw v2026.6.5 already ships Enter-to-send and dependency-backed Shift+Enter newline composition, but the canonical remaining work is still missing: the TUI guide and `/help` do not explain multiline input or terminal fallbacks, and configurable submit/newline keys require an explicit product decision.

## What This Changes

Document and optionally make configurable the TUI behavior where Enter submits and a modified Enter inserts a newline for multiline composition.

## Best Possible Solution

Document the shipped newline shortcut and terminal fallbacks in the TUI guide and `/help`, then add one canonical keybinding preference only if maintainers approve the config or CLI contract and its upgrade behavior.

## Reproduction Assessment

Yes at source level. Current main directly demonstrates the missing documentation, `/help` coverage, and public preference surface, although terminal-specific key sequences still need PTY or real-terminal validation during implementation.

## Solution Assessment

Yes for the narrowed direction: retain dependency-owned multiline editing and improve discoverability. Configurable submission behavior should be implemented only through an explicitly approved canonical contract, not an ad hoc wrapper shortcut.
