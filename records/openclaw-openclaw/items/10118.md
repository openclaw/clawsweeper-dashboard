## Summary

Keep open: current main has dependency-backed Shift+Enter newline support, but OpenClaw still lacks docs/help coverage and a CLI/config bridge for configurable send/newline preferences; recent discussion also kept this issue open for that remaining product work.

## What This Changes

The issue asks the TUI composer to keep Enter as send while supporting Shift+Enter newline composition, optional configurable send/newline keybindings, and docs/help hints.

## Best Possible Solution

Document current newline and fallback behavior in the TUI guide and /help, then expose configurable send/newline keybindings only after maintainers choose the config or CLI contract.

## Reproduction Assessment

Not applicable: this is a feature request rather than a broken current contract. Source and dependency inspection show partial Shift+Enter support plus remaining docs/help/config gaps.

## Solution Assessment

Not yet. The dependency-backed editor behavior is the right low-level path, but completing the issue needs user-facing docs/help and a deliberate keybinding preference design rather than another redundant CustomEditor Shift+Enter handler.
