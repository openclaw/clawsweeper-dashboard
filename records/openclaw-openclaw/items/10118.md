## Summary

Keep open because current main only partially satisfies the request: Shift+Enter newline handling is now provided by the pinned `pi-tui` editor, but OpenClaw still lacks user-facing docs/help for multiline composition and has no public CLI/config keybinding preference.

## What This Changes

Request support for composing multiline TUI messages with Shift+Enter while keeping Enter as send, with optional configurable send/newline keybindings and docs/help coverage.

## Best Possible Solution

Keep the dependency-backed Shift+Enter behavior, document multiline and fallback newline composition in the TUI guide and `/help`, then add a versioned CLI/config keybinding bridge only if maintainers want that public preference surface.

## Reproduction Assessment

Not applicable as a feature/configuration request. Source inspection does confirm the remaining gap: docs and `/help` omit multiline composition, and the CLI/runtime option surface has no send/newline keybinding preference.

## Solution Assessment

Unclear. The current dependency-backed Shift+Enter behavior is the narrow base implementation, but the configurable Enter/Ctrl+Enter/Shift+Enter part should wait for maintainer agreement on the public config and CLI contract.
