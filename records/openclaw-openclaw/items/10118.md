## Summary

Keep open: current main has dependency-backed Shift+Enter newline support, but the OpenClaw-facing docs, `/help` output, and configurable send/newline preference surface are still missing, and recent issue discussion explicitly preserved that remaining product scope.

## What This Changes

The issue requests TUI multiline composition where Enter sends and Shift+Enter, or a configurable equivalent, inserts a newline.

## Best Possible Solution

Document current newline and fallback behavior in the public TUI guide and `/help`, then expose configurable send/newline keybindings only after maintainers choose the config or CLI contract.

## Reproduction Assessment

Not applicable: this is a feature request rather than a broken current contract. Source and dependency inspection show partial Shift+Enter support plus remaining docs/help/config gaps.

## Solution Assessment

Not yet. The dependency-backed editor behavior is the right low-level path, but completing the issue needs user-facing documentation and a deliberate keybinding preference design rather than a redundant CustomEditor Shift+Enter handler.
