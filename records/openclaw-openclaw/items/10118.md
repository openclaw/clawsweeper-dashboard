## Summary

Current main and v2026.6.5 already provide dependency-backed Shift+Enter newline composition, but OpenClaw still omits shortcut and fallback guidance and has no intentional public customization bridge. The issue remains useful because alternate submit behavior requires a product decision rather than another editor callback.

## What This Changes

Requests reliable multiline TUI composition with Enter-to-send, Shift+Enter newline, optional configurable submit keys, and user-facing help.

## Best Possible Solution

Retain pi-tui as the canonical keybinding owner, document Shift+Enter and terminal fallback behavior in the TUI guide and `/help`, and expose configurable submit keys only if maintainers approve a narrowly scoped, backward-compatible preference contract.

## Reproduction Assessment

Yes at source level: v2026.6.5 and current main delegate to pi-tui's Shift+Enter newline behavior, while OpenClaw's docs and `/help` omit it and no OpenClaw keybinding preference exists. A terminal matrix was not run, so terminal-specific escape-sequence support was not runtime-reproduced here.

## Solution Assessment

No, the closed PR's extra Shift+Enter callback is not the best solution because it duplicates the dependency's editor contract. The maintainable solution is documentation and help for shipped behavior, followed by a separate maintainer decision on whether configurable submit keys justify a public setting.
