## Summary

Keep open: current main already gets Shift+Enter newline composition from `pi-tui`, but OpenClaw still does not document it in the TUI guide or `/help`, and there is no configurable send/newline keybinding surface. Prior discussion correctly narrowed this to remaining TUI docs/help and UX/config-contract work.

## What This Changes

Request support for multiline TUI composition with Shift+Enter as newline while keeping Enter as send, plus docs/help coverage and an optional configurable keybinding.

## Best Possible Solution

Document the current newline and fallback behavior in the TUI guide and `/help`, then expose a stable send/newline keybinding preference only after maintainers choose the config or CLI contract.

## Reproduction Assessment

Not applicable as a feature request. Source inspection establishes dependency-backed Shift+Enter support plus remaining docs/help/config gaps, so there is no broken current-main contract to reproduce.

## Solution Assessment

Not yet. The dependency behavior is the right low-level path, but the maintainable completion is docs/help plus a deliberate keybinding configuration design rather than a redundant `CustomEditor`-only handler.
