## Summary

Keep open: current main and the latest release have dependency-backed Shift+Enter newline handling, but OpenClaw still omits that behavior from TUI docs and `/help`, and configurable send/newline keybindings remain a maintainer product decision.

## What This Changes

Request multiline TUI composition where Enter sends, Shift+Enter inserts a newline, optional keybinding configuration is available, and docs/help describe the behavior.

## Best Possible Solution

Keep pi-tui as the newline/submit implementation, document terminal-specific Shift+Enter and fallback behavior in TUI docs and `/help`, and add a keybinding config/CLI bridge only after maintainer approval.

## Reproduction Assessment

Not applicable: this is an enhancement/keybinding/docs request rather than a broken current TUI contract. Source and dependency inspection show Shift+Enter is dependency-backed, while docs/help and configurable keybinding support remain absent.

## Solution Assessment
