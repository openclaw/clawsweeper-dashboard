## Summary

Keep open: current main and v2026.6.5 already have dependency-backed Shift+Enter newline insertion, but the TUI docs and `/help` still omit it and configurable Enter/Ctrl+Enter/Cmd+Enter behavior remains an unresolved product/config decision.

## What This Changes

The issue requests TUI multiline composition with Enter remaining send and Shift+Enter or configurable alternatives inserting newlines.

## Best Possible Solution

Keep the pi-tui newline/submit behavior, document the existing shortcut and fallback behavior in the TUI guide and `/help`, and add a configurable keybinding bridge only after maintainers approve the public TUI policy.

## Reproduction Assessment

Yes, by source inspection: baseline Shift+Enter newline behavior is present through pi-tui, while TUI docs, `/help`, and any configurable send/newline keybinding bridge are absent on current main and v2026.6.5.

## Solution Assessment

Unclear until maintainers decide whether OpenClaw should expose a public keybinding preference. The maintainable path is docs/help coverage plus an approved pi-tui keybinding bridge, not a duplicate custom Shift+Enter handler.
