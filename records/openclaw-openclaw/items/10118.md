## Summary

Keep open: current main and the latest release have dependency-backed Shift+Enter newline behavior through pi-tui, but OpenClaw still lacks the requested docs/help coverage and configurable send/newline keybinding surface; the linked implementation PR was closed unmerged and this issue remains the canonical follow-up.

## What This Changes

The issue requests TUI multiline composition where Enter sends by default and Shift+Enter or a configurable keybinding inserts newlines.

## Best Possible Solution

Keep pi-tui as the newline/submit implementation, document Shift+Enter and fallback behavior in the public TUI guide and /help, and add an OpenClaw keybinding config/CLI bridge only after maintainer approval.

## Reproduction Assessment

Not applicable as a bug reproduction: this is an enhancement request. Source and dependency inspection show Shift+Enter is partially available through pi-tui, while docs/help and configurable keybinding support remain absent on current main and v2026.5.12.

## Solution Assessment

Unclear until maintainers decide the config surface. A direct OpenClaw Shift+Enter handler is not the narrowest path because pi-tui already owns newline handling; the maintainable remainder is docs/help plus any approved keybinding bridge.
