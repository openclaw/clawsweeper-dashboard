## Summary

Keep open: current main and the latest release provide baseline Shift+Enter newline insertion through pi-tui, but OpenClaw still does not document the keybinding in TUI docs or /help, and configurable Enter/Ctrl+Enter/Shift+Enter behavior remains an unresolved public TUI UX/config decision.

## What This Changes

Request support for Shift+Enter multiline composition in `openclaw tui` while keeping Enter as send, with docs/help coverage and optional configurable send/newline keybindings.

## Best Possible Solution

Keep pi-tui as the newline/submit implementation, document Shift+Enter plus fallback behavior in the TUI guide and /help, and add a keybinding bridge only after maintainers approve the public TUI policy.

## Reproduction Assessment

Not applicable: this is an enhancement/keybinding/docs request rather than a broken current TUI contract. Source and dependency inspection show baseline Shift+Enter is dependency-backed, while docs/help and configurable keybinding support remain absent.

## Solution Assessment
