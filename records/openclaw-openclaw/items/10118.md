## Summary

Keep open: current main and v2026.6.5 already have baseline Shift+Enter newline insertion through pi-tui, but OpenClaw still omits the shortcut from TUI docs and `/help` and has no approved configurable send/newline keybinding surface.

## What This Changes

Request support and documentation for multiline TUI composition with Enter sending by default, Shift+Enter inserting a newline, and optional configurable send/newline keybindings.

## Best Possible Solution

Keep pi-tui editor delegation as the canonical runtime path, document newline and fallback behavior in the TUI guide and `/help`, then add a narrow pi-tui keybinding/config bridge only if maintainers approve that public option.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this is a TUI UX/config/docs request. Source and dependency inspection show baseline Shift+Enter newline behavior is present, while docs/help and configurable keybinding support remain absent.

## Solution Assessment

Unclear until maintainers decide whether OpenClaw should expose a public keybinding preference. The maintainable path is docs/help coverage plus an approved pi-tui keybinding bridge, not a duplicate custom Shift+Enter handler.
