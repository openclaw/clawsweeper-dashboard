## Summary

Keep open: current main and v2026.6.8 cover the default Enter-submit and Shift+Enter-newline behavior through pi-tui, but the requested TUI docs, /help coverage, and maintainer-approved configurable send/newline preference remain unresolved. ## What This Changes

Allow `openclaw tui` users to compose multiline prompts with Shift+Enter while keeping Enter-to-send and optionally exposing/documenting configurable send/newline keybindings.

## Best Possible Solution

Document the current Shift+Enter and fallback newline behavior in the public TUI guide and `/help`; if maintainers approve configuration, expose pi-tui keybinding overrides as the canonical TUI preference with focused tests.

## Reproduction Assessment

Yes, for the remaining source-level gap: current main and v2026.6.8 provide dependency-backed Shift+Enter newline behavior, but TUI docs, `/help`, CLI, and config schema do not expose the requested documentation or configurable send/newline bridge. ## Solution Assessment
