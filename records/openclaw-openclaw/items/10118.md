## Summary

Keep open: current main has dependency-backed Shift+Enter newline support through pi-tui, but OpenClaw still does not document it in the TUI guide or `/help`, and there is no OpenClaw config or CLI bridge for configurable send/newline keybindings. The prior detailed review and related items support keeping this as the canonical TUI keybinding/product follow-up rather than closing it as fully implemented.

## What This Changes

Request support for multiline TUI composition with Shift+Enter while keeping Enter as send, plus optional configurable send/newline keybindings and help/docs coverage.

## Best Possible Solution

Keep this issue open and narrow it to the remaining TUI product work: document Shift+Enter and fallback behavior in public TUI docs and `/help`, then decide whether to add an OpenClaw config/CLI bridge into pi-tui keybindings.

## Reproduction Assessment

Not applicable as a bug reproduction: this is an enhancement request. Source and upstream dependency inspection show Shift+Enter is partially available through pi-tui, while docs/help/configurable keybinding support remains absent on current main.

## Solution Assessment

Unclear until maintainers decide the config surface. A narrow direct Shift+Enter handler is not the best fix because pi-tui already owns that behavior; the maintainable remainder is docs/help plus any agreed keybinding-configuration bridge.
