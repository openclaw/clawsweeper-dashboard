## Summary

Keep open: current main and the latest release already get baseline Shift+Enter newline insertion from pi-tui, but OpenClaw still does not document the shortcut in the TUI guide or /help, and configurable Enter/Ctrl+Enter/Shift+Enter behavior remains a public TUI UX/config decision.

## What This Changes

Request support for multiline TUI composition with Enter-to-send by default, Shift+Enter for newline, optional configurable send/newline keybindings, and docs/help visibility.

## Best Possible Solution

Keep pi-tui as the newline/submit implementation, document Shift+Enter plus fallback behavior in the TUI guide and /help, and add a keybinding bridge only after maintainers approve the public TUI policy.

## Reproduction Assessment

Not applicable: this is an enhancement/keybinding/docs request rather than a broken current TUI contract. Source and dependency inspection show baseline Shift+Enter is dependency-backed, while docs/help and configurable keybinding support remain absent.

## Solution Assessment

Unclear until maintainers decide the public keybinding/config surface. The maintainable path is docs/help coverage plus an approved pi-tui keybinding bridge, not a duplicate custom Shift+Enter handler.
