## Summary

Keep open: current main and v2026.6.5 have baseline Shift+Enter newline insertion through pi-tui, but OpenClaw still does not document the shortcut in TUI docs or `/help`, and configurable Enter/Ctrl+Enter/Cmd+Enter behavior remains an unresolved product/config decision.

## What This Changes

The issue requests TUI multiline composition with Shift+Enter while keeping Enter as send, plus optional configurable send/newline keybindings and help/docs coverage.

## Best Possible Solution

Keep the pi-tui newline/submit behavior, document the existing shortcut and fallback behavior in the TUI guide and `/help`, and add a configurable keybinding bridge only after maintainers approve the public TUI policy.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this is a TUI UX/config/docs request. Source and dependency inspection show baseline Shift+Enter newline behavior is present through pi-tui, while docs/help and configurable keybinding support remain absent.

## Solution Assessment

Unclear until maintainers decide whether OpenClaw should expose a public keybinding preference. The maintainable path is docs/help coverage plus an approved pi-tui keybinding bridge, not a duplicate custom Shift+Enter handler.
