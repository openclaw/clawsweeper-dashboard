## Summary

Keep open: current main has dependency-backed Shift+Enter newline support through pi-tui, but the requested OpenClaw help/docs coverage and configurable send/newline keybinding surface are still unresolved and prior discussion explicitly kept this as the canonical follow-up.

## Best Possible Solution

## Solution Assessment

Unclear until maintainers decide the config surface. A direct OpenClaw Shift+Enter handler is not the narrowest fix because pi-tui already owns newline handling; the maintainable remainder is docs/help plus any approved keybinding bridge.
