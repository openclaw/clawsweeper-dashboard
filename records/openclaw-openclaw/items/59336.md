## Summary

Keep this PR open: current main still has the reported Raw-mode round-trip failure, but the proposed branch has a blocking exec safe-bin semantics regression and is missing the required user-facing changelog entry.

## What This Changes

The PR changes config raw redaction to compare restored raw snapshots against source config with undefined-key stripping, and changes exec safe-bin normalization to delete empty safe-bin profile/trusted-dir fields.

## Best Possible Solution

Land a maintainer-reviewed fix that keeps Raw mode available for source-vs-materialized snapshot differences while preserving exec safe-bin runtime semantics, adding focused regression tests and a `CHANGELOG.md` Fixes entry.

## Reproduction Assessment

Yes. Current main materializes own `undefined` safe-bin keys, compares restored JSON5 raw output against materialized `snapshot.config`, and then the Control UI disables Raw mode when that check drops `raw` to `null`.

## Solution Assessment

No. Comparing against source config and stripping undefined during the round-trip check is the right direction, but deleting safe-bin keys is not the safest fix; keep runtime policy semantics intact and limit compatibility normalization to the redaction comparison path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P1] Preserve safe-bin shadows when clearing fields:** `src/config/normalize-exec-safe-bin.ts:28-34`
  - body: Deleting these fields changes more than the raw-mode comparison. If an `exec` object has an own empty `safeBinProfiles` or `safeBinTrustedDirs` value while its prototype has one, this `delete` reveals the inherited value, and runtime policy resolution reads those properties through normal optional chaining. Please keep the runtime object shadowed or sanitize to own plain data, and strip `undefined` only for the raw round-trip comparison.
  - confidence: 0.78
- **[P3] Add the required changelog entry:** `CHANGELOG.md:33`
  - body: This is a user-facing Control UI/config editing fix, but the PR does not add an Unreleased `CHANGELOG.md` Fixes entry. OpenClaw policy requires one for user-visible fixes, with contributor credit that is safe to publish.
  - confidence: 0.9
