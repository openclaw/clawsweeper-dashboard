## Summary

Keep open: this PR targets a valid current and shipped Raw-mode regression, but it still lacks real behavior proof and introduces a security-sensitive safe-bin normalization regression that should block merge.

## What This Changes

The PR changes config raw-redaction round-trip comparison to use a source config with recursive undefined-key stripping and changes exec safe-bin normalization to delete empty safe-bin fields.

## Best Possible Solution

Land a revised fix that compares raw round-trip output against a JSON-serializable source-shaped config while preserving safe-bin policy clearing semantics, with focused regression tests and redacted real Control UI or terminal proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main writes undefined safe-bin own fields, compares restored raw text against materialized `snapshot.config`, and Control UI disables Raw mode when fallback sets `raw` to `null`; I did not run the UI in this read-only review.

## Solution Assessment

No. Comparing against a source-shaped config and stripping undefined values is the right direction, but deleting safe-bin fields is not the narrowest maintainable fix because it changes exec policy semantics.
