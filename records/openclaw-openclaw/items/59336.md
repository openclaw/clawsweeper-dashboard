## Summary

Keep open: the PR targets a valid current-main Raw mode regression, but the patch still has a blocking safe-bin policy/security concern and lacks inspectable after-fix real behavior proof.

## What This Changes

The PR changes config raw-redaction round-trip comparison to use the parsed source config with undefined-key stripping and changes exec safe-bin normalization to delete empty safe-bin fields.

## Best Possible Solution

Land a revised fix that compares raw round-trip output against the authored/source config shape while preserving safe-bin policy shadows or sanitizing prototypes, with focused regression tests and redacted real Control UI or terminal proof.

## Reproduction Assessment

Yes. Source inspection and a focused JavaScript proof show current main writes undefined safe-bin fields, compares restored raw text against materialized config, and disables Raw mode when fallback sets raw to null; I did not run the full UI because this review is read-only.

## Solution Assessment

No. Comparing against source config and recursively stripping undefined values is the right direction, but deleting safe-bin fields is not the narrowest maintainable fix because it changes exec policy semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve safe-bin shadows when clearing fields:** `src/config/normalize-exec-safe-bin.ts:28-34`
  - body: Deleting these fields removes the own-property shadow that previously masked inherited safeBinProfiles or safeBinTrustedDirs. Runtime policy later reads those properties through ordinary property access, so a polluted or inherited config object can regain policy values after normalization; keep the shadow, strip prototype data, or limit undefined removal to the raw comparison path.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The patch changes exec safe-bin normalization in a way that can expose inherited policy values after own fields are deleted.

Concerns:

- **[medium] Deleting safe-bin keys can reveal inherited policy data:** `src/config/normalize-exec-safe-bin.ts:28`
  - body: The diff deletes own safeBinProfiles and safeBinTrustedDirs when normalization makes them empty, while runtime policy resolution reads those properties through normal property access. On an object with inherited policy data, that can re-enable safe-bin profiles or trusted dirs that normalization meant to clear.
  - confidence: 0.86
