The PR changes config safe-bin normalization and raw snapshot redaction so the Control UI Config Raw tab can use source-config round-trip checks instead of materialized runtime-config checks.

## Best Possible Solution

Land the raw round-trip fix only after safe-bin normalization preserves sanitized own shadows or strips prototype data before runtime policy, with focused tests for both raw redaction and safe-bin policy boundaries.

## Reproduction Assessment

Yes. At source level, current main still creates undefined safe-bin own keys, compares restored raw against materialized `snapshot.config`, and nulls raw on mismatch; I did not run a live Control UI session in this read-only pass.

## Solution Assessment

No, not as written. Comparing against parsed/source config is the right direction, but deleting empty safe-bin fields is not the safest implementation because runtime policy later reads those fields through ordinary property access.
