## Summary

Keep open. The PR targets a real current-main Raw mode regression, but its safe-bin deletion changes a security-sensitive exec policy path and the external contributor has not provided after-fix real behavior proof.

## What This Changes

The PR changes config raw-redaction round-trip comparison to use source config with undefined-key stripping, and changes exec safe-bin normalization to delete empty safe-bin fields.

## Best Possible Solution

Land a revised fix that re-enables Raw mode for source-vs-materialized snapshot drift while preserving exec safe-bin policy semantics and the raw redaction safety fallback.

## Reproduction Assessment

Yes. Source inspection shows current main still writes own undefined safe-bin fields, compares restored raw text against materialized `snapshot.config`, and disables Raw mode when the fallback sets `raw` to `null`.

## Solution Assessment

No. Comparing against source config and stripping undefined during the round-trip check is the right direction, but deleting safe-bin fields is not the safest fix; preserve runtime policy shadows or sanitize prototypes and keep undefined stripping scoped to the comparison path.
