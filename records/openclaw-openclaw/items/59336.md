## Summary

Keep open: the Raw mode regression is still present on current main, but this PR is not mergeable as-is because it deletes safe-bin shadow properties on a security-sensitive exec policy path and lacks after-fix real behavior proof.

## What This Changes

The PR changes config raw-redaction round-trip comparison to use the source config with recursive undefined-key stripping, and changes exec safe-bin normalization to delete empty safe-bin fields.

## Best Possible Solution

Land a revised fix that compares raw round-trip output against a source-shaped config while preserving safe-bin own-property shadows or sanitizing prototypes, with focused regression tests and redacted real Control UI or terminal proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: current main writes undefined safe-bin fields, compares restored raw text against materialized `snapshot.config`, and Control UI disables Raw mode when the fallback sets `raw` to `null`; I did not execute the UI in this read-only review.

## Solution Assessment

No. Comparing against the source config and stripping undefined values in the round-trip check is the right direction, but deleting safe-bin fields is not the narrowest maintainable fix because it changes exec policy semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve safe-bin shadows when clearing fields:** `src/config/normalize-exec-safe-bin.ts:28-34`
