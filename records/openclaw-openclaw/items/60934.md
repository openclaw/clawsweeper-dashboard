## Summary

Keep open: the PR targets a real provider-auth contract gap, but the current patch is not safe to merge because it can still ignore the explicit token/oauth mode, leaves availability checks on different semantics, lacks real behavior proof, and is currently conflicting with main.

## What This Changes

This PR adds token/oauth filtering inside `resolveApiKeyForProvider` based on `models.providers.<id>.auth` before iterating auth profiles.

## Best Possible Solution

Centralize provider-level token/oauth profile eligibility and reuse it from resolver and availability paths, with focused regression coverage and redacted real setup proof.

## Reproduction Assessment

Yes. Source inspection shows `models.providers.*.auth` is an existing token/oauth config contract while current resolver and availability paths still scan profile order without a provider-level token/oauth eligibility filter.

## Solution Assessment

No. The inline resolver-only filter falls back to mismatched profiles and does not update availability checks; a shared strict eligibility helper is the narrower maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Do not fall back to mismatched auth profiles:** `src/agents/model-auth.ts:357`
  - body: When `models.providers.<id>.auth` is set to `"token"` or `"oauth"` and no matching profile type exists, this fallback reuses the unfiltered order, so the configured mode can still be ignored. Treat that as no eligible matching profile instead of silently trying the other auth mode.
  - confidence: 0.92
- **[P2] Reuse auth-mode filtering in availability checks:** `src/agents/model-auth.ts:350-357`
  - body: The new eligibility logic only changes `resolveApiKeyForProvider`; `hasAvailableAuthForProvider` still reports readiness from the unfiltered profile order. A bad matching token profile plus a valid OAuth profile can therefore look available but fail at execution, so both paths need the same helper.
  - confidence: 0.87
