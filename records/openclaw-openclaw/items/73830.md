## Summary

Keep open. The PR targets a real open installer report, but the proposed docs change is technically incorrect: `bash -s --` does not make child prompts terminal-backed, and current main still has the raw pipe in multiple install docs. This needs maintainer validation of the WSL2 failure and a consistent installer/docs direction, not cleanup closure.

## What This Changes

This PR changes the Getting Started macOS/Linux install snippet to pipe the hosted installer into `bash -s --` and adds a note saying those flags prevent stdin consumption.

## Best Possible Solution

Validate the linked WSL2 installer failure, then either harden `scripts/install.sh` so all interactive prompts and subprocesses use terminal-backed stdin or adopt one consistent public install command pattern across the install docs, with focused installer stdin regression coverage.

## Reproduction Assessment

Yes for the PR's claimed fix: a focused pipe smoke shows `bash -s --` still leaves child processes with non-TTY stdin. The full WSL2 hosted-installer hang from #73814 is still not fully reproduced in this read-only review.

## Solution Assessment

No. A single quickstart docs edit plus a false stdin guarantee is not the narrowest maintainable fix; the safer path is validated installer stdin handling or a consistent docs-wide caller pattern.
