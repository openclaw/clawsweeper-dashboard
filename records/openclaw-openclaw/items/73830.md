## Summary

Keep open: the PR addresses a real installer report, but the proposed docs change makes a false stdin-isolation claim and lacks after-fix real behavior proof.

## What This Changes

The PR changes the Getting Started macOS/Linux installer one-liner to `bash -s --` and adds a note saying those flags prevent stdin consumption.

## Best Possible Solution

Validate the WSL2 path, harden installer subprocess and onboarding stdin handling or document a tested caller pattern, then update all public install one-liners consistently with focused installer smoke coverage.

## Reproduction Assessment

Yes for the PR's claimed fix: a focused pipe smoke shows a child process can still consume the next script line under both `bash` and `bash -s --`. The full linked WSL2 hosted-installer hang was not executed.

## Solution Assessment

No. `bash -s --` is appropriate for passing arguments to a piped script, but it is not a stdin-isolation fix; installer-side `/dev/tty` handling or a tested download/process-substitution pattern is safer.
