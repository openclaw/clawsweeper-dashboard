## Summary

Keep this PR open. The linked installer stdin bug appears plausible, but this docs patch is not merge-ready because `bash -s --` does not isolate child-process stdin and the added note would incorrectly document the root issue as fixed.

## What This Changes

This PR changes the Getting Started macOS/Linux install snippet to pipe the hosted installer into `bash -s --` and adds a note claiming those flags prevent installer stdin consumption.

## Best Possible Solution

Validate the WSL2 installer failure, harden `scripts/install.sh` so interactive prompts and subprocesses use terminal-backed stdin or explicit noninteractive behavior, then update all public install commands consistently and add focused installer stdin regression coverage.

## Reproduction Assessment

Yes for the PR's claimed fix: a focused Bash pipe smoke shows `bash -s --` still lets a child read consume the remaining script stream. The full WSL2 hosted-installer hang from #73814 remains to be live-verified.

## Solution Assessment

No. Changing one quickstart snippet and adding a stdin-isolation guarantee is not the narrowest maintainable fix because the guarantee is false and the rest of the install documentation would remain inconsistent.
