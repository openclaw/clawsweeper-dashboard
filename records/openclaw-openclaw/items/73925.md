## Summary

Keep open. The PR targets a real Gateway handshake race, but the proposed connect-auth watchdog can be reset by duplicate preauth connect frames, so it would not actually bound unauthenticated auth work.

## What This Changes

The branch adds a Gateway WebSocket connect-auth watchdog, clears the original preauth timer after a valid connect frame, adds closed-state guards/tests, and updates protocol docs/changelog.

## Best Possible Solution

Repair the branch with a one-shot connect-auth deadline or explicit connecting state before the first auth await, reject duplicate preauth connects deterministically, add regression coverage, and preserve the contributor credit from #49751.

## Reproduction Assessment

Yes. Send a valid connect frame whose auth resolution stalls, then send another valid preauth connect before the watchdog fires; source inspection shows PR head keeps getClient() null until registration and re-arms the timer.

## Solution Assessment

No. The two-phase timer direction is narrow and maintainable, but this implementation is not the best fix while duplicate preauth connects can reset the auth-phase deadline.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Make the connect-auth watchdog one-shot:** `src/gateway/server/ws-connection.ts:408`
  - body: armConnectAuthTimer() clears and recreates the connect-auth timer while auth is still pending. Because the socket has no client until registration, another valid preauth connect can re-enter this path and extend the deadline indefinitely; reject duplicate connects or preserve the original auth watchdog deadline.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: The PR introduces a resettable unauthenticated connect-auth watchdog that can keep WebSocket resources open indefinitely.

Concerns:

- **[high] Resettable connect-auth watchdog:** `src/gateway/server/ws-connection.ts:408`
  - body: A second valid preauth connect frame can rearm the new watchdog while the first auth attempt is still awaiting, because no in-progress state is set before registration. This defeats the bounded-auth guarantee and can hold the socket plus preauth budget slot open.
