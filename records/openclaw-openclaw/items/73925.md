## Summary

Keep open. Current main still has the single preauth timer behavior this PR targets, but PR head 9a16c5b is not merge-ready because the new connect-auth watchdog can be reset by repeated valid preauth connect frames.

## What This Changes

The PR adds a connect-auth watchdog, moves preauth timer clearing to valid connect, updates Gateway protocol/changelog docs, and adds Gateway/client tests.

## Best Possible Solution

Repair the existing branch with a one-shot connect-auth deadline or explicit connecting state before the first auth await, reject duplicate preauth connects deterministically, and preserve the contributor credit from #49751.

## Reproduction Assessment

Yes. Send a valid connect frame whose auth resolution stalls, then send another valid preauth connect before the watchdog fires; because getClient remains null until registration, the PR re-enters the connect path and rearms the watchdog.

## Solution Assessment

No. The two-phase timer direction is the narrow maintainable fix, but this implementation is not the best solution while duplicate preauth connects can reset the auth-phase deadline.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Make the auth watchdog one-shot:** `src/gateway/server/ws-connection.ts:408`
  - body: armConnectAuthTimer() clears and recreates the connect-auth timer while auth is still pending. Because the socket has no client until registration, another valid preauth connect can re-enter this path and extend the deadline indefinitely; reject duplicate connects or keep the original auth watchdog deadline.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: The PR introduces a resettable unauthenticated connect-auth watchdog that can keep WebSocket resources open indefinitely.

Concerns:

- **[high] Resettable connect-auth watchdog:** `src/gateway/server/ws-connection.ts:408`
  - body: A second valid preauth connect frame can rearm the new watchdog while the first auth attempt is still awaiting, because no in-progress state is set before registration. This defeats the bounded-auth guarantee and can hold the socket plus preauth budget slot open.
