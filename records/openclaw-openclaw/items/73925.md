## Summary

Keep open. The PR addresses a real Gateway WebSocket auth-timing problem that current main still has, but the proposed connect-auth watchdog is resettable while auth is pending, so the patch does not yet provide the bounded unauthenticated-resource guarantee it is meant to add.

## What This Changes

The PR adds a two-phase Gateway WebSocket handshake/auth timeout, updates protocol docs and the changelog, and adds gateway tests for stalled connect auth and related client timeout behavior.

## Best Possible Solution

Repair the existing PR branch with a one-shot connect-auth watchdog or explicit connecting state set before the first auth await, reject or close duplicate preauth connect frames deterministically, preserve cleanup on close and successful registration, and add regression coverage for repeated valid connect frames during stalled auth while preserving the original contributor credit from #49751.

## Reproduction Assessment

Yes: send a valid connect frame whose auth path stalls in resolveConnectAuthState or resolveConnectAuthDecision, then send another valid preauth connect before the watchdog fires. With no in-flight state and getClient still null, the second frame re-enters the connect path and rearms the watchdog.

## Solution Assessment

No: the two-phase timer direction is the right narrow design, but this implementation is not the best fix while the auth-phase deadline is resettable. A one-shot watchdog or explicit connecting state before any auth await is the safer maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Make the auth watchdog one-shot:** `src/gateway/server/ws-connection/message-handler.ts:426`
  - body: `armConnectAuthTimer()` clears and recreates the connect-auth timer while the socket still has no client during the auth awaits. A second valid preauth `connect` frame can re-enter this block and extend the timeout indefinitely, so the new bounded-auth guarantee is bypassed; reject duplicate connects or make the watchdog one-shot before merging.
  - confidence: 0.91

## Security Review

Concerns:

  - body: The new watchdog can be rearmed by another valid `connect` frame while auth is pending because no in-progress state is set until `setClient`. An unauthenticated peer can repeatedly extend the auth deadline and keep the socket plus preauth budget slot open.
