## Summary

Keep open: the current PR head is not mergeable because duplicate valid preauth `connect` frames can rearm the new connect-auth watchdog before a client is registered, defeating the bounded-auth guarantee. Current `main` also has not landed an equivalent two-phase auth watchdog, so the useful underlying fix still needs repair rather than cleanup close.

## What This Changes

The PR moves Gateway WebSocket preauth timer clearing to a valid `connect` frame, adds a connect-auth watchdog and cleanup hooks, and updates Gateway docs, changelog, and focused tests.

## Best Possible Solution

Repair the branch with a one-shot connect-auth deadline or explicit connecting state, add duplicate-connect regression coverage, and preserve the original contributor credit when landing.

## Reproduction Assessment

Yes. PR-head source shows a valid connect whose auth stalls can be followed by another valid preauth connect before timeout; because no client is registered yet, the second path clears and re-arms the watchdog.

## Solution Assessment

No. The two-phase timer direction is the right fix boundary, but this implementation is not the best solution until duplicate preauth connects cannot extend the auth deadline.
