## Summary

Keep open. The PR still targets a real gateway handshake gap that current main has not fully fixed, but the current branch is not merge-ready because repeated valid pre-auth `connect` frames can reset the new connect-auth watchdog and keep unauthenticated sockets alive indefinitely.

## What This Changes

The PR adds a two-phase Gateway WebSocket handshake timeout by clearing the preauth timer after a valid `connect`, arming a connect-auth watchdog, adding close guards, and updating gateway docs/tests plus a changelog entry.

## Best Possible Solution

Land the two-phase handshake fix only after the connect-auth phase is one-shot or explicitly marked `connecting`, duplicate pre-client connect frames are rejected or ignored without extending the deadline, and focused regression coverage proves the original deadline is preserved.

## Reproduction Assessment

Yes, from source inspection rather than an executed test. On PR head, a second valid `connect` frame can arrive while the first auth await is pending, `getClient()` is still null, and `armConnectAuthTimer()` clears and replaces the existing watchdog.

## Solution Assessment

No. The two-phase timer is the right direction, but the current implementation must make connect-auth non-resettable or mark the handshake as connecting before the first await so duplicate valid connects cannot extend the deadline.
