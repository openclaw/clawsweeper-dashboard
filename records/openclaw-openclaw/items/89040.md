## Summary

Keep open. The performance direction is useful, but the PR still lacks real embedded_run proof and has a blocking compatibility issue: capped extra-bootstrap glob expansion can silently drop configured workspace context.

## What This Changes

The PR changes embedded-run bootstrap context loading to async workspace file reads, capped extra-bootstrap glob traversal, yielded internal hooks, bootstrap timing logs, and focused tests.

## Best Possible Solution

Keep the async/yield mitigation direction, but make capped extra-bootstrap omissions diagnosable or explicitly maintainer-approved, fix the async timing measurement, and require real embedded_run liveness proof before merge.

## Reproduction Assessment

Yes for the PR-introduced compatibility risk: the PR’s own cap test shows 140 matching files become 128 loaded files, and source shows no diagnostic for omitted matches. I did not reproduce the original 14-22 second embedded_run stall locally.

## Solution Assessment

No. The async file-read and hook-yield direction is plausible, but the hard extra-bootstrap cap is not upgrade-safe without diagnostics or explicit maintainer acceptance, and the new routing timing must await the async work it measures.

- **[P1] Report capped extra-bootstrap matches before omitting context:** `src/agents/workspace.ts:1257-1258`
  - body: The new glob cap stops adding matches once `matches.length >= 128`, and `loadExtraBootstrapFilesWithDiagnostics` only sees the already-capped set.   - confidence: 0.91

The new slow-bootstrap trace will miss the async routing work it is meant to diagnose; make this helper `async` and record after the awaited call settles.
