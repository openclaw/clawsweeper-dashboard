## Summary

Keep this PR open, but it is not merge-ready: the event-loop mitigation direction is plausible, yet the branch adds default hard caps that can drop configured model-visible bootstrap context, and the contributor proof is still unit-test-only for the reported embedded_run liveness path.

## What This Changes

The PR changes bootstrap workspace file loading, extra-bootstrap glob traversal, bootstrap hook scheduling, timing diagnostics, and related tests to reduce embedded_run bootstrap-context event-loop stalls.

## Best Possible Solution

Keep the event-loop mitigation, but preserve configured bootstrap context while yielding or batching expensive traversal, and require redacted embedded_run runtime proof before merge.

## Reproduction Assessment

Yes at source/log level: the linked issue shows a bootstrap-context liveness stall, and current main still has synchronous root bootstrap reads plus uninterrupted bootstrap hook execution. This read-only review did not run a live large-workspace embedded_run reproduction.

## Solution Assessment

No, not as currently shaped: async reads and hook yielding are reasonable, but truncating configured extra-bootstrap context by default is not the safest fix for a performance stall. The better shape is to keep configured context complete while yielding or batching the expensive work, unless maintainers explicitly accept the cap as a product change.
