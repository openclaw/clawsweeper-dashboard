## Summary

Keep open: current main and v2026.6.10 still have the implicated synchronous bootstrap read, `fs.glob`-first extra-bootstrap resolution, and sequential hook dispatch, while this PR is the active candidate fix for the linked bootstrap-context stall report. I found no discrete line-level correctness defect, but the custom glob walker and inter-handler yield remain maintainer-visible compatibility/session-state tradeoffs before merge.

## What This Changes

The PR makes embedded-run bootstrap file loading and extra-bootstrap glob resolution async/yielding, adds bootstrap substage timing, and moves hook scheduling/timing into an internal dispatch helper without changing public SDK exports.

## Best Possible Solution

Land this only after maintainer signoff on the custom glob compatibility and bootstrap hook interleaving tradeoffs; keep broader worker-thread, caching, and gateway-priority work in separate follow-ups.

## Reproduction Assessment

Yes for the narrow source path: current main and v2026.6.10 still synchronously read workspace bootstrap files, use `fs.glob` for extra-bootstrap patterns, and dispatch bootstrap hooks without an explicit yield. I did not run a full connected-channel reproduction in this read-only review.

## Solution Assessment

Yes, with maintainer signoff: async guarded reads plus a yielding full walker keep the fix in the workspace/bootstrap owner boundary and avoid public SDK changes. Worker-thread isolation, prep caching, and gateway priority scheduling are broader follow-up solutions, not required for this targeted patch.
