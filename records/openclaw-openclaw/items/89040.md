## Summary

Keep open for maintainer review: current main and the latest release still use the blocking bootstrap paths this PR targets, and the branch appears to address them, but it changes compatibility-sensitive glob resolution and shared `agent:bootstrap` timing semantics.

The PR replaces synchronous bootstrap file reads and Node `fs.glob` extra-bootstrap resolution with async/yielding workspace loading, adds `agent:bootstrap` inter-handler yielding plus timing diagnostics, and adds focused regression tests.

## Best Possible Solution
