## Best Possible Solution

Unclear as a final product choice: the PR touches the right hot path and preserves the public SDK signatures, but the custom glob walker and hook-handler interleaving are compatibility/session-state tradeoffs. If maintainers accept those tradeoffs, this is an acceptable mitigation; otherwise the safer solution is a narrower event-loop fix that preserves shipped semantics.
