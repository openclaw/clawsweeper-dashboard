## Summary

Keep open: current main still has the synchronous bootstrap file read and Node `fs.glob` path, while this PR contains useful, current work for the reported embedded-run stall. I found no discrete line-level blocker, but merge should remain human-gated because the fix replaces glob semantics and changes `agent:bootstrap` handler interleaving.

## What This Changes

The PR makes workspace bootstrap file loading asynchronous/yielding, replaces extra-bootstrap glob resolution with an OpenClaw walker, adds hook-yield/timing diagnostics, and expands focused regression coverage.

## Best Possible Solution

Land only after a maintainer explicitly accepts the glob-parity and hook-interleaving tradeoffs, with focused tests green and optional full connected `embedded_run` proof if maintainers want end-to-end message-loss validation.

## Reproduction Assessment

Yes, source-reproducible: current main still uses `readFileSync`/`closeSync` and `fs.glob` in the bootstrap path, and the linked issue provides event-loop delay logs for `bootstrap-context`. I did not run a full connected `embedded_run` reproduction in this read-only review.

## Solution Assessment

Acceptable mitigation, not an automatic merge: moving fd reads and glob walking to yielding async paths is in the right owner area, and worker-thread offload would be broader. The remaining question is maintainer acceptance of glob compatibility and hook scheduling semantics.
