## What This Changes

The PR changes embedded-run bootstrap-context loading to async/yielding workspace reads, a custom yielding extra-bootstrap glob walker, internal bootstrap-hook scheduling, and bootstrap substage timing.

## Best Possible Solution

Land this or an equivalent focused mitigation only after explicit maintainer acceptance of the custom glob resolver and bootstrap-hook interleaving tradeoffs, then close the linked issue with merge and release provenance.

## Reproduction Assessment

Source-reproducible: current main still performs sync fd reads and fs.glob-backed extra bootstrap resolution on the embedded-run bootstrap path, and the linked issue provides production timing logs. This review did not run a fresh live channel repro.

## Solution Assessment

Unclear as a best fix: this is a focused mitigation with good tests, but replacing fs.glob and interleaving bootstrap hooks are maintainer-level tradeoffs. A worker or runtime-isolation design would avoid some compatibility risk but is broader.
