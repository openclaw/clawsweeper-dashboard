The PR makes embedded-run bootstrap-context loading more event-loop friendly by using async workspace file reads, a yielding extra-bootstrap glob walker, yielded bootstrap hook dispatch, substage timing logs, and focused regression tests.

## Best Possible Solution

Keep the async/yielding bootstrap fix, preserve fs.glob-compatible match and no-match semantics, and let maintainers accept the remaining custom-walker availability tradeoff after exact-head checks.

## Solution Assessment

Mostly yes. Moving the implicated file reads, glob expansion, and bootstrap hook chain to async or yielding paths is a narrow maintainable fix, but the custom walker should preserve fs.glob-compatible no-match behavior before merge.
