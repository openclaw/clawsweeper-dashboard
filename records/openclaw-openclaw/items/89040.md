## Summary

Keep open: the PR targets a real P1 bootstrap-context stall, but it is not merge-ready because it hard-caps configured extra bootstrap glob matches at 128, the supported Node `fs.glob` branch still cannot bound traversal by visited entries, and the proof remains focused Docker Vitest output rather than a live patched `embedded_run` liveness run.

## What This Changes

The PR changes bootstrap-context loading to async/parallel file reads, adds extra-bootstrap glob caps/diagnostics and ignored dirs, adds hook handler yielding/timing, logs bootstrap substages, and adds focused workspace/hook tests.

## Best Possible Solution

Keep the async read and hook-yield direction, but preserve configured bootstrap context by default, use a traversal-bounded glob implementation on supported Node, and require redacted live `embedded_run` timing/liveness proof before merge.

## Reproduction Assessment

No, I do not have a high-confidence live reproduction from current main in this read-only review. The linked report has concrete liveness and prep-stage logs, and source inspection confirms synchronous bootstrap reads plus unbounded glob/hook paths, but the patched real `embedded_run` path was not exercised.

## Solution Assessment

No, this is not yet the best fix. Async file reads and hook yielding are useful, but the hard match cap and yielded-match counting in the Node glob branch are weaker than a traversal-bounded implementation that preserves configured context by default.
