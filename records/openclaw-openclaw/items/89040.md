## Summary

Keep open: current main still has the bootstrap-context stall path and this PR is the active candidate fix, but the branch is not merge-ready because it expands SDK-facing API surface and changes no-match glob behavior while still requiring maintainer acceptance of hook interleaving.

## What This Changes

The PR makes embedded-run bootstrap file loading async/yielding, replaces extra-bootstrap glob resolution with a cooperative walker, adds bootstrap timing diagnostics, and yields between bootstrap hook handlers.

## Best Possible Solution

Keep the async/yielding bootstrap repair, privatize scheduling and timing plumbing so the SDK surface stays stable, preserve fs.glob no-match parity, and land only after maintainers accept the hook interleaving tradeoff.
