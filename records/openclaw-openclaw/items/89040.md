## Summary

- **[P1] Bound the active Node glob path by visited entries:** `src/agents/workspace.ts:1248-1249`
  - body: On supported Node 24 this branch is the main path, but the new counters only run when `fs.glob` yields a match. A sparse pattern like `**/AGENTS.md` with one late match can still let Node traverse a huge tree internally without hitting the local visit limit or yield interval, so the reported event-loop stall remains possible. Please route this path through the traversal-counted walker or add an equivalent active-path bound plus a sparse late-match regression test.
  - confidence: 0.9

After upgrade those files can silently disappear from the model-visible bootstrap context, so keep existing context unless the cap is opt-in or make truncation visible and documented with upgrade coverage.
