## Summary

This PR contains useful async-read and timing work, but it replaces the existing asynchronous Node glob implementation without proving that implementation causes the reported stall, and the supplied proof validates an intermediate branch cutoff regression rather than the original current-main embedded-run failure.

## What This Changes

The PR makes guarded bootstrap reads asynchronous, replaces Node glob enumeration with a custom yielding walker, yields between bootstrap hook handlers, adds bootstrap timing diagnostics, and expands regression tests.

## Best Possible Solution

Retain the asynchronous guarded reads and diagnostic timing that are independently justified, profile current main in the reported embedded-run scenario, and change glob or hook scheduling only where that profile demonstrates a blocker while preserving existing contracts.

No, not for the complete reported issue. The relevant blocking-read source path and a synthetic helper liveness scenario are clear, but the supplied evidence does not reproduce the 14–22 second current-main embedded-run stall or identify its responsible substage.

## Solution Assessment
