## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Report capped extra-bootstrap matches before omitting context:** `src/agents/workspace.ts:1257-1258`
  - body: This new hard cap stops after 128 matched extra-bootstrap files, but `bootstrap-extra-files` is documented as loading configured workspace paths and the hook only logs loader diagnostics. Existing monorepos with more than 128 configured matches will silently lose later AGENTS.md/TOOLS.md context after upgrade, changing session instructions without a warning. Preserve the existing configured context or return a visible cap diagnostic before dropping files.
  - confidence: 0.91
- **[P2] Count walked entries before claiming a glob visit cap:** `src/agents/workspace.ts:1244-1249`
  - body: On Node versions with `fs.glob`, this counter increments only for yielded matches, not for every entry the glob implementation traverses. A large workspace with few or no `**/AGENTS.md` matches can still make `fs.glob` walk the whole tree before this counter reaches 20,000, leaving the unbounded traversal case that this PR is meant to fix. Use the bounded local walker or another path that can count actual visited entries.
  - confidence: 0.84
