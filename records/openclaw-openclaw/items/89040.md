## Summary

Keep this PR open: the central performance fix is still useful and not already on main, but the latest diff still has a glob parity regression and exposes hook scheduling controls through a public SDK subpath, with an additional maintainer decision needed on bootstrap hook interleaving.

## What This Changes

The PR makes workspace bootstrap file reads async, replaces extra-bootstrap glob resolution with a yielding walker, adds bootstrap/hook timing diagnostics, and expands focused tests.

## Best Possible Solution

Land a narrowed async/yielding bootstrap loader that preserves fs.glob empty-match semantics, keeps scheduling controls out of the public SDK unless explicitly approved, and has maintainer acceptance for bootstrap hook interleaving.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source and PR proof cover the loader/event-loop liveness path, and current main clearly uses sync bootstrap reads plus fs.promises.glob, but I did not reproduce the full connected channel message-loss symptom end to end.

## Solution Assessment

Is this the best way to solve the issue? No, not as-is: async/yielding bootstrap loading is the right boundary, but the branch should preserve empty-glob parity and settle the public hook API/session interleaving choices before merge.
