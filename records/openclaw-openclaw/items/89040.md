## Summary

Keep this PR open: it is aimed at a real P1 bootstrap-context stall, but the current patch still leaves the supported Node `fs.glob` path unbounded by walked entries, can truncate configured bootstrap context without a user-visible warning, and lacks real `embedded_run` timing/liveness proof.

## What This Changes

The PR changes workspace bootstrap loading to async root-file reads, adds extra-bootstrap glob filtering/caps, yields between bootstrap hook handlers, logs bootstrap-context substage timings, and adds focused workspace and hook tests.

## Best Possible Solution

Use one traversal-counted, ignored-dir-aware async walker for extra bootstrap globs, preserve or visibly warn on any configured-context truncation, and land only with redacted `embedded_run` timing/liveness proof.

## Reproduction Assessment

No live `embedded_run` reproduction was established in this review; the patch-level issues are source-reproducible from the active Node `fs.glob` branch and the debug-only diagnostic path.

## Solution Assessment

No. The PR is a plausible mitigation, but the best fix needs a real traversal bound on the active path, compatibility-safe truncation behavior, and live liveness proof.
