## Summary

This PR should stay open: current main still uses the blocking `fs.glob` path and synchronous fd reads, while the latest PR head preserves the user-configured broad-glob match set and adds focused regression coverage. I found no discrete patch bug, but merge still needs maintainer acceptance of the custom glob walker and `agent:bootstrap` scheduling tradeoff.

## What This Changes

The PR makes workspace bootstrap file loading async/yielding, replaces extra-bootstrap `fs.glob` resolution with a yielding walker, adds bootstrap/hook timing diagnostics, and expands focused tests.

## Best Possible Solution

Land after an agents/bootstrap owner accepts the glob compatibility and hook-scheduling tradeoffs, with connected `embedded_run` proof only if maintainers require end-to-end channel validation.

## Reproduction Assessment

No full high-confidence connected `embedded_run` reproduction was run in this review. The linked issue logs and PR terminal proof make the loader/liveness path source-reproducible, but channel message loss is not end-to-end reproduced here.

## Solution Assessment

Unclear whether this is the single best fix, but it is an acceptable mitigation if maintainers accept the custom glob and hook-scheduling tradeoffs. A safer alternative would need to preserve `fs.glob` parity while moving the blocking traversal off the event loop or otherwise yielding without changing hook timing.
