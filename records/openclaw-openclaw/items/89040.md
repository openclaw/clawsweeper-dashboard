## Summary

Keep open. The performance direction is useful, but the current head still drops configured extra-bootstrap matches after 20,000 scanned entries, and the proof remains Docker/unit-test output rather than a patched `embedded_run` liveness run.

## What This Changes

This PR converts bootstrap workspace reads to async fd reads, uses a yielding bounded walker for extra-bootstrap globs, yields/times bootstrap hooks, logs bootstrap-context substages, and adds focused tests.

## Best Possible Solution

Keep the event-loop-yielding approach, preserve configured extra-bootstrap context by default, and require redacted patched embedded_run timing/liveness proof before merge.

## Reproduction Assessment

No live high-confidence reproduction was run in this read-only review. The related issue has concrete liveness logs, and source inspection shows current main still has sync root bootstrap reads plus sequential awaited hook execution in the bootstrap-context path.

## Solution Assessment

No. Async reads and hook yielding are reasonable, but hard truncation of documented extra-bootstrap globs is not the safest fix for a liveness stall; preserving configured context while yielding or batching traversal is the better default unless maintainers explicitly accept the cap.
