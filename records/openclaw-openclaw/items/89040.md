## Summary

Keep open: the patch appears correct and has sufficient focused loader proof, but it still needs maintainer acceptance because replacing `fs.glob` behavior and yielding between bootstrap hook handlers are compatibility and session-timing risks.

## What This Changes

The PR makes embedded-run bootstrap context loading async/yielding, replaces extra-bootstrap glob resolution with a yielding walker, adds internal hook-dispatch scheduling/timing, and expands bootstrap tests.

## Best Possible Solution

Land this only after maintainer signoff on the compatibility and bootstrap scheduling tradeoffs; keep broader worker-thread, caching, and gateway-priority work in separate follow-ups.

## Reproduction Assessment

## Solution Assessment

Yes, with maintainer signoff: async guarded reads plus a yielding full walker keep the fix in the workspace/bootstrap owner boundary and avoid public SDK changes. Worker-thread isolation, prep caching, and gateway priority scheduling are broader follow-up solutions, not required for this targeted patch.
