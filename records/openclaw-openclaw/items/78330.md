## Summary

Keep open: current main does not expose an image-generation provider inventory over Gateway RPC, and the request is a plausible additive Gateway/API feature rather than a stale, duplicate, or ClawHub-only item.

## What This Changes

Request a read-only `image.providers` Gateway RPC returning image-generation provider metadata for external control UIs without spawning the CLI.

## Best Possible Solution

If accepted, add an additive typed Gateway method that reuses the existing image-provider listing data, registers under `operator.read`, omits secrets, and includes protocol/docs/tests plus client/UI follow-through as needed.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms current main has comparable `models.list` and `tts.providers` Gateway methods, but no `image.providers` handler, method-list entry, read-scope entry, protocol schema, or docs entry.

## Solution Assessment

Unclear until maintainers approve the Gateway API shape. The narrow maintainable path is to reuse the existing provider-list metadata behind a read-only Gateway handler, not to route dashboards through CLI process spawning.
