Keep open: current main still lacks the requested Gateway image-provider inventory RPC, while the local CLI and `image_generate` list path already expose the underlying provider data. This is a concrete additive Gateway protocol feature, but it needs maintainer API approval before implementation.

## What This Changes

The issue requests a read-only `image.providers` Gateway RPC returning secret-free image-generation provider metadata for external control UIs.

## Best Possible Solution

Add an additive, typed `operator.read` Gateway RPC that reuses the image-generation provider registry/list helpers and returns only derived, secret-free inventory metadata, with protocol docs and focused Gateway tests updated together.
