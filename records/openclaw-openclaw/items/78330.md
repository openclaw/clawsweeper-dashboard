## Summary

Keep open: current main and the latest release still expose `models.list` and `tts.providers` over the Gateway, but no equivalent image-generation provider inventory RPC is registered, documented, or schema-backed.

## What This Changes

Request a read-only Gateway RPC for image-generation provider metadata so control UIs can avoid spawning `openclaw infer image providers --json`.

## Best Possible Solution

Add an additive `operator.read` image provider inventory RPC that reuses the existing image-generation provider registry and returns only derived provider metadata with protocol, docs, and focused Gateway tests updated together.

## Reproduction Assessment

Yes. Source inspection on current main shows `models.list` and `tts.providers` registered as read-scope Gateway methods, while no `image.providers` descriptor, handler, protocol schema, or docs entry exists.

## Solution Assessment

Yes. An additive read-only Gateway method is the narrow maintainable solution because the canonical provider registry and CLI/tool list data already exist; the fix should not add config, expose credentials, or create a parallel registry.
