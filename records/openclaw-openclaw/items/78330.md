## Summary

Keep open: this is still a real feature/API gap. Current main has image-provider inventory through the local infer CLI and `image_generate action="list"`, but no typed Gateway RPC, read-scope entry, advertised method, handler, protocol schema, or Gateway docs for a read-only image-provider catalog.

## What This Changes

Request to add a read-only Gateway RPC such as `image.providers` that returns secret-free image-generation provider metadata for external control UIs.

## Best Possible Solution

Add an additive typed Gateway method under `operator.read` that reuses the existing image-generation provider registry and returns a documented, secret-free provider catalog with protocol schema, docs, tests, and client follow-through.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high-confidence absence evidence because current main has CLI/tool image-provider listing but no Gateway/protocol/docs `image.providers` surface.

## Solution Assessment

Unclear until maintainers approve the public API shape. An additive `operator.read` Gateway method appears aligned with `tts.providers`, but the exact schema and privacy boundary are product/API decisions.
