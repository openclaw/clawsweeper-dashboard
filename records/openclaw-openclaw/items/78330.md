## Summary

Keep open: current main still has local CLI and `image_generate action=list` provider inventory, but no read-only Gateway RPC, protocol schema, read-scope entry, advertised method, SDK helper, or UI surface for image-generation providers.

## What This Changes

Request a read-only Gateway RPC such as `image.providers` that returns secret-free image-generation provider metadata for external control UIs.

## Best Possible Solution

Add a typed, additive Gateway method under `operator.read` that reuses the existing image-generation provider registry and returns a secret-free provider catalog, with protocol schema, docs, focused Gateway tests, and changelog coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high-confidence absence evidence because current main has CLI/tool image provider listings but no Gateway/protocol/UI/SDK `image.providers` surface.

## Solution Assessment

Unclear pending maintainer API approval. An additive read-only Gateway method appears aligned with `tts.providers`, but the exact response contract and SDK/UI follow-through are public protocol decisions.
