## Summary

Keep open: current main still has image provider inventory through the local infer CLI and `image_generate` tool list action, but no read-only Gateway RPC, scope entry, advertised method, or protocol documentation for `image.providers`. This is a valid additive Gateway feature request, not a broken existing contract.

## What This Changes

Request to add a read-only `image.providers` Gateway RPC returning secret-free image-generation provider metadata for external control UIs.

## Best Possible Solution

Add a typed additive Gateway method under `operator.read` that reuses the image-generation provider registry/list helpers and returns secret-free provider metadata, with protocol schema, docs, SDK/client follow-through, and focused tests kept in sync.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high-confidence absence evidence because CLI/tool provider listing exists, but the Gateway method/scope/protocol surfaces do not expose it.

## Solution Assessment
