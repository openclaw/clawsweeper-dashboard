## Summary

Keep open: current main still has no read-only gateway RPC for image-generation provider inventory, while the existing image provider registry and CLI/tool list paths make this a bounded additive gateway feature.

## What This Changes

The issue requests a lightweight `image.providers` gateway RPC that returns registered image-generation provider metadata without credentials so control UIs do not need to spawn `openclaw infer image providers --json`.

## Best Possible Solution

Add an additive `operator.read` `image.providers` gateway RPC that reuses the existing image-generation provider registry, returns only derived/provider metadata, and updates protocol schemas, docs, SDK/client coverage if needed, and gateway tests together.

## Reproduction Assessment

Yes. Source inspection on current main shows `models.list` and `tts.providers` are registered gateway methods, while no `image.providers` protocol, descriptor, handler, SDK, or docs surface exists.

## Solution Assessment

Yes, an additive read-only gateway method is the narrow maintainable fix because the provider registry and CLI/tool list data already exist; it should not introduce config, credentials exposure, or a parallel provider registry.
