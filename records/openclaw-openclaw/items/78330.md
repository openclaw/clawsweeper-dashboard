Keep open: current main still lacks a read-only Gateway RPC for image-generation provider inventory. The request is concrete and source-backed, but it adds a public Gateway protocol/API surface, so maintainers should approve the method name and response contract before implementation.

## Best Possible Solution

Add an additive, typed operator.read Gateway method that reuses the existing image-generation provider registry and returns only derived, secret-free metadata with protocol docs and focused Gateway tests.
