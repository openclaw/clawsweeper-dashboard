## Summary

Keep open: current main and the latest release still lack the public Discord message `fetch`/`get` action, and an open same-author implementation PR explicitly closes this issue, so the issue should remain open until that PR lands or is closed.

## What This Changes

Request a Discord message-tool action that retrieves one specific message by `channelId` plus `messageId`, or from a Discord message URL.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81243, or a narrow equivalent, so the existing Discord fetch backend is exposed through the public message tool with direct-ID and URL coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows the internal Discord `fetchMessage` runtime exists, but the public action list, Discord discovery, and public handler do not expose `action: "fetch"`, so the requested call cannot reach that backend.

## Solution Assessment

Yes. The narrow maintainable solution is to expose the existing Discord `fetchMessageDiscord`/`Routes.channelMessage` path through the public message tool while preserving the existing messages gate; the linked PR is the active implementation path.
