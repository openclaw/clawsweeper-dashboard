## Summary

Keep open: current main and the latest release still lack the public Discord message `fetch`/`get` action, and the same-author open implementation PR explicitly closes this issue, so this issue should stay open until that PR lands or is closed.

## What This Changes

Request to expose a Discord message `fetch`/`get` action that retrieves one specific message by `channelId` plus `messageId` or by a Discord message URL.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81243, or a narrow equivalent, so the existing Discord fetch backend is exposed through the public message tool with direct-ID and URL coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows the internal Discord `fetchMessage` backend exists, but the shared action list, Discord discovery, and public Discord handler omit `fetch`/`get`, so the requested public action still falls through as unsupported.

## Solution Assessment

Yes. Reusing the existing `fetchMessageDiscord`/`Routes.channelMessage` backend and adding only the public action, schema, discovery, target normalization, URL alias, and tests is the narrow maintainable solution.
