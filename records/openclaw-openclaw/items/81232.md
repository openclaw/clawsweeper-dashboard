## Summary

Keep open: current main and the latest release still do not expose a public Discord `message` `fetch` action, and the open closing implementation at https://github.com/openclaw/openclaw/pull/81243 is unmerged and currently conflicting.

## What This Changes

Request to add a Discord message-tool `fetch` action for one message by `channelId` plus `messageId` or by Discord message URL.

## Best Possible Solution

Review, rebase, and land or replace https://github.com/openclaw/openclaw/pull/81243 after maintainers accept the public `fetch` and URL-alias contract, then close this issue once the implementation reaches main.

## Reproduction Assessment

Yes, at source level. Current main exposes an internal Discord `fetchMessage` helper, but the shared action list, Discord discovery, and public handler do not expose `action: "fetch"`, so a public fetch call falls through to the unsupported-action path.

## Solution Assessment

Yes, the best technical direction is to reuse the existing Discord `fetchMessage` runtime and wire it into the public message-tool surface. The remaining question is maintainer acceptance of the shared `fetch` action and Discord URL alias, not a missing lower-level Discord helper.
