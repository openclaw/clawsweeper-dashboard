Request to expose a Discord message-tool fetch/get action that returns one message by channelId plus messageId or by Discord message URL.

## Best Possible Solution

Land one canonical PR that maps public Discord fetch to the existing fetchMessage runtime, supports direct IDs and URL-only fetches, preserves messages gating/gateway execution, updates schema/docs/tests, then close this issue.

## Solution Assessment

Yes in direction, but not solved on main. The maintainable path is to expose the existing Discord fetch runtime through one reviewed public message action rather than adding a separate REST implementation.
