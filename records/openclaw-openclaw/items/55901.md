## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Negotiate batch before enabling multiline:** `extensions/irc/src/client.ts:333`
  - body: The IRCv3 multiline spec depends on the `batch` capability, but this branch only requests `draft/multiline` before emitting `BATCH` frames. Servers that enforce capability dependencies can reject or ignore the flow, so request and require ACKs for both capabilities before setting `multilineCap`.
  - confidence: 0.9
- **[P2] Split long multiline PRIVMSG payloads:** `extensions/irc/src/client.ts:249`
  - body: Each newline-delimited line is written as one tagged `PRIVMSG`. Long code or URL lines can still exceed IRC's 512-byte frame limit and be truncated or rejected; split each logical line into safe frames and use `draft/multiline-concat` when preserving a single original line.
  - confidence: 0.92
- **[P3] Scope LF preservation to message payloads:** `extensions/irc/src/protocol.ts:147-148`
  - body: `sanitizeIrcOutboundText` now preserves standalone LF for every caller, including NickServ, QUIT, and USER arguments that later pass through `sendRaw`. `sendRaw` removes LF entirely, so values like `secret\nJOIN` collapse into `secretJOIN`; keep newline preservation in the message-send path or flatten LF at non-message call sites.
  - confidence: 0.82
- **[P3] Add the IRC changelog entry:** `extensions/irc/src/channel.ts:339`
  - body: This changes user-visible IRC delivery behavior but does not update `CHANGELOG.md`. Add a single-line entry in the active section before landing.
  - confidence: 0.84
