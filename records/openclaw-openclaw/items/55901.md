## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Split long multiline lines before writing PRIVMSG frames:** `extensions/irc/src/client.ts:249`
  - body: The multiline send path writes each newline-delimited markdown line as one tagged `PRIVMSG`. A long code line or URL can exceed IRC's per-frame limit and be truncated or rejected, so split long batch payloads into IRC-safe frames before writing them.
  - confidence: 0.9
- **[P2] Negotiate batch before enabling multiline batches:** `extensions/irc/src/client.ts:333`
  - body: The IRCv3 multiline spec depends on the `batch` capability, but this branch enables multiline sends after only `draft/multiline` is ACKed. Request and confirm `batch` before sending BATCH frames.
  - confidence: 0.87
- **[P2] Enforce advertised multiline limits:** `extensions/irc/src/client.ts:332-333`
  - body: The CAP parser only checks whether the capability string contains `draft/multiline`, then the channel limit is hard-coded to 16384. Parse and enforce the advertised `max-bytes` and `max-lines` values so servers do not reject oversized batches.
  - confidence: 0.86
- **[P2] Honor concat tags when rebuilding batches:** `extensions/irc/src/client.ts:462`
  - body: Incoming multiline batches are always joined with `\n`, but `draft/multiline-concat` means the next fragment should be joined directly. Preserve concat semantics when reconstructing the combined text.
  - confidence: 0.84
- **[P3] Keep command sanitization single-line:** `extensions/irc/src/protocol.ts:147-148`
  - body: `sanitizeIrcOutboundText` now preserves standalone LF for messages, but the same helper still feeds NickServ, USER, and QUIT command fields that pass through `sendRaw`. Use a message-specific path or normalize LF for command fields so escaped newlines do not collapse tokens.
  - confidence: 0.78
