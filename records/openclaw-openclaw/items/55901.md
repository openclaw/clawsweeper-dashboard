## Summary

Keep open: current main still lacks IRC draft/multiline support, and the PR is a plausible IRC-plugin feature, but the head patch still misses required IRCv3 negotiation and limit handling that can reject or truncate multiline sends.

## What This Changes

The PR adds IRCv3 tag parsing, draft/multiline CAP negotiation, outbound and inbound BATCH handling, LF-preserving IRC sanitization, and newline-preserving chunking in the bundled IRC plugin.

## Best Possible Solution

Land a rebased IRC plugin change that negotiates batch plus draft/multiline, parses advertised limits, emits byte-safe draft/multiline-concat frames, scopes LF preservation to message payloads, and adds focused IRC tests.

## Reproduction Assessment

Yes. The current-main gap is source-reproducible because IRC output sanitization flattens newlines and sends plain PRIVMSG frames; the PR blockers are source/spec reproducible from the head code and the IRCv3 multiline contract.

## Solution Assessment

No, not as-is. The feature belongs in the IRC plugin, but this patch needs required batch negotiation, advertised-limit handling, byte-safe line splitting, and message-scoped LF preservation before it is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Negotiate batch before using draft/multiline:** `extensions/irc/src/client.ts:333`
  - body: The IRCv3 multiline spec depends on the batch capability, but this branch only requests draft/multiline before sending BATCH frames. Request and require ACKs for both capabilities before enabling multiline sends.
  - confidence: 0.91
- **[P2] Honor advertised multiline limits:** `extensions/irc/src/client.ts:332-333`
  - body: The draft/multiline capability value carries required max-bytes and optional max-lines limits, but this branch only checks for the capability name and then uses a hard-coded 16384 chunk limit. Parse the advertised limits and keep emitted batches within them so servers do not reject valid-looking sends.
  - confidence: 0.86
- **[P2] Split long multiline payload lines:** `extensions/irc/src/client.ts:249`
  - body: When multiline is active, each logical line is written as one tagged PRIVMSG. Long markdown code lines or URLs can exceed IRC's per-frame limit, so split long logical lines into safe frames and use draft/multiline-concat where the original line must be preserved.
  - confidence: 0.93
- **[P3] Scope LF preservation to message bodies:** `extensions/irc/src/protocol.ts:147-148`
  - body: sanitizeIrcOutboundText now preserves standalone LF, but NickServ, QUIT, and USER construction also use this helper before sendRaw strips newlines. Keep the generic sanitizer flattening CR/LF, or add a message-specific sanitizer for multiline payloads, so non-message command values do not concatenate tokens.
  - confidence: 0.82
