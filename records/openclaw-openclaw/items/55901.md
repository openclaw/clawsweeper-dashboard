Overall confidence: 0.9

Full review comments:

- **[P2] Negotiate batch before using draft/multiline:** `extensions/irc/src/client.ts:333`
  - body: The multiline spec depends on the `batch` capability, but this branch requests only `draft/multiline` before emitting `BATCH` and `@batch` frames. Servers that enforce CAP dependencies can reject or ignore the flow; request and require ACKs for both capabilities before enabling multiline sends.
  - confidence: 0.9
- **[P2] Split long multiline lines before writing PRIVMSG:** `extensions/irc/src/client.ts:249`
  - body: When multiline is active, each newline-delimited payload is written as one tagged `PRIVMSG`. A long markdown code line or URL can exceed IRC's per-frame limit and be truncated or rejected; split long logical lines into safe frames, using `draft/multiline-concat` where needed.
  - confidence: 0.92
- **[P3] Keep LF preservation message-scoped:** `extensions/irc/src/protocol.ts:147-148`
  - body: `sanitizeIrcOutboundText` now preserves standalone LF, but this helper still feeds NickServ, QUIT, and USER command arguments before `sendRaw` removes newlines entirely. Inputs with LF can concatenate tokens instead of the old space-flattened value, so keep newline preservation message-specific or flatten LF at non-message call sites.
  - confidence: 0.82
- **[P3] Add the IRC changelog entry:** `extensions/irc/src/channel.ts:339`
  - body: This is a user-facing IRC feature, but the diff does not update `CHANGELOG.md`. Add a single-line entry in the active section with contributor credit before landing.
  - confidence: 0.86
