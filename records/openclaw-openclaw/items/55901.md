Full review comments:

- **[P1] Split long multiline payloads before writing PRIVMSG:** `extensions/irc/src/client.ts:249`
  - body: With `textChunkLimit` raised to 16384, a response chunk can contain a single long markdown/code line. This branch writes that whole line as one tagged `PRIVMSG`, which still has to fit IRC's roughly 512-byte message frame; compliant servers can truncate or reject it. Split long logical lines into IRC-safe frames and use `draft/multiline-concat` for continuations before closing the batch. ([ircv3.net](https://ircv3.net/specs/extensions/multiline))
  - confidence: 0.94
- **[P2] Keep LF normalization out of generic IRC command sanitizing:** `extensions/irc/src/protocol.ts:147`
  - body: `sanitizeIrcOutboundText` is still used for NickServ, USER, QUIT, and other raw command payloads, but this change preserves standalone `\n` and leaves `sendRaw` to remove it entirely. That regresses current behavior where newline separators become spaces, so multiline message bodies need a separate sanitizer instead of changing the generic command sanitizer.
  - confidence: 0.86
