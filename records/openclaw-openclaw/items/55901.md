## Summary

Keep open: current main still lacks IRCv3 multiline support, and the PR has useful real-world proof, but the current head still has protocol-level blockers around capability negotiation, advertised limits, and byte-safe multiline framing.

## What This Changes

The PR adds IRCv3 draft/multiline CAP negotiation, IRCv3 tag parsing, outbound and inbound BATCH handling, LF-preserving IRC sanitization, and a newline-preserving IRC chunker.

## Best Possible Solution

Land a rebased IRC-plugin implementation that negotiates batch plus draft/multiline, parses advertised limits, emits byte-safe multiline frames with concat tags, and adds focused IRC tests.

## Reproduction Assessment

Yes. Current main flattens IRC newlines and sends plain PRIVMSG frames, while the PR blockers are source/spec reproducible from the head send path and the IRCv3 contracts.

## Solution Assessment

No, not as-is. The feature belongs in the IRC plugin, but this patch needs required batch negotiation, advertised-limit handling, byte-safe line splitting, and message-scoped LF preservation before it is maintainable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Negotiate batch before enabling multiline:** `extensions/irc/src/client.ts:333`
  - body: The IRCv3 multiline spec depends on the `batch` capability, but this branch only requests `draft/multiline` before sending BATCH frames. Request and require ACKs for both capabilities before enabling multiline sends.
  - confidence: 0.91
- **[P2] Honor advertised multiline limits:** `extensions/irc/src/client.ts:332-333`
  - body: The `draft/multiline` capability value carries required `max-bytes` and optional `max-lines` limits, but this branch only checks for the capability name and uses a hard-coded 16384 chunk limit. Parse the advertised limits and keep emitted batches within them.
  - confidence: 0.86
- **[P2] Split long multiline payload lines:** `extensions/irc/src/client.ts:249`
  - body: When multiline is active, each logical line is written as one tagged PRIVMSG. Long markdown code lines or URLs can exceed IRC's per-frame limit, so split long logical lines into safe frames and use `draft/multiline-concat` where the original line must be preserved.
  - confidence: 0.93
- **[P3] Scope LF preservation to message bodies:** `extensions/irc/src/protocol.ts:147-148`
  - body: `sanitizeIrcOutboundText` now preserves standalone LF, but NickServ, QUIT, and USER construction also use this helper before `sendRaw` strips newlines. Keep the generic sanitizer flattening CR/LF, or add a message-specific sanitizer for multiline payloads.
  - confidence: 0.82
