## Summary

Land a refreshed IRC-owned implementation that keeps the draft/multiline direction, preserves existing command sanitization, splits every outgoing IRC frame within the transport limit, uses concat semantics for long logical multiline lines, and adds focused regression tests.

## Review Findings

Full review comments:

- **[P1] Split long multiline lines before writing PRIVMSG:** `extensions/irc/src/client.ts:249`
  - body: With `textChunkLimit` raised to 16384, a response chunk can contain a single long markdown/code line. This branch writes that whole line as one tagged `PRIVMSG`, which still has to fit IRC's per-frame line limit, so compliant servers can truncate or reject it; split long logical lines into safe frames and preserve continuation semantics before closing the batch.
  - confidence: 0.94
- **[P2] Keep LF preservation out of command sanitizing:** `extensions/irc/src/protocol.ts:147-148`
  - body: `sanitizeIrcOutboundText` is still used for NickServ, USER, QUIT, and other raw command payloads, but this change preserves standalone `\n` and leaves `sendRaw` to remove it entirely. That regresses current behavior where newline separators become spaces, so multiline message bodies need a separate sanitizer instead of changing the generic command sanitizer.
  - confidence: 0.87
