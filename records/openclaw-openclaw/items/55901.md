## Summary

Keep open. Current main still flattens IRC newlines and has no IRCv3 `draft/multiline`/`BATCH` support, while this PR is a plausible plugin-local feature but still has protocol correctness blockers before merge.

## What This Changes

The PR adds IRCv3 `draft/multiline` CAP negotiation, BATCH send/receive handling, a newline-preserving IRC chunker, IRCv3 tag parsing, and LF-preserving sanitizer behavior for IRC output.

## Best Possible Solution

Land a rebased IRC plugin change that negotiates both `batch` and `draft/multiline`, emits byte-safe multiline frames, keeps LF preservation message-scoped, and adds focused IRC tests.

## Reproduction Assessment

Yes. Current main's feature gap is source-reproducible because IRC outbound text is sanitized into spaces and sent as plain `PRIVMSG`, and the remaining PR blockers are visible in the head source plus the IRCv3 multiline spec.

## Solution Assessment

No, not as-is. `draft/multiline` is the right IRC-plugin direction, but this patch needs `batch` negotiation, per-frame splitting, and message-scoped LF handling before it is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Negotiate batch before using draft/multiline:** `extensions/irc/src/client.ts:333`
  - body: The multiline spec depends on the `batch` capability, but this branch requests only `draft/multiline` and then sends `BATCH` frames. Servers that enforce capability dependencies can reject or ignore that flow, so request and require ACKs for both capabilities before enabling multiline sends.
  - confidence: 0.9
- **[P2] Split long multiline payload lines:** `extensions/irc/src/client.ts:249`
  - body: When multiline is active, each newline-delimited payload is written as one tagged `PRIVMSG`. A long markdown code line or URL can exceed IRC's per-frame limit and be truncated or rejected; split long logical lines into safe frames, using `draft/multiline-concat` where needed to preserve the original line.
  - confidence: 0.92
- **[P3] Scope LF preservation to message bodies:** `extensions/irc/src/protocol.ts:147-148`
  - body: `sanitizeIrcOutboundText` now preserves standalone LF, but this helper also feeds NickServ, QUIT, and USER command arguments before `sendRaw` strips newlines. Inputs with LF now concatenate tokens instead of the old space-flattened value, so keep LF preservation message-specific or flatten LF at non-message call sites.
  - confidence: 0.8
