## Summary

Keep open: current main still lacks IRC multiline support, but this PR is not merge-ready because the latest head misses required capability negotiation, can still emit oversized multiline frames, and changes generic IRC text sanitization in a way that affects non-message commands.

## What This Changes

The PR adds an IRC newline-preserving chunker, IRCv3 tag parsing, draft/multiline CAP negotiation, BATCH-based incoming/outgoing multiline handling, and a larger IRC outbound chunk target.

## Best Possible Solution

Land a rebased IRC plugin change that negotiates both `batch` and `draft/multiline`, emits byte-safe multiline frames with `draft/multiline-concat` where needed, scopes newline preservation to message bodies, and adds focused tests plus a changelog entry.

## Reproduction Assessment

Yes. Current main's feature gap is source-reproducible: IRC outbound text is newline-flattened and sent as plain `PRIVMSG`, while the PR regressions are visible in the head diff's CAP negotiation, multiline send path, and sanitizer call sites.

## Solution Assessment

No, not as-is. `draft/multiline` is the right IRC-plugin-local direction, but this implementation needs the IRCv3 batch dependency, byte-safe frame splitting, message-scoped LF handling, tests, changelog, and a clean rebase before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Negotiate batch before using draft/multiline:** `extensions/irc/src/client.ts:333`
  - body: The multiline spec depends on the `batch` capability, but this branch requests only `draft/multiline` and then sends `BATCH` frames. Servers that enforce CAP dependencies can reject or ignore this flow, so request and require ACKs for both capabilities before enabling multiline sends.
  - confidence: 0.9
- **[P2] Split long multiline payload lines:** `extensions/irc/src/client.ts:249`
  - body: When multiline is active, each newline-delimited payload is written as one tagged `PRIVMSG`. A long markdown code line or URL can exceed IRC's per-frame limit and be truncated or rejected; split long logical lines into safe frames, using `draft/multiline-concat` where needed to preserve the original line.
  - confidence: 0.92
- **[P3] Keep LF preservation out of command arguments:** `extensions/irc/src/protocol.ts:147-148`
  - body: `sanitizeIrcOutboundText` now preserves standalone LF, but this helper still feeds NickServ, QUIT, and USER command arguments before `sendRaw` removes newlines entirely. Inputs with LF now concatenate tokens instead of the old space-flattened value, so keep newline preservation message-specific or flatten LF at non-message call sites.
  - confidence: 0.8
- **[P3] Add the IRC changelog entry:** `extensions/irc/src/channel.ts:339`
  - body: This is a user-facing IRC feature, but the diff does not update `CHANGELOG.md`. Add a single-line entry in the active section with contributor credit before landing.
  - confidence: 0.84
