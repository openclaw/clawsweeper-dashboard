## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Negotiate `batch` before sending multiline batches:** `extensions/irc/src/client.ts:333`
  - body: The IRCv3 multiline spec depends on the `batch` capability, but this branch requests only `draft/multiline` and later enables BATCH sends from that ACK. Servers that require the dependency can reject or ignore the BATCH frames, so request and require `batch` with `draft/multiline` before setting `multilineCap`.
  - confidence: 0.94
- **[P1] Split long batch lines before writing PRIVMSG:** `extensions/irc/src/client.ts:249`
  - body: Each newline-separated logical line is written as one tagged `PRIVMSG` without the 350-character guard used by the fallback path. A long URL or code line can exceed IRC's 512-byte frame limit and be truncated or rejected; split oversized logical lines into IRC-safe frames, using `draft/multiline-concat` where needed.
  - confidence: 0.95
- **[P2] Honor advertised multiline size limits:** `extensions/irc/src/client.ts:332-333`
  - body: This only checks whether the capability string contains `draft/multiline`, but the spec requires servers to advertise `max-bytes` and may advertise `max-lines`. Sending a 16k logical chunk without parsing those values can make otherwise valid replies fail with multiline limit errors on stricter servers.
  - confidence: 0.9
- **[P2] Handle concat tags when combining incoming batches:** `extensions/irc/src/client.ts:462`
  - body: Incoming multiline batches are always reconstructed by joining every buffered line with `\n`. The spec says lines tagged `draft/multiline-concat` must be joined directly to the previous line, so long messages split by a compliant sender will be reconstructed with extra newlines.
  - confidence: 0.89
- **[P2] Keep LF preservation out of shared command sanitization:** `extensions/irc/src/protocol.ts:147-148`
  - body: `sanitizeIrcOutboundText` is shared by NickServ, USER, QUIT, and message paths. After this change, standalone LF survives the helper and `sendRaw` removes it with no separator, so payloads like `secret\nJOIN #bad` become `secretJOIN #bad`; keep LF preservation message-specific or flatten command payloads before `sendRaw`.
  - confidence: 0.87
