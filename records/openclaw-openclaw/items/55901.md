## Summary

Keep open: this PR addresses a real IRC plugin feature gap, and prior discussion shows several earlier review issues were fixed, but the latest head still has concrete IRCv3 protocol and sanitizer regressions and is dirty against current main.

## What This Changes

The PR adds IRC draft/multiline support with newline-preserving chunking, IRCv3 tag parsing, CAP negotiation, and BATCH-based send/receive handling.

## Best Possible Solution

Land a rebased IRC plugin change that negotiates both `batch` and `draft/multiline`, emits byte-safe multiline frames with `draft/multiline-concat` where needed, scopes LF preservation to message bodies, preserves current outbound sanitization, and adds tests plus a changelog entry.

## Reproduction Assessment

Yes. Current main's feature gap and the PR head regressions are source-reproducible from the IRC client/protocol code and IRCv3 spec, though I did not perform a live IRC server run in this read-only review.

## Solution Assessment

No, not as-is. `draft/multiline` is the right IRC-plugin-local direction, but this implementation needs batch negotiation, long-line splitting, message-scoped LF handling, tests, changelog, and a clean rebase before merge.

Full review comments:

- **[P2] Request batch before enabling multiline:** `extensions/irc/src/client.ts:333`
  - body: The IRCv3 multiline spec depends on the `batch` capability, but this branch only requests `draft/multiline` and then sends BATCH frames. Servers that enforce capability dependencies can reject or ignore this path, so request and require ACKs for both capabilities before enabling multiline sends. ([ircv3.net](https://ircv3.net/specs/extensions/multiline?utm_source=openai))
  - confidence: 0.9
- **[P2] Split long multiline payload lines:** `extensions/irc/src/client.ts:249`
  - body: When multiline is active, each newline-delimited payload is written as one tagged `PRIVMSG`. A long markdown code line or URL can still exceed IRC's per-frame limit; split long logical lines into safe frames and use `draft/multiline-concat` where needed to preserve the original line. ([ircv3.net](https://ircv3.net/specs/extensions/multiline?utm_source=openai))
  - confidence: 0.92
- **[P3] Keep LF preservation message-scoped:** `extensions/irc/src/protocol.ts:147-148`
  - body: `sanitizeIrcOutboundText` now preserves standalone LF, but this helper also feeds NickServ, QUIT, and USER arguments before `sendRaw` removes newlines entirely. Inputs with LF now concatenate tokens instead of the old space-flattened value, so preserve LF only for message bodies or flatten LF at non-message call sites.
  - confidence: 0.82
- **[P3] Add the IRC changelog entry:** `extensions/irc/src/channel.ts:339`
  - body: This is a user-facing IRC feature, but the diff does not update `CHANGELOG.md`. Add a single-line entry in the active section before landing.
  - confidence: 0.84
