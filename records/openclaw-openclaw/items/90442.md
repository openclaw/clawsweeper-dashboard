Keep open. Current main and v2026.6.8 already ship lower-level Slack Block Kit plumbing, but the requested trusted cron/report producer contract is still an explicit Slack plugin product/API decision rather than an implemented or duplicate cleanup item.

## What This Changes

Requested decision and documentation for whether Slack-native Block Kit blocks should be a supported advanced reporting surface for trusted cron/report producers.

## Best Possible Solution

Maintainers should choose and document one Slack-plugin-owned boundary: either bless trusted native Block Kit report payloads with fallback and validation rules, or keep portable presentation as the only supported producer API and track richer report/table layout separately.

## Reproduction Assessment

Not applicable. This is explicitly a feature/product-boundary request; source, docs, tests, and the issue discussion confirm raw Slack block paths exist while the documented producer contract remains unresolved.

## Solution Assessment

Unclear until maintainers choose the boundary. A narrow docs or Slack-plugin update is best only if maintainers bless provider-specific blocks; otherwise the maintainable answer is to keep presentation canonical and design richer report layout separately.
