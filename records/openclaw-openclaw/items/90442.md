Keep open. Current main has Slack Block Kit plumbing and tests, but the public producer-facing docs still steer shared sends to portable `presentation`; this issue is asking for a maintainer support-boundary decision that has not been made yet.

## What This Changes

The issue asks maintainers to decide and document whether trusted cron/report producers may use Slack-native Block Kit blocks as a supported advanced reporting surface.

## Best Possible Solution

Have maintainers choose one Slack-plugin-owned contract: either document trusted native Block Kit blocks with fallback-text and validation requirements, or keep `presentation` as the only supported shared producer API and track richer report/table layout separately.
