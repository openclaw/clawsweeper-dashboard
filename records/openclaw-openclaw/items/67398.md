## Summary

Keep open: current main still has the heartbeat outbound-session mismatch, but this PR needs revision because the proposed key swap also changes policy-sensitive outbound context for group/media delivery and the comments overstate transcript mirroring.

## What This Changes

The PR changes heartbeat outbound delivery to build its session context from `runSessionKey` and adds a regression test for isolated `:heartbeat` delivery keys on fresh-base and wake re-entry paths.

## Best Possible Solution

Carry two contexts: use the isolated `:heartbeat` key for outbound identity, diagnostics, and internal hooks while preserving the base session for payload planning, media access, and delivery policy, then update the comments or add explicit mirror wiring if transcript appending is intended.

## Reproduction Assessment

Yes. A source-level reproduction is clear: mock `deliverOutboundPayloads` for an isolated heartbeat and assert current main passes the base `session.key`; a group-session media-policy variant would reproduce the PR-introduced policy-context regression.

## Solution Assessment

No as written. The one-line identity change is directionally useful, but the safer maintainable fix must preserve the base policy key and align the transcript-mirror comments with actual `mirror` behavior.

## Review Findings

  - body: This replaces the outbound session key without carrying the base session as policy context. Delivery uses the session key for payload planning and media access; for a group key like `agent:main:whatsapp:group:ops`, the PR changes that lookup to `...:ops:heartbeat`, which no longer matches group id `ops` and can skip group-specific read/media restrictions. Please preserve the base session as `policySessionKey` or equivalent and ensure media access uses that policy context.

  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff can bypass group-specific outbound media/read policy by replacing the base group session key with a suffixed heartbeat key without preserving policy context.

Concerns:

- **[medium] Base group media policy can be skipped:** `src/infra/heartbeat-runner.ts:980`
  - body: `deliverOutboundPayloads` passes `params.session?.key` into outbound media access, and group policy resolution derives the group id from that key. The PR changes a base group key to a `:heartbeat`-suffixed key, which no longer matches the configured group id, so a group-specific read denial may not apply to heartbeat media delivery.
  - confidence: 0.86
