## Summary

Keep open. Current main still has the heartbeat delivery-key mismatch, but the PR should not merge as-is because it changes `session.key` broadly enough to affect delivery policy/media context, does not actually wire transcript mirroring without an explicit `mirror`, and still lacks contributor real behavior proof.

## What This Changes

The branch changes heartbeat outbound session context from the base session key to `runSessionKey` and adds isolated-heartbeat delivery-key regression tests.

## Best Possible Solution

Land a reviewed heartbeat fix that separates isolated run identity from base delivery/policy context, adds explicit mirror metadata only if transcript append is intended, and proves the real heartbeat delivery behavior after the patch.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the source-level delivery-key mismatch: current main runs the heartbeat turn with `runSessionKey` but builds outbound delivery from the base `sessionKey`; no for the transcript append claim as stated, because heartbeat sends do not pass an explicit `mirror`.

## Solution Assessment

Is this the best way to solve the issue? No as written. The safer fix should use the isolated key only for the intended identity or mirror surface while preserving the base delivery policy context, and should add mirror wiring if transcript appending is part of the behavior.
