## Summary

Keep open: current main still has the heartbeat outbound-key mismatch, but this PR is not merge-ready because it changes session identity without preserving base delivery-policy context and lacks after-fix real behavior proof.

## What This Changes

The PR changes heartbeat outbound delivery to build its outbound session context from the isolated heartbeat run key and adds a targeted regression test for isolated delivery keys.

## Best Possible Solution

Land a reviewed heartbeat fix that routes isolated heartbeat delivery identity to the stable `:heartbeat` run key while preserving the base session as delivery-policy context, with explicit mirror wiring only if transcript appending is intended.

## Reproduction Assessment

Yes. Current main sets the agent-turn `SessionKey` to the isolated key but builds outbound delivery context from the base key, and the PR's new test targets that call-argument mismatch.

## Solution Assessment

No as written. The one-line identity change is directionally useful, but the maintainable fix needs to preserve base policy/media context and either correct the mirror wording or add explicit mirror behavior and coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve the base policy key for outbound delivery:** `src/infra/heartbeat-runner.ts:980`
  - body: This swaps the outbound session key to the isolated `:heartbeat` key without carrying the base session as policy context. Delivery uses session context for payload planning and host-media policy, so group sessions can stop matching their configured group policy; pass the isolated key as identity and the base key as `policySessionKey` or equivalent.
  - confidence: 0.88
- **[P3] Align the comment with explicit mirror behavior:** `src/infra/heartbeat-runner.ts:969-976`
  - body: The new comment says changing `sessionKey` mirrors transcripts to the isolated session, but delivery appends a transcript only when `params.mirror` is supplied. Please describe this as outbound session metadata/internal hook routing, or add explicit mirror wiring and coverage if transcript appending is intended.
  - confidence: 0.86
- **[P3] Mock the outbound delivery seam directly:** `src/infra/heartbeat-runner.isolated-session-mirror.test.ts:7`
  - body: This argument-only test imports the broad outbound delivery module and spies on it, which conflicts with the scoped outbound test guidance. Use a hoisted mock for `./outbound/deliver.js`, matching nearby heartbeat tests, so the test does not load the full delivery runtime.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff can change group-scoped outbound media/read policy resolution by replacing the base delivery session key with a suffixed heartbeat key.

Concerns:

- **[medium] Base group media policy can be skipped:** `src/infra/heartbeat-runner.ts:980`
  - body: Outbound media access passes `params.session?.key` into group policy resolution, and the policy derives group candidates from that session key. A group base key such as `agent:main:whatsapp:group:ops` becomes `agent:main:whatsapp:group:ops:heartbeat`, so a group-specific read/media restriction may not apply unless the base key is preserved as policy context.
  - confidence: 0.86
