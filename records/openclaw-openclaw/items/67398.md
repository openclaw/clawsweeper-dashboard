## Summary

Keep open: current main still has a source-level heartbeat delivery-key mismatch, but this branch is not ready because it changes delivery policy/media context and does not actually wire transcript mirroring. The external PR also still lacks after-fix proof from a real heartbeat setup.

## What This Changes

The PR changes heartbeat outbound delivery context to use the isolated run session key and adds a regression test for isolated heartbeat delivery keys.

## Best Possible Solution

Land a reviewed heartbeat fix that uses the isolated run key only for the intended identity/internal hook or mirror surface, preserves base delivery policy context, and adds explicit mirror wiring and coverage if transcript appending is intended.

## Reproduction Assessment

Yes for the delivery-key mismatch at source level: current main runs the heartbeat agent with runSessionKey but builds outbound delivery from the base sessionKey. No for the transcript append claim as stated, because current heartbeat delivery passes no explicit mirror object.

## Solution Assessment

No as written. Passing only runSessionKey is too broad because it changes policy/media context and still does not append a transcript mirror without the existing mirror seam.
