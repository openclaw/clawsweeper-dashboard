## Summary

Keep open: this PR targets a real current-main heartbeat routing mismatch, but the patch is not merge-ready because it changes outbound session identity without preserving base delivery policy/media context, does not explicitly wire transcript mirroring, and lacks external contributor real behavior proof.

## What This Changes

The PR changes heartbeat outbound delivery context to use the isolated heartbeat run key and adds regression tests for fresh-base and wake re-entry isolated heartbeat delivery keys.

## Best Possible Solution

Route isolated heartbeat delivery identity and any explicit transcript mirror to the stable `:heartbeat` run key while preserving the base session for delivery policy/media context, then cover both the outbound key and actual mirror append behavior.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source level: current main uses the isolated `runSessionKey` for the agent turn but builds outbound delivery context from the base `sessionKey`; I did not run tests because this was a read-only review.

## Solution Assessment

Is this the best way to solve the issue? No as written. The patch moves the identity in the right direction, but it needs explicit mirror wiring and a split between isolated identity and base delivery-policy/media context.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

  - body: Changing the outbound session key to the isolated `:heartbeat` key means delivery planning and host-media policy no longer see the base group/session key. Current delivery only consults `policyKey` for payload planning and still passes `session.key` into media access, so the fix needs to carry the base policy/media context before routing identity to the heartbeat key.
  - confidence: 0.88
- **[P2] Wire an explicit mirror before relying on transcript append:** `src/infra/heartbeat-runner.ts:970-976`
  - body: The new comment and test treat `session.key` as the transcript mirror target, but current delivery appends transcript entries only when `params.mirror` is supplied. With this patch alone, the isolated session file can still remain unwritten for heartbeat replies; add mirror metadata and cover the transcript append path if that is the intended fix.
  - confidence: 0.86
- **[P3] Mock the outbound delivery seam directly:** `src/infra/heartbeat-runner.isolated-session-mirror.test.ts:7`
  - body: This argument test imports the broad outbound delivery module and spies on it, which goes against the scoped outbound guidance for narrow argument/payload tests. Use a hoisted mock for `./outbound/deliver.js`, matching nearby heartbeat tests, so the real delivery runtime is not loaded here.
  - confidence: 0.82
