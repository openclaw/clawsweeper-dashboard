## Summary

Keep open: this PR targets a real heartbeat delivery-key mismatch, but it is not merge-ready because the patch changes outbound session identity without preserving base delivery policy/media context, and the external PR still lacks real heartbeat proof.

## What This Changes

The branch changes heartbeat outbound delivery context to use `runSessionKey` and adds isolated-heartbeat delivery-key regression coverage.

## Best Possible Solution

Land a revised heartbeat fix that uses the isolated `:heartbeat` key for outbound identity/internal hooks while preserving the base session for delivery policy/media context, with explicit mirror wiring and coverage only if transcript appending is intended.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main sets the agent-turn `SessionKey` to `runSessionKey` but builds outbound delivery context from the base `sessionKey`. I did not run tests because this was a read-only review.

## Solution Assessment

No as written. The direction is useful, but the patch should split isolated delivery identity from base delivery-policy/media context and should not claim transcript mirroring unless it supplies the explicit mirror path.
