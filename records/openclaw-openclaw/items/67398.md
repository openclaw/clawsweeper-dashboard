## Summary

Keep open: current main still has a source-level heartbeat delivery-key mismatch, but this PR is not merge-ready because it changes outbound session identity without preserving base delivery policy and it overstates transcript mirroring.

## What This Changes

The branch changes heartbeat outbound delivery context to use `runSessionKey` and adds regression coverage for isolated heartbeat delivery receiving the `:heartbeat` session key.

## Best Possible Solution

Land a reviewed heartbeat fix that separates isolated run identity and internal hooks from base delivery policy context, with explicit mirror wiring and coverage only if heartbeat transcript append is intended.

## Reproduction Assessment

Yes for the delivery-key mismatch: current main uses `runSessionKey` for the heartbeat agent turn but builds outbound delivery from the base `sessionKey`. The transcript-append claim is not fully reproduced because current delivery only writes transcripts when an explicit `mirror` object is supplied.

## Solution Assessment

No as written. Passing only `runSessionKey` is too broad because it changes delivery policy/media context and does not actually enable transcript mirroring without the existing mirror seam.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Preserve the base policy context for delivery:** `src/infra/heartbeat-runner.ts:979`
  - body: Switching `session.key` to the isolated `:heartbeat` key without also passing the base key as `policySessionKey` changes outbound planning and host-media policy resolution. Keep the isolated key for identity/internal hooks, but preserve the base session for delivery policy/media context.
  - confidence: 0.88
- **[P2] Add mirror wiring before claiming transcript mirroring:** `src/infra/heartbeat-runner.ts:969-979`
  - body: This change does not append heartbeat text to the isolated transcript because outbound transcript writes only happen when an explicit `mirror` object is provided. Either add mirror metadata and coverage for the intended transcript write, or narrow the comment/test to delivery metadata only.
  - confidence: 0.87
