## Summary

Keep open: current main still builds heartbeat outbound delivery from the base session key, so the PR targets a real gap, but the patch is not safe as-is. Replacing the outbound session key without preserving the base policy context can make isolated group heartbeat deliveries miss group-specific media/tool policy, and the new comments overstate transcript mirroring that current delivery only performs with an explicit `mirror` object.

## What This Changes

The PR changes heartbeat outbound delivery to build its session context from `runSessionKey` and adds an isolated heartbeat regression test that asserts `deliverOutboundPayloads` receives the stable `:heartbeat` key.

## Best Possible Solution

Keep the PR's core direction, but carry two contexts: use the isolated `:heartbeat` key for outbound identity, diagnostics, and internal hooks, while preserving the base session as the policy/delivery context for payload planning and media access. Add regression coverage for group-policy media access plus the existing fresh-base and wake re-entry key assertions, and reword the mirror comments unless explicit mirror wiring is added.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: an isolated heartbeat with `deliverOutboundPayloads` mocked shows current main passes the base `session.key`; a group-session variant with outbound media policy would exercise the PR-introduced loss of base policy context when the key gains `:heartbeat`.

## Solution Assessment

No. The one-line change is a plausible identity fix, but as written it also changes policy-sensitive session resolution; the safer solution is to pass the isolated key and preserve the base policy key, then update media-policy wiring and comments accordingly.

## Security Review

Status: needs_attention

Summary: The diff can bypass group-specific outbound media/read policy by replacing the base group session key with a suffixed heartbeat key without preserving policy context.

Concerns:

- **[medium] Base group media policy can be skipped:** `src/infra/heartbeat-runner.ts:973`
  - body: `deliverOutboundPayloads` passes `params.session?.key` into outbound media access, and group policy resolution derives the group id from that key. The PR changes a base group key to a `:heartbeat` suffixed key, which no longer matches the configured group id, so a group-specific read denial may not apply to heartbeat media delivery.
