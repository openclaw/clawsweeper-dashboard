## Summary

Keep open: the PR targets a real heartbeat/session routing surface, but the one-line change is not merge-ready because it changes delivery policy context and does not implement the transcript mirror behavior claimed in the body.

## What This Changes

The PR changes heartbeat outbound session context to use `runSessionKey` instead of the base `sessionKey` and adds isolated-heartbeat delivery-key regression tests.

## Best Possible Solution

Land a reviewed heartbeat fix that separates isolated run identity from base delivery policy context and adds explicit mirror metadata only if transcript appending is intended.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the source-level delivery-key mismatch: current main runs the heartbeat turn with `runSessionKey` but builds outbound delivery from the base `sessionKey`; no for the transcript-append claim because heartbeat delivery does not pass explicit mirror metadata.

## Solution Assessment

Is this the best way to solve the issue? No as written; the safer path is to preserve base delivery policy context while using the isolated key only for the intended identity or mirror surface.
