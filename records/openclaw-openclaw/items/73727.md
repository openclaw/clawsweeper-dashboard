## Summary

Keep open. Current main does not implement the requested `@default` model sentinel, and this PR is a plausible implementation path, but the current head still has concrete consistency bugs around subagent session metadata and cron fallback classification, plus missing user-facing docs and changelog coverage required for exposing the new operator value.

## What This Changes

The PR adds a `@default` sentinel for stored cron and subagent model overrides, updates cron/subagent model-selection paths and persisted override handling, and adds focused unit and integration tests.

## Best Possible Solution

Land a revised version only if maintainers accept the sentinel behavior, with `@default` handled consistently as no stored model override across model selection, fallback policy, persisted/session metadata, public cron/subagent docs, changelog, and regression tests; then use related issue #37843 as the canonical follow-up to close after merge.

## Reproduction Assessment

Yes. The PR defects are reproducible by calling `resolveSubagentModelAndThinkingPlan` with `modelOverride: "@default"` and checking `initialSessionPatch.modelOverrideSource`, and by comparing `resolveCronFallbacksOverride` for omitted `payload.model` versus `payload.model: "@default"`.

## Solution Assessment

No, not as currently patched. The shared sentinel helper is a narrow maintainable direction, but the implementation needs sentinel-aware metadata, fallback classification, docs, changelog, and focused regression coverage before it is the best solution.
