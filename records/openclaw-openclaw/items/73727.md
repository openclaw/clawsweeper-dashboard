## Summary

Keep open. Current main still does not implement a runtime default model sentinel, so the PR is useful, but the proposed patch remains merge-blocked by sentinel consistency bugs plus required docs and changelog coverage. The earlier Greptile and ClawSweeper comments correctly identify the same narrow repair area.

## What This Changes

The PR adds a shared `@default` sentinel for stored cron and subagent model overrides, updates cron/subagent model selection to fall through to live defaults, and adds focused model-selection tests.

## Best Possible Solution

Land a revised version only if maintainers accept the sentinel behavior, with `@default` handled consistently across model selection, fallback policy, persisted/session metadata, public docs, changelog, and regression tests; then use related issue #37843 as the canonical follow-up to close once merged.

## Reproduction Assessment

Yes. The PR defects are reproducible by calling the subagent planning path with `modelOverride: "@default"` and expecting `modelOverrideSource: "auto"`, and by comparing `resolveCronFallbacksOverride` for omitted `payload.model` versus `payload.model: "@default"`.

## Solution Assessment

No, not as currently patched. The shared sentinel helper is a reasonable narrow approach, but the implementation needs the metadata predicate, cron fallback classification, docs, changelog, and focused tests tightened before it is the best maintainable solution.
