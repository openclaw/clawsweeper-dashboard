## Summary

Keep open. Current main still lacks the requested runtime default sentinel, and the PR branch is useful but not mergeable because sentinel handling is inconsistent in subagent metadata, cron fallback classification, docs, and changelog coverage.

## What This Changes

The PR adds a shared `@default` sentinel so stored cron and subagent model overrides can resolve to the live default model at execution time instead of pinning a concrete model string.

## Best Possible Solution

Land a revised version only if maintainers accept the sentinel contract, with `@default` handled consistently across selection, fallback policy, session metadata, public docs, changelog, and regression tests; then use #37843 as the canonical follow-up once merged.

## Reproduction Assessment

Yes. The PR defects are reproducible by planning a subagent with `modelOverride: "@default"` and checking for `modelOverrideSource: "auto"`, and by comparing `resolveCronFallbacksOverride` for omitted `payload.model` versus `payload.model: "@default"`.

## Solution Assessment

No, not as currently patched. The shared sentinel helper is a reasonable narrow implementation, but the branch needs the metadata predicate, cron fallback policy, docs, changelog, and focused tests tightened before it is the best maintainable solution.

  - confidence: 0.93

That makes `@default` use a different fallback path than an omitted model, so the sentinel is not fully equivalent to following the live default.
  - confidence: 0.82

  - body: This patch makes `@default` a supported operator value, but the cron and subagent docs still describe model fields as ordinary explicit overrides. Please add public docs for the migration path and precedence so users know when `@default` tracks the live default versus when a real model string stays pinned.
  - confidence: 0.85
