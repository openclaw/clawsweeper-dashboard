The PR adds a shared `@default` sentinel for stored cron and subagent model overrides, updates model-selection paths, and adds cron/subagent regression tests.

- **[P2] Exclude the sentinel from user override metadata:** `src/agents/subagent-spawn-plan.ts:75`
  - body: For `modelOverride: "@default"`, model selection falls through to the configured/default model, but this predicate still treats the non-empty sentinel as user-supplied.   - confidence: 0.93

  - body: This normalizes `payload.model: "@default"` away for model selection, but `resolveCronFallbacksOverride` still inspects the raw payload and classifies any non-empty model as a payload override. That makes `@default` use a different fallback policy than an omitted model, so the sentinel is not fully equivalent to following the live default.
  - confidence: 0.86

  - body: This test passes an object into `resolveSubagentModelAndThinkingPlan`, but that function's params still declare `modelOverride?: string`. The changed test should fail the repo's test typecheck; either widen the function type if object-shaped overrides are valid here, or move this assertion to the lower-level `resolveSubagentSpawnModelSelection` API that already accepts `unknown`.
  - confidence: 0.88
- **[P3] Add public docs and changelog for the sentinel:** `src/agents/model-default-sentinel.ts:9`
  - body: This exposes `@default` as a supported operator value, but the cron and subagent docs still describe model fields as ordinary explicit overrides and the changelog has no user-facing feature entry. Please document the migration path, precedence, and case-sensitive spelling before shipping it.
