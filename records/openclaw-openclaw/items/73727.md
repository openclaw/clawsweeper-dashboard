## Summary

Keep open. Current main still lacks the `@default` sentinel behavior, so the PR is not obsolete, but the proposed patch is not ready because sentinel-aware selection is applied without matching subagent metadata, cron fallback-policy, docs, and changelog updates.

## What This Changes

The PR adds a shared `@default` sentinel for cron and subagent stored model overrides, plus tests, so those paths resolve `defaults.primary` at execution time instead of pinning a concrete model string.

## Best Possible Solution

Land a revised version only if maintainers accept the `@default` product behavior, with sentinel semantics applied consistently to cron selection, cron fallback policy, subagent session metadata, persisted override resolution, public docs, changelog, and focused regression tests; then use the related runtime-default cron model request #37843 as the canonical follow-up to close once merged.

## Reproduction Assessment

Yes. The PR defects are reproducible by calling the subagent planning path with `modelOverride: "@default"` and expecting `modelOverrideSource: "auto"`, and by comparing `resolveCronFallbacksOverride` for omitted `payload.model` versus `payload.model: "@default"`.

## Solution Assessment

No, not as currently patched. The sentinel helper is a reasonable narrow approach for the requested behavior, but the patch needs metadata, fallback-policy, docs, changelog, and regression-test updates before it is the best maintainable solution.
