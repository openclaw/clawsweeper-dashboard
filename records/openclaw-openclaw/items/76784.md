## Summary

Keep open: the performance direction is useful, but the current diff can return stale cached list rows because invalidation is local while session mutations broadcast from other Gateway paths; it also lacks required changelog coverage.

## What This Changes

The PR adds a bounded cached/in-flight-coalesced `sessions.list` path, keeps default async rows from using transcript usage fallback, extends gateway/session tests, and relaxes two script shim signal typings.

## Best Possible Solution

Keep the lightweight list-row optimization, but tie cache freshness to a central session-change/store-version signal and add a user-facing changelog entry before merge.

## Reproduction Assessment

Yes. From source inspection, cache an identical `sessions.list` request, then create or update session state through chat lifecycle, session-event, or agent paths outside `server-methods/sessions.ts`; the next identical list can reuse stale cached rows until the 60s TTL.

## Solution Assessment

No. Whole-result caching can be maintainable, but this patch invalidates from only one local helper; central invalidation or a reliable store version/mtime key is the narrower safe fix.
