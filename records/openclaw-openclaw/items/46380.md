## Summary

Keep open: the MiniMax VLM SSRF concern is still valid on current main, but this stale branch is not merge-ready because it drops established proxy and timeout behavior and has no after-fix real behavior proof.

## What This Changes

The PR changes `src/agents/minimax-vlm.ts` to replace the raw MiniMax VLM `fetch()` call with `fetchWithSsrFGuard(withStrictGuardedFetchMode(...))` and release the guarded dispatcher afterward.

## Best Possible Solution

Rework MiniMax VLM to use guarded fetch with `auditContext`, timeout forwarding, release cleanup, strict direct mode, and `TRUSTED_ENV_PROXY` only when `shouldUseEnvHttpProxyForUrl(url)` permits it, with focused SSRF/proxy/timeout tests and real proof.

## Reproduction Assessment

Yes, by source inspection rather than live execution: current main builds the MiniMax VLM target from configurable host inputs and sends the bearer token and image body with raw fetch. The PR regressions are also source-backed by current proxy, timeout, and guarded-fetch contracts.

## Solution Assessment

No. SSRF-guarded fetch is the right primitive, but this branch hard-codes strict mode and omits timeout and audit context, so a safer fix should preserve the current proxy and timeout contracts while adding SSRF blocking.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve env-proxy routing for MiniMax VLM:** `src/agents/minimax-vlm.ts:80-81`
  - body: Hard-coding `withStrictGuardedFetchMode` ignores the current main env-proxy path for ordinary `HTTP_PROXY`/`HTTPS_PROXY` operator routing. In proxy-only environments this can fail or bypass the expected egress route; select trusted env-proxy mode when `shouldUseEnvHttpProxyForUrl(url)` permits it and strict mode otherwise.
  - confidence: 0.9
- **[P2] Keep the MiniMax VLM request timeout:** `src/agents/minimax-vlm.ts:80-81`
  - body: The replacement guarded-fetch call passes neither `timeoutMs` nor a signal, while current main computes the caller/default timeout and uses `AbortSignal.timeout(timeoutMs)`. A rebased version would drop MiniMax VLM timeout behavior and can hang on slow or stuck requests.
  - confidence: 0.86
- **[P3] Set a MiniMax audit context:** `src/agents/minimax-vlm.ts:80`
  - body: The guarded fetch call omits `auditContext`, so SSRF-blocked warnings fall back to the generic `url-fetch` label. Add a MiniMax-specific context so security logs identify this endpoint.
  - confidence: 0.78
