## Summary

Keep open: current main still has the configurable MiniMax VLM raw-fetch sink, and this PR is not merge-ready because it drops existing proxy/timeout behavior and lacks after-fix proof.

## What This Changes

The PR replaces the MiniMax VLM raw fetch with `fetchWithSsrFGuard(withStrictGuardedFetchMode(...))` and releases the guarded dispatcher afterward.

## Best Possible Solution

Use guarded fetch for MiniMax VLM with `auditContext`, timeout forwarding, release cleanup, and trusted env-proxy mode only when the proxy gate permits it, plus focused SSRF/proxy/timeout coverage and redacted real proof.

## Reproduction Assessment

Yes, by source inspection rather than live execution: current main builds the MiniMax VLM target from configurable host inputs and sends the bearer token and image payload with raw `fetch()`. The PR regressions are also source-backed by current proxy, timeout, and guarded-fetch contracts.

## Solution Assessment

No. SSRF-guarded fetch is the right primitive, but this branch hard-codes strict mode and omits timeout and audit context, so the safer fix should preserve the current proxy and timeout contracts while adding SSRF blocking.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve env-proxy routing for MiniMax VLM:** `src/agents/minimax-vlm.ts:80-81`
  - body: Hard-coding `withStrictGuardedFetchMode` ignores the current env-proxy path for ordinary `HTTP_PROXY`/`HTTPS_PROXY` operator routing. In proxy-only environments this can fail or bypass the expected egress route; select trusted env-proxy mode when `shouldUseEnvHttpProxyForUrl(url)` permits it and strict mode otherwise.
  - confidence: 0.9
- **[P2] Keep the MiniMax VLM request timeout:** `src/agents/minimax-vlm.ts:80-81`
  - body: The replacement guarded-fetch call passes neither `timeoutMs` nor a signal, while current main computes the caller/default timeout and uses `AbortSignal.timeout(timeoutMs)`. A rebased version would drop MiniMax VLM timeout behavior and can hang on slow or stuck requests.
  - confidence: 0.86
- **[P3] Set a MiniMax audit context:** `src/agents/minimax-vlm.ts:80`
  - body: The guarded fetch call omits `auditContext`, so SSRF-blocked warnings fall back to the generic `url-fetch` label. Add a MiniMax-specific context so security logs identify this endpoint.
  - confidence: 0.78
