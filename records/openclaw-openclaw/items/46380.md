## Summary

Keep open: current main still has the raw configurable MiniMax VLM fetch sink, but this branch is not mergeable as-is because it drops current env-proxy and timeout behavior and lacks required real behavior proof.

## What This Changes

The PR changes `src/agents/minimax-vlm.ts` to call MiniMax VLM through guarded fetch instead of raw `fetch()`.

## Best Possible Solution

Guard MiniMax VLM with SSRF-protected fetch while preserving timeout, trusted env-proxy routing via the existing gate, release cleanup, and MiniMax-specific audit context, then add focused tests and redacted runtime proof.

## Reproduction Assessment

Yes, by source inspection: current main builds the MiniMax VLM URL from configurable host inputs and sends a bearer token plus image data through raw `fetch()`. No live MiniMax credential test was run in this read-only review.

## Solution Assessment

No. SSRF-guarded fetch is the right primitive, but the current PR diff is stale against current main and should preserve env-proxy routing, timeout enforcement, and audit attribution.
