## Summary

Keep open: the SSRF concern is valid by source inspection, but this PR is not merge-ready because it hard-codes strict guarded fetch and would break or bypass normal env-proxy egress for MiniMax VLM requests.

## What This Changes

The PR wraps MiniMax image-understanding requests in `fetchWithSsrFGuard(withStrictGuardedFetchMode(...))` and releases the guarded dispatcher after response handling.

## Best Possible Solution

Use guarded fetch for MiniMax VLM while preserving current timeout and proxy behavior: strict mode by default, `TRUSTED_ENV_PROXY` when `shouldUseEnvHttpProxyForUrl(url)` qualifies, `auditContext: "minimax-vlm"`, `release()` in `finally`, and regression tests for private-IP blocking plus env-proxy mode.

## Reproduction Assessment

Yes by source inspection, not live execution: current main derives the MiniMax VLM host from user/configurable inputs and sends the API key and image body to that URL with raw `fetch`. The PR regression is also source-backed by guarded-fetch tests showing strict mode ignores env proxy by default.

## Solution Assessment

No. The SSRF guard family is the right direction, but the proposed strict-mode-only call is not the narrowest maintainable fix because existing provider/media helpers preserve env proxy routing when eligible.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Preserve env-proxy routing for MiniMax VLM:** `src/agents/minimax-vlm.ts:80`
  - body: Hard-coding `withStrictGuardedFetchMode` makes this call ignore normal `HTTP_PROXY`/`HTTPS_PROXY` routing by default. Current main explicitly enables env-proxy dispatch before MiniMax VLM fetches, and the guarded-fetch tests document that strict mode bypasses that path unless managed proxy mode is active. In proxy-only or audited-egress deployments this turns image understanding into a failure or sends traffic outside the operator proxy; select trusted env-proxy mode when `shouldUseEnvHttpProxyForUrl(url)` qualifies and keep strict mode otherwise.
  - confidence: 0.87
