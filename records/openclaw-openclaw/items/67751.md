Full review comments:

- **[P2] Preserve runtime conversation bindings when waking:** `extensions/slack/src/monitor/events/interactions.block-actions.ts:692-696`
  - body: This wake path derives SessionKey by applying resolveThreadSessionKeys directly to route.sessionKey. Regular Slack inbound routing first checks runtime conversation bindings for the routed thread or root conversation, so bound Slack conversations can dispatch button clicks to a different session than normal messages in the same thread.
  - confidence: 0.84
- **[P2] Resolve reply mode for the Slack chat type:** `extensions/slack/src/monitor/events/interactions.block-actions.ts:732-760`
  - body: The delivery path uses ctx.replyToMode, which is only the account default. Normal Slack inbound handling resolves the account's chat-type reply mode, so workspaces with direct/channel overrides can thread button-triggered replies differently from regular messages in the same conversation.
  - confidence: 0.8
