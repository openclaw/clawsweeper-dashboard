## Summary

Keep open. Current main still leaks leading session-recap blocks through the shared user-visible metadata stripper, and this PR targets the right sanitizer boundary, but the patch has two concrete content-loss regressions that should block merge until repaired.

## What This Changes

The PR adds a session-recap fast path/helper to strip leading `<session-recap>` or `<session_recap>` blocks from inbound user text and adds six colocated sanitizer tests.

## Best Possible Solution

Land a narrow sanitizer update that strips only complete leading `<session-recap>` and `<session_recap>` blocks in both exported strip functions, leaves malformed or mid-message literals untouched, preserves current-main timestamp/active-memory/untrusted-context behavior, and adds regression coverage plus a user-facing changelog entry.

## Reproduction Assessment

Yes. A leading closed recap-only block passed into `stripInboundMetadata()` or `stripLeadingInboundMetadata()` reproduces the current-main leak by taking the fast return; the PR regressions reproduce with an unclosed leading recap opener and with a non-leading recap marker surrounded by user-authored blank lines.

## Solution Assessment

No, not as-is. The shared sanitizer boundary is the right place to solve this, but the helper must require a matching closing tag and the exported functions must return the original string when no leading recap block was actually removed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Require a closing recap tag before stripping:** `src/auto-reply/reply/strip-inbound-meta.ts:135`
  - body: If the first non-blank line is `<session-recap>` or the underscore variant but no matching close tag exists, this helper advances to EOF and returns an empty slice. That silently deletes a user message or pasted XML snippet in both display paths; only strip after a closing tag is found.
  - confidence: 0.96
- **[P2] Return original text when recap is not stripped:** `src/auto-reply/reply/strip-inbound-meta.ts:173`
  - body: A non-leading recap marker still sends the text down the slow path, and when there are no standard sentinels this branch rejoins the unchanged lines while trimming surrounding newlines. Preserve the original string when the recap helper made no leading-block removal.
  - confidence: 0.91
