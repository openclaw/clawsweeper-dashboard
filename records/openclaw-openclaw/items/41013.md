## Summary

Keep open. Current main still leaks leading session-recap blocks, and this PR targets the right shared sanitizer, but the patch can erase malformed user text, mutates non-leading user-authored text, and is currently merge-conflicted against main.

## What This Changes

The PR adds a session-recap fast path/helper to the shared inbound metadata stripper and colocated tests for leading, blank-line, mid-message, and underscore-tag cases.

## Best Possible Solution

Land a narrow current-main sanitizer fix that strips only complete leading `<session-recap>` and `<session_recap>` blocks in both exported strip functions, preserves malformed or mid-message user literals and existing metadata behavior, and adds focused tests plus a changelog entry.

## Reproduction Assessment

Yes. On current main, a complete leading `<session-recap>...</session-recap>` string with no standard metadata sentinel follows the fast return in both exported strip functions, and the Control UI/TUI display paths call those functions for user text.

## Solution Assessment

No, not as-is. The shared sanitizer is the right boundary, but the helper must require a matching close tag before stripping and must return the original string when no leading recap block was removed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Require a closing recap tag before stripping:** `src/auto-reply/reply/strip-inbound-meta.ts:132`
  - body: If the first non-blank line is a recap opener but no matching close tag exists, this loop advances to EOF and returns `lines.slice(i)`, deleting the whole user message. Only strip after locating the closing line.
  - confidence: 0.96
- **[P2] Return original text when no recap block is stripped:** `src/auto-reply/reply/strip-inbound-meta.ts:173`
  - body: A mid-message recap marker forces the slow path, and with no standard sentinel this branch rejoins and trims the text even though no leading block was removed. Preserve the original string when the helper makes no change.
  - confidence: 0.91
