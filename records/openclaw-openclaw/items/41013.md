## Best Possible Solution

Update or replace the branch on current main so the shared sanitizer strips only complete leading `<session-recap>` and `<session_recap>` blocks, preserves malformed and mid-message literals plus current sanitizer behavior, adds regression coverage, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. On current main, a complete leading `<session-recap>...</session-recap>` string has no `SENTINEL_FAST_RE` match, so both exported strip functions return without removing it before the Control UI and TUI display paths render user text.

## Solution Assessment

No, not as-is. The shared sanitizer is the right boundary, but the patch must require a matching close tag, preserve the original text when no leading recap block is removed, rebase over current main, and add the required changelog entry.
