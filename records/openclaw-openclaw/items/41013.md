## Summary

Keep open: current main still has the reported session-recap leak in the shared sanitizer paths, but this PR is not merge-ready because the patch can delete malformed leading recap text, changes unrelated mid-message text, lacks the required changelog entry, and needs to be brought forward onto current main.

## What This Changes

The PR adds a session-recap fast-path/helper to the inbound metadata stripper and colocated tests for leading, blank-line, mid-message, and underscore-tag cases.

## Best Possible Solution

Update this PR or create a narrow replacement that strips only complete leading `<session-recap>`/`<session_recap>` blocks in the shared sanitizer, preserves malformed and mid-message literals, keeps current active-memory behavior intact, adds regression coverage, and includes the changelog entry.

## Reproduction Assessment

Yes, source-reproducible. On current main, a complete leading `<session-recap>...</session-recap>` string has no `SENTINEL_FAST_RE` match, so both exported strip functions return it unchanged before the Control UI and TUI display paths render user text.

## Solution Assessment

No, not as-is. The shared sanitizer is the right boundary, but the patch must require a matching close tag before stripping, preserve the original text when no leading block is removed, rebase over current main, and add the required changelog entry.
