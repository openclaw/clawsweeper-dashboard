## Summary

Keep open: current main still has the session-recap leak, but this PR is not safe to merge as-is because the proposed helper can erase malformed/literal user text, can alter content when no leading recap block is removed, and lacks the required changelog entry.

## What This Changes

The PR adds a session-recap fast path/helper and colocated tests to strip leading recap blocks from inbound user-message sanitization.

## Best Possible Solution

Repair this PR or open a narrow replacement that strips only complete leading `<session-recap>` and `<session_recap>` blocks in the shared sanitizer while preserving malformed, mid-message, and ordinary user text exactly.

## Reproduction Assessment

Yes. Source inspection on current main shows a complete leading `<session-recap>` string has no `SENTINEL_FAST_RE` match, so both exported strip functions return it unchanged before Control UI and TUI rendering paths display user text.

## Solution Assessment

No, not as-is. The shared sanitizer is the right boundary, but the patch must require a matching close tag, preserve the original string when no leading recap block is removed, and add the required changelog entry.
