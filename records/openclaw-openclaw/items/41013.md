## Summary

Keep this PR open, but it still needs changes before merge. The linked user-visible bug is valid on current main and the shared inbound sanitizer is the right boundary, but this PR can silently delete malformed/user-authored text and can alter non-leading user content; the branch also reports as dirty against the current base.

## What This Changes

The PR adds a session-recap fast-path/helper to the shared inbound metadata stripper and six colocated sanitizer tests for leading, blank-line, mid-message, and underscore-tag cases.

## Best Possible Solution

Land a narrow current-main sanitizer update that strips only complete leading `<session-recap>` and `<session_recap>` blocks in both exported strip functions, preserves malformed or mid-message user-authored literals, keeps the existing timestamp, active-memory, and untrusted-context behavior intact, and adds focused regression coverage plus a user-facing changelog entry.

## Reproduction Assessment

Yes. On current main, a complete leading `<session-recap>...</session-recap>` string with no standard metadata sentinel follows the fast return in both exported strip functions, and the Control UI/TUI display paths call those functions for user text.

## Solution Assessment

No, not as-is. The shared sanitizer is the right place to fix the leak, but this patch must require a matching closing tag before stripping and must return the original string when no leading recap block was actually removed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Require a closing recap tag before stripping:** `src/auto-reply/reply/strip-inbound-meta.ts:131-135`
  - body: If the first non-blank line is a recap opener but no matching close tag exists, this loop advances to EOF and `lines.slice(i)` drops the entire user message in both display paths. Only strip after locating the closing tag.
  - confidence: 0.96
- **[P2] Preserve unchanged text when no recap block is removed:** `src/auto-reply/reply/strip-inbound-meta.ts:173`
  - body: A mid-message recap marker forces the slow path, and when no standard sentinel is present this returns rejoined text with surrounding newlines removed even though no internal block was stripped. Preserve the original string when the helper made no removal.
  - confidence: 0.91
