## Summary

Keep open: current main and the latest release still have source-visible ACP paths that can finish a visible prompt as `end_turn` after a terminal `chat.send` OK or empty final event without assistant/tool dispatch evidence.

## What This Changes

The issue requests ACP visible-agent prompts to fail with a typed error or stream assistant/tool events instead of completing as a silent empty turn when no model/tool dispatch occurs.

## Best Possible Solution

Keep this as the canonical ACP visible-turn contract bug and define a narrow translator/Gateway handshake where visible prompts require dispatch or output evidence, or return a typed no-dispatch/unavailable-tools error.

## Reproduction Assessment

Yes at source level, but not from a live Workbench rerun. Current main and v2026.6.11 can resolve ACP prompts as `end_turn` from terminal OK or final chat events without assistant/tool evidence, and current tests codify the terminal OK path.

## Solution Assessment

Unclear as an implementation, but the requested contract is the right direction. The fix should target visible prompts with no dispatch/run evidence, not blanket-fail every ACP backend completion with no transcript.
