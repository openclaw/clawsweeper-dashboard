## Summary

Keep open: current main still has source-visible ACP paths that can resolve a visible prompt as `end_turn` without assistant/tool events, and the related raw dynamic-tool payload work does not cover this no-dispatch/empty-final contract.

## What This Changes

Require ACP-visible user prompts to either dispatch a model/tool turn and stream assistant/tool events or fail with a typed runtime/configuration error instead of finishing empty.

## Best Possible Solution

Keep this as the focused ACP visible-turn contract bug and fix the translator/gateway handshake so no-run or empty-final paths surface a typed error or wait for real run evidence without changing valid backend empty-event semantics broadly.

## Reproduction Assessment

Yes from source inspection, but not from a live Workbench rerun: current main can resolve ACP prompts as `end_turn` from terminal `ok` or empty final chat events without assistant/tool/run evidence.

## Solution Assessment

Yes, the requested direction is the right narrow fix: visible ACP prompt completion should require dispatch/run evidence or a typed no-dispatch error, not a blanket no-output failure for every ACP backend.

## Repair Work Prompt

Fix OpenClaw issue #98274. In `src/acp/translator.ts` and the narrow gateway handshake if needed, stop treating no-run terminal `ok` acknowledgements or empty final chat events as successful ACP `end_turn` completions for visible prompts. Preserve the normal `started` acknowledgement flow and existing timeout/error handling. Use existing run/final evidence, or add the smallest typed signal needed, so a visible prompt either observes assistant/tool/runtime events from a dispatched run or surfaces a typed runtime/configuration error when no model/tool dispatch starts. Add focused regressions for `chat.send` returning a no-run/ok acknowledgement and/or a `chat` final with no message so ACP does not emit empty `end_turn` without assistant/tool/runtime events. Do not change generic ACP backend completed-with-no-events semantics in `manager.turn-runner.ts` unless maintainers explicitly choose that broader contract.
