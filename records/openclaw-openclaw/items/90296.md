## What This Changes

The issue reports that `openclaw channels login` saves channel auth but then calls `channels.start`, which can leave an external channel stuck in `SETUP` with no inbound messages.

## Best Possible Solution

Keep this canonical issue open until one accepted fix lands on main with proof for the external-plugin login/start lifecycle, then close it against the merged PR or redirect remaining startup-loader work to the broader canonical issue.

## Reproduction Assessment

No live high-confidence reproduction was run in this read-only review. The source path is reproducible: current main still calls `channels.start` after login, and gateway channel normalization can reject an external channel missing from the active registry.

## Solution Assessment
