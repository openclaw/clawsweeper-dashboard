## What This Changes

The issue reports that `openclaw channels login` can save channel auth but leave the channel stuck in SETUP when the running gateway rejects the post-save `channels.start` call for an unloaded channel plugin.

## Best Possible Solution

Keep this issue open until one accepted channel-login lifecycle fix lands on main with proof for an external plugin login path, then close it against the merged PR and route any remaining startup-planner gap to its own canonical thread.

## Reproduction Assessment

No live high-confidence reproduction was run in this read-only review. The source-level path is clear: current main still calls `channels.start` after login, and gateway channel normalization can reject an external channel missing from the active registry with the reported error.

## Solution Assessment

Unclear as a final product choice: the restart fallback in the open linked PR is a plausible narrow mitigation, but maintainers still need to decide whether automatic safe restart from channel login or an earlier startup/plugin-loading repair is the best canonical fix.
