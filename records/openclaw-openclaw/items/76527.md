## Summary

Keep open: the handler change is plausible, but this PR is not merge-ready because it expands `/steer` syntax without synchronizing the command contract and still needs maintainer choice between sole-subagent shorthand and the main-session queue semantics raised in #76512 and #34881.

## What This Changes

The PR adds a targetless `/steer` fallback that auto-selects the sole active subagent when no explicit target is present or target resolution fails, plus two unit tests.

## Best Possible Solution

Settle the `/steer` grammar and land code, registry/help docs, changelog, and regression tests that all describe one command contract.

## Reproduction Assessment

Yes for the reported parser behavior: current main treats the first token after `/steer` as the subagent target and returns usage or target-resolution errors before steering. I did not run tests because this review was required to keep the checkout read-only.

## Solution Assessment

No. The code path may solve the one-active-subagent case, but it is not the best complete fix until maintainers choose the targetless `/steer` semantics and the command registry, help text, docs, changelog, and tests are synchronized.
