## Summary

Keep open. The handler/test fallback is plausible, but this PR changes the public `/steer` command grammar without updating the shared command registry, runtime help, public docs, or changelog, and the related discussion still needs maintainer confirmation on targetless `/steer` semantics.

## What This Changes

The PR adds a `/steer <message>` fallback that auto-selects the sole active subagent when no explicit target can be parsed, plus two steer-action unit tests.

## Best Possible Solution

Choose the targetless `/steer` grammar, then land the handler, command registry/help, docs, changelog, and focused regression coverage as one coherent command contract.

## Reproduction Assessment

Yes for the reported parser behavior: current-main source rebuilds Telegram native `/steer please check logs` as `/steer please check logs`, then the shared handler treats `please` as the subagent target and fails before steering. It is not a strict documented-contract bug because current docs define `/steer` as requiring a subagent target.

## Solution Assessment

Unclear. The PR's sole-active-subagent fallback is a plausible implementation for the linked report, but it is not the best complete fix until maintainers confirm the targetless grammar and the registry/help/docs/changelog are updated to match.
