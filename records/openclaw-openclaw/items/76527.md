## Summary

Keep this PR open: the implementation is small and relevant, but it changes the shared `/steer` command grammar while the command registry, docs, and changelog still describe only explicit subagent targets; the linked discussion also shows this syntax needs maintainer confirmation before landing.

## What This Changes

The PR adds a fallback in subagent steer handling that auto-selects the only active subagent for targetless or unresolved `/steer` input and adds two unit tests.

## Best Possible Solution

Choose the `/steer` grammar explicitly, then align the handler, command registry, Telegram/native command surfaces, docs, changelog, and regression coverage around that contract.

## Reproduction Assessment

Yes, from source inspection: current main parses the first token after `/steer` as a subagent target and requires a separate message before steering dispatch. I did not run a live Telegram session in this read-only review.

## Solution Assessment

Unclear. The patch is a plausible narrow interpretation for exactly one active subagent, but it does not settle or document the broader `/steer` grammar that the linked issue and older shorthand request raise.
