## Summary

Keep open: this draft, protected-label feature PR has useful terminal proof and focused tests, but the diff introduces blocking issues in handoff revocation authorization and browser-session transport normalization.

## What This Changes

The PR adds a shared Talk session controller and new Gateway Talk RPCs, moves realtime, transcription, handoff, native, and browser Talk surfaces onto shared primitives, and updates SDK, native bindings, docs, tests, and changelog coverage.

## Best Possible Solution

Keep the shared Talk controller direction, but require bearer/admin authorization for handoff revocation, normalize legacy provider transports before responses, and add focused regression coverage plus a live media smoke before merge.

## Reproduction Assessment

Not applicable for the feature as a whole. The blocking review findings are source-reproducible from the PR diff: revoke is id-only, and browser sessions are returned without transport normalization.

## Solution Assessment

No, not yet. The proposed architecture is plausible, but the current implementation leaves an authorization gap and a protocol compatibility gap that should be fixed before maintainer review can treat it as merge-ready.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Require bearer auth before revoking handoffs:** `src/gateway/server-methods/talk.ts:1444`
  - body: `talk.handoff.join` and the turn methods require the bearer token, but revoke accepts only the handoff id and immediately calls `revokeTalkHandoff`. The id is embedded in the room URL/result, so any authenticated client that learns that URL can tear down another room without the bearer token; require the token or an admin/owner scope before revoking.
  - confidence: 0.86
- **[P2] Normalize provider transports before returning sessions:** `src/gateway/server-methods/talk.ts:1636`
  - body: The new schema only accepts canonical `webrtc`/`provider-websocket`, and the migration docs say legacy `webrtc-sdp`/`json-pcm-websocket` names should normalize at adapter boundaries. This handler returns `createBrowserSession()` unchanged, so external or mixed-version providers returning old transport names produce successful responses that fail the advertised protocol schema; normalize/copy the session before comparing and responding.
  - confidence: 0.83

## Security Review

Status: needs_attention

Summary: Needs attention: the new handoff revoke RPC invalidates rooms by id without the bearer-token or admin check used by the rest of the handoff flow.
