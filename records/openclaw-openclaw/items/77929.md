## Summary

Keep open: this draft XL feature PR is still not merge-ready because the latest head keeps an id-only managed-room revoke path and can return browser session transports that violate the new canonical protocol surface.

## What This Changes

The branch adds a shared Talk session runtime, Gateway-managed `talk.session.*` RPCs, compatibility adapters across realtime/transcription/handoff/native/plugin/UI surfaces, and matching SDK, docs, generated client, and changelog updates.

## Best Possible Solution

Keep the shared Talk runtime direction, but add revoke authorization, normalize and validate browser provider session transports, add focused regressions, and complete maintainer plus live-media review before merge.

## Reproduction Assessment

Not applicable for the feature as a whole; yes for the blockers from source inspection. The latest PR head still has id-only revoke schema/handler code and an unchanged browser provider response path.

## Solution Assessment

No, not yet. The architecture may be the right direction, but the current patch leaves a narrow authorization gap and a protocol compatibility gap that should be fixed before treating it as merge-ready.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Require bearer auth before revoking handoffs:** `src/gateway/server-methods/talk.ts:748`
  - body: `talk.handoff.join` and all turn methods require the bearer token, but revoke validates only `id` and immediately calls `revokeTalkHandoff`. Since the handoff id is exposed in room metadata/URLs, any authenticated client that learns it can close another managed room; require the token or an admin/owner authorization path before revoking.
  - confidence: 0.86
- **[P2] Normalize provider transports before responding:** `src/gateway/server-methods/talk.ts:940`
  - body: The new protocol/type surface advertises only canonical `webrtc` and `provider-websocket`, but this handler returns provider `createBrowserSession()` output unchanged. A mixed-version or external provider that still returns `webrtc-sdp` or `json-pcm-websocket` can produce a successful response that violates the advertised schema; normalize a copied session before comparing and responding.
  - confidence: 0.83
