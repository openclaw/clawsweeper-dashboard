## Summary

Keep open: current main and the latest release expose context usage in status rows and selected-session UI, but they still do not define a native alert path for saturated terminal `done` sessions. The issue is recent, related context-pressure reports remain open, and the durable fix needs a maintainer decision about alert surface, threshold, and recovery/cleanup behavior.

## What This Changes

The issue asks OpenClaw to detect context-saturated sessions even when their state is `done`, so dead agents do not remain undetected.

## Best Possible Solution

Define one native context-pressure contract for persisted sessions that covers active and terminal rows, names the threshold and user-visible surface, and adds regression coverage without automatic cleanup unless maintainers explicitly choose that policy.
