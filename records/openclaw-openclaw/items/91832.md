## What This Changes

The issue requests native monitoring for context-saturated sessions in `done` state so dead agents are detected instead of running unresponsive for hours.

## Best Possible Solution

Define one native context-saturation contract for session monitoring that covers terminal `done` and active sessions, names the threshold and user-visible surface, and adds regression coverage without automatic cleanup unless maintainers explicitly choose that policy.

## Reproduction Assessment

## Solution Assessment

Unclear. Treating saturated `done` sessions as alert-worthy is plausible, but maintainers should first choose the durable surface and threshold semantics instead of adding an ad hoc monitor or cleanup shim.
