## Summary

Keep open: the screenshot matches a current Control UI rendering path where live tool messages keep their original timestamps but are appended after all history messages, so newer assistant groups can appear above older tool groups.

## What This Changes

The issue reports that Telegram-originated task messages can appear with non-monotonic timestamps in OpenClaw Control, with newer timestamps shown above older ones.

## Best Possible Solution

Keep transcript order canonical where possible, but render live history, stream segments, and tool cards in a monotonic visible order for the active chat without changing Telegram delivery semantics.

## Reproduction Assessment

No high-confidence live current-main reproduction was run. The screenshot plus source path gives a focused source repro: pass buildChatItems a newer history assistant message and an older live tool message, and the tool item is appended below the newer history.

## Solution Assessment

Unclear until a regression test is written, but the likely narrow fix is to interleave rendered chat items by canonical event order or timestamp while preserving canvas lifting and history pagination behavior.
