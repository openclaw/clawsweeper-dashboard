## What This Changes

The issue reports newer message timestamps appearing above older ones in Telegram and OpenClaw Control on v2026.5.10-beta.3.

## Best Possible Solution

Add a focused ordering invariant for displayed Control messages that preserves transcript semantics, then separately verify whether Telegram has the same timestamp source or a distinct delivery-order bug.

## Reproduction Assessment

No high-confidence live reproduction yet. The screenshot plus source inspection give a clear Control UI source path when a live tool message with an older timestamp is appended below newer history, but Telegram remains unverified.

## Solution Assessment

Unclear. The likely fix is to reconcile live tool/stream items into display order without losing transcript order, but sorting every message by timestamp would be riskier than first locking down the exact Control and Telegram paths.
