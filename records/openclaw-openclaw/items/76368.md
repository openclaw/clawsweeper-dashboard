## Summary

Keep open. Current main still omits the memory-core runtime config schema, and this PR is a narrow, appropriate fix with regression coverage and no blocking review findings found.

## What This Changes

The PR wires the memory-core manifest config schema into the runtime plugin entry, adds schema regression tests, and adds a changelog fix entry.

## Best Possible Solution

Runtime entry validation and packaged manifest validation should share the same memory-core config schema so documented dreaming settings are accepted and typos remain rejected.

## Reproduction Assessment

Yes by source inspection. Current main omits `configSchema` on the memory-core runtime entry, the SDK default rejects non-empty plugin config, and the docs direct users to set non-empty `plugins.entries.memory-core.config.dreaming` values.

## Solution Assessment

Yes. Reusing the existing manifest schema through `buildJsonPluginConfigSchema` is the narrowest maintainable fix because it avoids a second schema while preserving strict unknown-key rejection.
