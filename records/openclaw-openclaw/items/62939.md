## Summary

Keep open. Current main has useful targeted hardening for web fetches, inbound marker spoofing, and external-content wrappers, but it still lacks the requested shared trust-boundary contract for non-owner current messages, generic tool results, and stable prompt guidance; the narrow linked PR is closed unmerged and the broader CaMeL PR remains open.

## What This Changes

Request adds structural trust delimiters and prompt guidance for externally sourced tool results, web/file/API content, and non-owner messages, plus optional config and injection-pattern validation.

## Best Possible Solution

Adopt or explicitly supersede this with a core prompt-boundary design that covers stable prompt guidance, non-owner current-message marking, open-world tool-result classification, delimiter-safe framing, and provider regression tests.

## Reproduction Assessment

Yes. A source-level reproduction is to pass non-owner prompt text or a `toolResult` containing injection text through the current transform/provider conversion paths; current main preserves it as ordinary provider text without the requested shared boundary.

## Solution Assessment

Unclear. Structural data/instruction separation is directionally valid, but the exact wrapper layer and tag design need maintainer/security direction because delimiter escaping, provider payload shapes, and the CaMeL overlap are unresolved.
