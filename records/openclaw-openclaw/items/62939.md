## Summary

Keep open. Current main has useful source-specific external-content hardening, but the central shared boundary for non-owner prompt text, generic tool results, and cache-stable model guidance is still missing; the narrow implementation PR was closed unmerged and the broader CaMeL PR remains open.

## What This Changes

The issue requests a shared prompt-injection boundary contract that marks untrusted tool results, incoming non-owner messages, web/API/file content, and persisted transcript data as data rather than instructions.

## Best Possible Solution

Adopt or explicitly supersede this with a maintainer-approved prompt-boundary design covering stable guidance, delimiter-safe framing, non-owner current-message semantics, generic tool-result/source classification, and cross-provider regression coverage while preserving existing external-content wrappers.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: send non-owner prompt text or a generic toolResult containing instruction-looking content through the current transform and OpenAI/Anthropic conversion paths, and it remains ordinary provider text apart from invalid-surrogate sanitization.

## Solution Assessment

Unclear. Structural data/instruction separation is directionally appropriate, but the exact solution needs maintainer/security choice because delimiter safety, provider payload shapes, replay timing, and the open CaMeL proposal all affect the maintainable design.
