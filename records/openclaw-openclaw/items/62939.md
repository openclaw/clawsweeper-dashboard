## Summary

Keep open. Current main has partial prompt-injection hardening for external-content envelopes and sanitizer tests, but the core request remains unresolved: non-owner current prompts and generic provider-bound user/tool-result replay still lack a shared trust-boundary contract and stable system prompt guidance, and the related CaMeL PR is open rather than shipped.

## What This Changes

The issue requests core prompt-injection defenses that classify externally sourced messages and tool results as untrusted data at model-context boundaries, with stable prompt guidance and validation for suspicious injection patterns.

## Best Possible Solution

Define or explicitly supersede a core prompt-boundary design that covers stable guidance, source trust classification, escape-safe framing, current-turn and replay behavior, provider payload shapes, and regression tests across the existing external-content sanitizer paths.
