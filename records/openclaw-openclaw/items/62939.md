## Summary

Keep open. Current main still has only targeted prompt-injection hardening; it does not ship the requested shared trust-boundary contract for non-owner current messages, generic tool results, or stable system-prompt guidance, and the related narrow PR is closed unmerged while the broader CaMeL PR remains open.

## What This Changes

The issue requests structural trust boundaries for externally sourced prompt content, including non-owner messages, tool results, external content, and system-prompt guidance.

## Best Possible Solution

Land or explicitly supersede a core-owned prompt-injection boundary design with stable pre-cache guidance, non-owner prompt marking, generic tool-result classification, delimiter escaping, and provider-shape regression tests.

## Reproduction Assessment

Yes. A source-level reproduction is to pass external-user prompt text or a toolResult containing injection text through the current transform/provider conversion paths; current main preserves it as ordinary provider text without the requested shared boundary. A live model exploit would be nondeterministic, but the missing boundary behavior is directly observable.

## Solution Assessment

Unclear. The requested structural separation is directionally valid, but the exact XML-like wrapper approach is not the best proven fix until delimiter escaping, provider transcript shape, prompt-cache placement, and the relationship to CaMeL are settled.
