## Summary

Keep open: the useful Feishu UX change is not on current main, but this PR still has blocking correctness defects and lacks required real Feishu behavior proof before merge.

## What This Changes

The branch adds a hardcoded 8-second delayed Feishu p2p/private-message acknowledgment in `extensions/feishu/src/bot.ts` and Feishu bot tests for slow, fast, and group-chat cases.

## Best Possible Solution

Land a Feishu-local revision that sends the delayed acknowledgment only after a real no-visible-output interval, cancels or suppresses it when visible Feishu output starts, covers slow/fast/group/early-visible cases deterministically, and leaves the cross-channel config/API design to https://github.com/openclaw/openclaw/issues/8285.

## Reproduction Assessment

Yes, at source level for the PR defects. The diff cancels the delayed acknowledgment only after dispatcher settlement while current Feishu replies can become visible before settlement, and the fast-path test awaits a 9-second mock after advancing only 7,999 ms.

## Solution Assessment

No. The Feishu-local direction is plausible, but this implementation keys the delay to dispatcher settlement instead of visible reply start/delivery, and one added test is not deterministic under fake timers.

Overall confidence: 0.89
