## Summary

Keep open: current main still lacks this Feishu delayed-ack feature, but the PR has blocking source-level defects and no after-fix real behavior proof from a real Feishu setup.

## What This Changes

The branch adds an 8-second delayed acknowledgment for Feishu p2p/private-message dispatches and bot tests for slow, fast, and group-chat paths.

## Best Possible Solution

Land a Feishu-local revision that sends the delayed acknowledgment only after a real no-visible-output interval, suppresses it as soon as visible reply delivery starts, keeps final reply flow intact, and proves the behavior with deterministic tests plus redacted real Feishu evidence.

## Reproduction Assessment

Yes, at source level. A Feishu p2p/private dispatch that emits visible output before 8 seconds but settles after 8 seconds leaves this PR's timer active, and the fast-path fake-timer test awaits a 9-second mock after advancing only 7,999 ms.

## Solution Assessment

No. The Feishu-local boundary is reasonable, but cancellation tied only to outer dispatcher settlement is too late for the PR body's no-user-visible-reply condition; the safer path should hook the ack cancellation into reply start or visible delivery and make the fast test deterministic.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Cancel the delayed ack when visible replies start:** `extensions/feishu/src/bot.ts:1833-1848`
  - body: The timer is canceled only after `withReplyDispatcher` settles, but Feishu reply delivery can begin via `onReplyStart`/`deliver` and send streaming cards, structured cards, text, or media before settlement. A long direct-message run that already showed output can still send the delayed ack at 8s, violating the no-user-visible-reply condition and confusing users.
  - confidence: 0.9
- **[P2] Make the fast delayed-ack test settle before awaiting:** `extensions/feishu/src/bot.test.ts:537`
  - body: This case advances fake timers to 7,999 ms and then awaits `pending`, while the shared dispatch mock resolves after 9,000 ms. Under fake timers the promise cannot settle, so the test can hang instead of proving pre-threshold cancellation.
  - confidence: 0.88
