## Summary

Keep open: the Feishu-local feature is useful, but the current patch still sends the delayed ack based on dispatcher settlement rather than first visible output, has a fake-timer test that can hang, and lacks required real Feishu behavior proof.

## What This Changes

The branch adds an 8-second delayed acknowledgment for Feishu p2p/private-message dispatches plus Feishu bot tests for slow, fast, and group-chat behavior.

## Best Possible Solution

Land a Feishu-local revision that sends the delayed ack only after a real no-visible-output interval, suppresses it as soon as visible Feishu output starts or is delivered, preserves final replies, and leaves the cross-channel config/API design to https://github.com/openclaw/openclaw/issues/8285.

## Reproduction Assessment

Yes, at source level for the PR defects. The diff cancels the delayed ack only after dispatcher settlement while current Feishu replies can become visible before settlement, and the fast-path test awaits a 9-second mock after advancing only 7,999 ms.

## Solution Assessment

No. The Feishu-local scope is plausible, but this implementation keys the delay to dispatcher settlement instead of visible reply start/delivery, and one added test is not deterministic under fake timers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Cancel delayed ack when visible replies start:** `extensions/feishu/src/bot.ts:1833-1863`
  - body: The timer is canceled only after `withReplyDispatcher` settles, but Feishu can start or deliver streaming/card/plain-text/media replies before settlement. A long DM that already produced visible output can still receive the delayed ack at 8s, contradicting the no-visible-reply condition and creating duplicate messaging.
  - confidence: 0.9
- **[P2] Use a fast dispatcher in the fast-path test:** `extensions/feishu/src/bot.test.ts:537`
  - body: The fast direct-message case advances fake timers by only 7,999 ms and then awaits `pending`, while the shared dispatcher mock resolves after 9,000 ms. Under fake timers that promise cannot settle there, so the test can hang instead of validating fast-path cancellation.
  - confidence: 0.88
