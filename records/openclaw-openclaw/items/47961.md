## Summary

Keep open: current main does not contain this Feishu delayed-ack feature, and the PR still has source-confirmed correctness and test blockers before it should merge.

## What This Changes

The PR adds an 8-second delayed Feishu p2p/private-message acknowledgment timer in `extensions/feishu/src/bot.ts` plus bot tests for slow, fast, and group-chat paths.

## Best Possible Solution

Land a Feishu-local revision that sends the delayed acknowledgment only for p2p/private chats after a real no-visible-output interval, suppresses it as soon as visible Feishu reply delivery starts, preserves final reply flow, and includes deterministic tests plus the required changelog entry.

## Reproduction Assessment

Yes. Source inspection of the PR head shows a slow Feishu p2p/private dispatch that emits visible reply output before 8 seconds but settles after 8 seconds can still send the extra ack; the fast-path fake-timer test can also hang before its shared 9-second mock resolves.

## Solution Assessment

No. The Feishu-local boundary is appropriate, but cancellation tied only to dispatcher settlement is later than the PR body's no-visible-output condition; the safer fix should hook into the reply lifecycle or delivery path and make the fast test deterministic.
