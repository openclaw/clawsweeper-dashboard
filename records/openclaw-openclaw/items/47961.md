## Summary

Keep open: current main still lacks the requested Feishu delayed acknowledgment behavior, but this PR cannot merge as-is because it can send the extra acknowledgment after visible output has already started and one added test can hang under fake timers.

## What This Changes

The PR branch adds an 8-second Feishu p2p/private-message delayed acknowledgment timer and bot tests for slow, fast, and group-chat paths.

## Best Possible Solution

Land a Feishu-local revision that sends the delayed acknowledgment only after a real no-visible-output interval, cancels or suppresses it as soon as visible Feishu reply delivery starts, preserves final reply flow, and includes deterministic tests plus the required changelog entry.

## Reproduction Assessment

Yes. By inspection, a slow Feishu p2p/private dispatch that starts streaming/card/plain-text/media output before 8 seconds but settles after 8 seconds would still send the PRŌĆÖs delayed ack; the fast-path test can also hang because it awaits a 9-second fake-timer dispatch after advancing only 7.999 seconds.

## Solution Assessment

No. The Feishu-local boundary is appropriate, but tying cancellation only to dispatcher settlement is later than the PRŌĆÖs stated no-visible-output condition; the safer repair should connect cancellation to the reply lifecycle or delivery path.
