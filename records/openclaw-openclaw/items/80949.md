## Summary

Keep this PR open, but the current patch should not merge as written: the prompt and audit pieces are aligned with the existing tool-only contract, while the runtime fallback changes privacy-sensitive delivery behavior by posting normal final text that current docs and maintainer notes treat as private.

## What This Changes

The PR adds message-tool-only prompt guidance, suppressed-final diagnostics, and a runtime metadata path that auto-delivers normal final replies when no message-tool send was recorded.

## Best Possible Solution

Keep the prompt guidance and privacy-safe audit, but make same-turn auto-delivery diagnostic-only unless maintainers explicitly approve a gated fallback with live channel proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: group/channel turns resolve to message_tool_only, suppress normal final delivery, and current dispatch skips those finals unless an internal bypass marker is present; I did not run a live channel repro in this read-only review.

## Solution Assessment

No, not as written. The guidance and audit warning are narrow, but default auto-delivery of normal finals changes the privacy contract and should be warning-only or explicitly gated by maintainer-approved policy/config.
