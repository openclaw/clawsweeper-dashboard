## Summary

Keep open: the PR addresses a real message-tool-only visibility failure, but the runtime fallback changes the current privacy contract by making normal finals visible without an explicit policy decision.

## What This Changes

The PR adds message-tool-only prompt guidance, suppressed-final diagnostics, fallback metadata, dispatch bypass logic, tests, and prompt snapshots to recover missed normal finals when no message-tool delivery evidence exists.

## Best Possible Solution

Keep the prompt and privacy-safe diagnostic parts, but only add same-turn visible fallback behind an explicit maintainer-approved policy or config gate with live channel proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current main resolves group/channel turns to message_tool_only and suppresses ordinary final payloads when the agent does not call the message tool. I did not run a live channel repro in this read-only review.

## Solution Assessment

No, not as written. The guidance and audit warning are narrow, but auto-delivering normal finals in tool-only mode is broader than the current privacy policy unless maintainers explicitly approve or gate it.
