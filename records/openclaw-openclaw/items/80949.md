## Summary

Keep open: the PR is a plausible fix direction, but the new runtime fallback appears to weaken the existing message-tool-only privacy contract by posting normal final text that current prompts say should stay private. It also needs fresh real behavior proof after the fallback force-push.

## What This Changes

The PR strengthens message-tool-only prompt guidance, emits a privacy-safe diagnostic for suppressed finals, and adds a guarded fallback that can visibly deliver normal final payloads when no message-tool delivery evidence exists.

## Best Possible Solution

Keep the prompt and diagnostic pieces, but only land runtime recovery after maintainers choose an explicit privacy policy boundary, likely a narrower or config-gated fallback with live channel proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path for the original failure: current main resolves group/channel message-tool-only turns to suppressed delivery and skips ordinary final payloads when the agent did not call the message tool. I did not run a live Discord or Telegram reproduction in this read-only review.

## Solution Assessment

No. The diagnostic and guidance are narrow, but auto-delivering every normal final without message-tool evidence is broader than the current privacy contract and needs a maintainer policy decision or tighter guard.
