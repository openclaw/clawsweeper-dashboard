## Summary

Keep open: the underlying silent final path is source-reproducible, but this PR should not merge as written because its fallback can visibly post normal final text on tool-only channels without an explicit visible-delivery policy decision.

## What This Changes

The PR adds message-tool-only prompt guidance, suppressed-final diagnostics, and a same-turn final-delivery fallback when no committed message-tool send is recorded.

## Best Possible Solution

Keep the prompt guidance and privacy-safe warning diagnostic, then decide any final-text fallback separately as an explicit policy-gated behavior with live channel proof.

## Reproduction Assessment

Yes. Source inspection shows group/channel turns resolve to message_tool_only, suppress normal final delivery, and skip unmarked final payloads; related Slack, Telegram, and Discord reports provide live examples.

## Solution Assessment

No, not as written. The prompt and audit pieces are narrow, but default auto-delivery of normal finals changes the documented privacy contract and should be warning-only or explicitly policy-gated.
