## What This Changes

The PR emits a warning diagnostic event when substantive normal final replies are suppressed by message-tool-only delivery and adds focused reply-dispatch tests.

## Best Possible Solution

Land one canonical privacy-preserving diagnostic path for suppressed message-tool-only finals, preserving the no-auto-forward contract and avoiding suppressed final text in logs or diagnostic payloads.

Yes. Source inspection on current main shows group/channel delivery resolves to `message_tool_only`, sets `suppressDelivery`, and skips ordinary unmarked final replies; related Telegram and Slack reports provide live examples of the same class.

## Solution Assessment

Yes, this PR is a narrow maintainable direction because it emits aggregate diagnostic metadata rather than forwarding or logging private final text. The remaining maintainer decision is whether this diagnostic-event surface should be preferred over the overlapping verbose-log implementation.
