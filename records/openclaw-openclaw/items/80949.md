## Summary

Keep open: the prompt and diagnostic parts are useful, but the runtime fallback still crosses the existing message-tool-only privacy boundary and needs explicit maintainer privacy/product review plus stronger real-channel proof before merge.

## What This Changes

The PR adds message-tool-only prompt guidance, diagnostic warnings for suppressed substantive finals, fallback metadata for missed normal finals, and dispatch/tests that deliver marked final payloads despite source suppression.

## Best Possible Solution

Keep the prompt and privacy-safe diagnostic improvements, but redesign runtime recovery behind an explicit visible-delivery policy or config gate before any normal final can be posted visibly.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: group/channel message-tool-only turns resolve to suppressed delivery, and current dispatch skips ordinary final payloads when the agent did not call the message tool.

## Solution Assessment

No, not as written. The guidance and audit warning are narrow, but marking ordinary finals for visible fallback without an explicit policy decision is broader than the current privacy contract.
