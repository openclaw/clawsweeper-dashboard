## Summary

## What This Changes

## Best Possible Solution

Land this as a memory-core bugfix after the FTS indexed-text format is part of index identity or migration so existing clean indexes rebuild automatically.

## Solution Assessment

No. Prefixing the path into new FTS rows is the right local mechanism, but the PR needs an upgrade/reindex trigger so the fix applies to existing clean indexes too.
