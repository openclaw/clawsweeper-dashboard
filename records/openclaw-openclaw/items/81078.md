## What This Changes

The issue reports that WhatsApp block streaming can suppress the complete final assistant reply after only a partial streamed block was delivered.

## Best Possible Solution

Review and land the open shared reply-pipeline fix, or an equivalent narrow change, so successful block streaming suppresses only payloads proven delivered while preserving media follow-up behavior.

## Reproduction Assessment

Yes. Source inspection shows current main drops text-only final payloads when `didStream()` is true and `isAborted()` is false, and the adjacent test still codifies that drop even when the pipeline says the payload was not sent.

## Solution Assessment

Yes. The best path is a narrow shared reply-payload filter fix rather than a WhatsApp-only workaround, because the suppression happens in generic final-payload construction used by the WhatsApp path.
