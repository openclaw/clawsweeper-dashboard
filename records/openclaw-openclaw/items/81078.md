## What This Changes

The issue reports that WhatsApp block streaming can suppress a complete final assistant reply after only partial streamed content was delivered.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/93079 or an equivalent repair that preserves unsent text-only finals while still dropping finals proven delivered by the block pipeline.

## Reproduction Assessment

Yes, source-reproducible: a successful block stream with a different final text payload reaches the branch that drops text-only payloads before `hasSentPayload` is checked. I did not run the live WhatsApp path in this read-only review.

## Solution Assessment

No, current main is not the best way to solve the issue because it treats any successful partial stream as equivalent to full final delivery. The maintainable fix is to suppress only payloads proven delivered while preserving the existing media follow-up behavior.
