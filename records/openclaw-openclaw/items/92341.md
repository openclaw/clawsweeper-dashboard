- **[P2] Split CJK fallback terms before exact LIKE matching:** `extensions/memory-core/src/memory/manager-search.ts:126-127`
  - body: The patch routes every CJK token to `substringTerms`, but still pushes a multi-character query such as `飞书插件配置` as one exact substring. The PR body reports that this shape can still produce `textScore: 0`, and https://github.com/openclaw/openclaw/issues/92728 tracks the exact-phrase LIKE precision problem, so this needs a chosen split-term strategy before it closes the bug.
  - confidence: 0.9
- **[P2] Keep empty-MATCH fallback as a stricter filter:** `extensions/memory-core/src/memory/manager-search.ts:387`
  - body: The existing exception fallback requires every fallback term to match, but this new empty-MATCH branch switches to OR matching and then assigns fallback `textScore = 1`. If MATCH returns no rows because one term is unindexable, rows containing only one query token can now be treated as fully relevant memory hits.
  - confidence: 0.86
- **[P2] Preserve keyword-only fallback entries after overlap merges:** `extensions/memory-core/src/memory/hybrid.ts:143-153`
  - body: When a keyword hit overlaps a vector hit, this code merges it into the vector entry and drops the original keyword-only row. If the combined entry later falls below `minScore` or its expanded range no longer matches the manager's keyword key, the lexical hit can still be omitted; https://github.com/openclaw/openclaw/pull/92524 keeps this fix at the manager selection boundary.
