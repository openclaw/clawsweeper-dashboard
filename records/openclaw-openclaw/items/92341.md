## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Split CJK fallback terms before exact LIKE matching:** `extensions/memory-core/src/memory/manager-search.ts:117-118`
  - body: The patch routes every CJK token to `substringTerms`, but still pushes a multi-character query such as `飞书插件配置` as one exact substring. The PR body reports that example still gets `textScore: 0`, and https://github.com/openclaw/openclaw/issues/92728 shows exact-phrase LIKE can match issue-discussion files instead of relevant partial phrases, so this needs a chosen split-term strategy before it closes the bug.
  - confidence: 0.9
- **[P2] Keep empty-MATCH fallback as a stricter filter:** `extensions/memory-core/src/memory/manager-search.ts:371`
  - body: The existing exception fallback requires every query token to match, but this new empty-MATCH branch switches to OR matching and then assigns fallback `textScore = 1`. If MATCH returns no rows because one term is unindexable, rows containing only one query token can now be treated as fully relevant memory hits.
  - confidence: 0.86
- **[P2] Preserve keyword-only fallback entries after overlap merges:** `extensions/memory-core/src/memory/hybrid.ts:143-153`
  - body: When a keyword hit overlaps a vector hit, this code merges it into the vector entry and does not keep the original keyword-only row. If the combined score remains below `minScore`, `MemoryManager.search` filters relaxed lexical fallback by the original keyword source/path/line key, so the mutated vector-range entry can be dropped even though the lexical hit should be preserved.
  - confidence: 0.85
