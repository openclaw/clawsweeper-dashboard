## Summary

Keep open. ## What This Changes

The PR changes memory-core keyword planning, empty-MATCH fallback search, and hybrid result merging so CJK or short-token keyword hits can contribute `textScore` when FTS MATCH returns no rows or chunk IDs differ.

## Best Possible Solution

Repair this branch or a narrow replacement with an owner-approved CJK term decomposition strategy, shared LIKE fallback semantics, preserved keyword fallback behavior, focused regression tests, and a green lint lane before landing.

## Solution Assessment

No. The PR is a plausible partial fix, but the whole-token CJK LIKE path, OR empty-MATCH fallback, and overlap-merge interaction need correction before this is the best maintainable solution.

- **[P2] Split CJK fallback terms before exact LIKE matching:** `extensions/memory-core/src/memory/manager-search.ts:118`
  - body: The patch routes every CJK token to `substringTerms`, but still pushes a multi-character query such as `飞书插件配置` as one exact substring. The PR body records that this case still returns `textScore: 0`, and the linked follow-up describes wrong exact-phrase matches, so this needs an owner-approved split-term strategy with coverage before closing the CJK bug.
  - confidence: 0.91
- **[P2] Keep empty-MATCH fallback aligned with AND semantics:** `extensions/memory-core/src/memory/manager-search.ts:371`
  - body: The existing exception fallback and tests require every query token to match the row, but this new empty-MATCH branch switches to `OR`, giving fallback rows a full `textScore` when they contain only one token from a multi-token query. Reuse the existing AND-style fallback semantics or add a deliberately tested ranking strategy.
  - confidence: 0.84
- **[P2] Preserve keyword fallback after overlap merges:** `extensions/memory-core/src/memory/hybrid.ts:143-145`
  - body: When a keyword hit overlaps a vector hit, this code merges it into the vector entry and does not keep a keyword-only entry. If the combined score is still below `minScore`, `MemoryManager.search` filters by the original keyword source/path/line key, so the merged vector-range entry can be dropped even though the keyword hit should be preserved.
  - confidence: 0.86
- **[P3] Add braces around the range updates:** `extensions/memory-core/src/memory/hybrid.ts:144-145`
  - body: The new single-line range-update `if` statements are in the current diff and the live lint checks fail on `eslint(curly)`. Use braced blocks so the extension lint gate can pass.
