- **[P2] Split CJK fallback terms before exact LIKE matching:** `extensions/memory-core/src/memory/manager-search.ts:118`

  - confidence: 0.9
- **[P2] Keep empty-MATCH fallback as a stricter filter:** `extensions/memory-core/src/memory/manager-search.ts:371`
  - body: The existing exception fallback requires every query token to match the row, but this new empty-MATCH branch switches to OR. If MATCH returns no rows because one token is unindexable, rows containing only one token can now get fallback textScore=1, which can inject unrelated memory chunks into recall context.
  - confidence: 0.84
- **[P2] Preserve keyword-only fallback entries after overlap merges:** `extensions/memory-core/src/memory/hybrid.ts:143-153`
  - body: When a keyword hit overlaps a vector hit, this code merges it into the vector entry and does not keep the original keyword-only entry. If the combined score remains below minScore, MemoryManager.search filters the relaxed fallback by the original keyword source/path/line key, so the mutated vector-range entry can be dropped even though the lexical hit should be preserved.
  - confidence: 0.85
