- **[P2] Split CJK fallback terms before exact LIKE matching:** `extensions/memory-core/src/memory/manager-search.ts:118`
  - body: The patch routes every CJK token to substringTerms, but still pushes a multi-character query as one exact substring. The PR body reports a CJK query that remains textScore=0, and the linked follow-up explains that exact phrase LIKE can match issue-discussion files instead of relevant partial phrases.
  - confidence: 0.9
- **[P2] Keep empty-MATCH fallback as a stricter filter:** `extensions/memory-core/src/memory/manager-search.ts:371`
  - body: The existing exception fallback requires every query token to match the row, but this new empty-MATCH branch switches to OR and then assigns fallback textScore=1. If MATCH returns no rows because one token is unindexable, rows containing only one token can now enter memory recall as fully relevant.
  - confidence: 0.84
- **[P2] Preserve keyword-only fallback entries after overlap merges:** `extensions/memory-core/src/memory/hybrid.ts:143`
  - body: When a keyword hit overlaps a vector hit, this code merges it into the vector entry and drops the original keyword-only row. MemoryManager.search's relaxed fallback is keyed to the original keyword source/path/line range, so the lexical hit can be lost even though it should remain available.
  - confidence: 0.85
