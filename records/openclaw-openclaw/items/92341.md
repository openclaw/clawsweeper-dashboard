Full review comments:

- **[P2] Split CJK fallback terms before exact LIKE matching:** `extensions/memory-core/src/memory/manager-search.ts:118`
  - body: The patch routes every CJK token to `substringTerms`, but still pushes a multi-character query such as `飞书插件配置` as one exact substring. The PR body says that query still gets `textScore: 0`, and the related open issue reports this can match issue-discussion files instead of relevant partial phrases, so this should use an owner-approved split-term plan with coverage before closing the CJK bug.
  - confidence: 0.9
- **[P2] Avoid broad OR fallback for empty MATCH queries:** `extensions/memory-core/src/memory/manager-search.ts:371`
  - body: The existing exception fallback and tests require each query token to match the row, but this new empty-MATCH branch switches to `OR`, giving fallback `textScore: 1` to rows that contain only one token from a multi-token query. That can inject unrelated memory chunks whenever MATCH returns no rows; share the existing fallback semantics or add a deliberately tested ranking strategy.
  - confidence: 0.84
- **[P2] Preserve keyword fallback after overlap merges:** `extensions/memory-core/src/memory/hybrid.ts:143-149`
  - body: When a keyword hit overlaps a vector hit, this code merges it into the vector entry and does not keep a keyword-backed entry with the original source/path/line key. If the combined score is still below `minScore`, `MemoryManager.search`'s relaxed fallback can filter the merged row out even though the keyword hit should be preserved.
  - confidence: 0.84
- **[P3] Add braces around range updates:** `extensions/memory-core/src/memory/hybrid.ts:144-145`
  - body: The new single-line range-update `if` statements fail the live lint jobs with `Expected { after 'if' condition`; use braced blocks so the extension lint gate can pass.
  - confidence: 0.95
