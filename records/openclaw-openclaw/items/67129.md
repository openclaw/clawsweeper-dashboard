## Summary

Keep open: current main still sends only a static Slack assistant-thread status/clear call and exposes no config/schema/docs for custom status or `loading_messages`. The request is valid, but it needs maintainer API/product choice because related status work and prior closed PRs used competing config shapes.

## What This Changes

Request to add Slack config for custom assistant-thread `status` and `loading_messages` values passed to `assistant.threads.setStatus`.

## Best Possible Solution

Choose one Slack assistant-status contract, then add typed and validated Slack config, pass `status` plus optional `loading_messages` through the active Slack plugin wrapper, and update focused Slack tests and https://docs.openclaw.ai/channels/slack.

## Solution Assessment
