## Summary

Current main still has the reported singleton eviction path, and an open closing PR already owns the keyed-pool fix path; this issue should stay open until that PR is corrected and merged or closed.

## What This Changes

The issue asks OpenClaw to replace the Codex app-server shared-client singleton with per-key pooling so agents with different agentDir values do not evict each other's warm clients.

## Best Possible Solution

Land or revise the linked keyed-pool fix so agentDir remains in the compatibility key while shared-client lifecycle preserves cleanup for auth, env, and other incompatible start-option rotations.

## Reproduction Assessment

Yes by source inspection: current main deterministically creates distinct keys for distinct agentDir values, then clears the only global shared client when the key changes. I did not run a live Raspberry Pi or multi-agent gateway workload.

## Solution Assessment

Yes as a direction: keyed pooling is the narrow maintainable fix because it preserves per-agent HOME/CODEX_HOME/auth isolation while avoiding cross-agent eviction. The currently linked PR still needs to preserve stale auth/env/start-option cleanup before it is the best implementation.
