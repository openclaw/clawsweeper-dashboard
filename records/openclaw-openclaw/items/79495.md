## Summary

Current main still has the reported singleton eviction behavior, so this should stay open as a narrow Codex plugin bug rather than be swept closed.

## What This Changes

The issue requests changing the Codex app-server shared client from a single global key/client slot to per-start-options-key reuse so different agent directories do not evict one another.

## Best Possible Solution

Keep agentDir in the compatibility key, replace the singleton with a keyed shared-client pool, add regression coverage for alternating agent dirs, and treat compact dynamic-tool fingerprints as a separate follow-up unless the same patch can keep it small.

## Reproduction Assessment

Yes by source inspection: current main deterministically creates distinct keys for distinct agentDir values, then clears the only global shared client when that key changes. I did not run a live Raspberry Pi or multi-agent gateway scenario.

## Solution Assessment

Yes: a keyed pool is the narrow maintainable fix because it preserves per-agent HOME/CODEX_HOME/auth isolation while avoiding cross-agent eviction. Removing agentDir from the key would be riskier because docs and auth bridging depend on per-agent isolation.
