The issue asks OpenClaw to stop Codex app-server shared-client key changes for different agentDir values from evicting warm clients across agents.

Resolve or supersede https://github.com/openclaw/openclaw/pull/79513 against the shipped keyed-registry implementation, then keep startup retry/cold-spawn and CPU overhead work on https://github.com/openclaw/openclaw/issues/83959 and https://github.com/openclaw/openclaw/issues/84037.

## Reproduction Assessment

## Solution Assessment

Yes for the original bug: pooling by the existing full compatibility key is the narrow maintainable fix because it preserves per-agent auth/CODEX_HOME isolation without cross-agent eviction. The remaining release symptoms should be split to the narrower startup and CPU follow-ups rather than reopening the singleton design.
