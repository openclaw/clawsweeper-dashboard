## Summary

Current main still has the reported cross-agent singleton eviction behavior in the Codex app-server shared client, so this should stay open as a narrow Codex plugin bug rather than be swept closed.

## What This Changes

The issue requests replacing the Codex harness app-server's single shared-client singleton with a keyed pool so compatible per-agent clients are reused without evicting each other.

## Best Possible Solution

Keep agentDir in the compatibility key, replace the singleton with a keyed shared-client pool with bounded idle cleanup, and add regression coverage for alternating agent dirs without mixing in dynamic-tool fingerprint compaction.

## Reproduction Assessment

Yes by source inspection: current main deterministically creates distinct keys for distinct agentDir values, then clears the only global shared client when that key changes. I did not run a live Raspberry Pi or multi-agent gateway scenario.

## Solution Assessment

Yes for the requested keyed-pool direction: it preserves per-agent HOME/CODEX_HOME/auth isolation while avoiding cross-agent eviction. The related open PR's per-run isolation addresses a concurrent kill path, but it may trade away warm per-agent reuse and is not current-main behavior.
