Current main still has the reported Codex app-server singleton eviction path, and the issue should stay open because an open linked PR already targets the fix with GitHub closing syntax.

## What This Changes

The issue requests replacing the Codex app-server single shared-client slot with a keyed pool so agents with distinct `agentDir` values do not evict each other.

## Best Possible Solution

Review and land the linked keyed-pool PR after preserving stale auth/env/start-option cleanup, while treating compact dynamic-tool fingerprints as a separate follow-up unless explicitly scoped.

## Reproduction Assessment

Yes. Source inspection on current main shows distinct `agentDir` values produce distinct keys and a key mismatch clears the only shared client; the linked duplicate also supplies live overlapping-agent failure evidence.

## Solution Assessment

Yes. A keyed shared-client pool is the narrow maintainable direction because it preserves per-agent Codex home/auth isolation while preventing cross-agent eviction; the implementation should also keep the existing stale secret/start-option cleanup contract.
