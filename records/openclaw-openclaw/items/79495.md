## What This Changes

The issue asks for the Codex app-server shared client to become a compatibility-keyed pool so different agent directories stop evicting each other while preserving per-agent isolation.

## Best Possible Solution

Review and land the keyed shared-client pool in https://github.com/openclaw/openclaw/pull/79513, keep agentDir in the key, and handle compact dynamic-tool fingerprints as separate follow-up work.

## Reproduction Assessment

Yes. Source inspection on current main shows distinct agentDir values produce distinct compatibility keys, and any key change clears the single global shared client slot.

## Solution Assessment

Yes. Pooling by the existing full compatibility key is the narrow maintainable fix because it preserves per-agent CODEX_HOME/auth isolation while avoiding cross-agent eviction.
