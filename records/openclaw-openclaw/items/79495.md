## Summary

Keep open: current main still has the singleton cross-agent eviction path, and the open closing PR is unmerged with a focused review blocker, so this issue remains the canonical tracker.

## What This Changes

The issue asks for the Codex app-server shared client to reuse compatible clients per key instead of evicting warm clients when agents have different `agentDir` values.

## Best Possible Solution

Land a keyed Codex app-server client pool that preserves per-agent isolation and stale auth/env/start-option cleanup, using https://github.com/openclaw/openclaw/pull/79513 or a narrow replacement.

## Reproduction Assessment

Yes. Current-main source inspection shows distinct `agentDir` values produce distinct keys and a key mismatch clears the only shared client; the closed duplicate report adds live overlapping-agent failure logs.

## Solution Assessment

Yes, the keyed-pool direction is the right narrow fix because it keeps per-agent `HOME`/`CODEX_HOME` isolation while avoiding cross-agent eviction. The open PR still needs to preserve existing auth/env/start-option rotation cleanup before merge.
