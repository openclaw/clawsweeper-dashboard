Full review comments:

- **[P1] Scope the new language rules:** `src/security/skill-scanner.ts:182-186`
  - body: Most new Python and shell rules have no `includeExtensions`, so they run on every already-scanned JS/TS file. Critical matches in strings or comments can fail closed during plugin or skill install; scope these rules to the intended language and add false-positive coverage.
  - confidence: 0.9
- **[P2] Scan extensionless shell entrypoints:** `src/security/skill-scanner.ts:48-50`
  - body: Adding only `.sh` and `.bash` keeps discovery dependent on `path.extname()`, so common shebang entrypoints such as `bin/install` or `postinstall` remain skipped by directory and forced-file scans. Detect shell shebangs or declared entrypoints so the shell rules cover real scripts.
  - confidence: 0.91
- **[P2] Match non-literal Python command payloads:** `src/security/skill-scanner.ts:186`
  - body: The `os.system`/`os.popen`/`os.exec*` branch only matches quoted first arguments, so `cmd = ...; os.system(cmd)` bypasses the critical command-exec finding. Match non-literal payloads or add a broader source rule for these calls.
  - confidence: 0.88
