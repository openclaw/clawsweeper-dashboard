  - body: Most new Python and shell rules have no include-extension guard, so they also run on already-scanned JS/TS files.   - confidence: 0.9

  - body: Adding only `.sh` and `.bash` keeps discovery dependent on `path.extname()`, so common shebang entrypoints such as `bin/install` or `postinstall` remain skipped by directory and forced-file scans. Detect shell shebangs or declared entrypoints so the shell rules cover real scripts.
  - confidence: 0.91

  - body: The `os.system`/`os.popen`/`os.exec*` branch only matches quoted first arguments, so `cmd = ...; os.system(cmd)` bypasses the critical command-exec finding. Match non-literal payloads or add a broader source rule for these calls.
  - confidence: 0.88
