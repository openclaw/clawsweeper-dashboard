## Summary

Keep open: current main still lacks the requested Python/shell scanner coverage, but this external security PR has concrete scanner accuracy gaps and lacks real behavior proof, so it needs revision and security-owner review before merge.

## What This Changes

The PR extends the skill scanner to include `.py`, `.sh`, and `.bash` files and adds Python/shell detection rules with focused tests.

## Best Possible Solution

Land a consolidated scanner-hardening change that covers Python and shell skill scripts, extensionless shebang entrypoints, language-scoped rules, non-literal Python command execution, and regression tests for bypasses and false positives.

## Reproduction Assessment

Yes, via source inspection. Current main only scans JS/TS-family extensions, while the PR head remains extension-gated and has unscoped language rules that can be reasoned about directly in the diff.

## Solution Assessment

No. Extending the scanner is the right surface, but this branch should be revised or consolidated so the rules are language-scoped, cover extensionless shell entrypoints, and catch variable Python command payloads.
