## Summary

Keep open: the scanner expansion is useful and is not already on current main, but the PR still has blocking scanner coverage/accuracy issues and lacks after-fix real behavior proof from the contributor.

## What This Changes

The branch expands the skill scanner to scan `.py`, `.sh`, and `.bash` files and adds Python/shell detection rules plus focused scanner tests.

## Best Possible Solution

Land a consolidated scanner-hardening patch that covers Python and shell skill scripts, extensionless shebang entrypoints, language-scoped rules, variable Python command execution, and regression tests for bypasses and false positives.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main excludes Python and shell files, while the PR head still leaves extensionless shell entrypoints and `os.system(cmd)`-style Python payloads outside the new coverage.

## Solution Assessment

No. Extending the scanner is the right surface, but this branch should be revised or consolidated so the rules are language-scoped, handle extensionless shell entrypoints, and catch non-literal Python command execution.
