## Summary

Keep open. Current main has not implemented Python or shell skill scanning, and this external PR still contains concrete security-rule bypass and false-positive risks plus missing real behavior proof, so it needs revision and security-owner review rather than cleanup closure.

## What This Changes

The PR expands the skill scanner to scan `.py`, `.sh`, and `.bash` files and adds Python/shell detection rules plus focused scanner tests.

## Best Possible Solution

Land a consolidated scanner-hardening patch that covers Python and shell skill scripts, extensionless shebang entrypoints, language-scoped rules, variable Python command execution, and regression tests for bypasses and false positives.

## Reproduction Assessment

Yes. Source inspection shows current main only scans JS/TS-family extensions, while the PR head still skips extensionless shell entrypoints and misses `os.system(cmd)`-style Python payloads.

## Solution Assessment

No. Extending the scanner is the right surface, but this branch should be revised or consolidated so the rules are language-scoped, handle extensionless shell entrypoints, and catch non-literal Python command execution.
