## What This Changes

This PR extends `src/security/skill-scanner.ts` to scan `.py`, `.sh`, and `.bash` files, adds Python and shell detection heuristics, and expands scanner tests.

## Best Possible Solution

Land a secops-reviewed scanner update that language-scopes rules, detects extensionless shebang entrypoints and variable Python command execution, includes focused regression tests, and has redacted real scan proof.

## Reproduction Assessment

Yes for the PR-review findings: source inspection shows current main is extension-gated to JS/TS and the PR head still uses extension-only shell discovery plus literal-only `os.*` matching. I did not run tests because this was a read-only review.

## Solution Assessment

No. Extending the scanner is the right surface, but this branch should be revised so new rules are language-scoped, cover extensionless shell entrypoints, catch non-literal Python command execution, and land after security-owner review.
