## Review Findings

Overall confidence: 0.86

Full review comments:

- **[P1] Scope the new language rules:** `src/security/skill-scanner.ts:186`
  - body: Only `dynamic-code-execution-python` is extension-gated; the other new Python and shell rules run against every scannable JS/TS file. Because built-in critical findings block installs, a JS/TS file containing a shell snippet such as `eval "$cmd"` in a string can become an install-blocking false positive. Add file-type or shebang-aware guards for every new language rule and regression tests for JS/TS strings or comments.
  - confidence: 0.86
- **[P2] Scan extensionless shell entrypoints:** `src/security/skill-scanner.ts:49-50`
  - body: Adding `.sh` and `.bash` still misses common shebang scripts like `bin/install` or `postinstall`, because `isScannable()` continues to rely on `path.extname()`. Include shebang/interpreter detection or forced entrypoint scanning so shell coverage cannot be bypassed by removing the extension.
  - confidence: 0.84
- **[P2] Match variable Python command payloads:** `src/security/skill-scanner.ts:186`
  - body: The `os.system`/`os.popen`/`os.exec*` arm only reports quoted literal first arguments, so `cmd = payload; os.system(cmd)` avoids the new critical rule. Match non-literal call payloads, or add a source-level command-execution rule with false-positive coverage.
  - confidence: 0.82
