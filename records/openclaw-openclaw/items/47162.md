The branch extends `src/security/skill-scanner.ts` to scan `.py`, `.sh`, and `.bash` files, adds Python and shell heuristics, and expands scanner tests.

## Best Possible Solution

Land a security-owner-reviewed scanner update that language-scopes rules, detects extensionless shebang entrypoints, catches variable Python command execution, includes focused regression tests, and has redacted real scan proof.

## Reproduction Assessment

Yes for the PR-review findings: source inspection shows current main is extension-gated to JS/TS and the PR head still uses extension-only shell discovery plus literal-only `os.*` matching. I did not run tests because this was a read-only review.

## Solution Assessment

No. Extending the scanner is the right surface, but this branch should be revised to language-scope the new rules, cover extensionless shell entrypoints, catch non-literal Python command payloads, and land after security-owner review.
