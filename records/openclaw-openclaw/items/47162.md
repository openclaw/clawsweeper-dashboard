## What This Changes

The PR extends the skill scanner to `.py`, `.sh`, and `.bash` files, adds Python and shell detection rules, and adds focused scanner tests.

## Best Possible Solution

Land a consolidated scanner-hardening patch that covers Python and shell skill scripts, extensionless shebang entrypoints, language-scoped rules, variable Python command execution, and regression tests for bypasses and false positives.

## Reproduction Assessment

Yes for source-level reproduction. Current main still scans only JS/TS-family extensions, and source inspection of the PR head shows the extensionless shell and variable-payload Python gaps.

## Solution Assessment

No. Extending the scanner is the right surface, but this branch should be revised or consolidated so the new rules are language-scoped, cover extensionless shell entrypoints, and catch non-literal Python command execution.
