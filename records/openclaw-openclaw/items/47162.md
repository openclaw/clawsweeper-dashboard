## What This Changes

The PR extends the skill code-safety scanner to scan `.py`, `.sh`, and `.bash` files, adds Python and shell detection rules, and adds focused scanner tests.

## Best Possible Solution

Revise or replace the branch with a security-reviewed scanner update that scopes rules by language, covers shebang/extensionless shell entrypoints, detects variable Python command execution, and includes focused tests plus real scan proof.

## Reproduction Assessment

Yes, for source-level review findings: current main still scans only JS/TS-family extensions, and the PR head still skips extensionless shell entrypoints and misses variable Python command payloads. I did not run tests because this was a read-only review.

## Solution Assessment

No. Extending the scanner is the right surface, but this branch should be revised so the new rules are language-scoped, cover shebang entrypoints, catch non-literal Python execution, and land after security-owner review.
