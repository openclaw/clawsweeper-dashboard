Keep open: the PR targets a real linked nodes screenshot truncation bug, but it is not merge-ready because it changes the explicitly enabled raw invoke result contract, mishandles `screen.record` media, and lacks after-fix real node/Gateway proof.

## What This Changes

The PR adds a `nodes` tool invoke-result sanitizer that writes large direct `payload.base64` values to temp files and returns a `FILE:` text result instead of raw JSON.

## Best Possible Solution

Share the existing dedicated media parsing and file-writing contracts for each media command, preserve raw invoke compatibility unless maintainers explicitly approve a FILE-result contract change, and add focused tests plus real node/Gateway proof.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection and the linked issue show allowed raw invoke can return inline base64 that is vulnerable to tool-result truncation, but the Windows node/Gateway scenario was not executed here.

## Solution Assessment

No, not as submitted. Offloading large base64 is plausible, but the patch is in a compatibility-sensitive raw invoke path and routes `screen.record` through snapshot helpers instead of the established recording helpers.
