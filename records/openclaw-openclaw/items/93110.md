## Summary

Keep open: the lease-store direction matches maintainer feedback, but the patch is still incorrect because it stores the lease under `agentSessionKey` while completion lookups use requester/run keys, retires the lease before delayed harness/native completions can retry, and still has only test/build proof for a user-visible delivery bug.

## What This Changes

The PR adds an in-memory delivery-route lease store, wires subagent and harness announce-origin lookup to it, registers/retires leases from isolated cron runs, and adds lease-store plus test-plan coverage.

## Best Possible Solution

Register the route lease under the actual requester run key and retire it from the shared completion/delivery lifecycle only after supported background completions settle, with delayed-completion regression coverage and redacted live route proof.

## Reproduction Assessment

Yes, source-reproducible: current main creates a distinct run requester key for default cron execution and passes it into the harness, while this PR stores the lease under `agentSessionKey` and patched completion lookup uses `requesterSessionKey`. I did not run a live channel roundtrip in this read-only review.

## Solution Assessment

No. The lease-store direction matches the maintainer-requested design, but this implementation is not the best fix because its keying and retirement do not match the completion lifecycle it is meant to support.
