## Summary

Keep this PR open, but it is not merge-ready: the lease-store direction matches the maintainer-requested shape, while this implementation still keys and retires the route in ways that can drop the same completion messages it is meant to recover.

## What This Changes

The branch adds a process-local delivery lease store for isolated cron completion routes, wires announce-origin lookup to it, registers/retires leases from cron, and adds lease-store plus test-plan coverage.

## Best Possible Solution

Register the route lease under the actual requester run key and retire it from the shared completion/delivery lifecycle only after all supported background completions settle, with delayed-completion regression coverage and redacted live route proof.

## Solution Assessment

No. The lease-store direction matches the maintainer-requested design, but this implementation is not the best fix because keying and retirement do not match the completion lifecycle it is meant to support.
