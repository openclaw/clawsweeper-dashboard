## Summary

Keep open: the branch addresses the likely missing isolated-cron route propagation, but it still has no production retirement hook for the new route leases and the supplied proof does not show a real gateway/webchat completion reaching the controller.

## What This Changes

## Best Possible Solution

Own requester-route leases in the shared background-task completion/delivery lifecycle, retire them after final delivery settles, preserve delayed-completion coverage, and add redacted live gateway/webchat proof before merge.

## Reproduction Assessment

Yes from source and linked issue evidence: current main lacks a delivery-route fallback when the isolated cron requester session has no deliveryContext, and the linked issue gives a concrete webchat failure shape. I did not run a live gateway/webchat reproduction in this read-only review.

## Solution Assessment

No. The generic lease store is a plausible direction, but this PR is not the best fix until lease retirement is owned by the shared final-delivery lifecycle instead of TTL/cap cleanup.
