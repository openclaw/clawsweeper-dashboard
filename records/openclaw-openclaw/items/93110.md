## Summary

Keep this PR open, but not merge-ready: the run-key lease lookup fixes the right seam, yet the current TTL-only lifecycle can expire valid delayed completions and has no production retirement after completion settles.

## What This Changes

The PR adds a process-local delivery route lease store, registers isolated cron delivery targets under the run session key, lets subagent and harness completion lookup fall back to that store, and adds lease-store plus CI shard tests.

## Best Possible Solution

Keep the run-key lease direction, but make the lease owned by the shared completion/delivery lifecycle, keep it valid through supported delayed completions, retire it after final delivery settles, and add real route proof.

## Reproduction Assessment

Yes for the PR defect: source inspection shows leases are created before execution, expire after one hour, and have no production retirement call. I did not run a live cron/webchat roundtrip in this read-only review.

## Solution Assessment

No. The run-key lease lookup is the right direction, but TTL-only cleanup is not the best fix for a lifecycle-owned delivery route and can fail supported delayed completions.

- **[P1] Keep route leases through completion lifecycle:** `src/infra/delivery-lease-store.ts:31-39`
  - body: This lease is registered before the cron run starts, but the store prunes it after one hour.
