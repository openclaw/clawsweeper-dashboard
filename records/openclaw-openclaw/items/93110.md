## Summary

Keep open: this is a meaningful fix attempt for the isolated-cron completion-routing bug, but it is not merge-ready because the route lease is still TTL/cap-retained rather than retired by the shared completion lifecycle, the current head has a PR-introduced lint failure, and the supplied proof does not show a live gateway/webchat delivery.

## Best Possible Solution

Own the requester-route lease in the shared background completion/delivery lifecycle, retire it after final delivery settles, keep delayed-completion coverage, fix the lint error, and add live gateway or webchat proof before merge.
