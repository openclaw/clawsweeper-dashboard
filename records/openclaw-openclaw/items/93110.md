## Summary

Keep this PR open because it addresses the right isolated-cron delivery seam, but PR head still does not implement the maintainer-requested lifecycle-owned route lease: production code registers leases and looks them up, but never retires them after final delivery settles, leaving delivery/session routing governed by a 52h TTL and 2000-entry cap.

## What This Changes

The PR adds a process-local delivery route lease store, registers isolated cron delivery targets under the run session key, passes that key into subagent and harness completion routing, and adds lease-store plus CI shard tests.

## Best Possible Solution

Wire the route lease into the shared background completion/delivery lifecycle so leases survive supported delayed completions and retire immediately after final delivery settles, then add real route proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Retire route leases from the completion lifecycle:** `src/infra/delivery-lease-store.ts:54-62`
  - body: This store evicts the oldest active lease whenever the process exceeds 2000 entries, but PR head never calls `retireDeliveryLease` from production; completed isolated cron runs therefore keep route leases for 52h. A burst of completed runs can fill the cap and evict a still-active delayed completion route before `resolveAnnounceOrigin` looks it up, falling back to the channel-less session entry and reproducing the delivery failure. The lease needs a shared background completion/delivery retirement hook after final delivery settles, not TTL/cap-only cleanup.
  - confidence: 0.9
