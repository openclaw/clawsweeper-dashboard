## Summary

This PR adds a process-local delivery route lease store, wires cron/subagent/harness announcement resolution to look up requester routes by run session key, and adds infra tests, a repro script, and a CI shard expectation.

## Best Possible Solution

Own the requester-route lease in the shared background completion/delivery lifecycle, retire it after final delivery settles, keep delayed-completion coverage, fix lint, and add redacted live gateway or webchat proof before merge.

## Solution Assessment

No: the generic lease lookup is a plausible direction, but TTL/cap-only cleanup is not the best fix. The predecessor maintainer direction calls for shared lifecycle-owned retirement after final delivery settles.
