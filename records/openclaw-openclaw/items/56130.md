  - confidence: 0.98
- **[P1] Use message in invoke error payloads:** `apps/visionos/Sources/NodeManager.swift:160-161`

  - confidence: 0.95
- **[P2] Register APNs through node.event:** `apps/visionos/Sources/APNsManager.swift:59-62`
  - body: The Gateway APNs registration path is node.event with event push.apns.register; node.apns is not a supported Gateway RPC method. As written, APNs token publishing will be rejected and wake/notification setup will not work.
  - confidence: 0.94
- **[P2] Restrict handshake error handling to connect responses:** `apps/visionos/Sources/GatewaySocket.swift:165-176`
  - body: This treats every ok:false response as a handshake failure. Later RPC errors can legitimately return ok:false, and those would incorrectly flip the node into an error state even though the socket is still healthy.
  - confidence: 0.86
- **[P2] Guard delayed reconnects from active sessions:** `apps/visionos/Sources/NodeManager.swift:132-135`
  - body: scheduleReconnect always calls connect() after sleeping, even if a manual reconnect has already succeeded. Since connect() starts by disconnecting the current socket, the delayed task can drop a healthy connection.
  - confidence: 0.86
- **[P2] Prevent stale disconnect tasks from closing new sockets:** `apps/visionos/Sources/NodeManager.swift:117-120`
  - body: disconnect() waits for sendExitFrame() inside an async task before calling socket.disconnect(). If the user reconnects during that wait, the old task can close the new socket when it resumes.
  - confidence: 0.84
