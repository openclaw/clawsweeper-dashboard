Full review comments:

- **[P1] Add the missing build target metadata:** `apps/visionos/README.md:17`
  - body: The PR adds Swift source files and docs, but no `.xcodeproj`, `Package.swift`, workspace, or project generator. A fresh checkout cannot open `visionOS-node/visionOS-node.xcodeproj` as the README instructs, so reviewers and users cannot build the app from this branch.
  - confidence: 0.95
- **[P1] Decode `paramsJSON` before dispatch:** `apps/visionos/Sources/NodeManager.swift:261-263`
  - body: Gateway invoke events carry command arguments in `paramsJSON`; this handler only reads `payload["params"]`. Any command that relies on parameters will run with empty/default input, and the claimed local approval-field validation will not inspect the actual serialized payload.
  - confidence: 0.96
- **[P1] Send `error.message` in invoke failures:** `apps/visionos/Sources/NodeManager.swift:160-162`
  - body: `node.invoke.result` validates errors as `{ code, message }` with no extra properties. Building errors with `detail` makes failure responses invalid, so callers can time out or lose the intended diagnostic instead of receiving the node error.
  - confidence: 0.94
- **[P2] Register APNs through `node.event`:** `apps/visionos/Sources/APNsManager.swift:59-62`
  - body: The gateway does not advertise or handle a `node.apns` RPC method; APNs registrations are accepted as `node.event` with `event: "push.apns.register"`. Sending the token to `node.apns` will be rejected and leaves wake/notification registration unconfigured.
  - confidence: 0.92
- **[P2] Do not treat every failed response as a handshake error:** `apps/visionos/Sources/GatewaySocket.swift:165-176`
  - body: After connect, ordinary RPCs can return `ok:false`; this parser sends every failed `res` frame to `socketDidReceiveHandshakeError`, which flips the UI state to error even when the socket remains healthy. Restrict this path to the connect response or track request ids.
  - confidence: 0.86
- **[P2] Guard stale reconnect and disconnect tasks:** `apps/visionos/Sources/NodeManager.swift:132-135`
  - body: `scheduleReconnect()` reconnects after its sleep without checking whether a later manual connect already succeeded, and `disconnect()` closes the shared socket after awaiting `sendExitFrame()`. These delayed tasks can tear down a healthy replacement connection after quick reconnects.
  - confidence: 0.84
