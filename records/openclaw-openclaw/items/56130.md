  - body: The quick start tells users to open `visionOS-node/visionOS-node.xcodeproj`, but the PR only adds Swift source files, an Info.plist, README, and docs. A fresh checkout cannot build or run the app from the branch, so the app is not mergeable until a project/package/spec target is included.

- **[P1] Decode paramsJSON before dispatch:** `apps/visionos/Sources/NodeManager.swift:261-263`
  - body: Current gateway `node.invoke.request` events carry command arguments in `payload.paramsJSON`, but this handler reads `payload["params"]` and falls back to `{}`. Commands that depend on input will run with empty/default params, and the claimed params-smuggling validation will not see the real request body.
  - confidence: 0.94
- **[P1] Use message in invoke error payloads:** `apps/visionos/Sources/NodeManager.swift:160-161`
  - body: `node.invoke.result` only accepts `error.code` and `error.message`; `additionalProperties: false` rejects the `detail` key sent here. Error paths can therefore fail schema validation and leave callers with timeouts or opaque failures instead of the intended error.
  - confidence: 0.91
- **[P1] Register APNs through node.event:** `apps/visionos/Sources/APNsManager.swift:59-61`
  - body: The gateway method list includes `node.event`, and APNs registration is handled as `push.apns.register`; `node.apns` is not a supported RPC method. This token publish will be rejected, so APNs-backed wake/notification flows will not be configured.
  - confidence: 0.9
- **[P2] Limit handshake errors to connect responses:** `apps/visionos/Sources/GatewaySocket.swift:165-176`
  - body: This treats every `ok:false` response frame as a handshake rejection. After the node is connected, any ordinary RPC error response can incorrectly flip the UI/session to an error state even though the WebSocket is still usable.
  - confidence: 0.84
- **[P2] Guard delayed reconnects before reconnecting:** `apps/visionos/Sources/NodeManager.swift:132-135`
  - body: A scheduled reconnect always calls `connect()` after the sleep unless the task itself was cancelled. If a manual reconnect succeeds first, the stale task can still disconnect the healthy socket because `connect()` begins by calling `disconnect()`.
  - confidence: 0.82
- **[P2] Prevent stale disconnect tasks from closing new sockets:** `apps/visionos/Sources/NodeManager.swift:117-119`
  - body: `disconnect()` sends the exit frame in an async task and then calls `socket.disconnect()`. If the user reconnects quickly, the older task can complete later and close the new WebSocket session.
  - confidence: 0.78
