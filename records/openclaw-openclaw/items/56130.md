## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Add build metadata for the visionOS app:** `apps/visionos/README.md:17`
  - body: The README tells users to open `visionOS-node/visionOS-node.xcodeproj`, but the PR only adds Swift sources, an Info.plist, README, and docs. A fresh checkout has no project, package, entitlements, or target metadata to build the app, so this cannot land as a first-party app until the build surface is included.
  - confidence: 0.93
- **[P1] Decode paramsJSON before dispatch:** `apps/visionos/Sources/NodeManager.swift:261-263`
  - body: Current Gateway `node.invoke.request` events carry command arguments in `paramsJSON`, but this handler reads `payload["params"]`. That drops every command argument and also makes the smuggled-approval check run against an empty/default params object instead of the real payload.
  - confidence: 0.95
- **[P1] Use message in invoke error payloads:** `apps/visionos/Sources/NodeManager.swift:158-162`
  - body: `node.invoke.result` errors are schema-checked as `{ code?, message? }` with no additional properties, but `sendError` writes `detail`. Error paths like permission denied, background unavailable, and unknown commands will send invalid result frames and can leave callers with timeouts or opaque failures.
  - confidence: 0.94
- **[P2] Register APNs through node.event:** `apps/visionos/Sources/APNsManager.swift:59-62`
  - body: The gateway does not expose a `node.apns` method; APNs registration is handled as `node.event` with event `push.apns.register`. Sending this token through `node.apns` will be rejected, so the documented wake/notification path will not configure.
  - confidence: 0.9
- **[P2] Restrict handshake error handling to connect responses:** `apps/visionos/Sources/GatewaySocket.swift:165-176`
  - body: This treats every `ok:false` response frame as a handshake failure and flips the node to error. After the socket is connected, normal request failures can also return `ok:false`, so a healthy session can be marked failed by an unrelated RPC error.
  - confidence: 0.84
- **[P2] Guard delayed reconnects from active sessions:** `apps/visionos/Sources/NodeManager.swift:132-135`
  - body: A scheduled reconnect sleeps and then calls `connect()` without checking whether a manual reconnect already succeeded. Because `connect()` begins by disconnecting the current socket, an old backoff task can drop a healthy session.
  - confidence: 0.86
- **[P2] Prevent stale disconnect tasks from closing new sockets:** `apps/visionos/Sources/NodeManager.swift:117-120`
  - body: `disconnect()` waits for `sendExitFrame()` in an async task and then calls `socket.disconnect()`. If the user reconnects before that wait finishes, the older task can still close the replacement socket.
  - confidence: 0.86
- **[P2] Nest allowCommands under gateway.nodes:** `docs/nodes/visionos.md:97-110`
  - body: The docs show a root-level `allowCommands` object, but the current config contract is `gateway.nodes.allowCommands`. Users following this snippet will not enable the visionOS commands they are trying to test.
  - confidence: 0.88
