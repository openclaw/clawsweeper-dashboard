  - body: Current gateway node invoke events carry command arguments in `paramsJSON`, but this handler reads only `payload["params"]`. Commands that rely on arguments will run with empty params, and the claimed approval/bypass validation does not inspect the current wire field.
  - confidence: 0.95
- **[P1] Use message in invoke error payloads:** `apps/visionos/Sources/NodeManager.swift:160-161`
  - body: `node.invoke.result` error payloads are validated against `error.code` and `error.message` with no extra properties. Sending `detail` makes error results schema-invalid, so callers can see timeouts or opaque failures instead of the intended command error.
  - confidence: 0.94
- **[P2] Register APNs through node.event:** `apps/visionos/Sources/APNsManager.swift:61`
  - body: The gateway handles APNs registration as `node.event` with `event: "push.apns.register"`; `node.apns` is not a supported gateway method. This token publish request will be rejected, so APNs wake/notification registration will not be configured.
  - confidence: 0.91
- **[P2] Guard stale reconnect tasks:** `apps/visionos/Sources/NodeManager.swift:132-136`
  - body: `scheduleReconnect` always calls `connect()` after its sleep. If the user manually reconnects successfully first, the stale task still fires and `GatewaySocket.connect()` begins by disconnecting the healthy socket.
  - confidence: 0.86
- **[P2] Avoid closing new sockets from old disconnects:** `apps/visionos/Sources/NodeManager.swift:117-120`
  - body: `disconnect()` waits asynchronously on `sendExitFrame()` and then calls `socket.disconnect()` on the shared socket. A quick reconnect can create a new task before the old shutdown task resumes, letting the old task close the new connection.
  - confidence: 0.84
