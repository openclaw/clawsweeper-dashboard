## Summary

Keep open. Current main and the latest release still have the dry-run/apply mismatch, and this PR contains a useful narrow fix, but the patch should not merge as-is because it checks public npm before knowing the actual install registry and still lacks full CLI dry-run proof.

## What This Changes

The PR makes exact semver `openclaw update --dry-run --tag ...` targets call the npm package target status helper, return unresolved on public `HTTP 404`, add a dry-run note, and add focused helper tests.

## Best Possible Solution

Land one canonical update CLI fix that checks confirmed missing public npm targets against the actual install registry before dry-run/apply, preserves custom registry and non-registry specs, and includes full dry-run proof.

## Reproduction Assessment

Yes. Source inspection shows current main resolves exact semver tags locally before dry-run returns, and a live registry check confirms a deliberately absent version-like target returns 404.

## Solution Assessment

No. This PR is a plausible narrow mitigation, but the public npm lookup lives in `resolveTargetVersion` before the command knows the effective install registry, so it is not the safest owner boundary for existing registry setups.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Align exact-version checks with the install registry:** `src/cli/update-cli/shared.ts:103`
  - body: This call always checks `registry.npmjs.org` before `update-command` has resolved the package manager, install spec, or custom registry. For a private or mirrored registry where `openclaw@2026.x` exists but public npm returns 404, `targetVersion` becomes `null`, `downgradeRisk` can become true, and JSON/non-TTY apply can stop before the package manager uses the configured registry. Keep the confirmed-404 check on the actual public-npm install path, or preserve the old fallback when the registry is unknown.
  - confidence: 0.84
- **[P3] Gate the propagation note on confirmed 404s:** `src/cli/update-cli/update-command.ts:3226`
  - body: This condition fires for any unresolved explicit registry target, including an explicit dist-tag whose lookup timed out or returned another inconclusive error. That would tell users apply will fail until npm propagation even though the probe result was transient; carry the lookup reason through or only emit this note for confirmed 404 exact targets.
  - confidence: 0.78
