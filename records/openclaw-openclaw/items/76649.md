## Summary

Keep open. Current main still has a source-level path where embedded PI runs can reuse a startup-scoped plugin registry that omits explicitly enabled non-bundled hook plugins, so the reported Feishu main-agent hook miss remains plausible and actionable.

## What This Changes

The issue reports that custom plugin hooks `before_prompt_build` and `agent_end` do not fire for Feishu main-agent embedded PI turns in v2026.5.2 while heartbeat harness turns do.

## Best Possible Solution

Embedded PI runs should load or reuse a runtime registry that includes explicitly enabled hook-capable plugins before hook lookup, without hardcoding Feishu or a plugin id.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: gateway startup stores a scoped startup plugin list, embedded PI reuses that list as `onlyPluginIds`, and the hook call sites only run when the resulting global hook runner has the hook. I did not perform a live Feishu reproduction in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is generic runtime plugin scoping or startup-plan inclusion for explicitly enabled hook plugins. A Feishu-specific workaround would solve the symptom at the wrong layer.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/76649. Current main can omit explicitly enabled non-bundled hook-only plugins from embedded PI runs because `ensureRuntimePluginsLoaded` reuses `snapshot.startup.pluginIds` as `onlyPluginIds`, and embedded hook call sites only see hooks registered in that scoped runtime registry. Implement a generic fix so embedded PI/runtime plugin loading includes explicitly enabled hook-capable plugins before `getGlobalHookRunner()` is used. Do not hardcode Feishu, `memory-tencentdb`, or any bundled plugin id, and do not broadly load every installed plugin if a narrower manifest/config-derived scope is available. Preserve startup registry laziness and third-party plugin behavior. Add regression coverage for a non-bundled plugin with `before_prompt_build` and `agent_end` enabled under a startup-scoped snapshot. Update `CHANGELOG.md` because this is a user-facing plugin hook regression fix.
