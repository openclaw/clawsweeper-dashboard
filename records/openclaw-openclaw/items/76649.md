## Summary

Keep open. Current main still has a source-reproducible startup-scope gap for embedded PI hook plugins, and #76684 is an open closing PR for the same root cause, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

External hook plugins configured with `before_prompt_build` and `agent_end` should fire for Feishu main embedded PI runs, not only harness or heartbeat runs.

## Best Possible Solution

Land or revise #76684 so gateway startup planning includes explicitly enabled hook-capable plugins generically, with regression coverage, and let GitHub close this issue after merge.

## Reproduction Assessment

Yes. Source inspection shows embedded PI reads the global hook runner after startup-scoped runtime loading, while current startup planning can omit explicitly configured hook-only plugins.

## Solution Assessment

Yes. The narrow maintainable fix is a generic startup-planning change for explicit hook capability or hook policy, not a Feishu-specific workaround.
