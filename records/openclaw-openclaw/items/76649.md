## Summary

Keep open. Current `main` has the embedded PI hook call sites, but source inspection shows the reported v2026.5.2 regression is plausible: embedded runs can reuse a startup-scoped runtime registry before reading the global hook runner, while the startup plan can omit explicitly enabled non-bundled hook-only plugins.

## What This Changes

The issue asks OpenClaw to fire custom plugin `before_prompt_build` and `agent_end` hooks for Feishu main-agent embedded PI runs the same way they fire for the heartbeat harness.

## Best Possible Solution

Ensure embedded PI runs load or reuse a runtime registry that includes explicitly enabled hook-capable plugins before `getGlobalHookRunner`, without hardcoding Feishu or a specific plugin id.

## Reproduction Assessment

Yes from source inspection: the embedded PI path checks the global hook runner, and the current startup-snapshot reuse path can omit explicit non-startup hook plugins. I did not run a live Feishu reproduction in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is to adjust runtime plugin loading or startup-plan inclusion so explicit hook plugins are present for embedded PI attempts. A Feishu-specific workaround would be the wrong layer.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/76649. A custom non-bundled plugin that registers `before_prompt_build` and `agent_end` with `allowPromptInjection` / `allowConversationAccess` should fire for Feishu/main embedded PI runs. Current source loads runtime plugins via `ensureRuntimePluginsLoaded` before `runEmbeddedPiAgent` reads `getGlobalHookRunner`, but when a current metadata snapshot exists it scopes to `snapshot.startup.pluginIds`; `gateway-startup-plugin-ids.ts` can exclude generic non-bundled hook-only plugins from that startup plan. Make the embedded PI runtime load or reuse a registry that includes explicitly enabled hook-capable plugins without hardcoding Feishu, `memory-tencentdb`, or bundled plugin ids, and preserve startup registry reuse where it is safe. Add focused regression coverage with a fixture non-bundled hook-only plugin explicitly enabled in config and policy flags, then verify both hooks are visible before embedded attempt hook checks. Review whether #76568 shares the same `agent_end` omission, but do not solve queued-message merging from #76632 or internal `message_sending` from #73542 in this PR. Add a single-line `CHANGELOG.md` fix entry because this is user-facing behavior.
