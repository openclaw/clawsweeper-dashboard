## Summary

Keep open for maintainer review: the PR addresses a real Chrome MCP cached-session race and I did not find a blocking line-level defect, but the patch changes waiter accounting, abort propagation, pending-session eviction, and close timing in a session lifecycle path.

## What This Changes

The PR replaces Chrome MCP shared pending attach promises with per-waiter pending-session leases/controllers, adds lifecycle race coverage, and adjusts one command test mock helper.

## Best Possible Solution

Land the lifecycle change after browser-owner acceptance with the focused regression tests, requesting a live Chrome MCP/CDP smoke only if maintainers want proof beyond the controlled cache race coverage.

## Reproduction Assessment

Yes. Current main creates the shared Chrome MCP pending attach with the first caller's AbortSignal, and the linked issue at https://github.com/openclaw/openclaw/issues/88304 gives a concrete two-waiter abort shape.

## Solution Assessment

Yes, with owner acceptance. Fixing the cache owner by separating the shared attach controller from per-caller abort semantics is the narrow maintainable solution; caller-side workarounds or config knobs would leave the shared promise semantics wrong.
