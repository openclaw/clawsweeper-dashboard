## Summary

Keep open: current main still prepares native subagent child sessions with model/provider metadata but no inherited runtime/auth-profile handoff, the latest release does not contain a relevant fix, and the related open OAuth propagation issue does not fully cover this issue's accepted-then-401 child-run behavior.

## What This Changes

The issue reports native `sessions_spawn` child runs being accepted and then failing asynchronously on an OpenAI route with `HTTP 401 Missing scopes: api.responses.write` instead of completing or failing at spawn time.

## Best Possible Solution

Define and implement the native subagent auth/runtime inheritance boundary so child runs inherit the intended parent route when inheritance is expected, or fail before acceptance with clear diagnostics.

## Reproduction Assessment

No, we do not have a high-confidence live current-main reproduction. The source path is reproducible from current main and matches the reporter logs, but the exact OAuth scope failure still needs a live OpenAI/Codex run.

## Solution Assessment

Unclear. Inheriting the parent effective runtime/auth context appears to be the right direction, but maintainers need to choose whether incompatible child routes should fail before acceptance or preserve the current non-blocking acceptance semantics with better routing.
