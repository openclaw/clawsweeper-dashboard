## Best Possible Solution

Decide and implement one owner-boundary fix that makes child OpenAI/Codex auth/profile selection match the parent effective route or rejects the spawn before acceptance when the child route lacks required scopes, with focused regression coverage for inherited and explicit `openai/gpt-5.4` spawns.
