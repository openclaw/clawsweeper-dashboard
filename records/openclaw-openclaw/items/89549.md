## What This Changes

The issue reports that `sessions_spawn(..., mode:"run")` accepts native child runs that later fail on an OpenAI `api.responses.write` auth-scope error.

## Best Possible Solution

Preserve the parent effective model/runtime/auth context for native child runs when inheritance is intended, or reject incompatible child routes before acceptance, with focused regression and live OpenAI/Codex OAuth proof.

## Reproduction Assessment

No high-confidence live reproduction was run in this read-only review. Source inspection and the reporter's logs still show a plausible current-main path where a child run is accepted with model metadata but without inherited auth/runtime context.

## Solution Assessment

Unclear. Repairing child model/runtime/auth inheritance looks like the safer direction, but maintainers need to choose whether incompatible child routes should fail before acceptance or inherit the parent effective route.
