## Summary

Keep open. Current main now enforces Feishu media byte caps earlier, but over-limit or failed inbound media is still reduced to a logged download failure while placeholder text can keep normal agent dispatch reachable with no Feishu-visible warning.

## What This Changes

The issue asks Feishu to notify the user and avoid misleading normal agent handling when inbound media is dropped because it exceeds `mediaMaxMb`.

## Best Possible Solution

Add a Feishu-owned media resolution result that reports dropped media to the same chat or thread and prevents a normal agent turn when all requested media was undelivered, while preserving text and successful attachments on partial failures.

## Reproduction Assessment

Yes, at source level. A Feishu media message over `mediaMaxMb` causes the media save/download path to throw, `resolveFeishuMediaList` catches and logs the failure, the placeholder content remains nonblank, and normal dispatch can proceed without a user-visible warning.

## Solution Assessment

Yes. A plugin-local structured failure plus Feishu-visible warning is the narrow maintainable fix because the Feishu plugin owns the reply target, media parsing, account config, and inbound dispatch decision.
