## Summary

Keep open. Current main improved Feishu media byte-cap enforcement, but oversized or failed inbound media is still caught as a log-only failure and can proceed to normal agent dispatch with placeholder text and no Feishu-visible warning.

## What This Changes

Report a user-facing Feishu error and avoid misleading agent execution when inbound media exceeds `mediaMaxMb` or otherwise cannot be downloaded.

## Best Possible Solution

Add a Feishu-owned media resolution result that reports dropped media to the same chat/thread and prevents normal agent turns when all requested media was undelivered, while preserving successful attachments on partial failures.

## Reproduction Assessment

Yes, at source level. A focused test can mock Feishu media download/save to throw for an over-limit file event with a filename, then assert current main logs the failure and still reaches dispatch because parsed placeholder content is nonblank.

## Solution Assessment

Yes. A plugin-local structured failure plus Feishu-visible warning is the narrow maintainable fix because the Feishu plugin owns the reply target, media parsing, account config, and inbound dispatch decision.
