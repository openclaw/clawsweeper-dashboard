## Summary

Keep open. Current main enforces the Feishu media byte cap during download, but failed or over-limit inbound media is still converted into a log-only drop, and placeholder text can still reach normal agent dispatch without a Feishu-visible warning.

## What This Changes

Ask the Feishu plugin to notify the user and avoid a normal agent turn when inbound media is dropped because it exceeds `mediaMaxMb`.

## Best Possible Solution

Add a Feishu-owned media resolution result that reports dropped media to the same chat/thread and blocks normal agent turns when all requested media was undelivered, while preserving successful attachments on partial failures.

## Reproduction Assessment

Yes, at source level. A focused test can mock Feishu media download/save to reject for an over-limit file with a filename, then assert the current path logs the failure, retains placeholder text, and still reaches normal dispatch with no media attachment.

## Solution Assessment

Yes. A plugin-local structured failure plus Feishu-visible warning is the narrow maintainable fix because the Feishu plugin owns the reply target, media parsing, account config, and inbound dispatch decision.
