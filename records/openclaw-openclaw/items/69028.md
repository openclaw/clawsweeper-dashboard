## Summary

Keep open. Current main still has the Feishu silent media-loss path: oversized or otherwise failed inbound media saves are caught and logged in the Feishu media resolver, then media-only messages with placeholders can continue into normal agent dispatch without a chat-visible warning.

## What This Changes

The issue requests a Feishu plugin fix so users are notified when inbound media exceeds `mediaMaxMb` and the agent does not proceed as if the file was received.

## Best Possible Solution

Fix this inside the bundled Feishu plugin by returning structured media-resolution failures, sending a Feishu-visible warning with the configured `mediaMaxMb` limit to the same conversation/thread, and blocking ordinary agent dispatch when the user input was dropped media rather than delivered media.

## Reproduction Assessment

Yes. A focused unit test can configure Feishu `mediaMaxMb`, send a Feishu file/media event whose downloaded buffer makes `saveMediaBuffer` throw `Media exceeds 30MB limit`, and assert current main logs the failure but can still dispatch because the parsed placeholder text is nonblank.

## Solution Assessment

Yes. A plugin-local structured failure plus chat notification is the narrowest maintainable solution because Feishu owns the reply target, account config, embedded post parsing, and media download behavior.
