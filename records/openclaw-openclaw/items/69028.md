## Summary

Keep open. Current main still treats Feishu media download or size failures as log-only resolver errors, and nonblank media placeholders let the turn continue into normal agent dispatch without a Feishu-visible failure notice.

## What This Changes

The issue asks Feishu inbound media over `mediaMaxMb` to produce a user-visible error instead of silently logging and continuing an agent turn.

## Best Possible Solution

Add a Feishu-owned media resolution result that reports dropped media to the source chat/thread and prevents success-looking agent turns for undelivered media while preserving successful attachments in the normal payload.

## Reproduction Assessment

Yes. A focused unit test can mock Feishu media download/save so `saveMediaBuffer` throws for a file event with a filename, then assert current main logs the failure and still reaches dispatch because the parsed placeholder content is nonblank.

## Solution Assessment

Yes. A plugin-local structured failure plus Feishu-visible warning is the narrowest maintainable fix because the Feishu plugin owns the reply target, account config, media parsing, and download behavior.
