## Summary

Keep open. Current main still turns Feishu media download failures into log-only drops, so an over-limit file can leave placeholder text in the agent turn without a Feishu-visible warning or an all-media-failed dispatch block.

## What This Changes

Request user-facing Feishu feedback, and preferably block normal agent handling, when inbound media exceeds `mediaMaxMb` and is not delivered to the agent.

## Best Possible Solution

Add a Feishu-owned dropped-media result that carries the failed filename/type/reason, sends a concise warning to the same chat/thread, and prevents normal agent dispatch when all requested media failed to resolve.

## Reproduction Assessment

Yes, at source level. Mock `saveMessageResourceFeishu` to reject an over-limit file with a filename and assert the handler logs the failure, preserves placeholder text, sends no Feishu warning, and still reaches dispatch with no media attachment.

## Solution Assessment

Yes. A plugin-local structured failure result plus same-chat warning is the narrow maintainable fix because Feishu owns media parsing, reply targeting, and inbound dispatch decisions.
