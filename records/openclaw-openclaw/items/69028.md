## Summary

Keep open. Current main enforces the Feishu media byte cap, but download failures are still reduced to log-only drops, and a media placeholder can continue into normal agent dispatch without a Feishu-visible warning.

## What This Changes

Request a user-facing Feishu warning and dispatch guard when inbound media exceeds `mediaMaxMb` or otherwise fails to download.

## Best Possible Solution

Add a Feishu-owned media resolution result that carries dropped-media details, replies to the same chat/thread with a concise warning, and blocks normal agent turns when all requested media was undelivered while preserving partial successes.

## Reproduction Assessment

Yes, at source level. A focused test can mock `saveMessageResourceFeishu` to reject an over-limit file with a filename, then assert the current path logs the failure, keeps placeholder text, and reaches dispatch with no media attachment.

## Solution Assessment

Yes. A plugin-local structured failure plus Feishu-visible warning is the narrow maintainable fix because the Feishu plugin owns reply targeting, media parsing, account config, and inbound dispatch decisions.
