## Summary

Keep open: the underlying claude-cli continuity bug is real, but this patch crosses the documented auth-boundary no-raw transcript policy and adds an unbounded synchronous Claude transcript read on the run path, so it needs maintainer/security rework plus real after-fix proof before merge.

## What This Changes

The PR changes cli-runner reseeding to raw-reseed auth-profile/auth-epoch invalidations, keep recent tail history, raise the reseed cap, and fall back to Claude CLI on-disk JSONL when the OpenClaw reseed looks thin.

## Best Possible Solution

Preserve the auth-boundary no-raw default and solve continuity through an explicitly approved summary/compaction-derived reseed path or a narrowly scoped fallback with bounded I/O, focused tests, docs alignment, and live patched-run proof.

## Reproduction Assessment

Yes from source inspection, though not from a live patched run: current main excludes auth-profile/auth-epoch from raw reseed and the paired report gives concrete WSL2 claude-cli steps. The PR body adds before-state diagnostics but says live after-fix verification is still pending.

## Solution Assessment

No. The patch fixes continuity by raw replay across an auth boundary that current docs, tests, and prior maintainer guidance keep closed; the safer solution needs an approved summary/compaction-based design or a tightly bounded fallback policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve the auth-boundary no-raw reseed policy:** `src/agents/cli-runner/prepare.ts:461-462`
  - body: This makes every invalidated CLI session opt into raw transcript reseeding; combined with adding auth-profile/auth-epoch to the allowed reasons, it undoes the documented guardrail that auth profile or credential-epoch changes never replay raw transcript history. If the auth identity changed, previous conversation content can be sent to a fresh CLI under the new profile. Keep auth resets on a maintainer-approved non-raw summary/compaction path instead.
  - confidence: 0.93
- **[P2] Bound the Claude transcript read before reseeding:** `src/agents/cli-runner/session-history.ts:428-431`
  - body: The new fallback moves `readClaudeCliFallbackSeed` onto the run preparation path, but that reader synchronously loads the whole Claude JSONL before `loadClaudeCliReseedMessages` trims messages to the reseed window. A large or corrupted `~/.claude/projects/...jsonl` can block the agent process or allocate far more than the 64 KiB prompt cap. Add a byte cap or streaming tail reader before using this during a run.
  - confidence: 0.84
