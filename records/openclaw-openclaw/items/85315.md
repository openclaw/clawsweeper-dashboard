## Summary

## What This Changes

The PR changes cli-runner reseeding so invalidated Claude CLI sessions can raw-reseed history, keep a longer tail, and fall back to Claude CLI JSONL when OpenClaw history looks one-sided.

## Best Possible Solution

Preserve the no-raw auth-boundary default and solve continuity through a maintainer-approved summarized/redacted recovery path or another bounded fallback with focused tests, docs alignment, and live patched-run proof.

A large or corrupted ~/.claude/projects transcript can block the agent process or allocate far more than the 64 KiB prompt cap; add a byte cap or streaming tail reader before using this during a run.
  - confidence: 0.84
