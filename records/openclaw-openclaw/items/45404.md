## Summary

Keep open. Current main has the canonical managed browser profile on `openclaw` and preserves `clawd` only as a compatibility driver alias, but there is still no browser doctor or repair path that discovers and safely archives leftover `CONFIG_DIR/browser/clawd/user-data` residue.

## What This Changes

Add a doctor/repair check that detects legacy `clawd` managed browser profile residue and safely archives it after confirming `openclaw` is the active managed profile.

## Best Possible Solution

Implement a browser-owned doctor warning and explicit `--fix` archival path for unconfigured legacy `CONFIG_DIR/browser/clawd/user-data`, while preserving the legacy `clawd` driver alias and documenting the canonical profile path.

## Reproduction Assessment

Yes. A source-level reproduction is to place an unconfigured `CONFIG_DIR/browser/clawd/user-data` beside the canonical `openclaw` profile; current doctor/browser doctor code only inspects configured/default profiles and will not report or archive that directory.

## Solution Assessment

Yes. A doctor/repair warning plus explicit archival is the narrow maintainable solution; startup migration or silent deletion would be riskier for browser profile state.
