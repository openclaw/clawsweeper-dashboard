## Summary

Keep open. Current main already uses `openclaw` as the canonical managed browser profile and keeps `clawd` as a config compatibility alias, but it still does not detect or safely archive stale on-disk `CONFIG_DIR/browser/clawd/user-data` residue.

## What This Changes

Request a doctor or repair cleanup path for legacy `clawd` managed Brave/browser profile directories left behind after the rename migration.

## Best Possible Solution

Add a browser-owned doctor warning and explicit `--fix` archival path for unconfigured legacy `CONFIG_DIR/browser/clawd/user-data`, while preserving the `clawd` driver alias and documenting the canonical `openclaw` profile path.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to place an unconfigured `CONFIG_DIR/browser/clawd/user-data` beside the canonical `openclaw` profile; current doctor code only inspects configured/default profiles and will not report that directory.

## Solution Assessment

Yes. A doctor/repair warning with explicit archival is the narrow maintainable fix; silent startup migration or deletion would be riskier for browser profile data.
