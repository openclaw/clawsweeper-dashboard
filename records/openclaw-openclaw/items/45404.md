## Summary

Keep open. Current main consistently uses the `openclaw` managed browser profile path and preserves `clawd` only as driver compatibility, but there is still no doctor or repair path that detects and safely archives stale `browser/clawd/user-data` residue.

## What This Changes

The issue requests detection and safe cleanup of legacy `clawd` managed Brave profile directories left behind after the rename migration.

## Best Possible Solution

Add a browser-owned doctor diagnostic that reports legacy `clawd` profile residue and, in repair mode, moves only confirmed unused residue to a safe trash/archive location after explicit confirmation.

## Reproduction Assessment

Yes, from source inspection rather than a live migrated home. Current main resolves managed browser state through the `openclaw` profile path and has no source path that scans for or reports a stale `browser/clawd/user-data` sibling.

## Solution Assessment

Yes. A doctor/browser-doctor diagnostic with opt-in archive or trash repair is the narrowest maintainable direction; startup migration or unconditional deletion would be less safe.
