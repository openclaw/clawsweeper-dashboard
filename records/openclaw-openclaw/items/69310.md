## Summary

Keep open: the PR addresses a real source-backed MEDIA failure path, but the patch currently sends separate warning messages outside outbound hook/audit handling, duplicates warnings when all media is dropped, and lacks the required changelog entry.

## What This Changes

The branch adds DroppedMedia types, carries dropped media through reply normalization/routing/outbound projections, emits attachment-failure notices, and adds focused tests.

## Best Possible Solution

Keep the DroppedMedia pipeline, but deliver warning notices through the normal outbound hook/audit path, remove the legacy duplicate warning, and add regression coverage plus a changelog entry.

Yes, source inspection gives a high-confidence path: current main still silently loses individual failed MEDIA items when another item survives, and the catastrophic normalization catch clears media without a detailed user-visible notice. I did not execute a live channel repro in this read-only review.

## Solution Assessment

No. The direction is maintainable, but the current patch should not send user-visible notice messages outside outbound hooks, should avoid duplicate all-drop warnings, and should add the required changelog entry.
