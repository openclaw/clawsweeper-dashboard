## Summary

Keep this PR open for review, but it is not mergeable: the current head has typecheck/lint failures, a silent-reply regression, and no after-fix real behavior proof. Current main has a simpler all-dropped media warning, but this PR still proposes broader partial-drop/structured dropped-media behavior that is not fully implemented on main.

## What This Changes

This PR adds a DroppedMedia pipeline that records failed media normalization, carries it through reply/outbound delivery, and renders user-visible attachment warning notices with tests.

## Best Possible Solution

Make the DroppedMedia pipeline type-clean, preserve `NO_REPLY`/silent semantics, keep focused regression coverage, and demonstrate the visible warning in a real channel before merge.

## Reproduction Assessment

Yes for the PR blockers: latest CI on merge SHA `6d1ed859bd11ac5972388539b287be4c9f5a2f89` reproduces the typecheck error and the silent-media failing test. I did not run local tests because this review was read-only.

## Solution Assessment

No, not yet. The richer structured dropped-media approach is plausible, but the current patch must compile, avoid sending warnings for silent `NO_REPLY` payloads, and include real channel proof before it is the best merge path.
