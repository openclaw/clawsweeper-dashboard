## Summary

Keep open. Current main still leaves partial MEDIA normalization failures without user-visible feedback, so the PR addresses a real gap, but the latest head has a blocking duplicate-warning case and is missing the required changelog entry for a user-facing fix.

## What This Changes

The PR adds a dropped-media pipeline through reply normalization, routing, outbound delivery, JSON projection, notices, logs, transcript mirroring, and tests so failed MEDIA directives become visible warnings.

## Best Possible Solution

Keep the droppedMedia pipeline direction, but make it the single source of user-visible attachment failure text, remove the legacy generic warning from all-media-failed normalized payloads, add the required changelog entry, and then rerun focused reply/outbound tests plus the changed gate before merge.

## Reproduction Assessment

Yes. The current-main gap is reproducible from the code path where one media item throws and another normalizes, and the PR-head regression is reproducible statically from the all-media-failed branch returning both legacy warning text and droppedMedia that deliver.ts later renders again.

## Solution Assessment

No, not yet. The DroppedMediaItem pipeline is the maintainable shape for this bug, but this exact patch should replace the legacy all-failed warning instead of layering the new notice on top of it, and it needs the user-facing changelog entry.
