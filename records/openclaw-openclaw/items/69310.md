## Summary

Keep this PR open. It is still the canonical broader dropped-media pipeline candidate, but it is not mergeable as-is because the latest head fails typecheck/build, violates the existing silent-reply behavior, and lacks required real behavior proof.

## What This Changes

The PR adds a DroppedMedia pipeline that records failed MEDIA attachments, preserves those failures through reply routing/outbound projections, and emits user-visible attachment warning text.

## Best Possible Solution

Keep the broader DroppedMedia design on this PR, but require focused CI fixes, preserve the existing silent-reply contract, and add redacted real behavior proof before merge.

## Reproduction Assessment

Yes. Source inspection shows current main can still silently lose partial media drops, and completed CI on the PR head reproduces the proposed patch’s typecheck and silent-reply regression failures.

## Solution Assessment

No, not as currently implemented. The direction is plausible, but the patch must first fix the wrapper access/type errors, remove unused imports, keep NO_REPLY silent when media normalization fails, and provide real behavior proof.
