## Summary

Keep open: this is a plausible fix for source-visible MEDIA drop behavior, but the current PR head still has merge-blocking defects and lacks real after-fix behavior proof.

## What This Changes

The branch adds DroppedMedia metadata for failed MEDIA normalization, carries it through reply routing/outbound delivery/JSON projections, formats user/transcript notices, upgrades warning logs, and adds tests.

## Best Possible Solution

Keep the DroppedMedia direction, fix the wrapper access and duplicate warning behavior, then add redacted live proof that a failed MEDIA directive produces exactly one visible warning in a real channel.

## Reproduction Assessment

Yes. Source inspection shows current main can still drop failed mixed/catastrophic MEDIA paths without per-item user-visible diagnostics, and the PR review defects are directly reproducible from the PR-head code.

## Solution Assessment

No, not yet. The pipeline is a maintainable direction, but this head needs to read dropped media from entry.payload, avoid the legacy duplicate warning, and provide real after-fix behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Read droppedMedia from the normalized payload:** `src/infra/outbound/deliver.ts:1913`
  - body: `normalizedPayloads` entries are `{ index, payload }`, so this reads a non-existent `droppedMedia` property on the wrapper. That should fail type checking, and if bypassed the transcript mirror would never record the dropped-media notice; use `p.payload.droppedMedia` here.
  - confidence: 0.93
- **[P2] Drop the legacy warning when recording dropped media:** `src/auto-reply/reply/reply-media-paths.ts:240-246`
  - body: When every media item fails, this branch still appends the legacy `Media failed` text and also sets `droppedMedia`. The delivery layer then inlines the new per-file notice for no-media payloads, so users see two warnings for one failure; let the dropped-media notice be the single rendered warning.
  - confidence: 0.84
