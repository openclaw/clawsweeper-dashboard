## Summary

Keep open. The PR is aimed at a real dropped-media visibility bug, but the current branch has source-level merge blockers: unused imports under `noUnusedLocals`, duplicate user-facing warnings for all-failed media, and no after-fix real behavior proof yet.

## What This Changes

The branch adds dropped-media reason tracking through reply media normalization, routing/outbound projections, delivery notices, transcript mirroring, and focused tests.

## Best Possible Solution

Keep the dropped-media pipeline direction, remove stale imports and the legacy generic warning duplication, add focused regression coverage for the all-failed path, and provide real Telegram or equivalent channel proof.

## Reproduction Assessment

Yes, from source inspection: current main catches a MEDIA normalization failure, logs it verbosely, and continues, so a mixed valid/invalid media reply can hide the dropped item from the user. I did not run a live reproduction in this read-only review.

## Solution Assessment

No, not as currently patched. The approach is maintainable, but the branch should first remove compile-blocking unused imports and replace the legacy generic warning instead of layering the detailed dropped-media notice on top of it.
