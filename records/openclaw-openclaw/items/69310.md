  - body: `normalizedPayloads` contains entries shaped like `{ index, payload }`, so reading `p.droppedMedia` does not typecheck and would miss the transcript mirror notices at runtime. Use the nested payload when collecting dropped media.
  - confidence: 0.98

  - body: The new imports for `sendMediaWithLeadingCaption`, `chunkByParagraph`, and `chunkMarkdownTextWithMode` are unused, and CI also reports the new `resolveConfigDir` import in `reply-media-paths.ts` as unused. These TS6133 errors keep the branch from building.
  - confidence: 0.97
- **[P2] Preserve silent media failures as silent:** `src/auto-reply/reply/reply-payloads-base.ts:85`
  - body: Making `droppedMedia` renderable causes a `NO_REPLY` payload whose media normalization fails to survive as a warning-only reply, breaking the existing silent-turn contract and the `suppresses warning text when silent media payloads fail normalization` test.
  - confidence: 0.9
