- **[P2] Avoid emitting both media failure notices:** `src/auto-reply/reply/reply-media-paths.ts:240-246`
  - body: When every media item fails, this branch still appends the legacy `⚠️ Media failed.` text and also carries `droppedMedia`. Drop the legacy text mutation when `droppedMedia` is present, or consume only one notice path.
  - confidence: 0.92
- **[P2] Read dropped media from normalized payload entries:** `src/infra/outbound/deliver.ts:1913`
  - body: `normalizedPayloads` contains `{ index, payload }` entries, so `p.droppedMedia` is always absent and should also fail type checking. This means transcript mirroring will not record the new dropped-media notices; read `p.payload.droppedMedia` instead.
  - confidence: 0.95
