## Summary

Keep open: current main has a generic all-dropped media warning, but this PR still targets broader dropped-media propagation and per-item visibility; it is not merge-ready because it regresses `NO_REPLY` silent turns and lacks after-fix real behavior proof.

## What This Changes

The PR adds a `droppedMedia` payload pipeline, sanitized attachment-failure notices, warning logs, JSON/transcript propagation, and regression tests for failed `MEDIA:` normalization.

## Best Possible Solution

Keep sanitized media-failure visibility for normal replies, preserve `NO_REPLY` as silent, add the required changelog entry, and require redacted real-runtime proof before merge.

## Reproduction Assessment

Yes for the PR-introduced regression: latest CI shows a focused failing test where a silent media payload now emits a warning payload. The underlying remaining media-drop gap is source-reproducible in current main's partial-drop path, though I did not run a live channel send.

## Solution Assessment

No, not as currently proposed: the direction is reasonable, but the patch must preserve silent-turn behavior and include the missing changelog and real behavior proof. A narrow repair should filter dropped-media notices after `NO_REPLY` is consumed while keeping non-silent media failures visible.
