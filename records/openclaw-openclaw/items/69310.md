This PR threads dropped-media metadata through reply normalization and outbound delivery so failed attachments produce sanitized user-visible notices and transcript entries.

## Best Possible Solution

Land a corrected version that compiles cleanly, reads droppedMedia through the normalized payload wrapper, removes stale imports and duplicate generic warnings, and includes real after-fix channel proof.
