## Summary

Keep this PR open: the dropped-media pipeline is a useful fix, but the current diff composes or sends new user-visible warning text after `message_sending` has already run, so outbound policy hooks cannot rewrite or cancel the final delivered content.

## What This Changes

The PR adds a dropped-media tracking pipeline that sanitizes failed media sources, carries them through reply/outbound delivery, and emits user-visible attachment-failure notices with tests.

## Best Possible Solution

Thread dropped-media notices through the same outbound hook and audit path as delivered message content, then add focused regression coverage and an Unreleased changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows media normalization failures can still be logged and discarded without a structured user-visible dropped-media notice, especially for mixed-media and catastrophic normalization paths.

## Solution Assessment

No. The direction is right, but the final notice text must be visible to `message_sending` or sent through an equivalent hook/audit path before this is the narrowest maintainable fix.
