## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Read dropped media from the payload field:** `src/infra/outbound/deliver.ts:1921`
  - body: `normalizedPayloads` entries are `{ index, payload }`, but this mirror aggregation reads `p.droppedMedia` on the wrapper. That property does not exist, so the branch should fail typecheck and the mirror would not collect the dropped-media entries; use `p.payload.droppedMedia` or carry the list explicitly.
  - confidence: 0.97
- **[P2] Keep silent media failures silent:** `src/auto-reply/reply/agent-runner-payloads.ts:58-61`
  - body: When media normalization throws for a `NO_REPLY` payload, this catch adds `droppedMedia`; later code clears the text because no media remains, but `isRenderablePayload` now treats `droppedMedia` as visible. That turns a silent failed-media reply into a delivered warning, breaking the existing silent-turn contract and the branch's own regression test.
  - confidence: 0.91
- **[P2] Avoid sending two warnings for all-dropped media:** `src/auto-reply/reply/reply-media-paths.ts:246`
  - body: In the all-media-failed branch, the payload still carries the generic `Media failed` text while also carrying `droppedMedia`; delivery later appends the detailed `Attachment not sent` notice for no-media payloads. All-dropped replies would show two warnings for the same failure, so skip the old generic text when the detailed dropped-media notice will be rendered.
  - confidence: 0.86
