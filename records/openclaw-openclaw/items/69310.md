Keep this PR open: the requested direction is valid, but the current patch introduces regressions in silent-reply handling, hook reporting for separately sent notices, and lint cleanliness.

## What This Changes

The PR adds a dropped-media diagnostic pipeline that carries sanitized failed attachment details through reply normalization, routing, outbound projection, delivery warnings, and transcript mirroring.

## Best Possible Solution

Land a revised patch that surfaces failed `MEDIA:` attachments while preserving established silent-reply behavior, reporting every user-visible notice through the existing sent-hook/audit path, and passing the changed lint/test lane.

## Reproduction Assessment

Yes. Source inspection plus the unchanged `agent-runner-payloads.test.ts` case gives a high-confidence reproduction path for `NO_REPLY\nMEDIA: ./missing.png` when media normalization throws.

## Solution Assessment

No. The approach is close, but it is not yet the best mergeable solution because it changes silent-turn behavior, omits sent-hook reporting for separately delivered warnings, and leaves unused imports in a linted core file.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve silent media-failure suppression:** `src/auto-reply/reply/agent-runner-payloads.ts:57-60`
  - body: The existing `agent-runner-payloads.test.ts` contract expects `NO_REPLY\nMEDIA: ./missing.png` to produce no visible payload when normalization fails. This catch now adds `droppedMedia`, then the later silent handling clears only `text`; because `isRenderablePayload` now treats dropped media as content, silent turns will emit an attachment warning. Preserve the silent suppression path or intentionally update the contract with focused coverage.
  - confidence: 0.92
- **[P2] Report separate dropped-media notices to sent hooks:** `src/infra/outbound/deliver.ts:1235`
  - body: When media or structured payload delivery succeeds, the warning is sent as a second text message via `sendTextChunks`, but only the primary payload is passed to `emitMessageSent`. Hook consumers therefore miss a user-visible message, while the fallback branch now reports its composed fallback text. Emit a sent event for the notice or include the notice in the reported content consistently.
  - confidence: 0.86
- **[P2] Remove unused outbound delivery imports:** `src/infra/outbound/deliver.ts:1-4`
  - body: These new imports are not referenced anywhere else in `deliver.ts` at PR head. The core lint lane uses oxlint, so the PR should remove `sendMediaWithLeadingCaption`, `chunkByParagraph`, and `chunkMarkdownTextWithMode` before merge.
  - confidence: 0.9
