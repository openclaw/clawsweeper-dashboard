## Summary

Keep this PR open: it addresses a valid silent media-drop problem, but the current branch is not mergeable because PR-head CI catches a silent-reply regression, generated notice text bypasses `message_sending` policy hooks, and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR adds a `droppedMedia` payload pipeline that records failed `MEDIA:` normalization and sends sanitized attachment-failure warnings to users and transcript mirrors.

## Best Possible Solution

Keep the dropped-media user-notice direction, but preserve silent `NO_REPLY` semantics, route generated notices through outbound hook policy, and require real setup proof before merge.

## Reproduction Assessment

Yes. The PR-head CI annotation reproduces the regression in `agent-runner-payloads.test.ts`, and the current source shows why a dropped-media-only payload now survives the silent reply path.

## Solution Assessment

No. The approach is directionally useful, but the current implementation breaks the silent-reply contract and bypasses documented outbound hook controls.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve NO_REPLY suppression for dropped media:** `src/auto-reply/reply/reply-payloads-base.ts:85`
  - body: `NO_REPLY\nMEDIA: ./missing.png` is supposed to disappear when media normalization fails, but this change makes the resulting droppedMedia-only payload renderable. The PR-head CI failure shows that the silent-reply path now returns one payload instead of zero, so users can receive a warning from a deliberately silent response.
  - confidence: 0.92
- **[P2] Run dropped-media notices through sending hooks:** `src/infra/outbound/deliver.ts:1602`
  - body: `message_sending` hooks are documented as able to rewrite or cancel outbound content, but the notice is appended after `applyMessageSendingHook` and separate notice sends use `sendTextChunks` directly. Installations using these hooks for policy or moderation cannot filter or cancel the new user-visible warning text.
  - confidence: 0.86
