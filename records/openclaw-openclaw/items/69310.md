Keep this active, but the current PR head is not mergeable: it still contains build-breaking type/lint errors, a failing silent-reply behavior test, and no real after-fix behavior proof for the visible dropped-media warning path.

## What This Changes

The PR adds dropped-media metadata, user-facing attachment-failure notices, transcript/JSON propagation, warning logs, and regression tests across reply media normalization and outbound delivery.

## Best Possible Solution

Keep the dropped-media pipeline idea, but fix the build/test regressions, preserve existing silent-reply semantics, and add redacted live proof showing an invalid MEDIA item produces the expected warning in a real channel.

## Reproduction Assessment

Yes by source inspection and CI. Current main still has a mixed-media silent-drop path, and the PR head’s own CI reproduces the introduced type and silent-reply failures.

## Solution Assessment

No, not in the current form. The metadata pipeline is a plausible direction, but the patch must read the normalized payload shape correctly, remove stale imports, preserve NO_REPLY behavior, and provide real behavior proof before it is the maintainable fix.

Full review comments:

  - body: `normalizedPayloads` stores entries as `{ index, payload }`, so `p.droppedMedia` does not exist and required typecheck/lint lanes fail with TS2339. Read `p.payload.droppedMedia` here or explicitly carry the field on the wrapper before using it.
  - confidence: 0.98
- **[P2] Keep silent media failures silent:** `src/auto-reply/reply/reply-payloads-base.ts:85`
  - body: Treating any `droppedMedia` as renderable lets a `NO_REPLY` payload whose media normalization fails survive as a warning-only reply. The PR’s auto-reply shard already catches this: the silent media failure test expected zero payloads but received one, so this needs to preserve the silent-token contract.
  - confidence: 0.93
- **[P1] Remove the stale outbound imports:** `src/infra/outbound/deliver.ts:1-4`
  - body: The added imports for `sendMediaWithLeadingCaption`, `chunkByParagraph`, and `chunkMarkdownTextWithMode` are unused, so `pnpm tsgo:prod` fails with TS6133 before the PR can merge. Remove them or wire them into the actual implementation.
  - confidence: 0.97

  - body: `resolveConfigDir` is imported but never used in this file, and the required prod typecheck reports TS6133 for it. Drop the import or use it as part of the intended media normalization logic.
  - confidence: 0.97
