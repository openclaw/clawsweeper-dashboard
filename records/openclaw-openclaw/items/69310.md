## Summary

Keep this PR open: it is addressing a valid current-main media-drop visibility bug and the discussion shows several earlier privacy/routing review concerns were fixed, but the latest head still has a user-visible duplicate-warning regression plus stale imports that should be cleaned before merge.

## What This Changes

The branch adds DroppedMedia tracking through reply normalization and outbound delivery, emits sanitized attachment-failure notices, preserves dropped-media-only payloads across routing/projection paths, and adds regression tests.

## Best Possible Solution

Keep the DroppedMedia pipeline, but make droppedMedia the single source for the detailed user notice and remove stale imports before normal PR validation.

## Reproduction Assessment

Yes, by source inspection: current main drops individual failed MEDIA entries in the catch/continue path, and the PR head's all-failed path now combines the legacy generic warning with the new detailed droppedMedia notice.

## Solution Assessment

No, not quite yet: the proposed direction is maintainable, but the current patch should stop injecting the legacy generic warning when droppedMedia will render the user-visible diagnostic and should remove unused imports.
