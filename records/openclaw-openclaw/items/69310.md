- **[P1] Read dropped media from the wrapped payload:** `src/infra/outbound/deliver.ts:1921`
  - body: The mirror collection iterates `normalizedPayloads`, whose entries are `{ index, payload }`, so `p.droppedMedia` does not exist. This is why current `check-prod-types` and plugin boundary DTS fail at `deliver.ts(1921,58)`; read `p.payload.droppedMedia` or destructure `payload` before flattening.
  - confidence: 0.98

  - body: Making every payload with `droppedMedia` renderable lets a `NO_REPLY\nMEDIA: ...` payload whose media normalization fails survive after its silent text is stripped, so `buildReplyPayloads` emits a warning during a silent turn. The existing `suppresses warning text when silent media payloads fail normalization` test now receives one payload instead of zero; clear/drop these diagnostics for parsed silent replies unless silent policy explicitly allows visible output.
  - confidence: 0.94
