The PR adds dropped-media tracking, sanitized warning notices, routing/outbound JSON propagation, transcript mirroring, warning logs, and focused tests for failed `MEDIA:` normalization.

## Best Possible Solution

Land a revised version that preserves silent-reply suppression, emits exactly one dropped-media notice per failed attachment set, keeps the sanitized warning pipeline, adds regression coverage, and includes real behavior proof.

## Reproduction Assessment

Yes. Current main has a source-visible swallow path for failed `MEDIA:` normalization, and the latest PR regression has a concrete CI annotation on `agent-runner-payloads.test.ts:596`.

## Solution Assessment

No. The direction is maintainable, but the current patch is not the best fix until silent replies remain suppressed and the legacy generic warning is removed so the new formatter is the single user-visible notice.
