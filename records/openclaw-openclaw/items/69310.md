## Summary

## What This Changes

The PR adds dropped-media metadata, user-facing attachment-failure notices, outbound/transcript propagation, and focused tests around failed MEDIA normalization.

## Best Possible Solution

Keep the dropped-media notice design, fix the compile and silent-reply blockers, then require redacted real behavior proof before merge.

## Reproduction Assessment

Yes, at source level: current main drops individual failed media items with only verbose logging and clears media on catastrophic normalization failure. I did not run a live channel reproduction, but the current code paths and PR CI failures are concrete.

## Solution Assessment

No, not in the current form: the PR direction is maintainable, but the branch must read droppedMedia from the normalized payload wrapper correctly and preserve NO_REPLY/silent semantics before it is a safe fix.

  - body: `normalizedPayloads` contains `{ index, payload }` entries, so this reads a property that does not exist and currently fails TypeScript with TS2339. Use `p.payload.droppedMedia` when building the mirror notice.
  - confidence: 0.98
- **[P2] Honor silent replies after media normalization failures:** `src/auto-reply/reply/agent-runner-payloads.ts:58-61`

  - confidence: 0.92
