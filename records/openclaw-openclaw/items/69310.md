## Summary

Keep this PR open: it addresses a real reply-media visibility gap and the latest head fixed the earlier stale test-field issue, but the current patch would show duplicate warnings when every media item is dropped and it still lacks after-fix real behavior proof.

## What This Changes

The branch adds a DroppedMedia pipeline for MEDIA normalization failures, user/transcript notices, routing/projection preservation, warning logs, and focused tests across reply and outbound delivery.

## Best Possible Solution

Keep the DroppedMedia pipeline, remove the legacy all-dropped generic warning or otherwise ensure only one user-facing diagnostic is emitted, add regression coverage for that path, and require real OpenClaw behavior proof before merge.

## Reproduction Assessment

Yes for the review finding by source inspection: the PR returns text containing the legacy `⚠️ Media failed.` warning plus `droppedMedia`, and `deliver.ts` then appends the detailed dropped-media notice when no media survives. I did not run a live repro because this was a read-only review.

## Solution Assessment

No, not yet. The DroppedMedia direction is maintainable, but the all-dropped path should replace the legacy generic warning instead of stacking it with the new detailed notice.
