## Summary

Keep open. The PR is a useful fix for silent MEDIA drops, but the latest head still double-surfaces the all-media-failed case by preserving the legacy generic warning and then adding the new dropped-media notice; it also lacks the required user-facing changelog entry.

## What This Changes

The PR adds a dropped-media payload field and threads sanitized attachment-failure notices through reply media normalization, routing, outbound delivery, JSON projection, tests, and transcript mirroring.

## Best Possible Solution

Replace the legacy generic all-media-failed text with the new sanitized dropped-media notice, add focused coverage for that path, and include a single-line `CHANGELOG.md` Fixes entry.

## Reproduction Assessment

Yes. By inspection, an all-failed media payload gets `text: "⚠️ Media failed."` plus `droppedMedia` in normalization, then delivery appends the formatted dropped-media notice in the no-media text path.

## Solution Assessment

No. The PR direction is the right maintainable shape, but the all-failed path should emit one actionable attachment-failure notice and the user-facing fix needs the required changelog entry.
