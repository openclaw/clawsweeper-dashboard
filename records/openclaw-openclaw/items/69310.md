## Summary

Keep open. This is a plausible bug fix, but the current PR head is not mergeable: CI shows TypeScript build failures, an existing silent-reply regression test fails, and the external PR still lacks real after-fix behavior proof.

## What This Changes

The PR adds a dropped-media pipeline that records failed MEDIA normalization, formats user-visible attachment-not-sent notices, preserves dropped-media-only payloads through routing/outbound projections, and adds regression tests.

## Best Possible Solution

Keep the dropped-media visibility direction, but make it preserve explicit silent-reply policy, carry `droppedMedia` through the correct payload objects, remove compile blockers, and add redacted real runtime proof of the warning.

## Reproduction Assessment

Yes for the PR blockers: the current PR CI shows exact typecheck errors and a failing existing agent-runner test. The original silent-drop behavior is source-reproducible from current main, but I did not run a live channel reproduction in this read-only review.

## Solution Assessment

No. The requested direction is good, but this implementation is not yet the best mergeable fix because it violates silent-reply behavior and breaks the TypeScript build; the safer path is to keep notices only on visible payloads and mirror from the wrapped `payload` object.
