## Summary

This PR should stay open: current main still lacks structured per-attachment feedback for dropped reply media, but the proposed patch has a user-visible duplicate-warning regression and is missing the required changelog entry.

## What This Changes

The PR adds dropped-media metadata, sanitized notices, routing preservation, JSON projection, transcript mirror handling, and tests for failed reply media normalization.

## Best Possible Solution

Keep the PR open, remove the legacy generic warning from the droppedMedia delivery path, add an all-media-failed regression test, and add the required changelog entry.

## Reproduction Assessment

Yes, from source inspection: current main still has catch-and-continue media normalization paths without structured dropped-media delivery details. The PR regression is also source-reproducible by following an all-media-failed payload from `reply-media-paths.ts` into `deliver.ts`.

## Solution Assessment

No. The PR direction is the right narrow fix, but the implementation should not carry both the legacy `Media failed` text and the new detailed dropped-media notice for the same all-failed attachment set.
