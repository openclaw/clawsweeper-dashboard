## Summary

Keep this active PR open: it addresses a real dropped-media visibility bug, but the current head does not build, breaks an existing silent-reply guard, still duplicates warnings on the all-media-dropped path, and lacks required real behavior proof.

## What This Changes

The PR adds DroppedMedia types, normalization tracking, outbound delivery notices, JSON projection, transcript mirroring, and tests for MEDIA directive failures.

## Best Possible Solution

Keep the PR open, fix the type error, preserve NO_REPLY suppression, remove the legacy generic warning when droppedMedia will render the detailed notice, then attach redacted real behavior proof before merge.

## Reproduction Assessment

Yes. Source inspection shows current main still drops individual failed MEDIA items after logging, and PR CI gives focused failure paths for regressions introduced by the proposed fix.

## Solution Assessment

No. The droppedMedia pipeline is the right direction, but this patch needs the compile fix, silent-reply preservation, duplicate-warning cleanup, and real behavior proof before it is a maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Read droppedMedia from normalized payload entries:** `src/infra/outbound/deliver.ts:1921`
  - body: `normalizedPayloads` contains `{ index, payload }` entries, so `p.droppedMedia` does not typecheck and would also leave transcript mirroring without the collected drops at runtime. Read from `p.payload.droppedMedia` instead.
  - confidence: 0.98
- **[P2] Keep silent media failures silent:** `src/auto-reply/reply/reply-payloads-base.ts:85`
  - body: Making any `droppedMedia` payload renderable revives `NO_REPLY` replies whose media failed normalization after the text is cleared, which is why the existing silent-media test now emits one reply instead of zero. Preserve the silent decision before letting droppedMedia-only payloads through.
  - confidence: 0.94
- **[P2] Remove the legacy warning when droppedMedia is set:** `src/auto-reply/reply/reply-media-paths.ts:240-243`
  - body: When every MEDIA item fails, this branch still appends the old generic `Media failed` text and also returns `droppedMedia`; delivery then appends the detailed dropped-media notice, so users can receive two warnings for the same failed attachment.
  - confidence: 0.9
