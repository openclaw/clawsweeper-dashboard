## Summary

Keep this PR open: the underlying bug is valid and the direction is useful, but the current patch still introduces hook/audit regressions, duplicate user warnings, lint-level dead code, and misses the required user-facing changelog entry.

## What This Changes

The PR threads dropped reply-media failures through outbound payloads so users receive sanitized attachment-not-sent notices instead of silent media loss.

## Best Possible Solution

Revise the PR so dropped-media notices follow the normal outbound policy and audit path, remove duplicate legacy warning text and dead code, add the required changelog entry, and then validate the targeted reply-media/outbound tests plus the changed gate.

## Reproduction Assessment

Yes, at source level: current main still drops failed mixed media items in the reply-media normalizer, and the PR diff shows the new notice path bypassing hooks and retaining the old generic warning. I did not execute a live runtime repro in this read-only review.

## Solution Assessment

No; surfacing dropped media is the right narrow fix, but this patch should route notice delivery through the established outbound hook/audit semantics and avoid duplicate warnings before it is merge-ready.
