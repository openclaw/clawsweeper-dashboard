## Summary

Keep open: the PR targets a real source-visible dropped-media bug, but the current head does not build, breaks the existing silent-reply contract, and still lacks required real behavior proof.

## What This Changes

The branch adds a dropped-media pipeline for failed MEDIA normalization, preserving sanitized failure details through reply routing, outbound delivery, JSON projection, transcript mirroring, warning logs, and focused tests.

## Best Possible Solution

Keep the PR open, fix the compile blockers and silent-reply regression, preserve dropped-media transcript collection from the normalized payload object, then add redacted real behavior proof for a channel-visible failure notice.

## Reproduction Assessment

Yes, by source inspection: current main drops failed MEDIA items in the reply-media normalizer after only verbose logging, and the agent-runner fallback strips media on normalization failure. I did not run a live channel repro in this read-only review.

## Solution Assessment

No, not in the current branch state. The overall direction is reasonable, but this implementation must fix the type errors, keep NO_REPLY/silent turns silent, and provide real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Read dropped media from the normalized payload:** `src/infra/outbound/deliver.ts:1921`
  - body: `normalizedPayloads` contains entries shaped as `{ index, payload }`, so `p.droppedMedia` is both a TS2339 build error and always the wrong object at runtime. This means the transcript mirror would not record the dropped-media notice; collect `p.payload.droppedMedia` instead.
  - confidence: 0.98
- **[P2] Keep silent replies silent after media failure:** `src/auto-reply/reply/normalize-reply.ts:59-61`
  - body: The new `hasDroppedMedia` escape lets a `NO_REPLY` payload with a failed MEDIA directive survive as a dropped-media-only reply. That breaks the existing silent-turn contract and the current regression test, so silent payloads should still be suppressed instead of sending a warning to the user.
  - confidence: 0.93
- **[P2] Remove the unused imports that break typecheck:** `src/infra/outbound/deliver.ts:1-4`
  - body: The PR head adds unused imports such as `sendMediaWithLeadingCaption`, `chunkByParagraph`, `chunkMarkdownTextWithMode`, and `resolveConfigDir`; CI reports TS6133 for these, so the branch cannot pass production typechecking until they are removed.
  - confidence: 0.96
