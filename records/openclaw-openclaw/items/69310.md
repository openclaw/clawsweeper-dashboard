## Summary

Keep open: current main still does not carry per-item dropped-media details through the reply and outbound delivery pipeline, and this PR is aimed at a real remaining behavior gap. The patch needs one small correction before merge because the all-media-dropped path would show both the old generic warning and the new detailed attachment warning.

## What This Changes

The branch adds dropped-media metadata, sanitization helpers, delivery notices, transcript/JSON propagation, and focused tests for media normalization failures.

## Best Possible Solution

Keep the dropped-media pipeline, but remove the legacy generic warning once droppedMedia is available and add a regression for the all-media-dropped path so users see exactly one detailed notice.

## Reproduction Assessment

Yes by source inspection. Current main still has generic or silent media-drop paths, and the PR head has a deterministic all-dropped path that returns both legacy warning text and droppedMedia metadata.

## Solution Assessment

No as currently patched. The overall direction is maintainable, but the all-dropped path should stop appending the old generic warning and let the new formatter produce the single user-facing diagnostic.
