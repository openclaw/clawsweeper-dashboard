## What This Changes

The PR adds a structured `droppedMedia` pipeline so failed `MEDIA:` attachments are preserved through reply normalization, outbound delivery, JSON projection, notices, logs, transcript mirroring, and tests.

## Best Possible Solution

Keep the broader dropped-media pipeline only after it compiles, preserves silent-reply policy, and includes redacted real behavior proof showing failed media produces the intended warning without leaking paths or secrets.

## Reproduction Assessment

Yes for the PR blockers: the latest CI run reproduces the type/build failure and the silent-media test regression. For the original behavior, current-main source shows only the all-media-dropped case gets a warning, while partial media drops still continue without a per-item user notice.

## Solution Assessment

No, this is not the best mergeable form yet. The direction is reasonable, but the patch needs the wrapper/type fixes, unused import cleanup, silent-policy handling, and real runtime proof before maintainers can treat it as the right fix.

  - body: `normalizedPayloads` contains `{ index, payload }` entries, so `p.droppedMedia` does not exist and CI fails with TS2339 here; it would also miss every dropped-media item at runtime. Read from `p.payload.droppedMedia` or carry the field onto the wrapper explicitly.
  - confidence: 0.97
- **[P1] Remove the unused imports before merging:** `src/infra/outbound/deliver.ts:1-4`
  - body: The latest CI run fails `pnpm tsgo:prod` because this file imports `sendMediaWithLeadingCaption`, `chunkByParagraph`, and `chunkMarkdownTextWithMode` without using them. The branch needs these removed, along with the unused `resolveConfigDir` import in `reply-media-paths.ts`, before it can build.
  - confidence: 0.95
- **[P2] Preserve silent reply suppression for dropped media:** `src/auto-reply/reply/reply-payloads-base.ts:83-86`
  - body: By making dropped-media-only payloads renderable globally, a `NO_REPLY` payload whose media normalization fails now survives as a warning payload; the existing `suppresses warning text when silent media payloads fail normalization` test now gets one payload instead of none. Keep explicit silent turns silent while still surfacing non-silent media failures.
  - confidence: 0.91
