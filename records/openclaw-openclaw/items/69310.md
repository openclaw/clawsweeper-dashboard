## Summary

Keep open: the PR is a plausible fix for a real media-drop visibility problem, but the current head is not mergeable because exact-head CI shows a type error and a silent-reply regression, and the contributor still has not posted after-fix real behavior proof.

## What This Changes

The PR adds dropped-media tracking from reply media normalization through outbound delivery, JSON projection, and transcript paths, then sends sanitized attachment-failure notices to users.

## Best Possible Solution

Land a corrected implementation that preserves silent/no-reply semantics, threads droppedMedia through the existing typed payload-entry shape, keeps the privacy sanitization, and includes focused tests plus redacted real channel or CLI proof.

## Reproduction Assessment

Yes for the PR regressions: exact-head CI reproduces both the TypeScript error and the failing silent-media test. The original user problem is also source-backed, though current main already covers the all-dropped-media case with a simpler warning.

## Solution Assessment

No in the current form. The direction is maintainable, but the patch must fix the payload-entry access, preserve silent-reply suppression, and add real behavior proof before merge.
