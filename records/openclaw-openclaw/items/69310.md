  - body: `normalizedPayloads` contains wrapper objects shaped like `{ index, payload }`, so `p.droppedMedia` does not typecheck and would never collect the actual dropped-media entries for the transcript mirror. Read from `p.payload.droppedMedia` or destructure the payload before formatting the mirror notice.
  - confidence: 0.99
- **[P2] Preserve NO_REPLY silence when media normalization fails:** `src/auto-reply/reply/agent-runner-payloads.ts:58-61`
  - body: The catch path now synthesizes `droppedMedia` for a `NO_REPLY` payload whose media fails normalization, and the new renderability checks let that warning survive as a visible reply. The existing `suppresses warning text when silent media payloads fail normalization` test fails because this sends one payload instead of none, so silent directives need to keep suppressing synthesized drop notices.
  - confidence: 0.92
- **[P2] Remove the unused outbound imports:** `src/infra/outbound/deliver.ts:1-4`
  - body: These newly added imports are not used in the final file, and `tsgo:prod` fails with TS6133 for them. Drop `sendMediaWithLeadingCaption`, `chunkByParagraph`, and `chunkMarkdownTextWithMode` unless the implementation actually needs them.
  - confidence: 0.98

  - body: `resolveConfigDir` is imported but never read, which is another TS6133 failure in the latest CI run. Removing this import is required before the branch can typecheck.
