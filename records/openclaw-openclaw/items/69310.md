## Summary

Keep open. Current main only has a generic all-media-failed warning, and this PR is still useful, but the branch is not merge-ready because it can emit duplicate user warnings and lacks the required changelog entry.

## What This Changes

The PR threads dropped-media metadata through reply normalization, routing, outbound delivery, transcript/JSON projections, and tests so failed `MEDIA:` attachments become visible to users.

## Best Possible Solution

Merge a cleaned-up version of this PR that lets the dropped-media formatter be the only user-visible failure notice, preserves the privacy safeguards, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. The branch can be reproduced by an all-media-dropped `MEDIA:` payload: `reply-media-paths.ts` adds the legacy warning and `droppedMedia`, then `deliver.ts` appends the detailed dropped-media notice for the same payload.

## Solution Assessment

No, not as currently proposed. The dropped-media pipeline is the right direction, but the legacy generic warning should be removed from the dropped-media path and the required changelog entry added before merge.
