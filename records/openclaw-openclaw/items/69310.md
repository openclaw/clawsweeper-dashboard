## Summary

Keep open: the fix direction is useful, but the current PR head has blocking correctness/typecheck failures, regresses explicit silent replies, and still lacks required after-fix real behavior proof.

## What This Changes

The PR adds a DroppedMedia pipeline through reply media normalization, outbound delivery, transcript mirroring, JSON projection, routing, and tests so failed attachments produce user-visible warnings.

## Best Possible Solution

Land a corrected version that carries sanitized dropped-media details through reply and outbound paths, preserves silent/no-reply semantics, emits one warning per dropped event, mirrors the notice correctly, and includes real Telegram-visible proof.

## Reproduction Assessment

Yes, by source and CI evidence. Current main still has partial dropped-media visibility, and the PR head's silent-reply regression is reproduced by the failing `agent-runner-payloads` check annotation.

## Solution Assessment

No. The dropped-media pipeline is the right direction, but this patch must fix silent-response handling, the mirror object-shape bug, duplicate all-failed warnings, and typecheck/lint blockers before it is a maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve silent media failures as no reply:** `src/auto-reply/reply/agent-runner-payloads.ts:58-61`
  - body: The catch path always adds `droppedMedia` after media normalization fails. A payload like `NO_REPLY\nMEDIA: ./missing.png` then survives the new dropped-media checks, and CI is failing the existing silent-media contract because one payload is emitted instead of zero.
  - confidence: 0.95
- **[P1] Collect mirror drops from the wrapped payload:** `src/infra/outbound/deliver.ts:1921`
  - body: `normalizedPayloads` entries are `{ index, payload }`, so `p.droppedMedia` is a type error and would be undefined at runtime. Read `p.payload.droppedMedia` so transcript mirroring actually records the new dropped-media diagnostics.
  - confidence: 0.96
- **[P2] Drop the legacy all-failed warning when carrying droppedMedia:** `src/auto-reply/reply/reply-media-paths.ts:240-246`
  - body: When every media item fails, this branch still appends the old generic `Media failed` text while also setting `droppedMedia`. Delivery later inlines the formatted dropped-media notice for no-media payloads, so users can see duplicate warnings for one attachment failure.
  - confidence: 0.9
- **[P2] Remove unused delivery imports:** `src/infra/outbound/deliver.ts:1-5`
  - body: These new imports are never referenced, and the current `check-prod-types` annotation reports them as unused. Keeping them leaves the branch unable to typecheck with the core TypeScript settings.
  - confidence: 0.98
- **[P2] Remove the unused config-dir import:** `src/auto-reply/reply/reply-media-paths.ts:18`
  - body: `resolveConfigDir` is imported but never used in this file, and the branch's typecheck reports it as a `noUnusedLocals` failure. Drop the import before merge.
  - confidence: 0.98
