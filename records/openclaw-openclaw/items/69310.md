## Summary

Keep open: the direction is useful, but this PR still has a blocking correctness issue where all-failed media gets both the old generic warning and the new detailed dropped-media notice, plus the user-facing fix needs a changelog entry.

## What This Changes

The PR adds a DroppedMediaItem pipeline through reply media normalization, routing, outbound delivery, JSON projection, transcript mirroring, and tests so failed MEDIA directives produce user-visible warnings.

## Best Possible Solution

Land the dropped-media pipeline after removing the legacy generic warning from the droppedMedia path, adding a regression for single-notice rendering, and documenting the user-visible fix in the changelog.

## Reproduction Assessment

Yes. Source inspection shows current main silently continues on partial media drops, and the PR branch has a clear all-failed path that would compose both the legacy generic warning and the new detailed notice.

## Solution Assessment

No, not as currently written. The dropped-media pipeline is the right shape, but the all-failed normalizer branch should stop adding the old generic warning and let the formatted dropped-media notice be the single user-visible diagnostic.
