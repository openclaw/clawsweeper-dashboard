## Summary

Keep open: the dropped-media warning direction is useful, but current PR head still breaks the established silent-turn contract and lacks required after-fix real behavior proof, including Telegram-visible proof.

## What This Changes

This PR adds a DroppedMediaItem pipeline that records failed MEDIA normalization, formats user-visible attachment warnings, and preserves those warnings through routing, delivery, JSON projection, tests, and transcript mirroring.

## Best Possible Solution

Keep the dropped-media visibility direction, but preserve NO_REPLY/silent-turn suppression, prove the visible behavior in a real OpenClaw setup, and include Telegram proof before merge.

## Reproduction Assessment

Yes. Source inspection shows the existing silent-media failure test expects zero payloads, while PR head stores `droppedMedia` in the normalization fallback and makes dropped-media-only payloads renderable.

## Solution Assessment

No. The broad dropped-media pipeline is a maintainable direction, but it must clear or ignore `droppedMedia` for parsed silent replies and needs real channel proof before this implementation is merge-ready.
