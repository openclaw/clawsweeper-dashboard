## What This Changes

This PR adds a DroppedMedia pipeline that sanitizes failed MEDIA sources, carries them through reply/outbound routing, emits user/transcript notices, upgrades warning logs, and adds focused tests.

## Best Possible Solution

Keep the structured droppedMedia approach, but remove the duplicate legacy warning, fix the mirror/typecheck blockers, then verify the real channel notice path before merge.

## Reproduction Assessment

Yes, from source inspection: current main catches failed media normalization and continues without a structured dropped-media payload, so mixed-media failures can remain invisible. I did not run a live reproduction because this review was read-only.

## Solution Assessment

No, not yet: the pipeline direction is maintainable, but this PR still has concrete correctness/typecheck defects and no real behavior proof. The narrow fix is to let droppedMedia own the warning text, read mirror drops from the payload wrapper, remove unused code, and then prove the visible notice path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Remove the legacy all-failed warning:** `src/auto-reply/reply/reply-media-paths.ts:240-243`
  - body: When every media item fails, this branch still appends `⚠️ Media failed.` while also carrying `droppedMedia`. The delivery layer later formats `droppedMedia` into an attachment warning for no-media payloads, so users can receive both the generic legacy warning and the new detailed warning for the same failure. Leave the text unchanged here and let `droppedMedia` render the notice once.
  - confidence: 0.91
- **[P2] Read dropped entries from the payload wrapper:** `src/infra/outbound/deliver.ts:1921`
  - body: `normalizedPayloads` contains `{ index, payload }` entries, so `p.droppedMedia` is not a valid field. This should read `p.payload.droppedMedia`; otherwise the branch either fails typecheck or, if loosened, never mirrors the dropped-media diagnostics it claims to collect.
  - confidence: 0.96
- **[P2] Remove the unused deliver imports and tracking set:** `src/infra/outbound/deliver.ts:1-4`
  - body: The new `sendMediaWithLeadingCaption`, `chunkByParagraph`, and `chunkMarkdownTextWithMode` imports are not referenced, and this file also adds a `deliveredPayloadIndices` set that is only written. With core `noUnusedLocals` enabled, these dead symbols block the branch before behavior tests run.
  - confidence: 0.94
