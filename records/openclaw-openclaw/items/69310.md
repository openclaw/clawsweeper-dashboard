## Summary

Keep open: this PR addresses a real reply-media diagnostics gap that current main only covers with a coarse all-failed fallback, but the patch still double-warns users when every media item fails and it lacks the required changelog entry for a user-facing fix.

## What This Changes

The PR threads dropped-media metadata through reply normalization, routing, outbound projection, delivery, transcript mirroring, and tests so failed MEDIA attachments produce user-visible warnings.

## Best Possible Solution

Land the dropped-media pipeline after replacing the legacy generic fallback with the richer notice path and adding the required changelog coverage.

## Solution Assessment

No as written. The dropped-media pipeline is the right maintainable direction, but it should replace the current generic fallback instead of stacking two user-visible warnings, and the user-facing fix needs a changelog entry.
