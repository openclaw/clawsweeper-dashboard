## Summary

Keep open: this is a plausible user-facing media-drop fix, but the current head is not mergeable because it still has a type/runtime bug in transcript mirroring, can double-warn all-dropped media replies, and lacks required real behavior proof.

## What This Changes

The PR adds a dropped-media pipeline for failed `MEDIA:` attachments, carrying sanitized failure details through reply normalization, outbound delivery, JSON projection, transcript mirroring, and tests.

## Best Possible Solution

Keep the dropped-media contract, but fix the wrapped-payload mirror read, remove the legacy generic warning from detailed-notice flows, and require redacted real OpenClaw proof before merge.

## Reproduction Assessment

Yes for the blocking PR defects: source inspection shows the mirror reads `droppedMedia` from the wrapper object instead of `payload`, and live CI reports `check-prod-types` failing. The duplicate-warning path is also source-reproducible from the all-dropped normalizer branch plus the outbound notice append.

## Solution Assessment

No: the overall direction is maintainable, but this head is not the best mergeable implementation until it fixes the wrapper access, avoids duplicate warnings, and supplies real behavior proof. The narrow safer path is to keep the sanitized dropped-media pipeline while correcting those delivery edges.
