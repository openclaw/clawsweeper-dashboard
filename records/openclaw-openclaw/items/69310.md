## Summary

Keep open: the direction is valid, but the patch still leaves the legacy all-media-failed warning in the normalizer, so dropped-media-only failures render both the old generic warning and the new detailed notice.

## What This Changes

The PR threads dropped `MEDIA:` normalization failures through reply payloads and outbound delivery, emits sanitized user-visible attachment warnings, upgrades diagnostics, and adds focused tests.

## Best Possible Solution

Keep the structured `droppedMedia` pipeline, but make outbound delivery the single owner of user-visible attachment-failure notice text.

## Reproduction Assessment

Yes. On the PR branch, an all-invalid `MEDIA:` payload follows `reply-media-paths.ts:239-246` and then `deliver.ts:1164-1174`, producing both the legacy `⚠️ Media failed.` text and the new detailed dropped-media notice.

## Solution Assessment

No. The proposed pipeline is the right maintainable direction, but the normalizer should collect structured drops without also mutating reply text with the old generic warning.
