## Summary

Keep this PR open: current main still has incomplete dropped-media visibility, and this PR is the right area to fix it, but the latest patch can show duplicate warnings when all media in a payload fails normalization.

## What This Changes

The PR adds a DroppedMedia pipeline across reply normalization, routing, outbound delivery, JSON projection, transcript mirroring, and tests so failed MEDIA attachments produce user-visible warnings.

## Best Possible Solution

Revise the PR so a failed attachment is reported exactly once, keep the droppedMedia pipeline, and add the required changelog entry before landing.

## Reproduction Assessment

Yes, source-reproducible. A payload whose MEDIA entries all fail normalization returns text containing the legacy warning plus droppedMedia, and the PR's delivery path then appends a second formatted dropped-media notice when no media remains.

## Solution Assessment

No, not yet. The droppedMedia pipeline is the right maintainable direction, but the all-media-failed path should remove or replace the legacy generic warning so delivery has a single source of user-visible failure text.
