## Summary

Keep open. The PR addresses a real media-drop visibility problem and the author has worked through many review comments, but the latest patch is not mergeable: exact-head CI shows a type/build failure in the transcript mirror path and a regression where silent `NO_REPLY` media failures now emit a warning payload. The PR also still lacks after-fix real behavior proof from a real setup.

## What This Changes

The PR adds a dropped-media warning pipeline so failed `MEDIA:` attachments are surfaced to users, transcript mirrors, and JSON consumers instead of being silently discarded.

## Best Possible Solution

Keep the PR open, fix the dropped-media wrapper access and silent-reply regression, then land only after exact-head CI is green and the contributor adds redacted runtime proof showing the warning in a real media-drop scenario.

## Reproduction Assessment

Yes for the PR blockers: the exact-head CI run reproduces both the TypeScript failure and the silent-media regression test failure. The original media-drop problem is also source-reproducible on current main from the `reply-media-paths.ts` and `agent-runner-payloads.ts` drop paths.

## Solution Assessment

No, not yet. The PR direction is maintainable, but the current patch reads dropped media from the wrong wrapper object and lets `NO_REPLY` media failures become renderable warning payloads, so it needs a focused update before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Read dropped media from the payload wrapper:** `src/infra/outbound/deliver.ts:1908`
  - body: `normalizedPayloads` contains `{ index, payload }` entries, so `p.droppedMedia` does not exist. This is already failing exact-head typecheck with `TS2339`, and even in emitted JS it would skip the dropped-media transcript mirror; collect from `p.payload.droppedMedia` instead.
  - confidence: 0.98
- **[P2] Respect NO_REPLY when media normalization fails:** `src/auto-reply/reply/agent-runner-payloads.ts:58-61`
  - body: The catch path now preserves `droppedMedia` after media normalization fails. For a parsed silent `NO_REPLY` payload, the later silent branch only clears `text`, and `isRenderablePayload` treats the dropped-media metadata as content, so a `NO_REPLY\nMEDIA: ./missing.png` turn emits a warning payload instead of staying silent; exact-head CI catches this at `agent-runner-payloads.test.ts:596`.
  - confidence: 0.94
