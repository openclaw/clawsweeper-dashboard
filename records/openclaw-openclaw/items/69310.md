## Summary

Keep open: the PR addresses a real media-drop visibility gap that current main only partially covers, but the latest head would show duplicate warnings when all media in a payload fails and it lacks the required user-facing changelog entry.

## What This Changes

The PR threads sanitized `droppedMedia` entries through reply media normalization, outbound payload planning, routing, delivery notices, transcript mirroring, JSON projection, and focused tests.

## Best Possible Solution

Keep the dropped-media pipeline direction, but converge on one sanitized user notice per dropped item, preserve transcript/JSON/routing coverage, and document the user-facing fix in the changelog before merge.

## Reproduction Assessment

Yes from source inspection. A payload with text and one failing `MEDIA:` item enters the PR all-dropped branch with legacy warning text plus `droppedMedia`, then delivery appends the detailed dropped-media notice because no media remains.

## Solution Assessment

No, not yet. The proposed direction is maintainable, but the patch should remove or bypass the legacy generic warning once `droppedMedia` is present and add the required changelog entry.
