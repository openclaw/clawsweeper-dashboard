## Summary

Keep open: the PR addresses a real silent-media-drop gap, but the latest head still has blocking correctness defects in the new droppedMedia mirror/notice paths and the external real behavior proof gate is still missing.

## What This Changes

The PR threads dropped-media items through reply media normalization, outbound planning/delivery, JSON projection, transcript mirroring, and tests so failed MEDIA attachments produce user-visible warnings.

## Best Possible Solution

Keep the droppedMedia pipeline, fix the wrapper access in transcript mirroring, make the detailed dropped-media notice the single all-dropped warning source, add focused regressions, then attach redacted real send proof before merge.

## Reproduction Assessment

Yes. Source inspection of current main shows failed MEDIA normalization is caught and continued through without per-item user feedback, and the catastrophic fallback strips media without a structured dropped-media notice.

## Solution Assessment

No. The overall droppedMedia direction is appropriate, but the current PR head has a transcript-mirror property bug, duplicate all-dropped notices, and no required real behavior proof yet.

  - confidence: 0.96

Delivery later appends the new detailed dropped-media notice for the same failure, so users can receive two warnings for one dropped attachment; let the detailed droppedMedia notice be the single warning source.
  - confidence: 0.9
