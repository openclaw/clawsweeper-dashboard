## Summary

Keep open: current main only has a generic all-media-failed warning, so the PR still adds useful missing visibility, but the latest diff would emit both the legacy generic warning and the new detailed dropped-media notice for one all-failed media payload.

## What This Changes

The PR adds a dropped-media metadata pipeline and warning formatter across reply media normalization, routing, outbound delivery, JSON projection, and transcript mirroring.

## Best Possible Solution

Make `DroppedMediaItem` the single source for user-facing attachment failure text, preserving sanitized per-item notices while removing the old generic warning from the dropped-media path.

## Reproduction Assessment

Yes. Source inspection shows a current all-media-failed payload gets legacy warning text in the normalizer, and the PR head then appends the new detailed dropped-media notice during delivery.

## Solution Assessment

No. The PR direction is maintainable, but the all-failed path should suppress the legacy generic warning once `droppedMedia` is present so users receive one clear diagnostic.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

- **[P2] Avoid emitting two drop warnings for one failed payload:** `src/auto-reply/reply/reply-media-paths.ts:240-243`

Drop the legacy warning when `droppedMedia` will carry the detailed notice.
