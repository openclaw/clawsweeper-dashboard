## Summary

Keep open: the dropped-media visibility gap is real and this PR is pointed at the right area, but the current patch should not merge until it stops double-emitting warnings and cleans up validation blockers.

## What This Changes

The PR threads a new dropped-media payload field through reply media normalization, routing, outbound projection, delivery notices, transcript mirroring, and tests.

## Best Possible Solution

Land a single dropped-media notice pipeline that preserves partial and all-failed media diagnostics without duplicating warnings or exposing paths, with focused tests and a changelog entry.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes: current main source shows the per-item media catch continues after verbose logging, and mixed valid/invalid media has no user-facing notice for the dropped item; the duplicate-warning regression in the PR is also source-reproducible from the all-media-failed branch.

## Solution Assessment

Is this the best way to solve the issue? No, not yet: carrying structured droppedMedia through delivery is the right shape, but the patch needs to remove the legacy generic warning path, clean unused bindings, and add the required changelog before it is mergeable.
