The PR threads dropped-media metadata through reply normalization, outbound delivery, JSON projection, routing, and tests so failed MEDIA directives can produce user-visible attachment warnings.

## Best Possible Solution

Keep the PR open and land it after the all-failed MEDIA path emits exactly one specific dropped-media notice, unused delivery code is removed, and focused regression coverage is added.
