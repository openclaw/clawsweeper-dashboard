## Summary

Keep open. The PR addresses a real gap in current main, but the current head is not mergeable: CI reports lint and reply-agent-runner failures, and the all-media-failed path can show both the old generic warning and the new detailed dropped-media notice.

## What This Changes

The PR adds a DroppedMediaItem pipeline for failed MEDIA normalization, preserves dropped-media-only payloads through reply routing and outbound projections, formats sanitized user/transcript notices, upgrades warning logs, and adds focused tests.

## Best Possible Solution

Keep the dropped-media pipeline, but make it merge-ready by preserving the established silent-turn contract or explicitly updating it, emitting one detailed notice per failed-media case, cleaning lint blockers, adding the required changelog entry, and rerunning focused reply/outbound tests plus the changed gate before maintainer merge.

## Reproduction Assessment

Yes. Current main can be inspected directly: mixed media failures hit logVerbose plus continue, and catastrophic normalization clears media without droppedMedia. The PR head also has live CI annotations reproducing the new silent-media test failure and lint blockers.

## Solution Assessment

No, not as-is. The approach is directionally maintainable for non-silent failed MEDIA directives, but it needs a narrower repair around NO_REPLY semantics, duplicate warning composition, lint cleanup, and changelog coverage before this is the best merge path.
