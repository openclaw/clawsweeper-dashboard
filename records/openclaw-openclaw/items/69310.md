## Summary

Keep open. Current main still has the media-drop visibility gap for mixed failed media, and this PR is a valid direction, but the latest head is not merge-ready because public CI reproduces a silent-reply regression and lint failures; the user-facing fix also needs a changelog entry before merge.

## What This Changes

The PR adds a DroppedMediaItem pipeline for failed MEDIA normalization, threads dropped media through reply routing/outbound projections, emits user/transcript notices, upgrades warning logs, and adds focused dropped-media tests.

## Best Possible Solution

Keep the PR active, preserve dropped-media notices for non-silent failed media while keeping the existing NO_REPLY suppression contract, remove the stale imports, add the required changelog entry, and rerun the focused auto-reply/outbound tests plus changed-gate CI before merge.

Yes. The current-main media-drop behavior is visible in source, and PR-head CI gives a high-confidence regression path via `src/auto-reply/reply/agent-runner-payloads.test.ts:420`, where a silent failed-media payload is now emitted instead of suppressed.

## Solution Assessment

No, not as currently patched. The droppedMedia pipeline is the right general direction, but it needs a narrower renderability/silent-policy adjustment, lint cleanup, and changelog coverage before it is the best maintainable fix.
