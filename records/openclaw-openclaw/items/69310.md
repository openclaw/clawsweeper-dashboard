## Best Possible Solution

Fix the wrapper access in the mirror aggregation, add focused coverage for transcript mirroring and all-media-fail notice output, avoid duplicate legacy warnings, then require redacted real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P1] Read dropped media from the wrapped payload:** `src/infra/outbound/deliver.ts:1908`
  - body: `normalizedPayloads` is an array of `{ index, payload }`, so this reads a field that does not exist. CI already reports this as a type error, and the mirror path would never collect dropped-media notices; read `payload.droppedMedia` from the wrapper instead.
  - confidence: 0.98
- **[P2] Avoid sending both media failure warnings:** `src/auto-reply/reply/reply-media-paths.ts:243`
  - body: When every media item fails, this branch still appends the legacy `Media failed` text while also returning `droppedMedia`, and the no-media delivery path appends the new detailed notice. That makes users see two warnings for one failed attachment; suppress the legacy text when the detailed notice will be emitted.
  - confidence: 0.88
