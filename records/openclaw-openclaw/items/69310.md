## Summary

Keep open: this is a plausible focused bug-fix PR, but the latest head is not mergeable because it breaks typecheck and an existing silent-reply behavior test, and it still lacks real after-fix channel proof.

## What This Changes

The branch adds a DroppedMedia pipeline that records failed MEDIA normalization, carries it through reply/outbound projections, renders user-visible attachment warnings, mirrors them to transcripts, and adds regression tests.

## Best Possible Solution

Fix the current type and silent-reply blockers, remove unused imports, then verify the warning in a real channel flow before deciding whether the richer DroppedMedia pipeline should replace the current simpler warning path.

## Reproduction Assessment

Yes for the PR defects: current PR-head CI reproduces the type error and the existing silent-media regression test failure. The original user-visible media-drop problem is also source-backed by the current normalizer and the linked issue discussion.

## Solution Assessment

No for this revision: the proposed direction is maintainable, but the implementation currently breaks typecheck and changes silent reply suppression semantics. The narrowest fix is to read dropped media from the normalized payload wrapper and keep `NO_REPLY` media failures suppressed unless there is real deliverable media or an explicit error policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Read dropped media from the payload wrapper:** `src/infra/outbound/deliver.ts:1921`
  - body: `normalizedPayloads` entries are `{ index, payload }`, so reading `p.droppedMedia` does not compile and would collect no transcript mirror notices. Use `p.payload.droppedMedia` when building `allDropped`.
  - confidence: 0.98
- **[P2] Keep silent dropped-media replies suppressed:** `src/auto-reply/reply/reply-payloads-base.ts:85`
  - body: Treating every dropped-media-only payload as renderable lets `NO_REPLY\nMEDIA: ./missing.png` survive after the silent token clears text, which fails the existing silent-media test and would send a warning for a reply the agent explicitly marked silent. Gate dropped-media renderability so silent replies without deliverable media still drop.
  - confidence: 0.9
