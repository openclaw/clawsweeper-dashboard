## Summary

Keep open; this PR is useful but not merge-ready. The prior `reason` vs `code` test-shape blocker appears fixed at the latest head, but the all-media-failed path now shows duplicate user warnings because the normalizer keeps the legacy generic warning text while delivery also formats the new dropped-media notice.

## What This Changes

The PR adds dropped-media tracking for failed MEDIA normalization, carries it through reply routing and outbound payload projections, emits user/transcript notices, upgrades warning logs, and adds focused tests.

## Best Possible Solution

Land the dropped-media notification pipeline after the all-failed MEDIA path produces exactly one sanitized attachment-failure notice across direct, routed, and followup delivery, with focused tests covering that final user-visible output.

## Reproduction Assessment

Yes. Source tracing the PR head is enough: an all-failed media normalization result contains both legacy `⚠️ Media failed.` text and `droppedMedia`, and delivery later appends `Attachment not sent` for no-media dropped-media payloads.

## Solution Assessment

No. The PR direction is maintainable, but the all-failed path should remove or suppress the legacy generic warning once `droppedMedia` is populated, rather than sending both diagnostics to the user.
