## Summary

Keep open: current main still has source-visible silent-drop paths, and this active PR is useful, but it is not merge-ready because it lacks real behavior proof and currently double-reports all-failed media with both the legacy warning and the new detailed notice.

## What This Changes

The branch adds DroppedMediaItem plumbing through reply normalization, routing, outbound JSON, delivery, transcript mirroring, and tests so dropped MEDIA items produce user-visible attachment warnings.

## Best Possible Solution

Keep the droppedMedia pipeline, remove the legacy generic warning when droppedMedia will render the detailed notice, add regression coverage for the all-media-failed path, and require redacted real behavior proof before merge.

## Reproduction Assessment

Yes from source inspection, but not by executing a live channel flow: current main catches and continues dropped media in reply-media-paths, and the mixed-success path returns only surviving media without any dropped-item warning.

## Solution Assessment

No, not as-is: the droppedMedia design is the right fix direction, but this implementation should stop carrying the old generic warning alongside the new detailed notice and needs real after-fix behavior proof.
