## Summary

Keep open. Current main still silently loses mixed-media normalization failures, so the PR addresses a real gap, but the latest patch is not merge-ready because the all-media-failed path now produces both the legacy generic warning and the new dropped-media notice, and the user-facing fix lacks the required changelog entry.

## What This Changes

The PR adds a dropped-media payload pipeline for failed MEDIA normalization, renders user/transcript notices, preserves dropped-media-only payloads through routing/outbound projections, updates warning logs, and adds focused tests.

## Best Possible Solution

Keep the droppedMedia direction, but make the all-failed and partial-failed paths converge on one sanitized user notice, remove stale dead code from the delivery implementation, add the required changelog credit, and land only after focused reply/outbound tests plus the changed gate are green.

## Reproduction Assessment

Yes. The original gap is reproducible from current main by following the per-item catch in `createReplyMediaPathNormalizer`: one invalid MEDIA item is logged and skipped while surviving media continues without a detailed user notice. The PR regression is reproducible from the all-invalid branch because it appends the legacy warning before delivery appends the new dropped-media notice.

## Solution Assessment

No, not yet. The PR is the right narrow area for the fix, but it should replace the legacy all-failed warning with the new droppedMedia notice instead of stacking both, and it needs the repo-required changelog entry for a user-visible fix.
