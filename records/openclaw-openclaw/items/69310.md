## Summary

Keep this PR open: the dropped-media bug is valid on current main, but the proposed patch still has narrow lint and user-visible behavior blockers before it is merge-ready.

## What This Changes

This PR adds dropped-media metadata through reply normalization, routing, outbound delivery, transcript mirroring, JSON projection, and tests so failed MEDIA attachments produce user-visible warnings.

## Best Possible Solution

Keep the dropped-media pipeline direction, remove the lint blockers, suppress the legacy generic media-failed text when structured dropped-media notices will render, and land with focused regression coverage plus the changed gate.

## Reproduction Assessment

Yes from source inspection: current main still has paths where MEDIA normalization failures are logged or converted to a generic fallback without per-attachment user-visible detail. I did not run a live channel reproduction because this is a read-only PR review.

## Solution Assessment

No, not as currently proposed; the design is the right narrow fix, but the current diff has lint-blocking unused imports and a duplicate-warning path when all media fail. The maintainable solution is to keep the structured dropped-media metadata but remove the unused imports and replace, rather than stack on top of, the legacy generic warning.
