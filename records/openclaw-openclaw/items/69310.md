## Summary

Keep open. This PR addresses a real source-reproducible dropped-media visibility gap that current main still has for mixed and catastrophic media-normalization failures, but current head needs one small correctness fix and after-fix real behavior proof before merge.

## What This Changes

The branch adds DroppedMedia metadata, sanitization and reason-code helpers, outbound warning delivery, transcript/JSON propagation, routing preservation, and regression tests for failed MEDIA attachments.

## Best Possible Solution

Land the dropped-media pipeline after removing the legacy generic all-failed warning and adding real after-fix proof that a failed MEDIA directive produces one clear user-visible attachment notice.

## Reproduction Assessment

Yes from source inspection: current main still drops failed items silently in mixed-success normalization and clears all media in the catastrophic catch without an attachment-specific user notice. I did not run a live channel reproduction in this read-only pass.

## Solution Assessment

No, not as-is. The DroppedMedia boundary is the right maintainable direction, but the current branch double-warns in the all-failed path by preserving the old generic warning before adding the new detailed notice.
