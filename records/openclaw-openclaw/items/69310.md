  - confidence: 0.99
- **[P2] Remove the unused config-dir import:** `src/auto-reply/reply/reply-media-paths.ts:18`
  - body: `resolveConfigDir` is imported but never used, and the latest `check-prod-types` run reports TS6133 for this line. Drop the import before merge.
  - confidence: 0.99
- **[P2] Honor silent replies before rendering dropped media:** `src/auto-reply/reply/reply-payloads-base.ts:85`
  - body: After a `NO_REPLY` media normalization failure clears the text, this unconditional dropped-media renderability keeps a warning payload alive; CI shows the existing silent-turn test now gets one payload instead of zero. Clear/drop `droppedMedia` for silent replies or gate this path through the silent policy.
  - confidence: 0.95
- **[P3] Stop duplicating the legacy media warning:** `src/auto-reply/reply/reply-media-paths.ts:240-243`
  - body: When every media item fails, this branch still appends the old `⚠️ Media failed.` text while also carrying `droppedMedia`; delivery later appends the detailed `Attachment not sent` notice, so users can see two warnings for one failure. Let the new dropped-media notice replace the legacy generic text in this path.
  - confidence: 0.9
