## Summary

Keep open. Current main still has a valid mixed-media gap, because per-item media normalization failures are logged and skipped when another attachment survives, so this PR addresses a real user-visible problem. The latest head is close but not merge-ready: the all-media-failed path can now produce both the legacy generic warning and the new formatted notice, the user-facing fix has no changelog entry, and exact-head CI currently has failing check summaries.

## What This Changes

The PR adds a `DroppedMediaItem` pipeline for failed `MEDIA:` normalization, preserves dropped-media-only payloads through reply/outbound routing, formats sanitized user/transcript notices, upgrades warning logs, and adds focused tests.

## Best Possible Solution

Land this PR or an equivalent generic core fix after cleanup, keeping the channel-agnostic `droppedMedia` metadata, sanitized display names, JSON/transport preservation, and user/transcript notice behavior while removing the legacy duplicate all-dropped warning and documenting the shipped user-facing fix in the changelog.

## Reproduction Assessment

Yes. A current-main payload with one invalid `MEDIA:` source and one valid media source reproduces the silent mixed-drop path by taking the per-item catch/`continue` and returning only the surviving media; the PR's added data-URL plus valid-URL test exercises the same normalization branch.

## Solution Assessment

Mostly yes, but not as-is. A payload-level dropped-media pipeline is the narrow maintainable core solution for this cross-channel behavior, but the branch should remove the duplicate legacy warning, add the required changelog entry, and get the exact-head checks green before merge.
