- **[P1] Read droppedMedia from the wrapped payload:** `src/infra/outbound/deliver.ts:1921`
  - body: `normalizedPayloads` contains `{ index, payload }` entries, so this line reads `droppedMedia` from the wrapper instead of from `payload`. `check-prod-types` now fails with `Property 'droppedMedia' does not exist on type 'NormalizedPayloadForChannelDelivery'`, and the same run also reports stale unused imports added by this branch.   - confidence: 0.97
- **[P2] Suppress dropped-media payloads for silent replies:** `src/auto-reply/reply/agent-runner-payloads.ts:65-68`
  - body: When `normalizeMediaPaths` throws for a parsed `NO_REPLY` payload, this catch adds `droppedMedia`, making the otherwise silent payload renderable after its text is cleared. The existing CI test for `NO_REPLY\nMEDIA: ./missing.png` now gets one payload instead of none, so users can receive warning text during a silent turn. Drop or suppress the synthesized dropped-media entry when the parsed reply is silent and no media survived.
  - confidence: 0.94
