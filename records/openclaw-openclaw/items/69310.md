## Summary

Keep this PR open: the underlying dropped-media bug is still valid beyond current main's generic all-dropped warning, but the latest branch has blocking type/test failures, duplicate-warning behavior, and no after-fix real behavior proof.

## What This Changes

The PR adds a dropped-media metadata pipeline, user/transcript warning formatting, routing/outbound propagation, warning logs, and tests for failed `MEDIA:` attachment normalization.

## Best Possible Solution

Keep the dropped-media pipeline direction, but fix the compile/test regressions, honor silent turns, remove duplicate legacy warnings, and add redacted real channel or CLI proof before merge.

## Reproduction Assessment

Yes. Source inspection shows current main still silently loses partial and catastrophic media failures, and CI reproduces the PR's new type/test regressions.

## Solution Assessment

No, not in the current PR shape. The approach is maintainable, but it must read dropped media through the payload wrapper, preserve silent-reply semantics, remove unused imports and duplicate warnings, and provide real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P1] Read dropped media from the payload wrapper:** `src/infra/outbound/deliver.ts:1921`
  - body: `normalizedPayloads` entries are `{ index, payload }`, so `p.droppedMedia` does not type-check and would not collect mirror notices. Read `p.payload.droppedMedia` here instead.
  - confidence: 0.98
- **[P2] Keep silent media failures silent:** `src/auto-reply/reply/agent-runner-payloads.ts:58-60`
  - body: The catch now returns `droppedMedia`, and the existing `parsed.isSilent` branch only clears text when media is gone. That leaves a renderable dropped-media-only payload for `NO_REPLY\nMEDIA:...`, failing the existing silent-turn test and sending a warning when the model explicitly chose silence.
  - confidence: 0.94
- **[P2] Remove unused delivery imports:** `src/infra/outbound/deliver.ts:1-4`
  - body: `sendMediaWithLeadingCaption`, `chunkByParagraph`, and `chunkMarkdownTextWithMode` are imported but not referenced. Core TypeScript enables `noUnusedLocals`, so these will block typechecking once the earlier type error is fixed.
  - confidence: 0.93
- **[P2] Remove the unused config-dir import:** `src/auto-reply/reply/reply-media-paths.ts:18`
  - body: `resolveConfigDir` is imported but never used. With `noUnusedLocals` enabled, this is another typecheck blocker on the PR branch.
  - confidence: 0.95
- **[P2] Avoid sending duplicate all-dropped warnings:** `src/auto-reply/reply/reply-media-paths.ts:240-246`
  - body: When every media item is dropped, this branch still appends the legacy `Media failed` text and also carries `droppedMedia`, which `deliver.ts` later renders as the new detailed notice. Users would see two warnings for the same dropped attachment.
  - confidence: 0.9
