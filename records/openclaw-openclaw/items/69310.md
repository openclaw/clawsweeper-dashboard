## What This Changes

The branch threads dropped-media metadata through reply normalization, routing, outbound delivery, JSON projection, transcript mirroring, and tests so failed MEDIA attachments can produce user-visible warnings.

## Best Possible Solution

Keep the dropped-media pipeline active, fix the branch typecheck/runtime issues, remove legacy warning duplication, and require a redacted real run before merge.

## Reproduction Assessment

Yes, by source inspection. Current main still drops per-item media failures without a visible notice when another media item survives, and the catastrophic normalizer catch clears media without carrying a dropped-media warning.

## Solution Assessment

No, not in the latest PR shape. The approach is maintainable, but the branch must fix the typecheck blockers, mirror collection bug, duplicate warning, and proof gap before it is the best solution.
