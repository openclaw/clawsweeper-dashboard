## Summary

Keep open. Current main still only supports `streamTo="parent"`, so the PR is not obsolete, but the proposed patch has a blocking Slack/Mattermost relay-target bug and a missing user-facing changelog entry before it is merge-ready.

## What This Changes

The PR adds `streamTo="thread"` to ACP `sessions_spawn`, relays ACP progress directly to bound channel threads, updates ACP spawn/relay tests and docs, and includes two CI-related test expectation fixes.

## Best Possible Solution

Keep the feature in core, but revise the relay target resolver so Slack/Mattermost current-thread bindings retain or derive the parent channel when `parentConversationId` is absent, add a regression test for that missing-parent case, and include the required changelog entry before merge.

## Reproduction Assessment

Yes. A unit-level reproduction can bind a Slack or Mattermost current conversation with `conversationId` set to the thread id and no `parentConversationId`; the PR resolver then falls through and targets `channel:<thread_id>` instead of `channel:<parent>` with `threadId`.

## Solution Assessment

No. The overall feature direction is maintainable, but this patch is not the best mergeable form until the Slack/Mattermost fallback is fixed and the user-facing changelog entry is added.
