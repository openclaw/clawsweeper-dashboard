## Summary

Keep open: current main still only supports ACP `streamTo="parent"`, and this PR is the canonical implementation path for the closed linked feature request, but it is not mergeable as-is because Slack/Mattermost current-thread bindings can fall through to `channel:<thread_ts>` without a `threadId` and the external PR has no real platform proof.

## What This Changes

The PR extends ACP `sessions_spawn` with `streamTo="thread"`, routes ACP progress through gateway sends to bound thread targets, updates ACP spawn/relay tests and docs, and adjusts two test expectations.

## Best Possible Solution

Keep the feature in core, but fix the resolver to preserve or derive the parent channel for Slack/Mattermost thread relays or reject the missing-parent case, add focused regression coverage, then provide redacted real Discord/Slack proof before merge.

## Reproduction Assessment

Yes for the PR blocker at source level: bind a Slack or Mattermost current conversation with `conversationId` set to the thread timestamp and no `parentConversationId`; the PR resolver falls through to a non-thread `channel:<thread_ts>` target. I did not run a live platform smoke test.

## Solution Assessment

No. The feature direction is maintainable, but this patch needs the Slack/Mattermost missing-parent target handling fixed and proven before it is the best mergeable solution.
