## Summary

Keep open: current main still lacks the requested ACP thread stream target, and this branch remains useful but is not merge-ready because the Slack/Mattermost relay fallback can target the thread id as a channel and the required changelog entry is missing.

## What This Changes

The PR adds `streamTo="thread"` to ACP `sessions_spawn`, sends ACP relay progress to bound channel threads, updates ACP spawn/relay coverage and docs, and includes two test expectation fixes.

## Best Possible Solution

Keep the feature in core, but fix the relay target resolver to preserve or derive the parent channel for Slack/Mattermost current-thread bindings, add regression coverage for the missing-parent case, and add the active changelog entry before merge.

## Reproduction Assessment

Yes. A unit-level reproduction can bind a Slack or Mattermost current conversation with `conversationId` set to the thread id and no `parentConversationId`; the PR resolver then falls through to `channel:<thread_id>` without `threadId`.

## Solution Assessment

No. The overall feature direction is maintainable, but this patch is not the best mergeable form until the Slack/Mattermost fallback is fixed and the user-facing changelog entry is added.
