Keep open: current main still lacks ACP `streamTo="thread"`, and this useful PR is blocked by a source-proven Slack/Mattermost relay target defect plus missing real behavior proof.

## What This Changes

The PR adds ACP `sessions_spawn` `streamTo="thread"` support, direct ACP progress relay to bound thread targets, docs, ACP spawn/relay tests, and two unrelated test expectation fixes.

## Best Possible Solution

Refresh the branch on current main, preserve or derive the parent channel for Slack/Mattermost thread relay targets or reject unsupported cases with coverage, then add redacted live Discord or Slack proof.

## Reproduction Assessment

Yes, at source level: on the PR branch, a Slack or Mattermost current-thread binding with `conversationId` set to the thread timestamp and no `parentConversationId` falls through to a non-thread `channel:<thread_ts>` target. I did not run a live platform smoke test.

## Solution Assessment

No. The feature direction is maintainable and current main lacks it, but this patch needs the Slack/Mattermost missing-parent relay target fixed and real platform proof before it is mergeable.
