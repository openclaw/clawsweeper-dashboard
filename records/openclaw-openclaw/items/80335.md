## Summary

Keep open. The PR addresses a plausible Gateway event-pressure bug, but the patch still has two blocking correctness issues: a post-window text event can discard a buffered delta, and the normal abort path does not receive the new cleanup maps. The external proof is also mock-only, so it should not merge yet.

## What This Changes

The PR adds stream-specific throttling and buffering for assistant/thinking Gateway `agent` events, plus cleanup state and focused Gateway tests.

## Best Possible Solution

Keep the stream-specific throttle approach, but merge or flush buffered deltas before any later send and make the abort cleanup maps part of the required abort contract with regression coverage.

## Reproduction Assessment

Yes, from source inspection: deltas at 0 ms, 50 ms, and 200 ms take the PR path that buffers the middle delta, then deletes that buffer and emits only the latest payload. I did not execute the PR branch because this review is read-only.

## Solution Assessment

No. The throttling direction is reasonable, but the patch is not the best fix until it preserves byte-for-byte delta bursts and wires cleanup through the real abort caller.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Flush buffered deltas before sending later chunks:** `src/gateway/server-chat.ts:696-697`
  - body: When a third assistant/thinking delta arrives after the throttle window, this branch deletes the existing buffered payload and broadcasts only the current payload. Raw `agent` stream consumers that apply `data.delta` will miss the middle text, so flush or merge the buffered entry before sending the later chunk.
  - confidence: 0.91
- **[P2] Require abort cleanup maps in chat abort ops:** `src/gateway/chat-abort.ts:116-117`
  - body: The new maps are optional, and the real `createChatAbortOps()` caller still does not pass them, so normal aborts can leave `agentDeltaSentAt` and `bufferedAgentEvents` behind. Make these fields required and pass the chat-run state maps from the Gateway abort caller so stale agent text cannot flush after abort.
  - confidence: 0.88
