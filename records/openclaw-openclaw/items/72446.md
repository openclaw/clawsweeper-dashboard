## Summary

Keep this PR open for normal maintainer review and merge gating. Current main still offers and persists Claude deployments through the Microsoft Foundry OpenAI-compatible route, while the PR’s latest patch addresses both the picker and saved catalog paths with focused tests and a changelog entry.

## What This Changes

The PR filters Anthropic/Claude Microsoft Foundry deployments during discovered onboarding, persists only supported deployments, adds tests, and updates the changelog.

## Best Possible Solution

Land this provider-local guardrail after exact-head CI and maintainer review, while leaving first-class Azure Claude Messages onboarding to the broader separate work in #47181.

## Reproduction Assessment

Yes. A source-level reproduction exists on current main: discovered deployments feed the picker and persisted catalog unfiltered, while runtime URL normalization always routes the Microsoft Foundry provider through /openai/v1.

## Solution Assessment

Yes. For the guardrail option, a shared provider-local partition used by both selection and persistence is the narrowest maintainable fix; full Anthropic Foundry support should remain a separate provider/onboarding change.
