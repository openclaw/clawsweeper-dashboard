## Summary

Keep this PR open. The guardrail is useful and provider-scoped, but the current patch only filters the interactive deployment picker; the Entra onboarding path still persists the original discovered deployment list into the Microsoft Foundry model catalog, so a mixed GPT-plus-Claude resource can still expose the unsupported Claude deployment after onboarding.

## What This Changes

The PR adds a Microsoft Foundry onboarding guardrail that detects Claude model names, hides Anthropic deployments from the picker with custom-provider guidance, errors when only unsupported deployments exist, and adds detector tests.

## Best Possible Solution

Finish the provider-local Option B guardrail by deriving one supported-deployment list and using it consistently for picker selection, persisted catalog generation, and regression tests; leave full Anthropic-on-Azure Foundry runtime support to a separate provider/API implementation such as the broader Azure Claude work.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a discovered Foundry resource containing both GPT and Claude deployments: the patch hides Claude in `selectFoundryDeployment`, but current `auth.ts` still persists the raw `resourceDeployments` list into the provider catalog.

## Solution Assessment

No. The guardrail direction is narrow and maintainable, but this patch is incomplete because it filters only the prompt surface; the safer fix is to reuse the filtered supported set for both selection and persisted model catalog creation.
