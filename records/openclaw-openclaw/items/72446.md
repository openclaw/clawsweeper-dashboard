## Summary

Keep this PR open. Current main still exposes all succeeded Microsoft Foundry deployments in onboarding, persists the raw discovered catalog, and routes the provider through the OpenAI-compatible `/openai/v1` path; the PR is a narrow provider-local guardrail that filters Claude deployments from both picker and catalog with focused regression coverage.

## What This Changes

The PR adds a Microsoft Foundry deployment partition helper, filters Anthropic/Claude deployments during onboarding, persists only supported deployments, adds regression tests, and records the user-facing fix in the changelog.
