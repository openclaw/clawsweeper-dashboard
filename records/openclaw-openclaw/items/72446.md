## Summary

Keep this PR open. The guardrail is useful and targets the right provider-owned surface, but the patch still allows unsupported Claude deployments to be persisted in the Microsoft Foundry model catalog and also lacks the required user-facing changelog entry.

## What This Changes

The PR adds a Microsoft Foundry onboarding guardrail that detects Claude model names, omits Anthropic deployments from the deployment picker with user guidance/error handling, and adds detector tests.

## Best Possible Solution

Keep the guardrail provider-local, derive one supported deployment collection, use it for both prompt selection and saved provider models, cover mixed and all-Anthropic resource cases, and leave full Azure Claude Messages support to separate work such as #47181.

## Reproduction Assessment

Yes. A mixed Azure resource with one GPT deployment and one Claude deployment reproduces the source-level bug: the PR filters the picker but `auth.ts` still persists the unfiltered `resourceDeployments` catalog.

## Solution Assessment

No. The guardrail direction is reasonable, but this implementation is incomplete until the same supported deployment set feeds persisted catalog generation and tests.
