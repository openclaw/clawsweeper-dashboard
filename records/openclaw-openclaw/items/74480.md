## What This Changes

The PR centralizes Active Memory's embedded recall timeout budget and updates the setup-grace unit test so the embedded runner receives `timeoutMs + setupGraceTimeoutMs`.

## Best Possible Solution

Yes. The Active Memory unit harness can configure a tiny `timeoutMs`, add setup grace, simulate setup lasting longer than the raw timeout, and assert that `runEmbeddedPiAgent` receives the grace-extended timeout.

## Solution Assessment

Yes, with the changelog fix. A shared effective-timeout helper is the narrowest maintainable solution for the existing setup-grace contract because it keeps the embedded runner, watchdog, and hook registration on the same budget.
