The PR adds optional realtime voice-call background email delivery for caller-requested email responses or consult timeouts, with new realtime config fields, a delivery helper, consult-tool schema changes, and SDK exports.

## Best Possible Solution

Land this feature only after recipient availability is proven before acknowledgement, the full background research-and-delivery chain is tracked through shutdown, exec-based delivery is explicitly approved and documented, focused tests cover the new branches, and redacted Twilio-to-email proof is attached.

## Solution Assessment

No. The direction may be useful, but this patch should not promise email before a recipient is known, should track the complete background chain, and should get explicit approval for the exec-capable delivery path before merge.
