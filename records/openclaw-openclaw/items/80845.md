## Summary

Keep open: this is a new voice-call feature with concrete security and correctness blockers around tool-policy bypass and sensitive runtime logging, plus insufficient after-fix proof for an external PR.

## What This Changes

The PR adds asynchronous email delivery for realtime voice-call consult results, including new realtime config keys, a background email-delivery helper, consult-tool delivery preference parsing, and SDK exports.

## Best Possible Solution

Define a voice-call async delivery design that preserves realtime.toolPolicy, uses an explicit email-delivery capability or owner-gated tool surface, avoids logging caller content/results, and includes targeted runtime tests plus real Twilio/email proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a new feature. The blocking concerns are source-reproducible from the PR head: the new background delivery call lacks toolsAllow and added logs include caller content and consult result snippets.

## Solution Assessment

No. The current approach is not the safest maintainable path because it introduces background command-capable delivery without preserving the existing realtime consult policy boundary and stores sensitive call content in logs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Preserve the realtime tool policy for email delivery:** `extensions/voice-call/src/background-email-delivery.ts:98-107`
  - body: The new email-delivery agent starts a second `consultRealtimeVoiceAgent` without passing `toolsAllow`, even when the voice-call config uses the default `safe-read-only` policy. Because the prompt asks that agent to use exec-based email tools, a caller-triggered email path can bypass the configured realtime consult tool boundary; gate this behind an explicit delivery capability or pass a constrained allow-list that matches the intended policy.
  - confidence: 0.91
- **[P1] Stop logging caller prompts and consult answers:** `extensions/voice-call/src/runtime.ts:362-364`
  - body: These new info logs persist JSON-stringified tool args, partial transcripts, and answer snippets. Voice consults can include phone-call PII, customer data, query results, or secrets, so keep the logs to structural IDs/statuses or redact the content before merge.
  - confidence: 0.88
