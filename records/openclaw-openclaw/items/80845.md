  - body: Adding `backgroundEmailDeliveryEnabled: z.boolean().optional().default(false)` makes the parsed realtime config require that property, but the object default and fixtures omit it, which breaks prod and test type checks. Add the field to the default config/fixtures or adjust the schema shape so the output type remains compatible.
  - confidence: 0.9
- **[P2] Track the full deferred consult chain for shutdown:** `extensions/voice-call/src/runtime.ts:447-452`
  - body: The runtime returns before the background consult is registered anywhere, and `stop()` only waits for `pendingBackgroundDeliveries`, which is populated after research resolves. If the runtime stops while research is still running, delivery may never be queued, so track the complete research-and-delivery promise before returning the ack.
  - confidence: 0.86
- **[P2] Require a recipient before promising email delivery:** `extensions/voice-call/src/runtime.ts:466`
  - body: Both deferred paths can acknowledge that an email will be sent while `recipientEmail` is unset, leaving the helper to perform only a best-effort memory lookup later. Resolve or require a deliverable recipient before returning the email ack, or fall back to the normal voice path.
  - confidence: 0.87
