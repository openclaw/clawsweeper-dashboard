## Summary

Keep open: the latest branch addresses the earlier logging/tool-policy direction, but it still has blocking correctness issues, missing inspectable real behavior proof, and a security/product concern around caller-triggered exec-capable email delivery.

## What This Changes

The PR adds optional realtime voice-call background email delivery via new realtime config fields, a delivery preference tool argument, a background consult/email helper, SDK exports, and a small consult-tool test update.

## Best Possible Solution

Land async voice email delivery only after recipient availability is proven before acknowledgement, the full research-and-delivery chain is tracked for shutdown, exec-based delivery is approved and documented, focused tests are added, and redacted Twilio/email proof is attached.

## Reproduction Assessment

Not applicable as a user-reported bug; this is a new feature PR. The blocking defects are source-reproducible from the PR head: the deferred paths can ack without a known recipient, and shutdown only tracks delivery tasks after research finishes.

## Solution Assessment

No. The feature direction may be useful, but the current patch should require or resolve a recipient before acking, track the complete background chain, and make the exec-based delivery capability explicit before merge.
