## Summary

Keep open: the feature direction is plausible, and later commits addressed the earlier policy/logging/timer comments, but the patch still promises email without a verified recipient path, does not track background research promises during shutdown, and lacks inspectable after-fix Twilio/email proof.

## What This Changes

The PR adds optional realtime voice-call background email delivery with new config fields, a delivery preference tool argument, background consult/email paths, SDK exports, and a small consult-tool test update.

## Best Possible Solution

Land async voice delivery only after it has recipient resolution before acknowledgement, shutdown tracking for the full research-and-delivery chain, focused tests, documentation/security review for exec-based delivery, and real Twilio/email proof.

## Reproduction Assessment

Not applicable as a user-reported bug; this is a new feature PR. The introduced defects are source-reproducible from the PR diff: the deferred paths can ack without a known recipient, and only email-delivery tasks are tracked for shutdown.

## Solution Assessment

No. The direction may be useful, but the current patch should require or resolve a deliverable recipient before acking, track the complete background chain, and make the exec-based delivery capability explicit before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Verify the recipient before returning the email ack:** `extensions/voice-call/src/runtime.ts:443-480`
  - body: Both deferred paths return the email acknowledgement as soon as `backgroundEmailDeliveryEnabled` is true, even when `recipientEmail` is unset. The helper only tries a later memory lookup and logs a warning if it cannot find an address, so callers can be told an email is coming when no deliverable destination is known; resolve or require the address before acking, or keep the call on the normal voice path.
  - confidence: 0.87
- **[P2] Track the research promise for shutdown:** `extensions/voice-call/src/runtime.ts:447-452`
  - body: The stop path only flushes `pendingBackgroundDeliveries`, but the email-first and timeout branches return before the background `consultPromise` is added to that set. Stopping the runtime while research is still running can tear down resources before the consult resolves and before `spawnEmailDeliveryAgent` is queued, so track the research-and-delivery chain as background work.
  - confidence: 0.84
