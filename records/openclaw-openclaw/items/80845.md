## Summary

## What This Changes

The branch adds async email delivery for realtime voice-call consults, including new realtime config keys, delivery preference parsing, a background email helper, SDK exports, and a small consult-tool test update.

## Best Possible Solution

Design async voice-call delivery as an explicit, reviewed voice-call capability that preserves the existing consult policy boundary, redacts content logs, clears timeout resources, documents the new config, and includes focused tests plus real Twilio/email proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature PR. The blocking concerns are high-confidence from source review of the PR head against current main's documented realtime tool-policy contract.

## Solution Assessment

No. The async delivery idea may be useful, but this implementation is not the best path until it preserves or explicitly replaces the realtime toolsAllow boundary, removes sensitive content logging, and fixes the timeout race cleanup.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve the realtime tool policy for email delivery:** `extensions/voice-call/src/background-email-delivery.ts:98-119`
  - body: The new email-delivery agent starts a second `consultRealtimeVoiceAgent` run without passing `toolsAllow`, while the prompt asks that agent to use exec-based email tools. Under `safe-read-only`, the normal voice consult path is limited to read/web/memory tools, so this caller-triggered path can exceed the configured realtime policy; gate delivery behind an explicit capability or pass a constrained allow-list.
  - confidence: 0.91
- **[P1] Stop logging caller prompts and consult answers:** `extensions/voice-call/src/runtime.ts:362-364`
  - body: This new log persists JSON-stringified tool args and partial transcripts, and the added email/timeout logs also include original questions and answer snippets. Voice consults can contain phone-call PII, customer data, query results, or secrets, so keep these logs to structural IDs/statuses or redact the content before merge.
  - confidence: 0.89
- **[P2] Clear the timeout when the consult wins:** `extensions/voice-call/src/runtime.ts:489-490`
  - body: When the consult finishes before `backgroundConsultTimeoutMs`, the `setTimeout` created for the race is left scheduled until it fires. That can keep the process alive and accumulate timers on normal successful consults; keep the timer handle and clear it when the consult wins or in a `finally`.
  - confidence: 0.86
