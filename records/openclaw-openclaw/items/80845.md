## What This Changes

The branch adds opt-in background email delivery for realtime voice-call consults with new realtime config fields, a delivery-preference tool argument, runtime email-first and timeout paths, SDK exports, and a background delivery helper.

## Best Possible Solution

Land this only after the branch preserves email-first behavior across fast-context, tracks the full background consult-to-delivery lifecycle, documents or redesigns the exec-capable delivery policy, and includes focused tests plus redacted real Twilio/email proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature PR. The remaining blockers are source-reproducible from the PR head: fast-context can return before deliveryPreference is checked, and shutdown flushes only delivery promises, not the prior background consult.

## Solution Assessment

No. The direction may be useful, but this implementation is not the best merge candidate until email-first semantics, lifecycle tracking, command-capable policy, docs/tests, and real behavior proof are tightened.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Bypass fast context for email delivery:** `extensions/voice-call/src/runtime.ts:398`
  - body: `resolveRealtimeFastContextConsult` still runs before the PR checks `deliveryPreference`. With `realtime.fastContext.enabled`, an explicit email request can be handled as a spoken fast-context hit or miss and never enter the email-first branch, so parse first and skip fast context when the caller asked for email delivery.
  - confidence: 0.87
- **[P2] Track the full background consult before shutdown:** `extensions/voice-call/src/runtime.ts:447-451`
  - body: The email-first and timeout paths create an untracked `consultPromise`, and `pendingBackgroundDeliveries` only starts tracking after that consult resolves and `spawnEmailDeliveryAgent` runs. If the runtime stops while the background consult is still researching, `stop()` can finish without waiting for the eventual email delivery; track the whole consult-to-email task.
  - confidence: 0.84
