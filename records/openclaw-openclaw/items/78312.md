## Summary

Keep open: current `main` still trims each ACP text-delta chunk and inserts a synthetic space between chunks, so the CJK/path corruption report is source-backed. Related PR #74103 is useful but only changes the cap and terminal-summary fallback; it does not fix the chunk-join bug.

## What This Changes

The issue requests preserving ACP background-task progress summary token boundaries instead of trimming streamed chunks and inserting forced spaces.

## Best Possible Solution

Update the ACP progress-summary helper to append streamed text deltas without synthetic boundary spaces, add focused regression tests for CJK, paths, identifiers, whitespace-bearing chunks, empty chunks, and the length cap, and coordinate with #74103 if both changes land near each other.

## Reproduction Assessment

Yes, at the helper/source level: current `main` routes ACP `text_delta` chunks through a trim-and-forced-space join, and the deterministic source-level check reproduces the reported broken CJK/path output. I did not perform a full live ACP run during this read-only review.

## Solution Assessment

Yes, a narrow fix in `appendBackgroundTaskProgressSummary` is the best path. PR #74103 alone is not sufficient because it promotes the already-corrupted progress summary without changing the chunk join logic.
