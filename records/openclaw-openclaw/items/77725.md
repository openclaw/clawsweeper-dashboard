## Summary

Keep open: the PR addresses a real prompt-composition bug, and current main still has CLI and compaction override paths that can drop runtime extra context, but the patch needs a small `promptMode: "none"` correction before it is mergeable.

## What This Changes

The PR appends runtime `extraSystemPrompt` to configured `systemPromptOverride` prompts across CLI, embedded attempt, compaction, docs, config comments, changelog, and regression tests.

## Best Possible Solution

Preserve runtime context with prompt overrides in the remaining CLI and compaction paths, keep the already-merged embedded helper behavior, and add a `promptMode: "none"` guard plus focused regression coverage before merge.

## Reproduction Assessment

Yes. Source inspection shows the CLI and compaction override branches still bypass the generated prompt builder that receives `extraSystemPrompt`, and the contributor supplied after-fix CLI prompt-preparation output.

## Solution Assessment

No as written. Appending runtime extra context to override prompts is the right narrow repair, but the helper must preserve the existing `promptMode: "none"` behavior before this patch is safe to merge.
