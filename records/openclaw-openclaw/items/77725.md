## Summary

Keep this PR open: it fixes a real remaining prompt-context bug in the CLI and compaction override paths, but the current branch still has a blocking prompt-mode correctness issue before it is safe to merge.

## What This Changes

The PR appends runtime `extraSystemPrompt` to configured `systemPromptOverride` prompts across CLI, embedded attempt, compaction, docs, config comments, changelog, and regression tests.

## Best Possible Solution

Keep the override-plus-runtime-context behavior, but make the shared append helper preserve `promptMode: "none"` and cover that contract with focused regression tests before merge.

## Reproduction Assessment

Yes. Source inspection on current `main` shows CLI and compaction override branches still skip `extraSystemPrompt`, and the contributor supplied after-fix CLI prompt-preparation output for the branch.

## Solution Assessment

No as written. Appending runtime context to override prompts is the right narrow fix, but the helper must keep `promptMode: "none"` from appending runtime context before the patch is correct.
