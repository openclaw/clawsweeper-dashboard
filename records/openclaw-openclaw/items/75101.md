## Summary

Keep open. Current main does not yet have a deny-path exec policy, and the latest PR head still leaves concrete bypasses in common shell-wrapper and bare-filename cases, so the patch should not merge without another author update and maintainer/security review.

## What This Changes

Adds an opt-in `tools.exec.denyPathPatterns` config surface with schema, docs, changelog, an infra matcher, and pre-exec enforcement before allowlist/approval/safeBins checks.

## Best Possible Solution

Land a maintainer-reviewed version where one shared deny-path policy evaluates parsed command segments and inner shell payloads before every execution path, treats cwd-resolvable sensitive filenames as candidates, and includes gateway, sandbox/elevated, and node integration coverage.

## Reproduction Assessment

Yes. Configure `tools.exec.denyPathPatterns: ["~/.openclaw/secrets/**"]` and use the gateway/sandbox/elevated path with `bash -c "cat ~/.openclaw/secrets/foo.env"`; the latest pre-dispatch gate tokenizes the raw wrapper text and misses the tilde pattern. A second static reproduction is `tools.exec.denyPathPatterns: ["**/.env"]` with `cat .env`, which is skipped before cwd resolution.

## Solution Assessment

No. The deny-path gate is the right direction for #74379, but this implementation needs parsed shell-wrapper inputs on local/elevated paths and broader cwd-relative candidate handling before it is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Parse shell wrappers before deny matching:** `src/agents/bash-tools.exec.ts:1622`
  - body: The gateway/sandbox/elevated gate passes the entire raw command as `shellPayload`, so `bash -c "cat ~/.openclaw/secrets/foo.env"` is tokenized as one quoted inner string prefixed by `cat `. That does not match a documented `~/.openclaw/secrets/**` deny pattern, leaving the incident path reachable outside the node-host parsed policy phase.
  - confidence: 0.9
- **[P2] Treat bare filenames as cwd-relative candidates:** `src/infra/exec-deny-path.ts:86-95`
  - body: `isPathLikeArg` rejects bare relative filenames such as `.env` and `credentials.json` because they contain no slash. Those arguments never get resolved against `cwd`, so documented patterns like `**/.env` and `**/credentials.json` do not block `cat .env` or `cat credentials.json`.
  - confidence: 0.88
