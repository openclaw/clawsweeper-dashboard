## Summary

Keep open. Current main does not implement this exec path denylist, and the PR is a useful implementation candidate for the linked security feature, but the latest head still leaves concrete bypasses in the security boundary it introduces: local pre-dispatch checking scans raw shell text instead of parsed wrapper payloads, and bare cwd-relative sensitive filenames are skipped before resolution.

## What This Changes

The PR adds an opt-in `tools.exec.denyPathPatterns` hard-deny exec gate with schema, help, docs, changelog, tests, global-plus-agent union semantics, and pre-allowlist enforcement in local/gateway/sandbox and node-host exec paths.

## Best Possible Solution

Revise the PR so one shared exec-policy gate evaluates parsed argv plus extracted shell-wrapper payload before every gateway, sandbox, elevated, and node execution path, and make the matcher consistently cover cwd-relative bare sensitive filenames such as `.env` and `credentials.json` with focused regression tests.

## Reproduction Assessment

Yes. Configure `tools.exec.denyPathPatterns: ["~/.openclaw/secrets/**"]` and inspect `bash -c 'cat ~/.openclaw/secrets/foo.env'`: the local gate tokenizes the raw wrapper command and misses the tilde path. A second static reproduction is `tools.exec.denyPathPatterns: ["**/.env"]` with `cat .env`, which is skipped before cwd resolution.

## Solution Assessment

No. The requested control is the right direction for the linked security issue, but this implementation is not the best fix until it reuses the parsed shell-wrapper seam and resolves or recognizes bare cwd-relative sensitive filenames before filtering.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Parse shell wrappers before denying paths:** `src/agents/bash-tools.exec.ts:1621-1622`
  - body: The local pre-dispatch gate scans `params.command` as raw shell text with no argv. For `bash -c 'cat ~/.openclaw/secrets/foo.env'`, tokenization keeps the inner command as one token prefixed by `cat `, so a `~/.openclaw/secrets/**` deny pattern is missed before gateway, sandbox, or elevated-full dispatch. Feed parsed argv plus the extracted shell-wrapper inner payload into the matcher before host dispatch.
  - confidence: 0.91
- **[P2] Resolve bare filenames before filtering:** `src/infra/exec-deny-path.ts:211-212`
  - body: The matcher rejects bare args such as `.env` and `credentials.json` before resolving them against `cwd`. That means documented patterns like `**/.env` and `**/credentials.json` miss common commands such as `cat .env`; resolve candidates first or treat configured sensitive basenames as path candidates.
  - confidence: 0.9
