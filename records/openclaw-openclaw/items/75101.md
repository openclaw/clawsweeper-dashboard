## Summary

Keep this PR open, but do not merge the latest head as-is. The requested hard-deny layer is still missing from current main and directly addresses the linked secret-exfiltration incident, but the current patch leaves two concrete bypasses in the security control it adds.

## What This Changes

The PR adds an opt-in `tools.exec.denyPathPatterns` exec config with schema/help/docs/changelog coverage, global-plus-agent union semantics, matcher tests, and enforcement before approval/allowlist handling.

## Best Possible Solution

Land this config-backed denylist only after one shared exec-policy path evaluates parsed argv plus extracted shell-wrapper payload before every gateway, sandbox, elevated, and node execution path, and after cwd-relative bare sensitive filenames are covered consistently with the documented semantics.

## Reproduction Assessment

Yes. Configure `tools.exec.denyPathPatterns: ["~/.openclaw/secrets/**"]` and review the local gate for `bash -c 'cat ~/.openclaw/secrets/foo.env'`: raw tokenization produces an inner token prefixed by `cat `, so the tilde path pattern is missed. A second static reproduction is `tools.exec.denyPathPatterns: ["**/.env"]` with `cat .env`, which is skipped before cwd resolution.

## Solution Assessment

No. The requested control is the right direction for #74379, but this implementation is not the best fix until it reuses the existing shell-wrapper extraction seam and resolves or otherwise recognizes bare cwd-relative sensitive filenames before filtering.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Parse shell wrappers before denying paths:** `src/agents/bash-tools.exec.ts:1621-1622`
  - body: The local pre-dispatch gate scans `params.command` as raw shell text with no argv. For `bash -c 'cat ~/.openclaw/secrets/foo.env'`, tokenization keeps the inner command as one token prefixed by `cat `, so `~/.openclaw/secrets/**` will not match while gateway, sandbox, or elevated-full execution can continue. Feed parsed argv plus the extracted shell-wrapper inner payload into the matcher before host dispatch.
  - confidence: 0.91
- **[P2] Resolve bare filenames before filtering:** `src/infra/exec-deny-path.ts:211-212`
  - body: The matcher rejects bare args such as `.env` and `credentials.json` before resolving them against `cwd`. That means documented patterns like `**/.env` and `**/credentials.json` miss common commands such as `cat .env`; resolve candidates first or treat configured sensitive basenames as path candidates.
  - confidence: 0.9
