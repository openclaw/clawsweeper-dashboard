## Summary

Keep open. Current main does not implement the requested exec path denylist, and the latest PR head is a useful security-hardening direction, but the proposed patch still has concrete bypasses in the local exec gate and path matcher that should block merge.

## What This Changes

The PR adds opt-in `tools.exec.denyPathPatterns` config with schema/help/docs/changelog updates, matcher tests, agent exec and node-host `system.run` enforcement, plus a small config-schema preprocessing change for partial Codex web search location.

## Best Possible Solution

Land an exec path denylist only after one shared policy path evaluates parsed argv plus extracted shell-wrapper payloads before every gateway, sandbox, elevated, and node execution path, and after cwd-relative bare sensitive filenames are covered consistently with docs and tests.

## Reproduction Assessment

Yes. Configure `tools.exec.denyPathPatterns: ["~/.openclaw/secrets/**"]` and run a local gateway, sandbox, or elevated-full `bash -c 'cat ~/.openclaw/secrets/foo.env'`; the PR scans the raw wrapper command there. A second static reproduction is `tools.exec.denyPathPatterns: ["**/.env"]` plus `cat .env`, which is skipped before cwd resolution.

## Solution Assessment

No. The product direction is appropriate for #74379, but the current patch is not the best fix until it reuses existing shell-wrapper extraction for the local gate and resolves or otherwise recognizes bare cwd-relative sensitive filenames before filtering.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Extract shell wrapper payloads before denying:** `src/agents/bash-tools.exec.ts:1622`
  - body: The local pre-dispatch gate passes the whole raw command as `shellPayload` with no argv. For `bash -c 'cat ~/.openclaw/secrets/foo.env'`, tokenization keeps the inner command as one quoted token prefixed by `cat `, so a `~/.openclaw/secrets/**` pattern does not match while gateway, sandbox, or elevated-full execution can continue. Feed the matcher parsed argv plus the extracted shell-wrapper inner payload before host dispatch.
  - confidence: 0.91
- **[P2] Resolve bare sensitive filenames before filtering:** `src/infra/exec-deny-path.ts:211-212`
  - body: The matcher rejects bare args such as `.env` and `credentials.json` before resolving them against `cwd`. That means documented patterns like `**/.env` and `**/credentials.json` miss common commands such as `cat .env`; resolve candidates first or treat configured sensitive basenames as path candidates.
  - confidence: 0.9
