## Summary

Keep open. The requested security control is still absent from current main and this PR is a plausible implementation path for the related production-secret exposure report, but the latest patch still leaves concrete bypasses in the new exec deny gate, so it should not merge as-is.

## What This Changes

The PR adds opt-in `tools.exec.denyPathPatterns` config, schema, docs, changelog, tests, a new exec deny-path helper, and enforcement in the agent exec tool plus node-host `system.run` policy evaluation.

## Best Possible Solution

Land a maintainer-reviewed version that evaluates one shared deny-path policy against parsed argv plus the existing shell-wrapper inner-payload extraction before every gateway, sandbox, elevated, and node execution path, and that resolves cwd-relative sensitive filenames before filtering candidates. Keep config/schema/docs/changelog aligned and add focused tests for wrapper payloads, global-plus-agent union, allowlist/full bypass resistance, and bare `.env` or `credentials.json` cases.

## Reproduction Assessment

Yes. Configure `tools.exec.denyPathPatterns: ["~/.openclaw/secrets/**"]` and send `bash -c "cat ~/.openclaw/secrets/foo.env"` through the agent exec gateway path; the PR scans the raw wrapper string at `src/agents/bash-tools.exec.ts:1622`, so the tilde pattern is missed. A second static reproduction is `tools.exec.denyPathPatterns: ["**/.env"]` with `cat .env`, which is skipped before cwd resolution in `src/infra/exec-deny-path.ts`.

## Solution Assessment

No. The feature direction is appropriate for the related issue, but the current patch is not the narrow maintainable fix because it duplicates wrapper handling with a lossy raw-command tokenizer and filters out bare filenames before the documented resolved-path match can run.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Parse shell wrappers before deny matching:** `src/agents/bash-tools.exec.ts:1622`
  - body: The pre-dispatch gate passes the complete command string as `shellPayload` with no parsed argv. For `bash -c "cat ~/.openclaw/secrets/foo.env"`, the quoted inner command is tokenized with a leading `cat ` prefix, so a `~/.openclaw/secrets/**` pattern does not match and gateway/sandbox/elevated execution can continue.
  - confidence: 0.9
- **[P2] Resolve bare filenames before skipping:** `src/infra/exec-deny-path.ts:211-212`
  - body: `isPathLikeArg` returns false for bare filenames, so the loop continues before resolving against `cwd`. Documented patterns like `**/.env` and `**/credentials.json` therefore do not block common commands such as `cat .env` or `cat credentials.json`.
  - confidence: 0.88
