## Summary

Keep open: the PR now includes real behavior proof and is directionally useful, but latest head still leaves two concrete bypasses in the new security gate that should block merge.

## What This Changes

Adds `tools.exec.denyPathPatterns` config/schema/docs/changelog support plus a shared matcher and exec/node-host enforcement before allowlist, approval, and `safeBins` checks.

## Best Possible Solution

Reuse the existing wrapper-resolution path before deny payload extraction, split unquoted shell operator boundaries in the tokenizer, add focused regression coverage, then send the security-sensitive config surface through maintainer review.

## Reproduction Assessment

Yes. Source inspection of PR head shows the gateway-side gate extracts shell payloads only from the immediate executable, and the tokenizer splits only on whitespace; those paths reproduce the wrapper-hidden and operator-adjacent bypasses without needing a live secret file.

## Solution Assessment

No. The hard-deny config direction matches the linked security request, but the current implementation is not the best mergeable form until it reuses the established wrapper seam and handles ordinary shell operator boundaries.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Unwrap dispatch wrappers before extracting shell payloads:** `src/agents/bash-tools.exec.ts:1423-1427`
  - body: The new pre-dispatch gate only calls `extractShellWrappedCommandPayload` for `denyArgv[commandIdx]`, so `timeout 5s bash -lc 'cat ~/.openclaw/secrets/foo.env'` hides the inner shell payload as a plain argv token and can pass the gateway/sandbox/elevated-full gate. Reuse the existing dispatch-wrapper resolution before extracting the shell payload.
  - confidence: 0.9
- **[P2] Split shell operators around path tokens:** `src/infra/exec-deny-path.ts:142`
  - body: The tokenizer only ends tokens on whitespace, so exact denies documented by this PR, such as `~/.aws/credentials`, miss common shell forms like `cat ~/.aws/credentials;echo ok` or `cat ~/.aws/credentials>/tmp/leak`. Treat unquoted shell metacharacters as token boundaries or strip operator suffixes before matching.
  - confidence: 0.87
