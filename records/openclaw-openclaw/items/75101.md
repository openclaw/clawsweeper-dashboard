## Summary

Keep this PR open: current main still lacks the requested exec path-deny config surface, and the PR is a useful security direction, but the latest head still leaves concrete bypasses in the deny boundary it adds.

## What This Changes

The PR adds an opt-in `tools.exec.denyPathPatterns` hard-deny exec gate with config schema/help/docs/changelog updates, global-plus-agent union semantics, and matcher tests.

## Best Possible Solution

Revise the PR so one shared deny-path gate consumes parsed argv plus extracted shell-wrapper payload before every exec host path, and make cwd-resolved bare sensitive filenames match with focused regression tests.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: at PR head, `tools.exec.denyPathPatterns: ["~/.openclaw/secrets/**"]` misses `bash -c 'cat ~/.openclaw/secrets/foo.env'` in the local gate, and `["**/.env"]` misses `cat .env` because `.env` is skipped before cwd resolution.

## Solution Assessment

No. The config-driven hard-deny direction is appropriate for the linked security issue, but this patch is not merge-ready until the matcher uses the existing parsed wrapper seam consistently and resolves bare cwd-relative candidates.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Parse shell wrappers before denying paths:** `src/agents/bash-tools.exec.ts:1621-1623`
  - body: The local pre-dispatch gate passes `argv: []` and raw `params.command` into the matcher. For `bash -c 'cat ~/.openclaw/secrets/foo.env'`, tokenization keeps the inner command as one token prefixed by `cat `, so a `~/.openclaw/secrets/**` deny pattern is missed before gateway, sandbox, or elevated-full dispatch. Feed parsed argv plus the extracted shell-wrapper payload into this gate.
  - confidence: 0.91
- **[P2] Resolve bare filenames before filtering:** `src/infra/exec-deny-path.ts:210-212`
  - body: The matcher rejects bare args such as `.env` and `credentials.json` before resolving them against `cwd`. That means documented patterns like `**/.env` and `**/credentials.json` miss common commands such as `cat .env`; resolve candidates first or treat configured sensitive basenames as path candidates.
  - confidence: 0.9
