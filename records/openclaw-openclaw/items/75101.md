## Summary

Keep open. Current main does not have the requested exec path-deny surface, and the linked production incident in #74379 is a valid security hardening target, but the latest PR head still has blocking bypasses for documented shell-wrapper and bare-filename cases.

## What This Changes

The PR adds opt-in `tools.exec.denyPathPatterns` config, a new exec-deny matcher, local and node exec policy wiring, schema/help/docs/changelog coverage, and focused tests.

## Best Possible Solution

Land an exec path denylist only after one shared policy evaluates parsed argv plus extracted shell-wrapper payloads before every gateway, sandbox, elevated, and node execution path, and after cwd-relative bare sensitive filenames are covered consistently with the docs and tests.

## Reproduction Assessment

Yes. Configure `tools.exec.denyPathPatterns: ["~/.openclaw/secrets/**"]` and run `bash -c 'cat ~/.openclaw/secrets/foo.env'` through a local gateway/sandbox/elevated-full path; this PR scans the raw wrapper command there. A second static reproduction is `tools.exec.denyPathPatterns: ["**/.env"]` plus `cat .env`, which is skipped before cwd resolution.

## Solution Assessment

No. The product direction is appropriate for #74379, but this implementation is not the narrowest maintainable fix until it reuses the existing shell-wrapper extraction path for local exec and resolves bare cwd-relative sensitive filenames before filtering.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Extract shell wrapper payloads before denying:** `src/agents/bash-tools.exec.ts:1622`
  - body: The local pre-dispatch gate passes the full raw command as `shellPayload` with no argv. For `bash -c 'cat ~/.openclaw/secrets/foo.env'`, tokenization keeps the inner command as one quoted token prefixed by `cat `, so a `~/.openclaw/secrets/**` pattern does not match while gateway, sandbox, or elevated-full execution can continue. Feed the matcher parsed argv plus the extracted shell-wrapper inner payload before host dispatch.
  - confidence: 0.91
- **[P2] Resolve bare sensitive filenames before filtering:** `src/infra/exec-deny-path.ts:211-212`
  - body: The matcher rejects bare args such as `.env` and `credentials.json` before resolving them against `cwd`. That means documented patterns like `**/.env` and `**/credentials.json` miss common commands such as `cat .env`; resolve candidates first or otherwise treat configured sensitive basenames as path candidates.
  - confidence: 0.9
