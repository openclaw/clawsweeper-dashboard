## Summary

Keep open. Current main still lacks an exec path denylist, and this PR is aimed at a real security gap from #74379, but the latest head still has concrete bypasses in the gateway/sandbox local exec path and in bare-filename matching.

## What This Changes

The PR adds opt-in `tools.exec.denyPathPatterns` config with schema/help/docs/changelog updates, matcher tests, agent exec wiring, and node-host `system.run` enforcement.

## Best Possible Solution

Land the denylist only after one shared policy path evaluates parsed argv plus extracted shell-wrapper payload before every gateway, sandbox, elevated, and node execution path, and after cwd-relative bare sensitive filenames are covered consistently with the documented semantics.

## Reproduction Assessment

Yes. Configure `tools.exec.denyPathPatterns: ["~/.openclaw/secrets/**"]` and run local exec with `bash -c 'cat ~/.openclaw/secrets/foo.env'`; the PR’s local gate tokenizes raw wrapper text. A second static reproduction is `tools.exec.denyPathPatterns: ["**/.env"]` plus `cat .env`, which is skipped before cwd resolution.

## Solution Assessment

No. The requested control is the right direction for the linked incident, but this implementation is not the best fix until it reuses the existing shell-wrapper extraction seam and resolves or otherwise recognizes bare cwd-relative sensitive filenames before filtering.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Parse shell wrappers before denying paths:** `src/agents/bash-tools.exec.ts:1621-1622`
  - body: The local pre-dispatch gate passes the whole raw command as `shellPayload` with no argv. For `bash -c 'cat ~/.openclaw/secrets/foo.env'`, tokenization keeps the inner command as one quoted token prefixed by `cat `, so a `~/.openclaw/secrets/**` pattern will not match while gateway, sandbox, or elevated-full execution can continue. Feed parsed argv plus the extracted shell-wrapper inner payload into the matcher before host dispatch.
  - confidence: 0.91
- **[P2] Resolve bare filenames before filtering:** `src/infra/exec-deny-path.ts:211-212`
  - body: The matcher rejects bare args such as `.env` and `credentials.json` before resolving them against `cwd`. That means documented patterns like `**/.env` and `**/credentials.json` miss common commands such as `cat .env`; resolve candidates first or treat configured sensitive basenames as path candidates.
  - confidence: 0.9
