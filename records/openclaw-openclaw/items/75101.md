## Summary

Keep open. Current main still lacks the requested deny-path exec control, and this PR is a useful implementation candidate, but the latest diff still leaves concrete bypasses in the security boundary it adds.

## What This Changes

The PR adds an opt-in `tools.exec.denyPathPatterns` hard-deny exec policy with schema/help/docs/changelog updates, config merging, and tests for path and shell-payload matching.

## Best Possible Solution

Revise the PR so one shared deny-path gate evaluates parsed argv plus extracted shell-wrapper payload before every gateway, sandbox, elevated, and node execution path, and make the matcher cover cwd-relative bare sensitive filenames with focused regression tests.

## Reproduction Assessment

Yes. Static reproduction against PR head is high-confidence: with `tools.exec.denyPathPatterns: ["~/.openclaw/secrets/**"]`, the local gateway/sandbox/elevated-full gate tokenizes raw `bash -c 'cat ~/.openclaw/secrets/foo.env'` as wrapper text and misses the inner tilde path; with `["**/.env"]`, `cat .env` is skipped before cwd resolution.

## Solution Assessment

No. The requested hard-deny control is the right direction for the linked security issue, but this implementation is not the best fix until it reuses the parsed shell-wrapper seam consistently and resolves bare cwd-relative candidates before filtering.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Parse shell wrappers before denying paths:** `src/agents/bash-tools.exec.ts:1621-1622`
  - body: The local pre-dispatch gate scans `params.command` as raw shell text with no argv. For `bash -c 'cat ~/.openclaw/secrets/foo.env'`, tokenization keeps the inner command as one token prefixed by `cat `, so a `~/.openclaw/secrets/**` deny pattern is missed before gateway, sandbox, or elevated-full dispatch. Feed parsed argv plus the extracted shell-wrapper inner payload into the matcher before host dispatch.
  - confidence: 0.91
- **[P2] Resolve bare filenames before filtering:** `src/infra/exec-deny-path.ts:217-218`
  - body: The matcher rejects bare args such as `.env` and `credentials.json` before resolving them against `cwd`. That means documented patterns like `**/.env` and `**/credentials.json` miss common commands such as `cat .env`; resolve candidates first or treat configured sensitive basenames as path candidates.
  - confidence: 0.9
