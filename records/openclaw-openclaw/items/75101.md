## What This Changes

Adds `tools.exec.denyPathPatterns` config, schema/help/docs/changelog coverage, a shared path matcher, and exec/node-host gates before allowlist, approval, and safe-bin checks.

## Best Possible Solution

Revise the PR to reuse the shared wrapper-resolution path, split or strip shell metacharacters around path operands, add targeted regression coverage for those cases, then provide real denial output for maintainer/security review.

## Reproduction Assessment

Yes. Source inspection of PR head shows the local gate only extracts payloads from the immediate executable and the payload tokenizer splits only on whitespace; current main absence of this config was also verified.

## Solution Assessment

No. The hard-deny direction matches the linked security request, but this implementation is not the best mergeable form until it uses the existing wrapper seam and handles common shell operator adjacency.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Unwrap dispatch wrappers before extracting shell payloads:** `src/agents/bash-tools.exec.ts:1423-1427`
  - body: The local gate only treats `denyArgv[commandIdx]` as the shell executable, so transparent wrappers like `timeout 5s bash -lc 'cat ~/.openclaw/secrets/foo.env'` leave the inner command as one argv token. With a `~/.openclaw/secrets/**` deny pattern that misses before gateway/sandbox/elevated-full dispatch; reuse the shared wrapper-resolution seam here.
  - confidence: 0.9
- **[P2] Split shell operators from payload path tokens:** `src/infra/exec-deny-path.ts:142`
  - body: `tokenizeShellPayload()` only splits on whitespace, so exact deny patterns documented by this PR can be bypassed by common no-space syntax like `cat ~/.aws/credentials; echo ok` or `cat ~/.aws/credentials>/tmp/leak`. Treat shell metacharacters as token boundaries outside quotes, or strip operator suffixes before matching.
  - confidence: 0.86
- **[P3] Add real behavior proof before merge:** `docs/tools/exec-approvals-advanced.md:11`
  - body: The PR body lists unit/type/schema checks and an unchecked manual test plan, but it does not show after-fix runtime output from OpenClaw denying a configured path. Please add terminal output, copied console output, a screenshot, a linked artifact, or redacted logs showing the new denial in a real setup.
  - confidence: 0.95
