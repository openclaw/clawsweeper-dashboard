## Summary

Keep open. Current main still lacks the requested deny-path exec control, and this PR is the active implementation for the linked incident, but the latest head remains non-landable because the new security gate still misses wrapped shell payloads on local/elevated paths and skips bare sensitive filenames promised by the docs.

## What This Changes

The PR adds an opt-in `tools.exec.denyPathPatterns` hard-deny config with schema, docs, changelog, helper tests, and enforcement in the agent exec tool and node `system.run` policy path.

## Best Possible Solution

Land a maintainer-reviewed version where one shared deny-path policy evaluates parsed command segments and inner shell-wrapper payloads before every gateway, sandbox, elevated, and node execution path, treats cwd-resolvable sensitive filenames as candidates, and keeps schema, docs, changelog, and focused integration coverage aligned with that contract.

## Reproduction Assessment

Yes. Configure `tools.exec.denyPathPatterns: ["~/.openclaw/secrets/**"]` and run a gateway/sandbox/elevated command like `bash -c "cat ~/.openclaw/secrets/foo.env"`; the PR head tokenizes the raw wrapper string instead of the parsed inner payload. A second high-confidence static reproduction is `tools.exec.denyPathPatterns: ["**/.env"]` with `cat .env`, which is skipped before cwd resolution.

## Solution Assessment

No. The product direction is appropriate for #74379, but this implementation is not yet the narrow maintainable fix because the local gate does not reuse existing shell-wrapper parsing and the matcher excludes bare cwd-relative sensitive filenames that the docs advertise.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Parse shell wrappers before deny matching:** `src/agents/bash-tools.exec.ts:1621-1622`
  - body: The gateway/sandbox/elevated gate passes the full command string as `shellPayload`. For `bash -c "cat ~/.openclaw/secrets/foo.env"`, the tokenizer keeps the inner command as one token prefixed by `cat `, so `~/.openclaw/secrets/**` does not match and the run can continue on local/elevated execution paths.
  - confidence: 0.9
- **[P2] Resolve bare sensitive filenames before skipping:** `src/infra/exec-deny-path.ts:86-95`
  - body: The matcher rejects bare args such as `.env` and `credentials.json` because they have no slash, tilde, or absolute path shape. Those args never get resolved against `cwd`, so documented patterns like `**/.env` and `**/credentials.json` miss `cat .env` and `cat credentials.json`.
  - confidence: 0.88
