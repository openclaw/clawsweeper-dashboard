## Summary

Keep open: current main does not already implement the requested exec path denylist, and the linked production-secret leak report is valid core security context, but this PR still has blocking bypasses in the proposed deny gate before it should merge.

## What This Changes

The PR adds opt-in `tools.exec.denyPathPatterns` config, schema/help/docs/changelog coverage, tests, and pre-exec deny checks, while also touching WhatsApp package metadata and an unrelated config schema helper.

## Best Possible Solution

Land a maintainer-reviewed exec path denylist only after one shared policy evaluates parsed argv plus extracted shell-wrapper payloads before every gateway, sandbox, elevated, and node execution path, with cwd-relative bare sensitive filenames covered and package metadata cleanup split or rebased away.

## Reproduction Assessment

Yes. A high-confidence static reproduction is `tools.exec.denyPathPatterns: ["~/.openclaw/secrets/**"]` plus gateway/sandbox/elevated `bash -c 'cat ~/.openclaw/secrets/foo.env'`, where this PR scans raw wrapper text. A second reproduction is `tools.exec.denyPathPatterns: ["**/.env"]` plus `cat .env`, which is skipped before cwd resolution.

## Solution Assessment

No. The core direction is appropriate for #74379, but this implementation is not the best fix until it reuses existing shell-wrapper extraction for local exec paths, resolves bare cwd-relative filenames before filtering, and separates unrelated package metadata churn.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Parse shell wrappers before deny matching:** `src/agents/bash-tools.exec.ts:1619-1623`
  - body: The pre-dispatch gate passes the full raw command as `shellPayload` with no argv. For `bash -c 'cat ~/.openclaw/secrets/foo.env'`, tokenization keeps the inner command as one token prefixed by `cat `, so a `~/.openclaw/secrets/**` pattern does not match and gateway/sandbox/elevated execution can continue. Feed the matcher parsed argv plus extracted shell-wrapper inner payloads before dispatch.
  - confidence: 0.91
- **[P2] Resolve bare filenames before filtering:** `src/infra/exec-deny-path.ts:210-212`
  - body: The matcher rejects bare args such as `.env` and `credentials.json` before resolving them against `cwd`. That means documented patterns like `**/.env` and `**/credentials.json` miss common commands such as `cat .env`; resolve candidates first or treat configured sensitive basenames as path candidates.
  - confidence: 0.89
