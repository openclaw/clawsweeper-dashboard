## Summary

Keep open. Current main does not have the requested deny-path exec policy, and the PR is a plausible security-hardening direction, but the latest patch still leaves concrete bypasses in the new enforcement path: wrapped shell commands can miss the deny pattern before gateway/sandbox/elevated dispatch, and bare sensitive filenames are skipped before cwd resolution.

## What This Changes

The PR adds opt-in `tools.exec.denyPathPatterns` config/schema/docs/changelog support, a shared exec deny-path matcher, tests, and enforcement before exec allowlist/approval checks.

## Best Possible Solution

Land a maintainer-reviewed version that evaluates one shared deny-path policy against parsed argv plus existing shell-wrapper inner payload extraction before every gateway, sandbox, elevated, and node execution path. Resolve cwd-relative sensitive filenames before filtering candidates, remove stale duplicate helper files, and keep config/schema/docs/changelog/tests aligned with direct argv, wrapped shell payload, global-plus-agent union, allowlist/full bypass resistance, and bare `.env`/`credentials.json` coverage.

## Reproduction Assessment

Yes. Configure `tools.exec.denyPathPatterns: ["~/.openclaw/secrets/**"]` and send `bash -c 'cat ~/.openclaw/secrets/foo.env'` through the gateway/sandbox/elevated path; the PR scans raw wrapper text at `src/agents/bash-tools.exec.ts:1622`, so the tilde pattern is missed. A second static reproduction is `tools.exec.denyPathPatterns: ["**/.env"]` with `cat .env`, which is skipped before cwd resolution in `src/infra/exec-deny-path.ts`.

## Solution Assessment

No. The feature direction is appropriate for #74379, but the current patch is not the best implementation because it duplicates shell-wrapper parsing with a lossy tokenizer and filters out bare filenames before the documented resolved-path match can run.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Parse shell wrappers before deny matching:** `src/agents/bash-tools.exec.ts:1622`
  - body: The pre-dispatch gate passes the full raw command as `shellPayload`. For `bash -c 'cat ~/.openclaw/secrets/foo.env'`, the tokenizer keeps the inner command as one token prefixed by `cat `, so a tilde-based deny pattern does not match and gateway/sandbox/elevated execution can continue. Reuse the existing shell-wrapper extraction before matching here.
  - confidence: 0.9
- **[P2] Resolve bare sensitive filenames before skipping:** `src/infra/exec-deny-path.ts:211-212`
  - body: The matcher rejects bare args such as `.env` and `credentials.json` before resolving them against `cwd`. That means documented patterns like `**/.env` and `**/credentials.json` miss common commands such as `cat .env`; resolve candidates first or treat configured sensitive basenames as path candidates.
  - confidence: 0.88
