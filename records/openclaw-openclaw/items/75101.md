## Summary

Keep open. The PR targets a real, still-unimplemented exec security gap from the linked open report, but the latest head still leaves concrete bypasses for wrapped shell commands and bare sensitive filenames, and it mixes unrelated WhatsApp package metadata churn into the security change.

## What This Changes

The branch adds opt-in `tools.exec.denyPathPatterns` config/schema/docs/tests and enforcement in exec policy paths, while also removing the WhatsApp plugin's `qrcode` dependency.

## Best Possible Solution

Land a maintainer-reviewed version of the exec denylist that evaluates one shared policy against parsed argv plus extracted shell-wrapper inner payloads before every gateway, sandbox, elevated, and node execution path. It should resolve cwd-relative bare sensitive filenames before filtering, split unrelated package changes, and keep config, schema, docs, changelog, and targeted tests aligned with the documented v1 scope.

## Reproduction Assessment

Yes. A high-confidence static reproduction is `tools.exec.denyPathPatterns: ["~/.openclaw/secrets/**"]` plus `bash -c 'cat ~/.openclaw/secrets/foo.env'` through `createExecTool`; the PR scans the raw wrapper command rather than the extracted inner payload. A second reproduction is `tools.exec.denyPathPatterns: ["**/.env"]` plus `cat .env`, which is skipped before cwd resolution.

## Solution Assessment

No. The requested core direction is appropriate for the linked report, but this implementation is not the best maintainable fix until it reuses existing shell-wrapper extraction, resolves bare filenames before path-like filtering, and separates the WhatsApp package change from the security PR.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Parse shell wrappers before deny matching:** `src/agents/bash-tools.exec.ts:1621-1622`
  - body: The pre-dispatch gate passes the full raw command as `shellPayload` with no argv. For `bash -c 'cat ~/.openclaw/secrets/foo.env'`, tokenization keeps the inner command as one token prefixed by `cat `, so a `~/.openclaw/secrets/**` pattern does not match and gateway/sandbox/elevated execution can continue; feed the matcher parsed argv plus extracted shell-wrapper inner payloads before dispatch.
  - confidence: 0.9
- **[P2] Resolve bare filenames before filtering:** `src/infra/exec-deny-path.ts:210-212`
  - body: The matcher rejects bare args such as `.env` and `credentials.json` before resolving them against `cwd`. That means documented patterns like `**/.env` and `**/credentials.json` miss common commands such as `cat .env`; resolve candidates first or treat configured sensitive basenames as path candidates.
  - confidence: 0.88
