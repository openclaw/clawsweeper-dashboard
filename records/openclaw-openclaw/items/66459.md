## Summary

Keep open: current main still blanket-suppresses same-target message-tool follow-up replies after content/media dedupe, matching the later file/media-send repros, while open PR #59752 already targets that shared root cause.

## What This Changes

Telegram users report that a distinct final assistant text reply can be silently dropped after a same-target message-tool file/media send, even though the transcript contains the reply.

## Best Possible Solution

Land or supersede #59752 with content-level same-target dedupe that keeps exact duplicate suppression but allows distinct final text after file/media sends.

## Reproduction Assessment

Yes. Current main has a unit-level reproduction: same-target `messagingToolSentTargets` plus distinct final text still returns no reply payload, and existing tests assert that behavior.

## Solution Assessment

Yes. The narrow maintainable fix is in the shared auto-reply/follow-up dedupe path, not a Telegram streaming workaround.
