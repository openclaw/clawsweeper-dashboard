## Summary

Keep open, but not merge-ready: the branch is focused and has real proxy proof, yet it reverses a shipped/native-host Kimi fallback boundary and needs maintainer approval before custom Moonshot chat proxies are inherited implicitly.

## What This Changes

The PR makes Kimi web_search inherit any non-empty `models.providers.moonshot.baseUrl` when no explicit Kimi search baseUrl is set, and updates provider tests for proxy inheritance and 401 behavior.

## Best Possible Solution

Keep native `.ai`/`.cn` inheritance as the default, and require arbitrary custom Kimi search hosts through the explicit webSearch baseUrl unless maintainers approve broader inheritance with upgrade docs and proof.
