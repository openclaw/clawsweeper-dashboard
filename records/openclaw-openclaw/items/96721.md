## Summary

Keep open. The patch still broadens a shipped Moonshot/Kimi fallback from native hosts to arbitrary custom chat hosts, which conflicts with current docs and the prior merged boundary unless maintainers explicitly accept the credential-routing contract.

## What This Changes

The PR makes Kimi web search inherit any non-empty `models.providers.moonshot.baseUrl` when Kimi has no explicit search `baseUrl`, and updates Moonshot tests for custom proxy inheritance and 401 URL selection.

## Best Possible Solution

Keep native `.ai`/`.cn` inheritance as the default and require arbitrary custom Kimi search hosts through explicit web-search `baseUrl` unless maintainers approve, document, and test broad inheritance.
