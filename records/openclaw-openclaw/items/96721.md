## Summary

Keep open for maintainer review. The branch is focused and has real behavior proof, but it broadens Kimi provider fallback from native Moonshot hosts to arbitrary custom chat hosts, which conflicts with current docs/prior scope and can reroute search traffic and credentials for existing setups.

## What This Changes

The branch removes Kimi's native Moonshot host guard so web_search inherits any configured `models.providers.moonshot.baseUrl`, and updates Moonshot Kimi tests for custom proxy inheritance and 401 errors.

## Best Possible Solution

Keep native `.ai`/`.cn` inheritance as the default, and require arbitrary custom Kimi search hosts through the explicit webSearch baseUrl unless maintainers approve and document/test broad inheritance.

Yes for the PR's custom-baseUrl branch: current main rejects non-native Moonshot provider base URLs and falls back to the default Kimi host by source inspection. No high-confidence reproduction exists for the linked Windows service report because the full redacted config and failing request host are still missing.

## Solution Assessment

No. The patch is a plausible custom-proxy fix, but the best mergeable path needs maintainer approval for broad custom-host inheritance or should keep arbitrary custom search hosts explicit.
