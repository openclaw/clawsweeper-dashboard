## Summary

Keep open. The PR has useful after-fix proof for custom-host inheritance, but it broadens a shipped Kimi credential-routing fallback from native Moonshot hosts to any configured Moonshot chat base URL, which needs maintainer approval before merge.

## What This Changes

The PR makes Kimi web search inherit any non-empty `models.providers.moonshot.baseUrl` when no explicit Kimi search `baseUrl` is configured, and updates Moonshot tests for custom proxy inheritance and 401 handling.

## Best Possible Solution

Keep native `.ai`/`.cn` inheritance as the default and require arbitrary custom Kimi search hosts through explicit web-search `baseUrl` unless maintainers approve, document, and test broad inheritance.

## Solution Assessment

Is this the best way to solve the issue? No. The patch is a plausible custom-proxy behavior change, but the safest mergeable fix keeps arbitrary custom search hosts explicit unless maintainers approve the broader credential-routing contract.
