## Summary

Keep open: the PR is a narrow attempt at the Kimi authentication report, but it broadens endpoint and credential routing from custom Moonshot chat base URLs to Kimi search without maintainer-approved upgrade behavior or real runtime proof.

## What This Changes

The PR removes the native-Moonshot-host guard from Kimi web-search base URL fallback and updates one regression test to expect arbitrary `models.providers.moonshot.baseUrl` inheritance.

## Best Possible Solution

Keep native `.ai`/`.cn` inheritance as the default and require custom Kimi search hosts through explicit web-search baseUrl config unless maintainers approve broader inheritance with upgrade and credential-routing proof.

## Reproduction Assessment

Yes for the changed source path: current main rejects non-native Moonshot baseUrls and falls back to the default Kimi host. No live reproduction proves the linked reporter’s exact 401, because the full redacted config and failing request host are still missing.

## Solution Assessment

No. The patch is a plausible fix for one custom-proxy scenario, but the safer maintainable path is native-host inheritance plus explicit custom Kimi search baseUrl unless maintainers approve broader endpoint inheritance.
