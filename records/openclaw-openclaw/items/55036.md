## Summary

Keep open. Current main still intentionally suppresses commentary-phase assistant text before it can reach Discord partial, block, or final delivery, and the related merged phase-separation work fixed leakage rather than this early-status durability request.

## What This Changes

The issue asks Discord to commit early assistant commentary or status text as a standalone visible message before later work or a gateway restart can drop it.

## Best Possible Solution

Define an explicit safe user-visible status/progress contract and route only that approved text through embedded subscribe and Discord delivery, while keeping raw commentary suppressed and adding restart-sensitive coverage.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: emit commentary-phase assistant text before tool use and current main suppresses onPartialReply, onBlockReply, and assistantTexts, leaving Discord with nothing durable to commit before restart.

## Solution Assessment

No, not by forwarding raw commentary. The maintainable solution is a narrower approved-status lane that preserves the shipped planning-text suppression boundary while giving Discord an immediate durable update path.
