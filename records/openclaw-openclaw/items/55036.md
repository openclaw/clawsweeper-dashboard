## Summary

Keep open. Current main still suppresses commentary-phase assistant text before Discord delivery, while the Discord progress-draft path covers approved tool/work/reasoning progress rather than durable early commentary/status text.

## What This Changes

The issue asks for Discord to commit early assistant commentary immediately as its own visible message so restart-sensitive turns cannot lose it.

## Best Possible Solution

Define a narrow approved status/progress contract that can be persisted or streamed to Discord before restart-sensitive work, while keeping raw commentary-phase assistant text suppressed.

## Reproduction Assessment

Yes for the current source behavior: emit a commentary-phase assistant message before tool use and current main returns before onPartialReply, onBlockReply, assistantTexts, or Discord draft delivery. I did not live-restart a Discord gateway, so the restart-loss part remains inferred from that undelivered window.

## Solution Assessment

No for direct raw-commentary delivery; it conflicts with the current commentary/final_answer suppression boundary. The safer solution is an explicit approved status lane that Discord can durably commit without exposing internal planning text.
