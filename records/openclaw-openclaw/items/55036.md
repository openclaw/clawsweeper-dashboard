## Summary

Keep open. Current main still does not deliver early assistant commentary to Discord as a committed visible message; it deliberately suppresses `commentary` phase output before the Discord draft preview, partial reply, block reply, and final reply paths can see it. The related closed report #55236 is about final-result delivery latency, while this issue is specifically about safe early commentary/status durability before restart-sensitive work.

## What This Changes

The issue asks for Discord to commit early assistant status/commentary immediately as visible output so a gateway restart before final delivery cannot silently lose it.

## Best Possible Solution

Define an explicit safe status-commentary surface, keep raw `commentary` suppressed, and route only approved status text through the embedded subscribe plus Discord draft/dispatcher path with regression coverage for immediate commit and restart-sensitive delivery.
