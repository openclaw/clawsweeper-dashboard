## Summary

Keep this PR open, but it is not mergeable as-is: the linked display bug is real, while the proposed helper-wide semantic change would break fresh-only memory/compaction callers and the branch also carries broad unrelated release/version churn.

## What This Changes

The branch changes `resolveFreshSessionTotalTokens` to return preserved totals even when stale, updates freshness checking and the changelog, and also includes unrelated workflow, package version, release script, generated baseline, app version, and extension metadata changes across 163 files.

## Best Possible Solution

Split or rename the helper semantics so display/session-list consumers can show finite preserved stale totals with the freshness flag intact, while compaction-sensitive callers keep an explicit fresh-only path and the PR branch is reduced to the session fix plus focused tests.

Yes. Source inspection on current main shows finite `totalTokens` plus `totalTokensFresh: false` is filtered to `undefined` by the shared helper and current tests encode that display behavior; I did not run a live UI session in this read-only review.

## Solution Assessment

No. The PR applies the semantic change too broadly to a helper still used as a fresh-only guard, and it should instead scope the fix to display/session projection callers while preserving compaction-sensitive behavior.
