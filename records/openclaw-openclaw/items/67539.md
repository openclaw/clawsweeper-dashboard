## Summary

Keep open. Current main still injects only provider-neutral TTS guidance and the speech-provider contract has no provider-owned prompt-hint hook, while Gemini docs/tests show expressive tags work only after the model already emits them.

## What This Changes

Add an optional provider-owned, model-aware TTS prompt hint so agents learn expressive syntax for the active speech provider.

## Best Possible Solution

Add a backward-compatible speech-provider prompt-hint hook, append only the active provider's short hint from speech-core when TTS is enabled, and let Google own Gemini audio-tag wording with focused tests and docs.

## Reproduction Assessment

Yes for the feature gap. Source inspection on current main shows TTS prompt construction can only emit generic guidance because no provider hook or selected-provider context is wired into buildTtsSystemPromptHint.

## Solution Assessment

Yes. A provider-owned hook is the narrowest maintainable direction because it keeps vendor/model syntax out of generic speech-core while avoiding optional skills or human-only docs as the runtime guidance layer.

## Repair Work Prompt

Implement provider-specific TTS prompt hints for https://github.com/openclaw/openclaw/issues/67539. Add a backward-compatible optional hook to the speech provider contract so providers can return a short runtime prompt hint for the active TTS provider/model/voice/target/config. Wire speech-core's buildTtsSystemPromptHint to append only the selected provider's hint when TTS is enabled; do not hard-code Gemini wording in generic speech-core except through the Google provider hook. Include modelOverrides/policy context so hints do not advertise controls the model is not allowed to emit. Add a concise Google Gemini hint for expressive square-bracket cues inside [[tts:text]] blocks, based on the behavior introduced by https://github.com/openclaw/openclaw/pull/67515. Update public SDK exports/API baseline and plugin docs as needed. Add focused tests proving generic guidance remains, only the active provider hint is injected, inactive providers are not advertised, and off-policy controls are not advertised. Add a user-facing CHANGELOG.md entry under the active version Changes section, crediting @barronlroth if attribution is included. Do not add provider-specific strings, ids, or policy branches to unrelated core/provider code.
