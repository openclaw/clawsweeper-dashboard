## Summary

Keep open. Current main still only injects generic TTS prompt guidance, and the speech-provider contract has no provider-owned prompt-hint hook, while Gemini TTS docs/tests confirm expressive tags work only after the agent already emits them.

## What This Changes

Add provider-specific, model-aware TTS prompt hints so the active speech provider can advertise valid expressive controls to agents at runtime.

## Best Possible Solution

Add a backwards-compatible speech-provider prompt-hint hook, have speech-core append only the selected provider's short hint when TTS is enabled, and let Google own Gemini audio-tag wording with focused tests and docs.

## Reproduction Assessment

Yes. Source inspection on current main is a high-confidence reproduction path: enabling TTS can only produce the generic buildTtsSystemPromptHint output because no provider hook or provider context is wired into prompt construction.

## Solution Assessment

Yes. A provider-owned hook is the narrowest maintainable direction because it keeps vendor/model syntax in the owning speech provider instead of hard-coding Gemini guidance in speech-core or relying on optional skills/docs.
