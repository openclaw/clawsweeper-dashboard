## Summary

Keep open. Current main still injects only generic TTS guidance and the speech-provider contract has no provider-owned prompt-hint hook, so the requested active-provider expressive guidance remains missing.

## What This Changes

Add a provider-owned, model-aware TTS prompt hint hook and append only the active speech provider's short guidance when TTS is enabled.

## Best Possible Solution

Add a backwards-compatible provider prompt-hint hook, have speech-core append only the selected provider's short hint, and let the Google provider own Gemini audio-tag wording with focused contract/runtime tests.

## Reproduction Assessment

Yes. Source inspection on current main is a high-confidence reproduction path: enabled TTS can only produce the generic buildTtsSystemPromptHint output because no provider hook or provider context is wired into prompt construction.

## Solution Assessment

Yes. A provider-owned hook is the narrowest maintainable direction because it keeps vendor/model syntax in the owning speech provider instead of hard-coding Gemini guidance in speech-core or relying on optional docs/skills.
