## Summary

Keep open: current main still has only generic TTS prompt guidance, the speech-provider contract has no provider-owned prompt-hint hook, and the related open auto-emotion PR addresses provider mapping rather than this active-provider runtime prompt seam.

## What This Changes

Request an additive, provider-owned, model-aware TTS prompt-hint hook so speech-core can append only the active speech provider's expressive-control guidance.

## Best Possible Solution

Add a backwards-compatible speech-provider prompt-hint hook, append only the selected provider's short hint from speech-core, let Google own Gemini tag wording, and cover active-provider-only plus policy-gated behavior in tests and docs.

## Reproduction Assessment

Yes. Source inspection on current main shows buildTtsSystemPromptHint only receives config and agent id while SpeechProviderPlugin has no prompt hook, so selecting Google TTS cannot inject Gemini tag guidance at runtime.

## Solution Assessment

Yes. A provider-owned speech hook is the narrow maintainable direction; hard-coding Gemini in speech-core, relying on skills, or leaving this as docs-only would not scale across provider/model-specific TTS syntax.
