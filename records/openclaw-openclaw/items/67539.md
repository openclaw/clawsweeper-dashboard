## Summary

Keep open: current main still has only generic TTS prompt guidance, the speech-provider contract has no provider-owned prompt-hint hook, and the live auto-emotion PR tracks a complementary runtime-mapping feature rather than this active-provider prompt seam.

## What This Changes

Request provider-specific, model-aware TTS prompt hints so agents know which expressive controls are valid for the active speech provider.

## Best Possible Solution

Add a backwards-compatible speech-provider prompt-hint hook, append only the selected provider's short hint from speech-core, and let each provider own model/voice/target-specific wording and policy gating.

## Reproduction Assessment

Yes, via source inspection rather than a failing runtime test. Current main builds the TTS system hint without provider context and the SpeechProviderPlugin contract has no prompt hook, so selecting Google TTS cannot inject Gemini tag guidance at runtime.

## Solution Assessment

Yes with owner review. A provider-owned speech hook is the narrow maintainable direction; hard-coding Gemini in speech-core or relying on docs/skills would not scale across provider and model-specific TTS syntax.
