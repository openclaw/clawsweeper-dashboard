## Summary

Keep open: current main still has only generic TTS prompt guidance, the speech-provider contract has no provider-owned prompt-hint hook, and the related autoEmotion work is explicitly complementary rather than a replacement for this prompt seam.

## What This Changes

Request to add an additive speech-provider prompt-hint hook so the active TTS provider can inject model-aware expressive-syntax guidance into agent system prompts.

## Best Possible Solution

Add a backwards-compatible speech-provider prompt-hint hook, append only the selected provider's short hint from speech-core, and let each provider own model/voice/target-specific wording with tests and docs covering active-provider-only injection.

## Reproduction Assessment

Yes for the current gap, via source inspection: buildTtsSystemPromptHint only emits generic guidance and SpeechProviderPlugin has no prompt-hint hook, so selecting Google TTS cannot add Gemini tag guidance to the agent prompt. This is a feature request rather than broken documented behavior.

## Solution Assessment

Yes directionally, with owner review. A provider-owned speech hook is the narrow maintainable approach; hard-coding Gemini in speech-core or relying on docs/skills would not scale across provider and model-specific TTS syntax.
