## Summary

Keep open. Current main still has only the generic TTS runtime prompt and the speech-provider contract has no provider-owned prompt-hint hook, while the Google Gemini TTS provider and docs confirm the motivating expressive-tag behavior is available only when the agent already knows to emit it.

## What This Changes

Issue requests a provider-owned, model-aware TTS prompt-hint hook that speech-core appends only for the active speech provider when TTS is enabled.

## Best Possible Solution

Add a backwards-compatible speech-provider prompt-hint contract, have speech-core resolve the active provider and append only that provider's short hint when TTS is enabled, and let the Google provider own Gemini audio-tag guidance with active-provider-only and policy-gated tests.

## Reproduction Assessment

Yes. A high-confidence reproduction is source inspection on current main: build the TTS system prompt with TTS enabled and it can only include the generic buildTtsSystemPromptHint text because no provider hook or provider context exists.

## Solution Assessment

Yes. The provider-owned hook is the narrowest maintainable direction because it keeps vendor/model syntax in the owning provider instead of hard-coding Gemini rules in speech-core or relying on optional skills/docs.
