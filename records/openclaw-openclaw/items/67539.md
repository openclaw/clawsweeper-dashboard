## Summary

Keep open. Current main still builds only generic TTS system-prompt guidance, the speech-provider contract has no provider-owned hint hook, and the related open auto-emotion PR covers a different provider-mapping feature rather than this active-provider prompt seam.

## What This Changes

Request to add a backwards-compatible, provider-owned TTS prompt-hint hook so the active speech provider can inject model-aware expressive-syntax guidance into the system prompt.

## Best Possible Solution

Add an additive speech-provider prompt-hint hook, append only the selected provider's short hint from speech-core, let Google own Gemini audio-tag wording, and cover active-provider-only plus policy-gated behavior in tests and docs.

## Reproduction Assessment

Yes. Source inspection on current main is high-confidence: buildTtsSystemPromptHint only receives config and agent id, and SpeechProviderPlugin exposes no prompt-hint hook, so selecting Google TTS cannot inject Gemini tag guidance.

## Solution Assessment

Yes. A provider-owned speech hook is the narrow maintainable direction because the existing provider-owned system-prompt contribution is scoped to the active text model provider, not the speech provider selected for TTS.
