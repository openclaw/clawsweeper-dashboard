## Summary

Keep open: current main and the latest tagged release still expose only generic TTS prompt guidance, while the speech-provider contract has no provider-owned prompt-hint hook. The related autoEmotion PR and tracker are open and complementary rather than a superseding implementation, so this remains a product/API boundary decision for TTS/provider owners.

## What This Changes

The issue requests provider-owned, model-aware TTS prompt hints so the active speech provider can teach agents which expressive syntax is valid at runtime.

## Best Possible Solution

Add a backwards-compatible speech-provider prompt-hint hook, append only the selected provider's short hint from speech-core, and let each provider own model/voice/target-specific wording and policy gating after TTS/provider owner review.

## Reproduction Assessment

Yes, by source inspection. Current main builds only the generic TTS prompt, calls it without provider/model/target context, and the SpeechProviderPlugin contract has no prompt-hint hook.

## Solution Assessment

Yes, with owner review. A provider-owned speech hook is the narrow maintainable direction; hard-coding Gemini guidance in speech-core or relying only on docs/skills would not scale across provider and model-specific TTS syntax.
