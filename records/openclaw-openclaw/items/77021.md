## Summary

Keep open: the implementation matches the requested UX boundary, but it introduces raw terminal rendering of persisted session labels and the branch is currently dirty against main.

## What This Changes

The PR updates `src/commands/sessions-cleanup.ts`, its tests, and `CHANGELOG.md` to add a text-only per-label kept/pruned summary for `openclaw sessions cleanup --dry-run`.

## Best Possible Solution

Keep the dry-run-only summary design, sanitize displayed labels, add a malicious-label regression test, then resolve conflicts before landing.

## Reproduction Assessment

Yes for the review finding by source inspection: a label set through `sessions.patch` can contain ANSI or line controls, and the PR logs that persisted label directly in the dry-run summary. The feature request itself is not a bug reproduction case.

## Solution Assessment

No as currently patched: the feature boundary is right, but label display should use the repo's terminal-safe text helper before logging. The remaining merge conflict also needs resolution before this can land.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Sanitize labels before logging them:** `src/commands/sessions-cleanup.ts:102`
  - body: `SessionEntry.label` can be set through `sessions.patch`, and `parseSessionLabel` only trims and length-checks it. Logging the raw label here lets a stored label containing ANSI escapes or newlines forge or rewrite the dry-run output, so sanitize the display label with the existing terminal-safe helper and add a regression test for a control-character label.
  - confidence: 0.9
