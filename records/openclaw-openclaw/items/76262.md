## Summary

Keep open: the Teams SDK migration is active and substantially validated, but the PR introduces one blocking Dockerfile regression. The new Matrix native-addon guard does not honor the documented comma-separated `OPENCLAW_EXTENSIONS` format, so a build that includes `matrix` in a comma list can skip the verifier entirely.

## What This Changes

The branch migrates the Microsoft Teams plugin to `@microsoft/teams.apps`, rewires streaming/invokes/SSO/proactive sends, updates Docker/docs/dependency metadata, and refreshes msteams tests.

## Best Possible Solution

Keep the SDK migration direction, normalize `OPENCLAW_EXTENSIONS` parsing consistently across Dockerfile stages, then refresh maintainer approval with the same live/CI evidence intact.

## Reproduction Assessment

Yes for the review finding: the PR-head Dockerfile documents comma-separated extension lists, but the new guard only splits spaces before looking for `matrix`. That source path is enough to show `OPENCLAW_EXTENSIONS="diagnostics-otel,matrix"` skips the Matrix verifier.

## Solution Assessment

No, not as-is. The Teams SDK migration path looks maintainable after the latest review fixes, but the Docker guard should reuse the repository's existing comma-or-whitespace parsing before this PR merges.
