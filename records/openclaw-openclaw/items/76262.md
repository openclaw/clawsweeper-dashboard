## Summary

Keep open: this is useful active migration work with supplied real Teams proof, but source and dependency inspection still show a blocking Connector routing compatibility issue plus an unversioned dependency freshness bypass.

## What This Changes

This PR migrates the Microsoft Teams plugin from Bot Framework adapter/JWT/streaming plumbing to `@microsoft/teams.apps`, rewiring auth, invoke handling, streaming, proactive sends/edits/deletes, Docker defaults, tests, and package metadata.

## Best Possible Solution

Land the Teams SDK migration only after it preserves the existing Connector routing contract, or introduces an explicit documented cloud/serviceUrl migration path, and narrows the freshness exception to the reviewed version or removes it after the normal age window.

## Reproduction Assessment

Yes. The remaining blockers are source-reproducible: PR head drops stored `serviceUrl` when using SDK app-level send/API calls, and pnpm 10.33.2 treats a bare package-name freshness exclude as all versions.

## Solution Assessment

No. The SDK migration direction and public-cloud proof are promising, but the branch should preserve current serviceUrl/cloud routing compatibility and scope the package freshness bypass before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve the stored serviceUrl for proactive sends:** `extensions/msteams/src/send.ts:440`
  - body: `app.send()` builds its Bot Framework reference from the SDK app's configured `api.serviceUrl`, so this line drops the stored `baseRef.serviceUrl` that current main passes through `continueConversation`. Existing sovereign, regional, or tenant-specific Connector endpoints can misroute proactive sends, cards, polls, file-consent messages, edits, deletes, and threaded fallbacks unless the SDK call is scoped to the stored service URL or an explicit cloud/serviceUrl migration preserves that contract.
  - confidence: 0.9
- **[P3] Scope the Teams SDK freshness bypass:** `pnpm-workspace.yaml:22`
  - body: pnpm parses a bare package-name policy entry as matching all versions, so this exempts every future `@microsoft/teams.apps` release from the workspace freshness guard. Use an exact exception such as `@microsoft/teams.apps@2.0.10`, add exact companion packages only if needed, or remove the exception once the normal age window has passed.
  - confidence: 0.88
