## Summary

Keep open: the current head still changes Teams webhook auth, body parsing, proactive routing, and dependency-age contracts in ways source inspection shows can regress deployed bots, and external live proof is still not inspectable.

## What This Changes

The PR migrates the Microsoft Teams plugin from OpenClaw's local Bot Framework/JWT/REST/streaming adapter code to `@microsoft/teams.apps`, updating Teams streaming, poll, send, edit/delete, Docker, dependency, and test surfaces.

## Best Possible Solution

Keep the SDK migration open until it preserves auth, body-size/auth ordering, fallback route, serviceUrl/tenant/recipient routing, edit/delete serviceUrl behavior, dependency-age policy, changelog, and redacted live Teams proof.

## Reproduction Assessment

Yes. Source inspection against current main and `@microsoft/teams.apps@2.0.10` gives deterministic review-blocker paths for auth-after-body parsing and SDK proactive sends losing stored Teams routing metadata; no live Teams setup was run.

## Solution Assessment

No. SDK adoption may be reasonable, but this patch is not yet the safest implementation because it changes existing auth, routing, fallback-route, edit/delete, and supply-chain contracts without owner-approved compatibility decisions and inspectable live proof.

Overall confidence: 0.93

Full review comments:

- **[P1] Preserve OpenClaw's Teams JWT contract:** `extensions/msteams/src/monitor.ts:252-255`
  - body: This routes inbound webhook auth through the SDK `ServiceTokenValidator`. In `@microsoft/teams.apps@2.0.10`, that validator uses the Bot Framework issuer path and default SDK audiences instead of current main's accepted `https://api.botframework.com` audience plus tenant-scoped Entra v1/v2 issuers and app binding, so valid deployed Teams webhooks can start returning 401.
  - confidence: 0.94
- **[P1] Preserve stored routing metadata in reply fallback:** `extensions/msteams/src/messenger.ts:532-536`
  - body: `sendProactively` now calls `app.send`/`app.reply` with only the conversation id. SDK `App.send` builds a fresh reference from the app's default API service URL and omits the stored service URL, tenant id, Teams user id, and AAD object id that current main forwards for proactive reply routing, which can cause 403s or wrong regional routing.
  - confidence: 0.93
- **[P1] Preserve stored routing metadata in proactive tools:** `extensions/msteams/src/send.ts:440`
  - body: This helper backs proactive poll/card/file sends and now calls `app.send(baseRef.conversation.id, activity)` directly. That bypasses the current adapter path that rehydrates serviceUrl, tenant, recipient id, and AAD object id from the stored conversation reference, so message-tool sends can regress even outside reply fallback.
  - confidence: 0.91
- **[P2] Keep full auth before SDK body parsing:** `extensions/msteams/src/monitor.ts:252-255`
  - body: The retained pre-gate only checks for a Bearer-shaped header, then the SDK `ExpressAdapter` installs `express.json()` before `HttpServer.authorize`. This drops current main's full JWT-before-body-parsing behavior and explicit bounded-body contract for Bearer-shaped requests.
  - confidence: 0.92
- **[P2] Register the standard Bot Framework fallback route:** `extensions/msteams/src/monitor.ts:254`
  - body: Passing only `messagingEndpoint: configuredPath` makes the SDK register a single POST route. Current main also registers `/api/messages` when a custom path is configured, so existing Azure Bot endpoints still posting to the standard Bot Framework path can start getting 404s.
  - confidence: 0.88
- **[P2] Use the conversation service URL for edits and deletes:** `extensions/msteams/src/send.ts:606`
  - body: The edit/delete paths call the global `app.api` client with only the conversation id. That client is constructed from the SDK app's default service URL, while current main sends edits/deletes through the stored conversation serviceUrl, so regional Teams conversations can update or delete against the wrong Connector endpoint.
  - confidence: 0.86
- **[P2] Narrow the Teams release-age exception:** `pnpm-workspace.yaml:22`
  - body: Adding `@microsoft/teams.*` to `minimumReleaseAgeExclude` bypasses the package-age guard for every future Teams SDK package. Use a version-scoped exception or wait out the age gate unless an owner approves the broader supply-chain policy change.
  - confidence: 0.82
- **[P2] Restore visible allowlist resolution failures:** `extensions/msteams/src/monitor.ts:195`
  - body: This downgrades allowlist resolution failures from `runtime.error` to `runtime.log`. Current main intentionally logs these visibly because Graph-resolved allowlist members can be missing afterward, so operators may miss a degraded Teams access-control state.
  - confidence: 0.79
