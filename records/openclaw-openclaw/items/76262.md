## Summary

Keep open: the Teams SDK migration is meaningful and not implemented on current main, but the latest branch still has two concrete review findings before merge.

## What This Changes

This PR migrates the Microsoft Teams plugin from custom Bot Framework plumbing to `@microsoft/teams.apps`, updating webhook auth, invoke handling, streaming, proactive send/edit/delete paths, tests, docs, Docker defaults, and plugin dependencies.

## Best Possible Solution

Land the SDK migration after fixing the Docker extension-list parser and preserving the OpenClaw User-Agent fragment, with targeted regression coverage and the documented public-cloud boundary intact.

## Reproduction Assessment

Not applicable as a PR migration review rather than a standalone bug report. The two findings are source-reproducible from the PR head and the upstream SDK source; the PR's Teams behavior proof is supplied in comments/body, but I did not rerun it.

## Solution Assessment

No. The SDK migration direction is maintainable, but the branch should fix the Docker comma-list guard and the ineffective SDK User-Agent tagging before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.83

Full review comments:

- **[P2] Parse comma-separated Docker extension lists before skipping Matrix checks:** `Dockerfile:84`
  - body: `OPENCLAW_EXTENSIONS` is documented as space- or comma-separated, and the manifest-copy stage converts commas to spaces. This new skip guard only splits on spaces, so a build arg like `diagnostics-otel,matrix` installs Matrix but exits before verifying `matrix-sdk-crypto`. Split on commas here too, or reuse the same parser, before checking for `matrix`.
  - confidence: 0.93
- **[P3] Preserve the OpenClaw User-Agent fragment:** `extensions/msteams/src/sdk.ts:282`
  - body: The SDK constructor spreads `options.client.headers` and then overwrites `User-Agent` with `teams.ts[apps]/<version>`, so the object passed here drops the `OpenClaw/<version>` fragment. Use a client path the SDK actually merges, or add a regression test that inspects the constructed SDK client headers.
  - confidence: 0.9
