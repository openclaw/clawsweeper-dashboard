## Best Possible Solution

Fix the comma-aware Docker Matrix guard and preserve the OpenClaw SDK User-Agent fragment in the actual Teams SDK client, then rerun targeted Teams and Docker checks before merge.

## Solution Assessment

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Parse comma-separated extension lists before skipping Matrix checks:** `Dockerfile:82`
  - body: `OPENCLAW_EXTENSIONS` is documented and parsed earlier as space- or comma-separated. This guard only splits on spaces, so `OPENCLAW_EXTENSIONS=diagnostics-otel,matrix` copies Matrix into the build but exits before verifying `matrix-sdk-crypto`; reuse the comma-aware parsing here.
  - confidence: 0.97

Use a client path that keeps both fragments or add a regression test against the constructed SDK client.
  - confidence: 0.91
