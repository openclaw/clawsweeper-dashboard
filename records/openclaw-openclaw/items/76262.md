## Summary

Keep open. The draft PR is not mergeable yet because it drops current MS Teams auth and proactive-send invariants, and the Docker port change conflicts with the documented/default Teams webhook port.

## What This Changes

The PR migrates the MS Teams plugin toward `@microsoft/teams.apps` App/ExpressAdapter APIs while replacing adapter, streaming, card-action, send, tests, and Docker surfaces.

## Best Possible Solution

Keep the SDK migration draft open, but reapply current main's JWT audience/app-binding, proactive routing metadata, `/api/messages` fallback, and documented port behavior before owner review and validation.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to compare the PR head against current main's JWT global-audience tests and proactive-send metadata path, plus the compose/docs port mismatch.

## Solution Assessment

No. Moving to `@microsoft/teams.apps` may be the right direction, but this patch is not the narrowest maintainable implementation until it preserves current compatibility and security invariants.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve the Teams JWT validator:** `extensions/msteams/src/monitor.ts:247-253`
  - body: This delegates inbound webhook auth to the SDK ExpressAdapter path, but the pinned SDK validator's default audience list does not include `https://api.botframework.com`. Current main has explicit coverage for accepting those Bot Framework global-audience tokens with app binding, so valid Teams webhooks can start returning 401.
  - confidence: 0.91
- **[P1] Forward proactive send routing metadata:** `extensions/msteams/src/send.ts:371-372`
  - body: Calling `app.send(baseRef.conversation.id, activity)` discards the tenantId, aadObjectId, serviceUrl, user, and bot metadata that `buildConversationReference` recovered. Current main keeps that metadata because Bot Framework proactive sends can fail with HTTP 403 without it, so proactive messages, polls, cards, edits, and deletes can regress.
  - confidence: 0.93
- **[P2] Expose the port MS Teams actually listens on:** `docker-compose.yml:46`
  - body: The compose file publishes container port 3000 for MS Teams, but the plugin and docs still default `channels.msteams.webhook.port` to 3978. A default Docker Compose deployment would expose a port where the Teams webhook is not listening.
  - confidence: 0.86
- **[P2] Keep the standard /api/messages fallback:** `extensions/msteams/src/monitor.ts:253`
  - body: Passing only `messagingEndpoint: configuredPath` leaves route registration to the single configured SDK endpoint. Current main also registers `/api/messages`, so deployments with a custom path that still receive Bot Framework traffic on the standard endpoint would start returning 404.
  - confidence: 0.82
