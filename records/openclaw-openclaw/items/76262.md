## Summary

Keep open: this is an active, useful Microsoft Teams SDK migration that is not already implemented on current main, with sufficient live Teams proof supplied, but it still has a low-priority User-Agent tagging regression and should wait for normal maintainer/domain review plus CI completion.

## What This Changes

The PR migrates the Microsoft Teams plugin from custom Bot Framework plumbing to `@microsoft/teams.apps`, rewiring webhook auth, typed invoke routes, streaming, proactive send/edit/delete paths, docs, Docker defaults, tests, and plugin dependencies.

## Best Possible Solution

Land the SDK migration only after fixing the User-Agent tagging regression, completing CI, and getting maintainer/domain agreement that the documented public-cloud Teams boundary is acceptable for this release.

## Reproduction Assessment

Not applicable as a PR migration review rather than a standalone bug report. I inspected current main, the PR branch, and upstream SDK contracts, and the contributor supplied live Teams validation instead of a local repro path.

## Solution Assessment

Mostly yes: moving auth, invokes, streaming, and proactive activity plumbing onto the supported Teams SDK is a maintainable direction. The safer correction is to preserve OpenClaw User-Agent tagging through an SDK-supported client path or regression-tested wrapper because the current plain header injection is overwritten.
