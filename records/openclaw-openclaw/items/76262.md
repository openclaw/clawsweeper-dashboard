## Summary

Keep this PR open. The migration direction is plausible and the supplied live proof is useful, but the exact head still has source-reproducible Connector routing and dependency-freshness blockers, and the live review decision is still CHANGES_REQUESTED.

## What This Changes

The PR migrates the Microsoft Teams plugin from custom Bot Framework adapter/JWT/streaming/invoke plumbing to `@microsoft/teams.apps`, updating proactive sends, streaming, typed invokes, SSO token persistence, tests, Docker defaults, and dependency metadata.

## Best Possible Solution

Land the SDK migration only after preserving the current Connector routing contract, or making a documented cloud/serviceUrl compatibility decision, and narrowing the Teams SDK freshness exception to reviewed versions.

## Reproduction Assessment

Yes. The remaining blockers are source-reproducible: the exact PR head calls SDK app-level proactive APIs while current main carries the stored conversation reference, and pnpm documents that package-name release-age excludes apply to all versions.

## Solution Assessment

No. The SDK migration direction is reasonable and the live proof covers public-cloud behavior, but the branch should preserve or explicitly replace current serviceUrl/cloud routing compatibility and should scope the freshness bypass before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve the stored serviceUrl for proactive sends:** `extensions/msteams/src/send.ts:440`
  - body: `app.send()` builds its outgoing reference from the SDK app's single configured service URL, so this drops the stored `conversationReference.serviceUrl` that current main passes through `continueConversation`. Sovereign, GCC/DoD, regional, or tenant-specific Connector endpoints can misroute proactive sends, edits, deletes, and threaded fallbacks unless the SDK call is scoped to the stored service URL or an explicit cloud/service URL config preserves the old routing contract.
  - confidence: 0.9
- **[P3] Scope the Teams SDK freshness bypass:** `pnpm-workspace.yaml:22`
  - body: pnpm applies package-name `minimumReleaseAgeExclude` entries to all versions, and this unversioned entry exempts every future `@microsoft/teams.apps` release from the workspace freshness delay. Use an exact-version exception for the reviewed SDK version, or remove the bypass once the normal age window has passed.
  - confidence: 0.88
