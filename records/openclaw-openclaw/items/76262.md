## Summary

Keep open: the Teams SDK migration is still useful and current main has not absorbed it, but latest head still has a Docker extension-list parser regression and ineffective OpenClaw User-Agent tagging.

## What This Changes

The PR migrates the msteams plugin from custom Bot Framework plumbing to `@microsoft/teams.apps`, updating Teams webhook auth, invoke handling, streaming, proactive send/edit/delete paths, docs, Docker defaults, dependency manifests, and tests.

## Best Possible Solution

Land the Teams SDK migration after fixing the Docker extension-list parser and constructing the SDK client through a path that preserves the OpenClaw User-Agent fragment.

## Reproduction Assessment

Yes for the review findings: source inspection against PR head and upstream `@microsoft/teams.apps`/`@microsoft/teams.common` 2.0.10 proves the Docker parser mismatch and User-Agent overwrite. Re-running the full live Teams migration is not applicable to this read-only review; supplied live proof covers that separately.

## Solution Assessment

No. The SDK migration direction is maintainable, but the branch should fix the Docker comma-list guard and use a supported SDK client path that actually preserves the OpenClaw User-Agent fragment before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Parse comma-separated extension lists before skipping Matrix checks:** `Dockerfile:84`
  - body: `OPENCLAW_EXTENSIONS` is documented and parsed elsewhere as space- or comma-separated, but this skip guard only splits on spaces. A build arg like `diagnostics-otel,matrix` still installs Matrix, then exits before verifying `matrix-sdk-crypto`, so normalize commas here too or reuse the existing parser.
  - confidence: 0.94
- **[P3] Preserve the OpenClaw User-Agent fragment:** `extensions/msteams/src/sdk.ts:282`
  - body: `createMSTeamsApp` passes `OpenClaw/<version>` through a plain `client.headers.User-Agent` object, but `@microsoft/teams.apps` overwrites that header in the object-client constructor path. Use a real SDK `Client`/factory clone path or another supported hook, and add coverage so outbound Teams calls keep both the SDK and OpenClaw fragments.
  - confidence: 0.9
