## Summary

Keep open: this active PR is not implemented on main and still has source-level blockers around SSO invoke routing, webhook auth/body parsing, Connector serviceUrl routing, and package freshness policy, with maintainer changes requested still live.

## What This Changes

The PR migrates the Microsoft Teams plugin to `@microsoft/teams.apps`, replacing adapter/auth/streaming/invoke/proactive messaging plumbing and updating dependencies, Docker defaults, and tests.

## Best Possible Solution

Keep the migration open until the branch preserves SDK default sign-in semantics, auth/body hardening, per-conversation Connector routing, and exact dependency freshness scope, then get another Teams-domain maintainer pass.

## Reproduction Assessment

Yes. The blockers are source-reproducible from the PR head and `@microsoft/teams.apps@2.0.10` source: user sign-in routes replace SDK defaults, body parsing precedes the bearer gate, and proactive SDK calls use an app-level serviceUrl.

## Solution Assessment

No. The SDK migration direction is plausible, but this patch is not the safest merge path until the compatibility and security-sensitive contracts above are restored or explicitly accepted by maintainers.

  - body: This registers user routes for `signin.token-exchange` and `signin.verify-state`. The Teams SDK removes its system route when a user route has the same name, so this still disables the SDK 200/412 consent-fallback behavior that the PR body says is preserved.

  - confidence: 0.86

  - body: This rebuilds `baseRef` but sends through `app.send(baseRef.conversation.id, ...)`. The SDK builds that proactive reference from the app-level serviceUrl, so regional, sovereign, or tenant-specific stored conversation serviceUrls are ignored.
  - confidence: 0.88
- **[P2] Preserve the stored Connector serviceUrl for edits:** `extensions/msteams/src/send.ts:606`
  - body: The edit path likewise uses `app.api.conversations.activities(...)`, which is rooted at the app's configured/default serviceUrl instead of the stored conversation serviceUrl. Proactive edits can therefore target the wrong Connector endpoint.
  - confidence: 0.86
- **[P2] Scope the release-age bypass to the reviewed version:** `pnpm-workspace.yaml:22`
  - body: This excludes every future `@microsoft/teams.apps` release from the workspace freshness guard. The exception should be exact-version scoped for the reviewed 2.0.10 bump, or removed once the normal release-age window has elapsed.
