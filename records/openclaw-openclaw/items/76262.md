## Summary

Keep open: the SDK migration is useful and has live Teams proof, but current head still has source-reproducible Connector routing and webhook parsing regressions that should block merge.

## What This Changes

The PR migrates the Microsoft Teams plugin to `@microsoft/teams.apps` 2.0.10, replacing Bot Framework auth/adapter/stream/invoke/outbound plumbing and updating Teams tests, Docker defaults, package metadata, and unrelated `src/oc-path` files.

## Best Possible Solution

Restore per-conversation `serviceUrl` routing across every out-of-turn Teams path, keep cheap webhook auth rejection before body parsing, and land the SDK migration only after the Teams-domain review converges.

## Reproduction Assessment

Yes. The remaining blockers are source-reproducible from PR head plus `@microsoft/teams.apps@2.0.10`: `App.send`/`reply` use app-level serviceUrl, while the PR no longer passes stored conversation serviceUrl into proactive paths.

## Solution Assessment

No. The migration direction is plausible, but the current patch is not the safest implementation until it preserves stored Connector routing and the previous cheap unauthenticated webhook rejection behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Put the bearer gate before JSON parsing:** `extensions/msteams/src/monitor.ts:268`
  - body: This global parser now runs before the bearer check, so requests with no `Authorization` header still spend JSON parsing work before being rejected. Keep the bearer-presence middleware first, then install the bounded parser before the SDK route so the SDK parser remains capped without losing current main's cheap unauthenticated rejection path.
  - confidence: 0.84
- **[P2] Route proactive sends through the stored serviceUrl:** `extensions/msteams/src/send.ts:440`
  - body: This rebuilds `baseRef`, but then calls `app.send` with only the conversation id. The SDK's `App.send`/`reply` uses the app-level serviceUrl, so regional, sovereign, or tenant-specific conversations can be sent through the wrong Connector endpoint; the same pattern remains in threaded replies and typing fallback.
  - confidence: 0.9
- **[P2] Route edits and deletes through the stored serviceUrl:** `extensions/msteams/src/send.ts:606`
  - body: The proactive edit path also rebuilds `baseRef` and then calls the app-level `app.api.conversations.activities(...)` client. Existing stored conversation references carry the Connector `serviceUrl` needed for out-of-turn edits and deletes outside the original turn.
  - confidence: 0.88
