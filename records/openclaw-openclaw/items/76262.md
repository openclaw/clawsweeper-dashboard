## Summary

Keep this PR open: the supplied proof now covers visible Teams runtime flows, but source and dependency inspection still show unresolved webhook auth/body-limit parity, stored service URL routing, allowlist diagnostic, and supply-chain policy regressions.

## What This Changes

The PR migrates the bundled Microsoft Teams plugin from local Bot Framework/JWT/REST/streaming plumbing to `@microsoft/teams.apps`, updating streaming, card actions, proactive sends, Docker port exposure, dependency metadata, lockfile entries, and tests.

## Best Possible Solution

Keep the SDK migration open until it preserves the existing webhook security contract and stored-route behavior, narrows the dependency age exception, and gets maintainer review on any intentional compatibility break.

## Reproduction Assessment

Yes for the review blockers: current main, the PR head source, and `@microsoft/teams.apps@2.0.10` source provide a deterministic source-level reproduction of the auth/body-parsing and routing parity changes. I did not run a live Teams webhook reproduction.

## Solution Assessment

No. Adopting the Teams SDK may be a good direction, but this patch is not the safest maintainable solution until it either preserves current OpenClaw contracts or records explicit maintainer approval for the compatibility and supply-chain policy changes.

- **[P1] Preserve the Bot Framework token contract:** `extensions/msteams/src/monitor.ts:252-255`
  - body: This switches inbound webhook validation to the SDK route. In `@microsoft/teams.apps@2.0.10`, `ServiceTokenValidator` uses the cloud token issuer and default client-id audiences, so it does not preserve current main's accepted Bot Framework global audience or tenant-scoped Entra issuer contract; deployed Teams bots can regress to 401.

  - body: The retained middleware only checks that a Bearer-shaped header exists, then the SDK `ExpressAdapter` route runs `express.json()` before `HttpServer.authorize()`. That drops current main's full JWT-before-body-parsing behavior and explicit 1 MiB body limit for Bearer-shaped requests.
  - confidence: 0.92
- **[P2] Use the stored service URL for edits and deletes:** `extensions/msteams/src/send.ts:606`
  - body: The edit/delete paths now call the global `app.api` client with only the conversation id. That client is constructed from the app default service URL, while current main updates and deletes through the stored conversation `serviceUrl`; regional Teams conversations can target the wrong Connector endpoint.
  - confidence: 0.87
- **[P2] Keep allowlist resolution failures visible:** `extensions/msteams/src/monitor.ts:198`
  - body: This downgrades allowlist resolution failures from `runtime.error` to `runtime.log`. Current main intentionally logs these as errors because Graph-resolved allowlist IDs may be missing afterward, leaving access control degraded without an operator-visible failure.
  - confidence: 0.86
- **[P2] Narrow the Teams SDK release-age bypass:** `pnpm-workspace.yaml:22`
  - body: Adding `@microsoft/teams.*` to `minimumReleaseAgeExclude` bypasses the workspace age gate for every future Teams SDK package. Use a version-scoped exception or wait out the age gate unless a maintainer approves the broader supply-chain policy change.
  - confidence: 0.84
- **[P3] Pass the OpenClaw User-Agent through the SDK client:** `extensions/msteams/src/sdk.ts:164-167`
  - body: `createMSTeamsApp` passes `client.headers['User-Agent'] = OpenClaw/<version>`, but the SDK `App` constructor overwrites plain `options.client.headers['User-Agent']` with its own `teams.ts[apps]/<version>` value. The claimed OpenClaw traffic tagging will not be present on outbound SDK calls.
  - confidence: 0.82
