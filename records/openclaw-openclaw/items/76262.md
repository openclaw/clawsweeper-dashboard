## Summary

Keep open: current main has explicit Teams webhook auth, body-size, fallback-route, and proactive-routing contracts that this PR drops while delegating to the SDK; the branch also still lacks inspectable real Teams proof.

## What This Changes

The PR migrates the Microsoft Teams plugin from OpenClaw's local Bot Framework adapter, JWT validator, REST send/edit/delete, and custom streaming helpers to `@microsoft/teams.apps` with related routing, streaming, Docker, dependency, and test changes.

## Best Possible Solution

Keep the migration open until it preserves current Teams auth, body-parse, fallback-route, serviceUrl, tenant, and recipient-routing contracts or has explicit owner-approved compatibility changes, plus a narrow dependency-age exception, changelog entry, and redacted live proof.

## Reproduction Assessment

Yes. Source inspection against current main and `@microsoft/teams.apps@2.0.10` shows deterministic blocker paths: the SDK route parses JSON before full authorization and SDK proactive sends rebuild references from default app state instead of stored Teams routing metadata; no live Teams setup was run.

## Solution Assessment

No. SDK adoption may be reasonable, but this patch should preserve the current auth, body parsing, fallback-route, serviceUrl, tenant-routing, and release-age contracts or carry explicit maintainer approval and live proof for changing them.
