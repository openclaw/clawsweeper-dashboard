## Summary

Keep this PR open: the Teams SDK migration has useful live proof and the main auth/invoke/serviceUrl review discussion appears addressed, but the Dockerfile change introduces a blocking Matrix-bundle regression in documented comma-separated builds.

## What This Changes

This PR migrates the Microsoft Teams plugin to `@microsoft/teams.apps`, replacing Bot Framework adapter/auth/streaming plumbing and updating related Teams tests, docs, Docker, compose, and dependency metadata.

## Best Possible Solution

Fix the Dockerfile extension-list parsing so documented comma and space formats both keep the Matrix native-addon guard active, then let normal CI and maintainer review continue for the SDK migration.

## Reproduction Assessment

Yes, source-reproducible. The Dockerfile itself documents comma-separated `OPENCLAW_EXTENSIONS`, and the PR's new guard only splits on spaces before checking for `matrix`.

## Solution Assessment

No. The Teams SDK migration direction looks maintainable, but this exact patch should not merge until the Dockerfile guard preserves Matrix verification for comma-separated bundled-plugin lists.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Split comma-separated extension lists before skipping Matrix:** `Dockerfile:84`
  - body: The new guard only converts spaces to newlines before checking for `matrix`, but this Dockerfile documents `OPENCLAW_EXTENSIONS` as space- or comma-separated. With the documented `OPENCLAW_EXTENSIONS="diagnostics-otel,matrix"` form, `grep -qx matrix` never matches and the build skips the `matrix-sdk-crypto` native-addon verification even though Matrix is bundled. Normalize commas as delimiters here before deciding to skip the check.
  - confidence: 0.92
