## Summary

Keep open. The Teams SDK migration has substantial live proof and the latest Teams reviewer asks appear addressed, but the Dockerfile change introduces a packaging guard regression: comma-separated `OPENCLAW_EXTENSIONS` lists skip the matrix native-addon verification even when `matrix` is bundled.

## What This Changes

This PR migrates the Microsoft Teams plugin to `@microsoft/teams.apps`, rewires streaming/invoke/send paths, updates Teams docs/tests/dependencies, and adjusts Docker defaults for Teams builds.

## Best Possible Solution

Normalize commas and spaces in the Dockerfile matrix guard, keep the Teams public-cloud service URL documentation, and then continue normal maintainer review for the SDK migration.

## Reproduction Assessment

Yes. A focused shell check against the new guard shows the documented comma-separated form `diagnostics-otel,matrix` does not match `matrix`, while the space-separated form does.

## Solution Assessment

No. The Teams SDK direction looks maintainable after the reviewer follow-ups, but this PR should not merge until the Docker guard uses the same comma/space normalization as the extension copy stage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Split comma-separated extension lists before skipping matrix:** `Dockerfile:73-75`
  - body: `OPENCLAW_EXTENSIONS` is documented as space- or comma-separated, and the Dockerfile even gives `diagnostics-otel,matrix` as the example.
