## Summary

Keep open: the latest PR head still changes security-sensitive Microsoft Teams auth, body parsing, route compatibility, and proactive-send routing contracts, and it lacks inspectable real Teams proof.

## What This Changes

The PR migrates the Microsoft Teams plugin from local Bot Framework adapter/JWT/streaming plumbing to `@microsoft/teams.apps` 2.0.10 and rewires inbound routing, proactive sends, streaming, polls, Docker port exposure, and tests.

## Best Possible Solution

Keep the migration open until it preserves existing Teams auth, webhook, and routing contracts or has explicit owner-approved compatibility removals, plus a narrow supply-chain exception, changelog entry, and redacted live proof.

## Reproduction Assessment

Yes. Source inspection against current main and the `@microsoft/teams.apps@2.0.10` source gives a high-confidence reproduction path for the review blockers; no live Teams setup was run.

## Solution Assessment

No. SDK adoption may be a reasonable direction, but this patch should preserve the current JWT, body-parser, fallback-route, serviceUrl, and tenant-routing contracts or carry explicit owner approval and live proof for changing them.
