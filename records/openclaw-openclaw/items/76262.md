## Summary

Keep this PR open. Current main has not made the Teams SDK migration obsolete, and the latest head addresses the recent auth/poll concerns, but two source-backed blockers remain in the Docker Matrix guard and Teams SDK User-Agent path.

## What This Changes

The branch migrates the Microsoft Teams plugin to `@microsoft/teams.apps` 2.0.10, replacing Bot Framework adapter/streaming/auth plumbing while updating Teams docs, Docker defaults, tests, package metadata, and lockfile state.

## Best Possible Solution

Fix the Docker comma-aware Matrix guard and preserve the OpenClaw SDK User-Agent fragment, then land the SDK migration with the existing public-cloud support boundary intact.

## Reproduction Assessment

Yes for the merge blockers. The Docker parser failure is demonstrated by the same shell pipeline used in the PR-head guard, and the User-Agent loss is source-reproducible from PR `sdk.ts` plus the published Teams SDK constructor.

## Solution Assessment

No, not yet. The SDK migration direction is plausible and the latest auth fixes are in place, but the branch should still repair the Docker comma parser and User-Agent construction before merge.

## Review Findings

Full review comments:

- **[P2] Parse comma-separated extension lists before skipping Matrix checks:** `Dockerfile:82`
  - body: `OPENCLAW_EXTENSIONS` is documented and used as a space- or comma-separated list, and the manifest-copy stage already converts commas to spaces. Reuse the same comma-aware parser before deciding Matrix is absent.
  - confidence: 0.96

Use a client path the SDK actually merges, or add a regression test proving the constructed SDK client keeps both fragments.
  - confidence: 0.9
