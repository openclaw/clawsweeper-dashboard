## Summary

Keep open: this PR has useful live Teams proof, but current head still weakens webhook auth parsing order and drops stored Teams Connector serviceUrl on proactive send/edit/delete paths that current main preserves.

## What This Changes

Migrates the Microsoft Teams plugin to `@microsoft/teams.apps` 2.0.10, replacing Bot Framework adapter/auth/streaming/invoke plumbing and updating Teams messaging, dependencies, Docker defaults, and tests.

## Best Possible Solution

Land the SDK migration only after preserving auth-before-body hardening and per-conversation Connector serviceUrl routing, then get Teams-domain maintainer approval on the narrowed diff.

## Reproduction Assessment

Yes. The remaining blockers are source-reproducible from the PR head and the `@microsoft/teams.apps@2.0.10` implementation; I did not need a live tenant to verify the changed auth ordering or dropped stored serviceUrl.

## Solution Assessment

No. The migration direction is plausible, but the current patch is not the safest merge path until webhook auth parsing and Connector routing parity with current main are restored.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Move JSON parsing back behind auth:** `extensions/msteams/src/monitor.ts:268`
  - body: This installs `express.json()` before even the Bearer-token gate, while current main rejects missing Bearer headers and validates JWT before parsing the body. That means internet-facing unauthenticated webhook traffic can force JSON parsing before rejection, weakening the hardening this route currently has.
  - confidence: 0.9
- **[P2] Preserve stored serviceUrl for proactive sends:** `extensions/msteams/src/send.ts:440`
  - body: This call rebuilds `baseRef` but passes only the conversation id into the SDK. `App.send`/`reply` use the app-level serviceUrl, so regional, sovereign, or tenant-specific conversations can be sent through the wrong Connector endpoint instead of the stored conversation `serviceUrl`.
  - confidence: 0.86
- **[P2] Preserve stored serviceUrl for edits and deletes:** `extensions/msteams/src/send.ts:606`
  - body: The proactive edit/delete paths likewise rebuild `baseRef` and then call `app.api.conversations.activities(...)`, whose client is rooted at the app-level serviceUrl. Existing stored conversation references carry the Connector serviceUrl needed for these operations outside the original turn.
  - confidence: 0.86
