## Summary

Keep open: this draft PR still introduces blocking MS Teams auth, proactive-send, route, and Docker-port regressions relative to current main, and the external contributor has not attached real after-fix Teams proof.

## What This Changes

The PR migrates the Microsoft Teams plugin from custom Bot Framework JWT/proactive/streaming plumbing to `@microsoft/teams.apps`, updates tests, removes the custom streaming helper, and changes Docker metadata.

## Best Possible Solution

Keep the draft open until the SDK migration preserves current auth, proactive-reference, route, port, and changelog contracts, or explicit MS Teams owner/security approval documents why those contracts are being retired with real Teams proof.

## Reproduction Assessment

Yes. A high-confidence source reproduction compares PR head against current main's MS Teams JWT tests, proactive reference builder, route registration, Docker/docs defaults, and the published `@microsoft/teams.apps@2.0.9` `ExpressAdapter` and `app.send` implementation.

## Solution Assessment

No. Full SDK adoption may still be the right long-term direction, but this patch is not the best merge path until it preserves current compatibility contracts or gets explicit owner-approved retirements backed by real Teams evidence.
