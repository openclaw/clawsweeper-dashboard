## Summary

Keep this PR open: the SDK migration is not obsolete and has useful live proof, but the current head still has three source-backed blockers in the Teams authorization and Docker build paths.

## What This Changes

The PR migrates the Microsoft Teams plugin from custom Bot Framework plumbing to `@microsoft/teams.apps`, including webhook auth, typed invoke routes, streaming, proactive send/edit/delete paths, docs, Docker defaults, tests, and dependency updates.

## Best Possible Solution

Land the SDK migration after reusing the existing Teams sender/channel authorization gate for typed SSO and card-action routes, preserving SDK sign-in defaults, and parsing Docker extension lists consistently.

## Reproduction Assessment

Yes for review purposes: the Docker parsing gap and the two Teams authorization bypasses are source-reproducible from the PR head against current-main contracts. I did not run a live Teams instance.

## Solution Assessment

No. The SDK migration direction is maintainable and `app.reply` is supported by the published SDK, but this PR should not merge until the typed routes preserve existing sender policy and the Docker parser handles the documented comma form.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Gate SSO token persistence with Teams sender policy:** `extensions/msteams/src/monitor.ts:469`
  - body: The SDK `signin` event now writes delegated tokens directly to the SSO token store, but this path no longer checks the Teams DM, group, or channel allowlists before saving. A user OpenClaw would otherwise reject can still leave sensitive delegated state behind, so reuse the existing sender-access gate before `tokenStore.save` while keeping the SDK sign-in routes in control.
  - confidence: 0.9
- **[P2] Gate card-action poll votes before recording them:** `extensions/msteams/src/monitor.ts:373`
  - body: `card.action` invokes now record poll votes before the normal guarded handler runs, and the catch-all skips `adaptiveCard/action` afterward. An unallowlisted Teams user who can invoke the card can mutate poll state even though their messages would be dropped; apply the same sender/channel authorization before `recordVote`.
  - confidence: 0.91
- **[P2] Parse comma-separated extension lists in the Matrix guard:** `Dockerfile:84`
  - body: `OPENCLAW_EXTENSIONS` is documented as space- or comma-separated and the earlier Docker stage accepts commas, but this new skip guard only splits on spaces. A build arg such as `diagnostics-otel,matrix` installs Matrix dependencies and then exits before verifying `matrix-sdk-crypto`; split commas here too or share the parser.
  - confidence: 0.93
