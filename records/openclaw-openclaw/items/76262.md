## Summary

Keep open: this draft SDK migration still changes MS Teams webhook auth, proactive routing, route registration, and Docker port behavior in ways that conflict with current-main compatibility contracts, and the PR still lacks sufficient after-fix real behavior proof for an external contribution.

## What This Changes

The branch migrates the Microsoft Teams plugin from custom Bot Framework adapter/JWT/proactive/streaming plumbing to `@microsoft/teams.apps`, updates related tests, deletes the custom streaming helper, and changes Docker metadata.

## Best Possible Solution

Keep the draft open and either preserve current main's auth, proactive-reference, route, port, and changelog contracts inside the SDK migration or get explicit MS Teams owner approval with real Teams proof for intentionally changing them.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to compare the PR head against current main's MS Teams JWT tests, proactive send reference builder, route registration, Docker/docs port defaults, and the upstream SDK `app.send`/`ExpressAdapter` implementation.

## Solution Assessment

No. Full SDK adoption may still be the right long-term direction, but this diff is not the narrowest maintainable solution until it preserves or explicitly retires current main's compatibility and security contracts.
