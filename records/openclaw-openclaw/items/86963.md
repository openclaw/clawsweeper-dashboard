## What This Changes

The issue asks OpenClaw to prevent oversized or orphaned Codex native threads from making `chat.send` return `started` when no agent run executes and messages are silently lost.

## Best Possible Solution

Define a Codex app-server liveness boundary that either proves a turn is runnable before success, rotates an unrunnable persisted thread safely, or surfaces a visible and auditable failure, with a regression fixture for the no-run persisted-thread state.

## Reproduction Assessment

No high-confidence live current-main reproduction is available. Source inspection shows the ACK-before-dispatch window and threshold-dependent native-thread rotation, but the exact persisted 212k-token no-run state still needs a fixture or redacted live proof.

## Solution Assessment

Unclear: preventing silent loss is the right outcome, but force-rotation versus fail-fast or delayed success changes Codex continuity and `chat.send` semantics. Maintainers should choose that boundary before implementation.
