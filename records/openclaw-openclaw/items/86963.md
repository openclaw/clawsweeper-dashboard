## What This Changes

The issue reports a persisted oversized or orphaned Codex app-server thread that makes WebChat `chat.send` acknowledge messages without starting a run or surfacing an error.

## Best Possible Solution

Add a Codex app-server pre-start liveness and recovery path that either force-rotates an unrunnable persisted native thread with a safe handoff or rejects the send with a visible error and audit signal, with regression coverage for no silent `started` response without run progress.

## Solution Assessment

Unclear. The requested outcome is right, but maintainers should choose the recovery semantics for persisted native Codex threads before implementation because force-rotation, summary seeding, and visible fail-closed errors have different session-state tradeoffs.
