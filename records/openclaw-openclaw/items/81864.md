## Summary

Keep open: the configurable plain-language approval direction is useful and the PR has real proof plus green checks at the reviewed head, but simple mode still hides leading environment assignment prefixes while suppressing the raw command text.

## What This Changes

This PR adds `approvals.plugin.language` with `original`, `simple`, and `simple-technical` plugin approval renderers, wires it through plugin approval forwarding plus Telegram/Matrix fallback text, and updates config docs, schemas, baselines, and tests.

## Best Possible Solution

Keep the config-gated renderer with `original` as the default, but fail closed or show a redacted command preview whenever leading env assignments can affect a known command before raw details are hidden.

## Reproduction Assessment

Yes. A focused source reproduction is `buildPluginApprovalPendingReplyPayload(..., { language: "simple" })` with `Command: PATH=./bin ls` or `Command: LD_PRELOAD=./hook.so ls`; source inspection shows the prefix is stripped before classification and no preview is shown for known commands.

## Solution Assessment

No, not as-is. The UX direction is reasonable, but the safe fix is to preserve or conservatively fail closed on env-prefix context before simple mode hides the raw command.
