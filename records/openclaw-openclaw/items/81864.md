## Summary

Keep open. The PR is a useful, active feature branch with real behavior proof, but the new simple approval renderer still has blocking security gaps around curl credentials while it hides the raw technical command.

## What This Changes

The PR adds an opt-in `approvals.plugin.language` setting with simple/simple-technical plugin approval rendering, wires it through Telegram/Matrix/fallback approval paths, and updates schema, docs, baselines, and renderer tests.

## Best Possible Solution

Keep the opt-in plain-language renderer direction, but make the network-command summarizer fail closed on credential-bearing curl forms and redact cookie values before simple mode hides raw details.

## Reproduction Assessment

Yes. Source inspection of PR head plus curl's option contract gives high-confidence paths with `curl -H "Cookie: session=s3cr3t" --data-binary @notes.txt https://example.test/upload`, `curl -H "Authorization: Bearer s3cr3t" https://example.test`, `curl -u alice:s3cr3t https://example.test`, and `curl --oauth2-bearer s3cr3t https://example.test`.

## Solution Assessment

No, not yet. The renderer direction is plausible, but the safe implementation needs to surface or preview credential-bearing curl commands and redact cookie headers before raw command details are suppressed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Redact Cookie headers in approval previews:** `src/infra/plugin-approvals.ts:347-371`
  - body: When a high-risk curl command forces a command preview, the redactor still leaves `Cookie:` and `Set-Cookie:` header values intact. For example, `curl -H "Cookie: session=s3cr3t" --data-binary @notes.txt https://example.test/upload` would post the cookie value into chat while simple mode hides the raw technical block.
  - confidence: 0.92
- **[P1] Surface auth-only curl credentials before hiding raw commands:** `src/infra/plugin-approvals.ts:1674-1682`
  - body: The network summarizer only enables the high-risk preview branch for transfer/config/netrc cases, so auth-only commands such as `curl -H "Authorization: Bearer s3cr3t" https://example.test`, `curl -u alice:s3cr3t ...`, or `curl --oauth2-bearer s3cr3t ...` render as a generic medium network request with no preview. Simple mode then hides the raw command and the user cannot see that credentials are being sent.
  - confidence: 0.9
