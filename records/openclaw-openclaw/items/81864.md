## Summary

Keep open: the PR adds a useful config-gated plain-language approval renderer, but `simple` mode still has source-visible safety gaps where execution-changing shell context can be hidden while raw technical details are suppressed.

## What This Changes

The branch adds `approvals.plugin.language`, a plain-English plugin approval renderer, Telegram/Matrix/fallback forwarding support, docs/generated baselines, extensive renderer tests, plus unrelated oc-path formatting and a dev-dependency lockfile change.

## Best Possible Solution

Keep the config-gated plain-language renderer, but make environment assignment prefixes and leading input redirections visible or high-risk before `simple` mode hides raw details, with focused regression tests.

Yes for the review findings: source inspection shows `KEY=value` prefixes and leading redirection words are stripped before simple-mode summaries can hide raw details. I did not execute a runtime repro in this read-only pass.

## Solution Assessment

No, not as-is. The UX direction is maintainable, but the renderer needs conservative handling for these shell forms before the raw technical command is omitted.
