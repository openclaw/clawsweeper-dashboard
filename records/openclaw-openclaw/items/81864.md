Keep open: the PR is active and useful, but the Telegram plugin approval path still omits the effective allowed-decision list, so simple/simple-technical text can advertise `allow-always` while the buttons and backend only allow `allow-once|deny`.

## What This Changes

The PR adds an opt-in plain-language plugin approval renderer, `approvals.plugin.language` config, Matrix/Telegram/fallback wiring, docs, generated baselines, and approval-renderer tests.

## Best Possible Solution
