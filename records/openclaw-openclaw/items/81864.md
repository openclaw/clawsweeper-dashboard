Adds `approvals.plugin.language` modes, wires plain-language plugin approval rendering through fallback, Telegram, and Matrix paths, and updates config docs, generated baselines, and tests.

## Best Possible Solution

Keep the config-gated renderer direction, but conservatively classify or preview shell prefixes and leading stdin redirections, add focused regressions, and refresh Telegram-visible proof at the final head.

## Solution Assessment

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Fail closed on leading environment assignments:** `src/infra/plugin-approvals.ts:1796-1798`
  - body: `resolveCommandWords()` still drops leading `KEY=value` words before command classification, and `simple` mode can return without raw technical details. Prefixes such as `PATH=./bin ls` or `LD_PRELOAD=./hook.so ls` can change what actually runs while the user sees only a low-risk known-command summary, so fail closed or force a redacted preview for these prefixes.
  - confidence: 0.92
- **[P2] Preserve leading stdin redirection targets:** `src/infra/plugin-approvals.ts:1753`
  - body: Leading redirections are stripped before the command is summarized, but stdin targets are only used by the network upload path. In simple mode, a command like `< .env cat` is reduced to `cat` with no file target or preview even though it reads `.env`, so carry input targets into read summaries or fail closed/show a preview.
  - confidence: 0.86
