The PR adds an opt-in `approvals.plugin.language` config, plain-English plugin approval rendering, Telegram/Matrix/fallback wiring, docs/schema/test updates, and unrelated oc-path formatting churn.

## Best Possible Solution

Yes for the review finding via source inspection: `splitCommandSegments` does not split single `&`, so `sleep 1 & rm -rf /tmp/x` is summarized through the low-risk `sleep` branch while `simple` mode hides raw details. The broader PR is a feature, not a current-main bug report.

## Solution Assessment
