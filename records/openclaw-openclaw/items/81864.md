## Summary

Keep open: the feature is useful and has sufficient real-behavior proof, but the PR is not merge-ready because `simple` mode can still hide raw command details while understating known destructive or write-capable shell forms.

## What This Changes

The PR adds an opt-in `approvals.plugin.language` setting for plain-language plugin approval prompts and wires it through shared approval rendering, forwarding, Telegram, Matrix, config, docs, and tests.

## Best Possible Solution

Land the default-preserving approval-language feature only after the simple renderer fails closed for destructive `find`, read-command writes, and sourced script execution before hiding raw details.

## Reproduction Assessment

Yes for the PR-introduced review findings by source inspection. `find . -delete`, `find . -exec rm -rf {} \;`, `cat source > dest`, and `source ./setup.sh` all reach known-command branches that can avoid a raw preview while understating the real operation.

## Solution Assessment

No as currently proposed. The config and channel wiring are maintainable, but the command summarizer needs conservative fail-closed handling for known dangerous shell forms before `simple` mode hides technical details.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Fail closed on destructive find predicates:** `src/infra/plugin-approvals.ts:636-640`
  - body: The new simple renderer treats every `find` invocation as file search/listing and only raises risk for sensitive or system target paths. Commands like `find . -delete` or `find . -exec rm -rf {} \;` can therefore render as low-risk search with no command preview while `simple` mode hides the raw technical fields. Please detect destructive predicates such as `-delete`, `-exec`, `-execdir`, and `-ok` as high risk or fall back to a raw preview.
  - confidence: 0.9
- **[P2] Treat read-command writes as writes:** `src/infra/plugin-approvals.ts:665-673`
  - body: Known read commands still use the read summary even when the shell command writes through redirection or in-place options. For example, `cat notes.txt > out` or `sed -i ... file` can overwrite files while the prompt says it will read file contents and may stay low risk with no preview. Please detect these write forms and mark them medium/high risk or show the redacted command preview.
  - confidence: 0.82
- **[P2] Do not label sourced scripts as low-risk reads:** `src/infra/plugin-approvals.ts:564-570`
  - body: The `.`/`source` branch classifies non-sensitive paths as a low-risk read, but `source ./setup.sh` executes arbitrary shell in the current process. Because `simple` mode hides the raw command for this known branch, users can see a low-risk local-read prompt for executable script behavior. Please treat sourced scripts as code execution unless a narrowly recognized env-file case is intentionally allowed.
  - confidence: 0.76
