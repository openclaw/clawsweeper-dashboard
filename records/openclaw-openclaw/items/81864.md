## Summary

Keep open. The feature is not implemented on current main, and this PR still has a concrete approval-safety blocker: destructive `find` predicates can be rendered as low-risk file searching while simple mode hides the raw technical command.

## What This Changes

This PR adds an opt-in `approvals.plugin.language` setting for plain-language plugin approval prompts and wires it through shared renderers, forwarding, Telegram, Matrix, docs, schema, and tests.

## Best Possible Solution

Land the default-preserving plain-language approval feature only after destructive `find` predicates fail closed or show a raw preview, with focused renderer regression coverage.

## Reproduction Assessment

Yes for the review finding by source inspection. `find . -delete` reaches the new `find` summary branch, which does not inspect destructive predicates and can produce a low-risk file-search approval with no command preview.

## Solution Assessment

No as proposed. The opt-in renderer/config shape is maintainable, but command summaries must fail closed for destructive `find` predicates before simple mode hides raw technical fields.
