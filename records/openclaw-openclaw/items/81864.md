## Summary

Keep this PR open, but it still needs changes: the new simple-mode command preview redaction misses common prefixed environment credential names, so unknown-command approvals can post secrets to chat despite the redacted-preview behavior.

## What This Changes

The PR adds an opt-in `approvals.plugin.language` setting and a plain-language plugin approval renderer wired through chat approval payloads, config schema/help, docs, and tests.

## Best Possible Solution

Keep the opt-in approval language and unchanged `original` default, but broaden command-preview redaction for env/config key names containing credential words before merge.

## Reproduction Assessment

Yes, source-reproducible for the review finding: an approval description like `Command: GITHUB_TOKEN=s3cr3t custom-tool --send` reaches `buildCommandPreview()` for an unknown command, and the current redaction regex misses prefixed env credential names. I did not run tests because this review was read-only.

## Solution Assessment

No, not yet. The renderer/config direction is maintainable, but preview redaction needs to cover common env credential naming patterns before simple mode can safely hide the raw technical dump.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Redact prefixed env credential names in previews:** `src/infra/plugin-approvals.ts:411-413`

Broaden this to redact env/config keys that contain token/secret/password/api-key style words before emitting chat preview text.
