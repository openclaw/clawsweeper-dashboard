## Summary

Keep open: the feature direction is plausible and the supplied real-behavior proof is sufficient, but the patch still has two merge-blocking defects: an unrelated Telegram type dependency downgrade and an approval-safety parser gap for option-bearing shell `command` wrappers.

## What This Changes

The PR adds an opt-in `approvals.plugin.language` setting for plain-language plugin approval prompts and wires it through shared rendering, forwarding, Telegram, Matrix, SDK exports, docs, and tests.

## Best Possible Solution

Keep the opt-in renderer with `original` as the default, restore the root dependency pins to current main, and make shell-wrapper parsing fail closed whenever it cannot confidently expose the command being approved.
