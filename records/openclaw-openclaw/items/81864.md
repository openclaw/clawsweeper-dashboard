## Summary

Keep open: the feature has useful proof and the default remains legacy, but the branch still has a blocking approval-safety bug where simple mode can hide leading environment assignments before suppressing raw technical text.

## What This Changes

The PR adds configurable plain-language plugin approval prompts across config, plugin approval rendering, fallback forwarding, Telegram/Matrix approval text, docs, generated baselines, and tests.

## Best Possible Solution

Keep the config-gated plain-language renderer with `original` as the default, but fail closed or show a redacted command preview whenever leading environment assignments can affect a known command before raw text is hidden.

## Reproduction Assessment

Yes for the blocking review finding: source inspection shows leading `KEY=value` words are stripped before command classification, and `simple` mode returns before technical details are appended. I did not execute the renderer in this read-only review.

## Solution Assessment

No, not as-is. The UX direction is plausible, but the approval summarizer must preserve or conservatively fail closed on env-prefix context before it can safely hide the raw command.
