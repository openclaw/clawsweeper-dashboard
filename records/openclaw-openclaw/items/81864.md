## Summary

Keep open: the feature is plausible and the latest head has green CI/proof, but the simple fallback text can advertise decisions that the interactive payload intentionally disallows.

## What This Changes

The PR adds an opt-in `approvals.plugin.language` setting for simple/simple-technical plugin approval prompts and wires it through config, shared renderers, Telegram/Matrix delivery, docs, tests, baselines, and a dependency lockfile change.

## Best Possible Solution

Keep the opt-in plain-language approval feature, but make the manual fallback text use the same effective decision list as buttons/reactions and cover that path with a focused regression.

## Reproduction Assessment

Yes. From source, render a simple-language Matrix plugin approval whose view actions are `allow-once` and `deny`; the text path defaults to `allow-once|allow-always|deny` while the interactive payload uses the narrowed list.

## Solution Assessment

No. The overall direction is maintainable, but the current patch should thread effective allowed decisions into the message builder before merge.
