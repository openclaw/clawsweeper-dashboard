## Summary

Keep open; current main still uses raw plugin approval copy, and this PR’s config-gated plain-language direction is useful, but the patch still has approval-safety defects that can hide high-risk details in `simple` mode.

## What This Changes

Adds `approvals.plugin.language` modes, a plain-language plugin approval renderer, Telegram/Matrix fallback wiring, config/docs updates, and focused renderer tests.

## Best Possible Solution

Keep the config-gated plain-language mode, but make `simple` approval text preserve or preview every hidden medium/high-risk detail before raw command details are suppressed.

## Reproduction Assessment

Yes, by source inspection for the blocking review findings. A command like `pwd && ls && ps && cat notes.txt && rm -rf /tmp/x` can hide the delete target after truncation, and `bash -c 'rm -rf "$1"' sh /tmp/x` drops `/tmp/x` while unwrapping.

## Solution Assessment

No, not as-is. The shared config-gated renderer is a maintainable direction, but simple mode must prioritize dangerous summaries or force a redacted preview whenever truncation or shell-wrapper unwrapping hides actionable risk details.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Surface hidden high-risk steps before truncating:** `src/infra/plugin-approvals.ts:277-281`
  - body: `simple` mode removes raw command details, but this loop always emits only the first four unique action lines. A command like `pwd && ls && ps && cat notes.txt && rm -rf /tmp/x` can show `Risk: High` while replacing the actual delete target with `1 more background step(s)` and no command preview, so dangerous summaries need priority or a forced redacted preview before truncation.
  - confidence: 0.91
- **[P2] Preserve shell-wrapper arguments when unwrapping:** `src/infra/plugin-approvals.ts:734`
  - body: `extractShellWrapperCommand` returns only the `-c` body and drops later wrapper arguments. For `bash -c 'rm -rf "$1"' sh /tmp/x`, simple mode summarizes deletion of `$1` while hiding `/tmp/x` and the raw command, so wrapper unwrapping should fail closed or force a preview when the body can reference positional args.
  - confidence: 0.86
