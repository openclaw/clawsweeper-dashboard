## Best Possible Solution

Keep the opt-in plain-language renderer, but first fail closed on curl cookie sources and redact cookie header values before simple mode suppresses raw technical details.

## Solution Assessment

No, not yet. The renderer direction is plausible, but the safe solution must treat curl cookie headers, cookie files, and cookie jars like credential/file-transfer surfaces before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Redact Cookie headers in approval previews:** `src/infra/plugin-approvals.ts:347-371`
  - body: When a high-risk curl command forces a command preview, `redactSensitiveCommandText()` still leaves `Cookie:` or `Set-Cookie:` header values intact. For example, `curl -H "Cookie: session=s3cr3t" --data-binary @notes.txt https://example.test/upload` would post the cookie value into chat while simple mode hides the raw technical block.
  - confidence: 0.9
- **[P1] Surface curl cookie options before hiding raw details:** `src/infra/plugin-approvals.ts:1805-1808`
  - body: curl's `-b/--cookie` can send cookie data or read a cookie file, and `-c/--cookie-jar` writes cookies to a local file, but the transfer parser only handles netrc/config/upload/output branches and misses the short `-c` form. That lets simple mode render cookie credential use or `curl -c .env ...` as a generic medium network request with no preview.
  - confidence: 0.87
