## Summary

The PR is still useful and should stay open, but one security-sensitive renderer path still hides the exact pipe-to-shell command in `simple` mode.

## What This Changes

Adds `approvals.plugin.language` modes and wires plain-English plugin approval rendering through config, SDK reply payloads, Telegram/Matrix/fallback delivery, docs, generated baselines, and tests.

## Best Possible Solution

Keep the config-gated plain-language approval approach, but require every high-risk path that says command text is not visible to show a redacted command preview before simple mode hides raw details.

## Solution Assessment

No, not as-is. The default-preserving config-gated design is maintainable, but pipe-to-shell approvals need a redacted command preview before this can safely land.
