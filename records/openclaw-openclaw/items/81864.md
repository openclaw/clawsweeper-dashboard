## Summary

Keep open: the plain-language approval direction is useful, but PR head still has a security-sensitive renderer bug where leading environment assignments can disappear from simple approval text while raw details are hidden. The Telegram-visible proof also needs to be refreshed after the latest renderer changes before merge.

## What This Changes

The PR adds an `approvals.plugin.language` config option, a plain-language plugin approval renderer, Telegram/Matrix/fallback wiring, docs/generated baselines, broad renderer tests, and some adjacent dependency/formatting cleanup.

## Best Possible Solution

Keep the config-gated plain-language renderer, but fail closed or force a redacted preview for leading environment assignments on known commands, add focused regressions, and then refresh Telegram-visible proof at the final head.

## Reproduction Assessment

Yes for the blocking review finding: PR-head source inspection shows `resolveCommandWords()` strips leading `KEY=value` words before `simple` mode can hide raw details, so a command like `LD_PRELOAD=./hook.so ls` would be summarized as the inner `ls` action without a preview. I did not run tests in this read-only review.

## Solution Assessment

No, not as-is. The UX direction is maintainable, but the renderer needs to preserve, classify, or preview leading environment prefixes before raw technical command text is omitted.
