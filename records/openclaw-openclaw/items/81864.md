## Summary

Keep open: the UX direction is plausible, but the current patch still lets `simple` mode hide execution-changing shell context while suppressing raw technical details.

## What This Changes

The PR adds an `approvals.plugin.language` config option with original, simple, and simple-technical plugin approval rendering, channel wiring for Telegram/Matrix/fallback delivery, docs, tests, generated baselines, and a Telegram type dependency alignment.

## Best Possible Solution

Keep the config-gated plain-language renderer with `original` as the default, but make simple summaries fail closed or show a redacted preview whenever shell prefixes or stdin sources affect execution.

## Reproduction Assessment

Yes for the review findings: PR-head source inspection shows env assignment prefixes and leading stdin redirections are stripped before `simple` mode hides raw technical details. I did not run tests because this was a read-only review.

## Solution Assessment

No, not as-is. The proposed UX/config shape is maintainable, but the renderer needs conservative handling for these shell forms before raw technical fields can be hidden.
