## Summary

Keep open: the branch still needs a safety fix before maintainer approval because simple mode can hide a later dangerous action or target when more than four action summaries are produced while the raw command is intentionally hidden.

## What This Changes

The PR adds a configurable `approvals.plugin.language` option for plain-language plugin approval prompts and wires it through shared forwarding plus Telegram and Matrix approval payloads.

## Best Possible Solution

Fix simple-mode truncation so high-risk or preview-worthy actions are always visible, then let maintainers decide whether to land the default-preserving config-gated approval UX.

## Reproduction Assessment

Yes for the review finding by source inspection: a command with four benign steps before `rm -rf /tmp/x` will put the delete target behind the generic “more background step(s)” line because action truncation happens before high-risk preservation. Reproduction is not applicable to the feature request itself.

## Solution Assessment

No, not as-is. The shared config-gated renderer is the right shape, but simple mode must surface high-risk or preview-worthy actions before it hides the raw command.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Surface high-risk actions before truncating the summary:** `src/infra/plugin-approvals.ts:277-281`
  - body: `simple` mode hides the raw technical command, but this loop only renders the first four action summaries in command order and collapses the rest into a count. For a command like `pwd && ls && echo ok && ps && rm -rf /tmp/x`, the delete target falls behind `1 more background step(s)`; `rm` does not force a command preview, so the approval can show `Risk: High` without the dangerous action or target. Prioritize high-risk/preview actions or force a redacted preview whenever truncation would hide them.
  - confidence: 0.87
