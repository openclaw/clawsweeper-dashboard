## Summary

Keep open: the feature direction is useful and the PR has addressed several prior review blockers, but the latest head still has a blocking credential-redaction bug in the new simple approval preview. Current main still uses the raw plugin approval message, while this PR’s simple mode can emit a command preview that leaks part of a multi-cookie `Cookie:` header.

## What This Changes

The PR adds an opt-in `approvals.plugin.language` setting with `simple` and `simple-technical` plugin approval rendering across shared fallback, Telegram, Matrix, config docs/schema, and tests.

## Best Possible Solution

Keep the opt-in plain-language approval mode, but land it only after command-preview redaction fully covers quoted and semicolon-separated credential header values with focused regressions.

## Reproduction Assessment

Yes for the PR finding: source inspection plus a focused redaction-chain probe shows the preview for a multi-cookie `Cookie:` header still leaks a later cookie value. This is a PR-head reproduction, not a current-main bug path.

## Solution Assessment

No, not yet. The opt-in renderer is a maintainable direction, but hiding raw technical details is only safe if the redacted preview never leaks credential header values.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Redact the full Cookie header value:** `src/infra/plugin-approvals.ts:383`
  - body: The new preview redactor only replaces the first non-whitespace token after `Cookie:`. Redact the whole quoted header value, including semicolon-separated cookies, and add a regression for that form.
