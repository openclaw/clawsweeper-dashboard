## Summary

Needs changes before merge: the opt-in plain-language approval path is useful and has real behavior proof, but the current PR can still hide shell-valid attached redirections while simple mode suppresses raw command text.

## What This Changes

The PR adds an `approvals.plugin.language` config option, plain-language plugin approval rendering, docs/config schema updates, and Telegram/Matrix/fallback wiring.

## Best Possible Solution

Keep the default-conservative language option, but make shell redirection parsing detect attached `<`/`>` forms and add no-space regressions before hiding raw technical command text.

## Reproduction Assessment

Yes for the blocking review finding by source inspection: the PR regexes require whitespace before redirection operators, while shell accepts attached forms like `cat notes.txt>out.txt`. I did not execute the PR branch because this review is read-only.

## Solution Assessment

No, not as currently proposed: the opt-in default is conservative, but simple mode must detect shell-valid attached redirections before it can safely hide raw technical command text.
