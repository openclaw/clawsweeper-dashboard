## Summary

Keep this draft PR open, but it is not merge-ready: the new approval-risk classifier can understate destructive shell commands, and the provided proof does not show the changed approval prompt in a real channel flow.

## What This Changes

The PR adds an opt-in `approvals.plugin.language: "simple"` mode that renders plain-language plugin approval summaries and wires it into Telegram, Matrix, fallback delivery, config schema/help, and tests.

## Best Possible Solution

Keep the mode opt-in, make the plain-language classifier conservative for destructive or unknown shell constructs, and land only after real channel proof shows the resulting approval prompt.

## Reproduction Assessment

Yes for the review finding: source inspection shows `find . -delete` reaches the new `find` summary branch and is described as file listing with low risk when the target path is not sensitive/system. Reproduction of the PR's intended visible UX is not yet provided in a real channel.

## Solution Assessment

No. The opt-in direction is plausible, but the current classifier needs conservative handling for destructive command forms and real chat proof before this is the safest maintainable implementation.
