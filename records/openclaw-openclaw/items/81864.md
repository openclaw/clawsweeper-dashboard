## Summary

Keep open. The config-gated plain-language approval UX is useful and current main does not already provide it, but the current PR head still hides security-relevant command context in `simple` mode when raw technical details are intentionally removed.

## What This Changes

The PR adds `approvals.plugin.language` modes and wires plain-language plugin approval rendering through config, docs, shared reply payloads, Telegram, Matrix, and fallback forwarding.

## Best Possible Solution

Keep the `original` default and land the plain-language modes only after `simple` mode conservatively surfaces or fails closed on env prefixes and text-processor input operands, with focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction is `LD_PRELOAD=./hook.so ls` or `jq . .env` with `language: "simple"`: PR-head source strips or ignores the security-relevant context while `simple` returns before raw technical details are appended.

## Solution Assessment

No, not as-is. The config-gated UX direction is maintainable, but the summarizer must preserve a redacted preview or fail closed before it hides raw command details.
