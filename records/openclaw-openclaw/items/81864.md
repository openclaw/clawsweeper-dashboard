## Summary

This PR is still a useful feature direction with sufficient real-behavior proof, but it should not merge yet because `simple` mode can still hide security-relevant command context while suppressing the raw technical approval text.

## What This Changes

The PR adds `approvals.plugin.language` with plain-language plugin approval rendering, command summaries/redacted previews, config docs/schema updates, and Telegram/Matrix/forwarder wiring.

## Best Possible Solution

Keep the config-gated plain-language approval mode, but make `simple` output preserve or preview every hidden medium/high-risk command detail before raw technical fields are suppressed.

## Reproduction Assessment

Yes. Source inspection of PR head shows `pwd && ls && ps && cat notes.txt && rm -rf /tmp/x` can hide the delete target after the four-action cap, and `bash -c 'rm -rf "$1"' sh /tmp/x` can hide `/tmp/x` while `simple` mode suppresses raw details.

## Solution Assessment

No, not as-is. The shared configurable renderer is the right direction, but the implementation needs to prioritize or preview hidden risky summaries and preserve shell-wrapper positional arguments before merge.
