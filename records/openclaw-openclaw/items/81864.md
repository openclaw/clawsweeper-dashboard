## Summary

Keep this PR open: the plain-language approval mode is useful and current main does not already have it, but the current head still has concrete approval-safety blockers where `simple` mode can hide security-relevant command context while suppressing raw technical fields.

## What This Changes

The PR adds an `approvals.plugin.language` config option and a plain-language plugin approval renderer wired into fallback, Telegram, and Matrix delivery.

## Best Possible Solution

Keep the configurable plain-language renderer, but make `simple` mode preserve or preview every omitted medium/high-risk command detail before raw technical fields are hidden.

## Reproduction Assessment

Yes. Source inspection of PR head shows `pwd && ls && ps && cat notes.txt && rm -rf /tmp/x` can hide the delete target after the four-action cap, and `bash -c 'rm -rf "$1"' sh /tmp/x` can hide `/tmp/x` after shell-wrapper unwrapping.

## Solution Assessment

No, not as-is. The shared configurable renderer is the right direction, but `simple` mode must prioritize or preview hidden risky summaries and preserve shell-wrapper positional context before merge.

  - confidence: 0.88
