## Summary

Keep open: this is a plausible feature PR with supplied real behavior proof, but the latest head still has safety defects where simple mode can hide execution-changing shell context while omitting raw technical fields.

## What This Changes

The PR adds an `approvals.plugin.language` config surface and simple plugin-approval command summarization, wires it into shared, Telegram, Matrix, and fallback delivery, and updates docs, tests, generated baselines, and dependency metadata.

## Best Possible Solution

Keep the config-gated plain-language renderer with `original` as the default, but merge it only after hidden shell prefixes and redirections either appear in the prompt/preview or fail closed as high risk.

## Reproduction Assessment

Yes for the review findings: source inspection of PR head shows env assignments and leading input redirections are stripped before `simple` mode can hide raw technical details. I did not run tests because this was a read-only review.

## Solution Assessment

No, not as-is. The UX direction is maintainable, but the renderer needs conservative handling for these shell forms before plain-language mode can safely suppress raw technical fields.
