## Summary

Keep this PR open: the opt-in plain-language approval direction is useful, but the latest head still leaks later cookie pairs from a multi-cookie `Cookie:` header in the new simple-mode command preview while hiding the raw technical details.

## What This Changes

The PR adds an opt-in `approvals.plugin.language` config with simple and simple-technical plugin approval rendering across shared approval rendering, Telegram/Matrix forwarding, config docs/schema, generated baselines, and tests.

## Best Possible Solution

Land the opt-in approval language only after command-preview redaction consumes complete `Cookie` and `Set-Cookie` header values and regression coverage includes semicolon-space multi-cookie forms.

## Reproduction Assessment

Yes for the PR finding: applying the latest redaction pattern to a quoted multi-cookie `Cookie:` header leaves the later cookie pair in the preview. This is a PR-head reproduction, not a current-main bug path.

## Solution Assessment

No, not yet. The opt-in renderer and unchanged default are a maintainable direction, but simple mode cannot safely hide raw details until redacted previews stop leaking complete credential header values.
