## Summary

Keep open: the feature is useful, but the current PR head still has approval-safety blockers where simple mode can hide raw command text after under-summarizing risky commands.

## What This Changes

The PR adds an opt-in `approvals.plugin.language` setting for plain-language plugin approval prompts and wires it through config, shared approval renderers, Telegram, Matrix, docs, and tests.

## Best Possible Solution

Land the opt-in plain-language mode only after the shell summarizer fails closed for conditional bodies and command-specific flag parsing, with unrelated formatting churn split out or removed.

## Reproduction Assessment

Yes for the PR findings: PR-head source inspection shows conditional bodies after `then`/`do` are stripped and global option parsing can skip targets such as `cat -n .env`. This is not a current-main bug reproduction because the PR adds a new opt-in mode.

## Solution Assessment

No as proposed. The default-preserving config direction is maintainable, but simple mode must conservatively summarize or fail closed before raw command text is hidden.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Summarize conditional bodies before hiding raw commands:** `src/infra/plugin-approvals.ts:367-370`
  - body: Simple mode hides the raw command, so the shell-control parser cannot drop everything after `then` or `do`. A command like `if [ -f ok ]; then rm -rf data; fi` is summarized from only the condition while the delete step disappears; parse the body/else branches or mark the whole conditional high-risk.
  - confidence: 0.91
- **[P2] Parse command flags per command:** `src/infra/plugin-approvals.ts:1203-1229`
  - body: `getCommandTargetArgs` treats flags such as `-n` and `-f` as value-taking for every command. For commands where those are boolean, such as `cat -n .env`, the sensitive path is skipped before target and risk checks, leaving a low-risk read summary while simple mode hides the raw technical command.
  - confidence: 0.86

## Security Review

Status: needs_attention

  - confidence: 0.91
