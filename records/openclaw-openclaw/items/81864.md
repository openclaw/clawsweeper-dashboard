The PR adds `approvals.plugin.language`, a shared simple/simple-technical plugin approval renderer, and wiring through fallback forwarding plus Telegram, Matrix, config, docs, and tests.

## Best Possible Solution

Keep the configurable plain-language renderer, but make shell-wrapper paths preserve concrete argv in a redacted preview or fail closed before simple mode can hide raw command details.

## Solution Assessment

No, not yet. The shared renderer and config direction are maintainable, but shell-wrapper positional argv must stay visible or force a high-risk command preview before merge.
