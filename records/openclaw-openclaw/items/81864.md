## Summary

Keep open. The feature direction is useful and has real behavior proof, but the latest PR head still has two source-reproducible simple-mode parser holes that can hide important command context once raw technical details are suppressed.

## What This Changes

The PR adds `approvals.plugin.language` with `original`, `simple`, and `simple-technical` plugin approval rendering, wires it through shared fallback/Telegram/Matrix paths, and updates config docs, baselines, and renderer tests.

## Best Possible Solution

Keep the configurable plain-language renderer, but make simple mode fail closed whenever command parsing drops concrete execution context before hiding raw technical fields.

## Reproduction Assessment

Yes. Source inspection at PR head gives high-confidence repro paths with simple approval rendering for `bash -c 'rm -rf "$1"' sh /tmp/x` and `npm -w app exec create-vite@latest app`.

## Solution Assessment

No, not yet. The shared renderer/config approach is maintainable, but it must preserve shell-wrapper argv context and parse npm workspace shorthand correctly before simple mode can safely suppress raw details.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve shell-wrapper argv context:** `src/infra/plugin-approvals.ts:734`
  - body: `extractShellWrapperCommand` returns only the shell `-c` body and drops argv after it. For `bash -c 'rm -rf "$1"' sh /tmp/x`, simple mode can summarize deletion of `$1` without a command preview while the actual `/tmp/x` target disappears with the raw technical block, so keep those argv values visible or fail closed when positional parameters are referenced.
  - confidence: 0.91
- **[P2] Consume npm workspace values before subcommand parsing:** `src/infra/plugin-approvals.ts:1485-1486`
  - body: `-w` is treated as a value-less global before value-taking options, so `npm -w app exec create-vite@latest app` stops at `app` as the subcommand and renders generic medium-risk package-manager text without a preview. Handle npm workspace shorthand as value-taking while preserving pnpm workspace-root semantics, so later `exec`/`x` remains high-risk and visible.
  - confidence: 0.86
