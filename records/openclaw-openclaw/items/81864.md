## Summary

Keep open: the PR implements a useful config-gated plain-language approval renderer, but it still has a concrete approval-safety blocker. `simple` mode removes the raw command while truncating action summaries before guaranteeing that later high-risk actions remain visible.

## What This Changes

Adds `approvals.plugin.language` with `original`, `simple`, and `simple-technical` plugin approval rendering across shared approval rendering, forwarding, Telegram, Matrix, config docs, and tests.

## Best Possible Solution

Keep the default-preserving config-gated renderer design, but make `simple` mode always surface high-risk or preview-worthy actions before hiding raw technical fields.

## Reproduction Assessment

Yes for the blocking review finding by source inspection. A command like `pwd && ls && ps && cat notes.txt && rm -rf /tmp/x` reaches the first-four-action truncation while `rm` does not force a command preview, so the delete target can be hidden.

## Solution Assessment

No, not as-is. The shared config-gated renderer is the right direction, but truncation must prioritize high-risk or preview-worthy actions, or force a redacted command preview whenever truncation hides them.
