The branch adds an opt-in `approvals.plugin.language` renderer for plugin approvals, wires it through shared/Telegram/Matrix approval delivery, updates docs/schema/tests/baselines, and carries some generated/dependency/version churn.

## Best Possible Solution

## Reproduction Assessment

Yes for the review finding by source inspection: `summarizePackageManagerCommand` treats the first non-option argument as the subcommand, so value-taking global options can hide a later `install`/`update`. I did not run tests in this read-only review.
