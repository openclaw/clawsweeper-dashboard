Keep open: the PR is a useful opt-in plain-language approval feature, but simple mode still hides text-filter file reads before hiding raw technical details, which is an approval-safety blocker.

## What This Changes

The PR adds `approvals.plugin.language` with `original`, `simple`, and `simple-technical` plugin approval rendering, wires it through fallback/channel approval payloads, and updates config docs, generated baselines, and tests.

## Best Possible Solution
