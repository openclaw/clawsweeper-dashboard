## Summary

Keep open. The PR is a useful direction with supplied real behavior proof, but `simple` mode still suppresses raw technical details while hiding security-relevant command context in two source-reproducible command summaries.

## What This Changes

The PR adds `approvals.plugin.language` for original/simple/simple-technical plugin approval text, wires it through shared, Telegram, and Matrix approval rendering, and adds config docs/tests plus shell command summarization coverage.

## Best Possible Solution

Keep the config-gated plain-language approval mode, but make `simple` output preserve or preview every hidden medium/high-risk command detail before raw technical fields are suppressed.

## Reproduction Assessment

Yes. Source inspection at PR head shows `pwd && ls && ps && cat notes.txt && rm -rf /tmp/x` can hide the delete target after the four-action cap, and `bash -c 'rm -rf "$1"' sh /tmp/x` drops `/tmp/x` while summarizing only `$1`.

## Solution Assessment

No, not as-is. The shared config-gated renderer is the right direction, but `simple` mode needs to prioritize or preview hidden risky summaries and fail closed when shell-wrapper positional arguments affect the command body.

## Review Findings
