The branch adds `approvals.plugin.language` for plain/simple-technical plugin approval text, wires it through shared, Telegram, Matrix, and forwarding paths, updates config docs/schema/baselines/tests, and carries minor oc-path/Discord conflict-resolution edits.

## Best Possible Solution

Keep the configurable plain-language renderer, but preserve hidden shell-wrapper context or fail closed with a redacted original-command preview before simple mode can hide raw details.

## Reproduction Assessment

Yes. Source inspection at PR head shows `extractShellWrapperCommand` returns only the shell `-c` body, so `bash -c 'rm -rf "$1"' sh /tmp/x` loses `/tmp/x` before simple mode returns without the raw technical block.

## Solution Assessment

No, not yet. The shared configurable renderer is a maintainable direction, but it needs the shell-wrapper argv safety fix before the raw-details-hiding mode is safe to merge.
