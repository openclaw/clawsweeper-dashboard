## Summary

Keep open: the feature direction is useful and the proof gate is satisfied, but current PR head still has two source-reproducible approval-safety defects where `simple` mode can hide execution-changing shell context before raw technical details are omitted.

## What This Changes

The PR adds an `approvals.plugin.language` option with plain-language plugin approval rendering, wires it through fallback, Telegram, and Matrix delivery, and updates config docs, generated baselines, tests, and related dependency metadata.

## Best Possible Solution

Keep the config-gated plain-language approval renderer with `original` as the default, but make the simple renderer conservatively surface or fail closed on hidden shell context before it can suppress the raw technical block.

## Reproduction Assessment

Yes for the review findings: source inspection of PR head shows env assignments and leading input redirections are stripped before `simple` mode can hide raw technical details. I did not execute tests because this was a read-only review.

## Solution Assessment

No, not as-is. The config-gated UX direction is maintainable, but the renderer needs conservative handling for these shell forms before the plain-language mode can safely omit technical details.
