## Summary

Keep open: the opt-in approval-language feature is not on current main, and the PR still has a concrete approval-safety bug where escaped ampersands in shell URLs are split before URL credential classification. CI and real behavior proof are present, but this should be fixed before merge.

## What This Changes

The PR adds an opt-in `approvals.plugin.language` setting and simple/simple-technical plugin approval rendering, wired through shared reply payloads, fallback forwarding, Telegram, Matrix, config help/schema, docs, and tests.

## Best Possible Solution

Land the opt-in shared renderer after preserving escaped shell separators through credential classification and adding a focused renderer regression, while keeping `original` as the default.

## Reproduction Assessment

Yes for the review finding by source inspection: PR head does not preserve a backslash escape before `&`, so a shell-valid URL like `curl https://example.test/path?x=1\&access_token=secret` is split before URL credential analysis. I did not run tests in this read-only review.

## Solution Assessment

No. The shared opt-in renderer is a reasonable ownership boundary, but the parser should preserve escaped shell metacharacters or fail closed before simple mode can under-label credential-bearing network commands.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve escaped ampersands while splitting shell commands:** `src/infra/plugin-approvals.ts:561-563`
  - body: `splitCommandSegments` treats every non-redirection `&` as a separator without honoring a preceding backslash. A shell-valid command such as `curl https://example.test/path?x=1\&access_token=secret` is split before `hasNetworkUrlCredentials` sees the full URL, so simple mode can miss the intended high-risk credential-URL classification and only show a medium network/unknown summary. Keep escaped `&` inside the segment, or conservatively mark that case high-risk, and add a regression.
  - confidence: 0.86
