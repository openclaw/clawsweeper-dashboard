## Summary

Keep open: the feature direction and supplied real-behavior proof are useful, but the current PR head still has approval-renderer safety regressions where `simple` mode can hide security-relevant command context while suppressing the technical block.

## What This Changes

The PR adds `approvals.plugin.language`, plain-English plugin approval rendering, Telegram/Matrix/fallback wiring, config docs/schema/baseline updates, and focused renderer tests.

## Best Possible Solution

Keep `original` as the default and land the plain-language modes only after `simple` mode preserves, previews, or fails closed on environment prefixes and text-processor file operands.

## Reproduction Assessment

Yes, for the blocking review findings: PR-head source inspection shows `resolveCommandWords()` strips leading `KEY=value` before classification and `getCommandTargetArgs()` returns `[]` for text processors. Representative checks are `LD_PRELOAD=./hook.so ls` and `jq . .env` with `approvals.plugin.language: "simple"`.

## Solution Assessment

No, not as-is. The config-gated renderer with unchanged default is maintainable, but simple mode must fail closed or show a redacted preview before hiding raw details for these command shapes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve env prefixes before hiding commands:** `src/infra/plugin-approvals.ts:1796-1798`
  - body: `resolveCommandWords()` drops every leading `KEY=value` token before classifying the command, and `simple` mode returns before the technical block. A command like `LD_PRELOAD=./hook.so ls` can therefore render as an ordinary low-risk listing with no preview, even though the prefix changes execution behavior, so keep a redacted preview or fail closed for prefixed commands.
  - confidence: 0.94
- **[P1] Surface text-processor input files:** `src/infra/plugin-approvals.ts:2057-2063`
  - body: The text-processor branches classify `jq`, `awk`, `cut`, `sort`, `tr`, and `uniq` as low-risk formatting while target extraction returns no operands for them. Commands like `jq . .env` or `sort .env` can read credential files without the file appearing once `simple` hides raw details, so include operands and raise or fail closed for sensitive/system paths.
  - confidence: 0.91
