## Summary

Keep open: the feature direction is useful, but the current patch is not merge-ready because simple-mode approval text can still hide execution-changing shell context, and the PR head currently reports a dirty merge state.

## What This Changes

The PR adds configurable plain-language plugin approval rendering, exposes `approvals.plugin.language`, and wires the renderer through plugin approval forwarding plus Telegram and Matrix fallback payloads.

## Best Possible Solution

Keep the config-gated plain-language renderer with `original` as the default, but fail closed or show a redacted preview whenever shell prefixes or input sources affect execution before hiding raw details.

## Reproduction Assessment

Yes for the review findings: PR-head source inspection shows the simple renderer hides raw technical fields while command resolution strips env assignment prefixes and leading input redirections. I did not run tests because this was a read-only review.

## Solution Assessment

No, not as-is. The UX and config shape are maintainable, but the renderer needs conservative handling for these shell forms before simple mode can safely omit raw command details.

## Review Findings

Overall correctness: patch is incorrect

- **[P1] Fail closed on env assignment prefixes:** `src/infra/plugin-approvals.ts:1796-1798`
