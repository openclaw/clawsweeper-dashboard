## Summary

Keep open. Current main does not have the new config-gated plain-language plugin approval mode, and the PR is useful, but current head still has source-visible safety blockers before merge.

## What This Changes

The PR adds `approvals.plugin.language` for plain-language plugin approval prompts and wires it through config, SDK reply rendering, fallback forwarding, Telegram, Matrix, docs, and tests.

## Best Possible Solution

Keep the config-gated plain-language renderer, but make shell forms that alter execution or hide input targets visible or fail closed before `simple` mode omits raw details.

## Reproduction Assessment

Yes for the review findings: PR-head source shows leading `KEY=value` prefixes and leading redirection words are stripped before simple-mode summaries can hide raw technical details. I did not execute tests in this read-only pass.

## Solution Assessment

No, not as-is. The UX direction is maintainable, but the renderer should conservatively summarize or preview these shell forms before raw technical text is hidden.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Fail closed on env assignment prefixes:** `src/infra/plugin-approvals.ts:1796-1798`
  - body: `resolveCommandWords()` still drops leading `KEY=value` prefixes before classification, and `simple` mode can then omit the raw command. Prefixes such as `PATH=./bin ls` or `LD_PRELOAD=./hook.so ls` can change command lookup/loading while the user sees a low-risk known-command summary, so fail closed or force a redacted preview for these prefixes.
  - confidence: 0.92
- **[P2] Surface leading stdin redirections:** `src/infra/plugin-approvals.ts:1753`
  - body: `stripLeadingShellRedirectionWords()` removes leading input redirection targets before command-specific summaries, but non-network read commands do not carry those stdin targets into the visible simple summary. In simple mode, `< .env cat` is reduced to a low-risk `cat` read with no `.env` target or preview, so carry stdin targets into read summaries or fail closed/show a preview.
  - confidence: 0.88
