## Summary

Keep this PR open. The plain-language approval path is useful and the author has addressed many prior parser-safety findings, but the latest head still has one blocking approval-safety bug: shell wrappers using positional parameters can hide the real argv values while `simple` mode suppresses raw technical details.

## What This Changes

The PR adds configurable plain-language plugin approval rendering and wires it through config, shared approval payloads, Matrix, Telegram, docs, and tests.

## Best Possible Solution

Keep the shared renderer/config approach, but make shell-wrapper summaries preserve positional argv or fail closed with a command preview before `simple` mode hides raw details.

## Reproduction Assessment

Yes. PR-head source inspection gives a high-confidence path with `bash -c 'rm -rf "$1"' sh /tmp/x`: the wrapper extractor returns only the body, the delete summary does not force a preview, and simple mode omits raw details.

## Solution Assessment

No, not yet. The shared renderer/config direction is maintainable, but shell-wrapper positional arguments must stay visible or conservatively fail closed before this can safely suppress technical fields.
