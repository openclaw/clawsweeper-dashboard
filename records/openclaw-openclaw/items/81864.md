## Summary

Keep this PR open: current head still has concrete approval-safety defects where `simple` mode can remove raw command details while hiding security-relevant command context. The prior discussion and supplied proof are useful, but these are merge-blocking renderer issues rather than a reason to close the PR.

## What This Changes

The PR adds `approvals.plugin.language` and a plain-language plugin approval renderer wired through shared, Telegram, Matrix, and forwarding paths.

## Best Possible Solution

Keep the config-gated plain-language approval mode, but make `simple` output preserve or preview every hidden medium/high-risk command detail before raw technical fields are suppressed.

## Reproduction Assessment

Yes. Source inspection of PR head shows `pwd && ls && ps && cat notes.txt && rm -rf /tmp/x` can hide the concrete delete target after the four-action cap, and `bash -c 'rm -rf "$1"' sh /tmp/x` can hide `/tmp/x` while `simple` mode suppresses raw command details.

## Solution Assessment

No, not as-is. The shared config-gated renderer is the right direction, but `simple` mode needs to prioritize or preview hidden risky summaries and fail closed when shell-wrapper positional arguments affect the command body.
