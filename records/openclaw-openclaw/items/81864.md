## Summary

Keep open: the default-preserving approval-language feature is useful, but current head still has security-sensitive understatement cases where simple mode hides raw command details.

## What This Changes

The branch adds an opt-in `approvals.plugin.language` setting, plain-English plugin approval rendering, channel/forwarder wiring, docs/config updates, and renderer regressions.

## Best Possible Solution

Land the opt-in approval-language feature after the simple renderer fails closed for executable sourced scripts and read/search commands that write files.

## Reproduction Assessment

Yes for the PR-introduced review findings by source inspection. `source ./setup.sh`, `cat notes.txt > out`, and `sed -i ... file` still reach known-command summaries that can hide the raw command while understating execution or writes.

## Solution Assessment

No as currently proposed. The config and channel wiring are maintainable, but the command summarizer needs conservative handling for these forms before `simple` mode hides technical details.
