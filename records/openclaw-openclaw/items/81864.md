## Summary

Keep open: the feature is active, but current PR head still has a security-sensitive approval-summary bug where env-prefixed commands can be classified too safely while `simple` mode hides the raw command, and current-head Telegram-visible proof remains incomplete.

## What This Changes

Adds `approvals.plugin.language` with `simple` and `simple-technical` plugin approval rendering, wires it through shared fallback, Telegram, and Matrix paths, and updates config docs/schema/tests.

## Best Possible Solution

Keep the plain-language approval feature path, but make env assignments fail closed or force a redacted command preview before hiding raw technical text, then refresh proof through the affected Telegram approval path.

## Reproduction Assessment

Yes. Source inspection of PR head shows `KEY=value` prefixes are stripped before command classification, and `simple` mode can then omit the raw command entirely.

## Solution Assessment

No. The direction is maintainable, but the implementation needs conservative handling for env-prefixed commands and current-head Telegram-visible proof before merge.
