## Summary

Keep open; this PR is not ready to merge. The feature boundary and proof look reasonable, but source inspection shows `pnpm -w install` is still parsed as if `install` were the value for `-w`, so simple mode can hide a dependency install behind a medium-risk package-manager summary.

## What This Changes

This PR adds opt-in `approvals.plugin.language` rendering, shared simple/simple-technical plugin approval copy, Telegram/Matrix/fallback wiring, config/docs/baseline updates, and focused approval tests.

## Best Possible Solution

Land the opt-in shared renderer after making package-manager option parsing command-aware, adding a `pnpm -w install` regression, and preserving `original` as the default.

## Reproduction Assessment

Yes, by source inspection. `pnpm -w install` reaches `parsePackageManagerSubcommand()`, where exact `-w` is treated as value-taking and consumes `install`, leaving simple mode with a generic medium-risk package-manager summary.

## Solution Assessment

No for the latest patch as-is. The opt-in renderer and shared channel boundary are reasonable, but the parser must handle `-w` according to the selected package manager before raw command details are hidden.

## Review Findings
