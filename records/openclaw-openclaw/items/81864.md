## Summary

Keep open: the plain-language approval direction is useful, but current head still has approval-safety regressions where simple mode can hide command context that changes execution or reads credential files.

## What This Changes

The PR adds a configurable `approvals.plugin.language` setting, plain-language plugin approval rendering, channel wiring for Telegram/Matrix fallback text, docs, generated baselines, and focused renderer tests.

## Best Possible Solution

Keep `original` as the default and land the new plain-language modes only after simple mode conservatively preserves or fails closed on env prefixes and text-processor file operands.

## Reproduction Assessment

Yes. Source inspection of PR head gives high-confidence reproductions: `LD_PRELOAD=./hook.so ls` loses the prefix before classification, and `jq . .env` is treated as low-risk formatting while simple mode hides raw technical details.

## Solution Assessment

No, not as-is. The config-gated renderer approach is maintainable, but it must show a redacted preview or fail closed before suppressing raw command details for these cases.

## Review Findings

Prefixes such as `PATH=./bin ls`, `LD_PRELOAD=./hook.so ls`, or credential-bearing env vars can materially change a known command while rendering as an ordinary low-risk action, so keep a redacted preview or fail closed for prefixed commands.

Commands like `jq . .env` or `sort .env` can read credential files without the file appearing in the prompt once `simple` hides raw details, so include operands and raise or fail closed for sensitive/system paths.
