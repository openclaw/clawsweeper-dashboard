## Summary

Keep this PR open: the UX direction is useful and the default remains unchanged, but the current simple-mode renderer can still hide behavior-changing command context while suppressing the raw technical block.

## What This Changes

The PR adds `approvals.plugin.language` modes, a plain-language command summarizer for plugin approval prompts, config/docs/test updates, and Telegram/Matrix/forwarder wiring for the new prompt language.

## Best Possible Solution

Keep the config-gated plain-language renderer with `original` as the default, but make simple mode fail closed or show a redacted preview whenever env prefixes or text-processor operands could alter or hide command semantics.

## Reproduction Assessment

Yes. A focused source reproduction is a simple plugin approval for `LD_PRELOAD=./hook.so ls` or `jq . .env`; PR-head source strips the env prefix or drops text-processor operands while simple mode hides the raw technical details.

## Solution Assessment

No, not as-is. The config-gated UX direction is maintainable, but the summarizer must conservatively preserve hidden command context before the raw command is removed from the prompt.

  - confidence: 0.9
