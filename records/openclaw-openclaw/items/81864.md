## Summary

Keep open; current main still uses the raw plugin-approval text, so the PR is not obsolete, but the latest head still has approval-safety defects where `simple` mode can hide destructive targets while suppressing raw technical detail.

## What This Changes

The PR adds `approvals.plugin.language`, a plain-language command approval renderer, config/docs/baseline updates, Telegram/Matrix fallback wiring, and incidental merge-conflict cleanup in adjacent plugin files.

## Best Possible Solution

Keep the config-gated plain-language mode, but make `simple` approval text preserve or preview every hidden medium/high-risk detail before raw command details are suppressed.

## Reproduction Assessment

Yes. Source inspection against PR head shows concrete paths for both defects: a multi-step command can put a delete action after the four-action cap, and a `bash -c` body can reference positional wrapper arguments that the renderer drops.

## Solution Assessment

No, not as-is. The shared config-gated renderer is the right direction, but simple mode must prioritize or preview hidden risky summaries and fail closed when shell-wrapper positional arguments affect the command body.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Show high-risk actions before truncating:** `src/infra/plugin-approvals.ts:277-281`
  - body: `simple` mode removes raw technical details, but this loop always emits only the first four action strings. A command like `pwd && ls && ps && cat notes.txt && rm -rf /tmp/x` can show high risk and a delete reason while hiding the concrete delete target behind `1 more background step(s)` with no command preview, so risky summaries need priority or a forced preview before truncation.
  - confidence: 0.92
- **[P2] Fail closed on positional shell wrappers:** `src/infra/plugin-approvals.ts:734`
  - body: `extractShellWrapperCommand` returns only the `-c` body and drops later shell-wrapper arguments. For `bash -c 'rm -rf "$1"' sh /tmp/x`, simple mode can summarize deletion of `$1` while hiding `/tmp/x` and the raw command, so wrappers whose bodies reference positional parameters need to keep a redacted preview or fail closed.
  - confidence: 0.87
