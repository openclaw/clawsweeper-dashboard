## Summary

Keep open: the feature direction and real-behavior proof are useful, but the PR still has security-relevant simple-mode approval gaps where raw command details are hidden before the summary preserves all behavior-changing context.

## What This Changes

The PR adds `approvals.plugin.language` with `original`, `simple`, and `simple-technical` plugin approval prompt modes, wires it through shared forwarding plus Telegram/Matrix fallback rendering, and updates docs, schemas, baselines, and tests.

## Best Possible Solution

Keep the config-gated renderer with `original` as the default, but make simple-mode summaries fail closed or show a redacted preview whenever env prefixes or text-processor file operands could change the hidden raw command semantics.

## Reproduction Assessment

Yes. A focused source reproduction is a simple plugin approval for `LD_PRELOAD=./hook.so ls` or `jq . .env`; the PR-head source strips the env prefix or ignores the text-processor file target while simple mode hides the raw technical details.

## Solution Assessment

No, not as-is. The UX direction is maintainable, but the summarizer must preserve or conservatively fail closed on behavior-changing command context before raw details are removed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve env prefixes before hiding commands:** `src/infra/plugin-approvals.ts:1796-1798`
  - body: `resolveCommandWords()` still strips leading `KEY=value` tokens before classifying the command, and `simple` mode then omits the raw technical block. Prefixes such as `PATH=./bin ls`, `LD_PRELOAD=./hook.so ls`, or credential-bearing env vars can materially change a known command while rendering as an ordinary low-risk action, so surface a redacted preview or fail closed for prefixed commands.
  - confidence: 0.93
- **[P1] Surface text-processor file operands:** `src/infra/plugin-approvals.ts:1214-1230`
  - body: The `jq` and text-processing cases summarize non-redirected commands as low-risk formatting without using `targetText`, `sensitiveTarget`, or a preview. Commands like `jq . .env` or `sort .env` can read credential files while the simple prompt only says it will filter/process output, so include file operands and raise/fail closed for sensitive or system paths before hiding the raw command.
  - confidence: 0.86
