## Summary

This PR is useful but still blocked: the new `simple` approval mode can hide security-relevant command context while suppressing the raw technical details that would otherwise let an approver catch it.

## What This Changes

The PR adds an `approvals.plugin.language` config option with `original`, `simple`, and `simple-technical` plugin approval rendering, wired through shared reply rendering, Telegram/Matrix fallback delivery, docs, schema help, and regression tests.

## Best Possible Solution

Keep the config-gated plain-language approval mode, but make simple output preserve or preview every hidden medium/high-risk command detail before raw technical fields are suppressed.

## Reproduction Assessment

Yes. Source inspection of PR head shows `pwd && ls && ps && cat notes.txt && rm -rf /tmp/x` can hide the concrete delete target after the four-action cap, and `bash -c 'rm -rf "$1"' sh /tmp/x` can hide `/tmp/x` after shell-wrapper unwrapping.

## Solution Assessment

No, not as-is. The shared configurable renderer is the right direction, but simple mode must prioritize or preview hidden risky summaries and preserve shell-wrapper positional arguments before merge.

## Review Findings

  - confidence: 0.88
