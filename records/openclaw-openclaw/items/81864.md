## Summary

Keep open: the feature is useful and has real behavior proof, but the PR head still has approval-renderer security defects where `simple` mode can hide raw technical context while presenting low-risk summaries.

## What This Changes

The PR adds `approvals.plugin.language` with `original`, `simple`, and `simple-technical` plugin approval wording across config, the shared approval renderer, Telegram/Matrix fallback rendering, docs, and tests.

## Best Possible Solution

Keep the raw format as the default and land the plain-language modes only after simple mode preserves, previews, or fails closed on environment prefixes, text-processor file operands, and input redirection targets.

## Reproduction Assessment

Yes for the review findings: source inspection of PR head shows representative commands like `LD_PRELOAD=./hook.so ls`, `jq . .env`, and `< .env cat` can lose important context before `simple` mode hides the raw details. I did not run tests because this was a read-only review.

## Solution Assessment

No, not as-is. The config-gated renderer with unchanged default is maintainable, but simple mode must fail closed or show a redacted preview when parsing drops execution context or input files.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve env prefixes before hiding commands:** `src/infra/plugin-approvals.ts:1796-1798`
  - body: `resolveCommandWords()` still drops leading `KEY=value` assignments before classifying the command, and `simple` mode then returns without the raw technical block. A command like `LD_PRELOAD=./hook.so ls` can therefore render as an ordinary low-risk listing with no preview even though the prefix changes execution behavior; keep a redacted preview or fail closed for prefixed commands.
  - confidence: 0.94
- **[P1] Surface text-processor input files:** `src/infra/plugin-approvals.ts:2057-2063`
  - body: Target extraction returns no operands for `jq`, `awk`, `cut`, `sort`, `tr`, and `uniq`, while those command branches classify unredirected use as low-risk formatting. Commands like `jq . .env` or `sort .env` can read credential files without the file appearing once `simple` hides raw details, so include operands and raise or fail closed for sensitive/system paths.
  - confidence: 0.92
- **[P1] Preserve leading input redirects before summarizing:** `src/infra/plugin-approvals.ts:1753`
  - body: The generic resolver strips leading input redirections before command classification, but only the network-transfer path later inspects input redirection targets. Forms like `< .env cat` can render as a low-risk read with no file target or preview, so generic summaries need to preserve those targets or fail closed before simple mode suppresses the raw command.
  - confidence: 0.88
