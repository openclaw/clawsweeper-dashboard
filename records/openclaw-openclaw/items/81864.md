## Summary

Keep this PR open: the feature is active and useful, but PR-head source inspection still shows two approval-renderer safety gaps where simple mode can hide command context after removing the raw technical block.

## What This Changes

The PR adds `approvals.plugin.language` with `original`, `simple`, and `simple-technical` plugin approval rendering, wires it through shared forwarding plus Telegram/Matrix fallback payloads, and updates config docs/tests/baselines.

## Best Possible Solution

Keep the config-gated plain-language approval renderer with `original` as the default, but make simple-mode command summaries fail closed or show redacted previews whenever shell prefixes or input sources alter execution.

## Reproduction Assessment

Yes for the review findings: PR-head source inspection shows simple mode hides raw details while the resolver strips leading env assignments and input redirections before known-command summaries. I did not run tests because this was a read-only review.

## Solution Assessment

No, not as-is. The UX/config shape is maintainable, but the renderer needs conservative handling for hidden shell context before simple mode can safely omit raw technical details.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Fail closed on env assignment prefixes:** `src/infra/plugin-approvals.ts:1796-1798`
  - body: `resolveCommandWords()` still drops leading `KEY=value` words before classification, and `simple` mode returns before raw technical details are appended. Prefixes such as `PATH=./bin ls` or `LD_PRELOAD=./hook.so ls` can change command lookup or dynamic loading while rendering as an ordinary known command, so surface the prefix or fail closed with a preview.
  - confidence: 0.92
- **[P2] Surface leading input redirections:** `src/infra/plugin-approvals.ts:1753`
  - body: Leading redirections are stripped before the non-network command summaries, but read commands do not carry that source into the visible simple text. For example, `< .env cat` can become a low-risk `read file contents` summary with no `.env` source or preview while raw details are hidden, so include the input source or fail closed.
  - confidence: 0.9
