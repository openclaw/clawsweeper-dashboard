## Summary

Keep open. Current main still has only the raw plugin approval text, and this PR is the active implementation path, but PR head still has approval-safety gaps where simple mode can hide shell behavior that was previously visible in raw technical text.

## What This Changes

The PR adds a config-gated plain-language plugin approval renderer, wires it through fallback, Telegram, and Matrix approval paths, and updates config docs, baselines, and focused tests.

## Best Possible Solution

Keep the config-gated plain-language renderer direction, but make shell forms that alter execution or hide input targets fail closed or show a redacted preview before simple mode omits raw details.

## Reproduction Assessment

Yes for the review findings: source inspection of PR head shows leading KEY=value prefixes and leading stdin redirections are removed before simple mode hides raw details. I did not execute tests in this read-only pass.

## Solution Assessment

No, not as-is. The UX direction is maintainable, but simple mode should not suppress raw technical text until these command forms are represented conservatively or forced to show a redacted preview.

  - confidence: 0.87
