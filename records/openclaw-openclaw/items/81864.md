## Summary

Keep open: the feature is plausible, but this PR still has an approval-safety regression in simple mode and needs fresh current-head Telegram-visible proof before merge.

## What This Changes

The PR adds configurable `approvals.plugin.language` rendering for plain-language plugin approvals, wires it through shared forwarding plus Telegram/Matrix fallback payloads, and updates config docs/schema/tests and generated baselines.

## Best Possible Solution

Keep the plain-language approval direction, but make env-prefixed command summaries fail closed or include a redacted preview before hiding raw text, then refresh current-head Telegram proof.

## Reproduction Assessment

Yes. Source inspection of PR head shows leading `KEY=value` words are stripped before classification while `simple` mode can return without raw technical text, so the approval-safety issue is reproducible from the code path.

## Solution Assessment

No. The feature shape is maintainable, but the current implementation should conservatively treat env-prefixed execution as high-risk or show a redacted command preview before suppressing the raw command.
