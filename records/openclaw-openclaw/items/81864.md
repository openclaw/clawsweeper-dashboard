## Summary

This PR should stay open because the new `simple` approval mode still has a safety hole: single unknown commands can hide their arguments and raw command text while showing only a medium-risk generic summary. The feature direction is useful and the proof is substantially better now, but this blocker needs a focused fix before merge.

## What This Changes

The PR adds `approvals.plugin.language` plus a plain-language plugin approval renderer and wires it through shared forwarding, Telegram, Matrix, config help, docs, and tests.

## Best Possible Solution

Keep the new language option opt-in, but require the simple renderer to expose a redacted command preview whenever it cannot fully summarize a command before hiding technical details.

## Reproduction Assessment

Yes. Source inspection of the PR head shows the single-command unknown fallback lacks `showCommandPreview`, while `simple` mode returns before technical details are appended.

## Solution Assessment

No. The broad approach is plausible, but the current implementation is not the best safe merge shape until unknown commands still show a redacted preview and the branch is refreshed against main.
