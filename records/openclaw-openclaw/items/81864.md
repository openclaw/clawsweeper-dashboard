## Summary

Keep open: the plain-language approval direction is useful, but the current PR head can still hide security-relevant shell syntax when `simple` mode suppresses the raw technical block.

## What This Changes

The PR adds `approvals.plugin.language`, a plain-language plugin approval renderer, channel fallback wiring for Telegram/Matrix/shared forwarding, docs/tests/generated baseline updates, plus ancillary dependency and refresh churn.

## Best Possible Solution

Keep the config-gated plain-language renderer, but make shell forms that alter execution or hide input targets visible or fail closed before `simple` mode omits raw details.

## Reproduction Assessment

Yes for the review findings: PR-head source shows leading `KEY=value` prefixes and leading input redirections are stripped before simple-mode summaries are emitted. I did not execute tests in this read-only pass.

## Solution Assessment

No, not as-is. The UX direction is maintainable, but the renderer should conservatively summarize or preview these shell forms before raw technical text is hidden.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Fail closed on leading environment assignments:** `src/infra/plugin-approvals.ts:1796-1798`
  - body: `resolveCommandWords()` still drops leading `KEY=value` prefixes before classification, and `simple` mode can return without raw technical details. Prefixes such as `PATH=./bin ls` or `LD_PRELOAD=./hook.so ls` can change command lookup/loading while the user sees a low-risk known-command summary, so fail closed or force a redacted preview for these prefixes.
  - confidence: 0.92
- **[P2] Preserve leading stdin redirection targets:** `src/infra/plugin-approvals.ts:1753`
  - body: `stripLeadingShellRedirectionWords()` removes leading `< file` targets before command-specific summaries, but non-network read commands do not carry stdin targets into the visible simple summary. In simple mode, `< .env cat` is reduced to a low-risk `cat` read with no `.env` target or preview, so carry stdin targets into read summaries or fail closed/show a preview.
  - confidence: 0.87
