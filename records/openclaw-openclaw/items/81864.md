## Summary

Keep open: the shared renderer/config direction is useful and CI/proof look healthy, but the PR still has a security-sensitive simple-mode parser gap where implicit curl/wget downloads can create local files while the approval stays a generic medium-risk network prompt.

## What This Changes

The PR adds opt-in `approvals.plugin.language` rendering for plugin approvals, wires it through forwarding plus Telegram/Matrix payloads, updates config/docs/baselines/tests, and carries several main-refresh test/hardening fixes.

## Best Possible Solution

Keep the centralized renderer/config approach, but classify implicit network-output downloads or fail closed with a redacted preview before simple mode hides raw command text.

## Solution Assessment
