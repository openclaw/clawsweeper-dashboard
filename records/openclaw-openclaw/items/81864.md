## Summary

Keep open: the opt-in renderer direction is plausible, but the current branch still hides security-relevant command content in simple mode, so it should not merge yet.

## What This Changes

The branch adds opt-in `approvals.plugin.language` rendering for plain-language plugin approval prompts across shared, forwarded, Telegram, and Matrix paths, updates config/docs/baseline checks, and includes unrelated oc-path formatting churn.

## Best Possible Solution

Land the opt-in simple approval mode only after shell-control bodies and command targets are summarized conservatively before raw command text is hidden, with the unrelated formatting churn removed or split out.

## Reproduction Assessment

Yes for the PR findings: PR-head source inspection shows `if`/`while` bodies are dropped and global option parsing can skip sensitive command targets. This is not a current-main bug reproduction because the PR adds a new opt-in mode.

## Solution Assessment

No as proposed. The default-preserving config direction is maintainable, but the simple renderer must fail closed on conditional bodies and ambiguous option parsing before it is safe to hide raw technical commands.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Summarize conditional bodies before hiding raw commands:** `src/infra/plugin-approvals.ts:333-336`
  - body: Simple mode hides the raw command, so the shell-control parser cannot drop everything after `then` or `do`. A command like `if [ -f ok ]; then rm -rf data; fi` would be summarized as only a low-risk file check while the delete step disappears; parse the body/else branches or mark the whole conditional high-risk.
  - confidence: 0.91
- **[P2] Parse command flags per command:** `src/infra/plugin-approvals.ts:1169-1197`
  - body: `getCommandTargetArgs` treats `-n`, `-f`, and similar flags as value-taking for every command. For commands where those are boolean, such as `cat -n .env`, the sensitive path is skipped before target and risk checks, leaving a low-risk generic read summary while simple mode hides the raw command.
  - confidence: 0.86
