## Summary

Keep open: the feature direction, CI, and supplied real-behavior proof are good, but the simple renderer still hides arbitrary package-manager exec/dlx command text behind a medium-risk summary, which is too weak for an approval prompt.

## What This Changes

The PR adds configurable plugin approval language modes, wires simple/plain-English rendering through config, plugin SDK reply payloads, Telegram/Matrix/fallback delivery, docs, generated baselines, and tests.

## Best Possible Solution

Land the configurable plain-language approval modes after package-manager exec/dlx subcommands fail closed with a redacted preview and the unrelated dependency drift is removed.

## Reproduction Assessment

Yes for the review finding: source inspection shows simple mode hides raw details, while package-manager exec/dlx commands are summarized as medium-risk package actions without a command preview. I did not execute the renderer because the checkout is read-only.

## Solution Assessment

No, not as-is. The default-preserving config-gated approach is maintainable, but package-manager exec/dlx forms need to be treated as opaque/high-risk or parsed conservatively before simple mode hides the raw command.
