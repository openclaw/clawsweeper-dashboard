## Summary

Keep this PR open, but it still needs changes: the current branch fixes the earlier conditional/pipeline visibility concerns and has real behavior proof, yet simple mode can still hide sensitive file targets because target parsing treats flags globally across commands.

## What This Changes

The PR adds an opt-in `approvals.plugin.language` config for plain-language plugin approval prompts, wires it through shared/forwarded Telegram and Matrix approval rendering, and updates config/docs/tests.

## Best Possible Solution

Keep the default-preserving plain-language approval direction, but land it only after command target extraction is command-aware or fail-closed and unrelated formatting churn is split out or removed.

## Reproduction Assessment

Yes for the review finding: PR-head source shows `getCommandTargetArgs` applies one global option-value table, so a command like `cat -n .env` can skip `.env` before sensitive-target checks. This is not a current-main bug because the PR adds a new opt-in simple mode.

## Solution Assessment

No as proposed. The config and renderer direction is maintainable, but simple mode should not hide raw command text until target parsing is command-specific or conservatively high-risk for ambiguous flags.

## Review Findings

Overall correctness: patch is incorrect
