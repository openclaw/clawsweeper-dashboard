## Summary

Keep open: the feature is useful and real behavior proof is present, but `simple` mode still hides raw technical details while the command summarizer omits security-relevant environment prefixes and text-processor file operands.

## What This Changes

The PR adds an `approvals.plugin.language` config option, a plain-English plugin approval renderer, Telegram/Matrix/fallback wiring, docs/config schema updates, generated baselines, and focused renderer tests.

## Best Possible Solution

Keep `original` as the default and land the new plain-language modes only after simple mode conservatively preserves, previews, or fails closed on env prefixes and text-processor file operands.

## Reproduction Assessment

Yes for the blocking findings: source inspection at PR head shows `resolveCommandWords()` stripping leading `KEY=value` before classification and `getCommandTargetArgs()` dropping text-processor operands. Representative repro commands are `LD_PRELOAD=./hook.so ls` and `jq . .env` in `simple` mode.

## Solution Assessment

No, not as-is. The config-gated renderer and unchanged default are maintainable, but simple mode must fail closed or show a redacted preview before hiding raw details for these command shapes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve env prefixes before hiding commands:** `src/infra/plugin-approvals.ts:1796-1798`
  - body: `resolveCommandWords()` drops every leading `KEY=value` token before classifying the command, and `simple` mode returns before the technical block. A command like `LD_PRELOAD=./hook.so ls` can therefore render as an ordinary low-risk listing with no preview, even though the prefix changes execution behavior, so keep a redacted preview or fail closed for prefixed commands.
  - confidence: 0.94
- **[P1] Surface text-processor input files:** `src/infra/plugin-approvals.ts:1211-1218`
  - body: The `jq` and text-processor branches force low-risk formatting while target extraction returns no operands for `jq`, `awk`, `cut`, `sort`, `tr`, and `uniq`. Commands like `jq . .env` or `sort .env` can read credential files without the file appearing once `simple` hides raw details, so include operands and raise or fail closed for sensitive/system paths.
  - confidence: 0.9
