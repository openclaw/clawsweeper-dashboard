## Summary

Keep this PR open: the plain-language approval direction is useful and the supplied proof is sufficient, but the current head still has a blocking approval-safety gap for shell-capable git forms hidden by `simple` mode.

## What This Changes

This PR adds `approvals.plugin.language` and a plain-language plugin approval renderer, then wires it through shared approval forwarding plus Telegram and Matrix fallback payloads.

## Best Possible Solution

Keep the opt-in shared renderer/config approach, but make shell-capable or opaque git forms fail closed or show a redacted command preview before hiding raw technical fields.

## Reproduction Assessment

Yes for the blocking finding: source inspection of PR head shows unknown git subcommands use the medium-risk no-preview fallback, and Git documents `submodule foreach` as shell execution. The feature itself is not a bug reproduction case.

## Solution Assessment

No, not yet. The opt-in renderer/config direction is maintainable, but simple mode must preserve command context or fail closed for shell-capable git commands before merge.
