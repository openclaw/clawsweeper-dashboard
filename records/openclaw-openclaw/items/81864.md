## Summary

Keep open: the feature is not on current main and the PR has useful proof, but it is not safe to merge while simple-mode approval text can still hide leading environment assignments that change command execution before raw details are suppressed.

## What This Changes

Adds a config-gated `approvals.plugin.language` option and plain-language plugin approval renderer, wires it into forwarding plus Telegram/Matrix fallback payloads, updates docs/config/tests, and adjusts the Telegram type dev dependency.

## Best Possible Solution

Keep the config-gated plain-language approval renderer with `original` as the default, but make simple mode fail closed or show a redacted preview whenever environment assignments can alter command lookup, loading, credentials, or runtime behavior.

## Reproduction Assessment

Yes for the blocking review finding: PR-head source inspection shows `resolveCommandWords()` strips leading env assignments, while simple mode returns before raw technical details are appended. I did not execute the renderer because this review was read-only.

## Solution Assessment

No, not as-is. The UX/config direction is plausible, but the approval summarizer must preserve or fail closed on env-prefix context before it can safely hide raw technical text.
