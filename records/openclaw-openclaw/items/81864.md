## Summary

Keep open: current main still lacks the plain-language plugin approval path, but PR head still has a blocking approval-safety issue because simple mode can truncate away a later high-risk action while hiding raw command details.

## What This Changes

The PR adds an `approvals.plugin.language` setting and plain-language plugin approval rendering across the shared renderer, Telegram/Matrix fallbacks, docs/config schema, and tests.

## Best Possible Solution

Keep the config-gated plain-language approval direction, but make simple mode preserve or preview every medium/high-risk action before it omits raw technical details.

## Reproduction Assessment

Yes for the blocking finding by source inspection: a command such as `pwd && ls && ps && cat notes.txt && rm -rf /tmp/x` reaches first-four-action truncation while the later delete action has no command preview. I did not execute the renderer in this read-only review.

## Solution Assessment

No, not as-is. The shared config-gated renderer is a maintainable direction, but simple mode must prioritize high-risk summaries or force a redacted preview whenever truncation hides them.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Surface hidden high-risk approval steps before truncating:** `src/infra/plugin-approvals.ts:277-281`
  - body: `simple` mode hides raw command details, but this loop renders only the first four action summaries and collapses the rest. A command like `pwd && ls && ps && cat notes.txt && rm -rf /tmp/x` can show high risk while replacing the actual delete target with `1 more background step(s)`, and `rm` does not force a command preview. Prioritize dangerous/preview-worthy summaries before truncating, or force a redacted preview whenever truncation hides any medium/high-risk action.
  - confidence: 0.9
