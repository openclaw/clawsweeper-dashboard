## Summary

Keep this PR open: the feature is useful and actively maintained, but the current head still has a blocking approval-safety issue where simple mode can hide a later high-risk action after truncating the action list while also hiding the raw command.

## What This Changes

The branch adds configurable plain-language plugin approval rendering and wires it through shared approval payloads, Telegram/Matrix fallbacks, config docs/baselines, and tests, with incidental adjacent cleanup.

## Best Possible Solution

Keep the config-gated plain-language approval direction, but require simple mode to surface or preview any high-risk action before it hides raw technical details.

## Reproduction Assessment

Yes. Source inspection of PR head shows a command like `pwd && ls && ps && cat notes.txt && rm -rf /tmp/x` reaches the first-four-action truncation while the later delete target is replaced by a generic background-step line.

## Solution Assessment

No, not as-is. The shared config-gated renderer is a maintainable direction, but simple mode must prioritize high-risk or preview-worthy summaries, or force a redacted command preview whenever truncation hides them.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Surface hidden high-risk approval steps before truncating:** `src/infra/plugin-approvals.ts:277-281`
  - body: `simple` mode hides raw command details, but this loop renders only the first four action summaries and collapses the rest. A command like `pwd && ls && ps && cat notes.txt && rm -rf /tmp/x` can show `Risk: High` while replacing the actual delete target with `1 more background step(s)`, and the `rm` summary does not force a command preview. Prioritize dangerous/preview-worthy summaries before truncating, or force a redacted preview whenever truncation hides any medium/high-risk action.
  - confidence: 0.9
