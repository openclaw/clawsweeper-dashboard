## Summary

Keep open: the PR still has a concrete simple-mode approval safety bug. Current head drops shell wrapper argv after `-c`, so an approval such as `bash -c 'rm -rf "$1"' sh /tmp/x` can hide the real `/tmp/x` target while `simple` mode suppresses raw technical details.

## What This Changes

The PR adds configurable plain-language plugin approval rendering and wires it through shared plugin approval forwarding plus Telegram and Matrix fallback text, with config docs and tests.

## Best Possible Solution

Keep the configurable plain-language renderer, but make shell-wrapper summaries preserve positional argv or fail closed with a preview before `simple` mode hides raw approval details.

## Reproduction Assessment

Yes. PR-head source inspection gives a high-confidence path with `bash -c 'rm -rf "$1"' sh /tmp/x`: the wrapper extractor returns only the body, the delete summary does not force a preview, and simple mode omits raw details.

## Solution Assessment

No, not yet. The shared renderer/config approach is maintainable, but shell-wrapper positional arguments must stay visible or conservatively fail closed before this can safely suppress technical fields.
