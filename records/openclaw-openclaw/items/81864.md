The PR adds an opt-in `approvals.plugin.language` setting with plain-language plugin approval rendering, Telegram/Matrix/shared fallback wiring, docs/config/tests/generated baselines, plus package and unrelated formatting churn.

## Best Possible Solution

Land the opt-in approval language mode only after the curl `--url` classification gap is fixed, CI is green, and maintainer UX/security review accepts the remaining broad summarizer surface.

## Reproduction Assessment

## Solution Assessment

No, not as-is: the shared renderer and opt-in config direction are maintainable, but simple mode must fail closed for curl `--url` operands before raw technical command text is hidden.

## Review Findings

Full review comments:

- **[P1] Handle curl --url operands in the network summarizer:** `src/infra/plugin-approvals.ts:2166-2169`

  - confidence: 0.9
