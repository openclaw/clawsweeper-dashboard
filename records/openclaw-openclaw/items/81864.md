## Summary

Keep this PR open. The plain-language approval direction is useful and the proof gate is satisfied, but simple mode still has approval-safety blockers: command previews can expose curl OAuth bearer tokens, and curl netrc credential use is hidden behind a generic medium-risk network summary.

## What This Changes

Adds an opt-in `approvals.plugin.language` setting, plain-language plugin approval rendering, Telegram/Matrix/fallback wiring, config docs/schema tests, renderer coverage, and some oc-path formatting churn.

## Best Possible Solution

Keep the opt-in plain-language renderer, but require conservative credential redaction and fail-closed handling for curl credential sources before simple mode can suppress raw technical fields.

## Reproduction Assessment

Yes. Source inspection at PR head plus curl's documented option contracts gives a high-confidence path: render a simple approval for `curl --oauth2-bearer s3cr3t --data-binary @notes.txt https://example.test/upload` or `curl --netrc-file .netrc https://example.test`.

## Solution Assessment

No, not yet. The renderer and config wiring are plausible, but the safe solution must redact bearer-token flags and surface or fail closed on netrc credential use before merge.
