## Summary

Keep this PR open, but it still needs changes before merge: the plain-language renderer can hide execution-changing shell context in `simple` mode while omitting the raw technical approval details.

## What This Changes

The PR adds an `approvals.plugin.language` config option, plain-language plugin approval rendering, channel wiring for Telegram/Matrix/fallback delivery, docs, tests, generated baselines, and a dependency adjustment.

## Best Possible Solution

Keep the config-gated plain-language renderer with `original` as the default, but merge it only after hidden shell prefixes and leading input redirections are surfaced or fail closed with focused regression coverage.

## Reproduction Assessment

Yes for the review findings: PR-head source inspection shows env assignment prefixes and leading input redirections are stripped before `simple` mode hides raw technical details. I did not run tests because this was a read-only review.

## Solution Assessment

No, not as-is. The UX direction is plausible, but the renderer must conservatively handle these shell forms before plain-language mode can suppress raw technical fields.
