## Summary

Keep open. The plain-language approval direction is still useful and not present on current main, but the PR still has an approval-safety blocker: simple mode can hide a later high-risk command action while omitting the raw technical command.

## What This Changes

Adds `approvals.plugin.language` and routes `simple`/`simple-technical` plugin approval text through the shared renderer, Telegram/Matrix fallback paths, config schema/help, docs, and tests.

## Best Possible Solution

Keep the config-gated plain-language approval path, but make simple mode preserve or preview every medium/high-risk action before truncating and add a focused regression.

## Reproduction Assessment

Yes for the blocking finding by source inspection: a command such as `pwd && ls && ps && cat notes.txt && rm -rf /tmp/x` reaches the first-four-action truncation while the later delete target is hidden. I did not execute the renderer in this read-only review.

## Solution Assessment

No, not as-is. The shared config-gated renderer is a maintainable direction, but simple mode must prioritize dangerous summaries or force a redacted preview whenever truncation hides medium/high-risk actions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Surface hidden high-risk approval steps before truncating:** `src/infra/plugin-approvals.ts:277-281`
  - body: `simple` mode removes raw command details, but this loop renders only the first four action summaries and collapses the rest. A command like `pwd && ls && ps && cat notes.txt && rm -rf /tmp/x` can show high risk while replacing the actual delete target with `1 more background step(s)`, and `rm` does not force a command preview. Prioritize dangerous or preview-worthy summaries before truncating, or force a redacted preview whenever truncation hides any medium/high-risk action.
  - confidence: 0.9
