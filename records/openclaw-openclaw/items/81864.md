## Summary

Keep open: this is active feature/config work with solid proof and green CI, but the new simple approval renderer still has a safety-sensitive command-summary gap that should block merge.

## What This Changes

This PR adds an opt-in `approvals.plugin.language` setting, plain-language plugin approval rendering, Telegram/Matrix/fallback wiring, docs/schema updates, focused renderer tests, and related dependency/test-config adjustments.

## Best Possible Solution

Keep the opt-in language mode and unchanged `original` default, but fail closed or show a redacted preview for all `find` predicates that write, execute, or delete before merge.

## Reproduction Assessment

Yes for the PR finding: source inspection at PR head plus `find --help` shows `find . -fprint .env` reaches the read-only fallback even though GNU find writes to the named file. This is a PR-head safety regression, not a current-main bug path.

## Solution Assessment

No, not yet. The opt-in renderer and config shape are maintainable, but simple mode should not suppress raw technical details until `find` output-file predicates are classified as writes or conservatively high-risk with a preview.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Fail closed on find output-file predicates:** `src/infra/plugin-approvals.ts:812-816`
  - body: The `find` summarizer only special-cases `-delete` and `-exec*`, then treats the remaining forms as read-only search/list operations. GNU find also supports `-fprint`, `-fprint0`, `-fls`, and `-fprintf FILE FORMAT`, which create or overwrite the named file; in simple mode a command like `find . -fprint .env` would hide the raw command and present it as search/list activity. Detect these predicates as writes/high risk or fail closed with a command preview.
  - confidence: 0.88
