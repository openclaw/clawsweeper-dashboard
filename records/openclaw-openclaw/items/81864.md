## Summary

Needs changes before merge: `simple` mode still hides raw technical details while dropping environment assignment prefixes and ignoring file operands for several text processors, so approval prompts can understate security-sensitive commands.

## What This Changes

The PR adds configurable plain-language plugin approval prompts and wires them through shared rendering, Telegram, Matrix, config schema/help, docs, and tests.

## Best Possible Solution

Keep `original` as the default and land the plain-language modes only after `simple` mode conservatively surfaces or fails closed on env prefixes and text-processor file operands with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection of PR head gives high-confidence reproductions: `LD_PRELOAD=./hook.so ls` loses the prefix before classification, and `jq . .env` loses the file operand while `simple` mode hides raw technical details.

## Solution Assessment

No, not as-is. The config-gated renderer direction is maintainable, but the summarizer must preserve a redacted preview or fail closed before it suppresses raw command details.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve env prefixes before hiding commands:** `src/infra/plugin-approvals.ts:1796-1798`
  - body: `resolveCommandWords()` drops leading `KEY=value` tokens before classification, and `simple` mode then omits the raw technical block. Prefixes such as `PATH=./bin ls`, `LD_PRELOAD=./hook.so ls`, or credential-bearing env vars can materially change a known command while rendering as an ordinary low-risk action, so surface a redacted preview or fail closed for prefixed commands.
  - confidence: 0.94
- **[P1] Surface text-processor input files:** `src/infra/plugin-approvals.ts:2057-2062`
  - body: `getCommandTargetArgs()` returns no targets for `jq`, `awk`, `cut`, `sort`, `tr`, and `uniq`, and those cases summarize as low-risk formatting while `simple` hides the raw command. Commands like `jq . .env` or `sort .env` can read credential files without the file appearing in the prompt, so include operands and raise or fail closed for sensitive/system paths.
  - confidence: 0.9
