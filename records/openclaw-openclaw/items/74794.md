## Summary

Keep this PR open but do not merge as-is: it addresses a real cron announce mis-routing path, and the production incident comment reinforces the impact, but the patch can persist an inferred target under an incompatible caller-supplied channel and is missing the required changelog entry.

## What This Changes

The PR changes cron add-time delivery inference to populate `delivery.to` for channel-only announce deliveries and adds regression tests in `src/agents/tools/cron-tool.test.ts`.

## Best Possible Solution

Merge a guarded cron fix that infers `to` only from compatible context/session data, preserves explicit `to`, covers mismatch and no-source cases, and adds the active-version changelog entry.

## Reproduction Assessment

Yes by source inspection. Current main skips inference for channel-only announce delivery, and the PR-specific defect follows from `delivery.channel: "discord"` combined with a context/session key that infers a different channel.

## Solution Assessment

No as proposed. Gating inference on explicit `to` is the right direction, but the merge must guard channel compatibility or fail closed before storing `inferred.to` under a caller-supplied channel.
