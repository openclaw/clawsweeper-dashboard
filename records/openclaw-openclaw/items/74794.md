## Summary

Keep open: the production report and current code show a real cron announce routing bug, but this PR can mix a caller-supplied channel with an inferred target from another channel and still lacks the required changelog entry.

## What This Changes

The PR changes cron add-time delivery inference so channel-only announce deliveries populate `delivery.to`, and adds regression tests in `src/agents/tools/cron-tool.test.ts`.

## Best Possible Solution

Merge a guarded cron fix that infers only compatible targets, fail-closes or reports channel-only jobs with no target source, preserves explicit `to`, and adds regression coverage plus the active-version changelog entry.

## Reproduction Assessment

Yes by source inspection. Current main deterministically skips inference for channel-only announce delivery, and the PR-specific mismatch follows from combining an inferred target from one context/session with a different caller-supplied `delivery.channel`.

## Solution Assessment

No as proposed. Gating inference on explicit `to` is the right direction, but the merge must guard channel compatibility and define fail-closed behavior when no target can be inferred.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:
