## Summary

Keep open: the PR addresses a real cron announce misrouting bug that is still present on current main, and the production incident comment matches the source behavior. The proposed patch is not ready because it can persist an inferred `to` from one channel under a different caller-supplied `delivery.channel`, and it is missing the required changelog entry.

## What This Changes

The PR changes cron add-time delivery inference to infer `delivery.to` for channel-only announce deliveries and adds three regression tests in `src/agents/tools/cron-tool.test.ts`.

## Best Possible Solution

Land a guarded cron fix that infers `to` for channel-only announce jobs only when the inferred channel is compatible, handles no-source channel-only jobs deterministically, adds focused regression coverage, and records the user-facing fix in the active changelog.

## Reproduction Assessment

Yes, by source inspection. Current main deterministically skips inference for channel-only announce delivery, and the PR-specific cross-channel mismatch follows from combining a caller-supplied channel with `to: inferred.to`.

## Solution Assessment

No. Narrowing inference to explicit `to` is the right direction, but the patch must guard channel compatibility or fail closed before persisting an inferred target under an explicit different channel.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Guard inferred targets against channel mismatches:** `src/agents/tools/cron-tool.ts:749-751`
  - body: When a caller supplies `delivery.channel` without `to`, this merge can overwrite `inferred.channel` while forcing `to` from the previous context/session. That can persist an explicit target for a different channel and bypass the downstream same-channel `lastTo` guard; only reuse the inferred target when it is compatible with the explicit channel, or fail closed.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `src/agents/tools/cron-tool.ts:734-738`
  - body: This is a user-visible cron announce routing fix: channel-only jobs would now infer and persist `delivery.to` instead of falling through session routing. Repo policy requires a single-line `CHANGELOG.md` entry under the active release, but this PR only changes the cron tool and its tests.
  - confidence: 0.88
