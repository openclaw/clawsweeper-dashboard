## Summary

Keep open. The PR addresses a real current-main cron announce routing bug with a narrow code change, but it still needs the required user-facing changelog entry before merge.

## What This Changes

The PR changes cron-tool add-time delivery inference so announce agent-turn jobs infer `delivery.to` when only `delivery.channel` is supplied, and adds regression tests for channel-only, explicit-target, and mode-less announce delivery.

## Best Possible Solution

Land the focused inference fix after adding a single-line changelog entry and validating the touched cron-tool surface, while preserving the existing rules that explicit `delivery.to` wins and `webhook`/`none` modes do not infer chat announce targets.

## Reproduction Assessment

Yes. The current-main source path deterministically skips inference for `{ mode: "announce", channel: "slack" }` because `channel` makes `hasTarget` truthy, and the existing test helper can exercise that exact add-time cron-tool path; this review did not execute tests because it was read-only.

## Solution Assessment

Yes for the code path. Gating only on explicit `to` and restoring the inferred `to` after merging caller delivery fields is the narrowest maintainable fix; the safer merge path is to add the required changelog entry and validate before landing.
