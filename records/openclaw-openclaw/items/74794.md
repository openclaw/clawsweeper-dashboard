## Summary

Keep open: the cron announce mis-route is credible and current main still skips inference for channel-only delivery, but this PR can persist an inferred target under an incompatible caller-supplied channel and needs a changelog entry before merge.

## What This Changes

The PR changes cron add-time delivery inference so channel-only announce deliveries populate `delivery.to` from the current delivery context or agent session key, with regression tests in `cron-tool.test.ts`.

## Best Possible Solution

Merge a guarded version that infers `to` only from a compatible context/session, fails closed or refuses inference on explicit-channel mismatches, preserves explicit `to`, adds matching and mismatch regressions, and includes the active-version changelog entry.

## Reproduction Assessment

Yes by source inspection. Current main deterministically skips inference for channel-only announce delivery, and the PR-specific mismatch follows from combining an inferred target from one context/session with a different caller-supplied `delivery.channel`.

## Solution Assessment

No, not as proposed. Gating inference on explicit `to` is the right direction, but the patch must guard channel compatibility before persisting `inferred.to` under a caller-supplied channel.
