## Summary

Keep open: the underlying cron announce routing bug is still present on current main, but this PR cannot merge as-is because it can persist mismatched channel/target pairs and lacks required changelog coverage.

## What This Changes

The PR changes cron-tool add-time delivery inference to populate `delivery.to` for channel-only announce jobs and adds focused regression tests.

## Best Possible Solution

Merge a version that only reuses inferred targets when the explicit channel is absent or compatible, preserves explicit `to`, adds the active-version changelog entry, and passes focused cron tests plus the changed gate.

## Reproduction Assessment

Yes by source inspection. Current main deterministically skips inference for `delivery: { mode: "announce", channel: "slack" }`, and the PR-specific mismatch follows from combining a Matrix current context with caller `delivery.channel: "slack"` and no `to`.

## Solution Assessment

No as proposed. Gating inference on explicit `to` is the right direction for same-channel jobs, but the patch also needs a channel-compatibility guard before it persists `inferred.to` under a caller-supplied channel.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Guard inferred targets against channel mismatches:** `src/agents/tools/cron-tool.ts:751`
  - body: When a caller supplies `delivery.channel` without `to`, this merge can overwrite `inferred.channel` while forcing `to` from the previous context/session. A cron created from one channel with `{ channel: "slack" }` could persist a Slack delivery with a non-Slack target, bypassing the downstream same-channel fallback guard; only reuse the inferred target when it is compatible with the explicit channel.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `src/agents/tools/cron-tool.ts:734-738`
  - body: This changes user-visible cron announce routing: channel-only cron jobs now infer and persist `delivery.to` instead of falling back through session routing. Repo policy requires user-facing fixes to add a single-line `CHANGELOG.md` entry under the active release, but this PR only changes the cron tool and tests.
  - confidence: 0.88

## Security Review

Concerns:
