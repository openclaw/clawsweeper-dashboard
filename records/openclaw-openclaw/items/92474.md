## What This Changes

The issue requests that blocked stdio MCP env keys such as `PYTHONPATH` be diagnosed at config time or warned once instead of flooding the gateway journal.

## Best Possible Solution

Land one canonical upgrade-safe MCP fix that centralizes the blocked-key policy across write, validation, runtime-warning, and doctor-recovery boundaries, then close this issue as implemented.

## Reproduction Assessment

Yes, at source level. Current main saves arbitrary MCP env keys through `setConfiguredMcpServer`, and the stdio transport resolver logs whenever the same blocked key is dropped.

## Solution Assessment

Yes: a shared validation and runtime deduplication fix with doctor recovery is the best maintainable shape. A helper-only rejection would leave generic config paths and existing saved records inconsistent.
