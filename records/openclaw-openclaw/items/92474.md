## Summary

Current main has the reported save-then-warn mismatch, and the reporter’s open linked PR now owns a focused fix; the issue should remain open until that PR is reviewed and either merged or closed.

## What This Changes

Reject or clearly surface blocked stdio MCP environment variables and prevent their runtime warnings from flooding gateway logs.

## Best Possible Solution

Use one shared blocked-env predicate at the MCP write, schema-validation, and runtime-defense boundaries; reject new invalid stdio configs clearly, retain filtering for hostile or legacy raw input, and deduplicate fallback warnings per server and key.

## Reproduction Assessment

Yes, from current-main source with high confidence: `setConfiguredMcpServer` accepts the env map, while every later transport resolution filters blocked keys through an unconditional warning callback. This read-only review did not execute the CLI against a disposable config.

## Solution Assessment

Yes in direction: the linked PR addresses the narrowest maintainable boundaries by sharing one policy across writes, schema validation, and runtime fallback. Its upgrade behavior and real-environment proof still need to be settled during PR review.
