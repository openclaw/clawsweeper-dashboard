## Summary

Current main and v2026.6.6 still save blocked stdio MCP environment keys without a configuration error, then warn whenever the saved server is resolved; this contradicts the public MCP contract and remains a narrow, unowned bug.

## What This Changes

Reject blocked MCP stdio environment keys during configuration and stop repeated runtime warnings for already-saved invalid entries.

## Best Possible Solution

Use one shared blocked-key inspection contract to reject new invalid stdio server config before persistence, report and repair existing invalid config through doctor, and retain quiet runtime filtering as defense-in-depth.

## Reproduction Assessment

Yes at source level: `mcp set` persists the raw object, and later status, doctor, probe, or runtime resolution invokes the warning callback for each blocked key. A live command was not run because the review contract requires the checkout to remain byte-for-byte clean.

## Solution Assessment

Yes: shared set-time validation with doctor handling is the narrowest maintainable solution because warning throttling alone would preserve invalid canonical config and the documentation mismatch.
