## Summary

Keep open: current main and the latest release still accept blocked stdio MCP env keys at write time and warn from the runtime resolver on repeated resolutions, while two open linked PRs now own competing fix paths.

## What This Changes

The issue asks OpenClaw to stop silently saving or repeatedly warning about blocked stdio MCP env keys such as `PYTHONPATH`.

## Best Possible Solution

Land one canonical fix that enforces the documented blocked-env policy at the shared MCP config/write boundary, keeps runtime filtering as defense in depth, deduplicates legacy warnings, and includes upgrade recovery for released configs.

## Reproduction Assessment

Yes at source level. Current main saves arbitrary MCP `env` keys, then multiple runtime and CLI paths call the resolver that drops blocked keys and emits the reported warning.

## Solution Assessment
