The PR changes the package updater to convert completed nonzero `openclaw doctor --fix` results into advisory warning steps after package install verification, updates progress rendering for advisory stderr, and adds regression tests for advisory and killed doctor runs.

## Best Possible Solution

Land this only if maintainers want verified package installs to continue after completed doctor repair warnings, with killed or timed-out doctor runs still fatal and advisory stderr visible in progress and summaries.
