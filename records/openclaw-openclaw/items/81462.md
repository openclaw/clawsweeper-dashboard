Keep open: current main still has a source-reproducible path where a mentioned Telegram group turn is processed but ordinary final text is suppressed by the documented group `message_tool` visible-reply policy, and the reported Telegram client “version not compatible” placeholder still needs a live current-main payload capture. ## What This Changes

The issue reports that Telegram group mentions with `requireMention: true` are processed but do not produce a delivered group reply, while `/send` to the same chat works.

## Best Possible Solution

Run a live Telegram group mention probe first; fix a confirmed Telegram payload/client incompatibility narrowly, or keep the strict visible-reply policy and direct operators to `messages.groupChat.visibleReplies: "automatic"` unless maintainers explicitly approve the @mention auto-delivery exception.
