## Summary

Keep open: current main and latest release still carry the 15s Dockerfile healthcheck start period, and open fixing PRs #75766 and #75809 already target this issue with different proposed durations.

## What This Changes

Increase the Dockerfile `HEALTHCHECK --start-period` so legitimate Docker cold starts do not flip to `unhealthy` after 15 seconds.

## Best Possible Solution

Review #75766 and #75809 together, choose one Docker healthcheck grace period, decide whether Compose parity belongs in the same change, and land a single canonical fix with changelog coverage.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence because current main and `v2026.4.29` both schedule the image healthcheck after only 15s, while the issue gives a Docker run sequence and related startup reports document legitimate cold starts that can exceed that window.

## Solution Assessment

Yes for the direction: widening the Dockerfile start period is the narrowest maintainable fix for premature OCI `unhealthy` state without changing `/healthz` semantics. The unsettled part is the exact duration and whether the Compose template should be aligned.
