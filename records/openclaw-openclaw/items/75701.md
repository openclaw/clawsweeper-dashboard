## Summary

Keep open. Current main still ships the Docker image `HEALTHCHECK` with `--start-period=15s`, and two open PRs already target this report with closing syntax, so the issue should remain open until maintainers choose the grace period and land or supersede the fix PRs.

## What This Changes

Raise the Docker image healthcheck start-period so legitimate gateway cold starts do not flip containers to `unhealthy` before `/healthz` can respond.

## Best Possible Solution

Review the competing PRs, choose a cold-start grace period that matches current built-image behavior, update the Dockerfile and any intended Compose/docs/changelog follow-through, then land one closing PR and close or supersede the other.

## Reproduction Assessment

Yes. Current main directly reproduces the configuration mismatch because `Dockerfile` still has `--start-period=15s`; the live false-`unhealthy` timing should be rechecked with a built Docker image before finalizing the exact replacement duration.

## Solution Assessment

Yes for the direction. Widening only the Dockerfile healthcheck start period is the narrowest fix for premature OCI `unhealthy` state without changing `/healthz` semantics; the unresolved part is the exact duration and Compose parity.
