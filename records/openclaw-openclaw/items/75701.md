## Summary

Keep open: current main still ships the Docker image healthcheck with a 15 second start period, and two open PRs already use closing syntax for this issue. The issue should remain open until maintainers choose, adjust, land, or close the competing Dockerfile fixes.

## What This Changes

The issue requests widening the Dockerfile OCI HEALTHCHECK start grace period so legitimate Docker cold starts do not report unhealthy before the gateway binds.

## Best Possible Solution

Ship one consolidated Docker healthcheck timing fix that matches current image cold-start behavior, with any needed Compose, docs, and changelog follow-through, then let the merged PR close this issue.

## Reproduction Assessment

Yes. The configuration side is directly reproducible from current main because `Dockerfile:294` still has `--start-period=15s`; the runtime false-unhealthy path can be reproduced with the reporter's Docker cold-start command and `docker ps` inspection around the first healthcheck window.

## Solution Assessment

Yes for the direction: increasing the Dockerfile healthcheck start period is the narrowest fix for premature OCI unhealthy state without changing `/healthz` semantics. The exact duration and whether Compose should also change remain maintainer choices across the two open PRs.
