## Summary

Keep open: current main and the latest release still ship the short Docker image healthcheck grace period, and an open closing PR already targets this issue. The issue should stay open until the canonical Docker healthcheck fix is reviewed, updated if needed, and merged.

## What This Changes

The issue requests a longer Docker image healthcheck start period so legitimate cold starts do not flip to unhealthy before the gateway binds.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/75809, or a narrow replacement, after aligning the image and bundled Compose healthcheck grace periods to the accepted cold-start envelope.

## Reproduction Assessment

Partly. The short Dockerfile value and separate short Compose value are present on current main, and the issue gives a concrete `docker run` plus `docker ps` health-state check; I did not live-rerun a current-main cold start.

## Solution Assessment

Unclear. Widening the start period is the narrow Docker direction if slow first-bind still occurs, but maintainers still need to choose the canonical duration and whether the Compose healthcheck must change with it.
