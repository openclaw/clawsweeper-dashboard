## Summary

Keep open: current main and the latest released image still carry the reported 15-second Docker image healthcheck start period, and open closing PRs already target the concrete Dockerfile fix.

## What This Changes

Raise the Docker image healthcheck start period so legitimate cold starts do not trip OCI unhealthy state before the gateway binds.

## Best Possible Solution

Land one canonical Docker healthcheck fix that widens the image start period, decide Compose parity, and include changelog coverage for the user-visible deployment behavior.

## Reproduction Assessment

Yes. The configuration-level reproduction is high confidence because current main and v2026.4.29 still start the OCI healthcheck after 15 seconds; the issue body supplies the live Docker run sequence, but I did not rerun a full cold-start timing test in this read-only review.

## Solution Assessment

Yes for the direction: widening the image-level start period is the narrow maintainable fix without changing `/healthz` semantics. The unresolved decision is the canonical value and whether the Compose template should change at the same time.
