## What This Changes

The issue asks to increase the Docker image HEALTHCHECK start grace so legitimate cold starts are not marked unhealthy around 15 seconds.

## Best Possible Solution

Review and land one canonical Docker operator fix that aligns the image and bundled Compose healthcheck grace periods to the chosen cold-start envelope, keeps the `/healthz` probe semantics, and adds release-note/static coverage as needed.

## Reproduction Assessment

Partly. The configuration-level path is source-reproducible because current main and `v2026.5.2` still start OCI image health checking after 15 seconds and bundled Compose after 20 seconds; I did not run a live fresh-host Docker cold-start reproduction in this read-only review.

## Solution Assessment

Partly. Lengthening `--start-period` is the narrow fix for the reported image behavior, but the best maintainable solution should also handle the separate Compose healthcheck and pick one issue-backed grace period rather than closing this issue while competing PRs remain open.
