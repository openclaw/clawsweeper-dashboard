## Summary

Keep open: current main still ships the reported 15-second Docker image healthcheck start period, and two open closing PRs already target this issue with different proposed durations.

## What This Changes

The issue asks to widen the Docker image `HEALTHCHECK --start-period` so legitimate gateway cold starts do not become `unhealthy` after 15 seconds.

## Best Possible Solution

Land one canonical Docker healthcheck fix that widens the image start period, decide Compose parity, and include changelog coverage for the user-visible deployment behavior.

## Reproduction Assessment

Yes. The configuration-level reproduction is high confidence because current main still starts the OCI healthcheck after 15 seconds; the issue body supplies the Docker run sequence, while live timing was not rerun in this read-only review.

## Solution Assessment

Yes for the direction: widening the image-level start period is the narrow Docker-surface fix without changing `/healthz` semantics. The unsettled part is whether the canonical value should be 120s, 600s, or another maintainer-approved duration.
