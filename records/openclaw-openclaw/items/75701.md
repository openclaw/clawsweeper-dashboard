## Summary

Keep open: current main still ships the Docker image healthcheck with a 15 second start period, and open PR #75766 is already the linked implementation candidate with closing syntax. The issue should remain open until that PR is reviewed, adjusted, merged, or closed.

## What This Changes

Request to widen the Docker image `HEALTHCHECK --start-period` so legitimate cold starts do not prematurely mark containers unhealthy.

## Best Possible Solution

Review PR #75766, choose a cold-start grace period that matches current built-image behavior, and ship the Dockerfile healthcheck update with any deliberate compose/docs/changelog follow-through.

## Reproduction Assessment

Yes. Current source confirms the 15s image-level healthcheck value, and the issue provides a concrete `docker run` plus `docker ps` path to observe premature `unhealthy` state; exact timing should still be rechecked with a built image before selecting the final duration.

## Solution Assessment

Yes for the direction. Widening the Dockerfile `--start-period` is the narrowest fix for premature OCI health state without changing `/healthz` semantics; the remaining question is the final grace period and whether the compose template should align.
