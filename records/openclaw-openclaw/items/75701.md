## Summary

Keep open: current main and the latest release still ship the reported 15-second Docker image healthcheck, and two open closing PRs already target the same Dockerfile value. Current main has mitigated the original cold-start path by binding HTTP before startup plugin runtime loading, but maintainers still need to choose the canonical image grace period and whether Compose should follow.

## What This Changes

The issue asks to raise the Docker image HEALTHCHECK start grace from 15 seconds to a realistic cold-start window, with possible Compose parity.

## Best Possible Solution

Review and land one canonical healthcheck grace-period change, including Compose parity if desired, or explicitly close the linked PRs if current post-bind behavior makes the short grace acceptable.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Partly: the configuration path is source-reproducible because current main still starts OCI healthchecks after 15 seconds, but I did not live-rerun a current-main Docker cold start and startup plugin loading is now post-bind.

## Solution Assessment

Is this the best way to solve the issue? Unclear: widening the start period is the narrow Dockerfile fix if first-bind can still exceed the current grace, but maintainers should choose the duration and whether `docker-compose.yml` should be updated with it.
