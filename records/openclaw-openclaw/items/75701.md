## Summary

Keep open: current main and the latest tagged release still ship the reported 15-second image healthcheck, while two open closing PRs already target the Dockerfile value. Current main has an important mitigation by loading startup plugins after HTTP bind, but that does not settle the requested Docker healthcheck grace period or Compose parity.

## What This Changes

The issue asks to increase the Docker image healthcheck start period so OCI health consumers do not mark legitimate cold starts unhealthy.

## Best Possible Solution

Review and land one canonical Docker healthcheck change, or explicitly close the PRs if latest post-bind startup behavior makes the 15-second image grace acceptable.

## Reproduction Assessment

Partly. The configuration-level path is source-reproducible because current main still starts the OCI healthcheck after 15 seconds, but the full current-main cold-start unhealthy flip was not live-rerun and startup plugin loading is now post-bind.

## Solution Assessment

Unclear. Widening the start period is the narrow Dockerfile fix if slow first-bind still occurs, but current main also mitigates the reported cold-start cause by binding before startup plugin runtime work, so maintainers should choose the canonical duration and Compose parity.
