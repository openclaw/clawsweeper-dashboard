## Summary

Keep open: current main and the latest release still ship the reported 15-second Docker image healthcheck, and open closing PRs already target the same Dockerfile value.

## What This Changes

The issue asks to increase the Docker image HEALTHCHECK start-period so legitimate cold starts do not become unhealthy before `/healthz` can answer.

## Best Possible Solution

Land one canonical Docker healthcheck grace-period change, likely with an explicit Compose parity decision, then let the merged closing PR resolve this issue.

## Reproduction Assessment

No high-confidence current-main live reproduction was established. The configuration path is source-reproducible because current main and `v2026.5.2` still start OCI health checks after 15 seconds, but current main now defers startup plugin work until post-bind.

## Solution Assessment

Unclear. Widening the Dockerfile start period is the narrowest fix if slow first-bind still occurs, but maintainers need to choose the duration and whether the Compose healthcheck should match.
