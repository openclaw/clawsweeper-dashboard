## Summary

Keep open: current main and the latest release still ship the reported short Docker image healthcheck grace period, and two open PRs already reference this issue with closing syntax. Current main also moved startup plugin runtime work after HTTP bind, so maintainers should choose the canonical Dockerfile/Compose grace behavior instead of closing or spawning a competing repair branch.

## What This Changes

The issue requests increasing the Docker image HEALTHCHECK start period so legitimate cold starts are not marked unhealthy after about 15 seconds.

## Best Possible Solution

Review and land one canonical Docker healthcheck grace-period change, aligning Compose if desired, or explicitly close the linked PRs if the current post-bind startup behavior makes the existing grace acceptable.

## Reproduction Assessment

Partly. Current source proves the image healthcheck still starts after 15 seconds and Compose after 20 seconds, but I did not live-run a current-main Docker cold start; gateway startup now binds HTTP before post-attach runtime work.

## Solution Assessment

Unclear. Widening the start period is the narrow Dockerfile fix if slow first-bind still occurs, but current main has startup mitigation and the open PRs disagree on duration, so maintainers should choose the canonical value and Compose parity.
