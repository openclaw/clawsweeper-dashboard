## Summary

Keep open: current main still ships the reported 15-second Docker image healthcheck start period, and two open closing PRs already target the issue with different proposed durations.

## What This Changes

The issue requests widening the Dockerfile `HEALTHCHECK --start-period` so legitimate Docker cold starts are not marked unhealthy before `/healthz` can answer.

## Best Possible Solution

Land one canonical Docker healthcheck fix that widens the image start period, decide Compose parity, and include changelog coverage for the user-visible deployment behavior.

## Reproduction Assessment

Yes. The configuration-level reproduction is high confidence because current main and `v2026.4.29` still start the OCI healthcheck after 15 seconds; the issue body supplies the Docker run sequence, while live timing was not rerun in this read-only review.

## Solution Assessment

Yes for the direction: widening the image-level `--start-period` is the narrow Docker-surface fix without changing `/healthz` semantics. The remaining maintainer choice is the intended duration and whether the Compose start period should move with it.
