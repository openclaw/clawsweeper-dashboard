## Summary

Keep open: current main still ships the 15s Docker image healthcheck start period, and two open PRs already target the same fix with different grace periods.

## What This Changes

The issue requests increasing the Docker image HEALTHCHECK start period so legitimate cold starts do not mark the container unhealthy before the gateway binds.

## Best Possible Solution

Review #75766 and #75809 together, choose one Docker healthcheck grace period, decide whether Compose parity belongs with it, and land a single user-facing fix with changelog coverage.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence because current main still schedules the image healthcheck after 15s, the docs confirm Docker marks repeated failures as `unhealthy`, and the issue provides a concrete Docker run sequence plus related cold-start reports.

## Solution Assessment

Yes for the direction: widening the Dockerfile start period is the narrowest maintainable fix for premature OCI `unhealthy` state without changing `/healthz` semantics. The unsettled part is the exact duration and whether the Compose template should be aligned.
