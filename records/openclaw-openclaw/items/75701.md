## Summary

Keep open: current main still ships the 15s Dockerfile healthcheck grace period, and two open closing PRs already target this issue with competing durations.

## What This Changes

The issue requests increasing the Docker image `HEALTHCHECK --start-period` from 15s to a realistic cold-start window so orchestrators do not mark legitimate starts unhealthy.

## Best Possible Solution

Review #75766 and #75809 together, pick one Docker healthcheck grace period, decide whether Compose parity belongs in the same fix, and land a single changelog-covered PR.

## Reproduction Assessment

Yes. Current main still schedules the image healthcheck after 15s, while the issue gives a Docker run reproduction and linked startup reports document legitimate cold starts beyond that window.

## Solution Assessment

Yes for the direction: widening the Dockerfile start period is the narrowest maintainable fix for premature OCI `unhealthy` state without changing `/healthz` semantics. The unsettled part is the exact duration and Compose parity.
