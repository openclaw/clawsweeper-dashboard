## Summary

Keep open: the latest PR head is still incorrect because the new persistedConfig reads are not type-safe, and the gateway write/response path still discards the persisted result and responds with the pre-write config.

## What This Changes

The PR changes config mutation helpers to return persistedConfig from writes and adds a changelog entry for config RPC response behavior.

## Best Possible Solution

Finish the contract end to end: type the mutation IO result, return persistedConfig from commitGatewayConfigWrite, redact that persistedConfig in config.set/config.patch/config.apply responses, add focused regression coverage, and require patched gateway proof.

## Reproduction Assessment

Yes, from source inspection plus the linked bug report's live pre-fix proof. Current main writes through transforms that can change the persisted/readback config while the gateway handlers still respond from parsed or validated pre-write config; this read-only review did not perform a live write.

## Solution Assessment

No. Returning persistedConfig from the lower write helpers is the right direction, but this PR head stops before the gateway response path and leaves the injected writer return type as unknown.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.96

Full review comments:

- **[P1] Type the injected config writer before using persistedConfig:** `src/config/mutate.ts:266`
  - body: ConfigMutationIO.writeConfigFile is still declared as Promise<unknown>, but this patch now reads writeResult.persistedConfig. Under strict TypeScript this access is invalid, and injected writers that still resolve void will fail at runtime; update the IO contract and mocks before reading the field.
  - confidence: 0.95
- **[P1] Thread persistedConfig through the gateway response path:** `src/config/mutate.ts:50`
  - body: Adding persistedConfig to ConfigReplaceResult does not fix the user-visible RPCs while commitGatewayConfigWrite still discards replaceConfigFile's return value and config.set/config.patch/config.apply still respond with parsed or validated config. Return persistedConfig from the gateway write helper and redact that value in the three responses.
  - confidence: 0.97
