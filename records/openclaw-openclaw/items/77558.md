## Summary

Keep open: the PR is aimed at the right bug, but the updated patch still leaves one public writer path able to return no persisted result under the new contract, and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR threads a persisted config value from config writes through `replaceConfigFile`, `mutateConfigFile`, `commitGatewayConfigWrite`, and the `config.set`/`config.patch`/`config.apply` responses, with a changelog entry.

## Best Possible Solution

Finish the persistence-result contract end to end: every public writer branch should return a persisted result, the gateway responses should use that value, focused regression coverage should cover transformed writes, and the contributor should add real config RPC proof.

## Reproduction Assessment

Yes from source inspection. Current main writes through transforms that can change the persisted/readback config while the gateway handlers respond from the pre-write parsed or validated config; this read-only review did not perform a live write.

## Solution Assessment

No. The direction is the narrow maintainable fix, but this implementation is incomplete until the public writer's skip-refresh branch also returns a persisted result and after-fix behavior is shown from a real config RPC run.
