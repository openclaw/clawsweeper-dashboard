## Summary

Keep open. This is a focused bug-fix PR: current main still raw-compares `boundNode` and `requestedNode`, while the PR resolves both selectors through the existing node resolver before enforcing the binding and keeps mismatches fail-closed.

## What This Changes

The PR canonicalizes configured and requested exec-node selectors before comparing node-host bindings, adds regression coverage, and records the fix in the changelog.

## Best Possible Solution

Land the focused canonicalization fix after normal maintainer review, preserving the existing resolver contract and regression coverage.

## Reproduction Assessment

Yes. The failure is source-reproducible on current main because the guard compares raw selector strings before the resolver can map a display name, IP, or ID prefix to the canonical node ID.

## Solution Assessment

Yes. Resolving both selectors with `resolveNodeIdFromList` before comparison is the narrowest maintainable fix because it reuses the existing node selector contract and keeps stale or mismatched bindings fail-closed.
