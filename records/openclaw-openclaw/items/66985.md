## Summary

Keep this PR open for maintainer review and landing: current main still rejects valid bound/requested exec-node selector combinations before canonicalization, and the PR applies a focused fix with regression coverage and sufficient live proof.

## What This Changes

The PR resolves configured and requested exec-node selectors to canonical node IDs before enforcing node-host bindings, then adds regression tests and a changelog entry.

## Best Possible Solution

Land the focused canonicalization fix after normal maintainer review, preserving the existing resolver contract and fail-closed binding behavior.

## Reproduction Assessment

Yes. On current main, resolveNodeExecutionTarget compares raw boundNode/requestedNode values before the existing resolver can map a display name, IP, or ID prefix to a canonical node ID.

## Solution Assessment

Yes. Resolving both selectors through resolveNodeIdFromList before comparing canonical IDs is the narrowest maintainable fix, and the PR preserves fail-closed behavior for stale or mismatched bindings.
