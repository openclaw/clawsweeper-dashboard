## Summary

Keep open. Current main and the latest release still reject valid host=node requests when bound and requested selectors identify the same node through different supported forms, and this PR is a focused, proof-supplied fix with regression coverage; the competing open fix path is weaker because it still lacks real behavior proof.

## What This Changes

The PR canonicalizes bound and requested node selectors before enforcing node-host exec binding and adds regression tests for selector matches, mismatches, and missing nodes.

## Best Possible Solution

Land this PR or an equivalent canonicalization fix so node exec compares canonical node identity, then close https://github.com/openclaw/openclaw/issues/87213 after merge.

## Reproduction Assessment

Yes. Source inspection shows current main rejects a configured bound full node id plus requested display name before the shared resolver can prove both selectors point to the same node; I did not run a live paired-node repro in this read-only review.

## Solution Assessment

Yes. Resolving both selectors inside resolveNodeExecutionTarget is the narrow owner-boundary fix because the caller supplies raw config/tool selectors and this helper owns conversion to a concrete node execution target.
