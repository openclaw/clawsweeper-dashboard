## Summary

Keep open: current main still has the raw bound/requested node comparison that causes the reported failure, while this PR applies the narrow fix through the existing node resolver with targeted regression coverage, sufficient real behavior proof, and green PR checks.

## What This Changes

The PR canonicalizes configured and requested exec node selectors before enforcing node-host bindings, adds regression tests, and records the user-facing fix in the changelog.

## Best Possible Solution

Land this narrow canonicalization fix after normal maintainer merge gating, keeping node selection semantics centralized in the existing resolver and preserving fail-closed behavior for different nodes or stale bindings.

## Reproduction Assessment

Yes from source inspection. Current main compares boundNode and requestedNode as raw strings before selector resolution, so a configured canonical node ID and a requested display name or prefix for the same node fail before resolveNodeIdFromList can run.

## Solution Assessment

Yes. The PR uses the existing resolver contract for both selectors, keeps the binding comparison on canonical IDs, and preserves the stale bound-node failure path by falling back to the raw configured value for final resolution.
