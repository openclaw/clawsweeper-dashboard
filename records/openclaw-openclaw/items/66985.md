## What This Changes

The PR canonicalizes configured `boundNode` and per-call `requestedNode` through the existing node resolver before enforcing node-host bindings, adds regression tests, and updates the changelog.

## Best Possible Solution

Land the focused canonicalization fix after normal maintainer review, preserving the existing node selector resolver and regression coverage.

## Reproduction Assessment

Yes. The failure is source-reproducible on current main because the guard compares raw `boundNode` and `requestedNode` before the resolver can map display names, IPs, or prefixes to canonical IDs.

## Solution Assessment

Yes. Resolving both selectors with `resolveNodeIdFromList` before comparison is the narrowest maintainable fix because it reuses the existing node selector contract and keeps mismatched or stale bindings fail-closed.
