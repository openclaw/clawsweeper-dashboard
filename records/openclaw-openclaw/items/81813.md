Keep open for maintainer review. The patch is focused and appears correct: current main lacks an adapter-level runtime handoff, the PR adds one before `startAccount` with error isolation, and the supplied real Weixin logs plus current check runs support the behavior.

## What This Changes

The branch adds optional `ChannelGatewayAdapter.setChannelRuntime`, invokes it before `startAccount` with sync/async rejection guards, and adds gateway regression tests.

## Best Possible Solution

Land or adjust the backward-compatible gateway hook as an explicit plugin API, keeping `startAccount` context semantics unchanged and preserving startup error isolation.
