## Summary

Keep open. Current main still drops the top-level provider in the Telegram model keyboard fallback, and the PR is a narrow, correct fix for the reported OpenRouter case, but it is currently draft and still needs the required user-facing changelog entry before merge.

## What This Changes

The PR changes the Telegram model keyboard fallback label to prepend the provider for nested provider-local IDs and adds two regression tests for OpenRouter-style labels and current-model marking.

## Best Possible Solution

Land the narrow Telegram fallback-label fix with its regression tests after adding one Unreleased/Fixes changelog bullet, and keep any smarter long-label truncation design as a separate follow-up rather than expanding this patch.

## Reproduction Assessment

Yes. A source-level reproduction on current main is `buildModelsKeyboard({ provider: "openrouter", models: ["openai/gpt-5.4-mini"], currentPage: 1, totalPages: 1 })`, which follows the fallback path that displays only `openai/gpt-5.4-mini`.

## Solution Assessment

Yes, after the changelog entry is added. The proposed fallback is the narrowest maintainable fix for the reported short nested OpenRouter IDs because it preserves callback data, selection resolution, `modelNames` precedence, pagination, and current-model marking.
