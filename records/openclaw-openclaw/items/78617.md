## What This Changes

Ensure `sessions_spawn` child runs keep generated `extraSystemPrompt` context when the target agent uses `systemPromptOverride`.

## Best Possible Solution

Keep the current main prompt composition and ship it in the next release; the separate visible task-envelope work can continue independently.

Yes. The reporter provided a concrete installed-package failure on `2026.5.5`, and the old source path is clear: `sessions_spawn` puts the task in `extraSystemPrompt` while the override branch used to omit that field.

Yes. The narrow maintainable fix is the current main behavior: preserve `extraSystemPrompt` in the `systemPromptOverride` branch with regression coverage, while leaving the visible transcript-envelope feature to https://github.com/openclaw/openclaw/pull/78992.
