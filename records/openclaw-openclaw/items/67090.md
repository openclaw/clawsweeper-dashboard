This PR widens post-compaction AGENTS.md section extraction from H2/H3 headings to H1-H4 headings and adds regression tests for H1/H4 matching, H1 boundaries, and nested H1 sections.

## Best Possible Solution

Yes for the narrow parser behavior: current main's regex only matches H2/H3, so an AGENTS.md with `# Session Startup` is not extracted by this path. The unresolved part is product intent, not reproduction.
