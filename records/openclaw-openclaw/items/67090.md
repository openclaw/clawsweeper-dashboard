## Summary

Keep open. Current main still implements and documents H2/H3-only AGENTS.md section extraction, while this PR broadens a shared runtime parser to H1-H4 without resolving the maintainer’s heading-contract question or updating docs/config help.

## What This Changes

This PR widens `extractSections` to accept H1-H4 Markdown headings and adds regression tests for H1/H4 extraction plus H1 boundaries and nesting.

## Best Possible Solution

End with a maintainer-approved heading-level contract where parser behavior, regression tests, config comments, generated help/schema text, and public docs agree; if no real H1 case is accepted, keep the H2/H3 behavior.

## Reproduction Assessment

Yes for the source-level parser behavior: current main only matches H2/H3 headings, so H1/H4 headings are skipped. No for a high-confidence bug against the current public contract, because docs, typed config, generated help, and the template define H2/H3 extraction.

## Solution Assessment

No as proposed. The diff broadens shared runtime behavior to H1-H4 without settling the heading-level policy or updating all contract surfaces, so the safer path is maintainer approval followed by aligned code, docs, and tests.
