## Summary

Keep this PR open. Current main still has the OpenClawBrain package/manifest identity mismatch path, but the proposed core alias needs maintainer policy review and a rebase against current plugin discovery code before merge.

## What This Changes

The PR normalizes `@openclawbrain/openclaw` to the canonical `openclawbrain` plugin id in discovery and suppresses the matching install-time manifest/package info log, with targeted tests.

## Best Possible Solution

Keep the manifest id as the canonical install/config key; if this compatibility exception is accepted, represent package/manifest equivalence in one shared helper or metadata-backed table used by both discovery and install.

## Reproduction Assessment

Yes. By source inspection on current main, installing a native plugin with package name `@openclawbrain/openclaw` and manifest id `openclawbrain` makes `packageNameMatchesId(...)` return false and keeps the install info-log path live.

## Solution Assessment

Unclear. The PR fixes the observable warning narrowly, but duplicating a plugin-specific alias in core discovery and install is weaker than a shared helper or manifest/registry-backed compatibility seam.
