## Summary

Keep this PR open. Current main still has the OpenClawBrain package/manifest mismatch path, but the proposed core alias needs maintainer judgment because it hard-codes a plugin-specific exception and the branch is stale against current plugin discovery/install code.

## What This Changes

The PR adds a scoped OpenClawBrain package alias in plugin discovery, suppresses the install-time manifest/package mismatch info log for that alias, and adds targeted discovery/install tests.

## Best Possible Solution

Keep the manifest id as the canonical install/config key; if this compatibility exception is accepted, implement it through one shared package/manifest equivalence helper or metadata-backed table used by both discovery and install.

## Reproduction Assessment

Yes. By source inspection on current main, installing a native plugin with package name `@openclawbrain/openclaw` and manifest id `openclawbrain` makes `packageNameMatchesId(...)` return false and keeps the install info-log path live.

## Solution Assessment

Unclear. The PR addresses the observable warning narrowly, but duplicating a plugin-specific alias in discovery and install is weaker than a shared helper or manifest/registry-backed compatibility seam.
