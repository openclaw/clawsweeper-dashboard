## Summary

Keep open. Current main still has the OpenClawBrain package/manifest mismatch behavior the PR targets, but the merge direction needs maintainer judgment because the patch hard-codes a plugin-specific alias in core and one added discovery regression does not exercise the new alias branch.

## What This Changes

The PR adds a `@openclawbrain/openclaw` to `openclawbrain` alias in plugin discovery, suppresses the install mismatch info log for that pair, and adds targeted tests.

## Best Possible Solution

Keep the manifest id as the canonical install/config key; if this compatibility exception is accepted, implement it through one shared package/manifest equivalence helper or metadata-backed table used by both discovery and install.

## Reproduction Assessment

Yes. By source inspection on current main, `@openclawbrain/openclaw` with manifest id `openclawbrain` fails `packageNameMatchesId(...)` and keeps the install info-log path live; manifestless discovery would still derive `openclaw` from the package name.

## Solution Assessment

Unclear. The PR addresses the observable warning narrowly, but duplicating a plugin-specific alias in discovery and install is weaker than a shared helper or manifest/registry-backed compatibility seam.
