## Summary

Keep open. Current main still has the reported OpenClawBrain package/manifest identity mismatch behavior, so the PR is not obsolete; however, the branch is dirty against main and the proposed core hard-coded alias needs maintainer judgment before merge or replacement.

## What This Changes

The PR adds an `@openclawbrain/openclaw` to `openclawbrain` alias in plugin discovery, suppresses the install-time manifest/package mismatch info log for that known pair, and adds targeted discovery/install regression tests.

## Best Possible Solution

Keep the manifest id as the canonical install/config key; if maintainers accept this compatibility exception, implement it through one shared package/manifest equivalence helper or metadata-backed table used by both discovery and install, then rebase the PR and add tests that exercise the actual package-derived alias path and install log suppression.

## Reproduction Assessment

Yes. By code inspection on current main, installing a native plugin with package name `@openclawbrain/openclaw` and manifest id `openclawbrain` makes `packageNameMatchesId(...)` return false and keeps the install info log path live.

## Solution Assessment

Unclear. The PR addresses the observable warning narrowly, but duplicating a plugin-specific alias in discovery and install is weaker than a shared helper or manifest/registry-backed compatibility seam.
