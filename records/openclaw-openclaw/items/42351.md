## Summary

Keep open. Current main still lacks a Feishu handler for `im.chat.access_event.bot_p2p_chat_entered_v1`, the default WebSocket monitor passes that dispatcher into the Feishu SDK, and the only related implementation PR in the provided context is closed unmerged.

## What This Changes

The issue asks OpenClaw to handle or ignore Feishu `im.chat.access_event.bot_p2p_chat_entered_v1` events so opening a bot private chat does not destabilize the Gateway.

## Best Possible Solution

Register a Feishu-owned no-op or low-noise handler for `im.chat.access_event.bot_p2p_chat_entered_v1` and add focused monitor coverage so the SDK no longer treats the informational event as unknown.

## Solution Assessment
