## Summary

Keep open. Current main still lacks a Feishu handler for `im.chat.access_event.bot_p2p_chat_entered_v1`, the default WebSocket monitor still passes the fixed dispatcher into the Feishu SDK, and the only related implementation PR is closed unmerged.

## What This Changes

The issue asks OpenClaw's Feishu plugin to handle or ignore `im.chat.access_event.bot_p2p_chat_entered_v1` so opening a bot private chat does not produce missing-handler warnings or disrupt Gateway connections.

## Best Possible Solution

Add a Feishu-owned no-op or low-noise handler for `im.chat.access_event.bot_p2p_chat_entered_v1` with focused monitor coverage; leave generic Gateway supervision changes for a separate, reproduced restart bug if needed.

## Solution Assessment
