## Summary

Keep open. Current main still does not register `im.chat.access_event.bot_p2p_chat_entered_v1`, the default Feishu WebSocket path passes events through that dispatcher, and the only related implementation PR is closed unmerged.

## What This Changes

The issue asks OpenClaw to ignore or handle the Feishu `bot_p2p_chat_entered_v1` event so opening a bot private chat does not produce unknown-handler warnings or Gateway disruption.

## Best Possible Solution

Register a Feishu-owned no-op or low-noise handler for `im.chat.access_event.bot_p2p_chat_entered_v1` and add focused coverage proving the dispatcher no longer treats the event as unknown.
