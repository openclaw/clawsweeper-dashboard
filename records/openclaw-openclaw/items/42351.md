The issue asks OpenClaw to ignore or handle Feishu `bot_p2p_chat_entered_v1` private-chat entry events so they do not destabilize the Gateway.

Add a Feishu-owned no-op or low-noise handler for `im.chat.access_event.bot_p2p_chat_entered_v1` with focused monitor coverage; leave any broader Gateway restart-supervision work for a separately reproduced bug.

## Solution Assessment
