- **[P2] Include the issue payload in the invalid snapshot fingerprint:** `src/gateway/config-reload.ts:148-151`
  - body: `snapshot.hash` is derived from the persisted/root config bytes, but invalid reload issues are produced after include/env/plugin validation. If that resolved state changes while the root hash stays the same, this branch treats the snapshot as already logged and drops the new diagnostic operators need to fix the config; include the formatted issues in the dedupe fingerprint.
  - confidence: 0.9
- **[P2] Sanitize the future-version string before warning:** `src/config/io.ts:643-644`
  - body: `cfg.meta.lastTouchedVersion` is user-editable config text, and `loadConfig()` logs the returned future-version warning directly through `logConfigMessageOnce` rather than `formatConfigIssueDetails`. A value with a valid leading version plus newlines or ANSI escapes can still forge terminal/service log output; sanitize `touched` before interpolating it or route it through the issue formatter.
  - confidence: 0.86
