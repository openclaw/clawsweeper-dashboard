  - body: This branch sets the generated LaunchAgent throttle to 30s, while current main and v2026.6.10 use a 10s baseline with restart-responsiveness comments and newer plist coverage.   - confidence: 0.86
- **[P2] Include issue details in invalid reload fingerprints:** `src/gateway/config-reload.ts:148`
  - body: `snapshot.hash` is derived from the root config bytes, so include/env/plugin validation can produce different `snapshot.issues` under the same hash. After the first warning, this branch would suppress the updated diagnostic and leave operators looking at stale error text.
  - confidence: 0.9
- **[P2] Keep future-version warning text unforgeable:** `src/config/io.ts:644`
  - body: This PR head logs `meta.lastTouchedVersion` directly while that same head parses versions with a non-anchored regex, so user-edited config can append forged log text after a numeric version. Refresh to current main's anchored parser/guard, or sanitize/reject the value before logging.
  - confidence: 0.82
- **[P2] Use the current macOS supervisor log path:** `docs/gateway/logging.md:52`
  - body: The docs addition points macOS LaunchAgent users at `$OPENCLAW_STATE_DIR/logs/gateway*.log`, but current launchd installs resolve stdout to `~/Library/Logs/openclaw/gateway*.log` and pass stderr as `/dev/null`. This would send operators to the wrong files during recovery.
  - confidence: 0.88
