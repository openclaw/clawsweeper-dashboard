Overall correctness: patch is incorrect

Full review comments:

- **[P2] Sanitize the future-version warning before logging:** `src/config/io.ts:644`
  - body: `getFutureVersionWarning` still interpolates `cfg.meta.lastTouchedVersion` into the returned warning. On this branch, `compareOpenClawVersions` accepts a numeric prefix with trailing text, so a config value like `9999.0.0\nFORGED` reaches the warning path and can forge extra terminal/log lines. Sanitize `touched` or make the version parser reject trailing text before logging this warning.
  - confidence: 0.88
- **[P2] Include invalid issue text in the reload fingerprint:** `src/gateway/config-reload.ts:148`
  - body: Using `snapshot.hash` as the invalid reload fingerprint can suppress changed diagnostics when the root config bytes stay the same but resolved validation output changes. The PR's snapshot hash is computed from the root file before `$include` and validation, so include the formatted issues in the fingerprint instead of preferring the hash alone.
  - confidence: 0.77
