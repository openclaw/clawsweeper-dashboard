## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Include issue details in the invalid reload fingerprint:** `src/gateway/config-reload.ts:148`
  - body: This uses `snapshot.hash` before the formatted issue text, but that hash is the root config bytes. `$include` contents, environment substitution, or plugin-resolution state can change `snapshot.issues` while the root file hash stays the same, so the new diagnostic would be suppressed after the first warning. Include the formatted issues in the fingerprint.
  - confidence: 0.86
- **[P2] Sanitize the future-version value before logging:** `src/config/io.ts:644`
  - body: `meta.lastTouchedVersion` is user-edited config text and this PR head still interpolates it directly into a warning. In the same head, version parsing accepts a numeric prefix with trailing text, so a value like `9999.0.0\nFORGED` can reach service/terminal logs and forge output. Sanitize the value or require an exact version parse before returning the warning.
  - confidence: 0.82
