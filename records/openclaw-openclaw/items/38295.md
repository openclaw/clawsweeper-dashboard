  - body: Current main intentionally keeps the generated LaunchAgent throttle at 10s so crash loops back off while explicit supervised restarts recover promptly.   - body: Using `snapshot.hash` first means unchanged root config bytes suppress later invalid-reload warnings even when include/env/plugin-derived `snapshot.issues` change.   - body: Current launchd runtime resolves supervisor logs under `~/Library/Logs/openclaw`, but this docs addition points CLI LaunchAgent users at `$OPENCLAW_STATE_DIR/logs`. That sends users to the wrong files when diagnosing or truncating macOS service log growth.
  - confidence: 0.9
