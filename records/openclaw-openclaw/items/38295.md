- **[P2] Include diagnostics in the invalid snapshot fingerprint:** `src/gateway/config-reload.ts:148`
  - body: `snapshot.hash` is the root or persisted config byte hash, so validation diagnostics can change while this value stays the same, for example through includes, env resolution, or plugin metadata. Because the PR prefers that hash over `issues`, the reloader can suppress the new warning operators need to see; fold the formatted issue details into the fingerprint instead of using the hash alone.
  - confidence: 0.87
