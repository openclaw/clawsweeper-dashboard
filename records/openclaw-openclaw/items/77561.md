## Summary

Keep open: current main still skips the normal tool-result guard and its mid-turn precheck for `ownsCompaction` context engines, while Pi auto-compaction is disabled for that same owned-engine path.

## What This Changes

The issue reports that Codex-owned context engines can still miss mid-turn tool-loop compaction protection and overflow context under heavy tool-result load.

## Best Possible Solution

Add an owned-context-engine-safe mid-turn pressure signal that feeds the existing truncation/compaction recovery path without re-enabling Pi auto-compaction or bypassing context-engine assembly.

## Reproduction Assessment

Yes for the source-level bypass: current main still routes `ownsCompaction` sessions to a hook that lacks the normal mid-turn guard. I did not establish the full live Codex overflow timing from the report.

## Solution Assessment

No for blindly installing the normal guard on every session; the narrower maintainable fix is to add a last-resort owned-engine mid-turn check that uses the existing recovery route.

## Repair Work Prompt

Fix openclaw/openclaw#77561. Current main skips `installToolResultContextGuard` for `ownsCompaction: true` context engines and uses `installContextEngineLoopHook`, which only runs context-engine `afterTurn`/`ingest`/`assemble`; Pi auto-compaction is disabled for the same path. Add a focused last-resort mid-turn pressure check for owned context-engine sessions that routes through the existing mid-turn/preflight recovery machinery (`MidTurnPrecheckSignal` / `onMidTurnPrecheck` / existing truncation and compaction retry paths) without re-enabling Pi auto-compaction, adding a new config option, or bypassing context-engine assembly semantics. Update the current test that asserts mid-turn precheck is absent from the owned hook, and add regression coverage for `ownsCompaction: true` with `agents.defaults.compaction.midTurnPrecheck.enabled: true` producing recovery. Add a user-facing changelog fix entry. Stop and ask for maintainer guidance if the implementation requires a public context-engine API change or a default policy change for all sessions.
