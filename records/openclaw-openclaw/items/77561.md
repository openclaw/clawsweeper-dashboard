## Summary

Keep open: current main still excludes `ownsCompaction` sessions from the configured tool-result mid-turn precheck while Pi auto-compaction is disabled for the same sessions; source and tests show the prior #71325 fix covers pre-prompt safeguards, not this tool-loop gap.

## What This Changes

The issue asks OpenClaw to protect Codex/`ownsCompaction` tool loops from mid-turn context overflow instead of bypassing the normal tool-result guard and Pi auto-compaction.

## Best Possible Solution

Add an owning-engine-aware mid-turn pressure check that routes through existing precheck recovery or active engine compaction while preserving context-engine ownership semantics.

## Reproduction Assessment

Yes for the source-level bypass: with `ownsCompaction: true`, current main wires the context-engine loop hook instead of the normal tool-result mid-turn precheck, and a regression test asserts that exclusion. I did not reproduce the full live Codex overflow loop.

## Solution Assessment

No as proposed if it means installing the normal guard for every owned engine without regard for ownership. The maintainable fix is a narrow owned-engine fallback that detects mid-turn pressure and hands off to the existing recovery or engine compaction path.
