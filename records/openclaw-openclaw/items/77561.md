## Summary

Keep open: current main still skips the normal tool-result guard for `ownsCompaction` context engines, and the owned-engine loop hook does not run the same mid-turn pressure precheck or hard overflow throw. The prior #71325 release note covers pre-prompt/windowing safeguards, but this issue reports a distinct tool-loop gap that remains source-reproducible.

## What This Changes

Report that Codex/`ownsCompaction` sessions can miss mid-turn tool-result compaction protection and overflow during long tool loops.

## Best Possible Solution

Add a generic last-resort mid-turn pressure check for `ownsCompaction` sessions that routes through the existing precheck/overflow recovery machinery while preserving context-engine ownership semantics.

## Reproduction Assessment

Yes for the source-level guard bypass: current main deterministically routes `ownsCompaction: true` sessions away from the normal tool-result guard, and a test asserts that mid-turn precheck is not wired into that hook. No live Codex overflow run was executed in this read-only review.

## Solution Assessment

Unclear as proposed if the fix is to install the normal guard for every session, because that could conflict with owned context-engine semantics. The narrow maintainable fix is an owned-engine last-resort mid-turn pressure signal that reuses the existing recovery route and adds regression coverage for `ownsCompaction: true`.
