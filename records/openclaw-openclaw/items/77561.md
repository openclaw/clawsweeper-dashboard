## Summary

Keep open: current main still routes `ownsCompaction` sessions away from the normal tool-result guard and the opt-in mid-turn precheck, so the owned-engine tool-loop protection gap remains source-reproducible. The newer Codex sidecar rotation work is adjacent, but it does not add a last-resort mid-turn guard inside the owned-engine hook.

## What This Changes

The issue asks OpenClaw to protect `ownsCompaction`/Codex sessions from mid-turn tool-result context overflow instead of bypassing the guard.

## Best Possible Solution

Add a last-resort mid-turn pressure signal for `ownsCompaction` sessions that routes through existing recovery or active-engine compaction without directly rewriting engine-owned context.

## Reproduction Assessment

Yes for the source-level issue: current main still skips the normal tool-result guard and opt-in mid-turn precheck for `ownsCompaction: true`, and a regression test asserts that bypass. I did not run a live long-loop Codex overflow reproduction.

## Solution Assessment

No to the exact proposal of installing the normal guard for every session without qualification, because that can conflict with owned context-engine semantics. The narrower maintainable fix is a last-resort owned-engine mid-turn pressure check that signals the existing compaction/retry path.

Fix https://github.com/openclaw/openclaw/issues/77561. Current main installs `installContextEngineLoopHook` instead of `installToolResultContextGuard` when `activeContextEngine.info.ownsCompaction === true`, and that owned hook does not run `midTurnPrecheck` or a hard overflow signal during tool loops. Implement a last-resort mid-turn pressure check for owned context-engine sessions while preserving `ownsCompaction` semantics: do not directly truncate or rewrite engine-owned context unless the existing recovery path chooses that; prefer reusing `MidTurnPrecheckSignal`/`onMidTurnPrecheck` or the active engine compaction/retry path. Keep non-owned behavior unchanged. Update the regression test that currently asserts `midTurnPrecheck` is absent for owned hooks, add focused hook coverage, update docs only if public semantics change, and add a `CHANGELOG.md` fix entry unless maintainers decide this is internal-only.
