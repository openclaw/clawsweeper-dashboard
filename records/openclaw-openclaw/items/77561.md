## Summary

Keep open: current main still excludes `ownsCompaction: true` sessions from the normal tool-result guard and opt-in mid-turn precheck, while the owned-engine hook only delegates to the context engine and does not raise the existing mid-turn recovery signal. The Codex sidecar rotation fix is useful adjacent work, but it does not close this core tool-loop guard gap.

## What This Changes

The issue asks OpenClaw to protect Codex/context-engine-owned sessions from mid-turn context overflow during long tool loops.

## Best Possible Solution

Add a narrow owned-engine mid-turn pressure signal that reuses the existing recovery path or active context-engine compaction without directly truncating engine-owned context.

## Reproduction Assessment

Yes for the source-level issue: current main still routes `ownsCompaction: true` sessions away from the normal tool-result guard and the test suite asserts `midTurnPrecheck` is absent from that hook. I did not live-reproduce the reported heavy Codex tool-loop overflow.

## Solution Assessment

No to the exact proposal of unconditionally installing the normal guard for every owned-engine session, because that can violate the `ownsCompaction` contract. The safer fix is a small owned-engine pressure check that raises the existing mid-turn recovery signal or invokes active engine compaction.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/77561. Current main routes `ownsCompaction: true` sessions to `installContextEngineLoopHook`, while the normal `installToolResultContextGuard` is the only path that receives `midTurnPrecheck` and raises `MidTurnPrecheckSignal`. Add a last-resort mid-turn pressure signal for context-engine-owned sessions that preserves the `ownsCompaction` contract: do not simply install the normal guard over owned engines, do not destructively truncate inside the owned hook, and do not add a new user-facing config option. Reuse the existing `agents.defaults.compaction.midTurnPrecheck.enabled` setting and the existing `handleMidTurnPrecheckRequest` recovery path, or invoke active engine compaction through the existing owned-engine recovery surface. Update regression coverage so owned-engine sessions with `midTurnPrecheck.enabled: true` no longer silently lack mid-turn pressure handling, while non-owned sessions keep the existing guard behavior. Add a changelog entry because this is user-visible agent reliability work.
