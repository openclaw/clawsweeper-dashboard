## Summary

Keep open: current main still routes `ownsCompaction` sessions away from the normal mid-turn tool-result precheck, and the owned-engine loop hook does not raise the existing mid-turn recovery signal. The related Codex sidecar rotation work is useful adjacent context-window hardening, but it does not close this core owned-compaction gap.

## What This Changes

The issue reports that Codex/`ownsCompaction` sessions bypass mid-turn tool-result context protection and can lose active context during heavy tool loops.

## Best Possible Solution

Add an owned-engine-safe mid-turn pressure signal that preserves the `ownsCompaction` contract while routing overflow through existing recovery or context-engine compaction, without adding a new config knob.

## Reproduction Assessment

Yes for the source-level issue: current main still routes `ownsCompaction: true` sessions away from the mid-turn precheck wiring, and current tests explicitly assert that exclusion. I did not run a live heavy Codex tool-loop session.

## Solution Assessment

No to the exact proposal of unconditionally installing the normal guard for all sessions, because that can conflict with engine-owned compaction semantics. The maintainable fix is a narrow owned-engine pressure check that raises the existing mid-turn recovery signal or invokes the active engine's compaction path.

Fix https://github.com/openclaw/openclaw/issues/77561. Current main routes `ownsCompaction` sessions through `installContextEngineLoopHook`, while only `installToolResultContextGuard` receives `midTurnPrecheck`; `shouldDisablePiAutoCompaction` also disables Pi internal auto-compaction for owned engines. Add an owned-engine-safe mid-turn pressure path that detects prompt pressure after new tool results and before the next model call, then reuses `MidTurnPrecheckSignal`/`handleMidTurnPrecheckRequest` or the active context-engine compaction path. Update focused tests so `ownsCompaction: true` with `midTurnPrecheck.enabled: true` is covered by the new behavior instead of asserting the option is absent; preserve behavior for sessions without a context engine and for `ownsCompaction !== true`. Add a `CHANGELOG.md` fix entry unless maintainers classify the final patch as internal-only.
