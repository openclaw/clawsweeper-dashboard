## Summary

Keep open: current main still routes `ownsCompaction` sessions away from the normal tool-result guard and its mid-turn precheck, while Pi auto-compaction is disabled for those same sessions. The closed #71325 work covers pre-prompt/windowing safeguards, but this report identifies a separate mid-turn tool-loop gap that remains source-reproducible.

## What This Changes

Request to add mid-turn compaction protection for `ownsCompaction` Codex/context-engine sessions so heavy tool loops do not overflow and restart.

## Best Possible Solution

Add an owned-engine-aware last-resort mid-turn pressure check that triggers existing compaction/recovery without breaking context-engine ownership, and cover it with regression tests for `ownsCompaction: true`.

## Reproduction Assessment

Yes for the source-level bypass: current main routes `ownsCompaction` sessions to `installContextEngineLoopHook`, and a regression test asserts that `midTurnPrecheck` is not wired into that hook. No for the full live 20+ tool-loop overflow/restart sequence, which was not executed in this read-only review.

## Solution Assessment

Unclear as proposed: installing the normal tool-result guard for all sessions may violate owned context-engine semantics. A narrower fix is to add a last-resort mid-turn pressure check inside the owned-engine path and route it through the existing compaction/retry machinery.
