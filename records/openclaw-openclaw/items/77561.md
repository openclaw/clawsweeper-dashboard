## Summary

Keep open: current main still routes `ownsCompaction` sessions away from the normal tool-result guard and opt-in mid-turn precheck. The newer Codex sidecar rotation fix is useful adjacent work, but it does not add the missing owned-engine mid-turn pressure guard.

## What This Changes

The issue requests protecting Codex/context-engine-owned tool loops from mid-turn context overflow even when `ownsCompaction: true` bypasses the normal tool-result guard.

## Best Possible Solution

Add a last-resort mid-turn pressure signal for `ownsCompaction` sessions that uses the existing recovery or active-engine compaction path without directly truncating engine-owned context.

## Reproduction Assessment

Yes for the source-level issue: current main and `v2026.5.7` still route `ownsCompaction: true` sessions away from the normal guard, and current tests assert `midTurnPrecheck` is absent from that hook. I did not run the reported live heavy Codex tool-loop session.

## Solution Assessment

No to the exact proposal of unconditionally installing the normal guard, because that can violate owned-engine semantics. The maintainable fix is a small owned-engine pressure check that raises the existing mid-turn recovery signal or invokes active engine compaction.

## Repair Work Prompt

Fix the owned context-engine mid-turn overflow gap tracked by https://github.com/openclaw/openclaw/issues/77561. Current main installs `installContextEngineLoopHook` for `activeContextEngine.info.ownsCompaction === true`, so those sessions do not get the normal `installToolResultContextGuard` mid-turn precheck even when `agents.defaults.compaction.midTurnPrecheck.enabled` is true. Add a last-resort mid-turn pressure signal for owned-engine sessions that preserves the `ownsCompaction` contract: do not wholesale install the normal truncating guard and do not mutate engine-owned context directly. Prefer routing through the existing `MidTurnPrecheckSignal` / `handleMidTurnPrecheckRequest` recovery path or active engine compaction/retry machinery. Update regression coverage for an `ownsCompaction: true` context engine with midTurnPrecheck enabled, preserving existing non-owned behavior. Add a changelog fix entry if the change is user-visible.
