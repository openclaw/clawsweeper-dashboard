## Summary

Keep open: current main still skips the normal tool-result context guard for `ownsCompaction` context engines, and the owned-engine loop hook does not run the same mid-turn precheck or hard overflow guard. The source gap is narrow and still actionable after the earlier #71325 pre-prompt fix.

## What This Changes

The issue asks OpenClaw to restore mid-turn tool-loop context protection for `ownsCompaction` sessions such as Codex-backed runs.

## Best Possible Solution

Add a generic last-resort mid-turn pressure check for `ownsCompaction` sessions that routes through existing recovery or active-engine compaction, with regression coverage for the owned-engine hook.

## Reproduction Assessment

Yes for the source-level guard bypass: configure an active context engine with `ownsCompaction: true` and `midTurnPrecheck.enabled`, and current main installs the owned-engine loop hook without the normal precheck guard. I did not live-reproduce the full Codex 20+ tool-call overflow.

## Solution Assessment

Unclear as proposed: installing the normal guard for every session may conflict with context-engine ownership. The safer solution is a narrow last-resort owned-engine pressure signal that uses the existing mid-turn recovery path or the active engine's compaction route.

## Repair Work Prompt

Fix issue #77561 on current main. The observable bug is that `ownsCompaction: true` sessions skip `installToolResultContextGuard`, so the normal mid-turn precheck and hard overflow signal are absent during tool loops while Pi auto-compaction is disabled. Implement a narrow last-resort mid-turn pressure check for owned context-engine sessions in the embedded Pi runner. Prefer routing through the existing `MidTurnPrecheckSignal` / `handleMidTurnPrecheckRequest` recovery path or active context-engine compaction; do not simply install the non-owned guard wholesale if that would bypass context-engine ownership semantics. Add regression coverage proving an `ownsCompaction: true` engine can trigger the mid-turn pressure path after new tool results. Update CHANGELOG.md with a single-line user-facing fix entry. Do not add a new config option unless source inspection proves the existing recovery path cannot express the fix.
