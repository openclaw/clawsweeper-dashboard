## Summary

Keep open: current main still routes `ownsCompaction` sessions away from the normal tool-result context guard and its mid-turn precheck, while Pi auto-compaction is disabled for the same ownership mode.

## What This Changes

The issue reports that `ownsCompaction` sessions can miss mid-turn tool-loop compaction protection and overflow during heavy Codex-style tool runs.

## Best Possible Solution

Add a last-resort mid-turn pressure path for owned context-engine PI sessions that routes through the existing compaction/retry recovery without bypassing engine ownership.

## Reproduction Assessment

Yes for the source-level guard bypass, but not for the full live overflow loop. Current main clearly skips the normal tool-result guard for `ownsCompaction: true`, and a test asserts that the mid-turn precheck is not wired into that hook.

## Solution Assessment

No to the issue's broadest proposed fix of installing the normal guard for every session. A safer solution is a narrow owned-engine last-resort pressure signal that reuses the existing precheck recovery path and has regression coverage.
