## Summary

Keep open. The patch targets a real cron transcript-context gap, but it adds session transcript writes for any non-empty cron target session key, including routed peer sessions, and that needs to be reconciled with the active-session lock failure pattern before merge; the submitted proof is also mock/test-only.

## What This Changes

The branch adds a cron announce transcript mirror to `sendCronAnnouncePayloadStrict` and failure-alert delivery, covers the mirror payload in `src/cron/delivery.failure-notify.test.ts`, and adds a changelog fix entry.

## Best Possible Solution

Use one lock-safe cron mirror policy for both direct delivery and the failure-alert helper: mirror custom/non-conflicting session targets, avoid or defer active routed peer-session writes, and prove the final behavior with focused tests plus real channel evidence.

## Reproduction Assessment

No high-confidence live reproduction was run in this review. The source path is clear: current main omits the mirror in `src/cron/delivery.ts`, and the PR adds it, but the submitted proof is limited to mocked unit coverage and local checks.

## Solution Assessment

No, not yet. Passing `mirror` through the durable delivery path is the right mechanism for transcript visibility, but it is too broad until routed peer-session lock safety is handled or explicitly accepted by maintainers.
