## Summary

Keep this PR open for maintainer handling, but do not merge it as-is. The table-conversion direction is useful, yet the branch is stale against current Feishu card delivery on main and would miss the structured-card paths that now carry card-mode outbound and reply delivery.

## What This Changes

The PR adds a Feishu markdown-card helper that parses Markdown pipe tables into native Card JSON 2.0 table elements, delegates buildMarkdownCard to it, and adds unit/integration tests.

## Best Possible Solution

Build a current-main Feishu card table normalization layer that works for both markdown and structured cards, preserves width_mode, protects all supported fenced code blocks, handles table-count fallback deliberately, keeps quoted replies readable, and defines whether streaming cards should normalize tables or fall back to safe markdown/plain text.

## Reproduction Assessment

Yes. Static current-main inspection gives a high-confidence reproduction path: send Feishu text with a Markdown pipe table using renderMode=card or auto table detection, and the active structured-card path wraps it in one markdown element instead of native table elements. The PR branch can also be checked with multiple tables or tilde-fenced table examples to reproduce the introduced regressions.

## Solution Assessment

No. The PR’s basic conversion idea is useful, but hooking only buildMarkdownCard is not the best current-main fix because outbound adapter delivery, reply dispatch, structured headers/notes, quote extraction, and streaming card behavior now sit outside that helper.

  - body: Current main sends renderMode=card and auto table-like outbound text through sendStructuredCardFeishu/buildStructuredCard, not only through buildMarkdownCard.   - body: Current main uses config.width_mode = "fill" and tests assert wide_screen_mode is absent. Returning wide_screen_mode from the new helper would either fail the existing expectations after rebase or regress the current Card JSON 2.0 sizing contract.
  - confidence: 0.88
- **[P2] Protect tilde-fenced code blocks:** `extensions/feishu/src/markdown-card.ts:25`
  - body: The protected-span regex only recognizes backtick fences. Markdown tables inside valid ~~~ fenced code blocks will still be converted into live Feishu table components, corrupting code examples that should remain markdown text.
  - confidence: 0.86
- **[P2] Cap or fall back before table-count rejection:** `extensions/feishu/src/markdown-card.ts:184-186`
  - body: The splitter emits one native table component for every matched Markdown table. Related Feishu reports show cards can fail with the platform table-count limit, so messages containing several tables need a cap or fallback instead of blindly pushing every table element.
  - confidence: 0.84
- **[P2] Extract table text from quoted cards:** `extensions/feishu/src/send.ts:407`
  - body: After this change, outgoing cards can contain native table elements, but current interactive-card extraction only reads text-like elements. Replies quoting a table-only card can therefore lose the table content and fall back to the generic interactive-card text.
  - confidence: 0.82
