## Best Possible Solution

Rebase onto the current Feishu card pipeline, share table-aware rendering with structured card sends, preserve width_mode and quoted-card extraction, and fall back to text when table limits would reject delivery, with redacted live Feishu/Lark proof.

  - confidence: 0.86

  - body: The splitter appends one native table component for every Markdown table. The related Feishu table-limit issue shows multi-table cards can be rejected with 230099/11310, so this needs a count fallback or split strategy before switching card bodies to native tables.
  - confidence: 0.88
