## Summary

Keep open. The bug is source-reproducible on current main and the PR contains useful native-table work, but the branch is not merge-ready: it lacks real Feishu proof and misses current structured-card paths, schema-2.0 config, fence handling, table-limit fallback, and quoted-card extraction.

## What This Changes

The PR adds a Feishu markdown-card helper and tests, then delegates buildMarkdownCard to convert Markdown pipe tables into native Card JSON 2.0 table elements.

## Best Possible Solution

Build one shared Feishu card table layer used by markdown, structured, outbound, reply, and streaming-adjacent fallback paths while preserving width_mode, fence protection, quote extraction, and Feishu table-limit degradation.

## Reproduction Assessment

Yes, at source level. Current main marks Markdown-table content as card-worthy and then sends the relevant card paths as single markdown elements; I did not run a live Feishu send.

## Solution Assessment

No. Native table conversion is a reasonable direction, but this patch is too narrow for current main and needs structured-card coverage, schema config preservation, fence handling, table-limit fallback, quote extraction, and real Feishu proof.

Full review comments:

- **[P2] Wire table conversion through structured card sends:** `extensions/feishu/src/send.ts:407`
  - body: This only changes buildMarkdownCard, but current main routes renderMode=card and auto table-like outbound/reply text through sendStructuredCardFeishu and buildStructuredCard. Those primary card sends would still render pipe tables as raw markdown after rebase.
  - confidence: 0.91
- **[P2] Preserve schema-2.0 width_mode config:** `extensions/feishu/src/markdown-card.ts:232`
  - body: The new helper returns wide_screen_mode, while current schema-2.0 Feishu cards use config.width_mode = "fill" and tests assert wide_screen_mode is absent. Merging this would regress the established card sizing contract.
  - confidence: 0.89
- **[P2] Protect tilde-fenced code blocks:** `extensions/feishu/src/markdown-card.ts:25`
  - body: The protected-span regex only recognizes backtick fences. OpenClaw's markdown parser also treats ~~~ as a fence, so table-shaped examples inside tilde fences can still be converted into live Feishu tables and corrupt rendered snippets.
  - confidence: 0.84
- **[P2] Add fallback before exceeding Feishu table limits:** `extensions/feishu/src/markdown-card.ts:185-186`
  - body: The splitter emits one native table component for every matched Markdown table with no component-count cap or fallback. Feishu documents a five-table-per-card limit, so multi-table replies can become failed card sends instead of delivered text.
  - confidence: 0.88
- **[P2] Include table elements in quoted-card extraction:** `extensions/feishu/src/markdown-card.ts:185-186`
  - body: After this change, outgoing cards can contain tag: "table" elements, but current interactive-card extraction reads only text-like elements. Replies to a table-only card can lose that context and fall back to generic interactive-card text.
  - confidence: 0.82
