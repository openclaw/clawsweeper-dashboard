## Summary

Keep open: the reported Feishu table rendering problem is still source-reproducible on current main, but this PR is stale against the active structured-card and streaming-card paths and introduces merge-blocking regressions.

## What This Changes

The PR adds a Feishu markdown-card helper that converts Markdown pipe tables into native Card JSON 2.0 table elements, wires buildMarkdownCard to it, and adds tests.

## Best Possible Solution

Build a current-main Feishu card table normalization and fallback layer that covers markdown cards, structured cards, streaming cards, and quoted-card extraction while preserving width_mode.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: send Feishu text containing a Markdown pipe table with renderMode=card or auto, and current main emits raw markdown in card elements on the active card paths.

## Solution Assessment

No. The conversion idea is useful, but this PR hooks only buildMarkdownCard while current main primarily uses buildStructuredCard/sendStructuredCardFeishu and also needs width, quote extraction, streaming, and table-limit handling.

Full review comments:

- **[P2] Wire table conversion through structured cards:** `extensions/feishu/src/send.ts:407`
  - body: Current main sends renderMode=card and auto table-like outbound/reply text through sendStructuredCardFeishu/buildStructuredCard. Hooking only buildMarkdownCard leaves the primary card path rendering pipe tables as raw markdown after rebase.
  - confidence: 0.91

  - body: Current schema-2.0 Feishu cards use config.width_mode = "fill", and tests assert wide_screen_mode is absent. Returning wide_screen_mode from the new helper regresses that card sizing contract.
  - confidence: 0.88
- **[P2] Protect tilde-fenced code blocks:** `extensions/feishu/src/markdown-card.ts:25`
  - body: The protected-span regex only recognizes backtick fences, so Markdown tables inside valid ~~~ fenced code blocks will still be converted into live Feishu table components and corrupt code examples.
  - confidence: 0.86
- **[P2] Add a table-count fallback before emitting native tables:** `extensions/feishu/src/markdown-card.ts:184-186`
  - body: The splitter emits one native table component for every matched Markdown table. Related Feishu reports show cards can fail with the platform table-count limit, so multi-table messages need a cap or fallback instead of blindly pushing every table element.
  - confidence: 0.84
- **[P2] Extract native table text when quoting card replies:** `extensions/feishu/src/markdown-card.ts:126`
  - body: After this change, outgoing cards can contain native table elements, but current interactive-card extraction only reads text-like elements. Replies quoting a table-only card can lose that context and fall back to generic interactive-card text.
  - confidence: 0.82
