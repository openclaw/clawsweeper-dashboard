- **[P1] Preserve the schema-2.0 width config:** `extensions/feishu/src/markdown-card.ts:231-233`
  - body: This new builder returns `config: { wide_screen_mode: true }`, but current main already fixed Feishu schema-2.0 cards to use `width_mode: "fill"` because the legacy key caused parse-card failures. Rebase the helper on the current card config so this does not break card sends again.
  - confidence: 0.91

  - body: Routing only `buildMarkdownCard()` through the table builder misses the current main Feishu reply path, which sends card-mode replies via `sendStructuredCardFeishu()` and `buildStructuredCard()`. After rebase, table-heavy normal replies would still render as raw pipe text unless the shared structured-card builder also handles tables.
  - confidence: 0.86
- **[P2] Recover table text when quoted cards are read:** `extensions/feishu/src/markdown-card.ts:184-186`
  - body: The patch starts sending `tag: "table"` elements, but quoted-card extraction only reads text-like elements, so replies to table-only cards can fall back to `[Interactive Card]` and lose useful prior context. Add table-element serialization in the interactive-card parser before introducing native table cards.
  - confidence: 0.84
- **[P2] Protect tilde-fenced table examples:** `extensions/feishu/src/markdown-card.ts:20-25`
  - body: The protected-span regex only recognizes backtick fences, so valid `~~~` fenced code blocks containing table examples can still be rewritten into live Feishu tables. That regresses code/example rendering compared with the previous single-markdown card.
  - confidence: 0.82
