## Summary

Keep open: current main still has the Feishu card-table rendering gap, but this PR is not mergeable as-is because it targets an old buildMarkdownCard-only path and misses current structured/streaming card behavior plus several table-rendering edge cases.

## What This Changes

The PR adds a Feishu markdown-card helper that converts Markdown pipe tables into native Card JSON table elements, wires buildMarkdownCard through it, and adds tests.

## Best Possible Solution

Build one current-main Feishu card table normalization layer that covers markdown cards, structured static cards, streaming cards, and quoted-card extraction while preserving width_mode and falling back safely when Feishu table limits are hit.

## Reproduction Assessment

Yes. Source inspection on current main shows renderMode=card or auto table-like text selects Feishu card paths, and the builders place the original pipe table text into markdown card elements without normalization.

## Solution Assessment

No. Native table conversion is a useful direction, but this patch is not the narrowest maintainable fix because it hooks an older helper path and omits structured/streaming cards, width_mode, quote extraction, code-fence parity, and table-limit fallback.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Wire table conversion through structured cards:** `extensions/feishu/src/send.ts:407`
  - body: Current main sends renderMode=card and auto table-like outbound/reply text through sendStructuredCardFeishu/buildStructuredCard. Delegating only buildMarkdownCard leaves the primary card path rendering raw pipe tables after rebase.
  - confidence: 0.91
- **[P2] Preserve the schema-2.0 width config:** `extensions/feishu/src/markdown-card.ts:231-233`
  - body: Current main cards use config.width_mode = "fill" and tests assert wide_screen_mode is absent. Returning the legacy config here regresses the card sizing contract.
  - confidence: 0.88
- **[P2] Protect tilde fenced code blocks:** `extensions/feishu/src/markdown-card.ts:25`
  - body: The protected-span regex only recognizes backtick fences, so valid ~~~ fenced code blocks containing table-shaped lines are still converted into live table components and code snippets get corrupted.
  - confidence: 0.86
- **[P2] Add a table-count fallback before emitting tables:** `extensions/feishu/src/markdown-card.ts:184-186`
  - body: The splitter emits a native Feishu table for every Markdown table. Feishu rejects cards that exceed its table-count limit, so multi-table responses need a cap or fallback instead of blindly pushing every table element.
  - confidence: 0.84
- **[P2] Extract table elements when quoting card replies:** `extensions/feishu/src/markdown-card.ts:184-186`
  - body: After this helper emits tag:"table" elements, interactive-card text extraction still reads only text-like elements. Replies to table-only cards can lose the original table context and fall back to generic card text.
  - confidence: 0.82
