## Summary

Keep open. The PR targets a real Feishu card-table rendering gap, but it is still incomplete against current main's structured/reply/outbound/streaming card paths, has blocking correctness issues, and lacks required real Feishu after-fix proof.

## What This Changes

The branch adds a Feishu markdown-card helper that converts Markdown pipe tables into Card JSON 2.0 table elements, delegates buildMarkdownCard to it, and adds Feishu tests.

## Best Possible Solution

Build one coordinated Feishu card table layer that covers markdown, structured, reply, outbound, and streaming paths while preserving width_mode and falling back to text when native tables exceed platform limits.

## Reproduction Assessment

Yes, at source level. Current main emits pipe tables unchanged in Feishu card markdown paths, Feishu's Markdown docs do not list pipe-table syntax, and no live Feishu send was performed in this read-only review.

## Solution Assessment

No. Native table conversion is a useful direction, but the current patch misses current structured/reply/outbound/streaming paths and does not yet handle width_mode, table-count fallback, or quoted-card extraction safely.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Wire table conversion through structured cards:** `extensions/feishu/src/send.ts:408`
  - body: This only changes buildMarkdownCard, but current main routes renderMode=card and auto table-like outbound/reply text through sendStructuredCardFeishu/buildStructuredCard. Those primary card sends will still render pipe tables as raw markdown after rebase.
  - confidence: 0.91
- **[P2] Preserve the Card JSON 2.0 width config:** `extensions/feishu/src/markdown-card.ts:231-232`
  - body: Current schema-2.0 Feishu cards use config.width_mode = "fill" and tests assert wide_screen_mode is absent. Returning wide_screen_mode from the new helper regresses that card sizing contract.
  - confidence: 0.89
- **[P2] Protect tilde-fenced code blocks:** `extensions/feishu/src/markdown-card.ts:25`
  - body: The protected-span regex only recognizes backtick fences, so table-shaped content inside valid ~~~ fenced Markdown blocks can still be converted into live Feishu table elements and corrupt code examples.
  - confidence: 0.84
- **[P2] Add a table-count fallback before emitting native tables:** `extensions/feishu/src/markdown-card.ts:183-186`
  - body: The splitter emits one native table component for every matched Markdown table. Feishu's table component docs cap a single card at five table components, so multi-table responses can start failing card creation unless the builder caps or falls back.
  - confidence: 0.9
- **[P2] Extract native table text when quoting card replies:** `extensions/feishu/src/markdown-card.ts:183`
  - body: After this change, outgoing cards can contain tag:"table" elements, but current interactive-card extraction reads only text-like elements. Replies to a table-only card can lose that context and fall back to generic interactive-card text.
  - confidence: 0.82
