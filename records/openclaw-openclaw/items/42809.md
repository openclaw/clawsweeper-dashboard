## Summary

Keep open. The PR targets a real Feishu card-table bug, but the branch still has blocking correctness gaps against current main: it only wires the legacy markdown-card helper, regresses the schema 2.0 width config, misses valid tilde fences, can still exceed Feishu native table limits, and lacks required real Feishu/Lark proof.

## What This Changes

The branch adds a Feishu markdown-card helper and tests that convert Markdown pipe tables into native Card JSON 2.0 table elements, then delegates `buildMarkdownCard` to that helper.

## Best Possible Solution

Implement one coordinated Feishu card table/fallback layer used by markdown, structured, reply, outbound, streaming-close, and quoted-card paths, preserving `width_mode` and platform table limits, then verify with redacted real Feishu/Lark proof.

## Reproduction Assessment

Yes, source-level. Current main detects Markdown pipe tables as card-worthy and sends the main reply/outbound card paths through markdown-only structured cards; I did not run a live Feishu send in this read-only review.

## Solution Assessment

No. Native table conversion is a reasonable ingredient, but this PR is not the best complete fix until it covers structured-card paths, preserves schema 2.0 config, handles table/code-fence limits, extracts quoted table text, and includes real Feishu proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Wire table conversion through structured card sends:** `extensions/feishu/src/send.ts:407`
  - body: This only changes `buildMarkdownCard`, but current main routes primary reply and outbound card text through `sendStructuredCardFeishu`/`buildStructuredCard`. After rebase, those card sends would still emit raw pipe tables inside markdown elements.
  - confidence: 0.92
- **[P2] Keep schema 2.0 width_mode in generated cards:** `extensions/feishu/src/markdown-card.ts:232`
  - body: The new helper returns `config.wide_screen_mode`, while current schema 2.0 Feishu card tests require `config.width_mode = "fill"` and assert `wide_screen_mode` is absent. Merging this would regress the established card config contract.
  - confidence: 0.9
- **[P2] Protect tilde fences before converting tables:** `extensions/feishu/src/markdown-card.ts:25`
  - body: The protected-span regex only recognizes backtick fences. Markdown tables inside valid `~~~` fenced code blocks would still be converted into live Feishu table components, corrupting snippets that previously stayed as markdown text.
  - confidence: 0.86
- **[P2] Fallback before emitting too many table components:** `extensions/feishu/src/markdown-card.ts:185-186`
  - body: The splitter appends one native table component for every Markdown table. Feishu documents a five-table-per-card limit, so multi-table replies can still fail instead of falling back to deliverable text.
  - confidence: 0.87
- **[P2] Extract table elements when quoting native table cards:** `extensions/feishu/src/markdown-card.ts:185-186`
  - body: After this change, outgoing cards can contain `tag: "table"` elements, but current quoted interactive-card extraction only reads text-like elements. Replies to a table-only card can lose the table context and fall back to generic interactive-card text.
  - confidence: 0.82
