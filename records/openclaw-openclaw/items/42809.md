## Summary

Keep open: the Feishu card-table bug is real, but this branch is incomplete against current main because it only wires `buildMarkdownCard` while normal card delivery now flows through structured and streaming card paths, and the diff still has correctness gaps around width config, fenced code, platform limits, and quoted-card extraction.

## What This Changes

The PR adds a Feishu `markdown-card` helper that parses Markdown pipe tables into native Card JSON 2.0 table elements, delegates `buildMarkdownCard` to it, and adds unit/integration tests.

## Best Possible Solution

Add a shared Feishu card table-normalization and fallback layer used by markdown, structured, reply, and streaming card flows while preserving schema-2.0 width config and quote extraction.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Markdown pipe tables trigger card mode, then current main sends normal outbound and reply cards through `sendStructuredCardFeishu`/`buildStructuredCard`, which emits a single markdown element.

## Solution Assessment

No. The conversion idea is useful, but the PR's `buildMarkdownCard`-only hook is not the best current-main fix because structured cards, streaming cards, quote extraction, width config, and table-limit fallback sit outside that helper.

## Review Findings

Overall confidence: 0.91

Full review comments:

- **[P2] Wire table conversion through structured cards:** `extensions/feishu/src/send.ts:407`
  - body: Current main sends normal `renderMode=card` and auto table-like outbound/reply text through `sendStructuredCardFeishu`/`buildStructuredCard`, not only `buildMarkdownCard`. After rebase, this hook leaves the primary card path rendering pipe tables as raw markdown, so the reported behavior remains unfixed.
  - confidence: 0.91
- **[P2] Preserve current Card JSON width config:** `extensions/feishu/src/markdown-card.ts:231-232`
  - body: Current main uses schema-2.0 `config.width_mode = "fill"`, and tests assert `wide_screen_mode` is absent. Returning `wide_screen_mode` from the new helper regresses the current card sizing contract after rebase.
  - confidence: 0.88
- **[P2] Protect tilde-fenced code blocks:** `extensions/feishu/src/markdown-card.ts:25`
  - body: The protected-span regex only recognizes backtick fences. Markdown tables inside valid `~~~` fenced code blocks will still be converted into live Feishu table components, corrupting code examples that should remain markdown text.
  - confidence: 0.86
- **[P2] Fall back before table-count rejection:** `extensions/feishu/src/markdown-card.ts:184-186`
  - body: The splitter emits one native table component for every matched Markdown table. Related Feishu reports show cards can fail with the platform table-count limit, so multi-table messages need a cap or fallback instead of blindly pushing every table element.
  - confidence: 0.84
- **[P2] Extract table text when quoting card replies:** `extensions/feishu/src/send.ts:407`
  - body: After this change, outgoing cards can contain native `table` elements, but current interactive-card extraction reads only text-like elements. Replies quoting a table-only card can therefore lose the useful table content and fall back to generic interactive-card text.
  - confidence: 0.82
