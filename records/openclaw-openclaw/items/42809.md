## Summary

Keep open: current main still has the Feishu table-rendering bug, but this PR is not merge-ready after current-main changes because it only hooks buildMarkdownCard and misses structured/streaming card paths while adding native-table regressions around width config, tilde fences, table-count limits, and quoted card extraction.

## What This Changes

The PR adds a Feishu markdown-card helper that parses Markdown pipe tables into native Card JSON 2.0 table elements, wires buildMarkdownCard to it, and adds unit/integration tests.

## Best Possible Solution

Build a current-main Feishu card table normalization and fallback layer that covers structured cards, markdown cards, streaming cards, and quoted-card extraction while preserving width_mode and defining table-limit behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: send Feishu text with a Markdown pipe table using renderMode=card or auto, and current main routes the active card path into a single markdown element instead of native table elements.

## Solution Assessment

No. The conversion idea is useful, but hooking only buildMarkdownCard is not the narrowest maintainable current-main fix because structured card delivery, streaming cards, quote extraction, width config, and platform table limits sit outside that helper.

- **[P2] Wire table conversion through structured cards:** `extensions/feishu/src/send.ts:407`
  - body: Current main sends renderMode=card and auto table-like outbound/reply text through sendStructuredCardFeishu/buildStructuredCard, not only buildMarkdownCard.   - confidence: 0.91
- **[P2] Preserve current Card JSON width config:** `extensions/feishu/src/markdown-card.ts:231-232`
  - body: Current main uses config.width_mode = "fill" for schema-2.0 cards and tests assert wide_screen_mode is absent. Returning wide_screen_mode from the new helper would regress the current card sizing contract after rebase.
  - confidence: 0.88
- **[P2] Protect tilde-fenced code blocks:** `extensions/feishu/src/markdown-card.ts:25`
  - body: The protected-span regex only recognizes backtick fences. Markdown tables inside valid ~~~ fenced code blocks will still be converted into live Feishu table components, corrupting code examples that should remain markdown text.
  - confidence: 0.86
- **[P2] Fall back before table-count rejection:** `extensions/feishu/src/markdown-card.ts:184-186`
  - body: The splitter emits one native table component for every matched Markdown table. Related Feishu reports show cards can fail with the platform table-count limit, so multi-table messages need a cap or fallback instead of blindly pushing every table element.
  - confidence: 0.84
- **[P2] Extract table text when quoting card replies:** `extensions/feishu/src/send.ts:407`
  - body: After this change, outgoing cards can contain native table elements, but current interactive-card extraction reads only text-like elements. Replies quoting a table-only card can therefore lose the table content and fall back to generic interactive-card text.
  - confidence: 0.82
