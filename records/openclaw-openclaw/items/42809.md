## Summary

Keep open: current main still lacks native Feishu table conversion, but this PR is not merge-ready because it lacks real Feishu proof and still misses primary structured-card paths, current width config, fence protection, table-limit fallback, and quoted-card extraction.

## What This Changes

The PR adds a Feishu markdown-card helper and tests to convert Markdown pipe tables into native Card JSON 2.0 table elements, then delegates buildMarkdownCard to that helper.

## Best Possible Solution

Build one shared Feishu card table layer used by markdown, structured, outbound, reply, and streaming paths while preserving width_mode, protecting code fences, extracting table text for quotes, and falling back when Feishu table limits are hit.

## Reproduction Assessment

Yes, at source level. Current main keeps pipe tables as markdown in Feishu card builders and routes primary auto/card outbound and reply flows through structured cards that do not convert tables; I did not perform a live Feishu send.

## Solution Assessment

No. Native table conversion is a reasonable direction, but this PR is too narrow for current main and needs width-config preservation, table-limit fallback, quote extraction, fence handling, and real Feishu proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Wire table conversion through structured card sends:** `extensions/feishu/src/send.ts:407`
  - body: This only changes buildMarkdownCard, but current main sends renderMode=card and auto table-like outbound/reply text through sendStructuredCardFeishu and buildStructuredCard. Those primary card sends would still render pipe tables as raw markdown after rebase.
  - confidence: 0.91
- **[P2] Preserve schema-2.0 width_mode config:** `extensions/feishu/src/markdown-card.ts:232`
  - body: The new helper returns wide_screen_mode, while current schema-2.0 Feishu cards use config.width_mode = "fill" and tests assert wide_screen_mode is absent. Merging this would regress the established card sizing contract.
  - confidence: 0.89
- **[P2] Protect tilde-fenced code blocks:** `extensions/feishu/src/markdown-card.ts:25`
  - body: The protected-span regex only recognizes backtick fences. Markdown also permits ~~~ fences, so table-shaped code examples in tilde fences can still be converted into live Feishu tables and corrupt rendered snippets.
  - confidence: 0.84
- **[P2] Add fallback before exceeding Feishu table limits:** `extensions/feishu/src/markdown-card.ts:186`
  - body: The splitter emits one native table component for every matched Markdown table with no component-count cap or fallback. Related Feishu reports show multi-table cards can fail with 230099/11310, so this can turn readable raw text into a failed send.
  - confidence: 0.88
- **[P2] Include table elements in quoted-card extraction:** `extensions/feishu/src/markdown-card.ts:186`
  - body: After this change, outgoing cards can contain tag: "table" elements, but current interactive-card extraction reads only text-like elements. Replies to a table-only card can lose that context and fall back to generic interactive-card text.
  - confidence: 0.82
