## Summary

Keep this PR open but do not merge it as-is. The reported Feishu card table problem is real at source level, but the branch only rewires `buildMarkdownCard` while current main sends the main reply/outbound card flows through structured and streaming card paths, and the patch still lacks real Feishu proof plus several runtime safeguards.

## What This Changes

The PR adds a Feishu markdown-card helper that converts Markdown pipe tables into Card JSON 2.0 table elements, delegates `buildMarkdownCard` to it, and adds Feishu tests.

## Best Possible Solution

Build one shared Feishu card table layer used by markdown, structured, outbound, reply, and streaming paths while preserving `width_mode`, extracting table text for quotes, and falling back when Feishu table limits are hit.

## Reproduction Assessment

Yes, at source level: current main emits pipe tables unchanged in Feishu card markdown/structured paths, and Feishu's markdown docs list a syntax subset without standard pipe tables. I did not perform a live Feishu send in this read-only review.

## Solution Assessment

No. Native table conversion is a reasonable direction, but this PR is too narrow for current main and needs width-config preservation, table-limit fallback, quoted-reply extraction, and real Feishu proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Wire table conversion through structured card sends:** `extensions/feishu/src/send.ts:407`
  - body: This only changes `buildMarkdownCard`, but current main routes renderMode=card and auto table-like outbound/reply text through `sendStructuredCardFeishu` and `buildStructuredCard`. Those primary card sends would still render pipe tables as raw markdown after rebase.
  - confidence: 0.91
- **[P2] Preserve schema-2.0 width_mode config:** `extensions/feishu/src/markdown-card.ts:232`
  - body: The new helper returns `wide_screen_mode`, while current schema-2.0 Feishu cards use `config.width_mode = "fill"` and tests assert `wide_screen_mode` is absent. Merging this would regress the established card sizing contract.
  - confidence: 0.89
- **[P2] Protect tilde-fenced code blocks:** `extensions/feishu/src/markdown-card.ts:25`
  - body: The protected-span regex only recognizes backtick fences, so valid `~~~` fenced Markdown blocks containing table-shaped text can still be converted into live Feishu table elements and corrupt code examples.
  - confidence: 0.84
- **[P2] Add a fallback before exceeding Feishu table limits:** `extensions/feishu/src/markdown-card.ts:186`
  - body: The splitter emits one native `table` component for every matched Markdown table. Feishu's table component docs cap a single card at five table components, so multi-table responses can fail card creation unless this caps or falls back.
  - confidence: 0.9
- **[P2] Include table elements in quoted-card extraction:** `extensions/feishu/src/markdown-card.ts:186`
  - body: After this change, outgoing cards can contain `tag: "table"` elements, but current interactive-card extraction reads only text-like elements. Replies to a table-only card can lose that context and fall back to generic interactive-card text.
  - confidence: 0.82
