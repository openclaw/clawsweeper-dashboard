## Summary

Keep this PR open as a useful Feishu bug-fix candidate, but the current patch is not mergeable: it misses current structured-card send paths, regresses the established schema-2.0 width config, has table/fence edge cases, and lacks real Feishu after-fix proof.

## What This Changes

The PR adds a Feishu markdown-card helper and tests, then delegates `buildMarkdownCard` so Markdown pipe tables become native Feishu Card JSON 2.0 table elements.

## Best Possible Solution

Build one shared Feishu card table layer used by markdown, structured, outbound, and reply card paths while preserving schema-2.0 `width_mode`, code-fence protection, table-count fallback, quoted-card extraction, and real Feishu proof.

## Reproduction Assessment

Yes, at source level: current main marks table-containing Feishu text as card-worthy and emits it as markdown-only card elements, while Feishu’s card table component docs show native table limits that this implementation must respect. I did not run a live Feishu send.

## Solution Assessment

No. Native table conversion is a reasonable direction, but this PR is too narrow for current main and needs structured-card wiring, width config preservation, tilde-fence protection, table-limit fallback, quoted-card extraction, and live proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Wire table conversion through structured card sends:** `extensions/feishu/src/send.ts:407`
  - body: This only changes `buildMarkdownCard`, but current main routes primary outbound and reply card text through `sendStructuredCardFeishu` and `buildStructuredCard`. Those card sends would still render pipe tables as raw markdown after rebase.
  - confidence: 0.91
- **[P2] Preserve schema-2.0 width_mode config:** `extensions/feishu/src/markdown-card.ts:232`
  - body: The new helper returns `wide_screen_mode`, while current schema-2.0 Feishu cards use `config.width_mode = "fill"` and tests assert `wide_screen_mode` is absent. Merging this would regress the established card sizing contract.
  - confidence: 0.89
- **[P2] Protect tilde-fenced code blocks:** `extensions/feishu/src/markdown-card.ts:25`
  - body: The protected-span regex only recognizes backtick fences. OpenClaw’s own code-region handling treats `~~~` as a fence too, so table-shaped examples inside tilde fences can still be converted into live Feishu tables and corrupt rendered snippets.
  - confidence: 0.86
- **[P2] Add fallback before exceeding Feishu table limits:** `extensions/feishu/src/markdown-card.ts:185-186`
  - body: The splitter emits one native table component for every matched Markdown table with no component-count cap or fallback. Feishu documents a five-table-per-card limit, so multi-table replies can become failed card sends instead of delivered text.
  - confidence: 0.88
- **[P2] Include table elements in quoted-card extraction:** `extensions/feishu/src/markdown-card.ts:185-186`
  - body: After this change, outgoing cards can contain `tag: "table"` elements, but current interactive-card extraction reads only text-like elements. Replies to a table-only card can lose that context and fall back to generic interactive-card text.
  - confidence: 0.82
