## Summary

Keep open. The Feishu table rendering bug is source-reproducible and the PR contains useful work, but the branch is not mergeable as-is: it only wires `buildMarkdownCard` while current main primarily uses structured card paths, regresses the schema-2.0 width config, misses tilde-fence/table-limit/quoted-card coverage, and lacks real Feishu proof.

## What This Changes

The PR adds a Feishu markdown-card helper and tests to convert Markdown pipe tables into native Card JSON 2.0 table elements, then delegates `buildMarkdownCard` to that helper.

## Best Possible Solution

Implement a coordinated Feishu card table/fallback layer used by markdown, structured, outbound, reply, and quoted-card paths, preserving `width_mode` plus code-fence and table-limit safeguards, then verify it with redacted real Feishu/Lark proof.

## Reproduction Assessment

Yes, source-level: current main marks Markdown tables as card-worthy and still sends reply/outbound card text through markdown-only structured card elements. I did not run a live Feishu send in this read-only review.

## Solution Assessment

No. Native table conversion is a reasonable direction, but this PR is too narrow for current main and needs structured-card wiring, schema-2.0 config preservation, code-fence/table-limit safeguards, quoted-card extraction, and real Feishu proof.

  - body: This only changes `buildMarkdownCard`, but current main routes primary outbound and reply card text through `sendStructuredCardFeishu` and `buildStructuredCard`. After rebase, those card sends would still render pipe tables as raw markdown.

  - confidence: 0.89
- **[P2] Protect tilde-fenced code blocks:** `extensions/feishu/src/markdown-card.ts:25`
  - body: The protected-span regex only recognizes backtick fences. Markdown tables inside valid `~~~` fenced code blocks would still be converted into live Feishu tables, corrupting rendered snippets.
  - confidence: 0.86
- **[P2] Add table-count fallback before native table emission:** `extensions/feishu/src/markdown-card.ts:185-186`
  - body: The splitter emits one native table component for every matched Markdown table, but Feishu docs cap a card at five table components. Multi-table replies can still fail instead of falling back to deliverable text.
  - confidence: 0.86
- **[P2] Extract native table text from quoted cards:** `extensions/feishu/src/markdown-card.ts:185-186`
  - body: After this change, outgoing cards can contain `tag: "table"` elements, but current interactive-card extraction reads only text-like elements. Replies to a table-only card can lose that context and fall back to generic interactive-card text.
  - confidence: 0.82
