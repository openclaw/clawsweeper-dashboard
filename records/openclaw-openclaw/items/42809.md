## Summary

Keep open. The PR targets a real Feishu card rendering bug, but it is not correct for current main: it only wires `buildMarkdownCard`, while current outbound and reply card flows use `sendStructuredCardFeishu`/`buildStructuredCard`; it also conflicts with the current JSON 2.0 `width_mode` contract, misses tilde-fence protection, and lacks real Feishu proof.

## What This Changes

The PR adds a Feishu `markdown-card` helper that converts Markdown pipe tables into native card table elements, delegates `buildMarkdownCard` to it, and adds focused tests.

## Best Possible Solution

Build one coordinated Feishu card table layer shared by markdown, structured, outbound, reply, and relevant streaming/card extraction paths while preserving `width_mode`, code-fence protection, table-limit fallback, quoted-card extraction, and live Feishu proof.

## Reproduction Assessment

Yes, at source level: current main detects Markdown tables as card-worthy content and still emits them through markdown-only card elements on multiple Feishu card paths. I did not run a live Feishu send in this read-only review.

## Solution Assessment

No. Native table conversion is a reasonable fix direction, but this PR is too narrow for current main and needs structured-card wiring, schema 2.0 config preservation, tilde-fence protection, table-limit coordination, quoted-card extraction, and real Feishu proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Wire table conversion through structured card sends:** `extensions/feishu/src/send.ts:407`
  - body: This only changes `buildMarkdownCard`, but current main routes primary outbound and reply card text through `sendStructuredCardFeishu` and `buildStructuredCard`. After rebase, those card sends would still render pipe tables as raw markdown.
  - confidence: 0.91
- **[P2] Preserve schema 2.0 width_mode config:** `extensions/feishu/src/markdown-card.ts:232`
  - body: The new helper returns `wide_screen_mode`, while current schema 2.0 Feishu cards use `config.width_mode = "fill"` and tests assert `wide_screen_mode` is absent. Merging this would regress the established card sizing contract.
  - confidence: 0.89
- **[P2] Protect tilde-fenced code blocks:** `extensions/feishu/src/markdown-card.ts:25`
  - body: The protected-span regex only recognizes backtick fences. Markdown tables inside valid `~~~` fenced code blocks would still be converted into live Feishu tables, corrupting rendered snippets that should remain code.
  - confidence: 0.86
- **[P2] Fallback before exceeding Feishu table limits:** `extensions/feishu/src/markdown-card.ts:185-186`
  - body: The splitter emits one native table component for every matched Markdown table with no component-count guard or fallback. Feishu documents a five-table-per-card limit, so table-heavy replies can be rejected instead of delivered as text.
  - confidence: 0.86
- **[P2] Include table elements in quoted-card extraction:** `extensions/feishu/src/markdown-card.ts:186`
  - body: After this change, outgoing cards can contain `tag: "table"` elements, but current interactive-card extraction reads only text-like elements. Replies to a table-only card can lose that context and fall back to generic interactive-card text.
  - confidence: 0.82
