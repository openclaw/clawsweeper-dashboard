## Summary

Keep open. The native-table direction addresses a real Feishu card rendering gap, but this branch still misses current structured-card paths, violates the Schema 2.0 config contract, has table/fence/reply-context correctness gaps, and lacks required real Feishu/Lark proof.

## What This Changes

Adds a Feishu markdown-card helper that detects Markdown pipe tables, converts them into Card JSON 2.0 table elements, and wires buildMarkdownCard plus tests to that helper.

## Best Possible Solution

Implement one Feishu card table/fallback layer shared by markdown, structured, reply, outbound, streaming-close, and quoted-card paths, preserving Schema 2.0 width_mode and platform limits, then verify it with redacted real Feishu/Lark proof.

## Reproduction Assessment

Yes, at source level. Current main routes Markdown pipe tables into Feishu card markdown paths, Feishu documents a separate table component and no pipe-table Markdown syntax, and the PR evidence plus related reports show the affected card behavior; I did not run a live Feishu send.

## Solution Assessment

No for the current PR head. Native table conversion is a reasonable ingredient, but this branch needs to cover the current structured-card surfaces, preserve width_mode, respect Feishu table limits, retain quoted-reply context, and include real Feishu/Lark proof.
