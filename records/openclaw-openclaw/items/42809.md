## Summary

Keep open. Current main has not implemented native Feishu card table rendering, and this PR is not merge-ready against current main because it only wires the older `buildMarkdownCard` path while newer structured-card, streaming, quote-extraction, and platform-limit behavior still need a coordinated Feishu card solution.

## What This Changes

The branch adds a Feishu markdown-card helper that parses Markdown pipe tables into Card JSON 2.0 table elements, delegates `buildMarkdownCard` to it, and adds unit/integration tests.

## Best Possible Solution

Build a current-main Feishu card table normalization layer that is shared by markdown and structured static cards, preserves Card JSON 2.0 `width_mode`, has deliberate fallback/table-count behavior, protects both backtick and tilde fenced code blocks, keeps quoted table cards readable, and defines how streaming cards should handle table-heavy final output.

## Reproduction Assessment

Yes. Static current-main inspection gives a high-confidence reproduction path: send Feishu `renderMode=card` or auto-detected table text and it reaches markdown-only card elements through `sendStructuredCardFeishu`/`buildStructuredCard`, while the PR head only changes `buildMarkdownCard`.

## Solution Assessment

No. The PR direction is useful, but this branch is not the best complete fix against current main because it is scoped to an older helper and would miss or regress structured-card, streaming-card, width config, table-count, and quoted-reply behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Wire table conversion through structured cards:** `extensions/feishu/src/send.ts:407`
  - body: Current main sends `renderMode=card` and auto table-like outbound/reply text through `sendStructuredCardFeishu`/`buildStructuredCard`, not only through this helper. A rebase that only delegates `buildMarkdownCard` would still leave the main card path rendering pipe tables as raw markdown.
  - confidence: 0.91
- **[P2] Preserve schema 2.0 width_mode:** `extensions/feishu/src/markdown-card.ts:231-232`
  - body: Current main intentionally uses `config.width_mode = "fill"` and tests assert `wide_screen_mode` is absent. This new builder returns `wide_screen_mode`, so a current-main rebase would either fail the existing expectations or regress the Feishu card sizing contract.
  - confidence: 0.88
- **[P2] Protect tilde-fenced code blocks:** `extensions/feishu/src/markdown-card.ts:25`
  - body: The protected-span regex recognizes only backtick fences. Markdown tables inside valid `~~~` fenced code blocks would still be converted into live Feishu table components, corrupting code examples that should remain markdown text.
  - confidence: 0.86
- **[P2] Cap or fall back when too many native tables are emitted:** `extensions/feishu/src/markdown-card.ts:184-186`
  - body: The splitter emits a native `table` component for every matched Markdown table. Related Feishu reports show cards can fail with the platform table-count limit, so multi-table responses need a cap or fallback instead of blindly adding every table element.
  - confidence: 0.84
- **[P2] Keep table content available for quoted replies:** `extensions/feishu/src/send.ts:407`
  - body: After this change, outgoing cards can contain native `table` elements, but interactive-card extraction only reads text-like elements. Replies quoting a table-only card can therefore lose the table content and fall back to generic interactive-card text.
  - confidence: 0.82
