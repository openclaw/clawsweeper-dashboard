## Summary

Keep this PR open but do not merge the current head. The Feishu card-table problem is real and source-reproducible, but this branch is conflict-marked, lacks real Feishu/Lark proof, and still misses current structured-card paths while regressing the established Schema 2.0 config contract.

## What This Changes

The PR adds a Feishu markdown-card helper that converts Markdown pipe tables into native Card JSON table elements and wires buildMarkdownCard to use it.

## Best Possible Solution

Implement one Feishu card table/fallback layer used by markdown, structured, reply, outbound, streaming-close, and quoted-card paths, preserving Schema 2.0 width_mode, platform limits, and reply context, then verify it with redacted real Feishu/Lark proof.

## Reproduction Assessment

Yes, at source level. Current main routes Markdown pipe tables into Feishu card markdown paths, Feishu's markdown docs do not list pipe-table syntax, and the table component docs provide a separate native table component; I did not run a live Feishu send.

## Solution Assessment

No for the current PR head. Native table conversion is a reasonable ingredient, but the patch needs to be rebased through current structured-card paths, preserve width_mode, respect table/fence limits, extract table text for replies, and include real Feishu proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Wire table conversion through structured card sends:** `extensions/feishu/src/send.ts:407`
  - body: This only changes buildMarkdownCard, but current main routes primary reply and outbound card text through sendStructuredCardFeishu/buildStructuredCard. After rebase, those card sends would still emit raw pipe tables inside markdown elements.
  - confidence: 0.92
- **[P2] Keep schema 2.0 width_mode in generated cards:** `extensions/feishu/src/markdown-card.ts:231-232`
  - body: The new helper returns config.wide_screen_mode, while current Schema 2.0 Feishu card tests require config.width_mode = "fill" and assert wide_screen_mode is absent. Merging this would regress the established card config contract.
  - confidence: 0.9
- **[P2] Protect tilde fences before converting tables:** `extensions/feishu/src/markdown-card.ts:25`
  - body: The protected-span regex only recognizes backtick fences. Markdown tables inside valid ~~~ fenced code blocks would still be converted into live Feishu table components, corrupting snippets that previously stayed as markdown text.
  - confidence: 0.86
- **[P2] Fallback before emitting too many table components:** `extensions/feishu/src/markdown-card.ts:185-186`
  - body: The splitter appends one native table component for every Markdown table. Feishu documents a five-table-per-card limit, so multi-table replies can still fail instead of falling back to deliverable text.
  - confidence: 0.88
- **[P2] Extract native table cards when quoting replies:** `extensions/feishu/src/send.ts:407`
  - body: After this change, outgoing cards can contain tag: "table" elements, but current quoted interactive-card extraction only reads text-like elements. Replies to a table-only card can lose the table context and fall back to generic interactive-card text.
  - confidence: 0.82
