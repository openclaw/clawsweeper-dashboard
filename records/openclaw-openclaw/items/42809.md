## Summary

Keep open. Native Feishu table rendering is a useful direction, but this PR is still conflicting with current main, misses current structured-card delivery paths, regresses the schema-2.0 width config, has remaining table/code-fence/extraction edge cases, and lacks required real Feishu/Lark behavior proof.

## What This Changes

The PR adds a Feishu markdown-card helper that converts Markdown pipe tables into native card table elements, delegates buildMarkdownCard to it, and adds focused tests.

## Best Possible Solution

Build one coordinated Feishu card table layer used by markdown, structured, outbound, and reply card paths while preserving width_mode, code-fence protection, table-limit fallback, quoted-card extraction, and live Feishu proof.

## Reproduction Assessment

Yes, at source level: current main marks table-containing Feishu text as card-worthy and still emits it through markdown-only card elements on multiple paths. I did not run a live Feishu send.

## Solution Assessment

No. Native table conversion is a reasonable direction, but this PR is too narrow for current main and needs structured-card wiring, schema-2.0 config preservation, tilde-fence protection, table-limit coordination, quoted-card extraction, and real Feishu proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Wire table conversion through structured card sends:** `extensions/feishu/src/send.ts:407`
  - body: This only changes buildMarkdownCard, but current main routes primary outbound and reply card text through sendStructuredCardFeishu and buildStructuredCard. Those card sends would still render pipe tables as raw markdown after rebase.
  - confidence: 0.91
- **[P2] Preserve schema-2.0 width_mode config:** `extensions/feishu/src/markdown-card.ts:232`
  - body: The new helper returns wide_screen_mode, while current schema-2.0 Feishu cards use config.width_mode = "fill" and tests assert wide_screen_mode is absent. Merging this would regress the established card sizing contract.
  - confidence: 0.89
- **[P2] Protect tilde-fenced code blocks:** `extensions/feishu/src/markdown-card.ts:25`
  - body: The protected-span regex only recognizes backtick fences. OpenClaw's code-region handling treats ~~~ as a fence too, so table-shaped examples inside tilde fences can still be converted into live Feishu tables and corrupt rendered snippets.
  - confidence: 0.86
- **[P2] Add fallback before exceeding Feishu table limits:** `extensions/feishu/src/markdown-card.ts:185-186`
  - body: The splitter emits one native table component for every matched Markdown table with no component-count guard or fallback. Related Feishu 230099/11310 reports show table-heavy card payloads can be rejected, so multi-table replies can fail instead of delivering text.
  - confidence: 0.86
- **[P2] Include table elements in quoted-card extraction:** `extensions/feishu/src/markdown-card.ts:186`
  - body: After this change, outgoing cards can contain tag: "table" elements, but current interactive-card extraction reads only text-like elements. Replies to a table-only card can lose that context and fall back to generic interactive-card text.
  - confidence: 0.82
