## Summary

Keep open: current main still preserves recent image-bearing turns in provider replay, and https://github.com/openclaw/openclaw/pull/76732 is the open closing PR for the remaining fix.

## What This Changes

The issue asks OpenClaw to prevent current multimodal turns from receiving stale image references from injected context or recent history unless earlier-image comparison is explicit.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/76732, or an equivalent attempt-local pruning change that masks prior replay images on fresh-image turns while preserving the raw transcript and deliberate comparison support.

## Reproduction Assessment

Yes, at source level. Current main preserves recent prior image blocks and media markers in replay while separately loading current prompt images, so an image-heavy topic can still present old and new screenshots together.

## Solution Assessment

Yes for the issue direction, but not yet complete on current main. The narrow maintainable fix is attempt-local current-turn image pruning plus coverage and real Telegram/runtime proof, rather than closing the issue before the linked PR lands.
