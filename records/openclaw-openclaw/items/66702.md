## Summary

Keep open. Current main has partial mitigations for old replay markers and runtime context separation, but it still intentionally preserves recent prior image blocks and media markers in model replay, so the requested current-turn-only media boundary for Telegram vision turns is not implemented.

## What This Changes

The issue asks OpenClaw to prevent stale images from injected context and recent session history from being treated as current-turn vision input, unless the user explicitly requests comparison with earlier images.

## Best Possible Solution

Add an embedded-runner media policy for image-bearing turns: keep current-turn attachments hydratable, mask or omit prior replay image blocks and hydratable media markers by default, and provide a deliberate comparison path for earlier images. Keep the old-history scrub, add memory/untrusted-context coverage, and document the replay behavior in the session-pruning/media docs.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: current main preserves recent image blocks and media markers in replay while separately attaching current prompt images, so an image-heavy Telegram topic can present prior and current screenshots to the model together; no live Telegram run was performed in this read-only review.

## Solution Assessment

No. Current main is only a partial solution because the #71868 scrub handles older pruned history, while the requested current-turn media boundary and explicit comparison behavior remain missing.
