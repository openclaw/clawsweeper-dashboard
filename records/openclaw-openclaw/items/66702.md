## Summary

Keep open: current main still preserves recent prior image blocks and media markers in provider replay, and https://github.com/openclaw/openclaw/pull/76732 remains the open closing PR that should be reviewed, repaired, landed, or closed before this issue is resolved.

## What This Changes

The issue asks OpenClaw to prevent stale images from injected context and recent history from contaminating current-turn vision prompts in Telegram topic sessions.

## Best Possible Solution

Land a reviewed version of https://github.com/openclaw/openclaw/pull/76732, or an equivalent attempt-local media policy that masks prior replay images on fresh-image turns while preserving an explicit comparison path.

## Reproduction Assessment

Yes, at source level. Current main preserves the three most recent completed image turns and installs replay pruning without a current-turn attachment override, so an image-heavy topic can still present old and new screenshots together.

## Solution Assessment

No for current main; the prompt-local image loading change is only a partial mitigation because provider replay can still carry recent stale images. The linked pruning PR is the right direction, but it needs review for attempt-local state and comparison semantics.
