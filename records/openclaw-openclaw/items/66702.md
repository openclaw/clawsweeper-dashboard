## Summary

Keep open: current main still preserves the last three completed image turns and media markers in provider replay, and this issue is already paired with an open closing PR that should be reviewed rather than pre-closed.

## What This Changes

The issue requests current-turn-only vision media handling so stale images from injected context or recent history do not contaminate Telegram image analysis.

## Best Possible Solution

Land a reviewed version of https://github.com/openclaw/openclaw/pull/76732, or an equivalent attempt-local media policy that masks prior replay images on fresh-image turns while preserving an explicit comparison path.

## Reproduction Assessment

Yes, at source level. Current main preserves recent prior image blocks and media markers in replay while separately loading current prompt images, so an image-heavy Telegram topic can still present old and new screenshots together without a live Telegram run.

## Solution Assessment

No for current main; it is only a partial mitigation. The open pruning PR is the right direction, but it needs review for attempt-local state and explicit comparison semantics before this issue can close.
