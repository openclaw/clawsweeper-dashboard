## Summary

Keep open: current main has useful runtime-context separation, but recent prior image blocks and media markers are still intentionally preserved in replay, so the requested current-turn-only vision boundary remains unimplemented and privacy-sensitive.

## What This Changes

The issue asks OpenClaw to keep Telegram vision prompts scoped to current-turn media instead of reintroducing stale images from injected context or recent session history.

## Best Possible Solution

Add an embedded-runner media policy for image-bearing turns that keeps current attachments hydratable, masks prior replay image blocks and media markers by default, and provides an explicit comparison path for earlier images.

## Reproduction Assessment

Yes. The source-level path is high confidence: current prompt images are attached separately while activeSession.messages can still replay the last three completed turns with prior image blocks and media markers intact; live Telegram was not run.

## Solution Assessment

No. Current main's runtime-context separation helps injected context contamination, but preserving recent replay media is not the requested current-turn-only boundary and remains the narrower issue to solve.
