## Summary

Keep open: current main fixes the injected-context image-scan path, but recent replay history can still keep prior image blocks and media markers model-visible, and an open closing PR already owns the remaining fix path.

## What This Changes

The issue reports stale image attachments from injected memory/context blocks and recent replay history contaminating fresh multimodal Telegram turns.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/76732, or an equivalent attempt-local pruning policy that masks prior replay images on fresh-image turns while retaining the raw transcript and deliberate comparison support.

## Reproduction Assessment

Yes, at source level. Current main preserves recent prior image blocks and media markers in replay while separately loading current prompt images, so an image-heavy Telegram topic can still present old and new screenshots together.

## Solution Assessment

No for current main. The prompt-local image-loading split is only a partial mitigation because provider replay can still include recent history images; the linked pruning PR is the right direction but needs its existing blockers resolved before this issue closes.
