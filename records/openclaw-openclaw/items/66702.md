## Summary

Keep open: current main still preserves recent replay images while loading current prompt images separately, and an open PR already references this issue with closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw to prevent current-turn vision prompts from being contaminated by stale images from injected memory/context and recent message history.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/76732, or an equivalent attempt-local media policy that masks prior replay images on fresh-image turns while preserving a deliberate comparison path.

## Reproduction Assessment

Yes, at source level. Current main preserves recent prior image blocks and media markers in replay while separately loading current prompt images, so an image-heavy Telegram topic can still present old and new screenshots together without a live Telegram run.

## Solution Assessment

No for current main; the prompt-local loading split is only a partial mitigation. The open pruning PR is the right implementation path, but it needs review for attempt-local state and explicit comparison semantics before this issue can close.
