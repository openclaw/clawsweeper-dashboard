## Summary

Keep open: current main still preserves recent history images in provider replay, and an open closing PR already owns the remaining current-turn pruning fix.

## What This Changes

The issue requests that fresh multimodal turns use only current-turn images, not stale images from injected context or recent history unless comparison is explicit.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/76732, or an equivalent attempt-local pruning policy that masks prior replay images on fresh-image turns while retaining the raw transcript and deliberate comparison support.

## Reproduction Assessment

Yes, at source level. Current main preserves recent prior image blocks and media markers in replay while separately loading current prompt images, so an image-heavy Telegram topic can still present old and new screenshots together.

## Solution Assessment

No for current main; the prompt-local image loading split is only a partial mitigation because provider replay can still include recent history images. The linked pruning PR is the right direction, but it needs attempt-local state and proof before this issue should close.
