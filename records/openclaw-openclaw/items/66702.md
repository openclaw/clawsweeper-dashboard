## Summary

Keep open: current main has partial mitigations for runtime-context separation and old-history media scrubbing, but it still preserves recent prior image blocks and media markers in replay, so the requested current-turn-only media boundary for Telegram vision turns is not implemented.

## What This Changes

The issue requests that Telegram multimodal turns use only current-turn media unless the user explicitly asks to compare against earlier images.

## Best Possible Solution

Add an embedded-runner media policy for image-bearing turns that keeps current-turn attachments hydratable, masks prior replay image blocks and hydratable markers by default, and provides an explicit comparison path for earlier images.

## Reproduction Assessment

Yes. The source-level path is high confidence: recent prior image blocks and media markers remain in replay while current prompt images are attached separately, so an image-heavy Telegram topic can present prior and current screenshots together; live Telegram was not run.

## Solution Assessment

No. Current main is only a partial mitigation because old-history scrubbing and runtime-context separation do not implement the requested current-turn media boundary or explicit comparison behavior.
