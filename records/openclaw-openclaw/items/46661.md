## Summary

## What This Changes

The issue requests an opt-in custom ASR server/provider setting for voice input, including endpoint URL, auth, model, language, and fallback behavior.

## Best Possible Solution

Keep this open for a maintainer-scoped native ASR/Gateway transcription design that preserves platform speech as the default and reuses existing provider auth, network, and plugin contracts where possible.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request, not broken documented behavior. Source inspection gives high confidence that native Talk/Wake/PTT still use platform recognizers and no native custom ASR endpoint setting exists.

## Solution Assessment

Unclear until maintainers choose the product and security scope. A one-off native HTTP preference is not the best shape by default; the safer direction is a shared native ASR boundary or Gateway transcription reuse that aligns with existing provider/plugin contracts.
