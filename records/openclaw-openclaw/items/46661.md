## Summary

Keep open: current main still hardwires native Talk, Voice Wake, and push-to-talk transcription to Apple or Android platform recognizers, while the existing OpenAI-compatible transcription support is limited to media/Voice Call surfaces rather than native app voice input.

## What This Changes

Add a configurable custom OpenAI-compatible ASR endpoint for OpenClaw voice input while keeping the platform speech recognizer as the default.

## Best Possible Solution

Introduce a native ASR provider abstraction for app voice capture, keep platform speech as the default provider, then add a narrowly scoped HTTP/OpenAI-compatible provider with settings, docs, and tests once maintainers choose the covered surfaces.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high confidence that current native app voice input still uses platform recognizers and lacks a custom ASR endpoint setting.

## Solution Assessment

Unclear until maintainers choose the scope. The maintainable direction is a provider seam first; directly wiring a URL into each current recognizer path would duplicate policy and platform behavior.
