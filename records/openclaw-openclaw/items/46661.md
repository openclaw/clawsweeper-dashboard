## Summary

Keep open: current main and the latest release still do not provide a native custom ASR endpoint setting for Talk/Wake voice input, and the adjacent plugin/realtime/media transcription paths do not replace the requested native ASR provider boundary.

## What This Changes

The issue requests an opt-in custom ASR endpoint/provider setting for voice input while preserving platform speech recognition as the default.

## Best Possible Solution

Add a first-class native ASR provider/config boundary for the chosen Talk/Wake paths, keep platform recognition as the default, and reuse existing provider request, SecretRef, proxy, private-network, and redaction contracts instead of adding a one-off macOS-only HTTP preference.

Not applicable; this is a feature request, not broken documented behavior. Source inspection gives high confidence that native Talk/Wake still use platform recognizers and no native custom ASR endpoint config exists.

## Solution Assessment

Unclear until maintainer product/security scoping. The maintainable direction is a shared native ASR provider/config seam that reuses existing provider security contracts, with ClawHub/provider plugins used for engines that fit existing relay/plugin surfaces.
