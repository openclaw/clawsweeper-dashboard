## Summary

Keep this open: current main has useful custom/local transcription support for uploaded audio and voice-note media, but the requested native Talk/Voice Wake ASR provider setting is still not implemented in the first-party app speech paths.

## What This Changes

Request to add configurable custom ASR server support, including endpoint/auth/model/language settings, while preserving built-in platform speech recognition as the default.

## Best Possible Solution

Add a dedicated Talk/ASR transcription provider config surface that reuses existing media/realtime transcription provider contracts where possible, keeps platform speech as default, and then wire first-party native app clients to it deliberately.

## Reproduction Assessment

Not applicable; this is a feature request, not broken documented behavior. Source inspection gives high-confidence proof that native Talk and Voice Wake still use platform recognizers while media/audio custom endpoint support only covers uploaded audio and voice-note transcription.

## Solution Assessment

Unclear until maintainer design review; the direction is valid, but a one-off macOS preferences toggle would be weaker than a shared Talk transcription provider/config seam with app integrations. Existing media audio endpoint support should be reused where it fits, not mistaken for the full native ASR feature.
