## Summary

Keep open: current main and v2026.6.10 still route native macOS Push-to-Talk and Talk recognition through Apple Speech, while the existing custom OpenAI-compatible audio endpoint support is limited to the separate Gateway `tools.media.audio` media-understanding path. The narrower macOS PTT issue is a useful partial follow-up, but this broader request still needs maintainer product and security decisions.

## What This Changes

Request to add configurable custom ASR/STT server selection for voice input, including endpoint URL, auth, model, language hint, and fallback behavior.

## Best Possible Solution

Design a maintainer-scoped native ASR/STT boundary that preserves platform speech as the default, reuses Gateway media-audio or Talk provider contracts where appropriate, and documents credential, network, fallback, and platform scope before implementation.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and release inspection give high confidence that native macOS voice input still uses Apple Speech and has no custom ASR endpoint selector.

## Solution Assessment

Unclear until maintainers choose the product and security scope. Reusing existing Gateway media-audio, Talk session, provider request, and plugin contracts is safer than adding a parallel native HTTP preference by default.
