## Summary

Keep open: current main still lacks a native Talk/Voice Wake ASR provider setting. The related local/HTTP transcription work covers uploaded audio, media understanding, and Gateway transcription relay sessions, but native macOS/iOS/Android voice input is still wired to platform recognizers.

## What This Changes

Add a configurable custom ASR speech-to-text endpoint for native voice input while keeping platform speech recognition as the default.

## Best Possible Solution

Add a dedicated native ASR provider/config seam for Talk and any selected Voice Wake paths, keep platform speech as the default, and reuse existing transcription/provider-request security contracts where possible.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken documented behavior. Source inspection gives high confidence that native Talk and Voice Wake still use platform recognizers and that no native ASR endpoint config exists.

## Solution Assessment

Unclear until maintainer design review. The request is valid, but the maintainable direction is likely a shared native ASR provider/config seam rather than a one-off macOS preferences toggle.
