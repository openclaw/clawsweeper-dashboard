## Summary

Keep open: current main and v2026.6.6 still hard-wire native macOS Talk/PTT and mobile wake/talk paths to platform speech recognizers, while the existing configurable audio transcription surface is separate media-understanding/Gateway functionality. The request remains useful but needs maintainer product and security scoping before implementation.

## What This Changes

The issue requests an opt-in custom ASR speech-to-text endpoint/provider configuration instead of relying only on platform speech recognition.

## Best Possible Solution

Keep this open for a maintainer-scoped native ASR/STT design that preserves platform speech as the default and reuses Gateway/media-understanding, provider auth, and network-safety contracts where possible.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request, not broken documented behavior. Source inspection gives high confidence that native Talk/Wake/PTT still use platform recognizers and no native custom ASR endpoint setting exists.

## Solution Assessment

Unclear until maintainers choose the product and security scope. A one-off native HTTP preference is not the best shape by default; a shared native ASR boundary or Gateway transcription reuse is the safer direction.
