## Summary

Keep open: current main still hard-wires native Talk and Voice Wake recognition to platform speech APIs, while the existing media-understanding and realtime transcription plugin paths do not yet provide the requested native custom ASR endpoint setting.

## What This Changes

The issue requests an opt-in custom ASR endpoint setting for native voice input, with platform speech recognition remaining the default.

## Best Possible Solution

Add a shared native ASR provider/config boundary for the chosen Talk/Wake flows, keep platform recognition as the default, and reuse existing provider request, SecretRef, private-network, proxy, and redaction contracts.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken documented behavior. Source inspection gives high confidence that native Talk/Wake still use platform recognizers and no native custom ASR endpoint setting exists.

## Solution Assessment

Unclear until maintainers choose product and security scope. A one-off macOS HTTP preference is not the best shape; the maintainable path is a shared native ASR provider boundary or an explicit decision to route native voice through plugin-backed gateway relay.
