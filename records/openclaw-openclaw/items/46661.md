## Summary

Keep this open: current main has custom/local transcription support for uploaded audio and transcription-only Gateway relay sessions, but the requested native Talk/Voice Wake ASR provider setting is still missing from the first-party app speech paths.

## What This Changes

Allow users to configure a custom OpenAI-compatible ASR endpoint, auth, model, and language instead of native platform speech recognition for voice input.

## Best Possible Solution

Add a dedicated native Talk/ASR provider config surface that keeps platform speech as the default, reuses existing media/realtime transcription contracts where possible, and wires macOS/iOS/Android clients deliberately.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken documented behavior. Source inspection gives high-confidence proof that native Talk and Voice Wake still use platform recognizers while custom endpoint support covers other transcription surfaces.

## Solution Assessment

Unclear until maintainer design review. The requested direction is valid, but a one-off macOS preferences toggle would be weaker than a shared native ASR provider/config seam with platform integrations.
