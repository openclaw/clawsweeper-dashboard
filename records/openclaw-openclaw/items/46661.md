## Summary

Keep open: current main has adjacent uploaded-audio and Gateway transcription relay support, but native Talk and Voice Wake still use platform recognizers and there is no native custom ASR endpoint setting. This needs maintainer scoping because it affects platform behavior, audio privacy, endpoint auth, and fallback policy.

## What This Changes

Request to add an opt-in configurable custom ASR/Speech-to-Text endpoint for native voice input while preserving platform recognition as the default.

## Best Possible Solution

Add a first-class native ASR provider/config boundary for the selected Talk and Voice Wake paths, keep platform recognition as the default, and reuse existing provider request and SecretRef security contracts.

## Solution Assessment

Unclear until maintainer product review. The maintainable direction appears to be a shared native ASR provider/config seam that reuses existing media/provider request security, not a one-off macOS preferences field.
