## Summary

Keep open: current main and the latest release still route native macOS Talk and Push-to-Talk through Apple Speech, while the existing configurable audio transcription path is separate `tools.media.audio`/Gateway functionality and does not provide a native ASR endpoint selector.

## What This Changes

The issue requests an opt-in custom ASR/STT server setting for native voice input, including endpoint URL, optional auth, model, language hint, and fallback to the built-in macOS recognizer.

## Best Possible Solution

Design a maintainer-scoped native ASR/STT boundary that preserves Apple Speech as the default, reuses Gateway/media audio or Talk provider contracts where appropriate, and explicitly documents credential, network, fallback, and platform scope.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and release inspection give high confidence that native macOS voice input still uses Apple Speech and has no custom ASR endpoint setting.

## Solution Assessment

Unclear until maintainers choose the product and security scope. A one-off native HTTP preference is not the best shape by default; reusing existing Gateway/media audio, Talk, provider request, and plugin contracts is the safer direction.
