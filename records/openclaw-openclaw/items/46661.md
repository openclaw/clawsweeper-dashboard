## Summary

Keep open: current main still does not provide a native Talk or Voice Wake ASR provider setting, and the existing audio/media transcription support does not replace the platform recognizers used by native voice input.

## What This Changes

The issue requests an opt-in custom ASR server configuration for native voice input, including endpoint URL, auth, model, and language settings.

## Best Possible Solution

Add a dedicated native ASR provider/config seam for Talk and any selected Voice Wake paths, keep platform speech as the default, and reuse the existing media/provider request security contracts where possible.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken documented behavior. Source inspection gives high confidence that native Talk and Voice Wake still use platform recognizers and lack a native ASR endpoint setting.

## Solution Assessment

Unclear until maintainer design review. The request is valid, but the maintainable direction is likely a shared native ASR provider/config seam rather than a one-off macOS preferences toggle.
