## Summary

Keep open: current main and the latest release still route native Talk and Voice Wake through platform speech recognizers, while the existing OpenAI-compatible batch STT and Gateway realtime transcription paths do not provide the requested native custom ASR endpoint setting.

## What This Changes

Request to add an opt-in custom ASR server endpoint, auth, model, and language configuration for speech-to-text while keeping platform speech recognition as the default.

## Best Possible Solution

Add a shared native ASR provider/config boundary only after maintainers choose the Talk/Wake/platform scope, keep platform recognition as the default, and reuse the existing provider request, SecretRef, private-network, proxy, and redaction contracts instead of adding a parallel audio upload surface.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken documented behavior. Source inspection gives high confidence that native Talk/Wake still use platform recognizers and no native custom ASR endpoint setting exists.

## Solution Assessment

Unclear until maintainers choose product and security scope. A one-off macOS HTTP preference is not the best shape; the maintainable path is a shared native ASR provider boundary or an explicit decision to route native voice through plugin-backed Gateway relay.
