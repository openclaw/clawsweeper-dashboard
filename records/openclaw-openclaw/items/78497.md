## Summary

Keep open: the intended channel-login diagnostic is useful, but this branch does not fix the paths described in its own examples or the related report, and it includes unrelated churn outside the channel surface.

## What This Changes

The PR changes channel login/start/stop error text and also modifies an unrelated Codex test helper plus an agent cron-tool lint suppression.

## Best Possible Solution

Fix the login path before or during gateway reconciliation so missing or disabled channel config gets actionable guidance without implicitly writing config, update the unknown-channel gateway diagnostics on the actual plugin-miss branches, and add focused regression coverage.

## Reproduction Assessment

Yes, from source inspection rather than a live run. The CLI saves auth before gateway reconciliation, the gateway can return successful non-running state for unconfigured accounts, and normal unknown channel ids hit an unchanged plugin-miss branch.

## Solution Assessment

No. The proposed solution only appends a hint for thrown errors and changes the invalid-param branch, so it misses both the non-throwing unconfigured login path and the normal unknown-channel example.
