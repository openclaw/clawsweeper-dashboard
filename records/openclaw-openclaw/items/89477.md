Request an optional vendor-neutral plugin SDK rerank provider that reorders builtin memory search candidates before MMR without adding core config or a concrete reranker.

## Best Possible Solution

Use this issue as the design anchor while maintainers review https://github.com/openclaw/openclaw/pull/89584, accept or revise the SDK/manifest contract, and keep concrete reranker transports plugin-owned.
