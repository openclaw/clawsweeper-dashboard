## What This Changes

Request an optional vendor-neutral MemoryRerankProvider plugin-SDK capability for builtin sqlite-vec memory search before MMR, with no core config or bundled reranker implementation.

## Best Possible Solution

Make an explicit maintainer API decision, then implement a narrow plugin-agnostic rerank seam with contract tests, SDK docs, manifest/export baselines, and no concrete vendor reranker in core.
