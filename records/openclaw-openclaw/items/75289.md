## Summary

Keep open. Current main still matches the reported TUI lifecycle gap: `runTui` handles SIGINT/SIGTERM but not SIGHUP, shutdown still waits on an unbounded terminal drain, and the render/status timers are not explicitly stopped before exit cleanup; the closed fix PR that targeted this issue was not merged.

## What This Changes

The issue asks for `openclaw tui` to exit on terminal hangup or gateway restart instead of leaving orphaned high-CPU, high-RSS `openclaw-tui` processes that require SIGKILL.

## Best Possible Solution

Land narrow TUI lifecycle hardening: handle POSIX hangup, make shutdown bounded and idempotent with a hard fallback, stop render timers/loaders before terminal teardown, and add focused regression tests plus a changelog entry.

## Reproduction Assessment

Yes. The issue gives concrete macOS LaunchAgent terminal-close and gateway-restart steps, and current main still has matching source-level preconditions; I did not run the live macOS reproduction in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the TUI shutdown path and tests, reusing the existing helper seams rather than broadening into model discovery, session-store performance, or gateway CPU work.
