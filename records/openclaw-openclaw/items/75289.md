## Summary

Keep open. Current main still has the reported TUI lifecycle gap: the TUI shutdown path handles SIGINT/SIGTERM only, signal exit still waits on the normal unbounded terminal drain path, and the prior closing PR is now closed unmerged rather than implemented on main.

## What This Changes

The issue requests that `openclaw tui` exit cleanly on terminal hangup, gateway restart, and SIGTERM instead of leaving high-CPU, high-memory orphaned TUI processes.

## Best Possible Solution

Land a narrow replacement for PR #75294 that handles POSIX hangup, makes signal shutdown bounded and idempotent, stops loader/timer activity before terminal teardown, and keeps the broader model-normalization CPU work tracked separately through #75137/#75503 unless profiling proves a shared root cause.

## Reproduction Assessment

Yes. The report gives concrete macOS LaunchAgent and terminal-close/gateway-restart steps, and current main has matching source evidence for missing SIGHUP handling plus unbounded signal cleanup; this review did not run the live macOS reproduction.

## Solution Assessment

Yes at the direction level. The maintainable fix belongs in `src/tui/tui.ts` with focused tests, reusing the dependency's bounded `drainInput(maxMs, idleMs)` contract and avoiding broad changes to unrelated TUI CPU paths.
