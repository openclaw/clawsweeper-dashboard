## Summary

Keep open. Current main still matches the reported TUI lifecycle gap: the TUI child handles SIGINT/SIGTERM but not SIGHUP, signal shutdown still waits on the normal unbounded terminal drain path, and the PR that directly targeted this issue is closed unmerged rather than implemented on main.

## What This Changes

Make `openclaw-tui` exit promptly on terminal hangup, gateway restart, and SIGTERM instead of leaving high-CPU, high-RSS orphaned processes behind.

## Best Possible Solution

Land a focused TUI lifecycle fix in `src/tui/tui.ts`: handle POSIX hangup, make signal shutdown bounded and idempotent, stop loader/timer activity before terminal teardown, add regression tests in `src/tui/tui.test.ts`, and keep the broader model-normalization CPU work tracked separately unless profiling proves a shared root cause.

## Reproduction Assessment

Yes. The report gives concrete macOS LaunchAgent and terminal-close/gateway-restart steps, and current main has matching source evidence for missing SIGHUP handling plus unbounded signal cleanup; this review did not run the live macOS reproduction.

## Solution Assessment

Yes at the direction level. The narrow maintainable path is a TUI shutdown repair, not closing this under the broader CPU busy-loop PR, because orphan survival and SIGHUP/SIGTERM behavior remain observable gaps on current main.
