## Summary

Keep open. Current main still exposes the reported TUI lifecycle failure mode, and open PR #75294 directly targets this issue with closing syntax, so the issue should remain open until that PR or an equivalent fix lands. The related CPU reports also point to at least one separate hot path, so the lifecycle fix should not be conflated with all TUI/CLI CPU work.

## What This Changes

The issue requests that `openclaw-tui` exit on terminal hangup, gateway restart, and SIGTERM instead of surviving as a high-CPU, high-RSS orphan process.

## Best Possible Solution

Land PR #75294 or an equivalent narrow TUI lifecycle fix that handles SIGHUP, makes signal shutdown bounded and idempotent, stops loader/timer work before terminal cleanup, and leaves the broader model-normalization CPU work tracked through #75137/#75503 unless profiling proves a shared root cause.

## Reproduction Assessment

Yes. The issue provides concrete macOS LaunchAgent reproduction steps, and current main has matching source evidence: no explicit SIGHUP handler, an unbounded terminal drain in the exit path, and render/status timers not stopped before shutdown cleanup.

## Solution Assessment

Yes at the direction level. The narrow maintainable fix belongs in the TUI lifecycle path and tests; model-discovery CPU caching should remain a separate PR unless live profiling ties it to the orphaned terminal-close failure.
