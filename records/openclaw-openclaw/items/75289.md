## Summary

Keep open. Current main still has the reported TUI lifecycle gap: the child launcher forwards SIGHUP on Unix, but the TUI runtime only registers SIGINT/SIGTERM, and its signal exit path still uses the normal unbounded terminal drain without an explicit timer/loader stop or hard fallback.

## What This Changes

The issue requests that `openclaw tui` exit cleanly on terminal close, SIGTERM, and gateway restart instead of leaving high-CPU/high-RSS orphaned TUI processes that require SIGKILL.

## Best Possible Solution

Land a narrow TUI lifecycle fix that handles POSIX hangup, makes signal shutdown bounded and idempotent, stops status loader/timer activity before terminal teardown, adds focused regression tests, and leaves broader CPU normalization/session-bloat work to the existing related reports unless new profiling ties them together.

## Reproduction Assessment

Yes. The issue gives concrete macOS LaunchAgent and terminal-close/gateway-restart steps, and current main has matching source evidence for missing SIGHUP handling plus unbounded signal cleanup; I did not run the live macOS reproduction in this read-only review.

## Solution Assessment

Yes. The narrow maintainable path is to fix `src/tui/tui.ts` shutdown behavior and tests, reusing the existing child-process bridge and terminal-drain seams instead of changing unrelated TUI CPU or model-discovery code.
