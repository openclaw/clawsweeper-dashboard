## Summary

Keep open. Current main still has the reported TUI lifecycle failure surface, and the only PR that explicitly closes this issue is still open rather than merged.

## What This Changes

The issue requests that `openclaw tui` exit on terminal hangup and SIGTERM without leaving high-CPU, high-memory orphaned TUI processes after terminal close or gateway restart.

## Best Possible Solution

Review and land PR #75294 or an equivalent narrow TUI lifecycle fix that handles POSIX hangup, makes signal shutdown bounded and idempotent, stops loader/timer activity before terminal cleanup, and leaves broader model-normalization CPU work to the separate PR #75503 path unless profiling proves a shared root cause.

## Reproduction Assessment

Yes. The issue gives concrete macOS LaunchAgent and terminal-close/gateway-restart steps, and current main has matching source evidence for missing TUI SIGHUP handling plus unbounded signal cleanup.

## Solution Assessment

Yes at the direction level, but not yet on main. A focused TUI lifecycle fix is the narrow maintainable path for orphaned terminal-close processes, while the model-discovery CPU cache work should remain separate unless new profiling ties it to this shutdown bug.
