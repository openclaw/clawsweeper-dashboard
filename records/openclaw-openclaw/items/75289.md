## Summary

Keep open. Current main still has the TUI lifecycle gap behind this report: `runTui` handles SIGINT/SIGTERM but not SIGHUP, signal shutdown still waits on an unbounded terminal drain, and the exact fix PR linked from the issue is closed unmerged.

## What This Changes

The issue reports that `openclaw-tui` processes can spin at 100% CPU, ignore normal termination, and survive terminal or gateway shutdown.

## Best Possible Solution

Land a focused TUI lifecycle fix that handles terminal hangup, bounds signal shutdown, stops render timers/loaders before terminal teardown, and adds regression tests plus a changelog entry.

## Repair Work Prompt

Fix the TUI lifecycle bug from https://github.com/openclaw/openclaw/issues/75289. Current main registers only SIGINT/SIGTERM in `src/tui/tui.ts`, while terminal close/hangup and gateway restart paths can leave `openclaw-tui` stuck behind an unbounded terminal drain. Add POSIX hangup handling for the TUI child, make signal-triggered `requestExit` idempotent and bounded, stop status loader/waiting/status timers before terminal drain, call `drainInput` with explicit bounds or a timeout wrapper, and add a hard fallback so a hanging drain/stop cannot leave the process alive. Add focused regression coverage in `src/tui/tui.test.ts` for SIGHUP registration/cleanup, bounded drain, hanging drain fallback, timer/loader shutdown, and existing SIGTERM/Ctrl+C behavior. Add a single-line user-facing changelog fix entry. Use #75294 as useful prior source context, but keep the replacement narrow; do not make broad gateway restart, model-discovery cache, plugin loading, or session-store rewrites unless a targeted test proves they are necessary for this lifecycle bug.
