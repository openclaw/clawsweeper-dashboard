## Summary

Keep open. Current main still has the reported TUI lifecycle gap: the launched child process can receive SIGHUP, but the TUI runtime only handles SIGINT/SIGTERM, and the signal exit path still uses unbounded terminal drain without a hard fallback.

## What This Changes

The issue requests that `openclaw-tui` exit cleanly on terminal hangup, gateway restart, and SIGTERM instead of remaining as high-CPU, high-RSS orphaned processes.

## Best Possible Solution

Land a narrow TUI lifecycle fix that handles POSIX hangup, makes signal shutdown bounded and idempotent, stops loader/timer activity before terminal teardown, adds focused regression tests, and leaves model-discovery CPU normalization work to #75137/#75503 unless profiling proves the same root cause.

## Reproduction Assessment

Yes. The issue gives concrete macOS LaunchAgent and terminal-close/gateway-restart steps, and current main has matching source evidence for missing SIGHUP handling plus unbounded cleanup; I did not run the live macOS reproduction in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix belongs in `src/tui/tui.ts` and its tests, reusing the existing child-process bridge and terminal-drain seams rather than broadening this issue into unrelated gateway or model-discovery CPU work.
