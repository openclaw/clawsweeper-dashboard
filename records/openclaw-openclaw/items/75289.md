Keep open: current main still lacks the TUI SIGHUP and bounded-exit lifecycle handling needed for the reported orphaned high-CPU processes, and the focused fix PR is closed unmerged.

## What This Changes

The issue asks for TUI processes to exit on terminal hangup, gateway restart, and SIGTERM without spinning or accumulating CPU/RAM usage.

## Best Possible Solution

Land a narrow TUI lifecycle fix that treats POSIX hangup as shutdown, makes exit cleanup bounded and idempotent with a hard fallback, stops render timers before teardown, and adds focused regression coverage plus a changelog entry.
