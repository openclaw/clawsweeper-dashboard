## Summary

Current main now covers the reported TUI orphan lifecycle: interactive TUI launches avoid the generic respawn wrapper, terminal loss requests exit, shutdown drain is bounded, and repeated/blocked shutdown has a hard fallback. The central fix and follow-up TUI exit hardening are shipped in v2026.5.12.

## What This Changes

The issue reports that orphaned TUI processes remain alive after terminal close or gateway restart, peg CPU, and require SIGKILL.

## Best Possible Solution

Keep the shipped TUI lifecycle fixes; if the later non-TUI CLI/session-history CPU report still reproduces on current releases, track it as a separate issue with its own profile and session-data repro.

## Reproduction Assessment

Yes. The original report has concrete macOS terminal-close and gateway-restart steps, and the later merged PR for the adjacent disconnect path included a before/after PTY proof; I did not rerun the live macOS LaunchAgent scenario in this read-only review.

## Solution Assessment

Yes. The current solution fixes the narrow lifecycle boundaries: interactive respawn policy, terminal-loss detection, bounded terminal drain, and hard shutdown fallback, instead of adding a policy/config workaround.
