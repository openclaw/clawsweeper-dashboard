## Summary

Keep this open. Current main still matches the reported TUI shutdown failure mode: the TUI handles SIGINT/SIGTERM but not SIGHUP, its exit path only completes after terminal drain/stop succeeds, and busy status timers are not explicitly stopped as part of shutdown. There is also an open linked PR with closing syntax, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw to prevent orphaned `openclaw-tui` processes from pegging CPU and surviving terminal close, gateway restart, or SIGTERM.

## Best Possible Solution

Land PR #75294 or an equivalent TUI lifecycle fix that handles SIGHUP, makes shutdown idempotent and bounded, stops loader/status timers before terminal cleanup, and adds focused regression tests for terminal hangup and SIGTERM behavior. Treat the later large-session CLI profiling comment as a separate follow-up if it still reproduces after the TUI shutdown fix.

## Reproduction Assessment

Yes. The issue gives concrete macOS LaunchAgent/TUI steps, and current main has matching source evidence for the shutdown bug: no SIGHUP handler, unbounded terminal drain, and exit completion chained to terminal cleanup.

## Solution Assessment

Yes. The narrow maintainable solution is in the TUI lifecycle path, not gateway reconnect policy: bound cleanup, tolerate closed terminal streams, stop render timers, and explicitly handle terminal hangup signals.
