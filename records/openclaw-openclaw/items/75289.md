## Summary

Keep open. Current main still has the TUI lifecycle gap behind this report: the launched child can receive SIGHUP, but `runTui` only handles SIGINT/SIGTERM and signal shutdown still waits on an unbounded terminal drain; the prior fix PR is closed unmerged.

## What This Changes

The issue asks for TUI processes to exit cleanly on terminal hangup, gateway restart, and SIGTERM instead of spinning at 100% CPU and accumulating memory.

## Best Possible Solution

Land a narrow TUI lifecycle fix that handles POSIX hangup, makes signal shutdown bounded and idempotent with a hard fallback, stops render timers/loaders before terminal teardown, and adds focused regression tests plus a changelog entry.

## Reproduction Assessment

Yes, at source level. The issue gives concrete macOS LaunchAgent terminal-close and gateway-restart steps, and current main still has the matching SIGHUP and unbounded-drain preconditions; I did not run the live macOS reproduction in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the TUI lifecycle path, reusing existing shutdown helper seams; changing model discovery or gateway restart behavior alone would not guarantee orphaned TUI processes exit.
