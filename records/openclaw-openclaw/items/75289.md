## Summary

Keep open. Current main still has the TUI shutdown gap: the parent bridge forwards SIGHUP to the child, but `runTui` only handles SIGINT/SIGTERM and its signal exit path still waits on an unbounded terminal drain.

## What This Changes

Fix TUI lifecycle so `openclaw-tui` exits on terminal hangup, gateway restart, and SIGTERM instead of spinning and accumulating orphaned high-CPU processes.

## Best Possible Solution

Land a narrow TUI lifecycle fix that handles POSIX hangup, makes signal shutdown bounded and idempotent with a hard fallback, stops render timers/loaders before terminal teardown, and adds focused regression tests plus a changelog entry.

## Reproduction Assessment

Yes, at source level. The report gives concrete macOS LaunchAgent terminal-close and gateway-restart steps, and current main still has the matching SIGHUP and unbounded-drain preconditions; I did not run the live macOS reproduction.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the TUI lifecycle path, reusing existing shutdown helper seams; changing gateway restart behavior or model discovery alone would not guarantee orphaned TUI processes exit.
