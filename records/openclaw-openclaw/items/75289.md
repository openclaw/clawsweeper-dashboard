## Summary

Keep open. Current main still has the reported TUI lifecycle gap: the launcher forwards hangup on Unix, but the TUI runtime only handles SIGINT/SIGTERM and still exits through an unbounded terminal-drain path without explicit exit-time loader/timer shutdown or a hard fallback.

## What This Changes

Fix TUI processes that can peg CPU, ignore normal termination, and survive terminal close or gateway restart.

## Best Possible Solution

Land a narrow TUI lifecycle fix that handles POSIX hangup, makes signal shutdown bounded and idempotent, stops loader/timer render activity before terminal teardown, and adds focused regression tests while leaving broader CPU normalization work to the existing related items.

## Solution Assessment

Yes. The narrow maintainable path is to fix `src/tui/tui.ts` shutdown behavior and tests, reusing the existing child-process bridge and terminal-drain seams instead of changing unrelated model-discovery, session-bloat, or gateway CPU code.
