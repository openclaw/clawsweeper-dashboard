## Summary

Keep open. Current main still matches the reported TUI lifecycle failure surface: no explicit SIGHUP handling, signal shutdown still waits on the normal unbounded terminal drain path, and the linked closing PR is open rather than merged.

## What This Changes

The issue asks for TUI processes to exit on terminal hangup, gateway restart, and SIGTERM instead of spinning at high CPU/RSS as orphaned processes.

## Best Possible Solution

Review and land PR #75294 or an equivalent narrow TUI lifecycle fix that handles POSIX hangup, makes signal shutdown bounded and idempotent, stops status loader/timers before terminal cleanup, and keeps broader model-normalization CPU work tracked separately unless profiling proves a shared root cause.

## Reproduction Assessment

Yes. The report gives concrete macOS LaunchAgent and terminal-close steps, and current main has matching source evidence for missing SIGHUP handling and unbounded signal shutdown cleanup.

## Solution Assessment

Yes at the direction level, but it is not implemented on main yet. The narrow maintainable fix belongs in the TUI lifecycle path, while PR #75503-style model normalization caching should remain a separate performance fix unless profiling ties it to this orphaned-process path.
