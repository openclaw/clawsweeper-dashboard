## Summary

Keep open. Current main still has the reported TUI lifecycle gap: the TUI handles SIGINT/SIGTERM but not SIGHUP, signal shutdown still waits on unbounded terminal drain, and the current model-warmup CPU fix does not address orphaned TUI processes after terminal close.

## What This Changes

The issue reports orphaned `openclaw-tui` processes that peg CPU/RAM after terminal hangup or gateway restart and resist SIGTERM.

## Best Possible Solution

Land a narrow TUI lifecycle fix that handles POSIX hangup, makes shutdown bounded and idempotent with a hard fallback, stops render timers/loaders before terminal teardown, and adds focused regression tests plus a changelog entry.

## Reproduction Assessment

Yes. The issue gives concrete macOS LaunchAgent and terminal-close/gateway-restart steps, and current main still has matching source-level preconditions; I did not run the live macOS reproduction in this read-only review.

## Solution Assessment

Yes. The narrow maintainable solution belongs in `src/tui/tui.ts` and its tests, reusing the existing shutdown helper seams rather than changing unrelated model-normalization, session-store, or gateway CPU paths.
