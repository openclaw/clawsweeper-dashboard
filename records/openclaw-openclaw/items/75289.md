## Summary

Keep open. Current main still has the reported TUI lifecycle gap: the TUI runtime has no SIGHUP shutdown handler, signal exit still waits on an unbounded terminal drain path, and the narrow prior fix PR is closed unmerged rather than present on main.

## What This Changes

The issue requests that TUI processes exit cleanly on terminal hangup, SIGTERM, and gateway restart instead of orphaning, pegging CPU, and consuming large memory.

## Best Possible Solution

Land a narrow TUI lifecycle fix that handles POSIX hangup, makes signal-driven shutdown bounded and idempotent, stops status loader/timer activity before terminal teardown, and coordinates with #75379 for the disconnected-state exit path while keeping #75137/#75503 model-normalization CPU work separate.

Yes. The issue gives concrete macOS terminal-close and gateway-restart steps, and current-main source inspection confirms the same missing SIGHUP handling plus unbounded signal cleanup path; this review did not run the live macOS repro.

## Solution Assessment
