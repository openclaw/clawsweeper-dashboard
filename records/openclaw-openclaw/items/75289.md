## Summary

Keep open. Current main still has the reported TUI lifecycle gap: the runtime handles SIGINT/SIGTERM but not terminal hangup, shutdown waits on OpenClaw-unbounded terminal drain, and the current model-warmup CPU fix does not address orphaned TUI processes after terminal close.

## What This Changes

The issue reports orphaned `openclaw-tui` processes that survive terminal hangup or gateway restart, peg CPU, ignore SIGTERM, and accumulate memory.

## Best Possible Solution

Land a narrow TUI lifecycle fix that handles POSIX hangup, makes shutdown bounded and idempotent with a hard fallback, stops render timers/loaders before terminal teardown, and adds focused regression tests plus a changelog entry.

## Reproduction Assessment

Yes. The issue gives concrete macOS LaunchAgent terminal-close and gateway-restart steps, and current main still has matching source-level preconditions: no SIGHUP handler and an OpenClaw-unbounded requestExit drain chain; I did not run the live macOS reproduction in this read-only review.

## Solution Assessment

Yes. The maintainable fix belongs in the TUI lifecycle path, reusing the existing shutdown helper seams; the current model-normalization startup fix reduces a different CPU path but does not solve terminal hangup or SIGTERM-resistant orphaned TUI processes.

## Repair Work Prompt

Fix the TUI lifecycle bug reported in https://github.com/openclaw/openclaw/issues/75289. On current main, `runTui` handles SIGINT/SIGTERM but not POSIX SIGHUP, and `requestExit()` waits on `drainAndStopTuiSafely(tui)` without explicit drain bounds or a hard fallback. Implement a narrow TUI-owned fix: handle SIGHUP like SIGTERM on non-Windows platforms, make exit idempotent and bounded, stop status/waiting timers and loader render sources before terminal drain/stop, and add a fallback so a stuck drain/stop cannot leave `openclaw-tui` alive indefinitely. Preserve normal Ctrl+C double-press behavior where possible, and coordinate with related #75379 only for shared lifecycle coverage. Do not change unrelated model discovery, gateway CPU, session-store, or provider behavior. Add focused regression tests in the TUI test surface for SIGHUP registration/removal, bounded drain or hanging-drain fallback, SIGTERM/SIGHUP exit completion, and render timer/loader cleanup. Add a single-line `CHANGELOG.md` entry under Unreleased Fixes; do not add bot/maintainer Thanks attribution unless a human contributor is credited by the actual PR history.
