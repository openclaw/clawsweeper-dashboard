## Summary

Keep open. Current main still has the reported TUI lifecycle gap, and open PR #75294 already targets the exact terminal-hangup shutdown path with closing syntax, so the issue should wait for that PR or an equivalent fix rather than be closed now.

## What This Changes

The issue asks for `openclaw tui` to exit on terminal hangup, SIGTERM, and gateway restart instead of leaving high-CPU, high-RSS orphaned TUI processes behind.

## Best Possible Solution

Review and land PR #75294 or an equivalent narrow TUI lifecycle patch that adds POSIX hangup handling, makes shutdown idempotent and bounded, stops status loader/timers before terminal cleanup, and adds focused tests for SIGHUP/SIGTERM cleanup. Keep broader model-normalization CPU work such as #75137/#75503 tracked separately unless profiling proves a shared root cause.

## Reproduction Assessment

Yes. The issue provides concrete macOS steps, and current main has matching source evidence: no explicit SIGHUP handler, unbounded terminal drain, and signal shutdown that can wait on terminal cleanup before finishing.

## Solution Assessment

Yes at the direction level. The narrow maintainable solution belongs in `src/tui/tui.ts` and focused TUI shutdown tests; gateway service management and model-normalization CPU caching should not be treated as the sole fix for the orphaned terminal process path.
