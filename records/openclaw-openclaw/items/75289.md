## Summary

Keep open. Current main still lacks explicit TUI SIGHUP handling and the signal shutdown path can wait on unbounded terminal drain before finishing; open PR #75294 directly targets this issue and should be reviewed or replaced before the issue is closed.

## What This Changes

Ensure `openclaw tui` exits on terminal hangup, gateway restart, and SIGTERM instead of orphaning high-CPU, high-memory TUI processes.

## Best Possible Solution

Land PR #75294 or an equivalent narrow TUI lifecycle fix that handles POSIX hangup, makes shutdown idempotent and bounded, stops status loader/timers, and tolerates closed terminal streams; keep model-normalization CPU work such as #75503 tracked separately unless profiling proves a shared root cause.

## Reproduction Assessment

Yes. The issue gives concrete macOS steps, and current main has matching source evidence: no TUI SIGHUP handler, unbounded terminal drain on exit, and active status/render timers that are not stopped by `requestExit()`.

## Solution Assessment

Yes, at the direction level. The narrow maintainable fix belongs in `src/tui/tui.ts` and focused shutdown tests; gateway reconnect policy and unrelated model/session performance fixes should remain separate unless live profiling ties them together.
