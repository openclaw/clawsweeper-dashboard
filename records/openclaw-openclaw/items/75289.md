## Summary

Keep open. Current main still has the reported TUI lifecycle gap: the launcher forwards hangup signals, but the TUI runtime only handles SIGINT/SIGTERM and its signal-exit path can still wait on normal unbounded terminal drain while status timers remain active.

## What This Changes

The issue asks for TUI processes to exit on terminal close, SIGTERM, and gateway restart instead of becoming orphaned high-CPU, high-RSS processes.

## Best Possible Solution

Land a narrow TUI lifecycle fix that handles POSIX hangup, makes signal shutdown bounded and idempotent, stops loader/timer activity before terminal teardown, and leaves broader model-discovery or session-bloat CPU work to the existing related reports unless new profiling connects them.

## Reproduction Assessment

Yes. The report gives concrete macOS LaunchAgent terminal-close and gateway-restart steps, and current main still has matching source evidence for missing SIGHUP handling plus unbounded signal cleanup; I did not execute the live macOS reproduction.

## Solution Assessment

Yes. The narrow maintainable solution is in `src/tui/tui.ts` with focused tests, reusing the existing child-process bridge and terminal drain seams instead of changing unrelated model-discovery or session storage behavior.
