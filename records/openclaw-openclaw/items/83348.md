Yes, source-level. Current main routes bare `stop`, `abort`, and `wait` through a slash-command-only debounce predicate, while the existing abort-aware detector and trigger set already recognize those phrases.

## Solution Assessment

No, not in the latest repaired shape. Reusing `isControlCommandMessage()` at debounce boundaries is the right fix, but Mattermost and MSTeams should not use the broader inline-token auth probe as the ingress blocking flag.
