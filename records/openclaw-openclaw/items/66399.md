## Summary

Keep open. Current main still routes exec and CLI timeout/cancellation through a supervisor cancel path that invokes SIGKILL, and the remaining focused fix is still tracked by this canonical issue rather than an open fixing PR.

## What This Changes

Request graceful SIGTERM-to-SIGKILL escalation and bounded drain handling for exec-tool process supervision.

## Best Possible Solution

Implement a focused supervisor cancellation policy that sends SIGTERM first, waits an unref'ed grace period, then escalates to SIGKILL with bounded wait/drain behavior across child and PTY adapters.

## Reproduction Assessment

Yes, at source level. Exec and CLI runs pass timeouts into the supervisor, timeout/manual cancellation routes through requestCancel, and current tests still assert SIGKILL on those paths.

## Solution Assessment

Yes. A bounded SIGTERM-first supervisor policy with SIGKILL escalation is the narrow maintainable fix; the existing process-tree helper should be reused or aligned without dropping the hard fallback.
