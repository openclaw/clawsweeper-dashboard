## Summary

Keep open. Current main still cancels timeout, no-output timeout, scoped replacement, and manual cancellation through immediate `SIGKILL`, and the remaining work is still on this canonical issue rather than an open fixing PR.

## What This Changes

The issue asks for exec/process supervisor cancellation to use graceful SIGTERM-to-SIGKILL escalation with a bounded drain timeout instead of immediate hard kill.

## Best Possible Solution

Implement a focused supervisor cancellation policy that sends SIGTERM first, waits an unref'ed grace period, then escalates through SIGKILL with bounded wait/drain behavior across child and PTY adapters.

## Reproduction Assessment

Yes, at source level. Timeout, no-output timeout, scoped replacement, and manual cancellation all route through `requestCancel`, current tests assert `SIGKILL`, and exec passes `timeoutMs` into the same supervisor path.

## Solution Assessment

Yes. SIGTERM first with a bounded SIGKILL escalation is the narrow maintainable fix, while a SIGTERM-only change would miss the liveness concern already raised in related discussion.
