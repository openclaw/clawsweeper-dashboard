## Summary

Keep open. Current main still has backend TTS capabilities but no first-class Control UI Settings surface for enabling TTS, choosing a provider, choosing a default voice, or playing a sample; the linked implementation candidate is still open and draft in the provided GitHub context, so closing this issue before that lands would be premature.

## What This Changes

Request a Control UI Settings TTS section with an enable toggle, provider selector, provider-filtered default voice picker, and optional sample playback backed by the existing TTS preferences and RPC surface.

## Best Possible Solution

Ship a focused Control UI Quick Settings TTS card backed by tts.status/providers/enable/disable/setProvider, add the narrow default-voice persistence and preview RPCs if needed, and cover the UI flow plus gateway methods with targeted tests before closing this issue.
