## What This Changes

The issue requests a first-class Control UI Settings surface for Text-to-Speech enablement, provider selection, default voice selection, and optional sample playback.

## Best Possible Solution

Add a focused Text-to-Speech panel in Control UI Settings backed by existing Gateway TTS status/provider/mutation RPCs, plus a provider-filtered voice selector that respects provider-owned voice metadata and persists through the agreed TTS preference/config contract. The panel should include clear unavailable-provider states, provider-backed sample playback through `tts.convert`, focused UI/gateway tests, docs updates, and a user-facing changelog entry.
