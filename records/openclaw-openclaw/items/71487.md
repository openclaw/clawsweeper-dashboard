## Summary

Keep open. Current main still lacks the requested first-class Control UI TTS Settings surface, and the linked implementation candidate #75479 is open, draft, and unmerged, so closing this issue before that work lands would be premature.

## What This Changes

Add a Control UI Settings TTS section with an enable toggle, provider selector, provider-filtered default voice picker, and optional sample playback backed by the existing TTS preferences.

## Best Possible Solution

Ship a focused Control UI Quick Settings TTS card backed by existing tts.status/providers/enable/disable/setProvider methods, plus narrow additive voice preference and preview methods if needed, with UI and gateway tests for the settings flow.

## Reproduction Assessment

Yes. Source inspection on current main is enough to reproduce the gap: Quick Settings has no TTS props, render path, or app wiring, and the UI tree has no consumer for the gateway TTS RPCs needed by the requested controls.

## Solution Assessment

Yes. A first-class Quick Settings TTS card using the existing backend TTS surface, with minimal additive voice persistence and preview support, is the narrow maintainable solution; #75479 appears aimed at that path but is not merged.
