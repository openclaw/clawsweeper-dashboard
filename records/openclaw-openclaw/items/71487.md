## Summary

Keep open. Current main still has TTS CLI/docs/backend RPC support but no first-class Control UI Settings toggle, provider selector, provider-filtered voice picker, or sample playback; the connected draft PR #75479 targets this gap but is still open and should be reviewed or replaced before this issue closes.

## What This Changes

The issue requests a discoverable Control UI Settings TTS section with enable/disable, default provider selection, default voice selection, and optional sample playback backed by the existing TTS preferences.

## Best Possible Solution

Land a focused Control UI Settings implementation, likely by reviewing and finishing #75479 or replacing it if needed, that reads TTS status/providers, persists enable/provider/default voice preferences through the agreed TTS contract, shows unavailable providers clearly, filters voices by provider, and uses provider-backed sample playback with focused UI and gateway coverage.
