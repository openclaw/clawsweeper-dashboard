## Summary

Keep open. Current main still has backend TTS RPCs and docs, but the Control UI Quick Settings surface does not expose a TTS toggle, provider selector, default voice picker, or sample playback; the matching implementation candidate #75479 is still open in the provided GitHub context, so closing this issue before that PR lands would be premature.

## What This Changes

Add a first-class Control UI Settings surface for enabling/disabling TTS, selecting a default provider, selecting a provider-filtered default voice, and optionally playing a short sample.

## Best Possible Solution

Ship a focused Control UI Quick Settings TTS card backed by existing tts.status/providers/enable/disable/setProvider calls, add minimal default-voice persistence and provider-backed preview RPCs if needed, and close this issue only after #75479 lands or a replacement PR provides the same user-facing behavior with tests.

## Reproduction Assessment

Yes. Source inspection against current main is a high-confidence reproduction path: the Settings UI props, render tree, and app wiring have no TTS controls, while backend TTS RPCs remain available but incomplete for default voice and preview.

## Solution Assessment

Yes. A first-class Settings card using the existing TTS RPC surface plus narrow additive voice preference and sample playback support is the maintainable path; the open draft PR appears aimed at that path but is not merged.
