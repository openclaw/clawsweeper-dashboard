## Summary

Keep open; the branch targets a real Ollama multi-provider routing gap, but the current patch is not merge-ready because it can pair a per-request URL with first-provider headers and it drops unrelated changelog entries.

## What This Changes

The PR recomputes the Ollama chat URL and SSRF policy from runtime `model.baseUrl`, wraps configured streams to stamp a base URL, adds stream runtime tests, and adds a changelog entry.

## Best Possible Solution

Resolve the Ollama URL, SSRF policy, and headers from the same per-request provider/model data, preserve the current changelog, and add a regression that proves credentials do not cross providers.

## Reproduction Assessment

Yes. A source-level reproduction is clear: use a first registered Ollama-compatible stream with base URL and headers A, then invoke the shared `api: "ollama"` path with a runtime model carrying base URL and headers B; the proposed patch changes the URL to B while retaining A's headers.

## Solution Assessment

No. Per-request URL routing is the right direction, but the patch must also route headers from the same runtime provider data and resolve the changelog conflict before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Route headers with the per-request URL:** `extensions/ollama/src/stream.ts:1033-1034`
  - body: With this added per-request `chatUrl`, a later Ollama-compatible provider can route to its runtime `model.baseUrl`, but the request still merges `defaultHeaders` captured when the first provider's stream was created. That can send provider A's Authorization or custom headers to provider B, or omit B's headers, so derive URL, SSRF policy, and headers from the same runtime model/provider request data and cover the cross-provider case.
  - confidence: 0.9
- **[P2] Restore the dropped changelog entries:** `CHANGELOG.md:19`
  - body: This hunk replaces the current Unreleased Fixes section with the Ollama bullet and removes current-main entries, so merging would erase release notes unrelated to this PR. Rebase and add the Ollama line without deleting the existing fixes.
  - confidence: 0.96

## Security Review
