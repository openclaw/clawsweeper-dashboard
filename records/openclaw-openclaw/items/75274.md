## Summary

This PR should stay open: it addresses a real multi-provider Ollama routing bug, but the current patch still separates the per-request URL from the headers used for that request and also drops existing current-main changelog entries. The header mismatch is security-sensitive because configured provider credentials can be sent to the wrong Ollama-compatible host.

## What This Changes

The PR changes the Ollama native stream path to derive the chat URL and SSRF policy from runtime model base URLs, wraps configured Ollama streams to stamp an effective base URL, adds regression tests, and adds a changelog entry.

## Best Possible Solution

Keep the narrow Ollama-plugin direction, but resolve base URL, SSRF policy, and request headers from the same runtime model/provider request, add a regression proving cross-provider headers do not leak, and rebase the changelog so the Ollama entry is added without deleting existing Unreleased fixes.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to register two Ollama-compatible providers with `api: "ollama"`, distinct `baseUrl` values, and distinct headers; the first registered stream handles the later runtime model, and this PR routes to the later URL while still merging the first provider's captured headers.

## Solution Assessment

No. Per-request URL and SSRF policy resolution is the right seam, but the patch is not complete until headers come from the same per-request provider source and the changelog conflict is resolved.

## Review Findings

Full review comments:

- **[P1] Route headers with the per-request URL:** `extensions/ollama/src/stream.ts:1029-1030`
  - body: When a later Ollama-compatible provider enters the first registered stream, this code switches `chatUrl` to runtime `model.baseUrl`, but the request still merges `defaultHeaders` captured from the first provider. That can send provider A's Authorization header to provider B's URL, or omit provider B's required headers. Resolve runtime headers from the same model/provider source and cover the cross-provider case.
