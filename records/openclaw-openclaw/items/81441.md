## Summary

Keep open: this is a plausible fix for a source-reproducible heartbeat leak, but the patch introduces one sanitizer bug that should block merge as written. There is also an overlapping open fix at https://github.com/openclaw/openclaw/pull/81434, so maintainers should choose one canonical branch for https://github.com/openclaw/openclaw/issues/81369.

## What This Changes

The PR routes heartbeat reply text through the assistant-visible sanitizer and adds sanitizer/heartbeat regression coverage for raw protocol scaffolding and media-bearing heartbeat payloads.

## Best Possible Solution

Land one canonical heartbeat/sanitizer fix with focused runner and sanitizer regression coverage, then close the linked heartbeat leak issue after merge.

## Reproduction Assessment

Yes. Current main source shows heartbeat replies are normalized from raw text without the canonical assistant-visible sanitizer, so a heartbeat payload containing only `<tool_calls>` or `<file_contents ...>` scaffolding is a high-confidence source reproduction path.

## Solution Assessment

No, not as written. The canonical-sanitizer direction is right, but this PR should not unwrap singular tool-call/tool-result XML bodies into normal visible text; either strip those whole protocol blocks or limit the marker-line pass to the bare plural scaffolding markers being fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Strip whole standalone tool-result blocks:** `src/shared/text/assistant-visible-text.ts:648-650`
  - body: The standalone-line regex includes singular `tool_call`, `function_call`, and `tool_result`, but this pass removes only the tag lines after the existing XML stripper declines plain-text bodies. A reply like `<tool_result>\nsecret\n</tool_result>` would become `secret`, so internal tool-result content can still be delivered as ordinary chat. Please either strip the whole wrapped protocol block for these singular tags or restrict this pass to the bare plural marker lines this PR is fixing.
  - confidence: 0.83
