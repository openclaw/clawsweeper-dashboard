## Summary

This PR is useful and should stay open, but it is not merge-ready because the relayable exec-completion heartbeat fallback can still restore raw scaffolding after the new sanitizer empties the main reply text.

## What This Changes

The PR routes heartbeat reply text through the assistant-visible sanitizer, extends that sanitizer for `<file_contents>` blocks and standalone tool-call tag lines, and adds heartbeat/sanitizer regression coverage.

## Best Possible Solution

Keep the centralized sanitizer approach, but make every heartbeat delivery and fallback path use sanitized assistant-visible text before sending, with an exec-completion regression test.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main takes heartbeat `payload.text`, does not apply the assistant-visible sanitizer, and sends `normalized.text` through durable channel delivery. The linked report also includes a redacted 2026.5.12-beta.4 Discord configuration and visible leak examples.

## Solution Assessment

No, not yet. The PR fixes the normal heartbeat reply path with the right shared sanitizer, but the exec-completion fallback needs the same sanitization or suppression before this is the narrow maintainable fix.

## Review Findings
