## Summary

Keep open. The PR targets a valid Telegram partial-streaming regression and includes after-fix runtime proof, but the patch introduces a blocker: the new delta coalescing can append the first answer fragment after a tool-progress preview to the progress-status draft because it reads `answerLane.lastPartialText` before clearing answer-lane progress state.

## What This Changes

The PR adds Telegram draft partial coalescing logic and a regression test so delta-shaped partial fragments can reach the existing 30-character preview debounce cumulatively.

## Best Possible Solution

Keep the debounce-preserving delta coalescing, but make answer text coalesce only against prior answer text, not the tool-progress status draft that shares the answer lane.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: in current main, partial mode sends each short fragment to the draft stream independently, and the draft stream suppresses the first preview below `minInitialChars: 30`. I did not run a live Telegram bot in this read-only review.

## Solution Assessment

No. Coalescing delta fragments is the right direction, but this implementation resolves deltas against `lastPartialText` before clearing answer-lane tool-progress state, so a short answer fragment after tool progress can be appended to status text instead of replacing it.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Avoid coalescing answer text with progress drafts:** `extensions/telegram/src/bot-message-dispatch.ts:765`
  - body: In partial mode, tool progress is enabled by default and the dispatcher stores the rendered progress status in `answerLane.lastPartialText`. This new call resolves the next answer partial against that value before the answer-lane branch clears progress state, so a short first answer fragment after a tool event is treated as a delta and appended to the status draft instead of replacing it. Track the last answer partial separately, or clear/ignore progress text before coalescing.
  - confidence: 0.88
