## Summary

Keep this PR open: the reported current-main UI behavior is real and the dedicated tool-sidebar direction is maintainable, but the proposed patch introduces a render-path regression by building and JSON-parsing sidebar payloads eagerly for collapsed tool cards.

## What This Changes

The PR adds a tool-specific Control UI sidebar content kind, renders tool input/output in a dedicated sidebar view with raw fallback and tests, routes non-canvas tool-card summary clicks to the side panel, and adds a changelog entry.

## Best Possible Solution

Keep the dedicated `tool` sidebar contract, but make sidebar payload construction lazy on click and either defer or size-cap JSON parsing/pretty-printing so collapsed chat history cannot parse large tool results during normal render; add a focused regression test for that path before merge.

## Reproduction Assessment

Yes. Current main reproducibly sends tool details through markdown and the markdown renderer collapses JSON fences; the PR’s own focused tests model that path. The new review finding is also reproducible by rendering a collapsed non-canvas tool card with a large JSON output, because PR head constructs `buildToolCardSidebarContent(card)` during render.

## Solution Assessment

No, not as submitted. The dedicated sidebar renderer is the right product direction, but the render-time construction should be moved into the click handler or guarded with the existing large-JSON parsing limit before this is the narrowest maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Build tool sidebar content lazily:** `ui/src/ui/chat/tool-cards.ts:417-418`
  - body: This computes `sidebarActionContent` on every render, even when the card is collapsed and no click occurs. `buildToolCardSidebarContent` now runs `JSON.parse`/pretty-printing on tool input and output without the 20k render-loop cap used elsewhere, so a large JSON tool result can freeze chat history merely by being rendered. Please build this content inside the summary click/sidebar action path, or apply the same size guard before parsing.
  - confidence: 0.86
