## Summary

Keep open: the dedicated tool sidebar is the right direction, but the PR still introduces unbounded JSON parse/pretty-print work while rendering collapsed tool rows, so it should not merge until that path is capped or deferred.

## What This Changes

This PR adds a `tool` sidebar content kind, renders tool input/output in a dedicated Control UI side panel, routes non-canvas collapsed tool-card clicks to that panel, updates styles/tests, and adds a changelog entry.

## Best Possible Solution

Land a revised PR that keeps the dedicated escaped tool sidebar and summary-to-sidebar behavior, while bounding or lazily deferring JSON parsing/pretty-printing and adding focused large-payload coverage.

## Reproduction Assessment

Yes. Current main is source-reproducible by building tool sidebar content as markdown with JSON fences and rendering it through the `json-collapse` markdown path; the PR regression is source-reproducible by rendering a collapsed non-canvas tool card with a large JSON `outputText`.

## Solution Assessment

No, not as submitted. The dedicated `tool` sidebar content kind is the narrow maintainable fix, but unbounded JSON parsing in the render path should be capped or moved to the click/sidebar path before merge.

## Review Findings

Overall correctness: patch is incorrect
