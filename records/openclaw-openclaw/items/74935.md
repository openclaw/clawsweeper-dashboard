## Summary

Keep this PR open: it addresses a real current-main UI bug and includes the right dedicated-sidebar direction, but the latest PR head still parses and pretty-prints tool-controlled JSON during collapsed row rendering, creating a concrete Control UI freeze risk before merge.

## What This Changes

The PR adds a dedicated `tool` sidebar content kind and routes non-canvas tool-card clicks to a side panel that shows tool input/output as expanded blocks with raw-text fallback.

## Best Possible Solution

Land a revised version that keeps the dedicated escaped tool sidebar, but caps or lazily defers JSON detection/pretty-printing and adds large-payload regression coverage.

## Reproduction Assessment

Yes. Current main is source-reproducible by building tool sidebar content as markdown with JSON fences and rendering it through the `json-collapse` markdown path; the PR regression is source-reproducible from the unbounded parse path at PR head.

## Solution Assessment

No, not as submitted. The dedicated `tool` sidebar content kind is the narrow maintainable fix, but JSON parsing should be bounded or deferred before merge.
