## Summary

Keep open: the PR fixes the right UI surface, but PR head still parses and pretty-prints unbounded tool-controlled JSON while rendering collapsed tool rows, which can freeze the Control UI before the sidebar is opened.

## What This Changes

The PR adds a dedicated `tool` sidebar content kind, renders tool input/output in the Control UI side panel, routes non-canvas collapsed tool-card clicks to it, updates styles/tests, and adds a changelog entry.

## Best Possible Solution

Land a revised PR that keeps the dedicated escaped tool sidebar and summary-to-sidebar behavior, while bounding or lazily deferring JSON parsing/pretty-printing and adding focused large-payload coverage.

## Reproduction Assessment

Yes. Current main is source-reproducible by routing tool sidebar content through markdown JSON fences, and the PR regression is source-reproducible by rendering a collapsed non-canvas tool card with a multi-MB JSON `outputText`.

## Solution Assessment

No, not as submitted. The dedicated `tool` sidebar content kind is the narrow maintainable direction, but render-time unbounded JSON parsing should be capped or deferred before merge.
