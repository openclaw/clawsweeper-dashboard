## Summary

Keep open: the no-op preflight compaction fix is useful, but the budget forwarding is incomplete because the queued compaction wrapper can still pass the active model budget to harness/context-engine compaction without clamping to the selected compaction model.

## What This Changes

The branch accepts successful no-op preflight compaction, forwards the caller context budget into compaction, clamps direct runtime compaction budgets, and adds focused tests.

## Best Possible Solution

Clamp the effective compaction budget at the queued wrapper boundary before any harness, context-engine, or direct runtime delegation, while keeping the successful no-op preflight behavior and focused regression coverage.

## Reproduction Assessment

Yes. Source inspection shows current main throws for `{ ok: true, compacted: false }` preflight results, and the PR’s unit test models that path without needing a live provider.

## Solution Assessment

No. The no-op handling is narrow, but the budget clamp should happen in the queued `compactEmbeddedPiSession` boundary so direct runtime, harness, and ownsCompaction paths receive the same capped budget.
