## What This Changes

The PR makes `suppressDefaultToolProgressMessages` suppress default tool/progress messages even when session verbose mode is enabled, and updates two dispatch regression tests.

## Best Possible Solution

Settle channel-suppression precedence, then land this or an equivalent shared dispatch fix with regression coverage if channel config should beat session verbose.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: current main sets `verboseLevel: "on"`, passes `suppressDefaultToolProgressMessages: true`, and expects `sendToolResult` to deliver text/progress despite suppression.

## Solution Assessment

Yes, with a policy caveat. Returning the flag directly is the narrow shared dispatch fix if channel suppression is authoritative; a separate force-suppression flag would be safer only if maintainers want `/verbose` to keep overriding preview-dedupe callers.
