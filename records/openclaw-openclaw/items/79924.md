## What This Changes

The PR makes `suppressDefaultToolProgressMessages` unconditional in shared dispatch and updates two dispatcher tests so verbose mode no longer emits default tool, plan, or working-status progress when suppression is set.

## Best Possible Solution

Land this shared dispatch fix after current-head real Telegram proof is attached and CI is green, then let the linked bug close through the PR.

## Reproduction Assessment

Yes, source-reproducible. Current main’s dispatcher gates suppression on `!shouldEmitVerboseProgress()`, and existing tests assert default progress is delivered when verbose is on despite the suppression flag.

## Solution Assessment

Yes. Making the shared suppression predicate unconditional is the narrowest maintainable fix because it preserves the existing media/error/exec-approval exceptions and channel-owned progress callbacks while removing only the verbose escape hatch.
