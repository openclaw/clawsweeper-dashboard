## Summary

Keep open for normal maintainer review and CI. The PR is a narrow bug fix: current main lets verbose progress bypass an explicit default-progress suppression flag, while the option contract and Telegram docs describe suppression as authoritative for generic tool/progress chatter.

## What This Changes

The PR changes shared reply dispatch so `suppressDefaultToolProgressMessages` ignores verbose progress mode and updates two dispatch tests to expect no default tool/status messages when suppression is set.

## Best Possible Solution

Land the focused shared-dispatch fix with the updated regression tests, then let the linked bug report close from the merge path.

## Reproduction Assessment

Yes. A high-confidence source reproduction is dispatching with `verboseLevel: "on"` and `replyOptions.suppressDefaultToolProgressMessages: true`; current-main tests expect leaked `sendToolResult` calls in that setup.

## Solution Assessment

Yes. Making the suppression flag unconditional is the narrowest maintainable fix because it matches the option contract and preserves channel callbacks plus media, error, and exec-approval exceptions.
