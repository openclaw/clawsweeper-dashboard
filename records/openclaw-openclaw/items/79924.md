## What This Changes

The PR makes shared reply dispatch treat `suppressDefaultToolProgressMessages` as unconditional and updates two dispatcher tests so verbose mode no longer emits default tool, plan, or working-status progress when suppression is set.

## Best Possible Solution

Land the shared dispatch fix after the contributor attaches inspectable current-head Telegram proof and CI is green, then let the linked bug close through the PR.

Yes, source-reproducible. Current main’s dispatcher gates suppression on `!shouldEmitVerboseProgress()`, and current tests assert verbose mode delivers default tool, plan, and approval progress despite the suppression flag.

## Solution Assessment

Yes. Making the shared suppression predicate unconditional is the narrowest maintainable fix because it preserves existing media, error, and exec-approval exceptions while removing only the verbose escape hatch.
