Full review comments:

- **[P1] Reject hallucinated mutation paths instead of retargeting them:** `src/agents/agent-tools.params.ts:153`
  - body: `wrapToolParamValidation` uses this helper for write/edit path keys, so a mutation call such as `write({ path: "report.docodex" })` is silently executed against `report.docx` instead of forcing the model to retry with the intended path. That can mutate a different path than the model requested and conflicts with the safer sibling behavior.
  - confidence: 0.9
- **[P1] Keep the outer workspace guard from hiding hallucinated paths:** `src/agents/agent-tools.read.ts:774`
  - body: This guard can wrap write/edit tools before their inner validation, so correcting the suffix here hides the original `.docodex` path from any mutation-path rejection logic. The guard should remain XML-only and let the file tool boundary decide whether to correct or reject.
  - confidence: 0.86
