## Summary

Keep open: current main still emits deprecated Feishu `action` button containers, and the open closing PR https://github.com/openclaw/openclaw/pull/79832 is the active fix path, so this issue should stay open until that PR merges.

## What This Changes

The issue requests replacing deprecated Feishu V2 `action` button containers with V2-compatible `column_set` rows for approval, quick-action, and presentation cards.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/79832 or an equivalent Feishu-owned fix that emits V2 `column_set` button rows across sanitizer, presentation, approval, and launcher paths with focused tests and live Feishu proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Feishu approval, launcher, and presentation button cards still serialize `tag: "action"` into an interactive Feishu card on current main; I did not perform a live Feishu send, so this is source-reproducible rather than reproduced.

## Solution Assessment

Yes. A Feishu-plugin-local helper that emits `column_set` rows and is reused by the sanitizer, presentation renderer, approval card, and launcher card is the narrow maintainable fix; a JSON-stringify rewrite should remain only a local workaround.
