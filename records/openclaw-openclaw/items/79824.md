## Summary

Keep open: current main still emits Feishu interactive cards with deprecated `tag: "action"` button containers on the reported approval, launcher, and presentation paths, and the active fix is the open closing PR at https://github.com/openclaw/openclaw/pull/79832.

## What This Changes

Replace the Feishu plugin's deprecated card V2 `action` button containers with V2-supported `column_set` rows so approval and quick-action cards can be delivered.

## Best Possible Solution

Land a Feishu-owned `column_set` migration with focused tests and live Feishu/Lark validation, then close this issue as fixed by the merged PR.

## Reproduction Assessment

Yes. Source inspection on current main shows Feishu approval, launcher, and presentation-button cards serialize `tag: "action"` through `sendCardFeishu` as `msg_type: "interactive"`; I did not call Feishu live, so this is source-reproducible rather than reproduced.

## Solution Assessment

Yes. The narrow maintainable fix is local to `@openclaw/feishu`: render button rows as `column_set`, keep sanitizer compatibility, and update Feishu-focused tests; the linked PR should be reviewed and landed rather than closing this issue first.
