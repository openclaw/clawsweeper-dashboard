## Summary

Keep open: current main still emits Feishu schema-2.0 interactive cards with deprecated `tag: "action"` button containers on the reported approval, launcher, sanitizer, and presentation paths, and an open fixing PR already references this issue with closing syntax.

## What This Changes

The issue reports that Feishu approval, quick-action, and presentation button cards are rejected because `@openclaw/feishu` emits deprecated V1 `action` containers under Feishu card schema V2.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/79832, keeping the fix inside `@openclaw/feishu` with shared V2-compatible button-row rendering and focused Feishu tests.

## Reproduction Assessment

Yes. Source inspection on current main shows every relevant Feishu button-bearing card path still serializes `tag: "action"`; I did not call the live Feishu API, so this is source-reproducible rather than live-reproduced.

## Solution Assessment

Yes. The narrow maintainable fix is an owner-local Feishu helper that emits V2-compatible `column_set` button rows and is reused by the sanitizer, presentation renderer, approval card, and launcher card; the open fixing PR is the right object to review next.
