## Summary

Keep open: current main still emits Feishu schema-2.0 interactive cards with deprecated `tag: "action"` button containers on the reported approval, launcher, native-card, and presentation paths. The linked implementation PR is still open, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that Feishu approval and button-bearing cards emit the deprecated V1 `action` container, causing Feishu V2 schema error 230099/200861 and failed approval delivery.

## Best Possible Solution

Repair and land https://github.com/openclaw/openclaw/pull/79832 or an equivalent Feishu-owned `column_set` migration, then close this issue as implemented after the fix reaches main.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: any Feishu approval, quick-action launcher, native-card sanitizer input, or presentation button block currently serializes `tag: "action"` into an interactive card; I did not run a live Feishu send in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is an owner-local Feishu helper/sanitizer that emits schema-V2-compatible button rows and is reused by the sanitizer, presentation renderer, approval card, and launcher card; a JSON stringify rewriter would be a weaker workaround.
