## Label Justifications

- `P1`: Feishu approval and quick-action cards with buttons fail to deliver, breaking a real channel approval workflow for affected users.
- `impact:message-loss`: The reported failure drops button-bearing Feishu cards on the chat side before approvers can see or click them.

## Summary

Keep open: current main and the latest release still emit Feishu JSON 2.0 cards with deprecated `tag: "action"` button containers, and the linked open fix PR now owns the implementation path.

## What This Changes

The issue asks the bundled Feishu plugin to stop emitting V2-incompatible `action` card containers for button-bearing approval, quick-action, and presentation cards.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79832, or an equivalent Feishu-plugin-local helper that renders V2-compatible button rows and updates the affected tests.

## Reproduction Assessment

Yes. Source inspection on current main shows approval, launcher, presentation, and native-card sanitization paths still serialize `tag: "action"`; the reporter and linked PR add live Feishu logs for the rejected and working shapes.

## Solution Assessment

Yes. The narrow maintainable direction is to replace the Feishu plugin's V1 action-row output with a shared V2-compatible renderer while preserving safe legacy input normalization.
