## Summary

Keep open: current main still emits Feishu interactive cards with deprecated `tag: "action"` button containers on the reported approval, quick-action, native-card, and presentation-button paths. The linked open fix PR is the right closure path, so this issue should remain open until that PR lands or is replaced.

## What This Changes

Feishu approval, quick-action, and presentation button cards should stop emitting deprecated V2-incompatible `action` containers and use a supported button-row shape instead.

## Best Possible Solution

Complete and land the Feishu-plugin fix in https://github.com/openclaw/openclaw/pull/79832, or an equivalent replacement, then close this issue as implemented.

## Reproduction Assessment

Yes. Source inspection on current main shows approval, quick-action, native-card, and presentation-button paths still serialize `tag: "action"` into Feishu interactive cards; I did not run a live Feishu send in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is a Feishu-owned helper that emits V2-compatible button rows and is reused by sanitizer, presentation, approval, and launcher card builders; a JSON-stringify rewrite should remain only a local workaround.
