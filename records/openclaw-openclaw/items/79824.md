## Summary

Keep open: current main and the latest release still emit deprecated Feishu `tag: "action"` button rows on the reported paths, and the linked implementation PR remains open with closing syntax for this issue.

## What This Changes

The issue reports Feishu approval, quick-action, and presentation-button cards failing because the Feishu plugin emits V2-incompatible `tag: "action"` containers instead of `column_set` button rows.

## Best Possible Solution

Land `https://github.com/openclaw/openclaw/pull/79832` or an equivalent Feishu-plugin fix that emits V2-compatible `column_set` button rows across native-card, presentation, approval, and launcher paths with focused tests and Feishu proof.

## Reproduction Assessment

Yes. Source inspection on current main shows the reported Feishu approval, launcher, native-card sanitizer, and presentation-button paths still serialize `tag: "action"`; I did not perform a live Feishu API send in this read-only review.

## Solution Assessment

Yes. The narrow maintainable solution is an owner-local Feishu card helper that emits `column_set` rows and is reused by the sanitizer, shared presentation renderer, approval card, and launcher card; the linked PR is the current implementation candidate.
