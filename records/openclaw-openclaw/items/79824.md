## Summary

Keep open: the reported Feishu button-card failure is still present on current main, and https://github.com/openclaw/openclaw/pull/79832 is an open closing PR for the same issue, so closing this before merge would be premature.

## What This Changes

The issue asks OpenClaw to stop emitting deprecated Feishu card V2 `action` button containers and use a schema-compatible button row shape for approvals, quick actions, and presentation buttons.

## Best Possible Solution

Review and land or supersede https://github.com/openclaw/openclaw/pull/79832, then close this issue once the Feishu plugin emits V2-compatible button rows on main.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: Feishu approval, quick-action launcher, native-card sanitizer, and presentation-button rendering all serialize `tag: "action"`; I did not run a live Feishu send, so this is source-reproducible rather than independently reproduced.

## Solution Assessment

Yes. The narrow maintainable solution is an Feishu-plugin helper that emits schema V2-compatible `column_set` button rows and is reused by sanitizer, presentation, approval, and launcher paths; the open linked PR is the right implementation path to review before closure.
