## Summary

Keep open: current main and the latest release still emit deprecated Feishu `tag: "action"` button containers on the reported approval and presentation paths, and the issue is paired with an open closing PR that has not merged.

## What This Changes

The issue reports that Feishu exec/plugin approval cards and shared button cards fail because the Feishu plugin renders schema-2.0 interactive cards with deprecated `action` containers instead of V2-compatible button rows.

## Best Possible Solution

Land a Feishu-owned fix that replaces action containers with a shared V2-compatible button-row helper across native sanitization, presentation buttons, approval cards, and launcher cards, then close this issue after merge and focused Feishu validation.

## Reproduction Assessment

Yes. Source inspection of current main shows every reported button-bearing Feishu card path still serializes `tag: "action"`; I did not perform a live Feishu send in this read-only review, so this is source-reproducible rather than directly reproduced.

## Solution Assessment

Yes. The narrow maintainable direction is an owner-local Feishu helper that emits V2-compatible button rows and is reused by the sanitizer, presentation renderer, approval card, and launcher card; the linked open PR is the active implementation path.
