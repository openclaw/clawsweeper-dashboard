## Summary

Keep open: current main and the latest shipped tag still emit deprecated Feishu `tag: "action"` button rows on approval, quick-action, native-card, and shared presentation paths, and the linked open fix PR should land before this issue is closed.

## What This Changes

Replace deprecated Feishu card V2 `action` button containers with V2-compatible `column_set` rows for approval, quick-action, and presentation card sends.

## Best Possible Solution

Land a Feishu-owned helper/sanitizer migration through the linked PR, with tests updated to assert `column_set` output, then close this issue as implemented after merge.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: Feishu approval, launcher, native-card, and presentation button sends serialize `tag: "action"` into `sendCardFeishu` as an interactive card; I did not rerun a live Feishu send in this read-only review.

## Solution Assessment

Yes. The narrow maintainable solution is to fix the Feishu plugin’s card construction and sanitizer to emit V2-compatible `column_set` rows, rather than relying on a JSON stringify rewriter or a core workaround.
