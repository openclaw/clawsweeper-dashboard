## Summary

Keep open: current main and the latest release still emit deprecated Feishu `action` button containers, and the open linked PR https://github.com/openclaw/openclaw/pull/79832 is the implementation candidate that should resolve this issue after review/landing.

## What This Changes

Feishu button-bearing interactive cards should use a V2-compatible layout so exec/plugin approval cards and quick-action cards deliver successfully.

## Best Possible Solution

Land a Feishu-owned `column_set` migration through https://github.com/openclaw/openclaw/pull/79832 or an equivalent narrow fix, with focused Feishu card tests and live card-send validation when credentials are available.

## Reproduction Assessment

Yes. Current main source inspection gives a high-confidence reproduction path: any Feishu approval, launcher, or presentation card with buttons serializes `tag: "action"` into an interactive card, and the linked PR body supplies redacted live Feishu rejection logs for that path.

## Solution Assessment

Yes. The narrow maintainable solution is to fix the Feishu plugin renderer/sanitizer to emit V2-compatible `column_set` button rows while preserving legacy input compatibility, then close this issue after the fix merges.
