## Summary

Keep open: current main and the latest shipped release still emit Feishu Schema 2.0 cards with deprecated `tag: "action"` button containers, and an open fixing PR already owns this issue with closing syntax.

## What This Changes

The issue asks the bundled Feishu plugin to replace deprecated V2 `action` button containers with `column_set` rows for approval, quick-action, and presentation cards.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/79832 with a Feishu-owned `column_set` renderer/sanitizer, updated tests, and live Feishu validation for one approval card plus one presentation-button card.

## Reproduction Assessment

Yes. Source inspection on current main shows every affected Feishu button-card path still serializes `tag: "action"`, and the linked PR body supplies redacted live before/after logs for the Feishu API rejection and fixed click path.

## Solution Assessment

Yes. The narrow maintainable fix is to change the Feishu plugin’s local card renderer/sanitizer to emit V2-compatible `column_set` rows and keep the issue open until the linked implementation lands.
