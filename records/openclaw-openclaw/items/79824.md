## Summary

Keep open: current main and the latest release still emit Feishu Card JSON 2.0-rejected `action` rows for approval and quick-action cards, and the linked open PR is the active fix path for the same delivery failure.

## What This Changes

The issue reports Feishu/Lark rejecting OpenClaw approval and quick-action cards because the bundled Feishu plugin still wraps buttons in the deprecated `action` container.

## Best Possible Solution

Land a refreshed version of https://github.com/openclaw/openclaw/pull/79832, or an equivalent Feishu-plugin patch, that emits Card JSON 2.0-supported button rows for approval, quick-action, and native-card paths with focused regression coverage.

## Reproduction Assessment

Yes, at source level: current main and `v2026.6.8` still build `tag: "action"` rows in the Feishu approval and quick-action card paths. I did not run a live Feishu tenant in this read-only review, but multiple issue comments and the linked PR body include matching real Feishu `230099` / `200861` evidence.

## Solution Assessment

Yes in direction: converting the remaining Feishu-plugin action rows to a Card JSON 2.0-supported `column_set` or equivalent button layout is the narrow owner-boundary fix. The issue should stay open until the linked PR or an equivalent refreshed patch merges.
