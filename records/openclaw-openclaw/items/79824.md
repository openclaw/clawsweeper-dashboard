## Label Justifications

- `P1`: The report describes a production Feishu channel workflow where interactive approval cards fail to reach users, blocking real approval flows.
- `impact:message-loss`: The affected Feishu cards are rejected by the chat API before delivery, so approval and quick-action messages are lost on the chat side.

## Summary

Codex review keeps this open: current `origin/main` and the latest release still emit Feishu V2-rejected `action` card rows, and the linked open PR is the active fix path for the same approval/card delivery failure.

## What This Changes

Feishu approval and quick-action card rows should stop using the V2-rejected `action` container so button-bearing cards deliver as interactive cards.

## Best Possible Solution

Land a refreshed version of https://github.com/openclaw/openclaw/pull/79832, or an equivalent Feishu-plugin change, that emits `column_set` rows for approval, quick-action, and native button card paths with matching Feishu tests.

## Reproduction Assessment

Yes, at source level: current `origin/main` and `v2026.6.8` still construct the rejected `tag: "action"` Feishu card rows in the reported approval and quick-action paths. I did not run a live Feishu tenant repro in this read-only sweep, but the issue and linked PR include matching live error/proof logs.

## Solution Assessment

Yes, the plugin-local conversion from legacy `action` rows to Feishu V2 `column_set` rows is the narrow owner-boundary fix. The issue should stay open until the linked PR or an equivalent patch is merged.
