## Summary

Close as duplicate: the no-detail `sqlite-vec unavailable` warning in this MiniMax setup matches the already-open diagnostic bug tracked in https://github.com/openclaw/openclaw/issues/75624, while current docs require a supported `memorySearch` embedding provider for semantic vector recall.

## What This Changes

The issue reports repeated `chunks_vec not updated` warnings and degraded builtin SQLite vector recall when Active Memory runs on Ubuntu with MiniMax and no memory slot configured.

## Best Possible Solution

Keep https://github.com/openclaw/openclaw/issues/75624 as the canonical fix for the misleading warning, and have users configure a supported `memorySearch` embedding provider when they want semantic vector recall.

## Reproduction Assessment

Yes, by source inspection. A MiniMax-only/no-embedding-provider setup can write FTS-only chunks with `vectorReady: false` and no sqlite-vec `loadError`, producing the reported no-detail warning text.

## Solution Assessment

Yes. Closing this duplicate and routing the remaining diagnostic wording bug to https://github.com/openclaw/openclaw/issues/75624 is narrower than splitting the same warning surface across two open issues.
