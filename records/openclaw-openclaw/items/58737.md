## Summary

Keep open. Current main still has the reported Slack identity mismatch: initial Slack sends can carry the resolved agent name/avatar, but the draft, preview-finalize, and status-final edit paths still update messages without custom authorship. The open closing PR #58800 remains the right item to review before closing this issue.

## What This Changes

The issue requests that edited Slack messages preserve the configured agent display name and avatar instead of reverting to the Slack app default identity.

## Best Possible Solution

Keep the fix inside the Slack plugin: preserve agent identity for every OpenClaw-authored Slack update path if live Slack behavior confirms `chat.update` can retain custom authorship, or switch those paths to a Slack-specific fallback that avoids losing the custom author identity. The final patch should include regression coverage for edit helper payloads, draft streaming, preview finalization, and status-final completion updates.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: configure a Slack custom agent identity, trigger draft streaming, preview finalization, or `status_final`, and current main reaches edit/update calls that omit the identity fields present on the initial send path.

## Solution Assessment

Unclear until #58800 is validated against Slack's API behavior. The affected code boundary is right, but Slack's documented contract suggests custom authorship is a `chat.postMessage` feature, so the PR may need a live workspace check or an alternate Slack-local edit strategy.
