## Summary

Keep open: current main and v2026.6.8 still omit Slack agent identity from the `chat.update` edit paths, and there are now open closing PRs that should own the fix review instead of closing this issue early.

## What This Changes

Slack edited messages should preserve the configured agent display name and avatar after tool-call, streaming, or final-preview edits.

## Best Possible Solution

Land one narrow Slack-plugin fix that forwards the resolved agent identity through the shared edit helper and agent-reply edit callers, keeps missing-scope fallback parity with the send path, and leaves approval or interaction UI edits under the bot identity.

## Reproduction Assessment

Yes. Current main resolves and sends Slack agent identity on initial posts, but the draft-stream and preview-finalize edit paths call `chat.update` without that identity.

## Solution Assessment

Yes. The best fix is the narrow Slack-plugin path already represented by the open PRs: extend `editSlackMessage` to accept existing Slack identity data and forward it only from agent-reply edit callers, rather than adding new config or broad channel behavior.
