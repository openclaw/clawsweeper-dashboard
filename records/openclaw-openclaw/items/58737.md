## Summary

Keep open: current main still omits the resolved Slack agent identity from edit paths, and the live closing PR remains open and unmerged. The issue is source-reproducible and should stay open until https://github.com/openclaw/openclaw/pull/58800 or an equivalent Slack-plugin fix lands.

## What This Changes

The issue requests preserving a configured Slack agent display name and avatar when OpenClaw edits a previously posted Slack message.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/58800 with a Slack-plugin-only fix that preserves visible agent identity across edit helper, draft edit, preview finalization, and status-final draft updates, backed by regression tests and live/API proof.

## Reproduction Assessment

Yes, at source level. Configure Slack with `chat:write.customize`, route a custom-identity agent, and trigger a draft/preview/status edit path; current main resolves identity for sends but does not pass it through edit helper/callers.

## Solution Assessment

Yes on direction, but not complete on main. The maintainable boundary is Slack-plugin identity threading through edit paths, with send-path missing-scope fallback parity and live/API proof for Slack edit authorship behavior.
