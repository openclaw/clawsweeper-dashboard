## What This Changes

The issue asks OpenClaw to preserve a routed Slack agent's custom display name and avatar when editing existing Slack messages.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/58800 with a Slack-plugin fix that threads `SlackSendIdentity` through every edit path, preserves send-path missing-scope fallback behavior, and includes focused tests plus redacted live Slack proof.

## Reproduction Assessment

Yes, at source level. Current main resolves `slackIdentity` for outbound sends, but draft edit and preview finalization paths call `chat.update` without forwarding that identity, matching the reported Slack edit flow.

## Solution Assessment

Yes on boundary, but not complete on main. The maintainable solution belongs in the Slack plugin by reusing the existing identity contract for edit paths and proving the behavior in a real Slack workspace.
