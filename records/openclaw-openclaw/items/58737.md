## What This Changes

The issue asks for edited Slack agent replies to retain the configured custom display name and avatar instead of reverting to the Slack bot default identity.

## Best Possible Solution

Review and land one narrow Slack-plugin PR that forwards the existing `SlackSendIdentity` through `editSlackMessage` and all agent-reply edit callers, with live or otherwise convincing Slack update proof, then close this issue after merge.

## Reproduction Assessment

Yes. Current main resolves custom Slack agent identity for sends, but the edit helper and agent-reply edit callers do not pass identity into `chat.update`; I did not run a live Slack workspace reproduction in this read-only review.

## Solution Assessment

Yes, with proof. The narrow maintainable fix is to keep the behavior inside the Slack plugin by threading the existing `SlackSendIdentity` through edit paths; because Slack docs are asymmetric between post and update, the landing PR should prove real Slack behavior or document the accepted fallback.
