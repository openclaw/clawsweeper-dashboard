## Summary

Keep open: current main and the latest release still drop custom Slack identity on edit/update paths, and the open closing PR remains unmerged with review/proof blockers. The bug is source-reproducible, so this should stay open until https://github.com/openclaw/openclaw/pull/58800 or an equivalent Slack-plugin fix lands.

## What This Changes

The issue reports that Slack messages initially sent with a custom agent display name/avatar lose that identity when OpenClaw later edits the message with `chat.update`.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/58800 with a Slack-plugin-only fix that preserves visible agent identity across edit helper, draft edit, preview finalization, and status-final updates, backed by regression tests and live/API proof.

## Reproduction Assessment

Yes, at source level. Configure a Slack route with a custom agent identity and trigger a draft/preview/status edit path; current main resolves identity for sends but omits it from the edit helper and key edit callers.

## Solution Assessment

Yes on direction, but not yet on the open PR revision. The maintainable boundary is Slack-plugin identity threading with edit-path tests, plus Slack-supported proof or strategy for `chat.update` authorship behavior.
