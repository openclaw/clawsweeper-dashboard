## Summary

Keep open: current main and the latest shipped release still omit Slack agent identity from `chat.update` edit paths, while the linked closing PR is open, unmerged, and conflict-marked. The linked PR discussion is useful and matches the source-level root cause, but this issue should remain open until that PR or an equivalent Slack-plugin fix lands.

## What This Changes

The issue reports that Slack messages posted with a custom agent display name/avatar revert to the Slack app default after OpenClaw edits them.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/58800 with a Slack-plugin-only fix that threads the resolved `SlackSendIdentity` through edit helper, draft edit, preview finalization, and status-final update paths, with missing-scope fallback parity and redacted live Slack proof.

## Reproduction Assessment

Yes, at source level. Configure Slack with `chat:write.customize`, route a custom-identity agent, and trigger a draft/preview/status edit path; current main resolves identity for sends but edit helpers and preview finalization do not pass that identity to `chat.update`.

## Solution Assessment

Yes on direction, but it is not complete on main. The maintainable fix belongs in the Slack plugin by reusing the existing `SlackSendIdentity` contract for edit paths and proving Slack edit authorship behavior in a real workspace.
