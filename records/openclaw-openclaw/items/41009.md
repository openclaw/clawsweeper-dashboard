Keep this PR open: it targets a real config-migration bug that current main still lacks, but the branch is not merge-ready because it targets the retired migration surface, relaxes the `auto` bind validation guard, and still lacks after-fix real behavior proof.

## What This Changes

The PR adds a legacy config migration, legacy detection rule, shared bind helper, validation reuse, and regression tests to normalize explicit non-loopback Tailscale Serve/Funnel binds to `loopback`.

## Best Possible Solution

## Solution Assessment
