## Summary

Keep open. The underlying crash-loop bug is still source-reproducible on current main, but this PR is not mergeable as-is because it targets the retired config migration surface and relaxes the current loopback-only validation for explicit `gateway.bind="auto"`.

## What This Changes

The branch adds a legacy config rule, migration, helper reuse, and tests to normalize explicit non-loopback Tailscale Serve/Funnel gateway binds to `loopback`.

## Best Possible Solution

Port the normalization and legacy rule into `src/commands/doctor/shared/legacy-config-migrations.runtime.gateway.ts`, preserve strict validation for explicit `auto`, add focused regression coverage, and require redacted real behavior proof before merge.

## Reproduction Assessment

Yes, at source level. Current main rejects `gateway.bind="lan"` with `gateway.tailscale.mode="serve"`, and the active doctor/shared migration registry has no repair for that combination; no live launchd crash-loop was run in this read-only review.

## Solution Assessment

No. The migration direction is right, but this branch targets retired files and changes validation so explicit `auto` is treated as safe; the maintainable fix is to port the migration into the active registry while keeping the current validation invariant.
