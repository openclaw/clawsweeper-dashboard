## Summary

Keep open: the linked startup crash-loop bug still has a clear source path on current main, and this PR remains the canonical implementation path, but the branch is not correct as-is because it targets obsolete migration files, relaxes the Serve/Funnel validation contract for explicit `auto`, and lacks after-fix real behavior proof.

## What This Changes

The PR adds a legacy config migration/rule, shared bind helpers, validation reuse, and regression tests to normalize explicit non-loopback Tailscale Serve/Funnel binds to `loopback`.

## Best Possible Solution

Port explicit non-loopback Serve/Funnel bind normalization into the active doctor/shared gateway migration registry with a matching legacy rule, keep explicit `auto` rejected by validation, and add focused startup/doctor regression coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction is `gateway.bind="lan"` with `gateway.tailscale.mode="serve"`: current validation rejects it, startup asserts the invalid snapshot, and the active migration registry lacks a matching repair.

## Solution Assessment

No. The migration direction is right, but this PR should be ported to the current doctor/shared migration architecture and should keep validation-specific handling for explicit `auto` separate from the legacy migration predicate.
