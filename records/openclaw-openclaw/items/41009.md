## Summary

Keep open: the underlying crash-loop bug still exists on current main and this PR is the canonical implementation path, but the current branch is not merge-ready because it targets a retired migration surface, weakens the Serve/Funnel bind validation for explicit `auto`, and still lacks real behavior proof.

## What This Changes

The PR adds a legacy config migration, legacy detection rule, shared bind helper, validation reuse, and regression tests to normalize explicit non-loopback Tailscale Serve/Funnel binds to `loopback`.

## Best Possible Solution

Port explicit non-loopback Serve/Funnel bind normalization into the active doctor/shared gateway migration registry with a matching legacy rule and focused regression coverage, while preserving strict validation for explicit `auto`.

## Reproduction Assessment

Yes. Source inspection shows current validation rejects `gateway.bind="lan"` with `gateway.tailscale.mode="serve"`, startup asserts the invalid snapshot, and the active migration registry has no matching repair; no live launchd run was performed.

## Solution Assessment

No. The migration direction is right, but this branch targets retired migration files and makes validation-specific handling depend on a legacy predicate that excludes explicit `auto`.
