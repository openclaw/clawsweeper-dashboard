Keep open: the underlying crash-loop bug is still valid and this PR is the canonical implementation path, but the current branch is not mergeable because it targets retired migration files, relaxes explicit `auto` validation, and lacks after-fix real behavior proof.

## What This Changes

The PR adds legacy config detection, migration logic, helper reuse, and tests to normalize explicit non-loopback Tailscale Serve/Funnel gateway binds to `loopback`.

## Best Possible Solution

Port the normalization rule and legacy detection to `src/commands/doctor/shared/legacy-config-migrations.runtime.gateway.ts` before Control UI origin seeding, preserve current validation semantics including explicit `auto`, add focused regression coverage, and merge only after redacted real behavior proof is attached.

## Solution Assessment

No. The migration direction is right, but this branch needs to be rebased onto the active doctor/shared migration registry and must keep explicit `auto` under the loopback-only Serve/Funnel guard.
