## Summary

Keep open for maintainer review: the underlying WebSocket reasoning-stream bug is plausible, but this patch broadens reasoning visibility beyond the existing explicit stream and suppression gates and lacks real behavior proof.

## What This Changes

The PR changes embedded Pi reasoning-stream handling so WebSocket `thinking` events can emit without a channel callback and broadens the native, tagged, and final reasoning call sites.

## Best Possible Solution

Land a focused gateway reasoning fix that forwards stored `reasoningLevel`, decouples only the channel callback dependency for explicit stream visibility, preserves silent and deterministic-approval suppression, and adds targeted regression coverage plus real behavior proof.

## Reproduction Assessment

Yes. Static tracing is enough: current main persists gateway reasoning settings but callback-less WebSocket runs cannot pass the `onReasoningStream` gate before `emitAgentEvent({ stream: "thinking" })`.

## Solution Assessment

No. The PR fixes a real path too broadly; the safer solution is to remove only the callback dependency while keeping explicit reasoning visibility, thinking availability, silent-run, and deterministic-approval gates intact.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Keep thinking events behind reasoning visibility:** `src/agents/pi-embedded-subscribe.ts:636`
  - body: Removing the `state.streamReasoning` guard lets the broadened callers publish `stream: "thinking"` for ordinary non-silent runs where reasoning visibility is still off, including approval-suppressed turns. That exposes content the current visibility contract keeps hidden unless reasoning is explicitly `on` or `stream`.
  - confidence: 0.9
- **[P2] Guard tagged-thinking scans to explicit visibility:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:307-309`
  - body: `ctx.state.streamReasoning || !ctx.params.silentExpected` is true for nearly every visible non-reasoning run, so this repeatedly scans the growing full delta buffer on the streaming hot path even when no surface requested reasoning output.
  - confidence: 0.84
- **[P2] Extract final reasoning for callback-less streams:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:640`
  - body: This final call still depends on `rawThinking`, but `rawThinking` is only extracted when `includeReasoning || streamReasoning` is true. In callback-less gateway `reasoningMode: "stream"` runs, `streamReasoning` remains false, so providers that only surface thinking at `message_end` still produce no WebSocket thinking event.
  - confidence: 0.78
