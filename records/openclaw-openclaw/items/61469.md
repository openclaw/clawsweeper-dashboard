## Summary

Keep open. The callback gate reported by the PR is real on current main, but this patch removes too much reasoning visibility/suppression gating and still misses final-only callback-less streams, while related PRs track safer versions of the same gateway reasoning work.

## What This Changes

This PR changes agent subscription reasoning-stream handling so WebSocket `thinking` agent events can be emitted without requiring the channel `onReasoningStream` callback, and broadens native, tagged, and final reasoning call sites.

## Best Possible Solution

Land a narrow fix that makes reasoning streaming callback-optional only for explicit reasoning-stream visibility, preserves silent and deterministic-approval suppression, forwards gateway session `reasoningLevel`, adds focused regression tests, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. Static tracing is enough: on current main, `reasoningMode: "stream"` without `onReasoningStream` leaves `streamReasoning` false and cannot reach the `stream: "thinking"` event; with this PR, non-silent default/off or approval-suppressed runs with thinking content can reach that event path.

## Solution Assessment

No. Broadcasting unconditionally is broader than the current visibility contract; the safer solution is to decouple the callback dependency while keeping reasoning-mode, thinking-availability, and suppression gates intact.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Keep thinking events behind reasoning visibility:** `src/agents/pi-embedded-subscribe.ts:636`
  - body: Removing the `state.streamReasoning` guard means the broadened callers can publish `stream: "thinking"` events for ordinary non-silent runs where reasoning visibility is still off. Connected WebSocket clients would receive reasoning that the current docs and normal assistant paths keep hidden, including during deterministic-approval-suppressed turns.
  - confidence: 0.9
- **[P2] Guard tagged-thinking scans to explicit visibility:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:307-309`
  - body: `ctx.state.streamReasoning || !ctx.params.silentExpected` is true for nearly every visible non-reasoning run, so this runs full-buffer tagged-thinking extraction on every text delta. Because `deltaBuffer` grows across the turn, long normal replies can repeatedly rescan the full buffer on the streaming hot path.
  - confidence: 0.84
- **[P2] Extract final reasoning for callback-less streams:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:640`
  - body: This unconditional final call still depends on `rawThinking`, but `rawThinking` is only extracted when `includeReasoning || streamReasoning` is true. In the callback-less gateway stream case on current main, `streamReasoning` remains false, so providers that surface thinking only at `message_end` still produce no WS thinking event.
  - confidence: 0.78
