## Summary

Keep open. The PR targets a real callback-gating gap for WebSocket reasoning streams, but the patch broadens reasoning visibility beyond the current explicit on/stream contract and still leaves a final-only callback-less stream gap.

## What This Changes

The PR changes agent reasoning-stream handling so WebSocket `thinking` events can be emitted without requiring an `onReasoningStream` callback, and broadens native, tagged, and final reasoning call sites.

## Best Possible Solution

Land a narrow agent/gateway fix that makes explicit `reasoningMode: "stream"` callback-optional, preserves off/silent/deterministic-approval visibility gates, forwards gateway session `reasoningLevel` where needed, and adds focused regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. Static tracing is enough: current main blocks callback-less WS thinking through the `streamReasoning` callback requirement, while this PR’s broadened callers can emit thinking for non-stream visible runs and still miss final-only callback-less reasoning.

## Solution Assessment

No. The current patch is broader than the visibility contract; the safer solution is callback decoupling only inside explicit reasoning-stream mode, with suppression gates preserved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Keep thinking events behind reasoning visibility:** `src/agents/pi-embedded-subscribe.ts:636`
  - body: Removing the `state.streamReasoning` gate lets any broadened caller that passes thinking text publish `stream: "thinking"` for ordinary non-silent runs where reasoning mode is still off/default. That exposes reasoning the current prompt and assistant-output paths keep hidden unless reasoning is explicitly on or stream.
  - confidence: 0.9
- **[P2] Restore the explicit stream guard for tagged scans:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:307-309`
  - body: `ctx.state.streamReasoning || !ctx.params.silentExpected` is true for nearly every visible non-reasoning text delta, so this now runs full-buffer tagged-thinking extraction as `deltaBuffer` grows. Long normal replies can repeatedly rescan the whole buffer on the streaming hot path.
  - confidence: 0.84
- **[P2] Extract final reasoning for callback-less streams:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:640`
  - body: This unconditional final call still depends on `rawThinking`, but `rawThinking` is only populated when `includeReasoning || streamReasoning` is true. Since this PR leaves `streamReasoning` tied to `onReasoningStream`, callback-less gateway stream sessions still miss providers that only surface thinking at `message_end`.
  - confidence: 0.78
