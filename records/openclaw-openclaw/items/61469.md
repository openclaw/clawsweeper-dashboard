## What This Changes

The PR broadens embedded Pi reasoning-stream emission so WebSocket thinking events can broadcast without requiring the channel `onReasoningStream` callback.

## Best Possible Solution

## Reproduction Assessment

Yes. Source tracing is high-confidence: current main requires `onReasoningStream`, while gateway `chat.send` does not provide that callback, so callback-less WebSocket runs cannot emit live `thinking` events through the current path.

## Solution Assessment

No for this branch. The safer solution is the canonical PR because it removes only the callback dependency, preserves `canShowReasoning`, and adds focused no-callback plus `thinkingLevel: "off"` regression coverage with real behavior proof.
