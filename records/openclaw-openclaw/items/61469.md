## Summary

This PR should close as superseded because a newer canonical PR carries the same Gateway/WebSocket no-callback reasoning-stream fix with the narrower visibility gate, regression tests, and sufficient real behavior proof, while this branch broadens reasoning emission in ways that are unsafe and incomplete.

## What This Changes

The branch removes the `onReasoningStream` callback dependency from `emitReasoningStream` and broadens native, tagged, and final reasoning-stream call sites.

## Best Possible Solution

Keep https://github.com/openclaw/openclaw/pull/79687 as the canonical implementation, finish its docs/changelog cleanup, and close overlapping variants that do not add unique safe behavior.

## Reproduction Assessment

Yes. Source tracing is enough: current main requires `onReasoningStream` before emitting `stream: "thinking"`, while the Gateway `chat.send` path does not provide that callback.

## Solution Assessment

No for this PR. The safer solution is the canonical PR that removes only the callback dependency, preserves `canShowReasoning`, and adds no-callback plus `thinkingLevel: "off"` regression coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep thinking events behind reasoning visibility:** `src/agents/pi-embedded-subscribe.ts:636`
  - body: Removing the `state.streamReasoning` guard lets the broadened callers publish `stream: "thinking"` events for ordinary non-silent runs where `/reasoning` visibility is still off. Keep the WebSocket broadcast callback-optional, but still gated by explicit stream-mode visibility.
  - confidence: 0.9
- **[P2] Guard tagged-thinking scans to explicit stream mode:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:307-309`
  - body: `ctx.state.streamReasoning || !ctx.params.silentExpected` is true for almost every visible run, so normal text deltas repeatedly scan the growing full buffer and can surface `<think>` content outside stream mode. Restore a reasoning-visibility gate or use an incremental/tag-presence check.
  - confidence: 0.84
- **[P2] Extract final reasoning for callback-less stream runs:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:640`
  - body: The final `ctx.emitReasoningStream(rawThinking)` call still depends on `rawThinking`, but `rawThinking` is only extracted when `includeReasoning || streamReasoning` is true. In the callback-less Gateway stream case, `streamReasoning` remains false, so providers that only expose final thinking at `message_end` still emit nothing.
  - confidence: 0.78
