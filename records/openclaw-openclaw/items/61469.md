## Summary

## What This Changes

The PR changes embedded Pi reasoning streaming so WebSocket thinking events are emitted without requiring a channel-specific reasoning callback.

## Best Possible Solution

Keep https://github.com/openclaw/openclaw/pull/79687 as the canonical implementation, finish its remaining docs/changelog cleanup, and close duplicate reasoning-stream branches that add no unique safe behavior.

## Reproduction Assessment

Yes. Source tracing is high-confidence: current main requires `onReasoningStream`, while gateway `chat.send` does not provide that callback, so callback-less WebSocket runs cannot emit live `thinking` events through the current path.

## Solution Assessment

No for this branch. The safer solution is the canonical PR because it removes only the callback dependency, preserves `canShowReasoning` and silent-run gates, and adds focused no-callback regression coverage plus real behavior proof.

  - confidence: 0.84

In callback-less gateway `reasoningMode: "stream"` runs, `streamReasoning` remains false, so providers that only surface thinking at `message_end` still produce no WebSocket thinking event.
  - confidence: 0.78
