## Summary

Keep open. The PR is a narrow implementation for a valid embedded timeout/abort leak that current main still does not guard in payload assembly; the code change looks functionally correct, but it needs the required user-facing changelog entry before merge.

## What This Changes

The PR updates embedded run payload assembly to treat aborted assistant turns as error surfaces, suppress aborted answer/reasoning text, and add regression tests for partial timeout leakage.

## Best Possible Solution
