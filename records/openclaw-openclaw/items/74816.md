## Summary

Keep open. The PR targets a real embedded timeout leak, but the current diff suppresses every `stopReason: "aborted"` assistant payload, including user-initiated external aborts where existing failover policy expects partial assistant output to carry the turn; it also still needs the required changelog entry.

## What This Changes

The PR changes embedded run payload assembly to surface formatted errors for aborted assistant turns, suppress aborted answer/reasoning text, and add regression tests for timeout partial leaks.

## Best Possible Solution

Keep the shared payload-builder fix, but gate aborted-artifact suppression to timeout/internal error-surface cases rather than all aborted turns, then add the missing Unreleased changelog entry.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to call `buildEmbeddedRunPayloads` on current main with non-empty assistant text, `lastAssistant.stopReason: "aborted"`, a timeout-like `errorMessage`, and reasoning enabled; current main can emit the partial answer/reasoning instead of only a clean timeout error. The PR-introduced regression can be reproduced with an aborted assistant that has visible text but no error message, representing the existing external-abort contract.

## Solution Assessment

No. The shared payload-builder boundary is the right place for the timeout leak fix, but the current implementation needs an explicit timeout/internal-abort guard or caller flag so user-initiated external aborts still deliver their partial reply.
