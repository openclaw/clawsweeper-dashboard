## Summary

Keep open. Current main still routes only errored assistant turns through the formatted embedded error surface, while aborted turns remain eligible for reasoning and answer payload delivery; the PR is a narrow, appropriate fix, but it still needs the required user-facing changelog entry before merge.

## What This Changes

The PR updates embedded run payload assembly to treat aborted assistant turns as clean error surfaces, suppress aborted answer and reasoning text, and add regression tests for timeout/abort partial-content leakage.

## Best Possible Solution

Land this PR, or an equivalent guard in `buildEmbeddedRunPayloads`, after adding the Unreleased changelog entry; keep the fix at the shared embedded payload boundary instead of adding channel-specific filtering.
