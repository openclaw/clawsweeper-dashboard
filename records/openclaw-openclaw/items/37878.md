## Summary

Keep open. Current main and the latest shipped release still use raw numeric type checks for model-usage costs, and a focused current-main probe reproduces dropped numeric-string costs plus boolean/non-finite contamination; the linked implementation PR is closed unmerged and the open token-summary PR does not fix this bug.

## What This Changes

Request to update `model-usage` cost parsing so numeric-string costs count, booleans and non-finite values are ignored, and regression tests cover those cases.

## Best Possible Solution

Add a small shared cost-normalization helper that accepts finite numeric strings, rejects booleans and non-finite values, and route `aggregate_costs`, `pick_current_model`, and `latest_day_cost` through it with focused regression tests.

## Reproduction Assessment

Yes. A current-main `python3 -B` probe against `aggregate_costs`, `pick_current_model`, and `latest_day_cost` reproduces the dropped numeric string, accepted boolean, and propagated `NaN`/`Infinity` behavior.

## Solution Assessment

Yes. A shared finite `parse_cost`-style helper is the narrowest maintainable fix; it avoids a new config/product decision and should not broaden into the separate token-summary feature work.
