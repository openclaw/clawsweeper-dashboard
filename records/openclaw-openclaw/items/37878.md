## What This Changes

The issue requests that model-usage cost aggregation accept numeric-string costs while ignoring booleans and non-finite values.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/87861 after normal maintainer review, then let its closing reference resolve this issue.

## Reproduction Assessment

Yes. Current source shows aggregate_costs, pick_current_model, and latest_day_cost only accept native int/float costs, so numeric-string costs are skipped and NaN/Infinity floats can flow into totals.

## Solution Assessment

Yes. A shared finite-cost coercion helper used by all three cost-reading sites is the narrow maintainable repair, and the open linked PR is the active candidate for that path.
