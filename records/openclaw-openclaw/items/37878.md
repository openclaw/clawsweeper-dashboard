## Summary

Keep open. Current main and the latest release still use raw numeric type checks in the model-usage cost paths, and a focused current-main probe reproduces dropped numeric-string costs plus boolean and non-finite contamination; the linked implementation PR is closed unmerged and current open same-file PRs do not close this bug.

## What This Changes

The issue asks for model-usage cost handling to accept numeric-string cost values while ignoring booleans and non-finite numeric values.

## Best Possible Solution

Normalize cost values through one finite cost parser, reuse it in every model-usage cost consumer, and add focused regression coverage plus a changelog entry without taking on token-summary or test-harness changes beyond what the fix requires.

## Reproduction Assessment

Yes. A current-main Python -B probe against aggregate_costs, pick_current_model, and latest_day_cost reproduces the dropped numeric-string cost plus bool and non-finite contamination.

## Solution Assessment

Yes. A small parse_cost-style helper that accepts finite numeric strings and rejects bools, NaN, and Infinity is the narrowest maintainable fix; if adjacent PRs land first, the same helper should be applied to the renamed cost aggregation path.
