## Summary

Keep open. Current main and the latest release still use raw numeric checks in the model-usage cost paths, a current-main probe reproduces numeric-string costs being omitted plus boolean/non-finite values being accepted, and the direct fix PR is closed unmerged.

## What This Changes

Accept numeric-string model cost values in the bundled model-usage helper, ignore boolean and non-finite cost values, and add regression tests.

## Best Possible Solution

Add one shared finite cost-normalization helper and route aggregate_costs, pick_current_model, and latest_day_cost through it with focused regression tests and a changelog entry.

## Reproduction Assessment

Yes. A bytecode-disabled Python probe against current main reproduces the dropped numeric-string cost, accepted boolean cost, and propagated NaN/Infinity values in the affected helper paths.

## Solution Assessment

Yes. A shared finite cost parser is the narrowest maintainable fix because all three cost consumers need the same input-normalization contract without changing plugin APIs or product policy.
