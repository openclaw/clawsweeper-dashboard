## Summary

Keep open. Current main still drops numeric-string costs and lets bool, NaN, and Infinity flow through the model-usage cost helpers; the linked direct fix PR was closed unmerged, and the open token-summary PR touches the same script without owning this normalization.

## What This Changes

The issue requests accepting numeric-string model-usage costs while ignoring booleans and non-finite cost values.

## Best Possible Solution

Add one shared finite cost-normalization helper used by `aggregate_costs`, `pick_current_model`, and `latest_day_cost`, with focused regression tests for numeric strings, booleans, NaN, and Infinity.

## Reproduction Assessment

Yes. A bytecode-disabled Python probe against current main reproduces the dropped numeric-string cost, accepted boolean cost, and propagated NaN/Infinity values in the affected helper paths.

## Solution Assessment

Yes. A shared finite cost parser is the narrowest maintainable fix because all three cost consumers need the same normalization without changing the skill API or product policy.
