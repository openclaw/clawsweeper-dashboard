## Summary

Keep open. Current main and the latest release still use raw Python numeric checks for model-usage costs, so numeric-string costs are dropped while booleans and non-finite floats can affect totals; the directly linked implementation PR was closed unmerged, and the open token-summary PR leaves this cost-normalization work separate.

## What This Changes

The issue requests that the model-usage helper accept numeric-string cost values, ignore booleans and non-finite costs, and add regression tests.

## Best Possible Solution

Add a shared finite cost-normalization helper and route `aggregate_costs`, `pick_current_model`, and `latest_day_cost` through it with focused regression coverage for numeric strings, booleans, `NaN`, and `Infinity`.

## Reproduction Assessment

Yes. A current-main `python3 -B` probe against `aggregate_costs`, `pick_current_model`, and `latest_day_cost` reproduces the dropped numeric string, accepted boolean, and propagated `NaN`/`Infinity` behavior.

## Solution Assessment

Yes. A shared `parse_cost`-style helper is the narrowest maintainable fix, and the work should stay separate from the token-summary feature path.
