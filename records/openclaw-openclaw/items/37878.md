## Summary

Keep open: current main and latest release still drop numeric-string costs and allow boolean/non-finite values to corrupt model-usage summaries, while https://github.com/openclaw/openclaw/pull/87861 is the open closing PR that owns the fix.

## What This Changes

Make `skills/model-usage/scripts/model_usage.py` accept finite numeric-string costs and ignore booleans, NaN, and Infinity across aggregation, current-model selection, and latest-day reporting.

## Best Possible Solution

Land a focused fix like https://github.com/openclaw/openclaw/pull/87861 that centralizes finite-cost coercion for all three cost readers and adds regression coverage, then close this issue.

## Reproduction Assessment

Yes. A current-main Python probe against `skills/model-usage/scripts/model_usage.py` reproduces numeric-string cost loss, NaN poisoning, and wrong current-model selection without writing to the checkout.

## Solution Assessment

Yes. A single finite-cost coercion helper reused by `aggregate_costs`, `pick_current_model`, and `latest_day_cost` is the narrow maintainable fix shape for this bug.
