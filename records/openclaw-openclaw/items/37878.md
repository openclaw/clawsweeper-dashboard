Current main still has the reported parser gap, and the only linked implementation, #37877, is closed unmerged. The issue is valid, narrow, and suitable for a focused repair PR rather than cleanup closure.

## What This Changes

The issue requests that the model-usage helper accept finite numeric-string costs while ignoring booleans and non-finite numeric values in aggregation, model selection, and latest-day reporting.

## Best Possible Solution

Add one shared finite-cost parser in the model-usage script, use it consistently in the three cost-reading paths, and cover numeric strings, booleans, NaN, and Infinity with focused tests plus a changelog fix entry crediting @shuofengzhang.
