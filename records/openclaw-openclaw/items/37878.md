## Summary

Keep open. Current main and the latest release still drop numeric-string costs and allow booleans or non-finite floats through model-usage cost paths; the directly linked fix PR is closed unmerged, and the open token-summary PR does not supersede this cost-normalization bug.

## What This Changes

The issue asks the model-usage helper to parse finite numeric-string costs and ignore booleans, NaN, and Infinity values.

## Best Possible Solution

Add one shared finite cost-normalization helper and route aggregate_costs, pick_current_model, and latest_day_cost through it, with focused tests for numeric strings, booleans, NaN, and Infinity.

## Solution Assessment

Yes. A shared parse_cost-style helper is the narrowest maintainable fix because all three affected paths need the same finite-cost contract without changing plugin APIs or product policy.
