## Summary

Keep open. Current main and v2026.6.1 still use raw numeric type checks in all three model-usage cost paths, so numeric-string costs are still dropped and bool/non-finite values can still affect totals; the linked fix PR is open and should remain the implementation path until it merges or is closed.

## What This Changes

The issue requests that the model-usage Python helper accept finite numeric-string cost values while ignoring booleans, NaN, and Infinity across aggregation, model selection, and latest-day cost reporting.

## Best Possible Solution

Land the linked narrow fix PR or an equivalent patch that centralizes finite-cost coercion across all three model-usage cost readers and regression-tests numeric strings, booleans, and non-finite values.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence repro path with modelBreakdowns[].cost values like "1.75", true, or NaN; I did not execute the script because this cleanup review is read-only.

## Solution Assessment

Yes. A single finite-cost coercion helper reused by aggregate_costs, pick_current_model, and latest_day_cost is the narrow maintainable solution; the issue should stay open until that lands through the linked PR or an equivalent fix.
