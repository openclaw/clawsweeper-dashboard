## Summary

Keep open. Current main and the latest shipped release still use raw Python numeric checks for model-usage costs, so numeric-string costs are dropped while booleans and non-finite floats can contaminate totals; the linked implementation PR was closed unmerged.

## What This Changes

The issue asks the model-usage helper to accept finite numeric-string costs, ignore booleans and non-finite values, and add focused regression tests.

## Best Possible Solution

Add one shared finite cost-normalization helper and route `aggregate_costs`, `pick_current_model`, and `latest_day_cost` through it with regression coverage for numeric strings, booleans, `NaN`, and `Infinity`.

## Reproduction Assessment

Yes. A current-main `python3 -B` probe against `aggregate_costs`, `pick_current_model`, and `latest_day_cost` reproduces the dropped numeric string, accepted boolean, and propagated `NaN`/`Infinity` behavior.

## Solution Assessment

Yes. A shared `parse_cost`-style helper is the narrowest maintainable fix; it should not broaden into the separate token-summary feature work.
