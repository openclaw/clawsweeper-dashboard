## Summary

Keep open. Current main and the latest release still gate model-usage cost fields with raw numeric type checks, so numeric strings are omitted while booleans and non-finite floats can flow into summaries; the linked direct fix PR is closed unmerged, and the open token-usage PR is adjacent rather than canonical.

## What This Changes

The issue requests that the model-usage helper accept finite numeric-string cost values while ignoring booleans, NaN, and Infinity in cost aggregation and selection paths.

## Best Possible Solution

Add a shared finite cost-normalization helper and route aggregate_costs, pick_current_model, and latest_day_cost through it with focused regression tests and a small changelog fix entry.

## Reproduction Assessment

Yes. A bytecode-disabled Python probe against current main reproduces numeric-string costs being dropped, boolean costs being counted, and NaN/Infinity propagating through the affected helper paths.

## Solution Assessment

Yes. A shared finite cost parser is the narrow maintainable fix because all current cost consumers need the same input-normalization contract without changing plugin APIs or product policy.
