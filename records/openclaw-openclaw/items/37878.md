## What This Changes

The issue asks for model-usage cost parsing to accept numeric strings while ignoring booleans and non-finite values.

## Best Possible Solution

Keep this issue open while https://github.com/openclaw/openclaw/pull/87861 is reviewed and landed; after it merges, close this issue as implemented by that PR.

## Reproduction Assessment

Yes. A read-only current-main helper call against a cost payload with 1.50, "1.75", true, and NaN reproduces the bad aggregate and latest-day behavior in the implicated functions.

## Solution Assessment

Yes. The narrow maintainable fix is to centralize finite cost coercion and use it in aggregate_costs, pick_current_model, and latest_day_cost; the open linked PR is the current candidate for that implementation.
