## What This Changes

The issue requests that the model-usage helper accept finite numeric-string cost values while ignoring booleans and non-finite values such as NaN and Infinity.

## Best Possible Solution

Add one shared finite-cost parser in the model-usage script, use it in aggregate_costs, pick_current_model, and latest_day_cost, and cover numeric strings, booleans, NaN, and Infinity in the colocated Python tests with a changelog fix entry crediting the reporter.
