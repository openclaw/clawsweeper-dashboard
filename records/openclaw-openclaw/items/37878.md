## Summary

This issue is still necessary because current main and the latest release still use native int/float checks that drop numeric-string costs and allow bool/NaN values to affect model-usage summaries, but it already has an open closing fix candidate in https://github.com/openclaw/openclaw/pull/87861.

## What This Changes

The issue asks for the model-usage helper to count numeric-string costs, ignore booleans and non-finite values, and add regression coverage.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/87861, or an equivalent centralized finite-cost coercion fix with the same three-reader regression coverage, then close this issue through the merged fix.

## Reproduction Assessment

Yes. A read-only current-main probe against the helper functions reproduces numeric-string undercounting, bool miscounting, and missing numeric-string latest-day cost handling, and the source shows the shared native int/float checks that cause it.

## Solution Assessment

Yes. A single finite-cost coercion helper used by aggregate_costs, pick_current_model, and latest_day_cost is the narrowest maintainable fix; the open linked PR follows that shape and adds focused tests.
