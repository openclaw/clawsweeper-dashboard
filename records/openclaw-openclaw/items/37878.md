## Summary

Keep open. Current main and the latest release still use raw Python numeric type checks in the model-usage cost paths, a focused current-main probe reproduces dropped numeric-string costs plus boolean/NaN/Infinity leakage, and the linked implementation PR was closed unmerged while the open token-summary PR does not supersede this normalization bug.

## What This Changes

Normalize `model-usage` cost handling so numeric-string costs are accepted while booleans and non-finite values are ignored across aggregation, current-model selection, and latest-day cost lookup.

## Best Possible Solution

Add one shared finite cost-normalization helper and route `aggregate_costs`, `pick_current_model`, and `latest_day_cost` through it with focused regression tests.

## Reproduction Assessment

Yes. A bytecode-disabled Python probe against current main reproduces the dropped numeric string, accepted boolean, and propagated NaN/Infinity behavior in the affected helper paths.

## Solution Assessment

Yes. A shared finite cost parser is the narrowest maintainable fix because all three cost consumers need the same input-normalization contract without changing plugin APIs or product policy.
