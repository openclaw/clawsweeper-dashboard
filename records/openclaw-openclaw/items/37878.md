## Summary

Keep open. Current main and the latest release still drop numeric-string costs and allow bool, NaN, and Infinity through model-usage cost paths; the direct fix at https://github.com/openclaw/openclaw/pull/37877 closed unmerged, and the related token-summary PR at https://github.com/openclaw/openclaw/pull/78247 does not fully own this normalization.

## What This Changes

The issue asks the model-usage helper to accept numeric-string cost values while ignoring booleans and non-finite numeric values.

## Best Possible Solution

Add one shared finite cost-normalization helper and route aggregate_costs, pick_current_model, and latest_day_cost through it with focused regression tests.

## Reproduction Assessment

Yes. A bytecode-disabled Python probe against current main reproduces the dropped numeric-string cost, accepted boolean cost, and propagated NaN/Infinity values in the affected helper paths.

## Solution Assessment

Yes. A shared finite cost parser is the narrowest maintainable fix because all three cost consumers need the same normalization contract without changing plugin APIs or product policy.
