## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/37878. Current main in `skills/model-usage/scripts/model_usage.py` only accepts native `int`/`float` costs in `aggregate_costs`, `pick_current_model`, and `latest_day_cost`, so finite numeric strings are ignored while booleans and `NaN`/`Infinity` can leak into totals or current-model selection. Add a small shared parser/helper that accepts finite numeric values and finite numeric strings, rejects booleans, non-numeric strings, and non-finite values, and use it in all three paths while preserving the existing `modelsUsed` fallback. Add focused regression tests in `skills/model-usage/scripts/test_model_usage.py` for numeric-string aggregation, skipping bool/non-finite values, current-model selection from finite parsed costs, and latest-day cost for numeric strings. Treat closed unmerged PR https://github.com/openclaw/openclaw/pull/37877 as useful source context, but do not broaden the change to CodexBar invocation, skill metadata, or unrelated docs. Add a `CHANGELOG.md` fix entry if maintainers consider this user-facing skill behavior.
