## Summary

Keep open. Current main and v2026.5.19 still drop numeric-string costs while allowing bool, NaN, and Infinity through the model-usage cost helpers; the direct fix PR at https://github.com/openclaw/openclaw/pull/37877 was closed unmerged, and the related token-summary PR at https://github.com/openclaw/openclaw/pull/78247 does not own this normalization.

## What This Changes

Accept numeric-string costs and reject booleans, NaN, and Infinity in model-usage aggregation, current-model selection, and latest-day cost reporting.

## Best Possible Solution

Add one shared finite cost-normalization helper and route aggregate_costs, pick_current_model, and latest_day_cost through it with focused regression tests.

## Reproduction Assessment

Yes. A bytecode-disabled Python probe against current main reproduces the dropped numeric-string cost, accepted boolean cost, and propagated NaN/Infinity values in the affected helper paths.

## Solution Assessment

Yes. A shared finite cost parser is the narrowest maintainable fix because all three cost consumers need the same input-normalization contract without changing plugin APIs or product policy.
