## Summary

Keep open. Current main still drops numeric-string costs while accepting booleans and propagating NaN/Infinity; the linked implementation PR is closed unmerged, and the open token-summary PR does not supersede this cost-normalization bug.

## What This Changes

The issue requests that model-usage parse finite numeric-string cost values and ignore booleans, NaN, and Infinity.

## Best Possible Solution

Add one shared finite cost-normalization helper and route aggregate_costs, pick_current_model, and latest_day_cost through it with focused regression tests.

## Reproduction Assessment

Yes. A bytecode-disabled Python probe against current main reproduces the dropped numeric string, accepted boolean, and propagated NaN/Infinity behavior in the affected helper paths.

## Solution Assessment

Yes. A shared parse_cost-style helper is the narrowest maintainable solution because all three cost paths need the same finite-cost contract without changing plugin APIs or product policy.
