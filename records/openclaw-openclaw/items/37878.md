## Summary

Keep open. Current main and the latest release still use raw int/float checks for model costs, so numeric-string costs are skipped while booleans and non-finite values can flow into summaries; the linked implementation PR is closed unmerged and the open token-summary PR does not supersede this cost-normalization bug.

## What This Changes

The issue requests that the model-usage helper accept numeric string cost values while ignoring booleans and non-finite numeric values.

## Best Possible Solution

Add one shared finite cost-normalization helper and route `aggregate_costs`, `pick_current_model`, and `latest_day_cost` through it with focused regression tests.

## Reproduction Assessment

Yes. A bytecode-disabled Python probe against current main reproduces the dropped numeric string, accepted boolean, and propagated NaN/Infinity behavior in the affected helper paths.

## Solution Assessment

Yes. A shared finite cost parser is the narrowest maintainable fix because all three cost consumers need the same input-normalization contract without changing plugin APIs or product policy.
