## Summary

Keep open. Current main still drops finite numeric-string costs and lets boolean or non-finite native values contaminate totals/current-model selection; the linked implementation PR was closed unmerged.

## What This Changes

The issue requests that `model-usage` normalize cost fields so finite numeric strings count while booleans, `NaN`, and `Infinity` are skipped.

## Best Possible Solution

Add a shared cost-normalization helper, use it in aggregation/current/latest-day paths, and cover numeric strings, booleans, `NaN`, and `Infinity` with focused tests plus a user-facing changelog entry.

## Solution Assessment
