## Summary

Keep open. Current main still uses raw numeric type checks in the model-usage cost paths and a current-main probe reproduces dropped numeric-string costs plus boolean/non-finite contamination; the linked implementation PR is closed unmerged and no open canonical fix references this issue.

## What This Changes

The issue requests model-usage cost parsing to accept numeric-string costs and ignore booleans plus non-finite values across aggregation, current-model selection, and latest-day output.

## Best Possible Solution

Normalize cost values once, reject invalid values, and route every model-usage cost consumer through that helper with focused regression tests and a changelog entry.

## Reproduction Assessment

Yes. On current main, a `PYTHONDONTWRITEBYTECODE=1 python3 -B` probe against `aggregate_costs`, `pick_current_model`, and `latest_day_cost` reproduces the dropped numeric string and bool/non-finite contamination.

## Solution Assessment

Yes. A small finite `parse_cost`-style helper reused by `aggregate_costs`, `pick_current_model`, and `latest_day_cost` is the narrowest maintainable fix and does not require config or product policy changes.
