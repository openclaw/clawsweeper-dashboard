## Summary

Keep open. Current main still drops numeric-string model costs and accepts bool/NaN/Infinity through the model-usage cost paths; the linked implementation PR was closed unmerged, and the open token-summary PR does not supersede this normalization bug.

## What This Changes

The issue requests model-usage cost parsing to accept numeric-string costs, ignore booleans and non-finite values, and add regression tests.

## Best Possible Solution

Add one shared finite cost-normalization helper and route `aggregate_costs`, `pick_current_model`, and `latest_day_cost` through it with focused regression tests.

## Reproduction Assessment

Yes. A bytecode-disabled Python probe against current main reproduces the dropped numeric string, accepted boolean, and propagated NaN/Infinity behavior in the affected helper paths.

## Solution Assessment

Yes. A shared finite cost parser is the narrowest maintainable fix because all three cost consumers need the same input-normalization contract without changing plugin APIs or product policy.
