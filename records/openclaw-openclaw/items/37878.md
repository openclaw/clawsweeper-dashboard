## What This Changes

The issue asks the model-usage helper to count numeric-string cost values and ignore booleans plus non-finite numeric values during cost aggregation and model selection.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Yes. A shared finite-cost coercion helper used by `aggregate_costs`, `pick_current_model`, and `latest_day_cost` is the narrowest maintainable fix; patching only one reader would leave sibling behavior drift.
