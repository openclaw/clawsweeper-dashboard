## Summary

Keep open. Current main and latest release still use raw native-number checks in all three model-usage cost paths, so numeric-string costs are still skipped while bool and non-finite values can still affect totals; the open closing PR remains the canonical implementation path.

## What This Changes

The issue asks model-usage to parse finite numeric-string cost values and ignore booleans, NaN, and Infinity in all cost summary paths.

## Best Possible Solution

Review and either land or reject the open closing PR; the desired end state is one shared finite-cost coercion helper used by all three cost readers with focused regression tests.

## Reproduction Assessment

Yes. Source inspection shows `modelBreakdowns[].cost` values of `"1.75"`, `true`, or `NaN` reach native-number checks in all three cost paths on current main; I did not execute a runtime repro in this read-only review.

## Solution Assessment

Yes. A single finite-cost coercion helper reused by `aggregate_costs`, `pick_current_model`, and `latest_day_cost` is the narrow maintainable fix; the issue should stay open until the linked PR lands or is rejected.
