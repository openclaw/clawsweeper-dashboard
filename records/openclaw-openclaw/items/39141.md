## Summary

Keep open. Current main and the latest shipped release still lack the requested opt-in gateway watchdog alert engine, and the prerequisite public `sessions.activity` RPC/WebSocket contract is still open, so this remains a real feature/design item rather than cleanup to close.

## What This Changes

Request an opt-in gateway-side watchdog that consumes future session activity state and emits deduped/rate-limited operator alerts for stale or blocked sessions, with optional tokenized summaries.

## Best Possible Solution

Land the `sessions.activity` contract first, then add a disabled-by-default gateway watchdog with privacy-safe structured alerts, configurable thresholds/cooldowns/scoping, delivery sinks, docs, focused tests, and strictly opt-in token summaries.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken existing behavior report. Source and live issue inspection give a high-confidence current-main gap: there is no public `sessions.activity` RPC/event contract or opt-in watchdog alert engine, only adjacent session-list and diagnostic signals.

## Solution Assessment

Unclear as a complete implementation until the activity API, alert sinks, privacy defaults, cooldown semantics, and token-summary policy are settled. The dependency-first direction is maintainable, but this should not be treated as a narrow automatic bug fix.
