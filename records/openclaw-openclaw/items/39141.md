## Summary

Keep open. Current main has adjacent session list status, liveness diagnostics, and telemetry, but it still lacks the public `sessions.activity.*` contract and the requested opt-in Gateway watchdog alert engine; the prerequisite activity-contract issue at https://github.com/openclaw/openclaw/issues/39127 remains open.

## What This Changes

Request an opt-in Gateway watchdog that consumes session activity state, detects stale or blocked sessions, and emits deduped operator alerts with optional tokenized summaries.

## Best Possible Solution

Settle and document the per-session activity API first, then add a disabled-by-default Gateway watchdog with privacy-safe structured alerts, configurable thresholds/cooldowns, tests, and explicitly opt-in tokenized summaries.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken existing behavior report. Source and docs inspection provide a high-confidence verification path: current main lacks the requested activity RPC/events and only has adjacent diagnostics/session-list signals.

## Solution Assessment

Unclear as a complete design until the Gateway activity contract, alert outputs, privacy defaults, and token-summary policy are settled. The maintainable path is to resolve https://github.com/openclaw/openclaw/issues/39127 first and build this watchdog against that public surface.
