## Summary

Keep open. Current main has adjacent internal liveness diagnostics and channel-health checks, but it still lacks the prerequisite public `sessions.activity` RPC/WS contract and the requested opt-in watchdog alert engine.

## What This Changes

Request an optional gateway-side session activity watchdog that consumes `sessions.activity` state and emits rate-limited operator alerts, with tokenized summaries only as an opt-in.

## Best Possible Solution

Sequence this behind #39127, then design a disabled-by-default gateway watchdog on the public activity contract with privacy-safe structured alerts, cooldowns, docs, and protocol/runtime tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main shows adjacent diagnostics only, with no public `sessions.activity` RPC/event contract or opt-in watchdog alert surface to reproduce against.

## Solution Assessment

Unclear as a final product shape, but the sequencing is right: first land #39127 or an equivalent activity contract, then build the watchdog on that public surface instead of private diagnostics.
