## Summary

Keep open. Current main has adjacent liveness diagnostics and channel-health checks, but it still lacks the public `sessions.activity.get/list` plus activity-change event contract and the requested opt-in watchdog alert engine.

## What This Changes

The issue requests an optional gateway-side session activity watchdog with configurable stale/waiting/child-session alerts built on the separate sessions.activity API.

## Best Possible Solution

Sequence this behind #39127, then design a disabled-by-default gateway watchdog that consumes the public activity contract and emits privacy-safe structured alerts with rate limits, docs, and protocol/runtime tests.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection on current main shows adjacent diagnostics only, with no public `sessions.activity` RPC/event contract or opt-in watchdog alert surface.

## Solution Assessment

Unclear whether every proposed output belongs in core exactly as written. The maintainable path is to land #39127 or an equivalent activity contract first, then build generic privacy-safe watchdog alerts on that surface instead of private diagnostics.
