## Summary

Current main and v2026.6.5 provide narrower stale-session telemetry, but not the requested session activity/attention contract, thresholded alert engine, delivery outputs, dedupe/cooldowns, or optional summaries. The prerequisite remains open, and the feature needs an explicit product decision because OpenClaw’s plugin-first direction favors external alert policy once a generic activity seam exists.

## What This Changes

Add an opt-in gateway watchdog that consumes per-session activity state to detect stale or blocked sessions and emit rate-limited operator alerts, with optional model-generated summaries.

## Best Possible Solution

Approve a minimal, generic session activity and attention contract in the canonical prerequisite, then implement threshold evaluation, dedupe/cooldown state, and operator delivery as an external code plugin unless maintainers explicitly adopt session monitoring as a core gateway product.

## Reproduction Assessment

Not applicable: this issue requests a new watchdog and alerting capability rather than reporting broken behavior under an existing documented contract.

## Solution Assessment

No, the proposed all-in-one gateway implementation is not yet the narrowest maintainable solution. The generic activity contract should be decided first, with optional alert policy and delivery kept plugin-owned unless maintainers deliberately choose a core product surface.
