## Summary

Keep open. Current main still has only coarse session status plus adjacent stuck-session diagnostics/recovery; it does not provide the prerequisite session activity RPC/event contract or the requested opt-in operator alert watchdog.

## What This Changes

The issue requests an opt-in gateway-side session activity watchdog with configurable token-free alerts, optional summaries, and delivery targets, built on a future sessions activity API.

## Best Possible Solution

Land the shared session activity API first, then add a gateway-owned opt-in watchdog that consumes that state with privacy-safe alerts, dedupe/cooldowns, docs, and focused tests.

## Solution Assessment
