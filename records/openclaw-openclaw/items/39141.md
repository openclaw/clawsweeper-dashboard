## Summary

Keep open. Current main has adjacent internal diagnostics, channel health checks, and coarse session status, but the prerequisite public `sessions.activity` RPC/WS contract and the requested opt-in watchdog/alert delivery layer are still missing.

## What This Changes

The issue requests an optional gateway-side session activity watchdog that consumes activity state and emits deduped, privacy-safe operator alerts with configurable thresholds.

## Best Possible Solution

Land the shared session activity contract in https://github.com/openclaw/openclaw/issues/39127 first, then add a gateway-owned opt-in watchdog that consumes it with privacy-safe alerts, configurable thresholds, dedupe/cooldowns, docs, and focused tests.

## Solution Assessment

Yes directionally, but not as an automated cleanup fix. The maintainable path is to settle https://github.com/openclaw/openclaw/issues/39127 first and layer this watchdog on that contract instead of adding polling, log parsing, or diagnostic-only alert heuristics.
