## Summary

Keep open. Current main has adjacent session diagnostics and cron failure alerts, but it still lacks the prerequisite public `sessions.activity` RPC/WS contract and the requested opt-in operator watchdog for stale or blocked session alerts.

## What This Changes

The issue requests an opt-in gateway-side session activity watchdog that consumes a future `sessions.activity` API and emits alerts for stale, waiting, or stuck child-session states.

## Best Possible Solution

Land the shared `sessions.activity` API first, then add a gateway-owned opt-in watchdog that consumes that state with privacy-safe alerts, dedupe/cooldowns, docs, and focused tests.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken documented behavior. Source inspection gives a high-confidence current-main gap check for the missing activity RPC/event contract and alert watchdog.

## Solution Assessment

Yes directionally, but not as an automated cleanup fix. The maintainable path is to settle https://github.com/openclaw/openclaw/issues/39127 first and then layer this watchdog on that contract instead of adding separate polling, log parsing, or diagnostic-only alert heuristics.
