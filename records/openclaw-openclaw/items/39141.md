## Summary

Keep open: current main and v2026.6.10 still lack the requested `sessions.activity.*` RPC/statechange contract and the opt-in gateway watchdog/alerting layer. The prerequisite activity-state design remains open at https://github.com/openclaw/openclaw/issues/39127, so this is still a maintainer product/API decision rather than conservative cleanup.

## What This Changes

The issue requests an optional gateway-side session activity watchdog that consumes `sessions.activity` state and emits deduped operator alerts, with optional tokenized summaries and configurable thresholds.

## Best Possible Solution

First settle the canonical gateway session activity/attention contract, then build a default-off watchdog on top of that state with safe alert payloads, dedupe/cooldowns, focused docs, and tests for no-credential baseline alerts plus optional summaries.

## Reproduction Assessment

Not applicable as a bug reproduction: this requests a new opt-in gateway watchdog and alerting capability. Source and release inspection show the prerequisite activity API and requested alert layer are absent on current main and v2026.6.10.

## Solution Assessment

Unclear until maintainers settle the product/API boundary. The maintainable direction is to build on https://github.com/openclaw/openclaw/issues/39127 rather than inventing independent polling, diagnostic-only state, or alert-specific session truth.
