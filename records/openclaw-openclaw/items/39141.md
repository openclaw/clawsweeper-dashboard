## Summary

Keep open: current main and v2026.6.10 still lack the requested default-off session activity watchdog and alerting layer, and the prerequisite activity-state contract remains open at https://github.com/openclaw/openclaw/issues/39127. Existing diagnostic liveness telemetry and open PRs are useful adjacent work, but they do not replace operator-facing deduped alerts, thresholds, delivery targets, webhooks, or optional summaries.

## What This Changes

Request to add a configurable gateway-side watchdog that consumes future session activity state and emits operator alerts for stale, waiting, or stuck child-session conditions.

## Best Possible Solution

Settle the canonical session activity/attention API first, then design a default-off watchdog that emits payload-minimized alerts with dedupe, cooldowns, docs, and tests for no-credential baseline behavior plus gated optional summaries.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new opt-in gateway watchdog and alerting capability. Source and release inspection are enough to show the requested activity API dependency and alerting layer are absent.

## Solution Assessment

Unclear until maintainers settle the product/API boundary. The maintainable direction is to build on https://github.com/openclaw/openclaw/issues/39127 rather than inventing independent polling, diagnostic-only state, or alert-specific session truth.
