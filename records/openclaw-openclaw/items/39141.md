## Summary

Keep open: current main and v2026.6.5 still lack the prerequisite public per-session activity/attention contract and the requested alerting layer. Existing channel-health, diagnostic, plugin-hook, and subagent-recovery mechanisms are useful adjacent work, but they have different scopes and do not provide queryable session state, prolonged-wait alerts, deduplicated operator delivery, or optional summaries. The issue remains necessary, but its built-in gateway scope needs maintainer product and ownership review because OpenClaw policy favors a lean core and plugin-owned optional capability.

## What This Changes

Add an opt-in, event-driven session activity watchdog that detects stale runs, prolonged waiting, and child-session problems and emits rate-limited operator alerts with optional model summaries.

## Best Possible Solution

Land one additive, Gateway-owned session activity and attention contract with authorization, versioning, and reconnect semantics, expose a generic plugin observation API, and implement opt-in alert classification and delivery as an operator-monitoring plugin unless maintainers explicitly designate built-in monitoring as a core product.

## Reproduction Assessment

Not applicable: this requests a new built-in monitoring and alerting capability, and neither current documentation nor protocol schemas define it as existing expected behavior.

## Solution Assessment

No, not as one monolithic gateway subsystem. The narrowest maintainable solution separates the canonical core session-state contract from optional alert policy, transport delivery, webhooks, and model-generated summaries.
