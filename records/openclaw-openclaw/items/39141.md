## Summary

Keep open. Current main has coarse session status, merged UI live badges, and internal liveness diagnostics, but it still lacks the requested `sessions.activity.get/list` and statechange contract plus an opt-in operator alert watchdog; the prerequisite https://github.com/openclaw/openclaw/issues/39127 remains open.

## What This Changes

Add an opt-in gateway-side watchdog that consumes future session activity state to alert on stale runs, long waiting states, child-session stalls, operator delivery targets, webhooks, and optional model summaries.

## Best Possible Solution

Land the shared session-activity contract from https://github.com/openclaw/openclaw/issues/39127 first, then add a gateway-owned opt-in watchdog that reuses that state with token-free baseline alerts, dedupe/cooldowns, privacy-safe delivery, docs, and focused tests.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken documented behavior report. Source inspection gives a high-confidence current-main check that the requested activity RPC/event contract and watchdog alert engine are absent.

## Solution Assessment

Yes in direction, but not as an automated cleanup fix. The maintainable path is to settle the activity contract first and then layer the watchdog on that contract instead of adding separate polling, log parsing, or one-off alert heuristics.
