## Summary

Keep open. Current main has adjacent private liveness diagnostics and channel-health checks, but it still lacks the blocked public session activity RPC/event contract and the requested opt-in watchdog alert engine.

## What This Changes

Request an opt-in gateway-side watchdog that consumes session activity state to alert operators on stale, waiting, or stuck sessions with configurable thresholds, dedupe, delivery, and optional summaries.

## Best Possible Solution

Sequence this behind #39127, then design a disabled-by-default gateway watchdog that uses the public activity contract with privacy-safe structured alerts, rate limits, docs, and protocol/runtime tests.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection on current main shows adjacent diagnostics only, with no public sessions.activity RPC/event contract or opt-in watchdog alert surface.

## Solution Assessment

Unclear whether every proposed output belongs in core exactly as written. The maintainable direction is to first land #39127 or an equivalent activity contract, then build the watchdog on that public surface rather than private diagnostics.
