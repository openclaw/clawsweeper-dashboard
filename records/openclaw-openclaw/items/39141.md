## Summary

Keep open: current main and v2026.6.6 still lack the requested `sessions.activity`-backed watchdog, alert outputs, cooldown/dedupe policy, configurable alert classes, and optional model summaries. The prerequisite activity API remains open, so this is still a product/API design item rather than conservative cleanup.

## What This Changes

The issue requests an optional gateway-side session activity watchdog that consumes future session-activity state, emits deduped operator alerts, and optionally generates tokenized summaries.

## Best Possible Solution

Define and land the canonical session activity/attention contract first, then add a small opt-in alert observer that consumes that state without owning parallel session semantics.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this is a request for new opt-in gateway monitoring and alerting behavior. Source inspection gives high confidence that current main lacks the requested `sessions.activity`-backed watchdog and alert outputs.

## Solution Assessment

Unclear until the prerequisite activity contract is accepted. The gateway-side observer direction is plausible, but the best maintainable solution should reuse canonical session activity state rather than invent parallel polling, timers, or attention-state ownership.
