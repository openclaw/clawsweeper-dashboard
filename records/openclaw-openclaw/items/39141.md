Keep open. Current main still lacks the prerequisite `sessions.activity` RPC/WS contract and the requested opt-in gateway watchdog/alerting layer; adjacent diagnostics and channel-health checks are not a public per-session operator alert feature.

Add an optional gateway-side session activity watchdog with configurable stale/waiting/child-session alerts, privacy-safe outputs, and optional tokenized summaries after the activity API exists.

## Best Possible Solution

Land the shared activity contract first, then add a gateway-owned opt-in watchdog that consumes it with privacy-safe alerts, configurable thresholds, dedupe/cooldowns, docs, and focused tests.
