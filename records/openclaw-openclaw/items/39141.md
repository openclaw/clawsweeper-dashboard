## Summary

Keep open. Current main still lacks the prerequisite session-scoped activity RPC and WebSocket state-change contract, and the existing stuck-session diagnostics, channel/account activity fields, subagent announce checks, and Control UI attention items are adjacent pieces rather than the requested opt-in watchdog with configurable alert classes, dedupe/cooldowns, safe operator delivery, and optional gated summaries.

## What This Changes

Request an opt-in gateway-side session activity watchdog that consumes future `sessions.activity` state, alerts on stale or blocked sessions and child trees with configurable no-token outputs, and optionally produces gated model-generated summaries.

## Best Possible Solution

Keep this open until #39127 or an equivalent per-session activity/attention API lands, then add an opt-in gateway watchdog that consumes that evented contract, emits no-content/no-token structured alerts with configurable thresholds and cooldowns, exposes safe UI/log/webhook/plugin delivery, gates optional summaries behind explicit config and rate limits, and documents/tests the protocol and runtime behavior.
