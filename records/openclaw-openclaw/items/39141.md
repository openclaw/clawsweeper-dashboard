## Summary

Keep open: current main and v2026.6.6 still lack the prerequisite `sessions.activity.*` contract and the requested opt-in watchdog/alerting layer, while the related activity-state design remains open for product/API review.

## What This Changes

Requested an optional gateway-side session activity watchdog that consumes canonical session activity state and emits deduped operator alerts for stale runs, long waits, and stuck child sessions.

## Best Possible Solution

First land or explicitly decide the canonical session activity/attention API, then build a small opt-in watchdog on top of that state with no content leakage, dedupe/cooldowns, focused docs, and tests for credentials-free baseline alerts plus optional summaries.

## Reproduction Assessment

Not applicable as a bug reproduction: this requests a new opt-in gateway watchdog and alerting capability. Source inspection does confirm the prerequisite activity API and requested alert layer are absent on current main and v2026.6.6.

## Solution Assessment

Unclear until maintainers settle the product/API boundary. The right direction is to build on the canonical session activity contract at https://github.com/openclaw/openclaw/issues/39127 rather than inventing independent polling or diagnostic-only state.
