## Summary

Keep open. Current main and the latest shipped release still provide only adjacent session rows, diagnostic liveness classification, and recovery telemetry; the requested opt-in gateway watchdog alert engine depends on the still-open session activity contract and would add new public protocol, configuration, alert delivery, privacy, and optional model-summary policy.

## What This Changes

Add an optional gateway-side session activity watchdog that consumes a future sessions.activity API and emits deduped, rate-limited operator alerts for stale or blocked sessions.

## Best Possible Solution

Define and land the sessions.activity contract first, then add a disabled-by-default gateway watchdog with privacy-safe structured alerts, configurable thresholds/cooldowns/scoping, delivery sinks, docs, focused tests, and strictly opt-in token summaries.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken existing behavior report. Source and live issue inspection give a high-confidence current-main gap: there is no public sessions.activity RPC/event contract or opt-in watchdog alert engine, only adjacent session-list and diagnostics signals.

## Solution Assessment

Unclear as a complete design until the activity API, alert sinks, privacy defaults, cooldown semantics, and token-summary policy are settled. The dependency-first direction is maintainable, but this should not be treated as a narrow automatic bug fix.
