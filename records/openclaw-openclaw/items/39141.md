## Summary

Keep open. Current main has adjacent session liveness diagnostics, but it still lacks the public `sessions.activity.get/list` plus activity state-change WS contract and the requested opt-in watchdog alert engine with configurable alert classes, dedupe/cooldowns, operator delivery, and optional summaries.

## What This Changes

Request an optional gateway-side session activity watchdog that consumes per-session activity state and emits privacy-safe operator alerts.

## Best Possible Solution

Sequence this behind #39127, then add a disabled-by-default gateway watchdog that consumes the public activity state and emits structured no-token alerts with configurable thresholds, dedupe/cooldowns, privacy-safe delivery seams, optional rate-limited summaries, docs, and protocol/runtime tests.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection on current main show adjacent diagnostics only, with no public session activity RPC/event contract or opt-in watchdog alert surface.

## Solution Assessment

Unclear whether every proposed output should land exactly as written, but the maintainable direction is to wait for #39127 or an equivalent gateway-owned activity contract and build the watchdog on top of that. Current diagnostics and channel health checks are useful inputs, not a complete substitute.
