## Summary

Keep open. Current main has adjacent session list status, lifecycle events, and diagnostics, but it still lacks the prerequisite public `sessions.activity.*` contract and the requested opt-in gateway watchdog alert engine.

## What This Changes

The issue requests an opt-in gateway-side session activity watchdog that consumes a future `sessions.activity` API and emits deduped operator alerts, with optional tokenized summaries.

## Best Possible Solution

Land or settle the public session activity contract first, then design a disabled-by-default Gateway watchdog with structured privacy-safe alerts, documented configuration, dedupe/cooldowns, and focused protocol/runtime tests.
