## Summary

Keep open. Current main has diagnostic stuck-session telemetry/recovery and a coarse `hasActiveRun` session-list flag, but it does not provide the requested `sessions.activity` API or an opt-in event-driven operator alert watchdog; the prerequisite https://github.com/openclaw/openclaw/issues/39127 is still open.

## What This Changes

Request an optional gateway-side watchdog that consumes per-session activity state, detects stale or blocked sessions, and emits configurable operator alerts with optional tokenized summaries.

## Best Possible Solution

Define and land the session activity contract first, then add a gateway-owned opt-in alert engine that reuses that state, keeps baseline alerts token-free, and documents config, privacy, dedupe, and validation behavior.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives a high-confidence current-main check: the requested `sessions.activity` methods and activity statechange event are not present, so this remains unimplemented feature work.

## Solution Assessment

Not applicable as a patch. The direction is reasonable, but the maintainable path is to settle https://github.com/openclaw/openclaw/issues/39127 first and then layer alerting on that contract rather than building a separate polling/watchdog path.
