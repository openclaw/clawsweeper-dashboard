## Summary

Keep open: current main and the latest release still lack the prerequisite public `sessions.activity` RPC/WS contract and the requested opt-in watchdog/alert layer; existing diagnostics and channel-health logic are adjacent but internal, channel-scoped, or policy-specific rather than the user-facing session activity alerting requested here.

## What This Changes

The issue requests an opt-in Gateway session activity watchdog that consumes a future `sessions.activity` contract and emits deduped, rate-limited operator alerts with optional tokenized summaries.

## Best Possible Solution

Land the shared session activity contract first, then design an opt-in Gateway watchdog around that contract with privacy-safe outputs, dedupe/cooldowns, configurable thresholds, docs, and tests.
