## Summary

Keep open: current main and the latest release have internal diagnostic stuck-session telemetry/recovery and a coarse `sessions.changed` active-run projection, but they still do not provide the requested `sessions.activity.*` contract or opt-in gateway watchdog/alerting layer. The prerequisite activity-state issue remains open, so this needs maintainer product/API review rather than cleanup closure.

## What This Changes

Add an opt-in gateway-side session activity watchdog that consumes `sessions.activity` state and emits deduped, rate-limited operator alerts with optional model-generated summaries.

## Best Possible Solution

First land or explicitly decide the canonical session activity/attention API, then build a small opt-in watchdog on top of that state with no content leakage, dedupe/cooldowns, focused docs, and tests for credentials-free baseline alerts plus optional summaries.

## Reproduction Assessment

Not applicable as a bug reproduction: this requests a new opt-in gateway watchdog and alerting capability. Source inspection confirms the prerequisite activity API and requested alert layer are absent on current main and `v2026.6.8`.

## Solution Assessment

Unclear until maintainers settle the product/API boundary. The right direction is to build on the canonical session activity contract at https://github.com/openclaw/openclaw/issues/39127 rather than inventing independent polling or diagnostic-only state.
