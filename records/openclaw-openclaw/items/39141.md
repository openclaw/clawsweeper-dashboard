## Summary

Keep open. Current main has adjacent session liveness diagnostics and diagnostics exporter plugins, but it still lacks the requested public `sessions.activity.get/list` contract, activity state-change event family, and opt-in gateway watchdog/alert delivery engine; the blocking #39127 activity API is also still open.

## What This Changes

Request an opt-in gateway-side session activity watchdog with configurable stale/waiting/child-session alerts, no-token baseline delivery, and optional tokenized summaries.

## Best Possible Solution

Sequence this behind #39127, then design a disabled-by-default watchdog that consumes the public activity state and emits privacy-safe, deduped no-token alerts with optional rate-limited summaries if maintainers want that in core.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection on current main show adjacent diagnostics only, with no public session activity RPC/event contract or opt-in watchdog alert surface.

## Solution Assessment

Unclear whether every requested output mode belongs in core. The narrow maintainable path is to first land #39127 or an equivalent activity contract, then make a maintainer decision on which alert delivery pieces are core versus plugin-owned.
