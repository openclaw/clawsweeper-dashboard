## Summary

Keep open. Current main and the latest release still expose only adjacent session-list, liveness diagnostic, and telemetry signals; the requested opt-in Gateway watchdog alert engine depends on the still-open sessions.activity API work and would add new protocol, config, alert-delivery, privacy, and optional model-summary policy.

## What This Changes

The issue requests an opt-in Gateway session activity watchdog that emits stale-run, long-waiting, child-session, webhook/operator-channel, and optional token-summary alerts built on a future sessions.activity API.

## Best Possible Solution

Define and document the sessions.activity contract first, then add a disabled-by-default Gateway watchdog with privacy-safe structured alerts, configurable thresholds/cooldowns/scoping, delivery sinks, focused tests, and opt-in token summaries.
