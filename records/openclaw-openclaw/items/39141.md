## Summary

Keep open: current main and v2026.6.6 still lack the prerequisite `sessions.activity.*` contract and the requested opt-in alerting/watchdog layer, while related live issues show this is still an active session-state product area rather than stale cleanup.

## What This Changes

Request an opt-in gateway-side session activity watchdog that consumes future session activity state and emits deduped, rate-limited operator alerts, with optional tokenized summaries.

## Best Possible Solution

First land or decide the canonical session activity/attention API, then design a small opt-in alerting layer on top of that source with no content leakage, explicit cooldowns, focused docs, and tests for resync, dedupe, credentials-free baseline alerts, and optional model summaries.

## Reproduction Assessment

Not applicable as a bug reproduction: this requests a new opt-in gateway watchdog and alerting capability. Source inspection does confirm the prerequisite activity API and requested alert layer are not present on current main or v2026.6.6.

## Solution Assessment

Unclear until maintainers settle the product/API boundary. The right direction is to build on the canonical session activity contract rather than inventing independent polling or diagnostic-only state, but the alert delivery and model-summary surfaces need explicit scope decisions.
