## Summary

Keep open: this is the canonical PR for the linked Feishu 231001 reaction failure, but the latest diff is not merge-ready because several Feishu emoji_type values do not match the documented API vocabulary and the channel import crosses the existing lazy runtime boundary.

## What This Changes

The PR adds Feishu reaction emoji normalization, applies it in the generic Feishu react path and reaction helpers, and adds focused tests.

## Best Possible Solution

Keep the Feishu-owned normalization layer, but make it use exact documented emoji_type values from Feishu and place the pure normalizer in a lightweight helper that does not import the Feishu client/runtime modules.

## Reproduction Assessment

Yes, at source level. The linked issue gives a concrete message(action="react", emoji="👍") path, current main passes that value directly into Feishu reaction_type.emoji_type, and Feishu documents 231001 for invalid reaction types.

## Solution Assessment

No, not yet. Normalizing at the Feishu reaction API boundary is the right direction, but the current table must use exact documented values and stay out of the eager channel runtime path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve Feishu's exact emoji_type values:** `extensions/feishu/src/reactions.ts:133-150`
  - body: Feishu's emoji table is case-sensitive and lists values such as `LAUGH`, `ThumbsDown`, `CheckMark`, `CrossMark`, and `Fire`, but this PR invents uppercase/similar values like `LAUGHING`, `THUMBSDOWN`, `CHECK`, `CROSS`, and `FIRE`. Because the normalizer uppercases matching local entries, valid inputs such as `Fire` and `ThumbsDown`, plus Unicode aliases like 🔥/✅/❌/😂, can still be sent as invalid `emoji_type` values and fail with 231001.
  - confidence: 0.93
- **[P2] Keep normalization behind the lazy runtime boundary:** `extensions/feishu/src/channel.ts:73`
  - body: Importing `normalizeFeishuEmoji` from `reactions.ts` at channel module load pulls in `reactions.ts` and transitively `client.ts`, whose module load mutates the Lark SDK HTTP interceptor. Move the pure emoji map into a lightweight helper or keep the call behind `loadFeishuChannelRuntime()` so loading the Feishu channel does not eagerly evaluate the client runtime.
  - confidence: 0.86
