## Summary

Keep this PR open: the reported Feishu reaction bug is real and this is still the canonical fix path, but the current head has merge-blocking compatibility and lazy-runtime-boundary issues.

## What This Changes

The PR adds Feishu emoji normalization for reaction add/list paths, wires it into the generic Feishu react action, and adds Feishu reaction helper tests.

## Best Possible Solution

Use a Feishu-local lightweight emoji normalizer with exact official `emoji_type` values, preserve already-valid Feishu strings, and keep the helper free of client/runtime imports.

## Reproduction Assessment

Yes, at source level: current main trims raw emoji and serializes it unchanged into Feishu `reaction_type.emoji_type`, while the linked report shows Feishu returning code 231001 for that path. I did not run a live Feishu HTTP roundtrip in this read-only review.

## Solution Assessment

No. Normalizing at the Feishu reaction API boundary is the right direction, but this implementation should use Feishu's exact documented values and avoid importing reaction/client runtime modules from `channel.ts`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Preserve Feishu's exact emoji_type strings:** `extensions/feishu/src/reactions.ts:134-150`
  - body: This table invents all-caps names for values Feishu documents as mixed-case or different names: `ThumbsDown`, `Fire`, `CheckMark`, `CrossMark`, and `LAUGH` are not the same as `THUMBSDOWN`, `FIRE`, `CHECK`, `CROSS`, and `LAUGHING`.   - confidence: 0.96
- **[P2] Keep normalization out of the reaction runtime import:** `extensions/feishu/src/channel.ts:73`
  - body: This top-level import pulls in `reactions.ts`, which imports `client.ts`; `client.ts` mutates the Lark SDK request interceptor at module evaluation. `channel.ts` already uses `loadFeishuChannelRuntime()` for these helpers, so move the pure emoji map into a lightweight helper imported by both paths instead of forcing reaction client side effects during plugin load.
  - confidence: 0.91
