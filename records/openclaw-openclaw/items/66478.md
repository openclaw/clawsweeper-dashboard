## What This Changes

The PR adds a Feishu emoji_type normalizer, wires it into Feishu reaction add/list/channel paths, and adds regression tests for Unicode, documented casing, trimming, and unknown passthrough behavior.

## Best Possible Solution

Land the Feishu-owned normalizer once maintainers accept request-body proof, or ask for one live Feishu add/remove reaction roundtrip before merge if transport-level evidence is required.

## Solution Assessment

Yes. Normalizing inside the Feishu plugin at the reaction API boundary is the narrowest maintainable fix; the separate pure map also preserves the channel module's lazy runtime boundary.
