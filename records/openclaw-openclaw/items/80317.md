## Summary

Keep open: current `main` still gates OpenAI TTS voice-note compatibility to Opus only, while Telegram delivery and shared media compatibility already accept MP3; the linked open fix PR should be reviewed and landed or closed before this issue is resolved.

## What This Changes

Treat OpenAI-compatible TTS `responseFormat: "mp3"` output as voice-note compatible for Telegram so it can route through `sendVoice` instead of the audio-file fallback.

## Best Possible Solution

Land or close https://github.com/openclaw/openclaw/pull/80326 after it has adequate real behavior proof, or apply the same narrow OpenAI provider compatibility fix with focused regression coverage.

## Reproduction Assessment

Yes by source inspection, though not by a live Telegram run here. Current `main` resolves Telegram TTS as `voice-note`, OpenAI MP3 returns `voiceCompatible: false`, speech core leaves `audioAsVoice` unset, and Telegram then falls back to `sendAudio`.

## Solution Assessment

Yes. The narrow provider-gate change proposed by the linked PR matches the existing Telegram/media contract while preserving the Opus default and avoiding a broader conversion or config redesign.
