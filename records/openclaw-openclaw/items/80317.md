## Summary

Keep open: current main still gates OpenAI TTS voice-note compatibility to Opus only, and https://github.com/openclaw/openclaw/pull/80326 is an open closing PR rather than a merged fix.

## What This Changes

OpenAI-compatible TTS configured for MP3 should be treated as Telegram voice-note-compatible so speech-core routes it through Telegram sendVoice.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80326 after redacted real Telegram/local TTS proof, or replace it with the same narrow provider/test fix if that PR is abandoned.

## Reproduction Assessment

Yes. Source inspection on current main shows configured MP3 is preserved by the OpenAI provider, but voiceCompatible is false unless the format is Opus; speech-core then leaves Telegram voice delivery disabled.

## Solution Assessment

Yes. The narrow provider compatibility change in the linked PR is the maintainable fix for this bug; broader transcoding can remain a separate product decision if needed later.
