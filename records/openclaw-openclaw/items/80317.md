## Summary

Keep open: current main and the latest stable tag still gate OpenAI TTS voice compatibility to Opus only, while Telegram's sendVoice path and OpenClaw's shared media compatibility already accept MP3. This is a narrow provider/delivery mismatch rather than a duplicate of the closed generic Telegram voice-message work.

## What This Changes

Request OpenAI TTS configured with `responseFormat: "mp3"` to be treated as Telegram voice-compatible so TTS replies use `sendVoice` instead of `sendAudio`.

## Best Possible Solution

Align OpenAI TTS MP3 compatibility with Telegram's accepted voice-message formats while preserving cross-channel format safety, then add focused provider and delivery regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Telegram resolves TTS as `voice-note`, OpenAI with configured MP3 returns `voiceCompatible: false`, speech core leaves `audioAsVoice` false, and Telegram then takes the `sendAudio` fallback.

## Solution Assessment

Yes, the requested direction is likely the narrowest maintainable fix, but it should be implemented in source with tests rather than by patching generated output. The repair should preserve existing Opus defaults and only broaden compatibility where the channel/runtime contract supports it.

## Repair Work Prompt

Fix the OpenAI TTS MP3 voice-compatibility mismatch for Telegram. Current main returns `voiceCompatible: false` for OpenAI `responseFormat: "mp3"` even when the TTS target is `voice-note`, so Telegram falls back to `sendAudio` despite Telegram and OpenClaw shared media compatibility accepting MP3 for voice messages. Keep the existing Opus default for voice-note targets and do not add a new config option. Prefer a narrow source change in `extensions/openai/speech-provider.ts` with regression coverage in `extensions/openai/speech-provider.test.ts`; if the safe fix needs channel-aware format gating, keep it scoped to the existing TTS/channel compatibility contract and add the smallest necessary tests in speech-core/Telegram. Add a user-facing changelog entry because this affects Telegram TTS delivery. Do not edit generated dist files.
