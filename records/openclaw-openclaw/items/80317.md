## Summary

Keep open: current main still gates OpenAI TTS voice compatibility to Opus only, while Telegram delivery and the shared media contract already accept MP3 for voice sends. The linked narrow fix PR was closed unmerged, so the issue remains actionable.

## What This Changes

Allow OpenAI-compatible TTS MP3 output to be treated as Telegram voice-note-compatible instead of falling back to audio-file delivery.

## Best Possible Solution

Align the OpenAI provider's MP3 voice-note compatibility with the existing shared media and Telegram contracts, with focused provider and delivery regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Telegram requests voice-note synthesis, OpenAI with configured MP3 returns voiceCompatible false, speech-core leaves audioAsVoice false, and Telegram falls back to sendAudio.

## Solution Assessment

Yes. The narrow maintainable fix is to broaden the OpenAI provider's voice-note-compatible response formats to include MP3 while keeping unsupported formats such as WAV false and adding regression tests.
