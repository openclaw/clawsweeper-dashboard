## Summary

Keep open: current main and v2026.5.7 still gate OpenAI-compatible TTS voice compatibility to Opus only, while the matching open PR https://github.com/openclaw/openclaw/pull/80326 uses closing syntax and should be reviewed or landed before this issue is closed.

## What This Changes

Treat MP3 output from the OpenAI-compatible TTS provider as Telegram voice-compatible so it can route through `sendVoice` instead of the audio-file fallback.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80326 once real Telegram/local TTS behavior proof is supplied, then close this issue as implemented.

## Reproduction Assessment

Yes. Source inspection on current main traces OpenAI `responseFormat: "mp3"` plus `target: "voice-note"` to `voiceCompatible: false`, and speech core then leaves `audioAsVoice` false so Telegram takes the audio-file path.

## Solution Assessment

Yes. The narrow provider compatibility change in the linked PR is the maintainable fix because Telegram and OpenClaw's shared media policy already accept MP3 voice media, but this issue should remain open until that PR lands.
