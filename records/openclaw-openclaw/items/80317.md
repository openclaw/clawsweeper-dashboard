## Summary

Keep open: current main and v2026.6.8 still mark configured OpenAI-compatible MP3 voice-note output as non-voice-compatible, and the linked closing PR is open rather than merged.

## What This Changes

OpenAI-compatible TTS configured with `responseFormat: "mp3"` should be marked voice-compatible for Telegram voice-note delivery so it routes through `sendVoice` like Microsoft/Edge TTS.

## Best Possible Solution

Resolve and land the linked PR, or a narrow replacement on current main, that reuses shared media voice compatibility for OpenAI MP3 output and keeps provider, speech-core, and Telegram routing coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows configured OpenAI MP3 voice-note synthesis returns `voiceCompatible: false`, while speech-core and Telegram already use MP3 voice delivery when that flag is true.

## Solution Assessment

Yes. The best fix is the linked PR's provider-boundary repair that reuses shared media compatibility rather than adding another OpenAI-local allowlist or changing Telegram transport behavior.
