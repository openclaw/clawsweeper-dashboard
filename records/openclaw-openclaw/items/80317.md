## Summary

Keep open: current main still makes OpenAI TTS MP3 output non-voice-compatible for Telegram, and an open closing PR is already paired with this issue. The issue should stay open until that PR is reviewed and either landed or closed.

## What This Changes

OpenAI-compatible TTS with `responseFormat: "mp3"` should be treated as Telegram voice-note compatible so delivery uses `sendVoice` instead of `sendAudio`.

## Best Possible Solution

Review and land or revise https://github.com/openclaw/openclaw/pull/80326 after redacted real Telegram/local TTS proof, keeping the fix limited to OpenAI MP3 voice-note compatibility with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main traces OpenAI `responseFormat: "mp3"` with `target: "voice-note"` to `voiceCompatible: false`, which keeps `audioAsVoice` false and sends Telegram audio through the fallback path.

## Solution Assessment

Yes. The narrow maintainable fix is to broaden the OpenAI provider's voice-note compatibility to MP3 where the Telegram/media contract already accepts it, with tests; the open linked PR is the current implementation path.
