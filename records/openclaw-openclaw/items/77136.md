## Summary

Keep open. Current main still has bounded WebChat history and render windows that can make older assistant replies disappear from the browser view, and the exact Safari/transcript case has not been provided enough to close as fixed or non-reproducible.

## What This Changes

WebChat should render assistant replies from long transcripts consistently with TUI instead of silently losing visible messages after history reload or session switching.

## Best Possible Solution

Define WebChat's long-history contract and implement it visibly, preferably with pagination/load-older behavior or server-side truncation metadata so bounded history never looks like lost assistant replies.

## Reproduction Assessment

Partially. Source inspection provides a high-confidence bounded-history path: WebChat reloads with `limit: 200` and the gateway returns only recent transcript messages, but the reporter's exact transcript/Safari case was not provided.

## Solution Assessment

Unclear. Raising a fixed limit alone is not the best solution; the maintainable path is to expose truncation/has-more state or add a load-older flow while preserving payload caps.
