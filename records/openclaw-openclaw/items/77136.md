## Summary

Keep open: current main and the latest release still have the source-proven WebChat bounded-history gap, while the likely pagination fix path is an open maintainer-labeled PR that has not merged yet. The issue discussion is useful: Jim Dawdy’s comment narrows the render-window UX bug, and Sheila Mayo’s logs point to adjacent compaction/restart symptoms that should remain separate from this bounded-history path.

## What This Changes

The issue asks WebChat to show long-session assistant history from intact transcripts without silently making older assistant replies appear lost.

## Best Possible Solution

Review and land or revise https://github.com/openclaw/openclaw/pull/87111, or make an explicit maintainer decision to ship a narrower truncation notice; keep this issue open until that path merges.

## Reproduction Assessment

Yes, as a source-level reproduction: WebChat requests only 100 history rows, the renderer can only count hidden rows it received, and current `chat.history` has no older-history cursor or has-more signal. I did not run a browser repro in this read-only review.

## Solution Assessment

Unclear as a final product path. Additive cursor pagination is the most complete maintainable fix, but maintainers still need to choose it over a narrower truncation notice.
