## Summary

Keep open. Current main still has bounded WebChat history and rendering windows that can make older transcript-backed assistant replies vanish from the browser view, and the exact Safari/transcript case is not available to close as fixed or non-reproducible.

## What This Changes

The issue asks WebChat to render transcript-backed assistant messages consistently with TUI instead of dropping older or previously visible replies in long dashboard sessions.

## Best Possible Solution

Define WebChat's long-history contract and implement either pagination/load-older behavior or explicit truncation metadata and visible UI state so bounded history never looks like lost assistant replies.

## Reproduction Assessment

Partially. Source inspection gives a high-confidence bounded-history path with WebChat loading only 200 recent messages, but the exact Safari transcript case was not provided or live-reproduced.

## Solution Assessment

Unclear. Raising a fixed limit alone is not the best solution; the maintainable path is an additive history UX/API contract such as has-more metadata, pagination, or a clear truncation notice while preserving payload caps.
