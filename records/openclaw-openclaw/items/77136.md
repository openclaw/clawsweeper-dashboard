## Summary

Keep open: current main and the latest release still have the source-proven WebChat bounded-history gap, and the canonical pagination PR is open but conflicting and maintainer-labeled, so the work is not implemented or safely superseded yet.

## What This Changes

The issue asks for WebChat to keep assistant replies visible from intact transcript history in long sessions, matching TUI behavior instead of silently omitting older replies.

## Best Possible Solution

Resolve and review the open WebChat/Gateway pagination PR, or make an explicit maintainer decision to land a narrower truncation notice instead; keep this issue open until that path merges.

## Reproduction Assessment

Yes. Current main source shows WebChat requesting and rendering a bounded 100-message history window without a server truncation/pagination signal, so older transcript rows can look missing even though data is intact.

## Solution Assessment

Unclear as a final product path. Additive bounded pagination is the most complete maintainable fix, but maintainers still need to choose it over a narrower older-history notice.
