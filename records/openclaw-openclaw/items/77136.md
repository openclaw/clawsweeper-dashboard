## Summary

Keep open. Current main still has fixed WebChat, gateway, and renderer history windows that can make older assistant replies disappear after reload or session switches, and the exact Safari/transcript case has not been attached to prove this is only the known bounded-history behavior.

## What This Changes

The issue requests WebChat long-session rendering that continues to show assistant replies from intact transcripts instead of dropping older messages after reloads or session switches.

## Best Possible Solution

Define WebChat's long-history contract and implement visible pagination, load-older, or truncation metadata instead of silently replacing the browser view with a capped tail.

## Reproduction Assessment

Partially. Source inspection gives a clear bounded-history path with 201+ visible transcript messages, but there is no attached transcript to reproduce the exact Safari/DeepSeek rendering failure.

## Solution Assessment

Unclear. Raising the fixed limit alone is not the best fix; the maintainable direction is explicit pagination or visible truncation state while preserving bounded payloads.
