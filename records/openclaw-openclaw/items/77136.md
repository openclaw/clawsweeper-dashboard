## Summary

Keep open. Current main still has WebChat history and render windows that can make older persisted assistant replies vanish from the browser view, and the related closed satellite report points at the same long-history/truncation area rather than proving this issue is fixed.

## What This Changes

The issue requests WebChat to render persisted assistant messages from long transcripts consistently with TUI instead of dropping them from the browser view.

## Best Possible Solution

Define WebChat's long-history contract and add visible pagination, load-older, or truncation metadata so bounded history never looks like lost persisted assistant replies.

## Reproduction Assessment

Yes, source-reproducible. A transcript with more than 200 display-visible entries will reload through WebChat's `chat.history` request capped at 200 and replace local browser state with that bounded result; I did not run a live Safari transcript reproduction in this read-only review.

## Solution Assessment

Unclear. A fixed limit bump is not the best maintainable solution; the safer direction is explicit truncation/has-more state or a load-older flow while preserving payload caps.
