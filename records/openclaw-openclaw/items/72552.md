## What This Changes

The PR changes the Control UI Dreaming Advanced tab and view tests so the Daily Log Review summary and staged list use `dailyCount`-backed entries instead of `groundedCount`-backed entries.

## Best Possible Solution

Align the Daily Log Review summary, staged entries, section-local action, and regression coverage around one `dailyCount`-backed meaning, then verify the real Control UI against a store with daily-backed entries.

## Reproduction Assessment

Yes. Source inspection on current main shows a `dailyCount > 0` and `groundedCount === 0` entry is excluded from the Daily Log Review summary and staged section because those surfaces filter on `groundedCount > 0`.

## Solution Assessment

No, not as-is. Switching the summary and staged section to `dailyCount` is the right narrow direction, but the PR must also fix or remove the grounded-only clear action for that section and provide real behavior proof.
