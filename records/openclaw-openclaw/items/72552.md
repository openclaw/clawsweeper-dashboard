## Summary

## What This Changes

The PR updates Control UI Dreaming Advanced rendering and tests so Daily Log Review counts and listing use `dailyCount` instead of `groundedCount`.

## Best Possible Solution

Revise the PR so the Daily Log Review summary, staged entries, origin labels, controls, and regression coverage share one `dailyCount`-backed meaning, then verify the real Advanced tab against a store with daily-backed entries before landing.

## Reproduction Assessment

Yes, by source inspection. Current main filters the Daily Log Review summary and staged section on `groundedCount > 0`, so a `dailyCount > 0` / `groundedCount === 0` entry is excluded without needing a live store to see the code path.

## Solution Assessment

No, not as-is. Switching the summary and staged section to `dailyCount` is the narrow direction, but the section-local clear control still targets grounded-only cleanup and will not clear the rows the section now displays.
