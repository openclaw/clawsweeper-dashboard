## Summary

Keep open: current main still lacks one shared validation and diagnostic contract for prompt-bearing Markdown across bootstrap context, runtime SKILL.md loading, native ClawHub installs, and publish-side policy; the open scanner PR is only a partial, currently blocked implementation path.

## What This Changes

Request a Markdown validation or linting layer for files that can become model-visible prompt context, especially skill definitions and ClawHub-published skills.

## Best Possible Solution

Define a threat-model-backed shared trust and diagnostic contract for model-visible Markdown, then apply it consistently to bootstrap context, runtime skill loading, Skill Workshop, ClawHub install/update, and publish-side verdicts while keeping broad workspace-write blocking out unless maintainers explicitly choose it.

## Reproduction Assessment

Yes, at source level: current bootstrap and local skill paths can be traced and do not share the requested Markdown-content validation policy. This is not a strict bug reproduction because current docs and tests do not already promise such a validator.

## Solution Assessment

No, not as proposed wholesale. A centralized validation and diagnostics contract is the maintainable direction, while blanket HTML-comment rejection, automatic wrapping, nested Git tracking, and general workspace-write interception need separate maintainer decisions.
