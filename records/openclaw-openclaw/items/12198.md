## Summary

Keep open. Current main still has path, size, symlink, archive-root, and code-safety checks, but it does not implement a shared prompt-bearing Markdown validation contract before workspace bootstrap files, extra bootstrap files, SKILL.md reads, or native ClawHub skill installs can become model-visible.

## What This Changes

Request Markdown linting/validation for prompt-bearing files, especially skill definitions and workspace prompt files, to detect prompt-injection and security patterns such as HTML comments.

## Best Possible Solution

Define one reusable prompt-bearing Markdown validation contract with explicit severities, override/quarantine behavior, docs, and tests, then apply it consistently at bootstrap ingestion, extra bootstrap loading, SKILL.md discovery/read paths, native ClawHub install/update, and ClawHub publish/tagging surfaces.

## Reproduction Assessment

Yes. A source-level reproduction is to place an HTML comment or prompt-injection marker in AGENTS.md, an extra bootstrap file, a local SKILL.md, or a ClawHub-installed skill; current main reads or installs that Markdown without a shared validator before it can become model-visible.

## Solution Assessment

Unclear. The requested infrastructure is valid, but the HTML-comment example is under-scoped; the maintainable fix is a shared policy with explicit severities, warnings/quarantine, overrides, and tests.
