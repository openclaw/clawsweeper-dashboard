## Summary

Keep open: current main still has partial hardening, but no shared prompt-bearing Markdown validation contract across workspace bootstrap files, SKILL.md loading, native ClawHub skill install/update, and publish-side policy. The related open scanner PR is useful but unmerged, blocked, and narrower than the requested cross-surface contract.

## What This Changes

Request to validate Markdown files that become model-visible prompt content, especially skills and bootstrap files, for prompt-injection and security-risk patterns.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract and apply it consistently to bootstrap ingestion, extra bootstrap files, SKILL.md discovery/read paths, native ClawHub install/update, and ClawHub publish scanning or tagging with explicit severity, override behavior, docs, and tests.

## Reproduction Assessment

Yes, at source level. A Markdown marker such as an HTML comment or prompt-injection phrase in a recognized bootstrap file or SKILL.md body can still follow raw-content paths into Project Context or model-readable skill content without a shared Markdown validator.

## Solution Assessment

Unclear as a final implementation plan. A shared validator is the likely maintainable seam, but maintainers still need to decide blocking severity, customization or override behavior, ClawHub install/update behavior, and publish-side policy; https://github.com/openclaw/openclaw/pull/43469 is a useful but incomplete scanner-only path.
