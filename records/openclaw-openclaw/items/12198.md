## Summary

Keep open: current main still lacks a shared prompt-bearing Markdown validation contract across bootstrap/context files, model-visible SKILL.md reads, and native ClawHub skill installs. The open scanner PR at https://github.com/openclaw/openclaw/pull/43469 is useful related work, but it only covers one slice and does not settle the broader security/product contract already called out in prior discussion.

## What This Changes

The issue requests validation for prompt-bearing Markdown files, including bootstrap files and SKILL.md content, to reduce prompt-injection risk.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract and apply it consistently to bootstrap/context ingestion, local skill reads, native ClawHub install/update, and ClawHub publish scanning or tagging with documented severity and override behavior.

## Reproduction Assessment

Yes, at source level. A marker such as an HTML comment or prompt-injection phrase in a recognized bootstrap file or SKILL.md body still follows current raw-content paths into Project Context or model-readable skill content without a shared Markdown validator.

## Solution Assessment

Unclear as an implementation plan. A shared validator is the likely maintainable seam, but a scanner-only patch is not sufficient unless maintainers also settle severity, override/customization, native ClawHub, and publish-side behavior.
