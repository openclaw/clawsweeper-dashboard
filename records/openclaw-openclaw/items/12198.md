## Summary

Keep open. Current main still lacks a shared prompt-bearing Markdown validation contract; bootstrap/context files, model-readable `SKILL.md` content, and native ClawHub skill installs still rely on raw-content paths or code-oriented/optional scanners, and the recent keep-open comments remain accurate.

## What This Changes

Request a shared validator for Markdown files that become prompt-visible, including workspace bootstrap files, skills, and ClawHub publishing or install paths.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract and apply it consistently to bootstrap/context ingestion, local skill reads, native ClawHub install/update, and ClawHub publish/tagging surfaces.

## Reproduction Assessment

Yes, at source level. A marker such as an HTML comment in a recognized bootstrap file or `SKILL.md` still follows raw-content paths into prompt-visible context or model-readable skill content without a shared Markdown validator.

## Solution Assessment

Unclear. Validation infrastructure is the right direction, but maintainers need to decide the shared ruleset, severity model, override/quarantine behavior, and ClawHub scope before implementation.
