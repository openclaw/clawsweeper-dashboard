## Summary

Keep open. Current main still lacks a shared prompt-bearing Markdown validation contract across bootstrap/context files, model-readable SKILL.md content, and native ClawHub skill installs, and the recent keep-open discussion remains accurate.

## What This Changes

Request validation infrastructure for Markdown files that become prompt-visible, including bootstrap files, SKILL.md, and ClawHub skill publishing or install paths.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract and apply it consistently to bootstrap/context ingestion, local skill reads, native ClawHub install/update, and ClawHub publish or tagging surfaces.

## Reproduction Assessment

Yes, at source level. A marker such as an HTML comment in a recognized bootstrap file or SKILL.md still follows raw-content paths into prompt-visible context or model-readable skill content without a shared Markdown validator.

## Solution Assessment

Unclear. A shared validation contract is the right direction, but the proposed second-git-folder cache and exact policy are not yet proven to be the narrowest maintainable solution; maintainers should decide rules, severity, overrides, and ClawHub scope first.
