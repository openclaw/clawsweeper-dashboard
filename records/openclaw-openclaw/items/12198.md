## Summary

Keep open. Current main still lacks one shared validation contract for prompt-bearing Markdown; the existing code has path/size/frontmatter guards, a code-oriented scanner, and a Skill Workshop-only scanner, but not a common policy for bootstrap Markdown, local `SKILL.md`, or native ClawHub skill installs.

## What This Changes

Add Markdown validation/linting for prompt-bearing files, especially skills and workspace bootstrap Markdown, to catch injection/security patterns before they become model-visible.

## Best Possible Solution

Define a shared prompt-bearing Markdown validation contract with explicit severities, override or quarantine behavior, docs, and tests, then apply it consistently to workspace bootstrap ingestion, extra bootstrap files, local skill reads, native ClawHub install/update, and ClawHub publish/tagging surfaces.

## Reproduction Assessment

Yes, at source level. A prompt-bearing marker such as an HTML comment in `AGENTS.md`, an extra bootstrap file, or `SKILL.md` still follows raw-content paths into prompt-visible context or model-readable skill files without a shared Markdown validator.

## Solution Assessment

Unclear. The requested validation infrastructure is directionally right, but the maintainable fix needs maintainer agreement on rules, severity, override/quarantine behavior, and ClawHub scope before implementation.
