## Summary

Keep open: current main still lacks a shared validation policy for prompt-bearing Markdown, and prior maintainer/Codex comments also scoped this as unresolved across bootstrap files, SKILL.md reads, and ClawHub skill installs.

## What This Changes

The issue requests Markdown validation for prompt-bearing files, especially bootstrap Markdown and skill definitions, to flag prompt-injection patterns such as HTML comments before the content becomes model-visible.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract and apply it consistently to bootstrap ingestion, extra bootstrap files, SKILL.md discovery/read paths, native ClawHub skill install/update, and ClawHub publish scanning or tagging with documented severity, override behavior, and tests.

## Reproduction Assessment

Yes, at source level. A marker such as an HTML comment or prompt-injection phrase in a recognized bootstrap file or SKILL.md body can still follow raw-content paths into Project Context or model-readable skill content without a shared Markdown validator.

## Solution Assessment

Unclear as a final implementation plan. A shared validator is the likely maintainable seam, but maintainers still need to decide severity, override or customization behavior, native ClawHub behavior, and publish-side policy.
