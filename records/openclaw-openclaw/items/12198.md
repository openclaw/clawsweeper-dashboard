## Summary

Keep open. Current main and v2026.6.5 still provide partial safeguards only: Skill Workshop scans proposals and install policy can gate staged sources, but bootstrap Markdown, extra bootstrap files, and model-visible SKILL.md reads still lack one shared prompt-bearing Markdown validation contract. The open scanner PR is related but only covers part of the requested surface.

## What This Changes

Request a shared Markdown validator/linter for prompt-bearing files such as workspace bootstrap Markdown and SKILL.md, including ClawHub publish/install policy.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract and reuse it across bootstrap ingestion, extra bootstrap files, SKILL.md loading, native ClawHub install/update, and ClawHub publish-side scan/tagging.

## Reproduction Assessment

Yes. Source inspection shows bootstrap Markdown and model-visible SKILL.md content flowing toward prompt context without a shared Markdown validator; this is source-reproducible rather than a live runtime bug repro.

## Solution Assessment

Unclear until maintainers decide the security contract. A one-path HTML-comment linter or scanner-only change would be too narrow; the maintainable shape is a shared validator/policy reused by prompt ingestion, skill loading, ClawHub install/update, and publish-side scan/tagging.
