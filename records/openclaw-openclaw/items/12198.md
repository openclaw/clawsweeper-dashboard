## Summary

Keep open: current main and the latest release still have only boundary, size, and code-scan checks; they do not provide a shared prompt-bearing Markdown validation contract for bootstrap files, SKILL.md reads, or ClawHub skill installs.

## What This Changes

Request to add Markdown validation for prompt-bearing files such as workspace bootstrap Markdown and SKILL.md, including ClawHub install or publishing policy.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract and apply it consistently to bootstrap ingestion, extra bootstrap files, SKILL.md discovery/read paths, native ClawHub install/update, and ClawHub publish scanning or tagging with explicit severity, override behavior, docs, and tests.

## Reproduction Assessment

Yes, at source level. A Markdown marker such as an HTML comment or prompt-injection phrase in a recognized bootstrap file or SKILL.md body can still follow raw-content paths into Project Context or model-readable skill content without a shared Markdown validator.

## Solution Assessment

Unclear as a final implementation plan. A shared validator is the likely maintainable seam, but maintainers still need to decide blocking severity, customization or override behavior, ClawHub install/update behavior, and publish-side policy.
