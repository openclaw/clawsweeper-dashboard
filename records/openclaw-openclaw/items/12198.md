## Summary

Keep open: current main has partial Skill Workshop prompt-content scanning and ClawHub verification/install-policy hooks, but it still lacks one shared validation contract for prompt-bearing Markdown across bootstrap files, runtime SKILL.md reads, native skill installs, and ClawHub policy.

## What This Changes

The issue requests Markdown validation/linting for prompt-bearing files, especially skill definitions and bootstrap context, to reduce prompt-injection and sensitive-data risks.

## Best Possible Solution

Define a threat-model-backed shared validation and diagnostics contract for model-visible Markdown, then apply it consistently to bootstrap context, runtime skill loading, Skill Workshop, ClawHub install/update, and publish-side verdicts.

## Reproduction Assessment

Yes, at source level: current bootstrap, prompt assembly, and runtime skill-loading paths can be traced and do not share the requested Markdown-content validation policy. This is not a strict bug reproduction because current docs and tests do not already promise such a validator.

## Solution Assessment

No, not as proposed wholesale. A centralized validation and diagnostics contract is the maintainable direction, while blanket HTML-comment rejection, automatic wrapping, nested Git tracking, general workspace-write interception, and ClawHub publish semantics need maintainer decisions.
