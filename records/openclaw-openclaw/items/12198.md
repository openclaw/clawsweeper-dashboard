## Summary

Keep open: current main has adjacent skill-content and code-safety scanning, but it still lacks a shared prompt-bearing Markdown validation contract across workspace bootstrap files, local SKILL.md reads, ClawHub installs, and publish-side policy.

## What This Changes

Request Markdown validation for model-visible prompt files and skill definitions to detect or block prompt-injection and security-risk patterns.

## Best Possible Solution

Define a threat-model-backed shared trust and diagnostic contract for model-visible Markdown, then apply it consistently to prompt ingestion, skill loading, Skill Workshop, ClawHub install/update, and publish-side verdicts without broad workspace-write blocking unless maintainers explicitly choose it.

## Reproduction Assessment

Yes, at source level: current bootstrap and local skill paths can be traced and do not share the requested Markdown-content policy. This is not a strict bug reproduction because current docs and tests do not promise a forbidden-pattern validator.

## Solution Assessment

No, not as proposed wholesale. A centralized trust and diagnostics contract is the maintainable path, while blanket HTML-comment rejection, automatic wrapping, nested Git tracking, and general workspace-write interception need separate decisions or rejection.
