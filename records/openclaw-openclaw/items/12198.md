## Summary

Keep this open. Current main and the latest release have useful partial safeguards for skill proposals, install policy, path containment, and external-content warnings, but they still do not provide the shared prompt-bearing Markdown validation contract requested for bootstrap files, model-visible SKILL.md reads, native ClawHub installs/updates, and publish-side scan/tag behavior.

## What This Changes

The issue requests Markdown validation for prompt-bearing files such as bootstrap Markdown and SKILL.md, with security-focused rules like disallowing HTML comments and optional ClawHub/power-user policy controls.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract, then apply it consistently to workspace bootstrap ingestion, extra bootstrap files, normal SKILL.md discovery/read paths, native ClawHub skill install/update, and ClawHub publish-side scan/tag behavior with explicit docs and tests.

## Reproduction Assessment

Yes. Source inspection shows current-main paths where bootstrap Markdown and model-visible SKILL.md content flow toward prompt context without a shared Markdown validation contract; this is source-reproducible rather than a live runtime bug repro.

## Solution Assessment

Unclear until maintainers decide the security contract. A one-path HTML-comment linter would be too narrow; the best shape is a shared validator/policy reused by prompt ingestion, skill loading, ClawHub install/update, and publish-side scan/tagging.
