## Summary

Keep open: current main has adjacent size/boundary checks, proposal-scoped skill scanning, and ClawHub trust metadata, but it still lacks a shared validation contract for prompt-bearing Markdown across bootstrap context, runtime SKILL.md loading, native skill install/update, and ClawHub policy.

## What This Changes

The issue requests validation or linting for prompt-bearing Markdown files, especially skill definitions, to reduce prompt-injection and security risk.

## Best Possible Solution

Define and apply one shared, threat-model-backed prompt-bearing Markdown validation and diagnostics contract across bootstrap context, runtime skill loading, Skill Workshop, ClawHub install/update, and publish-side verdicts.

## Reproduction Assessment

Yes, at source level: current bootstrap, prompt assembly, skill loading, and scanner paths show no shared Markdown-content validation policy. This is not a strict bug reproduction because current docs/tests do not already promise such a validator.

## Solution Assessment

No, not as the full proposal or a one-off scanner-only change. The maintainable path is a shared validation and diagnostics contract, with maintainers deciding rules, severity, overrides, workspace-write behavior, and ClawHub policy.
