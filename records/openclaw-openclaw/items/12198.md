## Summary

Current main still lacks the shared prompt-bearing Markdown validation contract requested here; existing safeguards cover path containment, size limits, code scanning, and Skill Workshop-generated content, but not all raw Markdown paths that feed bootstrap/context files, normal `SKILL.md` reads, or native ClawHub skill installs.

## What This Changes

The issue requests Markdown validation infrastructure for prompt-bearing files, especially skill definitions and bootstrap/context Markdown, to reduce prompt-injection risk.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract and apply it consistently to bootstrap/context ingestion, local skill reads, native ClawHub skill install/update, and ClawHub publish-side scan or tagging behavior.

## Reproduction Assessment

Yes, at source level. A marker such as an HTML comment in a recognized bootstrap file or `SKILL.md` still follows raw-content paths into prompt-visible context or model-readable skill content without a shared Markdown validator.

## Solution Assessment

Unclear. Validation infrastructure is the right direction, but maintainers still need to decide the shared ruleset, severity model, override or quarantine behavior, and ClawHub publish scope before implementation.
