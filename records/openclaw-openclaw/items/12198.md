## Summary

Keep open. Current main still lacks a shared prompt-bearing Markdown validation contract across bootstrap/context files, model-readable SKILL.md files, and native ClawHub skill install/update paths; the recent comments on this issue reached the same conclusion and no superseding implementation or canonical issue now owns the core gap.

## What This Changes

Request to add validation infrastructure for Markdown files that become prompt-visible, especially SKILL.md and workspace bootstrap files, with ClawHub publish/install policy consideration.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract with explicit severities, override/quarantine behavior, docs, and tests, then apply it consistently to bootstrap/context ingestion, local skill reads, native ClawHub install/update, and ClawHub publish/tagging surfaces.

## Reproduction Assessment

Yes, at source level. A marker such as an HTML comment in a recognized bootstrap file or SKILL.md still follows raw-content paths into prompt-visible context or model-readable skill files without a shared Markdown validator.

## Solution Assessment

Unclear. The requested validation infrastructure is directionally right, but maintainers need to decide the ruleset, severity model, override/quarantine behavior, and ClawHub scope before implementation.
