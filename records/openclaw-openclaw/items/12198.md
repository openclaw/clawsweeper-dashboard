## Summary

Keep open. Current main still has path, size, symlink, archive-root, code-safety, and optional Skill Workshop checks, but no shared prompt-bearing Markdown validation contract before workspace bootstrap files, extra bootstrap files, SKILL.md reads, or native ClawHub skill installs can become model-visible.

## What This Changes

Request Markdown validation for prompt-bearing files, especially skills and ClawHub publishing/install paths, to detect prompt-injection patterns such as HTML comments.

## Best Possible Solution

Define one reusable prompt-bearing Markdown validation contract with explicit severities, override or quarantine behavior, docs, and tests, then apply it consistently at bootstrap ingestion, extra bootstrap loading, SKILL.md discovery/read paths, native ClawHub install/update, and ClawHub publish/tagging surfaces.

## Reproduction Assessment

Yes, at source level. A prompt-injection marker or HTML comment in AGENTS.md, an extra bootstrap file, a local SKILL.md, or a native ClawHub-installed skill currently follows a raw-content path into prompt-visible context or model-readable skill content without a shared Markdown validator.

## Solution Assessment

Unclear. The requested infrastructure is directionally right, but the HTML-comment example is under-scoped; the maintainable fix needs a shared validation policy with explicit severity, warning/quarantine, override, publish, install, and test behavior.
