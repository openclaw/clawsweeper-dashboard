## Summary

Keep open. Current main still has path, size, symlink, archive-root, and code-oriented scanning, but no shared prompt-bearing Markdown validation contract before workspace bootstrap files, extra bootstrap files, local SKILL.md reads, or native ClawHub skill installs can become model-visible.

## What This Changes

Requests mandatory or configurable Markdown validation for prompt-bearing files such as workspace bootstrap Markdown and SKILL.md, including ClawHub install/publish policy.

## Best Possible Solution

Define one reusable prompt-bearing Markdown validation contract with explicit severity, warning/quarantine/override behavior, docs, and tests, then apply it consistently at bootstrap ingestion, extra bootstrap loading, SKILL.md discovery/read paths, native ClawHub install/update, and ClawHub publish/tagging surfaces.

## Reproduction Assessment

Yes. A source-level reproduction is to place an HTML comment or prompt-injection phrase in AGENTS.md, an extra bootstrap file, a local SKILL.md, or a ClawHub-installed skill; current main reads or installs that Markdown without a shared validator before it can become model-visible.

## Solution Assessment

Unclear. The requested infrastructure is valid, but the HTML-comment example is under-scoped; the maintainable fix is a shared policy with explicit severities, warnings/quarantine, overrides, and tests.
