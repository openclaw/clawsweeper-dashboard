## Summary

Keep open. Current main still has path, size, symlink, archive-root, and code-safety checks, but it does not implement a shared validation contract for prompt-bearing Markdown before workspace bootstrap files, extra bootstrap files, model-readable SKILL.md files, or native ClawHub skill installs become model-visible. The prior maintainer/Codex comments reached the same conclusion, and source inspection at current main still matches that gap.

## What This Changes

Request a mandatory or configurable linter for Markdown prompt files, especially workspace prompt files and SKILL.md, to detect prompt-injection or security-risk patterns such as HTML comments before read, write, publish, or install paths expose them.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract and apply it consistently before workspace bootstrap files, extra bootstrap files, normal SKILL.md discovery/read paths, native ClawHub skill install/update, and ClawHub publish-side scan/tag behavior expose content to the model. The contract should document severities, overrides, warnings or quarantine behavior, ClawHub tagging, and focused regression tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to place an HTML comment or prompt-injection marker in AGENTS.md, an extra bootstrap file, a local SKILL.md, or a ClawHub-installed skill; current loaders read or install that Markdown without a shared validator before it can become model-visible.

## Solution Assessment

Unclear. The requested infrastructure is valid, but an HTML-comment-only linter is under-scoped and may conflict with intentional internal markers; the maintainable solution is a reusable validation policy with explicit enforcement, exception, and override behavior across all prompt-bearing Markdown paths.
