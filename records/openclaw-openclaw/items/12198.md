## Summary

Keep open. Current main still has boundary checks, size limits, optional Skill Workshop scanning, and code-oriented install scanning, but not one shared prompt-bearing Markdown validation contract for bootstrap/context files, `SKILL.md` reads, or native ClawHub skill installs.

## What This Changes

The issue asks for validation of prompt-bearing Markdown files to detect prompt-injection or security-risk patterns before those files become model-visible.

## Best Possible Solution

Define a shared prompt-bearing Markdown validation contract, then apply it consistently to bootstrap/context ingestion, local skill reads, native ClawHub skill install/update, and ClawHub publish-side scan or tagging behavior.

## Reproduction Assessment

Yes, at source level. A marker such as an HTML comment in a recognized bootstrap file or `SKILL.md` still follows raw-content paths into prompt-visible context or model-readable skill content without a shared Markdown validator.

## Solution Assessment

Unclear. Validation infrastructure is the right direction, but maintainers need to decide the shared ruleset, severity model, override or quarantine behavior, and ClawHub publish scope before implementation.
