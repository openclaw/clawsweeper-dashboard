## Summary

Keep open: current main still lacks a shared prompt-bearing Markdown validation contract across workspace bootstrap/context files, model-visible SKILL.md reads, and native ClawHub skill installs. The related open scanner PR is useful, but it only covers one subsystem and does not settle the broader security/product policy already called out in prior discussion.

## What This Changes

The issue requests Markdown validation or linting for prompt-bearing files such as workspace bootstrap Markdown and skill definitions to reduce prompt-injection and security risk.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract and apply it consistently to bootstrap/context ingestion, local skill reads, native ClawHub install/update, and ClawHub publish scanning or tagging with documented severity and override behavior.

## Reproduction Assessment

Yes, at source level. A marker such as an HTML comment or prompt-injection phrase in a recognized bootstrap file or SKILL.md body still follows current raw-content paths into Project Context or model-readable skill content without a shared Markdown validator.

## Solution Assessment

Unclear as a final implementation plan. A shared validator is the likely maintainable seam, but maintainers still need to decide rule severity, override/customization behavior, native ClawHub behavior, and publish-side policy.
