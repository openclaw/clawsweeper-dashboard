## Summary

Keep open: current main and the latest release still lack a shared prompt-bearing Markdown validation contract across bootstrap context, runtime skill loading, ClawHub installs/publish verdicts, and Skill Workshop only covers its proposal path.

## What This Changes

Request to add validation for Markdown files that become model-visible prompt context, especially skill definitions and ClawHub-published skill content.

## Best Possible Solution

Define a threat-model-backed shared trust and diagnostics contract for model-visible Markdown, then apply it consistently to bootstrap context, runtime skill loading, Skill Workshop, ClawHub install/update, and publish-side verdicts.

## Reproduction Assessment

Yes, at source level: current bootstrap, prompt assembly, and runtime skill-loading paths can be traced and do not share the requested Markdown-content validation policy. This is not a strict bug reproduction because current docs and tests do not already promise such a validator.

## Solution Assessment

No, not as proposed wholesale. A centralized validation and diagnostics contract is the maintainable direction, while blanket HTML-comment rejection, automatic wrapping, nested Git tracking, general workspace-write interception, and ClawHub publish semantics need maintainer decisions.
