## Summary

Keep this open. Current main still lacks a shared validation contract for prompt-bearing Markdown; the existing code has boundary checks, size limits, JS/TS code scanning, ClawHub install hardening, and Skill Workshop proposal scanning, but workspace bootstrap content and model-visible SKILL.md reads still flow without a common Markdown policy. The open scanner PR is related but only covers one slice and currently targets an obsolete path, so this issue remains the canonical security/product decision.

## What This Changes

Request a Markdown validation and linting contract for prompt-bearing files such as workspace bootstrap Markdown and SKILL.md, including ClawHub install/publish policy and possible severity or override behavior.

## Best Possible Solution

Define one maintainer-approved prompt-bearing Markdown validation contract and apply it consistently to workspace bootstrap files, extra bootstrap files, SKILL.md loading, native ClawHub skill install/update, and ClawHub publish-side scan or tagging behavior.

## Reproduction Assessment

Yes, at source level: current bootstrap/context and local skill paths can be traced into model-visible content without a shared Markdown-content validation step. This is not a strict bug reproduction because current docs/tests do not promise such a validator.

## Solution Assessment

No, not as proposed wholesale. The maintainable path is a centralized trust and diagnostics contract; blanket HTML-comment rejection, automatic wrapping, nested Git tracking, workspace-write interception, and customization knobs need separate maintainer decisions or rejection.
