## Summary

Keep open. Current main still lacks a shared prompt-bearing Markdown validation contract across workspace/bootstrap prompt files, local SKILL.md reads, and native ClawHub skill install/update paths, and the recent maintainer/Codex discussion still applies.

## What This Changes

Request to add validation or linting for Markdown files that become prompt-visible, including SKILL.md and workspace bootstrap files, with ClawHub enforcement/tagging and optional policy controls.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract with explicit severities, override/quarantine behavior, docs, and tests, then apply it consistently to bootstrap/context ingestion, local skill reads, native ClawHub install/update, and ClawHub publish/tagging surfaces.

## Reproduction Assessment

Yes, at source level. A marker such as an HTML comment in a recognized bootstrap file or SKILL.md still follows raw-content paths into prompt-visible context or model-readable skill files without a shared Markdown validator.

## Solution Assessment

Unclear. The requested validation infrastructure is directionally right, but maintainers need to decide the shared ruleset, severity model, override/quarantine behavior, and ClawHub scope before implementation.
