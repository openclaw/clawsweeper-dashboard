## Summary

Keep open: current main still lacks a shared prompt-bearing Markdown validation contract. Existing guards cover path containment, size limits, code-oriented scans, and Skill Workshop writes, but bootstrap/context Markdown, normal SKILL.md reads, and native ClawHub skill installs still have raw model-visible paths.

## What This Changes

The issue requests validation or linting for Markdown files that can become prompt-bearing content, especially skill definitions, workspace bootstrap files, and ClawHub skill publishing or install flows.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract and apply it consistently to bootstrap/context ingestion, local skill reads, native ClawHub skill install/update, and ClawHub publish scanning or tagging with documented severity and override behavior.

## Reproduction Assessment

Yes, at source level. A marker such as an HTML comment in a recognized bootstrap file or SKILL.md body still follows current raw-content paths into Project Context or model-readable skill content without a shared Markdown validator.

## Solution Assessment

Unclear. A shared validator is the right likely seam, but maintainers still need to settle the ruleset, severity, override/quarantine behavior, and ClawHub publish scope before implementation.
