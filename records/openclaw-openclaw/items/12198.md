## Summary

Keep open: current main still has no shared prompt-bearing Markdown validation contract across workspace bootstrap/context files, model-visible SKILL.md reads, and native ClawHub skill installs. The related open scanner PR is useful but only covers the scanner subsystem, so it does not settle the broader security/product policy called out in prior discussion.

## What This Changes

The issue requests mandatory or configurable validation for Markdown files that become prompt-visible, especially skill definitions and workspace prompt files, to reduce prompt-injection risk.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract and apply it consistently to bootstrap/context ingestion, local skill reads, native ClawHub install/update, and ClawHub publish scanning or tagging with documented severity and override behavior.

## Reproduction Assessment

Yes, at source level. A marker such as an HTML comment or prompt-injection phrase in a recognized bootstrap file or SKILL.md body still follows raw-content paths into Project Context or model-readable skill content without a shared Markdown validator.

## Solution Assessment

Unclear as a final implementation plan. A shared validator is the likely maintainable seam, but maintainers still need to decide rule severity, override/customization behavior, native ClawHub behavior, and publish-side policy.
