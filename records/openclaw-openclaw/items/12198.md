## Summary

Current main still lacks a shared validation contract for model-visible Markdown across workspace bootstrap files, prompt templates, and skill reads. ClawHub now provides registry verification/security verdicts, but that does not cover local prompt ingestion or define the requested rules; because the issue is security-sensitive, protected, and requires policy choices about severity and overrides, it should remain open for owner review.

## What This Changes

Add configurable security validation for prompt-bearing Markdown during local ingestion and skill install, update, and publishing workflows.

## Best Possible Solution

Define one threat-model-backed contract for model-visible Markdown, align it with ClawHub security verdicts, and explicitly specify warning versus blocking severities, diagnostics, trusted overrides, caching, and ownership for local ingestion versus registry lifecycle enforcement.

## Reproduction Assessment

Yes at source level: current workspace bootstrap and prompt-template paths demonstrably ingest Markdown without a common content-policy validator. The issue does not, however, provide a concrete exploit transcript or agreed severity threshold for individual patterns.

## Solution Assessment

No, not as fully proposed: the secondary Git index, blanket HTML-comment ban, and workspace-wide secret blocking are not established as the narrowest reliable controls. A shared model-visible-content policy integrated with existing scanner and ClawHub verdict surfaces is the better direction.
