## Summary

Keep open. Current main still lets prompt-visible Markdown flow through workspace bootstrap, SKILL.md loading, and native ClawHub skill installs without a shared validation policy; existing scanners are either code-focused or limited to Skill Workshop-generated content, matching the prior maintainer/Codex comments.

## What This Changes

Add validation or linting for Markdown files that become prompt context, including workspace bootstrap files and skill definitions, to reduce prompt-injection and security risks.

## Best Possible Solution

Define one reusable prompt-bearing Markdown validation contract with explicit severity, warning/quarantine, override, docs, and test coverage, then apply it consistently to bootstrap ingestion, extra bootstrap files, normal SKILL.md discovery/read paths, native ClawHub install/update, and ClawHub publish/tagging surfaces.

## Reproduction Assessment

Yes. A source-level reproduction is to place an HTML comment or prompt-injection marker in AGENTS.md, an extra bootstrap file, a local SKILL.md, or a ClawHub-installed skill; current main still reads or installs that Markdown without a shared validator before it can become model-visible.

## Solution Assessment

Unclear. The requested infrastructure is the right direction, but an HTML-comment-only ban is under-scoped; the maintainable solution is a shared policy with explicit severities, override/quarantine behavior, docs, and tests.
