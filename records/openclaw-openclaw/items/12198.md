## Summary

Keep open. Current main still lacks a shared prompt-bearing Markdown validation contract across bootstrap/context files, local skill reads, and native ClawHub skill install/update paths; the open scanner PR covers only part of the SKILL.md install/audit surface and does not resolve the broader policy gap.

## What This Changes

Add validation for Markdown prompt sources such as skill definitions and bootstrap files to catch prompt-injection or security patterns, with possible ClawHub enforcement and configurable policy.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract with explicit severities, override/quarantine behavior, docs, and tests, then apply it consistently to bootstrap/context ingestion, local skill reads, native ClawHub install/update, and ClawHub publish/tagging surfaces.

## Reproduction Assessment

Yes, at source level. A marker such as an HTML comment in a recognized bootstrap file or SKILL.md still follows raw-content paths into prompt-visible context or model-readable skill files without a shared Markdown validator.

## Solution Assessment

Unclear. The requested validation infrastructure is directionally right, and the open scanner PR may cover a useful slice, but maintainers need to decide the shared ruleset, severity model, override/quarantine behavior, and ClawHub scope before implementation.
