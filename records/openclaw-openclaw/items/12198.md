## Summary

Keep open: current main still has source-level paths where prompt-bearing Markdown is loaded into model-visible context without a shared validation policy, and the open scanner PR only covers the scanner surface rather than workspace bootstrap, SKILL.md reads, ClawHub install/update, and publish policy.

## What This Changes

Request to add validation or linting for prompt-bearing Markdown files, especially skill definitions and bootstrap/context files, to reduce prompt-injection risk.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract and reuse it across workspace bootstrap ingestion, extra bootstrap files, local SKILL.md reads, native ClawHub install/update, and ClawHub publish scanning or tagging with documented severity, override, and tests.

## Reproduction Assessment

Yes, at source level. A marker such as an HTML comment or prompt-injection phrase in a recognized bootstrap file or SKILL.md body still follows raw-content paths into Project Context or model-readable skill content without a shared Markdown validator.

## Solution Assessment

Unclear as a final implementation plan. A shared validator is the likely maintainable seam, but maintainers still need to decide rule severity, override/customization behavior, native ClawHub behavior, and publish-side policy.
