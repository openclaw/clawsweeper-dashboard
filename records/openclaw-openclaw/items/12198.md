## Summary

Keep open. Current main still lacks a shared prompt-bearing Markdown validation contract; existing guards cover paths, sizes, frontmatter, code-oriented skill scanning, and Skill Workshop-generated content, but not all bootstrap Markdown, local `SKILL.md` reads, or native ClawHub skill install/update paths.

## What This Changes

Markdown prompt files should be linted for prompt-injection-prone patterns such as HTML comments before they are used in model-visible contexts.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract with explicit severities, override or quarantine behavior, docs, and tests, then apply it consistently to bootstrap/context ingestion, local skill reads, native ClawHub install/update, and ClawHub publish/tagging surfaces.

## Reproduction Assessment

Yes, at source level. A marker such as an HTML comment in a recognized bootstrap file or `SKILL.md` still follows raw-content paths into prompt-visible context or model-readable skill files without a shared Markdown validator.

## Solution Assessment

Unclear. The requested validation infrastructure is directionally right, but maintainers need to decide the rules, severity, override/quarantine behavior, and ClawHub scope before implementation.
