## Summary

Keep open: current main still has model-visible Markdown paths for workspace bootstrap files and SKILL.md content without one shared validation policy; related scanner/security PRs are partial or unmerged and do not close the full gap.

## What This Changes

Request to add validation for Markdown files that feed prompts, especially workspace bootstrap files and SKILL.md, to catch prompt-injection patterns such as HTML comments.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract and apply it consistently to workspace bootstrap ingestion, extra bootstrap files, local SKILL.md reads, native ClawHub install/update, and ClawHub publish scanning or tagging with documented severity, override, and tests.

## Reproduction Assessment

Yes, at source level. A marker such as an HTML comment or prompt-injection phrase in a recognized bootstrap file or SKILL.md body can still follow raw-content paths into Project Context or model-readable skill content without a shared Markdown validator.

## Solution Assessment

Unclear as a final implementation plan. A shared validator is the likely maintainable seam, but maintainers still need to decide rule severity, override/customization behavior, native ClawHub behavior, and publish-side policy.
