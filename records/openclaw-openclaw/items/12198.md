## Summary

Keep open: current main still lacks one shared prompt-bearing Markdown validation policy across workspace bootstrap files, `SKILL.md` reads, and native ClawHub skill installs. The related open PR https://github.com/openclaw/openclaw/pull/43469 covers part of the scanner gap, but it is not merged and does not settle the broader prompt-ingestion, override, and ClawHub policy decisions.

## What This Changes

Request to add validation for Markdown files that can become prompt-visible, including skills, workspace/bootstrap files, and ClawHub publish or install policy.

## Best Possible Solution

Define a shared prompt-bearing Markdown validation contract and apply it consistently to bootstrap ingestion, extra bootstrap files, `SKILL.md` discovery/read paths, native ClawHub install/update, and ClawHub publish scanning or tagging with documented severity, override behavior, and tests.

## Reproduction Assessment

Yes, at source level. A marker such as an HTML comment or prompt-injection phrase in a recognized bootstrap file or `SKILL.md` body can still follow raw-content paths into Project Context or model-readable skill content without a shared Markdown validator.

## Solution Assessment

Unclear as a final implementation plan. A shared validator is the likely maintainable seam, but maintainers still need to decide rule severity, override/customization behavior, native ClawHub behavior, and publish-side policy.
