## Summary

Keep open: current main still has prompt-visible Markdown paths for workspace bootstrap files and SKILL.md content without one shared validation policy, and the related scanner or guardrail work is partial, closed, or not equivalent.

## What This Changes

Request Markdown validation infrastructure for prompt-bearing files, especially skill definitions and bootstrap Markdown, to detect prompt-injection patterns such as HTML comments.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract and apply it consistently to bootstrap ingestion, extra bootstrap files, SKILL.md discovery/read paths, native ClawHub skill install/update, and ClawHub publish scanning or tagging with documented severity, override behavior, and tests.

## Reproduction Assessment

Yes, at source level. A marker such as an HTML comment or prompt-injection phrase in a recognized bootstrap file or SKILL.md body can still follow raw-content paths into Project Context or model-readable skill content without a shared Markdown validator.

## Solution Assessment

Unclear as a final implementation plan. A shared validator is the likely maintainable seam, but maintainers still need to decide severity, override/customization behavior, native ClawHub behavior, and publish-side policy.
