## Summary

Keep open: current main still has partial hardening, but it does not provide one shared prompt-bearing Markdown validation contract for bootstrap files, SKILL.md reads, and native ClawHub skill installs.

## What This Changes

Requested Markdown validation for prompt-bearing files, including skill definitions, workspace bootstrap Markdown, ClawHub skill publishing/install paths, and optional read/write policy controls.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract and apply it consistently to bootstrap ingestion, extra bootstrap files, SKILL.md discovery/read paths, native ClawHub skill install/update, and ClawHub publish scanning or tagging with explicit severity, override behavior, docs, and tests.

## Reproduction Assessment

Yes, at source level. A Markdown marker such as an HTML comment or prompt-injection phrase in a recognized bootstrap file or SKILL.md body can still follow raw-content paths into Project Context or model-readable skill content without a shared Markdown validator.

## Solution Assessment

Unclear as a final implementation plan. A shared validator is the likely maintainable seam, but maintainers still need to decide blocking severity, customization or override behavior, ClawHub install/update behavior, and publish-side policy.
