## Summary

Keep open: current main still lacks a shared prompt-bearing Markdown validation contract across bootstrap/context files, normal SKILL.md reads, and native ClawHub skill installs, and the existing issue discussion already scoped this as unresolved security/product work.

## What This Changes

The issue requests validation or linting for Markdown files that become prompt content, especially skill definitions and bootstrap/context files, to reduce prompt-injection and secret-exposure risk.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract and apply it consistently to bootstrap/context ingestion, local skill reads, native ClawHub install/update, and ClawHub publish scanning or tagging with documented severity and override behavior.

## Reproduction Assessment

Yes, at source level. A marker such as an HTML comment or prompt-injection phrase in a recognized bootstrap file or SKILL.md body still follows current raw-content paths into Project Context or model-readable skill content without a shared Markdown validator.

## Solution Assessment

Unclear. A shared validator is the likely maintainable seam, but maintainers still need to settle the ruleset, severity, override/quarantine behavior, and ClawHub publish scope before implementation.
