## Summary

Keep open. Current main still lacks a shared prompt-bearing Markdown validation contract; existing guards cover paths, sizes, frontmatter, code-oriented scanning, and Skill Workshop-generated content, but not all bootstrap Markdown, local `SKILL.md` reads, or native ClawHub skill install/update paths.

## What This Changes

Request to add prompt-injection-oriented Markdown validation for prompt-bearing files such as workspace bootstrap Markdown, `SKILL.md`, ClawHub skill publishing/installing, and possibly Markdown writes.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract with explicit severities, override or quarantine behavior, docs, and tests, then apply it consistently to bootstrap/context ingestion, local skill reads, native ClawHub install/update, and ClawHub publish/tagging surfaces.

## Reproduction Assessment

Yes, at source level. A marker such as an HTML comment in a recognized bootstrap file or `SKILL.md` still follows raw-content paths into prompt-visible context or model-readable skill files without a shared Markdown validator.

## Solution Assessment

Unclear. The requested validation infrastructure is directionally right, but maintainers need to decide the ruleset, severity model, override/quarantine behavior, and ClawHub scope before implementation.
