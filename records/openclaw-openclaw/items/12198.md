## Summary

Keep open: current main still lacks a shared prompt-bearing Markdown validation contract. Existing safeguards cover path containment, size limits, code scanning, and Skill Workshop writes, but bootstrap/context files, normal SKILL.md reads, and native ClawHub skill installs can still carry raw Markdown into model-visible paths.

## What This Changes

Request a shared validation policy for Markdown files that become prompt context or skill instructions, including ClawHub publishing/install behavior and possible override modes.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract and apply it consistently to bootstrap/context ingestion, local skill reads, native ClawHub skill install/update, and ClawHub publish-side scan or tagging behavior.

## Reproduction Assessment

Yes, at source level. A marker such as an HTML comment in a recognized bootstrap file or SKILL.md still follows raw-content paths into prompt-visible context or model-readable skill content without a shared Markdown validator.

## Solution Assessment

Unclear. Validation infrastructure is the right direction, but maintainers still need to decide the ruleset, severity, override/quarantine behavior, and ClawHub publish scope before implementation.
