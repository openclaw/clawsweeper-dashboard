## Summary

Keep open. Current main and the latest release have useful boundary checks, code-oriented install scanning, external-content warnings, and Skill Workshop proposal scanning, but they still do not provide one shared prompt-bearing Markdown validation contract across workspace bootstrap files, extra bootstrap files, model-visible SKILL.md reads, native ClawHub skill installs, and publish-side policy.

## What This Changes

Request to add Markdown validation or linting for prompt-bearing files such as workspace bootstrap Markdown and skill definitions, including ClawHub publishing/install handling.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract and apply it consistently to bootstrap ingestion, extra bootstrap files, SKILL.md discovery/read paths, native ClawHub install/update, and ClawHub publish-side scan/tag behavior.

## Reproduction Assessment

Yes, at source level. A prompt-bearing Markdown file such as AGENTS.md, an extra bootstrap file, or SKILL.md can include suspicious Markdown/HTML-comment text and current main will read and inject or expose it without a shared Markdown validation contract; I did not run a live write-based repro because this review is read-only.

## Solution Assessment

Unclear as proposed. The best fix is not the suggested secondary .git tracking mechanism; it is a shared validation contract with explicit severity, override, tagging, docs, and tests across all prompt-bearing Markdown entry points.
