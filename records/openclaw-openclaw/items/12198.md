## Summary

Keep open. Current main still lacks a shared prompt-bearing Markdown validation contract for bootstrap/context files, model-visible SKILL.md reads, and native ClawHub skill installs, and the recent maintainer/Codex comments remain accurate.

## What This Changes

Add Markdown validation for prompt-visible files such as workspace bootstrap Markdown and SKILL.md, with ClawHub install/publish policy and optional override behavior.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract and apply it consistently to bootstrap/context ingestion, local skill reads, native ClawHub install/update, and ClawHub publish/tagging surfaces.

## Reproduction Assessment

Yes, at source level. A marker such as an HTML comment in a recognized bootstrap file or SKILL.md still follows raw-content paths into prompt-visible context or model-readable skill files without a shared Markdown validator.

## Solution Assessment

Unclear. Validation infrastructure is the right direction, but maintainers need to decide the shared ruleset, severity model, override/quarantine behavior, and ClawHub scope before implementation.
