## Summary

Keep open. Current main still lacks a shared prompt-bearing Markdown validation contract across workspace bootstrap files, extra bootstrap files, local SKILL.md reads, and native ClawHub skill install/update paths; the recent maintainer/Codex comments remain accurate.

## What This Changes

Request Markdown validation for files appended to prompts, especially SKILL.md and bootstrap Markdown, to catch prompt-injection patterns such as HTML comments before use or publication.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract with explicit severities, override/quarantine behavior, docs, and tests, then apply it consistently to bootstrap/context ingestion, local skill reads, native ClawHub skill install/update, and ClawHub publish/tagging surfaces.

## Reproduction Assessment

Yes, at source level. A marker such as an HTML comment in a recognized bootstrap file or SKILL.md still follows raw-content paths into prompt-visible context or model-readable skill files without a shared Markdown validator.

## Solution Assessment

Unclear. Validation infrastructure is directionally right, but maintainers need to decide the shared ruleset, severity model, override/quarantine behavior, and ClawHub scope before implementation.
