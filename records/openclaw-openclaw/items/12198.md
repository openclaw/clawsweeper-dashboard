## Summary

Keep open. The prior maintainer/Codex comments correctly identified a remaining gap, and current main still lacks one shared validation contract for prompt-bearing Markdown before bootstrap files, extra bootstrap files, local skills, or native ClawHub-installed skills become model-visible.

## What This Changes

The issue requests a reusable Markdown validation layer for prompt-bearing files such as skills and bootstrap Markdown, including ClawHub publish/install policy and optional power-user controls.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract with explicit severities, warning/quarantine or override behavior, docs, and tests, then apply it consistently at bootstrap ingestion, extra bootstrap loading, `SKILL.md` discovery/read paths, native ClawHub install/update, and ClawHub publish/tagging surfaces.

## Reproduction Assessment

Yes, at source level. A prompt-bearing Markdown marker such as an HTML comment in `AGENTS.md`, an extra bootstrap file, a local `SKILL.md`, or a native ClawHub-installed `SKILL.md` still follows raw-content paths into prompt-visible or model-readable surfaces without a shared Markdown validator.

## Solution Assessment

Unclear. The requested validation infrastructure is directionally right, but the exact rule set and enforcement behavior need maintainer security/product policy before implementation should be automated.
