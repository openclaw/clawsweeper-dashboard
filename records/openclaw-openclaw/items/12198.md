## Summary

Keep open. Current main still lacks a shared prompt-bearing Markdown validation contract for bootstrap/context files, local SKILL.md reads, and native ClawHub installs; related open PRs cover useful slices but not the broader policy gap.

## What This Changes

Request a Markdown validation layer for prompt-visible files such as SKILL.md and workspace bootstrap files to flag prompt-injection or security patterns.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation policy with explicit severities, overrides, docs, and tests, then apply it consistently to bootstrap/context ingestion, local skill reads, native ClawHub install/update, and ClawHub publish/tagging surfaces.

## Reproduction Assessment

Yes, at source level. A recognized bootstrap Markdown file or SKILL.md body still follows raw-content paths into prompt-visible context or model-readable skill files without a shared Markdown validator.

## Solution Assessment

Unclear. The validation direction is right, but maintainers need to decide the shared ruleset, severity model, override behavior, and ClawHub scope before a narrow implementation is safe.
