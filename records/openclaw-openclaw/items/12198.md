## Summary

Keep open: current main has partial skill-content scanning, but still lacks a shared prompt-bearing Markdown validation contract across workspace bootstrap/context files, model-visible SKILL.md loading, native skill install/update, and ClawHub publish/install policy.

## What This Changes

The issue requests validation for Markdown files that become model-visible prompt context, especially skill definitions and other prompt-bearing Markdown, to reduce prompt-injection risk.

## Best Possible Solution

Define a threat-model-backed shared validation and diagnostics contract for model-visible Markdown, then apply it consistently to prompt ingestion, skill loading, Skill Workshop, native skill install/update, and ClawHub publish/install verdicts while splitting off broad workspace-write blocking unless maintainers explicitly choose it.

## Reproduction Assessment

Yes. Source inspection on current main shows bootstrap/context Markdown and normal SKILL.md loading do not share the requested validation contract, while the current scanner still excludes .md files from directory scans.

## Solution Assessment

No, not as proposed wholesale. A centralized trust and diagnostics contract is the maintainable path; blanket HTML-comment rejection, automatic wrapping, nested Git tracking, and general workspace-write interception need separate maintainer decisions.
