## Summary

Keep open. Current main still lacks one shared prompt-bearing Markdown validation contract; bootstrap/context files, `SKILL.md` discovery/read paths, native ClawHub skill installs, and the existing code-oriented scanner do not cover the requested policy.

## What This Changes

The issue requests validation infrastructure for Markdown files that can become prompt-visible context, especially skill definitions and workspace bootstrap files.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract and apply it consistently to bootstrap/context ingestion, local skill reads, native ClawHub install/update, and ClawHub publish-side scan or tagging behavior.

## Reproduction Assessment

Yes, at source level. A marker such as an HTML comment in a recognized bootstrap file or `SKILL.md` still follows raw-content paths into prompt-visible context or model-readable skill content without a shared Markdown validator.

## Solution Assessment

Unclear. Validation infrastructure is the right direction, but maintainers need to decide the shared ruleset, severity model, override or quarantine behavior, and ClawHub scope before implementation.
