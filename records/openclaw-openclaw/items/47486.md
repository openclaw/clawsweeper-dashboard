## Summary

Keep open: current main still lacks an equivalent broadcast tool, but this branch has blocking provenance, dependency, lane-policy, and restricted-surface gaps before it can safely add a core cross-session injection primitive.

## What This Changes

The branch adds a core `sessions_broadcast` agent tool, registers it in the default tool catalog/tool set, and adds unit coverage for channel-scoped multi-session delivery.

## Best Possible Solution

If maintainers want this capability, rebuild it as a policy-aware sibling of `sessions_send`: declared dependencies, valid `inter_session` provenance, per-target nested lanes, matching deny/mutation surfaces, docs/changelog, and focused regression coverage on current main.

## Reproduction Assessment

Yes. Source inspection is enough: the PR sends an invalid provenance kind through a schema-validated Gateway method and exposes a new cross-session injection tool without matching existing `sessions_send` policy surfaces.

## Solution Assessment

No. The branch is a useful implementation sketch and addressed the first automated feedback, but it is not the narrowest maintainable solution until the core trust-boundary decision and mechanical policy gaps are resolved.
