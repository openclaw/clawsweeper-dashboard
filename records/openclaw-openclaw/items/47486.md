## Summary

Keep this PR open. Current main still lacks the requested tool, but the branch has blocking correctness/security issues and only mocked proof for a new cross-session injection primitive.

## What This Changes

Adds a core `sessions_broadcast` agent tool, registers it in the session tool catalog/default tool set, and adds mocked multi-session broadcast coverage.

## Best Possible Solution

Land this only if maintainers accept the core feature and the implementation reuses or extends the existing session-send provenance, thread-session, subagent-policy, lane, dependency, docs, and real-environment proof contracts.

## Reproduction Assessment

Yes for the PR review findings by source inspection: the latest head uses an unsupported provenance kind, can include thread-scoped sessions, exposes the tool to subagent contexts, uses a shared nested lane, and imports an undeclared TypeBox package. I did not run a live broadcast because this read-only review found no real behavior proof in the PR body or comments.

## Solution Assessment

No, not as currently patched. The maintainable path is to preserve the existing `sessions_send` safety boundaries or intentionally extend those contracts with maintainer approval before adding a broadcast primitive.

  - confidence: 0.87
