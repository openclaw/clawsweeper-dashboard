## Summary

Keep this PR open, but it is not merge-ready: current main still lacks `sessions_broadcast`, while the proposed core broadcast primitive violates current gateway provenance, session-safety, subagent-policy, lane, dependency, and real-proof expectations.

## What This Changes

The PR adds a core `sessions_broadcast` agent tool, registers it in the tool catalog/default tool set, and adds mocked tests for multi-session broadcasts.

## Best Possible Solution

Land this only if maintainers accept the core feature and the implementation follows the existing `sessions_send` provenance, thread-session, subagent-policy, lane, and dependency contracts with real multi-session proof.

## Reproduction Assessment

Yes for the PR review findings by source inspection: the latest head sends an unsupported provenance kind, can include thread sessions, exposes broadcast to subagent contexts, uses a shared nested lane, and imports an undeclared TypeBox package. I did not run a live broadcast because this review is read-only.

## Solution Assessment

No, not as currently patched. The maintainable path is to reuse or formally extend the gateway provenance contract, preserve `sessions_send` safety boundaries, and prove the behavior in a real multi-session setup.
