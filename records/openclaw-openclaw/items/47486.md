  - body: `sessions_broadcast` sends `inputProvenance.kind: "broadcast"`, but the gateway validates `agent` params against the existing provenance enum before dispatch.   - body: Adding this tool to the default OpenClaw tool set exposes a broader cross-session injection path to subagents, while current policy denies `sessions_send` so subagents communicate through the announce chain.   - confidence: 0.91

  - body: The target loop adds every visible session key, so broadcast can target active thread-scoped chat sessions. Current `sessions_send` rejects thread sessions for inter-agent coordination; apply the same guard before adding a broadcast target.
  - confidence: 0.89
- **[P2] Use per-target nested lanes for broadcast sends:** `src/agents/tools/sessions-broadcast-tool.ts:140`
  - body: Every target send uses the shared nested lane, which can reintroduce cross-session head-of-line blocking. Use `resolveNestedAgentLaneForSession(internalKey)` for each target, matching the current `sessions_send` behavior.
  - confidence: 0.87
- **[P2] Import TypeBox from the declared package:** `src/agents/tools/sessions-broadcast-tool.ts:2`
  - body: Current main imports TypeBox from `typebox`, and the root package declares `typebox`, not `@sinclair/typebox`. Match the declared dependency so installs and type resolution do not rely on an undeclared transitive package.
  - confidence: 0.88
