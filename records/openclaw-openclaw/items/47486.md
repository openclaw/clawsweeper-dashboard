## Summary

Keep this PR open. The proposed tool still has blocking correctness and security-boundary issues: broadcast sends use an invalid Gateway provenance kind, and default tool wiring exposes a new cross-session injection path without updating the subagent deny policy; the PR also still lacks after-fix real behavior proof.

## What This Changes

The PR adds a core `sessions_broadcast` agent tool, registers it in the session tool catalog/profiles, wires it into `createOpenClawTools`, and adds unit tests.

## Best Possible Solution

Keep the feature work open, but require a valid Gateway provenance contract, explicit subagent policy handling with regression coverage, docs for the new tool surface, and real cross-session runtime proof before considering merge.

## Reproduction Assessment

Yes, from source inspection. The PR sends `inputProvenance.kind: "broadcast"`, while current main's Gateway contract only accepts `external_user`, `inter_session`, or `internal_system`, so the targeted `agent` calls can be rejected before delivery.

## Solution Assessment

No. The current PR shape is not mergeable until it uses an existing or intentionally added provenance contract and preserves the subagent announce-chain boundary instead of adding an unhandled cross-session send path.
