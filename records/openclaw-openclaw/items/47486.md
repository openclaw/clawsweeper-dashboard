## Summary

Keep this PR open, but do not merge it as-is. Current main still lacks `sessions_broadcast`, but the branch sends an invalid provenance kind that current Gateway validation rejects, exposes the new cross-session tool to subagents by default, and lacks after-fix real behavior proof.

## What This Changes

The PR adds a core `sessions_broadcast` agent tool, registers it in the tool catalog and runtime tool set, and adds unit coverage for broadcasting to visible sessions with optional channel scope.

## Best Possible Solution

Land a rebased core implementation only after it uses accepted provenance, preserves subagent/embedded/sandbox/mutation guards, updates focused tests and docs, and includes redacted real Gateway proof.

## Reproduction Assessment

Yes for the blocking source paths. The PR sends `inputProvenance.kind: "broadcast"`, while current main only accepts three provenance kinds and validates `agent` params before any run is enqueued.

## Solution Assessment

No. The feature direction may be useful, but this patch should first fit the existing provenance contract and cross-session policy surfaces instead of adding a new default tool path that bypasses current subagent and mutation guards.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use a valid provenance kind for broadcast sends:** `src/agents/tools/sessions-broadcast-tool.ts:142`
  - body: `inputProvenance.kind` is set to `"broadcast"`, but the current Gateway schema only accepts `external_user`, `inter_session`, or `internal_system` and rejects invalid `agent` params before enqueueing the run. As written, each broadcast `agent` call will fail validation; use an accepted kind and keep `sourceTool: "sessions_broadcast"` for specificity.
  - confidence: 0.94
- **[P1] Keep subagents from receiving sessions_broadcast by default:** `src/agents/openclaw-tools.ts:189`
  - body: Adding this factory to the default tool set makes it survive subagent filtering because the current subagent deny list only blocks `sessions_send`. That opens a direct cross-session injection path for subagents that are supposed to communicate through the announce chain, so add the new tool to the same deny/default policy surfaces and tests before exposing it.
  - confidence: 0.88
- **[P2] Classify sessions_broadcast as a mutating tool:** `src/agents/tools/sessions-broadcast-tool.ts:36`
  - body: The new tool injects messages into other sessions, but `sessions_broadcast` is neither in the explicit mutating-tool lists nor matched by the current name heuristic. That can make side-effectful broadcast calls look replay-safe or non-mutating in embedded runner error handling and approval classification; add the new name to the mutation classifier and tests.
  - confidence: 0.78
