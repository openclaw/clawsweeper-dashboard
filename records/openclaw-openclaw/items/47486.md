## Summary

Keep open. Current main still lacks `sessions_broadcast`, and this remains a meaningful feature PR, but the branch needs product/security review, contributor real-behavior proof, and fixes for the dependency, provenance, and restricted-surface defects before merge.

## What This Changes

The PR adds a core `sessions_broadcast` agent tool, registers it in the tool catalog and default tool factory, and adds unit tests for channel-scoped broadcast delivery.

## Best Possible Solution

If maintainers want this in core, land it as a policy-aware sibling of `sessions_send` with the declared schema package, valid inter-session provenance, matching restricted-surface coverage, docs/changelog, and real multi-session proof.

## Reproduction Assessment

Yes for the PR defects. Source inspection shows the invalid provenance kind, undeclared import, and missing restricted-surface policy coverage against current-main contracts.

## Solution Assessment

No. The branch is a useful implementation sketch, but the current shape is not the narrowest maintainable solution until the trust boundary and policy details are settled.
