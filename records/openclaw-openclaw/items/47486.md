## What This Changes

## Best Possible Solution

Land a rebased and documented core tool only after it uses a valid Gateway provenance contract, preserves the subagent announce-chain boundary with tests, and includes real cross-session proof.

No. The current PR shape is not mergeable until it uses an existing or intentionally added provenance contract and explicitly handles subagent policy for this new cross-session send path.
