- **[P1] Use a valid provenance kind for broadcasts:** `src/agents/tools/sessions-broadcast-tool.ts:142`
  - body: `inputProvenance.kind` is set to `"broadcast"`, but the current Gateway schema only accepts `external_user`, `inter_session`, or `internal_system` and rejects invalid `agent` params before enqueueing the run. Use an accepted kind and keep `sourceTool: "sessions_broadcast"` for specificity.
  - confidence: 0.94
- **[P1] Deny sessions_broadcast for subagents by default:** `src/agents/openclaw-tools.ts:189`
  - body: Adding this factory to the default tool set makes it available in subagent contexts, while the current subagent deny list only blocks `sessions_send`. That opens a direct cross-session injection path for subagents that are supposed to communicate through the announce chain, so update the same policy surface and tests before exposing it.
  - confidence: 0.88
- **[P2] Classify sessions_broadcast as mutating:** `src/agents/tools/sessions-broadcast-tool.ts:36`
  - body: The new tool injects messages into other sessions, but `sessions_broadcast` is neither in the explicit mutating-tool list nor matched by the current name heuristic. That can make side-effectful broadcast calls look replay-safe or non-mutating in embedded runner handling; add the new name to the mutation classifier and tests.
  - confidence: 0.78
