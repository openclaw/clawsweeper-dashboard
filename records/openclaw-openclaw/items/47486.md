## Summary

Keep open, but not mergeable as-is: the feature is not on current main, and the PR still has source-level blockers around gateway provenance, subagent/thread safety, lane/dependency drift, plus no real behavior proof.

## What This Changes

The branch adds a core `sessions_broadcast` agent tool, registers it in the sessions tool catalog/default tool set, and adds mocked tests for broadcasting messages to active sessions.

## Best Possible Solution

Keep this as the implementation candidate only after it uses supported provenance semantics or a documented protocol extension, preserves `sessions_send` safety boundaries, follows current lane/dependency contracts, and includes redacted live proof.

## Reproduction Assessment

Yes for the blocking review findings by source inspection: the PR sends a provenance kind rejected by the current schema, bypasses current subagent/thread targeting boundaries, and uses stale lane/dependency contracts. I did not run a live broadcast in this read-only review.

## Solution Assessment

No, not as currently patched. The maintainable path is to reuse or formally extend the gateway provenance contract, mirror existing `sessions_send` safety rules where applicable, and require live proof after the source-level blockers are fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.96

Full review comments:

- **[P1] Use a valid provenance kind for broadcast sends:** `src/agents/tools/sessions-broadcast-tool.ts:142`
  - body: `sessions_broadcast` sends `inputProvenance.kind: "broadcast"`, but the current gateway provenance schema only accepts `external_user`, `inter_session`, and `internal_system`. Use `inter_session` with `sourceTool: "sessions_broadcast"` or add a documented protocol extension before sending these agent requests.
  - confidence: 0.94
- **[P1] Deny sessions_broadcast for subagent sessions:** `src/agents/openclaw-tools.ts:189-194`
  - body: Registering this tool in the default OpenClaw tool set exposes a new cross-session injection path to subagents, while current policy only always-denies `sessions_send`. Add `sessions_broadcast` to the subagent deny policy and cover it with regression tests.
  - confidence: 0.9
- **[P1] Skip thread-scoped sessions before broadcasting:** `src/agents/tools/sessions-broadcast-tool.ts:118`
  - body: The target loop pushes every visible session key, but `sessions_send` rejects thread-scoped chat sessions so inter-agent tool messages do not enter active human-facing threads. Apply the same thread-session guard before adding a broadcast target.
  - confidence: 0.88
- **[P2] Scope each broadcast lane to the target session:** `src/agents/tools/sessions-broadcast-tool.ts:140`
  - body: Each target send uses the shared `AGENT_LANE_NESTED` lane, which can reintroduce cross-session head-of-line blocking. Use `resolveNestedAgentLaneForSession(internalKey)` for each broadcast target.
  - confidence: 0.84
- **[P2] Import TypeBox from the current package:** `src/agents/tools/sessions-broadcast-tool.ts:2`
  - body: Current main imports TypeBox from `typebox` and the root package only declares `typebox`; this new `@sinclair/typebox` import can fail after merge. Match the surrounding tools and import `Type` from `typebox`.
  - confidence: 0.87
