## Summary

Keep open: current main does not implement `sessions_broadcast`, and the PR is not merge-ready because it introduces invalid Gateway provenance, an undeclared TypeBox import, and a new cross-session injection path without matching restricted-surface policy coverage.

## What This Changes

The PR adds a core `sessions_broadcast` agent tool, registers it in the catalog and default tool factory, and adds unit tests for channel-scoped broadcast delivery.

## Best Possible Solution

If maintainers want this in core, land it as a policy-aware sibling of `sessions_send`: use the declared `typebox` dependency, valid `inter_session` provenance, matching subagent and Gateway HTTP restrictions, focused regression tests, and docs/changelog for the new trust boundary.

## Reproduction Assessment

Yes. Source inspection is enough: the PR sends `inputProvenance.kind: "broadcast"` through the validated Gateway `agent` method, while current main only accepts `external_user`, `inter_session`, and `internal_system`; the policy gap is visible by comparing the new default registration with existing `sessions_send` deny surfaces.

## Solution Assessment

No. The branch is a useful implementation sketch and addressed the first automated feedback, but it is not the narrowest maintainable solution until the dependency, provenance, restricted-surface policy, and core trust-boundary questions are resolved.
