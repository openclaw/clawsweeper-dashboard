## Summary

Keep this PR open. The requested broadcast tool is still absent from current main and the linked feature request was consolidated onto this PR, but the patch still has blocking defects in the gateway provenance contract and subagent tool-policy boundary, and the external contributor has not provided real behavior proof.

## What This Changes

The PR adds a core `sessions_broadcast` agent tool, registers it in the tool catalog and OpenClaw tool set, and adds mocked tests for broadcasting a system event to active sessions.

## Best Possible Solution

Keep this PR as the canonical implementation candidate, but require valid existing or documented provenance semantics, subagent policy gating with tests, and redacted live proof that a broadcast reaches another real session before merge.

## Reproduction Assessment

Yes for the blocking defects by source inspection: the PR sends a provenance kind that current gateway schema rejects, and the current subagent policy would not deny the new broadcast tool. I did not run a live broadcast in this read-only review.

## Solution Assessment

No, not as currently patched. The maintainable path should reuse `inter_session` provenance or add a documented gateway contract, and it must preserve the existing subagent announce-chain boundary before live validation.
