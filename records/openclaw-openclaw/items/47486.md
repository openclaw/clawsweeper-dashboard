## Summary

Keep open. Current main still lacks this tool, but the PR head is not merge-ready because it adds a core cross-session injection primitive with invalid provenance, an undeclared schema import, and missing restricted-surface policy coverage.

## What This Changes

The PR adds a core `sessions_broadcast` agent tool, registers it in the catalog and default tool factory, and adds unit tests for broadcasting system-event messages to active sessions with optional channel filtering.

## Best Possible Solution

If maintainers want this capability in core, land it as a policy-aware sibling of `sessions_send`: use the current `typebox` dependency, valid provenance, matching subagent and Gateway HTTP restrictions, focused regression tests, and trust-boundary docs or changelog. If the desired product is operator notification rather than agent context injection, fold it into the admin-scoped design tracked in #53228.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: the PR sends `inputProvenance.kind: "broadcast"` through the validated Gateway `agent` method while current main only accepts three provenance kinds, and the policy gap is visible by comparing the new default tool registration with the existing `sessions_send` deny surfaces.

## Solution Assessment

No. The branch is a useful implementation sketch and it addressed the first automated feedback, but it is not the narrowest maintainable solution until the dependency, Gateway provenance, cross-session trust boundary, restricted-surface policy, and docs/changelog contracts are resolved.
