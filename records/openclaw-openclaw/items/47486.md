## What This Changes

Adds a core `sessions_broadcast` agent tool, registers it in the tool catalog/default tool set, and adds mocked tests for multi-session system-event broadcasts.

## Best Possible Solution

Land this only if maintainers accept the core broadcast feature and the implementation follows the existing provenance, thread-session, subagent-policy, lane, dependency, and real multi-session proof contracts.

## Reproduction Assessment

Yes for the PR review findings by source inspection: the latest head sends an unsupported provenance kind, can include thread sessions, exposes broadcast to subagent contexts, uses a shared nested lane, and imports an undeclared TypeBox package. I did not run a live broadcast because this review is read-only and the PR does not include real behavior proof.

## Solution Assessment

No, not as currently patched. The maintainable path is to reuse or formally extend the gateway provenance contract, preserve `sessions_send` safety boundaries, and prove the behavior in a real multi-session setup.
