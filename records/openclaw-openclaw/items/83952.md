## Summary

Keep this PR open for maintainer review and landing; current main still has the global WebChat file-input selector, while the branch applies a narrow scoped fix with focused coverage and sufficient browser-level proof, and I found no blocking correctness or security issue.

## What This Changes

This PR changes WebChat's Upload button to click the hidden file input inside its own composer, sets the control to `type="button"`, adds a focused attachment-picker test, and records a changelog entry.

## Best Possible Solution

Land the narrow WebChat UI fix after ordinary maintainer review and CI, keeping the existing attachment serialization and upload pipeline unchanged.

## Reproduction Assessment

Source-level yes: current main routes the visible Upload button through a document-wide `.agent-chat__file-input` lookup, so multiple or stale composers can target the wrong hidden input. I did not reproduce the exact Windows Chrome native-dialog failure locally.

## Solution Assessment

Yes: scoping from the clicked button to the nearest composer is the narrowest maintainable fix, and `type="button"` prevents accidental form submission without changing the upload pipeline.
