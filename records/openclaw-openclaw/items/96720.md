- **[P1] Gate bootstrap reads to explicit agentDir:** `src/agents/bootstrap-files.ts:328-329`
  - body: `resolveAgentDir()` falls back to the default per-agent state directory when no `agents.list[].agentDir` is configured. This new block would start treating bootstrap-named files in state as prompt instructions for any run with config plus agentId, so gate the load on an explicitly configured `agentDir`.
  - confidence: 0.92
- **[P1] Keep bootstrap hooks as the final override:** `src/agents/bootstrap-files.ts:335`
  - body: This block runs after `applyBootstrapHookOverrides()` has produced the effective bootstrap list, then replaces same-named files with agentDir content. That lets local agentDir files undo hook-provided safety or customization output; merge agentDir files before hook application so hooks remain the final override.
  - confidence: 0.9
- **[P2] Preserve BOOTSTRAP completion filtering:** `src/agents/bootstrap-files.ts:334-337`
  - body: `filterCompletedWorkspaceBootstrapFile()` compares `BOOTSTRAP.md` against the workspace root. After replacing a hook or workspace entry with an agentDir `BOOTSTRAP.md`, the stale first-run bootstrap file can remain injected after workspace setup completion.
  - confidence: 0.84
- **[P2] Align bootstrap diagnostics with agentId context:** `src/agents/bootstrap-files.ts:327-328`
  - body: AgentDir loading only happens when `params.agentId` is present, but bootstrap-size doctor callers still call this resolver with config only. If this lands, runtime sessions can inject or truncate agentDir files while diagnostics report workspace-only bootstrap sizes.
  - confidence: 0.82
