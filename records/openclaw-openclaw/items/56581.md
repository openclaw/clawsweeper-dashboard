Full review comments:

- **[P2] Gate profile sourcing to live runs:** `test/test-env.ts:338`
  - body: `installTestEnv()` now calls `loadProfileEnv(realHome)` for every test process before HOME isolation. Regular unit/integration runs can inherit local `~/.profile` variables such as provider keys, making tests non-hermetic and increasing accidental secret exposure; keep profile loading behind live-test or explicit real-home opt-in.
  - confidence: 0.93
- **[P2] Snapshot live env after profile import:** `test/test-env.ts:336`
  - body: `liveEnvSnapshot` is captured before `loadProfileEnv(realHome)`, so profile-defined `OPENCLAW_CONFIG_PATH` or `OPENCLAW_STATE_DIR` are missing when `stageLiveTestState()` decides which config/auth state to copy. Live tests can stage the wrong default config even though the profile was sourced.
  - confidence: 0.9
- **[P2] Honor real-home opt-in loaded from profile:** `test/test-env.ts:334`
  - body: `allowRealHome` is computed before the profile import, so `OPENCLAW_LIVE_USE_REAL_HOME=1` in `~/.profile` is ignored. That contradicts the new docs and silently keeps live runs on temp HOME unless the parent shell already exported the flag.
  - confidence: 0.9
