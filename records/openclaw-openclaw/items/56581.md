Full review comments:

- **[P2] Gate profile sourcing to live runs:** `test/test-env.ts:338`
  - body: `installTestEnv()` sources `~/.profile` for every test process before HOME isolation. Regular unit/integration runs can inherit local provider keys or tokens, making tests non-hermetic and increasing accidental secret exposure; keep profile loading behind live-test or explicit opt-in.
  - confidence: 0.93
- **[P2] Snapshot live env after profile import:** `test/test-env.ts:336`
  - body: `liveEnvSnapshot` is captured before `loadProfileEnv(realHome)`, so profile-defined `OPENCLAW_CONFIG_PATH` or `OPENCLAW_STATE_DIR` are missing when live state is staged. Live tests can copy the wrong config even after profile sourcing.
  - confidence: 0.9
- **[P2] Honor profile-loaded real-home opt-in:** `test/test-env.ts:334`
  - body: `allowRealHome` is computed before profile import, so `OPENCLAW_LIVE_USE_REAL_HOME=1` set in `~/.profile` is ignored. That contradicts the documented real-home opt-out unless the parent shell exported the flag first.
  - confidence: 0.9
