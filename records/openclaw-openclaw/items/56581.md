## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Gate profile sourcing to live runs:** `test/test-env.ts:338`
  - body: `installTestEnv()` now calls `loadProfileEnv(realHome)` for every test process before HOME isolation. Normal unit/integration tests can inherit local `~/.profile` variables such as provider keys, so profile loading should stay behind live-test or explicit real-home opt-in.
  - confidence: 0.93
- **[P2] Snapshot live env after loading the profile:** `test/test-env.ts:336`
  - body: `liveEnvSnapshot` is captured before `loadProfileEnv(realHome)`, so profile-defined `OPENCLAW_CONFIG_PATH` or `OPENCLAW_STATE_DIR` are missing when `stageLiveTestState()` chooses which config/auth state to copy. Live tests can stage the wrong default config even though the profile was sourced.
  - confidence: 0.9
- **[P2] Honor real-home opt-in loaded from profile:** `test/test-env.ts:334`
  - body: `allowRealHome` is computed before the profile import, but the docs say users can set `OPENCLAW_LIVE_USE_REAL_HOME=1` for live runs. If that flag lives in `~/.profile`, this code still forces temp-HOME isolation unless it was already exported by the parent shell.
  - confidence: 0.9
