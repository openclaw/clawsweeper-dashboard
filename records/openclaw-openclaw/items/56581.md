  - body: `installTestEnv()` calls `loadProfileEnv(realHome)` unconditionally, so ordinary unit and integration test runs can execute/source a developer profile and inherit local provider keys or tokens before HOME isolation. Keep profile loading behind live-test or explicit opt-in.
  - confidence: 0.93
- **[P2] Snapshot live env after profile import:** `test/test-env.ts:336`
  - body: `liveEnvSnapshot` is captured before `loadProfileEnv(realHome)`, so profile-defined `OPENCLAW_CONFIG_PATH` or `OPENCLAW_STATE_DIR` are absent when live state is staged. Live tests can copy the wrong config/auth location after profile sourcing.
  - confidence: 0.9
- **[P2] Honor profile-loaded real-home opt-in:** `test/test-env.ts:334`
  - body: `allowRealHome` is computed before the profile import, so `OPENCLAW_LIVE_USE_REAL_HOME=1` set in `~/.profile` is ignored. Recompute this after loading the profile so the documented real-home opt-in works.
  - confidence: 0.9
