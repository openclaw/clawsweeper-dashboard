Keep open: current main still only blocks or skips unauthorized WhatsApp group traffic, and the requested auto-leave setting, group-join event/hook, and WhatsApp leave-group action are not implemented.

## What This Changes

Requests WhatsApp plugin support to auto-leave unauthorized groups, expose a group-add hook/event, and add a manual leave-group action.

## Best Possible Solution

Define an opt-in WhatsApp plugin-owned feature with config/schema/docs, verified group-add detection and group leave behavior, explicit target authorization, owner notification, and regression coverage coordinated with the broader group-pairing design.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high confidence the capability is absent: existing WhatsApp allowlist paths block or skip traffic, and action discovery/runtime expose no leave-group action.

## Solution Assessment

Unclear until maintainers settle the product and security model. The maintainable direction is WhatsApp plugin-owned opt-in behavior with explicit authorization and owner notification, not hidden core default behavior.
