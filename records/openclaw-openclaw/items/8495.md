## Summary

Keep open: current main and the latest release still block or skip unauthorized WhatsApp group traffic, but they do not provide opt-in auto-leave, a WhatsApp group-join hook/event, or a WhatsApp leave-group message action. The related broader group-pairing issue remains open design context, not a merged or shipped replacement.

## What This Changes

The issue requests opt-in WhatsApp behavior to leave unauthorized groups, emit a group-join hook/event, and expose a manual leave action.

## Best Possible Solution

Design this as an opt-in WhatsApp plugin feature with schema/docs, authorized Baileys-backed groupLeave behavior, join or participant-update handling, minimal metadata exposure, focused regression tests, and coordination with https://github.com/openclaw/openclaw/issues/11489.
