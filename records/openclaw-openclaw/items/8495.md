## Summary

Keep open: current main still enforces WhatsApp group allowlists by blocking or skipping unauthorized traffic, but it does not implement opt-in auto-leave, a group-join hook/event, or a WhatsApp leave-group message action; related #11489 is broader and remains open.

## What This Changes

The issue requests opt-in WhatsApp auto-leave for unauthorized groups, a group-join hook/event, and a manual WhatsApp leave-group action.

## Best Possible Solution

Define this as an opt-in bundled WhatsApp plugin feature with config/schema/docs, verified leave and join detection paths, explicit authorization and notification semantics, regression coverage, and coordination with #11489.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main shows existing allowlist paths block or skip unauthorized WhatsApp group traffic without auto-leave, group-join events, or a leave-group action.

## Solution Assessment

Unclear until maintainers choose the product/security shape. The narrow maintainable direction is a WhatsApp plugin implementation with explicit opt-in configuration and authorization, not hidden core behavior.
