## Summary

Keep open: current main and v2026.5.12 still only block or skip unauthorized WhatsApp group traffic and expose no auto-leave setting, group-join event, or WhatsApp leave-group action. The request is a new security-sensitive product feature, not conservative cleanup.

## What This Changes

Request an opt-in WhatsApp auto-leave path for unauthorized groups, a group-join hook/event, and a manual WhatsApp group leave message action.

## Best Possible Solution

Design this as an opt-in WhatsApp plugin feature with config/schema/docs, authorized Baileys group-leave behavior, join/participant update detection or owner notification, and focused tests, coordinated with https://github.com/openclaw/openclaw/issues/11489.

Not applicable: this is a feature request, not a broken current contract. Current-main source, existing tests, docs, live issue context, and a v2026.5.12 spot check give a high-confidence verification path that the requested surface is absent.

## Solution Assessment

Unclear until maintainers settle the product and security model. Auto-leave, group-join notifications, and a manual leave action should be scoped together enough to avoid inconsistent authorization and privacy behavior.
