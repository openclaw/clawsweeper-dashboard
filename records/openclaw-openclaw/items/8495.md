## Summary

Keep open. Current main still blocks or skips unauthorized WhatsApp group traffic, but it does not implement opt-in auto-leave, a WhatsApp group-join hook/event, or a WhatsApp leave-group message action; related #11489 remains open and broader rather than superseding this WhatsApp-specific request.

## What This Changes

Request opt-in WhatsApp auto-leave for unauthorized groups, a group-join notification/hook, and a manual leave-group message action.

## Best Possible Solution

Implement this deliberately in the bundled WhatsApp plugin with opt-in config/schema/docs, authorized group-leave behavior, group-add or participant-update detection with owner notification semantics, regression tests, and coordination with #11489.

## Reproduction Assessment

Not applicable as a feature request, but current main has a high-confidence source-level verification path. The config/schema, inbound access control, group gating, hooks, docs, and message-action support all show block/skip behavior without auto-leave or group-join handling.

## Solution Assessment

Unclear until maintainers define the product semantics. The narrow maintainable direction is WhatsApp-plugin-owned opt-in behavior with explicit authorization, tests, and docs rather than a generic core policy first.
