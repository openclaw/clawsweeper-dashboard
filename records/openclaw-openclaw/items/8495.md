## Label Justifications

- `P2`: This is a meaningful WhatsApp security and privacy enhancement, but existing gating already prevents unauthorized group messages from starting agent work.
- `impact:security`: The issue concerns unauthorized group presence, reconnaissance exposure, and authorization for a destructive group-membership action.

## Summary

Current main and v2026.6.5 still only suppress unauthorized WhatsApp group traffic; they do not leave unadmitted groups, emit a group-add event, or expose a leave-group action. The request remains useful and is not superseded by the broader open group-pairing issue, but its original sender-allowlist trigger is not the correct group-admission boundary and the destructive action needs explicit product and security decisions.

## What This Changes

Add opt-in WhatsApp auto-leave for unapproved groups, a group-join notification/event, and an authorized manual leave-group action.

## Best Possible Solution

Keep the capability in the bundled WhatsApp plugin: define a separate opt-in response for chat IDs rejected by the groups admission policy, emit an owner-visible join/admission event aligned with the shared group-pairing design, and expose group departure only through a narrowly authorized account-and-target operation with live WhatsApp validation.

## Reproduction Assessment

Not applicable: source, docs, and the latest release confirm that dropping unauthorized group traffic is current intended behavior; the issue asks for new membership-management capabilities rather than reporting a broken existing contract.

## Solution Assessment

No, not exactly as proposed. Auto-leave should be driven by the chat-level groups admission decision rather than groupAllowFrom, and any manual leave action needs a narrower authorization and notification design coordinated with the broader group-pairing work.
