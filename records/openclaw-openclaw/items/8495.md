## Summary

Keep open: current main still blocks or skips unauthorized WhatsApp group traffic, but it does not implement opt-in auto-leave, a group-join hook/event, or a WhatsApp leave-group action; related #11489 is broader and remains open.

## What This Changes

Request opt-in WhatsApp auto-leave for non-allowlisted groups, group-join notification/event hooks, and a manual WhatsApp leave action.

## Best Possible Solution

Define this as an opt-in bundled WhatsApp plugin feature with schema/docs, verified Baileys leave support, join/participant-update detection, explicit authorization and owner-notification semantics, regression coverage, and coordination with #11489.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main shows the existing allowlist paths block or skip unauthorized WhatsApp group traffic without auto-leave, group-join events, or a leave-group action.

## Solution Assessment

Unclear until maintainers choose the product/security shape. The narrow maintainable direction is a WhatsApp plugin implementation with explicit opt-in configuration and authorization, not hidden core behavior.
