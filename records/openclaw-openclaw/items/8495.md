## Summary

Keep open: current main still blocks or skips unauthorized WhatsApp group traffic but does not implement opt-in auto-leave, a WhatsApp group-join hook/system event, or a WhatsApp leave-group message action. The prior maintainer-side review also kept this open, and related #11489 is broader and still open rather than a shipped superseding fix.

## What This Changes

Request opt-in WhatsApp behavior to leave unauthorized groups, surface group-join events/hooks, and optionally add a leave-group message action.

## Best Possible Solution

Keep this as a WhatsApp-owned, opt-in feature: define the policy/event/action shape first, then add schema, docs, runtime handling, and focused tests around verified group-leave and group-join behavior.

## Reproduction Assessment

Not applicable for a feature request. Current source and docs give high-confidence evidence of the present behavior: allowlists block or skip group traffic, while no config, hook, event listener, or WhatsApp leave action exists.

## Solution Assessment

Unclear until maintainers choose the product shape. The narrow maintainable direction is a WhatsApp plugin implementation coordinated with #11489, not a generic core migration or an unconditional auto-leave side effect.
