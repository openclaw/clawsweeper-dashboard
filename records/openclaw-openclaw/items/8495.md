## Summary

Keep open: current main and v2026.5.22 still block or drop unauthorized WhatsApp group traffic, but they do not implement opt-in auto-leave, a group-join hook/event, or a WhatsApp leave-group message action. Prior maintainer-side comments reached the same conclusion, and the broader group-pairing request at https://github.com/openclaw/openclaw/issues/11489 remains open rather than superseding this WhatsApp-specific feature.

## What This Changes

Requested an opt-in WhatsApp feature to auto-leave unauthorized groups, emit a group-join hook/event, and support a manual leave-group action.

## Best Possible Solution

Design this as an explicit WhatsApp plugin feature with opt-in schema/docs, authorized Baileys groupLeave wiring, minimal group-add metadata handling, focused tests, and coordination with https://github.com/openclaw/openclaw/issues/11489.
