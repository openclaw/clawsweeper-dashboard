## Summary

Keep open: current main still blocks or skips unauthorized WhatsApp group traffic, but it does not implement opt-in auto-leave, a WhatsApp group-join hook/system event, or a WhatsApp leave-group message action. The related broader group-pairing request #11489 remains open and does not supersede this WhatsApp-specific feature request.

## What This Changes

The issue requests an opt-in WhatsApp setting to leave unauthorized groups, a group-join webhook/system event, and a manual leave-group message action.

## Best Possible Solution

Keep this as a deliberate WhatsApp plugin feature: define the opt-in policy/event/action semantics first, then add schema, docs, runtime handling, and focused tests for verified group-leave and group-join behavior.

## Reproduction Assessment

Not applicable for a feature request. Current source and docs give high-confidence evidence of the present behavior: allowlists block or skip group traffic, while no auto-leave config, group-join listener, or WhatsApp leave action exists.

## Solution Assessment

Unclear until maintainers choose the product shape. The narrow maintainable direction is a WhatsApp-owned opt-in implementation coordinated with #11489, not a generic core migration or unconditional auto-leave side effect.
