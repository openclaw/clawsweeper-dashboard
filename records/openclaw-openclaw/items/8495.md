The broader group-pairing issue at https://github.com/openclaw/openclaw/issues/11489 and the Telegram-only PR at https://github.com/openclaw/openclaw/pull/50682 are useful design context, not replacements for this WhatsApp-specific request.

## What This Changes

Add opt-in WhatsApp auto-leave for unauthorized groups, a group-join webhook/event, and a manual leave action.

## Best Possible Solution

Design this as an opt-in WhatsApp plugin feature with schema/docs, authorized group-leave behavior, join/participant event handling or an equivalent add-detection path, minimal metadata exposure, focused tests, and coordination with https://github.com/openclaw/openclaw/issues/11489.
