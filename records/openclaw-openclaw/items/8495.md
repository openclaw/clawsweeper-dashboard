## Summary

Keep open: current main and the latest release still block or skip unauthorized WhatsApp group traffic, but they do not provide an opt-in auto-leave setting, a group-join event/hook, or a WhatsApp leave-group message action. The related broader group-pairing request remains open and overlaps the design, but it does not supersede this WhatsApp-specific security feature.

## What This Changes

Request an opt-in WhatsApp feature to leave unauthorized groups, expose group-join handling, and add a manual leave action.

## Best Possible Solution

Design this as an opt-in WhatsApp plugin feature with schema/docs, authorized Baileys-backed groupLeave behavior, join or participant-update handling, minimal metadata exposure, focused regression tests, and coordination with https://github.com/openclaw/openclaw/issues/11489.
