## Summary

Keep open: current main and the latest shipped release still block or skip unauthorized WhatsApp group traffic, but they do not implement opt-in auto-leave, a group-join hook/event, or a WhatsApp leave-group message action. The related group-pairing issue and WhatsApp admin PR are adjacent, but both remain open and neither supersedes this WhatsApp-specific feature.

## What This Changes

Request to add opt-in WhatsApp auto-leave for unauthorized groups, a group-join hook/event, and a manual leave-group message action.

## Best Possible Solution

Define an explicit WhatsApp plugin-owned opt-in feature with schema/docs, Baileys group-participant join detection, authorized groupLeave behavior, owner notification or hook payload minimization, regression tests, and coordination with https://github.com/openclaw/openclaw/issues/11489.

## Reproduction Assessment

Not applicable as a feature request. Current-main source inspection gives a high-confidence verification path that the requested capability is absent: allowlist paths block or skip traffic, action discovery has no leave action, and no group-join event path is wired.

## Solution Assessment

Unclear until maintainers settle the product and security shape. The maintainable direction is an explicit, opt-in WhatsApp plugin feature with authorization and owner notification, not hidden default leave behavior inside the existing allowlist checks.
