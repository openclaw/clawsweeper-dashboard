## Summary

Keep open: current main still enforces WhatsApp group allowlists by blocking or skipping unauthorized traffic, but it does not implement opt-in auto-leave, a group-join event/hook, or a WhatsApp leave-group message action. The related group-pairing request remains broader and open, so it does not supersede this WhatsApp-specific feature.

## What This Changes

The issue requests opt-in WhatsApp auto-leave for groups outside the allowlist, a group-join hook/event, and a manual leave action.

## Best Possible Solution

Define this as a deliberate bundled WhatsApp plugin feature with opt-in config/schema/docs, a verified leave operation, explicit authorization and notification semantics, regression coverage for denied-group paths, and coordination with the broader group-pairing design in #11489.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main blocks or skips unauthorized WhatsApp group traffic but does not implement auto-leave, group-join events, or a leave-group message action.

## Solution Assessment
