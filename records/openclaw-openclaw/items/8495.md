## Summary

Keep open: current main still blocks or skips unauthorized WhatsApp group traffic, but it does not add opt-in auto-leave, a WhatsApp group-join event/hook, or a WhatsApp leave-group message action. Related #11489 is broader and still open, so it does not supersede this WhatsApp-specific request.

## What This Changes

Add opt-in WhatsApp auto-leave for unauthorized groups, a group-join webhook/event, and a manual leave action.

## Best Possible Solution

Define this as a deliberate bundled WhatsApp plugin feature: add opt-in config/schema/docs, a verified Baileys-backed leave operation, group-add detection or notification semantics, focused tests, and alignment with the broader group-pairing design in #11489.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main blocks/skips unauthorized WhatsApp group messages but does not implement the requested auto-leave, group-join event, or leave action.

## Solution Assessment

Unclear until maintainers choose the product/security shape. The narrow maintainable path is a WhatsApp plugin implementation with explicit opt-in configuration and tests, not a generic hidden core behavior change.
