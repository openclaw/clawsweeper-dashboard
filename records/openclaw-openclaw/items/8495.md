## Summary

Keep open. Current main still blocks or skips unauthorized WhatsApp group traffic but does not implement opt-in auto-leave, a WhatsApp group-join hook/event, or a WhatsApp leave-group message action; the prior maintainer-side Codex review also kept this open, and broader related #11489 is still open rather than superseding the WhatsApp-specific request.

## What This Changes

The issue requests WhatsApp support for automatically leaving non-allowlisted groups, emitting a group-join hook/event, and exposing a manual leave-group message action.

## Best Possible Solution

Keep this open for deliberate bundled WhatsApp plugin work: add opt-in config/schema/docs for unauthorized-group auto-leave, implement an account- and target-authorized Baileys group leave operation, add group-add or participant-update detection with owner notification/event semantics, consider a WhatsApp leaveGroup message action, and coordinate the event shape with #11489.

## Reproduction Assessment

Not applicable as a feature request, but we do have a high-confidence source-level verification path for current behavior. Current main’s config/schema, group gating, inbound monitor, docs, and tests show block/skip behavior without auto-leave, group-join events, or a WhatsApp leaveGroup action.

## Solution Assessment

Unclear until maintainers settle the product semantics. The narrow maintainable direction is to implement this inside the bundled WhatsApp plugin with opt-in config, explicit account/target authorization, tests, docs, and coordination with #11489 rather than closing it or adding a generic core policy first.
