## Summary

Keep open: current main and the latest release still only block or drop unauthorized WhatsApp group traffic; they do not auto-leave the group, emit a WhatsApp group-join event, or expose a WhatsApp leave-group action. The request is valid but needs maintainer product/security decisions before implementation.

## What This Changes

Request auto-leaving unauthorized WhatsApp groups, a group-join hook/event, and a manual WhatsApp leave-group message action.

## Best Possible Solution

Keep this open for a deliberate bundled WhatsApp plugin design: add an opt-in leave-on-chat-admission-denial path, wire Baileys group-add/participant events into an owner-visible notification contract, and expose any leave action only behind narrow authorization and live WhatsApp validation.

## Reproduction Assessment

Not applicable: this is a feature request rather than a report of broken existing behavior. Source, docs, tests, and v2026.6.6 confirm the current behavior is to block/drop unauthorized group traffic, not leave groups or emit join events.

## Solution Assessment

No, not exactly as proposed. The safer solution should key auto-leave off chat-level group admission, not groupAllowFrom alone, and should coordinate destructive leave authorization plus join notifications with the broader group-pairing design.
