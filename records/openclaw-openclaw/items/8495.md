## Summary

Keep open: current main still only blocks or skips unauthorized WhatsApp group traffic and lacks the requested opt-in auto-leave setting, group-join event, and WhatsApp leave-group action.

## What This Changes

Request an opt-in WhatsApp feature to auto-leave unauthorized groups, emit a group-join hook or system event, and expose a manual leave action.

## Best Possible Solution

Define an opt-in WhatsApp plugin feature with config/schema/docs, verified group leave support, join or participant-update detection, explicit authorization and owner-notification semantics, regression coverage, and coordination with https://github.com/openclaw/openclaw/issues/11489 and related cross-channel auto-leave work.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high confidence the requested capability is absent: existing WhatsApp allowlist paths block or skip traffic without auto-leave, group-join events, or a leave-group action.

## Solution Assessment

Unclear until maintainers choose the product/security shape. The narrow maintainable direction is WhatsApp plugin-owned opt-in behavior with explicit authorization and owner notification, not hidden core default behavior.
