## Summary

Keep open: current main still blocks or skips unauthorized WhatsApp group traffic, but it does not implement opt-in auto-leave, a WhatsApp group-join hook/event, or a WhatsApp leave-group message action. The related group-pairing issue and group-admin PR are adjacent, but neither is merged or a replacement for this WhatsApp-specific feature.

## What This Changes

Add opt-in WhatsApp auto-leave for unauthorized groups, a group-join hook/event, and a manual leave-group action.

## Best Possible Solution

Define a WhatsApp plugin-owned opt-in design with schema/docs, verified group-add detection, authorized leave behavior, owner notification, and focused regression coverage coordinated with the broader group-pairing request.

## Reproduction Assessment

Not applicable as a feature request. Current-main source inspection gives a high-confidence verification path that the requested capability is absent: allowlist paths block or skip traffic, action discovery has no leave action, and no group-join event path is wired.

## Solution Assessment

Unclear until maintainers settle the product and security shape. The maintainable direction is an explicit, opt-in WhatsApp plugin feature with authorization and owner notification, not hidden default leave behavior inside the existing allowlist checks.
