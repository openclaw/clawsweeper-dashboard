## Summary

Keep open: current main and v2026.6.10 still only log, block, or drop unauthorized WhatsApp group traffic; they do not auto-leave groups, emit a WhatsApp group-join hook/event, or expose a WhatsApp leave action. Related cross-channel and Telegram work overlaps but does not supersede this WhatsApp-specific security feature.

## What This Changes

Request opt-in WhatsApp auto-leave for unauthorized groups, a group-add webhook/event, and a manual WhatsApp leave action.

## Best Possible Solution

Keep this as a deliberate WhatsApp plugin feature: define opt-in auto-leave keyed to group admission, owner-visible group-add notification semantics, and an authorized leave action aligned with https://github.com/openclaw/openclaw/issues/11489.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high-confidence current behavior: unauthorized WhatsApp group senders or unregistered groups are blocked/dropped without leave, group-join hook, or leave action.

## Solution Assessment

No, not exactly as proposed. The safer solution should separate chat-level group admission from sender allowlists and coordinate destructive leave authorization plus group-add notification with the broader group-pairing design.
