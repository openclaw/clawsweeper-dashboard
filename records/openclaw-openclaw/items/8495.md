## Summary

Keep open: current main and the latest release still only block or skip unauthorized WhatsApp group traffic. The requested opt-in auto-leave, group-join event or hook, and WhatsApp leave-group message action are not implemented, and the related open items are adjacent rather than replacements.

## What This Changes

Request an opt-in WhatsApp feature to leave unauthorized groups, notify or hook on group joins, and expose a manual leave-group message action.

## Best Possible Solution

Define a WhatsApp plugin-owned opt-in design with schema/docs, verified group-add detection, authorized leave behavior, owner notification or hook semantics, and focused regression coverage coordinated with the broader group-pairing and hooks-only requests.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainers settle the product and security shape. The maintainable direction is an explicit opt-in WhatsApp plugin feature with authorization and owner notification, not hidden default leave behavior inside the existing allowlist checks.
