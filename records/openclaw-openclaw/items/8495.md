## Summary

Keep open: current main and the latest release still block or skip unauthorized WhatsApp group traffic, but they do not add an opt-in auto-leave setting, a group-join event/hook, or a WhatsApp leave-group message action. The related broader group approval request at https://github.com/openclaw/openclaw/issues/11489 remains open and overlaps notification/approval design without superseding this WhatsApp-specific request.

## What This Changes

The issue requests opt-in WhatsApp auto-leave for groups outside the allowlist, plus a group-join notification/hook and a manual leave action.

## Best Possible Solution

Design this as an opt-in WhatsApp plugin feature with config/schema/docs, authorized Baileys-backed groupLeave behavior, group-add or participant-update detection, minimal join metadata, focused tests, and coordination with https://github.com/openclaw/openclaw/issues/11489.
