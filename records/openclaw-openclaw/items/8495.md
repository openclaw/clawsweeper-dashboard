## Summary

Keep open: current main and v2026.6.6 still block or skip unauthorized WhatsApp group traffic, but they do not auto-leave unadmitted groups, emit a WhatsApp group-join event, or expose a WhatsApp leave-group action.

## What This Changes

The issue requests opt-in WhatsApp behavior to leave unauthorized groups, emit group-join hooks/events, and expose a manual leave action.

## Best Possible Solution

Keep this open for maintainer product/security design in the bundled WhatsApp plugin: make unauthorized-group departure opt-in, base it on chat admission rather than sender allowlists, and coordinate any owner notification with the broader group-pairing contract.

## Reproduction Assessment

Not applicable: current source and docs already show the existing behavior is to deny or drop unauthorized group traffic, while the issue asks for new membership-management capability.

## Solution Assessment

No, not exactly as proposed. The safer design is to drive auto-leave from chat-level group admission through channels.whatsapp.groups, not from groupAllowFrom sender authorization, and to coordinate group-add notification with the broader group-pairing work.
