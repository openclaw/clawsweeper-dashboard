## Summary

Keep open: current main and v2026.5.22 still block or drop unauthorized WhatsApp group traffic, but do not add opt-in auto-leave, a group-join hook/event, or a WhatsApp leave-group action. The request is security-sensitive and overlaps the broader, still-open group-pairing work at https://github.com/openclaw/openclaw/issues/11489, so conservative cleanup should not close it.

## What This Changes

The issue requests WhatsApp allowlist behavior that can auto-leave unauthorized groups, emit a group-join hook/event, and expose a manual leave-group message action.

## Best Possible Solution

Design this as an explicit WhatsApp plugin feature with opt-in schema/docs, authorized group-leave behavior, group-add event semantics with minimal metadata, focused tests, and coordination with https://github.com/openclaw/openclaw/issues/11489.
