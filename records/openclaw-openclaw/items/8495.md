## Summary

Keep open: current main and v2026.6.6 still block or drop unauthorized WhatsApp group traffic, but they do not auto-leave the group, emit a WhatsApp group-join event, or expose a WhatsApp leave-group action. The request is valid and security-relevant, but it needs maintainer product/security decisions before implementation.

## What This Changes

Request for WhatsApp to auto-leave unauthorized groups, expose a group-join hook/event, and add a manual leave action.

## Best Possible Solution

Keep this open for a deliberate WhatsApp plugin design that keys auto-leave to chat-level group admission, exposes owner-visible group-add notification semantics, and gates any leave action behind narrow authorization plus live WhatsApp proof.

## Reproduction Assessment

Not applicable: this is a feature request rather than a report of broken existing behavior. Source, docs, tests, and v2026.6.6 show current behavior is to block/drop unauthorized WhatsApp group traffic without leaving the group or emitting a join event.

## Solution Assessment

No, not exactly as proposed. The safer solution should separate chat-level group admission from sender allowlists and coordinate group-add notification plus destructive leave authorization with the broader group-pairing design.
