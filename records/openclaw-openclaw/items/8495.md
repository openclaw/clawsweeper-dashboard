## Summary

Keep open: current main and the latest release still only block or drop unauthorized WhatsApp group traffic; they do not auto-leave groups, emit a WhatsApp group-join event, or expose a WhatsApp leave action. The request remains useful, security-relevant, and needs maintainer product/security decisions before implementation.

## What This Changes

Request opt-in WhatsApp auto-leave for unapproved groups, a group-join event or hook, and a manual WhatsApp leave action.

## Best Possible Solution

Keep this as a deliberate WhatsApp plugin feature: add opt-in auto-leave keyed to group admission, define owner-visible group-add notification semantics, and gate any leave action behind narrow authorization plus live WhatsApp proof while coordinating with https://github.com/openclaw/openclaw/issues/11489.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source and release inspection show current behavior is to block/drop unauthorized WhatsApp group traffic without leaving the group or emitting a join event.

## Solution Assessment

No, not exactly as proposed. The safer solution should separate chat-level group admission from sender allowlists and coordinate group-add notification plus destructive leave authorization with the broader group-pairing design.
