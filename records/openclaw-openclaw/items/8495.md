## Summary

Keep open: current main and v2026.6.10 still block or drop unauthorized WhatsApp group traffic without leaving the group, emitting a group-join hook/event, or exposing a WhatsApp leave action. Related group-pairing, Telegram unknown-group, WhatsApp listen-only, and WhatsApp admin-privacy work overlaps but does not supersede this WhatsApp-specific security feature request.

## What This Changes

Request adds opt-in WhatsApp unauthorized-group auto-leave, a group-add hook/event, and a manual WhatsApp leave action for preapproved group deployments.

## Best Possible Solution

Keep this open for a deliberate WhatsApp plugin feature that ties opt-in auto-leave to chat admission, adds an owner-visible group-add event or hook, and exposes any leave action only behind explicit authorization, schema/docs, tests, and live WhatsApp proof.

## Reproduction Assessment

Yes at source level, not as a bug reproduction: current main blocks or drops unauthorized WhatsApp group traffic without leaving, notifying, or exposing a leave action. The same absence is visible in v2026.6.10.

## Solution Assessment

No, not exactly as proposed. The maintainable path should distinguish chat-level group admission from sender allowlists and settle destructive leave authorization plus owner notification before adding config or message actions.
