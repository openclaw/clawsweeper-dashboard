## Summary

Keep open: current main and the latest release still only block/drop unauthorized WhatsApp group traffic; they do not auto-leave groups, emit a group-join event, or expose a WhatsApp leave action. The request is security-relevant, but it needs maintainer product/security decisions before implementation.

## What This Changes

Request an opt-in WhatsApp unauthorized-group auto-leave mode, group-join notification/hook, and manual group leave action.

## Best Possible Solution

Keep this open for a deliberate WhatsApp plugin feature that keys auto-leave to chat-level admission, defines owner-visible group-add notification semantics, and gates any leave action behind narrow authorization plus live WhatsApp proof.

## Reproduction Assessment

Not applicable: this is a feature request rather than a broken-behavior report. Source, tests, docs, and v2026.6.6 show current behavior is to block/drop unauthorized WhatsApp group traffic without leaving the group or emitting a join event.

## Solution Assessment

No, not exactly as proposed. The safer solution should separate chat-level group admission from sender allowlists and coordinate group-add notification plus destructive leave authorization with the broader group-pairing design.
