## Summary

Keep open: current main still blocks or skips unauthorized WhatsApp group traffic and does not implement opt-in auto-leave, a WhatsApp group-join hook/event, or a WhatsApp leave-group message action. Related group-pairing and hook/listen-only requests remain open and adjacent, but none is a merged or shipped replacement for this WhatsApp-specific feature.

## What This Changes

The issue requests opt-in WhatsApp behavior to leave unauthorized groups, emit a group-join hook/event, and expose a manual group-leave action.

## Best Possible Solution

Define a WhatsApp plugin-owned opt-in design with schema/docs, verified Baileys group-add detection, authorized leave behavior, owner notification, and focused regression coverage coordinated with the broader group-pairing and hook/listen-only requests.

## Reproduction Assessment

Not applicable as a feature request. Current-main source inspection gives high confidence the requested capability is absent: WhatsApp allowlist paths block or skip traffic without auto-leave, group-join events, or a leave-group action.

## Solution Assessment

Unclear until maintainers settle the product and security shape. The maintainable direction is an explicit WhatsApp plugin-owned opt-in feature with authorization and owner notification, not hidden default leave behavior in existing allowlist checks.
