## Summary

Keep open for maintainer review. The patch looks correct after the latest follow-up, but it changes the source-of-truth for a generated diffs viewer runtime and the PR already carries maintainer/protected handling plus build/packaging merge risk.

## What This Changes

The PR stops tracking `extensions/diffs/assets/viewer-runtime.js`, ignores it as generated output, and updates diffs tests to generate the runtime fixture when a clean checkout lacks it.

## Best Possible Solution

Land only after maintainers accept the generated runtime as build-owned output and keep the generator, package build hook, source-test fixture path, and packaging checks aligned.

## Reproduction Assessment

Not applicable: this is a cleanup PR, not a reported runtime bug. The relevant check is whether clean-checkout build/test/package paths regenerate the ignored runtime, and the PR body lists focused local commands for that path.

## Solution Assessment

Yes, the direction is maintainable if maintainers want the minified viewer runtime to be generated output. The safer merge posture is to keep the build/package proof as the gate because the runtime route still depends on that generated file existing.
