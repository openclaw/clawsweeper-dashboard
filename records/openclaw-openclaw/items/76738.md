## Summary

Keep open: this protected maintainer-labeled PR still has a blocking correctness issue in the Discord retargeting path. The latest head resolves channel aliases, but it still moves status reactions off the inbound message before proving the tracked message can accept reactions.

## What This Changes

The PR adds `trackToolCalls`/`track_tool_calls` to reaction tool schemas and docs, forwards sanitized tool-start args, retargets Discord status reactions to an opted-in reaction target, updates status-reaction tool emoji resolution, tests, snapshots, and the changelog.

## Best Possible Solution

Keep the opt-in schema and emoji-display direction, but make Discord retain or restore the inbound status controller until the tracked target has accepted a reaction, with regression coverage for failed target or permission cases.

## Reproduction Assessment

Yes for the review finding: source inspection of the PR head shows the controller is swapped and the old controller cleared before a tracked reaction succeeds. A focused test can make the tracked target reaction fail and assert progress/done reactions stay on the inbound message.

## Solution Assessment

No: the feature direction is reasonable, but the current patch still swaps controllers before proving the new Discord target works. The narrower maintainable fix is to defer the swap until a successful tracked reaction or restore the original controller on first tracked-target failure.
