## Summary

Keep open. Current main still exposes node commands as string allowlist entries only, with no per-command input/output schema declaration, discovery surface, or command-specific invoke validation. Related #8287 is adjacent node-tool registration work, but it does not supersede this narrower gateway command-schema protocol gap.

## What This Changes

The issue requests a gateway protocol mechanism for nodes to declare per-command input and output schemas for custom node commands.

## Best Possible Solution

Define an additive command descriptor contract that preserves `commands: string[]` compatibility while adding optional per-command schema metadata, then carry it through connect, pairing persistence, node catalog/list/describe, generated protocol clients, node-host/plugin APIs, docs, and command-specific `node.invoke` input/output validation.
