## Summary

Close as duplicate/superseded: current main still models node commands as names without input/output schemas, and the open #8287 thread already tracks the broader node-registered tools design with parameter schemas and direct `node.invoke` routing.

## What This Changes

Nodes should be able to declare input/output schemas for custom commands so the gateway can validate calls, route them, and expose proper APIs to callers.

## Best Possible Solution

Keep #8287 as the canonical design thread for a versioned node tool/schema advertisement protocol and close this narrower duplicate.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection confirms the current node protocol only carries command names plus raw `unknown` invoke payloads.

## Solution Assessment

Yes. Consolidating into #8287 is the better path because command schemas are part of the larger node-registered tool contract, including naming, parameter schemas, policy, lifecycle, and routing.
