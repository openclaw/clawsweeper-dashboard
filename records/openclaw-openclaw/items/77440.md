## Summary

Keep open: current main still builds the WebChat Default label from global session defaults, and an open draft PR explicitly closes this issue, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

WebChat should display the active agent's effective default model in the Default option instead of always using the global default model.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/77690, or an equivalent narrow fix, so the selector resolves the active agent/session default before falling back to global defaults and keeps focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main renders defaultLabel directly while resolveDefaultModelValue builds it from global sessionsResult.defaults, despite gateway rows carrying agent-scoped model identity.

## Solution Assessment

Yes. The maintainable fix belongs in the selector-state resolution boundary so the UI uses the active agent or session row before global fallback; the open draft PR is already aimed at that boundary.
