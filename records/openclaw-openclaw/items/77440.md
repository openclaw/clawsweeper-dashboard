## Summary

Keep open: current main still has the WebChat selector-state bug, and an open draft PR already references this issue with closing syntax, so the issue should remain open until that PR is reviewed, proven, and merged or closed.

## What This Changes

The issue asks WebChat to show the active agent's effective default model in the model selector instead of always labeling `default` with the global default model.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/77690, or an equivalent narrow fix, so the default label resolves from the active agent/session model before falling back to global defaults and has focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: default-selected sessions render `defaultLabel`, and current main builds that label from global `sessionsResult.defaults` even though active session rows expose an agent-scoped model.

## Solution Assessment

Yes. The narrow maintainable solution is to resolve the selector label from the active agent or active session row before falling back to global defaults; the open draft PR is already pointed at that boundary.
