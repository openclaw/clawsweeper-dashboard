## Summary

Keep open: current main still resolves the chat model selector's `Default (...)` label from global session defaults, while the active session row can already expose the agent-scoped effective model.

## What This Changes

The issue asks the WebChat/dashboard model selector to label the default option with the current agent's effective default model instead of the global default.

## Best Possible Solution

Derive the model `Default (...)` label from the active session row's effective model when present, fall back to global defaults only when the row is absent, and add focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a session row can expose an agent-scoped `model`/`modelProvider`, but the model default label is still formatted from `sessionsResult.defaults`; I did not run a live UI session because this review is read-only.

## Solution Assessment

Yes. A narrow Control UI selector-state fix is the best path because backend/session row model resolution already accounts for the active agent, and no new config or product decision is needed.
