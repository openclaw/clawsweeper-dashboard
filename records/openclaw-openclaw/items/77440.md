## Summary

Keep open: current main still has a source-reproducible Control UI bug where the chat model selector's `Default (...)` label is derived from global session defaults instead of the active agent/session effective default.

## What This Changes

Fix the WebChat model selector so the `Default` option label reflects the current agent's effective default model instead of always using the global default.

## Best Possible Solution

Derive the selector's `Default (...)` label from the active session row's effective model when available, fall back to global defaults only when needed, and add focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with global default A, active agent/session row model B, and `chatModelOverrides[sessionKey] === null`, the resolver selects the empty default option but formats it from `sessionsResult.defaults`, producing `Default (A)`.

## Solution Assessment

Yes. A narrow Control UI selector-state fix is the maintainable path because backend model resolution and session-row construction already account for the current agent; no new config or product decision is needed.
