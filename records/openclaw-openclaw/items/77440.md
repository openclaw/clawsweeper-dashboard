## Summary

Keep open: current main still derives the chat model selector's `Default (...)` label from global session defaults, so an agent-scoped default can execute correctly while the UI names the wrong model.

## What This Changes

The issue asks WebChat/Control UI to label the `default` model option with the active agent's effective default model instead of the global `agents.defaults.model.primary`.

## Best Possible Solution

Derive the default-selected label from the active session row's effective model when available, fall back to global defaults only when needed, and add focused selector-state regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with an active agent session whose row model is agent-scoped and no explicit UI override selected, `Default (...)` still formats from `sessionsResult.defaults`, which is the global model.

## Solution Assessment

Yes. A narrow Control UI selector-state fix is the best path for the reported default/cleared state because backend execution and session-row model resolution already account for the current agent.
