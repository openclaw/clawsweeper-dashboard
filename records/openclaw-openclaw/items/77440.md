Keep open: current main and the latest release still build the WebChat `Default (...)` model label from global session defaults, and the direct fixing PR is still an open draft with review blockers rather than a merged implementation.

## What This Changes

The issue requests that WebChat display the `Default (...)` model label using the active agent's effective default model instead of the global `agents.defaults.model.primary`.

## Best Possible Solution

Review and land a narrow WebChat fix that derives the default label from the active agent's configured model while preserving the existing session-row fallback and default-agent alias contract.

## Reproduction Assessment

## Solution Assessment

Yes, the requested direction is the narrow maintainable fix: update the WebChat selector-state helper to resolve the active agent default before falling back to the session row/global defaults, with regression coverage and real WebChat proof.
