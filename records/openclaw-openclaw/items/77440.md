## Label Justifications

- `P2`: This is a normal-priority WebChat display bug with limited blast radius but clear user confusion around active model routing.
- `impact:session-state`: The WebChat selector state can display a default model label that does not match the selected session or agent context.
- `impact:auth-provider`: The misleading label can make users think the session will use a different configured provider/model route than it actually uses.

## Summary

Keep open: current main and the latest release still build the WebChat Default label from global session defaults, while a direct open draft PR is already linked as the candidate fix.

## What This Changes

The issue requests that WebChat show the currently selected agent's effective default model in the model selector instead of always labeling Default from the global agent default.

## Best Possible Solution

Review, repair, and land or replace the linked narrow WebChat selector-state fix so Default labels resolve from the active agent model before session/global fallbacks.

## Reproduction Assessment

Yes, at source level: current main builds the WebChat Default label from global session defaults while agent rows expose per-agent model data. I did not run a live browser scenario in this read-only review.

## Solution Assessment

Yes, the best fix is a narrow selector-state change that resolves active-agent defaults before session/global fallbacks, with regression coverage and real WebChat proof on the linked PR.
