## Summary

Current main still has the reported source behavior: WebChat renders the Default label from `sessionsResult.defaults`, while the gateway already exposes per-agent model data. The issue should stay open because the direct linked fixing PR is still open/draft and unmerged.

## What This Changes

The issue asks WebChat to resolve the model selector's Default label from the active agent's effective default model instead of always using the global default.

## Best Possible Solution

Land or replace the linked narrow WebChat selector-state fix so Default labels resolve from the active agent model while preserving session-row and global fallbacks.

## Reproduction Assessment

Yes, at source level: current main builds the WebChat Default label from global session defaults while agent rows expose per-agent model data. I did not run a live browser scenario in this read-only review.

## Solution Assessment

Yes: selector-state is the narrow maintainable seam for this display-only bug, and the linked PR/replacement should resolve active-agent defaults before session/global fallbacks and add regression coverage.
