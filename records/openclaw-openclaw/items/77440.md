## Summary

Keep open: current main still has a source-reproducible Control UI bug where the chat model selector's `Default (...)` label is computed from global session defaults instead of the active agent/session default.

## What This Changes

The issue asks WebChat to show the active agent's effective default model in the chat model selector instead of always showing the global default.

## Best Possible Solution

Resolve the default option label from the active session's agent-specific effective default when available, fall back to global defaults only when necessary, and add focused selector-state coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with global model A and an agent-specific model B, the gateway/session code can expose B for the active session while `resolveDefaultModelValue` still labels the default option from global defaults A.

## Solution Assessment

Yes. A narrow Control UI selector-state fix is the maintainable path because backend model execution and agent default resolution already account for the selected agent.
