## Summary

Keep open: current main still has the WebChat selector-state bug, and the issue is already paired with an open draft PR that uses closing syntax, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks for the WebChat model selector's `Default (...)` label to resolve from the active agent's effective default model instead of always using the global default.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/77690, or an equivalent narrow fix, so the selector label resolves from the active agent/session model before falling back to global defaults and has focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: default-selected sessions render `defaultLabel`, and current main builds that label from global `sessionsResult.defaults` even though active session rows expose an agent-scoped model.

## Solution Assessment

Yes. The narrow maintainable solution is to update the selector-state helper to resolve the active agent or active session row before falling back to global defaults; the linked draft PR is already pointed at that boundary.
