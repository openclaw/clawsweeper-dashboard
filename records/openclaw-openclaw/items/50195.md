## Summary

Keep this open: current main and the latest shipped release still lack the WhatsApp pre-built link-preview pass-through, an opt-in automatic preview config, and a public batch send RPC. The related discussion and prior PRs are useful prior art, but the merged implementation was reverted and the automatic-preview attempt closed unmerged.

## What This Changes

The issue requests WhatsApp support for caller-supplied link preview metadata, an opt-in automatic preview setting, and a batch send RPC for bulk message sends.

## Best Possible Solution

Stage the work: first add typed WhatsApp-owned pre-built `linkPreview` pass-through with tests/docs, then separately design automatic preview fetching controls and batch Gateway semantics.

## Reproduction Assessment

Yes for the reported capability gap. Source inspection on current main shows preview metadata has no WhatsApp send path, the socket/config path has no high-quality preview option, and Gateway exposes only the single-message `send` RPC.

## Solution Assessment

No as one combined change. The pre-built `linkPreview` pass-through is a narrow first step, while automatic URL fetching and batch RPC behavior need separate security and protocol decisions.
