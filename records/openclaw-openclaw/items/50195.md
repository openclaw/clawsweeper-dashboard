## Summary

Keep open: current main and v2026.6.6 still lack the requested WhatsApp pre-built link-preview pass-through, WhatsApp config/socket opt-in for Baileys high-quality preview generation, and a batch Gateway send RPC. The narrow pass-through is dependency-supported, but automatic preview fetching and batch send need maintainer security/protocol decisions before implementation.

## What This Changes

The issue requests WhatsApp support for caller-supplied link-preview payloads, an opt-in Baileys high-quality preview setting, and a batch send RPC path for bulk sends.

## Best Possible Solution

Stage the work: first decide and implement a typed WhatsApp-owned pre-built linkPreview pass-through if maintainers accept the API shape, then handle automatic fetching and batch Gateway sending as separate security/protocol designs.

## Reproduction Assessment

Yes. Source inspection on current main shows no WhatsApp linkPreview input path, no generateHighQualityLinkPreview config/socket option, and no send.batch Gateway method; this is a feature gap rather than broken existing behavior.

## Solution Assessment

No, not as one bundled change. The narrow preview pass-through is maintainable first, while automatic URL fetching and batch RPC need separate maintainer decisions for security and protocol semantics.
