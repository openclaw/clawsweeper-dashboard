## Summary

Keep open: current main and v2026.6.10 still lack the requested WhatsApp pre-built linkPreview pass-through, opt-in automatic preview fetching, and send.batch Gateway RPC. The narrow pass-through is dependency-supported, but the automatic fetch and batch RPC pieces need maintainer security and protocol direction before implementation.

## What This Changes

Request support for pre-built WhatsApp link previews, an opt-in high-quality preview setting, and a batch send Gateway RPC for bulk outbound sends.

## Best Possible Solution

Split the work: implement a typed WhatsApp-owned pre-built linkPreview pass-through first, then design automatic preview fetching and batch Gateway sending separately with explicit security and protocol semantics.

## Reproduction Assessment

Yes, by source inspection rather than a live runtime repro. Current main has no WhatsApp linkPreview input path, no preview-fetching config/socket option, and no send.batch Gateway method while the pinned Baileys dependency supports the underlying text linkPreview pass-through.

## Solution Assessment

No as one combined change. The pre-built preview pass-through is a narrow dependency-supported increment, but automatic URL fetching and batch Gateway sending should be separate maintainer-approved designs.
