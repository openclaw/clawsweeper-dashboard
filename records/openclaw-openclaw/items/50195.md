## Summary

Keep open: current main and latest release v2026.6.10 still lack the WhatsApp pre-built linkPreview pass-through, the opt-in automatic preview setting, and any send.batch Gateway method. The prior discussion and related PRs are useful context, but this still needs maintainer security and protocol direction before implementation.

## What This Changes

Request support for pre-built WhatsApp link preview payloads, an opt-in automatic high-quality preview setting, and a batch Gateway send RPC for bulk WhatsApp sends.

## Best Possible Solution

Split the work: add a typed WhatsApp-owned pre-built linkPreview pass-through first, then design automatic preview fetching and batch Gateway sending separately with explicit security and protocol semantics.

## Reproduction Assessment

Yes, by source inspection rather than a failing runtime repro. Current main has no WhatsApp linkPreview input path, no preview-fetching socket/config option, and no send.batch Gateway method, while the pinned Baileys dependency supports the underlying text linkPreview pass-through.

## Solution Assessment

No as one combined change. The pre-built preview pass-through is a narrow dependency-supported increment, but automatic URL fetching and batch Gateway sending should be separate maintainer-approved designs.
