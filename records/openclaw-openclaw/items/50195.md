## Summary

Keep this issue open. Current main still lacks the requested WhatsApp pre-built link-preview pass-through, a WhatsApp high-quality preview opt-in, and a batch Gateway send RPC; the request is valid but should be split before implementation because it mixes narrow WhatsApp plumbing with SSRF-sensitive URL fetching and a new protocol surface.

## What This Changes

The issue requests WhatsApp pre-built link-preview pass-through, an opt-in for Baileys high-quality previews, and a batch send RPC for bulk message sends.

## Best Possible Solution

Split this into staged maintainable work: first add a typed WhatsApp-owned pre-built linkPreview pass-through with focused tests/docs, then separately design automatic preview fetching and batch Gateway sending with explicit security and protocol semantics.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction exists: current main rejects extra Gateway send fields, builds WhatsApp text sends as { text }, has no WhatsApp preview-generation config/socket option, and advertises no send.batch method.

## Solution Assessment

No, not as one combined automated change. The pre-built preview pass-through is narrow, but automatic URL fetching and batch Gateway sending are separate security/protocol decisions that should be split and reviewed by maintainers.
