## Summary

Keep this issue open. Current main still lacks WhatsApp pre-built `linkPreview` pass-through, has no WhatsApp config/socket opt-in for Baileys high-quality preview generation, and only exposes the single-message Gateway `send` RPC; the related prior work is either reverted or closed unmerged.

## What This Changes

Request adds WhatsApp pre-built link-preview pass-through, an opt-in high-quality preview setting, and a batch Gateway send RPC for multi-message workflows.

## Best Possible Solution

Stage the work: first add a typed WhatsApp-owned pre-built `linkPreview` pass-through with tests and docs, then separately design opt-in automatic preview fetching and additive `send.batch` semantics.

## Reproduction Assessment

Yes. Source inspection on current main shows the WhatsApp send path drops preview metadata, the config/socket path has no high-quality preview option, and Gateway exposes only single-message `send`.

## Solution Assessment

No, not as one combined change. The pre-built preview pass-through is a narrow first step, while automatic URL fetching and batch RPC behavior need separate maintainer-reviewed security and protocol decisions.
