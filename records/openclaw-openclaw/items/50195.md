## Summary

Keep this issue open. Current main still lacks the requested WhatsApp pre-built preview pass-through, opt-in automatic preview generation, and Gateway batch-send RPC; the related prior work is either reverted, closed unmerged, or explicitly consolidated into this tracker.

## What This Changes

Request support for WhatsApp pre-built `linkPreview` payloads, optional Baileys high-quality preview generation, and a Gateway batch send RPC.

## Best Possible Solution

Split the request: land a typed WhatsApp-owned pre-built `linkPreview` pass-through first, then separately design opt-in automatic preview fetching and additive Gateway batch sending with security and protocol docs.

## Reproduction Assessment

Yes. Source inspection on current main shows the WhatsApp send path drops preview metadata, the config/socket path has no preview-generation option, and Gateway exposes only single-message `send`.

## Solution Assessment

No, not as one combined change. The pre-built preview pass-through is narrow, but automatic URL fetching and batch RPC semantics should be separate maintainer-reviewed stages.
