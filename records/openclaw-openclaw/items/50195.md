## Summary

Keep this issue open. Current main still lacks the WhatsApp pre-built `linkPreview` pass-through, the proposed WhatsApp high-quality preview opt-in, and a batch Gateway `send` RPC; the related discussion is useful prior art, but the remaining work spans extension plumbing, SSRF-sensitive URL fetching, and an additive Gateway protocol decision.

## What This Changes

The issue requests WhatsApp pre-built link preview metadata in outbound sends, a configurable high-quality preview fetch option, and a batch Gateway send RPC for bulk message delivery.

## Best Possible Solution

Split the work into staged changes: first add a typed WhatsApp-owned pre-built `linkPreview` pass-through through the outbound and Gateway send path with focused tests and docs, then separately design automatic preview fetching and batch sending with explicit security and protocol semantics.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction exists: current main rejects extra Gateway `send` fields, sends WhatsApp text as `{ text }`, has no WhatsApp preview-generation config/socket option, and advertises no `send.batch` method.

## Solution Assessment

No, not as one bundled automated change. The pre-built preview pass-through is a narrow WhatsApp plumbing change, but automatic URL fetching and batch Gateway sending are separate security and protocol decisions that should be reviewed and split by maintainers.
