## Risks / Open Questions

- Automatic high-quality preview fetching would make the Gateway host fetch arbitrary URLs from outbound WhatsApp text, so SSRF, private-network access, redirects, proxy behavior, timeouts, response-size limits, and logging/privacy semantics need explicit design.
- A public batch send RPC changes Gateway protocol, idempotency, partial-failure reporting, transcript mirroring, and rate-limit behavior; it should not be bundled into a narrow WhatsApp pass-through patch.
- The earlier per-message preview implementation was reverted after an accidental merge, so reintroducing it should use current typed plugin/Gateway contracts rather than reviving the old path directly.
