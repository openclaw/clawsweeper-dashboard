## Summary

Keep this open. Current main still lacks the requested WhatsApp pre-built preview pass-through, automatic preview opt-in, and Gateway batch send RPC; the request is valid but combines a narrow WhatsApp change with SSRF-sensitive fetching and a new protocol surface.

## What This Changes

The issue requests WhatsApp pre-built link-preview pass-through, a configurable Baileys high-quality preview option, and a batch Gateway send RPC for bulk sends.

## Best Possible Solution

Split the request into staged work: land a typed WhatsApp-owned pre-built `linkPreview` pass-through first, then design automatic preview fetching and batch Gateway sending as separate security/protocol changes.

## Reproduction Assessment

Yes. Source inspection on current main shows the WhatsApp send path drops preview metadata, the config/socket path has no preview-generation option, and Gateway exposes only single-message `send`.

## Solution Assessment

No, not as one combined automated repair. The pre-built preview pass-through is narrow, but automatic URL fetching and a new batch RPC require separate maintainer design decisions.
