## Summary

Keep open: current main and the latest v2026.6.8 release still lack WhatsApp pre-built link-preview pass-through, an opt-in automatic high-quality preview setting, and a batch Gateway send RPC. The narrow pass-through is dependency-supported, but automatic URL fetching and Gateway batching need explicit maintainer security and protocol decisions before implementation.

## What This Changes

The issue requests WhatsApp support for caller-supplied Baileys link preview metadata, an opt-in high-quality preview config, and a batch Gateway send method for multi-message workflows.

## Best Possible Solution

Split the work: add a typed WhatsApp-owned pre-built linkPreview pass-through first, then handle automatic preview fetching and batch Gateway sending as separate security and protocol designs.

## Reproduction Assessment

Yes, from source inspection. Current main has no WhatsApp linkPreview input path, no preview-fetching socket/config option, and no send.batch Gateway method, while the pinned Baileys dependency supports the underlying text linkPreview pass-through.

## Solution Assessment

No as one bundled change. The pre-built preview pass-through is a narrow dependency-supported increment, but automatic fetching and batch Gateway sending should be separate maintainer-approved designs.
