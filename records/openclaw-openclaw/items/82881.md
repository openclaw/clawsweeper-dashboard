## Summary

Keep open. This is a fresh external performance regression with detailed profiler and downgrade evidence; current main still has the broad manifest-read and conditional Jiti-transform surfaces that plausibly match the report, and the related open issues cover sibling problems rather than this exact Linux 2026.5.12 regression. The fix crosses plugin discovery, runtime loading, and cache policy, so it needs maintainer profiling and design review rather than cleanup closure.

## What This Changes

The issue requests a fix for a 2026.5.12 regression where short CLI commands spend tens of seconds in Jiti/Babel plugin loading and repeated manifest discovery.

## Best Possible Solution

Run a packaged current-main and 2026.5.12 profile, then land an owner-reviewed loader/discovery change that reuses explicit metadata snapshots or registries and avoids Jiti for compiled JS where the SDK alias contract permits, while coordinating manifest-discovery filtering with https://github.com/openclaw/openclaw/issues/70533.

## Reproduction Assessment

No high-confidence local packaged reproduction was established. The reporter provides detailed timing, profile, strace, and downgrade evidence, and source inspection confirms the relevant manifest-read and Jiti-transform paths still exist on current main.

## Solution Assessment

Unclear as written. Fixing the regression is the right direction, but the persistent cache suggestion is not the best first implementation under the plugin cache-boundary docs; a safer solution should carry explicit metadata through hot paths and narrow Jiti use for already-built artifacts.
