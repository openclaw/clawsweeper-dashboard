## Summary

Keep open. The report is recent and concrete, current main still has matching broad manifest-registry and Jiti source-transform paths, and the related open issues do not fully cover this 2026.5.12 Linux/package startup regression.

## What This Changes

The issue reports a v2026.5.12 CLI startup regression where `openclaw models` spends tens of seconds in Jiti/Babel traversal and repeated plugin manifest discovery.

## Best Possible Solution

Profile the packaged CLI path on current main and then remove avoidable Jiti transforms and duplicate manifest reads by carrying explicit metadata/load-context objects through hot paths.

## Reproduction Assessment

No high-confidence live current-main reproduction was established in this read-only review. The issue provides strong release-to-release timing evidence, and source inspection confirms matching manifest-read and Jiti-transform paths, so this is source-reproducible but still needs packaged profiling.

## Solution Assessment

Unclear as written. The performance bug should be fixed, but the proposed persistent cache direction needs maintainer approval; the safer path is an owner-reviewed loader/discovery change that respects the explicit metadata boundary.
