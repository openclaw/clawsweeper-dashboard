The issue reports a post-v2026.5.26 regression where managed `web_search` times out through `fetchWithSsrFGuard` for Kimi/Moonshot and DuckDuckGo while direct network calls and `web_fetch` still work.

## Best Possible Solution

Partially: the reporter provided concrete logs and current source confirms Kimi and DuckDuckGo web_search calls still enter `fetchWithSsrFGuard`, pinned DNS, and runtime Undici fetch when no proxy is used. I did not reproduce the exact timeout in the reporter's network and provider environment.

## Solution Assessment

Unclear: a web-search-only workaround would be too narrow, and closing as fixed is not supported by current source or release evidence. The safer direction is a shared guarded-fetch fix coordinated with the broader pinned-dispatcher issue while retaining web_search proof requirements.
