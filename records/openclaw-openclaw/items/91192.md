## Summary

Keep open: this PR targets a real source-reproducible TTS delivery bug and the shared fix shape is close, but the suppressed block-media bypass normalizes media before checking the trusted-local-media gate, so it needs one narrow repair before merge.

## What This Changes

This PR changes shared embedded tool-media merging and auto-reply dispatch so trusted local TTS media can bypass `message_tool_only` suppression as media-only, with regression tests for trusted and untrusted block/final paths.

## Best Possible Solution

Keep the shared merge/dispatch approach, but gate raw block payloads for `trustedLocalMedia === true` and sendable media before media normalization, then land with focused regression coverage and the existing live transport proof.

## Solution Assessment

No, not quite as submitted. The shared dispatch/media-merge boundary is the right place to fix the bug, but the block bypass should check trust before media normalization to avoid processing suppressed untrusted media.
