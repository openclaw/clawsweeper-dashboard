## Summary

Keep open. Current main and v2026.5.12 still use the reported Gateway-spawned `imsg rpc` readiness path, the report has concrete current-release logs, and the newer workaround comment shows a plausible architectural escape hatch rather than an implemented core fix.

## What This Changes

The issue reports that iMessage startup hangs on the `imsg rpc` readiness check when Gateway runs under the macOS LaunchAgent.

## Best Possible Solution

Keep this open and make LaunchAgent iMessage startup either work under a supported macOS process identity or fail fast with clear launchd/TCC guidance and reliable cleanup of hung `imsg` children.

## Reproduction Assessment

No high-confidence live reproduction was established in this review. The source path is clear on current main and the issue provides detailed v2026.5.12 logs, but proving the hang still needs a real macOS LaunchAgent plus imsg setup.

## Solution Assessment

Unclear. A narrow diagnostic and child-cleanup fix is likely useful, but choosing a signed app/session wrapper, documented launch mode, or `imsg-server` endpoint support needs maintainer product and security judgment.
