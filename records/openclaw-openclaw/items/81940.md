- `P1`: The report describes a current-release macOS LaunchAgent startup path that leaves the bundled iMessage channel unavailable for real users.
- `impact:message-loss`: The channel fails readiness and exits, so iMessage receive/send workflows are unavailable under the reported startup mode.

## Summary

Keep open. Current main and the latest release still use the reported macOS LaunchAgent to Node to stdio `imsg rpc` readiness path, and the newer comments add a second reproduction plus workaround detail rather than making the report obsolete.

## What This Changes

Report asks OpenClaw to fix or clearly diagnose iMessage `imsg rpc` readiness hangs when the Gateway runs as the macOS LaunchAgent.

## Best Possible Solution

Keep this open and make LaunchAgent iMessage startup either work under a supported macOS process/RPC model or fail quickly with clear launchd/TCC/session guidance while reliably cleaning up stuck `imsg` children.

## Reproduction Assessment

No, not as a live high-confidence reproduction. The issue has detailed macOS 26.3.1 logs and a second same-version repro comment, and current source still uses the same LaunchAgent-to-stdio `imsg rpc` readiness path.

## Solution Assessment

Unclear. A narrow diagnostics and child-cleanup fix is plausible, but deciding whether OpenClaw should support bare launchd, an app/session-owned launcher, or a first-class RPC endpoint needs maintainer product and security judgment.
