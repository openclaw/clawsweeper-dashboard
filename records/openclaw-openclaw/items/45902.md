## Summary

Keep this PR open: current main still does not implement Telegram parent-chat future-thread model defaults, and the proposed branch has concrete correctness blockers plus unrelated memory, gateway, and live E2E tooling changes that need maintainer review before merge.

## What This Changes

The PR adds Telegram `/model` future-thread default persistence/seeding, `direct_messages_topic` thread-id routing, memory auth diagnostics and retry changes, gateway probe/call tweaks, and optional Telegram live E2E/recovery scripts.

## Best Possible Solution

Keep the PR open for a scoped maintainer revision or replacement that implements only Telegram model inheritance first: persist same-chat parent future-thread defaults from both `/model` paths, seed genuinely new Telegram forum-topic and DM-topic sessions from that default, avoid cross-chat leakage, tolerate metadata pre-created rows, and cover those paths with focused tests. Split memory, gateway, recovery-script, and live E2E tooling into separate PRs.

## Reproduction Assessment

Yes. A high-confidence reproduction path is to set `/model` in a Telegram topic/thread on current main, create a new topic/thread in the same chat, and observe that the new session has no parent future-thread default to inherit because the schema, persistence, and seeding paths are absent.

## Solution Assessment

No. The feature direction is reasonable, but this PR is not the best current solution because it mixes unrelated changes and still has actionable correctness and supply-chain review blockers; a narrower Telegram-only patch is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Scope main-thread defaults by Telegram chat:** `src/sessions/session-key-utils.ts:173`
  - body: For `agent:...:main:thread:<chatId>:<topicId>` keys, the helper collapses the parent to the shared `main` key. That lets `/model` in one Telegram DM chat seed future threads in another chat, so the parent key needs to retain chat scope or skip this key shape.
  - confidence: 0.91
- **[P2] Seed defaults before metadata rows block inheritance:** `src/auto-reply/reply/session.ts:474`
  - body: Telegram inbound metadata can create a session row before `initSessionState` runs. Because seeding requires `!hasExistingSessionEntry`, the first real turn for a brand-new topic can skip the parent default and start without the inherited model.
  - confidence: 0.86
- **[P2] Gate parent default writes on actual /model changes:** `src/auto-reply/reply/directive-handling.impl.ts:428`
  - body: The directive-only write callback persists the parent entry whenever it exists, even for non-model directives. A concurrent `/queue`, `/think`, or similar directive can write a stale parent snapshot and revert a `/model` future-thread default.
  - confidence: 0.84
- **[P2] Gate inline parent writes on actual /model changes:** `src/auto-reply/reply/directive-handling.persist.ts:236`
  - body: The inline directive path has the same stale-parent overwrite window: any updated inline directive writes the loaded parent entry back even when this request did not modify the parent default.
  - confidence: 0.82
- **[P2] Preserve device identity for remote authenticated probes:** `src/gateway/probe.ts:77`
  - body: For non-loopback gateways with token/password auth, the probe sends `deviceIdentity: null`. That can break remote paired-device authenticated probes that require device identity in the handshake.
  - confidence: 0.8
- **[P2] Use live Telegram context when persisting directive defaults:** `src/auto-reply/reply/directive-handling.impl.ts:321`
  - body: Directive-only `/model` derives the Telegram hint only from persisted session fields, so older main-scoped Telegram thread entries without `channel` or `lastChannel` skip parent default persistence even though the live context is Telegram.
  - confidence: 0.78
- **[P2] Use live Telegram context for inline /model defaults:** `src/auto-reply/reply/directive-handling.persist.ts:88`
  - body: Inline `/model` uses only persisted session channel hints. Migrated main-scoped Telegram threads without those fields update the current session but never persist the future-thread default for new Telegram threads.
  - confidence: 0.77
