## Summary

Keep open: the cleanup direction is useful, but the submitted patch is incorrect against current main. The selector implementation has moved to `ui/src/ui/chat/session-controls.ts`, current main still has a source-reproducible Telegram label gap, and the related open work at https://github.com/openclaw/openclaw/issues/7406 and https://github.com/openclaw/openclaw/pull/68850 is adjacent rather than a complete superseder.

## What This Changes

The PR changes Control UI session selector rendering to clean Telegram DM/slash/group labels, dedupe the `main` alias, add `origin.label` to UI session-row typing, and add selector tests.

## Best Possible Solution

Port the safe label cleanup to `ui/src/ui/chat/session-controls.ts`: use existing row metadata, consider `origin.label` only where appropriate, strip only known Telegram transport noise, preserve arbitrary user labels, and keep broader topic-name metadata work with https://github.com/openclaw/openclaw/issues/7406.

## Reproduction Assessment

Yes, source-level. Current main's display-name resolver ignores `origin.label`, so Telegram rows whose only clean label is in `origin.label` or whose display name is `telegram:<name>` still fall through to noisy or fallback labels without needing a live run to see the code path.

## Solution Assessment

No, not as submitted. The intended UI cleanup is maintainable, but this patch targets the old helper layout and introduces overbroad main aliasing, prefix stripping, and label filtering.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve bare main when the canonical row is absent:** `ui/src/ui/app-render.helpers.ts:139`
  - body: The patch rewrites `state.sessionKey === "main"` to `agent:main:main` even when the rendered options only contain `main`. In that stale-list state the select has no matching value and can visually show the wrong active session.
  - confidence: 0.93
- **[P2] Restrict prefix stripping to Telegram noise:** `ui/src/ui/app-render.helpers.ts:748-749`
  - body: `cleanValue()` strips any leading `<token>:` from every label and display name. Free-form labels such as `project:alpha` or `maintainer:v2` would be truncated even when they are not Telegram transport prefixes.
  - confidence: 0.89
- **[P2] Match only the canonical main alias:** `ui/src/ui/app-render.helpers.ts:777-779`
  - body: The `key.endsWith(":main")` guard labels every agent main session as `main`, not only the `main`/`agent:main:main` alias pair. That can collapse unrelated agent session rows before disambiguation can preserve them.
  - confidence: 0.9
- **[P2] Dedupe only the main alias pair:** `ui/src/ui/app-render.helpers.ts:896-900`
  - body: The new filter removes later options whose visible label is `main`, including a real contact, room, or renamed session. The dedupe should target only the `main` and `agent:main:main` keys when both are present.
  - confidence: 0.91
