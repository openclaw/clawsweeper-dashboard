## Summary

Keep open. Current main still does not implement the Telegram session selector cleanup, and the submitted branch remains useful source work but needs a rebase to the current session-controls module plus fixes for the review blockers around main-session aliasing and broad label cleanup.

## What This Changes

The PR changes the Control UI session selector to clean Telegram DM/slash/group labels, expose `origin.label` on UI session rows, dedupe the visible main alias, and add chat view regression tests.

## Best Possible Solution

Land an updated implementation in the current session-controls module that normalizes only known Telegram transport noise, uses `origin.label` when it is the only human conversation label, preserves free-form labels, and dedupes only the `main`/`agent:main:main` alias pair by key. The topic-name portion should stay aligned with open #7406 so the final UI behavior is covered by focused regression tests.

## Reproduction Assessment

Yes. The current-main proof is source-level: the resolver and scoped option label paths still ignore origin-only Telegram metadata, and the PR's dummy test cases show the expected `Alice DM`, `Monk-tech`, slash-label, and main-alias scenarios.

## Solution Assessment

No, the submitted patch is not the best fix as-is. The right solution is a smaller current-main patch that applies the Telegram cleanup in `ui/src/ui/chat/session-controls.ts`, keeps the bare-main fallback selected unless the canonical row exists, and avoids arbitrary prefix stripping or label-based dedupe.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Preserve the bare main value:** `ui/src/ui/app-render.helpers.ts:139`
  - body: The new selected value rewrites `state.sessionKey === "main"` to `agent:main:main` even when the canonical row is not present. If the rendered options only contain `main`, the select has no matching value and can visually show the wrong active session.
  - confidence: 0.93
- **[P2] Limit prefix stripping to Telegram noise:** `ui/src/ui/app-render.helpers.ts:748-749`
  - body: `cleanValue()` strips any leading `<token>:` from every label/displayName. Session labels are user-visible free-form text, so names like `project:alpha` or `maintainer:v2` would be truncated even when they are not Telegram transport prefixes.
  - confidence: 0.89
- **[P2] Match only the canonical main key:** `ui/src/ui/app-render.helpers.ts:777-779`
  - body: The `key.endsWith(":main")` guard labels every agent's main session as `main`, not only `main` and `agent:main:main`. That can collapse unrelated session rows before the duplicate-label logic can disambiguate them.
  - confidence: 0.9
- **[P2] Dedupe main by key, not label:** `ui/src/ui/app-render.helpers.ts:896-900`
  - body: This filter removes every later option whose visible label is `main`, including a real room, contact, or renamed session. The dedupe should target only the `main` and `agent:main:main` alias keys when both are present.
  - confidence: 0.91
