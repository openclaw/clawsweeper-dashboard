## Summary

Keep open for human follow-up: the visual cleanup is useful and source-reproducible on current main, but the branch targets stale selector code, lacks real behavior proof, and introduces blocking selector regressions. The related trackers https://github.com/openclaw/openclaw/issues/7406 and https://github.com/openclaw/openclaw/pull/68850 are adjacent, not full supersedes for this PR.

## What This Changes

The branch changes Control UI session selector label rendering for Telegram rows, adds optional origin labels to UI session rows, and adds selector tests.

## Best Possible Solution

Port only the safe visual cleanup into `ui/src/ui/chat/session-controls.ts`, use explicit session metadata where available, strip only known Telegram transport noise, dedupe only the `main` alias pair, and leave full topic-name metadata to https://github.com/openclaw/openclaw/issues/7406.

## Reproduction Assessment

Yes, source-level. Current main's selector resolver ignores `origin.label` and returns raw non-key `displayName` values, so Telegram rows with noisy display names or useful origin labels can still render noisy or fallback labels without a live browser run.

## Solution Assessment

No, not as submitted. The cleanup should be narrowed and ported to the current selector module while avoiding broad prefix stripping, broad main-session aliasing, and label-based option filtering.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve bare main when the canonical row is absent:** `ui/src/ui/app-render.helpers.ts:139`
  - body: The patch rewrites `state.sessionKey === "main"` to `agent:main:main` even when the rendered options only contain `main`. In that stale-list state the select has no matching value and can visually show the wrong active session.
  - confidence: 0.93
- **[P2] Restrict prefix stripping to Telegram transport noise:** `ui/src/ui/app-render.helpers.ts:741-749`
  - body: `cleanValue()` strips any leading `<token>:` from every label and display name. Free-form labels such as `project:alpha` or `maintainer:v2` would be truncated even when they are not Telegram transport prefixes.
  - confidence: 0.89
- **[P2] Match only the canonical main alias:** `ui/src/ui/app-render.helpers.ts:777-779`
  - body: The `key.endsWith(":main")` guard labels every agent main session as `main`, not only the `main` / `agent:main:main` alias pair. That can collapse unrelated agent session rows before disambiguation can preserve them.
  - confidence: 0.9
- **[P2] Dedupe only the main alias pair:** `ui/src/ui/app-render.helpers.ts:896-900`
  - body: The new filter removes later options whose visible label is `main`, including a real contact, room, or renamed session. The dedupe should target only the `main` and `agent:main:main` keys when both are present.
  - confidence: 0.91
