## Summary

Keep open: current main still has the Control UI Telegram label gap, but this PR is stale against the moved `session-controls` implementation and the submitted diff introduces selector/label regressions that should block merge as-is.

## What This Changes

This PR changes the Control UI session selector to clean Telegram DM/slash/group labels, expose `origin.label` on UI session rows, dedupe the `main` alias, and add chat view regression tests.

## Best Possible Solution

Land or replace this with a narrow current-main patch in `ui/src/ui/chat/session-controls.ts` that uses existing origin/display metadata, strips only known Telegram transport noise, preserves free-form labels, and dedupes only the `main`/`agent:main:main` alias pair while leaving broader topic-name storage/display work aligned with #7406.

## Reproduction Assessment

Yes. A source-level reproduction is clear: construct Telegram session rows like the PR body describes, especially rows whose human label exists only in `origin.label`, and current main falls back to raw label/displayName or parsed key rest in the selector path.

## Solution Assessment

No. The direction is useful, but this patch is not the best implementation because it targets the old helper file shape and introduces overbroad `main` remapping, prefix stripping, alias matching, and visible-label dedupe.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve bare main when the canonical row is absent:** `ui/src/ui/app-render.helpers.ts:139`
  - body: The patch rewrites `state.sessionKey === "main"` to `agent:main:main` even when the rendered options only contain `main`. In that stale-list state the select has no matching value and can visually show the wrong active session.
  - confidence: 0.93
- **[P2] Restrict prefix stripping to Telegram noise:** `ui/src/ui/app-render.helpers.ts:748-749`
  - body: `cleanValue()` strips any leading `<token>:` from all labels and display names. Free-form labels such as `project:alpha` or `maintainer:v2` would be truncated even when they are not Telegram transport prefixes.
  - confidence: 0.89
- **[P2] Match only the canonical main alias:** `ui/src/ui/app-render.helpers.ts:777-779`
  - body: The `key.endsWith(":main")` guard labels every agent main session as `main`, not only the `main`/`agent:main:main` alias pair. That can collapse unrelated agent session rows before disambiguation can preserve them.
  - confidence: 0.9
- **[P2] Dedupe only the main alias pair:** `ui/src/ui/app-render.helpers.ts:896-900`
  - body: The new filter removes later options whose visible label is `main`, including a real contact, room, or renamed session. The dedupe should target only the `main` and `agent:main:main` keys when both are present.
  - confidence: 0.91
