## Summary

Keep open: the requested UI cleanup is useful, but the submitted patch has blocking selector-label regressions, lacks external after-fix real behavior proof, and overlaps the open canonical Telegram topic-label tracker.

## What This Changes

The PR changes the Control UI session selector to clean Telegram DM/slash/group labels, dedupe `main` alias options, add `origin.label` to the UI session-row type, and add selector tests.

## Best Possible Solution

Port the safe cleanup to `ui/src/ui/chat/session-controls.ts`: use appropriate row metadata, strip only known Telegram transport noise, preserve arbitrary user labels, dedupe only the `main` alias pair, and keep topic-name metadata work with https://github.com/openclaw/openclaw/issues/7406.

## Reproduction Assessment

Yes, source-level. Current main's selector resolver ignores `origin.label` and returns raw non-key `displayName` values, so Telegram rows with noisy `telegram:<name>` display names or useful origin labels can still render noisy or fallback labels without a live run.

## Solution Assessment

No, not as submitted. The intended visual cleanup is maintainable, but this patch edits the old helper layout and introduces overbroad aliasing, prefix stripping, and label filtering that should be replaced with a narrower current-module change.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve bare main when the canonical row is absent:** `ui/src/ui/app-render.helpers.ts:139`
  - body: The patch rewrites `state.sessionKey === "main"` to `agent:main:main` even when the rendered options only contain `main`. In that stale-list state the select has no matching value and can visually show the wrong active session.
  - confidence: 0.93
- **[P2] Restrict prefix stripping to Telegram transport noise:** `ui/src/ui/app-render.helpers.ts:746-749`
  - body: `cleanValue()` strips any leading `<token>:` from every label and display name. Free-form labels such as `project:alpha` or `maintainer:v2` would be truncated even when they are not Telegram transport prefixes.
  - confidence: 0.89
- **[P2] Match only the canonical main alias:** `ui/src/ui/app-render.helpers.ts:777-779`
  - body: The `key.endsWith(":main")` guard labels every agent main session as `main`, not only the `main` / `agent:main:main` alias pair. That can collapse unrelated agent session rows before disambiguation can preserve them.
  - confidence: 0.9
- **[P2] Dedupe only the main alias pair:** `ui/src/ui/app-render.helpers.ts:896-900`
  - body: The new filter removes later options whose visible label is `main`, including a real contact, room, or renamed session. The dedupe should target only the `main` and `agent:main:main` keys when both are present.
  - confidence: 0.91
