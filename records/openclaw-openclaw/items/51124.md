## Summary

Keep open. Current main still does not implement the requested Telegram label cleanup, and the submitted patch has concrete selector regressions that should block merge until the work is rebased onto the current `session-controls` implementation and narrowed.

## What This Changes

This PR changes the Control UI session selector to clean Telegram DM/slash/group labels, expose `origin.label` on UI session rows, dedupe the visible main alias, and add chat view regression tests.

## Best Possible Solution

Land a narrow current-main patch in `ui/src/ui/chat/session-controls.ts` that uses known Telegram display/origin metadata, strips only known Telegram transport noise, preserves free-form labels, and dedupes only the `main`/`agent:main:main` alias pair by key. Keep the larger topic-name storage/display behavior aligned with #7406 and cover the label cases with focused UI/helper tests.

## Reproduction Assessment

Yes. A focused source-level reproduction is enough: construct session rows like the PR body describes and current main will either use raw `label`/`displayName` strings or fall back to the parsed key/rest because `origin.label` is ignored in the selector path.

## Solution Assessment

No. The submitted patch is useful source work, but it is not the best fix as-is because it rewrites selected `main` unconditionally, strips arbitrary `<token>:` prefixes from free-form labels, matches every key ending in `:main`, and dedupes by visible label instead of the main alias keys.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve bare main when the canonical row is absent:** `ui/src/ui/app-render.helpers.ts:139`
  - body: The patch rewrites `state.sessionKey === "main"` to `agent:main:main` even when the rendered options only contain `main`. In that stale-list state the select has no matching value and can visually show the wrong active session.
  - confidence: 0.93
- **[P2] Restrict prefix stripping to Telegram noise:** `ui/src/ui/app-render.helpers.ts:748-749`
  - body: `cleanValue()` strips any leading `<token>:` from all labels and display names. Free-form labels such as `project:alpha` or `maintainer:v2` would be truncated even when they are not Telegram transport prefixes.
  - confidence: 0.89
- **[P2] Match only the canonical main alias:** `ui/src/ui/app-render.helpers.ts:777-779`
  - body: The `key.endsWith(":main")` guard labels every agent main session as `main`, not only the `main`/`agent:main:main` alias pair. That can collapse unrelated agent session rows before disambiguation can preserve them.
  - confidence: 0.9
- **[P2] Dedupe only the main alias pair:** `ui/src/ui/app-render.helpers.ts:896-900`
  - body: The new filter removes later options whose visible label is `main`, including a real contact, room, or renamed session. The dedupe should target only the `main` and `agent:main:main` keys when both are present.
  - confidence: 0.91
