## Summary

Keep open. Current main still hardcodes the overflow compaction retry cap at 3 and the existing compaction config/schema/docs expose adjacent knobs but not a max-overflow-attempts setting; because this would add a new operator-facing config policy, it needs maintainer product/runtime review rather than cleanup close.

## What This Changes

Request to make the embedded agent overflow-compaction retry limit configurable, defaulting to the current value of 3.

## Best Possible Solution

Decide whether OpenClaw should support a bounded overflow-retry knob; if yes, add it to the config type, zod schema, help/labels/docs, and thread the resolved value into the embedded overflow recovery loop while preserving default 3.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new config feature request rather than broken documented behavior. Source inspection does verify the current-main behavior is hardcoded at 3 and not exposed in the compaction config contract.

## Solution Assessment

Unclear until maintainers decide the config policy. The requested direction is plausible, but adding a knob changes the fail-fast versus recovery tradeoff and should be aligned with the broader overflow recovery work.
