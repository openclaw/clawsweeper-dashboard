## Summary

Keep open: current main still hardcodes the overflow compaction retry ceiling at 3, and the public config type, strict schema, docs, and tests do not expose an equivalent knob. The request is a plausible cost-control config feature, but it needs maintainer approval for the public config contract before implementation.

## What This Changes

The issue requests exposing the overflow compaction retry ceiling as `agents.defaults.compaction.maxOverflowAttempts` so operators can lower the current default of 3.

## Best Possible Solution

If maintainers approve the knob, add a bounded `agents.defaults.compaction` option that preserves default behavior at 3 and wire it through runtime, types, schema, help/docs, and focused overflow-compaction tests.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a bug reproduction, but yes for source confirmation: current main still has the hardcoded three-attempt overflow retry path and no matching public config key.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers approve the public config contract; the proposed placement is plausible, but naming, bounds, and whether this should be a raw retry ceiling or broader cost guard need product judgment.
