## Summary

Keep open: current main still hardcodes the overflow compaction retry ceiling at 3 and the public compaction config/schema/docs do not expose a matching knob. This is a narrow, plausible config feature request rather than a broken existing contract, so it needs maintainer product/config approval before implementation.

## What This Changes

Request to make the embedded runner's overflow-triggered compaction retry limit configurable under `agents.defaults.compaction` instead of fixed at 3.

## Best Possible Solution

If maintainers approve the knob, add a bounded `agents.defaults.compaction` option, preserve default behavior at 3, wire it through type/schema/help/docs, and cover both default and fail-fast behavior in focused overflow compaction tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection on current main does confirm the reported hardcoded three-attempt behavior and the absence of an existing config key.

## Solution Assessment

Unclear until maintainers approve the public config contract. The proposed placement under `agents.defaults.compaction` is plausible, but naming, validation bounds, and default semantics should be settled before implementation.
