## Summary

Keep open: current main still hardcodes the overflow compaction retry ceiling at 3, and the public compaction config/schema/docs do not expose a matching knob. This is a narrow config feature request that needs maintainer approval for the public contract, not a broken existing behavior under current docs or tests.

## What This Changes

Request to make the overflow compaction retry ceiling configurable under `agents.defaults.compaction` instead of fixed at three attempts.

## Best Possible Solution

If approved, add a bounded `agents.defaults.compaction` option that preserves default behavior at 3, wire it through types/schema/help/docs, and cover default plus fail-fast behavior in focused overflow compaction tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a config feature request. Source inspection on current main confirms the hardcoded three-attempt behavior and the absence of an existing config key.

## Solution Assessment

Unclear until maintainers approve the public config contract. The proposed placement under `agents.defaults.compaction` is plausible, but naming, validation bounds, and default semantics should be settled before implementation.
