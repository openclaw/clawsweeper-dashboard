## Summary

Current main and the latest local release tag route custom `api: "ollama"` providers through the Ollama provider hook and read the selected provider's own `baseUrl`, so the reported multi-host dispatch path is implemented.

## What This Changes

The issue asks Ollama dispatch for custom `ollama-*` providers to honor each `models.providers.<id>.baseUrl` instead of routing remote fallbacks to localhost.

## Best Possible Solution

Keep the shipped per-provider Ollama routing behavior and have users on 2026.4.24 or `@openclaw/ollama-provider` 2026.4.20 upgrade to `v2026.4.29` or newer.

## Reproduction Assessment

Yes. The issue provides concrete multi-provider config and fallback steps, and current tests cover the same custom-provider hook and per-provider `baseUrl` selection paths without needing the reporter's fleet.

## Solution Assessment

Yes. Resolving the selected provider through its `api` owner and reading that provider's own `baseUrl` is the narrow provider-owned fix; separate plugin instances are not needed for this case.
