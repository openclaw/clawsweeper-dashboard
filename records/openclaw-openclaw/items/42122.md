## Summary

Keep open: the speedup is useful, but the PR is not already implemented on main and its skip path is too broad for current install-smoke image reuse, because package/update and Bun-global paths still need full Control UI assets.

## What This Changes

The PR adds an opt-in Docker build argument that lets install-smoke Docker builds create a minimal Control UI placeholder instead of running the full UI build.

## Best Possible Solution

Keep the Docker speedup only for fast startup-only image coverage, or build a separate full-asset root image for package, update, Bun-global, release, and package-acceptance paths.

## Reproduction Assessment

Yes. Source inspection is enough: the PR placeholder would omit dist/control-ui/assets/, while current main reuses the root Dockerfile image for package/update and Bun-global paths that pack or validate dist/.

## Solution Assessment

No. The Docker build-arg mechanism is viable after the PR's ARG scoping fix, but passing the skip arg to a reusable full root image is broader than the maintainable boundary on current main.
