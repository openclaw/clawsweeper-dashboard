## What This Changes

The issue reports Discord streaming losing a final assistant reply when a later failed-tool warning final is delivered in the same turn.

## Reproduction Assessment

## Solution Assessment

Yes, the maintainable fix is to preserve an already-finalized non-error Discord preview when a later error final is delivered, while still sending the warning separately. The open fix PR follows that boundary rather than suppressing tool warnings globally.
