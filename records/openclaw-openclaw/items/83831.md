## What This Changes

Discord streamed final replies can disappear when a later failed-tool warning final runs through the inbound Discord live-preview finalization path.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83844 or an equivalent Discord delivery fix that preserves finalized assistant previews while keeping standalone terminal error cleanup behavior covered by tests.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: Pi can append an `isError` warning final after a normal reply, Discord skips preview editing for that error payload, and the shared finalizer can clear the attached draft. I did not run a credentialed Discord repro during this read-only review.

## Solution Assessment

Yes. The narrow solution is in Discord preview finalization/lane handling; globally suppressing tool warnings would remove useful status messages without fixing the lifecycle bug.
