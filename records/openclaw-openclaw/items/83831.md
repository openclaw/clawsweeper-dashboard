## Summary

Keep open: current main still has a source-supported path that can delete a finalized Discord partial-stream preview when a later tool-error warning payload is delivered, matching the latest 2026.5.18 report; I did not find an open canonical duplicate or merged fix for this narrower variant.

## What This Changes

Fix Discord partial streaming so a tool-call failure warning does not remove or supersede the assistant's final text reply.

## Best Possible Solution

Preserve an already-finalized Discord preview when later tool-error warning payloads are delivered, and send the warning separately without clearing the final answer.

## Reproduction Assessment

Yes, source-level: Pi can emit assistant text followed by an `isError` tool-warning payload, and Discord's finalizer can clear the attached draft when processing that later warning. I did not run a live Discord reproduction in this read-only pass.

## Solution Assessment

Yes, the best fix appears to be in Discord preview finalization/lane handling, not in suppressing tool-warning payloads globally. The warning should remain visible while the finalized answer remains durable.
