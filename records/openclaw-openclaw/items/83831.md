## Summary

Keep open because current main still has the source-supported Discord draft-finalization path, and the open closing fix is https://github.com/openclaw/openclaw/pull/84169; this issue should close only after that PR lands or is replaced.

## What This Changes

The issue reports that a Discord streamed assistant final reply can disappear when a later failed-tool warning is delivered in the same turn.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/84169 so the finalized Discord preview remains durable while later tool-warning payloads are delivered separately.

## Reproduction Assessment

Yes, source-level: Pi can emit a normal assistant final followed by an `isError` tool-warning payload, and Discord's current preview fallback path can clear the attached draft when processing that later warning. I did not run a live Discord reproduction in this read-only pass.

## Solution Assessment

Yes, the narrowest maintainable fix belongs in Discord live-preview finalization so error warning finals cannot clear an already-finalized answer. Suppressing tool warnings globally would lose useful error visibility and is not the best fix.
