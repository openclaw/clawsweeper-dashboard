## Summary

Keep open. Current main still has no provider-level `models.providers.<id>.enabled` contract, and the two implementation PRs that explicitly tried to close this work were both closed unmerged.

## What This Changes

Add a provider-level `models.providers.<id>.enabled` option that disables a configured model provider while preserving its config block.

## Best Possible Solution

Add a documented default-true provider-enabled contract and apply it consistently across model planning, fallback selection, configured catalogs, auth/status output, embedded-runner resolution, and regression tests.

## Reproduction Assessment

Yes. Source inspection on current main shows the provider type/schema omit `enabled`, and the relevant planning, fallback, catalog, and status paths have no disabled-provider gate.

## Solution Assessment

Yes in direction. A provider-level boolean defaulting to enabled is the narrowest maintainable shape, but it should be implemented as a full config contract rather than a type-only patch.
