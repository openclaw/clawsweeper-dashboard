## Best Possible Solution

Approve one shared provider eligibility contract that defaults omitted values to enabled, preserves disabled config blocks, and consistently gates schema, planning, runtime resolution, fallback, catalog/listing, auth/probe, status output, docs, and focused tests.

## Reproduction Assessment

Yes, at source level: current main's provider type/schema omit `enabled`, and planning, fallback, configured catalog, and status/list paths consume providers without a disabled-provider gate. I did not run a live config because this review is read-only.

## Solution Assessment

Unclear pending maintainer product direction. A provider-level flag is plausible, but the best implementation should be a shared provider eligibility contract rather than scattered per-consumer filters.
