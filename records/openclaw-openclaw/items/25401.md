## Summary

Keep open. Current main still lacks the provider-level `models.providers.<id>.enabled` contract, and the linked implementation PR was closed unmerged, so this remains the active tracker for the requested feature.

## What This Changes

Add an optional provider-level `enabled` boolean that preserves configured provider blocks while excluding disabled providers from model generation, fallback selection, catalogs, and status output.

## Best Possible Solution

Add a documented provider-level flag that defaults to enabled, preserves disabled config blocks, filters disabled providers consistently from runtime selection surfaces, and shows disabled state in `openclaw models status` JSON and rich output.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: add `enabled: false` under any `models.providers.<id>` block and the strict provider schema and selection/status paths still have no provider-level disabled contract to honor it.

## Solution Assessment

Yes. The requested provider-level flag is the narrowest maintainable solution for temporary provider shutdowns, provided it is implemented across config validation, generation, fallback/catalog selection, status/probe display, docs, and regression tests rather than only as a type addition.
