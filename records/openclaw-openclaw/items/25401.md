## Summary

Keep open. Current main still lacks a provider-level disabled-provider contract, the two linked implementation PRs are closed unmerged, and no canonical merged replacement now owns the remaining work.

## What This Changes

Add a default-enabled `models.providers.<id>.enabled` flag so operators can temporarily disable a configured model provider without deleting its config block.

## Best Possible Solution

Approve a default-true provider-enabled contract, then implement it through a shared helper across schema/types, models.json planning, model selection/fallback, configured catalogs, status/probe output, embedded runner resolution, docs, and focused tests.

## Reproduction Assessment

Yes. Source inspection on current main shows the proposed key is not in the provider type/schema and the central planning, fallback, catalog, and status paths still process every configured provider; no runtime command was run because this review is read-only.

## Solution Assessment

Yes in direction, but not yet implemented. A default-true provider-level boolean is the narrowest maintainable shape, provided it is landed as a full public config/status contract rather than a type-only change.
