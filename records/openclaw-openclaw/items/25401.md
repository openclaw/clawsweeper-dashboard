## Summary

Keep open. Current main still lacks a provider-level `models.providers.<id>.enabled` contract: config validation omits the field, model planning and catalog paths include every configured provider, fallback construction has no disabled-provider filter, and `openclaw models status` has no disabled-provider state.

## What This Changes

The issue requests a default-true `models.providers.<provider>.enabled` config flag so providers can be temporarily disabled without deleting their config.

## Best Possible Solution

Approve a default-true provider-enabled contract, then implement it consistently across schema/types, models.json planning, fallback/catalog/selection, status/probe output, docs, and focused regression tests.

## Reproduction Assessment

Yes. Source inspection on current main shows `models.providers.<id>.enabled` is not accepted by the provider type/schema and the central planning, fallback, catalog, and status paths do not check disabled providers.

## Solution Assessment

Yes in direction. A default-true provider-level boolean is the narrowest maintainable shape, but it should land as a complete public config/status contract rather than a partial type-only patch.
