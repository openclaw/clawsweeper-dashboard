## Summary

Keep open. Current main still lacks a provider-level `models.providers.<id>.enabled` contract across config validation, models.json planning, fallback/model catalog selection, and `models status`, while the implementation PR that attempted this feature was closed unmerged.

## What This Changes

Add an optional `models.providers.<id>.enabled` flag so a configured model provider can be temporarily disabled without deleting its config block.

## Best Possible Solution

Add a documented provider-level flag that defaults to enabled, preserves disabled config blocks, filters disabled providers consistently from runtime selection surfaces, and exposes disabled state in `openclaw models status`.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: add `enabled: false` under any `models.providers.<id>` block and the strict provider schema plus model planning, fallback, catalog, and status paths still have no complete provider-disabled contract to honor it.

## Solution Assessment

Yes. The provider-level boolean is the narrowest maintainable direction for temporary provider shutdowns, provided it is implemented across config schema/types, runtime filtering, status/probe display, docs, and regression tests rather than only as a type addition.
