## Summary

Keep open. Current main still lacks a provider-level `models.providers.<id>.enabled` contract, and the linked implementation PR was closed unmerged; this is a public config/status feature that needs maintainer product approval before implementation.

## What This Changes

Add an optional default-true `models.providers.<id>.enabled` flag so users can temporarily disable a configured model provider without deleting its config block.

## Best Possible Solution

Approve a default-true provider enablement contract, then implement it consistently across config types/schema, models.json planning, fallback/catalog selection, status/probe output, docs, and focused regression tests.

## Reproduction Assessment

I did not run a runtime repro because this read-only review could prove the gap directly from source.

## Solution Assessment

Yes directionally, but not as automated cleanup. A provider-level default-true boolean is a maintainable shape only after maintainers approve the public config/status contract and all runtime surfaces honor it together.
