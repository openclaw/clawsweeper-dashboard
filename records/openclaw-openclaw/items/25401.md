## Summary

Keep open. Current main still lacks the requested provider-level disable contract, and both known implementation attempts are closed unmerged rather than shipped fixes.

## What This Changes

The issue requests a provider-level `models.providers.<id>.enabled` option that disables a configured model provider without deleting its config block.

## Best Possible Solution

Add a documented default-true provider-enabled contract and make all model planning, selection, catalog, auth/status, docs, and regression tests honor disabled providers consistently.

## Reproduction Assessment

Yes for the current missing capability. Source inspection on current main shows the provider type/schema omit `enabled`, and the planning, fallback, catalog, and status paths have no disabled-provider gate.

## Solution Assessment

Yes. A provider-level boolean defaulting to enabled is the narrowest maintainable direction, but it needs a full contract update rather than a type-only patch.
