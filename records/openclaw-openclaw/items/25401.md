## What This Changes

Request to add a default-true `models.providers.<id>.enabled` flag that disables a configured model provider without deleting its config.

## Best Possible Solution

Approve and document a default-true provider enablement contract, then implement it consistently across config types/schema, models.json planning, configured catalogs, fallback selection, runtime model resolution, status/probe output, docs, and focused tests.

## Reproduction Assessment

Yes for the source-level capability gap. Current main and the latest release omit `enabled` from the provider type/schema and do not carry a provider-disabled state through planning, fallback, catalog, status, or runtime model resolution.

## Solution Assessment

Yes directionally, but not as automated cleanup. A provider-level default-true boolean is coherent only after maintainers approve the public config/status contract and all runtime surfaces honor it together.
