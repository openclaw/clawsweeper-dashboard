## Summary

Keep open. Current main still lacks the requested provider-level enablement contract, all inspected runtime paths still treat configured providers as active, and the exact implementation attempts found for this request are closed unmerged.

## What This Changes

Add a default-true `models.providers.<id>.enabled` flag that can temporarily disable a configured provider while preserving its config and showing that state in model status output.

## Best Possible Solution

Approve a default-true provider enablement contract, then implement it consistently across config types/schema, models.json planning, fallback/catalog selection, status/probe output, docs, and focused regression tests.

## Reproduction Assessment

Yes for the capability gap. Source inspection on current main shows the strict provider type/schema omit `enabled`, and planning, fallback, catalog, status, and docs paths have no provider-config disabled state.

## Solution Assessment

Yes directionally, but not as automated cleanup. A provider-level default-true boolean is a coherent fit only after maintainers approve the public config/status contract and all runtime surfaces honor it together.
