## Summary

Keep open. Current main still does not define or honor a provider-level `models.providers.<id>.enabled` contract, and the linked implementation PR was closed unmerged.

## What This Changes

Requested support for a provider-level `models.providers.<id>.enabled` flag that disables a model provider while preserving its configuration.

## Best Possible Solution

Approve a default-true provider-enabled contract, then implement it consistently across schema/types, models.json planning, fallback/catalog/selection paths, status/probe output, docs, and focused regression tests.

## Reproduction Assessment

Yes. Source inspection on current main shows `models.providers.<id>.enabled` is not a provider type/schema field and the central planning, fallback, catalog, and status paths have no disabled-provider gate.

## Solution Assessment

Yes in direction. A default-true provider-level boolean is the narrowest maintainable shape, but it should land as a complete public config/status contract rather than a type-only patch.
