## Summary

Keep open. Current main still lets the missing-configured-plugin install repair choose the downloadable Feishu channel package from `channels.feishu` without applying the existing alternative-owner/preferOver arbitration; related PR #44735 covers auto-enable, not this installer path.

## What This Changes

Prevent `openclaw doctor --repair` from installing `@openclaw/feishu` when an enabled `openclaw-lark` plugin already owns `channels.feishu`.

## Best Possible Solution

Make missing-configured-plugin install repair honor the same generic channel ownership and `preferOver` arbitration before downloading channel-selected candidates, with a regression test for `channels.feishu` plus enabled `openclaw-lark`.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with `channels.feishu` configured, `plugins.entries.openclaw-lark.enabled=true`, no usable `feishu` install record, and the packaged Feishu plugin externalized, the collector can still select the `feishu` catalog candidate by channel id.

## Solution Assessment

Yes. The narrow maintainable repair is to apply the existing generic channel-owner/preferOver arbitration in the missing-install path; a hard-coded `feishu`/`openclaw-lark` skip would leave the shared replacement-plugin contract inconsistent.
