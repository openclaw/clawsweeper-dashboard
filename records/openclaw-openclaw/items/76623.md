## Summary

Keep open. Current main still lets the doctor missing-plugin install repair choose the downloadable `feishu` channel candidate from `channels.feishu` without checking that an enabled replacement plugin already owns that channel.

## What This Changes

Doctor install repair should avoid installing the downloadable Feishu plugin when an enabled Lark plugin already owns the Feishu channel config.

## Best Possible Solution

Apply the existing channel-owner and `preferOver` arbitration before installing channel-selected missing-plugin candidates, with regression coverage for `channels.feishu` plus enabled `openclaw-lark`.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configured `channels.feishu`, enabled `openclaw-lark`, and no usable `feishu` install record can still collect and install the `feishu` catalog entry.

## Solution Assessment

Yes. The narrow maintainable fix is generic owner/preferOver arbitration in the install-repair candidate path; a Feishu/Lark-only guard would solve this case but leave the shared replacement-plugin contract inconsistent.
