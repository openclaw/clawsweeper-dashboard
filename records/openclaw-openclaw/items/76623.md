## Summary

Keep open. Current main still lets doctor install repair select the downloadable `feishu` channel candidate from `channels.feishu` without checking whether an enabled alternative plugin already owns that channel.

## What This Changes

The issue asks `openclaw doctor --repair` to avoid installing `@openclaw/feishu` when enabled `openclaw-lark` already owns `channels.feishu`.

## Best Possible Solution

Extend the missing-configured-plugin install repair to honor existing channel-owner arbitration before installing channel-selected candidates, then cover the `channels.feishu` plus enabled `openclaw-lark` case with a regression test.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure `channels.feishu`, enable `plugins.entries.openclaw-lark`, leave `feishu` uninstalled, and the collector can still select the `feishu` catalog entry because `configuredChannelIds` contains `feishu`.

## Solution Assessment

Yes. The narrow maintainable fix is generic owner arbitration using the existing channel ownership and `preferOver` contract; a hard-coded `feishu` to `openclaw-lark` guard would be less durable.
