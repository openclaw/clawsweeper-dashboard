## Summary

Keep open. Current main still lets the doctor missing-configured-plugin install repair select and install the downloadable `feishu` channel package from `channels.feishu` without checking that an enabled replacement plugin already owns that channel; the related open PR only covers adjacent auto-enable behavior.

## What This Changes

Prevent doctor missing-configured-plugin install repair from installing `@openclaw/feishu` when `openclaw-lark` is already configured/enabled for `channels.feishu`.

## Best Possible Solution

Make missing-configured-plugin install repair honor the same generic channel ownership and `preferOver` arbitration as auto-enable, with a regression test for `channels.feishu` plus enabled `openclaw-lark`.

## Reproduction Assessment

Yes. Source inspection of current main gives a high-confidence path: `channels.feishu` makes `feishu` a downloadable install candidate, and the installer loop has no alternative-owner check before installing it.

## Solution Assessment

Yes. The narrow maintainable fix is generic channel-owner/`preferOver` filtering in the install-repair candidate path; a hard-coded `feishu`/`openclaw-lark` guard would solve this report but leave the shared replacement-plugin contract inconsistent.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/76623. Current main lets `repairMissingConfiguredPluginInstalls` install a channel-selected downloadable candidate such as `feishu` from `channels.feishu` even when another enabled/configured plugin already owns the same channel via `channelConfigs.<id>.preferOver`. Add a generic guard in `src/commands/doctor/shared/missing-configured-plugin-install.ts` so channel-selected install candidates are skipped when an enabled/configured alternative plugin claims the same channel and prefers over that candidate, matching `plugin-auto-enable` behavior. Preserve explicit user selection of both plugins and existing blocked/disabled-plugin behavior; do not add a Feishu-only hard-code. Add regression coverage in `src/commands/doctor/shared/missing-configured-plugin-install.test.ts` for `channels.feishu`, `plugins.entries.openclaw-lark.enabled=true`, no `plugins.entries.feishu`, no install record, mocked catalog entry for `feishu`, and manifest metadata showing `openclaw-lark` owns `feishu` with `preferOver: ["feishu"]`; assert npm install is not called and no install record is written. Keep #44735 in mind as adjacent auto-enable work, but cover this installer path. Add a user-facing `CHANGELOG.md` fix entry crediting the human reporter if appropriate under repo policy.
