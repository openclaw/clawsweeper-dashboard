## Summary

Keep open. Current main still lets the missing configured plugin install repair select the downloadable `@openclaw/feishu` candidate from `channels.feishu` without checking that `openclaw-lark` already owns the channel; the related open PR covers adjacent auto-enable behavior, not this installer path.

## What This Changes

Doctor repair should avoid installing the built-in/community Feishu plugin when an enabled official Lark plugin already handles the Feishu channel.

## Best Possible Solution

Make missing configured plugin install repair honor the same manifest channel ownership and `preferOver` arbitration used by startup and auto-enable, with regression coverage for `channels.feishu` plus enabled `openclaw-lark`.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `channels.feishu` can select the `feishu` downloadable catalog entry, and the install loop has no alternative-owner or `preferOver` guard before calling the installer.

## Solution Assessment

Yes. The narrow maintainable fix is a generic ownership/preferOver filter in the missing install repair path; a hard-coded `feishu`/`openclaw-lark` skip would fix this case but leave the shared replacement-plugin contract inconsistent.
