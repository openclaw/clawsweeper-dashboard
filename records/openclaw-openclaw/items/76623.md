## Summary

Keep open. Current main still leaves the missing-configured-plugin install repair path able to select the downloadable `@openclaw/feishu` channel package from `channels.feishu` without checking that an enabled alternative plugin already owns that channel; the related PR covers adjacent auto-enable behavior, not this installer path.

## What This Changes

Prevent `openclaw doctor --repair` from installing `@openclaw/feishu` when enabled `openclaw-lark` is already handling `channels.feishu`.

## Best Possible Solution

Make missing-configured-plugin install repair honor the same manifest/channel ownership and `preferOver` arbitration used by startup and auto-enable, with a regression test for `channels.feishu` plus enabled `openclaw-lark`.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with `channels.feishu` configured, `plugins.entries.openclaw-lark.enabled=true`, no usable `feishu` install record, and packaged `feishu` excluded from bundled dist, the current candidate collector can still select the `feishu` catalog entry by channel id.

## Solution Assessment

Yes. The narrow maintainable fix is to apply the existing generic channel-owner/preferOver arbitration before installing channel-selected candidates; a hard-coded `feishu`/`openclaw-lark` guard would cover this report but would leave the shared replacement-plugin contract inconsistent.
