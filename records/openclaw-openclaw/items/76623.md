## Summary

Keep open. Current main still lets the missing-configured-plugin install repair path select the downloadable `@openclaw/feishu` candidate from `channels.feishu` without checking that an enabled replacement plugin already owns that channel.

## What This Changes

Doctor repair should avoid installing or enabling the built-in/community Feishu plugin when `@larksuite/openclaw-lark` is active for `channels.feishu`.

## Best Possible Solution

Make missing-configured-plugin install repair apply generic channel-owner and `preferOver` arbitration before installing channel-selected candidates, with regression coverage for `channels.feishu` plus enabled `openclaw-lark`.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configured `channels.feishu` can select the `feishu` catalog candidate, and the install loop lacks an enabled-alternative-owner skip before installing it.

## Solution Assessment

Yes. The narrow maintainable solution is to reuse the existing generic channel ownership and `preferOver` contract; a hard-coded `feishu`/`openclaw-lark` guard would fix this case but leave the repair path inconsistent with the rest of plugin activation.
