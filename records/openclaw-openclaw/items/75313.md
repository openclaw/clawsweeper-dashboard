## Summary

Keep this issue open. Current main still removes blank-only user messages during session-file repair, and the linked fix PR #75368 is open and unmerged, so this issue should stay open until that implementation or an equivalent lands.

## What This Changes

The issue asks for session-file repair to rewrite blank-only user-role messages to a non-empty placeholder instead of dropping them from persisted JSONL history.

## Best Possible Solution

Land PR #75368 or an equivalent narrow change that rewrites blank-only user session entries to a neutral non-empty placeholder, keeps media-block cleanup behavior intact, updates the existing regression test away from destructive drops, and adds coverage for a session that would otherwise repair to assistant-only history.

## Reproduction Assessment

Yes. The current helper and unit test provide a high-confidence source reproduction: a persisted blank user entry is removed during repair, and the repair call runs before session replay for attempts and compaction.

## Solution Assessment

Yes. The narrowest maintainable fix belongs in `src/agents/session-file-repair.ts` with focused tests; a provider-specific OpenAI-compatible transport guard should only be added if review shows file repair does not cover the failing path.
