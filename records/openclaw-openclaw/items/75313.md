## Summary

Keep this issue open. Current main still drops blank-only user session entries during session-file repair, and the linked fix PR #75368 is open and unmerged, so the issue should remain open until that PR or an equivalent change lands.

## What This Changes

Session-file repair should preserve blank-only user-role session entries instead of dropping them, so repaired transcripts do not become assistant-only histories rejected by strict OpenAI-compatible providers.

## Best Possible Solution

Land PR #75368 or an equivalent narrow change in `src/agents/session-file-repair.ts` that rewrites blank-only user entries to a neutral non-empty placeholder, preserves media-block cleanup behavior, updates the existing regression test, and adds coverage for a repaired transcript that would otherwise contain no user role.

## Reproduction Assessment

Yes. The current helper and unit test provide a high-confidence source reproduction: a persisted blank user entry is removed during repair, and that repair runs before session replay for embedded attempts and compaction.

## Solution Assessment

Yes. The best fix is the narrow session-file repair change proposed by the linked PR; a provider-specific transport guard should only be added if review shows file repair does not cover a real remaining path.
