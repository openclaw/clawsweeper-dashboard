Current main still clears followup queue state during gateway session reset and does not rebind queued followups to the replacement session, so the PR is not obsolete; however, the PR head is not mergeable as-is because it has compile-blocking merge artifacts and stale reset-entry logic that would drop persisted session settings.

## What This Changes

The PR preserves followup queue work through gateway session resets, rebinds queued and in-flight reply runs to replacement sessions, and suppresses stale streamed or final deliveries after session supersession.
