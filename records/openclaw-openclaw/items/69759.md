## Summary

Keep open. Current main still counts every session `.jsonl` file in `openclaw memory status`, while the indexing path and artifact tests exclude exact compaction checkpoint snapshots from usage-counted session transcripts; the related open PR handles QMD behavior and does not fix this CLI denominator mismatch.

## What This Changes

The issue asks for `openclaw memory status` to exclude compaction checkpoint session transcript files from the indexed file-count denominator.

## Best Possible Solution

Align the memory status session denominator with the usage-counted session transcript contract and add focused CLI status coverage for a primary transcript plus checkpoint snapshot.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: place one primary session transcript and one exact `<session>.checkpoint.<uuid>.jsonl` in an agent sessions directory; status scanning counts both while indexing lists only usage-counted transcripts.

## Solution Assessment

Yes. Reusing the existing usage-counted transcript predicate for the status scan is the narrow maintainable fix; indexing checkpoint snapshots would double-index derived compaction artifacts.
