## Summary

Keep open. Current main still counts every session `.jsonl` file for the `openclaw memory status` denominator, while the session indexing/export path already filters out compaction checkpoint snapshots, so the reported mismatch remains valid.

## What This Changes

Make `openclaw memory status` report session file totals consistently by excluding compaction checkpoint `.jsonl` snapshots from the status denominator.

## Best Possible Solution

Have memory status reuse the existing usage-counted transcript predicate for session totals, with focused CLI status regression coverage and a user-facing changelog fix.

## Reproduction Assessment

Yes. Create one normal session `.jsonl` and one `<session>.checkpoint.<uuid>.jsonl` in an agent sessions directory; current source shows `scanSessionFiles` counts both while the indexing file list excludes the checkpoint.

## Solution Assessment

Yes. Reusing the existing session artifact predicate is the narrow maintainable fix; indexing checkpoint snapshots would broaden the memory corpus and conflict with current checkpoint handling.
