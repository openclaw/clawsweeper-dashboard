## Summary

Keep open. Current main and the latest shipped release still count every session `.jsonl` file in the memory status denominator, while the session indexing path uses the usage-counted transcript predicate that excludes compaction checkpoints.

## What This Changes

The issue asks for `openclaw memory status` to exclude compaction checkpoint transcripts from the session file-count denominator so indexed totals match the indexed corpus.

## Best Possible Solution

Align the memory status session denominator with the existing usage-counted transcript contract, and add focused CLI status coverage for primary plus checkpoint transcript files.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: put one primary session transcript and one exact `<session>.checkpoint.<uuid>.jsonl` in an agent sessions directory; status scanning counts both while indexing lists only usage-counted transcripts.

## Solution Assessment

Yes. Reusing the existing usage-counted session transcript predicate is the narrow maintainable fix; indexing checkpoint snapshots instead would broaden the corpus and risk double-indexing derived compaction snapshots.
