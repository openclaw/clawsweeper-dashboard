## Summary

Keep open. Current main still has the reported mismatch: memory status counts raw session `.jsonl` files for the denominator, while the indexing/export corpus already filters session files through the usage-counted transcript predicate that excludes compaction checkpoints.

## What This Changes

The issue requests excluding compaction checkpoint session snapshots from `openclaw memory status` session-file totals so displayed indexed counts match the files indexing actually considers.

## Best Possible Solution

Align the memory status denominator with the existing usage-counted session transcript contract and cover that behavior in focused CLI status tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create one normal session `.jsonl` and one `<session>.checkpoint.<uuid>.jsonl` in an agent sessions directory; current status scanning counts both while indexing lists only usage-counted transcripts.

## Solution Assessment

Yes. Reusing the existing session artifact predicate is the narrow maintainable fix; indexing checkpoint snapshots would broaden the corpus and conflict with the current checkpoint design.
