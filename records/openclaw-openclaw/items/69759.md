## Summary

Keep open. Current main still has the reported mismatch: `openclaw memory status` counts all session `.jsonl` files for the denominator while the indexing path filters session transcripts through the usage-counted predicate that excludes compaction checkpoints.

## What This Changes

The issue asks `openclaw memory status` to exclude compaction checkpoint `.jsonl` snapshots from the session-file denominator so indexed/total counts stay consistent.

## Best Possible Solution

Align the memory status denominator with the existing usage-counted session transcript predicate and cover checkpoint snapshots in focused CLI status output tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: put one primary session transcript and one exact `<session>.checkpoint.<uuid>.jsonl` in an agent sessions directory; status scanning counts both while indexing lists only usage-counted transcripts.

## Solution Assessment

Yes. Reusing the existing usage-counted predicate is the narrow maintainable solution; indexing checkpoint snapshots would broaden the corpus and risks double-indexing derived compaction snapshots.
