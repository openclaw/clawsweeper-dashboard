## Summary

Keep open: the reported Feishu config bug is still present on current main, but this PR is stale against the active resolver location and now overlaps an open semantics alternative in #73958, so it is not mergeable as-is.

## What This Changes

The PR changes Feishu group session resolution and one bot test so non-topic group inbound thread metadata no longer enables thread replies when `replyInThread` is disabled.

## Best Possible Solution

Land one coordinated Feishu-plugin patch that updates `resolveFeishuGroupSession` in `bot-content.ts`, separates session topic anchoring from outbound `reply_in_thread`, preserves topic/direct-send delivery, updates regression tests, and adds the required changelog entry.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists on current main: configure a normal Feishu group with `replyInThread: "disabled"`, provide inbound `thread_id`, and `resolveFeishuGroupSession` still returns `replyInThread: true` and `threadReply: true`.

## Solution Assessment

No. The PR direction is plausible, but this branch edits the moved resolver, can drop topic-thread delivery for direct sends, and needs reconciliation with #73958 before merge.
