## Summary

Keep open: current main still has a source-reproducible Feishu disabled-thread bug, but this branch is stale against the active resolver split and overlaps an unsettled root_id/thread_id semantics decision now also tracked in https://github.com/openclaw/openclaw/pull/73958.

## What This Changes

The PR changes Feishu group-session routing and one bot test so non-topic quote-reply metadata no longer forces thread replies when `replyInThread` is disabled.

## Best Possible Solution

Land one current-main Feishu-plugin fix that updates `bot-content.ts` and dispatcher behavior together, preserves topic direct-send routing, and explicitly resolves the `root_id` versus `thread_id` semantics alongside https://github.com/openclaw/openclaw/pull/73958.

## Reproduction Assessment

Yes. Source inspection on current main shows inbound `threadId` or `rootId` still forces `replyInThread` true despite disabled config, and the current bot test asserts that behavior for a disabled non-topic group with inbound `thread_id`.

## Solution Assessment

No. The direction is plausible for disabled-as-hard-off semantics, but this branch edits the stale resolver location, can drop thread delivery for topic direct sends, and needs reconciliation with https://github.com/openclaw/openclaw/pull/73958 before it is the best fix.
