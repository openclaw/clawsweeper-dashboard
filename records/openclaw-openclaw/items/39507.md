## What This Changes

The PR changes Feishu group-session resolution and one bot test so quote-reply metadata no longer forces thread replies when `replyInThread` is disabled.

## Best Possible Solution

Land one current-main Feishu-plugin fix in `bot-content.ts` that separates session topic anchoring from outbound `reply_in_thread`, preserves topic/direct-send routing, updates regression tests and changelog, and resolves #73958's root/thread semantics explicitly.

## Reproduction Assessment

Yes. Source inspection on current main shows `threadId`/`rootId` still force `replyInThread` true despite disabled config, and the existing bot test locks in that behavior for a normal group with inbound `thread_id`.

## Solution Assessment

No. The PR direction is plausible, but this branch is not the best fix as written because it patches the stale resolver location, can break topic-thread direct sends, and needs reconciliation with #73958 before merge.
