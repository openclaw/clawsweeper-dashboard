## Summary

Keep open: the reported Feishu behavior is still source-reproducible on current main, but this PR is stale against the current resolver layout and conflicts with the open root/thread semantics work in #73958.

## What This Changes

The PR changes Feishu group session resolution and one bot test so non-topic group inbound thread metadata no longer enables thread replies when `replyInThread` is disabled.

## Best Possible Solution

Land one coordinated Feishu-plugin patch that updates `resolveFeishuGroupSession` in `bot-content.ts`, separates session topic anchoring from outbound `reply_in_thread`, preserves topic/direct-send delivery, updates regression tests, and adds the required changelog entry.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: configure a normal Feishu group with `replyInThread: "disabled"`, provide inbound `thread_id`, and the resolver/test path still produces `replyInThread: true` and `threadReply: true`.

## Solution Assessment

No. The PR direction is plausible, but this branch is not the best fix as written because it edits the moved resolver, can drop topic-thread delivery for direct sends, and needs reconciliation with #73958 before merge.
