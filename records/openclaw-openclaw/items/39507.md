## Summary

Keep open: the reported Feishu disabled-mode bug is real, but this PR is stale and not correct as-is because it edits the old resolver location while current main resolves sessions in `bot-content.ts`; the remaining semantics also overlap with the open root/thread routing discussion in #73958.

## What This Changes

The PR changes Feishu group session resolution and a bot test so `replyInThread: "disabled"` suppresses inbound root/thread metadata from forcing thread replies in non-topic groups.

## Best Possible Solution

Land one coordinated Feishu plugin patch that fixes the active resolver, separates session topic anchoring from outbound `reply_in_thread`, preserves topic/direct-send delivery, reconciles #73958, updates regression tests, and adds the required changelog entry.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: configure a normal Feishu group with `replyInThread: "disabled"`, provide inbound `thread_id`, and the active resolver/test path still produces `replyInThread: true` and `threadReply: true`.

## Solution Assessment

No. The desired disabled-mode fix is valid, but this branch edits the stale resolver location and needs a maintainer decision against #73958's root-preserving semantics before it can be the best fix.
