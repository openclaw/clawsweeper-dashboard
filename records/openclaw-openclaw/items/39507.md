## What This Changes

The branch changes Feishu group-session routing so non-topic `replyInThread: "disabled"` ignores inbound `thread_id`/`root_id` and updates the matching bot test expectation.

## Best Possible Solution

Refresh or replace this with one Feishu plugin patch after maintainers choose hard-off versus root-preserving semantics; implement it in `bot-content.ts` and the dispatcher/direct-send boundary with focused regression coverage for normal groups, topic scopes, and fallback behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows `resolveFeishuGroupSession` sets `threadReply` from `threadId || rootId` and ORs it into `replyInThread`, and the current disabled-config test expects both dispatcher flags to be true.

## Solution Assessment

No. The requested direction is plausible, but this branch edits a stale resolver location and can break topic direct-send thread delivery; a current-main fix should update the active helper and outbound boundary after the semantics choice.
