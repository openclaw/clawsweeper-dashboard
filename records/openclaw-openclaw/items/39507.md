## Summary

Keep this PR open: it targets a real Feishu bug, but the branch is stale against current main, still lacks artifact-backed real behavior proof, and needs maintainer agreement on `root_id` versus `thread_id` semantics before it is mergeable.

## What This Changes

The PR changes Feishu group-session resolution and one bot test so disabled non-topic group quote-reply metadata no longer sets `replyInThread` or `threadReply` true.

## Best Possible Solution

Land one current-main Feishu plugin fix that updates `bot-content.ts`, dispatcher/direct-send routing, and regression tests after maintainers choose the canonical `root_id` and `thread_id` semantics.

## Reproduction Assessment

Yes. Source inspection on current main shows inbound `threadId` or `rootId` still forces `replyInThread` true despite disabled config, and the current bot test asserts that behavior for a disabled non-topic group with inbound `thread_id`.

## Solution Assessment

No. The direction is plausible for disabled-as-hard-off semantics, but this branch edits a stale resolver location, risks topic direct-send behavior, and needs reconciliation with https://github.com/openclaw/openclaw/pull/73958 before it is the best fix.
