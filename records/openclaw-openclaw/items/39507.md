## Summary

Keep open: the underlying Feishu bug is still source-reproducible on current main, but this PR is not merge-ready because it is conflicting, edits the pre-split resolver location, has a topic direct-send regression risk, lacks acceptable real behavior proof, and overlaps an open replacement semantics path.

## What This Changes

The PR changes Feishu group-session routing and one bot test so quote-reply metadata no longer forces thread replies when `replyInThread` is disabled.

## Best Possible Solution

Land one current-main Feishu plugin fix that updates `bot-content.ts` and `reply-dispatcher.ts` together, explicitly chooses the `root_id` versus `thread_id` semantics, preserves topic/direct-send routing, and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows inbound `threadId` or `rootId` still forces `replyInThread` true despite disabled config, and the current bot test asserts that behavior for a disabled non-topic group with inbound `thread_id`.

## Solution Assessment

No. The direction is plausible for disabled-as-hard-off semantics, but this branch edits the stale resolver location, can drop thread delivery for topic direct sends, and needs reconciliation with https://github.com/openclaw/openclaw/pull/73958 before it is the best fix.
