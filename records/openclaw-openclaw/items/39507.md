## Summary

Keep open: the underlying Feishu disabled-mode bug is still present on current main, but this branch is stale against the moved resolver and conflicts with an open root/thread semantics alternative, so it should not merge as-is.

## What This Changes

The PR changes Feishu group session resolution and one bot test so `replyInThread: "disabled"` no longer follows inbound `thread_id`/`root_id` in non-topic groups while preserving topic-session `threadReply`.

## Best Possible Solution

Land one coordinated Feishu-plugin patch that updates the active `bot-content.ts` resolver, separates session topic anchoring from outbound `reply_in_thread`, preserves topic/direct-send delivery, refreshes regression tests, and adds the required changelog entry.

## Reproduction Assessment

Yes, statically. Current main with a normal Feishu group, `replyInThread: "disabled"`, and inbound `thread_id` still resolves `replyInThread: true` and `threadReply: true`, and the current bot test expects that behavior.

## Solution Assessment

No. The PR direction is plausible, but this branch edits stale resolver code, can drop topic-thread delivery for direct sends, and needs reconciliation with the open #73958 semantics path before merge.
