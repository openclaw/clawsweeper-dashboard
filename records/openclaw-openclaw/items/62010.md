## Summary

Keep open. Current main does not have this WhatsApp config option, and the PR remains potentially useful, but the latest head is not merge-ready because configured roots stay outside the loader’s preferred `mediaAccess` path, `file://` roots are accepted without normalization, real WhatsApp send proof is missing, and the branch is conflicting with current main.

## What This Changes

The PR adds a WhatsApp-scoped `mediaLocalRoots` config field, schema validation, generated config baseline updates, and outbound delivery tests for channel/account/default-account root merging.

## Best Possible Solution

Keep the feature WhatsApp-scoped or move it into a generic configured-root resolver, normalize configured roots into canonical `mediaAccess.localRoots`, preserve strict root validation, and add real WhatsApp media-send proof before merge.

## Reproduction Assessment

Yes, for the PR defects by source inspection. Configure existing `mediaAccess.localRoots` plus `channels.whatsapp.mediaLocalRoots`; the PR passes the configured root only through `mediaLocalRoots`, while current WhatsApp media loading can prefer `mediaAccess.localRoots` and ignore the fallback.

## Solution Assessment

No. The direction is useful, but the current patch should merge normalized configured roots into the canonical media-access path and avoid hard-coding WhatsApp policy in shared outbound delivery unless maintainers choose that product boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Merge configured roots into mediaAccess:** `src/infra/outbound/deliver.ts:304-305`
  - body: The patch builds a merged WhatsApp root list but still returns `mediaAccess: params.mediaAccess`. Current WhatsApp media loading prefers `mediaAccess.localRoots` whenever present, so files under the new configured root can still be rejected in normal outbound sends. Merge the normalized roots into the `mediaAccess.localRoots` object, or make the loader combine both inputs consistently.
  - confidence: 0.93
- **[P2] Normalize or reject file URL roots:** `src/config/zod-schema.providers-whatsapp.ts:42`
  - body: The schema accepts `file://` roots, but media URLs are converted to filesystem paths before allowlist checks while configured root entries are compared as raw paths. A root like `file:///tmp/wa-media` validates but will not match `/tmp/wa-media/photo.png`, so normalize root entries before delivery or reject `file://` roots here.
  - confidence: 0.89
- **[P2] Keep WhatsApp root policy out of core delivery:** `src/infra/outbound/deliver.ts:273-288`
  - body: This adds WhatsApp-specific config and account resolution inside the shared outbound delivery context. The repo boundary keeps owner-specific channel policy in the owning plugin or behind a generic channel contract, so this merge should live in the WhatsApp outbound path or a shared configured-root resolver instead.
  - confidence: 0.84
