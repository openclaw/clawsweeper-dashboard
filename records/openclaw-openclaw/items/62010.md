## Summary

Keep open: current main still lacks the requested WhatsApp config field, and the PR head still has blocking correctness/boundary issues plus no real after-fix WhatsApp media-send proof.

## What This Changes

The PR adds a WhatsApp `mediaLocalRoots` config option, schema and generated-config updates, and outbound delivery tests that merge channel/account roots into WhatsApp media send options.

## Best Possible Solution

Implement WhatsApp-scoped root resolution in the WhatsApp plugin path or a generic channel contract, normalize configured roots to filesystem paths, merge them into the canonical media access consumed by the loader, and require redacted live WhatsApp media-send proof.

## Reproduction Assessment

Yes, for the PR defects by source inspection. PR head passes configured roots only through `mediaLocalRoots` while the WhatsApp loader prefers `mediaAccess.localRoots`, and `file://` roots validate without being normalized to the filesystem paths used for allowlist checks.

## Solution Assessment

No. The feature direction is plausible, but the narrow maintainable fix should normalize roots and merge them into the canonical media access in the WhatsApp-owned path or behind a generic channel contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Merge configured roots into mediaAccess:** `src/infra/outbound/deliver.ts:304-305`
  - body: The patch computes a merged WhatsApp root list but still returns `mediaAccess: params.mediaAccess`. The WhatsApp loader prefers `mediaAccess.localRoots` whenever present and only falls back to `mediaLocalRoots`, so sends with existing media access roots can still reject files under the new configured root.
  - confidence: 0.93
- **[P2] Normalize or reject file URL roots:** `src/config/zod-schema.providers-whatsapp.ts:42`
  - body: This schema accepts `file://` roots, but media URLs are converted to filesystem paths before allowlist checks while root entries are resolved as raw paths. A root like `file:///tmp/wa-media` validates but will not match `/tmp/wa-media/photo.png`.
  - confidence: 0.89
- **[P2] Keep WhatsApp root policy in the plugin:** `src/infra/outbound/deliver.ts:273-291`
  - body: This adds a WhatsApp-specific config branch to core outbound delivery. OpenClaw's channel boundary keeps owner-specific channel policy in the owning plugin or a generic SDK seam, so the root merge should live in the WhatsApp outbound path or a channel contract instead.
  - confidence: 0.86
