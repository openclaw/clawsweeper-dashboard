## Summary

Keep open. The PR is a useful direction, but the current patch still routes configured WhatsApp roots through a side `mediaLocalRoots` field that can be ignored by the WhatsApp media loader, lacks after-fix real WhatsApp proof, and expands trusted host media roots enough to need maintainer security/product review.

## What This Changes

The PR adds a WhatsApp `mediaLocalRoots` config field, schema validation and generated config baseline updates, plus outbound delivery tests and channel/account root merging.

## Best Possible Solution

Route normalized configured channel/account roots through the shared outbound media-access resolver or a WhatsApp-owned resolver seam, keep strict schema validation, and require redacted real WhatsApp media-send proof before merge.

## Reproduction Assessment

Yes, for the PR defects by source inspection: current main builds `mediaAccess` before channel send, and the WhatsApp media loader prefers `mediaAccess.localRoots` over `mediaLocalRoots`. No live WhatsApp send was run in this read-only review.

## Solution Assessment

No. The requested capability may be useful, but this patch should merge normalized roots into the canonical media-access path instead of adding a WhatsApp-specific side list in shared delivery.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Merge configured roots into mediaAccess:** `src/infra/outbound/deliver.ts:304-305`
  - body: The patch only puts WhatsApp config roots in `mediaLocalRoots` while returning `mediaAccess` unchanged. `sendMessageWhatsApp` passes both values, and `loadOutboundMediaFromUrl` prefers `mediaAccess.localRoots` whenever it is present, so sends without a `readFile` override can still reject files under the configured roots.
  - confidence: 0.92
- **[P2] Normalize configured file roots before merging:** `src/config/zod-schema.providers-whatsapp.ts:42`
  - body: The schema accepts `file://` roots, but the merge stores those raw URL strings. `loadWebMedia` converts the media URL to a filesystem path before checking roots, so `file:///tmp/wa-media` will not allow `/tmp/wa-media/photo.png`. Normalize file URLs to paths before merging, or reject them.
  - confidence: 0.89
- **[P2] Keep WhatsApp account policy out of shared delivery:** `src/infra/outbound/deliver.ts:273-280`
  - body: `createChannelOutboundContextBase` now imports account helpers and reads `channels.whatsapp` directly inside the shared outbound delivery module. This hard-codes one bundled channel into core delivery policy; the configured-root lookup should live in shared media access or a channel-owned seam so other channels do not need duplicate branches.
  - confidence: 0.81

## Security Review

Status: needs_attention

Summary: The diff expands a configured host-filesystem read allowlist and should route normalized roots through canonical media-access policy before merge.
