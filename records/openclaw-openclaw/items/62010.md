## Summary

Keep open: the feature is plausible, but the current PR head still has blocking correctness issues in the media-root plumbing and lacks after-fix real behavior proof from a real WhatsApp send.

## What This Changes

The PR adds WhatsApp `mediaLocalRoots` config/types/schema, generated config baseline updates, and outbound delivery tests that pass channel/account configured roots into WhatsApp media send options.

## Best Possible Solution

Keep the feature WhatsApp-scoped, normalize configured roots into filesystem paths, and feed them into the `mediaAccess.localRoots` allowlist consumed by the WhatsApp loader; maintainer-owned changelog work can happen during landing.

## Reproduction Assessment

Yes. From source inspection, configure `mediaAccess.localRoots` to one directory and `channels.whatsapp.mediaLocalRoots` to another; the PR forwards the extra root only as `mediaLocalRoots`, while the WhatsApp loader prefers `mediaAccess.localRoots` and ignores the fallback list.

## Solution Assessment

No. The current patch splits roots across two option channels; the safer fix is to merge normalized WhatsApp roots into the canonical media-access local root list, or make the loader combine both sources consistently.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Merge configured roots into mediaAccess:** `src/infra/outbound/deliver.ts:305`
  - body: The PR leaves `mediaAccess` unchanged while adding configured WhatsApp roots only to `mediaLocalRoots`. When `mediaAccess.localRoots` is already present, the WhatsApp media loader uses that list first and never falls back to `mediaLocalRoots`, so files under the new channel/account roots can still be rejected.
  - confidence: 0.93
- **[P2] Normalize or reject file URL roots:** `src/config/zod-schema.providers-whatsapp.ts:42`
  - body: The schema accepts `file://` roots, but media URLs are converted to filesystem paths before allowlist checks while root entries are compared as raw strings. A root such as `file:///tmp/wa-media` validates but will not match `/tmp/wa-media/photo.png`; normalize configured roots before matching or reject file URLs.
  - confidence: 0.89
