Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Route configured roots through media access policy:** `src/infra/outbound/deliver.ts:282-290`
  - body: This appends WhatsApp-configured roots after `resolveAgentScopedOutboundMediaAccess` has decided whether host media reads are allowed. When that resolver withholds a `readFile` capability because sender/group/tool policy denies host reads, the extra `mediaLocalRoots` can still be passed to WhatsApp media loading and allow reads from those configured directories outside the shared policy gate.
  - confidence: 0.88
- **[P2] Normalize file URL roots before accepting them:** `src/config/zod-schema.providers-whatsapp.ts:38-43`
  - body: The schema accepts `file://` roots, but the local-root checker resolves root entries with filesystem path logic, so `file:///tmp/wa-media` will not match `/tmp/wa-media/...`. Convert file URLs to paths in the root resolver or reject them to avoid a config value that validates but silently does not authorize the intended directory.
  - confidence: 0.86
