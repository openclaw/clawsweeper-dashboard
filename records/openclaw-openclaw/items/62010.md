## Summary

Keep open. The latest PR update fixes the earlier default-account, validation, and generated-baseline concerns, but the core feature still does not work in normal WhatsApp media delivery because the merged roots are not propagated into the canonical `mediaAccess.localRoots` path that the WhatsApp media loader prefers.

## What This Changes

The PR adds WhatsApp channel/account `mediaLocalRoots` config, schema validation and generated config baseline updates, outbound root merging, and focused config/outbound regression tests.

## Best Possible Solution

Revise the PR so configured WhatsApp roots are merged into the media access object consumed by real media loaders, normalize or reject `file://` root entries consistently, rebase onto current main, and rerun the focused outbound/config tests plus the appropriate changed gate.

## Reproduction Assessment

Yes. Configure a WhatsApp media send with normal scoped `mediaAccess.localRoots` plus `channels.whatsapp.mediaLocalRoots` outside those roots; the PR passes both fields onward, but the real WhatsApp media runtime selects `mediaAccess.localRoots` and drops the configured root.

## Solution Assessment

No. The feature direction is reasonable, but the implementation should use one canonical root list for both `mediaAccess.localRoots` and `mediaLocalRoots`, or merge both inside the WhatsApp media loader, instead of keeping diverging root channels.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Merge roots into mediaAccess before sending:** `src/infra/outbound/deliver.ts:305`
  - body: The PR only exposes the configured roots through `mediaLocalRoots`, while leaving `mediaAccess` unchanged. Real WhatsApp sends pass both fields into `loadOutboundMediaFromUrl`, and that runtime prefers `mediaAccess.localRoots` whenever present, so normal media delivery still ignores the configured WhatsApp roots and rejects files outside the managed roots.
  - confidence: 0.93
- **[P2] Reject or normalize file URL roots:** `src/config/zod-schema.providers-whatsapp.ts:42-43`
  - body: The schema accepts `file://` entries as roots, but the local media allowlist treats root entries as filesystem paths and never converts root file URLs before `fs.realpath`/`path.resolve`. A config like `file:///tmp/wa-media` validates but cannot match a media path under `/tmp/wa-media`, so either reject file URL roots or normalize them before they reach the media loader.
  - confidence: 0.86
