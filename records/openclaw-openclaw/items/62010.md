## Summary

Keep open. The branch still misses the canonical media-access path, has a file-url root normalization bug, lacks real behavior proof, and overlaps active generic local-media-root work that needs maintainer security/product judgment.

## What This Changes

The PR adds WhatsApp `mediaLocalRoots` config, schema validation, generated config baseline updates, and outbound delivery tests/root merging for channel and account scopes.

## Best Possible Solution

Define one canonical configured local-media-root contract, validate and normalize roots into shared media access, keep channel-specific account policy in owner seams or a generic resolver, and require redacted real WhatsApp media-send proof.

## Reproduction Assessment

Yes, for the PR defects by source inspection: current main builds `mediaAccess` through `resolveAgentScopedOutboundMediaAccess`, and the PR only augments `mediaLocalRoots` while WhatsApp loading prefers `mediaAccess.localRoots`. No live WhatsApp send was run in this read-only review.

## Solution Assessment

No. The requested direction may be useful, but the patch should feed normalized configured roots into the canonical media-access path or a WhatsApp-owned seam instead of a WhatsApp-specific side list in shared delivery.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Merge configured roots into mediaAccess:** `src/infra/outbound/deliver.ts:305`
  - body: The merged WhatsApp roots are passed only as `mediaLocalRoots`; `mediaAccess` remains `params.mediaAccess`. The WhatsApp media loader prefers `mediaAccess.localRoots` whenever present, and current delivery always builds that object through `resolveAgentScopedOutboundMediaAccess`, so configured roots can still be rejected in normal sends.
  - confidence: 0.93
- **[P2] Normalize file URL roots before allowing them:** `src/config/zod-schema.providers-whatsapp.ts:42`
  - body: This accepts `file://` root entries, but `loadWebMedia` converts media URLs to filesystem paths before checking roots. A configured root like `file:///tmp/wa-media` will not match `/tmp/wa-media/photo.png`, so normalize those roots into paths before merging or reject them.
  - confidence: 0.89
- **[P2] Move WhatsApp root lookup out of shared delivery:** `src/infra/outbound/deliver.ts:273-280`
  - body: Shared outbound delivery now imports WhatsApp account helpers and reads `channels.whatsapp` directly. That leaves configured-root behavior owned by a WhatsApp-only branch in core rather than the shared media-access resolver, so adjacent outbound paths and future channels need duplicate policy.
  - confidence: 0.82
