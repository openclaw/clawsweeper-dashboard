## Summary

Keep open. Current main has partial startup deferral and lazy-cron fixes, but the remaining cron/channel race is still source-reproducible because cron is not gated on delivery transport readiness and hot reload still starts cron before restarting affected channels.

## What This Changes

Ensure cold-start and config hot-reload cron execution waits for delivery-capable channel transports, especially Discord, before due LLM or announcement jobs run.

## Best Possible Solution

Add a bounded generic delivery-readiness gate at the gateway/cron boundary for cold startup and hot reload, using channel status/health signals rather than sidecar handoff alone.

## Reproduction Assessment

Yes, at source level. A configured Discord account whose startAccount lifecycle has not reached gateway.isConnected can coexist with cron.start running missed or due agent jobs after sidecar handoff, and config.patch reload starts cron before restarting affected channels.

## Solution Assessment

No. Current main reduces the window with startup deferral and a fixed missed-agent delay, but the narrow maintainable fix is a bounded delivery-readiness gate plus hot-reload ordering coverage.
