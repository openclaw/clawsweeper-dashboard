Keep open: this PR is still the live candidate fix for the linked Feishu message-tool media upload failure. Current main and v2026.6.10 still delegate SDK multipart Buffer uploads without normalization, while this head adds a narrow wrapper-boundary FormData conversion, fixes the prior test type blocker, and has green exact-head checks.

## What This Changes

Normalizes Feishu SDK multipart Buffer upload requests to native FormData inside the timeout-aware HTTP wrapper and adds a regression test for file upload request delegation.

## Best Possible Solution

Land the narrow wrapper-boundary FormData normalization with the regression test and live Feishu proof, then close the linked issue after merge.

## Solution Assessment

Yes. The HTTP wrapper is the narrow layer where OpenClaw diverges from the standalone SDK upload path, and the PR preserves the canonical SDK upload APIs instead of adding a downstream workaround.
