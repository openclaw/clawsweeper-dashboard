## What This Changes

Fix `sessions_yield` on the MCP/claude-cli gateway loopback path so it yields with active session context instead of returning `No session context`.

## Best Possible Solution

Review and validate the linked gateway fix PR, or replace it with an equivalent narrow fix that binds MCP `sessions_yield` to the active run and proves the real claude-cli loopback flow.

Yes. Source inspection on current main shows MCP loopback tools are resolved without `sessionId` or `onYield`, and the `sessions_yield` implementation deterministically returns `No session context` when `sessionId` is missing.

## Solution Assessment

Yes at the issue direction level. The maintainable fix is to pass run-bound yield context through the MCP loopback path with MCP-specific pause semantics; the open PR is the current vehicle but still needs revision and real proof.
