## Summary

Keep open: current main still constructs the MCP/claude-cli loopback tool surface without the run-bound yield context that `sessions_yield` requires, and the linked closing PR is still open with review/proof gaps.

## What This Changes

The issue reports that `sessions_yield` always returns `No session context` on the MCP/claude-cli runtime path because gateway loopback tool resolution does not provide `sessionId` and `onYield`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/77437 after it binds MCP `sessions_yield` to the active run and proves a real claude-cli loopback flow, or replace it with an equivalent narrow fix.

## Reproduction Assessment

Yes, source-reproducible. Current main builds MCP loopback tools without `sessionId` or `onYield`, and the tool implementation deterministically returns `No session context` when `sessionId` is absent.

## Solution Assessment

Yes for the issue direction. The maintainable fix is to pass run-bound yield context through the MCP loopback path with MCP-specific pause semantics; the current linked PR remains the active vehicle but still needs revision and real proof.
