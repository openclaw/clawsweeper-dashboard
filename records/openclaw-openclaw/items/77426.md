## Summary

Keep open: current main still has the source-reproducible MCP/claude-cli failure path, and open PR #77437 already references this issue with closing syntax, so the issue should stay open until that PR is reviewed and merged or replaced.

## What This Changes

Fix `sessions_yield` on the MCP/claude-cli gateway runtime path so it receives run context and can yield instead of returning `No session context`.

## Best Possible Solution

Review and land a narrow fix like #77437 that passes run-bound yield context into loopback tools and records MCP/CLI yield state without reusing embedded-runner abort semantics.

## Reproduction Assessment

Yes. Source inspection shows MCP `tools/call` resolves `sessions_yield` through the gateway loopback cache without `sessionId` or `onYield`, so the tool deterministically reaches the `No session context` branch.

## Solution Assessment

Yes at the direction level. The narrow maintainable fix is to add run-bound MCP yield context in the shared loopback tool path; channel-specific workarounds would miss the gateway-level failure.
