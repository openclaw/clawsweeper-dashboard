## Summary

Keep open: current main still has the source-reproducible MCP/claude-cli `sessions_yield` error path, and open PR #77437 is already linked as the implementation candidate, so this issue should stay open until that PR merges or is replaced.

## What This Changes

Reported regression where `sessions_yield` returns `No session context` for the claude-cli/MCP loopback runtime because gateway tool resolution does not provide run-bound yield context.

## Best Possible Solution

Add run-bound yield context to the MCP loopback tool path and record MCP/CLI yield pause state with focused regression coverage, then let the linked PR close this issue after merge.

## Reproduction Assessment

Yes from source inspection: MCP `tools/call` resolves `sessions_yield` through the loopback cache without `sessionId` or `onYield`, so the tool reaches the `No session context` branch. I did not execute a live claude-cli setup.

## Solution Assessment

Yes at the direction level: the maintainable fix belongs in shared gateway MCP loopback/run binding, with an MCP-specific `onYield` that records pause state. PR #77437 appears to target that boundary but still needs normal review and validation before this issue closes.
