## Summary

Keep open: current main still builds the MCP loopback `sessions_yield` tool without `sessionId` or `onYield`, and the open closing PR is the right implementation candidate to review before this issue closes.

## What This Changes

Fix `sessions_yield` on the MCP/claude-cli runtime path so it receives run-bound yield context instead of returning `No session context`.

## Best Possible Solution

Review and land PR #77437 or an equivalent narrow fix that passes run-bound yield context through the loopback tool path and preserves CLI-specific yield semantics.

## Reproduction Assessment

Yes. From source inspection, MCP `tools/call` resolves `sessions_yield` through `McpLoopbackToolCache` and `resolveGatewayScopedTools`, neither of which supplies the `sessionId` and `onYield` that the tool requires.

## Solution Assessment

Yes at the direction level. The maintainable fix belongs in shared gateway/MCP tool resolution with a CLI-aware `onYield`, not in a channel-specific workaround.
