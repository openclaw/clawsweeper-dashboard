## Summary

Keep open. The PR is still a useful pointer to a real Google Vertex ADC auth area, but the proposed patch is not merge-ready on current main: it targets the old "<authenticated>" placeholder instead of the current `gcp-vertex-credentials` marker, and it mixes in an unrelated compaction-only GitHub Copilot token exchange.

## What This Changes

The PR attempts to add a Google Vertex ADC placeholder guard to embedded run and compaction runtime auth handling, adds focused model-auth coverage, and also changes GitHub Copilot token exchange behavior in compaction.

## Best Possible Solution

The maintainable end state is a current-main Google Vertex ADC path that treats `gcp-vertex-credentials` as a non-secret marker consistently across run, compaction, and provider transport, preserves real API-key auth, explicitly defines which ADC credential types are supported, and keeps Copilot token-flow changes in a separate tested PR.

## Reproduction Assessment

Yes for the authorized_user ADC marker path: current source and tests show `gcp-vertex-credentials` being converted to a Bearer token. It is unclear for generic ADC because current auth evidence only checks that a local credentials file exists, while the transport helper requires an `authorized_user` credentials file.

## Solution Assessment

No. This PR is not the best way to solve the issue on current main because it targets the obsolete `<authenticated>` placeholder and includes an unrelated Copilot compaction auth change; a rebased fix should use the current marker/helper contract and stay scoped to Google Vertex.
