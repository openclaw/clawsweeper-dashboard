The PR still targets the old `"<authenticated>"` Vertex ADC placeholder while current main uses `gcp-vertex-credentials`; it also adds duplicate env-var fallback code and an unrelated compaction-only GitHub Copilot token exchange. The PR adds a google-vertex ADC placeholder runtime-auth guard, applies it to embedded run and compaction flows, adds focused model-auth tests, and changes GitHub Copilot token handling during compaction.

A current-main fix should use the current `gcp-vertex-credentials` marker or the shared non-secret marker helpers, keep the behavior aligned across `run/auth-controller.ts`, compaction, and the Google provider transport, and split any GitHub Copilot compaction behavior into a separate reviewed change.
