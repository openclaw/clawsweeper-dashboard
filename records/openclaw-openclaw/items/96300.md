## What This Changes

The PR adds safe-integer rejection for byte-size parsing and config validation, plus focused tests for string and numeric unsafe byte-size inputs.

## Best Possible Solution

Land the shared safe-integer invariant at the parser/helper/schema boundaries once maintainers accept the stricter treatment for previously unsafe byte-size config values.

## Reproduction Assessment

## Solution Assessment

Yes, with a maintainer compatibility decision. The parser/helper/schema boundaries are the narrow owner boundaries; a schema-only fix would miss runtime helper consumers, while a compatibility shim would preserve corrupted values without a clear shipped contract.
