## Summary

Keep open: current main and v2026.6.10 only partially satisfy the hook trace-context request. Agent lifecycle runId and some message-hook correlation fields exist, but outbound message_sending still lacks run/message identity, agent hook context still lacks message/span aliases, and subagent hooks still do not expose requesterRunId for run-based trace hierarchy.

## What This Changes

The issue requests message, run, sender, and span correlation fields across plugin hook events and contexts so observability plugins can build accurate distributed traces.

## Best Possible Solution

Define one backward-compatible plugin-hook correlation contract, then land focused slices for agent inbound identity, outbound delivery correlation, subagent child/requester run IDs, and trace/span aliases with SDK docs and tests aligned.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.10 still show missing outbound run/message identity, missing first-class agent message/span aliases, and no requesterRunId in subagent hook contracts.

## Solution Assessment

No, not as a one-shot mechanical patch. The direction is valid, but the best fix is a maintainer-approved public SDK contract across agent, message, diagnostics, and subagent hooks before implementation.
