## Summary

Keep open. Current main has addressed part of the original run correlation gap for agent start/end hooks, but the broader public plugin hook contract still lacks several requested correlation fields across agent, outbound message, and subagent hook surfaces.

## What This Changes

The issue requests run, message, sender, and span correlation fields across plugin hook events and contexts so observability plugins can build accurate traces for concurrent group chats and nested agent/subagent/tool activity.

## Best Possible Solution

Define a backwards-compatible plugin-hook correlation contract, then land focused slices for agent inbound identity, outbound delivery correlation, subagent child/requester run IDs, and trace/span aliases with SDK baseline and plugin docs updated together.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the remaining gaps: agent hook contexts omit inbound identity/span aliases, generic message_sending lacks correlation, and subagent hooks still lack requesterRunId.

## Solution Assessment

Unclear. The requested direction is valid, but the best implementation needs maintainer agreement on the public SDK shape now that current main already has ctx.trace and partial runId backfill.
