## Summary

Keep open. Current main and the latest release have partial runId and diagnostic trace plumbing, but the public plugin-hook contract still does not satisfy the central request across agent inbound identity, outbound message_sending correlation, span aliases, and subagent requester/child run hierarchy.

## What This Changes

Request to add complete trace and correlation fields such as runId, messageId, senderId, parent span identifiers, and subagent parent/child run IDs across plugin hooks.

## Best Possible Solution

Define one backward-compatible plugin-hook correlation contract, then land focused slices for agent inbound identity, outbound delivery correlation, subagent child/requester run IDs, and trace/span aliases with SDK docs and contract tests.

## Reproduction Assessment

Yes. Current-main source inspection reproduces the contract gap: agent hook contexts and subagent context types lack requested fields, and message_sending calls still omit run/message/trace correlation on some outbound paths.

## Solution Assessment

Unclear for immediate implementation. The requested direction is sound, but the best fix needs maintainer-approved public SDK field names and rollout slices rather than a one-off mechanical patch.
