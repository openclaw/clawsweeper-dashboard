## Summary

Keep open. Current main has useful partial hook-correlation work, but the public plugin-hook contract still does not satisfy the issue’s central request for consistent agent identity, outbound delivery correlation, span aliases, and subagent requester/child run hierarchy.

## What This Changes

The issue asks OpenClaw to expose stable trace and correlation identifiers across plugin hooks, including runId, messageId, senderId, parentSpanId/current span data, and subagent parent/child run IDs.

## Best Possible Solution

Define one backward-compatible plugin-hook correlation contract, then land focused slices for agent inbound identity, outbound delivery correlation, subagent child/requester run IDs, and trace/span aliases with SDK docs and contract tests.

## Reproduction Assessment

Yes. Current-main source inspection is enough to reproduce the missing-contract claim: the requested public fields are absent or only partially present in hook types and call sites.

## Solution Assessment

Unclear for immediate implementation. The requested direction is sound, but the best fix requires a maintainer-approved public SDK field shape rather than a one-off mechanical patch.
