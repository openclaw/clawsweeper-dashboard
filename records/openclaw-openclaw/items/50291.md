## Summary

Keep open. Current main and v2026.5.22 have partial runId/trace plumbing, but the public plugin-hook contract still misses central correlation fields for agent identity, outbound message_sending, span aliases, and subagent parent/child run hierarchy.

## What This Changes

Request to expose complete trace correlation fields across plugin hooks, including messageId, senderId, runId, span context, and subagent requester/child run IDs.

## Best Possible Solution

Define one backward-compatible plugin-hook correlation contract, then land focused slices for agent inbound identity, outbound delivery correlation, subagent child/requester run IDs, and trace/span aliases with SDK docs and contract tests.

## Reproduction Assessment

Yes. Current-main source inspection reproduces the contract gap: several hook contexts and events still omit the requested correlation fields, and outbound message_sending still lacks run/message/trace context on the generic delivery path.

## Solution Assessment

Unclear for immediate implementation. The requested direction is sound, but the maintainable fix needs a public SDK contract decision and rollout slices rather than a one-off mechanical field dump.
