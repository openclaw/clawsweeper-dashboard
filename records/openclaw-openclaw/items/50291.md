## Label Justifications

- `P2`: The issue blocks robust observability for multi-run and multi-agent deployments, but it is a bounded plugin SDK improvement rather than an urgent runtime outage.
- `impact:session-state`: The missing correlation fields cause plugins to mis-associate messages, runs, and subagent activity into the wrong logical session or trace.

## Summary

Keep open: current main partially implements hook observability correlation, including agent runId backfill, message-context trace fields, and a diagnostics trace carrier, but outbound message hook correlation and subagent requester/child run hierarchy are still incomplete. The remaining work is public Plugin SDK contract design rather than a fully solved issue on main.

## What This Changes

Request richer plugin-hook trace and correlation fields across agent, message, compaction, and subagent hooks so observability plugins can build accurate per-run trace trees.

## Best Possible Solution

Approve one backward-compatible plugin-hook correlation contract, then land focused slices for outbound delivery run/message/trace correlation, subagent child/requester run IDs, and trace/span aliases with docs and SDK tests.

## Reproduction Assessment

Yes. Source inspection on current main shows the remaining gap: agent runId is now backfilled, but outbound message hooks still lack runId/messageId/trace correlation and subagent lifecycle hooks still lack requesterRunId.

## Solution Assessment

The requested direction is sound, but the best fix is a maintainer-approved public SDK contract rather than piecemeal optional field additions.
