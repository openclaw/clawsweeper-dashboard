## Summary

Keep open. Current main has partial hook-correlation support, but the requested public hook contract is still incomplete for agent inbound identity, outbound `message_sending` correlation, and subagent requester/child run hierarchy.

## What This Changes

The issue requests richer plugin hook trace and correlation fields across agent, message, and subagent hooks for observability plugins.

## Best Possible Solution

Define a backward-compatible plugin-hook correlation contract, then land coordinated slices for agent identity, outbound delivery correlation, subagent child/requester run IDs, and trace/span aliases with SDK baseline, docs, and contract tests updated together.

## Reproduction Assessment

Yes for the missing-contract claim: source inspection on current main shows the remaining gaps in `PluginHookAgentContext`, `PluginHookMessageSendingEvent`, and subagent hook types without needing a live Feishu or Opik setup. I did not run a live concurrent group-chat trace reproduction because the API absence is directly visible in the public types and call sites.

## Solution Assessment

Unclear. The requested direction is valid, but the best implementation needs maintainer agreement on field names, optional versus required compatibility, and how `DiagnosticTraceContext` maps to hook-level span aliases.
