## Summary

Keep open: current main partially satisfies the request with agent lifecycle runId and some message correlation fields, but outbound message_sending still lacks run/message identity and subagent hooks still lack requesterRunId for run-based trace hierarchy.

## What This Changes

The issue requests richer plugin hook trace context so observability plugins can correlate agent, message, and subagent hooks by run, message, sender, and span relationships.

## Best Possible Solution

Define one backward-compatible plugin-hook correlation contract, then land focused slices for agent inbound identity, outbound delivery correlation, subagent child/requester run IDs, and trace/span aliases with SDK docs and tests aligned.

## Reproduction Assessment

Yes, at source level. Current main still shows missing outbound run/message identity, missing first-class agent message/span aliases, and no requesterRunId in subagent hook contracts.

## Solution Assessment

No, not as a one-shot mechanical patch. The direction is valid, but the best fix is a maintainer-approved public SDK contract across agent, message, diagnostics, and subagent hooks before implementation.
