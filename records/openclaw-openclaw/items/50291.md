## Summary

Keep open because current main and v2026.6.6 only partially satisfy the requested plugin-hook trace context: runId now reaches some agent lifecycle hooks, but outbound message hooks and subagent hierarchy still lack the requested run/message/span correlation contract.

## What This Changes

The issue requests a fuller plugin-hook observability contract with message/run/span identifiers across agent, message, tool, compaction, and subagent hooks.

## Best Possible Solution

Define one backward-compatible hook correlation contract, then land focused slices for agent message/span identity, outbound delivery run/message correlation, and subagent child/requester run IDs with SDK baseline, docs, and tests updated together.

## Reproduction Assessment

Yes. Source inspection reproduces the remaining gap: current hook types and call sites still omit key message, outbound run, span, and subagent requester identifiers even though some runId plumbing exists.

## Solution Assessment

No for a one-shot mechanical patch. The requested direction is sound, but the best fix is a maintainer-approved SDK contract across hook families before implementation.
