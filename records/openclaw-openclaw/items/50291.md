## Summary

Keep open. Current main has partial hook-correlation support, including agent runId backfill and richer message-hook context, but the public plugin-hook contract still does not cover the requested all-hook agent identity, span aliases, outbound message_sending event correlation, and subagent requester/child run hierarchy.

## What This Changes

Request adding stable trace and correlation fields such as messageId, senderId, runId, parent/current span ids, callDepth, and subagent parent/child run ids to plugin hook events and contexts.

## Best Possible Solution

Define one backward-compatible plugin-hook correlation contract, then land focused slices for agent inbound identity, outbound delivery correlation, subagent child/requester run IDs, and span aliases with SDK docs and contract tests.

## Reproduction Assessment

Yes. Current-main source inspection reproduces the missing-contract claim: requested fields are absent or only partially present in the public hook types and embedded runner call sites.

## Solution Assessment

Unclear for direct automation. The requested direction is sound, but the best fix needs maintainer-approved public SDK field shape and rollout slices rather than a one-off mechanical field add.
