## Summary

Keep open. Current main has partial hook-correlation support, including runId backfill for before_agent_start/agent_end and richer message hook contexts, but it still does not satisfy the full public plugin hook trace-context request across agent inbound identity, generic outbound message_sending correlation, and subagent requester-run hierarchy.

## What This Changes

Request to expose stable message, run, and span correlation fields in plugin hook event/context payloads so observability plugins can build accurate concurrent and nested traces.

## Best Possible Solution

Define one backward-compatible plugin-hook correlation contract, then land focused slices for agent inbound identity, generic outbound delivery correlation, subagent child/requester run IDs, and trace/span aliases with SDK baseline, docs, and contract tests updated together.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the remaining gaps: agent hook contexts omit inbound identity/span aliases, generic outbound message_sending lacks run/session/message correlation, and subagent hooks lack requesterRunId despite child run IDs being available.

## Solution Assessment

Unclear. The requested direction is valid, but the best implementation needs maintainer agreement on the public SDK shape now that current main already exposes ctx.trace and partial runId/message-context plumbing.
