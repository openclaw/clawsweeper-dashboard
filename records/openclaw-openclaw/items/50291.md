## Summary

Keep open: current main only partially addresses the requested observability contract. `runId` and diagnostic trace plumbing exist for some agent/message hooks, but outbound delivery, agent-context message identity/span aliases, and requester/child subagent run correlation still remain incomplete.

## What This Changes

The issue requests richer plugin hook observability context, including message/run identifiers across hook families and parent/child span fields for trace-tree reconstruction.

## Best Possible Solution

Define one additive plugin SDK trace-correlation contract across agent, message, tool, compaction, and subagent hooks, then update the emitters, docs, API baselines, and focused contract tests together.

## Reproduction Assessment

Yes for a source-level reproduction: current hook types and emitters show missing fields and an explicit outbound `runId` caveat. I did not run a live plugin scenario because the remaining gap is visible in the public hook contract.

## Solution Assessment

Unclear as a direct implementation request: adding fields is plausible, but the best fix is a maintainer-approved plugin SDK trace contract rather than piecemeal aliases on individual hooks.
