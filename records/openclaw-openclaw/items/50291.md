## Summary

Keep open. Current main has partial hook-correlation support, but the public plugin hook contract still does not cover the core observability request across agent inbound identity, outbound message_sending correlation, and subagent requester-run hierarchy.

## What This Changes

The issue requests richer plugin hook trace context, including runId, messageId, senderId, span parent/current fields, call depth, and child/requester run IDs so observability plugins can build accurate trace trees.

## Best Possible Solution

Define one backwards-compatible plugin-hook correlation contract, then land focused slices for agent inbound identity, outbound delivery correlation, subagent child/requester run IDs, and trace/span aliases with SDK baseline, docs, and contract tests updated together.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the remaining gaps: agent hook context omits inbound identity and span aliases, message_sending lacks event correlation fields, and subagent hooks lack requesterRunId despite child run IDs being available.

## Solution Assessment

Unclear. The requested direction is valid, but the best implementation needs maintainer agreement on the public SDK shape now that current main already has ctx.trace and partial runId backfill.
