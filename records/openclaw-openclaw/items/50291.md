## Summary

Keep open. Current main has partial hook-correlation support, but the public plugin hook contract still does not cover the requested agent inbound identity fields, `message_sending` event correlation, and subagent requester/child run hierarchy.

## What This Changes

The issue requests richer plugin-hook trace and correlation context across agent, message, and subagent hooks, including run IDs, message/sender identity, and parent/child span fields.

## Best Possible Solution

Define a backward-compatible plugin-hook correlation contract, then land coordinated slices for agent identity, outbound delivery correlation, subagent requester/child run IDs, and trace/span aliases with SDK baseline, docs, and contract tests updated together.

## Reproduction Assessment

Yes. Source inspection on current main shows the remaining gaps in `PluginHookAgentContext`, `PluginHookMessageSendingEvent`, and subagent hook types without needing a live channel setup.

## Solution Assessment

Unclear. The requested direction is valid, but the best fix needs maintainer agreement on the public plugin SDK field names and how `DiagnosticTraceContext` should map to first-class span aliases.
