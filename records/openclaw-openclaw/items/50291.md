## Summary

Keep open because current main only partially addresses the requested plugin-hook observability contract: runId and trace plumbing exist for some agent/message hooks, but message identity, span aliases, outbound delivery correlation, and requester/child subagent run correlation remain missing or explicitly caveated.

## What This Changes

Expose complete trace and correlation context across plugin hooks, including per-turn run IDs, inbound message identity, span parentage, outbound message correlation, and subagent parent/child run IDs.

## Best Possible Solution

Define one backward-compatible plugin hook correlation contract, then implement it in focused slices for agent inbound identity and span aliases, outbound delivery run/message correlation, and subagent requester/child run IDs with SDK baseline, docs, and tests updated together.

## Reproduction Assessment

Yes, source inspection is enough to reproduce the missing contract: current hook types and dispatch paths expose runId in some lifecycle contexts but still omit the requested message/span/subagent correlation fields.

## Solution Assessment

The requested direction is sound, but the best fix is a maintainer-approved SDK contract across hook families rather than another isolated field addition.
