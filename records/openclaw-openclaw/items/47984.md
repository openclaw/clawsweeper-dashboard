## Summary

Keep open: current main still lacks Discord channel-edit applied tag editing, but this branch is not merge-ready because it drops explicit empty arrays and targets stale runtime paths; it also lacks attached real behavior proof.

## What This Changes

The PR adds an optional Discord `appliedTags` field to channelEdit handling so agents can send Discord `applied_tags` when editing forum/media threads.

## Best Possible Solution

Refit the change onto the current Discord extension channel-edit runtime path, preserve explicit empty arrays through parsing and REST mapping, and cover both non-empty and empty-array tag updates before merge.

## Reproduction Assessment

Yes, source-level. Current main lacks `appliedTags` in the shared channel-edit parser, type, and REST body, and the PR branch drops explicit empty arrays before the REST layer can clear tags.

## Solution Assessment

No. The feature direction is right, but the patch should use the current shared Discord runtime parser and a field-specific array parser that treats `appliedTags: []` as a valid update.
