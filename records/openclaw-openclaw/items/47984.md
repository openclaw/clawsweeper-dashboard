## Summary

Keep open: the requested Discord channelEdit appliedTags capability is still missing from current main, and the PR is useful, but the branch needs a small refit before merge because explicit empty arrays are still dropped before the REST layer and the Discord action runtime has moved since the branch was cut.

## What This Changes

The PR adds Discord channelEdit appliedTags plumbing through the send type, REST PATCH body mapping, dispatcher paths, and a dispatcher test.

## Best Possible Solution

Refit this onto the current Discord shared channel-edit path: add appliedTags to DiscordChannelEdit, parse camelCase and preferably snake_case while preserving explicit empty arrays, map it to applied_tags, and cover non-empty, empty, dispatch, and send-body behavior.

## Reproduction Assessment

Yes, source-level. Current main has no channelEdit appliedTags type or PATCH body mapping, and the PR's current parsing path would still swallow appliedTags: [] before Discord can clear tags; I did not run a live Discord repro.

## Solution Assessment

No. The direction is right, but the patch should be adapted to current main's shared parser and use a channel-edit-specific array reader that preserves explicit empty arrays.
