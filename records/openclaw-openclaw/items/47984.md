## Summary

Keep open: the requested Discord channelEdit capability is still useful and not present on current main, but this PR cannot merge as-is because it targets stale/deleted dispatcher files, drops explicit empty appliedTags arrays before the REST layer, and lacks contributor-provided real behavior proof.

## What This Changes

The PR adds an appliedTags parameter to Discord channel editing so agents can PATCH applied_tags on forum/media threads.

## Best Possible Solution

Refit this onto the current Discord shared channel-edit path: add appliedTags to DiscordChannelEdit, parse it in readDiscordChannelEditParams with explicit empty-array preservation, map it to applied_tags, and cover non-empty and empty tag updates in runtime/send tests.

## Reproduction Assessment

Yes, source-level. Current main lacks appliedTags in the Discord channel-edit type, shared parser, and PATCH body, and the PR’s parsing choice would still drop appliedTags: [] before Discord can clear all tags.

## Solution Assessment

No. The direction is correct, but the current patch is stale and should be implemented through the shared Discord parser with a channel-edit-specific array reader that preserves explicit empty arrays.
