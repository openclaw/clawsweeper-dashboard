## Summary

Keep this PR open. The requested Discord `channelEdit` capability is still missing on current `main`, and Discord's Modify Channel thread params support `applied_tags`, but the branch is stale against the current Discord plugin runtime layout and still drops explicit empty arrays before the REST layer.

## What This Changes

The PR adds `appliedTags` to Discord channel-edit typing, dispatch, REST PATCH body mapping, and a non-empty-array dispatcher test.

## Best Possible Solution

Land a current-main implementation in the Discord plugin runtime: add `appliedTags` to the channel edit contract, parse camelCase and snake_case inputs through the shared channel-edit parser while preserving explicit `[]`, map it to `applied_tags` in `editChannelDiscord`, and cover both non-empty updates and empty-array clearing in focused tests.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: on current main, `channelEdit` ignores `appliedTags` entirely; on the PR branch, sending `appliedTags: []` is normalized to `undefined` before the REST layer, so clearing all tags is a no-op.

## Solution Assessment

No. The REST mapping is the right direction, but the PR is not the best current fix because it targets removed dispatcher files and does not preserve empty arrays through dispatch; the safer fix belongs in the current shared Discord channel-edit parser and extension runtime tests.
