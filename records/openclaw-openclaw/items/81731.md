## What This Changes

## Best Possible Solution

Land this focused cron scheduler fix or an equivalent probe-past-the-candidate-second change, then close https://github.com/openclaw/openclaw/issues/81691 while keeping overflow-deferral work on its separate canonical track.

## Solution Assessment

Yes. The one-second probe is the narrowest maintainable fix because it preserves the existing stale-future repair policy and only moves the previous-run lookup past Croner's candidate-second boundary.
