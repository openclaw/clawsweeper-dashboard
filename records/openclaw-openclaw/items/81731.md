## Summary

Keep this PR open: current main and v2026.6.10 still use the reported `runAtMs + 1` cron-slot probe, while this branch is a focused candidate fix with sufficient terminal proof and no blocking review findings.

## What This Changes

The PR changes cron stale-future slot validation to probe one second after a candidate run and adds a regression test for preserving an exact-second Pacific/Honolulu cron slot.

## Best Possible Solution

Land this focused helper/test change, or an equivalent probe-past-the-candidate-second fix with the same regression coverage, then close https://github.com/openclaw/openclaw/issues/81691.

## Reproduction Assessment

Yes. Current source still probes `runAtMs + 1`, the maintenance repair path depends on that classification, and a live Croner 10.0.1 probe with the issue input shows the 1ms probe returns the prior slot while the 1s probe returns the candidate slot.

## Solution Assessment

Yes. The helper-level one-second probe is the narrowest maintainable fix because it preserves the existing stale-future repair policy, matches existing cursor stepping, and adds focused regression coverage instead of introducing a parallel scheduler path.
