Keep open for normal maintainer handling: current main and the latest release still have the cron slot bug, while this PR is a focused fix with matching regression coverage and sufficient terminal proof. No cleanup close reason applies.

## What This Changes

The PR changes cron stale-future slot detection to probe one second past a candidate run and adds a regression test for preserving a Honolulu exact-second future cron slot.

## Best Possible Solution

Land this PR or an equivalent patch with the regression test, then close the linked cron issue and keep the superseded duplicate fix closed.

## Reproduction Assessment

Yes, at source level. Current main uses the `runAtMs + 1` probe, the pinned Croner behavior is second-granular, and the linked issue plus PR proof show `slot + 1ms` returning the prior slot while `slot + 1000ms` returns the candidate slot.

## Solution Assessment

Yes. Probing one second past the candidate is the narrowest maintainable fix for Croner's second-granular boundary and matches the existing one-second cursor step in the same helper family.
