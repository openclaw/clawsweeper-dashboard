## What This Changes

This PR changes the cron stale-future slot check from a 1 ms probe to a 1 second probe and adds a regression test for an exact-second Pacific/Honolulu cron slot.

## Best Possible Solution

Land this focused scheduler-state fix with the regression test, then close the linked cron issue after the merge reaches main.

## Reproduction Assessment

Yes. The current-main helper still probes `runAtMs + 1`, and Croner 10.0.1 source shows previous-run lookup strips milliseconds and steps by seconds; the linked issue's exact slot exercises that path.

## Solution Assessment

Yes. Moving the probe past the candidate second is the narrowest maintainable fix because the rest of the repair logic already distinguishes valid natural slots, retry backoff, and stale future slots.
