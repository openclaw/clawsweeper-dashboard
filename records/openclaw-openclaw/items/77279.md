## Summary

Keep open: current main still has the duplicate `path | path` prompt-note behavior, and this recent PR is the active, focused fix path for the paired open bug report.

## What This Changes

This PR suppresses the media-note URL suffix when the sanitized URL equals the local path and updates formatter tests for duplicate, distinct, mixed, and sanitized inputs.

## Best Possible Solution

Land the formatter-level dedupe with its regression tests, while leaving broader all-image transcription policy to existing media attachment configuration or a separate product discussion.

## Reproduction Assessment

Yes. Passing equal `MediaPath`/`MediaUrl` or equal `MediaPaths`/`MediaUrls` into `buildInboundMediaNote` is a high-confidence source reproduction, and Telegram plus the shared channel media payload can produce that shape on current main.

## Solution Assessment

Yes. Comparing sanitized path and URL in the shared formatter is the narrowest maintainable fix because it removes redundant prompt text without changing media collection, attachment selection, or distinct remote URL rendering.
