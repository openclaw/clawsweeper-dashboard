## Summary

Keep open for maintainer review: current main still has a source-reproducible duplicate `path | path` media-note bug, and this PR contains the narrow formatter/test fix with sufficient terminal proof and no blocking findings.

## What This Changes

This PR makes the inbound media-note formatter omit a URL suffix when the sanitized URL matches the local path, updates media-note regression tests, and adds an Unreleased changelog entry.

## Best Possible Solution

Land the formatter-level dedupe with its regression tests, while leaving broader all-image transcription policy to the existing media attachment configuration.

## Reproduction Assessment

Yes, source-level. Current main renders duplicate `path | path` when equal `MediaPath`/`MediaUrl` or `MediaPaths`/`MediaUrls` reach `buildInboundMediaNote`, and Telegram currently creates that equal local path/url shape.

## Solution Assessment

Yes. Comparing sanitized path and URL in the shared formatter is the narrowest maintainable fix because it removes redundant prompt text without changing media collection, attachment selection policy, or genuinely distinct remote URL rendering.
