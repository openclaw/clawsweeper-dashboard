## Summary

Keep open: current main still has the duplicate media-note behavior, and this PR is a focused, adequately proven fix with no blocking correctness or security findings found.

## What This Changes

The PR changes the inbound media-note formatter to omit a URL suffix when it equals the sanitized local path and updates media-note tests for duplicate, distinct, mixed, and sanitized cases.

## Best Possible Solution

Land the formatter-level dedupe with its regression tests, while leaving broader multi-image transcription policy to the existing media attachment configuration or a separate product decision.

## Reproduction Assessment

Yes. Source inspection on current main shows equal `MediaPath`/`MediaUrl` or equal `MediaPaths`/`MediaUrls` render `path | path`, and Telegram/shared media context can produce that shape.

## Solution Assessment

Yes. Comparing sanitized path and URL in the shared formatter is the narrowest maintainable fix because it removes redundant prompt text without changing media collection, attachment selection, or genuinely distinct remote URL rendering.
