## What This Changes

The PR skips the inbound media-note URL suffix when the sanitized URL is identical to the local media path, updates media-note regression tests, and adds an Unreleased changelog entry.

## Best Possible Solution

Land the shared formatter dedupe with its regression tests, while keeping broader all-image transcription behavior tied to the documented `tools.media.*.attachments` policy unless maintainers choose a separate product change.

## Reproduction Assessment

Yes. Source inspection shows current main renders duplicate `path | path` for equal `MediaPath`/`MediaUrl` values, and Telegram plus shared channel context currently produce that equal local path/url shape for local-only media.

## Solution Assessment

Yes. Comparing sanitized path and URL in the shared formatter is the narrowest maintainable fix because it removes redundant prompt text without changing Telegram media collection or suppressing genuinely distinct remote URLs.
