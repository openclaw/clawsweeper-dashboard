## Summary

Keep open. Current main still has the duplicated local path/url media-note behavior, and this PR carries a focused formatter/test/changelog fix with sufficient terminal proof; the remaining step is maintainer review and landing, not cleanup closure.

## What This Changes

The PR changes `formatMediaAttachedLine` to omit a URL suffix when the sanitized URL equals the sanitized local path, updates media-note regression tests, and adds a changelog entry.

## Best Possible Solution

Land the formatter-level dedupe with its regression tests, while leaving all-image transcription defaults to the existing media attachment policy unless maintainers choose a separate product change.

## Reproduction Assessment

Yes. Source inspection on current main shows equal `MediaPath`/`MediaUrl` or `MediaPaths`/`MediaUrls` inputs render duplicate `path | path` output, and Telegram plus shared channel context can produce that equal local path/url shape.

## Solution Assessment

Yes. Comparing sanitized path and URL in the shared formatter is the narrowest maintainable fix because it removes redundant prompt text without changing media collection, attachment selection policy, or genuinely distinct remote URL rendering.
