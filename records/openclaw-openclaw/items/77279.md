## Summary

Keep open for normal review and merge handling. The patch is a narrow formatter fix for a source-reproducible duplicate `path | path` prompt note, and the updated PR body now includes sufficient terminal proof from the production formatter export.

## What This Changes

The PR omits an inbound media-note URL suffix when it sanitizes to the same value as the local path, adds regression tests, and records the fix in the changelog.

## Best Possible Solution

Land the formatter-level dedupe with the regression tests, while keeping broader multi-image transcription policy tied to the documented `tools.media.*.attachments` configuration rather than expanding this PR.

## Reproduction Assessment

Yes, source-level. Current main renders duplicate `path | path` when equal MediaPath/MediaUrl values reach `buildInboundMediaNote`, and Telegram currently creates that exact shape for local inbound media.

## Solution Assessment

Yes. Comparing sanitized path and URL in the shared formatter is the narrowest maintainable fix because it removes redundant prompt text without changing channel media collection or hiding genuinely distinct remote URLs.
