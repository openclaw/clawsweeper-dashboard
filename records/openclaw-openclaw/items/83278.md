## Summary

Keep open: the cleanup goal is plausible, but this patch is not merge-ready because it clears the Telegram draft stream without rearming the stream lifecycle, so later progress previews in the same turn can be silently dropped. The external PR also still needs after-fix real Telegram proof.

## What This Changes

The PR changes Telegram progress-mode final delivery to clear the progress draft and send the final reply without rotating the draft stream, and updates the dispatch regression test expectation.

## Best Possible Solution

Keep the stale progress-draft cleanup, but preserve a clear-and-rearm lifecycle reset, add coverage for another progress update after a progress-mode final, then collect live Telegram proof.

## Reproduction Assessment

Yes, source-reproducible: clear() stops the draft stream, update() ignores stopped streams, and this patch removes the forceNewMessage rearm before later progress callbacks could reuse the stream. I did not run tests because this review is read-only.

## Solution Assessment

No. Deleting the stale draft before the durable final is the right direction, but the implementation should keep or encapsulate the stream rearm after clear instead of only resetting lane state.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep the draft stream reset after clearing progress:** `extensions/telegram/src/bot-message-dispatch.ts:1128-1129`
  - body: After clear(), the underlying draft stream is stopped: update() ignores stopped/final streams until forceNewMessage() resets that state. This replacement resets only the lane state, so a progress-mode turn that emits another assistant/progress message after the final will silently drop later preview updates instead of creating a new progress draft. Keep the cleanup, but rearm the stream or add a clear-and-reset helper and cover that multi-message path.
  - confidence: 0.9
