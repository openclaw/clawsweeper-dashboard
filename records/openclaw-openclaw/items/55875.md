## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Fix the AWS secret regex before this file can compile:** `src/security/skill-scanner.ts:522`
  - body: The negative lookahead's character class escapes the closing bracket, so the regex literal is unterminated. As written, the scanner module cannot parse, which blocks the security scanner and any tests/imports that touch this file.
  - confidence: 0.95
- **[P2] Redact every matched secret before storing evidence:** `src/security/skill-scanner.ts:610`
  - body: `redactSecrets` uses each non-global secret regex with `replace`, so only the first same-pattern secret on a line is masked. Scanner findings can still retain later secrets in `evidence`, which is exactly the output likely to be logged or shown to users during audit/install review.
  - confidence: 0.9
- **[P2] Validate manifest field types before trimming:** `src/security/skill-scanner.ts:1118`
  - body: `validateManifest` calls `.trim()` on `name` and `description` without first proving they are strings. A malformed or block-list frontmatter value can throw a `TypeError` instead of returning a structured invalid-manifest result, interrupting the security-check flow for untrusted input.
  - confidence: 0.86
- **[P2] Reject non-array trigger fields explicitly:** `src/security/skill-scanner.ts:1169`
  - body: Trigger checks run only when `manifest.triggers` is an array, so a scalar value such as `triggers: always` bypasses the broad-trigger validation and can leave a risky manifest looking clean. Treat present-but-non-array trigger and capability fields as validation errors.
  - confidence: 0.82
