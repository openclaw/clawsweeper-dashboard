Full review comments:

- **[P1] Fix the AWS secret regex before merge:** `src/security/skill-scanner.ts:522`
  - body: The new regex literal is unterminated: the negative lookahead escapes the only closing bracket, and Node reports `Unterminated regexp literal` for this expression. The scanner module cannot parse until this character class is closed correctly.
  - confidence: 0.98
- **[P1] Redact every matched secret in evidence:** `src/security/skill-scanner.ts:610`
  - body: `redactSecrets` calls `replace` with non-global secret patterns, so only the first token of a given type on a line is masked. If a line contains two GitHub tokens, AWS keys, or similar same-type secrets, later matches stay in `finding.evidence`, which makes the security scanner leak secrets it detected.
  - confidence: 0.9
- **[P2] Validate manifest field types before trimming:** `src/security/skill-scanner.ts:1118-1122`
  - body: `validateManifest` calls `.trim()` on `manifest.name` and `manifest.description` after only truthiness checks. A malformed SKILL.md frontmatter value parsed as an array or object throws a TypeError instead of returning validation errors, so untrusted manifests can interrupt the scan path.
  - confidence: 0.86
