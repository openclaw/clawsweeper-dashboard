## Summary

Keep open. Current main has a narrower built-in skill/plugin code-safety scanner, but it does not implement this PR's YARA-style categories, hardcoded secret detection, unicode steganography checks, manifest validation, trust verdict API, or `shouldBlockSkill`/`validateManifest` helpers. The request hardens an existing core install/audit gate, so this is not a conservative ClawHub or stale-close candidate.

## Best Possible Solution

Keep the PR open or ask the author to refresh it against current main. The best path is to extend the existing built-in skill scanner and install/audit gate with well-tested YARA-style rule coverage, secret redaction, unicode/manifest checks, and a clear trust/blocking contract, while resolving the review comments about false positives and manifest parser correctness.
