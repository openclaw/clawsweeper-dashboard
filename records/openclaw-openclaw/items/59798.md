## Summary

Keep this PR open for maintainer follow-up. Current main already avoids the ACP locally-caught throw shape through a turn-stream helper, but the PR's ACP refactor moves failure bookkeeping behind fallible cleanup, and the separate OAuthProviderId cleanup is still not present on main.

## What This Changes

The PR moves ACP turn failure handling into a post-finally `turnError` path and changes OAuth auth-profile provider typing from deprecated `OAuthProvider` to `OAuthProviderId`.

## Best Possible Solution

Keep current main's ACP helper split and failure-bookkeeping order. Rebase or split this PR so the obsolete ACP refactor is dropped, then land the dependency-backed `OAuthProviderId` cleanup as a focused type-contract change; add ACP cleanup-failure regression coverage if maintainers still want to touch this path.

## Reproduction Assessment

Yes. For the blocking PR finding, make an ACP turn fail while `reconcileRuntimeSessionIdentifiers` rejects during cleanup; in the PR diff, the `turnError` bookkeeping runs only after `finally`, so cleanup exits first and the failed turn is left without the intended terminal/error updates.

## Solution Assessment

No. The ACP part is not the best current fix because main already removes the local caught-throw warning while preserving failure state before cleanup; the OAuth type cleanup is better handled as a narrow rebased change.
