## Summary

Keep open: the ACP hunk should not merge as-is because it can skip failure bookkeeping when post-turn cleanup rejects, while the separate OAuth type cleanup needs to be rebased and proven against current main.

## What This Changes

The PR moves ACP turn failure bookkeeping into a post-finally `turnError` path and changes OAuth provider type annotations from `OAuthProvider` to `OAuthProviderId`.

## Best Possible Solution

Keep current main's ACP helper/catch-time failure ordering, and land the OAuthProviderId cleanup only as a small rebased change with dependency-verified proof.

## Reproduction Assessment

Yes. The source-level reproduction is an ACP turn failure followed by a rejecting post-turn cleanup call before the PR's post-finally `turnError` block, which skips the intended terminal/error updates.

## Solution Assessment

No. Current main already removes the local caught-throw shape while preserving failure bookkeeping before cleanup, so the safer solution is to drop or rework the ACP hunk and split the OAuth type cleanup if still wanted.
