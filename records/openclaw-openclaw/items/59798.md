## Summary

## What This Changes

## Best Possible Solution

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Persist turn failure state before cleanup:** `src/acp/control-plane/manager.core.ts:943`
  - body: The new `turnError` block runs only after `finally`. If post-turn cleanup rejects before this block, `recordTurnCompletion`, background-task terminal marking, and `setSessionState(..., state: "error")` are skipped, masking the ACP failure and leaving the failed turn non-terminal. Keep failure bookkeeping in the catch path or make cleanup unable to preempt it before moving this block.
