## Summary

Keep open: the PR still has a blocking ACP cleanup-order regression relative to current main. Current main already removes the original local stream-error throw shape through the turn-stream helper while preserving failure bookkeeping before fallible cleanup; the OAuth type cleanup is separable useful work, but it should not ride along with the obsolete ACP refactor as-is.

## What This Changes

The PR refactors ACP turn error handling to use a post-finally `turnError` path and changes OAuth auth-profile provider typing from `OAuthProvider` to `OAuthProviderId`.

## Best Possible Solution

Rebase or replace the PR so the ACP path keeps current main's helper split and catch-time failure bookkeeping. If maintainers want the OAuth cleanup, land it as a focused type-contract change after confirming the upstream `pi-ai` type export and running the targeted auth-profile tests.

## Reproduction Assessment

Yes. Make an ACP turn fail, then make `reconcileRuntimeSessionIdentifiers` reject during the post-turn cleanup path; in the PR diff, the `turnError` block runs only after `finally`, so cleanup exits first and the failed turn misses the intended terminal/error updates.

## Solution Assessment

No. The ACP refactor is not the best fix because current main already avoids the local caught-throw shape while preserving failure bookkeeping before cleanup; the safer path is to keep that ordering and split out the OAuth type cleanup if it is still desired.
