## Summary

Keep open: the PR addresses a valid source-reproducible installer bug, but the current patch moves Homebrew setup only into the Node branch and drops the existing macOS bootstrap path needed when Git is missing.

## What This Changes

Moves `install_homebrew` from the unconditional preparation stage into the failed `check_node` branch and adds a static installer regression assertion.

## Best Possible Solution

Keep the lazy Homebrew direction, but make it cover every macOS brew consumer: bootstrap Homebrew before `install_node` when Node is missing and before `install_git` when Git is missing, with regression coverage for supported-Node/Git-present and supported-Node/Git-missing paths.

## Reproduction Assessment

Yes at source level: current main calls `install_homebrew` before `check_node`, and `install_homebrew` can exit before a supported Node on PATH is considered. I did not establish a live non-admin macOS reproduction in this read-only review.

## Solution Assessment

No: moving Homebrew solely into the Node branch is too narrow because the same installer still uses Homebrew to install Git on macOS. The safer solution is a lazy Homebrew helper used by both Node and Git install branches.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve Homebrew bootstrap for Git installs:** `scripts/install.sh:2825`
  - body: Moving `install_homebrew` only under `if ! check_node` means a macOS user with supported Node but no Git and no Homebrew will skip Homebrew setup, then the npm path reaches `install_git`, whose macOS branch runs `brew install git`. Current main bootstraps Homebrew before that path and the docs still promise Git is installed when missing, so this breaks an existing first-run setup. Please also bootstrap Homebrew before the macOS Git install path, or factor the dependency so both Node and Git callers use it.
  - confidence: 0.9
