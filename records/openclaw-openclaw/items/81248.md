## Summary

Keep open: the current-main race is plausible and the fix direction is useful, but this PR still leaves the old relay able to enqueue writes while `drain()` is awaiting and also misses the ACP spawn relay-handle test double update. External real behavior proof is still mock-only.

## What This Changes

The PR adds an async `drain()` method to ACP parent stream relays, awaits it before replacing a relay that shares a stream log path, and adds drain-focused unit tests.

## Best Possible Solution

Land a focused fix that closes the old relay's event intake before waiting for its write chain, covers the reassignment path in ACP spawn tests, and includes redacted real runtime proof or an explicit maintainer proof override.

## Reproduction Assessment

No high-confidence live reproduction is established. Source inspection shows the current-main mismatched-runId path can create two independent write chains for one log path, and the PR tests exercise the promise-ordering contract with mocks.

## Solution Assessment

No. Awaited draining is the right narrow direction, but the old relay should stop accepting events before the wait, and the ACP spawn test doubles need the new handle method before this branch is mergeable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Stop relay event intake before awaiting drain:** `src/agents/acp-spawn.ts:1423`
  - body: This awaits `drain()` while the old relay remains subscribed. If an ACP event arrives while an existing append is pending, `writeLogLine` can extend `logWriteChain` after `drain()` has already started awaiting the previous promise, so the branch can still create the replacement relay while the old chain is appending to the same log file. Dispose/unsubscribe before awaiting, or make `drain()` close intake before it waits.
  - confidence: 0.87
- **[P2] Update relay test handles before awaiting drain:** `src/agents/acp-spawn.ts:1423`
  - body: Adding this await changes the relay handle contract, but `createRelayHandle()` in `src/agents/acp-spawn.test.ts` still returns only `dispose` and `notifyStarted`. Existing stream-to-parent tests use that helper and enter this mismatched-runId branch because the gateway mock returns `runId: "run-1"`, so the suite can throw `parentRelay.drain is not a function` before reaching its assertions.
  - confidence: 0.92
