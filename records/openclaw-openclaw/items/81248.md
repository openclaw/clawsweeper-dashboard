## Summary

Keep open: current main still has the fire-and-forget relay log flush, and this PR is a plausible fix direction, but the branch still has focused correctness/test-harness blockers and only mock-based proof.

## What This Changes

The PR adds an async `drain()` method to the ACP parent stream relay, awaits it before replacing a relay for a mismatched ACP run id, and adds drain timing tests.

## Best Possible Solution

Implement stable relay shutdown/drain that prevents new events before awaiting `logWriteChain`, update ACP spawn tests and handles, and provide redacted real runtime proof or a maintainer proof override.

## Reproduction Assessment

Source-reproducible only: current main pre-registers a relay under the idempotency key, then disposes it and constructs another relay with the same `streamLogPath` if the gateway returns a different `runId`, while `dispose()` does not await the write chain. I did not establish a live mismatched-runId gateway run.

## Solution Assessment

No: awaiting a drain is the right direction, but this branch should stop or unsubscribe the old relay before waiting, or make `drain()` close intake and await the stable final chain. It also needs the ACP spawn relay-handle test doubles updated.
