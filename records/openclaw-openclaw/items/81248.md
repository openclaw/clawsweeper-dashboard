## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Stop relay intake before awaiting drain:** `src/agents/acp-spawn.ts:1455`
  - body: Calling `drain()` while the old relay is still subscribed leaves a race in the path this PR is trying to fix. If an old-run event arrives while an append is pending, it can enqueue another `logWriteChain` after `drain()` already captured the earlier promise, so the replacement relay can still start while the old chain writes to the same log path. Close intake before awaiting, or make `drain()` close intake and await the stable final chain.
  - confidence: 0.84
- **[P2] Update ACP spawn relay test handles:** `src/agents/acp-spawn.ts:1455`
  - body: Adding this awaited call changes the relay handle contract, but the existing `createRelayHandle()` helper in `src/agents/acp-spawn.test.ts` still returns only `dispose` and `notifyStarted`. Stream-to-parent tests use the default gateway mock `runId: "run-1"`, enter this mismatched-runId branch, and can throw `parentRelay.drain is not a function` before their assertions.
  - confidence: 0.92
