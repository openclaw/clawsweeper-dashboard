## Summary

Keep open: the patch targets a real source-level race in ACP parent stream logging, but the branch is not merge-ready because existing ACP spawn test handles still lack the new `drain()` method and the supplied proof is mocked unit output only.

## What This Changes

The branch adds `AcpSpawnParentRelayHandle.drain()`, awaits it before replacing a parent stream relay after a mismatched ACP run id, and adds drain-focused unit tests.

## Best Possible Solution

Land a focused fix that stops the old relay from accepting events before awaiting its log chain, updates ACP spawn relay-handle test doubles and coverage, and includes redacted real runtime proof or an explicit maintainer proof override.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source-reproducible only: current main has a clear path to two independent relay write chains for the same `streamLogPath`, but no live mismatched-runId gateway run was established.

## Solution Assessment

Is this the best way to solve the issue? No: an awaited drain is the right direction, but this branch awaits while the old relay can still accept events and leaves existing ACP spawn test handles without the new method.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Stop relay intake before awaiting drain:** `src/agents/acp-spawn.ts:1455`
  - body: Calling `drain()` while the old relay is still subscribed leaves a race in the path this PR is trying to fix. If an old-run event arrives while the first append is pending, it can enqueue another `logWriteChain` after `drain()` has already captured the earlier promise; `drain()` then resolves, `dispose()` flushes fire-and-forget, and the replacement relay can still start while the old chain is appending to the same log path. Dispose/unsubscribe before awaiting the chain, or make `drain()` close intake and await the stable final chain.
  - confidence: 0.84
- **[P2] Update relay test handles before awaiting drain:** `src/agents/acp-spawn.ts:1455`
  - body: Adding this await changes the relay handle contract, but `createRelayHandle()` in `src/agents/acp-spawn.test.ts` still returns only `dispose` and `notifyStarted`. Existing stream-to-parent tests use that helper and the default gateway mock returns `runId: "run-1"`, so they enter this mismatched-runId branch and can throw `parentRelay.drain is not a function` before reaching their assertions.
  - confidence: 0.92
