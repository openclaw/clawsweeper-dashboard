## Summary

Keep open: the previous ClawSweeper blockers still match the current PR head, and the branch remains incorrect because it awaits drain while the old relay can still receive events, leaves existing relay test handles without the new method, and supplies only mock-based proof.

## What This Changes

The PR adds `AcpSpawnParentRelayHandle.drain()`, awaits it when replacing an ACP parent stream relay after a mismatched run id, and adds drain-focused unit tests.

## Best Possible Solution

Land a focused relay fix that closes intake before draining or otherwise awaits a stable final log chain, updates ACP spawn handle test doubles and coverage, and includes real runtime proof or an explicit maintainer proof override.

## Reproduction Assessment

No: there is not a high-confidence live reproduction path. Source inspection does show current main can create two independent relay write chains for the same `streamLogPath` when the gateway returns a run id different from the idempotency key.

## Solution Assessment

No: awaiting a drain is the right direction, but this branch waits while the old relay can still accept events and does not update all existing relay test handles for the new contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Stop relay intake before awaiting drain:** `src/agents/acp-spawn.ts:1455`
  - body: At this point the old relay is still subscribed while `drain()` awaits the current `logWriteChain`. An event for the pre-dispatch run can arrive during that await and enqueue a newer append that the already-evaluated await will not wait for; `dispose()` then only fire-and-forget flushes, so the replacement relay can still write the same log file concurrently. Close intake before awaiting, or make `drain()` close intake and await the final stable chain.
  - confidence: 0.84
- **[P2] Update relay test handles before awaiting drain:** `src/agents/acp-spawn.ts:1455`
  - body: Adding this await extends the relay handle contract, but `createRelayHandle()` in `src/agents/acp-spawn.test.ts` still returns only `dispose` and `notifyStarted`. Existing stream-to-parent tests create both relay handles from that helper and the default gateway mock returns `runId: "run-1"`, so they can enter this branch and throw before reaching their assertions unless the helper and coverage are updated.
  - confidence: 0.92
