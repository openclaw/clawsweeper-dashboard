## Summary

Keep open: the fix is aimed at a real source-level race, but the branch still drains while the old relay can accept events, leaves existing ACP spawn relay test handles without the new method, and has only mock-based after-fix proof.

## What This Changes

The branch adds an async drain method to ACP spawn parent stream relays, awaits it before replacing a relay after a mismatched run id, and adds drain-focused unit tests.

## Best Possible Solution

Land a focused relay shutdown/drain contract that stops intake before awaiting the final log write chain, updates ACP spawn relay-handle tests, and includes redacted real runtime proof or an explicit maintainer proof override.

## Reproduction Assessment

No high-confidence live reproduction is available. Current main source shows the two-relay same-logPath path, but no failing current-main test or live mismatched-runId gateway run was established.

## Solution Assessment

No. Awaited draining is the right direction, but this implementation awaits while the old relay can still accept events and does not update existing ACP spawn relay test handles.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Stop relay intake before awaiting drain:** `src/agents/acp-spawn.ts:1455`
  - body: Calling `drain()` before `dispose()` leaves the old relay subscribed while the awaited append is in flight. An old-run event can enqueue a later `logWriteChain` after `drain()` captured the previous promise, so the replacement relay can still start writing the same `logPath` while the old chain is appending. Dispose/unsubscribe first, or make `drain()` close intake and await the final stable chain.
  - confidence: 0.84
- **[P2] Add drain to ACP spawn relay test handles:** `src/agents/acp-spawn.ts:1455`
  - body: This await makes `drain` part of the relay handle contract, but `createRelayHandle()` in `src/agents/acp-spawn.test.ts` still returns only `dispose` and `notifyStarted`. Existing stream-to-parent tests use that helper and the default gateway mock returns a different run id, so the mismatched-runId branch can throw `parentRelay.drain is not a function` before assertions.
  - confidence: 0.92
