## Summary

Keep open: the PR addresses a plausible source-reproducible relay log ordering bug, but it still has merge blockers. The new drain is awaited before the old relay is disposed, which can still allow a later old-relay log append to race the replacement relay, and the ACP spawn test double used by existing mismatched-runId tests still lacks the new `drain` method.

## What This Changes

The PR adds `drain()` to ACP parent stream relay handles, awaits it before replacing a mismatched-runId relay, and adds drain-focused unit tests.

## Best Possible Solution

Land a focused fix that stops the old relay before awaiting all pending writes, updates ACP spawn relay test doubles and replacement-path coverage, and includes redacted real runtime proof or a maintainer proof override.

## Reproduction Assessment

No: source inspection shows current main can replace a parent relay while the first relay's `logWriteChain` is still pending, but there is no live gateway setup that reliably returns a mismatched `runId`. The PR's own proof is a mocked unit test, not a real reproduction.

## Solution Assessment

No: an awaited drain is the right direction, but this implementation drains before disposal, leaving the old relay able to enqueue later writes while the drain awaits. The branch also needs the ACP spawn test handle updated before it is a safe merge candidate.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Dispose the old relay before awaiting its log drain:** `src/agents/acp-spawn.ts:1423`
  - body: The new call awaits `drain()` while the old relay is still subscribed. If another event for the old run arrives while a prior append is pending, its microtask can extend `logWriteChain` after `drain()` has already captured the previous promise; `dispose()` then returns without awaiting that later append, so the replacement relay can still append to the same log file concurrently. Dispose/unsubscribe before awaiting the drain, or make drain close the relay and wait for a stable chain.
  - confidence: 0.83
- **[P2] Update relay test handles before awaiting drain:** `src/agents/acp-spawn.ts:1423`
  - body: This call changes the mocked relay handle contract, but `createRelayHandle()` in `src/agents/acp-spawn.test.ts` still returns only `dispose` and `notifyStarted`. Existing stream-to-parent tests use that helper and enter this mismatched-runId branch with the default gateway mock, so they can throw `parentRelay.drain is not a function` before reaching their assertions.
  - confidence: 0.92
