## What This Changes

The PR adds an async `drain()` method to ACP parent stream relays, awaits it before replacing a relay that reuses the same stream log path, and adds unit coverage for drain ordering.

## Best Possible Solution

Land a focused fix that drains the old relay before replacement, updates all relay-handle test doubles, covers the reassignment ordering in `acp-spawn.test.ts`, and includes redacted real runtime proof or an explicit maintainer proof override.

## Solution Assessment

Is this the best way to solve the issue? Partly: adding an awaited drain is a narrow fix direction, but this patch also needs the existing `acp-spawn.test.ts` relay mocks and reassignment coverage updated before it is safe to merge.
