## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Close relay intake before awaiting drain:** `src/agents/acp-spawn.ts:1455`
  - body: At `await parentRelay.drain()`, the old relay is still subscribed. If an event for the idempotency-key run arrives while an earlier append is pending, `drain()` awaits the chain that existed when it was called; the later event can replace `logWriteChain`, so `dispose()` still flushes fire-and-forget and the replacement relay can write the same file concurrently. Close intake before the await, or make `drain()` close intake and await the final stable chain.
  - confidence: 0.84
- **[P2] Update ACP spawn relay test doubles:** `src/agents/acp-spawn.ts:1455`
  - body: This call makes `drain` part of the relay handle contract, but `createRelayHandle()` in `src/agents/acp-spawn.test.ts` still only returns `dispose` and `notifyStarted`. Existing stream-to-parent tests use that helper while the default gateway mock returns `runId: "run-1"`, so this branch can call `drain` on a test handle that does not implement it.
  - confidence: 0.92
