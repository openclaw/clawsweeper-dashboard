## Summary

Keep open: the PR targets a real source-visible ACP relay log-ordering bug, but the proposed drain still awaits while the old relay can accept events and the branch leaves existing ACP spawn relay test doubles without the new method.

## What This Changes

The PR adds an async drain method for ACP spawn parent relays, awaits it before replacing a relay with the same stream log path, and adds drain-focused unit tests.

## Best Possible Solution

Land a focused ACP relay fix that closes old relay intake before awaiting the final stable log chain, updates ACP spawn test handles and branch coverage, and includes redacted real runtime proof or an explicit maintainer proof override.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source-reproducible only: current main has a clear path to two independent relay write chains for the same streamLogPath, but no live mismatched-runId gateway run was established.

## Solution Assessment

Is this the best way to solve the issue? No: an awaited drain is the right direction, but this branch awaits while the old relay can still accept events and leaves ACP spawn test handles without the new method.
