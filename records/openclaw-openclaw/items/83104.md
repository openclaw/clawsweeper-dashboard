## Summary

Keep this PR open: the production escape at the two attribute sinks looks like the right fix for the reported XSS, but the branch still carries unsafe proof helpers and the PR body has not shown the browser-rendered export path after the fix.

## What This Changes

The PR escapes exported session entry IDs at the HTML `id` and `data-entry-id` attribute sinks, adds linkedom security coverage, and adds two local XSS proof scripts.

## Best Possible Solution

Land the attribute escaping and regression coverage after proof helpers either fail closed or are removed, and after real browser or DOM export proof shows no script or attribute breakout plus copy-link round-trip.

## Reproduction Assessment

Yes by source inspection: current main inserts raw entry IDs into `data-entry-id` and `id` attributes at `src/auto-reply/reply/export-html/template.js:1312` and `src/auto-reply/reply/export-html/template.js:1323`. I did not execute a current-main browser repro in this read-only review.

## Solution Assessment

Partly. Escaping exactly at both attribute sinks is the right production fix, but the committed proof helpers should fail closed or be removed and real browser/DOM proof is still needed before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Remove the raw-HTML-only XSS proof:** `scripts/xss-proof.mjs:71-83`
  - body: `scripts/xss-proof.mjs` never runs `template.js`, so it checks the template before the base64 session data is decoded and before the vulnerable `entry.id` attribute interpolation happens. This helper can pass on the unfixed code and gives false assurance for the security fix; delete it or replace it with the DOM/browser-rendered proof path.
  - confidence: 0.9
- **[P2] Fail the DOM proof when checks fail:** `scripts/xss-dom-proof.mjs:188`
  - body: The DOM proof prints `FAIL` for injected scripts or attribute breakout but never records a failure or exits nonzero, then reaches the unconditional `All DOM checks passed` line. If this stays in `scripts/`, make every failed check set a failing exit status so the proof cannot silently pass a regression.
  - confidence: 0.88
