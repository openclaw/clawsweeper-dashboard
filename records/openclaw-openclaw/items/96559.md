## Summary

Keeping this open: current main and the latest release still route Gemini web_search through API-key credentials, while ADC support remains scoped to the Google Vertex model-provider path, so the requested Enterprise ADC web_search capability is not implemented yet.

## What This Changes

Request to add ADC-style authentication for Google Enterprise/Gemini web_search, similar to the existing Google Vertex model-provider ADC path.

## Best Possible Solution

Implement maintainer-approved support inside the Google plugin boundary, reusing the existing Vertex ADC helper or a clearly named web_search provider contract, with docs and tests that cover both API-key and ADC behavior.

## Reproduction Assessment

Yes. Source inspection shows Gemini web_search currently accepts API-key-style credentials only and returns missing_gemini_api_key when none are present; I did not run a live Enterprise ADC web_search request.

## Solution Assessment

Unclear. ADC support belongs in the Google plugin/provider boundary, but maintainers still need to choose the permanent auth/config shape before a fix PR is safe.
