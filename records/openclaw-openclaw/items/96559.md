## Summary

Keep open: current main and the latest release still route Gemini web_search through API-key credentials, while ADC support exists on the Google Vertex model-provider transport path only. The request is aligned with provider support goals, but it changes auth/config surface and needs maintainer API direction before implementation.

## What This Changes

Request ADC authentication support for the Gemini web_search provider used by Google Enterprise users.

## Best Possible Solution

Design this inside the Google plugin as an explicit, documented ADC-capable web_search auth path that preserves the existing API-key Gemini flow and tests both fresh setup and upgrade behavior.

## Reproduction Assessment

Yes. Source inspection reproduces the gap: Gemini web_search accepts only API-key credentials and returns a missing_gemini_api_key path when no key is available; I did not run a live ADC request without credentials.

## Solution Assessment

Unclear as an implementation path. The requested webSearch.auth adc mode is plausible, but the best fix needs maintainer direction on whether the durable API is a new auth mode, google-vertex reuse, or a separate provider entry.
