## Summary

Keep open: current main and v2026.6.10 still make Gemini web_search API-key based, while ADC support is only wired through the google-vertex model-provider path. This remains a valid Google provider-auth feature request that needs maintainer API/config direction before implementation.

## What This Changes

The issue requests adding an ADC authentication mode for Google/Gemini-backed web_search, similar to the existing Google Vertex provider ADC path.

## Best Possible Solution

Implement accepted support inside the Google plugin boundary after maintainers choose the durable ADC web_search auth/config shape.

## Reproduction Assessment

Yes, by source inspection: the current Gemini web_search runtime and docs resolve only API-key-style credentials and emit a missing-key response without one. I did not run a live Google Enterprise ADC search request, so this is source-reproducible rather than runtime-reproduced.

## Solution Assessment

Unclear: ADC support should live in the Google plugin boundary, but maintainers still need to choose whether the permanent API is a google-vertex web_search provider, a Gemini webSearch auth mode, or provider-auth linkage.
