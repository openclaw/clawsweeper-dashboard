## Summary

Keep open: current main and the latest release still expose Gemini web_search as an API-key based provider, while ADC support exists only on the Google Vertex model-provider path. This is a valid provider-auth feature request, but the durable API shape needs maintainer direction before implementation.

## What This Changes

The issue requests ADC-based Google Enterprise authentication support for web_search, similar to the existing Google Vertex provider ADC path.

## Best Possible Solution

Implement accepted support inside the Google plugin/provider boundary by reusing the existing Vertex ADC helpers and adding focused docs plus tests for API-key and ADC web_search behavior, without moving Google-specific auth probing into core.

## Reproduction Assessment

Yes, by source inspection: current main's Gemini web_search credential resolver only accepts API-key-style inputs and returns a missing-key response when none are present. I did not run a live Google Enterprise ADC web_search request, so this is source-reproducible rather than runtime-reproduced.

## Solution Assessment

Unclear: ADC support belongs in the Google plugin boundary, but maintainers should choose whether the permanent API is a google-vertex web_search provider, a webSearch auth mode, or reuse of provider-auth metadata before implementation.
