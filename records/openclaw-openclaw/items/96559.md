## Summary

Keep open: current main and the latest release still make Gemini web_search API-key-only, while ADC support exists on the separate Google Vertex model-provider path. This is a valid Google provider-auth feature request, but the durable web_search API/config shape needs maintainer direction before implementation.

## What This Changes

Request support for Google Enterprise Application Default Credentials authentication in Gemini web_search, similar to the Google Vertex provider ADC path.

## Best Possible Solution

Add accepted ADC support inside the Google plugin web-search boundary, reusing the Vertex ADC helper where appropriate, with docs and focused tests for both API-key and ADC behavior.

## Reproduction Assessment

Yes, by source inspection: current main's Gemini web_search resolver only accepts API-key-style inputs and returns missing_gemini_api_key when none are present. I did not run a live Google Enterprise ADC web_search request, so this is source-reproducible rather than runtime-reproduced.

## Solution Assessment

Unclear: the Google plugin boundary is the right place, but maintainers still need to choose whether the permanent API is a google-vertex web_search provider, a Gemini webSearch auth mode, or provider-auth reuse.
