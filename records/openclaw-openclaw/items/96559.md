## Summary

Keep open: current main and v2026.6.10 still authenticate Gemini web_search with API-key inputs, while ADC support is scoped to the google-vertex model-provider path. This remains a valid auth-provider feature request, but the durable web_search auth/API shape needs maintainer direction before implementation.

## What This Changes

Request support for Google Enterprise Application Default Credentials as an authentication mode for Gemini-backed web_search.

## Best Possible Solution

Choose a maintainer-approved Google web_search auth contract, then implement ADC support inside the Google plugin with aligned manifest/config schema, docs, and focused coverage for both API-key and ADC behavior.

## Reproduction Assessment

Yes for the source-level limitation: current main's Gemini web_search path resolves only API-key-style credentials and returns missing_gemini_api_key when none are present. I did not run a live Google Enterprise ADC web_search request because the missing behavior is a new auth/config capability, not a broken documented workflow.

## Solution Assessment

Unclear until maintainers choose the API shape. Implementing ADC is plausible, but the maintainable fix should live in the Google plugin's web_search/provider-auth boundary and avoid adding a second competing config source without an accepted contract.
