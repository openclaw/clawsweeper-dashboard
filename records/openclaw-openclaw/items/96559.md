## Summary

Keep open: Gemini managed web_search still requires API-key auth on current main and in v2026.6.10, while ADC support is scoped to the Google Vertex model-provider transport. This is a valid provider-auth feature request, but it needs maintainer/API direction before implementation.

## What This Changes

Request support for Google Enterprise/Vertex ADC authentication in managed Gemini web_search.

## Best Possible Solution

Add accepted ADC support inside the Google plugin’s web-search/provider boundary, reusing Vertex ADC helpers after maintainers choose the stable web_search auth/config contract.

## Reproduction Assessment

Yes, source reproducible: the managed Gemini search resolver only accepts API-key sources and returns missing_gemini_api_key when none are present. No live Google Enterprise ADC request was run for this review.

## Solution Assessment

Unclear until maintainers choose the product/API shape. The Google plugin boundary is the right ownership area, but the stable auth/config contract is unresolved.
