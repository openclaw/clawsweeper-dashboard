## Summary

Keep open: current main supports Gemini web_search only through API-key-style credentials, while Google Vertex ADC is implemented for the model provider path but not wired into web_search. This is a valid provider-auth feature request, but it needs maintainer direction because the proposed fix adds or changes auth/config surface.

## What This Changes

Request ADC authentication support for Google Enterprise AI/Vertex-backed web_search, similar to the existing google-vertex model provider ADC handling.

## Best Possible Solution

Implement this, if accepted, inside the Google plugin/provider boundary by reusing the existing Vertex ADC helpers and adding focused docs and tests for fresh config plus ADC auth, without generic core credential probing.

## Reproduction Assessment

Yes, by source inspection: current Gemini web_search only resolves API-key-style credentials and emits a missing-key response when none are present. I did not perform a live ADC request, so this is source-reproducible rather than runtime-reproduced.

## Solution Assessment

Unclear: the proposed webSearch.auth adc mode is plausible, but the best maintainable shape may be a distinct google-vertex web_search provider or an auth-provider linkage that reuses existing Vertex ADC helpers. Maintainers should choose that API boundary before a repair PR.
