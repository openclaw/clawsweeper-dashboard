## Summary

Keep open: current main and the latest release still expose Gemini web_search through API-key credentials only, while ADC support is implemented on the separate Google Vertex model-provider path. The request is valid, but it changes provider-auth/config semantics and needs maintainer product/API direction before implementation.

## What This Changes

The issue requests Google Enterprise ADC authentication support for the Gemini-backed web_search provider.

## Best Possible Solution

Implement accepted ADC support inside the Google plugin boundary by reusing the existing Vertex ADC token helper, keeping core web_search provider selection generic, and adding docs plus focused tests for both API-key and ADC behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows Gemini web_search only accepts API-key-style credentials and returns missing_gemini_api_key when none are present; I did not run a live Google Enterprise ADC web_search request.

## Solution Assessment

Unclear. ADC support belongs in the Google plugin boundary, but maintainers should choose the durable public shape before implementation because a provider id, webSearch auth mode, or provider-auth linkage would each create different config semantics.
