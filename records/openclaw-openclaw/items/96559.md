## Summary

Keep open: current main and v2026.6.10 still expose Gemini web_search as an API-key-backed provider, while ADC support lives on the separate Google Vertex model-provider path. This is a valid provider-auth feature request, but the durable web_search API/config shape needs maintainer direction before implementation.

## What This Changes

Request to add Google Enterprise Application Default Credentials support for the Gemini-backed web_search provider.

## Best Possible Solution

Implement accepted support inside the Google plugin/provider boundary by reusing the existing Vertex ADC helpers and adding focused docs plus tests for API-key and ADC web_search behavior.

## Reproduction Assessment

Yes, by source inspection: current main's Gemini web_search path only accepts API-key-style inputs and returns a missing-key response when none are present. I did not run a live Google Enterprise ADC web_search request, so this is source-reproducible rather than runtime-reproduced.

## Solution Assessment

Unclear: ADC support belongs in the Google plugin boundary, but maintainers should choose whether the permanent API is a google-vertex web_search provider, a webSearch auth mode, or reuse of provider-auth metadata before implementation.
