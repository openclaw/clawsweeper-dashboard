## Summary

Keep open: current main still requires API-key auth for Gemini `web_search`, while Google’s Enterprise/Vertex grounding docs support ADC-style bearer-token usage; the remaining blocker is maintainer choice of the OpenClaw config/auth shape.

## What This Changes

Request support for `web_search` to use Google Enterprise/Application Default Credentials instead of requiring Gemini API-key auth.

## Best Possible Solution

Add ADC support in the Google plugin’s Gemini web_search provider through a maintainer-approved auth/config shape that reuses existing Vertex ADC helpers where appropriate and avoids core-specific policy.

## Reproduction Assessment

Yes from source inspection: on current main, Gemini web_search has no ADC path and fails before runtime search when no API key is resolved. I did not run a live ADC request because no Google Enterprise ADC credential is available in this review environment.

## Solution Assessment

Unclear: the provider boundary is the Google plugin, but maintainers need to choose whether this is a new `webSearch.auth` mode, reuse of `google-vertex`, or another config/API shape before implementation.
