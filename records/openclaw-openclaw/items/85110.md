## Summary

Keep open. The PR is aimed at the right ACP/subagent task-state bug and now includes current-head live-output proof, but the new classifier still treats colon- or dash-delimited final answers after a progress prefix as progress-only, which can falsely mark a valid required completion as blocked.

## What This Changes

The PR adds a shared required-completion contract for ACP and subagent finalization so missing or progress-only required output and exhausted completion delivery set `terminalOutcome=blocked`, with focused regressions.

## Best Possible Solution

Land the shared required-completion contract after the classifier accepts clear non-progress deliverables after a progress prefix and tests cover those shapes while keeping missing, pure-progress, and delivery-failed required completions blocked.

## Reproduction Assessment

Yes. Current main source shows parented ACP/subagent success paths can persist progress-only or failed-delivery required completions as succeeded, and the PR classifier false-positive is source-reproducible from the new boundary check.

## Solution Assessment

No as-is. The shared finalization contract is the right maintainable shape, but the classifier needs to accept clear colon/dash/same-sentence final deliverables before this is safe to merge.
