## Summary

Keep open: this PR targets the right ACP/subagent task-state boundary and addresses the earlier progress-then-final false positive, but the current classifier still lets progress-intent completions pass when they contain words like `report`, leaving part of the linked false-success bug unfixed.

## What This Changes

The PR adds a required-completion contract for ACP and subagent task finalization so missing, progress-only, or delivery-exhausted required outputs become blocked while real final reports remain successful.

## Best Possible Solution

Land the required-completion contract after tightening progress-intent detection so future-looking progress text remains blocked even when it mentions the artifact it plans to produce, while progress followed by a real final report still succeeds.

## Reproduction Assessment

Yes. Source inspection shows current main can complete ACP/subagent child task records as succeeded when required completion text is only progress or delivery failure is only in `deliveryStatus`, and the PR classifier false negative is reproducible from the exact regex order.

## Solution Assessment

No. The owner boundary and terminalOutcome reuse are the right shape, but the current heuristic should not let progress-intent text bypass blocking merely because it mentions `report`, `result`, or similar artifact words.
