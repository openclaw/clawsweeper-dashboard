## Summary

Keep open: the PR is aimed at the right ACP/subagent completion-state bug, but the current classifier can still block valid final answers that start with progress text and do not contain one of the new allow-list words. The supplied live proof also predates the current head after the force-push, so current-head behavior needs fresh proof before merge.

## What This Changes

The PR adds a shared required-completion contract for ACP and subagent task finalization so missing, progress-only, and delivery-exhausted required completions record a blocked terminal outcome while tested final reports remain successful.

## Best Possible Solution

Land the required-completion contract after replacing the keyword allow-list with a safer pure-progress test, adding regressions for progress-prefixed final diagnoses, and refreshing real behavior proof on the current head.

## Reproduction Assessment

Yes: current main source inspection shows parented ACP/subagent ok outcomes are finalized without a required-final-deliverable check, and the linked issue supplies observed source-repro context. I did not execute tests because this review is read-only.

## Solution Assessment

No: the shared contract is the right boundary, but the classifier should identify truly pure progress text instead of requiring a small set of final-output keywords. A safer repair is to add regressions for progress-prefixed final diagnoses that lack the current indicator words.
