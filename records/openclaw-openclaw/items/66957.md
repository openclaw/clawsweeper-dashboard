## Summary

Keep open. Current main still runs implicit provider discovery before reading `models.mode`, and an open closing PR now owns the direct fix, so the issue should remain open until that PR lands or is closed.

## What This Changes

`models.mode="replace"` should skip implicit provider discovery and use only configured providers during models.json planning.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/82638, keeping the fix limited to a pre-discovery replace-mode guard and regression coverage while leaving pricing-fetch behavior to its separate track.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: call `resolveProvidersForModelsJsonWithDeps()` with `models.mode: "replace"` and an injected implicit resolver, and current main still reaches that resolver before mode handling.

## Solution Assessment

Yes. The narrow maintainable solution is to branch on replace mode before `resolveImplicitProviders()` runs, return only explicit providers for that phase, and keep adjacent pricing or discovery-toggle behavior out of scope.
