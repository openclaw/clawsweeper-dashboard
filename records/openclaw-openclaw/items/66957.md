## Summary

Keep open. Current main still calls implicit provider discovery before it reads `models.mode`, so `replace` mode can still pay the discovery path and merge implicit providers contrary to the documented configured-providers-only contract. The linked #73557 is an open, unmerged closing PR, so this issue should remain open until that fix or an equivalent one lands.

## What This Changes

The issue requests that `models.mode: "replace"` use only explicitly configured providers and skip implicit provider discovery during models.json planning and legacy embedded-agent startup.

## Best Possible Solution

Keep this issue open and land a narrow fix that branches on `cfg.models?.mode === "replace"` before any implicit provider discovery, returns only explicit providers on that path, preserves normalization and source-managed secret enforcement, and adds regression coverage proving the injected implicit resolver is not called in replace mode while merge/default mode still discovers providers.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `planOpenClawModelsJsonWithDeps()` with `cfg.models.mode: "replace"` and an injected implicit resolver; current main reaches the resolver before reading `mode`. The live timing symptom was not rerun here.

## Solution Assessment

Yes in direction. The maintainable fix is to gate discovery before `resolveImplicitProviders()` runs for replace mode, with tests around the injected resolver; the exact open PR still needs normal review before merge.
