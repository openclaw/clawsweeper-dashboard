## Summary

Keep open: current `main` and the latest release still call implicit provider discovery before honoring `models.mode`, and live GitHub shows an open linked fix PR that closes this issue. The right cleanup is not to close or queue another fix, but to review and land or reject the linked PR.

## What This Changes

The issue reports that `models.mode="replace"` still invokes implicit provider discovery, delaying startup despite documented replace-mode semantics.

## Best Possible Solution

Review and land the linked fix PR if it remains correct, preserving merge/default behavior while skipping implicit discovery only for replace mode.

## Reproduction Assessment

Yes. Source inspection on current `main` shows `resolveProvidersForModelsJsonWithDeps` awaits implicit discovery before mode-specific handling, while the config help defines `replace` as using only configured providers.

## Solution Assessment

Yes. The narrow best fix is to branch before implicit discovery for `models.mode === "replace"` and leave merge/default discovery behavior intact; the open linked PR is the current candidate for that path.
