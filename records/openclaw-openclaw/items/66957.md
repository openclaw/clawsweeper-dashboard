## Summary

Keep open: current main and the latest release still call implicit provider discovery before replace-mode handling, and https://github.com/openclaw/openclaw/pull/82638 is the open linked fix path.

## What This Changes

The issue asks for `models.mode="replace"` to skip implicit provider discovery and use only configured providers so startup is not delayed by runtime provider scans.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82638, or replace it with an equivalent early guard plus regression coverage, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes, source-reproducible with high confidence: current main documents replace mode as configured-only but still awaits implicit provider discovery before the mode branch. I did not run the full live startup benchmark.

## Solution Assessment

Yes, the linked PR's early guard is the narrow maintainable fix because it skips only implicit discovery while preserving downstream explicit-provider normalization and merge/default behavior.
