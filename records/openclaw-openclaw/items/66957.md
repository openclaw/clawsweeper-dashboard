## Summary

Keep open. Current main and v2026.5.7 still call implicit provider discovery before `models.mode` is read, and the open fix PR at https://github.com/openclaw/openclaw/pull/73557 already targets this issue with closing syntax.

## What This Changes

The issue requests that `models.mode: "replace"` skip implicit provider discovery and use only explicitly configured providers to avoid startup delays.

## Best Possible Solution

Land the open fix PR or an equivalent narrow guard that skips implicit discovery before provider resolution in replace mode while preserving merge/default behavior and regression coverage.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `resolveProvidersForModelsJsonWithDeps()` or `planOpenClawModelsJsonWithDeps()` with `cfg.models.mode: "replace"` and an injected implicit resolver; current main still reaches that resolver before mode handling.

## Solution Assessment

Yes. The narrow maintainable fix is to branch on replace mode before `resolveImplicitProviders()` runs, keep explicit-provider normalization and secret handling intact, and cover replace versus merge behavior in tests.
