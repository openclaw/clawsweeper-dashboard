## What This Changes

The issue reports that `models.mode="replace"` still runs implicit provider discovery before honoring replace-mode semantics, causing slow startup under explicit-provider configs.

## Best Possible Solution

Review and land or reject the linked fix PR; the desired end state is an early replace-mode guard that skips implicit discovery while preserving merge/default behavior.

## Reproduction Assessment

Yes. Source inspection on current `main` shows `resolveProvidersForModelsJsonWithDeps` awaits implicit discovery before any replace-mode handling, matching the reporter's source reproduction.

## Solution Assessment

Yes. The narrow best fix is to branch before implicit discovery for `models.mode === "replace"` and leave merge/default discovery intact; the open linked PR is the active candidate for that path.
