## Summary

Keep open. Current main still calls implicit provider discovery before reading `models.mode`, so `replace` mode can still violate the documented configured-providers-only contract; the direct fix PR was closed unmerged.

## What This Changes

The issue asks for `models.mode: "replace"` to skip implicit provider discovery and use only explicitly configured providers during models.json planning.

## Best Possible Solution

Add a focused replace-mode guard before implicit provider discovery, preserve explicit-provider normalization and merge/default behavior, and cover replace versus merge/default paths with regression tests.

## Reproduction Assessment

Yes. At source level, calling `resolveProvidersForModelsJsonWithDeps()` or `planOpenClawModelsJsonWithDeps()` with `models.mode: "replace"` and an injected implicit resolver will still reach that resolver before mode handling on current main.

## Solution Assessment

Yes. The narrow maintainable fix is to branch on replace mode before `resolveImplicitProviders()` runs, return only explicit providers for that phase, and keep broader provider-discovery performance plus pricing-fetch behavior out of scope.
