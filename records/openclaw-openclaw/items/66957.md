## What This Changes

The issue asks for `models.mode: "replace"` to skip implicit provider discovery so explicit-provider startup avoids the discovery cost.

## Solution Assessment

Yes. The linked early guard is the narrow maintainable fix because it skips only implicit discovery for replace mode while preserving downstream explicit-provider normalization and merge/default behavior.
