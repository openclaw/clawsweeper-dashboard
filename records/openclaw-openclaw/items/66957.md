## What This Changes

The issue reports that `models.mode="replace"` still runs implicit provider discovery, violating explicit-only provider semantics and causing large startup delays.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/82638; the desired implementation is an early replace-mode guard that skips implicit discovery while preserving merge/default behavior.

## Reproduction Assessment

Yes, at source level. Current main still awaits implicit provider discovery before any replace-mode handling in `resolveProvidersForModelsJsonWithDeps`; I did not rerun the wall-clock startup benchmark.

## Solution Assessment

Yes. The narrow maintainable fix is to branch before implicit discovery when `models.mode === "replace"` and leave merge/default discovery intact; the open linked PR is the active candidate for that path.
