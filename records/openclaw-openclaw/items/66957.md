## Summary

Keep open. Current main still calls implicit provider discovery before reading `models.mode`, and open unmerged PR #73557 already targets this exact bug.

## What This Changes

The issue asks for `models.mode="replace"` to skip implicit provider discovery and use only explicitly configured providers during models.json planning/startup.

## Best Possible Solution

Keep this issue open until #73557 or an equivalent narrow fix lands with regression coverage proving replace mode never invokes the implicit resolver while merge mode still does.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `planOpenClawModelsJsonWithDeps()` with `cfg.models.mode: "replace"` and an injected implicit resolver; current main reaches that resolver before it reads the mode.

## Solution Assessment

Yes in direction. The narrow maintainable fix is to branch on replace mode before `resolveImplicitProviders()` runs, preserve explicit-provider normalization and secret handling, and cover the injected resolver behavior in tests.
