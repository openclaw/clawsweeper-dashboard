## Summary

Keep open. Current main still resolves implicit providers before reading `models.mode`, so the documented `replace` behavior is still violated; the direct fix PR was closed unmerged and the remaining repair is narrow.

## What This Changes

The issue requests that `models.mode: "replace"` use only configured providers and skip implicit provider discovery during models.json planning.

## Best Possible Solution

Land a focused replace-mode guard before implicit provider discovery while keeping explicit-provider normalization, secret handling, merge/default behavior, and the separate pricing-fetch issue out of scope.

## Reproduction Assessment

Yes, at source level. Calling `resolveProvidersForModelsJsonWithDeps()` or `planOpenClawModelsJsonWithDeps()` with `models.mode: "replace"` and an injected implicit resolver will still reach that resolver before mode handling.

## Solution Assessment

Yes. The narrow maintainable fix is to branch on replace mode before `resolveImplicitProviders()` runs, return only explicit providers for that phase, and cover replace versus merge/default behavior in focused tests.
