## Summary

Keep open. Current main still calls implicit provider discovery before `models.mode` is evaluated, and the direct fix PR at https://github.com/openclaw/openclaw/pull/73557 was closed unmerged rather than replaced by a shipped implementation.

## What This Changes

The issue reports that `models.mode="replace"` still runs implicit provider discovery and delays startup despite the documented replace-mode contract.

## Best Possible Solution

Land a narrow replace-mode guard before implicit discovery while preserving explicit-provider normalization, secret handling, and existing merge/default behavior; keep pricing-fetch behavior tracked separately.

## Reproduction Assessment

Yes, at source level. Call `resolveProvidersForModelsJsonWithDeps()` or `planOpenClawModelsJsonWithDeps()` with `cfg.models.mode: "replace"` and an injected implicit resolver; current main reaches that resolver before mode handling.

## Solution Assessment

Yes. The narrow maintainable fix is to branch on replace mode before `resolveImplicitProviders()` runs, return only explicit providers for that phase, and cover replace versus merge/default behavior in focused tests.
