## Summary

Keep open. Current main still calls implicit provider discovery before it reads `models.mode`, so `replace` mode can still pay the discovery cost and violate the documented configured-providers-only contract; the direct fix PR was closed unmerged.

## What This Changes

The issue asks OpenClaw to make `models.mode="replace"` use only explicit providers and skip implicit provider discovery during models.json planning.

## Best Possible Solution

Add a focused replace-mode guard before implicit provider discovery, preserve explicit-provider normalization and merge/default behavior, and cover replace versus merge/default paths with regression tests.

## Reproduction Assessment

Yes, at source level. Calling `resolveProvidersForModelsJsonWithDeps()` or `planOpenClawModelsJsonWithDeps()` with `models.mode: "replace"` and an injected implicit resolver will still reach that resolver before mode handling on current main.

## Solution Assessment

Yes. The narrow maintainable fix is to branch on replace mode before `resolveImplicitProviders()` runs, return only explicit providers for that phase, and keep broader provider-discovery performance and pricing-fetch behavior out of scope.
