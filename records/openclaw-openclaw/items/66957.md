## Summary

Keep open. Current main still invokes implicit provider discovery before `models.mode` is evaluated, and the provided GitHub context shows open PR #73557 already references this issue with closing syntax, so the issue should remain open until that fix or an equivalent one lands.

## What This Changes

The issue requests that `models.mode="replace"` skip implicit provider discovery and use only explicitly configured providers during model planning.

## Best Possible Solution

Land #73557 or an equivalent narrow fix that branches before implicit discovery in replace mode while preserving merge-mode discovery and explicit-provider normalization, with regression coverage for both paths.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `planOpenClawModelsJsonWithDeps()` or `resolveProvidersForModelsJsonWithDeps()` with `cfg.models.mode: "replace"` and an injected implicit resolver; current main invokes that resolver before reading the mode.

## Solution Assessment

Yes in direction. The narrow maintainable solution is to skip `resolveImplicitProviders()` when `models.mode` is `replace`, keep explicit-provider normalization/secret handling, and add regression coverage proving merge mode still discovers implicit providers.
