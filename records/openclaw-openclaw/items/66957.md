## Summary

Keep open. Current main still calls implicit provider discovery before `models.mode` is read, so `replace` mode can still violate the documented configured-providers-only contract; the direct fix PRs are closed unmerged and the open related PRs cover adjacent scoping/pricing work rather than this exact skip.

## What This Changes

`models.mode="replace"` should skip implicit provider discovery and use only explicitly configured providers during models.json planning.

## Best Possible Solution

Add a focused guard before implicit discovery in `src/agents/models-config.plan.ts`, keep explicit-provider normalization and merge/default behavior intact, and leave pricing-fetch behavior to the separate pricing issue.

## Reproduction Assessment

Yes. A high-confidence source-level repro exists: call `resolveProvidersForModelsJsonWithDeps()` or `planOpenClawModelsJsonWithDeps()` with `models.mode: "replace"` and an injected implicit resolver, and current main still reaches that resolver before mode handling.

## Solution Assessment

Yes. The narrow maintainable fix is to branch on `replace` mode before `resolveImplicitProviders()` runs, return only explicit providers for that phase, and keep broader discovery scoping plus pricing fetches out of scope.
