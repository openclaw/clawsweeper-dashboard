## Summary

Keep open. Current main still collapses Gemini CLI OAuth credential extraction failures into the generic setup error, while this PR remains a narrow diagnostics and regression-test change for the active bundled Google plugin. Open #54289 may supersede it if maintainers remove or gate automatic extraction, but that product/security decision has not landed.

## What This Changes

The PR adds diagnostic state to Google Gemini CLI OAuth credential extraction, appends failure details to `resolveOAuthClientConfig()`, resets diagnostics with the credential cache, and adds focused regression tests in `extensions/google/oauth.test.ts`.

## Best Possible Solution

If automatic Gemini CLI credential extraction remains supported, land or adapt the focused diagnostics with any desired path redaction. If #54289 changes the extraction policy, rework this PR to match that decision or close it as superseded after the policy fix lands.

## Reproduction Assessment

Yes. On current main, the source path is enough to reproduce: with no OAuth env vars and a missing or unparseable Gemini CLI credential source, `resolveOAuthClientConfig()` falls through to the generic error at `extensions/google/oauth.credentials.ts:270`; the PR adds regression coverage for those cases.

## Solution Assessment

Unclear as a final product direction, but the PR is a narrow maintainable fix if extraction remains supported. The safer alternative is to first resolve #54289 and then keep only diagnostics that match the chosen consent or removal policy.
