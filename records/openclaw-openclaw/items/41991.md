## Summary

Keep open. Current main still collapses Gemini CLI OAuth extraction failures into the generic setup error, and this PR addresses that narrow diagnostics problem, but it remains blocked by the missing changelog entry and maintainer review of local path/raw error disclosure while #54289 is open.

## What This Changes

The PR adds diagnostic state to the Google plugin's Gemini CLI OAuth credential extraction, appends failure details to `resolveOAuthClientConfig()` errors, resets the diagnostic cache, and adds regression tests.

## Best Possible Solution

Keep these diagnostics in the Google plugin only if maintainers keep automatic Gemini CLI extraction, add the changelog entry, and settle path/error disclosure; otherwise rework or supersede it with the #54289 policy fix.

## Reproduction Assessment

Yes. Source inspection on current main shows env lookup falls through to `extractGeminiCliCredentials()`, extraction failures return `null` without preserving details, and `resolveOAuthClientConfig()` throws the generic setup error; the PR tests model those failure paths.

## Solution Assessment

Unclear. The diagnostics approach is narrow if automatic extraction remains supported, but #54289 may change that behavior and the patch still needs a changelog entry plus maintainer-approved redaction policy.
