## Summary

Keep open: current main still collapses Gemini CLI OAuth extraction failures into the generic install-or-set-env-var setup error, while this PR adds focused diagnostic detail and regression coverage. The patch is narrow and proof-backed, but merge needs maintainer security-policy review because it exposes local path and read-error metadata in an auth setup path and may be superseded by the open extraction-policy work.

## What This Changes

The PR adds Gemini CLI OAuth extraction diagnostic state to the Google plugin and tests resolver errors for missing binary, missing credential source, parse failure, and read/read-traversal failures.

## Best Possible Solution

Land the diagnostic categories only if automatic extraction remains supported and maintainers accept the metadata disclosure; otherwise carry the useful failure categories into the gated or removal path tracked by https://github.com/openclaw/openclaw/issues/54289.

## Reproduction Assessment

Yes. Source inspection on current main shows failed Gemini CLI extraction returns null and resolveOAuthClientConfig() then throws the same generic setup error; the PR body also supplies Windows terminal proof for the after-fix missing-binary detail.

## Solution Assessment

Yes for the narrow diagnostics problem if automatic extraction stays. The final merge path is unclear until maintainers decide whether the local metadata disclosure is acceptable and how this relates to the open extraction-policy issue.
