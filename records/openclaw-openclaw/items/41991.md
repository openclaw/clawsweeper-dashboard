## Summary

Keep open. Current main still collapses Gemini CLI OAuth credential extraction failures into the generic setup error, so the PR is not obsolete; the remaining blockers are maintainer review against the open #54289 security/product thread, path/error-detail disclosure, and the missing required changelog entry.

## What This Changes

The PR adds Gemini CLI OAuth extraction diagnostics in the bundled Google plugin, appends the captured detail to `resolveOAuthClientConfig()` failures, resets diagnostic state with the credential cache, and adds focused OAuth tests.

## Best Possible Solution

If automatic Gemini CLI client-config extraction remains supported, land or adapt these focused diagnostics after adding a changelog entry and applying whatever path/error redaction maintainers want. If #54289 changes the extraction policy, rework this patch around that policy or close it as superseded after the security decision lands.

## Reproduction Assessment

Yes. With no OAuth client env vars and a missing, unreadable, or unparseable Gemini CLI credential source, current main reaches `resolveOAuthClientConfig()` and throws the generic setup error; the PR's focused tests model those failure paths.

## Solution Assessment

Unclear as the final product direction because #54289 may change the underlying extraction behavior. As a narrow implementation, this is a maintainable diagnostics fix if extraction remains, with path redaction and changelog handled before merge.
