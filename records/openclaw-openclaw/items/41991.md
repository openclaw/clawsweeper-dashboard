## Summary

Keep open. Current main still has the generic Gemini CLI OAuth setup failure, and the PR is a focused diagnostics improvement, but it should not merge until the required changelog entry is added and maintainers decide how this interacts with the open security/product question in #54289 and diagnostic path disclosure.

## What This Changes

The PR adds Gemini CLI OAuth extraction diagnostics in the bundled Google plugin, appends the captured failure detail to `resolveOAuthClientConfig()` errors, resets that state with the credential cache, and adds focused OAuth tests.

## Best Possible Solution

If automatic Gemini CLI client-config extraction remains supported, land or adapt these diagnostics with a single-line changelog entry and maintainer-approved redaction of local path/error details. If #54289 changes the extraction policy, rework this patch around that policy or close it as superseded after the security decision lands.

## Reproduction Assessment

Yes. With OAuth client env vars absent and the Gemini CLI binary, oauth2.js source, parseable credentials, or readable files missing, current main reaches `resolveOAuthClientConfig()` and throws the generic setup error; the PR’s added tests model those failure paths.

## Solution Assessment

Unclear as the final product direction because #54289 may change the underlying extraction behavior. As a narrow diagnostics fix, the approach is maintainable if extraction stays, provided the changelog and diagnostic disclosure questions are resolved first.
