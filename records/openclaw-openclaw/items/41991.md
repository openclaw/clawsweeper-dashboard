## Summary

Keep open: current main still collapses Gemini CLI OAuth extraction failures into the generic setup error, while this PR is not merge-ready because it lacks the required changelog entry and still needs maintainer judgment on diagnostic disclosure and the related extraction-policy issue.

## What This Changes

The PR adds Gemini CLI OAuth extraction diagnostics in the bundled Google plugin, appends those details to `resolveOAuthClientConfig()` setup errors, resets diagnostic cache state, and adds focused OAuth tests.

## Best Possible Solution

If automatic Gemini CLI client-config extraction remains supported, land or adapt these diagnostics with a single-line changelog entry and maintainer-approved redaction of local path/error details; if #54289 changes the extraction policy, rework this patch around that decision or close it as superseded after the policy fix lands.

## Reproduction Assessment

Yes. Source inspection shows that with OAuth client env vars absent and the Gemini CLI binary/source/parse/read path failing, current main reaches `resolveOAuthClientConfig()` and throws the generic setup error; the PR’s added tests model those failure paths.

## Solution Assessment

Unclear as the final product direction because #54289 may change the underlying extraction behavior. As a narrow diagnostics fix, the approach is maintainable if extraction stays, provided the changelog and diagnostic disclosure questions are resolved first.
