## Summary

Keep open. Current main still lacks a per-run browser cleanup dispatch guard, and this PR is the active narrow fix for the linked duplicate-cleanup bug with no blocking review findings from this read-only pass.

## What This Changes

The PR adds `browserCleanupDispatchedAt` to subagent run records, guards browser cleanup dispatch in `completeSubagentRun`, resets the flag on run replacement, and adds a parallel-completion regression test.

## Best Possible Solution

Land this PR or an equivalent lifecycle-controller guard so browser cleanup dispatch is deduped per subagent run while preserving the current announce cleanup and steer-restart reset semantics.

## Reproduction Assessment

Yes. The source-level reproduction is two parallel `completeSubagentRun` calls for the same `runId` with `triggerCleanup: true`; current main reaches the unguarded browser cleanup wrapper from both completion paths.

## Solution Assessment

Yes. A per-record dispatch flag set synchronously before the awaited cleanup wrapper, with reset on run replacement, is the narrowest maintainable fix and avoids changing browser extension IPC semantics.
