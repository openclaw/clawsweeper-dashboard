## Summary

Keep open: current main still lacks a per-run browser cleanup dispatch guard, and this PR is a narrow lifecycle-controller fix with no blocking code or security findings found in review.

## What This Changes

The PR adds a per-run browser cleanup dispatch timestamp to subagent run records, resets it on rearm, guards cleanup dispatch in the lifecycle controller, and adds a parallel-completion regression test.

## Best Possible Solution

Land this PR or an equivalent lifecycle-controller dispatch guard after required exact-head checks complete, then let https://github.com/openclaw/openclaw/issues/68668 close through the merge.

## Solution Assessment

Yes for the code shape. A synchronous per-run guard in the shared lifecycle helper, reset on run replacement, is the narrow maintainable fix for this duplicate wrapper dispatch.
