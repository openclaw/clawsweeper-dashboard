## Summary

Keep open. Current main still has the unguarded browser cleanup wrapper path, and this PR is a narrow, relevant fix with regression coverage and no blocking correctness or security finding from this read-only review.

## What This Changes

The PR adds a `browserCleanupDispatchedAt` dispatch flag, guards browser cleanup in `completeSubagentRun`, resets the flag on steer-restart replacement, and adds a parallel-completion regression test.

## Best Possible Solution

Land this PR or an equivalent controller-level guard so browser lifecycle cleanup dispatch is deduped per subagent run while preserving the existing announce cleanup flow and restart reset semantics.

## Reproduction Assessment

Yes. The high-confidence reproduction is two parallel `completeSubagentRun` calls for the same `runId` with `triggerCleanup: true`; current main has no guard before the browser cleanup wrapper, and the PR adds that focused unit coverage.

## Solution Assessment

Yes. A per-record dispatch flag set synchronously before the await, plus reset on run replacement, is the narrowest maintainable fix for this duplicate side effect and avoids changing browser extension IPC semantics.
