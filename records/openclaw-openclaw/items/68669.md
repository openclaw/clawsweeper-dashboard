## Summary

Keep open. Current main still calls the browser cleanup wrapper without a per-run dispatch guard, and the latest PR head now scopes the new guard only to that wrapper while preserving runtime retirement and announce cleanup for duplicate completion callers.

## What This Changes

The PR adds a per-run browser cleanup dispatch timestamp, resets it on rearm, skips duplicate browser cleanup wrapper calls during subagent completion, and adds parallel-completion lifecycle regression tests.

## Best Possible Solution

Land this narrow guard if maintainers want the focused duplicate-wrapper fix now, or intentionally supersede it with the broader lifecycle-race PR only if that larger ordering change becomes the chosen canonical path.

## Reproduction Assessment

Yes. Source inspection shows both completion paths converge on `completeSubagentRun`, and current main has no per-run guard around the browser cleanup wrapper; the PR body also supplies terminal before/after counts for the duplicate dispatch.

## Solution Assessment

Yes. A per-run dispatch flag scoped only to the browser cleanup wrapper is the narrow maintainable fix, and the latest head preserves the completion finalization tail that the earlier version accidentally skipped.
