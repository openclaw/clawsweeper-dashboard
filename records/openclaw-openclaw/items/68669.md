## Summary

This PR should stay open because the latest head still returns from `completeSubagentRun` when the duplicate browser-cleanup flag is already set, before the existing runtime-retirement and announce-cleanup tail can run. The dispatch flag is the right narrow mechanism, but the duplicate path needs to skip only the browser wrapper and then continue finalization.

## What This Changes

The PR adds a per-run browser cleanup dispatch timestamp to subagent records, resets it on rearm, guards duplicate cleanup dispatch in the lifecycle controller, and adds a parallel-completion regression test.

## Best Possible Solution

Keep the dispatch marker, but make the duplicate path skip only `cleanupBrowserSessionsForLifecycleEnd` and then continue through runtime retirement and `startSubagentAnnounceCleanupFlow`, with a regression test where the first browser cleanup is slow and the second completion signal can start the tail.

## Reproduction Assessment

Yes. Source inspection shows both completion paths can call `completeSubagentRun(..., triggerCleanup: true)` for the same run, and the PR body supplies before/after terminal output for 100 parallel-completion trials.

## Solution Assessment

No, not as currently written. The dispatch flag is the right narrow mechanism, but the duplicate branch should skip only browser cleanup rather than returning before the completion tail.
