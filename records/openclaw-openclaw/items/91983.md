## Summary

The report is still necessary: current main can recover when Anthropic rejects an invalid historical signature, but it still concatenates every signature delta received for a block and has no invariant or regression test preventing an oversized value from being persisted. The reporter provides fresh, quantified production evidence after the recovery fix shipped, while the suggested cross-retry explanation is not yet confirmed because transport state is created per stream invocation.

## What This Changes

Prevent anomalously oversized Anthropic thinking signatures from being persisted and poisoning subsequent session replay.

## Best Possible Solution

Preserve the shipped replay recovery, capture and test the malformed event sequence, then enforce one complete signature per Anthropic thinking block at the proven stream or persistence boundary without relying on an undocumented fixed-size threshold.

## Reproduction Assessment

Yes for the poisoned-replay symptom: current source recognizes Anthropic invalid-thinking-signature errors and the related report includes repeated live failures. No high-confidence current-main reproduction yet establishes how the oversized signature is created.

## Solution Assessment

No, not fully. Resetting at the OpenClaw stream-call retry boundary appears redundant because assembly state is already invocation-local, and a hard size cap lacks contract proof; the narrowest maintainable fix must follow a captured malformed event sequence.
