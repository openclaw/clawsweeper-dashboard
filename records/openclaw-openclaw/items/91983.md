## Summary

Keep open: current main and v2026.6.5 still replay any non-empty Anthropic thinking signature without a size or shape guard, while this fresh report describes newly generated oversized signatures after the merged recovery work in https://github.com/openclaw/openclaw/pull/90697.

## What This Changes

Report requests prevention and recovery for oversized Anthropic `thinkingSignature` values being persisted to session JSONL and later rejected on replay.

## Best Possible Solution

Confirm the generation path with redacted transcript/SSE evidence, then fix signature assembly so buffers cannot cross message or retry boundaries and add a deliberate tested fallback for anomalous persisted signatures if maintainers accept one.

## Reproduction Assessment

Partly. Source inspection gives a high-confidence replay path for any persisted non-empty oversized `thinkingSignature`, but the exact live path that creates 64KB+ signatures still needs redacted transcript or SSE proof.

## Solution Assessment

Unclear. Resetting signature buffers at the proven accumulation boundary would be the clean fix, while a write-time cap is a policy fallback because current code intentionally avoids signature length heuristics.
