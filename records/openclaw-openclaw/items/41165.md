## Summary

Keep open. Current main still has the source-reproducible heartbeat/session precedence path described in the later comments, and open PR #56092 is already the implementation candidate with closing syntax rather than a reason to close early.

## What This Changes

The issue reports Telegram direct-message and heartbeat/session routing bleed where DM or system-event-triggered heartbeat traffic can enter the wrong session.

## Best Possible Solution

Land or replace #56092 for configured heartbeat-session precedence and system-event handling, then separately decide whether Telegram DMs should continue sharing the main session by default or switch to isolated defaults with docs/tests updated.

## Reproduction Assessment

Yes, from source inspection. An exec completion can enqueue and wake heartbeat with the originating session key, and current `resolveHeartbeatSession` consumes that forced key before `heartbeat.session`; I did not run a live Telegram gateway repro in this read-only review.

## Solution Assessment

Yes for keeping this issue open behind #56092, because that PR targets the narrow heartbeat/session precedence bug described in the later comments. The requested default Telegram DM isolation is broader because current docs and tests still define main-session sharing by default.
