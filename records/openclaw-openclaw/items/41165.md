## Summary

Keep open. Current main still has the source-reproducible heartbeat/session routing path described in the comments, and the related open PR #56092 is the correct implementation candidate to review rather than closing this issue before merge.

## What This Changes

The issue asks for Telegram DM and heartbeat/system-event traffic to stop sharing or polluting `agent:main:main` and to respect isolated Telegram or configured heartbeat sessions.

## Best Possible Solution

Land or replace #56092 for the configured-heartbeat-session precedence and system-event migration bug, then handle the Telegram default-DM-to-main behavior as a separate explicit product/docs decision.

## Reproduction Assessment

Yes, from source inspection. A background exec completion in a Telegram DM can enqueue and wake heartbeat with that DM session key, and current `resolveHeartbeatSession` consumes the forced key before `heartbeat.session`; I did not run a live Telegram gateway reproduction in this read-only review.

## Solution Assessment

Yes for keeping this issue open behind #56092, because that PR targets the narrow heartbeat/session precedence bug described in the later comments. The requested default Telegram DM isolation is broader because current docs and tests still define main-session sharing by default.
