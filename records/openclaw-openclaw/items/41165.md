## Summary

Keep open. Current main still has the source-reproducible system-event heartbeat routing path, and the related open PR #56092 is the right implementation candidate to review rather than closing this issue before merge.

## What This Changes

The issue asks for Telegram DM and heartbeat sessions to stay isolated so ordinary DM traffic and system-triggered heartbeats do not pollute each other’s transcripts.

## Best Possible Solution

Land or replace #56092 for configured `heartbeat.session` precedence and system-event migration, then separately decide whether Telegram DMs should keep documented main-session sharing or become isolated by default.

## Reproduction Assessment

Yes, from source inspection. An exec completion in a Telegram DM can enqueue and wake heartbeat with that DM session key, and current heartbeat resolution consumes the forced key before the configured heartbeat session.

## Solution Assessment

Yes for keeping this issue open behind #56092, because that PR targets the narrow source-reproducible heartbeat/session precedence bug. The default Telegram DM isolation request is broader and needs an explicit docs/tests/product decision.
