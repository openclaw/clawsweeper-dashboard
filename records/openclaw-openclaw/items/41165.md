## Summary

Keep open. Current main still has a source-reproducible path where an exec/system-event heartbeat wake carries the originating session key into heartbeat resolution before the configured `heartbeat.session`, and the linked fix PR is still open rather than merged.

## What This Changes

The issue requests stronger isolation so Telegram DM traffic and system-event-triggered heartbeat runs cannot pollute `agent:main:main` or an explicitly configured heartbeat session.

## Best Possible Solution

Review, land, or replace https://github.com/openclaw/openclaw/pull/56092 for configured heartbeat-session precedence, then handle Telegram default DM isolation separately with matching docs and tests if maintainers choose that policy.

## Reproduction Assessment

Yes, source-level. Current main lets exec notify-on-exit queue and wake heartbeat with the originating session key, and heartbeat resolution accepts that forced key before the documented `heartbeat.session` setting.

## Solution Assessment

Yes for keeping this issue open behind the linked implementation candidate. The narrow maintainable fix is configured heartbeat-session precedence for system-event-triggered wakes; changing Telegram DM defaults should be decided separately.
