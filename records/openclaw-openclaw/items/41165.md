## Summary

Keep open: current main still has a source-level path where exec/system-event heartbeat wakes can carry the originating session into heartbeat resolution before the documented `heartbeat.session` setting. The open implementation candidate at https://github.com/openclaw/openclaw/pull/56092 uses closing syntax for this issue, so the issue should stay open until that PR or a replacement lands.

## What This Changes

Telegram DM and heartbeat/system-event traffic can be associated with the wrong session, polluting `agent:main:main` or an active Telegram DM transcript.

## Best Possible Solution

Land a rebased or replacement fix where configured `heartbeat.session` wins for system-event-triggered wakes, queued system events move safely without cross-session loss, and the default Telegram DM sharing behavior is either explicitly preserved in docs or changed with maintainer-approved compatibility coverage.

## Reproduction Assessment

Yes, source-level. Current main queues exec notify-on-exit events with the originating session key, forwards that key through the wake scheduler, and resolves it before the documented `heartbeat.session` setting.

## Solution Assessment

Yes for keeping this issue open behind the linked implementation candidate. The narrow maintainable fix is heartbeat-session precedence for event-triggered wakes; changing default Telegram DM-to-main sharing should remain a separate compatibility decision.
