## Summary

Keep open. Current main still has the source-reproducible heartbeat/session precedence path, and https://github.com/openclaw/openclaw/pull/56092 is still the open implementation candidate that references this issue.

## What This Changes

The issue asks for Telegram DMs and heartbeat runs to remain session-isolated so direct-chat traffic cannot pollute the main/heartbeat session.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/56092 for configured heartbeat-session precedence and safe system-event handling, then settle default Telegram DM isolation as a separate product/docs decision.

## Reproduction Assessment

Yes, source-level. Current main lets exec notify-on-exit queue and wake heartbeat with the originating session key, and heartbeat resolution accepts that forced key before the documented `heartbeat.session` setting.

## Solution Assessment

Yes for keeping this issue open behind the linked implementation candidate. The narrow maintainable fix is to make configured heartbeat sessions win for system-event-triggered wakes; changing default Telegram DM isolation should be handled separately.
