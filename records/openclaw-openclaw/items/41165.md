## Summary

Keep open. Current main still lets exec/system-event heartbeat wakes carry the originating session into heartbeat resolution before an explicit `heartbeat.session`, and https://github.com/openclaw/openclaw/pull/56092 is still an open closing PR for the narrow fix.

## What This Changes

The issue asks for Telegram DM and heartbeat/system-event runs to avoid sharing or polluting `agent:main:main`, especially when an explicit `heartbeat.session` is configured.

## Best Possible Solution

Make configured `heartbeat.session` win for system-event-triggered wakes while preserving pending system events correctly, then settle Telegram default DM sharing as a separate documented product decision or explicit routing change.

## Reproduction Assessment

Yes, source-level. Current main queues and wakes exec notify-on-exit with the originating session key, the wake scheduler forwards that key, and heartbeat resolution accepts the forced key before the documented `heartbeat.session` setting.

## Solution Assessment

Yes for keeping this issue open behind the linked implementation candidate. The narrow maintainable fix is configured heartbeat-session precedence for system-event-triggered wakes; default Telegram DM isolation should remain a separate maintainer product decision.
