## Summary

Keep open. Current main still has the source-reproducible heartbeat/session precedence path, and https://github.com/openclaw/openclaw/pull/56092 remains the open implementation candidate with closing syntax for this issue rather than a reason to close it early.

## What This Changes

The issue asks for Telegram DM and heartbeat-triggered traffic to stop polluting `agent:main:main` and to remain in the intended direct or configured heartbeat session.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/56092 for configured heartbeat-session precedence and safe system-event migration, then handle Telegram default DM isolation as a separate documented policy decision.

## Reproduction Assessment

Yes, from source inspection. An exec completion in a Telegram DM can enqueue and wake heartbeat with that DM session key, and current heartbeat resolution consumes the forced key before the configured heartbeat session.

## Solution Assessment

Yes for keeping this issue open behind https://github.com/openclaw/openclaw/pull/56092, because that PR targets the narrow source-reproducible heartbeat/session precedence bug. The default Telegram DM isolation request is broader and needs a separate maintainer product decision.
