## Summary

Keep open. Current main still has a source-confirmed heartbeat session-routing path where an originating session from exec/system events wins before an explicit `heartbeat.session`, and Telegram DMs still default to the agent main session by documented behavior; #56092 is the open closing PR to review rather than closing this issue first.

## What This Changes

The issue asks for Telegram DM and heartbeat session isolation so direct-chat traffic and heartbeat/system events cannot pollute `agent:main:main` or a configured heartbeat session.

## Best Possible Solution

Land or replace #56092 for the heartbeat configured-session precedence bug, then make an explicit maintainer decision on whether Telegram DMs should continue documented main-session sharing or move to isolated default sessions with matching tests and docs.

## Reproduction Assessment

Yes, source-reproducible: an exec `notifyOnExit` event queues a wake with the originating session key, and `resolveHeartbeatSession` returns that forced key before `heartbeat.session`. I did not run a live Telegram gateway repro in this read-only review.

## Solution Assessment

Yes for the heartbeat half: making configured heartbeat sessions win over event-originating sessions is the narrowest maintainable fix. The Telegram default-DM isolation half needs a separate product/docs decision because current docs and tests still define DM-to-main as the default.
