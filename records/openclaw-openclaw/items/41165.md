## Summary

Keep open: current main still resolves an event wake's forced session before the documented `heartbeat.session` setting, and the linked open implementation PR should be reviewed or replaced rather than closing this issue ahead of a merged fix.

## What This Changes

Telegram DM and heartbeat-triggered system-event traffic should remain session-isolated so heartbeat/main history is not polluted by ordinary direct-chat turns.

## Best Possible Solution

Land a reviewed current-main fix where configured `heartbeat.session` wins for system-event-triggered wakes without losing queued events, then separately settle or document Telegram DM default-to-main behavior with compatibility tests.

## Reproduction Assessment

Yes, source-level. Current main forwards exec/system-event wake session keys into heartbeat resolution, and `resolveHeartbeatSession` consumes that forced key before the documented `heartbeat.session` setting.

## Solution Assessment

Yes for keeping this issue open behind the linked implementation candidate. The narrow fix is heartbeat-session precedence for event wakes; changing Telegram's default DM sharing remains a separate maintainer product/compatibility decision.
