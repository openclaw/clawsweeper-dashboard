## Summary

Keep open. Current main still has a source-reproducible path where an exec/system-event heartbeat wake can carry an originating session key that is resolved before the configured heartbeat session, and the linked implementation candidate is still open rather than merged.

## What This Changes

The issue asks for Telegram DM and heartbeat session routing to avoid polluting `agent:main:main` or another active conversation session.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/56092 for configured heartbeat-session precedence and safe system-event handling, then handle default Telegram DM isolation as a separate cross-channel product/docs decision.

## Reproduction Assessment

Yes, source-level. Current main lets exec notify-on-exit queue and wake heartbeat with the originating session key, and heartbeat resolution accepts that forced key before the documented `heartbeat.session` setting.

## Solution Assessment

Yes for keeping this issue open behind the linked implementation candidate. The narrow maintainable fix is configured heartbeat-session precedence for system-event-triggered wakes; changing Telegram DM defaults should be decided separately.
