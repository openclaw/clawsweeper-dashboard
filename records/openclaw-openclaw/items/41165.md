## Summary

Keep open. Current main still has a source-reproducible heartbeat/session-routing path where an exec-triggered wake carries the originating session into heartbeat resolution before the configured `heartbeat.session`, and an open linked PR uses closing syntax for this issue rather than being merged.

## What This Changes

The issue asks OpenClaw to prevent Telegram DMs and heartbeat/system-event traffic from polluting each other's sessions, especially `agent:main:main`.

## Best Possible Solution

Review, rebase, land, or replace https://github.com/openclaw/openclaw/pull/56092 for configured heartbeat-session precedence, then settle Telegram default DM isolation separately with docs and tests.

## Reproduction Assessment

Yes, source-level. Current main lets exec notify-on-exit enqueue and wake heartbeat with the originating session key, and heartbeat resolution accepts that forced key before the documented `heartbeat.session` setting.

## Solution Assessment

Yes for keeping this issue open behind the linked implementation candidate. The narrow fix is configured heartbeat-session precedence for system-event-triggered wakes; Telegram default DM isolation should remain a separate maintainer product decision.
