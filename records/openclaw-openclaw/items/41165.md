## Summary

Keep open. Current main and the latest release still preserve a source-reproducible path where exec/system-event heartbeat wakes can force the originating session before the configured heartbeat session, and Telegram private DMs still default to the agent main session unless configured otherwise; the open implementation PR at https://github.com/openclaw/openclaw/pull/56092 uses closing syntax and should be reviewed rather than closing this issue early.

## What This Changes

Telegram and heartbeat session routing should prevent direct-message and system-event heartbeat traffic from polluting the main or configured heartbeat session.

## Best Possible Solution

Make configured heartbeat.session win for system-event-triggered wakes while preserving pending system events correctly, then settle Telegram default DM sharing as a separate documented product decision or explicit routing change.

## Reproduction Assessment

Yes, source-level. Current main lets exec notify-on-exit enqueue and wake heartbeat with the originating session key, and heartbeat resolution accepts that forced key before the documented heartbeat.session setting.

## Solution Assessment

Yes for keeping this issue open behind the linked implementation candidate. The narrow maintainable fix is configured heartbeat-session precedence for system-event-triggered wakes; default Telegram DM isolation should remain a separate maintainer product decision.
