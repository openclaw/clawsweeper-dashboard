## Summary

Keep open: current main still has a source-level path where exec/system-event heartbeat wakes carry an originating session key that is resolved before configured `heartbeat.session`, and the open closing PR is the right next implementation candidate rather than closing this issue early.

## What This Changes

Telegram DMs and system-event-triggered heartbeat wakes should not pollute `agent:main:main` and should respect explicit heartbeat session isolation.

## Best Possible Solution

Land a reviewed fix where configured `heartbeat.session` wins for system-event-triggered wakes without losing queued system events, then handle Telegram default DM sharing through an explicit compatibility decision with tests and docs.

## Reproduction Assessment

Yes, source-level. Current main queues exec notify-on-exit events with the originating session key, forwards that key through the wake scheduler, and resolves it before the documented `heartbeat.session` setting.

## Solution Assessment

Yes for keeping this issue open behind the linked implementation candidate. The narrow maintainable fix is heartbeat-session precedence for system-event-triggered wakes; changing Telegram's default DM sharing should remain a separate compatibility decision.
