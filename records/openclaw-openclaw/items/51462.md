## Best Possible Solution

Rebase the focused handler fix, add regression coverage for tool-call-only `message_end`, and land it after redacted vLLM proof; track relay-visible tool-call metadata separately if maintainers want that UX.

## Solution Assessment

Yes for the scoped subscription-handler state bug. A presence-only block-type check is narrower than adding synthetic relay markers, while the existing ID-based extraction helper is too strict for providers that omit tool-call IDs.
