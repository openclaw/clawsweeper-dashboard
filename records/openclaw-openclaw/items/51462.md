## Summary

Keep open: current main still gates final assistant updates on text or media, so this PR remains useful for textless structured tool-call turns, but it needs porting to current main plus changelog and focused regression coverage before merge.

## What This Changes

The PR adds a tool-call-block presence check to the embedded message-end handler and includes it in the assistant update emission gate.

## Best Possible Solution

Land a rebased, current-main version that emits the assistant update for textless structured tool-call messages, adds focused regression coverage and a changelog entry, and leaves relay-visible tool-call metadata for a separate follow-up.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `handleMessageEnd` with an assistant message whose content is only a `toolCall`/`toolUse`/`functionCall` block and observe that current main's `(cleanedText || hasMedia)` guard does not emit an assistant event.

## Solution Assessment

Yes for the scoped subscription-handler state bug. A presence-only block-type check is narrower than adding relay markers, and using the existing ID-based extraction helper would be too strict for providers that omit tool-call IDs.
