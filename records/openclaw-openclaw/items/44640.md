Full review comments:

- **[P1] Preserve atomic writes in the rename fallback:** `src/infra/json-files.ts:45`
  - body: The EPERM/EEXIST fallback writes directly to dest with fs.writeFile, so a crash, ENOSPC, or interruption can leave a previously valid pairing/auth JSON file truncated. Keep the all-or-nothing write guarantee for writeTextAtomic callers instead of overwriting the destination in place.
  - confidence: 0.86
- **[P2] Delay done reactions until Telegram sends a reply:** `extensions/telegram/src/bot-message-dispatch.ts:829-830`
  - body: This marks the status reaction done whenever dispatchError is false, before checking hasFinalResponse. Non-throwing send or fallback failures can therefore leave the user with no final message but a success reaction; only set done after a final or fallback was actually delivered.
  - confidence: 0.78
