## Summary

Keep open: the classifier change is useful, but it still routes the Anthropic long-context 429 into the normal overflow compaction path without proving the retry leaves the active 1M Anthropic context budget/model path, so the same ineligible-credential failure can repeat instead of becoming the promised standard-window compact+retry.

## What This Changes

The PR adds an Anthropic long-context usage error classifier, routes that string through `isLikelyContextOverflowError` before the billing exclusion, re-exports it, and updates focused tests plus troubleshooting docs.

## Best Possible Solution

Make the Anthropic long-context 429 recovery carry an explicit standard-window retry budget or retry mode, add focused tests that prove the next attempt no longer uses the 1M long-context path, then update docs to describe that implemented behavior.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main matches the exact Anthropic body as billing before overflow handling, and the PR’s diff shows only classifier reordering without a standard-window retry proof. I did not run a live Anthropic 429 because the review is read-only and the PR body says that condition is hard to trigger on demand.

## Solution Assessment

No, the classifier-before-billing step is necessary but not enough. The maintainable fix should also make the retry leave the 1M long-context budget/path, or prove with focused runtime tests that existing compaction already does so.
