## Summary

Keep this PR open: the classifier change is focused and directionally useful, but current main still makes exhausted overflow recovery terminal, so the branch can stop configured model fallback for this Anthropic error unless the compact-then-fallback contract is completed.

## What This Changes

The PR reclassifies one Anthropic long-context 429 body as context overflow, exports and tests that classifier, and updates two docs pages to describe compact plus retry.

## Best Possible Solution

Keep the Anthropic-specific classifier, but make exhausted compact/truncate recovery emit a fallback-safe terminal result and prove the ordered raw 429 -> compaction -> fallback path.

## Reproduction Assessment

Yes for the classifier bug at source level: current main matches the exact Anthropic body as billing before context-overflow classification. The full live gateway path for upstream Anthropic 429 recovery has not been reproduced end to end.

## Solution Assessment

No. The classifier move is necessary, but the best complete fix also preserves fallback after bounded compaction/truncation recovery is exhausted and proves that sequence.
