## Summary

Keep open: the diff is a focused attempt at the right classifier seam, but it reclassifies the Anthropic long-context entitlement 429 as overflow without proving the retry actually leaves the long-context tier or avoids unsafe session compaction.

## What This Changes

The PR adds an Anthropic long-context extra-usage classifier, routes it through likely context-overflow handling before billing exclusion, exports it, adds focused tests, and updates docs to describe compact-and-retry recovery.

## Best Possible Solution

Make this a provider-aware recovery: either cap/remove the Anthropic 1M request mode for the retry and prove it with tests, docs, and runtime evidence, or keep this body in eligibility/billing guidance.

## Reproduction Assessment

Yes at source level: current main matches the exact body as billing and excludes billing before overflow handling. No live Anthropic long-context 429 was reproduced against the patched runtime.

## Solution Assessment

No: the classifier reorder alone is not the best fix because current overflow recovery still uses the 1M context budget and leaves context1m request behavior intact. The safer fix needs provider-aware standard-tier retry semantics or should stay in eligibility handling.
