## Summary

Keep open: current main still lets synthetic heartbeat route metadata reach the inbound Conversation info prompt path, and a GitHub-closing fix PR remains open rather than merged.

## What This Changes

The issue reports that cron heartbeat turns can inject partial untrusted Conversation info metadata with an unqualified chat_id, causing false prompt-injection warnings and cross-channel alert noise.

## Best Possible Solution

Review and land one narrow fix PR that suppresses untrusted inbound conversation metadata for synthetic heartbeat, cron-event, and exec-event turns while preserving normal channel inbound metadata.

## Reproduction Assessment

Yes. Source inspection shows a heartbeat context with OriginatingTo reaches buildInboundUserContextPrefix, and that function emits Conversation info when chat_id is present.

## Solution Assessment

Yes. Suppressing the untrusted inbound metadata block for synthetic heartbeat-like providers is the narrowest maintainable fix because the heartbeat prompt already carries the instruction and normal inbound channel metadata should remain unchanged.
