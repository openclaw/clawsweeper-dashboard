## Summary

Keep open: current main still suppresses Telegram ack reactions for unmentioned always-on group messages under the default `group-mentions` scope, and the PR’s code path is narrowly aimed at that gap. The diff looks correct, but it remains merge-blocked by a conflicting branch and missing real Telegram behavior proof.

## What This Changes

This PR makes Telegram treat the default `group-mentions` ack scope as `group-all` only for groups with `requireMention: false`, with a regression test and Telegram docs note.

## Best Possible Solution

Rebase onto current main, preserve the Telegram-local effective-scope fix, and land after redacted real Telegram proof or an explicit maintainer override.

I did not run a live Telegram setup in this read-only review.

## Solution Assessment

Yes. The PR’s Telegram-local effective-scope rewrite is the narrowest maintainable fix because it leaves explicit scopes and non-Telegram channel behavior untouched instead of weakening the shared gate.
