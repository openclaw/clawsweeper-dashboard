## Summary

Keep open. The report is fresh and source-backed: current Discord status reactions do move to `done` and schedule cleanup, but Discord reaction deletion is still best-effort in the cleanup path, so a failed delete can leave a non-terminal lifecycle emoji visible after a completed turn.

## What This Changes

Discord lifecycle status reactions should not leave a permanent non-terminal emoji, especially `🤔`, after a successful completed turn.

## Best Possible Solution

Make Discord lifecycle reaction cleanup reliable: terminal completion should either leave only the terminal `done` reaction when cleanup is disabled or remove all lifecycle reactions after the hold period when cleanup is enabled, with retry-backed deletion and useful emoji/message-level diagnostics.

## Reproduction Assessment

Yes. The issue includes a concrete live Discord reproduction, and current source supports a deterministic unit reproduction by making `removeReaction` fail for a lifecycle emoji during clear; live Discord was not run in this read-only review.

## Solution Assessment

Yes, a narrow fix in the Discord reaction deletion path and shared status-reaction tests is the best direction. Changing defaults or moving this into plugin/product policy would not address the stale terminal UI bug.
