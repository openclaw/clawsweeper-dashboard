## Summary

Keep open. Current main includes the merged deferred-cleanup work from #75582, but terminal completion still only applies done/error and relies on later clear/restore cleanup; it does not remove stale non-terminal lifecycle reactions at the terminal transition or retry/log per-emoji removal failures. The closed unmerged #75570 remains useful source work for the narrow terminal-cleanup direction.

## What This Changes

The issue reports that Discord lifecycle status reactions can leave a non-terminal thinking emoji visible after a successful turn instead of leaving the done reaction or clearing status reactions.

## Best Possible Solution

Keep #75582's no-flicker behavior during active runs, but add a terminal cleanup step that removes non-terminal tracked/known lifecycle reactions after done/error and before the hold-period clear. The fix should also make Discord removal failures diagnosable per emoji/message and preserve the delayed full clear when removeAckAfterReply is enabled.

## Reproduction Assessment

Yes. The issue gives a concrete live Discord reproduction on v2026.4.29, and current source supports a deterministic regression test by walking queued/thinking/tool to done with a remove-capable adapter and asserting stale non-terminal reactions are cleaned at terminal completion and after removeAckAfterReply clear.

## Solution Assessment

No. The current #75582 approach is a good fix for reaction flicker, but it is incomplete for this issue because terminal completion can still depend on later best-effort cleanup; the safer fix is to adapt #75570's terminal cleanup direction to the current active-reaction controller.
