## Summary

Keep open: current main still does not perform terminal cleanup of stale non-terminal lifecycle reactions. The shared controller sets the terminal emoji in `finishWithEmoji`, but stale reactions are only removed later by `clear()` or `restoreInitial()`, and the reported release already had delayed broad `clear()` cleanup yet still produced a permanent thinking reaction.

## What This Changes

Discord status reactions should end a successful turn with the terminal done reaction briefly and then no lifecycle reactions when `removeAckAfterReply` is enabled, never with a stale thinking/status emoji left behind.

## Best Possible Solution

Keep this issue open and land a narrow shared-controller fix that removes non-terminal lifecycle reactions when entering done/error, then still clears all reactions after the configured hold when `removeAckAfterReply` is enabled. The fix should include focused controller coverage and a Discord success-path regression test.

## Reproduction Assessment

Yes. The issue provides a concrete live Discord reproduction on `v2026.4.29`, and a deterministic unit reproduction can walk queued/thinking/tool to done while simulating stale prior reactions that must be removed at terminal completion.

## Solution Assessment

No, current main is not the best complete solution. The narrow maintainable fix is to adapt the terminal cleanup direction from #75570 to the current active-emoji controller and improve failure diagnostics or retry behavior for Discord removals.
