## Summary

Keep open. Current main includes the later active-reaction tracking change from #75582, but terminal completion still does not remove non-terminal lifecycle reactions at the done/error transition, and the Discord success path only schedules a later asynchronous clear. The closed unmerged #75570 remains the closer repair direction for this issue rather than proof that the issue is implemented.

## What This Changes

Discord status reactions should not leave a stale non-terminal thinking or lifecycle emoji after a successful turn, especially when `removeAckAfterReply` should clear lifecycle reactions after the done hold.

## Best Possible Solution

Land a narrow shared-controller or Discord-finalization fix that guarantees terminal completion cannot leave thinking/tool/coding/web/stall reactions as the final state, while preserving #75582's active-reaction tracking and later full clear for `removeAckAfterReply`. Add focused controller coverage plus a Discord successful-turn cleanup regression.

## Reproduction Assessment

Yes. The issue gives a concrete live Discord reproduction on `v2026.4.29`, and current tests provide a deterministic controller harness that can model queued/thinking/tool/done plus failed or delayed removal behavior.

## Solution Assessment

No. Current main improves reaction tracking and reduces progress flicker, but it is not the best complete solution for this issue because terminal completion still relies on a later async cleanup and lacks success-path Discord regression proof.
