## Summary

The PR changes Matrix QA request and media-upload response parsing to use `readResponseWithLimit` with a shared 16 MiB cap, with regression tests for oversized and malformed in-bounds responses.

## Best Possible Solution

Land the shared bounded-reader change if maintainers accept the 16 MiB fail-closed Matrix QA policy and CI remains green.

## Reproduction Assessment

## Solution Assessment

Yes. Reusing `readResponseWithLimit` at both Matrix QA JSON parse sites is the narrowest owner-boundary fix; a new Matrix-specific stream reader would duplicate existing SDK behavior.
