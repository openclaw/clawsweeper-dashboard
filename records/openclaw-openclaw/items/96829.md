Full review comments:

- **[P2] Recognize SSE data lines without a following space:** `src/agents/provider-transport-fetch.ts:119`
  - body: This regex only matches `data:` when the next character is whitespace. SSE allows `data:{...}` as well, and both the OpenAI SDK decoder and OpenClaw's existing `hasReadableSseData` helper accept that form, so a valid JSON-labeled SSE response can still be rewrapped as `data: data:{...}` and fail parsing. Reuse the existing SSE classifier/helper instead of this narrower regex.
  - confidence: 0.92
