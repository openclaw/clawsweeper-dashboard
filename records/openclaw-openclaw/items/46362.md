## Summary

Keep open for maintainer review: current main does not implement voice-triggered model escalation, while this branch is stale against the Discord plugin refactor, mixes in a gateway fix that current main already solved differently, and introduces concrete voice-command UX/privacy bugs.

## What This Changes

The PR branch adds a gateway plugin HTTP registry bridge and hard-coded Discord voice transcript triggers that persist session model overrides to Anthropic Sonnet/Haiku or reset to default.

## Best Possible Solution

If accepted, port only the voice capability into the current Discord plugin voice segment path with explicit command detection, configurable/allowed model targets, command-only utterances skipped before agent dispatch, first-turn session persistence, and focused regression tests; leave the gateway registry fix to the current main implementation.

## Reproduction Assessment

Yes for the PR findings: the PR head source shows the escalation handler is awaited before falling through to agentCommandFromIngress, returns when the session entry is missing, and uses a broad natural-language regex. I did not execute the branch because this review is read-only and pnpm is unavailable.

## Solution Assessment

No. The proposed branch is not the best current solution because it hard-codes Anthropic targets, uses broad transcript matching, targets an old Discord path, and includes an unrelated gateway registry fix already replaced on main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Stop forwarding voice mode commands to the agent:** `src/discord/voice/manager.ts:693-700`
  - body: After a trigger matches, `handleEscalationTrigger` returns `void` and execution continues into `agentCommandFromIngress`, so utterances like `go deep` both change the override and get sent as a normal prompt. Return a boolean or command result and skip the agent turn for command-only triggers.
  - confidence: 0.94
- **[P2] Create the session before persisting voice overrides:** `src/discord/voice/manager.ts:786-788`
  - body: When the spoken trigger is the first voice turn for a route, the session store entry may not exist yet; this early return makes the mode command a no-op until the user repeats it. Ensure the target session entry exists before applying the override.
  - confidence: 0.86
- **[P2] Use explicit medium-tier trigger phrases:** `src/discord/voice/manager.ts:84`
  - body: The `think about that/this/it` pattern matches ordinary conversation and can silently switch future turns to Haiku. Keep voice mode changes behind explicit command phrases such as `medium mode` or `use haiku`.
  - confidence: 0.9
- **[P3] Allow clearing the gateway registry without casts:** `src/plugins/runtime.ts:58-59`
  - body: `getGatewayPluginRegistry` returns nullable state, but `setGatewayPluginRegistry` only accepts `PluginRegistry`, forcing tests to clear it with `null as never`. Widen the setter type or provide a clear helper so teardown stays type-safe.
  - confidence: 0.83
