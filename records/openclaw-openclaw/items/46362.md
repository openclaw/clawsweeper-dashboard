Keep open: the gateway-registry half is already superseded on main, but the Discord voice model-switching feature is still unique and needs a fresh implementation path, proof, and maintainer policy decision before merge.

## What This Changes

The branch adds spoken Discord voice phrases that persist session model overrides and adds a global gateway plugin-registry bridge with tests.

## Best Possible Solution

If accepted, rework the feature in `extensions/discord/src/voice/*` with explicit configurable commands, consume command utterances before routing, handle first-turn sessions, and keep current main's pinned route registry.

## Reproduction Assessment

Yes for the PR defects from source inspection: the diff matches broad conversational phrases and then forwards the same transcript into agent routing. The actual Discord voice behavior is not live-proven because the manual checks remain unchecked.

## Solution Assessment

No. The gateway fix should stay on main's pinned registry implementation, while the voice feature needs a new plugin-path design with explicit command semantics, configuration, first-turn handling, and real Discord proof.

  - body: Current main's Discord voice runtime lives under `extensions/discord/src/voice/*`, but this PR adds the feature to `src/discord/voice/manager.ts`. As written, the model-switching logic targets the obsolete core path instead of the active plugin runtime.

  - body: After `handleEscalationTrigger` runs, execution still sends the same transcript to `agentCommandFromIngress`. A command-only phrase like `go deep` would both change the override and be answered as a normal user prompt.
  - confidence: 0.94
- **[P2] Require explicit medium-mode commands:** `src/discord/voice/manager.ts:84-85`
  - body: The `think about this/that/it` pattern is ordinary speech, so normal prompts can silently persist a Haiku override. Make the medium trigger an explicit mode-switch command instead of a conversational phrase.
  - confidence: 0.9
- **[P2] Persist overrides for first-turn voice sessions:** `src/discord/voice/manager.ts:781-784`
  - body: When the session store entry is missing, the handler returns without applying the override. A first utterance after joining can therefore drop `go deep` before the later agent turn creates the session entry.
  - confidence: 0.82
- **[P3] Allow registry cleanup without unsafe casts:** `src/plugins/runtime.ts:58-59`
  - body: The PR's getter returns `PluginRegistry | null`, but the setter accepts only `PluginRegistry`, forcing tests to clear state with `null as never`. Either widen the setter or drop this superseded bridge in favor of current main's pinned registry surface.
  - confidence: 0.8
