## Summary

Keep open for maintainer handling: the branch still contains a unique Discord voice escalation feature, but it targets a removed legacy path and has correctness, proof, and model-routing policy blockers. The separate gateway registry fix is already superseded by https://github.com/openclaw/openclaw/pull/47902.

## What This Changes

The PR adds hardcoded spoken Discord voice triggers that persist session model overrides and also adds a gateway plugin HTTP-route registry bridge with tests.

## Best Possible Solution

If maintainers want this capability, implement it in extensions/discord/src/voice with explicit operator-configured trigger phrases and allowed model targets, consume command-only utterances, handle first-turn session state, and keep the gateway registry solution from https://github.com/openclaw/openclaw/pull/47902.

## Reproduction Assessment

Yes for the PR defects by source inspection: the diff matches broad phrases, writes a persistent model override, and then continues to normal agent routing while current main no longer uses src/discord/voice/manager.ts. I did not run a live Discord voice session.

## Solution Assessment

No. The current branch is not the best solution because it modifies a removed path, hardcodes provider/model choices, leaves command utterances in the agent pipeline, and duplicates a gateway fix already solved on main.
