## Summary

Keep open, but the PR is not mergeable as-is: the voice feature targets a stale Discord path, forwards command utterances to the agent, uses broad hardcoded cloud-model triggers, and lacks real Discord voice proof. The gateway registry part has already been superseded on current main by the pinned HTTP-route registry implementation.

## What This Changes

The PR adds voice-triggered Discord voice model escalation plus a gateway plugin registry bridge for late HTTP route registration.

## Best Possible Solution

Redesign the feature in extensions/discord/src/voice with explicit configurable command semantics and model targets, consume command utterances before routing, handle first-turn session state, and keep main's pinned gateway registry implementation.

## Reproduction Assessment

Yes for the PR defects from source inspection: the branch matches broad spoken phrases, persists overrides, and then continues routing the same transcript, while current main's active Discord voice path is under extensions/discord/src/voice. No live Discord proof is present for the intended behavior.

## Solution Assessment

No. The gateway fix should remain on current main's pinned-registry design, and the voice feature needs an active-plugin-path implementation with explicit command/config policy before it is safe to merge.
