## Summary

Keep open: the branch is not mergeable as-is because the voice feature targets an obsolete Discord path, has source-level UX and security defects, lacks real Discord proof, and the unrelated gateway portion has already been superseded on current main.

## What This Changes

The branch adds spoken Discord voice model-switch phrases to the legacy `src/discord` voice manager and adds a global gateway plugin-registry bridge with tests.

## Best Possible Solution

If maintainers want this feature, redesign it in `extensions/discord/src/voice/*` with explicit configurable commands and model targets, consume command utterances before routing, handle first-turn sessions, and leave main's pinned gateway route registry in place.

## Reproduction Assessment

Yes for the PR defects from source inspection: the diff matches broad conversational phrases and then forwards the same transcript into agent routing. No live Discord proof is present for the intended voice behavior.

## Solution Assessment

No. The gateway fix should stay on current main's pinned-registry implementation, and the voice feature needs a plugin-path design with explicit command semantics, configuration, first-turn handling, and real Discord proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Port escalation to the active Discord plugin path:** `src/discord/voice/manager.ts:59`
  - body: Current main's Discord voice runtime lives under `extensions/discord/src/voice/*`, but this PR adds the feature to `src/discord/voice/manager.ts`. As written, the model-switching logic targets a stale path instead of the active runtime.
  - confidence: 0.91
- **[P2] Consume model-switch utterances before agent routing:** `src/discord/voice/manager.ts:692-693`
  - body: After `handleEscalationTrigger` runs, execution still sends the same transcript to `agentCommandFromIngress`. A command-only phrase like `go deep` would both change the override and be answered as a normal user prompt.
  - confidence: 0.94
- **[P2] Require explicit medium-mode commands:** `src/discord/voice/manager.ts:84-85`
  - body: The `think about this/that/it` pattern is ordinary speech, so normal prompts can silently persist a Haiku override. Make the medium trigger an explicit mode-switch command instead of a conversational phrase.
  - confidence: 0.9
- **[P2] Persist overrides for first-turn voice sessions:** `src/discord/voice/manager.ts:781-784`
  - body: When the session store entry is missing, the handler returns without applying the override. A first utterance after joining can therefore drop `go deep` before the later agent turn creates the session entry.
  - confidence: 0.82
- **[P3] Allow registry cleanup without unsafe casts:** `src/plugins/runtime.ts:58-59`
  - body: The PR's getter returns `PluginRegistry | null`, but the setter accepts only `PluginRegistry`, forcing tests to clear state with `null as never`. Either widen the setter or drop this superseded bridge in favor of current main's pinned registry surface.
  - confidence: 0.8
