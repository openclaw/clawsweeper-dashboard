- **[P1] Rebase the feature onto the active Discord voice plugin:** `src/discord/voice/manager.ts:21`
  - body: Current main no longer uses `src/discord/voice/manager.ts`; Discord voice lives under `extensions/discord/src/voice`. As a result this branch is conflicting and the feature would miss the current realtime/agent-proxy voice paths until it is moved and redesigned there.
  - confidence: 0.95
- **[P1] Gate spoken model switches behind configured allowed targets:** `src/discord/voice/manager.ts:75-86`
  - body: These hardcoded Anthropic targets write a user session override from ambient voice text. Session overrides are exact and respect allowlists/auth, so this can either fail the next turn or silently move a voice session onto an unconfigured cloud provider; the targets and trigger phrases need explicit config/allowlist semantics.
  - confidence: 0.9
- **[P2] Consume voice commands before routing them as prompts:** `src/discord/voice/manager.ts:692-693`
  - body: `handleEscalationTrigger` returns `void`, so after changing the model the same utterance still falls through to the normal agent turn. A command-only phrase like `go deep` should not also be sent as the user's prompt.
  - confidence: 0.95
- **[P2] Create the session entry before writing the override:** `src/discord/voice/manager.ts:782-784`
  - body: When the voice session has not been persisted yet, this early return drops the escalation while the later agent turn may create the session. A first utterance of `go deep` therefore does nothing and requires the user to repeat it.
  - confidence: 0.86
- **[P2] Make the medium-tier trigger an explicit command:** `src/discord/voice/manager.ts:84-85`
  - body: `think about that/this/it` is common conversational speech and would persistently de-escalate the session to Haiku even when the user did not ask to change modes. Voice model switching needs a much more explicit command phrase.
  - confidence: 0.88
