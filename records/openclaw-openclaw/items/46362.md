## Summary

Keep this PR open, but it is not merge-ready: the gateway registry half is already shipped on main, while the voice escalation half targets the old Discord core path and still has several command-handling defects called out in review.

## What This Changes

The PR adds spoken Discord voice phrases that persist session model overrides, and also carries an older gateway HTTP route registry pinning fix.

## Best Possible Solution

Rework this as a fresh Discord plugin change in `extensions/discord/src/voice/segment.ts`, omit the already-shipped gateway registry fix, consume pure mode commands, and add explicit owner/config policy plus focused voice tests.

## Reproduction Assessment

Yes for the PR-level defects from source inspection: the patch edits a legacy path absent from current main, and its escalation handler returns `void` before the transcript continues to agent dispatch. I did not run tests because the review was read-only.

## Solution Assessment

No. The PR mixes an already-shipped gateway fix with a new voice feature, targets the old Discord path, and hard-codes persistent cloud model tiers without command consumption or a clear permission/config boundary.

## Review Findings

Overall confidence: 0.87

Full review comments:

- **[P1] Move voice changes to the current Discord plugin path:** `src/discord/voice/manager.ts:21`
  - body: Current main no longer has `src/discord/voice/manager.ts`; Discord voice lives under `extensions/discord/src/voice`, with transcript dispatch in `segment.ts`. As written, this branch cannot apply cleanly and would miss the live code path after rebase.
  - confidence: 0.94
- **[P2] Consume escalation-only utterances before agent dispatch:** `src/discord/voice/manager.ts:692-693`
  - body: `handleEscalationTrigger` returns `void`, so after applying an override the same transcript still flows into the normal agent request. A pure command like `go deep` would both switch models and get answered as user content.
  - confidence: 0.9
- **[P2] Narrow the medium-tier trigger phrases:** `src/discord/voice/manager.ts:84-85`
  - body: The `think about that/this/it` regex is common conversational language, so normal speech can silently downgrade the session to Haiku. Please require an explicit mode command instead.
  - confidence: 0.86
- **[P2] Persist overrides for missing session entries:** `src/discord/voice/manager.ts:784-785`
  - body: When the session key is not yet present in `sessions.json`, the handler returns without applying the override. Saying an escalation phrase as the first voice utterance after joining can therefore be ignored.
  - confidence: 0.78
