## Summary

Keep open for maintainer handling: the voice escalation idea is still unique, but this branch is not merge-ready because it targets the deleted legacy Discord path, duplicates a gateway fix already handled elsewhere, lacks real Discord voice proof, and introduces provider/session/security risks.

## What This Changes

The PR adds spoken Discord voice triggers that persist model overrides for cloud/local model switching and also carries an older gateway plugin HTTP-route registry bridge.

## Best Possible Solution

If maintainers want this capability, implement it in `extensions/discord/src/voice` with explicit configured trigger phrases, allowed model targets, command-utterance consumption, first-turn session handling, and no gateway registry diff.

## Reproduction Assessment

Yes for the PR defects by source inspection: the diff hardcodes trigger phrases and model targets, continues to normal agent routing after trigger detection, and writes only to an existing session-store entry. I did not run a live Discord voice session in this read-only review.

## Solution Assessment

No. The current patch mixes a superseded gateway fix with a stale-path voice feature; the maintainable path is an active-plugin implementation with configured phrases, allowed targets, and real Discord proof.
