## Summary

Keep open for maintainer review: the gateway-registry half is already superseded on main, but the central Discord voice feature is still a meaningful new capability and the proposed patch is not merge-ready against the active voice implementation.

## What This Changes

The PR branch adds spoken Discord voice model escalation/reset triggers in a legacy voice manager and adds a gateway HTTP-route registry bridge with tests.

## Best Possible Solution

If maintainers want this capability, implement it in extensions/discord/src/voice with explicit configured command phrases and allowed model targets, consume command-only utterances before agent routing, handle first-turn session state, and leave the gateway registry fix to the merged mainline implementation.

## Reproduction Assessment

Yes for the PR defects from source inspection: the branch modifies a legacy voice path, matches broad phrases, writes persistent overrides, and then continues routing the same transcript. No live Discord voice run proves the intended behavior after the patch.

## Solution Assessment

No. The gateway half is already solved on main, and the voice half should be redesigned on the active Discord plugin path with explicit operator-controlled configuration and command-consumption semantics.
