## Summary

Keep open, but this PR is not merge-ready: it lacks real Discord voice proof, targets a removed legacy Discord voice path, keeps a gateway registry fix already superseded on main, and hardcodes spoken model-routing policy into session state.

## What This Changes

The PR adds spoken Discord voice triggers for persistent session model overrides and also adds an older plugin HTTP-route registry bridge with tests.

## Best Possible Solution

Implement this, if wanted, in `extensions/discord/src/voice` with explicit configured trigger phrases, allowed model targets, command-utterance consumption, first-turn session handling, and no gateway registry diff.

## Reproduction Assessment

Yes for the PR defects by source inspection: the diff hardcodes trigger phrases and model targets, continues to normal agent routing after trigger detection, and writes only to an existing session-store entry. I did not run a live Discord voice session in this read-only review.

## Solution Assessment

No. The current patch mixes a superseded gateway fix with a stale-path voice feature; the maintainable path is an active-plugin implementation with configured phrases, allowed targets, and real Discord proof.
