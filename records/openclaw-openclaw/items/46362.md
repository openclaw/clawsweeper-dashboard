## Summary

Keep open: the central voice-escalation feature is not implemented on current main, but this branch is not mergeable as-is because it targets the removed legacy Discord path, forwards command utterances to the agent, has broad spoken triggers, lacks real Discord proof, and carries a cloud-model switching security/product decision. The gateway registry portion has already been superseded on main by the pinned HTTP-route registry design.

## What This Changes

This PR adds spoken Discord voice model-switch phrases in the legacy voice manager path and a global plugin HTTP-route registry bridge with tests.

## Best Possible Solution

Redesign the feature in extensions/discord/src/voice with explicit configurable command semantics and model targets, consume command utterances before routing, handle first-turn session state, and keep main's pinned gateway registry implementation.

## Reproduction Assessment

Yes for the PR defects from source inspection: the branch matches broad spoken phrases, persists model overrides, and then continues routing the same transcript. No live Discord proof is present for the intended feature behavior.

## Solution Assessment

No. The gateway part should stay on current main's pinned-registry implementation, and the voice feature needs an active-plugin-path design with explicit command/config policy before implementation.
