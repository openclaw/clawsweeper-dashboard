## Summary

Keep open: the feature idea is still unique, but this branch is not merge-ready because the voice change targets a removed Discord path, needs product/security direction for spoken cloud-model switching, and lacks real Discord voice proof; the gateway-registry half is already superseded by https://github.com/openclaw/openclaw/pull/47902.

## What This Changes

The branch adds hardcoded Discord voice utterance triggers that write session model overrides and includes a gateway plugin HTTP route registry bridge plus tests.

## Best Possible Solution

If maintainers want this capability, implement it in `extensions/discord/src/voice` with explicit configured trigger phrases and allowed model targets, consume command-only utterances, handle first-turn session state, and drop the gateway registry diff already covered by https://github.com/openclaw/openclaw/pull/47902.

## Reproduction Assessment

Yes for the PR defects by source inspection: the diff adds broad hardcoded trigger phrases, persists a model override, and continues to normal routing while current main no longer has the modified `src/discord` path. I did not run a live Discord voice session.

## Solution Assessment

No. The branch is not the best solution because the useful gateway fix is already merged separately and the voice behavior should be implemented in the active Discord plugin with explicit configuration and allowlisted targets.
