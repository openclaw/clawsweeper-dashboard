## Summary

Keep open for maintainer judgment: the gateway-registry half is already superseded by merged mainline work, but the central Discord voice feature is still meaningful and the branch implements it on a removed legacy path with product/security blockers and no real Discord proof.

## What This Changes

The PR adds spoken Discord voice model switching in a legacy voice manager and adds a gateway plugin HTTP-route registry bridge with tests.

## Best Possible Solution

If maintainers want this capability, implement it in extensions/discord/src/voice with explicit configured command phrases and allowed model targets, consume command-only utterances before agent routing, handle first-turn session state, and leave the gateway registry fix to the merged mainline implementation.

## Reproduction Assessment

Yes for the PR defects from source inspection: the branch modifies a removed legacy voice path, matches broad phrases, writes persistent overrides, and then continues routing the same transcript. No live Discord voice run proves the intended behavior after the patch.

## Solution Assessment

No. The gateway half is already solved on main, and the voice half should be redesigned on the active Discord plugin path with explicit operator-controlled config and command-consumption semantics.
