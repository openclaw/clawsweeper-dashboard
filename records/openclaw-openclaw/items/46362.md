## Summary

Keep this PR open for maintainer review, but it is not merge-ready. Current main has moved Discord voice into the bundled Discord plugin and already supersedes the unrelated gateway-registry fix, while the remaining spoken model-switching change has product, UX, and security-policy issues.

## What This Changes

The PR adds spoken Discord voice model switching and an unrelated gateway plugin HTTP-route registry bridge with regression tests.

## Best Possible Solution

If maintainers want this capability, rebuild it as a focused Discord plugin change under `extensions/discord/src/voice` with explicit commands, command consumption, session-entry creation, configured or allowlisted model targets, focused tests, and docs.

## Reproduction Assessment

Yes for the PR review issues: the patch shows the broad trigger regex, missing early return after a match, and missing-session no-op. Current main cannot reproduce the intended spoken-trigger feature because that behavior is absent from the current Discord voice plugin.

## Solution Assessment

No. This PR is not the best way to solve the request because it edits a retired path, hardcodes cloud targets, forwards matched commands as prompts, and bundles unrelated gateway work already superseded on main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Require explicit voice mode commands:** `src/discord/voice/manager.ts:84-85`
  - body: The medium-tier regex matches ordinary speech such as `think about this`, so normal conversation can silently persist a model downgrade. Use explicit command phrases with low false-positive risk before changing model state from voice transcripts.
  - confidence: 0.91
- **[P2] Consume matched voice commands before dispatch:** `src/discord/voice/manager.ts:692-693`
  - body: After applying an escalation trigger, execution continues into `agentCommandFromIngress`, so a phrase like `go deep` both changes the model and is sent to the agent as a normal prompt. Return a boolean from the trigger handler and skip dispatch when it matches.
  - confidence: 0.9
- **[P2] Persist overrides for missing session entries:** `src/discord/voice/manager.ts:783-785`
  - body: The session-store update returns when the session entry is absent, so the first spoken escalation in a fresh voice session can be dropped before the later agent command path creates the entry. Create or update the target entry when applying the override.
  - confidence: 0.84
- **[P3] Allow clearing the gateway registry without casts:** `src/plugins/runtime.ts:58-59`
  - body: The new getter returns `PluginRegistry | null`, but the setter only accepts `PluginRegistry`, forcing test cleanup to use `null as never`. Widen the setter parameter if this registry API remains in the branch.
  - confidence: 0.86
