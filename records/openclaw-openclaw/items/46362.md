## Summary

Keep this PR open for maintainer judgment rather than cleanup-close: the gateway registry portion is already superseded by a merged mainline fix, but the Discord voice escalation feature is still unique and the proposed patch is not mergeable as-is because it targets a deleted legacy voice path, lacks real Discord proof, and introduces hardcoded spoken cloud-model switching.

## What This Changes

The PR adds spoken Discord voice model-switch triggers in the legacy voice manager and a gateway plugin HTTP-route registry bridge with tests.

## Best Possible Solution

If maintainers want this capability, redesign it in extensions/discord/src/voice with explicit configurable command semantics and allowed model targets, consume command utterances before agent routing, handle first-turn session state, and keep the current pinned gateway registry implementation.

## Reproduction Assessment

Yes for the PR defects from source inspection: the branch matches broad phrases, writes persistent overrides, and then continues routing the same transcript. The intended user behavior still lacks live Discord proof.

## Solution Assessment

No. The gateway half is already solved on main, and the voice half needs an active-plugin-path implementation with explicit operator-controlled config/trust semantics before it is the maintainable fix.
