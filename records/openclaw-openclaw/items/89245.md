## What This Changes

Issue reports Feishu direct-session recovery failing after user `/reset` or `/new`, leaving session-family IDs without transcript files and later generic external-run fallback replies.

## Best Possible Solution

Land one generic session-lifecycle repair that makes `/new`, `/reset`, and automatic rotations durable or self-repairing before session-family metadata can point at unusable transcript IDs.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection supports the failure shape: reset/rotation persists new IDs and paths before durable transcript creation is guaranteed, and the runner tolerates missing transcript files until later persistence.

## Solution Assessment

Yes, a generic core session-lifecycle repair is the best direction; a Feishu-only patch would be too narrow. The fix should either create a minimal durable transcript before switching metadata or avoid retaining new family IDs until durability is proven.
