## Summary

Keep open. Current main has the adjacent compaction-failure preserved-session fix and a Feishu doctor cleanup path, but it still allows user-initiated `/new` or `/reset` metadata to advance before a durable transcript artifact is guaranteed.

## What This Changes

The issue reports that a Feishu direct session can rotate session-family metadata after `/reset` or `/new` without durable transcript artifacts, causing later messages to hit the generic external-run failure reply.

## Best Possible Solution

Add a generic session lifecycle invariant so user-initiated reset/new rotations either create a durable minimal transcript before the family index advances or avoid retaining a new family member until the transcript exists.

## Reproduction Assessment

No high-confidence live current-main replay was run. Source inspection supports the reported missing-transcript shape because reset/new metadata is persisted before transcript creation is guaranteed.

## Solution Assessment

Yes, the best direction is a core session lifecycle invariant plus Feishu doctor cleanup as fallback. A Feishu-only runtime patch would miss the generic reset/new persistence boundary.
