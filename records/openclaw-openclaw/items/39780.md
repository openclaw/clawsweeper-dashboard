## Summary

Keep open: current main and v2026.5.12 still render the named Control UI recommendations as text guidance, while the Config editor and gateway already have separate diff/apply primitives. The remaining work is a security-sensitive feature and UX policy decision, and the related open issues cover adjacent CLI migration/default-security questions rather than this UI action flow.

## What This Changes

The issue requests Control UI recommendation cards that can show a before/after config diff and apply accepted config changes instead of telling users to edit JSON manually.

## Best Possible Solution

Define an allowlisted Control UI recommendation-action contract that reuses config.patch and existing diff/redaction behavior, with explicit confirmation and policy gates for security-sensitive toggles.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high-confidence evidence that current main and v2026.5.12 still show guidance only while config diff/apply primitives are scoped to the Config editor.

## Solution Assessment

Unclear until maintainers settle policy. Reusing config.patch is plausible, but the maintainable solution should be an allowlisted recommendation-action system rather than ad hoc Apply buttons for each warning, especially around allowInsecureAuth.
