## Summary

Keep open: current main has `config.patch` plus Config-tab diff/apply primitives, but the specific Overview/login recommendation surfaces still render text-and-docs guidance, and one-click application for `gateway.controlUi.allowInsecureAuth` needs UX/security policy before implementation.

## What This Changes

Request that Control UI config recommendations show before/after diffs with accept/dismiss controls and apply approved changes through existing config APIs instead of manual JSON edits.

## Best Possible Solution

Define a Control UI recommendation-action contract that reuses `config.patch` and existing diff/redaction behavior for approved safe changes, while keeping security-sensitive toggles behind explicit maintainer-approved confirmation or manual guidance.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high-confidence evidence that the named hints are currently text-only while the Config editor already has separate diff/apply primitives.

## Solution Assessment

Unclear until maintainers settle policy. Reusing `config.patch` and existing diff UI is plausible, but adding one-click recommendations for auth/security flags should be designed as an allowlisted recommendation system rather than ad hoc buttons in each hint.
