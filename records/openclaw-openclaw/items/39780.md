## Summary

Keep open: current main and v2026.5.12 still render the named Control UI remediation as guidance text, while diff/apply behavior remains scoped to the Config editor and config write RPCs. The requested one-click recommendation workflow is useful but security-sensitive and needs maintainer product/security policy before implementation.

## What This Changes

The issue asks Control UI recommendation sections to offer one-click config changes with before/after diffs, accept/dismiss controls, and confirmation instead of manual edit instructions.

## Best Possible Solution

Define an allowlisted Control UI recommendation-action contract that reuses `config.patch` and existing redacted diff behavior, with explicit confirmation, dismiss state, and policy gates for security-sensitive toggles.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence that current main lacks an Apply/Diff/Dismiss recommendation path while the Config editor has separate scoped diff/apply primitives.

## Solution Assessment

Unclear until maintainers settle policy. Reusing `config.patch` is plausible, but a reusable allowlisted recommendation-action system is safer than ad hoc Apply buttons for each warning, especially around `allowInsecureAuth`.
