## Summary

Keep open. Current main has config diff/apply primitives, but the requested actionable recommendation flow is still missing: insecure-context guidance renders manual config text and docs links, attention items only support docs links, and the diff UI is confined to manual config editing.

## What This Changes

The issue requests Control UI recommendation cards that can apply config changes with a before/after diff, explicit accept/dismiss controls, and confirmation.

## Best Possible Solution

Design a generic actionable config recommendation surface that can preview a narrowly scoped patch, use existing config write APIs with base-hash protection, require explicit acceptance, support dismissal, and coordinate upgrade-migration diff behavior with #38249.

## Reproduction Assessment

Not applicable as a bug. Source inspection against current main confirms the existing behavior: recommendation guidance still renders manual text/docs links, while diff/apply support exists only in the Config editing flow.

## Solution Assessment

No current implementation solves this issue yet. The existing config diff/apply primitives look like the right lower-level seam, but the best fix needs a designed recommendation/action layer rather than adding ad hoc buttons to one warning.
