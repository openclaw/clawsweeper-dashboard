## Summary

Keep open: current main and v2026.5.20 still show guidance-only Control UI remediation for the insecure-auth path, while before/after diff and apply behavior is scoped to the Config editor. Turning auth/security recommendations into one-click config writes still needs maintainer UX and security policy.

## What This Changes

The issue requests Control UI recommendation sections that can show a before/after diff, accept or dismiss the suggestion, and apply safe config changes automatically instead of telling users to edit config files manually.

## Best Possible Solution

Define an allowlisted Control UI recommendation-action contract that reuses existing config write and redacted diff behavior, with explicit confirmation, dismiss state, and policy gates for security-sensitive toggles.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence current main lacks an Apply/Diff/Dismiss recommendation path while the Config editor has separate scoped diff/apply primitives.

## Solution Assessment

Unclear until maintainers settle policy. Reusing existing config write and redacted diff behavior is plausible, but a reusable allowlisted recommendation-action system is safer than ad hoc Apply buttons for each warning, especially around allowInsecureAuth.
