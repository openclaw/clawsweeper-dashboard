## Summary

Keep open: current main has config staging, before/after diff, and save/apply plumbing, but the relevant Control UI recommendation still renders manual guidance and docs links instead of an apply/diff/dismiss flow.

## What This Changes

Control UI recommendation cards should apply safe config changes through the existing config API, with a before/after diff and user acceptance, instead of telling users to edit JSON manually.

## Best Possible Solution

Add a reusable Control UI recommendation action pattern that stages safe config patches, shows the same redacted diff machinery before commit, and then applies or dismisses the suggestion with explicit user consent.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/UX request. Source inspection confirms the current Overview warning still renders manual guidance and docs links rather than an apply/diff flow.

## Solution Assessment

Unclear until maintainers define the recommendation safety policy. The maintainable direction is to reuse existing staged config/diff/apply plumbing rather than special-case direct writes from warning text.
