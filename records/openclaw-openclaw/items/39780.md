## Summary

Keep open: current main still renders the named Control UI recommendations as text/docs guidance, while diff/apply primitives exist only in the Config editor and gateway config RPCs. The requested one-click recommendation flow is useful, but it needs maintainer UX/security policy because one path would persist `gateway.controlUi.allowInsecureAuth`.

## What This Changes

The issue requests Apply/Diff/Dismiss controls for Control UI config hardening recommendations instead of telling users to edit config files manually.

## Best Possible Solution

Define an allowlisted Control UI recommendation-action contract that reuses `config.patch` plus existing diff/redaction behavior, with explicit confirmation, dismiss state, and policy gates for security-sensitive toggles.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence that current main still lacks an Apply/Diff/Dismiss recommendation path on the Overview/login hints while the Config editor has separate scoped diff/apply primitives.

## Solution Assessment

Unclear until maintainers settle policy. Reusing `config.patch` is plausible, but a reusable allowlisted recommendation-action system is safer than ad hoc Apply buttons for each warning, especially around `allowInsecureAuth`.
