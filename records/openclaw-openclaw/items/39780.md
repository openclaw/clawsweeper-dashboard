## Summary

Keep open: current main still renders the named Control UI recommendations as text guidance, while diff/apply primitives remain scoped to the Config editor and gateway config RPCs. The request is useful, but it needs maintainer UX/security policy before one-click recommendation actions can safely persist settings such as `gateway.controlUi.allowInsecureAuth`.

## What This Changes

Add Control UI recommendation actions that show a before/after diff and let users accept, dismiss, and apply suggested config changes automatically.

## Best Possible Solution

Define an allowlisted Control UI recommendation-action contract that reuses `config.patch` and existing diff/redaction behavior, with explicit confirmation, dismiss state, and policy gates for security-sensitive toggles.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence that current main still lacks an Apply/Diff/Dismiss recommendation path on the Overview/login hints while the Config editor has separate scoped diff/apply primitives.

## Solution Assessment

Unclear until maintainers settle policy. Reusing `config.patch` is plausible, but a reusable allowlisted recommendation-action system is safer than ad hoc Apply buttons for each warning, especially around `allowInsecureAuth`.
