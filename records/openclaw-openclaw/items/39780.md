## Summary

Keep open: current main and the latest release still render the named Control UI recommendations as text/docs guidance, while the Config editor and gateway already have separate diff/apply primitives. The remaining work is a security-sensitive feature and UX policy decision, especially for one-click changes to `gateway.controlUi.allowInsecureAuth`.

## What This Changes

Make Control UI config hardening recommendations actionable with before/after diff, accept/dismiss controls, and automatic config patching instead of manual JSON instructions.

## Best Possible Solution

Define an allowlisted Control UI recommendation-action contract that reuses `config.patch` and existing diff/redaction behavior for safe changes, with explicit confirmation and policy gates for security-sensitive toggles.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high-confidence evidence that the named hints remain text-only while the Config editor already has separate diff/apply primitives.

## Solution Assessment

Unclear until maintainers settle policy. Reusing `config.patch` is plausible, but the maintainable solution should be an allowlisted recommendation system rather than ad hoc Apply buttons for each warning, especially around `allowInsecureAuth`.
