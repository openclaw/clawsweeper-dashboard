## Best Possible Solution

If maintainers approve explicit operator-home exposure, land it with docs, changelog, focused regression coverage, and real Codex harness proof; otherwise keep `HOME` isolated and document the supported runbook workaround.

## Solution Assessment

No as-is. The v2 code now populates the default `REAL_HOME` value, but exposing that variable should be treated as a maintainer-approved isolation contract and paired with docs, tests, changelog, and real harness proof before merge.
