## Summary

Keep open: the proposed change misses the default stdio path because it derives `REAL_HOME` only from explicit start-option overrides, and the external PR still lacks after-fix real behavior proof.

## What This Changes

This PR adds a `REAL_HOME` environment variable to the Codex app-server start environment before the harness rewrites `CODEX_HOME` and `HOME`.

## Best Possible Solution

If maintainers approve explicit operator-home exposure, derive it from an existing `REAL_HOME` override or the inherited process `HOME` before remapping `HOME`, then add regression coverage, docs, and a real harness proof artifact.

## Reproduction Assessment

Yes. Current main source and tests show stdio Codex app-server starts remap `HOME` to the per-agent native home, and the PR source shows the proposed `REAL_HOME` path remains empty when callers do not explicitly set `startOptions.env.HOME`.

## Solution Assessment

No. The direction may be acceptable if maintainers approve it, but this implementation misses the inherited process `HOME` path that the reported default workflow depends on.
