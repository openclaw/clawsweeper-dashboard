## Summary

Keep open. The linked bug is still real, but this PR should not merge as-is because it repurposes the shared `deliver` flag and would send exec approval followup prompts directly to external chats instead of delivering an agent-synthesized reply.

## What This Changes

The PR adds a direct payload bypass in `agentCommandInternal` for `--deliver` requests without `internalEvents`, updates CLI help text, and adds CLI/gateway tests for direct delivery.

## Best Possible Solution

Introduce an explicit direct-delivery mode for the CLI notification path while preserving generic `deliver` as delivery of the agent-produced reply for gateway and internal workflows.

## Reproduction Assessment

Yes. Source inspection is enough: current main has exec followups call the gateway with `deliver: true` and no `internalEvents`, and the PR early return would deliver `buildExecApprovalFollowupPrompt(...)` text before any agent turn runs.

## Solution Assessment

No. The PR fixes the direct notification case, but the maintainable fix is to gate the bypass behind a separate direct-delivery signal from the CLI/gateway path instead of broadening the shared `deliver` contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve agent synthesis for followups:** `src/agents/agent-command.ts:382`

Gate the bypass behind a separate direct-delivery flag from the CLI path; keep generic `deliver` on the agent-reply path.
