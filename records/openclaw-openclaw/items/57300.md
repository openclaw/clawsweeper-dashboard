## Summary

Keep open. The PR targets a real delivery pain point, but the current patch overloads the shared `deliver` flag and would send exec-approval followup prompts directly to external channels instead of letting the agent synthesize a safe reply.

## What This Changes

The PR adds a direct `--deliver` bypass in `agentCommandInternal`, updates CLI help text, and adds CLI/gateway tests for direct delivery behavior.

## Best Possible Solution

Keep `deliver` as agent-reply delivery and add an explicit direct-delivery contract for raw notification text, with regression coverage for CLI direct delivery, exec followups, and internal-event announcements.

## Reproduction Assessment

Yes for the PR-introduced regression. Source inspection shows exec approval followups call the agent with `deliver: true`, no `internalEvents`, and an instruction prompt, which the PR's early return would deliver verbatim.

## Solution Assessment

No. The direct notification case should be separated from the shared `deliver` flag so existing agent-synthesized delivery workflows keep their current behavior.
