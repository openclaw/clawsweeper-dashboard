## Summary

Keep open. Current main still has the raw-vs-canonical split path: explicit inbound session keys can remain raw, while Gateway/session resolution canonicalizes the same bare structured key to an agent-scoped key and can prune the raw row as a legacy alias.

## What This Changes

The issue asks OpenClaw to preserve continuity for stable external raw session keys after internal sessions_send or nested agent activity touches the same logical conversation.

## Best Possible Solution

Add a shared raw-key alias contract so existing external raw rows either remain durable aliases or resolve consistently through both inbound and Gateway paths before any pruning occurs.

## Reproduction Assessment

Yes, at source level: seed a store row under a structured raw key such as conversation:pair:user_a::user_b:space:space_123, resolve or write through a Gateway path with that key, then send another inbound turn using the same explicit SessionKey. Current main can canonicalize and prune the raw row before the inbound lookup reuses it.

## Solution Assessment

Yes. The narrowest maintainable fix is a generic session-store alias/lookup rule for raw external keys; UUID-only or plugin-specific fixes leave the reported structured-key path open.
