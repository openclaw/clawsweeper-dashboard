## Summary

This should stay open: current `main` still has the Discord channel-management action implementation and discovery, but the public `openclaw message channel` CLI/docs surface exposes only `info` and `list`, so the reported executable-surface mismatch remains actionable.

## What This Changes

The issue asks OpenClaw to align Discord multi-account readiness/capability reporting with an executable channel-admin surface, or explicitly report that admin actions are unavailable.

## Best Possible Solution

Add CLI/docs parity for the already implemented Discord channel-management actions, or deliberately mark those actions as capabilities-only wherever readiness and help text are shown.

## Reproduction Assessment

Yes. Source/docs inspection on current `main` reproduces the mismatch: plugin discovery and handlers include Discord channel-management actions, while the CLI registration and CLI docs expose only `channel info` and `channel list`.

## Solution Assessment

Yes. The narrow maintainable solution is to wire the existing Discord action handlers into `openclaw message channel` subcommands with focused CLI/docs/tests, without changing Discord permission checks or action-gate defaults.
