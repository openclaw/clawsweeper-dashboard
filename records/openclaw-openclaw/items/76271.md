## Summary

Keep this open: current main has Discord channel-admin runtime support, but the public CLI and readiness surfaces still do not line up with that capability, so the reporter's confusion is valid.

## What This Changes

The issue requests that Discord multi-account readiness and executable channel-admin surfaces agree, either by exposing channel creation/admin actions or reporting them unavailable.

## Best Possible Solution

Expose the already-implemented Discord channel-admin actions consistently through CLI/help/docs and make multi-account readiness reflect the executable admin capability or its absence.

## Reproduction Assessment

Yes. Static current-main inspection reproduces the CLI gap: only `channel info` and `channel list` are registered, while the runtime and message-tool discovery already support channel-admin actions behind a gate.

## Solution Assessment

No. The current split is not the best state because it makes the agent/tool runtime, docs, skill readiness, and CLI tell different stories about Discord channel administration.
