## Summary

This should stay open: current main still exposes Discord channel-management actions through plugin discovery/runtime, while the public `openclaw message channel` CLI and CLI docs only expose `info` and `list`.

## What This Changes

The issue asks OpenClaw to either expose Discord channel creation/admin actions when Discord is available or report that those admin actions are not executable.

## Best Possible Solution

Add CLI and docs parity for the existing Discord channel/category admin actions, or explicitly document that those actions are capability/tool-only if maintainers do not want a public CLI surface.

## Reproduction Assessment

Yes. Source/docs inspection on current main reproduces the mismatch: the Discord plugin advertises and handles channel-management actions, while `openclaw message channel` and the CLI docs only expose `info` and `list`.

## Solution Assessment

Yes. The narrow maintainable solution is to register and document the existing Discord channel/category admin actions without changing runtime gates or Discord permission defaults.
