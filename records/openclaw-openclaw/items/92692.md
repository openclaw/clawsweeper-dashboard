## Best Possible Solution

Keep the scanner-layer fix, but make the parent warning suppression apply only to fallback-only parent records while preserving diagnostics for implicit default accounts and account scopes.

## Solution Assessment

No. The scanner is the right layer, but the suppression condition should distinguish fallback-only parent records from top-level records that still act as implicit default accounts.
