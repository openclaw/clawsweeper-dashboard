## Summary

Keep open: the linked doctor false-positive is real, but this branch suppresses parent group warnings whenever any active account exists, which is too broad for Telegram configs that keep top-level credentials as an implicit default account.

## What This Changes

The PR changes the doctor empty-allowlist scanner to suppress top-level group allowlist diagnostics when channel account records exist.

## Best Possible Solution

Keep the scanner-layer fix, but make parent-scope group warning suppression apply only when the parent record is truly fallback-only, not when it represents an implicit default account.

## Reproduction Assessment

Yes. Source inspection shows current main checks the parent scope, and PR-head source shows that parent group warnings are suppressed whenever any active account entry exists.

## Solution Assessment

No. The scanner is the right layer, but the suppression condition should distinguish fallback-only parent records from top-level records that still act as implicit default accounts.
