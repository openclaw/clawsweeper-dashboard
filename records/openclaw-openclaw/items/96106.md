## Summary

Keep open. The latest head fixes the earlier cross-channel widening, but Discord still opts into non-stream reasoning unconditionally and only applies `progress.thinking` at the draft renderer, leaving visible status/summary side effects when the new default-off key is unset or false; the branch is also still dirty against current main.

## What This Changes

The PR routes Anthropic reasoning and pre-tool commentary into Discord progress/durable lanes, adds a Discord-only `streaming.progress.thinking` option, updates docs/generated config metadata/tests, and adjusts plugin-SDK surface budgets.

## Best Possible Solution

Keep the Anthropic commentary tagging and Discord lane reuse, but enforce `channels.discord.streaming.progress.thinking` before any non-stream reasoning callback side effects, add false/unset coverage, then rebase/regenerate baselines before merge.

## Reproduction Assessment

Yes. Source inspection of the latest head shows the Discord reasoning callback is invoked in non-stream modes before the `progress.thinking` renderer gate, so the default-off setting can still leave visible status/summary side effects.

## Solution Assessment

No. Reusing the existing Anthropic/commentary and Discord progress lanes is the right direction, but the config gate must own the subscriber opt-in or callback side effects rather than only the final compositor render call.
