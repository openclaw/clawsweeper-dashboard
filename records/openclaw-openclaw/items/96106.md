- **[P1] Preserve the reasoning-off visibility gate:** `src/agents/embedded-agent-subscribe.ts:188-190`
  - body: `reasoningMode` now streams whenever it is not `"on"`, so the default/off state starts forwarding thinking to any channel that has an `onReasoningStream` callback. Discord and Slack both install that callback for progress/status previews, while the docs define `/reasoning off` as hidden and reserve preview rendering for `/reasoning stream`; this can expose internal reasoning in chats that intentionally kept reasoning disabled.
  - confidence: 0.9

Either keep the existing opt-in default for compatibility or make the broader default change explicit with maintainer approval and matching docs/tests.
