Full review comments:

- **[P1] Preserve the reasoning-off visibility gate:** `src/agents/embedded-agent-subscribe.ts:187-190`
  - body: This changes `streamReasoning` from the current `reasoningMode === "stream"` gate to every mode except `on`. Discord always supplies an `onReasoningStream` hook, so sessions with `/reasoning off` can now render hidden model reasoning in the channel progress draft, contrary to the current runtime contract that reasoning is hidden unless on/stream. Restore the existing gate or make this an explicitly approved opt-in policy.
  - confidence: 0.93
- **[P2] Keep commentary progress opt-in:** `src/channels/streaming.ts:787-790`
  - body: Changing the default to `true` makes `streaming.progress.commentary` visible for existing progress-mode users who never opted in. Current Discord and Telegram docs say this default is `false`, so this silently changes channel behavior and can expose pre-tool narration in temporary drafts. Keep the default false unless maintainers approve the new policy and the docs/schema/tests are updated together.
  - confidence: 0.9
- **[P2] Declare or remove the progress thinking key:** `src/channels/streaming.ts:799-808`
  - body: This adds a `streaming.progress.thinking` reader, but the inspected config schema/UI/docs still only expose the existing progress keys. That leaves users without a valid documented way to disable the new reasoning draft lane and drifts from the config contract. Either wire the key through schema, types, hints, docs, and tests, or remove the hidden option and rely on the existing reasoning modes.
  - confidence: 0.84
