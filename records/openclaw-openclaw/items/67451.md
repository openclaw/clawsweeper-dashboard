The PR changes WhatsApp target normalization and tests so `group:` and `whatsapp:group:` prefixes are stripped before validating group JIDs, but it does so under `src/whatsapp/*`.

## Solution Assessment

No. The branch applies the right small idea to the wrong file path; the maintainable solution is the same behavior change on the active bundled WhatsApp plugin files under `extensions/whatsapp/src/*`.
