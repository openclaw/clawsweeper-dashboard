## Summary

Keep open: current main still rejects the requested group-prefixed WhatsApp target aliases, but this branch cannot land as-is because it edits obsolete `src/whatsapp/*` files, broadens parsing too far, and still lacks real behavior proof.

## What This Changes

The PR changes legacy WhatsApp target normalization to strip `group:` as well as `whatsapp:` prefixes and updates legacy tests to accept prefixed group JIDs.

## Best Possible Solution

If maintainers want to restore this compatibility alias, refresh it in `extensions/whatsapp/src/normalize-target.ts` with focused coverage in `extensions/whatsapp/src/resolve-target.test.ts` and accept only valid group JIDs.

## Reproduction Assessment

Yes, source-level for the requested behavior: current main leaves `group:` in the candidate string, tests assert those aliases are invalid, and outbound resolution fails when normalization returns null. This is not a high-confidence documented bug because current docs describe native group JIDs rather than the prefixed alias.

## Solution Assessment

No. The branch edits obsolete files and strips `group:` before target classification; the safer solution is an active-plugin change only if maintainers decide to support this alias.
