## Summary

Keep open: the hot-path optimization is plausible, but the PR currently lets unresolved child-session data become explicit undefined event fields, which the Control UI treats as a deletion. It also has focused test output only, not real behavior proof from a live gateway or Control UI run.

## What This Changes

The branch adds a lightweight gateway session-row option and uses it for session lifecycle/message event snapshots, with focused tests around skipped child-session resolution.

## Best Possible Solution

Keep the lightweight hot-path optimization only after event payloads omit unresolved childSessions instead of sending an own undefined field, with UI patch regression coverage and redacted live/runtime proof.

## Reproduction Assessment

Yes for the code-path risk: current Control UI code deletes event row fields whose own value is undefined, and this PR feeds undefined childSessions through lightweight event rows. The original profiler performance symptom is source/profiler-reported rather than live-replayed here.

## Solution Assessment

No as written: the lightweight-row direction addresses the hot path, but the event projection must preserve patch semantics for omitted fields before merge.

Full review comments:

- **[P1] Preserve child session links in lightweight events:** `src/gateway/server-chat.ts:319-323`
  - body: Passing `includeChildSessions: false` makes the row's `childSessions` value `undefined`, but the event snapshot still emits `childSessions` as an own property. Omit the field when it was not resolved, or keep resolving/carrying the previous value.
