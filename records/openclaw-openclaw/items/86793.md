## Summary

Keep open: the patch has a blocking session-state regression because lightweight snapshots make `childSessions` an undefined row-patch field that the Control UI deletes, and the external real-behavior proof is still test-only.

## What This Changes

The branch adds a lightweight gateway session-row option and uses it in session lifecycle/message event snapshots, with focused tests for skipped child-session resolution.

## Best Possible Solution

Keep the lightweight event-row performance direction, but preserve child-session lineage in event patches or change patch semantics so omitted child data does not clear existing rows, then add focused regression coverage and real behavior proof.

## Reproduction Assessment

Yes for the patch defect: source inspection shows `includeChildSessions: false` produces an owned `childSessions: undefined` field, and the Control UI deletes undefined event patch fields. The original performance issue still has profiler-excerpt evidence rather than a live replay in this PR.

## Solution Assessment

No. The performance target is reasonable, but dropping child-session data from row patches is not the safest solution; keep lineage metadata or adjust patch semantics while skipping transcript-heavy work.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve child session links in lightweight events:** `src/gateway/session-utils.ts:1807-1808`
  - body: Setting `includeChildSessions: false` makes `buildGatewaySessionRow` return an owned `childSessions: undefined` field. These rows are sent as `sessions.changed`/`session.message` patches, and the Control UI deletes patch fields whose value is `undefined`, so a parent row with active children can lose its live child-session tree after any lightweight event. Please keep this metadata in event rows or change patch handling so omitted child data does not clear existing lineage.
