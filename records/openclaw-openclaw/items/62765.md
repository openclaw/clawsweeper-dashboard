## Summary

Keep open: current main still records unpaired MS Teams personal DM pairing requests and returns before sending the standard in-band pairing challenge, while the linked repair PR is closed unmerged.

## What This Changes

The issue asks for MS Teams `dmPolicy=pairing` to surface pending pair codes instead of recording and silently dropping unpaired personal DMs.

## Best Possible Solution

MS Teams should call the existing `pairing.issueChallenge` helper for unpaired personal DMs, preserving conversation-reference capture and drop semantics while sending and logging the one-time pair-code challenge.

## Reproduction Assessment

Yes. Source inspection on current main shows a personal MS Teams message from an unapproved sender with `channels.msteams.dmPolicy: "pairing"` reaches the direct upsert branch and returns before any Teams reply is sent.

## Solution Assessment

Yes. Reusing `pairing.issueChallenge` is the narrowest maintainable repair because the existing helper centralizes request creation, one-time replies, code-bearing creation callbacks, and reply-error handling used by other channels.
