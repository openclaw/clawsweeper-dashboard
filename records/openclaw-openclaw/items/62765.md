## Summary

Keep open: current main still records and drops unpaired MS Teams personal DMs without issuing the in-band pairing challenge, and the linked fix PR is closed unmerged. The affected branch and regression-test surface are narrow enough for a focused repair.

## What This Changes

The issue asks MS Teams pairing-mode DMs to surface unpaired senders with a log line and/or pair-code reply instead of silently storing and dropping them.

## Best Possible Solution

MS Teams should reuse `pairing.issueChallenge` for unpaired personal DMs, preserving conversation-reference capture and drop semantics while sending and logging the one-time pair-code challenge.

## Reproduction Assessment

Yes, source-reproducible on current main. A personal MS Teams message from an unapproved sender with `channels.msteams.dmPolicy: "pairing"` reaches the direct upsert branch and returns before any Teams reply is sent.

## Solution Assessment

Yes. Reusing `pairing.issueChallenge` is the narrowest maintainable repair because it already centralizes request creation, one-time replies, code-bearing creation callbacks, and reply-error handling used by other channels.
