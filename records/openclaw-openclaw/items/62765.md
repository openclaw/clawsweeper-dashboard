## Summary

Keep open: current `main` still records and drops unpaired MS Teams personal DMs without sending the pairing challenge reply, and the linked repair PR is closed unmerged. The bug is narrow and source-reproducible, with the shared pairing challenge helper already available for the missing behavior.

## What This Changes

MS Teams should surface a pairing challenge and useful log for unpaired direct messages under `channels.msteams.dmPolicy = "pairing"` instead of silently recording and dropping them.

## Best Possible Solution

Update the MS Teams personal-DM pairing rejection path to use the existing `pairing.issueChallenge` flow while preserving conversation-reference capture, access drop semantics, and repeat-request behavior.

## Reproduction Assessment

Yes, from current source inspection. A personal MS Teams DM from an unapproved sender with `channels.msteams.dmPolicy: "pairing"` reaches the pairing branch, calls `upsertPairingRequest`, and returns without any `sendActivity` or `issueChallenge` call.

## Solution Assessment

Yes. Reusing `pairing.issueChallenge` is the narrowest maintainable fix because the shared helper already implements create-if-missing behavior, standard reply text, code-bearing creation callbacks, and reply-error handling used by other channel plugins.
