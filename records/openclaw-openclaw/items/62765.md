## Summary

Keep open: current main and the latest release still record and drop unpaired MS Teams personal DMs without sending the standard pairing challenge, and the linked repair PR is closed unmerged.

## What This Changes

The issue requests visible MS Teams pairing feedback for unpaired direct-message senders under `dmPolicy: "pairing"` instead of a silent HTTP 200 drop.

## Best Possible Solution

MS Teams should reuse the existing `pairing.issueChallenge` helper for unpaired personal DMs while preserving conversation-reference capture, authorization decisions, and drop semantics.

## Reproduction Assessment

Yes, by source inspection. A personal MS Teams message from an unapproved sender under `dmPolicy: "pairing"` reaches the pairing branch, records a request, and returns before any Teams reply path is called.

## Solution Assessment

Yes. Reusing `pairing.issueChallenge` is the narrowest maintainable fix because the existing helper centralizes request creation, first-time reply text, code-bearing callbacks, and reply-error handling.
