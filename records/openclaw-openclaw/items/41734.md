## Summary

Keep open: current main does not contain this verifier, and the PR still needs rebase/security fixes because it targets retired owner-only policy names, stale tool/policy fixtures, and fail-open AST extraction paths.

## What This Changes

The PR adds parser-only `src/verify` AST tooling, types, README documentation, a Vitest suite, and a standalone runner for extracting tool catalog and policy metadata for future SMT/Z3 verification.

## Best Possible Solution

If maintainers want this in core, land a rebased parser-only lane that targets the live catalog, owner-only approval-class map, and subagent policies, fails closed on unsupported AST forms, and documents only functionality present in the PR.

## Reproduction Assessment

Yes. Source comparison against current main reproduces the blockers: the parser searches for a retired owner-only symbol, and the tests assert old catalog and policy fixtures.

## Solution Assessment

No. Parser-only tooling is a reasonable incremental direction, but this implementation is not the best current solution until it is rebased to live contracts, fails closed, and removes docs for absent SMT generation.
