## Summary

Keep open. The branch is not implemented or superseded on current main, but the current patch is not mergeable because it targets stale catalog, owner-only, and pipeline contracts, and the attached proof does not show current after-fix parser behavior.

## What This Changes

The PR adds internal `src/verify` AST parsers, parser tests, shared types, a standalone runner, and a README for extracting tool catalog, policy, and pipeline data for future formal verification.

## Best Possible Solution

Land a maintainer-approved verification lane that parses the current catalog, approval-class owner-only map, subagent arrays, and sender-policy pipeline fail-closed, with docs limited to implemented parser surfaces until SMT generation exists.

## Reproduction Assessment

Yes for review findings from source inspection. Current main has 33 tools, an 8-step sender-policy pipeline, and approval-class owner-only policy while this branch targets 25 tools, 7 steps, and a retired owner-only symbol.

## Solution Assessment

No. Parser-only tooling is a plausible increment, but this branch should first be rebased to current contracts, fail closed on unparsed AST data, and remove generator docs until generator files exist.
