## Summary

Keep open: this PR is not implemented on current main and the branch still targets retired/stale policy contracts, so it would not verify the live tool-policy surface without a rebase and parser fixes.

## What This Changes

The PR adds a new `src/verify` module with AST parsers, README, Vitest tests, and a standalone runner for extracting tool catalog, policy, and pipeline metadata for future SMT/Z3 verification.

## Best Possible Solution

Land a maintainer-approved verification lane that parses the current catalog, approval-class owner-only policy, subagent policy arrays, and pipeline steps fail-closed, with docs and proof scoped to the code actually added.

## Reproduction Assessment

Yes for review purposes. Source comparison shows the PR expects 25 tools and 7 pipeline steps while current main has 33 tool IDs and an 8-step sender-policy pipeline, and the parser targets a retired owner-only symbol.

## Solution Assessment

No. Parser-only tooling is a plausible increment, but this branch should first be rebased to current contracts, made fail-closed for unparsed AST data, and paired with current parser-only behavior proof.

  - confidence: 0.96
