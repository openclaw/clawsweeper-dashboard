## What This Changes

Adds internal `src/verify` AST parsers, parser tests, shared types, a standalone runner, and a README for extracting tool catalog, policy, and pipeline data for future SMT/Z3 verification.

## Best Possible Solution

Land a maintainer-approved verification lane that parses the current catalog, approval-class owner-only map, subagent arrays, and sender-policy pipeline fail-closed, with docs limited to implemented parser surfaces until SMT generation exists.

## Reproduction Assessment

Yes for the review findings from source inspection: current main has 33 tools, an 8-step sender-policy pipeline, and an approval-class owner-only policy while this branch targets 25 tools, 7 steps, and a retired owner-only symbol.

## Solution Assessment

No. Parser-only tooling is a plausible increment, but this branch should first be rebased to current contracts, made fail-closed on unparsed AST data, and have generator docs removed until generator files exist.
