## Summary

This PR should remain open, but it is not merge-ready: current main still has no `src/verify` implementation, and the branch adds security-adjacent verification tooling that is stale against current tool-policy contracts and can silently omit parsed policy data.

## What This Changes

The PR adds a new `src/verify` parser module, README, Vitest tests, and standalone runner for extracting tool catalog, policy, and pipeline metadata for future SMT/Z3 verification.

## Best Possible Solution

Land a maintainer-approved verification lane that parses current catalog, approval-class owner-only policy, subagent policy arrays, and pipeline steps fail-closed, with docs and proof scoped to the code actually added.

## Reproduction Assessment

Yes for review purposes. Source comparison against current main shows the PR expectations conflict with the live 30-tool catalog, approval-class owner-only map, and sender-policy pipeline shape.

## Solution Assessment

No. Parser-only tooling is a plausible increment, but this branch should first be rebased to live contracts, made fail-closed for unparsed AST data, and paired with current parser-only behavior proof.
