## Summary

Keep this PR open for maintainer review rather than cleanup-close: current main does not contain the proposed `src/verify` module, and the branch is useful source work, but it is stale against the live tool-policy contracts and has correctness/security-verification issues that should block merge.

## What This Changes

The PR adds a parser-only `src/verify` module with AST parsers, shared parsed-policy types, README documentation, a Vitest suite, and a standalone runner for extracting OpenClaw tool catalog and policy metadata for future SMT-LIB2/Z3 verification.

## Best Possible Solution

If maintainers want this verification lane in core, land it as a rebased parser-only change that targets the current catalog, owner-only approval-class map, and subagent policy shapes, fails closed on unsupported AST forms, and documents only generator/Z3 functionality that is actually included.

## Reproduction Assessment

Yes. A high-confidence reproduction path is source comparison against current main: the PR parser searches for a retired owner-only symbol, while current main has a 32-tool catalog and different owner-only/subagent policy fixtures than the branch asserts.

## Solution Assessment

No. Parser-only tooling is a reasonable incremental direction, but this implementation is not the best current solution until it targets live contracts, rejects partial extraction, and avoids documenting absent SMT generator workflows.
