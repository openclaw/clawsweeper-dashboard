## Summary

Keep open: current main does not already implement the verifier module, but this PR is not merge-ready because it targets stale tool-policy contracts and can produce partial verification inputs.

## What This Changes

Adds a new parser-only `src/verify` module with AST parsers, parsed policy types, README documentation, a Vitest parser suite, and a standalone runner for future SMT/Z3 tool-policy verification.

## Best Possible Solution

If maintainers want this verifier lane in core, land a rebased parser-only module that targets the current catalog, approval-class map, and subagent policy shapes, fails closed on unsupported AST forms, and documents only shipped parser functionality.

## Reproduction Assessment

Yes. A high-confidence source reproduction is comparing the PR parser and fixtures against current main: the PR searches for `OWNER_ONLY_TOOL_NAME_FALLBACKS`, while main uses `OWNER_ONLY_TOOL_APPROVAL_CLASS_FALLBACKS` and a 33-tool catalog.

## Solution Assessment

No. Parser-only tooling is a reasonable incremental direction, but this implementation is not the best current solution until it targets live contracts, rejects partial extraction, and removes docs for generator workflows that are not included.

  - confidence: 0.95

  - body: `parseStringArray` filters unsupported entries and callers only check that the result is non-empty. If a deny list later uses a constant, spread, or template entry, extraction can silently omit that denied tool and reason about a weaker policy than runtime enforces.
  - confidence: 0.88

  - body: `extractStringArray` filters profile entries it cannot decode and returns the truncated array. A catalog refactor through constants, spreads, or templates would silently remove profile memberships from the verification model instead of stopping extraction.
  - confidence: 0.84
- **[P2] Remove references to absent SMT generator files:** `src/verify/README.md:17-45`
  - body: The README documents `generate-smt.ts`, `emit-smt/`, generated model files, and Z3 commands, but this branch only adds parser code. Users following these docs would hit missing-module or missing-output errors.
