## Summary

Keep open for maintainer review; the branch still contains useful verification work but is stale against current main contracts, has security-verifier fail-open gaps, and lacks sufficient after-fix real behavior proof.

## What This Changes

Adds a new `src/verify` parser-only module with TypeScript AST parsers, parsed-policy types, README docs, a Vitest suite, and a standalone runner for future SMT/Z3 tool-policy verification.

## Best Possible Solution

Land only a maintainer-approved core verification lane that parses the current tool catalog, approval-class map, subagent deny lists, and 8-step policy pipeline without silently dropping unparsed entries.

## Reproduction Assessment

Yes. Source reproduction is comparing the PR against current main: current main defines `OWNER_ONLY_TOOL_APPROVAL_CLASS_FALLBACKS` and has 33 tools and 8 pipeline steps, while the PR parser/tests target `OWNER_ONLY_TOOL_NAME_FALLBACKS`, 25 tools, and 7 steps.

## Solution Assessment

No. Parser-only tooling is a plausible incremental direction, but this branch should be rebased onto live contracts, fail closed on partial AST extraction, and either remove or include the documented SMT generator before merge.

  - confidence: 0.86

  - confidence: 0.96
