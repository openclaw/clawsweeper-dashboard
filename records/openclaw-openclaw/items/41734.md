## Summary

Keep open for maintainer review: current main does not include this verifier module, but the PR is stale against live tool-policy and catalog contracts and has false-proof risks before merge.

## What This Changes

The PR adds parser-only `src/verify` tooling with AST extractors, tests, and README documentation for future SMT/Z3 tool-policy verification.

## Best Possible Solution

If maintainers want this verifier lane in core, land a rebased parser-only change that targets the current catalog, owner-only approval-class map, and subagent policy shapes, fails closed on unsupported AST forms, and documents only functionality included in the diff.

## Reproduction Assessment

Yes. Source comparison is enough: the PR parser targets a retired owner-only symbol while current main uses approval-class fallbacks, and the PR tests still expect 25 tools while current main lists newer catalog entries.

## Solution Assessment

No. Parser-only tooling is a reasonable incremental direction, but this implementation is not the best current solution until it targets live contracts, rejects partial extraction, and removes docs for absent SMT generator workflows.

  - confidence: 0.95
