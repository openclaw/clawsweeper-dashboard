## Summary

Keep this PR open, but it is not ready to merge: the parser-only verification surface is useful in direction, yet the branch targets retired policy/catalog contracts and its proof is stale against current main.

## What This Changes

The PR adds a new `src/verify` parser module, README, Vitest coverage, and a standalone runner for extracting tool catalog, policy, and pipeline metadata for future SMT/Z3 verification.

## Best Possible Solution

Land only a maintainer-approved verification lane rebased to live policy/catalog contracts, with fail-closed parsing and README/proof aligned to the files actually included.

## Reproduction Assessment

Yes. Source comparison is high confidence: current main uses an approval-class owner-only map, 33 catalog tools, and eight pipeline steps, while the PR targets the old fallback set, 25 tools, and seven steps.

## Solution Assessment

No. Parser-only tooling is a plausible increment, but this branch should be rebased onto live contracts, fail closed on every unparsed policy/catalog element, and remove or add the documented SMT generator before merge.
