## Summary

Keep open: the underlying liveness bug is valid on current main, but this PR targets a retired legacy tool path and lacks after-fix real behavior proof, so it cannot merge as-is.

## What This Changes

The branch wraps the legacy `src/agents/tools/memory-tool.ts` `memory_search` execution in a 30-second timeout, adds timeout-specific unavailable text, and adds a fake-timer unit test.

## Best Possible Solution

Port the fail-open deadline to `extensions/memory-core` around runtime loading, manager acquisition, and active search, return timeout-specific unavailable metadata, preserve corpus behavior, and add focused regression coverage.

## Reproduction Assessment

Yes for source-level reproduction: stub `getMemorySearchManager` or `memory.manager.search` to return a never-settling promise, and current `extensions/memory-core/src/tools.ts` will await it without a deadline. I did not run a live Telegram/iCloud setup in this read-only review.

## Solution Assessment

No. The hard fail-open timeout is the right direction, but this PR applies it to legacy `src/agents/tools/memory-tool.ts` instead of the active `extensions/memory-core` implementation.
