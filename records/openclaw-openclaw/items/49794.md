## Summary

Keep this PR open. The linked liveness bug is still valid, but the branch targets the old `src/agents/tools` memory tool path while current `main` registers and runs `memory_search` from `extensions/memory-core`, where manager acquisition and search still await directly.

## What This Changes

The PR wraps the old `src/agents/tools` `memory_search` implementation in a 30-second `Promise.race` timeout, adds timeout-specific unavailable metadata, and adds a fake-timer timeout test.

## Best Possible Solution

Port the timeout behavior to `extensions/memory-core`: bound the active `memory_search` execution around manager acquisition plus search, return timeout-specific unavailable metadata, preserve existing corpus and supplement behavior, and add a fake-timer regression test on the active tool.

## Reproduction Assessment

Yes. A memory-core test can set the mocked search implementation to a never-resolving promise and observe that current `main` awaits it directly; the same code inspection also shows manager acquisition is unbounded.

## Solution Assessment

No. The tool-boundary timeout direction is reasonable, but this PR is not the best current implementation because it patches the retired path instead of the active memory-core plugin.
