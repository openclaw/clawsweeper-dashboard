## Summary

Keep open. The liveness bug is still plausible on current main, but this PR is stale because it modifies the removed legacy core memory_search files while active memory_search is provided by memory-core and still awaits manager acquisition/search without a hard fail-open guard.

## What This Changes

The PR adds a 30-second Promise.race timeout plus timeout-specific unavailable metadata and tests for the legacy core memory_search tool.

## Best Possible Solution

Port the liveness guard to memory-core around manager acquisition and active search, return timeout-specific unavailable metadata, preserve corpus/supplement behavior, and keep #49524 linked until the port merges.

## Reproduction Assessment

Yes. A memory-core test can set the mocked search implementation to a never-resolving promise and observe that the active execute path awaits it directly; code inspection also shows manager acquisition is unbounded.

## Solution Assessment

No. The tool-boundary fail-open direction is reasonable, but this PR is not the best current implementation because it patches a retired path instead of the active memory-core plugin.
