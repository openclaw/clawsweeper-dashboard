## What This Changes

The PR adds a 30-second Promise.race timeout and timeout-specific unavailable-result assertions to the legacy src/agents/tools memory_search implementation.

## Best Possible Solution

Port or replace this PR with a memory-core fix that wraps the full active memory_search boundary in a hard fail-open deadline and adds stalled-promise regression coverage.

No. A hard fail-open timeout is the right fix direction, but this PR applies it to retired src/agents/tools files instead of the active extensions/memory-core implementation.
