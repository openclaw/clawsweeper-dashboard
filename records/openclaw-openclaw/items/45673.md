## Summary

Keep this PR open: current main has the basic Feishu session-account-before-default behavior, but it still lacks the PR's channel-origin guard and normalized inherited-account handling, and the PR needs a required changelog entry before landing.

## What This Changes

The PR threads Feishu tool account context through tool call sites so explicit `accountId` wins, Feishu-origin session account IDs can override configured `defaultAccount`, non-Feishu inherited account IDs fall back to the configured Feishu default, and targeted tests cover those routes.

## Best Possible Solution

Land a rebased Feishu-plugin fix that keeps explicit `accountId` first, accepts normalized inherited account IDs only for Feishu-origin sessions, falls back to configured `defaultAccount`, and records the user-facing fix in the changelog.

## Solution Assessment
