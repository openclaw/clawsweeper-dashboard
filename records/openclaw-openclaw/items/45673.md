## Summary

Keep this PR open: current main has the central explicit > contextual > configured default Feishu precedence, but it still lacks the PR's unique Feishu-origin guard for inherited session account context, and the external PR still needs inspectable real behavior proof before merge.

## What This Changes

The PR changes Feishu tool account resolution and tests so explicit accountId wins, then Feishu-origin session account context, then configured defaultAccount fallback.

## Best Possible Solution

Feishu tools should inherit session account context only from Feishu-origin turns, while explicit accountId remains the override and configured defaultAccount remains the fallback.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main forwards ctx.agentAccountId into Feishu tool clients without ctx.messageChannel, so a non-Feishu session whose account id collides with a Feishu account can select that tenant over defaultAccount.

## Solution Assessment

Yes for direction. Scoping inherited account precedence to Feishu-origin tool contexts is the narrow maintainable fix, but the branch needs rebase work and inspectable real Feishu behavior proof before merge.
