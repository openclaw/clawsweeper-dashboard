## Summary

Keep open: this draft, maintainer-labeled PR is useful cleanup but still has merge-blocking installability, public SDK compatibility, security, and real-behavior-proof gaps.

## What This Changes

The PR moves filesystem safety helpers to `@openclaw/fs-safe`, rewires many core/plugin call sites, updates Plugin SDK exports/docs/tests, and adds release-check coverage for local fs-safe specs.

## Best Possible Solution

Use a published semver dependency or an in-repo workspace package with lockfile integrity, preserve public SDK compatibility aliases or defer removals to a major-version plan, then attach real behavior proof from a clean setup.

## Reproduction Assessment

Yes for the review findings: latest PR head source inspection shows a runtime `link:../fs-safe` dependency and removal of existing public SDK exports. I did not run an install because this review is read-only, but the source path is enough to reproduce in a clean checkout without the sibling package.

## Solution Assessment

No: extracting shared filesystem primitives is a reasonable direction, but shipping via a sibling link and dropping public SDK names is not the narrowest maintainable path. The safer fix is a published or workspace dependency plus compatibility aliases or an explicit breaking-release plan.
