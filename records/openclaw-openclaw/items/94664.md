The branch changes `WizardSessionPrompter.progress()` to emit latest-wins non-interactive `progress` steps over the RPC wizard protocol and adds `WizardSession` tests for delivery, cancellation, non-answerability, and concurrent `next()` calls.

## Best Possible Solution

## Reproduction Assessment

Yes, source-reproducible: current main drops `WizardSessionPrompter.progress()` updates, and the PR returns progress ids that current bundled clients answer while `session.answer()` rejects unregistered step ids. I did not run a live RPC client in this read-only review.

## Solution Assessment

No. Emitting progress from `WizardSession` is the right owner boundary, but the best fix must preserve or explicitly gate current RPC clients and include real-client proof.
