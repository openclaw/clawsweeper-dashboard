## Summary

This should stay open: current main and v2026.6.1 still contain the reported memory-index state where missing metadata plus existing chunks leaves the manager dirty and search paused instead of repairing the metadata or rebuilding.

## What This Changes

The issue asks for gateway memory auto-sync to recover when `memory_index_meta_v1` is missing while indexed chunks already exist, so `memory_search` no longer returns zero results due to missing index metadata.

## Best Possible Solution

Make the memory manager recover missing metadata with existing chunks by forcing the existing safe reindex path or by writing validated metadata, then cover that gateway-style state with a focused regression test.

## Reproduction Assessment

Yes, at source level: current code can be put in the reported state by indexing chunks, deleting `memory_index_meta_v1`, and running non-CLI sync/search. I did not run a live gateway repro in this read-only pass.

## Solution Assessment
