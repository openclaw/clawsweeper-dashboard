## Summary

Keep open: current main still drops read-tool image content when the result lacks a MEDIA token or details.path, and the open closing pull request https://github.com/openclaw/openclaw/pull/81415 has not merged yet.

## What This Changes

The issue reports that WhatsApp replies after reading an image send text-only because processed image media is not attached to outbound messages.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81415, or an equivalent shared read-tool media bridge, so image bytes are persisted to inbound media and delivered through the existing outbound media URL path.

## Reproduction Assessment

Yes. A read tool result with image content but no MEDIA token or details.path makes the extractor return undefined, so the handler never queues pending media and WhatsApp receives no primary media URL.

## Solution Assessment

Yes. The linked PR’s shared read-tool media bridge is the narrow maintainable direction because it feeds the existing outbound media URL path instead of adding a WhatsApp-only workaround.
