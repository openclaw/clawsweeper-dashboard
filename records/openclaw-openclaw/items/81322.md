## Summary

Keep open: current main still has a source-visible gap between read-tool image processing and outbound WhatsApp media delivery. WhatsApp can send media when a media URL reaches the send path, but Pi SDK read-image results currently provide base64 image content without a media path, so the pending tool-media bridge has nothing to attach.

## What This Changes

Restore WhatsApp outbound image attachments when an agent reads an image and replies, so processed images reach the WhatsApp media send path instead of text-only delivery.

## Best Possible Solution

Bridge trusted read-tool image outputs into the existing pending tool-media/outbound media pipeline with scoped local-media handling and regression coverage, without adding WhatsApp-specific special cases.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection explains the symptom: read-image results can be processed as base64 images while producing no media URL for WhatsApp outbound delivery.

## Solution Assessment

Yes, the best fix direction is the shared read-tool media bridge, not the WhatsApp sender itself. WhatsApp already sends media when a media URL reaches `sendMessageWhatsApp`.
