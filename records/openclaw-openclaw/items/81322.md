## What This Changes

The issue reports that WhatsApp replies produced after reading an image process the image but send text-only messages because no outbound media is attached.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81415, or an equivalent shared read-tool media bridge, so read image bytes are persisted into inbound media and delivered through existing outbound media URLs.

## Reproduction Assessment

Yes. The narrow maintainable fix belongs in the shared read-tool media bridge rather than a WhatsApp-only workaround, and the linked PR targets that boundary.
