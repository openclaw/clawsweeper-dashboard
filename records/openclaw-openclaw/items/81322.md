## What This Changes

The issue reports that WhatsApp replies send text but no attached image after the read tool processes image content, leaving outbound logs with hasMedia false.

## Best Possible Solution

Land a reviewed shared read-tool media bridge that persists trusted image content into the existing inbound media store and queues it for normal outbound delivery, without adding WhatsApp-specific special cases.

## Reproduction Assessment

No live end-to-end reproduction was run. The source path is source-reproducible: read-image results without MEDIA or details.path produce no media URL, and WhatsApp hasMedia false follows when no URL reaches sendMessageWhatsApp.

## Solution Assessment

Yes. The narrow maintainable fix is in the shared read-tool media bridge because WhatsApp already sends media when media URLs reach its outbound path.
