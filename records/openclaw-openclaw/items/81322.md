## What This Changes

The issue reports that WhatsApp replies send text only after reading an image because the processed image is not attached to outbound messages.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81415, or an equivalent shared read-tool media bridge, so read image bytes are persisted and delivered through the existing outbound media pipeline.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection reproduces the failure condition: a read result with image content but no MEDIA token or details.path produces no media path, leaving WhatsApp with hasMedia false.

## Solution Assessment

Yes. The narrow maintainable fix is in the shared read-tool media bridge so all outbound channels can receive media URLs through existing delivery paths, and the linked PR is already aimed at that boundary.
