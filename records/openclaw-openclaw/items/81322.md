## What This Changes

Restore WhatsApp outbound attachment delivery for images produced by the read tool when the tool result contains image bytes but no `MEDIA:` token or `details.path`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81415, or an equivalent shared read-tool media bridge, then close this issue once the fix is merged.

Yes. Source inspection shows a read result with image content but no `MEDIA:` token or `details.path` produces no media artifact, so the pending-media queue stays empty and WhatsApp receives no primary media URL.

## Solution Assessment

Yes. The linked PR’s shared read-tool media bridge is the right direction because it feeds the existing outbound media URL path instead of adding a WhatsApp-specific workaround.
