## Summary

Keep open. Current main still captures a same-line serialized JSON suffix as part of a local MEDIA path, and the earlier closing PR is closed and unmerged, so the reported parser bug remains actionable.

## What This Changes

Harden shared MEDIA directive parsing so local media paths do not include trailing serialized JSON before channel delivery.

## Best Possible Solution

Add a shared parser-level fix in src/media/parse.ts with focused regression coverage in src/media/parse.test.ts for MEDIA:/path/to/image.png followed by same-line serialized JSON. Channel-specific sanitization can be defense-in-depth, but the primary contract should prevent polluted mediaUrls from leaving the shared parser.

Yes. Source inspection gives a high-confidence reproduction path: MEDIA_TOKEN_RE captures the full same-line payload, cleanCandidate leaves the embedded JSON boundary intact, and the local read path later opens that polluted string. I did not run Vitest because this sweep is read-only.

## Solution Assessment

Yes. A narrow shared-parser fix with regression coverage is the best repair direction; a channel-only workaround would leave other outbound consumers exposed to the same polluted mediaUrl.
