## Summary

Keep open. Current main still lets a same-line serialized JSON suffix remain inside a slash-prefixed MEDIA local path, and the exact fix PR that referenced this issue is closed and unmerged; the open multiline MEDIA PR is adjacent but does not supersede this suffix-parsing bug.

## What This Changes

The issue reports that malformed MEDIA directives can pass a local path with appended serialized JSON into outbound media delivery.

## Best Possible Solution

Keep this open until the shared MEDIA parser prevents polluted mediaUrls from leaving src/media/parse.ts. Channel-level checks can add defense in depth, but the durable fix belongs in the shared parser contract with regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: MEDIA_TOKEN_RE captures the full line, cleanCandidate does not truncate embedded JSON after the extension, and slash-prefixed candidates are accepted as local media before outbound delivery tries to normalize them. I did not run tests because this review is read-only.

## Solution Assessment

Yes. A narrow parser-level fix plus src/media/parse.test.ts regression coverage is the best repair direction; a Weixin-only or channel-only workaround would leave other outbound consumers exposed to the same polluted mediaUrl.
