## Summary

Keep open. Current main still lets a same-line serialized JSON suffix remain inside a slash-prefixed local `MEDIA:` path, the focused fix PR #75202 is closed and unmerged, and the open multiline parser PR #68338 is adjacent but does not supersede this suffix-parsing bug.

## What This Changes

The issue requests hardening shared `MEDIA:` directive parsing so a local media path cannot absorb trailing serialized tool-result JSON before outbound channel delivery.

## Best Possible Solution

Keep this open until the shared parser prevents polluted `mediaUrls` from leaving `src/media/parse.ts`, with focused regression coverage for same-line JSON suffixes and no channel-specific workaround as the primary fix.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: `MEDIA_TOKEN_RE` captures the whole same-line payload, `cleanCandidate` does not truncate the embedded JSON suffix, and slash-prefixed local candidates are accepted before outbound delivery tries to read them.

## Solution Assessment

Yes. A narrow shared parser fix plus `src/media/parse.test.ts` regression coverage is the best repair path; channel-level checks can be defense in depth but would leave other outbound consumers exposed.
