## Summary

Keep open. Current main still accepts the reported polluted MEDIA token shape through the shared parser, and the focused closing PR remains open and unmerged, so this issue should wait for that PR to land or be replaced.

## What This Changes

The issue requests hardening shared MEDIA directive parsing so local media paths followed by serialized tool-result JSON are trimmed or rejected before channel delivery attempts local file I/O.

## Best Possible Solution

Add or land a shared parser-level fix that prevents serialized JSON from becoming part of a local MEDIA path, with focused regression coverage in `src/media/parse.test.ts`; channel adapters may add defense-in-depth but should not be the primary owner of the fix.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a one-line `MEDIA:/path/to/image.png"}],"details":...` token is captured as one payload, not split without whitespace, accepted as a local path, and later read as that exact polluted path.

## Solution Assessment

Yes. The narrow maintainable solution is a shared parser fix with parser regression coverage; the existing linked PR is the right review path unless maintainers replace it with an equivalent parser-level change.
