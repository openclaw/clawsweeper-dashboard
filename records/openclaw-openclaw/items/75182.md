## Summary

Keep open. Current main still accepts the reported polluted local MEDIA path shape, and unmerged PR #75202 already targets the exact parser regression, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Harden shared MEDIA directive parsing so a local media path followed immediately by serialized JSON does not pass the polluted path to channel media delivery.

## Best Possible Solution

Land or revise PR #75202, or an equivalent shared parser fix, so MEDIA local paths are truncated or rejected at a valid media/document boundary before serialized JSON reaches channel delivery, with focused parser regression coverage.

## Reproduction Assessment

Yes. Static inspection of current main gives a high-confidence reproduction path: a same-line `MEDIA:/path/image.png"}],"details":...` payload is captured, not truncated, accepted as a local path, and later opened as that exact path. I did not run tests to keep this review read-only.

## Solution Assessment

Yes. A shared parser-level fix with regression coverage is the narrowest maintainable solution; a channel-only cleanup would not protect all outbound media delivery paths.
