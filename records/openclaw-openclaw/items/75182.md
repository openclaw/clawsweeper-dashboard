## Summary

Keep open. Current main still has the shared MEDIA parsing path that can absorb a same-line serialized JSON suffix into a slash-prefixed local path, and #75202 is open/unmerged as the linked closing PR for this issue.

## What This Changes

The issue reports that malformed `MEDIA:` directives can include trailing serialized JSON in parsed local media paths, causing channels to try opening a polluted path.

## Best Possible Solution

Land or revise the linked shared-parser fix so MEDIA local paths stop at a valid media boundary before serialized JSON, with focused parser regression coverage, then let the linked PR close this issue after merge.

## Reproduction Assessment

Yes. The current source gives a high-confidence reproduction path: a one-line `MEDIA:/path/to/image.png"}],...` payload is captured as one token, cleaned without truncating the embedded quote, accepted as a local path, and later opened as that exact path. I did not run tests or a live channel because this review was read-only.

## Solution Assessment

Yes. A shared parser-level fix with regression coverage is the narrowest maintainable solution; fixing only one channel adapter would leave other MEDIA consumers exposed to the same polluted parser output.
