## Summary

Keep open. Current checked-out main still accepts the reported no-whitespace `MEDIA:` payload as a local path, and PR #75202 explicitly fixes this issue but remains open and unmerged, so this issue should wait for that PR or an equivalent shared-parser fix to land.

## What This Changes

Harden MEDIA directive parsing so a local media path followed by serialized JSON is extracted as the valid file path or rejected before channel delivery.

## Best Possible Solution

Land or revise PR #75202 as a shared MEDIA parser fix with focused regression coverage in `src/media/parse.test.ts`, then let GitHub close this issue after the fix merges.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: the reported one-line `MEDIA:/path/to/image.png\"}],\"details\":...` payload remains one candidate, passes local-path validation, and is later read as that exact path. Tests were not run because this review was read-only.

## Solution Assessment

Yes. A shared parser-level fix with parser regression coverage is the narrowest maintainable solution; channel adapters can add defense-in-depth, but they should not be the primary fix.
