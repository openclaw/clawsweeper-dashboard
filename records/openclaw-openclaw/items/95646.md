## Summary

Keep open: current main and latest v2026.6.9 still have the Mattermost private-channel chat_type split where inbound type `P` maps to `group`, but outbound/session derivation can still turn the same `channel:<id>` target into `channel`.

## What This Changes

The issue asks OpenClaw to make Mattermost private-channel target-to-chat-type inference deterministic so private channels do not split across `group` and `channel` session namespaces.

## Best Possible Solution

Mattermost should keep chat-type classification at the plugin boundary and derive private-channel outbound/session routes from authoritative channel kind or resolved target metadata while preserving public channels as `channel`.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: inbound Mattermost private-channel posts use `P -> group`, while outbound/session reconstruction still builds `channel:<id>` as `channel`. I did not run a live Mattermost instance.

## Solution Assessment

Yes, the best fix is a narrow plugin-owned classification and route repair, with shared fallback ordering adjusted only if needed so plugin-owned semantics can override lossy target strings.
