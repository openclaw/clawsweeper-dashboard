## Summary

Keep open: this is a real P1 bugfix direction with sufficient live-log proof, but the latest patch still has a blocking final-output boundary bug in live Claude sessions where pre-tool commentary can be surfaced as final assistant text.

## What This Changes

The PR adds Claude JSONL parser fallback logic that preserves streamed text when a final empty result arrives, records `CliOutput.hadToolCalls`, and adds parser plus runner regression tests.

## Best Possible Solution

Preserve true streamed final assistant text on empty Claude results while using the same commentary/tool-use boundary for buffered final output and live-session finalization; keep true empty and tool-only fallback policy unchanged.

## Reproduction Assessment

Yes, source-reproducible: live Claude sessions push raw lines through the streaming parser but finalize with `parseCliOutput(raw)`, while the PR's buffered parser returns accumulated deltas on empty result without applying the commentary/tool-use split. I did not run tests because this review is read-only.

## Solution Assessment

No, not yet: preserving streamed text is the right fix direction, but the current parser-only shape is incomplete because the live-session final-output path can lose the commentary boundary. A safer repair is to reuse the streaming parser's normalized output or mirror the same boundary state in the buffered parser.
