## Summary

Keep open: current main and v2026.6.9 still replace the accumulated Claude streamed text with an empty final result, while this parser-only PR preserves the runner's existing empty-response failover contract. The remaining work is maintainer acceptance of the provider-routing/message-delivery behavior plus exact-head gate cleanup.

## What This Changes

The branch changes Claude CLI JSONL parsing to preserve accumulated streamed assistant text when an empty final result event arrives, adds a `CliOutput.hadToolCalls` parser signal, and adds parser/runner regression tests for empty-result and tool-only fallback behavior.

## Best Possible Solution

Land the parser-only fix after exact-head gates are clean and maintainer review accepts that already-streamed Claude text should beat a later empty result; keep synthetic-placeholder observability in the separate open thread.

## Reproduction Assessment

Yes for the parser-level shape: current main returns the empty final Claude result before preserving earlier deltas, and the PR adds focused JSONL fixtures for both buffered and streaming parsers. The live Claude event remains intermittent, so this was not independently reproduced against a real Claude CLI stream in this review.

## Solution Assessment

Yes, this is the best current fix shape: it preserves streamed text in the parser, where the information is still available, while leaving the runner's empty-response failover policy intact. A runner-level bypass would be broader and riskier, and the related synthetic-placeholder case should remain separate.
