## Summary

Keep open: the PR targets a real Claude CLI empty-result bug and has useful production proof, but the current patch can still turn pre-tool commentary into final assistant text on empty-result live-session turns because the buffered parser does not preserve the streaming parser's commentary/tool-use boundary.

## What This Changes

This PR changes Claude CLI JSONL parsing to preserve accumulated streamed deltas when an empty final result event arrives, records `hadToolCalls`, and adds parser plus runner regression tests.

## Best Possible Solution

Preserve streamed final assistant text on empty Claude results while carrying the streaming parser's commentary/tool-use boundary into the final live-session output; keep the existing empty-response fallback for truly empty or tool-only turns unless silence was already allowed.

## Reproduction Assessment

## Solution Assessment

No, not yet: preserving streamed text is the right direction, but the current parser-only shape is incomplete because it does not preserve the commentary/tool-use boundary in the live-session final-output path.
