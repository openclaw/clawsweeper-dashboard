## Summary

Keep this PR open as the active focused fix for the linked multimodal memory-ranking bug. Current main and v2026.6.8 still use uniform hybrid scoring that discounts vector-only image/audio hits, while the PR targets that fusion boundary with derived modality information and focused edge-case tests.

## What This Changes

The PR changes memory-core hybrid result fusion so vector-only classified image/audio candidates can be scored on their vector signal while preserving keyword-only and both-signal media scoring, and adds focused regression tests.

## Best Possible Solution

Land the focused fusion-boundary fix after maintainer acceptance of the changed multimodal recall ordering, keeping the repair schema-free and config-free.

## Reproduction Assessment

Yes. Current main seeds vector-only candidates with `textScore: 0` and applies the default `0.7 * vector + 0.3 * text` score, matching the linked issue's discount path and the PR's before/after proof.

## Solution Assessment

Yes. The patch fixes the fusion boundary with derived modality information from the existing classifier and avoids a new config, schema, or persisted modality column; persisting modality would be a broader migration-heavy alternative for no clear payoff here.
