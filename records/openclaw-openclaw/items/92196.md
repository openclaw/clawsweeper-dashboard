## Summary

Keep this PR open, but it is not merge-ready: the fix is at the right hybrid-fusion boundary and the supplied proof is strong, but the current guard still drops BM25/text relevance for classified media that also has an FTS hit under positive vector weight.

## What This Changes

The branch makes memory-core hybrid fusion modality-aware for classified image/audio results, wires the manager to the multimodal path classifier, and adds regression tests for vector-only, keyword-only, text-only, and zero-vector-weight cases.

## Best Possible Solution

Keep modality-aware scoring at the hybrid merge boundary, but drop text weight only for vector-only classified media and preserve the configured BM25 blend when a keyword hit exists.

## Reproduction Assessment

Yes. Current main source shows vector-only media is discounted because textScore is 0 and the unconditional hybrid formula still applies textWeight; the PR-introduced both-signals edge is also source-reproducible from the new guard and production caller.

## Solution Assessment

No. The PR uses the right owner boundary, but the condition is too broad because it removes BM25 relevance even when classified media actually matched FTS; preserving textScore when present is the narrower fix.
