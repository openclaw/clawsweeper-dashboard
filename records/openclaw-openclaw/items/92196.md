## Summary

The PR correctly targets the reported default-weight ranking bug and supplies useful full-pipeline proof, but the new condition regresses an explicitly supported keyword-only weight configuration, so it should remain open for a narrow repair.

## What This Changes

The PR makes hybrid memory fusion ignore synthetic text scoring for vector-backed non-text media and adds regression coverage for default mixed weights and keyword-only media without vector results.

## Best Possible Solution

Keep the modality-aware normalization, but drop the synthetic media text signal only when the candidate has a vector result with positive configured vector weight; add a regression test for media present in both result sets under `vectorWeight: 0, textWeight: 1`.

## Reproduction Assessment

Yes. The reported default-weight bug follows directly from the current fusion formula and is supported by the PR's full-pipeline before/after output; the introduced zero-vector-weight regression is also deterministic from the schema and changed scoring branch.

## Solution Assessment

No, not yet. Reusing the existing modality classifier is the right owner-correct approach, but the condition must account for zero vector weight so it does not remove the only meaningful configured signal.
