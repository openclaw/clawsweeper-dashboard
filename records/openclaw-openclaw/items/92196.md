## Summary

Keep this PR open, but not merge-ready: the fix is in the right hybrid-fusion boundary with strong proof for the original vector-only media bug, but the current guard also suppresses real keyword scoring when classified media appears in both vector and FTS results.

## What This Changes

The PR makes hybrid memory fusion modality-aware for non-text media, wires the memory manager to the existing multimodal path classifier, and adds regression tests for vector-only, keyword-only, text-only, and zero-vector-weight cases.

## Best Possible Solution

Keep the modality-aware normalization at the existing hybrid merge boundary, but discard the media text weight only when there is no keyword/text signal to preserve and retain focused coverage for both-signal media.

## Reproduction Assessment

Yes. Current main source shows vector-only media is discounted because textScore is 0 and the unconditional hybrid formula still applies the text weight; the PR-introduced both-signals edge is also source-reproducible from the new guard and production caller.

## Solution Assessment

No. The chosen boundary is right, but the current condition is too broad because it also removes BM25 relevance for classified media that actually matched FTS; preserving textScore when present is the narrower fix.

- **[P2] Preserve keyword score when media also matches FTS:** `extensions/memory-core/src/memory/hybrid.ts:126-129`
  - body: The guard drops the BM25/text signal for every classified media item that has any vector hit. The existing flow indexes media labels and has a manager test expecting `search("image")`/`search("audio")` to find media, so only discard the text weight when there is no keyword signal to preserve, or add equivalent coverage for the both-signals case.
