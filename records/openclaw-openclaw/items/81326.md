## Summary

Keep open: current main still does not surface the missing configured Codex owner plugin in `plugins doctor`, and this issue is already paired with the open fixing PR at https://github.com/openclaw/openclaw/pull/81674. A newer startup-install PR at https://github.com/openclaw/openclaw/pull/82571 may cover more of the runtime failure, but it is still open and under review, so this issue should close only after a fix lands.

## What This Changes

The issue reports that Codex runtime config can remain active without the Codex plugin installed or enabled, leading to `Requested agent harness "codex" is not registered` and weak doctor diagnostics.

## Best Possible Solution

Land a policy-respecting diagnostic or startup repair that ensures Codex runtime routes have an enabled owner plugin before user traffic, then close this issue as implemented by the merged PR.

## Reproduction Assessment

Yes, source-reproducible. Current main still has a `plugins doctor` no-issues path without configured Codex owner-plugin checks, and the harness selector still throws when an explicit plugin runtime is selected but no matching harness is registered; I did not run a live CLI repro in this read-only review.

## Solution Assessment

Yes for the doctor-warning direction. The open diagnostic PR matches maintainer guidance and is narrower than hard-failing startup; the broader startup-install PR may also help but needs its own review to avoid unsafe install side effects.
