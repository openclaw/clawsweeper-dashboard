## Summary

Keep open. Current main still routes fallback retry marker/prelude text through the embedded runner's visible prompt path, so the transcript-facing user turn is not guaranteed to remain the original body; an overlapping open PR appears to be the active implementation candidate but is not merged or linked with closing syntax.

## What This Changes

The issue asks OpenClaw to preserve the original user prompt as the visible transcript body during model fallback retries instead of persisting synthetic retry text as user-authored content.

## Best Possible Solution

Keep the fallback model's retry/prelude context available through hidden runtime context, lifecycle metadata, or carefully scoped duplicate-persistence suppression, while ensuring the persisted visible user turn remains the original body. Coordinate with #63696 before starting a replacement so the repository does not grow two conflicting fixes for the same fallback transcript boundary.

## Reproduction Assessment

Yes. The current helper and tests deterministically show the marker being prepended for fallback retries with history or a prelude, and the embedded attempt call path forwards that value as the submitted prompt without transcriptPrompt separation.

## Solution Assessment

No, not as the issue's helper-only proposal. The better fix preserves fallback context for the model but separates or suppresses it from user-authored transcript storage; #63696 looks like a plausible active candidate but needs maintainer validation against this exact issue.
