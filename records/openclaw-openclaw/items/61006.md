## Summary

Keep open. Current main still sends fallback retry marker/prelude text through the embedded runner's visible prompt path, and the overlapping PR that may suppress the duplicate transcript write is open rather than merged.

## What This Changes

The issue asks OpenClaw to preserve the original user-authored prompt in persisted fallback retry transcript turns instead of storing synthetic retry text.

## Best Possible Solution

Review and land #63696 if it keeps the persisted visible user turn as the original body in all fallback cases; otherwise implement the same boundary with `transcriptPrompt` or hidden runtime context so retry/prelude text reaches the model without becoming user-authored transcript content.

## Reproduction Assessment

Yes. Source inspection reproduces the path deterministically: `resolveFallbackRetryPrompt` returns marker-prefixed text for fallback retries with history/prelude, and `runAgentAttempt` passes that value as the embedded runner's visible prompt without `transcriptPrompt`.

## Solution Assessment

No, not as the helper-only proposal. Returning only `body` would remove useful retry/prelude context from the model; the safer fix is to separate model-only fallback context from transcript persistence or suppress only the duplicate retry user write.
