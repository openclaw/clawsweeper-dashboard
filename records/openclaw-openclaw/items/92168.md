## Summary

Keep open: current main still has the reported resolver divergence in `/btw`, while the documented/current agent runtime contract already says canonical Anthropic model refs may use `agentRuntime.id: "claude-cli"`. ## What This Changes

Make `/btw` honor existing model-scoped `agentRuntime.id: "claude-cli"` canonical aliases instead of throwing `Unknown model`.

## Best Possible Solution

Align BTW model resolution with the main agent loop's existing runtime-aware resolver for `agentRuntime` policies, with a regression test for canonical Anthropic aliases routed through `claude-cli` and no new config syntax.

## Reproduction Assessment

## Solution Assessment
