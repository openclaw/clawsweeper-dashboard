## Summary

Keep open: the PR targets a real doctor false positive, but the proposed skip is too broad for channel configs where root credentials still create an active default account alongside named accounts, and the external real-behavior proof gate is still mock-only.

## What This Changes

The branch changes the shared doctor empty-allowlist scanner to skip top-level channel warnings whenever enabled account records exist, with regression tests for the parent-fallback false-positive case.

## Best Possible Solution

Preserve the phantom-parent fix only for true parent fallback records, keep diagnostics for active root/default accounts, add a regression for root credentials plus named accounts, and include redacted real `openclaw doctor` output before merge.

## Reproduction Assessment

Yes. Current main reproduces the linked false positive from source because the scanner checks the top-level channel record before accounts; the PR regression is also source-reproducible because the new account-entry condition skips that top-level scan whenever any enabled account exists.

## Solution Assessment

No. The direction is plausible for pure parent fallback configs, but the implementation is not the best fix because existing account helpers allow a real implicit default account to coexist with named accounts.
