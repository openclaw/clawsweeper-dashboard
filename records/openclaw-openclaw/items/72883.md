Keep open: current main still rejects the reported safe `diagnostics.cacheTrace` patch, and the active implementation path is the open, unmerged closing PR at https://github.com/openclaw/openclaw/pull/73098.

## What This Changes

Allow gateway `config.patch` to enable `diagnostics.cacheTrace` only when cache-trace content capture remains disabled.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73098, or an equivalent narrow guard that permits cache-trace decision observability only when all content-capture fields resolve disabled.

## Reproduction Assessment

Yes. Source inspection on current main shows the reported patch changes `diagnostics.cacheTrace` leaves that are not allowlisted, so the guard rejects them before forwarding `config.patch`.

## Solution Assessment

Yes. The conditional-safe guard direction is the narrowest maintainable fix; unconditionally allowing the subtree would weaken the existing content guard because payload-capture flags default to true.
