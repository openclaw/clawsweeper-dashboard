Keep this issue open: current main still blocks the reported safe `diagnostics.cacheTrace` patch, and the linked implementation path at https://github.com/openclaw/openclaw/pull/73098 is open and unmerged with closing syntax.

## What This Changes

Allow `gateway config.patch` and `config.apply` to enable `diagnostics.cacheTrace` only when message, prompt, and system content capture are disabled.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73098, or an equivalent narrow guard that permits cache-trace decision observability only when all content-capture fields resolve disabled.

## Reproduction Assessment

Yes. Source inspection on current main shows the reported patch changes `diagnostics.cacheTrace` leaves that are not allowlisted, so the guard rejects them before forwarding `config.patch`.

## Solution Assessment

Yes. The conditional-safe direction is the narrowest maintainable fix; unconditionally allowing the subtree would weaken the existing content guard because payload-capture flags default to true.
