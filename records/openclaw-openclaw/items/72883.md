## Summary

Keep this issue open: current `main` still rejects the reported safe `diagnostics.cacheTrace` patch, and open PR #73098 remains the paired implementation path with closing syntax rather than a reason to close early.

## What This Changes

Allow the gateway config mutation tool to enable cache-trace decision diagnostics while sensitive content capture remains disabled.

## Best Possible Solution

Review and land PR #73098, or an equivalent narrow guard that allows cache-trace decision observability only when all content-capture flags resolve disabled.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: the reported patch changes `diagnostics.cacheTrace` leaves, those leaves are not allowlisted, and the agent-facing `config.patch` branch rejects them before applying the gateway write.

## Solution Assessment

Yes. The conditional-safe direction is the narrowest maintainable fix because unconditionally allowing the subtree would weaken the existing content guard while the payload-capture flags default to true.
