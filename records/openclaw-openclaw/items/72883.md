## Summary

Keep this issue open. Current main still rejects the reported safe `diagnostics.cacheTrace` patch through the agent-facing gateway tool, and PR #73098 is open and unmerged with closing syntax for this issue.

## What This Changes

The issue requests a supported way for `gateway config.patch` to enable `diagnostics.cacheTrace` safely when sensitive content capture flags are disabled.

## Best Possible Solution

Review and land PR #73098, or an equivalent narrow change, so cache tracing can be enabled only when the resulting cache-trace configuration keeps all content capture flags disabled.

## Reproduction Assessment

Yes. Static inspection of current main is enough: the reported patch changes `diagnostics.cacheTrace` leaf paths, those paths are absent from the gateway tool allowlist, and the guard rejects non-allowlisted changes before forwarding `config.patch`.

## Solution Assessment

Yes, with the conditional-safe shape. The maintainable fix is to allow cache-trace enablement only when the resulting config has `includeMessages`, `includePrompt`, and `includeSystem` all false; unconditionally allowing the subtree would weaken the current content guard.
