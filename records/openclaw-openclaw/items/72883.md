## Summary

Keep open: current main still rejects the reported safe `diagnostics.cacheTrace` patch, and the linked closing implementation remains open and unmerged at https://github.com/openclaw/openclaw/pull/73098.

## What This Changes

The issue requests that agent-facing `gateway config.patch` allow cache-trace diagnostics only when message, prompt, and system content capture are disabled.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73098, or an equivalent narrow guard that permits cache-trace decision observability only when content-capture fields resolve disabled.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes. Source inspection on current main shows the reported patch changes `diagnostics.cacheTrace` leaves that are not allowlisted, so the guard rejects them before forwarding `config.patch`.

## Solution Assessment

Is this the best way to solve the issue? Yes. The conditional-safe guard direction is the narrowest maintainable fix; unconditionally allowing the subtree would weaken the existing content guard because payload-capture flags default to true.
