## Summary

Keep this issue open: current main still rejects the reported safe `diagnostics.cacheTrace` patch, and the open closing PR is the active implementation path rather than a reason to close early.

## What This Changes

Allow the agent-facing gateway `config.patch` path to enable cache-trace observability when message, prompt, and system content capture stay disabled.

## Best Possible Solution

Review and land PR #73098, or an equivalent narrow conditional guard that permits cache-trace decision observability only when all content capture flags resolve disabled.

## Reproduction Assessment

Yes. Static inspection is high-confidence: the reported patch changes `diagnostics.cacheTrace` leaves, those leaves are not allowlisted, and the agent-facing `config.patch` branch rejects them before forwarding the write.

## Solution Assessment

Yes, the conditional-safe direction is the best fit. Unconditionally allowing the subtree would weaken the existing content guard because the payload-capture flags default to true.
