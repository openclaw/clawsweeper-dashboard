## Summary

Keep this open: current main still blocks the reported safe `diagnostics.cacheTrace` patch, and PR #73098 is the open, unmerged closing implementation path.

## What This Changes

Allow the agent-facing gateway config mutation path to enable `diagnostics.cacheTrace` decision tracing when prompt, system, and message content capture remain disabled.

## Best Possible Solution

Review and land PR #73098, or an equivalent narrow guard that permits cache-trace observability only when the resulting `includeMessages`, `includePrompt`, and `includeSystem` settings are all disabled.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: the reported patch changes `diagnostics.cacheTrace` leaves, `config.patch` runs the mutation guard before RPC forwarding, and those leaves are absent from the allowlist on current main.

## Solution Assessment

Yes, the conditional-safe direction is the best fit. Unconditionally allowing the subtree would weaken the existing content guard because cache-trace payload-capture flags default to enabled.
