## Summary

Keep open: current main still rejects the reported safe cache-trace patch, and the open closing PR is the active implementation path rather than a reason to close early.

## What This Changes

Allow `gateway config.patch` to enable `diagnostics.cacheTrace` decision tracing when `includeMessages`, `includePrompt`, and `includeSystem` are all false.

## Best Possible Solution

Review and land PR #73098, or an equivalent narrow guard that allows cache-trace observability only when the resulting content-capture flags are disabled.

## Reproduction Assessment

Yes. Static current-main inspection is high-confidence: the reported patch changes `diagnostics.cacheTrace` leaves, those leaves are not allowlisted, and the agent-facing `config.patch` path rejects them before forwarding the write.

## Solution Assessment

Yes, the conditional-safe direction is the best fit. Unconditionally allowlisting the subtree would weaken the existing model/operator trust boundary because cache-trace content capture defaults to true.
