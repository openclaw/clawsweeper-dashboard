## Summary

Keep this issue open: current main still rejects the safe `diagnostics.cacheTrace` patch, and PR #73098 is the active unmerged implementation path.

## What This Changes

Allow the agent-facing gateway config mutation tool to enable `diagnostics.cacheTrace` when message, prompt, and system content capture remain disabled.

## Best Possible Solution

Review and land PR #73098 or an equivalent narrow guard that permits cache-trace observability only when content capture remains off in the resulting config.

## Reproduction Assessment

Yes. Static inspection of current main is high-confidence: the reported patch changes non-allowlisted `diagnostics.cacheTrace` leaves, and the guard rejects those leaves before forwarding `config.patch`.

## Solution Assessment

Yes. The conditional-safe direction is the narrowest maintainable fix; allowing the whole subtree unconditionally would weaken the current content-capture protection.
