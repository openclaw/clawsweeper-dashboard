## Summary

Keep open: current main still rejects the reported safe `diagnostics.cacheTrace` patch, and #73098 is the open implementation path with closing syntax rather than a reason to close this issue early.

## What This Changes

Allow the agent-facing gateway `config.patch` path to enable `diagnostics.cacheTrace` when sensitive content capture remains disabled.

## Best Possible Solution

Add a narrow guard that permits decision-only cache tracing when all content-capture flags resolve disabled, while keeping payload capture protected.

## Reproduction Assessment

Yes. Source inspection is high-confidence: the reported patch changes `diagnostics.cacheTrace` leaves, those leaves are not allowlisted, and the agent-facing `config.patch` branch rejects them before the gateway write.

## Solution Assessment

Yes. The conditional-safe direction is the narrowest maintainable fix; unconditionally allowing the subtree would weaken the existing guard because the payload-capture flags default to true.
