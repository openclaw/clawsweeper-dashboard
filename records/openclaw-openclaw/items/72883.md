## Summary

Keep open: current main still rejects the reported safe `diagnostics.cacheTrace` patch, and PR #73098 is the open closing implementation path rather than a reason to close early.

## What This Changes

The issue asks for `gateway config.patch` to permit cache-trace decision tracing when message, prompt, and system content capture are disabled.

## Best Possible Solution

Review and land PR #73098, or an equivalent narrow guard that permits cache-trace decision observability only when content capture resolves disabled.

## Reproduction Assessment

Yes. Source inspection shows the reported patch changes `diagnostics.cacheTrace` leaves, those leaves are not allowlisted, and the agent-facing `config.patch` path rejects them before the gateway write.

## Solution Assessment

Yes. The conditional-safe direction is the narrowest maintainable fix; unconditionally allowing the subtree would weaken the existing content guard.
