## Summary

Keep this open. Current main still has no cache-trace exception in the agent-facing gateway config mutation guard, and PR #73098 is open and unmerged with closing syntax for this issue, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Allow the gateway config patch/apply tool path to enable `diagnostics.cacheTrace` when raw message, prompt, and system-prompt capture remain disabled.

## Best Possible Solution

Review and land PR #73098, or an equivalent narrow change, so the gateway tool permits cache-trace enablement only when the resulting diagnostics config has all raw-content capture disabled, with regression coverage for safe enablement and unsafe attempts.

## Reproduction Assessment

Yes. Static inspection of current main is enough: the reported patch changes `diagnostics.cacheTrace` leaves, those leaves are not allowlisted, and the guard rejects non-allowlisted changed paths before calling the gateway write.

## Solution Assessment

Yes. A conditional allow based on the resulting `cacheTrace` config is the narrowest maintainable fix; unconditionally allowing the subtree would be less safe because the content-capture fields default to `true`.
