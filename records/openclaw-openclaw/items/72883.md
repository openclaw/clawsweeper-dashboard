## Summary

Keep this open. Current main still rejects the reported safe `diagnostics.cacheTrace` patch through the agent-facing gateway mutation guard, and the linked fixing PR is open and unmerged with closing syntax, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Permit `gateway config.patch`/`config.apply` to enable cache-trace diagnostics only when raw message, prompt, and system capture remain disabled.

## Best Possible Solution

Review and land PR #73098, or an equivalent narrow change, so the gateway tool permits cache-trace enablement only when the resulting diagnostics config has all raw-content capture disabled, with regression coverage for safe enablement and unsafe attempts.

## Reproduction Assessment

Yes. Static inspection is enough: the reported patch changes non-allowlisted `diagnostics.cacheTrace` leaves, and the guard rejects those leaves before the gateway write path runs.

## Solution Assessment

Yes, if implemented as a conditional allow based on the resulting cache-trace config. Unconditionally permitting the subtree would be less safe because the content-capture fields default to enabled.
