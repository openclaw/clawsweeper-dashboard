## Summary

Keep this issue open: current main still rejects the reported safe `diagnostics.cacheTrace` patch, and open PR #73098 is the active closing implementation path.

## What This Changes

The issue requests a supported gateway-tool path to enable cache-trace observability while keeping message, prompt, and system content capture disabled.

## Best Possible Solution

Review and land PR #73098 or an equivalent narrow conditional guard that permits cache-trace enablement only when the resulting config keeps all content capture flags disabled.

## Reproduction Assessment

Yes. Static inspection is high-confidence: the reported patch changes `diagnostics.cacheTrace.*` leaves, no allowlist pattern matches them, and the guard throws before forwarding `config.patch`.

## Solution Assessment

Yes. The conditional-safe direction is the narrow maintainable fix; unconditionally allowing the cache-trace subtree would weaken the current sensitive-content protection.
