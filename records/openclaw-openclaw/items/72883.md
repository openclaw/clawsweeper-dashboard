## Summary

Keep this issue open. Current main still blocks the reported safe cache-trace `config.patch`, and PR #73098 is open with closing syntax for this issue.

## What This Changes

Allow the agent-facing gateway config tool to enable `diagnostics.cacheTrace` when the resulting config keeps message, prompt, and system content capture disabled.

## Best Possible Solution

Review and land PR #73098, or an equivalent narrow guard change, so cache tracing can be enabled only when the resulting cache-trace config keeps all content capture flags disabled.

## Reproduction Assessment

Yes. Static inspection of current main is enough: the reported patch changes `diagnostics.cacheTrace` leaves, those leaves are not allowlisted, and the gateway tool rejects disallowed changes before forwarding `config.patch`.

## Solution Assessment

Yes, the conditional-safe approach is the best direction. Unconditionally allowing the subtree would weaken the current sensitive-content guard, while a result-state check can allow observability without enabling payload capture.
