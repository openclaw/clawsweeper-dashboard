## Summary

Keep this issue open: current main still rejects the safe `diagnostics.cacheTrace` patch through the agent-facing gateway tool, and the unmerged linked PR is the active implementation path.

## What This Changes

Allow `gateway config.patch`/`config.apply` to enable cache tracing only when message, prompt, and system capture stay disabled.

## Best Possible Solution

Land PR #73098 or an equivalent narrow conditional guard that permits cache-trace observability only when all content capture flags remain disabled.

## Reproduction Assessment

Yes. Static inspection of current main is high-confidence: the reported patch changes `diagnostics.cacheTrace` leaves, those leaves are not allowlisted, and the guard rejects them before forwarding `config.patch`.

## Solution Assessment

Yes, with the conditional-safe shape. Unconditionally allowing the subtree would weaken the current content guard because the payload-capture flags default to true.
