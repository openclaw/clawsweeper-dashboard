## Summary

Keep open: the PR is useful and has positive live proof, but the current patch can suppress a configured Gateway SecretRef warning when a fallback environment token authenticates the status probe.

## What This Changes

The branch adds a doctor-only Gateway status probe before unresolved `gateway.auth.token` SecretRef warnings and adds unit coverage for inline-token and healthy-gateway suppression paths.

## Best Possible Solution

Keep suppression only when the check proves the configured SecretRef/runtime credential path is healthy, or explicitly approve and test a fallback-authenticated suppression policy.

## Reproduction Assessment

Yes, at source level. The PR resolves the configured SecretRef with fallback disabled, then calls generic `callGateway`, which can authenticate from `process.env` with env-first local token precedence.

## Solution Assessment

No. The branch is close, but the safer fix needs to preserve the configured SecretRef warning unless maintainers explicitly approve fallback-authenticated status as sufficient.
