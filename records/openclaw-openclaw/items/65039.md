## Summary

Keep open. Current main and the latest shipped release still pass `daveEncryption=false` through to `@discordjs/voice`, wait for `Ready`, and surface a pre-Ready abort as the same failed `/vc join`; the unresolved part is the intended contract for non-DAVE Discord voice joins.

## What This Changes

The issue reports Discord `/vc join` aborting after Hello and before Ready when Discord voice is configured with `daveEncryption=false`.

## Best Possible Solution

Make the Discord voice join contract explicit: either support DAVE-disabled joins through Ready with live coverage, or reject `daveEncryption=false` before join with a clear diagnostic and docs.

## Reproduction Assessment

Yes, with source-level confidence. The issue includes live repro steps and a detailed trace, and current main still has a pre-Ready `entersState()` rejection path that returns the same `/vc join` failure while `daveEncryption=false` remains accepted config.

## Solution Assessment

Unclear. A narrow code change is possible, but the maintainable fix depends on whether maintainers want non-DAVE Discord voice joins to be supported or to fail early as unsupported because Discord may require DAVE.
