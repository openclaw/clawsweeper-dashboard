## Summary

Keep open: current main preserves pre-compaction checkpoints, but the model-facing `sessions_history` path still reads only the active successor transcript and has no checkpoint/archive opt-in.

## What This Changes

Request to let `sessions_history` recover pre-compaction rotated transcripts when `truncateAfterCompaction` leaves the active successor transcript nearly empty.

## Best Possible Solution

Add an explicit, redacted checkpoint/archive history path that reuses current session visibility guards and compaction checkpoint metadata, with tests proving active-only default behavior and opt-in archived recall.

## Reproduction Assessment

Yes, at source level. Create a session entry with an active successor transcript plus a pre-compaction checkpoint, call `sessions_history`, and current main will route through `chat.history` to only the active `entry.sessionFile`.

## Solution Assessment

Unclear. The explicit opt-in direction is safer than implicit fallback, but maintainers should choose whether this belongs in `sessions_history`, `chat.history`, or a separate checkpoint recall API/tool.
