## Summary

Keep open: current main still routes sessions_history through active-transcript chat.history only, while pre-compaction checkpoint transcripts are stored on a separate recovery surface and not model-facing history.

## What This Changes

The issue requests an explicit or fallback way for sessions_history to include rotated pre-compaction transcripts when truncateAfterCompaction leaves the active transcript short.

## Best Possible Solution

Add an explicit, redacted checkpoint-history recall path that reuses existing session visibility guards, compaction checkpoint metadata, bounded reads, docs, and tests instead of silently changing active-branch history semantics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create a session with an active successor transcript plus a pre-compaction checkpoint, call sessions_history, and the tool will request chat.history for only the active transcript.

## Solution Assessment

Unclear. An explicit opt-in appears safer than automatic fallback, but maintainers should decide whether this belongs in sessions_history, chat.history, or a separate checkpoint recall tool.
