## Summary

Keep open: current main preserves compaction checkpoints and exposes operator/UI recovery controls, but the model-facing `sessions_history` path still reads only the active successor transcript and has no archived/checkpoint opt-in.

## What This Changes

Request a `sessions_history` or SDK mode that can include pre-rotation transcript data after `truncateAfterCompaction` rotates the active JSONL.

## Best Possible Solution

Add an explicit, redacted checkpoint/archive history path that reuses current session visibility guards and checkpoint metadata instead of silently changing active-branch history semantics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: create a session with an active successor transcript plus a pre-compaction checkpoint, call `sessions_history`, and it will only request `chat.history` for the active transcript.

## Solution Assessment

Unclear. The requested explicit opt-in direction is likely safer than implicit fallback, but maintainers should decide whether this belongs in `sessions_history`, a separate checkpoint recall tool, or operator-only checkpoint controls.
