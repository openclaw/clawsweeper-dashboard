## Summary

Keep open: current main still has no model-facing way for `sessions_history` to request pre-compaction checkpoint content, even though operator checkpoint list/branch/restore controls exist.

## What This Changes

Request an archived or checkpoint transcript recall path for `sessions_history` after compaction rotates the active transcript.

## Best Possible Solution

Add an explicit, permission-checked checkpoint recall path that reuses current `sessions_history` redaction and session visibility guards instead of silently changing active-branch history semantics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create a session with an active successor transcript plus a pre-compaction checkpoint, call `sessions_history`, and it can only request active `chat.history`.

## Solution Assessment

Unclear. Explicit opt-in is safer than implicit fallback, but maintainers should choose whether this belongs in `sessions_history`, a separate checkpoint recall tool, or operator-only checkpoint controls.
