## Summary

Keep open: current main still exposes only the active successor transcript through `sessions_history`/`chat.history`; pre-compaction checkpoint transcripts exist and are recoverable through separate operator checkpoint paths, but no model-facing archived-history contract exists yet.

## What This Changes

Request that `sessions_history` optionally recover pre-rotation transcript content after `truncateAfterCompaction: true` leaves a short active successor transcript.

## Best Possible Solution

Add an explicit, redacted checkpoint/archive history retrieval path that reuses current session visibility guards, with maintainers deciding whether it belongs in `sessions_history` or a separate checkpoint recall surface.

## Reproduction Assessment

Yes, at source level. A session with a current active successor transcript plus a stored `preCompaction.sessionFile` checkpoint will have `sessions_history` call `chat.history`, which reads only the active `entry.sessionFile` path.

## Solution Assessment

Unclear. The explicit opt-in direction is likely safer than implicit fallback, but maintainers should choose the archive/checkpoint API and privacy semantics before implementation.
