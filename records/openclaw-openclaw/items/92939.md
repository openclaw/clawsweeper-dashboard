## Summary

Keep open: the PR is a useful direction for the linked ACP workspace bug, but it is not merge-ready because it can skip intentionally configured workspaces and only covers one of the ACP-reachable workspace setup paths.

## What This Changes

The PR adds a `runtimeManaged` workspace flag, exposes `runtime` on resolved agent config, and passes the flag from `getReplyFromConfig` for non-embedded runtimes.

## Best Possible Solution

Centralize the runtime-managed workspace decision so only non-embedded agents without an explicit workspace skip default scaffolding, and apply that policy consistently across reply, agent-command, and cron entry points.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction for the remaining PR defects: explicit ACP workspaces would still pass `runtimeManaged: true`, while `agentCommand` and isolated cron ACP paths still scaffold through the old call.

## Solution Assessment

No. Skipping default scaffolding is the right direction, but the best fix needs an explicit-workspace gate and a shared entry-point policy rather than only changing `getReplyFromConfig`.
