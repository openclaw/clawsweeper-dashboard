## Summary

Keep open. The PR targets a valid bug, but the current patch is not merge-ready because it skips workspace setup for all non-embedded runtime agents, including agents with an explicit workspace, and it only changes one caller while other ACP-reachable workspace setup paths remain unresolved.

## What This Changes

The PR adds a `runtimeManaged` skip to `ensureAgentWorkspace`, exposes runtime metadata on resolved agent config, and passes the skip from the auto-reply path for non-embedded runtime agents.

## Best Possible Solution

Gate the skip on both a selected non-embedded runtime and absence of the selected agent's explicit workspace, then place the predicate at a shared workspace-decision boundary or add sibling-path proof for every ACP-reachable caller.

## Reproduction Assessment

Yes for the PR defect by source inspection: an ACP/non-embedded agent with an explicit `agents.list[].workspace` would still pass `runtimeManaged: true` and skip setup. I did not run the ACP startup scenario in this read-only review.

## Solution Assessment

No. Skipping implicit runtime-managed workspace scaffolding is the right direction, but this implementation is too narrow and needs the explicit-workspace gate plus sibling caller coverage before merge.
