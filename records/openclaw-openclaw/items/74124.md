## Summary

Keep open. Current main still lacks an intentional global/shared skill install target: the CLI exposes only workspace/agent targeting, the ClawHub installer resolves under `workspaceDir/skills`, and #74466 is an open, unmerged closing PR for the requested behavior.

## What This Changes

Add an explicit global/shared install target for `openclaw skills install` so ClawHub skills can be installed under shared roots such as `~/.openclaw/skills` instead of only an agent workspace.

## Best Possible Solution

Land #74466 or an equivalent narrow implementation that adds an opt-in shared/global target, preserves the current workspace default, keeps `--global` and `--agent` mutually exclusive, scopes tracking metadata to the selected root, and updates tests/docs/changelog.

## Reproduction Assessment

Yes. A high-confidence read-only reproduction exists on current main: command registration has no global/target option, and the installer resolves every ClawHub skill target under `workspaceDir/skills`.

## Solution Assessment

Yes. An explicit opt-in global/shared target is the narrowest maintainable solution; changing the default workspace behavior would be broader and could surprise existing multi-agent setups.
