## Summary

Keep open. Current main still exposes only workspace/agent targets for ClawHub skill install/update, and #74466 is an open, unmerged closing PR carrying the candidate implementation.

## What This Changes

Request an explicit global/shared install target for `openclaw skills install` and related update flows so ClawHub skills can be installed outside an agent workspace.

## Best Possible Solution

Review and land #74466 or an equivalent narrow CLI/docs/test/changelog implementation that adds an opt-in shared target while preserving the workspace default.

## Reproduction Assessment

Yes. Read-only source inspection on current main shows `skills install`/`update` accept workspace/agent routing but no `--global` or `--target`, and the installer derives target directories from `<workspaceDir>/skills`.

## Solution Assessment

Yes. An explicit opt-in global/shared target is the narrowest maintainable direction; changing the default workspace behavior would be broader and could surprise existing multi-agent setups.
