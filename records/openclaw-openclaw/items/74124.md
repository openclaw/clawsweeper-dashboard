## Summary

Keep open. Current main still lacks an explicit global/shared install target for ClawHub skills: the CLI exposes workspace/agent targeting only, and the installer still derives targets from the selected workspace. The supplied GitHub context also shows open closing PR #74466, so this issue should remain open until that PR or an equivalent replacement lands.

## What This Changes

Add an explicit global/shared target for `openclaw skills install` so users can install ClawHub skills into `~/.openclaw/skills` while preserving the existing workspace-scoped default.

## Best Possible Solution

Land #74466 or an equivalent narrow replacement that adds an opt-in shared/global target for install and update, preserves existing workspace defaults, keeps ClawHub origin/lock metadata scoped to the selected root, documents precedence and conflicts, and adds focused CLI tests.

## Reproduction Assessment

Yes. A high-confidence read-only reproduction is available from current main: `skills install` and `skills update` only resolve an agent workspace, and the installer appends `skills/<slug>` beneath that workspace.

## Solution Assessment

Yes. An explicit opt-in global/shared target is the narrowest maintainable direction; changing the default install location would be broader and could surprise existing multi-agent setups.
