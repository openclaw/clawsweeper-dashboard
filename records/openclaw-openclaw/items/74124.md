## Summary

Keep open. Current main still has only active-workspace ClawHub skill install/update targeting, and the provided GitHub context shows open PR #74466 already paired with this issue but not merged.

## What This Changes

Add an explicit global/shared target for `openclaw skills install`, and possibly `skills update`, so ClawHub skills can be installed into shared skill roots instead of only an agent workspace.

## Best Possible Solution

Land #74466 or an equivalent narrow replacement that adds an opt-in shared/global target, preserves existing workspace defaults, keeps ClawHub tracking metadata scoped to the selected root, documents precedence, and covers CLI conflict behavior with focused tests.

## Reproduction Assessment

Yes. A high-confidence read-only reproduction exists on current main: command registration has no global/target option, and the ClawHub installer resolves targets under `workspaceDir/skills`.

## Solution Assessment

Yes. An explicit opt-in global/shared target is the narrowest maintainable direction; changing the default workspace behavior would be broader and could surprise existing multi-agent setups.
