## Summary

Keep open. Current main still lacks an intentional global/shared skill install target: the CLI and ClawHub installer remain workspace-targeted, and #74466 is an open, unmerged closing PR for this exact request.

## What This Changes

Add an explicit global/shared target for `openclaw skills install` and related update flow so ClawHub skills can be installed under a shared skills directory instead of only the active workspace.

## Best Possible Solution

Land #74466 or an equivalent narrow implementation that adds an opt-in shared/global target, preserves the current workspace default, keeps `--global` and `--agent` mutually exclusive, and updates tests, docs, and changelog.

## Reproduction Assessment

Yes. A high-confidence read-only reproduction exists on current main: command registration has no global target, and the installer resolves every target below `workspaceDir/skills`.

## Solution Assessment

Yes. An explicit opt-in global/shared target is the narrowest maintainable direction; changing the default install scope would be broader and could surprise existing multi-agent setups.
