## Summary

Keep open. Current main still lacks an intentional global/shared skill install target, and #74466 is an open, unmerged closing PR for the same behavior, so this issue should remain the canonical tracker until that work lands or is replaced.

## What This Changes

The issue requests an explicit global/shared target for `openclaw skills install`, such as `--global`, while preserving the current workspace-scoped default.

## Best Possible Solution

Land #74466 or an equivalent narrow implementation that adds an opt-in shared/global target, preserves the current workspace default, handles `--global` and `--agent` conflicts, scopes tracking metadata to the selected root, and updates tests/docs/changelog.

## Reproduction Assessment

Yes. A high-confidence read-only reproduction exists on current main: command registration lacks `--global` or `--target`, and the installer resolves every ClawHub skill under `workspaceDir/skills`.

## Solution Assessment

Yes. An explicit opt-in global/shared target is the narrowest maintainable direction; changing the default workspace behavior would be broader and could surprise existing multi-agent setups.
