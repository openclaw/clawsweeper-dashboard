## Summary

Keep open. Current main still lacks an explicit shared/global install target for ClawHub skills, and the issue is paired with open implementation PR #74466, so this should stay open until that PR or an equivalent replacement lands.

## What This Changes

Request an explicit global/shared target for `openclaw skills install` so ClawHub skills can be installed into `~/.openclaw/skills` or another shared skill root instead of only an agent workspace.

## Best Possible Solution

Land #74466 or an equivalent narrow replacement that adds an opt-in shared/global target, preserves the current workspace default, keeps ClawHub origin and lock metadata scoped to the selected root, documents precedence, and adds focused CLI tests for conflict handling and update behavior.

## Reproduction Assessment

Yes. A high-confidence read-only reproduction is available from current main: command registration has no global/target option, and the installer resolves every ClawHub skill install under `path.resolve(workspaceDir)/skills`.

## Solution Assessment

Yes. An explicit opt-in shared/global target is the narrowest maintainable solution; changing the default workspace behavior would be broader and could surprise existing multi-agent setups.
