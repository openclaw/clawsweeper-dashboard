## Summary

Keep open. Current main still has workspace/agent-only ClawHub skill install and update paths, and #74466 is an open, unmerged closing PR carrying the requested global-target implementation.

## What This Changes

Add an explicit CLI target, such as `--global`, so `openclaw skills install` can intentionally install shared skills outside the active agent workspace.

## Best Possible Solution

Review and land #74466, or an equivalent narrow CLI/docs/tests/changelog implementation that adds an opt-in shared target while preserving the workspace default.

## Reproduction Assessment

Not applicable as a feature request, but source inspection gives high-confidence current-main verification: `skills install` and `skills update` expose only workspace/agent options while the installer derives targets from `<workspaceDir>/skills`.

## Solution Assessment

Yes. An explicit opt-in global/shared target is the narrowest maintainable direction; changing the default workspace target would be broader and could surprise existing multi-agent setups.
