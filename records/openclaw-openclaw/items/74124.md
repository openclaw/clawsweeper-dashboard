## Summary

Keep open. Current main still has only workspace-scoped ClawHub skill install/update routing, while #74466 is an open, unmerged closing PR for the requested global/shared target.

## What This Changes

Add an explicit global/shared target for `openclaw skills install` and related update behavior while preserving the current workspace default.

## Best Possible Solution

Land #74466 or an equivalent narrow implementation that adds an opt-in shared target, keeps workspace installs as the default, rejects conflicting target options, stores tracking metadata under the selected root, and updates tests/docs/changelog.

## Reproduction Assessment

Yes. A high-confidence read-only reproduction exists: current command registration lacks a global target, and the installer always resolves ClawHub skill installs through the selected workspace directory.

## Solution Assessment

Yes. An explicit opt-in global/shared target is the narrowest maintainable direction; changing the default workspace behavior would be broader and could surprise existing multi-agent setups.
