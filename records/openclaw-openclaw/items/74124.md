## Summary

Keep open. Current main still lacks an intentional global/shared skill install target, and #74466 is an open, unmerged closing PR for the same behavior, so this issue should remain the tracker until that work lands or is replaced.

## What This Changes

Add an explicit global/shared target for `openclaw skills install` so ClawHub skills can be installed under the shared skills root instead of only the active workspace.

## Best Possible Solution

Review and land #74466 or an equivalent narrow implementation that adds an opt-in shared/global target while preserving the current workspace default and updating tests, docs, and changelog.

## Reproduction Assessment

Yes. A high-confidence read-only reproduction exists on current main: command registration lacks `--global`/`--target`, and the ClawHub installer derives targets from `workspaceDir/skills`.

## Solution Assessment

Yes. An explicit opt-in global/shared target is the narrowest maintainable direction; changing the default workspace behavior would be broader and could surprise existing multi-agent setups.
