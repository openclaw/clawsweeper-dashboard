## Summary

Keep open. Current main still lacks an intentional global/shared target for `openclaw skills install` or `update`, and the linked closing PR #74466 is open and unmerged, so this issue should remain open until that implementation or an equivalent one lands.

## What This Changes

Add an explicit global/shared install target for ClawHub skill install/update commands while preserving the existing workspace-scoped default.

## Best Possible Solution

Review and land #74466 or an equivalent narrow CLI/docs/test/changelog change that adds an opt-in shared target while preserving workspace-scoped defaults.

## Reproduction Assessment

Yes. Source inspection on current main shows `skills install` and `skills update` expose only workspace/agent routing and the installer derives targets from `<workspaceDir>/skills`.

## Solution Assessment

Yes. An explicit opt-in global/shared target is the narrow maintainable direction; changing the default workspace behavior would be broader and could surprise existing multi-agent setups.
