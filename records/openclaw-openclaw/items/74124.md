## What This Changes

The issue requests a new `openclaw skills install`/`update` global target, such as `--global`, for installing shared skills under the global managed skills directory.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/74466, or an equivalent narrow CLI/docs/test change that preserves workspace defaults and adds an explicit shared target.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence in the gap because current main lacks a global install/update target while the shared skill root is already loaded.

## Solution Assessment

Yes, pending maintainer approval of the CLI surface. An explicit opt-in shared target is the narrowest maintainable direction because it preserves existing workspace behavior and avoids changing default multi-agent install semantics.
