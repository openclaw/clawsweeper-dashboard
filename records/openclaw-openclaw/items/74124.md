## Summary

Keep open. Current main still only supports active-workspace ClawHub skill install/update targeting, while shared skill roots are already loadable; the linked closing PR is open and unmerged, so this issue should remain the tracker until that PR or an equivalent fix lands.

## What This Changes

The issue requests an explicit global/shared install target for `openclaw skills install`, such as `--global` or `--target global`, so ClawHub skills can be installed under shared skill roots instead of only an agent workspace.

## Best Possible Solution

Land #74466 or an equivalent narrow implementation that adds an opt-in shared/global target, preserves the current workspace default, rejects conflicting `--global` and `--agent` usage, scopes ClawHub tracking metadata to the selected install root, updates docs/changelog, and covers install/update behavior with focused CLI tests.

## Reproduction Assessment

Yes. A high-confidence read-only reproduction exists on current main: command registration has no global/target option, and the installer resolves every ClawHub target under `workspaceDir/skills`.

## Solution Assessment

Yes. An explicit opt-in global/shared target is the narrowest maintainable direction; changing the default workspace behavior would be broader and could surprise existing multi-agent setups.
