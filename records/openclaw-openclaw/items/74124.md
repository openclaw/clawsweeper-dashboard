## Summary

Keep open. Current main still has only workspace/agent-targeted ClawHub skill installs, and the explicit shared/global install target is represented by #74466, which is open and unmerged.

## What This Changes

Add an explicit `openclaw skills install` target for installing ClawHub skills into a shared/global skills directory instead of only an agent workspace.

## Best Possible Solution

Land #74466 or an equivalent narrow change that adds an opt-in shared/global target, preserves the current workspace default, handles `--global` and `--agent` conflicts, keeps tracking metadata scoped to the selected root, and updates tests/docs/changelog.

## Reproduction Assessment

Yes. A high-confidence read-only reproduction exists: current main has no global install CLI option, and the ClawHub installer always resolves targets below `workspaceDir/skills`.

## Solution Assessment

Yes. An explicit opt-in global/shared target is the narrowest maintainable fix; changing the default install scope would be broader and risk surprising existing multi-agent setups.
