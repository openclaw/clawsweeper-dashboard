## Summary

Keep open. Current main still only supports active-workspace ClawHub skill install/update targeting while shared skill roots are already loadable, and #74466 is an open, unmerged closing PR for the requested behavior.

## What This Changes

Request an explicit global/shared install target for `openclaw skills install` and related update behavior so ClawHub skills can be installed under shared roots such as `~/.openclaw/skills` instead of only an agent workspace.

## Best Possible Solution

Land #74466 or an equivalent narrow implementation that adds an opt-in shared/global install target, preserves the current workspace default, rejects conflicting target options, keeps ClawHub tracking metadata scoped to the selected install root, and updates CLI docs, changelog, and focused command/installer tests.

## Reproduction Assessment

Yes. A high-confidence read-only reproduction exists on current main: the command surface has no global/target option, and the installer resolves every ClawHub skill under `workspaceDir/skills`.

## Solution Assessment

Yes. An explicit opt-in global/shared target is the narrowest maintainable direction; changing the default workspace behavior would be broader and could surprise existing multi-agent setups.
