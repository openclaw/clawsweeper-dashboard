## Summary

Keep open. Current main still only routes ClawHub skill install/update through active or selected agent workspace directories, and the linked closing implementation at https://github.com/openclaw/openclaw/pull/74466 is still open and unmerged.

## What This Changes

Add an explicit global/shared target for `openclaw skills install` and `openclaw skills update` so ClawHub skills can be installed under the shared managed skills directory.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/74466 or an equivalent narrow CLI/docs/test implementation that preserves workspace defaults and adds an explicit shared target.

## Reproduction Assessment

Not applicable as a bug reproduction, but the feature gap is high-confidence from source inspection: current main exposes no global/shared install or update target while shared managed skills are already loadable.

## Solution Assessment

Yes. An explicit opt-in shared target is the narrowest maintainable solution; changing the default workspace behavior would be broader and more surprising for multi-agent setups.
