## Summary

Keep open. Current main still routes ClawHub skill install/update only through active workspace or agent targets, and #74466 is an open, unmerged closing PR carrying the focused implementation candidate.

## What This Changes

Request an explicit global/shared target for ClawHub skill installs so shared skills can be installed outside an agent workspace.

## Best Possible Solution

Review and land #74466 or an equivalent narrow CLI/docs/test/changelog implementation that adds an opt-in shared target while preserving the workspace default.

## Reproduction Assessment

Yes. Source inspection on current main shows no `--global` or `--target` selector, and the installer derives target directories from `<workspaceDir>/skills`.

## Solution Assessment

Yes. An explicit opt-in global/shared target is the narrowest maintainable direction; changing the default workspace behavior would be broader and could surprise existing multi-agent setups.
