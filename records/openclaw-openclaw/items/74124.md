## Summary

Keep open. Current main still only supports active-workspace ClawHub skill installs/updates, while the provided GitHub context shows open, unmerged closing PR #74466 already paired with this issue.

## What This Changes

Add an explicit global/shared target for `openclaw skills install` so ClawHub skills can intentionally install under shared roots like `~/.openclaw/skills` instead of only the active agent workspace.

## Best Possible Solution

Land #74466 or an equivalent narrow replacement that adds an opt-in shared/global install target, preserves the current workspace default, keeps tracking metadata scoped to the selected root, and updates CLI docs plus focused command tests.

## Reproduction Assessment

Yes. A high-confidence read-only reproduction exists on current main: command registration has no global/target option, and the ClawHub installer always resolves targets under `workspaceDir/skills`.

## Solution Assessment

Yes. An explicit opt-in global/shared target is the narrowest maintainable direction; changing the default workspace behavior would be broader and could surprise existing multi-agent setups.
