## Summary

Keep open. Current main still lacks an explicit global/shared install target for ClawHub skill installs or updates, and the supplied GitHub context shows an open closing PR, #74466, already paired with this work.

## What This Changes

Request adding an explicit global/shared target, such as `--global` or `--target global`, so ClawHub skill installs can intentionally land in `~/.openclaw/skills` instead of only an agent workspace.

## Best Possible Solution

Land #74466 or an equivalent narrow replacement that adds an opt-in shared/global target, preserves existing workspace defaults, keeps install/update metadata consistent for the selected root, documents precedence, and covers CLI conflict behavior with focused tests.

## Reproduction Assessment

Yes. A high-confidence read-only reproduction is available from current main: command registration has no global/target option, and the installer resolves targets under `path.resolve(workspaceDir)/skills`.

## Solution Assessment

Yes. An explicit opt-in shared/global target is the narrowest maintainable direction; changing the default workspace behavior would be broader and could surprise existing multi-agent setups.
