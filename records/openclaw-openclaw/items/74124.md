## Summary

Keep open. Current main still lacks an explicit global/shared install target for ClawHub skills, and the remaining work is already paired with open closing PR #74466, so this issue should stay open until that PR lands or is superseded.

## What This Changes

The issue requests an explicit global/shared target for `openclaw skills install` and related updates while preserving the existing workspace-scoped default.

## Best Possible Solution

Land #74466 or an equivalent narrow replacement that adds an opt-in shared/global target for install and update, keeps current workspace defaults unchanged, writes ClawHub metadata consistently for the chosen root, documents precedence, and covers conflict behavior with focused CLI tests.

## Reproduction Assessment

Yes. A high-confidence read-only reproduction is available from current main: command registration has no global/target option, the installer resolves targets under `workspaceDir/skills`, and a targeted search found no `--global` implementation in the relevant CLI/docs/tests.

## Solution Assessment

Yes. An explicit opt-in global/shared target is the narrowest maintainable direction because shared roots are already loaded, while changing the default workspace target would be broader and potentially surprising for existing multi-agent setups.
