## Summary

Keep open. Current main still lacks an intentional global/shared ClawHub skill install target even though shared skill roots are already loadable; the linked implementation PR #74466 is open and unmerged, so this issue should remain the tracker.

## What This Changes

Request an explicit `openclaw skills install` target, such as `--global` or `--target global`, for installing shared skills outside the active agent workspace.

## Best Possible Solution

Land #74466 or an equivalent narrow implementation that adds an opt-in shared/global target for install and update, rejects conflicting workspace/global targeting, preserves existing workspace behavior, updates docs and changelog, and covers the selected install root with focused CLI tests.

## Reproduction Assessment

Yes. A high-confidence read-only reproduction exists on current main: the CLI exposes no global/target option, and the installer resolves every ClawHub skill path under `workspaceDir/skills`.

## Solution Assessment

Yes. An explicit opt-in global/shared target is the narrowest maintainable direction; changing default workspace resolution would be broader and could surprise existing multi-agent setups.
