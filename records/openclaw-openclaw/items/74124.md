## Summary

Keep open. Current main still has only workspace-scoped ClawHub skill install/update routing, while the linked implementation PR is open and uses closing linkage for this issue.

## What This Changes

Request an explicit global/shared target for `openclaw skills install` and related update flows so skills can be installed into the shared skills directory instead of an agent workspace.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/74466, or an equivalent narrow CLI/docs/test change that preserves workspace defaults and adds an explicit shared target.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence in the feature gap because current main lacks a global install/update target while shared skill roots are already documented and loaded.

## Solution Assessment

Yes. An explicit opt-in shared target is the narrowest maintainable direction because it preserves existing workspace behavior and avoids changing default multi-agent install semantics.
