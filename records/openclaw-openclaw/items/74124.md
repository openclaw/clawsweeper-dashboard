## Summary

Keep open. Current main still only targets workspace skill directories for ClawHub skill installs/updates, and the open closing PR #74466 has not merged, so this issue should remain the canonical tracker.

## What This Changes

Request an explicit global/shared install target for `openclaw skills install` so ClawHub skills can land in `~/.openclaw/skills` instead of only a workspace `skills/` directory.

## Best Possible Solution

Land #74466 or an equivalent narrow implementation that preserves workspace defaults, adds an explicit shared/global target, handles `--global` and `--agent` conflicts, scopes tracking metadata to the selected root, and updates tests/docs/changelog.

## Reproduction Assessment

Yes. A high-confidence read-only reproduction exists on current main: the CLI has no `--global` or `--target` option, and the ClawHub installer always resolves targets under `workspaceDir/skills`.

## Solution Assessment

Yes. An opt-in global/shared target is the narrowest maintainable fix; changing the default install location would be broader and more disruptive for existing workspace-scoped users.
