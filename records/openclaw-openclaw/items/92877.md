## Summary

Keep open: the Telegram readability goal is valid and has live proof, but the patch is broader than the stated fix because it changes the shared stock footer and Discord footer, dropping token/cache telemetry that current docs and tests still define for `/usage full`. ## What This Changes

The PR modifies the built-in `/usage full` usage-bar template to make the default footer more compact and easier to wrap on Telegram/mobile surfaces.

## Best Possible Solution

Add a Telegram-specific compact usage footer surface, preserve the current shared default and Discord telemetry unless maintainers intentionally change the product contract, and remove the redundant template-leading newline.

## Reproduction Assessment

Yes for the review findings: source inspection shows the shared default/Discord footer is changed while `replyToChannel` already allows a Telegram-specific surface, and CI confirms the missing token/cache atoms plus the extra blank line. I did not run a local Telegram client in this read-only review.

## Solution Assessment

No. The Telegram wrap goal is reasonable, but the current patch is too broad; the maintainable fix is to use a scoped Telegram surface and keep the existing stock telemetry contract unless maintainers choose a product-wide default change.
