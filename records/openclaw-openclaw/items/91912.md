## Summary

Keep open: current main and the latest release intentionally deliver model fallback notices and expose no `suppressFallbackNotice` config key, while OpenClaw policy treats new config and fallback behavior as maintainer product decisions rather than automatic bug fixes.

## What This Changes

The issue requests a per-session or per-agent opt-out config option to suppress user-visible model fallback notices.

## Best Possible Solution

Keep fallback notices visible by default; if maintainers accept the ergonomics request, add a narrow opt-in suppression policy with aligned schema/help/docs and regression coverage around fallback state and source-reply suppression.

## Reproduction Assessment

Yes from source inspection, not from a live run: current runtime, docs, and tests show fallback notices are intentionally delivered when fallback state changes and bypass normal source-reply suppression.

## Solution Assessment

Unclear: the requested config key is plausible, but new config and fallback suppression need a maintainer product decision before a code path is chosen.
