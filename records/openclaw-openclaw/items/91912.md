## Summary

Keep open: current main and v2026.6.5 intentionally deliver model fallback notices and expose no `suppressFallbackNotice` key, while adding one would change config and fallback-notice product behavior.

## What This Changes

Request to add a per-session or per-agent `suppressFallbackNotice` option so non-interactive sessions can opt out of model fallback notices in chat surfaces.

## Best Possible Solution

Keep fallback notices visible by default unless maintainers accept a narrow opt-in suppression policy with aligned schema, help text, docs, and regression coverage.

## Reproduction Assessment

Yes from source inspection: current main builds and prepends visible fallback notices when fallback state changes, and the public docs document that this bypasses normal source-reply suppression.

## Solution Assessment

Unclear: a suppression option is plausible, but the scope, default, and operational-safety tradeoff for hiding fallback state need maintainer product direction before implementation.
