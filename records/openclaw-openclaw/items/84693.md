## Summary

Keep this PR open for maintainer review: the patch appears directionally correct with sufficient live heap evidence, but it changes session-store cache and snapshot behavior on session-state and availability-sensitive paths that should get owner acceptance before merge.

## What This Changes

The PR adds immutable session-store snapshot/entry read APIs, interns repeated skills prompts, bounds serialized session-cache retention, migrates several read-only session lookups, and isolates agent attempt lifecycle callbacks.

## Best Possible Solution

Land the compatibility-preserving snapshot-read and capped-cache path after session/cache owner acceptance; track any stricter loadSessionStore deprecation or guardrail plan separately.

## Reproduction Assessment

No local high-confidence current-main reproduction was run in this read-only review. Source inspection shows the full-store clone and unbounded serialized-cache paths, and the PR body supplies credible live before/after heap evidence for the high-session workload.

## Solution Assessment

Yes, mostly: preserving loadSessionStore compatibility while adding immutable snapshot reads and bounding serialized cache state is the maintainable fix shape. Any stricter deprecation of mutable full-store reads should remain a separate maintainer API decision.
