## Summary

Keep open. Current main still has a credible source-level path for stale raw-stream logging under LaunchAgent: raw-stream itself treats unset and `0` as disabled, but the generated LaunchAgent service-env wrapper can preserve an enabled environment across a normal restart, and the docs still only describe enabling raw-stream logging.

## What This Changes

The issue asks for macOS LaunchAgent-managed raw-stream logging to stop after `OPENCLAW_RAW_STREAM` is unset or set to `0`, with docs clarifying supported disable values.

## Best Possible Solution

Define raw-stream disable semantics, add a focused LaunchAgent service-env refresh or diagnostic path for stale raw-stream metadata, update the docs, and cover the disable flow with regression tests.

## Reproduction Assessment

Yes, with source-level confidence. The issue provides concrete launchd steps, and current main still has env-gated raw-stream writes plus persistent generated service-env metadata that a successful `kickstart -k` restart does not refresh.

## Solution Assessment

Unclear. The expected disable behavior is valid, but the narrowest maintainable fix likely belongs in LaunchAgent service-env refresh/diagnostics and docs, not in the raw boolean parser that already treats unset and `0` as disabled.
