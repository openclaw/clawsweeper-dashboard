## Summary

Keep open. Current main still supports the reported source-level failure mode: raw-stream writes depend on the gateway process environment, `0` and unset are already parsed as disabled, the LaunchAgent restart path can still kickstart an unchanged plist, and docs still do not state disable semantics or LaunchAgent environment interaction.

## What This Changes

The issue asks OpenClaw to stop raw-stream JSONL writes after `OPENCLAW_RAW_STREAM` is unset or set to `0` under macOS LaunchAgent restarts, and to document supported disable values.

## Best Possible Solution

Document accepted truthy and disabled raw-stream values, then make LaunchAgent restart/install/doctor refresh or warn about stale raw-stream environment state so disabled values actually stop logging, with live macOS proof.

## Reproduction Assessment

Yes, with source-level confidence: the reporter supplied concrete LaunchAgent steps and current code shows that a stale truthy gateway process environment would keep `appendRawStream` writing. I did not run the live macOS launchd scenario.

## Solution Assessment

Unclear. Changing raw-stream boolean parsing alone is not the best fix because `0` is already disabled; the maintainable path is likely LaunchAgent environment refresh or diagnostics plus docs.
