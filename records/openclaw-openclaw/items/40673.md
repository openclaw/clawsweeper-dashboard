## Summary

Keep open. Current main still has a credible source-level LaunchAgent/service-env stale logging path, and the user-facing docs only describe how to enable raw-stream logging, not how to disable it safely under launchd.

## What This Changes

Requests that macOS LaunchAgent raw-stream logging stop after `OPENCLAW_RAW_STREAM` is unset or set to `0`, with docs clarifying disabled values and LaunchAgent behavior.

## Best Possible Solution

Document truthy and disabled raw-stream values, then make macOS LaunchAgent restart/install/doctor refresh or warn about stale generated service-env metadata that can keep raw-stream enabled.

## Reproduction Assessment

Yes, with source-level confidence: a generated LaunchAgent env wrapper can continue exporting `OPENCLAW_RAW_STREAM=1` while `restartLaunchAgent` only kickstarts the already-loaded service, and raw-stream writes trust process env. I did not run the live macOS launchd repro.

## Solution Assessment

Unclear. The parser already treats unset and `0` as disabled, so the maintainable fix is likely a LaunchAgent service-env refresh/diagnostic plus docs update rather than changing raw-stream boolean parsing.
