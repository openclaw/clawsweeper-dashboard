## Summary

Keep open. Current main and v2026.5.22 still do not document raw-stream disable values, and the normal macOS LaunchAgent restart path still uses `launchctl kickstart -k` when the plist is unchanged, so the reported post-disable append behavior is not proven fixed.

## What This Changes

The issue asks OpenClaw to make `OPENCLAW_RAW_STREAM` disable semantics clear and ensure LaunchAgent-managed gateways stop appending raw-stream logs after unset or false-like values.

## Best Possible Solution

Document accepted raw-stream truthy and disabled values, then make LaunchAgent restart/status/doctor diagnose or clear stale raw-stream environment so unset, `0`, and documented disabled values stop new writes, with live macOS proof.

## Reproduction Assessment

No, not with high confidence. The reporter gave concrete macOS LaunchAgent steps and current source shows a plausible stale-environment path, but this review did not live-verify current main on macOS.

## Solution Assessment

Unclear. A parser-only fix is not the right solution because `0` and unset already parse false; the maintainable path likely combines LaunchAgent environment diagnostics or reload behavior with explicit docs.
