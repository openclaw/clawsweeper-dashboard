## Summary

Keep open. Current main still has a credible source-level gap: raw-stream writes are controlled by the running gateway process environment, while macOS LaunchAgent restart and documentation still do not clearly prove or define the disable path for `launchctl unsetenv`, `0`, and stale service environment state.

## What This Changes

The issue asks OpenClaw to make `OPENCLAW_RAW_STREAM` disable semantics clear and ensure LaunchAgent-managed gateways stop appending raw stream logs after the variable is unset or set to `0`.

## Best Possible Solution

Document the accepted truthy and disabled raw-stream values, then make macOS LaunchAgent restart, install, or doctor detect and resolve stale raw-stream launchctl/service-env state so disabling actually stops writes, with live macOS proof.

## Reproduction Assessment

No live high-confidence reproduction was established in this review. The reporter provided concrete LaunchAgent steps and current source shows a credible path whenever the running gateway retains a truthy raw-stream environment value.

## Solution Assessment

Unclear. Changing boolean parsing alone is not the best fix because `0` already disables the writer; the maintainable solution likely combines LaunchAgent environment refresh or diagnostics with explicit docs.
