## Summary

Keep open. Current main still leaves the user-facing disable semantics undocumented, and source inspection does not give high-confidence proof that the macOS LaunchAgent path clears every raw-stream enablement source after `unsetenv` or `0`.

## What This Changes

The issue requests that disabling `OPENCLAW_RAW_STREAM` via unset or `0` under a macOS LaunchAgent reliably stop raw-stream writes and that supported disabled values be documented.

## Best Possible Solution

Document accepted raw-stream truthy and disabled values, then make LaunchAgent status/doctor/restart expose or clear every raw-stream enablement source, with live macOS proof for unset, `0`, and any documented disabled values.

## Reproduction Assessment

No high-confidence current-main reproduction is available from this review. The reporter provided concrete macOS LaunchAgent steps and logs, but this read-only review did not run the live macOS LaunchAgent scenario and source inspection does not isolate the exact stale enablement source.

## Solution Assessment
