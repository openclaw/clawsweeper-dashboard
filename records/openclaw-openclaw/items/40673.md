## Summary

Keep open. Current `main` still has a credible source-level LaunchAgent path where raw-stream logging can remain enabled through the generated service-env wrapper after an operator unsets or sets `OPENCLAW_RAW_STREAM=0`, and the docs still only describe enabling the logger.

## What This Changes

The issue reports that LaunchAgent-managed raw-stream logging can keep appending after `OPENCLAW_RAW_STREAM` is unset or set to `0`.

## Best Possible Solution

Document truthy and disabled raw-stream values, then make LaunchAgent restart/install/doctor refresh or warn on stale generated service-env metadata that can keep raw-stream logging enabled.

## Reproduction Assessment

Yes, with source-level confidence: a generated LaunchAgent env wrapper can export `OPENCLAW_RAW_STREAM=1`, normal restart can kickstart without refreshing that file, and raw-stream writes trust the resulting process env. I did not run the live macOS launchd repro.

## Solution Assessment

Unclear. The parser already treats unset and `0` as disabled, so the maintainable fix is likely LaunchAgent service-env refresh/diagnostics plus docs rather than changing boolean parsing.
