## Summary

Keep open: current main and v2026.6.8 only prove that `0` and unset are disabled inside a correctly restarted process; the macOS LaunchAgent/service-env path and docs still do not prove or define the reported sensitive-log disable behavior.

## What This Changes

The issue asks OpenClaw to stop raw-stream appends after `OPENCLAW_RAW_STREAM` is unset or set to `0` for a restarted macOS LaunchAgent, and to document the supported disable values.

## Best Possible Solution

Define the supported raw-stream disable semantics across process env and managed LaunchAgent service env, document them, and prove on macOS that post-disable restarts stop jsonl appends.

## Reproduction Assessment

No high-confidence live reproduction yet. Source confirms the in-process parser rejects `0` and unset values and shows the LaunchAgent env-file path to inspect, but the reported macOS post-restart append behavior still needs live proof on current main or v2026.6.8.

## Solution Assessment

Unclear. A raw parser fix is not the best solution because current code already treats `0` and unset as disabled; the maintainable fix is LaunchAgent env-source proof plus documented disable semantics.
