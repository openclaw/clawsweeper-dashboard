## Summary

Keep open: current main still treats `OPENCLAW_RAW_STREAM=0` as disabled in-process, but neither current main nor the latest release documents or proves the reported macOS LaunchAgent disable path, and the issue is security-sensitive because raw-stream logs can contain private model-stream data.

## What This Changes

The issue reports that disabling `OPENCLAW_RAW_STREAM` for a macOS LaunchAgent can still leave `raw-stream.jsonl` receiving new events after a gateway restart.

## Best Possible Solution

Define the supported disable path for raw-stream logging, document it, and verify LaunchAgent-managed restart behavior with before/after append proof on macOS.

## Reproduction Assessment

Partly. Source shows the parser rejects `0` and unset values and the append path is clear, but the macOS LaunchAgent restart append behavior has not been reproduced in this review.

## Solution Assessment

Unclear. A parser-only fix is not the best solution because current main already rejects `0`; the likely solution is LaunchAgent restart/env propagation proof plus docs for supported disable semantics.
