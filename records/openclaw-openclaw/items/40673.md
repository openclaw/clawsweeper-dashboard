## Label Justifications

- `P2`: The historical bug could keep an opt-in sensitive diagnostic active after an operator attempted to disable it.
- `impact:security`: Raw-stream logs can contain private prompts, model output, and tool results.

## Summary

The reported stale disable behavior is fixed: current main re-evaluates the raw-stream environment flag before every append instead of caching it at module load, and that correction first shipped in v2026.3.28. The docs could still state the false values more explicitly, but the central runtime bug no longer remains.

## What This Changes

Ensure raw-stream logging stops after `OPENCLAW_RAW_STREAM` is unset or set to a false value, including for a restarted macOS LaunchAgent.

## Best Possible Solution

Keep the shipped per-append truthy check; a separate low-risk documentation update may explicitly state that unset, `0`, and `off` disable raw-stream logging.

## Reproduction Assessment

Yes for v2026.3.7 from source: the enabled state was cached at module load, matching stale in-process environment behavior. Current main and v2026.3.28 onward no longer reproduce that path.

## Solution Assessment

Yes. Re-evaluating the existing strict truthy parser before each append fixes the stale-state root cause without adding configuration, compatibility branches, or another logging path.
