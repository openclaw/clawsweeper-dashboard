## Summary

Keep open: current main still treats `OPENCLAW_RAW_STREAM=0` and unset as disabled inside the raw-stream writer, but the macOS LaunchAgent wrapper/service-env path can still supply a process-level enabled value that `launchctl print` may not make obvious, and no merged PR or release proof closes that LaunchAgent disable gap.

## What This Changes

The issue asks OpenClaw to stop appending raw-stream logs after `OPENCLAW_RAW_STREAM` is unset or set to `0` for a LaunchAgent-managed gateway restart, and to document supported disable values.

## Best Possible Solution

Define the canonical raw-stream disable path for managed macOS gateways, prove it with a LaunchAgent before/after append check, and document the exact values and service-env/global-dotenv precedence users should rely on.

## Reproduction Assessment

Partly. Source inspection shows the writer depends on the final process value of `OPENCLAW_RAW_STREAM`, and the LaunchAgent wrapper can source a generated env file before exec, but I did not establish a live macOS LaunchAgent repro against current main.

## Solution Assessment

Unclear. A parser-only fix is not the best solution because `0` is already false; the likely fix is in managed service env/source precedence plus docs and live proof.
