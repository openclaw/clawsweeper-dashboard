## Summary

Keep open. Current main still leaves the reported LaunchAgent raw-stream disable path unresolved: the raw-stream writer is disabled for `0`/unset only if the running gateway process actually sees that environment, while the macOS restart path can still take the unchanged-plist `kickstart -k` path and the docs still only describe enablement.

## What This Changes

The issue asks for `OPENCLAW_RAW_STREAM` disable behavior to stop `raw-stream.jsonl` appends after LaunchAgent restart and for docs to clarify supported disable values.

## Best Possible Solution

Document accepted truthy and disabled raw-stream values, then make macOS LaunchAgent restart, install, or doctor surface stale raw-stream environment state so disabling actually stops writes, with live macOS proof.

## Reproduction Assessment

Yes, with source-level confidence: the reporter provided concrete LaunchAgent steps and current code shows continued writes whenever the running gateway process still has truthy `OPENCLAW_RAW_STREAM`. I did not run the live macOS launchd scenario.

## Solution Assessment

Unclear. Changing raw-stream boolean parsing alone is not the best fix because `0` is already disabled; the maintainable solution likely combines LaunchAgent environment refresh or diagnostics with explicit docs.
