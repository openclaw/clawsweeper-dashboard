## Summary

Keep open: current main still has a plausible LaunchAgent env-file override path for raw-stream logging, the disable precedence is not documented, and no merged PR or live macOS proof closes this sensitive logging report.

## What This Changes

The issue asks OpenClaw to stop raw-stream JSONL writes after a LaunchAgent-managed gateway unsets or sets `OPENCLAW_RAW_STREAM=0`, and to document supported disable values and precedence.

## Best Possible Solution

Define one raw-stream disable contract across `launchctl`, generated service env files, state-dir `.env`, config `env`, and CLI flags, then update docs, tests, doctor guidance, and macOS proof.

## Reproduction Assessment

Yes for a source-level path, but not a live macOS run in this review. The writer only appends when the final process env is truthy, while the LaunchAgent wrapper sources a generated env file before `exec`, creating a clear path where a service env file can override launchd-inherited disable state.

## Solution Assessment

No current close path is proven best. The maintainable fix needs an explicit service-env precedence contract for sensitive logging, plus docs, regression coverage, and live macOS proof.
