## Summary

Keep open. Current main still has a source-level path where raw-stream logging can remain enabled through generated LaunchAgent service environment metadata even after the visible launchctl value is unset or 0, and the docs still describe only how to enable raw-stream logging.

## What This Changes

The issue requests clear OPENCLAW_RAW_STREAM disable semantics and reliable raw-stream shutdown for LaunchAgent-managed gateways after unset or 0 plus restart.

## Best Possible Solution

Document accepted truthy and disabled raw-stream values, then make LaunchAgent restart/install/doctor refresh or warn on stale generated service-env entries that keep raw-stream enabled.

## Reproduction Assessment

Yes, with source-level confidence: a generated LaunchAgent env wrapper can export OPENCLAW_RAW_STREAM=1, raw-stream writes trust process.env, and a successful restart kickstart returns without rewriting the loaded service metadata. I did not run the live macOS launchd repro.

## Solution Assessment

Unclear. The parser already treats unset and 0 as disabled, so the maintainable fix is likely LaunchAgent service-env refresh/diagnostics plus docs rather than changing boolean parsing.
