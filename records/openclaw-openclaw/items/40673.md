## Summary

Keep open. Current main still documents how to enable raw-stream logging but not the disable semantics, and the source does not prove the macOS LaunchAgent path clears stale raw-stream process environment after `unsetenv` or `0`.

## What This Changes

The issue asks OpenClaw to stop raw-stream writes after `OPENCLAW_RAW_STREAM` is unset or set to `0` under a macOS LaunchAgent restart, and to document the disable values.

## Best Possible Solution

Document accepted raw-stream truthy and disabled values, then make LaunchAgent restart/status/doctor detect or clear stale raw-stream service environment so disabling logging stops writes, with live macOS proof.

## Reproduction Assessment

No, not with high confidence in this review. The reporter provided concrete macOS LaunchAgent steps and current source shows a plausible stale-environment path, but current main was not live-verified on macOS.

## Solution Assessment

Unclear. A parser-only fix is not the best path because `0` is already falsey; the maintainable fix likely combines LaunchAgent environment diagnostics or refresh behavior with explicit docs.
