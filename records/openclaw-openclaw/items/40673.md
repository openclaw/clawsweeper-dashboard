## Summary

Keep open. Current main still has a credible LaunchAgent stale-service-env path: raw-stream itself treats unset and `0` as disabled, but a normal macOS restart only kickstarts the existing service and does not refresh generated service-env wrapper metadata; the docs still only cover enabling raw-stream logging.

## What This Changes

The issue requests clear and enforced disable semantics for `OPENCLAW_RAW_STREAM` on LaunchAgent-managed gateways, including docs for disabled values and stale-service-env behavior.

## Best Possible Solution

Add a focused macOS service-env and docs fix: document truthy and disabled raw-stream values, clarify LaunchAgent precedence, make restart/install/doctor refresh or diagnose stale raw-stream service metadata, and add regression coverage for disabling.

## Reproduction Assessment

Yes, with source-level confidence. The issue gives concrete LaunchAgent steps, and current main still has process-env-gated raw-stream writes plus persistent generated service-env metadata that normal restart does not refresh; I did not run the macOS launchd repro live.

## Solution Assessment

Unclear. The requested behavior is valid, but the best fix is likely a narrow LaunchAgent service-env refresh/diagnostic plus docs update, not a change to the raw boolean parser that already disables `0` and unset.
