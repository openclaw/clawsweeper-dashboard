## Summary

Keep open. Current `main` still supports the reported source-level failure mode: raw-stream writes depend on the gateway process environment, `0` is already parsed as disabled, but the LaunchAgent restart path can still use a kickstart when the plist is unchanged and docs still do not explain disable values or LaunchAgent environment interaction.

## What This Changes

The issue asks for `OPENCLAW_RAW_STREAM` unset or `0` to reliably stop raw-stream appends under a macOS LaunchAgent after restart, with disable semantics documented.

## Best Possible Solution

Document accepted truthy and disabled raw-stream values, then make LaunchAgent restart/install/doctor refresh or warn about stale raw-stream environment state so disabled values actually stop logging, with live macOS proof.

## Reproduction Assessment

Yes, with source-level confidence: the reporter supplied concrete LaunchAgent steps, raw-stream writes still trust `process.env.OPENCLAW_RAW_STREAM`, and current restart can kickstart an unchanged loaded service without a targeted environment refresh. I did not run the live macOS launchd repro.

## Solution Assessment

Unclear. Changing raw-stream boolean parsing alone is not the best fix because `0` is already disabled; the maintainable path is likely LaunchAgent environment refresh/diagnostics plus docs.
