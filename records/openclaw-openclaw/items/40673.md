## Label Justifications

- `P2`: The original report was a bounded runtime logging bug with medium operator impact and a shipped fix.
- `impact:security`: The affected raw-stream log can contain prompts, tool output, and other sensitive debugging data.

## Summary

Current main and released builds no longer have the reported cached raw-stream enable state: raw-stream appends now re-check `OPENCLAW_RAW_STREAM` before every write, and the shared parser only enables `1`, `on`, `true`, or `yes`. The docs could still spell out false values more explicitly, but the central sensitive logging bug is fixed and shipped.

## What This Changes

The issue reports that disabling `OPENCLAW_RAW_STREAM` for a macOS LaunchAgent with unset or `0` still allowed `raw-stream.jsonl` to receive new events after restart.

## Best Possible Solution

Keep the shipped per-append truthy check; any remaining work should be a separate docs-polish change clarifying false disable values.

## Reproduction Assessment

Yes for the reported release from source: v2026.3.7 cached `RAW_STREAM_ENABLED` at module load. No current-main failing path remains because `appendRawStream()` now re-checks the env value before each append.

## Solution Assessment

Yes. The shipped fix reuses the existing strict truthy parser and removes the stale cached state without adding a LaunchAgent-specific workaround or new config surface.
