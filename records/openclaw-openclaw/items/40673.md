## Summary

Keep open. Current main still leaves the LaunchAgent raw-stream disable path unresolved: raw-stream writes stop only when the running gateway process sees a falsey `OPENCLAW_RAW_STREAM`, while the macOS restart path can still use `kickstart -k` without a plist reload and the docs still only describe enablement.

## What This Changes

The issue asks OpenClaw to clarify `OPENCLAW_RAW_STREAM` disable semantics and stop `raw-stream.jsonl` appends after a LaunchAgent restart when the env is unset or set to `0`.

## Best Possible Solution

Document the accepted truthy and disabled raw-stream values, then make LaunchAgent restart, install, or doctor detect and resolve stale raw-stream launchctl environment state so disabling actually stops writes, with live macOS proof.

## Reproduction Assessment

Yes, with source-level confidence: the reporter provided concrete LaunchAgent steps and current code shows continued writes whenever the running gateway process still has truthy `OPENCLAW_RAW_STREAM`. I did not run the live macOS launchd scenario.

## Solution Assessment

Unclear. Changing boolean parsing alone is not the best fix because `0` already disables the writer; the maintainable solution likely combines LaunchAgent environment refresh or diagnostics with explicit docs.
