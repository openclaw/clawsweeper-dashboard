## Summary

Keep open. Current main and the latest release still have a credible path where raw-stream writes follow the gateway process environment while macOS LaunchAgent restart can kickstart an already-loaded service without refreshing generated service-env metadata; the docs still only describe enabling raw-stream logging.

## What This Changes

Request to make disabling `OPENCLAW_RAW_STREAM` under a macOS LaunchAgent stop raw-stream writes and document supported disable values.

## Best Possible Solution

Define raw-stream disable and precedence semantics, refresh or diagnose stale LaunchAgent service-env metadata on restart/install/doctor, and update the debugging and gateway docs.

## Reproduction Assessment

Yes, with source-level confidence: a generated LaunchAgent env wrapper can keep `OPENCLAW_RAW_STREAM=1`, restart can kickstart without refreshing that file, and raw-stream writes trust the resulting process env. I did not run the live macOS launchd repro.

## Solution Assessment

Unclear. The parser already treats unset and `0` as disabled, so the maintainable fix is likely LaunchAgent service-env refresh or diagnostics plus docs, but env-source precedence needs maintainer judgment.
