## Summary

Keep open. Current main and the latest release still lack documented raw-stream disable semantics, and the LaunchAgent restart path still has a normal unchanged-plist path that uses `launchctl kickstart -k` rather than a full reload, so the reported macOS disable behavior is not proven fixed.

## What This Changes

Ensure raw-stream logging stops after a LaunchAgent-managed gateway restart when `OPENCLAW_RAW_STREAM` is unset or `0`, and document the supported disable values.

## Best Possible Solution

Document accepted raw-stream truthy and disabled values, then make LaunchAgent restart/status/doctor diagnose or clear stale raw-stream environment so unset, `0`, and documented disabled values stop new writes, with live macOS proof.

## Reproduction Assessment

No, not with high confidence. The reporter gave concrete macOS LaunchAgent steps and current source shows the plausible stale-environment path, but this review did not live-verify current main on macOS.

## Solution Assessment

Unclear. A parser-only fix is not the right solution because `0` and unset already parse false; the maintainable path likely combines LaunchAgent environment diagnostics or reload behavior with explicit docs.
