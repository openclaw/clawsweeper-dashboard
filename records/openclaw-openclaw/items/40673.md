## Summary

Keep open: current source says `OPENCLAW_RAW_STREAM=0` or unset should disable in-process raw-stream appends, but the reported macOS LaunchAgent path is still not proven on current main or documented clearly, and the surface is security-sensitive because raw-stream logs can include private model data.

## What This Changes

The issue asks OpenClaw to stop raw-stream JSONL appends after a LaunchAgent-managed gateway disables `OPENCLAW_RAW_STREAM` via unset or `0`, and to document the supported disable values.

## Best Possible Solution

Define the canonical raw-stream disable behavior for LaunchAgent-managed gateways, document it, and verify on macOS that new agent turns do not append after restart with the env unset and with `OPENCLAW_RAW_STREAM=0`.

## Reproduction Assessment

No. The issue gives concrete LaunchAgent steps, but current source would return before appending when the process env is `0` or absent, and this review did not establish live macOS LaunchAgent proof on current main.

## Solution Assessment

Unclear. A parser-only fix is not the best solution because current main already rejects `0`; the better path is LaunchAgent env/restart proof plus docs for the supported disable semantics.
