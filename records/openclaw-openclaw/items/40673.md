## Summary

Keep open. Current main still has a source-level path where generated LaunchAgent service-env metadata can keep `OPENCLAW_RAW_STREAM=1` effective after the visible `launchctl` environment is unset or set to `0`, and the public docs still only show enablement. The related reports at https://github.com/openclaw/openclaw/issues/18753 and https://github.com/openclaw/openclaw/pull/18740 cover CLI enablement, not this LaunchAgent disable path.

## What This Changes

Disabling `OPENCLAW_RAW_STREAM` via `launchctl unsetenv` or `OPENCLAW_RAW_STREAM=0` should stop raw-stream logging after a LaunchAgent restart, and docs should define supported disabled values.

## Best Possible Solution

Document accepted truthy and disabled raw-stream values, then make LaunchAgent restart/install/doctor refresh or warn on stale generated service-env entries that keep raw-stream enabled.

## Reproduction Assessment

Yes, with source-level confidence: a generated LaunchAgent env wrapper can export `OPENCLAW_RAW_STREAM=1`, raw-stream writes trust `process.env`, and a successful restart kickstart returns without rewriting the loaded service metadata. I did not run the live macOS launchd repro.

## Solution Assessment

Unclear. The parser already treats unset and `0` as disabled, so the maintainable fix is likely LaunchAgent service-env refresh/diagnostics plus docs rather than changing boolean parsing alone.
