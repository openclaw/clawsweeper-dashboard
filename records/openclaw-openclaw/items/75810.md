## Summary

Keep this PR open, but it should not merge as-is. Current main still has the CRLF/non-global managed-markdown bug and the PR is aimed at the right shared helper, but the proposed cleanup rewrites unrelated markdown whitespace outside managed blocks; the existing ClawSweeper comment correctly identifies that narrow blocker.

## What This Changes

The PR changes `replaceManagedMarkdownBlock` to match CRLF headed blocks, globally collapse duplicate managed blocks, adds helper regression tests, and adds an Unreleased changelog entry for the memory/dreaming/wiki fix.

## Best Possible Solution

Land one narrow helper fix that matches CRLF headed managed blocks and removes duplicate blocks while preserving all bytes outside matched managed ranges, with focused regression coverage and the existing changelog credit; then use the landed fix to resolve #75491 and supersede the duplicate PR path if needed.

## Reproduction Assessment

Yes. Pure string inputs to `replaceManagedMarkdownBlock` reproduce the current-main bug: a headed CRLF block misses the `\n` regex, and repeated LF blocks leave later copies because replacement is not global.

## Solution Assessment

No. The CRLF/global replacement direction is the right boundary, but the whole-document blank-line normalization is not the safest implementation; duplicate removal should preserve unrelated markdown exactly and only manage spacing introduced by removed duplicate matches.
