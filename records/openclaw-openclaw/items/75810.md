## Summary

Keep open. Current main still has the reported managed-markdown bug, and this PR targets the right shared helper with useful CRLF and duplicate-block coverage, but the patch should not merge as-is because it normalizes blank lines across the entire document and can rewrite user-authored memory/wiki content outside managed markers.

## What This Changes

The PR changes the shared managed-markdown helper to match CRLF heading boundaries and replace duplicate managed blocks, adds regression tests, and adds an Unreleased changelog entry for the memory/dreaming/wiki fix.

## Best Possible Solution

Land a narrow helper fix that matches CRLF headed blocks and collapses pre-existing duplicate managed blocks while preserving all bytes outside the matched managed blocks, with tests covering CRLF, duplicate collapse, and outside-whitespace preservation; then use that shipped fix to close #75491.

## Reproduction Assessment

Yes. The current-main helper can be reproduced with pure string inputs: a headed CRLF block misses the existing regex and a document with repeated matching blocks leaves later copies because replacement is not global.

## Solution Assessment

No. The regex direction and tests are appropriate, but whole-document blank-line normalization is not the best implementation; the safer fix is to remove duplicate matches while preserving unrelated markdown exactly.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

- **[P2] Limit cleanup to removed duplicate blocks:** `src/plugin-sdk/memory-host-markdown.ts:42`

The helper should preserve text outside managed markers and only normalize whitespace created by deleting duplicate blocks.
