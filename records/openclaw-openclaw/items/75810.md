## Summary

Keep open. Current main still has the managed-markdown CRLF/non-global replacement bug, and this PR targets the right helper with useful tests and changelog coverage, but the proposed cleanup can rewrite unmanaged markdown outside the managed block area and should not merge as-is.

## What This Changes

The PR changes the shared managed-markdown helper to match CRLF headed blocks, collapse duplicate managed blocks, adds regression tests, and adds an Unreleased changelog entry for the memory/dreaming/wiki fix.

## Best Possible Solution

Land a narrow helper fix that matches CRLF headed blocks and collapses pre-existing duplicate managed blocks while preserving all bytes outside matched managed-block ranges, with tests covering CRLF, duplicate collapse, and unmanaged whitespace preservation.

## Reproduction Assessment

Yes. The current-main helper can be checked with pure string inputs: a headed CRLF block misses the LF-only pattern, and a document with repeated matching blocks leaves later copies because replacement is not global.

## Solution Assessment

No. The regex direction and test coverage are appropriate, but whole-document newline normalization is broader than needed; the safer implementation removes duplicate matched ranges without rewriting unrelated markdown.
