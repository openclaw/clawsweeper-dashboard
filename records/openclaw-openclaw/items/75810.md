## Summary

Keep open. The PR targets a real current-main managed-markdown bug and includes focused tests plus a changelog entry, but it should not merge as-is because the patch rewrites unrelated user-authored markdown outside managed markers whenever an existing block is replaced.

## What This Changes

The PR changes `replaceManagedMarkdownBlock` to match CRLF headed blocks, replace matching managed blocks globally, remove later duplicate matches, add helper regression tests, and add an Unreleased changelog entry for the memory/dreaming/wiki fix.

## Best Possible Solution

Land a narrow helper fix that accepts CRLF headed blocks and collapses duplicate managed blocks while preserving bytes outside the matched managed blocks, with tests for CRLF, duplicate collapse, and outside-whitespace preservation; then use that shipped fix to close #75491.

## Reproduction Assessment

Yes. Current main can be reproduced with pure string inputs: a CRLF separator after the heading misses the existing LF-only regex, duplicated matching blocks are only partially replaced, and the PR finding is reproducible with unrelated text containing three or more blank lines outside managed markers.

## Solution Assessment

No. The CRLF/global matching direction is the right shared-helper boundary, but whole-document blank-line normalization is not the safest implementation because managed-block cleanup must not rewrite user-authored markdown outside the markers.
