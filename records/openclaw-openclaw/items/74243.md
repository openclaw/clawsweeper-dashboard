## Summary

Keep open: the latest head is useful and fixes the earlier diagnostic-test mismatch, but it is not merge-ready because the new fallback can return unoptimized `image/*` bytes without a positive dimension guard when the optimizer is unavailable, and the required user-facing changelog entry is still missing.

## What This Changes

The PR adds a shared media fallback for under-cap non-HEIC images when optimization fails, adds media regression tests, and marks Discord system-event text untrusted.

## Best Possible Solution

Land the fallback only after it proves the original image has passed the intended pixel/dimension guard, or restricts fallback to formats covered by that guard, and documents the user-facing fix in `CHANGELOG.md`.

## Reproduction Assessment

Yes. Current main has a source-level reproduction where an optimizer failure aborts `loadWebMedia`, and the PR adds a mock-backed under-cap PNG reproduction; the guard concern is also source-reproducible from the `image/*` classification plus unknown-metadata path.

## Solution Assessment

No, not as submitted. The shared media boundary is the right place for the fallback, but it needs a positive guard before returning original bytes and a changelog entry before it is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Require a positive image guard before fallback:** `src/media/web-media.ts:407-418`
  - body: The new fallback accepts any under-cap non-HEIC `image/*` after optimizer failure. Because `kindFromMime` sends all `image/*` through this path and the pixel guard returns without error when dimensions cannot be parsed on non-sips backends, an unavailable optimizer can now pass through original bytes for unvalidated image types instead of failing closed. Restrict the fallback to dimension-validated formats or require an explicit successful preflight before returning the original.
  - confidence: 0.87
- **[P2] Add the media fallback changelog entry:** `src/media/web-media.ts:407-418`
  - body: This changes user-facing image-tool and outbound media behavior and is intended to close active bug reports, but the PR head has no `CHANGELOG.md` Unreleased Fixes entry for the optimizer fallback. Add a single-line entry before merge.
  - confidence: 0.96

## Security Review

Status: needs_attention

Summary: Needs attention: the media fallback can return original bytes for unvalidated `image/*` inputs when the optimizer backend is unavailable, weakening the intended image guard boundary.

Concerns:

- **[medium] Fallback can bypass unknown-dimension image guards:** `src/media/web-media.ts:407`
  - body: The fallback returns the original image based on byte size and HEIC status only. For image MIME types whose dimensions are not parsed before the optimizer fails, that can bypass the fail-closed decoded-pixel or unknown-dimension guard and send unoptimized attacker-controlled image bytes onward.
  - confidence: 0.86
