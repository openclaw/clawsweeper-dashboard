## Summary

Keep this PR open for maintainer review rather than cleanup-close: current main still lacks the requested hook media exposure, but the branch is not merge-ready because it patches a stale type surface, exposes raw pre-staging media references, and has no after-fix real behavior proof.

## What This Changes

The PR adds `mediaPaths`, `mediaUrls`, and `mediaTypes` to the `before_agent_reply` plugin hook context and populates them from inbound `MsgContext` media metadata.

## Best Possible Solution

Define a rebased hook API with explicit staged or structured inbound media semantics, focused `before_agent_reply` tests, plugin docs, and maintainer-approved security boundaries.

## Reproduction Assessment

Yes. Source inspection shows current main passes only `cleanedBody` plus agent/session/channel context to `before_agent_reply`, and a handled hook returns before `stageSandboxMedia` runs.

## Solution Assessment

No. The feature direction is plausible, but this patch should be rebased to the live `src/plugins/hook-types.ts` contract and should expose staged or structured attachment data rather than raw pre-policy paths and URLs.
