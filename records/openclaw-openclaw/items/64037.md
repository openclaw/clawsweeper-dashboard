## Summary

Keep open: the feature need is plausible and still not implemented for `before_agent_reply` on current `main`, but this PR is currently incorrect because it targets an obsolete type location, exposes raw pre-staging media references, and lacks contributor real behavior proof.

## What This Changes

The branch adds `mediaPaths`, `mediaUrls`, and `mediaTypes` to the `before_agent_reply` hook context and populates them from inbound `MsgContext` media metadata.

## Best Possible Solution

Define a rebased hook API with explicit staged or structured inbound media semantics in `src/plugins/hook-types.ts`, plus focused hook tests, plugin docs, and maintainer-approved security boundaries.

## Reproduction Assessment

Yes. Source inspection shows current `main` passes no media fields to `before_agent_reply`, and a handled hook returns before `stageSandboxMedia` runs.

## Solution Assessment

No. The feature direction is plausible, but this patch should target the live hook contract and expose staged or structured attachment data rather than raw pre-policy paths and URLs.
