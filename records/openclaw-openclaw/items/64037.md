## Summary

This PR should stay open because it addresses a real plugin API gap, but the current branch is not mergeable as-is: it edits a stale hook type surface on current main, exposes raw pre-staging media references to plugins, and still lacks contributor-provided real behavior proof.

## What This Changes

The PR adds optional mediaPaths, mediaUrls, and mediaTypes fields to the before_agent_reply plugin hook context and populates them from inbound MsgContext media metadata.

## Best Possible Solution

Define a rebased before_agent_reply media contract in src/plugins/hook-types.ts, preferably using staged or structured attachment data with focused hook tests and public plugin documentation.

## Reproduction Assessment

Yes. Source inspection shows before_agent_reply can handle and return before stageSandboxMedia, while the current hook context lacks media fields, so the plugin media-loss path is reproducible from current main.

## Solution Assessment

No. The additive feature direction is plausible, but this branch updates a stale type surface and forwards pre-staging media references; the maintainable fix should define explicit staged or structured media semantics in the live hook contract.
