## Summary

Keep open: the feature gap is real, but this patch is not ready because it updates an obsolete hook type location, exposes raw inbound media before staging/allowed-root rewriting, and lacks after-fix real behavior proof.

## What This Changes

The PR adds `mediaPaths`, `mediaUrls`, and `mediaTypes` to the `before_agent_reply` hook context from inbound `MsgContext` media metadata.

## Best Possible Solution

Define a structured or staged inbound-media hook contract in `src/plugins/hook-types.ts`, wire it through `before_agent_reply`, and cover it with focused hook tests/docs plus real plugin proof.

## Reproduction Assessment

Yes, from source inspection: current main passes `cleanedBody` plus agent/session/channel fields to `before_agent_reply`, and a handled hook returns before media staging. This demonstrates a capability gap rather than a broken documented bug contract.

## Solution Assessment

No. The direction is reasonable, but the current diff updates the wrong public type surface and forwards pre-staging media references; a staged or structured contract is the safer maintainable path.
