## Summary

Keep open: the requested hook media gap still exists on current main, but this PR is not merge-ready because it updates a stale type surface, exposes pre-staging media references that need API/security approval, and lacks contributor real-behavior proof.

## What This Changes

The PR adds mediaPaths, mediaUrls, and mediaTypes to the before_agent_reply plugin hook context and wires them from inbound MsgContext media metadata.

## Best Possible Solution

Define a rebased hook API in src/plugins/hook-types.ts with explicit staged or structured media semantics, focused before_agent_reply tests, public plugin docs, and maintainer-approved security boundaries.

## Reproduction Assessment

Yes. Source inspection shows current main passes only cleanedBody plus agent/session/channel context to before_agent_reply and returns before stageSandboxMedia when a hook handles the turn.

## Solution Assessment

No. The additive direction is plausible, but this branch updates the stale type surface and exposes raw pre-staging media references; the safer path is a rebased, explicit media contract after API/security review.
