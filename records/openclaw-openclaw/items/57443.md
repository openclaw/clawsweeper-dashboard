## Summary

Keep open. Current main and the latest shipped tag still leave the BytePlus/Ark tool-schema path without `patternProperties` stripping, and the linked closing PR is open/conflicting rather than merged.

## What This Changes

The issue asks OpenClaw to prevent BytePlus Ark tool-call 400s by stripping unsupported JSON Schema keywords such as `patternProperties` from outbound tool schemas.

## Best Possible Solution

Add provider-owned BytePlus/Ark compatibility metadata or hooks, and include affected Ark-family providers such as Volcengine if confirmed, with focused regression coverage and redacted live Ark proof.

## Reproduction Assessment

Yes, at source level. `exec.env` still emits a TypeBox record schema with `patternProperties`, tool preparation reaches `normalizeToolParameterSchema()`, and BytePlus has no current compat hook that strips this keyword.

## Solution Assessment

No for the issue's broad universal-default proposal without maintainer agreement. A targeted BytePlus/Ark compat hook using the existing unsupported-keyword seam is the safer default unless maintainers explicitly choose the broader policy.
