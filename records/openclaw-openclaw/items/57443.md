## Summary

Keep open. Current main still routes BytePlus Ark tool schemas through the active normalizer without `patternProperties` compatibility, and the open closing PR has not landed or been updated for the current helper path.

## What This Changes

The issue reports BytePlus Ark rejecting tool schemas containing `patternProperties` and asks for schema keyword cleaning beyond the Gemini-specific path.

## Best Possible Solution

Update the active schema helper or BytePlus/Ark provider compat metadata to strip `patternProperties` for BytePlus Ark, preserve Anthropic/Gemini behavior, add focused regression coverage, and land or replace https://github.com/openclaw/openclaw/pull/57480.

## Reproduction Assessment

Yes at source level: `exec.env` still emits a map schema, the current tool factory passes BytePlus through the active normalizer, and BytePlus has no unsupported-keyword compat metadata. I did not run a live BytePlus Ark API request.

## Solution Assessment

No for the issue's proposed universal non-Anthropic default without maintainer agreement. The safer maintainable path is targeted BytePlus/Ark provider compat feeding the existing generic unsupported-keyword stripping, or an updated active-helper default if maintainers explicitly choose that policy.
