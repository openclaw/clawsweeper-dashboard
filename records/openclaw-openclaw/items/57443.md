## Summary

Keep open. Current main still leaves the reported BytePlus/Volcengine Ark path uncovered: schema cleaning is automatic only for Google/Gemini providers, generic keyword stripping only happens when `modelCompat.unsupportedToolSchemaKeywords` is present, and the bundled Ark-family provider manifests do not declare `patternProperties` unsupported. The issue is also paired with open PR #57480 using closing syntax, so it should remain open until that implementation candidate is reviewed, landed, replaced, or closed.

## What This Changes

The issue requests making tool-parameter JSON Schema keyword cleaning cover BytePlus/Volcengine Ark providers, or more broadly OpenAI-compatible providers, so `patternProperties` from tools like `exec.env` does not cause provider HTTP 400 errors.

## Best Possible Solution

Ship a focused fix that removes `patternProperties` from outgoing tool schemas for BytePlus/Volcengine Ark models while preserving Anthropicâ€™s full JSON Schema behavior, with regression coverage around the `exec.env` schema and the selected provider/model compatibility path.

Yes. The static path is high-confidence: `exec.env` still uses a record schema, current normalization does not clean non-Gemini providers without compat metadata, and the Ark provider manifests do not provide that metadata.
