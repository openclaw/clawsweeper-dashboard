## Summary

Keep this open: current main and v2026.6.8 still leave BytePlus Ark `patternProperties` unstripped because BytePlus has no provider-owned tool-schema compatibility hook, while the linked implementation attempts are closed, obsolete, or not a viable merged fix.

## What This Changes

The issue reports BytePlus Ark rejecting tool schemas containing `patternProperties` and asks for schema cleaning to cover BytePlus/Ark instead of only Gemini.

## Best Possible Solution

Add provider-owned schema compatibility for BytePlus and Volcengine Ark models that strips `patternProperties`, with focused regression tests and live Ark proof; avoid a global non-Anthropic policy unless maintainers explicitly choose that direction.

## Reproduction Assessment

Yes, by source inspection: BytePlus model routes can still receive an `exec.env` record schema while current cleanup only strips Gemini schemas or compat-declared keywords. I did not run a live BytePlus Ark request in this read-only review.

## Solution Assessment

No, current main is not the best end state because BytePlus lacks the provider-owned compat metadata that the active schema path expects. The best fix should use that provider/plugin seam rather than broadly changing every non-Anthropic provider without maintainer approval.
