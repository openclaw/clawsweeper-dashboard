## Summary

Keep open. Current main still has a source-level BytePlus/Ark path where tool schemas can carry `patternProperties`, and the linked OpenClaw fix PR is still open/conflicting rather than merged.

## What This Changes

The issue reports that BytePlus/Ark OpenAI-compatible tool calls can fail because `patternProperties` is still sent in tool JSON schemas.

## Best Possible Solution

Land a rebased targeted BytePlus/Ark provider compat patch, or an explicitly approved broad active-normalizer policy, with focused regression coverage and redacted Ark proof.

## Reproduction Assessment

Yes, at source level: `exec.env` still emits a TypeBox record schema with `patternProperties`, tool preparation reaches `normalizeToolParameterSchema()`, and BytePlus has no current compat hook that strips this keyword. I did not run a live Ark request in this read-only review.

## Solution Assessment

No for the issue's broad universal-default proposal without maintainer agreement. A targeted BytePlus/Ark compat hook using the existing unsupported-keyword seam is the safer default unless maintainers explicitly choose the broader policy.
