## Summary

Keep this open: current main and v2026.6.6 still only strip `patternProperties` through Gemini cleanup or explicit model compatibility metadata, while BytePlus has no tool-schema compat hook and the linked fix PR is still open rather than merged.

## What This Changes

The issue asks OpenClaw to prevent BytePlus Ark/doubao tool-bearing requests from sending JSON Schema keywords such as `patternProperties` that the provider rejects.

## Best Possible Solution

Add targeted provider-owned compatibility metadata or a provider hook for BytePlus/Ark and Volcengine Ark models, include regression coverage for stripping `patternProperties`, and keep this issue open until that fix merges with real provider proof.

## Reproduction Assessment

Yes, source-reproducible: current main still sends record-derived schema keywords unless the provider is Gemini or the model declares unsupported keywords, and BytePlus does not declare that compat metadata. I did not run a live BytePlus Ark call in this read-only review.

## Solution Assessment

Unclear as proposed: the bug is real, but the best fix should use the existing provider-owned compatibility seam rather than globally stripping schemas for every non-Anthropic provider without a maintainer policy decision.
