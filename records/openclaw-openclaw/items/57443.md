Keep open. Current main and the latest release still leave the BytePlus Ark tool-schema path without `patternProperties` stripping, and the live closing PR is still open and conflicting rather than merged.

## What This Changes

The issue asks OpenClaw to strip unsupported JSON Schema keywords such as `patternProperties` for BytePlus Ark/OpenAI-compatible tool schemas instead of only relying on Gemini-specific cleaning.

## Best Possible Solution

Land a rebased, maintainer-approved fix that adds BytePlus/Ark unsupported schema-keyword compat in provider metadata or an explicitly approved active-normalizer policy, with regression coverage and redacted live Ark proof.

## Reproduction Assessment

Yes, at source level: `exec.env` is a TypeBox record whose upstream type includes `patternProperties`, and current normalization only cleans Gemini or declared compat keywords. I did not run a live BytePlus Ark request.

## Solution Assessment

No, not as the issue's universal default without maintainer approval. A targeted BytePlus/Ark provider compat hook is the safer default unless maintainers explicitly choose broad non-Anthropic schema cleaning.
