## What This Changes

The issue requests that tool parameter schema cleanup cover BytePlus/Ark `patternProperties` failures instead of only the current Google/Gemini-oriented path.

## Best Possible Solution

Add provider-owned schema compatibility for BytePlus and Volcengine/Ark models that strips `patternProperties`, with focused regression tests and live Ark proof; use a global non-Anthropic scrubber only if maintainers explicitly choose that policy.

## Reproduction Assessment

Yes, source-level: BytePlus exposes Ark OpenAI-compatible routes without schema-compat metadata, and the active normalizer strips `patternProperties` only through Gemini cleanup or declared unsupported keywords. I did not run a live BytePlus Ark API request in this read-only review.

## Solution Assessment

No, the universal scrubber suggested in the issue is not clearly the best fix because it changes all non-Anthropic provider schemas. The narrower maintainable path is provider-owned compatibility metadata for BytePlus/Volcengine unless maintainers approve a broader policy.
