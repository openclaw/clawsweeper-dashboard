## Best Possible Solution

Land a rebased targeted BytePlus/Ark provider compat patch, or an explicitly approved broad active-normalizer policy, with focused regression coverage and redacted Ark proof; otherwise replace or close the stale implementation PR after preserving the useful context.

## Solution Assessment

No for the broad universal-default proposal without maintainer agreement. A targeted BytePlus/Ark compat hook using the existing unsupported-keyword seam is the safer default unless maintainers explicitly choose the broader policy.
