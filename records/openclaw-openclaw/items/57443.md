## Summary

Keep open. Current main and the latest release still leave BytePlus/Ark tool schemas uncleaned unless explicit compat metadata supplies the unsupported keyword list, and the open implementation candidate at https://github.com/openclaw/openclaw/pull/57480 still references this issue with closing syntax.

## What This Changes

Strip unsupported JSON Schema keywords such as `patternProperties` from BytePlus/Ark tool schemas instead of limiting schema cleaning to Gemini or explicit compat metadata.

## Best Possible Solution

Land or replace the linked PR with a focused fix in the active schema normalizer or BytePlus/Ark provider compat path, preserving Anthropic full JSON Schema behavior and adding regression coverage for `exec.env`.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: `exec.env` produces the `patternProperties` schema surface, BytePlus uses an OpenAI-compatible Ark endpoint, and current normalization does not strip that keyword for BytePlus without compat metadata.

## Solution Assessment

Unclear. Universal non-Anthropic stripping would address the report, but provider-owned BytePlus/Ark compat metadata or an active-helper update may be the narrower maintainable fix; the existing open PR needs review or replacement before this issue closes.
