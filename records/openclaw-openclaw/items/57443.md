## Summary

Keep open. Current main still does not strip `patternProperties` for BytePlus/Volcengine Ark unless compat metadata declares it, and open PR #57480 already targets this issue with closing syntax, so this should wait for that PR to be reviewed, landed, replaced, or closed.

## What This Changes

Make tool-parameter schema cleaning cover BytePlus/Volcengine Ark so `patternProperties` from `exec.env` does not cause HTTP 400s on OpenAI-compatible Doubao models.

## Best Possible Solution

Ship a focused fix that removes `patternProperties` from outgoing BytePlus/Volcengine Ark tool schemas, preserves Anthropic full JSON Schema behavior, and adds regression coverage for the `exec.env` tool schema path.

## Reproduction Assessment

Yes. The static reproduction path is high-confidence: `exec.env` still uses a record schema, current normalization leaves non-Google providers raw without compat metadata, and the Ark manifests do not provide that metadata.

## Solution Assessment

Unclear. Universal non-Anthropic cleaning would likely fix the report, but explicit Ark-family compat metadata plus a regression test may be the narrower maintainable path unless maintainers deliberately choose the broader policy.
