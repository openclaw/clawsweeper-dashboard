The issue should stay open because the report includes concrete recent logs, and current source still leaves a plausible stalled-model-call gap: diagnostics only abort-drain stalled embedded runs, while restart scheduling and run-loop shutdown can each wait on active work. ## What This Changes

Ensure safe gateway restarts cannot be pinned by stalled model calls or delivery recovery, and ensure subagent completion notifications route back to the requester channel.

## Best Possible Solution

Add a focused fix that prevents restart requests from exceeding the configured drain budget for stalled model-call work, while preserving explicit `deferralTimeoutMs: 0`, and add a mixed Weixin/Feishu subagent delivery regression that keeps channel and target fields paired.
