## What This Changes

The issue reports a regression where custom `openai-responses` providers using non-env SecretRef-managed `apiKey` values fail auth after upgrade.

## Best Possible Solution

Land a narrow resolver fix that lets runtime-resolved provider `apiKey` values satisfy custom provider auth without adding a new config shape or broad fallback.

## Reproduction Assessment

Yes at source level: a custom provider with `apiKey: secretref-managed` returns null in the direct key path, then exits synthetic runtime auth before reading the resolved runtime snapshot when no plugin hook matches. I did not execute the live gateway request in this read-only review.

## Solution Assessment

Yes for the direction: the fix belongs in `src/agents/model-auth.ts` so resolved runtime snapshot credentials can satisfy the existing provider auth contract. The open PR at https://github.com/openclaw/openclaw/pull/92113 is the current candidate to review rather than opening a second repair lane.
