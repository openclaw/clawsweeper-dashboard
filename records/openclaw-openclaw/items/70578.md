## Summary

Keep open. Current main mitigates the silent clobber paths with source-preserving projection, validation-before-write, destructive-write rejection, and plugin-install allowlist updates, but the requested root-schema passthrough and bundled command allowlist behavior remain unresolved config-contract and plugin-security policy decisions.

## What This Changes

The issue asks OpenClaw to prevent config round-trip data loss from strict root validation, runtime-shape projection, plugin installs, and restrictive bundled-command allowlists.

## Best Possible Solution

Maintainers should decide the config contract explicitly: either keep strict root validation with fail-closed diagnostics and regression coverage, or preserve unknown top-level keys as opaque data with matching docs, doctor behavior, and write tests; the bundled command exemption should be handled as a separate plugin-security policy change.

## Reproduction Assessment

Yes. Current source gives focused reproductions: unknown root keys are rejected by strict schema validation, while command aliases like `wiki` are blocked unless their parent plugin is in `plugins.allow`; the historical Windows byte-drop sequence was not rerun in this read-only review.

## Solution Assessment

Unclear. The suggested `.passthrough()` change would preserve opaque unknown keys, but it is not clearly the best maintainable fix until maintainers choose between typo/security strictness and legacy-data preservation; bundled command allowlist behavior needs the same policy call.
