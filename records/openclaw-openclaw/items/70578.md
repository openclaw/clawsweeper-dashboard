## Summary

Keep open. Current main and v2026.5.18 have mitigated several destructive write paths, but the remaining root config contract and bundled `plugins.allow` behavior are still intentional source-backed policy decisions rather than a terminal fix.

## What This Changes

The issue asks OpenClaw to stop config round-trips from silently losing schema-unknown top-level fields and to revisit bundled command behavior under restrictive `plugins.allow`.

## Best Possible Solution

Keep the shipped no-clobber protections, then make an explicit maintainer decision on unknown root-key preservation and bundled command allowlist semantics with matching schema, doctor, docs, and regression coverage.

## Reproduction Assessment

Yes, source-level. Current main still rejects unknown root keys through the strict root schema and still blocks bundled command aliases when the owning plugin is excluded by `plugins.allow`; I did not replay the original Windows global-npm byte-drop sequence.

## Solution Assessment

Unclear. A bare root `.passthrough()` change or a blanket bundled-command exemption is not the best standalone fix because both change documented config and plugin trust policy without the matching doctor, docs, tests, and security review.
