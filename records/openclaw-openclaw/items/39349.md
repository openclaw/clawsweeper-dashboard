## Summary

Keep open. Current main still silently turns an explicit sandbox `pty:true` request into non-PTY execution, and this PR is the open closing fix for the linked user report, but it still lacks required after-fix real behavior proof.

## What This Changes

This PR adds an informational exec warning in `src/agents/bash-tools.exec.ts` when `pty:true` is requested while sandbox execution disables PTY.

## Best Possible Solution

Land this warning-only fix or an equivalent patch after contributor proof and focused coverage confirm the behavior, while keeping sandbox PTY disabled.

## Reproduction Assessment

Yes by source inspection. With a sandbox runtime selected by `host=auto` or `host=sandbox` and `pty:true`, current main computes `usePty` as false and has no sandbox-specific warning branch.

## Solution Assessment

Yes, the warning-only direction is the narrowest maintainable fix because it explains the ignored explicit request without changing the sandbox security boundary. The merge path still needs real behavior proof and focused regression coverage.
