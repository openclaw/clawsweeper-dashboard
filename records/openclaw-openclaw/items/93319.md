## Summary

Keep open: the latest patch fixed the earlier delete-retention problem, but it now creates reset-suffixed trajectory runtime files that current artifact helpers classify as usage-counted transcripts, and the external PR still has only in-memory test proof.

## What This Changes

The PR changes session reset archiving to also archive adjacent trajectory runtime and pointer sidecars when `archiveSessionTranscriptsDetailed` runs with reason `reset`.

## Best Possible Solution

Archive trajectory sidecars on reset, but first update artifact classification and regression tests so reset-preserved trajectory archives are retained and cleaned as archives without becoming transcript, usage, memory-hit, or session-discovery inputs.

## Reproduction Assessment

Yes. Source-level evidence shows current main archives only transcript candidates on reset, and the PR-introduced classifier regression follows from the new `*.trajectory.jsonl.reset.*` files passing current usage-counted transcript helpers.

## Solution Assessment

No. Archiving sidecars in the reset archive helper is the right layer, but the PR must also keep those archives out of transcript/usage/session-discovery classifiers and provide real Gateway reset proof.
