## Summary

Keep open. The PR addresses a real gap in current main, but the proposed `agentId` schema text is incomplete because current `sessions_send` also supports `agentId` as a direct configured-agent target, and the external contributor has not supplied after-fix real behavior proof.

## What This Changes

The PR adds `sessions_send` target-selection bullets to the session-tool docs and TypeBox field descriptions for `sessionKey`, `label`, and `agentId`.

## Best Possible Solution

Consolidate schema descriptions, tool description, docs, runtime precedence, and focused tests around the current selector contract: `sessionKey` authoritative when present, `label` only when no key is available, and `agentId` usable both for direct configured-agent targeting and label disambiguation.

## Solution Assessment

No as submitted. Schema/docs guidance is the right narrow layer, but the `agentId` description must preserve existing direct targeting and the final wording should align with the open runtime-precedence work.
