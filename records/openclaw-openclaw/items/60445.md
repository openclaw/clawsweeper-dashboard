Keep open. The PR adds `channelData.execApprovalUnavailable` to unavailable exec approval reply payloads, updates the tool-result delivery filter to pass that marker, and adds focused regression tests for the builder and group delivery path.

No, not as-is. The marker approach is the narrow maintainable solution, but it must be applied consistently to the progress-suppression gate and covered by a regression test before merge.
