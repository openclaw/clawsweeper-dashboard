## Summary

Keep open: current main still keeps ordinary chat-visible tool warnings compact by default and does not emit a user-visible retry-success confirmation. The earlier related PR showed a useful direction, but it was closed unmerged after the author concluded the approach needed more work, and the remaining fix needs maintainer policy for safe redaction and channel visibility.

## What This Changes

The issue requests chat-visible tool failure messages that include a safe failure reason and a retry-success confirmation when a later matching tool call succeeds.

## Best Possible Solution

Define allowlisted or redacted tool-failure summaries by tool and channel, then emit retry-success confirmation only for a previously surfaced mutating failure resolved by a matching successful action.

## Reproduction Assessment

Yes, by source inspection. Current main gates ordinary tool-error details behind `/verbose full` or narrow cron timeout handling, and retry success only clears `lastToolError` state.

## Solution Assessment

No, not as raw error passthrough. The maintainable solution is sanitized or allowlisted failure summaries plus explicit retry-confirmation rules for matching surfaced mutating failures.
