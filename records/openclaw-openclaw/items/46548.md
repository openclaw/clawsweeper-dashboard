## Summary

Keep open. Current main still emits ordinary non-verbose tool-warning chat text without the failure reason and clears matching retry failures only in runner state; the closed linked PR did not land, and the currently open edit-mismatch PR is only a narrow partial fix.

## What This Changes

The issue requests chat-visible tool failure messages that include a safe failure reason and a follow-up confirmation when a retry succeeds.

## Best Possible Solution

Define sanitized or allowlisted failure summaries by tool and channel, then emit retry-success confirmation only when a previously surfaced mutating failure is resolved by the matching successful action.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: build a normal non-verbose payload with a mutating `lastToolError` and the reason is omitted, then follow a failed edit through a matching successful edit where state clears without a chat payload.

## Solution Assessment

No current implementation fully solves it. The safer direction is sanitized summaries plus explicit retry-success rules; the closed raw-reason PR and the open edit-mismatch PR are partial and do not settle the broader channel/redaction policy.
