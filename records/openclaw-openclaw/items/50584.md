Full review comments:

- **[P1] Keep reset memory capture off the reply path:** `src/hooks/bundled/session-memory/handler.ts:97`
  - body: Current main starts session-memory capture in the background so `/new` and `/reset` replies are not delayed by transcript reads, file writes, slugging, or model work. This branch makes the hook itself async and awaits the whole capture path, so reset acknowledgements can block again, especially when synthesis or slug generation runs.
  - confidence: 0.94
- **[P1] Keep model-backed slugging opt-in:** `src/hooks/bundled/session-memory/handler.ts:193`
  - body: Current main only calls the slug LLM when `llmSlug === true`. The branch changes that to `hookConfig?.llmSlug !== false`, so default `/new` and `/reset` captures can send recent transcript content to the configured provider without explicit opt-in.
  - confidence: 0.95
- **[P1] Use an exported fs-safe append API:** `src/hooks/bundled/session-memory/handler.ts:26`
  - body: `appendFileWithinRoot` is not exported by current `src/infra/fs-safe.ts`, so this import will fail after merge. Rebase this code onto the current `root(...)` or `appendRegularFile` safe append surface before adding canonical daily writes.
  - confidence: 0.9
- **[P2] Preserve collision-safe session filenames:** `src/hooks/bundled/session-memory/handler.ts:210`
  - body: The branch writes `${dateStr}-${slug}.md` directly. Current main allocates a suffix when the same timestamp or generated slug already exists, so this would reintroduce silent overwrites for repeated same-minute resets or repeated LLM slugs.
  - confidence: 0.9
