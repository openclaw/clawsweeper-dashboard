## Summary

Keep this PR open: current main still lacks the opt-in synthesis mode and canonical daily append, but the branch should not merge as-is because current main already loads both exact daily memory files and date-prefixed slug files at startup, so this patch would make saved session entries appear twice on the next boot.

## What This Changes

The PR adds opt-in session-memory LLM synthesis, appends saved sessions to `memory/YYYY-MM-DD.md`, updates hook docs/changelog, and adds focused handler tests.

## Best Possible Solution

Decide the memory-loading contract first, then refresh the branch so session-memory either appends canonical entries without duplicating startup context or relies on the existing slug-file startup loader while keeping any accepted synthesis behavior documented and tested.

## Reproduction Assessment

Yes. Source inspection shows current main startup loads both `memory/YYYY-MM-DD.md` and newest `memory/YYYY-MM-DD-*.md`, while the PR writes the same entry to both paths; running the hook and startup prelude would load duplicated notes.

## Solution Assessment

No. The synthesis path is narrow and prior review feedback appears addressed, but the canonical append is no longer the narrowest safe fix now that current main already reads slugged files at startup.

## Review Findings
