## Summary

Keep open: the PR addresses a real Codex/OpenClaw sandbox networking failure and includes usable live-output proof, but the current patch ties the network opt-in to `sandbox.enabled` rather than to the app-server actually running under an external sandbox, which reopens host-network egress for normal stdio Codex turns.

## What This Changes

The PR adds a `workspaceWriteNetworkAccess` runtime flag, passes it into Codex turn sandbox-policy construction, and adds focused tests so OpenClaw-sandboxed Codex turns can opt into workspace-write network access.

## Best Possible Solution

Revise the fix so network delegation is tied to an actual external sandbox policy or app-server execution boundary, while preserving default host `workspace-write` network denial and covering both paths with regression tests.

## Reproduction Assessment

Yes by source inspection and PR live output: current main clamps active OpenClaw sandbox turns to Codex `workspace-write` with `networkAccess: false`, while the PR body shows a hotpatched heartbeat completing after a network opt-in. I did not run the Docker/Codex live path in this read-only review.

## Solution Assessment

No. Enabling `workspaceWrite.networkAccess` from `sandbox.enabled` is not the narrowest safe fix because the stdio app-server is still spawned on the gateway host; use Codex's external-sandbox policy or another gate that proves the app-server/network path is actually sandbox-owned.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Gate Codex network on a real external sandbox:** `extensions/codex/src/app-server/run-attempt.ts:429`
  - body: This flag is set whenever `resolveSandboxContext` returns enabled, but the default stdio app-server still starts via `child_process.spawn` on the gateway host. Passing the flag through makes native Codex `workspaceWrite` commands network-enabled on the host for ordinary sandboxed sessions, so egress is no longer owned by the Docker/SSH sandbox policy. Gate this on an actual external sandbox execution path, or use Codex's external-sandbox policy instead of turning on host workspace-write networking.
  - confidence: 0.91
