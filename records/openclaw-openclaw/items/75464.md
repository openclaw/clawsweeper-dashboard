## Summary

Keep open. Current main still has the RTT preflight regression this PR targets, but the PR head still has blocking credential-boundary and lease-cleanup bugs before it is safe to merge.

## What This Changes

The PR defaults `pnpm rtt` to the current checkout and adds Convex-aware credential handling for package RTT runs, with a Docker credential helper, tests, and changelog entry.

## Best Possible Solution

Preserve the RTT wrapper fix, but split install from credential-bearing runtime execution like the existing live lane and guarantee Convex leases are released after every successful acquire attempt.

## Reproduction Assessment

Yes. The original bugs are visible from current-main preflight code, and the PR blockers reproduce by static control-flow inspection of the PR head without live Telegram or Docker credentials.

## Solution Assessment

No. The PR addresses the requested preflight shape, but the safest maintainable solution needs a stricter install/runtime secret boundary and acquire-time lease cleanup before merge.
