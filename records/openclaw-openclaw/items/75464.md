## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep Convex secrets out of the install phase:** `scripts/e2e/npm-telegram-rtt-docker.sh:153-156`
  - body: The PR appends Convex broker secrets to the same `docker_env` used for the container that runs `npm install -g "$install_source"`. Package install lifecycle code can read those secrets, so split installation from the credential-bearing runtime phase like the existing live lane.
  - confidence: 0.9
- **[P2] Release leases when RTT payload validation fails:** `scripts/e2e/npm-telegram-rtt-credentials.ts:347-349`
  - body: After a broker acquire succeeds, `parseTelegramPayload(response.payload)` runs before the lease is returned to `runCommand` and before its release `finally` is active. A malformed Telegram payload can leave the pooled credential leased until TTL.
  - confidence: 0.87
