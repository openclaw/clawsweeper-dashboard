## Summary

Keep open. The PR fixes the default harness-root bug and moves the RTT wrapper preflight in the right direction, but the Convex credential fix is incomplete because the wrapper still launches the dedicated RTT Docker/config/driver path, and that path requires raw Telegram env vars instead of acquiring Convex credentials.

## What This Changes

The PR updates RTT wrapper preflight to accept Convex QA credential env, defaults `pnpm rtt` harness root to the current checkout, and adds focused regression tests for both behaviors.

## Best Possible Solution

Keep the PR open and update it so the full `pnpm rtt` execution path supports the same Convex credential contract it accepts at preflight, while preserving raw-env operation and the `--harness-root` override.

## Reproduction Assessment

Yes. A Convex-only environment would pass the PR's new wrapper preflight, then `runHarness` invokes `scripts/e2e/npm-telegram-rtt-docker.sh`, which still exits on missing raw Telegram env vars before Docker work proceeds.

## Solution Assessment

No. Changing only `assertRequiredEnv` is not the best fix because it accepts a credential mode the launched RTT Docker/driver path cannot use; the safer repair is to align that downstream path with the existing live lane Convex lease contract or delegate to it.
