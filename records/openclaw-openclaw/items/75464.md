## Summary

Keep open: current main still has the RTT wrapper regressions, and this PR fixes the intended preflight shape but introduces two blocking issues around Convex secret exposure and lease cleanup.

## What This Changes

The PR defaults `pnpm rtt` to the current checkout and adds Convex-backed credential leasing, Docker wiring, tests, and a changelog entry for package Telegram RTT runs.

## Best Possible Solution

Land a narrow RTT wrapper fix that mirrors the live lane’s install-before-runtime-secret split, guarantees acquired lease release on every post-acquire failure path, and keeps focused regression coverage plus credential-bearing validation before merge.

## Reproduction Assessment

Yes. Current main source still requires raw Telegram env in the RTT preflight and defaults `pnpm rtt` to the old home-relative harness root; the PR-head blockers are also source-reproducible from the Docker wrapper and credential helper control flow.

## Solution Assessment

No. The PR addresses the original preflight regressions, but the maintainable fix needs to split package installation from credential-bearing runtime execution and release any acquired Convex lease if payload validation fails.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep Convex secrets out of the install phase:** `scripts/e2e/npm-telegram-rtt-docker.sh:155-156`
  - body: The PR adds Convex broker secrets to `docker_env` and passes that same env into the container that runs `npm install -g "$install_source"`. Package install lifecycle scripts can read those broker credentials, so split installation from the credential-bearing runtime phase like the live package lane.
  - confidence: 0.9
- **[P2] Release the lease when payload validation fails:** `scripts/e2e/npm-telegram-rtt-credentials.ts:347-349`
  - body: After a successful broker acquire, `parseTelegramPayload(response.payload)` runs before `runCommand` has a lease and before its release `finally` is active. If the broker returns malformed Telegram payload data, the credential remains leased until TTL; release the acquired lease on this failure path.
  - confidence: 0.87
