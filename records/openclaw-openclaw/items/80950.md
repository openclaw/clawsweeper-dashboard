## Summary

Keep open: this is a protected maintainer security PR with useful direction, but the patch still has blocking redaction-contract issues in the central append helper before it should merge.

## What This Changes

The branch adds a shared transcript redactor and routes JSONL append, gateway injection, Telegram transcript mirroring, delivery mirror updates, and guarded tool-result persistence through the redacted path with focused tests.

## Best Possible Solution

Land a narrowed security patch that applies the shared redaction logic to every append payload, exposes the existing OpenClawConfig redaction contract on the exported append helper, and preserves redacted transcript update emission with focused regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a non-role object passed to the exported append helper takes the PR fallback at src/config/sessions/transcript-append.ts:299 and is JSONL-stringified without the redaction that current main applies unconditionally.

## Solution Assessment

No. Centralizing transcript redaction is the right direction, but the best fix should keep append-layer redaction broad for unknown payloads and widen the exported config type so callers can actually use the existing logging redaction settings.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep redacting non-role append payloads:** `src/config/sessions/transcript-append.ts:297-299`
  - body: The append helper still accepts `message: unknown` and is exported through the agent harness runtime SDK. Current main sends every value through `redactSecrets` before writing JSONL, but this branch only redacts objects with a string `role`; a caller appending `{ content: "sk-..." }` or another custom transcript payload will hit the fallback and persist plaintext. Make the shared redactor accept `unknown` and apply it unconditionally, or keep a redacted fallback for non-AgentMessage values.
  - confidence: 0.87
- **[P2] Expose the redaction config on the append API:** `src/config/sessions/transcript-append.ts:241`
  - body: `appendSessionTranscriptMessage` is a public SDK export, but its params still type `config` as `SessionWriteLockAcquireTimeoutConfig`. That hides and rejects object-literal `logging.redactSensitive` / `logging.redactPatterns` usage even though this patch now depends on those fields in the central append path. Use `OpenClawConfig` or an explicit intersection here, and refresh the SDK API baseline if the public type changes.
  - confidence: 0.84
