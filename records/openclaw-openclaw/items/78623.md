## Summary

Keep open: the contributor has now added sufficient visual proof for the UI path, but the PR still has blocking correctness issues in the picker helper around label formatting and actual parent-child ordering.

## What This Changes

The PR adds Control UI session-picker logic, tests, and changelog text to mark spawned subagent sessions as child options with a `└─ ` prefix when their parent row is present.

## Best Possible Solution

Keep the fix in the Control UI picker, but build options from lineage so visible children render immediately under their visible parent and add regression coverage for both label formatting and child-before-parent input order.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main receives lineage fields in `sessions.list` rows but the picker ignores them and renders a flat row-order list. I did not run a live Control UI session in this read-only review.

## Solution Assessment

No. The PR is in the right helper, but it should align the child label with existing subagent prefix behavior and explicitly order/group children under their parent instead of relying on gateway recency order.
