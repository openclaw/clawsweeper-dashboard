## Summary

Keep open: the PR targets the right Control UI surface, but the proposed patch has blocking correctness issues and the contributor has not provided after-fix real behavior proof.

## What This Changes

The PR adds Control UI session-picker logic, tests, and a changelog entry to mark spawned subagent sessions with a `└─ ` prefix and parent metadata when their parent is present.

## Best Possible Solution

Keep the fix in the Control UI picker, but build options from parent-child lineage so children render immediately under their parent with labels that do not duplicate the existing `Subagent:` prefix.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path for current main: `sessions.list` rows expose lineage, but the picker ignores it and appends a flat recency-ordered option list. I did not run a live Control UI session in this read-only review.

## Solution Assessment

No. The PR is aimed at the right helper, but it should account for existing subagent label prefixes and reorder or group child options under their parent before it can be considered the best fix.

## Review Findings
