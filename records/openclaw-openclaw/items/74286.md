## Summary

Keep open. Current main still builds the announce rewrite turn from child task-completion data and a rewrite instruction, then forwards that same trigger through active and direct delivery paths without an explicit bounded requester-conversation handoff; #74913 addresses a related delivered-message mirror path but not this parent rewrite context gap.

## What This Changes

Ensure subagent and cron announce completion rewrites have enough bounded parent/requester context to produce on-topic replies.

## Best Possible Solution

Add an explicit, bounded, sanitized requester-context handoff for announce rewrite turns, with additive event/schema support and regression coverage across subagent, cron, active steered, direct/dormant, and nested requester paths.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists on current main: construct a subagent or cron completion announce and inspect that triggerMessage is generated from task_completion fields plus the rewrite instruction, then forwarded unchanged by active and direct delivery paths.

## Solution Assessment

Not exactly as proposed. Passing raw last-N transcript text would be risky; the maintainable fix is a bounded requester-context summary or safe requester-session context reference that excludes raw child traces and untrusted arbitrary transcript splices.
