## Summary

Current main still matches the reported failure mode: the announce rewrite turn is built from child completion data and a rewrite instruction, then forwarded unchanged through active and direct requester delivery without an explicit bounded requester-conversation handoff. The related #74913 work helps the later transcript-mirroring/follow-up half, but it does not fully solve initial rewrite grounding.

## What This Changes

Ensure subagent and cron announce rewrite turns receive enough bounded requester context to produce completion replies grounded in the active parent conversation.

## Best Possible Solution

Add an explicit, bounded requester-context handoff for announce rewrite turns while keeping child results untrusted data. The contract should cover subagent, cron, active steered, direct/dormant, and nested-subagent paths, with additive schema/types, docs, and regression tests that preserve existing idempotency and fallback behavior.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: construct a subagent or cron completion announce and observe that the trigger is generated from task_completion fields plus the rewrite instruction, then forwarded unchanged by active and direct delivery paths.

## Solution Assessment

No for the literal last-N transcript splice suggested in the report; that is too broad. The maintainable solution is a narrow bounded requester-context handoff or context reference, with #74913 handled separately for delivered-message transcript mirroring.
