## Summary

Keep open. Current main still builds announce rewrite turns from the child completion event plus a rewrite instruction, and the active/direct delivery paths forward that trigger without an explicit bounded requester-conversation handoff.

## What This Changes

The issue requests that sub-agent and cron `delivery.mode: "announce"` completions include enough parent-session context for the parent rewrite reply to stay relevant.

## Best Possible Solution

Add an explicit bounded requester-context handoff for announce rewrite turns, with typed schema coverage and regression tests for active steered, dormant/direct, cron, and nested requester paths.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: construct a subagent or cron completion announce and observe that the generated trigger contains only `task_completion` fields plus the rewrite instruction before delivery forwards it unchanged.

## Solution Assessment

Yes. The narrow maintainable fix is to pass a sanitized requester-context summary or typed session-context reference into the announce rewrite path, not raw child traces or an arbitrary transcript splice.
