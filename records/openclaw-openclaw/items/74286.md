## Summary

Keep open. Current main still builds the announce rewrite turn from child completion data plus a rewrite instruction, and the active-session/direct-delivery paths forward that same trigger without an explicit bounded requester-conversation handoff.

## What This Changes

The issue asks OpenClaw to keep sub-agent and cron announce completion replies grounded in the parent/requester conversation context.

## Best Possible Solution

Add an explicit bounded requester-context handoff for announce rewrite turns, with protocol/schema coverage and regression tests across subagent, cron, active steered, dormant/direct, and nested requester paths.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: construct a subagent or cron completion announce and observe that the generated trigger contains task_completion fields plus the rewrite instruction, while delivery forwards that trigger unchanged.

## Solution Assessment

Yes. The narrow maintainable fix is to pass a sanitized requester-context summary or typed session-context reference into the announce rewrite path, not raw child traces or an arbitrary transcript splice.
