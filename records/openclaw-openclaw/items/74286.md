## Summary

Keep open. Current main still builds the announce rewrite turn from child completion metadata plus a rewrite instruction, then forwards that trigger without any explicit bounded requester-conversation context.

## What This Changes

The issue asks OpenClaw to include enough parent/requester conversation context when subagent or cron announce completions are rewritten for user delivery.

## Best Possible Solution

Add a typed, sanitized, size-bounded requester-context summary or reference to the announce handoff, then update schema/docs/tests so active, queued, direct, cron, and nested paths use it without raw transcript splicing.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: construct a subagent or cron completion announce and inspect that the generated trigger contains task_completion fields plus the rewrite instruction while delivery forwards that trigger unchanged.

## Solution Assessment

Yes. The issue's suggested direction is maintainable if implemented as an explicit bounded requester-context contract, not an arbitrary transcript append or child-trace leak.
