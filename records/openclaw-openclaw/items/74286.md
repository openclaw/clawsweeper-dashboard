## Summary

Keep open. Current main still builds the announce rewrite turn from child completion metadata plus a rewrite instruction, with no explicit bounded requester-conversation context in the typed event or forwarded trigger, and the latest report says the same failure class still reproduces on v2026.5.2.

## What This Changes

The issue asks OpenClaw to keep subagent and cron `announce` completion rewrites grounded in the requester session context instead of letting child completion events drive out-of-context parent replies.

## Best Possible Solution

Add a typed, sanitized, size-bounded requester-context handoff and stale-turn ordering guard for announce rewrite turns, with protocol/schema coverage and regression tests for subagent, cron, active steered, dormant/direct, and nested requester paths.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists on current main: construct a subagent or cron completion announce and inspect that the generated trigger contains child task-completion fields plus the rewrite instruction while delivery forwards that same trigger without requester-conversation context.

## Solution Assessment

Yes. The narrow maintainable fix is an explicit bounded requester-context summary or typed session-context reference in the announce rewrite path, not delivery mirroring alone or an arbitrary raw transcript splice.
