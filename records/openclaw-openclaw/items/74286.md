## Summary

Keep open. Current main still builds the announce rewrite turn from child completion metadata plus a rewrite instruction, then forwards that same trigger to the requester without a bounded requester-conversation handoff.

## What This Changes

The issue asks for sub-agent and cron announce completion rewrites to include bounded parent/requester conversation context so generated parent replies stay relevant.

## Best Possible Solution

Add a typed, sanitized, size-bounded requester-context handoff to announce rewrite turns, with protocol/schema coverage and regression tests for subagent, cron, active steered, dormant/direct, and nested requester paths.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: construct a subagent or cron completion announce and inspect that the generated trigger contains task_completion fields plus the rewrite instruction while delivery forwards that trigger unchanged.

## Solution Assessment

Yes. The narrow maintainable fix is an explicit bounded requester-context summary or typed session-context reference in the announce rewrite path, not raw child traces or an arbitrary transcript splice.
