## Summary

Keep open. Current main still routes announce-mode child completions through a requester rewrite turn built from child completion metadata and a rewrite instruction, while the related open work only covers adjacent delivery/mirroring problems rather than the bounded requester-context handoff reported here.

## What This Changes

The issue reports that announce-mode subagent or cron completions can trigger requester replies without enough requester-session context, causing out-of-context visible responses.

## Best Possible Solution

Add a sanitized, size-bounded requester-context handoff or equivalent stale-announce suppression to the announce rewrite path while preserving inter-session provenance and existing channel routing semantics.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists on current main: build a subagent or cron `task_completion` announce and inspect that the requester rewrite trigger is child-result metadata plus the rewrite instruction, with no typed bounded requester-context field.

## Solution Assessment

Yes. The narrow maintainable fix is in the core announce/internal-event path, not a broad channel routing rewrite or a raw transcript splice; #74913 can help follow-up grounding but does not fully solve this report.
