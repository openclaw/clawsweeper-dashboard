## Summary

Keep open. Current main still constructs subagent/cron announce rewrite turns from child completion metadata and a rewrite instruction, with no typed bounded requester-context handoff; the latest issue comment also reports the same class reproducing on v2026.5.2.

## What This Changes

The issue asks OpenClaw to keep subagent and cron completion announce replies grounded in the requester session context instead of generating a parent-session reply from only the child result.

## Best Possible Solution

Add a typed, sanitized, size-bounded requester-context handoff for announce rewrite turns, with schema coverage and regression tests across subagent, cron, active steered, dormant/direct, nested requester, and stale direct-user-message paths.

## Reproduction Assessment

Yes. Source inspection on current main shows a high-confidence reproduction path: construct a subagent or cron completion announce and inspect that the trigger contains task_completion result/stats/replyInstruction fields while delivery forwards that trigger without bounded requester conversation context.

## Solution Assessment

Yes, with guardrails. The narrow maintainable direction is an explicit bounded requester-context handoff or hidden runtime-event path for announce rewrites, not raw child traces or an arbitrary requester transcript splice.

The observable bug is that subagent/cron completion announce rewrite turns are built from child completion metadata plus a rewrite instruction and delivered to the requester without a typed, bounded view of requester conversation context, so the requester can generate out-of-context replies. Add the smallest maintainable fix in the agent/subagent announce path: introduce a sanitized, size-bounded requester-context handoff or hidden runtime-event mechanism for announce rewrites; update the AgentInternalEvent protocol/schema if adding fields; preserve provider compatibility and do not raw-splice full requester or child transcripts. Cover active steered requester delivery, dormant/direct delivery, nested requester sessions, cron/subagent paths, and the case where a newer direct user message should not be overridden by a stale announce. Coordinate with but do not depend on #74913; it addresses delivery mirroring, not this rewrite-context issue. Add a user-facing CHANGELOG.md fix entry unless the final patch is tests-only.
