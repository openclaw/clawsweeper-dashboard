## Summary

Keep open. Current main still builds the announce rewrite turn from child completion metadata plus a rewrite instruction, then forwards that trigger without an explicit bounded requester-conversation handoff; #74913 may help the delivered-message mirror/follow-up half, but it does not cover this parent rewrite context gap.

## What This Changes

The issue requests that subagent and cron announce rewrites include bounded parent session context so completion replies stay relevant to the active conversation.

## Best Possible Solution

Add a typed, sanitized, size-bounded requester-context handoff to announce rewrite turns, with additive schema/docs coverage and regression tests across subagent, cron, active steered, dormant/direct, and nested requester paths.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: construct a subagent or cron completion announce and inspect that the generated trigger contains task_completion fields plus the rewrite instruction while delivery forwards that trigger unchanged.

## Solution Assessment

Yes, if implemented as an explicit bounded requester-context summary or typed session-context reference. A raw transcript splice or child-trace forwarding would be less maintainable and risk leaking unrelated context.
