## Summary

Keep open. Current main still constructs announce rewrites from child completion data plus a rewrite instruction and forwards that trigger without an explicit bounded requester-session context, while the related #74913 work only covers the delivered-message mirror half.

## What This Changes

The issue asks OpenClaw to keep sub-agent and cron `delivery.mode: "announce"` replies grounded in the parent/requester conversation context.

## Best Possible Solution

Add an additive, typed requester-context handoff for announce rewrite turns, with formatting/protocol/docs coverage and focused tests across subagent, cron, active steered, direct, and nested requester paths.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: construct a subagent or cron completion announce and the generated trigger contains child task-completion fields plus the rewrite instruction, while delivery forwards that trigger unchanged.

## Solution Assessment

Yes. The narrow maintainable solution is a bounded requester-context summary or typed session-context reference in the announce rewrite path, not raw child traces or an arbitrary parent transcript splice.
