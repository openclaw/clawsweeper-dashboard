## Summary

Keep open. Current main still constructs the announce rewrite turn from child completion metadata, result text, stats, and a rewrite instruction, then forwards that same trigger through active-session and direct-delivery paths without an explicit bounded requester-conversation handoff.

## What This Changes

Ensure subagent and cron announce completion rewrites include enough bounded requester context for parent-session replies to stay grounded in the ongoing conversation.

## Best Possible Solution

Add a bounded, sanitized requester-context handoff for announce rewrite turns through the typed internal event/schema contract, with regression coverage for subagent, cron, active steered, direct/dormant, and nested requester paths.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists on current main: construct a subagent or cron completion announce and observe that triggerMessage is generated only from task_completion fields plus the rewrite instruction, then forwarded unchanged by active and direct delivery paths.

## Solution Assessment

Yes. A narrow announce-handoff repair is the best direction; it should pass bounded requester context or a safe requester-session context reference, not raw last-N transcript text or child traces.
