## Summary

Keep open. Current main has adjacent transcript/display hygiene fixes, but the announce rewrite path still builds a task-completion handoff from child result metadata plus a rewrite instruction without an explicit bounded requester-context handoff.

## What This Changes

The issue asks OpenClaw to prevent sub-agent or cron announce completions from causing parent-session replies that lack the requester conversation context.

## Best Possible Solution

Add a typed, sanitized, size-bounded requester-context handoff to announce rewrite turns, with regression coverage for active, queued, direct, cron, and nested requester paths.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: construct a subagent or cron task_completion announce and inspect that the generated trigger still contains child completion fields plus the rewrite instruction, without a bounded requester-context field.

## Solution Assessment

Yes. The narrow maintainable fix is an explicit bounded requester-context summary or typed session-context reference in the announce handoff; raw transcript splicing or relying only on unrelated Telegram/cron fixes would leave this bug class exposed.
