## What This Changes

The issue requests deriving or configuring the Codex app-server terminal-idle watchdog from the effective run timeout and surfacing terminal-idle-specific timeout guidance.

## Best Possible Solution

Land or refresh the focused runtime-budget fix, then add terminal-idle-specific timeout guidance that names the watchdog/effective budget without adding a new public config or env surface unless maintainers explicitly choose one.

## Reproduction Assessment

Yes, at source level: current main resolves the terminal-idle watchdog from an explicit override only, does not pass the effective run timeout, and skips terminal-specific prompt timeout outcomes. I did not run a live 30-minute scheduled Codex turn.

## Solution Assessment

Yes: the narrow maintainable fix is to derive the existing internal watchdog from the effective run budget and add terminal-idle-specific guidance. A new config or environment variable should remain optional policy, not the default repair path.
