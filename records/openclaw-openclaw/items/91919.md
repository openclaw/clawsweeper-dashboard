## Summary

Keep open: current main and the latest inspected release still call the shared state database permission helper without an open-time guard, so a chmod failure can abort gateway startup on affected filesystems. The adjacent NFS WAL PR does not cover this chmod failure mode.

## What This Changes

The issue reports that shared state SQLite startup can crash when chmodSync fails on filesystems that do not support POSIX permissions.

## Best Possible Solution

Make state SQLite permission hardening best-effort only when chmod cannot be applied, preserve secure modes where supported, and add focused regression coverage for shared and sibling SQLite open paths.

## Reproduction Assessment

Yes, from source: make `chmodSync` throw while opening the default shared state database and `openOpenClawStateDatabase` will propagate before the SQLite handle is returned. I did not run a live unsupported-filesystem repro in this read-only pass.

## Solution Assessment

Yes, a narrow best-effort chmod hardening fix is the maintainable path; adding a new env var or relocating only the SQLite path is unnecessary for the reported crash.
