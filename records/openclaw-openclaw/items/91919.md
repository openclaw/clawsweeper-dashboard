## Summary

Keep open: current main and the latest release still call the shared state database permission helper without an open-time guard, so a chmod failure can abort gateway startup on affected filesystems. The related NFS WAL PR covers a different SQLite-on-NFS failure mode and does not fix this chmod path.

## What This Changes

The issue reports that shared state database startup can crash when open-time chmod calls fail on filesystems without POSIX permission support.

## Best Possible Solution

Make startup permission tightening best-effort for database opens while preserving hard failures for path-escape validation, and cover chmod failures with focused state DB regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: make `chmodSync` throw from `ensureOpenClawStatePermissions` during `openOpenClawStateDatabase`, and the exception is unhandled before SQLite opens.

## Solution Assessment

Yes. The narrow maintainable fix is to keep security validation hard while treating POSIX mode tightening as best-effort at open time, with sibling per-agent coverage checked before landing.

## Repair Work Prompt

Fix the shared state database startup crash reported in https://github.com/openclaw/openclaw/issues/91919. Current `openOpenClawStateDatabase` calls `ensureOpenClawStatePermissions` before SQLite open and after schema setup; if `chmodSync` throws on a chmod-hostile filesystem, gateway startup aborts. Preserve hard failures for path-escape validation, but make POSIX permission tightening best-effort during database open in a way consistent with the existing post-write best-effort policy. Check the sibling per-agent database opener in `src/state/openclaw-agent-db.ts` and either include the same owner-boundary fix there or explain with code/tests why it is unaffected. Add focused Vitest regression coverage by mocking or stubbing chmod failure around open-time permission enforcement; do not add new env vars or config options unless maintainers explicitly ask. Do not touch CHANGELOG.md.
