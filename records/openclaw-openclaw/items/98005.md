## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Resolve child commands through the spawn resolver:** `src/plugin-sdk/json-rpc-plugin.ts:445-449`
  - body: This public SDK helper starts the JSON-RPC runtime with raw `spawn(command, args)`, while existing OpenClaw process surfaces use `resolveWindowsSpawnProgram` and `materializeWindowsSpawnProgram` for Windows wrapper commands and shell-fallback policy. Without that, plugin authors can get a binding that works on Unix but fails for npm/pnpm `.cmd` wrappers on Windows.
  - confidence: 0.88
- **[P2] Make cleanup actually retire the child process:** `src/plugin-sdk/json-rpc-plugin.ts:394-395`
  - body: `stop()` only sends `child.kill()` and clears the pointer, so cleanup can report success while a runtime that ignores the signal, or a descendant it spawned, keeps running. Since the docs promise the child is stopped on disable/restart and `gateway_stop`, cleanup should wait for or forcefully retire the process tree before treating the runtime as stopped.
  - confidence: 0.84
