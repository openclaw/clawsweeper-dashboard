## Summary

Keep open. Current main still has the cron-tool path that attaches the caller chat session key to a normalized cron.add job, and https://github.com/openclaw/openclaw/pull/81376 is an open same-author fix PR with a closing reference, so the issue should remain open until that PR is reviewed and merged or closed.

## What This Changes

The issue requests preventing implicit main/systemEvent cron jobs from inheriting a live chat session key while preserving explicit session keys and isolated agentTurn delivery context.

## Best Possible Solution

Land a narrow fix that only suppresses implicit sessionKey stamping for main/systemEvent cron jobs, keeps explicit sessionKey and isolated agentTurn behavior intact, and includes focused regression coverage.

## Reproduction Assessment

Yes, by source inspection: create a cron tool with a chat agentSessionKey, add a systemEvent job without an explicit sessionKey, and current main normalizes it to main then stamps the caller sessionKey before cron.add. I did not run tests or live restart proof because this review is read-only.

## Solution Assessment

Yes in direction: the linked PR’s tool-level guard is the narrowest maintainable fix if cleaned up, because it leaves explicit session binding and isolated agentTurn announce context alone. The issue should stay open until that implementation path is merged or rejected.
