## Summary

Keep open: the report is source-reproducible on current main and is already paired with an open closing PR, so the issue should remain open until that PR is reviewed and merged or explicitly closed.

## What This Changes

The issue reports that cron jobs created from live chat can inherit the chat session key as implicit main/systemEvent work, causing restart catch-up jobs to occupy the same live chat lane before real inbound replies.

## Best Possible Solution

Land a focused fix that prevents implicit main/systemEvent cron jobs from adopting a live chat session key while preserving explicit session keys and isolated agentTurn delivery context.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: add a systemEvent cron job from a tool context with `agentSessionKey`; current main stamps that caller key onto the job, and the existing test asserts that behavior.

## Solution Assessment

Yes. The proposed direction is the narrowest maintainable fix: guard only implicit main/systemEvent session-key stamping, preserve explicit `job.sessionKey`, and keep isolated agentTurn context behavior intact.
