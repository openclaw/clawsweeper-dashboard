## Summary

Keep open. Current main still has the cron-tool path that stamps the caller session key onto implicit main/systemEvent jobs, and the open closing PR has not merged.

## What This Changes

The issue asks for cron.add to avoid binding implicit main/systemEvent cron jobs to the caller's live chat session key while preserving explicit or isolated agentTurn session context.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/81376 with the scoped cron-tool guard and regression coverage, then close this issue after merge.

## Reproduction Assessment

Yes, at source level: current main defaults systemEvent cron jobs to main, stamps a missing job.sessionKey from the caller session, and then uses that key for main cron enqueue and wake. I did not rerun the full live WhatsApp restart scenario.

## Solution Assessment

Yes, the narrow maintainable fix is to prevent caller sessionKey stamping for implicit main/systemEvent jobs while preserving explicit job.sessionKey and isolated agentTurn context.
