## Summary

Keep open. Current main and v2026.5.12 still have the cron-tool path that stamps the caller session key onto implicit main/systemEvent jobs, and the linked closing pull request remains open and conflicting; this issue should close only after that fix or a replacement merges.

## What This Changes

Prevent cron.add-created implicit main/systemEvent jobs from inheriting the live chat session key while preserving explicit session keys and isolated agentTurn context.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/81376 with a scoped cron-tool normalization fix and focused regression coverage, then close this issue after the fix merges.

## Reproduction Assessment

Yes, at source level: current main defaults systemEvent cron jobs to main, stamps a missing job.sessionKey from the caller session, and then uses that key for main cron enqueue/wake. I did not run the full live WhatsApp restart scenario in this read-only review.

## Solution Assessment

Yes, the linked fix direction is the narrow maintainable solution: guard implicit main/systemEvent jobs from caller sessionKey stamping while preserving explicit job.sessionKey and isolated agentTurn context.
