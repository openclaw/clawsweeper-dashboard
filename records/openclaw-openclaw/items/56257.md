## Summary

Keep open. The PR is still useful as a security-hardening proposal, but it is not merge-ready because the new audit check can miss real installed plugins, can misreport disabled plugins, lacks real behavior proof, and changes a public security-boundary manifest contract.

## What This Changes

The branch adds an optional plugin-manifest `capabilities` field, normalization helpers, a legacy-unrestricted plugin audit warning, and a proposal document for phased runtime enforcement.

## Best Possible Solution

Land a smaller secops-approved schema and diagnostics phase only after the audit resolves installed manifest roots correctly, scopes or labels the finding accurately, adds focused regression coverage, rebases, and includes redacted real audit output.

## Reproduction Assessment

Yes for the PR-introduced defects: source inspection shows `pluginDirs` are basenames, `loadPluginManifest` expects a plugin root, and the new loop bypasses enabled-plugin scoping. No runtime command was run because this review is read-only.

## Solution Assessment

No. A declaration-only phase may be the right first step, but this implementation should first fix the audit root/scoping bugs, rebase cleanly, and get explicit security/product approval for the public capability contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Resolve plugin manifests from the extensions root:** `src/security/audit-extra.async.ts:833`
  - body: `pluginDirs` are directory names from `stateDir/extensions`, but this calls `loadPluginManifest(pluginDir, false)`, so installed plugins will usually be resolved relative to the process cwd and skipped. Join the id with `extensionsDir` before loading the manifest so the new security finding can actually report enabled legacy plugins.
  - confidence: 0.94
- **[P2] Restrict legacy capability warnings to loaded plugins:** `src/security/audit-extra.async.ts:831-834`
  - body: The loop scans every discovered extension directory even when plugins are globally disabled, denied, or per-entry disabled, yet the finding says these plugins run in legacy unrestricted mode. Scope this to enabled plugin ids, or relabel it as installed-plugin inventory so disabled plugins are not reported as active security exposure.
  - confidence: 0.88
