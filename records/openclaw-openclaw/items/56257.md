## Summary

Keep open for maintainer review: current main still lacks the proposed plugin capability model, but this external PR changes a security contract, has concrete audit correctness bugs, and lacks real behavior proof.

## What This Changes

The branch adds manifest `capabilities` types and normalization, a `plugins.capabilities.legacy_unrestricted` audit finding, and an RFC for phased plugin capability enforcement.

## Best Possible Solution

Define the capability manifest semantics with security owner approval, implement the audit in `src/security/audit-plugins-trust.ts` against resolved enabled manifest roots, and add focused tests plus public manifest docs before merging.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `listInstalledPluginDirs` returns directory names while `loadPluginManifest` expects a plugin root path, and the PR scans all names rather than the enabled plugin set.

## Solution Assessment

No. A declaration-only phase may be reasonable, but this patch is not the best implementation until it uses resolved enabled plugin manifests in the current trust-audit module and gets security/product approval.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Resolve installed plugin paths before loading manifests:** `src/security/audit-extra.async.ts:833`
  - body: `pluginDirs` are directory names under the extensions root, but `loadPluginManifest` resolves `openclaw.plugin.json` relative to its argument. Passing a bare name can skip real installed plugins, so the new legacy-capabilities warning may never fire for the plugins it is meant to audit.
  - confidence: 0.94
- **[P2] Scope legacy capability warnings to active plugins:** `src/security/audit-extra.async.ts:831`
  - body: The new loop scans every discovered extension directory, including globally disabled, denied, unallowlisted, or per-entry disabled plugins. The finding says those plugins run without declared capabilities, so inactive plugins can produce false security warnings.
  - confidence: 0.88
