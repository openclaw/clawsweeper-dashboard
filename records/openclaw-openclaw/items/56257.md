## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Resolve plugin manifests from the plugin root:** `src/security/audit-extra.async.ts:833`
  - body: `pluginDirs` are names returned from `stateDir/extensions`, while `loadPluginManifest` expects a plugin root path and resolves `openclaw.plugin.json` under that root. Passing `pluginDir` here will look relative to the process cwd and usually skip real installed plugins, so `plugins.capabilities.legacy_unrestricted` can silently fail to report the exposure.
  - confidence: 0.94
- **[P2] Scope legacy capability warnings to enabled plugins:** `src/security/audit-extra.async.ts:831-834`
  - body: The loop scans every discovered directory instead of the enabled plugin ids already computed by this audit path, but the finding says those plugins run unrestricted. With global disable, deny/allow filters, or per-entry disabled config, inactive installs can be reported as active exposure; scope this to enabled plugin ids or relabel it as installed-plugin inventory.
  - confidence: 0.88
