## Summary

Keep open. Current main still does not implement the requested manifest capability contract or audit warning, but this PR is not merge-ready because its audit addition can miss normal installed plugins, can warn for disabled plugins, and now targets a stale pre-split audit file path for a security-sensitive plugin trust surface.

## What This Changes

The PR adds a proposed `capabilities` field and normalizers to plugin manifests, a legacy-unrestricted plugin audit warning, and an RFC for phased capability and budget enforcement.

## Best Possible Solution

Land a maintainer-approved capability manifest contract in the current manifest, docs, tests, and `src/security/audit-plugins-trust.ts`, with legacy-mode auditing limited to enabled plugin roots and future enforcement semantics clearly versioned before users rely on the field.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: an installed plugin under `<stateDir>/extensions/<id>/openclaw.plugin.json` is not found when the PR passes only `<id>` to `loadPluginManifest`, and disabled or denied plugin directories are still scanned because the loop uses all installed directory names.

## Solution Assessment

No. A declaration-only phase may be a reasonable direction, but this patch is not the best implementation until it is rebased to the current audit split, fixes enabled-plugin scoping and path resolution, and gets explicit security-contract review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Join plugin dirs before loading manifests:** `src/security/audit-extra.async.ts:831`
  - body: `pluginDirs` are entry names from the extensions directory, but `loadPluginManifest` treats its argument as a plugin root and resolves `openclaw.plugin.json` below it. Passing the bare name makes the new audit skip normal installed plugins instead of warning, so join each entry with `extensionsDir` before loading.
  - confidence: 0.94
- **[P2] Scope legacy warnings to enabled plugins:** `src/security/audit-extra.async.ts:829`
  - body: This loop scans every installed directory, including plugins disabled globally, denied, absent from `plugins.allow`, or disabled per entry. The finding says those plugins run without declared capabilities, so it should use the enabled plugin set or be relabeled as installed inventory.
  - confidence: 0.88

## Security Review
