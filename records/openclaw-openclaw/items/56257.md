## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Load manifests from the installed plugin root:** `src/security/audit-extra.async.ts:833`
  - body: `pluginDirs` are directory names from the extensions directory, but this calls `loadPluginManifest` with the bare name. That resolves relative to the process CWD, so the new legacy-unrestricted warning can silently miss installed plugins that lack declarations.
  - confidence: 0.96
- **[P2] Restrict the warning to enabled plugins:** `src/security/audit-extra.async.ts:831-833`
  - body: This loop scans every discovered extension directory, so configs with `plugins.enabled=false`, deny entries, or disabled plugin entries can still be reported as plugins that run unrestricted. Scope the check to the enabled plugin ids or relabel it as install inventory.
  - confidence: 0.9
