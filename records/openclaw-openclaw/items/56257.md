## Best Possible Solution

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path for the PR-introduced defects: `pluginDirs` are basenames, `loadPluginManifest` expects a plugin root, and current main computes enabled plugin ids separately from discovered directories.

No. A declaration-only phase may be reasonable, but this branch should first audit resolved enabled manifest roots and get maintainer approval for the public plugin capability contract.
