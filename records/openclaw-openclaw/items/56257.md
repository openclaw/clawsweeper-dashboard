## Summary

Keep open for maintainer review; this is plausible security hardening, but the PR is not mergeable as-is because the new audit signal is wired to the wrong plugin roots/scope and the contributor has not supplied real behavior proof.

## What This Changes

The PR adds a `PluginManifest.capabilities` declaration model, manifest normalizers, a legacy-unrestricted plugin audit warning, and an RFC for phased runtime enforcement.

## Best Possible Solution

Land a maintainer-approved capability contract with audit code that resolves enabled plugin manifest roots, focused tests/docs, and redacted real `openclaw security audit` output.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path for the PR-introduced defects: `pluginDirs` are basenames, `loadPluginManifest` expects a plugin root, and an enabled-plugin set is computed but not used by the new loop.

## Solution Assessment

No. A declaration-only phase may be reasonable, but this branch should first audit resolved enabled manifest roots and get maintainer approval for the public plugin capability contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Resolve plugin manifests from the extensions root:** `src/security/audit-extra.async.ts:833`
  - body: `pluginDirs` are directory names from the extensions directory, but this calls `loadPluginManifest(pluginDir, false)`, whose root argument is used to find `openclaw.plugin.json`. Installed plugins under `<stateDir>/extensions/<id>` will usually be skipped, so the new security warning can miss enabled legacy plugins.
  - confidence: 0.94
- **[P2] Scope legacy capability warnings to enabled plugins:** `src/security/audit-extra.async.ts:831-834`
  - body: The new loop scans every discovered extension directory instead of the enabled plugin ids already computed above. Disabled, denied, or globally disabled plugins can be reported as "run without declared capabilities" even though they are not loaded, creating false-positive security audit output.
  - confidence: 0.88
