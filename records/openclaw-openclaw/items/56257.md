## Summary

Keep open. Current main still does not implement the proposed manifest `capabilities` contract or `plugins.capabilities.legacy_unrestricted` audit finding, so the PR is not obsolete; however, the current patch has concrete audit correctness bugs and needs maintainer security/design review before merge.

## What This Changes

The PR adds an optional `capabilities` field and normalizers to plugin manifests, adds a legacy-unrestricted plugin audit warning, and introduces an RFC for phased capability and budget enforcement.

## Best Possible Solution

Rebase the PR onto current main, get maintainer approval for the capability schema and empty-object semantics, then implement the manifest contract, public docs, focused tests, and the legacy audit inside `src/security/audit-plugins-trust.ts` using joined installed-plugin paths and the enabled plugin set.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: an installed plugin under `<stateDir>/extensions/<id>/openclaw.plugin.json` is not found when the PR passes only `<id>` to `loadPluginManifest`, and a config with plugins disabled would still be scanned because the loop uses all installed directories.

## Solution Assessment

No. A declaration-only capability model may be a reasonable first step, but this patch is not the best implementation until it is rebased, fixes the audit path/enabled-scope bugs, and has explicit security-contract review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Join plugin dirs before loading manifests:** `src/security/audit-extra.async.ts:831`
  - body: `pluginDirs` are entry names from the extensions directory, but `loadPluginManifest` resolves `openclaw.plugin.json` under the `rootDir` argument. Passing the bare name usually searches the process cwd instead of `<stateDir>/extensions/<plugin>`, so installed legacy plugins are skipped and the new audit finding can fail to fire.
  - confidence: 0.94
- **[P2] Scope legacy warnings to enabled plugins:** `src/security/audit-extra.async.ts:829`
  - body: This loop scans every discovered extension directory, including plugins disabled globally, denied, or marked disabled. The finding says those plugins run without declared capabilities, which creates false-positive security warnings for plugins that are not loaded; reuse the enabled plugin set or relabel the check as installed inventory.
  - confidence: 0.88
