## Summary

Keep open: the WhatsApp upload-file failure is source-reproducible, but this patch should not merge as-is because it puts WhatsApp-specific upload lowering into the core runner instead of a plugin-owned or generic channel action seam.

## What This Changes

The PR changes outbound message-action routing to lower WhatsApp `upload-file` calls with path/URL media into `send` plus media, adds regression tests, updates the generated Plugin SDK API hash, and adds a changelog entry.

## Best Possible Solution

Keep the fix, but implement it as a channel-owned compatibility lowering hook or Plugin SDK helper that preserves the caller's original allowlist action and keeps core free of bundled-channel IDs.

## Reproduction Assessment

Yes. Source inspection shows current main resolves WhatsApp, then routes `upload-file` through plugin action dispatch where WhatsApp only supports `react`, so the action is unsupported without a live WhatsApp run.

## Solution Assessment

No. Lowering `upload-file` to `send` is the right direction, but hardcoding `whatsapp` in the core runner is not the maintainable solution; use a plugin-owned or generic action-lowering seam instead.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Move WhatsApp upload lowering out of core:** `src/infra/outbound/message-action-runner.ts:1165-1166`

That prevents any WhatsApp plugin implementation from owning a native or buffer-capable `upload-file` action later, because core intercepts and rejects it before plugin dispatch. Move this lowering behind a generic channel-owned hook or into the WhatsApp plugin surface.
