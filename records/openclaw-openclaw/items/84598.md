## Summary

Keep open: current main and the latest release still count structured memory-wiki claim rows as `claimCount` while claim IDs remain optional, so the reported ambiguity is real and not superseded by a narrower fix.

## What This Changes

The issue asks memory-wiki digest/status wording or fields to distinguish stable claim IDs from idless claim/cache rows.

## Best Possible Solution

Preserve `claimCount` as the compatible structured-row total, then add explicit ID-bearing/idless counts or relabel user-facing copy as claim rows and cover the contract in tests/docs.

## Reproduction Assessment

Yes. Source inspection shows `buildAgentDigest` counts all `page.claims` rows while `claims.jsonl` conditionally omits `id`, so a vault with idless claims will reproduce the mismatch; I did not run compile because this review is read-only.

## Solution Assessment

Unclear as proposed. Counting only ID-bearing claims may be incompatible with the shipped digest contract, so the safer solution is additive fields and clearer wording rather than silently changing `claimCount`.
