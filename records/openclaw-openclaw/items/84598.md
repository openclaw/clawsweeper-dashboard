## Summary

Keep open: current main and v2026.6.11 still count every structured Memory Wiki claim row in `claimCount` while idless rows can be emitted to `claims.jsonl`. The visible palace UI wording is partially fixed, but the machine-facing digest and prompt semantics still need a maintainer compatibility decision.

## What This Changes

The issue asks OpenClaw to clarify Memory Wiki claim metrics so counts do not conflate stable claim IDs with idless structured claim rows.

## Best Possible Solution

Choose a compatible digest contract: keep `claimCount` as row count and add explicit ID-bearing/idless metrics plus prompt/docs wording, or approve a breaking semantic change with migration and docs.

## Reproduction Assessment

Yes, by source inspection. A structured claim with text but no `id` is normalized as a claim, counted by `page.claims.length`, and emitted to `claims.jsonl` without an `id`.

## Solution Assessment

Unclear as a direct semantic change. Reusing `claimCount` for only ID-bearing claims could break stable digest consumers, so additive metrics and clearer wording are the safer maintainable path unless maintainers approve a breaking contract.
