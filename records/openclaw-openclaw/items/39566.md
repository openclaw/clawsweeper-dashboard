## Summary

Keep open: the wildcard topic-default feature is still missing from the active Telegram plugin, but this branch is not mergeable as-is because it changes retired `src/telegram` runtime/tests and lacks required real Telegram proof.

## What This Changes

This PR adds `topics."*"` fallback semantics for Telegram group/direct topic config resolution, docs/type comments, old `src/telegram` tests, and `.secrets.baseline` line metadata.

## Best Possible Solution

Port wildcard topic resolution to the active Telegram plugin resolver, add focused plugin tests for group and DM wildcard/exact precedence, update docs/types, and include redacted live Telegram proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the merge-blocking PR defect: source inspection shows the changed runtime file is absent on current main while the active plugin resolver is exact-only. Not applicable as an end-user bug report because this is an additive feature.

## Solution Assessment

Is this the best way to solve the issue? No: the fallback idea is narrow, but this implementation targets retired runtime files and relies only on synthetic tests. The maintainable path is a plugin-port with focused tests and live Telegram evidence.
