The PR adds `topics["*"]` fallback semantics to the legacy Telegram bot, documents the wildcard topic default, updates Telegram config comments, adds legacy bot tests, and refreshes `.secrets.baseline` line numbers.

## Best Possible Solution

Yes, for the requested behavior gap: current-main source inspection shows the active Telegram plugin resolves only exact topic IDs and that `topicConfig` then drives agent routing and allowlist behavior. This is a new config behavior rather than a broken documented contract.

## Solution Assessment

No. Exact-topic-before-wildcard semantics look like the right direction, but the current branch applies them to removed legacy files and includes unrelated secret-baseline churn; the maintainable fix belongs in the active Telegram plugin with real Telegram proof.
