## Summary

Keep open, but the patch is not correct as-is: the requested wildcard topic default is still useful and not implemented on current main, while this PR changes removed legacy Telegram files and unrelated secret-scanning baseline metadata.

## What This Changes

The PR adds `topics["*"]` wildcard fallback semantics for Telegram group and DM topic configs, updates tests/docs/types, and refreshes `.secrets.baseline`.

## Best Possible Solution

Implement exact-topic-before-wildcard lookup in `extensions/telegram/src/bot-core.ts`, cover group and DM topic fallback in active plugin tests, update docs/types, omit `.secrets.baseline`, and require redacted real Telegram proof before merge.

## Reproduction Assessment

## Solution Assessment

No. The fallback idea may be reasonable, but this PR edits removed legacy paths and unrelated baseline metadata; the maintainable fix is to port the change to `extensions/telegram` with active-plugin tests and real proof.
