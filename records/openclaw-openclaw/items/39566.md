## What This Changes

The PR adds exact-before-wildcard `topics["*"]` fallback handling to the old Telegram bot implementation, adds old-layout Telegram tests, updates Telegram docs/type comments, and refreshes `.secrets.baseline` line numbers.

## Best Possible Solution

Implement the exact-topic-then-`topics["*"]` fallback in the active Telegram plugin resolver for both group and DM topic configs, lock it with plugin tests for `allowFrom`/`agentId` defaults and exact-topic precedence, update user-facing docs/types and changelog, and leave secret-scan baselines unchanged.

Yes. Configure an active Telegram plugin group or DM with only `topics["*"].allowFrom` or `topics["*"].agentId`, send an inbound topic message with an unconfigured `message_thread_id`, and current main resolves `topicConfig` as `undefined` because it only checks the exact thread key.

## Solution Assessment

No. The intended exact-before-wildcard semantics are the right solution, but this PR applies them to obsolete files and bundles unrelated baseline metadata; the safer path is a narrow replacement against `extensions/telegram`.
