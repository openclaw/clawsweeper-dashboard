## Summary

Keep open for normal maintainer review and merge consideration; current main still accepts fractional numeric ACP timeouts by rounding them, while this PR is a narrow contract-aligned fix with sufficient terminal proof and no blocking review findings.

## What This Changes

This PR changes ACP runtime timeout validation to reject fractional numeric `timeoutSeconds` patches and adds a colocated regression test.

## Best Possible Solution

Land the focused validation and regression-test change after required maintainer approval, preserving integer-only timeout semantics across numeric patch and string/config paths.

## Reproduction Assessment

Yes. Current main source shows the numeric timeout helper accepts finite numbers and rounds with `Math.round`, while the PR proof exercises the same module after the fix and shows fractional values rejected.

## Solution Assessment

Yes. Adding the integer check inside `validateRuntimeTimeoutSecondsInput` is the narrowest maintainable fix because both direct numeric patches and parsed string/config values pass through that helper before persistence or backend control handling.
