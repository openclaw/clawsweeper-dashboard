Keep this PR open for normal maintainer review: current main still silently rounds fractional numeric ACP timeout patches, and the branch is a narrow validator/test fix with sufficient terminal proof and no blocking findings.

## What This Changes

The PR makes ACP numeric `timeoutSeconds` runtime option patches reject fractional values and adds a regression test for that case.

## Best Possible Solution

Land the narrow validation and regression-test change after required maintainer review and checks pass, preserving integer-only timeout semantics across numeric patch and string/config paths.

## Reproduction Assessment

Yes. Source inspection of current `origin/main` shows the numeric `timeoutSeconds` patch path rounds finite numbers with `Math.round`, while the string/config path rejects non-integer text.

## Solution Assessment

Yes. Tightening `validateRuntimeTimeoutSecondsInput` is the narrowest maintainable fix because ACP initialization and runtime option updates already pass through that shared helper.
