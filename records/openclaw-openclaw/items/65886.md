## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Kill the shell process group or narrow the cleanup guarantee:** `src/tui/tui-statusline.ts:83-84`
  - body: This runner starts the configured command with `shell: true`, but timeout and stop cleanup still signal only the returned shell process. On Linux, shell descendants can survive killing their parent, so the docs claim that SIGTERM plus SIGKILL prevents leaking processes across ticks is too strong unless the implementation kills the process group/tree.
  - confidence: 0.91
- **[P3] Cap buffered bytes instead of decoded string length:** `src/tui/tui-statusline.ts:105-108`
  - body: The docs and constant describe a 4 KB byte cap, but this code appends decoded UTF-8 text and compares `stdout.length`, which counts JavaScript string code units. Multibyte output can exceed the documented byte cap; cap buffered bytes or document this as a character/code-unit cap.
  - confidence: 0.78
