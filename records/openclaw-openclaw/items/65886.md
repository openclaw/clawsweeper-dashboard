## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Kill the whole status command process tree:** `src/tui/tui-statusline.ts:83-84`
  - body: This spawns the configured command through a shell, but `killChild()` only signals that direct child. Existing shell runners use detached process groups plus `killProcessTree`; without that, a script or shell pipeline can leave descendants alive after timeout/stop, so the documented leak protection is not reliable.
  - confidence: 0.88
- **[P2] Use the TUI config namespace for the status line:** `src/config/zod-schema.ts:421-425`
  - body: This publishes a TUI-only option under generic `ui`, while current main already keeps terminal-only settings under `tui.footer.showRemoteHost`. Shipping `ui.statusLine` would create an inconsistent public config contract; move it under `tui` or use a maintainer-approved footer/plugin seam before regenerating schema/docs.
  - confidence: 0.82
