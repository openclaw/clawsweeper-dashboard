## Summary

Keep open. Current main still does not implement `ui.statusLine`, and the PR remains a useful implementation candidate, but the latest patch still leaves a security-sensitive shell descendant cleanup bug and needs the required changelog entry before merge.

## What This Changes

The PR adds an opt-in `ui.statusLine` config block, TUI command runner and rendering hook, generated config schema/help metadata, TUI docs, and focused lifecycle tests.

## Best Possible Solution

Land the feature only after rebasing and making the command lifecycle either kill the whole spawned process tree/process group reliably or explicitly narrow the contract so shell descendants are not promised to be cleaned up.

## Reproduction Assessment

Yes. A high-confidence reproduction path is a `ui.statusLine.command` that starts a long-lived shell descendant; the PR signals only the spawned shell while Node documents that Linux descendants can survive parent termination.

## Solution Assessment

No, not as-is. The status-line direction is plausible, but the proposed shell-backed cleanup and docs guarantee do not match the dependency contract; process-tree cleanup or a safer non-shell/narrower command contract is needed first.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Kill the shell command tree, not only the shell:** `src/tui/tui-statusline.ts:83-84`
  - body: Because this uses `shell: true`, `killChild()` signals only the shell process returned by `spawn`. Node documents that Linux child processes of child processes can survive killing their parent, so a status command can leak descendants across timeout or TUI exit despite the docs promise.
  - confidence: 0.92
- **[P3] Add the required changelog entry:** `docs/web/tui.md:42`
  - body: This PR adds a user-facing TUI/config feature, but the branch does not update `CHANGELOG.md`. Please add a single-line Unreleased entry with appropriate `Thanks @nanami-he` credit so release notes cover the new status-line command surface.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The diff adds an auto-start trusted local shell command surface, and cleanup can still leave shell descendants running on Linux.

Concerns:

- **[medium] Shell descendants can survive status-line cleanup:** `src/tui/tui-statusline.ts:83`
  - body: `ui.statusLine.command` runs with `shell: true`; timeout and stop paths send SIGTERM/SIGKILL only to the direct shell `ChildProcess`, so descendant commands can continue outside the TUI lifecycle.
