## Summary

Keep open. Current main still lacks a configurable TUI status-line command, and this PR is a relevant implementation candidate, but the latest patch still signals only the direct shell process while using `shell: true`, so shell descendants can survive timeout or TUI exit; it also still needs the required changelog entry.

## What This Changes

The PR adds an opt-in `ui.statusLine` config block, generated config metadata, TUI rendering, docs, and a command runner that periodically executes a shell command and displays stdout.

## Best Possible Solution

Land the feature only after the command lifecycle matches the shell contract, either by reliably terminating the whole process tree/group or by narrowing the command contract away from shell descendants, with corrected docs, focused tests, and a changelog entry.

## Reproduction Assessment

Yes. A high-confidence reproduction path is a `ui.statusLine.command` that starts a long-lived shell descendant, such as a script that backgrounds `sleep`, then triggering timeout or `stop()`; the PR signals only the direct shell while Node documents that shell descendants can survive parent termination on Linux.

## Solution Assessment

No, not as-is. The config/schema/docs/test direction is useful, but the current direct-child SIGTERM/SIGKILL fallback does not satisfy the documented no-leak behavior for a `shell: true` command runner.

## Review Findings
