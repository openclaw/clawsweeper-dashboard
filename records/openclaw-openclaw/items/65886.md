## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Narrow the process-cleanup guarantee:** `docs/web/tui.md:55`
  - body: This line promises that a SIGTERM-ignoring script cannot leak processes, but the runner starts commands with `shell: true` and only signals the returned `ChildProcess`. A script can spawn or background descendants that survive after the shell exits or is killed, so either kill a process group/tree or narrow the documented guarantee.
  - confidence: 0.92
- **[P3] Cap bytes before decoding or change the contract:** `src/tui/tui-statusline.ts:105-108`
  - body: The code and docs describe a 4 KB output cap, but this appends decoded UTF-8 and compares `stdout.length`, which counts JavaScript string code units. Multibyte output can exceed the promised byte cap, so buffer/cap bytes before decoding or document this as a character limit.
  - confidence: 0.82
