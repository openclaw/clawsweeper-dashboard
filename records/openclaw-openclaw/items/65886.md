## Summary

Keep open. Current main does not implement this feature, and the latest PR head still has a merge-blocking process cleanup issue for shell descendants plus a missing changelog entry.

## What This Changes

The PR adds an opt-in `ui.statusLine` config block, generated schema/help/docs metadata, a TUI status-line command runner, TUI layout wiring, and lifecycle tests.

## Best Possible Solution

Land the feature only after using reliable process-tree/process-group cleanup or narrowing the command contract away from shell descendants, keeping generated config artifacts aligned, and adding the user-facing changelog entry.

## Reproduction Assessment

Yes. Configure `ui.statusLine.command` to start a long-lived shell descendant; the PR kills only the shell `ChildProcess`, and Node documents that shell descendants can survive parent termination on Linux.

## Solution Assessment

No, not as-is. The status-line direction is plausible, but the cleanup path needs a process-tree strategy or a safer command contract before this auto-running local command surface is maintainable.

## Review Findings
