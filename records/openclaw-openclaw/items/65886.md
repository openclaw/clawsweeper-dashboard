## Summary

Keep open. Current main still lacks this configurable TUI status-line feature, and the PR remains a relevant implementation candidate, but the latest patch still has a blocking process-tree cleanup problem for the new auto-running local command surface plus a missing changelog entry.

## What This Changes

The PR adds an opt-in `ui.statusLine` config surface that periodically runs a local shell command from the TUI, renders stdout, and updates config schema/help/docs metadata.

## Best Possible Solution

Land the feature only after rebasing, either killing the whole command tree reliably or narrowing the command contract away from shell descendants, keeping generated config artifacts aligned, and adding the required user-facing changelog entry.

## Reproduction Assessment

Yes. A high-confidence reproduction path is a `ui.statusLine.command` that starts a long-lived shell descendant; the PR times out or stops only the direct shell process, while Node documents that shell descendants can survive parent termination on Linux.

## Solution Assessment

No, not as-is. The status-line direction is plausible, but the current shell-backed cleanup does not satisfy the documented no-leak guarantee; a process-group/tree kill strategy or a safer non-shell command contract is needed first.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Kill the spawned command tree, not only the shell:** `src/tui/tui-statusline.ts:83-84`
  - body: Because this runner uses `shell: true`, `child.kill()` and the SIGKILL fallback target the shell process returned by `spawn`, not necessarily the command tree it launched. Node documents that shell descendants can survive killing their parent on Linux, so status scripts can leak work past timeout or TUI exit despite the docs/help promise.
  - confidence: 0.91
- **[P3] Add the required changelog entry:** `docs/web/tui.md:42`
  - body: This PR adds a user-facing TUI/config feature, but the branch does not update `CHANGELOG.md`. Please add a single-line Unreleased entry with appropriate `Thanks @nanami-he` credit so release notes cover the new status-line command surface.
  - confidence: 0.88
