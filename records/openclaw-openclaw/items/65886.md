Full review comments:

- **[P1] Kill the whole status command process tree:** `src/tui/tui-statusline.ts:83-84`
  - body: This spawns the configured command through a shell, but `killChild()` only signals that direct child. Existing shell runners use detached process groups plus `killProcessTree`; without that, a script or shell pipeline can leave descendants alive after timeout/stop, so the documented leak protection is not reliable.
  - confidence: 0.88
