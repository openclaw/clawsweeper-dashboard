## Summary

Keep open. Current main still lacks the requested media credential guidance, and a docs-only clarification matches the existing runtime behavior, but this PR should not merge as-is because the new `models.providers` example omits the required provider `models` array.

## What This Changes

The PR adds a Provider credentials section to `docs/nodes/media-understanding.md` explaining credential sources for `tools.media` provider models and showing a `models.providers`-based Moonshot example.

## Best Possible Solution

Land the docs clarification after the example is made schema-valid: either show only env/auth-profile credential setup, or include a complete `models.providers.<id>` object with the required model catalog for the custom provider path.

## Reproduction Assessment

Yes for the actionable docs defect: statically copying the new provider object into config conflicts with `ModelProviderSchema`, which requires `models`. For the linked setup confusion, the static path is also clear because `tools.media.*.models[]` has no `apiKey` field and runtime credentials resolve through auth profiles, env, or `models.providers`.

## Solution Assessment

No, not as-is. A focused docs update is the right approach, but the new custom-provider example needs to satisfy the current config contract before it can be the recommended fix.
