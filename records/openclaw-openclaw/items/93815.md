## Summary

Keep this PR open for maintainer review, but the current patch is not merge-ready: it reintroduces automatic-mode final suppression from heuristic source-reply/prose evidence, which prior maintainer discussion rejected as message-loss-prone, and it has no real behavior proof beyond a focused unit-test command.

## What This Changes

The PR adds a core dispatch heuristic and tests to suppress short group/channel final status text after a source reply has already been delivered while leaving direct chats and substantive final text visible.

## Best Possible Solution

Keep automatic-mode finals eligible unless a typed terminal source-reply signal with thread identity proves the source reply owns the final answer; otherwise use `message_tool_only` for tool-owned visible replies.

## Solution Assessment

No. The current patch is a plausible duplicate-status mitigation, but the safer and previously requested solution is an explicit terminal source-reply signal with thread identity, not prose/target heuristics in automatic mode.
