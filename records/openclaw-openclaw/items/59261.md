## Summary

Keep this PR open. Current main still lacks Agent Computer hosting coverage, but the proposed docs are not merge-ready because the recommended shortcut is currently a 404 and the CLI instructions do not match Agent Computer's public install and port-publishing contract.

## What This Changes

This PR adds an Agent Computer hosting guide and wires it into install, VPS, platform, FAQ, Mintlify navigation/redirect, and zh-CN glossary docs surfaces.

## Best Possible Solution

Keep the docs contribution, but amend it to match Agent Computer's live public contract: use the official CLI installer, document only supported port publishing flags, and either remove the browser shortcut claim or point it at a live OpenClaw template before merging.

## Reproduction Assessment

Yes. The blocking docs defects reproduce with read-only checks: the shortcut URL returns 404, the published npm package has no CLI bin, and Agent Computer's official docs/OpenAPI do not expose the PR's port flags.

## Solution Assessment

No. A first-party hosting guide is the right surface, but this PR should be corrected against the live Agent Computer docs and API before it is the maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Install the actual Agent Computer CLI:** `docs/install/agentcomputer.md:26`
  - body: The guide tells users to run `npm install -g aicomputer` before `computer login`, but the current `aicomputer` npm package is an SDK with no `bin`, while Agent Computer's own docs install the CLI via `curl -fsSL https://agentcomputer.ai/install.sh | bash`. Fresh users will not get a `computer` command from the documented npm install, so both the quick path and manual path fail before login.
  - confidence: 0.94
- **[P2] Use supported port publishing flags:** `docs/install/agentcomputer.md:33`
  - body: The documented publish command includes `--subdomain openclaw --protocol https`, but Agent Computer's public CLI docs and OpenAPI only expose port publishing with `port`, optional `name`, and `visibility`/`--private`. Unless the CLI has undocumented flags, users will be blocked at the publish step; update the command to the supported form instead of documenting flags outside the public contract.
  - confidence: 0.86
- **[P2] Point the shortcut at a live page:** `docs/install/agentcomputer.md:16`
  - body: The recommended browser shortcut currently redirects to `www.agentcomputer.ai/openclaw` and returns 404. Since the guide presents this as the recommended setup path and the first quick-path step, users hit a dead page before they can create an OpenClaw machine; either use a live template URL or remove the shortcut flow until it exists.
  - confidence: 0.95
