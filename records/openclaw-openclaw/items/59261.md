## Summary

Keep open. Current main still lacks Agent Computer hosting docs, and the PR is useful, but the latest head remains blocked by live-provider mismatches: the recommended shortcut returns 404, the documented npm package does not install a `computer` CLI, and the port-publish command uses flags outside Agent Computer's public CLI/API contract.

## What This Changes

This PR adds an Agent Computer hosting guide and wires it into install, VPS, platforms, FAQ, Mintlify navigation/redirect, and zh-CN glossary docs surfaces.

## Best Possible Solution

Keep the contribution, but amend the guide to match Agent Computer's public contract: use the official CLI installer, remove or replace the dead OpenClaw shortcut until it is live, and document supported port publishing syntax while preserving the gateway token and non-loopback bind guidance.

## Reproduction Assessment

Yes. The blocking docs defects reproduce with read-only checks against current public contracts: the shortcut returns 404, the npm package metadata has no CLI bin, and the public CLI/OpenAPI docs do not expose the PR's port flags.

## Solution Assessment

No. A first-party hosting guide is the right surface, but this PR should be corrected against the live Agent Computer docs and API before it is the narrow maintainable solution.
