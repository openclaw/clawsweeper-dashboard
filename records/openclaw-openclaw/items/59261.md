## Summary

This PR should stay open: the docs target is plausible, but the current guide still conflicts with Agent Computer's public CLI/API contract and omits the OpenClaw public-origin requirement for a non-loopback Control UI flow.

## What This Changes

The PR adds an Agent Computer hosting guide and links it from OpenClaw install, VPS, platforms, FAQ, Mintlify navigation, redirects, and zh-CN glossary docs.

## Best Possible Solution

Revise the guide against Agent Computer's current public CLI/API contract, remove or provider-verify the shortcut/template path, use supported publish syntax, and include the required OpenClaw allowed-origin/auth setup before merge.

## Reproduction Assessment

Yes. The PR head contains the affected instructions, the shortcut returns 404, Agent Computer's public docs/npm/OpenAPI contradict the CLI and publish commands, and current OpenClaw source rejects non-loopback Control UI without allowed origins.

## Solution Assessment

No. A first-party Agent Computer hosting guide may be appropriate, but this branch needs provider-contract corrections and Gateway security setup guidance before it is the maintainable solution.
