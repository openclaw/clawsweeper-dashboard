## Summary

Keep this open. Current main still leaves the reported stale A2UI renderer output failure class reachable during in-place git updates: ignored `vendor/a2ui/renderers/lit/dist` files pass the updater clean check, then the source build runs the A2UI renderer `tsgo` step in the same checkout without cleaning that output first.

## What This Changes

The issue requests that in-place git updates of source installs, especially under `/usr/lib/node_modules/openclaw`, either succeed or detect/clean stale generated A2UI build state before building.

## Best Possible Solution

Clean or fail fast on stale A2UI renderer output before the renderer `tsgo` step in source builds, with narrowly scoped updater recovery messaging only if needed.

## Reproduction Assessment

Yes, by source inspection rather than by mutating the checkout: a mutable git checkout with ignored `vendor/a2ui/renderers/lit/dist` output can pass the updater clean check and then run the A2UI renderer compile in that same tree.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the A2UI source-build path; treating all ignored files as dirty would be broader than this failure requires.
