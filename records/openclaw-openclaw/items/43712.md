## Repair Work Prompt

Fix openclaw/openclaw#43712. Current main can leave ignored stale A2UI renderer output under `vendor/a2ui/renderers/lit/dist` in an in-place source checkout; the git updater clean check does not see ignored files, and `pnpm build` runs `canvas:a2ui:bundle`, which invokes the renderer `tsgo` step without cleaning that output. Make the source-build path clean or fail fast before the renderer compile so stale generated declarations cannot trigger TS5055/TS2742 during update. Keep the fix focused in the A2UI build/update surface, add regression coverage for stale renderer output, add a user-facing changelog entry, and do not broaden updater policy to reject all ignored files unless the build-path fix proves insufficient.
