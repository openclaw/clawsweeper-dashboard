## Summary

Keep this open. Current main still has the same source-update failure class: ignored A2UI renderer output can survive the git clean check, and the build still recompiles A2UI in the same checkout without first cleaning or guarding that output.

## What This Changes

The issue asks OpenClaw to make in-place git checkout updates/builds recover from stale generated A2UI state, especially when the source checkout lives under a global node_modules path.

## Best Possible Solution

Add a focused source-build guard that removes or safely rejects stale A2UI renderer output before renderer compilation, plus a precise recovery message and source-checkout recovery docs where useful.

## Reproduction Assessment

Yes. A high-confidence synthetic reproduction is a mutable git checkout with ignored stale `vendor/a2ui/renderers/lit/dist` output; current main's clean check can pass and `pnpm build` still runs A2UI compilation in that same checkout.

## Solution Assessment

Yes. The narrowest maintainable fix belongs in the A2UI source-build path, with updater/docs messaging only where needed; treating all ignored files as dirty would be broader than this failure requires.
