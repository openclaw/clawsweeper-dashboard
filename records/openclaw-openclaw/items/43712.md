## Summary

Keep this open. Current main still allows stale ignored A2UI renderer output to survive the git-update clean check and reach the source build, which matches the reporter's in-place checkout failure class.

## What This Changes

The issue asks for in-place git updates under `/usr/lib/node_modules/openclaw` to succeed or detect/clean stale generated build state before building.

## Best Possible Solution

Clean or fail fast on stale A2UI renderer output before the renderer `tsgo` step in source builds, with updater recovery messaging only if the build path cannot own the cleanup cleanly.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the A2UI source-build path; broadening the updater to reject all ignored files would be a larger behavior change than this failure requires.
