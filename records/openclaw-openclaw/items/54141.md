## Summary

Keep open: the bug is real and the branch is useful, but the current patch still misclassifies dual-field auth profiles and lacks required after-fix real CLI proof for an external PR.

## What This Changes

The PR adds bare `$VAR` env SecretRef parsing, changes auth-profile audit classification for inline env refs, and adds parser/audit regression tests.

## Best Possible Solution

Keep the parser support, classify authored inline credential values before plaintext fallback while preserving real plaintext findings, add dual-field auth-profile coverage, and require redacted CLI proof.

## Reproduction Assessment

Yes. Source inspection shows current main still misclassifies auth-profile env-looking values, and the PR branch still leaves explicit-ref plus inline env-template profiles on the `PLAINTEXT_FOUND` path because `inlineRef` is null when a sibling ref exists.

## Solution Assessment

No. Adding `$VAR` parsing is the right direction, but the audit path should classify `entry.value` independently when an explicit sibling ref exists instead of relying only on `inlineRef`.
