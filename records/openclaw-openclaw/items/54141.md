## Summary

Keep open: the PR is directionally useful, but the latest diff still misses auth profiles where `keyRef`/`tokenRef` coexists with an inline env ref, and it also lacks the required changelog entry.

## What This Changes

The PR adds bare `$VAR` env SecretRef parsing, skips auth-profile plaintext findings for inline SecretRefs, and adds parser/audit regression tests.

## Best Possible Solution

Update auth audit to classify the authored inline value as an env SecretRef before plaintext fallback even when an explicit sibling ref exists, then add changelog and regression coverage for explicit-ref plus inline-env-ref cases.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main does not parse bare `$VAR`, and auth audit can still fall through to `PLAINTEXT_FOUND`; the PR's remaining dual-field miss follows from `explicitRef` forcing `inlineRef` to null.

## Solution Assessment

No. The shorthand parser addition is appropriate, but the auth audit fix should inspect the inline value independently before plaintext classification rather than relying on `inlineRef` after explicit-ref suppression.
