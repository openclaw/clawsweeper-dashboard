## Summary

Keep open: the PR targets a real secrets-audit bug on current main, but it is not merge-ready because the auth-audit change still misclassifies dual-field profiles that have an explicit ref plus an inline env ref, and the external PR lacks after-fix real behavior proof.

## What This Changes

The PR adds bare `$VAR` SecretRef parsing, skips auth-profile plaintext findings for detected inline env refs, and adds parser/audit regression tests.

## Best Possible Solution

Keep the parser improvement, but classify the authored inline credential value before plaintext fallback, add explicit-ref-plus-inline-env-ref regression coverage, and require redacted CLI proof before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main still misclassifies env-looking auth values, and the PR branch still sends explicit-ref-plus-inline-env-ref profiles to `PLAINTEXT_FOUND`.

## Solution Assessment

No. Adding `$VAR` parsing is reasonable, but relying on `inlineRef` after `resolveSecretInputRef` is not sufficient because explicit refs intentionally suppress `inlineRef`.
