## Summary

Keep open: the PR addresses a real secrets-audit false positive, but the current branch still misses the explicit-ref plus inline env-template case and lacks required real CLI behavior proof.

## What This Changes

The PR adds bare `$VAR` SecretRef parsing, changes auth-profile secrets-audit plaintext classification, and adds parser/audit regression tests.

## Best Possible Solution

Keep the parser support, classify authored env-ref strings before plaintext fallback even when explicit refs exist, preserve real plaintext findings, add dual-field coverage, and require redacted CLI proof.

## Reproduction Assessment

Yes. Source inspection shows current main still misclassifies auth-profile env-looking values, and the PR branch still leaves explicit-ref plus inline env-template profiles on the `PLAINTEXT_FOUND` path because `inlineRef` is null.

## Solution Assessment

No. Adding `$VAR` parsing is useful, but the audit path should classify `entry.value` independently when an explicit sibling ref exists instead of relying only on `inlineRef`.
