## Summary

Keep open: the PR fixes the right current-main bug class, but the branch still misclassifies inline auth env refs when an explicit sibling ref is present, and the external contributor has not provided after-fix real CLI/runtime proof.

## What This Changes

The PR adds bare `$VAR` parsing to env SecretRefs, skips auth-profile plaintext audit findings for inline SecretRefs, and adds parser/audit regression tests.

## Best Possible Solution

Update auth audit to classify the authored inline credential value as an env SecretRef before plaintext fallback, while preserving `PLAINTEXT_FOUND` for real leftover plaintext when explicit `keyRef` or `tokenRef` is set.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main does not parse bare `$VAR`, auth audit falls through to `PLAINTEXT_FOUND`, and the PR's remaining dual-field miss follows from `explicitRef` forcing `inlineRef` to null.

## Solution Assessment

No. The shorthand parser addition is appropriate, but the auth audit fix should inspect the inline value independently before plaintext classification instead of relying only on `inlineRef` after explicit-ref suppression.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Classify inline auth env refs before plaintext fallback:** `src/secrets/audit.ts:303-305`
