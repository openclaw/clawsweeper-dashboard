## Summary

Keep this PR open, but it is not ready to merge: current main still has the secrets-audit false positive, while this branch still misses the explicit-ref plus inline env-template case and lacks after-fix real CLI proof.

## What This Changes

The PR adds bare `$VAR` parsing as an env SecretRef and updates auth-profile secrets-audit behavior/tests to avoid plaintext findings for inline env refs while preserving explicit-ref plaintext findings.

## Best Possible Solution

Keep the parser support if maintainers want `$VAR`, classify the authored auth-profile value before plaintext fallback even when an explicit sibling ref exists, add dual-field coverage, and require redacted CLI proof.

## Reproduction Assessment

Yes. Source inspection shows current main still misclassifies env-looking auth-profile values, and the PR branch still leaves explicit-ref plus inline env-template profiles on the `PLAINTEXT_FOUND` path because `inlineRef` is null when a sibling ref exists.

## Solution Assessment

No. Adding `$VAR` parsing is useful, but the audit path should classify `entry.value` independently before plaintext fallback instead of relying only on `inlineRef`.
