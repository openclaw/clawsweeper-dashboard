## Summary

Keep open: the PR addresses a real current-main secrets-audit false positive, but the current head still misses a dual-field auth-profile case and lacks contributor real behavior proof.

## What This Changes

The PR adds bare `$VAR` SecretRef parsing, changes auth-profile audit classification for inline env refs, and adds parser/audit regression tests.

## Best Possible Solution

Keep the parser improvement, classify the authored inline credential value before plaintext fallback, add explicit-ref-plus-inline-env-ref regression coverage, and require redacted CLI proof before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main still misclassifies env-looking auth values, and the PR branch still sends explicit-ref-plus-inline-env-ref profiles to `PLAINTEXT_FOUND`.

## Solution Assessment

No. Adding `$VAR` parsing is the right direction, but relying on `inlineRef` after `resolveSecretInputRef` is not sufficient because explicit refs intentionally suppress `inlineRef`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Classify explicit-ref env values before plaintext fallback:** `src/secrets/audit.ts:303-305`
