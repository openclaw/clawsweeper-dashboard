## Summary

This PR targets a real auth-profile audit false positive, but it is not ready to merge because the proposed audit guard still misses dual-field auth profiles and the PR body only reports unit/integration tests rather than after-fix CLI behavior proof.

## What This Changes

The PR adds `$VAR` shorthand parsing for env SecretRefs, skips auth-profile plaintext findings for detected inline env refs, and adds parser/audit regression tests.

## Best Possible Solution

Update auth audit to classify the authored inline credential value as an env SecretRef before plaintext fallback, while keeping `PLAINTEXT_FOUND` for real leftover plaintext alongside explicit refs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main does not parse bare `$VAR`, and the PR's remaining dual-field miss follows from `resolveSecretInputRef` forcing `inlineRef` to null when an explicit ref is present.

## Solution Assessment

No. Adding shorthand parsing is appropriate, but the auth audit fix should inspect the authored inline value independently before plaintext classification instead of relying on `inlineRef` after explicit-ref suppression.
