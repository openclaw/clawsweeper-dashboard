## Summary

Keep open: this PR targets a valid current-main secrets-audit false positive, but the branch still misses dual-field auth profiles because `inlineRef` is suppressed when an explicit ref exists, and the external PR still lacks required real behavior proof.

## What This Changes

The PR adds bare `$VAR` env SecretRef parsing, skips auth-profile plaintext audit findings for detected inline env refs, and adds parser/audit regression tests.

## Best Possible Solution

Keep the parser improvement, but update auth audit to classify the authored inline credential value independently before plaintext fallback, add dual-field regression coverage, and require redacted CLI proof before merge.

## Reproduction Assessment

Yes. Source inspection shows current main still misclassifies env-looking auth profile values, and the PR's remaining dual-field miss follows directly from `resolveSecretInputRef` nulling `inlineRef` whenever an explicit ref is present.

## Solution Assessment

No. Adding `$VAR` parsing is reasonable, but the auth audit fix should inspect the authored inline value before plaintext classification instead of relying on `inlineRef` after explicit-ref suppression.
