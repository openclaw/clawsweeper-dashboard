## Summary

Keep open: current main still activates the top-level managed proxy through `global-agent`, the docs/tests still preserve that contract, and Discord's explicit proxy path is only a workaround for the failing process-wide hook.

## What This Changes

Fix the documented top-level proxy so HTTP-forward-proxied Discord/TLS traffic keeps validating certificates against the target host rather than localhost.

## Best Possible Solution

Keep the managed proxy contract, but replace or narrow the unsafe Node core hook with SNI-safe HTTP CONNECT coverage and update docs/tests for any supported-coverage change.

## Reproduction Assessment

Yes. The report gives exact config, logs, and a minimal `global-agent` reproduction, and current main still activates that same top-level bootstrap path.

## Solution Assessment

No current fix is present. The owner-specific Discord and Telegram proxy settings are useful workarounds, but the durable fix needs to address the documented managed proxy behavior.
