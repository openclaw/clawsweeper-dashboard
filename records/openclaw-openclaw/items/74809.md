## Summary

Keep open: current main still enables the documented top-level managed proxy through `global-agent`, and the Discord-specific explicit proxy path is only a workaround for the failing process-wide hook.

## What This Changes

Fix the documented top-level proxy so HTTP-forward-proxied Discord/TLS traffic validates certificates against the target host instead of the local proxy host.

## Best Possible Solution

Preserve the managed proxy contract while replacing or narrowing the unsafe Node core hook with SNI-safe HTTP CONNECT coverage, then update tests and docs for any supported-coverage change.

## Reproduction Assessment

Yes. The report gives exact config, logs, and a minimal `global-agent` reproduction, and current main still activates the same top-level bootstrap path.

## Solution Assessment

No current fix is present. The narrowest durable fix should correct target-host TLS validation in the managed proxy path rather than relying on owner-specific proxy settings or disabling certificate checks.
