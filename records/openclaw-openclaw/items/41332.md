## Summary

Keep open: current main does not implement this PR's full intent, and the patch still has blocking correctness issues around the gateway exit proof, cooldown failover contract, and public cron config handling.

## What This Changes

The branch adds a gateway-specific runtime exit path, changes model-fallback cooldown probing, adds cron startup staggering config, updates related tests, and allowlists fake-token strings in one sanitizer test.

## Best Possible Solution

Split or refresh the work into focused changes that prove a shutdown strategy, preserve or intentionally redesign the cooldown probe contract, and route cron catch-up controls through the generated config/docs pipeline and #42039.

## Reproduction Assessment

Yes for the PR review findings by source inspection: Node source shows the exit-path proof gap, current tests encode the cooldown contract, and generated schema files show the config exposure gap. The original GGML Metal crash itself remains unclear without live Metal-host proof.

## Solution Assessment

No. The PR bundles three behavior changes and is not the narrowest maintainable fix because the runtime dependency claim is unproven, the fallback change removes covered recovery behavior, and the cron config addition bypasses generated public schema/docs handling.

- **[P2] Prove the gateway exit path skips native at-exit cleanup:** `src/runtime.ts:61`
  - body: `createGatewayRuntime().exit` relies on `process.reallyExit` while documenting that it skips native/C++ at-exit handlers. Node v22.11.0 source shows `ReallyExit` calls `RunAtExit(env)` before exiting, so this patch does not prove it avoids the GGML native cleanup path identified in the PR.
  - confidence: 0.86
- **[P2] Preserve transient same-provider cooldown probes:** `src/agents/model-fallback.ts:445-451`
  - body: Changing active-cooldown attempts to only primary probes removes same-provider fallback attempts during transient `rate_limit`, `overloaded`, and `timeout` cooldowns. Current main intentionally allows those probes, now capped to one per provider per run, and has regression tests asserting `allowTransientCooldownProbe: true` for that path.
  - confidence: 0.9
- **[P2] Regenerate the public config schema:** `src/config/zod-schema.ts:512`
  - body: Adding `startupStaggerMs` only to the zod/type surface leaves the generated base config schema unchanged, but `config.schema` and `config.schema.lookup` are served from that generated schema path. The new public key will not be exposed consistently without regenerating schema/docs or folding this into #42039.
  - confidence: 0.84

  - body: This PR contains user-facing gateway shutdown, model fallback cooldown, and cron configuration behavior changes, but it does not update `CHANGELOG.md`. Repo policy requires a changelog entry for user-facing fix, feat, and perf changes.
