## Summary

Keep this PR open for revision: current `main` does not implement the bundled change, and the PR still has merge-blocking correctness issues around the gateway exit proof, cooldown fallback policy, and generated config schema handling.

## What This Changes

The PR adds a gateway-specific runtime exit path using `process.reallyExit`, changes model-fallback cooldown probing, adds `cron.startupStaggerMs` catch-up staggering, updates tests, and allowlists fake token strings in one sanitizer test.

## Best Possible Solution

Refresh or split the PR so the shutdown fix is proven against Node/native exit behavior, cooldown probing preserves or intentionally redesigns the current tested recovery policy, and cron catch-up controls go through the generated schema/docs pipeline with #42039 handled deliberately.

## Reproduction Assessment

Yes for the review findings: source inspection against current main and the PR diff reproduces the cooldown regression and generated-schema gap, and Node source inspection reproduces the `process.reallyExit` proof problem. The original GGML Metal shutdown crash itself is not live-reproduced here.

## Solution Assessment

No. The current PR is not the narrowest maintainable solution because it bundles three behavior changes, relies on an exit-path claim contradicted by Node source, removes tested cooldown recovery behavior, and bypasses generated public config schema/docs handling.
