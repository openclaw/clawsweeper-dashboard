## Summary

Keep open: the PR still has blocking correctness concerns, lacks real Metal-host shutdown proof, and conflicts with current main's intentional cooldown recovery contract plus the canonical cron config follow-up.

## What This Changes

The PR adds a gateway-specific exit runtime, tightens model fallback cooldown probing, adds a cron startup stagger config key, and updates related tests/config typings.

## Best Possible Solution

Split or refresh the work into focused changes: prove or redesign gateway shutdown, preserve the established transient cooldown probe behavior unless maintainers change policy, and resolve cron missed-job controls through the canonical public config design.

## Reproduction Assessment

Yes for the PR review findings: source inspection and upstream Node source show the exit-contract gap, current tests show the cooldown contract conflict, and current config/schema code plus https://github.com/openclaw/openclaw/issues/42039 show the cron API gap. The original GGML Metal crash itself is not reproduced because the PR includes no live Metal-host proof.

## Solution Assessment

No. The branch combines three independent behavior changes, and the maintainable path is to split them while proving shutdown behavior, preserving established cooldown semantics unless maintainers choose otherwise, and routing cron catch-up configuration through https://github.com/openclaw/openclaw/issues/42039.
