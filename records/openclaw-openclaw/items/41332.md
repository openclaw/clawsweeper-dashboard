## Summary

Keep open for human follow-up: current main does not implement the shutdown proposal, while the cooldown and cron parts now overlap or conflict with later mainline decisions and still have substantive review blockers.

## What This Changes

The branch adds a gateway-specific `process.reallyExit` runtime, changes model fallback cooldown probing, adds `cron.startupStaggerMs`, updates tests, and allowlists fake sanitizer tokens.

## Best Possible Solution

Split or refresh the work into focused changes: prove or redesign gateway shutdown, preserve the current transient cooldown probe contract unless policy changes, and route cron catch-up configuration through the generated config/docs path tracked by https://github.com/openclaw/openclaw/issues/42039.

## Reproduction Assessment

Yes for the PR review findings: source inspection shows the Node exit contract gap, the current cooldown contract conflict, and the missing generated/public cron config surface. The original GGML Metal shutdown crash itself is not reproduced here because the PR does not include live Metal-host proof.

## Solution Assessment

No. The branch combines three independent behavior changes, and the maintainable path is to split them while proving the shutdown path, keeping the established cooldown semantics unless maintainers change policy, and resolving public cron config design in the canonical follow-up.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Prove the gateway exit path bypasses native shutdown:** `src/runtime.ts:61`
  - body: The new `createGatewayRuntime().exit` documents `process.reallyExit` as skipping C++ atexit handlers, but Node v22's `ReallyExit` implementation calls `RunAtExit(env)` before exiting. As written, the patch does not prove it avoids the GGML cleanup path it is meant to fix.
  - confidence: 0.86
- **[P2] Keep transient same-provider cooldown probes:** `src/agents/model-fallback.ts:445-451`
  - body: The PR changes active cooldown handling so same-provider fallback models are skipped. Current main intentionally permits one transient same-provider probe for `rate_limit`, `overloaded`, and `timeout`, with regression coverage for that contract, so this regresses model-scoped recovery unless maintainers explicitly change the policy.
  - confidence: 0.9
- **[P2] Expose cron catch-up knobs through generated config:** `src/config/zod-schema.ts:512`
  - body: Adding `cron.startupStaggerMs` only to Zod/types leaves the public generated schema, labels, help, docs, and the canonical missed-job policy in https://github.com/openclaw/openclaw/issues/42039 unresolved. The cron configuration should go through that generated public surface instead of introducing a partial key.
  - confidence: 0.84
