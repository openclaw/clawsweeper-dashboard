## Summary

Keep open: the branch is still useful context, but it remains conflicting and has blocking proof, dependency-contract, cooldown-policy, and cron-config issues that should not be merged as-is.

## What This Changes

The branch adds a gateway-specific runtime using `process.reallyExit`, removes active-cooldown probe paths from model fallback, exposes `cron.startupStaggerMs`, and updates related tests plus test-only secret-scan allowlist comments.

## Best Possible Solution

Split or refresh the work into focused changes: prove or redesign the gateway shutdown fix, preserve current transient cooldown recovery unless maintainers choose a new policy, and route cron missed-run controls through the canonical public cron policy work.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the PR review findings by source inspection of the PR head, current-main tests, cron schema/deps, and Node source; no live Metal-host reproduction exists for the original shutdown crash.

## Solution Assessment

Is this the best way to solve the issue? No. The branch combines three independent changes, relies on an unproven undocumented exit contract, changes cooldown recovery policy, and exposes a one-off cron key instead of the broader missed-run policy design.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Avoid relying on reallyExit for native cleanup bypass:** `src/runtime.ts:61`
  - body: `process.reallyExit` is described here as bypassing native shutdown handlers, but Node v22.16.0 calls `RunAtExit(env)` before exiting. That leaves the GGML Metal crash fix unproven while moving gateway shutdown onto an undocumented API, so this needs a different exit path or real Metal-host proof.
  - confidence: 0.86
- **[P2] Preserve transient same-provider cooldown probes:** `src/agents/model-fallback.ts:452`
  - body: This reduces cooldown attempts to primary-only, so same-provider fallback models are skipped during transient `rate_limit`, `overloaded`, or `timeout` cooldowns. Current main intentionally preserves one transient same-provider probe and tests that recovery behavior.
  - confidence: 0.91
- **[P2] Route missed-job controls through the public cron policy:** `src/config/zod-schema.ts:512`
  - body: Adding only `cron.startupStaggerMs` creates a partial public key that does not cover the existing `maxMissedJobsPerRestart`/`missedJobStaggerMs` controls or the canonical skip-policy request. The public cron catch-up API should be handled through https://github.com/openclaw/openclaw/issues/42039 rather than landing this one-off key.
  - confidence: 0.84
