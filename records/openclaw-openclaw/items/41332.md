## Summary

Keep open: the PR still targets real gateway, fallback, and cron pain points, but the current branch has blocking correctness findings and still lacks after-fix real behavior proof for the Metal shutdown claim.

## What This Changes

The PR adds a gateway-specific exit runtime, changes model fallback cooldown probing, adds a public cron startup stagger option, and updates related tests plus fake-token allowlist comments.

## Best Possible Solution

Split or refresh the work into focused changes: prove or redesign the gateway shutdown fix, preserve current transient cooldown recovery unless maintainers choose a new policy, and route cron missed-run controls through https://github.com/openclaw/openclaw/issues/42039.

## Reproduction Assessment

Yes for the PR review findings: PR-head source, current-main cooldown tests, current cron schema/deps, and Node v22.16.0 source show the blockers. No live Metal-host reproduction or proof exists for the original shutdown crash.

## Solution Assessment

No. The branch combines independent changes, relies on undocumented process.reallyExit without proving the native cleanup bypass, reverses current cooldown recovery behavior, and exposes a one-off cron key instead of resolving the canonical missed-run policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Avoid relying on reallyExit for native cleanup bypass:** `src/runtime.ts:59-61`
  - body: `process.reallyExit` is claimed to bypass C++ atexit handlers, but Node v22.16.0's `ReallyExit` runs `RunAtExit(env)` and then exits through the environment handler. That leaves the GGML Metal shutdown fix unproven while moving gateway shutdown onto an undocumented API, so use a proven exit path or provide real Metal-host proof.
  - confidence: 0.88
- **[P2] Preserve transient same-provider cooldown probes:** `src/agents/model-fallback.ts:452`
  - body: This reduces cooldown attempts to primary-only, so same-provider fallback models are skipped during transient `rate_limit`, `overloaded`, or `timeout` cooldowns. Current main intentionally preserves a transient same-provider probe and has tests covering that recovery behavior.
  - confidence: 0.91
- **[P2] Route missed-job controls through the cron policy:** `src/config/zod-schema.ts:512`
  - body: Adding only `cron.startupStaggerMs` creates a partial public key that does not cover the existing `maxMissedJobsPerRestart`/`missedJobStaggerMs` controls or the skip-policy request. The public cron catch-up API should be handled through https://github.com/openclaw/openclaw/issues/42039 instead of landing this one-off key.
  - confidence: 0.84
