## Summary

Keep this PR open: current main does not implement the proposed gateway exit runtime, and the PR still has merge-blocking correctness concerns around the Node exit contract, model-fallback cooldown policy, and public config schema generation.

## What This Changes

The PR adds a gateway-specific runtime exit path, changes model-fallback cooldown probing, exposes cron startup staggering config, updates related tests, and allowlists fake-token strings in one sanitizer test.

## Best Possible Solution

Refresh or split the PR into focused changes: prove a shutdown path that really avoids the failing native cleanup, preserve or explicitly redesign cooldown probing, and handle cron public config through the generated schema/docs pipeline or #42039.

## Reproduction Assessment

Yes for the PR review findings: source inspection against current main reproduces the cooldown-policy regression and generated-schema gap, and Node source inspection reproduces the process.reallyExit proof concern. The original GGML Metal shutdown crash remains unclear without live Metal-host proof.

## Solution Assessment

No. The current PR is not the narrowest maintainable fix because it relies on an unproven runtime exit assumption, removes tested fallback recovery behavior, and adds public cron config without regenerating the public schema/docs surfaces.
