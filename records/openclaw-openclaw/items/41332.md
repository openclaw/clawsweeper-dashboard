## Summary

Keep open for human follow-up: the PR still has blocking correctness questions around the shutdown exit path, cooldown recovery behavior, and cron missed-run policy, plus no after-fix real behavior proof for the Metal shutdown claim.

## What This Changes

The PR adds a gateway-specific runtime using `process.reallyExit`, tightens model fallback cooldown probing, exposes `cron.startupStaggerMs`, and updates related tests plus fake-token allowlist comments.

## Best Possible Solution

Split or refresh the work into focused changes: prove or redesign the gateway shutdown fix, preserve transient cooldown recovery unless maintainers choose a new policy, and route cron missed-run controls through https://github.com/openclaw/openclaw/issues/42039.

## Reproduction Assessment

Yes for the PR review findings: source inspection of the PR head, current-main tests, current-main cron schema/deps, and Node source shows the blockers. No live Metal-host reproduction exists for the original shutdown crash.

## Solution Assessment

No. The branch combines independent changes, relies on an undocumented exit path whose native cleanup bypass is unproven, reverses current cooldown recovery behavior, and exposes a one-off cron key instead of resolving the broader missed-run policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Avoid relying on reallyExit for native cleanup bypass:** `src/runtime.ts:59-61`
  - body: `process.reallyExit` is documented here as bypassing native shutdown handlers, but Node v22.16.0 calls `RunAtExit(env)` before exiting. That leaves the GGML Metal crash fix unproven while moving gateway shutdown onto an undocumented API, so this needs a different exit path or real Metal-host proof.
  - confidence: 0.86
- **[P2] Preserve transient same-provider cooldown probes:** `src/agents/model-fallback.ts:452`
  - body: This reduces cooldown attempts to primary-only, so same-provider fallback models are skipped during transient `rate_limit`, `overloaded`, or `timeout` cooldowns. Current main intentionally preserves one transient same-provider probe and tests that recovery behavior.
  - confidence: 0.91
- **[P2] Route missed-job controls through the cron policy:** `src/config/zod-schema.ts:512`
  - body: Adding only `cron.startupStaggerMs` creates a partial public key that does not cover the existing `maxMissedJobsPerRestart` and `missedJobStaggerMs` controls or the canonical skip-policy request. The public cron catch-up API should be handled through https://github.com/openclaw/openclaw/issues/42039 rather than landing this one-off key.
  - confidence: 0.84
