## Summary

Keep open: current main only partially covers this bundle, and the PR still has blocking review issues around the native shutdown claim, model fallback cooldown policy, and public cron config shape.

## What This Changes

The PR adds a gateway-specific `process.reallyExit` runtime, changes model fallback cooldown probing, adds a `cron.startupStaggerMs` catch-up delay option, updates tests, and adds fake-token allowlist comments in a sanitizer test.

## Best Possible Solution

Split or refresh the work into focused changes: prove or redesign the shutdown exit path, preserve the tested cooldown probe contract unless maintainers deliberately change it, and expose cron catch-up controls through the generated config/docs pipeline tracked by #42039.

## Reproduction Assessment

Yes for the PR review findings: source inspection and current tests show the Node exit claim, cooldown contract conflict, and missing generated config surface. The original GGML Metal shutdown crash itself was not reproduced in this review because no live Metal-host shutdown proof is available here.

## Solution Assessment

No. The PR bundles three independent behavior changes, the exit API does not support the stated native-bypass claim without more proof, the fallback change removes tested recovery behavior, and the cron option should go through the existing generated schema/docs path or #42039.
