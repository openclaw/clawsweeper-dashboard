## Summary

Keep open: current main does not implement this PR, and the proposed patch still has blocking correctness issues around the gateway exit proof, existing cooldown-probe behavior, and generated public config schema handling.

## What This Changes

The PR adds a gateway-specific runtime exit helper using `process.reallyExit`, changes model fallback cooldown probing, adds cron startup staggering config/schema/types wiring, updates tests, and allowlists fake-token sanitizer test strings.

## Best Possible Solution

Refresh or split the PR into focused changes: prove a shutdown path that actually avoids the failing native cleanup, preserve or explicitly redesign transient cooldown probe policy with tests, and route cron catch-up controls through the generated config schema/docs pipeline or the existing #42039 discussion.

No. The current PR is not the narrowest maintainable fix because it depends on an unproven runtime exit claim, removes tested fallback recovery policy, and adds public cron config without generated schema/docs follow-through.
