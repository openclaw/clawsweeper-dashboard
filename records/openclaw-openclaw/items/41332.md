## Summary

Keep open: current main does not implement the full PR, and the patch still has merge-blocking correctness issues around the shutdown proof, model-fallback cooldown policy, and public cron config exposure.

## What This Changes

The PR branch adds a gateway-specific shutdown runtime, changes model-fallback cooldown probing, adds a cron startup stagger config field, updates related tests, and allowlists fake-token strings in one sanitizer test.

## Best Possible Solution

Split the work into focused changes: prove a supported gateway shutdown strategy on macOS/Metal, preserve or intentionally redesign the cooldown-probe contract with maintainer approval, and route cron catch-up public config through #42039 with generated schema, docs, and changelog coverage.

## Reproduction Assessment

Yes for the PR review findings: source inspection of Node v22.11.0, current fallback tests, and generated config schema establishes the blockers. The original GGML Metal crash itself was not live-reproduced here.

## Solution Assessment

No. The PR is not the narrowest maintainable fix because the shutdown dependency claim is unproven, the fallback change removes a tested recovery path, and the cron config addition bypasses the generated public schema/docs pipeline.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Prove the gateway exit path skips native cleanup:** `src/runtime.ts:61`
  - body: `createGatewayRuntime().exit` relies on `process.reallyExit` while documenting that it skips native/C++ at-exit handlers. Node v22.11.0 source shows `ReallyExit` calls `RunAtExit(env)` and then reaches the default C `exit(...)` path, so this patch does not prove it avoids the GGML native cleanup path identified in the PR.
  - confidence: 0.86
- **[P2] Preserve transient same-provider cooldown probes:** `src/agents/model-fallback.ts:445-451`
  - body: Changing active-cooldown attempts to only primary probes removes same-provider fallback attempts during transient `rate_limit`, `overloaded`, and `timeout` cooldowns. Current main intentionally allows one such probe per provider per run and has regression tests asserting `allowTransientCooldownProbe: true` for that path.
  - confidence: 0.9
- **[P2] Regenerate the public config schema:** `src/config/zod-schema.ts:512`
  - body: Adding `startupStaggerMs` only to the zod/type surface leaves the generated base config schema, help, and labels unchanged, but `config.schema` and `config.schema.lookup` are served from that generated schema path. The new public key will not be exposed consistently without regenerating schema/docs or folding this into #42039.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `CHANGELOG.md:1`
  - body: This PR contains user-facing gateway shutdown, model fallback cooldown, and cron configuration behavior changes, but it does not update `CHANGELOG.md`. Repo policy requires a changelog entry for user-facing fix, feat, and perf changes.
  - confidence: 0.8
