## Summary

Keep open: this PR still has correctness blockers, is reported dirty/unmergeable in the provided GitHub context, and current main already carries more focused cooldown and cron work that conflicts with parts of this branch.

## What This Changes

This PR changes gateway runtime exit behavior, model-fallback cooldown probing, cron missed-job startup staggering/config typing, gateway CLI runtime mocks, and fake-token allowlist comments in an agents sanitizer test.

## Best Possible Solution

Split or replace the branch with focused work: prove the gateway shutdown strategy against Node/native behavior, keep or explicitly redesign the current cooldown probe contract, and handle cron catch-up controls through the #42039 config/schema/docs path.

## Reproduction Assessment

Yes for the PR review findings by source inspection: Node's reallyExit path calls RunAtExit, current tests encode transient same-provider probes, and config.schema is served from the generated base schema. The original GGML Metal shutdown crash remains unclear without a Metal-host reproduction.

## Solution Assessment

No. The PR bundles three independent behavior changes, relies on an unproven shutdown primitive, removes covered fallback recovery behavior, and introduces cron config surface that should be folded into the generated schema/docs work tracked by #42039.
