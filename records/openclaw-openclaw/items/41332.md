## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Do not rely on reallyExit to bypass native shutdown:** `src/runtime.ts:61`
  - body: `process.reallyExit` is documented here as bypassing C++ atexit handlers, but Node 22/24 source calls `RunAtExit(env)` before exiting. That leaves the implicated GGML native cleanup path unproven while moving gateway shutdown onto an undocumented API, so this needs a different exit path or live Metal proof.
  - confidence: 0.86
- **[P2] Preserve transient same-provider cooldown probes:** `src/agents/model-fallback.ts:452`
  - body: This makes only the primary eligible during cooldown, so same-provider fallback models are skipped. Current main intentionally allows one transient same-provider probe for rate-limit, overloaded, timeout, and related recovery and tests that behavior, so this regresses fallback recovery unless maintainers change the policy.
  - confidence: 0.91
- **[P2] Route missed-job controls through the public cron policy:** `src/config/zod-schema.ts:512`
  - body: Adding `cron.startupStaggerMs` alone creates a partial public key that does not cover the canonical `maxMissedJobsPerRestart`/`missedJobStaggerMs`/skip-policy request or generated config help/docs. The cron catch-up API should be handled through the open public-config design instead of landing a one-off key.
  - confidence: 0.84
