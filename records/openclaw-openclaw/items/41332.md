## Summary

Current `main` has not absorbed this PR, and the proposed branch still has concrete merge blockers: the gateway exit change relies on an unproven Node internal behavior, the cooldown rewrite removes behavior that current tests intentionally preserve, and the cron config addition does not follow the generated public schema/docs path. ## Best Possible Solution

  - body: `createGatewayRuntime().exit` relies on `process.reallyExit` while documenting that it skips native/C++ atexit handlers. Node v22.11.0 source shows `ReallyExit` calls `RunAtExit(env)` before exiting, so this patch does not prove it avoids the GGML native cleanup path identified in the PR.

  - confidence: 0.9

  - body: Adding `startupStaggerMs` only to the zod/type surface leaves the generated base config schema unchanged, but `config.schema` and `config.schema.lookup` are served from that generated schema path.
