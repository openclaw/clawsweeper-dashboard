## Summary

Keep open: the patch is not mergeable as-is because it still lacks real Metal-host shutdown proof, relies on an unproven `process.reallyExit` native-shutdown contract, regresses current transient cooldown fallback behavior, and adds a one-off cron config key while the public missed-run policy remains tracked elsewhere.

## What This Changes

The branch changes gateway shutdown to use a gateway-specific `process.reallyExit` runtime, removes active-cooldown probe paths from model fallback, adds `cron.startupStaggerMs`, and updates related tests plus test-only secret-scan allowlist comments.

## Best Possible Solution

Split or refresh the work into focused changes: prove or redesign gateway shutdown, preserve current transient cooldown recovery unless maintainers choose a new policy, and route cron missed-run controls through https://github.com/openclaw/openclaw/issues/42039.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the PR review findings: source inspection shows the exit-contract gap, the cooldown behavior conflict, and the partial cron config surface. No for the original Metal crash itself because this read-only review did not run a live Metal-host shutdown reproduction.

## Solution Assessment

Is this the best way to solve the issue? No. The branch combines three independent changes, and the maintainable path is to split the work while proving the shutdown fix, preserving current cooldown semantics unless policy changes, and handling cron catch-up settings through the canonical public-config design.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Do not rely on reallyExit to bypass native shutdown:** `src/runtime.ts:61`
  - body: `process.reallyExit` is documented here as bypassing C++ atexit handlers, but Node v22.16.0 source calls `RunAtExit(env)` before exiting.   - confidence: 0.86
- **[P2] Preserve transient same-provider cooldown probes:** `src/agents/model-fallback.ts:452`
  - body: This makes only the primary eligible during cooldown, so same-provider fallback models are skipped. Current main intentionally allows one transient same-provider probe for rate-limit, overloaded, timeout, and related recovery and tests that behavior, so this regresses fallback recovery unless maintainers change the policy.
  - confidence: 0.91
- **[P2] Route missed-job controls through the public cron policy:** `src/config/zod-schema.ts:512`
  - body: Adding `cron.startupStaggerMs` alone creates a partial public key that does not cover the canonical `maxMissedJobsPerRestart`/`missedJobStaggerMs`/skip-policy request or generated config help/docs. The cron catch-up API should be handled through https://github.com/openclaw/openclaw/issues/42039 instead of landing a one-off key.
  - confidence: 0.84
