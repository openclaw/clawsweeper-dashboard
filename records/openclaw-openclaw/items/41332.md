## Summary

Keep open: this PR still has blocking correctness concerns and missing real behavior proof. The gateway shutdown path depends on an undocumented Node exit API whose native-cleanup claim does not match the Node source, the cooldown change reverses current tested transient fallback behavior, and the cron config change overlaps an open policy/API decision tracked separately.

## What This Changes

The PR adds a gateway-specific `process.reallyExit` shutdown runtime, tightens model-fallback cooldown probing, exposes `cron.startupStaggerMs`, and updates related tests.

## Best Possible Solution

Split or refresh the work into focused changes: prove or redesign the gateway shutdown fix, preserve transient cooldown recovery unless maintainers choose a new policy, and route cron missed-run configuration through the canonical cron policy issue.

## Reproduction Assessment

Yes for the PR review findings: PR-head source, current-main cooldown tests, current cron schema/deps, and Node v22.16.0 source show the blockers. No live Metal-host reproduction or after-fix proof exists for the original shutdown crash.

## Solution Assessment

No. The branch combines independent gateway, agents, and cron changes, relies on an unproven undocumented exit API for the native crash, reverses current cooldown recovery behavior, and exposes a one-off cron key instead of resolving the canonical missed-run policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Avoid relying on reallyExit for native cleanup bypass:** `src/runtime.ts:59-61`
  - body: `process.reallyExit` is claimed to bypass C++ atexit handlers, but Node v22.16.0's `ReallyExit` calls `RunAtExit(env)` before exiting. That leaves the GGML Metal shutdown fix unproven while moving gateway shutdown onto an undocumented API, so use a proven exit path or provide real Metal-host proof.
  - confidence: 0.88
- **[P2] Preserve transient same-provider cooldown probes:** `src/agents/model-fallback.ts:452`
  - body: This reduces cooldown attempts to primary-only, so same-provider fallback models are skipped during transient `rate_limit`, `overloaded`, or `timeout` cooldowns. Current main intentionally preserves a transient same-provider probe and has tests covering that recovery behavior.
  - confidence: 0.91
- **[P2] Route missed-job controls through the cron policy:** `src/config/zod-schema.ts:512`
  - body: Adding only `cron.startupStaggerMs` creates a partial public key that does not cover the existing `maxMissedJobsPerRestart`/`missedJobStaggerMs` controls or skip-policy request. The public cron catch-up API should be handled through https://github.com/openclaw/openclaw/issues/42039 instead of landing this one-off key.
  - confidence: 0.84
