## Summary

Keep open: the PR still has blocking review issues around the gateway exit dependency claim, removal of current cooldown recovery behavior, and an incomplete public cron config surface.

## What This Changes

The PR adds a gateway-specific runtime exit path using `process.reallyExit`, tightens model-fallback cooldown probing, adds cron startup catch-up staggering config, updates related tests, and allowlists fake token strings in one sanitizer test.

## Best Possible Solution

Refresh or split the PR into focused changes: prove a shutdown path that actually avoids the failing native cleanup, preserve or explicitly redesign transient cooldown probe policy with tests, and route cron catch-up controls through the generated config schema/docs pipeline.

The original GGML Metal shutdown crash remains unclear because the PR record does not include live Metal-host reproduction proof.

## Solution Assessment

No. The current PR bundles three behavior changes and is not the narrowest maintainable fix because the runtime dependency claim is unproven, the fallback policy change removes tested recovery behavior, and the cron config path bypasses generated public schema/docs handling.
