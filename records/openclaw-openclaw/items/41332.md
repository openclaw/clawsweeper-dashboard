## Summary

Keep open, but the current patch should not merge: it lacks real Metal-host shutdown proof, relies on a Node exit contract that does not prove native atexit bypass, removes current cooldown recovery behavior, and only partially overlaps the open cron missed-run API work.

## What This Changes

The branch adds a gateway-specific runtime using `process.reallyExit`, removes active-cooldown fallback probe paths, exposes `cron.startupStaggerMs`, and updates related tests plus test-only secret-scan allowlist comments.

## Best Possible Solution

Split or refresh the work into focused changes: prove or redesign gateway shutdown, preserve current transient cooldown recovery unless maintainers choose a new policy, and route cron missed-run controls through https://github.com/openclaw/openclaw/issues/42039.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the PR review blockers by source inspection: the exit-contract gap, cooldown behavior conflict, and partial cron config surface are visible in the diff and current main. No for the original Metal crash itself because this read-only review did not run a live Metal-host shutdown reproduction.

## Solution Assessment

Is this the best way to solve the issue? No. The branch combines three independent changes, relies on an unproven shutdown dependency contract, removes current cooldown recovery behavior, and exposes a one-off cron key instead of the canonical missed-run policy design.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Do not rely on reallyExit to bypass native shutdown:** `src/runtime.ts:61`
  - body: The new comment says `process.reallyExit` bypasses C++ atexit handlers, but Node v22.16.0 source calls `RunAtExit(env)` before exiting. That leaves the implicated GGML native cleanup path unproven while moving gateway shutdown onto an undocumented API, so this needs a different exit path or real Metal-host proof.
  - confidence: 0.86
- **[P2] Preserve transient same-provider cooldown probes:** `src/agents/model-fallback.ts:452`
  - body: This makes only the primary eligible during cooldown, so same-provider fallback models are skipped. Current main intentionally allows one transient same-provider probe and tests that it passes `allowTransientCooldownProbe: true`, so this regresses fallback recovery unless maintainers choose a new cooldown policy.
  - confidence: 0.91
- **[P2] Route missed-job controls through the public cron policy:** `src/config/zod-schema.ts:512`
  - body: Adding `cron.startupStaggerMs` alone creates a partial public key that does not cover the canonical `maxMissedJobsPerRestart`/`missedJobStaggerMs`/skip-policy request tracked at https://github.com/openclaw/openclaw/issues/42039, nor the related docs/help surface. The cron catch-up API should be handled as that policy instead of a one-off key.
  - confidence: 0.84
