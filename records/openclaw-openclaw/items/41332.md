## Summary

Keep open: the PR still has concrete correctness blockers against current main, especially the unproven `process.reallyExit` shutdown premise, removal of currently tested transient same-provider cooldown probes, and an incomplete public cron config/schema path.

## What This Changes

The PR adds a gateway-specific runtime exit path, changes model-fallback cooldown probing behavior, adds cron startup catch-up staggering config, updates related tests, and allowlists fake-token strings in one sanitizer test.

## Best Possible Solution

Split or refresh the PR into focused changes: prove a shutdown path that really avoids the failing native cleanup, preserve or deliberately redesign transient cooldown probes with regression coverage, and route cron catch-up controls through the generated config schema/docs pipeline and #42039.

## Reproduction Assessment

Yes for the PR review findings by source and patch inspection: current main tests and code reproduce the cooldown-contract conflict and generated-schema gap, and Node source inspection reproduces the `reallyExit` proof concern. The original GGML Metal shutdown crash itself remains unclear without live Metal-host proof.

## Solution Assessment

No. The PR bundles three behavior changes and is not the narrowest maintainable fix because the runtime dependency claim is unproven, the fallback policy removes tested recovery behavior, and the cron config addition bypasses generated public schema/docs handling.
