## Summary

Keep open: this PR is still not mergeable as-is because the shutdown change relies on unproven undocumented Node exit behavior, the cooldown change conflicts with current regression-covered behavior and a later merged fix, and the cron config work overlaps an open public API design item.

## What This Changes

This PR adds a gateway-specific `process.reallyExit` runtime, removes active cooldown probing from model fallback, adds `cron.startupStaggerMs`, updates related tests, and allowlists fake sanitizer tokens in a test.

## Best Possible Solution

Split or refresh the work into focused changes: prove or redesign gateway shutdown, preserve current transient cooldown probe semantics unless policy changes, and route missed-job public config through the canonical cron API design.

## Reproduction Assessment

Yes for the review findings: source inspection shows the Node exit contract gap, the current cooldown contract conflict, and the missing generated/public cron config surface. The original GGML Metal shutdown crash itself is not reproduced because this PR still lacks live Metal-host proof.

## Solution Assessment

No. The branch combines three independent behavior changes, and the maintainable path is to split them while proving shutdown behavior, keeping established cooldown semantics unless maintainers change policy, and resolving public cron config in https://github.com/openclaw/openclaw/issues/42039.
