## Summary

Keep open: current main still limits `browser.upload` to the temp uploads root while WebChat can stage attachments as managed inbound media, and open implementation PRs already reference this issue with closing links: https://github.com/openclaw/openclaw/pull/83572 and https://github.com/openclaw/openclaw/pull/83660.

## What This Changes

Allow `browser.upload` to use managed WebChat inbound media paths or refs without first copying them into the temp browser uploads directory.

## Best Possible Solution

Land one owner-reviewed shared upload resolver that is used by tool preflight, CLI prevalidation, the hook route, and Playwright use-time validation, accepting only temp uploads plus constrained managed inbound media while preserving traversal, symlink, hardlink, and outside-root rejection tests.

## Reproduction Assessment

Yes, source-level: current WebChat staging can expose attachments as `media/inbound/*`, while all current browser upload validators still require `DEFAULT_UPLOAD_DIR`. I did not run the live WebChat-to-browser path in this read-only review.

## Solution Assessment

Yes, the requested direction is maintainable if it reuses the existing managed-media resolver pattern and keeps the upload boundary narrow. The preferred path is to review and land one of the existing closing PRs rather than queue another repair branch.
