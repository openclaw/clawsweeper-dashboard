## Summary

Keep open: current main and the latest release still validate browser uploads only against the temp uploads root, while WebChat can stage attachments as managed inbound media. Two open closing PRs already cover this issue, so this should stay open until maintainers choose and land or close the implementation path.

## What This Changes

Allow `browser.upload` to use WebChat attachments stored as managed inbound media without first copying them into the temporary browser uploads directory.

## Best Possible Solution

Land a shared browser upload resolver, preferably the broader current approach in https://github.com/openclaw/openclaw/pull/83660 after boundary approval, then let the merged PR close this issue and supersede https://github.com/openclaw/openclaw/pull/83572 if appropriate.

## Reproduction Assessment

Yes, source-level: WebChat attachment staging can produce `media/inbound/*` paths, while current browser upload validation accepts only `DEFAULT_UPLOAD_DIR`. I did not run the full live WebChat-to-browser flow in this read-only review.

## Solution Assessment

Yes, the maintainable solution is a shared resolver used by tool preflight, CLI prevalidation, the hook route, and Playwright use-time validation, limited to temp uploads plus verified managed inbound media. The linked PRs should be reviewed rather than closing this issue early.
