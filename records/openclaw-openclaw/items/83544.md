## Summary

Keep open: current main still rejects managed WebChat inbound media for browser uploads, and the open linked fix PR is the right place to finish the security-boundary review before this issue closes.

## What This Changes

Allow `browser.upload` to accept OpenClaw-managed WebChat inbound media paths in addition to the temporary browser uploads directory.

## Best Possible Solution

Land one canonical browser upload resolver that validates both temp uploads and managed inbound media across tool preflight, CLI, server route, and Playwright use-time paths, then close this issue through the merged PR.

## Reproduction Assessment

Yes, source-level: WebChat attachment handling can produce `media://inbound/<id>` and `media/inbound/*` paths, while current browser upload validators accept only `DEFAULT_UPLOAD_DIR`. I did not run a live WebChat-to-browser upload flow in this read-only review.

## Solution Assessment

Yes, the linked shared-resolver direction is the narrowest maintainable fix because it can cover the tool, CLI, browser server, and Playwright validation paths consistently. The remaining decision is explicit maintainer approval of the browser upload boundary expansion.
