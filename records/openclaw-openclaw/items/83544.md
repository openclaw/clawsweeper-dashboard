## Label Justifications

- `P2`: The report describes a real but bounded WebChat attachment-to-browser upload workflow break.
- `impact:security`: The affected surface is the browser file-upload allowlist, which controls what local files may be sent to web forms.

## Summary

Keep open: current `main` and the latest release still validate browser uploads against the temp uploads root only, while WebChat can hand agents managed inbound media paths. The open linked fix PR should be reviewed and either landed or closed before this issue is resolved.

## What This Changes

The issue asks for `browser.upload` to accept OpenClaw-managed WebChat inbound media references and staged `media/inbound/*` files without requiring a copy into the browser temp uploads directory.

## Best Possible Solution

Review and land one canonical fix that centralizes upload validation for temp uploads plus verified managed inbound media refs, then close this issue after the linked PR merges.

## Reproduction Assessment

Yes at source level: WebChat staging can produce `media/inbound/*` paths, while current browser upload validation accepts only `DEFAULT_UPLOAD_DIR`. I did not run the full live WebChat/Gateway/browser flow in this read-only review.

## Solution Assessment

Yes, the best direction is a shared browser-owned resolver used by tool preflight, CLI, hook route, and Playwright use-time validation. The open linked PR is the right place to review that implementation and boundary approval.
