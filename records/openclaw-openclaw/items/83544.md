## Summary

Keep open: current main still validates browser uploads only against the temp uploads root, while WebChat attachments are exposed as managed inbound media paths. The linked open fix at https://github.com/openclaw/openclaw/pull/83660 uses closing syntax for this issue, so the right cleanup path is to review, update, and land or close that PR rather than closing this issue first.

## What This Changes

Allow `browser.upload` to accept OpenClaw-managed WebChat inbound media paths and media refs instead of only the temp uploads root.

## Best Possible Solution

Land one canonical browser upload fix that centralizes validation for temp uploads and verified managed inbound media refs across tool preflight, the server route, CLI, docs, and Playwright use-time validation, then close this issue.

## Reproduction Assessment

Yes, at source level: WebChat saves attachments as managed inbound media and exposes `media://inbound/<id>` or `media/inbound/*`, while current browser upload validators only accept `DEFAULT_UPLOAD_DIR`. I did not run a full live WebChat-to-browser upload flow in this read-only review.

## Solution Assessment

Yes, the best direction is a shared browser-owned resolver for temp uploads plus verified managed inbound media refs. This issue should stay open until the linked PR is reviewed, updated if needed, and either landed or closed by maintainers.
