## Summary

Keep open: current main and the latest release still validate browser uploads against the temp uploads root only, while WebChat staging can hand the agent managed inbound media paths. ## What This Changes

The issue asks `browser.upload` to accept managed inbound WebChat media paths such as `media/inbound/<id>` instead of requiring a copy into `DEFAULT_UPLOAD_DIR`.

## Best Possible Solution

Land one shared browser-owned resolver that accepts `DEFAULT_UPLOAD_DIR` plus verified managed inbound media refs/paths across tool preflight, hook route, CLI, and Playwright validation after maintainer approval, then close the overlapping PR and let this issue close from the merged fix.
