## Summary

Keep open: current main still validates browser uploads only against the browser temp upload root, while WebChat-managed attachments can reach the agent as inbound media paths. Two open fix PRs already reference this issue with closing syntax, so the issue should stay open until one reviewed fix lands.

## What This Changes

Allow `browser.upload` to accept WebChat-managed inbound media files instead of only files under the browser temp uploads root.

## Best Possible Solution

Land one reviewed browser-owned resolver that accepts `DEFAULT_UPLOAD_DIR` plus verified managed inbound media references in both tool preflight and use-time/server validation, then close or supersede the duplicate open fix PR.

## Reproduction Assessment

Yes, source-level: WebChat can stage attachments as `media/inbound/*`, while the browser upload tool and server paths still validate only against `DEFAULT_UPLOAD_DIR`. I did not run the full live WebChat/browser flow in this read-only review.

## Solution Assessment

Yes, the requested direction is the right shape if implemented through a narrow shared resolver and applied to both preflight and use-time validation. The broader fix in https://github.com/openclaw/openclaw/pull/83660 appears closer to that end state than only adding a physical inbound-directory fallback.
