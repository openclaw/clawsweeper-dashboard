## What This Changes

The PR updates `scripts/install.sh` so the Linux NodeSource apt setup runs with `DEBIAN_FRONTEND=noninteractive` and verifies the active Node version before printing the Node install success message.

## Best Possible Solution

Land the narrow installer fix after a redacted clean Ubuntu or WSL2 run proves the patched NodeSource apt path and maintainers accept the stricter post-install failure behavior.

## Reproduction Assessment

Partly: current source and the linked report provide a high-confidence path for the false-success symptom, because success is printed before the active Node version is verified. I did not reproduce the full WSL2 NodeSource apt failure live in this read-only review.

## Solution Assessment

Yes for the narrow bug: checking `node_is_at_least_required` before success is the maintainable fix for the false success. The explicit `DEBIAN_FRONTEND` on the NodeSource setup call is mostly belt-and-suspenders because current main already exports it for Linux, but it preserves `sudo -E` and does not create a new code path.
