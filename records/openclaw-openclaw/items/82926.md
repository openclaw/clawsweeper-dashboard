## What This Changes

The branch updates `scripts/install.sh` and `test/scripts/install-sh.test.ts` so the apt NodeSource setup runs with `DEBIAN_FRONTEND=noninteractive` and Linux Node installation validates the active Node version before reporting success.

## Best Possible Solution

Merge the focused installer fix after redacted Ubuntu or WSL2 terminal output, logs, or a recording shows the patched installer reaches the NodeSource apt path without a supported Node and either activates Node 24 or exits with the new clear error.

## Reproduction Assessment

## Solution Assessment

Likely yes: reusing `node_is_at_least_required` after `activate_supported_node_on_path` is the narrow maintainable fix for false success, and `sudo -E env DEBIAN_FRONTEND=noninteractive` preserves the prior sudo environment behavior. The solution still needs real clean Linux proof before merge.
