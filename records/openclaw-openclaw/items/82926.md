## Summary

Keep open: the post-install Node version guard is useful, but the diff still drops the existing `sudo -E` environment preservation on the NodeSource setup path and the supplied proof does not exercise the Linux NodeSource/WSL2 installer path.

## What This Changes

## Best Possible Solution

Keep the focused post-install version guard, preserve sudo environment while adding the noninteractive setting, and require redacted WSL2 or equivalent Linux installer proof before merge.

## Reproduction Assessment

## Solution Assessment

No. The post-install guard is the right direction, but the safest fix is to preserve the existing `sudo -E` behavior while setting `DEBIAN_FRONTEND=noninteractive`.
