## What This Changes

The PR adds per-registration tokens to `TwitchClientManager.onMessage` cleanup and regression tests for stale cleanup, same-callback re-registration, and current cleanup.

## Best Possible Solution

Merge the registration-token guard after clearing the token map in the test reset helper and after maintainers either obtain or explicitly waive redacted live Twitch runtime proof; then close https://github.com/openclaw/openclaw/issues/83888 when the fix lands.

## Reproduction Assessment

Yes, source-level. On current main, register handler A, register handler B for the same account key, then call cleanup A; the cleanup deletes the shared key without checking ownership, so B is removed.

## Solution Assessment

Yes, with one minor cleanup. A per-registration token guard is the right fix because it also handles same-callback re-registration; the reset helper should clear the new token map too.
