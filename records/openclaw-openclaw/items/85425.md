## What This Changes

The PR adds a token-owned cleanup guard to `TwitchClientManager.onMessage` and regression tests for stale cleanup, same-callback re-registration, and current cleanup removal.

## Best Possible Solution

Merge the token-owned cleanup guard after clearing test-only token state and after live Twitch proof is supplied or maintainers explicitly accept source and test proof.

## Reproduction Assessment

Yes, source-level. On current main, registering handler A, registering handler B for the same account key, then calling cleanup A deletes the shared key without checking ownership, and the monitor stop path calls that cleanup.

## Solution Assessment

Yes, with a small cleanup gap. The token guard is stronger than handler equality because the same callback can be re-registered, but the new token map should also be cleared by `clearForTest()`.
