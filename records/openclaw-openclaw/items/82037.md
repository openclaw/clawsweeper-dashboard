## Summary

Keep open: the reported protocol-3 client failure is still source-reproducible on current main, and the latest issue comments say web and app connections still fail after reinstall. A merged current-main Control UI service-worker mitigation likely addresses the stale web bundle part, but it is not shipped in the latest release and does not prove the macOS app/update path is fixed.

## What This Changes

The issue reports that upgrading to v2026.5.12 leaves web and macOS app clients connecting with protocol 3, causing protocol-mismatch failures against a protocol-4 gateway.

## Best Possible Solution

Ship and verify the service-worker fix, then decide whether stale native app clients need an update prompt or compatibility path; close after a packaged upgrade smoke shows web and app clients advertising protocol 4.
