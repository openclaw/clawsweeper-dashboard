## Summary

Keep open: the MiMo bug is plausible and source-backed, but this PR broadens generic OpenAI-compatible replay policy, changes the strict-key payload contract, and still lacks artifact-backed real behavior proof.

## What This Changes

The PR preserves or synthesizes `reasoning_content` in OpenAI-compatible replay and skips historical reasoning stripping for model IDs containing `mimo`.

## Best Possible Solution

Add Xiaomi/MiMo replay handling through a provider-owned or explicit capability-gated seam with exact model matching, focused regression tests, and no changes to generic strict-key sanitization.

## Reproduction Assessment

No live high-confidence reproduction is available from this review. The source path is clear enough to be source-reproducible: Xiaomi reasoning models use OpenAI-compatible replay, current generic policy drops historical reasoning, and MiMo docs recommend preserving `reasoning_content` for multi-turn thinking-mode conversations.

## Solution Assessment

No. The intended fix is directionally useful, but the implementation should be provider-owned or capability-gated and must not add `reasoning_content` to unrelated strict-key payloads or generic replay fallbacks.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep MiMo replay out of generic fallbacks:** `src/agents/transcript-policy.ts:146`
  - body: This fallback runs for arbitrary custom OpenAI-compatible providers. Keying it on `modelId.includes("mimo")` means any unrelated model name containing that substring stops stripping historical reasoning, so hidden thinking can be replayed where the current policy intentionally removes it; use a Xiaomi/provider capability or exact model-family hook instead.
  - confidence: 0.86
- **[P2] Preserve the strict-key role/content contract:** `src/agents/openai-completions-string-content.ts:50-60`
  - body: `strictMessageKeys` exists for backends that only accept `role` and `content`; docs and tests cover that contract. Adding `reasoning_content` for assistant array-content messages sends an extra key to non-MiMo strict backends, so MiMo-specific reasoning shaping should not live in this sanitizer.
  - confidence: 0.84
