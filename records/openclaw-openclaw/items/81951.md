## Summary

Keep this PR open: the reported MiMo replay failure is plausible from current source, but the patch puts MiMo-specific policy into generic OpenAI-compatible replay paths and broadens strict-key payloads, and it still needs real behavior proof before merge.

## What This Changes

The PR preserves or synthesizes `reasoning_content` in OpenAI-compatible replay and skips historical reasoning stripping for model IDs containing `mimo`.

## Best Possible Solution

Add provider-owned replay handling for Xiaomi and any supported MiMo proxy providers, with focused regression tests that preserve required `reasoning_content` without changing generic core fallbacks or strict-key sanitization.

## Reproduction Assessment

No live high-confidence reproduction is available from this review. The source path is clear enough to be source-reproducible: current Xiaomi/OpenAI-compatible replay can drop historical reasoning, but the PR lacks artifact-backed MiMo output showing the timeout and fix.

## Solution Assessment

No. The intended fix is directionally useful, but the implementation should be provider-owned or capability-gated and must not add `reasoning_content` to unrelated strict-key payloads.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep MiMo replay out of generic fallbacks:** `src/agents/transcript-policy.ts:146`
  - body: This generic fallback runs for arbitrary custom OpenAI-compatible providers. Keying it on `modelId.includes("mimo")` means any unrelated model name containing that substring will stop stripping historical reasoning, so hidden thinking can be replayed where the current policy intentionally removes it. Move this to a Xiaomi/OpenCode provider hook or an explicit compat/replay capability instead.
  - confidence: 0.86
- **[P2] Do not widen strict-key messages:** `src/agents/openai-completions-string-content.ts:50-60`
  - body: This helper is used for `compat.strictMessageKeys`, where existing coverage expects only `role` and `content` to survive. Adding `reasoning_content` for every assistant array-content message sends an extra key to non-MiMo strict backends, which is the payload shape this path exists to avoid.
  - confidence: 0.84
