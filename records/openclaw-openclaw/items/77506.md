## Summary

Keep open: current main still hardcodes `memory_recall` in the active-memory embedded recall run while the bundled `memory-core` plugin only declares and registers `memory_search` and `memory_get`. The open linked PR #77512 uses closing syntax for this issue, so this issue should remain open until that PR lands, is replaced, or is closed.

## What This Changes

Ensure active-memory does not request an unregistered `memory_recall` tool when only memory-core recall tools are available.

## Best Possible Solution

Make active-memory derive its recall tool allowlist from registered memory tools, preserving memory-lancedb `memory_recall` and memory-core `memory_search`/`memory_get`, then add targeted regression coverage.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection confirms the active-memory allowlist names `memory_recall` while memory-core exposes only `memory_search` and `memory_get`, and the embedded runner has the reported failure mode when no callable tools remain.

## Solution Assessment

Yes, the narrow maintainable direction is a backend-aware allowlist rather than registering a dummy `memory_recall` in memory-core or removing memory-lancedb support. The linked open PR appears aimed at that boundary and should be reviewed or landed before closing this issue.
