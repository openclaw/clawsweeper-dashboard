## What This Changes

The issue requests fixing Telegram replies that are processed and saved but never sent to Telegram, leaving no `sendMessage` log or visible response.

## Best Possible Solution

Patch the current-main partial-preview path so Telegram commits exactly one visible final reply: suppress duplicate extra bubbles only after the Telegram finalizer has retained or finalized the preview, and keep normal fallback delivery for unfinalized previews.

## Reproduction Assessment

## Solution Assessment

The existing generic preview-stream suppression is not the best solution because it can remove the final payload before Telegram finalization. A safer fix should make dedupe/finalization explicit to Telegram or to finalizable-preview channels while preserving duplicate-final suppression.
