## Summary

Keep open: current main still sends pre-auth bootstrap-token handshakes into the mutex-backed verifier without a bootstrap-token limiter, and the linked open fix PR has not merged.

## What This Changes

The issue asks to rate-limit pre-auth bootstrap-token verification before it reaches the mutex-backed device bootstrap verifier.

## Best Possible Solution

Review and update https://github.com/openclaw/openclaw/pull/77527 so bootstrap-token verification is throttled before verifyDeviceBootstrapToken, with the always-on versus configured-limiter policy explicitly settled, then close this issue after merge.

## Reproduction Assessment

Yes, from source inspection rather than a live run: a signed device handshake with auth.bootstrapToken reaches verifyBootstrapToken before any bootstrap-token limiter, and that verifier serializes under withLock.

## Solution Assessment

Yes on direction, but not fully settled: gating before verifyDeviceBootstrapToken is the right fix shape, while the linked PR still needs an owner/security decision on mandatory versus configured limiter behavior.
