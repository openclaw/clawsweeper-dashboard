## Summary

Keep open: the report matches a credible current-main gap between delivery-context-aware system-event routing and subagent active-wake delivery, and the related open PR/issue cluster does not clearly own this narrower final-reply route preservation problem.

## What This Changes

The issue requests preserving the original channel delivery routing for a parent session’s final reply after `sessions_yield` resumes from a subagent completion.

## Best Possible Solution

Carry the original requester `DeliveryContext` through the subagent completion wake/resume path so the resumed parent final reply gets the same effective turnSource as the original channel request, with regression coverage for an external channel route.

## Reproduction Assessment

No fresh high-confidence live reproduction was established in this review. The issue is source-reproducible enough to keep open because current main has delivery-context-aware turnSource routing, but the subagent active-wake queue path does not carry that context into the resumed parent turn.

## Solution Assessment

Directionally yes: preserving requester delivery context across the wake boundary is the right invariant. The exact implementation should be reviewed against the related yield-resume cluster before a fix PR lands.
