## Summary

Keep open. Current main still lacks the requested Control UI quote-reply workflow: it can display assistant-injected reply markers, but there is no message-level Reply action, no selected quote state in the composer, and no `chat.send` reply/quote payload. The open #7817 tracker overlaps the broader threading problem but does not precisely replace this lightweight quote-reply request.

## What This Changes

Request to add a Control UI chat workflow for replying to a specific earlier message, including a message Reply action, quoted composer context, and quoted context on the sent reply.

## Best Possible Solution

Decide whether Control UI replies should be a lightweight quoted-message affordance or part of the broader thread/project model, then implement the chosen path with additive transcript/protocol support plus desktop hover and mobile long-press UI.

## Reproduction Assessment

Not applicable as a feature request, but the current gap is high-confidence from source inspection. The chat view, controller, and protocol schema show no Reply action, selected quote composer state, or send-time quote metadata.

## Solution Assessment

Unclear until maintainers choose the product shape. A narrow quote-reply implementation could be maintainable, but it should not rely only on display-only inline directive tags if replies need durable context and reload-safe rendering.
