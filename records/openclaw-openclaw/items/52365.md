## Summary

Keep open: this PR is useful and now has real behavior proof, but the latest head still has merge-blocking correctness issues around the propagated cron deadline and stop-error handling.

## What This Changes

The PR adds a model-fallback pre-attempt stop hook, threads cron deadline data into isolated cron execution, skips low-budget cron fallback attempts, adjusts changed-test routing for `src/cron`, adds regression tests, and updates the changelog.

## Best Possible Solution

Land a rebased patch that keeps the generic `beforeAttempt` seam, derives the cron fallback deadline from the same execution-start moment as the deferred timeout, preserves explicit stop errors, and adds focused regression coverage for both paths.

## Reproduction Assessment

Yes. The PR body includes production logs, and current-main source inspection shows cron-selected fallbacks have no pre-attempt budget gate before `runFallbackAttempt`.

## Solution Assessment

No. The approach is maintainable, but the current patch is not the best mergeable form until the deadline uses the deferred execution-start clock and stop-only failures preserve the hook's explicit error.

## Review Findings

  - confidence: 0.86
