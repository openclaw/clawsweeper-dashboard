## Summary

## What This Changes

The PR adds a `runWithModelFallback` pre-attempt stop hook, threads cron deadlines into isolated cron runs, and adds regression coverage for skipping fallback attempts when remaining cron budget is too low.

## Best Possible Solution

Land a rebased version that preserves explicit stop errors when no real model attempt has failed, keeps the budget gate aligned with the execution-start deadline, and coordinates the timeout contract with the terminal-abort work.

## Reproduction Assessment

Yes. Source inspection shows current main has no pre-attempt veto before starting a fallback candidate, and the PR discussion includes production logs plus a local low-budget repro; I did not run live providers in this read-only review.

## Solution Assessment

No, not as-is. The `beforeAttempt` seam and per-attempt budget gate are the right shape, but the patch should preserve explicit stop errors after skip-only candidates and needs fresh current-head proof before merge.
