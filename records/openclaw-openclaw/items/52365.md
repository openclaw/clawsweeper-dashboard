## Summary

Keep open: this PR addresses a real cron fallback bug and the main approach is useful, but current head still has merge-blocking edge cases in the new stop hook and the real-behavior proof is stale after the latest force-push.

## What This Changes

Adds an optional model-fallback `beforeAttempt` hook, threads cron deadline access into isolated cron execution, skips low-budget fallback attempts, and updates cron/fallback tests, test routing, and the changelog.

## Best Possible Solution

Land a refreshed, narrow version that keeps the pre-attempt stop seam, preserves explicit stop reasons, avoids consuming cooldown probe slots for stopped attempts, and includes current-head terminal proof.

## Reproduction Assessment

Yes. Current main has no cron `beforeAttempt` fallback veto, and the PR body includes production logs plus terminal output showing the intended low-budget skip path; I did not run a live provider repro.

## Solution Assessment

No. The generic hook and execution-start deadline getter are the right direction, but the current patch still drops the stop reason in a skip-only path, burns cooldown probe slots before the stop check, and needs refreshed current-head proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve stop errors when no candidate has failed:** `src/agents/model-fallback.ts:1120-1121`
  - body: When `beforeAttempt` stops after only skip-only candidates have accumulated, `lastError` is still `undefined`, so this branch breaks into the generic fallback summary and drops the explicit stop message. Store or throw the hook error when there is no prior real failure so callers see the cron-budget stop reason.
  - confidence: 0.86
- **[P3] Run stop checks before marking cooldown probes:** `src/agents/model-fallback.ts:1092-1101`
  - body: The new hook runs after `markProbeAttempt`, so a low-budget stop on a cooldown probe consumes the provider probe throttle even though no provider call happens. Move the stop gate ahead of probe bookkeeping or mark the probe only once the attempt will run.
  - confidence: 0.78
