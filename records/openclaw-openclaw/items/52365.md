The PR adds a model-fallback pre-attempt stop hook and cron deadline plumbing so isolated cron jobs can skip fallback model attempts when too little cron budget remains.

## Best Possible Solution

Land a rebased version that preserves explicit stop errors when no real attempt has failed, keeps the execution-start deadline budget gate, and includes fresh redacted current-head runtime proof.

## Reproduction Assessment

Yes. Source inspection shows current main lacks a pre-attempt fallback veto and the cron abort check happens only after a candidate is selected; the PR discussion also includes production logs and a local low-budget repro, though I did not run live providers.

## Solution Assessment

No, not as-is. The internal hook and per-attempt budget direction are appropriate, but the patch should carry the explicit stop error when previous entries were only skips and needs fresh current-head proof.
