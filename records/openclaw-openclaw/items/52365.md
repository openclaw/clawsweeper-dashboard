## Summary

Keep this PR open: it targets a real cron fallback budget bug and the main implementation direction is useful, but the latest head still has stop-hook edge cases and the real behavior proof in the PR body was captured on an older head rather than the current force-pushed commit.

## What This Changes

The PR adds a generic model-fallback pre-attempt stop hook, threads cron deadline access into isolated cron execution, and skips fallback attempts when too little cron budget remains.

## Best Possible Solution

Land a rebased cron fallback-budget fix that preserves explicit stop reasons, checks the stop gate before consuming cooldown probe slots, refreshes current-head real behavior proof, and coordinates semantics with the related terminal-abort PR.

## Reproduction Assessment

Yes, by source inspection rather than a live run: current main has no pre-attempt fallback veto or cron deadline getter, and the PR's tests model the low-budget skip path. I did not run a live cron reproduction in this read-only review.

## Solution Assessment

No, not yet. The hook and execution-start deadline direction are maintainable, but the patch should preserve explicit stop errors for skip-only histories, avoid burning cooldown probe slots before a stopped attempt, and refresh current-head proof.
