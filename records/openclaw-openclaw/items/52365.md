## Summary

Keep open. The underlying cron fallback-budget bug is still present on current main, but this PR head is not merge-ready because it regresses current main's deferred isolated-timeout boundary and still drops the explicit stop reason in a skip-only fallback chain.

## What This Changes

The PR adds a model-fallback pre-attempt stop hook, propagates a cron deadline into isolated cron execution, skips fallback candidates when little cron budget remains, and adds cron/model-fallback regression tests.

## Best Possible Solution

Land a rebased, narrow fix that keeps current deferred-timeout cleanup semantics, adds the generic pre-attempt stop seam, computes the propagated deadline from the actual execution-start timeout, preserves explicit stop errors, and records a changelog fix.

## Reproduction Assessment

Yes. The PR body provides concrete production logs, and current-main source inspection confirms the fallback loop has no pre-attempt budget guard before candidate execution.

## Solution Assessment

No. The direction is maintainable, but this patch is not the best mergeable solution until it preserves current main's deferred cron timeout boundary and surfaces beforeAttempt stop errors when no previous candidate actually failed.
