## Summary

Keep open: the reported cron fallback bug is valid and the PR now has sufficient terminal proof, but the latest patch still has blocking correctness issues around the propagated deadline clock and stop-hook behavior.

## What This Changes

The PR adds a generic model-fallback `beforeAttempt` stop hook, threads cron deadline data into isolated cron jobs, skips low-budget cron fallbacks, updates tests and test routing, and adds a changelog entry.

## Best Possible Solution

Land a rebased patch that keeps the generic stop hook, derives the cron fallback deadline from the same execution-start clock as the abort timer, preserves explicit stop errors, avoids burning cooldown-probe throttle slots on stopped attempts, and adds focused regression coverage.

## Reproduction Assessment

Yes. The PR body includes production logs and terminal after-fix proof, and current-main source inspection shows cron-selected fallbacks have no pre-attempt budget gate before candidate execution.

## Solution Assessment

No. The direction is maintainable, but this patch is not the best mergeable form while the propagated deadline uses the wrong clock origin and stop-only paths can lose the hook's explicit error or mutate probe throttle state.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Anchor fallback deadlines to execution start:** `src/cron/service/timer.ts:117`
  - body: For detached `agentTurn` jobs, current main intentionally starts the real cron abort timer only when `onExecutionStarted` fires. Computing `deadlineAtMs` here before setup makes the new fallback guard count setup or queue time as spent, so it can skip a valid fallback while the actual timeout still has a full execution budget remaining.
  - confidence: 0.92
- **[P2] Preserve stop errors when no candidate failed:** `src/agents/model-fallback.ts:1120-1121`
  - body: If earlier candidates were only skipped for cooldown/auth and the budget hook stops the next candidate, `lastError` is still undefined. This branch breaks into the generic `All models failed` summary and drops the explicit `Skipping fallback...` error returned by the hook.
  - confidence: 0.87
- **[P3] Run stop hooks before marking cooldown probes:** `src/agents/model-fallback.ts:1092`
  - body: The new hook runs after cooldown probe bookkeeping, so a near-deadline stop can call `markProbeAttempt` for a provider that was never actually probed. The next run can then skip a real recovery probe until the throttle expires.
  - confidence: 0.78
