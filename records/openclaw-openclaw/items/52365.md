Full review comments:

- **[P2] Run stop hooks before cooldown probe bookkeeping:** `src/agents/model-fallback.ts:1676-1685`
  - body: `beforeAttempt` runs after the cooldown branch has already called `markProbeAttempt()` and logged a `probe_cooldown_candidate`. If the cron budget gate stops there, this records a provider probe slot even though no request was made, so a later cron run can skip the legitimate cooldown-recovery probe for the probe interval. Move the stop check ahead of probe bookkeeping, or delay marking the probe until an attempt will actually run.
  - confidence: 0.88
- **[P2] Preserve stop errors when no candidate failed:** `src/agents/model-fallback.ts:1704-1705`
  - body: When earlier candidates were only skipped for auth or cooldown, `lastError` is still undefined when `beforeAttempt` returns `stop`. This branch only logs `beforeAttempt.error` and then falls through to the generic fallback summary, so the explicit cron-budget stop reason is lost; store or throw the stop error for that path too.
  - confidence: 0.86
