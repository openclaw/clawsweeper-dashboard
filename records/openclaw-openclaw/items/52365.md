## Summary

Keep open. The reported production bug is still plausible on current main, but this active contributor PR is not merge-ready: the branch is outdated against the current cron timeout lifecycle, and the proposed stop hook still drops the explicit stop reason in a skip-only fallback chain.

## What This Changes

The PR adds a model-fallback pre-attempt stop hook, threads cron deadline metadata into isolated cron execution, skips fallback attempts when remaining budget is too low, and adds cron/model-fallback regression tests.

## Best Possible Solution

Land a rebased, narrow fix that preserves current `onExecutionStarted` and timed-out-run cleanup semantics, computes the propagated deadline when the outer cron timeout is actually armed, keeps the generic `beforeAttempt` seam, uses the prepared per-attempt timeout for the fallback threshold, and preserves explicit stop errors when no real candidate failure exists yet.

## Reproduction Assessment

Yes. The PR body gives a concrete production log where the primary consumes nearly all cron budget and the fallback is killed at 0 ms, and current-main source inspection confirms there is no pre-attempt stop hook before fallback candidate execution.

## Solution Assessment

No. The direction is maintainable, but the current patch is not the best mergeable solution because it starts deadline accounting before current main's execution-start boundary and still loses caller-provided stop messages after skip-only attempts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve the execution-start timeout boundary:** `src/cron/service/timer.ts:89-93`
  - body: Current main arms isolated cron timeouts from `onExecutionStarted`, so lane queue time does not consume the job's execution budget. This patch computes `deadlineAtMs` before `executeJobCore`, which lets queued work drain the propagated fallback budget and can skip fallbacks before the agent has actually started running.
  - confidence: 0.88
- **[P2] Preserve explicit stop errors after skip-only attempts:** `src/agents/model-fallback.ts:912-913`

This branch breaks and falls through to the generic fallback summary, dropping the hook's explicit `error` message, so the cron budget stop can still be reported as a misleading all-models-failed summary.
  - confidence: 0.82
