## What This Changes

The issue reports that host=node exec in full/off YOLO mode can return SYSTEM_RUN_DENIED after exec-approvals.json contains a persisted socket token.

## Best Possible Solution

Review and land a corrected version of the linked PR that makes full/off independent of socket-token presence while preserving token-file hardening and non-YOLO approval enforcement.

## Reproduction Assessment

Yes from source inspection for the core condition: current main only takes the no-socket full/off fast path when no socket token is present. The full Linux headless denial path still needs live reproduction because current node exec has a direct local-policy skip path.

## Solution Assessment

Yes in direction but not necessarily as the current linked PR stands. The narrow maintainable fix is to bypass socket approval in full/off mode regardless of token presence while still hardening token-bearing files and leaving non-YOLO approval paths strict.
