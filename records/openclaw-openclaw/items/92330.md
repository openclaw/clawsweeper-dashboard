## Summary

Keep open: current main and the latest release still carry the token-sensitive `resolveExecApprovals` fast path, and an open closing PR already owns the proposed fix path.

## What This Changes

The issue reports node-host exec denial in YOLO mode after remote approvals writes persist a socket token in `exec-approvals.json`.

## Best Possible Solution

Fix `resolveExecApprovals` so full/off never depends on an unavailable approval socket, while preserving owner-only handling for token-bearing approvals files when they are touched.

## Reproduction Assessment

No high-confidence live repro was established. Source and docs confirm the token-bearing YOLO policy edge, but I did not run a paired Linux node/gateway scenario in this read-only pass.

## Solution Assessment

Unclear until the linked PR is reviewed. Removing the token condition is the likely narrow fix, but maintainers should ensure the final patch does not accidentally weaken token-file hardening behavior.
