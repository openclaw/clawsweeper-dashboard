## What This Changes

Report that a token-bearing `exec-approvals.json` can make full/off `host=node` exec fall through the socket approval path and return `SYSTEM_RUN_DENIED`.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/92334 or a replacement that bypasses socket-token presence in full/off mode while preserving 0600 token-file hardening and proving the node `system.run` path.

## Reproduction Assessment

Yes for the source-level resolver path: create a token-bearing full/off approvals file and inspect `resolveExecApprovals`, which still falls through instead of returning the empty-token fast path. I did not run a live paired-node Kali reproduction.

## Solution Assessment

Yes directionally: the narrow fix belongs in `resolveExecApprovals`. The active PR or replacement should keep non-YOLO behavior strict and preserve token-file hardening.
