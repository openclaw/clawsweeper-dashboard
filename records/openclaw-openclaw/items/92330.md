Keep this open: current main and the latest release still have the token-gated YOLO fast path, and the issue is already paired with an open closing PR that should be reviewed rather than closing the issue early.

## What This Changes

The issue reports that node-host exec in YOLO mode returns `SYSTEM_RUN_DENIED` after the node approvals file contains a socket token.

## Best Possible Solution

Fix the YOLO path so `security=full, ask=off` does not fall through to socket approval because a token exists, while preserving token-file safety or making any permission tradeoff explicit in the linked PR review.

## Reproduction Assessment

Yes from source inspection, though not via live multi-node E2E in this read-only review. Current main still requires the socket token to be absent before the full/off fast path returns, while `ensureExecApprovals()` persists a token.

## Solution Assessment

Yes, the right repair is a narrow exec-approval fast-path change plus regression coverage. The issue should stay open until the linked fix PR lands or maintainers replace it with an equivalent safe patch.
