## Summary

Keep this issue open: current main and the latest release still have the token-gated full/off resolver path, and https://github.com/openclaw/openclaw/pull/92334 is the open closing PR for the same report. Source inspection supports the stored-token fast-path drift, but the exact Linux headless denial chain still needs live proof because current source only requires the socket exec host for macOS app-exec mode.

## What This Changes

The issue reports that externally writing a node `exec-approvals.json` can persist a socket token that makes `exec host=node` deny commands in YOLO mode.

## Best Possible Solution

Land a fix through the open closing PR or a replacement that bypasses socket use in full/off mode while preserving token-bearing file hardening, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes for the source-level token-gated resolver: current main still skips the no-token fast path when a socket token is present, and `ensureExecApprovals()` persists tokens. The full Linux paired-node denial needs live proof, so this is not a high-confidence end-to-end reproduction.

## Solution Assessment

Unclear as currently proposed: removing the token condition is the narrow likely direction, but the fix must preserve `0600` hardening for token-bearing files and prove the real node execution path.
