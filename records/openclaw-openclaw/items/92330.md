## What This Changes

Fix node-host exec so YOLO/full-off approvals do not fall through to socket-dependent denial when `exec-approvals.json` contains a socket token.

## Best Possible Solution

Review and land a narrow exec-approvals fix that keeps full/off on the no-persistence path regardless of socket-token presence, with focused regression coverage and node-host proof.

## Reproduction Assessment

Partially. Source inspection confirms that a token-bearing approvals file skips the current full/off no-persistence fast path; I did not run the full paired-node reproduction.

## Solution Assessment

Likely yes. Removing the socket-token predicate from the full/off fast path is the narrowest repair for no-approval mode; implementing a new socket server would be broader than the reported denial path needs.
