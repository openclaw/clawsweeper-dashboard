Keep this issue open: current main has not implemented the linked one-line token fast-path change, and there is an open linked fix PR that should be reviewed rather than closing the report early. ## What This Changes

The issue reports that `exec host=node` returns `SYSTEM_RUN_DENIED` in YOLO mode after the node approvals file contains a persisted socket token.

## Best Possible Solution

Keep this issue open until the linked PR is reviewed or replaced by a fix that proves `exec host=node` succeeds in full/off mode with a token-bearing node approvals file and adds focused regression coverage for the node-host path.

## Reproduction Assessment

No fresh paired-node live repro was run, so we do not have a high-confidence live reproduction. Source inspection does show a token-gated full/off fast path plus a node-host policy default/propagation path that can explain an approval-required result and warrants keeping the report open.

## Solution Assessment

Unclear. Removing the socket-token condition is a plausible narrow fix for the no-persistence invariant, but the best fix needs to be validated against the actual gateway-to-node `system.run` path because security/ask are not passed through the node invoke payload today.
