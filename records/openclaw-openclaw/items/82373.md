## Summary

Keep this PR open: the startup diagnostics direction is useful, but the current branch still has blocking channel lifecycle, typed-callsite, CI, security-classification, docs-navigation, and real-behavior-proof gaps, with a maintainer follow-up already asking for fixes and proof.

## What This Changes

The PR adds gateway startup runtime-state reporting, a `startup-doctor` CLI command, safe-mode startup flags, per-channel startup summaries/timeouts, redaction tests, and a new gateway startup resilience docs page.

## Best Possible Solution

Preserve the existing channel restart/backoff lifecycle, make startup summaries respect unmanaged channels, classify the local diagnostic socket probe, expose the docs through Gateway navigation, and validate with redacted real `openclaw gateway startup-doctor --json` output plus green focused checks.

## Reproduction Assessment

Yes for the review findings: source inspection shows the PR's early throw bypasses current restart/backoff flow and its `lastError` scan conflicts with the unmanaged-account health contract; live checks also show related failures. The claimed user-facing startup diagnostic behavior still lacks real runtime proof.

## Solution Assessment

No. The direction is useful, but the current implementation should derive startup summaries without bypassing existing lifecycle handling, preserve disabled/unconfigured channel semantics, satisfy the raw-socket guard, and get maintainer review for the new diagnostic contract.
