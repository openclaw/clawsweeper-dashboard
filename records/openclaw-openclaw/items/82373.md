  - body: `startAccount` returns the long-lived channel account task that cleanup and restart handling depend on. Calling it with `void` lets the tracked task resolve right after startup handoff, so the channel can be recorded as exited and restarted while transport work is still alive.

  - body: `getImmediateChannelFailure` treats any stopped account with `lastError` as a startup failure, but disabled and unconfigured accounts use `lastError` for expected reasons. Those normal states should not increment `channelsFailed` or appear as startup errors.
