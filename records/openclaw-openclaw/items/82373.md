Full review comments:

- **[P1] Track the channel account task through startup:** `src/gateway/server-channels.ts:548-558`
  - body: `startAccount` returns the long-lived channel account promise, but this line discards it. That lets the tracked task resolve after startup handoff, so cleanup/restart logic can mark the channel exited while the transport is still active; keep and await the returned promise in the tracked task as current main does.
  - confidence: 0.95
- **[P1] Preserve post-ready sidecar handles:** `src/gateway/server-startup-post-attach.ts:641`
  - body: Returning only `pluginServices` and `channelStartupSummary` drops the `postReadySidecars` handle path that current main uses for gateway-lifetime cleanup of watchers and other post-ready tasks. Carry the sidecar handles through the new summary return shape instead of replacing them.
  - confidence: 0.9
- **[P2] Do not classify skipped accounts as startup failures:** `src/gateway/server-channels.ts:820-826`
  - body: Disabled and unconfigured accounts intentionally record a `lastError` reason, but this getter treats any non-running account with `lastError` as a startup failure. That makes healthy installs with intentionally skipped channels look degraded, so gate this on actual startup failure state instead of generic status text.
  - confidence: 0.88
- **[P2] Move startup runtime state out of a JSON sidecar:** `src/gateway/startup-runtime-state.ts:152-154`
  - body: This writes OpenClaw-owned runtime state to a JSON file under the state directory and makes `startup-doctor` consume it. Root policy requires runtime state in state dirs to live in SQLite/shared state storage unless maintainers explicitly accept a named artifact contract.
  - confidence: 0.9
- **[P2] Remove or add the canon script target:** `package.json:1363-1365`
  - body: These new package scripts call `node scripts/canon.mjs`, but that file is not present on the PR head. Anyone running the scripts will hit a missing-module failure until the script is added or the package entries are removed.
  - confidence: 0.96
- **[P2] Classify the TCP diagnostic probe:** `src/cli/gateway-cli/register.ts:357`
  - body: This production CLI path adds a raw `net.createConnection` call. The repository raw-socket guard requires new net/tls/http2 client callsites to be classified as local-only, managed/proxied, diagnostic guarded, or unsupported before landing.
  - confidence: 0.91
