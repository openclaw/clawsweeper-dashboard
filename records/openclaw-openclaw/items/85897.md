## Summary

This should stay open: current main still stores IRC group reply context with the `irc:#channel` spelling and transient IRC sends still connect, `PRIVMSG`, and quit without joining the target channel first.

## What This Changes

The issue asks to canonicalize IRC group reply-route targets as `channel:#name` and make transient IRC channel sends join before `PRIVMSG`.

## Best Possible Solution

Implement the change inside the bundled IRC plugin: use `channel:#name` for group `To` and `OriginatingTo`, pass a transient join list only for channel targets, keep direct nick sends unchanged, and add focused regression tests.

## Reproduction Assessment

Yes, at source level: current main builds IRC group `To` and `OriginatingTo` as `irc:${peerId}` and transient sends do not pass channel join options before `PRIVMSG`. I did not run a live IRC server reproduction.

## Solution Assessment

Yes, the suggested plugin-local path is narrow and maintainable: reuse existing target normalization and join-option plumbing, while adding tests that direct nick sends stay unchanged.
