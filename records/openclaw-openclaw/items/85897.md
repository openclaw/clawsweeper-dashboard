## Summary

Keep this open: current main and the latest release still use the inconsistent IRC channel route spelling and still send transient channel messages without joining first. https://github.com/openclaw/openclaw/pull/85906 is now the linked implementation path, but it is open and not merged, so the issue should remain open until that PR is fixed and landed or closed.

## What This Changes

The issue requests IRC plugin changes to persist group reply routes as `channel:#name` and join channel targets before transient `PRIVMSG` sends while leaving direct nick sends unchanged.

## Best Possible Solution

Land a corrected plugin-local fix through https://github.com/openclaw/openclaw/pull/85906 or a replacement PR: use `channel:#name` for IRC group reply-route fields, join transient channel sends before `PRIVMSG`, preserve direct nick behavior, and add focused regression tests.
