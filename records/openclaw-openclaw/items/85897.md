## Summary

Keep this open: current main and the latest release still keep IRC group reply targets in the `irc:#channel` shape and transient channel sends still send `PRIVMSG` without first joining. The open implementation path is https://github.com/openclaw/openclaw/pull/85906, so this issue should stay open until that PR lands or is replaced.

## What This Changes

Request to store IRC group reply route metadata as `channel:#name` and make transient IRC channel sends join before `PRIVMSG`, while leaving direct nick sends unchanged.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/85906 or a narrow replacement that updates only the IRC plugin route metadata, transient channel join behavior, and focused regression tests.

## Reproduction Assessment

Yes, at source level: current main still builds IRC group reply metadata as `irc:${peerId}` and transient sends do not pass channel join options before `PRIVMSG`. I did not run a live IRC server reproduction.

## Solution Assessment

Yes. The narrow plugin-local fix is to reuse existing target normalization and connect-option channel joining, preserve direct nick sends, and add focused inbound/send tests.
