## Summary

Keep open: current main and v2026.6.6 still provide authenticated Gateway WebSocket WebChat, browser pairing, PWA/Web Push, DM session scoping, and provider-backed Talk/WebRTC voice, but not the requested browser-held E2EE chat envelopes, browser key negotiation, or per-device/per-agent encrypted browser chat transport.

## What This Changes

The issue requests a native browser-based end-to-end encrypted chat transport for OpenClaw instances without third-party chat setup.

## Best Possible Solution

Define a maintainer-approved encrypted-browser-channel capability and threat model first, then implement protocol, UI, key lifecycle, routing, transcript, relay/recovery, docs, and contract tests under the chosen core or plugin boundary.
