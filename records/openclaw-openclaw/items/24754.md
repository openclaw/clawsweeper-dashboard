## Summary

## What This Changes

The issue requests a native browser-based E2EE chat transport with no third-party chat provider, browser-held keys, and per-agent/per-device isolation.

## Best Possible Solution

Define an additive Gateway/browser E2EE chat architecture first, then implement protocol fields, browser-held key lifecycle, isolated routing, transcript/storage semantics, relay policy, UI, docs, and contract tests after product and security review.
