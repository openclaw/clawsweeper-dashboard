## Summary

Keep open: current main and the latest release still provide authenticated Gateway WebSocket WebChat, device identity/pairing, DM session isolation options, and provider-backed Talk/WebRTC voice, but not the requested browser-held E2EE chat transport with encrypted envelopes, browser key negotiation, or per-device/per-agent encrypted browser channels.

## What This Changes

Request a native browser-based E2EE chat layer for OpenClaw so users can chat with agents from a browser without a third-party messaging app or account.

## Best Possible Solution

Define a maintainer-approved encrypted browser-channel capability and threat model first, then implement protocol, UI, key lifecycle, routing, transcript, relay/recovery, docs, and contract tests under the chosen core or plugin boundary.

## Solution Assessment
