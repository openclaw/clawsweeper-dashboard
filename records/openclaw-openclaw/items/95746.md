## Summary

## What This Changes

The issue reports that memory-core Dream Diary/dreaming work can run multiple local-model narrative subagents in parallel and exhaust LM Studio context/KV cache despite configured concurrency limits.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/95775 or an equivalent narrow fix that queues heartbeat/internal Dream Diary narrative subagent work while preserving existing cron-detached behavior; avoid adding a separate public config knob unless maintainers choose that broader contract.

## Reproduction Assessment

No live high-confidence reproduction was run against LM Studio. Source inspection shows the non-detached heartbeat/internal path still calls the base narrative generator directly, and the issue includes concurrent local-model context/KV exhaustion logs.

## Solution Assessment

Yes, the best path is a narrow fix that reuses the existing queue for heartbeat/internal Dream Diary narratives while preserving cron detached behavior. A new public memory-core concurrency option is a separate product choice, not required for this bug fix.
