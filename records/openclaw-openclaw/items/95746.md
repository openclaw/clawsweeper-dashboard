## Summary

Keep open: current main still has a source-reproducible gap where managed dreaming handled through the heartbeat path can call Dream Diary narrative generation without the detached narrative queue, and the directly linked fix PR is open but unmerged.

## What This Changes

The issue asks memory-core Dream Diary/dreaming narrative subagent work to respect concurrency limits, or to expose a memory-core narrative concurrency knob for local-model users.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/95775 or an equivalent narrow fix that bounds heartbeat-triggered Dream Diary narrative runs without adding a conflicting memory-core config surface.

## Reproduction Assessment

No live high-confidence reproduction was run against LM Studio. Source inspection does show the managed heartbeat path can bypass the detached narrative queue while the reporter's logs show concurrent local-model context/KV exhaustion.

## Solution Assessment

Yes, the best path is a narrow implementation that reuses the existing narrative queue for heartbeat-triggered subagent narratives. A new config option may be a later product choice, but it is not required to fix this bug.
