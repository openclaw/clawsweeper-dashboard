## Summary

Keep open: current main still has a source-reproducible path where heartbeat-handled managed dreaming can call Dream Diary narrative subagents without the shared narrative concurrency queue, and the linked fix PR is open but unmerged.

## What This Changes

The issue asks memory-core Dream Diary/dreaming subagent work to respect concurrency limits or otherwise provide a way to serialize local-model narrative runs.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/95775 or an equivalent narrow fix that reuses the shared narrative queue for heartbeat/non-detached Dream Diary subagent runs while preserving intentional detached-cron behavior.

## Reproduction Assessment

Source-reproducible, but not live-reproduced: current main shows managed dreaming can be handled as a heartbeat trigger and then call non-queued Dream Diary subagent generation, matching the reporter's concurrent local-model slot exhaustion logs.

## Solution Assessment

Yes: the best fix is to reuse the existing narrative queue for heartbeat/non-detached subagent narratives. A new memory-core config knob is not required for the bug unless maintainers later want tunable policy.
