- `P1`: The report describes a user-facing WebChat regression where each keystroke can stall for 500ms-1s in the primary composer workflow.

## Summary

Keep open. Current main and latest v2026.5.18 still route each WebChat input event through a host update path, and the focused fix remains open, conflicting, and unmerged.

## What This Changes

The issue asks to eliminate 500ms-1s keystroke latency in the Control UI WebChat composer.

## Best Possible Solution

Land or replace the focused WebChat fix so ordinary typing does not update Lit host state on each input, while preserving slash-menu, history recall, failed-send restore, and detached-send draft sync with tests and browser proof.

## Reproduction Assessment

Yes, at source level. Current main wires textarea input to `updateSlashMenu` and `onDraftChange`, and `onDraftChange` assigns a Lit `@state()` draft value, although no live browser timing trace was run.

## Solution Assessment

Yes, keeping this open is the right outcome. The maintainable fix is a current-main WebChat repair that moves ordinary draft typing out of reactive state and explicitly refreshes only the programmatic draft paths that need DOM sync.
