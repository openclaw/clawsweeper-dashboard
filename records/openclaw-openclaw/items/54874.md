## Summary

Keep open. Current main still has the WebChat per-keystroke reactive draft render path, and the focused implementation PR remains open, conflicting, and unmerged.

## What This Changes

Typing in the Control UI WebChat input is reported to lag 500ms-1s per keystroke because draft changes drive reactive Lit renders.

## Best Possible Solution

Land or replace the focused WebChat fix so ordinary draft typing does not update Lit reactive state while slash menu, history recall, failed-send restore, and detached-send draft sync stay explicit and tested.

## Reproduction Assessment

Yes, at source level. Current main wires textarea input to `onDraftChange`, which assigns `chatMessage` while it is Lit `@state()`, matching the reported per-keystroke host render path.

## Solution Assessment

Yes for the overall direction, but not by closing this issue yet. The best path is to review, update, and land or replace https://github.com/openclaw/openclaw/pull/62790 with current-main coverage and real browser proof.
