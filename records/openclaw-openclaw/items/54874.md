## Summary

Keep open. Current main and the latest release still keep the WebChat draft as Lit reactive state and still route every textarea input through that state, while https://github.com/openclaw/openclaw/pull/62790 remains the open, unresolved implementation path.

## What This Changes

The issue reports a WebChat regression where typing in the composer can lag 500ms-1s per keystroke because draft input appears to trigger expensive UI rendering.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/62790 with a current-main fix that keeps ordinary typing outside expensive host renders while preserving slash-menu, history recall, abort/send draft restore, and detached-send behavior.

## Reproduction Assessment

Yes, at source level: current main routes textarea input through `updateSlashMenu` and `onDraftChange`, and `onDraftChange` mutates Lit `@state()` draft state. I did not run a browser performance trace, so the exact delay duration is not measured here.

## Solution Assessment

Yes, keeping this issue open is the best current outcome because neither current main nor v2026.5.22 contains the fix. The narrow solution is to repair, land, or replace the focused WebChat PR rather than closing this issue first.
