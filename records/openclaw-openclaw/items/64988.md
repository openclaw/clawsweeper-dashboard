## Summary

Keep this PR open: the requested provider inference is still useful and not fully implemented on current main for selected persisted session overrides, but the proposed gateway post-processor has blocking resolver-ordering defects and still lacks real behavior proof.

## What This Changes

The PR adds a gateway post-processing step and tests to infer a unique configured provider for bare `resolveSessionModelRef()` model overrides.

## Best Possible Solution

Implement selected-model configured-provider inference in the shared model-selection resolver while preserving explicit providers, stored runtime identity, nested slash refs, ambiguity handling, and focused regression coverage.

## Reproduction Assessment

Yes. Source review of the PR diff gives high-confidence paths for the defects: slash-qualified models are parsed before `providerOverride`, stored runtime providers return before inference, and non-OpenAI defaults skip inference.

## Solution Assessment

No. The requested behavior is useful, but a gateway-only post-processor duplicates shared model-selection semantics and introduces ordering holes; the safer path is shared resolver behavior with regression tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve explicit providers before parsing slash models:** `src/gateway/session-utils.ts:1048-1054`
  - body: `normalizedModel.includes("/")` runs before checking `entry.providerOverride`, so `providerOverride: "openrouter"` with `modelOverride: "anthropic/claude-haiku-4.5"` is rewritten to provider `anthropic`, breaking wrapper-provider routing.
  - confidence: 0.91
- **[P2] Do not let stored runtime providers suppress inference:** `src/gateway/session-utils.ts:1073-1078`
  - body: When `entry.modelProvider` is present, this branch returns the stored runtime provider before the inferred configured provider can be used, so existing sessions can keep routing a bare selected override through stale/default runtime metadata.
  - confidence: 0.86
- **[P2] Infer configured providers for non-OpenAI defaults:** `src/gateway/session-utils.ts:1079-1084`
  - body: The new inference only applies when the resolved provider is empty or `openai`; with a non-OpenAI default such as `anthropic`, a unique configured `openai-codex/gpt-5.4` match is ignored and the session stays on the wrong provider.
  - confidence: 0.84
