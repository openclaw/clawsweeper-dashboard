## Summary

Keep open: the PR targets a useful gateway model-routing gap, but the current diff is not merge-ready because it can override explicit wrapper providers, misses existing-session and non-OpenAI-default cases, and still lacks real behavior proof.

## What This Changes

The PR adds a `resolveSessionModelRef()` normalization step and tests so bare canonical model overrides can infer a unique configured provider.

## Best Possible Solution

Implement configured-provider inference in the shared selected-model contract while preserving explicit providers, wrapper slash-models, stored runtime metadata, ambiguity handling, and focused regression coverage plus redacted runtime proof.

## Reproduction Assessment

Yes for the PR defects: the diff shows slash-qualified models are parsed before providerOverride, stored runtime providers return before inference, and the inference gate only handles the default OpenAI provider. No live runtime proof is present.

## Solution Assessment

No. The requested behavior is useful, but a gateway-only post-processor duplicates shared model-selection semantics and creates ordering holes; the maintainable solution belongs in the shared selected-model resolver contract.
