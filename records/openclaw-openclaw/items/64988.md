## What This Changes

The PR adds a gateway-side normalization step and focused tests so bare canonical model overrides can infer a unique provider from configured model families.

## Best Possible Solution

Implement configured-provider inference in the shared selected-model resolver while preserving explicit provider overrides, slash-qualified wrapper models, stored runtime metadata, ambiguity handling, and regression coverage.

## Reproduction Assessment

Yes for the PR defects: the diff shows slash-qualified models are parsed before providerOverride and stored runtime providers return before inference. No live runtime proof was run because this was a read-only review.

## Solution Assessment

No. The requested behavior is useful, but a gateway-only post-processor duplicates shared model-selection semantics and creates ordering holes; the maintainable solution belongs in the shared resolver contract.

  - confidence: 0.84
