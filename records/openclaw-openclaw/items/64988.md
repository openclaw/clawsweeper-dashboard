## Summary

Keep open: current main still does not implement bare-model configured-provider inference for this selected-session path, and the PR branch has concrete provider-routing regressions plus missing after-fix real behavior proof.

## What This Changes

The PR adds a gateway session model-ref normalization helper and a focused test file so bare `modelOverride` values can infer a unique configured provider.

## Best Possible Solution

Move bare-model configured-provider inference into the shared selected-model resolver semantics, preserving explicit providers, wrapper model ids, runtime-only sessions, ambiguous configured matches, and non-default defaults with focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists from the PR diff: slash-containing override models are parsed before explicit providers, stored runtime providers can suppress inference, and inference only runs for empty or `openai` fallback providers.

## Solution Assessment

No. The requested behavior is useful, but a gateway-only post-processor is not the best fix because selected-model semantics already live in shared resolver code and current tests protect wrapper-provider behavior there.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Preserve explicit providers before parsing slash models:** `src/gateway/session-utils.ts:1049-1054`
  - body: `normalizedModel.includes("/")` is handled before `entry.providerOverride`, so `providerOverride: "openrouter"` with `modelOverride: "anthropic/claude-haiku-4.5"` is rewritten to provider `anthropic`. That regresses wrapper-provider routing and sends the session through the wrong provider.
  - confidence: 0.91
- **[P2] Do not let stored runtime providers suppress inference:** `src/gateway/session-utils.ts:1065-1078`
  - body: When `entry.modelProvider` is present, this branch returns the stored runtime provider before the inferred configured provider can be used. Existing sessions that have already run can keep routing a bare selected override through stale/default runtime metadata instead of the unique configured provider this feature is trying to select.
  - confidence: 0.86
- **[P2] Infer from configured models for any fallback provider:** `src/gateway/session-utils.ts:1077-1081`
  - body: The new inference only applies when `params.ref.provider` is empty or equal to `DEFAULT_PROVIDER`. If the configured default provider is `anthropic` and `gpt-5.4` uniquely belongs to `openai-codex`, the resolver leaves the session on `anthropic/gpt-5.4`, so the feature fails for non-OpenAI defaults.
  - confidence: 0.84
