## Summary

Keep this PR open, but the current patch should not merge as-is: the requested gateway behavior still is not on current main, while the PR introduces routing regressions and still lacks real after-fix behavior proof.

## What This Changes

The PR adds gateway session model-reference normalization so bare canonical model overrides can infer a unique configured provider from configured model families.

## Best Possible Solution

Implement bare selected-model configured-provider inference in the shared selected-model resolver path while preserving explicit wrapper providers, stored runtime-session semantics, ambiguous matches, and non-default configured defaults with focused regression coverage.

## Reproduction Assessment

Yes. Source review gives a high-confidence path for the patch defects: slash-qualified models are parsed before `providerOverride`, stored `runtimeProvider` returns before inference, and the `DEFAULT_PROVIDER` gate skips non-OpenAI configured defaults.

## Solution Assessment

No. The behavior is useful, but this gateway-only post-processor is not the best fix because selected-model semantics and configured-provider inference already live in the shared model-selection layer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Preserve explicit providers before parsing slash models:** `src/gateway/session-utils.ts:1049-1054`
  - body: `normalizedModel.includes("/")` is handled before `entry.providerOverride`, so `providerOverride: "openrouter"` with `modelOverride: "anthropic/claude-haiku-4.5"` is rewritten to provider `anthropic`. That breaks existing wrapper-provider routing and sends the session through the wrong provider.
  - confidence: 0.91
- **[P2] Do not let stored runtime providers suppress inference:** `src/gateway/session-utils.ts:1065-1078`
  - body: When `entry.modelProvider` is present, this branch returns the stored runtime provider before the inferred configured provider can be used. Existing sessions can keep routing a bare selected override through stale/default runtime metadata instead of the unique configured provider this feature is trying to select.
  - confidence: 0.86
- **[P2] Infer configured providers for non-OpenAI defaults:** `src/gateway/session-utils.ts:1077-1081`
  - body: The new inference only applies when `params.ref.provider` is empty or equal to `DEFAULT_PROVIDER`. If the configured default provider is `anthropic` and `gpt-5.4` uniquely belongs to `openai-codex`, the resolver leaves the session on `anthropic/gpt-5.4`, so the feature fails for non-OpenAI default setups.
  - confidence: 0.84
