## Summary

Keep open: the idea is still unique on current main, but the proposed resolver ordering remains unsafe for stored providers and non-OpenAI defaults, and the external PR still lacks real behavior proof.

## What This Changes

The PR adds a gateway resolver post-processing helper and tests so a bare selected model can infer a unique provider from configured model families.

## Best Possible Solution

Move configured-provider inference into the shared selected-model resolver contract while preserving explicit overrides, wrapper-provider refs, stored runtime semantics, ambiguity handling, and current Codex routing policy.

## Reproduction Assessment

Yes for the PR defects: source inspection of the PR diff and GitHub potential merge result shows stored providers can suppress inference and non-OpenAI defaults skip configured-provider inference. No live configured OpenClaw run was performed in this read-only review.

## Solution Assessment

No. The requested behavior is useful, but a gateway-only post-processor is not the best merge shape until the shared resolver contract defines override, runtime, wrapper-provider, and Codex legacy routing semantics.
