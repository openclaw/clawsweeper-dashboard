## Summary

Keep open: current main still does not cover the selected-session bare model override inference this PR targets, but the PR should not merge as-is because the new gateway post-processor breaks explicit provider precedence, misses existing-session and non-OpenAI-default cases, and lacks required real behavior proof.

## What This Changes

The PR adds gateway session model-reference normalization so bare canonical model overrides can infer a unique configured provider from configured model families.

## Best Possible Solution

Implement bare selected-model configured-provider inference in the shared selected-model resolver path while preserving explicit providers, runtime identity semantics, slash/nested wrapper refs, ambiguous matches, and non-default configured defaults with focused regression coverage.

## Reproduction Assessment

Yes. Source review gives high-confidence paths for the PR defects: slash-qualified models are parsed before `providerOverride`, stored `runtimeProvider` returns before inference, and the `DEFAULT_PROVIDER` gate skips non-OpenAI configured defaults.

## Solution Assessment

No. The requested behavior is useful, but a gateway-only post-processor after selected-model resolution duplicates shared model-selection semantics and introduces ordering holes; the safer fix belongs in the shared selected-model resolver path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve explicit providers before parsing slash models:** `src/gateway/session-utils.ts:1048-1051`
  - body: `normalizedModel.includes("/")` is handled before `entry.providerOverride`, so `providerOverride: "openrouter"` with `modelOverride: "anthropic/claude-haiku-4.5"` is rewritten to provider `anthropic`. That breaks wrapper-provider routing and sends the session through the wrong provider.
  - confidence: 0.91
- **[P2] Do not let stored runtime providers suppress inference:** `src/gateway/session-utils.ts:1073-1078`
  - body: When `entry.modelProvider` is present, this branch returns the stored runtime provider before the inferred configured provider can be used. Existing sessions can keep routing a bare selected override through stale/default runtime metadata instead of the unique configured provider this feature is trying to select.
  - confidence: 0.86
- **[P2] Infer configured providers for non-OpenAI defaults:** `src/gateway/session-utils.ts:1079-1084`
  - body: The new inference only applies when `params.ref.provider` is empty or `openai`. If an agent default is `anthropic` and `gpt-5.4` uniquely belongs to `openai-codex`, the resolver leaves the session on `anthropic/gpt-5.4`, so the feature fails for non-OpenAI default setups.
  - confidence: 0.84
