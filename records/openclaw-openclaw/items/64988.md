## Summary

Keep open: current main still does not implement bare session model override provider inference, but this PR cannot merge as-is because the proposed gateway post-processor regresses explicit wrapper-provider routing, misses existing-session/runtime-provider cases, and lacks contributor real behavior proof.

## What This Changes

The PR adds gateway session model-reference normalization and focused tests so bare canonical model overrides can infer a unique configured provider.

## Best Possible Solution

Implement bare-model configured-provider inference in the shared selected-model resolver path, preserving explicit wrapper providers, ambiguous matches, stored runtime sessions, and non-default configured defaults with focused regression coverage.

## Reproduction Assessment

Yes. Source reproduction is high-confidence from the PR diff: slash-qualified models are parsed before providerOverride, runtimeProvider returns before inference, and the DEFAULT_PROVIDER gate skips non-OpenAI defaults.

## Solution Assessment

No. The behavior is useful, but a gateway-only post-processor is not the best fix because persisted selected-model semantics and configured-provider inference already live in shared model-selection code.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Preserve explicit providers before parsing slash models:** `src/gateway/session-utils.ts:1049-1054`
  - body: normalizedModel.includes("/") is handled before entry.providerOverride, so providerOverride: "openrouter" with modelOverride: "anthropic/claude-haiku-4.5" is rewritten to provider anthropic. That regresses wrapper-provider routing and sends the session through the wrong provider.
  - confidence: 0.91
- **[P2] Do not let stored runtime providers suppress inference:** `src/gateway/session-utils.ts:1065-1078`
  - body: When entry.modelProvider is present, this branch returns the stored runtime provider before the inferred configured provider can be used. Existing sessions can keep routing a bare selected override through stale/default runtime metadata instead of the unique configured provider this feature is trying to select.
  - confidence: 0.86
- **[P2] Infer from configured models for any fallback provider:** `src/gateway/session-utils.ts:1077-1081`
  - body: The new inference only applies when params.ref.provider is empty or equal to DEFAULT_PROVIDER. If the configured default provider is anthropic and gpt-5.4 uniquely belongs to openai-codex, the resolver leaves the session on anthropic/gpt-5.4, so the feature fails for non-OpenAI defaults.
  - confidence: 0.84
