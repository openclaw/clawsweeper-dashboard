## Summary

Keep this PR open. The requested selected-session model routing behavior is still not fully covered on current main, but the proposed normalizer introduces semantic regressions around explicit wrapper providers, stored runtime metadata, and non-default provider setups.

## What This Changes

The PR adds a gateway session model-ref normalization helper and a focused test file so bare session model overrides can infer a unique configured provider.

## Best Possible Solution

Implement selected-session bare-model inference in the shared selected-model resolver flow, preserving explicit `providerOverride`, slash-containing wrapper model ids, stored runtime-only sessions, ambiguous configured matches, and non-default configured providers. Add focused gateway regression coverage for those cases.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: with `providerOverride: "openrouter"` and `modelOverride: "anthropic/claude-haiku-4.5"`, the PR's slash branch reparses the model as provider `anthropic`; with a non-OpenAI default provider and bare `modelOverride: "gpt-5.4"`, the PR skips configured-provider inference.

## Solution Assessment

No. The goal is valid, but the PR's post-processing normalizer is not the best fix because it partially bypasses existing selected-model semantics; extending the shared resolver path is safer and narrower.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Preserve explicit providers before parsing slash models:** `src/gateway/session-utils.ts:1049-1054`
  - body: `normalizedModel.includes("/")` is handled before `entry.providerOverride`, so a wrapper selection such as `providerOverride: "openrouter"` with `modelOverride: "anthropic/claude-haiku-4.5"` is rewritten to provider `anthropic`. That regresses the current wrapper-provider contract and routes the session through the wrong provider.
  - confidence: 0.91
- **[P2] Do not let stored runtime providers suppress inference:** `src/gateway/session-utils.ts:1065-1078`
  - body: When an existing session has `entry.modelProvider`, this branch returns that stored runtime provider before the inferred configured provider can be used. Sessions that have already run once can keep routing a bare selected override through stale/default runtime metadata instead of the unique configured provider this feature is trying to select.
  - confidence: 0.86
- **[P2] Infer from configured models for any fallback provider:** `src/gateway/session-utils.ts:1077-1081`
  - body: The new inference only applies when `params.ref.provider` is empty or equal to `DEFAULT_PROVIDER`. If the configured default provider is `anthropic` and `gpt-5.4` uniquely belongs to `openai-codex`, the resolver leaves the session on `anthropic/gpt-5.4`, so the feature fails for non-OpenAI defaults.
  - confidence: 0.84
