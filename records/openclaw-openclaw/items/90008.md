## What This Changes

The issue requests stdin-safe installer subprocess handling so `curl | bash` cannot corrupt the script stream while direct `bash install.sh` keeps interactive prompt stdin.

## Best Possible Solution

Land a focused installer fix that isolates stdin for piped installs while preserving terminal stdin for direct or logged interactive installs, then close this issue and link the older related installer report.

## Reproduction Assessment

Yes, source-reproducible: the documented `curl | bash` path feeds the installer through stdin, current main lets gum-wrapped and fallback subprocesses inherit stdin, and gum v0.17.0 passes `os.Stdin` to the wrapped command. I did not run a full live WSL2 installer reproduction in this read-only review.

## Solution Assessment
