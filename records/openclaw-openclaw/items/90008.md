## Summary

Keep open: current main and the latest release still expose the documented piped installer path with child commands that can inherit the installer stdin stream, and the linked implementation PR is open rather than merged.

## What This Changes

The issue reports that piped installer runs let gum/npm child processes consume the script stdin while direct runs should preserve terminal stdin for prompts.

## Best Possible Solution

Review and land, reject, or replace https://github.com/openclaw/openclaw/pull/87799; once focused stdin isolation reaches main, close this issue as implemented.

## Reproduction Assessment

Yes, source-reproducible: docs pipe the installer through stdin, current main lets key installer subprocesses inherit stdin, and gum v0.17.0 forwards `os.Stdin` to wrapped commands. I did not run a full live WSL2 installer reproduction in this read-only review.
