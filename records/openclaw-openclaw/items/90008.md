## Summary

Keep open: current main and latest release v2026.6.6 still expose the installer stdin-inheritance path, and the linked closing PR is open and unmerged, so this issue remains necessary as the tracking item.

## What This Changes

The issue reports that `curl | bash` installer subprocesses can inherit the installer stream under `gum spin`, while direct `bash install.sh` fallback paths must preserve interactive stdin.

## Best Possible Solution

Land or replace the focused installer fix that isolates subprocess stdin for piped or non-interactive installs while preserving terminal stdin for direct interactive installs, then close this issue and the related installer issue through the merged PR.

## Reproduction Assessment

Yes, source-reproducible: the documented install command feeds the script through stdin, current main lets installer subprocesses inherit stdin, and gum v0.17.0 forwards `os.Stdin` when the spinner path is active. I did not run a live installer in this read-only review.

## Solution Assessment

Yes for routing: the best way to solve this issue is to review and land or replace the linked installer PR. Closing before merge would lose tracking for a bug that is still present on current main and v2026.6.6.
