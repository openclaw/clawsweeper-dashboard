## Summary

Keep open: current main and latest stable release still expose the documented piped installer path with unisolated child-command stdin, and the open linked PR is the active implementation candidate rather than merged fix provenance.

## What This Changes

The issue asks OpenClaw to prevent piped installer child commands from consuming the `curl | bash` script stream while preserving terminal stdin for direct interactive installer runs.

## Best Possible Solution

Review and land, reject, or replace https://github.com/openclaw/openclaw/pull/87799; once focused stdin isolation reaches main, close this issue as implemented.

## Reproduction Assessment

Yes, source-reproducible: current docs still pipe the installer through stdin, current main lets gum-wrapped and fallback installer subprocesses inherit stdin, and gum v0.17.0 forwards `os.Stdin` to wrapped commands. I did not run a full live WSL2 installer reproduction in this read-only review.

## Solution Assessment

Yes for routing: the best path is focused stdin-consumer hardening through the open linked PR or a maintainer-approved replacement. Closing before that PR merges would drop tracking for behavior still present on main and v2026.6.8.
