## Summary

Keep open: current main and latest release v2026.6.8 still have installer subprocess paths that can inherit the piped script stdin, and https://github.com/openclaw/openclaw/pull/87799 is still the open closing implementation candidate rather than a merged fix.

## What This Changes

The issue requests installer stdin handling that prevents `curl | bash` child processes from consuming the script stream while preserving terminal stdin for direct interactive installs.

## Best Possible Solution

Review and land, reject, or replace https://github.com/openclaw/openclaw/pull/87799; once a focused stdin-isolation fix reaches main, close this issue and the related duplicate as implemented.

## Reproduction Assessment

Yes, source-reproducible: current docs pipe the installer through stdin, current main lets key installer subprocesses inherit stdin, and gum v0.17.0 forwards `os.Stdin` to wrapped commands. I did not run a full live installer reproduction in this read-only review.

## Solution Assessment

Yes for routing: the best fix path is the focused consumer-level stdin hardening in the open linked PR or a maintainer-approved replacement. Closing this issue before that PR merges would drop tracking for behavior still present on main and v2026.6.8.
