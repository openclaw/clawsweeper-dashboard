## Summary

Keep open. This PR is not safe maintainer cleanup: it is dirty against current main, mixes a narrow config-message request with broad Feishu localization and validation behavior changes, and still has concrete review blockers in the proposed patch.

## What This Changes

The PR adds Zod `unrecognized_keys` formatting in config validation, adds several plugin/channel schema compatibility and defaulting changes, and localizes multiple Feishu error messages and tests into Chinese.

## Best Possible Solution

Split this into a narrow config-validation change: teach the shared JSON Schema/AJV formatter to include `additionalProperty`, keep Zod key-name behavior covered, and add focused tests for channel/plugin config failures. Feishu localization or agent-prompt wording should be handled separately with explicit maintainer approval.

## Solution Assessment

No. The proposed patch is not the best way to solve the issue because it handles Zod `unrecognized_keys` but misses the AJV `additionalProperties` path shown in the example, while adding unrelated Feishu and plugin-validation behavior changes.
