## Summary

Keep open: current main still does not surface AJV additional-property names, and this PR is not safe to merge as-is because it mixes the formatter change with unrelated Feishu localization and a plugin allowlist validation downgrade.

## What This Changes

The PR adds Zod unrecognized-key formatting, channel/plugin config validation/defaulting compatibility changes, and Chinese-localized Feishu error-message updates with test expectation changes.

## Best Possible Solution

Split this into a narrow shared AJV formatter fix that includes `params.additionalProperty` in `additionalProperties` errors, with focused config/schema-validator tests; handle Feishu localization and plugin allowlist policy separately with owner approval.

## Reproduction Assessment

Yes. A focused test with `validateJsonSchemaValue` against an object schema using `additionalProperties: false` and an unknown key, or a `channels.feishu` config containing an unknown field, reproduces the generic AJV message path.

## Solution Assessment

No. The PR handles Zod `unrecognized_keys`, but the reported Feishu example uses the AJV path; the maintainable fix belongs in `src/plugins/schema-validator.ts` rather than bundling unrelated Feishu and plugin policy changes.
