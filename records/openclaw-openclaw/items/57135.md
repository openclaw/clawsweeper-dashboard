## Summary

Keep open, but do not merge this branch as-is. The useful user problem is real, but the patch handles Zod `unrecognized_keys` while the reported Feishu example flows through JSON Schema/AJV `additionalProperties`, and the branch also carries unrelated Feishu localization plus plugin allowlist validation changes.

## What This Changes

The PR adds Zod `unrecognized_keys` formatting in config validation, adds plugin/channel schema compatibility and defaulting behavior, and localizes several Feishu error messages and tests into Chinese.

## Best Possible Solution

Replace this with a narrow config-validation patch that formats AJV `additionalProperties` errors with the offending property name, adds focused tests for JSON Schema/channel config validation, and leaves Feishu localization plus plugin allowlist policy for separate maintainer-reviewed work.

## Reproduction Assessment

Yes. A focused test can validate a JSON Schema object with `additionalProperties: false` and an unknown key through `validateJsonSchemaValue` or channel config validation; current main still formats that path from AJV's generic `error.message`.

## Solution Assessment

No. The PR is not the best way to solve the issue because it changes Zod issue formatting while the example uses the AJV path, and it bundles unrelated Feishu prompt/localization and plugin allowlist behavior.
