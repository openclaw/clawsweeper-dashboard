## Summary

Keep open: the classifier addition may be useful, but the PR still relaxes the Codex app-server refresh callback in a way that conflicts with upstream Codex's Unauthorized recovery contract and is not merge-ready.

## What This Changes

The PR changes Codex app-server OAuth refresh handling to reuse still-valid credentials and updates refresh-failure classification for app-session termination messages.

## Best Possible Solution

Preserve real provider refresh for Unauthorized app-server callbacks; split the classifier-only change if useful, or make any no-op optimization reason-aware and prove it with redacted live Codex app-server evidence.

## Reproduction Assessment

Yes, source-level: upstream Codex emits account/chatgptAuthTokens/refresh for Unauthorized recovery, and this PR can return the same locally usable token instead of forcing refresh. I did not run a live OAuth repro in this read-only review.

## Solution Assessment

No. The proposed fix is too broad because it treats an Unauthorized recovery callback as a local expiry check; the safer solution is to keep forced refresh for Unauthorized callbacks and split any proactive no-op behavior into a separate path.
