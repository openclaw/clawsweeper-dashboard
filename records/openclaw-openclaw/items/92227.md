## Summary

Keep open: current main has not implemented this PR, and the branch still appears incorrect because it can bypass a real refresh on Codex Unauthorized callbacks; it is also mock-proof-only and currently merge-conflicting.

## What This Changes

The PR makes Codex app-server token refresh callbacks reuse locally usable OpenAI/Codex OAuth credentials and broadens OAuth refresh-failure classification for app-session termination messages.

## Best Possible Solution

Preserve real OAuth refresh for Unauthorized app-server callbacks; split the classifier-only change if useful, or make any no-op optimization reason-aware and prove it with redacted live Codex app-server evidence.

## Reproduction Assessment

Yes, source-level: upstream Codex emits `account/chatgptAuthTokens/refresh` for Unauthorized recovery, and this PR can return the same locally usable token instead of forcing refresh. I did not run a live OAuth repro in this read-only review.

## Solution Assessment

No. The proposed fix is too broad because it treats an Unauthorized recovery callback as a local expiry check; the safer solution is to keep forced refresh for Unauthorized callbacks and split any proactive no-op behavior into a separate path.
