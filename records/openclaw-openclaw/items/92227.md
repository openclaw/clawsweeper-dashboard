## Summary

Keep open: this is a focused Codex auth bug-fix attempt, but the current patch is not merge-ready because it can answer Codex app-server 401 refresh callbacks with the same locally-valid token instead of refreshing it, and the PR proof is unit-test-only.

## What This Changes

The branch changes the Codex app-server auth bridge so OAuth refresh callbacks reuse still-usable credentials, preserves near-expiry refreshes, and broadens OAuth refresh failure classification for OpenAI app-session termination messages.

## Best Possible Solution

Keep the sign-in-required classifier, but preserve real OAuth refresh for app-server `Unauthorized` callbacks and require live Codex app-server proof before merge.

## Reproduction Assessment

Source-level yes for the introduced regression: upstream Codex emits this callback for `Unauthorized`, and the patch can return the same locally-valid token instead of refreshing. The original incident still lacks live after-fix proof.

## Solution Assessment

No. The fix is too broad because it treats an unauthorized recovery callback as an expiry check; the safer solution is to keep forced refresh for unauthorized callbacks and move any no-op behavior to a separate proactive path.
