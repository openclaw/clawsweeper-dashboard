## Summary

Keep open: the patch is not merge-ready because upstream Codex uses this app-server callback for 401 Unauthorized recovery, and the PR can return the same locally-valid but backend-rejected token instead of refreshing it. The PR also still has mock-only proof for a sensitive auth-provider behavior change.

## What This Changes

The PR changes the Codex app-server auth bridge so refresh callbacks can reuse still-usable OAuth credentials and expands OpenAI app-session refresh-failure classification.

## Best Possible Solution

Keep the sign-in-required classifier if useful, but preserve real OAuth refresh for app-server Unauthorized callbacks and require redacted live Codex app-server proof before merge.

## Reproduction Assessment

Yes, source-level: upstream Codex emits account/chatgptAuthTokens/refresh for Unauthorized recovery, and the PR can return the same locally-usable token instead of forcing refresh. I did not run a live OAuth repro in this read-only review.

## Solution Assessment

No. The proposed fix is too broad because it treats an unauthorized recovery callback as an expiry check; the safer solution is to keep forced refresh for Unauthorized callbacks and split any proactive no-op behavior into a separate path.
