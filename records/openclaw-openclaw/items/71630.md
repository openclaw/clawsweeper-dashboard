## Summary

Keep this open. Current main still has composer action controls, Talk-only composer status text, compaction/fallback notices, and chat lifecycle state, but it does not expose the requested compact composer runtime chip for In progress, Interrupted, and Done; the broader #39127 and #52640 items remain open and do not supersede this narrower Web UI request.

## What This Changes

Add a compact Web UI composer runtime status indicator beside the existing voice and file controls, with session-scoped In progress, Interrupted, and Done states.

## Best Possible Solution

Ship a focused Control UI composer chip driven by explicit local chat lifecycle, abort, reconnect, and error signals now, with stable desktop/mobile layout, accessible labels, no transcript messages, and a clean future handoff to the gateway-owned activity API from #39127.

Not applicable as a bug reproduction because this is a feature request. Static inspection of current main gives a high-confidence verification path: the relevant Web UI composer, run controls, lifecycle handlers, docs, and gateway method list lack the requested chip and backing display state.

## Solution Assessment

Yes, the narrow Control UI chip is the best first step. It solves the user-facing ambiguity without waiting for the larger cross-client sessions.activity design, as long as it avoids text heuristics and keeps the state session-scoped.
