## Summary

Keep this open. Current `main` still has Web UI local run state, action controls, Talk-only composer status text, and reconnect cleanup, but it does not render the requested compact `In progress` / `Interrupted` / `Done` status chip beside the composer actions.

Add a compact, session-scoped runtime status indicator beside the Web UI composer action buttons.

Ship a focused Control UI composer chip driven by explicit chat lifecycle, abort/error, disconnect, and reconnect-resync signals, with stable desktop/mobile layout, accessible labels/tooltips, localized strings, no transcript messages, and a later path to consume #39127.
