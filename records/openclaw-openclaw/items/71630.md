## Summary

Keep open. Current main still lacks the requested compact In progress/Interrupted/Done chip beside the Control UI composer actions; it has local run state, action buttons, Talk-only composer status text, compaction/fallback notices, and silent reconnect cleanup instead. A prior unmerged prototype in https://github.com/openclaw/openclaw/pull/69571 covered this idea, but it was closed as split/superseded and the UI run-state slice is not present on main.

## What This Changes

Add a compact Web UI composer runtime status indicator for In progress, Interrupted, and Done beside the existing composer action controls.

## Best Possible Solution

Ship a focused Control UI composer status chip driven by explicit local chat lifecycle, abort/error, disconnect, and reconnect-resync signals now, with a clean later path to consume the gateway activity API when it lands.

Not applicable as a bug reproduction because this is a feature request. Static inspection gives a high-confidence verification path: current main renders composer controls and local lifecycle state without the requested runtime status chip or gateway activity API.

Yes, the requested narrow composer chip is the best first implementation path for this issue. It solves the immediate Web UI ambiguity without reviving the broad unmerged PR wholesale or blocking on the larger activity API tracked at https://github.com/openclaw/openclaw/issues/39127.
