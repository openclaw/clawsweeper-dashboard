## Summary

Keep this open. Current main still has local chat run state, action controls, Talk-only status text, compaction/fallback toasts, and reconnect cleanup, but it does not render the requested compact composer runtime indicator for In progress, Interrupted, and Done. The broader gateway activity API and channel task-status requests remain open and do not supersede this narrower Control UI composer feature.

## What This Changes

Add a compact, localized runtime status chip beside the Web UI composer voice/file controls for in-progress, interrupted, and done states without adding transcript messages.

## Best Possible Solution

Ship a focused Control UI composer chip driven by explicit local chat lifecycle, abort/error, disconnect, and reconnect-resync signals, with stable desktop/mobile layout, accessible labels/tooltips, localized strings, no transcript messages, and a documented path to consume #39127 later.

## Repair Work Prompt

Implement the Web UI composer runtime status chip requested in openclaw/openclaw#71630. Add a compact localized indicator beside the composer attachment/Talk controls that supports at least In progress, Interrupted, and Done without adding transcript messages or changing conversation history. Drive the first version from existing Control UI lifecycle signals: chatSending/chatRunId/stream/tool/finalizer activity for in-progress, terminal final for done, abort/error/disconnect/reconnect orphan cleanup or uncertain ownership for interrupted/non-running. Keep the layout stable on desktop and mobile, collapse or shorten safely on narrow widths, expose aria-label/title with useful state text and timestamp/elapsed detail where practical, and avoid distracting flicker on fast turns. Do not implement the full gateway sessions.activity API here; leave #39127 as the future authoritative source. Update English i18n source and generated locale outputs via the repo i18n pipeline, add focused tests for render state, terminal/interrupted transitions, reconnect cleanup, and run controls if touched, update Control UI docs and add a user-facing CHANGELOG entry. Do not add status messages to transcripts and do not hard-code brittle assistant text heuristics.
