## Summary

Keep this open. Current main has Web UI composer controls, Talk-only composer status text, compaction/fallback indicators, and chat lifecycle fields, but it still lacks the requested compact composer runtime chip for In progress, Interrupted, and Done. The related gateway activity API work remains open and does not supersede this narrower Control UI request.

## What This Changes

Add a compact, session-scoped runtime status indicator beside the Web UI composer voice/file controls so users can see in-progress, interrupted, and done states without transcript noise.

## Best Possible Solution

Ship a focused Control UI composer chip driven by explicit local chat lifecycle and reconnect/abort/error signals, with stable desktop/mobile layout, accessible labels, and no transcript messages; keep the authoritative cross-client activity model scoped to #39127.
