## Summary

Keep open. Current main still has only implicit Web UI chat run state and coarse session liveness; the composer toolbar does not render the requested compact `In progress` / `Interrupted` / `Done` status chip, and the broader related activity/status issues remain open rather than superseding this narrower UI request.

## What This Changes

Add a compact, accessible runtime status chip beside the Web UI composer voice and file controls showing in-progress, interrupted, and done states for the current session turn.

## Best Possible Solution

Implement a focused composer chip from explicit chat lifecycle and existing session liveness signals, keep it free of transcript text heuristics, and leave richer authoritative activity semantics to https://github.com/openclaw/openclaw/issues/39127.
