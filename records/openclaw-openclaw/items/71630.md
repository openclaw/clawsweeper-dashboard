## Repair Work Prompt

Implement the compact Web UI composer runtime status indicator requested by this issue. Add a small accessible chip beside the existing composer action controls in `ui/src/ui/views/chat.ts`/`ui/src/ui/chat/run-controls.ts` and style it in the chat layout CSS without resizing or wrapping the toolbar on desktop or mobile. Drive the first version from existing session-local chat lifecycle signals: active send/run/tool/finalizer -> In progress, normal terminal completion -> Done with a short fade/expiry, abort/error/disconnect/reconnect uncertainty or cleared orphaned run -> Interrupted/non-running. Do not append status messages to chat history or infer state from assistant text. Add English i18n keys and regenerate locale output via the repo i18n command. Update Control UI docs and CHANGELOG.md because this is a user-facing feature. Keep #39127 as the future authoritative gateway activity source; do not implement the full gateway `sessions.activity` API here.
