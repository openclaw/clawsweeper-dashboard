## Summary

Keep open. Current main still has local chat run state, action controls, Talk-only composer status text, compaction/fallback notices, and reconnect cleanup, but it does not render the requested compact In progress/Interrupted/Done runtime chip beside the Control UI composer actions.

## What This Changes

Add a compact session-scoped runtime status chip beside the Control UI chat composer actions for In progress, Interrupted, and Done states.

Ship a focused Control UI composer chip driven by explicit local chat lifecycle, abort/error, disconnect, and reconnect-resync signals, with stable desktop/mobile layout, accessible labels/tooltips, localized strings, no transcript messages, and a later path to consume #39127.
