## Summary

Keep this open. Current main still has local chat run state, action controls, Talk-only composer status text, compaction/fallback notices, and reconnect cleanup, but it does not render the requested compact runtime status chip for In progress, Interrupted, and Done beside the Web UI composer actions.

## What This Changes

Add a compact, accessible Web UI composer runtime status indicator for active, interrupted, and completed chat turns without adding transcript messages.

## Best Possible Solution

Ship a focused Control UI composer chip driven by explicit local chat lifecycle, abort/error, disconnect, and reconnect-resync signals, with stable desktop/mobile layout, accessible labels/tooltips, localized strings, no transcript messages, and a later path to consume #39127 when that gateway API exists.

Implement a focused Control UI composer runtime status indicator for https://github.com/openclaw/openclaw/issues/71630. Add a compact chip beside the existing file/Talk controls in the Web UI composer toolbar that supports at least In progress, Interrupted, and Done, does not add chat transcript messages, is session-scoped, has accessible aria-label/title text, keeps desktop/mobile layout stable, and avoids flicker on very fast turns. Use current client-side chat lifecycle signals first: active chatSending/chatRunId/tool/finalizer state should become In progress after a short smoothing delay; final success should become Done briefly; abort, send error, disconnect/reconnect uncertainty, stale lost terminal event, or gateway restart with unknown run ownership should become Interrupted or another clearly non-running state.
