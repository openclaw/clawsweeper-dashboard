## Summary

Keep this open. Current main still has composer controls and chat lifecycle state, but it does not render the requested always-visible composer status chip, and the broader gateway activity API remains a separate open dependency rather than an implementation of this UI feature.

## What This Changes

The issue requests a compact, accessible, session-scoped runtime status chip in the Control UI composer showing In progress, Interrupted, and Done beside the existing file and voice controls.

## Best Possible Solution

Ship a focused Control UI composer chip driven by explicit local chat lifecycle and reconnect signals, with stable desktop/mobile layout and accessible labels; keep the authoritative cross-client activity model scoped to #39127.
