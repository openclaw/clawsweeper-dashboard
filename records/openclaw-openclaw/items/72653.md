## Summary

Keep open. Current main still drops the accepted browser timeout budget through navigate and snapshot paths, and the open snapshot-only PR does not cover this broader Ubuntu/CDP report where open, snapshot, and navigate all time out; the FSB link is a useful external workaround/comparison point, not a core fix.

## What This Changes

The issue reports OpenClaw browser open, snapshot, and navigate timing out on Ubuntu despite the configured CDP endpoint working with curl and direct Playwright.

## Best Possible Solution

Thread one normalized browser operation timeout through open, navigate, and snapshot internals while preserving SSRF/navigation guards, and coordinate the snapshot portion with https://github.com/openclaw/openclaw/pull/75702.

Yes for source-level triage: the reporter gives concrete CDP steps, and current main independently shows accepted timeout data cannot reach the navigate and snapshot seams that can hang. I did not establish a live Ubuntu runtime reproduction in this read-only review.

## Solution Assessment

Yes. A targeted browser-plugin timeout propagation repair is the narrow maintainable path; changing gateway architecture, adding a new config knob, or closing in favor of an external workaround would not address the supported core browser tool behavior.
