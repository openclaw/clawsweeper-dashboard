## Repair Work Prompt

Fix the browser-plugin timeout/control propagation bug reported in https://github.com/openclaw/openclaw/issues/72653. Current main drops top-level `timeoutMs`/navigation wait controls on `action="navigate"`, still omits snapshot timeout propagation, and does not pass caller timeout controls from `/tabs/open` into the inner open helpers. Keep the fix inside `extensions/browser/`; do not add a new config option, do not broaden core policy, and preserve existing SSRF checks and default budgets. Normalize positive timeout values at the tool/route boundary, thread them through `browserNavigate`, `callBrowserProxy`, `/navigate`, Chrome MCP navigation, Playwright navigation, snapshot, and open-tab helper seams where applicable, and only support navigation `loadState` values that the existing Playwright/Chrome MCP layers can safely honor. Coordinate with open PR #75702: if it has landed, build on its snapshot timeout wiring; if it is still open, avoid duplicating unrelated snapshot changes beyond what this issue needs. Add focused regression coverage for host and node proxy paths, plus route/helper tests showing explicit timeout values reach the underlying browser operations. Add a user-facing CHANGELOG entry because this fixes browser tool behavior.
