## Summary

Keep open. Current main still has browser timeout-control gaps after a green CDP readiness probe: navigate drops caller timeout/load-state controls before both the node proxy and local client, the HTTP route still invokes Playwright navigation without those controls, and snapshot timeout propagation is still incomplete on main. The open #75702 work is relevant to the snapshot subset but explicitly scopes this broader Linux/CDP all-action report out, so it does not supersede this issue.

## What This Changes

The issue asks for browser open, navigate, and snapshot operations to stop timing out when CDP health checks and direct Playwright CDP access already work.

## Best Possible Solution

Implement a browser-plugin fix that threads normalized operation timeouts and navigation wait-state controls through the agent tool, CLI/client, node proxy, /navigate and /snapshot routes, and Playwright/Chrome MCP helpers, while preserving SSRF pre/post navigation guards and existing-session limits.

Yes. The report gives a high-confidence reproduction path with a local CDP endpoint, curl and direct Playwright success, and OpenClaw operation timeouts; current-main source still matches the timeout-prone boundaries described.

## Solution Assessment

Unclear because there is no complete patch for this issue on main. The narrow maintainable solution is targeted timeout/load-state propagation and clearer timeout classification, not a blanket timeout increase or broad CDP transport rewrite.
