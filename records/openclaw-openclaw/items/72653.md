## Repair Work Prompt

Fix the browser plugin timeout/control propagation bug reported in this issue. On current main, `browser action=navigate` drops `timeoutMs`/`loadState`, `/navigate` ignores them, snapshot drops `timeoutMs` before the client/proxy/route/helper layers, and `/tabs/open`/Chrome MCP open-tab uses fixed internal budgets. Implement a focused fix inside `extensions/browser/`: normalize caller timeouts at the tool boundary, pass them through direct and proxied paths, parse/validate them in routes/plans, thread them into Chrome MCP and Playwright helpers, and preserve existing default budget values unless tests justify a narrow change. Preserve SSRF/navigation guards, existing-session limits, profile behavior, and plugin boundaries. Coordinate with #75702: if it has landed, reuse its snapshot timeout implementation; if it is still open, avoid duplicating conflicting snapshot changes and keep the PR scope clear. Add regression tests for navigate timeout/loadState propagation, snapshot timeout propagation, and open-tab Chrome MCP timeout behavior. Add a user-facing changelog entry if behavior changes ship.
