## Repair Work Prompt

Fix the broader browser timeout propagation bug tracked by https://github.com/openclaw/openclaw/issues/72653. Current main still drops caller timeout controls in `browser action=navigate` and `browser action=snapshot`, and snapshot helpers can hang below a green CDP readiness probe. Keep the work inside the browser plugin boundary. Normalize a positive `timeoutMs` once at tool/client boundaries, pass it through host and node-proxy requests, parse it in `/navigate` and `/snapshot` route planning, and forward it to Chrome MCP `navigate_page`/`take_snapshot`/labeled screenshot calls plus Playwright snapshot helpers (`snapshotAiViaPlaywright`, `snapshotRoleViaPlaywright`, `snapshotAriaViaPlaywright`) and navigation helpers. Preserve the existing SSRF pre/post navigation checks, route auth behavior, existing-session capability limits, and screenshot timeout behavior. Coordinate with related snapshot-only work in https://github.com/openclaw/openclaw/pull/75702 if it is still open; do not create a duplicate snapshot-only fix. Add focused unit coverage for each seam and a user-facing `CHANGELOG.md` fix entry unless the final change is tests-only.
