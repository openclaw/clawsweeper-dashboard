## Summary

Keep open. Current main still has source-visible browser timeout propagation gaps in the browser plugin: snapshot drops accepted timeout input, navigate drops timeout/load-state controls through the tool, client, and route, and the linked snapshot-only PR remains open while the reported Ubuntu/CDP all-action failure is broader. The FSB link is useful as an external workaround/comparison point, but it does not replace an OpenClaw browser-plugin fix.

## What This Changes

The issue reports browser tool operations timing out even though the same Chromium CDP endpoint works through curl and direct Playwright CDP connection.

## Best Possible Solution

Keep this as the canonical all-action browser-timeout tracker, land or replace the snapshot timeout propagation PR, and add the remaining navigate/open timeout propagation in the browser plugin while preserving SSRF and navigation guards.
