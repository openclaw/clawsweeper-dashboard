Keep open. Current main still has source-visible browser timeout propagation gaps for navigate and snapshot, and the related snapshot-only PR remains open rather than merged; the broader Ubuntu/CDP all-action report is not implemented on main.

## What This Changes

The issue reports that browser open, snapshot, and navigate time out on Ubuntu even though the configured CDP endpoint is reachable and direct Playwright CDP use works.

## Best Possible Solution

Keep this as the canonical browser-timeout tracker, land or replace the snapshot propagation PR, and add the remaining navigate/open timeout propagation fix while preserving existing SSRF and navigation guards.

Yes. A targeted browser-plugin timeout propagation fix is the narrow maintainable direction; changing global gateway timeouts or broad browser navigation policy would be larger than the current evidence supports.
