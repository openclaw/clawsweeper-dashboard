## Summary

Keep open. Current main still contains the browser plugin timeout/control gaps that match this report: navigate ignores caller timeout/load-state controls, snapshot still lacks end-to-end timeout propagation, and the open-tab path can still run fixed internal CDP/navigation budgets despite the outer tool call being timed.

## What This Changes

The issue requests a fix for browser open, snapshot, and navigate timing out through OpenClaw even though the same local CDP endpoint works via curl and direct Playwright CDP connection.

## Best Possible Solution

Implement the fix inside the browser plugin by threading normalized timeout/load-state controls through open, navigate, and snapshot seams, while preserving SSRF checks, existing-session limits, and current default budgets unless regression tests justify a narrower adjustment.

## Reproduction Assessment

Yes, source-level. The issue gives a concrete live CDP reproduction, and current main statically reproduces the dropped browser tool controls for navigate and snapshot; I did not run the reporter's live Ubuntu CDP setup.

## Solution Assessment

Yes. A targeted browser-plugin propagation and timeout-boundary repair is the narrow maintainable solution; a generic retry loop or blanket timeout increase would leave the route/helper gaps intact.
