## Summary

Keep open. Current main still has browser-plugin timeout propagation gaps that match the reporter's CDP-ready-but-actions-time-out symptom, and the related open #75702 snapshot PR explicitly leaves this broader Linux/all-action report out of scope.

The issue reports that OpenClaw browser open, snapshot, and navigate operations time out on Ubuntu even though the same CDP endpoint is reachable by curl and direct Playwright CDP connection.

## Best Possible Solution

Implement a browser-plugin repair that threads normalized operation timeout and navigation wait controls through the agent tool, CLI/client, node proxy, /navigate and /snapshot routes, Chrome MCP helpers, and Playwright snapshot/navigation helpers while preserving existing SSRF guards and profile capability limits.
