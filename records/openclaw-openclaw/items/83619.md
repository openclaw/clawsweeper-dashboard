## Summary

## What This Changes

The issue reports a 2026.5.12 regression where exec approvals persistence fails before every exec call on Kubernetes fsGroup-owned PVC mounts.

## Best Possible Solution

Make exec approvals directory chmod hardening best-effort for expected permission/filesystem errors, preserve all existing approval-store safety guards, and add focused regression coverage for the fsGroup/EPERM case.

## Reproduction Assessment

Yes. Source inspection plus reporter logs provide a high-confidence path: on a non-Windows fsGroup mount where chmod returns `EPERM`, `ensureDir` rethrows before `exec-approvals.json` can be written.

## Solution Assessment

Yes. The narrow maintainable fix is to tolerate expected chmod permission/filesystem errors in `src/infra/exec-approvals.ts` while keeping the directory, symlink, hard-link, and file-mode guards intact.
