## What This Changes

Request to add configurable retry/backoff for main LLM requests on transient provider errors such as overloaded/529, 5xx, network failures, and timeouts.

Keep this as the canonical retry/backoff product tracker until maintainers choose an opt-in public contract that fits existing SDK retry caps, auth-profile cooldowns, model fallback, and idempotency rules.
