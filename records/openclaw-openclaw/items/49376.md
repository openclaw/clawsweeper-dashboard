## Summary

Keep this open. Current main still documents and implements SDK-level short retry plus auth-profile/model failover rather than the requested configurable retry-with-backoff surface for main LLM conversation requests, and an open PR with closing syntax is still attached.

## What This Changes

The issue requests configurable retry-with-backoff for transient LLM provider failures such as Anthropic overloaded/529 responses.

## Best Possible Solution

Pick one canonical retry design for main LLM requests, rebase it onto current main, preserve abort/timeout/failover semantics, update schema/docs/tests, and close this issue through the merged PR.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this is a feature/config request. Source inspection does confirm the requested LLM retry config is absent from current main while transient provider errors are handled by classification, SDK retry caps, auth-profile rotation, and model fallback.

## Solution Assessment

Unclear until maintainers choose the retry scope. The direction is plausible, but the maintainable path is to review and land https://github.com/openclaw/openclaw/pull/49800, https://github.com/openclaw/openclaw/pull/61443, or a canonical replacement rather than closing this issue now.
