Keep this open. ## What This Changes

The issue requests configurable retry/backoff for main LLM provider requests on transient provider errors such as overloaded, server error, network, and timeout failures.

Land one canonical opt-in retry contract that reuses the shared retry infrastructure, preserves abort and SDK retry caps, documents interaction with auth.cooldowns/model fallback, and closes this issue through the merged PR.
