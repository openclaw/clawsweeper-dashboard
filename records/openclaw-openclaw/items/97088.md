## Summary

Keep open: current main and v2026.6.10 have safeguards for no-content and historical WhatsApp events, but they still retain a real pairing-reply send path for unknown direct senders and there is no live proof that the reported reconnect/address-book variant is fixed.

## What This Changes

## Best Possible Solution

Keep this as the canonical security-sensitive tracker, capture a live redacted reconnect event, and tighten the WhatsApp-owned inbound gate so reconnect/history/contact-sync side effects cannot send pairing replies while deliberate live first-DM pairing still works.
