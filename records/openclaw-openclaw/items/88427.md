## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Require a usable main credential before dropping local OAuth:** `src/agents/auth-profiles/store.ts:195`
  - body: This still treats a same-provider main OAuth profile as the owner whenever the local expiry is Date-invalid, even if the main credential is merely date-valid but expired/older and the local record has different refresh material. Saving the child store can drop the only recoverable local profile, and refresh ownership can route to the stale main store; only identical credentials or a usable/newer main credential should suppress the local record.
  - confidence: 0.85
- **[P2] Do not clamp browser tab idle retention:** `extensions/browser/src/browser/config.ts:236`
  - body: `idleMinutes` is not the timer delay; it is converted to `idleMs` for `sweepTrackedBrowserTabs`, so clamping it to about 24.8 days silently closes tabs earlier for users who configured longer primary-session retention. Keep the retention budget intact and clamp only the sweep timer interval.
  - confidence: 0.9
- **[P2] Preserve remote batch timeout budgets:** `extensions/memory-core/src/memory/manager-sync-ops.ts:1571`
  - body: `remote.batch.timeoutMinutes` is a logical wait budget compared against elapsed time in the embedding batch loops, not a single `setTimeout` delay. Clamping it here makes valid long-running batch settings time out early; keep this budget intact and clamp individual poll sleeps instead.
  - confidence: 0.88
