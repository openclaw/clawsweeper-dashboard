## Summary

Keep open: current main no longer appears to use the reported 15k reserve as the effective floor, but it still defaults embedded Pi compaction to 20k, below the requested 25k-30k headroom, and the long-session quality failure has not been live-proved against current main.

## What This Changes

Raise the default Pi compaction reserve headroom so long-session summarization has enough room to preserve structured task state.

## Best Possible Solution

Choose a maintained default or adaptive reserve policy, update `src/agents/pi-settings.ts` plus compaction docs/tests, and live-compare current-main 20k behavior against the selected 25k-30k-or-adaptive target on a long-session compaction.

## Reproduction Assessment

No high-confidence current-main live reproduction yet. Source inspection confirms the threshold mechanics and current 20k effective floor, while the reported transcript-tail failure still needs a long-session run against current main.

## Solution Assessment

Unclear. Raising the floor is plausible and narrow, but the best fix should be selected after maintainer threshold/adaptive-policy review and live proof; quality-guard diagnostics are complementary, not a substitute for enough summarization headroom.
