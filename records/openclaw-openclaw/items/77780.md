## Summary

Keep open: current main has improved from the reported 15k Pi default by enforcing a 20k floor, but it still does not implement the requested 25k-30k default headroom or transcript-tail quality warning.

## What This Changes

The issue requests increasing the default compaction reserve-token headroom so long-session summarization runs before the context window is nearly full.

## Best Possible Solution

Validate summary quality at realistic 200k-session sizes, choose a new default floor if the 20k current default is still too late, and update tests, docs, and changelog; treat transcript-tail detection as a follow-up observability improvement if needed.

## Reproduction Assessment

No high-confidence live reproduction was established. Source and tests do show the current effective default floor is 20k, so the threshold concern is source-checkable, but the reported poor summary quality at 182k tokens was not reproduced here.

## Solution Assessment

Unclear. Raising the default to 25k-30k may be the right fix, but current main already enforces 20k and caps small-context models, so maintainers should choose the threshold after validating quality, cost, and compaction-frequency tradeoffs.
