## Summary

Keep open: current main and v2026.6.10 still use a fixed 20,000-token reserve floor over the 16,384 harness default, and no merged change implements the requested higher or context-window-aware default for long-session structured summaries.

## What This Changes

The issue asks OpenClaw to increase or adapt the default compaction reserve headroom so long sessions still produce structured summaries instead of transcript-tail summaries.

## Best Possible Solution

Choose a maintainer-approved context-window-aware default reserve policy, preserve explicit overrides and small-context caps, then validate it with long-session compaction-quality proof and docs/tests.

## Reproduction Assessment

No high-confidence live reproduction on current main. Source confirms current main still applies only a fixed 20,000-token reserve floor over the 16,384 harness default, but this review did not run a 180k-token compaction scenario.

## Solution Assessment

Unclear. Raising the fixed floor to 25,000-30,000 is plausible, but a context-window-aware default backed by long-session proof is the safer maintainable path because small-context caps and operator overrides must remain safe.
