## Summary

Keep open. Current main and v2026.6.10 still use a 20,000-token embedded compaction reserve floor over the 16,384 harness default, while the requested higher or context-window-aware default for long-session structured summaries remains unresolved and needs maintainer direction plus live proof.

## What This Changes

The issue requests raising or making context-aware the default compaction reserve headroom so long-session summaries retain structured task and session state instead of degrading into transcript tails.

## Best Possible Solution

Choose one maintainer-approved context-window-aware reserve policy for default compaction headroom, preserve explicit overrides and the small-context cap, then validate it with long-session compaction-quality evidence and aligned docs/tests.

## Reproduction Assessment

No high-confidence current-main reproduction yet. The issue gives a detailed historical long-session observation and source confirms current main still applies only a 20,000-token floor, but this review did not run a live 180k-token compaction proof.

## Solution Assessment

Unclear. Raising the fixed floor to 25,000-30,000 is plausible, but a context-window-aware default backed by proof is the safer maintainable path because small-context caps and operator overrides must remain safe.
