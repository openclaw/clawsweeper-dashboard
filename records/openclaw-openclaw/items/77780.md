## Summary

Keep open. Current main and v2026.6.8 still use a 20,000-token OpenClaw reserve floor over the 16,384 harness default, so the requested 25,000-30,000 or adaptive headroom policy is not implemented.

## What This Changes

The issue asks OpenClaw to increase or adapt default compaction reserve headroom because late compaction can starve structured summarization in long sessions.

## Best Possible Solution

Choose one maintained reserve policy across embedded and Codex-adjacent compaction paths, preserve the small-context cap, and add focused coverage for high-pressure structured summaries.
