## Summary

Keep open for maintainer review: current main still gates live thinking broadcasts on a channel callback, while this branch makes the narrow gate change, preserves the visibility/silent gates, adds focused regression coverage, and supplies real WebSocket proof. The remaining decision is whether maintainers accept the expanded `/reasoning stream` message-delivery scope for no-callback gateway clients.

## What This Changes

The PR loosens the embedded agent reasoning-stream gate so no-callback gateway/WebSocket-style runs emit `agent.stream === "thinking"`, updates reasoning docs, and adds focused regression tests.

## Best Possible Solution

Land the narrow gate change after maintainer acceptance of the event-delivery expansion, keeping `canShowReasoning`, `silentExpected`, and the optional channel callback behavior intact.

## Reproduction Assessment

Yes from source inspection: on current main, a `reasoningMode: "stream"` subscription without `onReasoningStream` leaves `streamReasoning` false and returns before `emitAgentEvent`. I did not run tests because this review is read-only, but the failing path is direct and covered by the proposed regression tests.

## Solution Assessment

Yes, this is the best fix shape: callers already guard on `state.streamReasoning`, so the state gate must stop depending on the optional callback, and the callback call itself should be optional. Moving only the broadcast would still leave no-callback runs unreachable.
