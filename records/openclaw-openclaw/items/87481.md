## Summary

Keep open for maintainer review: current main and the latest release still gate live thinking events on the optional channel callback, while this PR removes that callback dependency with focused regression coverage and matching docs. I did not find a blocking correctness or security defect in the patch, but the intentional message-delivery expansion should be accepted by a maintainer before merge.

## What This Changes

The PR decouples embedded-agent reasoning-stream broadcasts from the optional channel callback, adds no-callback thinking-stream regression coverage, updates reasoning docs, and carries a few formatting-only hunks outside the central path.

## Best Possible Solution

Land the small runtime/docs/test fix once a maintainer accepts the message-delivery expansion and the current shared-path proof, or ask for one targeted live transport proof if that confidence is required before merge.

## Reproduction Assessment

Yes from source inspection: current main requires onReasoningStream before setting streamReasoning and returns before emitAgentEvent when the callback is absent, while chat.send does not provide that callback. I did not run a local failing test because this review is read-only.

## Solution Assessment

Yes: the PR removes only the callback dependency, keeps the existing canShowReasoning and silentExpected gates, optional-chains the channel callback, and adds focused regression coverage. The remaining question is maintainer comfort with the supplied shared-path proof rather than a different implementation direction.
