## Summary

Keep open: the PR targets a real auth-profile cooldown problem, but the current patch is not merge-ready because it treats every raw HTTP 400 as a malformed-payload case and bypasses cooldowns for failures that current main intentionally classifies as profile-level problems.

## What This Changes

The PR changes auth-profile failure marking to skip usage-state mutation for `format` failures or HTTP 400s, plumbs embedded-runner error text into status extraction, adds unit tests, and adds a changelog entry.

## Best Possible Solution

Keep the narrow fix direction, but skip auth-profile usage mutation only after the failure is classified as `format`, while preserving cooldown or disable behavior for specific `billing`, `auth`, `rate_limit`, and model errors even when they arrive with HTTP 400.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: under this PR, `markAuthProfileFailure({ reason: "billing", statusCode: 400 })` would return early, while current main's classifier and failure policy intentionally preserve `billing` as a profile-level failure reason.

## Solution Assessment

No. The current PR is too broad because raw HTTP 400 is only a transport hint in this codebase; the safer solution is to reuse or extend the existing message-sensitive classification path and test the prompt plus assistant failover call sites.

  - body: This guard returns before recording profile failures whenever a caller forwards raw status 400, even if the already-classified reason is `billing`, `auth`, `rate_limit`, or another specific profile problem. Current main treats 400/422 as ambiguous and inspects the message before falling back to `format`, so gate the skip on a confirmed `format` classification instead of raw status alone.

If provider or proxy text contains another number before the actual status, such as a request-count or upstream note before a final 429, the cooldown policy can receive a false 400 and skip a real rate-limit failure. Use a structured status source or a deterministic helper with tests.
