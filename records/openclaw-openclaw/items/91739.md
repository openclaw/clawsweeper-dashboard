## What This Changes

The issue reports that Gateway image attachments for `google/gemini-3.5-flash` routed through `google-gemini-cli` are rejected before Gemini CLI image prestage runs.

## Best Possible Solution

Land one canonical gateway capability fix for `google-gemini-cli` Gemini image-capable model refs, with regression coverage for stale and missing catalog rows plus a text-only rejection case.

## Reproduction Assessment

Yes, this is source-reproducible: current main routes attachments through `resolveGatewayModelSupportsImages`, which has no `google-gemini-cli` image-capable fallback and causes the parser to reject images when catalog metadata is stale or absent. I did not run a live Google OAuth/Gemini CLI round trip.

## Solution Assessment
