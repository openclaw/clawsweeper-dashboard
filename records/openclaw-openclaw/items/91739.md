## What This Changes

The issue requests that Gateway image attachments for image-capable `google-gemini-cli` Gemini models pass the attachment gate before CLI image prestage.

## Best Possible Solution

Land one canonical targeted fallback for Google Gemini CLI pro/flash chat models with stale or missing catalog metadata, keep text-only Gemini IDs fail-closed, then close this issue and duplicate fix PRs after merge.

## Reproduction Assessment

Yes, at source level. Current main resolves image support before attachment parsing, lacks a `google-gemini-cli` fallback, and the parser throws the reported `UnsupportedAttachmentError` when inline image support is false.

## Solution Assessment

Yes. Repairing the existing Gateway image-capability resolver with a targeted Google Gemini CLI pro/flash fallback plus negative text-only coverage is the narrowest maintainable solution.
