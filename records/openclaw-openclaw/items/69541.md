## Summary

Keep open. Current main still lacks the requested declarative injected-context tag registry, and the live discussion now includes a concrete third-party Crystal/MemPalace adopter that had to hand-roll a `<palace-recall>` strip/ingest contract.

## What This Changes

Request adding a manifest-declared injected-context tag registry and core getter so downstream strip, sanitize, and ingest consumers do not maintain hardcoded plugin tag lists.

## Best Possible Solution

Keep this open for maintainer API/security review, then implement a staged manifest-backed registry and migrate core sanitizers plus downstream consumers to that contract.

## Reproduction Assessment

No end-to-end failure fixture was established, but yes for the contract gap: current main has no registry symbols and still relies on hardcoded producer/consumer strings in the inspected paths.

## Solution Assessment

Unclear pending maintainer review. A manifest-backed registry fits OpenClaw's manifest-first plugin architecture, but strip, sanitize, ingest, dynamic registration, and provenance boundaries need product and security decisions first.
