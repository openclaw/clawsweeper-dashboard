## Summary

Keep open. Current main still lacks a declarative injected-context tag registry, and the recent Crystal/MemPalace adopter comment is concrete evidence that third-party plugins are hitting the same strip/sanitize/ingest contract gap.

## What This Changes

Request a declarative plugin manifest registry for prompt-injected context tags and downstream strip/sanitize/ingest behavior.

## Best Possible Solution

Keep this open for maintainer API/security review, then implement an additive manifest/SDK registry with a normalized getter and staged consumer adoption across display, replay, compaction, memory ingestion, export, and docs.

## Reproduction Assessment

No high-confidence end-to-end bug fixture was established because this is an RFC rather than a narrow failing flow. Yes for source-level reproduction: current main lacks registry symbols while producers inject raw tagged strings and consumers use local hardcoded sanitizers.

## Solution Assessment

Unclear pending maintainer API/security review. A manifest-backed registry is the most maintainable direction visible from current code because manifests already provide static plugin metadata, but the final schema should be coordinated with context provenance and replay hygiene.
