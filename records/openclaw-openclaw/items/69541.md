## Summary

Keep open. Current main still lacks a declarative injected-context tag registry, the issue has a concrete third-party adopter report from Crystal/MemPalace, and the remaining work is a public plugin manifest/SDK plus privacy contract decision rather than conservative cleanup.

## What This Changes

Request an additive plugin manifest/SDK registry for injected prompt context tags so downstream consumers can share strip, sanitize, and ingest rules.

## Best Possible Solution

Keep this open for maintainer API/security review, then implement a manifest-backed registry/getter with staged adoption by display, replay, compaction, memory ingestion, export, and plugin docs.

## Reproduction Assessment

No high-confidence end-to-end bug fixture was established because this is an RFC, but source inspection gives high-confidence reproduction of the contract gap: current main lacks registry symbols while producers and consumers still use hardcoded strings. The Crystal/MemPalace comment adds concrete third-party evidence that the gap is active outside core.

## Solution Assessment

Unclear pending maintainer review. A manifest-backed registry fits OpenClaw's manifest-first plugin architecture, but strip/sanitize/ingest semantics, dynamic registration, and the relationship to context provenance need product and security decisions first.
