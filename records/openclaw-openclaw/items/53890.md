## Summary

Keep open because current main still lacks a Telegram config-level outbound topic default per agent. The existing topic config remains inbound/session routing only, strict validation rejects the proposed extra topic field, and outbound threading only resolves an explicit thread/topic target or the current Telegram tool context.

## What This Changes

Request a Telegram config-level per-agent outbound topic/thread default so proactive sends can target configured forum topics without passing threadId on each send.

## Best Possible Solution

Implement an additive Telegram-owned config and threading resolver path: topic-level outbound-default metadata, schema/docs/generated metadata, and precedence of explicit target/thread first, current Telegram context second, and per-agent configured default last.

## Reproduction Assessment

Yes. Source inspection shows the requested `outbound: true` topic field is not accepted by the strict Telegram topic schema, and proactive message-action threading has no agent-aware lookup for a configured topic default.

## Solution Assessment

Yes. The maintainable direction is a Telegram plugin/config feature plus an additive channel-threading resolver input, not hard-coded Telegram policy in shared core outbound code.
