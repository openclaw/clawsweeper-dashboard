## Summary

Keep open because current main still does not provide a Telegram config-level outbound topic default per agent. The existing topic config is inbound/session routing only, strict schema validation rejects the requested extra field, and outbound threading still resolves only an explicit thread target or the current Telegram tool context.

## What This Changes

Add a Telegram topic config option so an agent can default proactive outbound sends to its configured topic/thread without passing `threadId` on every call.

## Best Possible Solution

Implement this in the bundled Telegram plugin/config surface: add typed topic-level outbound-default metadata, update schema/generated metadata/docs, and resolve outbound threads with precedence for explicit `threadId` or `:topic:` target first, current Telegram context second, and configured per-agent default last.

## Reproduction Assessment

Yes. A Telegram topic config containing `{ "agentId": "main", "outbound": true }` is rejected by the strict schema today, and a proactive send to a bare Telegram group target has no code path that maps the sending agent to a configured topic thread.

## Solution Assessment

Yes for the feature direction, with one API detail to settle in code. The maintainable fix is Telegram-owned config plus an additive channel threading resolver input, not a core hard-coded Telegram lookup.
