## Summary

Keep open because current main still lacks the requested Telegram config-level outbound topic binding, and no duplicate or fixing PR was found. The gap is source-provable in the strict topic schema, config tests, docs, and outbound threading resolver path.

## What This Changes

The issue requests a Telegram configuration feature that binds an agent's proactive outbound messages to a default forum topic/thread without specifying `threadId` per send.

## Best Possible Solution

Add a Telegram-owned outbound default topic binding with explicit thread/target precedence first, current-context auto-threading second, and the per-agent configured default last, with schema, docs, and tests updated together.

## Reproduction Assessment

Yes. Source inspection against current main shows the requested topic-level `outbound` field is not accepted by the strict schema, and the proactive auto-threading path cannot resolve a per-agent topic default.

## Solution Assessment

Yes. The narrow maintainable direction is additive Telegram config plus a small generic channel-threading resolver input, while keeping Telegram-specific policy inside the Telegram plugin.
