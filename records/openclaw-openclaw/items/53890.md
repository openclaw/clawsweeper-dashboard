## Summary

Keep open: current main still lacks a Telegram config-level outbound topic default, while the issue has recent live evidence that a proactive no-thread `message.send` can return success and still miss the active forum topic. This is an actionable Telegram product/config decision, not conservative cleanup-close material.

## What This Changes

Request a per-agent Telegram outbound topic/thread default so proactive sends can target a configured forum topic without every call passing `threadId`.

## Best Possible Solution

Add a Telegram-owned outbound topic default contract with deterministic precedence below explicit `threadId` and explicit topic targets, while preserving existing inbound topic routing and same-chat context inheritance.

## Reproduction Assessment

Yes at source level, but not as a live current-main reproduction in this review. Current code still has only explicit or context-derived thread resolution, and the issue comment adds a recent v2026.5.18 live repro for the proactive no-thread path.

## Solution Assessment

Unclear until maintainers choose the public contract. The requested `outbound: true` shape is plausible, but the durable solution needs explicit precedence for explicit `threadId`, topic-qualified targets, active same-chat context, and configured defaults.
