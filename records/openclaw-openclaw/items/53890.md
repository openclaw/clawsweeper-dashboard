## Summary

Keep open because current main and the latest release still lack a Telegram config-level outbound topic binding; topic config remains inbound/session routing only, and proactive outbound threading has no per-agent default-topic lookup.

## What This Changes

The issue requests a Telegram config option that binds each agent's proactive outbound messages to a default forum topic/thread.

## Best Possible Solution

Add a Telegram-owned topic-level outbound default that preserves explicit target/threadId precedence, then current-context auto-threading, then configured per-agent fallback.

## Reproduction Assessment

Yes. Source inspection shows the requested `topics.<threadId>.outbound` field is rejected by the strict topic schema, and the proactive auto-threading path cannot resolve a per-agent configured topic default.

## Solution Assessment

Yes. The narrow maintainable fix is additive Telegram config plus an agent-aware threading resolver input, with Telegram-specific lookup policy kept inside the Telegram plugin.
