## Summary

Keep open. Current main still has no Telegram config-level outbound topic/thread default per agent: topic config remains inbound-routing oriented, strict schema validation would reject the requested `outbound` flag, and outbound threading only resolves explicit thread targets or current Telegram tool context.

## What This Changes

Request a Telegram topic config option so proactive outbound sends from an agent can default to that agent's configured forum topic without passing `threadId` on every send.

## Best Possible Solution

Add a typed Telegram topic-level outbound default in the bundled Telegram plugin/config surface if maintainers accept the behavior, with explicit target/thread id taking precedence, current conversation auto-threading next, configured per-agent default after that, and deterministic validation for duplicate defaults.
