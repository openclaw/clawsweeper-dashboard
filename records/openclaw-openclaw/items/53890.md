## Summary

Keep open: current main and the latest shipped release still lack a Telegram config-level per-agent outbound topic default, and the recent live note shows the proactive no-thread path remains user-visible on v2026.5.18.

## What This Changes

The issue requests a Telegram config contract that binds each agent to a default outbound forum topic/thread when a proactive send omits `threadId`.

## Best Possible Solution

Add a Telegram-owned outbound default topic contract with deterministic precedence: explicit target/threadId first, active current topic context second, configured per-agent fallback last, with validation for duplicate defaults.

## Reproduction Assessment

Yes, at source level: current main has no outbound topic field in the Telegram topic type/schema, strict validation rejects unknown topic fields, and the auto-thread resolver has no configured per-agent fallback. A recent live comment also reports the same no-thread proactive-send path on v2026.5.18.

## Solution Assessment

Unclear until maintainers choose the public config contract and ambiguity policy. The narrow maintainable direction is Telegram-owned precedence and validation, not a broad core default or silent override of current context.
