## Summary

Keep open: current main and v2026.5.12 still support per-topic inbound agent routing and current-context auto-threading, but not a configured per-agent outbound default topic for proactive Telegram sends.

## What This Changes

Add a Telegram config-level per-agent outbound default topic/thread binding so proactive sends can target a configured forum topic without passing `threadId` on each call.

## Best Possible Solution

Add an additive Telegram-owned outbound default topic setting with clear precedence: explicit thread/target first, current-context auto-threading second, configured per-agent fallback last.

## Reproduction Assessment

Yes. Source inspection against current main shows the requested `outbound` topic field is absent from the type/schema and rejected by strict validation, while proactive auto-threading has no agent-aware config lookup.

## Solution Assessment

Unclear until maintainers confirm the public config shape and duplicate-default behavior. The maintainable direction is Telegram-owned lookup policy plus the smallest agent-aware extension to the shared threading contract.
