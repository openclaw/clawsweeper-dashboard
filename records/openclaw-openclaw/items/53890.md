## Summary

Keep open: current main still lacks the requested Telegram config-level per-agent outbound topic binding, while the adjacent active-context and explicit-topic fixes do not cover the proposed outbound default contract.

## What This Changes

The issue requests a Telegram topic config flag that lets each agent default proactive outbound sends to its configured forum topic when no explicit threadId is provided.

## Best Possible Solution

Add a narrow Telegram-owned outbound topic-default contract only after maintainers approve the config shape and precedence, with regression coverage for proactive message.send from forum-topic sessions and docs for explicit-topic alternatives.

## Reproduction Assessment

Yes for source-level reproduction of the missing config surface: current main has no topic-level outbound/default binding in the strict Telegram schema or types. The message-loss symptom has credible v2026.5.18 live evidence, but this review did not establish a current-main live repro.

## Solution Assessment

Unclear: the requested outbound flag is plausible, but current code already supports explicit topic targets, channel/account defaultTo, and active-context inheritance. Maintainers should choose whether a new config default is better than extending active-context inheritance or warning/failing when no topic can be resolved.
