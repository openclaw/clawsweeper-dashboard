## Summary

Keep open: current main still lacks a Telegram per-agent outbound topic default, and the adjacent fixes only cover explicit or current-context thread paths while the live issue evidence shows proactive no-thread sends can land off-topic.

## What This Changes

The issue requests a Telegram config-level default outbound forum topic/thread binding per agent for proactive sends.

## Best Possible Solution

Add a Telegram-owned outbound default topic contract with deterministic precedence, duplicate-default validation, docs, and regression coverage for proactive message sends.

## Reproduction Assessment

Yes, at source level: current main has strict topic config with no outbound/default field, and auto-threading only reuses tool context when present. The May 20, 2026 comment adds live evidence that a proactive Telegram send without `threadId` produced a successful no-thread message.

## Solution Assessment

Unclear until maintainers choose the public config shape and ambiguity policy. The maintainable direction is a Telegram plugin config contract and precedence rules, not a broad core default or silent override of explicit/current thread context.
