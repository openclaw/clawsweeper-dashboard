## Label Justifications

- `P2`: This is a normal-priority Telegram enhancement with a real multi-topic workflow impact and an available per-call threadId workaround.
- `impact:message-loss`: The request is about preventing proactive Telegram sends from landing outside the intended forum topic when a threadId is omitted.

## Summary

Keep open: current main and the latest release still support inbound per-topic agent routing and explicit/current-context Telegram thread targeting, but not a configured per-agent outbound default topic for proactive sends.

## What This Changes

Add a Telegram config option that binds each agent to a default outbound forum topic/thread so proactive sends can omit per-call threadId.

## Best Possible Solution

Add an additive Telegram-owned outbound default topic setting with deterministic precedence: explicit target/thread first, current-context auto-threading second, configured per-agent fallback last.

## Reproduction Assessment

Yes, at source level: the requested outbound topic field is absent from TelegramTopicConfig and rejected by the strict topic schema, while the Telegram auto-thread resolver has no agent-aware fallback lookup. I did not run a live Telegram proactive-send flow because the missing config contract is visible in current source and latest release source.

## Solution Assessment

Unclear until maintainers choose the public config shape and duplicate-default behavior. The safest direction is an additive Telegram-owned fallback that never overrides explicit targets or current-context threading.
