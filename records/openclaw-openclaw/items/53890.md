## Summary

Keep open: current main and the latest release still lack the requested per-agent Telegram outbound topic default, while adjacent fixes only cover explicit topic targets and active-context thread inheritance. This remains a Telegram config/product decision rather than an implemented or stale item.

## What This Changes

The issue requests a Telegram config-level per-agent default outbound forum topic/thread binding so proactive sends do not need an explicit threadId.

## Best Possible Solution

Keep the existing explicit topic-target and active-context inheritance behavior, then add a narrow Telegram-owned outbound-default contract only if maintainers approve the config surface, precedence rules, docs, and regression coverage.

## Reproduction Assessment

Yes for the missing config surface: current main and v2026.6.10 have no outbound topic binding field in the strict Telegram topic schema or topic config type. The older nested proactive-send symptom has live evidence from v2026.5.18 but was not live-reproduced on current main in this read-only review.

## Solution Assessment

Unclear: the requested outbound flag is plausible, but current code already has explicit topic targets, top-level defaultTo, and active-context inheritance while repository policy sets a high bar for new config/default surfaces.
