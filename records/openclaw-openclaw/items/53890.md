## Summary

Keep open: current main and v2026.5.18 still lack a Telegram config-level per-agent outbound default topic. Existing behavior supports inbound per-topic agent routing, explicit `:topic:` targets, and current-context auto-threading, but the requested configured fallback needs a product/config contract decision.

## What This Changes

The issue requests a Telegram config option that binds each agent to a default outbound topic/thread for proactive messages.

## Best Possible Solution

Add an additive Telegram-owned outbound default topic contract with deterministic precedence: explicit target/thread first, active chat context second, and configured per-agent fallback last.

## Reproduction Assessment

Yes, at source level. Current main has no outbound topic field in the type/schema, strict validation rejects unknown topic fields, and the auto-thread resolver has no agent-aware config lookup path.

## Solution Assessment

Unclear until maintainers choose the public config contract and duplicate-default behavior. The maintainable direction is Telegram-owned policy plus the smallest threading seam needed to see the active agent without overriding explicit or current-context routing.
