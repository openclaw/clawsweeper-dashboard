## Summary

Keep this issue open because current main still lacks a Telegram config-level per-agent outbound topic default: topic config accepts inbound routing metadata, strict schema validation rejects the requested extra field, and outbound threading only uses explicit thread targets or current Telegram context auto-threading.

## What This Changes

Requested a Telegram topic config flag that lets proactive agent outbound sends default to that agent's configured topic without passing `threadId` on every call.

## Best Possible Solution

Implement the feature in the bundled Telegram plugin/config surface with typed topic-level outbound-default metadata, schema/generated metadata/help/docs updates, and tests. Preserve precedence as explicit `threadId` or `:topic:` target first, current Telegram tool context next, then the configured per-agent outbound default, with clear handling for duplicate defaults.

## Reproduction Assessment

Yes. A Telegram config topic containing `{ "agentId": "main", "outbound": true }` is rejected today because `TelegramTopicSchema` is strict and has no `outbound` property; outbound sends to a bare Telegram group target also only resolve explicit/current-context thread IDs.

## Solution Assessment

Unclear for the exact proposed field name, but yes for the feature direction. The maintainable path is a typed Telegram-owned config addition plus a generic outbound-thread resolver input, rather than a core hard-coded Telegram lookup.
