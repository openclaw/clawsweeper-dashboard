- `impact:message-loss`: Successful Telegram sends without topic metadata can land outside the active forum topic and become effectively hard for users to find.

## Summary

Keep open: current main still lacks the requested Telegram config-level per-agent outbound topic binding, while existing explicit topic targets and active-context auto-threading cover only adjacent paths.

## What This Changes

Add a Telegram config-level per-agent outbound topic/thread default so proactive sends can target an agent's forum topic without per-call threadId.

## Best Possible Solution

Define a Telegram-owned outbound topic-default contract only after maintainers choose the config shape and precedence, then land docs and regression coverage for proactive message.send from forum-topic sessions.

## Reproduction Assessment

Yes for source-level reproduction of the missing config surface: current main has no topic-level outbound/default binding in the strict Telegram schema or types. The live message-loss symptom has credible v2026.5.18 evidence, but this review did not rerun a current-main Telegram live repro.

## Solution Assessment

Unclear as the final product shape. The requested outbound flag is plausible, but maintainers should decide whether config defaulting, stronger active-context inheritance, or fail/warn behavior is the durable solution.
