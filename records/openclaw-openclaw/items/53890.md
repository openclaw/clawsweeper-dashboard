## Summary

Keep open: current main still has inbound per-topic Telegram agent routing and current-context auto-threading, but it does not provide a configured per-agent outbound default topic for proactive sends.

## What This Changes

Request to add a Telegram config-level outbound default topic binding per agent for proactive messages.

## Best Possible Solution

Add an additive Telegram-owned outbound default topic contract with deterministic precedence: explicit target or threadId first, current active topic second, and a configured per-agent fallback last.

## Reproduction Assessment

Yes, at source level. Current main has no outbound topic field in the type/schema, strict validation rejects unknown topic fields, and the auto-thread resolver has no agent-aware config lookup path.

## Solution Assessment

Unclear until maintainers choose the public config contract and duplicate-default behavior. The narrow maintainable direction is Telegram-owned policy plus the smallest threading seam needed to see the active agent.
