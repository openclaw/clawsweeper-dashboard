## Summary

Keep open: current main supports inbound per-topic Telegram agent routing and current-context auto-threading, but it still lacks a config-level per-agent outbound topic default for proactive sends.

## What This Changes

Request a Telegram per-agent outbound topic/default-thread binding so proactive sends can target configured forum topics without a per-call threadId.

## Best Possible Solution

Add an additive Telegram-owned outbound default topic contract with validation for ambiguous defaults and deterministic precedence: explicit target/threadId first, active topic context second, configured per-agent fallback last.

## Reproduction Assessment

Yes, at source level. Current main has no outbound topic field in the Telegram topic type/schema, strict validation rejects unknown topic fields, and the auto-thread resolver has no agent-aware configured fallback path.

## Solution Assessment

Unclear until maintainers choose the public config contract and duplicate-default behavior. The narrow maintainable direction is Telegram-owned policy plus the smallest outbound threading seam needed to see the active agent.
