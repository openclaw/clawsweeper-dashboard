## Summary

This PR should stay open for maintainer review: current main does not implement the requested opt-in rewrite, but the branch has source-reproducible punctuation bugs, is behind the current outbound payload-plan architecture, and still lacks real behavior proof.

## What This Changes

The PR adds a `messages.outbound.text.emDash` config mode that rewrites outbound Unicode em dashes to commas across shared outbound normalization, gateway sends, mirrors, post-hook delivery, schema metadata, generated config docs, and tests.

## Best Possible Solution

Decide whether a global outbound punctuation rewrite belongs in core, then implement it in the current payload-plan flow with edge-case tests, generated config docs, and unchanged default behavior.

## Reproduction Assessment

Yes for the PR defects: source inspection and a direct regex probe show leading em dashes become leading commas and existing trailing commas are removed when comma mode runs. The requested behavior itself is a new opt-in feature, not a current-main bug.

## Solution Assessment

No. The shared outbound pipeline is a plausible place for the feature, but this patch needs punctuation edge-case fixes, real behavior proof, and a port to current main's payload-plan architecture before it is the maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve commas that were already in the message:** `src/infra/outbound/payloads.ts:59`
  - body: When comma mode is enabled, this cleanup runs even if the input did not contain an em dash, so a message like `Hello,` is delivered as `Hello`. Only trim commas introduced by rewriting a trailing em dash, or return early when there is no em dash to rewrite.
  - confidence: 0.93
- **[P2] Handle leading em dashes without adding a comma:** `src/infra/outbound/payloads.ts:57`
  - body: Replacing `\s*—\s*` with `, ` turns leading em dashes into visible leading commas, so `— hello` becomes `, hello`. Add a leading-edge case or cleanup so the opt-in transform does not produce malformed outbound text.
  - confidence: 0.91
