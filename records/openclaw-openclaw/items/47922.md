## Summary

This PR should stay open: the feature is not implemented on current main, but the submitted patch has blocking punctuation edge-case bugs and still lacks external real behavior proof.

## What This Changes

This PR adds an opt-in `messages.outbound.text.emDash` config, resolver, outbound text transform wiring for direct/gateway/hook send paths, generated config docs, and focused tests.

## Best Possible Solution

If maintainers want this core style knob, rebase it onto the current payload-plan architecture, fix the comma edge cases, document the hook/config behavior, update the current generated-doc format, and attach real after-fix proof.

## Reproduction Assessment

Yes for the review findings: source inspection of the added replacement chain shows `—hello` becomes `, hello` and `hello,` becomes `hello` whenever comma mode is enabled. No current-main product bug is reproduced because the feature is not on main.

## Solution Assessment

No, not as submitted. The opt-in transform approach is plausible, but the current patch corrupts punctuation edge cases and needs a current-main rebase plus real behavior proof before it is the best maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve messages that already end with a comma:** `src/infra/outbound/payloads.ts:58-60`
  - body: In comma mode every outbound text is passed through `rewriteEmDashesAsCommas`, even if it contains no em dash. The final `.replace(/,\s*$/g, "")` therefore turns `Thanks,` into `Thanks`, silently rewriting user-visible content beyond this feature; only trim a comma that was introduced for a trailing em dash.
  - confidence: 0.94
- **[P2] Handle leading em dashes without a leading comma:** `src/infra/outbound/payloads.ts:58`
  - body: The first replacement turns `—hello` or ` — hello` into `, hello`, and that text is then delivered across all enabled send paths. Add a leading-edge cleanup or branch so leading em-dash text does not gain a visible comma.
  - confidence: 0.9
