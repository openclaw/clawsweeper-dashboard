## Summary

Keep this PR open for maintainer judgment rather than cleanup-close: it proposes a new core outbound style-policy config that is not implemented on current main, but the branch is conflicting, lacks required real behavior proof, and has concrete punctuation bugs in the proposed transform.

## What This Changes

The PR adds an opt-in `messages.outbound.text.emDash` config option and applies em-dash-to-comma rewriting across outbound payload normalization, gateway sends, mirrors, hook-mutated content, config metadata, generated docs, and tests.

## Best Possible Solution

Decide whether this belongs in core; if yes, rebase onto the current payload-plan flow, fix punctuation edge cases, align schema/docs, and require redacted live send-path proof, otherwise steer it to a ClawHub code plugin using `message_sending`.

## Reproduction Assessment

Yes for the PR defects: source inspection of the proposed helper shows leading em dashes become leading commas and unrelated trailing commas can be stripped in comma mode. This is not a current-main bug because current main does not implement the feature.

## Solution Assessment

No, not as-is. The direction may be acceptable after maintainer product approval, but the branch needs a current-main port, punctuation fixes, and real configured send-path proof.

- **[P2] Preserve existing trailing commas in comma mode:** `src/infra/outbound/payloads.ts:59`
  - body: When comma mode is enabled, this helper always runs `.replace(/,\s*$/g, "")`, so a message like `Hello,` with no em dash becomes `Hello` across every transformed delivery path. Trim only punctuation introduced by a trailing em dash, or return early when the input contains no em dash.
  - confidence: 0.93
- **[P2] Handle leading em dashes without emitting a leading comma:** `src/infra/outbound/payloads.ts:57-59`
  - body: Replacing `\s*—\s*` with `, ` turns leading em dashes into visible leading commas, so `— hello` becomes `, hello`. Add a leading-edge case or cleanup so the opt-in transform does not produce malformed outbound text.
  - confidence: 0.91
