## Summary

Keep this PR open. Current main still lacks the proposed outbound em-dash config and transform, so the PR is not obsolete, but the patch is not merge-ready because the helper can mutate punctuation beyond em dashes and mishandle leading em dashes.

## What This Changes

The PR adds a global `messages.outbound.text.emDash` config option and outbound text transform applied across payload normalization, direct and gateway sends, mirroring, and post-`message_sending` hook output, with schema, docs-baseline, and test updates.

## Best Possible Solution

Land a small, opt-in outbound text normalization surface only after the rewrite helper is made content-preserving outside actual em-dash replacements, edge-case tests cover leading/trailing punctuation, hook ordering is documented or adjusted, and the branch is rebased onto the current outbound payload-plan architecture.

## Reproduction Assessment

Yes. The patch bugs reproduce by evaluating `rewriteEmDashesAsCommas("Hello,")` in `comma` mode, which drops the comma, and `rewriteEmDashesAsCommas("— hello")`, which emits a leading comma artifact.

## Solution Assessment

No, not as currently proposed. A global opt-in transform can be a maintainable solution, but the current helper is too broad and the hook-ordering semantics need a deliberate maintainer decision before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve existing trailing commas:** `src/infra/outbound/payloads.ts:59`
  - body: When `messages.outbound.text.emDash` is `comma`, this cleanup runs even if the input had no em dash, so a message like `Hello,` is delivered as `Hello`. Restrict trailing-comma removal to commas introduced by replacing a trailing em dash, or skip the transform entirely when no em dash is present.
  - confidence: 0.92
- **[P2] Handle leading em dashes without adding a comma:** `src/infra/outbound/payloads.ts:57`
  - body: Replacing `\s*—\s*` with `, ` turns leading em dashes into visible leading commas, so `— hello` becomes `, hello`. Add a leading-edge case or cleanup so opt-in rewriting does not produce malformed outbound text.
  - confidence: 0.9
