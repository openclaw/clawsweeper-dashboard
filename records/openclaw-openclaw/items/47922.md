## Summary

Keep open: current main still lacks the requested outbound em-dash config, and this PR remains useful source work, but the patch is dirty against the current payload-plan architecture and has source-reproducible punctuation bugs plus missing changelog coverage.

## What This Changes

The branch adds an opt-in `messages.outbound.text.emDash` config, schema/help/docs metadata, outbound text normalization across direct/gateway/mirror/hook send paths, and focused tests.

## Best Possible Solution

Port the opt-in transform into the current outbound payload-plan send/mirror/hook flow, keep default behavior as `preserve`, avoid changing punctuation unrelated to em dashes, and land with targeted tests, generated config docs, and changelog coverage.

## Reproduction Assessment

Yes for the PR defects: source inspection of `rewriteEmDashesAsCommas()` shows `— hello` becomes `, hello`, and `Hello,` loses its existing comma when comma mode runs. Current main lacks the feature, so these are PR-branch defects rather than current-main bugs.

## Solution Assessment

No. An opt-in outbound transform is a plausible narrow solution, but this patch needs edge-case fixes, changelog coverage, and a port to current main's payload-plan architecture before it is the maintainable implementation.
