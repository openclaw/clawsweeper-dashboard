## Summary

Keep open: current main still lacks the requested outbound em-dash config and transform, but this PR is not merge-ready because the helper has confirmed punctuation bugs, the branch is dirty against the current payload-plan architecture, and contributor real behavior proof is missing.

## What This Changes

The PR adds an opt-in `messages.outbound.text.emDash` config, schema/help/generated docs, outbound text transform logic, and send-path tests for rewriting Unicode em dashes before delivery.

## Best Possible Solution

If maintainers accept this core config surface, implement the transform in the current outbound payload-plan architecture with default `preserve`, edge-case-safe punctuation handling, focused coverage, generated config docs, and maintainer-owned changelog handling at landing.

## Reproduction Assessment

Yes for the PR-branch defects: source inspection of `rewriteEmDashesAsCommas()` shows `— hello` becomes `, hello`, and `Hello,` can lose its existing comma. Current main lacks this feature, so there is no current-main bug reproduction for the requested behavior.

## Solution Assessment

No. An opt-in outbound transform is a plausible direction, but this patch needs edge-case fixes, real behavior proof, and a port to current main's payload-plan architecture before it is the maintainable implementation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve existing trailing commas:** `src/infra/outbound/payloads.ts:59`
  - body: When comma mode is enabled, this cleanup runs for every string, even if no em dash was present. A message like `Hello,` is delivered as `Hello`, so the transform changes punctuation outside the requested em-dash rewrite; only trim commas introduced by replacing a trailing em dash or return early when no em dash exists.
  - confidence: 0.93
- **[P2] Handle leading em dashes without adding a comma:** `src/infra/outbound/payloads.ts:57`
  - body: Replacing `\s*—\s*` with `, ` turns leading em dashes into visible leading commas, so `— hello` becomes `, hello`. Add a leading-edge case or cleanup so opt-in rewriting does not produce malformed outbound text.
  - confidence: 0.91
