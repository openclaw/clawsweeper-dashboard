## Summary

Close as not reproducible: current `main`, latest release source, and the reporter's stated release all emit visible text immediately after a reasoning close tag for the described same-chunk path, without waiting for stream flush.

## What This Changes

The issue reports that Qwen-style mirrored reasoning streams hold the first visible content after a closing reasoning tag until final stream flush and may delay or lose leading newlines.

## Best Possible Solution

Keep the existing partitioner behavior and ask for a new raw current-main stream trace only if a different provider chunk shape still delays the first visible text.

## Reproduction Assessment

No. The reported same-chunk `</think>` transition is contradicted by current and v2026.6.5 source/tests, which emit the visible suffix from the same `push()` call rather than only at `flush()`.
