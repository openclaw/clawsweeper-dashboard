## Summary

Keep open. Current main has adjacent Feishu streaming and subagent completion routing, but it does not implement this PR's outbound real-time subagent streaming seam, and the proposed patch still has blocking delivery, thread-routing, boundary, and throttling regressions plus missing real Feishu proof.

## What This Changes

The PR adds a ChannelPlugin outbound dispatcher hook and Feishu reply-dispatcher/card changes so subagent runs can stream partial output, reasoning, and tool status through Feishu cards.

## Best Possible Solution

Land a refreshed, SDK-safe Feishu subagent streaming implementation that is subagent-only, preserves thread/root routing and block-streaming defaults, and includes focused regression tests plus redacted real Feishu proof.

## Reproduction Assessment

Yes for the PR-introduced regressions: the diff shows the broad delivered-run bridge, dropped Feishu thread id, opt-out block-streaming default, and unthrottled replacement updates. The intended Feishu streaming improvement itself was not live-reproduced and lacks an inspectable artifact.

## Solution Assessment

No. The current PR is not the best solution because it is broader than subagent delivery, loses Feishu topic routing, and crosses the plugin/core boundary; the safer path is a narrow SDK-safe streaming seam that preserves existing delivery contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Restrict the bridge to subagent runs:** `src/commands/agent.ts:738`
  - body: The bridge is created for `opts.deliver === true`, but delivered main-agent runs still go through normal final delivery after the attempt. On Feishu that can create a streaming card and then send the same final reply again through standard delivery for every delivered main-agent command.
  - confidence: 0.92
- **[P1] Preserve the Feishu thread target:** `extensions/feishu/src/channel.ts:499`
  - body: `createOutboundDispatcher` turns `threadId` into a boolean and drops the actual topic/root id. When `replyToId` is absent, the streaming session has no reply target or root id, so topic subagent streams can be posted as top-level cards instead of staying in the originating thread.
  - confidence: 0.9
- **[P2] Keep block streaming opt-in:** `extensions/feishu/src/reply-dispatcher.ts:433-434`
  - body: `account.config?.blockStreaming !== false` enables block streaming whenever the option is unset. Current Feishu behavior keeps internal block chunks disabled unless `blockStreaming` is explicitly true, so default accounts would start receiving fragmented block messages.
  - confidence: 0.9
- **[P2] Use SDK-safe types at the plugin boundary:** `extensions/feishu/src/reply-dispatcher.ts:10`
  - body: This production plugin file imports `../../../src/auto-reply/types.js`, which crosses the extension boundary and ties Feishu to core source layout. Expose or reuse the needed type through the plugin SDK or Feishu runtime API instead.
  - confidence: 0.86
- **[P2] Keep replacement updates under the existing throttle:** `extensions/feishu/src/streaming-card.ts:361-371`
  - body: `replaceContent()` queues a Cardkit content update every time it is called, while the new model-name animation calls the replace path once per character at 30 ms intervals. That bypasses the existing throttle used by normal streaming updates and can burst Feishu API calls.
  - confidence: 0.82
