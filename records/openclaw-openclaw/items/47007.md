## Review Findings

Overall confidence: 0.91

Full review comments:

- **[P1] Restrict the bridge to subagent runs:** `src/commands/agent.ts:738`
  - body: The PR creates the outbound streaming bridge for every delivered run, but normal delivered main-agent runs still go through standard final delivery. On Feishu that can produce a streamed card and then a second final reply for the same assistant turn; the bridge should be limited to subagent delivery or otherwise suppress the final send.
  - confidence: 0.9
- **[P1] Preserve the Feishu thread/root id:** `extensions/feishu/src/channel.ts:498-499`
  - body: The new outbound dispatcher reduces threadId to a boolean and does not pass a root/thread identifier into createFeishuReplyDispatcher. In thread-only subagent contexts without replyToId, FeishuStreamingSession.start cannot target the original topic and can create a top-level card instead.
  - confidence: 0.91
- **[P1] Keep block streaming opt-in:** `extensions/feishu/src/reply-dispatcher.ts:433-434`
  - body: account.config?.blockStreaming !== false turns block streaming on for accounts that never set the option, while current main keeps core block streaming disabled unless Feishu blockStreaming is explicitly true. That can send fragmented internal block chunks to existing chats after upgrade.
  - confidence: 0.88
- **[P2] Throttle replacement card updates:** `extensions/feishu/src/streaming-card.ts:361-371`
  - body: replaceContent bypasses the update throttling used by update, and the PR calls it for model-name/tool/reasoning animations as often as every 30 ms. That can overrun the code's own Feishu CardKit pacing and cause rate or sequence failures during active streaming.
  - confidence: 0.8
- **[P2] Use an SDK-safe channel seam:** `extensions/feishu/src/reply-dispatcher.ts:10`
  - body: Feishu plugin production code imports ModelSelectedContext via a relative path into ../../../src/auto-reply/types.js, which violates the extension boundary that bundled plugins use the public openclaw/plugin-sdk/* surface. Promote the needed type or callback contract through the SDK seam instead.
  - confidence: 0.86
