## Summary

Keep open: current main has absorbed some Feishu streaming polish, but it does not implement the PR's outbound subagent streaming seam, and the proposed branch still has source-visible delivery, thread-routing, block-streaming, API-boundary, proof, and reasoning-visibility blockers.

## What This Changes

The PR adds a ChannelPlugin outbound dispatcher hook and Feishu reply-dispatcher/card changes to stream subagent output, reasoning, model status, and tool activity in real time.

## Best Possible Solution

Land a refreshed, narrow Feishu subagent streaming implementation through an SDK-safe delivery seam that preserves thread/root routing, keeps block streaming opt-in, throttles card updates, preserves reasoning-preview gates, and includes focused tests plus redacted Feishu proof.

No high-confidence real reproduction of the intended Feishu improvement is available because no inspectable after-fix proof was attached.

## Solution Assessment

No. The current patch is broader than the subagent-only problem and weakens current Feishu delivery and visibility contracts; a narrower SDK-safe bridge with preserved routing, gates, tests, and proof is the maintainable path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Limit the bridge to subagent runs:** `src/commands/agent.ts:738`
  - body: The bridge is created for ordinary `--deliver` runs as well as subagent runs, but the command still performs normal final delivery after the agent run. On Feishu this can stream a card and then send the final reply again for the same main-agent command.
  - confidence: 0.92
- **[P1] Preserve Feishu thread ids in the outbound dispatcher:** `extensions/feishu/src/channel.ts:499`
  - body: The dispatcher reduces `threadId` to a boolean and drops the actual topic/root identifier. When there is no `replyToId`, the streaming session has no routing id and can create the card outside the originating thread.
  - confidence: 0.9
- **[P2] Keep Feishu block streaming opt-in:** `extensions/feishu/src/reply-dispatcher.ts:433-434`
  - body: `account.config?.blockStreaming !== false` enables block streaming whenever the option is unset, and the PR forces `disableBlockStreaming: false`. Default Feishu accounts could receive fragmented block messages that current main keeps disabled unless explicitly enabled.
  - confidence: 0.9
- **[P2] Use SDK-safe types at the plugin boundary:** `extensions/feishu/src/reply-dispatcher.ts:10`
  - body: This production Feishu plugin file imports a type from `../../../src/auto-reply/types.js`, crossing the extension boundary. The type should come from the plugin SDK or a Feishu-owned runtime API before adding a plugin-facing seam.
  - confidence: 0.86
- **[P2] Throttle replacement card updates:** `extensions/feishu/src/streaming-card.ts:361-371`
  - body: `replaceContent()` queues a CardKit update every call, while the new model-name animation reaches it once per character at 30 ms intervals. That bypasses the existing streaming throttle and can burst Feishu API calls or sequence updates.
  - confidence: 0.84
