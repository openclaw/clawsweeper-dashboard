## Summary

Keep this PR open as the canonical Feishu subagent-progress thread, but it is not merge-ready: the diff lacks inspectable real Feishu proof and introduces source-visible delivery, routing, compatibility, and availability regressions.

## What This Changes

The PR adds a Feishu outbound streaming dispatcher, expands the channel plugin contract, and rewrites Feishu reply-dispatcher and streaming-card behavior so agent/subagent output can stream into Feishu cards.

## Best Possible Solution

Refresh or replace this with a narrower Feishu-owned subagent progress bridge that preserves topic routing, keeps block streaming opt-in, uses an SDK-reviewed seam, throttles card updates, and includes redacted real Feishu proof.
