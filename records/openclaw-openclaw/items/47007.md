## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Restrict the outbound bridge to subagent progress:** `src/commands/agent.ts:738`
  - body: The bridge is created for every `opts.deliver === true` run, but normal delivered runs still go through final delivery afterward. On Feishu this can post a streamed card and then send the final payload again for the same main-agent command; limit this bridge to the subagent lane or otherwise suppress the existing final delivery path.
  - confidence: 0.91
- **[P1] Preserve the Feishu thread or root identifier:** `extensions/feishu/src/channel.ts:499`
  - body: This turns `threadId` into a boolean and never passes the actual topic/root id into `createFeishuReplyDispatcher`. When a subagent only has a thread/root anchor and no `replyToId`, `FeishuStreamingSession.start()` has no route anchor and will create a top-level card instead of streaming in the originating topic.
  - confidence: 0.9
- **[P1] Keep block streaming opt-in for Feishu accounts:** `extensions/feishu/src/reply-dispatcher.ts:433-434`
  - body: `account.config?.blockStreaming !== false` enables block streaming whenever the setting is absent, which reverses the documented/current default for non-Telegram channels. Existing Feishu chats with unset config can start receiving fragmented block chunks unless this stays `=== true`.
  - confidence: 0.9
- **[P2] Throttle replacement card updates:** `extensions/feishu/src/streaming-card.ts:365-371`
  - body: `replaceContent()` clears pending text and sends every call through `updateCardContent()`, bypassing the 100 ms throttle used by `update()`. The model/tool status paths call it rapidly, so this can produce Feishu CardKit 429s or sequence-order failures during normal streaming.
  - confidence: 0.84
- **[P2] Use a public SDK seam for model-selection context:** `extensions/feishu/src/reply-dispatcher.ts:7`
  - body: The Feishu plugin imports `ModelSelectedContext` from `../../../src/auto-reply/types.js`, crossing the plugin/core source boundary. Expose the needed type through the plugin SDK or keep this logic in core so the channel plugin does not depend on core internals.
  - confidence: 0.78
