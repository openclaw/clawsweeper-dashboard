## Summary

Keep open. Current main does not implement the proposed outbound Feishu streaming bridge, but this PR is not mergeable as-is because the diff introduces duplicate-delivery, thread-routing, block-streaming, boundary, and update-throttling regressions, and it still lacks inspectable real Feishu proof.

## What This Changes

The PR adds a `ChannelPlugin.createOutboundDispatcher` hook and Feishu dispatcher/card changes so subagent output, reasoning, and tool status can stream through Feishu cards.

## Best Possible Solution

Land a refreshed Feishu subagent streaming path through an SDK-safe additive live-delivery seam that preserves topic/root routing, opt-in block streaming, throttled card updates, focused tests, and redacted real Feishu proof.

## Reproduction Assessment

Yes for the PR-introduced regressions: the diff source shows the delivered-run bridge, dropped thread/root id, opt-out block-streaming default, core-internal import, and unthrottled replacement update path. No high-confidence real reproduction is available for the requested Feishu streaming improvement because no inspectable Feishu artifact was attached.

## Solution Assessment

No. The current patch is broader than the subagent-only problem, crosses the plugin/core boundary, and weakens existing Feishu delivery contracts; a narrower SDK-safe live-delivery seam is the maintainable direction.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Limit the bridge to subagent runs:** `src/commands/agent.ts:735`
  - body: The new bridge is built for `opts.deliver === true` as well as the subagent lane, but delivered main-agent runs still go through normal durable delivery. On Feishu this can stream a card and then send the final reply again for ordinary delivered commands.
  - confidence: 0.92
- **[P1] Preserve Feishu thread ids in the outbound dispatcher:** `extensions/feishu/src/channel.ts:497`
  - body: `createOutboundDispatcher` reduces `threadId` to a boolean and never passes the actual topic/root id. When `replyToId` is absent, the streaming session has no routing identifier and can post the subagent stream as a top-level card.
  - confidence: 0.9
- **[P2] Keep Feishu block streaming opt-in:** `extensions/feishu/src/reply-dispatcher.ts:431-433`
  - body: `account.config?.blockStreaming !== false` enables block streaming whenever the option is unset, and the PR forces `disableBlockStreaming: false`. Default Feishu accounts could receive fragmented block messages that current main keeps disabled unless explicitly enabled.
  - confidence: 0.9
- **[P2] Use SDK-safe types at the plugin boundary:** `extensions/feishu/src/reply-dispatcher.ts:10`
  - body: This production Feishu plugin file imports `../../../src/auto-reply/types.js`, which crosses the extension boundary and ties the plugin to core source layout. The type should come from the plugin SDK or a Feishu-owned runtime API.
  - confidence: 0.86
- **[P2] Throttle replacement card updates:** `extensions/feishu/src/streaming-card.ts:361-374`
  - body: `replaceContent()` queues a Cardkit update every call, while the new model-name animation calls it once per character at 30 ms intervals. That bypasses the existing streaming throttle and can burst Feishu API calls or sequence updates.
  - confidence: 0.82
