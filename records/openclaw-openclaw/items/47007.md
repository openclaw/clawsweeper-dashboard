The PR adds a Feishu outbound streaming bridge so subagent runs can stream text, reasoning, and tool status through Feishu cards.

Rebase onto current main and implement the feature through the current channel message lifecycle/SDK direction, preserving Feishu thread targets, avoiding duplicate final delivery, keeping block streaming opt-in, and providing redacted real Feishu proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Restrict the streaming bridge to subagent-only delivery:** `src/commands/agent.ts:738`
  - body: Creating the bridge for `opts.deliver === true` means normal main-agent deliveries get streaming callbacks and then still go through final outbound delivery, producing duplicate Feishu replies/cards. Gate this bridge to the subagent path or make final delivery aware that the bridge already handled the turn.
  - confidence: 0.91
- **[P2] Preserve Feishu thread targets for streaming cards:** `extensions/feishu/src/channel.ts:499`
  - body: The new dispatcher turns `threadId` into only `replyInThread: true` and drops the actual id. When there is no `replyToId`, `FeishuStreamingSession.start()` has no reply/root target and creates the card at top level instead of inside the originating topic/thread.
  - confidence: 0.89
- **[P2] Keep block streaming opt-in by default:** `extensions/feishu/src/reply-dispatcher.ts:433-434`
  - body: `account.config?.blockStreaming !== false` enables block delivery whenever the key is unset, which conflicts with the current Feishu/docs contract that non-Telegram block streaming is off unless explicitly set to true. This can send intermediate block chunks to existing Feishu accounts.
  - confidence: 0.93
- **[P2] Throttle replacement card updates:** `extensions/feishu/src/streaming-card.ts:361-371`
  - body: `replaceContent()` bypasses the same update throttle used by `update()`, while the PR calls it during 30 ms model-name/tool animations. That can burst Feishu Cardkit update calls and risk rate-limit or sequence errors.
  - confidence: 0.82
- **[P2] Keep Feishu code off core-internal type imports:** `extensions/feishu/src/reply-dispatcher.ts:10`
  - body: The Feishu plugin imports `../../../src/auto-reply/types.js`, crossing the plugin boundary from production extension code into core internals. Export the needed callback type through an SDK/local runtime API surface instead.
  - confidence: 0.78
