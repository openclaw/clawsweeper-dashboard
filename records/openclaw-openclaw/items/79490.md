## Summary

Keep open: current main still does not read `SPAWN_ALLOWLIST` or `OPENCLAW_SPAWN_ALLOWLIST`, while the reported failure is actively covered by open implementation PRs and needs a maintainer decision on a security-sensitive env/config contract.

## What This Changes

The issue asks OpenClaw to honor a Docker-friendly spawn allowlist environment variable so `sessions_spawn` can target configured agents without an `openclaw.json` allowlist.

## Best Possible Solution

Review and land one maintainer-approved env-to-config mapping with docs and tests, preferably fallback-only unless env override precedence is explicitly accepted.

## Reproduction Assessment

Yes, by source inspection: current main has no `SPAWN_ALLOWLIST` reader, and the `sessions_spawn` authorization path only consults config-backed `subagents.allowAgents`. I did not run a live Docker reproduction in this read-only review.

## Solution Assessment

Unclear as proposed by the issue alone. The config-only behavior is the current documented contract, while env-only Docker control would need a deliberate, tested config surface and a clear precedence rule.
