## Summary

Keep this open: current main still does not read a spawn allowlist environment variable, and open linked PRs already cover the requested behavior but need maintainer review for the security-sensitive precedence contract.

## What This Changes

Request support for a Docker/env `SPAWN_ALLOWLIST` or `OPENCLAW_SPAWN_ALLOWLIST` value so `sessions_spawn` can use an environment-provided subagent target allowlist.

## Best Possible Solution

Land one canonical env-to-config implementation with documented precedence, regression coverage, and security review, or document that operators must configure `agents.defaults.subagents.allowAgents` instead.

## Reproduction Assessment

Yes, source-reproducible. Current main has no `SPAWN_ALLOWLIST` reader, and the `sessions_spawn` and `agents_list` paths only consult config-backed `allowAgents`, matching the env-only failure shape without needing a Docker run for this review.

## Solution Assessment

No for closing or queueing another repair. The narrow implementation direction is clear, but the security-sensitive env-vs-config precedence should be settled in one of the existing open PRs before the issue closes.
