Keep this open: current `main` still does not read `SPAWN_ALLOWLIST`, and two open PRs explicitly reference this issue with closing syntax, so the issue should remain open until maintainers land or reject the canonical fix path.

## What This Changes

The issue requests Docker/Coolify support for `SPAWN_ALLOWLIST=*` so `sessions_spawn` can target configured subagents instead of failing with `allowed: none`.

## Best Possible Solution

Review the open linked PRs and land one maintainer-approved env allowlist implementation with fallback/precedence semantics, docs, tests, and Docker proof, or document that operators must use `agents.defaults.subagents.allowAgents`.

## Reproduction Assessment

Yes. Source inspection on current `main` shows no `SPAWN_ALLOWLIST` reader, while direct spawn, ACP spawn, and `agents_list` only consult config-backed `subagents.allowAgents`.

## Solution Assessment

Unclear. Supporting an env alias may be useful for Docker deployments, but the best implementation depends on maintainer approval of the new config surface and whether env should be fallback-only or override explicit config.
