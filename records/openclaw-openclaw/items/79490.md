## Summary

Keep open: current main still does not read `SPAWN_ALLOWLIST`, and the issue is already paired with open implementation PRs that use closing references. The remaining decision is whether OpenClaw should add an env-to-config alias for a spawn authorization allowlist.

## What This Changes

The issue requests honoring a Docker/Coolify `SPAWN_ALLOWLIST=*` environment variable so `sessions_spawn` can target a configured agent without `openclaw.json` allowlist configuration.

## Best Possible Solution

Review the focused env-to-config PR and either land it with docs and authorization tests or explicitly keep `agents.defaults.subagents.allowAgents` as the supported Docker configuration path.

## Reproduction Assessment

Yes. By source inspection, setting only `SPAWN_ALLOWLIST=*` leaves `allowAgents` undefined on current main, so explicit cross-agent `sessions_spawn` calls follow the `allowed: none` rejection path.

## Solution Assessment

Unclear. An env alias may be the right Docker ergonomics fix, but because it controls spawn authorization, the narrow maintainable path is maintainer review of the open focused PR rather than treating this as already-resolved cleanup.
