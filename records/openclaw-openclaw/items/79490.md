## Summary

Keep this open: current main still has no `SPAWN_ALLOWLIST` reader, and the active path is an open implementation PR with closing syntax plus a security/precedence review concern that needs maintainer judgment before this issue can close.

## What This Changes

The issue asks OpenClaw to honor a Docker/container `SPAWN_ALLOWLIST` environment variable as the target-agent allowlist for `sessions_spawn`.

## Best Possible Solution

Review and land one canonical env-to-config implementation with documented fallback/precedence semantics and regression coverage, or explicitly keep `openclaw.json` as the only supported allowlist contract.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, by source inspection: current main has no `SPAWN_ALLOWLIST` reader, so a Docker env value alone cannot populate the `sessions_spawn` allowlist path.

## Solution Assessment

Is this the best way to solve the issue? Unclear as an issue-only request; the Docker env alias is plausible, but the maintainable solution needs a maintainer-approved precedence policy because this controls spawn authorization.
