## Summary

Keep open: current main still ignores `SPAWN_ALLOWLIST`, and two open PRs already reference this issue with implementation candidates, so closing before a canonical PR lands would be premature.

## What This Changes

The issue asks for Docker/Coolify `SPAWN_ALLOWLIST=*` to authorize `sessions_spawn` instead of returning `allowed: none`.

## Best Possible Solution

Review and land one canonical env allowlist implementation with docs, tests, real Docker proof, and explicit precedence semantics, or document the supported config-only path if maintainers decline the env surface.

## Reproduction Assessment

Yes, by source inspection: current main has no `SPAWN_ALLOWLIST` reader, and explicit cross-agent spawns consult only config-backed `subagents.allowAgents`, matching the reported `allowed: none` symptom. I did not run a live Docker stack for this read-only review.

## Solution Assessment

Unclear: supporting an env allowlist may be the right Docker/Coolify fix, but the maintainable path needs a maintainer decision on whether the env value is fallback-only or can override explicit config. The existing open PRs are the right place to settle that before closing this issue.
