## Summary

Keep open: current main still does not read `SPAWN_ALLOWLIST`, and the issue is actively paired with open fix PRs that reference it with closing syntax, so it should stay open until maintainers choose and land one canonical env allowlist contract.

## What This Changes

The issue asks for Docker/Coolify `SPAWN_ALLOWLIST=*` to be honored by `sessions_spawn` instead of returning `allowed: none`.

## Best Possible Solution

Review and land one canonical env allowlist implementation that defines precedence, covers direct spawn, ACP spawn, and `agents_list`, updates docs/tests, and then close this issue as fixed.

## Reproduction Assessment

Yes. By source inspection, a deployment that sets only `SPAWN_ALLOWLIST=*` and no config-backed `subagents.allowAgents` still reaches config-only spawn authorization and can produce the reported `allowed: none` denial.

## Solution Assessment

No. Closing this issue now is not the best path because current main is not fixed and open PRs already reference it; the maintainable solution is a maintainer-approved env mapping with clear precedence, docs, and regression coverage.
