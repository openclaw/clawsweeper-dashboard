## Summary

Keep open: current `main` and `v2026.6.8` still do not read `SPAWN_ALLOWLIST` or `OPENCLAW_SPAWN_ALLOWLIST`, while native and ACP `sessions_spawn` policy still depends on configured `subagents.allowAgents`. Two open closing PRs, https://github.com/openclaw/openclaw/pull/79913 and https://github.com/openclaw/openclaw/pull/79538, already target this issue, so maintainers should resolve one canonical fix rather than close or queue another repair.

## What This Changes

The issue asks OpenClaw to honor `SPAWN_ALLOWLIST` for `sessions_spawn` in Docker/env-only deployments where the target agent is configured through `AGENTS`.

## Best Possible Solution

Review and land one canonical fix that treats env-derived spawn allowlists as a fallback only when explicit config is absent, preserves configured-agent wildcard limits, and then closes this issue from the merged PR.

## Reproduction Assessment

Yes, source-reproducible: current main has no `SPAWN_ALLOWLIST` or `OPENCLAW_SPAWN_ALLOWLIST` reader, and both native and ACP spawn gates only receive allowlists from per-agent/default config. I did not run the Docker Compose scenario in this read-only review.

## Solution Assessment

Yes with constraints: an env fallback is a narrow fit for env-only Docker deployments if it fills missing config only and preserves explicit `allowAgents` plus configured-agent wildcard limits. The remaining decision is which linked PR should become the canonical, security-reviewed fix.
