## Summary

Keep open: current main and the latest release do not read `SPAWN_ALLOWLIST`, while `sessions_spawn` authorization still depends only on configured `subagents.allowAgents`. Open fix PRs https://github.com/openclaw/openclaw/pull/79913 and https://github.com/openclaw/openclaw/pull/79538 reference this issue, so the issue should remain open until one canonical fix lands.

## What This Changes

The issue asks OpenClaw to honor `SPAWN_ALLOWLIST` or `OPENCLAW_SPAWN_ALLOWLIST` from Docker environment configuration as a fallback target allowlist for `sessions_spawn`.

## Best Possible Solution

Land one canonical fix that treats env-derived spawn allowlists as a safe fallback without overriding explicit config, preserves configured-agent wildcard constraints, and then close this issue from the merged PR.

## Reproduction Assessment

Yes, source-reproducible: current main has no `SPAWN_ALLOWLIST` reader, and both native and ACP spawn gates only receive allowlists from per-agent/default config. I did not run the Docker Compose scenario in this read-only review.

## Solution Assessment

Yes with constraints: an env fallback is a narrow fit for env-only Docker deployments, but it must fill missing config only and preserve explicit `allowAgents` plus configured-agent wildcard limits.
