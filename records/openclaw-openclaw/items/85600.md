## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Do not replace prompt usage with message_delta usage:** `src/agents/cli-output.ts:175-176`
  - body: `message_delta` usage is a cumulative stream update and can contain only `output_tokens`, while the session-store contract needs prompt/cache tokens. With this branch, a stream with `message_start.usage.cache_read_input_tokens` followed by `message_delta.usage.output_tokens` will overwrite `lastCallUsage` with output-only data, so CLI `totalTokens` becomes stale instead of preserving the last-call cache read. Keep `message_delta` out of `lastCallUsage`, or merge it without discarding the existing input/cache fields.
  - confidence: 0.82
