## Summary

Keep open: this is a focused, recent bug-fix PR for a source-proven live chat projection bug. Current main still drops delta-only assistant events at both projection gates, and the PR’s approach aligns those gates with the existing SDK/event contract without broadening provider or protocol behavior.

## What This Changes

The PR adds a shared `hasLiveAssistantContent` predicate, uses it in Gateway and embedded TUI projection gates to accept `text` or `delta`, and adds focused regression tests.

## Best Possible Solution

Land the focused gate normalization with regression coverage after normal merge checks, while preserving commentary suppression and the existing merge semantics.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main requires `data.text` at both live chat projection gates, while SDK normalization accepts string `data.delta` as an assistant delta. I did not execute a full live Gateway/TUI provider path in this read-only review.

## Solution Assessment

Yes, the PR is the narrowest maintainable fix I found: admit text-or-delta assistant content at the two projection gates and reuse existing normalization and merge helpers. A broader SDK, provider, or protocol change is not needed for this bug.
