## Summary

Keep open: current main and the latest release still select the first stored GitHub Copilot profile in the shared Copilot auth helper, and the open linked fix path at https://github.com/openclaw/openclaw/pull/48682 is still unresolved.

## What This Changes

The issue requests that GitHub Copilot implicit auth selection honor `auth.order` instead of using the first stored `auth.profiles` entry.

## Best Possible Solution

Fix the shared Copilot plugin auth helper, or a narrow SDK auth seam it uses, so catalog and embeddings honor ordered eligible profiles while preserving env-token precedence, discovery disablement, and explicit auth-order exclusions.

## Reproduction Assessment

Yes. A high-confidence source path is to configure multiple stored `github-copilot` token profiles, set `auth.order.github-copilot` to prefer a non-first profile, omit env tokens, and follow catalog discovery into `resolveFirstGithubToken`, which selects `profileIds[0]`.

## Solution Assessment

No landed solution exists yet. The narrow maintainable fix should live in the shared Copilot plugin helper or an existing order-aware SDK auth seam; the open linked PR still needs repair, replacement, or closure with another fix path.
