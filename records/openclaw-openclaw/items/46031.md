## Summary

Keep open. Current main still has order-blind GitHub Copilot token selection in the plugin helper, and an open PR already references this issue with closing syntax, so cleanup closure would be premature.

## What This Changes

The issue requests that GitHub Copilot implicit auth profile selection honor `auth.order` instead of raw `auth.profiles` object order.

## Best Possible Solution

Use the order-aware auth profile seam in the GitHub Copilot plugin token helper, preserve environment-token precedence and explicit `auth.order` exclusions, and cover catalog discovery plus embeddings with focused regression tests.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: configure multiple stored `github-copilot` token profiles, set `auth.order.github-copilot` to prefer a non-first profile, and run implicit Copilot discovery or Copilot embeddings without env tokens; the helper reaches `profileIds[0]`.

## Solution Assessment

Yes. The narrow maintainable fix is to make `resolveFirstGithubToken` use order-aware eligible profile ids and token resolution, instead of adding a broad core fallback that can bypass plugin discovery controls.
