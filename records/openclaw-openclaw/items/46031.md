## Summary

Keep open. Current main still selects the raw first stored GitHub Copilot profile in the shared helper, and an open PR already references this issue with closing syntax, so closing this issue before a corrected merge would be premature.

## What This Changes

The issue requests that GitHub Copilot implicit auth selection honor `auth.order` instead of raw `auth.profiles` object order.

## Best Possible Solution

Make the GitHub Copilot plugin token helper use the existing order-aware auth profile seam while preserving env-token precedence, discovery disablement, explicit `auth.order` exclusions, and focused coverage for catalog discovery plus embeddings.

## Reproduction Assessment

Yes. By source inspection, configure multiple stored `github-copilot` token profiles, set `auth.order.github-copilot` to prefer a non-first profile, omit env tokens, and run implicit Copilot discovery or Copilot embeddings; current main reaches `profileIds[0]`.

## Solution Assessment

Yes for the bug boundary, but the best implementation should live in the plugin-owned `resolveFirstGithubToken` path or a provider-auth SDK helper it uses, not a broad core fallback that can bypass plugin discovery controls.
