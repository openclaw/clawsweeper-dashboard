## Summary

Keep open: current main and v2026.5.22 still select the first stored GitHub Copilot profile through `resolveFirstGithubToken`, while https://github.com/openclaw/openclaw/pull/48682 remains the open linked fix path.

## What This Changes

The issue requests that GitHub Copilot implicit auth profile selection honor `auth.order` instead of JSON insertion order.

## Best Possible Solution

Review, repair, or replace https://github.com/openclaw/openclaw/pull/48682 so the shared Copilot token helper uses ordered eligible profile ids across catalog discovery and embeddings.

Yes. A high-confidence source path is to configure multiple stored `github-copilot` token profiles, set `auth.order.github-copilot` to prefer a non-first profile, omit env tokens, and follow catalog discovery into `resolveFirstGithubToken`, which selects `profileIds[0]`.
