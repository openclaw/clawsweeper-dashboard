## Summary

Keep open: current main and v2026.5.20 still select the first stored GitHub Copilot profile in the shared helper, while the linked fix PR remains open and unmerged.

## What This Changes

The issue asks OpenClaw to respect `auth.order` when choosing the implicit GitHub Copilot auth profile instead of using the first stored profile.

## Best Possible Solution

Update the shared GitHub Copilot auth helper, or a narrow SDK auth seam it uses, so catalog discovery and embeddings use ordered eligible profiles consistently.

## Reproduction Assessment

Yes. A high-confidence source path is to configure multiple stored `github-copilot` token profiles, set `auth.order.github-copilot` to prefer a non-first profile, omit env tokens, and follow catalog discovery into `resolveFirstGithubToken`, which selects `profileIds[0]`.

## Solution Assessment

No landed solution exists yet. The maintainable fix should repair the shared Copilot plugin helper or reuse the existing order-aware SDK route, and the open linked PR should be reviewed, repaired, replaced, or closed by maintainers.
