## Summary

Keep open. Current main still selects the raw first GitHub Copilot auth profile, and open PR #48682 references this issue with closing syntax, so the issue should stay open until that PR is reviewed, merged, reworked, or closed.

## What This Changes

Make GitHub Copilot auth profile selection respect `auth.order` instead of raw `auth.profiles` insertion order.

## Best Possible Solution

Update the plugin-owned Copilot token helper to iterate order-aware eligible profile ids for catalog discovery and embeddings while preserving environment-token precedence and stale-order fallback behavior.

## Reproduction Assessment

Yes. Configure two stored `github-copilot` token profiles, set `auth.order.github-copilot` to prefer the non-first profile, and run implicit Copilot discovery or Copilot embeddings with no env token; current main reaches `profileIds[0]` in `resolveFirstGithubToken`.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing auth-order seam from the GitHub Copilot plugin helper, then close this only after the linked PR or an equivalent replacement lands.
