## Summary

Keep open: current main and v2026.5.20 still route GitHub Copilot token selection through a helper that takes the first stored profile, while the linked fix PR remains open and conflicting.

## What This Changes

The issue reports that GitHub Copilot auth selection ignores `auth.order` and uses the first stored `auth.profiles` entry instead.

## Best Possible Solution

Review, repair, or replace https://github.com/openclaw/openclaw/pull/48682 so the shared Copilot token helper uses ordered eligible profile ids for catalog discovery and embeddings.

## Reproduction Assessment

Yes. A high-confidence source path is to configure multiple stored `github-copilot` token profiles, set `auth.order.github-copilot` to prefer a non-first profile, omit env tokens, and follow catalog discovery into `resolveFirstGithubToken`, which selects `profileIds[0]`.

## Solution Assessment

No landed solution exists yet. The maintainable path is to repair the shared Copilot auth helper or reuse the existing order-aware SDK route, then review and land a focused fix PR.
