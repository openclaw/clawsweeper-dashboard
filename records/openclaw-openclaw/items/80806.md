## Summary

Keep open: current main still lacks sender attribution on agent turn hook contexts, and the overlapping open implementation at https://github.com/openclaw/openclaw/pull/72796 needs API-scope and propagation repair before it can resolve this issue.

## What This Changes

Expose optional senderId, senderName, senderUsername, and senderE164 metadata on agent turn hook contexts such as before_model_resolve, agent_turn_prepare, before_prompt_build, before_agent_finalize, and agent_end.

## Best Possible Solution

Land a rebased additive SDK change, likely by repairing or replacing https://github.com/openclaw/openclaw/pull/72796, that defines the approved sender fields, threads them through all agent turn hook contexts, and updates focused docs/tests.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a bug reproduction, but yes by source inspection: current main carries sender metadata in run params while PluginHookAgentContext and the live hookCtx builders omit it.

## Solution Assessment

Is this the best way to solve the issue? The requested direction is plausible, but the best fix needs an explicit public SDK contract decision and complete propagation through current hook builders, docs, and focused tests.
