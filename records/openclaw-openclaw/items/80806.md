The issue requests optional sender attribution fields on agent turn hook contexts so plugins can identify the user behind a turn.

## Best Possible Solution

Keep the shipped `senderId`/`chatId` support and have maintainers decide whether agent hook contexts should also expose the remaining sender metadata fields, with docs and focused SDK/runtime tests if approved.
