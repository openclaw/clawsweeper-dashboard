## Summary

Keep open: current main still omits sender attribution from public agent hook contexts and the relevant hookCtx builders, while the overlapping implementation PR at https://github.com/openclaw/openclaw/pull/72796 is still open rather than merged.

## What This Changes

The issue requests optional sender attribution fields on public agent turn hook contexts so plugins can identify the inbound sender during prompt/model/agent lifecycle hooks.

## Best Possible Solution

Land a narrow additive SDK change, likely by repairing or replacing the open sender/chat context PR, that defines approved sender fields and threads them through all agent hook context builders with docs, API baseline updates, and focused tests.

## Reproduction Assessment

Yes by source inspection: current main carries sender metadata in embedded run params, but the public agent hook context type and hookCtx builders omit those fields. This is not a failing runtime bug so much as an unimplemented SDK capability.

## Solution Assessment

Unclear until maintainer review defines the public SDK/privacy contract. The requested direction is plausible, but the maintainable path is to repair or replace the existing open PR rather than start a separate speculative implementation.
