## Summary

Keep open: current main still does not expose sender attribution on PluginHookAgentContext or the live agent-turn hook context builders, and the overlapping implementation PR remains open/conflicting rather than making this issue obsolete.

## What This Changes

Expose optional senderId, senderName, senderUsername, and senderE164 on agent turn hook contexts used by before_model_resolve, agent_turn_prepare, before_prompt_build, before_agent_start, before_agent_finalize, and agent_end.

## Best Possible Solution

Land a rebased additive SDK change, likely through https://github.com/openclaw/openclaw/pull/72796 or a narrow replacement, that defines the approved optional sender fields, threads them through every agent-turn hook context, and updates focused docs/tests.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection on current main confirms the requested agent hook contexts omit sender metadata while run params and message/before_agent_run surfaces already carry sender identity.

## Solution Assessment

The requested additive direction is plausible, but the best solution needs an explicit SDK/privacy contract decision and should preserve existing context-window fields and channel context normalization.
