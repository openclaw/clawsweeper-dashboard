## Summary

Keep open: current main still lacks sender attribution on the public agent hook context type and the live agent hook context builders, while the overlapping open implementation at https://github.com/openclaw/openclaw/pull/72796 is not a merged fix and does not fully cover the reported hook propagation path.

## What This Changes

Request to expose optional sender attribution fields on agent turn hook contexts so plugins can identify who triggered a turn.

## Best Possible Solution

Land a narrow additive SDK change, likely by repairing or replacing https://github.com/openclaw/openclaw/pull/72796, that defines the approved sender fields, threads them through every agent turn hook context, and updates focused docs and tests.

## Reproduction Assessment

Yes for source inspection, not as a runtime bug reproduction. Current main shows sender metadata on run params while PluginHookAgentContext and the agent hookCtx builders omit it.

## Solution Assessment

Unclear until maintainers settle the SDK contract. The additive direction is plausible, but the best fix should preserve channel normalization, choose the approved sender fields, and document privacy semantics.
