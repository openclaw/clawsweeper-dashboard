## Summary

Keep open: current main still carries sender metadata in embedded run params but omits it from the public agent hook context and the live hook context builders; the overlapping implementation PR is still open and the public SDK/privacy contract needs maintainer review.

## What This Changes

Request to add optional senderId, senderName, senderUsername, and senderE164 fields to agent turn hook contexts so plugins can attribute turns to the inbound sender.

## Best Possible Solution

Land a narrow additive plugin SDK change, likely by repairing or replacing https://github.com/openclaw/openclaw/pull/72796, that defines approved sender identity fields, threads them through embedded, CLI, and harness hook contexts, and updates docs, API baseline, and focused tests while preserving existing channel and context-window fields.
