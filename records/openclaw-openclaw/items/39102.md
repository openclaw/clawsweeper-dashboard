## Summary

Keep open: current main still has only global A2A allowlists, and this PR is a useful but stale authorization/config feature that needs a rebase, shared visibility-policy work, and docs/schema follow-through before merge.

## What This Changes

The PR adds `agents.list[].tools.agentToAgent.allow` as a per-requester outbound allowlist for A2A session tools, plus denial formatting, config pruning, schema/help labels, and focused tests.

## Best Possible Solution

Refresh the PR onto current main, keep existing `session_status` visibility behavior, implement per-agent A2A decisions in `src/plugin-sdk/session-visibility.ts`, and update generated config schema, public docs, changelog, and focused tests together.

## Reproduction Assessment

Yes, by source comparison. Current main is global-only for A2A policy, while the PR head adds the requested feature but also keeps `session_status` visibility guarded only for sandboxed calls and places the policy outside the current shared SDK path.

## Solution Assessment

No. The requested direction is plausible, but this PR is not the best current implementation until it is rebased, preserves current visibility hardening, centralizes the policy in the shared SDK module, and publishes the config/docs contract.
