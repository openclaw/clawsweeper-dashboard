## Summary

Keep open. Current main still lacks the requested per-agent outbound A2A allowlist, and this PR is useful source work, but it is not merge-ready against the current code shape because the shared A2A visibility policy now lives in `src/plugin-sdk/session-visibility.ts` while the PR implements the new policy in the older wrapper surface, and generated/public config surfaces still need to be aligned.

## What This Changes

The PR adds `agents.list[].tools.agentToAgent.allow` as a per-requester outbound allowlist for A2A session tools, updates denial handling and config help/schema labels, prunes deleted agent IDs from per-agent allowlists, and adds focused tests.

## Best Possible Solution

Refresh the PR onto current main, implement the structured A2A access decision in `src/plugin-sdk/session-visibility.ts`, keep `src/agents/tools/sessions-access.ts` as a thin wrapper, update strict types/schema/help/labels/generated schema/public docs/changelog, and preserve the existing tests for deny-all, global participation, outbound-only target behavior, label lookup, status, and pruning.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: current main rejects `agents.list[].tools.agentToAgent` in the strict per-agent tool schema and the active policy reads only global `tools.agentToAgent`, so the hub-spoke override in the PR body cannot work on main today.

## Solution Assessment

No. The requested direction is plausible, but the current PR patch is not the best implementation against main because the authorization policy must move to the shared plugin-sdk visibility module and the generated/public config surfaces must be regenerated and documented.
