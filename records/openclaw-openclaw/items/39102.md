## Summary

Keep this PR open: the requested per-agent A2A authorization feature is still absent on current main, but the branch is not ready because it puts new authorization decisions on the agent-tools wrapper path while current main centralizes session visibility in the shared plugin SDK module, and the public config contract is incomplete.

## What This Changes

The PR adds `agents.list[].tools.agentToAgent.allow` as an outbound-only per-agent A2A allowlist, updates session tool denial handling and config pruning/schema labels, and adds focused tests.

## Best Possible Solution

Refresh the branch onto current main, implement structured per-agent A2A decisions in `src/plugin-sdk/session-visibility.ts`, keep `src/agents/tools/sessions-access.ts` as a thin wrapper, preserve existing `session_status` visibility guards, and update generated schema, public docs, changelog, and focused tests together.

## Reproduction Assessment

Yes for the review findings by source comparison: current main's A2A policy is global-only and shared through `src/plugin-sdk/session-visibility.ts`, while the PR patch adds per-agent decisions in the older wrapper path and omits generated/docs/changelog surfaces. I did not run tests in this read-only review.

## Solution Assessment

No, not as currently structured. The feature direction is plausible, but the maintainable solution is to put the authorization decision in the shared visibility module and publish the full config contract instead of adding a wrapper-local policy.
