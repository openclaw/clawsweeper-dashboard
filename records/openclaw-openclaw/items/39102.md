## Summary

Keep open. Current main still has only global A2A/session policy and this PR provides a focused implementation with real behavior proof, but the latest head still introduces a blocking public SDK compatibility regression for existing `AgentToAgentPolicy` callers.

## What This Changes

The PR adds requester-owned `agents.list[].tools.agentToAgent.allow` outbound A2A restrictions for session tools, structured deny reasons/errors, schema/docs updates, deleted-agent allowlist pruning, tests, and a changelog entry.

## Best Possible Solution

Preserve Plugin SDK compatibility in this PR, then either land the narrow outbound-only A2A allowlist or fold it into the broader per-agent session/A2A scoping design tracked by the open canonical issue and related PRs.
