## Summary

Keep open: the POSIX chmod-less and read-only credential DB gaps are substantially repaired, but the Windows success path still trusts chmod without ACL privacy proof, so this is not merge-ready.

## What This Changes

The PR replaces raw per-agent SQLite chmod hardening with a private-or-refuse permission gate, applies it to read-only auth-profile reads, and adds chmod-less volume regression coverage.

## Best Possible Solution

Land a single credential-store policy that proves POSIX privacy after chmod and either verifies Windows ACL privacy or explicitly refuses Windows/network-backed agent DB locations that cannot prove it, while keeping the shared non-secret state DB best-effort.

## Reproduction Assessment

Yes. Current main and v2026.6.8 still call raw chmodSync for the agent DB, and the PR body supplies before/after terminal proof for POSIX chmod-less and silent-success cases; the remaining Windows gap is source-reproducible from the PR branch plus the Node fs contract.

## Solution Assessment

No. The POSIX and read-only auth-profile parts are now the right shape, but the Windows success branch still returns without ACL proof; the best fix needs ACL-aware verification or an explicit fail-closed Windows policy accepted by state/security ownership.
