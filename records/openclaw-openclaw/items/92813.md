## Summary

Keep this PR open, but it is not merge-ready: the normal agent SQLite open path is now private-or-refuse, while the short-lived read-only auth-profile SQLite reader can still open and trust the same credential database without that privacy gate.

## What This Changes

The PR replaces raw chmod hardening for the per-agent SQLite database with a private-or-refuse mode check and adds chmod-less volume regression coverage.

## Best Possible Solution

Make the private-or-refuse check a shared agent-DB contract used by every credential-store open, then have state/security ownership accept the fail-closed policy before merge.

No. The private-or-refuse policy is a plausible canonical fix, but this implementation is not yet the best fix because read-only auth-profile resolution still opens the same secret DB without the privacy gate.
