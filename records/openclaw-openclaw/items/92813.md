## Summary

Keep this PR open: current main and the latest release still have the raw agent credential DB chmod/read-only-loader gap, and this recent active PR is the canonical implementation candidate. ## What This Changes

The branch replaces raw per-agent SQLite chmod handling with a private-or-refuse credential-store gate, applies it to read-only auth-profile loads, and adds chmod-less and Windows-relocation regression tests.

## Best Possible Solution

Merge the scoped fail-closed credential-store policy only after a state/security owner accepts the Windows and relocated-storage upgrade behavior; track native ACL verification separately if maintainers want to support more Windows locations.

## Reproduction Assessment

Yes. Current main and v2026.6.10 still call raw chmodSync on the per-agent credential DB path, and the PR body provides terminal before/after proof for chmod failure and silent-success cases; native Windows SMB/ACL behavior is policy-backed rather than directly reproduced.

## Solution Assessment

Yes for the POSIX and read-only credential-store gap: verifying the resulting mode after chmod and sharing the gate with read-only auth loads is the narrowest safe fix. The Windows part is best only if maintainers accept scoped fail-closed behavior instead of requiring native ACL verification in this PR.
