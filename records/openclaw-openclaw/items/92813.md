## Label Justifications

- `P1`: The PR changes agent/auth credential-store behavior that can block real users from loading credentials on affected storage setups.
- `merge-risk: 🚨 compatibility`: The new fail-closed policy can make existing relocated or chmod-less agent credential database locations stop opening after upgrade.
- `merge-risk: 🚨 security-boundary`: The diff defines when OpenClaw may trust credential-bearing SQLite files as private enough to read or write.
- `merge-risk: 🚨 availability`: A refused agent database prevents credential loading and agent runs until the operator moves storage to a provably private location.

## Summary

Keep open: current main and the latest release still use raw agent DB chmod/read-only auth behavior, while this PR appears to address the credential-store privacy gap without a concrete patch defect. The remaining blocker is maintainer acceptance of the fail-closed Windows and relocated-state-dir policy because it can intentionally stop existing agent credential stores from opening.

## What This Changes

The PR replaces raw per-agent SQLite chmod handling with a private-or-refuse credential-store gate, applies it to read-only auth-profile loads, and adds regression tests for chmod-less and Windows-relocated storage cases.

## Best Possible Solution

Merge this scoped fail-closed credential-store policy after the state/security owner accepts the Windows and relocated-storage upgrade behavior, then track native ACL verification separately if maintainers want to widen supported Windows locations.

## Reproduction Assessment

Yes. Current main and v2026.6.9 still call raw chmodSync on the per-agent credential DB path and the PR body supplies terminal before/after proof for chmod failure and silent-success cases; native Windows SMB/ACL behavior is policy-backed rather than directly reproduced.

## Solution Assessment

Yes for the POSIX and read-only credential-store gap: verifying the resulting mode after chmod and sharing the gate with read-only auth loads is the narrowest safe fix. The Windows part is best only if maintainers accept scoped fail-closed behavior instead of requiring native ACL verification in this PR.
