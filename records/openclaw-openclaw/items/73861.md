Keep open. Current main still exposes both reported source-level symptoms: `sessions_send` can return a hard wait timeout after a successful start, and the Gateway call wrapper still formats code 1000 as `gateway closed (1000 normal closure)`; the related #68196/#68065 work only covers the narrower timeout-misclassification overlap.

## What This Changes

Investigate intermittent `sessions_send` timeouts and `gateway closed (1000 normal closure)` failures in a Windows multi-agent deployment while Gateway health checks remain normal.

## Best Possible Solution

Keep this as the broader tracking issue: land or supersede #68196 for the accepted-run timeout overlap, then gather Windows multi-agent Gateway/session diagnostics before adding targeted 1000-normal-close recovery or routing instrumentation.

## Reproduction Assessment

## Solution Assessment
