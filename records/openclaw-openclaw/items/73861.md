## Summary

Keep open. Current main still contains the direct `sessions_send` timeout return path and the Gateway `1000 normal closure` error shape, while the linked #68065 only covers the accepted-run timeout misclassification and does not fully cover the Windows multi-agent normal-close/routing failure described here.

## What This Changes

The issue reports intermittent `sessions_send` timeouts and `gateway closed (1000 normal closure)` failures in a Windows multi-agent deployment even though Gateway health checks pass.

## Best Possible Solution

Keep this open and scope it into two concrete tracks: land the accepted-run wait compensation tracked by #68065, and separately add recoverable 1000-normal-close handling or diagnostics for Windows multi-agent session routing once logs or a repeatable scenario identify the failing boundary.

## Reproduction Assessment

No for the full issue. Static source inspection gives a high-confidence path for `sessions_send` returning `status: "timeout"` and for the `gateway closed (1000 normal closure)` error string, but the Windows 7+ agent routing loop still needs logs, a diagnostic bundle, or a runnable scenario.

## Solution Assessment

Unclear because this is an issue, not a patch. A narrow fix around accepted-run wait compensation plus targeted Gateway close recovery/diagnostics would be safer than a broad routing-table reset without proof.
