## Summary

Keep open. Current main still has the reported `sessions_send` timeout classification path and the Gateway `1000 normal closure` error shape, while the related timeout issue does not fully cover the Windows multi-agent normal-close/routing failure described here.

## What This Changes

The issue reports intermittent `sessions_send` timeouts and `gateway closed (1000 normal closure)` failures in Windows multi-agent deployments despite a healthy Gateway.

## Best Possible Solution

Keep this open and scope it against #68065. The desired end state is a targeted repair for accepted-run wait compensation plus recoverable Gateway close handling or better diagnostics for Windows multi-agent session routing, backed by focused unit tests and a Windows repeated-send regression or diagnostic harness.

## Reproduction Assessment

No for the full issue. Static source inspection gives a high-confidence path for `sessions_send` returning `status: "timeout"`, but the Windows multi-agent `gateway closed (1000 normal closure)` loop still needs logs, a diagnostic bundle, or a runnable scenario.

## Solution Assessment

Unclear because this is an issue, not a patch. A narrow fix around wait compensation, recoverable close classification, and session-routing diagnostics would be more maintainable than a broad routing-table reset without proof.
