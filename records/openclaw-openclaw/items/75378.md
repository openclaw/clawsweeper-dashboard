## Summary

Keep open. Current main still permits the reported heavy subagent fan-out through one global subagent lane, and recent registry/stuck-session fixes only address adjacent delivery or recovery symptoms rather than the underlying event-loop starvation during startup/completion work.

## What This Changes

The issue reports gateway event-loop starvation and 1012 restarts or silent processing windows during heavy `sessions_spawn` startup/completion, especially with parallel or large-context subagents.

## Best Possible Solution

Profile current main under both the original parallel-heavy-subagent burst and the newer single large-context fork, then bound or offload blocking startup/completion/tool work while preserving delivery, cleanup, liveness, and fairness semantics.
