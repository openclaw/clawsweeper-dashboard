## Summary

Keep open. The issue is recent, has concrete event-loop and timeout metrics plus an independent macOS confirmation, and current main still allows the reported three-subagent burst through the shared subagent lane while completion wait, output capture, announce delivery, and cleanup remain Gateway-process work. Current main has a post-accept yield before agent dispatch, but that is only partial evidence and does not prove the reported completion bottleneck is fixed.

## What This Changes

The issue asks OpenClaw to keep the Gateway responsive when spawning three thread-bound session subagents with a heavy Ollama cloud model, avoiding 1012 restarts and announce timeouts.

## Best Possible Solution

Profile the reported three-heavy-subagent burst on current main and implement a measured Gateway responsiveness fix that bounds or offloads heavyweight subagent startup, completion capture, session persistence, and announce delivery without breaking existing delivery and cleanup semantics.
