## Summary

Keep open. Current main has a small accepted-ack yield in the Gateway agent RPC, but it still allows the reported parallel subagent burst, still dispatches child runs onto the shared subagent lane, and still performs startup and completion work inside the Gateway process. The later macOS confirmation makes this look like an active reliability issue, not stale cleanup.

## What This Changes

The issue reports that spawning several heavy-model subagents in parallel can saturate the Gateway event loop, trigger liveness warnings, restart with 1012, and lose all child work.

## Best Possible Solution

Keep the issue open and profile the reported startup and completion paths under load, then keep Gateway RPC/liveness responsive with the narrowest measured change: a bounded subagent startup/model-prep queue, async or offloaded completion/session-store work, targeted yielding/caching, or a liveness/supervisor adjustment if restart policy is the actual failure point.

## Reproduction Assessment

Unclear. The report and macOS confirmation provide a strong reproduction recipe and concrete logs, and code review confirms current main still permits the parallel burst, but I did not live-run the heavy Ollama workload end to end.

## Solution Assessment

Not applicable because there is no patch. Current main’s 10ms accepted-ack yield is useful but not a complete solution for startup saturation or the confirmed completion-processing waits; the final fix should follow profiling.
