## Summary

Keep open. Current main still allows the reported three-way subagent burst, routes child starts through one shared subagent lane, and performs completion wait/announce handling in the Gateway process. The report has concrete metrics plus an independent macOS confirmation, and the related yield-only mitigation PR was closed unmerged, so this remains an active reliability/performance issue rather than cleanup.

## What This Changes

The issue requests that parallel heavy `sessions_spawn` subagent starts and completion handling stop saturating the Gateway event loop and causing 1012 service-restart failures.

## Best Possible Solution

Keep this open and profile packaged v2026.4.29 plus current main under the reported three-subagent workload. The end state should preserve Gateway RPC/readiness responsiveness with a measured bounded startup/model-prep queue, chunked or offloaded completion/session-store work, targeted yielding/caching, or a liveness/supervisor adjustment only if profiling proves restart policy is the real failure point.
