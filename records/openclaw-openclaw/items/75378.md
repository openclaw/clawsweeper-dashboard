## Summary

Keep open. Current main still permits the reported three-way parallel subagent burst, still routes child runs through the shared subagent lane, and still performs completion capture and announce delivery inside the Gateway process. The issue also has a same-release macOS confirmation pointing at completion processing, so this is an active reliability/performance report rather than stale cleanup.

## What This Changes

Fix Gateway event-loop saturation and 1012 restart behavior when multiple heavy `sessions_spawn` subagents start and complete in parallel.

## Best Possible Solution

Keep this open and profile packaged v2026.4.29 plus current main under the reported three-subagent workload. The fix should preserve Gateway RPC/liveness responsiveness with a measured bounded startup/model-prep queue, offloaded or chunked completion/session-store work, targeted yielding/caching, or a supervisor/liveness adjustment only if profiling proves restart policy is the real failure point.
