## Summary

Keep open. Current main still allows the reported three-way subagent burst, still routes child runs through the shared subagent lane, and still performs completion capture and announce cleanup in the Gateway process; the same-release macOS confirmation makes this an active reliability report rather than cleanup material.

## What This Changes

The issue reports Gateway event-loop saturation and 1012 restarts when three heavy `sessions_spawn` subagents are started concurrently in OpenClaw v2026.4.29.

## Best Possible Solution

Keep this open and profile packaged v2026.4.29 plus current main under the reported three-subagent workload. The durable fix should preserve Gateway responsiveness with measured bounded startup/model-prep scheduling, chunked/offloaded completion/session-store work, or a narrowly justified liveness/supervisor adjustment based on profiling evidence.
