## What This Changes

The issue reports Gateway event-loop saturation and a 1012 restart when three heavy `sessions_spawn` subagents run in parallel.

## Best Possible Solution

Profile the current three-heavy-model fan-out, then bound or offload the blocking Gateway subagent startup/completion path so liveness, cleanup, and delivery semantics remain intact under parallel spawns.

No high-confidence live current-main reproduction was established. Source inspection shows current main still permits the reported concurrent heavy-subagent path, and the issue comments provide concrete released-version logs across multiple environments.
