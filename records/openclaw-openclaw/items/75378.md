## Summary

Keep open. Current main still allows the reported three-way subagent burst and still performs child dispatch plus completion capture/announce inside the Gateway process; adjacent session-list responsiveness work does not clearly fix this spawn/completion bottleneck, and the issue has concrete logs plus an independent macOS confirmation.

## What This Changes

Reports that spawning three heavy-model subagents in parallel via `sessions_spawn` saturates the Gateway event loop, causes liveness degradation and a 1012 service restart, and leaves subagent announcements timing out.

## Best Possible Solution

Keep this open and profile packaged v2026.4.29 plus current main under the reported three-subagent workload. The durable fix should preserve Gateway RPC/liveness responsiveness by addressing the measured hotspot: bounded subagent startup/model-prep concurrency, offloaded or chunked completion/session-store work, or a liveness/supervisor adjustment only if profiling proves restart policy is the failure point.
