## Summary

Keep open. The report is recent, concrete, and independently confirmed, and current main still permits the reported three-subagent burst through the shared Gateway subagent lane while completion wait, persistence, output capture, and announce delivery remain Gateway-process work.

## What This Changes

The issue requests preventing parallel heavy `sessions_spawn` subagent bursts from saturating the Gateway event loop and causing 1012 restarts.

## Best Possible Solution

Profile current main under the reported burst and land a targeted design that keeps the Gateway responsive while preserving subagent delivery and cleanup semantics.
