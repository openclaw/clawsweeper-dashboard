## Summary

Keep open: current main still permits missing fetch-timeout context because `operation` and `url` are optional and at least two direct callers omit them; the fix is small but should be careful around the exported plugin SDK helper.

## What This Changes

Ensure `buildTimeoutAbortSignal` callers provide operation and available URL context so fetch timeout warnings are actionable.

## Best Possible Solution

Keep this open and fix the in-repo timeout call sites so timeout logs always carry operation and available URL context, with regression coverage and SDK-compatible future enforcement.

## Reproduction Assessment

Yes, by source inspection: current main has direct timeout helper call sites that omit `operation` and `url`, and the helper only emits those fields when supplied. A focused fake-timer regression test can turn this into an executable repro without live network access.

## Solution Assessment

Partly: auditing callers is the right narrow fix, but making `operation` required on the exported SDK helper needs a backwards-compatible or versioned approach. The optional lane/phase context should be skipped unless an existing runtime fact is already available.
