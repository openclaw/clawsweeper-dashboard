## Summary

Keep open: current main and the latest release still copy the full environment in the shared channel ingress queue opener, and open closing PRs now own the fix path.

## What This Changes

The issue reports that Telegram isolated polling ingress repeatedly copies a large `process.env` when opening the shared channel ingress queue DB with a custom `stateDir`.

## Best Possible Solution

Merge one focused PR that replaces the env spread in the shared channel ingress queue opener with a non-enumerating env overlay or direct state-dir path plus regression coverage, then close this issue from the merged fix.

## Reproduction Assessment

Yes. Source inspection shows that a queue created with `stateDir` repeatedly calls the opener that spreads `process.env`; a large env plus `recoverStaleClaims`, `listClaims`, or `listPending` reproduces the hot-path enumeration without needing a full Telegram run.

## Solution Assessment

Yes. Fixing the shared queue DB opener is the narrowest maintainable solution because Telegram’s drain path reaches the same helper for all affected queue operations.
