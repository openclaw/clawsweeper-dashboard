## Summary

Keep this issue open because current main still does not parse OpenAI-owned `[[tts:speed=...]]` directives, and the open linked PR with closing syntax owns the focused fix path.

## What This Changes

Allow OpenAI TTS to honor `[[tts:speed=...]]` by parsing provider-owned speed directives and forwarding speed to OpenAI speech requests.

## Best Possible Solution

Review and land, or intentionally replace, https://github.com/openclaw/openclaw/pull/74089 with a provider-local OpenAI parser fix that validates `0.25..4.0`, respects `allowVoiceSettings`, and covers normal plus telephony request serialization.

## Reproduction Assessment

Yes. Source inspection on current main shows `[[tts:speed=1.5]]` reaches provider directive parsing, but OpenAI does not claim `speed`, and `parseTtsDirectives` only stores provider overrides after `handled: true`.

## Solution Assessment

Yes for the linked PR direction, not for closing now. The maintainable fix belongs in `extensions/openai/speech-provider.ts` rather than adding a generic core speed override that could conflict with provider-specific ranges.
