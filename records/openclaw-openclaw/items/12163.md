## Summary

Keep open: current main and v2026.5.19 still leave the OpenAI speed directive unclaimed, and the linked closing PR is open rather than merged.

## What This Changes

The issue asks OpenAI TTS to honor `[[tts:speed=...]]` directives by passing speed through to the OpenAI speech request.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/74089 with provider-local OpenAI speed directive parsing, focused regression coverage, non-fatal invalid-directive handling, and docs alignment, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes, source-reproducible. `parseTtsDirectives` calls the selected provider parser for `speed`, but current main's OpenAI `parseDirectiveToken` has no speed case and returns unhandled, so no OpenAI override is stored.

## Solution Assessment

Yes. The maintainable fix is provider-local parsing in `extensions/openai/speech-provider.ts`, gated by `allowVoiceSettings` and bounded to OpenAI's `0.25..4` speed contract, rather than a core-wide generic speed override.
