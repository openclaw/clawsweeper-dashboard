## Summary

Keep this issue open. Current main still misses the reported OpenAI `[[tts:speed=...]]` directive path, and an open linked PR with closing syntax already owns the focused implementation path.

## What This Changes

The issue requests that OpenAI TTS honor per-reply `[[tts:speed=...]]` directives and pass the value through to the OpenAI speech request.

## Best Possible Solution

Review and land, or intentionally replace, https://github.com/openclaw/openclaw/pull/74089 so the fix stays in the OpenAI provider parser with OpenAI range validation, policy gating, and focused directive/request-body coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows `[[tts:speed=1.5]]` reaches provider directive parsing, but OpenAI does not claim the `speed` key, and `parseTtsDirectives` only writes provider overrides after `handled: true`; no runtime command was run because this review is read-only.

## Solution Assessment

Yes for the linked PR direction, not for the original core-level sketch. The maintainable fix belongs in `extensions/openai/speech-provider.ts` with OpenAI-owned parsing, `allowVoiceSettings` gating, `0.25..4.0` validation, and focused regression tests.
