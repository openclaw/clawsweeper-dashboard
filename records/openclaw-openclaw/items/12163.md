## Summary

Keep open: current main still drops OpenAI `[[tts:speed=...]]` directive overrides at the OpenAI provider parser, and open PR #74089 is already the active closing implementation path.

## What This Changes

Add OpenAI TTS directive support so `[[tts:speed=...]]` passes through to OpenAI speech synthesis like provider-config speed already does.

## Best Possible Solution

Review and land or replace PR #74089 with a narrow OpenAI speech-provider parser fix and regression tests.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: parse `[[tts:speed=1.5]]` with OpenAI selected and the router requires `handled: true`, but the OpenAI parser returns unhandled for `speed`.

## Solution Assessment

Yes, provider-owned OpenAI directive handling is the narrow maintainable fix because core routing already delegates provider-specific keys and OpenAI has its own `0.25..4.0` speed range.
