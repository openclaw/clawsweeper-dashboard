## Summary

Keep open: current main still drops OpenAI `[[tts:speed=...]]` directive tokens at the OpenAI provider parser, and the linked implementation PR remains open rather than merged.

## What This Changes

The issue requests OpenAI TTS support for `[[tts:speed=...]]` directives so one reply's speech rate reaches the OpenAI audio request.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/74089 with provider-owned OpenAI speed directive parsing and focused regression coverage, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes, from source inspection. With OpenAI selected or explicitly declared, `[[tts:speed=1.5]]` reaches the OpenAI provider parser, but current main returns unhandled for `speed`, so no OpenAI provider override is stored.

## Solution Assessment

Yes. The narrow maintainable fix is provider-local OpenAI parsing, gated by `allowVoiceSettings` and bounded to OpenAI's `0.25..4.0` speech speed contract, rather than a core-generic parser that could drift across provider ranges.
