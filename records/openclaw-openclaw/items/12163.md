## Summary

Keep open: current main still drops OpenAI `[[tts:speed=...]]` directive overrides in provider directive parsing, and the linked open implementation PR is the active closing path.

## What This Changes

The issue requests that OpenAI TTS honor `speed` from `[[tts:speed=...]]` directives and pass it to the OpenAI speech request.

## Best Possible Solution

Review and land, replace, or close https://github.com/openclaw/openclaw/pull/74089; close this issue only after provider-owned speed directive parsing and regression tests merge.

## Reproduction Assessment

Yes, from source inspection. With OpenAI selected, `[[tts:speed=1.5]]` reaches provider directive parsing, but current main's OpenAI parser falls through for `speed`, so no OpenAI provider override is stored.

## Solution Assessment

Yes. The narrow maintainable fix is OpenAI-owned directive handling in `extensions/openai/speech-provider.ts`, using the existing provider parser and synthesis/request-body plumbing instead of broad core parser changes.
