## Summary

Keep open: current main still drops OpenAI `[[tts:speed=...]]` directive overrides at the OpenAI provider parser, and an open linked PR is already the active fix path.

## What This Changes

The issue asks for OpenAI TTS to honor `[[tts:speed=...]]` by carrying speed through directive parsing into OpenAI speech synthesis.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/74089, then close this issue only after provider-owned speed directive parsing and focused regression tests are on main.

## Reproduction Assessment

Yes, from source inspection. With OpenAI selected or explicitly declared, `[[tts:speed=1.5]]` reaches provider directive parsing, but current main's OpenAI parser falls through for `speed` and never stores a provider override.

## Solution Assessment

Yes. The narrow maintainable fix is OpenAI-owned directive handling in `extensions/openai/speech-provider.ts`, using OpenAI's range and existing request-body plumbing instead of broad core parser changes.
