## Summary

Keep open: current main still drops OpenAI `[[tts:speed=...]]` directive overrides at provider directive parsing, and the linked open PR is the active fix path.

## What This Changes

OpenAI TTS `[[tts:speed=...]]` directives should be accepted and passed through to OpenAI speech synthesis.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/74089; close this issue only after provider-owned speed directive parsing and focused regression tests are on main.

## Reproduction Assessment

Yes, from source inspection. With OpenAI selected or explicitly declared, `[[tts:speed=1.5]]` reaches provider directive parsing, but OpenAI's parser falls through for `speed`, so no provider override is stored.

## Solution Assessment

Yes. The narrow maintainable fix is OpenAI-owned directive handling in `extensions/openai/speech-provider.ts`, using the existing directive-routing and synthesis/request-body plumbing instead of broad core parser changes.
