## Summary

Keep open: current main still does not parse OpenAI `[[tts:speed=...]]` directive tokens into OpenAI provider overrides, and the issue has an open closing implementation path at https://github.com/openclaw/openclaw/pull/74089.

## What This Changes

The issue requests that OpenAI TTS honor `[[tts:speed=...]]` directives by passing speed overrides through to `/audio/speech`.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/74089 with provider-owned OpenAI speed directive parsing and focused regression coverage, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes, from source inspection. With OpenAI selected or explicitly declared, `[[tts:speed=1.5]]` reaches the OpenAI provider parser, but current main returns unhandled for `speed`, so no OpenAI provider override is stored.

## Solution Assessment

Yes. Provider-local OpenAI parsing, gated by `allowVoiceSettings` and bounded to OpenAI's `0.25..4.0` speech speed contract, is the narrow maintainable fix; the linked open PR is the current implementation path.
