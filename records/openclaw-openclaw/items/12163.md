## Summary

Keep open: current main still drops OpenAI `[[tts:speed=...]]` directive tokens because the OpenAI provider parser does not claim `speed`, and the linked implementation PR is still open rather than merged.

## What This Changes

The issue requests support for passing TTS speed directives through to the OpenAI speech provider.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/74089 with provider-owned OpenAI speed directive parsing and focused regression coverage, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes. Source inspection shows `[[tts:speed=1.5]]` reaches provider directive parsing, but current main's OpenAI parser returns unhandled for `speed`, so no OpenAI provider override is stored.

## Solution Assessment

Yes. The narrow maintainable fix is provider-local OpenAI parsing, gated by `allowVoiceSettings` and bounded to OpenAI's `0.25..4.0` speech speed contract, not a core-generic speed parser.
