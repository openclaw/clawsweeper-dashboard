## Summary

Keep open: current main still drops OpenAI `[[tts:speed=...]]` directives at the OpenAI provider parser, and the linked open PR is the active fix path rather than a reason to close the issue early.

## What This Changes

Request support for OpenAI TTS `[[tts:speed=...]]` directives so users can control speech rate per reply.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/74089 with provider-owned speed directive parsing and focused regression tests, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes, from source inspection. With OpenAI selected or explicitly declared, `[[tts:speed=1.5]]` reaches provider directive parsing, but current main's OpenAI parser returns unhandled for `speed`, so no OpenAI provider override is stored.

## Solution Assessment

Yes. The narrow maintainable fix belongs in `extensions/openai/speech-provider.ts`, using OpenAI's provider-specific range and existing synthesis/request-body plumbing rather than broad core parser changes.
