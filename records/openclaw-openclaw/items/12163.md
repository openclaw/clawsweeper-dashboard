## Summary

Keep open: current main still drops OpenAI `[[tts:speed=...]]` directives at the OpenAI provider parser, and https://github.com/openclaw/openclaw/pull/74089 is the active closing implementation path rather than a reason to close this issue before merge.

## What This Changes

Make OpenAI TTS honor `[[tts:speed=...]]` directives by converting them into OpenAI provider overrides and sending speed to the speech request.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/74089 with provider-owned OpenAI speed directive parsing and focused regression tests, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes, from source inspection. With OpenAI selected or explicitly declared, `[[tts:speed=1.5]]` reaches provider directive parsing, but current main's OpenAI parser returns unhandled for `speed`, so no OpenAI provider override is stored.

## Solution Assessment

Yes. The narrow maintainable fix belongs in `extensions/openai/speech-provider.ts`, using OpenAI's provider-specific `0.25..4.0` range and existing synthesis/request-body plumbing rather than broad core parser changes.
