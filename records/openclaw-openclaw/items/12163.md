## Summary

Keep open: current main still drops OpenAI `[[tts:speed=...]]` directives in the OpenAI provider parser, and the issue already has an open closing fix path at https://github.com/openclaw/openclaw/pull/74089.

## What This Changes

The issue asks for OpenAI TTS to honor `[[tts:speed=...]]` directives and pass the speed value to OpenAI speech synthesis.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/74089 with provider-owned OpenAI speed directive parsing plus focused regression tests, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes, from source inspection. With OpenAI selected or explicitly declared, `[[tts:speed=1.5]]` reaches the OpenAI provider parser, but current main returns unhandled for `speed`, so no OpenAI provider override is stored.

## Solution Assessment

Yes. The narrow maintainable fix is in `extensions/openai/speech-provider.ts`, gated by `allowVoiceSettings` and bounded to OpenAI's `0.25..4.0` speech speed contract, with regression tests for directive parsing and request-body propagation. ([platform.openai.com](https://platform.openai.com/docs/api-reference/audio/create?lang=node&utm_source=openai))
