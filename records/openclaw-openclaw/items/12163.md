## Summary

Keep open: current main still does not parse OpenAI `[[tts:speed=...]]` directives into OpenAI provider overrides, and the active closing implementation path is still the open PR at https://github.com/openclaw/openclaw/pull/74089 rather than a merged fix.

## What This Changes

The issue reports that OpenAI TTS ignores model-emitted `[[tts:speed=...]]` directives even though speed already works through other TTS paths.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/74089 with provider-owned OpenAI speed directive parsing and focused regression tests, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes, from source inspection. With OpenAI selected or explicitly declared, `[[tts:speed=1.5]]` reaches the OpenAI provider parser, but current main returns unhandled for `speed`, so no OpenAI provider override is stored.

## Solution Assessment

Yes. The narrow maintainable fix belongs in `extensions/openai/speech-provider.ts`, using OpenAI's provider-specific `0.25..4.0` range and existing synthesis/request-body plumbing rather than broad core parser changes.
