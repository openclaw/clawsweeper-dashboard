## Summary

Keep open: current main still drops OpenAI `[[tts:speed=...]]` directive overrides at the OpenAI provider parser, and open PR #74089 is already the active implementation path.

## What This Changes

Allow OpenAI TTS to honor `[[tts:speed=...]]` directives by turning them into OpenAI provider speed overrides.

## Best Possible Solution

Land or replace PR #74089 with a narrow OpenAI speech-provider parser fix, OpenAI `0.25..4.0` range validation, and focused regression tests while leaving core directive routing unchanged.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: parse `[[tts:speed=1.5]]` with OpenAI selected, then `parseTtsDirectives` requires provider `handled: true` while the OpenAI parser falls through for `speed`.

## Solution Assessment

Yes. Provider-owned OpenAI directive handling is the narrow maintainable fix because core routing already delegates provider-specific keys and OpenAI synthesis already serializes speed when an override reaches it.
