## Summary

Keep open: current main still drops OpenAI `[[tts:speed=...]]` directive overrides at the provider parser, while lower-level OpenAI speed config and request serialization already exist. The issue is already paired with open PR #74089, so the right path is to review and land or replace that PR.

## What This Changes

Add support for `[[tts:speed=...]]` directives to control OpenAI TTS speech rate.

## Best Possible Solution

Land a narrow OpenAI-provider parser fix that accepts speed aliases, gates them on `allowVoiceSettings`, validates OpenAI's `0.25..4.0` range, and adds focused directive regression tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is `parseTtsDirectives` with OpenAI selected and `[[tts:speed=1.5]]`: current routing requires `handled: true`, but the OpenAI parser returns unhandled for `speed`.

## Solution Assessment

Yes. The best fix direction is provider-owned OpenAI directive handling, because OpenAI has its own supported speed range and the core directive router already delegates provider-specific keys.
