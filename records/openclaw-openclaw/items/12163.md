## Summary

Keep open: current main still drops OpenAI `[[tts:speed=...]]` directive tokens at the provider parser, and the open linked implementation PR has not merged yet.

## What This Changes

The issue asks OpenAI TTS to honor per-reply speech-rate directives and forward the parsed speed to OpenAI speech requests.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/74089 with provider-owned OpenAI speed directive parsing and focused regression coverage, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes, from source inspection. With OpenAI selected or explicitly declared, `[[tts:speed=1.5]]` reaches the OpenAI provider parser, but current main returns unhandled for `speed`, so no OpenAI provider override is stored.

## Solution Assessment

Yes. Provider-local OpenAI parsing, gated by `allowVoiceSettings` and bounded to OpenAI's `0.25..4.0` speech speed contract, is the narrow maintainable fix; a core-generic parser would risk provider-range drift.
