## Summary

Keep open: current main still drops OpenAI `[[tts:speed=...]]` directive overrides in the OpenAI provider parser, and the linked open implementation PR is the active closing path.

## What This Changes

Users want OpenAI TTS to honor model-driven `[[tts:speed=...]]` directives by forwarding the speed override to the OpenAI speech request.

## Best Possible Solution

Review and land, replace, or close https://github.com/openclaw/openclaw/pull/74089; close this issue only after the provider-owned parser/test/changelog fix lands.

## Reproduction Assessment

Yes from source inspection. With OpenAI selected, `[[tts:speed=1.5]]` reaches provider directive parsing, but current main's OpenAI parser returns unhandled for `speed`, so no OpenAI provider override is stored.

## Solution Assessment

Yes. The narrow maintainable fix is in `extensions/openai/speech-provider.ts`, matching the provider-owned directive contract and OpenAI's documented `0.25..4.0` speed range without broad core parser changes.
