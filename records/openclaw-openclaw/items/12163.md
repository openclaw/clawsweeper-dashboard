## Summary

Keep open: current main still does not claim OpenAI `[[tts:speed=...]]` directive tokens in the OpenAI speech provider parser, and the active closing pull request is still open rather than merged.

## What This Changes

The issue asks OpenAI TTS to accept speed overrides from `[[tts:speed=...]]` directives and pass them through to the OpenAI speech request.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/74089 with provider-owned OpenAI speed directive parsing, focused regression coverage, and OpenAI's provider-specific speed range, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes. Source inspection shows `[[tts:speed=1.5]]` reaches provider directive parsing, but current main's OpenAI parser returns unhandled for `speed`, so no OpenAI provider override is stored.

## Solution Assessment

Yes. The narrow maintainable fix is provider-local OpenAI parsing, gated by the existing voice-settings policy and bounded to the OpenAI speech speed contract, rather than a core-generic speed parser.
