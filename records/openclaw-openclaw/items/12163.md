## Summary

Keep open: current main still drops OpenAI `[[tts:speed=...]]` directive overrides in the OpenAI provider parser, and https://github.com/openclaw/openclaw/pull/74089 is the active closing implementation path.

## What This Changes

The issue requests that OpenAI TTS honor `[[tts:speed=...]]` directives and pass the speed value through to the OpenAI speech request.

## Best Possible Solution

Review and land, replace, or close https://github.com/openclaw/openclaw/pull/74089; close this issue only after the provider-owned parser/test/changelog fix lands.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with OpenAI selected, `[[tts:speed=1.5]]` reaches provider directive parsing, but the OpenAI parser returns unhandled for `speed`, so no provider override is written.

## Solution Assessment

Yes. The narrow maintainable fix is in `extensions/openai/speech-provider.ts`, matching the provider-owned directive contract and OpenAI's documented `0.25..4.0` range without broad core parser changes.
