## Summary

Keep open: current main still does not parse OpenAI `[[tts:speed=...]]` directive tokens into provider overrides, and the linked implementation PR is open rather than merged.

## What This Changes

OpenAI TTS speed directives should be parsed into OpenAI provider overrides and serialized to OpenAI speech requests.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/74089 with provider-owned OpenAI speed directive parsing and focused regression coverage, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes. Source inspection shows `[[tts:speed=1.5]]` reaches the OpenAI provider parser, but current main returns unhandled for `speed`, so no OpenAI provider override is stored.

## Solution Assessment

Yes. The narrow maintainable fix is provider-local OpenAI parsing, gated by `allowVoiceSettings` and bounded to OpenAI's `0.25..4.0` speech speed contract, not a core-generic speed parser.
