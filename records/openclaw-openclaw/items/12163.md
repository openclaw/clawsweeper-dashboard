Keep this issue open. Current main still lacks OpenAI-owned parsing for `[[tts:speed=...]]`, while an open linked PR with closing syntax already owns the focused fix path, so the issue should remain open until that PR lands or is intentionally replaced.

## What This Changes

The issue requests OpenAI TTS support for the `[[tts:speed=...]]` directive so provider requests include OpenAI's speech speed parameter.

## Best Possible Solution

Review and land, or intentionally replace, https://github.com/openclaw/openclaw/pull/74089 with a provider-owned OpenAI parser fix that validates `0.25..4.0`, respects `allowVoiceSettings`, returns directive warnings for invalid model output, and keeps normal plus telephony request-body coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows `[[tts:speed=1.5]]` reaches provider directive parsing, but OpenAI does not claim `speed`, and `parseTtsDirectives` only stores provider overrides after `handled: true`.

## Solution Assessment

Yes for the linked PR direction, not for closing now. The maintainable fix belongs in `extensions/openai/speech-provider.ts` rather than adding a generic core speed override that could conflict with provider-specific ranges.
