## Summary

Keep open: current main still drops OpenAI `[[tts:speed=...]]` directives because the OpenAI parser does not claim speed tokens, and the active closing path is the still-open PR at https://github.com/openclaw/openclaw/pull/74089.

## What This Changes

The issue requests OpenAI TTS support for per-reply `[[tts:speed=...]]` directives so speed reaches the OpenAI `/audio/speech` request.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/74089 with provider-owned OpenAI speed directive parsing and focused regression tests, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes, from source inspection. With OpenAI selected or explicitly declared, `[[tts:speed=1.5]]` reaches the OpenAI provider parser, but current main returns unhandled for `speed`, so no OpenAI provider override is stored.

## Solution Assessment

Yes. The narrow maintainable solution is provider-local OpenAI directive handling using OpenAI's `0.25..4.0` range and the existing synthesis/request-body plumbing, not a broad core parser change.
