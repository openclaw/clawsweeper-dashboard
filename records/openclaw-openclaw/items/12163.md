## Summary

Keep open: current main and the latest release still leave OpenAI `[[tts:speed=...]]` directive tokens unhandled, and the active closing pull request is still open rather than merged.

## What This Changes

The issue asks for `[[tts:speed=...]]` directives to pass through to OpenAI TTS using OpenAI's supported speed range.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/74089 with provider-local OpenAI speed directive parsing, focused regression coverage, and docs alignment, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes, source-reproducible. `parseTtsDirectives` calls the selected provider parser for `speed`, but current main's OpenAI `parseDirectiveToken` has no speed cases and returns unhandled, so no OpenAI override is stored.

## Solution Assessment

Yes. The narrow maintainable solution is provider-local parsing in `extensions/openai/speech-provider.ts`, gated by `allowVoiceSettings` and bounded to OpenAI's `0.25..4` speed contract, rather than core-generic speed handling.
