## Summary

Keep open: current main still drops OpenAI `[[tts:speed=...]]` directives at the OpenAI provider parser, and the linked closing fix is still the open pull request at https://github.com/openclaw/openclaw/pull/74089.

## What This Changes

Support OpenAI TTS speed directives so `[[tts:speed=...]]` reaches OpenAI provider overrides and the `/audio/speech` request body.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/74089 with provider-owned OpenAI speed directive parsing and focused regression tests, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes, from source inspection. With OpenAI selected or explicitly declared, `[[tts:speed=1.5]]` reaches the OpenAI provider parser, but current main returns unhandled for `speed`, so no OpenAI provider override is stored.

## Solution Assessment

Yes. The narrow maintainable fix belongs in `extensions/openai/speech-provider.ts`, gated by `allowVoiceSettings` and bounded to OpenAI's provider range, with regression tests for directive parsing and request-body propagation.
