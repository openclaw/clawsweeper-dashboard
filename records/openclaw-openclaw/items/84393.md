## Summary

Keep open: current main and the latest shipped tag still intentionally route official OpenAI agent turns through the Codex app-server harness, and the Codex runtime contract says native Codex keeps Codex-owned base/personality instructions. The report raises a real prompt-boundary and runtime-policy question, not an obsolete cleanup item.

## What This Changes

The issue requests gating, observability, preflight checks, quarantine tooling, and regression coverage so operational agents do not silently inherit Codex coding-agent base instructions.

## Best Possible Solution

Make the Codex prompt boundary explicit and auditable, then have maintainers choose between keeping the documented default with stronger diagnostics/warnings or moving operational agents to an opt-in Codex route with a tested migration.
