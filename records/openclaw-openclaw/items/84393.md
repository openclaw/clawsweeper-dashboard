## Summary

Keep open: current main still intentionally routes official OpenAI agent turns through Codex by default, and upstream Codex supplies coding-agent base/model instructions unless OpenClaw overrides them; this remains a live prompt-boundary and runtime-policy decision, not implemented cleanup.

## What This Changes

## Best Possible Solution

Define the supported Codex runtime/base-prompt contract first, then land a staged fix that makes runtime selection visible, preserves explicit OpenClaw opt-out, and gates or overrides Codex base instructions according to that decision.
