## Summary

Keep open: current main still intentionally routes official OpenAI agent turns through the Codex app-server harness, and the Codex prompt-boundary concern is not resolved by the existing diagnostics and doctor cleanup work.

## What This Changes

The issue asks OpenClaw to prevent operational agents from silently receiving Codex coding-agent base instructions unless the Codex runtime is explicitly selected and auditable.

## Best Possible Solution

Make a maintainer-approved Codex prompt/runtime boundary explicit: either reaffirm the documented default with stronger base-prompt diagnostics and quarantine tooling, or move operational agents to an opt-in Codex path with a tested migration coordinated with https://github.com/openclaw/openclaw/issues/78938.
