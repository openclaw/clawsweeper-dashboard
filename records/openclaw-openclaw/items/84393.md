## Summary

Keep open: current main still routes official OpenAI auto-mode agent turns through the Codex harness when available, OpenClaw still does not override Codex base instructions, and upstream Codex still falls back to coding-agent base instructions. Changing that documented default needs maintainer security/product direction, and the open candidate PR is conflicting rather than a landed fix.

## What This Changes

Request to prevent normal operational OpenClaw agents from silently inheriting Codex coding-agent base prompts and to add visible/auditable runtime controls.

## Best Possible Solution

Maintainers should decide the intended OpenAI/Codex default, then land one coherent change that either gates Codex runtime explicitly or overrides/audits the base-prompt surface with doctor, docs, and upgrade tests.

## Reproduction Assessment

Yes from source inspection: OpenAI auto-mode resolves to Codex on current main, the Codex thread start payload lacks a base-instructions override, and upstream Codex defaults to coding-agent base instructions. I did not run a live gateway/Codex session in this read-only review.

## Solution Assessment

Unclear pending maintainer decision: a narrow diagnostics-only fix is not enough, and an unconditional default flip could break documented Codex deployments. The best fix needs policy, observability, migration/doctor, and regression coverage together.
