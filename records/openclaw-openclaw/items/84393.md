## Summary

Keep open: current main and v2026.5.19 intentionally route official OpenAI agent turns through the Codex app-server harness, and the reporter's native-log evidence raises an unresolved prompt-boundary/security policy question rather than an obsolete or already-fixed bug.

## What This Changes

The issue asks OpenClaw to prevent, gate, or clearly disclose Codex coding-agent base prompts in operational direct-chat agents unless Codex is explicitly requested.

## Best Possible Solution

Make Codex runtime selection and native base-prompt identity explicit and auditable, then either reaffirm the documented default with stronger diagnostics/warnings or move operational agents to an upgrade-safe opt-in Codex route.
