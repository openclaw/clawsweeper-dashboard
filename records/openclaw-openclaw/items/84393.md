## Summary

Keep open: current main intentionally routes official OpenAI agent turns through the Codex harness and exposes that runtime in docs/status, but the reported hidden native Codex base-prompt boundary and requested opt-in, warning, diagnostic, and quarantine behavior are not fully covered by the existing contract.

## What This Changes

The issue asks OpenClaw to prevent operational agents from using the Codex app-server runtime without explicit, visible opt-in and runtime/base-prompt diagnostics.

## Best Possible Solution

Define the intended Codex runtime boundary explicitly: either document and surface the native Codex base prompt as expected for Codex-backed sessions, or change operational-agent defaults to PI unless Codex is deliberately selected, with doctor/status diagnostics and regression coverage aligned to that decision.

## Reproduction Assessment

Partly. Source inspection gives a high-confidence path for official OpenAI agent turns resolving to the Codex harness, but the exact native Codex base_instructions content was not reproduced with a live app-server run in this review.

## Solution Assessment

Unclear. The requested direction is plausible, but the safest fix depends on a maintainer decision about whether Codex should remain the default OpenAI agent runtime or become an explicit coding/session mode with new diagnostics and migration handling.
