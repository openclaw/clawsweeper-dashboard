## Summary

Keep open: current main still routes official OpenAI agent turns to Codex by default, OpenClaw’s Codex thread-start payload sends developer instructions but no base-instructions override, and upstream Codex falls back to its coding-agent base/model instructions. This remains a live prompt-boundary and runtime-policy issue, with an open but conflicting related PR rather than a merged fix.

## What This Changes

The issue requests that operational OpenClaw agents stop silently inheriting Codex coding-agent base instructions and gain explicit runtime opt-in, diagnostics, warnings, quarantine tooling, and regression coverage.

## Best Possible Solution

Maintainers should decide the supported Codex runtime/base-prompt contract, then land a staged fix that makes harness selection visible, preserves explicit OpenClaw opt-out, gates or overrides Codex base instructions according to that decision, and covers direct-chat regression cases.

## Reproduction Assessment

Yes from source, not live: official OpenAI routes resolve to Codex by default, OpenClaw sends Codex developerInstructions without a base-instructions override, and upstream Codex falls back to coding-agent model/base instructions. A live rollout replay would still be useful before implementation.
