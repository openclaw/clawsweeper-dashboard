## Summary

Keep open. Current main still lacks the requested workflow seams, and the latest PR head remains an unmerged, dirty implementation branch with concrete merge blockers: attachment delivery can still fall back to global runtime config, finalize retry merging can discard a later valid retry, and the public SDK/Gateway surface has no changelog entry.

## What This Changes

This PR adds generic Plugin SDK workflow seams for session actions, host-mediated attachments, scheduled session turns, finalize retry support, Gateway/Swift protocol models, advanced contract fixtures, and QA/CI hardening.

## Best Possible Solution

Land the generic workflow seams only after the attachment helper passes registry-scoped config into outbound delivery, finalize-retry merging ignores invalid retry objects when a later valid retry exists, the active changelog is updated, the branch is rebased to a mergeable exact head, and the changed SDK/Gateway/build/API gates are green.

## Reproduction Assessment

Yes. The attachment issue is statically reproducible by using a non-default registry config and observing that sendMessage is called without cfg; the retry issue is reproducible with two before_agent_finalize revise hooks where the first retry has a blank instruction and the second has a valid retry instruction.

## Solution Assessment

No. The additive SDK direction is plausible and consistent with the merged host-hook foundation, but this PR is not the best mergeable solution until the scoped outbound config, retry merge semantics, changelog, and dirty head are fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Pass scoped config into attachment sends:** `src/plugins/host-hook-workflow.ts:168-177`
  - body: `sendPluginSessionAttachment` uses `params.config` to find the session delivery route, but the actual `sendMessage` call omits `cfg`. In multi-workspace or custom-store contexts, delivery then falls back to global runtime config and can use the wrong channel/account behavior. Pass `cfg: params.config` and assert it in the workflow test.
  - confidence: 0.91
- **[P2] Preserve later valid finalize retries:** `src/plugins/hooks.ts:341`
  - body: When two finalize hooks both return `revise`, this merge keeps `acc.retry` before normalization. If the earlier retry has an empty instruction and a later hook returns a valid retry, normalization drops the invalid retry and the valid retry is lost. Select the first retry with a non-empty instruction, or strip invalid retry objects before merging.
  - confidence: 0.86
- **[P2] Add the required changelog entry:** `src/plugins/types.ts:2523-2532`
  - body: This PR adds public Plugin SDK and Gateway protocol features, but the PR file list has no `CHANGELOG.md` change. Repo policy requires user-facing feature work to add an active-version changelog bullet with eligible contributor credit before merge.
  - confidence: 0.88
