## Summary

Keep open. Current main does not already contain the workflow SDK/Gateway seams, and the PR remains useful, but the current head is dirty and still has blocking scoped-config, finalize-retry, and changelog issues before it is mergeable.

## What This Changes

This PR adds generic Plugin SDK workflow seams for session actions, host-mediated attachments, scheduled session turns, finalize retry support, Gateway/Swift protocol models, advanced contract fixtures, and QA hardening.

## Best Possible Solution

Land the additive workflow seams only after every host-mediated workflow path carries caller-scoped config, finalize retry merging ignores malformed retry objects when a later valid retry exists, the active changelog is updated, the branch is rebased cleanly, and exact-head SDK/Gateway/build checks pass.

## Reproduction Assessment

Yes. The attachment issue is statically reproducible by using a non-default config and observing sendMessage is called without cfg; the session-extension issue is reproducible through before_tool_call on the createOpenClawTools path because the wrapper omits options.config. The retry issue is reproducible with two revise hooks where the first retry has a blank instruction and the second has a valid instruction.

## Solution Assessment

No. The additive SDK direction is plausible and consistent with the merged host-hook foundation, but this PR head is not the best mergeable solution until scoped config propagation, retry merge semantics, changelog, and the dirty branch are fixed.

## Review Findings

Overall confidence: 0.91

Full review comments:

- **[P2] Pass scoped config into attachment sends:** `src/plugins/host-hook-workflow.ts:168-177`
  - body: `sendPluginSessionAttachment` uses `params.config` to find the session delivery route, but the actual `sendMessage` call omits `cfg`. In custom-store or multi-workspace contexts, delivery can fall back to global runtime config and use the wrong channel/account behavior. Pass `cfg: params.config` and assert it in the workflow test.
  - confidence: 0.92
- **[P2] Require scoped config for session extension reads:** `src/plugins/host-hook-state.ts:411-422`
  - body: The new session-extension read helper still accepts a missing `cfg` and falls back to global runtime config. The main `createOpenClawTools` before-tool-call wrapper does not pass `options.config`, so trusted policies and before_tool_call hooks can read plugin session state from the wrong store. Either make this read path require scoped config or guarantee every caller threads it through.
  - confidence: 0.88
- **[P2] Preserve later valid finalize retries:** `src/plugins/hooks.ts:341`
  - body: When two finalize hooks both return `revise`, this merge keeps `acc.retry` before normalization. If the earlier retry has an empty instruction and a later hook returns a valid retry, normalization drops the invalid retry and the valid retry is lost. Select the first retry with a non-empty instruction, or strip invalid retry objects before merging.
  - confidence: 0.87
- **[P2] Add the required changelog entry:** `src/plugins/types.ts:2524-2533`
  - body: This PR adds public Plugin SDK and Gateway protocol features, but the current head has no active `CHANGELOG.md` entry for them. Repo policy requires user-facing feature work to add an active-version changelog bullet with eligible contributor credit before merge.
