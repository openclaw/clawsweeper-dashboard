## Summary

Keep open. Current main does not implement the new workflow seams, and the PR remains useful, but the current head still has merge-blocking issues: attachment delivery drops the registry-scoped config before the outbound send, finalize retry merging can discard a later valid retry request, and the public SDK/Gateway feature lacks the required changelog entry.

## What This Changes

The PR adds plugin SDK workflow seams for session actions, host-mediated attachments, scheduled session turns, finalize retry, protocol/Swift model updates, advanced workflow fixtures, and QA/CI hardening.

## Best Possible Solution

Land the generic workflow seams after the attachment helper passes the registry-scoped config through delivery execution, finalize-retry merging ignores invalid retry objects when a later valid retry exists, the required changelog entry is added, weak test assertions are corrected, and exact-head changed/build/API gates are green.

## Reproduction Assessment

Yes. The attachment issue has a high-confidence static reproduction by calling sendSessionAttachment from a registry with a non-default config and asserting the mocked sendMessage call lacks cfg; the retry issue is reproducible with two before_agent_finalize revise hooks where the first has a blank retry instruction and the second has a valid retry.

## Solution Assessment

No. The additive SDK direction is plausible and consistent with the merged #72287 foundation, but this PR head is not the best mergeable solution until the scoped outbound config, retry merge semantics, changelog, and focused test gap are fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Pass the scoped config into attachment sends:** `src/plugins/host-hook-workflow.ts:168-177`
  - body: sendPluginSessionAttachment resolves the route with params.config, but the actual sendMessage call omits cfg. In a custom session store or multi-workspace runtime, lookup and delivery can use different configs, which can fail delivery or send artifacts through the wrong channel/account. Pass cfg: params.config and assert it in the workflow test.
  - confidence: 0.91
- **[P2] Preserve later valid finalize retries:** `src/plugins/hooks.ts:341-342`
  - body: When two finalize hooks both return revise, this merge keeps acc.retry before normalization. If the earlier retry has an empty instruction and a later hook returns a valid retry, normalization drops the invalid retry and the valid retry is lost. Select the first retry with a non-empty instruction, or strip invalid retry objects before merging.
  - confidence: 0.84

  - body: This PR adds public Plugin SDK and Gateway protocol features, but the PR file list has no CHANGELOG.md entry. Repo policy requires user-facing feat/fix/perf changes to add an active-version changelog bullet with contributor credit.
  - confidence: 0.87
