## What This Changes

The PR adds generic Plugin SDK workflow APIs, Gateway protocol/client models, scheduler/attachment/finalize-retry handling, generated API baselines, contract fixtures, and QA/CI hardening changes.

## Best Possible Solution

Keep this PR as the implementation vehicle for the generic workflow layer, repair the two narrow regressions on the branch, keep the recipe documentation split in the separate docs PR, and then require exact-head validation before maintainer review.

## Reproduction Assessment

Yes for the review blockers. The Telegram scenario bug reproduces with a one-line Node expression when OPENCLAW_NPM_TELEGRAM_SCENARIOS is set, and the run-context race follows directly from dispatching a slow non-terminal subscription before a terminal lifecycle event while the PR waits only on terminal-dispatch promises.

## Solution Assessment

No for the latest PR head as-is. The generic SDK/Gateway direction is still the right maintainable shape, but the branch should preserve current main's run-level pending-handler cleanup and keep scenario env parsing split before building the Set.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Wait for all run handlers before clearing context:** `src/plugins/host-hook-runtime.ts:305-306`
  - body: Terminal cleanup now waits only for subscriptions dispatched by the terminal event. Earlier async subscriptions for the same run remain tracked in `pendingAgentEventHandlersByRunId`, but they are excluded here, so they can observe cleared run context after lifecycle end. Use the run-level pending set before calling `clearPluginRunContext`.
  - confidence: 0.91
- **[P2] Split scenario env before building the set:** `scripts/e2e/npm-telegram-rtt-driver.mjs:13-18`
  - body: When `OPENCLAW_NPM_TELEGRAM_SCENARIOS` is set, this passes the raw string to `new Set`, creating a set of characters. Then `.has("telegram-mentioned-message-reply")` is false and an explicitly requested RTT scenario is skipped. Wrap the fallback before `.split()` so both env and default values become scenario-id arrays.
  - confidence: 0.98
