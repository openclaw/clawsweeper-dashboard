## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Pass scoped config to attachment delivery:** `src/plugins/host-hook-workflow.ts:168-177`
  - body: `sendPluginSessionAttachment` resolves the session route with `params.config`, but the actual `sendMessage` call omits `cfg`. In custom-store or multi-workspace contexts, `sendMessage` falls back to global runtime config and can deliver through the wrong channel/account configuration. Pass `cfg: params.config` and assert it in the workflow test.
  - confidence: 0.93
- **[P2] Preserve later valid finalize retries:** `src/plugins/hooks.ts:341`
  - body: When two finalize hooks both return `revise`, this merge keeps `acc.retry` before normalization. If the earlier retry has a blank instruction and a later hook returns a valid retry, normalization drops the invalid retry and the valid retry is lost. Select the first retry with a non-empty instruction, or strip invalid retry objects before merging.
  - confidence: 0.89
- **[P2] Split scenario env values before building the set:** `scripts/e2e/npm-telegram-rtt-driver.mjs:13-18`
  - body: When `OPENCLAW_NPM_TELEGRAM_SCENARIOS` is set, `new Set(envString)` creates a set of characters, so `scenarioIds.has("telegram-mentioned-message-reply")` is false and the requested RTT scenario is skipped. Split/trim/filter the env string before passing it to `Set`.
  - confidence: 0.9
- **[P2] Add the required changelog entry:** `src/plugins/types.ts:2524-2533`
  - body: This PR adds public Plugin SDK and Gateway protocol features, but the current head has no active `CHANGELOG.md` entry for them. Repo policy requires user-facing feature work to add an active-version changelog bullet with eligible contributor credit before merge.
  - confidence: 0.88
