Full review comments:

- **[P2] Pass scoped config to attachment delivery:** `src/plugins/host-hook-workflow.ts:168-177`
  - body: `sendPluginSessionAttachment` resolves the session route with `params.config`, but the `sendMessage` call omits `cfg`. In custom-store or multi-workspace contexts, `sendMessage` falls back to global runtime config and can deliver through the wrong channel/account configuration. Pass `cfg: params.config` and assert it in this helper’s test.
  - confidence: 0.93
- **[P2] Split scenario env values before building the Set:** `scripts/e2e/npm-telegram-rtt-driver.mjs:13-19`
  - body: When `OPENCLAW_NPM_TELEGRAM_SCENARIOS` is set, `new Set(envString)` creates a set of characters, so `scenarioIds.has("telegram-mentioned-message-reply")` is false and the requested RTT scenario is skipped. Split, trim, and filter the env string before passing it to `Set`.
  - confidence: 0.95
- **[P2] Select a valid finalize retry when merging:** `src/plugins/hooks.ts:341`
  - body: When two finalize hooks return `revise`, this merge keeps `acc.retry` before normalization. If the earlier retry has a blank instruction and a later hook returns a valid retry, normalization drops the invalid retry and the valid retry is lost. Pick the first retry with a non-empty instruction or strip invalid retry objects before merging.
  - confidence: 0.89
- **[P2] Do not clear run context before handlers settle:** `src/plugins/host-hook-runtime.ts:94`
  - body: `waitForTerminalEventHandlers` resolves after a fixed 5-second timer, and the terminal path clears run context as soon as it resolves. Any agent-event subscription handler that does async work longer than that and then reads run-scoped state can observe missing context mid-handler. Keep context until the tracked handlers settle, or move the timeout to diagnostics without clearing live handler state.
  - confidence: 0.82
- **[P2] Add the required changelog entry:** `src/plugins/types.ts:2524-2533`
  - body: This PR adds public Plugin SDK and Gateway protocol features, but the current head has no active `CHANGELOG.md` entry for the workflow APIs. Repo policy requires user-facing feature work to add an active-version changelog bullet with eligible contributor credit before merge.
  - confidence: 0.88
