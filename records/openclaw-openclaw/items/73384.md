Keep open. Current main still lacks the workflow-session action, attachment, scheduled-turn, and finalize-retry SDK seams, so this PR remains the active implementation vehicle; the latest head still has two narrow P2 correctness blockers before merge.

## What This Changes

The PR adds generic Plugin SDK workflow APIs, Gateway protocol/session-action handling, scheduler and attachment workflow helpers, finalize-retry behavior, generated API/protocol baselines, contract fixtures, and QA/CI hardening changes.

## Best Possible Solution

Keep this PR as the implementation vehicle for the generic workflow layer, repair the two narrow correctness issues on the branch, keep the recipe documentation split in the separate docs PR, and then require exact-head SDK/Gateway validation before maintainer review.

## Reproduction Assessment

Yes for the review blockers. The loader issue can be reproduced with an active registry loaded under skip-runtime-deps followed by a resolve that explicitly requests installBundledRuntimeDeps:true; the finalize-retry issue follows from two before_agent_finalize revise results where the first retry has a blank instruction and the second has a valid one.

## Solution Assessment

No for the latest head as-is. The generic SDK/Gateway direction is the maintainable shape, but the branch should not treat explicit runtime-dependency installation as skip-compatible and should merge finalize retry requests by selecting a valid retry instruction.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Honor explicit runtime-deps installs:** `src/plugins/loader.ts:996`
  - body: This compatibility key always adds an `installBundledRuntimeDeps:false` variant whenever the request is not already false. A caller that explicitly asks for dependency installation can therefore reuse an active registry that was loaded in skip-runtime-deps mode, so a repair or runtime resolve can keep serving plugins that never ran the dependency-install pass. Only use the skip-compatible key when the request itself is allowed to skip installation.
  - confidence: 0.86
- **[P2] Choose a valid finalize retry when merging revisions:** `src/plugins/hooks.ts:341`
  - body: When two `before_agent_finalize` hooks return `revise`, this keeps `acc.retry` even if its `instruction` trims empty. `normalizeBeforeAgentFinalizeResult` then ignores that invalid retry and never considers `next.retry`, so one malformed higher-priority hook can suppress a later valid bounded retry request. Pick the first retry with a non-empty instruction, or strip invalid retry objects before merging.
  - confidence: 0.88
