## Summary

Keep open: the helper-level stale-thread guard is aimed at a real routing bug, but the current patch can drop valid fallback thread context when the same channel target is encoded differently. The PR discussion supports the narrowed helper-only scope and CI is green, but this external PR also still needs real after-fix behavior proof before merge.

## What This Changes

The PR updates `mergeDeliveryContext` and its unit tests to suppress fallback `threadId` inheritance for same-channel `to` mismatches and orphan fallback thread IDs.

## Best Possible Solution

Compare same-channel targets with channel-aware canonicalization where available, or keep target-mismatch suppression in caller paths that already have parsed route context, then add a regression for equivalent target encodings preserving the stored thread.

## Reproduction Assessment

Yes. Source inspection shows current main inherits fallback `threadId` on same-channel different-`to` merges, and the PR tests encode that failing helper path; I did not run tests because this review is read-only.

## Solution Assessment

No, not as written. The helper-level guard is a maintainable place for true cross-conversation mismatches, but the raw string comparison should not drop thread context before channel target equivalence is considered.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Compare same-channel targets canonically before dropping threadId:** `src/utils/delivery-context.shared.ts:128`
  - body: The new `targetsDiffer` check treats any raw `to` string mismatch as a different conversation. Channel targets are not canonical at this helper boundary: Slack-style routes can name the same target as `C1` and `channel:C1`, and the gateway follow-up uses a channel parser specifically to preserve threads for that equivalent-target case. With this check, merging a current `{ channel: "slack", to: "C1" }` route with stored `{ channel: "slack", to: "channel:C1", threadId: "1700.1" }` would drop the valid thread and flatten the reply.
  - confidence: 0.82
