## Summary

Keep open. The PR addresses a plausible stale-thread leak, but the raw `to` mismatch guard can also drop a valid fallback `threadId` when a channel parser treats two raw target strings as the same conversation, and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR adds `mergeDeliveryContext` guards and unit tests to suppress fallback `threadId` inheritance on same-channel target mismatch and orphan fallback-thread merges.

## Best Possible Solution

Preserve the stale-thread protection, but compare parsed or canonical channel routes before dropping a fallback thread, or keep the guard in callers that already have loaded channel target parsing available.

## Reproduction Assessment

Yes, source-reproducible. Current main inherits fallback `threadId` for same-channel delivery-context merges, and the PR regression is also source-reproducible because channel grammars can map different raw target strings to the same conversation.

## Solution Assessment

No, not as written. The helper is a reasonable place to block true cross-conversation leakage, but target mismatch needs parsed/canonical route equivalence or a caller layer with loaded channel parsing.
