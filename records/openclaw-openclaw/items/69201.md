## Summary

Keep open. The PR is a focused delivery-context fix with useful tests, but the current patch can drop valid thread routing when equivalent channel targets are represented by different raw strings, and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR adds `mergeDeliveryContext` guards and unit tests to stop fallback `threadId` inheritance when same-channel `to` values differ or the fallback has a `threadId` without a `to`.

## Best Possible Solution

Keep the stale-thread protection, but compare parsed or canonical route targets before suppressing fallback `threadId`, then add redacted real behavior proof for mismatch, orphan-threadId, and matching-target preservation paths.

## Reproduction Assessment

Yes, at source level. Current main inherits fallback `threadId` on same-channel merges, and the PR unit cases map directly to the mismatched-`to` and orphan-threadId cases, but there is no live transport proof yet.

## Solution Assessment

Unclear as written. The helper-level fix is narrow, but it should compare parsed/canonical route targets or move the guard to call sites with route facts before maintainers rely on it for shared delivery behavior.
