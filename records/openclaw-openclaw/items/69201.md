## Summary

Keep open. The PR addresses a real delivery-context gap, but as written it uses raw `to` inequality in a helper that does not parse plugin target grammars, which can drop valid thread routing; it also still lacks real behavior proof.

## What This Changes

The PR adds guards in `mergeDeliveryContext` and unit tests to stop fallback `threadId` inheritance when same-channel targets differ or fallback has an orphan `threadId`.

## Best Possible Solution

Keep the stale-thread protection, but compare same-channel targets using the channel route parsing/share-conversation contract or a caller layer that already has canonical route facts, with tests for equivalent raw targets and true mismatches.

## Reproduction Assessment

Yes. Current `main` source shows fallback `threadId` is inherited whenever channels match, and existing target-parsing tests show the PR's raw-inequality guard can misclassify equivalent raw target forms.

## Solution Assessment

No, not as written. The helper is a reasonable place to block true stale-thread leakage, but same-channel target mismatch needs canonical route comparison or a caller-layer guard with parsed route facts.
