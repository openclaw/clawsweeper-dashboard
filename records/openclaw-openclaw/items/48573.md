## Summary

Keep open. Current main still treats an existing ACP child session row as sufficient backing for an active ACP task, while parent-owned ACP orphan cleanup skips rows that still have active task references, so the stale-row zombie class remains source-reproducible.

## What This Changes

The issue asks for parent-terminated embedded or ACP child runs to be terminated and their persisted session state removed so later sessions do not see zombie contexts.

## Best Possible Solution

Define an ACP liveness or recovery contract behind Tasks/TaskFlow, then add regression coverage proving a stale parent-owned ACP row cannot keep an active task alive after parent abort or gateway restart.

## Reproduction Assessment

Yes, source-level. Seed a running ACP task older than the lost grace period with a childSessionKey pointing to an existing parent-owned ACP row; current maintenance treats the row as backing state and skips orphan cleanup because an active task exists.

## Solution Assessment

No, current main is not complete. Terminal and no-task ACP cleanup are useful, but the active stale-row case needs an explicit liveness or recovery contract rather than deleting rows by age alone.
