## Summary

Keep open. Current main contains useful ACP task cleanup, but active ACP tasks still treat a persisted ACP child session row as authoritative backing state, so the stale-row zombie class remains source-reproducible and the related durable-executor work still needs rework.

## What This Changes

The issue requests cleanup of embedded-run/ACP child session state so parent termination cannot leave zombie subagent or ACP session entries for later runs.

## Best Possible Solution

Define an ACP liveness or recovery contract behind Tasks/TaskFlow, such as a lease, heartbeat, runtime owner, or recovery hook, and add a focused regression proving a stale parent-owned ACP row cannot keep an active task alive after parent abort or gateway restart.

## Reproduction Assessment

Yes, from source inspection. Seed a running ACP task older than the lost grace period with a childSessionKey pointing to an existing parent-owned ACP session row; current maintenance treats the row as backing state and skips orphan cleanup because an active task exists.

## Solution Assessment

No. Current main's terminal and no-task ACP cleanup is useful but insufficient for active stale rows; the safer fix needs an explicit liveness/recovery contract rather than deleting by age alone.
