## Summary

Keep open. Current main and v2026.5.18 still treat an existing ACP child session-store row as backing state, so stale ACP rows can keep grace-expired tasks alive; the focused fix path remains the open PR at https://github.com/openclaw/openclaw/pull/64616.

## What This Changes

Issue reports stale embedded-run or ACP child session state persisting after parent termination and confusing later runs.

## Best Possible Solution

Land a narrow task-maintenance fix that detects stale parent-owned ACP backing rows through the canonical session and transcript liveness contract while preserving legitimate persistent and bound ACP sessions.

## Reproduction Assessment

Yes, source-level. A grace-expired running ACP task whose childSessionKey still has an ACP session-store row will pass hasBackingSession because current main returns Boolean(acpEntry.entry), so maintenance will not mark it lost.

## Solution Assessment

No, current main is incomplete. The best current path is to review and repair the focused PR at https://github.com/openclaw/openclaw/pull/64616, then close this issue after that fix lands with proof.
