## Summary

Keep open. Current main and v2026.5.22 still treat any existing ACP child-session row as task backing, so the zombie-session state leak remains source-reproducible; the focused remaining implementation path is the open PR at https://github.com/openclaw/openclaw/pull/64616.

## What This Changes

Ensure embedded-run or ACP child sessions are reconciled after parent termination so stale zombie session state does not persist across later runs.

## Best Possible Solution

Land a narrow ACP task-maintenance fix that uses a real liveness contract, resolves transcript/session paths through the canonical session resolver, and preserves legitimate persistent or bound ACP sessions.

## Reproduction Assessment

Yes, source-level. A grace-expired running ACP task whose childSessionKey still has any ACP session row passes hasBackingSession(), so shouldMarkLost() returns false and the stale task remains active.

## Solution Assessment

No, current main is incomplete. The best path is to finish the focused PR direction with canonical path/liveness handling and proof that stale ACP rows are reconciled without breaking live sessions.
