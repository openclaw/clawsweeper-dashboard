## Summary

Keep open. Current main and the latest release still treat any existing ACP child session row as backing state, so a stale ACP row can keep a grace-expired running task alive; the focused implementation path is still the open PR at https://github.com/openclaw/openclaw/pull/64616.

## What This Changes

Clean up embedded-run or ACP child session state so parent termination cannot leave zombie session/task records that later runs treat as live.

## Best Possible Solution

Land a narrow ACP task-maintenance fix that uses the canonical session/transcript liveness contract, preserves legitimate persistent/bound ACP sessions, and then close this issue from the merged PR.
