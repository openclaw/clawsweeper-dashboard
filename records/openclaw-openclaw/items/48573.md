## Summary

Keep open. Current main and v2026.5.18 still accept an existing ACP child session row as backing state, while the focused fix path at https://github.com/openclaw/openclaw/pull/64616 remains open and needs follow-up before this issue can close.

## What This Changes

The issue asks OpenClaw to prevent embedded-run or ACP child session state from surviving parent termination as stale zombie execution context.

## Best Possible Solution

Land a narrow task-maintenance fix that detects stale parent-owned ACP backing rows through the canonical session/transcript liveness contract while preserving legitimate persistent and bound ACP sessions.

## Reproduction Assessment

Yes, source-level. A grace-expired running ACP task whose `childSessionKey` still has an ACP session row will pass `hasBackingSession()` because current main returns `Boolean(acpEntry.entry)`, so maintenance will not mark it lost.

## Solution Assessment

No, current main is incomplete. The open PR direction is narrow, but it still needs canonical session-file/liveness handling and real behavior proof before this issue can close.
