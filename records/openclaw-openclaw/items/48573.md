## Summary

Keep open. Current main and v2026.5.22 still accept an existing ACP child-session row as live task backing, so the stale ACP zombie-session path remains source-reproducible; https://github.com/openclaw/openclaw/pull/64616 is the focused implementation path.

## What This Changes

The issue asks OpenClaw to clean up embedded-run or ACP child session state when a parent agent terminates so stale zombie session contexts are not reused.

## Best Possible Solution

Land or replace the linked ACP task-maintenance fix so stale ACP rows are reconciled through a real liveness contract while preserving legitimate persistent and bound sessions.

## Reproduction Assessment

Yes, source-level. A grace-expired running ACP task whose childSessionKey still has any ACP session row passes hasBackingSession(), so shouldMarkLost() returns false and the stale task remains active.

## Solution Assessment

No, current main is incomplete. The best path is to finish the focused PR direction with canonical liveness handling and proof that stale ACP rows are reconciled without breaking live sessions.
