## Summary

Keep open. Current main and v2026.5.20 still treat any existing ACP child-session row as task backing state, so a stale ACP row can keep zombie work alive; the focused fix path is still the open PR at https://github.com/openclaw/openclaw/pull/64616.

## What This Changes

The issue reports embedded-run or ACP child session state persisting after parent termination and poisoning later session/task state.

## Best Possible Solution

Land a narrow ACP task-maintenance fix that uses the canonical session/transcript liveness contract, preserves legitimate persistent or bound ACP sessions, and closes this issue after the merged PR ships.

## Reproduction Assessment

Yes, source-level. A grace-expired running ACP task whose childSessionKey still has an ACP session row will pass hasBackingSession() because current main returns true for any existing ACP entry.

## Solution Assessment

No, current main is incomplete. The best path is to finish and review the focused PR direction with canonical liveness handling and proof that stale ACP rows are reconciled without breaking live sessions.
