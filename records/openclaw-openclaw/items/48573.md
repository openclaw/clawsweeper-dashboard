## Summary

Keep open. Current main still makes an active ACP task trust the mere existence of its persisted child session row, and orphan cleanup explicitly skips rows that have an active task reference, so the ACP stale-row zombie class remains source-reproducible.

## What This Changes

The issue requests cleanup or recovery for embedded-run/ACP child session state that can survive parent termination as zombie session data.

## Best Possible Solution

Define an ACP liveness or recovery contract behind Tasks/TaskFlow, then add regression coverage proving a stale parent-owned ACP row cannot keep an active task alive after parent abort or gateway restart.

## Reproduction Assessment

Yes, from source inspection. Seed a running ACP task older than the lost grace period with a `childSessionKey` pointing to an existing parent-owned ACP row; current maintenance treats the row as backing state and skips orphan cleanup because an active task exists.

## Solution Assessment

No, current main is not a complete solution. Terminal/no-task ACP cleanup is useful, but the active stale-row case needs an explicit liveness or recovery contract rather than deleting rows by age alone.
