## Summary

Keep open: this is a focused bug-fix PR for a source-provable heartbeat pending-delivery loop, but the latest patch still has a P2 correctness issue in pending replay cleanup and the supplied proof does not show the actual session-store write/clear behavior.

## What This Changes

This PR suppresses or clears acknowledgement-only `HEARTBEAT_OK` heartbeat pending final-delivery state, preserves real heartbeat content, adjusts the heartbeat defer check, and adds regression tests plus a changelog entry.

## Best Possible Solution

Keep the PR focused on heartbeat pending delivery, thread the same resolved `ackMaxChars` policy through write, defer, and replay cleanup paths, and add redacted terminal or log proof of the actual session-store state transitions.

## Reproduction Assessment

Yes, source-reproducible. The linked report gives deterministic v2026.5.7 steps, and current main source shows the pending write, heartbeat replay attempt update, and defer-window loop; I did not run a live gateway reproduction in this read-only review.

## Solution Assessment

No, not yet. Reusing `stripHeartbeatToken` is the right direction, but the replay cleanup path must use the same resolved `ackMaxChars` policy as the runner/write path before this is the narrow maintainable fix.
