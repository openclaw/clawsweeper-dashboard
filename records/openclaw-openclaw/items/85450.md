## Label Justifications

- `P1`: A real Feishu group button-click workflow can silently drop the agent command when mention gating is enabled.
- `impact:message-loss`: The card-action callback becomes an inbound synthetic message that is dropped before agent dispatch.

## Summary

Keep open: current main and the latest release still build Feishu card-action synthetic group messages without mention metadata, so the existing `requireMention` gate can drop the callback before agent dispatch. Two open fix PRs already reference this issue, so the issue should stay open until one focused fix merges.

## What This Changes

The issue requests that Feishu structured card action callbacks in `requireMention` group chats dispatch to the agent instead of being filtered as unmentioned group messages.

## Best Possible Solution

Review and land one focused Feishu plugin fix, preferably one that marks synthetic group card callbacks as bot-addressed without weakening group and sender authorization, then close this issue and supersede duplicate fix attempts.

## Reproduction Assessment

Yes, from source inspection: a group card-action callback is synthesized without mention metadata, parses as `mentionedBot=false`, and reaches the existing `requireMention` drop path. I did not run a live Feishu environment in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix belongs in the Feishu card-action dispatch path by carrying a valid bot-addressed fact or mention metadata into `handleFeishuMessage`. A broad bypass of activation gates is riskier than preserving group and sender authorization while satisfying mention activation.
