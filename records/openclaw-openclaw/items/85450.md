## Summary

Keep open: current main and the latest release still appear to drop Feishu structured card action callbacks in `requireMention: true` groups because the synthetic card-action message is not marked as mentioning the bot, and the group activation gate then returns the same missing-mention drop reported in the logs.

## What This Changes

The issue requests fixing Feishu structured card action callbacks so button clicks in `requireMention: true` group chats reach the agent instead of being silently filtered as unmentioned messages.

## Best Possible Solution

Fix the Feishu plugin so card-action synthetic group callbacks carry an explicit bot-addressed activation fact or valid bot mention metadata before `handleFeishuMessage`, while preserving group and sender authorization gates.

## Reproduction Assessment

Yes, from source inspection: a group card-action callback builds a synthetic event with no mentions, parses as `mentionedBot=false`, and then hits the existing require-mention drop path. I did not run a live Feishu environment in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is inside the Feishu card-action dispatch path, not a new config option: mark synthetic card callbacks as bot-addressed while keeping group and sender allowlists intact.
