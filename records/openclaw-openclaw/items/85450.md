## Summary

Keep open: current main and the latest release still build Feishu group card-action callbacks as synthetic text messages without mention metadata, so `requireMention: true` can drop the callback before agent dispatch. Two open PRs already reference this issue with closing syntax, so the issue should stay open until one canonical fix lands or the PRs are closed.

## What This Changes

Feishu structured card button callbacks in group chats should reach the agent even when group `requireMention` gating is enabled.

## Best Possible Solution

Land a narrow Feishu plugin fix that marks synthetic card callbacks as bot-addressed before the existing mention gate while preserving group admission and sender allowlist checks, then close this issue through the merged PR.

## Reproduction Assessment

Yes. From source inspection, a group card-action callback builds a synthetic text event without `message.mentions`, parses as `mentionedBot=false`, and then hits the existing `requireMention` drop path; I did not run a live Feishu environment.

## Solution Assessment

Yes, the best fix is a narrow Feishu card-action dispatch repair that supplies a valid bot-addressed activation fact or mention metadata before `handleFeishuMessage`, not a new config option or a broad bypass of group policy.
