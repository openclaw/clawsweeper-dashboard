## Summary

Keep open: current main and latest release still source-reproduce the Feishu card-action drop in mention-gated groups because the synthetic card callback reaches the normal group mention gate without mention metadata.

## What This Changes

The issue reports that Feishu structured card button callbacks in group chats with `requireMention: true` are filtered before reaching the agent.

## Best Possible Solution

Fix the Feishu plugin so card-action synthetic group callbacks carry an explicit bot-addressed activation fact or valid bot mention metadata before `handleFeishuMessage`, while preserving group and sender authorization gates.

## Reproduction Assessment

Yes. From source inspection, a group card-action callback builds a synthetic event with no mentions, parses as `mentionedBot=false`, and then hits the existing require-mention drop path; I did not run a live Feishu environment.

## Solution Assessment

Yes. The narrow maintainable fix is in the Feishu card-action dispatch path: mark synthetic card callbacks as bot-addressed while keeping group and sender allowlists intact.
