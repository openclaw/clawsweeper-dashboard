## Summary

Keep open: current main and the latest release still synthesize Feishu group card-action callbacks without mention metadata, so the normal group `requireMention` gate can still drop the callback before agent dispatch. The issue is already paired with an open closing PR, so cleanup should not close it before that PR or an equivalent Feishu plugin fix lands.

## What This Changes

The issue reports Feishu structured card action callbacks being filtered in group chats when `requireMention` is enabled.

## Best Possible Solution

Land the narrow Feishu plugin fix that makes bot-originated group card callbacks satisfy the existing mention gate while preserving `requireMention` for ordinary group messages and keeping p2p callbacks unchanged.

## Reproduction Assessment

Yes, at source level. A group card callback is synthesized without mention metadata in `extensions/feishu/src/card-action.ts`, and `extensions/feishu/src/bot.ts` passes `ctx.mentionedBot` into the `requireMention` gate that logs and returns when the bot was not detected.

## Solution Assessment

Yes, the linked PR's group-only synthetic mention path appears to be the narrowest maintainable fix because it reuses the existing Feishu mention gate instead of bypassing group policy. The issue should remain open until that PR or an equivalent patch is merged.
