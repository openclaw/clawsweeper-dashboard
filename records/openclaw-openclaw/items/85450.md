## Summary

Keep open: current main and v2026.6.8 still synthesize Feishu group card-action messages without mention metadata, while the group activation gate still requires a positive bot mention when `requireMention` is enabled. The linked closing PR remains open and unmerged, so this issue should stay open until that PR or an equivalent Feishu plugin fix lands.

## What This Changes

Requests a Feishu bug fix so structured card button callbacks in `requireMention` group chats reach the agent instead of being dropped by mention gating.

## Best Possible Solution

Keep this issue open and review, rebase, prove, and land https://github.com/openclaw/openclaw/pull/85481, or replace it with an equivalent focused Feishu plugin fix that preserves the existing `requireMention` policy.

## Reproduction Assessment

Yes, at source level: current main builds group card-action synthetic messages without `message.mentions`, and the group activation gate requires `ctx.mentionedBot` when `requireMention` applies. I did not perform a live Feishu click in this read-only review.

## Solution Assessment

Yes for the current fix direction: adding group-only bot mention metadata reuses the existing Feishu mention gate, while bypassing `requireMention` would be broader and weaker. The issue itself should remain open until the linked focused PR or an equivalent patch lands.
