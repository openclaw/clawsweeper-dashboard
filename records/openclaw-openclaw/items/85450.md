Keep open: current main and the latest release still synthesize Feishu card-action group callbacks without mention metadata, so the existing `requireMention` gate can drop the callback before agent dispatch; two open closing PRs are already the active fix candidates.

## What This Changes

The issue reports Feishu structured card button callbacks being dropped in `requireMention` group chats before the agent receives the command.

## Best Possible Solution

Review and land a narrow Feishu fix that marks synthetic group card-action callbacks as bot-addressed while preserving group admission and sender allowlists, then close this issue after the fixing PR merges.

## Reproduction Assessment

Yes, source-reproducible: current main builds a group card-action text event without `message.mentions`, parses it as `mentionedBot=false`, and then reaches the existing `requireMention` drop path. I did not run a live Feishu environment in this read-only review.

## Solution Assessment

Yes, the best fix is a narrow Feishu plugin change in the card-action synthesis or dispatch path, not a new config option. The safer shape is to satisfy the existing activation contract for card callbacks while keeping group and sender authorization gates intact.
