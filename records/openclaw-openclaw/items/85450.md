## Label Justifications

- `P1`: A real Feishu group button-click workflow can silently drop the agent command when mention gating is enabled.
- `impact:message-loss`: The structured card callback becomes an inbound synthetic message that is dropped before agent dispatch.

## Summary

Keep open: current main and the latest release still build Feishu card-action callbacks as synthetic group text messages without bot mention metadata, so `requireMention: true` can still reject the callback before agent dispatch. Two open PRs already reference this issue, so the issue should stay open until one narrow Feishu fix lands.

## What This Changes

Feishu structured card action callbacks in `requireMention: true` group chats should reach the agent instead of being filtered as unmentioned group messages.

## Best Possible Solution

Land one narrow Feishu plugin fix that carries a bot-addressed activation fact or valid bot mention metadata for synthetic card callbacks, then close the issue after the merged PR proves the regression path.

## Reproduction Assessment

Yes. Source inspection shows a group card-action callback builds a synthetic event without `mentions`, parses as `mentionedBot=false`, and then reaches the existing require-mention drop path.

## Solution Assessment

Yes. The best fix is a narrow Feishu card-action dispatch change that marks synthetic card callbacks as bot-addressed without disabling group admission or sender authorization globally.
