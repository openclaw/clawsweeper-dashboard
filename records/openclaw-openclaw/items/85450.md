## Summary

This issue should stay open because current main still routes Feishu card-action callbacks through normal group mention gating, and the synthetic callback event does not provide any mention fact for `requireMention: true` groups, matching the reported silent drop path.

## What This Changes

The issue reports Feishu structured card action callbacks in group chats with `requireMention: true` being treated as unmentioned messages and dropped before agent dispatch.

## Best Possible Solution

Treat verified Feishu card-action callbacks as bot-directed for group mention admission while preserving group admission and sender allowlist checks, then add a regression test for a group callback with `requireMention: true`.

## Reproduction Assessment

Yes, at source level: current main builds a card-action text event without mentions, parses `mentionedBot` as false, and then the group ingress path drops it when `requireMention` is true. I did not run a live Feishu callback against real credentials.

## Solution Assessment

Yes, a narrow fix in the Feishu card-action synthetic dispatch path is the maintainable direction. The fix should not disable group or sender policy; it should only mark authenticated card-action callbacks as bot-directed for mention admission.
