## Summary

Current main still drops the reported Feishu path: synthetic card-action group messages are built without `message.mentions`, while the group `requireMention` gate admits dispatch only when the parsed message context says the bot was mentioned. The issue should stay open because https://github.com/openclaw/openclaw/pull/85481 is still the active open closing PR and the latest release has not shipped the fix.

## What This Changes

Keep Feishu structured card-action callbacks from being filtered by group `requireMention` when a user clicks a bot-sent card button.

## Best Possible Solution

Keep this issue open and land the linked focused Feishu plugin PR, or replace it with an equivalent patch that supplies valid bot-addressed activation metadata without weakening normal group gates.

## Reproduction Assessment

Yes. Source inspection on current main shows group card-action synthetic messages are built without `message.mentions`, while the group `requireMention` gate depends on `ctx.mentionedBot`; reporter logs show the same drop path.

## Solution Assessment

Yes for the current direction. The narrow maintainable fix is to make synthetic group card callbacks satisfy the existing Feishu mention-gate contract, as the linked PR attempts, rather than bypassing ordinary group admission broadly.
