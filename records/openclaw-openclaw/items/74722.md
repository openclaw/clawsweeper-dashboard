## Summary

## What This Changes

Request language-aware TTS provider/model/voice routing so inbound voice replies can synthesize with a route matching the detected input or reply language.

## Best Possible Solution

Keep this open for maintainer product/API review, then implement a generic speech language-routing contract with detected-language metadata propagation and fallback to the current fixed TTS behavior when disabled.

## Reproduction Assessment
