## Summary

Keep open. Current main still has a narrow plugin-update diagnostics gap: npm metadata probe failures are not retried and the updater falls through to the installer path, while the exact valid `acpx@0.5.3` to unsupported-spec transition is not fully proven from source.

## What This Changes

The issue asks plugin update checks to retry or clearly report transient npm metadata failures instead of surfacing a misleading unsupported npm spec error for a valid tracked npm plugin.

## Best Possible Solution

Add focused regression coverage, then retry transient or incomplete npm metadata probes and preserve the registry-failure context so valid plugin specs are not reported as invalid.

No, not as a high-confidence live reproduction. Source inspection shows the metadata probe failure and installer fallback path, but current main does not prove the specific `acpx@0.5.3` unsupported-spec result.

## Solution Assessment

Yes, a narrow repair in the npm metadata resolver and plugin update failure formatting is the best direction. Keep registry spec validation strict and avoid changing ClawHub/catalog migration behavior.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/77616. Current main makes a single `npm view` metadata probe in `resolveNpmSpecMetadata`, logs a warning in `updateNpmInstalledPlugins` when it fails, and then falls through to `installPluginFromNpmSpec`; if the installer path also fails, the final diagnostic can lose the transient registry context. Add regression coverage for incomplete/transient npm view metadata failure plus fallback failure, then implement a narrow fix in `src/infra/install-source-utils.ts` and/or `src/plugins/update.ts`: retry transient/incomplete/invalid-JSON npm metadata probes with bounded backoff/no-output timeout, or preserve the precheck registry failure in the final npm update error for valid specs. Do not loosen `parseRegistryNpmSpec` beyond exact versions and dist-tags, do not change ClawHub/catalog migration behavior, and keep third-party plugin contracts unchanged. Add a user-facing changelog fix entry.
