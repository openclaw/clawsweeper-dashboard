The exact follow-on `unsupported npm spec` line is not proven on current main because exact versions and beta tags validate now, but the report still describes a real narrow plugin update reliability gap.

## What This Changes

The issue asks OpenClaw to avoid misleading plugin update errors after transient npm metadata lookup failures during `openclaw update` or post-core plugin sync.

## Best Possible Solution

Centralize bounded retry/no-output handling in `resolveNpmSpecMetadata`, keep strict registry spec validation, and add regression coverage for clear transient npm metadata errors in update and install flows.
