## Summary

Keep open. The PR is a useful partial fix, and the earlier internal-default and zero-mode review concerns appear addressed, but it still leaves the sandboxed Write/Edit path unchanged; that path is the one tied to the current stat/EACCES evidence in #44077.

## What This Changes

The PR adds optional file-mode plumbing to host safe writes, passes `0o644` from host workspace Write/Edit operations, and changes the pinned helper mode fallback to nullish coalescing.

## Best Possible Solution

Extend the readable user-document mode through both host and sandboxed Write/Edit paths, keep internal and sensitive writes private by default, preserve existing regular-file modes during replacement, and add focused permission regression tests plus the required user-facing changelog entry.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a sandboxed session with workspace writes enabled: `pi-tools.ts` routes to sandboxed Write/Edit, those call `bridge.writeFile` without a mode, and the sandbox mutation helper creates replacement files with `0o600`; #44077 adds concrete stat and EACCES evidence for that path.

## Solution Assessment

No. The host-only override is directionally correct but incomplete; the narrower maintainable fix is nullable-safe mode/default-mode plumbing through the sandbox bridge and mutation helper as well, with tests proving both `0o644` user-facing creation and `0o600` internal defaults.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Apply readable mode in sandboxed Write/Edit flows:** `src/agents/pi-tools.read.ts:786`
  - body: Passing `mode: 0o644` only from the host workspace operations leaves sandboxed sessions unchanged: `pi-tools.ts` selects `createSandboxedWriteTool`/`createSandboxedEditTool` when a sandbox root is active, and those still call `bridge.writeFile` without any mode. The sandbox bridge's mutation helper creates replacement files with `0o600`, so sandboxed Write/Edit still produce owner-only files and the Syncthing/EACCES failure remains. Thread the mode through the sandbox bridge/mutation helper and pass `0o644` from the sandboxed user-facing operations too.
