## What This Changes

The PR adds a `mode` option to the legacy fs-safe write path and passes `0o644` from host Write/Edit workspace writes.

## Best Possible Solution

Approve or defer the user-output permission policy; if approved, implement it through current `root.write` and sandbox bridge paths with focused host/sandbox permission tests and real redacted `stat` proof.

## Reproduction Assessment

Yes at source level: current host workspace Write/Edit omits a mode and `@openclaw/fs-safe@0.2.7` defaults new writes to `0o600`; sandbox Write/Edit also has no mode contract and its mutation helper creates new temp files with `0o600`. I did not run a live Write/Edit session in this read-only review.

## Solution Assessment

No, not as-is. The useful fix should target the current fs-safe root API and sandbox bridge path after maintainers approve the `0644` user-output policy or defer it to the configurable permissions design.
