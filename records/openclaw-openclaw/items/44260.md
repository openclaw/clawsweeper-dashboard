## Summary

Keep open: the PR is useful but incomplete because it only updates host workspace Write/Edit writes, while sandboxed Write/Edit still create new files through bridge paths that default to 0o600. Current main's related #44077 fix preserves existing sandbox file modes, but it does not make newly created Write/Edit files readable for sandboxed sessions.

## What This Changes

The PR adds optional file-mode plumbing to safe workspace writes, passes 0o644 from host Write/Edit operations, and switches one pinned-helper mode fallback to nullish coalescing.

## Best Possible Solution

Thread an explicit user-document mode through both host and sandbox Write/Edit creation paths, preserve existing target modes on replacement, keep generic/internal safe-write defaults private, and add focused tests plus changelog coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: sandboxed sessions route to sandbox Write/Edit tools, those call bridge.writeFile without a mode, and the mutation helper creates new write temp files with 0o600.

## Solution Assessment

No. The host-only override is directionally correct but incomplete; the maintainable fix should extend mode plumbing through SandboxFsBridge and both bridge implementations while preserving internal private defaults and existing target modes.
