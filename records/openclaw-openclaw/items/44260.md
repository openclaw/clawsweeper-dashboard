## What This Changes

The PR adds a mode option to the old fs-safe write path and passes 0o644 from host Write/Edit operations.

## Best Possible Solution

Define a maintainer-approved user-output file-mode policy, then implement it consistently for host workspace and sandbox Write/Edit while preserving restrictive internal writes.

## Reproduction Assessment

Yes, at source level. Current main host workspace Write/Edit calls root.write without mode, fs-safe defaults missing modes to 0o600, and sandbox Write/Edit has no mode field while its mutation helper creates new files at 0o600.

## Solution Assessment

No. The branch is not the best current fix because it targets the old in-repo fs-safe path, does not carry a mode through sandbox Write/Edit, and should be aligned with the broader user-output permission policy.
