## Summary

Keep open: the intent is useful, but the PR is not merge-ready because it only changes host Write/Edit paths, targets fs-safe internals that current main has since replaced with the package-backed root writer, lacks sandbox coverage, and still needs maintainer approval for the broader file-permission policy.

## What This Changes

The PR threads an optional file mode through the old fs-safe write pipeline and passes `0o644` from host Write/Edit workspace writes.

## Best Possible Solution

Approve a scoped user-output permission policy, then implement it consistently through current host `root.write` and sandbox bridge/mutation paths with focused permission tests, or defer the hard-coded default to the configurable policy tracked in https://github.com/openclaw/openclaw/issues/51660.
