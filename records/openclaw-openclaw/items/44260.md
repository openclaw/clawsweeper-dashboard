## Summary

Keep this PR open: the direction is useful, but the current diff only updates the host workspace Write/Edit path and leaves sandboxed Write/Edit creating new files through the 0o600 sandbox mutation helper. The related sandbox regression was partly fixed on main for existing-file mode preservation, but that does not implement this PR’s new-file 0o644 behavior.

## What This Changes

The PR adds an optional file mode through safe workspace writes and passes 0o644 from host workspace Write/Edit while preserving the internal 0o600 default and nullish mode handling.

## Best Possible Solution

Apply the readable default consistently to user-facing Write/Edit new-file creation in both host and sandbox flows, preserve existing target modes on edits, keep internal sensitive defaults private, and cover the contract with targeted tests plus a changelog entry.

## Reproduction Assessment

Yes from source inspection. Current main routes sandboxed Write/Edit through bridge.writeFile and a helper that creates brand-new temp files with 0o600; I did not run a live chmod/stat reproduction because this review is read-only.

## Solution Assessment

No. Passing 0o644 only through host workspace Write/Edit is not the narrow complete fix because sandboxed Write/Edit uses a separate bridge path that still creates new files as 0o600.
