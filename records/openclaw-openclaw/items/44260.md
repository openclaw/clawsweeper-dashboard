## Summary

Keep this PR open: the host workspace change is useful, but the patch still leaves sandboxed Write/Edit flows creating new files through the private-mode sandbox bridge path, so the reported readability problem is not fully fixed.

## What This Changes

The PR adds optional file-mode plumbing to safe workspace writes, passes `0o644` from host workspace Write/Edit operations, and switches one pinned-helper fallback to nullish coalescing.

## Best Possible Solution

Thread an explicit user-document mode through both host and sandboxed Write/Edit creation paths, preserve existing file modes during replacements, keep generic/sensitive safe-write defaults private, and cover the behavior with permission tests and changelog.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: sandboxed sessions route to sandbox Write/Edit tools, those call `bridge.writeFile` without mode support, and the mutation helper creates new write temp files with `0o600`.

## Solution Assessment

No. The host-only override is directionally correct but incomplete; the maintainable fix should extend mode plumbing through `SandboxFsBridge` and both bridge implementations while preserving internal private defaults and existing target modes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Thread the user-document mode through the sandbox bridge path as well so sandboxed Write/Edit no longer reproduce the readability failure.
