## Summary

Keep this open: current main still has the EACCES-to-fallback config path and the write blocker still misses unreadable/last-known-good shape-loss reasons; the sudo/plugin state ownership half also remains unresolved while the related PR only covers the config guard half.

## What This Changes

Prevent sudo-driven OpenClaw update/doctor/plugin flows from corrupting user-owned config/plugin state and block config overwrites after unreadable config snapshots.

## Best Possible Solution

Land or supersede the focused config write-guard PR, then add a separate owner-aware guard for sudo/root-run state and plugin mutations under a user-owned OpenClaw state directory.

Yes. Source inspection shows EACCES still produces a fallback snapshot that can flow into later writes, and plugin update/install paths still write managed state without an effective-UID/state-owner guard; I did not run a live sudo reproduction in this read-only pass.

## Solution Assessment

Partly. The open config write-guard PR is the narrow maintainable fix for the config clobber half, but the full issue also needs a maintainer decision for sudo-owned plugin/state mutation behavior.
