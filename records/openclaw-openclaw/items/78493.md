## Summary

Keep this open: current main still has a source-reproducible unsafe path from an EACCES-derived fallback config into later config writes, and the sudo/root-owned plugin/state mutation half is not fixed. The open related PR covers only the config write-guard half and intentionally leaves sudo-owned plugin/state artifacts for this issue.

## What This Changes

The issue asks OpenClaw to fail closed when sudo/root-run update or doctor paths make user config/state unreadable or root-owned instead of clobbering config or leaving mixed ownership.

## Best Possible Solution

Land or supersede the focused config write-guard PR, then add a separate ownership guard for sudo/root-run state and plugin mutations under a user-owned OpenClaw state directory.

## Reproduction Assessment

Yes. Source inspection shows EACCES produces a fallback snapshot and current write blockers do not reject unreadable-base or last-known-good shape loss, while update/plugin install paths still write managed state without an effective-UID owner check.

## Solution Assessment

Partly. The focused config write-guard direction in https://github.com/openclaw/openclaw/pull/79299 is the right narrow fix for the clobber half, but the sudo-owned plugin/state half needs a separate maintainer decision on abort, re-exec, or ownership repair.
