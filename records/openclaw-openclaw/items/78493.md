## Summary

Keep this open: current main still has the EACCES-to-fallback config read path, write blocking still does not include unreadable-config or last-known-good shape-loss reasons, and the sudo/root-owned plugin-state half remains unresolved while the related config-only PR is still open.

## What This Changes

The issue asks OpenClaw to fail closed when sudo/root-run commands make user config or plugin state unreadable or owner-mixed, instead of overwriting config or leaving root-owned state artifacts.

## Best Possible Solution

Land or supersede the focused config write-guard fix, then add an owner-aware guard for sudo/root-run config, plugin npm root, plugin ledger, and plugin-skills mutations under a user-owned OpenClaw state directory.

## Reproduction Assessment

Yes, at source level. Current main still converts EACCES into a fallback snapshot and lets doctor/update/plugin write paths rely on blockers that do not include unreadable-config or last-known-good shape-loss reasons; I did not run a live sudo reproduction.

## Solution Assessment

Partly. https://github.com/openclaw/openclaw/pull/79299 is the narrow maintainable fix for the config clobber half, but the full issue also needs an explicit owner-aware policy for sudo/root-run state and plugin mutations.
