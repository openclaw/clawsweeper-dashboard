## Summary

Keep open: current main still has the EACCES-to-fallback config path and the remaining sudo/plugin-state ownership issue is not implemented; the related PR only covers the config write-guard half and is still open.

## What This Changes

The issue requests fail-closed config writes and owner-aware sudo/update/plugin-state guards after sudo-run commands can leave mixed root/user ownership and let doctor clobber an unreadable config.

## Best Possible Solution

Land or supersede the focused config write-guard PR, then add an owner-aware guard for sudo/root-run state and plugin mutations under a user-owned OpenClaw state directory.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path and the issue has macOS plus Linux sudo evidence. I did not execute a live sudo flow in this read-only review, so this is source-reproducible rather than directly reproduced here.

## Solution Assessment

Partly. The open config write-guard PR is the narrow maintainable fix for the config clobber path, but the remaining sudo-owned plugin/state mutation behavior needs an explicit maintainer policy choice.
