## Summary

## What This Changes

This PR edits old Pi model discovery to materialize sanitized static provider/model catalog headers onto resolved models, adds resolver tests, and documents `ctx.model.headers` for provider wrappers.

## Best Possible Solution

Retarget the fix to the current ModelRegistry and resolveModel path, reuse the existing loader/ownership filtering, preserve SecretRef stripping, and land it only after maintainers accept the static-header wrapper contract with real proof.

## Reproduction Assessment

Yes at source level: current main stores models.json provider/model headers for request auth while leaving registry-loaded model.headers undefined, and provider wrappers receive the resolved model object. I did not run a live provider scenario in this read-only review.

## Solution Assessment

No: the goal is valid, but this branch is not the best fix because it reparses stale Pi-era files instead of reusing the current ModelRegistry loader and resolveModel header path.
