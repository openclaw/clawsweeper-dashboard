## What This Changes

This PR materializes sanitized provider/model headers from generated models.json onto PI-discovered runtime models, adds resolver regression coverage, and documents ctx.model.headers for provider stream wrappers.

## Best Possible Solution

Materialize sanitized catalog headers using PI-compatible parsing or registry data, cover provider headers, model row headers, modelOverrides headers, and SecretRef stripping, then attach redacted after-fix behavior proof.

## Reproduction Assessment

Yes, as a source-level reproduction: PI stores models.json provider/model headers for request auth while custom model objects can have model.headers undefined, and OpenClaw passes that model into wrapStreamFn. I did not run tests because this review is read-only.

## Solution Assessment

No, not yet: exposing sanitized catalog headers is the right narrow direction, but the patch should reuse PI-compatible parsing or registry data and include all supported header sources before merge.
