## Summary

## What This Changes

The branch materializes sanitized provider/model catalog headers onto resolved models in the old Pi discovery path, adds resolver tests, and documents `ctx.model.headers` for provider stream wrappers.

## Best Possible Solution

If maintainers accept wrapper-visible static catalog headers, forward-port the fix into the active `ModelRegistry` and embedded resolver/request-config path for both root JSONC and generated plugin catalogs, preserve SecretRef stripping, add focused tests/docs, and require redacted real behavior proof.

## Solution Assessment

No. Exposing sanitized headers may be the right bug-fix direction, but this branch targets removed Pi files and adds a parallel root-only parser instead of using the current registry/request-config catalog path.
