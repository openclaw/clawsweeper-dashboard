## Summary

Keep this PR open: the fix direction is right, but the current patch has a parser mismatch with the PI models.json contract and the external-PR real behavior proof is still missing.

## What This Changes

The PR materializes sanitized provider/model headers from generated models.json onto discovered PI runtime models, adds resolver regression tests, and documents ctx.model.headers for provider wrappers.

## Best Possible Solution

Reuse PI-compatible models.json parsing or registry data for header materialization, keep SecretRef stripping, add coverage for comments/trailing commas, then attach real after-fix proof.

## Reproduction Assessment

Yes, as a source-level reproduction: PI stores models.json provider/model headers for request auth while generated custom model objects have headers undefined, and OpenClaw passes that resolved model into provider stream wrappers. I did not run tests because this review is read-only.

## Solution Assessment

No, not yet: exposing sanitized catalog headers is the right narrow fix, but the patch should honor PI's JSONC-compatible models.json parsing instead of adding a stricter raw JSON.parse path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Reuse the same parser/data source or add equivalent parsing and coverage.
