## Summary

Keep open: this is a plausible narrow bug fix for provider catalog headers, but the branch has a correctness gap for supported `models.json` syntax and still lacks inspected real behavior proof.

## What This Changes

The PR materializes sanitized catalog headers in `src/agents/pi-model-discovery.ts`, adds generated `models.json` header regression tests, and documents `ctx.model.headers` for provider wrappers.

## Best Possible Solution

Materialize sanitized provider/model catalog headers through the model-discovery bridge using PI-compatible parser semantics, keep SecretRef values stripped, and document `ctx.model.headers` once real proof is supplied.

## Reproduction Assessment

Yes, as a source-level reproduction: pi-coding-agent accepts provider/model headers for request auth but emits generated custom model objects with `headers: undefined`, and current main forwards only `discoveredModel.headers` to wrappers when no OpenClaw provider config is present.

## Solution Assessment

No, not as-is: the direction is narrow and maintainable, but the patch should parse `models.json` with the same comments/trailing-comma tolerance as PI and include contributor-owned real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86
