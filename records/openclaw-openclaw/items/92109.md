## Label Justifications

- `P1`: The reproduced regression prevents cron jobs and subagent spawns from completing on affected Btrfs-backed deployments.
- `impact:session-state`: The session-file takeover fence misclassifies unchanged session state and aborts embedded attempts.

## Summary

The regression is present on current main and has strong Btrfs evidence, but this issue should remain open because two linked fix PRs are still open; the content-verifying approach in https://github.com/openclaw/openclaw/pull/92123 better preserves the takeover safety boundary than globally removing ctimeNs.

## What This Changes

Fix false session-takeover detection when Btrfs changes only a session file’s ctime metadata.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/92123 as the canonical fix if its content-verification boundary and checks remain sound, close the competing broader implementation, and let this issue close only after the canonical PR merges.

## Reproduction Assessment

Yes. The reporter supplied a deterministic Btrfs stat probe showing ctime-only drift and a repeated v2026.6.5 failure path that maps directly to current main’s fingerprint comparison.

## Solution Assessment

No for unconditional ctimeNs removal; verifying that transcript content is unchanged before accepting ctime-only drift is the narrowest maintainable fix because it resolves the filesystem false positive without discarding protection against actual same-size rewrites.
