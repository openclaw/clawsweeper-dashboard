## Summary

Keep open for maintainer review: the Mattermost-local code path is a reasonable fit for the requested Slack/Telegram parity, but the PR changes default reply engagement in existing Mattermost rooms and the user-facing docs still describe the old mention-only behavior.

## What This Changes

This PR adds a cached Mattermost root-post lookup and mention-gate input so replies in bot-authored threads can bypass `requireMention` and `onchar`, with focused unit coverage.

## Best Possible Solution

Land the Mattermost-local implementation after docs reflect the new default-on implicit reply behavior and maintainers accept the expanded reply surface; keep https://github.com/openclaw/openclaw/pull/95552 as the separate participated-thread path.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature PR. Source inspection does show current main drops unmentioned Mattermost channel/group replies under `requireMention` or untriggered `onchar`, which is the behavior the PR intentionally changes.

## Solution Assessment

Yes, with one docs gap: a Mattermost-local gate input plus cached root-post lookup is the right owner boundary because the plugin owns Mattermost mention gating and the upstream post model exposes only `root_id` plus root `user_id`. The remaining decision is whether default-on parity is acceptable for existing rooms.
