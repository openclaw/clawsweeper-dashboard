## Summary

Keep open. Current main and the latest release still lack the requested subscribeEmbeddedPiSession-level markdown-table integration coverage, and the underlying block chunker remains fence-aware rather than table-aware. Open PR #66344 explicitly closes this issue, and #73981 overlaps the same table-safe streaming work, so this should stay open until one of those PRs lands or is closed.

## What This Changes

The issue requests an integration test proving block streaming does not split a markdown table across subscribeEmbeddedPiSession block replies.

## Best Possible Solution

Land one coordinated table-safe streaming change that adds protected markdown table spans to the shared chunking path and includes the requested subscribeEmbeddedPiSession Intro/table/Outro integration guard, then close this through the merged PR alongside the related table-splitting bug.

## Reproduction Assessment

Yes. The coverage gap is reproducible by inspecting the current integration test file, and the runtime gap is traceable through createParagraphChunkedBlockReplyHarness to EmbeddedBlockChunker, whose safe-break logic only checks fence spans.

## Solution Assessment

Yes. The requested integration test is the right guardrail for the production block-reply pipeline, but it should land with or after the table-aware chunker fix rather than as a standalone failing test.
