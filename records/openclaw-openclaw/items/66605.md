## Summary

Keep open. Current main still lacks the requested subscribeEmbeddedPiSession-level markdown-table integration test, the active chunking code remains fence-aware rather than table-aware, and open PR #66344 explicitly closes this issue while PR #73981 overlaps the same table-safe streaming work.

## What This Changes

The issue requests an integration test proving block streaming emits Intro, a complete markdown table, and Outro as separate block replies without splitting the table.

## Best Possible Solution

Land one table-safe streaming fix that adds protected markdown table spans and the requested subscribeEmbeddedPiSession Intro/table/Outro integration coverage, then close this issue through the merged PR.

## Reproduction Assessment

Yes. The coverage gap is reproducible by inspecting the current integration test file, and the runtime gap is traceable through subscribeEmbeddedPiSession to EmbeddedBlockChunker, whose safe-break logic only checks fence spans; I did not run tests because this review is read-only.

## Solution Assessment

Yes. The requested integration test is the narrowest maintainable guardrail for the production block-reply pipeline, but it should land with or after the table-aware chunking fix rather than as a standalone failing test.
