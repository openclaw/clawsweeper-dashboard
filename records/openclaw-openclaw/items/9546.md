The issue requests a WebChat option to disable, replace, or customize the New messages notification banner.

## Best Possible Solution

Add a maintainer-chosen WebChat new-message indicator mode separate from chatAutoScroll, preserve a usable manual catch-up path, and update focused UI/config tests plus public WebChat docs.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main confirms the New messages indicator still exists and no disable/customize setting is present.

## Solution Assessment

Unclear until maintainers choose the UX and config surface. A first-class persisted indicator mode is more maintainable than CSS hiding, but the design must preserve an obvious way to catch up when automatic scrolling is off.
