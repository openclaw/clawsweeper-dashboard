## Summary

Keep open; current main added an adjacent persisted auto-scroll selector, but it still intentionally preserves the WebChat New messages button as the manual catch-up affordance and has no dedicated setting to disable, dismiss, customize, or replace that banner. A maintainer comment after https://github.com/openclaw/openclaw/pull/81629 explicitly treats the banner preference as a separate product decision.

## What This Changes

The issue requests a WebChat setting to disable, customize, or replace the New messages notification banner.

## Best Possible Solution

Add a maintainer-chosen WebChat new-message indicator mode, wire it through scroll scheduling and chat rendering, and cover it with focused UI/settings/config docs and tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high confidence that the New messages indicator still exists and no dedicated disable/customize setting is present.

## Solution Assessment

Unclear until maintainers choose the UX/config surface. The current auto-scroll selector is a good adjacent fix, but a first-class persisted indicator mode would be narrower and more maintainable than hiding the button with CSS or treating auto-scroll modes as a full substitute.
