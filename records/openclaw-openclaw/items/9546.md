## Summary

Current main and v2026.6.5 still retain the New messages catch-up control separately from the shipped auto-scroll selector, and the discussion explicitly identifies its visibility and replacement behavior as an unresolved UX decision.

## What This Changes

Add a WebChat preference to disable, customize, or replace the New messages catch-up indicator.

## Best Possible Solution

Define a browser-local indicator presentation mode that preserves a discoverable manual catch-up action for every auto-scroll mode, then cover persistence, scroll behavior, rendering, accessibility, and localized UI copy.

## Reproduction Assessment

Yes. Current main has a direct source path from incoming content that is not auto-followed to `chatNewMessagesBelow` and the rendered New messages catch-up control.

## Solution Assessment

No to an unconditional disable flag or new gateway config as proposed; the narrower maintainable solution is a persisted browser-local presentation mode that retains manual catch-up behavior.
