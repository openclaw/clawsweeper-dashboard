## Summary

Current main and v2026.6.5 include the adjacent auto-scroll selector, but the New messages catch-up button is still driven directly by scroll state with no disable, dismiss, customization, or replacement preference. A repository member explicitly kept this issue open after the adjacent feature landed because the remaining behavior requires a separate UX decision.

## What This Changes

Add a WebChat preference to disable, dismiss, customize, or replace the New messages indicator.

## Best Possible Solution

Add a persisted browser-local indicator mode after maintainers choose whether hidden mode retains another visible catch-up affordance; avoid a gateway config key unless a cross-client policy requirement is established.

## Reproduction Assessment

Yes at source level: current main deterministically sets and renders the New messages indicator whenever automatic scrolling does not follow new content. No live browser run is needed to establish that the requested preference is absent.

## Solution Assessment

Unclear: a browser-local persisted UI setting is the narrowest likely surface, but maintainers must decide hidden versus dismissible behavior and how users catch up to unseen output.
