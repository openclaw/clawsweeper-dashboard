## Summary

Keep open: current main still has no WebChat setting or gateway config to disable, replace, or customize the `New messages` banner, and the remaining choice is a product/UX decision rather than a narrow bug fix.

## What This Changes

Add a WebChat option to disable, auto-scroll instead of, or otherwise customize the `New messages` notification banner.

## Best Possible Solution

Implement a WebChat new-message indicator mode with a default that preserves current behavior, then wire it through scroll scheduling, render gating, persisted/configured settings, tests, and public docs.

## Reproduction Assessment

Yes for the reported current behavior: source inspection and the focused scroll test show `scheduleChatScroll` flips `chatNewMessagesBelow`, and the render path turns that state into the `New messages` button with no suppressing setting. This is a feature request rather than a broken existing contract.

## Solution Assessment

Unclear. The requested direction is valid, but the best maintainable fix depends on maintainer choice of setting surface and exact semantics for hide versus auto-scroll versus dismissible banner behavior.
