## Summary

Keep open. Current main and the latest release have the WebChat auto-scroll selector, but the New messages indicator is still rendered from scroll state with no separate hide, dismiss, or customization preference.

## What This Changes

The issue requests a WebChat setting to disable, customize, or replace the New messages notification banner.

## Best Possible Solution

Add a maintainer-approved browser-local WebChat new-message indicator mode, preserve an accessible jump-to-latest affordance, and avoid reintroducing retired gateway.webchat config unless maintainers explicitly choose a config policy surface.

## Reproduction Assessment

Yes, at source level. With chatAutoScroll set to Off or the user scrolled away from bottom, scheduleChatScroll sets chatNewMessagesBelow and renderChat shows New messages; focused tests assert that behavior.

## Solution Assessment

No for the proposed openclaw.json gateway.webchat shape because that config is retired. A browser-local persisted setting is the narrower implementation direction, but maintainers first need to choose the indicator and catch-up semantics.
