## Summary

Keep open. Current main and the latest release include the related WebChat auto-scroll mode selector, but the New messages indicator is still raised from scroll state and rendered as its own catch-up button with no separate hide, dismiss, or customization preference.

## What This Changes

Request to add a WebChat setting or behavior that disables, customizes, dismisses, or replaces the New messages notification banner.

## Best Possible Solution

Add a maintainer-approved browser-local WebChat new-message indicator mode, or another accessible catch-up design, while avoiding retired `gateway.webchat` config unless maintainers explicitly choose that policy surface.

## Reproduction Assessment

Yes, at source level. Current source and focused tests show `scheduleChatScroll` sets `chatNewMessagesBelow` when the view does not auto-scroll, including `chatAutoScroll: "off"`, and `renderChat` shows the New messages button from that flag.

## Solution Assessment

No final solution is selected yet. The landed auto-scroll selector is useful but intentionally preserves the banner, and the issue's proposed `openclaw.json` config shape is not the best fit while WebChat config is retired.
