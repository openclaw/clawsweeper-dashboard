## Summary

Keep open. Current main and v2026.5.22 have the adjacent WebChat auto-scroll selector, but the New messages indicator is still driven by scroll state and still has no disable, dismiss, customization, or banner-mode preference; the latest member comment also explicitly kept this issue open as a separate product decision.

## What This Changes

The issue requests a WebChat setting to disable, customize, or replace the New messages banner that appears when new content arrives below the current scroll position.

## Best Possible Solution

Add a maintainer-approved persisted WebChat new-message indicator mode, wire it through scroll scheduling and chat rendering, and document the chosen UI or gateway config surface with focused tests.

## Reproduction Assessment

Yes, at source level. Current main still sets chatNewMessagesBelow and renders the New messages button without any banner preference, and the focused auto-scroll-off test confirms the indicator remains active when automatic scrolling is disabled.

## Solution Assessment

Unclear until maintainers choose the UX/config surface and semantics. A dedicated indicator mode is plausible, while the merged auto-scroll selector intentionally does not solve this request.
