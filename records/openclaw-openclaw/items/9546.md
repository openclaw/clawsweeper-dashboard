## Summary

Current main now provides a persisted auto-scroll selector, but it intentionally preserves the “New messages” catch-up button and still has no preference to hide, dismiss, or restyle that indicator; maintainer discussion also explicitly keeps this separate UX decision open.

## What This Changes

Add a WebChat preference to disable, replace, or make the “New messages” indicator less intrusive.

## Best Possible Solution

Add a browser-local persisted indicator mode in the existing Control UI settings path, preserve a clear manual catch-up affordance when auto-scroll is off, and cover the interaction of indicator and auto-scroll modes in focused scroll and render tests.

## Reproduction Assessment

Yes for the current behavior: source and focused tests show the indicator appears when new content arrives while the view is not following the bottom. The missing preference is directly verifiable on current main.

## Solution Assessment

No, the suggested `openclaw.json` surface is not yet shown to be the best fit; the existing auto-scroll preference is browser-local, so the narrowest consistent solution is likely a persisted Control UI setting after maintainers choose the desired catch-up UX.
