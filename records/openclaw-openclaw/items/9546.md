## Summary

Keep open: current main still shows the WebChat `New messages` indicator directly from scroll state, with no persisted UI setting or `gateway.webchat` config to disable or replace it. This is a valid feature request that needs a maintainer UX/config-surface decision before a narrow implementation is safe.

## What This Changes

Add a WebChat preference or configuration mode to disable, replace, or customize the `New messages` banner.

## Best Possible Solution

Implement a default-preserving WebChat new-message indicator mode, wire it through scroll scheduling and chat rendering, expose it through the chosen settings/config surface, and update focused tests plus WebChat docs.

## Reproduction Assessment

Yes for the current behavior by source inspection: `scheduleChatScroll` sets `chatNewMessagesBelow`, `renderApp` passes that state into `renderChat`, and `renderChat` shows the `New messages` button. This is a feature request rather than a broken existing contract.

## Solution Assessment

Unclear. The requested direction is reasonable, but the best maintainable fix depends on maintainer choice of setting surface and exact hide versus autoscroll versus dismissible-banner semantics.
