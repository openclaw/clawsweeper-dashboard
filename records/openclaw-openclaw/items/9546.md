## Summary

Current main still has no preference to hide or customize the New messages control. The merged auto-scroll selector deliberately preserves it as a catch-up affordance, leaving a distinct product decision about indicator visibility and replacement access.

## What This Changes

Add a WebChat preference to disable, dismiss, customize, or replace the New messages indicator.

## Best Possible Solution

Add a persisted browser-local indicator mode only if maintainers approve the UX, preserve an accessible jump-to-latest action, and avoid adding a gateway configuration key.

## Reproduction Assessment

Yes, at source level. Current code and focused tests show that unseen output sets chatNewMessagesBelow and renders the New messages control, with no setting that suppresses it.

## Solution Assessment

No for the proposed openclaw.json shape. A browser-local persisted setting is the narrowest maintainable direction, but maintainers must first choose hide-only, dismissible, or alternative catch-up semantics.
