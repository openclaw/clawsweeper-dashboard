## Summary

Keep open: current main still has no WebChat setting or gateway configuration to disable, replace, or customize the “New messages” banner. The current scroll path still raises `chatNewMessagesBelow`, the render path still turns that into the banner, and docs/config/tests still describe only the existing behavior.

## What This Changes

Add a WebChat option to disable, auto-scroll instead of, or make less intrusive the “New messages” banner.

## Best Possible Solution

Implement a WebChat new-message indicator mode with a default that preserves current behavior, feed that mode into scroll scheduling and render gating, and update focused scroll/render tests plus the public WebChat/config documentation before closing this issue.

## Reproduction Assessment

Yes. The current source and focused scroll test reproduce the behavior by setting a scrolled-up chat host and calling `scheduleChatScroll`, which flips `chatNewMessagesBelow`; the render path then displays the `New messages` button with no suppressing setting.

## Solution Assessment

Unclear. The feature request is valid, but the best maintainable fix depends on maintainer UX/product choice between a browser-local preference and a strict gateway config option, and between hide, auto-scroll, or dismissible banner semantics.
