## Summary

Current main implements persisted WebChat auto-scroll modes but intentionally retains the New messages control whenever incoming output is not followed. This issue remains necessary because no independent preference gates that indicator, and the discussion correctly identifies its removal or redesign as a separate UX decision.

## What This Changes

Add a user preference to disable, customize, or replace WebChat’s New messages indicator.

## Best Possible Solution

Add a browser-local persisted indicator mode that can reduce or hide the banner while preserving an accessible jump-to-latest affordance; avoid a gateway-wide configuration key unless a cross-client policy need is demonstrated.

## Reproduction Assessment

Yes. Current source and focused tests provide a high-confidence path: incoming content that is not auto-followed sets the state flag and renders the New messages control.

## Solution Assessment

No to the proposed gateway configuration as the default solution. The existing browser-local `UiSettings` boundary is narrower and better aligned, but maintainers must first define the catch-up behavior when the indicator is hidden.
