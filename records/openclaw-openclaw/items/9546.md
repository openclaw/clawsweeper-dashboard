## Summary

Keep open. Current main and v2026.6.10 still render the New messages button from scroll state, while the landed auto-scroll selector and backscroll fix intentionally preserve that catch-up affordance; the remaining request is a separate WebChat indicator preference and product decision.

## What This Changes

The issue requests a WebChat option or setting to disable, replace, or customize the New messages notification banner.

## Best Possible Solution

Add a maintainer-approved browser-local WebChat new-message indicator mode that preserves an accessible jump-to-latest affordance and avoids reviving retired WebChat config unless maintainers explicitly choose that surface.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.10 set chatNewMessagesBelow when chat does not auto-stick, render the New messages button from that state, and focused tests assert that auto-scroll Off still shows the catch-up indicator.

## Solution Assessment

No for the proposed openclaw.json WebChat shape because that config surface is retired. A browser-local persisted indicator setting is the narrower likely direction, but maintainers first need to choose the indicator and catch-up semantics.
