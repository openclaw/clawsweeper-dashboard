## Summary

Keep this open: current main still has the WebChat new-message indicator hard-wired to scroll state, with no persisted UI setting or gateway.webchat configuration to disable or change it.

## What This Changes

Add a WebChat option to disable, auto-scroll instead of, or otherwise customize the New messages banner.

## Best Possible Solution

Add a documented WebChat new-message indicator mode in the chosen settings/config surface, then gate scheduleChatScroll and renderChat through that mode with focused scroll/render coverage.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request for a new preference/config mode. Source inspection does confirm the requested option is absent on current main.

## Solution Assessment

Unclear until maintainers choose the product surface and exact mode semantics. The narrow maintainable path is to add one explicit WebChat indicator mode and wire it through existing scroll state, rendering, settings/config, tests, and docs.
