## Summary

Keep open. Current main and v2026.6.5 have the adjacent WebChat auto-scroll selector, but the New messages button is still driven directly by scroll state with no disable, dismiss, customization, or indicator-mode setting; the latest member comment also explicitly kept this issue open as a separate product decision.

## What This Changes

The issue requests a WebChat setting or UI control to disable, customize, or replace the New messages notification banner.

## Best Possible Solution

Keep the landed auto-scroll selector, then add a separate persisted WebChat indicator preference only after maintainers choose the UX surface and banner semantics.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: when auto-scroll does not stick, current main still sets chatNewMessagesBelow and renders the New messages button from that state. I did not run a browser repro because this review is read-only and the missing behavior is a feature preference rather than a failing documented contract.

## Solution Assessment

No, the already-merged auto-scroll selector is only an adjacent fix because it intentionally keeps the New messages manual catch-up affordance. The best solution is a dedicated indicator/banner preference after maintainer UX direction.
