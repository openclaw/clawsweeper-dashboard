## Summary

Keep open. Current main and v2026.6.8 still render the New messages button from scroll state, while the landed auto-scroll selector intentionally preserves that manual catch-up affordance; the remaining request is a separate WebChat indicator preference decision.

## What This Changes

Request to add a WebChat option or setting to disable, customize, or replace the New messages notification banner.

## Best Possible Solution

Add a maintainer-approved browser-local WebChat new-message indicator mode that preserves an accessible jump-to-latest affordance and avoids reviving retired WebChat config unless maintainers explicitly choose that surface.

## Reproduction Assessment

Yes, at source level. With chatAutoScroll set to Off or the user scrolled away from bottom, scheduleChatScroll sets chatNewMessagesBelow and renderChat shows New messages; focused tests assert that behavior.

## Solution Assessment

No for the proposed openclaw.json WebChat shape because that config surface is retired. A browser-local persisted indicator setting is the narrower likely direction, but maintainers first need to choose the indicator and catch-up semantics.
