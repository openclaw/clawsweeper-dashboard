## Summary

Keep open. Current main and the latest release have the WebChat auto-scroll mode selector, but the New messages indicator is still rendered from scroll state with no separate hide, dismiss, or customization preference.

## What This Changes

Request a WebChat setting or behavior change to disable, customize, or replace the New messages indicator banner.

## Best Possible Solution

Add a maintainer-approved browser-local WebChat new-message indicator mode that preserves an accessible jump-to-latest affordance and avoids resurrecting retired gateway.webchat config unless maintainers explicitly choose that policy surface.

## Reproduction Assessment

Yes, at source level. With chatAutoScroll set to Off or the user scrolled away from bottom, scheduleChatScroll sets chatNewMessagesBelow and renderChat shows New messages; focused tests assert that behavior.

## Solution Assessment

No for the proposed openclaw.json gateway.webchat shape because that config is retired. A browser-local persisted setting is the narrower implementation direction, but maintainers first need to choose the indicator and catch-up semantics.
