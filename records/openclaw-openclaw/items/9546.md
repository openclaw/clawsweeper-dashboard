## Summary

Keep open; current main has the adjacent WebChat auto-scroll selector, but the New messages indicator is still a separate, ungated manual catch-up affordance and the latest maintainer comment explicitly treats banner disabling/customization as a separate product decision.

## What This Changes

The issue requests a WebChat option to disable, replace, or make the New messages notification banner less intrusive.

## Best Possible Solution

Add a maintainer-chosen WebChat new-message indicator mode separate from chatAutoScroll, preserve a usable manual catch-up path, and cover the selected UI/config surface in focused tests and public WebChat docs.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main confirms the New messages indicator still exists and no disable/customize setting is present.

## Solution Assessment

Unclear until maintainers choose the UX and config surface. A first-class persisted indicator mode is more maintainable than CSS hiding, but it must preserve an obvious way to catch up when automatic scrolling is off.
