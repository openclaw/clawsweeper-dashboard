## Summary

Keep open. Current main and the latest release still preserve the New messages button as the manual catch-up affordance, and there is no separate WebChat setting to hide, dismiss, or customize that indicator.

## What This Changes

The issue requests a WebChat setting or UX mode to disable, replace, or customize the New messages banner.

## Best Possible Solution

Add a maintainer-approved browser-local WebChat new-message indicator mode that preserves an accessible jump-to-latest affordance and avoids reviving retired gateway.webchat config unless maintainers explicitly choose that policy surface.

## Reproduction Assessment

Yes, at source level. With chatAutoScroll set to Off or the user scrolled away from bottom, scheduleChatScroll sets chatNewMessagesBelow and renderChat shows New messages; focused tests assert that behavior.

## Solution Assessment

No for the proposed openclaw.json gateway.webchat shape because that config is retired. A browser-local persisted setting is the narrower likely direction, but maintainers first need to choose the indicator and catch-up semantics.
