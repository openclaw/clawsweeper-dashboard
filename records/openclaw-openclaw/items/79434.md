## Summary

Keep open: the patch is a useful first slice, but it only updates Telegram account merging while the runtime group authorization path still resolves raw account groups and can keep producing the reported `not-allowed` drop. The PR body now has merge-helper output, but not proof that a Telegram update is delivered through the gateway after the fix.

## What This Changes

The PR changes Telegram account config merging to treat an explicit empty account `groups` object as unset for single-account root group fallback and adds account-config regression tests.

## Best Possible Solution

Have Telegram group resolution use one consistent helper that treats empty account-level groups as absent only for single-account inheritance, while preserving multi-account isolation and covering runtime authorization with a regression test.

## Reproduction Assessment

Yes, source-reproducible: the reported config reaches both `mergeTelegramAccountConfig` and `resolveChannelGroupPolicy` with `account.groups` set to `{}`, and both current-main paths skip root `channels.telegram.groups`. I did not run a live Telegram gateway scenario in this read-only review.

## Solution Assessment

No. Updating the merge helper is necessary, but the runtime policy lookup still reads raw account groups, so the narrowest maintainable fix should align that lookup and add a runtime authorization regression.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Align the raw group-policy lookup too:** `extensions/telegram/src/account-config.ts:76`
  - body: This fixes the merged account config, but the group authorization path that logs the reported `not-allowed` still calls `resolveChannelGroupPolicy` on the raw config. With `accounts.default.groups: {}`, that helper keeps the empty account map, skips root `channels.telegram.groups`, and returns no matching group config, so the reported update can still be rejected. Please make the policy lookup share the same empty-as-absent single-account semantics or add a runtime regression that proves the group update is delivered.
