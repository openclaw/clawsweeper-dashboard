## Summary

Keep open: the PR fixes the Telegram merge helper, but the runtime group-policy path still resolves the raw empty account groups map, so the reported inbound group drop can remain; it also lacks a changelog entry and sufficient runtime proof.

## What This Changes

The branch treats empty Telegram account-level groups as root-group fallback in the merge helper, adds merge-helper tests, and includes unrelated memory-core test type casts.

## Best Possible Solution

Align the empty-as-absent single-account inheritance rule across Telegram merge and runtime group-policy resolution, preserve multi-account isolation, add regression coverage and changelog coverage, then merge after redacted runtime proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `accounts.default.groups: {}` is treated as defined by both the merge helper and `resolveChannelGroups`, and Telegram inbound authorization can still reject the configured root group before delivery.

## Solution Assessment

No. Updating `mergeTelegramAccountConfig` is necessary but incomplete because inbound authorization calls `resolveChannelGroupPolicy` on the raw config; the maintainable fix should centralize or share the same empty-as-absent rule there.
