## Summary

Keep open: the branch fixes the Telegram merge helper but leaves the raw runtime group-policy resolver unchanged, so the reported inbound group rejection can still occur.

## What This Changes

The branch changes Telegram account-config merging so an explicit empty account-level `groups` map falls back to root groups for single-account bots and adds two merge-helper tests.

## Best Possible Solution

Apply the empty-as-absent single-account inheritance rule consistently in Telegram account merging and shared runtime group-policy resolution, preserve multi-account isolation, add regression and changelog coverage, then merge after redacted gateway-path proof.

## Reproduction Assessment

Yes, by source inspection. Current main and the PR-diff scope show that `accounts.default.groups: {}` still suppresses root groups in the runtime resolver used by Telegram inbound authorization, although I did not run a live Telegram gateway repro in this read-only review.

## Solution Assessment

No. Updating `mergeTelegramAccountConfig` is necessary but incomplete because the inbound authorization path calls `resolveChannelGroupPolicy` on raw config; the same fallback rule should be applied there with regression coverage.
