## Summary

Keep open: the PR addresses the merge helper, but current evidence shows the Telegram inbound runtime path still resolves group policy from the raw config and can still drop the root group allowlist.

## What This Changes

The branch changes Telegram account config merging so an explicit empty account-level `groups` object falls back to root groups in single-account setups and adds merge-helper tests.

## Best Possible Solution

Align the empty-as-absent single-account inheritance rule across Telegram merge and runtime group-policy resolution, preserve multi-account isolation, add regression coverage and changelog coverage, then merge after redacted runtime proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: root `channels.telegram.groups` plus `accounts.default.groups: {}` is still treated as an empty account map by the runtime resolver, and Telegram inbound access can reject the configured group.

## Solution Assessment

No. Updating `mergeTelegramAccountConfig` is necessary but incomplete because inbound authorization calls `resolveChannelGroupPolicy` on the raw config; the narrow maintainable fix should share the same empty-as-absent rule there.

## Review Findings
