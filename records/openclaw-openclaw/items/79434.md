The PR treats empty Telegram account-level groups as absent in `mergeTelegramAccountConfig`, adds merge-helper tests, and pins `fast-uri` through root pnpm overrides and the lockfile.

## Best Possible Solution

Apply the empty-as-absent rule consistently for Telegram single-account group inheritance across merged config and raw policy lookup, preserve multi-account isolation, add focused regression coverage and changelog, and split dependency pinning into a separate approved security/dependency change.
