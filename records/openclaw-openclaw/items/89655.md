## What This Changes

The issue reports that macOS runtimes with `NODE_USE_SYSTEM_CA=1` can break OpenAI Codex OAuth/keychain auth reads and fresh launch paths with `SecItemCopyMatching failed -50`.

## Best Possible Solution

Reproduce the failure on macOS, then fix the narrow auth/startup boundary so Codex keychain/status/runtime launch reads do not inherit the problematic CA mode while preserving launchd TLS reliability.

No high-confidence live reproduction was run in this review. The reporter provides concrete macOS commands, and current source still has the relevant service CA default plus keychain-capable Codex auth paths, so the source path remains credible.

## Solution Assessment
