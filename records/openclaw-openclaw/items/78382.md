Keep open: current main still lacks a microphone dictation control, while the current docs/changelog and recent history show server-side chat-draft dictation was added and then removed by a later latency PR. ## What This Changes

The issue asks to restore microphone dictation that writes recognized speech into the chat input for review before sending, instead of replacing that flow with Talk mode.

## Best Possible Solution

Restore the previously reviewed server-side chat-draft dictation path beside Talk, or explicitly retire that documented promise and update docs/tests/changelog so the intended voice UX is consistent.

## Solution Assessment

Unclear: removing Talk wholesale is not the best fix, and the current direct Web Speech PR is not the safest path without privacy/product approval. The narrow maintainable direction is to restore the Gateway-mediated dictation flow that writes transcript text into the draft before send.
