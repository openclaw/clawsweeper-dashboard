## Summary

Keep open: current main still has the one-shot startup-only skip path, while read and timer maintenance callers can still recompute without the overflow-deferral exemption; https://github.com/openclaw/openclaw/pull/94022 is the open candidate fix and should be reviewed before this issue closes.

## What This Changes

The issue reports that startup overflow cron catch-up deferrals can be advanced to the next natural schedule by later maintenance recomputes, dropping the deferred missed run.

## Best Possible Solution

Review and land a focused state-level pending-deferral fix, or close the linked PR and replace it with an equivalent narrow fix, then close this issue after the merged code is on main.

## Reproduction Assessment

Yes. The issue gives an in-process CronService path, and current main still shows read/timer maintenance callers reaching `recomputeNextRunsForMaintenance(state)` without the deferred catch-up skip set.

## Solution Assessment

Yes. A single `CronServiceState` pending-deferral marker is the narrow maintainable fix shape because it covers every maintenance recompute caller without threading one-off skip sets through each read and timer path.
