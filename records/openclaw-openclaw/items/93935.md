## Summary

Keep open: current main still protects startup overflow catch-up deferrals only through the startup-local skip set, while read/status and timer maintenance recomputes can still run without that exemption. The issue is also paired with open closing PRs, so it should remain open until one candidate lands or is replaced.

## What This Changes

The issue reports that startup cron overflow catch-up deferrals can be advanced to the next natural cron slot by later maintenance recomputes, dropping the missed run.

## Best Possible Solution

Review and land one canonical fix PR that moves pending startup catch-up deferral IDs into shared service state, covers read/status plus timer maintenance recomputes, and then closes or supersedes duplicate fix attempts.

## Reproduction Assessment

Yes. Current source still has `ensureLoadedForRead` and timer maintenance paths invoking `recomputeNextRunsForMaintenance` without the startup deferral skip set, matching the reported source-level reproduction.

## Solution Assessment

Yes. The state-level deferral exemption is the narrow maintainable fix because `runMissedJobs` creates the deferred IDs and every later maintenance recompute shares the same future-slot repair invariant.
