## Summary

Keep open: current main still lacks durable pending catch-up deferral tracking, and the active candidate fix is https://github.com/openclaw/openclaw/pull/94022 rather than a merged implementation.

## What This Changes

Preserve cron startup overflow catch-up deferrals across read RPC and timer maintenance recomputes until the staggered slot fires.

## Best Possible Solution

Land the active canonical cron fix, then close this issue and duplicate same-root-cause PR attempts after the merged commit is on main.

## Reproduction Assessment

Yes. The source path is clear: startup deferral IDs are only threaded into `start()`'s immediate recompute, while read and timer maintenance callers still run without that exemption.

## Solution Assessment

Yes. A state-owned pending deferral set shared by all maintenance recompute callers is the narrow maintainable fix; threading new one-off skip sets through each caller would keep the same fragile ownership split.
