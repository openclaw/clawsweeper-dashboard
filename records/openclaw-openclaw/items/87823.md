## Summary

Keep open: the PR targets a real multi-agent Usage page regression, but the proposed protocol shape should not merge as-is because it overloads a valid `agentId` value and still lacks real behavior proof.

## What This Changes

## Best Possible Solution

Keep the default scoped behavior, but expose all-agent usage through a non-colliding protocol contract with focused tests, docs/schema follow-through, and redacted real Control UI proof.

## Solution Assessment

No: explicit all-agent opt-in is the right direction, but overloading `agentId: "all"` is not the safest protocol shape because it collides with valid agent ids and lacks real behavior proof.
