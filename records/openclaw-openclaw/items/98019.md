## Summary

Keep open: current main and v2026.6.10 still hard-code the Telegram chat-window selection at 10 recent messages with a reply-target window of 2, and no supported Telegram config key exposes those values. Related work fixed duplicate persistent-DM context or broader direct-session recency, but the configurable recentLimit request remains a product/config-surface decision.

## What This Changes

The issue requests an openclaw.json setting for Telegram conversation context limits such as recentLimit and replyTargetWindowSize.

## Best Possible Solution

Have maintainers decide the Telegram plugin-owned config contract first; if accepted, add a narrow schema/type/docs/test update that keeps the default at 10 and applies only to the selected chat-window context path.

## Reproduction Assessment

Yes for the current limitation: source inspection shows the Telegram chat-window builder is called with recentLimit 10 and replyTargetWindowSize 2, and schema search found no matching config key. This is source reproduction of a missing feature, not a runtime failure reproduction.

## Solution Assessment

Unclear. A config knob is plausible, but because OpenClaw already has several Telegram history/context controls, the best fix depends on maintainer choice of public config shape rather than an automatic bug-fix patch.
