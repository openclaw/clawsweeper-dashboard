## Summary

Keep open: current main and v2026.6.10 still hardcode Telegram selected conversation context at `recentLimit: 10` and `replyTargetWindowSize: 2`, while existing `historyLimit` and `dmHistoryLimit` knobs cover different history paths. The request is valid as a feature/config-surface proposal, but it needs maintainer product/API direction before implementation.

## What This Changes

Request to make Telegram's selected conversation-context window size, and possibly reply-target window size, configurable through `openclaw.json` instead of fixed 10/2 values.

## Best Possible Solution

Add a maintainer-approved Telegram-plugin config contract for selected conversation context, with bounded defaults preserving 10/2 behavior, account-aware schema/docs/tests, and no reuse of `historyLimit` for a different context window.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection does confirm current main and v2026.6.10 hardcode the selected Telegram context window and lack the requested config keys.

## Solution Assessment

Unclear until maintainers choose the config contract. A Telegram-plugin-owned bounded `conversationContext` shape is cleaner than overloading `historyLimit`, because current docs and code use `historyLimit` for a separate group/DM history path.
