## Summary

Keep open. Current main still lacks Discord MESSAGE_UPDATE listener wiring, edit-specific config/docs, and edit-aware replay semantics; the broader open delete-to-cancel tracker does not supersede this edit-only request.

## What This Changes

Request opt-in Discord MESSAGE_UPDATE handling so edited Discord user messages can trigger a new agent turn with debounce and edit-aware behavior.

## Best Possible Solution

Implement this in the bundled Discord plugin with typed MESSAGE_UPDATE dispatch, content-change filtering, opt-in config/docs, per-message debounce, edit-aware replay handling, changelog, and focused monitor tests.

## Reproduction Assessment

Yes. Source inspection on current main is high-confidence: MESSAGE_UPDATE has no registered Discord monitor listener, and same-message replay currently has no edit-aware path.

## Solution Assessment

Yes on the owner boundary, but final policy is still a maintainer decision. The Discord plugin is the right place to solve it, while defaults, partial update handling, debounce, and abort/replay semantics should be decided before implementation.
