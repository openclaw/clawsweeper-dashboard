## Summary

Current main still ignores Discord message updates, while the open PR https://github.com/openclaw/openclaw/pull/92091 now owns the implementation. Keep this issue open until that PR’s combined edit/delete policy and live transport proof are resolved.

## What This Changes

Add Discord message-update handling so edited user messages can trigger agent processing with edit-aware debounce and session semantics.

## Best Possible Solution

Land a live-proven Discord-plugin implementation that sends genuine content edits through normal inbound gating, associates revisions with the exact source message, and applies a maintainer-approved supersede policy; split delete-to-cancel if that separate policy delays edit handling.

## Reproduction Assessment

Yes. Current main’s Discord inbound listener and startup registration are create-only, so message edits cannot reach the normal handler; source inspection provides a high-confidence reproduction path.

## Solution Assessment

Unclear as a final design. Reusing normal inbound preflight is the correct owner boundary, but configuration defaults, run abortion, revision replay keys, and the combined delete behavior require maintainer approval plus live Discord proof.
