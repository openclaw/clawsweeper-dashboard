## What This Changes

The issue requests opt-in Discord MESSAGE_UPDATE handling so edited user messages can be debounced and reprocessed as updated inbound agent turns.

## Best Possible Solution

Implement this in the bundled Discord plugin after maintainer policy confirmation: typed message-update dispatch, content-change filtering, opt-in config/docs, per-message debounce, edit-aware replay/session handling, mention-gating coverage, and focused monitor tests.

## Reproduction Assessment

Yes, at source level. Current main has exact event-type listener dispatch, no Discord MessageUpdate listener registration, no requested edit config surface, and a same-message-id replay guard that would treat naive edit handling as a duplicate.

## Solution Assessment

Unclear as an automatic repair. The bundled Discord plugin is the right owner boundary, but maintainers still need to choose edit policy, default/config shape, debounce and rate limits, partial update hydration, and replay or abort semantics.
