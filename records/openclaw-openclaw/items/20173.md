## Summary

Keep open. Current main still has no Discord `MESSAGE_UPDATE` listener or monitor registration path, no edit-specific config/docs surface, and the same-message replay guard would treat a naive edited-message reprocess as a duplicate.

## What This Changes

Add optional Discord `MESSAGE_UPDATE` handling so edited user messages can be debounced and reprocessed as new agent turns.

## Best Possible Solution

Implement this in the bundled Discord plugin with typed `MESSAGE_UPDATE` dispatch, content-change filtering, opt-in config/docs, per-message debounce, edit-aware replay handling, and focused monitor tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a Discord `MESSAGE_UPDATE` dispatch has no registered listener and same-message edits have no edit-aware replay key.

## Solution Assessment

Unclear as an automatic repair. The Discord plugin is the right owner boundary, but config defaults, debounce/rate-limit behavior, partial update hydration, and abort/replay semantics need maintainer direction first.
