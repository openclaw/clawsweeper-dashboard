## Summary

Keep open: this is a real config bug on current main, but the PR still needs contributor-supplied real behavior proof and has blocking implementation/baseline issues before merge.

## What This Changes

The PR makes `agents.defaults.cliBackends.<backendId>.command` optional in config schema/types, adds a resolved backend type, narrows normal CLI spawn command usage, and adds a modelAliases-only regression test.

## Best Possible Solution

Accept partial CLI backend override input while preserving `command` as a resolved non-empty string through prepared and live execution paths, refresh the config docs baseline hash, and require redacted real-run proof before merge.

## Reproduction Assessment

Yes, source-reproducible. Current main requires `command` in the schema/type while the resolver merges registered backends with overrides and validates the resolved command after merge, so a `modelAliases`-only override is blocked before inheritance.

## Solution Assessment

No. Making input `command` optional is the right direction, but the branch should also carry the resolved command contract through prepared/live execution paths and update the config docs baseline.
