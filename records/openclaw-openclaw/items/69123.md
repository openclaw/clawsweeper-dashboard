## Summary

Keep open: current main still has the reported partial `cliBackends` override bug, and this PR targets the right config boundary, but the branch is not mergeable as-is because it drops the resolved-command guarantee outside the guarded normal spawn path, misses the config docs baseline update, and still lacks contributor-supplied real behavior proof.

## What This Changes

The branch makes `cliBackends.<backendId>.command` optional in config schema/types, adds a resolved backend type plus spawn guard, and adds a regression test for a `modelAliases`-only override.

## Best Possible Solution

Accept partial CLI backend override input while preserving `command` as a resolved non-empty string through every execution path, refresh the config-doc baseline, and require redacted real-run proof before merge.

## Reproduction Assessment

Yes, source-reproducible. Current main requires `command` in the schema/type while the resolver already merges registered backends with overrides and validates the resolved command after merge, so a `modelAliases`-only override is blocked before inheritance.

## Solution Assessment

No. Making input `command` optional is the right direction, but the branch should also carry the resolved command contract through prepared/live execution paths and update the config docs baseline.
