## What This Changes

The PR relaxes CLI backend config so `command` can be omitted in partial overrides, adds a resolved-command type/child-spawn guard, and adds a regression test for a `modelAliases`-only override.

## Best Possible Solution

Accept partial CLI backend overrides at the config input boundary while keeping resolved backends command-required everywhere execution can spawn, then refresh config baselines and show a real-run proof.

## Reproduction Assessment

Yes. Source inspection shows current main requires `command` in both the zod schema and config type while docs/runtime merge semantics support inherited backend defaults, so a `modelAliases`-only override is rejected before inheritance can run.

## Solution Assessment

No. Making `command` optional for input is the right direction, but this branch should rebase, preserve the resolved non-empty command invariant through prepared/live execution, refresh the config baseline, and add real behavior proof.
