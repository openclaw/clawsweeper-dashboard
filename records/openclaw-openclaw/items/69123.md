## Summary

Keep this PR open. It targets a real current-main bug and the core direction is right, but the branch is incomplete because it leaves schema-driven config consumers rejecting the same partial override and appears to make the live CLI backend path type-unsafe after `command` becomes optional.

## What This Changes

The PR makes `CliBackendConfig.command` and `CliBackendSchema.command` optional, adds a resolved CLI backend type, narrows the normal CLI spawn command, and adds a regression test for a `modelAliases`-only `claude-cli` override.

## Best Possible Solution

Finish this PR by treating CLI backend config input as partial while preserving a resolved `command: string` invariant across prepare, bundle MCP, normal spawn, and live-session execution paths; then regenerate config schema/docs baselines, keep the regression test, and add the changelog entry.

## Reproduction Assessment

Yes. A config object with `agents.defaults.cliBackends.claude-cli.modelAliases` and no `command` reproduces the current-main validation rejection because the source Zod schema still requires `command`; the PR's remaining defects are reproducible by generated-schema inspection and the static type flow after `command` becomes optional.

## Solution Assessment

No. The requested source schema/type relaxation is the right direction, but this patch is not yet the best complete fix because it leaves generated schema consumers stale and does not preserve the resolved command guarantee through the live prepared-backend path.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Carry the resolved command type through live sessions:** `src/config/types.agent-defaults.ts:96`
  - body: After `CliBackendConfig.command` becomes optional, `CliPreparedBackend.backend` still exposes a plain `CliBackendConfig`, and `runClaudeLiveSessionTurn` still builds `argv` from `context.preparedBackend.backend.command`. The new guard in `execute.ts` only narrows the normal child-spawn local variable, so the Claude live-session path is left with `string | undefined` for `argv[0]`. Use the resolved backend type through prepare/bundle MCP, or pass a narrowed command into the live-session path.
  - confidence: 0.9

Schema-driven config UI/inspect/validation flows would still reject the `modelAliases`-only override this PR is meant to allow.
  - confidence: 0.96
