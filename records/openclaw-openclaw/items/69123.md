## What This Changes

The PR makes `agents.defaults.cliBackends.<backendId>.command` optional in config input, adds a resolved backend command type, guards the child-spawn command use, and adds a modelAliases-only regression test.

## Best Possible Solution

Accept partial CLI backend override input while preserving `command` as a resolved non-empty string through prepared and live execution paths, refresh the config baseline hash, and require redacted real-run proof before merge.

## Reproduction Assessment

Yes. Current main requires `command` in both the zod schema and config type while the resolver merges user overrides over plugin defaults, so a modelAliases-only override is rejected before inheritance.

## Solution Assessment

No. Making `command` optional for input is the right direction, but the branch should rebase, carry the resolved command contract through prepared/live execution, refresh the config baseline, and add real behavior proof.
