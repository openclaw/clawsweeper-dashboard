## Summary

Keep this PR open because it targets a real source-reproducible CLI backend config bug, but the branch is conflicting, lacks external real behavior proof, and still has blocking type/baseline gaps before merge.

## What This Changes

The PR makes `agents.defaults.cliBackends.<backendId>.command` optional for partial CLI backend overrides, adds a resolved backend command type, guards child-spawn command use, and adds a modelAliases-only regression test.

## Best Possible Solution

Accept partial CLI backend override input, but preserve `command` as a resolved non-empty string through prepared and live execution paths, refresh the config baseline hash, rebase, and require redacted real-run proof before merge.

## Reproduction Assessment

Yes, from source inspection: current main requires `command` in both the zod schema and config type while the resolver can inherit a base command after merging overrides. I did not run a local repro because this review was required to keep the checkout read-only.

## Solution Assessment

No. Making `command` optional for config input is the right direction, but this PR should rebase, preserve the resolved command invariant through prepared/live execution, refresh the config baseline, and add real behavior proof.
