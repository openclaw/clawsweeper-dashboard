## Summary

Keep open: this is a focused security PR with sufficient proof and green CI, but the latest diff still leaves the bootstrap-token verifier reachable without protection when the normal auth limiter is absent or loopback-exempt. The author and reviewer discussion already confirmed that gap, so this needs secops/owner policy review rather than cleanup closure.

## What This Changes

The PR adds a bootstrap-token auth rate-limit scope, gates pre-auth bootstrap-token verification, adds gateway auth tests, and updates the changelog.

## Best Possible Solution

Use a dedicated bootstrap-token limiter before verifyDeviceBootstrapToken that is always constructed and loopback-non-exempt, or an explicitly approved equivalent, with coverage for no-config, default-loopback, browser-origin, and configured remote cases.

## Reproduction Assessment

Yes. Source inspection shows a signed device handshake with auth.bootstrapToken reaches verifyDeviceBootstrapToken before any bootstrap limiter on current main, and that verifier serializes under withLock; the PR body also supplies configured-limiter runtime proof and an in-process WebSocket test path.

## Solution Assessment

No. Mirroring the optional device-token limiter is a useful partial fix, but the mutex-backed bootstrap-token verifier needs an always-present preflight limiter or an explicitly approved security policy alternative.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use an always-on limiter for bootstrap-token verifies:** `src/gateway/server/ws-connection/auth-context.ts:167`
  - body: Because this gate only runs when `params.rateLimiter` is supplied, deployments without `gateway.auth.rateLimit` and loopback clients under the default `exemptLoopback:true` setting still enter `verifyDeviceBootstrapToken`. Use an always-created bootstrap-token limiter for this mutex-backed pre-auth path, or an explicitly approved equivalent, and cover no-config/default-loopback cases.
  - confidence: 0.9
