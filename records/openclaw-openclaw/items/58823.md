## Summary

Keep open: current main and the latest release still leave the shared `agent-scope` helper with agent-primary precedence ahead of the global subagent default, while the docs and linked issue support this PR's expected order. ## What This Changes

This PR reorders shared subagent model selection to prefer per-agent subagent config, then `agents.defaults.subagents.model`, then the agent primary model, with updated agent/cron tests and a local proof script.

## Best Possible Solution

If maintainers confirm the documented global-subagent-default contract, land this PR or a narrow equivalent for the shared helper and cron fallback path; if shipped agent-primary precedence is intended, update docs/tests and close the linked issue with that product decision.

## Solution Assessment

Yes, if maintainers accept the documented contract: moving the global default before the agent primary in the shared helper is the narrowest fix because the direct helper already has that order and cron uses the shared helper.
