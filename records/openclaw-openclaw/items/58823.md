## Summary

Keep this PR open: current main and the latest release still leave the shared agent-scope helper and isolated cron consumer using the old helper precedence, while this branch is the active narrow candidate for the linked issue. ## What This Changes

The PR changes shared and cron subagent model selection so per-agent subagent config wins first, `agents.defaults.subagents.model` wins second, and the agent primary model is only the fallback.

## Best Possible Solution

Keep one documented subagent model precedence contract across native spawn, shared helper, cron, session runtime persistence, and docs; if maintainers accept the documented global-default precedence, land this narrow fix before rebasing broader runtime work.

## Reproduction Assessment

Yes. Source inspection on current main shows the shared helper evaluates `agentConfig.model` before `agents.defaults.subagents.model`, and isolated cron calls that helper.

## Solution Assessment

Yes, if maintainers accept the documented compatibility tradeoff. Applying the same precedence to the shared helper and cron tests is the narrowest maintainable fix for the remaining gap; the broader runtime persistence PR should be sequenced around it, not treated as a superseding clean replacement yet.
