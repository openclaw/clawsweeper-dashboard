## Label Justifications

- `P2`: This is a normal-priority model-routing bugfix with bounded but real impact on subagent and cron provider/model selection.
- `merge-risk: 🚨 compatibility`: Existing configurations can observe a different winning default model after upgrade when both an agent primary and a global subagent default are configured.
- `merge-risk: 🚨 auth-provider`: Changing the selected model can also change provider, auth profile, runtime policy, and fallback behavior for subagent execution.

## Summary

Keep open: current main still has the shared agent-scope and isolated-cron ordering that lets an agent primary shadow the global subagent default, and this PR is the active narrow fix candidate, but the routing contract is compatibility-sensitive.

## What This Changes

The PR reorders shared subagent model selection so per-agent subagent config wins first, then `agents.defaults.subagents.model`, then the agent primary model, with agent/cron tests and a terminal proof script.

## Best Possible Solution

Land this narrow helper/cron precedence fix if maintainers accept the documented contract, and coordinate it with https://github.com/openclaw/openclaw/pull/72984 so runtime persistence and selection semantics converge.

## Reproduction Assessment

Yes, at source level. Configure an agent primary model plus a different `agents.defaults.subagents.model`; current `agent-scope` checks the agent primary before the global subagent default, and isolated cron consumes that helper.

## Solution Assessment

Yes for the remaining shared-helper and cron gap. The narrow reorder matches the already-corrected native helper and docs, but maintainers still need to accept the compatibility impact and sequence it with the broader runtime-persistence PR.
