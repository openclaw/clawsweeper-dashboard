## Summary

Keep this PR open: current main only partially implements the intended precedence, while the shared agent-scope helper and isolated cron path still let an agent primary model shadow `agents.defaults.subagents.model`. The patch is a narrow candidate fix, but it changes compatibility-sensitive model/provider precedence and overlaps broader subagent runtime/session model work, so maintainer acceptance is still needed before merge.

## What This Changes

The PR reorders shared subagent model selection so per-agent `subagents.model` wins first, `agents.defaults.subagents.model` wins over the agent primary model, and the agent primary remains the last fallback, with focused tests and a proof script.

## Best Possible Solution

Choose and document one canonical subagent model precedence contract, then either land this narrow shared-helper/cron fix if the documented global-default behavior is intended or close it with source/docs/tests updated to preserve the shipped agent-primary order.

## Reproduction Assessment

Yes. Source inspection on current main shows `resolveSubagentModelConfigSelectionResult` still evaluates `agentConfig.model` before `agents.defaults.subagents.model`, and isolated cron calls that helper.

## Solution Assessment

Yes for the narrow shared-helper/cron gap if maintainers accept the compatibility tradeoff. The broader runtime/session persistence problem should stay coordinated through https://github.com/openclaw/openclaw/pull/72984 rather than being folded into this PR.
