## Summary

Keep open: current main still has the shared agent-scope helper, and therefore isolated cron, selecting an agent primary before the global subagent default; this PR remains the active narrow candidate for that remaining gap, with no blocking patch defect found but a real compatibility decision before merge.

## What This Changes

The PR changes shared subagent model selection and related cron/test proof paths so `agents.defaults.subagents.model` wins over an agent primary when no per-agent subagent model is set.

## Best Possible Solution

Adopt one documented subagent model precedence across the shared helper, cron selection, native spawn planning, runtime persistence, and docs; if maintainers reject that precedence, preserve the shipped order and update the public docs and linked issue accordingly.

## Reproduction Assessment

Yes. Source inspection on current main shows the shared helper still evaluates the agent primary before the global subagent default, and isolated cron calls that helper.

## Solution Assessment

Yes, if maintainers accept the compatibility tradeoff. Moving the ordering in the shared helper is the narrowest remaining code fix because the direct native resolver is already fixed and cron now consumes the helper.
