## Summary

Keep this PR open. Current main already has the direct `model-selection.ts` precedence fixed, but the shared `agent-scope` helper still gives the agent primary model priority over `agents.defaults.subagents.model`, and cron/fallback paths consume that helper; the PR still contains meaningful unique work with no blocking code finding found.

## What This Changes

This PR changes subagent model resolution and tests so per-agent `subagents.model` wins first, global `agents.defaults.subagents.model` wins second, and the agent primary model is the fallback, including the shared agent-scope and cron paths.

## Best Possible Solution

Land this PR or a narrow equivalent as the canonical helper-precedence fix after a maintainer explicitly accepts the provider/auth upgrade behavior and decides how it relates to the broader runtime persistence work.

## Reproduction Assessment

Yes. Source inspection on current main shows `resolveSubagentModelConfigSelectionResult` selecting the agent model before `agents.defaults.subagents.model`, while docs and the direct resolver expect the global subagent default to win when configured.

## Solution Assessment

Yes for the narrow helper-precedence bug: moving `default-subagent` before `agent` in the shared helper aligns cron/fallback consumers with the documented and direct resolver behavior. The remaining question is merge ownership of the upgrade-visible provider/model routing change, not a different code location for this narrow fix.
