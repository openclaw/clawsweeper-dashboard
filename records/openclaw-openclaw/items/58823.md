## Summary

Keep open: current main and the latest release still leave the shared `agent-scope`/cron path with agent-primary precedence ahead of the global subagent default, while this PR is the narrow open implementation candidate with sufficient proof. The remaining blocker is maintainer acceptance of the upgrade-visible model-routing change, not lack of a concrete fix.

## What This Changes

This PR reorders shared subagent model config selection to prefer per-agent subagent config, then `agents.defaults.subagents.model`, then the agent primary model, with agent/cron regression tests and a local proof script.

## Best Possible Solution

Land this PR or a narrow equivalent if maintainers confirm the documented global-subagent-default contract; otherwise preserve the shipped agent-primary behavior by updating docs and tests instead of merging this reversal.

## Reproduction Assessment

Yes, at source level. Configure an agent with its own model plus `agents.defaults.subagents.model`; current main's shared `agent-scope` helper still chooses the agent model before the global subagent default, and cron consumes that helper.

## Solution Assessment

Yes, if maintainers accept the documented contract: moving the global default before the agent primary in the shared helper is the narrowest remaining fix because direct spawn selection already has that order. If maintainers prefer shipped agent-primary behavior, the better solution is docs/test alignment instead of this reversal.
