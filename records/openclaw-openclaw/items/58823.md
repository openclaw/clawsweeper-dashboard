## Summary

Keep this PR open. Current main fixes the direct spawn resolver, but the shared `agent-scope` helper used by cron and fallback paths still selects the agent model before the global subagent default, so the branch still has unique useful work and needs maintainer choice against the broader open runtime fix.

## What This Changes

The PR reorders subagent model selection so per-agent subagent config wins first, global `agents.defaults.subagents.model` wins before the agent primary model, and related tests/proof cover the resolver paths.

## Best Possible Solution

Land one maintainer-approved canonical fix that aligns direct spawn, shared helper, cron/fallback, and any required runtime persistence behavior with the documented subagent default contract, with explicit upgrade/release-note context for provider/auth routing changes.
