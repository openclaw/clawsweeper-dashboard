## Summary

Keep open: current main and the latest release still leave the shared agent-scope and isolated-cron path using agent primary before the global subagent default, while this PR fixes that remaining gap. ## What This Changes

The branch moves shared subagent model selection to per-agent subagent model, then global subagent default, then agent primary, and updates cron, fallback, test, and proof coverage.

## Best Possible Solution

Land this narrow precedence fix after maintainers explicitly accept the documented global-subagent-default precedence, then rebase or narrow the broader runtime-persistence work around that contract.

## Solution Assessment

Yes, pending maintainer acceptance of the compatibility tradeoff. Moving the global subagent default ahead of agent primary in the shared helper is the narrow fix for the remaining mismatch, while the broader runtime persistence work should be sequenced separately.
