## Summary

Keep this PR open: current main and v2026.6.10 still leave the shared agent-scope helper selecting an agent primary before the global subagent default, and isolated cron consumes that helper. The patch is a focused candidate fix for the linked open issue, but it needs maintainer acceptance because it intentionally changes model/provider precedence for existing configs.

## What This Changes

The PR reorders shared subagent model config selection so `agents.defaults.subagents.model` beats an agent primary, updates agent and cron tests/mocks, and adds a terminal proof script.

## Best Possible Solution

Choose one canonical subagent model precedence. If the documented contract stands, land this narrow helper/cron fix and sequence it against https://github.com/openclaw/openclaw/pull/72984; if the shipped agent-primary order is intended, update docs and tests to match that decision.

## Solution Assessment

Yes, pending maintainer acceptance of the compatibility tradeoff. Moving the shared helper and its cron-facing tests to the same precedence as the already-fixed native resolver is the narrowest maintainable fix; the broader runtime persistence PR should be sequenced separately.
