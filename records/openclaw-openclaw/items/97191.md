## Summary

Keep open: the patch is focused and includes useful live-output proof, but the new guard only runs for newline-delimited lines and misses the final/no-newline buffer path, so it is not yet a complete proxy SSE OOM fix.

## What This Changes

The branch adds a 1 MiB byte-length check for newline-delimited proxy SSE lines in `streamProxy` and tests oversized and coalesced SSE lines.

## Best Possible Solution

Cover complete lines, the final no-newline frame, and the buffered tail with one byte-accurate proxy SSE size policy, coordinated with the 16 MiB total-body guard if maintainers accept both limits.

## Reproduction Assessment

Yes, by source inspection. After this patch, an oversized final SSE frame without a trailing newline still reaches `processSseLine(buffer)` with no byte check, and an open no-newline tail can keep growing before any complete-line cap runs.

## Solution Assessment

No. The complete-line check is a plausible mitigation, but the best fix must guard the shared line/tail drain path or coordinate with the total-byte guard so all proxy SSE buffering paths are bounded.
