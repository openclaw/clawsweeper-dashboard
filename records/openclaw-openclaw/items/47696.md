## Summary

Keep this PR open for maintainer/security review. Current main still has no `/trust` or `/untrust` implementation, but the proposed patch snapshots a temporary full-exec grant into queued run state and leaves status reporting unaware of the active grant, so it should not merge as-is.

## What This Changes

The PR adds `/trust [minutes]` and `/untrust` slash commands, in-memory session trust-window state, exec-security override resolution, docs, changelog text, and unit tests.

## Best Possible Solution

Land one consolidated trust-window design that validates time-bounded authority at execution time, discloses active trust and remaining duration in status/help surfaces, and keeps docs/changelog scoped to that design.

## Reproduction Assessment

Yes. Source inspection is enough: the PR diff resolves trust into `execOverrides` during directive parsing, while current main later replays stored `run.execOverrides` for queued execution.

## Solution Assessment

No. The feature direction is plausible, but a time-bounded full-exec grant should be represented or revalidated at execution time and surfaced in `/exec` status instead of being snapshotted silently at parse time.

  - body: The patch resolves active trust into `execOverrides.security` while parsing the message. Queued and follow-up runs later replay the stored `run.execOverrides`, so a run accepted near expiry can execute with `security=full` after the window has elapsed.   - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff introduces temporary exec privilege elevation but currently snapshots trust before queued execution and hides active trust from status reporting.

Concerns:

- **[high] Queued runs can outlive the trust window:** `src/auto-reply/reply/get-reply-directives.ts:75`
  - body: Trust is resolved into a stored `execOverrides.security` value at message parse time, while queued and follow-up runs replay that stored value later. This can extend `security=full` beyond the promised trust window for delayed execution.
