## Summary

Keep open: current main still has no `/trust` or `/untrust` implementation, and this PR is a security-sensitive competing implementation with execution-time authority, status visibility, and changelog-scope blockers that need maintainer/security direction before merge.

## What This Changes

The PR adds `/trust [minutes]` and `/untrust` slash commands, in-memory session trust-window state, exec-security override resolution, docs, changelog text, and unit tests.

## Best Possible Solution

Land one consolidated trust-window design that validates time-bounded authority at execution time, exposes active trust and remaining duration in user-visible status/help surfaces, and keeps docs/changelog scoped to that design.

## Reproduction Assessment

Yes. Source inspection shows the PR resolves trust into `execOverrides` during message parsing, while current main stores and replays `run.execOverrides` for queued/follow-up execution.

## Solution Assessment

No. The feature direction is plausible, but a parse-time `security=full` override is not the narrowest safe model for time-bounded privilege; execution-time revalidation or an explicit expiring authority plus visible status/audit is safer.

Queued and follow-up runs later replay the stored `run.execOverrides`, so a run accepted near expiry can execute with `security=full` after the window has elapsed; represent trust as an expiring authority or recheck it at execution time.

## Security Review

Status: needs_attention

Summary: The patch introduces temporary exec privilege elevation and currently snapshots that privilege before delayed execution and omits operator-visible trust status.

Concerns:

- **[high] Queued runs can outlive the trust window:** `src/auto-reply/reply/get-reply-directives.ts:75`
  - body: Trust is resolved into a stored `execOverrides.security` value at message parse time, while queued and follow-up runs replay that stored value later. This can extend `security=full` beyond the promised trust window for delayed execution.
  - confidence: 0.88
- **[medium] Exec status can misrepresent effective security:** `src/auto-reply/reply/get-reply-directives.ts:75`
  - body: The `/exec` status path does not include the active trust override, so an operator can see a lower-security status while execution is effectively full-trust.
  - confidence: 0.82
