## Summary

Keep open: this targets a real documented Discord elevated-authorization bug, but the current PR still has blocking implementation and diagnostic-output regressions plus missing external real behavior proof.

## What This Changes

The PR adds a shared Discord elevated allowFrom fallback resolver, uses it from runtime authorization and `sandbox explain`, derives account context for diagnostics, and adds focused unit tests.

## Best Possible Solution

Land a rebased fix that implements the Discord fallback through the Discord channel plugin `elevated.allowFromFallback` adapter, shares that generic path with `sandbox explain` including account context, preserves current human diagnostics, and includes regression tests plus redacted real behavior proof.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is clear: current docs promise the fallback, runtime only calls the channel plugin fallback hook, Discord does not register that hook, and `sandbox explain` reads only explicit elevated allowlists.

## Solution Assessment

No. The intended behavior is correct, but this PR puts Discord-specific elevated policy in core and regresses the human `sandbox explain` formatter; the narrower maintainable fix is a Discord plugin adapter with preserved diagnostics.
