## Summary

## What This Changes

This PR adds a Discord elevated `allowFrom` fallback resolver, reuses it from runtime elevated authorization and `sandbox explain`, infers account context for diagnostics, and adds focused regression tests.

## Best Possible Solution

Land a rebased or replacement fix that registers Discord `elevated.allowFromFallback` in the Discord channel adapter, shares that generic resolution path with `sandbox explain`, preserves the current human output contract, and includes real after-fix proof.

## Reproduction Assessment

Yes, source-level. Current main documents the Discord fallback, but runtime only asks the channel `elevated.allowFromFallback` hook, Discord does not register that hook, and diagnostics read only explicit elevated allowlists.

## Solution Assessment

No. The PR fixes the right user-visible behavior, but the maintainable fix should keep Discord-specific fallback policy behind the Discord channel adapter and preserve the existing `sandbox explain` human output.
