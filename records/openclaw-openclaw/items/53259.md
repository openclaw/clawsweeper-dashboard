## What This Changes

This PR adds Discord elevated `allowFrom` fallback resolution for runtime authorization and `sandbox explain`, infers Discord account context for diagnostics, and adds focused regression tests.

## Best Possible Solution

Land a rebased narrow fix that uses a Discord-owned elevated fallback adapter shared by runtime and `sandbox explain`, preserves existing human CLI output, and keeps regression coverage for fallback, explicit empty override, and multi-account diagnostics.

## Reproduction Assessment

Yes. A source-level current-main reproduction is clear: configure only `channels.discord.allowFrom`, omit `tools.elevated.allowFrom.discord`, and runtime plus `sandbox explain` never consult the documented Discord fallback because no Discord `elevated.allowFromFallback` adapter exists.

## Solution Assessment

No. The PR is close to the right fix direction, but it should preserve the current human `sandbox explain` output and preferably keep Discord-specific fallback ownership in the Discord plugin or an extension-owned adapter instead of a core special case.
