## Summary

Keep open: the PR targets a valid documented Discord elevated-auth bug, but the current patch still needs changes before merge because it puts Discord-owned fallback policy in core, regresses human `sandbox explain` diagnostics, and lacks the required changelog entry.

## What This Changes

This PR adds a Discord elevated `allowFrom` fallback resolver, wires it into runtime elevated authorization and `sandbox explain`, infers account context for diagnostics, and adds focused regression tests.

## Best Possible Solution

Land a focused fix that implements Discord fallback through the Discord channel plugin adapter, shares generic fallback resolution with diagnostics, preserves existing human `sandbox explain` output, and adds regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows the docs promise the Discord fallback, runtime only asks the plugin fallback hook, the Discord plugin lacks that adapter, and `sandbox explain` reads only the explicit elevated allowlist path.

## Solution Assessment

No. The intended behavior is correct, but this patch is not the best implementation because it embeds Discord policy in core and removes existing human diagnostics; the narrower maintainable fix is a Discord plugin fallback adapter plus preserved output.
