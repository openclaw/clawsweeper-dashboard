## Summary

Keep open: current main still does not honor the documented Discord elevated allowFrom fallback, and this PR targets a real bug, but the patch still has merge-blocking review issues and only mock/unit-test proof.

## What This Changes

The PR adds a Discord elevated allowFrom fallback resolver, uses it in runtime elevated authorization and `sandbox explain`, infers Discord account context for diagnostics, and adds focused regression tests.

## Best Possible Solution

Land a focused fix that implements Discord fallback through the Discord channel plugin adapter, shares generic fallback resolution with diagnostics, preserves the existing human `sandbox explain` output, and covers fallback, empty override, and account-scoped diagnostics.

## Reproduction Assessment

Yes. Source inspection on current main shows the docs promise the fallback, runtime only asks the plugin fallback hook, the Discord plugin has no fallback adapter, and `sandbox explain` reads only the explicit elevated allowlist path.

## Solution Assessment

No. The intended behavior is correct, but this patch puts Discord-owned fallback policy in core and regresses the human diagnostics output; the narrower maintainable fix is a Discord plugin adapter plus preserved formatter behavior.
