Keep this PR open: it targets a real documented Discord elevated-authorization bug that current main still has, but the proposed patch still needs contributor proof and maintainer review before merge.

## What This Changes

The PR adds Discord elevated `allowFrom` fallback resolution to runtime authorization and `sandbox explain`, threads account context into diagnostics, and adds focused regression tests.

## Best Possible Solution

Land a rebased fix that implements the Discord fallback through the Discord channel adapter, shares the generic resolution path with `sandbox explain` including account context, preserves current human diagnostics, and includes regression tests plus redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: docs promise the Discord fallback, runtime only calls the channel plugin fallback hook, Discord does not implement that hook, and diagnostics read only explicit elevated allowlists.

## Solution Assessment

No. The intended behavior is right, but this PR special-cases Discord elevated policy in core and regresses existing human `sandbox explain` output; the safer fix belongs in the Discord adapter while preserving the output contract.
