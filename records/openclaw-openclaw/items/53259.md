## Summary

Keep open: this is a valid fix for a documented Discord elevated-authorization bug, but the PR is not merge-ready because it still regresses human `sandbox explain` output, lacks real Discord/gateway proof, and is dirty against current main.

## What This Changes

This PR adds a Discord elevated allowFrom fallback resolver, uses it in runtime elevated authorization and `sandbox explain`, infers account context for diagnostics, and adds focused unit tests.

## Best Possible Solution

Land a rebased fix that puts Discord fallback behind the Discord `elevated.allowFromFallback` adapter, shares that generic path with `sandbox explain`, preserves existing human diagnostics, and includes regression tests plus redacted real Discord/gateway proof.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: configure only `channels.discord.allowFrom`, omit `tools.elevated.allowFrom.discord`, and runtime gets no fallback because Discord has no `elevated.allowFromFallback` adapter while `sandbox explain` reads only explicit elevated allowlists.

## Solution Assessment

No. The intended behavior is correct, but this PR hard-codes Discord fallback policy in core and regresses the human diagnostics output; the narrower maintainable fix is a Discord plugin adapter plus preserved formatter behavior.
