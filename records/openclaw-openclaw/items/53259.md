## Summary

Keep open: the PR targets a real documented Discord elevated-authorization regression, but the current patch still has merge-blocking review issues around the plugin ownership boundary, human `sandbox explain` diagnostics, and the required changelog entry.

## What This Changes

The PR adds a Discord elevated `allowFrom` fallback resolver, reuses it in runtime elevated auth and `sandbox explain` with account context, and adds focused regression tests.

## Best Possible Solution

Land a focused fix that implements Discord fallback through the Discord channel plugin adapter, shares the generic fallback path with diagnostics, preserves current human `sandbox explain` output, adds regression coverage, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. Source inspection on current main shows the docs promise the fallback, runtime only calls a plugin hook, the Discord plugin has no fallback adapter, and `sandbox explain` reads only the explicit elevated allowlist path.

## Solution Assessment

No. The intended behavior is correct, but the current PR is not the narrowest maintainable fix because it puts Discord-owned fallback policy in core and removes existing human diagnostics output; the safer path is a Discord plugin adapter plus preserved formatter output.
