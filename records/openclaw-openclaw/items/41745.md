## Summary

Keep open: current main still has the premature `systemSent` persistence window, and this PR is a plausible fix path, but the latest PR head still fails type checking at `session-run-accounting.ts:95` and lacks contributor-provided real behavior proof.

## What This Changes

This PR removes the first-turn `systemSent` disk write from `ensureSkillSnapshot()`, adds post-success persistence for reply and queued followup runs, and adds regression tests.

## Best Possible Solution

Land a corrected version that keeps first-turn snapshot persistence from marking `systemSent` on disk, persists the marker only after successful reply and followup completions, treats marker persistence as best-effort, and includes redacted real-run retry proof.

## Reproduction Assessment

Yes, by source inspection. Current main persists `systemSent: true` while preparing the first-turn skill snapshot before the later reply/model execution path runs, so a first-call provider failure can leave the disk marker set too early.

## Solution Assessment

No, not as submitted. Deferring persistence until post-success reply/followup runs is the right narrow design, but the optional `sessionEntry` assignment must be fixed and real behavior proof is still missing.
