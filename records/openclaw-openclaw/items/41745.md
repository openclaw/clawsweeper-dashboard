## Summary

Keep open: the bug is source-reproducible and the PR is useful, but the latest head still has blocking correctness issues and lacks required real behavior proof.

## What This Changes

The PR removes the first-turn `systemSent` disk write from `ensureSkillSnapshot`, adds post-success persistence for reply/followup runs, and adds regression tests.

## Best Possible Solution

Land a rebased patch that defers `systemSent` disk writes until verified successful reply/followup completion, preserves current snapshot hydration, and proves the provider-failure retry flow.

## Reproduction Assessment

Yes, source-reproducible. Current main writes `systemSent: true` during `reply.ensure_skill_snapshot` before model execution, so a first-call provider failure can leave disk state saying the system prompt was delivered.

## Solution Assessment

No, not as submitted. Deferring persistence is the right narrow fix, but this head must guard the optional session entry, preserve current-main snapshot hydration, and include real retry proof.
