## Summary

Keep open: the reported session corruption bug is still source-reproducible on current main and the PR is aimed at the right boundary, but the current head still has a blocking optional-entry mutation/type failure, drops current-main skills snapshot hydration, and lacks required real behavior proof.

## What This Changes

The PR removes pre-run `systemSent` persistence from `ensureSkillSnapshot`, adds post-success persistence for reply and followup runs, and adds regression coverage.

## Best Possible Solution

Land a rebased patch that defers `systemSent` disk writes until verified successful reply/followup completion, preserves current snapshot hydration, and proves the provider-failure retry flow.

## Reproduction Assessment

Yes, source-reproducible. Current main writes `systemSent: true` during `reply.ensure_skill_snapshot` before the model call, so a first-call provider failure can leave disk state saying the system prompt was delivered.

## Solution Assessment

No, not as submitted. Deferring persistence is the right narrow fix, but this head must avoid mutating an optional session entry, preserve current-main snapshot hydration, and include real retry proof.
