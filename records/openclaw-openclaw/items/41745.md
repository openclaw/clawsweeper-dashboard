## Summary

Keep open: current `main` still has the premature `systemSent` persistence path, and this PR is a focused fix for a real session-corruption bug, but the latest head is not mergeable because it still has a strict TypeScript blocker and lacks contributor-provided real behavior proof.

## What This Changes

The PR defers `systemSent` disk persistence from `ensureSkillSnapshot()` to post-success reply and followup accounting, with regression tests for failed-first-call retry and silent followup success paths.

## Best Possible Solution

Land a corrected version that keeps first-turn snapshot persistence from marking `systemSent` on disk, persists the marker only after successful reply/followup completions, treats marker persistence as best-effort, and includes redacted real-run retry proof.

## Reproduction Assessment

Yes, by source inspection. Current `main` persists `systemSent: true` in `ensureSkillSnapshot()` before the model run, so a first-call provider failure can leave the disk marker set before the system prompt and tools were successfully delivered.

## Solution Assessment

No, not as submitted. Deferring persistence until post-success reply/followup runs is the right narrow design, but the optional `sessionEntry` assignment must be guarded and real behavior proof is still missing.
