## Summary

Keep open: current main still has the early `systemSent` persistence path, so this PR is not obsolete, but the latest PR head still has a blocking type error and lacks external after-fix behavior proof.

## What This Changes

The PR removes the first-turn `systemSent` disk write from `ensureSkillSnapshot()`, adds post-success persistence for reply and followup runs, and adds regression tests.

## Best Possible Solution

Land a rebased version that preserves current snapshot hydration, defers the disk marker until successful reply/followup completion, guards optional in-memory entries, and includes redacted real-run retry proof.

## Reproduction Assessment

Yes, by source inspection. Current main persists `systemSent: true` in `ensureSkillSnapshot()` before model execution, so a first-call provider failure can leave the disk marker set too early.

## Solution Assessment

No, not as submitted. Deferring persistence until post-success reply/followup runs is the right narrow design, but the optional-entry assignment must be fixed and the branch needs proof plus rebase care.
