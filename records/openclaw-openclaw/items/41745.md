## Summary

The PR addresses a real current-main session-state bug, but it still needs changes before merge because the new optional in-memory assignment is not type-safe and the user-facing fix lacks the required changelog entry.

## What This Changes

The PR removes first-turn `systemSent` disk persistence from skill snapshot setup and adds post-success persistence for normal reply and followup runs with regression tests.

## Best Possible Solution

Land the deferred marker-persistence design after guarding the optional in-memory assignment and adding one Unreleased changelog fix bullet.

## Reproduction Assessment

Yes. Source inspection on current main shows `ensureSkillSnapshot()` persists `systemSent: true` before `runReplyAgent()`, which is enough to reproduce the corrupt-marker path after a first-call provider failure.

## Solution Assessment

No, not as submitted. Deferring persistence until after successful reply/followup runs is the narrow maintainable fix, but the optional `sessionEntry` assignment must be guarded and the user-facing fix needs a changelog entry.
