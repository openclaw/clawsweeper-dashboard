## Summary

Keep open: the underlying session-state bug is real and this PR is still a focused fix path, but the current head has a blocking TypeScript error, drops current-main skill snapshot hydration, and only provides test/CI proof rather than a redacted real provider-failure retry run.

## What This Changes

The branch removes the first-turn `systemSent` disk write from `ensureSkillSnapshot`, adds post-success persistence for reply/followup runs, and adds regression tests.

## Best Possible Solution

Land a rebased, type-clean patch that defers `systemSent` disk writes until verified successful reply/followup completion, preserves current-main skill snapshot hydration, and proves the retry flow in a real run.

## Reproduction Assessment

Yes, source-reproducible. Current main persists `systemSent: true` during `reply.ensure_skill_snapshot` before model execution, so a first-call provider failure can leave disk state saying the system prompt was delivered.

## Solution Assessment

No, not as submitted. Deferring persistence is the right narrow fix, but this head must guard the optional session entry, preserve `hydrateResolvedSkills(...)`, and include real retry proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Guard the optional session entry before mutating it:** `src/auto-reply/reply/session-run-accounting.ts:95`
  - body: `sessionEntry` is optional, and the guard is still true when it is absent; after a successful store update this assigns `sessionEntry.systemSent = true`. That is the current TS18048 CI failure and can also log a false persistence failure, so only mutate when an entry exists or use the returned store entry.
  - confidence: 0.97
- **[P2] Preserve hydrated skill snapshots on reuse:** `src/auto-reply/reply/session-updates.ts:173`
  - body: Current main hydrates reused snapshots because persisted session entries strip `resolvedSkills`; this branch reuses `current.skillsSnapshot` directly when no refresh is needed. A cold-resumed first turn can pass a stripped snapshot to the runner and lose runtime skill definitions, so keep the deferred marker change while preserving `hydrateResolvedSkills(...)`.
  - confidence: 0.9
