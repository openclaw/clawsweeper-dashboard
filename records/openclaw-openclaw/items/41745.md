## Summary

Keep open: current main still persists `systemSent: true` before model execution, so the PR is addressing a real source-visible bug, but the latest PR head still has a concrete TypeScript/lint blocker and lacks required real behavior proof before merge.

## What This Changes

The PR moves `systemSent` disk persistence out of `ensureSkillSnapshot()` and into post-success reply/followup accounting, with regression coverage for failed first-turn retry behavior.

## Best Possible Solution

Land a corrected version that defers disk persistence until after successful reply/followup runs, guards optional `sessionEntry`, and includes redacted real-run proof of the retry path.

## Reproduction Assessment

Yes, by source inspection. Current main persists `systemSent: true` in `ensureSkillSnapshot()` before `runReplyAgent()`, so a first-call provider failure can leave the disk marker set before the system prompt and tools were successfully delivered.

## Solution Assessment

No, not as submitted. Deferring persistence until post-success reply/followup runs is the right narrow design, but the optional `sessionEntry` assignment must be guarded and real behavior proof is still missing.
