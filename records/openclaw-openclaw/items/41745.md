## Summary

Keep open. Current main still persists `systemSent: true` before the model call, and this PR targets the right bug, but the PR head still fails type checking on the optional `sessionEntry` assignment and lacks contributor-provided real behavior proof.

## What This Changes

The PR removes `systemSent` disk persistence from `ensureSkillSnapshot()` and adds post-success `persistSystemSentAfterSuccess()` calls for reply and followup runs with regression tests.

## Best Possible Solution

Keep the deferred marker-persistence design, guard the optional in-memory assignment, get checks green, and require redacted real-run proof for the retry path before merge.

## Reproduction Assessment

Yes, by source inspection. Current main persists `systemSent: true` in `ensureSkillSnapshot()` before `runReplyAgent()`, so a first-call provider failure can leave the on-disk marker set without a delivered system prompt.

## Solution Assessment

No, not as submitted. Deferring persistence until post-success reply and followup runs is the right narrow design, but the optional `sessionEntry` assignment must be guarded and real behavior proof is still missing.
