## Summary

Keep open: current main still writes `systemSent` before the model run, and this PR is the focused fix path, but the latest head is blocked by a TypeScript error and still lacks contributor-provided real behavior proof.

## What This Changes

The PR removes pre-call `systemSent` disk persistence, adds post-success reply/followup persistence, and adds regression tests for failed first-turn retry behavior.

## Best Possible Solution

Land a corrected version that defers disk persistence until after successful reply/followup runs, guards the optional `sessionEntry`, and includes redacted real-run proof of the retry path.

## Reproduction Assessment

Yes, by source inspection. Current main persists `systemSent: true` in `ensureSkillSnapshot()` before `runReplyAgent()`, so a first-call provider failure can leave the disk marker set before the system prompt and tools were successfully delivered.

## Solution Assessment

No, not as submitted. Deferring persistence until post-success reply/followup runs is the right narrow design, but the optional `sessionEntry` assignment must be guarded and real behavior proof is still missing.
