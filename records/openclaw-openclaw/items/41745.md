## Summary

This PR should stay open because current main still has the early `systemSent` disk write and this is the only live in-repo PR I found for the deferred-persistence fix. It is not merge-ready yet: latest PR head still fails type/lint checks at the optional `sessionEntry` assignment and the contributor has not provided after-fix real behavior proof.

## What This Changes

The PR defers session `systemSent` persistence until after successful LLM reply and followup runs, and adds regression coverage for failed first-turn retry behavior.

## Best Possible Solution

Land a corrected version that removes pre-call disk persistence, writes `systemSent` best-effort only after successful reply/followup runs, and includes redacted real-run proof of the retry path.

## Reproduction Assessment

Yes, by source inspection. Current main computes first-turn state, persists `systemSent: true` in `ensureSkillSnapshot()`, and only later calls `runReplyAgent()`, so a first-call provider failure can leave the disk marker set before the system prompt/tools were successfully delivered.

## Solution Assessment

No, not as submitted. Deferring persistence until post-success reply and followup runs is the narrow maintainable design, but the optional `sessionEntry` assignment must be guarded and real behavior proof is still missing.
