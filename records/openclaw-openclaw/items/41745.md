## Summary

Keep open. Current main still persists `systemSent` during pre-run skill snapshot preparation, while this PR takes the right direction but still has a blocking optional `sessionEntry` assignment error and is missing the required changelog entry.

## What This Changes

The PR removes the pre-run `systemSent` disk write from skill snapshot persistence, adds post-success persistence for normal and followup reply runs, and adds regression coverage for failed first calls and silent followup successes.

## Best Possible Solution

Land the deferred-persistence design after guarding the optional in-memory assignment, keeping marker writes best-effort after successful runs, and adding the required changelog bullet.

## Reproduction Assessment

Yes. Source inspection on current main shows the marker is persisted in `ensureSkillSnapshot()` before `runReplyAgent()`, and the PR-specific blocker is confirmed by public `check-test-types` and `check-lint` annotations on the latest head.

## Solution Assessment

No, not as submitted. The design is the narrow maintainable fix, but the optional `sessionEntry` assignment must be guarded and the changelog requirement satisfied before merge.
