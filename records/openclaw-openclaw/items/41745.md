## Summary

Keep open. Current main still writes `systemSent: true` during `ensureSkillSnapshot()` before reply/model execution, so the PR targets a real bug, but PR head still has a blocking optional-entry assignment/typecheck failure and lacks external real behavior proof.

## What This Changes

The PR removes the first-turn `systemSent` disk write from `ensureSkillSnapshot()`, adds post-success persistence for reply and followup runs, and adds regression tests.

## Best Possible Solution

Land a corrected version that defers the disk marker until successful reply and followup completion, guards optional in-memory entries, treats marker persistence as best-effort, and includes redacted real-run retry proof.

## Reproduction Assessment

Yes, by source inspection. Current main persists `systemSent: true` while preparing the first-turn skill snapshot before later reply/model execution, so a first-call provider failure can leave the disk marker set too early.

## Solution Assessment

No, not as submitted. Deferring persistence until post-success reply/followup runs is the right narrow design, but the optional `sessionEntry` assignment must be fixed and real behavior proof is still missing.
