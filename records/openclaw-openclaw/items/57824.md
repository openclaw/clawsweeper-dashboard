## Summary

Keep open. Current main still has both targeted behaviors, and the provided PR diff remains a focused implementation candidate with the earlier zero-budget throttling concern addressed on the branch.

## What This Changes

The PR removes the ACP path-only gate for image attachments, changes Synology Chat invalid-token throttling to lock at the exhausted budget, clamps zero pre-auth budgets to one attempt, and adds focused regression tests.

## Best Possible Solution

Land this focused fix after a clean refresh and targeted validation, keeping the behavior changes in the ACP attachment resolver and the Synology Chat plugin rather than adding broader core policy.

## Reproduction Assessment

Yes. On current main, a URL-only image context reaches `resolveAcpAttachments` but is skipped by the local-path check; for Synology, `rateLimitPerMinute=5` still allows the fifth invalid token attempt because `recordFailure()` locks only when the count is greater than the limit.

## Solution Assessment

Yes. The proposed path is narrow: remove only the ACP pre-cache path gate while preserving local-path URL stripping, and fix the Synology plugin's limiter boundary plus zero-budget edge case with focused tests.
