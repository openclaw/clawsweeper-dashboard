## Summary

## What This Changes

The PR forwards URL-only ACP image attachments through the existing media cache and changes Synology Chat invalid-token throttling to lock at the configured threshold with zero-budget coverage.

## Best Possible Solution

Land the focused fixes after owners accept the earlier Synology lockout boundary and guarded ACP remote-media fetch; split or request live proof only if maintainers want separate risk decisions.

## Reproduction Assessment

Yes. Current main source shows URL-only MediaUrl attachments are normalized but blocked by the path guard before cache.getBuffer, and Synology invalid-token lockout uses strict > in isLocked and recordFailure.

## Solution Assessment

Yes. Removing only the ACP path gate lets the existing MediaAttachmentCache handle URL-only images while preserving local-path URL stripping, and Synology thresholding belongs in the pre-auth limiter; the remaining question is owner acceptance of the boundary changes.
