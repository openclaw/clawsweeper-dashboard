## Summary

Keep open: current main still has both behaviors this PR targets, and the patch is a focused bug fix with no blocking code finding. The remaining decision is maintainer acceptance of the operator-visible Synology lockout threshold change and the ACP remote-media security boundary expansion.

## What This Changes

The PR removes the current-turn ACP path-only gate so URL-only image attachments can be fetched and forwarded into ACP turns, and changes Synology invalid-token throttling to lock at the configured threshold with zero-budget coverage.

## Best Possible Solution

Land the bounded fixes after maintainer owner review accepts the compatibility and security-boundary risks, or request live Synology/ACP proof if that risk bar is not met.

## Reproduction Assessment

Yes at source level: current main returns before cache loading for URL-only ACP image attachments and uses strict `>` invalid-token checks, so both reported behaviors are traceable without running tests. I did not run local tests because this was a read-only checkout review.

## Solution Assessment

Yes, this is the best code layer for the reported bugs: ACP should reuse the existing media cache rather than add a new fetch path, and Synology thresholding belongs in the pre-auth limiter. The safer alternative is not a code change but stronger live proof or splitting the PR if maintainers want separate risk decisions.
