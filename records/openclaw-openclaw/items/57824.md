## Best Possible Solution

Keep the ACP fix on the existing MediaAttachmentCache path and the Synology threshold fix inside the pre-auth limiter, then land after maintainer owner review accepts the behavior-boundary changes.

## Reproduction Assessment

Yes. Current main still returns before cache loading for URL-only ACP image attachments and uses strict > invalid-token checks, so both behaviors are high-confidence source reproductions without running the test suite.

## Solution Assessment

Yes. This is the best code layer for the reported bugs: ACP should reuse the existing guarded media cache, and Synology thresholding belongs in the pre-auth limiter; the safer alternative is stronger live proof or splitting the PR for separate owner decisions.
