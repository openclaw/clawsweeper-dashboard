## What This Changes

The PR removes the ACP local-path prerequisite for image attachments, changes Synology Chat invalid-token throttling to lock at the threshold with a zero-limit clamp, and adds focused regression tests.

## Best Possible Solution

## Reproduction Assessment

Yes. On current main, a URL-only image reaches normalized attachments but is skipped by the ACP local-path check, and a Synology `rateLimitPerMinute` of 5 returns 401 on the fifth invalid token because the lock uses `>` instead of `>=`.

## Solution Assessment

Yes for the code path. Removing only the ACP pre-cache path gate while preserving local-path URL stripping, and tightening the Synology limiter with a zero-budget clamp, is the narrow maintainable fix; the missing changelog is the remaining mechanical gap.
