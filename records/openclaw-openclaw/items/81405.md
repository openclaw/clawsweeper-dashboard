## Summary

Keep open: current main and the latest stable tag still lack the requested browser-callback-settled or cancellable manual-prompt contract for remote OpenAI Codex OAuth. This is a real provider auth UX/design feature, but it is not a broken documented contract and it needs a maintainer contract choice before implementation.

## What This Changes

Request an explicit callback-settled signal or cancellable manual-input contract so remote OpenAI Codex OAuth can dismiss a visible paste prompt after an SSH-tunneled browser callback completes sign-in.

## Best Possible Solution

Keep this issue open until maintainers choose and implement a provider-owned callback-settled or cancellable manual-input contract that preserves immediate paste for headless users and automatic completion for tunneled users.

## Reproduction Assessment

Not applicable as a failing bug. Source inspection gives a high-confidence current behavior check: the remote path starts manual paste immediately, and no current OpenAI OAuth helper or prompt API exposes a browser-callback-settled signal to cancel it.

## Solution Assessment

Unclear until maintainers choose the contract shape. The requested direction is reasonable, but the maintainable solution should keep OpenAI-specific OAuth behavior provider-owned and add any shared prompt-cancellation seam intentionally.
