## Summary

Keep open: current main has the remote OAuth wording clarification and local fallback suppression, but it still lacks the requested provider-owned browser-callback-settled signal or cancellable manual-prompt contract. This remains a feature/design item for the OpenAI Codex OAuth path, not a broken documented contract.

## What This Changes

Add a callback-settled signal so remote OpenAI Codex OAuth can dismiss the manual paste prompt after browser callback completion.

## Best Possible Solution

Keep this open as a provider-owned feature/design item and add a callback-settled or cancellable manual-input contract without delaying paste-only remote users or inferring success from token-exchange traffic.

## Reproduction Assessment

Not applicable as a failing bug, but source inspection gives a high-confidence current behavior check: the remote path starts the paste prompt immediately and the pinned OAuth helper exposes only a browser/manual race, not a callback-settled event.

## Solution Assessment

Unclear until maintainers choose the contract shape. The requested direction is reasonable, but the maintainable solution should keep OpenAI-specific OAuth behavior provider-owned and add any shared prompt-cancellation seam intentionally.
