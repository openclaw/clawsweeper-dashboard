## Summary

Keep open: current main and v2026.6.10 still lack a prompt-scoped callback-settled or cancellation signal for the remote OpenAI ChatGPT/Codex OAuth manual paste prompt, and the open candidate PR still needs maintainer review before this issue can close.

## What This Changes

Request a provider-owned callback-settled signal so remote OpenAI ChatGPT/Codex OAuth can dismiss the manual paste prompt when the browser callback wins.

## Best Possible Solution

Adopt a provider-owned prompt cancellation or callback-settled contract that keeps immediate remote paste input, dismisses it when browser callback completion wins, and covers paste-only plus SSH-forwarded paths with focused tests or live proof.

## Reproduction Assessment

Yes, at source level: current main shows the remote manual prompt can be started without a prompt-scoped cancellation or callback-settled event while the browser callback can still win. I did not run a live SSH-tunneled OAuth replay.

## Solution Assessment

Unclear as a final implementation: the provider-owned signal direction fits the issue, but maintainers still need to accept the exact prompt-contract shape and the tradeoffs in the active candidate PR.
