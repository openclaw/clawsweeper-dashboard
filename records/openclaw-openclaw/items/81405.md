## Summary

Keep open: current main still has the copy-only remote OAuth clarification and local fallback settlement suppression, but it does not expose the requested callback-settled or cancellable prompt contract. This is a provider/API design feature, not a broken documented behavior, so it needs maintainer direction before implementation.

## What This Changes

Request an explicit provider-owned signal so remote OpenAI Codex OAuth can dismiss the manual paste prompt when the browser callback completes first.

## Best Possible Solution

Keep this open until maintainers choose a callback-settled or cancellable manual-input contract that preserves immediate paste input for remote users and avoids token-exchange inference.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection does show the current remote path starts the paste prompt immediately while the relevant helper and prompt APIs lack a callback-settled or cancellation signal.

## Solution Assessment

Unclear until maintainers choose the contract shape. The requested direction is reasonable, but the maintainable solution should keep OpenAI-specific OAuth behavior provider-owned and add any shared prompt-cancellation seam intentionally.
