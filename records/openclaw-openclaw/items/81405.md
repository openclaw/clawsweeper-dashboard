## Summary

Keep open: current main and v2026.6.8 still expose the remote OpenAI ChatGPT/Codex OAuth manual prompt without a callback-settled or cancellable prompt signal, while https://github.com/openclaw/openclaw/pull/89491 is only an open candidate that still needs maintainer decisions.

## What This Changes

Request an explicit provider-owned signal or prompt-cancellation contract so remote OpenAI ChatGPT/Codex OAuth can dismiss manual paste input when the browser callback wins.

## Best Possible Solution

Adopt a provider-owned prompt cancellation or callback-settled contract that keeps immediate remote paste input, dismisses it when browser callback completion wins, and covers paste-only plus SSH-forwarded paths with focused tests or live proof.

## Reproduction Assessment

Yes by source inspection: the remote OAuth path still starts or reuses a manual prompt without a prompt-scoped cancellation signal, and the browser callback path can win without notifying that prompt. No live remote SSH-tunnel OAuth replay was run.

## Solution Assessment

Unclear as a final implementation: the provider-owned signal direction fits the issue, but maintainers still need to accept the exact prompt-contract shape and the tradeoffs in https://github.com/openclaw/openclaw/pull/89491.
