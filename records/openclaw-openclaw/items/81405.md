## Summary

Keep open: current main still lacks an explicit callback-settled or prompt-cancellation contract for remote OpenAI Codex OAuth. The merged wording fix in https://github.com/openclaw/openclaw/pull/81301 clarifies the two remote paths, but it intentionally leaves OAuth timing, callback handling, and prompt dismissal unchanged.

## What This Changes

Request to add an explicit OpenAI Codex OAuth callback-settled signal so remote/VPS paste prompts can be dismissed when an SSH-forwarded browser callback completes.

## Best Possible Solution

Add a provider-owned callback-settled or cancellable-prompt contract, or adopt an upstream equivalent, while preserving immediate paste input for remote users and keeping any shared prompt-cancellation seam generic.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this is a feature/API request. Source inspection establishes the missing contract and the current remote flow, but no live SSH-tunneled OAuth run was performed.

## Solution Assessment

Unclear: the requested callback-settled event is the right problem shape, but the exact API belongs to maintainer/provider ownership and may need upstream dependency support.
