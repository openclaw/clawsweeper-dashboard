## Summary

Keep open: current main still lacks the requested provider-owned browser-callback-settled signal or cancellable manual-prompt contract, while the issue is a feature/design request rather than a broken documented contract.

## What This Changes

Add a callback-settled signal for remote OpenAI Codex OAuth so a visible manual paste prompt can be dismissed when the browser callback wins.

## Best Possible Solution

Keep this as an OpenAI-provider feature/design item and add a callback-settled or cancellable manual-input contract without delaying paste-only remote users or inferring success from token-exchange traffic.

## Reproduction Assessment

Not applicable as a failing bug. Source inspection gives a high-confidence current-behavior check: the remote path starts the paste prompt immediately and the pinned OAuth helper exposes only a browser/manual race, not a callback-settled event.

## Solution Assessment

Unclear until maintainers choose the contract shape. The requested direction is reasonable, but the maintainable solution should keep OpenAI-specific OAuth behavior provider-owned and add any shared prompt-cancellation seam intentionally.
