## Summary

Keep open: current main has the wording-only remote OAuth clarification and local fallback settlement suppression, but remote prompt dismissal still lacks the requested provider-owned callback-settled signal. This is a feature/design item, not a broken documented contract.

## What This Changes

Request an explicit callback-settled or cancellable manual-input signal so remote OpenAI Codex OAuth can dismiss stale paste prompts after browser callback success.

## Best Possible Solution

Keep this open as a provider-owned feature/design item and add a callback-settled or cancellable manual-input contract without delaying paste-only remote users or inferring success from token exchange traffic.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection does confirm the current remote path starts the paste prompt immediately and the locked OAuth helper exposes only a race API, not a callback-settled event.

## Solution Assessment

Unclear until maintainers choose the contract shape. The direction is sound, but the maintainable solution should keep OpenAI-specific OAuth behavior provider-owned and use a generic prompt-cancellation seam only if that broader API is intentionally added.
