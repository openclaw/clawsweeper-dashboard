## Summary

Keep open: current main still lacks a provider-owned browser-callback-settled signal or cancellable manual-prompt contract. The related wording PR improved the remote instructions but intentionally left OAuth timing, callback handling, prompt cancellation, provider contracts, and config surface unchanged, so this remains a feature/design item rather than a fixed bug.

## What This Changes

The issue requests a callback-settled signal so remote OpenAI Codex OAuth can dismiss a manual paste prompt after a browser callback succeeds.

## Best Possible Solution

Add a provider-owned callback-settled or cancellable manual-input contract for OpenAI Codex OAuth while keeping remote paste input immediate and avoiding token-exchange inference.

## Reproduction Assessment

Not applicable as a failing bug. Source inspection gives a high-confidence current behavior check: remote auth starts the paste prompt immediately, and the pinned OAuth helper only resolves the internal browser/manual race without exposing a browser-wins signal.

## Solution Assessment

Unclear until maintainers choose the contract shape. The requested direction is reasonable, but the maintainable solution should keep OpenAI-specific OAuth behavior provider-owned and add any shared prompt-cancellation seam intentionally.
