## Summary

Keep open: current main still lacks the requested provider-owned browser-callback-settled signal or cancellable manual-prompt contract. The existing remote path is documented and tested as a browser/manual race, and the merged wording-only PR intentionally left lifecycle behavior unchanged, so this remains a feature/design item rather than conservative cleanup.

## What This Changes

Add a provider-owned callback-settled or prompt-cancellation signal so remote OpenAI Codex OAuth can dismiss a visible manual paste prompt when the browser callback wins.

## Best Possible Solution

Keep this open as a provider-owned feature/design item and add an explicit callback-settled or cancellable manual-input contract without delaying paste-only remote users or inferring success from token-exchange traffic.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a failing bug, but source inspection gives a high-confidence current behavior check: remote auth starts a paste prompt immediately and the pinned OAuth helper exposes only a browser/manual race, not a callback-settled event.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose the contract shape; the requested direction is sound, but the maintainable solution should keep OpenAI-specific OAuth behavior provider-owned and add any shared prompt-cancellation seam intentionally.
