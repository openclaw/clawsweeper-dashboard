## Summary

Keep open: current main still lacks an explicit browser-callback-settled signal, and the request is a real OpenAI provider lifecycle feature rather than a stale duplicate or already-implemented bug fix.

## What This Changes

Add a provider-owned callback-settled signal so remote OpenAI Codex OAuth can dismiss manual paste input when the browser callback completes first.

## Best Possible Solution

Adopt a provider-owned callback-settled or prompt-cancellation contract in the OpenAI plugin/upstream helper, keep the prompt cancellation seam generic, and cover both paste-only and SSH-forwarded callback paths.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: remote onAuth starts the paste prompt immediately, while the locked helper can complete through the browser callback without exposing a callback-won event to cancel that prompt.

## Solution Assessment

Yes in direction, but the exact API shape needs maintainer/provider decision; a callback-settled contract is cleaner than token-exchange inference or delaying remote paste input.
