## Summary

Keep open: current main has the remote OAuth wording clarification and local fallback suppression, but it still does not expose a provider-owned browser-callback-settled event or cancellable manual-prompt handle for the remote OpenAI Codex OAuth paste prompt. This is a feature/API design item, not a narrow bug-fix candidate.

## What This Changes

Request a callback-settled or cancellable manual-prompt contract so remote SSH-tunneled OpenAI Codex OAuth browser callbacks can dismiss an already visible paste prompt.

## Best Possible Solution

Keep this as provider-owned design work and add an explicit callback-settled or cancellable manual-input contract while preserving immediate remote paste input and the local delayed fallback behavior.

## Reproduction Assessment

Not applicable as a failing bug; this is a feature/API request. Source inspection gives a high-confidence current behavior check: remote mode starts paste input immediately and the dependency exposes only a browser/manual race, not a callback-settled/cancellable prompt event.

## Solution Assessment

Unclear until maintainers choose the contract shape. The requested provider-owned signal is a reasonable direction, but a safe solution should avoid token-exchange inference, avoid delaying paste-only remote users, and keep OpenAI-specific OAuth behavior in the provider/plugin path.
