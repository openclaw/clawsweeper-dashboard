## Summary

Keep open. Current main has the wording-only improvement from https://github.com/openclaw/openclaw/pull/81301, but it still lacks a callback-settled or prompt-cancellation contract for remote OpenAI Codex OAuth, so the requested lifecycle improvement remains real work.

## What This Changes

Request an explicit browser-callback-settled or prompt-cancellation signal so remote OpenAI Codex OAuth can dismiss manual paste UI after SSH-forwarded callback completion.

## Best Possible Solution

Add a provider-owned callback-settled or cancelable manual-input contract in the OpenAI Codex OAuth path, keep remote paste input immediate, and cover paste-only plus SSH-forwarded callback paths with focused tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction. Source inspection on current main confirms the requested lifecycle signal is absent while the remote manual prompt is created immediately.

## Solution Assessment

Unclear until maintainers choose the contract shape. The issue points at the right boundary, but the safest implementation should avoid token-exchange inference and either use upstream callback-settled support or add a generic prompt-cancellation seam.
