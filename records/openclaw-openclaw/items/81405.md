## Summary

Keep open: current main has the wording-only remote OAuth clarification and local fallback settlement suppression, but it still lacks the requested provider-owned callback-settled signal or prompt-cancellation contract for an already-visible remote paste prompt. This remains a feature/design item rather than a broken documented contract.

## What This Changes

Request an explicit callback-settled signal so remote OpenAI Codex OAuth can dismiss a manual paste prompt when SSH-tunneled browser callback completion wins.

## Best Possible Solution

Keep this open as a provider-owned feature/design item and add a callback-settled or cancellable manual-input contract without delaying paste-only remote users or inferring success from token exchange traffic.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection does confirm the current remote path starts the paste prompt immediately and the locked OAuth helper exposes only a race API, not a callback-settled event.

## Solution Assessment

Unclear until maintainers choose the contract shape. The direction is sound, but the maintainable solution should keep OpenAI-specific OAuth behavior provider-owned and use a generic prompt-cancellation seam only if that broader API is intentionally added.
