## Summary

Keep open: current main and the pinned OAuth helper confirm the remote flow can show manual paste input while the browser callback wins, and no existing main implementation or related item provides the requested callback-settled/cancel signal. This is a concrete provider/prompt lifecycle feature, not cleanup-closeable.

## What This Changes

Request an explicit callback-settled signal so remote OpenAI Codex OAuth can dismiss manual paste prompts when the browser callback completes first.

## Best Possible Solution

Add an explicit provider-owned callback-settled or cancellable-manual-input contract, preserving immediate remote paste input and local delayed fallback while avoiding token-exchange inference.

## Reproduction Assessment

Yes, at source level. Current main creates the remote manual prompt during onAuth, while the pinned pi-ai helper can let the browser callback win internally without exposing a separate settled event to cancel that prompt.

## Solution Assessment

Unclear as an implementation shape, but the requested direction is sound. The maintainable fix is an explicit provider/upstream signal or generic prompt-cancellation seam, not token-request monitoring or arbitrary timing.
