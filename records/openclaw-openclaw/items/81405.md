## Summary

This remains a provider-auth feature/design item, not closeable cleanup.

## What This Changes

Request an explicit OpenAI Codex OAuth callback-settled or manual-prompt cancellation contract so remote SSH-tunneled browser callback completion can dismiss the visible paste prompt.

## Best Possible Solution

Add a provider-owned callback-settled or cancellable manual-input contract for OpenAI Codex OAuth, with any shared prompt-cancellation seam kept generic and covered for paste-only plus callback-winning remote flows.

## Reproduction Assessment

Not applicable as a failing-bug reproduction. Source and dependency inspection give a high-confidence current-behavior check: the remote path starts paste input immediately and the helper exposes only an internal browser/manual race, not a callback-settled event.

## Solution Assessment

Unclear until maintainers choose the contract shape. The requested direction is reasonable, but the maintainable solution should keep OpenAI-specific OAuth behavior provider-owned and add any shared prompt-cancellation API intentionally.
