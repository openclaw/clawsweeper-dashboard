## Summary

Keep open: current main still has only the copy clarification and local fallback suppression; it does not expose the requested provider-owned browser-callback-settled signal or cancellable manual-prompt contract. The prior ClawSweeper comment remains consistent with current source, and the related merged wording PR explicitly left OAuth lifecycle and prompt cancellation unchanged.

## What This Changes

The issue requests a callback-settled signal for remote OpenAI Codex OAuth so OpenClaw can dismiss a visible manual paste prompt after browser callback completion.

## Best Possible Solution

Add a provider-owned callback-settled or cancellable manual-input contract, keep the remote paste prompt immediate, align the core wrapper and bundled OpenAI runtime, and cover both remote completion paths with focused tests or real-flow proof.

## Reproduction Assessment

Not applicable as a failing bug. Source inspection on current main gives a high-confidence behavior check: the remote path starts paste input immediately and the published helper only exposes a browser/manual race, not the requested settled signal.

## Solution Assessment

Unclear until maintainers choose the contract shape. The direction is reasonable, but the maintainable solution should keep OpenAI-specific OAuth behavior provider-owned and avoid inferring success from token-exchange traffic.
