## Summary

Keep open: current main and v2026.6.6 still lack a provider-owned callback-settled or prompt-cancellation signal for the remote OpenAI ChatGPT/Codex OAuth paste prompt, and the closest open implementation candidate still needs maintainer decisions before this issue can close.

## What This Changes

The issue requests an explicit callback-settled signal so remote OpenAI ChatGPT/Codex OAuth can dismiss manual paste input when the browser callback completes first.

## Best Possible Solution

Adopt a provider-owned callback/prompt cancellation contract that keeps immediate remote paste input, dismisses it when browser callback completion wins, and covers both paste-only and SSH-tunneled paths with focused tests or live proof.

## Reproduction Assessment

Yes for the current behavior by source inspection: remote OAuth still starts or reuses a manual prompt without a callback-settled cancellation signal. No live remote SSH-tunnel OAuth replay was run in this read-only review.

## Solution Assessment

Unclear as a final implementation: the requested provider-owned signal is the right direction, but maintainers still need to accept the prompt-contract shape and the tradeoffs in https://github.com/openclaw/openclaw/pull/89491 before this can close.
