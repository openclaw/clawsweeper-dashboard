## Summary

Current main still appears to have the reported Control UI config discoverability regression: Advanced Settings renders through the grouped accordion path, that path still lacks the visible search control, and the form renderer still filters to the active section before applying search.

## What This Changes

Restore v1-level discoverability in the v2 Config UI by making Advanced Settings searchable/filterable without forcing users through isolated grouped sections.

## Best Possible Solution

Advanced Settings should retain any intended v2 grouping, but expose search/filter controls in accordion mode and let nonempty searches find matching settings across the included Advanced Settings sections.

## Reproduction Assessment

Yes. Static inspection against current main reproduces the issue: Advanced Settings selects accordion layout, the accordion branch does not render the search input, and form filtering applies activeSection before search matching.

## Solution Assessment

Yes for a narrow repair. Restoring accordion-mode search/filter and making search global across Advanced Settings is smaller and safer than reverting the whole v2 settings redesign.

## Repair Work Prompt

Fix the Control UI config discoverability regression reported in openclaw/openclaw#45688. Current main renders Advanced Settings with `settingsLayout: "accordion"`, but the `Search settings` control only exists in the non-accordion branch and `renderConfigForm()` filters by `activeSection` before applying search. Implement a narrow repair: expose the existing config search/tag query UI in accordion layout, and when Advanced Settings has a nonempty query, allow matches across the included Advanced Settings sections instead of only the currently active accordion section. Preserve scoped settings panels such as Communication, Appearance, Automation, Infrastructure, and AI Agents so their search behavior remains appropriately scoped. Add focused tests for accordion-mode search visibility/clear behavior and cross-section text/tag search matching. Add one single-line `CHANGELOG.md` fix entry because this is user-facing. Do not revert the v2 Quick Settings/accordion redesign and do not change config schema or runtime gateway behavior.
