## Summary

Keep open: current main still has the reported Control UI config usability regression. Advanced Settings still uses the grouped accordion path, that accordion branch still omits the visible settings search control, and the form renderer still applies active-section scoping before search matching.

## What This Changes

The issue asks to restore v1-level Config usability in the v2 Control UI, especially a clear searchable/filterable settings list instead of only grouped top-level sections.

## Best Possible Solution

Advanced Settings should keep the v2 grouping unless maintainers choose otherwise, but regain discoverability by exposing search/filter controls in accordion mode and allowing nonempty searches to find matching settings across the included Advanced Settings sections.

## Reproduction Assessment

Yes. Static inspection against current main reproduces the gap: Advanced Settings selects accordion layout, the accordion branch lacks the search input, and form filtering applies activeSection before search matching.

## Solution Assessment

Yes for a narrow repair. Restoring accordion-mode search/filter and making search global across the Advanced Settings section set is the smallest maintainable fix; reverting the whole v2 settings redesign should remain a separate product decision.
