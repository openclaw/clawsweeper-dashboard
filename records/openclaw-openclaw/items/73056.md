## Summary

Keep open. Current main still has the raw-session outer idle guard before compaction, and manual `/compact` still does not pass observed token usage into that path; open PR #73092 is already the linked implementation candidate.

## What This Changes

The issue reports that manual `/compact` can skip a tool-heavy Discord thread session as idle when the limited raw message tail lacks a nearby user turn even though the session is over the compaction token threshold.

## Best Possible Solution

Land PR #73092 or an equivalent patch that preserves the empty-session skip while letting over-threshold manual compaction proceed using observed token usage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a manual `/compact` call without `currentTokenCount` can reach the outer guard after history limiting, and a tool-heavy tail whose meaningful user turn is outside the 20-message lookback is rejected before compaction runs.

## Solution Assessment

Yes. The narrow maintainable solution is to pass fresh manual token usage and bypass only the outer idle skip when the session is over the configured compaction threshold, rather than weakening the shared real-conversation classifier.
