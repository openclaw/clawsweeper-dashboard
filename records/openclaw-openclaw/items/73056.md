## Summary

Keep open. Current main still has the raw-session outer idle pre-check before compaction, and open PR #73092 is already the linked implementation candidate for this issue.

## What This Changes

Fix manual `/compact` so over-threshold tool-heavy sessions are not skipped as idle by the outer raw-message compaction pre-check.

## Best Possible Solution

Land PR #73092 or an equivalent threshold-gated fix that preserves the empty-session skip while allowing over-threshold manual compaction to proceed.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a limited `session.messages` slice with heartbeat/tool-call/tool-result entries, or tool results whose meaningful user ask is outside the 20-message lookback, combined with over-threshold token usage on manual `/compact`.

## Solution Assessment

Yes. The narrowest maintainable direction is to propagate fresh manual token usage and bypass only the outer idle skip when the session is over threshold, rather than weakening the shared real-conversation classifier.
