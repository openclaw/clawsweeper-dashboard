## Summary

Keep open. Current main still has the outer raw-session compaction pre-check that can return `no real conversation messages` before Pi SDK preparation can consider turn-prefix context, and open PR #73092 is the active unmerged implementation candidate.

## What This Changes

Manual `/compact` should not classify tool-heavy, over-threshold Discord sessions as idle when the limited raw message slice lacks a recent user turn.

## Best Possible Solution

Keep the idle-session skip for genuinely empty below-threshold sessions, but let over-threshold manual compaction proceed past the outer raw-message guard using fresh observed token usage.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a limited raw message slice containing heartbeat/tool-result entries with the meaningful user turn outside the 20-message lookback, plus an over-threshold manual compaction request.

## Solution Assessment

Yes. The threshold-bypass plus manual token propagation described by PR #73092 is the narrowest maintainable fix path; the issue should remain open until that PR or an equivalent patch lands.
