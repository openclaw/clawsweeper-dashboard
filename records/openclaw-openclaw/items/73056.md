## Summary

Keep open because current main still has the raw-session outer idle guard that can return `no real conversation messages` before compaction, and open PR #73092 is already the linked implementation candidate.

## What This Changes

The issue requests fixing manual `/compact` so tool-heavy over-threshold sessions are not misclassified as idle by the outer compaction pre-check.

## Best Possible Solution

Land PR #73092 or an equivalent patch that propagates observed manual token usage and bypasses only the over-threshold outer idle skip while preserving the empty-session guard.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a limited `session.messages` slice containing heartbeat/tool-result entries, or tool results whose meaningful user ask is outside the 20-message lookback, still reaches the outer `no real conversation messages` return before compaction.

## Solution Assessment

Yes. The narrow maintainable solution is to pass fresh manual token usage into compaction and allow over-threshold sessions past only the outer idle skip, instead of weakening the shared real-conversation classifier.
