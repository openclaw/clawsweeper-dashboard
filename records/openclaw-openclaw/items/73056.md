## Summary

Keep open. Current main still has the raw-session outer compaction pre-check that can return `no real conversation messages` before Pi SDK preparation can account for turn-prefix context, and open PR #73092 is the active implementation candidate rather than a shipped mainline fix.

## What This Changes

The issue requests that manual `/compact` avoid misclassifying over-threshold, tool-heavy Discord thread sessions as idle when the raw trimmed message slice lacks a recent meaningful user message.

## Best Possible Solution

Keep the idle-session skip for genuinely empty below-threshold sessions, but let over-threshold manual compaction proceed past the outer raw-message guard by propagating fresh observed token usage and comparing it against the active caller/session budget; review and land PR #73092 or an equivalent narrow fix.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a limited raw message slice containing only heartbeat/tool-result entries with the meaningful user turn outside the 20-message lookback, plus an over-threshold token count; current main can return the outer `no real conversation messages` skip before Pi SDK preparation can add prefix context.

## Solution Assessment

Yes for the requested direction, with one caveat. The threshold-bypass plus manual token propagation is the narrow maintainable fix, but the active PR still needs maintainer review fixes before merge.
