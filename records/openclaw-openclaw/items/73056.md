## Summary

Keep open. Current main still has the raw-session outer compaction pre-check that can return `no real conversation messages` before Pi SDK preparation can account for turn-prefix context, and open PR #73092 is still the active unmerged implementation candidate.

## What This Changes

The issue requests that manual `/compact` avoid misclassifying over-threshold, tool-heavy Discord thread sessions as idle when the trimmed raw message slice lacks a recent meaningful user message.

## Best Possible Solution

Keep the idle-session skip for genuinely empty below-threshold sessions, but let over-threshold manual compaction proceed past the outer raw-message guard by propagating fresh observed token usage and comparing it against the active caller/session budget.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a limited raw message slice containing only heartbeat/tool-result entries with the meaningful user turn outside the 20-message lookback, plus an over-threshold token count; current main can return the outer `no real conversation messages` skip before Pi SDK preparation can add prefix context.

## Solution Assessment

Yes for the requested direction. The threshold-bypass plus manual token propagation is the narrow maintainable fix, with PR #73092 or an equivalent reviewed patch as the implementation path.
