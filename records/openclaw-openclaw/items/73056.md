## Summary

Keep open. Current main still has the raw-session outer idle guard that can return `no real conversation messages` before compaction, and the linked implementation PR #73092 remains open rather than merged.

## What This Changes

Manual `/compact` should not classify an over-threshold, tool-heavy Discord-thread session as idle merely because the limited raw message slice lacks recent meaningful user text.

## Best Possible Solution

Land PR #73092 or an equivalent patch that preserves the empty-session skip while allowing over-threshold manual compaction to proceed using observed token usage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: manual `/compact` can pass a limited tool-heavy message slice into the outer guard without `currentTokenCount`, and that guard returns before compaction when the recent slice lacks a meaningful user message.

## Solution Assessment

Yes. The narrow maintainable solution is to propagate fresh manual token usage and bypass only the outer idle skip when the session is over threshold, instead of weakening the shared real-conversation classifier.
