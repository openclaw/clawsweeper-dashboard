## Summary

Keep open. Current main still has the outer raw-session idle guard before compaction, manual `/compact` still does not pass observed token pressure into that guard, and the linked implementation PR remains open with closing syntax.

## What This Changes

Manual `/compact` on long tool-heavy sessions can be misclassified as idle by the outer compaction pre-check before Pi preparation context is available.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/73092 or an equivalent narrow fix that forwards manual token pressure and bypasses only the outer idle skip for over-threshold sessions.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a limited `session.messages` slice with heartbeat/tool-result entries, or tool results whose meaningful user ask is outside the 20-message lookback, still reaches the outer `no real conversation messages` return before compaction.

## Solution Assessment

Yes for the linked direction. The narrow maintainable fix is to propagate fresh manual token usage and bypass only the outer idle skip when the session is over the configured compaction threshold, rather than weakening the shared real-conversation classifier.
