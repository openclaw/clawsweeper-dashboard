## Summary

Keep open. Current main still has the outer idle guard and the manual command path does not supply the token data needed to bypass it; linked open PR #73092 is the implementation candidate.

## What This Changes

Manual `/compact` should not skip over-threshold, tool-heavy sessions as idle when the trimmed raw session tail lacks a recent meaningful user message.

## Best Possible Solution

Land PR #73092 or an equivalent narrow patch that preserves the empty-session skip while allowing over-threshold manual compaction to proceed using observed token usage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a limited `session.messages` slice with heartbeat/tool-result entries, or tool results whose meaningful user ask is outside the 20-message lookback, still reaches the outer `no real conversation messages` return before compaction.

## Solution Assessment

Yes. The narrow maintainable solution is to propagate fresh manual token usage and bypass only the outer idle skip when the session is over the configured compaction threshold, rather than weakening the shared real-conversation classifier.
