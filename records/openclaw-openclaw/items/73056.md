## Summary

Keep open. Current main still has the raw-session outer compaction pre-check that can skip before compaction when the limited session slice is tool-heavy, and open PR #73092 is the active unmerged implementation candidate.

## What This Changes

Manual `/compact` should not classify over-threshold, tool-heavy sessions as idle when the limited raw message window lacks a recent user turn.

## Best Possible Solution

Land PR #73092 or an equivalent patch that preserves the idle-session skip for genuinely empty sessions while allowing over-threshold manual compaction to proceed past the outer raw-message guard using fresh observed token usage.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a limited `session.messages` slice containing only heartbeat/tool-result entries or tool results whose meaningful user ask is outside the 20-message lookback, plus an over-threshold context count; current main can return `no real conversation messages` before compaction starts.

## Solution Assessment

Yes for the requested direction. The narrow maintainable fix is to propagate manual token usage and bypass the outer idle skip only when the session is over the active compaction threshold, rather than broadly relaxing message-classification heuristics.
