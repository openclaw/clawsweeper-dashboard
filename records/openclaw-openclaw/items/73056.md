## Summary

Keep open. Current main still has the raw-session outer compaction pre-check that can return `no real conversation messages` before the Pi SDK preparation path can see turn-prefix context, and open PR #73092 is already the linked implementation candidate for this issue.

## What This Changes

The issue asks manual `/compact` to stop treating over-threshold, tool-heavy Discord thread sessions as idle when the limited raw message window lacks a recent meaningful user message.

## Best Possible Solution

Land PR #73092 or an equivalent narrow fix that keeps the idle-session skip for genuinely empty below-threshold sessions, passes fresh token usage into manual `/compact`, and lets over-threshold sessions proceed past the outer raw-message pre-check.

## Reproduction Assessment

Yes. A source-level reproduction is a limited raw message slice containing only heartbeat/tool-result entries with the meaningful user turn outside the 20-message lookback, plus an over-threshold session; current main returns the outer `no real conversation messages` skip before Pi SDK preparation can add prefix context.

## Solution Assessment

Yes. The threshold-bypass direction is the narrowest maintainable fix visible from current code because it preserves the idle-session optimization and avoids changing the shared real-conversation classifier.
