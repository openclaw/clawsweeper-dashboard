## Summary

Keep open. Current main still has the raw-session outer compaction pre-check that can return `no real conversation messages` before Pi SDK preparation sees split-turn prefix context, and the linked implementation PR #73092 is open/unmerged rather than a shipped fix.

## What This Changes

The issue asks for manual `/compact` to avoid treating over-threshold, tool-heavy Discord thread sessions as idle when the trimmed raw message slice lacks a recent meaningful user message.

## Best Possible Solution

Keep the idle-session skip for genuinely empty below-threshold sessions, but pass fresh observed token usage into manual `/compact` and allow over-threshold sessions past the outer raw-message pre-check; review and land PR #73092 or an equivalent narrow fix.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a limited raw message slice containing only heartbeat/tool-result entries with the meaningful user turn outside the 20-message lookback, plus an over-threshold token count; current main returns the outer `no real conversation messages` skip before Pi SDK preparation can add prefix context.

## Solution Assessment

Yes. The threshold-bypass direction in the linked PR is the narrowest maintainable fix visible from current code because it preserves the idle-session optimization and avoids changing the shared real-conversation classifier.
