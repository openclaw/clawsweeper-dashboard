## Summary

Keep open. Current main still has the raw-session outer compaction pre-check that can return `no real conversation messages` before Pi SDK preparation can account for turn-prefix context, and open PR #73092 is the active implementation candidate rather than a shipped mainline fix.

## What This Changes

The issue requests that manual `/compact` avoid misclassifying over-threshold, tool-heavy sessions as idle when the raw limited message slice lacks a recent meaningful user message.

## Best Possible Solution

Keep the idle-session skip for genuinely empty below-threshold sessions, but let over-threshold manual compaction proceed past the outer raw-message guard by propagating observed token usage and reviewing or landing PR #73092 or an equivalent narrow fix.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a limited raw message slice containing only heartbeat/tool-result entries with the meaningful user turn outside the 20-message lookback, plus an over-threshold token count; current main returns the outer `no real conversation messages` skip before Pi SDK preparation can add prefix context.

## Solution Assessment

Yes. The threshold-bypass direction is the narrowest maintainable fix visible from current code because it preserves the empty-session optimization and avoids changing the shared real-conversation classifier.
