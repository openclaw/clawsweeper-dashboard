## Summary

Keep open. Current main still has the outer raw-session idle guard before compaction, manual `/compact` still does not pass observed token pressure into that guard, and https://github.com/openclaw/openclaw/pull/73092 remains the open implementation candidate linked with closing syntax.

## What This Changes

Manual `/compact` should not classify a long over-threshold, tool-heavy Discord-thread session as idle when the trimmed raw message slice lacks a recent user anchor.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73092 or an equivalent narrow patch that forwards observed manual token usage and bypasses only the outer idle skip when the session is over threshold.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a limited `session.messages` slice with heartbeat/tool-result entries, or tool results whose meaningful user ask is outside the 20-message lookback, still reaches the outer `no real conversation messages` return before compaction.

## Solution Assessment

Yes for the linked direction. The narrow maintainable fix is to propagate fresh manual token usage and bypass only the outer idle skip when the session is over the configured compaction threshold, instead of weakening the shared real-conversation classifier.
