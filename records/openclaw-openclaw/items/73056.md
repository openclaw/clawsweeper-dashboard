## Summary

Keep open. Current main still has the raw-session outer compaction pre-check that can return `no real conversation messages` after history limiting and before Pi SDK preparation can account for turn-prefix context, while the linked fix PR remains open and unmerged.

## What This Changes

Manual `/compact` should not classify over-threshold tool-heavy sessions as idle before compaction can use the fuller session context.

## Best Possible Solution

Keep the idle-session skip for genuinely empty below-threshold sessions, but propagate observed manual token usage and bypass only the raw-message guard when active usage is already over the compaction threshold.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a history-limited raw message slice containing heartbeat/tool-result entries with the meaningful user turn outside the 20-message lookback, plus an over-threshold context count on manual `/compact`.

## Solution Assessment

Yes. The threshold-bypass plus manual token propagation is the narrow maintainable fix because it preserves the idle-session optimization while avoiding an over-threshold false idle decision.
