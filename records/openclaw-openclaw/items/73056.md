## Summary

Keep open. Current main still has the outer raw-session idle guard before compaction, the tool-result classifier still fails a tool-heavy tail without a nearby real user anchor, and the focused implementation remains in an open PR rather than merged on main.

## What This Changes

The issue reports that manual `/compact` can skip long, tool-heavy Discord sessions as idle when the outer compaction pre-check sees only a trimmed raw session tail.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/73092 or an equivalent narrow fix that forwards manual token pressure and bypasses only the outer idle skip for over-threshold sessions while preserving the empty-session skip.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: a limited `session.messages` slice of heartbeat/tool-result entries, or tool results whose real ask is outside the 20-message lookback, still reaches the outer `no real conversation messages` return before compaction. I did not replay a live Discord-thread transcript.

## Solution Assessment

Yes for the linked direction. A threshold-aware bypass on the outer idle skip is narrower than weakening the shared real-conversation classifier and preserves the intended empty-session protection.
