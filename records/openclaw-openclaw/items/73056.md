## Summary

Keep open. Current main still has the outer raw-session idle guard before compaction, manual `/compact` still does not pass observed token usage into that guard, and open PR #73092 is already the linked implementation candidate.

## What This Changes

Manual `/compact` should not skip over-threshold, tool-heavy Discord thread sessions as idle when the limited raw transcript tail lacks a recent user message.

## Best Possible Solution

Review and land PR #73092 or an equivalent narrow patch that keeps the empty-session skip while allowing over-threshold manual compaction to proceed using observed token usage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a limited `session.messages` slice containing heartbeat/tool-result entries, or tool results whose meaningful user ask is outside the 20-message lookback, still reaches the outer `no real conversation messages` return before compaction.

## Solution Assessment

Yes. The narrow maintainable fix is to propagate fresh manual token usage and bypass only the outer idle skip when the session is over the configured compaction threshold, rather than weakening the shared real-conversation classifier.
