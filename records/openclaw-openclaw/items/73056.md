## Summary

Keep open. Current main still runs the raw-session outer idle guard before compaction and manual `/compact` still does not pass observed token usage into that guard; open PR #73092 is already the linked implementation candidate.

## What This Changes

The issue asks for manual `/compact` to avoid treating an over-threshold, tool-heavy session tail as idle before compaction starts.

## Best Possible Solution

Land PR #73092 or an equivalent patch that preserves the empty-session skip while allowing over-threshold manual compaction to proceed using observed token usage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a limited `session.messages` slice with heartbeat/tool-result entries, or tool results whose meaningful user ask is outside the 20-message lookback, still reaches the outer `no real conversation messages` return before compaction.

## Solution Assessment

Yes. The narrow maintainable solution is to propagate fresh manual token usage and bypass only the outer idle skip when the session is over the configured compaction threshold, instead of weakening the shared real-conversation classifier.
