## Summary

Keep open. Current main still has the outer raw-session no-real-conversation guard before Pi preparation, manual `/compact` still does not pass observed token usage, and open PR #73092 remains the linked unmerged implementation path.

## What This Changes

The issue requests fixing manual compaction so tool-heavy over-threshold sessions are not misclassified as idle by the outer compaction pre-check.

## Best Possible Solution

Preserve the idle-session skip for genuinely empty sessions, but pass fresh token usage into manual compaction and let over-threshold sessions proceed past the outer raw-message guard with focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a limited raw message slice containing heartbeat or tool-result entries with the meaningful user turn outside the 20-entry lookback, combined with an over-threshold token count on the manual `/compact` path.

## Solution Assessment

Yes for the requested direction. Bypassing the outer idle guard only when observed usage is over the active budget is narrower than changing message-classification heuristics, and PR #73092 or an equivalent reviewed patch is the right implementation path.
