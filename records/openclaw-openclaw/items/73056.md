## Summary

Keep open. Current main still contains the raw-session outer compaction pre-check that can classify a limited tool-heavy session slice as having no real conversation before compaction starts, and open PR #73092 is already the linked implementation candidate.

## What This Changes

The issue requests that manual compaction for over-threshold, tool-heavy sessions not be short-circuited as idle by the outer raw-message pre-check.

## Best Possible Solution

Land PR #73092 or an equivalent patch that preserves the empty-session skip while allowing over-threshold manual compaction to proceed past the outer guard using fresh observed token usage.

## Reproduction Assessment

Yes. A source-level reproduction is a limited `session.messages` slice containing heartbeat/tool-result entries, or tool results whose meaningful user ask is outside the 20-message lookback, combined with over-threshold context usage; current main can return `no real conversation messages` before compaction starts.

## Solution Assessment

Yes. The narrow maintainable fix is to propagate manual token usage and bypass the outer idle skip only when the session is over the configured compaction threshold, instead of weakening the shared real-conversation classifier.
