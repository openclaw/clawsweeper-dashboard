## Summary

This PR is still useful and not implemented on current main: current main still returns the user-scope systemd unit before considering a same-name system unit. The patch looks coherent and proof-backed, but it intentionally changes gateway service ownership and now overlaps with the open doctor-gated alternative at https://github.com/openclaw/openclaw/pull/91221, so the remaining action is maintainer policy review rather than cleanup closure.

## What This Changes

The branch changes Linux systemd gateway owner selection, user-install handoff, restart/stop conflict retirement, and regression tests for same-name user/system unit conflicts.

## Best Possible Solution

Land one canonical reviewed systemd conflict policy: this PR is suitable if maintainers want automatic lifecycle-bound conflict retirement, while https://github.com/openclaw/openclaw/pull/91221 is the alternative if they prefer doctor-gated cleanup.
