## Summary

## What This Changes

The PR updates `src/daemon/systemd.ts` and `src/daemon/systemd.test.ts` to select a single owner for same-name user/system gateway units, retire live opposite-scope conflicts, and cover the handoff cases with regression tests.

## Best Possible Solution

Merge the focused systemd fix after normal CI if maintainers accept the single-owner same-name unit policy and its operator-visible service retirement behavior.

## Reproduction Assessment

Yes, source inspection shows current main prefers the user unit when both same-name scopes exist, and the linked issue plus PR proof describe the resulting dueling-service restart loop. I did not run live systemd locally in this read-only review.

## Solution Assessment

Yes, this is the right layer: the fix centralizes ownership in `findInstalledSystemdGatewayScope()` and handles retirement at install/control boundaries instead of changing Telegram handling or stale-PID detection.
