## Summary

Keep the PR open. The model input/output capture path and real OTEL proof look solid, but the patch currently exposes `inputMessages`/`outputMessages` as public diagnostic event fields while the implementation intentionally keeps that content in trusted privateData, so the SDK/security contract needs correction before merge.

## What This Changes

The PR wires opt-in model input/output content capture from model-call diagnostics into diagnostics-otel attributes through trusted diagnostic privateData, adds a shared capture-content resolver export, and expands focused tests.

## Best Possible Solution

Remove or re-scope the public model-call event fields so the SDK contract matches the privateData-only implementation, then land the opt-in model input/output capture if maintainers accept the raw-content telemetry boundary.

## Reproduction Assessment

Yes. Current main's model-call wrapper lacks final payload and normalized text-delta content capture, while the documented `captureContent` contract expects OTEL content attributes when explicitly enabled.

## Solution Assessment

Mostly yes, but not merge-ready as written. Capturing final provider payloads at `onPayload` and routing raw content through trusted privateData is the right owner-boundary fix for model input/output, but the public event payload fields must not contradict that privateData design.
