## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Capture input only after final onPayload replacement:** `src/agents/embedded-agent-runner/run/attempt.model-diagnostic-events.ts:611`
  - body: This records `inputMessages` from the original request before the caller/provider `onPayload` hook gets a chance to replace it. If that hook returns a sanitized payload with no extractable messages, the second `observeInputMessages(replacement ?? payload)` call does not clear the already stored raw input, so `privateData.modelContent` can still carry the original prompt into diagnostics-otel when `captureContent.inputMessages` is enabled. Capture from the final payload only, or clear the previous `inputMessages` when the final payload has none, and add a regression for a replacement that removes messages.
  - confidence: 0.95
- **[P1] Preserve the public capture policy shape:** `src/infra/diagnostic-llm-content.ts:8`
  - body: `DiagnosticModelContentCapturePolicy` is exported through `openclaw/plugin-sdk/diagnostic-runtime`, so adding a required `logBodies` property breaks third-party plugins that construct this public SDK type with the current fields. Keep log-body state in an internal OTEL policy, make the new member non-breaking, or add an explicit compatibility/deprecation path before updating the API baseline.
  - confidence: 0.9
