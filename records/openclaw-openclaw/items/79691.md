## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Capture input only after the final payload is chosen:** `src/agents/embedded-agent-runner/run/attempt.model-diagnostic-events.ts:632`
  - body: The wrapper stores `inputMessages` from the original payload before the caller's `onPayload` can return a sanitized replacement. If that hook replaces the payload with `{}` or `messages: []`, the later observe call does not clear `state.modelContent.inputMessages`, so the original raw prompt can still be exported through trusted diagnostic private data when `captureContent.inputMessages` is enabled. Capture only from the final payload or clear stored input when the final payload has no capturable messages.
  - confidence: 0.92
- **[P1] Keep the capture policy type source-compatible:** `src/infra/diagnostic-llm-content.ts:16`
  - body: `DiagnosticModelContentCapturePolicy` is exported through `openclaw/plugin-sdk/diagnostic-runtime` in v2026.6.1 without `logBodies`. Making `logBodies` required means third-party plugins that construct the shipped policy type stop compiling even though this PR only needs an OTEL log-body flag, so keep that field optional/internal or use an OTEL-local extended type.
  - confidence: 0.9
