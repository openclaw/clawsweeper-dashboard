This PR adds policy-gated `inputMessages`/`outputMessages` population for model-call diagnostic events, centralizes the OTEL content-capture policy helper, exports it through the diagnostic runtime SDK seam, refreshes the SDK baseline hash, and adds focused tests.

## Best Possible Solution

Land the focused model-call content-capture fix after maintainer privacy/scope review, and keep https://github.com/openclaw/openclaw/issues/77391 open or split until tool input/output capture is fixed or deliberately descoped.

Yes. The bug is source-reproducible on current main: docs/config and diagnostics-otel expect model content fields, while the current model-call event type and emitter do not populate them.

## Solution Assessment

Yes for the PR's model-call scope. Capturing final request content at `onPayload`, reading normalized output chunks, and gating emission through shared capture policy is the narrow maintainable path; the broader tool-content issue should remain tracked separately.
