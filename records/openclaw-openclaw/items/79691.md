Full review comments:

- **[P1] Capture only the final onPayload request body:** `src/agents/embedded-agent-runner/run/attempt.model-diagnostic-events.ts:610`
  - body: This records `inputMessages` from the original payload before `originalOnPayload` runs. Provider streams use the hook result as the final request body when it is not `undefined`, so a hook that returns a sanitized body with no extractable messages can still leave the original raw prompt in diagnostic privateData and export it through OTEL. Move the capture to the final `replacement ?? payload` decision and add a regression where the replacement strips messages.
  - confidence: 0.88
- **[P1] Preserve the public capture policy shape:** `src/infra/diagnostic-llm-content.ts:8`
  - body: `DiagnosticModelContentCapturePolicy` is re-exported from `openclaw/plugin-sdk/diagnostic-runtime`, so adding required `logBodies` changes the public structural type and can break existing plugins that construct this policy. Keep log-body state in an internal/local policy extension or make the new field non-breaking, then refresh the SDK baseline accordingly.
  - confidence: 0.86
