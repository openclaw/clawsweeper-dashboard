The PR adds a `doctor:node-runtime` health contribution with Node runtime diagnostics, advisory warnings, unit tests, and Doctor contribution registration.

## Best Possible Solution

Keep the Doctor contribution direction, repair advisory coverage and path display using current shared contracts, sync the minimum-version text, then require redacted real `openclaw doctor` output before landing.

## Reproduction Assessment

Yes, source-reproducible. On the PR head, `buildNodeRuntimeWarnings()` only knows Node 22 while current main accepts all Node versions >=22.16.0, and `buildNodeRuntimeSummary()` only shortens `/` home paths.

## Solution Assessment

No. The Doctor contribution is the right surface, but the implementation should reuse `shortenHomePath()`, cover every runnable release line that needs advisory output, keep minimum-version text in sync, and include real behavior proof.
