## Summary

Keep open. Current main still lacks this Doctor runtime contribution, and the PR remains useful, but the patch is not ready because its advisory table omits Node release lines that pass OpenClaw's runtime gate, its home-path shortening still misses Windows backslash paths, and the user-facing Doctor change lacks the required changelog entry. The latest PR commit did address the earlier POSIX home-prefix and comment-year feedback.

## What This Changes

The PR adds a `doctor:node-runtime` health contribution, diagnostics helper/tests, and registration in `resolveDoctorHealthContributions()`.

## Best Possible Solution

Land the Doctor runtime diagnostic with complete advisory coverage for Node release lines that OpenClaw can run on, reuse the shared path display helper, add regression tests for the missed cases, and document the user-facing change in the active changelog.

## Reproduction Assessment

Yes. By inspection, `buildNodeRuntimeWarnings()` returns no advisory for Node 23 or Node 25 because the schedule table only includes Node 22, and `buildNodeRuntimeSummary()` cannot shorten `C:\Users\...` paths because it only checks a `/` boundary.

## Solution Assessment

No. The Doctor contribution is the right surface, but the implementation should avoid duplicating path-shortening logic and should make the release-line advisory data cover every Node major that can pass the runtime gate.
