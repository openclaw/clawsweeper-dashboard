## Summary

Keep open. Current main still lacks the Node runtime Doctor contribution, and the proposed patch is useful but has merge-blocking review findings: the advisory schedule omits accepted Node lines that are already in maintenance or EOL, and the path shortening misses Windows home paths in support-ready Doctor output.

## What This Changes

The PR adds a `doctor:node-runtime` health contribution that prints Node version, executable path, version-manager diagnostics, EOL/recommended-version advisories, tests, and Doctor registration.

## Best Possible Solution

Keep the feature direction, but cover all Node majors that can pass OpenClaw's runtime guard and are known EOL or in maintenance, reuse the shared home-path display helper, add regression tests for Node 23/25 and Windows backslash paths, and include the required user-facing changelog entry.

## Reproduction Assessment

Yes. `buildNodeRuntimeWarnings()` with Node 23 after 2025-06-01, or Node 25 on 2026-05-01, returns no advisory because the schedule only includes Node 22; `buildNodeRuntimeSummary()` with `USERPROFILE=C:\Users\Alice` and a backslash `execPath` does not shorten the home path.

## Solution Assessment

No. Adding a Doctor runtime contribution is the right surface, but the current implementation should use the existing path-shortening helper and complete the release-line advisory logic before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Warn for non-LTS Node lines that pass startup:** `src/commands/doctor-node-runtime.ts:129-136`
  - body: `runtimeSatisfies()` accepts any Node version >=22.14, so users on Node 23 or 25 reach this Doctor check. The upstream schedule marks Node 23 EOL on 2025-06-01 and Node 25 in maintenance with EOL on 2026-06-01, but this table only includes major 22, so those runtimes get no EOL or maintenance advisory. Please add the known non-LTS lines or make the warning logic data-driven before merge.
  - confidence: 0.91
- **[P2] Handle Windows home paths when shortening execPath:** `src/commands/doctor-node-runtime.ts:242-243`
  - body: On Windows, `USERPROFILE` and `process.execPath` commonly use `\`, but this check only accepts `${homeDir}/`. A version-managed Node path under `C:\Users\...` will print the full username in Doctor output users paste into issues; reuse `shortenHomePath()` or accept a backslash boundary too.
  - confidence: 0.88
- **[P3] Add the required changelog entry:** `src/flows/doctor-health-contributions.ts:620-625`
  - body: This registers new user-facing `openclaw doctor` output, but the diff does not include a `CHANGELOG.md` entry. Repo policy requires user-facing `feat` changes to add a single-line changelog bullet with valid contributor credit.
  - confidence: 0.83
