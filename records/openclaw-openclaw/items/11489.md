## Summary

Keep open. Current main still limits group admission to open, disabled, or allowlist behavior; DM pairing does not authorize groups, and no landed related item provides group approval, owner notification, or pending-group routing. The feature is directionally aligned with OpenClaw’s security and channel priorities, but its authorization, persistence, notification, and pre-approval observation semantics require maintainer product and security decisions.

## What This Changes

Add cross-channel group pairing with owner notifications, approve/reject controls, and optional restricted observation while group approval is pending.

## Best Possible Solution

Define additive group approval as a shared ingress security and persisted-state contract with authenticated owner actions, then implement channel-specific join notifications; scope silent observation or reader-agent routing as a separate explicit opt-in after its trust boundary is approved.

## Reproduction Assessment

Not applicable: current main consistently documents and implements DM-only pairing, so this is a missing capability rather than broken existing behavior.

## Solution Assessment

No, not as one combined change. Shared group approval is a plausible direction, but pre-approval reader-agent processing materially changes the trust and session-state boundary and should receive a separate product and security decision.
