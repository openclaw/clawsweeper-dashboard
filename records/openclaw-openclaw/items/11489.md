## Summary

Current main and release v2026.6.5 still intentionally limit pairing to direct messages; the requested group admission, owner notification, and restricted observation capabilities remain absent and require maintainer product and security decisions.

## What This Changes

Requests owner-notified group admission pairing plus an optional restricted observation mode for unapproved Telegram and WhatsApp groups.

## Best Possible Solution

Define an additive cross-channel group-admission contract with durable channel/account/group approval state, authorized owner notifications and approve/reject operations, then scope restricted observation as a separate opt-in capability unless maintainers explicitly combine the security models.

## Reproduction Assessment

Not applicable: this is a feature request rather than broken defined behavior. Current main and v2026.6.5 provide a high-confidence source check showing the requested group-pairing path does not exist.

## Solution Assessment

No, not yet as one combined feature. Group admission approval and pre-approval observation have distinct authorization and session-state consequences and should be designed separately unless maintainers explicitly choose a unified contract.
