## Summary

Keep open: current main and the latest shipped release still keep pairing on the DM path while group admission remains allowlist/open/disabled, and the related open items cover narrower notification or listen-only pieces rather than this owner-approved group-pairing contract.

Request adds a cross-channel group pairing approval flow with owner notification, group approve/reject controls, and optional read-only handling for unconfigured groups.

## Best Possible Solution

Define an additive cross-channel group approval contract with persisted pending state, owner-visible join notifications, authorized approve/reject operations, and channel-specific Telegram/WhatsApp wiring; treat pending read-only observation as an explicitly approved mode within or beside that contract.
