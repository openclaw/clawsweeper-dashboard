## Summary

Keep open: current main and v2026.5.12 still scope pairing to DMs, while group ingress remains allowlist/open/disabled plus mention and ambient-room behavior. Adjacent work covers listen-only plugins or WhatsApp auto-leave, but not owner-notified pending group approval, approve/reject state, or safe pending-group routing.

## What This Changes

The issue requests a cross-channel group pairing flow with owner notification, approve/reject state, and optional read-only pending-group routing.

## Best Possible Solution

Define an additive cross-channel group approval contract with explicit owner authority, persisted pending state, owner-visible join notifications, authorized approve/reject operations, and Telegram/WhatsApp implementations; keep pending read-only observation as an explicitly approved mode within or beside that contract.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken current contract. Source, docs, live issue state, related-item review, and the v2026.5.12 tag show that current group behavior is allowlist/open/disabled while pairing remains DM-only.

## Solution Assessment

Unclear until maintainers settle the product and security model. The maintainable path is a deliberate group approval contract, not reusing DM pairing state or treating ambient/listen-only group observation as a complete owner-approval flow.
