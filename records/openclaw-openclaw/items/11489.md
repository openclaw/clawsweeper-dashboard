## Summary

Keep open: current main and v2026.6.8 still keep pairing DM-only while group access remains open, disabled, or allowlist/drop behavior. The related Telegram and WhatsApp items cover adjacent warn, leave, audit, or suppression paths, but none provide the requested cross-channel pending group approval, owner notification, approve/reject, or read-only pending-group contract.

## What This Changes

Request an additive group-pairing flow that notifies the owner when a bot joins an unconfigured group and optionally keeps that group in read-only pending approval mode.

## Best Possible Solution

Define a shared additive group-approval contract with durable pending group state keyed by channel/account/group, owner-visible notifications, authorized approve/reject operations, and Telegram/WhatsApp integration; split reader-agent observation unless maintainers explicitly include it.

## Reproduction Assessment

Not applicable as a bug reproduction; source and docs provide high-confidence proof that group pairing is absent while DM pairing and group allowlist behavior are separate.

## Solution Assessment

No direct implementation path is ready yet; reusing DM pairing state for shared-room authorization would be the wrong default without a separate group approval and security contract.
