## Summary

Keep open: current main still implements DM-only pairing, while group access remains open/disabled/allowlist and the related listen-only/plugin discussion does not provide owner notification, pending group approval, or group approve/reject controls for https://github.com/openclaw/openclaw/issues/11489.

## What This Changes

Request adds group pairing with owner notification, approve/reject controls, and optional read-only pending-group routing for Telegram/WhatsApp-style groups.

## Best Possible Solution

Define an additive cross-channel group-pairing contract with persisted pending group state, owner-visible join notifications, authorized approve/reject operations, and Telegram/WhatsApp implementations; split reader-agent observation unless maintainers explicitly include it.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high confidence the capability is absent: schemas reject group pairing, docs keep pairing DM-only, and Telegram/WhatsApp group paths allow or block without pending approval.

## Solution Assessment

Unclear until maintainers settle the product and security model. The maintainable direction is a deliberate group approval contract, not a narrow bug fix or automatic repair.
