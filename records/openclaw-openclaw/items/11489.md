## Summary

Current main and release v2026.6.5 still support pairing only for direct messages; group ingress remains open, disabled, or allowlist-based, with no pending group approval, owner notification, rejection command, or restricted pending-group agent path. The request remains useful but needs explicit product and security decisions before implementation.

## What This Changes

Add owner-approved group pairing, group-join notifications, and optional restricted observation for groups awaiting approval.

## Best Possible Solution

Define an additive shared pending-group authorization contract with stable scoped identities, owner-visible join events, and authorized approve/reject operations; keep silent observation or reader-agent execution as a separately approved mode unless maintainers intentionally include it.

## Reproduction Assessment

Not applicable; this requests new group authorization, notification, and observation capabilities rather than reporting a failure of documented existing behavior. Source inspection does provide high-confidence confirmation that those capabilities are currently absent.

## Solution Assessment

No, not as one undivided feature. Shared group approval and owner notification should be designed first, while read-only agent routing should remain separately scoped unless maintainers explicitly accept its session and security implications.
