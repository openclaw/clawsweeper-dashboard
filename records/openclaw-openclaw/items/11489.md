## Summary

Keep open: current main and v2026.6.8 still implement DM-only pairing while group admission remains open, disabled, or allowlist-based; the requested owner notification, pending group approval state, group approve/reject controls, and reader-agent pending mode remain unimplemented and need product/security review.

## What This Changes

Request adds a cross-channel group pairing mode with owner notification, approve/reject controls, and optional read-only reader-agent handling for unconfigured Telegram and WhatsApp groups.

## Best Possible Solution

Define an additive cross-channel group-admission contract with durable pending approvals, owner-visible join notifications, authorized approve/reject operations, and coordinated Telegram/WhatsApp behavior; treat reader-agent observation as a separately reviewed mode unless maintainers explicitly include it.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source and release inspection confirm current behavior is still DM-only pairing plus group allowlist/open/disabled handling, without pending group approval or owner notification.

## Solution Assessment

Unclear until maintainers settle the product/security contract. The requested direction is reasonable, but the maintainable solution should separate core approval state and notifications from optional reader-agent observation unless they are deliberately designed together.
