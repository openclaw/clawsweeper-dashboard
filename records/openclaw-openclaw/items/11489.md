## Summary

Keep open: current main and v2026.6.8 still keep pairing scoped to DMs while group admission remains open, allowlist, or disabled. Related Telegram, WhatsApp, audit, and listen-only items overlap, but none has merged a cross-channel owner-notified group approval flow or pending read-only group mode.

## What This Changes

The issue requests a cross-channel group pairing mode that notifies the owner when a bot is added to an unconfigured group and optionally routes pending groups through a read-only agent.

## Best Possible Solution

Define a deliberate cross-channel group admission contract: fail closed by default, persist pending approvals by channel/account/group, send redacted owner notifications, authorize approve/reject operations, and scope read-only observation separately.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source and release inspection give high-confidence proof that current behavior is DM-only pairing plus group allowlist/open/disabled handling, without group approval state or read-only pending-group routing.

## Solution Assessment

No current implementation is the best solution yet. The maintainable path is a product/security-reviewed group admission contract rather than stretching DM pairing or merging the narrower Telegram/WhatsApp adjacent proposals independently.
