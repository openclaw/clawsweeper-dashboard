## Summary

Keep open: current main and v2026.5.12 still keep group access to open, disabled, or allowlist while pairing remains DM-only. Current ambient room events partially cover quiet context for already-allowed rooms, but they do not provide owner notification, pending group approval state, group approve/reject controls, or unconfigured WhatsApp group handling.

## What This Changes

Request to add a cross-channel group pairing flow with owner notification, approve/reject controls, and optional read-only pending-group routing.

## Best Possible Solution

Define an additive cross-channel group approval contract with explicit owner authority, persisted pending state, owner-visible join notifications, authorized approve/reject operations, and Telegram/WhatsApp implementations; keep pending read-only observation as an explicitly approved mode within or beside that contract.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken current contract. Current-main source, latest-release inspection, docs, and related live issue state show group behavior remains allowlist/open/disabled while pairing remains DM-only.

## Solution Assessment

Unclear until maintainers settle the product and security model. The maintainable path is a deliberate group approval contract, not reusing DM pairing state or treating ambient/read-only observation as the whole approval flow.
