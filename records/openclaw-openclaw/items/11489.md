## Summary

Keep open: current main and v2026.6.8 still keep pairing scoped to DMs while group admission remains open, allowlist, or disabled. Related Telegram, WhatsApp, audit, and listen-only items overlap, but none replaces the requested cross-channel owner-notified group approval flow or pending read-only group mode.

## What This Changes

The issue requests a cross-channel group pairing flow with owner notification, approve/reject operations, and read-only handling for unconfigured groups.

## Best Possible Solution

Define a deliberate cross-channel group admission contract: fail closed by default, persist pending approvals by channel/account/group, send redacted owner notifications, authorize approve/reject operations, and scope read-only observation separately.

## Reproduction Assessment

Yes at source level for the missing capability, not as a bug: current main keeps group policy to open/disabled/allowlist and keeps pairing on the DM path only.

## Solution Assessment

No current implementation is best yet. The maintainable path is a product/security-reviewed group admission contract rather than stretching DM pairing or merging the narrower Telegram/WhatsApp adjacent proposals independently.
