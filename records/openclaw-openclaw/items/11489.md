## Summary

Keep open. Current main and v2026.6.5 still keep pairing DM-only while group admission remains open, disabled, or allowlist-based, and related items only cover adjacent alert, listen-only, or auto-leave behavior rather than the owner-approved group-pairing contract.

## What This Changes

Request to add group pairing with owner notifications, approve/reject controls, and optional read-only pending-group routing for Telegram/WhatsApp-style group chats.

## Best Possible Solution

Define an additive group approval contract with persisted pending state, owner-visible join notifications, authorized approve/reject operations, and channel-specific Telegram/WhatsApp wiring; keep read-only pending observation explicitly scoped within or beside that contract.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken current contract. Source and v2026.6.5 inspection show current groups remain open/disabled/allowlist while pairing remains DM-only.

## Solution Assessment

Unclear until maintainers settle the product and security model. The maintainable path is a deliberate group approval contract, not reusing DM pairing state or treating ambient/listen-only observation as the whole approval flow.
