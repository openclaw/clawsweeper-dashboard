## Summary

Keep open: current main still models pairing as DM-only, while group access remains open, disabled, or allowlist; the related open items are adjacent and do not cover owner-approved group pairing.

## What This Changes

Requested a group pairing mode that notifies an owner when a bot is added to a group, supports approve/reject, and optionally routes pending groups through a read-only agent.

## Best Possible Solution

Define an additive cross-channel group-pairing contract with persisted pending group state, owner-visible join notifications, authorized approve/reject operations, and Telegram/WhatsApp implementations; split reader-agent observation unless maintainers explicitly choose to include it.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high confidence the capability is absent: schemas reject group pairing, docs keep pairing DM-only, and Telegram/WhatsApp group paths allow or block without pending approval.

## Solution Assessment

Yes, keeping this open is the best current action. A safe implementation needs a maintainer-approved group approval and notification contract rather than a narrow bug fix.
