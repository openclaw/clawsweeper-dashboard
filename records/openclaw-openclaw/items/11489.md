## Summary

Keep open: current main and the latest release still keep pairing scoped to DMs while group admission is open, allowlist, or disabled. The related Telegram and WhatsApp items overlap, but they do not replace the requested cross-channel owner-notified group approval flow or pending read-only mode.

## What This Changes

Requests a cross-channel group pairing flow with owner notification, group approve/reject controls, and optional read-only handling for unconfigured Telegram/WhatsApp groups.

## Best Possible Solution

Define a shared group admission contract that fails closed by default, stores pending approvals by channel/account/group, sends redacted owner notifications, authorizes approve/reject operations, and scopes reader-agent observation as a separate explicit mode.

## Reproduction Assessment

Yes at source level, not as a bug: current main keeps group policies to open/allowlist/disabled and keeps pairing on the DM path only.

## Solution Assessment

No current implementation is best yet. The maintainable path is a product/security-reviewed group admission contract rather than stretching DM pairing or merging narrower channel-specific proposals independently.
