## Summary

Current main and release v2026.6.5 intentionally keep pairing limited to direct messages; group admission, owner notification, and isolated pending-group observation remain unimplemented and require maintainer product and security decisions.

## What This Changes

Add owner-approved group admission, group-add notifications, and restricted observation for unconfigured groups.

## Best Possible Solution

Define an additive admission-state contract keyed by channel, account, and group; keep join detection and notification delivery channel-owned, preserve sender and mention authorization, and design isolated observation as a separate opt-in security mode.

## Reproduction Assessment

Not applicable; current source, tests, documentation, and release v2026.6.5 consistently define pairing for direct messages and group access through open, disabled, or allowlist policies.

## Solution Assessment

No, not as one bundled implementation; group admission, transport join notification, and untrusted observation have different ownership and security contracts and should be designed separately.
