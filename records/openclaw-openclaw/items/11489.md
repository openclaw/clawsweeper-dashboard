## Summary

Keep open: current main and v2026.5.20 still keep pairing scoped to DMs, while group admission remains open, disabled, or allowlist-only. The related open items cover narrower notification, auto-leave, or listen-only pieces, but they do not replace the requested owner-approved group pairing contract.

## What This Changes

The issue requests a cross-channel group pairing flow with owner notification, approve/reject controls, and optional read-only handling for unconfigured groups.

## Best Possible Solution

Define an additive group approval contract with persisted pending state, owner-visible join notifications, authorized approve/reject operations, and channel-specific Telegram/WhatsApp wiring; treat pending read-only observation as an explicitly approved mode within or beside that contract.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken current contract. Source, docs, the latest release tag, and related live issue state show groups remain allowlist/open/disabled while pairing remains DM-only.

## Solution Assessment

Unclear until maintainers settle the product and security model. The maintainable path is a deliberate group approval contract, not reusing DM pairing state or treating ambient/listen-only observation as the whole approval flow.
