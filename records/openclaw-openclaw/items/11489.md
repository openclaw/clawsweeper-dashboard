## Summary

Keep open: current main and v2026.5.22 still implement pairing only for DMs, while group admission remains open, disabled, or allowlist-based. Related open items cover narrower auto-leave, audit, or listen-only behavior, but none supersedes the requested owner-approved group pairing contract.

## What This Changes

Request group pairing for unconfigured Telegram/WhatsApp groups with owner notification, approve/reject controls, and optional read-only pending-group routing.

## Best Possible Solution

Define an additive group approval contract with persisted pending state, owner-visible join notifications, authorized approve/reject operations, and channel-specific Telegram/WhatsApp wiring; keep read-only pending observation explicitly scoped within or beside that contract.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken current contract. Source and latest-release inspection show current groups remain open/disabled/allowlist while pairing remains DM-only.

## Solution Assessment

Unclear until maintainers settle the product and security model. The maintainable path is a deliberate group approval contract, not reusing DM pairing state or treating ambient/listen-only observation as the whole approval flow.
