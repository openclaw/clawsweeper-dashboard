## Summary

Current main and v2026.6.6 still support pairing only for DMs; group ingress remains open, disabled, or allowlist-based, with no pending approval state or owner notification. The request remains useful and is not superseded, but it bundles cross-channel authorization, notification routing, persisted state, and optional restricted observation, so maintainers need to choose the product and security contract before implementation.

## What This Changes

Add owner-approved group pairing, group-add notifications, and an optional restricted read-only agent mode for groups awaiting configuration.

## Best Possible Solution

Define an additive cross-channel group-admission state machine keyed by channel, account, and group, with an explicitly authorized owner notification and approve/reject path; scope silent observation or a restricted reader agent as a separate opt-in design after its isolation guarantees are defined.

## Reproduction Assessment

Not applicable; this is a request for new group authorization and notification capabilities, and source plus v2026.6.6 inspection confirms those capabilities do not currently exist.

## Solution Assessment

No, not as one bundled implementation: shared group pairing and owner notification are a coherent product contract, but read-only reader-agent processing has separate isolation and session-state implications that should be designed independently.

## Risks / Open Questions

- A single combined feature could blur the boundary between group admission authorization and post-admission sandboxing, leading operators to assume that message observation is safer or more isolated than the actual agent and session contract guarantees.
- Pending approvals need a canonical persistent owner and identity key across channel, account, and group; an underspecified design could approve the wrong account or reuse stale group state.
- Owner notifications and inline approval actions require an explicit authorization target and trusted reply path so an untrusted group cannot approve itself or redirect the decision.
