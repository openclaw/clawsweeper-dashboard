## Summary

Keep open: current main still silently prefers stdio for mixed MCP `command` plus `url` configs, and this PR is a focused, correct fix. ## What This Changes

This PR adds an early MCP transport resolver guard and tests so server configs with both non-empty `command` and `url` are skipped with a clear diagnostic instead of falling through to stdio.

## Best Possible Solution

Land the focused resolver guard and regression tests after the contributor supplies redacted real behavior proof, so OpenClaw enforces one transport per MCP server at config resolution.
