## What This Changes

The PR adds a guard and tests so MCP server configs with both non-empty `command` and `url` are rejected instead of silently resolving as stdio.

## Best Possible Solution

Land the resolver-layer rejection only after redacted real OpenClaw proof and maintainer acceptance of the stricter mixed-config behavior.

## Reproduction Assessment

## Solution Assessment

Yes, pending proof and compatibility acceptance: the shared resolver is the narrowest seam where both fields are visible and callers already handle `null` as a skipped server. Supporting command-launched HTTP lifecycle would be a different feature direction, not a smaller bug fix.
