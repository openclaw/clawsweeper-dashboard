## Summary

Keep open: current main and the latest release still use stdio-first MCP transport resolution for mixed `command` plus `url` configs, and this PR is the focused same-repo candidate fix. ## What This Changes

The PR adds resolver-level rejection and tests for MCP server configs that set both `command` and a non-empty `url`.

## Best Possible Solution

Align runtime with the documented MCP config contract by rejecting mixed `command` and `url` at the resolver layer, after clarifying the `transport: "stdio"` diagnostic and getting real behavior proof plus maintainer acceptance of the fail-closed upgrade behavior.
