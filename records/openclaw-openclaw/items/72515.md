## What This Changes

The PR adds a resolver guard and tests so MCP server configs with both `command` and non-empty `url` are skipped with a diagnostic instead of silently selecting stdio.

## Best Possible Solution

Merge a resolver-level mixed `command`+`url` rejection after real behavior proof and maintainer acceptance of the compatibility scope, or replace it with an equivalent maintainer-owned guard that rejects the same ambiguous shape.

## Reproduction Assessment

## Solution Assessment

Yes for the reported embedded runtime path: `resolveMcpTransportConfig` is the narrow owner layer because it sees both fields before transport construction. The remaining question is whether maintainers want schema validation or sibling projection alignment in the same PR.
