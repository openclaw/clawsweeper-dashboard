Adds an MCP transport resolver guard that skips configs with both non-empty `command` and `url`, logs a sanitized diagnostic, and adds regression tests for mixed configs, whitespace-only URLs, and warning sanitization.

## Best Possible Solution

## Solution Assessment

Yes for the code path. The resolver is the narrowest maintainable place to reject the ambiguous shape because it sees both transport fields before selecting stdio or HTTP; the remaining gap is release-note hygiene.
