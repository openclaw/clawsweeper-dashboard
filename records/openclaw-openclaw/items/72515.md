The PR adds a `resolveMcpTransportConfig` guard that logs and skips MCP server entries with both non-empty `command` and `url`, plus four resolver tests.

## Best Possible Solution

## Solution Assessment

Yes, with merge gates. The resolver is the right layer because it sees both `command` and `url` before constructing stdio or HTTP transports; a timeout-only fix would leave the silent transport mismatch in place.
