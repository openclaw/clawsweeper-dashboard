## Summary

Keep open. Current main still records embedded-run `livenessState` in metadata and lifecycle events, but the outbound reply payload contract and delivery projections still do not carry a stable user-visible blocked/abandoned signal for channels or plain CLI delivery.

## What This Changes

The issue requests surfacing embedded-run `livenessState` in user-facing reply payloads for blocked or abandoned terminal exits.

## Best Possible Solution

Add a generic terminal-run surfacing layer so final visible payload text carries a stable blocked/abandoned state and short recovery hint, while keeping the existing metadata and lifecycle event channels intact.

## Reproduction Assessment

Yes. The existing strict-agentic blocked regression produces a payload with only blocked text and `isError: true` while `meta.livenessState` is `blocked`; following that through auto-reply and CLI delivery shows no projection from meta into visible payload text.

## Solution Assessment

Yes. The narrowest maintainable first fix is the issue's Option A: append or otherwise synthesize a stable visible footer for blocked/abandoned terminal payloads. Option B needs separate public payload/API review.
