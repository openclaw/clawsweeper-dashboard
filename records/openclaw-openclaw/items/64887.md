## Summary

Keep open. Current main still carries terminal liveness through run metadata, lifecycle events, and some gateway-facing records, but the visible reply payload path still has no stable blocked/abandoned signal.

## What This Changes

Request to surface blocked or abandoned agent-run liveness in user-facing reply payloads with regression coverage.

## Best Possible Solution

Add a generic terminal-run surfacing layer that appends a stable blocked/abandoned footer and short recovery hint to final visible payload text, while leaving envelope redesign for a separate API discussion.

## Solution Assessment

Yes. The issue's footer-style Option A is the narrowest maintainable first fix; adding a channel-aware envelope would be cleaner long term but changes the public payload contract.
