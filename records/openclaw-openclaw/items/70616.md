## Summary

Keep open. Current main and the latest shipped release partially address cheaper Dream Diary routing, but the requested direct `dreaming.execution.model` shape and configured fallback-model chain are still absent and need maintainer API judgment.

## What This Changes

The issue requests a memory-core Dreaming execution config field that supports cheaper model overrides and ordered fallback models.

## Best Possible Solution

Choose one public Dreaming execution contract, then either document the existing `dreaming.model`/`execution.defaults.model` route as canonical or implement a backward-compatible fallback-chain through schema, resolver, runtime, docs, and tests.

## Reproduction Assessment

Yes. Source inspection on current main shows the strict manifest rejects the requested direct `dreaming.execution.model` and fallback-list shape, while runtime fallback only retries with the session default.

## Solution Assessment

Unclear. Adding the exact requested field may be useful, but it overlaps with existing `dreaming.model` and `execution.defaults.model`, so maintainers should decide the stable API before implementation.
