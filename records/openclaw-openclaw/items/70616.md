## Summary

Keep open. Current main and v2026.5.7 cover the cheaper Dream Diary model path through `dreaming.model` and `dreaming.execution.defaults.model`, but the requested direct `dreaming.execution.model` shape and ordered configured fallback chain are still absent and need public config/API judgment.

## What This Changes

Request adds a Dreaming execution config field with explicit model selection and configured fallback models for background Dreaming runs.

## Best Possible Solution

Choose one canonical Dreaming model-routing contract, then either document the existing `dreaming.model`/`execution.defaults.model` path as the supported solution or add a backward-compatible fallback-chain through schema, resolver, runtime, docs, and tests.

## Reproduction Assessment

Yes. Source inspection on current main shows the strict manifest rejects the requested direct `dreaming.execution.model` and fallback-list shape, while the runtime only retries a configured model with the session default.

## Solution Assessment

Unclear. The exact requested field may be useful, but it overlaps with shipped `dreaming.model` and `dreaming.execution.defaults.model`, so maintainers should decide the stable public API before implementation.
