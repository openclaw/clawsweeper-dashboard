## Summary

Keep open. Current main still rejects Docker bind sources outside the sandbox/workspace allowlist unless the dangerous external-source override is enabled, while the custom bind docs still show arbitrary host-path examples without clearly documenting the default root constraint or the override; the open #59718 is useful adjacent least-privilege design context, not a superseding tracker for this docs/runtime mismatch.

## What This Changes

The issue requests that `sandbox.docker.binds` accept or clearly document host paths outside the per-session sandbox and agent workspace roots.

## Best Possible Solution

Maintainers should decide the Docker bind contract explicitly, then either document the default roots plus `dangerouslyAllowExternalBindSources` next to the examples, or intentionally relax configured external binds with security-focused regression coverage.

## Reproduction Assessment

Yes. Configure a Docker bind whose source is outside `workspaceDir` and `agentWorkspaceDir`, leave `dangerouslyAllowExternalBindSources` unset, and current `buildSandboxCreateArgs` reaches `validateBindMounts` with bounded roots and throws `outside allowed roots`; the inspected tests assert this path.

## Solution Assessment

Unclear. The current allowlist is a defensible safer default, so documentation clarification is likely the narrowest maintainable fix unless maintainers intentionally want configured external binds to be trusted by default.
