## Summary

Keep open. Current main and the latest release still reject configured Docker bind sources outside the runtime sandbox/workspace roots unless the dangerous override is set, while the custom-bind docs still show arbitrary host-path examples without documenting that default root limit or the override in the same section.

## What This Changes

The issue asks OpenClaw to make `sandbox.docker.binds` accept configured host paths outside the per-session sandbox/workspace roots, matching the public custom-bind examples.

## Best Possible Solution

Maintainers should explicitly choose the Docker bind contract, then either document the default allowed roots plus `dangerouslyAllowExternalBindSources` next to the examples or intentionally relax configured external binds with security-focused regression coverage.

## Reproduction Assessment

Yes, at source level. Configure a Docker bind whose source is outside `workspaceDir` and `agentWorkspaceDir`, leave `dangerouslyAllowExternalBindSources` unset, and current `buildSandboxCreateArgs` reaches `validateBindMounts` with bounded roots and throws `outside allowed roots`; tests assert that path.

## Solution Assessment

Unclear. The current allowlist is a defensible safer default, so documentation clarification may be the narrowest maintainable fix unless maintainers intentionally want configured external binds to be trusted by default.
