## Summary

Keep open. Current main still rejects configured Docker bind sources outside the per-session sandbox/workspace roots by default, while the custom bind docs still show arbitrary host paths without naming the default root constraint or the dangerous override; #59718 is adjacent least-privilege design work, not a superseding tracker for this docs/runtime mismatch.

## What This Changes

The issue requests that `sandbox.docker.binds` accept operator-configured host paths outside the per-session sandbox/workspace roots, or that the docs be corrected to match the default restriction.

## Best Possible Solution

Maintainers should define the Docker bind contract explicitly, then either document the default roots plus `dangerouslyAllowExternalBindSources` next to the examples, or intentionally relax configured external binds with focused security regression coverage.

## Reproduction Assessment

Yes. Configure a Docker bind whose source is outside `workspaceDir` and `agentWorkspaceDir`, leave `dangerouslyAllowExternalBindSources` unset, and `buildSandboxCreateArgs` reaches `validateBindMounts` with bounded roots and throws `outside allowed roots`; current tests assert this path.

## Solution Assessment

Unclear. The current allowlist is a defensible safer default, so documentation clarification is likely the narrowest maintainable fix unless maintainers intentionally want configured external binds to be trusted by default.
