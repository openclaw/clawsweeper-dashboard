## Summary

Keep open. Current main still limits Docker bind sources to the active sandbox workspace and agent workspace unless `dangerouslyAllowExternalBindSources` is set, and latest release `v2026.5.22` has the same behavior; the merged Codex writable-bind fix at https://github.com/openclaw/openclaw/pull/83849 addressed an adjacent write-policy regression, not this source-validation/docs contract.

## What This Changes

The issue asks OpenClaw to reconcile Docker custom-bind docs and runtime behavior because external bind sources are rejected unless a dangerous override is set.

## Best Possible Solution

Maintainers should choose the Docker bind contract, then either document the default allowed roots plus `dangerouslyAllowExternalBindSources` beside the custom-bind examples or intentionally relax configured external binds with security-focused regression coverage.

## Reproduction Assessment

Yes, at source level. Configure a Docker bind source outside `workspaceDir` and `agentWorkspaceDir`, leave `dangerouslyAllowExternalBindSources` unset, and current `buildSandboxCreateArgs` reaches `validateBindMounts` with bounded roots and throws `outside allowed roots`.

## Solution Assessment

Unclear until maintainers decide the security contract. Documentation clarification is the narrowest safe fix if the allowlist is intended; runtime relaxation needs explicit owner/security approval and coverage for blocked roots, symlink canonicalization, and writable mounts.
