## Summary

Keep open. Current main and v2026.5.19 still reject Docker bind sources outside the sandbox/workspace roots unless `dangerouslyAllowExternalBindSources` is set, while the custom-bind docs still show arbitrary host-path examples and do not clearly document the default root limit beside them.

## What This Changes

The issue requests that Docker sandbox custom bind sources outside the per-session sandbox/workspace roots either work as documented or clearly document the required dangerous override.

## Best Possible Solution

Maintainers should choose the Docker bind contract, then either document the default allowed roots plus `dangerouslyAllowExternalBindSources` beside the examples or intentionally relax configured external binds with security-focused regression coverage.

## Reproduction Assessment

Yes, at source level. Configure a Docker bind source outside `workspaceDir` and `agentWorkspaceDir`, leave `dangerouslyAllowExternalBindSources` unset, and current `buildSandboxCreateArgs` reaches `validateBindMounts` with bounded roots and throws `outside allowed roots`.

## Solution Assessment

Unclear until maintainers decide the security contract. Documentation clarification is the narrowest safe fix if the allowlist is intended; runtime relaxation needs explicit owner/security approval and coverage for blocked roots, symlink canonicalization, and writable mounts.
