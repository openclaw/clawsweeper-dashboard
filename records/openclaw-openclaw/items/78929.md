## Best Possible Solution

Keep the current main delivery-correlation fix and ship it in the next release so affected Telegram users can update from builds before 2026-05-07T03:39:36Z.

No current-main failing reproduction path remains from source inspection. The pre-fix path is clear from history: same-chat message-tool sends were not counted as inbound delivery, so the direct-chat silent fallback could be emitted before the visible tool-routed reply path completed.

## Solution Assessment

Yes. The current fix is narrow because it only marks matching Telegram session/chat/account outbound success as delivered, avoiding the duplicate filler while preserving fallback behavior for truly empty or failed turns.
