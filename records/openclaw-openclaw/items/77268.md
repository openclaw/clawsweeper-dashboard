## Summary

Keep open: the fix is in the right Feishu-owned area, but it does not cover the reported payload shape. The new parser delegates directly to a sanitizer that only accepts `body.elements`, while the linked regression uses top-level `elements` and discussion also calls out wrapped interactive-card JSON.

## What This Changes

The PR adds Feishu card-JSON detection for plain `message`/`text` values in `send` and `thread-reply`, exposes the helper through the Feishu runtime, adds channel tests, and updates the changelog.

## Best Possible Solution

Keep the Feishu-local approach, but normalize the legacy top-level `elements` and wrapped interactive-card shapes before sanitizing, then add regressions using the exact linked payload.

## Reproduction Assessment

Yes. The linked issue provides an exact `message(action=send)` payload, comments confirm it on 2026.4.25, and source inspection shows the proposed parser still returns undefined for that payload shape.

## Solution Assessment

No. Feishu-local parsing is the narrow maintainable direction, but this implementation must cover the existing compatibility shapes before it can solve the reported regression.
