## Summary

Keep open: this PR targets a real Feishu regression and uses the right extension-owned path, but the proposed parser still misses the reported card shape, so it would not fully fix the linked bug.

## What This Changes

The PR adds Feishu `message`/`text` JSON card detection for `send` and `thread-reply`, wires it through the Feishu channel runtime, adds channel tests, and updates the changelog.

## Best Possible Solution

Extend the Feishu text-card parser to accept the reported card shapes, especially top-level `elements`, while still normalizing through the existing safe card output and adding a regression fixture from #53486.

## Reproduction Assessment

Yes. Source inspection of current `main` shows the proactive Feishu send path cannot convert plain `message` JSON into a card, and the linked bug provides a concrete payload; I did not execute tests because this was a read-only review.

## Solution Assessment

No. The proposed hook point is appropriate, but delegating directly to the current body-only sanitizer misses the reported top-level-elements Feishu card format, so the fix is incomplete.
