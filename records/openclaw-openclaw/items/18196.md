## Summary

Keep open. Current `main` does not implement the PR's core client-side skill security model: skills do not parse or store declared capabilities, the before-tool-call path has no skill-aware enforcement gate, `skills list/info/check` still report setup readiness rather than trust/capability/scan status, and the existing skill scanner is limited to install-time code scanning rather than SKILL.md content/runtime enforcement.

## Evidence

- **Skill metadata on main has no capability or scan fields:** `OpenClawSkillMetadata` only carries install/setup metadata (`always`, `skillKey`, `primaryEnv`, `emoji`, `homepage`, `os`, `requires`, `install`), and `SkillEntry` has no scan result/trust-tier fields. That means the PR's declared capability model is not present on current `main`.
