## Summary

Keep the PR open because the patch is still not a safe fix for the reported memory embedding provider routing problem: source inspection shows it leaves baseUrl-only generic providers and gateway memory-specific provider overrides on the wrong adapter path, and the posted proof does not exercise those remaining failure paths.

## What This Changes

## Best Possible Solution
