## Summary

Keep open: the PR is a useful partial fix for memory-core provider resolution, but it is not merge-ready because the `/v1/embeddings` gateway path still resolves by the visible `openai` provider id and the supplied proof is adapter/mock-script evidence rather than a real OpenClaw endpoint run.

## What This Changes

This PR adds `actualProvider` to memory-search resolution and memory-core embedding creation so configured `openai` aliases can use a resolved embedding adapter, with focused tests and a small Cohere stream typing cleanup.

## Best Possible Solution

Thread the resolved adapter identity through every embedding creation surface, including `/v1/embeddings`, while keeping `provider` as the user-visible configured id, then add gateway coverage and real endpoint proof.

## Reproduction Assessment

Yes, source-reproducible: current main and the PR diff show the gateway embeddings path still uses `memorySearch.provider` and direct provider lookup before the configured alias. I did not run a live Ollama endpoint in this read-only review.

## Solution Assessment

No; the memory-core portion is a good partial repair, but the best fix must include the sibling `/v1/embeddings` runtime path so one agent config cannot use different embedding adapters by entry point.
