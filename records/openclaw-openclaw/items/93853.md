## Summary

Keep open. The memory-core routing change is useful, but the patch is still incomplete because the gateway `/v1/embeddings` bridge can keep resolving the visible `openai` provider id to the direct OpenAI adapter while memory indexing resolves the same config to Ollama.

## What This Changes

The branch adds an `actualProvider` field through memory-search and memory-core embedding creation, adds focused regression coverage, and includes a small Cohere stream wrapper type cleanup.

## Best Possible Solution

Thread the resolved adapter identity through every embedding creation surface, including `/v1/embeddings`, while preserving `provider` as the user-visible configured id, then add gateway coverage and real endpoint proof.

## Reproduction Assessment

Yes, source-reproducible: current `main` and the PR diff show the gateway embeddings path still uses `memorySearch.provider` and direct provider lookup before the configured alias. I did not run a live Ollama endpoint in this read-only review.

## Solution Assessment

No. The memory-core portion is a good partial repair, but the best fix must include the sibling `/v1/embeddings` runtime path so one agent config cannot use different embedding adapters by entry point.
