## Summary

Keep open. Current main has useful prompt-cost controls for workspace bootstrap context, but it still lacks the requested per-file conditional injection contract, frontmatter/config-driven policies, lazy TOC/on-demand mode, and budget-aware summarization; #13968 covers only the narrower HEARTBEAT.md subset.

## What This Changes

Request selective, lazy, and token-budget-aware workspace bootstrap context injection so files such as HEARTBEAT.md are included only when needed or summarized/on-demand.

## Best Possible Solution

Keep this as a design umbrella, handle the HEARTBEAT.md-only path through #13968, and define a core bootstrap/context-engine contract for per-file policies, TOC/on-demand loading, and summarization with docs and focused regression tests.
