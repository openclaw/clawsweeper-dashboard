## Summary

Keep open. Current main has partial prompt-cost controls for workspace bootstrap context, but it still lacks the requested per-file conditional injection contract, frontmatter/config-driven inject rules, lazy TOC/on-demand mode, and budget-aware summarization. The related open issue #13968 covers only the HEARTBEAT.md subset, and the referenced frontmatter-injection commit is diverged/unmerged rather than implemented on main.

## What This Changes

Request configurable selective and lazy workspace context-file injection, including per-file rules, TOC/on-demand loading, and token-budget-aware summarization.

## Best Possible Solution

Keep this as a design umbrella, split the HEARTBEAT.md-only behavior through #13968, and separately define a core bootstrap/context-engine contract for per-file policies, TOC/on-demand loading, and summarization with docs and focused regression tests.
