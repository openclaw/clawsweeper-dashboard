## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Scope cross-account fallback before rewriting mentions:** `extensions/discord/src/directory-cache.ts:123-130`
  - body: The fallback searches every other account cache and returns the first matching handle, so duplicate display names can resolve by insertion order and ping the wrong Discord user; scope this by guild/context or require an unambiguous authorized match.
  - confidence: 0.91
- **[P1] Move directory cache writes behind policy gates:** `extensions/discord/src/monitor/message-handler.preflight.ts:386-437`
  - body: Preflight caches author and mentioned-user handles before later allowlist, allowBots, mention-required, and command authorization drops. Rejected traffic can seed the shared cache later used for outbound rewrites.
  - confidence: 0.89
- **[P1] Keep guild nicknames out of the account-global cache:** `extensions/discord/src/monitor/message-handler.preflight.ts:393`
  - body: Guild nicknames are scoped to a guild, but this cache is keyed only by account and outbound rewriting has no guild context, so the same nickname in another guild can rewrite to the wrong user.
  - confidence: 0.9
- **[P2] Keep mention matching out of URLs and canonical mentions:** `extensions/discord/src/mentions.ts:9`
  - body: The broad non-letter/non-number prefix treats `/` and `<` as mention separators, so cached handles can rewrite URL paths or already-canonical Discord mentions.
  - confidence: 0.84
- **[P1] Preserve verified bot identity resolution:** `extensions/discord/src/monitor/provider.startup.ts:229-236`
  - body: Falling back from a failed fetchUser("@me") to applicationId can keep the monitor running with an unverified bot user ID, weakening self-message and mention-required checks.
  - confidence: 0.82
