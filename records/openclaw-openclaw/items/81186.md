## Summary

Keep open: current main still runs the normal heartbeat path for a literal `PAUSED` `HEARTBEAT.md`, and the open linked PR is the active implementation/product-decision path for this report. The documented controls today are `0m`, `openclaw system heartbeat disable`, and empty-file skipping; accepting `PAUSED` would add a new user-facing pause contract.

## What This Changes

The issue asks OpenClaw to stop idle heartbeat polling from consuming credits, including when `HEARTBEAT.md` says `PAUSED`, and to make disabling the default heartbeat more visible and durable.

## Best Possible Solution

Resolve the linked PR with an explicit heartbeat pause-contract decision: either land a documented `PAUSED` sentinel with diagnostics and coverage, or keep `0m` and `openclaw system heartbeat disable` as the canonical disable paths and close the PR with that rationale.

## Reproduction Assessment

Partially yes: source inspection shows current main treats literal `PAUSED` as non-empty `HEARTBEAT.md` content and falls through to the interval heartbeat path. The broader credit-burn amount and hosted restart/config-overwrite claim are not high-confidence reproduced from the available evidence.

## Solution Assessment

Unclear: a `PAUSED` sentinel is a plausible workflow fix, but it adds a new user-facing contract and may duplicate the existing `0m` and `openclaw system heartbeat disable` controls unless maintainers accept that product direction.
