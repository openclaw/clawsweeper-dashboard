## Summary

Keep open: current main still lacks a `HEARTBEAT.md` `PAUSED` sentinel, and https://github.com/openclaw/openclaw/pull/81276 is the open closing PR carrying the implementation and product-boundary discussion. The documented disable path remains `0m` / `openclaw system heartbeat disable`, so this issue should stay open until that PR is landed or explicitly rejected.

## What This Changes

The issue reports costly default heartbeat polling while idle and asks for a safer pause/disable workflow, better onboarding/docs visibility, persistent config behavior, and possibly a UI toggle.

## Best Possible Solution

Resolve the linked PR’s product question: either land and document a `PAUSED` sentinel with diagnostics and coverage, or reaffirm `0m` / `system heartbeat disable` as the canonical pause path and address any remaining docs or config-persistence evidence separately.

## Reproduction Assessment

Partially, yes: source inspection shows a literal `PAUSED` `HEARTBEAT.md` is treated as non-empty content and does not skip the interval preflight. The broader hosted credit total and config-overwrite claim lacks a live current-main reproduction path in the available evidence.

## Solution Assessment

Unclear: a `PAUSED` file sentinel could solve the reporter’s workflow, but it is a new product contract beside the existing `0m` and CLI disable controls. The maintainable path is to decide that contract in the linked PR before changing behavior.
