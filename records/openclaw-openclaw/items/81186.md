## Summary

Keep open: current `main` still runs the default heartbeat cadence and does not treat `PAUSED` in `HEARTBEAT.md` as a skip condition, while https://github.com/openclaw/openclaw/pull/81276 is the active implementation/discussion path for this report.

## What This Changes

The issue asks OpenClaw to prevent idle default heartbeats from burning provider credits, especially when a user tries to pause them through `HEARTBEAT.md`.

## Best Possible Solution

Resolve the heartbeat pause contract through https://github.com/openclaw/openclaw/pull/81276: either land a documented `PAUSED` sentinel with diagnostics and coverage, or keep `0m`/`openclaw system heartbeat disable` as the canonical pause path and close the PR with that rationale.

## Reproduction Assessment

Partially yes: source inspection shows current `main` treats a literal `PAUSED` `HEARTBEAT.md` as non-empty content and proceeds with the interval heartbeat path. The broader hosted credit total and config overwrite claim do not have a high-confidence live reproduction from the available evidence.

## Solution Assessment

Unclear: adding a `PAUSED` sentinel at preflight is plausible, but it is a new user-facing contract that may duplicate the existing `0m` config and `openclaw system heartbeat disable` controls unless maintainers choose to support it.
