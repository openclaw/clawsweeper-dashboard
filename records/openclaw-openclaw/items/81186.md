## Summary

Keep open: current main still has no PAUSED heartbeat-file sentinel or activity-based idle gate, and https://github.com/openclaw/openclaw/pull/81276 is an open closing PR for this issue. The shipped contract already documents `0m` and `openclaw system heartbeat disable`, so the remaining work is maintainer product review of the linked PR rather than cleanup closure.

## What This Changes

Request that idle/default heartbeat polling stop consuming credits by respecting a PAUSED or no-work state and by making disable controls persist and easier to discover.

## Best Possible Solution

Resolve the linked PR by choosing one pause contract: land and document a PAUSED sentinel with regression and runtime proof, or keep `0m`/CLI disable as canonical and close the PR with that rationale while leaving broader activity-based idle timeout to https://github.com/openclaw/openclaw/issues/14051.

## Solution Assessment

Unclear: the linked PR changes the plausible preflight location, but recognizing PAUSED is a new user-facing pause contract and maintainers need to decide whether it should exist beside the documented `0m` and CLI disable controls.
