## Summary

Keep open: current main still lacks a documented `HEARTBEAT.md` `PAUSED` sentinel, and this issue is paired with the open closing PR https://github.com/openclaw/openclaw/pull/81276, so it should not be closed before that PR is reviewed, landed, or rejected.

## What This Changes

The issue asks OpenClaw to stop idle/default heartbeats from consuming credits, including when `HEARTBEAT.md` is set to `PAUSED` or other disable controls are intended to suppress scheduled polling.

## Best Possible Solution

Resolve the linked PR/product decision by either landing and documenting an accepted `PAUSED` sentinel, or closing that PR and reinforcing the existing `0m` and empty-file contracts.

## Reproduction Assessment

Partially: source inspection gives a clear current-main path where literal `PAUSED` is non-empty `HEARTBEAT.md` content and does not skip the heartbeat preflight. The exact hosted restart overwrite and daily credit total are not reproduced.

## Solution Assessment

Unclear: adding an exact `PAUSED` sentinel in the preflight is plausible, but it creates a new user-facing pause contract alongside the existing `0m` and CLI disable controls, so maintainer product agreement is needed first.
