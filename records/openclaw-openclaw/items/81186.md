Keep open: current main still lacks a PAUSED heartbeat-file sentinel or activity-idle gate, and the open closing PR is the active implementation/product-boundary discussion for this report.

## What This Changes

The issue asks OpenClaw to stop costly idle heartbeat polling, including a PAUSED HEARTBEAT.md skip, clearer disable docs/onboarding, and persistent disable controls.

## Best Possible Solution

Resolve the linked PR by choosing one pause contract, then either land a documented PAUSED sentinel with regression coverage or keep 0m/CLI disable as canonical; leave broader activity-idle timeout work to https://github.com/openclaw/openclaw/issues/14051.
