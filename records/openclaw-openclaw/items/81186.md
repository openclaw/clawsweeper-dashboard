## Summary

Keep open: current main still has no `HEARTBEAT.md` `PAUSED` sentinel, and https://github.com/openclaw/openclaw/pull/81276 is an open closing PR carrying the implementation and product-direction discussion.

## What This Changes

The issue asks OpenClaw to avoid idle heartbeat credit burn, including respecting a `PAUSED` `HEARTBEAT.md` state and making heartbeat disablement more visible and persistent.

## Best Possible Solution

Review https://github.com/openclaw/openclaw/pull/81276 and decide the heartbeat pause contract: either land a documented `PAUSED` sentinel with diagnostics and coverage, or keep `0m`/CLI disable as canonical and close the PR with that rationale.

## Reproduction Assessment

Partially, yes: source inspection shows current main treats a literal `PAUSED` `HEARTBEAT.md` as non-empty content and proceeds past the file gate. The claimed daily credit total and hosted restart/config overwrite path were not live-reproduced.

## Solution Assessment

Unclear: adding a `PAUSED` sentinel at preflight is a plausible narrow implementation, but it creates a new user-facing pause contract that maintainers are already comparing against the existing `0m` and `openclaw system heartbeat disable` paths.
