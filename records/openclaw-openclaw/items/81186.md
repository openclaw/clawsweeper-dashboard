## Summary

Keep open for maintainer product judgment. Current main still intentionally runs scheduled heartbeats for non-empty `HEARTBEAT.md` prose such as `PAUSED`, while narrower cost mitigations and related trackers cover only parts of the broader idle-cost problem.

## What This Changes

The issue asks OpenClaw to stop or reduce idle heartbeat credit use from the default 30-minute cadence and make disable/pause behavior more visible and persistent.

## Best Possible Solution

Define a maintainer-approved heartbeat idle/paused policy, land or supersede the narrow scaffold fix in https://github.com/openclaw/openclaw/pull/75047, and keep https://github.com/openclaw/openclaw/issues/14051 as the API-level tracker for activity-gated heartbeats.

## Reproduction Assessment

Yes for the current source behavior: a configured interval with non-empty `HEARTBEAT.md` prose such as `PAUSED` reaches the heartbeat prompt path instead of the empty-file skip. No for the exact billed-credit amount or hosted restart overwrite, which would need logs or live hosted reproduction.

## Solution Assessment

No as a narrow bug-fix cleanup. The maintainable path needs a product/API decision on whether `PAUSED`, activity idle windows, onboarding warnings, or an existing `0m`-backed UI control should be the supported contract.
