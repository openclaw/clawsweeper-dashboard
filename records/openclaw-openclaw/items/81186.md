## Summary

Keep open for maintainer product judgment: current main has important heartbeat cost mitigations, but it still intentionally runs scheduled heartbeats when `HEARTBEAT.md` is missing or contains non-empty prose such as `PAUSED`, so the remaining request is not an already-fixed bug cleanup.

## What This Changes

The issue asks OpenClaw to avoid significant model-credit use from the default idle heartbeat and to make the disable, paused, and onboarding behavior durable and visible.

## Best Possible Solution

Maintainers should consolidate this with the heartbeat auto/idle policy discussion and decide whether to keep proactive heartbeats with stronger cost guidance, or add a supported `auto`/paused/disable UX that preserves the existing `0m` and empty-file contracts.

## Reproduction Assessment

No, not for the full issue. Source inspection clearly shows current main will still run scheduled heartbeats for missing files and non-empty prose such as `PAUSED`, but the reported ~2,500 credits/day and hosted restart overwrite are not reproduced with logs or a live setup.

## Solution Assessment

Unclear. The current `0m` and empty-file skips are the narrow existing controls; treating default idle, missing-file, or `PAUSED` states as disabled would be a product/config contract change best decided with the related open auto-toggle issue.
