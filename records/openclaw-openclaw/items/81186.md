## Summary

Keep open: current main still runs the normal heartbeat path for a non-empty `HEARTBEAT.md` containing `PAUSED`, and the linked open PR is the active implementation/product discussion path for this report.

## What This Changes

The issue asks OpenClaw to stop or clearly disable idle heartbeat polling that can consume provider credits, especially when a user attempts to pause it with `HEARTBEAT.md`.

## Best Possible Solution

Use the linked PR to decide one explicit heartbeat pause contract, then either land a documented sentinel with diagnostics and coverage or keep `0m`/CLI disable as the canonical pause path.

## Reproduction Assessment

Partially, yes: source inspection on current main shows a literal `PAUSED` `HEARTBEAT.md` is treated as non-empty content and does not skip preflight. The broader daily credit total and hosted restart/config overwrite claim were not live-reproduced.

## Solution Assessment

Unclear: the preflight hook is the right place if a sentinel is accepted, but the current requested direction duplicates existing `0m` and `openclaw system heartbeat disable` controls unless maintainers choose that new contract.
