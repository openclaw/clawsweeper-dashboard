## Summary

Keep open: current `main` still has the documented 30-minute heartbeat default and no `PAUSED` file sentinel, and the open linked PR is the active implementation path for this report. Maintainer discussion on that PR is still deciding whether a file-based pause contract should exist alongside the existing `0m` disable path.

## What This Changes

The issue asks OpenClaw to avoid costly idle heartbeat polling by making the default cadence/disable path clearer, preserving disable configuration, and possibly treating `HEARTBEAT.md` as a pause signal.

## Best Possible Solution

Decide the heartbeat pause contract through the linked PR: either land a documented `PAUSED` sentinel with diagnostics and docs, or keep `0m`/CLI disable as the canonical pause path and close the PR with that rationale.

## Reproduction Assessment

Partially, yes: source inspection shows current `main` treats a literal `PAUSED` `HEARTBEAT.md` as non-empty content and proceeds with the interval heartbeat path. The broader daily credit total and hosted restart/config overwrite claim were not live-reproduced from the available evidence.

## Solution Assessment

Unclear: a file-level `PAUSED` sentinel is a plausible fix for the reporter’s workflow, but it creates a new user-facing contract that maintainers are already questioning because `0m` and `openclaw system heartbeat disable` exist.
