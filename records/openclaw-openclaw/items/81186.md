## Summary

Keep open for maintainer judgment: current main has heartbeat cost mitigations, but it still intentionally runs scheduled heartbeats when `HEARTBEAT.md` is missing or contains non-empty prose such as `PAUSED`, so changing the idle/default policy is not a narrow already-fixed bug cleanup.

## What This Changes

The issue asks OpenClaw to avoid costly default idle heartbeat polling, surface or document the default cost, and preserve a disabled heartbeat configuration across gateway restarts.

## Best Possible Solution

A maintainer should decide whether proactive heartbeat should remain the default with stronger first-run cost guidance, or whether OpenClaw should add a documented idle/paused contract using the existing `0m` disable path or a new explicit sentinel.

## Reproduction Assessment

No, not as a high-confidence current-main bug. Source inspection shows current main skips empty `HEARTBEAT.md` files but intentionally runs when the file is missing or contains non-empty prose, and the reported hosted restart overwrite lacks logs or live reproduction.

## Solution Assessment

Unclear. Treating `PAUSED` or all idle/missing-file heartbeats as disabled might reduce cost, but it would change documented proactive behavior; the safer path is a maintainer product decision on default/onboarding policy and the supported disable workflow.
