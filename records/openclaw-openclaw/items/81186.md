## Summary

Keep open: current main still does not recognize a PAUSED heartbeat file, and https://github.com/openclaw/openclaw/pull/81276 is the open closing PR where maintainers are already discussing whether that should become a supported pause contract.

## What This Changes

The issue reports idle heartbeat polling from the default 30-minute cadence and asks for clearer or stronger controls to prevent credit use when no heartbeat work should run.

## Best Possible Solution

Resolve the linked PR by choosing one pause contract, then either land a documented PAUSED sentinel with regression coverage or keep `0m` and `openclaw system heartbeat disable` as the canonical controls.

## Reproduction Assessment

Yes for the narrow source behavior: current main treats PAUSED as ordinary non-empty HEARTBEAT.md content and continues through the interval heartbeat path. No live reproduction was established for the exact daily credit total or hosted config overwrite claim.

## Solution Assessment

Unclear: the linked PR changes the plausible preflight location, but accepting PAUSED needs maintainer agreement and documentation because current docs already expose `0m` and CLI disable controls.
