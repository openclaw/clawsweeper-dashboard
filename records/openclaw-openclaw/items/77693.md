## Summary

Keep open: current main still has a docs/config discoverability mismatch. The key is documented and supported by the runtime, but the config materialization path used by `openclaw config get` does not currently add `archiveAfterMinutes` when it is only relying on the runtime default.

## What This Changes

The issue asks for `agents.defaults.subagents.archiveAfterMinutes` to resolve through `openclaw config get`, or for the subagents docs to clarify the correct configurable path.

## Best Possible Solution

Materialize the existing 60-minute archive default in the runtime config read path so `config get` and the docs agree, while preserving `archiveAfterMinutes: 0` as disable and avoiding writes to the user config file.

## Reproduction Assessment

Yes, from source inspection and the reporter's command output. `runConfigGet` walks `snapshot.config`, `materializeRuntimeConfig` delegates defaults to `applyAgentDefaults`, and that defaulting code does not add `archiveAfterMinutes` even though the runtime archive resolver falls back to 60.

## Solution Assessment

Yes, the narrowest maintainable fix is to surface the already-existing runtime default through config materialization and add regression coverage. A docs-only clarification is a fallback only if maintainers intentionally want optional runtime defaults hidden from `config get`.
