## Summary

Keep open: current main and v2026.6.10 implement Chat Completions `tool_calls` as caller-supplied client tools, not a feed of every internal OpenClaw tool call. Exposing internal tool telemetry through this HTTP API would be a new product and security contract, and the report does not include a concrete qwen request/response that proves a regression.

## What This Changes

The issue asks for OpenClaw's Chat Completions API to return internal tool-call information when an agent is accessed with a base URL, API key, and model name.

## Best Possible Solution

Keep the existing client-tool Chat Completions contract unless maintainers explicitly design an opt-in, redacted internal-tool telemetry or diagnostic API with security review, docs, and compatibility tests.

## Reproduction Assessment

No high-confidence live repro. Source review confirms the endpoint currently returns structured OpenAI tool calls only for pending client tool calls, but the issue lacks a concrete request/response showing a regression for internal OpenClaw tools.

## Solution Assessment

No as a bug fix. The narrow maintainable path is a maintainer API/security decision first: either keep internal tools out of Chat Completions responses or add a deliberate redacted diagnostic/telemetry mode.
