The PR adds memory-core startup and non-force CLI catch-up logic that compares session transcript files with persisted index file state, marks stale session files dirty, and covers it with focused tests.

## Solution Assessment

Yes. The PR uses durable DB file state plus on-disk mtime/size comparison to mark only missing/newer/resized session files dirty, which is narrower than forcing full session scans on every session-start/watch sync.
