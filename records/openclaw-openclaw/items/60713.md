## Summary

Keep open. Current main still resolves an accessible `/tmp/openclaw` before consulting `os.tmpdir()`, which matches the reported Windows `C:\tmp\openclaw` behavior, and the open closing PR has not merged.

## What This Changes

The issue requests that `resolvePreferredOpenClawTmpDir()` skip the POSIX `/tmp/openclaw` preference on Windows and use the normal Windows temp directory for logs, TTS, and other temp files.

## Best Possible Solution

Review and land PR #73533, or replace it with an equivalent narrow change that gates the POSIX preferred path off on `win32`, keeps the trusted `os.tmpdir()` fallback, and adds Windows plus non-Windows regression coverage.

## Reproduction Assessment

Yes. Current source gives a high-confidence reproduction path: when `/tmp/openclaw` is accessible, the resolver returns it before `tmpdir()` is called, and Windows maps that POSIX-looking path to the current drive root.

## Solution Assessment

Yes for the proposed direction, but it is not merged yet. Skipping the POSIX preference on Windows while reusing the existing fallback is the narrowest maintainable fix.
