## Summary

Keep open: this is a focused performance optimization with sufficient Testbox proof, but the new cached metadata path introduces a workspace-scoping regression and the protected `maintainer` label also rules out cleanup close.

## What This Changes

The PR changes persisted read-only model catalog normalization to reuse one prepared manifest model-id normalization metadata snapshot, with focused regression tests and a changelog entry.

## Best Possible Solution

Keep the one-snapshot performance shape, but make the cached manifest metadata load preserve the active plugin registry workspace fallback and cover that case with a focused regression test.

## Reproduction Assessment

Yes. The PR body provides a concrete before/after Testbox benchmark for the performance problem, and source inspection gives a high-confidence path for the new workspace-scope regression without needing a local write-producing test run.

## Solution Assessment

No. The metadata reuse direction is maintainable, but the implementation should pass or derive the same workspace scope that `normalizeProviderModelIdWithManifest()` currently uses before it is the best fix.
