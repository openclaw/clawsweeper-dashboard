Keep open: the report matches active Telegram polling and gateway-scope code paths on current main, but the evidence still does not isolate whether authorized Telegram updates fail in isolated spool handoff, DM pairing/allowlist gating, or the separate CLI scope-upgrade path.

## What This Changes

## Best Possible Solution

Live-prove the Telegram polling path with a paired or allowlisted sender, then patch the precise boundary so spooled updates reach handleUpdate, authorized DMs update activity and reply, and any separate CLI scope loop gets its own regression fix.

## Solution Assessment
