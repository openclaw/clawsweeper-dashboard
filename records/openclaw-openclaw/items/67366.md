## Summary

Keep open. Current main and v2026.5.18 still call `.trim()` directly on prompt results in the Telegram credential replacement path, and multiple reporters confirmed the onboarding crash; no open closing PR currently owns this exact fix.

## What This Changes

The issue reports that replacing an existing Telegram bot token during onboarding throws `TypeError: Cannot read properties of undefined (reading 'trim')` instead of saving or validating the new token.

## Best Possible Solution

Normalize unexpected or nullable text prompt results at the shared prompter/setup boundary, preserve cancel/keep/use-env semantics, and add focused regression coverage for Telegram token replacement.

## Reproduction Assessment

Yes. The issue has reporter steps and current-main source still shows the configured-token replacement path can reach a direct `.trim()` on the prompt result; I did not execute the interactive flow in this read-only pass.

## Solution Assessment

Yes. A narrow shared prompt-result normalization fix with regression tests is the maintainable route; broad unrelated setup changes or caller-only guards risk missing the same boundary elsewhere.
