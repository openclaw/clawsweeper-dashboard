Keep open. Current main and the latest release still have the unsafe channel setup prompt paths that can call `.trim()` on a non-string text result, and the issue is currently owned by open linked repair PRs rather than an implemented main-branch fix.

The issue requests that `openclaw onboard` accept or gracefully reject a replacement Telegram bot token instead of throwing a TypeError during the token replacement prompt.

## Best Possible Solution

Land or replace the linked repair with a narrow shared setup-wizard guard that normalizes text prompt results before trimming and adds regression tests for Telegram token replacement plus the generic text-input path.

## Reproduction Assessment

Yes for source reproduction: current main still calls `.trim()` directly on awaited setup prompt text in the Telegram token replacement path and the generic setup wizard text-input path. I did not run the full interactive macOS onboarding sequence in this read-only review.

## Solution Assessment

Yes, the best fix shape is a narrow shared setup-wizard boundary guard with regression coverage. A Telegram-only workaround is too narrow, and no config migration or new option is needed for this crash.
