Keep open. Current main and v2026.6.6 still contain the unsafe prompt-result trimming paths, and the remaining work is already owned by open linked repair PRs rather than an implemented main-branch fix.

## What This Changes

The issue reports that replacing an existing Telegram bot token through onboarding can crash when a prompt returns undefined and the setup flow calls `.trim()`.

## Best Possible Solution

Land or replace the focused channel setup prompt guard so non-string text results cannot crash onboarding, with regression coverage for the Telegram token replacement path and the generic text-input path.

## Reproduction Assessment

Yes, from source inspection rather than a live terminal run: if `WizardPrompter.text` resolves to undefined in the reported token-replacement branch, `promptSingleChannelToken` calls `.trim()` on that value and throws.

## Solution Assessment

Yes, the narrow maintainable fix is to guard the channel setup prompt boundary and add regression tests; the issue should remain open until a linked repair PR lands or is replaced.
