## Summary

Keep open. Current main still has the unsafe Telegram token replacement `.trim()` path, the latest stable release and current beta tag also contain it, and https://github.com/openclaw/openclaw/pull/77813 is an open fixing PR with closing syntax for this issue.

## What This Changes

The issue reports `openclaw onboard` crashing with a TypeError when replacing an already-configured Telegram bot token during channel setup.

## Best Possible Solution

Review and land, split, or replace the setup-wizard prompt normalization from https://github.com/openclaw/openclaw/pull/77813, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an already-configured Telegram channel can decline the keep-token prompt, then the replacement token prompt result is trimmed without a nullish guard.

## Solution Assessment

Yes. The narrow maintainable fix is to normalize or reject nullish prompt results at the shared setup-wizard prompt boundary and add regression coverage; the open PR implements that, though it may need refresh or splitting before merge.
