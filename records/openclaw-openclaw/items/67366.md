## Summary

Keep open. Current main and the latest release still contain the unsafe setup prompt paths that can call `.trim()` on a non-string text result, and open linked repair PRs still own the fix path.

## What This Changes

The issue reports that replacing an existing Telegram bot token through `openclaw onboard` can crash because a prompt text result is treated as a string before trimming.

## Best Possible Solution

Land or replace the linked repair with a narrow shared setup-wizard guard that normalizes text prompt results before trimming and adds regression coverage for Telegram token replacement plus the generic text-input path.

## Reproduction Assessment

Yes for source reproduction: current main still calls `.trim()` directly on awaited setup prompt text in the Telegram token replacement path and the generic setup wizard text-input path. I did not run the full interactive macOS onboarding sequence in this read-only review.

## Solution Assessment

Yes, the best fix shape is a narrow shared setup-wizard boundary guard with regression coverage. A Telegram-only workaround is too narrow, and no config migration or new option is needed for this crash.
