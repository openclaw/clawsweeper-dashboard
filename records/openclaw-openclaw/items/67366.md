## Summary

Keep open. Current main and the latest release still contain the unsafe raw `.trim()` paths, and an open PR with closing syntax is the active implementation candidate, so this issue should stay open until that PR lands or is replaced.

## What This Changes

Replace an existing Telegram bot token during onboarding without crashing when the text prompt returns a nullish value.

## Best Possible Solution

Review and land, split, or replace the setup-wizard guard from https://github.com/openclaw/openclaw/pull/77813, then close this issue once the fix reaches main.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an already-configured Telegram channel can decline the keep-token prompt, then the replacement token prompt result is trimmed without a nullish guard.

## Solution Assessment

Yes. The maintainable fix is to normalize or reject nullish prompt results at the shared setup prompt boundary and add regression coverage; the linked PR contains that guard, though maintainers may choose to split it from unrelated channel streaming work.
