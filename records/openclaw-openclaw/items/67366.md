Fix `openclaw onboard` so replacing an existing Telegram bot token does not crash with a `trim` TypeError.

Normalize nullable text prompt results at the shared setup-wizard prompt boundaries, preserve existing valid-token behavior, and add focused regression coverage for undefined replacement-token prompt returns.
