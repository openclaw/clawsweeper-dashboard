## Summary

Current main no longer has the reported unguarded synchronous chmod path: the active secrets text/json writers delegate to fs-safe-backed private-store or atomic-replace helpers, whose sync chmod calls are best-effort. The reporter’s 2026.5.7 check is valid for the latest release, but the implementation evidence is present on current main only, not in that release.

## What This Changes

The issue asks OpenClaw to stop crashing when synchronous atomic config/auth writes hit chmod EPERM on Windows-mounted Docker volumes.

## Best Possible Solution

Keep the shared fs-safe-backed implementation on main and let the stale helper-only contributor PR be handled separately as obsolete; related auth-profile EPERM cascade work remains tracked by its narrower open issue if needed.
