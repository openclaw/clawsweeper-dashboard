## Summary

Keep open: the docs direction is useful, but the current PR is not mergeable because it documents the plugin logger contract incorrectly, makes a command-specific stdout claim without a current repro, and lacks required real behavior proof.

## What This Changes

The PR adds a `Logging best practices` section to `docs/plugins/building-plugins.md` advising plugin authors to use `api.logger` instead of `console.log` in `register()`.

## Best Possible Solution

Revise the section to match the required logger API, keep stdout guidance generalized or repro-backed, then add rendered docs or preview proof before merge.

## Reproduction Assessment

Yes for the review findings: comparing the PR diff with `OpenClawPluginApi`, `PluginLogger`, `buildPluginApi`, and `runConfigSchema` on current main proves the mismatch. A runtime bug reproduction is not applicable because this is a docs-only change.

## Solution Assessment

No, not as written. The maintainable version should use `api.logger.info`/`warn`/`error` as required methods, optional-chain only `debug`, and avoid naming `openclaw config schema` unless the PR adds a current repro.

## Review Findings
