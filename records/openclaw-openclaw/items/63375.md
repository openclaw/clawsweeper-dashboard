## Summary

Keep open: the docs-only PR addresses a real plugin-author logging docs gap, but the current text teaches the wrong SDK optionality and makes a command-specific `openclaw config schema` claim that is not supported by the current source path.

## What This Changes

The PR adds a logging best-practices section to `docs/plugins/building-plugins.md` advising plugin authors to use `api.logger` instead of `console.log` in `register()`.

## Best Possible Solution

Revise the docs to show `api.logger.info` / `warn` / `error` as required methods, optional-chain only `debug` if mentioned, and keep the stdout warning general unless a current `openclaw config schema` repro is added.

## Reproduction Assessment

Yes for the review findings: the PR diff plus `PluginLogger`, `OpenClawPluginApi`, `buildPluginApi`, and `runConfigSchema` source inspection show the docs mismatch. A runtime bug reproduction is not applicable because this is a docs-only PR.

## Solution Assessment

No, not as written. The narrow maintainable fix is a docs-only edit that matches the required logger contract and avoids the unsupported command-specific schema claim unless the PR adds proof.
