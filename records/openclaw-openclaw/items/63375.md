## Summary

Keep open. The docs direction is useful, but current main does not already contain the requested logging guidance and the proposed section still has concrete accuracy issues: it treats required logger members as optional and overstates `openclaw config schema` as a `register()`-time stdout corruption path.

## What This Changes

This PR adds a logging best-practices section to `docs/plugins/building-plugins.md` advising plugin authors to use `api.logger` instead of `console.log` in `register()`.

## Best Possible Solution

Revise the docs patch into concise plugin logging guidance that matches the SDK contract: show `api.logger.info/warn/error(...)` directly, optional-chain only `debug` if shown, and describe stdout writes as a general JSON-output CLI risk unless a current command-specific reproduction is added.

## Reproduction Assessment

Not applicable as a runtime reproduction for this docs-only PR. The docs defects are high-confidence by comparing the PR patch with `OpenClawPluginApi`/`PluginLogger` types and the current `config schema` implementation and tests.

## Solution Assessment

No, not as written. The proposed direction is useful, but the maintainable solution is a narrower docs update aligned with the current SDK contract and manifest-based schema path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Align logger optionality with the SDK contract:** `docs/plugins/building-plugins.md:305-308`
  - body: This paragraph says the `logger` property and its methods are optional and the example uses `api.logger?.info?.(...)`, but `OpenClawPluginApi.logger` is required and `PluginLogger.info`, `warn`, and `error` are required; only `debug` is optional. Please show required logger calls directly and optional-chain only `debug` if needed.
  - confidence: 0.95
- **[P3] Keep the config-schema example accurate:** `docs/plugins/building-plugins.md:298-299`
  - body: The new text ties `console.log` in `register()` to corrupting `openclaw config schema`, but current main builds that command's payload from manifest/config-schema metadata rather than plugin entry `register()` execution. The stdout warning is still useful, but this should be phrased as a broader JSON-output CLI risk unless a current command-specific repro is added.
  - confidence: 0.82
