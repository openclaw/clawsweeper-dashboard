## Summary

Keep open: the docs idea is useful, but the current branch still teaches the wrong plugin logger contract and makes an unproven command-specific stdout corruption claim.

## What This Changes

This PR adds a logging best-practices section to `docs/plugins/building-plugins.md` advising plugin authors to use `api.logger` instead of `console.log` in `register()`.

## Best Possible Solution

Keep the docs addition, but revise it to match the SDK logger contract and describe stdout pollution accurately.

## Reproduction Assessment

Yes for the review findings: the PR diff and current `PluginLogger`, `OpenClawPluginApi`, `buildPluginApi`, and config-schema source paths show the docs mismatch. A runtime bug reproduction is not applicable because this is a docs-only PR.

## Solution Assessment

No, not as written. The narrow maintainable path is a docs-only repair that documents `api.logger` and required logger methods correctly and generalizes the stdout warning unless a current command-specific repro is added.
