## Summary

Keep open. Current main still lacks this specific logging best-practices section, but the PR text does not match the current plugin logger contract and overstates the `openclaw config schema` path.

## What This Changes

The PR adds a logging best-practices section to `docs/plugins/building-plugins.md` telling plugin authors to use `api.logger` instead of `console.log` in `register()`.

## Best Possible Solution

Revise the docs to match the required logger API, keep the stdout warning generalized or repro-backed, and land it as a docs-only update after docs sanity checks pass.

## Reproduction Assessment

Yes for the review findings: source inspection of the PR diff, `PluginLogger`, `OpenClawPluginApi`, `buildPluginApi`, and `runConfigSchema` shows the docs mismatch. A runtime bug reproduction is not applicable because this is a docs-only PR.

## Solution Assessment

No, not as written. The narrow maintainable fix is to document `api.logger.info` / `warn` / `error` as required methods, optional-chain only `debug` when needed, and avoid the command-specific schema claim without a current repro.
