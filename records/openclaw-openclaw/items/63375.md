## What This Changes

This PR adds a logging best-practices section to `docs/plugins/building-plugins.md` advising plugin authors to use `api.logger` instead of `console.log` in `register()`.

## Best Possible Solution

Keep the section, but make it source-backed: document `api.logger` as required, `debug` as optional, `info`/`warn`/`error` as required, and phrase stdout pollution generally unless a current config-schema repro is added.
