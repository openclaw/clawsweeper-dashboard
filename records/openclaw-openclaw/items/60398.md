The issue reports that `openclaw gateway install` fails with launchd error 5 when the macOS home directory is on an external APFS volume.

## Best Possible Solution

Land one canonical boot-volume-aware LaunchAgent path across install, restart, uninstall, doctor detection, and update restart paths, then let the merged closing PR close this issue.

## Reproduction Assessment

Yes. The source path on current main still writes and bootstraps a HOME-derived LaunchAgent plist, and the linked PR gives live macOS APFS proof of error 5 from an external-volume plist path.

## Solution Assessment

Yes. A single boot-volume-aware LaunchAgent resolver reused by install, restart, uninstall, doctor, and update restart paths is the narrowest maintainable direction; an install-only copy would split lifecycle state.
