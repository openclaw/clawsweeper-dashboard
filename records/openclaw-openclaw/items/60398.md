## Summary

Keep open. Current main still resolves the macOS Gateway LaunchAgent plist under the resolved HOME directory and bootstraps that same path, so the external-APFS-home install failure remains source-backed and actionable.

## What This Changes

Handle macOS Gateway installation when the user's home directory lives on an external APFS volume by using a launchd-bootstrap-safe per-user LaunchAgents plist path.

## Best Possible Solution

Add one shared macOS LaunchAgent plist-location resolver and wire it through install, stage, bootstrap repair, uninstall, restart, status/doctor/audit, update restart, and the macOS app snapshot while preserving Linux/Windows behavior.

## Reproduction Assessment

Yes. The report gives concrete external-APFS HOME steps, and current-main code inspection reproduces the path decision: with HOME under `/Volumes/...`, the resolver returns `/Volumes/.../Library/LaunchAgents/ai.openclaw.gateway.plist` and install bootstraps that path.

## Solution Assessment

Yes. A narrow shared resolver plus lifecycle wiring is the maintainable fix; a docs-only workaround would leave install, restart, doctor/status, update restart, and app inspection inconsistent.
