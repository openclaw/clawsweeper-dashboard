## What This Changes

The issue reports that macOS gateway install fails with launchctl error 5 when the user home directory is on an external APFS volume.

## Best Possible Solution

Add one shared macOS LaunchAgent plist-location resolver that chooses a launchd-safe boot-volume user LaunchAgents path when HOME is external, then use it consistently across install, uninstall, restart, repair, audit/status, update restart, infra restart, macOS app snapshots, and tests.
