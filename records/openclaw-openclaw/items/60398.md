## Summary

Keep open. Current main still writes and bootstraps the macOS Gateway LaunchAgent from the resolved HOME path, so an external-APFS home still routes launchctl to a /Volumes/... plist and the reported error remains source-backed and actionable.

## What This Changes

The issue requests a macOS LaunchAgent plist location fallback for gateway install when HOME is on an external APFS volume.

## Best Possible Solution

Add one shared macOS LaunchAgent plist-location resolver that chooses a launchd-safe boot-volume path when HOME is external, then use it consistently across install, repair, restart, status/audit, update restart, and the macOS app snapshot.

## Reproduction Assessment

Yes, source-backed. The report gives concrete external-APFS HOME steps, and current main still resolves the plist to HOME/Library/LaunchAgents and passes that path to launchctl bootstrap.

## Solution Assessment

Yes. A shared LaunchAgent plist resolver is the narrow maintainable fix; an install-only workaround would leave restart, update restart, repair, audit, and app status paths inconsistent.
