## Summary

Keep this issue open: current main and the latest release still derive macOS LaunchAgent plist paths from the user's HOME, while an open linked PR now carries the focused fix and real APFS launchd proof.

## What This Changes

Make macOS gateway LaunchAgent install and lifecycle commands use a boot-volume plist path when the user's HOME is on an external APFS volume.

## Best Possible Solution

Review and land a canonical boot-volume-aware LaunchAgent resolver across install, restart, uninstall, doctor detection, and update restart paths, then close this issue through the merged fix.

## Reproduction Assessment

Yes, source-level and linked live proof are high confidence: current main builds and bootstraps the plist under HOME, and the linked PR reproduces launchd error 5 from an external APFS volume. I did not run a local macOS APFS repro in this Linux checkout.

## Solution Assessment

Yes, the best direction is a single boot-volume-aware LaunchAgent path resolver reused by all launchd lifecycle call sites. An install-only copy would leave restart, repair, uninstall, and doctor paths split.
