## Summary

Keep this issue open: current main and the latest release still derive the macOS LaunchAgent plist path from `HOME`, and the open linked PR is the active fix candidate rather than a merged implementation.

## What This Changes

The issue requests boot-volume LaunchAgent plist placement for macOS users whose home directory is on an external APFS volume.

## Best Possible Solution

Land or replace the linked boot-volume-aware LaunchAgent resolver so install, restart, uninstall, repair, status, and doctor detection use one canonical launchd plist path for external-home macOS users.

Yes, source inspection gives a high-confidence path: current main resolves LaunchAgent plists from `HOME`, and the linked PR proof shows launchd returns error 5 when that plist is on a distinct external APFS volume.

## Solution Assessment

Yes, the best fix shape is a canonical boot-volume-aware launchd plist resolver reused by every lifecycle path, not an install-only copy that leaves restart, uninstall, and inspection split across paths.
