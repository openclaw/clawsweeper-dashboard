## Summary

Keep open for maintainer review: current main still has the post-tool-result abort gap, while this PR is a focused shared agent-core fix with a regression test, sufficient terminal proof, clean mergeability, and no blocking review findings.

## What This Changes

The PR adds a shared agent-loop abort check after tool results and `turn_end`, plus a regression test that fails if an aborted tool run triggers a second model turn.

## Best Possible Solution

Land the shared loop-boundary guard after normal maintainer review, since it stops the extra provider turn without moving cleanup into a gateway or session-specific workaround.

## Reproduction Assessment

Yes. Current main source shows the loop can emit `turn_end` after tool results and then enter next-turn handling without checking the shared abort signal; the PR body supplies comparative terminal output showing the second model call before the fix and one call after it.

## Solution Assessment

Yes. The shared agent-core loop boundary is a better fix than a gateway/session-specific workaround because it preserves existing tool result and `turn_end` events while preventing any caller from starting another provider request on an already-aborted signal.
