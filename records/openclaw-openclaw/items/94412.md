## Best Possible Solution

Land the narrow agent-core guard after normal maintainer validation; no broader session, provider, or harness refactor is indicated.

## Reproduction Assessment

## Solution Assessment

Yes: the guard is placed after existing tool result and `turn_end` emission but before `prepareNextTurn`, stop hooks, queue polling, and another provider call. Provider- or harness-specific alternatives would be narrower than the shared low-level loop invariant.
