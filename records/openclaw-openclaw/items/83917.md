## Summary

Keep open: current main still keeps a module-level diffs viewer controller collection and appends to it during hydration, so the reported lifecycle risk remains source-backed even though it still needs an executed jsdom/browser repro.

## What This Changes

The issue asks OpenClaw to stop the diffs plugin viewer from retaining stale controllers across repeated hydration.

## Best Possible Solution

Clear or clean up the existing controller collection at the hydration boundary, add a focused repeated-hydration regression test, and regenerate the bundled viewer runtime asset with the source change.

## Reproduction Assessment

No executed high-confidence repro was run in this read-only review. Source inspection gives a focused path: repeated hydration in one module instance would append new controllers while toolbar sync still iterates the old entries.

## Solution Assessment

Yes, the proposed direction is narrow: rebuild or clear the controller list during hydration and cover that lifecycle directly. If old `FileDiff` instances are actively evicted, use the dependency's `cleanUp(recycle?)` API rather than a non-existent `destroy()` method.
