## What This Changes

Ensure diff viewer hydration does not retain stale FileDiff controllers across repeated hydration in the same document/module instance.

## Best Possible Solution

Make hydration rebuild the controller set for the current document, dispose prior FileDiff instances when replacing live controllers, refresh the generated runtime asset, and add a repeated-hydration regression test.

## Reproduction Assessment

No executed repro was run. Source inspection gives a clear same-module repeated-hydration path: a second `hydrateViewer()` call appends new controllers while toolbar sync still iterates the old entries.

## Solution Assessment

Yes. The narrow maintainable fix is to reset or replace the controller collection at the hydration boundary, use `FileDiff.cleanUp()` for evicted instances if needed, and cover the lifecycle directly.
