## What This Changes

The issue asks OpenClaw to clear or rebuild the diffs viewer controller list during re-hydration so stale `FileDiff` instances are not retained.

## Best Possible Solution

Rebuild the active controller list at the hydration boundary and add a focused jsdom regression that calls `hydrateViewer()` twice in the same document without manually clearing `controllers`.

## Reproduction Assessment

No executed repro was run in this read-only review. Source inspection gives a medium-confidence same-module double-hydration path because `hydrateViewer()` appends to the exported module array and toolbar sync later iterates every retained controller.

## Solution Assessment

Yes, the best fix appears narrow: reset or replace the controller collection during hydration and test that two hydrations leave exactly one active controller per card. Any cleanup of old `FileDiff` instances should be dependency-informed because `@pierre/diffs` exposes `cleanUp()`, not `destroy()`, and its default cleanup can remove the host container.
