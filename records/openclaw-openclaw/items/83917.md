## What This Changes

The issue asks for the diffs viewer hydration lifecycle to clear or rebuild its module-level controller list so repeated hydration does not retain stale FileDiff controllers.

## Best Possible Solution

Rebuild the controller collection at the hydration boundary, use the real FileDiff cleanup API if old instances must be released, refresh the runtime asset, and add a focused repeated-hydration regression test.

No high-confidence executed repro was run in this read-only pass. Source inspection shows a same-module second hydration would append new controllers while toolbar sync still iterates the old entries, but the issue still needs a focused jsdom or browser regression to prove the full failure path.

## Solution Assessment
