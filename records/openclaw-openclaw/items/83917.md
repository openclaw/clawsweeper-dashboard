## Summary

Keep open: current main still keeps a module-level diffs viewer controller list, `hydrateViewer()` appends to it without clearing, and no merged canonical fix resets the list. The related open fix PR covers per-card hydration failures for https://github.com/openclaw/openclaw/issues/83914, but it does not close this controller lifecycle report.

## What This Changes

The issue asks for the diffs viewer hydration lifecycle to rebuild or clear its controller collection so stale `FileDiff` instances are not synchronized after re-hydration.

## Best Possible Solution

Reset or replace the diffs viewer controller collection at the hydration boundary, add a focused lifecycle regression test for two hydrations in one document, and regenerate the bundled viewer runtime asset if the source changes.

## Reproduction Assessment

No, not as an executed high-confidence repro in this read-only review. Source inspection shows that a second `hydrateViewer()` call in the same module instance would append new controllers while toolbar sync still iterates the older entries, but a focused jsdom/browser test should prove the full failure path.

## Solution Assessment

Yes. The narrow maintainable fix is to clear or rebuild the controller collection during hydration, with cleanup if the upstream `FileDiff` API exposes it; the existing related PR for per-card failures does not replace this lifecycle fix.
