## Summary

Keep open: current main and the latest release still keep the diffs viewer controller list at module scope and append during hydration without clearing it. The remaining work is narrow, but it still needs an executed lifecycle regression before treating the report as fully proven.

## What This Changes

The issue asks OpenClaw to stop the diffs viewer from accumulating stale module-level controller entries when the viewer is hydrated more than once in one document/module instance.

## Best Possible Solution

Reset or rebuild the controller collection at the `hydrateViewer()` boundary, add focused repeated-hydration coverage, and refresh the generated viewer runtime asset.

## Reproduction Assessment

No high-confidence executed repro was run. Source inspection on current main shows repeated hydration in one module instance would append more controllers while toolbar sync still iterates the old entries, so a focused jsdom or browser lifecycle test should prove the failure before landing.

## Solution Assessment

Yes. The narrow maintainable fix is to clear or replace the controller list during hydration, while first checking the `FileDiff` cleanup API if the implementation tries to destroy evicted instances.

Refresh `extensions/diffs/assets/viewer-runtime.js` with the diffs viewer build, keep the change local to the diffs plugin, and coordinate with https://github.com/openclaw/openclaw/pull/84775 if it lands first so the per-card failure work for https://github.com/openclaw/openclaw/issues/83914 is not duplicated.
