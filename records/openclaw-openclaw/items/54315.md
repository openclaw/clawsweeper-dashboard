## Summary

Keep open. Current main still does not implement the prioritized SOUL/HARD/AGENTS bootstrap contract, and the PR branch remains a draft with a blocking prompt-ordering mismatch plus a maintainer security/product decision around a new privileged bootstrap filename.

## What This Changes

The PR branch adds a bootstrap-priority diagnostics page, recognizes `HARD_EXECUTION_RULES.md` during workspace bootstrap loading, reorders bootstrap budget selection to prefer SOUL/HARD/AGENTS, emits skip-budget warnings, and updates focused bootstrap tests.

## Best Possible Solution

Maintainers should choose one bootstrap contract; if this priority model is accepted, discovery, budget selection, final prompt sorting, docs, diagnostics, and tests should all encode the same SOUL/HARD/AGENTS behavior, otherwise the remaining custom-file work should stay with #39120/#74735.
