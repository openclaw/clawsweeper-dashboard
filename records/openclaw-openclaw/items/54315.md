## Summary

Keep this draft PR open for maintainer review. Current main has not implemented the proposed HARD_EXECUTION_RULES.md bootstrap contract or SOUL/HARD/AGENTS budget priority, and the proposed patch still has a blocking ordering bug because final Project Context rendering uses a separate sort that the PR does not update.

## What This Changes

The PR branch adds a bootstrap-priority diagnostics doc, recognizes HARD_EXECUTION_RULES.md as a workspace bootstrap file, prioritizes SOUL.md and HARD_EXECUTION_RULES.md before AGENTS.md during bootstrap budget selection, emits skipped-budget warnings, and updates focused bootstrap tests.

## Best Possible Solution

Maintainers should choose one bootstrap contract. If SOUL/HARD/AGENTS priority is accepted, the branch should align discovery, budget selection, final Project Context sorting, docs, diagnostics, and integration tests around that same contract; if not, the work should defer to the configurable extra-bootstrap-file path tracked by #39120 and #74735.

## Solution Assessment

No. The PR is directionally useful but not mergeable as the best solution until the final prompt order, docs, diagnostics, tests, and maintainer-approved trust model all match one bootstrap policy.
