Keep open. ## What This Changes

The issue asks for Control UI access from Docker/VPS or LAN remote HTTP to connect despite the current secure-context device-identity requirement.

## Best Possible Solution

Keep the secure default, review or replace https://github.com/openclaw/openclaw/pull/56830 for clearer guidance, and decide any remote plain-HTTP device-identity support in https://github.com/openclaw/openclaw/issues/53274.

Yes, source-backed. A remote plain-HTTP browser lacks `crypto.subtle`, the UI omits device identity, and the gateway rejects the non-local Control UI connect without a device.

## Solution Assessment

Unclear. Clarifying the message is a narrow maintainable step, but accepting device identity or operator auth over remote plain HTTP changes a documented security boundary and needs maintainer threat-model approval.
