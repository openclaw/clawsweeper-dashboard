## Summary

This PR is still needed because current main still buffers Mattermost successful REST JSON/text responses without a success-path cap. The JSON, upload, and probe parts are on the right boundary, but the non-JSON success fallback currently returns a truncated 64 KiB string as success instead of failing on overflow, so the patch needs a narrow fix before merge.

## What This Changes

The PR changes the Mattermost client, upload, and probe success paths to use bounded response readers and adds regression tests for oversized streamed JSON cancellation.

## Best Possible Solution

Use the shared bounded JSON reader and a throwing bounded text-success reader in the Mattermost plugin, then merge after maintainer acceptance of the response-size budgets.

## Reproduction Assessment

Yes. Source inspection on current main shows successful Mattermost client, upload, and probe paths using unbounded res.json()/res.text(), and the PR body includes terminal proof from a local no-Content-Length streaming server; I did not run tests in this read-only review.

## Solution Assessment

No as written. The JSON, upload, and probe fix uses the right shared SDK helper, but the text fallback should fail on overflow instead of returning a truncated successful value.
