## Summary

Keep this PR open for normal maintainer review: current main still has the unbounded Mattermost success-body reads, and the branch applies the existing bounded JSON reader to the client, upload, and probe paths with focused regression coverage. I found no blocking correctness finding; the maintainer-visible decision is whether the new hard caps are the intended security/compatibility tradeoff for unusually large Mattermost success responses.

## What This Changes

The PR bounds Mattermost successful REST JSON reads in the client, file upload, and probe paths, caps the non-JSON success fallback, and adds stream-cancellation regression tests.

## Best Possible Solution

Merge the bounded-reader fix after maintainer acceptance of the response-size budgets, keeping the behavior local to the Mattermost plugin and existing SDK helpers.
