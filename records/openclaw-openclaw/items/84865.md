## Summary

Keep open: current main still leaves user-sourced model overrides with an empty fallback list, and the linked implementation PR is open with a policy question about whether exact user model selections may fall back automatically.

## What This Changes

The issue asks OpenClaw to recover from provider outages when a user-selected model is active, without permanently replacing the user's chosen model.

## Best Possible Solution

Keep the issue open until maintainers choose the policy: either preserve strict user model pins and fix the stall into a visible failure, or intentionally allow non-persistent fallback for user-selected models with aligned docs and regression coverage.

## Reproduction Assessment

Partly: source and tests show user-sourced model overrides resolve to an empty fallback list on current main. The full provider-outage stall was not reproduced end to end in this read-only review.

## Solution Assessment

Unclear: allowing fallbacks for user-selected models would address the incident path, but it changes the current documented strict-selection contract. Maintainers should decide the policy before landing the linked PR or choosing a visible-failure fix.
