## What This Changes

The issue reports that user-selected session models have no fallback candidates during provider outages, causing the session to stall instead of recovering or failing visibly.

## Best Possible Solution

Review the linked PR and decide the model-selection policy: either keep user pins strict while making outage failures visible and non-stalling, or intentionally add fallback eligibility with aligned docs, tests, and proof.
