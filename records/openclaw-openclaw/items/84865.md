## Summary

Keep open: current main still disables configured fallbacks for user-sourced model overrides, and https://github.com/openclaw/openclaw/pull/84867 is an open closing PR where maintainers can decide the exact model-pin policy.

## What This Changes

The issue requests per-call fallback recovery when a user-selected session model becomes unavailable, without persisting the fallback selection.

## Best Possible Solution

Resolve the policy in the linked PR: either keep user pins exact and make outages fail visibly without stalls, or explicitly approve fallback-eligible user pins with aligned docs, tests, and live proof.

## Reproduction Assessment

Partly: current source and tests show user-sourced model overrides resolve to an empty fallback list on main. The full provider-outage deadlock was not run end-to-end in this read-only review.

## Solution Assessment

Unclear: fallback for user-selected models could address availability, but it changes the documented exact-selection contract. Maintainers should choose the policy before landing the linked PR or a strict visible-failure alternative.
