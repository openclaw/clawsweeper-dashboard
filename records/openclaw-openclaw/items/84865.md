## Summary

Keep open: current main and the latest release still make user-sourced model overrides strict, and the issue already has an open closing PR where maintainers need to decide the policy change.

## What This Changes

The issue asks OpenClaw to avoid session deadlock when a user-pinned model fails, potentially by allowing configured model fallbacks for user-switched models.

## Best Possible Solution

Prevent provider outages from wedging sessions while keeping user-model routing semantics explicit: either preserve strict pins and fail visibly, or adopt a maintainer-approved fallback policy with aligned tests, docs, and runtime proof.

## Reproduction Assessment

Partly: current source and tests show user-sourced model overrides resolve to an empty fallback list and make only one attempt. The full provider-outage deadlock was not reproduced end to end in this read-only review.

## Solution Assessment

Unclear: allowing fallbacks for user-selected models may fix availability, but it changes documented exact-pin semantics. Maintainers should choose the policy before landing the linked PR or a narrower visible-failure repair.
