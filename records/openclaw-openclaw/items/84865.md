## Summary

Keep open: current main and the latest release still keep user-selected models strict, while the linked closing PR and competing opt-in policy PR remain open and unmerged. The remaining decision is the provider/model routing contract for exact user selections, not conservative cleanup.

## What This Changes

The issue requests that user-selected models keep a fallback path during provider outages instead of becoming strict session pins with no recovery chain.

## Best Possible Solution

Maintainers should choose one user-selected model fallback contract, then land a single aligned implementation with resolver behavior, docs, tests, and real runtime proof.

## Reproduction Assessment

Yes for the source-level behavior: current main returns `[]` for user-sourced session model overrides, and the current command test asserts one failed selected-model attempt with no fallback. The full live provider-outage deadlock was not reproduced in this read-only review.

## Solution Assessment

Unclear: enabling fallback for every user-selected model is a plausible availability fix, but it changes the documented exact-selection contract. A maintainer-approved direct change or strict-default opt-in policy is needed before implementation can be called the best fix.
