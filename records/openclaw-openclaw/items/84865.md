## What This Changes

The issue asks for user-selected models to have a recovery fallback path during provider outages instead of leaving a session stuck on the failing selected model.

## Best Possible Solution

Maintainers should decide the user-pin contract, then land or rework the linked fallback PR or an opt-in policy PR with matching docs, tests, and runtime proof while keeping strict visible failure if fallback routing is rejected.

## Reproduction Assessment

## Solution Assessment

Unclear. Allowing fallback for every user-selected model is a plausible availability fix, but it conflicts with the documented exact-selection contract; an explicit opt-in fallback policy or visible-failure repair may be safer depending on maintainer direction.
