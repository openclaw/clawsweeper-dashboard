## What This Changes

The issue reports that a manually selected session model has no configured fallback chain and can leave a session stalled during a provider outage.

## Best Possible Solution

Keep user model selection semantics explicit: either preserve strict selections and fix the stall as a visible failure, or land a maintainer-approved opt-in fallback policy with matching docs, tests, and runtime proof.

## Solution Assessment
