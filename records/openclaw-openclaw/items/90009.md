## Summary

Keep open: current main and the latest release still persist raw standalone debug-proxy headers, and the linked fixing PR is open rather than merged.

## What This Changes

Request redaction of sensitive request, response, and CONNECT headers before standalone debug proxy capture events store `headers_json`.

## Best Possible Solution

Refresh, review, and land or replace https://github.com/openclaw/openclaw/pull/82951 so the standalone proxy reuses the shared sensitive-header redaction at all current persistence sites.

## Reproduction Assessment

Yes. Source inspection on current main shows `openclaw proxy start` stores raw HTTP response, HTTP request, and CONNECT headers into `headers_json`; a live repro would start the proxy, send sensitive headers through it, and query the capture SQLite database.

## Solution Assessment

Yes. Reusing the existing sensitive-header redaction logic for the standalone proxy is the narrow maintainable fix, but this issue should stay open until the linked PR or equivalent lands.
