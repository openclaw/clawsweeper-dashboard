## Summary

Keep open: current main and v2026.6.8 still persist raw standalone debug-proxy HTTP response, HTTP request, and CONNECT headers, while the linked fixing PR remains open rather than merged.

## What This Changes

The issue asks for standalone debug proxy capture storage to redact sensitive request, response, and CONNECT headers before writing `headers_json`.

## Best Possible Solution

Land or refresh https://github.com/openclaw/openclaw/pull/82951 so future captures redact all standalone proxy header persistence sites, then decide separately whether old capture rows need purge or rotation guidance.

## Reproduction Assessment

Yes. Source inspection on current main shows `openclaw proxy start` persists raw HTTP response, HTTP request, and CONNECT headers into `headers_json`; a live repro would start the proxy, send sensitive headers, and query `capture_events`.

## Solution Assessment

Yes. Reusing the existing sensitive-header redaction policy at the standalone proxy persistence sites is the narrow maintainable fix, but this issue should stay open until the linked PR or equivalent lands.
