## Summary

Keep open: current main and the latest release still persist raw standalone debug-proxy request, response, and CONNECT headers, and the linked fixing PR remains open rather than merged.

## What This Changes

The issue asks for standalone debug proxy capture to redact sensitive headers before storing request, response, and CONNECT events.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/82951 so standalone proxy storage uses the same sensitive-header redaction contract as runtime capture.

## Reproduction Assessment

Yes. Source inspection on current main shows the standalone proxy serializes raw request, response, and CONNECT headers into capture events; a live repro would start the proxy, send sensitive headers, and query capture_events.

## Solution Assessment

Yes. Reusing the existing runtime sensitive-header predicate at the standalone proxy persistence sites is the narrow maintainable fix, but this issue should stay open until that fix or an equivalent one lands.
