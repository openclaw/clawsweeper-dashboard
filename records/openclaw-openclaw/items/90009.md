## What This Changes

The issue reports that `openclaw proxy start` persists raw request, response, and CONNECT headers to debug capture storage instead of applying the runtime capture redaction policy.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/82951 so standalone proxy storage redacts all current header persistence sites and documents any old-capture cleanup expectation.

## Reproduction Assessment

Yes. Source inspection on current main shows `openclaw proxy start` persists raw HTTP response, request, and CONNECT headers into `headers_json`; a live repro would start the proxy, send sensitive headers, and query `capture_events`.

## Solution Assessment

Yes. Reusing the existing runtime sensitive-header redaction policy at the standalone proxy persistence sites is the narrow maintainable fix, but this issue should stay open until the linked PR or equivalent lands.
