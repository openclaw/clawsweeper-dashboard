## What This Changes

The issue requests redacting standalone debug proxy captured request, response, and CONNECT headers with the same sensitive-header policy used by the runtime capture path.

## Best Possible Solution

## Reproduction Assessment

Yes. Source inspection on current main shows `openclaw proxy start` persists raw HTTP response, HTTP request, and CONNECT headers into `headersJson`; a live repro would start the proxy, send sensitive headers, and query `capture_events`.

## Solution Assessment

Yes. Reusing the existing runtime sensitive-header policy at the standalone proxy persistence sites is the narrow maintainable fix, but this issue should stay open until the linked PR or equivalent lands.
