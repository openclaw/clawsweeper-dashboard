## Summary

Keep this PR open. Current main still has the stale bookkeeping path for hook-adjusted message and cron tool params, and the PR contains a plausible fix with focused tests, but it needs to be ported onto current main and needs the required changelog entry before merge.

## What This Changes

The PR changes embedded agent tool-end bookkeeping to consume `before_tool_call` adjusted params for message/cron state, adds regression tests, and broadens transcript repair for snake_case raw tool-call blocks.

## Best Possible Solution

Port the adjusted-param message/cron bookkeeping fix onto current main with focused regression coverage and a single-line changelog entry; split the transcript-repair change unless maintainers intentionally keep it bundled.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a `before_tool_call` hook rewrites a `message` or `cron` call after `tool_execution_start`, the wrapped tool executes the adjusted params, but current main commits message/cron bookkeeping from the original start args.

## Solution Assessment

Yes, with cleanup. Consuming adjusted params once at the start of `handleToolExecutionEnd` and reusing that value for message, cron, and after-hook bookkeeping is the narrow maintainable fix; the safer landing path is to port it cleanly and split unrelated transcript repair.
