## What This Changes

The PR changes embedded agent tool-end bookkeeping to commit hook-adjusted message and cron tool params, adds message/cron regression tests, and also broadens transcript repair for snake_case raw tool-call block types.

## Best Possible Solution

Port the adjusted-param message/cron bookkeeping fix onto current main with focused regressions and a changelog entry, while leaving the transcript-repair change for a separate reviewed PR unless maintainers intentionally keep it bundled.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a `before_tool_call` hook rewrites a `message` send after `tool_execution_start`, the wrapped tool executes the adjusted params, but current main commits message/cron bookkeeping from the original start args before consuming the adjusted params.

## Solution Assessment

Yes, with cleanup. Consuming adjusted params once at the start of `handleToolExecutionEnd` and reusing that value for message, cron, and after-hook bookkeeping is the narrow maintainable fix; the safer landing path is to rebase/port it and split the unrelated transcript repair.
