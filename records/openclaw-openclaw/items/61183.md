## Summary

Keep this PR open. Current main still commits embedded message-tool sent text, targets, and media from the stale tool-start args before adjusted `before_tool_call` params are consumed, while the PR applies the right source of truth for message and cron bookkeeping. It still needs maintainer rebase/review because the branch is stale against current main, lacks the required changelog entry, and includes a separate transcript-repair expansion.

## What This Changes

The PR changes embedded tool-end bookkeeping to consume adjusted `before_tool_call` params for message and cron commits, adds message/cron regression tests, and broadens transcript repair/tests to recognize snake_case raw tool-call block types.

## Best Possible Solution

Rebase the adjusted-param bookkeeping fix onto current main, keep the message and cron regression coverage, add the required changelog entry, and either split the transcript-repair change or land it as an explicitly reviewed companion fix.

## Reproduction Assessment

Yes. A high-confidence reproduction is a `before_tool_call` hook that rewrites a `message` send after `tool_execution_start`; current main executes the adjusted params but commits sent-message bookkeeping from the original start args before consuming the adjusted params.

## Solution Assessment

Yes, with cleanup. Consuming adjusted params once at the start of `handleToolExecutionEnd` and reusing that value for message, cron, and `after_tool_call` bookkeeping is the narrow maintainable fix; the safer landing path is to rebase and separate the unrelated transcript repair if needed.
