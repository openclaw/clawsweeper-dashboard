## Summary

Keep open. Current main still commits message and cron bookkeeping from stale `tool_execution_start` args even though `before_tool_call` can rewrite the params actually executed; the PR addresses the right bug but still needs a current-main landing path and required changelog coverage.

## What This Changes

The PR updates embedded agent tool-end bookkeeping to use `before_tool_call`-adjusted message and cron params, adds regressions, and broadens transcript repair for snake_case raw tool-call blocks.

## Best Possible Solution

Land a focused current-main fix that consumes adjusted params once in `handleToolExecutionEnd` for message and cron bookkeeping, with regression tests and a changelog entry; handle transcript repair separately unless maintainers intentionally include it.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a `before_tool_call` hook rewrites a `message` or `cron` call, the wrapper executes adjusted params, but current main commits message/cron bookkeeping from the original start args.

## Solution Assessment

Yes, with cleanup. Consuming adjusted params once at the start of `handleToolExecutionEnd` and reusing them for message, cron, and after-hook bookkeeping is the narrow maintainable fix; the safer landing path is to port/rebase it and split unrelated transcript repair.
