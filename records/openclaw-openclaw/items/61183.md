## Summary

Keep open. The PR addresses a real current-main bookkeeping gap, but the current head still drops route-scoped text/media evidence needed by final-payload dedupe and the external PR has no real after-fix behavior proof.

## What This Changes

The PR updates embedded agent message/cron tool-end bookkeeping to consume adjusted `before_tool_call` params, adds regression tests for message-tool hook rewrites, and includes snake_case transcript-repair changes.

## Best Possible Solution

Port the adjusted-param bookkeeping fix while preserving target-level text/media evidence, keep cron bookkeeping aligned with adjusted params, and drop obsolete transcript-repair churn during rebase.

## Reproduction Assessment

Yes. Current main stores and executes adjusted `before_tool_call` params, but embedded message bookkeeping still commits from start-event args, so a `message` hook rewrite can leave stale sent-text/target/media evidence.

## Solution Assessment

No, not as-is. Consuming adjusted params at tool end is the right direction, but the PR must attach the adjusted text/media to the committed target record and should avoid unrelated transcript-repair changes.
