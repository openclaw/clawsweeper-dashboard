## Summary

Keep open. Current main still lacks the direct `anthropic` Claude-name image-support fallback, and this PR remains a focused fix with regression coverage, but the branch is not merge-ready because the user-facing gateway fix needs a changelog entry.

## What This Changes

The PR extends `resolveGatewayModelSupportsImages` so direct `anthropic` Claude model names share the existing `claude-cli` image-support fallback when catalog metadata is stale or missing, and adds two regression tests.

## Best Possible Solution

Add the required changelog entry, then land this focused fallback and its regression tests unless maintainers choose the longer-term provider-owned capability/catalog route. The desired end state is that direct Anthropic Claude models keep image attachments usable when catalog metadata lags behind model releases.

Yes. A focused resolver check with `provider: "anthropic"`, `model: "claude-opus-4-7"`, and either an empty catalog or a text-only catalog entry follows current main to `false`, which then feeds the chat attachment parser as `supportsImages: false`.

Mostly yes. The functional patch is the narrowest change against the existing `claude-cli` fallback pattern, though a provider-owned capability or catalog hook would be cleaner long-term; the current PR still needs the changelog entry before it is merge-ready.
