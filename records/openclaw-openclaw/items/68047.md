## Summary

Keep this PR open. Current main has mitigations for text-only image offload, but `resolveGatewayModelSupportsImages` still returns `false` for direct `anthropic` Claude names when catalog metadata is stale or missing, so the PR remains relevant for preserving native image handling; it is not merge-ready because the user-facing fix lacks the required changelog entry.

## What This Changes

The PR extends the gateway Claude-name image-support fallback from `claude-cli` to direct `anthropic` models and adds stale-catalog and missing-catalog regression tests in `src/gateway/session-utils.test.ts`.

## Best Possible Solution

Add the required single-line changelog entry, then either land this focused fallback with its regression tests or replace it with a provider-owned capability seam that keeps direct Anthropic Claude models image-capable when gateway catalog metadata lags.

## Reproduction Assessment

Yes. A focused resolver call with `provider: "anthropic"`, `model: "claude-opus-4-7"`, and either an empty catalog or a text-only catalog row returns `false` on current main, which makes chat attachment handling use text-only media-path routing instead of native image blocks.

## Solution Assessment

Unclear as the final architecture. The patch is a narrow functional fix for the observed resolver gap, but it is not merge-ready without the changelog entry, and a provider-owned capability hook would be the cleaner long-term alternative if maintainers want to avoid gateway provider-id heuristics.
