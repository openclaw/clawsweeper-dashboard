## Summary

Keep open: current main blocks unauthorized Telegram inbound traffic and has config-focused security audit checks, but it does not provide the requested runtime alert, event, or recent blocked-attempt history surface.

## What This Changes

Add a supported Telegram runtime audit and alert path for blocked unauthorized inbound DMs, groups, group additions, and rejected commands.

## Best Possible Solution

Design a Telegram-owned runtime security-event path with redacted metadata, rate limiting, durable inspection, and optional owner/webhook delivery, adding a generic channel seam only if other channels need the same contract.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection show current main enforces Telegram allowlists and exposes static security-audit checks, but no runtime alert or recent blocked-attempt history surface.

## Solution Assessment

Unclear until maintainers choose the product/security shape. A Telegram-owned event/audit implementation with privacy-safe metadata is more maintainable than documenting log scraping or routing unauthorized content into agent turns.
