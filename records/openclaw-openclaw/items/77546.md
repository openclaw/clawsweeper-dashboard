## Summary

Keep open: current main enforces Telegram allowlist blocks and exposes static config security audit findings, but it still does not provide the requested runtime alert, notification, or recent blocked-attempt audit surface.

## What This Changes

Request adding Telegram runtime security alerts and audit history for unauthorized DM, group, allowlist, and command attempts with configurable delivery and privacy controls.

## Best Possible Solution

Design a Telegram-owned runtime security-event pipeline with redacted metadata, rate limiting, durable inspection, and optional owner/webhook delivery, adding a generic channel seam only if other channels need the same contract.

## Reproduction Assessment

Not applicable as a feature request. Source, tests, and docs show current main blocks unauthorized Telegram events and audits static config, but lacks the requested runtime alert/history surface.

## Solution Assessment

Unclear until maintainers choose the product and security shape. The requested Telegram-owned audit path is more maintainable than log scraping, but delivery modes, storage, redaction, and cross-channel scope need design first.
