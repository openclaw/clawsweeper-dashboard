## Best Possible Solution

Land one canonical TUI fix that suppresses only a visible replay/live-final duplicate, preserves stale-history recovery and external reset reloads, and includes focused tests plus redacted real TUI/Gateway or Telegram proof.

## Solution Assessment

No. The current PR is a plausible event-handler direction, but the latest display-map clearing makes late finals pass even when history replay already restored the row; a visible-row-aware ChatLog or replay reconciliation fix is safer.
