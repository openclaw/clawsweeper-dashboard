## Summary

Keep open. Current main still has the direct-first completion dispatch and lacks the persisted delivery-claim/user-payload state this PR adds, but the PR head has two blocking correctness bugs in claim cleanup and iMessage chunk normalization before it is mergeable.

## What This Changes

The PR adds persisted subagent delivery-claim and user-delivery payload state, changes completion dispatch toward queue-first active-parent routing, hardens iMessage outbound normalization, and updates focused agent/iMessage tests.

## Best Possible Solution

Keep the hardened-delivery direction, but make claim lifecycle cleanup cover every early exit, perform dangling-fence closure only at a full-message boundary before iMessage chunking, update the subagents docs/changelog for the delivery behavior change, and then rerun the focused agent and iMessage tests before maintainer merge review.

## Reproduction Assessment

Yes for the PR-head defects: source inspection shows a persisted claim can be created before an already-aborted return that bypasses cleanup, and iMessage text is normalized before chunking and again during per-chunk send. The original production leak/duplicate-delivery path was not live-reproduced in this read-only review.

## Solution Assessment

No. The queue-first and persisted-payload architecture appears to address a real current-main gap, but this patch is not the best mergeable form until claim cleanup and iMessage normalization boundaries are corrected.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Release the claim on early abort:** `src/agents/subagent-announce-dispatch.ts:185`
  - body: `beginPersistedAnnounceDeliveryClaim` can write a fresh `claimed` state before the already-aborted signal guard. That return runs before the `finally`, so later retries for the same announce can be rejected as `delivery-already-in-flight` until the lease expires.
  - confidence: 0.92
- **[P2] Skip per-chunk fence normalization:** `extensions/imessage/src/send.ts:146`
  - body: `deliverReplies` normalizes the whole reply before chunking, but each chunk now passes through `sendMessageIMessage`, which closes dangling fences again. A long fenced code block split across chunks can get an injected closing fence in the middle of the delivered response.
  - confidence: 0.87
