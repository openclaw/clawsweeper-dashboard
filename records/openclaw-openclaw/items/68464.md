## Summary

Keep open: current main has not absorbed this PR's central durable delivery-claim/user-payload design, but the branch is not mergeable as-is because source inspection confirms blocking claim-cleanup and iMessage normalization defects, missing runtime proof, and overlap with active subagent completion work.

## What This Changes

The PR adds persisted subagent delivery claims and user-facing completion payload storage, changes completion announce dispatch toward queue-first delivery, and hardens iMessage outbound sanitization and chunking.

## Best Possible Solution

Consolidate this hardening into the maintained subagent completion-delivery design, preserving durable user payloads and queue-first semantics while fixing claim finalization, iMessage normalization boundaries, docs, and runtime proof before merge.

## Reproduction Assessment

Yes for the PR-head defects: source inspection shows a persisted claim can be created before an already-aborted return that bypasses cleanup, and chunked iMessage text can be normalized again per send. The original intermittent production reports remain discussion-backed but were not live-reproduced in this read-only pass.

## Solution Assessment

No. The direction is plausible, but this PR is not the best mergeable solution until claim cleanup, iMessage normalization, docs alignment, real behavior proof, and overlap with https://github.com/openclaw/openclaw/pull/76927 and https://github.com/openclaw/openclaw/pull/43992 are resolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Release delivery claims on early abort:** `src/agents/subagent-announce-dispatch.ts:200-204`
  - body: `beginPersistedAnnounceDeliveryClaim` can create a claim before this already-aborted return. Because the return happens before the `finally` block, retries for the same announce can be rejected as `delivery-already-in-flight` until the lease expires.
  - confidence: 0.92
- **[P2] Skip per-chunk iMessage fence normalization:** `extensions/imessage/src/send.ts:146`
  - body: `deliverReplies` normalizes the full reply before chunking, but `sendMessageIMessage` normalizes each chunk again. A fenced block split across chunks can receive extra closing fences and render corrupted output.
  - confidence: 0.87
- **[P3] Align docs with queue-first completion delivery:** `src/agents/subagent-announce-dispatch.ts:213-215`
  - body: This dispatch path now attempts queue delivery first, but the subagent docs on the branch still describe direct `agent` delivery first with queue fallback. Update the docs so the released contract matches the code.
  - confidence: 0.84
