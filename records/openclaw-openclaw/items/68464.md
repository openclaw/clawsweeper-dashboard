## Summary

Keep open. Current main still lacks the persisted delivery claim/user-payload state and iMessage internal-envelope stripping that this PR targets, so the work is not obsolete; however, the PR head still has merge-blocking correctness issues around claim cleanup and iMessage chunk normalization, plus missing user-facing docs/changelog updates.

## What This Changes

The PR adds persisted subagent completion delivery-claim and user-payload fields, switches completion announce dispatch toward queue-first routing, and expands iMessage outbound sanitization/chunking coverage with focused tests.

## Best Possible Solution

Keep the hardened-delivery direction, but make claim cleanup fail-closed on every early exit, keep iMessage fence closing at a single full-message boundary, add public docs and changelog coverage for the queue-first behavior change, and validate the fix against the linked missing/dropped payload reports before maintainer merge review.

## Reproduction Assessment

Yes for the PR-head defects: source inspection shows a persisted claim can be created before an already-aborted return that bypasses cleanup, and iMessage text is normalized before chunking and again during each send. The original production leak/reconnect scenarios are supported by related reports but were not live-reproduced in this read-only review.

## Solution Assessment

No. The queue-first and persisted-payload architecture appears to address a real current-main gap, but this PR is not yet the best mergeable form until claim finalization, iMessage normalization boundaries, docs, and changelog coverage are fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Release the claim on early abort:** `src/agents/subagent-announce-dispatch.ts:200-204`
  - body: `beginPersistedAnnounceDeliveryClaim` can persist a fresh claimed state before this already-aborted return. Because the return happens before the `finally`, the claim is not deleted or finalized, so later retries for the same announce can be rejected as `delivery-already-in-flight` until the lease expires.
  - confidence: 0.92
- **[P2] Skip per-chunk fence normalization:** `extensions/imessage/src/send.ts:146`
  - body: Monitor delivery already normalizes the full reply before splitting it into chunks, but this send path normalizes each chunk again. If a long fenced code block is split across chunks, an intermediate chunk can have an odd fence count and this call appends a closing fence, corrupting the delivered formatting.
  - confidence: 0.86
- **[P3] Document the queue-first delivery change:** `src/agents/subagent-announce-dispatch.ts:213-214`
  - body: This changes completion dispatch to queue-first, but the PR does not update docs or `CHANGELOG.md`; `docs/tools/subagents.md` still says manual-spawn completion tries direct `agent` delivery first. Add the public docs/changelog updates so released behavior is described correctly.
  - confidence: 0.86
