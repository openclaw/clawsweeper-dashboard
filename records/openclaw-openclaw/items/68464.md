## Summary

Keep open: current main does not implement the central queue-first/durable-claim behavior, and this PR still has source-backed blockers plus missing real behavior proof before it is mergeable.

## What This Changes

The branch adds persisted subagent delivery claims and stored user delivery payloads, switches completion announces to queue-first routing, and hardens iMessage outbound sanitization/chunking.

## Best Possible Solution

Consolidate the hardening around one durable completion-delivery design, fix claim cleanup and iMessage normalization boundaries, align docs at landing, and require redacted runtime proof before merge.

## Reproduction Assessment

Yes for the PR-head defects: source inspection shows a persisted claim can be created before an already-aborted return that bypasses cleanup, and chunked iMessage text can be normalized again per send. The original intermittent production reports remain discussion-backed but were not live-reproduced here.

## Solution Assessment

No. The direction is plausible, but this PR is not the best mergeable solution until the claim-finalization bug, iMessage normalization boundary, docs mismatch, real behavior proof, and overlap with https://github.com/openclaw/openclaw/pull/76927 and https://github.com/openclaw/openclaw/pull/43992 are resolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Release delivery claims on early abort:** `src/agents/subagent-announce-dispatch.ts:200-204`
  - body: When `announceId` is present, `beginPersistedAnnounceDeliveryClaim` can create a claim before this already-aborted return. Because the return happens before the `finally` block, later retries can be rejected as `delivery-already-in-flight` until the lease expires.
  - confidence: 0.92
- **[P2] Avoid normalizing delivered iMessage chunks:** `extensions/imessage/src/send.ts:146`
  - body: `deliverReplies` normalizes the full reply before chunking, but `sendMessageIMessage` normalizes each chunk again. A long fenced code block split across chunks can get extra closing fences appended to individual chunks and corrupt the delivered formatting.
  - confidence: 0.87
- **[P3] Align docs with queue-first completion delivery:** `src/agents/subagent-announce-dispatch.ts:213-214`
  - body: This changes completion dispatch to queue-first, but the subagent docs on the branch still describe direct `agent` delivery first with queue fallback. Update the docs so released behavior matches the new delivery order.
  - confidence: 0.84
