Overall correctness: patch is incorrect

Full review comments:

- **[P2] Release delivery claims on early abort:** `src/agents/subagent-announce-dispatch.ts:200-204`
  - body: When `announceId` is set, `beginPersistedAnnounceDeliveryClaim` can persist a claim before this already-aborted return. Because this return is outside the `try`/`finally`, the claim is not deleted or finalized, so later retries can be blocked as `delivery-already-in-flight` until the lease expires.
  - confidence: 0.92
- **[P2] Avoid normalizing delivered iMessage chunks:** `extensions/imessage/src/send.ts:146`
  - body: `deliverReplies` normalizes the full reply before chunking, but this line normalizes each chunk again. A long fenced code block split across chunks can have an odd fence count per chunk, causing extra closing fences to be appended and corrupting delivered formatting.
  - confidence: 0.88
- **[P3] Update docs for queue-first completion delivery:** `src/agents/subagent-announce-dispatch.ts:213-214`
  - body: This change makes completion dispatch queue-first, but the branch's task delivery docs still describe direct delivery first with session-queued delivery only as fallback. Update the docs before landing so released behavior matches the documented delivery order.
  - confidence: 0.82
