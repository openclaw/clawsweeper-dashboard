  - body: This early return runs after `beginPersistedAnnounceDeliveryClaim()` has already written `entry.deliveryClaim`, but it is still before the `try/finally` that calls `finalizePersistedAnnounceDeliveryClaim()`.   - confidence: 0.93
- **[P2] Avoid closing fences inside each iMessage chunk:** `extensions/imessage/src/send.ts:146`
  - body: `deliverReplies` already normalizes the whole payload before splitting it into chunks, but this send path normalizes each chunk again. Because `normalizeIMessageDeliveryText()` closes an odd fenced block, any chunk split inside a long code fence can get an extra closing fence and corrupt the rendered code output.
  - confidence: 0.88
