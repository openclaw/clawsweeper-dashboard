## Summary

Keep open. Current main still lacks the durable claim, user-facing payload, queue-first completion path, and iMessage internal-context filtering this PR targets, so the work is not obsolete. The PR head still has blocking correctness issues in persisted-claim cleanup and iMessage code-fence normalization, with related reports showing this delivery area remains active.

## What This Changes

The PR adds persisted subagent delivery-claim and user-payload state, changes completion dispatch toward queue-first routing, and updates iMessage outbound sanitization, chunking, send behavior, and focused tests.

## Best Possible Solution

Keep the hardened-delivery direction, but fix claim cleanup on every early exit, make iMessage fence normalization happen at one full-message boundary, update docs/changelog for the behavior change, and validate against the related missing-payload reports before maintainer merge review.

## Reproduction Assessment

Yes for the PR-head defects: source inspection shows a persisted claim can be created before an already-aborted return that bypasses cleanup, and iMessage text is normalized before chunking and again during per-chunk send. The original production leak/duplicate-delivery scenario was not live-reproduced in this read-only review.

## Solution Assessment

No. The queue-first and persisted-payload direction appears appropriate for the current-main gap, but this patch is not the best mergeable form until claim cleanup and iMessage normalization boundaries are corrected.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Release the claim on early abort:** `src/agents/subagent-announce-dispatch.ts:185`
  - body: `beginPersistedAnnounceDeliveryClaim` can persist a fresh `claimed` state before the already-aborted signal return below it. Because that return happens before the `finally`, the claim is not deleted or finalized, so later retries for the same announce can be rejected as `delivery-already-in-flight` until the lease expires.
  - confidence: 0.92
- **[P2] Skip per-chunk fence normalization:** `extensions/imessage/src/send.ts:146`
  - body: Monitor delivery already normalizes the full reply before splitting it into chunks, but this send path normalizes each chunk again. If a long fenced code block is split across chunks, an intermediate chunk can have an odd fence count and this call appends a closing fence, corrupting the delivered formatting.
  - confidence: 0.86
