## Summary

Keep open. Current main still lacks the PR's durable delivery-claim/user-payload fields, queue-first active-parent completion routing, and OpenClaw-internal iMessage sanitization, so the branch is not obsolete; however, the current PR head has concrete merge blockers in claim cleanup, iMessage chunk handling, config preservation, and docs/changelog coverage.

## What This Changes

The PR adds persisted subagent announce delivery claims and user-facing payloads, changes completion announce dispatch to queue-first with direct fallback, and expands legacy iMessage outbound sanitization/chunking behavior and tests.

## Best Possible Solution

Land a repaired version of this hardening that keeps durable completion claims and user-safe payload persistence, routes active parent completions through the queue first, preserves iMessage formatting/config contracts, and documents the released behavior so the related completion-delivery reports can be retested against one coherent implementation.

## Reproduction Assessment

Yes for the PR-head blockers: source inspection shows the persisted claim is created before an already-aborted return that bypasses finalization, monitor delivery normalizes before chunking while send normalizes each chunk again, and the diff removes the existing `chunkMode` resolution. The original production reconnect/leak scenarios are supported by related reports but were not live-reproduced in this read-only pass.

## Solution Assessment

No. The architecture is a plausible direction for the reported durability and leakage bugs, but this PR is not the best mergeable form until claim cleanup, single-boundary iMessage normalization, `chunkMode` preservation, docs, and changelog coverage are fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Release the claim on early abort:** `src/agents/subagent-announce-dispatch.ts:200-204`
  - body: `beginPersistedAnnounceDeliveryClaim` can persist a fresh claimed state before this already-aborted return. Because the return happens before the `finally`, the claim is not deleted or finalized, so later retries for the same announce can be rejected as `delivery-already-in-flight` until the lease expires.
  - confidence: 0.92
- **[P2] Normalize iMessage text before chunking only once:** `extensions/imessage/src/send.ts:146`
  - body: Monitor delivery normalizes the full reply before splitting it into chunks, but this send path normalizes each chunk again. If a long fenced code block is split across chunks, an intermediate chunk can have an odd fence count and this call appends a closing fence, corrupting the delivered formatting.
  - confidence: 0.88
- **[P2] Keep honoring iMessage chunkMode:** `extensions/imessage/src/monitor/deliver.ts:123-125`
  - body: This replacement no longer calls `resolveChunkMode`, so `channels.imessage.chunkMode` and account-level overrides are ignored. Preserve the existing `length`/`newline` contract while adding the safer code-aware splitting behavior.
  - confidence: 0.84
- **[P3] Update subagent delivery docs and changelog:** `src/agents/subagent-announce-dispatch.ts:213-214`
  - body: The PR changes completion delivery from direct-first to queue-first, but `docs/tools/subagents.md` still documents direct-first behavior and there is no user-facing `CHANGELOG.md` entry for this fix. Update both before merge.
  - confidence: 0.86
