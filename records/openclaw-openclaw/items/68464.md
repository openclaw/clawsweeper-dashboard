  - body: `beginPersistedAnnounceDeliveryClaim` has already persisted `state: "claimed"` here, but this early aborted-signal return happens before the `try`/`finally` that calls `finalizePersistedAnnounceDeliveryClaim`. The next retry for the same announce can therefore see `delivery-already-in-flight` until the 5-minute lease expires, delaying or dropping the completion delivery path this PR is meant to harden.
  - confidence: 0.92
- **[P2] Avoid normalizing fenced chunks in sendMessageIMessage:** `extensions/imessage/src/send.ts:146`
  - body: Monitor delivery normalizes the full iMessage reply before chunking, then sends each chunk through `sendMessageIMessage`. Closing dangling fences again on every chunk means a long fenced block split across chunks can receive extra closing fences in the middle of the response and render corrupted code to the user.
  - confidence: 0.87
