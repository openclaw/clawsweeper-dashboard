## Summary

Keep open. Current main does not already contain the PR's central durable delivery-claim/user-payload work or queue-first completion dispatch, related completion-delivery issues remain open, and this PR still has blocking correctness findings plus missing real behavior proof.

## What This Changes

The PR adds persisted subagent completion delivery claims and user-facing payloads, changes completion announce dispatch toward queue-first routing, and expands iMessage outbound sanitization/chunking coverage.

## Best Possible Solution

Rebase the branch, keep the durable completion-delivery direction, fix claim finalization and iMessage normalization boundaries, update the public subagent docs for queue-first behavior, and add real redacted runtime proof before merge.

## Reproduction Assessment

Yes for the PR-head defects: source inspection shows a persisted claim can be created before an already-aborted return that bypasses cleanup, and iMessage text can be normalized once before chunking and again per chunk. The original intermittent production reports are discussion-backed but were not live-reproduced here.

## Solution Assessment

No. The hardening direction is appropriate, but this PR is not the best mergeable form until claim cleanup, iMessage normalization boundaries, documentation, rebase conflicts, and real behavior proof are addressed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Release the claim on early abort:** `src/agents/subagent-announce-dispatch.ts:200-204`
  - body: When `announceId` is present, `beginPersistedAnnounceDeliveryClaim` can create a fresh claim before this already-aborted return. Because the return occurs before the `finally`, the claim remains in-flight until lease expiry and later retries can be rejected as `delivery-already-in-flight`.
  - confidence: 0.92
- **[P2] Skip per-chunk fence normalization:** `extensions/imessage/src/send.ts:146`
  - body: Monitor delivery normalizes the full reply before splitting it into chunks, but this send path normalizes each chunk again. A long fenced code block split across chunks can get an extra closing fence appended to an intermediate chunk, corrupting the delivered formatting.
  - confidence: 0.86
- **[P3] Document queue-first completion delivery:** `src/agents/subagent-announce-dispatch.ts:213-214`
  - body: This changes completion dispatch to queue-first, but the public subagent docs still describe requester-agent direct handoff first and queue fallback afterward. Update the docs so released behavior matches the new delivery order.
  - confidence: 0.84
