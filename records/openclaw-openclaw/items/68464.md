## Summary

Keep open. Current main has not fully absorbed the PR's hardened completion-delivery behavior, and the latest discussion still points to concrete needs-changes work: claim cleanup on early abort, iMessage chunk normalization, docs alignment, and real behavior proof before merge.

## What This Changes

The branch adds persisted subagent completion delivery claims and user payloads, changes active-parent completion dispatch toward queue-first routing, and expands iMessage outbound sanitization/chunking coverage.

## Best Possible Solution

Rebase the branch, fix fail-closed claim finalization and iMessage normalization boundaries, update the queue-first docs, and validate with targeted tests plus redacted real reconnect/iMessage delivery proof.

## Reproduction Assessment

Yes for the PR-head defects: source inspection shows a persisted claim can be created before an early aborted return that bypasses finalization, and iMessage text can be normalized once before chunking and again per chunk. The original production leak/reconnect bugs are supported by linked reports but were not live-reproduced here.

## Solution Assessment

No. The direction addresses a real current-main gap, but this PR is not the best mergeable form until the claim cleanup, per-chunk code-fence normalization, docs mismatch, stale base, and real behavior proof are handled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Release the claim on early abort:** `src/agents/subagent-announce-dispatch.ts:200-204`
  - body: `beginPersistedAnnounceDeliveryClaim` can persist a fresh claimed state before this already-aborted return. Because this return happens before the `finally`, the claim is not deleted or finalized, so later retries for the same announce can be rejected as `delivery-already-in-flight` until the lease expires.
  - confidence: 0.92
- **[P2] Avoid closing code fences per chunk:** `extensions/imessage/src/send.ts:146`
  - body: Monitor delivery normalizes the full reply before splitting it into chunks, but this send path normalizes each chunk again. If a long fenced code block is split across chunks, an intermediate chunk can have an odd fence count and this call appends a closing fence, corrupting the delivered formatting.
  - confidence: 0.86
- **[P3] Document queue-first completion dispatch:** `src/agents/subagent-announce-dispatch.ts:213-214`
  - body: This changes completion dispatch to queue-first, but the public subagent docs still describe top-level completion delivery as direct-first. Update the docs with the new queue-first behavior so released behavior and user guidance stay aligned.
  - confidence: 0.82
