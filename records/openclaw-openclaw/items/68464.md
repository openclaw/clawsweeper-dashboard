## Summary

Keep open: this PR is still a plausible fix direction, but the current branch is not mergeable because two source-reproducible defects remain, real behavior proof is missing, and active related completion-delivery work still needs consolidation.

## What This Changes

The PR adds persisted subagent delivery claims and user-safe payloads, switches completion announces to queue-first routing, and hardens iMessage outbound sanitization and chunking.

## Best Possible Solution

Consolidate one durable completion-delivery fix that preserves user-safe result payloads, cleans up delivery claims on every exit path, keeps iMessage normalization at one boundary, aligns docs, and includes redacted runtime proof.

## Reproduction Assessment

Yes, for the PR-head defects at source level: the claim is persisted before an already-aborted return outside the cleanup finally block, and monitor delivery can send already-normalized chunks through another fence-closing normalization pass. The original intermittent production bugs were not live-reproduced here.

## Solution Assessment

No. The direction is plausible, but this is not the best mergeable solution until the claim cleanup bug, iMessage normalization boundary, docs mismatch, real behavior proof, and overlap with https://github.com/openclaw/openclaw/pull/76927 and https://github.com/openclaw/openclaw/pull/43992 are resolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Release delivery claims on early abort:** `src/agents/subagent-announce-dispatch.ts:200-204`
  - body: When `announceId` is set, `beginPersistedAnnounceDeliveryClaim` can persist a claim before this already-aborted return. Because this return is outside the `try`/`finally`, the claim is not deleted or finalized, so later retries can be blocked as `delivery-already-in-flight` until the lease expires.
  - confidence: 0.92
- **[P2] Avoid normalizing delivered iMessage chunks:** `extensions/imessage/src/send.ts:146`
  - body: `deliverReplies` normalizes the full reply before chunking, but this line normalizes each chunk again. A long fenced code block split across chunks can have an odd fence count per chunk, causing extra closing fences to be appended and corrupting delivered formatting.
  - confidence: 0.88
- **[P3] Align docs with queue-first completion delivery:** `src/agents/subagent-announce-dispatch.ts:213-214`
  - body: This change makes completion dispatch queue-first, but the branch's task delivery docs still describe direct delivery first with session-queued delivery only as fallback. Update the docs before landing so released behavior matches the documented delivery order.
  - confidence: 0.82
