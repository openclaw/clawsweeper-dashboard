## Summary

Keep open. Current main still has direct-first completion dispatch, no persisted delivery claim/user payload fields, and iMessage outbound sanitization does not strip the new internal-context envelopes; this PR is a relevant hardening attempt for the related delivery reports, but PR head still has two concrete P2 defects before it is mergeable.

## What This Changes

The PR adds persisted subagent delivery claim and user-facing payload state, changes completion dispatch toward queue-first delivery for active parents, and expands iMessage sanitization, chunking, send behavior, and focused tests.

## Best Possible Solution

Keep the source-level hardening direction, but first release or avoid persisted claims on all early exits, make iMessage fence closure happen at a single full-message boundary, rebase onto current main while preserving queue deferral behavior, and validate against the linked missing-payload reports before maintainer merge review.

## Reproduction Assessment

Yes for the PR-head defects: raw PR head shows a fresh claim can be persisted before an already-aborted return that bypasses cleanup, and iMessage text is normalized both before chunking and again per sent chunk. The original production leak/reconnect scenario was not live-reproduced in this read-only review.

## Solution Assessment

No. The queue-first and persisted-payload architecture looks like the right direction for the current-main gap, but the proposed patch is not the best mergeable solution until claim cleanup covers early aborts and code-fence normalization is owned by one full-message delivery boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Release the claim on early abort:** `src/agents/subagent-announce-dispatch.ts:200-204`
  - body: After `beginPersistedAnnounceDeliveryClaim` creates a persisted `claimed` state, this already-aborted return exits before the `finally` that clears or finalizes it. Later retries for the same announce can be rejected as `delivery-already-in-flight` until the lease expires.
  - confidence: 0.92
- **[P2] Skip per-chunk fence normalization in iMessage send:** `extensions/imessage/src/send.ts:146`
  - body: `deliverReplies` normalizes the full reply before chunking, but every chunk then goes through this normalization again. A long fenced block split across chunks can get an extra closing fence appended to an intermediate chunk, corrupting the delivered code formatting.
  - confidence: 0.86
