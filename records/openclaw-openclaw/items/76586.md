## Summary

Keep open. Current main has partial upgrade-recovery fixes, but source inspection still shows a live invalid-config loop for configured externalized plugins: update-time doctor can defer the install, post-core plugin sync still skips invalid configs, and Brave’s manual install path remains outside the narrow invalid-config recovery exception.

## What This Changes

The issue reports that upgrading from 2026.4.29 to 2026.5.2 can strand configs that reference newly externalized plugins because install, doctor repair, and downgrade recovery paths do not complete end to end.

## Best Possible Solution

Make the upgrade repair install or reconcile configured official external plugins before the invalid-config sync gate, then revalidate config while preserving fail-closed behavior for unrelated invalid configs.

## Reproduction Assessment

Yes, from source inspection. A config selecting `tools.web.search.provider: "brave"` without an active Brave plugin is invalid, update-mode doctor can defer the install, and the post-core plugin sync returns before repair when the config snapshot is invalid.

## Solution Assessment

No, current main is not yet the best solution. The safer fix is a narrow configured-official-plugin recovery path before the invalid-config skip, plus regression coverage for the package-update and manual recovery cases.
