## Summary

Keep open. Current main has several partial fixes for the 2026.5.2 externalized-plugin fallout, but the core upgrade recovery loop is still live: the package-update doctor pass defers configured plugin installs, the release repair step is skipped when the source config is invalid, post-core plugin sync skips invalid configs, and `plugins install` only grants invalid-config recovery to a narrow opt-in path that does not cover Brave.

## What This Changes

The issue reports that upgrading from 2026.4.29 to 2026.5.2 can strand configs that reference newly externalized plugins such as Brave, Discord, voice-call, and WhatsApp.

## Best Possible Solution

Make the upgrade and doctor repair path converge configured official external plugin installs before config-valid plugin sync gates, while keeping invalid-config bypasses narrowly limited to missing official plugin state.

## Reproduction Assessment

Yes, from source inspection rather than a live VPS run. A config selecting Brave web search without an active Brave plugin is invalid, update-time doctor defers the install, the release repair skips invalid source config, and post-core sync also skips invalid config.

## Solution Assessment

No, current main is not yet the best way to solve the issue. The maintainable fix is a narrow recovery path for configured official external plugins, not a general invalid-config bypass or a Brave package rename.
