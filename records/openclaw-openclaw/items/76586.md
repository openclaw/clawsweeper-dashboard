## Summary

Keep open. Current main has fixed some fallout, including the managed npm-root sibling-pruning bug, but the main upgrade recovery loop can still skip the install that would make the config valid again.

## What This Changes

The issue requests an end-to-end 2026.4.29 to 2026.5.2 upgrade recovery path for configs that reference newly externalized official plugins such as Brave, Discord, voice-call, and WhatsApp.

## Best Possible Solution

Make update recovery install configured official plugins before the invalid-config post-core skip, then rerun the normal plugin sync and keep the managed npm-root sibling preservation already on main.

## Reproduction Assessment

Yes, from source inspection. A config selecting Brave web search without an active Brave plugin is invalid, update-time doctor defers that install, and post-core plugin sync then skips because the config is invalid.

## Solution Assessment

No, current main is not the best solution yet. The narrow maintainable fix is to let the update recovery path install or repair configured official plugins before relying on config-valid plugin sync.
