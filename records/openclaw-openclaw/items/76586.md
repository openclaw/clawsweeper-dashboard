## Summary

Keep open. Current main still has a source-reproducible 2026.5.2 externalized-plugin upgrade loop: update-time doctor defers configured plugin installs, the later repair is gated behind a valid config, and manual install remains denied for the Brave invalid-config case.

## What This Changes

The issue asks OpenClaw to make upgrades from 2026.4.29 to 2026.5.2 recover configured externalized plugins without manual npm-global copying, invalid-config deadlocks, or downgrade traps.

## Best Possible Solution

Make the 2026.5.2 configured-plugin upgrade repair install or otherwise carry forward actively used official external plugins before any invalid-config gate can strand the gateway, with narrow regression tests for Brave web search and channel-selected plugins.

## Reproduction Assessment

Yes, from source inspection. A config selecting Brave web search without an active Brave plugin becomes invalid, update-time doctor defers the official plugin install, and both the later release repair and post-core plugin sync are skipped once the config is invalid.

## Solution Assessment

No, current main is not yet the best solution. The safer fix is a narrow recovery path for configured official external plugins during upgrade or install recovery, rather than relying on a doctor pass that cannot run after config validation fails.
