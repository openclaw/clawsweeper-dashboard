## Summary

Keep open. Current main has partial externalized-plugin repair work, but the source still leaves the reported upgrade loop reachable: update-time doctor can defer configured plugin installs, post-core plugin sync skips invalid configs, and `plugins install` still fails closed for the Brave invalid-config recovery case.

## What This Changes

The issue asks OpenClaw to make the 2026.4.29 to 2026.5.2 upgrade path recover automatically when config references newly externalized plugins such as Brave, Discord, and voice-call.

## Best Possible Solution

Repair configured official externalized plugin installs during the update/doctor recovery path before the config-valid plugin sync gate, while keeping arbitrary invalid-config plugin installs denied.

## Reproduction Assessment

Yes, from source inspection. A config selecting Brave web search without an active Brave plugin becomes invalid, update-mode doctor defers the configured install, and the post-core plugin sync then skips because the config is invalid.

## Solution Assessment

No, current main is not the best solution yet. The narrow maintainable fix is to complete official configured-plugin install repair before the invalid-config gate, not to trust arbitrary global npm installs or change plugin precedence.
