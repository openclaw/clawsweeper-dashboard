## Summary

Keep open. Current main has partial 5.2 externalized-plugin repair work, but source inspection still shows the reported upgrade recovery loop can remain unresolved when the config is invalid because update-mode doctor defers installs and post-core plugin sync skips invalid configs.

## What This Changes

The issue reports a 2026.4.29 to 2026.5.2 upgrade failure for configs using externalized official plugins, including Brave package naming, plugin install location, doctor repair reachability, and downgrade recovery problems.

## Best Possible Solution

Make the 5.2 externalized-plugin upgrade repair install or carry forward configured official plugin candidates before any config-valid plugin sync gate, and keep invalid-config install recovery narrowly scoped to the affected official plugin id.

## Reproduction Assessment

Yes, source-reproducible on current main. A config selecting tools.web.search.provider=brave without an active Brave plugin becomes invalid, update-mode doctor defers the missing install, and post-core plugin sync then skips because the config is invalid.

## Solution Assessment

No, current main is not yet the best solution. The narrow maintainable fix is to let the upgrade recovery path complete for configured official externalized plugins without broadening plugin install into a general invalid-config bypass.

## Repair Work Prompt

Fix openclaw/openclaw#76586 on current main. The observable bug is that upgrading from 2026.4.29 to 2026.5.2 with configured official externalized plugins such as brave/discord/voice-call can leave config invalid and recovery unreachable: update-mode doctor defers configured installs, post-core plugin sync skips invalid configs, and `plugins install` does not recover the Brave web-search-provider invalid-config case. Implement a narrow repair so configured official downloadable plugins needed by the existing config are installed or carried forward before the config-valid post-core sync gate blocks recovery. Keep the boundary narrow: do not rename npm packages, do not search arbitrary global npm prefixes as plugin sources, do not add a new config option, and do not turn `plugins install` into a broad invalid-config bypass. Add/update regression tests for update doctor deferral, release configured-plugin install repair, post-core invalid-config recovery, and the Brave provider-only config case. Add a user-facing changelog fix entry if runtime behavior changes. Related refs: #76586 and #76605, but #76605 should remain a separate diagnostic/shadowing issue unless the same narrow code path naturally improves it.
