## Summary

Keep open. Current main documents and partially repairs 5.2 externalized plugin fallout, but the central invalid-config upgrade loop is still source-reproducible: doctor skips the release repair when the source config is invalid, update-mode doctor defers configured plugin installs, and post-core plugin sync skips invalid configs.

## What This Changes

The issue asks for the 2026.4.29 to 2026.5.2 upgrade path to recover configurations that reference newly externalized plugins such as Brave, Discord, voice-call, and WhatsApp.

## Best Possible Solution

Make the one-time configured official plugin install repair run before or through the missing-plugin invalid-config gate, while keeping invalid-config bypass scoped to actively configured official external plugin ids.

## Reproduction Assessment

Yes, from source inspection rather than a live upgrade run. A config selecting Brave web search without an active Brave plugin becomes invalid, doctor skips release repair on invalid source config, update-mode doctor defers installation, and post-core plugin sync returns `invalid-config`.

## Solution Assessment

No, current main is only a partial solution. The narrow maintainable fix is to let doctor/update repair actively configured official external plugin installs before marking the release repair complete or skipping post-core work because config validation is blocked by that same missing plugin.
