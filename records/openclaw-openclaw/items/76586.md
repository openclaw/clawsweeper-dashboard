## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/76586 on current main. Observable bug: upgrading from 2026.4.29 to 2026.5.2+ with config-selected official externalized plugins such as brave web search, discord, voice-call, or whatsapp can leave the new binary with invalid config because update-time doctor defers installs and post-core plugin sync skips invalid config snapshots. Implement the narrowest recovery so configured official external plugins are installed or their missing payloads repaired under the managed ~/.openclaw/npm install path before service restart. Keep unofficial/third-party plugin installs fail-closed, do not rename @openclaw/brave-plugin, do not broaden general invalid-config install bypasses, and split the WhatsApp readWebAuthState runtime crash into a separate issue unless a test proves it is the same root cause. Add/update regression tests for package-update doctor, post-core invalid-config recovery, Brave provider-only config, and missing recorded install payloads. Add a user-facing changelog fix entry; update docs only if the recovery command or downgrade hint changes.
