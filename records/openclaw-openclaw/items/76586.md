## Summary

Keep open. Current main has partial 5.2 externalized-plugin recovery work, but source inspection still shows the reported invalid-config upgrade loop can survive the package update path.

## What This Changes

Fix the 2026.4.29 to 2026.5.2 upgrade path for configs that reference externalized plugins so official plugin installs recover without manual npm copying or downgrade surgery.

## Best Possible Solution

Make update recovery install or carry forward configured official external plugins before any config-valid plugin sync gate can skip, and keep manual install recovery limited to the requested official plugin id rather than broad invalid-config bypasses.

## Reproduction Assessment

Yes, from source inspection. A config selecting Brave search without an active Brave plugin is invalid, update-time doctor defers that install, and post-core plugin sync then skips because the config is invalid.

## Solution Assessment

No, current main is not yet the best solution. The narrow maintainable fix is to complete configured official plugin install recovery before marking the release repair complete or reaching invalid-config-gated sync/install paths.
