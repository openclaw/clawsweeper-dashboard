## Summary

Keep open. Current main has partial recovery work, but the source still shows a live 5.2 upgrade trap where configured external plugins can leave config invalid before doctor/install/update can complete repair.

## What This Changes

The issue asks OpenClaw to make the 2026.4.29 to 2026.5.2 upgrade path recover configured externalized plugins without manual npm copying or downgrade config edits.

## Best Possible Solution

Make update-time recovery install configured official external plugins before the invalid-config sync gate, then keep manual install recovery narrowly scoped to the requested official plugin id and document any remaining downgrade escape hatch.

## Reproduction Assessment

Yes, from source inspection. A config selecting Brave web search without an active Brave plugin becomes invalid, update-mode doctor defers the install, and the post-core plugin sync skips because the config is invalid.

## Solution Assessment

No, current main is not the best solution yet. The narrow maintainable fix is to complete the configured official plugin install repair before the invalid-config block, with regression coverage for package-update doctor and invalid-config install recovery.
