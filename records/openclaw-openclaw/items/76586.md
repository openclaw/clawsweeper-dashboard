## Summary

Keep open. Current main still has a live 5.2 externalized-plugin upgrade recovery gap: the missing official plugin makes config invalid, update-time doctor defers the install, and post-core plugin sync then skips because the config is invalid.

## What This Changes

The issue reports that upgrading from 2026.4.29 to 2026.5.2 can strand configs that reference newly externalized plugins such as Brave, Discord, voice-call, and WhatsApp.

## Best Possible Solution

Make the 2026.5.2 configured official plugin install repair complete before invalid-config gates can strand the updater, while keeping plugin installs explicit and scoped to known official configured plugins.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: configure tools.web.search.provider as brave with no active Brave plugin, run the package update doctor path with OPENCLAW_UPDATE_IN_PROGRESS=1, and the install is deferred while the later post-core sync skips invalid config. I did not run a live upgrade because this review is read-only.

## Solution Assessment

No, current main is not the best solution yet. The narrow maintainable fix is to repair configured official external plugin installs before the invalid-config plugin sync gate, not to add a broad config bypass or rename published packages.
