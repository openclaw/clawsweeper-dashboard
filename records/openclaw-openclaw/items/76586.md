## Summary

Keep open. Current main has partial recovery improvements, but the reported 5.2 upgrade path still has a source-reproducible loop: update-time doctor defers configured official plugin installs, post-core plugin update skips invalid configs, and manual `plugins install brave` is still not allowed through the invalid-config recovery gate.

## What This Changes

The issue asks OpenClaw to make upgrades from bundled plugins to externalized official plugins recover automatically, including Brave, Discord, and voice-call install state plus clearer downgrade guidance.

## Best Possible Solution

Make the official externalized-plugin repair complete before config validity blocks plugin sync, while keeping invalid-config bypasses scoped to configured official plugin ids and preserving the package security model.

## Reproduction Assessment

Yes, from source inspection. A config with `tools.web.search.provider: "brave"` and no active Brave install is invalid, the update doctor pass defers the missing install under `OPENCLAW_UPDATE_IN_PROGRESS`, and post-core plugin update skips because the config is invalid.

## Solution Assessment

No, current main is not the best way to solve the issue. The safer repair is a narrow upgrade recovery path that installs configured official external plugins before the invalid-config sync gate, instead of requiring a user to discover manual npm-root copying or a separate later doctor run.
