## Summary

Keep open. Current main still has the post-provider `extra_body` path that can overwrite DeepSeek thinking to disabled after `reasoning_content` has been added, and this PR is a focused fix; merge still needs contributor real behavior proof and maintainer scope review.

## What This Changes

The PR adds post-`extra_body` cleanup in the OpenAI-completions extra-params wrapper to delete `reasoning_content` from messages when the final payload has `thinking.type: "disabled"`, plus a DeepSeek V4 regression test.

## Best Possible Solution

Land this or an equivalent final-payload cleanup after proof, coordinated with https://github.com/openclaw/openclaw/pull/74418 and the broader precedence contract in https://github.com/openclaw/openclaw/issues/75040.

## Reproduction Assessment

Yes. Source inspection shows the provider wrapper can set enabled DeepSeek thinking and add `reasoning_content`, then the later `extra_body` wrapper can overwrite `thinking` to disabled without cleanup on current main; no live DeepSeek request was run in this read-only pass.

## Solution Assessment

Yes, with scope confirmation. Cleaning after the final `extra_body` merge is the narrow fix for this collision; the safer alternative is gating the same cleanup to DeepSeek/MiMo-compatible OpenAI-completions payloads if maintainers do not want generic behavior.
