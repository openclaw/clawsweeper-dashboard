## What This Changes

This PR adds post-`extra_body` cleanup to remove `reasoning_content` from OpenAI-completions messages when the final payload has `thinking.type: "disabled"`, plus a DeepSeek V4 regression test.

## Best Possible Solution

Land this or an equivalent final-payload cleanup with regression coverage and a one-line changelog entry, while preserving enabled-thinking replay behavior and leaving the broader `extra_body` collision discussion to the separate framework-level issue.

## Reproduction Assessment

Yes. The reproducible path is the payload mutation chain: DeepSeek V4 can add or preserve `reasoning_content`, then `extra_body` can make the final payload `thinking.type: "disabled"` on current main without cleanup.

## Solution Assessment

Yes, with a scope check. Cleaning the final payload after `extra_body` matches the actual request state; if maintainers want stricter owner boundaries, the safer alternative is the same cleanup gated to DeepSeek V4-compatible payloads.
