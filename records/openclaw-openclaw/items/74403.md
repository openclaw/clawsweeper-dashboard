## Summary

Keep open. Current main still lacks the final-payload cleanup, and this PR is the active narrow implementation for the DeepSeek disabled-thinking `extra_body` failure; it needs a changelog entry and maintainer review before merge.

## What This Changes

The PR adds post-`extra_body` cleanup that removes `reasoning_content` from OpenAI-completions messages when the final payload has `thinking.type: "disabled"`, plus a DeepSeek regression test.

## Best Possible Solution

Land this or equivalent final-payload cleanup with regression coverage and a one-line changelog entry, while leaving the broader `extra_body` collision policy to #75040 and the thinking-alias normalization to #74418.

## Reproduction Assessment

Yes, from source inspection rather than an executed test. Current main lets the DeepSeek wrapper add or preserve `reasoning_content` before post-plugin `extra_body` can set `thinking.type` to `disabled`, and there is no final cleanup in that wrapper.

## Solution Assessment

Yes, with the changelog addition. Cleaning the final payload after `extra_body` is the narrowest fix for the request as actually sent; if maintainers want stricter provider ownership, the safer variant is the same cleanup gated to DeepSeek-compatible payloads.
