## Summary

Keep open. Current main still applies `extra_body` after provider payload wrappers without stripping stale `reasoning_content`, and this PR is a narrow fix with regression coverage; the remaining merge cleanup is the required changelog entry.

## What This Changes

This PR adds post-`extra_body` cleanup to remove `reasoning_content` from OpenAI-completions messages when the final payload disables thinking, plus a DeepSeek regression test.

## Best Possible Solution

Land this or an equivalent final-payload cleanup with regression coverage and a one-line changelog entry, while keeping the broader thinking-alias and field-collision work in #74418 and #75040 separate.

## Reproduction Assessment

Yes. Source inspection on current main shows DeepSeek V4 can backfill `reasoning_content`, then post-plugin `extra_body` can overwrite the final payload to `thinking.type: "disabled"` without cleanup.

## Solution Assessment

Yes, with a small process fix. Cleaning the final OpenAI-completions payload after `extra_body` matches the actual outgoing request state; the PR still needs the required changelog entry before merge.
