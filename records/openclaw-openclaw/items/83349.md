## What This Changes

The issue reports that upgrade/doctor/plugin state can orphan an explicit Codex harness and allow live Telegram sessions configured for GPT-5.5 to silently fall back to Sonnet.

## Best Possible Solution

Land or supersede https://github.com/openclaw/openclaw/pull/83400 with a typed non-retryable missing-harness guard, then track any remaining doctor/plugin registration repair separately if it still reproduces.

## Reproduction Assessment

Yes, source-reproducible: current main throws a generic missing-harness error for explicit `codex` selection, and the outer model fallback loop continues unknown errors while candidates remain. I did not run the live Telegram path in this read-only review.

## Solution Assessment

Yes: making explicit missing-harness errors non-retryable by model fallback is the narrowest maintainable fix for the documented fail-closed contract. Auto-install, doctor repair, or session rebind behavior should not mask the runtime routing failure.
