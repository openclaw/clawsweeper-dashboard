## Summary

Keep open: current main still appears to allow the reported routing-integrity failure because the strict Codex harness selection error is thrown inside the model-fallback wrapper, which can continue to the next configured model candidate.

## What This Changes

The issue asks OpenClaw to prevent a missing configured Codex harness after upgrade or doctor from silently falling live Telegram sessions back from GPT-5.5 to another model.

## Best Possible Solution

Fail closed before model fallback for explicit missing harness selection/runtime errors, then add regression coverage for a forced Codex OpenAI model with configured Anthropic fallbacks.

## Reproduction Assessment

Yes, source-reproducible: direct strict harness selection throws when `codex` is missing, and the outer model fallback loop can continue on unrecognized errors when another candidate exists. I did not run the live Telegram path in this read-only review.

## Solution Assessment

Yes: making explicit missing-harness errors non-retryable by model fallback is the narrowest fix for the documented fail-closed contract. Auto-install or session rebind behavior can remain doctor/repair work rather than masking routing failure at runtime.
