## Summary

Keep open: current main still appears to allow the reported routing-integrity failure because a strict missing Codex harness error is thrown inside the model-fallback wrapper, which can continue to another configured model candidate.

## What This Changes

The issue reports that upgrade/doctor/plugin state can orphan the configured Codex harness and then mask the failure by falling back live Telegram sessions from GPT-5.5 to Sonnet.

## Best Possible Solution

Fail closed before model fallback for explicit missing harness selection/runtime errors, while leaving normal provider availability and auth/rate-limit fallback behavior intact.

## Reproduction Assessment

Yes, source-reproducible: a forced missing `codex` harness throws from `selectAgentHarness`, and `runWithModelFallback` can continue on unrecognized errors when another model candidate exists. I did not run the live Telegram path in this read-only review.

## Solution Assessment

Yes: making explicit missing-harness/runtime-selection errors non-retryable by model fallback is the narrowest maintainable fix for the documented fail-closed contract. Auto-install or session rebind behavior can remain separate doctor/repair work.
