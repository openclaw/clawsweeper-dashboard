## What This Changes

The issue asks OpenClaw to prevent missing explicit Codex harness state from silently falling back live Telegram or background sessions from GPT-5.5/Codex to Sonnet.

## Best Possible Solution

Land a narrow fail-closed guard for missing explicit agent harness selection before model fallback, with regression coverage, while leaving plugin reinstall and stale-session repair to separate doctor/runtime follow-up if needed.

## Reproduction Assessment

Yes. The source path is clear: current main throws an untyped missing-harness error from strict harness selection, and the outer model fallback loop can continue unrecognized errors when fallback candidates remain.

## Solution Assessment

Yes. A typed missing-harness error that model fallback treats as non-retryable is the narrowest maintainable fix for the documented fail-closed runtime contract; auto-install or session rebind behavior can be separate repair work.
