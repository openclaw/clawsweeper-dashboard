## What This Changes

The branch changes Slack routing so top-level public channel messages in `replyToMode="all"` use per-message thread session keys, adds a routing matrix test, updates the existing session-key test, and adds a changelog entry.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes. Source inspection on current main shows two top-level Slack room messages with `replyToMode="all"` both resolve to the shared channel session because `autoThreadId` is discarded by the `isRoomish` branch, while reply delivery targets each message timestamp.

## Solution Assessment

Is this the best way to solve the issue? Yes for the functional code: changing the room/group predicate is the narrowest route to let public rooms use `autoThreadId` in `all` mode while preserving DMs, group DMs, thread replies, and non-`all` modes. The changelog line should be narrowed before merge.
