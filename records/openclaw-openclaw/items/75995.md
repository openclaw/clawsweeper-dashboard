## What This Changes

The PR changes Slack routing so top-level public-channel messages in `replyToMode="all"` use per-message thread session keys, adds a routing matrix test, updates the existing session-key test, and adds a changelog entry.

## Best Possible Solution

Land the narrow Slack routing fix after targeted Slack tests and CI pass, letting the linked Slack bug reports close via the PR.

## Reproduction Assessment

Yes. By source inspection on current main, two top-level public Slack room messages with `replyToMode="all"`, `isRoom=true`, and `isRoomish=true` both fall through to the shared channel session because `canonicalThreadId` ignores `autoThreadId` for roomish channels.

## Solution Assessment

Yes. The PR is the narrowest maintainable fix I found: it lets public rooms use `autoThreadId` in `all` mode while preserving shared sessions for DMs, group DMs, and public rooms in `off`, `first`, and `batched` modes.
