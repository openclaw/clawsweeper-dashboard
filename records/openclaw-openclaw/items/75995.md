## Summary

Keep open, but the proposed patch is not merge-ready. The new tests assert that unseeded top-level Slack room messages with `replyToMode=all` should get per-message thread-scoped session keys, while current main intentionally keeps ordinary top-level room messages on the per-channel session and only thread-scopes actual replies or seeded mentions.

## What This Changes

The PR adds a new Slack Vitest file asserting session linkage behavior for roomish Slack channels when `replyToMode=all`.

## Best Possible Solution

Clarify whether the desired change is a real Slack session-routing behavior change; if not, replace this draft with tests that preserve current per-channel session routing while separately verifying Slack reply-thread delivery metadata.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes. Static inspection shows the proposed assertions conflict with current `resolveSlackRoutingContext` behavior and existing main-branch tests; I did not run Vitest because this review was required to keep the checkout read-only.

## Solution Assessment

Is this the best way to solve the issue? No. A test-only PR that asserts the old per-message thread-session behavior would fail against current main and risks undoing the deliberate per-channel room session contract.
