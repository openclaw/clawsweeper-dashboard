## Summary

Keep open: current main still routes unseeded top-level Slack room messages in `replyToMode="all"` to the shared channel session while reply delivery targets a per-message thread; the PR is a narrow source-aligned fix and I found no blocking review findings.

## What This Changes

The PR changes Slack routing so top-level public/private channel messages in `replyToMode="all"` use per-message thread session keys, adds a routing matrix test, updates the existing session-key test, and adds a changelog entry.

## Best Possible Solution

Land this PR, or an equivalent maintainer patch, after targeted Slack routing tests pass and the changelog wording is tightened to say Slack rooms/channels rather than all roomish chats.

## Solution Assessment
