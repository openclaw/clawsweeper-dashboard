## Summary

Keep open. This draft tests-only PR is tied to active Slack routing reports and intentionally asserts a new roomish `replyToMode="all"` session contract that current `main` does not implement, so it needs Slack owner/product review rather than cleanup closure.

## What This Changes

Adds Slack routing tests that expect `replyToMode="all"` room and fallback top-level messages to use distinct `:thread:<ts>` sessions while preserving shared sessions for other modes and chat types.

## Best Possible Solution
