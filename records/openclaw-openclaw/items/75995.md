## What This Changes

The PR changes Slack routing so top-level channel messages in `replyToMode="all"` use per-message thread session keys, adds a routing matrix test, updates an existing session-key expectation, and adds a changelog entry.

## Best Possible Solution

Land the narrow Slack routing fix after maintainer review; the linked Slack bug reports should close via this PR.

## Reproduction Assessment

Yes. By source inspection on current main, two top-level Slack room messages with `replyToMode="all"` both fall back to the shared channel session because `isRoomish` drops `autoThreadId`.

## Solution Assessment

Yes. The PR keeps DMs, group DMs, and non-`all` channel modes on their existing shared behavior while routing channel `all` mode through the timestamp-derived thread key.
