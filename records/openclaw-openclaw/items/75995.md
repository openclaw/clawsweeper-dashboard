## Summary

Keep open: current main still has the Slack public-channel `replyToMode="all"` session-key bug and this PR appears narrowly targeted, but it lacks required after-fix real behavior proof before merge.

## What This Changes

The PR changes Slack routing so top-level public channel messages in `replyToMode="all"` use per-message thread session keys, adds a routing matrix test, updates an existing session-key test, and adds a changelog entry.

## Best Possible Solution

Land the narrow Slack routing fix after the contributor adds real Slack/runtime proof and targeted Slack validation passes.

## Reproduction Assessment

Yes by source inspection. On current main, two top-level public Slack room messages with `replyToMode="all"`, `isRoom=true`, and `isRoomish=true` both lose `autoThreadId` and share the base channel session key.

## Solution Assessment

Yes, with a merge gate. The PR is the narrowest maintainable direction I found: allow public rooms to use `autoThreadId` in `all` mode while preserving shared sessions for DMs, group DMs, and non-`all` public-room modes.
