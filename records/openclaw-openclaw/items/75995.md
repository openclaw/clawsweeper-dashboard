## Summary

Keep open: this PR is a current implementation candidate for the linked Slack routing bugs. Current main still sends unseeded top-level Slack room messages in `replyToMode="all"` through the shared room session while reply delivery targets the message thread; the PR narrowly aligns the session key with that per-message thread target, and I found no blocking correctness or security issue in the diff.

## What This Changes

The PR changes Slack routing so top-level Slack room messages with `replyToMode="all"` use per-message `:thread:<ts>` session keys, adds resolver matrix tests, updates the existing thread-session-key test, and adds a changelog entry.

## Best Possible Solution

Land this PR, or an equivalent maintainer patch, after targeted Slack routing tests pass and the adjacent routing comment is aligned with the accepted `replyToMode="all"` contract.

## Solution Assessment

Yes. The PR keeps DMs and group DMs flat while allowing Slack rooms in `replyToMode="all"` to use `autoThreadId`, which is the narrowest code change for the linked session/thread mismatch.
