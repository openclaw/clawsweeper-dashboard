## Summary

Keep open. The PR is a focused current-helper implementation for a source-reproducible Slack routing bug, and I did not find a concrete functional or security regression in the proposed diff.

## What This Changes

The branch changes Slack routing so public-channel top-level messages with `replyToMode="all"` use per-message thread session keys, adds a routing matrix/regression coverage, updates the existing session-key contract, and adds a changelog entry.

## Best Possible Solution

Land this or an equivalent Slack-owned routing fix after CI, then close #64078 and #75969 and supersede the older overlapping #64080 as appropriate.

## Reproduction Assessment

Yes, source-level. On current main, `resolveSlackRoutingContext` for a top-level public Slack channel message with `replyToMode="all"`, no `thread_ts`, and no seeded mention resolves `SessionKey` to the shared channel key while `MessageThreadId` is the message timestamp.

## Solution Assessment

Yes. The PR is the narrowest maintainable fix I found: it lets public rooms in `all` mode keep `autoThreadId` while preserving DMs, group DMs, true thread replies, seeded room mentions, and `off`/`first`/`batched` behavior.
