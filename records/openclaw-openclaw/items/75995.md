## What This Changes

The branch changes Slack routing so public-channel top-level messages in `replyToMode="all"` use per-message thread session keys, adds Slack routing/session tests, and updates the changelog.

## Best Possible Solution

Slack room/channel routing should use one conversation thread id for both inbound session routing and reply delivery under `replyToMode="all"`, while preserving current DM, MPIM, `off`, `first`, `batched`, true-thread-reply, and seeded-mention behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows a top-level Slack room message with `replyToMode="all"`, no `thread_ts`, and no seeded thread can keep the base channel `SessionKey` while `MessageThreadId` is the message timestamp.

## Solution Assessment

Yes, subject to real behavior proof. The PR targets the narrow Slack-owned routing boundary by allowing public rooms to use `autoThreadId` in `all` mode while preserving shared sessions for DMs, group DMs, and non-`all` public-room modes.
