## Summary

Keep open. Current main still has the Slack `replyToMode="all"` public-channel session mismatch, and this PR is a focused fix with no blocking code findings, but the external contributor still needs after-fix real Slack/OpenClaw proof before merge.

## What This Changes

The PR changes Slack routing so public-channel top-level messages in `replyToMode="all"` use per-message thread session keys, adds Slack routing/session tests, and updates the changelog.

## Best Possible Solution

Land one Slack routing fix after redacted real Slack/OpenClaw proof shows distinct per-message sessions in a public channel and maintainers choose between this PR and https://github.com/openclaw/openclaw/pull/64080.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the bug: a top-level public Slack room message with `replyToMode="all"`, no `thread_ts`, and no seeded root computes `autoThreadId` but resolves `SessionKey` to the base channel while `MessageThreadId` remains the message timestamp.

## Solution Assessment

Yes, subject to real behavior proof. The PR is a narrow maintainable fix because it lets public rooms use `autoThreadId` only for `all` mode while preserving shared sessions for DMs, group DMs, and non-`all` public-room modes.
