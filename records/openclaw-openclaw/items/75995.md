## Summary

Keep this draft PR open: it raises a real Slack routing contract question, but the test-only diff is not merge-ready because current main deliberately keeps ordinary top-level room messages on the per-channel session.

## What This Changes

The PR adds a Slack Vitest file with a routing matrix for roomish, group-DM, and DM session keys under several `replyToMode` values.

## Best Possible Solution

Decide the Slack contract for ordinary room `replyToMode="all"`, then either implement a narrow `prepare-routing.ts` change with aligned tests and changelog or keep the existing per-channel behavior and adjust/close the draft.

## Reproduction Assessment

Yes. Static inspection is enough: current main drops `autoThreadId` for unseeded `isRoomish` top-level messages, so the PR's distinct-session public-channel row and unseeded root/reply equality assertion would fail against current behavior.

## Solution Assessment

No. A tests-only draft is not the best final fix because its assertions reverse an explicit current-main contract without a production change or maintainer decision.
