## What This Changes

The PR changes Slack routing so public channel top-level messages in `replyToMode="all"` use per-message thread session keys, adds a routing matrix test, updates the existing session-key expectation, and adds a changelog entry.

## Best Possible Solution

## Reproduction Assessment

Yes. By source inspection on current main, two top-level public Slack room messages with `replyToMode="all"`, `isRoom=true`, and `isRoomish=true` both route without a thread id and therefore share the channel session.

## Solution Assessment

Yes. The PR is the narrowest maintainable code direction I found: allow public rooms to use `autoThreadId` in `all` mode while preserving shared sessions for DMs, group DMs, and non-`all` public-room modes.
