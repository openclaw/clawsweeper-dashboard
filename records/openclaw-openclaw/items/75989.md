## Summary

Keep open. This PR is related to the blank Control UI regression, but the patch is incomplete: it guards some `home-dir` and `resolveHomeDir` process access while the browser-facing slash-command path still imports Node/plugin-loader code and `src/utils.ts`; it also lacks the required changelog entry.

## What This Changes

The PR adds browser-safe home-directory accessors, guards two direct `process.env` reads in `src/utils.ts`, and adds home-dir unit tests for empty-env fallback behavior.

## Best Possible Solution

Move the Control UI slash-command metadata and `/think` choices onto a browser-safe shared leaf, keep server/provider thinking policy on the Gateway side, add a browser-import regression test, and document the user-facing fix in the changelog.

## Reproduction Assessment

Yes. The related issue provides a concrete browser stack, and current main has a source-level path from the Control UI slash-command module to Node-only `src/utils.ts` and `home-dir` side effects; the PR tests do not reproduce the process-less browser import.

## Solution Assessment

No. Guarding `home-dir` defaults is not the narrowest maintainable fix because the browser bundle should not import provider plugin loaders, `src/utils.ts`, or home-dir in the first place.
