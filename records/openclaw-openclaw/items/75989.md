## Summary

Keep open. The PR guards one `process` symptom, but the current Control UI slash-command import graph still pulls Node/plugin-loader code into the browser-facing bundle, and the branch lacks the required changelog entry for a user-facing blank-screen regression.

## What This Changes

The PR adds browser-safe home-directory/process guards in `src/infra/home-dir.ts` and `src/utils.ts`, plus empty-env home-dir regression tests.

## Best Possible Solution

Move Control UI slash-command metadata and static `/think` choices onto a browser-safe shared leaf, keep provider-specific thinking policy on the Gateway side, add a process-less browser import regression test, and add a one-line Unreleased Fixes changelog entry.

## Reproduction Assessment

Yes. The linked issue provides a browser stack, and current main has a source-level reproduction path where importing the UI slash-command module initializes fallback commands through Node-only shared registry and plugin policy modules.

## Solution Assessment

No. Guarding `home-dir` defaults treats the reported crash as a symptom; the narrower maintainable fix is to stop the browser bundle from importing `src/utils.ts`, provider policy loading, and bundled-plugin resolution at all.
