## Label Justifications

- `impact:session-state`: The reported failure invalidates stored CLI session bindings and replaces the live session handle.
- `impact:auth-provider`: The trigger is auth epoch calculation across local CLI credentials and OpenClaw auth profiles.

## Summary

Keep open: current main and the latest release still combine the local CLI credential fingerprint with the selected auth-profile credential for the Claude CLI path, so a local credential source-presence flip can still invalidate live sessions. The linked fix PR was closed unmerged, so no merged implementation currently owns the remaining bug.

## What This Changes

The issue asks OpenClaw to keep CLI session auth epochs stable when local Claude CLI credential storage appears or disappears without an identity change.

## Best Possible Solution

Decide the auth-epoch ownership policy, then land a focused core fix plus regression coverage so identity-unchanged local credential source flips do not reset live CLI sessions while real auth changes still do.

## Reproduction Assessment

Yes. The current source still has a clear source-repro path: local credential missing versus present changes the auth-epoch parts while the selected identity-less OAuth profile remains unchanged, and the session layer invalidates on the resulting epoch mismatch.

## Solution Assessment

Unclear until maintainers choose the auth-epoch policy. The proposed direction is narrow and plausible, but the final fix should explicitly preserve invalidation for genuine auth changes and cover the Claude CLI identity-less profile case in tests.
