## Summary

Keep open: current main and v2026.6.10 still compose Claude CLI auth epochs from the local credential part when present plus the selected auth-profile part, so a same-identity local credential source-presence flip can still move the epoch and invalidate live CLI sessions.

## What This Changes

The issue asks for Claude CLI session auth epochs to stay stable when local credential storage presence changes but the OAuth identity has not changed.

## Best Possible Solution

Land or sponsor a focused Claude CLI auth/session policy fix that keeps unchanged identity from resetting live sessions across local source-presence flips while preserving resets for logout, real account changes, and unprovable identity changes.

## Reproduction Assessment

Yes. Current-main source still changes the auth-epoch part set when a Claude CLI local credential read flips between absent and present while an auth profile is selected, and the session reuse code invalidates same-version epoch mismatches.

## Solution Assessment

Unclear. The requested direction is plausible and narrow, but maintainers still need to choose the Claude CLI identity boundary so unchanged storage presence does not reset sessions while real or unprovable auth changes still do.
