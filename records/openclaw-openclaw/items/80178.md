## Summary

Keep open: current main and the latest release still combine local CLI credential presence with auth-profile state, and an open closing PR already owns the proposed fix path.

## What This Changes

The issue reports that CLI auth-epoch hashing can reset live reusable Claude CLI sessions when the local credential file appears or disappears even though the OAuth identity is unchanged.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/80265; the end state should stabilize auth epochs across local storage-presence flips while preserving invalidation for real auth identity changes.

## Reproduction Assessment

Yes. Source inspection plus the reporter's deterministic mock hashes show that changing the Claude local credential reader from null to an identity-less OAuth credential while the profile stays unchanged changes the resolver's part list, and cli-session maps that mismatch to auth-epoch invalidation.

## Solution Assessment

Unclear for the issue's broad resolver-level snippet, because it can bypass the combined versus profile-owned policy seam. The active PR's narrower prepare/auth-epoch seam approach is the better direction if live proof confirms the runtime behavior.
