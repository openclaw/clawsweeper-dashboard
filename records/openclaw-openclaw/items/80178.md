## Summary

Keep this open. Current main still allows the reported storage-presence flip to change the CLI auth epoch, and the issue is already paired with the open closing PR at https://github.com/openclaw/openclaw/pull/80265, so the right cleanup path is to review and land or reject that PR rather than close this issue now.

## What This Changes

Stabilize live CLI session reuse when a Claude local credential file appears or disappears but the selected OAuth auth-profile identity is unchanged.

## Best Possible Solution

Review and land, revise, or replace https://github.com/openclaw/openclaw/pull/80265 with redacted live Claude CLI proof and focused regression coverage, keeping session reuse stable across storage-presence flips while preserving real auth-change invalidation.

## Reproduction Assessment

Yes, with high confidence at source level. Mock the Claude local credential reader from null to an identity-less OAuth credential while the auth-profile credential stays unchanged; current main hashes a different part list and `resolveCliSessionReuse` maps the mismatch to `invalidatedReason: "auth-epoch"`.

## Solution Assessment

Unclear for the original broad resolver-level proposal because it can bypass the documented combined-mode auth ownership contract. The better path is the narrower prepare/auth-epoch policy-seam fix represented by the open closing PR, with real runtime proof before merge.
