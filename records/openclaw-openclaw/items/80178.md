## Summary

Keep this open: current main still has the reported auth-epoch instability, and an open closing PR is already the active implementation path. The issue should close only after https://github.com/openclaw/openclaw/pull/80265 or an equivalent fix lands.

## What This Changes

The issue requests that CLI auth-epoch computation stop invalidating live CLI sessions when only local credential file presence changes while the auth profile identity is unchanged.

## Best Possible Solution

Review, rebase if needed, and land https://github.com/openclaw/openclaw/pull/80265 or a maintainer-approved equivalent that stabilizes identity-less Claude CLI epochs across file-presence flips while preserving invalidation for real identity changes.

## Reproduction Assessment

Yes. Source inspection shows current main still hashes local and profile epoch parts together, and the issue supplies deterministic hashes plus live `auth-epoch` reset logs for the same path.

## Solution Assessment

Yes in direction: the maintainable fix should use the prepare/backend auth-epoch policy seam, with regression coverage and live Claude CLI proof. A resolver-wide suppression like the closed alternate PR is riskier because it bypasses combined-mode ownership.
