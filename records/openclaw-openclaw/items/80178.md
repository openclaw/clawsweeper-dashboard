## Summary

This should stay open: current main and the latest release still combine local CLI credential presence with the auth-profile epoch, and the issue is already paired with an open fixing PR that uses closing syntax.

## What This Changes

The issue requests stable CLI auth epochs when the Claude CLI local credential file appears or disappears while the auth-profile OAuth identity is unchanged.

## Best Possible Solution

Review and land the linked policy-seam fix once redacted live Claude CLI proof shows session reuse stays stable across the file-presence flip while real identity changes still invalidate.

## Reproduction Assessment

Yes. Source inspection shows current main hashes different part lists when the local Claude credential reader changes from null to a credential while the auth profile is unchanged, and the session reuse path invalidates on that epoch mismatch.

## Solution Assessment

Yes for keeping this issue open until the fixing PR lands. The narrow maintainable fix should use the prepare/auth-epoch ownership seam, not a broad resolver-level suppression that hides all local credential changes.
