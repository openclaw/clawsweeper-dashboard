## Summary

Keep this issue open: current main still combines local CLI credential fingerprints with auth-profile credentials, and the linked closing PR is open rather than merged.

## What This Changes

The issue reports that CLI auth epochs change when the local Claude credential file appears or disappears even though the selected auth-profile identity is unchanged.

## Best Possible Solution

Review and land the narrower policy-seam fix in https://github.com/openclaw/openclaw/pull/80265 after proof, or close that PR with an alternate maintainer-approved auth-boundary fix.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: mock the Claude local credential reader from null to an identity-less OAuth credential while the auth-profile credential stays unchanged, and current main changes the part list that is hashed; the session reuse path then maps that mismatch to "auth-epoch".

## Solution Assessment

Partly. The broad resolver-level approach from the closed related PR is risky, while the open PR's narrower prepare/auth-epoch policy seam is the better direction but should not close this issue until it is reviewed and merged.
