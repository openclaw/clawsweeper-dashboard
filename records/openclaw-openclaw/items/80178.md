## Summary

This should stay open: current main still has the reported source-level auth-epoch instability, and two open PRs already reference this issue with closing syntax but have not merged.

## What This Changes

The issue asks OpenClaw to keep reusable Claude CLI sessions valid when only the local credential file source appears or disappears while the auth-profile identity is unchanged.

## Best Possible Solution

Review and land one canonical fix PR that stabilizes the auth epoch for unchanged Claude CLI identity while preserving real reauth invalidation and focused regression coverage.

## Reproduction Assessment

Yes. Mocking the Claude local credential reader from `null` to an OAuth credential while keeping the auth-profile credential unchanged exercises the current resolver path that changes the epoch and then invalidates session reuse as `auth-epoch`.

## Solution Assessment

Unclear which open PR shape is best. The maintainable fix should use the existing auth-epoch policy seam or a narrow resolver change, with tests for storage-presence flips, bootstrap behavior, and real identity changes.
