## Summary

This should stay open: current main still lets Claude local credential presence change the auth-epoch shape, which can invalidate reusable CLI sessions even when the auth-profile identity is unchanged.

## What This Changes

The issue requests stable CLI auth epochs when Claude local credential storage appears or disappears without an identity change.

## Best Possible Solution

Stabilize Claude CLI auth epochs across local credential storage-presence flips when the selected auth-profile identity is unchanged, while preserving invalidation for real auth-state and account changes with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection shows the current resolver hashes different part lists when the Claude local credential read changes from null to a value while the profile credential is unchanged, and the session reuse path maps that same-version epoch drift to `auth-epoch`.

## Solution Assessment

Unclear for the exact proposed resolver change. The safest fix should use the existing profile-owned seam or a narrowly scoped Claude auth-epoch rule, with tests for storage-presence stability and true identity-change invalidation.
