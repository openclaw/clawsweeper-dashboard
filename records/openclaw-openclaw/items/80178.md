## Summary

This should stay open: current main still has the source-reproducible auth-epoch reset path, and the open closing PR is the right place to review the fix before the issue is closed.

## What This Changes

Prevent local Claude CLI credential file presence changes from changing the CLI auth epoch and resetting live sessions when the auth-profile identity is unchanged.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80265, or an equivalent policy-seam fix, with regression coverage and redacted live Claude CLI proof that file presence flips no longer cause `auth-epoch` resets while real identity changes still invalidate.

## Reproduction Assessment

Yes, source-reproducible with high confidence: mock the Claude local credential reader from null to an identity-less OAuth credential while the auth-profile credential stays unchanged, and current main hashes a different part list. The session reuse path then maps the epoch mismatch to `invalidatedReason: "auth-epoch"`.

## Solution Assessment

Yes for keeping this issue open until the linked fix PR lands; current main is not the desired solution. The safer implementation path is the prepare/auth-epoch ownership seam used by https://github.com/openclaw/openclaw/pull/80265, not the broader resolver-level bypass that was closed in https://github.com/openclaw/openclaw/pull/80425.
