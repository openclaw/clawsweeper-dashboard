## Summary

Keep open: current main and the latest shipped release still combine local CLI credential fingerprints with auth-profile epochs, and the open closing PR https://github.com/openclaw/openclaw/pull/80265 is the active fix path rather than a reason to close before merge.

## What This Changes

Stabilize CLI auth epochs so local Claude CLI credential-file presence changes do not invalidate live sessions when the OAuth identity/profile is unchanged.

## Best Possible Solution

Review and land the linked PR or an equivalent policy-seam fix that skips local fingerprints for identity-less OAuth/profile cases, bumps the epoch version if needed, and proves real Claude CLI session reuse across the file-presence flip.

## Reproduction Assessment

Yes. Source inspection shows the current resolver can produce different part lists when the local Claude credential reader flips between null and a credential while the profile credential stays the same, and session reuse maps the resulting epoch mismatch to auth-epoch invalidation.

## Solution Assessment

Yes at the direction level: the maintainable fix should use the prepare/backend auth-epoch policy seam for identity-less OAuth profiles while preserving identity-bearing invalidation. The open PR is the concrete candidate, but it should be reviewed and proven before merge.
