## Summary

This should stay open: current main still combines the local CLI credential fingerprint with the auth-profile credential, so the reported storage-presence flip can still change the auth epoch, and the direct closing PR is open but unmerged.

## What This Changes

Prevent local Claude CLI credential file-presence changes from invalidating live CLI sessions when the stored auth-profile identity is unchanged.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/80265 or a safer replacement lands, preserving the combined/profile-only auth-epoch contract while adding regression coverage and redacted real-session proof.

## Reproduction Assessment

Yes. ## Solution Assessment

No for closing this issue now. The direct linked PR is still open and unmerged, and the maintainable fix should preserve the documented combined/profile-only auth-epoch boundary rather than broadly suppressing host credential invalidation.
