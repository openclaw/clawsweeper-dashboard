## What This Changes

The issue reports that CLI session auth epochs change when the local Claude credential file appears or disappears even though the selected auth-profile identity is unchanged.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80265, or a successor with equivalent regression coverage and redacted live Claude CLI proof, then close this issue as implemented.

## Reproduction Assessment

Yes. Mock the Claude local credential reader from null to an OAuth credential while the auth-profile store stays unchanged; current main hashes a different part list, and resolveCliSessionReuse maps the changed epoch to invalidatedReason: "auth-epoch".

## Solution Assessment

Yes for the current direction: the linked PR uses the prepare-time skip policy seam and keeps identity-bearing profiles in combined mode. The issue should remain open until that fix or an equivalent successor is merged.
