## Summary

Keep open: current main still has a source-level path that can turn a missing persisted catalog `reasoning` flag into `reasoning: false`, and the `/think` gate then returns an off-only profile before the OpenAI/OpenAI Codex provider-owned `xhigh` policy can run. The reporter's follow-up usefully narrows this from pure effective-routing mismatch to catalog/profile precedence, so this should be fixed rather than closed as handled by the earlier auth-order PR.

## What This Changes

The issue requests that `/think xhigh` work for supported OpenAI/OpenAI Codex GPT-5 models even when canonical routing or catalog hydration is involved.

## Best Possible Solution

Make provider-owned thinking profiles authoritative for OpenAI/OpenAI Codex when catalog reasoning is absent or stale, while keeping explicit configured `reasoning: false` as a real opt-out for providers such as Google/Gemma.

## Reproduction Assessment

Yes, at source level. A selected OpenAI/OpenAI Codex GPT-5 row with missing persisted reasoning normalized to `false` flows through `resolveThinkingCatalog` into `resolveThinkingProfile`, which returns only `off` before the provider policy can expose `xhigh`; the reporter supplied matching live `/think` output.

## Solution Assessment

Yes. The maintainable fix is a narrow catalog/profile precedence repair with regression tests, not a new flag or routing policy change; likely candidates are preserving `undefined` for missing persisted reasoning and/or only treating explicitly configured `reasoning: false` as an off-only override.
