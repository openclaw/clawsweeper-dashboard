## Summary

Keep open. Current main has adjacent Codex OAuth refresh hardening and Codex `server_error` failover handling, but it still does not prove or implement the reported post-refresh recovery path for the same selected `openai-codex/gpt-5.4` route.

## What This Changes

The issue asks for `openai-codex/gpt-5.4` to keep working after OAuth refresh without manually switching to `gpt-5.3-codex` and back.

## Best Possible Solution

Collect live Codex OAuth evidence on a current release, then implement a Codex-owned credential rebind and one-time same-selected-model retry only where replay-safe, with focused regression coverage.
