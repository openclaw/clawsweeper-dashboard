Yes. Source inspection on current main shows invalid gateway-rpc timeout input flows through `Number(opts.timeout ?? 10_000)` in `src/cli/gateway-rpc.runtime.ts`, then `callGateway` treats non-finite values as unset and falls back instead of failing at the CLI boundary.

## Solution Assessment

No. Strict validation is the right direction, but this branch must update the active runtime path and preserve the channels probe fallback before it is the narrow maintainable merge shape.
