## Summary

Keep open: the patch is a plausible narrow fix for the forced-refresh path, but this external PR still needs after-fix real behavior proof before merge; I also found one non-blocking test fidelity issue around the OAuth refresh margin.

## What This Changes

The PR removes the Codex app-server auth-refresh forced OAuth rotation, updates focused auth-bridge tests, and adds a changelog entry.

## Best Possible Solution

Land the narrow auth-bridge change with a production-faithful regression test and redacted patched-run proof that a current Codex OAuth token is reused while expired or near-expiring credentials still refresh through the owner store.

## Reproduction Assessment

Partially: source inspection gives a high-confidence current-main path where app-server refresh requests force OAuth rotation by writing `expires: 0`. A full live reproduction of the OpenAI `refresh_token_reused` response still depends on a real Codex OAuth account/session and was not established in this read-only review.

## Solution Assessment

Yes, the implementation direction is the narrowest maintainable fix because it removes the app-server-specific forced expiry and leaves the existing auth resolver to decide reuse versus refresh. The test fixture should be aligned with the core five-minute refresh margin, and merge should wait for real after-fix behavior proof.

- **[P3] Use a token outside the refresh margin:** `extensions/codex/src/app-server/auth-bridge.test.ts:1050`

Use an expiry comfortably beyond the margin or make the mock honor the shared margin.
