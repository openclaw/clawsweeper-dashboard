## Summary

Keep open: the PR targets a real opencode-go liveness bug, but the implementation still counts boundary events before any provider delta and can shorten the first-event window that current main deliberately preserves; contributor real behavior proof is also missing.

## What This Changes

The PR adds text_end, thinking_end, toolcall_start, and toolcall_end to the opencode-go stalled-stream liveness predicate and adds a regression test for boundary-event idle resets.

## Best Possible Solution

Gate boundary-event liveness on a post-delta state, preserve the longer first-event timer, add pre-delta regression coverage, and include redacted after-fix wrapper or OpenClaw proof.

## Reproduction Assessment

Yes, source-reproducible. Current main forwards boundary events but only re-arms idle liveness on delta events, and the canonical issue includes a focused failing timing sequence; I did not run tests in this read-only review.

## Solution Assessment

No. The fix belongs in the opencode-go provider wrapper, but this patch applies boundary liveness statelessly; it should track that a real provider delta has occurred before using boundaries to refresh idle.
