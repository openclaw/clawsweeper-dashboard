## Summary

Keep open for maintainer/contributor action: this draft external PR targets a plugin metadata cache surface that current main has removed, conflicts with the scoped plugin cache-boundary guidance, and still lacks real gateway behavior proof.

## What This Changes

The PR changes old plugin discovery and manifest registry caches to use 30-second default TTLs and adds regression tests asserting cache hits after 1.5 seconds.

## Best Possible Solution

Fix the linked gateway polling cost on current main by carrying a flow-owned plugin metadata snapshot or prepared registry through auth/provider checks, without reintroducing persistent discovery or manifest registry caches.

## Reproduction Assessment

No high-confidence live reproduction on current main was established. The linked issue provides useful v2026.5.7 strace/latency evidence, and source inspection shows this PR targets cache APIs that no longer exist on current main.

## Solution Assessment

No. Extending the old cache TTL is not the best fix because current main intentionally removed persistent metadata caches and the scoped plugin guide directs this surface toward explicit flow-owned snapshots or registries.
