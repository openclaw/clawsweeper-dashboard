## Summary

Keep open: this draft external PR is not mergeable as written because it adds default persistent plugin metadata TTL caches in a control-plane path that current policy and tests keep fresh by default. The linked latency issue remains valid, but this patch needs real gateway proof and a maintainer-guided redesign around flow-owned snapshots or narrower auth/status coalescing.

## What This Changes

The branch adds 30-second in-process caches and environment controls around plugin discovery and manifest registry loading, plus tests that expect cache hits across short gateway poll intervals.

## Best Possible Solution

Fix the gateway polling cost by reusing a gateway-owned PluginMetadataSnapshot, PluginLookUpTable, or explicit manifest registry through the auth/provider flow, or by narrowly coalescing unchanged auth/status no-ops while preserving fresh default plugin metadata calls.

## Reproduction Assessment

No high-confidence live reproduction of the gateway latency was run during this review. The stale-cache regression introduced by this PR is source-reproducible from the diff because the new tests intentionally keep removed or edited plugin metadata visible inside the default 30-second cache window.

## Solution Assessment

No. A module-level 30-second metadata cache is not the best fix because current plugin policy keeps control-plane metadata fresh unless the caller owns an explicit snapshot or registry for the current flow.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Drop the persistent discovery cache:** `src/plugins/discovery.ts:78-82`
  - body: This default 30-second cache means plugin add/remove events and path-safety changes can be hidden until the TTL expires. Current plugin guidance keeps discovery metadata fresh unless a caller owns an explicit snapshot or lookup table, so the gateway performance fix should reuse a flow-owned metadata object instead of caching discoverOpenClawPlugins globally.
  - confidence: 0.89
- **[P2] Keep manifest registries fresh by default:** `src/plugins/manifest-registry.ts:183-187`
  - body: Caching loadPluginManifestRegistry for 30 seconds returns stale manifest records after openclaw.plugin.json or package metadata changes, contradicting the current next-load freshness regression and delaying manifest safety diagnostics. Pass a flow-owned registry or snapshot through the gateway path instead of adding a default module-level TTL.
  - confidence: 0.87
