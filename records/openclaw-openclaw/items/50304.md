## Summary

Keep open. The PR targets a real linked Qianfan replay bug, but the branch drops Qianfan's OpenAI-compatible replay defaults and does not strip the immediate tool-call turn it tests; the supplied proof is static terminal syntax/search output rather than after-fix runtime behavior.

## What This Changes

Adds a Qianfan provider replay-policy hook for `qianfan-code` model IDs and regression tests for provider registration plus session-history sanitization.

## Best Possible Solution

Keep the Qianfan-owned replay hook boundary, compose it with OpenAI-compatible replay defaults, handle the actual current tool-call reasoning shape, and require redacted runtime proof before merge.

## Reproduction Assessment

No live Qianfan Code reproduction was established. Source inspection is high-confidence for review: the PR hook skips existing OpenAI-compatible fallback defaults, and the current `dropThinkingBlocks` helper preserves the latest assistant tool-call turn.

## Solution Assessment

No. Provider-owned Qianfan replay policy is the right boundary, but this implementation should preserve OpenAI-compatible defaults and address the current tool-call replay shape before merge.
