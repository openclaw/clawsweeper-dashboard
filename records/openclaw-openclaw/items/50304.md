## Summary

Keep open. The branch targets the right provider-owned replay seam, but the proposed hook would replace Qianfan's existing OpenAI-compatible fallback with an incomplete policy, and the supplied proof only demonstrates static code shape rather than a replayed tool-call behavior run.

## What This Changes

The PR adds a Qianfan provider replay-policy hook for model IDs containing `qianfan-code` and adds provider plus sanitize-session-history coverage for Qianfan Code replay.

## Best Possible Solution

Keep Qianfan on a complete OpenAI-compatible replay policy and layer a narrow Qianfan Code rule with focused regression tests plus real replay proof.

## Reproduction Assessment

No high-confidence live reproduction was established here. The linked issue gives a concrete Qianfan Code tool-call error, and source inspection shows the replay-policy path clearly enough to review the proposed fix.

## Solution Assessment

No. A provider-owned hook is the right location, but this patch should compose with the OpenAI-compatible replay policy instead of replacing the fallback with an undefined or minimal policy.
