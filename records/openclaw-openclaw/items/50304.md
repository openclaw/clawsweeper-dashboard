## Summary

Keep open: the linked bug is plausible, but this patch regresses Qianfan's existing OpenAI-compatible replay behavior by replacing the transport-family policy with a minimal provider hook, and the supplied proof is static-only.

## What This Changes

The PR adds a Qianfan provider replay-policy hook that drops thinking blocks for model IDs containing `qianfan-code` and adds provider plus sanitize-session-history coverage.

## Best Possible Solution

Keep a Qianfan-owned replay hook, but compose it with the OpenAI-compatible replay family and layer the Qianfan Code thinking-block rule on top with focused regression coverage and real replay proof.

## Reproduction Assessment

No high-confidence live reproduction was established. The linked issue gives a concrete Qianfan Code tool-call error, and source inspection shows the replay-policy path clearly enough to review the proposed fix.

## Solution Assessment

No. A provider-owned hook is the right location, but this patch should compose with the OpenAI-compatible replay policy instead of replacing it with `undefined` or a minimal `{ dropThinkingBlocks: true }` policy.
