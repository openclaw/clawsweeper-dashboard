## What This Changes

The issue reports that Active Memory's embedded recall runner cannot expose the allowlisted `memory_recall` tool when the hidden runner uses `deepseek/deepseek-v4-flash`.

## Best Possible Solution

Add targeted proof for Active Memory with `plugins.slots.memory: "memory-lancedb"`, `toolsAllow: ["memory_recall"]`, and `deepseek/deepseek-v4-flash`; then fix the narrow embedded/provider tool bridge or diagnostics path without broadening the hidden recall sub-agent's tool surface.
