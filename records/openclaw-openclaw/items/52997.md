## Summary

Keep open. Current main and v2026.5.19 still intentionally isolate Telegram forum topics and configured DM threads, while the related shared-session trackers do not replace this issue's Telegram-specific metadata and reply-routing contract.

## What This Changes

Request an opt-in Telegram configuration mode that routes forum topics and DM threads into one shared agent session while preserving topic/thread metadata and origin reply routing.

## Best Possible Solution

Design an explicit opt-in shared-session contract aligned with the broader shared-session work, preserving default isolation while carrying Telegram topic/thread metadata and origin reply routing through schema, docs, and focused routing tests.

## Reproduction Assessment

Yes, at source level. Current source, tests, docs, and the v2026.5.19 tag show forum topics become :topic: sessions, configured DM threads can become scoped thread sessions, and no Telegram shared-session config exists.

## Solution Assessment

Unclear as a final implementation. The requested sessionMode is plausible, but the maintainable path should align with the broader shared-session contract and explicitly define privacy, sandbox, metadata, and reply-routing behavior.
