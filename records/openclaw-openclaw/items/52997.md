## Summary

Keep open. Current main still isolates Telegram forum topics with topic-qualified session keys and exposes no Telegram shared-session config surface; the DM half is partly addressed by flat default DM handling, but configured DM topics/thread replies still isolate. This remains a real feature/config request that needs maintainer product and privacy judgment before implementation.

## What This Changes

Request an opt-in Telegram mode that routes forum topics and DM threads into one shared session while preserving topic/thread metadata for replies.

## Best Possible Solution

If accepted, add an explicit opt-in Telegram shared-session mode in the Telegram plugin config, preserve MessageThreadId/IsForum/topic metadata and reply-thread routing, and document the visibility tradeoff with focused routing/schema tests.

## Reproduction Assessment

Yes. Source and tests show the current-main behavior: forum topics produce topic-qualified session keys, while DM threads stay flat only by default and become thread-scoped when configured.

## Solution Assessment

Unclear. The requested shared sessionMode is plausible, but the exact config shape and cross-topic visibility semantics need maintainer product design before code changes are safe.
