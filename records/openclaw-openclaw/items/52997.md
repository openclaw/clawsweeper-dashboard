## Summary

Keep open. Current main now keeps incidental Telegram DM topic metadata on the flat DM session by default, but forum topics still route through topic-qualified session keys and there is no Telegram shared-session config surface; the remaining request is a product/config feature, not a stale cleanup close.

## What This Changes

Request an opt-in Telegram mode that shares one main agent session across forum topics and DM threads while preserving topic/thread metadata for replies.

## Best Possible Solution

Keep this as a Telegram plugin product/design issue: if accepted, add an explicit opt-in shared mode for forum topics and any needed configured DM-topic cases, preserve MessageThreadId/IsForum/TopicName and reply targets, and document the isolation tradeoff with focused routing/schema tests.

## Reproduction Assessment

Yes. Source and existing tests show the current behavior: forum topics use topic-qualified session keys, while incidental DM message_thread_id values stay on the flat DM session unless explicitly configured for thread isolation.

## Solution Assessment

Unclear. The requested shared-session mode is plausible, but the best maintainable fix depends on maintainer decisions about privacy, session tooling, and whether the option should cover only forum topics or also configured DM topic isolation.
