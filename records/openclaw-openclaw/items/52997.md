## Summary

Keep open. Current main still intentionally isolates Telegram forum topics and Telegram DM topics into topic/thread-specific session keys, while preserving topic/thread metadata for prompt context and reply routing; there is still no Telegram config schema or docs support for an opt-in shared session mode.

## What This Changes

Request an opt-in Telegram setting that routes all forum topics and DM threads into one shared main session while carrying topic/thread metadata and replying to the originating topic/thread.

## Best Possible Solution

Keep this as a Telegram-plugin-owned product/design issue. If accepted, add an explicit opt-in shared mode for group topics and DM topics, preserve per-message thread metadata and reply delivery targets, document the isolation tradeoff, and cover the new behavior with Telegram config, routing, and delivery tests.

## Reproduction Assessment

Yes. The current behavior can be reproduced deterministically through existing Telegram routing tests and source inspection: forum topics produce topic-qualified group session keys, and DM topics produce `:thread:<chatId>:<threadId>` session keys.

## Solution Assessment

Unclear. The requested shared-session mode is plausible, but it is not yet clearly the best fix until maintainers decide the privacy and session-tool boundary; if approved, the narrowest maintainable path is an explicit Telegram opt-in rather than changing default isolation.
