## Summary

Keep open. Current main and v2026.5.18 still isolate Telegram forum topics and configured DM threads, and the related shared-session issues/PR cover the broader design but do not replace this issue's Telegram-specific metadata and reply-routing requirements.

## What This Changes

Request an opt-in Telegram mode that routes forum topics and configured DM threads into one shared agent session while preserving topic/thread metadata and reply routing.

## Best Possible Solution

Design an explicit opt-in shared-session contract, preferably aligned with the broader shared-session work, that preserves default isolation while carrying Telegram topic/thread metadata and origin reply routing through schema, docs, and focused routing tests.

## Reproduction Assessment

Yes, at source level. Current source, docs, tests, and the latest release show forum topics become :topic: sessions, configured DM threads can become :thread: sessions, and there is no Telegram shared-session config surface.

## Solution Assessment

Unclear as a final implementation. The requested sessionMode is plausible, but the maintainable solution should be aligned with the broader shared-session contract and explicitly define privacy, sandbox, metadata, and reply-routing behavior.
