## Summary

Keep open. Current main still has intentional Telegram forum-topic session isolation, only partly-flattened DM thread behavior, and no shared-session configuration surface; the remaining request is a feature/product decision rather than conservative cleanup.

## What This Changes

Request an opt-in Telegram configuration mode that routes forum topics and DM threads into one shared agent session while preserving topic/thread metadata for replies.

## Best Possible Solution

If accepted, add an explicit opt-in Telegram shared-session mode with documented privacy tradeoffs, preserve MessageThreadId/IsForum/topic metadata and reply routing, and cover forum plus DM thread cases in schema, docs, and routing tests.

## Reproduction Assessment

Yes for the current behavior: source and tests show forum topics still produce topic-qualified session keys and configured DM topics still create thread-scoped sessions. I did not run a live Telegram forum reproduction in this read-only review.

## Solution Assessment

Unclear. The requested shared sessionMode is plausible, but the safest solution needs maintainer product/privacy design and should avoid colliding with adjacent group-to-main binding proposals.
