## Summary

Keep open: current main and v2026.5.19 still isolate Telegram forum topics and configured DM topic sessions, and the related shared-session trackers do not fully replace this issue's Telegram-specific metadata and reply-routing requirements.

## What This Changes

The issue requests an opt-in Telegram shared-session mode for forum topics and DM threads while preserving topic/thread metadata and origin reply routing.

## Best Possible Solution

Define an explicit opt-in shared-session contract that preserves default isolation, carries Telegram topic/thread metadata through the shared context, and routes replies back to the originating topic/thread with schema, docs, and focused routing tests.

## Reproduction Assessment

Yes at source level, but not as a bug reproduction. Current source, tests, docs, and the latest release show forum topics remain :topic: sessions, configured DM topics can become :thread: sessions, and no Telegram shared-session config surface exists.

## Solution Assessment

Unclear as a final implementation. The requested sessionMode direction is plausible, but the maintainable solution should align with the broader shared-session contract and define privacy, sandbox, metadata, and reply-routing behavior before implementation.
