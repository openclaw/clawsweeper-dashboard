## Summary

Keep open. Current main and the latest release still preserve Telegram forum topics and configured DM threads as separate session scopes, and the open broader group-scope PR does not replace this issue’s Telegram-specific shared-session, metadata, and reply-routing contract.

## What This Changes

The issue requests an opt-in Telegram session mode that shares one main agent session across forum topics or DM threads while preserving topic/thread metadata for routing.

## Best Possible Solution

Design an explicit opt-in shared-session contract, preferably aligned with the broader binding-session work, that preserves default isolation while carrying Telegram topic/thread metadata and origin reply routing through schema, docs, and focused routing tests.

## Reproduction Assessment

Yes, at source level. Current source, tests, docs, and v2026.5.19 show forum topics become topic-qualified sessions, configured DM topics can become thread-qualified sessions, and no Telegram shared-session config surface exists.

## Solution Assessment

Unclear as a final implementation. The requested sessionMode is plausible, but the maintainable solution should align with the broader shared-session/binding contract and explicitly define privacy, metadata, sandbox, and reply-routing behavior.
