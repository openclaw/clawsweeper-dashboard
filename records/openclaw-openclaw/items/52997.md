## Summary

Keep open. Current main still routes Telegram forum topics through topic-qualified session keys and exposes no Telegram shared-session config surface; the DM half is partly addressed by flat default DM handling, but configured DM topic isolation still has no shared override, so the remaining request is a product/config feature.

## What This Changes

Add an opt-in Telegram mode that routes forum topics and DM threads into one shared session while preserving topic/thread metadata for context and replies.

## Best Possible Solution

If accepted, add an explicit opt-in Telegram shared-session mode in the Telegram plugin config, preserve `MessageThreadId`, `IsForum`, `TopicName`, and reply-thread routing, and document the visibility tradeoff with focused routing/schema tests.

## Reproduction Assessment

Yes. Source and tests show the current-main behavior: forum topics still resolve to topic-qualified session keys, while incidental DM thread metadata stays on the flat session unless DM topic isolation is explicitly configured.

## Solution Assessment

Unclear. The requested `sessionMode: "shared"` direction is plausible, but the exact config shape and privacy/session-tool behavior need maintainer product design before implementation.
