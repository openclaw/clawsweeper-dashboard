## Summary

Keep open: current main and the latest release still lack the requested `messages` config option, and Signal still synthesizes `<media:audio>` / `<media:${kind}>` as the outbound body for blank media-only sends. The iMessage visible-send path has been improved, but that only covers one channel and does not resolve the canonical cross-channel product/config request.

## What This Changes

Request a default-off `messages` config option to suppress automatic media placeholder text for media-only outbound sends while preserving current fallback behavior by default.

## Best Possible Solution

Approve a documented default-off `messages` media-placeholder policy, keep the current iMessage attachment-only visible send behavior, and wire Signal plus any remaining outbound placeholder paths through schema, docs, metadata, and focused tests.

## Reproduction Assessment

Yes, by source inspection. Blank Signal media sends with recognized audio MIME types still synthesize `<media:${kind}>`, and the strict `messages` schema has no accepted suppression key.

## Solution Assessment

Yes, directionally. A default-off `messages` option is the narrow maintainable direction, but the option shape and Signal transport fallback policy still need maintainer approval.
