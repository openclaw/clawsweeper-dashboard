## Summary

Keep open: current main still generates visible `<media:...>` placeholder text for blank Signal and iMessage media sends, and the strict `messages` config surface still has no suppression option. The prior focused implementation PRs are closed unmerged, so this remains the canonical feature/config request.

## What This Changes

Add a default-off messages config option to suppress visible media placeholder text for media-only outbound messages, especially audio/voice sends.

## Best Possible Solution

Approve a default-off `messages` config contract, wire it through outbound Signal and iMessage media-only delivery, preserve existing fallback behavior by default, update schema/help/docs/generated metadata, and add focused regression tests.

## Reproduction Assessment

Yes, by source inspection. Blank Signal and iMessage media sends with recognized media MIME types still enter placeholder-producing branches, and the strict messages schema has no accepted suppression key.

## Solution Assessment

Yes, directionally. A default-off messages option is a narrow maintainable direction, but the exact option shape and per-channel delivery policy should be approved before implementation.
