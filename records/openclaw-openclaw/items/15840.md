## Summary

Keep open: current main and the latest release still synthesize Signal media placeholder text for captionless media sends, and the requested `messages` suppression config surface is still absent. The iMessage visible-send case was fixed by a later merged PR, but the canonical Signal/config request remains a product decision.

## What This Changes

The issue requests a default-off configuration option to suppress visible media-placeholder text such as `<media:audio>` for media-only messaging sends.

## Best Possible Solution

Approve a default-off `messages` media-placeholder policy, document it in the strict config surface, wire it through Signal’s media-only send path, and preserve current fallback behavior by default with focused regression coverage.

## Reproduction Assessment

Yes, by source inspection. Captionless Signal media replies pass empty text plus `mediaUrl` into `sendMessageSignal`, audio MIME maps to `audio`, and the send path synthesizes `<media:audio>` before dispatch.

## Solution Assessment

Yes directionally, but the exact contract still needs maintainer approval. A default-off `messages` option is the narrow maintainable path because it preserves current fallback behavior for existing users while allowing clean voice-only sends.
