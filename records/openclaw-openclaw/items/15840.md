## Summary

Keep open: current main still lacks the requested `messages` suppression config, and the remaining Signal media-only path still synthesizes visible `<media:${kind}>` text for blank sends. The recent iMessage fix covers one visible-send path, but the canonical cross-channel config request is still unresolved.

## What This Changes

Add an opt-in `messages` configuration option to suppress automatic media placeholder text such as `<media:audio>` for media-only outbound messaging sends.

## Best Possible Solution

Approve a default-off `messages` media-placeholder policy, keep the current iMessage attachment-only visible-send behavior, wire Signal and any remaining outbound placeholder paths through the chosen config, and update schema, docs, and focused tests together.

## Reproduction Assessment

Yes, by source inspection. Blank Signal media sends with recognized audio/media MIME types still synthesize `<media:${kind}>`, and the strict `messages` schema has no accepted suppression key.

## Solution Assessment

Yes, directionally. A default-off `messages` option is the narrow maintainable path, but boolean versus granular policy and Signal's hidden/non-empty send body need maintainer approval before implementation.
