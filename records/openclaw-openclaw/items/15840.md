## Summary

Keep open: current main and the latest release still synthesize Signal media placeholder text for captionless audio/media sends, and there is still no accepted `messages` config surface to suppress it. The iMessage visible-send case has been fixed separately, but this canonical Signal/config request remains unresolved and needs maintainer product approval.

## What This Changes

The issue requests a default-off `messages` option to suppress visible `<media:audio>` fallback text for media-only outbound messages, especially voice/audio sends.

## Best Possible Solution

Approve a default-off `messages` media-placeholder policy, document it in the strict config surface, wire it through Signal’s media-only send path, and keep current fallback behavior as the default with focused regression coverage.

## Reproduction Assessment

Yes, by source inspection. Captionless Signal media replies pass empty text plus `mediaUrl` into `sendMessageSignal`, audio MIME maps to `audio`, and the send path synthesizes `<media:audio>` before dispatch.

## Solution Assessment

Yes, directionally, but the exact contract still needs a maintainer decision. A default-off `messages` option is the narrow maintainable path because it preserves current fallback behavior for existing users while allowing clean voice-only sends.
