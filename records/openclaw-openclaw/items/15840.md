## Summary

Keep open: current main still emits visible media placeholder text for blank Signal and iMessage media sends, and the requested opt-in `messages` configuration surface has not been added. The related implementation PRs are closed unmerged, so this remains the canonical feature/config request.

## What This Changes

The issue requests a default-off configuration option to suppress visible `<media:audio>` and other media placeholder text for media-only messaging sends.

## Best Possible Solution

Approve a default-off `messages` config contract, wire it through outbound media-only channel paths that synthesize placeholders, preserve existing fallback behavior by default, update schema/help/docs/generated metadata, and add focused regression tests.

## Reproduction Assessment

Yes, by source inspection. Blank Signal and iMessage media sends with recognized audio/media MIME types still enter placeholder-producing branches, and the strict messages schema has no accepted suppression key.

## Solution Assessment

Yes, directionally. A default-off messages option is a narrow maintainable direction, but the exact option shape and per-channel suppression policy should be approved before implementation.
