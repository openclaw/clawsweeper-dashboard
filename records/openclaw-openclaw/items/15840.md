## Summary

Keep open: current main and v2026.5.20 still synthesize Signal media placeholder text for captionless media sends, and the requested `messages` suppression config surface is still absent. The iMessage visible-send case was fixed separately, but the remaining Signal/config request needs a maintainer product decision.

## What This Changes

The issue requests an opt-in `messages` configuration option to suppress visible `<media:audio>` fallback text for media-only messaging sends while preserving current placeholder behavior by default.

## Best Possible Solution

Approve a default-off `messages` media-placeholder policy, document it in the strict config surface, wire it through Signal’s media-only send path, preserve current fallback behavior by default, and keep the already-fixed iMessage visible-send behavior intact.

## Reproduction Assessment

Yes, by source inspection. Captionless Signal media replies pass empty text plus `mediaUrl` into `sendMessageSignal`, audio MIME maps to `audio`, and the send path synthesizes `<media:audio>` before dispatch.

## Solution Assessment

Yes directionally, but the exact contract still needs maintainer approval. A default-off `messages` option is the narrow maintainable path because it preserves current fallback behavior while allowing clean voice-only sends.
