## Summary

Keep open: current main and the latest release still have the remaining Signal placeholder behavior and no accepted `messages` suppression config; the iMessage visible-send case was fixed separately, so this is now a focused Signal/config product decision rather than cleanup.

## What This Changes

Requests a default-off configuration option to suppress visible `<media:audio>` and related synthetic media placeholder text for media-only outbound channel messages.

## Best Possible Solution

Approve a default-off `messages` media-placeholder policy, wire it through Signal media-only sends with focused coverage, and preserve the shipped iMessage attachment-only behavior.

## Reproduction Assessment

Yes, by source inspection. Captionless Signal media replies call `sendMessageSignal` with empty text plus `mediaUrl`, `audio/*` maps to `audio`, and `sendMessageSignal` assigns `<media:audio>` before dispatch.

## Solution Assessment

Yes directionally, but the exact contract needs maintainer approval. A default-off `messages` policy is the narrow maintainable shape because it preserves existing fallback behavior while allowing clean media-only sends.
