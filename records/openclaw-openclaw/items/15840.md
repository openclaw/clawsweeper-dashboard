## Summary

Keep this open as the canonical feature/config request: current main still synthesizes visible Signal and iMessage media placeholder text for blank media-only sends, the strict messages config surface still has no suppression option, and the related implementation PRs are closed unmerged.

## What This Changes

Request a messages config option to suppress media placeholder text such as `<media:audio>` for media-only messaging sends while preserving the current fallback behavior by default.

## Best Possible Solution

Add a documented, default-off messages setting for media placeholder suppression, wire it through Signal and iMessage media-only outbound paths, preserve existing fallback behavior by default, and cover the config/schema/docs plus focused channel tests.

## Reproduction Assessment

Yes, by source inspection. Current main still turns blank Signal/iMessage media sends with audio MIME into `<media:audio>`, and the strict config schema has no accepted suppression key.

## Solution Assessment

Yes, directionally. A default-off messages option is the narrow maintainable path, but the exact public option shape should be maintainer-approved before implementation.
