## Summary

Keep open: current main still lacks a `messages` suppression config and Signal still synthesizes visible media placeholders for blank media-only sends. The merged iMessage work covers one channel's visible-send behavior, but it does not satisfy the requested cross-channel opt-in config surface.

## What This Changes

Add an opt-in `messages` configuration option to suppress fallback media placeholder text such as `<media:audio>` for media-only messaging sends.

## Best Possible Solution

Approve a default-off `messages` media-placeholder policy, keep the current iMessage attachment-only send behavior, wire Signal and any remaining outbound placeholder paths through the chosen config, and update schema/docs/tests together.

## Reproduction Assessment

Yes, by source inspection. Blank Signal media sends with recognized audio/media MIME types still synthesize `<media:${kind}>`, and the strict `messages` schema has no accepted suppression key.

## Solution Assessment

Yes, directionally. A default-off `messages` option is the narrow maintainable path, but the exact option shape and Signal transport fallback policy need maintainer approval before implementation.
