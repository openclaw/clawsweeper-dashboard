## Summary

Keep open as the canonical feature/config request: current main still synthesizes visible media placeholder text for blank Signal and iMessage media sends, the strict messages config surface has no suppression option, and the related implementation PRs are closed unmerged.

## What This Changes

Add an opt-in messages setting to suppress automatic `<media:audio>` and related media placeholder fallback text for media-only voice/audio sends while preserving the default fallback behavior.

## Best Possible Solution

Approve a default-off messages config shape, wire it through Signal and iMessage media-only outbound delivery, preserve current fallback behavior by default, update schema/docs/generated metadata, and add focused regression tests.

## Reproduction Assessment

Yes, by source inspection. Blank Signal and iMessage media sends with audio MIME still enter placeholder-producing branches, and the strict messages schema has no accepted suppression key.

## Solution Assessment

Yes, directionally. A default-off messages option is the narrow maintainable direction, but the exact public option shape should be maintainer-approved before implementation.
