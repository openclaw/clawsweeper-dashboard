## Summary

Keep open. Current main still synthesizes visible placeholders for blank media-only Signal and iMessage sends, `audio/*` still maps to the `audio` placeholder kind, the `messages` config/schema/help surface has no suppression option, and the related implementation PRs are closed unmerged.

## What This Changes

Request a default-off `messages` config option to suppress visible media placeholder text such as `<media:audio>` for media-only channel sends.

## Best Possible Solution

Keep this open as the canonical request and implement a documented default-off `messages` media-placeholder policy, wired through the affected channel send paths while preserving existing fallback behavior by default.

## Reproduction Assessment

Yes for the current behavior, though this is a feature request rather than a broken-contract bug. Source inspection shows media-only Signal and iMessage replies pass blank captions into send paths that synthesize `<media:${kind}>`, and `audio/*` maps to `audio`.

## Solution Assessment

Unclear until maintainers choose the public config shape. The narrow maintainable direction is a default-off `messages` option with strict schema, generated config metadata/help/docs, and focused Signal/iMessage media-only tests.
