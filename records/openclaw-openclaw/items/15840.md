## Summary

Keep open. Current main still synthesizes visible media placeholders for blank media-only Signal and iMessage sends, audio MIME types still resolve to the `audio` placeholder kind, and the strict `messages` config surface still has no suppression option; the related implementation PRs were closed unmerged.

## What This Changes

Request a default-off `messages` config option to suppress visible `<media:audio>` or other `<media:...>` fallback text for media-only outbound messages.

## Best Possible Solution

Implement a documented, default-off `messages` setting wired through Signal and iMessage media-only send paths while preserving current fallback behavior by default.

## Reproduction Assessment

Yes. A high-confidence source-level path exists: a media-only Signal or iMessage reply with an empty caption and an `audio/*` content type reaches the channel send path, which constructs `<media:audio>` before delivery.

## Solution Assessment

Yes. A default-off shared `messages` option plus channel-owned Signal and iMessage wiring is the narrow maintainable path; removing placeholders globally would change the existing fallback behavior.
