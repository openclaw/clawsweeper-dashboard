## Summary

Keep this open as the canonical config request. Current main still synthesizes visible `<media:...>` fallback text for blank Signal and iMessage media sends, audio MIME types still map to `audio`, and the strict `messages` config surface still has no suppression option; the related implementation PRs were closed unmerged.

## What This Changes

The issue requests a default-off `messages` config option to suppress visible media placeholder text for media-only outbound messages, especially audio voice notes.

## Best Possible Solution

Add a documented default-off `messages.suppressMediaPlaceholders` setting, keep existing fallback behavior by default, and wire the setting through affected outbound media-only channel paths with focused tests.

## Reproduction Assessment

Yes, by source inspection. Current `main` still turns blank media-only Signal/iMessage sends with audio MIME into `<media:audio>`, and the strict config schema has no accepted suppression key.

## Solution Assessment

Yes. The narrow maintainable path is a default-off `messages` option plus channel-specific wiring and tests; removing placeholders globally would discard the existing accessibility/fallback behavior.

## Repair Work Prompt

Implement the canonical media-placeholder suppression request for https://github.com/openclaw/openclaw/issues/15840. Add a default-off `messages.suppressMediaPlaceholders?: boolean` config option, update strict zod validation, config labels/help/generated metadata/docs as required, and preserve current placeholder fallback behavior when the option is unset or false. Wire the option through Signal and iMessage outbound media-only send paths so blank captions with audio/image/video/document attachments can omit the visible `<media:...>` text when enabled. Add focused regression tests for Signal and iMessage media-only sends, including audio MIME resolving to the suppressed path, and add config-schema coverage that accepts the new key while preserving strict rejection for unrelated keys. Include a user-facing CHANGELOG.md entry. Do not remove inbound media placeholders, do not make suppression the default, and do not introduce bundled-channel ids into core beyond the generic config contract.
