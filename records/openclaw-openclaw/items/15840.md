## Summary

Keep open: current main and latest release still synthesize Signal media placeholder text for captionless media sends, and the strict `messages` config still has no suppression option. The iMessage visible-send case is already fixed separately, so the remaining work is a Signal/config product decision rather than conservative cleanup.

## What This Changes

Add a config option to suppress automatic media placeholder text such as `<media:audio>` for media-only channel sends while preserving default fallback behavior.

## Best Possible Solution

Approve and document a default-off `messages` media-placeholder policy, wire it through Signal media-only sends while preserving default fallback behavior, and keep the shipped iMessage attachment-only behavior intact.

## Reproduction Assessment

Yes, by source inspection. Captionless Signal media replies call `sendMessageSignal` with empty text plus `mediaUrl`, `audio/*` maps to `audio`, and `sendMessageSignal` assigns `<media:audio>` before dispatch.

## Solution Assessment

Yes directionally, but the exact contract needs maintainer approval. A default-off `messages` option is the narrow maintainable path because it preserves existing fallback behavior while allowing clean voice-only sends.
