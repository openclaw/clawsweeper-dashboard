## Summary

Keep open: current main and v2026.5.20 still synthesize Signal media placeholder text for captionless media sends, and the strict `messages` config has no suppression option. The iMessage visible-send case was fixed separately, so the remaining Signal/config request needs maintainer product approval.

## What This Changes

Request a default-off `messages` configuration option to suppress synthesized `<media:audio>` or other `<media:...>` text for media-only voice/audio sends.

## Best Possible Solution

Approve and document a default-off `messages` media-placeholder policy, wire it through Signal media-only sends while preserving default fallback behavior, and keep iMessage's shipped attachment-only delivery intact.

## Reproduction Assessment

Yes, by source inspection. Captionless Signal media replies call `sendMessageSignal` with empty text plus `mediaUrl`, `audio/*` maps to `audio`, and `sendMessageSignal` assigns `<media:audio>` before dispatch.

## Solution Assessment

Yes directionally, but the exact contract needs maintainer approval. A default-off `messages` option is the narrow maintainable path because it preserves existing fallback behavior while allowing clean voice-only sends.
