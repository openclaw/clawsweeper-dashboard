## Summary

Keep open: current main and v2026.5.20 still synthesize Signal media placeholder text for captionless media sends, and there is still no `messages` config/schema/docs surface to suppress it. The iMessage visible-send case was fixed separately, but the remaining Signal/config request needs maintainer product approval.

## What This Changes

Requests a default-off `messages` config option to suppress visible media placeholder text such as `<media:audio>` for audio/voice media-only sends.

## Best Possible Solution

Approve and document a default-off `messages` media-placeholder policy, wire it through Signal media-only sends while preserving default fallback behavior, and keep iMessage's shipped attachment-only send behavior intact.

## Reproduction Assessment

Yes, by source inspection. Captionless Signal media replies call `sendMessageSignal` with empty text plus `mediaUrl`, `audio/*` maps to `audio`, and `sendMessageSignal` assigns `<media:audio>` before dispatch.

## Solution Assessment

Yes directionally, but the exact contract needs maintainer approval. A default-off `messages` option is the narrow maintainable path because it preserves existing fallback behavior while allowing clean voice-only sends.
