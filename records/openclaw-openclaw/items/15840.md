## Summary

Keep this open as the canonical feature/config request: current main still synthesizes visible media placeholder text for blank Signal and iMessage media sends, and the strict messages config surface has no suppression option.

## What This Changes

Add an opt-in messages config option to suppress automatic media placeholder text for media-only messaging sends while preserving current fallback behavior by default.

## Best Possible Solution

Add a documented, default-off `messages.suppressMediaPlaceholders` setting and wire it through the affected outbound media-only channel paths with regression coverage.

## Reproduction Assessment

Yes, by source inspection. Current main still turns blank Signal/iMessage media sends with audio MIME into `<media:audio>`, and the strict config schema has no accepted suppression key.

## Solution Assessment

Yes. A default-off messages option is the narrow maintainable path because it gives users clean media-only sends without removing the existing accessibility/fallback behavior for everyone.
