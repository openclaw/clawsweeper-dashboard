## Summary

Keep open: current main still lacks the requested config option, and Signal still sends `<media:audio>` / `<media:${kind}>` as the outbound body for blank media-only sends. The iMessage-specific visible-send fix is present, but it only resolves one channel and not the cross-channel opt-in config request.

## What This Changes

Add a `messages` configuration option to suppress automatic media-placeholder text for media-only channel sends while preserving the default fallback behavior.

## Best Possible Solution

Approve a default-off `messages` media-placeholder policy, keep the current iMessage attachment-only send behavior, and wire Signal plus any remaining outbound placeholder paths through schema, docs, metadata, and focused tests.

## Reproduction Assessment

Yes, by source inspection. Blank Signal media sends with recognized audio/media MIME types still synthesize `<media:${kind}>`, and the strict `messages` schema has no accepted suppression key.

## Solution Assessment

Yes, directionally. A default-off `messages` option is the narrow maintainable path, but the option shape and Signal transport fallback policy still need maintainer approval.
