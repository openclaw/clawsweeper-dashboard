## Summary

Keep open as the canonical feature/config request: current main still synthesizes visible media placeholder text for blank Signal and iMessage media sends, the strict messages config surface has no suppression option, and the related implementation PRs are closed unmerged.

## What This Changes

Request a default-off messages configuration option to suppress visible `<media:audio>` or other media placeholder text for media-only outbound messages.

## Best Possible Solution

Approve a default-off messages config contract, wire it through outbound Signal and iMessage media-only delivery, preserve existing fallback behavior by default, update schema/docs/generated metadata, and add focused regression tests.

## Reproduction Assessment

Yes, by source inspection. Blank Signal and iMessage media sends with recognized audio/media MIME types still enter placeholder-producing branches, and the strict messages schema has no accepted suppression key.

## Solution Assessment

Yes, directionally. A default-off messages option is a narrow maintainable direction, but the exact option shape and per-channel delivery policy should be approved before implementation.
