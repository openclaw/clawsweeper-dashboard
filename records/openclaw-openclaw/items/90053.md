## What This Changes

The PR adds a `skills.proposals.requestRevision` gateway RPC and routes Skill Workshop revision sends through it so the UI queues only the visible user instructions while the server validates the pending proposal and builds the hidden agent handoff.

## Best Possible Solution

Land a refreshed branch that preserves current-main Skill Workshop session routing while keeping the revision handoff server-built and hidden from the chat transcript.

## Reproduction Assessment

Yes, source-reproducible: current main builds the Skill Workshop revision handoff and user instructions into one string before sending it to chat. I did not run a live UI reproduction because this review is read-only.

## Solution Assessment

Yes, this is the best fix shape I found: the gateway owns the agent-only instruction and the UI persists/retries only the visible user instructions. The safer alternative, letting the UI construct hidden provenance for `chat.send`, would put server-owned prompt context back in the client.
