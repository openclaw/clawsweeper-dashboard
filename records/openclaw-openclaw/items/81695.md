## Summary

Close as a configuration/support issue, not a core bug: current OpenClaw treats MiniMax-M2.7 as a text-only chat model, routes MiniMax image understanding through MiniMax-VL-01/VLM, and already base64-encodes image blocks for image-capable Anthropic-compatible models.

## What This Changes

The issue requests making MiniMax-M2.7 work as the image tool's imageModel by sending base64 image content through the MiniMax anthropic-messages path.

## Best Possible Solution

Keep MiniMax-M2.7 on the text chat path and configure agents.defaults.imageModel.primary to minimax/MiniMax-VL-01 or minimax-portal/MiniMax-VL-01 for image understanding.

## Reproduction Assessment

No. Current main does not use the bundled MiniMax-M2.7 chat model as an image-capable model; the supported MiniMax image path is MiniMax-VL-01 through the VLM endpoint with prompt plus base64 image_url.

## Solution Assessment

No. Encoding images in a MiniMax M2.7 anthropic-messages adapter is not the right fix because that MiniMax text endpoint does not support image/document input, while OpenClaw already has a separate MiniMax VLM image-understanding path.
