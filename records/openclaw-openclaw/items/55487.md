## Summary

Keep this PR open for now, but it is not merge-ready: current main has a newer benign uncaught-exception seam, the branch is conflicting and missing real gateway proof, and the proposed classifier still misses the reported TLS `setSession` null-deref while risking fatal-wrapper suppression.

## What This Changes

The PR adds strict transient network/TLS handling to uncaught-exception handlers, classifier tests, and a raw gateway upgrade socket error listener.

## Best Possible Solution

Rebase onto current main's benign uncaught-exception seam, preserve fatal-wrapper precedence, add a narrow TLS `setSession` null-deref detector with focused regression coverage, and provide redacted live gateway proof.

## Reproduction Assessment

No high-confidence live reproduction is available for the intermittent WSL2/concurrent-agent crash. The linked logs and source inspection do give a source-level way to check the PR gaps: fatal wrappers with nested transient causes can be suppressed, and the reported uncoded TLS `setSession` TypeError is not matched.

## Solution Assessment

No, not as-is. The direction is plausible only after rebasing onto current main, keeping fatal wrappers fatal, explicitly covering the reported TLS null-deref path, and adding real gateway proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve fatal precedence before transient suppression:** `src/cli/run-main.ts:164-169`
  - body: The handler returns when `isStrictTransientNetworkError(error)` is true, and that classifier walks nested causes. A fatal wrapper such as `ERR_WORKER_UNCAUGHT_EXCEPTION` with a transient network cause can be logged as non-fatal and keep the process running after a fatal uncaught exception, so check fatal wrappers before transient suppression or make the strict classifier ignore them.
  - confidence: 0.86
- **[P1] Cover the reported TLS setSession crash:** `src/infra/unhandled-rejections.ts:161-167`
  - body: The linked crash reports show `TypeError: Cannot read properties of null (reading 'setSession')` in `TLSSocket.setSession`, but the strict classifier only matches codes, network-specific names, and generic TLS/socket snippets. That uncoded TypeError still falls through to `process.exit(1)`, so add a narrow stack/message detector or source-scoped handler with regression coverage.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The patch changes global uncaught-exception policy and can keep the process alive after a fatal wrapper when a nested transient cause matches.
