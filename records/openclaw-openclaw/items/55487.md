## Summary

Keep open; current main has not fully implemented the requested crash hardening, and the PR remains useful but is not merge-ready because its uncaught-exception suppression can bypass fatal-wrapper handling.

## What This Changes

The PR adds strict transient network/TLS suppression to uncaughtException handlers, a raw gateway upgrade-socket error listener, and classifier unit coverage.

## Best Possible Solution

Update the fix against current main's shared uncaught-exception policy seam, preserve fatal handling, add focused coverage for the actual TLS/undici signatures, and keep the upgrade-socket guard if it still closes a real raw-socket gap.

## Reproduction Assessment

No high-confidence deterministic reproduction is available. The linked reports provide detailed intermittent WSL2/concurrent-subagent crash evidence and current source inspection confirms gaps, but the PR's unstable-network verification remains manual.

## Solution Assessment

No. The direction is plausible, but this patch is not the best current solution because it targets an older handler shape and can suppress fatal wrappers before the current fatal/registered-handler policy gets a chance to run.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve fatal precedence before suppressing uncaught errors:** `src/cli/run-main.ts:164-169`
  - body: The strict classifier recurses through nested causes, and this handler returns before the fatal path runs. A fatal wrapper such as `ERR_WORKER_UNCAUGHT_EXCEPTION` carrying a transient network cause can be logged as non-fatal and leave the CLI or gateway running after a fatal worker crash, so keep fatal wrappers ahead of transient suppression or make the strict check ignore them.
  - confidence: 0.86
