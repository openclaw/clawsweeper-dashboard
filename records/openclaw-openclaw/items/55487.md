Keep open: the PR addresses a real gateway crash class, but the branch is still missing contributor real-behavior proof and has correctness/security blockers in the uncaught-exception suppression path.

## What This Changes

The PR adds strict transient network/TLS classification to uncaughtException handlers, adds classifier tests, and attaches a gateway upgrade socket error listener.

## Best Possible Solution

Rebase onto the current uncaught-exception seam, preserve fatal/config precedence, cover the reported TLS `setSession` crash with a narrow signature or scoped dependency handler, and add focused tests plus redacted real gateway proof.

## Reproduction Assessment

No high-confidence live reproduction is available. Source inspection and the linked reports show the current-main allowlist gap and intermittent WSL2/concurrent-agent TLS crash evidence, but the crash has not been deterministically reproduced here.

## Solution Assessment

No, not as-is. The direction is useful, but the PR should integrate with the current `isBenignUncaughtExceptionError` seam, keep fatal wrappers ahead of transient suppression, and explicitly cover the linked `setSession` null-deref signature.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve fatal precedence before suppressing uncaught errors:** `src/cli/run-main.ts:164-169`
  - body: This handler returns for `isStrictTransientNetworkError(error)` before fatal handling, and the strict classifier walks nested causes. A fatal wrapper such as `ERR_WORKER_UNCAUGHT_EXCEPTION` with a transient network cause can be logged as non-fatal and leave the process running after a fatal exception, so keep fatal wrappers ahead of transient suppression or make the strict check ignore them.
  - confidence: 0.86
- **[P1] Cover the reported setSession TLS crash signature:** `src/infra/unhandled-rejections.ts:159-165`
  - body: The linked crash reports show `Cannot read properties of null (reading 'setSession')`, but this strict classifier only matches codes, names, and the listed TLS/socket message snippets. A plain TypeError from the TLS setSession path can still fall through to `process.exit(1)`, so add a narrow signature or scoped handler with regression coverage before closing those reports.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff changes global uncaught-exception handling and can keep the process running after a fatal wrapper when a nested transient cause matches.
