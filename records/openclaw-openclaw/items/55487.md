## Summary

Keep open: the gateway crash is still plausible, but this branch is not mergeable as-is because it conflicts with current main, lacks contributor real-behavior proof, and still misses the reported TLS `setSession` null-deref while risking fatal uncaught-exception suppression.

## What This Changes

The PR adds strict transient network/TLS classification for uncaught exceptions in both CLI entrypoints, classifier tests, and a raw gateway upgrade socket error listener.

## Best Possible Solution

Rebase onto the current benign-uncaught seam, preserve fatal precedence, add a narrow TLS `setSession` null-deref detector or source-scoped handler, keep focused regression tests, and provide redacted live gateway proof.

## Reproduction Assessment

No high-confidence live reproduction is available. Source inspection plus the linked crash reports show current main still lacks a `setSession` TLS null-deref match, but the intermittent WSL2/concurrent-agent crash was not reproduced deterministically here.

## Solution Assessment

No, not as-is. The direction is useful, but the patch should integrate with current main's benign uncaught-exception seam, keep fatal wrappers fatal, and explicitly cover the reported TLS null-deref path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve fatal precedence before suppressing uncaught errors:** `src/cli/run-main.ts:164-169`
  - body: The uncaught-exception handler returns for `isStrictTransientNetworkError(error)`, and that classifier walks nested causes. A fatal wrapper such as `ERR_WORKER_UNCAUGHT_EXCEPTION` with a transient network cause can be logged as non-fatal and leave the process running after a fatal uncaught exception, so keep fatal handling ahead of transient suppression or make the strict check ignore fatal wrappers.
  - confidence: 0.86
- **[P1] Cover the reported setSession TLS crash signature:** `src/infra/unhandled-rejections.ts:161-167`
  - body: The linked crash reports show `Cannot read properties of null (reading 'setSession')`, but the strict classifier only matches codes, names, and the listed TLS/socket message snippets. A plain TypeError from the TLS `setSession` path can still fall through to `process.exit(1)`, so add a narrow signature or scoped handler with regression coverage before closing those reports.
  - confidence: 0.82

## Security Review

Status: needs_attention
