Full review comments:

- **[P1] Handle the reported TLS setSession crash:** `src/infra/unhandled-rejections.ts:193`
  - body: The linked crash reports show an uncoded `TypeError: Cannot read properties of null (reading 'setSession')` in `TLSSocket.setSession`, but this strict classifier only matches codes, selected names, and generic TLS/socket snippets. That reported TypeError still falls through to `process.exit(1)`, so add a narrow stack/message detector with regression coverage before claiming the gateway crash is fixed.
  - confidence: 0.86
- **[P1] Preserve fatal precedence before transient suppression:** `src/cli/run-main.ts:164-169`
  - body: The new handler returns when `isStrictTransientNetworkError(error)` matches, and that classifier walks nested causes. A fatal wrapper such as `ERR_WORKER_UNCAUGHT_EXCEPTION` with a transient network cause can be logged as non-fatal and keep the process running after a fatal uncaught exception, so check fatal wrappers before transient suppression or make the strict classifier ignore them.
  - confidence: 0.84
