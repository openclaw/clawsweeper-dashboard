- **[P1] Handle the reported TLS setSession crash:** `src/infra/unhandled-rejections.ts:213-214`
  - body: The linked crashes are an uncoded `TypeError: Cannot read properties of null (reading 'setSession')` from `TLSSocket.setSession`, but this strict classifier only accepts network codes, strict names, code text, or the listed socket/TLS snippets. That error will still fall through to `process.exit(1)`, so the PR may not fix the gateway crash it closes.
  - confidence: 0.88
- **[P1] Preserve fatal precedence before suppression:** `src/infra/unhandled-rejections.ts:192-194`
  - body: `isStrictTransientNetworkError` walks nested causes and returns true as soon as any candidate has a transient network code. Since the uncaughtException callers do not check fatal codes first, a fatal wrapper such as an out-of-memory or worker-fatal error with a nested network cause can be logged and ignored instead of failing closed.
  - confidence: 0.78
