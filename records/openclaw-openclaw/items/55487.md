## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve fatal precedence before suppressing uncaught errors:** `src/cli/run-main.ts:164-169`
  - body: This handler returns for `isStrictTransientNetworkError(error)` before any fatal/config-style classification, and the strict classifier walks nested causes. A fatal wrapper such as `ERR_WORKER_UNCAUGHT_EXCEPTION` with a transient network cause can be logged as non-fatal and leave the CLI or gateway running after a fatal exception, so keep fatal wrappers ahead of transient suppression or make the strict check ignore them.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `src/index.ts:96-110`
  - body: This is a user-facing `fix(gateway)` crash-behavior change, but the PR only changes runtime and test files. OpenClaw policy requires a `CHANGELOG.md` entry for user-facing fixes before merge.
  - confidence: 0.9
