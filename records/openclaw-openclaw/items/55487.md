## Summary

Keep open. The PR targets a real gateway crash class, but the current diff still misses the specific TLS `setSession` TypeError claimed by the linked reports, can suppress fatal wrappers before the fatal path, and lacks contributor real-behavior proof.

## What This Changes

The PR adds strict transient network/TLS uncaught-exception suppression in the CLI entrypoints, classifier tests, and a raw gateway upgrade socket error listener.

## Best Possible Solution

Rebase onto the current `isBenignUncaughtExceptionError` seam, keep fatal wrappers fatal, add a narrow TLS `setSession` fingerprint or source-scoped handler, and require focused tests plus redacted live gateway proof.

## Reproduction Assessment

No high-confidence live reproduction is available. Source inspection and the linked crash logs show the classifier gap, but the WSL2/concurrent-agent crash was not reproduced deterministically here.

## Solution Assessment

No, not as-is. The direction is useful, but the patch should integrate with the current benign-uncaught seam, preserve fatal precedence, and explicitly cover the reported TLS `setSession` null-deref path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve fatal precedence before suppressing uncaught errors:** `src/cli/run-main.ts:164-169`
  - body: This handler returns for `isStrictTransientNetworkError(error)`, and the strict classifier walks nested causes. A fatal wrapper such as `ERR_WORKER_UNCAUGHT_EXCEPTION` with a transient network cause can be logged as non-fatal and leave the process running after an uncaught fatal exception, so keep fatal handling ahead of transient suppression or make the strict check reject fatal wrappers.
  - confidence: 0.86
- **[P1] Cover the reported setSession TLS crash signature:** `src/infra/unhandled-rejections.ts:159-165`
  - body: The linked crash reports show `Cannot read properties of null (reading 'setSession')`, but this strict classifier only matches codes, names, and the listed TLS/socket message snippets. A plain TypeError from the TLS `setSession` path can still fall through to `process.exit(1)`, so add a narrow signature or scoped handler with regression coverage before closing those reports.
  - confidence: 0.86

## Security Review
