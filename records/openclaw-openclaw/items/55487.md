## Summary

Keep open. The PR is a plausible fix for real crash reports, but it still misses the reported TLS `setSession` null-deref signature, changes global uncaught-exception behavior in a way that can suppress fatal wrappers, and lacks contributor real-behavior proof.

## What This Changes

The PR adds strict transient network/TLS classification for uncaught exceptions, wires it into both CLI entrypoint handlers, adds classifier tests, and attaches a raw gateway upgrade socket error listener.

## Best Possible Solution

Rebase onto the current benign-uncaught seam, preserve fatal precedence, add a narrow TLS `setSession` null-deref detector or source-scoped handler, and include focused tests plus redacted live gateway proof.

## Reproduction Assessment

No high-confidence live reproduction is available. Source inspection plus the linked crash reports show the reported `setSession` signature is not covered by the PR's strict classifier, but the intermittent WSL2/concurrent-agent crash was not reproduced deterministically here.

## Solution Assessment

No, not as-is. The direction is useful, but the patch should integrate with current main's benign uncaught-exception seam, keep fatal wrappers fatal, and explicitly cover the reported TLS null-deref path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve fatal precedence before suppressing uncaught errors:** `src/cli/run-main.ts:164-169`
  - body: The uncaught-exception handler returns for `isStrictTransientNetworkError(error)`, and that classifier walks nested causes. A fatal wrapper such as `ERR_WORKER_UNCAUGHT_EXCEPTION` with a transient network cause can be logged as non-fatal and leave the process running after a fatal uncaught exception, so keep fatal handling ahead of transient suppression or make the strict check ignore fatal wrappers.
  - confidence: 0.86
- **[P1] Cover the reported setSession TLS crash signature:** `src/infra/unhandled-rejections.ts:159-165`
  - body: The linked crash reports show `Cannot read properties of null (reading 'setSession')`, but the strict classifier only matches codes, names, and the listed TLS/socket message snippets. A plain TypeError from the TLS `setSession` path can still fall through to `process.exit(1)`, so add a narrow signature or scoped handler with regression coverage before closing those reports.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff changes global uncaught-exception policy and can keep the process alive after a fatal wrapper when a nested transient cause matches.

Concerns:

  - body: Because uncaught-exception suppression checks nested transient causes before fatal handling, a fatal worker/runtime wrapper with a network cause can bypass process exit and leave the gateway or CLI in an undefined state.
