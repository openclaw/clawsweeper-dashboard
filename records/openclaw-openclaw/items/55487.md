Full review comments:

- **[P1] Handle the reported TLS setSession crash:** `src/infra/unhandled-rejections.ts:210-214`
  - body: The linked crash reports show an uncoded `TypeError: Cannot read properties of null (reading 'setSession')` in `TLSSocket.setSession`, but this classifier only checks codes, selected names, and message snippets. That reported TypeError still falls through to the fatal path, so add a narrow stack/message detector with regression coverage before claiming this fixes the gateway crash.
  - confidence: 0.88
- **[P1] Preserve fatal precedence before suppression:** `src/cli/run-main.ts:164-169`
  - body: The new uncaughtException handler returns when `isStrictTransientNetworkError(error)` matches, and that classifier walks nested causes. A fatal wrapper with a transient network cause can be logged as non-fatal and keep the process running after a fatal uncaught exception, so check fatal wrappers before transient suppression or make the strict classifier ignore them.
  - confidence: 0.83
