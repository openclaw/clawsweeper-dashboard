## Summary

Keep this PR open, but it is not mergeable as-is: the branch tries to harden transient network/TLS uncaught exceptions, yet it misses the linked `setSession` null-dereference signature, predates current main's benign-uncaught handler seam, and still lacks contributor real-behavior proof.

## What This Changes

The PR adds strict network/TLS classification to global uncaught-exception handlers, adds classifier tests, and installs a raw gateway upgrade socket error listener.

## Best Possible Solution

Rebase onto current main's benign-uncaught handler seam, add a narrow TLS `setSession` null-deref detector or source-scoped handler, preserve fatal precedence, add focused regression tests, and include redacted real gateway proof.

## Reproduction Assessment

No deterministic live reproduction was established. Source inspection plus the linked crash logs show that the reported plain TLS `setSession` TypeError would still fall through the current and proposed classifiers to the fatal uncaught-exception path.

## Solution Assessment

No, not as-is. The direction is useful, but the PR should integrate with current main's benign uncaught-exception seam, keep fatal wrappers fatal, and explicitly cover the reported TLS null-deref signature.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Cover the reported TLS setSession null dereference:** `src/infra/unhandled-rejections.ts:161-166`
  - body: The linked crash reports show `TypeError: Cannot read properties of null (reading 'setSession')`, but this strict allowlist only matches codes, names, and generic TLS/socket snippets. A plain TypeError from the Node TLS `setSession` path still returns false and reaches `process.exit(1)`, so this PR does not fix the reported crash.
  - confidence: 0.86
- **[P1] Preserve fatal precedence before transient suppression:** `src/cli/run-main.ts:164-169`
  - body: The uncaught-exception handler returns for `isStrictTransientNetworkError(error)`, and that classifier walks nested causes and aggregate errors. A fatal wrapper such as a worker uncaught exception with a nested transient network cause can now be logged as non-fatal, so fatal wrappers should be rejected before suppressing transient network errors.
  - confidence: 0.82

## Security Review
