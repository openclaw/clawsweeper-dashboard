## Summary

Keep this PR open. Current main still does not cover the TLS/undici uncaught-exception signatures or the raw upgrade-socket listener gap, but the PR branch is not merge-ready because its uncaught-exception suppression can run before fatal wrapper handling.

## What This Changes

The PR adds uncaught-exception suppression for AbortError and strict transient network/TLS errors, a gateway upgrade-socket error listener, and classifier unit coverage.

## Best Possible Solution

Rebase or replace the PR on current main, implement the new TLS/undici signatures through the existing benign uncaught-exception policy seam with fatal precedence preserved, keep the upgrade-socket guard if still needed, and add focused tests plus a changelog entry.

## Reproduction Assessment

No. The linked reports provide detailed intermittent WSL2/concurrent subagent crash evidence and source inspection confirms the current-main gap, but there is no deterministic local reproduction path for the TLS setSession race.

## Solution Assessment

No. The direction is right, but this branch is not the best current implementation because it layers stale handler logic over the newer uncaught-exception seam and can suppress fatal wrappers with transient causes.
