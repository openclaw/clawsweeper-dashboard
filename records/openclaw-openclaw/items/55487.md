## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve fatal precedence before transient suppression:** `src/cli/run-main.ts:164-169`
  - body: The handler returns when `isStrictTransientNetworkError(error)` is true, and that classifier walks nested causes. A fatal wrapper such as `ERR_WORKER_UNCAUGHT_EXCEPTION` with a transient network cause can be logged as non-fatal and keep the process running after a fatal uncaught exception, so check fatal wrappers before transient suppression or make the strict classifier ignore them.
  - confidence: 0.86
- **[P1] Handle the reported TLS setSession crash:** `src/infra/unhandled-rejections.ts:161-167`
  - body: The linked crash reports show `TypeError: Cannot read properties of null (reading 'setSession')` in `TLSSocket.setSession`, but the strict classifier only matches codes, network-specific names, and generic TLS/socket snippets. That uncoded TypeError still falls through to `process.exit(1)`, so add a narrow stack/message detector with regression coverage.
  - confidence: 0.84
