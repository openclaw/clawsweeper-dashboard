## Summary

Keep open: this is a plausible gateway bug fix, but the branch is not mergeable as-is because it misses the linked TLS `setSession` crash signature, can suppress fatal wrappers with nested transient causes, lacks after-fix real behavior proof, and is currently conflicting.

## What This Changes

The PR adds strict transient network/TLS classification for uncaught exceptions, wires it into both CLI entrypoint handlers, adds classifier tests, and attaches an upgrade socket error listener.

## Best Possible Solution

Rebase onto the current `isBenignUncaughtExceptionError` seam, preserve fatal precedence, add a narrowly fingerprinted TLS `setSession` detector or source-scoped handler, and include focused tests plus redacted live gateway proof.

## Reproduction Assessment

No high-confidence live reproduction is available. Source inspection and the linked reports show the current-main allowlist gap and the PR's missing `setSession` signature, but the WSL2/concurrent-agent crash was not reproduced deterministically here.

## Solution Assessment

No, not as-is. The direction is useful, but the patch should integrate with the current benign-uncaught seam, keep fatal wrappers fatal, and explicitly cover the reported TLS `setSession` null-deref path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve fatal precedence before suppressing uncaught errors:** `src/cli/run-main.ts:164-169`
  - body: This handler returns for `isStrictTransientNetworkError(error)` before the fatal path, and the strict classifier walks nested causes. A fatal wrapper such as `ERR_WORKER_UNCAUGHT_EXCEPTION` with a transient network cause can be logged as non-fatal and leave the process running after an uncaught fatal exception, so keep fatal handling ahead of transient suppression or make the strict check ignore fatal wrappers.
  - confidence: 0.86
- **[P1] Cover the reported setSession TLS crash signature:** `src/infra/unhandled-rejections.ts:159-165`
  - body: The linked crash reports show `Cannot read properties of null (reading 'setSession')`, but this strict classifier only matches codes, names, and the listed TLS/socket message snippets. A plain TypeError from the TLS `setSession` path can still fall through to `process.exit(1)`, so add a narrow signature or scoped handler with regression coverage before closing those reports.
  - confidence: 0.82

## Security Review
