## Summary

Keep open: current main still has the shipped SDK zod resolution bug, and this PR is a plausible fix path, but the added artifact verifier has a blocking gap and the zod fix should be consolidated with the overlapping open PR.

## What This Changes

The PR adds zod to the tsdown always-bundle predicate and adds postpublish verifier/test coverage for the public `openclaw/plugin-sdk/zod` package artifact.

## Best Possible Solution

Land one zod inline-bundling fix with an installed-package guard that follows reachable chunks or imports the packed subpath, then close the linked issue and retire overlapping PRs.

## Reproduction Assessment

Yes. Current main still exposes `openclaw/plugin-sdk/zod` through a bare zod re-export and does not always-bundle zod, while the issue and PR body provide pnpm-global failure and after-fix packed-install evidence.

## Solution Assessment

Mostly yes. Extending the existing tsdown always-bundle predicate is the narrow maintainable fix, but this PR should strengthen the artifact verifier by following local chunks or running an installed subpath import smoke.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Follow local zod chunks before accepting the artifact:** `scripts/openclaw-npm-postpublish-verify.ts:229-233`
  - body: The verifier returns success after parsing only `dist/plugin-sdk/zod.js`. A package can pass if that file imports a package-local `../zod-*.js` chunk while the chunk still imports bare `zod`; because the root manifest declares zod, the broader root-dependency scan skips it and pnpm global imports can still fail. Walk reachable local chunks or run the installed subpath import smoke before accepting the artifact.
