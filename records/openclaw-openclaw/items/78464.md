## Summary

Keep open: the pnpm-global packaged zod resolution bug is real and current main still lacks the fix, but this draft/dirty branch has a verifier blind spot and overlaps an open narrower PR for the same runtime bundling issue.

## What This Changes

The branch adds zod subpaths to tsdown's always-bundle predicate, adds postpublish verifier/test coverage for the plugin SDK zod artifact, and updates the changelog.

## Best Possible Solution

Land one zod inline-bundling fix for the public SDK artifact, with an installed-package guard that follows reachable local chunks or performs a real installed subpath import smoke.

## Reproduction Assessment

Yes. Current main still has the bare plugin-sdk/zod source and no zod always-bundle rule, while the linked issue and PR body provide pnpm-global MODULE_NOT_FOUND and after-fix packed import evidence.

## Solution Assessment

Mostly yes. Extending the existing tsdown always-bundle predicate is the narrow maintainable code fix, but the added verifier should follow reachable local chunks or run an installed import smoke before this branch is the best landing vehicle.
