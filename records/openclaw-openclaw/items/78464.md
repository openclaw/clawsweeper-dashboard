## What This Changes

The PR adds zod to the tsdown always-bundle predicate, adds postpublish verifier/test coverage for the plugin SDK zod artifact, and updates the changelog.

## Best Possible Solution

Land one zod bundling fix that keeps the public SDK artifact self-contained and preserve an installed-package guard that follows reachable local chunks or runs an installed subpath import smoke.

## Reproduction Assessment

Yes. Current main still has the bare `plugin-sdk/zod` source and no zod always-bundle rule, while the linked issue and PR body provide pnpm-global `MODULE_NOT_FOUND` and after-fix packed import evidence.

## Solution Assessment

Mostly yes. Extending the existing tsdown always-bundle predicate is the narrow maintainable fix, but the added verifier should follow local chunks or run an installed import smoke before this branch is the best landing vehicle.
