## Summary

Keep this PR open, but it is not mergeable as reviewed: the Codex watchdog fix still lets raw assistant progress move the completion-idle timestamp, and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR changes Codex app-server watchdog activity for `rawResponseItem/completed` and also carries Ollama ambient-discovery warning suppression plus Windows plugin-skill junction changes.

## Best Possible Solution

Revise the Codex fix so raw assistant progress can update diagnostics without changing completion-idle timing, add focused regression coverage and changelog, and drop or split the unrelated already-landed work before merge.

## Reproduction Assessment

Yes, source-reproducible: after an armed turn-scoped request response, `rawResponseItem/completed` updates the same activity timestamp used by the completion watchdog, and silence then reaches the timeout/re-arm path. No live Codex app-server run was performed in this read-only sweep.

## Solution Assessment

No. The proposed option skips immediate rescheduling, but it still moves `turnCompletionLastActivityAt`, so the active timer can re-arm from the raw progress timestamp; use separate diagnostic state or skip the completion timestamp update for this notification.

## Review Findings
