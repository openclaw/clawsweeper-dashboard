The PR adds `channels.discord.implicitReplyMentions`, threads it into Discord preflight mention resolution, updates config schema/generated metadata/docs/changelog, and adds preflight coverage for user, bot, explicit-mention, and PluralKit reply cases.

## Best Possible Solution

Land the policy after fixing the docs wording and adding redacted real Discord evidence showing bot-authored native replies are suppressed with `fromBots: false` while explicit mentions still wake the agent.
