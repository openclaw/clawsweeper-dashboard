## Summary

Keep open: current main still has the Discord reply-to-bot implicit mention behavior, and this recent PR is the active, well-scoped implementation path with supplied live Discord proof and no blocking code-review finding.

## What This Changes

The PR adds `channels.discord.implicitReplyMentions` so Discord deployments can disable bot-authored native reply-only activation while preserving defaults, explicit mentions, docs, schema metadata, changelog, and regression tests.

## Best Possible Solution

Approve or adjust the Discord config API, then land the PR with green checks so it closes https://github.com/openclaw/openclaw/issues/80234.

## Reproduction Assessment

Yes, at source level. Current main passes `message.referencedMessage?.author?.id` into `resolveDiscordMentionState`, which emits `reply_to_bot` when it matches the bot id, and `allowBots="mentions"` accepts that implicit mention.

## Solution Assessment

Yes, pending maintainer API approval. The patch keeps the policy in Discord-owned preflight/config, preserves compatibility defaults and explicit textual mentions, and avoids changing the shared mention-gating contract.
