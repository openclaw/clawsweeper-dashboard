## Summary

This PR should stay open: it targets the paired Discord multi-bot reply issue, but the current diff can drop real explicit mentions in native replies when the new policy disables reply-only activation, and external real Discord proof is still absent.

## What This Changes

The branch adds a Discord `implicitReplyMentions` config object, threads it through preflight reply-to-bot mention detection, and updates schema metadata, docs, tests, and changelog.

## Best Possible Solution

Fix the explicit-mention suppression case, add regression coverage for a bot-authored native reply that also contains `<@botId>`, then land the config knob only after maintainer API review and redacted real Discord proof.

## Reproduction Assessment

Yes, source-level: current main computes `reply_to_bot` from `referencedAuthorId === botId` and `allowBots="mentions"` accepts either explicit or implicit mentions. I did not run a live Discord gateway reproduction in this read-only review.

## Solution Assessment

No for the current patch: the config-knob direction is plausible, but this implementation can suppress a real explicit mention after `baseText` normalization and still lacks real Discord behavior proof.
