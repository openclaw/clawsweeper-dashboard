## Summary

Keep this PR open but needs changes before merge: the Discord policy direction and supplied live proof are useful, but the latest head can still drop a real textual bot mention on a native reply because it checks normalized `baseText` when deciding whether a mention is reply-only.

## What This Changes

The PR adds `channels.discord.implicitReplyMentions` config, schema/docs/metadata/changelog, and Discord preflight tests to control whether user- or bot-authored native replies count as implicit mentions.

## Best Possible Solution

Preserve raw textual bot mentions before reply-only suppression, add native-reply-plus-explicit-mention regression coverage, then land only after maintainers accept the new Discord config API.

## Reproduction Assessment

Yes, source-level and supported by supplied live proof: current main treats a Discord native reply whose referenced author is the bot as an implicit `reply_to_bot` mention. I did not run a live Discord gateway reproduction in this read-only review.

## Solution Assessment

No for the current patch: the config direction is plausible, but checking normalized `baseText` can suppress a real explicit `<@botId>` mention on a native reply. The safer fix is to use raw `message.content` or an equivalent pre-normalized signal and add the missing regression test.
