## Summary

Keep this PR open: the config direction is plausible and the contributor supplied live Discord proof, but the latest head still checks normalized message text when deciding whether a native reply includes a real textual bot mention, so a bot-authored native reply with an explicit mention can be suppressed as reply-only.

## What This Changes

The PR adds `channels.discord.implicitReplyMentions` config, schema/docs/metadata, changelog, and Discord preflight tests to let deployments disable bot-authored native reply-only activation while preserving current defaults.

## Best Possible Solution

Preserve raw textual bot mentions before reply-only suppression, add native-reply-plus-explicit-mention regression coverage, then have maintainers review the new Discord config API shape.

## Reproduction Assessment

Yes, at source level and with supplied live proof for the central issue: current main treats a Discord native reply whose referenced author is the bot as an implicit `reply_to_bot` mention. The remaining PR regression is source-reproducible by combining a bot-authored native reply to the bot with content containing `<@botId>` while `fromBots:false`.

## Solution Assessment

No for the current patch: the config direction is plausible, but the suppression predicate should not use normalized `baseText` to detect raw Discord mention syntax. The safer fix is to use raw `message.content` or an equivalent pre-normalized signal and cover that edge in tests.
