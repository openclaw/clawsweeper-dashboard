## Summary

Keep this PR open with changes requested: the config direction and supplied live proof are useful, but the current patch can still suppress a real explicit Discord mention when it appears on a native reply.

## What This Changes

The PR adds `channels.discord.implicitReplyMentions` config, schema/docs/metadata, changelog, and Discord preflight tests so deployments can disable bot-authored native reply-only activation while preserving current defaults.

## Best Possible Solution

Preserve raw textual bot mentions before suppressing reply-generated mentions, add native-reply-plus-explicit-mention regression coverage, then have maintainers review the new Discord config API shape.

## Reproduction Assessment

Yes. The central issue is source-reproducible on current main and backed by supplied live Discord proof; the remaining PR regression is source-reproducible with a bot-authored native reply that also contains a real `<@botId>` mention while `fromBots:false`.

## Solution Assessment

No for the current patch. The config direction is plausible, but the suppression predicate should use raw `message.content` or an equivalent pre-normalized signal and cover that edge in tests.
