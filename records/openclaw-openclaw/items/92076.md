## Summary

Keep open: current main and the latest release still have a source-reproducible pure-text subagent completion loss path after active requester wake failure plus requester-agent lock failure, and the targeted open PR is not merged.

## What This Changes

The issue asks for top-level subagent completions to use a concise direct-message text fallback when the requester run is inactive or locked, without mirroring long child output through the requester transcript.

## Best Possible Solution

Land or replace the targeted open PR with bounded direct text fallback and regression coverage, then keep broader session-lock cleanup under the canonical lock issues if it remains separate.

## Reproduction Assessment

Yes at source level, but I did not run a live Feishu reproduction in this read-only review. Current main still throws after a SessionWriteLock handoff failure for pure-text completions with no generated media, matching the reported log sequence.

## Solution Assessment

Yes for the narrow direct-message fallback direction, provided it bounds long text and is proven in a real external-channel setup before merge. It is not enough to close broader session-lock recovery reports unless those remaining symptoms are ruled out or tracked separately.
