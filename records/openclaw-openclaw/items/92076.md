## Summary

Keep open: current main and the latest release still have the pure-text direct-message loss path after requester inactivity or SessionWriteLock handoff failure; the direct-text fix candidates are closed unmerged, and the remaining open fallback PR is broader and not a safe canonical replacement yet.

## What This Changes

The issue requests reliable capped direct-message delivery for subagent completion text when the requester run is inactive or the requester transcript path is locked.

## Best Possible Solution

Land a focused agents-path fix that sends capped, mirrored direct text for direct-message subagent completions when the requester is already inactive, active wake fails with the relevant reasons, or requester-agent handoff hits SessionWriteLock; keep orphan lock cleanup and cron route leases separate unless maintainers explicitly widen scope.

## Reproduction Assessment

Yes. Source inspection of current main and v2026.6.9 shows pure-text subagent completions still fall through to requester-agent handoff and can rethrow SessionWriteLock failures instead of using a capped direct-message text fallback.

## Solution Assessment

Yes, the issue's owner module and suggested direction are the right narrow layer, but the best implementation should cover both already-inactive requesters and active wake failures with bounded mirrored DM delivery rather than bundling unrelated lock cleanup or cron route work.
