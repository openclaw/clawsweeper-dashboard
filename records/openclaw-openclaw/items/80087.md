Keep open: current main still has a source-backed path for stale persisted plugin diagnostics to be printed by `openclaw doctor`, and I did not find a merged fix or exact canonical duplicate for this `lossless-claw` deleted-path warning.

## What This Changes

The issue asks `openclaw doctor` to stop warning about a deleted `extensions/lossless-claw` path when the active plugin is the managed npm package.

## Best Possible Solution

Make persisted plugin registry diagnostics fresh: filter or invalidate diagnostics whose source path no longer exists, especially when the plugin id now resolves to a healthy managed npm package, and cover the deleted source-dir plus npm replacement case with a regression test.
