## Summary

Keep open: current main still has a source-backed stale-presence path for the default/root WhatsApp setup because `channels.whatsapp.selfChatMode` changes are treated as hot no-ops while global presence is only sent when the socket attaches.

## What This Changes

The issue asks for WhatsApp personal-phone to separate-phone config changes to update the live presence behavior instead of leaving the existing session in personal-phone mode.

## Best Possible Solution

Add a narrow WhatsApp-owned reload path or live presence refresh for root `channels.whatsapp.selfChatMode`, with regression coverage proving personal-to-separate mode changes reapply presence without broadening unrelated root WhatsApp no-ops.

## Reproduction Assessment

Yes at source level: setup can write `channels.whatsapp.selfChatMode=false`, reload metadata classifies root WhatsApp fields as no-op, and the live socket only sends presence on attach. I did not run a live WhatsApp account repro in this read-only review.

## Solution Assessment

Yes, a narrow plugin-owned reload metadata or presence-refresh fix is the best shape; adding another config option or changing all WhatsApp root reload behavior would be broader than needed.
