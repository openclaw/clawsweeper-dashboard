## Summary

Keep open: current main still treats root WhatsApp config changes such as `selfChatMode` as reload no-ops, while the connect-time presence value is only applied when the WhatsApp listener attaches; the latest release contains the same no-op reload rule.

## What This Changes

The issue reports that WhatsApp presence stays in personal-phone behavior after switching `channels.whatsapp.selfChatMode` from personal-phone mode to separate-phone mode.

## Best Possible Solution

Add a focused reload rule and regression coverage so changes to `channels.whatsapp.selfChatMode` restart the affected WhatsApp runtime without adding a new config option or broadening unrelated root WhatsApp no-op behavior unnecessarily.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection gives a clear reproduction path: change root `channels.whatsapp.selfChatMode` while the listener is running, and the reload planner classifies that path as a no-op rather than restarting the listener that sends presence on connect.

## Solution Assessment

Yes: the best fix is a narrow reload-rule and test update for `channels.whatsapp.selfChatMode`, not a new presence setting or a broad WhatsApp reload policy change without evidence.
