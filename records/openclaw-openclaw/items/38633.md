## Summary

Keep open: current main still documents `pidsLimit: 256` as a Docker sandbox example while the resolver, schema, and Docker argv path leave PID limits unset unless explicitly configured. The issue requests either a Docker sandbox `pidsLimit` default or a docs clarification that the documented `256` value is only an explicit example.
