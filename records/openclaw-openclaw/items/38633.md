Keep open: current main still shows `pidsLimit: 256` as a Docker sandbox config example while resolver, schema, and Docker argv generation leave PID limits unset unless explicitly configured. The issue asks OpenClaw to resolve a docs mismatch where `pidsLimit: 256` appears as a sandbox Docker example even though runtime configuration currently has no default PID limit.

## Best Possible Solution

Update the public sandbox config docs to label `pidsLimit: 256` as an explicit example value and state that Docker PID limits are unset unless configured, unless an approved runtime default lands first.
