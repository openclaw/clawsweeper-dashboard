## Summary

Keep open: current `main` still shows `pidsLimit: 256` as a Docker sandbox config example while the resolver, schema, and Docker argv builder leave PID limits unset unless configured. The related docs PR closed unmerged, the runtime-default PR is still open, and the linked WASM sandbox idea is useful broader context but does not resolve this Docker docs mismatch.

## What This Changes

Clarify the sandbox Docker docs so `pidsLimit: 256` is not implied to be a default when OpenClaw currently leaves PID limits unset unless configured.

## Best Possible Solution

Update the public sandbox config docs to label `pidsLimit: 256` as an explicit example value and state the current unset-unless-configured behavior, unless a runtime default lands first.
