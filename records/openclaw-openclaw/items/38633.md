## Summary

Keep open: current main still shows `pidsLimit: 256` in the sandbox Docker config example while runtime config resolution, schema validation, and Docker create arguments keep PID limits unset unless configured.

## What This Changes

Clarify the sandbox Docker docs so `pidsLimit: 256` is not presented as a default when the runtime leaves it unset unless configured.

## Best Possible Solution

Update the sandbox docs to label the shown PID limit as an explicit example value and state that OpenClaw leaves `pidsLimit` unset unless the operator configures it.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the mismatch by comparing the docs example with `resolveSandboxDockerConfig`, optional schema validation, and conditional Docker `--pids-limit` emission.

## Solution Assessment

Yes for a docs-scoped repair. Clarifying the example is the narrowest maintainable fix here; choosing a runtime default belongs to the separate sandbox hardening discussion in https://github.com/openclaw/openclaw/issues/38604 and related resource-limit work.
