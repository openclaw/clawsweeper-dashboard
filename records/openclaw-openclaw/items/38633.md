## Summary

Keep open: current main still presents `pidsLimit: 256` as a concrete sandbox Docker example while the resolver, schema, and Docker create arguments leave it unset unless configured. The related docs PR #38914 was closed unmerged, and #38604 remains the broader runtime-default security discussion.

## What This Changes

The issue requests aligning sandbox docs with code because the docs show `pidsLimit: 256` while OpenClaw has no built-in Docker PID-limit default.

## Best Possible Solution

Update the sandbox docs to state that `pidsLimit: 256` is an explicit example value and OpenClaw leaves Docker PID limits unset unless configured, while leaving runtime-default policy to #38604.

## Reproduction Assessment

Yes. A high-confidence static reproduction is available by comparing the current docs example with `resolveSandboxDockerConfig`, the optional schema/type contract, and conditional Docker `--pids-limit` emission on current main.

## Solution Assessment

Yes for a docs-only repair: clarifying the example is the narrowest maintainable fix for this issue. Adding a runtime default would be a separate sandbox hardening decision that belongs with #38604.
