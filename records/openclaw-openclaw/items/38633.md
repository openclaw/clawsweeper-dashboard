## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/38633 with a docs-only clarification. Current main shows `pidsLimit: 256` in `docs/gateway/config-agents.md`, while `resolveSandboxDockerConfig` leaves `pidsLimit` undefined unless agent/global Docker config sets it and `buildSandboxCreateArgs` only emits `--pids-limit` for positive numbers. Update the docs to make the sample value explicit and state that no PID cap is applied unless `agents.defaults.sandbox.docker.pidsLimit` or a per-agent override is configured. Do not add a runtime default, schema change, config option, or wording that promises 256/1024 as the default; leave that policy to https://github.com/openclaw/openclaw/issues/38604 and https://github.com/openclaw/openclaw/pull/44098. Prefer `docs/gateway/config-agents.md`; add a short `docs/gateway/sandboxing.md` note only if it improves discoverability. No changelog is expected for this docs-only clarification unless a maintainer asks for one.
