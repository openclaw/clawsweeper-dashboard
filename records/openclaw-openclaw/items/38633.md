## Summary

Keep open: current main still presents `pidsLimit: 256` as a sandbox Docker config example while the resolver, schema, and Docker create-args path leave the PID limit unset unless the operator configures it.

## What This Changes

The issue asks OpenClaw to resolve a docs/runtime mismatch where docs show `pidsLimit: 256` but sandbox Docker config has no default PID limit.

## Best Possible Solution

Clarify the public sandbox config docs so `pidsLimit: 256` is explicitly an example override and document the current unset-unless-configured behavior, while leaving runtime default policy to the separate sandbox hardening thread.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the mismatch by comparing the docs example with `resolveSandboxDockerConfig`, optional schema validation, and conditional Docker `--pids-limit` emission.

## Solution Assessment

Yes for a docs-scoped repair. Clarifying the example is the narrowest maintainable fix here; adding or choosing a runtime PID cap belongs to the separate security/defaults discussion.
