## Summary

Keep open: current main still documents `pidsLimit: 256` as a Docker sandbox example while the resolver, schema, and Docker argv path leave PID limits unset unless explicitly configured. The direct docs PR was closed unmerged, and the open runtime-default PR tracks broader sandbox hardening rather than this docs mismatch.

## What This Changes

The issue requests either a Docker sandbox `pidsLimit` default or a docs clarification that the documented `256` value is only an explicit example.

## Best Possible Solution

Update the public sandbox config docs to label `pidsLimit: 256` as an explicit example and state the current unset-by-default behavior, unless an approved runtime default lands first.
