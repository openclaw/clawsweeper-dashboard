## Summary

Keep open: current main still shows `pidsLimit: 256` as a concrete sandbox config example while the resolver, schema, and Docker create args leave it unset unless users configure it. The closed docs PR #38914 addressed this mismatch but was not merged, and #38604 remains the broader runtime-default security discussion.

## What This Changes

The issue asks to align sandbox `pidsLimit` documentation with current behavior by either documenting the value as opt-in or adding a runtime default.

## Best Possible Solution

Update the sandbox docs to state that `pidsLimit: 256` is an example opt-in value and that OpenClaw leaves Docker PID limits unset unless configured; keep any runtime default decision coordinated with #38604.

## Reproduction Assessment

Yes. A high-confidence static reproduction is available by comparing the current docs example with `resolveSandboxDockerConfig`, the optional schema/type contract, and conditional Docker `--pids-limit` emission on current main.

## Solution Assessment

Yes for a docs-only repair: clarifying the example is the narrowest maintainable fix for this issue. A runtime default would be a separate sandbox hardening decision that belongs with #38604.
