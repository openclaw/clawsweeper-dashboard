## Repair Work Prompt

Fix the docs mismatch reported in https://github.com/openclaw/openclaw/issues/38633. Current main shows `pidsLimit: 256` in the `agents.defaults.sandbox.docker` example, but `resolveSandboxDockerConfig` leaves `pidsLimit` undefined unless agent/global config sets it and `buildSandboxCreateArgs` only emits `--pids-limit` for positive configured values. Make a docs-only clarification on the current English docs surface, likely `docs/gateway/config-agents.md`, that `pidsLimit: 256` is an explicit example value and PID limits are unset unless configured. Do not add a runtime `DEFAULT_SANDBOX_PIDS_LIMIT`, change sandbox policy, or settle the security-default value; that belongs to https://github.com/openclaw/openclaw/issues/38604 and https://github.com/openclaw/openclaw/pull/44098. No changelog is expected for a narrow docs-only clarification.
