Keep open: current main and the latest release still show `pidsLimit: 256` as a Docker sandbox docs example while runtime code leaves PID limits unset unless configured. The focused docs clarification PR is closed unmerged, and the runtime-default work remains open under separate security review.

The issue asks to align Docker sandbox documentation with runtime behavior by either adding a default `pidsLimit` or clarifying that the shown value is only an example.

## Best Possible Solution

Clarify the public agent configuration docs so `pidsLimit` and nearby Docker resource-limit values are marked as explicit examples unless configured, while leaving runtime default policy to https://github.com/openclaw/openclaw/issues/38604 and https://github.com/openclaw/openclaw/pull/44098.
