## What This Changes

The issue asks OpenClaw to align Docker sandbox documentation and runtime behavior for `pidsLimit`, because docs show `pidsLimit: 256` while runtime leaves the value unset unless configured.

## Best Possible Solution

Coordinate with https://github.com/openclaw/openclaw/issues/38604, https://github.com/openclaw/openclaw/pull/44098, and https://github.com/openclaw/openclaw/issues/60827: either land an approved PID-cap default with aligned schema/types/tests/docs, or clarify the docs now that `pidsLimit` is only an explicit example value unless configured.

## Reproduction Assessment

Yes. Static current-main inspection reproduces the mismatch: the docs example shows `pidsLimit: 256`, while the resolver and Docker create-args leave PID limits unset unless configured.

## Solution Assessment

Unclear until maintainers choose the docs-vs-runtime direction. A docs clarification is narrow and maintainable now, while changing omitted `pidsLimit` behavior needs security/product approval, tests, docs, and an explicit opt-out contract.
