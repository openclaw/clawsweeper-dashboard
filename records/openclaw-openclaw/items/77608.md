## Summary

Keep open: current `main` has diagnostics and optional disk-budget cleanup for some orphan artifacts, but default `openclaw sessions cleanup` still does not reconcile unindexed `.jsonl` / trajectory files after a restart-driven index desync.

## What This Changes

The issue asks for session cleanup or startup maintenance to remove or archive session files on disk that are no longer referenced by `sessions.json`.

## Best Possible Solution

Add explicit orphan artifact reconciliation to session maintenance with dry-run reporting, retention or mtime protection, writer-lock safety, and coverage for primary transcripts, trajectory sidecars, checkpoints, and archived reset/deleted artifacts.

## Reproduction Assessment

Yes from source inspection, but not by a live run: a sessions directory with unindexed `.jsonl` or trajectory files will not be touched by default cleanup unless disk-budget config is set and exceeded. The code path shows cleanup works from indexed entries first and the physical sweep returns `null` when `maxDiskBytes` is unset.

## Solution Assessment

No current solution fully handles the report. Doctor can interactively archive primary orphan transcripts and disk budget can sweep some unreferenced artifacts when configured, but neither provides the requested default startup or periodic reconciliation for restart-created orphans.
