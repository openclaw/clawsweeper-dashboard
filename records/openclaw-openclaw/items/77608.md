## Summary

Keep open: current main has partial mitigations, but normal session cleanup still does not reconcile unindexed transcript and trajectory files unless a disk budget is configured or doctor is run interactively, so the reported disk bloat remains actionable.

## What This Changes

The issue requests cleanup of session transcript and trajectory files left on disk but no longer referenced by `sessions.json` after gateway crashes or restarts.

## Best Possible Solution

Extend existing session maintenance so cleanup can dry-run/report and prune unreferenced session artifacts using current retention safeguards, without requiring users to configure a disk budget first.

## Reproduction Assessment

Yes. From source inspection, default cleanup leaves `diskBudget` null and only walks indexed store entries, so unindexed primary transcripts and trajectory sidecars are not reported or removed.

## Solution Assessment

No. The current mitigations are incomplete because doctor is manual/interactive and physical orphan cleanup is opt-in through disk-budget settings; the narrow fix belongs in the existing sessions cleanup/maintenance path.
