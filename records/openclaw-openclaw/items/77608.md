## Summary

Keep open: current main has partial mitigations, but normal session cleanup still does not reconcile unindexed transcript and trajectory files unless an optional disk budget is configured, so the reported disk bloat remains actionable.

## What This Changes

The issue requests cleanup or reconciliation for unindexed session transcript and trajectory files left behind after gateway crashes or restarts.

## Best Possible Solution

Extend session maintenance so `openclaw sessions cleanup` reports and prunes unreferenced session artifacts with existing retention safeguards, including dry-run visibility and regression coverage.

## Reproduction Assessment

Yes, from source inspection: with default maintenance config, unindexed `.jsonl` and trajectory files are only detected by doctor or removed when an optional disk budget is exceeded. I did not run a live filesystem repro because this review is read-only.

## Solution Assessment

No, current main is not the best complete solution: doctor is manual and disk-budget cleanup is opt-in. The narrow maintainable fix is to make the normal sessions cleanup path reconcile unreferenced physical artifacts safely.
