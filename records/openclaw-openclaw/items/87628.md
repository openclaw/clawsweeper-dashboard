## Summary

Keep open. The updated PR body now supplies credible live Gateway proof, but the patch still gives bare `inherit` special meaning before configured model aliases are resolved, so it can break an existing `inherit` alias in the same model fields it changes.

## What This Changes

The PR adds native `sessions_spawn` support for `model: "inherit"`, threads requester model context into sub-agent planning, updates docs, and adds focused model-selection and spawn tests.

## Best Possible Solution

Land this only after the `inherit` mode preserves the existing model-alias contract or maintainers explicitly accept and document the reserved-word break with upgrade guidance.

No. The requester-model threading is plausible and now has live proof, but the best merge shape should preserve configured aliases first or use a non-colliding inherit contract.
