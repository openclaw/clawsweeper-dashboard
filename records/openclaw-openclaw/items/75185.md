## Summary

Keep open: the PR targets a real current-main Telegram DM bug, but the patch has a blocking correctness issue because it can turn a failed non-silent delivery fallback into handled silence.

## What This Changes

The PR changes Telegram dispatch so successful no-visible DM turns stay silent, updates the focused Telegram regression test, and adds a changelog fix entry.

## Best Possible Solution

Land one canonical Telegram plugin fix that treats only clean successful no-visible turns as handled silence, preserves non-silent failure/error fallback status, keeps the regression test and changelog, then close #70628 and supersede #70630.

## Reproduction Assessment

Yes. Current main has a seam reproduction in `extensions/telegram/src/bot-message-dispatch.test.ts`; the PR regression can be reproduced by making a non-silent final skip/fail, returning `{ delivered: false }` from fallback delivery, and asserting the status remains error rather than done.

## Solution Assessment

No, not yet. The Telegram-owned approach is the right boundary, but the handled-silence guard needs to exclude prior non-silent skip/failure fallback paths before this is the safest fix.
