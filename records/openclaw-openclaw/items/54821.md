## Summary

Keep this PR open, but do not merge the current head: the requested Gateway thinking capability is still useful and not implemented on current main, while the branch is conflicting and has reasoning-visibility, payload-contract, and cleanup defects.

## What This Changes

Adds a Gateway `thinking-events` client capability, a thinking-recipient registry, cap-gated thinking event routing, and embedded reasoning delta changes with tests.

## Best Possible Solution

Land a rebased, protocol-aware implementation that preserves `canShowReasoning`, keeps core and Gateway thinking payloads raw, finalizes run-scoped recipients on all failure paths, and coordinates with the narrower no-callback and Control UI stream work.

## Reproduction Assessment

Yes for the bug portion via source inspection: current main requires `onReasoningStream` for streamed reasoning while Gateway `chat.send` does not pass that callback. The cap-gated subscription itself is a new Gateway protocol feature rather than a failing current-main behavior.

## Solution Assessment

No, not as-is. The useful direction needs a rebase plus fixes for reasoning visibility, raw payload alignment, and recipient cleanup before maintainers can evaluate the protocol change.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve the reasoning visibility gate:** `src/agents/pi-embedded-subscribe.ts:77`
  - body: Current main gates streamed reasoning on `canShowReasoning`, but this branch enables thinking broadcasts from `reasoningMode` alone. A cap-subscribed Gateway client could receive reasoning when `thinkingLevel: "off"` should hide it, so keep the visibility gate while making the callback optional.
  - confidence: 0.9
- **[P2] Keep Gateway thinking payloads raw:** `src/agents/pi-embedded-subscribe.ts:688-689`
  - body: Current main sends raw reasoning through core and formats at channel edges, but this emits `formatReasoningMessage` output as Gateway `data.text`. That reintroduces `Reasoning:` Markdown into the Gateway protocol and breaks clients expecting raw text plus raw deltas.
  - confidence: 0.88
- **[P2] Finalize preregistered recipients on dispatch failure:** `src/gateway/server-methods/chat.ts:1703`
  - body: Recipients are registered before dispatch, but if dispatch rejects before lifecycle end/error, the catch/finally path only broadcasts a chat error and deletes the abort controller. Mark the run-scoped recipient registries final on that path so stale grants do not live until TTL.
  - confidence: 0.82
