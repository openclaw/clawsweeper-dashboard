## Summary

Keep open: the requested Gateway thinking-event capability is not implemented on current main, and the PR still carries blocking protocol/security issues against current reasoning-stream behavior.

## What This Changes

The PR adds a Gateway `thinking-events` client capability, routes thinking agent events to registered recipients, revises reasoning delta handling, and updates Gateway/agent tests plus generated API baselines.

## Best Possible Solution

Refresh or replace this branch with a protocol-reviewed Gateway `thinking-events` design that preserves reasoning visibility, keeps core payloads raw, finalizes recipient grants on all early-failure paths, and includes focused Gateway/agent coverage.

## Reproduction Assessment

Yes for the bug portion: current main still requires `onReasoningStream` before `emitReasoningStream` can emit, while Gateway `chat.send` does not provide that callback. The cap-gated subscription behavior is a new protocol feature rather than a pure current-main reproduction.

## Solution Assessment

No. The capability direction may be valid, but this PR head drops the reasoning visibility gate, reintroduces formatted core thinking payloads, and leaves pre-registered recipients alive on early dispatch failures.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve reasoning visibility gates:** `src/agents/pi-embedded-subscribe.ts:77`
  - body: This enables streamed reasoning from `reasoningMode === "stream"` alone, dropping current main's `thinkingLevel !== "off"` guard. A cap-subscribed client could receive reasoning when visibility is disabled, so keep the visibility gate while decoupling event emission from the optional callback.
  - confidence: 0.9
- **[P2] Keep thinking payloads raw:** `src/agents/pi-embedded-subscribe.ts:664-689`
  - body: Current main's raw-reasoning refactor sends core thinking `text` and `delta` as raw payloads. Formatting `data.text` here would put `Reasoning:` Markdown back into the Gateway protocol and break clients that consume raw thinking deltas.
  - confidence: 0.88
- **[P2] Finalize pre-registered recipients on early failure:** `src/gateway/server-methods/chat.ts:1558`
  - body: The PR registers tool/thinking recipients before dispatch, but the rejection path only broadcasts the error and deletes the abort controller. If dispatch fails before lifecycle end/error, those run-scoped grants remain until TTL; mark both recipient registries final in the failure path.
  - confidence: 0.8
