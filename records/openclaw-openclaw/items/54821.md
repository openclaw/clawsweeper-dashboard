## Summary

Keep this PR open because current main still lacks the requested cap-gated thinking-event subscription, but the branch is not mergeable as-is after later main changes to raw reasoning and hidden-run visibility semantics.

## What This Changes

Adds a `thinking-events` gateway client capability with run-scoped recipient routing, reasoning delta changes, pre-dispatch recipient registration, and related gateway/agent tests.

## Best Possible Solution

Rebase the cap-gated thinking-event delivery onto current main, preserve raw core reasoning payloads and all visibility gates, finalize recipients on early dispatch failure, and document the new gateway capability contract.

## Reproduction Assessment

Yes. Source inspection shows current main still lacks `thinking-events`, keeps thinking emission tied to `onReasoningStream`, and routes any emitted thinking as a generic non-tool agent event; I did not run a live model session.

## Solution Assessment

No. The cap-gated direction is useful, but this branch must preserve current raw payloads, silent/thinking visibility gates, and recipient cleanup before it is the narrow maintainable fix.

  - confidence: 0.87
- **[P3] Finalize recipients after early dispatch failures:** `src/gateway/server-methods/chat.ts:1553-1558`
  - body: The new pre-dispatch registration can run before any lifecycle end/error event exists to call `markFinal`. If `dispatchInboundMessage` rejects early, the catch/finally path removes the active run but leaves tool/thinking recipient entries alive until TTL, which can accumulate and misroute if a run id is reused.
  - confidence: 0.74
