## Summary

Keep this PR open: current main still lacks the requested cap-gated thinking-event subscription path, but the branch is stale against newer raw-reasoning and visibility-gate behavior and should not merge as-is.

## What This Changes

The PR adds a gateway `thinking-events` client capability, cap-gated thinking stream routing, and reasoning delta fixes for WebSocket clients.

## Best Possible Solution

Add the `thinking-events` capability on top of current main's raw reasoning stream contract, preserving all reasoning visibility gates and covering cap-only delivery in gateway tests and protocol documentation.

## Reproduction Assessment

Yes, from source inspection: current main exposes only `tool-events` and still requires `onReasoningStream` before emitting thinking events, which gateway WebSocket runs do not provide. I did not run a live model session, so this is source-reproducible rather than runtime-reproduced.

## Solution Assessment

No. The cap-gated routing direction is right, but this branch should be rebased onto current main and preserve raw reasoning payloads plus the existing visibility gates before merge.
