## Summary

## What This Changes

The PR adds a Gateway `thinking-events` client capability, run-scoped thinking recipient routing, raw reasoning delta tracking, chat/agent registration plumbing, and related tests/generated API baseline updates.

## Best Possible Solution

Land a rebased cap-gated thinking-event implementation that preserves thinking visibility policy, defines robust raw delta versus full-snapshot handling, finalizes recipient registries on all failure paths, and updates Gateway protocol docs plus the changelog.

Yes, source inspection gives a high-confidence path: current main lacks `thinking-events` and still requires `onReasoningStream`, while the PR-head blockers reproduce from the raw delta, visibility, and dispatch-failure code paths without needing a live model.

## Solution Assessment

No. The opt-in capability design is the right direction, but this patch is not mergeable until the privacy guard, delta handling, recipient cleanup, docs, and changelog are corrected.
