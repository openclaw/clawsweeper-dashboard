## Summary

Keep open: current main still does not implement cap-gated thinking subscriptions, and this PR contains unique Gateway protocol work, but the branch is not merge-ready without maintainer protocol/security direction and rebase fixes for current-main reasoning invariants.

## What This Changes

This PR adds a Gateway `thinking-events` client capability, thinking-recipient routing, reasoning-stream delta changes, recipient pre-registration, and related Gateway/agent tests plus generated baseline updates.

## Best Possible Solution

Decide whether cap-gated thinking subscriptions belong in the Gateway protocol; if yes, rebase this work over current main while preserving visibility gates, raw payloads, and recipient cleanup, otherwise land the narrower callback fix in https://github.com/openclaw/openclaw/pull/79687.

## Reproduction Assessment

Yes for the bug portion: current-main source requires `onReasoningStream`, while Gateway `chat.send` does not supply that callback. The cap-gated subscription behavior is a new protocol feature rather than a current-main bug reproduction.

## Solution Assessment

No. The capability direction may be valid, but this PR head drops the visibility gate, sends formatted thinking payloads after the raw-payload refactor, and leaves pre-registered recipient grants alive on early dispatch failures.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve the reasoning visibility gate:** `src/agents/pi-embedded-subscribe.ts:77`
  - body: Current main gates streaming on `canShowReasoning`, but this branch sets `streamReasoning` from `reasoningMode` alone. A cap-subscribed Gateway client could receive thinking when `thinkingLevel: "off"` should hide it, so keep the visibility gate while making the callback optional.
  - confidence: 0.9
- **[P2] Keep Gateway thinking payloads raw:** `src/agents/pi-embedded-subscribe.ts:688-689`
  - body: Current main's raw-reasoning refactor sends raw thinking through core and formats at channel edges, but this branch emits `formatReasoningMessage` as Gateway `data.text`. That reintroduces `Reasoning:` Markdown into the Gateway protocol and breaks clients expecting raw text plus raw deltas.
  - confidence: 0.88
- **[P2] Finalize preregistered recipients on dispatch failure:** `src/gateway/server-methods/chat.ts:1558`
  - body: Recipients are registered before dispatch, but if dispatch rejects before a lifecycle end/error event, the catch/finally path only broadcasts a chat error and deletes the abort controller. Mark the run-scoped recipient registries final on that path so stale grants do not live until TTL.
  - confidence: 0.82
