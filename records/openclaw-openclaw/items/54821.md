## Summary

Keep open: current main still lacks the cap-gated `thinking-events` subscription surface, but this PR is not safe to merge as-is because it changes reasoning-token delivery and still has blocking visibility, payload-contract, and cleanup defects.

## What This Changes

The PR adds a `thinking-events` Gateway client capability, routes thinking agent events to cap-registered WebSocket clients, and changes reasoning stream delta tracking.

## Best Possible Solution

Keep the raw reasoning stream contract, land a narrow no-callback fix that preserves `canShowReasoning`, and handle cap-gated thinking subscriptions as an explicit Gateway protocol change with cleanup coverage.

## Reproduction Assessment

Yes for the bug portion: current main requires `onReasoningStream` for `streamReasoning`, while Gateway `chat.send` does not pass that callback. The cap-gated subscription itself is a new Gateway protocol feature rather than a current-main bug reproduction.

## Solution Assessment

No, not as-is. The narrower fix in https://github.com/openclaw/openclaw/pull/79687 covers the no-callback bug more safely, while this broader PR still needs visibility gating, raw payload alignment, recipient cleanup, and maintainer protocol review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve the reasoning visibility gate:** `src/agents/pi-embedded-subscribe.ts:77`
  - body: Current main gates streamed reasoning on `canShowReasoning`, but this branch enables `streamReasoning` from `reasoningMode` alone. A cap-subscribed Gateway client could receive thinking when `thinkingLevel: "off"` should hide it, so keep the visibility gate while making the callback optional.
  - confidence: 0.9
- **[P2] Keep Gateway thinking payloads raw:** `src/agents/pi-embedded-subscribe.ts:688-689`
  - body: Current main sends raw thinking through core and formats at channel edges, but this branch emits `formatReasoningMessage` as Gateway `data.text`. That reintroduces `Reasoning:` Markdown into the Gateway protocol and breaks clients expecting raw text plus raw deltas.
  - confidence: 0.88
- **[P2] Finalize preregistered recipients on dispatch failure:** `src/gateway/server-methods/chat.ts:1553-1558`
  - body: Recipients are registered before dispatch, but if dispatch rejects before a lifecycle end/error event, the catch/finally path only broadcasts a chat error and deletes the abort controller. Mark the run-scoped recipient registries final on that path so stale grants do not live until TTL.
  - confidence: 0.82
