## Summary

Keep open: this PR addresses a real Gateway thinking-events gap, but the current head is not mergeable because it bypasses current reasoning visibility gates, conflicts with the raw reasoning payload contract from https://github.com/openclaw/openclaw/pull/78397, and leaves pre-registered event recipients alive until TTL on early dispatch failure.

## What This Changes

This PR adds a Gateway `thinking-events` client capability, cap-gated routing for thinking agent events, reasoning delta changes, gateway registration plumbing, tests, and generated API baseline updates.

## Best Possible Solution

Refresh or replace this with a minimal Gateway capability design that decouples thinking events from `onReasoningStream` while preserving `thinkingLevel`, raw core payloads, cap-gated delivery, and guaranteed recipient cleanup.

## Reproduction Assessment

Yes for the bug portion by source inspection: current main still requires `onReasoningStream` before emitting `stream:"thinking"`, while Gateway `chat.send` does not provide that callback. The cap-gated subscription behavior is new protocol surface rather than an existing-behavior reproduction.

## Solution Assessment

No. The direction may be useful, but this PR head is not the best merge path until it preserves visibility gates, keeps core thinking payloads raw, finalizes recipient grants on early failure, and gets maintainer agreement on the capability contract.

## Review Findings

Overall correctness: patch is incorrect

- **[P1] Preserve reasoning visibility gates:** `src/agents/pi-embedded-subscribe.ts:77`
  - body: This sets streamed reasoning from `reasoningMode === "stream"` alone, dropping current main's `thinkingLevel` visibility guard. A client with the new cap could receive reasoning even when reasoning visibility is disabled, so keep the existing gate while decoupling the callback.
  - confidence: 0.9

  - confidence: 0.88
- **[P2] Finalize recipients on early dispatch failure:** `src/gateway/server-methods/chat.ts:1558`
  - body: The PR pre-registers tool/thinking recipients before dispatch, but the rejection path only broadcasts the error and deletes the abort controller. If dispatch fails before lifecycle end/error, those run-scoped grants remain until TTL; mark both recipient registries final in the catch/finally path.
  - confidence: 0.78
