## Summary

Keep open. The no-callback Gateway thinking-stream bug is still source-reproducible on current main, but this PR is a broader protocol/security change and the current branch still has blocking review issues around reasoning visibility, raw payload shape, and recipient cleanup.

## What This Changes

The PR adds a Gateway `thinking-events` client capability, cap-scoped thinking-event recipient routing, and reasoning stream delta changes across agent and Gateway code/tests.

## Best Possible Solution

Land the narrow callback-decoupling fix while preserving `canShowReasoning` and raw reasoning payloads; add a documented cap-gated protocol only if maintainers approve that surface and the cleanup/routing tests are complete.

## Reproduction Assessment

Yes for the bug portion by source inspection: current main requires `onReasoningStream`, while Gateway `chat.send` does not provide it. The cap-gated subscription behavior is a new protocol feature rather than a current-main bug reproduction.

## Solution Assessment

No, not as-is. The narrower open PR https://github.com/openclaw/openclaw/pull/79687 covers the callback bug more safely, while this branch still needs maintainer approval and fixes for visibility, raw payloads, and recipient cleanup.

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Preserve the reasoning visibility gate:** `src/agents/pi-embedded-subscribe.ts:77`
  - body: Current main gates streamed reasoning on `canShowReasoning`, but this branch enables `streamReasoning` from `reasoningMode` alone. A cap-subscribed Gateway client could receive thinking when `thinkingLevel: "off"` should hide it, so keep the visibility gate while making the callback optional.
  - confidence: 0.9
- **[P2] Keep Gateway thinking payloads raw:** `src/agents/pi-embedded-subscribe.ts:688-689`
  - body: Current main sends raw thinking through core and formats at channel edges, but this branch emits formatted `Reasoning:` Markdown as Gateway `data.text`. That breaks the raw Gateway stream contract established by the merged raw-reasoning refactor.
  - confidence: 0.88
- **[P2] Finalize preregistered recipients on dispatch failure:** `src/gateway/server-methods/chat.ts:1558`
  - body: Recipients are registered before dispatch, but if dispatch rejects before a lifecycle end/error event, the catch/finally path only broadcasts a chat error and deletes the abort controller. Mark the run-scoped recipient registries final on that path so stale grants do not live until TTL.
  - confidence: 0.82
