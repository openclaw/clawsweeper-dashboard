## Summary

Keep open. The no-callback reasoning-stream bug is source-reproducible on current main, but this PR head is conflicting and still has blocking reasoning visibility, raw-payload, and recipient-cleanup defects; the new capability subscription policy also needs maintainer review alongside active related work.

## What This Changes

The PR adds a Gateway `thinking-events` client capability, routes thinking agent events to cap-registered WebSocket connections, and changes embedded Pi reasoning-stream delta handling.

## Best Possible Solution

Land a coordinated, rebased Gateway reasoning-stream implementation that preserves `canShowReasoning`, keeps Gateway thinking payloads raw, adds opt-in subscription only if maintainers want that protocol boundary, and finalizes recipient grants on every dispatch exit.

## Reproduction Assessment

Yes for the bug portion via source inspection: current main requires `onReasoningStream`, while Gateway `chat.send` does not pass that callback. The `thinking-events` capability itself is a new protocol feature, not a failing current-main behavior.

## Solution Assessment

No, not as-is. The direction may be useful, but the current patch needs a rebase plus fixes for reasoning visibility, raw payload alignment, and recipient cleanup before maintainers can evaluate the protocol change.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve the reasoning visibility gate:** `src/agents/pi-embedded-subscribe.ts:77`
  - body: Current main gates streamed reasoning with `canShowReasoning`, but this branch sets `streamReasoning` from `reasoningMode` alone. A client with `thinking-events` could receive reasoning when `thinkingLevel: "off"` should suppress it, so keep `canShowReasoning` while making the callback optional.
  - confidence: 0.9
- **[P2] Keep Gateway thinking payloads raw:** `src/agents/pi-embedded-subscribe.ts:688`
  - body: Current main moved reasoning formatting to channel edges; this branch sends formatted output as Gateway `data.text`. That puts `Reasoning:`/Markdown back into the Gateway protocol and breaks clients that apply raw `text` and `delta` increments.
  - confidence: 0.88
- **[P2] Finalize preregistered recipients on dispatch failure:** `src/gateway/server-methods/chat.ts:1703`
  - body: The branch registers tool/thinking recipients before dispatch, but if dispatch rejects before lifecycle end/error, this `finally` block only deletes the abort controller. Mark the run recipient registries final on that path so stale run-to-connection grants do not survive until TTL.
  - confidence: 0.82
