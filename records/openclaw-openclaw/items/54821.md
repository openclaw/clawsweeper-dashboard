## Summary

Keep open: the no-callback reasoning-stream bug is source-reproducible and the capability idea is not on current `main`, but this PR is conflicting and still has security/compatibility defects around reasoning visibility, raw Gateway payloads, and stale recipient cleanup.

## What This Changes

The PR adds a Gateway `thinking-events` capability, routes thinking agent events to subscribed WebSocket connections, changes embedded reasoning-stream delta state, and updates Gateway tests plus generated SDK baselines.

## Best Possible Solution

Land a coordinated, rebased Gateway reasoning-stream implementation that preserves `thinkingLevel` visibility, keeps Gateway thinking payloads raw, and adds opt-in subscription only if maintainers want that protocol boundary.

## Reproduction Assessment

Yes for the bug portion by source inspection: current `main` requires `onReasoningStream`, while Gateway `chat.send` does not provide that callback. The `thinking-events` capability itself is a new protocol feature rather than a failing current-main behavior.

## Solution Assessment

No, not as-is. The direction may be useful, but the current head needs a rebase plus fixes for reasoning visibility, raw payload alignment, and recipient cleanup before maintainers can judge the protocol change.

- **[P1] Preserve the reasoning visibility gate:** `src/agents/pi-embedded-subscribe.ts:77`
  - body: Current main computes `streamReasoning` with `canShowReasoning`, but this branch makes it depend on `reasoningMode === "stream"` alone.   - confidence: 0.91
- **[P2] Keep Gateway thinking payloads raw:** `src/agents/pi-embedded-subscribe.ts:688`
  - body: Current main moved reasoning formatting to channel edges, but this branch sends `formatReasoningMessage(...)` as Gateway `data.text`. That reintroduces `Reasoning:`/Markdown into the Gateway protocol and breaks clients that apply raw `text` and `delta` increments.
  - confidence: 0.88
- **[P2] Finalize preregistered recipients on dispatch failure:** `src/gateway/server-methods/chat.ts:1703`
  - body: The branch registers tool/thinking recipients before dispatch, but if dispatch rejects before lifecycle end/error, this `finally` block only deletes the abort controller. Mark the run recipient registries final on that path so stale run-to-connection grants do not survive until TTL.
  - confidence: 0.82
