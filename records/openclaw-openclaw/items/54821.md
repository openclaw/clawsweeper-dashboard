## Summary

Keep open: current main still lacks the cap-gated `thinking-events` protocol surface, but this PR head is not mergeable as written because it regresses current reasoning visibility and raw-payload contracts and leaves stale recipient grants on early dispatch failure.

## What This Changes

The PR adds a Gateway `thinking-events` client capability, cap-scoped thinking event routing, revised reasoning delta handling, and related gateway/agent tests plus generated API baseline updates.

## Best Possible Solution

Refresh or replace this branch with a cap-gated Gateway design that preserves `canShowReasoning`, keeps core thinking payloads raw, finalizes recipient grants on every failure path, and documents/tests the protocol contract.

## Reproduction Assessment

Yes for the bug portion by source inspection: current main still requires `onReasoningStream`, while Gateway `chat.send` does not pass that callback. The cap-gated subscription behavior is a new protocol feature rather than a pure current-main bug reproduction.

## Solution Assessment

No. The capability direction is plausible, but this PR head is stale against current raw-reasoning and reasoning-visibility contracts and should be refreshed or split before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve reasoning visibility gates:** `src/agents/pi-embedded-subscribe.ts:77`
  - body: This enables streamed reasoning from `reasoningMode === "stream"` alone, dropping current main's `canShowReasoning` / `thinkingLevel !== "off"` gate. A cap-subscribed client could receive reasoning when reasoning visibility is disabled, so keep the visibility gate while decoupling event emission from the optional callback.
  - confidence: 0.9
- **[P2] Keep core thinking payloads raw:** `src/agents/pi-embedded-subscribe.ts:664-689`
  - body: Current main's raw-reasoning refactor sends core reasoning `text` and `delta` as raw payloads and pushes formatting to channel edges. Calling `formatReasoningMessage` here would put `Reasoning:` Markdown back into the Gateway protocol and break clients that consume raw thinking deltas.
  - confidence: 0.88
- **[P2] Finalize pre-registered recipients on dispatch failure:** `src/gateway/server-methods/chat.ts:1558`
  - body: This pre-registers tool/thinking recipients before dispatch, but the rejection path only broadcasts the error and deletes the abort controller. If dispatch fails before lifecycle end/error, those run-scoped grants remain until TTL; mark both recipient registries final in the catch/finally path.
  - confidence: 0.8
