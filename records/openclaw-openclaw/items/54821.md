## Summary

Keep this PR open, but it is not merge-ready: the branch still addresses a real Gateway reasoning-stream gap, while the current head conflicts with main and introduces security/contract regressions in reasoning visibility, raw Gateway payloads, and recipient cleanup.

## What This Changes

The PR adds a Gateway `thinking-events` client capability, routes thinking agent events to subscribed WebSocket connections, and changes embedded Pi reasoning-stream delta handling.

## Best Possible Solution

Use a rebased Gateway reasoning-stream implementation that preserves visibility gates, keeps Gateway `text`/`delta` raw, finalizes recipient grants on all dispatch exits, and introduces the opt-in capability only with explicit maintainer agreement.

## Reproduction Assessment

Yes for the bug portion: current main requires `onReasoningStream`, while Gateway `chat.send` does not pass that callback. The `thinking-events` capability itself is a new protocol feature rather than a failing current-main behavior.

## Solution Assessment

No, not as-is. The direction is plausible, but the branch needs a rebase plus fixes for visibility gating, raw payload alignment, and recipient cleanup before maintainers decide whether the new cap is the right protocol boundary.
