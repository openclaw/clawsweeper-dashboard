## Summary

Keep open for maintainer review: the bug portion is real and the contributor’s screenshots are adequate proof, but this branch is not correct against current main because it bypasses reasoning visibility, sends formatted thinking after the raw-payload refactor, and leaves pre-registered recipient grants alive on early failures.

## What This Changes

This PR adds a Gateway `thinking-events` client capability, cap-gated reasoning event routing, reasoning delta changes, and related Gateway/agent tests and generated API baseline updates.

## Best Possible Solution

Refresh or replace this with a protocol-reviewed Gateway design that preserves `canShowReasoning`, keeps core thinking payloads raw, finalizes recipient grants on every failure path, and aligns with the narrower no-callback fix if maintainers do not want a new capability.

## Reproduction Assessment

Yes for the bug portion from source inspection: current main still requires `onReasoningStream` before `emitReasoningStream` can emit, while Gateway `chat.send` does not provide that callback. The cap-gated subscription behavior is a new protocol feature, not a current-main bug reproduction.

## Solution Assessment

No. The capability direction may be valid, but this PR head drops the reasoning visibility gate, sends formatted thinking payloads after the raw-payload refactor, and leaves pre-registered recipient grants alive on early dispatch failures.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve the reasoning visibility gate:** `src/agents/pi-embedded-subscribe.ts:77`
  - body: This changes `streamReasoning` to depend only on `reasoningMode === "stream"`, and the branch also lacks current main’s `canShowReasoning` guard for included reasoning. A cap-subscribed client could receive reasoning when visibility is disabled, so keep the `thinkingLevel !== "off"` gate while making the callback optional.
  - confidence: 0.9
- **[P2] Keep Gateway thinking payloads raw:** `src/agents/pi-embedded-subscribe.ts:688-689`
  - body: Current main moved reasoning formatting to channel edges, but this branch sends `Reasoning:` Markdown as `data.text` in the Gateway event. That reintroduces presentation formatting into the core protocol and breaks clients expecting raw thinking text.
  - confidence: 0.88
- **[P2] Finalize pre-registered thinking recipients on failure:** `src/gateway/server-methods/chat.ts:1558`
  - body: Thinking recipients are registered before dispatch, but if dispatch rejects before a lifecycle end/error event, the catch/finally path only reports the chat error and deletes the abort controller. Mark the recipient registry final on this early-failure path so stale grants do not remain until TTL.
  - confidence: 0.82
