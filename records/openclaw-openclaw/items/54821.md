## Summary

Keep open for maintainer review: the no-callback Gateway thinking-stream bug is still source-reproducible and the `thinking-events` capability is not on current main, but this PR head is not mergeable as-is because it drops the reasoning visibility gate, formats Gateway thinking payloads against the current raw-stream contract, and leaves preregistered recipient grants alive on early dispatch failure.

## What This Changes

The PR adds a Gateway `thinking-events` client capability, routes thinking agent events to cap-registered connections, adjusts reasoning delta tracking, and updates related Gateway/agent tests and generated API baselines.

## Best Possible Solution

Either land the narrower no-callback bug fix in https://github.com/openclaw/openclaw/pull/79687, or rebase this PR and preserve `canShowReasoning`, raw Gateway thinking payloads, and recipient cleanup if maintainers approve the new cap-gated protocol surface.

## Reproduction Assessment

Yes for the bug portion: current main requires `onReasoningStream`, while Gateway `chat.send` does not supply that callback. The cap-gated subscription behavior is a new protocol feature rather than a current-main bug reproduction.

## Solution Assessment

No. The capability direction may be valid, but this PR head needs maintainer approval and fixes for visibility gating, raw payloads, and recipient cleanup before it is the best path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve the reasoning visibility gate:** `src/agents/pi-embedded-subscribe.ts:77`
  - body: Current main gates streamed reasoning on `canShowReasoning`, but this branch enables `streamReasoning` from `reasoningMode` alone. A cap-subscribed Gateway client could receive thinking when `thinkingLevel: "off"` should hide it, so keep the visibility gate while making the callback optional.
  - confidence: 0.9
- **[P2] Keep Gateway thinking payloads raw:** `src/agents/pi-embedded-subscribe.ts:688-689`
  - body: Current main sends raw thinking through core and formats at channel edges, but this branch emits `formatReasoningMessage` as Gateway `data.text`. That reintroduces `Reasoning:` Markdown into the Gateway protocol and breaks clients expecting raw text plus raw deltas.
  - confidence: 0.88
- **[P2] Finalize preregistered recipients on dispatch failure:** `src/gateway/server-methods/chat.ts:1558`
  - body: Recipients are registered before dispatch, but if dispatch rejects before a lifecycle end/error event, the catch/finally path only broadcasts a chat error and deletes the abort controller. Mark the run-scoped recipient registries final on that path so stale grants do not live until TTL.
  - confidence: 0.82
