## Summary

Keep open: current main still lacks the requested `thinking-events` Gateway capability, but this PR is not mergeable as-is because the head is conflicting and still has blocking reasoning-token visibility and protocol-contract issues.

## What This Changes

The branch adds a `thinking-events` Gateway client capability, routes thinking agent events to cap-registered WebSocket recipients, changes reasoning delta tracking, and updates related gateway/agent tests plus generated SDK baselines.

## Best Possible Solution

Keep the raw reasoning stream contract, land the narrow no-callback fix that preserves `canShowReasoning`, and treat cap-gated thinking subscriptions as a separate Gateway protocol change with security cleanup and coverage.

## Reproduction Assessment

Yes for the bug portion by source inspection: current main gates `streamReasoning` on `onReasoningStream`, while Gateway `chat.send` does not pass that callback. The cap-gated `thinking-events` subscription itself is a new protocol feature rather than a current-main bug reproduction.

## Solution Assessment

No, not as-is. The branch combines a useful no-callback fix with a new protocol capability and currently drops the visibility gate, sends formatted Gateway payloads, and leaves pre-registered recipients stale on early dispatch failure.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve the reasoning visibility gate:** `src/agents/pi-embedded-subscribe.ts:77`
  - body: Current main gates streamed reasoning on `canShowReasoning`, but this branch enables `streamReasoning` from `reasoningMode` alone. A cap-subscribed Gateway client could receive thinking when `thinkingLevel: "off"` should hide it, so keep the visibility gate while making the callback optional.
  - confidence: 0.9
- **[P2] Keep Gateway thinking payloads raw:** `src/agents/pi-embedded-subscribe.ts:688-689`
  - body: Current main sends raw thinking through core and formats at channel edges, but this branch emits `formatReasoningMessage` as Gateway `data.text`. That reintroduces `Reasoning:` Markdown into the Gateway protocol and breaks clients expecting raw text plus raw deltas.
  - confidence: 0.88
- **[P2] Finalize preregistered recipients on dispatch failure:** `src/gateway/server-methods/chat.ts:1558`
  - body: Recipients are registered before dispatch, but if dispatch rejects before a lifecycle end/error event, the catch/finally path only broadcasts a chat error and deletes the abort controller. Mark the run-scoped recipient registries final on that path so stale grants do not live until TTL.
  - confidence: 0.82
