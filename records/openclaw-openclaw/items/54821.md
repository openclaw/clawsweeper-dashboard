## Summary

Keep open. Current main does not implement the `thinking-events` capability, so this PR is not obsolete, but the PR head is stale against main and still has blocking reasoning visibility, delta handling, and recipient cleanup issues.

## What This Changes

The PR adds a Gateway `thinking-events` client capability, routes thinking agent events to capability subscribers, decouples reasoning emission from `onReasoningStream`, adjusts raw reasoning delta tracking, and updates related gateway tests/API baselines.

## Best Possible Solution

Land a rebased version of the cap-gated thinking-event design that preserves current reasoning visibility guards, separates full-snapshot and delta-only reasoning inputs, finalizes run-recipient registries on every failure path, and documents/changelogs the Gateway capability.

## Reproduction Assessment

Yes. Current main source gives a high-confidence reproduction path: a Gateway WebSocket client cannot advertise `thinking-events`, and reasoning emission still returns unless `onReasoningStream` is present. The PR-head regressions are also source-reproducible with a prior raw reasoning value followed by a delta-only chunk and with dispatch rejection after pre-registration.

## Solution Assessment

No. The capability-gated direction is the right shape, but this PR is not the best mergeable solution until it is rebased and the visibility, delta/full-snapshot, recipient cleanup, docs, and changelog gaps are fixed.
