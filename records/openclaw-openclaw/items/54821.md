## Summary

This PR should stay open for maintainer handling: it addresses a real source-visible Gateway reasoning-stream gap, but the current head is conflicting, overlaps a narrower open fix, and still has correctness/security blockers in reasoning visibility, payload format, and recipient cleanup.

## What This Changes

The branch adds a Gateway `thinking-events` client capability, routes thinking agent events to registered WebSocket clients, and rewrites embedded reasoning delta tracking.

## Best Possible Solution

Land a rebased, protocol-aware implementation that preserves `canShowReasoning`, keeps Gateway thinking payloads raw, finalizes recipient grants on failure paths, and coordinates the narrow no-callback fix with https://github.com/openclaw/openclaw/pull/79687.

## Reproduction Assessment

Yes for the bug portion: current main requires `onReasoningStream`, while Gateway `chat.send` does not provide that callback. The cap-gated subscription is a new Gateway protocol feature rather than a current-main bug reproduction.

## Solution Assessment

No, not as-is. The narrow callback-decoupling bug is better handled by https://github.com/openclaw/openclaw/pull/79687, while this broader protocol branch needs rebase, visibility gating, raw payload alignment, and recipient cleanup before it is the maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve the reasoning visibility gate:** `src/agents/pi-embedded-subscribe.ts:77`
  - body: Current main gates streamed reasoning with `canShowReasoning`, so `thinkingLevel: "off"` suppresses thinking. This branch sets `streamReasoning` from `reasoningMode` alone, so a cap-subscribed Gateway client could receive reasoning tokens that should remain hidden.
  - confidence: 0.9
- **[P2] Keep Gateway thinking payloads raw:** `src/agents/pi-embedded-subscribe.ts:688-689`
  - body: Current main keeps core reasoning stream payloads raw and leaves formatting to channel edges. This branch emits `formatReasoningMessage(...)` as Gateway `data.text`, reintroducing `Reasoning:`/Markdown into a protocol payload whose `text` and `delta` clients may append as raw text.
  - confidence: 0.88
- **[P2] Finalize pre-registered recipients on dispatch failure:** `src/gateway/server-methods/chat.ts:1535-1536`
  - body: The branch registers tool/thinking recipients before `dispatchInboundMessage`, but an early rejection before lifecycle end/error leaves those run-scoped grants alive until TTL because the catch/finally path only sends a chat error and removes the run. Mark both registries final on that failure path.
  - confidence: 0.82
