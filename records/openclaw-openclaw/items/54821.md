## Summary

Keep open: the useful bug report is real, but this PR is now stale against current main and still has correctness/security blockers around reasoning visibility, raw Gateway payloads, and stale recipient grants. The narrower callback-only bug is separately covered by https://github.com/openclaw/openclaw/pull/79687, while the new `thinking-events` protocol capability still needs maintainer product/security direction.

## What This Changes

The branch adds a Gateway `thinking-events` client capability, cap-gated thinking event recipients, reasoning stream delta changes, and related Gateway/agent tests plus generated API baseline updates.

## Best Possible Solution

Refresh or replace this branch with a protocol-reviewed `thinking-events` design that preserves reasoning visibility, keeps Gateway thinking payloads raw, and finalizes recipient grants on every failure path; if the capability is not desired, land the narrower callback fix in https://github.com/openclaw/openclaw/pull/79687.

## Reproduction Assessment

Yes for the bug portion: current main still requires `onReasoningStream` before `emitReasoningStream` can emit, while Gateway `chat.send` does not provide that callback. The cap-gated subscription behavior is a new protocol feature, so it is not a current-main bug reproduction.

## Solution Assessment

No. The capability direction may be valid, but this PR head drops the reasoning visibility gate, sends formatted core thinking payloads after the raw-payload refactor, and leaves pre-registered recipient grants alive on early dispatch failures.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve the reasoning visibility gate:** `src/agents/pi-embedded-subscribe.ts:77`
  - body: This changes `streamReasoning` to depend only on `reasoningMode === "stream"`, dropping current main's `thinkingLevel !== "off"` guard. A cap-subscribed client could receive reasoning when visibility is disabled, so keep the visibility gate while making the callback optional.
  - confidence: 0.9
- **[P2] Keep Gateway thinking payloads raw:** `src/agents/pi-embedded-subscribe.ts:688-689`
  - body: Current main's raw-reasoning refactor moved formatting to channel edges, but this branch sends `Reasoning:` Markdown as `data.text` in the Gateway event. That breaks clients expecting raw thinking text and reintroduces formatting into the core protocol.
  - confidence: 0.88
- **[P2] Finalize pre-registered thinking recipients on failure:** `src/gateway/server-methods/chat.ts:1558`
  - body: Thinking recipients are registered before dispatch, but if dispatch rejects before a lifecycle end/error event, the catch/finally path only reports the chat error and deletes the abort controller. Mark the recipient registry final on this early-failure path so stale grants do not remain until TTL.
  - confidence: 0.82
