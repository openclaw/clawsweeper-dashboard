## Summary

Keep open. The PR still carries unique cap-gated Gateway thinking-event work, but the current head is not safe to merge because it drops the reasoning visibility gate, conflicts with the raw reasoning contract now on main, and leaves preregistered recipients stale on early dispatch failures.

## What This Changes

The PR adds a `thinking-events` Gateway client capability, routes thinking agent events to cap-registered WebSocket clients, adjusts embedded reasoning delta tracking, and threads a thinking recipient registry through Gateway runtime code.

## Best Possible Solution

Land a rebased protocol-aware implementation that preserves `canShowReasoning`, keeps Gateway thinking payloads raw, finalizes recipient grants on all failure paths, and lets the narrower no-callback fix proceed separately.

## Reproduction Assessment

Yes for the bug portion: current main requires `onReasoningStream` for `streamReasoning`, while Gateway `chat.send` does not pass that callback. The cap-gated subscription itself is a new Gateway protocol feature rather than a current-main bug reproduction.

## Solution Assessment

No, not as-is. The no-callback bug is better handled by https://github.com/openclaw/openclaw/pull/79687, while this PR's broader cap-gated protocol work needs rebase, visibility gating, raw payload alignment, and recipient cleanup.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve the reasoning visibility gate:** `src/agents/pi-embedded-subscribe.ts:77`
  - body: Current main gates streamed reasoning on `canShowReasoning`, but this branch enables `streamReasoning` from `reasoningMode` alone. A cap-subscribed Gateway client could receive thinking when `thinkingLevel: "off"` should hide it, so keep the visibility gate while making the callback optional.
  - confidence: 0.9
- **[P2] Keep Gateway thinking payloads raw:** `src/agents/pi-embedded-subscribe.ts:688-689`
  - body: Current main sends raw thinking through core and formats at channel edges, but this branch emits `formatReasoningMessage` as Gateway `data.text`. That reintroduces `Reasoning:` Markdown into the Gateway protocol and breaks clients expecting raw text plus raw deltas.
  - confidence: 0.88
- **[P2] Finalize preregistered recipients on dispatch failure:** `src/gateway/server-methods/chat.ts:1703`
  - body: Recipients are registered before dispatch, but if dispatch rejects before a lifecycle end/error event, the catch/finally path only broadcasts a chat error and deletes the abort controller. Mark the run-scoped recipient registries final on that path so stale grants do not live until TTL.
  - confidence: 0.82
