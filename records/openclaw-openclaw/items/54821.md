## Summary

Keep open: this is still the canonical open PR for cap-gated Gateway thinking events, but the current head is not merge-ready because it is stale against main's reasoning visibility and raw-payload direction. The raw delta portion was merged separately, and the narrower callback-gate bug is now covered by a separate open PR.

## What This Changes

The PR adds a Gateway `thinking-events` capability, routes thinking stream events to run-scoped subscribed connections, and changes agent reasoning-stream delta handling.

## Best Possible Solution

Keep the cap-gated thinking-event design open for maintainer protocol review while landing the narrow callback fix separately if that is the desired immediate bug fix.

## Reproduction Assessment

Yes for the bug portion: current main requires `onReasoningStream` before emitting `stream:"thinking"`, and Gateway `chat.send` does not provide that callback. The cap-gated subscription behavior itself is a new protocol feature rather than a current-main bug reproduction.

## Solution Assessment

No. The broad cap-gated direction may be useful, but this PR head is not the best merge path until it preserves current visibility gates, keeps core thinking payloads raw, finalizes recipient grants on all failure paths, and has maintainer agreement on the Gateway capability contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve reasoning visibility gates:** `src/agents/pi-embedded-subscribe.ts:77`
  - body: This sets streamed reasoning from `reasoningMode === "stream"` alone, dropping current main's `canShowReasoning` guard. A client with the new cap could receive reasoning even when `thinkingLevel` is off, so the branch needs to keep the visibility gate when decoupling the callback.
  - confidence: 0.9
- **[P2] Keep core thinking payloads raw:** `src/agents/pi-embedded-subscribe.ts:664-689`
  - body: Current main now treats core reasoning `text` and `delta` as raw payloads and leaves Markdown formatting to channel edges. Reintroducing `formatReasoningMessage(...)` here would send formatted `Reasoning:` Markdown through the Gateway protocol and conflict with that merged contract.
  - confidence: 0.88
- **[P2] Finalize recipients after early dispatch failures:** `src/gateway/server-methods/chat.ts:1558`
  - body: `chat.send` pre-registers thinking recipients before dispatch, but the catch/finally path only deletes the abort controller. If dispatch rejects before lifecycle events run, the run-to-connection grant survives until TTL and can accumulate or misroute if a run id is reused.
  - confidence: 0.76
