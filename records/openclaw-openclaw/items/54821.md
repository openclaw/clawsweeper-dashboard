## Summary

Keep open. The PR targets a real Gateway reasoning-stream gap, but the current branch is not mergeable as-is because it conflicts with recent raw-reasoning and Gateway fanout work and introduces concrete reasoning-token visibility and cleanup regressions.

## What This Changes

The PR adds a Gateway `thinking-events` client capability, routes thinking agent events to capability subscribers, changes reasoning stream delta handling, and updates related tests and generated API baselines.

## Best Possible Solution

Refresh or replace this with a narrow Gateway `thinking-events` capability implementation that preserves raw reasoning payloads, reasoning visibility gates, recipient cleanup, and protocol docs/tests.

## Reproduction Assessment

Yes for the bug portion by source inspection: current main still requires `onReasoningStream` before emitting `stream: "thinking"`, while Gateway `chat.send` does not provide that callback. The cap-gated subscription behavior is new protocol surface rather than an existing current-main failure.

## Solution Assessment

No. The capability direction is reasonable, but this branch is stale against current raw-payload and Gateway fanout contracts and needs a refreshed implementation before merge.

  - confidence: 0.8
