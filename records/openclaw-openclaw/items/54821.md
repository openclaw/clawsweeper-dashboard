## Summary

Keep open. This PR still contains meaningful cap-gated thinking-event protocol work that current main does not implement, but the current branch is conflicting and has blocking reasoning-visibility, payload-contract, and stale-recipient cleanup issues.

## What This Changes

The PR adds a Gateway `thinking-events` client capability, routes thinking agent events to cap-registered WebSocket connections, changes reasoning stream delta tracking, and updates related Gateway/agent tests and generated baselines.

## Best Possible Solution

Land the narrow no-callback bug fix separately, or rebase this capability PR only if maintainers approve the new cap-gated protocol surface while preserving `canShowReasoning`, raw payloads, and recipient cleanup.

## Reproduction Assessment

Yes for the bug portion: current main requires `onReasoningStream`, while Gateway `chat.send` does not provide that callback. The cap-gated subscription behavior is a new protocol feature rather than a current-main bug reproduction.

## Solution Assessment

No, not as-is. The narrower callback-decoupling fix in https://github.com/openclaw/openclaw/pull/79687 and the merged raw-stream refactor cover much of the bug surface, while this broader capability PR still needs maintainer approval and fixes for visibility, payload shape, and cleanup.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve the reasoning visibility gate:** `src/agents/pi-embedded-subscribe.ts:77`
  - body: Current main gates streamed reasoning on `canShowReasoning`, but this branch enables `streamReasoning` from `reasoningMode` alone. A cap-subscribed Gateway client could receive thinking when `thinkingLevel: "off"` should hide it, so keep the visibility gate while making the callback optional.
  - confidence: 0.9
- **[P2] Keep Gateway thinking payloads raw:** `src/agents/pi-embedded-subscribe.ts:688-689`
  - body: Current main sends raw thinking through core and formats at channel edges, but this branch emits `formatReasoningMessage` as Gateway `data.text`. That reintroduces `Reasoning:` Markdown into the Gateway protocol and breaks clients expecting raw text plus raw deltas.
  - confidence: 0.88
- **[P2] Finalize preregistered recipients on dispatch failure:** `src/gateway/server-methods/chat.ts:1555-1558`
  - body: Recipients are registered before dispatch, but if dispatch rejects before a lifecycle end/error event, the catch/finally path only broadcasts a chat error and deletes the abort controller. Mark the run-scoped recipient registries final on that path so stale grants do not live until TTL.
  - confidence: 0.82
