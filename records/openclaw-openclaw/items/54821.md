## Summary

Keep this PR open, but it should not merge in its current form: the bug is real and the cap-gated idea is useful, yet the current head conflicts with later reasoning-stream direction and has concrete visibility/security blockers.

## What This Changes

The PR adds a Gateway `thinking-events` client capability, routes thinking agent events to cap-registered WebSocket recipients, revises reasoning delta handling, and updates gateway/agent tests and generated API baselines.

## Best Possible Solution

Refresh or replace the branch with a protocol-reviewed Gateway thinking-events design that preserves reasoning visibility policy, keeps core payloads raw, finalizes recipient grants on every failure path, and includes focused Gateway routing coverage.

## Reproduction Assessment

Yes for the bug portion, by source inspection: current main requires `params.onReasoningStream` before `emitReasoningStream` can emit, while Gateway `chat.send` does not pass that callback. The cap-gated subscription behavior is a new protocol capability rather than a current-main bug reproduction.

## Solution Assessment

No. The capability direction may be valid, but this PR head drops the reasoning visibility gate, sends formatted core thinking payloads, and leaves pre-registered grants alive on early dispatch failure; the safer path is a refreshed protocol-aware fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve reasoning visibility gates:** `src/agents/pi-embedded-subscribe.ts:77`
  - body: This makes streamed reasoning depend on `reasoningMode === "stream"` alone, dropping current main's `thinkingLevel !== "off"` visibility guard. A cap-subscribed client could receive reasoning when visibility is disabled, so keep that gate while decoupling event emission from the optional callback.
  - confidence: 0.9
- **[P2] Keep thinking payloads raw:** `src/agents/pi-embedded-subscribe.ts:664-689`
  - body: Current main's raw-reasoning refactor sends core thinking `text` and `delta` as raw payloads. Formatting `data.text` here would put `Reasoning:` Markdown back into the Gateway protocol and break clients that consume raw thinking deltas.
  - confidence: 0.88
- **[P2] Finalize pre-registered recipients on dispatch failure:** `src/gateway/server-methods/chat.ts:1558`
  - body: The PR registers tool/thinking recipients before dispatch, but the rejection path only broadcasts the error and deletes the abort controller. If dispatch fails before lifecycle end/error, those run-scoped grants remain until TTL; mark both recipient registries final in the failure path.
  - confidence: 0.8
