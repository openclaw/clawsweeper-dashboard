## Summary

Keep open for maintainer decision: current main already covers the raw reasoning delta part and a narrower open PR covers the no-callback bug, but this branch still carries a unique Gateway capability proposal and has blocking correctness/security issues.

## What This Changes

The PR adds a Gateway `thinking-events` client capability, run-scoped thinking-event recipient routing, and Embedded Pi reasoning-stream delta changes for WebSocket clients.

## Best Possible Solution

Choose the Gateway thinking-stream contract explicitly: land the narrow no-callback bug fix from https://github.com/openclaw/openclaw/pull/79687 if that is the desired behavior, and only add a `thinking-events` capability after preserving visibility gates, raw payloads, cleanup, docs, and protocol tests.

## Reproduction Assessment

Yes, for the original bug path: current main still requires `onReasoningStream`, while Gateway `chat.send` does not provide that callback. I did not run a live current-main Gateway reproduction in this read-only review.

## Solution Assessment

No. The cap-gated direction may be viable, but this branch is not the best version until it preserves current visibility gates, keeps core thinking payloads raw, finalizes pre-registered recipients on early failure, and gets maintainer protocol approval.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve reasoning visibility gates:** `src/agents/pi-embedded-subscribe.ts:77`
  - body: Current main only streams reasoning when visibility allows it. This branch enables `streamReasoning` from `reasoningMode` alone, so a Gateway client with the new cap could receive live reasoning even when `thinkingLevel: "off"` should suppress it.
  - confidence: 0.9
- **[P2] Keep core thinking payloads raw:** `src/agents/pi-embedded-subscribe.ts:688-689`
  - body: Current main now keeps core thinking event `text` and `delta` as raw strings after the raw-payload refactor. Emitting `formatReasoningMessage(...)` as Gateway `data.text` would reintroduce formatted `Reasoning:\n_..._` payloads for protocol clients.
  - confidence: 0.88
- **[P2] Finalize pre-registered recipients on dispatch failure:** `src/gateway/server-methods/chat.ts:1558`
  - body: `chat.send` registers tool and thinking recipients before dispatch, but if `dispatchInboundMessage` rejects before lifecycle events are emitted, no `markFinal` call runs and the run-to-connection grant survives until TTL. Finalize those recipient registries in the early failure path too.
  - confidence: 0.78
