## Summary

Keep open: the PR targets a real gateway thinking-stream gap, but the current branch is not merge-safe because it drops the reasoning visibility gate, conflicts with the raw-reasoning payload refactor already on main, and leaves pre-registered recipient grants alive on early dispatch failures.

## What This Changes

Adds a Gateway `thinking-events` client capability, cap-gated thinking recipient routing, reasoning-stream delta changes, early recipient registration, and related Gateway/agent tests plus generated baseline updates.

## Best Possible Solution

Decide whether the new cap-gated protocol is wanted; if yes, rebase it over current main while preserving visibility gating, raw thinking payloads, and recipient cleanup, otherwise land the narrower callback fix in https://github.com/openclaw/openclaw/pull/79687.

## Reproduction Assessment

Yes for the bug portion: current-main source still requires `onReasoningStream`, while Gateway `chat.send` does not supply it. The cap-gated subscription behavior is a new protocol feature rather than a current-main bug reproduction.

## Solution Assessment

No. The capability direction may be valid, but this PR head drops the visibility gate, sends formatted thinking payloads after the raw-payload refactor, and leaves pre-registered grants alive on early dispatch failures.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve the reasoning visibility gate:** `src/agents/pi-embedded-subscribe.ts:77`
  - body: This changes `streamReasoning` to depend only on `reasoningMode === "stream"`, dropping current main's `thinkingLevel !== "off"` guard. A cap-subscribed client could receive reasoning when visibility is disabled, so keep the visibility gate while making the callback optional.
  - confidence: 0.9
- **[P2] Keep Gateway thinking payloads raw:** `src/agents/pi-embedded-subscribe.ts:688-689`
  - body: Current main's raw-reasoning refactor moved formatting to channel edges, but this branch sends `Reasoning:` Markdown as `data.text` in the Gateway event. That breaks clients expecting raw thinking text and reintroduces formatting into the core protocol.
  - confidence: 0.88
- **[P2] Finalize preregistered recipients on dispatch failure:** `src/gateway/server-methods/chat.ts:1558`
  - body: Thinking recipients are registered before dispatch, but if dispatch rejects before a lifecycle end/error event, the catch/finally path only reports the chat error and deletes the abort controller. Mark the recipient registry final on this early-failure path so stale grants do not remain until TTL.
  - confidence: 0.82
