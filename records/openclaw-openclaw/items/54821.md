## Summary

Keep open. The PR targets a real source-level gap in callback-less Gateway thinking events, and the PR body has adequate screenshot proof, but the current head is stale against main and would regress current reasoning visibility/raw-payload contracts while leaving stale recipient grants after early dispatch failures.

## What This Changes

The PR adds a Gateway `thinking-events` client capability, run-scoped recipient routing for thinking agent events, and reasoning delta changes for WebSocket clients.

## Best Possible Solution

Land a rebased Gateway thinking-events contract that preserves `canShowReasoning`, keeps core thinking payloads raw, finalizes recipient grants on early failures, and adds focused routing/cleanup tests plus protocol documentation if maintainers accept the cap.

## Reproduction Assessment

Yes, source-reproducible. Current main gates `emitReasoningStream` on `onReasoningStream`, while Gateway `chat.send` does not provide that callback, so callback-less WebSocket clients cannot receive live thinking events through this path.

## Solution Assessment

No. Cap-gated delivery is a plausible direction, but this PR head is not the best merge path until it is rebased and preserves the current security and raw-payload contracts.
