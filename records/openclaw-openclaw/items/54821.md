## Summary

Keep open: current main still does not provide cap-gated thinking events, but this branch is not mergeable as-is because it drops reasoning visibility safeguards, conflicts with the merged raw-reasoning stream contract, and leaves pre-registered recipients alive after early dispatch failure.

## What This Changes

The PR adds a `thinking-events` gateway client capability, recipient registry/routing, reasoning-stream delta state, gateway registration plumbing, and related tests/generated SDK baseline updates.

## Best Possible Solution

Rebase or replace this with a focused cap-gated gateway delivery that preserves `canShowReasoning`/`silentExpected`, emits raw core `text`/`delta`, finalizes recipient registries on early failures, and updates the protocol/docs/tests for the new cap.

## Reproduction Assessment

Yes. Source inspection shows current main lacks `thinking-events`, still requires `onReasoningStream` before thinking events are emitted, and routes thinking through generic non-tool agent delivery; I did not run a live model session in this read-only review.

## Solution Assessment

No. The opt-in cap direction is reasonable, but this branch is not the best fix until it is reconciled with current raw reasoning payloads, existing visibility gates, and recipient cleanup behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve reasoning visibility gates:** `src/agents/pi-embedded-subscribe.ts:77`
  - body: Current main only streams reasoning when `thinkingLevel` allows it and also suppresses silent turns before emitting. This branch sets `streamReasoning` from `reasoningMode` alone and the new emit path no longer checks `silentExpected`, so a cap-subscribed client could receive reasoning tokens current main intentionally hides.
  - confidence: 0.86
- **[P2] Keep core thinking payloads raw:** `src/agents/pi-embedded-subscribe.ts:664-689`
  - body: The merged raw-reasoning refactor now treats core thinking `text` and `delta` as raw strings, leaving markdown formatting to channel edges. This branch calls `formatReasoningMessage` before emitting thinking events/callbacks, which would regress gateway and channel consumers back to formatted `Reasoning:\n_..._` payloads.
  - confidence: 0.88
- **[P3] Finalize pre-registered recipients on dispatch failure:** `src/gateway/server-methods/chat.ts:1553-1558`
  - body: These new pre-dispatch registrations can happen before any lifecycle end/error event exists to call `markFinal`. If `dispatchInboundMessage` rejects early, the catch/finally path only removes the active run, leaving tool/thinking recipient entries alive until TTL and possibly reused with the same idempotency run id.
  - confidence: 0.72
