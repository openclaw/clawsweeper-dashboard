## Summary

Keep open for maintainer handling: the underlying Gateway thinking-stream bug is real on current main, but this PR is not ready as the canonical fix because it adds a new capability/security contract, is merge-conflicting, and predates current-main raw reasoning payload work.

## What This Changes

The PR adds a Gateway `thinking-events` capability, run-scoped recipient routing for thinking agent events, and embedded Pi reasoning delta changes for WebSocket clients.

## Best Possible Solution

Land a narrow callback-decoupling fix that preserves `canShowReasoning` and raw payloads, then handle any cap-gated reasoning-event protocol as a separate documented Gateway contract with routing and cleanup tests.

## Reproduction Assessment

Yes. Source inspection shows current main still requires `onReasoningStream`, while Gateway WebSocket chat runs need callback-less agent-event delivery for live thinking events.

## Solution Assessment

No. The cap-gated direction may be useful, but this PR is not the best merge path until maintainers settle the Gateway protocol/security contract and reconcile it with raw payload work and narrower active fixes.
