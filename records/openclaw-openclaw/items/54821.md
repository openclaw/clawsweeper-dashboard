## What This Changes

The PR adds a Gateway `thinking-events` client capability, routes thinking agent events to registered WebSocket recipients, and changes embedded Pi reasoning-stream delta handling.

## Best Possible Solution

Land one coordinated, rebased Gateway reasoning-stream implementation that preserves `canShowReasoning`, keeps Gateway thinking payloads raw, adds opt-in subscription only if maintainers want that protocol boundary, and finalizes recipient grants on every dispatch exit.

## Solution Assessment

No, not as-is. The direction may be useful, but the current patch needs a rebase plus fixes for reasoning visibility, raw payload alignment, and recipient cleanup before maintainers can evaluate the protocol change.
