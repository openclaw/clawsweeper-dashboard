## Summary

Keep this PR open: current main still lacks the `thinking-events` capability path and Gateway `chat.send` still does not pass `onReasoningStream`, but this branch is conflicting and would regress newer reasoning visibility/raw-payload contracts if merged as-is.

## What This Changes

The PR adds a Gateway `thinking-events` client capability, routes thinking agent events to capability subscribers, changes reasoning delta handling, and updates Gateway/agent tests and generated API baselines.

## Best Possible Solution

Refresh or replace this with a minimal Gateway `thinking-events` capability that preserves reasoning visibility gates, keeps core payloads raw, finalizes recipient grants on every failure path, and documents/tests the protocol contract.

## Reproduction Assessment

Yes for the bug portion by source inspection: current main still requires `onReasoningStream` before emitting `stream: "thinking"`, while Gateway `chat.send` does not provide that callback. The cap-gated subscription behavior is new protocol surface rather than a current-main failure reproduction.

## Solution Assessment

No. The capability direction is reasonable, but this branch is stale and would regress current raw-payload and visibility contracts; the maintainable path is a refreshed, narrow Gateway cap implementation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve reasoning visibility gates:** `src/agents/pi-embedded-subscribe.ts:77`
  - body: This enables streamed reasoning from `reasoningMode === "stream"` alone, dropping current main's `thinkingLevel` visibility guard. A cap-subscribed client could receive reasoning when reasoning visibility is disabled, so keep the visibility gate while decoupling event emission from the optional callback.
  - confidence: 0.9
- **[P2] Keep thinking payloads raw:** `src/agents/pi-embedded-subscribe.ts:664-689`
  - body: Current main sends core reasoning `text` and `delta` as raw payloads after the raw-reasoning refactor. Formatting with `formatReasoningMessage` here would put `Reasoning:` Markdown back into the Gateway protocol and break clients that consume raw thinking deltas.
  - confidence: 0.88
- **[P2] Finalize pre-registered recipients after dispatch failures:** `src/gateway/server-methods/chat.ts:1558`
  - body: The PR registers tool/thinking recipients before dispatch, but the rejection path only broadcasts the error and deletes the abort controller. If dispatch fails before lifecycle end/error, those run-scoped grants remain until TTL; mark both recipient registries final in the catch/finally path.
  - confidence: 0.8
