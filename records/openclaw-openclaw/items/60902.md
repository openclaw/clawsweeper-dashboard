## What This Changes

The PR adds a Discord `/vc switch <agent>` subcommand, a `DiscordVoiceManager.switchAgent()` implementation, command tests, Discord voice docs, and a changelog entry.

## Best Possible Solution

Land a rebased Discord plugin implementation that preserves configured voice target sessions, snapshots route state through current helper APIs, constrains target agents by an explicit policy, updates both docs references, and includes redacted real Discord voice proof.

## Reproduction Assessment

Not applicable as a current-main bug. The blocking PR defects are source-reproducible from the PR head against current main: target validation, session-key choice, and helper route reads.

## Solution Assessment

No, not as-is. The feature should be rebased onto current Discord voice routing and define whether target agents are resolved through existing route policy or an explicit allow rule before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Constrain switch targets to permitted agents:** `extensions/discord/src/voice/manager.ts:543-545`
  - body: `switchAgent()` accepts any configured agent ID after voice-command access succeeds, bypassing Discord route/binding selection for the active guild/channel/speaker. Resolve the target through the existing routing policy or require an explicit owner/allowlist rule before mutating the shared voice session.
  - confidence: 0.86
- **[P2] Preserve configured voice session targets:** `extensions/discord/src/voice/manager.ts:547-552`
  - body: This builds the new session key from `entry.sessionChannelId`, which is the voice channel session. Current main supports `channels.discord.voice.agentSession.mode: "target"`, where agent turns intentionally use a configured text-channel or DM target; switching agents this way would move the conversation back to the voice channel session.
  - confidence: 0.9
- **[P2] Thread the route snapshot through current helpers:** `extensions/discord/src/voice/manager.ts:641-647`
  - body: The snapshot is passed only into this branch's inline `processSegment()` path. Current main delegates queued audio through `processDiscordVoiceSegment()` and `runDiscordVoiceAgentTurn()`, which still read `entry.route`, so pre-switch queued audio can run under the new agent after rebase unless the snapshot is carried through those helpers.
  - confidence: 0.88
- **[P3] Update the shared slash-command reference:** `docs/channels/discord.md:1033-1038`
  - body: The Discord channel page documents `/vc switch`, but the shared slash-command page still lists only `/vc join|leave|status`. Please update that reference too so the public command docs do not disagree.
  - confidence: 0.86
