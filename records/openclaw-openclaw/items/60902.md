## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Constrain switch targets to permitted agents:** `extensions/discord/src/voice/manager.ts:543-545`
  - body: `switchAgent()` only checks that the requested ID exists, while `/vc switch` authorization is for the current voice channel. That lets anyone with voice command access move the shared session to any configured agent, bypassing the route or binding policy that selected the original agent.
  - confidence: 0.86
- **[P2] Preserve configured voice-session targets:** `extensions/discord/src/voice/manager.ts:547-552`
  - body: For `voice.agentSession.mode: "target"`, current main routes agent turns through the configured target channel or DM. Rebuilding `sessionKey` from `entry.sessionChannelId` moves switched turns back to the voice channel session and breaks that documented setup.
  - confidence: 0.84
- **[P2] Thread route snapshots through the current voice helpers:** `extensions/discord/src/voice/manager.ts:641-647`
  - body: Current main now delegates STT and agent turns to `processDiscordVoiceSegment()` and `runDiscordVoiceAgentTurn()`, which read `entry.route`. After rebasing, the snapshot must be passed through those helpers or queued pre-switch audio can still run under the new agent.
  - confidence: 0.82
