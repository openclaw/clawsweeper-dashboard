Keep open: current main still lacks `/vc switch`, and this PR is a useful feature direction, but it is not mergeable as-is against current Discord voice routing. The branch needs a current-main rebase, real Discord voice proof, and fixes for agent-switch authorization, routed target preservation, queued/realtime route snapshots, and the shared slash-command docs reference.

## What This Changes

Adds a Discord `/vc switch <agent>` subcommand, `DiscordVoiceManager.switchAgent()`, command tests, Discord voice docs, and a changelog entry.

## Best Possible Solution

Land a rebased Discord plugin implementation that preserves existing voice routing contracts, constrains agent switching through an explicit policy, carries captured route state through current helper paths, aligns all docs references, and includes redacted real Discord voice proof.

## Reproduction Assessment

Not applicable as a current-main bug because this is a new feature PR. The blocking PR defects are source-reproducible from current main: current helpers read mutable `entry.route`, target voice sessions can route away from the voice channel, and the PR adds unrestricted agent selection.

## Solution Assessment

No, not as-is. The feature should be rebased onto current Discord voice routing and should define the authorization/target-session semantics before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Constrain switch targets to permitted agents:** `extensions/discord/src/voice/manager.ts:543-545`
  - body: `switchAgent()` currently accepts any configured agent ID after only voice-channel access succeeds. That bypasses Discord route/binding selection for the active channel or speaker, so a user allowed to operate voice in one context could move the shared session to an agent with different tool policy; resolve the target through the existing route policy or require an explicit owner/allowlist rule.
  - confidence: 0.83
- **[P2] Preserve configured voice session targets:** `extensions/discord/src/voice/manager.ts:547-552`
  - body: This rebuilds the new session key from `entry.sessionChannelId`, which is the voice channel id. Current main supports `channels.discord.voice.agentSession.mode: "target"`, where agent turns intentionally use a configured text-channel/direct target; switching agents this way would silently move the conversation back to the voice channel session.
  - confidence: 0.9
- **[P2] Thread the route snapshot through current helpers:** `extensions/discord/src/voice/manager.ts:644-647`
  - body: The snapshot is passed only to the branch's older inline `processSegment()` path. Current main delegates queued audio to `processDiscordVoiceSegment()` and `runDiscordVoiceAgentTurn()`, which still read `entry.route`, so pre-switch queued audio can be dispatched under the new agent unless the snapshot is passed through those helpers too.
  - confidence: 0.9
- **[P3] Update the shared slash-command reference:** `docs/channels/discord.md:1033-1038`
  - body: The Discord channel doc adds `/vc switch`, but the shared slash-command page still lists `/vc join|leave|status`. Landing only this page leaves the public command references inconsistent.
  - confidence: 0.86
