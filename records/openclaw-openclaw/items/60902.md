## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Constrain switch targets to permitted agents:** `extensions/discord/src/voice/manager.ts:543-545`
  - body: `switchAgent()` accepts any configured agent ID after voice-command access succeeds, bypassing Discord route/binding selection for the active guild, channel, and speaker. Resolve the target through the existing routing policy or require an explicit owner/allowlist rule before mutating the shared voice session.
  - confidence: 0.86
- **[P2] Preserve configured voice session targets:** `extensions/discord/src/voice/manager.ts:547-552`
  - body: This builds the new session key from `entry.sessionChannelId`, which is the joined voice channel. Current main supports `channels.discord.voice.agentSession.mode: "target"`, where agent turns intentionally use a configured text-channel or DM target; switching agents this way would move the conversation back to the voice channel session.
  - confidence: 0.9
- **[P2] Thread the route snapshot through current helpers:** `extensions/discord/src/voice/manager.ts:641-647`
  - body: The snapshot fix is only wired into this branch's inline `processSegment()` path. Current main delegates queued audio through `processDiscordVoiceSegment()` and `runDiscordVoiceAgentTurn()`, which still read `entry.route`, so pre-switch queued audio can run under the new agent after rebase unless the snapshot is carried through those helpers.
  - confidence: 0.88
- **[P3] Update the shared slash-command reference:** `docs/channels/discord.md:1033-1038`
  - body: The Discord channel page documents `/vc switch`, but the shared slash-command page still lists only `/vc join|leave|status`. Update that reference too so public command docs do not disagree.
  - confidence: 0.86
