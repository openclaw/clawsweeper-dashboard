## Summary

Keep this PR open: it targets a real current-main setup bug, but the latest head still has two script-local correctness blockers and lacks after-fix real behavior proof.

## What This Changes

The PR changes `scripts/setup-auth-system.sh` to copy the auth-monitor service template to the user systemd path and render `ExecStart` plus notification values there instead of editing the tracked template.

## Best Possible Solution

Materialize and validate a temporary service unit from the template, match any existing `Environment=NOTIFY_*=` line by key, then install it only after renderer success while keeping the tracked template unchanged.

## Reproduction Assessment

Yes, via source inspection on current main: the setup script edits the tracked service template with `sed -i` before copying it, and the template still contains a hard-coded checkout path. I did not execute setup because this review is read-only.

## Solution Assessment

No. The copy-then-render direction is right, but the current diff still replaces the installed unit before validation and does not tolerate arbitrary previously rendered notification environment values.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Render the service before replacing the installed unit:** `scripts/setup-auth-system.sh:72`
  - body: The PR copies the unrendered template over `SERVICE_TARGET` before checking `python3` or validating placeholders. If the renderer is missing or fails, setup exits after replacing a working user unit with the hard-coded template; render a temporary unit and move it into place only after success.
  - confidence: 0.9
- **[P2] Match already-rendered NOTIFY lines by key:** `scripts/setup-auth-system.sh:113-116`
  - body: Older setup runs could leave `Environment=NOTIFY_NTFY=<user topic>` or `Environment=NOTIFY_PHONE=<user phone>` in the tracked template. This helper only matches the default value, so reruns from those dirty checkouts still fail with `placeholder not found`; replace any existing line for the key.
  - confidence: 0.92
