## What This Changes

The PR changes `scripts/setup-auth-system.sh` so auth-monitor setup renders a temporary systemd service unit, substitutes the checkout-local `ExecStart` and notification environment lines, then installs the rendered unit atomically.

## Best Possible Solution

Land the materialized-unit rendering fix after maintainers either accept the guarded `python3` renderer or request a shell-only renderer, and after current-head validation is clean or explicitly waived.

## Reproduction Assessment

Yes. Current-main source shows that providing notification values makes `scripts/setup-auth-system.sh` edit the tracked service template with `sed -i` before copying it into the user systemd directory; I did not run the interactive setup because this review is read-only.

## Solution Assessment

Yes, with one maintainer choice left. Rendering a temporary copy, validating required replacements, preserving the installed unit on renderer failure, and matching `NOTIFY_*` lines by key is the right fix shape; if the new `python3` prerequisite is not acceptable, a shell-only renderer is the safer alternative.
