## Summary

Keep open: this is a focused setup-script fix that is not implemented on current main, but it still needs normal maintainer merge handling because required CI is red and the new ExecStart renderer has one low-priority systemd escaping edge case.

## What This Changes

The PR changes `scripts/setup-auth-system.sh` to render the auth-monitor user service into a temporary unit, validate placeholders, install it atomically, and avoid mutating the tracked systemd template.

## Best Possible Solution

Keep the temp-rendered, validated service install, complete systemd ExecStart literal escaping, and land only after required checks are green or explicitly waived.

## Reproduction Assessment

Yes. Current-main source shows the setup script edits the tracked systemd template with `sed -i` before copying it, so running setup with a notification value would dirty the checkout and can install the hardcoded template path; I did not execute the interactive setup in this read-only review.

## Solution Assessment

Mostly yes. Rendering a temporary copy, validating markers, tolerating previously rendered `NOTIFY_*` lines, and avoiding a Python dependency is the maintainable fix; escaping literal `$` in the rendered ExecStart path would make the systemd quoting complete.
