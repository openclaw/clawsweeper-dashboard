## Summary

Keep open: current main and the latest release still mutate the tracked auth-monitor service template, so the PR targets a real setup-script bug, but the latest head still has two blocking correctness issues and lacks inspectable after-fix real behavior proof.

## What This Changes

The branch changes `scripts/setup-auth-system.sh` to render the auth-monitor user systemd service from the tracked template with Python instead of patching the repository template in place.

## Best Possible Solution

Render and validate a materialized service into a temporary file, install it only after success, and replace any existing `NOTIFY_*` environment line by key while keeping repository templates untouched.

## Reproduction Assessment

Yes. By source inspection, current main still runs `sed -i` against `scripts/systemd/openclaw-auth-monitor.service` before copying it, so entering notification values mutates a tracked template.

## Solution Assessment

No. Copy-then-render is the right direction, but the current PR head still overwrites the installed unit before renderer validation and does not tolerate arbitrary previously-rendered `NOTIFY_*` values.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Render before replacing the installed service:** `scripts/setup-auth-system.sh:72`
  - body: This copies the unrendered template directly over `SERVICE_TARGET` before the `python3` preflight and placeholder validation run. If either step fails, setup exits after replacing an existing working user unit with the raw hard-coded template; render to a temporary file and install it only after validation succeeds.
  - confidence: 0.93
- **[P2] Replace existing NOTIFY lines by key:** `scripts/setup-auth-system.sh:113-116`
  - body: The helper only accepts the original placeholder value or that same value already uncommented. Older setup runs can leave `Environment=NOTIFY_NTFY=<user topic>` or `Environment=NOTIFY_PHONE=<user phone>` in the tracked template, so reruns from those dirty checkouts still fail with placeholder not found; match the key and replace the whole line.
  - confidence: 0.92
