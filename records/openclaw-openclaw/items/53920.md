## Summary

Keep open: the PR targets a real source-reproducible setup bug, but the latest head still has two setup-script correctness blockers and lacks inspectable real behavior proof for the after-fix systemd setup path.

## What This Changes

The PR changes scripts/setup-auth-system.sh to render the auth-monitor user systemd unit from the tracked template and update ExecStart plus optional notification environment values there.

## Best Possible Solution

Render and validate the service into a temporary materialized unit, install it only after success, and replace notification environment lines by key while leaving the tracked template untouched.

## Reproduction Assessment

Yes, by source inspection rather than execution: current main still edits scripts/systemd/openclaw-auth-monitor.service with sed -i before copying it, so providing notification values would dirty the tracked template.

## Solution Assessment

No. Copy-then-render is the right direction, but this PR still overwrites the installed unit before renderer validation and only accepts prior NOTIFY lines that exactly match placeholder defaults.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Render before overwriting the installed service:** `scripts/setup-auth-system.sh:72`
  - body: This copies the unrendered template to SERVICE_TARGET before the python3 preflight or placeholder validation runs. If the renderer is unavailable or exits on a mismatch, setup has already replaced a working user service with the hard-coded template; render to a temporary file and install it only after validation succeeds.
  - confidence: 0.93
- **[P2] Match prior NOTIFY values by key:** `scripts/setup-auth-system.sh:113-115`
  - body: The rerender helper only accepts the original placeholder value or that same value already uncommented. Older setup runs can leave Environment=NOTIFY_NTFY=<user topic> or Environment=NOTIFY_PHONE=<user phone> in the tracked template, so reruns from those dirty checkouts still exit with placeholder not found; replace any existing line for the key.
  - confidence: 0.92
