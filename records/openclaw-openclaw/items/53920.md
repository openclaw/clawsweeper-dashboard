## What This Changes

The PR changes `scripts/setup-auth-system.sh` to render the auth-monitor service into temporary files before installing it, updates changed-test routing for that script, and adds a focused planner test.

## Best Possible Solution

Land the render-into-temporary-unit approach once maintainers accept the supplied terminal proof or add a live Linux user-systemd smoke for the final install path.

## Reproduction Assessment

## Solution Assessment

Yes, this is the right layer for the fix: render a copy of the host-specific systemd unit before install, keep the tracked template canonical, and avoid changing auth-monitor runtime behavior. A broader doctor/config migration is not needed for this optional host-specific script.
