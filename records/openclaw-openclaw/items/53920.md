## Summary

Keep this PR open: it targets a real current-main setup bug, but the latest head still has two script-local blocking regressions and the external contributor has not attached inspectable after-fix real behavior proof.

## What This Changes

The PR changes `scripts/setup-auth-system.sh` to render the auth-monitor systemd service copy with an inline Python helper instead of editing the tracked template in place.

## Best Possible Solution

Keep the copy-then-render direction, but render and validate a temporary service unit, replace the installed target only after success, and accept any existing `NOTIFY_*` environment line left by dirty templates.

## Reproduction Assessment

Yes. Current main is source-reproducible: it edits the tracked systemd template with `sed -i` before copying it, and the template still contains the hard-coded auth-monitor checkout path.

## Solution Assessment

No. The PR direction is right, but the latest patch still overwrites the installed unit before renderer validation can fail and does not tolerate arbitrary previously rendered notification environment values.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Render into a temp unit before replacing the target:** `scripts/setup-auth-system.sh:72`
  - body: This copies the unrendered template over `SERVICE_TARGET` before checking `python3` or validating placeholders. If either step fails, setup exits after replacing the user's working unit with the hard-coded template, so render a temp file and move it into place only after validation succeeds.
  - confidence: 0.9
- **[P2] Match existing NOTIFY environment values:** `scripts/setup-auth-system.sh:113-116`
  - body: Older setup runs could leave `Environment=NOTIFY_NTFY=<user topic>` or `Environment=NOTIFY_PHONE=<user phone>` in the tracked template. The helper only matches the commented defaults and exact default values, so reruns from those dirty checkouts still fail instead of repairing the install.
  - confidence: 0.93
