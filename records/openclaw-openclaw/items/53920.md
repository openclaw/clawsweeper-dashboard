## Summary

Keep open. Current main still has the reported setup-script bug, and the PR is useful, but the latest head still has two script-local correctness blockers before it should merge.

## What This Changes

The PR changes `scripts/setup-auth-system.sh` to render the auth-monitor user service from a copied template with an inline Python helper instead of mutating the tracked systemd template.

## Best Possible Solution

Keep the copy-then-render direction, but render and validate a temporary service file before replacing `SERVICE_TARGET`, accept any existing `Environment=NOTIFY_*=` line, and add the required changelog entry.

## Reproduction Assessment

Yes. Current main is source-reproducible: `scripts/setup-auth-system.sh` edits the tracked service template with `sed -i` before copying it, and the template still has a hard-coded `ExecStart` path.

## Solution Assessment

No. The PR’s direction is narrow and maintainable, but it still writes the installed unit before render validation and only handles default notification placeholders instead of arbitrary previously rendered values.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Render the unit before replacing the installed service:** `scripts/setup-auth-system.sh:72`
  - body: The script copies the raw template over `SERVICE_TARGET` before the `python3` preflight and placeholder validation. With `set -e`, a missing Python binary or placeholder drift exits after clobbering the user's working unit with the hard-coded template, so render a temp file and move it into place only after validation succeeds.
  - confidence: 0.9
- **[P2] Accept arbitrary existing NOTIFY environment lines:** `scripts/setup-auth-system.sh:113-116`
  - body: This only accepts the commented defaults or exact uncommented default values. Older setup runs could have dirtied the tracked template with `Environment=NOTIFY_NTFY=<user topic>` or `Environment=NOTIFY_PHONE=<user phone>`, and rerunning this PR would still fail with `placeholder not found`; match any existing line for the key.
  - confidence: 0.94
- **[P3] Add the required changelog entry:** `scripts/setup-auth-system.sh:63`
  - body: This is a user-facing setup-script fix, but the PR only changes `scripts/setup-auth-system.sh`. Add an active `CHANGELOG.md` `### Fixes` entry before merge.
  - confidence: 0.84
