## What This Changes

The PR changes `scripts/setup-auth-system.sh` to render the auth-monitor systemd service through temporary files, replace `ExecStart` and `NOTIFY_*` lines, and install the rendered unit without mutating the checked-in template.

## Best Possible Solution

Keep the temporary render-and-install approach, but make `systemd_quote_arg` cover systemd’s literal-dollar contract and prove the generated unit with a checkout path containing both spaces and `$`.

## Reproduction Assessment

Yes. Current main is source-reproducible: `scripts/setup-auth-system.sh` edits the tracked systemd template with `sed` before copying it, and the template contains a hardcoded `/home/admin/openclaw` `ExecStart`; I did not execute the mutating script against the checkout.

## Solution Assessment

No, not quite. The copy-render-install shape is the right boundary, but the renderer is not yet the best fix because it misses systemd’s required `$$` escaping for literal dollar signs in command paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Escape dollar signs in rendered ExecStart paths:** `scripts/setup-auth-system.sh:84-89`
  - body: systemd still performs environment-variable substitution on service command lines, and its docs require literal dollar signs to be written as `$$`. `systemd_quote_arg` escapes backslashes, `%`, and quotes but leaves `$` untouched, so a checkout under a path like `/home/me/$work/openclaw` installs an `ExecStart` that systemd expands to the wrong path or an empty segment and the timer cannot start `auth-monitor.sh`. Please double literal dollar signs before writing the rendered command path.
  - confidence: 0.87
