## What This Changes

The PR changes `scripts/setup-auth-system.sh` to render the auth-monitor user systemd service from a temporary copied template before installing it, instead of editing the tracked template in place.

## Best Possible Solution

Proceed with the narrow renderer fix after normal maintainer review; request one live Linux user-systemd smoke only if maintainers want supervisor-level proof beyond the supplied terminal rendering evidence.

## Reproduction Assessment

Yes. Source inspection of current main and `v2026.6.8` shows the setup script still mutates the tracked service template with `sed -i` before copying it whenever notification inputs are provided; I did not execute the setup script because this review is read-only.

## Solution Assessment

Yes. Rendering a copied template into temporary files, validating the expected service lines, escaping the generated command path, and atomically installing the rendered unit is the narrow maintainable fix for the implicated setup path.
