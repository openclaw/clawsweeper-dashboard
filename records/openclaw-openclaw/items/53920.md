## Summary

Keep open: current main still mutates the tracked auth-monitor systemd template, and this PR is pointed at the right bug, but the latest head is not merge-ready because two setup failure/rerun cases remain and the required changelog entry is missing.

## What This Changes

The PR changes `scripts/setup-auth-system.sh` to copy the auth-monitor systemd template into the user unit path, render `ExecStart` and notification environment values in that copy with a Python helper, and leave the tracked template unchanged.

## Best Possible Solution

Keep the copy-then-render approach, but render into a temporary materialized unit after preflight checks, move it over the installed user unit only after successful validation, accept arbitrary existing `Environment=NOTIFY_*=` values from previously mutated templates, and add the required active changelog entry before maintainer review/merge.

## Reproduction Assessment

Yes. Current main is reproducible by source inspection: it edits the tracked template with `sed -i` before copying it, and the service template still has a hard-coded `ExecStart`; the remaining PR bugs are reproducible with a previously mutated `Environment=NOTIFY_NTFY=oldtopic` line or a render/preflight failure after the PR overwrites the user unit.

## Solution Assessment

No. The high-level solution is right, but the current patch is not the narrowest safe implementation because it still rejects arbitrary previously rendered notification values and overwrites the installed unit before it knows rendering can succeed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Handle arbitrary existing notification values:** `scripts/setup-auth-system.sh:113-116`
  - body: Older setup runs can leave the tracked template with `Environment=NOTIFY_NTFY=<user topic>` or `Environment=NOTIFY_PHONE=<user phone>`. This helper only accepts the default placeholder value, so rerunning setup from those dirty checkouts still exits with `placeholder not found`; match any existing `Environment=<key>=...` line before replacing it.
  - confidence: 0.94

If either step fails, setup exits with the user's unit file replaced by the hard-coded template; preflight first or render to a temp file and move it into place only after success.
  - confidence: 0.87
