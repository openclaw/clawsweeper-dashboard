## Summary

Keep this PR open: it addresses a real current-main bug, but the latest head still has two blocking script correctness problems and lacks after-fix real behavior proof from the contributor.

## What This Changes

The PR changes `scripts/setup-auth-system.sh` so auth-monitor setup renders a copied user systemd unit instead of mutating the tracked service template.

## Best Possible Solution

Render and validate a temporary service unit from the template, replace any existing `Environment=NOTIFY_*=` line by key, then install the rendered unit only after success while keeping the tracked template unchanged.

## Reproduction Assessment

Yes. Source inspection on current main shows `scripts/setup-auth-system.sh` edits the tracked service template with `sed -i` before copying it, and the template still contains the hard-coded `/home/admin/openclaw/scripts/auth-monitor.sh` path.

## Solution Assessment

No. Copy-then-render is the right direction, but this PR should render a temporary unit before replacing the installed one and should match prior rendered notification environment lines by key rather than by the default placeholder value.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Render before replacing the installed unit:** `scripts/setup-auth-system.sh:72`
  - body: The PR copies the unrendered template over `SERVICE_TARGET` before the `python3` check or placeholder validation runs. If the renderer is unavailable or exits on a placeholder mismatch, setup has already replaced a working user unit with the hard-coded template; render a temporary unit and move it into place only after success.
  - confidence: 0.92
- **[P2] Match existing NOTIFY lines by key:** `scripts/setup-auth-system.sh:113-116`
  - body: Older setup runs can leave `Environment=NOTIFY_NTFY=<user topic>` or `Environment=NOTIFY_PHONE=<user phone>` in the tracked template. This helper only accepts the default placeholder value, so reruns from those dirty checkouts still fail with `placeholder not found`; replace any existing line for the key.
  - confidence: 0.91
