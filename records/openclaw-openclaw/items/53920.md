## Summary

Keep open: the bug still exists on current main and in the latest shipped release, and this PR is aimed at the right root cause, but the latest branch still has two blocking setup-script issues plus missing real behavior proof.

## What This Changes

The PR changes `scripts/setup-auth-system.sh` to render the auth-monitor systemd service in the copied user unit instead of editing the tracked service template.

## Best Possible Solution

Render and validate the auth-monitor service in a temporary file, then install the rendered user unit only after success while accepting prior `Environment=NOTIFY_*=` lines by key.

## Reproduction Assessment

Yes. Source inspection on current main and `v2026.5.7` shows the setup script edits the tracked systemd template with `sed -i` before copying it, which is the reported dirty-working-tree path.

## Solution Assessment

No. Copy-then-render is the right direction, but the current PR head should render before overwriting the installed unit and should match previously rendered notification environment lines by key.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Render before overwriting the installed service:** `scripts/setup-auth-system.sh:72`
  - body: This copies the unrendered template to `SERVICE_TARGET` before the `python3` preflight or placeholder validation runs. If the renderer is unavailable or exits on a mismatch, setup has already replaced a working user service with the hard-coded template; render to a temporary file and install it only after validation succeeds.
  - confidence: 0.93
- **[P2] Match prior NOTIFY values by key:** `scripts/setup-auth-system.sh:114-115`
  - body: The rerender helper only accepts the original placeholder value or that same value already uncommented. Older setup runs can leave `Environment=NOTIFY_NTFY=<user topic>` or `Environment=NOTIFY_PHONE=<user phone>` in the tracked template, so reruns from those dirty checkouts still exit with `placeholder not found`; replace any existing line for the key.
  - confidence: 0.92
