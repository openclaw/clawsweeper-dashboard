## Summary

Keep open: the PR targets a real current and shipped setup bug, but the latest head still has two setup-script correctness blockers and lacks inspectable real behavior proof.

## What This Changes

The PR changes `scripts/setup-auth-system.sh` to render the auth-monitor systemd service into the copied user unit while leaving the tracked template unchanged.

## Best Possible Solution

Keep the scoped setup fix, but render and validate a temporary service unit, replace notification environment lines by key, install only after success, and add redacted terminal proof of setup and rerun behavior.

## Reproduction Assessment

Yes, by source inspection: current main and `v2026.5.18` still mutate the tracked service template with `sed -i` and copy a unit whose `ExecStart` is hard-coded to `/home/admin/openclaw/scripts/auth-monitor.sh`. I did not run the interactive setup because this review is read-only.

## Solution Assessment

No. Copy-then-render is the right direction, but the branch should render into a temporary unit before overwriting the installed service and should replace existing notification environment lines by key rather than only default placeholder text.

## Review Findings

Full review comments:

- **[P2] Render before overwriting the installed service:** `scripts/setup-auth-system.sh:72`
  - body: This copies the raw template over `SERVICE_TARGET` before the `python3` preflight or placeholder validation runs. If the renderer is unavailable or exits on a mismatch, setup has already replaced a working user unit with the hard-coded template, so render to a temporary file and install it only after validation succeeds.
  - confidence: 0.93
- **[P2] Replace prior NOTIFY lines by key:** `scripts/setup-auth-system.sh:113-115`
  - body: This only accepts the original commented placeholder or that same default value already uncommented. Existing dirty checkouts can contain `Environment=NOTIFY_NTFY=<user topic>` or `Environment=NOTIFY_PHONE=<user phone>`, so reruns still fail with "placeholder not found" instead of repairing the unit; match the key regardless of the old value.
  - confidence: 0.92
