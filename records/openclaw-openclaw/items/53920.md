## Summary

Keep open: current main still has the reported setup-script bug, but the latest PR head still has two blocking script behavior issues and lacks after-fix real behavior proof.

## What This Changes

This PR changes `scripts/setup-auth-system.sh` to copy the auth-monitor systemd template into the user unit path and render `ExecStart` and notification environment values there instead of editing the tracked template.

## Best Possible Solution

Render and validate a temporary service unit from the tracked template, replace existing notification environment lines by key, then install the rendered unit only after success with real user-systemd proof.

## Reproduction Assessment

Yes. Source inspection on current main shows `scripts/setup-auth-system.sh` edits the tracked service template with `sed -i` before copying it, while the template still contains the hard-coded `ExecStart` path.

## Solution Assessment

No. Copy-then-render is the right direction, but this PR should render a temporary unit before replacing the installed one and should match prior rendered notification environment lines by key rather than by the default placeholder value.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Render before overwriting the installed unit:** `scripts/setup-auth-system.sh:72`
  - body: This copies the unrendered template over `SERVICE_TARGET` before the `python3` check or placeholder validation runs. If the renderer is unavailable or exits on a mismatch, setup has already replaced a working user unit with the hard-coded template; render to a temporary file and install it only after validation succeeds.
  - confidence: 0.92
- **[P2] Replace existing NOTIFY lines by key:** `scripts/setup-auth-system.sh:113-119`
  - body: Older setup runs can leave `Environment=NOTIFY_NTFY=<user topic>` or `Environment=NOTIFY_PHONE=<user phone>` in the tracked template. This helper only accepts the original placeholder values, so reruns from those dirty checkouts still fail with `placeholder not found`; match any existing line for the key.
  - confidence: 0.91
