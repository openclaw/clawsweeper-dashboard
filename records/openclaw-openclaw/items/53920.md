## Summary

Keep open: this is a valid focused bug fix, but the current PR head still has two blocking setup-script regressions and lacks after-fix real behavior proof from a real setup.

## What This Changes

The PR changes `scripts/setup-auth-system.sh` to render the auth-monitor systemd unit under the user systemd directory instead of editing the tracked service template.

## Best Possible Solution

Render and validate a temporary service unit from the tracked template, replace notification environment lines by key, then install the rendered unit only after success with redacted setup proof.

## Reproduction Assessment

Yes. Source inspection on current main shows the setup script edits the tracked template with `sed -i` and then copies it, while the template still contains a hard-coded `ExecStart` path.

## Solution Assessment

No. Copy-and-render is the right direction, but the current PR should render to a temporary unit before replacing the installed one and should match existing NOTIFY environment lines by key rather than by only the default values.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Render before replacing the installed unit:** `scripts/setup-auth-system.sh:72`
  - body: This copies the unrendered template to the user systemd path before the `python3` check or placeholder validation runs. If either step fails, setup has already replaced a working installed unit with the hard-coded template, so render and validate a temporary file before moving it into place.
  - confidence: 0.92
- **[P2] Match existing NOTIFY lines by key:** `scripts/setup-auth-system.sh:113-116`
  - body: The helper only accepts the original commented placeholder or the uncommented default value. Older setup runs can dirty the tracked template with `Environment=NOTIFY_NTFY=<user topic>` or `Environment=NOTIFY_PHONE=<user phone>`, so reruns from those checkouts still exit with `placeholder not found`; match the `Environment` key regardless of prior value.
  - confidence: 0.91
