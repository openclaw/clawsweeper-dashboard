## Summary

Close as superseded by #73402; the PR targets a real live-tail bug, but its chat-event implementation misses external-channel ingress and broadens raw user text delivery, while the maintainer PR tracks the scoped transcript-subscription fix.

## What This Changes

The PR adds a new gateway `user-message` chat event from `chat.send`, extends TUI handling/types/protocol schema/tests for that event, and includes unrelated Anthropic model-selection regression coverage.

## Best Possible Solution

Close this PR in favor of the maintainer transcript-subscription PR and land the scoped `session.message` implementation for the user-facing fix.

## Reproduction Assessment

Yes, source-reproducible. Current main has scoped `session.message` transcript events, but the TUI only routes `chat`, `chat.side_result`, and `agent` events and has no active-session transcript subscribe path.

## Solution Assessment

No. The proposed broad `chat` event is not the best fix because current main already has a scoped transcript event contract, and #73402 tracks wiring the TUI to that contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Emit user turns from the transcript stream:** `src/gateway/server-methods/chat.ts:1274-1275`
  - body: The new `user-message` event is emitted only inside `chat.send`, but the reported Telegram/external-channel flow records inbound turns through the shared channel turn and dispatch path, bypassing this RPC. Wire the TUI to `session.message` or emit from the shared transcript path instead.
  - confidence: 0.91
- **[P2] Target raw user turns to session subscribers:** `src/gateway/server-methods/chat.ts:1274`
  - body: `context.broadcast("chat", userMessagePayload)` sends raw user text over the broad chat broadcast path to every operator-read client. Raw transcript turns should stay on `session.message` or an equivalent subscriber-scoped event so unrelated clients do not receive them.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff introduces a privacy-scope concern by broadcasting raw user message text on the broad `chat` event.

Concerns:

- **[medium] Raw user text is broadcast to all operator read clients:** `src/gateway/server-methods/chat.ts:1274`
  - body: The proposed `context.broadcast("chat", userMessagePayload)` uses the broad chat broadcast path, while existing transcript user turns can be delivered through scoped `session.message` subscribers. This widens exposure of channel user text beyond active session watchers.
  - confidence: 0.86
