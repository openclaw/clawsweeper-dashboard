## Summary

Keep this PR open for maintainer handling. Current main still lacks live TUI rendering for inbound user turns, but this PR’s `chat.send`-scoped `user-message` event misses the external-channel ingress path from the linked report and widens raw message fanout beyond the existing subscription-gated transcript stream.

## What This Changes

The PR emits a new `user-message` chat event from `chat.send`, extends TUI chat handling/types/protocol schema/tests for that event, and adds unrelated Anthropic model-selection regression coverage.

## Best Possible Solution

Use the existing transcript stream for this feature: have TUI `--session` subscribe to `sessions.messages.subscribe`, render active-session `session.message` user turns with local echo/dedupe protection, unsubscribe on session switch/shutdown, and keep raw user text on scoped subscriber delivery rather than a broad `chat` event. The unrelated model-selection test should be separated from this PR.

## Reproduction Assessment

Yes. The code path is high-confidence: current main loads history and handles run-scoped `chat`/`agent` events, but it neither subscribes to nor routes `session.message`; the PR emits `user-message` only from `chat.send`, so the external-channel ingress path remains uncovered.

## Solution Assessment

No. The proposed patch is not the best solution as written because it creates a new global chat event carrying raw user text, while the repository already has a session-scoped transcript event contract designed for this exact class of update.
