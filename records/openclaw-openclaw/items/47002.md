## Summary

Keep open: current main and the latest release still do not accept the requested `mediaLocalRoots` config keys, while the existing supported path is a narrower `tools.fs` read-policy model. The remaining work is a security-sensitive product/config decision rather than an already-implemented bug fix.

## What This Changes

The issue requests a public configuration path for extra outbound local-media roots, especially Telegram or agent-default `mediaLocalRoots`, so external local files can be uploaded without staging them under OpenClaw-managed directories.

## Best Possible Solution

Choose one canonical local-media trust contract: either document the current `tools.fs` read-policy path as intentional, or add a narrow shared, validated root allowlist with schema, docs, tests, and security review.

## Reproduction Assessment

Yes, source-level: current strict agent-default and Telegram schemas omit `mediaLocalRoots`, while bundled channel validation rejects unknown channel keys. I did not run a live config command in this read-only review.

## Solution Assessment

No for the exact proposed keys until maintainers choose the security contract. Current main already supports a narrower policy-based path for host-local outbound media when the agent is allowed to read the file, so a new allowlist needs a shared design rather than an ad hoc schema addition.
