## Summary

Keep this open. Current main still has no accepted `mediaLocalRoots` config surface for Telegram or agent defaults, and the shared outbound media resolver does not read channel/account configured roots; however, the fix is a filesystem-permission/config-product decision rather than a narrow already-defined bug fix.

## What This Changes

The issue asks OpenClaw to accept configurable outbound local media roots for Telegram or agent defaults so local files outside built-in roots can be sent.

## Best Possible Solution

Define one canonical operator-controlled filesystem/media-root policy, update schemas/types/docs, thread it through the shared outbound media-access resolver, and verify with focused config/media tests plus redacted real Telegram or Feishu send proof.

## Reproduction Assessment

Yes, source-level. Current main’s strict Telegram and agent-default schemas omit `mediaLocalRoots`, and the active outbound resolver builds roots from defaults, agent workspace, and media-source expansion rather than a configured root list.

## Solution Assessment

Unclear as proposed. Accepting the key in several ad hoc locations is not obviously the best fix; maintainers should choose a single filesystem/media-root policy surface and wire that through the shared resolver.
