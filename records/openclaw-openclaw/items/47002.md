## Summary

Keep open. Current main still rejects the reported persistent `mediaLocalRoots` config locations, and the remaining fix changes the host-local file read/send permission boundary, so it belongs in the broader shared filesystem/media policy rather than a Telegram-only schema exception.

## What This Changes

Requests a durable config surface for operator-defined outbound local-media roots so Telegram can send local files outside OpenClaw-managed directories.

## Best Possible Solution

Ship a shared, schema-backed filesystem/media root allowlist with docs, generated config help, and regression coverage across local media loading, outbound delivery, prompt images, and channel adapters.

## Reproduction Assessment

Yes. Source inspection on current main shows strict Telegram and agent-defaults schemas omit `mediaLocalRoots`, while Telegram outbound still forwards roots when supplied by runtime code.

## Solution Assessment

No for a Telegram-only schema exception. The narrow maintainable direction is a shared filesystem/media root policy, such as the open #52951 direction, because this is a host filesystem permission boundary.
