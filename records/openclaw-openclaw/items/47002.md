## Summary

Keep open: current main has fixed several Telegram/media-root propagation paths, but it still has no supported config surface for additional outbound local media roots, and adding one is a filesystem-access product/security decision.

## What This Changes

## Best Possible Solution

Define one canonical scoped local-media access contract, align it with filesystem policy, then wire Telegram outbound through schema, docs, validation, and regression tests.

## Solution Assessment

No, simply adding a Telegram-only schema key is not the best fix because this needs a canonical scoped filesystem/media-root contract that avoids duplicating or weakening existing tool FS policy.
