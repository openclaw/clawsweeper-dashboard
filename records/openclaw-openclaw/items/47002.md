- `impact:security`: The requested configuration would expand trusted host-filesystem roots that outbound delivery may read and upload.

## Summary

Keep open. Current main and the latest release still reject `mediaLocalRoots` under strict Telegram and agent-default config schemas; current code only supports default/agent roots plus source-parent expansion when host media reads are already allowed, while the broader explicit-roots design is still an open product/security thread.

## What This Changes

The issue asks for a persistent outbound media root config contract so Telegram can send local media outside OpenClaw-managed directories.

## Best Possible Solution

Choose one canonical explicit local-media root contract, preferably aligned with the broader filesystem-root policy, then validate narrow roots, route them through shared outbound media access, document the contract, and cover schema plus delivery behavior with focused tests.

## Reproduction Assessment

Yes, at source level. The strict Telegram and agent-default schemas omit `mediaLocalRoots`, and the latest release tag shows the same schema gap; I did not run a mutating config-parse repro in the read-only checkout.

## Solution Assessment

Unclear. A persistent root allowlist is useful, but adding Telegram-only `mediaLocalRoots` may be the wrong shape if `tools.fs.roots` becomes the canonical filesystem/media access contract.
