- `P2`: The report is a valid user-facing outbound media configuration gap with limited blast radius, but the safe fix requires product/security review.
- `impact:security`: The requested config would expand trusted host-filesystem roots that outbound media delivery may read and upload.
- `impact:message-loss`: When local media is outside allowed roots, Telegram outbound media delivery can fail instead of sending the intended attachment.

## Summary

Keep open. Current main and the latest shipped tag still do not expose a validated `mediaLocalRoots` config for Telegram or agent defaults; they only carry managed roots, agent workspace roots, and conditional concrete-source widening, so the requested operator-configured media-root policy still needs maintainer product/security review.

## What This Changes

The issue asks OpenClaw to accept configured outbound media local roots for Telegram or agent defaults so Telegram can send local files outside managed roots.

## Best Possible Solution

Choose one canonical explicit local-media root contract, validate and normalize narrow roots, route it through shared outbound media access and Telegram delivery, document it, and cover schema plus delivery behavior with focused tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: strict Telegram and agent-default schemas omit `mediaLocalRoots`, while the shared outbound resolver has no configured-root merge point.

## Solution Assessment

Unclear. Accepting a narrowly scoped `mediaLocalRoots` key may be useful, but maintainers should first choose whether the canonical contract is channel-specific media roots, agent defaults, or the broader `tools.fs.roots` model.
