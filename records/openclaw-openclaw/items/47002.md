## Summary

Keep open. Current main and the latest release still do not expose a validated persistent local-media root config for Telegram or agent defaults, and the closest active path is the broader tools.fs.roots product/security decision rather than a cleanup close.

## What This Changes

The issue asks OpenClaw to accept a documented config surface for explicit outbound media local roots so Telegram can send local files outside OpenClaw-managed directories.

## Best Possible Solution

Choose one canonical narrow host-root contract, likely through the active tools.fs.roots design or an explicitly approved media-root variant, then wire schema, docs, media loading, outbound delivery, and Telegram coverage through that single policy.

## Reproduction Assessment

Yes, at source level. The strict Telegram and agent-default schemas omit mediaLocalRoots, so adding that key remains rejected on current main and in v2026.5.20 without running a mutating config-parse command.

## Solution Assessment

Unclear. A narrow persistent root allowlist is useful, but maintainers need to decide whether the correct surface is the active tools.fs.roots contract or a media-specific config key before implementation.
