## Summary

Keep open. Current main still rejects the reported persistent `mediaLocalRoots` config locations and has no shared `tools.fs.roots` allowlist; runtime paths can consume roots and current fs-policy expansion helps some explicit media sends, but the durable allowlist belongs in the broader security-reviewed filesystem/media policy tracked by #52951.

## What This Changes

Request that OpenClaw accept a persistent local-media root allowlist for Telegram outbound media at `channels.telegram`, `agents.defaults`, or an equivalent shared config surface.

## Best Possible Solution

Ship a shared, schema-backed filesystem/media root allowlist with docs, generated config help, and regression coverage across local media loading, outbound delivery, prompt images, and channel adapters.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: strict current schemas omit `mediaLocalRoots` at both reported config locations, while Telegram outbound still forwards roots when it receives them.

## Solution Assessment

No for a Telegram-only schema exception. The narrow maintainable direction is a shared filesystem/media root policy that consistently governs host-local reads and outbound sends.
