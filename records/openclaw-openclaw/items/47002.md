## Summary

Keep open. Current main still has no public `mediaLocalRoots` config contract for Telegram or `agents.defaults`, and the remaining work would add a trusted host-filesystem root surface that needs maintainer product/security review.

## What This Changes

The issue asks OpenClaw to accept and apply configurable `mediaLocalRoots` for Telegram outbound media and agent defaults instead of rejecting the key during config validation.

## Best Possible Solution

Define one canonical, documented local-media root contract, validate it narrowly, route it through shared outbound media access and Telegram delivery, and cover schema plus delivery behavior with focused tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: strict Telegram and agent-default schemas omit `mediaLocalRoots`, while outbound media access has no configured-root merge point.

## Solution Assessment

Unclear. Accepting `mediaLocalRoots` may be part of the answer, but maintainers should first choose the canonical filesystem/media-root contract so Telegram, generic outbound delivery, and broader tool media access do not diverge.
