Keep open. Current main still does not expose a `mediaLocalRoots` config key for Telegram or agent defaults, and the related open PR is still an incomplete implementation path rather than a merged fix.

## What This Changes

The issue asks for configurable outbound media local roots to be accepted by config validation and applied to Telegram media sends.

## Best Possible Solution

Settle the canonical root config surface, then update schema/types/docs and thread the chosen roots through the shared outbound media-access resolver with focused Telegram coverage.

## Reproduction Assessment

Yes, source-level. Current main's strict Telegram and agent-default schemas omit `mediaLocalRoots`, so the reported config keys are still rejected by the validation contract.

## Solution Assessment

Unclear. A narrow Telegram-only `mediaLocalRoots` key may work, but the better maintainable path likely needs a shared filesystem/media-root policy decision rather than another channel-specific allowlist.
