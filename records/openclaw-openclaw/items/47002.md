## Summary

Keep open. Current main still rejects the requested persistent `mediaLocalRoots` config surface at both Telegram channel config and agent defaults, while Telegram outbound runtime paths can consume roots when core supplies them. The active shared filesystem-root proposal in #52951 is the better path, but it is still open and security-sensitive, so this issue should not be closed or queued as a narrow Telegram-only automated fix.

## What This Changes

The issue requests a schema-backed way to configure additional local roots for outbound Telegram media so files outside OpenClaw-managed directories can be sent without manual staging or symlinks.

## Best Possible Solution

Ship a shared, documented, schema-backed operator allowlist such as `tools.fs.roots` or an equivalent filesystem/media policy surface, with config types, generated schema/help, docs, and regression tests aligned across local media loading, outbound delivery, prompt images, and channel adapters.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: strict current schemas omit `mediaLocalRoots` at both reported config locations, while the Telegram outbound adapter still forwards supplied roots when it receives them.

## Solution Assessment

No, not as a Telegram-only schema exception. The maintainable fix should be a shared filesystem/media root policy reviewed across host FS tools, local media reads, outbound delivery, and channel/plugin adapters.
