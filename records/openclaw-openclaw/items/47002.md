## Summary

Keep open. Current main still has no public `mediaLocalRoots` config contract for Telegram or `agents.defaults`, and the remaining fix expands trusted local filesystem roots, so it needs maintainer product/security judgment rather than cleanup closure.

## What This Changes

Allow operators to configure additional trusted outbound local-media roots for Telegram, agent defaults, or a canonical media-root surface so files outside default OpenClaw roots can be sent.

## Best Possible Solution

Define one canonical, documented local-media root contract, validate it narrowly, route it through shared outbound media access and Telegram delivery, and cover schema plus delivery behavior with focused tests.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction for the validator gap: strict Telegram and agent-default schemas on current main omit `mediaLocalRoots`, so that key is rejected before delivery can use it. I did not run validation because this review is read-only.

## Solution Assessment

Unclear. Accepting `mediaLocalRoots` may be part of the answer, but maintainers should first choose the canonical filesystem/media-root contract so Telegram, generic outbound delivery, and broader tool media access do not diverge.
