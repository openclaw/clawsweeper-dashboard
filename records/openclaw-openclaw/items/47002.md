## Summary

Keep open. Current main still rejects the reported `mediaLocalRoots` config surface for Telegram and agent defaults, and the remaining fix overlaps active outbound-media and filesystem-root policy work rather than a shipped implementation.

## What This Changes

The issue asks OpenClaw to accept configurable outbound media local roots, especially for Telegram, so local files outside OpenClaw-managed directories can be sent without symlink/copy workarounds.

## Best Possible Solution

Define one supported, documented local-media root contract, add schema/types/docs, route it through the shared outbound media-access resolver and Telegram delivery, and cover validator plus delivery behavior with focused tests.

## Reproduction Assessment

Yes, for the reported validator gap: current-main source shows strict Telegram and agent-default schemas omit `mediaLocalRoots`, while the outbound resolver does not read configured roots. I did not run validation because this review was read-only, but the schema path is clear from source.

## Solution Assessment

Unclear. Accepting `mediaLocalRoots` may be a valid direction, but maintainers should first choose the canonical filesystem/media-root contract so Telegram, generic outbound delivery, and broader tool media access do not diverge.
