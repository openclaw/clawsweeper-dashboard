## Summary

Keep open: current main still rejects the requested `mediaLocalRoots` config keys, while the supported host-local media path now runs through narrower `tools.fs` read policy, so the remaining work is a security-sensitive product decision rather than an already-implemented bug fix.

## What This Changes

The issue asks OpenClaw to accept configurable outbound media local roots for Telegram, channel config, or agent defaults so local files outside managed roots can be sent.

## Best Possible Solution

Decide the canonical local-media trust contract first; either document the current `tools.fs` read-policy path as the supported answer or add a narrow validated root allowlist with schema, docs, tests, and security review.

## Reproduction Assessment

Yes, source-level: current strict agent-default and Telegram schemas omit `mediaLocalRoots`, while bundled channel validation reports unknown channel keys. I did not run a live config command in this read-only review.

## Solution Assessment

No for the exact proposed config keys until maintainers choose the security contract; current main already provides a narrower policy-based path for host-local outbound media when filesystem reads are allowed.
