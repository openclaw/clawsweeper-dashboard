## Summary

Keep open. Current main and the latest shipped tag still reject `mediaLocalRoots` under `channels.telegram` and `agents.defaults`; adjacent fixes now cover explicit CLI media paths and agent workspace roots, but not an operator-configured outbound media root contract.

## What This Changes

The issue asks OpenClaw to accept and apply a `mediaLocalRoots` configuration surface for Telegram outbound media or agent defaults so local media outside managed roots can be sent intentionally.

## Best Possible Solution

Choose one canonical explicit local-media root contract, validate and normalize narrow roots, route it through shared outbound media access and Telegram delivery, document it, and cover schema plus delivery behavior with focused tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: strict Telegram and agent-default schemas omit `mediaLocalRoots`, while the shared outbound resolver has no configured-root merge point.

## Solution Assessment

Unclear. Accepting `mediaLocalRoots` may be useful, but maintainers should first choose the canonical local-root contract so Telegram, generic outbound delivery, and broader filesystem policy do not diverge.
