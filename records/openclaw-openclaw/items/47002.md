## Label Justifications

- `P2`: This is a real configuration gap for outbound Telegram media with limited blast radius, but the fix needs product and security review rather than emergency handling.
- `impact:security`: The requested configuration changes which host-local filesystem paths OpenClaw may read and upload through outbound media delivery.
- `impact:message-loss`: Telegram outbound media can fail when an attachment path sits outside the currently allowed roots, so the intended message content is not delivered.

## Summary

Keep open. Current main and v2026.5.22 still do not expose a validated persistent `mediaLocalRoots`-style config for Telegram or `agents.defaults`, and the closest active path is the broader open `tools.fs.roots` product/security design rather than an already-merged cleanup fix.

## What This Changes

Request a validated config surface for additional outbound local-media roots so Telegram can send local files outside OpenClaw-managed directories without symlink or copy workarounds.

## Best Possible Solution

Choose one canonical opt-in host-root policy, likely the shared `tools.fs.roots` path or an explicitly approved media-root variant, then wire schema, types, docs, Telegram outbound delivery, and regression coverage through that single contract.

## Reproduction Assessment

Yes, at source level. The strict Telegram and agent-default schemas omit `mediaLocalRoots` on current main and in v2026.5.22, so the reported config key remains rejected without running a mutating parse command.

## Solution Assessment

Unclear. A narrow allowlist is plausible, but maintainers need to decide whether the durable solution is `tools.fs.roots`, a media-specific root key, or a Telegram-scoped variant before implementation.
