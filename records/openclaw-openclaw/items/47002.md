## Summary

Keep open. Current main still has no public `mediaLocalRoots` config contract for `channels.telegram` or `agents.defaults`, while the shared outbound media resolver still does not merge configured local roots; resolving it would add a security-sensitive host-filesystem allowlist that needs maintainer product/security review.

## What This Changes

Request to accept and apply configurable local-media roots for Telegram outbound media and/or agent defaults so external local files can be sent safely.

## Best Possible Solution

Define one canonical explicit local-media root contract, validate and normalize narrow roots, route it through shared outbound media access and Telegram delivery, document it, and cover schema plus delivery behavior with focused tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: strict Telegram and agent-default schemas omit `mediaLocalRoots`, while the shared outbound resolver has no configured-root merge point.

## Solution Assessment

Unclear. Accepting `mediaLocalRoots` may be useful, but maintainers should first choose the canonical local-root contract so Telegram, generic outbound delivery, and broader filesystem policy do not diverge.
