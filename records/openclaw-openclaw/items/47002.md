## Summary

Keep open. Current main still rejects `mediaLocalRoots` under strict Telegram and agent-default schemas, and the remaining fix is a security/product decision about the canonical explicit-root contract rather than a safe cleanup close.

## What This Changes

The issue requests a validated configuration path for explicit outbound local media roots so Telegram can send local files outside OpenClaw-managed directories.

## Best Possible Solution

Choose one canonical explicit local-media or filesystem-root contract, validate narrow roots, wire it through shared outbound media access, document the supported config, and cover schema plus Telegram delivery behavior with focused tests.

## Solution Assessment

Unclear. A persistent root allowlist is useful, but adding a Telegram-only `mediaLocalRoots` field may be the wrong shape if the broader `tools.fs.roots` contract becomes canonical.

- Configured outbound media roots would expand which host paths OpenClaw may read and upload, so root normalization, filesystem-root refusal, sender/account policy, and tool-policy interactions need explicit security review.

- Current `tools.fs.workspaceOnly=false` behavior helps some explicit local sends, but it is not the same as a narrow persistent allowlist for operator-managed outbound media directories.
