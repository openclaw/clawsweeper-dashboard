## Summary

Keep open. Current main and v2026.5.22 still omit a validated persistent local-media root config for Telegram or agent defaults, and the active related direction is the broader tools.fs.roots security/product design rather than a cleanup-close fix.

## What This Changes

The issue asks OpenClaw to accept a persistent config key for additional outbound local-media roots, especially for Telegram sends from directories outside OpenClaw-managed roots.

## Best Possible Solution

Choose one canonical narrow host-root contract, likely the active tools.fs.roots design or an explicitly approved media-root variant, then wire schema, docs, media loading, outbound delivery, and Telegram coverage through that single policy.

## Reproduction Assessment

Yes, at source level. The strict Telegram and agent-default schemas omit mediaLocalRoots on current main and in v2026.5.22, so adding that key remains rejected without needing a mutating config-parse run.

## Solution Assessment

Unclear. A narrow persistent allowlist is useful, but maintainers need to decide whether the correct surface is tools.fs.roots or a media-specific config key before implementation.
