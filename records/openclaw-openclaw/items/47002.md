## Summary

Keep open. Current main still has no accepted `mediaLocalRoots` config surface for Telegram or agent defaults; runtime media plumbing can consume roots, but the remaining fix is a security-sensitive product/API decision that overlaps the open `tools.fs.roots` proposal.

## What This Changes

Request to make OpenClaw accept and apply configured local-media roots for Telegram outbound media instead of rejecting `mediaLocalRoots` in strict config validation.

## Best Possible Solution

Choose one canonical explicit-root contract, wire schema/runtime/docs/tests around it, and avoid channel-specific drift unless maintainers intentionally choose that scope.

## Reproduction Assessment

Yes, at source level. The strict Telegram and agent-default schemas omit `mediaLocalRoots`, while current runtime roots are managed, agent-scoped, or source-parent-expanded only.

## Solution Assessment

Unclear. Adding a Telegram-only field would address the immediate report, but the safer maintainable path is a maintainer-chosen canonical root contract such as the broader `tools.fs.roots` proposal.
