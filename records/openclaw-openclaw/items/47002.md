## Summary

Keep open. Current main and v2026.5.22 still do not accept a persistent `mediaLocalRoots` config key under Telegram or agent defaults, and the remaining work is a security/product decision between a canonical `tools.fs.roots` contract and a media-specific allowlist.

## What This Changes

The issue requests a validated config surface for additional outbound local-media roots so Telegram can send local files outside OpenClaw-managed directories.

## Best Possible Solution

Adopt one explicit narrow host-root policy and wire it through config schema, docs, media access, outbound delivery, and Telegram tests while preserving current safe defaults.

## Reproduction Assessment

Yes, at source level. The strict Telegram and agent-default schemas omit `mediaLocalRoots`, so adding that key remains rejected on current main and in v2026.5.22 without needing a mutating config-parse run.

## Solution Assessment

Unclear. A narrow allowlist is plausible, but adding Telegram or agent-default `mediaLocalRoots` directly may duplicate the open `tools.fs.roots` design; maintainers should choose the canonical surface first.
