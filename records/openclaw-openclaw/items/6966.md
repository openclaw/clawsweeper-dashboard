## Summary

Keep open. Current main has ordered fallback, model-scoped cooldowns, and cooldown probes, but it still lacks a health-aware model strategy, per-model health store, or automatic rate-limit-driven downgrade plus recovery policy; the external aashari fork is useful prior art, not upstream behavior.

## What This Changes

Request automatic downgrade and later upgrade-back model routing when API rate-limit pressure is detected.

## Best Possible Solution

Define a first-class health-aware model-routing contract in core, with explicit policy selection, persisted recovery state, visible downgrade/recovery events, schema/docs, and regression coverage.

## Solution Assessment

Unclear. The capability likely belongs in core fallback policy, but maintainers still need to choose the routing strategy, config shape, health-state persistence, thresholds, and recovery semantics before implementation.
