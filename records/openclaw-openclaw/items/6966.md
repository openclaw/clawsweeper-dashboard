## Summary

Keep open. Current main has ordered fallback, model-scoped cooldown metadata, and cooldown probes, but it still lacks the requested rate-limit-driven model routing policy: no model strategy config, per-model health store, health-aware selector, or automatic downgrade plus upgrade-back behavior.

## What This Changes

Request automatic model downgrade on API rate-limit spikes and gradual upgrade-back when errors subside.

## Best Possible Solution

Define a first-class health-aware model-routing contract in core, with explicit policy selection, persisted recovery state, visible downgrade/recovery events, schema/docs updates, and regression coverage for rate-limit scenarios.
