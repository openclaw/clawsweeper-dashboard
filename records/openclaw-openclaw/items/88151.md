## Summary

Keep open: current main and the latest release still hardcode the gateway RSS memory-pressure thresholds, and `diagnostics` only exposes the snapshot toggle. The requested operator knob is plausible but needs a maintainer decision on the config contract before implementation.

## What This Changes

Request to make the gateway diagnostics RSS memory-pressure threshold configurable or derived from the Node heap/system capacity instead of a hardcoded 1.5 GiB warning threshold.

## Best Possible Solution

Add a canonical diagnostics memory-pressure threshold contract only after deciding whether thresholds are explicit bytes or derived from heap/system limits, then wire schema, help, docs, hot-reload behavior, and focused tests together.

## Reproduction Assessment

Partly yes: source inspection of current main reproduces the central missing capability because RSS thresholds are hardcoded and no config key exists. I did not establish a live current-main reproduction of the reported fleet cron aborts.

## Solution Assessment

Unclear: a config key is a plausible fix, but maintainers should first choose the threshold model and default behavior because this changes diagnostics and operator tuning surface.
