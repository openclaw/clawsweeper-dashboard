- **[P2] Strip every detected inbound metadata block before capture:** `extensions/memory-lancedb/index.ts:668-675`
  - body: `sanitizeForMemoryCapture` only removes the explicit sentinel list here, but current `buildInboundUserContextPrefix` also emits labels such as `Reply target of current user message (untrusted, for context):`, `Reply chain of current user message (untrusted, nearest first):`, `Location (untrusted metadata):`, and custom structured-context labels. Please make sanitization cover the same current inbound-meta contract as detection and add regression cases for these labels.
