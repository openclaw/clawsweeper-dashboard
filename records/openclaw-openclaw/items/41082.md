## Summary

Keep this issue open. Current main has useful compaction-summary safeguards and context-engine prompt forwarding, but the default `legacy` embedded path still passes assembled history through and submits the fresh prompt without a generic marker that makes the latest post-compaction ask the active task over older summary context.

## What This Changes

Request to reduce post-compaction context-switch lag so a fresh unrelated user prompt is treated as the active task instead of stale compressed context.

## Best Possible Solution

Add a focused embedded/default-session regression with an old-topic compaction summary and a fresh unrelated post-compaction prompt, then make the generic embedded prompt assembly path explicitly preserve latest-user priority while keeping the summary as historical background. Avoid channel-specific keyword heuristics, and document the intended post-compaction semantics.

## Reproduction Assessment

Unclear: the report provides plausible steps and timestamps, and source inspection confirms the default embedded path lacks a deterministic latest-ask marker. I do not have a high-confidence live model reproduction because the wrong-answer symptom depends on model behavior.

## Solution Assessment

No: keyword-only topic-switch detection would be brittle. The best fix is a generic prompt/context assembly contract, proven by a deterministic embedded regression, that treats the incoming prompt as active and compaction summaries as prior context.
