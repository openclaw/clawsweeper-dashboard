## Summary

Keep open. Current main has partial compaction mitigations, but I could not verify that they solve the reported post-compaction topic-switch lag, and no merged fix or canonical issue fully covers this compaction-specific behavior.

## What This Changes

The issue asks OpenClaw to prioritize new messages after compaction so the assistant does not keep answering the old summarized topic when the user switches topics.

## Best Possible Solution

Add a focused latest-message priority invariant for compacted sessions, ideally with a regression fixture that builds a compacted transcript plus an unrelated post-compaction user ask and proves the next prompt steers the model toward that new ask.

## Reproduction Assessment

No. The issue provides plausible timestamps and steps, and the source shows the summary plus post-boundary messages are both presented, but I did not establish a deterministic current-main failing path in this read-only review.

## Solution Assessment

Unclear. The narrowest maintainable direction is probably a compaction/context prompt or transcript freshness invariant, but the proposed topic-switch deweighting behavior needs maintainer product judgment before automation changes it.
