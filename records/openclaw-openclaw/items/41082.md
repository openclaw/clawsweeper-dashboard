## Summary

Keep this open. Current main still does not provide a generic embedded/default-session contract that makes a post-compaction user prompt unambiguously outrank prior compaction summary context: the default legacy engine is pass-through, the embedded runner submits the visible prompt plainly, and only the CLI reseed path has an explicit latest-message wrapper.

## What This Changes

The issue asks OpenClaw to avoid stale post-compaction replies by prioritizing the user's new message when generating the next response.

## Best Possible Solution

Add a generic embedded/default-session prompt assembly contract that treats the incoming post-compaction prompt as the active request and compaction summaries as prior context, backed by deterministic regression coverage.

## Reproduction Assessment

No high-confidence live reproduction was established; the stale-answer symptom depends on model behavior. Source inspection does show the current default embedded path lacks the explicit active-request marker that the CLI reseed path already uses.

## Solution Assessment

Yes for the keep-open direction: a generic prompt/context assembly contract is narrower and more maintainable than keyword topic-switch heuristics or provider-specific behavior.
