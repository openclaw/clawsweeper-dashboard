## Summary

Keep this open. Current main still has the source-level gap from the earlier review: default embedded runs use the legacy pass-through context engine and submit the fresh prompt plainly, while the CLI reseed path has an explicit current-message wrapper.

## What This Changes

The issue asks OpenClaw to reduce post-compaction context-switching lag so fresh user prompts are treated as the active request instead of being overshadowed by older compaction summaries.

## Best Possible Solution

Add a generic embedded/default-session contract that marks the incoming post-compaction prompt as the active request while keeping compaction summaries as prior context.

## Reproduction Assessment

No high-confidence live reproduction was established; the wrong-answer symptom depends on model behavior. Source inspection does show the current default embedded path lacks the explicit latest-ask priority marker that the CLI reseed path already uses.

## Solution Assessment

Yes for the keep-open direction: a generic prompt/context assembly contract is narrower and more maintainable than keyword-based topic-switch heuristics. The repair should avoid provider-, channel-, and plugin-specific policy.

## Repair Work Prompt

Investigate and fix the embedded/default OpenClaw agent path for post-compaction topic changes. The observable problem is that after compaction, the model can over-weight older compaction summary context and answer the old topic instead of the fresh user prompt. Keep the fix generic: do not add provider-, channel-, or keyword-specific topic-switch heuristics; do not change plugin-owned context-engine contracts except through a backward-compatible seam if needed. Likely direction: add a prompt/context assembly marker or equivalent contract for the current embedded prompt, similar in spirit to the CLI reseed `<next_user_message>` wrapper, so compaction summaries remain prior context and the fresh ask is clearly the active request. Add deterministic regression coverage around the embedded runner/context-engine prompt path, and update compaction/context docs plus CHANGELOG.md if behavior changes user-facing semantics.
