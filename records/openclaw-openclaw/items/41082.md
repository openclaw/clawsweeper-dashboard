## Summary

Keep this open. Current main still has the same product gap the earlier ClawSweeper review identified: default embedded runs keep compaction summaries in history and submit the fresh user prompt plainly, while only the CLI reseed path has an explicit latest-message wrapper.

## What This Changes

The issue asks OpenClaw to prevent post-compaction sessions from answering a new topic as if the older compressed topic were still the active request.

## Best Possible Solution

Add a generic embedded/default-session contract that treats the incoming post-compaction prompt as the active request and compaction summaries as prior context, backed by deterministic regression tests and updated docs.

## Reproduction Assessment

No high-confidence live reproduction was established; the wrong-answer symptom depends on model behavior. Source inspection does show the current default embedded path lacks the explicit latest-ask priority marker that the CLI reseed path already uses.

## Solution Assessment

Yes for the keep-open direction: a generic prompt/context assembly contract is narrower and more maintainable than keyword topic-switch heuristics. The repair should avoid provider- or channel-specific behavior.
