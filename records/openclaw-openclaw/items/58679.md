## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Populate provenance on invalid schema snapshots:** `src/config/io.ts:2319-2332`
  - body: When validation fails, this return path builds a snapshot without `provenance`, so `openclaw config explain` falls back to empty entries for schema-invalid configs. That removes include/env/key-source context in one of the main troubleshooting cases this command is meant to cover.
  - confidence: 0.9
- **[P2] Track skipped and ancestor includes as unapplied:** `src/config/io.ts:1865-1869`
  - body: `applied` only checks the terminal failed path and circular ancestors. Because include resolution aborts on the first thrown include error, later siblings in an include array and non-circular parent includes can still be printed as applied even though their merge never completed.
  - confidence: 0.87
- **[P2] Scan include-resolved config for env references:** `src/config/io.ts:1902-1908`
  - body: This scans only the root parsed payload, but env substitution runs after `$include` resolution. A `${VAR}` that exists only inside an included file affects runtime config but will not produce the `env` provenance entry, making `config explain` under-report substitutions.
  - confidence: 0.84
- **[P2] Use neutral status text for unapplied includes:** `src/commands/config-explain.ts:35-37`
  - body: Every unapplied include is rendered as `(not found)`, but the builder also marks entries false for circular chains and other include-resolution failures. That sends operators toward the wrong root cause for parse, security, depth, or ancestor failures.
  - confidence: 0.86
