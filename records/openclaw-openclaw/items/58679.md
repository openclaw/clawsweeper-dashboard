Full review comments:

- **[P2] Populate provenance for invalid snapshots:** `src/config/io.ts:2320`
  - body: When schema validation fails after includes/env resolution, this return path builds a snapshot without `provenance`. `config explain` then falls back to empty provenance for schema-invalid configs, removing the include/env/key-source diagnostics that the command is meant to provide while troubleshooting invalid config files.
  - confidence: 0.9
- **[P2] Detect env refs after includes are resolved:** `src/config/io.ts:1902`
  - body: This traversal only inspects `params.parsed`, and the valid snapshot call passes the root `effectiveParsed` value before `$include` files are merged. Any `${ENV}` reference that exists only in an included config file is substituted by runtime config loading but omitted from `config explain`, so the new env provenance can be false-negative for real configs.
  - confidence: 0.86
- **[P2] Stop labeling every unapplied include as not found:** `src/commands/config-explain.ts:36`
  - body: `applied: false` is used for more than missing files, including circular include chains, parse/security failures, and failed ancestors. Rendering all of those as `(not found)` sends operators toward the wrong root cause when the include was present but rejected or not fully applied.
  - confidence: 0.88
