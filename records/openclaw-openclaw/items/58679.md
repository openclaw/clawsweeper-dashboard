- **[P2] Populate provenance for invalid config snapshots:** `src/config/io.ts:2320-2325`
  - body: The schema-invalid snapshot path does not set `provenance`, so `openclaw config explain` falls back to an empty provenance object for invalid configs. That removes include/env/key-source visibility in one of the main troubleshooting cases this command is meant to serve.
  - confidence: 0.88
- **[P2] Scan include-resolved config for env provenance:** `src/config/io.ts:2347-2351`
  - body: Successful snapshots call `buildConfigProvenance` with the root `effectiveParsed`, not the include-resolved tree that `resolveConfigForRead` processes. If `${ENV}` appears only in an included file, OpenClaw still substitutes it, but `config explain` reports no env provenance.
  - confidence: 0.84
- **[P2] Use accurate status text for unapplied includes:** `src/commands/config-explain.ts:35-37`
  - body: `applied: false` is used for circular chains, ancestor failures, parse/security failures, and missing files, but the CLI renders every case as `(not found)`. That points users at the wrong root cause when diagnosing invalid include trees.
  - confidence: 0.9
