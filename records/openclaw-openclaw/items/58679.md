  - body: The schema-invalid snapshot return omits the new `provenance` field while other paths attach it. `config explain` then reports empty entries for parsed-but-invalid configs, hiding include/env/key-source context during troubleshooting.
  - confidence: 0.9

  - confidence: 0.88
- **[P2] Mark failed include ancestors for non-circular errors:** `src/config/io.ts:2281-2284`
  - body: Only circular include errors populate `failedIncludeAncestors`. If a root include loads a file whose nested include is missing, invalid, or blocked, the root include can still render as applied even though the merge never completed.
  - confidence: 0.84
- **[P2] Render unapplied includes with neutral status text:** `src/commands/config-explain.ts:36`
  - body: The CLI labels every unapplied include as `(not found)`, but `applied: false` also covers circular, parse, security, and ancestor failures. That can point operators at the wrong root cause.
  - confidence: 0.86
- **[P2] Use non-mutating include ordering:** `src/config/io.ts:1853`
  - body: The new `[...includePaths].sort()` trips the repo lint rule that requires `toSorted()`, so `pnpm check` fails before the branch can merge.
  - confidence: 0.92
