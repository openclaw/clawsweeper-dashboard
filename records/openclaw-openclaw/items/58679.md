## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Populate provenance for invalid snapshots:** `src/config/io.ts:2320`
  - body: The validation-failure return builds a `ConfigFileSnapshot` without `provenance`, so `openclaw config explain` falls back to empty provenance for parsed-but-invalid configs. Build the same provenance snapshot here that valid snapshots return.
  - confidence: 0.9
- **[P2] Scan the include-resolved tree for env provenance:** `src/config/io.ts:2350-2351`
  - body: `buildConfigProvenance` scans `effectiveParsed`, but config loading resolves `$include` before applying `${ENV}` substitutions. Env refs that exist only inside included files can be substituted while `config explain` reports no env provenance.
  - confidence: 0.88
- **[P2] Mark failed parent includes on nested failures:** `src/config/io.ts:2275-2284`
  - body: For non-circular nested include failures, only the inner failed path is recorded while root provenance only lists the outer `$include`. That outer include can be reported as applied even though its merge never completed.
  - confidence: 0.84
- **[P2] Use neutral status text for unapplied includes:** `src/commands/config-explain.ts:36`
  - body: The renderer labels every `applied: false` include as `(not found)`, but that flag can also mean circular chains, parse/security failures, or ancestor failures. Use neutral text or carry the actual failure reason.
  - confidence: 0.86
- **[P2] Use non-mutating include path ordering:** `src/config/io.ts:1853`
  - body: CI's `pnpm lint` fails because this new include path ordering uses mutating `Array#sort()`. Switch the copied array to `toSorted()` so the check lane can pass.
  - confidence: 0.92
