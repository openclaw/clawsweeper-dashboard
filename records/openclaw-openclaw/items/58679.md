## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Populate provenance on invalid snapshots:** `src/config/io.ts:2320`
  - body: The schema-validation failure branch creates a snapshot without `provenance`, so `openclaw config explain` reports empty provenance exactly when a parsed config is invalid. Build the same provenance snapshot here that the valid path returns.
  - confidence: 0.9
- **[P2] Scan the include-resolved config for env provenance:** `src/config/io.ts:2350`
  - body: `buildConfigProvenance` scans the pre-include root tree, but config loading resolves `$include` before applying `${ENV}` substitutions. Env refs that exist only inside included files can be substituted while `config explain` reports no env provenance.
  - confidence: 0.88
- **[P2] Mark failed parent includes on nested failures:** `src/config/io.ts:2275-2284`
  - body: For non-circular nested include failures, only the inner failed path is recorded while the root provenance only lists the outer `$include`. That outer include can be reported as applied even though its merge never completed.
  - confidence: 0.84
- **[P2] Use neutral status text for unapplied includes:** `src/commands/config-explain.ts:36`
  - body: The renderer labels every `applied: false` include as `(not found)`, but the flag can also mean circular chains, parse/security failures, or ancestor failures. Use neutral text or carry the actual failure reason.
  - confidence: 0.86
- **[P2] Use toSorted for include path ordering:** `src/config/io.ts:1853`
  - body: CI's `pnpm check` fails because this new include path ordering uses mutating `Array#sort()`, which the repo lint rule rejects. Switch the copied array to `toSorted()` so the PR can pass lint.
  - confidence: 0.92
