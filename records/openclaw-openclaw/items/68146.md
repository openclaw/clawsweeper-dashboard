## What This Changes

The PR adds a Gateway `StartupGateBarrier` that holds startup-gated RPCs until post-attach sidecars finish or a 20 second timeout fires, threads it through request context, and adds focused gateway dispatch tests.

## Best Possible Solution

Refresh the branch onto current main, keep the descriptor-derived startup method set as the source of truth, and land the bounded server-side wait only after exact-head restart proof and maintainer acceptance of the delayed-failure behavior.

Yes. ## Solution Assessment

Conditionally yes. A server-side barrier is the right layer for cached pre-retry browser bundles because client-only retry cannot change an already-loaded old UI bundle, but the branch needs refresh, exact-head proof, and maintainer acceptance of the delayed-failure tradeoff.
