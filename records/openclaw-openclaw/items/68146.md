## What This Changes

The PR adds a Gateway startup barrier that holds startup-gated RPCs until post-attach sidecars open, threads it through request context/startup, and adds focused unit tests.

## Best Possible Solution

Merge a Gateway-owned bounded barrier or equivalent queue after breaking the architecture cycle, preserving the descriptor-derived method set and canonical retry details, and adding current-head restart proof.

No high-confidence end-to-end reproduction was run in this review. Source inspection shows current main still returns immediate retryable `UNAVAILABLE` while startup-gated methods remain in the post-attach set, and the PR body includes historical real logs for the same race.
