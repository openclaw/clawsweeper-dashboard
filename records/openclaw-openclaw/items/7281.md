## Summary

Keep open: current main still lacks configurable post-failure hook delivery failover, and the related closed PR did not merge or supersede this request.

## What This Changes

Request is to add a `fallbackChannel` or ordered `fallbackChannels` hook mapping option when primary hook delivery fails.

## Best Possible Solution

Define an owner-approved ordered hook delivery fallback contract and thread it through config schema/types, mapping transforms, hook dispatch, cron delivery storage, outbound send failover, docs, and focused regression tests.

## Reproduction Assessment

Yes, source-level. Current main rejects the proposed fallback mapping keys, carries only one channel/to route through hook mapping and dispatch, and sends one resolved cron delivery target.

## Solution Assessment

Unclear pending maintainer product direction. The fallback chain is plausible, but it should be a typed ordered contract rather than overloading `channel: "last"`, `bestEffort`, `failureDestination`, or pre-delivery fallback helpers.
