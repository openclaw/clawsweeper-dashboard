## Summary

Current main now addresses the reported WebChat freeze path with bounded transcript tail reads, smaller WebChat history requests, a 100-message render window, and nonblocking chat refresh behavior; the remaining cursor/virtualization ideas are better treated as future older-history browsing enhancements.

## What This Changes

The issue requests bounded, paginated, or virtualized WebChat history loading/rendering so accumulated sessions do not freeze the Control UI.

## Best Possible Solution

Keep the current bounded WebChat history implementation; if users need full older-history browsing, track cursor pagination or virtualization as a separate feature request.

## Solution Assessment

Yes. For this bug, bounded latest-window reads plus capped/nonblocking Control UI rendering is the narrow maintainable fix; pagination or virtualization can be a separate enhancement for browsing older history.
