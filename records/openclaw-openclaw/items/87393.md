## Summary

Keep open: the PR fixes the main empty/missing inferred transcript-file fallback and supplies real Telegram/log proof, but the latest diff also makes parakeet-mlx `--output-dir` authoritative when `--output-format` is omitted, which conflicts with upstream's default `srt` output.

## What This Changes

The PR changes media CLI audio output resolution and tests so recognized Whisper, parakeet, and local-whisper transcript files are authoritative instead of falling back to stdout when inferred `.txt` output is empty or missing.

## Best Possible Solution

Keep authoritative file behavior for known txt-producing paths, but only mark parakeet-mlx authoritative when txt output is explicitly requested or OpenClaw itself adds the txt output flag; preserve stdout fallback for upstream defaults.
