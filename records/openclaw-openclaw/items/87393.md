## Summary

Keep open for maintainer review. The patch fixes the current-main CLI stdout fallback bug with focused tests and sufficient real behavior proof, but it intentionally changes fallback behavior for deterministic file-output STT setups and adds a core local-whisper wrapper exception that maintainers need to accept before merge.

## What This Changes

The PR makes recognized CLI audio transcript-file outputs authoritative, adds regression coverage for empty or missing transcript files, and updates media docs for parakeet output-format behavior.

## Best Possible Solution

Land the file-authoritative media runner fix after explicit maintainer acceptance of the compatibility behavior, or narrow/remove the local-whisper exception if core should not recognize that wrapper.

## Reproduction Assessment

Yes. Current main and v2026.6.6 read inferred transcript files only when non-empty and otherwise fall through to stdout, so an audio CLI that writes an empty inferred file and progress stdout can inject progress text.

## Solution Assessment

Mostly yes. Making known deterministic transcript-file outputs authoritative is the narrow repair for progress stdout contamination; the remaining question is whether maintainers want the path-based local-whisper wrapper exception in core.
