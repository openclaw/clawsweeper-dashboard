## What This Changes

The PR makes deterministic CLI audio transcript files authoritative for whisper-cli, Python whisper, explicit-txt parakeet-mlx, and a local-whisper node wrapper, with regression tests and docs updates.

## Best Possible Solution

Land the file-output media runner fix after maintainer acceptance of the compatibility behavior, or narrow it to known CLIs and track custom wrapper output policy separately.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.6.8 shows an inferred transcript file is only used when non-empty and otherwise stdout is returned, so empty or missing file output plus progress stdout is source-reproducible.

## Solution Assessment

Mostly yes. Making deterministic transcript-file outputs authoritative is the narrow fix for progress stdout contamination; the safer alternative is to remove the wrapper-specific path match if maintainers want custom wrappers handled by a generic output-source contract.
