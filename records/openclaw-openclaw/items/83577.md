## Summary

Keep open. Current main and v2026.5.12 still contain the reported collect-mode false positive and drain-wide force-individual state, while the broader lifecycle-notification issue does not supersede this narrower bug fix.

## What This Changes

The issue asks OpenClaw to preserve collect-mode batching for same-route subagent announce queues when one queued item has no resolvable origin route.

## Best Possible Solution

Patch the existing collect queue contract so unresolved-origin non-cross items can join a single resolved-route batch, while explicit cross-channel and runtime-context items still drain individually with regression coverage.

## Reproduction Assessment

Yes for source-level reproduction: current main classifies a keyed-plus-unkeyed collect queue as cross-channel and keeps force-individual collect state across the drain. I did not run the full live multi-subagent macOS scenario.

## Solution Assessment

Yes, the narrow two-part fix fits the existing queue contract better than waiting for the broader lifecycle-notification redesign. The patch should add regression tests for keyed-plus-unkeyed batching and preserve explicit cross-channel/runtime-context individual draining.

Fix the collect-mode queue bug reported in https://github.com/openclaw/openclaw/issues/83577.
