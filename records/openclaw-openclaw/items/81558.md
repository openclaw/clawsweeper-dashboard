## Summary

Keep open: current main still has a real CLI context-engine lifecycle gap. CLI turns are persisted to OpenClaw JSONL and may run compaction maintenance, but they do not run the documented post-turn `afterTurn`/`ingestBatch`/`ingest` lifecycle or `reason: "turn"` maintenance that context-engine plugins such as Lossless Claw rely on.

## What This Changes

The issue requests wiring claude-cli runs into context-engine/LCM post-turn finalization so CLI-routed turns feed memory and recovery context.

## Best Possible Solution

Review and repair https://github.com/openclaw/openclaw/pull/76279, or land an equivalent focused fix that runs the documented post-turn context-engine lifecycle for CLI turns after transcript persistence while preserving existing CLI compaction and reseed behavior.

## Reproduction Assessment

Yes, source-reproducible with high confidence: current main lacks CLI post-turn `afterTurn`/ingest/turn-maintenance calls while embedded/harness paths have them. I did not run a live Telegram/LCM setup in this read-only review.

## Solution Assessment

Yes: the maintainable fix is to mirror the shared harness finalization pattern for CLI turns, including `afterTurn` first and `ingestBatch`/`ingest` fallback, then `reason: "turn"` maintenance on success.
