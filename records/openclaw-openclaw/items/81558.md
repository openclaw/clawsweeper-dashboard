## Summary

Keep open: current main still has the context-engine turn lifecycle wired for embedded/Codex harnesses, but the CLI runner path has no contextEngine parameter, no finalizer call, and no maintenance scheduling. The related missing-transcript issue is adjacent, not a duplicate, because this report identifies a separate LCM ingestion gap for successful claude-cli turns.

## What This Changes

The issue requests that claude-cli routed turns run the same context-engine after-turn ingestion and maintenance lifecycle as embedded provider turns.

## Best Possible Solution

Wire successful CLI runs through the shared context-engine turn finalization and maintenance lifecycle, using the persisted session messages when available and keeping transcript writes single-owned.

Yes. ## Solution Assessment

Yes in direction. The maintainable fix should reuse the shared harness context-engine lifecycle around CLI turn completion rather than adding dist-only patches or duplicating persistence logic in each caller.

## Repair Work Prompt

Fix the claude-cli/CLI backend context-engine lifecycle gap reported in https://github.com/openclaw/openclaw/issues/81558. On current main, runCliAgent and the CLI auto-reply/cron call paths do not pass or resolve a context engine and never call finalizeHarnessContextEngineTurn/runContextEngineMaintenance after a successful CLI turn. Implement a narrow fix that makes successful CLI turns feed the completed user+assistant turn into the active context engine via afterTurn or ingest fallback and schedule turn maintenance, matching the embedded/Codex harness contract. Avoid duplicate session JSONL appends: agent-command already persists CLI transcripts, while auto-reply/cron direct runCliAgent paths may need a canonical persisted snapshot before finalization. Do not add a config option or change provider routing policy. Add focused tests for a CLI-routed turn with a mock context engine asserting afterTurn or ingestBatch and maintenance are invoked, plus coverage for the auto-reply CLI path if the fix touches it. Keep https://github.com/openclaw/openclaw/issues/77974 separate; do not solve the missing-transcript retry race in this PR except as needed to preserve existing behavior. Add a changelog entry if the final patch is user-facing.
