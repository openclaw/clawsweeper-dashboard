Adds optional `mcp.servers.<name>.toolsListTimeoutMs`, threads it through MCP transport resolution, applies it to catalog `client.listTools` calls only when configured, and updates schema/docs/generated baseline plus focused tests.

## Solution Assessment

Yes. A separate opt-in `toolsListTimeoutMs` is the narrowest merge-safe fix because it bounds hung catalog listing without changing the documented connection-only meaning of `connectionTimeoutMs`.
