## Summary

Keep open: current main still misses the doctor migration for legacy top-level plugin `tools`, and the linked open fix PR already references this issue with closing syntax, so the issue should wait for that PR to merge or be closed.

## What This Changes

`openclaw doctor --fix` should migrate legacy top-level plugin `tools` into `contracts.tools` and remove duplicate top-level `tools` when `contracts.tools` already exists.

## Best Possible Solution

Review and land the focused fix in https://github.com/openclaw/openclaw/pull/81112, then close this issue after the migration reaches main.

## Reproduction Assessment

Yes. Current main omits `tools` from doctor’s legacy manifest contract migration list, while runtime registration rejects plugin tools unless they are declared under `contracts.tools`.

## Solution Assessment

Yes. Adding `tools` to the existing legacy manifest contract migration list is the narrow maintainable fix and keeps the stricter runtime contract enforcement intact.
