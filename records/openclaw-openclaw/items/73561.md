## Summary

Keep open: the PR addresses a real static-hint problem and fixes earlier coverage/formatting gaps, but the latest patch can suppress heartbeat-bleed guidance by applying the 200k default in a shared context-window resolver, and its changelog entry conflicts with current main.

## What This Changes

The PR replaces the static 20000 context-overflow hint with context-window-aware `reserveTokensFloor` tiers, adds helper/fallback tests, and adds a changelog entry.

## Best Possible Solution

Land a rebased version that keeps the context-aware reset hint, scopes the default 200k fallback to the reset-hint path, preserves heartbeat-bleed diagnostics, and places the changelog entry in the active fixes section.

## Reproduction Assessment

Yes. Current main reaches the static `20000 or higher` hint through `buildContextOverflowRecoveryText()` without heartbeat evidence; the PR regression can be reproduced with a heartbeat-matching runtime model whose context is unknown and a primary model with a known context below 200k.

## Solution Assessment

No as written. The context-aware reset hint is the right narrow direction, but the default-context fallback should not participate in heartbeat window comparisons, and the branch needs a changelog rebase before merge.
