## Summary

Keep open. The PR targets a real current-main user-facing bug, but the latest head is unmergeable and the new default-context fallback can suppress the existing heartbeat-model-bleed diagnostic in an unknown-runtime-window case.

## What This Changes

The PR replaces the static context-overflow `reserveTokensFloor` hint with context-window-aware tiers, adds focused fallback/tier tests, and adds a changelog entry.

## Best Possible Solution

Rebase onto current main, keep the context-aware generic reset hint, but scope the `DEFAULT_CONTEXT_TOKENS` fallback so heartbeat-bleed detection still treats missing runtime metadata as unknown.

## Reproduction Assessment

Yes. Current main reproduces the reported static `20000 or higher` hint through `buildContextOverflowRecoveryText()` without heartbeat evidence, and the PR regression is source-reproducible when a configured heartbeat model matches the active session but lacks runtime context metadata while the primary model has a known window at or below 200k.

## Solution Assessment

No, not as currently proposed. The tiered hint direction is narrow and maintainable, but the fallback should be applied only when building the generic reserve-floor recommendation, not when deciding whether heartbeat model bleed should be diagnosed.
