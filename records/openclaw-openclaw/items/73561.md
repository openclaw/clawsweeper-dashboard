## Summary

Keep open. The PR is still useful, and the latest force-push fixed the earlier locale-formatting and direct tier-test gaps, but the fallback path can still emit the old 20000 hint when model metadata is unavailable because the session reset clears `contextTokens` before the hint is built and the new resolver call has no `DEFAULT_CONTEXT_TOKENS` fallback.

## What This Changes

This PR changes the context-overflow reset hint to compute tiered `reserveTokensFloor` recommendations from model/session context, adds tests for the tier helper and session fallback, and adds a changelog entry.

## Best Possible Solution

Keep the context-aware hint direction, but compute it from the same pre-reset session or default context budget used by runtime accounting, then make the changelog describe the context-window-aware guidance actually being introduced.

## Reproduction Assessment

Yes. On the PR head, pass an overflow recovery path with unavailable model metadata after `resetReplyRunSession()` has cleared `activeSessionEntry.contextTokens`, or directly call `buildContextOverflowRecoveryText()` with no session context and empty model metadata; the new resolver produces undefined and the hint still says 20000.

## Solution Assessment

No. The direction is maintainable, but the mergeable solution should preserve or explicitly pass a fallback context budget such as `DEFAULT_CONTEXT_TOKENS` before defaulting to the 20k tier.
