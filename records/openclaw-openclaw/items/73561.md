## Summary

Keep open; the PR targets a real user-facing hint problem, but the latest patch can still recommend 20000 for large-context sessions when model metadata is not already resolved.

## What This Changes

The PR changes the context-overflow recovery hint from a static 20000 recommendation to tiered values based on the resolved primary model context window and adds a changelog entry.

## Best Possible Solution

Keep the context-aware hint direction, but compute it from the same active-session/default fallback source used by runtime accounting and lock the tier boundaries with focused tests.

## Reproduction Assessment

Yes. On the PR head, call `buildContextOverflowRecoveryText()` with empty model metadata and `activeSessionEntry.contextTokens: 200000`; the resolver-only path returns undefined and the hint falls back to 20000 instead of the 50k tier.

## Solution Assessment

No. The tier policy is reasonable, but the implementation should seed the hint from `activeSessionEntry.contextTokens` or `DEFAULT_CONTEXT_TOKENS` before dropping to the 20k default.
