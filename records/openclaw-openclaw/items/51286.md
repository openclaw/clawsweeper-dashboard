## Summary

Keep open: this PR is a narrow, meaningful half-fix for the Telegram elevated diagnostics bug, but it is not merge-ready because current main still needs static bundled-channel resolution after the key is scoped and the external PR lacks real behavior proof.

## What This Changes

The branch wraps non-agent-scoped colon session keys in `openclaw sandbox explain` with the current agent ID and adds regression tests for raw Telegram slash keys and already agent-scoped keys.

## Best Possible Solution

Land one reconciled diagnostic fix that scopes raw Telegram slash keys, resolves bundled channel IDs without requiring a loaded plugin registry, preserves existing `global` and already-agent-scoped behavior, and proves the output with redacted CLI or Telegram evidence.

## Reproduction Assessment

Yes, by source inspection: current main leaves `telegram:slash:<id>` unscoped, the agent-session parser rejects it, and elevated diagnostics keep `allowedByConfig` false when channel identity is undefined. I did not run live Telegram or local tests in this read-only review.

## Solution Assessment

No as a standalone PR. Raw-key wrapping is necessary, but it must be paired with static bundled-channel resolution and real after-fix proof before it is the best maintainable fix.
