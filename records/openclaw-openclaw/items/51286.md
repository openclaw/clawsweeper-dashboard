## Summary

Keep open: the branch fixes the raw session-key normalization half of the bug, but it still needs bundled-channel fallback coverage and after-fix real behavior proof before merge.

## What This Changes

The PR wraps non-agent-scoped colon session keys in `openclaw sandbox explain` and adds regression assertions for Telegram slash session channel and allowFrom resolution.

## Best Possible Solution

Land one reconciled fix that normalizes raw channel-prefixed session keys, falls back to static bundled channel IDs when plugins are unloaded, covers the elevated diagnostic output, and includes redacted real CLI or Telegram proof.

## Reproduction Assessment

Yes, source-level. Current main leaves `telegram:slash:<id>` unscoped, `parseAgentSessionKey` rejects it, and `allowedByConfig` stays false when channel identity is unresolved.

## Solution Assessment

No. Wrapping raw keys is necessary, but this branch is not sufficient because channel inference still depends on the runtime plugin registry instead of falling back to bundled channel IDs.
