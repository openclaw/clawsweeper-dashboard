## Summary

Keep this PR open, but it is not merge-ready. It carries the useful session-key normalization half of the reported Telegram slash elevated bug, but current code evidence shows channel inference still depends on loaded plugin registry state, and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR wraps colon-containing non-agent-scoped `sandbox explain` session keys with `buildAgentMainSessionKey` and adds tests for raw Telegram slash keys plus already agent-scoped keys.

## Best Possible Solution

Land one reconciled fix that normalizes raw channel session keys, falls back to bundled channel ids when plugins are unloaded, adds no-registry regression coverage, and includes redacted real CLI or Telegram proof.

## Reproduction Assessment

Yes, source-level. Current `main` leaves `telegram:slash:<id>` unscoped, and this PR would reach an agent-scoped key but still relies on registry-only channel inference in a CLI path that normally does not load plugins.

## Solution Assessment

No. Key normalization is necessary, but the maintainable fix also needs bundled-channel fallback such as `normalizeChatChannelId`, or an explicit plugin-loading strategy, with no-registry regression coverage.
