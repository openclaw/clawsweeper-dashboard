## Summary

Keep open: this PR fixes one necessary normalization step, but it still leaves channel inference dependent on an initialized plugin registry and lacks required after-fix real behavior proof.

## What This Changes

The branch wraps non-agent-scoped colon session keys in `sandbox explain` as agent-scoped keys and adds tests for raw Telegram slash keys plus already agent-scoped keys.

## Best Possible Solution

Land one reconciled fix that normalizes raw channel-prefixed session keys, resolves bundled channel IDs without requiring a loaded plugin registry, and proves the Telegram elevated diagnostic path with focused regression coverage plus redacted real behavior output.

## Reproduction Assessment

Yes, source-level. Current main leaves `telegram:slash:<id>` unscoped, `parseAgentSessionKey` only accepts `agent:<id>:...`, and unresolved channel identity keeps `allowedByConfig` false; I did not run a live Telegram gateway flow in this read-only review.

## Solution Assessment

No. Key wrapping is necessary, but this PR is not sufficient because it still relies on `normalizeAnyChannelId` in a CLI path that normally has no loaded channel plugin registry; the safer fix adds bundled-channel fallback or explicit plugin loading with no-registry coverage.
