## Summary

Keep open: this PR contains useful session-key normalization work, but it is not sufficient by itself because channel inference still depends on an initialized plugin registry, and the external PR still lacks required after-fix real behavior proof.

## What This Changes

The PR wraps non-agent-scoped colon session keys in `sandbox explain` and adds regression tests for Telegram slash-session elevated diagnostics.

## Best Possible Solution

Land one reconciled fix that normalizes raw channel session keys, falls back to bundled channel ids when plugins are unloaded, adds no-registry regression coverage, and includes redacted real CLI or Telegram proof.

## Reproduction Assessment

Yes, source-level. Current `main` leaves `telegram:slash:<id>` unscoped, and this PR would reach an agent-scoped key but still relies on registry-only channel inference in a CLI path that normally does not load plugins; I did not run a live Telegram gateway flow in this read-only review.

## Solution Assessment

No. Key normalization is necessary, but the maintainable fix also needs bundled-channel fallback such as `normalizeChatChannelId`, or an explicit plugin-loading strategy, with no-registry regression coverage.
