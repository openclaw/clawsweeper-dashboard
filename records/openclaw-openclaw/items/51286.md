## What This Changes

This PR changes `sandbox explain` to wrap non-agent-scoped colon session keys into `agent:<id>:...` form and adds regression tests for Telegram slash/elevated output.

## Best Possible Solution

Land one reconciled diagnostic fix that canonicalizes raw channel-prefixed keys with the existing session-key helper, resolves bundled channel IDs without loaded plugins, and proves the Telegram allowFrom output after the patch.

## Reproduction Assessment

Yes, source-level: current main returns bare colon keys unchanged, then channel inference rejects non-agent-scoped keys and `allowedByConfig` remains false without a channel. I did not run live Telegram or CLI proof in this read-only review.

## Solution Assessment

No as a standalone PR. Wrapping raw keys is necessary, but the fix also needs bundled-channel fallback when plugins are unloaded and should preserve existing raw session-key canonicalization semantics.

  - confidence: 0.84
