## Summary

Keep this PR open. Current main still has the reported sandbox explain failure path, but the PR branch is not the complete fix by itself because channel inference still depends on active plugin registration unless the related built-in channel fallback is included.

## What This Changes

The PR normalizes non-agent-scoped colon session keys passed to `openclaw sandbox explain` and adds regression tests for Telegram slash-session elevated diagnostics.

## Best Possible Solution

Land one coherent sandbox explain fix that wraps non-agent-scoped colon keys, resolves bundled channel IDs without requiring an active plugin registry, preserves already agent-scoped keys and `global`, and keeps end-to-end assertions for `sessionKey`, `elevated.channel`, and `allowedByConfig`.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: the documented `telegram:slash:<id>` input remains unscoped on current main, `parseAgentSessionKey` rejects it, and elevated diagnostics cannot resolve the Telegram channel.

## Solution Assessment

No. Session-key normalization is necessary, but this branch should also include the static bundled channel fallback from #51584 or an equivalent resolver change before it reliably fixes CLI diagnostics.
