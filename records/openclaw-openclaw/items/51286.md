## Summary

Keep this PR open, but it is not merge-ready: it fixes the raw colon-key normalization half of the Telegram slash-session diagnostic bug, while channel inference still depends on a loaded plugin registry and can remain `(unknown)` in the CLI path.

## What This Changes

The PR changes `openclaw sandbox explain` to wrap raw colon session keys such as `telegram:slash:<id>` into agent-scoped session keys and adds regression assertions for session key, channel, and `allowedByConfig` output.

## Best Possible Solution

Land one reconciled fix that both normalizes raw channel-prefixed session keys and falls back to static bundled channel IDs when plugins are unloaded, with regression coverage and redacted real behavior proof.

## Reproduction Assessment

Yes, source-level. Current main leaves `telegram:slash:<id>` unscoped, `parseAgentSessionKey` rejects it, and unresolved channel identity keeps `allowedByConfig` false.

## Solution Assessment

No. Wrapping raw keys is necessary, but this branch is not sufficient because channel inference still relies on `normalizeAnyChannelId` without the static bundled-channel fallback needed when CLI plugins are not loaded.
