## Summary

Keep this PR open: it contains a useful raw session-key normalization fix, but it is not ready to merge because it lacks required real behavior proof and still needs reconciliation with the open bundled-channel fallback fix for the same reported path.

## What This Changes

The PR changes `sandbox explain` so bare colon session keys like `telegram:slash:200` are wrapped into agent-scoped form, with regression tests for the normalized key and elevated diagnostic fields.

## Best Possible Solution

Land one reconciled `sandbox explain` fix that normalizes raw channel-prefixed session keys, falls back to bundled channel IDs when plugins are not loaded, keeps focused regression coverage, and includes redacted real output proof.

## Reproduction Assessment

Yes, by source inspection. Current main leaves `telegram:slash:<id>` unscoped, `parseAgentSessionKey` requires an `agent:` prefix, and unresolved channel identity keeps `allowedByConfig` false.

## Solution Assessment

No. This PR fixes the raw-key normalization half and adds useful assertions, but the maintainable fix should also include the bundled-channel fallback from https://github.com/openclaw/openclaw/pull/51584 and real behavior proof.
