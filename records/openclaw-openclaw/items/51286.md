The PR wraps raw colon-containing sandbox explain session keys such as `telegram:slash:<id>` into agent-scoped keys unless they already parse as agent-scoped, and adds regression tests for normalized session key, channel, and `allowedByConfig` output.

## Best Possible Solution

Land one reconciled fix for the `sandbox explain` diagnostic path that both canonicalizes raw channel-prefixed session keys and resolves bundled channel IDs reliably when plugins are not loaded, with focused regression coverage and redacted real CLI or Telegram proof.

## Solution Assessment
