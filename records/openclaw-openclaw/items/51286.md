## Summary

Keep open: this PR contains the session-key normalization half of the fix, but it still depends on registry-backed channel resolution, so the real no-plugin `sandbox explain` CLI path can still report an unknown channel.

## What This Changes

The branch changes `sandbox explain` to wrap non-agent-scoped colon session keys as agent-scoped keys and adds tests for bare Telegram slash keys plus already agent-scoped passthrough.

## Best Possible Solution

Land one consolidated sandbox-explain fix that keeps the non-agent colon-key normalization, adds registry-independent bundled-channel fallback, and covers the no-registry CLI path with focused regression tests.

## Reproduction Assessment

Yes, by source inspection. Current main leaves `telegram:slash:<id>` unscoped, and this PR still relies on `normalizeAnyChannelId` after wrapping, while the CLI path does not load channel plugins.

## Solution Assessment

No. The key normalization is necessary, but the best fix also needs registry-independent channel resolution and a test that does not inherit the shared default Telegram test registry.
