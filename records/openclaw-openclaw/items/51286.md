## Summary

Keep this PR open, but do not merge it as-is: it fixes the bare session-key normalization half of the reported bug, while channel inference still depends on the active plugin registry in a CLI path that normally does not load plugins, and the external PR still lacks after-fix real behavior proof.

## What This Changes

This PR wraps non-agent-scoped colon session keys in `sandbox explain` and adds tests for Telegram slash-session normalization and elevated diagnostics.

## Best Possible Solution

Land one reconciled fix that normalizes raw channel session keys, falls back to bundled channel ids when plugins are unloaded, adds focused regression coverage, and includes redacted real CLI or Telegram proof.

## Reproduction Assessment

Yes, source-level: current `main` leaves `telegram:slash:<id>` unscoped, and this PR reaches an agent-scoped key but still relies on registry-only channel inference in a no-plugin CLI path. I did not run a live Telegram gateway flow in this read-only review.

## Solution Assessment

No. Key normalization is necessary, but the maintainable fix also needs bundled-channel fallback such as `normalizeChatChannelId`, or an explicit plugin-loading strategy, with no-registry regression coverage.
