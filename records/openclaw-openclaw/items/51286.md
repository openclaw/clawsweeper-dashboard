## Summary

Keep open. The normalization is a useful half of the fix, but this PR still leaves `sandbox explain` channel inference dependent on the runtime plugin registry, while the real CLI path does not load plugins; the complementary open PR at https://github.com/openclaw/openclaw/pull/51584 covers that other half. The PR also still has only synthetic/code-path proof, not after-fix real behavior proof.

## What This Changes

The PR wraps non-agent-scoped colon session keys passed to `sandbox explain` as agent-scoped keys and adds tests for Telegram slash-session diagnostics.

## Best Possible Solution

Land one reconciled fix that normalizes non-agent-scoped channel session keys, falls back to bundled channel ids when plugins are unloaded, and includes no-registry regression coverage plus redacted real behavior proof.

## Reproduction Assessment

Yes, by source inspection. Current main leaves `telegram:slash:<id>` unscoped, and this PR reaches an agent-scoped key but still relies on registry-only channel inference in a CLI path that does not load plugins.

## Solution Assessment

No. Key normalization is necessary, but the maintainable fix also needs `normalizeChatChannelId`-style bundled fallback or an explicit plugin-loading strategy with a no-registry regression test.
