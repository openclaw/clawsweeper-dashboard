## Summary

Keep open: the normalization is useful, but this PR still does not prove or complete the real no-registry `sandbox explain` path, and external real behavior proof is missing before merge.

## What This Changes

The branch wraps non-agent-scoped colon session keys in `sandbox explain` as agent-scoped keys and adds tests for Telegram slash normalization and passthrough.

## Best Possible Solution

Land one consolidated fix that keeps the key normalization, adds registry-independent bundled-channel fallback in channel inference, and includes a no-registry `sandbox explain` regression test.

## Reproduction Assessment

Yes, by source inspection. Current main leaves `telegram:slash:<id>` unscoped, and this PR still relies on registry-only channel inference while the CLI path does not load plugins.

## Solution Assessment

No. The key normalization is necessary, but the narrow maintainable fix also needs bundled-channel fallback coverage for the no-registry CLI path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Resolve bundled channels without a loaded registry:** `src/commands/sandbox-explain.ts:55-59`
  - body: Wrapping `telegram:slash:200` into `agent:main:telegram:slash:200` only gets the key to `inferProviderFromSessionKey`; that function still resolves `telegram` through `normalizeAnyChannelId`, which depends on an active plugin registry. Add a static bundled-channel fallback or deliberately load the registry and cover the no-registry case.
