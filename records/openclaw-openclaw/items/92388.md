## What This Changes

The PR adds a `refreshCurrentModelFromRegistry()` call before `checkCompaction` reads `contextWindow` in `AgentSession`.

## Best Possible Solution

Make the selected provider/model the canonical live session identity before post-turn `AgentSession` reads, then add cross-provider regression coverage and real Telegram or CLI proof.

## Reproduction Assessment

Yes, source-level. Persisting a different provider/model selection can leave a running `AgentSession` with its old `agent.state.model`, and current post-turn code then reads that stale snapshot for compaction and related session behavior.

## Solution Assessment

No. The proposed call is a plausible local mitigation, but it looks up the registry from the stale current identity; the maintainable fix should start from the newly selected provider/model or update the live session at the shared switch boundary.
