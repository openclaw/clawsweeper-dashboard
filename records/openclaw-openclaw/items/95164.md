Overall correctness: patch is incorrect

Full review comments:

- **[P1] Forward search_context_size on the Search API path:** `extensions/perplexity/src/perplexity-web-search-provider.runtime.ts:226`
  - body: The default direct Perplexity route is `search_api`, but this added parameter is never written into the request body and the call into `runPerplexitySearchApi` does not pass the parsed value. As a result, `web_search({ search_context_size: "high" })` still sends no context-size control for the main Perplexity path. Add `body.search_context_size`, pass the parsed value, and handle the documented conflict with manual token budgets.
  - confidence: 0.97
- **[P1] Stop rejecting the chat-completions context size:** `extensions/perplexity/src/perplexity-web-search-provider.runtime.ts:429-436`
  - body: This guard runs when `runtime.transport === "chat_completions"`, so Sonar/OpenRouter/baseUrl override calls with `search_context_size` return `unsupported_search_context_size` before the later `runPerplexitySearch` call can send `web_search_options.search_context_size`. Either remove this rejection for the supported chat-completions route or stop advertising the parameter for that route.
  - confidence: 0.96
- **[P2] Add the field to the Perplexity provider schema too:** `src/agents/tools/web-search.ts:69-73`
  - body: The generic built-in schema now lists the argument, but `createPerplexityParameters` is still the provider-owned schema returned by `createPerplexityWebSearchProvider().createTool(...)` and it omits `search_context_size`. Provider-specific tool discovery will not advertise the new capability until that schema and its tests are updated too.
  - confidence: 0.86
