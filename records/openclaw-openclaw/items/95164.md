- **[P1] Let chat completions reach the forwarding path:** `extensions/perplexity/src/perplexity-web-search-provider.runtime.ts:429-436`
  - body: `structured` is false for the Sonar/OpenRouter chat-completions route, so this new guard returns `unsupported_search_context_size` before `runPerplexitySearch` can send `web_search_options.search_context_size`. A valid Sonar call with the new option therefore fails instead of using the feature this PR advertises.
  - confidence: 0.97
- **[P1] Reject native context size before caching it:** `extensions/perplexity/src/perplexity-web-search-provider.runtime.ts:512`
  - body: Direct Perplexity defaults to the native Search API path, but this patch adds `searchContextSize` to the cache key while the native request body still never serializes it and the native call does not reject it before lookup. Different `search_context_size` values can cache the same default-depth native result under separate keys, leaving the model with a knob that appears accepted but has no effect.
  - confidence: 0.93
- **[P2] Expose the option through the provider schema:** `src/agents/tools/web-search.ts:69-73`
  - body: Only the shared schema is updated; `createPerplexityParameters` still leaves chat-completions tools with `query`, `count`, and `freshness` only, so provider-specific discovery does not advertise the new Sonar parameter or its transport limits. Keep the shared schema, provider schema, docs, and runtime contract in sync so agents see the same supported surface they can execute.
  - confidence: 0.86
