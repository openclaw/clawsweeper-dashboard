## Summary

Keep open. Current main still matches the reported macOS `screen.snapshot` behavior, and the same reporter has open unmerged PR #68186 using closing syntax for this issue, so this should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue asks macOS `screen.snapshot` to reject malformed `paramsJSON` and enforce a response-size guard before serializing inline base64 JSON payloads.

## Best Possible Solution

Review and land #68186, or apply an equivalent focused macOS runtime fix that distinguishes omitted params from malformed params, returns `INVALID_REQUEST` before capture, enforces the gateway payload budget before base64 JSON emission, and adds targeted `MacNodeRuntimeTests` coverage.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: call `MacNodeRuntime.handleInvoke` with malformed `screen.snapshot` params to hit the `try?` default fallback, or inject a large snapshot result to exercise the unguarded base64 response path.

## Solution Assessment

Yes. Fixing this inside the macOS node runtime handler, with focused Swift regression tests and no transport redesign for in-bounds snapshots, is the narrowest maintainable solution.
