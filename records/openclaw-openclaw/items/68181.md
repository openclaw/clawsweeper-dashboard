## Summary

Keep open. Current main still matches the reported macOS screen.snapshot behavior, and the same-author fix PR at https://github.com/openclaw/openclaw/pull/68186 remains open with a closing reference, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue asks for macOS screen.snapshot to reject malformed params and bound inline base64 JSON snapshot responses before emission.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/68186, or apply an equivalent focused macOS runtime fix that separates omitted params from malformed params, bounds snapshot result frames before emission, and adds regression tests.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: malformed params hit the try? default fallback, and large snapshot bytes are base64-encoded into payloadJSON without a size guard.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the macOS screen.snapshot handler and tests; the open linked PR is the active implementation path, subject to normal review and proof before merge.

  - body: Current main treats malformed screen.snapshot params as default params, so invalid JSON can still reach snapshotScreen instead of failing closed with INVALID_REQUEST.
