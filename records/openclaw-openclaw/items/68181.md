## Summary

Keep open. Current main still contains both reported `screen.snapshot` failure modes, and the linked closing PR is still open, conflicting, and not merged, so this issue should wait for that PR or an equivalent merged fix.

## What This Changes

The issue reports that macOS `screen.snapshot` accepts malformed `paramsJSON` by falling back to defaults and can emit oversized inline base64 JSON without a payload-size guard.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/68186 after its branch is repaired and real macOS node/gateway proof is added, or apply an equivalent focused runtime fix with regression tests.

## Reproduction Assessment

Yes. Current-main source inspection shows malformed `screen.snapshot` params still fall back to defaults and snapshot bytes are still emitted inline as base64 without a size guard; I did not run the macOS Swift repro locally in this read-only sweep.

## Solution Assessment

Yes. The narrow maintainable solution is to fix `handleScreenSnapshotInvoke` so omitted params and malformed params are distinct, bound the emitted snapshot frame before serialization, and add focused macOS runtime regression tests.
