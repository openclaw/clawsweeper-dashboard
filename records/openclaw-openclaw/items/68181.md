## Summary

Keep open. Current main still matches the reported macOS `screen.snapshot` behavior, and open PR #68186 is already the same-author implementation candidate using closing syntax, so this should remain open until that PR or an equivalent fix lands.

## What This Changes

Reject malformed macOS `screen.snapshot` params and oversized snapshot payloads before capture or base64 JSON emission.

## Best Possible Solution

Review and land #68186, or apply an equivalent focused macOS runtime fix that distinguishes omitted params from malformed params, enforces the gateway payload budget before encoding, and adds targeted regression tests.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: malformed `paramsJSON` hits the `try?` fallback before `snapshotScreen`, and large returned `Data` is base64-encoded into `payloadJSON` without a size check.

## Solution Assessment

Yes. The narrow maintainable fix belongs in `handleScreenSnapshotInvoke`: decode nil params separately from malformed params, fail with `INVALID_REQUEST` before capture, reject over-budget payloads before JSON/base64 emission, and cover those cases in `MacNodeRuntimeTests`.
