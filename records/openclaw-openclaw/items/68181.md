## Summary

Keep open. Current main still contains the reported macOS screen.snapshot fallback and unbounded inline payload behavior, and the same-author fix PR with a closing reference is still open, so this issue should wait for that PR or an equivalent merged fix.

## What This Changes

The issue requests hardening macOS `screen.snapshot` so malformed params fail with `INVALID_REQUEST` and oversized snapshot responses are rejected before base64 JSON emission.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/68186, or apply an equivalent focused macOS runtime fix that separates omitted params from malformed params, bounds snapshot result frames before emission, and adds regression tests.

## Reproduction Assessment

Yes, by source inspection. Current main still falls back from malformed `screen.snapshot` params to defaults and emits inline base64 `payloadJSON` without a size guard; I did not run the macOS Swift repro locally in this read-only sweep.

## Solution Assessment

Yes. The narrow maintainable path is to fix the macOS `screen.snapshot` handler and tests; because the linked closing PR is still open, this issue should close only after that PR or an equivalent fix merges.
