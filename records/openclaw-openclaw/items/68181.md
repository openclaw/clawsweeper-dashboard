## Summary

Keep open. Current main still contains the malformed-params fallback and unbounded inline base64 response, and https://github.com/openclaw/openclaw/pull/68186 is the active same-author closing PR but remains open and conflicting.

## What This Changes

The issue requests fail-closed validation for malformed macOS `screen.snapshot` params and a size guard for inline snapshot payload responses.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/68186, or apply an equivalent focused fix that separates omitted params from malformed params, bounds snapshot response frames before emission, and adds regression tests.

## Reproduction Assessment

Yes, by source inspection. Current main still falls back from malformed `screen.snapshot` params to defaults and emits inline base64 `payloadJSON` without a size guard; I did not run the macOS Swift repro in this read-only review.

## Solution Assessment

Yes. The narrow maintainable solution is to fix the macOS `screen.snapshot` handler and its Swift runtime tests; because the linked closing PR is still open, this issue should close only after that PR or an equivalent fix merges.
