## Summary

Keep open. Current main still matches the reported macOS screen.snapshot behavior, and open PR #68186 is the linked unmerged implementation candidate, so this issue should close only after that PR or an equivalent fix lands.

## What This Changes

The issue requests that macOS screen.snapshot reject malformed paramsJSON with INVALID_REQUEST and reject oversized snapshot payloads before emitting base64 JSON.

## Best Possible Solution

Review and land #68186, or apply an equivalent macOS runtime fix that separates omitted params from malformed params, returns INVALID_REQUEST before capture on malformed JSON, enforces the gateway payload budget before response emission, and adds focused MacNodeRuntimeTests coverage.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: call MacNodeRuntime.handleInvoke with malformed screen.snapshot params or an injected large snapshot result; current main still contains the fallback decode and unguarded base64 emission paths.

## Solution Assessment

Yes. The linked PR direction is the narrowest maintainable fix path: keep valid in-bounds snapshot behavior unchanged, add fail-closed validation and payload guards in the macOS runtime, and cover the behavior with targeted Swift tests.
