Keep open. The report is source-valid on current main, but the protected `maintainer` label means this should not be auto-closed or routed without explicit maintainer handling.

## What This Changes

Fix doctor configured-plugin install repair so provider ids referenced only by `channels.modelByChannel` are included in provider plugin repair.

## Best Possible Solution

Include channel model override refs when deriving provider ids for configured-plugin repair, then add a focused regression test around `collectReleaseConfiguredPluginIds` and provider catalog mapping.

## Reproduction Assessment

Yes, source-level. A config whose only installable provider reference is under `channels.modelByChannel` is skipped because the repair helper explicitly disables channel model override scanning before passing provider plugin ids to repair.

## Solution Assessment

Yes. The narrow maintainable fix is to include channel override model refs in the configured-plugin provider scan, with regression coverage, without broadening unrelated plugin auto-enable semantics.
