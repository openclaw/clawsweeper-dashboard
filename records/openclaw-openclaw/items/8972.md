## Summary

Keep open. Current main still treats Discord guild channel entries as exact id/name/slug keys, thread-parent fallbacks, or the literal `*` fallback; `ws-*` remains new access-control/config behavior rather than an implemented bug fix.

## What This Changes

Request support for Discord guild channel allowlist patterns such as `ws-*`, with category-based matching offered as an alternative.

## Best Possible Solution

Define a Discord-owned pattern syntax and precedence model, preserve exact/parent/`*` behavior, update config schema/types/docs/diagnostics, and add focused regression coverage for dynamically named channels.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that `ws-*` is currently treated as a literal key and will not match `ws-task-1` unless an exact, parent, or literal `*` entry also matches.

## Solution Assessment

Unclear until maintainers choose the public glob syntax and access-control precedence. A narrow Discord allow-list implementation is plausible, while category inheritance should remain tied to https://github.com/openclaw/openclaw/issues/60582 unless the designs are intentionally merged.
