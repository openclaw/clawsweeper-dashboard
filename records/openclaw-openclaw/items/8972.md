## Summary

Keep open. Current main still treats Discord guild channel keys as exact id/name/slug entries, optional parent fallbacks, or the literal `*`; glob keys such as `ws-*` are not implemented, and the related category/access-list issues do not supersede this narrower pattern request.

## What This Changes

Add glob or wildcard pattern support to the Discord guild channel allowlist so dynamically named channels can be allowed without per-channel config edits and gateway restarts.

## Best Possible Solution

Define a Discord-owned pattern syntax and precedence model, then implement it in the existing allow-list path with aligned exported types, strict schema, docs/status/audit behavior, and focused regression coverage.

## Reproduction Assessment

Yes, at source level. A guild channel config containing `ws-*` and an incoming channel slug `ws-task-1` will miss because the matcher uses exact lookups plus parent and literal `*` fallback, and current tests show unmatched configured channels deny.

## Solution Assessment

Unclear until maintainers choose the public pattern syntax and access-control precedence. A narrow Discord allow-list implementation is plausible, but it should follow product/security review rather than automatic cleanup.
