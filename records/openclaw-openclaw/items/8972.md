## Summary

Keep open: current main still treats Discord channel allowlist keys as exact id/name/slug entries, thread-parent fallbacks, or the literal `*`; it does not implement glob matching for keys like `ws-*`, and prior maintainer/bot discussion also kept this as remaining Discord authorization design work.

## What This Changes

Add glob/wildcard matching for Discord guild channel allowlist entries so dynamically-created channels like `ws-task-1` can match `ws-*` without per-channel config edits.

## Best Possible Solution

Implement a documented Discord-owned pattern form with explicit precedence, tests, schema/types/docs updates, and preserved exact/thread-parent/literal-star behavior; keep category inheritance under #60582 unless maintainers deliberately merge the designs.

## Reproduction Assessment

Yes for the current missing capability: source inspection shows `channels: { "ws-*": { enabled: true } }` will not match a channel named `ws-task-1` because only exact id/name/slug candidates, parent keys, and literal `*` are evaluated. I did not run tests because this review is constrained to read-only inspection.

## Solution Assessment

Unclear. The proposed `ws-*` UX is plausible, but the best implementation needs maintainer-defined authorization precedence and should stay separate from category inheritance unless #60582 is folded into the same design.
