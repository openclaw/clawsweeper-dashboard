## Summary

Keep open. Current main still lacks Discord glob/pattern matching for guild channel allowlist keys; the issue is a feature request that changes access-control semantics and needs maintainer design for public syntax and precedence.

## What This Changes

Request pattern/glob support for Discord guild channel allowlist entries so dynamically created channel names can match without per-channel config edits or gateway restarts.

## Best Possible Solution

Define a documented Discord-owned pattern syntax and precedence model, preserve exact/parent/literal `*` behavior, update config schema/types/docs/diagnostics, and add focused regression coverage for dynamically named channels.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that `ws-*` is currently treated as a literal key and will not match `ws-task-1` unless an exact, parent, or literal `*` entry also matches.

## Solution Assessment

Unclear until maintainers choose the public glob syntax and access-control precedence. A narrow Discord allowlist implementation is plausible, while category inheritance should remain tied to https://github.com/openclaw/openclaw/issues/60582 unless the designs are intentionally combined.
