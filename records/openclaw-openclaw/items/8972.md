## Summary

Keep open. Current main still treats Discord guild channel keys as exact id/name/slug matches, thread-parent fallbacks, or the literal `*`, so `ws-*` remains an unimplemented access-control feature rather than an obsolete cleanup item.

## What This Changes

The issue requests glob or wildcard matching for Discord guild channel allowlist entries so dynamic channels can be accepted without per-channel config edits and gateway restarts.

## Best Possible Solution

Define a documented Discord-owned pattern syntax and precedence model, preserve exact id/name/slug, parent, and literal * behavior, then update schema, types, docs, diagnostics, and focused allowlist tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that ws-* is currently treated as a literal record key and will not match ws-task-1 unless an exact, parent, or literal * entry also matches.

## Solution Assessment

Unclear until maintainers approve the public glob syntax and access-control precedence. A narrow Discord allowlist implementation is plausible, while category inheritance should remain tied to https://github.com/openclaw/openclaw/issues/60582 unless maintainers intentionally combine the designs.
