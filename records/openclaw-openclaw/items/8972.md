## Label Justifications

- `P2`: This is a normal-priority Discord access-control enhancement with a clear affected surface but no urgent production regression.
- `impact:security`: The request changes which Discord guild channels are authorized to trigger OpenClaw from a configured allowlist.
- `impact:message-loss`: Exact-only allowlists can cause intended dynamically created Discord channels to be ignored until configuration is patched and the gateway is restarted.

## Summary

Keep open. Current main still treats Discord guild channel entries as exact id/name/slug keys, thread-parent fallbacks, or the literal `*`; a key like `ws-*` is not evaluated as a glob, and adding that behavior is an access-control-sensitive config feature that needs maintainer product/security review.

## What This Changes

Request support for glob or wildcard Discord guild channel allowlist keys such as `ws-*` so dynamically created channels can be accepted without config edits and gateway restarts.

## Best Possible Solution

Keep this as an open Discord plugin design task: define a documented pattern syntax and precedence model, preserve exact and literal `*` behavior, update types/schema/docs/status/audit output, and add focused regression tests while coordinating category inheritance through https://github.com/openclaw/openclaw/issues/60582 and broader access-list redesign through https://github.com/openclaw/openclaw/issues/69748.

## Reproduction Assessment

Yes, at source level. A guild channel key like `ws-*` is stored as an ordinary `channels` record key, while runtime lookup only checks exact id/slug/name candidates, thread-parent fallback, and the literal `*`, so `ws-task-1` remains denied unless a broader fallback is configured.

## Solution Assessment

Unclear until maintainers choose the public config contract. Glob support is a coherent solution for the reported dynamic-channel workflow, but the safest implementation needs explicit access-control precedence and coordination with the separate category and access-list redesign issues.
