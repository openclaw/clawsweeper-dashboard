## Summary

Keep open. Current main still treats Discord guild channel keys as exact ids, names, slugs, parent fallbacks, or the literal `*`, so a key like `ws-*` is stored literally and matching dynamic channels remain denied.

## What This Changes

Request support for glob or wildcard Discord guild channel allowlist keys so dynamically created channels like `ws-task-1` can match `ws-*` without config edits or gateway restarts.

## Best Possible Solution

Define and document a Discord-owned pattern channel entry form with deterministic precedence and regression coverage; keep category inheritance in #60582 unless maintainers choose to merge the designs.

## Reproduction Assessment

Yes, from source inspection. A guild `channels` map containing `ws-*` and an inbound `ws-task-1` channel follows the exact-key matcher, misses the literal key, has no `*` fallback, and resolves to denied.

## Solution Assessment

Unclear until maintainers settle the access-control semantics. The narrow maintainable direction is a Discord/channel matcher extension with explicit ordering, docs, schema/status/audit updates, and tests for exact, parent, pattern, and literal wildcard precedence.
