## Summary

Keep open. Current main still resolves Discord guild channel entries through exact id/name/slug keys, thread-parent fallback, and the literal `*` fallback, so glob support remains a new access-control/config feature rather than cleanup that can be closed.

## What This Changes

Request support for Discord guild channel allowlist keys such as `ws-*` so newly created matching channels do not need per-channel config edits and gateway restarts.

## Best Possible Solution

Define a Discord-owned glob syntax and precedence model, update the type/schema/docs/audit/status surfaces, and add focused allowlist regression tests while leaving category inheritance to its canonical tracker.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that `ws-*` is currently treated as a literal key and will not match `ws-task-1` unless an exact, parent, or literal `*` entry also matches.

## Solution Assessment

Unclear until maintainers approve the public pattern syntax and access-control precedence. A narrow implementation in the Discord allowlist path looks plausible, but it should not silently absorb category inheritance or the broader access-list redesign.
