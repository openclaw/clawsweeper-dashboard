## Summary

Keep open. Current main still supports exact Discord channel id/name/slug matching, thread-parent fallback, and the literal `*` fallback only; glob/category allowlist semantics are absent from code, schema, docs, and tests, so this remains a valid access-control feature request needing maintainer design.

## What This Changes

Request support for glob or wildcard Discord guild channel allowlist entries so dynamically created matching channels do not require config edits and gateway restarts.

## Best Possible Solution

Define a documented Discord-owned pattern syntax and precedence model, update types/schema/docs/status/audit behavior, and add regression coverage; keep category inheritance scoped to #60582 unless maintainers merge those designs.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that `ws-*` is treated as a literal channel key and would not match `ws-task-1` unless an exact, parent, or literal `*` entry matches.

## Solution Assessment

Unclear until maintainers decide the access-control syntax and precedence. A narrow Discord allow-list implementation with schema, docs, diagnostics, and tests is plausible, while category inheritance should remain tied to #60582 unless designs are combined.
