## Summary

Current main still lacks pattern evaluation, and no related item safely supersedes this distinct request. The feature remains useful but needs an explicit maintainer decision because it expands Discord authorization semantics and overlaps adjacent category and access-list designs.

## What This Changes

Add pattern-based Discord guild channel allowlist entries so dynamically created matching channels work without per-channel configuration changes.

## Best Possible Solution

Define a Discord-owned selector contract with explicit syntax and deterministic precedence—exact ID, name, slug, and thread-parent matches first, approved pattern matches next, and literal `*` last—then align config types and schema, public docs, status and security-audit reporting, and focused allow/deny regression tests.
