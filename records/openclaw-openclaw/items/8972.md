- `impact:security`: The requested matcher controls which Discord channels are authorized to reach the agent.
- `impact:message-loss`: Dynamic channels that are not individually listed are currently denied and their messages are suppressed.

## Summary

Current main still supports only exact channel identifiers/names, thread-parent matching, and the literal `*` fallback. Partial patterns remain unimplemented, but their authorization semantics overlap open category and access-policy designs and require maintainer product and security judgment.

## What This Changes

Add glob or wildcard matching for Discord guild channel allowlist entries so dynamically created matching channels do not require individual config edits.

## Best Possible Solution

Choose a Discord-owned pattern contract, then implement deterministic exact and thread-parent precedence before ordered pattern matches and the literal `*` fallback, with schema/help/docs, audit metadata, and focused access-control tests.

## Reproduction Assessment

Yes at source level: current main treats `ws-*` as an ordinary record key, so it cannot match a channel named `ws-task-1`; no runtime execution is needed to establish that contract gap.

## Solution Assessment

Unclear: partial wildcard support is maintainable if Discord-scoped, but maintainers must first choose its syntax and precedence relative to category inheritance and broader Discord access-policy proposals.
