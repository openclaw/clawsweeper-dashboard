## Label Justifications

- `P2`: This is a meaningful but non-emergency Discord configuration improvement with a bounded user impact and design-sensitive implementation.
- `impact:security`: Guild channel matching determines which Discord ingress is authorized to reach the agent.
- `impact:message-loss`: Messages from dynamically created channels are suppressed when their names do not exactly match a configured entry.

## Summary

Current main still interprets Discord guild channel entries as exact id/name/slug keys, thread-parent fallbacks, or the literal `*` fallback, so the requested glob behavior remains missing and is not owned by the separate category or access-policy issues. The item should remain open because the public syntax and authorization precedence require explicit maintainer product and security judgment.

## What This Changes

Add glob-style Discord guild channel allowlist entries so newly created matching channels are accepted without per-channel config edits or gateway restarts.

## Best Possible Solution

Define a Discord-owned pattern contract with exact and thread-parent matches taking precedence, deterministic ordering for overlapping patterns, literal `*` retained as the final broad fallback, and aligned types/schema/docs/status/audit behavior plus focused access-control tests.

## Reproduction Assessment

Yes. Current main treats `ws-*` as an ordinary record key, so `ws-task-1` remains unmatched and is denied when the guild has a nonempty channel map.

## Solution Assessment

No, the example syntax alone is not yet the best complete solution; maintainers must specify normalization, escaping, overlap ordering, parent precedence, and diagnostic metadata before implementation.
