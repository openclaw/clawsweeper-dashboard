## Summary

Keep open: current main and the latest release still do not support Discord channel glob/pattern entries like `ws-*`; this remains a security-sensitive Discord config feature that needs maintainer-approved syntax and precedence.

## What This Changes

Request pattern-based Discord guild channel allowlist entries so dynamically created matching channels can be admitted without per-channel config edits and restarts.

## Best Possible Solution

Approve and document one Discord-owned selective pattern contract, then update the matcher, Discord allow-list path, types/schema, docs, audit/status output, and focused regression tests together.

## Reproduction Assessment

Yes, at source level: with a nonempty Discord guild `channels` map containing `ws-*`, a channel named `ws-task-1` has no exact, parent, normalized, or literal `*` match and resolves denied.

## Solution Assessment

Unclear until maintainers choose the permanent pattern syntax and authorization precedence; a narrow Discord-owned pattern contract is the right implementation boundary if that direction is approved.
