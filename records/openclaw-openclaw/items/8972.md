## Summary

Current main still treats Discord guild channel entries as exact or normalized keys, thread-parent fallbacks, or the literal `*` fallback; pattern matching remains a useful but security-sensitive feature requiring maintainer agreement on syntax and precedence.

## What This Changes

Add pattern-based Discord guild channel allowlist entries so dynamically created matching channels do not require config edits and gateway restarts.

## Best Possible Solution

Add an explicit Discord-owned pattern form with deterministic precedence and audit visibility while preserving exact keys, thread-parent behavior, and literal `*` semantics.

## Reproduction Assessment

Yes at source level: configure a `ws-*` entry and send in `ws-task-1`; current main performs equality-based matching plus a literal `*` fallback, so the configured pattern is not selected.

## Solution Assessment

No, ordinary map keys should not silently acquire glob semantics; an explicit typed pattern form is a safer and more maintainable direction.
