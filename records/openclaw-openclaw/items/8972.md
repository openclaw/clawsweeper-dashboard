## Summary

Keep open. Current main still treats Discord guild channel entries as exact id/name/slug keys, thread-parent fallbacks, or the literal `*` fallback, so a key like `ws-*` remains unimplemented access-control feature work; the category alternative is still tracked separately at https://github.com/openclaw/openclaw/issues/60582 and the additive-overrides semantic request was closed as superseded at https://github.com/openclaw/openclaw/issues/73387.

## What This Changes

Request glob/wildcard support for Discord guild channel allowlist entries so dynamically named channels can match without config edits and gateway restarts.

## Best Possible Solution

Define a documented Discord-owned pattern syntax and precedence model, preserve existing exact/parent/`*` behavior, update config types/schema/docs/audit or status surfaces, and add regression coverage for dynamically named matching channels.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that `ws-*` is currently treated as a literal channel key and will not match `ws-task-1` unless an exact, parent, or literal `*` entry also matches.

## Solution Assessment

Unclear until maintainers decide the access-control syntax and precedence. A narrow Discord allow-list implementation with schema, docs, diagnostics, and tests is plausible, while category inheritance should remain tied to https://github.com/openclaw/openclaw/issues/60582 unless maintainers merge the designs.
