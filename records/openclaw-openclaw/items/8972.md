## Summary

Keep open. Current main still treats Discord guild channel allowlist keys as exact id/name/slug entries, thread-parent fallbacks, or the literal `*` fallback, so glob keys like `ws-*` remain unimplemented access-control feature work.

## What This Changes

The issue requests glob or wildcard matching for Discord guild channel allowlists so dynamically created channels such as `ws-task-1` can be accepted without editing config and restarting the gateway.

## Best Possible Solution

Define a documented Discord-owned pattern syntax and precedence model, update the exported config types/schema/docs/status or audit surfaces, and add regression tests proving matching dynamic channel names are accepted without per-channel config edits.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that `ws-*` is currently treated as a literal channel key and will not match `ws-task-1` unless an exact, parent, or literal `*` entry also matches.

## Solution Assessment

Unclear until maintainers decide the access-control syntax and precedence. A narrow Discord allow-list implementation with schema, docs, diagnostics, and tests is plausible, while category inheritance should remain tied to #60582 unless maintainers merge the designs.
