## Summary

Keep open: current main still treats Discord guild channel allowlist entries as exact id/name/slug keys, thread-parent fallbacks, or the literal `*`; glob keys such as `ws-*` are not implemented, and the remaining work needs maintainer agreement on Discord authorization semantics.

## What This Changes

Request Discord channel allowlist support for glob/wildcard keys such as `ws-*` so dynamically created matching channels can be allowed without per-channel config edits and gateway restarts.

## Best Possible Solution

Implement a documented Discord-owned pattern matching contract with tests, schema/types/docs/status/audit updates, while keeping category inheritance under #60582 unless maintainers deliberately merge the designs.

## Reproduction Assessment

Yes for the current missing capability: source inspection shows `channels: { "ws-*": { enabled: true } }` will not match a channel named `ws-task-1` because only exact id/name/slug candidates, parent keys, and literal `*` are evaluated. I did not run tests because this review is constrained to read-only inspection.

## Solution Assessment

Unclear. The proposed glob UX is plausible, but the maintainable fix needs owner-defined authorization precedence and should stay separate from category inheritance unless maintainers choose to combine those designs.
