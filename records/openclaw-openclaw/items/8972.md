## Label Justifications

- `P2`: This is a bounded Discord ergonomics enhancement with normal backlog urgency but meaningful authorization implications.
- `impact:security`: Pattern entries would broaden which Discord channels are authorized for agent message processing.
- `impact:message-loss`: The current exact allowlist suppresses messages from newly created channels until configuration is updated.

## Summary

Current main still treats Discord guild channel entries as exact ID/name/slug keys, thread-parent fallbacks, or the literal `*`; no glob matching exists, and the unique request remains useful but needs maintainer agreement on security-sensitive precedence before implementation.

## What This Changes

Add glob or wildcard matching for Discord guild channel allowlist entries so dynamically created channels can be accepted without per-channel config edits and gateway restarts.

## Best Possible Solution

Introduce a documented Discord-owned pattern form with deterministic precedence after exact and thread-parent matches but before the literal `*` fallback, including audit/status visibility and focused authorization regression tests.

## Reproduction Assessment

Yes, at source level. The matcher performs exact record-key checks plus a literal `*` fallback, while Discord supplies exact ID, slug, and name candidates, so a configured `ws-*` entry cannot match `ws-task-1`.

## Solution Assessment

Unclear pending maintainer direction. Pattern support is a maintainable solution to this specific workflow, but its syntax and precedence must be chosen explicitly because channel matching is an authorization boundary and category inheritance remains separately tracked at https://github.com/openclaw/openclaw/issues/60582.
