## Summary

Keep open. Current main still treats Discord guild channel entries as exact id/name/slug keys, parent fallback keys, or the literal `*`; glob support would add a new access-control surface that needs maintainer design for syntax and precedence.

## What This Changes

Request support for glob-style Discord guild channel allowlist keys such as `ws-*` so newly created matching channels are accepted without config edits or restarts.

## Best Possible Solution

Define a documented Discord-owned pattern syntax and precedence model, preserve exact/parent/literal `*` behavior, update config schema/types/docs/diagnostics, and add focused regression coverage for dynamically named channels.

## Reproduction Assessment

No: this is not applicable as a bug reproduction because it requests a new feature. Source inspection gives high confidence that `ws-*` is currently treated as a literal key and will not match `ws-task-1` unless an exact, parent, or literal `*` entry also matches.

## Solution Assessment

Unclear until maintainers choose the public glob syntax and access-control precedence. A narrow Discord allowlist implementation is plausible, while category inheritance should remain tied to https://github.com/openclaw/openclaw/issues/60582 unless the designs are intentionally combined.
