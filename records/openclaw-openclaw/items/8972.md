## Summary

Keep open: current main and v2026.6.8 still only support exact/normalized/parent Discord channel keys plus the literal `*` fallback, so an entry like `ws-*` remains an ordinary unmatched key and the requested pattern allowlist behavior is still missing.

## What This Changes

The issue requests Discord channel allowlist glob/wildcard entries so dynamically created channels can be authorized without per-channel config edits and restarts.

## Best Possible Solution

Approve and document one Discord-owned pattern contract, then update allow-list matching, types/schema, docs, audit/status behavior, and focused regression tests together.

## Reproduction Assessment

Yes, at source level: with a nonempty Discord guild `channels` map containing `ws-*`, a channel named `ws-task-1` has no exact, parent, normalized, or literal `*` match and resolves denied. I did not run a live Discord repro in this read-only review.

## Solution Assessment

Unclear until maintainers choose the permanent pattern syntax and authorization precedence. A narrow Discord-owned pattern contract is the right implementation boundary if that direction is approved.
