## Summary

Keep open. Current main still treats Discord guild channel entries as exact id/name/slug keys, thread-parent fallbacks, or the literal `*` fallback; adding `ws-*` style matching would be a new Discord access-control/config feature that needs maintainer product and security review.

## What This Changes

The issue requests glob or wildcard matching for Discord guild channel allowlists so dynamically created channels such as `ws-*` can be allowed without per-channel config edits and gateway restarts.

## Best Possible Solution

Define a documented Discord channel-pattern contract with explicit precedence, keep literal `*` as the broad fallback, update config types/Zod schema/docs/status/audit behavior, and add focused regression tests for dynamically created matching channels.

## Reproduction Assessment

Not applicable as a bug reproduction path because this is a feature request. Source inspection gives high confidence that current main lacks glob evaluation and still denies unmatched configured Discord channels.

## Solution Assessment

Unclear until maintainers choose the public syntax and precedence model. The right implementation should stay in the Discord-owned allow-list/config path rather than changing generic channel matching without an approved access-control contract.
