## Summary

Keep open. Current main still treats Discord guild channel entries as exact id/name/slug keys, parent fallbacks, or the literal `*`; glob keys such as `ws-*` remain unimplemented feature work with access-control implications.

## What This Changes

Request adding glob/wildcard pattern support to Discord guild channel allowlists so dynamic channel names can be accepted without config edits and gateway restarts.

## Best Possible Solution

Define a documented Discord-owned pattern syntax and precedence model, preserve existing exact/parent/`*` behavior, update config schema/types/docs/diagnostics, and add focused regression coverage for dynamically named channels.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that `ws-*` is currently treated as a literal key and will not match `ws-task-1` unless an exact, parent, or literal `*` entry also matches.

## Solution Assessment

Unclear until maintainers choose the public glob syntax and access-control precedence. A narrow Discord allow-list implementation is plausible, while category inheritance should remain tied to https://github.com/openclaw/openclaw/issues/60582 unless the designs are intentionally combined.
