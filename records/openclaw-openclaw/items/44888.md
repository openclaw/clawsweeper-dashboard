## What This Changes

The PR adds a Kimi-specific tool-schema compactor in agent tool normalization, omits heavy `sessions_spawn` fields for detected Kimi OpenAI-compatible models, and adds regression tests.

## Best Possible Solution

Rebase the fix onto the provider-owned `normalizeToolSchemas` path, share a narrow Kimi/OpenAI-compatible schema compaction helper, gate it away from native Kimi Coding routes, and add focused provider/runtime tests plus a changelog entry.

## Solution Assessment
