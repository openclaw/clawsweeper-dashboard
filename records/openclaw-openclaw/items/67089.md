- **[P1] Close the write options object:** `src/config/io.ts:1912`
  - body: The options object passed to `io.writeConfigFile` is opened above this line but the patch closes only the function call. TypeScript cannot parse the file, so the branch cannot build.
  - confidence: 0.97
- **[P1] Bootstrap missing signatures before rejecting configs:** `src/config/io.ts:1112-1113`
  - body: This rejects established configs when the `.sig` file is missing. Existing installations with a normal `openclaw.json` and a gateway token will not have a sidecar yet, so first boot after upgrade can drop a valid config by returning `{}`.
  - confidence: 0.94
- **[P2] Propagate signature sidecar write failures:** `src/config/io.hmac-integrity.ts:85-86`
  - body: The sidecar writer swallows every write error, but later reads can reject missing or empty signatures. A successful config write followed by a failed `.sig` write can therefore become a later startup failure with no immediate signal.
  - confidence: 0.91
- **[P2] Resolve the HMAC key through injected config deps:** `src/config/io.hmac-integrity.ts:26-35`
  - body: `readGatewayToken` reads `os.homedir()` and `process.env` directly while config IO tests and alternate homes use injected dependencies. That can make tests read config from a fake home while resolving the HMAC key from the real home, skipping verification or disabling integrity in non-default state-dir layouts.
  - confidence: 0.88
- **[P2] Remove the unused async HMAC writer import:** `src/config/io.ts:34`
  - body: `writeConfigHmacSig` is imported here, but the diff only calls the sync writer. After the syntax error is fixed, the unused import should still fail the lint gate.
  - confidence: 0.9

  - body: The docs say mismatched configs are loaded normally, but the sync load path returns `{}` and reload rejects snapshots with `integrityWarning`. The user-facing security doc needs to match the actual enforcement behavior.
  - confidence: 0.86
