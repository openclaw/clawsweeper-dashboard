## Summary

Keep open: current main does not implement config HMAC integrity, and this security-sensitive PR is not merge-ready because the branch has a parse error plus migration and enforcement risks.

## What This Changes

This PR adds config HMAC sidecar signing/verification, config integrity audit docs/findings, explicit config method admin checks, and a CodexBar onboarding option.

## Best Possible Solution

Land a maintainer/security-owned config-integrity design narrowly, with bootstrap or migration, injected token/home/env dependencies, consistent write/read failure semantics, focused upgrade/tamper/reload tests, aligned docs, and unrelated onboarding split out.

## Reproduction Assessment

Yes for the PR blockers: the raw branch shows the unclosed options object, current check-runs fail on the exact head, and the missing-signature lockout follows directly from the `no-sig && suspicious` rejection path.

## Solution Assessment

No. The HMAC direction may be useful, but this patch is not the narrowest maintainable solution until the compile issue, bootstrap behavior, failure semantics, dependency injection, docs, and unrelated CodexBar change are resolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Close the write options object:** `src/config/io.ts:1912`
  - body: The options object passed to `io.writeConfigFile` is opened here but the changed line closes only the function call. TypeScript cannot parse this file, so the branch cannot build.
  - confidence: 0.97
- **[P1] Bootstrap signatures before enforcing reads:** `src/config/io.ts:1112-1113`
  - body: This branch rejects established configs when the `.sig` file is missing. Existing installations with a normal `openclaw.json` and a gateway token will not have a sidecar yet, so first boot after upgrade can drop a valid config by returning `{}`.
  - confidence: 0.94
- **[P2] Surface signature write failures:** `src/config/io.hmac-integrity.ts:87-89`
  - body: The sidecar writer swallows every write error, but later reads can reject missing or empty signatures. A successful config write followed by a failed `.sig` write can therefore become a later startup failure with no immediate signal.
  - confidence: 0.91
- **[P2] Use injected deps for token lookup:** `src/config/io.hmac-integrity.ts:23-39`
  - body: `readGatewayToken` reads `os.homedir()` and `process.env` directly while config IO supports injected homes and env. Tests and alternate homes can therefore read config from one location but resolve the HMAC key from another.
  - confidence: 0.88
- **[P2] Remove the unused HMAC writer import:** `src/config/io.ts:34`
  - body: `writeConfigHmacSig` is imported here, but this diff only calls the sync writer. After the syntax error is fixed, the unused import should still fail the lint gate.
  - confidence: 0.9
- **[P2] Align the docs with rejection behavior:** `docs/security/config-protection.md:13`
  - body: The docs say mismatched configs are loaded normally, but the sync load path returns `{}` and reload rejects snapshots with `integrityWarning`. The security doc needs to match the actual enforcement behavior.
  - confidence: 0.86
- **[P3] Separate the security audit log or fix the doc:** `docs/security/config-protection.md:50`
  - body: The new audit helper writes to `config-audit.jsonl`, but the docs describe the security audit log as separate from the existing `config-audit.jsonl`. Either the filename or the documentation needs to change.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The patch is security-sensitive and introduces concrete availability and assurance risks in the proposed config-integrity enforcement.

Concerns:

- **[high] Missing signatures can lock out existing configs:** `src/config/io.ts:1112`
  - body: The PR treats missing signatures on established configs as suspicious and rejects them, but existing installations will not have generated `.sig` sidecars before this feature lands.
  - confidence: 0.94
- **[medium] Sidecar write failures fail open before reads fail closed:** `src/config/io.hmac-integrity.ts:87`
  - body: Signature write errors are silently ignored while read paths can reject missing signatures later, so an I/O error during persistence can become a later startup failure.
  - confidence: 0.91
- **[low] HMAC key resolution bypasses injected dependencies:** `src/config/io.hmac-integrity.ts:26`
  - body: Direct `os.homedir()` and `process.env` reads make tests and alternate homes use a different token source from the config IO dependency graph, weakening coverage of the new security path.
  - confidence: 0.88
