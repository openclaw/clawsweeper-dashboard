## What This Changes

The issue reports an upgrade regression where unset or legacy `memorySearch.provider` resolves to OpenAI instead of the previous auto/Gemini behavior, leaving semantic memory vector search dirty or unavailable.

## Best Possible Solution

Maintainers should choose one upgrade contract for unset, legacy `auto`, and Google/Gemini memory provider ids, then align resolver, memory-core runtime, gateway embeddings, doctor migration, docs, tests, and upgrade proof before closing this issue.
