## Label Justifications

- `P2`: The issue describes a latest-release memory provider regression with meaningful but bounded impact on semantic memory recall.
- `impact:auth-provider`: The affected behavior is memory embedding provider selection and Google/Gemini auth-provider mapping.
- `impact:session-state`: The reported failure leaves the memory index identity dirty and semantic memory unavailable until provider/index state is repaired.

## Summary

Keep open: current main and v2026.6.1 still normalize unset or legacy auto memorySearch provider to openai, while the Google embedding adapter is registered under the canonical memory provider id gemini. The report is not implemented away; the remaining question is whether maintainers want to change the shipped upgrade contract, add a google-to-gemini repair path, or keep the documented OpenAI default.

## What This Changes

The issue asks OpenClaw to preserve or correctly migrate memorySearch provider selection for Google/Gemini users after upgrading to 2026.6.1 so vector memory does not stay dirty or unavailable.

## Best Possible Solution

Maintainers should codify the upgrade contract: either keep the OpenAI default and improve diagnostics/workaround guidance, or add a doctor-backed migration/repair for affected legacy auto or google memorySearch configs to the canonical gemini provider with regression coverage for status, deep probe, and reindex behavior.

## Reproduction Assessment

Yes, source-reproducible: current main and v2026.6.1 map unset or auto memorySearch provider to openai, while a literal google memory provider has no canonical memory adapter/default model and can produce an empty expected model in index identity. I did not run a live Rocky Linux/systemctl upgrade reproduction.

## Solution Assessment

No, not yet: the supported current workaround is to configure memorySearch.provider as gemini, but the best repository fix depends on a maintainer decision about whether legacy auto or google should be migrated, aliased, or only documented as unsupported.
