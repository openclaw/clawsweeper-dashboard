## What This Changes

The issue reports an upgrade regression where memorySearch provider selection falls back to OpenAI and leaves vector memory search permanently dirty with a blank expected model after Google/Gemini provider changes.

## Best Possible Solution

Define one maintainer-approved memorySearch upgrade contract, then land the linked default-provider fix together with any needed Google/Gemini alias, provider.model backfill, doctor/docs, and regression coverage before closing this cluster.

Yes. Source inspection on current main shows the unresolved path: unset or legacy auto resolves to OpenAI, `google` is not registered as the Google memory adapter id, and the latest shipped tag still lacks the current empty-model guard.
