## What This Changes

The issue reports that upgrading memorySearch from an auto/Google-Gemini setup resets provider resolution toward OpenAI and leaves vector memory permanently dirty because index identity can compare against an empty expected model.

## Best Possible Solution

Maintain a canonical memory-core upgrade fix plan that decides provider migration semantics, repairs empty expected-model identity handling, and proves upgrade behavior for existing Google/Gemini and OpenAI memory indexes.

## Reproduction Assessment

Yes. The reporter's logs map directly to current and v2026.6.6 source: auto still normalizes to OpenAI, the Google memory provider contract is `gemini`, and pre-init identity can still compare against an empty fallback model.

## Solution Assessment

No. The open empty expected-model PR is a useful narrow repair, but this issue also needs a maintainer decision on the upgrade/provider migration contract before it is fully solved.
