## Summary

Keep open: current main still keys bundled MCP runtime ownership and in-flight creation by session id, so separate agent sessions with the same workspace and MCP config can create separate persistent stdio children before idle cleanup runs.

## What This Changes

Prevent gateway bundled MCP lazy loading from spawning multiple persistent stdio MCP children for the same workspace/config across agent sessions.

## Best Possible Solution

Maintain one live bundled MCP runtime per compatible workspace/config fingerprint with single-flight creation, session leases, and deterministic disposal when leases expire or config changes.

## Reproduction Assessment

Yes. The reporter supplied concrete process/log evidence, and source inspection shows a high-confidence current-main path where different session ids with the same workspace/config create independent stdio MCP runtimes.

## Solution Assessment

Yes. A workspace/config-keyed single-flight runtime manager with reference-counted leases is the narrow maintainable fix; process-table cron cleanup only mitigates after duplicate children already exist.

## Repair Work Prompt

Fix the gateway bundled MCP duplicate-stdio-child leak described in https://github.com/openclaw/openclaw/issues/75621. Current main caches bundle MCP runtimes and create-in-flight promises by session id, while embedded attempts pass a per-run session id; repeated sessions with the same workspace and identical mcp.servers config can spawn duplicate stdio children. Implement a narrow source fix in the bundle MCP runtime manager so compatible sessions share one live runtime creation keyed by a stable workspace/config fingerprint, with per-session leases/sessionKey bookkeeping and deterministic disposal on config changes, session cleanup, idle expiry, gateway shutdown, and explicit one-shot cleanup. Preserve tool-policy filtering and do not add a user-facing config option or plugin-specific special case. Add regression coverage for two different session ids with the same workspace/config awaiting/reusing one runtime, in-flight single-flight behavior, cleanup with active leases, config-change disposal, and stdio process disposal. Update CHANGELOG.md with a user-facing fix entry crediting the reporter if policy allows.
