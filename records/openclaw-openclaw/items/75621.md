## Summary

Keep open: current main and the latest release still key bundled MCP runtime ownership and in-flight creation by session id, so separate gateway sessions with the same workspace and MCP config can create separate persistent stdio children before idle eviction runs.

## What This Changes

The issue reports duplicate bundled stdio MCP child processes for the same gateway parent and MCP config, causing memory and CPU leakage during lazy startup.

## Best Possible Solution

Maintain one live bundled MCP runtime per compatible workspace/config fingerprint with reference-counted session leases and deterministic teardown on config, session, and gateway cleanup paths.

## Reproduction Assessment

Yes. The reporter supplied concrete process/log evidence, and source inspection shows a high-confidence static reproduction path: request bundled MCP runtimes for two different session ids with the same workspace/config and each id can own a separate stdio transport.

## Solution Assessment

Yes. A workspace/config-keyed single-flight runtime and lease manager is the narrow maintainable fix; a cron or process-table cleanup only mitigates after duplicate children already exist.
