## Summary

Keep this open. The reporter provided concrete v2026.4.29 process and gateway-log evidence, and current main still caches and single-flights bundled MCP runtimes by `sessionId`, so different gateway sessions with the same workspace and MCP config can still create separate stdio child processes until idle cleanup, session cleanup, reload, or shutdown disposes them.

## What This Changes

The issue asks OpenClaw to prevent duplicate long-lived stdio MCP child processes for the same effective gateway MCP server configuration across lazy-loaded agent sessions.

## Best Possible Solution

Implement a bundled MCP runtime ownership layer that single-flights and reuses live stdio runtimes by effective workspace plus normalized MCP server/config fingerprint, with reference-counted session bindings and explicit disposal on config reload, session delete/reset, gateway shutdown, and idle expiry. Avoid command-name process sweeping as the primary fix.

## Reproduction Assessment

Yes. The issue has live process-table and gateway-log evidence, and current main has a static reproduction path: request bundle-MCP materialization for two different session ids with the same workspace/config and observe that the manager creates separate stdio-backed runtimes.

## Solution Assessment

Yes. A config-keyed single-flight/reuse layer with explicit leases is the narrowest maintainable direction; relying on the 10-minute idle TTL or process-name reaping would reduce symptoms but would not prevent duplicate startup work across sessions.

## Repair Work Prompt

Fix the bundled MCP duplicate stdio child leak reported in openclaw/openclaw#75621. Current main caches and single-flights `SessionMcpRuntime` by `sessionId`, so two gateway sessions with the same effective workspace and MCP server config can each start their own stdio child. Implement a narrow runtime ownership layer in the bundled MCP runtime manager that single-flights and reuses live stdio runtimes by effective workspace plus normalized MCP config/server fingerprint, with per-session bindings/leases and explicit disposal on config reload, session delete/reset, gateway shutdown, and idle expiry.
