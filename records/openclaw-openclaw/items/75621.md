## Summary

Keep open: the current manager still owns bundled MCP runtimes and in-flight creation by `sessionId`, so separate sessions with the same workspace and MCP config can still create separate stdio children before idle eviction or cleanup catches up.

## What This Changes

Prevent Gateway bundled MCP lazy loading from spawning duplicate stdio MCP children for the same effective workspace and MCP config.

## Best Possible Solution

Use a compatible workspace/config fingerprint as the runtime ownership key, with single-flight creation and ref-counted session leases, while keeping teardown deterministic on run end, session reset/delete, config reload, idle sweep, and gateway shutdown.

## Reproduction Assessment

Yes. The reporter supplied concrete process/log evidence, and current source shows different session ids with the same workspace/config can each own a separate bundled MCP runtime that spawns stdio children.

## Solution Assessment

Yes. A config-compatible single-flight runtime manager with leases is the narrow maintainable fix; cron-based process cleanup or shorter idle TTLs only reduce the damage after duplicate children already exist.

## Repair Work Prompt

Fix the bundled MCP duplicate stdio child leak reported at https://github.com/openclaw/openclaw/issues/75621. Current `createSessionMcpRuntimeManager` keys live runtimes and `createInFlight` by `sessionId`, so different sessions with the same effective workspace and MCP config can each spawn the same stdio MCP children. Implement a backwards-compatible ownership key based on compatible workspace/config fingerprint, or an equivalent single-flight design, with session leases/refcounts so one live runtime is reused safely across compatible sessions and disposed only when no active leases remain and cleanup/idle rules allow it. Preserve config/workspace changes creating a fresh runtime, preserve session reset/delete, one-shot run-end cleanup, config reload, and gateway shutdown teardown, and do not add a new config option or plugin-specific workaround. Add regression coverage for two different session ids/session keys with the same workspace/config, including concurrent in-flight creation, active lease protection, config-change recreation, and materialization failure releasing leases. Update MCP docs and CHANGELOG if the user-visible lifecycle wording changes.
