## Summary

Keep open: current main still keys bundled MCP runtime reuse and in-flight creation by session id, so two gateway sessions with the same workspace and MCP config can create separate persistent stdio children before idle cleanup runs.

## What This Changes

The issue requests preventing duplicate stdio MCP child processes for the same gateway-owned MCP server/config across separate lazy-spawned agent sessions.

## Best Possible Solution

Add config-fingerprinted single-flight ownership for bundled MCP runtimes, with explicit leases and lifecycle cleanup preserved across reload, shutdown, idle expiry, and failed startup.

## Reproduction Assessment

Yes. The reporter provided live process/log evidence, and current main has a static reproduction path: request bundle-MCP runtimes for two different session ids with the same workspace/config and observe that each can create its own stdio transport.

## Solution Assessment

Yes. A config-keyed single-flight runtime/lease manager is the narrow maintainable fix; an external process-table sweeper only mitigates after duplicate children already exist.
