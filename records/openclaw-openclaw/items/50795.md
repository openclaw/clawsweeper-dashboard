The issue reports that live embedded compaction zeroes all assistant usage snapshots, causing the TUI context counter to stay at 0 after compaction.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82874, or replace it with an equivalent narrow handler fix that applies the existing compaction boundary pattern to live session usage cleanup.

## Reproduction Assessment

Yes, at source level. A session with a compactionSummary followed by a fresh assistant message with nonzero usage reaches handleCompactionEnd with willRetry false, then the current helper zeroes both assistant snapshots.

## Solution Assessment

Yes. The narrow maintainable fix is to apply the existing replay sanitizer's timestamp/index compaction boundary pattern to the live subscribe handler rather than changing UI or provider code.
