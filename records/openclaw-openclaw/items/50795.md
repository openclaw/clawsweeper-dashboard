## Summary

Keep open: current main and the latest release still run the live embedded compaction-end helper that zeroes every assistant usage snapshot, while adjacent replay sanitization already preserves fresh post-compaction usage. The direct fix PR is closed unmerged, and the merged OpenAI-compatible usage PR explicitly left this helper out of scope.

## What This Changes

The issue asks OpenClaw to preserve assistant usage snapshots created after compaction so the context token counter does not stay at zero.

## Best Possible Solution

Patch the embedded subscribe compaction handler to apply latest-compaction boundary semantics, preserving fresh post-compaction usage while keeping stale pre-compaction snapshots structurally zeroed.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a session message list with a compactionSummary followed by an assistant message with nonzero usage; current main reaches the final compaction_end helper and zeroes that fresh usage.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing replay sanitizer's latest-compaction timestamp/index boundary pattern in the subscribe handler; provider, UI, or status display changes would not stop the live mutation that erases usage.
