## Summary

Keep open: current main and v2026.5.22 still clear every assistant usage snapshot after a non-retry compaction, and the open draft fix PR at https://github.com/openclaw/openclaw/pull/82874 owns the narrow repair path.

## What This Changes

The issue reports that the live embedded compaction handler zeroes fresh post-compaction assistant usage, leaving the context counter at zero after compaction.

## Best Possible Solution

Make the live subscribe compaction handler use the same latest-compaction timestamp/index boundary as replay sanitization, with regression coverage for stale-before and fresh-after usage.

## Reproduction Assessment

Yes, at source level. A session containing a compactionSummary followed by a fresh assistant message with nonzero usage reaches handleCompactionEnd with willRetry false, then the current helper zeroes both assistant snapshots.

## Solution Assessment

Yes. The narrow maintainable fix is to make the live subscribe helper use the existing replay sanitizer boundary pattern; UI or provider changes would not stop the live mutation.
