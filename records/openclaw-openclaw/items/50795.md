## Summary

Keep open: current main and v2026.5.22 still zero every assistant usage snapshot after a non-retry compaction, and the open draft fix PR at https://github.com/openclaw/openclaw/pull/82874 is the active closing path.

## What This Changes

This issue reports that the TUI context token counter stays at zero after compaction because the live compaction-end handler clears assistant usage snapshots indiscriminately.

## Best Possible Solution

Review and land the focused boundary-aware handler fix in https://github.com/openclaw/openclaw/pull/82874, or close that PR only if maintainers choose a different narrow fix for the same helper.

## Reproduction Assessment

Yes, at source level. A session with a compactionSummary followed by a fresh assistant message with nonzero usage reaches handleCompactionEnd with willRetry false, then the current helper zeroes both assistant snapshots.

## Solution Assessment

Yes. The narrow maintainable solution is to apply the existing replay sanitizer's timestamp/index compaction boundary pattern to the live subscribe handler rather than changing UI or provider code.
