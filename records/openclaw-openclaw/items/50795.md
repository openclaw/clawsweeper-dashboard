## Summary

## What This Changes

Fix the embedded compaction handler so fresh assistant usage after compaction is preserved for the TUI context counter.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82874, or an equivalent boundary-aware subscribe-handler patch with focused regression coverage.

## Reproduction Assessment

Yes, at source level. A session with a compactionSummary followed by a fresh assistant message with nonzero usage reaches handleCompactionEnd with willRetry false, then the current helper zeroes both assistant usage snapshots.

## Solution Assessment

Yes. The narrow maintainable fix is to make the live subscribe compaction helper use the same latest-compaction timestamp/index boundary as the replay sanitizer; UI or provider changes would not stop the live mutation.
