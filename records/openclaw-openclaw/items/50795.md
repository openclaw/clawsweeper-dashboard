## Summary

Keep open: current `main` still calls the live subscribe compaction-end helper and that helper still zeroes every assistant usage snapshot without checking the latest `compactionSummary` boundary. The direct fix PR is closed unmerged, while the merged OpenAI-compatible usage PR explicitly left this helper out of scope.

## What This Changes

The issue reports that the TUI context token counter stays at zero after compaction because fresh post-compaction assistant usage is cleared.

## Best Possible Solution

Patch the embedded subscribe compaction handler to identify the latest compaction summary and zero only stale pre-compaction assistant usage, with focused regression coverage for fresh post-compaction usage.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a session message list with a `compactionSummary` followed by an assistant message with nonzero usage; current `main` reaches the non-retry compaction-end helper and zeroes that fresh usage.

## Solution Assessment

Yes. The narrow maintainable fix is to apply the existing replay sanitizer's latest-compaction timestamp/index boundary semantics in the subscribe handler; UI or provider changes would not stop this live mutation.
