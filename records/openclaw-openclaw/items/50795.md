## Summary

Keep open: current main and v2026.5.18 still zero every assistant usage snapshot on non-retry compaction, and the open closing implementation has not merged yet.

## What This Changes

The issue asks to preserve post-compaction assistant usage so the TUI context counter no longer stays at zero after compaction.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82874, or an equivalent handler patch, so the subscribe compaction helper uses the latest compactionSummary timestamp/index boundary with focused regression coverage.

## Reproduction Assessment

Yes, at source level. A session with a compactionSummary followed by a fresh assistant message with nonzero usage reaches handleCompactionEnd with willRetry false, then the current helper zeroes both assistant usage snapshots.

## Solution Assessment

Yes. The narrow maintainable fix is to make the subscribe compaction helper follow the existing replay sanitizer's latest-compaction timestamp/index boundary; provider or UI changes would not stop the live mutation that erases usage.
