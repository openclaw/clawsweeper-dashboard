## Summary

Keep open: current main still has the live embedded compaction bug. The non-retry compaction end path still calls a helper that zeroes every assistant usage snapshot, while the replay sanitizer already has boundary-aware logic and tests that preserve fresh post-compaction usage; the linked fix PR is closed unmerged in the provided issue context.

## What This Changes

The issue asks for context token usage after compaction to preserve fresh post-compaction assistant usage instead of always showing zero.

## Best Possible Solution

Make the live embedded subscribe compaction cleanup use the same latest-compaction boundary semantics as replay sanitization, so stale pre-compaction snapshots are structurally zeroed and fresh post-compaction usage remains available for status/context accounting.

## Reproduction Assessment

Yes. A high-confidence static reproduction is a live session message array with a compactionSummary followed by an assistant message with nonzero usage; current main reaches the non-retry compaction_end helper and zeroes that fresh assistant usage anyway.

## Solution Assessment

Yes. The narrow maintainable fix is to apply the existing timestamp/index boundary pattern to the subscribe handler; UI-only, provider-only, or session-store-only changes would not stop the live mutation that erases fresh usage.
