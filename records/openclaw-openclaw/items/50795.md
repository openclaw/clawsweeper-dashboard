## Summary

Keep open: current main and the latest release still run the live compaction-end helper that zeroes every assistant usage snapshot, and the open draft PR at https://github.com/openclaw/openclaw/pull/82874 is the active closing fix path.

## What This Changes

The issue asks OpenClaw to preserve fresh post-compaction assistant usage so the TUI context counter no longer stays at zero after compaction.

## Best Possible Solution

Land or replace the linked compaction-handler fix so live compaction uses the same latest-summary boundary as replay sanitization, with regression coverage for stale-before and fresh-after assistant usage.

## Reproduction Assessment

Yes, at source level. A session with a compaction summary followed by a fresh assistant message with nonzero usage reaches `handleCompactionEnd` with `willRetry: false`, then the current helper zeroes both stale and fresh assistant snapshots.

## Solution Assessment

Yes. The narrow maintainable fix is to make the live subscribe helper use the existing replay sanitizer boundary pattern; UI or provider changes would not stop the live mutation.
