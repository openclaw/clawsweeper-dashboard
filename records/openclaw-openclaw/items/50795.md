## Summary

Keep open: current main and the latest release still zero every assistant usage snapshot after a non-retry compaction, and the open closing PR has not merged yet.

## What This Changes

The issue reports that the live embedded compaction handler zeroes fresh post-compaction assistant usage snapshots, causing context token usage to display as zero after compaction.

## Best Possible Solution

Apply the boundary-aware stale-usage clearing used by replay sanitization to the live subscribe compaction handler, with focused regression coverage for stale-before and fresh-after compaction usage.

## Reproduction Assessment

Yes, at source level. A session with a compactionSummary followed by a fresh assistant message with nonzero usage reaches handleCompactionEnd with willRetry false, then the current helper zeroes both assistant usage snapshots.

## Solution Assessment

Yes. The narrow maintainable fix is to make the live subscribe compaction helper use the same latest-compaction timestamp/index boundary as the replay sanitizer; UI or provider changes would not stop the live mutation.
