## Summary

Keep open: current main still has the reported live embedded compaction behavior. The handler still zeros every assistant usage snapshot after a non-retry compaction end, while the replay sanitizer has the boundary-aware implementation needed here; the referenced fix PR is closed unmerged.

## What This Changes

The issue requests preserving fresh post-compaction assistant usage so the TUI context token counter does not remain at 0 after compaction.

## Best Possible Solution

Patch the embedded subscribe compaction handler to identify the latest compaction summary and clear only assistant usage before that boundary, matching the replay sanitizer's timestamp/index fallback behavior. Add focused regression coverage in the subscribe/handler path and leave provider usage parsing, UI display, and replay sanitizer behavior unchanged unless a tiny shared helper is extracted cleanly.

Yes. A high-confidence static reproduction is a session containing a `compactionSummary` followed by an assistant message with nonzero `usage`, then a `compaction_end` event with `willRetry: false`; current main reaches the helper and zeroes that fresh assistant usage.

## Solution Assessment

Yes. The narrow fix is to reuse the existing boundary-aware compaction pattern in the live subscribe handler; UI-only or provider-only changes would not stop the live session mutation that erases fresh usage.
