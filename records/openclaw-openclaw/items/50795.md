## Summary

Keep open: current main and the latest stable release still run the live compaction-end helper that zeroes every assistant usage snapshot, while adjacent replay sanitization already preserves fresh post-compaction usage. The direct fix PR is closed unmerged, and the merged OpenAI-compatible usage PR explicitly left this helper out of scope.

## What This Changes

The issue asks to preserve fresh assistant usage after compaction so the context token counter does not stay at 0.

## Best Possible Solution

Patch the embedded subscribe compaction handler to identify the latest compaction summary and zero only stale pre-compaction assistant usage, with focused regression coverage for fresh post-compaction usage.

## Reproduction Assessment

Yes, at source level. A session message list with a compactionSummary followed by a fresh assistant message with nonzero usage reaches handleCompactionEnd with willRetry false, then the current helper zeroes both assistant usage snapshots.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse or extract the existing replay sanitizer's latest-compaction timestamp/index boundary pattern in the subscribe handler; provider, UI, or status display changes would not stop the live mutation that erases usage.
