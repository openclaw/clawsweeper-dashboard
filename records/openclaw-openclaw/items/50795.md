Keep open: current main still has the reported live embedded compaction bug, because the non-retry compaction end path still zeroes every assistant usage snapshot while the replay sanitizer already preserves fresh post-compaction usage.

## What This Changes

The issue requests that compaction cleanup preserve fresh post-compaction assistant usage so the TUI context counter no longer shows 0 after compaction.

## Best Possible Solution

Patch the embedded subscribe compaction handler to identify the latest compaction boundary and zero only stale pre-compaction assistant usage, with focused regression coverage and a changelog entry.

## Solution Assessment

Yes. The narrow maintainable fix is to apply the existing timestamp/index boundary semantics to the subscribe handler; UI, provider, or session-store-only changes would not stop the live mutation.
