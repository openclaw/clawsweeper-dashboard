## Summary

## What This Changes

The issue requests actionable recovery for generic LiteLLM Vertex beta INVALID_ARGUMENT failures that can wedge long agent sessions.

## Best Possible Solution

Add a narrow maintainer-approved Vertex/LiteLLM recovery policy that preserves redacted provider detail, distinguishes recoverable request-shape or context signals from generic validation, cleanly fails the lane, and only compacts, falls back, or resets under that rule.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainers choose the recovery policy. A narrow classifier plus lane cleanup is likely the right direction, while automatic reset or fallback for every generic INVALID_ARGUMENT would be too broad.
