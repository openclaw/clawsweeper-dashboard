## Summary

Keep open. Current main and the latest release still leave the reported generic LiteLLM/Vertex beta `INVALID_ARGUMENT` shape outside the context-overflow and provider-request recovery classifiers, and the related stuck-session/context-overflow work only partially overlaps this provider-specific recovery decision.

## What This Changes

The issue asks OpenClaw to classify and recover from LiteLLM Vertex beta 400 `INVALID_ARGUMENT` failures in long sessions instead of surfacing generic failure copy and leaving the session unreliable.

## Best Possible Solution

Add a narrow maintainer-approved LiteLLM/Vertex provider-session recovery path for the proven error shape, preserve redacted provider details, and prove the failed turn releases or marks the session lane cleanly.

## Reproduction Assessment

No, not for the full wedged-session behavior. The classifier and user-facing recovery gap is source-reproducible on current main, but a live LiteLLM Vertex beta run is still needed to prove the exact provider object shape and lane state.

## Solution Assessment

Unclear. The issue identifies a real gap, but maintainers need to choose whether the right recovery is provider-owned classification, compaction, fallback, non-beta routing, session cleanup, or some combination.
