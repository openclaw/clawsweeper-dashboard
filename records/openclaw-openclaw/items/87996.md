## Summary

Keep open. ## What This Changes

The issue requests actionable recovery, structured provider details, and clean session-lane failure handling when LiteLLM Vertex beta returns a generic 400 `INVALID_ARGUMENT` during long sessions.

## Best Possible Solution

Keep this open for a maintainer-led provider/session recovery decision: add narrow LiteLLM/Vertex-aware classification and sanitized logging only when the provider shape is proven, and verify that the failed turn cleanly releases the session lane without treating every generic `INVALID_ARGUMENT` as context overflow.
