## Summary

Keep open: current main still hardcodes the 0.9 Pi preemptive overflow threshold and the config schema/docs do not expose a ratio knob, while an open linked PR is already the implementation candidate for this issue.

## What This Changes

Request to add a public `agents.defaults.compaction.preemptiveOverflowRatio` config option, defaulting to `0.9`, so Pi tool-loop preemptive overflow compaction can trigger earlier than the hardcoded 90% threshold.

## Best Possible Solution

Review and either land the linked PR after product/docs approval, or choose a higher-level compaction policy while keeping the config type, schema, help text, docs, and runtime behavior aligned.

## Reproduction Assessment

Not applicable; this is a config-surface request rather than a broken documented behavior. Source inspection confirms current main still hardcodes `0.9` and the strict compaction config has no `preemptiveOverflowRatio` key.

## Solution Assessment

Unclear until maintainers approve the public contract. Exposing the ratio is a plausible narrow implementation, but a higher-level compaction policy may be safer than committing to a raw threshold knob.
