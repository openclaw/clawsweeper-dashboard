Keep open: current main and the latest release still use a hardcoded 0.9 Pi tool-result preemptive overflow threshold, and the public compaction config contract does not expose a ratio. Expose the Pi tool-result preemptive overflow threshold as an `agents.defaults.compaction` configuration option with the current 0.9 behavior preserved by default.

## Best Possible Solution
