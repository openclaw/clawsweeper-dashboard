## Summary

Keep open: current main still hardcodes the 0.9 preemptive overflow threshold and does not expose `agents.defaults.compaction.preemptiveOverflowRatio` in the public config contract. The open implementation PR at https://github.com/openclaw/openclaw/pull/80889 explicitly closes this issue, so this issue should remain open until that PR is reviewed and either lands or is abandoned.

## What This Changes

Request to expose the Pi embedded runner's preemptive overflow threshold as a typed `agents.defaults.compaction` configuration option while preserving the current `0.9` default.

## Best Possible Solution

Review https://github.com/openclaw/openclaw/pull/80889 as the implementation path; if maintainers approve the contract, land a typed, validated, documented `agents.defaults.compaction.preemptiveOverflowRatio` with default `0.9` and focused runtime/config coverage.
