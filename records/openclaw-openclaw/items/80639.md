## Summary

Keep open: the existing ClawSweeper comment still matches current `origin/main`: the Pi tool-loop threshold is hardcoded at `0.9`, and the strict `agents.defaults.compaction` type/schema/docs do not expose an equivalent knob. This is a new public compaction config contract, so it needs maintainer product/config review rather than cleanup closure.

## What This Changes

Request to expose the hardcoded Pi tool-loop preemptive overflow threshold ratio as a typed `agents.defaults.compaction` config option with default `0.9`.

## Best Possible Solution

If maintainers want this tunable, add a bounded documented `agents.defaults.compaction` setting, preserve the `0.9` default, wire it through the guard, and cover schema/runtime/docs/tests; otherwise keep the threshold internal.

## Reproduction Assessment

Not applicable; this is a config-surface feature request rather than a broken documented behavior. Source inspection is high confidence that the ratio remains hardcoded and the proposed key is absent from the strict config contract.

## Solution Assessment

Unclear; exposing a ratio is plausible, but maintainers should decide whether the stable public contract is a raw `(0, 1)` ratio, a higher-level compaction aggressiveness setting, or no new knob.
