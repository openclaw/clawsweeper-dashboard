## Summary

Keep open. Current main still hardcodes the 0.9 tool-result guard threshold and the strict compaction config schema does not expose a ratio knob, but adding one is a new runtime config/API decision rather than a broken existing contract.

## What This Changes

Request to expose the hardcoded preemptive compaction overflow ratio as a configurable agent compaction setting with a default of 0.9.

## Best Possible Solution

If maintainers want this tuning, add a validated compaction setting with default-preserving behavior, wire it into the tool-result guard, document how it interacts with reserve-token prechecks, and add focused schema/runtime tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that the requested knob is absent and the current threshold is still hardcoded at 0.9.

## Solution Assessment

Unclear. Exposing a ratio may be useful, but maintainers should decide whether this belongs as a new ratio setting, an extension of `midTurnPrecheck`, or documentation around existing reserve-token controls.
