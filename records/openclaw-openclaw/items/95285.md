Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep bridge reads behind the capability contract:** `extensions/memory-wiki/src/bridge.ts:217-220`
  - body: The fallback reaches into `../../memory-core/src/public-artifacts.js` whenever the active helper returns no artifacts and the slot string is `memory-core`. If the capability is absent because memory-core is disabled, not activated, or failed registration, memory-wiki can still enumerate memory artifacts and bypass the documented `publicArtifacts` contract; repair the active-memory handoff instead.
  - confidence: 0.9
- **[P2] Require cfg before calling artifact providers:** `extensions/memory-wiki/src/bridge.ts:205-207`
  - body: `resolveBridgePublicArtifacts` accepts `cfg` as optional but immediately passes the params object to providers whose contract requires `cfg: OpenClawConfig`. Make `cfg` required here or return before provider calls so strict TypeScript and runtime callers cannot pass undefined config through the provider path.
  - confidence: 0.82
