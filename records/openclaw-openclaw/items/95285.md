Overall correctness: patch is incorrect

Full review comments:

- **[P1] Keep bridge reads behind the capability contract:** `extensions/memory-wiki/src/bridge.ts:214-218`
  - body: This fallback keys off `plugins.slots.memory` and imports memory-core private source when the active helper returns no artifacts. If the capability is absent because memory-core is disabled, not activated, or failed registration, memory-wiki can still enumerate artifacts outside the documented `publicArtifacts` contract; repair the active-memory handoff instead.
  - confidence: 0.9
- **[P2] Require cfg before calling artifact providers:** `extensions/memory-wiki/src/bridge.ts:205-208`
  - body: `resolveBridgePublicArtifacts` accepts `cfg?: OpenClawConfig` but immediately passes the params object to artifact providers whose contract requires `cfg: OpenClawConfig`. Keep `cfg` required here or return before provider calls so strict TypeScript and runtime callers cannot pass undefined config through the provider path.
  - confidence: 0.82
