- **[P1] Keep bridge reads behind the publicArtifacts contract:** `extensions/memory-wiki/src/bridge.ts:216-218`
  - body: This fallback imports `../../memory-core/src/public-artifacts.js` from memory-wiki whenever the active helper returns no artifacts and the slot string is `memory-core`. That can enumerate memory-core artifacts when the capability is absent because the plugin was disabled, not activated, or failed registration, and it violates the documented plugin boundary; fix the active-memory capability handoff instead.
  - confidence: 0.9
- **[P2] Require config before calling artifact providers:** `extensions/memory-wiki/src/bridge.ts:204-206`
  - body: `resolveBridgePublicArtifacts` accepts `cfg?: OpenClawConfig` but passes the whole params object to `listActiveMemoryPublicArtifacts`, whose contract requires `cfg: OpenClawConfig`. Make `cfg` required here or return before calling providers so strict TypeScript and runtime callers cannot pass undefined config through the provider path.
  - confidence: 0.82
