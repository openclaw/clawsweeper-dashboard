## Best Possible Solution

Keep bridge/status artifact reads aligned with the effective active memory plugin: either repair the publicArtifacts capability handoff, or constrain the memory-core fallback to the same activation rules, then add redacted real CLI proof.

## Solution Assessment

No. The public SDK seam is better than the earlier private import, but the fallback still checks only the slot id and not effective plugin enablement, so a generic capability handoff or enablement-gated fallback is safer.
