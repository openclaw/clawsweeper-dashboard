- **[P1] Keep bridge reads behind the capability contract:** `extensions/memory-wiki/src/bridge.ts:215-220`
  - body: This fallback keys only off `plugins.slots.memory` and then imports memory-core private source. If the capability is absent because memory-core is disabled, not activated, or failed registration, the bridge can still enumerate workspace memory artifacts; fix the active-memory registration/loading path instead of bypassing the publicArtifacts contract.
  - confidence: 0.9
- **[P2] Require cfg before calling artifact providers:** `extensions/memory-wiki/src/bridge.ts:205-207`
  - body: `resolveBridgePublicArtifacts` accepts `cfg?: OpenClawConfig` but immediately passes that object to providers whose contract requires `cfg: OpenClawConfig`. Keep `cfg` required for this helper or return before calling artifact helpers so strict TypeScript and runtime callers cannot pass an undefined config through the provider path.
