## What This Changes

The PR changes memory-core phase-signal reads to rethrow unexpected I/O errors instead of returning an empty store.

## Best Possible Solution

Keep the shipped state-backed implementation and the canonical merged fix path; do not merge this obsolete conflicting branch.

## Reproduction Assessment

Yes for the historical bug: the old reader swallowed nonrecoverable phase-signal read failures and the write path could replace stored signals afterward. Current main no longer has that failing path because read errors propagate before writePhaseSignalStore runs.

## Solution Assessment

Yes. The best solution is the shipped canonical fix/current state-backed reader, which removes the silent empty-store fallback on nonrecoverable read failure while keeping the existing workspace-level failure handling.
