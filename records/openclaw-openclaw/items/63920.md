## Summary

Keep open. Current main still does not expose Slack global shortcut or message-action shortcut payloads through the bundled Slack plugin: the interaction entrypoint only wires block actions and modal lifecycle handlers, the docs/types/tests still describe block/modal interactions only, and the pinned Bolt dependency has the shortcut listener and payload types needed for a narrow plugin-local implementation.

## What This Changes

The issue asks OpenClaw’s bundled Slack plugin to expose Slack global shortcut and message-action shortcut events through `channels.slack` system events.

## Best Possible Solution

Add first-class shortcut handling inside `extensions/slack/src/monitor/events`, using Bolt `app.shortcut(...)`, existing sanitizer/session/auth helpers, tests for global and message shortcuts, Slack docs, and a user-facing changelog entry.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists on current main: Slack interaction registration and docs cover block actions and modal lifecycle events only, while searches find no shortcut registration or shortcut payload handling.

## Solution Assessment

Yes. A plugin-local Slack shortcut handler is the narrowest maintainable solution; a new core `SystemEventType`, broad SDK shortcut API, or config surface is not required for the first pass.
