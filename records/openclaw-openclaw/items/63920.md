## Summary

Keep open. Current main still does not expose Slack global shortcut or message-action shortcut payloads through the bundled Slack plugin: the interaction entrypoint registers block actions and modal lifecycle handlers only, the structured payload types/docs omit shortcuts, and Bolt 4.7.2 has the `app.shortcut(...)` API needed for a narrow plugin-local implementation.

## What This Changes

Expose Slack global shortcut and message-action shortcut payloads from the bundled Slack plugin as structured Slack interaction system events.

## Best Possible Solution

Add first-class shortcut handling inside `extensions/slack/src/monitor/events`, using Bolt `app.shortcut(...)`, existing mismatch checks, existing sanitizer/session helpers, and `authorizeSlackSystemEventSender` for interactive auth. Cover global and message shortcuts in tests, document the emitted payloads, and add a user-facing changelog entry credited to the reporter.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: current main has block-action and modal registration but no shortcut registration or shortcut tests/docs, while the pinned Bolt dependency exposes the shortcut listener and payload types needed for the feature.

## Solution Assessment

Yes. The requested plugin-local handler is the narrowest maintainable fix; a new core `SystemEventType` or broad SDK shortcut registration API is not required for the first pass.
