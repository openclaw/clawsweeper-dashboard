## Summary

Keep open: current main and v2026.6.8 still route Slack block actions and modal lifecycle events, but not Slack global or message shortcuts, so the requested system-event path remains missing and belongs in the Slack plugin boundary.

## What This Changes

The issue requests Slack global shortcut and message-action shortcut events to be routed through the Slack channel as OpenClaw system events.

## Best Possible Solution

Add a Slack-plugin-owned shortcut handler that ACKs first, filters and authorizes like other Slack interactions, emits compact sanitized system events for global and message shortcuts, and updates focused Slack tests and docs without adding core or config surface.

## Reproduction Assessment

Yes. Source inspection reproduces the gap: block actions and modal lifecycle events are registered and tested, while current main and v2026.6.8 have no Slack shortcut listener or tests.

## Solution Assessment

Yes. A Slack-plugin-local handler is the narrowest maintainable solution; it should reuse the existing ACK-first, sanitizer, auth, and session-event patterns, with a user-scoped routing choice for global shortcuts.
