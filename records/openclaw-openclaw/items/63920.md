## Summary

Keep open. Current main still does not expose Slack global shortcut or message-action shortcut payloads through the bundled Slack channel: the Slack interaction monitor registers block actions and modal lifecycle events only, the structured interaction type and docs omit shortcut variants, and Bolt 4.7.1 provides the `app.shortcut(...)` API needed for a narrow Slack-plugin fix.

## What This Changes

Expose Slack `shortcut` and `message_action` interactions through `channels.slack` as sanitized system events with stable session/context routing.

## Best Possible Solution

Add first-class shortcut handling inside the bundled Slack plugin, using Bolt `app.shortcut(...)`, existing mismatch/auth checks, existing interaction payload sanitization, and stable Slack system-event session/context keys. The docs should list `shortcut` and `message_action` alongside block actions and modal lifecycle events, with focused tests for global shortcut, message shortcut, authorization, and redaction behavior.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: current main has no `ctx.app.shortcut` registration or shortcut tests, while Bolt's packaged types provide the shortcut listener and payload types the issue describes.

## Solution Assessment

Yes. The narrowest maintainable fix is in `extensions/slack/src/monitor/events`, reusing the existing interaction sanitizer/auth/session helpers; a broader core API or generic SystemEventType change is not required for the first pass.
