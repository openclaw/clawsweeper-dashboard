## Summary

Keep open. Current main still matches the reported literal per-agent `session.store` failure mode: the gateway `sessions.list` path depends on the combined-session loader, and that loader still returns from the non-template literal branch before all-agent discovery runs. The issue is already paired with open closing PR #54716, so it should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue asks for `sessions.list` and the Control UI Sessions dashboard to include all active sessions, including Telegram and newly created sessions, instead of only the main session when multiple per-agent session stores exist.

## Best Possible Solution

Review and land #54716, or an equivalent current-main fix, so literal paths matching `agents/<id>/sessions/sessions.json` use all-agent discovery while true shared literal stores keep their current single-store semantics. The fix should include regression coverage for a literal `agents/main/sessions/sessions.json` configuration.
