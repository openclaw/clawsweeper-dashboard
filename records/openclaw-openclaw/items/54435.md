## Summary

Keep open: current main and the latest release still short-circuit non-template literal session stores, and the linked closing PR remains open rather than merged.

## What This Changes

The issue requests that `sessions.list` and the Control UI show sessions from sibling per-agent stores when `session.store` is configured as a literal `agents/<id>/sessions/sessions.json` path.

## Best Possible Solution

Port, land, or replace https://github.com/openclaw/openclaw/pull/54716 so literal per-agent paths use same-root discovery while true shared literal stores preserve single-store behavior, with focused regression coverage.

## Reproduction Assessment

Yes, at source level. Configure `session.store` to a literal `.../agents/main/sessions/sessions.json` path with a sibling agent store and call `sessions.list`; current main returns from the literal branch before sibling discovery, and the linked proof shows the count moving from 1 to 2 on the fix branch.

## Solution Assessment

Yes for the requested direction, but not by closing this issue now. The maintainable path is to land or replace the linked PR after porting it to the active combined-store loader and preserving shared literal-store compatibility.
