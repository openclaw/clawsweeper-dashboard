## Summary

Keep open: current `main` and the latest release still return early for non-template literal store paths, and the linked closing PR remains open and conflicting rather than merged.

## What This Changes

The issue requests that `sessions.list` and the Sessions dashboard include all per-agent sessions when `session.store` is configured as a literal per-agent store path.

## Best Possible Solution

Port or replace https://github.com/openclaw/openclaw/pull/54716 so literal per-agent paths under `agents/<id>/sessions/sessions.json` discover same-root sibling stores while true shared literal stores keep single-store behavior, with focused regression coverage.

## Reproduction Assessment

Yes, at source level. ## Solution Assessment

Yes for the requested direction, but not by closing this issue now. The maintainable path is to land or replace the linked PR after porting it to the active combined-store loader and preserving shared literal-store compatibility.
