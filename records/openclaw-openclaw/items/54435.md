## Summary

Keep open: current main still returns early for non-template literal `session.store` paths, so sibling per-agent stores remain invisible, and the linked closing PR is still open and conflicting rather than merged.

## What This Changes

The issue reports that `sessions_list`/`sessions.list` and the Sessions dashboard only show the main session when multiple active sessions exist.

## Best Possible Solution

Review, repair, and land https://github.com/openclaw/openclaw/pull/54716 or an equivalent current-main fix so literal per-agent paths under `agents/<id>/sessions/sessions.json` use same-root all-agent discovery while true shared literal stores keep single-store behavior.

## Reproduction Assessment

Yes. Configure `session.store` to a literal `.../agents/main/sessions/sessions.json` path with a sibling agent store and call `sessions.list`; current-main source returns from the literal-store branch before sibling discovery.

## Solution Assessment

Yes for the issue direction, but not by closing this issue now. The narrow maintainable solution is to land the linked PR or an equivalent current-main port in `src/config/sessions` with regression coverage for the literal per-agent path.
