## Summary

Keep open: current main and the latest release still short-circuit literal non-template `session.store` paths before sibling agent-store discovery, and the open linked PR that closes this issue has not merged.

## What This Changes

The issue reports that `sessions_list` and the Control UI only show the main session when multiple per-agent sessions exist under a literal session-store path.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/54716, or an equivalent port, so literal per-agent paths under `agents/<id>/sessions/sessions.json` use same-root all-agent discovery while true shared literal stores preserve single-store behavior.

## Reproduction Assessment

Yes, at source level. Configure `session.store` to a literal `.../agents/main/sessions/sessions.json` path with a sibling agent store and call `sessions.list`; current main returns from the literal branch before sibling discovery.

## Solution Assessment

Yes, the linked PR's direction is the right narrow fix, but the issue should not close before that PR or an equivalent port lands. The fix should preserve shared literal-store compatibility and add a regression for the literal per-agent path.
