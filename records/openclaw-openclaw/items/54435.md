## Summary

Keep open. Current main still has a literal `session.store` branch that reads one default-agent store and returns before sibling session-store discovery, and the linked closing PR at https://github.com/openclaw/openclaw/pull/54716 is still open with review work remaining.

## What This Changes

The issue asks for `sessions.list` and the Control UI sessions dashboard to show all relevant active sessions instead of only the main session, especially when `session.store` is a literal per-agent path.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/54716 or an equivalent current-main fix that sends literal `agents/<id>/sessions/sessions.json` stores through same-root session-store discovery while preserving true shared literal stores.

## Reproduction Assessment

Yes. Configure `session.store` to a literal per-agent path such as `.../agents/main/sessions/sessions.json` with a sibling agent store, then call `sessions.list`; source inspection shows current main reads only the literal default-agent store and returns before sibling discovery.

## Solution Assessment

Yes. The narrow maintainable direction is to detect literal agent-layout stores and use existing session-store discovery for the same agents root, while keeping non-agent-layout literal stores on the single-store path.
