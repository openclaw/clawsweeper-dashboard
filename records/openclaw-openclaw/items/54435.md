## Summary

Keep open: current `main` still short-circuits non-template literal `session.store` paths before all-agent discovery, and the open closing PR is unmerged and currently conflicting. The linked PR and its structured proof are useful evidence for the fix path, but the issue should not close until that work or an equivalent port lands.

## What This Changes

The issue reports that `sessions_list` and the Control UI Sessions dashboard only show the main session when additional agent or channel sessions exist.

## Best Possible Solution

Review, port, and land https://github.com/openclaw/openclaw/pull/54716 or an equivalent fix so literal per-agent paths under `agents/<id>/sessions/sessions.json` use same-root sibling discovery while true shared literal stores keep single-store behavior.

## Reproduction Assessment

Yes, at source level. ## Solution Assessment

Yes for the proposed direction, but not by closing this issue now. The maintainable path is to port or replace the linked PR against the current combined-store loader and preserve compatibility for true shared literal stores.
