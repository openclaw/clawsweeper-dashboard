Keep open. Current main still returns early for literal non-template `session.store` paths, so sibling per-agent stores remain invisible, and the linked closing PR is open rather than merged.

## What This Changes

The issue reports that `sessions_list` and the Sessions dashboard only show the main session when other active agent/channel sessions exist.

## Best Possible Solution

Review, repair, and land https://github.com/openclaw/openclaw/pull/54716 or an equivalent current-main fix in the `src/config/sessions` loader, preserving true shared literal-store behavior and adding a literal `agents/main/sessions/sessions.json` regression.

## Reproduction Assessment

Yes. Configure `session.store` to a literal `.../agents/main/sessions/sessions.json` path with a sibling agent store and call `sessions.list`; current-main source returns from the literal-store branch before sibling discovery.

## Solution Assessment

Yes for the issue direction, but not by closing this issue now. The narrow maintainable solution is to land the linked PR or an equivalent current-main port in `src/config/sessions` with regression coverage.
