## Summary

Keep open: current main and the latest shipped release still short-circuit literal `session.store` paths before sibling agent-store discovery, and the open closing PR remains unmerged and conflicting.

## What This Changes

The issue reports that `sessions_list`/`sessions.list` and the Control UI Sessions dashboard only show the main session when other active agent or channel sessions exist.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/54716 with a current-main fix that lets literal per-agent paths under `agents/<id>/sessions/sessions.json` use same-root all-agent discovery while preserving true shared literal-store behavior.

## Reproduction Assessment

Yes. Configure `session.store` to a literal `.../agents/main/sessions/sessions.json` path with a sibling agent store and call `sessions.list`; current-main source returns from the literal-store branch before sibling discovery.

## Solution Assessment

Yes for the issue direction, but not by closing this issue now. The maintainable fix is to land the linked PR or an equivalent current-main port with regression coverage for the literal per-agent path.
