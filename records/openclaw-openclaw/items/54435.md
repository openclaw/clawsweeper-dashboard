## Summary

Keep open. Current main still short-circuits literal `session.store` paths through a single default-agent store, and the open closing PR at https://github.com/openclaw/openclaw/pull/54716 is unmerged and conflicting with review work remaining.

## What This Changes

The issue reports that `sessions.list` and the Control UI only show the main session when multiple per-agent sessions exist, especially with a literal per-agent `session.store` path.

## Best Possible Solution

Review, repair, and land https://github.com/openclaw/openclaw/pull/54716 or an equivalent current-main fix so literal `agents/<id>/sessions/sessions.json` stores discover sibling agent stores while true shared literal stores remain single-store.

## Reproduction Assessment

Yes. Configure `session.store` to a literal per-agent path such as `.../agents/main/sessions/sessions.json` with a sibling agent store, then call `sessions.list`; current-main source inspection shows the loader reads only the literal default-agent store and returns before sibling discovery.

## Solution Assessment

Yes for the issue direction. The narrow maintainable fix is to detect literal agent-layout stores in the active config/sessions loader and route them through same-root session-store discovery, while preserving true shared literal stores on the single-store path.
