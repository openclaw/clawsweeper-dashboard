## Summary

Keep open. Current main and latest release still short-circuit literal non-template `session.store` paths through one default-agent store, and the linked closing PR remains open and conflicting rather than merged.

## What This Changes

The issue reports that `sessions_list` returns only the main session when multiple sessions, including Telegram or newly spawned sessions, exist.

## Best Possible Solution

Review, repair, and land https://github.com/openclaw/openclaw/pull/54716 or an equivalent current-main fix that treats literal `agents/<id>/sessions/sessions.json` paths as same-root multi-agent stores while preserving true shared literal stores.

## Reproduction Assessment

Yes. Configure `session.store` to a literal per-agent path such as `.../agents/main/sessions/sessions.json` with a sibling agent store, then call `sessions.list`; current-main source returns from the literal-store branch before sibling discovery.

## Solution Assessment

Yes for the issue direction, but not by closing this issue now. The narrow maintainable solution is to repair and land the linked closing PR or an equivalent current-main fix with regression coverage for the literal path.
