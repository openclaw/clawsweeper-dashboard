The issue asks for `sessions_list` and the Control UI Sessions view to include all active/per-agent sessions instead of only the main session when `session.store` is configured as a literal per-agent path.

## Best Possible Solution

Refresh and land the linked fix, or an equivalent narrow patch, so literal paths matching `agents/<id>/sessions/sessions.json` discover sibling agent stores while true shared literal stores keep their existing single-store behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: configure `session.store` to a literal per-agent `.../agents/main/sessions/sessions.json`, add a sibling agent store, and current main returns before all-agent discovery.

## Solution Assessment

Yes, the linked fix direction is the right narrow solution, but it needs to be refreshed for the current moved loader. The maintainable end state is scoped multi-agent discovery for literal agent-layout stores without changing arbitrary shared literal stores.
