## What This Changes

The PR updates `docs/platforms/windows.md` to add a `dbus-x11` prerequisite, replace the WSL boot task action with `dbus-launch true`, run the task as the current Windows user, and explain the WSL idle-termination workaround.

## Best Possible Solution

Land the docs update if maintainers accept the per-user Ubuntu WSL recipe as the supported manual headless path; otherwise add a scoped caveat or alternate recipe for unsupported account and distro layouts before merge.

## Reproduction Assessment

## Solution Assessment

Yes, if maintainers accept the common per-user Ubuntu WSL scope. For a docs-only manual boot recipe, updating the copied commands plus the package prerequisite is the narrowest maintainable fix; native Windows Gateway task code is a separate path.
