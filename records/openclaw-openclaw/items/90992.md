## Summary

Keep open: this docs-only PR is still useful and has no blocking correctness finding, but it changes an operator-copied WSL boot recipe in a compatibility-sensitive way that needs maintainer acceptance before merge.

## What This Changes

The PR updates `docs/platforms/windows.md` to install `dbus-x11`, replace the WSL boot task's `/bin/true` action with `dbus-launch true`, switch the task from `SYSTEM` to the current Windows user, and explain the WSL idle-termination workaround.

## Best Possible Solution

Land the docs update with the `dbus-x11` prerequisite if maintainers accept the per-user WSL scheduled-task recipe as the supported headless path; otherwise add a precise scope caveat before merge.

## Reproduction Assessment

Yes for source/provided evidence: current main still documents `/bin/true` and `/ru SYSTEM`, external WSL evidence matches the idle-termination failure, and the PR body supplies after-fix WSL 2.7.3 health polling. I did not run a live Windows WSL repro locally.

## Solution Assessment

Yes. For a docs-only manual recipe, updating the exact commands plus the missing `dbus-x11` prerequisite is the narrow maintainable fix; the native Windows Scheduled Task runtime is a separate path and is not implicated by this WSL boot recipe.
