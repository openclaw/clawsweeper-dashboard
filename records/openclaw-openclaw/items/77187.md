## Summary

Keep open: current main still lacks the additional per-list resolver caches, and the production change is focused, but the new perf test leaks process-wide runtime config state under the repo's non-isolated Vitest lanes.

## What This Changes

The PR extends gateway sessions-list rowContext caching for model identity, thinking default, and model cost resolution, adds a perf smoke test, and updates the changelog.

## Best Possible Solution

Land the per-call cache extension after adding test cleanup for runtime config state, keeping the cache scoped to one list operation and avoiding cross-call invalidation.

## Reproduction Assessment

Yes at source level for the remaining non-lightweight list path: current main still invokes display identity, estimated cost, and thinking default resolution per row without per-list memoization. The exact Control UI async profile was not live-replayed, and current main already skips several of those resolvers on lightweight polling rows.

## Solution Assessment

Mostly yes, but the new test should reset the runtime config snapshot before merge. The production approach is narrow because the cached inputs are stable within a single list call and callers without rowContext keep the direct resolver path.
