## Summary

Keep open for maintainer review. The cache implementation is narrowly scoped and CI is green, but the added regression test targets the async lightweight path, which does not exercise the newly cached non-lightweight resolvers.

## What This Changes

The PR extends `SessionListRowContext` with per-list caches for display model identity, thinking default, and model cost resolution, adds a sessions-list perf smoke test, and updates the changelog.

## Best Possible Solution

Keep the per-call cache approach, but retarget the regression test to the non-lightweight row path before landing so the new cache behavior is actually protected.

## Reproduction Assessment

Yes, at source level. Current main's non-lightweight row construction still invokes the deterministic display identity, model-cost, and thinking-default resolvers per row; I did not replay the production CPU profile.

## Solution Assessment

Yes for the production code direction: per-list `rowContext` memoization is the narrow maintainable fix with no public API or cross-call invalidation surface. The test should be adjusted because `listSessionsFromStoreAsync` uses lightweight rows and does not reach the newly cached calls.
