## What This Changes

The branch memoizes repeated gateway sessions.list row resolver work, adds a 1000-row perf smoke, and includes broad release/workflow/plugin changes from an older base.

## Best Possible Solution

Land a narrow gateway-only replacement or rebased PR that memoizes per-list resolver work, fixes regression coverage, and leaves release/workflow changes for separate review.

## Reproduction Assessment

Yes for the review blockers: source inspection shows the new perf smoke omits opts.limit while listSessionsFromStoreAsync defaults to 100 rows, so the 1000-row assertion cannot pass. ## Solution Assessment

No as currently proposed. The cache direction is maintainable, but this PR should be narrowed onto current main, pass an explicit test limit, thread the cache through the remaining cost probe, and split the CI/release changes.
