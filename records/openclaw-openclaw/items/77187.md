## Summary

Keep open: the PR is a focused performance fix that extends the existing sessions-list rowContext cache beyond the current main thinking-level cache, preserves the row shape/API boundary, includes a changelog entry and regression smoke, and the PR head checks I inspected are green.

## What This Changes

The PR adds per-list memoization for sessions-list model identity, thinking default, and model cost resolvers, threads that cache through transcript cost fallback, and adds a gateway perf regression test plus changelog entry.

## Best Possible Solution

Land the focused per-call cache extension after maintainer review, keeping the cache scoped to a single list operation and avoiding public API or cross-call invalidation changes.

## Reproduction Assessment

Yes at source level, not as a live profile: current main still calls the deterministic display identity, thinking default, and model cost resolvers from buildGatewaySessionRow per non-lightweight row, and the PR adds a many-row/few-tuple regression smoke. I did not run that test locally in this read-only review.

## Solution Assessment

Yes. A per-call rowContext cache is the narrow maintainable fix because the cached inputs are stable within one sessions-list call, and callers that do not pass rowContext still use the original direct resolver path.
