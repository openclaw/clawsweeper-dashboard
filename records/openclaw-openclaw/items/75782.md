## Summary

Keep open. Current main still routes non-plugin-harness embedded runs through a sequential `auth` bucket that resolves profiles and awaits provider runtime auth hooks, and I did not find source, test, changelog, or release proof that the reported 8-15s auth latency is fixed.

## What This Changes

The issue asks OpenClaw to diagnose and reduce embedded-run startup `auth` latency for API-key-only model auth profiles.

## Best Possible Solution

Add redacted auth substage timing first, then optimize the measured hot path with a hook-aware cache or fast path that preserves provider runtime auth semantics.

## Reproduction Assessment

Yes for the shipped 2026.4.29 behavior: the issue gives concrete trace commands and repeated timings, and local related context records a maintainer-side staging reproduction around 8-9s. Current-main attribution still needs the same live setup or finer substage instrumentation.

## Solution Assessment

No for the requested blanket API-key skip, because API-key credentials can be provider-transformed today. The maintainable path is instrumentation followed by a targeted optimization for the proven expensive suboperation.
