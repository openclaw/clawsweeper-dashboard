## Summary

Keep open. Current main still has a sequential embedded-run startup auth bucket that includes auth profile selection and runtime auth preparation, and I did not find a current-main or shipped-release fix proving the reported API-key-only latency is gone.

## What This Changes

The issue asks OpenClaw to diagnose and reduce embedded-run startup auth-stage latency for API-key-only model auth profiles.

## Best Possible Solution

Add fine-grained auth substage timing, then optimize the measured hot subpath with a hook-aware fast path or cache that preserves provider runtime auth semantics.

## Reproduction Assessment

Yes for the reported 2026.4.29 behavior: the issue gives concrete trace commands and repeated timings, and source confirms the same stage boundary. Current-main exact wall time still needs the reporter's live setup or equivalent instrumentation.

## Solution Assessment

No for a blanket API-key skip. The maintainable path is instrumentation first, followed by a targeted optimization that keeps prepareRuntimeAuth working for providers that need it.
