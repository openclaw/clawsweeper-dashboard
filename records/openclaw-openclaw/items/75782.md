## Best Possible Solution

Keep this as the focused API-key/custom-provider embedded auth latency tracker, add substage timing around auth-store load, getApiKeyForModel, and prepareProviderRuntimeAuth, then land a measured cache or fast path that preserves provider runtime-auth and profile-failover semantics.
