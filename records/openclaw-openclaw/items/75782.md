## Summary

Keep open. Current main still has a sequential embedded-run startup `auth` bucket that includes auth-store/profile work plus `initializeAuthProfile()`, and there is no source, test, release, or discussion evidence that the reported 10-15s production latency is fixed.

## What This Changes

The issue asks for investigation and optimization of the embedded-run startup `auth` stage, which reportedly blocks 10-15 seconds per turn even with only API-key model auth profiles.

## Best Possible Solution

Add fine-grained auth substage timing first, then optimize the proven hot subpath while preserving the provider runtime auth hook contract.

## Reproduction Assessment

Unclear. The issue gives concrete trace commands and repeated production timings, and current source confirms the bucket boundary, but a high-confidence wall-time reproduction needs the reporter's provider/config/profile setup or equivalent instrumentation.

## Solution Assessment

Unclear for the requested skip or parallelization path. A behavior-preserving substage instrumentation PR is the safest next step; a blanket API-key skip is not currently the best fix because API-key credentials can legitimately pass through provider runtime auth hooks.
