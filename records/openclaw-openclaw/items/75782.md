Keep open: current main still has a sequential embedded-run `auth` startup bucket, and the recent auth/runtime performance work does not prove the reported 10-15s latency is gone.

## What This Changes

Investigate and reduce the embedded-run startup `auth` stage that blocks each turn even when only an API-key model auth profile remains.

## Best Possible Solution

Add low-noise substage timing for embedded-run auth, collect current-main production traces, then optimize the identified store or provider-hook hot path without bypassing runtime auth contracts.
