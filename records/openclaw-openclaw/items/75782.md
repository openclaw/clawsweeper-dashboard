## Summary

Keep open. Current main still awaits embedded-run auth initialization before marking the startup `auth` stage, API-key credentials still pass through provider runtime auth preparation, and the newer open auth reports cover adjacent OAuth or Vertex/tool cases without clearly superseding this API-key custom-gateway report.

## What This Changes

The issue reports a synchronous 10-15s embedded-run startup `auth` stage even when only API-key model auth profiles remain.

## Best Possible Solution

Add fine-grained auth substage timing for this API-key custom-gateway path, then optimize the proven hot subpath while preserving provider `prepareRuntimeAuth` semantics and credential refresh behavior.

## Reproduction Assessment

No high-confidence live reproduction of the 10-15s timing is available from this review. Source inspection does confirm the reported `auth` bucket boundary and the API-key provider-runtime hook path on current main.

## Solution Assessment

No for the withdrawn blanket skip/config direction. The safer solution is targeted instrumentation followed by a measured cache, fast path, or parallelization only where provider hook contracts show it is safe.
