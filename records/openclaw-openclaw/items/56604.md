## Summary

Keep open. Current main still lacks per-dialed-number Voice Call routing: the plugin captures the provider `to` number, but schema, manifest metadata, docs, inbound greeting, response model/prompt/agent selection, realtime consults, and TTS still resolve from one global Voice Call config. A recent maintainer comment also explicitly kept the request open, and the related outbound-task-call item covers a broader outbound workflow rather than this inbound per-number persona route table.

## What This Changes

Add per-phone-number routing for the Voice Call plugin so different dialed numbers can use different voices, personas, prompts, models, greetings, and agents.

## Best Possible Solution

Implement this inside the bundled Voice Call plugin: add a typed, documented per-number route table with global fallback; resolve and persist the matched route or effective config when a call is created; and make inbound greeting, response agent/model/system prompt, realtime consult behavior, and TTS voice selection use that per-call route consistently.
