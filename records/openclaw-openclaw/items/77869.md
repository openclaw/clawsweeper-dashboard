## What This Changes

The issue reports Feishu group messages reaching the agent but not producing visible replies, with DMs and some other group sessions still replying.

## Best Possible Solution

Keep remaining compatibility and diagnostic decisions on https://github.com/openclaw/openclaw/issues/77666, while affected deployments use the documented automatic visible-reply setting or a reliable message-tool send path.

## Reproduction Assessment

Yes from source inspection: a Feishu group turn reaches the shared dispatch path, group/channel visible replies default to `message_tool_only`, and ordinary final text is skipped before final delivery, matching `queuedFinal=false, replies=0`.
