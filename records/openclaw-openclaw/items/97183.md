Full review comments:

- **[P2] Use a value matcher in the Anthropic regression test:** `src/agents/anthropic-transport-stream.test.ts:2431-2434`
  - body: The added test calls `toBe(expect.stringContaining(...))`, so Vitest compares object identity and the current CI shard fails even though the received string contains the expected text. Use `toEqual`, `toContain`, or another value matcher so this regression actually passes.
  - confidence: 0.98
- **[P2] Remove the converter-local structured-output cap:** `src/llm/providers/tool-result-text.ts:25-26`
  - body: `stringifyStructuredBlock` truncates every structured block to 1,000 characters before the existing live tool-result and prompt-history truncation policy runs. Large JSON or resource payloads can still be silently delivered as partial text even when the provider budget could carry more, so this should use the established truncation boundary or have an explicit tested product limit.
  - confidence: 0.86
- **[P2] Make the structured fixtures type-check:** `src/llm/providers/mistral.test.ts:262-266`
  - body: The new Mistral tests use `satisfies Context` while inserting `resource` and `resource_link` blocks and assistant messages without required `usage`, so `check-test-types` fails before the regression coverage can protect the behavior. Cast only the intentional structured-content boundary or construct valid messages.
  - confidence: 0.9
- **[P2] Validate Anthropic image fields before stringifying:** `src/agents/anthropic-transport-stream.ts:330-331`
  - body: The widened Anthropic conversion now accepts `unknown[]` but then stringifies `record.mimeType` and `record.data` for image blocks. If a malformed image-like block reaches this boundary, the provider payload can contain `[object Object]` or empty image data; add a narrow string guard instead of default stringification.
  - confidence: 0.78
