## Summary

Keep open. Current main still rejects POSIX backslash-newline before exec allowlist analysis, and this PR is the canonical implementation path, but the PR head still lets durable approval bypass the new unsupported-shell-syntax denial and lacks required real behavior proof.

## What This Changes

The PR normalizes POSIX backslash-newline continuations before exec allowlist analysis and adds distinct unsupported-shell-syntax denial messages across gateway, node-host, and user-message paths.

## Best Possible Solution

Land an adjusted PR that keeps POSIX continuation normalization, preserves true allowlist-miss semantics, hard-denies unsupported syntax before durable approval reuse unless maintainers explicitly document an exception, and includes focused tests plus real behavior proof.

## Reproduction Assessment

Yes. Source inspection on current main shows backslash-newline returns analysisFailure before allowlist analysis, and the PR blocker is source-reproducible with analysisOk=false plus durableApprovalSatisfied=true in the gateway or node policy path.

## Solution Assessment

No, not as currently proposed. The normalization direction is narrow and useful, but unsupported shell syntax should be denied before durable trust is considered unless maintainers deliberately approve and document a different policy.
