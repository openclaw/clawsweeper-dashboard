## Best Possible Solution

Land an adjusted implementation that keeps POSIX continuation normalization, denies `analysisOk=false` before durable trust unless maintainers explicitly approve otherwise, rebases around the command authorization planner, and includes focused tests plus redacted real exec proof.

## Reproduction Assessment

Yes, source-reproducible. Current main still hard-fails POSIX backslash-newline before allowlist analysis, and PR head still has a source-visible `analysisOk=false` plus `durableApprovalSatisfied=true` bypass path in gateway and node policy.

## Solution Assessment

No, not as currently proposed. The normalization approach is useful, but unsupported shell syntax should fail closed before durable approval reuse unless maintainers explicitly choose and document a different policy.
