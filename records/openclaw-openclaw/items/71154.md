## Summary

Keep open. Current main still rejects backslash-newline commands before allowlist analysis, so the PR is not obsolete, but the branch is not merge-ready because the new unsupported-syntax denial can still be bypassed by durable approval and the changelog entry misses required contributor credit.

## What This Changes

The PR normalizes POSIX backslash-newline continuations before exec allowlist analysis, adds an unsupported-shell-syntax denial signal across gateway and node-host paths, updates tests, and adds a changelog entry.

## Best Possible Solution

Land an adjusted version that keeps POSIX continuation normalization for analyzable allowlisted commands, makes unparseable syntax a hard deny or explicitly documented maintainer-approved exception before durable approval reuse, and aligns tests, docs, and changelog credit.

## Reproduction Assessment

Yes. Current main is source-reproducible: backslash-newline reaches hasShellLineContinuation and returns analysisOk=false, and the existing test asserts that behavior. The PR blocker is also source-reproducible by setting analysisOk=false with durableApprovalSatisfied=true in the gateway or node-host policy paths.

## Solution Assessment

No, not as currently proposed. The normalization and distinct denial direction are narrow and useful, but unsupported shell syntax must be denied before durable trust is considered unless maintainers deliberately approve and document another policy.
