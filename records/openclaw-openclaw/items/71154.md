## Summary

Keep this PR open, but do not merge it as-is: the central bug fix is still useful and not on current main, but the patch leaves unsupported shell syntax bypassable through durable approval paths and still lacks contributor-provided real behavior proof.

## What This Changes

The PR normalizes POSIX backslash-newline continuations before exec allowlist analysis, adds unsupported-shell-syntax denial plumbing across gateway/node paths, updates tests, and adds a changelog entry.

## Best Possible Solution

Land an adjusted PR that keeps the desired line-continuation handling, fail-closes analysisOk=false before durable approval reuse unless maintainers explicitly document an exception, and includes focused tests plus redacted real behavior proof.

## Reproduction Assessment

Yes. Current main source and tests show backslash-newline still produces analysis failure before allowlist analysis, and the PR blocker is source-reproducible with analysisOk=false plus durableApprovalSatisfied=true in gateway or node policy.

## Solution Assessment

No, not as currently proposed. The normalization direction is useful, but unsupported shell syntax is still evaluated after durable trust in the node path and through a gateway helper that excludes durable approvals.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Deny unparseable gateway commands before durable trust:** `src/agents/bash-tools.exec-host-gateway.ts:101-103`
  - body: `resolveGatewayDeniedReason()` first calls `hasGatewayAllowlistMiss()`, and that helper returns false when `durableApprovalSatisfied` is true. With `analysisOk=false`, durable exact-command trust can therefore skip the new unsupported-shell-syntax denial and continue toward execution.
  - confidence: 0.88
- **[P1] Deny unparseable node commands before durable trust:** `src/node-host/exec-policy.ts:124-126`
  - body: The durable approval branch still returns `allowed: true` before the new unsupported-shell-syntax branch runs. In allowlist mode, a persisted durable approval can override `analysisOk=false` instead of returning `SYSTEM_RUN_DENIED: unsupported shell syntax`.
  - confidence: 0.86
