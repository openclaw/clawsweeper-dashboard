## Summary

Keep this PR open: it addresses a real exec allowlist gap that is still present on current main, but the proposed denial split still lets analysisOk=false commands pass through durable approval paths and the external PR lacks real behavior proof.

## What This Changes

The PR normalizes POSIX backslash-newline continuations before exec allowlist analysis, adds an unsupported-shell-syntax denial reason across gateway/node paths, updates tests, and adds a changelog entry.

## Best Possible Solution

Land an adjusted PR that keeps POSIX continuation normalization, preserves true allowlist-miss semantics, hard-denies unsupported syntax before durable approval reuse unless maintainers explicitly document an exception, and includes focused tests plus redacted real behavior proof.

## Reproduction Assessment

Yes. Current main source and tests show backslash-newline still produces analysisFailure before allowlist analysis, and the PR blocker is source-reproducible with analysisOk=false plus durableApprovalSatisfied=true in the gateway or node policy path.

## Solution Assessment

No, not as currently proposed. The normalization direction is narrow and useful, but unsupported shell syntax should be denied before durable trust is considered unless maintainers deliberately approve and document a different policy.
