## What This Changes

The PR normalizes POSIX backslash-newline continuations before exec allowlist analysis, adds an `unsupported-shell-syntax` denial signal across gateway/node-host messaging, updates regression tests, and adds a changelog entry.

## Best Possible Solution

Land an adapted version that preserves the POSIX continuation normalization and clearer parse-failure signal, but makes unparseable allowlist analysis a hard denial before durable approval reuse unless maintainers intentionally define a narrower override. The final branch should include gateway/node-host durable-denial tests and a single-line changelog entry with valid contributor credit.

Yes. For the original bug, current main has a source/test reproduction: a backslash-newline command hits the early `hasShellLineContinuation()` failure path. For the PR blocker, set `analysisOk=false`, `allowlistSatisfied=false`, and durable approval true; the PR head's gateway and node policy paths skip the new unsupported-syntax denial.

## Solution Assessment

No, not as currently proposed. The normalization helper and distinct denial direction are the right narrow fix, but unsupported shell syntax must be denied before durable trust is considered or explicitly handled by a maintainer-approved policy with tests.
