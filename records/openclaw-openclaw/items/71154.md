## Summary

Keep open: current main still has the reported exec allowlist behavior, and this PR contains useful normalization and tests, but the PR head is not merge-ready because the new unsupported-syntax denial can be skipped by durable approvals and no real behavior proof is present.

## What This Changes

The PR normalizes POSIX backslash-newline continuations before exec allowlist analysis, adds unsupported-shell-syntax denial plumbing, and updates gateway/node-host/infra tests plus the changelog.

## Best Possible Solution

Land an adjusted implementation that preserves the continuation normalization, fail-closes unsupported syntax before durable trust unless maintainers approve another policy, and coordinates with the command authorization planner path if it supersedes this PR.

## Reproduction Assessment

Yes. Source inspection of current main shows backslash-newline commands still return `analysisOk=false` before allowlist resolution, and PR-head inspection shows the blocker with `analysisOk=false` plus `durableApprovalSatisfied=true` in both gateway and node policy.

## Solution Assessment

No, not as currently proposed. The normalization approach is useful, but the unsupported-shell-syntax denial needs to run before durable approval reuse or be covered by an explicit maintainer-approved policy exception.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Deny unparseable gateway commands before durable trust:** `src/agents/bash-tools.exec-host-gateway.ts:101-102`
  - body: `resolveGatewayDeniedReason()` delegates to `hasGatewayAllowlistMiss()`, which returns false when durable approval is satisfied. With `analysisOk=false`, durable exact-command trust skips the new unsupported-shell-syntax denial and can proceed toward execution.
  - confidence: 0.88
- **[P1] Deny unparseable node commands before durable trust:** `src/node-host/exec-policy.ts:124-136`
  - body: The new unsupported-shell-syntax branch runs only after the durable approval branch has already returned `allowed: true`. In allowlist mode, a persisted durable approval can still override `analysisOk=false` instead of returning `SYSTEM_RUN_DENIED: unsupported shell syntax`.
  - confidence: 0.86
