## Summary

Keep open. Current main and v2026.6.8 still expose persistent model override behavior only: `sessions.patch` writes session state, `chat.send` has no request-local model field, and the ACK does not report the resolved model; the remaining work needs a maintainer API/product decision.

## What This Changes

Request an additive one-turn model override for `sessions.patch` or `chat.send` so a single agent turn can use a selected model without changing persistent session state.

## Best Possible Solution

Define one additive request-local model override contract at the Gateway/session/runner boundary, then cover non-persistence, resolved-model reporting, provider/auth validation, hooks, docs, and focused protocol tests.

## Reproduction Assessment

Yes, at the source-contract level. Current main rejects a `chat.send` model field, has no `sessions.patch` next-turn scope, persists `sessions.patch.model`, and returns a `chat.send` ACK without resolved model information.

## Solution Assessment

Unclear until maintainers choose the public API shape. The maintainable solution is one canonical additive protocol contract rather than continuing the two-patch sticky workaround or reviving the closed conflicted inline `/model` branch.
