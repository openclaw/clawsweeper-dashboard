## Summary

Keep open. Current main and v2026.6.10 still expose persistent session model overrides only; the requested one-turn Gateway/chat contract remains missing and needs maintainer API direction before implementation.

## What This Changes

Add an atomic one-turn model override for `sessions.patch` or `chat.send` that reports the resolved model without persisting session model state.

## Best Possible Solution

Define one additive request-local model override contract, then implement it across Gateway schema, session/chat handling, runner model selection, allowlist/auth validation, hooks, docs, and focused non-persistence tests.

## Reproduction Assessment

Yes, at the source-contract level. Current main shows persistent `sessions.patch` model changes, no `chat.send` model field, no `agent` modelOverrideSource, and no resolved-model ACK for `chat.send`.

## Solution Assessment

Unclear until maintainers choose the public API shape. The maintainable path is one canonical additive protocol contract, not continuing the two-patch sticky-switch workaround or reviving the closed mixed-message branch.
