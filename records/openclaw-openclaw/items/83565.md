## Summary

Keep open. Current main still lacks an atomic one-turn model override: `sessions.patch` persists model selection into session state, `chat.send` has no caller model field, and the request changes public Gateway/session routing semantics that need maintainer API direction.

## What This Changes

Request an additive one-turn model override API on `sessions.patch` or `chat.send` that auto-reverts, reports the resolved turn model, and distinguishes agent-sourced overrides.

## Best Possible Solution

Choose one canonical public contract, then implement it at the Gateway protocol and runner boundary with SDK, docs, and regression tests proving no durable session mutation and correct resolved-model reporting.

## Reproduction Assessment

Yes at the source-contract level: current schemas and handlers show `sessions.patch` persists model changes and `chat.send` has no caller model field. A live repro is not needed because this is a feature request for a missing API surface, not a regression report.

## Solution Assessment

Unclear until maintainers choose the API shape. The maintainable direction is a single additive protocol contract with session-store non-mutation proof rather than continuing the two-patch sticky-switch workaround.
