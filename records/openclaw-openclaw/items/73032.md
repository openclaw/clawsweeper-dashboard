Full review comments:

- **[P2] Handle assistant audio transcript events:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:397-398`
  - body: xAI voice output can include `response.output_audio_transcript.delta` and `.done`, but this switch only handles text-mode transcript events. In an audio session, clients can receive speech audio without assistant transcript events, so route the audio transcript events through the assistant transcript path.
  - confidence: 0.9
- **[P2] Wait for session.updated before setup ready:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:276-283`
  - body: `session.created` can arrive before the adapter's `session.update` has been acknowledged. Resolving setup here can make the gateway emit `session.ready` and flush queued media before the configured voice, instructions, audio settings, and tools are applied; complete setup on `session.updated` instead.
  - confidence: 0.87
- **[P2] Defer response.create until all tool outputs are sent:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:165-168`
  - body: The adapter tracks pending tool calls, but still sends `response.create` after each individual `tool.result`. If multiple function calls are outstanding, the first completed tool can resume generation before the remaining `function_call_output` items are sent.
  - confidence: 0.84

  - body: This PR adds a user-facing realtime provider and docs, and latest head also adds iOS Share extension audio attachment behavior, but it does not change `CHANGELOG.md`. Repo policy requires a single-line active-version changelog entry before landing.
