## Summary

Keep open. Current main still only wires Gemini for `/voiceclaw/realtime`, while this draft xAI implementation has concrete adapter semantics and changelog blockers plus an open design proposal (#73019) that needs maintainer direction before merge.

## What This Changes

The branch adds an xAI Voice Agent adapter for `/voiceclaw/realtime`, provider dispatch/types, tests, and gateway/xAI provider docs.

## Best Possible Solution

Land a revised xAI `/voiceclaw/realtime` implementation after maintainer design agreement that matches xAI setup, audio transcript, and parallel tool semantics, preserves Gemini behavior, and records the user-facing feature in the changelog.

## Reproduction Assessment

Yes. Mocked adapter tests can reproduce the review findings by injecting `response.output_audio_transcript.delta`, sending `session.created` before `session.updated`, and simulating multiple pending `response.function_call_arguments.done` events before tool results.

## Solution Assessment

No. Provider-aware dispatch is plausible, but the current patch should not merge until the xAI event mapping, setup acknowledgement, parallel tool-call resume behavior, and changelog gap are fixed or maintainers choose a shared adapter design.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Handle assistant audio transcript events:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:403-404`
  - body: xAI emits assistant voice captions as `response.output_audio_transcript.delta` and `.done`, but this switch only handles text-mode `response.text.delta` / `response.output_text.delta`. In normal audio sessions the client can receive speech audio without transcript events, so route the audio transcript events through the assistant transcript path.
  - confidence: 0.91
- **[P2] Wait for session.updated before marking setup ready:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:283-285`
  - body: `session.created` is sent automatically on WebSocket connection, while `session.updated` acknowledges the adapter's `session.update`. Resolving setup on `session.created` can make the gateway emit `session.ready` and flush queued work before the configured voice, audio settings, instructions, and tools are applied.
  - confidence: 0.88
- **[P2] Defer response.create until all tool outputs are sent:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:154-176`
  - body: xAI documents that parallel function calls may arrive together and clients must send every `function_call_output` before a single `response.create`. This sends `response.create` after each individual result, so the first completed tool can resume generation without the remaining tool outputs.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `docs/gateway/index.md:191-204`
  - body: This adds a user-facing `/voiceclaw/realtime` provider and docs, but the PR diff does not include `CHANGELOG.md`. Repo policy requires a single-line Unreleased changelog entry before landing.
  - confidence: 0.96
