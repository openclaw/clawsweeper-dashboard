## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Wait for session.updated before marking setup ready:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:281-283`
  - body: `session.created` is sent automatically when the WebSocket opens, before the adapter's `session.update` has been acknowledged. Resolving setup here can emit `session.ready` and flush queued media before voice, instructions, audio format, and tools are actually applied; complete setup on `session.updated` instead.
  - confidence: 0.9
- **[P2] Handle assistant audio transcript events:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:401-402`
  - body: xAI voice sessions emit spoken assistant captions as `response.output_audio_transcript.delta` and `.done`, but this switch only handles text-mode deltas. Clients can therefore receive assistant audio without the matching transcript; route the audio transcript events through the assistant transcript path.
  - confidence: 0.91
- **[P2] Send response.create only after all tool outputs:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:169-173`
  - body: The adapter tracks pending tool calls but still sends `response.create` after each individual `tool.result`. With parallel function calls, the first completed tool can resume generation before the remaining `function_call_output` items are sent, which xAI documents as incomplete context; wait until the pending set is drained.
  - confidence: 0.88
