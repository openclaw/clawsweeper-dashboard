## Summary

Keep open. Current main still only wires the `/voiceclaw/realtime` surface to Gemini, while this draft PR has concrete xAI protocol gaps and unresolved maintainer design questions tied to the open proposal it tracks.

## What This Changes

The PR adds an xAI `/voiceclaw/realtime` provider by extending provider selection/types, adding an xAI Realtime adapter with tests, and updating gateway and xAI provider docs.

## Best Possible Solution

Land a revised implementation after maintainer design agreement that keeps the Gemini path stable, maps xAI realtime voice events correctly, preserves the server-side key boundary, handles parallel tool-call resume semantics, and closes the linked proposal only after merge.

## Reproduction Assessment

Yes for the review findings. Focused mocked adapter tests can inject `response.output_audio_transcript.delta`, send `session.created` before `session.updated`, and simulate multiple pending tool calls to observe the missing captions, early readiness, and premature resume.

## Solution Assessment

No, not yet. Provider-aware dispatch is a plausible direction, but the current PR should not merge until the documented xAI event mapping and ordering gaps are fixed or maintainers choose a different shared adapter design.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Map audio transcript events:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:397-398`
  - body: xAI emits assistant voice captions as `response.output_audio_transcript.delta` and `.done`, but this switch only maps text-mode `response.text.delta` / `response.output_text.delta`. A normal voice session can forward audio while never emitting assistant transcript events, so route the audio transcript events through the assistant transcript path.
  - confidence: 0.91
- **[P2] Wait for session.updated before setup completion:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:277-282`
  - body: xAI sends `session.created` automatically when the socket opens and `session.updated` after applying this adapter's `session.update`. Resolving setup on `session.created` can make the gateway emit `session.ready` and flush queued work before the configured voice, instructions, audio settings, and tools are acknowledged; treat `session.created` as informational and complete setup on `session.updated`.
  - confidence: 0.87
- **[P2] Defer response.create until all tool results are sent:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:165-169`
  - body: xAI documents that parallel tool calls may arrive together and that clients must send every `function_call_output` before a single `response.create`. This method sends `response.create` after each individual tool result, so the first completed tool can resume generation without the remaining tool outputs.
  - confidence: 0.82
