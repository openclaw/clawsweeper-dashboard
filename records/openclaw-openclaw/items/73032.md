## Summary

Keep open, but do not merge as-is: current main still lacks xAI support for `/voiceclaw/realtime`, and the draft PR has protocol-semantics blockers against xAI's documented setup, transcript, and parallel tool-call behavior.

## What This Changes

The PR adds an xAI `/voiceclaw/realtime` provider adapter, provider selection helpers/types, tests, and gateway/xAI provider docs.

## Best Possible Solution

Land a revised implementation after maintainer design agreement that preserves Gemini behavior, maps xAI Voice Agent events faithfully, waits for the session update acknowledgement, handles multiple tool calls correctly, and records the feature in the changelog.

## Reproduction Assessment

Yes. A mocked adapter test can inject `response.output_audio_transcript.delta`, send `session.created` before `session.updated`, and simulate multiple `response.function_call_arguments.done` events to observe missing captions, early readiness, and premature `response.create`.

## Solution Assessment

No. Provider-aware dispatch is a plausible direction, but this patch should not merge until the xAI event mapping, setup acknowledgement, parallel tool-call resume behavior, changelog gap, and maintainer design questions are addressed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Handle assistant audio transcript events:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:397-398`
  - body: xAI emits assistant voice captions as `response.output_audio_transcript.delta` and `.done`, but this switch only handles text-mode deltas. In audio sessions, clients can receive speech audio without matching assistant transcript events, so route the audio transcript events through the assistant transcript path.
  - confidence: 0.91
- **[P2] Wait for session.updated before setup ready:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:277-279`
  - body: `session.created` is sent when the socket opens, before this adapter's `session.update` is acknowledged. Resolving setup here can make the gateway emit `session.ready` and flush queued work before the configured voice, instructions, audio settings, and tools are applied; complete setup on `session.updated` instead.
  - confidence: 0.88
- **[P2] Defer response.create until all tool outputs are sent:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:165-169`
  - body: xAI documents that parallel function calls may arrive together and clients must send every `function_call_output` before one `response.create`. This sends `response.create` after each individual result, so the first completed tool can resume generation without the remaining tool outputs.
  - confidence: 0.83
- **[P3] Add the required changelog entry:** `docs/gateway/index.md:194-204`
  - body: This is a user-facing feature and docs change, but the diff does not add an Unreleased `CHANGELOG.md` entry. Repo policy requires a single-line changelog entry before landing.
  - confidence: 0.96
