## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Handle assistant audio transcript events:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:397-398`
  - body: xAI is documented as OpenAI-Realtime-compatible, and OpenAI Realtime audio responses can emit assistant captions as `response.output_audio_transcript.delta` and `.done`. This switch only routes text-mode `response.text.delta` / `response.output_text.delta`, so an audio session can deliver speech without any assistant transcript events.
  - confidence: 0.9
- **[P2] Wait for session.updated before setup ready:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:277-279`
  - body: `session.created` can arrive before the adapter's `session.update` has been acknowledged. Resolving setup here lets the gateway emit `session.ready` and flush queued media before the configured voice, instructions, audio settings, and tools are applied; complete setup on `session.updated` instead.
  - confidence: 0.87
- **[P2] Defer response.create until all tool outputs are sent:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:165-169`
  - body: The adapter tracks pending tool calls, but still sends `response.create` immediately after each individual `tool.result`. If multiple function calls are outstanding, the first completed tool can resume generation before the remaining `function_call_output` items are sent.
  - confidence: 0.9
- **[P3] Add the required changelog entry:** `docs/gateway/index.md:194-204`
  - body: This PR adds a user-facing realtime provider and user-facing docs, and the latest head also adds iOS Share extension audio attachment behavior, but the patch does not touch `CHANGELOG.md`. Repo policy requires a single-line active-version changelog entry before landing.
  - confidence: 0.96
