## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Send xAI's documented audio format object:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:325-326`
  - body: The adapter sends flat `input_audio_format` and `output_audio_format` fields, but xAI documents audio configuration under `session.audio.input.format` and `session.audio.output.format` with `audio/pcm` plus a sample rate. These fields can be ignored or rejected, leaving the session format uncontrolled. ([docs.x.ai](https://docs.x.ai/developers/model-capabilities/audio/voice-agent?campaign=custom-voices-blog))
  - confidence: 0.9
- **[P2] Wait for session.updated before setup completes:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:280-287`
  - body: `session.created` is sent automatically on WebSocket connection, while `session.updated` acknowledges the client's `session.update`. Resolving setup on `session.created` can flush queued media before voice, instructions, tools, and audio settings are applied. ([docs.x.ai](https://docs.x.ai/developers/rest-api-reference/inference/voice))
  - confidence: 0.9
- **[P2] Forward assistant audio transcript events:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:399-402`
  - body: The adapter handles text-mode `response.text.delta` but not `response.output_audio_transcript.delta`/`done`, which xAI sends for spoken assistant captions. Spoken assistant transcripts can be dropped even while audio is forwarded. ([docs.x.ai](https://docs.x.ai/developers/rest-api-reference/inference/voice))
  - confidence: 0.88
- **[P2] Resume only after pending tool outputs drain:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:169-173`
  - body: `sendToolResult` sends `response.create` after each individual tool result even though the adapter tracks pending tool calls. With parallel calls, the first result can resume generation before the remaining `function_call_output` items are attached, so the model continues with incomplete context.
  - confidence: 0.88
- **[P3] Count image attachments separately from audio:** `apps/ios/ShareExtension/ShareViewController.swift:400-401`
  - body: The image branch still checks total `attachments.count` against the image cap. If an audio item is appended first, a share with one audio plus three images only keeps two images; count image attachments independently.
  - confidence: 0.82
