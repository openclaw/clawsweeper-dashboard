## Summary

Keep open. Current main does not implement xAI on `/voiceclaw/realtime`, and this active draft PR still has blocking adapter semantics plus a missing changelog entry before it can merge.

## What This Changes

The PR adds an xAI `/voiceclaw/realtime` adapter, provider dispatch/types, tests, and gateway/xAI provider docs.

## Best Possible Solution

Land a revised implementation after maintainer design agreement that preserves the Gemini path, matches xAI's documented event/setup/tool-call semantics, keeps the server-side key boundary, and records the user-facing feature in the changelog.

## Reproduction Assessment

Yes. Mocked adapter tests can inject `response.output_audio_transcript.delta`, send `session.created` before `session.updated`, and simulate parallel `response.function_call_arguments.done` events to observe missing captions, early readiness, and premature `response.create`.

## Solution Assessment

No. Provider-aware dispatch is a plausible direction, but the current patch should not merge until the xAI event mapping, setup acknowledgement, parallel tool-call resume behavior, and changelog gap are fixed or maintainers choose a shared adapter design.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Handle assistant audio transcript events:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:397-398`
  - body: xAI emits assistant voice captions as `response.output_audio_transcript.delta` and `.done`, but this switch only handles text-mode `response.text.delta` / `response.output_text.delta`. In normal audio sessions the client can receive speech audio without transcript events, so route the audio transcript events through the assistant transcript path.
  - confidence: 0.91
- **[P2] Wait for session.updated before marking setup ready:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:277-279`
  - body: `session.created` is sent when the socket opens, before the adapter's `session.update` has been acknowledged. Resolving setup here can make the gateway emit `session.ready` and flush queued work before the configured voice, instructions, audio settings, and tools are applied; complete setup on `session.updated` instead.
  - confidence: 0.87
- **[P2] Defer response.create until all tool outputs are sent:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:165-169`
  - body: xAI documents that parallel function calls may arrive together and clients must send every `function_call_output` before a single `response.create`. This sends `response.create` after each individual result, so the first completed tool can resume generation without the remaining tool outputs.
  - confidence: 0.82
- **[P3] Add the required changelog entry:** `docs/gateway/index.md:194-204`
  - body: This is a user-facing feature and docs change, but the diff does not touch `CHANGELOG.md`. Repo policy requires a single-line Unreleased changelog entry before landing.
  - confidence: 0.96
