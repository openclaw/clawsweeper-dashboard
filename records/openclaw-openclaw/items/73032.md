## Summary

Keep open. Current main is still Gemini-only for `/voiceclaw/realtime`, and this draft PR is useful but not merge-ready because the xAI adapter misses documented voice-transcript events, marks setup ready before `session.updated`, resumes too early for parallel tool calls, and lacks the required user-facing changelog entry.

## What This Changes

The PR adds an xAI Voice Agent adapter for `/voiceclaw/realtime`, provider dispatch and env-key defaults, mocked adapter/session tests, and gateway/xAI provider docs.

## Best Possible Solution

Land a revised implementation after maintainer design agreement that keeps Gemini stable, maps xAI's documented realtime events correctly, gates readiness on the applied session configuration, handles parallel tool-call resume semantics, preserves the server-side key boundary, and includes focused tests plus the required changelog entry.

## Reproduction Assessment

Yes for the review findings: a mocked PR-head adapter test can inject `response.output_audio_transcript.delta`, `session.created` before `session.updated`, and multiple tool calls to observe the missing transcript, early readiness, and premature `response.create` behavior. Current-main absence is also directly reproducible by inspecting the provider union, forced Gemini session config, and xAI docs.

## Solution Assessment

No, not yet. Provider-aware `/voiceclaw/realtime` dispatch is plausible, but the patch should not merge until the documented protocol gaps and changelog policy gap are fixed or maintainers choose a shared adapter direction first.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Map audio transcript events:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:397-398`
  - body: xAI's realtime reference emits assistant voice captions as `response.output_audio_transcript.delta` and `.done`, but this switch only handles text-mode `response.text.delta` / `response.output_text.delta`. Normal voice sessions will forward audio without assistant captions or transcript history, so route the audio transcript events through the assistant transcript path.
  - confidence: 0.91
- **[P2] Wait for session.updated before marking setup ready:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:277-279`
  - body: xAI sends `session.created` on socket open and `session.updated` only after applying this adapter's `session.update`. Resolving setup on `session.created` can emit `session.ready` and flush queued audio/control before voice, tools, and audio settings are acknowledged, so complete setup on `session.updated`.
  - confidence: 0.87
- **[P2] Defer response.create until all tool results are sent:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:165-169`
  - body: xAI documents that parallel function calls may arrive together and clients must send every `function_call_output` before a single `response.create`. This method sends `response.create` after each tool result, allowing generation to resume before remaining outputs arrive.
  - confidence: 0.82
- **[P3] Add the required changelog entry:** `docs/gateway/index.md:200-203`
  - body: This user-facing `feat` adds a new xAI `/voiceclaw/realtime` provider, but the diff does not include `CHANGELOG.md`. Repo policy requires user-facing feat/fix/perf changes to add a single-line changelog entry with appropriate attribution.
  - confidence: 0.93
