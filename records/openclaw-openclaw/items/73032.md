## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Handle assistant audio transcript events:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:401-402`
  - body: xAI audio sessions can emit assistant captions as `response.output_audio_transcript.delta` and `.done`, but this switch only routes text-mode `response.text.delta` / `response.output_text.delta`. A client can receive spoken audio without assistant transcript events, so route the audio transcript events through the same assistant transcript path. ([]())
  - confidence: 0.91
- **[P2] Wait for session.updated before setup is ready:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:282`
  - body: `session.created` is sent automatically when the WebSocket opens, before the adapter's `session.update` is acknowledged. Resolving setup here can make the gateway emit `session.ready` and flush queued media before voice, instructions, audio settings, and tools are applied; complete setup on `session.updated` instead. ([]())
  - confidence: 0.88
- **[P2] Defer response.create until tool outputs are complete:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:169-173`
  - body: The adapter tracks pending tool calls but still sends `response.create` after each individual `tool.result`. With multiple outstanding function calls, the first completed tool can resume generation before the remaining `function_call_output` items are sent, so only request the next response once the pending set is drained.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `docs/gateway/index.md:200-204`
  - body: This PR adds a user-facing realtime provider and docs, and the current head also adds iOS Share audio attachment behavior, but it does not update `CHANGELOG.md`. Repo policy requires a single-line active-version changelog entry before landing.
  - confidence: 0.96
