## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Send the documented xAI audio session shape:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:325-326`
  - body: The xAI adapter sends flat `input_audio_format`, `output_audio_format`, and top-level `voice` fields in `session.update`, but current xAI Voice Agent sessions use nested audio input/output configuration. That can leave format, voice, and VAD settings unapplied and break clients expecting the selected audio format.
  - confidence: 0.9
- **[P2] Wait for session.updated before marking setup ready:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:281-283`
  - body: `session.created` can arrive before the adapter's `session.update` has been acknowledged. Resolving setup here can emit `session.ready` and flush queued media before voice, instructions, audio format, and tools are actually applied; complete setup on `session.updated` instead.
  - confidence: 0.9
- **[P2] Handle assistant audio transcript events:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:399-402`
  - body: The adapter forwards assistant audio but only turns text-mode deltas into assistant transcript events. Spoken assistant captions can therefore be lost; route audio transcript delta/done events through the assistant transcript path too.
  - confidence: 0.88
- **[P2] Resume only after all tool outputs are attached:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:169-173`
  - body: The adapter tracks pending tool calls but still sends `response.create` after each individual `tool.result`. With parallel function calls, the first completed tool can resume generation before the remaining `function_call_output` items are sent, so the model continues with incomplete context.
  - confidence: 0.88
- **[P3] Keep audio attachments out of the image cap:** `apps/ios/ShareExtension/ShareViewController.swift:400-401`
  - body: After adding audio attachments, the image branch still checks `attachments.count < maxImageAttachments`. If an audio item is processed first, it consumes one of the three image slots and a share with one audio plus three images only keeps two images; count image attachments separately.
  - confidence: 0.82
