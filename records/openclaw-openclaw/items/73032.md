## Summary

Keep open. Current main still has a Gemini-only `/voiceclaw/realtime` implementation, and this active draft is paired with the still-open xAI realtime proposal, but the patch is not mergeable as-is because it misses documented xAI audio-transcript, setup-acknowledgement, parallel tool-call, and changelog requirements.

## What This Changes

The PR adds an xAI `provider: "xai"` path for `/voiceclaw/realtime`, a new xAI realtime adapter and tests, provider dispatch helpers, and gateway/xAI provider docs.

## Best Possible Solution

Land a revised implementation after design agreement that preserves the Gemini path, matches xAI's documented realtime event and tool-call semantics, keeps the upstream key server-side, and records the user-facing feature in the changelog.

## Reproduction Assessment

Yes. Mocked adapter tests can inject `response.output_audio_transcript.delta`, send `session.created` before `session.updated`, and simulate parallel `response.function_call_arguments.done` events to observe missing captions, early readiness, and premature `response.create`.

## Solution Assessment

No. Provider-aware dispatch is a plausible direction, but this patch should not merge until the xAI event mapping, setup acknowledgement, parallel tool-call resume behavior, changelog entry, and design-scope question are resolved.
