- **[P2] Handle assistant audio transcript events:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:397-398`
  - body: Audio realtime sessions can emit assistant captions through audio-transcript events, but this switch only handles text output events. A client can receive speech audio without any assistant transcript, so route audio transcript delta/done events through the assistant transcript path.
  - confidence: 0.9
- **[P2] Wait for session.updated before setup ready:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:276-279`
  - body: `session.created` can arrive before the adapter's `session.update` is acknowledged. Resolving setup here can make the gateway emit `session.ready` and flush queued media before voice, instructions, audio settings, and tools are applied.
  - confidence: 0.87
- **[P2] Defer response.create until all tool outputs are sent:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:165-168`
  - body: The adapter tracks pending tool calls but still sends `response.create` after each individual `tool.result`. If multiple function calls are outstanding, the first completed tool can resume generation before the remaining outputs are sent.
  - confidence: 0.84

  - body: This PR adds a user-facing realtime provider and user-facing docs, and the latest head also adds iOS Share extension audio attachment behavior, but the patch does not touch `CHANGELOG.md`. Repo policy requires a single-line active-version changelog entry before landing.
