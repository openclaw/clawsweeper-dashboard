## Summary

Current main still does not expose Telegram `streamThrottleMs` or `minInitialChars` through the Telegram config/schema/docs metadata path, and the dispatcher still omits `throttleMs` when creating the draft stream. This should stay open because PR #38255 is the active open implementation path and has not merged.

## What This Changes

Expose Telegram draft preview cadence settings, especially `streamThrottleMs` and possibly `minInitialChars`, through Telegram config and wire them into draft streaming for smoother previews.

## Best Possible Solution

Review and land PR #38255, or an equivalent maintainer-approved change, that adds strict top-level and per-account Telegram preview cadence config, updates generated config metadata and public Telegram docs, passes resolved values into `createTelegramDraftStream`, and covers defaults, invalid values, account overrides, fallback, and final flush behavior with focused tests.
