## Summary

Current main still lacks the requested Telegram configuration surface: the draft stream accepts internal `throttleMs` and `minInitialChars` parameters, but dispatch only passes the fixed initial-character threshold and the strict Telegram config schema does not accept either requested key. The prior target-repo implementation PRs are closed unmerged, so this remains valid work rather than a closable duplicate.

## What This Changes

Expose configurable Telegram draft preview cadence, especially `streamThrottleMs` and possibly `minInitialChars`, so native DM draft streaming can update more smoothly than the current fixed cadence.

## Best Possible Solution

Add a documented, validated Telegram account config surface for draft preview cadence, merge it through the existing Telegram account resolution path, and pass the resolved values into `createTelegramDraftStream` while preserving current defaults unless maintainers choose otherwise.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: current dispatch creates Telegram draft lanes without a `throttleMs`, so the draft stream uses the 1000 ms default, and the strict config schema rejects the proposed keys.

## Solution Assessment

Yes for the configurable-key direction, with one caveat: exposing optional validated top-level and per-account settings is the narrow maintainable fix, while changing the default cadence should be treated as a separate maintainer decision.
