## Best Possible Solution

Keep the recovered plugin runtime available through a scoped/request-owned handle or a non-narrowing registry merge, with regression coverage proving cached descriptors still work and unrelated registry entries survive.

## Reproduction Assessment

## Solution Assessment

No. Retaining the recovered runtime is the right direction, but this implementation installs a scoped cold-loaded registry as global active state; the safer fix should preserve broader active/channel/http registry state.
