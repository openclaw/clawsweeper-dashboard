## Summary

Keep open: this PR targets a real source-confirmed plugin tool lifecycle regression and supplies live proof, but the current diff installs a scoped recovery registry as global active state, which can evict unrelated active/channel/http registry state.

## What This Changes

The branch exports the standalone registry installer, installs cold-loaded tool-discovery registries for cached plugin tool execution, adds a lifecycle regression test, and updates the changelog.

## Best Possible Solution

Retain or reuse the successful scoped recovery registry for cached descriptor execution without replacing broader active/channel/http state, and cover the fix with a regression that unrelated live registry entries survive.

## Reproduction Assessment

Yes, source-reproducible. The linked issue provides current-main failure context, and current main still has the cached descriptor path that rebuilds an `activate:false` tool-discovery registry and can throw when the resolved runtime registry lacks the matching tool.

## Solution Assessment

No. Retaining the scoped recovery registry is the right direction, but installing it as global active state is not the narrowest maintainable solution because it can replace unrelated runtime surfaces.
