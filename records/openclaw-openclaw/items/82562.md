## Summary

## What This Changes

The PR makes cached descriptor-backed plugin tools retain a scoped cold-loaded runtime registry after active registry replacement and adds focused regression coverage.

## Best Possible Solution

Rebase this PR onto current main, preserve the provider-preservation behavior from https://github.com/openclaw/openclaw/pull/93276, and land the descriptor-scoped retention fix only after focused plugin-runtime validation and maintainer acceptance of the lifecycle model.

## Solution Assessment

Yes, conditionally. Descriptor-scoped retention is the narrowest fix I found because it avoids reinstalling a tool-discovery registry globally, but the branch must be rebased and the lifecycle choice needs maintainer acceptance.
