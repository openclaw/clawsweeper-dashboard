## What This Changes

The PR retains a descriptor-scoped cold-loaded plugin runtime registry so cached plugin tool descriptors can continue executing after active registry replacement.

## Best Possible Solution

Rebase this PR onto current main, preserve the merged tool-discovery request-local behavior, and keep descriptor-scoped retained registry only for cached descriptor execution if maintainers accept that lifecycle contract.

## Reproduction Assessment

Yes, at source level: current main still has cached descriptor execution rebuilding a tool-discovery load and throwing `plugin tool runtime missing` when the recovered registry cannot produce the requested tool on later cached descriptor calls. The private lossless-claw loop was not live-run here, but the linked issue includes reporter and third-party confirmations.

## Solution Assessment

Yes, likely, but it needs rebase and maintainer acceptance. Descriptor-scoped retention is narrower than reinstalling a cold-loaded tool registry globally, and current main's merged tool-discovery guard makes that boundary more important.
