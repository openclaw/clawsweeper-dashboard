## Summary

Keep open: the linked regression is concrete and this PR targets the right resolver area, but the patch can replace broader active plugin runtime state with a scoped recovery registry during cached descriptor execution.

## What This Changes

The PR exports the standalone runtime registry installer, retains forced tool-discovery registries for cached plugin descriptors, adds a lifecycle regression test, and updates the changelog.

## Best Possible Solution

Retain or reuse the successful scoped recovery registry for cached descriptor execution without replacing broader active/channel/http registry state, and add coverage proving unrelated live registry entries survive.

## Reproduction Assessment

Yes, source-reproducible. The linked issue gives concrete logs and a current-main confirmation, and current main still has the cached descriptor path that force-loads an uninstalled scoped registry and can later throw `plugin tool runtime missing`.

## Solution Assessment

No. Retaining the recovery registry is the right direction, but installing the scoped registry as global active state is not the safest maintainable fix because it can evict broader runtime state.
