## Summary

Keep open. Current main still lacks a first-class Gateway session envelope, and same-author PR #74799 is already the active implementation candidate, so cleanup closure would race maintainer and security review.

## What This Changes

Add an optional Gateway session envelope for `sessions.create` and `sessions.patch`, with session-owned tool/path/network policy enforcement and an optional external authorization callback.

## Best Possible Solution

Review and land PR #74799, or an accepted narrower core design that stores a typed envelope, enforces it fail-closed before tool execution and after rewrites, emits stable rejection/audit events, and defers unenforceable network limits to sandbox/proxy work.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `sessions.create` or `sessions.patch` with `envelope`; current main rejects it because the closed TypeBox/AJV schemas omit that field.

## Solution Assessment

Unclear. A Gateway-owned session envelope is a plausible core solution, but the exact policy semantics, fail-closed boundary, path/network coverage, and event contract still need maintainer/security review.
