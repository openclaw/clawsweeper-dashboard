## Summary

Keep open. Current main still lacks the Gateway session-owned envelope, and same-author PR #74799 is the active implementation candidate; closing this issue would race design and security review.

## What This Changes

Add a Gateway session-owned runtime envelope on `sessions.create` and `sessions.patch` to constrain tools, paths, network or Bash policy, and deterministic rejection behavior.

## Best Possible Solution

Land one accepted core design that stores a typed session envelope, enforces it fail-closed before execution and after parameter rewrites, emits stable audit/rejection events, and leaves unenforceable network limits to a runtime boundary.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `sessions.create` or `sessions.patch` with `envelope`; the closed TypeBox schemas omit that field and AJV rejects additional properties before storage.

## Solution Assessment

Unclear. A Gateway-owned envelope is a plausible core solution, but the exact policy model should be settled through review of PR #74799 or an accepted replacement design.
