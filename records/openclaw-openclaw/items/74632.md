## What This Changes

Add an optional per-session runtime-policy envelope to `sessions.create` and `sessions.patch` so Gateway clients can declare and update tool, path, command, and network scope for session runs.

## Best Possible Solution

Keep this issue open until maintainers land PR #74799 or choose a replacement design that stores a typed session envelope, enforces it fail-closed before execution and after parameter rewrites, documents deterministic rejection semantics, and tracks network limits as runtime sandbox/proxy work.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to send `envelope` in `sessions.create` or `sessions.patch`; current main targets closed TypeBox/AJV schemas that do not define the field.

## Solution Assessment

Unclear pending maintainer review. A session-owned Gateway envelope is a plausible narrow core solution, but the exact authorization semantics and any network/external-callback scope need owner approval before this is considered solved.
