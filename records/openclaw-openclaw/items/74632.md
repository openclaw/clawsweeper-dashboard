## Summary

Keep open. Current main still lacks a first-class Gateway session envelope, and same-author PR #74799 is already the active implementation candidate, so this should wait for maintainer/security review rather than cleanup closure.

## What This Changes

Add an optional session-owned runtime envelope to `sessions.create` and `sessions.patch`, with pre-execution policy enforcement and an optional external authorization callback.

## Best Possible Solution

Keep this issue open until maintainers review and either land #74799 or replace it with an accepted narrower design that stores a typed session envelope, enforces it fail-closed before execution and after parameter rewrites, documents stable rejection semantics, and defers unenforceable network claims.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to send `envelope` in `sessions.create` or `sessions.patch`; current main rejects it because both TypeBox/AJV schemas omit that field and disallow additional properties.

## Solution Assessment

Unclear. A session-owned envelope is a plausible additive Gateway solution, but the exact authorization boundary, path semantics, network scope, callback behavior, and interaction with trusted policies/hooks need maintainer security review; #74799 is the current candidate to evaluate.
