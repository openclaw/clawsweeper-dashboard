## Summary

Keep open: current main and v2026.6.10 still lack a session-owned `envelope` field on `sessions.create` or `sessions.patch`, and the linked implementation candidate remains open, conflicting, and security/product-sensitive rather than merged.

## What This Changes

Requests a first-class Gateway session runtime envelope on `sessions.create`/`sessions.patch`, plus an optional external authorization callback, so clients can constrain tool, path, command, and network scope per session.

## Best Possible Solution

Keep this issue as the concrete design anchor, then land, rework, or replace https://github.com/openclaw/openclaw/pull/74799 after maintainers approve the authorization contract, active enforcement path, upgrade behavior, and proof requirements.
