## What This Changes

The issue requests an additive Gateway protocol session envelope on `sessions.create` and `sessions.patch` for per-session tool, path, network, and authorization policy.

## Best Possible Solution

Keep the issue open until maintainers either land https://github.com/openclaw/openclaw/pull/74799 or approve a narrower typed session-envelope design with clear enforcement, audit, and network/callback boundaries.

## Reproduction Assessment

Yes, source-level. Passing `envelope` to `sessions.create` or `sessions.patch` on current main hits closed protocol schemas that omit the field and are validated before session storage.

## Solution Assessment

Unclear. A Gateway-owned session envelope is a plausible core solution, but the exact enforcement layer, audit event contract, and optional network/callback scope need maintainer/security review before it is the accepted design.
