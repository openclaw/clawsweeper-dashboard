## Summary

Keep open. Current main still lacks a first-class Gateway session envelope, and open same-author PR #74799 is the active implementation candidate for maintainer and security review.

## What This Changes

Add a first-class, session-owned runtime policy envelope to Gateway `sessions.create` and `sessions.patch` so tool, path, and network limits can be enforced for all runs in that session.

## Best Possible Solution

Keep this open until maintainers either land #74799 or choose a narrower accepted core design that stores a typed session envelope, enforces it fail-closed before execution and after parameter rewrites, documents stable rejection semantics, and defers unenforceable network limits to sandbox/proxy work.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `sessions.create` or `sessions.patch` with `envelope`; current main rejects it because the closed TypeBox/AJV schemas omit that field.

## Solution Assessment

Unclear. A Gateway-owned session envelope is a plausible core solution, but the exact policy semantics and enforcement boundary need maintainer/security review before this is considered the best fix.
