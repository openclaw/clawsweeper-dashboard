## Summary

Keep open. Current main still rejects a session-owned `envelope` on `sessions.create` and `sessions.patch`, and same-author PR #74799 is already the active implementation surface that needs maintainer/security review rather than cleanup closure.

## What This Changes

Add an optional per-session runtime policy envelope to Gateway `sessions.create` and `sessions.patch` so OpenClaw can enforce tool and scope constraints for that session.

## Best Possible Solution

Keep the issue open until maintainers land PR #74799 or choose an accepted narrower design that adds a typed Gateway session envelope, persists it on session rows, and enforces it fail-closed before tool execution and after parameter rewrites.

## Reproduction Assessment

Yes. A source-level reproduction is to send `envelope` in `sessions.create` or `sessions.patch`; the closed TypeBox schemas and AJV settings reject it before the handlers can persist anything.

## Solution Assessment

Unclear. A session-owned Gateway envelope is a plausible core design, but the precise policy semantics and security boundary should be settled through review of PR #74799 or a maintainer-approved replacement.
