## Summary

Keep open. Current main still has no first-class Gateway session envelope, and open same-author PR #74799 is already the active implementation candidate for this security-sensitive tool authorization surface.

## What This Changes

The issue requests an optional per-session runtime policy envelope on `sessions.create` and `sessions.patch` that OpenClaw stores and enforces on tool calls.

## Best Possible Solution

Keep this open until maintainers land #74799 or choose another design that adds a typed session-owned envelope, fail-closed enforcement before execution and after parameter rewrites, stable rejection/audit semantics, and docs/tests for the accepted scope.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to send `envelope` in `sessions.create` or `sessions.patch` against current main; the closed TypeBox/AJV schemas omit that field and reject it before any store mutation.

## Solution Assessment

Unclear. A session-owned envelope is a plausible core Gateway solution for external clients, but the exact policy semantics, enforcement order, and omitted network/callback pieces need maintainer and security review before this is the best fix.
