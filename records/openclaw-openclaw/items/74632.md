## Summary

Keep open. Current main still lacks a first-class Gateway session envelope, and the same-author implementation PR at https://github.com/openclaw/openclaw/pull/74799 is open and conflicting, so closing this issue would preempt maintainer/security review.

## What This Changes

The issue requests an optional session-owned runtime policy envelope on Gateway `sessions.create` and `sessions.patch`, with enforcement and stable rejection semantics for later tool calls.

## Best Possible Solution

Keep this issue open until maintainers either land https://github.com/openclaw/openclaw/pull/74799 or approve a narrower design for typed session envelopes, fail-closed enforcement, stable rejection events, and separately scoped network/callback behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to pass `envelope` to `sessions.create` or `sessions.patch`; current main's closed schemas omit the field and AJV validation rejects unknown properties before storage.

## Solution Assessment

Unclear. A Gateway-owned session envelope is a plausible core solution, but the exact authorization semantics, enforcement layer, and network/callback scope require maintainer/security review; reviewing the paired PR or a narrower replacement is the right next step.
