## Summary

Keep open. Current main still lacks a first-class Gateway session envelope, and same-author PR #74799 is already the active implementation candidate, so cleanup closure would race maintainer/security review.

## What This Changes

Add an optional per-session runtime-policy envelope to `sessions.create` and `sessions.patch`, persist it with the session, and enforce it on tool calls.

## Best Possible Solution

Keep this open until maintainers either land PR #74799 or choose a narrower accepted core design with typed protocol fields, persisted session state, fail-closed pre-execution enforcement, stable rejection events, docs, and tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `sessions.create` or `sessions.patch` with `envelope`; current main rejects it because the closed TypeBox/AJV schemas omit that field.

## Solution Assessment

Unclear. A Gateway session-owned envelope is a plausible core solution, but the exact policy model and enforcement boundary should be settled through maintainer/security review of PR #74799 or a replacement design.
