## Summary

Keep open. Current main still lacks a first-class session-owned runtime envelope in the Gateway session RPC contract, and open PR #74799 is the same-author implementation candidate that should be reviewed rather than bypassed with a cleanup close.

## What This Changes

Request adds an optional per-session runtime envelope to `sessions.create` and `sessions.patch` so Gateway clients can declare and update tool, path, network, and authorization policy that OpenClaw enforces on later tool calls in that session.

## Best Possible Solution

Review #74799 or a replacement design as the canonical implementation path. The durable end state should be an additive typed Gateway protocol field, persisted `SessionEntry` state, pre-execution enforcement that cannot be bypassed by trusted policy, hook, or approval parameter rewrites, focused create/patch/tool-blocking tests, docs, and an explicit follow-up decision for network and out-of-process callback enforcement.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: send `envelope` in `sessions.create` or `sessions.patch`; current main validates against strict AJV schemas that omit that property and format it as an unexpected property.

## Solution Assessment

Unclear. A session-owned envelope is a plausible narrow solution for the Gateway-client gap, but the authorization boundary, enforcement ordering, path semantics, network scope, and external callback behavior need maintainer/security review; #74799 is the active candidate to evaluate.
