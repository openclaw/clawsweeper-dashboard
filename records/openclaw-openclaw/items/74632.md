## Summary

Keep open. Current main still lacks a first-class Gateway session envelope, and the same-author implementation PR at https://github.com/openclaw/openclaw/pull/74799 is still open, so closing this issue would preempt maintainer/security review.

## What This Changes

Add optional Gateway session envelope fields to `sessions.create` and `sessions.patch` so external clients can attach session-owned tool/path/network policy.

## Best Possible Solution

Keep this issue open until maintainers either land https://github.com/openclaw/openclaw/pull/74799 or choose a narrower accepted design for typed session envelopes, fail-closed enforcement, stable rejection events, and deferred network-boundary work.

## Reproduction Assessment

Yes. A high-confidence source-level check is to send `envelope` in `sessions.create` or `sessions.patch`; current main's closed TypeBox/AJV schemas omit the field and validation rejects it before store mutation.

## Solution Assessment

Unclear. A Gateway-owned session envelope is a plausible core solution, but the exact policy semantics and enforcement boundary need maintainer/security review; the paired PR's decision to defer network constraints looks safer than hook-level network matching.
