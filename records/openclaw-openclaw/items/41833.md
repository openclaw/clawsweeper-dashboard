## Summary

Keep open: current main has opt-in inferred follow-up commitments plus cron run status/failure handling, but it still lacks the requested native deferred-task primitive with explicit add/done lifecycle, durable cron/task linkage, completion reconciliation, escalation, and startup surfacing.

## What This Changes

The issue requests a native commitment/deferred-task primitive with explicit scheduling, completion tracking, escalation, and startup surfacing.

## Best Possible Solution

Design a first-class deferred-work layer above cron and task records with explicit add/list/done/cancel operations, durable scheduled-run links, completion reconciliation, overdue escalation, and unresolved-commitment surfacing.

## Reproduction Assessment

Yes for the source-level product gap, not as a bug reproduction. Current main exposes inferred commitment list/dismiss behavior and heartbeat delivery, while the requested explicit add/done/cron-link/completion/escalation workflow is absent.

## Solution Assessment

Unclear. A native layer above cron and task records is plausible, but the API, capture policy, verifier, escalation policy, and startup surfacing need maintainer product design before implementation is safe.
