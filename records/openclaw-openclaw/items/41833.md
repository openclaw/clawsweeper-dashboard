## Summary

Keep this open: current main has useful inferred commitments and a reminder guard note, but it still lacks the requested explicit deferred-work primitive with scheduling, done/completion tracking, cron/task reconciliation, escalation, and startup surfacing.

## What This Changes

The issue requests a native commitment/deferred-task CLI and runtime layer that schedules promised future work, tracks completion, escalates missed work, and exposes status.

## Best Possible Solution

Design a first-class commitment layer above cron and task records with explicit add/list/done/cancel operations, durable links to scheduled runs, completion reconciliation, overdue escalation, and unresolved-commitment surfacing in session/startup context.

## Reproduction Assessment

Yes for the source-level gap, not as a bug reproduction. Current main exposes inferred `openclaw commitments list/dismiss`, heartbeat check-in delivery, and a warning guard, while the requested explicit scheduling and completion-verification path is absent.

## Solution Assessment

Unclear. A native layer over cron/tasks is directionally appropriate, but the exact capture, completion, escalation, and user-visible API semantics need maintainer product design before a safe narrow PR exists.
