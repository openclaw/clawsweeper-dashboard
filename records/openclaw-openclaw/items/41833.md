## Summary

Keep this open: current main has inferred commitments and a reminder guard, but it still lacks the requested native deferred-task primitive with explicit add/done operations, scheduler linkage, completion reconciliation, escalation, and startup surfacing.

## What This Changes

Add a native commitment/deferred-task primitive that can schedule commitments, track completion, escalate missed work, expose status, and let users add/list/mark-done commitments.

## Best Possible Solution

Design a first-class deferred-work layer above cron and task records with explicit lifecycle commands, durable schedule/task links, completion reconciliation, overdue escalation, and unresolved-commitment surfacing.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature gap. Source inspection gives high confidence that current main has inferred follow-up delivery but not the requested explicit scheduling, completion, and escalation workflow.

## Solution Assessment

Unclear. A native layer over cron/tasks is plausible, but the API shape, auto-capture policy, completion verifier, and escalation behavior need maintainer product design before a safe implementation exists.
