## Summary

Keep open: current main now has opt-in inferred commitments and reminder guardrails, but they only cover heartbeat-delivered check-ins and warning notes, not the requested native add/done primitive with completion verification and overdue escalation.

## What This Changes

Request a native commitment/deferred-task primitive that turns conversational promises into tracked due work with completion status and escalation.

## Best Possible Solution

Design one native surface that composes commitments, cron scheduling, and the task ledger with explicit add/done/overdue states and notification semantics.

## Reproduction Assessment

Not applicable as a feature request. Source and docs confirm the closest current surfaces are inferred heartbeat check-ins, reminder warning notes, cron scheduling, and a separate task ledger.

## Solution Assessment

No, current main is only a partial adjacent solution. Closing as implemented would be premature because the requested durable API, manual done state, completion verification, and escalation policy are still undefined and unimplemented.
