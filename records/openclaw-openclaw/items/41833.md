## Summary

Keep open: current main has inferred follow-up commitments plus cron/task completion tracking, but it still lacks the requested native deferred-task primitive with explicit add/done lifecycle, durable cron/task linkage, completion reconciliation, escalation, and startup surfacing.

## What This Changes

The issue requests a native commitment/deferred-task CLI and runtime layer that schedules promised future work, tracks completion, escalates missed work, and exposes unresolved commitments.

## Best Possible Solution

Design a first-class deferred-work layer above cron and task records with explicit add/list/done/cancel operations, durable scheduled-run links, completion reconciliation, overdue escalation, and unresolved-commitment surfacing.

## Reproduction Assessment

Yes for the source-level product gap, not as a bug reproduction. Current main exposes inferred commitment list/dismiss behavior, heartbeat delivery, cron jobs, tasks, and Task Flow, while the requested explicit add/done/cron-link/completion/escalation workflow is absent.

## Solution Assessment

Unclear. A native layer above cron and task records is plausible, but the API, automatic capture policy, verifier, escalation policy, and startup surfacing need maintainer product design before implementation is safe.
