## Summary

Keep open. Current main has opt-in inferred commitments that partially address orphaned conversational follow-ups, but the requested native deferred-task primitive with explicit add/done, cron completion linkage, completion tracking, and missed escalation is not implemented as a defined product/API contract.

## What This Changes

The issue requests a native commitment/deferred-task primitive that records conversational promises, schedules due work, tracks completion, escalates missed commitments, and surfaces overdue items in future sessions.

## Best Possible Solution

Decide the product boundary first; if explicit deferred-task commitments are wanted, extend the existing commitments, heartbeat, cron, and task-ledger surfaces instead of creating a parallel scheduler.

Not applicable. This is a feature request, and current source/docs confirm related behavior but no broken existing contract to reproduce.

## Solution Assessment
