## Summary

Keep this open: current main has shipped inferred commitments and a reminder guard, but it still does not provide the requested explicit deferred-work primitive with add/done operations, cron or task linkage, completion verification, escalation, and startup surfacing.

## What This Changes

The issue requests a native commitment/deferred-task primitive with scheduling, completion tracking, escalation, and user-visible status.

## Best Possible Solution

Design a first-class deferred-work layer above cron and task records with explicit add/list/done/cancel operations, durable scheduled-run links, completion reconciliation, overdue escalation, and unresolved-commitment surfacing.

## Reproduction Assessment

Yes for the source-level product gap, not as a bug reproduction. Current main exposes inferred `openclaw commitments` list/dismiss behavior and heartbeat delivery, while the requested explicit scheduling, completion, and escalation workflow is absent.

## Solution Assessment

Unclear. A native layer over cron/tasks is directionally plausible, but the exact API, capture policy, completion verifier, and escalation behavior need maintainer product design before a safe implementation exists.
