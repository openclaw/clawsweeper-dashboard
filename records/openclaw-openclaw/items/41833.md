## Summary

Keep open: current main has opt-in inferred follow-up commitments and heartbeat delivery, but it still lacks the requested first-class deferred-task primitive with explicit add/done lifecycle, durable cron/task linkage, completion verification, escalation, and startup surfacing.

## What This Changes

The issue requests a native commitment/deferred-task command surface that schedules promised work, tracks completion, escalates misses, and shows overdue commitments.

## Best Possible Solution

Design a first-class deferred-work layer above cron and tasks with explicit lifecycle commands, durable scheduled-run links, completion reconciliation, overdue escalation, and unresolved-commitment surfacing.

## Reproduction Assessment

Yes for the source-level product gap, not as a bug reproduction. Current main exposes inferred `openclaw commitments` list/dismiss behavior and heartbeat delivery, while the requested explicit scheduling, completion, and escalation workflow is absent.

## Solution Assessment

Unclear. A native layer over cron/tasks is directionally plausible, but the API, capture policy, completion verifier, and escalation behavior need maintainer product design before implementation is safe.
