## Summary

Keep open: current main has opt-in inferred follow-up commitments, but it still lacks the requested native deferred-work primitive with explicit scheduling, done/completion tracking, cron/task reconciliation, overdue escalation, and startup/session surfacing.

## What This Changes

Request a native commitment layer such as `openclaw commit add/list/done` that schedules deferred commitments, tracks completion, escalates missed work, and surfaces overdue items.

## Best Possible Solution

Design a first-class commitment layer above cron/tasks with explicit operations, durable links to scheduled runs or background tasks, completion reconciliation, overdue or failed-run escalation, and unresolved-commitment surfacing in session/startup context.

## Reproduction Assessment

Yes for the source-level feature gap, not as a runtime bug reproduction. Current main exposes inferred `openclaw commitments list/dismiss` and heartbeat delivery, while the requested explicit add/done scheduling and completion-verification path is absent.

## Solution Assessment

Unclear. A native layer over cron/tasks is directionally plausible, but capture rules, completion semantics, escalation policy, CLI shape, and channel behavior need maintainer product design before a safe narrow PR exists.
