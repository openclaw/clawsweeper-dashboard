## Summary

Keep this open: current main has opt-in inferred follow-up commitments, but it does not implement the requested native deferred-work primitive with explicit scheduling, done/completion tracking, reconciliation against cron/task runs, overdue escalation, and startup/session surfacing.

## What This Changes

The issue requests a first-class commitment API for deferred agent promises with add/list/done operations, scheduling, completion verification, status, and escalation.

## Best Possible Solution

Design a first-class commitment layer above cron and task records, with explicit operations, durable links to scheduled runs or background tasks, completion reconciliation, overdue or failed-run escalation, and unresolved-commitment surfacing in session/startup context.

## Reproduction Assessment

Yes for the source-level gap, not as a bug reproduction: current main exposes inferred `openclaw commitments list/dismiss` and heartbeat check-in delivery, while the requested explicit `add/done` scheduling and completion-verification path is absent.

## Solution Assessment

Unclear. A native layer over cron/tasks is directionally appropriate, but the capture rules, escalation policy, completion semantics, and user-visible API need maintainer product design before a safe narrow PR exists.
