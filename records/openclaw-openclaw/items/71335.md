## Best Possible Solution

Choose the supported gateway memory freshness policy, then either close this as intentionally default-on with warnings or implement a gateway-only default-off path with docs and fresh-install plus upgrade tests.

## Reproduction Assessment

## Solution Assessment

Unclear. Default-off in gateway mode is a plausible reliability improvement, but the merged warning PR and maintainer comment show it is a default-policy tradeoff against live memory freshness for smaller gateway users.
