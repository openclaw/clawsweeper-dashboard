## Summary

Keep this open. Current main still resolves an unset memory sync watcher to true, and normal gateway memory_search use still opens the full long-lived memory manager path where builtin and QMD backends create chokidar watchers unless resolved watch is false.

## What This Changes

Request defaulting `agents.defaults.memorySearch.sync.watch` to false only for headless gateway mode while preserving explicit config overrides and CLI/desktop watch behavior.

## Best Possible Solution

Add a runtime-aware memory sync default policy so unset watch resolves to false only for the intended headless gateway manager context, while explicit true/false config remains authoritative and current interactive freshness paths stay covered by focused tests and docs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: leave sync.watch unset, invoke the normal gateway `memory_search` tool, and the full builtin or QMD manager path can start chokidar watchers because the resolver returns watch true.

## Solution Assessment

Unclear as originally phrased. The product direction is reasonable, but the maintainable fix should pass an explicit gateway/headless default-policy context into memory sync resolution or manager creation instead of keying only on raw `gateway.mode`.
