## Summary

Keep open. Current main and the latest release still resolve an unset memory watcher to enabled, and the merged warning mitigation plus maintainer comments explicitly preserve default-on behavior while leaving the gateway default-off policy decision unresolved.

## What This Changes

The issue requests making `memorySearch.sync.watch` default to `false` in gateway mode while preserving explicit user overrides and CLI/desktop default behavior.

## Best Possible Solution

Maintainers should decide the gateway memory freshness policy, then either implement a documented gateway-only default-off path with fresh-install and upgrade coverage, or close this as intentionally default-on in favor of warnings, lower-FD watcher backends, and explicit opt-out.

## Reproduction Assessment

Yes, at source level. With unset `sync.watch`, current main resolves the value to `true`, and the memory-core runtime creates watchers whenever `settings.sync.watch` is true; I did not run a live FD-count reproduction in this read-only review.

## Solution Assessment

Unclear. Gateway default-off is a plausible reliability improvement, but the merged mitigation and maintainer discussion show it is a product tradeoff against live memory freshness for smaller gateway deployments.
