## Summary

Keep open. Current main and the latest release still resolve unset memorySearch.sync.watch to true, and the merged watcher-pressure work intentionally kept default-on behavior while adding warnings, so the remaining request is a gateway default-policy decision with compatibility tradeoffs.

## What This Changes

Request defaulting memorySearch sync.watch to false only in gateway mode while preserving explicit user overrides and CLI/desktop default-on behavior.

## Best Possible Solution

Maintainers should choose the gateway memory-freshness policy, then either implement a gateway-only default-off path with upgrade coverage or formally keep default-on with warning/adaptive-watcher mitigations.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.10 both resolve an unset memorySearch.sync.watch to true, and watcher setup runs whenever the resolved value is true.

## Solution Assessment

Unclear. Gateway default-off is a plausible reliability improvement, but the merged warning-first mitigation and maintainer comment show this is a product tradeoff against live memory freshness for smaller gateway deployments.
