## Summary

Keep this open. Current main still resolves an unset memory watcher to enabled, and normal gateway `memory_search` calls still use long-lived full memory managers that can create chokidar watchers; the related open ENOSPC PR mitigates watcher-limit crashes but does not implement the requested gateway default policy.

## What This Changes

The issue requests a gateway-aware default where unset `memorySearch.sync.watch` resolves to false for headless gateway mode while explicit overrides and CLI/desktop defaults keep working.

## Best Possible Solution

Add a runtime-aware memory sync default policy so unset `watch` resolves to false only for intended headless gateway long-lived manager contexts, while explicit true/false and CLI/status/interactive freshness paths remain covered by tests, docs, and changelog.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: leave `memorySearch.sync.watch` unset, trigger gateway `memory_search`, and the resolver returns `watch: true` before the full builtin or QMD manager can create chokidar watchers.

## Solution Assessment

Yes, with one implementation constraint. The maintainable fix is to pass an explicit gateway/headless default-policy context into sync resolution or manager creation, not to key blindly on raw config in a way that changes desktop or one-shot CLI behavior.
