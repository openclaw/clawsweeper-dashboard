## Summary

Keep open. Current main and the latest release still resolve an unset memory watcher to enabled, the gateway memory_search path still opens full memory managers that can arm watchers, and the related ENOSPC and large-extraPaths items do not implement or supersede this gateway-default policy request.

## What This Changes

Request to default memorySearch.sync.watch to false in gateway/headless mode while preserving explicit overrides and desktop/CLI default behavior.

## Best Possible Solution

Define a runtime-aware memory sync policy where unset watch is false only for approved long-lived headless gateway contexts, explicit true/false still wins, and CLI/status/desktop freshness remains covered by tests and docs.

## Reproduction Assessment

Yes for the default-behavior path: source inspection shows unset memorySearch.sync.watch resolves to true and gateway memory_search uses full manager paths that can arm watchers. I did not live-reproduce the exact FD count from the report.

## Solution Assessment

Unclear as an automated fix. Defaulting watch off in gateway mode is plausible, but maintainers should first define the exact gateway/headless contexts and coordinate with the open large-extraPaths watcher issue.
