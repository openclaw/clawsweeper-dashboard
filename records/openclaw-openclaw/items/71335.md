## Summary

Keep open. Current main and v2026.6.10 still resolve unset memorySearch.sync.watch to true, and the merged watcher-pressure work intentionally kept the default enabled while adding runtime warnings. The remaining request is a gateway default-policy decision with compatibility tradeoffs, not an already-implemented fix.

## What This Changes

The issue requests making unset memorySearch.sync.watch default to false when OpenClaw runs in gateway mode while preserving explicit user overrides and CLI/desktop defaults.

## Best Possible Solution

Maintainers should choose the gateway memory-freshness policy, then either implement a gateway-only default-off path with fresh-install and upgrade coverage or formally keep default-on with warning/adaptive-watcher mitigations.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.10 both resolve an unset memorySearch.sync.watch to true, and watcher setup runs whenever the resolved value is true.

## Solution Assessment

Unclear. Gateway default-off is a plausible reliability improvement, but the merged mitigation and maintainer discussion show it is a product tradeoff against live memory freshness for smaller gateway deployments.
