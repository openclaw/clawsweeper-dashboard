## Summary

Keep this issue open. Current main still resolves an unset memory sync watcher to `true`, and normal gateway `memory_search` calls still enter long-lived manager paths that can start chokidar watchers unless config explicitly disables them.

## What This Changes

The issue requests changing the default memory search sync watcher behavior so headless gateway mode resolves an unset `memorySearch.sync.watch` to `false` while preserving explicit overrides and desktop/CLI defaults.

## Best Possible Solution

Add a runtime-aware memory sync default policy so an unset `watch` becomes false only for intended headless gateway long-lived manager contexts, while explicit true/false remains authoritative and CLI/desktop/status paths stay covered by tests and docs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: leave `memorySearch.sync.watch` unset, trigger gateway `memory_search`, and the default manager context resolves `watch: true` before builtin or QMD full-manager code can create chokidar watchers.

## Solution Assessment

Yes, with one constraint. The maintainable fix is to pass an explicit gateway/headless default-policy context into memory sync resolution or manager creation, not to key broadly on raw config in a way that changes interactive or one-shot manager behavior.

## Repair Work Prompt

Fix issue #71335. Current main resolves unset `agents.defaults.memorySearch.sync.watch` to true and gateway `memory_search` enters full memory managers, which can start chokidar watchers for many agents/files. Implement a runtime-aware default policy so unset `sync.watch` resolves to false only for headless gateway long-lived manager contexts. Preserve explicit `sync.watch: true` and `sync.watch: false`; preserve CLI/status/desktop/interactive defaults where watch should remain enabled or transient managers should avoid watchers as today. Prefer passing a typed context/default-policy through the memory sync resolver or manager creation seam over hard-coding bundled plugin ids or broad raw `gateway.mode` checks. Update focused tests for resolver/default behavior, gateway memory tool/manager paths, builtin manager watcher config, QMD manager watcher config, and startup boot sync. Update schema help/docs if the user-visible default changes and add a `CHANGELOG.md` entry crediting `Thanks @chrisabad`. Do not change unrelated memory indexing, QMD search ranking, ENOSPC handling, provider defaults, or explicit user config semantics.
