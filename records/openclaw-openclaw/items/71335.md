## Summary

Keep open. Current main and the latest release still resolve unset `memorySearch.sync.watch` to enabled, while the merged watcher-pressure warning work and member discussion explicitly leave the gateway default-off policy unresolved.

## What This Changes

The issue asks for unset `memorySearch.sync.watch` to default to `false` in gateway mode while preserving explicit user overrides and desktop/CLI default-on behavior.

## Best Possible Solution

Maintainers should choose the gateway memory-freshness policy, then either implement and document a gateway-only default-off path with fresh-install and upgrade coverage or explicitly keep default-on with warning/adaptive-watcher mitigations.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.6.8` both resolve an unset `memorySearch.sync.watch` to `true`, and watcher setup runs whenever that resolved value is true.

## Solution Assessment

Unclear. Gateway default-off is a plausible reliability improvement, but the merged mitigation and member comment show it is a product tradeoff against live memory freshness for smaller gateway deployments.
