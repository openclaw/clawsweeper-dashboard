## Summary

Keep open. Current main still resolves an unset memory watcher to enabled and gateway `memory_search` still enters long-lived manager paths that can arm chokidar watchers; the merged ENOSPC work only mitigates watcher-limit crashes, and the newer large-`extraPaths` report is adjacent rather than a replacement for this gateway default policy.

## What This Changes

Request to make `memorySearch.sync.watch` default to `false` only in gateway/headless mode while preserving explicit overrides and CLI/desktop watch defaults.

## Best Possible Solution

Define an explicit runtime-aware memory sync default policy: unset watch is false only for approved headless gateway long-lived manager contexts, explicit true/false still wins, and CLI/status/desktop freshness remains covered by tests and docs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: leave `memorySearch.sync.watch` unset, trigger gateway `memory_search`, and current main resolves watch to true before entering full builtin or QMD managers that arm chokidar watchers.

## Solution Assessment

Unclear for automation. The requested default is a plausible narrow mitigation, but maintainers should first define the exact gateway/headless contexts and preserve explicit opt-ins plus desktop freshness semantics.
