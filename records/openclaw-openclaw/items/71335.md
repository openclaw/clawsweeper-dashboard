## Summary

Keep open. Current main still defaults unset `memorySearch.sync.watch` to enabled and the gateway `memory_search` path still opens long-lived watcher-capable managers; related merged fixes reduce watcher FD pressure but do not implement the requested gateway-mode default policy.

## What This Changes

Request defaulting `memorySearch.sync.watch` to false in gateway/headless mode while preserving explicit user overrides and CLI/desktop watch behavior.

## Best Possible Solution

Define a runtime-aware memory sync policy where unset watch is false only for approved long-lived headless gateway contexts, explicit true/false still wins, and tests/docs cover builtin, QMD, CLI, desktop, and gateway behavior.

## Reproduction Assessment

Yes for the default-behavior path: source inspection shows unset `memorySearch.sync.watch` still resolves to true and gateway memory search still opens watcher-capable managers. I did not live-reproduce the original FD count after the merged FD-pressure mitigation.

## Solution Assessment

Unclear as an automated fix. Defaulting watch off in gateway/headless mode is plausible, but maintainers should first decide which gateway contexts are headless versus desktop-like and how to preserve explicit freshness expectations.
