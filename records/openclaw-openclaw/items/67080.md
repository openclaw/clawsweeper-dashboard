Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep scoped route loading out of request dispatch:** `src/plugins/runtime/http-route-registry-loader.ts:30-38`
  - body: `createGatewayPluginRequestHandler` can call this loader on a startup-registry miss, so any remote unmatched HTTP request can reach `loadOpenClawPlugins`. Because the call does not pass `activate:false`, the loader may clear and re-activate global plugin state from request dispatch instead of using a gateway-lifecycle-owned route snapshot.
  - confidence: 0.91
- **[P1] Run auth after scoped route candidates are known:** `src/gateway/server/plugins-http.ts:85-90`
  - body: Scoped routes are discovered only after the earlier `plugin-auth` stage decided whether gateway auth was required from the startup registry. A route that exists only in this fallback and declares `auth: "gateway"` can reject even valid credentials because `gatewayAuthSatisfied` and runtime auth context were never populated for that candidate set.
  - confidence: 0.88
- **[P2] Isolate failures between scoped route owners:** `src/plugins/runtime/http-route-registry-loader.ts:30-33`
  - body: The scoped owner set is loaded as one registry with `throwOnLoadError:true`, so one broken `gateway-plugin-http` owner throws before route matching can select a healthy owner. Since the plugin HTTP stage is `continueOnError`, that skips all scoped plugin routes for the request instead of falling through to unaffected handlers.
  - confidence: 0.84
