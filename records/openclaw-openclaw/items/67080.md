## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Avoid activating plugin registries from requests:** `src/plugins/runtime/http-route-registry-loader.ts:30-38`
  - body: This scoped route loader is called from the HTTP request path after a startup-registry miss, but it calls `loadOpenClawPlugins` without `activate:false`. The loader activates registries by default, so a network request can replace process-global plugin runtime state and rerun plugin initialization instead of using a gateway-owned stable route snapshot.
  - confidence: 0.91
- **[P2] Run auth before dispatching scoped gateway routes:** `src/gateway/server/plugins-http.ts:83-90`
  - body: Scoped routes are matched only after `plugin-auth` has already decided whether gateway auth is required from the startup registry. A scoped route with `auth: "gateway"` on a non-protected path will then be blocked without `gatewayAuthSatisfied`, making that route class unreachable even when the caller sends valid credentials.
  - confidence: 0.86
- **[P2] Keep one bad route owner from blocking scoped routes:** `src/plugins/runtime/http-route-registry-loader.ts:32`
  - body: The scoped loader uses `throwOnLoadError: true` for the whole planned owner set. If one route-owner plugin is broken or unconfigured, the exception skips the plugin-http stage for this request, so healthy scoped routes do not get a chance to match.
  - confidence: 0.84
