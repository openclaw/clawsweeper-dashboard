Full review comments:

- **[P1] Keep scoped route loading out of request dispatch:** `src/plugins/runtime/http-route-registry-loader.ts:30-38`
  - body: This loader is called from the plugin HTTP miss path and invokes `loadOpenClawPlugins` without `activate:false`, so an unmatched remote HTTP request can load plugin modules and mutate the active global registry. Move scoped route-owner resolution to gateway startup or explicit reload and dispatch against a prepared non-activating snapshot.
  - confidence: 0.9
- **[P2] Do not let one owner failure block all scoped routes:** `src/plugins/runtime/http-route-registry-loader.ts:32`
  - body: `throwOnLoadError:true` makes one broken `gateway-plugin-http` owner abort the entire scoped registry load before path matching can choose a healthy route owner. Load owners with failure isolation or build the prepared route registry so unrelated scoped routes continue to work.
  - confidence: 0.84
- **[P2] Run auth after scoped route candidates are known:** `src/gateway/server/plugins-http.ts:85-90`
  - body: Scoped candidates are discovered after the earlier `plugin-auth` stage has already decided whether gateway auth is needed from the startup registry. A scoped `auth: "gateway"` route on a non-default-protected path can therefore fail closed even with valid credentials; auth needs to see the prepared scoped candidates before dispatch.
  - confidence: 0.82
