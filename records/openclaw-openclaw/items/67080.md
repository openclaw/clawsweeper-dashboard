Overall confidence: 0.9

Full review comments:

- **[P1] Keep scoped route loads out of request dispatch:** `src/plugins/runtime/http-route-registry-loader.ts:30-38`
  - body: This fallback runs from the HTTP request path and calls `loadOpenClawPlugins` without `activate: false`. The loader default activates the loaded registry, so an unmatched remote request can mutate process-wide plugin state and execute route-owner plugin code outside a controlled startup or reload lifecycle.
  - confidence: 0.92
- **[P1] Do not fail all scoped routes on one owner error:** `src/plugins/runtime/http-route-registry-loader.ts:32`
  - body: `throwOnLoadError: true` makes any failing plugin in the `gateway-plugin-http` owner set throw before route matching can choose a healthy route. Because plugin HTTP is a `continueOnError` gateway stage, that exception skips the whole plugin-route stage for the request.
  - confidence: 0.88
- **[P2] Run auth against scoped route candidates:** `src/gateway/server/plugins-http.ts:85-90`
  - body: The scoped registry is resolved only after the earlier plugin-auth stage has decided whether to authenticate using the startup registry. Scoped-only routes with `auth: "gateway"` outside protected prefixes can be rejected because valid gateway credentials were never evaluated against those candidates.
  - confidence: 0.82
