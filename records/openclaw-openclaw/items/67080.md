## What This Changes

The PR adds manifest `activation.onRoutes` route hints, a scoped gateway plugin HTTP route-owner planner, and a request-path fallback that loads scoped plugin registries after startup-registry misses.

## Best Possible Solution

Land this concept only after scoped route registries are prepared or cached under gateway lifecycle, non-activating, failure-isolated, and used by the same auth decision path as dispatch.

## Reproduction Assessment

Yes. Source inspection of the PR head shows a startup-registry miss reaches `loadOpenClawPlugins` from HTTP dispatch, and current main shows that loader activates global plugin state unless `activate:false` is set; I did not run live gateway traffic.

## Solution Assessment

No. Manifest route-owner hints are plausible, but request-time activating registry loads are the wrong layer for a gateway hot path; the safer fix is lifecycle-owned scoped route snapshots plus shared auth gating.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep scoped route loading out of request dispatch:** `src/plugins/runtime/http-route-registry-loader.ts:30-38`
  - body: `loadScopedGatewayPluginHttpRouteRegistry` is called after an HTTP route miss and then calls `loadOpenClawPlugins` without `activate:false`. Current loader semantics make that an active load, so a remote request can clear/restore plugin runtime state and install a new active registry; move this to gateway startup/reload or use a gateway-owned non-activating cached snapshot before dispatch.
  - confidence: 0.9
- **[P1] Run auth after scoped route candidates are known:** `src/gateway/server/plugins-http.ts:96-99`
  - body: The `plugin-auth` stage decides whether to authorize before the scoped registry exists, and `shouldEnforcePluginGatewayAuth` checks only the startup/active registry. A scoped-only `auth: "gateway"` route on an unprotected path will reach this check with `gatewayAuthSatisfied` false and be rejected even when the request includes valid credentials; auth gating needs the same route set that dispatch uses.
  - confidence: 0.86
- **[P2] Isolate scoped route owner load failures:** `src/plugins/runtime/http-route-registry-loader.ts:32-33`
  - body: Using `throwOnLoadError: true` for the whole `gateway-plugin-http` owner set means one broken route-owner plugin throws before route matching can select a healthy scoped route. Because the plugin HTTP stage is continue-on-error, the request skips all scoped plugin routes instead of serving the healthy one; load/filter owners independently or return the healthy registry while reporting failed owners.
  - confidence: 0.82
