Add cumulative estimated session cost to slash `/status`/`session_status` output using existing transcript-backed cost data.

## Best Possible Solution

Reuse the existing transcript-backed session cost summary for the chat `/status` and `session_status` card, label it as estimated or partial when appropriate, and leave broader customizable statusline work to its separate product discussion.

## Reproduction Assessment

Source-reproducible: current main computes `/status` cost from the latest token/cache counters and never calls the cumulative session-cost summary loader. I did not run a live chat turn because this was a read-only review.

## Solution Assessment

Yes, the best fix is a narrow status-card change that reuses `loadSessionCostSummary` rather than parsing transcripts again or relying on `estimatedCostUsd`, which is only a run snapshot.
