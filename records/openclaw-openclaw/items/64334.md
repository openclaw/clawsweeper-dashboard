## Summary

Keep this issue open. Current main still aggregates transcript `usage.cost.total`, and `/usage cost` can report a session total, but the shared `/status` and `session_status` renderer only displays the token-derived per-response `Cost:` estimate and has no cumulative `Session cost` line.

## What This Changes

Add cumulative session cost to `/status` output by summing transcript `usage.cost.total` values for the current session.

## Best Possible Solution

Thread the existing transcript-derived session cost through the shared status renderer and show it as a distinct cumulative session-cost line without changing transcript persistence or the existing per-response estimate semantics.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that the requested line is absent from current `/status` while the cost data source already exists.

## Solution Assessment

Yes. The narrow maintainable fix is to extend the shared status helper and tests so both `/status` and `session_status` render the existing aggregate, rather than duplicating cost aggregation in individual command handlers.
