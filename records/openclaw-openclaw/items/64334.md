## Summary

Keep open: current main still does not expose the requested transcript-backed cumulative session cost in status output. Slash `/status` formats a cost, but it is estimated from tokens and model pricing; the CLI `openclaw status` sessions table still has no cost field, and all linked implementation PRs are closed unmerged.

## What This Changes

Add cumulative session cost from existing transcript or session cost data to status output.

## Best Possible Solution

Surface recorded session cost through both status display paths, prefer stored or transcript `costUsd` when available, and keep model-price estimation only as a fallback.

## Reproduction Assessment

Yes, source-reproducible: current slash status reads transcript usage but drops `costUsd`, estimates visible cost from tokens/model pricing, and CLI status session rows do not carry or render cost. I did not run a live daemon because this review is read-only.

## Solution Assessment

Yes. The narrow maintainable solution is to reuse existing stored `estimatedCostUsd` and transcript `costUsd` plumbing in the status display surfaces, not add a new usage subsystem or config option.
