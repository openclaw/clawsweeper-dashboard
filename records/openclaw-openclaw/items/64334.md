## Summary

Keep this issue open. Current main still has transcript-backed cost data, but the shared `/status` and `session_status` renderer does not pass or display cumulative session cost; it only renders the existing estimated `Cost:` from token fields, while `/usage cost` remains a separate command.

## What This Changes

Add a cumulative transcript-backed session cost line to the `/status` status card.

## Best Possible Solution

Thread transcript-derived `costUsd` through the shared status data path and render it as a distinct cumulative session-cost line for both chat `/status` and `session_status`, while keeping the existing last-response estimate and `/usage cost` summary intact.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: current main sums transcript `usage.cost.total` into `snapshot.costUsd`, then drops it before the shared status renderer and only displays a token-derived `Cost:` estimate.

## Solution Assessment

Yes. The narrowest maintainable solution is to extend the shared status helper and renderer with focused tests, not duplicate cost aggregation in individual command handlers or change transcript persistence.
