## Summary

Keep this open. Current main still aggregates transcript cost, but the shared status renderer does not carry or render cumulative session cost; no direct duplicate, paired author PR, or closing PR was found.

## What This Changes

Request to add cumulative session cost from transcript `usage.cost.total` to the `/status` output.

## Best Possible Solution

Thread the existing transcript-derived `costUsd` through the shared status renderer and render a distinct cumulative session-cost line without changing transcript aggregation or `/usage cost` semantics.

## Reproduction Assessment

Yes. Source inspection on current main shows transcript `usage.cost.total` is summed into `snapshot.costUsd`, then dropped before status rendering, while `/status` only renders a token-derived `Cost:` estimate.

## Solution Assessment

Yes. The narrow maintainable fix is to extend the shared status helper and tests so both `/status` and `session_status` get the same cumulative session-cost display.
