## Summary

Keep open. Current main and the latest release still only render a last-response estimated cost in `/status`; the cumulative session total exists in the `/usage cost` path but is not wired into the status card.

## What This Changes

Add a cumulative session-cost line to `/status` by summing transcript `usage.cost.total` values for the active session.

## Best Possible Solution

Reuse `loadSessionCostSummary` in the shared async status path, pass a formatted cumulative value into `buildStatusMessage`, and preserve the existing per-response `Cost:` line.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection on current main confirms `/status` lacks a cumulative session-cost line while `/usage cost` can compute the same session total.

## Solution Assessment

Yes. The narrow maintainable solution is to reuse the existing session-cost summary loader rather than adding another transcript parser or changing the current last-response cost semantics.
