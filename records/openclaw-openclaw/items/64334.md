## Summary

Keep open. Current main still lacks a cumulative session-cost line in `/status`; the renderer only shows estimated last-response `Cost:`, while `/usage cost` already has the full-session cost source this feature should reuse.

## Best Possible Solution

Load the full session cost summary in `buildStatusText`, pass a formatted cumulative value into `buildStatusMessage`, and preserve the existing last-response `Cost:` line.

## Solution Assessment

Yes. The narrow maintainable solution is to reuse `loadSessionCostSummary` in the shared async status path and render the cumulative value without changing the existing per-response cost semantics.
