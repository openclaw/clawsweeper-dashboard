## Summary

Keep open. Current main still lacks a cumulative session-cost line in `/status`; it only renders the existing estimated last-response `Cost:` line, while the full-session cost data is available through the separate `/usage cost` path.

## What This Changes

The issue requests adding a cumulative session-cost line to the `/status` output.

## Best Possible Solution

Load the full session cost summary in the shared async status path and pass a formatted cumulative value into the renderer while preserving the existing last-response `Cost:` line.

## Solution Assessment

Yes, the narrow maintainable path is to reuse `loadSessionCostSummary` in `buildStatusText` and render the cumulative value through `buildStatusMessage`. Avoid using the bounded transcript tail snapshot as the cumulative source.
