## Summary

Keep this issue open. Current main still aggregates transcript `usage.cost.total` into `snapshot.costUsd`, but `/status` and `session_status` do not render that cumulative value; the only status cost line remains the token-derived per-response estimate.

## What This Changes

Add a cumulative transcript-backed session cost line to the shared `/status` status card.

## Best Possible Solution

Thread the existing transcript-derived `costUsd` through the shared status renderer and display it as a distinct cumulative session-cost line for both `/status` and `session_status`.

## Reproduction Assessment

Yes. Source inspection on current main shows transcript `usage.cost.total` is aggregated into `snapshot.costUsd`, then dropped before rendering, while status only displays a token-derived `Cost:` estimate.

## Solution Assessment

Yes. The narrow maintainable path is to extend the shared status helper and tests, not duplicate cost aggregation in command handlers or alter transcript persistence.
