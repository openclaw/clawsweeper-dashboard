## Summary

Keep open. Current main still does not render a cumulative session-cost line in `/status`; it only estimates the last response cost, while `/usage cost` already has the full-session summary source this feature should reuse.

## What This Changes

The issue requests adding a cumulative session-cost line to `/status` output by summing transcript usage cost totals.

## Best Possible Solution

Load the full session cost summary in `buildStatusText`, pass a formatted cumulative value into `buildStatusMessage`, and preserve the existing last-response `Cost:` line.

## Solution Assessment

Yes. The narrow maintainable solution is to reuse `loadSessionCostSummary` in the shared async status path and render the cumulative value without changing existing per-response cost semantics.
