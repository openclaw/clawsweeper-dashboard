## Summary

Keep open: current main still does not expose the requested transcript-backed cumulative session cost in status output. Slash `/status` formats a Cost value, but it is estimated from token counts and current model pricing; the CLI `openclaw status` sessions table still has no cost field, and both linked implementation PRs are closed unmerged.

## What This Changes

Request to show cumulative per-session API cost in `/status` by summing recorded transcript `usage.cost.total` values.

## Best Possible Solution

Surface recorded session cost through the existing status summary and slash-status formatter paths, prefer stored or transcript `costUsd` when available, and keep model-price estimation only as a fallback.

## Reproduction Assessment

Yes. Source inspection shows the slash-status cost is estimated from token/model pricing instead of transcript `usage.cost.total`, and the CLI status table still lacks a Cost column; I did not run a live daemon because this review is read-only.

## Solution Assessment

Yes. The narrow maintainable path is to reuse existing stored `estimatedCostUsd` and transcript `costUsd` plumbing in the status display surfaces, not add a new usage subsystem or config option.
