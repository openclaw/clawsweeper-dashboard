## Summary

Keep this issue open. Current main still does not show cumulative session cost in `/status`; it only renders the existing per-response estimated `Cost:` line, while full session-cost summary data already exists through the `/usage cost` path.

## What This Changes

Add a cumulative session-cost line to the `/status` status card using transcript `usage.cost.total` data.

## Best Possible Solution

Load the full session cost summary in the shared async status path and pass a formatted cumulative value into the renderer, preserving the existing last-response cost line.

## Reproduction Assessment

Yes for the current behavior: source inspection shows the requested cumulative line is absent from `/status`, and the status renderer only emits the token-priced `Cost:` estimate. This is a feature request rather than a broken documented contract.

## Solution Assessment

Yes, with one refinement: use the existing full-session cost summary path, not the tail-limited transcript fallback, then render it through the shared status implementation used by both surfaces.
