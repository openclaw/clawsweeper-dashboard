## Summary

Keep open. Current main still lacks a cumulative session-cost line in `/status`; it only renders the existing estimated `Cost:` line from the current status token counters, while the full-session cost source already exists in the `/usage cost` path.

## What This Changes

Add cumulative session cost to `/status` output.

## Best Possible Solution

Load the full session cost summary in the shared async status path, pass a formatted cumulative value into the renderer, and preserve the existing per-response `Cost:` estimate.

## Reproduction Assessment

Yes for source-level current behavior: current main has no `Session cost` status field or renderer line, and the status cost line is only the existing estimated per-status token cost. This is a feature request, not a broken documented contract.

## Solution Assessment

Yes. The narrow maintainable solution is to reuse `loadSessionCostSummary` from the `/usage cost` path in the shared status implementation, rather than summing a tail-limited transcript snapshot or adding a separate cost parser.
