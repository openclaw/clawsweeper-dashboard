## What This Changes

The issue requests adding cumulative session cost to `/status` output by summing transcript `usage.cost.total` values.

## Best Possible Solution

Land a focused PR, likely https://github.com/openclaw/openclaw/pull/94308 if review passes, that reuses recorded transcript/session cost data for `/status` and `openclaw status` without adding config or a new usage subsystem.

## Solution Assessment
