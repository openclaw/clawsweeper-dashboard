## Summary

Keep this issue open. Current main still does not add a cumulative transcript-backed session cost line to `/status` or `session_status`; it only shows the existing estimated current `Cost:` value and keeps transcript/session cost summaries on separate paths.

## What This Changes

The issue requests a cumulative session cost line in `/status`, derived by summing `usage.cost.total` from the session transcript.

## Best Possible Solution

Add the cumulative session-cost display in the shared status renderer so chat `/status` and the `session_status` tool stay consistent, reusing the existing transcript/session cost aggregation and leaving `/usage cost` as the detailed cost summary command.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: current main reads transcript `costUsd` into the transcript snapshot, but the status helper discards it and the final status renderer never emits a cumulative session-cost line.

## Solution Assessment

Yes. The narrowest maintainable direction is to extend the shared status data path and focused tests, rather than duplicating output in the chat command or changing session persistence semantics.
