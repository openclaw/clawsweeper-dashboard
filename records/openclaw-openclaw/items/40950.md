The PR adds a `media.maxBytes` config field and attempts to replace fixed media store, media route, and sandbox staging byte limits with a mutable effective limit.

Ship a current-main repair for https://github.com/openclaw/openclaw/issues/40880 that adds a bounded `media.maxBytes` contract, wires startup, store, sandbox staging, and chat preflight, preserves stricter caller caps, updates config docs/tests, and includes real behavior proof.

  - body: The remote download guard compares streamed bytes against the module-global media limit. If an operator raises `media.maxBytes`, callers that intentionally pass a smaller channel or per-call cap can persist payloads that should remain rejected.
