## Summary

Keep open for maintainer review. Current main still has the fixed 5 MiB media-store/sandbox cap, so the underlying request is real, but this PR head is conflicting and incorrect: it calls an unimported getter, never applies the new setting on gateway startup, and needs a resource-limit precedence decision before merge.

## What This Changes

The branch adds a top-level `media.maxBytes` config field and mutable media-store limit helpers, then switches media store, sandbox staging, and media serving paths toward the configured limit.

## Best Possible Solution

Land a rebased or replacement implementation for https://github.com/openclaw/openclaw/issues/40880 that defines a bounded media staging limit, threads it through store, sandbox staging, gateway preflight, config help/docs, and regression coverage, and preserves stricter caller-specific caps.

## Reproduction Assessment

Yes, by source inspection. Current main still has fixed 5 MiB store, sandbox staging, and chat preflight caps, and current gateway coverage asserts a 6 MiB sandboxed PDF is rejected.

## Solution Assessment

No. A bounded `media.maxBytes` setting is plausible, but this branch is conflicting, lacks startup wiring and real behavior proof, calls an unimported getter, and needs a clear precedence rule for stricter resource caps.

  - body: `stageSandboxMedia` now calls `getMediaMaxBytes()`, but the module still imports only `getMediaDir` and `MEDIA_MAX_BYTES`. Every staging attempt can throw a `ReferenceError` inside the per-file `try` and skip otherwise valid inbound media.

  - confidence: 0.94
- **[P2] Wire media.maxBytes into gateway startup:** `src/config/zod-schema.ts:517-522`
  - body: Adding `media.maxBytes` to the schema does not apply it. PR head has no startup call to `setMediaMaxBytes`, so the new config option keeps using the default 5 MiB limit.
  - confidence: 0.93
- **[P2] Preserve stricter per-call media caps:** `src/media/store.ts:253-254`
  - body: The remote download path compares streamed bytes only against the module-global `mediaMaxBytes` and redirects also drop any local cap. A raised global limit can bypass stricter channel or caller limits unless the effective cap keeps the smaller explicit limit.
  - confidence: 0.82
