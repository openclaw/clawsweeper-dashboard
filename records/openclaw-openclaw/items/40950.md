The PR adds a top-level `media.maxBytes` config field and switches media store, route, and sandbox staging code to a mutable effective media byte limit.

Ship a current-main replacement that exposes one bounded inbound media store/sandbox limit, applies it before every sandbox gate, preserves stricter per-call/channel caps, updates generated config metadata/docs/changelog, and adds focused regression coverage.
