## Label Justifications

- `P2`: This is a bounded runtime correctness bug in embedded agent lifecycle metadata rather than an emergency or release-blocking outage.
- `impact:session-state`: The issue concerns incorrect terminal liveness and replay metadata for agent run/session state.

## Summary

Keep open. Current main still collapses aborted zero-payload embedded runs into `blocked`, and a rejected embedded attempt can leave before a normalized attempt exists to set terminal lifecycle metadata.

## What This Changes

The issue asks for an embedded-runner error boundary plus a distinct `cancelled` liveness state for user-initiated aborts.

## Best Possible Solution

Add `cancelled` at the shared embedded liveness contract/resolver boundary, propagate it through lifecycle consumers, and add a runner-level terminal metadata backstop for uncaught attempt failures.

## Reproduction Assessment

Yes, by source inspection on current main. An aborted, non-timeout, zero-payload run reaches the `blocked` branch, and a rejected `runEmbeddedAttemptWithBackend` exits before a normalized attempt can set terminal lifecycle metadata.

## Solution Assessment

Yes. The narrow maintainable fix is to update the central liveness contract/resolver and runner error backstop, then cover abort and thrown-attempt paths in the existing embedded-runner and lifecycle tests.
