## Summary

Keep open. Current main and the latest stable tag still have the two reported lifecycle gaps: user aborts without payloads resolve as `blocked`, and a rejected embedded-attempt backend exits through a rethrow path before the runner has an attempt result to carry terminal lifecycle metadata.

## What This Changes

The issue asks for `runEmbeddedPiAgent` to guarantee terminal lifecycle metadata on uncaught embedded-attempt failures and to expose user aborts as a distinct `cancelled` liveness state.

## Best Possible Solution

Add `cancelled` at the central embedded-runner liveness contract/resolver boundary, add a runner/backend terminal-metadata backstop for thrown attempts, and cover both paths with focused regression tests.

## Reproduction Assessment

Yes, by source inspection on current main. Passing `aborted: true`, `timedOut: false`, and `payloadCount: 0` reaches the `blocked` branch, and a `runEmbeddedAttemptWithBackend` rejection rethrows before an attempt result can provide terminal lifecycle metadata.

## Solution Assessment

Yes. The maintainable fix belongs at the shared liveness type/resolver plus the embedded-runner terminal boundary, with downstream lifecycle consumers updated for the new state.
