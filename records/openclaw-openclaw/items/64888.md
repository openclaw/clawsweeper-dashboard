## Summary

Keep open. Current main still lacks the requested `cancelled` embedded-run liveness state and still lets thrown embedded attempts cross the runner boundary without replay/liveness terminal metadata, so the prior source-level reproduction remains valid.

## What This Changes

Request to add runner error-boundary terminal lifecycle metadata and a distinct `cancelled` embedded-run liveness state.

## Best Possible Solution

Extend the shared embedded liveness type/resolver and lifecycle consumers for `cancelled`, then add a runner-level terminal metadata backstop before rethrowing uncaught attempt failures.

## Reproduction Assessment

Yes, by source inspection. On current main, `resolveRunLivenessState({ payloadCount: 0, aborted: true, timedOut: false, ... })` reaches the `blocked` branch, and a rejected `runEmbeddedAttemptWithBackend` is rethrown before attempt metadata is available.

## Solution Assessment

Yes. The narrow maintainable fix is to update the central embedded-run liveness contract/resolver plus the runner error backstop, with regression tests around abort and thrown-attempt paths.
