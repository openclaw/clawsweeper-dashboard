## Summary

Keep open. Current main still collapses user aborts into `blocked`, the liveness union has no `cancelled` state, and `runEmbeddedAttemptWithBackend` is awaited without a local error boundary that can set terminal replay/liveness metadata before rethrowing.

## What This Changes

Request to add a distinct `cancelled` embedded-run liveness state and a terminal lifecycle metadata backstop around uncaught embedded-attempt failures.

## Best Possible Solution

Extend the embedded liveness contract with `cancelled`, return it only for user aborts that are not timeouts, and add a narrow runner-level error backstop that emits explicit terminal replay/liveness metadata before rethrowing.

## Reproduction Assessment

Yes. A focused unit check of `resolveRunLivenessState` with `payloadCount: 0`, `aborted: true`, and `timedOut: false` currently returns `blocked`, and a mocked `runEmbeddedAttemptWithBackend` rejection follows the unguarded await path before terminal metadata can be set.

## Solution Assessment

Yes. The narrow maintainable path is to update the central liveness resolver and typed contract, adjust lifecycle consumers/tests, and add an error-boundary fallback around attempt dispatch rather than redesigning the runner.

## Repair Work Prompt

Fix the embedded-run terminal lifecycle gaps tracked by https://github.com/openclaw/openclaw/issues/64888. Add `cancelled` to `EmbeddedRunLivenessState`, update `resolveRunLivenessState` so `aborted && !timedOut` with no payloads returns `cancelled` while hard errors/timeouts remain `blocked`, and update typed lifecycle/subscription consumers and tests that assume the four-state set. Add a narrow `runEmbeddedPiAgent` error-boundary/backstop around `runEmbeddedAttemptWithBackend` or the smallest safe surrounding scope so uncaught attempt failures emit terminal lifecycle metadata with `replayInvalid: true` and `livenessState: "blocked"` before rethrowing. Preserve existing handled branches, do not redesign the runner or harness SPI, and do not overwrite richer terminal metadata already set by successful/handled paths. Add a changelog entry if the lifecycle state is user-visible in release notes policy.
