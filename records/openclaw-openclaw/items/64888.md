## Summary

Keep open. Current main and v2026.4.29 still lack the requested cancelled embedded-run liveness state, and uncaught embedded-attempt error paths can still rethrow without replay/liveness terminal metadata.

## What This Changes

The issue requests an embedded-runner error boundary around runEmbeddedAttempt plus a distinct cancelled liveness state for user-initiated aborts.

## Best Possible Solution

Add cancelled to the embedded liveness contract, return it for user-initiated aborts that are not timeouts, preserve blocked for hard failures/timeouts, and ensure every embedded rethrow/error path emits explicit replay/liveness terminal metadata with focused regression coverage.

## Reproduction Assessment

Yes. A unit check of resolveRunLivenessState with payloadCount 0, aborted true, and timedOut false currently returns blocked, and a mocked runEmbeddedAttemptWithBackend throw follows the unguarded await path before terminal metadata can be set.

## Solution Assessment

Yes. The narrow maintainable fix is to extend the liveness contract and central resolver, update downstream typed consumers/tests, and add an error-boundary/backstop for uncaught attempt failures rather than redesigning the runner.

## Repair Work Prompt

Fix the embedded runner lifecycle gaps tracked by https://github.com/openclaw/openclaw/issues/64888. Add `cancelled` to `EmbeddedRunLivenessState`, make `resolveRunLivenessState` return `cancelled` for user-initiated aborts where `aborted === true`, `timedOut === false`, and no higher-priority paused/abandoned state applies, and keep `blocked` for timeouts, provider errors, strict-agentic blocked exits, and other hard failures. Add an uncaught-error boundary/backstop around the `runEmbeddedAttemptWithBackend` path so raw attempt throws and generic prompt-error rethrows produce terminal lifecycle metadata with `replayInvalid: true` and an appropriate blocked/error liveness state before rethrowing or via the caller lifecycle backstop when no attempt object exists. Sweep downstream typed consumers and tests that assume only working/paused/blocked/abandoned. Keep the change scoped to embedded lifecycle metadata; do not redesign the harness SPI or change unrelated provider/auth behavior. Add or update focused regression tests for forced attempt throw and user abort distinct from timeout. Add a user-facing CHANGELOG.md fix entry if the lifecycle/state change is visible to operators, crediting `Thanks @100yenadmin`.
