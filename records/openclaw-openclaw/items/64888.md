## Summary

Keep open. Current main still has the two reported gaps: user aborts without timeout still collapse into `blocked`, and a thrown embedded attempt can leave the runner before terminal replay/liveness metadata is available.

## What This Changes

The issue requests a runner error boundary around embedded attempts and a distinct `cancelled` liveness state for user-initiated aborts.

## Best Possible Solution

Add `cancelled` at the shared embedded liveness contract/resolver boundary, propagate it through lifecycle consumers, and add a runner-level terminal metadata backstop for uncaught attempt failures with focused regression tests.

## Reproduction Assessment

Yes, by source inspection on current main. An aborted, non-timeout, zero-payload run reaches the `blocked` branch, and a rejected `runEmbeddedAttemptWithBackend` exits before an attempt result can provide terminal lifecycle metadata.

## Solution Assessment

Yes. The narrow maintainable fix is to update the central liveness contract/resolver and the runner error backstop, then cover abort and thrown-attempt paths in existing embedded-runner and lifecycle tests.
