## Summary

Keep open. Current main still has the reported lifecycle gap: user aborts without timeout are indistinguishable from blocked terminal failures, and the embedded attempt dispatch path can throw before attempt-scoped terminal lifecycle metadata can be attached.

## What This Changes

The issue requests embedded-agent lifecycle hardening: terminal metadata on uncaught attempt failures and a distinct `cancelled` liveness state for user aborts.

## Best Possible Solution

Extend the embedded liveness contract with `cancelled`, return it for user aborts that are not timeouts, and add a runner-level error backstop that emits explicit replay/liveness metadata before rethrowing uncaught attempt failures.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `resolveRunLivenessState({ payloadCount: 0, aborted: true, timedOut: false, ... })` reaches the current `blocked` branch, and a rejected `runEmbeddedAttemptWithBackend` crosses an unguarded await before fallback lifecycle metadata can be set.

## Solution Assessment

Yes. The proposed direction is the narrow maintainable fix: update the central liveness type/resolver and embedded runner dispatch backstop, then adjust downstream lifecycle/backstop tests for the new value.
