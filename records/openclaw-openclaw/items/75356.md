This PR should stay open: the marker-backed routing direction is right, but the latest diff still allows stale heartbeat sessionIds to survive the rewrite and still lacks the required user-facing changelog entry.

## What This Changes

The PR projects isolated-heartbeat base-session metadata into Gateway/UI session rows, hides heartbeat-only rows in the Control UI picker, normalizes stale WebChat operations to the base session, adds regression tests, and restores a tool-display serializer export.

## Best Possible Solution

Keep the marker-backed UI/Gateway routing approach, but clear or re-resolve stale heartbeat backing sessionIds during normalization, add the active-release changelog entry, and cover the stale `currentSessionId` case in tests.
