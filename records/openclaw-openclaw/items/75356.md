## Summary

Keep this PR open: the marker-based routing direction is right, but the current patch can still carry a stale heartbeat transcript id after rewriting to the base session, and it is missing the required user-facing changelog entry.

## What This Changes

This PR projects isolated-heartbeat base-session metadata into Gateway session rows, hides marker-backed heartbeat rows from the Control UI picker, rewrites stale UI/Gateway chat sends to the base key, restores a tool-display serializer export, and adds focused tests.

## Best Possible Solution

Keep the marker-backed UI/Gateway normalization, but invalidate or re-resolve backing session ids whenever a heartbeat key is rewritten, add stale-`currentSessionId` regression coverage, and add the Unreleased Fixes changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a marker-backed heartbeat row can be the only listed row, the PR rewrites the key to the base session, but stale heartbeat `currentSessionId`/`p.sessionId` can still be forwarded and reused when the base entry is absent.

## Solution Assessment

No, not yet. Marker-backed normalization is the narrow maintainable fix, but it must also treat the backing transcript id as invalid when the selected key changes and record the user-visible fix in the changelog.
