## Summary

Keep open: the PR is directionally right, but it still forwards a stale heartbeat `currentSessionId` after rewriting the active key to the base session, so pruned-base sends can still use the heartbeat transcript id. The current head also lacks the required user-facing changelog entry.

## What This Changes

The PR projects isolated-heartbeat base-session metadata into session rows, hides marker-backed heartbeat rows in the Control UI picker, normalizes UI/Gateway chat sends to the base key, adds focused tests, and restores a tool-display serializer export.

## Best Possible Solution

Keep the marker-backed routing approach, but clear or re-resolve the backing session id whenever a heartbeat row is normalized to its base key, add regression coverage for stale `currentSessionId`, and add the Unreleased Fixes changelog entry before merge.

Yes.
