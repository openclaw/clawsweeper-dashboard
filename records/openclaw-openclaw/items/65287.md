## Summary

Keep open. Current main still lacks the requested Anthropic-native compaction controls, Anthropic `context_management.edits` request wiring, compaction-block stream/replay handling, and mixed cache-TTL policy; related #10213 and #54041 overlap but do not cover the full stable-prefix/short-trailing-cache acceptance criteria.

## What This Changes

Add opt-in Anthropic server-side compaction and a mixed prompt-cache retention policy that keeps stable prefixes at `1h` while leaving volatile conversation content at `5m`.

## Best Possible Solution

Keep a canonical Anthropic context-management effort open that defines the public config/API, gates required beta headers, preserves compaction blocks through streaming/history, and applies long TTL only to stable prefix material.
