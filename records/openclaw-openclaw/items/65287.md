## Summary

Keep open. Current main and the latest release still lack Anthropic-native `context_management` compaction, compaction block stream/history replay, and the requested mixed stable-prefix/trailing-content cache TTL policy; the related open trackers overlap but do not fully replace the mixed-cache portion.

## What This Changes

Request Anthropic native server compaction support plus mixed prompt-cache TTLs that keep stable Anthropic prompt prefixes on 1-hour retention and trailing conversation content on 5-minute retention.

## Best Possible Solution

Define one maintainer-approved Anthropic context-management and cache-policy path that gates the right betas, preserves compaction blocks through stream/history replay, and keeps stable prefixes on long retention while volatile trailing content remains short-lived.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source, docs, release, and related-tracker checks give high confidence that current main does not expose the requested Anthropic-native compaction and mixed TTL behavior.

## Solution Assessment

Unclear for autonomous implementation. The direction is plausible, but maintainers should first choose the public API/config split across per-model Anthropic params, compaction config, or a broader context-management contract.
