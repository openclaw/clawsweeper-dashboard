## Summary

Keep open. The latest revision moved the broad fallback inheritance out of the shared agent helper, but the new cron guard is still evaluated against the rewritten cron defaults during preflight, so explicit differing string agent models can still inherit global fallback providers when the primary fails preflight.

## What This Changes

This PR narrows cron fallback inheritance for string agent model configs, adds focused cron fallback-policy tests, and adds a local proof script under `scripts/repro`.

## Best Possible Solution

Keep cron inheritance scoped to the default-primary shorthand by computing that decision from raw runtime config before defaults are rewritten, or by threading an explicit cron-local inheritance signal through preflight and execution.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: string agent model configs currently produce an empty fallback override, while this PR's remaining bug follows from preflight passing `cfgWithAgentDefaults` after per-agent strings have been copied into defaults.

## Solution Assessment

No. The fix is in the right cron-owned area, but comparing the agent string to the already-rewritten default primary is not the best repair; the inheritance decision needs raw-config provenance or an explicit forwarded signal.
