## Summary

Keep open. Current main still has the source-proven Telegram callback retry path, and an open closing PR is the active fix path, so this issue should remain open until that PR merges or is closed.

## What This Changes

The issue reports that expired Telegram exec approval callback queries are retried forever from the persisted polling spool, blocking later Telegram messages.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/82388; once a fix merges, close this issue as implemented with the merge commit proof.

## Reproduction Assessment

Yes at source level: an expired approval resolver error follows the retryable callback wrapper, and the polling spool keeps thrown handler errors for retry. I did not run a live Telegram credentialed reproduction in this read-only pass.

## Solution Assessment

Yes: the narrow maintainable solution is to classify approval-not-found or expired callback-query failures as handled for stale approval callbacks while preserving retries for transient resolver failures.
