## Summary

Current main still treats expired Telegram exec approval callbacks as retryable, and isolated ingress keeps failed spooled updates for retry, so this is a valid narrow bug rather than a cleanup-close candidate.

## What This Changes

The issue asks Telegram polling to discard expired callback_query exec approval updates instead of retrying their persisted spool files forever.

## Best Possible Solution

Handle stale approval IDs and expired callback-query acknowledgements as completed non-retryable Telegram callback clicks, while keeping retries for transient callback failures.

## Reproduction Assessment

Yes at source level: an expired approval resolver error currently follows the callback retry wrapper and the isolated ingress spool keeps the update after any thrown handler error. I did not run a live Telegram reproduction in this read-only pass.

## Solution Assessment

Yes: the narrow maintainable fix is to classify approval-not-found or Telegram expired-callback errors as non-retryable completion for approval callbacks only, reusing the existing approval-not-found helper where possible.
