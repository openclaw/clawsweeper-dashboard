## Summary

Current main still rethrows expired Telegram approval callback failures and polling ingress keeps failed spooled updates for retry. This issue should stay open because the linked closing PR is open and unmerged.

## What This Changes

The issue requests treating expired Telegram approval callback queries as non-retryable so persisted polling spool entries are discarded instead of replayed forever.

## Best Possible Solution

Land a focused Telegram fix that treats approval-not-found callback resolutions as handled while preserving retries for transient resolver and edit failures.

## Reproduction Assessment

Yes, at source level: make the Telegram approval resolver reject with 'unknown or expired approval id' and the callback handler rethrows while polling-session keeps the spooled update. I did not run live Telegram in this read-only pass.

## Solution Assessment

Yes: the narrow maintainable fix is to classify stale approval-id callback failures as completed/non-retryable in the Telegram approval callback path only, leaving generic approval resolver failures retryable.
