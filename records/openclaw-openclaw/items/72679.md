## Summary

Keep open. The underlying Nextcloud Talk reaction bug is real on current main, but this branch is not merge-ready because it mishandles shared reaction semantics and overlaps another open implementation path that a maintainer should consolidate before landing.

## What This Changes

The PR adds a Nextcloud Talk `react` message-action adapter, registers it on the channel plugin, and adds unit tests for discovery, validation, dispatch, and tool-context fallback.

## Best Possible Solution

Land one consolidated Nextcloud Talk action implementation that wires `react`, scopes discovery to configured accounts, uses shared reaction helpers, explicitly rejects or implements unsupported removal semantics, covers sender request shape, and adds the required changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows Nextcloud Talk advertises reactions and has `sendReactionNextcloudTalk`, but the channel plugin has no `actions.handleAction`, so the shared dispatcher cannot reach the sender.

## Solution Assessment

No. The channel-action seam is the right place to fix the bug, but this branch is not the best merge form until it follows shared reaction semantics, scopes discovery, adds changelog coverage, and is reconciled with #72348.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Reject or implement reaction removals:** `extensions/nextcloud-talk/src/channel.adapters.ts:78-84`
  - body: The shared `react` action accepts `remove`, but this handler ignores it and always calls the POST/add sender. A caller asking to remove a Nextcloud Talk reaction would add or refresh the reaction instead, so reject `remove: true` or implement the DELETE path before sending.
  - confidence: 0.9
- **[P2] Use the shared reaction message-id resolver:** `extensions/nextcloud-talk/src/channel.adapters.ts:74`
  - body: This only accepts string `params.messageId` and string `toolContext.currentMessageId`, so the documented `message_id` alias and numeric current-message fallback are ignored. Use `resolveReactionMessageId` and stringify the resolved id before sending.
  - confidence: 0.87
- **[P3] Scope discovery to configured accounts:** `extensions/nextcloud-talk/src/channel.adapters.ts:59`
  - body: `describeMessageTool` advertises `react` without checking `cfg` or an account-scoped target, so the shared message tool can expose Nextcloud Talk reactions even when no enabled account has both a base URL and bot secret. Mirror the other channel action adapters and return no action until a configured account is available.
  - confidence: 0.8
- **[P3] Add the required changelog entry:** `extensions/nextcloud-talk/src/channel.ts:97`
  - body: This is a user-facing Nextcloud Talk bug fix, but the PR does not update `CHANGELOG.md`. Add a single-line entry under the active Fixes section with appropriate non-bot contributor/report credit.
  - confidence: 0.83
