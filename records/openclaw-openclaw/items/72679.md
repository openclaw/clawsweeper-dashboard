## Summary

Keep open. Current main still has the underlying Nextcloud Talk reaction dispatch gap, and this PR uses the right channel-action seam, but the patch is not merge-ready because it misses parts of the shared reaction contract, advertises the action without configured-account scoping, and overlaps with the still-open #72348 implementation for the same #70110 bug.

## What This Changes

This PR adds a Nextcloud Talk message-action adapter for `react`, registers it on the channel plugin, and adds unit tests for dispatching reactions through the existing sender.

## Best Possible Solution

Land one consolidated Nextcloud Talk message-actions implementation that registers `react`, scopes discovery to configured accounts, resolves message IDs through the shared helper, handles removal via the documented DELETE path or rejects it explicitly, covers the sender request shape, and adds the required changelog entry before closing #70110.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists on current main: Nextcloud Talk advertises reactions and has `sendReactionNextcloudTalk`, but the plugin lacks `actions.handleAction`, so `dispatchChannelMessageAction` returns `null` before a reaction request can be sent.

## Solution Assessment

No. Registering a channel action adapter is the right seam, but this patch is not the best form because it bypasses shared reaction helpers, mishandles removal requests, advertises unconfigured accounts, and lacks changelog coverage while #72348 is already tracking a broader implementation.
