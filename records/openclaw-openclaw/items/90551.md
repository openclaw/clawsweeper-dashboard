## Summary

Keep open. Current main and v2026.6.8 still run update finalization doctor in repair mode, and the repair preflight can still replace an invalid non-plugin-local config wholesale from last-known-good, so the reported stale rollback and restore-observability gap remain unresolved.

## What This Changes

The issue asks update-time doctor config recovery to stop silently replacing the active openclaw.json with a stale last-known-good copy, or to surface durable restore evidence when full restore remains possible.

## Best Possible Solution

Maintainers should define one update-time config-recovery contract: preserve unrelated current config or require explicit restore acceptance, keep intentional last-good recovery policy-bounded, and emit durable audit/status/reason evidence for every full restore.

## Reproduction Assessment

Yes, at source level: current update finalization still runs repair-mode doctor, and repair preflight can still restore invalid non-plugin-local config from last-known-good. I did not run a live end-to-end openclaw update repro in this read-only review.

## Solution Assessment

Unclear. Adding restore observability is a safe part of the fix, but changing when whole-file last-good restore is allowed needs maintainer policy because it trades off against startup/channel availability recovery.
