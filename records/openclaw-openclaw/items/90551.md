## Label Justifications

## Summary

Keep open. Current main and v2026.6.8 still allow update finalization to run repair-mode doctor, and repair preflight can still replace an invalid non-plugin-local config wholesale from last-known-good; the stale rollback and restore-observability problem remains a maintainer policy decision.

## What This Changes

The issue asks OpenClaw to prevent `openclaw update` from silently restoring a stale last-known-good config and discarding intentional config changes, or at least make that restore durable and visible.

## Best Possible Solution

Maintainers should define one update-time config recovery contract that preserves unrelated current config, bounds any whole-file last-good restore, and emits durable audit/status/reason evidence for every restore.

## Reproduction Assessment

Yes at source level: current update finalization still runs repair-mode doctor, and repair preflight can still restore an invalid non-plugin-local config from last-known-good. I did not run a live end-to-end update repro in this read-only review.

## Solution Assessment

Unclear. Adding durable restore observability is a safe part of the fix, but changing when whole-file last-good restore is allowed needs maintainer policy because it trades off against startup/channel availability recovery.
