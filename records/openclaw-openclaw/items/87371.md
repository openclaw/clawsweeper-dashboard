## Best Possible Solution

Keep the shipped YOLO contract and have a maintainer either close this protected security-labeled issue as already shipped or open a narrower security-policy follow-up if current `security=full`, `ask=off`, `host=gateway` still misbehaves.

## Reproduction Assessment

The original v2026.5.26 failure is credible from the report and older related issues, but current source points to fixed behavior.

## Solution Assessment

Yes for the core bug: the current implementation bypasses the extra script preflight only for trusted gateway YOLO while keeping ambiguous interpreter forms fail-closed elsewhere. The remaining decision is whether maintainers are comfortable closing a protected security-labeled item based on that shipped behavior.
