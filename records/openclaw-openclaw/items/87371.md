## Summary

Keep open: current code and docs make the no-preflight path depend on YOLO mode (`security=full` plus `ask=off`), while this report only proves `security=full` and asks for a security-policy/heuristic change that should not be auto-closed or auto-fixed without the effective host/ask details.

## What This Changes

The issue asks OpenClaw exec to stop blocking trusted Python/Node script commands combined with pipes or redirects when `tools.exec.security` is `full`.

## Best Possible Solution

Add focused coverage for the exact effective policy and command, then narrow the parser or policy guard so concrete script-file pipes/redirections work without weakening ambiguous interpreter fail-closed cases.

## Reproduction Assessment

No high-confidence reproduction path is pinned down yet. The current source and release tag skip preflight for gateway YOLO mode, so maintainers need the effective `ask`, host, host approvals, and exact logged command to confirm the reported block.

## Solution Assessment

Unclear. A blanket `security=full` preflight bypass or new opt-out knob is a security-policy decision; the narrower maintainable path is to prove the false positive and adjust only the concrete-script pipeline/redirection heuristic if that is the actual failing case.
