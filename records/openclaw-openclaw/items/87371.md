## Summary

Close as implemented: current main and the latest release keep trusted host exec in YOLO mode out of the Python/Node script-preflight rejection layer, and direct script-file invocations with pipes are now routed through script-target validation instead of the ambiguous fail-closed branch.

## What This Changes

The issue requests that trusted exec configurations stop rejecting legitimate interpreter script commands combined with shell pipes or redirects.

## Best Possible Solution

Keep the shipped YOLO contract and current parser behavior; open a new focused issue only if a current-release `security=full`, `ask=off`, `host=gateway` command still hits this preflight with full effective-policy details.

## Reproduction Assessment

No current-main failure reproduction remains from source inspection: trusted gateway YOLO skips script preflight, and direct `python3 <file>.py | head` forms are parsed as script-file targets before the ambiguous-interpreter fail-closed branch.

## Solution Assessment

Yes, the current solution is the best boundary: trusted host exec bypasses the extra preflight only in full YOLO mode, while non-YOLO ambiguous interpreter forms remain fail-closed for security.
