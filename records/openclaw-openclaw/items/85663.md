## Best Possible Solution

Keep the hardening only with an explicit compatibility plan: preserve existing defaults or make stricter behavior maintainer-approved, documented, tested, and proven for upgrades.

No: the security direction is reasonable, but the current patch changes compatibility defaults without a maintainer-approved upgrade plan and leaves test contracts failing. A compatibility-preserving default or documented opt-in strict mode is the safer path.
