## Summary

Close as duplicate/superseded: the central async tool-result governance gap remains real on current main, but the remaining API design is now better consolidated in the open async hook thread and the broader pre-model sanitization thread.

## What This Changes

The issue requests async-capable `tool_result_persist` handling so governance plugins can call external DLP/policy services before tool results enter transcript or model context.

## Best Possible Solution

Keep the remaining async tool-result contract discussion in https://github.com/openclaw/openclaw/issues/62123 and use https://github.com/openclaw/openclaw/issues/29132 for the broader pre-model sanitization boundary.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a Plugin SDK feature request. Source inspection gives high confidence that current main still ignores async `tool_result_persist` and `before_message_write` handler results.

## Solution Assessment

Yes for triage: consolidating this issue into the open async hook thread is the narrowest maintainable cleanup. Directly awaiting the existing sync persistence hook is not clearly the best implementation until maintainers settle the trusted pre-model/pre-transcript API contract.
