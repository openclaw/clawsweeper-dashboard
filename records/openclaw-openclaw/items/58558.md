## Summary

Close as duplicate/superseded: the async governance gap is real on current main, but the remaining API design is already covered by the open async tool-result mutation thread and the broader pre-model sanitization thread.

## What This Changes

Add Promise-return support to `tool_result_persist` and `before_message_write` so governance plugins can asynchronously scan or mutate tool results before transcript/model exposure.

## Best Possible Solution

Keep the remaining async tool-result contract design in https://github.com/openclaw/openclaw/issues/62123 and the broader pre-model sanitization boundary in https://github.com/openclaw/openclaw/issues/29132, rather than changing the existing sync persistence hook directly.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a Plugin SDK feature request. Source inspection gives high confidence that current main still ignores async `tool_result_persist` and `before_message_write` handler results.

## Solution Assessment

Yes for triage: consolidating this into the open async hook thread is the narrowest maintainable cleanup. Directly awaiting the existing sync hot-path persistence hook is not clearly the best implementation until maintainers settle the trusted pre-model/pre-transcript API contract.
