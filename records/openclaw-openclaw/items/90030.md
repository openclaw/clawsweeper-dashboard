## What This Changes

The branch skips `manager.sync({ reason: "search", force: true })` for QMD-backed zero-hit `memory_search` calls and adds a regression test preserving non-QMD retry behavior.

## Best Possible Solution

Resolve the current main conflict, keep the backend-specific guard if maintainers accept bounded QMD search latency over forced immediate freshness, and preserve the non-QMD zero-hit retry path.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.6 still await forced sync after zero `memory_search` hits, and QMD sync awaits update work with a much longer timeout budget. I did not run a live QMD timing reproduction in this read-only pass.

## Solution Assessment

Yes, with maintainer acceptance: this is the narrowest owner-local fix to keep QMD update work out of `memory_search` while preserving non-QMD zero-hit repair. If forced freshness remains a contract, a nonblocking refresh path is safer than the current synchronous fallback.
