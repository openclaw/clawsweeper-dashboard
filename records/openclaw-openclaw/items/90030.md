## What This Changes

This PR adds a QMD backend guard to skip the zero-hit forced sync/retry in `memory_search` and adds a regression test while leaving non-QMD retry behavior intact.

## Best Possible Solution

Resolve conflicts against current main and land the backend-aware guard if maintainers accept prompt zero-hit QMD results over forced immediate freshness; otherwise replace the synchronous retry with async or debounced QMD refresh.

## Reproduction Assessment

Yes, at source level: current main awaits forced sync after zero memory hits, and QMD sync awaits update maintenance with a longer timeout budget. The linked issue and PR include live QMD evidence, but I did not run an independent live QMD timing repro in this read-only pass.

## Solution Assessment

Yes with maintainer acceptance: the backend guard is the narrowest owner-local fix that keeps non-QMD retry behavior while keeping QMD update work out of the interactive search path. If forced QMD freshness is required, a nonblocking refresh path is the safer alternative.
