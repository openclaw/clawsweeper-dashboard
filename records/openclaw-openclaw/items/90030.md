## What This Changes

This PR skips the synchronous forced sync/retry path for QMD-backed zero-hit `memory_search` results and adds a regression test while leaving non-QMD retry behavior intact.

## Best Possible Solution

Resolve the conflicts and land the backend-aware guard if maintainers accept prompt QMD zero-hit results over synchronous forced freshness; otherwise replace the forced retry with bounded or background QMD refresh outside `memory_search`.

## Reproduction Assessment

Yes at source level: current main still synchronously forces `manager.sync({ reason: "search", force: true })` after zero-hit `memory_search` results, and QMD sync waits on update maintenance. I did not run an independent live large-corpus QMD timing repro in this read-only pass.

## Solution Assessment

Yes with maintainer acceptance: the guard is the narrow owner-local fix and preserves non-QMD retry behavior. If immediate QMD freshness repair is contractual, a bounded or background refresh path is the safer alternative.
