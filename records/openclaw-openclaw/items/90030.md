## What This Changes

The PR changes `memory_search` so QMD zero-hit searches skip the synchronous forced sync-and-retry path while preserving the retry for other backends, with a regression test.

## Best Possible Solution

Resolve the dirty branch and land the backend-aware guard if maintainers accept prompt QMD zero-hit results over synchronous forced freshness; otherwise replace the forced retry with nonblocking QMD refresh outside `memory_search`.

## Reproduction Assessment

Yes at source level: current `memory_search` still awaits forced sync after zero hits, and QMD sync runs update work that can exceed the tool deadline. I did not run an independent live QMD timing reproduction in this read-only pass.

## Solution Assessment

Yes with maintainer acceptance: the guard is the narrow owner-local fix and preserves non-QMD retry behavior. If immediate QMD freshness repair is required, a nonblocking refresh path is the safer alternative.
