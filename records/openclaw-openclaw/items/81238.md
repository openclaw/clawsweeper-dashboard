## Summary

Keep open: the PR is a useful bundled MCP schema-compatibility fix, but the current resolver still drops validation siblings on resolved local refs, which is a concrete merge blocker. The protected `maintainer` label also rules out cleanup-closing this PR.

## What This Changes

The PR inlines local JSON Schema refs in tool-parameter normalization, applies that to bundled MCP tool materialization, adds Notion-style regression tests, and updates the changelog.

## Best Possible Solution

Keep the shared normalizer and bundled MCP materialization boundary, but preserve adjacent `$ref` validation siblings, preferably by composing them with the resolved schema and adding focused regression coverage.

## Solution Assessment

No. Normalizing local refs at the shared tool-schema boundary is the right shape, but the resolver must preserve or compose adjacent validation siblings instead of returning only the referenced schema plus metadata.
