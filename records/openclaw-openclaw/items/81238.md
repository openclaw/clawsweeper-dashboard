## Summary

Keep open: the latest PR head still drops validation siblings when resolving local `$ref`, which can weaken MCP JSON Schema 2020-12 validation before tool dispatch.

## What This Changes

## Best Possible Solution

Keep the PR open, preserve or compose non-`$ref` sibling keywords when inlining local refs, and add focused regression coverage for that case.

## Reproduction Assessment

No for the original Notion rejection on current main: maintainer live validation reached Notion on both main and PR head. Yes for the PR regression: source inspection shows resolved `$ref` schemas copy only the referenced schema plus annotations, dropping validation siblings.

## Solution Assessment

No. Normalizing local refs at the shared tool-schema boundary is the right shape, but the resolver must preserve or compose `$ref` sibling keywords to keep JSON Schema semantics intact.
