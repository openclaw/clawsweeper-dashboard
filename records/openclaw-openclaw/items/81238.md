## Summary

Keep open: the PR is a plausible bundled MCP schema compatibility fix with sufficient real-behavior proof, but the latest head still has a P2 correctness bug because resolved local `$ref`s drop adjacent validation keywords that JSON Schema 2020-12 allows. The `maintainer` label also means this PR should not be cleanup-closed.

## What This Changes

The PR adds local `$ref` inlining for tool parameter schemas, applies it to bundled MCP tool materialization, adds Notion-style regression coverage, and updates the changelog.

## Best Possible Solution

Keep the shared bundled MCP schema-normalization approach, but repair local `$ref` inlining so it preserves JSON Schema 2020-12 validation semantics and add focused regression coverage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No for the original current-main Notion pre-dispatch rejection, because maintainer live validation reached Notion on both main and PR head; yes for the PR regression, because the resolver source drops adjacent `$ref` validation siblings after successful local resolution.

## Solution Assessment

Is this the best way to solve the issue? No. Normalizing local refs at the shared tool-schema boundary is the right shape, but the resolver must preserve or compose adjacent sibling validation keywords rather than returning only the referenced schema plus metadata.
