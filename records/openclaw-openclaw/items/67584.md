## Summary

Keep this PR open: current main still has a single registration-time memory-wiki vault path, while the PR adds a focused plugin-owned feature for context-aware vault resolution and addresses the main fail-closed review concerns. The PR adds memory-wiki `vault.path` placeholder expansion for wiki tool invocations, refactors all wiki tools to guarded context-aware factories, skips memory prompt/corpus supplements for placeholder paths, and updates docs/tests.

## Best Possible Solution
