## Summary

Keep this PR open: current main still lacks ctx-aware memory-wiki vault path resolution, and the PR is a plausible scoped plugin-side implementation, but the external contributor still needs real after-fix behavior proof and a small stale comment cleanup before merge.

## What This Changes

The PR adds invocation-context templating for memory-wiki `vault.path`, routes all wiki tools through per-context factories, skips memory supplements for templated paths, and updates docs/tests.

## Best Possible Solution

Land this scoped memory-wiki templating after real behavior proof, stale comment cleanup, and fresh green gates; keep broader per-agent plugin config on https://github.com/openclaw/openclaw/issues/63829 and separate CLI multi-vault work on its own tracker.
