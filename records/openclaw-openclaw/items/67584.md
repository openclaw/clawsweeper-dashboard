## Summary

## What This Changes

The PR adds memory-wiki `vault.path` template tokens resolved from tool context, converts wiki tools to factories with config-error stubs, skips prompt/corpus supplements for templated paths, documents the behavior, and adds tests.

## Best Possible Solution

Land a rebased version that resolves templates only on trusted context-bearing surfaces, rejects or disables placeholder configs on CLI/gateway/no-context writes, keeps the prompt/corpus limitation intentional and documented, and includes redacted real-run proof of vault isolation.

Yes, at source level: configure `vault.path` as `{workspaceDir}/wiki`, then invoke a no-context gateway or CLI write path such as `wiki.init`; the unresolved config reaches `initializeMemoryWikiVault`, which calls recursive `fs.mkdir` on that path.

## Solution Assessment

No. Tool-side templating and stub errors are a plausible direction, but the best fix must also fail closed on no-context CLI/gateway paths or define a maintainer-approved context contract for those surfaces.
