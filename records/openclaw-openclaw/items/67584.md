## What This Changes

The branch adds memory-wiki `vault.path` placeholders resolved from plugin tool context, converts wiki tools to context-aware factories, skips prompt/corpus supplements for placeholder configs, documents the behavior, and adds tests.

## Best Possible Solution

Land a rebased version that resolves templates only where trusted context exists, fails closed or disables no-context CLI/gateway filesystem writes for placeholder paths, documents `memory_*` limitations, and includes redacted real-run proof of two isolated vaults.

Yes, by source inspection: configure `vault.path` as `{workspaceDir}/wiki`, then invoke a no-context gateway or CLI write path such as `wiki.init`; it passes the unresolved config to `initializeMemoryWikiVault`, which calls recursive `fs.mkdir` on that path.

## Solution Assessment

No. Tool-side templating is a plausible direction, but the best fix must also fail closed on no-context CLI/gateway paths or establish a maintainer-approved context contract for those surfaces.
