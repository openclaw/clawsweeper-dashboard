## Summary

Keep this issue open: current main and the latest checked release still route the enabled shell-env fallback into the POSIX `/bin/sh` probe on native Windows, and the active closing PR is the right place to finish the fix.

## What This Changes

Skip the login-shell environment fallback on native Windows so startup/status does not warn with `spawnSync /bin/sh ENOENT`.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/84838, then close this issue once main contains the win32 fallback skip and focused regression coverage.

## Reproduction Assessment

Yes for source-level reproduction: with shell-env import enabled and an expected key missing on win32, current main reaches `probeLoginShellEnv` and the `/bin/sh` exec path. I did not run a native Windows host in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is a platform-aware no-op in the fallback/probe path with regression coverage. A PowerShell or Windows profile importer would be a separate product decision, not required for this bug.
