## Summary

Keep open. The reported native-Windows startup warning is still source-reproducible on current main: the enabled shell-env fallback still reaches the `/bin/sh` probe path without a win32 guard, while only the separate PATH helper skips Windows.

## What This Changes

The issue asks OpenClaw to skip the POSIX login-shell environment fallback on native Windows so startup does not warn about missing `/bin/sh`.

## Best Possible Solution

Add a testable native-win32 no-op for the shell-env fallback/probe path, preserve POSIX behavior, and avoid replacing `/bin/sh` with COMSPEC because the current probe uses POSIX flags and `env -0`.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: enable shell-env import on native Windows with a missing expected key and current main calls the `/bin/sh` probe path before logging the reported warning.

## Solution Assessment

Yes. The narrow maintainable fix is a platform-aware skip in the fallback/probe path with regression coverage; changing the fallback shell to cmd.exe would still conflict with the POSIX `-l -c env -0` command shape.

## Repair Work Prompt

Fix the native Windows shell-env fallback warning reported in https://github.com/openclaw/openclaw/issues/84795. In `src/infra/shell-env.ts`, make shell-env import a no-op on `win32` before resolving or executing `/bin/sh`; use a testable platform seam if needed. Preserve existing POSIX `/bin/sh` and trusted-shell behavior, and do not switch Windows to COMSPEC/cmd.exe because the probe command uses POSIX `-l -c env -0`. Add focused regression coverage in `src/infra/shell-env.test.ts` for `loadShellEnvFallback` enabled on win32 with a missing expected key: no exec call, no warning, ok result with no applied keys. Keep the existing `getShellPathFromLoginShell` win32 behavior intact. Add a `CHANGELOG.md` Fixes entry because this is a user-visible startup warning fix. Validate with the focused shell-env test and, if available, a native Windows or Testbox smoke of `OPENCLAW_LOAD_SHELL_ENV=1 openclaw status` confirming no `/bin/sh` warning.
