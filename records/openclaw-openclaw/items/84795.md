## Summary

This should stay open: current main and the latest tagged release still have the failing fallback path, and the linked open fix PR is the correct implementation path before closing the issue.

## What This Changes

The issue asks OpenClaw to skip the optional login-shell environment fallback on native Windows so startup does not try to spawn `/bin/sh`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84838 after the remaining proof or compatibility decision, then close this issue as implemented.

## Reproduction Assessment

Yes. Source inspection shows that enabling shell-env import on native Windows with missing expected keys reaches `probeLoginShellEnv`, which can fall back to `/bin/sh`; I did not run a live native Windows host.

## Solution Assessment

Yes. The narrow maintainable solution is a testable `win32` no-op in the fallback/probe path while preserving POSIX behavior, and the linked open PR implements that shape.
