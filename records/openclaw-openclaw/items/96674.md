This PR changes the Control UI e2e Chromium capability helper to run `--headless=new --version` with a 5-second `spawnSync` timeout and treat `ETIMEDOUT` as runnable.

## Best Possible Solution

Use one bounded headless Chromium capability probe for both the Vitest helper and the preflight installer script, with regression coverage for timeout fallback.

## Solution Assessment
