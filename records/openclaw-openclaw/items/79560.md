## Summary

Keep open: current main still lacks a general per-channel ACP deny/warn policy and a same-shape completion coalescer for noisy ACP children. The adjacent group/channel completion-routing fix reduces one raw-status path, but it does not close the Telegram DM runaway-spawn policy gap described here.

## What This Changes

The issue requests channel-level controls, and possibly announcement coalescing or bounded permission-denial retry behavior, to stop non-interactive Telegram DM sessions from spawning runaway ACP children.

## Best Possible Solution

Define a small channel-aware ACP safety contract, then implement the selected guard with focused tests for `sessions_spawn(runtime="acp")`, task/announcement delivery, and Telegram DM/topic behavior.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection on current main confirms the relevant ACP spawn and task-delivery paths exist and that the requested channel deny/warn policy and same-shape coalescing are absent.

## Solution Assessment

Unclear as proposed: per-channel deny/warn is plausible, but maintainers should decide whether the durable fix belongs in channel policy, completion coalescing, acpx retry handling, or a combination.
