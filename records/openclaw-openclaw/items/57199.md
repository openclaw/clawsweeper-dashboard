## Summary

Keep open: the requested agent targeting is still missing on current main, but this PR is not mergeable as-is because current main already has newer `sessionKey` wake contract and timer semantics that this branch would regress, and the external PR still lacks attached real-behavior proof.

## What This Changes

This PR adds `--agent-id` and `--session-key` to `openclaw system event` and forwards them through wake validation, cron handlers, cron service wake/timer code, and CLI tests.

## Best Possible Solution

Refresh or replace this with one additive wake-target contract update that adds `agentId` end to end while preserving current `sessionKey` validation, forward compatibility, generated clients, docs, and targeted immediate wake behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows `--agent-id` is absent and `CronServiceContract.wake` accepts only `mode`, `text`, and optional `sessionKey`; comparing the PR diff shows it would also overwrite newer `sessionKey` schema and timer semantics.

## Solution Assessment

No. The direction is useful, but this exact branch is stale against current wake semantics; the safer solution is a refreshed additive patch that widens the current contract without regressing `sessionKey` behavior.

  - confidence: 0.84
