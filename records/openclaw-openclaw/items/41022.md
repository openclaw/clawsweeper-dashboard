## Summary

Keep open: current main still has the old CLI watchdog caps and this PR is a useful narrow mitigation, but it still needs focused coverage and a changelog entry before landing.

## What This Changes

The PR raises shared CLI watchdog no-output defaults so fresh sessions use 5/30 minute bounds and resumed sessions can wait up to 30 minutes.

## Best Possible Solution

Land the timeout-default mitigation with direct resolver/backend coverage and one Unreleased changelog line, leaving configurable no-cap behavior and Monitor idle exemption to separate issues.

## Reproduction Assessment

Yes, from source inspection. Current main resolves resumed CLI sessions through a 180_000 ms resume max and passes that timeout into the live-session or supervisor no-output enforcement path; I did not run a live CLI session in this read-only review.

## Solution Assessment

Yes for the narrow mitigation: changing the shared defaults is the smallest maintainable way to remove the hard 3-minute resumed cap for bundled CLI backends. It should not also implement the optional no-cap config or Monitor-idle exemption in this PR.
