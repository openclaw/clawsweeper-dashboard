## What This Changes

The PR raises the shared fresh/resume CLI no-output watchdog default bounds in `src/agents/cli-watchdog-defaults.ts` from 3-10 minute windows to 5-30 minute windows.

## Best Possible Solution

Land this narrow timeout-default mitigation with refreshed watchdog coverage and one Unreleased changelog line, while leaving configurable no-cap and Monitor idle-exemption behavior to separate follow-up work.

Yes. On current main, a resumed CLI backend with no override resolves through `CLI_RESUME_WATCHDOG_DEFAULTS.maxMs: 180_000`, and the runner passes that timeout to live-session or supervisor enforcement.

## Solution Assessment

Yes for the narrow mitigation: raising the shared defaults is the smallest maintainable change for the 3-minute cap. It is not complete as submitted because release notes and coverage need to move with the user-facing behavior change.
