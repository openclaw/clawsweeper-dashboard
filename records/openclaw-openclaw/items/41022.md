## Summary

Keep open: this PR is a narrow, useful mitigation for the CLI no-output watchdog cap, but it still needs regression coverage and a changelog entry before merge.

## What This Changes

This PR raises shared CLI no-output watchdog defaults so fresh sessions use 5/30 minute bounds and resumed sessions can wait up to 30 minutes.

## Best Possible Solution

Land the timeout-default mitigation with direct resolver/backend coverage and one Unreleased changelog line, leaving configurable no-cap and Monitor idle-exemption behavior to separate issues.

## Reproduction Assessment

Yes, from source inspection. Current main resolves resumed CLI sessions through CLI_RESUME_WATCHDOG_DEFAULTS.maxMs of 180_000 and passes the resolved timeout into live-session or supervisor enforcement; I did not run a live CLI in this read-only review.

## Solution Assessment

Yes for the narrow mitigation: changing the shared defaults is the smallest maintainable way to remove the hard 3-minute resumed cap. It is not complete as submitted because regression coverage and release notes need to move with the user-facing behavior change.
