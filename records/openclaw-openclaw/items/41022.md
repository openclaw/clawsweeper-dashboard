## Summary

Keep open: current main still has the 3-minute resumed CLI no-output cap, and this PR is a narrow mitigation, but the constants-only diff should gain regression coverage and a changelog entry before merge.

## What This Changes

The PR raises shared CLI watchdog defaults so fresh runs use a 5-minute floor and 30-minute ceiling, and resumed runs use a 30-minute ceiling.

## Best Possible Solution

Land the timeout-default mitigation with direct resolver/backend coverage and one Unreleased changelog line, while leaving configurable no-cap and Monitor idle-exemption behavior to separate follow-up work.

## Solution Assessment
