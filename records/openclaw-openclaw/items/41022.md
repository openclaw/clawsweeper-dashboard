## Summary

Keep open: current main still enforces the 3-minute resumed CLI no-output cap, and this PR is a valid narrow mitigation but still needs default-policy coverage and changelog alignment before merge.

## What This Changes

The PR raises the shared fresh and resumed CLI watchdog no-output default bounds in `src/agents/cli-watchdog-defaults.ts`.

## Best Possible Solution

Land the timeout-default mitigation with focused resolver/backend coverage and one Unreleased changelog line, while leaving no-cap configuration and Monitor expected-idle behavior to separate follow-up work.

## Reproduction Assessment

Yes, from source inspection. A resumed CLI run with a long outer timeout still resolves through `CLI_RESUME_WATCHDOG_DEFAULTS.maxMs: 180_000`, so the no-output watchdog can fire at 3 minutes even when the session timeout is much higher.

## Solution Assessment

Yes for the narrow mitigation: raising the shared defaults is the smallest maintainable fix for the hard cap. The patch should also update coverage and release notes, and should not fold in the separate no-cap or Monitor-specific behavior.
