## Summary

Keep open. The underlying Feishu topic-lane bug is valid on current main, but this PR is not merge-ready because the patch misses native `topic_group` events and does not match current main's topic-group session-key precedence.

## What This Changes

The PR adds Feishu topic-aware sequential queue keys, tests, a changelog entry, extracted group-session scope logic, and unrelated CI/type/lint cleanup across non-Feishu surfaces.

## Best Possible Solution

Land a narrow Feishu-only current-main fix that derives queue lanes from the same group-chat classification, effective group-session scope, and topic-id semantics as `resolveFeishuGroupSession`, while preserving chat-wide `/stop` and `/btw` lanes.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: two Feishu topic events in the same chat still resolve to the same current-main base sequential key, and `createSequentialQueue()` serializes work by that key.

## Solution Assessment

No. The direction is right, but this patch is not the best merge candidate until it includes `topic_group`, mirrors current session topic precedence, and drops unrelated cleanup.

## Review Findings

  - confidence: 0.88
