## What This Changes

The PR adds Feishu topic-scoped sequential queue keys and tests, plus unrelated CI-unblock edits across Mattermost, OpenAI, Microsoft, Telegram, channels, gateway, and SDK files.

## Best Possible Solution

Land a narrowed current-main Feishu fix that routes native `topic_group` messages through topic-aware lanes, matches the current thread-first topic session contract, preserves chat-wide `/stop` and `/btw`, and includes focused tests plus real Feishu proof.

## Reproduction Assessment

Yes. Source inspection on current main shows Feishu message handling enqueues by `getFeishuSequentialKey()`, and that key ignores topic metadata outside `/stop` and `/btw`, so same-chat topic messages serialize on one lane.

## Solution Assessment

No. The proposed direction is right, but the branch is stale against current main's native `topic_group` contract and should be repaired around `isFeishuGroupChatType` plus thread-first native topic lane ids before merge.
