## Summary

Keep open. Current main still converts imsg RPC tapback/reaction payloads into ordinary inbound text and can dispatch them through the normal message path; open closing PRs, especially https://github.com/openclaw/openclaw/pull/80713, should land or be closed before this issue is resolved.

## What This Changes

The issue asks for iMessage imsg RPC tapback reactions to stop reaching the agent as ordinary text messages.

## Best Possible Solution

Land or replace the active iMessage PR with a plugin-local classifier that handles imsg reaction metadata and legacy tapback text before ordinary message dispatch.

## Reproduction Assessment

Yes. A source-level reproduction is clear: an imsg RPC notification with is_reaction=true is converted to ordinary messageText, and a legacy body like Loved "hello" also reaches resolveIMessageInboundDecision with no tapback branch before dispatch.

## Solution Assessment

Yes for the triage outcome. Keeping this issue open while the closing PRs are unmerged is the maintainable path; the code fix belongs in extensions/imessage before normal inbound dispatch.
