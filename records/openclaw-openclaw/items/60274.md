## Summary

Keep open. Current main still lacks imsg RPC tapback filtering in the iMessage plugin, and open PR #60677 is an explicit implementation candidate with closing syntax, so this issue should wait for that PR or an equivalent owner-local fix to land.

## What This Changes

The issue requests that iMessage imsg RPC tapbacks stop being dispatched to the agent as normal text messages.

## Best Possible Solution

Land or replace PR #60677 with a plugin-local guard that handles imsg reaction metadata and quoted tapback text before dispatch; keep full reaction-event forwarding tracked by #39031.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: pass an allowed inbound imsg RPC message with body text like Loved "hello" through resolveIMessageInboundDecision and current main can return dispatch because no tapback-specific branch exists.

## Solution Assessment

Yes for the triage outcome. The maintainable fix belongs in extensions/imessage, and the active PR should be reviewed or repaired rather than spawning a second issue-driven branch.
