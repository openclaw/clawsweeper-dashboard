## Summary

Keep open. Current main still lacks an imsg RPC tapback filter in the iMessage plugin, and open PR #60677 explicitly targets this bug, so cleanup should wait for that PR or an equivalent owner-local fix to land.

## What This Changes

The issue requests that iMessage imsg RPC tapbacks stop being dispatched to the agent as normal text messages.

## Best Possible Solution

Add a narrow `extensions/imessage` guard that recognizes imsg RPC reaction metadata and known tapback text patterns before pairing or agent dispatch, with regression tests; keep full reaction-event forwarding tracked separately by #39031.

## Reproduction Assessment

Yes. A code-level reproduction is clear: pass an allowed inbound imsg RPC message with body text like `Loved "hello"` through `resolveIMessageInboundDecision()` and current main can return `dispatch` because no tapback branch exists.

## Solution Assessment

Yes. Keeping this issue open is the right cleanup outcome because #60677 is already an open implementation candidate; the maintainable fix belongs in the iMessage plugin, not core.
