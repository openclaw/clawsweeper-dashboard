## Summary

Keep open: the opencode-go-local watchdog is the right ownership boundary, but the current patch is not merge-ready because the synthetic `start` event can consume the longer first-event timeout and abort slow but valid streams too early.

## What This Changes

This PR adds an opencode-go stream termination wrapper and focused tests so stalled OpenAI-compatible streams abort at the provider boundary instead of waiting for stuck-session recovery.

## Best Possible Solution

Keep the opencode-go-local watchdog, but preserve the longer first-event timer until real provider SSE progress arrives and use the shorter idle window only after actual progress.

## Reproduction Assessment

Yes for the PR defect from source: current `openai-completions` emits `start` before SSE iteration, and the new wrapper treats that event as progress and switches to the shorter idle timer.

## Solution Assessment

No for the current patch: provider-owned abort is the right boundary, but the implementation must distinguish synthetic `start` from real provider progress before merge.
