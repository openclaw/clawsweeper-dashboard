## Summary

Current main does not have an opencode-go provider-owned stall guard, so the PR remains necessary, but the submitted fix still surfaces the provider watchdog as an aborted assistant rather than a failover-classifiable timeout/error. That leaves the linked cron fallback symptom unresolved for configured fallbacks.

## What This Changes

The PR wraps the opencode-go provider stream with provider-local first-event and inter-event stall timers, aborts the underlying request signal on timeout, emits an aborted terminal assistant event, and adds wrapper tests.

## Best Possible Solution

Keep the provider-owned raw SSE watchdog, but surface its internal stall as a runner/failover-classifiable timeout or provider error and add an embedded-run regression proving configured fallback rotates instead of only returning an aborted assistant payload.
