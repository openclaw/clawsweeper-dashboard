## Summary

Keep open: the execute-time identity gap still exists on current main, but this branch is stale and not safe to merge without a current-main repair.

## What This Changes

The PR wraps plugin tool `execute` calls to append an identity/session context, adds tests and a type, and updates an adjacent compaction auth caller and mocks.

## Best Possible Solution

Ship a refreshed additive plugin SDK contract that appends a frozen identity-only context after the existing dispatcher arguments, exports and documents it, and leaves current compaction auth behavior intact.

## Reproduction Assessment

Yes. Static inspection is enough: plain registered plugin tool objects can be reused without closing over request context, `resolvePluginTools` returns them unwrapped, and the dispatcher never passes a fifth identity/session argument.

## Solution Assessment

No, not as this branch currently stands. The wrapper direction is the right narrow shape, but the branch must be rebased onto current plugin surfaces and completed as a public SDK/docs/changelog change.
