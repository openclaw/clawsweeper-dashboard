## Summary

Keep open: current main still lacks an execute-time plugin tool identity context, and this PR is useful but not mergeable as-is because it is stale against current plugin resolver/cache surfaces and carries a compaction auth regression.

## What This Changes

The PR wraps plugin tool execution to append an identity/session context, adds tests, and updates adjacent compaction auth mocks.

## Best Possible Solution

Land a refreshed additive plugin SDK contract that appends a frozen identity-only execute context after `signal` and `onUpdate` for both live and cached plugin tool execution, exports and documents the type publicly, and leaves current compaction auth behavior intact.

## Reproduction Assessment

Yes. Source inspection of current main shows plugin tools are dispatched with `toolCallId`, params, `signal`, and `onUpdate`, while the factory context already carries identity fields and no execute-time context type or wrapper exists.

## Solution Assessment

No, not as currently proposed. The execute-wrapper direction is narrow and compatible, but this branch needs a current-main refresh, cached-path coverage, public SDK/docs/changelog updates, and removal or repair of the stale compaction auth hunk.
