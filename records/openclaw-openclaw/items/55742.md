## Summary

Keep open: current main still lacks execute-time plugin context, but this PR is stale/conflicting, misses current cached/live plugin tool paths and public SDK alignment, carries a compaction auth regression, and lacks real behavior proof.

## What This Changes

The PR adds an identity/session execute context for plugin tool execution, wraps resolved plugin tools to append it, adds tests, and updates an adjacent compaction auth caller and mocks.

## Best Possible Solution

Ship a refreshed additive plugin-tool execute-context contract that covers cached and live tool execution, exports and documents the fifth-argument SDK surface, and drops or fixes the stale compaction auth hunk.

## Reproduction Assessment

Yes, by source inspection. Current main passes identity into plugin tool factories, but cached and live execute paths call tools with only toolCallId, params, signal, and onUpdate.

## Solution Assessment

No, not as currently submitted. The wrapper direction is plausible, but the branch needs current-main path coverage, public SDK/docs alignment, and removal or repair of the stale compaction auth change.
