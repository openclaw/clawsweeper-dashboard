## Summary

Keep open: the reported execute-time agent identity gap is real by source inspection, but this external PR is not mergeable as submitted because it targets an older plugin resolver shape, misses the public SDK contract, carries a stale compaction auth hunk, and still lacks real behavior proof.

## What This Changes

The PR adds an execute-time plugin tool identity context wrapper and type, tests for context injection, and a compaction-safeguard auth API update.

## Best Possible Solution

Ship a rebased additive plugin SDK/runtime contract that appends a frozen identity context in both cached descriptor and live tool execution paths, exports and documents the public type, and preserves current compaction auth header behavior.

## Reproduction Assessment

Yes, by source inspection. Current main passes identity into plugin tool factories, but cached descriptor and live execution paths still call plugin execute with only toolCallId, params, signal, and onUpdate.

## Solution Assessment

No, not as submitted. The wrapper direction is plausible, but it must be rebased to the current resolver paths, exported through the supported SDK, and stripped of the stale compaction auth regression.
