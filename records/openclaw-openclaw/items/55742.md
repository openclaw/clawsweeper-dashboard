## Summary

Keep open. The underlying plugin execute-context gap still exists on current main, but this branch is conflicted, targets stale plugin/compaction surfaces, and needs public SDK/docs coverage plus real behavior proof before merge.

## What This Changes

The PR wraps plugin tool execution to append an identity/session context, adds an internal execute-context type and tests, and updates an adjacent compaction auth caller and mocks.

## Best Possible Solution

Ship a refreshed additive plugin-tool execute-context contract on current main that appends a frozen identity-only context after signal/onUpdate, covers live and cached plugin tool execution, and documents/exports the contract through the public SDK.

## Reproduction Assessment

Yes. Source inspection on current main shows plugin tool factories receive identity fields, but both cached and live tool execution paths still call execute with only toolCallId, params, signal, and onUpdate.

## Solution Assessment

No, not as currently submitted. The execute-wrapper direction is plausible, but the branch needs a current-main refresh, cached-path coverage, SDK/docs alignment, and removal or repair of the stale compaction auth hunk.
