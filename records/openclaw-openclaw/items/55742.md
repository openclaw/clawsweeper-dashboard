## Summary

Keep open. Current main still lacks execute-time plugin tool identity context, so this PR addresses a real gap, but the branch is stale and the proposed compaction auth hunk would regress header-based credentials that current main now preserves.

## What This Changes

This PR wraps resolved plugin tools so `execute` receives a frozen identity/session context as a fifth argument, adds the corresponding type and tests, and updates an adjacent compaction auth caller and mocks.

## Best Possible Solution

Ship a refreshed, documented plugin-tool execute-context contract on current main that appends a frozen identity-only context after existing dispatcher arguments and preserves current compaction auth behavior.

## Reproduction Assessment

Yes. Static inspection is enough: plain registered plugin tools can be reused without closing over request context, `resolvePluginTools` returns them unwrapped, and the dispatcher never passes a fifth identity/session context.

## Solution Assessment

No, not as currently proposed. The execute-wrapper direction is narrow and plausible, but the branch is stale, drops resolved auth headers, and misses the public SDK/docs/changelog follow-through needed for a plugin contract change.
