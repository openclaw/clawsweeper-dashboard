## Summary

Keep open, but the patch is not merge-ready as submitted. The requested execute-time agent/session context is a useful plugin SDK/runtime feature and the gap is source-reproducible on current main, but this branch is stale against the current cached descriptor/live tool paths, misses the public SDK surface, carries a compaction auth regression, is merge-conflicting, and lacks real behavior proof.

## What This Changes

The PR adds a plugin tool execute context type, wraps resolved plugin tools to append agent/session identity to execute calls, adds tests, and updates compaction-safeguard auth API mocks/usage.

## Best Possible Solution

Ship a rebased additive SDK/runtime contract that appends a frozen identity context in both cached descriptor and live plugin tool execution paths, exports and documents the public type, and preserves current compaction auth/header behavior.

## Reproduction Assessment

Yes, by source inspection. Current main passes identity into plugin tool factories, but cached descriptor and live execution paths still call plugin execute with only toolCallId, params, signal, and onUpdate.

## Solution Assessment

No, not as submitted. The wrapper direction is plausible, but it must be rebased to cover current cached/live tool execution, expose the SDK contract, and drop or repair the stale compaction auth hunk.
