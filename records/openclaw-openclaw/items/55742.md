## Summary

Keep open. Current main still lacks execute-time plugin tool identity context, so this PR addresses a real remaining gap; however, the branch is not correct as-is because the adjacent compaction auth hunk drops header-based credentials and the branch targets stale paths. Related PR #56036 covers hook dispatch context, not this execute-time plugin tool contract.

## What This Changes

The PR wraps resolved plugin tools so `execute` receives a frozen identity/session context as a fifth argument, adds the corresponding SDK type and tests, and updates an adjacent compaction auth caller and mocks.

## Best Possible Solution

Ship a rebased, documented plugin-tool execute-context contract on current main that appends a frozen identity-only context after the existing dispatcher arguments, with tests for context injection, signal/onUpdate preservation, own-property preservation, and config non-leakage while preserving current compaction auth header behavior.

## Reproduction Assessment

Yes. Static inspection is enough: plain registered plugin tools can be reused without closing over the request context, `resolvePluginTools` returns them unwrapped, and the dispatcher never passes a fifth identity/session context.

## Solution Assessment

No, not as currently proposed. The execute-wrapper direction is narrow and plausible, but the branch includes a stale compaction auth hunk that regresses header credentials; the safer path is a current-main rebase that preserves existing auth behavior or removes that unrelated change.
