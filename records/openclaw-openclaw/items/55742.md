## Summary

Keep open: current main still does not provide an execute-time plugin tool identity context, and related #56036 covers hook dispatch rather than this execute contract. The PR is useful source work, but it is not mergeable as-is because it targets stale paths, risks regressing compaction header auth, and misses public SDK/docs/changelog follow-through.

## What This Changes

The PR branch wraps plugin tool execution to append an identity/session context, adds a new type and tests, and updates an adjacent compaction auth caller and mocks.

## Best Possible Solution

Ship an additive, documented plugin-tool execute-context contract on current main that appends a frozen identity-only context after existing `signal` and `onUpdate` arguments, covers cached and live plugin tool execution, and preserves current compaction auth behavior.

## Reproduction Assessment

Yes. Static inspection of current main is enough: registered plugin tools are returned or proxied with only `toolCallId`, params, `signal`, and `onUpdate`, while the factory context already contains identity fields.

## Solution Assessment

No, not as currently proposed. The execute-wrapper direction is narrow and plausible, but this branch needs a current-main refresh, complete SDK/docs/changelog coverage, and removal or repair of the stale compaction auth hunk before merge.
