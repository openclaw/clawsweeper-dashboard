## Best Possible Solution

Rebase onto current main, append a frozen identity context in both cached descriptor and live plugin tool execute paths, expose and document the supported SDK type, preserve compaction auth headers or drop that unrelated hunk, and include redacted real setup proof.

## Solution Assessment

No, not as submitted. The wrapper direction is plausible, but it must target current resolver paths, be promoted through the public plugin SDK/docs, and avoid the stale compaction auth regression.
