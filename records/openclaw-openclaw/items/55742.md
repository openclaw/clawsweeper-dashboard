## Summary

Keep open: the execute-context direction addresses a real current-main gap, but this branch is stale and incomplete because it wraps only an older live tool path, misses the public SDK/docs contract, carries an obsolete compaction auth hunk, and lacks real behavior proof.

## What This Changes

The PR adds execute-time identity/session context for plugin tool calls, tests for context injection, and a compaction safeguard auth API update.

## Best Possible Solution

Ship a rebased additive SDK/runtime contract that appends a frozen identity context in both cached descriptor and live plugin tool execution paths, exports and documents the type, and preserves current compaction auth/header behavior.

## Reproduction Assessment

Yes, by source inspection. Current main passes identity into plugin tool factories, but cached descriptor and live tool execution paths still call plugin execute with only toolCallId, params, signal, and onUpdate.

## Solution Assessment

No, not as submitted. The wrapper direction is plausible, but the branch needs current-main path coverage, public SDK/docs alignment, real behavior proof, and removal or repair of the stale compaction auth hunk.
