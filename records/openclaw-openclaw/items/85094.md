## Summary

Keep open: current main still routes Linux service status through the user-systemd unit only, while the docs support system-scoped gateway units, so a local gateway can be reachable while the service rows say the user service is not installed.

## What This Changes

Report `openclaw status --deep` service rows correctly when the gateway is running under a system-scoped systemd unit instead of a user-scoped unit.

## Best Possible Solution

Teach Linux status to recognize an active system-scoped OpenClaw gateway as externally managed, surface the scope clearly, and keep user-systemd lifecycle commands as the managed default.

## Reproduction Assessment

Yes by source inspection, though not by running a live systemd-system fixture here. On Linux, current main reads and probes only the user unit, then formats a missing user unit as not installed even when the gateway probe can be reachable.

## Solution Assessment

Yes, a narrow status-only system-scope fallback or reachable-local-gateway external-management path is the maintainable fix. Adding a `--system` flag alone would leave the default diagnostic misleading for a documented deployment mode.

## Repair Work Prompt

Fix the Linux status false positive from https://github.com/openclaw/openclaw/issues/85094. On Linux, `openclaw status --deep` and the overview rows must not report the Gateway service as `systemd user not installed` when a local gateway is reachable and/or a matching system-scoped OpenClaw systemd unit is active. Keep OpenClaw-managed install/start/restart behavior user-systemd by default; add a status/diagnostic-only path that detects system-scope ownership as externally managed and surfaces the scope clearly. Be conservative with custom unit names such as `openclaw.service`: do not require sudo or broad-scan unrelated services, and prefer an externally-managed/unknown-scope status over a false not-installed warning when the local gateway probe is green. Add focused Vitest coverage for user-unit present, system-unit active with no user unit, neither unit present, and permission/error cases. Add a changelog entry if user-visible status wording changes.
