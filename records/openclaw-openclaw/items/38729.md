## Summary

Keep open: the PR still targets the deprecated `supportsXHighThinking` hook, while current main uses Anthropic's `resolveThinkingProfile` as the source of truth. Current docs and tests keep Claude 4.6 adaptive-only and reserve `xhigh`/`max` advertisement for Claude Opus 4.7.

## What This Changes

## Best Possible Solution

## Reproduction Assessment

Yes. A high-confidence static path exists: current main returns Anthropic's non-empty `resolveClaudeThinkingProfile` before the legacy `supportsXHighThinking` hook, and the Anthropic regression test asserts Claude Opus 4.6 excludes `xhigh`/`max`.

## Solution Assessment

No. Adding `supportsXHighThinking` is not the best fix because Anthropic already uses the profile-driven path; the maintainable fix, if approved, is to update `resolveClaudeThinkingProfile` and matching tests/docs.
