The PR adds an Anthropic `supportsXHighThinking` callback for Claude Opus 4.6 model IDs in `extensions/anthropic/register.runtime.ts`.

## Best Possible Solution

Keep Claude thinking support defined through `resolveClaudeThinkingProfile`; if maintainers approve Opus 4.6 max effort as user-facing, update that profile plus Anthropic/provider-policy tests and docs.

## Reproduction Assessment

Yes. Source inspection against current main shows Anthropic returns a non-empty `resolveClaudeThinkingProfile` before the legacy xhigh hook is consulted, and current tests assert Opus 4.6 excludes `xhigh`/`max`.

## Solution Assessment

No. Adding `supportsXHighThinking` is not the maintainable fix because Anthropic already uses the profile-driven path; the safer implementation belongs in `resolveClaudeThinkingProfile` with matching tests/docs after maintainer approval.
