## Summary

Keep this PR open but do not merge the patch as-is. The proposed Anthropic `supportsXHighThinking` hook is a legacy capability hook, while current `main` resolves Anthropic thinking through `resolveThinkingProfile` first; current tests and docs make Claude Opus 4.6 adaptive-only in the advertised profile even though the transport can map an internal `xhigh` value to Anthropic `max` effort.

## What This Changes

The branch adds an Anthropic provider `supportsXHighThinking` callback that returns true for Claude Opus 4.6 model IDs.

## Best Possible Solution

Keep Claude thinking support profile-driven. If maintainers approve user-facing Opus 4.6 max effort, update `resolveClaudeThinkingProfile` with matching Anthropic tests and thinking docs; otherwise close or revise this PR with Opus 4.6 remaining adaptive-only in the advertised profile.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: resolve Anthropic thinking levels for `claude-opus-4-6` on current `main` and observe that `resolveThinkingProfile` returns the Anthropic profile before `supportsXHighThinking` is consulted; the Anthropic test also asserts that profile lacks `xhigh` and `max`.

## Solution Assessment

No. Adding `supportsXHighThinking` is not the best fix because that hook is a legacy compatibility path and is bypassed by Anthropic's non-empty provider profile; the maintainable fix, if approved, is to update the shared Claude profile source of truth and its tests/docs.
