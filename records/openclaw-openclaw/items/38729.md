## Summary

Keep open. The requested Anthropic Opus 4.6 capability is not implemented on current main, and this PR’s hook is stale because Anthropic now resolves thinking through provider profiles before legacy xhigh hooks are consulted.

## What This Changes

The PR adds a `supportsXHighThinking` callback to `extensions/anthropic/register.runtime.ts` for Claude Opus 4.6 model IDs.

## Best Possible Solution

If maintainers want Opus 4.6 max effort to be user-facing, update `resolveClaudeThinkingProfile` with Anthropic tests/docs and require redacted runtime proof; otherwise close with the rationale that Opus 4.6 remains adaptive-only in the advertised profile.

## Reproduction Assessment

Yes for the blocking PR finding: source inspection shows Anthropic’s non-empty provider profile returns before the legacy `supportsXHighThinking` hook. The requested Opus 4.6 `xhigh` advertisement itself is a capability decision rather than a strict bug reproduction.

## Solution Assessment

No. Adding a deprecated compatibility hook is not the maintainable path on current main; the implementation, if approved, belongs in `resolveClaudeThinkingProfile` with matching tests/docs and proof.
