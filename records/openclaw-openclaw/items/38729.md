## Summary

Keep this PR open, but it is not mergeable as-is: current main resolves Anthropic thinking through provider profiles, so the added legacy hook is bypassed and current tests deliberately keep Claude Opus 4.6 adaptive-only.

## What This Changes

The PR adds an Anthropic `supportsXHighThinking` callback for Claude Opus 4.6 model IDs in `extensions/anthropic/register.runtime.ts`.

## Best Possible Solution

Keep Claude model levels defined in `resolveClaudeThinkingProfile`; if maintainers approve Opus 4.6 max effort, update that profile plus Anthropic tests/docs and require runtime proof.

Yes for the blocking PR finding: source inspection shows Anthropic's non-empty provider profile returns before legacy `supportsXHighThinking` is consulted. The requested Opus 4.6 `xhigh` capability itself is a feature/capability decision rather than a strict bug reproduction.

## Solution Assessment

No. Adding a deprecated compatibility hook is not the maintainable path on current main; the approved implementation, if any, belongs in `resolveClaudeThinkingProfile` with matching tests/docs and proof.
