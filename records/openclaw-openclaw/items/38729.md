## Summary

Keep open. Current main does not implement the proposed user-facing Anthropic xhigh capability, and this patch is not mergeable as-is because Anthropic already supplies a provider thinking profile that takes precedence over the legacy xhigh hook. Current docs and regression tests keep Claude Opus/Sonnet 4.6 adaptive-only, so exposing xhigh/max needs an explicit provider capability decision and a profile-source update.

## What This Changes

The branch adds an Anthropic provider `supportsXHighThinking` callback that returns true for Claude Opus 4.6 model-id prefixes.

## Best Possible Solution

Keep Claude thinking support defined through `resolveClaudeThinkingProfile`. If maintainers approve Opus 4.6 xhigh/max as user-facing, update that shared profile plus Anthropic/profile tests and thinking docs; otherwise revise or close this PR with the adaptive-only policy preserved.

## Reproduction Assessment

Yes. A static reproduction path is high-confidence: current main returns Anthropic's provider profile before checking legacy xhigh hooks, and the Anthropic regression test locks Opus 4.6 as adaptive-only without `xhigh` or `max`.

## Solution Assessment

No. Adding `supportsXHighThinking` is not the best way to solve this because Anthropic already uses `resolveThinkingProfile`; the narrow maintainable fix, if approved, is to update `resolveClaudeThinkingProfile` and its tests/docs.
