## What This Changes

The PR adds a `supportsXHighThinking` callback to the Anthropic provider registration for Claude Opus 4.6 model IDs.

## Best Possible Solution

Keep Claude thinking support defined through `resolveClaudeThinkingProfile`; if Opus 4.6 max effort is approved, update that profile with tests, docs, and changelog instead of adding the legacy hook.

## Reproduction Assessment

Yes. A high-confidence static path exists: Anthropic returns a non-empty provider profile before the legacy `supportsXHighThinking` hook is consulted, and the current Opus 4.6 profile/test excludes `xhigh` and `max`.

## Solution Assessment

No. Adding `supportsXHighThinking` is not the best fix because Anthropic is already on the profile-driven path; the maintainable solution is a profile/test/docs update if maintainers approve the capability.
