## Summary

Keep open: the requested user-facing capability is not implemented on current main, and this PR's patch is not correct because Anthropic's non-empty `resolveThinkingProfile` path wins before the legacy `supportsXHighThinking` hook can add `xhigh`. Current tests and docs still define Claude Opus/Sonnet 4.6 as adaptive-only for advertised levels.

## What This Changes

The PR adds an Anthropic provider `supportsXHighThinking` callback for Claude Opus 4.6 model ID prefixes.

## Best Possible Solution

Keep Claude thinking support profile-driven; if maintainers approve Opus 4.6 max effort as user-facing, update `resolveClaudeThinkingProfile` plus Anthropic/provider-policy tests and thinking docs, otherwise revise or close this PR as adaptive-only.

## Reproduction Assessment

Yes. Source inspection is enough: current main returns Anthropic's non-empty provider profile for `claude-opus-4-6` before the legacy `supportsXHighThinking` hook is consulted, and the Anthropic tests assert that profile excludes `xhigh`/`max`.

## Solution Assessment

No. Adding the deprecated compatibility hook is not the best fix because Anthropic already uses the profile-driven path; the maintainable implementation, if approved, belongs in `resolveClaudeThinkingProfile` with matching tests/docs.
