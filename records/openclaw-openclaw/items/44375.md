## Summary

Keep open. Current main still makes the first configured `agents.list` entry the default when no entry is marked `default: true`, and the raw `openclaw config set agents.list ...` path still only prints generic update output. The related fallback-routing privacy issue at https://github.com/openclaw/openclaw/issues/60712 is adjacent, but it does not replace this first-list default-displacement problem.

## What This Changes

The issue asks for a warning or routing-contract change so adding a non-main ACP agent does not silently replace the implicit `main` fallback for unbound traffic.

## Best Possible Solution

Add a non-breaking config-set and/or doctor/startup warning when a first `agents.list` assignment excludes `main` and has no explicit default while `main` state exists, and reserve any fallback contract change for an explicit maintainer product decision.

## Reproduction Assessment

Yes, at source level: set `agents.list` to one non-main ACP entry without `default: true`, and `resolveDefaultAgentId` plus `resolveAgentRoute` makes that entry the default for unbound traffic. I did not run a live Telegram/ACP gateway.

## Solution Assessment

Unclear. A warning or doctor diagnostic is the narrowest compatible direction, while auto-including `main` or requiring explicit defaults would change documented routing behavior.
