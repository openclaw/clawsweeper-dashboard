## Summary

Keep open. Current main still makes a non-empty `agents.list` default to `default: true` or the first list entry, raw `openclaw config set agents.list ...` still emits only generic restart guidance, and the related routing/privacy reports are adjacent rather than a canonical replacement for this default-displacement path.

## What This Changes

The issue asks OpenClaw to prevent or warn about a single ACP agent in `agents.list` becoming the default fallback for all unbound traffic instead of the implicit `main` agent.

## Best Possible Solution

Add a non-breaking `config set`, doctor, or startup diagnostic when `agents.list` is first populated without `main` or an explicit default while `main` state exists; reserve changes to first-entry fallback semantics for maintainer product review.

## Reproduction Assessment

Yes, at source level: set `agents.list` to one non-main ACP entry without `default: true`; `resolveDefaultAgentId` picks that entry and `resolveAgentRoute` uses it after binding misses. I did not run a live Telegram/ACP gateway.

## Solution Assessment

Unclear. A warning or doctor diagnostic is the narrowest compatible direction, while auto-including `main` or requiring explicit defaults would change documented routing behavior.
