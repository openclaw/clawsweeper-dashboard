## Summary

Keep open. Current main still makes the first `agents.list` entry the default when no entry has `default: true`, and neither the CLI/RPC config write path nor validation adds the displacement warning requested here; the remaining decision is the safest UX/routing contract, not an already-implemented fix.

## What This Changes

Request a warning or guard so adding one ACP-backed agent to `agents.list` for a specific binding does not silently move default routing away from implicit `main`.

## Best Possible Solution

Pick an explicit default-agent safety contract, likely warning or doctor guidance when `agents.list` is first introduced without `default: true` while `main` state exists, then align config write UX, docs, and focused resolver tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a config with a single non-`main` `agents.list` entry and no `default: true` makes `resolveDefaultAgentId` return that entry, and existing tests assert the same first-entry session fallback.

## Solution Assessment

Unclear. A warning or doctor diagnostic is the narrowest low-risk path, but requiring explicit defaults or auto-adding `main` changes documented routing semantics and needs maintainer product judgment.
