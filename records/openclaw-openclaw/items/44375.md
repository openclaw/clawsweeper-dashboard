- `impact:session-state`: The reported path changes session keys and default workspace ownership from `main` to the first configured agent.

## Summary

Keep open: current main still makes the first `agents.list` entry the default whenever no entry has `default: true`, so the reported implicit-main displacement remains real and compatibility-sensitive.

## What This Changes

The issue asks for a warning, guard, or default-preservation behavior when adding an ACP agent to `agents.list` would silently displace implicit `main` routing.

## Best Possible Solution

Add a maintainer-approved safety path around first-time or non-default `agents.list` writes so existing implicit `main` state is not silently displaced, while preserving documented explicit-default behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows that a populated `agents.list` with no explicit default makes the first list entry the default for unmatched routes and bare session keys; I did not run a live gateway reproduction in this read-only review.

## Solution Assessment

Unclear as a final policy. Warning or blocking unsafe raw `agents.list` writes looks safer than silently changing runtime fallback, but maintainers need to choose the compatibility behavior.
