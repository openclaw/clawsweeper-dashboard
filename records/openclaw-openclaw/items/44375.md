## Summary

Keep open. Current main still makes a non-empty `agents.list` default to `default: true` or the first entry, raw `openclaw config set agents.list ...` still prints only generic restart guidance, and the open related routing/privacy issues do not supersede this first-list default-displacement path.

## What This Changes

The issue asks OpenClaw to avoid or warn about silently replacing implicit `main` routing when the first `agents.list` entry is a non-main ACP agent.

## Best Possible Solution

Add a non-breaking `config set`, doctor, or startup diagnostic when `agents.list` is first populated without `main` or an explicit default while `main` state exists; reserve routing fallback changes for maintainer product review.

## Reproduction Assessment

Yes, at source level: set `agents.list` to one non-main ACP entry without `default: true`; `resolveDefaultAgentId` picks that entry and `resolveAgentRoute` uses it after binding misses. I did not run a live Telegram/ACP gateway.

## Solution Assessment

Unclear. A warning or doctor diagnostic is the narrowest compatible direction, while auto-including `main` or requiring explicit defaults would change documented routing behavior.
