- `impact:session-state`: The reported path changes session keys and default workspace ownership from implicit `main` to the first configured agent.

## Summary

Keep open: current main and the latest release still make the first `agents.list` entry the default when no entry has `default: true`, so adding a single ACP agent can still reroute unmatched traffic away from implicit `main`. The issue is compatibility- and security-sensitive, and the open related PR only covers the narrower explicit-`main` binding problem rather than this default-hijack policy.

## What This Changes

The issue asks OpenClaw to prevent a first ACP entry in `agents.list` from silently becoming the default for all unmatched routing and orphaning implicit `main` state.

## Best Possible Solution

Add a maintainer-approved safety path around first-time or non-default `agents.list` writes so existing implicit `main` state is not silently displaced, while preserving documented explicit-default behavior or migrating it deliberately.

## Reproduction Assessment

Yes. Source inspection shows a non-empty `agents.list` with no explicit default makes the first list entry the default for unmatched routes and bare session keys; I did not run a live gateway reproduction in this read-only review.

## Solution Assessment

Unclear as final policy. Warning or blocking unsafe raw `agents.list` writes looks safer than silently changing runtime fallback, but maintainers need to choose the compatibility behavior.
