## Summary

Keep open. Current main and v2026.5.7 still choose the first `agents.list` entry as the default when no entry has `default: true`; the remaining question is a config-safety/product decision, not an already-fixed bug.

## What This Changes

Request a warning or changed default-agent behavior when adding an ACP/named agent would displace the implicit `main` agent for unbound traffic.

## Best Possible Solution

Define the default-agent safety contract explicitly, then add a high-signal `config set`/startup/doctor warning or a deliberate default requirement without weakening explicit bindings.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: configure `agents.list` with only `codex-cubby`, omit `default: true`, send traffic with no matching binding, and `resolveAgentRoute` will select `codex-cubby` through `resolveDefaultAgentId`.

## Solution Assessment

Unclear. A warning or doctor diagnostic is safer than silently auto-injecting `main`, but maintainers need to choose whether the documented first-entry default remains the contract.
