## Summary

Keep this open. Current main still preserves the documented first-entry default-agent fallback, and `openclaw config set agents.list ...` still writes the config with only the generic restart message, so the silent implicit-main displacement remains a valid config UX gap; the related #60712 is broader fallback/privacy policy work and does not supersede this narrower warning request.

## What This Changes

The issue requests a warning or guard when adding an ACP agent to `agents.list` would make it the default for unbound traffic instead of the implicit `main` agent.

## Best Possible Solution

Add a conservative config mutation warning for non-empty `agents.list` values that lack both `default: true` and an explicit `main` entry, while preserving the documented first-entry fallback.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `resolveDefaultAgentId({ agents: { list: [{ id: "codex-cubby" }] } })` resolves to `codex-cubby`, unmatched routing uses that default, and the config writer has no special warning for this mutation.

## Solution Assessment

Yes. A warning is the narrowest maintainable fix because it addresses the silent UX hazard without auto-inserting `main` or requiring explicit defaults, either of which would change the documented routing contract.
