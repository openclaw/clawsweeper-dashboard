## Summary

Keep open. Current main still implements and documents first-entry default-agent selection, while raw config writes still do not warn when a new non-main ACP agent displaces the implicit main agent.

## What This Changes

Request a warning or safer default handling when adding an ACP-backed agent to `agents.list` would silently replace the implicit `main` default route.

## Best Possible Solution

Add a non-breaking config-set and doctor warning when a first `agents.list` assignment would displace an on-disk implicit `main` agent, and leave fallback semantics for a deliberate contract change.

## Reproduction Assessment

Yes, at source level: set `agents.list` to one non-main ACP entry without `default: true`, and current `resolveDefaultAgentId` plus `resolveAgentRoute` makes it the default for unbound traffic. I did not run a live Telegram/ACP gateway.

## Solution Assessment

Unclear. A warning or doctor diagnostic is the narrowest safe direction, while auto-including `main` or requiring explicit defaults would change documented routing compatibility.
