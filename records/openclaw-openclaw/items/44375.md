## Summary

Keep open. Current main still makes a non-empty `agents.list` default to `default: true` or the first configured entry, raw `openclaw config set agents.list ...` still only prints generic restart guidance, and the related open privacy issue covers stale binding fallback rather than this default-displacement path.

## What This Changes

The issue requests a warning or safer default behavior when a newly populated `agents.list` would displace the implicit `main` agent and route unbound traffic to a new ACP agent.

## Best Possible Solution

Add a non-breaking `config set`, startup, or doctor diagnostic when `agents.list` is first populated without `main` or an explicit default while `main` state exists; reserve fallback semantic changes for maintainer product review.

## Reproduction Assessment

Yes, at source level: set `agents.list` to one non-main ACP entry without `default: true`; `resolveDefaultAgentId` picks that entry and `resolveAgentRoute` uses it after binding misses. I did not run a live Telegram/ACP gateway.

## Solution Assessment

Unclear until maintainers choose the product contract. A warning or doctor diagnostic is the narrowest compatible direction, while auto-including `main` or requiring explicit defaults would change documented routing behavior.
