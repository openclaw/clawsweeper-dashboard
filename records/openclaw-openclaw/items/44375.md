## Summary

Keep this open. Current main still preserves the documented first-entry default-agent fallback, and `openclaw config set agents.list ...` still writes the config with only the generic restart message, so the reported silent implicit-main displacement remains a valid config UX gap; the related #60712 is broader fallback/privacy policy work and does not supersede this narrower warning request.

## What This Changes

The issue requests a warning or guard when adding an ACP agent to `agents.list` would move unbound traffic away from the implicit `main` agent.

## Best Possible Solution

Add a conservative `config set`/patch warning for non-empty `agents.list` mutations that lack both `default: true` and `id: "main"`, while preserving the documented first-entry fallback contract.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `resolveDefaultAgentId({ agents: { list: [{ id: "codex-cubby" }] } })` resolves to `codex-cubby`, unmatched routing uses that default, and the config writer has no special warning for this mutation.

## Solution Assessment

Yes. A warning/guard is the narrowest maintainable solution because it addresses the silent UX hazard without auto-migrating `main` or requiring explicit defaults, both of which would change the documented routing contract.
