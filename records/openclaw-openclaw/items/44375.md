## Summary

Keep open. Current main still implements and documents first-entry default-agent selection, while raw `config set agents.list` still has no warning for displacing implicit `main`; the fix needs maintainer choice because changing fallback semantics would alter a documented compatibility contract.

## What This Changes

The issue asks OpenClaw to warn or preserve `main` when adding a non-main ACP agent to `agents.list` would otherwise make that new agent the default route for unbound traffic.

## Best Possible Solution

Add a non-breaking config-set and doctor warning when a first `agents.list` assignment would displace an on-disk implicit `main` agent; reserve runtime fallback changes for a deliberate contract update.

## Reproduction Assessment

Yes, at source level: set `agents.list` to one non-main ACP entry without `default: true`, and current `resolveDefaultAgentId` plus `resolveAgentRoute` will make it the default for unbound traffic. I did not run a live Telegram/ACP gateway.

## Solution Assessment

Unclear. A warning or doctor diagnostic is the narrowest safe direction, while auto-including `main` or requiring explicit defaults would change documented routing compatibility and needs maintainer approval.
