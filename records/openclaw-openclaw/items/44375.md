## Summary

Keep open: current main still implements the reported fallback behavior and does not surface a config-set warning when a newly populated `agents.list` without `main` or `default: true` changes unmatched routing away from the implicit main agent. The issue is narrow, reproducible from source/docs/tests, and not resolved by the related broader fallback-routing discussion.

## What This Changes

Adding a single ACP-backed agent to `agents.list` without an explicit `main` or `default: true` can change all unmatched routing from the implicit main agent to the first configured agent without warning.

## Best Possible Solution

Add a conservative CLI/config UX guard that warns when `agents.list` becomes non-empty without `default: true` and without `id: "main"`, especially from an implicit-main setup, while leaving the documented first-entry fallback and session-key behavior intact unless maintainers separately choose a breaking routing-policy change.
