## Summary

Keep open. Current main still implements and documents first-entry default-agent selection, so the report is a real routing foot-gun, but the fix requires maintainer choice about warning versus changing a documented compatibility contract.

## What This Changes

Request a warning or guard so adding an ACP agent to `agents.list` does not silently displace implicit `main` as the default for unbound traffic.

## Best Possible Solution

Add a maintainer-approved, non-breaking config/doctor warning first, and only change runtime fallback semantics if the multi-agent default contract is deliberately revised with docs and tests.

## Reproduction Assessment

Yes, at source level: set `agents.list` to a single `codex-cubby` entry without `default: true`, and `resolveDefaultAgentId` plus `resolveAgentRoute` will make it the default for unbound traffic. I did not run a live Telegram/ACP gateway.

## Solution Assessment

Unclear. A warning is the safest narrow direction, but auto-including `main` or requiring explicit defaults would change documented routing compatibility and needs maintainer product judgment.
