## Summary

Keep open. Current main still makes the first configured `agents.list` entry the default when no entry has `default: true`, raw `openclaw config set agents.list ...` still emits only generic update/restart output, and the adjacent fallback-routing privacy issue does not cover this first-list default-displacement path.

## What This Changes

The issue asks for a warning or safer default behavior when adding a non-main ACP agent to `agents.list` would displace the implicit `main` default agent.

## Best Possible Solution

Add a non-breaking config-set and/or doctor/startup warning when a first `agents.list` assignment excludes `main` and has no explicit default while `main` state exists; reserve fallback contract changes for maintainer product review.

## Reproduction Assessment

Yes, at source level: set `agents.list` to one non-main ACP entry without `default: true`; `resolveDefaultAgentId` picks that entry and `resolveAgentRoute` uses it after binding misses. I did not run a live Telegram/ACP gateway.

## Solution Assessment

Unclear. A warning or doctor diagnostic is the narrowest compatible direction, while auto-including `main` or requiring explicit defaults would change documented routing behavior.
