## Summary

Keep open. Current main still has the Discord outbound mention reliability gap: normal, webhook, and structured sends call the known-mention rewriter, but that rewriter only consults an in-memory account-scoped directory cache and intentionally preserves unknown handles as plaintext. I did not find a config-backed alias-to-userId map or inbound-message seeding path that would make known teammate mentions deterministic.

## What This Changes

Request deterministic Discord outbound mention resolution so configured or otherwise known `@AgentName` references are sent as real Discord `<@userId>` mentions instead of plaintext when the transient cache is missing.

## Best Possible Solution

Implement this in the Discord plugin as an explicit, stable alias-to-userId mapping that resolves before the transient directory cache, scoped by account and preferably guild. Keep unknown handles, `@everyone`, `@here`, and markdown code spans/fences untouched, then document the config and cover normal, webhook, structured, and multi-account sends.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction exists: with no `rememberDiscordDirectoryUser` entry for an account, `rewriteDiscordKnownMentions("@Vladislava", { accountId })` preserves the plaintext handle, and all outbound send paths depend on that same rewriter.

## Solution Assessment

Yes for the requested direction. A plugin-local config-backed alias map is the narrowest maintainable fix because it provides deterministic userId mapping without moving Discord-specific policy into core or changing unknown-handle behavior.
