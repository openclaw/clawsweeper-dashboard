## Summary

Keep open. Current main still has the Discord outbound mention gap: sends call the known-mention rewriter, but that rewriter only resolves names from the transient account-scoped directory cache and intentionally preserves unknown handles as plaintext. I did not find a config-backed alias map or normal inbound-message cache seeding path in the Discord plugin/config surface.

## What This Changes

The issue asks for Discord outbound messages to deterministically convert known @AgentName text into real Discord <@userId> mentions instead of relying only on a transient cache.

## Best Possible Solution

Implement the fix in the Discord plugin by adding an explicit, stable mention alias map that resolves before the transient directory cache, scoped at least by account and preferably by guild. Keep unknown and reserved mentions untouched, document the config shape, and cover normal sends, webhook sends, structured sends, account isolation, and markdown code spans/fences.

## Reproduction Assessment

Yes. The current-main reproduction is code-level and test-backed: with no prior rememberDiscordDirectoryUser entry, rewriteDiscordKnownMentions("@Vladislava", {accountId}) returns plaintext because resolveDiscordDirectoryUserId has no cache entry, while sends depend on that same rewriter before delivery.

## Solution Assessment

Yes for a fix direction, not for the current implementation. A config-backed alias map in the Discord plugin is the narrowest maintainable repair because it gives operators deterministic userId mapping without broadening core or changing unknown-handle behavior.
