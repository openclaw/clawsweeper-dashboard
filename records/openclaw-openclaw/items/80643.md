## Summary

Keep this PR open: it fixes a real documented `session_status` default-resolution bug and includes sufficient after-fix live output, but the new per-agent lookup uses raw `entry.id` equality and can miss valid agent ids that OpenClaw normalizes elsewhere.

## What This Changes

The PR changes `session_status` to resolve displayed reasoning from session override, per-agent config, global config, then `off`, with focused tests for inherited defaults and explicit session override precedence.

## Best Possible Solution

Reuse the existing normalized agent-config lookup, or normalize both sides, and add a regression case for a normalized agent id before merge.

## Reproduction Assessment

Yes. Source inspection shows current main forces missing `sessionEntry.reasoningLevel` to `off` before the status renderer can apply the documented per-agent and global reasoning defaults.

## Solution Assessment

No, not quite. The approach is the right narrow fix, but the per-agent lookup should follow OpenClaw's normalized agent-id contract instead of raw `entry.id` equality.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Use normalized agent lookup for per-agent defaults:** `src/agents/tools/session-status-tool.ts:85`
  - body: `agentId` is normalized before this helper is called, while configured `agents.list[].id` values are matched elsewhere through `normalizeAgentId`/`resolveAgentConfig`. With a valid config such as `id: "Main"`, this raw equality misses the per-agent `reasoningDefault` and `session_status` falls back to the global default or `off`, even though the existing status path would resolve that agent entry. Please reuse `resolveAgentConfig` or normalize both sides and cover that case.
  - confidence: 0.78
