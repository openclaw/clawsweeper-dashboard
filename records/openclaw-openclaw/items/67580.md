## Summary

Keep open. Current main still has the reported promotion leak shape: inline `status: staged - Candidate:` snippets are not classified as contaminated, rehydration has no managed-dreaming fence guard, and promotion still writes raw snippets into `MEMORY.md`; open PR #68774 remains the active implementation candidate rather than merged proof.

## What This Changes

The issue requests that memory-core dreaming promotion filter or distill raw Candidate metadata before appending to `MEMORY.md` and stop feeding managed dreaming artifacts back into long-term memory.

## Best Possible Solution

Review and land PR #68774 or an equivalent memory-core fix for inline metadata and managed-fence rehydration, then track broader distillation and recall-threshold policy in #67363 or a maintainer-directed follow-up.

## Reproduction Assessment

Yes. A source-level reproduction against current main shows the reported inline metadata prefix is not classified as contaminated, and code inspection confirms raw snippet writing plus missing managed-fence rehydration checks.

## Solution Assessment
