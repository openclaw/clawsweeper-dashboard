## Summary

Close as duplicate/superseded: the reported Docker/Ollama `timeoutSeconds` idle-watchdog failure is the same provider-timeout-to-idle-watchdog root cause already tracked by the open canonical issue and active focused PR.

## What This Changes

The issue reports that `models.providers.<id>.timeoutSeconds` is accepted for slow local providers but the LLM idle watchdog still aborts Ollama requests after the default 120 seconds.

## Best Possible Solution

Keep https://github.com/openclaw/openclaw/issues/77744 and https://github.com/openclaw/openclaw/pull/83979 as the single work path, with this Docker/Ollama host-gateway evidence preserved there if maintainers need it.

## Reproduction Assessment

Yes, at source level: a provider `requestTimeoutMs` above 120 seconds with a non-local-classified Docker host URL is capped to `DEFAULT_LLM_IDLE_TIMEOUT_MS`. I did not run the full Docker/Ollama cold-start replay in this read-only pass.

## Solution Assessment

Yes for cleanup: this should be solved through the canonical timeout issue and active resolver PR, not tracked as a second queueable work item. The fix should honor explicit provider timeouts while preserving the default 120-second guard when no provider timeout is configured.
