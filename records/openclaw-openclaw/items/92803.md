## Best Possible Solution

## Reproduction Assessment

Yes, source-level: current main ranks `adaptive` the same as `medium`, MiniMax M3 exposes only `off` and `adaptive`, and cron uses the shared fallback when a configured default is unsupported. I did not run a live MiniMax cron or gateway request.

## Solution Assessment

Yes for the code shape: changing the shared fallback resolver is the narrow maintainable location and avoids MiniMax-specific drift, while preserving explicit `adaptive` requests. The remaining question is whether maintainers accept `off` as the desired compatibility behavior for non-adaptive requests on adaptive-only profiles.
