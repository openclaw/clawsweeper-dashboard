## Summary

Keep open: the PR is a plausible fix for a real P1 sandbox regression, but it is incomplete and still needs contributor real-behavior proof before merge.

## What This Changes

The PR changes sandbox mount, local FS bridge, config-hash, memory-flush gating, docs, and tests so `workspaceAccess: "none"` is treated as writable for the isolated sandbox workspace.

## Best Possible Solution

Land a rebased complete fix or replacement that makes only `ro` read-only across Docker/browser mounts, local and remote bridges, memory/docs/tests, while preserving host isolation and including redacted sandbox proof.

## Reproduction Assessment

Yes. Source inspection on current `main` shows `none` selects an isolated sandbox workspace, while the Docker/browser mount helper and local/remote bridge write checks still treat only `rw` as writable; https://github.com/openclaw/openclaw/issues/37634 and related reports include mount/log evidence.

## Solution Assessment

No. The direction is the narrow maintainable fix, but this branch misses remote bridge parity, leaves docs inconsistent, is stale against current mount-format code, and lacks required after-fix real behavior proof.
