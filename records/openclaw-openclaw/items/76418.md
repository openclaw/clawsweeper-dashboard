## What This Changes

The PR late-binds bundled web-search owner lookup config in `createWebSearchTool`, adds a focused regression test, and records the fix in `CHANGELOG.md`.

## Best Possible Solution

Close this PR as implemented by the current-main #76449 fix; keep the main-branch implementation and ship it in the next release.

## Reproduction Assessment

Yes, source-reproducible on the PR base and linked report: late-bound `web_search` could pass no config into bundled owner lookup, then route through an active runtime registry that omitted Brave. Current main no longer has that failing config path.

## Solution Assessment

Yes. The current-main solution uses the active resolved secrets/runtime config for both execution and bundled owner lookup, which is a narrow maintainable fix for sub-agent sessions without broadening their active plugin registry.
