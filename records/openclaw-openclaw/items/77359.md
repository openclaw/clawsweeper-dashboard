## Summary

Keep this open: current main still has the shared Discord command-deploy cache behavior, and the linked open PR #77367 uses closing syntax for this issue but has not merged yet.

## What This Changes

The issue asks for Discord slash commands to register independently for every configured bot account in a multi-account Discord setup.

## Best Possible Solution

Scope the Discord command-deploy cache by application ID or account ID and add regression coverage for two Discord clients with identical command specs but distinct applications.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: two Discord accounts with the same native command set and distinct application IDs share one cache file and the same global:reconcile key, so the later account can skip its own REST reconciliation.

## Solution Assessment

Yes. The narrow maintainable fix is to scope the existing deploy cache by Discord application/account while preserving the rate-limit cache; open PR #77367 appears aimed at that exact boundary.
