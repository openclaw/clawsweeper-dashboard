## Summary

Keep open: current main still has the Discord multi-account slash-command deploy-cache bug, and the open closing PR already owns the fix path, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Register Discord native slash commands for every configured Discord application in multi-account setups instead of letting a shared deploy cache skip non-default accounts.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/77367, or an equivalent app/account-scoped deploy-cache fix, then close this issue once the fix is merged and release provenance is clear.

## Reproduction Assessment

Yes. Source inspection shows two Discord applications can share one persisted hash entry such as `global:reconcile`, and the later account can return before its own REST reconciliation; issue comments corroborate the path on v2026.5.7.

## Solution Assessment

Yes. Scoping the cache by Discord application or account while preserving the same-application cache skip is the narrow maintainable fix; disabling the cache would avoid the symptom but remove the intended startup rate-limit protection.
