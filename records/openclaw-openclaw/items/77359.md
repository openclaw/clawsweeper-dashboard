## Summary

Keep open: current main still shares the Discord slash-command deploy cache across Discord applications, and an open linked PR specifically closes this issue but has not merged yet.

## What This Changes

Register Discord native slash commands independently for every configured Discord account/application in multi-bot setups.

## Best Possible Solution

Land a focused Discord plugin fix that scopes the command deploy cache by application or account while preserving the cache's rate-limit protection.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: two Discord accounts with identical command specs share one persisted cache, so the second application can skip reconciliation when `global:reconcile` already has the same hash.

## Solution Assessment

Yes. Scoping the existing cache by Discord application or account is the narrow maintainable fix; disabling the cache would avoid the symptom but lose the rate-limit protection it provides.
