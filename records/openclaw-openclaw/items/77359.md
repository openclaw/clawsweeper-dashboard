## Summary

Keep open: current main still has the source-level bug, and an open closing PR already owns the fix path for this issue.

## What This Changes

The issue requests Discord slash commands to be registered for every configured Discord account in a multi-bot setup, not only the default account.

## Best Possible Solution

Land a focused Discord plugin fix that scopes command-deploy cache entries per Discord application or account while preserving the cache's rate-limit protection and safe concurrent persistence.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: two Discord accounts with identical command sets share one persisted cache and the second can return before its own REST reconciliation when `global:reconcile` already has the same hash.

## Solution Assessment

Yes. Application/account-scoped cache keys are the narrow maintainable solution; disabling the cache would avoid the symptom but remove the intended startup rate-limit protection.
