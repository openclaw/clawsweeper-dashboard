## Summary

Keep open: current main still has the Discord deploy-cache scope bug, and open PR #77367 specifically closes this issue but has not merged yet.

## What This Changes

Fix Discord multi-account slash-command registration so non-default bot accounts register native commands when configured with their own application IDs.

## Best Possible Solution

Review and land PR #77367, or an equivalent focused fix that scopes the persisted command-deploy cache by Discord application or account while preserving restart rate-limit protection.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: two Discord accounts with identical serialized command specs share `discord/command-deploy-cache.json`, so after the first writes `global:reconcile`, the second can skip before listing or creating commands.

## Solution Assessment

Yes. Scoping the existing cache by application or account is the narrow maintainable fix; disabling the cache would avoid the symptom but remove the restart rate-limit protection it provides.
