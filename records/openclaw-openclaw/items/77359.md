## Summary

Keep open: current main and the latest release still share the Discord slash-command deploy cache across applications, and open PR #77367 specifically closes this issue but has not merged.

## What This Changes

The issue reports that Discord slash commands register for the default bot account but are absent for secondary bot accounts in a documented multi-bot setup.

## Best Possible Solution

Land PR #77367 or an equivalent focused fix that scopes the command deploy cache by Discord application or account while preserving the cache's rate-limit protection.

## Reproduction Assessment

Yes. Source inspection on current main shows two Discord accounts with identical command specs share `global:reconcile`, so the second application can return from the cache path before any Discord command reconciliation runs.

## Solution Assessment

Yes. Scoping the existing cache by application ID or account is the narrow maintainable fix; disabling the cache would avoid the symptom but remove the intended rate-limit protection.
