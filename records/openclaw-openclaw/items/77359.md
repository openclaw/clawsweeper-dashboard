## Summary

Keep open: current main still uses a shared Discord slash-command deploy cache keyed only by command scope, and open PR #77367 specifically closes this issue but has not merged yet.

## What This Changes

The issue asks for Discord slash commands to register for every configured bot account in a multi-account setup, not only the default account.

## Best Possible Solution

Land a focused Discord plugin fix that scopes command-deploy cache entries by Discord application or account while preserving the cache's rate-limit protection.

## Reproduction Assessment

Yes. Source inspection on current main shows two Discord accounts with identical command specs can share one persisted `global:reconcile` hash, causing the second account to return before listing or creating commands.

## Solution Assessment

Yes. Scoping the existing cache by Discord application or account, as PR #77367 proposes, is the narrow maintainable fix; disabling the cache would avoid the symptom but lose rate-limit protection.
