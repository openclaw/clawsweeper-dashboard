## Summary

Keep open: current main still shares Discord command-deploy hashes across applications, and #77367 is the unmerged closing PR that targets the narrow fix.

## What This Changes

The issue asks for Discord slash commands to be registered for every configured Discord account, not only the default account, in multi-bot setups.

## Best Possible Solution

Land a focused Discord plugin change that scopes deploy-cache keys by application ID or account while preserving the persisted cache's rate-limit protection.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: two Discord accounts with identical serialized commands share one bare `global:reconcile` hash, so the second application can skip reconciliation without listing or creating its own commands.

## Solution Assessment

Yes. Scoping the existing cache by Discord application or account is the narrow maintainable fix; disabling the cache would avoid the symptom but remove the intended rate-limit protection.
