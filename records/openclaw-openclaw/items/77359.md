## Summary

Keep open: current main still shares the Discord slash-command deploy cache across applications, and the linked open PR #77367 specifically closes this issue but has not merged.

## What This Changes

This issue asks for secondary Discord accounts in a multi-bot setup to receive the same slash-command registration as the default account.

## Best Possible Solution

Land #77367 or an equivalent focused fix that scopes the persisted command-deploy cache by Discord application or account while preserving the cache's rate-limit protection.

## Reproduction Assessment

Yes. Source inspection on current main shows two Discord applications with identical command sets can share the bare `global:reconcile` hash, causing the second deployer to return before listing or creating commands.

## Solution Assessment

Yes. Scoping the existing persisted cache by application or account is the narrow maintainable fix; disabling the cache would avoid the symptom but remove the intended rate-limit protection.
