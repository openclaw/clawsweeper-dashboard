## Summary

Keep open: the report matches current main, and an open closing PR already targets the root cause, so the issue should not be closed before that PR merges.

## What This Changes

Discord multi-account setups should register native slash commands for each configured bot account, not only the default account.

## Best Possible Solution

Land or replace #77367 so Discord command-deploy cache entries are scoped by application or account while preserving the restart cache behavior and adding multi-account regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: two Discord accounts with the same native command set and different application IDs share `command-deploy-cache.json`, so the second account can skip reconciliation after the first writes `global:reconcile`.

## Solution Assessment

Yes. The narrow maintainable fix is to scope the persisted deploy hash by Discord application or account; the linked open PR targets that root cause, so the issue should remain open until it is merged or replaced.
