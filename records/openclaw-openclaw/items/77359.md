## Summary

Keep open: current main still has the Discord multi-account slash-command deploy-cache bug, and an open closing PR already owns the likely fix path.

## What This Changes

The issue reports that secondary Discord bot accounts in a multi-bot setup can handle chat but do not get native slash commands registered.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/77367, or an equivalent app/account-scoped deploy-cache fix, then close this issue once the fix is merged and release provenance is clear.

## Reproduction Assessment

Yes. Source inspection on current main shows two Discord applications can share one persisted key such as `global:reconcile`, letting the later account return before its own REST reconciliation; issue comments add live v2026.5.7 confirmations.

## Solution Assessment

Yes. Scoping the deploy cache by Discord application or account while preserving same-application cache skips is the narrow maintainable fix; disabling the cache would avoid the symptom but remove the intended startup rate-limit protection.
