## Summary

Keep open: current main still has the shared Discord slash-command deploy cache behavior, and open PR #77367 is the active closing fix for this issue.

## What This Changes

Ensure Discord slash commands register independently for every configured Discord bot account in multi-account setups.

## Best Possible Solution

Review and land or revise PR #77367 so the Discord command-deploy cache is scoped by application or account while preserving the existing cache protection and adding two-application regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: two enabled Discord accounts with distinct application IDs and identical command specs share the same persisted `global:reconcile` hash, so the later account can skip REST reconciliation for its own application.

## Solution Assessment

Yes. Scoping the existing deploy cache by application/client ID is the narrowest maintainable fix because it keeps the rate-limit cache while making each Discord application reconcile independently.
