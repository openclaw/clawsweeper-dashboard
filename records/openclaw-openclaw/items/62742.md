## Summary

Keep open. Current main and the latest release still have the unsafe Microsoft Foundry subscription lookup, while this PR is a narrow provider-local guard that appears correct and still useful; no blocking functional or security issue was found in the diff.

## What This Changes

The PR updates `extensions/microsoft-foundry/auth.ts` so the selected subscription lookup is assigned through a checked `match` variable and throws a descriptive error when no subscription matches.

## Best Possible Solution

Land the provider-local guard, preferably with a small regression test that drives `entraIdAuthMethod.run` through an invalid selected subscription id, after confirming the aggregate CI failures are unrelated to this diff.

## Reproduction Assessment

Yes. A focused unit harness can mock the subscription list to contain multiple enabled subscriptions and make `ctx.prompter.select` return an id not in that list; current main then dereferences `undefined.tenantId`. A live Clack reproduction is less direct because the normal CLI selector returns one of the listed option values.

## Solution Assessment

Yes. The PR is the narrowest maintainable fix for the unsafe boundary: it keeps the existing flow and replaces only the non-null assertion with an explicit error. A regression test would improve coverage, but the code direction is sound.
