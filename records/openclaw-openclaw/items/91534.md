## Summary

Current main and the latest stable release already provide the requested non-agent cron command mode, including shell or exact-argv input, working directory, environment, stdin, timeouts, output capture, normal delivery, and failure handling.

## What This Changes

Add a `cron add --command` mode that runs a Gateway subprocess without an agent turn and preserves cron delivery and failure behavior.

## Best Possible Solution

Keep the shipped structured command payload and its exact-argv alternative as the canonical implementation; use `--command` for shell syntax or `--command-argv` when shell parsing is undesirable.

## Reproduction Assessment

No. The reported unrecognized-option behavior no longer reproduces on current main or `v2026.6.5`, where `cron add --command` and the related command controls are implemented.

## Solution Assessment

Yes. The shipped implementation supports the requested shell form while also providing exact argv execution, and it reuses the established cron scheduling, diagnostics, delivery, and failure paths instead of creating a parallel scheduler.
