## Summary

Keep open: current main and the latest release still exact-compare Node's active compile-cache directory against OpenClaw's desired packaged cache directory, while the linked fix PRs are closed unmerged and the raw-root versus scoped-cache contract remains unsettled.

## What This Changes

The issue asks OpenClaw to avoid unnecessary packaged launcher respawns caused by comparing Node's active versioned compile-cache path with OpenClaw's desired cache root.

## Best Possible Solution

Settle the raw-root versus scoped-cache contract, then land matching launcher logic, docs, and regression coverage while preserving package-scoped cache invalidation.

## Reproduction Assessment

Yes, at source level. Current launcher code still has the exact comparison, current docs still recommend raw cache roots, and Node's active cache directory proof shows the versioned child path shape that triggers the mismatch.

## Solution Assessment

Unclear until maintainers choose the contract. The best fix should either make documented raw roots avoid repeated packaged respawns or update docs to require scoped cache paths, with launcher regression coverage for the chosen behavior.
