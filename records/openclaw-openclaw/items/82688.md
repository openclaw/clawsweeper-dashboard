## Summary

Keep open: current main and v2026.6.10 still exact-compare Node's active compile-cache directory against OpenClaw's desired packaged cache directory, while the linked fix PRs are closed unmerged and the remaining raw-root versus scoped-cache behavior needs maintainer direction.

## What This Changes

The issue asks OpenClaw's launcher to avoid unnecessary repeated compile-cache respawns when Node reports a versioned active cache directory under the desired cache root.

## Best Possible Solution

Settle the raw-root versus scoped-cache contract, then align launcher logic, docs, and regression coverage while preserving package-scoped invalidation and the Windows Node 24 guard.

## Reproduction Assessment

Yes, at source level: current main still exact-compares the active and desired cache paths, docs still recommend raw cache roots, and a live Node probe shows `NODE_COMPILE_CACHE` produces a versioned active child path. I did not run a full two-invocation packaged OpenClaw repro because this review is read-only.

## Solution Assessment

Unclear until maintainers choose the contract. The best fix should either make documented raw roots avoid repeated packaged respawns or update docs to require scoped cache paths, with launcher regression coverage for the chosen behavior.
