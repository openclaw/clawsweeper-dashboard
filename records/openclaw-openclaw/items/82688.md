## Summary

Keep open: current main still has the reported packaged launcher exact path comparison, the documented operator setup still uses a raw `NODE_COMPILE_CACHE` root, and the active fix path is the open draft PR at https://github.com/openclaw/openclaw/pull/82721 rather than a merged implementation.

## What This Changes

The issue asks the packaged launcher to avoid repeated compile-cache respawns when Node reports an active versioned cache directory that should already be compatible with OpenClaw's intended cache root.

## Best Possible Solution

Resolve the linked draft PR by either fixing documented raw-root repeated respawns without losing package-scoped invalidation, or explicitly changing the supported setup to scoped cache paths with docs and launcher E2E coverage aligned.
