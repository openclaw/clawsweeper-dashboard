Keep open: current main still uses strict equality in the packaged compile-cache respawn guard, and two open PRs explicitly target this issue, so closing before a merge would be premature.

## What This Changes

The issue asks the packaged launcher to avoid repeated compile-cache respawns when Node reports a versioned cache child rather than OpenClaw's computed cache root.

## Best Possible Solution

Keep this issue open until one closing PR lands or maintainers decide the raw-root versus scoped-cache contract, with regression coverage for the packaged launcher respawn marker.

Yes. Source inspection shows the current guard still compares the active cache directory to the scoped desired root, and the existing ClawSweeper comment records a current-main packaged-fixture repro that respawned on repeated launches.
