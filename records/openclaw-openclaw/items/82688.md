The issue reports redundant packaged launcher respawns when the compile-cache guard compares Node's active versioned cache directory with OpenClaw's desired cache root.

## Best Possible Solution

Resolve the linked draft PR by settling the raw-root versus scoped-cache contract, then land launcher regression coverage and keep docs, doctor guidance, and packaged launcher behavior aligned.

Yes. Existing ClawSweeper review evidence reports a current-main packaged fixture reproducing the repeated respawn, and current source plus docs still show the same raw-root and exact-comparison path.

## Solution Assessment
