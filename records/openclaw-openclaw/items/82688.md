## What This Changes

The issue asks the packaged launcher to avoid repeated compile-cache respawns when Node's active cache directory is a versioned child of OpenClaw's desired cache root.

## Best Possible Solution

Resolve the linked draft PR by either fixing documented raw-root repeated respawns without losing package-scoped invalidation, or explicitly narrowing the supported setup to scoped cache paths and updating docs/tests accordingly.

## Reproduction Assessment
