## Summary

Keep open: current main still has the launcher guard that can compare Node's versioned active compile-cache directory against OpenClaw's unversioned desired root, and the issue is already paired with open fixing PRs that use closing references.

## What This Changes

The issue requests avoiding redundant packaged launcher respawns when the active Node compile-cache directory is nested under OpenClaw's desired cache root.

## Best Possible Solution

Land one focused launcher fix that treats cache directories nested under the desired root as already anchored, adds repeated packaged-launch regression coverage, and then closes this issue through the merged PR.
