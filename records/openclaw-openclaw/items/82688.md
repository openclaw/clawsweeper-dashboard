## Summary

Keep open: current main and the latest release still have the packaged launcher exact cache-path comparison and raw-root compile-cache docs, while the related fix PRs are closed unmerged and the remaining behavior needs a launcher contract decision.

## What This Changes

The issue reports that packaged launcher compile-cache handling can respawn on every CLI invocation when `NODE_COMPILE_CACHE` starts as a raw cache root.

## Best Possible Solution

Settle the raw-root versus scoped-cache contract, then land launcher, docs, and regression-test changes that preserve package-scoped invalidation before closing this issue.

## Reproduction Assessment

Yes. A prior ClawSweeper comment records a current-main packaged-fixture reproduction, and this pass verified the same source comparison plus Node's versioned active cache-directory behavior.

## Solution Assessment

Unclear until maintainers choose the contract. The maintainable fix should either support documented raw roots without per-run respawn or update docs to require scoped cache paths while preserving package-scoped invalidation.
