## Summary

Keep this issue open. Current main still sends truthy `choice.delta.content` directly to the OpenAI-completions text append path, so an array-shaped Mistral thinking/text delta can still be coerced into visible `[object Object]` output; the focused fix remains open in #68418 and has not landed.

## What This Changes

The issue reports that `mistral/mistral-small-latest` with thinking enabled renders repeated `[object Object]` text before the normal assistant output.

## Best Possible Solution

Land #68418 or an equivalent generic parser fix that preserves the existing string fast path, unpacks array-shaped OpenAI-completions `delta.content` into ordered text and thinking deltas, and adds focused regression coverage for Mistral typed-block streams.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is an OpenAI-completions stream chunk whose `choice.delta.content` is an array of `{type:"thinking"}` and `{type:"text"}` blocks; current main passes that non-string value into the text append path, which can produce `[object Object]`.

## Solution Assessment

Yes for the linked fix direction. The narrow maintainable solution is to fix the shared OpenAI-completions stream parser generically, rather than hiding thinking in the UI or adding a Mistral-only workaround.
