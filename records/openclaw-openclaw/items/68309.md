## Summary

Keep this issue open. Current main still forwards any truthy OpenAI-completions `choice.delta.content` into the text append path without a runtime string guard or typed-block unpacker, and Mistral Small thinking is a supported provider path. Open PR #68418 is the targeted implementation candidate and has not merged in the provided GitHub context, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports literal `[object Object]` text appearing before normal replies when `mistral/mistral-small-latest` is used with thinking enabled through the Mistral provider.

## Best Possible Solution

Land #68418 or an equivalent generic OpenAI-completions parser fix that unpacks array-shaped typed content into ordered thinking/text deltas, preserves the existing string fast path, and adds regression coverage in `src/agents/openai-transport-stream.test.ts`.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is an OpenAI-completions stream chunk whose `choice.delta.content` is an array of typed thinking/text blocks; current main treats that truthy non-string value as text before reaching the reasoning parser.

## Solution Assessment

Yes for the linked fix direction. The narrow maintainable solution is to fix the shared OpenAI-completions stream parser generically, not to hide thinking in the UI or add a Mistral-only display workaround.
