## Summary

Keep open: the PR is a useful narrow bug fix, but it is not merge-ready because current main moved three of the live helpers and the PR has only test claims, not after-fix real behavior proof.

## What This Changes

The branch updates plugin allowlist advisory text in CLI missing-plugin messages, config validation, plugin loader warnings, and security audit remediation to warn that non-empty `plugins.allow` disables unlisted plugins.

## Best Possible Solution

Rebase or replace the patch against `loader-provenance.ts`, `audit-plugins-trust.ts`, `run-main-policy.ts`, and the current validation warning, keep the empty-allowlist wording conditional, and add focused assertions for the user-facing text.

## Reproduction Assessment

Yes. Source inspection is high-confidence: current activation still treats a non-empty `plugins.allow` as exclusive while the live advisory strings in loader, audit, CLI policy, and validation omit that warning.

## Solution Assessment

No. The direction is good, but this PR is not the best implementation as-is because it edits old locations for three live helpers; the maintainable path is a current-main patch plus targeted assertions.
