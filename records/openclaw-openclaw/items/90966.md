## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Publish WS status from SDK readiness callbacks:** `extensions/feishu/src/monitor.transport.ts:270-276`
  - body: The pinned Feishu SDK does not make `start()` a handshake barrier: it assigns the dispatcher and calls `reConnect(true)` without awaiting readiness. Publishing `connected: true` here can mark a slow or failed first connection healthy, so wire connected/disconnected status to `onReady`, `onReconnected`, `onReconnecting`, and `onError` instead.
  - confidence: 0.93
- **[P1] Only set transport activity from transport proof:** `extensions/feishu/src/monitor.transport.ts:273-274`
  - body: `lastTransportActivityAt` drives stale-socket restarts, and the shared policy says ordinary app events are not socket liveness. Setting it from startup and refreshing it from app messages can restart quiet healthy WebSocket accounts after the timestamp ages out; keep this field unset until Feishu exposes heartbeat, pong, poll, or raw frame proof.
  - confidence: 0.88
