## Summary

Keep open. The FD-reduction direction is useful, but this patch would stop refreshing a supported direct-root `SKILL.md` layout because chokidar now ignores regular files and the new polling scan only registers child directories.

## What This Changes

The PR changes the skills refresh watcher to ignore regular files in chokidar, poll `SKILL.md` files with `fs.watchFile`, and add tests for initial, dynamic, and cleanup polling paths.

## Best Possible Solution

Keep the no-per-file-chokidar direction, but poll every `SKILL.md` shape the loader can load, including the watched root itself, and add a focused regression test before choosing this as the canonical fix.

## Reproduction Assessment

Yes. The original FD issue is source-reproducible from current main because `SKILL.md` files are admitted into chokidar and chokidar 5 opens file watchers for included files; the linked issue also provides macOS `lsof` steps, though I did not run that live proof here.

## Solution Assessment

No. The proposed shape is close, but it is not the best fix until it preserves refresh coverage for direct-root skill directories as well as child skill directories.
