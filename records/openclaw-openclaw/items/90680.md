## Summary

Keep open. The latest head addresses the earlier direct-root `SKILL.md` polling gap, but the new `fs.watchFile` listener schedules refreshes for every callback even though Node documents that callbacks can fire on access, not only modification, so one narrow repair remains before merge.

## What This Changes

The PR changes the skills refresh watcher to ignore regular files in chokidar, poll `SKILL.md` paths with `fs.watchFile`, and add watcher lifecycle tests for initial, dynamic, direct-root, and cleanup paths.

## Best Possible Solution

Keep the directory-watch plus `SKILL.md` polling approach, but gate polling callbacks to real create/delete/modify stat changes and land after the focused watcher tests and CI remain green.

## Reproduction Assessment

Yes. The original issue is source-reproducible because current main and v2026.6.1 still allow `SKILL.md` files through the chokidar ignore predicate, while the linked report provides concrete macOS `lsof` evidence.

## Solution Assessment

No, not as-is. The owner boundary is right and the latest head fixed the direct-root gap, but the `fs.watchFile` callback must filter stat changes to preserve the documented change-only refresh contract.
