## Summary

Keep open: the fix is pointed at the right gateway/WhatsApp boundary, but it only skips the health-monitor sweep path while the generic channel manager still auto-restarts a WhatsApp task after a terminal stop returns.

## What This Changes

The PR adds a `terminalDisconnect` status flag, maps it to a `terminal-disconnect` health reason, skips health-monitor restarts for that reason, adds tests, and updates the changelog.

## Best Possible Solution

Carry the terminal-disconnect signal into the generic channel-manager recovery path so terminal WhatsApp exits skip both manager auto-restart and health-monitor restart, while transient stops still retry normally.

## Reproduction Assessment

Yes: source inspection shows the loggedOut/conflict path exits the WhatsApp monitor and the generic manager then schedules non-manual task restarts. I did not run a live WhatsApp unlink in this read-only review.

## Solution Assessment

No: the health-monitor guard is necessary but incomplete because it does not cover the earlier ChannelManager auto-restart path. The narrower maintainable fix is to suppress manager recovery when the latest runtime snapshot is marked `terminalDisconnect`.

## Review Findings
