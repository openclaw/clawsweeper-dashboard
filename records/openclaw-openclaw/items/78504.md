## Summary

Keep open: current main still has a documented Control UI tweakcn import path that fetches `https://tweakcn.com/r/themes/...` from the browser, while the served Control UI CSP does not allow that origin in `connect-src`.

## What This Changes

Fix the Control UI CSP so importing a browser-local custom theme from a tweakcn share or registry link can complete.

## Best Possible Solution

Update the Control UI CSP to allow the exact tweakcn registry origin required by the existing direct-fetch design, add focused CSP regression coverage, include the user-facing changelog entry, and prove a real theme import succeeds.

## Solution Assessment
