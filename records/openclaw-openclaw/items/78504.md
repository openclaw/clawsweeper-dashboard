## Summary

Keep open: current main still serves a Control UI `connect-src` policy without `https://tweakcn.com`, while the documented importer still fetches tweakcn registry URLs from the browser. The right path is the already-open canonical fix at https://github.com/openclaw/openclaw/pull/80168, which uses closing syntax and should close this issue after it lands.

## What This Changes

The issue reports that Control UI Appearance tweakcn imports fail because CSP `connect-src` blocks direct browser fetches to `https://tweakcn.com/r/themes/{id}`.

## Best Possible Solution

Land or close the canonical PR after review; the desired end state is an exact `https://tweakcn.com` CSP allowlist with generated and served-header regression coverage, not a broad HTTPS wildcard or config escape hatch.

## Reproduction Assessment

Yes. Current-main source gives a high-confidence reproduction path: the Appearance import calls the browser fetch helper for `https://tweakcn.com/r/themes/<id>`, while the served Control UI CSP excludes that origin.

## Solution Assessment

Yes. The narrow maintainable fix is to align the existing direct-fetch design with the exact tweakcn origin and add CSP regression coverage; a proxy or configurable CSP is broader than this documented-feature bug requires.
