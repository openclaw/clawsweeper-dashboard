## Summary

Keep open. Current main and v2026.5.19 still have ordinary `before_tool_call` block/approval/param-rewrite semantics plus subagent delegation, but not a plugin-returned quarantine directive, per-call constrained execution context, or isolated-result trust metadata; this remains a security-sensitive Plugin SDK design item.

## What This Changes

The issue requests an additive Plugin SDK/runtime capability for plugins to request isolated processing of untrusted work with capability constraints and provenance/trust metadata.

## Best Possible Solution

Keep this as a core Plugin SDK/security design item, then split implementation into targeted PRs for an additive isolation contract with enforced constraints, explicit failure semantics, operator policy, docs, tests, and structured trust/provenance metadata.
