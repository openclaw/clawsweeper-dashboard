## Summary

Keep open: current main and v2026.6.6 still have subagent/session isolation primitives, but no plugin-returned trust-boundary directive, constrained isolated execution result, or quarantined provenance metadata. This remains a security-sensitive public plugin API design item, not stale cleanup.

## What This Changes

The issue requests a plugin API that can ask OpenClaw to run untrusted work in an isolated session with constrained capabilities and provenance metadata.

## Best Possible Solution

Design a narrow plugin/runtime isolation contract that reuses existing subagent, sandbox, session, and provenance primitives while explicitly documenting enforcement guarantees and trust metadata semantics.
