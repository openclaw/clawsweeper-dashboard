## Summary

Keep open: current main and v2026.6.8 have before_tool_call blocking, parameter rewrite, approval, plugin subagent helpers, and subagent sandbox/session primitives, but they do not implement a plugin-returned quarantine/isolation directive or structured trust/provenance result metadata. The related CaMeL hook/provenance issues remain open and the broad CaMeL PR is closed unmerged, so this is still a security-sensitive Plugin SDK/runtime design item rather than a cleanup close.

## What This Changes

Request a Plugin SDK/runtime capability for plugins to run untrusted work in an isolated constrained context and receive results tagged with trust/provenance metadata.

## Best Possible Solution

Keep this open for maintainer security/product design, then define a narrow plugin SDK trust-boundary contract that composes with the related hook context and persistence-provenance issues.
