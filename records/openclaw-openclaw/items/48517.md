## Summary

Keep open: current main and v2026.6.6 still provide separate hook enforcement, subagent, and sandbox primitives, but not a plugin-returned isolation directive with constrained execution and trust/provenance metadata. This remains a security-sensitive public Plugin SDK/runtime design item rather than a conservative cleanup close.

## What This Changes

The issue requests a plugin API that can ask runtime hooks to run selected work in an isolated, constrained session and return provenance/trust metadata.

## Best Possible Solution

Design a narrow, additive plugin/runtime isolation contract that reuses existing subagent, sandbox, session, and provenance primitives while clearly documenting what is isolated, which constraints are enforceable, and how returned trust metadata should be interpreted.
