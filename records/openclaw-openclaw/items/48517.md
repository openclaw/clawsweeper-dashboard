## Summary

Keep open: current main and v2026.6.6 have subagent/session isolation primitives, but the requested plugin-requestable trust-boundary directive and quarantined-result metadata are still missing and need maintainer product/security design.

## What This Changes

Request a plugin API that lets hooks ask OpenClaw to run untrusted work in an isolated constrained context and return provenance/trust metadata.

## Best Possible Solution

Design a narrow plugin/runtime isolation contract that composes with sessions_spawn, sandbox=require, tool policy, and provenance metadata while documenting the exact trust guarantees and limits.
