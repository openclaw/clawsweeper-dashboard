## Summary

Keep open: current main and the latest release still expose before_tool_call blocking, approval, and param rewriting plus subagent/session primitives, but not the requested plugin-returned isolation directive with constrained execution and provenance/trust metadata. This remains a security-sensitive public Plugin SDK/runtime design item rather than a conservative cleanup close.

## What This Changes

The issue requests a new Plugin SDK/runtime capability for plugins to ask OpenClaw to run untrusted processing in an isolated session and return provenance/trust metadata.

## Best Possible Solution

Design an additive Plugin SDK/runtime isolation contract that reuses subagent, sandbox, tool-policy, session, and provenance primitives while explicitly documenting enforceable guarantees and trust metadata semantics.

Not applicable: this is a feature/API design proposal rather than a broken existing behavior report. Source and release inspection confirm the requested hook result shape and trust metadata contract are absent on current main and v2026.6.8.

## Solution Assessment
