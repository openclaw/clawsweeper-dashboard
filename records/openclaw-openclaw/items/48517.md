## Summary

Keep open: current main and v2026.6.10 still have blocking, approval, params rewriting, and subagent primitives, but not a plugin-returned isolation directive with constrained execution and provenance/trust metadata. The AKF comment is useful prior art for trust metadata, but this remains a security-sensitive public Plugin SDK/runtime design decision.

## What This Changes

The issue requests a public Plugin SDK/runtime capability for plugins to request isolated execution contexts with constrained capabilities and provenance/trust metadata.

## Best Possible Solution

Design an additive Plugin SDK/runtime isolation contract that reuses subagent, sandbox, tool-policy, session, and provenance primitives while explicitly documenting enforceable guarantees and trust metadata semantics.
