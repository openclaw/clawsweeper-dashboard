## Summary

Keep open. Current main still has block, approval, parameter rewrite, ordinary subagent spawning, and sandbox-required subagent modes, but it does not expose a plugin-returned quarantine directive, per-request capability constraints, or structured trust/provenance metadata for isolated results.

## What This Changes

The issue requests a Plugin SDK/runtime contract that lets plugins route selected work through an isolated constrained session and receive trust-boundary metadata with the result.

## Best Possible Solution

Keep this as a core Plugin SDK/security design item for an additive isolation contract with enforceable constraints, explicit failure semantics, and structured trust/provenance metadata.

## Reproduction Assessment

Not applicable as a feature request. The gap is source-verifiable from the hook result type, hook runner, before_tool_call runtime, plugin subagent runtime, gateway dispatch, and plugin docs.

## Solution Assessment

Unclear. A core Plugin SDK contract is the right level for a trust-boundary feature, but maintainers still need to choose the directive shape, enforcement guarantees, and result metadata before implementation would be safe.
