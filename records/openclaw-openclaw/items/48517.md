## Summary

Keep open. Current main still has sub-agent sessions, sandbox/tool policy, and plugin subagent runtime helpers, but it does not provide a plugin-returned isolation directive, per-tool-call quarantined execution with constraints, or structured trust/provenance metadata for isolated results. This remains a core/plugin API and security design item, not a ClawHub-only plugin request.

## What This Changes

The issue requests a plugin hook/runtime API that lets `before_tool_call` or a dedicated hook request quarantined per-operation execution with constrained capabilities and structured trust/provenance metadata.

## Best Possible Solution

Keep this as a core/plugin API design item. The maintainable endpoint is an additive, documented hook/runtime capability that builds on `before_tool_call`, subagent spawning, sandbox state, and tool policy, but explicitly defines constrained execution, failure/timeout behavior, and structured trust/provenance metadata before plugins can rely on it.

## Reproduction Assessment

Not applicable as a feature request. The current-main source and docs provide a high-confidence verification path for the gap: inspect the hook result type, hook runtime branch, plugin subagent runtime, and `sessions_spawn` schema to confirm no isolation directive or trust-result contract exists.

## Solution Assessment

Unclear. An additive core/plugin API is the right level for this trust-boundary feature, but the exact directive shape should wait for maintainer/security design and coordination with #48503, #48506, and #48509.
