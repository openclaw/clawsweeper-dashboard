## Summary

Keep open: current main and the latest stable release still provide before_tool_call blocking, approval, and param rewriting plus subagent sandbox/session primitives, but not a plugin-returned isolation directive with constrained execution and provenance/trust metadata. This remains a public Plugin SDK/runtime security-boundary design item rather than conservative cleanup.

## What This Changes

The issue requests a public Plugin SDK/runtime capability for plugins to request isolated, constrained execution and receive provenance/trust metadata.

## Best Possible Solution

Design an additive Plugin SDK/runtime isolation contract that reuses sessions_spawn, sandbox, tool-policy, session, and provenance primitives with explicit enforceable guarantees before exposing it to plugins.
