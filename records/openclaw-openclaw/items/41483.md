## Summary

## What This Changes

The issue reports that `openclaw agent --agent ... --channel ... --to ...` ignores the recipient target and reuses `agent:<id>:main` for distinct channel users.

## Best Possible Solution

Refresh and review the linked fix PR or a narrow replacement, preserve `--session-id` precedence, settle the recipient-key privacy/upgrade contract, and close this issue only after the fix merges.

## Reproduction Assessment

Yes, from source inspection: current main still resolves an explicit agent id to the agent main key before the recipient `--to` fallback, and the gateway caller does not pass channel into session-key resolution.

## Solution Assessment

Yes in direction: the fix belongs in the shared agent session resolver and gateway caller, but the final merge shape needs maintainer acceptance of the recipient-key privacy and upgrade contract.
