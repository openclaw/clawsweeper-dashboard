Keep open: current main and latest release still route the reported explicit-agent direct target through the agent main session, and the open canonical PR that references this issue is the active fix path. Closing before that PR merges would drop a still-valid user bug.

## What This Changes

The issue requests per-recipient session routing for `openclaw agent --agent ... --channel ... --to ...` instead of routing every target to `agent:<id>:main`.
