## What This Changes

The issue asks for `openclaw agent --agent ... --channel ... --to ...` to derive distinct session keys per target instead of collapsing all direct targets to the agent main session.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73403 or an equivalent channel-aware resolver fix that preserves explicit session precedence and avoids unsafe raw-recipient key behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows `--agent` creates an explicit agent main session key before `--to` can influence routing, and neither CLI entry point passes `channel` into the resolver.
