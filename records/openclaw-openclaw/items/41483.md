## Summary

## What This Changes

The issue requests that `openclaw agent --agent ... --channel ... --to ...` create separate per-recipient sessions instead of collapsing every recipient to `agent:<id>:main`.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/73403 or an equivalent focused resolver change that threads normalized channel context into gateway and embedded session resolution while preserving explicit session-key and session-id precedence.

## Reproduction Assessment

Yes for source-level reproduction. Current main still resolves explicit agent requests through the agent main-session fallback without a channel-aware recipient key path.

## Solution Assessment

Yes. The narrow maintainable direction is a session resolver change with regression coverage for explicit session key, session id, explicit channel targets, and non-channel/global-session behavior.
