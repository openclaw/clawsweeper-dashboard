## Summary

Keep open: current main still resolves `--agent ... --to ...` to the agent main session before considering the recipient, and the linked open PR #73403 is the active fix candidate for this behavior.

## What This Changes

The issue requests that `openclaw agent --agent ... --channel ... --to ...` route each explicit channel recipient to its own agent session instead of `agent:<id>:main`.

## Best Possible Solution

Review and land the open focused PR if its regression coverage and precedence handling are sound, otherwise close that PR and make the same narrow resolver/CLI fix on a replacement branch.

## Reproduction Assessment

Yes. Source inspection on current main shows the resolver chooses the explicit agent main key before recipient-derived routing, and both CLI entry points omit `channel` when resolving the session key.

## Solution Assessment

Yes. The narrow maintainable fix is to thread the normalized channel into session resolution for explicit `--agent --channel --to` calls, derive an agent-scoped channel-recipient key, and preserve explicit `--session-key` and `--session-id` precedence.
