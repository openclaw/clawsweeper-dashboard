## Summary

Keep this issue open: current main and v2026.6.8 still resolve `--agent --channel --to` direct-recipient runs without channel-aware per-recipient session keys, and the open linked fix PR remains the active candidate.

## Best Possible Solution

Review, rebase, and land or replace the linked PR with a narrow channel-aware recipient key fix that preserves `--session-key` and `--session-id` precedence and bounds sensitive recipient-derived key material.
