## Summary

Keep this issue open: current main and the latest release still let an explicit `--agent` select the agent main session before the reported `--to` recipient can isolate the session, and the open closing PR is still unmerged and conflicting.

## What This Changes

The issue asks for `openclaw agent --agent ... --channel ... --to ...` to create recipient-specific agent sessions instead of collapsing every target into the agent main session.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/73403 lands or maintainers replace it with an equivalent focused fix that preserves session-key and session-id precedence while addressing security review.

## Reproduction Assessment

Yes. Source inspection shows `--agent` can produce the agent main session key before `--to` participates, and both Gateway and embedded CLI paths feed the resolver without channel-aware session routing; I did not run the CLI because this review is read-only.

## Solution Assessment

Yes in direction. The narrow maintainable solution is a resolver/caller fix that derives isolated agent session keys for explicit channel recipients while preserving `--session-key` and `--session-id` precedence.
