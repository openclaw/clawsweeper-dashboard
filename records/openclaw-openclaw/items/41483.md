## Summary

Keep open: current main and the latest release still resolve explicit `--agent --channel --to` requests through the agent main-session fallback before channel context can affect session routing, and the linked closing PR is still open rather than merged.

## What This Changes

The issue reports that `openclaw agent --agent ... --channel ... --to ...` collapses distinct channel recipients into `agent:<agentId>:main` instead of creating per-recipient agent sessions.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/73403 or an equivalent focused resolver change that threads normalized channel context into gateway and embedded session resolution while preserving explicit `--session-key` and `--session-id` precedence.

## Solution Assessment

Yes. The narrow maintainable fix is channel-aware session resolution with regression coverage for explicit session key, session id, explicit channel targets, and non-channel/global-session behavior.
