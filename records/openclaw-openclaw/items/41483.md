## Summary

Keep open: current main still lets explicit `--agent` select the agent main session before recipient routing, and open PR #73403 is the active linked fix candidate. Closing this issue before that PR merges would be premature.

## What This Changes

The issue asks for `openclaw agent --agent ... --channel ... --to ...` to route distinct recipients to distinct agent sessions instead of collapsing them into the agent main session.

## Best Possible Solution

Land a focused resolver and CLI change that threads the normalized channel into explicit-agent recipient routing while preserving explicit `--session-key` and `--session-id` precedence.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: `--agent` creates an explicit agent main key before `--to` can influence routing, and neither CLI entry point passes `channel` into session resolution.

## Solution Assessment

Yes. The narrow maintainable path is to review and land PR #73403 if its coverage and precedence handling are sound, or replace it with the same focused resolver/CLI fix.
