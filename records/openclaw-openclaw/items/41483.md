## Summary

Keep open: current main still resolves explicit `--agent --channel --to` direct targets through the agent main-session path, and the live closing PR for this issue is open and conflicting rather than merged.

## What This Changes

The issue asks OpenClaw to honor `--to` for explicit channel targets with `--agent`, so different recipients get separate agent-scoped sessions instead of all sharing `agent:<id>:main`.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/73403 or an equivalent focused resolver change that threads normalized channel/direct-target context through embedded and gateway session resolution while preserving explicit `--session-key`, `--session-id`, and existing DM-scope behavior.

## Reproduction Assessment

Yes for source-level reproduction. Current main resolves an explicit agent to the agent main key before the `to` value can derive a recipient key, and neither gateway nor embedded resolution passes channel into the resolver.

## Solution Assessment

Yes. The narrow maintainable solution is a channel-aware session resolver change with regression coverage for explicit session keys, session ids, explicit channel targets, and non-channel/global-session behavior.
