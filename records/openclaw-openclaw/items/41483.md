## Summary

Keep open: current `main` and the latest shipped tag still resolve explicit `--agent --channel --to` direct targets through the agent main-session path, and https://github.com/openclaw/openclaw/pull/73403 is the open closing PR for the fix.

## What This Changes

The issue requests that `openclaw agent --agent ... --channel ... --to ...` create distinct per-recipient sessions instead of collapsing all targets to the agent main session.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/73403 or an equivalent focused resolver change that threads normalized channel through gateway and embedded session resolution while preserving explicit `--session-key` and `--session-id` precedence.

## Reproduction Assessment

Yes, for source-level reproduction: current `main` resolves an explicit `agentId` to `agent:<id>:main` before `to` can derive a recipient-specific key, and neither gateway nor embedded caller passes channel into the resolver.

## Solution Assessment

Yes. The best fix is the narrow channel-aware resolver path already represented by the open closing PR, with regression coverage for explicit session keys, session ids, explicit channel targets, and non-channel/global-session behavior.
