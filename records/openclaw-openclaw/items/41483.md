## Summary

Keep open: current main still has the reported source-level routing bug, and the open closing PR is the active fix path rather than a reason to close this issue before merge.

## What This Changes

The issue asks OpenClaw to route explicit `--agent --channel --to` CLI messages into separate per-recipient agent sessions instead of sharing `agent:<id>:main`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73403 or an equivalent focused resolver change that derives per-recipient keys for explicit deliverable channel targets while preserving `--session-key` and `--session-id` precedence and addressing recipient-key bounds/redaction.

## Reproduction Assessment

Yes. Source inspection on current main shows an explicit `agentId` is converted to the agent main session key before the direct `to` value can produce a distinct per-recipient key, and the resolver has no channel parameter.

## Solution Assessment

Yes. The narrow maintainable solution is a channel-aware session resolver fix with regression coverage for `--session-key`, `--session-id`, explicit direct targets, and non-channel/global-session behavior.
