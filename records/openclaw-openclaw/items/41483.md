## Summary

Keep open: current main and the latest release still route explicit `--agent --channel --to` direct targets through the agent main-session path, and the linked closing PR is still open, so this issue should close only after that fix or an equivalent lands.

## What This Changes

The issue reports that `openclaw agent --agent ... --channel ... --to ...` ignores distinct `--to` recipients and maps them all to the agent main session.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73403 or an equivalent focused resolver change that derives per-recipient keys for explicit deliverable channel targets while preserving `--session-key` and `--session-id` precedence and addressing recipient-key bounds/redaction.

## Reproduction Assessment

Yes for source-level reproduction. Current main shows an explicit `agentId` becomes the agent main session key before the direct `to` value can create a distinct per-recipient key, and neither gateway nor embedded resolution passes channel context.

## Solution Assessment

Yes. The narrow maintainable solution is a channel-aware session resolver fix with regression coverage for explicit session keys, session ids, explicit direct channel targets, and non-channel/global-session behavior.
