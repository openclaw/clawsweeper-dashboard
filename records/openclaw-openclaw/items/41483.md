## Summary

Keep open: current main and the latest release still resolve explicit `--agent --channel --to` runs through the agent main-session fallback before channel context can affect session routing, and https://github.com/openclaw/openclaw/pull/73403 is still open rather than merged.

## What This Changes

`openclaw agent --agent ... --channel ... --to ...` should create distinct per-recipient session keys instead of mapping all targets to the agent main session.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/73403 or an equivalent focused resolver change that threads normalized channel context into gateway and embedded session resolution while preserving `--session-key` and `--session-id` precedence.

## Reproduction Assessment

Yes for source-level reproduction: current main resolves explicit agent requests to `agent:<id>:main` before channel context is available. I did not run a live Chatwoot, WhatsApp, or Instagram bridge repro in this read-only pass.

## Solution Assessment

Yes. The narrow maintainable fix is a channel-aware session resolver with regression coverage for explicit session key, session id, explicit channel targets, and non-channel/global-session behavior.
