## Summary

Keep open: current main and `v2026.5.12` still resolve explicit `--agent --channel --to` calls to the agent main session before the channel can participate in session routing, and https://github.com/openclaw/openclaw/pull/73403 is the open closing fix path.

## What This Changes

The issue requests distinct sessions for `openclaw agent --agent ... --channel ... --to ...` targets instead of collapsing all recipients into the agent main session.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73403 or an equivalent focused channel-aware resolver fix that preserves explicit `--session-key` and `--session-id` precedence.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: `--agent` selects the agent main session before `--to` and `--channel` can produce a per-recipient key, and the latest release has the same shape.

## Solution Assessment

Yes. The narrow maintainable solution is a channel-aware session resolver change with focused regression coverage, preferably through the existing open canonical PR rather than a duplicate repair lane.
