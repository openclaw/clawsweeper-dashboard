## Summary

## What This Changes

The issue reports that `openclaw agent --agent ... --channel ... --to ...` ignores the target recipient and collapses multiple recipients into one `agent:<id>:main` session.

## Best Possible Solution

Keep this issue open as the canonical bug until the linked fix PR lands or maintainers replace it with a narrower branch that preserves selector precedence and settles the recipient-key contract.

## Reproduction Assessment

Yes, from source: current `main` resolves explicit-agent direct targets without channel context, and the shared resolver collapses direct targets to the agent main bucket. I did not run the CLI because this read-only review cannot create session-store artifacts.
