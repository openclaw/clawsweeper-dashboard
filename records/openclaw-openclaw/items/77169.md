## Summary

Keep open: current main has native video-understanding internals and CLI video description, including Moonshot/Kimi `video_url` payload construction, but the requested agent-facing `video` analysis tool is still missing and needs maintainer API-surface direction.

## What This Changes

Request an agent-facing video analysis tool, or an image-tool extension, that accepts local or URL videos and routes native provider video input such as Kimi `video_url`, with optional chunking for long videos.

## Best Possible Solution

Add a generic agent-facing video analysis surface patterned after `image`, backed by the existing media-understanding runtime and current media URL/file safety policies, with docs for Kimi/Moonshot video input.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence gap: current main can describe videos through media-understanding internals and the CLI, but no agent-facing `video` analysis tool exists.

## Solution Assessment

Unclear until maintainers choose the user-facing surface. The narrowest maintainable direction is a generic `video` tool that reuses `describeVideoFile` and provider selection rather than adding Kimi-specific logic to core.
