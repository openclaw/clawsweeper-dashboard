## Summary

Keep open: current main already has native Moonshot/Kimi video-understanding plumbing and a CLI video-description path, but it still lacks the requested agent-facing `video` analysis tool analogous to `image`.

## What This Changes

Request an agent-facing native video analysis tool that accepts video paths or URLs and uses multimodal provider video input such as Kimi `video_url`.

## Best Possible Solution

Add a generic agent-facing video analysis surface backed by the existing media-understanding runner and provider registry, with provider-neutral docs and Kimi/Moonshot handled inside the Moonshot plugin.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection verifies current main has video-understanding internals and CLI description, but no agent-facing `video` analysis tool in the registered tool set.

## Solution Assessment

Unclear until maintainers choose the public surface. The narrowest maintainable implementation is likely a provider-neutral `video` analysis tool that reuses existing media-understanding video providers rather than adding Kimi-specific logic to core.
