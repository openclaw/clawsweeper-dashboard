## Summary

Keep open: current main has Moonshot/Kimi native video-understanding internals and `openclaw infer video describe`, but it still lacks the requested agent-facing video analysis tool analogous to `image`.

## What This Changes

Add a native agent-facing video analysis tool, or extend the image tool, so agents can analyze local video paths or URLs through multimodal providers such as Kimi K2.6.

## Best Possible Solution

Define a provider-neutral agent video analysis surface that reuses `src/media-understanding` and the existing Moonshot `video_url` path, then update tool catalog, docs, and focused tests for path/URL policy.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that video understanding exists in provider/CLI runtime but no agent-facing `video` analysis tool is registered.

## Solution Assessment

Unclear until maintainers choose the user-facing surface. The narrowest maintainable direction is to reuse the existing media-understanding runtime and security controls rather than adding Kimi-specific agent-tool logic.
