## Summary

Keep open: current main still matches the prior Codex review, with native video-understanding internals and `openclaw infer video describe` present, but no agent-facing `video` analysis tool analogous to `image`; this is a feature/API-surface decision rather than a cleanup close.

## What This Changes

Request to add a native video analysis tool for local or remote videos, routing multimodal models such as Kimi K2.6 through provider-native video inputs instead of frame extraction.

## Best Possible Solution

Add a generic agent-facing `video` analysis tool that reuses media-understanding providers and native `video_url` support, with explicit path/URL, size, and long-video policy.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main lacks an agent `video` analysis tool while already having video-understanding internals and a file-only CLI path.

## Solution Assessment

Unclear until maintainers choose the public surface. The narrowest maintainable direction is likely a generic `video` tool reusing the existing media-understanding runtime rather than adding Kimi-specific logic to core.
