## Summary

Keep open: current main has native Moonshot/Kimi video-understanding plumbing and a file-based `openclaw infer video describe` path, but it still lacks the requested agent-facing `video` analysis tool and needs maintainer direction on the public surface.

## What This Changes

Request an agent-facing native video analysis tool, analogous to `image`, that can pass local videos or URLs to multimodal models such as Kimi K2.6 using native `video_url` input.

## Best Possible Solution

Add or explicitly defer a generic agent-facing video analysis surface that reuses media-understanding runtime/provider selection and keeps Kimi-specific payload details in the Moonshot provider.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence gap: current main has native video-understanding internals and CLI description, but no agent tool named `video` or equivalent video-analysis tool.

## Solution Assessment

Unclear until maintainers choose the user-facing surface. The narrowest maintainable implementation would reuse the existing media-understanding runtime and Moonshot `video_url` provider path rather than adding Kimi-specific logic to the agent core.
