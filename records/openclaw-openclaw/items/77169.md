## Summary

Keep open: current main already has provider-backed video understanding and Moonshot/Kimi `video_url` internals, but it still lacks the requested agent-facing video analysis tool analogous to `image`; the remaining work is a public tool/API-surface decision.

## What This Changes

Add a native agent-facing video analysis tool, or extend `image`, so agents can analyze video paths or URLs using multimodal providers' native video input.

## Best Possible Solution

Define one generic video analysis surface that reuses the existing media-understanding runtime/provider contracts and documents the chosen path, URL, and long-video behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence gap: current main can describe videos through media-understanding and CLI paths, but no agent-callable video analysis tool exists.

## Solution Assessment

Unclear until maintainers choose the user-facing surface. The narrow maintainable direction is to reuse existing media-understanding provider selection and Moonshot `video_url` support rather than adding provider-specific agent-tool logic.
