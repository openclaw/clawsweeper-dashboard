## Summary

Keep open: current main has video-understanding internals, including Moonshot/Kimi-style `video_url` payloads and `openclaw infer video describe`, but it still lacks the requested agent-facing video analysis tool analogous to `image`.

## What This Changes

Request a native agent-facing video analysis tool for local paths or URLs that uses model-native video input, especially Kimi/Moonshot `video_url`, with possible long-video chunking.

## Best Possible Solution

Add a generic agent-facing video analysis surface, or document an intentional alternative, that reuses media-understanding providers and preserves existing sandbox, SSRF, size, and timeout controls.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that OpenClaw has CLI/inbound video understanding and `video_generate`, but no agent tool named `video` or equivalent video-analysis tool.

## Solution Assessment

Unclear until maintainers choose the user-facing surface. The narrow maintainable direction is a generic video-analysis tool reusing media-understanding runtime and the existing `video_url` provider helpers, not Kimi-specific logic in core.
