## Summary

Keep open: this is a plausible TUI feature, but the current PR head still regresses current main's bare `/context` selector and leaves valid null context reports as permanent loading; it also lacks real TUI behavior proof.

## What This Changes

The PR adds a `/context visualize`/`/context viz` TUI overlay, slash-command completions/help, a TUI client fetcher for `sessions.usage`, and related visualizer modules/tests.

## Best Possible Solution

Integrate the visualizer as an option in the existing `/context` selector, keep gateway subcommands forwarded, show a clear empty/unavailable state when no run report exists, and add focused tests plus contributor real TUI proof.

## Reproduction Assessment

Yes. Source inspection against current main and the PR head shows bare `/context` now opens a selector on main while the PR sends it to the gateway; `sessions.usage` may return `contextWeight: null`, and the overlay keeps null reports as loading.

## Solution Assessment

No. The feature direction is reasonable, but the branch should be rebased onto current selector behavior and treat null `contextWeight` as a loaded empty/unavailable state before it is the maintainable solution.
