## Summary

Keep open. Current main still lacks the requested interactive `/context visualize`/`/context viz` TUI overlay, so the PR is not obsolete, but the proposed patch has concrete correctness/performance blockers around missing cached reports and repeated `sessions.usage` scans before it is mergeable.

## What This Changes

The PR adds a TUI `/context visualize`/`/context viz` overlay with bar charts, sparkline history, drill-down views, slash-command completions/help, a `GatewayChatClient.getContextReport()` wrapper over `sessions.usage`, and adjusted TUI command-handler tests.

## Best Possible Solution

Land a rebased implementation that plugs visualization into current main's existing `/context` selector/forwarding flow, preserves all gateway subcommands, handles missing cached context reports with an explicit empty/estimated state, and avoids using the full usage scanner as a short-interval refresh source.

## Reproduction Assessment

Yes for the blocking patch behavior: when `sessions.usage` returns a session with `contextWeight: null`, `getContextReport()` returns null, `refresh()` ignores it, and `render()` keeps showing the loading state. The feature request itself is not a bug report, but the PR's broken empty-state path is reproducible from the source contracts.

## Solution Assessment

No. A TUI-owned visualizer is a plausible direction, but this PR should not rely solely on cached `sessions.usage` contextWeight for data or poll the usage scanner for live refreshes; a lightweight context-report seam or reuse of the existing `/context` report builder would be safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Show an empty state when no report is cached:** `src/tui/components/context-viz-overlay.ts:71-85`
  - body: `fetchReport()` can successfully return `null` because `sessions.usage` exposes `contextWeight` from the stored `systemPromptReport`, which is absent for fresh/new sessions or sessions before a first run. Since this branch only updates state inside `if (report)`, `/context viz` stays on `Loading context data...` indefinitely instead of showing an empty state or computing the same estimate as the gateway `/context` command.
  - confidence: 0.9
- **[P2] Avoid polling the usage scanner for refreshes:** `src/tui/gateway-chat.ts:269-271`
  - body: The visualizer refresh path calls `sessions.usage`, and the gateway handler loads `loadSessionCostSummary(...)` before returning each session, which scans the transcript file. Leaving the overlay open therefore repeats a full session usage scan every 10 seconds even though the UI only needs the cached context report; use a lightweight context-report path or avoid automatic polling of the usage endpoint.
  - confidence: 0.83
