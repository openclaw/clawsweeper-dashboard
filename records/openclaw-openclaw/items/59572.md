The PR adds a TUI `/context visualize`/`/context viz` overlay for context usage charts and updates slash-command help, gateway report fetching, and TUI command-handler tests.

## Best Possible Solution

Rebase/update the PR so `/context viz` is additive to the existing selector, `/context map` remains visible, null reports show a clear unavailable state, focused tests cover those paths, and real TUI proof is attached before merge.

Yes. Source inspection gives a high-confidence reproduction path for the review findings: compare current main's bare `/context` selector and nullable `sessions.usage` contract with the PR head's command branch, completion list, and overlay loading state.

  - confidence: 0.91

  - body: Current main supports and documents `/context map`, but this new local `context` slash command lists only `visualize`, `viz`, `list`, `detail`, and `json` in completions/help. Add `map` so the TUI does not hide a valid current mode.
  - confidence: 0.84
