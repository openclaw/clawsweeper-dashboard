## Summary

Keep open. Current main does not implement the visual TUI context overlay, but this PR still has merge-blocking integration issues against current main plus a missing required changelog entry.

## What This Changes

The PR adds a `/context visualize`/`/context viz` TUI overlay with context charts and drill-downs, a TUI client fetcher for `sessions.usage`, slash-command completions/help, and related command-handler test updates.

## Best Possible Solution

Keep the visualizer as TUI-owned UI, but integrate it with the current `/context` selector, handle unavailable context reports explicitly, add focused tests, and include the required changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: on current main bare `/context` opens a selector, while the PR branch sends it to the gateway; additionally, a session whose `sessions.usage` entry has `contextWeight: null` leaves the PR overlay stuck on loading.

## Solution Assessment

No. The feature direction is plausible, but the current PR is not the narrowest maintainable solution until it preserves existing `/context` UX and handles null context reports instead of an indefinite loading state.
