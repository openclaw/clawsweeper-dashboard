## Summary

Keep open: the feature direction is plausible, but the PR still has source-proven regressions against current `main` and lacks after-fix real TUI behavior proof.

## What This Changes

Adds a TUI `/context visualize`/`/context viz` overlay, context slash-command completions/help, a `sessions.usage` context report fetcher, visualization helpers, and TUI command-handler test updates.

## Best Possible Solution

Integrate the visualizer into the current `/context` selector alongside list/detail/json/map/help behavior, handle unavailable context reports explicitly, keep focused tests, and require real TUI proof before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction of the review findings: current `main` opens a selector for bare `/context`, while PR head forwards bare `/context`, and `sessions.usage` can validly return `contextWeight: null`.

## Solution Assessment

No. The visualizer is a reasonable feature direction, but this branch should be rebased onto current selector and `/context map` behavior, add an explicit unavailable state for null reports, and show real TUI proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Preserve the bare /context selector:** `src/tui/tui-command-handlers.ts:518-524`
  - body: Current main opens a local mode selector for bare `/context`, with tests asserting it does not send immediately. This branch only opens the overlay for `visualize` or `viz`, so bare `/context` falls through to `sendMessage(raw)` and would regress the current TUI flow; add the visualizer to the selector while keeping direct `/context viz` support.
  - confidence: 0.9
- **[P2] Show an unavailable state for null context reports:** `src/tui/components/context-viz-overlay.ts:70-71`
  - body: `sessions.usage` can successfully return `contextWeight: null` for sessions without a stored report. Since `refresh()` ignores null reports and `render()` treats `this.report === null` as loading, `/context viz` can stay on `Loading context data...` forever; track a loaded-empty/error state or reuse the existing unavailable-report copy.
  - confidence: 0.91
- **[P3] Include /context map in context completions:** `src/tui/commands.ts:120-122`
  - body: Current main supports and documents `/context map`, but this new local `context` slash command lists only `visualize`, `viz`, `list`, `detail`, and `json` in completions/help. Add `map` so the TUI does not hide a valid current mode.
  - confidence: 0.82
