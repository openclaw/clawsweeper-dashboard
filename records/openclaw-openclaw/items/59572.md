## Summary

Keep open: the feature is useful, but the PR still regresses current main’s bare `/context` selector, can stay in a loading state for valid null context reports, omits the current `/context map` mode from new TUI help/completions, and lacks real TUI proof.

## What This Changes

The branch adds a TUI `/context visualize`/`/context viz` overlay, slash-command completions/help, a `sessions.usage` context fetcher, visualization helpers, and TUI tests.

## Best Possible Solution

Integrate the visualizer into the existing `/context` selector alongside current modes, keep direct gateway subcommands forwarding, handle unavailable context reports explicitly, update focused tests, and require real TUI proof before merge.

## Reproduction Assessment

Yes. Source inspection is high-confidence: current main keeps bare `/context` local while the PR sends it to the gateway, and `sessions.usage` can return `contextWeight: null` while the overlay treats null as loading.

## Solution Assessment

No. The feature direction is reasonable, but the branch should be rebased onto current selector and `/context map` behavior and should show a clear empty/unavailable state before it is a maintainable solution.

  - confidence: 0.82
