## Summary

Keep open: current main still has separate Google Chat paths for automatic reply delivery and message-tool delivery. The automatic path receives and edits or clears `typingMessageName`, while current-source `message(action=send)` reaches the Google Chat action handler and sends directly without consuming that placeholder.

## What This Changes

Ensure Google Chat current-source `message(action=send)` replies consume or edit the message-backed typing placeholder instead of leaving it visible.

## Best Possible Solution

Keep the model-facing message tool unchanged and add a Google Chat-owned lifecycle bridge that edits the first same-target tool reply into the placeholder, or deletes the placeholder when delivery is bypassed.

## Reproduction Assessment

Yes, at source level. With `typingIndicator: "message"` and group visible replies set to `message_tool`, the monitor stores `typingMessageName`, automatic delivery is suppressed, and the Google Chat action send path does not consume that name.

## Solution Assessment

Yes. The best direction is a narrow Google Chat delivery/action integration that reuses the existing placeholder edit/delete behavior without adding a model-visible API or config knob.
