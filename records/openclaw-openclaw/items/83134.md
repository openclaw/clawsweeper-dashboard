## Summary

Keep open: current main and the latest shipped release still use `mailto:openclaw@localhost` as the default Web Push VAPID subject, while the implemented PWA/Web Push feature expects that VAPID subject to be used for browser push sends. The report is credible and source-backed, but the fix needs maintainer direction on the safe default/config/upgrade behavior.

## What This Changes

The issue requests changing or warning about the Gateway Web Push VAPID subject because auto-generated keys default to `mailto:openclaw@localhost`, which Apple Web Push rejects for iOS PWAs.

## Best Possible Solution

Define a supported VAPID subject resolution policy for Web Push, preserve the existing env override, add focused validation/warning and docs, and include upgrade guidance for users who already subscribed with the localhost subject.

## Reproduction Assessment

Yes, source-reproducible: on current main, unset `OPENCLAW_VAPID_SUBJECT`, generate Web Push keys, subscribe an iOS PWA, then send `push.web.test`; OpenClaw will sign with `mailto:openclaw@localhost`. I did not live-test Apple Web Push from this worker.

## Solution Assessment

Unclear. The report is valid, but the requested fixes range from warning-only to new config and Tailscale auto-detection, so maintainers should choose the subject policy before implementation.
