## Summary

Keep open: current main and v2026.5.12 still use the localhost VAPID fallback, and an open closing PR now targets the fix. The issue should remain open until https://github.com/openclaw/openclaw/pull/83181 merges or is closed.

## What This Changes

The issue asks OpenClaw to stop auto-generating Web Push VAPID keys with a localhost mailto subject that Apple rejects for iOS PWA push.

## Best Possible Solution

Land a narrow fix that replaces the no-env fallback with a stable valid OpenClaw https: or mailto: subject, updates tests/docs, and keeps OPENCLAW_VAPID_SUBJECT as the operator override.

## Reproduction Assessment

Yes for source-level current-main reproduction: with no OPENCLAW_VAPID_SUBJECT, resolveVapidKeys resolves to mailto:openclaw@localhost and the send path passes that subject to web-push. I did not run a live Apple endpoint send in this read-only review.

## Solution Assessment

Yes for the narrow fix direction: changing the built-in fallback is the smallest maintainable repair for first-install users, while the existing env override still covers operator-specific subjects. The linked PR should stay the implementation path unless it is closed.
