## Label Justifications

- `P1`: The bug can break iOS PWA Web Push notifications for first-install users without an override, which is a user-facing delivery regression.
- `impact:message-loss`: The reported failure causes Web Push notifications to be rejected instead of delivered to subscribed iOS PWA clients.

## Summary

Keep open: current main and v2026.5.12 still default the no-env Web Push VAPID subject to `mailto:openclaw@localhost`, and https://github.com/openclaw/openclaw/pull/83317 is the active closing PR for the narrow fix.

## What This Changes

The issue asks OpenClaw to stop auto-generating Web Push VAPID keys with a localhost subject that Apple Web Push rejects for iOS PWA notifications.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83317, or an equivalent narrow fix that uses a stable valid OpenClaw-owned VAPID subject while preserving `OPENCLAW_VAPID_SUBJECT`.

## Reproduction Assessment

Yes at source level: current main resolves the no-env VAPID subject to `mailto:openclaw@localhost` and passes it to `webPush.setVapidDetails()`. I did not independently hit Apple's live push endpoint in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to change the built-in fallback to a stable valid `https:` or `mailto:` subject and update tests/docs; a new config key is not required because the env override already exists.
