## Summary

Keep open: current main and the latest release still use the localhost VAPID subject and pass it to `web-push`; the open closing PR is the right fix path, so this issue should remain open until that PR or a replacement lands.

## What This Changes

Fix Web Push VAPID key generation so the no-env default subject is accepted by Apple/Safari instead of `mailto:openclaw@localhost`.

## Best Possible Solution

Land or replace the open linked PR that changes the built-in default to a stable valid URI, keeps the env override, and updates focused tests/docs, then close this issue.

## Reproduction Assessment

Yes. Source inspection on current main shows no-env VAPID resolution falls back to `mailto:openclaw@localhost`, and the send path passes that subject to `web-push`; I did not hit Apple's live endpoint in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to replace the built-in fallback with a valid stable subject, keep the existing env override, and update the focused tests/docs; the open linked PR follows that shape.
