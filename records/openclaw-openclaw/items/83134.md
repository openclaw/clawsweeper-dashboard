## Summary

Keep open: current main and the latest release still use the localhost VAPID subject, and there is now an open closing PR that should be reviewed or landed before this issue closes.

## What This Changes

The issue reports that auto-generated Web Push VAPID keys default to a localhost mailto subject that Apple Web Push rejects for iOS PWA notifications.

## Best Possible Solution

Review and land the focused default-subject PR, keeping `OPENCLAW_VAPID_SUBJECT` as the operator override and avoiding a new config surface unless maintainers want a broader push configuration design.

## Reproduction Assessment

Yes at source level: current main resolves the no-env VAPID subject to `mailto:openclaw@localhost` and passes it into `webPush.setVapidDetails()`. I did not independently exercise Apple's live push endpoint.

## Solution Assessment

Yes. The narrow maintainable fix is to replace the built-in fallback with a stable valid `https:` or `mailto:` subject and update focused tests/docs; the linked open PR follows that shape.
