## Summary

Keep open: current main and v2026.5.12 still default Web Push VAPID to `mailto:openclaw@localhost`, and the active closing PR at https://github.com/openclaw/openclaw/pull/83317 owns the narrow fix but has not merged.

## What This Changes

The issue reports that OpenClaw's no-env Web Push VAPID key generation uses `mailto:openclaw@localhost`, causing Apple Web Push/iOS PWA sends to fail with `BadJwtToken`.

## Best Possible Solution

Land the active narrow default-subject fix after maintainer review, keep `OPENCLAW_VAPID_SUBJECT` as the override, and close this issue only after the closing PR merges.

## Reproduction Assessment

Yes at source level: current main resolves the no-env VAPID subject to `mailto:openclaw@localhost` and passes it into `webPush.setVapidDetails()`. I did not independently run a live Apple endpoint send.

## Solution Assessment

Yes. A narrow constant, test, and docs update like https://github.com/openclaw/openclaw/pull/83317 is the maintainable fix; a new config key is not required because the env override already exists.
