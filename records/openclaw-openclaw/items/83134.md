## Label Justifications

- `P1`: The reported default can break iOS PWA push delivery for first-install users without an env override.
- `impact:message-loss`: The failure mode is silently missed Web Push notifications for subscribed iOS PWA clients.

## Summary

Keep open: current main and the latest shipped release still generate Web Push VAPID keys with `mailto:openclaw@localhost`, and the active closing PR should be reviewed or landed before this issue closes.

## What This Changes

The issue reports that OpenClaw's auto-generated Web Push VAPID subject uses a localhost mailbox that Apple Web Push rejects for iOS PWA notifications.

## Best Possible Solution

Review and land the active narrow PR if maintainers accept its project-owned default subject, keeping `OPENCLAW_VAPID_SUBJECT` as the customization path and avoiding a new config key unless a broader push-config design is desired.

## Reproduction Assessment

Yes at source level: current main resolves the no-env VAPID subject to `mailto:openclaw@localhost` and passes it to `webPush.setVapidDetails()`. I did not independently hit Apple's live push endpoint in this read-only review.

## Solution Assessment

Yes. The narrowest maintainable fix is to replace the invalid built-in fallback with a stable valid `https:` or `mailto:` subject, update focused tests/docs, and preserve the existing env override.
