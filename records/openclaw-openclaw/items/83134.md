## Summary

Keep open: current main and v2026.5.12 still use `mailto:openclaw@localhost` as the no-env Web Push VAPID subject, and the linked fix PR is open but unmerged. The issue should stay open until https://github.com/openclaw/openclaw/pull/83317 lands or maintainers choose a different default-subject fix.

## What This Changes

The issue asks OpenClaw to stop auto-generating Web Push VAPID subjects with `mailto:openclaw@localhost`, which Apple Web Push rejects for iOS PWA notifications.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83317 or an equivalent narrow fix that uses a stable project-owned `https:` or `mailto:` default, preserves `OPENCLAW_VAPID_SUBJECT`, and updates focused tests/docs.

Yes at source level: current main resolves the no-env VAPID subject to `mailto:openclaw@localhost` and passes it into `webPush.setVapidDetails()`. I did not repeat a live Apple endpoint send in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to replace the module-private fallback with a valid stable contact URI, keep the existing env override, and refresh tests/docs; a new config key is not required for this bug fix.
