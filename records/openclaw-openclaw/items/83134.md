- `P1`: The report describes a broken first-install iOS PWA Web Push workflow affecting real users until the default is fixed.
- `impact:message-loss`: The failure prevents Web Push notifications from being delivered to subscribed iOS PWA clients.

## Summary

Keep open: current main and v2026.5.12 still use the localhost VAPID subject, and an open closing PR already owns the narrow fix, so this issue should close only after that PR lands or is replaced.

## What This Changes

The issue reports that auto-generated Web Push VAPID keys default to a localhost subject that Apple Web Push rejects for iOS PWAs.

## Best Possible Solution

Land or replace the open closing PR with a narrow valid default-subject fix, keep `OPENCLAW_VAPID_SUBJECT` as the override, update focused tests/docs, and close this issue after merge.

## Reproduction Assessment

Yes at source level: current main resolves the no-env VAPID subject to `mailto:openclaw@localhost` and passes it into `webPush.setVapidDetails()`. I did not independently exercise Apple’s live push endpoint in this read-only review.

## Solution Assessment

Yes: replacing the module-private fallback with a stable public `https:` or `mailto:` subject while preserving the existing env override is the narrowest maintainable fix. A new config field is not required for the minimal bug fix.
