## What This Changes

The PR changes plugin CLI metadata normalization so malformed descriptor rows, fields, and parent-path entries are isolated instead of dropping healthy sibling command metadata or shortening nested command roots.

## Best Possible Solution

Keep the registry-owned row-level normalization and land after the draft state, required checks, and maintainer review are satisfied.

## Solution Assessment

Yes. Normalizing individual CLI metadata rows inside `src/plugins/registry.ts` is the narrow owner-boundary fix; sibling CLI loaders consume the sanitized registry entries, and the tests cover the descriptor and parent-path failure modes.
