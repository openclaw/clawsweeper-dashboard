## Summary

Keep open: current main still has the source path that recomputes external-auth overlays and logs the unchanged Codex bootstrap-only local-OAuth decision on every relevant auth-store overlay. The related non-bootstrap-only stale-profile PR does not cover this Codex case, and the broader plugin-discovery reports do not fully own the idle auth-poll trigger.

## What This Changes

The issue asks OpenClaw to stop idle gateway polls from repeatedly resolving/logging openai-codex bootstrap-only auth decisions and synchronously scanning plugin manifests.

## Best Possible Solution

Preserve Codex bootstrap-only local-OAuth precedence while making unchanged auth-status/idle polls cheap: suppress repeated unchanged debug noise and avoid redundant plugin/external-auth work using clear invalidation rather than broad stale metadata caches.

## Reproduction Assessment

No live idle-gateway reproduction was run, so there is not a high-confidence runtime reproduction of the 30-second cadence. Source inspection does show the repeated decision/log path whenever the auth overlay is invoked for a stored `openai-codex:default` OAuth profile.

## Solution Assessment

Yes, a narrow auth-path repair is the best first fix. It should keep local Codex OAuth canonical, avoid persisting or overwriting Codex CLI credentials, and either log only on state change or memoize the bootstrap-only no-op decision with auth/config/scope invalidation.

## Repair Work Prompt

Implement a narrow fix for https://github.com/openclaw/openclaw/issues/81143. Current main recomputes external-auth overlays and emits the unchanged `openai-codex:default` bootstrap-only local-OAuth debug decision whenever `ensureAuthProfileStore` overlays external auth; this can also pull plugin external-auth resolution into idle/status reads. Keep Codex `bootstrapOnly` semantics intact: do not persist, overwrite, or shadow the local Codex OAuth credential with Codex CLI credentials once local token material exists. If touching plugin manifest scanning, do not add a broad persistent metadata cache that conflicts with `src/plugins/AGENTS.md`; use prepared snapshots or narrow coalescing only where the caller owns freshness.
