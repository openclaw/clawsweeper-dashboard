## Summary

Keep open: current main still has the reported bootstrap-only Codex auth decision logging on every external auth overlay resolution. The related stale-profile fix path covers non-bootstrap-only external CLI OAuth behavior and does not resolve this `openai-codex` case.

## What This Changes

Reduce repeated `openai-codex` bootstrap-only auth debug logging and provider/auth resolution while the gateway is idle.

## Best Possible Solution

Preserve Codex bootstrap-only precedence while suppressing unchanged debug noise and avoiding unnecessary repeated external-auth work when auth/profile/config state has not changed.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows the repeated decision/log happens on each external auth overlay call for a stored `openai-codex:default` OAuth profile, but the exact idle 30-second gateway cadence still needs runtime proof.

## Solution Assessment

Yes, a narrow fix in the auth-profile external CLI path is the right direction. It should not persist or overwrite Codex CLI credentials; it should rate-limit/state-change the debug log or memoize the bootstrap-only no-op decision with clear invalidation.
