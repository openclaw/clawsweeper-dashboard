## Summary

Keep open. Current main still has a source-reproducible path where Discord auto-presence evaluates every 30s, reads the auth store through the external-auth overlay path, and hits the bootstrap-only openai-codex branch that emits the reported DEBUG line when a local OAuth profile exists.

## What This Changes

The issue asks OpenClaw to stop re-running and logging the unchanged openai-codex bootstrap-only local-OAuth decision every 30 seconds while an idle Discord-enabled gateway is running.

## Best Possible Solution

Move Discord auto-presence availability polling onto a no-external-overlay auth-store read, or an equivalently lightweight cached availability seam, so idle presence checks do not resolve provider/plugin external auth or re-emit unchanged bootstrap decisions.

## Reproduction Assessment

Yes at source level, provided Discord auto-presence is enabled: the controller starts a 30s interval, each evaluation calls ensureAuthProfileStore, and that path overlays external auth profiles before hitting the bootstrap-only openai-codex debug branch. I did not live-run a gateway with Discord credentials in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to stop this idle status poll from using the full external-auth overlay path; rate-limiting the debug line alone would reduce noise but leave the repeated idle auth/plugin work in place.
