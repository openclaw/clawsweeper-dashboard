## Summary

Keep open: the requested observability is still useful, but this PR targets retired memory/CLI paths and would not affect the active `memory-core` implementation on current `main`. The earlier timestamp/logging review bugs were addressed on the PR branch, so the remaining blocker is a current-path port rather than abandoning the work.

## What This Changes

The PR adds session sync status fields and logging, renders session state in `openclaw memory status --deep`, and adds memory manager/CLI tests.

## Best Possible Solution

Port the observability into `extensions/memory-core`: expose an additive `custom.sessionSync` payload, track last sync time/reason/error in active sync paths, render concise session state with deep-only pending counters, and add focused tests plus changelog/docs if user-facing output changes.
