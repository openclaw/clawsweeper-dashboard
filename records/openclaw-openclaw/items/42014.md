## Summary

Keep open: the observability idea is useful and is not implemented on current main, but this branch is not merge-ready because it adds the CLI/status payload under legacy `src/cli` and `src/memory` paths while the active `openclaw memory status` command now lives in `extensions/memory-core`; the external PR also still has only test-based proof.

## What This Changes

The PR adds session sync status fields, warn/debug logging, CLI memory status rendering, and tests under the legacy memory CLI and manager paths.

## Best Possible Solution

Port the payload, renderer, tests, and any docs wording into `extensions/memory-core`, with status labels that distinguish pending in-process session sync from durable disk/index freshness.
