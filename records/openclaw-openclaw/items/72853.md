## Summary

Keep open. Current main still has the cron missing-id collision, while this PR is a focused fix with no blocking correctness or security findings from the read-only review.

## What This Changes

The PR backfills and persists stable IDs for loaded cron jobs missing both `id` and legacy `jobId`, guards cron outcome application against invalid job IDs, and adds cron tests, docs, and changelog coverage for #72849.

## Best Possible Solution

Land the focused cron fix after maintainer review and exact-head validation, keeping ID backfill in the cron store load seam and the defensive outcome guard in the timer path.

## Reproduction Assessment

Yes. The source-backed reproduction is to load multiple persisted cron jobs missing both `id` and `jobId`, then apply a cron outcome with an undefined job ID; current main can match the first id-less row via `entry.id === result.jobId`.

## Solution Assessment

Yes. Backfilling stable IDs at load time fixes future scheduler identity, while the timer guard prevents any remaining malformed outcome from mutating the wrong stored row.
