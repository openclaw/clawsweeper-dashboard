## Summary

Keep open. Current main still lacks the cron missing-ID backfill and invalid outcome guard, while this PR is a focused implementation for the linked valid bug with no blocking review findings found.

## What This Changes

This PR backfills and persists stable IDs for loaded cron jobs missing both `id` and legacy `jobId`, guards cron outcome application against invalid job IDs, and updates cron tests, docs, and changelog coverage.

## Best Possible Solution

Land the focused cron fix after maintainer review and current required checks, keeping the identity repair in the cron store load seam and the defensive malformed-outcome guard in the timer path.

## Reproduction Assessment

Yes. A high-confidence reproduction is to load multiple persisted cron jobs missing both `id` and `jobId`, then let a run outcome with no valid job ID apply state; current main can match the first id-less row through the unguarded `entry.id === result.jobId` lookup.

## Solution Assessment

Yes. The proposed direction is the narrow maintainable fix: repair persisted job identity at load time so future scheduler state is keyed correctly, and fail closed if any malformed run outcome still reaches the timer.
