## What This Changes

The branch backfills and persists cron job UUIDs for loaded jobs missing both `id` and legacy `jobId`, guards malformed cron outcomes, and updates cron tests, docs, and changelog.

## Best Possible Solution

Land this focused cron fix after maintainer review and exact-head validation, keeping identity repair in the cron store load path and the malformed-outcome guard in the timer path.

## Reproduction Assessment

Yes. The reproduction is to load multiple persisted cron jobs missing both `id` and `jobId`, then let a run outcome apply state; current main can match the first id-less row through the unguarded `entry.id === result.jobId` lookup.

## Solution Assessment

Yes. The PR is the narrow maintainable solution: generate stable IDs once at load time, persist the canonical shape, and fail closed if any malformed outcome still reaches the timer.
