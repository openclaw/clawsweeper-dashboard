The PR changes `saveAuthProfileStore` so persisted auth files still exclude runtime-only external OAuth profiles while active runtime snapshots preserve them, and adds regression coverage for `anthropic:claude-cli`.

## Best Possible Solution

Land the narrow auth-profile store fix once required CI is green, keeping external OAuth material out of persisted files while preserving it in active runtime snapshots.

## Solution Assessment

Yes, the PR takes the narrow maintainable path by separating disk-filtered persistence from runtime snapshot preservation. It keeps inherited main OAuth and runtime-only external OAuth ownership rules intact instead of adding a broader auth reload workaround.
