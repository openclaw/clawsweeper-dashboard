## Summary

Keep this PR open: it is a focused auth-profile bugfix with useful proof, but the patch still refreshes active runtime snapshots from the save input only, so save callers that start from a disk-only/local-update store can still drop runtime-only external OAuth profiles from in-process state.

## What This Changes

The PR changes auth-profile save handling to keep external OAuth profiles out of persisted auth files while adding regression coverage for `anthropic:claude-cli` remaining available in-process after save.

## Best Possible Solution

Refresh active runtime snapshots from the saved local state plus existing or reattached runtime external OAuth profiles for every save path, while keeping disk persistence filtered and inherited-main OAuth exclusions intact.

## Reproduction Assessment

Yes, source-level: current main writes the disk-filtered `localStore` into active runtime snapshots, and the PR still misses save inputs that do not already contain the external profile. I did not run tests because this review is constrained to read-only inspection.

## Solution Assessment

No. The PR fixes the direct save-input path, but the maintainable fix should preserve or reattach runtime external OAuth profiles when the save input is disk-only as well.
