## Summary

Keep this PR open for the normal merge path: the patch addresses the linked auth-provider/session-state bug, I found no blocking correctness finding, and the remaining concern is merge-risk proof/gating rather than a code repair.

## What This Changes

The PR changes auth-profile save/load/merge logic to keep runtime-only external CLI OAuth profiles in in-process snapshots while continuing to filter them from persisted auth files, with regression coverage for that boundary.

## Best Possible Solution

Land the runtime/disk boundary fix after current-head CI or automerge gates pass, keeping runtime-only external OAuth available in memory and absent from persisted auth files.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main filters external profiles into localStore for disk persistence and then installs that same filtered store into the runtime snapshot. I did not run the live gateway/Claude CLI flow in this read-only review.

## Solution Assessment

Yes, this is the right fix shape: it separates the disk-filtered store from the runtime-safe snapshot and adds focused coverage for preservation, removal, and merge semantics. A live external CLI smoke would improve confidence but is not a different implementation direction.
