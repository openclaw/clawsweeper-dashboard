## Best Possible Solution

Keep the protected remediation concept, but make the plan phase metadata-only, move every body/comment read into the approved remediation job, fix the lint failures, and require rollout proof that `secret-remediation` is protected.

## Solution Assessment

No. The protected remediation direction is sound, but the best fix is to keep plan mode metadata-only, move content reads behind the approved remediation job, and prove the environment gate before using write scopes.
