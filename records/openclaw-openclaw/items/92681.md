## Summary

Keep open: this is a protected maintainer/security workflow PR, and the patch has a blocking security-boundary issue because plan mode can fetch full issue, PR, and comment bodies before protected-environment approval.

## What This Changes

The PR adds a secret-scanning remediation workflow and helper that plans remediation, waits on a `secret-remediation` environment, then redacts supported issue/PR bodies or deletes and recreates supported issue comments.

## Best Possible Solution

Keep the protected remediation approach, but make plan mode classify only from alert metadata and move every issue/PR/comment body fetch into the environment-gated remediation job with regression coverage.

## Reproduction Assessment

Yes. Source inspection shows plan mode calls `buildRemediationPlan`, which fetches issue, PR, and comment resources, and live API probes confirm those resources include `body` fields.

## Solution Assessment

No. The protected-environment direction is sound, but the best fix is to keep the plan phase metadata-only and postpone all body/comment reads until the approved remediation job.
