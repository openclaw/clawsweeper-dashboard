## What This Changes

The PR changes Memory Wiki import-insight snippet truncation to use the shared UTF-16-safe truncation helper and adds a regression test for an emoji at the truncation boundary.

## Best Possible Solution

Land the focused Memory Wiki truncation fix after normal maintainer/check gating, keeping the fix at the import-insights source and using the existing plugin SDK text helper.

## Reproduction Assessment

Yes. Current main uses raw `slice(0, maxLength - 1)` in `shortenSentence`, so an astral character crossing that boundary can leave a lone surrogate in `listMemoryWikiImportInsights` output.

## Solution Assessment

Yes. Replacing the single raw truncation point with the existing SDK UTF-16-safe helper is narrower and cleaner than masking replacement glyphs in the UI or duplicating a Memory Wiki-local Unicode helper.
