## Summary

Keep open for maintainer review. Current main still lacks a periodic WhatsApp presence keepalive and still has the 30-minute fallback message timeout, while this focused PR has sufficient real-log proof and no blocking correctness finding from this review.

## Best Possible Solution

Land the focused WhatsApp plugin fix after maintainer review and CI, preferably with a small regression test for the keepalive timer lifecycle and timeout override behavior.

Yes. The patch stays inside the WhatsApp plugin, uses Baileys' existing presence primitive over the authenticated socket, preserves explicit timeout overrides, and avoids a new config surface.
