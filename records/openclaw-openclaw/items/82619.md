The PR changes plugin metadata snapshot memoization to a bounded multi-entry cache, expands stale persisted-registry snapshot caching and invalidation, and adds focused memo regression tests.

## Best Possible Solution

Keep the bounded process-local multi-entry metadata snapshot cache so distinct bootstrap keys stay warm while registry and discovery freshness invalidation remain file-sensitive.

## Solution Assessment

Is this the best way to solve the issue? Yes; a bounded in-process cache keyed by the existing memo key is narrower than changing persistent registry behavior or CLI bootstrap policy.
