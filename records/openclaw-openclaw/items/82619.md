## Summary

Keep open: this is a focused bug-fix PR for a current-main single-slot plugin metadata cache limitation, and the patch shape matches the reported bootstrap timeout mechanism. I found no blocking correctness or security findings in the changed files.

## What This Changes

This PR changes the plugin metadata snapshot memo from one process slot to an 8-entry LRU keyed by the existing memo key, separates registry freshness memo state, and adds focused memo regression tests.

## Best Possible Solution

Land the bounded process-local cache fix after normal CI/package validation, keeping freshness checks and non-persistent plugin metadata policy intact.

## Reproduction Assessment

Yes, with source-level confidence rather than a local timeout run: current main has one memo slot, and the added regression scenario demonstrates why alternating workspace-scoped and unscoped callers rebuild instead of hitting cache. The PR body also supplies real CLI timing proof from a patched beta2 install.

## Solution Assessment

Yes. A small bounded LRU keyed by the existing memo key is the narrow maintainable fix because it avoids a persistent cache, preserves clone-on-hit behavior, and keeps registry freshness validation separate from snapshot entries.
