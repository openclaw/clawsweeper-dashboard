## Summary

Keep open: this is a focused active bug-fix PR. Current main still has the single-slot metadata memo behavior, the branch addresses the multi-key bootstrap path with bounded keyed memoization and focused tests, and I found no blocking correctness or security findings.

## What This Changes

This PR changes the plugin metadata snapshot memo from one process slot to a bounded keyed LRU, adds registry-state memoing for stale-source cases, and expands memo regression tests.

## Best Possible Solution

Land the bounded keyed memo fix after required CI/package validation, while preserving freshness checks and the non-persistent metadata cache policy.

## Reproduction Assessment

Yes, with source-level confidence: current main has one metadata memo slot, and the PR’s regression scenario demonstrates why alternating workspace-scoped and unscoped callers rebuild instead of hitting cache. The PR body also supplies real beta2 CLI timing proof from an equivalent compiled patch.

## Solution Assessment

Yes. A small bounded LRU keyed by the existing memo key is the narrow maintainable fix because it avoids persistent caching, preserves clone-on-hit behavior, and keeps freshness validation tied to registry state.
