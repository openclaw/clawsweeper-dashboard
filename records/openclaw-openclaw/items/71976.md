## Summary

Keep open. Current main still has the reported memory-core behavior, and the remaining work is split across open signal-gate and rehydration threads while the Light ordering and overall promotion contract still need maintainer judgment.

## What This Changes

Ensure memory-core light dreaming and promotion prioritize true recall evidence instead of daily/session freshness when surfacing and promoting short-term recall candidates.

## Best Possible Solution

Define one memory-core promotion contract for true recalls, daily/session freshness, grounded evidence, Light ordering, scoring gates, and rehydration, then update implementation, docs, diagnostics, and regression tests together.

## Reproduction Assessment

Yes, at source level. Current main still refreshes daily/session signals before a recency-first Light sort, still gates Deep ranking/apply on aggregate signalCount, and still drops candidates that fail narrow rehydration; I did not replay the reporter's private memory store.

## Solution Assessment

Unclear as a simple patch. The proposed sort and span changes address real symptoms, but the maintainable fix needs to align with the open gate work at https://github.com/openclaw/openclaw/pull/74114 and the rehydration work at https://github.com/openclaw/openclaw/pull/77761 without regressing current daily/grounded contracts.
