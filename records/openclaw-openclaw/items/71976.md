## Summary

Keep open. Current main still contains the reported Memory Core dreaming behavior, and open PR #74051 is an active implementation candidate with closing syntax, so this issue should wait for that PR or a focused replacement to land.

## What This Changes

The issue requests Memory Core dreaming to prioritize real recall evidence over daily/session freshness, keep non-recall signals out of recall gates and scoring, and widen rehydration matching.

## Best Possible Solution

Memory Core should separate true recall and grounded evidence from daily/session freshness across Light ordering, Deep gates/scoring, and rehydration, then land PR #74051 or a focused replacement with docs and tests.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the mechanism, and the existing test suite explicitly expects repeated daily-only signals with recallCount 0 to clear the default promotion gates.

## Solution Assessment

No for the original sort-only fix. The maintainable fix should address signal-gate semantics, score behavior, Light ordering, and rehydration together; PR #74051 is the active candidate to review or replace.
