## Summary

Keep open. Current main still has the reported Memory Core behavior: Light dreaming ranks by freshness before recall count, daily/session signals refresh `lastRecalledAt` without increasing `recallCount`, Deep promotion still gates on aggregate signal count, and the rehydration search span remains narrow. Open PR #74051 is an active, unmerged implementation candidate with closing syntax, so this issue should wait for that PR or a focused replacement to land.

## What This Changes

The issue asks Memory Core dreaming to prioritize true recall evidence over daily/session freshness and to widen promotion rehydration so valid recalled candidates are not hidden or dropped.

## Best Possible Solution

Memory Core should separate true recall and grounded evidence from daily/session freshness across Light ordering, Deep promotion gates, scoring, and rehydration, with tests and docs locking that contract.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the mechanism, and the existing test suite explicitly expects repeated daily-only signals with `recallCount: 0` to clear the default promotion gates.

## Solution Assessment

No for the original sort-only fix. The maintainable fix should address signal-gate semantics, score behavior, Light ordering, and rehydration together; #74051 is the active candidate to review or replace.
