## Summary

Keep open. Current `origin/main` still has the reported Memory Dreaming behavior, and open PR #74051 is an active implementation candidate with closing syntax, so this issue should wait for that PR or a focused replacement to land.

## What This Changes

The issue reports that Memory Dreaming lets daily/session recency and combined daily/session signal counts hide or filter genuine short-term recall candidates during light dreaming and deep promotion.

## Best Possible Solution

Review and land #74051, or a focused replacement, so real recalls drive Light ordering and default recall-gate eligibility, daily/session signals can still contribute to scoring without satisfying recall gates alone, rehydration handles shifted snippets, and tests/docs define the corrected Memory Dreaming contract.

## Reproduction Assessment

Yes. Source inspection on current `origin/main` reproduces the mechanism, and the existing test suite explicitly expects repeated daily-only signals to clear default promotion gates with `recallCount: 0`.

## Solution Assessment

No for the issue's initial sort-only patch. The maintainable fix should separate recall-gate semantics from daily/session scoring, preserve grounded evidence handling, address rehydration, and update focused tests/docs, which is the direction of open PR #74051.
