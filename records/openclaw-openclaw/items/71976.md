## Summary

Keep open. Current main still has the reported Memory Core mechanisms: light dreaming refreshes daily/session signals before sorting by `lastRecalledAt`, deep promotion still gates on aggregate signal count, and rehydration still has the narrow span/drop path; related narrower issues and PRs cover pieces, but none fully supersedes this issue's light-sort behavior.

## What This Changes

The issue asks Memory Core dreaming promotion to prioritize real recall data over fresh daily/session signals and make rehydration less likely to drop valid candidates.

## Best Possible Solution

Define one Memory Core promotion contract for true recalls, daily/session freshness, grounded evidence, light ordering, scoring gates, and rehydration, then update implementation, docs, and regression tests together.

## Reproduction Assessment

Yes, at source level. Current main still shows the recency-first Light sort after daily/session refresh, aggregate Deep gates, daily-only eligibility tests, and the narrow rehydration/drop path; I did not replay the reporter's private memory store.

## Solution Assessment

Unclear as a simple patch. The proposed sort and span changes target real symptoms, but the durable fix must align with the open gate discussion at https://github.com/openclaw/openclaw/pull/74114 and the rehydration work tracked by https://github.com/openclaw/openclaw/issues/74334.
