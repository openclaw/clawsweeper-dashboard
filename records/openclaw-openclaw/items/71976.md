## Summary

Keep open. Current main still has the reported Memory Core mechanisms: Light dreaming orders by freshness before recall count, daily/session ingestion refreshes entries without increasing real recall count, and Deep promotion still applies `minRecallCount` to aggregate signal count; the earlier implementation PR for this issue was closed unmerged, while related narrower work remains split across https://github.com/openclaw/openclaw/issues/72021 and https://github.com/openclaw/openclaw/issues/74334.

## What This Changes

This issue requests Memory Core dreaming to rank real recall data ahead of daily/session freshness and prevent daily-only signals or narrow rehydration from hiding valid promotion candidates.

## Best Possible Solution

Define and implement a single Memory Core signal contract that separates true recalls, daily/session freshness, and grounded evidence across Light ordering, Deep gates/scoring, rehydration, docs, and tests.

## Reproduction Assessment

Yes, at source level. Current main shows the recency-first Light sort, daily/session timestamp refresh without `recallCount`, aggregate Deep gating, and a regression test that ranks daily-only signals under default gates.

## Solution Assessment

Unclear for the issue as a whole. The sort-first suggestion is too narrow by itself; the maintainable fix should resolve signal semantics with the open gate discussion at https://github.com/openclaw/openclaw/pull/74114 and keep rehydration aligned with https://github.com/openclaw/openclaw/issues/74334.
