## Summary

Keep open. Current main still has the reported Memory Dreaming behavior, and open PR #74051 is an active implementation candidate with closing syntax for this issue, so this should remain open until that PR or a focused replacement lands.

## What This Changes

Memory Dreaming should prioritize genuine recall signals over daily/session ingestion recency, keep daily-only signals out of recall-count gates, and make promotion rehydration robust enough for shifted broad snippets.

## Best Possible Solution

Review and land #74051, or a focused replacement, so real recalls drive Light ordering and recall-gate eligibility, daily/session signals can still contribute to scoring without satisfying recall gates alone, rehydration handles shifted snippets, and tests/docs define the corrected Memory Dreaming contract.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the mechanism: daily/session ingestion refreshes lastRecalledAt, Light sorts by that timestamp before recallCount, and deep/apply gates still use combined signalCount rather than real recallCount.

## Solution Assessment

No for the original sort-only patch. The better solution is a memory-core fix that separates recall-gate semantics from daily/session scoring, preserves intentional grounded evidence, hardens rehydration, and updates focused tests/docs.
