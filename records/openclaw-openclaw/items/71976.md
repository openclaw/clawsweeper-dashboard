## Summary

Keep open. Current main still contains the reported Memory Core dreaming behavior, and open PR #74051 is an active implementation candidate with closing syntax, so this issue should wait for that PR or a focused replacement to land.

## What This Changes

The issue reports Memory Core dreaming promotion mixing daily/session freshness with real recall evidence, causing Light output and Deep promotion to hide true recall candidates.

## Best Possible Solution

Memory Core should separate true recall/grounded evidence from daily/session freshness across Light ordering, Deep promotion gates, scoring, and rehydration, with docs and tests locking that contract.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the mechanism, and the existing test suite explicitly expects repeated daily-only signals with recallCount 0 to clear the default promotion gates.

## Solution Assessment

No for the original sort-only fix. The maintainable fix should address signal-gate semantics, score behavior, Light ordering, and rehydration together; #74051 is the active candidate to review or replace.
