## Summary

Keep open. Current main still contains the reported Memory Core behavior, and open PR #74051 has closing syntax for this issue but is not merged, so this should wait for that implementation or a focused replacement to land.

## What This Changes

The issue asks Memory Core to stop daily/session freshness signals from burying true recall entries in Light dreaming and blocking Deep promotion, and to widen rehydration for moved snippets.

## Best Possible Solution

Memory Core should separate true recall and grounded evidence from daily/session freshness across Light ordering, Deep promotion gates, scoring, and rehydration, with regression tests and docs locking that contract.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the mechanism: Light sorts by freshness first, daily/session signals refresh `lastRecalledAt` while leaving `recallCount` at zero, aggregate signal count can satisfy the Deep gate, and rehydration still uses the narrow span.

## Solution Assessment

No for the original sort-only fix. The maintainable fix should address signal-gate semantics, score behavior, Light ordering, and rehydration together; #74051 is the active candidate to review or replace.
