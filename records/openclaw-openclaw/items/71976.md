## Summary

Keep open. Current main still contains the reported Memory Core dreaming behavior, and open PR #74051 is an active implementation candidate with closing syntax for this issue, so this should not be closed before that PR or a focused replacement lands.

## What This Changes

The issue reports that Memory Core dreaming ranks daily/session freshness ahead of real recall evidence and uses too narrow a rehydration search span, causing valid short-term recall candidates to be hidden or dropped.

## Best Possible Solution

Memory Core should make Light and Deep distinguish true recall or grounded evidence from daily/session freshness, keep daily ingestion as reinforcement rather than a recall gate, broaden or align rehydration, and lock the contract with tests and docs.

## Reproduction Assessment

Yes. Source inspection on current main shows the exact sorting, signal-count, and rehydration-span mechanisms described, and the existing test suite explicitly expects daily-only signals with `recallCount: 0` to pass the default promotion path.

## Solution Assessment

No for the original sort-only fix. The maintainable solution needs to address signal-gate semantics, scoring behavior, and rehydration together; #74051 is the active candidate to review or replace.
