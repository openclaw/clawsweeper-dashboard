## Summary

Keep open. Current main still has the reported Memory Core dreaming behavior, and open PR #74051 is an active implementation candidate with closing syntax, so this issue should remain open until that PR or a focused replacement lands.

## What This Changes

The issue asks Memory Core dreaming to prioritize real recall evidence over daily/session freshness in Light output, keep daily-only signals out of recall-count promotion gates, widen rehydration search, and recalibrate scoring so valid recalled candidates are not hidden.

## Best Possible Solution

Memory Core should separate recall evidence from daily/session freshness: Light output should surface true recalls before daily-only freshness, Deep gates should require real recall or grounded evidence rather than daily-only signals, rehydration should tolerate shifted snippets without promoting stale ranges, and docs/tests should lock in that contract.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the mechanism, and the existing test suite explicitly expects repeated daily-only signals with recallCount 0 to clear the default promotion gates.

## Solution Assessment

No for the original sort-only fix. The maintainable fix needs to address signal-gate semantics, grounded evidence, score behavior, and rehydration together; #74051 is the active open candidate but should be reviewed or replaced before this issue closes.
