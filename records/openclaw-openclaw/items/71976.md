## Summary

Keep open. Current main still has the reported Memory Core behavior, and open PR #74051 is an unmerged implementation candidate with closing syntax, so this issue should wait for that PR or a focused replacement to land.

## What This Changes

The issue asks Memory Core dreaming to prioritize real recall evidence over daily/session freshness in Light output and Deep promotion, and to widen rehydration matching for shifted snippets.

## Best Possible Solution

Land or rework #74051 so Memory Core separates true recall and grounded evidence from daily/session freshness across Light ordering, Deep gates/scoring, and rehydration, with regression coverage and docs updated to match.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the mechanism: daily/session signals refresh freshness without increasing `recallCount`, Light sorts by freshness first, Deep gates on aggregate signal count, and a test explicitly allows daily-only promotion eligibility.

## Solution Assessment

No for the original sort-only fix. The maintainable path is the broader active PR direction: align Light ordering, Deep recall-gate semantics, scoring expectations, and rehydration behavior together.
