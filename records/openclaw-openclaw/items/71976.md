## Summary

Keep open. Current main still has the reported Memory Dreaming mechanisms, and open PR #74051 is an active implementation candidate with closing syntax for this issue, so this should wait for that PR or a focused replacement to land.

## What This Changes

The issue asks to make Memory Dreaming prioritize real recallCount over daily recency, keep daily/session signals out of recall-count gates, and widen or repair promotion rehydration for shifted snippets.

## Best Possible Solution

Review and land #74051, or a focused replacement, so Light selection favors real recalls, daily/session signals can influence scoring without satisfying recall gates alone, rehydration handles shifted snippets correctly, and tests/docs define the corrected Memory Dreaming contract.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the mechanism: daily/session ingestion refreshes lastRecalledAt, Light sorts by that timestamp before recallCount, and deep/apply gates still use combined signalCount instead of real recallCount.

## Solution Assessment

No for the report's initial sort-only fix. The maintainable fix should separate recall-gate eligibility from daily/session scoring, preserve intentional grounded evidence semantics, harden rehydration, and update focused tests/docs.
