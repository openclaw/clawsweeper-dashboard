## Summary

Keep open. Current main still has the reported Memory Core dreaming behavior, and open PR #74051 is an active implementation candidate with closing syntax, so this issue should stay open until that PR or a focused replacement lands.

## What This Changes

The issue asks Memory Core dreaming to prioritize true recalls over daily/session recency, keep daily-only signals out of recall-count promotion gates, and widen/repair candidate rehydration so valid short-term recalls can promote.

## Best Possible Solution

Review and land #74051, or a focused replacement, so Light ordering prefers real recalls, default promotion/apply gates require recall or grounded evidence rather than daily-only signals, rehydration handles shifted snippets, and docs/tests define the corrected Memory Dreaming contract.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the mechanism, and the existing test suite explicitly expects repeated daily-only signals with recallCount 0 to clear default promotion gates.

## Solution Assessment

No for the original sort-only suggestion. The maintainable solution should separate recall-gate semantics from daily/session scoring, preserve grounded evidence, address rehydration, and update focused docs/tests, which is the direction of open PR #74051.
