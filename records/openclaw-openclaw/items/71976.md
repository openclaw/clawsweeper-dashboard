## Summary

Keep open. Current main still matches the reported memory-core behavior: Light sleep refreshes daily/session signals before a recency-first sort, Deep promotion still gates on aggregate signal counts, and rehydration still drops candidates that cannot be relocated with the narrow strict comparator.

## What This Changes

The issue asks memory dreaming to prioritize real recall signals over refreshed daily/session freshness and to avoid promotion failures from narrow rehydration and mixed signal gates.

## Best Possible Solution

Define one memory-core promotion contract for true recalls, daily/session freshness, grounded evidence, Light ordering, scoring gates, and rehydration, then update implementation, docs, diagnostics, and regression tests together.

Yes, at source level. Current main still refreshes daily/session signals before a recency-first Light sort, still gates Deep ranking/apply on aggregate signalCount, and still drops candidates that fail narrow rehydration; I did not replay the reporter's private memory store.

## Solution Assessment

Unclear as a simple patch. The proposed sort and span changes address real symptoms, but the maintainable fix needs to align with the open gate and rehydration work without regressing current daily/grounded contracts.
