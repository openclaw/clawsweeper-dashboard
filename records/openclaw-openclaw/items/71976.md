## Summary

Keep open. Current main and the latest release still show the reported Light recency-first selection, aggregate signal gates, and bounded rehydration span; related open work covers pieces, but no merged or canonical item fully owns the Light ordering plus promotion semantics.

## What This Changes

The issue reports that memory-core Light dreaming sorts by recency before recallCount, mixes daily/session signals into recall promotion gates and scoring, and uses a narrow rehydration range that can drop valid promotion candidates.

## Best Possible Solution

Have memory-core owners decide the contract for daily/session/grounded signals versus real recalls, then update Light ordering, promotion gates/scoring, and rehydration behavior with focused regression coverage.

## Reproduction Assessment

Yes, from source: run Light dreaming with fresher daily-ingested entries and older high-recall entries and the current sort chooses lastRecalledAt before recallCount. Current tests also source-reproduce the aggregate daily-signal promotion gate with recallCount 0.

## Solution Assessment

Unclear: the suggested recall-first sort is plausible for the Light report, but it does not resolve the aggregate gate, score denominator, or rehydration span questions. The best fix needs a maintainer decision on signal semantics before code changes.
