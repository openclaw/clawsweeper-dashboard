## Summary

Keep open. Current main and v2026.5.19 still show the reported Light recency-first selection, aggregate signal gates, and narrow rehydration/drop path; related open threads cover pieces, but no merged fix or narrower canonical item fully owns the Light ordering plus promotion contract.

## What This Changes

Issue requests that memory dreaming surface real recall signals over daily/session freshness in Light and Deep promotion paths, and avoid dropping valid candidates during rehydration.

## Best Possible Solution

Define one memory-core promotion contract for true recalls, daily/session freshness, grounded evidence, Light ordering, scoring gates, and rehydration, then update implementation, docs, diagnostics, and regression tests together.

## Reproduction Assessment

Yes, at source level. Current main still refreshes daily/session signals before a recency-first Light sort, still gates Deep ranking/apply on aggregate signalCount, and still drops candidates that fail narrow rehydration; I did not replay the reporter's private memory store.

## Solution Assessment

Unclear as a simple patch. The proposed sort and span changes address real symptoms, but the maintainable fix needs to align the open gate work at https://github.com/openclaw/openclaw/pull/74114 and rehydration work at https://github.com/openclaw/openclaw/pull/77761 without accidentally reversing current daily/grounded contracts.
