## Summary

Keep open. Current main still has the reported Memory Dreaming behavior, and open PR #74051 uses closing syntax for this issue, so the issue should remain open until that PR or a focused replacement lands.

## What This Changes

Fix Memory Dreaming so real recall signals drive Light ordering and promotion eligibility, daily/session signals cannot hide recalled entries, and broad shifted snippets can be rehydrated reliably.

## Best Possible Solution

Review and land #74051, or a focused replacement, so Light prioritizes real recalls, daily/session signals can contribute to scoring without satisfying recall gates alone, rehydration handles shifted snippets, and tests/docs define the corrected Memory Dreaming contract.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the mechanism: daily/session ingestion refreshes lastRecalledAt, Light sorts by that timestamp before recallCount, and rank/apply gates still use combined signalCount rather than real recallCount.

## Solution Assessment

No for the original sort-only fix. The maintainable fix should separate recall-gate semantics from daily/session scoring, preserve intentional grounded evidence, widen or improve rehydration, and update focused tests/docs.
