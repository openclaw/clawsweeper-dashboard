- `impact:session-state`: The issue concerns short-term memory recall state, dreaming promotion ranking, and durable memory promotion drift.

## Summary

Keep open. Current main and the latest stable tag still match the reported memory-core behavior: Light dreaming ingests daily/session signals before a recency-first sort, Deep promotion still gates on aggregate signal counts, and rehydration still drops candidates that cannot be relocated by the narrow strict matcher. Related open items cover the signal-gate and rehydration pieces, but they do not fully supersede the Light ordering and overall promotion-contract question.

## What This Changes

The issue reports that memory-core dreaming promotion hides true recall signals because Light ordering prefers freshness over recall count, Deep promotion mixes daily/session signals with true recalls, and rehydration searches too narrowly.

## Best Possible Solution

Define one memory-core promotion contract for true recalls, daily/session freshness, grounded evidence, Light ordering, scoring gates, and rehydration, then update implementation, docs, diagnostics, and regression tests together.

## Reproduction Assessment

Yes, at source level. Current main still refreshes daily/session signals before a recency-first Light sort, still gates Deep ranking/apply on aggregate signalCount, and still drops candidates when the narrow rehydration matcher cannot relocate them; I did not replay the reporter's private memory store.

## Solution Assessment

Unclear as a simple patch. The proposed sort and span changes address real symptoms, but the maintainable fix needs maintainer agreement on recall vs daily vs grounded signal semantics and coordination with https://github.com/openclaw/openclaw/pull/74114 and https://github.com/openclaw/openclaw/pull/77761.
