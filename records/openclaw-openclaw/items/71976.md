The issue asks memory-core dreaming to prioritize true recall data over daily/session recency and to stop daily-only signals from hiding or displacing valid short-term recall candidates.

## Best Possible Solution

Keep this open as the broad memory-core dreaming signal-policy bug, settle the intended recall/daily/grounded promotion contract, then land one focused fix that aligns light ordering, rank/apply gates, diagnostics, docs, and regression tests.

## Solution Assessment

Unclear as a one-line fix. Recall-first sorting is narrow, but the promotion-gate change needs maintainer direction because current tests intentionally let repeated daily-only signals pass default gates.
