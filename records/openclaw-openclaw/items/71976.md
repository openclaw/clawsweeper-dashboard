## Summary

Keep open. Current main still matches the reported memory-core behavior: Light dreaming refreshes daily/session signals before a recency-first sort, Deep promotion still gates on aggregate signal counts, and rehydration still has the narrow/drop path; related open PRs cover parts of the cluster but do not settle the full promotion contract.

## What This Changes

The issue asks memory-core dreaming promotion to prioritize genuine recall data over daily/session freshness and to avoid dropping valid promotion candidates during rehydration.

## Best Possible Solution

Define one memory-core promotion contract for true recalls, daily/session freshness, grounded evidence, Light ordering, scoring gates, and rehydration, then update implementation, docs, diagnostics, and regression tests together.

## Reproduction Assessment

Yes, at source level. Current main still refreshes daily/session signals before a recency-first Light sort, still gates Deep ranking/apply on aggregate signalCount, and still drops candidates that fail narrow rehydration; I did not replay the reporter's private memory store.

## Solution Assessment

Unclear as a simple patch. The proposed sort and span changes address real symptoms, but the maintainable fix must align with https://github.com/openclaw/openclaw/pull/74114 and https://github.com/openclaw/openclaw/pull/77761 without accidentally retiring current daily or grounded promotion contracts.
