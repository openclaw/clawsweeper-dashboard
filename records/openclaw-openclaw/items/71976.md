## Summary

Keep open. Current main still has the reported source-level mechanisms, and the remaining work is not terminally superseded because light ordering, aggregate promotion semantics, and rehydration normalization are split across open follow-ups without a merged fix.

## What This Changes

The issue asks Memory Core dreaming to surface real recalled entries ahead of daily/session freshness, keep daily-only signals from hiding recall data in promotion gates/scoring, and fix rehydration for shifted snippets.

## Best Possible Solution

Define one Memory Core promotion contract that separates true recalls, daily/session freshness, and grounded evidence, then align Light ordering, Deep gates/scoring, rehydration normalization, docs, and tests to that contract.

## Reproduction Assessment

Yes, at source level. Current main shows the recency-first Light sort, daily/session timestamp refresh without recallCount, aggregate Deep gating, and whitespace-only rehydration against richer daily-ingestion snippets.

## Solution Assessment

Unclear as a one-line sort or recallCount-only gate. The maintainable fix should resolve signal semantics with the open gate PR and align rehydration with the open normalization reports while preserving the live-source safety guard.
