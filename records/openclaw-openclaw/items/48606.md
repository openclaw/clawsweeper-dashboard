## Summary

Keep open: the reported macOS Edge default-browser bug is still source-reproducible on current main, and this PR is directionally right but not merge-ready because it still lacks focused regression tests and required after-fix real macOS behavior proof.

## What This Changes

The PR changes the browser plugin’s macOS default Chromium detection to fall back from recognized bundle IDs to known system and user Applications executable paths when osascript/defaults resolution fails.

## Best Possible Solution

Land a browser plugin fix that preserves dynamic app resolution first, then falls back to known existing system/user Applications executables for recognized Chromium bundle IDs, with focused regression tests and redacted macOS `openclaw browser status --json` proof.

## Reproduction Assessment

Yes, by source inspection: current main reads `com.microsoft.edgemac`, returns `null` when osascript/defaults resolution fails, then checks Chrome before Edge in the generic macOS fallback. The existing no-app-path test also encodes Chrome as the current result.

## Solution Assessment

Yes in direction, but not merge-ready. A known bundle-ID path fallback is a narrow maintainable fix for this failure mode, but this branch still needs regression coverage, contributor real behavior proof, and fresh validation.
