## Best Possible Solution

Land a rebased browser plugin fix that keeps dynamic app resolution first, falls back to known existing system/user Applications executables for recognized Chromium bundle IDs, and includes focused regression tests plus redacted macOS status proof.

## Solution Assessment

Yes in direction, but not merge-ready. A known bundle-ID path fallback is a narrow maintainable fix for this failure mode, but this branch still needs regression coverage, contributor real behavior proof, and a clean rebase.
