## Summary

Keep open: current main still does not implement the known-path fallback, and this PR is a useful narrow repair, but it should not merge without focused regression coverage and the required changelog entry.

## What This Changes

The PR adds a macOS known bundle-ID-to-executable fallback for Chromium-based default browsers when dynamic `osascript`/`defaults` path resolution fails.

## Best Possible Solution

Land a rebased browser-plugin fallback that keeps dynamic macOS app resolution first, falls back only for recognized Chromium bundle IDs with existing system or user Applications executables, and covers both Edge-present and Edge-missing cases in tests plus changelog.

## Reproduction Assessment

Yes, by source inspection: mock LaunchServices as `com.microsoft.edgemac`, make `osascript`/`defaults` fail, and make both Edge and Chrome paths exist; current main returns `null` from default detection and then selects Chrome from the generic fallback. I did not run the test because this was a read-only review.

## Solution Assessment

Yes in direction, but not as-is. A known bundle-ID path fallback is the narrow maintainable fix, but the PR still needs regression tests, changelog coverage, and rebase validation before merge.
