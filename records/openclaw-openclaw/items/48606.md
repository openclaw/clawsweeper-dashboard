## What This Changes

The PR adds a macOS known browser bundle-ID to executable-path fallback in the browser plugin’s Chromium default-browser resolver when `osascript` or `defaults` path resolution fails.

## Best Possible Solution

Land a rebased browser plugin fix that keeps dynamic macOS app resolution first, falls back to existing system/user Applications executables for recognized Chromium bundle IDs, and includes focused regression tests plus redacted macOS status proof.

## Reproduction Assessment

Yes, source-reproducible. Current main recognizes `com.microsoft.edgemac`, returns `null` when `osascript` or `defaults` resolution fails, and then the generic macOS fallback checks Chrome before Edge.

## Solution Assessment

Yes in direction, but not merge-ready. A known bundle-ID path fallback is a narrow maintainable fix for this failure mode, but this branch still needs focused tests, real macOS proof, and a clean rebase.
