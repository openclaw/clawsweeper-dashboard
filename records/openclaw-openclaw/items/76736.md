## Summary

Keep open. Current main has a beta-first stable promotion policy and release validation lanes, but it still exposes only stable/beta/dev update channels and does not define the requested LTS-style or stricter production-stable support policy.

## Best Possible Solution

Define a maintainer-owned release/support policy, then align docs, update-channel semantics, publish workflows, and release checks with that policy while keeping concrete Telegram regressions tracked separately.
