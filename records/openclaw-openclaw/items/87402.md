## Summary

## What This Changes

The issue asks OpenClaw to treat the managed LaunchAgent gateway listener as the restart target, recover embedded Codex direct sessions from stale compaction/rebind state, and show wedged agent sessions distinctly from gateway/channel health.

## Best Possible Solution

Define a maintainer-owned scope that first fixes managed LaunchAgent restart semantics for gateway-descendant callers, then either proves or tightens embedded Codex stale-binding recovery and adds a status surface for wedged sessions.
