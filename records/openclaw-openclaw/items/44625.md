## Summary

Keep open. Current main still only tries Slack's legacy `auth.scopes` and `apps.permissions.info` methods for capability scope discovery, so the reported `unknown_method | unknown_method` path remains possible for modern `xoxb-*` bot tokens; the related fix PR was closed without merging and no open replacement PR was found.

## What This Changes

The issue requests that Slack channel capability discovery report scopes successfully for modern Slack bot tokens instead of surfacing `unknown_method` errors from legacy Slack scope APIs.

## Best Possible Solution

Fix scope discovery inside the Slack plugin by adding a modern bot-token path, preferably `auth.test` plus `response_metadata.scopes`/`acceptedScopes` from the Slack SDK, while preserving legacy workspace-app fallbacks and adding focused tests for fallback behavior and capability diagnostics.

## Solution Assessment
