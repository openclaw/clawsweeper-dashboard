## Summary

Close as implemented on current main. Slack capability diagnostics now use `auth.test` response metadata before legacy scope APIs, which addresses the reported modern `xoxb-*` bot-token `unknown_method` failure; the fix is present on main but not in the latest shipped `v2026.4.29` release.

## What This Changes

Slack capability scope discovery should work for modern Slack bot tokens instead of failing after `auth.scopes` and `apps.permissions.info` return `unknown_method`.

## Best Possible Solution

Keep the Slack plugin-local `auth.test` metadata implementation on main and let the next OpenClaw release ship it.

## Reproduction Assessment

Yes. The issue has concrete CLI steps, the latest release source still shows the two-legacy-method failure path, and current main has a focused regression test for the modern bot-token `auth.test` recovery path.

## Solution Assessment

Yes. A Slack plugin-local scope discovery change is the narrowest maintainable solution; it relies on the Slack SDK’s documented header-to-`response_metadata.scopes` behavior and does not need a core channel API change.
