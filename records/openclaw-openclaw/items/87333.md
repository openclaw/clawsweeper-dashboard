## Summary

Keep this PR open: current main still has the source-reproducible channel-registry overwrite path, the PR is focused, proof is sufficient, and no blocking patch defect was found. It should stay in maintainer review because the change sits on mutable global registry state used by channel delivery.

## What This Changes

The PR adds a channel-only guard so empty tool-discovery registries update the active registry but do not overwrite the pinned channel registry, plus regression tests for channel and http-route pin behavior.

## Best Possible Solution

Land this PR or an equivalent small guard after maintainer review, preserving active registry updates for tool discovery while skipping only empty channel-surface pin replacement.

## Reproduction Assessment

Yes, source-reproducible. Current main traces tools.catalog into a channel-surface, activate:false, toolDiscovery:true standalone registry load, and current installStandaloneRegistry can pin an empty channel registry over the startup-pinned registry.

## Solution Assessment

Yes. Guarding installStandaloneRegistry is narrower than changing the tools.catalog surface or adding outbound fallback behavior, because it preserves active registry updates for tool discovery and changes only empty channel pin replacement.
