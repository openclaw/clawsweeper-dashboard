## Summary

Keep open: the force-pushed patch addresses the earlier code-shape concerns by timing the upload step only and leaving interactive replies on the existing untimed path, but this external PR still has mock-only proof and the 35s fail-fast behavior needs live Zalo validation before merge.

## What This Changes

The branch adds an optional zalouser `sendTimeoutMs`, applies a 35s timeout to direct outbound media sends, routes timed non-GIF media through `uploadAttachment` plus a custom uploaded-attachment send, and adds mocked regression coverage and zca-js result typing.

## Best Possible Solution

Land a narrow zalouser fix that either delivers non-interactive media through the valid live upload path or fails boundedly with maintainer-approved semantics, backed by live Zalo proof and without changing the working interactive reply path.

## Reproduction Assessment

Yes, at source level. The linked issue gives concrete live steps, and current main routes non-interactive zalouser media through zca-js upload/send paths where zca-js waits on upload callbacks for non-image attachments; I did not independently run a live Zalo account.

## Solution Assessment

Unclear as a final fix. Timing only the upload-callback wait is a cleaner mitigation than the previous full-send race, but maintainers still need to decide whether bounded failure is sufficient for the linked delivery issue or whether non-interactive sends should route through a live session and succeed.
