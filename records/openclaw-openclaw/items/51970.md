## Summary

Keep open: current main still has the credential-exposure surface, but this PR should not merge as-is because it increases disclosure for 5- and 6-character secrets and still lacks real `/models list` or chat-output proof.

## What This Changes

The PR changes `maskApiKey` to reveal only a leading prefix and updates the helper tests for that masking policy.

## Best Possible Solution

Adopt a masking policy that removes suffix exposure without increasing disclosed material for short secrets, then cover the helper and user-visible auth output surfaces with focused tests and real behavior proof.

## Reproduction Assessment

Yes. Source comparison is enough: current main keeps first/last masking, while the PR diff changes 5- and 6-character values to `abcd...`, increasing disclosed material for that length range.

## Solution Assessment

No. Removing suffix exposure is the right direction, but the catch-all `> 4` branch is not the safest fix because it reveals most short credentials; use constant redaction or a dedicated short-secret branch.
