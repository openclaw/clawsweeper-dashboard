## Summary

## What This Changes

The PR changes `maskApiKey` and its tests to show one leading character for keys up to four characters and a four-character prefix for longer keys.

## Best Possible Solution

Adopt a masking policy that removes suffix exposure without increasing visible material for any credential length, cover boundary behavior in tests, and attach redacted real CLI or chat output.

## Reproduction Assessment

Yes, by source comparison. Current main expects `short` to mask as `s...t`, while this branch would return `shor...` through the helper used by CLI and chat-visible auth labels.

## Solution Assessment

No. Removing suffix exposure is the right direction, but this implementation increases visible material for short credentials; a short-value guard or constant non-secret marker is safer.
