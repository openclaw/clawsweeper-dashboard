## Summary

Keep open: the credential-exposure problem is valid, but this PR still introduces a short-secret disclosure regression and needs a small test cleanup plus the required changelog entry before merge.

## What This Changes

This PR changes `maskApiKey` to stop showing key suffixes, expose one leading character for values up to four characters and four leading characters for longer values, and updates the unit tests.

## Best Possible Solution

Adopt a credential-display policy that never shows suffixes and never reveals more short-secret material than current main, then cover the helper and auth-output surfaces with tests plus a changelog entry.

## Reproduction Assessment

Yes. From source comparison, `abcde` changes from current `a...e` to proposed `abcd...`, and `abcdef` changes from `a...f` to `abcd...`, so the PR regression is high-confidence without needing to run the branch.

## Solution Assessment

No. Removing suffix exposure is the right direction, but the catch-all `> 4` branch is not the best fix because it reveals most of 5- and 6-character secrets; use a safer short-secret branch or constant mask.
