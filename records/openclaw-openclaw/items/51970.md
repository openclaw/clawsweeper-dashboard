## Summary

Keep open: this is a focused credential-display hardening PR, but the current branch still has a blocking short-secret disclosure regression and lacks required after-fix real behavior proof. Current main has not implemented the requested helper change, and the related full-redaction attempts were closed unmerged rather than shipped.

## What This Changes

The PR changes `maskApiKey` to reveal only a leading prefix plus ellipsis and updates the focused helper tests for that masking policy.

## Best Possible Solution

Adopt a masking policy that removes suffix exposure without increasing disclosure for short secrets, then cover the helper and user-visible auth output surfaces with focused tests and redacted real behavior proof.

## Reproduction Assessment

Yes. Source comparison is enough for the blocking finding: current main returns `a...e` and `a...f` for 5- and 6-character inputs, while the PR would return `abcd...`; the helper is used in auth display output.

## Solution Assessment

No. Removing suffix exposure is the right direction, but the catch-all `> 4` branch is not the safest fix because it increases disclosure for short secrets; a short-secret branch or constant redaction is safer.
