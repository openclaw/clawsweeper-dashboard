## Summary

Keep open: current main and v2026.6.9 still expose `sessions_yield` as a subagent-oriented turn-ending primitive with no cron-completion wait selector or general parent-session wake. The docs-only closing PR and the broader runtime PR are both still open, so this issue still needs maintainer product-contract handling before cleanup closure.

## What This Changes

Clarify or change `sessions_yield` and isolated cron completion behavior so agents do not wait on an unsupported cron-completion wake path.

## Best Possible Solution

Keep this issue open until maintainers decide whether the accepted outcome is docs/tool-text clarification, an explicit cron wait/wake API, or the broader yielded-parent lifecycle work.

## Reproduction Assessment

## Solution Assessment

Unclear. A docs/tool-description change is the lowest-risk mitigation, but adding cron completion resume semantics would create a new product/runtime contract that maintainers need to choose explicitly.
