## Summary

Keep open: current main and the latest release still have the broad `sessions_yield` wording and no isolated-cron completion wake, while an open draft PR owns the wording mitigation and a separate RFC owns the opt-in runtime API decision.

## What This Changes

Clarify or change `sessions_yield` and isolated cron semantics so a yielded session is not left waiting for a cron completion event that never wakes it.

## Best Possible Solution

Land a narrow wording fix if maintainers accept docs-only mitigation, keep the opt-in cron completion wake/wait decision in https://github.com/openclaw/openclaw/issues/96190, and avoid automatic wakes for every isolated cron completion.

## Reproduction Assessment

Yes. Source inspection shows `sessions_yield` ends the current turn without triggering a new turn, and detached isolated cron completion returns through `runIsolatedAgentJob` without a parent-session enqueue or heartbeat wake.

## Solution Assessment

Unclear as a final product solution. Docs/tool-text clarification is the lowest-risk mitigation, but an explicit opt-in cron completion wake or wait API needs maintainer product approval before implementation.
