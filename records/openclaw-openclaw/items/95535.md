## Summary

Keep open: current main and v2026.6.9 still leave `sessions_yield` described as a subagent-oriented yield primitive, while isolated cron completion remains intentionally separate from general parent-session wake behavior. The new open draft PR is the active narrow docs/tool-text path, and the broader cron wait/resume behavior still needs maintainer product direction.

## What This Changes

Request to clarify that `sessions_yield` does not resume a main session for isolated cron completion, or define an explicit cron-completion wait/wake contract.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/95569 for the narrow tool-text clarification, then track any opt-in cron completion wait/resume API separately if maintainers want runtime support.

## Reproduction Assessment

Yes at source level. `sessions_yield` records a hidden yielded context without triggering a turn, while detached isolated cron completion is tested to avoid main-session event/heartbeat wake behavior.

## Solution Assessment

Unclear until maintainers choose the product contract. A tool-description clarification is the lowest-risk immediate fix, while a wait selector or cron completion wake is a new runtime API that should be designed separately.
