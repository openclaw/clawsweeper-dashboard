## Label Justifications

- `impact:session-state`: The affected behavior is whether a yielded main session receives or remains stuck waiting for isolated cron completion state.

## Summary

Keep open: current main and the latest release still treat `sessions_yield` as a cooperative end-turn primitive, and isolated cron completion is not a general parent-session wake contract. The useful next step is a maintainer choice between clarifying the model-facing docs/tool text and adding an explicit cron completion notification API.

## What This Changes

The issue asks OpenClaw to clarify or change `sessions_yield` behavior when a main session waits for an isolated cron run to complete.

## Best Possible Solution

Clarify `sessions_yield` as a subagent/follow-up-event turn-ending primitive and document `cron.runs` or CLI `--wait` for cron completion waits; add main-session cron completion wake behavior only as an explicit designed contract.

## Reproduction Assessment

Yes. Source inspection shows `sessions_yield` records a yielded turn and needs a later event, while isolated cron completion does not generally request an immediate parent-session wake.

## Solution Assessment

Unclear. A docs/tool-description clarification is the narrowest low-risk solution, while a runtime wake or wait selector would create a new cron/session contract that maintainers need to choose explicitly.
