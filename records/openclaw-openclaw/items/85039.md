## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Normalize CRLF markdown before scanning blocks:** `extensions/zalouser/src/outbound-format.ts:69`
  - body: `normalizeZalouserOutboundText` splits only on `\n`, while the existing markdown parser accepts CRLF by normalizing `\r\n?` first. With input like `- a\r\n\r\n- b` or `---\r\n`, the retained `\r` prevents the HR/list/fence matchers from firing, so Windows-style agent output still reaches Zalo with the broken spacing or literal HR markers. Normalize CRLF before this scan or make the matchers consume `\r`, and add regression coverage.
  - confidence: 0.88
- **[P3] Include plus list markers in the collapse rules:** `extensions/zalouser/src/outbound-format.ts:142-145`
  - body: The current zalouser markdown parser treats `+ item` as an unordered list, but the new blank-line collapse regexes only match numbered, `-`, and `*` items. A `+` list separated by blank lines will still render fragmented in Zalo, so the normalizer should include `+` in the shared list-item pattern and test that case.
  - confidence: 0.82
