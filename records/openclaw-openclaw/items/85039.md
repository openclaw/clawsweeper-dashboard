Full review comments:

- **[P1] Preserve default plain sends unless markdown is explicit:** `extensions/zalouser/src/send.ts:45`
  - body: `sendMessageZalouser` currently treats absent `textMode` as plain and the existing test suite says default sends stay literal. This change still normalizes default/plain sends when no manual styles are present, so direct `zalouser` tool sends can lose literal `---`, `***`, or intentional blank-line list spacing. Gate this normalizer to `options.textMode === "markdown"` or another explicit rendering path.
  - confidence: 0.9
- **[P2] Match fenced-block closers to the opener:** `extensions/zalouser/src/outbound-format.ts:77-82`
  - body: The normalizer flips `inFence` on any 3+ backtick or tilde line, so a backtick fence containing a `~~~` line is treated as closed and later code lines such as `---` can be stripped as prose. Track the opener marker and length, matching the existing `text-styles.ts` fence rules, and add mixed-marker/short-closer regressions.
  - confidence: 0.88
