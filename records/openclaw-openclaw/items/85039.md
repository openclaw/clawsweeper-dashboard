## Best Possible Solution

Preserve plain/default send compatibility, apply normalization only on explicit markdown or agent-rendering paths, and make the normalizer opener-aware before merging.

## Reproduction Assessment

Yes for the review findings: source inspection shows default/plain sends now pass through the normalizer, and mixed-marker fences can leave fence mode early before `---` stripping runs. ## Solution Assessment

No: the rendering direction is useful, but the patch should preserve plain-mode compatibility and use opener-aware fence matching before it rewrites outbound text. The existing markdown send path and parser provide a narrower maintainable pattern to follow.
