## Summary

Keep open for maintainer review: current main still lacks the unreadable-config write blocker, and this PR is a focused, well-proven fix, but changing doctor/update writes to fail closed with `CONFIG_WRITE_REJECTED` is an intentional compatibility and operator-behavior decision.

## What This Changes

The branch records config read failures on `ConfigFileSnapshot`, blocks full-file config writes from unreadable base snapshots, and adds EACCES write-guard regression tests.

## Best Possible Solution

Land the scoped config IO fail-closed guard after maintainer acceptance of the operator behavior, while keeping sudo/wrong-UID ownership policy tracked separately.

## Reproduction Assessment

Yes at source level: current main catches EACCES into an invalid fallback snapshot, and later config writes have no unreadable-base blocker. I did not run a live sudo/systemd or LaunchAgent reproduction during this read-only review.

## Solution Assessment

Yes, with a maintainer compatibility decision: recording the read failure on the config snapshot and blocking at the existing config IO write gate is the narrow owner-boundary fix for the clobber path. The wrong-UID prevention policy is separate work.
