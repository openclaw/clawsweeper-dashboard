## Summary

Keep open: current main still lacks full returned tool-result payload preservation, but this PR needs changes before merge because it can suppress timeout error payloads and has only mocked test proof.

## What This Changes

The PR buffers embedded Pi `onToolResult` `ReplyPayload`s, replays them into non-streamed final run payloads, resets the buffer on compaction retry, and adds focused regression tests plus a changelog entry.

## Best Possible Solution

Land a rebased fix that preserves full non-streamed `ReplyPayload`s while keeping current timeout error plus tool-media behavior, backed by focused tests and redacted real-run proof.

## Reproduction Assessment

Yes, at source level. Current main has a media-only attempt fallback but no full `toolResultPayloads` return contract, while the PR's mocked path demonstrates the dropped returned-payload shape; I did not run tests because this review is read-only.

## Solution Assessment

No, not in the current head. Buffering full `ReplyPayload`s in the attempt layer is the right seam, but this branch should integrate with current `payloadsWithToolMedia` timeout handling instead of letting buffered tool results hide timeout failures.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Keep timeout errors when replaying tool results:** `src/agents/pi-embedded-runner/run.ts:1523`
