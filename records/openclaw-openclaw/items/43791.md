## Summary

Keep open: this is a plausible bug fix, and the author already addressed earlier review gaps around singular `mediaUrl`, metadata preservation, streamed duplicate suppression, and compaction retry reset, but the latest branch still has blocking correctness issues and only mocked-test proof.

## What This Changes

The PR buffers embedded `ReplyPayload` tool results, prepends them to final embedded Pi run payloads for non-streaming callers, resets the buffer on compaction retry, and adds regression tests plus a changelog entry.

## Best Possible Solution

Rebase the payload preservation onto current main by making buffered tool-result payloads optional/defaulted, replaying only non-streamed final deliverables, and preserving explicit timeout errors for tool-only summaries.

## Reproduction Assessment

Yes, by source inspection. Current main preserves pending tool media but still drops non-streamed arbitrary `ReplyPayload` text/channel metadata because the callback path is gated on `params.onToolResult`.

## Solution Assessment

No as-is. The direction is useful, but the branch is not the narrowest maintainable fix until it keeps the harness result contract backward-compatible and avoids treating buffered tool summaries as successful timeout responses.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Default missing tool-result payload arrays:** `src/agents/pi-embedded-runner/run.ts:1520`
  - body: `AgentHarnessAttemptResult` is the plugin harness result type, and existing or third-party harnesses can return results without the new field. When `params.onToolResult` is absent, this spread reads `undefined`, so the run crashes instead of returning assistant payloads; make the field optional and default it to `[]` before spreading.
  - confidence: 0.86
- **[P2] Preserve timeout errors for tool-only summaries:** `src/agents/pi-embedded-runner/run.ts:1520`
  - body: Buffered tool-result text/metadata is folded into `payloads` before the timeout branch, so a run that emits only a tool summary and then times out skips the explicit timeout error and looks successful. Gate timeout handling on assistant/media deliverables, or append buffered summaries after the timeout error path.
  - confidence: 0.78
