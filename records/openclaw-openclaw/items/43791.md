## Summary

Keep open: this PR addresses a real embedded-run payload gap that current main still only covers for tool media, but the current head is conflicting, lacks real behavior proof, and can suppress explicit timeout errors when buffered tool-result payloads exist.

## What This Changes

The PR buffers embedded Pi `ReplyPayload` tool results in `runEmbeddedAttempt`, returns them before assistant payloads for non-streaming callers, resets the buffer on compaction retry, and adds regression tests plus a changelog entry.

## Best Possible Solution

Land a rebased fix that preserves full non-streamed tool-result `ReplyPayload`s while integrating with current tool-media merge and always preserving explicit prompt-timeout errors.

## Reproduction Assessment

Yes, at source level. Current main lacks the full `toolResultPayloads` return contract, and the PR head has a clear source-level timeout regression when a timed-out attempt has buffered tool-result payloads.

## Solution Assessment

No, not at the current PR head. Buffering at the attempt layer is plausible, but it must be rebased onto current `payloadsWithToolMedia` handling and preserve explicit timeout errors.
