## Summary

Keep open: current main only covers a narrower media fallback, while this PR still has unique full `ReplyPayload` preservation work, but the branch is not mergeable as-is because it can suppress timeout errors and lacks real behavior proof.

## What This Changes

The PR buffers embedded Pi `onToolResult` `ReplyPayload`s, returns them before assistant payloads for non-streamed callers, resets the buffer on compaction retry, and adds regression tests plus a changelog entry.

## Best Possible Solution

Land a rebased fix that preserves non-streamed full `ReplyPayload`s while keeping current media fallback and timeout behavior, backed by focused tests and redacted real-run proof.

## Reproduction Assessment

Yes, at source level. Current main lacks the full `toolResultPayloads` return contract, and the PR head also has a clear source-level timeout regression when a timed-out attempt has buffered tool-result payloads.

## Solution Assessment

No, not at the current PR head. Buffering at the attempt layer is a plausible seam, but the branch must be rebased onto current `payloadsWithToolMedia` handling and must preserve explicit timeout errors.

## Review Findings

Overall correctness: patch is incorrect
