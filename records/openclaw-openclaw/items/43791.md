## Summary

Keep this PR open: the bug is still source-visible on current main, but the branch needs a rebase, real behavior proof, and a timeout-regression fix before merge.

## What This Changes

The branch buffers embedded Pi `onToolResult` `ReplyPayload`s, returns them ahead of assistant payloads for non-streamed callers, resets the buffer on compaction retry, and adds regression tests plus a changelog entry.

## Best Possible Solution

Land a rebased fix that preserves non-streamed full `ReplyPayload`s without regressing current timeout errors or the media artifact fallback, backed by focused tests and redacted real-run proof.

## Reproduction Assessment

Yes, at source level. Current main has a media-only attempt fallback but no full `toolResultPayloads` return contract, while the PR's mocked path demonstrates the dropped returned-payload shape; I did not run tests because this review is read-only.

## Solution Assessment

No, not at the current PR head. Buffering at the attempt layer is a plausible seam, but the branch prepends buffered payloads before the old timeout guard and needs to be rebased onto the current `payloadsWithToolMedia` path.
