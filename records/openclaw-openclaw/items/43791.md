## Summary

Keep this PR open. Current main still lacks the full embedded tool-result payload preservation contract this PR targets, but the current branch is not ready because buffered tool-result payloads can mask timeout failures, and the user-facing changelog entry is missing required attribution.

## What This Changes

The PR buffers embedded Pi runner `ReplyPayload` tool-result payloads, prepends them to non-streaming final run payloads, clears the buffer on compaction retry, extends related attempt/subscribe types, adds regression tests, and updates the changelog.

## Best Possible Solution

Land a rebased patch that captures complete non-empty tool-result `ReplyPayload`s in the attempt layer, clears retry state, integrates with current main’s tool-media merge path, and only exposes buffered payloads after timeout/error handling cannot be hidden.

## Reproduction Assessment

Yes. The intended bug is reproducible with a mocked embedded attempt that emits tool-result payloads plus assistant text and then inspecting `runEmbeddedPiAgent(...).payloads`; the PR regression is reproducible from the added timed-out mocked attempt that returns buffered payloads without an assistant reply.

## Solution Assessment

No. The implementation direction is maintainable, but the branch currently uses buffered tool-result payloads to satisfy the timeout guard; a safer repair should decide timeout/failure state first, then attach buffered non-streaming tool-result payloads only where they do not turn a failed run into a partial success.
