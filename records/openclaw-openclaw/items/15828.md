## Summary

Keep open. Current main still implements bundled session-memory as command-scoped persistence, while the requested idle, shutdown, periodic message-count checkpoint, trigger config, and summary-output modes remain missing; related open PRs cover only the automatic rollover subset and have not merged.

## What This Changes

The issue requests configurable session-memory checkpoints on idle, shutdown, periodic message count, and command events, plus optional LLM summarization instead of raw transcript storage.

## Best Possible Solution

Keep this as the broader session-memory automation tracker, consolidate the automatic rollover PR path first, then define one supported trigger/config contract with docs and targeted coverage.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main confirms the bundled hook is command-scoped and lacks the requested trigger/config surface.

## Solution Assessment

Unclear as a one-shot implementation. The maintainable path is to settle the automatic rollover implementation first, then make an explicit maintainer decision for shutdown, message-count checkpoints, and summary-output semantics.
