- `P2`: This is a normal-priority session-memory persistence improvement with real user impact but pending product and security decisions.
- `impact:session-state`: The affected behavior is session rollover, transcript archival, checkpointing, and memory continuity across session boundaries.

## Summary

Keep open. Current main and the latest release still implement bundled session-memory as command-only capture, while idle, shutdown, message-count checkpoints, trigger-selection config, and summary-output behavior remain unimplemented; open PRs cover only the daily/idle subset.

## What This Changes

The issue requests automatic session-memory checkpoints for idle rollover, gateway shutdown, periodic message-count thresholds, configurable triggers, and optional summarized memory output.

## Best Possible Solution

Keep this as the umbrella tracker while maintainers choose one automatic checkpoint contract, consolidate the daily/idle PR path, and then scope shutdown, message-count, trigger-selection, opt-in summaries, docs, and tests together.
