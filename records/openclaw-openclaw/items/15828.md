## Summary

Keep open. Current `main` and the latest release still implement bundled session-memory as a command-only `/new` and `/reset` capture, while the requested idle, shutdown, message-count, trigger-selection, and summary-output behavior remains missing; the related open PRs cover only part of the daily/idle path and still need maintainer consolidation.

## What This Changes

The issue requests configurable session-memory checkpoints for idle, shutdown, command, and message-count lifecycle events plus an optional LLM-generated summary output mode.

## Best Possible Solution

Keep this as the umbrella tracker while maintainers choose one lifecycle memory contract, consolidate the daily/idle PR path, and scope opt-in idle, shutdown, message-count, and summary behavior with privacy bounds, docs, and focused tests.

Yes at source level, but not as a strict existing-bug reproduction: the handler, metadata, docs, and release tag all show session-memory only runs for `/new` and `/reset`. ## Solution Assessment
