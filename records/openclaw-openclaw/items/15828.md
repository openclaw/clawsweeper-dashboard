## Summary

Keep open. Current main still keeps bundled session-memory scoped to `/new` and `/reset`, while idle/daily rollover persistence, shutdown persistence, message-count checkpoints, trigger configuration, and summary-output mode remain unimplemented; the active daily/idle PRs are partial and not merged.

## What This Changes

Request automatic session-memory checkpoints on idle, shutdown, periodic message counts, and configurable summarized output in addition to the existing `/new` path.

## Best Possible Solution

Keep this as the broad session-memory automation tracker and land a documented trigger surface that preserves command persistence while adding maintained idle/daily, shutdown-safe, message-count, and optional summary-output paths with regression coverage.

## Reproduction Assessment

Yes, by source inspection for the current gap: enable session-memory and exercise idle/daily rollover, gateway shutdown, or normal message flow; current main only registers and accepts `command:new` and `command:reset`. This is still a feature request rather than a broken documented contract.

## Solution Assessment

Unclear as a single one-shot change. The requested direction is valid, but the maintainable path should first reconcile the overlapping daily/idle PRs, then separately scope shutdown durability, message-count state, and summary-output behavior.
