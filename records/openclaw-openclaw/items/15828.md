## Label Justifications

## Summary

This issue remains necessary because current main still limits the bundled session-memory hook to explicit `/new` and `/reset` events, while idle, shutdown, periodic checkpoints, and model-generated summaries remain absent. The two overlapping daily/idle PRs are still open and conflicting, so neither provides a safe canonical resolution for the broader request.

## What This Changes

Add automatic session-memory persistence on idle, shutdown, and message-count checkpoints, with optional LLM-generated summaries.

## Best Possible Solution

Keep this as the canonical automation tracker, converge daily and idle persistence on the modern session lifecycle path, and separately specify shutdown, periodic, and opt-in summary behavior with privacy, deduplication, documentation, and focused lifecycle tests.

## Reproduction Assessment

Yes, at source level. Current main’s hook metadata and public documentation establish a command-only contract, and the requested idle, shutdown, periodic, trigger-selection, and summary-output surfaces are absent; this read-only review did not execute a runtime scenario.

## Solution Assessment

No, not as one undifferentiated patch. Daily and idle rollover should reuse the canonical session lifecycle seam, while shutdown, message-count, and model-summary behavior require separate timing, privacy, compatibility, and ownership decisions.
