## Summary

Keep this broader tracker open. Current main now provides typed `session_end` events for idle/daily rollover and bounded shutdown/restart finalization, but the bundled session-memory hook still consumes only explicit `/new` and `/reset`; message-count checkpoints and summary-output policy also remain unimplemented. The two overlapping daily/idle PRs are still open, conflicting, and incomplete substitutes for the broader request.

## What This Changes

Add automatic session-memory persistence for idle rollover, graceful shutdown, and periodic message checkpoints, with an optional LLM-generated summary output mode.

## Best Possible Solution

Have the active memory plugin consume the canonical typed `session_end` contract for idle, daily, shutdown, and restart finalization, then design periodic snapshots and LLM-authored summaries as separately approved memory policy with deduplication, privacy, timeout, and upgrade-safe configuration.

## Reproduction Assessment

Yes at source level. Current main exposes automatic rollover and shutdown session lifecycle events, while the bundled session-memory hook is still registered and guarded only for explicit `/new` and `/reset`; this establishes the missing capability without treating it as a broken existing contract.

## Solution Assessment

No, the proposed independent trigger matrix is not the best implementation boundary. Reusing typed `session_end` for finalization and separately deciding periodic and summary policy is narrower, avoids duplicate lifecycle logic, and better matches current plugin ownership.
