## Summary

Keep open. Current main still limits the bundled session-memory hook to explicit `/new` and `/reset` commands, while the requested idle, shutdown, periodic checkpoint, configurable trigger, and summary-output behavior remains broader than any merged implementation.

## What This Changes

Request to extend bundled session-memory so it can checkpoint on idle, shutdown, message-count intervals, and `/new`, with configurable triggers and optional LLM summaries.

## Best Possible Solution

Keep this as the broad session-memory automation tracker and land a documented trigger surface that preserves command persistence while adding maintained idle/daily, shutdown-safe, message-count, and optional summary-output paths with regression coverage.

## Reproduction Assessment

Yes for the current gap, by source inspection: enable session-memory and exercise idle/daily rollover, gateway shutdown, or normal message flow; current main only registers and accepts `command:new` and `command:reset`. This is a feature request rather than a broken documented contract.

## Solution Assessment

Unclear as a single one-shot change. The requested direction is valid, but the maintainable path should first reconcile the overlapping daily/idle PRs, then separately scope shutdown durability, message-count state, and summary-output behavior.
