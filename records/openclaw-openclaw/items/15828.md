## Summary

Keep open. Current main still has a command-only bundled session-memory hook, while the requested idle, shutdown, message-count trigger config and LLM summary-output mode remain absent; the two related open PRs cover only part of the idle/daily path.

## What This Changes

The issue requests automatic bundled session-memory checkpoints on idle, shutdown, periodic message count, and `/new`, plus optional LLM-generated summary output.

## Best Possible Solution

Keep this as the broader session-memory automation tracker, consolidate the daily/idle PR path first, then scope shutdown, message-count, and summary-output config with docs and targeted tests.
