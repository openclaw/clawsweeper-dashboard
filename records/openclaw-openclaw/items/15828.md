## Summary

Keep open. Current main and the latest release still keep the bundled session-memory hook command-only; open PRs cover parts of daily/idle rollover and synthesis, but none has landed and shutdown, message-count checkpoints, trigger config, and summary semantics still need maintainer/API consolidation.

## What This Changes

Request to extend bundled session-memory beyond `/new` and `/reset` with idle, shutdown, periodic message-count checkpoints, configurable triggers, and optional summarized output.

## Best Possible Solution

Keep this as the broad tracker, land one canonical daily/idle reset implementation, then deliberately scope bounded shutdown, message-count checkpointing, trigger config, and opt-in summary output with docs and focused tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that the requested trigger and config surfaces are absent from current main and latest release.

## Solution Assessment

Unclear as a one-shot implementation. The maintainable path is for maintainers to choose one automatic-reset contract across the open PRs, then separately define shutdown, periodic checkpoint, and summary-output semantics.
