## Summary

Keep open. Current main still implements the bundled session-memory hook as command-only, while the broader requested idle, shutdown, message-count trigger configuration, and summary-output behavior remain unimplemented; the open daily/idle PRs cover only part of the request and are not merged.

## What This Changes

The issue requests automatic session-memory checkpoints on idle, shutdown, message-count, and command triggers, plus optional summarized memory output.

## Best Possible Solution

Keep this as the broad session-memory automation tracker, then consolidate the daily/idle reset path and separately scope shutdown, periodic checkpoints, trigger config, summary output, docs, tests, and sensitive-content behavior.

## Reproduction Assessment

Yes. Source inspection reproduces the gap: the bundled session-memory handler and metadata accept only command:new and command:reset, while requested idle, shutdown, periodic, trigger-config, and summary-output behavior is absent.

## Solution Assessment

No, this is not a safe one-shot automated fix. The best solution needs maintainer scoping across overlapping daily/idle PRs, shutdown timing, persisted memory semantics, and security-sensitive transcript handling.
