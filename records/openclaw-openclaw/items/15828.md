## Summary

Keep open. Current main and v2026.5.20 still make bundled session-memory a command-only capture for `/new` and `/reset`; shutdown lifecycle hooks and session lifecycle hooks exist, but they do not make session-memory save on idle, shutdown, message-count checkpoints, trigger-selection config, or summary-output mode. The open daily/idle PRs are useful partial paths, but they remain unmerged and do not cover the broader shutdown, periodic, summary, privacy, and provider-exposure decisions.

## What This Changes

The issue requests automatic session-memory checkpoints on idle, shutdown, periodic message counts, configurable trigger selection, and optional LLM-summarized memory output.

## Best Possible Solution

Keep this as the umbrella tracker while maintainers choose one lifecycle memory contract, consolidate the daily/idle PR path, and then scope shutdown, message-count, trigger-selection, opt-in summaries, docs, tests, and privacy bounds together.
