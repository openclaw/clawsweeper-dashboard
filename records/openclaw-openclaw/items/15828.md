## Summary

Keep open: current main still implements bundled session-memory as command-only `/new` and `/reset` persistence, while idle, shutdown, periodic checkpoint, and summary-output modes remain absent and the daily/idle subset overlaps active unmerged PRs.

## What This Changes

The issue requests configurable automatic session-memory checkpoints on idle, shutdown, periodic message count, and `/new`, plus an optional LLM-generated summary output mode.

## Best Possible Solution

Keep this as the broader session-memory automation tracker, consolidate the active daily/idle PRs, then define the supported trigger/config contract with docs and targeted coverage.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main confirms the bundled hook is command-only and lacks the requested trigger/config surface.

## Solution Assessment

Unclear as a one-shot implementation plan. The maintainable path is to consolidate the open daily/idle PRs first, then make maintainer-scoped decisions for shutdown, message-count checkpoints, and summary-output semantics.
