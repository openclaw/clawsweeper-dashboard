## Summary

Keep open: this PR is a plausible narrow fix and has addressed the earlier layout/changelog feedback, but current head introduces a strict TypeScript nullability error at the new day-chip helper call and is still marked dirty against current main.

## What This Changes

The PR moves Dreams diary day-chip rendering from the scrollable entry body into the sticky diary chrome and adds an Unreleased changelog entry for the long-entry navigation issue.

## Best Possible Solution

Keep the moved-inside-sticky-chrome design, fix the nullable helper contract, rebase or resolve current-main conflict, then run the focused UI test and changed gate before merge.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, by source inspection and the linked #76638 steps: current main scrolls `.dreams-diary` while rendering non-sticky day chips before long article content, so long entries can push the date switcher out of view.

## Solution Assessment

Is this the best way to solve the issue? No, not yet: moving the chips into the existing sticky chrome is the right maintainable direction, but the new helper's `string` parameter must accept or guard `null` before the patch can pass strict TypeScript.

## Review Findings

Overall correctness: patch is incorrect

Widen the helper parameter or guard before calling it.
