## Summary

Keep this PR open, but it is not merge-ready: current main still has the reported day-chip scroll-away behavior, while the proposed `top: 0` sticky rule conflicts with the existing sticky diary chrome and the user-facing UI fix lacks a changelog entry.

## What This Changes

The PR changes `ui/src/styles/dreams.css` so `.dreams-diary__daychips` becomes sticky at `top: 0`, raises its z-index, and adds a background.

## Best Possible Solution

Rework the PR so the day-chip navigation remains visible and clickable below or inside the existing sticky diary chrome across long entries and responsive header wrapping, then add the required changelog entry.

## Reproduction Assessment

Yes. Source inspection plus the linked issue steps gives a high-confidence path: Dreams -> Diary with multiple days and a long entry scrolls `.dreams-diary`, while current main renders `.dreams-diary__daychips` below the sticky header with only `position: relative`, so the chips scroll away.

## Solution Assessment

No. Making the chip row sticky is the right direction, but `top: 0` pins it under the already-sticky `.dreams-diary__chrome`; a safer fix is to render it inside that sticky chrome or offset it below the dynamic chrome height.
