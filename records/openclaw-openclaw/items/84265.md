## Summary

Keep open: the PR targets a valid agent failover bug, but the patch is not merge-ready because the prompt-side same-profile retry skips existing auth-profile failure marking/cooldown and the supplied proof does not demonstrate the changed retry behavior in a real setup.

## What This Changes

The PR adds transient failover reason classification, same-profile retries for overloaded, rate-limit, and server-error failures when no fallback is configured, regression tests, and a changelog entry.

## Best Possible Solution

Keep the same-profile transient retry direction, but make prompt and assistant paths share failure marking/cooldown semantics, report same-profile retries truthfully, keep the retry policy bounded, and require redacted real behavior proof.

## Reproduction Assessment

Yes from source inspection: current main resolves transient prompt/assistant failures to profile rotation first, then surfaces the error when rotation is exhausted and no fallback is configured. I did not run a live provider overload repro.

## Solution Assessment

Not yet. Same-profile retry is a plausible fix boundary, but this implementation should preserve prompt-side cooldown marking, use truthful retry telemetry, and be backed by real behavior proof before merge.
