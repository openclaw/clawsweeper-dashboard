## Summary

Keep open. The picker half has already shipped on current main via #61382, but the root `llm` shorthand is still absent and the branch is not mergeable as-is because it adds a credential-bearing config surface with build, correctness, validation, and secret-coverage issues that need maintainer config/product review.

## What This Changes

The PR adds a root `llm` config shorthand that materializes a custom model provider/default model and changes the chat model picker to submit provider-qualified model refs.

## Best Possible Solution

If maintainers want this shorthand, implement it as a config-owner change in the current materialization pipeline with strict provider/ref/base URL validation, non-destructive merging, generated schema/help/docs/changelog updates, secret registry/UI/redaction coverage, and focused config plus picker regression tests.

## Reproduction Assessment

Yes for the PR defects by source inspection: the diff has deterministic build, merge, validation, and secret-surface failures, while current main confirms the root `llm` feature is absent and the picker behavior is already covered elsewhere.

## Solution Assessment

No. The branch mixes an already-shipped picker fix with a new root credential-bearing config alias, and the shorthand needs rebasing plus config/security design work before this is the narrowest maintainable solution.
