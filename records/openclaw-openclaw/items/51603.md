## Summary

Keep this PR open for maintainer handling. The Control UI picker part is already covered on current main, but the new root `llm` config shorthand is not implemented there and the PR branch still has correctness and secret-surface gaps that should block merge as-is.

## What This Changes

The branch adds a top-level `llm` config shorthand that expands into model provider/default settings and changes the chat model picker to submit provider-qualified model refs.

## Best Possible Solution

A maintainer should first decide whether OpenClaw wants a root `llm` shorthand at all. If yes, it should be rebuilt as a config-owner change with strict provider/ref validation, non-destructive provider/model merging, fallback-preserving default assignment, generated config metadata, SecretRef/redaction coverage, docs, and focused regression tests, while leaving the current main picker implementation in place.

## Reproduction Assessment

Yes for the PR defects: the diff and current parsing/defaulting code give deterministic static cases for fallback loss, dangling provider/model refs, slash-containing `provider_id`, and free-form API values. Current-main inspection also confirms the `llm` key is absent while the picker behavior is already covered by tests.

## Solution Assessment

No. This PR mixes an already-solved picker bug with a new credential-bearing root config alias that needs product and security review plus several correctness fixes before it would be the narrowest maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve fallbacks when seeding the primary model:** `src/config/defaults.ts:593`
  - body: When `agents.defaults.model` already contains `fallbacks` but no `primary`, this assignment replaces the whole model object with `{ primary }`. Adding `llm` would silently drop the configured failover chain on every load, so spread the existing model object before setting `primary`.
  - confidence: 0.93
- **[P2] Add the shorthand model before pointing primary at it:** `src/config/defaults.ts:576-578`
  - body: If `models.providers[providerId]` already exists, the PR preserves that provider unchanged but can still set `agents.defaults.model.primary` to `providerId/modelId`. When the existing provider lacks that model, the new default points at a model that cannot resolve.
  - confidence: 0.91
- **[P2] Reject provider IDs that cannot form model refs:** `src/config/zod-schema.core.ts:270`
  - body: `provider_id` is later used as the left side of a `provider/model` ref, but the schema allows values containing `/`. A value like `foo/bar` becomes a primary ref that parses as provider `foo`, while the provider was inserted under `models.providers["foo/bar"]`.
  - confidence: 0.89
- **[P2] Validate llm provider API values:** `src/config/zod-schema.core.ts:265`
  - body: `llm.provider` is accepted as any string and then cast into `ModelProviderConfig["api"]`. Typos pass validation and reach downstream transport selection as unsupported API ids; validate this against the supported shorthand values plus the existing model API enum.
  - confidence: 0.87
- **[P2] Cover the new secret path in generated config surfaces:** `src/config/zod-schema.core.ts:267`
  - body: The PR adds a credential-bearing `llm.api_key` path but does not update generated schema/help/labels, SecretRef documentation, or redaction/UI coverage for that public path. Secret-bearing config aliases need the same coverage as `models.providers.*.apiKey`.
  - confidence: 0.8
