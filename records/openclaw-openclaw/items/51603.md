Keep this PR open but blocked: the picker-ref half is already covered on main, while the remaining root `llm` shorthand is a new credential-bearing config contract with correctness, validation, SecretRef-surface, and proof gaps.

## What This Changes

The branch adds a top-level `llm` config shorthand that materializes custom model-provider defaults and changes the old Control UI model picker to send provider-qualified model values.

## Best Possible Solution

Keep the shipped picker implementation; if maintainers want a root shorthand, land it as a separate rebased config design with strict validation, SecretRef coverage, docs, tests, and redacted real proof.

The root shorthand itself is a new feature, not a current-main bug reproduction.

## Solution Assessment

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Preserve existing model defaults when seeding primary:** `src/config/defaults.ts:591`
  - body: When `agents.defaults.model` already has `fallbacks`, `timeoutMs`, or other fields but no `primary`, assigning `{ primary: modelRef }` drops those settings during config load, contradicting the non-destructive shorthand contract.
  - confidence: 0.93
- **[P2] Add the shorthand model before setting primary:** `src/config/defaults.ts:574-576`
  - body: If `models.providers[providerId]` already exists, the branch preserves that provider unchanged but still points `agents.defaults.model.primary` at `providerId/modelId`; when the preserved provider lacks that model, the default cannot resolve.
  - confidence: 0.91
- **[P2] Reject provider IDs that cannot form model refs:** `src/config/zod-schema.core.ts:271`
  - body: `provider_id` is later used as the left side of a `provider/model` ref, but the schema allows values containing `/`; those parse as a different provider/model pair at runtime and miss the injected provider entry.
  - confidence: 0.89
- **[P2] Restrict llm.provider to supported API values:** `src/config/zod-schema.core.ts:266`
  - body: `llm.provider` accepts any non-empty string and is cast into `ModelProviderConfig["api"]`, so typos pass config validation and fail later in provider selection instead of being rejected at load time.
  - confidence: 0.87
- **[P2] Require base_url before materializing a provider:** `src/config/zod-schema.core.ts:269`
  - body: `base_url` is optional, but `applyLlmConfig()` materializes `baseUrl: ""`; the normal provider schema requires a non-empty base URL, so a schema-valid `llm` block can create an invalid provider.
  - confidence: 0.84
- **[P2] Cover llm.api_key in secret surfaces:** `src/config/zod-schema.core.ts:268`
  - body: The branch adds credential-bearing `llm.api_key`, but it does not add target-registry, configure/apply, audit, generated docs, or equivalent handling comparable to `models.providers.*.apiKey`.
  - confidence: 0.82
