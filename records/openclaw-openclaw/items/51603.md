## Summary

Keep open for maintainer judgment: the picker-ref portion has already shipped, but the remaining root `llm` shorthand is a new credential-bearing config contract and this branch still has correctness, validation, SecretRef coverage, proof, and rebase blockers.

## What This Changes

The PR adds a root `llm` config shorthand that expands into model provider/default-agent config and changes the older Control UI picker to send provider-qualified model values.

## Best Possible Solution

Keep the shipped picker implementation; if maintainers want the root shorthand, land it as a rebased config design with strict validation, SecretRef coverage, docs, tests, and redacted real behavior proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: the branch drops existing model defaults, can create dangling primary refs, accepts bad provider/ref values, and lacks complete credential-surface coverage. The root shorthand itself is a new feature, not a current-main bug reproduction.

## Solution Assessment

No. The picker fix already landed in https://github.com/openclaw/openclaw/pull/61382, and the remaining shorthand should be a rebased config/security design PR rather than this mixed stale branch.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Preserve existing model defaults when seeding primary:** `src/config/defaults.ts:591`
  - body: When `agents.defaults.model` already has `fallbacks`, `timeoutMs`, or other fields but no `primary`, assigning `{ primary: modelRef }` drops those settings during config load, contradicting the non-destructive shorthand contract.
  - confidence: 0.93
- **[P2] Add the shorthand model before setting primary:** `src/config/defaults.ts:577-580`
  - body: If `models.providers[providerId]` already exists, the branch preserves that provider unchanged but still points `agents.defaults.model.primary` at `providerId/modelId`; when the preserved provider lacks that model, the default cannot resolve.
  - confidence: 0.91
- **[P2] Reject provider IDs that cannot form model refs:** `src/config/zod-schema.core.ts:271`
  - body: `provider_id` is later used as the left side of a `provider/model` ref, but the schema allows values containing `/`; `parseModelRef()` splits at the first slash, so runtime lookups miss the injected provider key.
  - confidence: 0.89
- **[P2] Restrict llm.provider to supported API values:** `src/config/zod-schema.core.ts:266`
  - body: `llm.provider` accepts any non-empty string and is cast into `ModelProviderConfig["api"]`, so typos pass config validation and fail later in provider selection instead of being rejected at load time.
  - confidence: 0.87
- **[P2] Require base_url before materializing a provider:** `src/config/zod-schema.core.ts:269`
  - body: `base_url` is optional, but `applyLlmConfig()` materializes `baseUrl: ""`; the normal provider schema requires a non-empty base URL, so a schema-valid `llm` block can create an invalid provider entry.
  - confidence: 0.84
- **[P2] Cover llm.api_key in secret surfaces:** `src/config/zod-schema.core.ts:268`
  - body: The branch adds credential-bearing `llm.api_key`, but it does not add equivalent target-registry, configure/apply, audit, generated docs, and runtime collector coverage for the new root path.
  - confidence: 0.82
