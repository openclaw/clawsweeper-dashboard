## Summary

Keep open: current main already covers the model-picker wrong-provider behavior, but it still does not implement the requested root `llm` config shorthand. The PR also has blocking correctness and secret-surface issues, so it needs maintainer/config-owner review rather than cleanup closure.

## What This Changes

The PR adds a root `llm` config shorthand that expands into custom model provider/default-model settings and changes the Control UI chat model picker to submit provider-qualified model refs.

## Best Possible Solution

If maintainers want this shorthand, land it as a config-owner change through the current `materializeRuntimeConfig` pipeline with strict provider/ref/base URL validation, non-destructive provider/model/default merging, generated schema/help/docs, secret registry/UI/redaction coverage, and focused config plus picker regression tests; keep the already-shipped picker behavior from main.

## Reproduction Assessment

Yes for the PR defects: the diff shows deterministic cases such as existing provider plus a new `llm.model`, existing fallbacks with no primary, slash-containing `provider_id`, typoed provider APIs, and the new `llm.api_key` secret path. Current-main inspection also confirms the root `llm` key is absent while the picker behavior is already covered by tests.

## Solution Assessment

No. The branch mixes an already-shipped UI picker fix with a new credential-bearing root config alias that needs product approval, current-pipeline rebasing, and multiple correctness/security fixes before it is maintainable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve fallbacks when seeding the primary model:** `src/config/defaults.ts:593`
  - body: When `agents.defaults.model` already has `fallbacks` but no `primary`, this assignment replaces the whole model object with `{ primary }`. Adding `llm` would silently drop the configured fallback chain on every load, so preserve the existing model object while setting `primary`.
  - confidence: 0.93
- **[P2] Add the shorthand model before pointing primary at it:** `src/config/defaults.ts:576-578`
  - body: If `models.providers[providerId]` already exists, the PR preserves that provider unchanged but can still set `agents.defaults.model.primary` to `providerId/modelId`. When the existing provider lacks that model, the new default points at a model that cannot resolve.
  - confidence: 0.91
- **[P2] Reject provider IDs that cannot form model refs:** `src/config/zod-schema.core.ts:270`
  - body: `provider_id` is later used as the left side of a `provider/model` ref, but the schema allows values containing `/`. A value like `foo/bar` is inserted under `models.providers["foo/bar"]` while later parsing resolves the primary as provider `foo`, so validate this before accepting the config.
  - confidence: 0.89
- **[P2] Restrict llm.provider to supported API values:** `src/config/zod-schema.core.ts:265`
  - body: `llm.provider` accepts any string and is cast into `ModelProviderConfig["api"]`. Typos pass validation and reach downstream transport selection as unsupported API ids; validate against the supported shorthands plus the existing `ModelApi` enum.
  - confidence: 0.87
- **[P2] Require a base URL for custom provider shorthand:** `src/config/zod-schema.core.ts:268`
  - body: `base_url` is optional, but `applyLlmConfig()` materializes a provider with `baseUrl: ""` when it is omitted even though normal `models.providers.*.baseUrl` validation requires a non-empty string. A valid `llm` block can therefore create a runtime provider that would fail the regular provider schema and may fail transport setup.
  - confidence: 0.82
- **[P2] Cover the new secret path in generated config surfaces:** `src/config/zod-schema.core.ts:267`
  - body: The PR adds a credential-bearing `llm.api_key` path but does not update generated schema/help, secret target registry, config UI, or redaction coverage for that public path. Secret-bearing aliases need the same handling as `models.providers.*.apiKey`.
  - confidence: 0.8
