## Summary

Keep open. The Control UI picker part has been superseded by the merged model-picker fix in https://github.com/openclaw/openclaw/pull/61382, but the remaining root `llm` shorthand is a new credential-bearing config feature that is not on current main and the PR still has blocking correctness, security-surface, and real-behavior-proof gaps.

## What This Changes

The PR adds a root `llm` config shorthand that materializes model provider/default config and changes the Control UI chat model picker to submit provider-qualified values.

## Best Possible Solution

If maintainers want this shorthand, replace or rebase the branch as a dedicated config-owner PR with strict provider/ref/base URL validation, non-destructive merging, generated schema/help/docs, secret registry/audit coverage, and focused config tests; otherwise keep the existing `models.providers` plus `agents.defaults` contract.

## Reproduction Assessment

Yes for the PR defects by source inspection: the missing import, fallback overwrite, dangling model-ref path, and validation gaps are deterministic in the diff. The root `llm` shorthand itself is a new feature, so there is no current-main bug reproduction path for that request.

## Solution Assessment

No. The picker fix has already landed via https://github.com/openclaw/openclaw/pull/61382, and the remaining shorthand should be a current-main config/security design PR rather than this stale mixed-scope branch.

Overall confidence: 0.93

Full review comments:

- **[P1] Import the primary model resolver before using it:** `src/config/defaults.ts:588`
  - body: `applyLlmConfig()` calls `resolveAgentModelPrimaryValue`, but this branch never imports that symbol from `src/config/model-input.ts`. The config build/typecheck will fail before the new shorthand can run.
  - confidence: 0.96
- **[P1] Preserve existing model defaults when seeding primary:** `src/config/defaults.ts:591`
  - body: When `agents.defaults.model` already has `fallbacks` or `timeoutMs` but no `primary`, this assignment replaces the entire object with `{ primary }`, silently dropping configured fallback behavior on every load.
  - confidence: 0.93
- **[P2] Add the shorthand model before pointing primary at it:** `src/config/defaults.ts:572-574`
  - body: If `models.providers[providerId]` already exists, the branch preserves that provider unchanged but can still set `agents.defaults.model.primary` to `providerId/modelId`. When the preserved provider lacks that model, the default points at a model that cannot resolve.
  - confidence: 0.91
- **[P2] Reject provider IDs that cannot form model refs:** `src/config/zod-schema.core.ts:270`
  - body: `provider_id` is later used as the left side of a `provider/model` ref, but the schema allows values containing `/`. A value like `foo/bar` is stored under one provider key while model-ref parsing resolves it as provider `foo`.
  - confidence: 0.89
- **[P2] Restrict llm.provider to supported API values:** `src/config/zod-schema.core.ts:265`
  - body: `llm.provider` accepts any non-empty string and is cast into `ModelProviderConfig["api"]`. Typos pass validation and reach downstream provider selection as unsupported API ids instead of failing at config load.
  - confidence: 0.87
- **[P2] Require base_url before materializing a provider:** `src/config/zod-schema.core.ts:268`
  - body: `base_url` is optional, but `applyLlmConfig()` materializes `baseUrl: ""` when omitted even though `ModelProviderSchema` requires a non-empty provider `baseUrl`. A schema-valid `llm` block can create an invalid runtime provider.
  - confidence: 0.84
- **[P2] Cover llm.api_key in secret surfaces:** `src/config/zod-schema.core.ts:267`
  - body: The PR adds credential-bearing `llm.api_key`, but only registers the Zod field. It needs the same target-registry, configure/apply/audit, redaction, and generated credential-surface coverage as `models.providers.*.apiKey`.
  - confidence: 0.82
- **[P3] Add the public config contract docs:** `src/config/types.openclaw.ts:95-96`
  - body: A new root config key needs generated schema/help and user-facing configuration docs so operators and config tooling have a stable contract for the shorthand.
  - confidence: 0.76
