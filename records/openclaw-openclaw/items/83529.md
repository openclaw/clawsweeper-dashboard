## Summary

Keep open: the PR is a useful direction, but it is not merge-ready because the upgrade path still leaves existing native Codex bound sidecars without callable plugin tools, and the supplied proof is repo-contained rather than a real bound conversation invoking a plugin tool.

## What This Changes

The PR changes the bundled Codex plugin so native conversation bindings build OpenClaw dynamic tool specs, pass them into new app-server threads, and dispatch dynamic tool calls back through plugin handlers.

## Best Possible Solution

Expose plugin tools for both new and upgraded native bound conversations with an explicit upgrade-safe refresh, fork, or tested rebind path, then merge after redacted live proof of a plugin tool invoked through the native Codex binding.

## Reproduction Assessment

Yes from source inspection: current main starts native bound Codex threads without `dynamicTools` and returns a fixed unsupported response for `item/tool/call`. I did not run a live bound Codex conversation in this read-only review.

## Solution Assessment

No, not yet: the PR fixes new and fingerprinted bindings but explicitly preserves no-fingerprint sidecars, which are the upgrade state current users have from main. The safer solution is an explicit tested legacy refresh, fork, or rebind path plus live proof.
