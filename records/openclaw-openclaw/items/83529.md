## Label Justifications

- `P1`: The PR targets a broken native Codex workflow where registered plugin tools are hidden from bound conversations, but the patch still carries merge-blocking session-state risk.
- `merge-risk: 🚨 session-state`: The patch can replace existing no-fingerprint bound Codex sidecars with new threads and lose prior conversation context on upgrade.
- `merge-risk: 🚨 message-delivery`: Plugin tools invoked through bound Codex turns may target channel messages, so routing correctness needs proof beyond mocked app-server tests.
- `merge-risk: 🚨 security-boundary`: The diff lets app-server tool-call requests execute plugin handlers and relies on strict thread/turn scoping to preserve the dispatch boundary.

## Summary

This PR is a plausible fix for the native Codex binding gap, but it is not merge-ready: the patch still rotates no-fingerprint bound threads even when there is no dynamic tool catalog to gain, and the supplied after-fix proof is mocked app-server coverage rather than a live bound conversation invoking a plugin tool.

## What This Changes

The PR updates the bundled Codex plugin to project OpenClaw plugin dynamic tools into native bound Codex conversations, dispatch matching `item/tool/call` requests, pass runtime config into the binding, and add regression tests.

## Best Possible Solution

Merge a guarded native dynamic-tool bridge that rotates only when a non-empty tool catalog needs a tool-aware thread, preserves or records empty-catalog legacy bindings without losing context, and is backed by redacted live bound-session proof.

## Reproduction Assessment

Yes from source inspection: current main creates native bound Codex threads without dynamic tools and returns an unsupported response for `item/tool/call`. I did not run a live gateway repro in this read-only review.

## Solution Assessment

No. The dynamic-tool bridge and thread/turn guard are the right direction, but the no-fingerprint refresh should preserve empty-catalog bindings and the PR still needs live bound-session proof.
