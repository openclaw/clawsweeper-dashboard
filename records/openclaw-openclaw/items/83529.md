## Best Possible Solution

Yes. Current main source shows native bound Codex turns omit dynamic tools and return a fixed unsupported response for item/tool/call, and the PR adds a focused mocked regression path for that failure mode.

## Solution Assessment

No. Reusing the existing Codex dynamic-tool bridge is the right direction, but the legacy refresh rule needs an empty-catalog compatibility guard before this is the safest fix.
