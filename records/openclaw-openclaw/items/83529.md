## Summary

Keep open. The PR fixes a real native Codex binding gap, but current head still has a P1 upgrade/session-state defect for legacy sidecars with an empty dynamic-tool catalog, and the supplied proof is repo-local tests plus CLI output rather than a live native bound Codex conversation invoking a plugin tool.

## What This Changes

The PR changes the bundled Codex plugin to project OpenClaw plugin dynamic tools into native bound Codex conversations, dispatch matching app-server tool calls through plugin handlers, and add regression coverage.

## Best Possible Solution

Merge a guarded dynamic-tool bridge that preserves legacy sidecars when the resolved catalog is empty, rotates only when a non-empty catalog needs a tool-aware thread, and includes focused regression coverage plus redacted live bound-session proof.

## Reproduction Assessment

Yes from source inspection: current main creates native bound Codex threads without dynamic tools and returns a fixed unsupported response for item/tool/call. I did not run a live gateway repro in this read-only review.

## Solution Assessment

No. The dynamic-tool bridge and thread/turn scoping are the right direction, but the refresh condition should preserve empty-catalog legacy threads and the PR still needs live bound-session proof.
