## Summary

Keep open: the central Codex plugin-tool bridge is useful and source-reproducible, but the current patch still has a P1 session-state regression for explicit thread binds and lacks real bound-session proof.

## What This Changes

The PR updates the bundled Codex plugin's native conversation binding to pass plugin dynamic tools into app-server threads, dispatch scoped `item/tool/call` requests, pass runtime config, and add regression tests.

## Best Possible Solution

Keep the dynamic tool bridge, but preserve explicit `/codex bind <thread-id>` and `/codex resume` thread semantics, add a regression for that upgrade path, and require redacted live bound-session proof before merge.

## Reproduction Assessment

Yes. Source inspection shows current main rejects bound `item/tool/call` requests, and the PR branch has a clear source-level repro for explicit thread binds rotating because attached sidecars still lack `dynamicToolsFingerprint`.

## Solution Assessment

No. The plugin-tool bridge is the right direction, but it should not silently replace explicitly selected Codex threads and it still needs real bound-session proof.
