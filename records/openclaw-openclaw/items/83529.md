## What This Changes

The PR wires OpenClaw plugin dynamic tools into native Codex bound conversations, adds dynamic-tool call dispatch, passes runtime config into the Codex plugin binding, and adds focused regression tests.

## Best Possible Solution

Use the existing Codex dynamic-tool bridge, but dispatch only calls for the active thread and turn, provide a tested legacy no-fingerprint refresh/rebind policy, and prove a real bound conversation can invoke a plugin tool.

## Reproduction Assessment

Yes. Current-main source starts native bound Codex threads without dynamicTools and returns a hardcoded unsupported response for item/tool/call; I did not run a live bound Codex conversation in this read-only review.

## Solution Assessment

No. Reusing the dynamic-tool bridge is the right direction, but the patch must isolate tool calls by thread/turn and settle the legacy sidecar upgrade path before it is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Filter tool calls to the active thread and turn:** `extensions/codex/src/conversation-binding.ts:481-489`
  - body: This handler returns a tool result for every `item/tool/call` request on the shared app-server client. Since the client stops at the first handler that returns a value, concurrent bound conversations can have one active bridge execute another thread's plugin tool with the wrong channel/session context; parse the call, require matching `threadId` and current `turnId`, and return `undefined` for nonmatching requests.
  - confidence: 0.9
- **[P1] Handle legacy sidecars before preserving threads:** `extensions/codex/src/conversation-binding.ts:434-441`
  - body: This guard skips refresh whenever the sidecar lacks `dynamicToolsFingerprint`, which is what current main writes. Because plugin `dynamicTools` are only sent on `thread/start`, upgraded bound conversations can keep an old thread where Codex still cannot see plugin tools; add a tested migration, fork, or explicit rebind path, or clearly scope the PR to new bindings only.
  - confidence: 0.88
