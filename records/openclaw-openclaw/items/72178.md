## Summary

This PR should stay open because current main still lacks the requested `chat.history` cursor pagination, but the branch is not ready to merge: the cursor window is computed without being applied before history projection, generated protocol models are missing, and contributor real behavior proof is absent.

## What This Changes

The PR adds a `before` cursor and `cursor`/`hasMore` response metadata to `chat.history`, Web UI load-older state/rendering, partial `NO_REPLY` suppression, docs, and tests.

## Best Possible Solution

Repair the Gateway handler so cursor requests page a stable projected transcript window or reuse the existing session-history pagination contract, regenerate protocol clients, keep focused regression coverage, and require real WebChat/Gateway proof before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence PR-head reproduction path: create a transcript longer than one page, request the latest `chat.history`, then request `before=cursor`; the handler computes a raw window but still projects/caps the full transcript, so older-page requests can overlap the latest page.

## Solution Assessment

No. Cursor pagination is a reasonable additive direction, but this implementation should apply the cursor window before projection/capping or reuse the existing session-history pagination helper and keep generated protocol outputs synchronized.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Apply the cursor window before projecting history:** `src/gateway/server-methods/chat.ts:1517-1523`
  - body: The handler computes `start` and `end` from `before`, but still passes the full `rawMessages` array into `projectRecentChatDisplayMessages`. That helper caps to the newest messages, so older-page requests can return the latest page again and make the UI prepend duplicates; slice or otherwise project the requested stable window before capping.
  - confidence: 0.93

  - body: Adding `before` changes the public Gateway schema, but the generated Swift `ChatHistoryParams` output is not included in the PR file list. Typed clients and protocol drift checks will remain out of sync until those artifacts are regenerated.
  - confidence: 0.9
