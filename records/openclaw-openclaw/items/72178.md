Yes. Source inspection of PR head gives a high-confidence path: load a transcript longer than the page limit, request the latest `chat.history` page, then request `before` with the returned cursor; the handler still projects the full raw transcript and can return the newest page again.

## Solution Assessment

No. Cursor pagination is a reasonable direction, but this branch should either reuse the existing cursor-capable session-history path or apply the raw window before projection/capping, then regenerate protocol artifacts and prove the real WebChat flow.
