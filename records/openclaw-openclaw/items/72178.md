## Summary

Keep open: current main still lacks chat.history pagination, but this active external PR is not merge-ready because PR head still ignores the requested cursor window during projection, omits generated protocol model updates, and lacks real behavior proof.

## What This Changes

The PR adds chat.history before/cursor/hasMore pagination, WebChat load-older state and UI handling, partial NO_REPLY suppression, docs, and focused tests.

## Best Possible Solution

Revise this branch or a focused replacement PR to implement additive chat.history pagination with stable cursor/window semantics, regenerated protocol clients, regression coverage, and redacted real WebChat/Gateway proof.

## Reproduction Assessment

Yes. Source inspection of PR head gives a high-confidence path: with a transcript longer than the page limit, request the latest chat.history page, then request before with the returned cursor; the handler still projects the full transcript and can return the newest page again.

## Solution Assessment

No. Cursor pagination is a reasonable additive direction, but this implementation should apply pagination before projection/capping or align with the existing session-history cursor helper, then regenerate protocol artifacts.
