## Summary

Keep open: current main still lacks the requested `chat.history` cursor pagination, so this PR is not obsolete, but the latest PR head still has pagination correctness and protocol-generation blockers plus missing real behavior proof.

## What This Changes

The PR adds cursor pagination for Gateway `chat.history`, Web UI load-older behavior, partial `NO_REPLY` suppression, related docs, and tests.

## Best Possible Solution

Implement `chat.history` pagination over a stable projected transcript window or reuse the existing session-history pagination contract, regenerate protocol clients, add focused regression coverage, and collect real WebChat/Gateway proof before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence PR-head path: request the latest `chat.history` page for a transcript longer than the page size, then request `before=cursor`; the handler computes a window but still projects the full raw transcript.

## Solution Assessment

No. Cursor pagination is a reasonable additive direction, but this implementation should page a stable projected/windowed transcript, avoid object-identity cursor recovery, and keep generated clients in sync.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Apply the cursor window before projecting history:** `src/gateway/server-methods/chat.ts:1526-1530`
  - body: The handler computes `start`/`end` from `before`, but still passes the full `rawMessages` array into `projectRecentChatDisplayMessages`. That helper keeps the newest messages, so an older-page request can return the latest page again and the UI will prepend duplicates.
  - confidence: 0.94
- **[P2] Track the first returned raw index without object identity:** `src/gateway/server-methods/chat.ts:1541-1548`
  - body: `firstBounded` is a projected/sanitized message, so `rawMessages.indexOf(firstBounded, start)` can fail whenever projection clones the first returned row or byte trimming leaves a cloned row first. Falling back to `start` makes the cursor point before the actual first returned message and can skip or overlap byte-trimmed history.
  - confidence: 0.82
- **[P2] Regenerate the Swift protocol models:** `src/gateway/protocol/schema/logs-chat.ts:31`
  - body: Adding `before` changes the public Gateway schema, but the generated Swift `ChatHistoryParams` on the PR head still lacks that field. Typed clients and protocol drift checks will remain broken until the generated model is refreshed.
  - confidence: 0.9
