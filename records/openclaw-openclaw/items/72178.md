Full review comments:

- **[P1] Window raw history before projection:** `src/gateway/server-methods/chat.ts:1527-1530`
  - body: The handler computes `start`/`end` from `before`, but still calls `projectRecentChatDisplayMessages(rawMessages, ...)`.   - confidence: 0.95
- **[P2] Update generated protocol clients:** `src/gateway/protocol/schema/logs-chat.ts:31`
  - body: Adding `before` changes the public `chat.history` request shape, but the generated Swift `ChatHistoryParams` still lacks the field and the PR edits the old schema location while current main owns this under `packages/gateway-protocol`. Regenerate the protocol artifacts from the current schema surface before merge.
  - confidence: 0.91
- **[P2] Keep older-page rendering bounded:** `ui/src/ui/chat/build-chat-items.ts:200`
  - body: `loadOlderChatHistory` accumulates pages in `chatMessages`, but this change removes the render limit and renders every retained history item. Repeated page loads can rebuild a large DOM and recreate the WebChat long-history performance problem; keep a bounded/virtualized rendering path or an equivalent render budget.
  - confidence: 0.8
