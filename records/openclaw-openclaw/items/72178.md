## Summary

Keep open. Current main does not implement the requested API, and the latest PR head still has a blocking cursor-window bug plus generated-protocol and changelog drift; the work is useful but needs repair before merge.

## What This Changes

The PR adds `before`/`cursor`/`hasMore` pagination to `chat.history`, load-older UI behavior, partial `NO_REPLY` stream suppression, protocol docs, and focused tests.

## Best Possible Solution

Repair the PR at the Gateway handler boundary: page the requested raw transcript window or carry stable raw indices through projection/capping, preserve bounded reads, regenerate protocol artifacts, add the changelog entry, and cover the behavior with handler-level tests.

## Reproduction Assessment

Yes. On the PR head, create a session with more than one page of raw messages, request the latest `chat.history`, then request with `before` set to the returned cursor; because the handler still projects the full raw array with a latest-slice limiter, the older-page request can return overlapping latest messages.

## Solution Assessment

No. Cursor pagination is a reasonable direction, but this implementation is not the narrowest maintainable fix until the server applies the cursor window before projection/capping and keeps the public protocol artifacts in sync.

## Review Findings

Overall correctness: patch is incorrect
