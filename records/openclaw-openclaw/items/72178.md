## Summary

Keep open: current main still lacks `chat.history` cursor pagination, but the latest PR head has blocking server pagination and generated-protocol defects, plus no after-fix real behavior proof from the contributor.

## What This Changes

The PR adds a `before` cursor to Gateway `chat.history`, Web UI load-older state/rendering, partial `NO_REPLY` suppression, and related docs/tests.

## Best Possible Solution

Repair the Gateway handler so cursor requests paginate a stable projected transcript window or reuse the existing session-history pagination helper, regenerate protocol artifacts, add focused regression coverage, and require real WebChat/Gateway proof before merge.

## Reproduction Assessment

Yes, source inspection gives a high-confidence PR-head reproduction path: request the latest `chat.history` page for a transcript longer than one page, then request `before=cursor`; the handler computes a window but still limits the full raw transcript to the newest messages.

## Solution Assessment

No. Cursor pagination is a reasonable additive direction, but this implementation should apply the cursor window before projection/capping or reuse the existing session-history pagination contract, and keep generated clients in sync.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Apply the cursor window before projecting history:** `src/gateway/server-methods/chat.ts:1526-1530`
  - body: The handler computes `start`/`end` from `before`, but still passes the full `rawMessages` array into `projectRecentChatDisplayMessages`. That helper caps to the newest messages, so an older-page request can return the latest page again and the UI will prepend duplicates; slice the requested raw window or carry stable sequence/index metadata through projection before capping.
  - confidence: 0.93
- **[P2] Regenerate the Swift protocol models:** `src/gateway/protocol/schema/logs-chat.ts:31`
  - body: Adding `before` changes the public Gateway schema, but the generated Swift `ChatHistoryParams` outputs are not in the PR. `pnpm protocol:check` already fails with generated diffs adding `before`, so typed clients and protocol drift checks remain broken until those artifacts are refreshed.
  - confidence: 0.9
