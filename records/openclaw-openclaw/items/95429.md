## What This Changes

The issue reports that Anthropic thinking-signature recovery is bypassed when the embedded-agent stream function returns a Promise, causing replay errors to wedge the session instead of self-healing.

## Best Possible Solution

## Reproduction Assessment

Yes. Source inspection shows current main has an async embedded auth wrapper returning a Promise and a recovery wrapper Promise branch that does not pump resolved stream error events; I did not run a live Anthropic replay reproduction in this read-only review.

## Solution Assessment

Yes at the issue boundary. Pumping Promise-resolved streams through the existing recovery loop is the narrow owner-boundary fix, but this issue should stay open until the linked PR lands or maintainers choose a replacement.
