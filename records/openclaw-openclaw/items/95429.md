## Summary

Keep open. Current main and v2026.6.10 still use the Promise-only branch that bypasses `pumpStreamWithRecovery` for async embedded stream functions, and the open linked PR is the candidate fix rather than a reason to close this issue early.

## What This Changes

Fix Anthropic thinking-signature recovery for embedded agent streams whose stream function returns a Promise so mid-stream error events are pumped through recovery.

## Best Possible Solution

Review and land or explicitly reject https://github.com/openclaw/openclaw/pull/95430, while keeping broader signed-thinking replay policy work tracked separately in the related Anthropic thinking-signature issues.

## Reproduction Assessment

Yes, source-reproducible but not live-reproduced here. A focused current-main repro can make `wrapEmbeddedAgentStreamFn` return a Promise that resolves to an event stream whose first event is a recoverable Anthropic thinking error, which the current Promise branch never pumps.

## Solution Assessment

Yes, the linked PR direction is the narrowest maintainable fix: make Promise-resolved streams go through the same `pumpStreamWithRecovery` path as synchronous streams. Broader stale-signature prevention remains adjacent work, not a replacement for this async wrapper bug.
