## Summary

Keep open. Current main and v2026.6.8 still have a Promise-valued stream branch in `wrapAnthropicStreamWithRecovery` that only handles Promise rejection, while the embedded resolver returns an async stream wrapper when injecting auth, so recoverable pre-output stream error events can bypass the recovery pump.

## What This Changes

The issue requests fixing Anthropic thinking replay recovery so async embedded-agent stream functions still route pre-output stream error events through the recovery wrapper and durable transcript repair path.

## Best Possible Solution

Route Promise-valued stream returns through the same outer `pumpStreamWithRecovery` path as synchronous streams, add a focused async stream-error regression, and keep the broader Anthropic replay cluster open until its separate carriers land or are rejected.

## Reproduction Assessment

Yes at source level. A focused regression can use an async `StreamFn` that resolves to an assistant event stream emitting a recoverable pre-output error and assert that recovery retries once; I did not run a live provider repro in this read-only pass.

## Solution Assessment

Yes, the narrow fix is to normalize Promise-valued streams into the existing recovery pump instead of adding a second recovery path. The broader create/replay policy work should remain with the canonical Anthropic cluster.
