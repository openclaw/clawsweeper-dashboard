## Summary

Keep open. Current main and v2026.6.9 still have the Promise-valued recovery bypass, and https://github.com/openclaw/openclaw/pull/95430 is the open linked closing PR for this exact bug.

## What This Changes

The issue reports that async embedded Anthropic stream functions return a Promise, causing the recovery wrapper to skip mid-stream error-event pumping and leave invalid thinking-signature sessions unrecovered.

## Best Possible Solution

Land a narrow fix that routes Promise-resolved embedded streams through pumpStreamWithRecovery with focused regression coverage and real provider proof, then close this issue from the merged PR.

## Reproduction Assessment

Yes, at source level. The embedded auth/API-key wrapper returns a Promise, while current main's recovery wrapper only catches Promise rejection and does not pump resolved stream error events.

## Solution Assessment

Yes, the right fix boundary is the recovery wrapper. The linked PR's direction of reusing pumpStreamWithRecovery for Promise-resolved streams is the narrow maintainable path, pending proof and review.
