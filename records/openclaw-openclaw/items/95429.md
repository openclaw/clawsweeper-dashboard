## Summary

## What This Changes

The issue reports that Anthropic thinking replay recovery is bypassed on the embedded agent path when the wrapped stream function resolves asynchronously.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/95430; the durable fix should route Promise-valued streams through the existing recovery pump and add focused async stream-error regression coverage while broader Anthropic replay-policy work stays in its own cluster.

## Reproduction Assessment

Yes, source-level: current `main` has an async embedded auth wrapper that returns a Promise, and the recovery wrapper's Promise branch does not pump resolved stream error events. I did not run a live Anthropic/Windows reproduction in this read-only pass.

## Solution Assessment

Yes. The narrow maintainable fix is to normalize Promise-valued stream returns into the existing `pumpStreamWithRecovery` path rather than adding a second recovery mechanism; the open linked PR is the candidate for that fix.
