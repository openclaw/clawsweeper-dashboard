## Summary

Keep open. Current main and v2026.6.8 still have the Promise-valued recovery bypass described in this issue, and https://github.com/openclaw/openclaw/pull/95430 is now an open Fixes-linked implementation candidate that should be reviewed or rejected before this issue closes.

## What This Changes

The issue requests that Anthropic thinking-signature recovery run for embedded-agent stream functions that resolve asynchronously and then emit recoverable stream error events.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/95430; the durable fix should route Promise-valued streams through the existing recovery pump and add a focused async stream-error regression while leaving broader Anthropic replay-policy work in its own cluster.

## Reproduction Assessment

Yes, source-level: current main has an async embedded auth wrapper that returns a Promise, and the recovery wrapper's Promise branch does not pump resolved stream error events. I did not run a live Anthropic/Windows reproduction in this read-only pass.

## Solution Assessment

Yes. The narrow maintainable fix is to normalize Promise-valued stream returns into the existing `pumpStreamWithRecovery` path rather than adding a second recovery mechanism.
