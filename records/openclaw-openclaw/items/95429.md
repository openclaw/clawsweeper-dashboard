## Summary

Keep open. Current main and the latest release still have the Promise-valued embedded-agent recovery bypass, and https://github.com/openclaw/openclaw/pull/95430 is an open clean closing PR for this exact bug.

## What This Changes

The issue reports that Anthropic thinking-signature recovery is skipped when the embedded agent stream function returns a Promise, so mid-stream replay errors brick the session instead of triggering strip-and-retry recovery.

## Best Possible Solution

Keep this issue open while maintainers review and either land or reject https://github.com/openclaw/openclaw/pull/95430; close this issue only after the fix lands.

## Reproduction Assessment

Yes, source-level: runtime-auth embedded stream functions can return a Promise, and current main's recovery wrapper skips the recovery pump for Promise-resolved streams.

## Solution Assessment

Yes. The narrow maintainable fix is to route Promise-resolved streams through the existing recovery pump; the linked PR is the current candidate for that exact change.
