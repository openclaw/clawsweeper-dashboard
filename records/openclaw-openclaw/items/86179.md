- **[P2] Require Token Plan catalog to see a base URL:** `extensions/xiaomi/index.ts:82-85`
  - body: `hasConfiguredProviderEntry` treats any provider object as enough for `requireConfiguredProvider`, but this PR also accepts `models.providers["xiaomi-token-plan"]` with only `timeoutSeconds` and an empty normalized `baseUrl`. With a Token Plan key, that exposes the catalog using the manifest's Singapore default instead of forcing the user-selected regional endpoint, so China or Europe users can silently route to the wrong Token Plan host.
  - confidence: 0.94
- **[P2] Keep Token Plan docs costs tiered:** `docs/providers/xiaomi.md:201`
  - body: The runtime manifest uses `tieredPricing` for Token Plan cache-read costs at the 256k boundary, but this copyable config example flattens the costs to a single `cacheRead` value. Users who paste it lose the pricing behavior the PR adds and usage estimates can diverge from the bundled catalog; include the same tiered blocks for both Token Plan models or avoid copying full costs here.
  - confidence: 0.9
