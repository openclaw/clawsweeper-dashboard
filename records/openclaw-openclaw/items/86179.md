Full review comments:

- **[P2] Require a Token Plan baseUrl before activating the catalog:** `extensions/xiaomi/index.ts:97-99`
  - body: `requireConfiguredProvider` treats any `xiaomi-token-plan` config object as enough to enable the catalog.   - confidence: 0.9
- **[P2] Reject keys that do not match the selected Xiaomi path:** `extensions/xiaomi/index.ts:126-134`
  - body: The new mismatch helper only rejects the opposite known prefix, so a value like `bad-key` can pass onboarding and be persisted even though the docs and linked request define `sk-...` for pay-as-you-go and `tp-...` for Token Plan. Enforce the expected prefix for each auth path or remove the documented validation promise.
  - confidence: 0.87
