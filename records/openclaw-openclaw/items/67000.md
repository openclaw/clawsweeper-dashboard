## Summary

Keep open. Current main still lacks warm standby or persistent session reuse for Active Memory embedded recalls; #50963 overlaps, but it is broader expert-subagent reuse and does not supersede these Active Memory-specific latency reports and runner-boundary questions.

## What This Changes

Add warm standby or session reuse for Active Memory embedded-agent recalls so repeated invocations skip cold embedded-run setup.

## Best Possible Solution

Design a bounded per-parent-session Active Memory warm cache or generic embedded-runner reuse seam with explicit TTL, invalidation, and isolation semantics before implementing it.

## Reproduction Assessment

Yes for the missing reuse behavior: static inspection shows uncached recalls create a fresh Active Memory sub-agent session id/key and enter runEmbeddedPiAgent each time. The exact wall-clock latency still needs a live gateway/provider timing run.

## Solution Assessment

Unclear. Warm standby/session reuse is plausible, but the best maintainable boundary still needs maintainer design because it may belong in the Active Memory plugin, a generic embedded-runner API, or a hybrid.
