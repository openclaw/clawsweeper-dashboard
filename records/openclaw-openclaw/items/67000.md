## Summary

Keep open: current main still lacks Active Memory warm standby or persistent embedded-session reuse, and the related latency reports narrow adjacent overhead sources without replacing this lifecycle/API design request.

## What This Changes

Request Active Memory warm standby or session reuse for embedded recall subagents so each eligible message does not cold-start `runEmbeddedPiAgent`.

## Best Possible Solution

Design a bounded per-parent-session Active Memory warm cache or generic embedded-runner reuse seam with explicit TTL, invalidation, isolation semantics, docs, and regression coverage.

## Reproduction Assessment

Yes for the missing reuse behavior: source inspection shows uncached Active Memory recalls still create a fresh subagent session id/file and call `runEmbeddedPiAgent` each time. I did not establish a current-main wall-clock latency benchmark.

## Solution Assessment

Unclear. Warm standby/session reuse is plausible, but maintainers should first choose whether the boundary belongs in the Active Memory plugin, a generic embedded-runner reuse API, or a hybrid with strict isolation.
