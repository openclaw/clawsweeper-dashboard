## Summary

Keep open: current main still lacks Active Memory warm standby or persistent embedded-session reuse, and the live thread plus related searches show this remains an active lifecycle/performance request rather than a cleanup-close candidate.

## What This Changes

Request to add warm standby, pre-initialization, or persistent session reuse for Active Memory embedded agents to reduce per-message cold-start latency.

## Best Possible Solution

Design a bounded per-parent-session Active Memory warm cache or generic embedded-runner reuse contract with explicit TTL, invalidation, isolation, locking, docs, and regression coverage.

## Reproduction Assessment

Not applicable as a strict bug reproduction, but yes for the missing behavior at source level: current main still creates a fresh Active Memory embedded session and runs the embedded setup path on cache misses. I did not run a live timing benchmark in this read-only review.

## Solution Assessment

Unclear. Warm standby is plausible, but maintainers should first choose whether lifecycle ownership belongs in Active Memory, the generic embedded runner, or a hybrid API with strict TTL and isolation rules.
