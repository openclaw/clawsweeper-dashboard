## Summary

Keep open: current main still lacks Active Memory warm standby or persistent embedded-session reuse, and the remaining work is a feature/API lifecycle design rather than conservative cleanup.

## What This Changes

The issue requests warm standby or persistent session reuse for Active Memory embedded-agent runs to reduce per-message cold-start latency.

## Best Possible Solution

Design a bounded per-parent-session Active Memory warm cache or generic embedded-runner reuse seam with explicit TTL, invalidation, isolation semantics, docs, and regression coverage.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is an enhancement request. Source inspection gives a high-confidence path for the missing behavior: uncached Active Memory recalls still create a fresh embedded session and run setup each time.

## Solution Assessment

Unclear. Warm standby is plausible, but maintainers should first choose whether the boundary belongs in Active Memory, the generic embedded runner, or a hybrid API with strict TTL and isolation rules.
