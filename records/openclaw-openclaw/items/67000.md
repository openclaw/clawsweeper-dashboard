## Summary

Keep open: current main still lacks Active Memory warm standby or persistent embedded-session reuse, and multiple reporters added latency evidence showing this is still a meaningful lifecycle/performance feature request.

## What This Changes

Request warm-up or session reuse for Active Memory embedded agent runs so repeated recalls avoid per-invocation setup latency.

## Best Possible Solution

Design a bounded per-parent-session Active Memory warm cache or generic embedded-runner reuse seam with explicit TTL, invalidation, isolation semantics, docs, and regression coverage.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is an enhancement request. Source inspection gives a high-confidence missing-behavior path: uncached Active Memory recalls still create fresh embedded-runner setup instead of reusing a warm session.

## Solution Assessment

Unclear. Warm standby is plausible, but maintainers should first choose whether the lifecycle boundary belongs in Active Memory, the generic embedded runner, or a hybrid API with strict TTL and isolation rules.
