## Summary

Keep open: the patch is aimed at the right A2A announce helper, but it still misses valid account-scoped direct-session keys produced by current routing, so it should not merge as-is.

## What This Changes

The branch adds direct/dm fallback parsing in `resolveAnnounceTargetFromKey` and regression tests for Feishu direct, dm alias, and null delivery-target fallback.

## Best Possible Solution

Use a canonical direct-session parsing path that handles both simple and account-scoped direct/dm keys, preserves `accountId`, and leaves existing group/channel/thread parsing under the current plugin conversation contracts.

## Reproduction Assessment

Yes. Source inspection shows current main rejects direct/dm keys before announce target construction, and the remaining account-scoped failure follows directly from `buildAgentPeerSessionKey` output plus the PR's `raw[1]` kind check.

## Solution Assessment

No. The patch touches the right helper and keeps plugin delivery routing first, but the ad hoc segment indexing needs to handle an optional account segment and return `accountId`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Handle account-scoped direct session keys:** `src/agents/tools/sessions-send-helpers.ts:38-42`

Parse the optional account segment and return it as `accountId`.
