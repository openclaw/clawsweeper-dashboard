## Summary

Keep open: current main and v2026.6.10 still miss the direct/dm announce-target fallback, but this PR is not merge-ready because it routes direct peer ids through the conversation-oriented delivery hook before direct-user normalization, which can misroute Slack-like direct messages.

## What This Changes

The PR adds direct/dm and account-scoped direct/dm fallback parsing in `resolveAnnounceTargetFromKey`, plus regression tests for parser misses and null plugin delivery-target fallback.

## Best Possible Solution

Land a revised fallback that recognizes direct/dm keys while preserving direct-user target semantics, preferably by reusing the existing session-delivery parser or normalizing `user:<id>` before any conversation routing.

## Reproduction Assessment

Yes, at source level: current main rejects direct/dm keys in the session conversation parser, then `resolveAnnounceTargetFromKey` returns null and A2A announce delivery has no target to send.

## Solution Assessment

No. The helper is the right narrow surface, but the PR needs to preserve direct-user semantics before conversation routing and add Slack-like coverage before it is the best fix.
