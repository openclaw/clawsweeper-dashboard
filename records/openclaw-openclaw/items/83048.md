## Summary

Keep open: the PR targets a real direct/dm A2A announce gap, but the current patch can route direct peer ids through conversation-target hooks before user-target normalization, which can misroute Slack/Mattermost-style direct announces.

## What This Changes

The PR adds direct/dm and account-scoped direct/dm fallback parsing in `resolveAnnounceTargetFromKey`, with regression tests for direct, dm, account-scoped variants, and null plugin target fallback.

## Best Possible Solution

Land a revised fallback that recognizes direct/dm keys while preserving direct-user target semantics, preferably by reusing the existing direct-aware session delivery parser or normalizing `user:<id>` before any conversation routing.

## Reproduction Assessment

Yes, at source level: current main rejects direct/dm keys in the session conversation parser, then `resolveAnnounceTargetFromKey` returns null and A2A announce delivery has no target to send.

## Solution Assessment

No. The helper is the right narrow surface, but the PR needs to preserve direct-user semantics before conversation routing and should reuse or mirror the existing direct-aware delivery route parser.
