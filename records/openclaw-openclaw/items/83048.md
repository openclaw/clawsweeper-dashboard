## Summary

Keep open. The PR fixes a real source-proven A2A announce gap for simple direct/dm session keys, but the new fallback still misses account-scoped direct session keys that OpenClaw itself can generate, so the patch needs a small parser/accountId follow-up before merge.

## What This Changes

The PR adds a direct/dm fallback parser in `resolveAnnounceTargetFromKey` and regression tests for Feishu direct, dm alias, and null plugin delivery-target fallback handling.

## Best Possible Solution

Use a canonical direct-session parsing path that handles both simple and account-scoped direct/dm keys, preserves `accountId`, and leaves existing group/channel/thread parsing under the current plugin conversation contracts.

## Reproduction Assessment

Yes. Source inspection shows current main rejects direct/dm keys before announce target construction, and the PR's new tests cover the simple direct/dm fallback path; the remaining account-scoped gap is also source-reproducible from the router's generated key shape.

## Solution Assessment

No. The fix is aimed at the right helper and keeps plugin delivery routing first, but the ad hoc segment indexing should be replaced or extended so account-scoped direct/dm keys resolve and carry `accountId`.
