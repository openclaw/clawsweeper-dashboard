## Summary

Keep open: current main still has only coarse memory/session source fields, citation string decoration, inline promotion markers, and separate memory-wiki provenance metadata; it does not implement the requested mandatory durable-memory provenance model across records, revisions, merges/edits, retrieval output, and review flows.

## What This Changes

Add mandatory source attribution for durable memories, including source refs, multi-source support, merge/edit preservation, retrieval display, and promotion rejection without provenance.

## Best Possible Solution

Define one canonical durable-memory source-reference and revision contract, then enforce it through schema, promotion/write paths, merge/edit logic, retrieval output, review flows, and Memory Wiki bridge boundaries.

## Reproduction Assessment

Not applicable. This is a feature/model request rather than broken existing behavior, and source inspection shows partial attribution surfaces instead of a current-main failing path to reproduce.

## Solution Assessment

Yes as a product direction, but not as a narrow citation-only fix. The maintainable solution is a canonical source-reference and revision model that every durable-memory writer and reader uses.
