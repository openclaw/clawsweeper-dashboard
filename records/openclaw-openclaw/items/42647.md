## Summary

Keep open: current main and v2026.6.6 have citation-level source metadata, promotion annotations, memory events, and a provenance-rich memory-wiki layer, but they do not implement the requested durable multi-source source-reference/revision model or a gate that prevents source-less durable memories.

## What This Changes

Request to add a durable memory provenance model with source references for session/message/file origins, multi-source preservation across merges/edits, retrieval attribution, and promotion blocking for source-less memories.

## Best Possible Solution

Keep this in the Memory MVP cluster and decide the canonical durable memory schema, provenance authority, merge/edit lineage, and promotion/review semantics before code locks the contract.
