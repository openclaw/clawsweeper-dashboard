## Summary

Keep open: current main has useful path/line citations and short-term promotion metadata, but it does not implement the requested durable provenance model with required source refs, multi-source support, merge/edit provenance retention, and promotion blocking for source-less memories.

## What This Changes

Requests a first-class source-attribution and provenance model for every durable memory.

## Best Possible Solution

Keep this as the provenance-focused memory MVP item and resolve it with the v1 schema/write-pipeline work: durable source-ref records, revision/merge provenance history, promotion gates for source-less candidates, and retrieval/review output that exposes attribution clearly.

Not applicable: this is a design and feature request, not a current-main bug with a failing reproduction path. Source inspection shows partial citation/provenance metadata exists, but not the complete durable provenance model requested.

## Solution Assessment

Unclear as an implementation path: source attribution is the right requirement, but the final shape should be decided with the v1 memory schema, write pipeline, and review-surface issues rather than patched into one path.
