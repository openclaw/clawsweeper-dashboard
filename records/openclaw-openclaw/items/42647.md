## Summary

Keep open: current main and v2026.5.12 still have coarse memory source labels, citation display, promotion source strings, session-input provenance, and separate Memory Wiki provenance, but not the requested mandatory durable-memory provenance model across records, revisions, merges/edits, retrieval output, and review flows. Related Memory MVP schema/write-pipeline/trust-tagging work remains open and this issue already has maintainer/product-review labels, so it should stay in the Memory MVP design queue.

## What This Changes

The issue requests a mandatory source-attribution/provenance model for durable memories, including source refs, multi-source support, merge/edit preservation, retrieval attribution, and rejection of source-less durable promotion.

## Best Possible Solution

Define one canonical durable-memory source-reference and revision contract, then enforce it across schema, promotion/write paths, merge/edit logic, retrieval output, review flows, and wiki bridge boundaries.

## Reproduction Assessment

Not applicable. This is a feature/model request rather than broken existing behavior, and current source inspection shows partial attribution features instead of a failing current-main path to reproduce.

## Solution Assessment

Yes as a product direction, but it should be solved as a cross-cutting durable-memory contract rather than by adding another display-level citation marker.
