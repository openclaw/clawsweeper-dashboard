## Summary

Keep open: current main and v2026.6.6 have adjacent source labels, citation metadata, promotion annotations, event logs, and Memory Wiki provenance, but they still do not implement the requested durable multi-source provenance model, merge/edit lineage, or gate that prevents source-less durable memories.

## What This Changes

Request to add mandatory source attribution for durable memories, including session/message/file source refs, multi-source support, retained provenance across merges or edits, retrieval display, and promotion enforcement.

## Best Possible Solution

Settle one canonical Memory MVP provenance contract, then implement required source refs, merge/edit lineage, retrieval attribution, and promotion/review enforcement at the memory-plugin storage and tool boundaries.
