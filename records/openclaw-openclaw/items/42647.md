## Summary

Keep open: current main still has coarse memory/session sources, presentation citations, single-source promotion markers, and Memory Wiki provenance, but not the requested mandatory durable-memory provenance model across records, revisions, merges/edits, retrieval output, and review flows.

## What This Changes

Add canonical source attribution and provenance for durable memories, including source refs, multi-source support, merge/edit preservation, retrieval display, and promotion blocking for source-less memories.

## Best Possible Solution

Define one canonical durable-memory source-reference and revision contract, then enforce it consistently in promotion, direct writes, merge/edit logic, retrieval output, and review flows.

## Reproduction Assessment

Not applicable. This is a feature/model request rather than broken existing behavior, and current-main source inspection shows only selective citation, session source, and Memory Wiki provenance paths today.

## Solution Assessment

Yes as a product direction, but it should be designed across durable records, revisions, write paths, retrieval, and review flows rather than added as a one-file citation marker.
