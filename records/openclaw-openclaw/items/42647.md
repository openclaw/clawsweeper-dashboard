## Summary

Keep open: current main and the latest shipped tag still only provide retrieval citations, chunk/file source fields, wiki-local provenance, and inline promotion source markers, not the requested mandatory durable-memory provenance model with multi-source refs, revision/merge preservation, and source-less durable promotion rejection.

## What This Changes

Request a durable-memory provenance model that requires source references, supports multiple sources, preserves attribution across merges/edits, and exposes attribution in retrieval and review flows.

## Best Possible Solution

Define one canonical durable-memory source-reference and revision contract, then enforce it through schema, promotion/write paths, merge/edit logic, retrieval output, review flows, and Memory Wiki bridge boundaries.
