## Summary

Keep open. Current main has useful adjacent provenance pieces, but it does not yet implement the requested end-to-end durable memory provenance model, and the paired schema/write-pipeline MVP issues are still open.

## What This Changes

Add a provenance model so durable memories carry source references, preserve provenance through edits/merges, expose attribution on retrieval, and block source-less durable promotion.

## Best Possible Solution

Define the durable record and revision provenance contract as part of the Memory MVP schema/write pipeline, reusing existing citation and memory-wiki evidence concepts where they fit.

## Reproduction Assessment

Not applicable. This is a feature/design request, and source inspection shows partial attribution surfaces rather than a bug with a current-main reproduction path.

## Solution Assessment

Unclear until the schema and write pipeline are settled. The narrowest maintainable direction is to make provenance a required field in the durable record model and enforce it in the promotion/write path, rather than layering it only in retrieval citations.
