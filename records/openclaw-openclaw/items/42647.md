## Summary

Keep open: current main and the latest release still have only coarse source labels, citation display, promotion source markers, session input provenance, and Memory Wiki provenance, not the requested mandatory durable-memory provenance model across records, revisions, merges/edits, retrieval output, and review flows.

## What This Changes

The issue requests a durable memory provenance model with mandatory source references, merge/edit preservation, retrieval attribution, and promotion blocking for source-less memories.

## Best Possible Solution

Define one canonical durable-memory source-reference and revision contract, then enforce it across schema, promotion/write paths, merge/edit logic, retrieval output, review flows, and wiki bridge boundaries.

## Solution Assessment
