## Summary

Keep open: current main has retrieval citations, promotion source markers, and memory-wiki provenance tools, but it still lacks a mandatory durable-memory provenance contract that covers canonical records, revisions, merges/edits, retrieval, and review flows. The adjacent schema, write-pipeline, and trust-tagging work remains open in https://github.com/openclaw/openclaw/issues/42646, https://github.com/openclaw/openclaw/issues/42648, and https://github.com/openclaw/openclaw/issues/7707.

## What This Changes

Add a durable-memory provenance model requiring source references across promotion, merges/edits, retrieval, and review flows.

## Best Possible Solution

Design and implement canonical durable memory records with source-reference and revision storage, then wire promotion, direct writes, merge/edit logic, retrieval, and review flows so source-less durable writes are rejected and provenance survives updates.

## Reproduction Assessment

Not applicable; this is a feature/model request rather than a broken existing contract. Source inspection on current main gives a high-confidence state check: attribution exists in retrieval and memory-wiki, but mandatory durable provenance is not implemented.

## Solution Assessment

No; treating current main as the solution would overstate partial source markers and wiki-layer provenance. The maintainable fix should be scoped with the open schema/write-pipeline work and preserve source history as part of the durable record contract.
