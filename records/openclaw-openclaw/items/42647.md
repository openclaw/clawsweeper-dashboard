## Summary

Keep open: current main has coarse memory/session source labels, retrieval citations, and a provenance-rich Memory Wiki companion path, but it still does not implement the requested mandatory durable-memory source-reference model across durable records, revisions, merges/edits, retrieval, and review flows.

## What This Changes

Request to add mandatory provenance for durable memories, including source refs, multiple supporting sources, preservation across edits/merges, and clear attribution in retrieval and review flows.

## Best Possible Solution

Define one canonical durable-memory source-reference and revision contract, then enforce it in promotion, direct writes, merge/edit logic, retrieval output, and review flows.

## Reproduction Assessment

Not applicable; this is a feature/model request rather than broken existing behavior. Source inspection gives a high-confidence current-state check that attribution exists only in selected citation/wiki paths, not as mandatory durable provenance.

## Solution Assessment

Yes for triage: keeping this open for maintainer design is the best current path. A one-file marker or citation-only change would not satisfy durable source references across records, revisions, merges, retrieval, and review.
