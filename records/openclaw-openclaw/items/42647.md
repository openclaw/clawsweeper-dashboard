## Summary

Keep this open. Current main still has adjacent citations, source labels, memory-host events, and Memory Wiki provenance, but it does not implement the requested canonical durable-memory provenance model with required typed source references, multiple supporting sources, merge/edit lineage, or a gate that prevents source-less durable promotion.

## What This Changes

Add a durable memory provenance model that requires source references, preserves attribution across merges and edits, and exposes provenance in retrieval and review flows.

## Best Possible Solution

Implement provenance as part of the canonical Memory MVP schema and write pipeline: typed source references for session/message/file origins, support for multiple supporting sources, required provenance before durable promotion, provenance-preserving create/merge/edit/review history, and retrieval/review rendering that can show those references. Memory Wiki can consume or mirror that model, but should not be treated as the core durable-memory contract.

## Reproduction Assessment

Not applicable as a feature/schema request rather than a runtime bug. The high-confidence verification path is source and docs inspection of current main, which shows the requested durable provenance contract is still absent.

## Solution Assessment

Unclear until the related record-model and write-pipeline design is settled. The requested direction is appropriate for core Memory MVP work, but display-only citations and Memory Wiki provenance are not sufficient as the final solution.
