## Summary

Keep open: current main has coarse memory/session source labels, citation formatting, short-term promotion source markers, session input provenance, and Memory Wiki sourceIds, but it still does not implement the requested mandatory durable-memory provenance model across records, revisions, merges/edits, retrieval output, and review flows.

## What This Changes

Add a durable-memory provenance model requiring source references for every durable memory, preserving source history across merges/edits, and surfacing attribution in retrieval and review flows.

## Best Possible Solution

Define one canonical durable-memory source-reference and revision contract, then enforce it across schema, promotion/write paths, merge/edit logic, retrieval output, review flows, and wiki bridge boundaries.

## Reproduction Assessment

Not applicable. This is a feature/model request rather than broken existing behavior, and current-main source inspection shows only selective citation, source labels, session provenance, and Memory Wiki provenance paths today.

## Solution Assessment

Yes as a product direction, but it should be solved as a cross-cutting durable-memory contract rather than by adding another one-file citation marker.
