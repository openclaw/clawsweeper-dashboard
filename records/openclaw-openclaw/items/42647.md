## Summary

Keep open. Current main has partial source labels, citations, and promotion source markers, but it still does not implement the requested durable provenance model with mandatory source refs, multi-source history, merge/edit retention, and promotion blocking for source-less memories.

## What This Changes

Add a durable memory provenance model that records source references, preserves them through merges and edits, and exposes attribution in retrieval and review flows.

## Best Possible Solution

Define provenance as part of the Memory MVP storage, write-pipeline, retrieval, and review contract, then implement it in the memory plugin boundary with migration and regression coverage across current backends.

## Reproduction Assessment

Not applicable; this is a feature and data-model request, not a current-main bug report. Source inspection shows current main has partial attribution surfaces but not the requested durable provenance model.

## Solution Assessment

Unclear until maintainers choose the Memory MVP data boundary. A first-class provenance model is the right class of solution, but it should be designed with the open schema, write-pipeline, trust-tagging, and retrieval issues rather than patched into one backend.
