## Summary

Keep open: current main has citation labels, short-term promotion source strings, an event journal, and Memory Wiki provenance, but not the requested mandatory durable-memory provenance model across records, revisions, merges/edits, retrieval output, and review flows.

## What This Changes

Add a durable-memory provenance model so every durable memory carries source references, preserves provenance across merges/edits, and exposes attribution in retrieval and review flows.

## Best Possible Solution

Define one canonical durable-memory source-reference and revision contract, then enforce it across schema, promotion/write paths, merge/edit logic, retrieval output, review flows, and Memory Wiki bridge boundaries.
