## Summary

Keep open: current main and v2026.6.10 still have source labels, citations, session-input provenance, promotion source metadata, and Memory Wiki provenance-adjacent features, but not the requested required durable multi-source provenance model with merge/edit lineage or a gate blocking source-less durable memories.

## What This Changes

Add a required source-attribution and provenance model for durable memory, including multiple sources, merge/edit lineage, retrieval attribution, and promotion blocking for source-less memories.

## Best Possible Solution

Settle one Memory MVP provenance contract for the active memory plugin, then implement persistent source refs, merge/edit lineage, retrieval attribution, and source-required promotion across storage, write, retrieval, and review boundaries.

## Reproduction Assessment

Not applicable: this is a feature and data-model request, not a broken existing behavior report. Source inspection shows the requested durable provenance contract is still absent on current main and v2026.6.10.

## Solution Assessment

Unclear as a concrete implementation until maintainers choose the Memory MVP storage, write, retrieval, and review boundaries. The direction is sound, but it should land with the schema and lifecycle decisions rather than as a narrow one-off patch.
