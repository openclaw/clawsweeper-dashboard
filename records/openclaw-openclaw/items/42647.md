## Summary

Keep open: current main has retrieval citations, coarse memory/session source fields, inter-session input provenance, and separate memory-wiki provenance metadata, but it still lacks the requested mandatory durable-memory provenance model across records, revisions, merges/edits, retrieval output, and review flows.

## What This Changes

The issue requests mandatory source attribution and provenance for durable memory records, including multi-source support, merge/edit preservation, retrieval attribution, and rejection of source-less durable promotion.

## Best Possible Solution

Define one canonical durable-memory source-reference and revision contract, then enforce it through schema, promotion/write paths, merge/edit logic, retrieval output, review flows, and Memory Wiki bridge boundaries.

## Reproduction Assessment

Not applicable. This is a feature/model request rather than broken existing behavior, and source inspection shows partial attribution surfaces instead of a failing current-main path to reproduce.

## Solution Assessment

Yes as a product direction, but not as a narrow one-off citation fix. The maintainable solution is a canonical source-reference and revision model that every durable-memory writer and reader uses.
