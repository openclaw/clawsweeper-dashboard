## Summary

Keep open: current main has useful retrieval citations and source labels for some dreaming promotions, but it does not implement the requested mandatory durable provenance model across all memory writes, merges, edits, and retrieval/review flows.

## What This Changes

The issue requests a mandatory provenance model for durable memories, including source references, merge/edit retention, retrieval attribution, and blocking source-less durable promotion.

## Best Possible Solution

Define a memory-core durable provenance contract alongside the schema and write-pipeline MVP work, while keeping memory-wiki as an optional provenance-rich knowledge layer rather than the only enforcement path.

## Reproduction Assessment

Not applicable; this is a feature/model request rather than a broken existing contract. Source inspection gives a high-confidence current-state check: citations and promotion source markers exist, but mandatory durable provenance is not implemented.

## Solution Assessment

No; treating current main as the solution would overstate partial attribution. The maintainable fix needs a designed durable record/provenance model that covers promotion, direct writes, merges, edits, retrieval, and review surfaces.
