## Summary

Keep open: current main has source labels, retrieval citations, and Memory Wiki provenance, but it still lacks a mandatory durable-memory source-reference contract covering durable records, revisions, merges/edits, retrieval, and review flows.

## What This Changes

Add a durable-memory provenance model requiring source references for session, message, and file origins, preserving them through writes, merges, edits, retrieval, and review.

## Best Possible Solution

Design a canonical durable-memory source-reference and revision contract, then enforce it in promotion, direct writes, merge/edit logic, retrieval output, and review flows across the active memory surfaces.

## Reproduction Assessment

Not applicable; this is a feature/model request rather than broken existing behavior. Source inspection gives a high-confidence state check that attribution exists in selected presentation and wiki paths, but mandatory durable provenance is not implemented.

## Solution Assessment

Yes for triage: keeping this open for maintainer design is the best current path. A one-file marker or citation-only change would not satisfy durable source references across records, revisions, merges, retrieval, and review.
