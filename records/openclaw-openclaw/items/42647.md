## Summary

Keep open: current main still has source labels, citations, promotion metadata, event logs, and Memory Wiki provenance-adjacent features, but it does not implement a durable multi-source provenance model with merge/edit lineage or a gate blocking source-less durable memories.

## What This Changes

Request a durable memory provenance model with source refs for session/message/file origins, multi-source support, merge/edit retention, retrieval attribution, and promotion enforcement.

## Best Possible Solution

Settle one canonical Memory MVP provenance contract, then implement required source refs, merge/edit lineage, retrieval attribution, and promotion/review enforcement at the memory-plugin storage and tool boundaries.
