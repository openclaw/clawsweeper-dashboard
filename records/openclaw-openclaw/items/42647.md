## Summary

Keep open: current main and v2026.6.10 have source buckets, citations, and path-line promotion metadata, but they still do not implement the requested required durable multi-source provenance model with merge/edit lineage or a gate blocking source-less durable memories.

## What This Changes

The issue requests a durable memory provenance model that records source attribution for every promoted memory, preserves provenance through merges/edits, and exposes attribution in retrieval and review flows.

## Best Possible Solution

Settle one Memory MVP provenance contract for active memory, then implement persistent source refs, merge/edit lineage, retrieval attribution, and source-required promotion across storage, write, retrieval, and review boundaries.

## Reproduction Assessment

Not applicable: this is a Memory MVP data-contract request, not a broken current behavior report. Source inspection shows the requested durable provenance contract is still absent on current main and v2026.6.10.

## Solution Assessment

Unclear as a concrete implementation until maintainers choose the durable memory storage, promotion, retrieval, and review boundaries. The direction is sound, but it should land with the sibling schema/write/review surface decisions rather than as a one-off patch.
