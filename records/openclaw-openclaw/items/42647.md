## Summary

Keep open: current main and v2026.6.8 still have source labels, citations, session input provenance, promotion metadata, and Memory Wiki provenance-adjacent features, but not the requested required durable multi-source provenance model with merge/edit lineage or a gate blocking source-less durable memories.

## What This Changes

Add a durable memory provenance model with required source references, multiple supporting sources, merge/edit lineage retention, retrieval attribution, and promotion gating.

## Best Possible Solution

Settle one Memory MVP provenance contract across storage, promotion, retrieval, and review/edit boundaries, then implement it consistently across the active memory path and documented plugin seams.

## Reproduction Assessment

Not applicable: this is a feature and data-model request, not a broken existing behavior report. Source inspection shows the requested durable provenance contract is still missing on current main.

## Solution Assessment

Unclear as a concrete implementation until maintainers choose the Memory MVP storage and review boundaries. The requested direction is sound, but the best solution must line up with schema, write pipeline, retrieval, and review/edit work.
