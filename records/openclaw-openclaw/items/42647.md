## Summary

Keep open: current main has useful retrieval citations, short-term promotion source markers, and a provenance-rich memory-wiki layer, but it still does not implement mandatory durable provenance across all memory records, merges, edits, retrieval, and review flows.

## What This Changes

Request to add a mandatory source-attribution and provenance model for durable memories, including source refs, multi-source support, merge/edit retention, retrieval display, and blocking sourceless durable promotion.

## Best Possible Solution

Define an explicit durable-memory provenance contract in the memory-core/host schema and write pipeline, then let memory-wiki consume that provenance as a richer knowledge layer rather than being the only enforcement path.

## Reproduction Assessment

Not applicable; this is a feature/model request rather than a broken existing contract. Source inspection gives a high-confidence current-state check: attribution exists in retrieval and one promotion path, but mandatory durable provenance is not implemented.

## Solution Assessment

No; treating current main as the solution would overstate partial attribution. The maintainable fix needs a designed durable record and provenance model that covers promotion, direct writes, merges, edits, retrieval, and review surfaces.
