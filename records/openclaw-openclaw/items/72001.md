## Summary

Keep open. Current main still has the Gmail wizard gap, so the PR remains relevant, but the proposed patch is not merge-ready: it only seeds the prefix needed for startup validation, while mapped Gmail delivery still fails the templated-session runtime gate, and the unconditional prefix can invalidate supported configs that shadow the Gmail preset with a static mapping. The discussion supports this as a real Gmail setup path, including AS76's tested Google Workspace/Gmail context, but the PR needs a tighter session-policy fix before merge.

## What This Changes

This PR adds a Gmail hook prefix merge helper, has `openclaw webhooks gmail setup` write `hooks.allowedSessionKeyPrefixes: ["hook:"]`, adds Gmail/gateway tests, and adds a changelog fix entry.

## Best Possible Solution

Revise the wizard so it emits a complete Gmail hook session policy for the effective mapping it installs: if the templated Gmail preset remains effective, write a bounded `hook:` prefix and enable the templated/request session-key path deliberately; if a user-provided static mapping or default session shadows that preset, preserve the existing compatible policy. Add coverage for `resolveHooksConfig`, mapped `/hooks/gmail` dispatch, and the static-shadow/default-session compatibility case, then keep the changelog entry on one physical line.

## Reproduction Assessment

Yes. The original bug is reproducible by source inspection on current main: the wizard writes the Gmail preset, the preset has a templated session key, and `resolveHooksConfig` requires prefixes. The PR regressions are also reproducible by following the mapped dispatch path to `resolveHookSessionKey` and the existing default-session/static-shadow tests; tests were not run because this was a read-only review.

## Solution Assessment

No. The patch is a useful partial fix, but prefix seeding alone only gets past startup validation; it does not enable templated Gmail mapped delivery and can break existing compatible hook configs. A safer solution needs to emit the full policy only when the templated preset is the effective mapping, or switch Gmail setup to a static session strategy.
