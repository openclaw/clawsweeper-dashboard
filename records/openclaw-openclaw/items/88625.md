## Summary

Keep open: this maintainer-labeled PR is a focused session-state bug fix for the linked reset/lifecycle race, and source review found no blocking correctness or security finding. It should remain open for maintainer merge handling and as the canonical replacement for https://github.com/openclaw/openclaw/pull/88583.

## What This Changes

The branch stamps lifecycle events with the owning run session id, skips stale session lifecycle persistence and snapshot projection after a reset rotates the session row, and adds regression coverage for the emit, persist, projection, and follow-up compaction paths.

## Best Possible Solution

Land one canonical session-id guarded lifecycle fix after maintainer review accepts the bounded session-state risk, then retire the replaced implementation at https://github.com/openclaw/openclaw/pull/88583.

## Reproduction Assessment

Yes, source-reproducible: current main writes lifecycle state by sessionKey without a sessionId identity check and projects lifecycle snapshots the same way. I did not run the full live Slack race, but the linked issue provides a concrete reset/late-event sequence and sanitized state evidence.

## Solution Assessment

Yes, the proposed fix is the narrow maintainable path: stamp the owning session id once on lifecycle events, guard both persistence and snapshot projection, and preserve legacy behavior when the owning id is unknown. The remaining choice is proof depth before merge, not a different implementation direction.
