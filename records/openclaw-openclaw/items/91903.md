## Summary

## What This Changes

The PR adds sender and transport-native chat identity fields plus extensible nested channel metadata to plugin agent hook contexts, and propagates native chat IDs through Feishu, reply, queued-followup, CLI, command, and embedded-agent paths.

## Best Possible Solution

Place the augmentable identity declarations in a stable public SDK module that core itself consumes, remove `senderExternalId` unless a shipped contract is proven, add an external-plugin type contract test, and demonstrate a redacted live channel hook payload.

## Solution Assessment
