## Summary

Keep open: the PR carries a useful SDK feature that is not already implemented on current main, but it is not merge-ready because it collides with the current public `ctx.channel` string contract and expands exec subprocess identity exposure without an agreed boundary.

The PR adds structured sender/chat metadata types to plugin hook contexts, threads `channelContext` through inbound/agent/follow-up/exec paths, and exposes it to `exec` via `OPENCLAW_CHANNEL_CONTEXT`.

## Best Possible Solution

Keep `ctx.channel` as the channel/plugin id, expose structured sender/chat metadata under a non-conflicting field such as `ctx.channelContext`, and land exec exposure only after the metadata boundary is documented, redacted, or explicitly approved.
