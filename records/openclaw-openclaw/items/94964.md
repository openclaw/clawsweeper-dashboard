## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Check cancellation after the poll sleep:** `src/gateway/server-reload-handlers.ts:318-326`
  - body: `pendingChannelReloadAborted` is only checked before the 500ms sleep. If SIGUSR1 arrives during that sleep and active work drains before the timer fires, this returns success and the reload still restarts channels, preserving the reported race window.
  - confidence: 0.93
- **[P2] Abort only after SIGUSR1 is accepted:** `src/cli/gateway-cli/run-loop.ts:724-733`
  - body: This abort runs before the handler reads restart intent or authorization. An unauthorized SIGUSR1 that current code would ignore can now cancel a pending channel reload without any restart to apply the new channel config.
  - confidence: 0.9
- **[P2] Recheck cancellation before starting the channel:** `src/gateway/server-reload-handlers.ts:529-533`
  - body: Once the wait has returned, a restart accepted while `stopChannel()` is in flight is never observed before `startChannel()` runs. Recheck the durable cancellation signal after stop and before start so the old reload cannot race the restarted gateway.
  - confidence: 0.84
