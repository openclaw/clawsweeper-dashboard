Full review comments:

- **[P2] Cancel reloads before shutdown reaches channel work:** `src/infra/restart.ts:116-124`
  - body: This calls `abortPendingChannelReloads()` from `resetGatewayRestartStateForInProcessRestart()`, but current run-loop code reaches that reset only after `server.close(...)` completes and the next restart iteration begins. A deferred reload can wake during the shutdown window, before this flag is set, and still run the channel stop/start path that causes the duplicate listener; cancellation needs to be owned by the shutdown/restart lifecycle or active config-reloader drain.
  - confidence: 0.9
- **[P1] Preserve shell-style escaped QMD commands:** `packages/memory-host-sdk/src/host/backend-config.ts:397-411`
  - body: This branches on any backslash before using the shell-aware tokenizer, so a valid POSIX-style command like `/Applications/QMD\ Tools/qmd --flag` will be split as `/Applications/QMD\` instead of preserving the escaped space. Restrict the special case to actual Windows drive/UNC command paths, or keep shell parsing for non-Windows command strings.
  - confidence: 0.88
- **[P2] Split the unrelated QMD command fix:** `packages/memory-host-sdk/src/host/backend-config.ts:390-411`
  - body: This gateway PR also changes `memory.qmd.command` parsing and its commit message says it fixes https://github.com/openclaw/openclaw/issues/92302.
