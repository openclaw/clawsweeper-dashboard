## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Signal reload cancellation before shutdown proceeds:** `src/infra/restart.ts:113-115`
  - body: `abortPendingChannelReloads()` is called from `resetGatewayRestartStateForInProcessRestart()`, but that reset runs in the next loop iteration after the current server has already closed. The reported stale reload resumes during the old lifecycle shutdown window, so this can set the flag after `stopChannel`/`startChannel` already raced; signal cancellation when restart is accepted or when the config reloader is stopped/drained.
  - confidence: 0.91
- **[P2] Gate channel restart after the deferral wait:** `src/gateway/server-reload-handlers.ts:313-316`
  - body: The new flag is only checked inside the polling loop. If the wait returns because active work drains or times out, a restart arriving before or during the later channel restart loop still lets stale reload code stop/start channels; carry a cancelled result through `applyHotReload` and recheck before channel start.
  - confidence: 0.88
- **[P2] Split the unrelated QMD command parser fix:** `packages/memory-host-sdk/src/host/backend-config.ts:390-412`
  - body: This gateway PR also changes `memory.qmd.command` parsing and tests for the separate Windows QMD issue. That behavior has its own open focused candidate at https://github.com/openclaw/openclaw/pull/95274; keeping it here makes the gateway fix carry unrelated config compatibility risk.
  - confidence: 0.95
