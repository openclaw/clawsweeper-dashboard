## Summary

Keep this PR open but not merge-ready: the gateway abort hook is wired too late to cancel the shutdown-window reload that caused the Telegram duplicate listener race, and the branch also bundles an unrelated QMD Windows command parser fix.

## What This Changes

The PR adds a deferred channel reload abort flag invoked from in-process restart reset, and also changes QMD command parsing/tests for Windows paths.

## Best Possible Solution

Thread shutdown/restart cancellation through config hot reloads, stop and drain active reload work before channel teardown, add targeted regression coverage, and land the QMD Windows command fix separately through its focused PR.

## Reproduction Assessment

Yes, at source level: current main lets an active deferred channel reload resume into channel stop/start while restart shutdown is in progress. I did not run a live Telegram webhook/SIGUSR1 reproduction in this read-only review.

## Solution Assessment

No. The abort flag is a plausible mitigation, but it is wired at the next-start reset boundary and does not cancel the active reload during `server.close()`; lifecycle-owned cancellation and drain is the narrower maintainable fix.
