## Summary

Keep open: current main still has the deferred channel reload/restart race, and this PR is a relevant candidate fix, but it still misses an accepted restart path. ## What This Changes

The branch exports a deferred channel reload abort hook, calls it from SIGUSR1/restart reset paths, adds generation checks to channel reload waits and restarts, adds gateway reload tests, and removes a stale Telegram reasoning comment.

## Best Possible Solution

Land one canonical gateway-owned cancellation/drain fix that aborts stale deferred reload work for every accepted restart path, including file-intent SIGUSR1 restarts, with focused regression coverage; then close or supersede competing candidate PRs.

## Reproduction Assessment

Yes, at source and log level. The canonical issue gives a concrete Telegram EADDRINUSE timeline, current main shows deferred channel reloads resume into stop/start without cancellation, and the PR gap is traceable through the file-intent SIGUSR1 path.

## Solution Assessment

No. The generation-based abort is a plausible mitigation, but this patch does not abort every accepted restart path; the narrow fix is to cancel before any restartIntent request, or use the lifecycle-owned config-reloader cancellation shape if maintainers choose that candidate.
