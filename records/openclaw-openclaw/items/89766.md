## Summary

Keep open. Current main and the latest release still classify a processing session with fresh CLI-live progress but no active work kind as recoverable stuck state, and the focused repair remains open in https://github.com/openclaw/openclaw/pull/89831.

## What This Changes

Prevent isolated `claude-cli` cron sessions that are still streaming from being classified as `queued_work_without_active_run` and recovered with `release_lane released=0`.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/89831; after it merges, close this issue from that fix and track any remaining runtime-plugin stall separately.

## Reproduction Assessment

Yes, at source level: the cron CLI path can emit fresh `cli_live:stream_progress` while `activeWorkKind` is undefined, and the current classifier still marks that state recoverable stuck. I did not run the multi-hour live gateway scenario.

## Solution Assessment

Yes for the source-confirmed false recovery path: the open PR's freshness gate is the narrowest current repair. A broader CLI active-work taxonomy can be considered later, but is not required to stop fresh progress from being recovered as stale.
