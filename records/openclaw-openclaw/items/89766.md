## Summary

Keep open. Current main and v2026.6.8 still have the no-active-work classifier fallback that treats fresh CLI-live progress as recoverable stuck state, and the focused repair in https://github.com/openclaw/openclaw/pull/89831 is still open rather than merged.

## What This Changes

The issue asks OpenClaw to stop classifying isolated cron claude-cli runs as stale while they are actively streaming `cli_live:stream_progress`, because recovery then reports `released=0` and later isolated runs stall until restart.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/89831 if maintainers accept the fresh-progress recovery gate, then close this issue from the merged fix and track any remaining isolated-cron stall under the narrower adjacent reports.

## Reproduction Assessment

Yes. Source inspection shows a current-main path where `cli_live:stream_progress` updates fresh progress without setting `activeWorkKind`, then the classifier fallback marks queued no-active-work sessions as recoverable stuck; the multi-hour gateway stall was not live-reproduced here.

## Solution Assessment

Yes for the source-confirmed classifier bug. The linked PR is the narrowest current repair because it gates only fresh untracked progress while leaving stale or missing-progress sessions recoverable; maintainers still need to accept the recovery-delay tradeoff.
