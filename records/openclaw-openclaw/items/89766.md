## Summary

Keep open. Current main and v2026.6.8 still classify fresh CLI-live progress without an active work kind as recoverable stuck state, while the focused repair remains open in https://github.com/openclaw/openclaw/pull/89831.

## What This Changes

The issue reports isolated cron claude-cli runs being misclassified as stuck while actively streaming, followed by release_lane returning released=0 and later isolated cron stalls.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/89831 for the classifier false-recovery path, then track any remaining runtime-plugins stall as a narrower follow-up if it still reproduces.

## Reproduction Assessment

Yes. Source inspection shows the isolated cron CLI path can emit fresh cli_live:stream_progress with no activeWorkKind, and the current classifier fallback marks that state recoverable stuck.

## Solution Assessment

Yes for the source-confirmed classifier bug. The open PR is the narrowest current repair, while a broader CLI active-work taxonomy is optional follow-up and the bounded recovery-delay policy still needs maintainer acceptance.
