## Summary

Keep open. Current main and the latest release still classify fresh `cli_live:stream_progress` with no active work kind as recoverable stuck state, and the focused repair is still open in https://github.com/openclaw/openclaw/pull/89831.

## What This Changes

The issue reports isolated cron jobs on the `claude-cli` backend being misclassified as stuck while actively streaming, followed by ineffective `release_lane released=0` recovery and later stalled cron runs.

## Best Possible Solution

Review and land or revise https://github.com/openclaw/openclaw/pull/89831 for the source-confirmed classifier bug, then track any remaining runtime-plugin stall as a separate follow-up if it persists.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: a cron CLI run can emit fresh `cli_live:stream_progress` while `activeWorkKind` remains undefined, and the classifier fallback still marks that state recoverable stuck. I did not run the multi-hour live gateway scenario.
