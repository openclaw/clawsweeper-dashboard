## What This Changes

The PR adds Voice Call realtime agent context and consult-policy config, Google Meet Twilio session routing/audio updates, provider lazy-loading/reconnect changes, docs, changelog, generated config baseline, and tests.

## Best Possible Solution

Build realtime instructions from the per-call effective route, make consultPolicy semantics match docs, and derive requester-session routing from trusted context before merge.

## Reproduction Assessment

Yes. Source inspection gives high-confidence paths: routed inbound calls still build provider instructions from global config, `consultPolicy: always` schedules a forced consult for any final transcript, and legacy voice_call mode forwards raw requesterSessionKey.

## Solution Assessment

No. The feature direction is useful, but the current patch should resolve route-effective instructions, forced consult semantics, and trusted requester-session derivation before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Do not trust caller-supplied requester sessions:** `extensions/voice-call/index.ts:793-795`
  - body: The legacy voice_call path forwards rawParams.requesterSessionKey into the outbound call. That value later becomes spawnedBy for realtime consults and is used to fork/read delivery context, so a model/tool caller can attach a new voice call to an arbitrary guessed session. Derive this from trusted tool context or keep it internal to the Google Meet gateway path.
  - confidence: 0.78
- **[P2] Build realtime context from the effective route:** `extensions/voice-call/src/runtime.ts:322-330`
  - body: This builds provider instructions once from the top-level Voice Call config during runtime startup. Per-number routes can override agentId, and realtime consults already resolve the route-effective config, so routed calls can speak with the global persona while consults use the routed agent. Build the instructions after route resolution or pass a per-call instruction resolver.
  - confidence: 0.9
- **[P2] Avoid forcing consults for every final transcript:** `extensions/voice-call/src/webhook/realtime-handler.ts:569-580`
  - body: The docs describe consultPolicy as model guidance and `always` as consulting before substantive answers, but every final user transcript schedules a forced full-agent consult when the provider did not call the tool. Greetings or short acknowledgements can get duplicate forced responses; gate the fallback to substantive/tool-needed turns or document an explicit forced mode.
  - confidence: 0.82
- **[P3] Honor action-form session keys:** `extensions/voice-call/index.ts:185-188`
  - body: The action-form schema now advertises sessionKey, but the initiate_call branch still calls initiateCall with an undefined session key. Callers using the documented action form lose the intended session pinning while only the legacy mode path preserves it.
  - confidence: 0.78
