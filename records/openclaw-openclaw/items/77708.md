## Summary

Keep open. This maintainer-labeled feature PR is not cleanup-closeable, and the latest diff still has blocking source-backed findings plus missing external real behavior proof.

## What This Changes

The PR adds Voice Call realtime agent-context and consult-policy config/docs/tests, plus Google realtime reconnect/audio handling and capability-provider loading changes.

## Best Possible Solution

Make realtime instruction construction route-aware, keep capability-provider activation generic and manifest-backed, then add redacted live Voice Call proof before merge.

## Reproduction Assessment

Yes, from source inspection: configure Voice Call with a route-specific number `agentId`, enable `realtime.agentContext`, and start a realtime inbound call to that route. The PR builds provider instructions from global config before `numberRouteKey` is available while consults resolve the route-effective config.

## Solution Assessment

No. The maintainable fix is to derive realtime instructions after route resolution or pass a per-call instruction resolver into `RealtimeCallHandler`, and to express provider activation through generic manifest/plugin contracts instead of bundled plugin IDs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Build realtime context from the effective call route:** `extensions/voice-call/src/runtime.ts:321-326`
  - body: This builds provider instructions once from the top-level Voice Call config at runtime startup. Per-number routing stores the matched route on the call record, and realtime consults already resolve the route-effective config, so a routed line with its own `agentId` can inject the global persona into the realtime provider while consults use the routed agent. Move instruction construction into the per-call path after the route is known.
  - confidence: 0.88
- **[P2] Keep capability loading generic:** `src/plugins/capability-provider-runtime.ts:348-359`
  - body: This adds core reads of `plugins.entries.voice-call` and `plugins.entries.google-meet` to discover realtime providers. That creates a bundled-plugin-only path in the generic provider resolver and leaves external plugins with the same capability outside the activation model; use manifest metadata or a generic plugin-owned contract instead.
  - confidence: 0.8
