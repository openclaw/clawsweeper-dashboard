## Summary

Keep open: this draft, maintainer-labeled external PR adds a useful Voice Call realtime feature, but it is not merge-ready because the PR lacks after-fix real behavior proof and the new agent context is built from the global config instead of the effective per-call route config.

## What This Changes

The PR adds opt-in Voice Call realtime agent context and consult-policy config, wires the generated instructions into realtime sessions, updates plugin metadata/docs/config baselines/changelog, and adds focused tests.

## Best Possible Solution

Keep the PR open and make realtime instruction construction route-aware before merge, then require a real Twilio/Gemini or Twilio/OpenAI voice-call proof artifact.

## Reproduction Assessment

Yes, from source inspection: configure `numbers` with a route-specific `agentId`, enable `realtime.agentContext`, and start an inbound realtime call. The PR builds provider instructions once from the global config while the consult path uses the route-effective config.

## Solution Assessment

No. The narrow maintainable fix is to derive the realtime instructions after the call record route is known, or pass a route-aware instruction resolver into `RealtimeCallHandler`, then cover the mismatch with a regression test.

## Review Findings
