## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Build realtime context from the effective route:** `extensions/voice-call/src/runtime.ts:322-327`
  - body: This builds provider instructions once from the top-level Voice Call config during runtime startup. Per-number routing can override `agentId`, and realtime consults already resolve that route-effective config, so a routed line can inject the global persona into the realtime provider while consults use the routed agent. Build instructions after route resolution or pass a per-call instruction resolver into `RealtimeCallHandler`.
  - confidence: 0.9
- **[P2] Avoid forcing consults for every final transcript:** `extensions/voice-call/src/webhook/realtime-handler.ts:569-580`
  - body: The docs say `consultPolicy` controls model instructions and that `always` consults before substantive answers, but this schedules a forced full-agent consult for every non-empty final user transcript when the provider does not call the tool. A simple greeting can trigger a second forced consult response; gate this fallback to substantive/tool-needed turns or make the forced behavior an explicit documented mode.
  - confidence: 0.82
