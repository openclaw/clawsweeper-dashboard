## Summary

## What This Changes

## Best Possible Solution

Land the status fix once maintainers accept the bounded wait; if finished events must never wait on alert transport, move the status update to a separate post-finish persistence path.

## Reproduction Assessment

## Solution Assessment

Yes, this is a targeted fix for durable failure-alert status if maintainers accept the bounded completion wait; otherwise the safer shape is a post-finish status update path.
