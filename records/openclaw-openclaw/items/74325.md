## What This Changes

The issue reports deterministic gateway restart stalls on 2026.4.26 where `sidecars.channels` blocks startup for about 45 seconds and a second post-ready event-loop freeze causes local plugin WebSocket handshakes to hit the 30 second timeout.

## Best Possible Solution

Keep the issue open until maintainers capture a current-main packaged restart with startup trace, diagnostics timeline event-loop samples, per-channel startup timings, and a loopback plugin WebSocket probe; then move any measured synchronous work off the gateway hot path or keep readiness/client access gated until the event loop is actually responsive.
