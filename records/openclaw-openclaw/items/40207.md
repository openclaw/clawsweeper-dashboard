## Repair Work Prompt

Fix openclaw/openclaw#40207. Current macOS LaunchAgent generation uses `resolveGatewayLogPaths` to put `StandardOutPath` and `StandardErrorPath` under the OpenClaw state directory, which launchd can reject when `~/.openclaw` or `OPENCLAW_STATE_DIR` is symlinked or on an external volume. Make the macOS LaunchAgent stdout/stderr paths use a launchd-safe local user log directory such as `~/Library/Logs/openclaw`, preserving profile/log-prefix behavior where appropriate, and make install, restart plist rewrite, status/diagnostics/runtime hints, docs, and tests agree. Add regression coverage proving symlinked/external state dirs are not used for LaunchAgent `StandardOutPath`/`StandardErrorPath`. Do not attempt to solve the separate launchd hang in #46153 or external-home bootstrap failure in #60398. Add a user-facing `CHANGELOG.md` fix entry.
