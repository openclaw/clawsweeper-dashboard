## Repair Work Prompt

Fix the macOS Gateway LaunchAgent stdout/stderr path bug described in https://github.com/openclaw/openclaw/issues/40207. Current main writes `StandardOutPath` and `StandardErrorPath` through `resolveGatewayLogPaths`, which points at `$OPENCLAW_STATE_DIR/logs`; launchd can reject those paths when the state dir is symlinked or on an external volume. Implement a narrow macOS LaunchAgent-safe default for stdout/stderr, update restart fallback plist rewrite, diagnostics/status/log hints, tests, docs, and CHANGELOG.md. Preserve Linux systemd, Windows schtasks, runtime `logging.file`, and `OPENCLAW_STATE_DIR` semantics unless directly needed for the LaunchAgent log-path fix. Do not add a config option, broad state-dir migration, or unrelated launchd restart rewrite. Keep #46153 and #60398 separate. Add/update regression coverage proving generated macOS plist stdout/stderr no longer use a symlink-prone state-dir path while restart-attempt logging remains intentional or is updated consistently. Before handoff, run targeted formatting/tests and `pnpm check:changed` in Testbox; include manual macOS smoke instructions for a symlinked/external state dir.
