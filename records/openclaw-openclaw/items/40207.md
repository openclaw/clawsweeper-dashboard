## Summary

Keep open. Current main and the latest release still derive the macOS LaunchAgent stdout path from the OpenClaw state directory, so the independently confirmed symlinked/external-volume `EX_CONFIG` failure remains actionable.

## What This Changes

The issue requests that macOS LaunchAgent log paths avoid symlinked or external `~/.openclaw/logs` locations so launchd does not fail with `EX_CONFIG`.

## Best Possible Solution

Generate macOS LaunchAgent stdout logs from a launchd-safe user log directory outside the OpenClaw state directory, then align status output, diagnostics, docs, and focused tests with that contract.
