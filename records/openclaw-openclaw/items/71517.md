## Summary

Keep open: current main still lacks the shared multi-encoding filename decoder and Feishu inbound filename normalization, but this PR needs real behavior proof plus SDK-surface and rebase cleanup before it is mergeable.

## What This Changes

The PR adds shared media filename decoding helpers, routes remote media `Content-Disposition` parsing through them, normalizes Feishu inbound filenames, and adds focused media/Feishu tests.

## Best Possible Solution

Land a rebased focused version that keeps the generic decoder in `src/media`, exposes any plugin-needed helper through the narrow documented SDK/runtime surface maintainers choose, coordinates the Feishu facade and outbound filename PRs, and leaves the broader channel migration tracked in https://github.com/openclaw/openclaw/issues/48788.
