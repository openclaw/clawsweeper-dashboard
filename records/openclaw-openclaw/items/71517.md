## Summary

Keep open. Current main still lacks the shared filename decoder and Feishu inbound filename normalization this PR targets, but the PR is not ready to merge because its public SDK surface is not aligned with the repo contract and it still touches the removed Feishu-specific SDK facade while separate compatibility work now owns that surface.

## What This Changes

The PR adds shared media filename decoding helpers, routes remote media Content-Disposition parsing through them, and normalizes Feishu inbound media filenames with focused tests.

## Best Possible Solution

Land a rebased version that keeps the generic decoder in `src/media`, routes `fetchRemoteMedia` and Feishu inbound saves through it, updates SDK docs/API baseline and the changelog for any public/user-facing contract change, drops or explicitly coordinates the stale Feishu facade hunk with #74223, and leaves #48788 open for remaining channel migrations.

Yes. ## Solution Assessment

Partly. A shared media decoder plus Feishu inbound normalization is the narrow maintainable direction for this scoped fix, but this PR is not the best mergeable form until it is rebased and the public SDK/changelog/facade contract is aligned with current main.
