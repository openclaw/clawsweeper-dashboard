## Summary

Keep open: current `main` still has the reported `config.patch` gap, and https://github.com/openclaw/openclaw/pull/98281 is an open closing PR rather than a merged fix.

## What This Changes

The issue reports that gateway `config.patch` rejects unrelated patches when bundled provider runtime defaults inject an empty `baseUrl` for built-in providers.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/98281, or replace it with an equivalent focused fix that makes `config.patch` share the existing bundled-provider default stripping invariant.

## Reproduction Assessment

Yes at source level: current `main` materializes missing built-in provider `baseUrl` as `""`, and `config.patch` validates the restored runtime-shaped merge without stripping that value. I did not run a live RPC repro in this read-only pass.

## Solution Assessment

Yes: applying the existing bundled-provider default stripping to `config.patch` is the narrowest maintainable fix, while loosening `baseUrl` validation would be broader and less consistent with the current config contract.
