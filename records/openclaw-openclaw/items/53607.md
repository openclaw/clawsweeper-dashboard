## Summary

Keep open: current main still lacks Discord component transcript mirroring, and this PR is useful but still has blocking regressions and transcript trust/size concerns before merge.

## What This Changes

The PR adds Discord component transcript summarization, returns it as outbound result metadata, changes shared delivery mirroring to prefer that metadata, and adds Discord/outbound regression tests.

## Best Possible Solution

Land a refreshed implementation that mirrors Discord component UI text through one idempotent write path, gates enriched media merging to explicit adapter summaries, dedupes mixed batches, and bounds or labels UI-derived transcript text.

## Reproduction Assessment

Yes. Source inspection on current main shows component sends register interaction entries and return ids, while shared mirroring only uses params.mirror.text/mediaUrls and never sees component labels.

## Solution Assessment

No. The direction is right, but this diff changes ordinary media mirror semantics and promotes unbounded adapter text into assistant transcript history; a narrower enriched-text signal with bounds is safer.

## Security Review

Status: needs_attention

Summary: The patch promotes adapter result metadata into persisted assistant transcript text without an explicit size or trust boundary.

Concerns:

- **[medium] Bound adapter transcript metadata before persistence:** `src/infra/outbound/deliver.ts:808`
  - body: `deliverOutboundPayloadsCore` would join `meta.transcriptText` from delivery results and persist it as assistant transcript text without a character, line, or result cap. Channel adapters/plugins can populate result metadata, so this should be bounded before it can inflate session files or downstream model context.
  - confidence: 0.78
- **[medium] Treat UI-derived transcript text as lower-trust context:** `src/infra/outbound/deliver.ts:816`
  - body: The PR stores component labels/options and adapter-provided transcript metadata as an assistant message. That may be acceptable for context recovery, but the persisted text should be explicitly labeled or constrained so UI-derived content is not silently promoted to high-authority assistant instructions.
  - confidence: 0.72
