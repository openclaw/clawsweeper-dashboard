## Summary

Keep open: this targets a real Discord transcript bug and current main still lacks enriched component mirroring, but the PR is not merge-ready because it lacks real behavior proof, regresses shared media transcript semantics, and needs a trust/size boundary for adapter-provided transcript text.

## What This Changes

The PR adds Discord component transcript summarization, returns it as delivery-result metadata, and changes shared outbound mirroring to prefer that metadata.

## Best Possible Solution

Land a rebased Discord component transcript mirror that carries bounded, labeled component text through one idempotent mirror path and preserves existing media-only transcript semantics for non-enriched sends.

## Reproduction Assessment

Yes, source-reproducible: current main sends and registers Discord component messages, but the shared mirror path only uses plain mirror text/media and ignores any component-label summary. The linked issue provides concrete button-click context-loss steps, but I did not run a live Discord repro in this read-only review.

## Solution Assessment

No. The PR's direction is useful, but the current patch broadens shared media transcript behavior and persists generic adapter metadata without trust and size boundaries.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Gate media-text transcript merge to enriched sends:** `src/config/sessions/transcript-mirror.ts:44-49`
  - body: The new branch in `resolveMirroredTranscriptText` runs for every caller with both `text` and `mediaUrls`, not only adapter-enriched Discord components. Current delivery coverage expects captioned media to mirror only `report.pdf`, so this would change ordinary media transcripts across channels to `caption\nreport.pdf` and duplicate assistant text; keep existing media-only behavior unless the caller explicitly passes enriched transcript text.
  - confidence: 0.92

## Security Review

Status: needs_attention

Add hard character, line, or item bounds before persistence to avoid unbounded transcript growth and downstream context cost.
  - confidence: 0.78
