## What This Changes

The PR adds Discord component transcript summaries, attaches them as outbound result metadata, changes shared outbound mirroring to prefer that metadata, and adds Discord/outbound regression tests.

## Best Possible Solution

Land a rebased implementation that mirrors bounded, labeled Discord component text through an explicit enriched-transcript path, preserves ordinary media-only mirror semantics, and proves one idempotent real Discord component-click flow.

## Reproduction Assessment

Yes, source-based: current main registers and routes Discord component interactions, but the component send result carries no enriched transcript summary and the mirror path only writes params.mirror.text/mediaUrls. I did not run a live Discord click in this read-only review.

## Solution Assessment

No, not as written. The bug fix direction is valid, but the maintainable solution should use an explicit bounded enriched-transcript contract instead of changing all text-plus-media mirroring and trusting a generic metadata bag.
