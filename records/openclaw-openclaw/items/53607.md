## Summary

Keep open: this PR targets a real Discord transcript bug, but the current patch still changes shared captioned-media mirroring behavior, lacks real Discord after-fix proof, and leaves the adapter transcript trust/size boundary unresolved.

## What This Changes

The PR adds Discord component transcript summarization, returns it via delivery-result metadata, teaches the shared outbound mirror path to prefer that metadata, and adds Discord/outbound regression tests.

## Best Possible Solution

Land a narrow Discord component transcript mirror through one idempotent path, gated to explicit enriched adapter summaries, with bounds/trust handling and no change to ordinary media transcript semantics.

## Reproduction Assessment

Yes, source-reproducible. Current main sends and registers Discord component messages but returns no enriched transcript metadata, and the shared mirror path ignores delivery metadata while the linked bug gives concrete live button-click context-loss steps.

## Solution Assessment

No. The direction is useful, but this patch broadens shared captioned-media transcript behavior and promotes generic adapter metadata into assistant transcript memory without an explicit enriched-text trust and size boundary.
