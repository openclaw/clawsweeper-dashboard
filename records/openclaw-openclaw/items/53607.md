## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Use an explicit enriched transcript contract:** `src/infra/outbound/deliver.ts:807-808`
  - body: This reads `meta.transcriptText` from every outbound adapter result and persists it as assistant transcript text, but `OutboundDeliveryResult.meta` is exported as generic channel-specific plugin metadata. That creates an implicit public contract and lets unrelated adapters override session transcript contents without a bounded/trusted signal; use an explicit enriched transcript field or caller-side path with limits instead.
  - confidence: 0.86
- **[P2] Gate text-plus-media mirroring to enriched sends:** `src/config/sessions/transcript-mirror.ts:43-50`
  - body: The new text+media branch runs for every caller with both text and media URLs, not only Discord component summaries. Current main expects a captioned media send to mirror `report.pdf`; this patch would persist `caption\nreport.pdf` across channels and duplicate assistant content in normal media transcripts.
  - confidence: 0.92
- **[P2] Deduplicate mixed enriched mirror batches:** `src/infra/outbound/deliver.ts:813-815`
  - body: For mixed result batches, this prepends the full `params.mirror.text` to adapter summaries even though the Discord summary already starts with the component text. A component payload plus a plain payload can record repeated base text in one assistant entry; merge per-result text or dedupe prefixes before writing.
  - confidence: 0.75
