## Summary

Keep open: current main still lacks enriched transcript mirroring for Discord component messages, but this PR has a blocking shared media transcript regression, missing real Discord proof, and unresolved transcript metadata trust/size concerns.

## What This Changes

The PR adds Discord component transcript summaries, propagates them through outbound result metadata into shared delivery mirroring, adjusts media transcript resolution, and adds Discord/outbound regression tests.

## Best Possible Solution

Land a rebased implementation that mirrors bounded, labeled Discord component text and interactive labels through one idempotent transcript path while preserving ordinary captioned-media mirror semantics.

## Reproduction Assessment

## Solution Assessment

No. The direction is useful, but the current patch changes shared captioned-media transcript semantics and persists generic adapter metadata as assistant-visible memory without size or trust boundaries.
