## Summary

Keep open. This targets a real source-reproducible Discord bug, but the current PR is not ready to merge because it lacks real Discord proof and introduces a shared captioned-media transcript regression plus unresolved transcript metadata trust and size concerns.

## What This Changes

The PR adds Discord component transcript summarization, returns that summary as delivery-result metadata, updates shared transcript mirroring to prefer it, and adds Discord/outbound regression tests.

## Best Possible Solution

Land a narrow Discord component transcript mirror that carries bounded, labeled component text through one idempotent path and preserves existing media-only mirror behavior for ordinary captioned media.

## Reproduction Assessment

Yes, source-reproducible: current main sends and registers Discord component messages but does not add enriched transcript metadata, and the shared mirror path ignores delivery-result metadata while the linked bug provides concrete button-click context-loss steps. I did not run a live Discord repro in this read-only review.

## Solution Assessment

No. The direction is useful, but this patch broadens shared media transcript behavior and promotes generic adapter metadata into assistant transcript memory without explicit enriched-text trust and size boundaries.
