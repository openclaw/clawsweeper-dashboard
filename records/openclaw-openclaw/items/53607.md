## Summary

Keep this PR open: it targets a real Discord Components v2 transcript bug, but the current patch still has a shared media-transcript regression, an unresolved transcript-metadata trust/size boundary, and no after-fix proof from a real Discord setup.

## What This Changes

The PR adds Discord component transcript summarization, returns it as delivery-result metadata, changes shared transcript mirroring to prefer that metadata, and adds Discord/outbound regression tests.

## Best Possible Solution

Land a narrow Discord component transcript mirror through one idempotent path in the current outbound payload structure, gated to explicit enriched adapter summaries, with bounds/trust handling and preserved media mirror semantics.

## Reproduction Assessment

Yes, source-reproducible: current main sends and registers Discord component messages but returns no enriched transcript metadata, and the shared mirror path ignores delivery metadata while the linked bug gives concrete button-click context-loss steps.

## Solution Assessment

No: the direction is useful, but this patch broadens shared captioned-media transcript behavior and promotes generic adapter metadata into assistant transcript memory without an explicit enriched-text trust and size boundary.
