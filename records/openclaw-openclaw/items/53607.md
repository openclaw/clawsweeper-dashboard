## Summary

Keep open. The PR targets a real Discord Components v2 transcript gap, but the current patch introduces a shared captioned-media transcript regression, still needs an explicit trust/size boundary for adapter-provided transcript metadata, and lacks after-fix proof from a real Discord setup.

## What This Changes

The PR adds Discord component transcript text extraction, returns it through delivery-result metadata, changes shared outbound transcript mirroring to prefer that metadata, and adds focused Discord/outbound tests.

## Best Possible Solution

Land a narrow Discord component transcript mirror that uses one idempotent write path, preserves ordinary captioned-media mirror semantics, and bounds or labels UI-derived transcript text before it becomes assistant session memory.

## Reproduction Assessment

Yes, source-reproducible. Current main registers and routes Discord component interactions but does not add component prompt or label context to the mirrored assistant transcript, and the linked report provides concrete live steps for the resulting orphaned-click behavior.

## Solution Assessment

No. The proposed direction is useful, but this patch broadens shared captioned-media transcript behavior and promotes generic adapter metadata into assistant memory without an explicit enriched-text trust and size boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Gate media transcript merging to enriched sends:** `src/config/sessions/transcript-mirror.ts:49`
  - body: The new text+media branch runs for every caller of resolveMirroredTranscriptText, not only Discord component results. Current delivery mirror coverage expects a normal captioned media send to persist the filename (`report.pdf`) rather than `caption\nreport.pdf`, so this changes ordinary media transcripts across channels and duplicates assistant text. Keep the old media-only behavior unless the caller explicitly passes enriched adapter transcript text.
  - confidence: 0.92
