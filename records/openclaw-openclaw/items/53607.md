## Summary

Keep open: this PR targets a real source-reproducible Discord Components v2 transcript bug, but it should not merge as-is because it broadens captioned-media transcript behavior, lacks real Discord after-fix proof, and needs explicit trust/size bounds around adapter-provided transcript text.

## What This Changes

The PR adds Discord component transcript summarization, returns it as delivery-result metadata, changes shared outbound transcript mirroring to prefer that metadata, and adds Discord/outbound regression coverage.

## Best Possible Solution

Land a narrow Discord component transcript mirror through one idempotent path, with explicit enriched-metadata bounds/trust handling and no change to ordinary captioned-media mirror semantics.

## Reproduction Assessment

Yes, source-reproducible. Current main sends and registers Discord component messages while returning plain delivery results, and the shared mirror path ignores delivery metadata; the linked bug report gives concrete live button-click context-loss steps.

## Solution Assessment

No. The direction is useful, but this patch changes shared captioned-media behavior and promotes generic adapter metadata into assistant transcript memory without an explicit enriched-text trust and size boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Gate media transcript merging to enriched sends:** `src/config/sessions/transcript-mirror.ts:44`

## Security Review

Status: needs_attention

Summary: The diff adds a generic metadata-to-assistant-transcript path that needs explicit trust and size constraints before merge.

  - confidence: 0.78
