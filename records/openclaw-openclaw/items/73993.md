## Summary

Keep open. Current main still lacks the accepted/completed Telegram polling watermark split, while this PR is a useful but currently dirty, non-maintainer-modifiable bot repair that should be replaced or rebased onto current main before landing.

## What This Changes

The PR adds split Telegram accepted/completed polling offset persistence, fetched-offset durability before `getUpdates` results return to grammY, recovery logging, API exports, regression tests, and changelog text.

## Best Possible Solution

Land a clean current-main Telegram plugin repair that records fetched/accepted offsets before grammY can confirm them, stores completed handler offsets separately for restart replay/logging, preserves token-fingerprint rotation behavior, and includes focused restart regression coverage.

## Reproduction Assessment

Yes by source inspection. `@grammyjs/runner` advances its polling offset after a successful batch while current main persists only one local watermark after handler completion, leaving a restart window for in-flight updates to be silently lost.

## Solution Assessment

Yes in direction, but not as this branch verbatim. The maintainable path is a current-main replacement or rebase that keeps the split-watermark design, preserves token-fingerprint offset invalidation, and adds the completed-watermark monotonic guard.
