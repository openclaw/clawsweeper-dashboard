## Summary

This PR should stay open: the Telegram silent-drop race is still source-reproducible on current main and the branch heads in the right direction, but it needs a clean replacement or repair that integrates current token-fingerprint offset state and carries the completed-watermark guard from review feedback.

## What This Changes

The PR adds separate fetched/accepted and completed Telegram polling watermarks, recovery logging, regression coverage, API exports, and a changelog entry for the in-flight update loss race.

## Best Possible Solution

Land a clean Telegram plugin-local replacement that preserves current token-fingerprint offset invalidation, adds fetched/accepted versus completed watermarks, logs recovery state, and keeps focused restart regression coverage.

## Reproduction Assessment

Yes by source inspection: grammY can advance its polling offset after a successful batch while current main persists only a single local watermark tied to handler completion. A live Telegram crash/restart probe was not run in this read-only pass.

## Solution Assessment

Yes in direction, but not as a direct application of this stale head. The safest solution is a fresh patch that keeps the PR's durability barrier, adds the completed-watermark monotonic guard, and integrates the already-merged token-fingerprint offset state.
