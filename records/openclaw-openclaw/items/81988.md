## Summary

Keep open. Current main and the latest shipped release still route `opencode-go` through passthrough Gemini replay hooks that omit `dropReasoningFromHistory`, while the documented OpenAI-compatible Chat Completions contract strips historical reasoning before replay.

## What This Changes

The issue reports multi-turn `opencode-go/kimi-k2.6` tool runs replaying a top-level `reasoning` field to an OpenAI-compatible Chat Completions proxy, causing 400 extra-input validation errors.

## Best Possible Solution

Update the passthrough Gemini replay policy or opencode-go replay hook so OpenAI-compatible Chat Completions proxy routes drop historical assistant reasoning while preserving Gemini signature sanitation.

## Reproduction Assessment

No high-confidence live reproduction was run because the path needs opencode-go credentials and a tool-using model turn. Source inspection gives a clear current-main reproduction path through opencode-go's plugin-owned replay policy and the missing `dropReasoningFromHistory` flag.

## Solution Assessment

Yes, the suggested direction is the narrowest maintainable fix: make the provider-owned passthrough replay policy account for `modelApi: "openai-completions"` instead of adding a new config option or broad transcript rewrite.
