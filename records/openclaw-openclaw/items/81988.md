## Summary

Close as implemented on current main: the rejected `reasoning` replay field is now stripped in the OpenAI Chat Completions transport for Kimi-style OpenAI-compatible replay, with regression coverage, though the fix is not in a stable release yet.

## What This Changes

The issue reports that opencode-go passthrough OpenAI-compatible completions models replay assistant `reasoning` into multi-turn history, causing upstream providers to reject the second turn.

## Best Possible Solution

Keep the current transport-level replay sanitizer, since it removes disallowed Chat Completions reasoning fields at the wire boundary while preserving `reasoning_content` for providers that require it.

## Reproduction Assessment

Yes at source level, but not with a fresh live provider call. The reported release path and current opencode-go model metadata identify an OpenAI-compatible Kimi reasoning replay path, and current main now sanitizes the rejected field before outbound Chat Completions requests.

## Solution Assessment

Yes. The current transport-boundary fix is a maintainable solution because it strips invalid replay fields even when a provider keeps a minimal passthrough replay policy, while retaining the Kimi `reasoning_content` contract.
