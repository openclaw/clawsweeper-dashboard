## Summary

Keep open: the latest PR revision addresses the original whitespace/quote serialization problem, but it still accepts empty `args` values that the acpx string parser drops and the changelog entry uses a forbidden bot credit.

## What This Changes

The PR adds optional `plugins.entries.acpx.config.agents.<id>.args` schema/manifest support, flattens command plus shell-quoted args for the acpx agent registry, adds config tests, and adds a changelog entry.

## Best Possible Solution

Land the feature after making `agents.<id>.args` either lossless or explicitly non-empty, adding regression coverage and docs for that contract, and keeping the changelog credit policy-compliant.

## Reproduction Assessment

Yes for the PR findings: a source-level parser check using the PR quoting logic and acpx 0.6.1 parser shows empty args are dropped, and the changelog policy violation is directly visible in the diff. I did not run the PR test suite in this read-only checkout.

## Solution Assessment

No. Shell-quoting is the right direction for whitespace and quotes, but the implementation should not accept values the downstream string parser cannot represent, and the docs/changelog should match OpenClaw policy before merge.
