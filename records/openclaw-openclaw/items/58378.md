If the branch is rebased around this helper, users with an explicit configured pin can connect under a different trust source; thread `GatewayRemoteConfig.resolveTLSFingerprint` through this path or keep `MacNodeModeCoordinator.tlsParams` as the owner.
  - confidence: 0.83
- **[P1] Keep first-use pinning behind system trust or config:** `apps/macos/Sources/OpenClaw/GatewayTLSPinningSupport.swift:155-157`
  - body: The probe returns a fingerprint from any server-trust challenge and `confirmSelection` saves it after the prompt, bypassing current main's `GatewayTLSFirstUsePolicy` and docs that only first-use pin after normal system trust unless a fingerprint is explicitly configured. A spoofed discovered endpoint could become the stored pin if the user accepts the prompt, so preserve system-trust validation by default or gate untrusted fingerprint import behind an explicit maintainer-approved path.
