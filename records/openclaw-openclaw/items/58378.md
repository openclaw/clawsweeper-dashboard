- **[P1] Preserve the current TLS trust gate before saving pins:** `apps/macos/Sources/OpenClaw/GatewayTLSPinningSupport.swift:155-157`

Current main documents and enforces first-use pinning only after normal system trust succeeds, with explicit `gateway.remote.tlsFingerprint` or SSH for private/self-signed certs; this branch would let a first-connection MITM become the stored pin unless the user performs out-of-band verification perfectly. Keep the current system-trust gate by default, or get explicit maintainer approval and docs/proof for changing the trust model.
  - confidence: 0.84
- **[P1] Preserve configured TLS fingerprints when building sessions:** `apps/macos/Sources/OpenClaw/NodeMode/MacNodeModeCoordinator.swift:174-178`
  - body: This branch's macOS node session builder now delegates to a helper that only resolves stored Keychain pins and TOFU state. Current main prefers `gateway.remote.tlsFingerprint` for remote WSS sessions, so existing users with configured pins for private/self-signed gateways could lose that source of trust during the rebase and fall into TOFU or connection failure. Thread the current-main configured-fingerprint path through the new helper or keep the current `MacNodeModeCoordinator.tlsParams` shape.
