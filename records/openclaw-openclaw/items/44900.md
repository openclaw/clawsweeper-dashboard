## Summary

Keep this PR open, but the current patch should not merge as written because it changes the retired `src/telegram` tree while current `main` runs Telegram from `extensions/telegram/src`. The underlying bug still has a narrow active-code repair path.

## What This Changes

The PR branch adds a route-level guard intended to preserve ACP-bound Telegram DM-topic session keys by skipping an extra DM-thread suffix in inbound, native-command, and session-resolution paths.

## Best Possible Solution

Port the bound-route authority signal into `extensions/telegram/src`, consume it in inbound routing, native command target resolution, and session-state lookup, and keep unbound DM-topic isolation governed by the current opt-in policy.

## Reproduction Assessment

Yes. The source-level path is clear: current `main` can rewrite a Telegram DM-topic route to a bound ACP session, then active inbound/native code can still append a DM-topic suffix when DM-topic session isolation is enabled.

## Solution Assessment

No, not as written. The guard is the right shape, but it must be applied to the active Telegram plugin files and aligned with `shouldUseTelegramDmThreadSession`.
