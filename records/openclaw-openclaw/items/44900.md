## Summary

Keep open. The Telegram ACP DM-topic bug is still a plausible active-runtime issue, but this PR edits retired `src/telegram` paths instead of the current `extensions/telegram` plugin tree, so it would not fix current `main`.

## What This Changes

The PR adds a route-level `skipDmThreadSuffix` guard in legacy Telegram routing and uses it in inbound and native-command paths to preserve ACP-bound DM-topic session keys, with regression tests.

## Best Possible Solution

Port the bound-route authority signal into `extensions/telegram/src`, consume it in inbound routing, native command target resolution, and session-state lookup, and preserve unbound DM-topic isolation under the current opt-in DM threading policy.

## Reproduction Assessment

Yes. The source-level path is clear: current `main` can rewrite a Telegram DM-topic route to a bound ACP session, then active inbound/native code can still append a DM-topic suffix when DM-topic session isolation is enabled.

## Solution Assessment

No, not as written. The guard is the right general shape, but it must be applied to the active Telegram plugin files and aligned with the newer `shouldUseTelegramDmThreadSession` policy.
