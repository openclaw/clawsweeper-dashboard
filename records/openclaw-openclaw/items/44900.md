## Summary

Keep open: the reported Telegram ACP DM-topic session-key bug is valid on current main, but this PR edits the retired `src/telegram` paths instead of the active Telegram plugin implementation under `extensions/telegram/src`.

## What This Changes

The branch adds a `skipDmThreadSuffix` flag and Telegram regression tests under the old `src/telegram` tree to avoid adding DM-topic suffixes to already-bound ACP session keys.

## Best Possible Solution

Port the bound-route authority signal into the active Telegram plugin routing, inbound message, native command, and session-state lookup paths, with regression coverage and a user-facing changelog entry.

## Reproduction Assessment

Yes, from source inspection. Current main can rewrite a Telegram DM-topic route to a bound ACP session key, then active inbound/native/session-state code can append a DM-topic suffix when DM-topic session isolation is enabled.

## Solution Assessment

No, not as written. The guard is the right shape, but it must be applied to the active `extensions/telegram/src` files and cover the same suffixing sites without weakening unbound DM-topic isolation.
