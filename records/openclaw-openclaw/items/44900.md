## Summary

Keep open. The reported Telegram ACP DM-topic bug is still credible on current main, but this PR is not the right patch because it changes retired `src/telegram` files while the active Telegram plugin runtime lives under `extensions/telegram/src`.

## What This Changes

The PR changes legacy Telegram routing, inbound-message, native-command, and test files under `src/telegram` to add a `skipDmThreadSuffix` guard for ACP-bound DM-topic session keys.

## Best Possible Solution

Port the route-level bound-session authority signal into `extensions/telegram/src`, consume it in inbound message routing, native command target resolution, and session-state lookup, and add regression tests proving bound DM topics keep the exact ACP key while unbound DM topics still get `:thread:<chatId>:<topicId>` isolation.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: when a runtime conversation binding resolves a DM topic to an ACP target key, the active Telegram paths then append a DM-topic thread suffix after route resolution.

## Solution Assessment

No, not as written. The proposed guard is the right shape, but it must be applied to the active Telegram plugin tree instead of the retired `src/telegram` paths.
