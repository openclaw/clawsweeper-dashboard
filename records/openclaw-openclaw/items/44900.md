## Summary

Keep open: the reported Telegram ACP DM-topic session-key bug is valid from current source inspection, but this PR targets the retired `src/telegram` tree and would not change the active Telegram plugin runtime on current `main`.

## What This Changes

The branch changes the old `src/telegram` Telegram channel files to carry a `skipDmThreadSuffix` route flag through inbound, native-command, and session-state suffix handling with regression tests.

## Best Possible Solution

Port the bound-route authority signal into `extensions/telegram/src/conversation-route.ts`, consume it in inbound routing, native command target resolution, and session-state lookup, and keep unbound DM-topic isolation governed by the existing opt-in policy.

## Reproduction Assessment

Yes, from source inspection. Current `main` can rewrite a Telegram DM-topic route to a bound ACP session key, then active inbound/native/session-state code can append a DM-topic suffix when DM-topic sessions are enabled.

## Solution Assessment

No, not as written. The guard is the right shape, but it must be applied to the active `extensions/telegram/src` files and cover the same suffixing sites without weakening unbound DM-topic isolation.
