## Summary

Keep open for maintainer review: the patch addresses a real docs gap against the current create/verify-only backup CLI, but it would publish a manual restore path for archives that can contain credentials, auth profiles, sessions, state, and workspaces, so a maintainer should explicitly accept that security boundary before merge.

## What This Changes

Adds a `Restore from an archive` section to `docs/cli/backup.md` covering trusted archives, structural verification, private extraction, manifest inspection, selective copy-back, restart, and `openclaw doctor`.

## Best Possible Solution

Land the docs only after an owner accepts the interim trusted-archive/manual-copy restore posture, or narrow the page to state that no restore command exists and track a first-class restore command separately.

## Reproduction Assessment

Yes. Current main is source-reproducible: `docs/cli/backup.md` documents create/verify only, and `src/cli/program/register.backup.ts` registers create and verify without a restore command.

## Solution Assessment

Unclear as a merge decision. The patch is a narrow docs-only way to describe current CLI behavior, but maintainers should choose whether public manual restore guidance is acceptable before a first-class restore command exists.
