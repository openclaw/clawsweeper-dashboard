## What This Changes

The PR changes trusted-proxy Control UI WebSocket auth so fresh device identities must complete pairing before requested operator scopes are honored, clears unbound scopes for device-less proxy sessions, and updates focused gateway auth tests.

## Best Possible Solution

Land a maintainer-reviewed version that bounds trusted-proxy scoped access to approved pairings, preserves intentional bypass paths, updates the trusted-proxy/Control UI docs, and includes redacted live gateway proof.

## Reproduction Assessment

Yes. Current main has a clear source-level path: `trustedProxyAuthOk` skips Control UI pairing, and the existing suite asserts an unpaired trusted-proxy device identity keeps operator scopes and can run an admin RPC; I did not run tests because this review is read-only.

## Solution Assessment

Unclear until maintainer review. The patch targets the right connect-policy and scope-clearing seams, but the docs and live proof need to catch up, and the token-issuance change should be explicitly accepted for this auth mode.
