## Summary

Keep open. The performance problem is real on current main, but this PR trades the per-task full registry relist for a stale sweep-start snapshot, which can make terminal ACP cleanup miss active work created later in the asynchronous maintenance pass; it also needs the required changelog entry.

## What This Changes

The branch changes task-registry maintenance to pass the initial sweep snapshot into terminal ACP session cleanup and adds a regression test counting task-record listing calls.

## Best Possible Solution

Land a version that keeps maintenance linear without weakening the current-state safety check, preferably by using a current indexed lookup for the affected child session or an explicitly maintained active-session view, with focused regression coverage and a changelog entry.

## Reproduction Assessment

Yes. A focused test can count listTaskRecords calls while seeding many terminal ACP tasks to show current main relists per task; a companion safety test should add an active same-child-session ACP task after the initial snapshot and assert cleanup does not close that session.

## Solution Assessment

No. Reusing the initial array fixes the O(n^2) call pattern, but the safer maintainable fix is to remove the full relist while keeping the terminal ACP close guard based on current task state.
