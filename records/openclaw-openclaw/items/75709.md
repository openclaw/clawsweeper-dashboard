## Summary

Keep open. The PR targets a real current-main performance problem, but the proposed one-line optimization changes terminal ACP cleanup from a fresh registry view to the sweep-start snapshot, which can miss active same-session work created during the async maintenance pass; it also lacks the required changelog entry for a user-facing fix.

## What This Changes

The PR changes task-registry maintenance to pass the initial task snapshot into terminal ACP session cleanup and adds a regression test that counts task-record listing calls during maintenance.

## Best Possible Solution

Land a version that removes the full per-task registry relist while preserving a current same-child-session guard for terminal ACP cleanup, preferably via an indexed current-session lookup or an explicitly maintained active-session view, and add focused regression coverage plus the changelog entry.

## Reproduction Assessment

Yes. A focused reproduction can inject a `listTaskRecords` counter, seed many terminal ACP tasks, and run `runTaskRegistryMaintenance()` to show current main relists once per task; a companion regression should create an active same-child-session task after the initial snapshot and verify cleanup does not close that session.

## Solution Assessment

No, not as written. Reusing the sweep-start array fixes the quadratic call pattern, but the safer maintainable solution is to avoid the full relist with a current indexed lookup rather than making the active-task guard stale.
