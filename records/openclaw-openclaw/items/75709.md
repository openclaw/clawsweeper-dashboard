## Summary

Keep open. Current main still relists the full task registry inside each maintenance-loop terminal ACP cleanup, but this PR’s snapshot reuse makes the ACP session close guard stale and still needs a changelog entry before it is safe to merge.

## What This Changes

The PR changes task registry maintenance to pass the initial sweep task snapshot into terminal ACP session cleanup and adds a regression test that counts maintenance `listTaskRecords()` calls.

## Best Possible Solution

Land a version that avoids the full per-task relist with a current session-specific lookup or maintained active-session view, preserving ACP close safety, focused regression coverage, and a single-line changelog entry.

## Reproduction Assessment

Yes. A focused check can seed many terminal ACP tasks, count `listTaskRecords()` calls during `runTaskRegistryMaintenance()`, and add a companion case where an active same-child ACP task appears after the initial snapshot but before cleanup.

## Solution Assessment

No, not as written. Reusing the initial snapshot fixes the call count but is not the narrowest safe fix because the ACP close guard needs current task state without cloning and sorting the whole registry.
