## Summary

Keep open: current main still has both reported behaviors. The compaction wrapper emits `before_compaction` for engine-owned compaction but only emits `after_compaction` when `result.compacted` is true, while the direct compaction path can legitimately return `ok: true, compacted: false`; the gateway validator also still formats additional-property errors with single quotes. The related closed reports at https://github.com/openclaw/openclaw/issues/78300 and https://github.com/openclaw/openclaw/issues/78478 are adjacent compaction fixes, not a fix for this remaining observer path.

## What This Changes

Request that no-op successful compaction still notify compaction observers and that gateway validation errors quote unexpected property names in a tool-caller-friendly way.

## Best Possible Solution

Land a focused bug fix that closes successful no-op compaction cycles for observers and makes additional-property validation messages unambiguous, with targeted regression coverage for both paths.

## Reproduction Assessment

Yes at source level, but not with a live voice setup in this read-only review. Current main has an ok/no-op compaction return path that bypasses the queued wrapper's `after_compaction` gate, and the validator formatter still emits the single-quoted unexpected-property string asserted by tests.

## Solution Assessment

Yes: the narrowest maintainable fix is to emit an honest after-compaction event for `ok && !compacted` without adding a new recovery-strategy config, and to change the additional-property delimiter plus update exact-string tests.
