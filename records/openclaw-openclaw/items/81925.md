## Summary

Keep open: current main still has both source-proven behaviors. Engine-owned compaction emits `before_compaction` but only emits `after_compaction` for `ok && compacted`, while successful no-op compaction remains a real return path; the gateway validator still formats additional-property names with single quotes that are fed into cron/tool errors.

## What This Changes

The issue requests two focused fixes: emit `after_compaction` for successful no-op compaction and make additional-property validation errors use an unambiguous delimiter.

## Best Possible Solution

Land a focused bug fix that closes successful no-op compaction lifecycles for observers and makes additional-property validation errors unambiguous for tool-calling models.

## Solution Assessment
