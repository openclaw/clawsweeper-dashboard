## Summary

Keep open: current main still contains both source-level behaviors, so this is not implemented, stale, duplicate, or outside the repository. The report is narrow enough for a focused fix PR: close the compaction hook lifecycle for successful no-op compactions and make additional-property validation messages less copy-prone for tool-calling models.

## What This Changes

The issue reports that queued compaction skips `after_compaction` for successful no-op compactions and that validation errors quote unexpected property names in a way tool-calling models can copy back incorrectly.

## Best Possible Solution

Land a focused regression-tested bug fix that emits an honest after-compaction notification for successful no-op compactions and changes additional-property delimiters without adding a new compaction recovery strategy setting.

## Reproduction Assessment

Yes, at source level: current main has a successful no-op compaction return path that bypasses the queued `after_compaction` gate, and the formatter still emits the single-quoted additional-property text. I did not establish the full live voice-channel reproduction in this read-only pass.

## Solution Assessment

Yes: the narrow maintainable path is to close the hook lifecycle for `ok && !compacted` results and update the validation error string plus exact-string tests. The optional recovery-strategy config should be left out unless maintainers explicitly choose that product direction.
