## Summary

Keep open: current main still has the two reported source behaviors. Successful no-op compaction can return before the lifecycle is closed for hook observers, and gateway additional-property validation still emits single-quoted property names that are fed back through cron/tool errors.

## What This Changes

The issue asks OpenClaw to emit an honest `after_compaction` notification for successful no-op compactions and to make additional-property validation messages unambiguous for LLM tool callers.

## Best Possible Solution

Land a focused bug fix that closes successful no-op compaction cycles for observers on both relevant paths and changes additional-property error delimiters with targeted regression coverage.

## Reproduction Assessment

Yes at source level: current main has a successful no-op compaction return that can occur after `before_compaction` but before any after notification, and the validator still emits the single-quoted additional-property string. I did not run the full live voice/llama-server setup in this read-only review.

## Solution Assessment

Yes: the narrowest maintainable fix is to emit an honest after-compaction event for `ok && !compacted` successful skips and to change the additional-property delimiter plus exact-string tests. The optional recovery-strategy config should stay out of this bug fix unless a maintainer explicitly chooses a larger product change.
