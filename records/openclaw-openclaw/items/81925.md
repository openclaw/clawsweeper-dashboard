## Summary

Keep open: current main still shows both reported source-level behaviors, so this is not implemented, duplicate, stale, or outside the repository. The queued compaction wrapper can emit `before_compaction` and then skip the matching after hook on a successful no-op result, and the gateway validator still returns additional-property names in single quotes that are surfaced through tool errors.

## What This Changes

The issue requests two focused fixes: close successful no-op compaction cycles for observers and make additional-property validation errors unambiguous for tool-calling models.

## Best Possible Solution

Land a focused bug fix with regression coverage that emits an honest after-compaction event for successful no-op compactions and changes the additional-property delimiter without adding a new recovery-strategy config.

## Reproduction Assessment

Yes at source level: current main has a successful no-op compaction return path that bypasses the queued wrapper's `after_compaction` gate, and the validator formatter still emits the single-quoted additional-property text. I did not establish a live voice-channel reproduction in this read-only pass.

## Solution Assessment

Yes: the narrowest maintainable solution is to close the hook lifecycle for `ok && !compacted` results and update the validation error string plus exact-string tests. The optional recovery-strategy config would be a product choice and should not be part of the first bug fix.
