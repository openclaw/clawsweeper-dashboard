## Summary

Keep open. Current main still has the OpenRouter gap described by the reporter: the OpenRouter `openai-completions` path adds OpenRouter reasoning metadata but does not strip Anthropic trailing assistant prefills, while the direct Anthropic and Cloudflare Anthropic paths already use the shared sanitizer. The linked fixing PR was closed unmerged, so there is no merged implementation to close this against.

## What This Changes

The issue requests that OpenRouter-routed Anthropic models using the OpenAI-compatible adapter strip trailing assistant prefill turns when reasoning/thinking is enabled, matching the direct Anthropic behavior.

## Best Possible Solution

Add an OpenRouter-owned stream/payload fix for Anthropic model IDs on the `openai-completions` route that treats OpenRouter `reasoning`/`reasoning_effort` as thinking-enabled, strips only trailing non-tool assistant prefills, and keeps non-Anthropic OpenRouter behavior unchanged.

## Reproduction Assessment

Yes. The issue provides concrete OpenRouter config plus repeated 400 logs, and current-main source inspection shows the OpenRouter path still lacks the sanitizer used by direct Anthropic and Cloudflare Anthropic routes.

## Solution Assessment

Yes. A provider-local OpenRouter wrapper or small shared-helper extension is the narrowest maintainable fix; changing every `openai-completions` provider would risk unrelated provider behavior.
