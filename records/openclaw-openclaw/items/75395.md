## Summary

Keep open. Current main still does not apply the shared Anthropic thinking-prefill sanitizer on the OpenRouter `openai-completions` path for Anthropic model IDs, while direct Anthropic and Cloudflare Anthropic routes do. The report is recent, concrete, and the only linked fixing PR was closed unmerged.

## What This Changes

The issue requests stripping trailing assistant prefill turns for Anthropic models routed through OpenRouter's OpenAI-compatible adapter when reasoning/thinking is enabled.

## Best Possible Solution

Add an OpenRouter-owned wrapper or replay-policy extension that detects Anthropic model IDs on verified OpenRouter OpenAI-compatible routes and reuses the shared Anthropic prefill sanitizer only when thinking/reasoning is enabled, with focused regression tests and a changelog entry.

## Reproduction Assessment

Yes. The issue provides concrete config and repeated provider 400 logs, and current-main source inspection shows the OpenRouter path still lacks the sanitizer used by direct Anthropic and Cloudflare Anthropic routes.

## Solution Assessment

Yes. A provider-local OpenRouter fix is the narrowest maintainable solution; applying this to every `openai-completions` provider would risk changing unrelated proxy behavior.
