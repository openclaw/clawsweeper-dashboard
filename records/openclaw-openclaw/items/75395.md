## Summary

Keep open. Current main still lacks the OpenRouter-specific sanitizer for Anthropic-routed `openai-completions` payloads, and the linked open PR #75400 explicitly closes this issue, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that Claude Opus 4.6 routed through OpenRouter fails because OpenClaw does not strip trailing assistant-prefill turns on the OpenRouter OpenAI-compatible path.

## Best Possible Solution

Land #75400 or an equivalent provider-local OpenRouter wrapper change that reuses the shared Anthropic thinking-prefill sanitizer only for Anthropic-routed OpenRouter payloads, with focused regression coverage preserving existing OpenRouter routing and reasoning behavior.

## Reproduction Assessment

Yes. The issue provides concrete OpenRouter configuration and repeated 400 logs, and current-main source inspection shows the OpenRouter stream path does not apply the sanitizer already used by direct Anthropic and Cloudflare AI Gateway paths.

## Solution Assessment

Yes. A narrow OpenRouter provider wrapper that reuses the shared Anthropic prefill sanitizer is more maintainable than changing the generic `openai-completions` path for every compatible provider.
