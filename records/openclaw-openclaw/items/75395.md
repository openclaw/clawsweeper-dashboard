## Summary

Keep open. Current main still leaves OpenRouter on the proxy/OpenAI-compatible path without the Anthropic assistant-prefill payload sanitizer, while the direct Anthropic and Cloudflare AI Gateway routes already use that shared sanitizer. The linked fix PR #75400 was closed unmerged, so there is no active implementation candidate to wait on.

## What This Changes

OpenRouter Anthropic models using the `openai-completions` adapter should strip trailing assistant prefill payloads when reasoning/thinking is enabled so translated Anthropic requests end on a user message.

## Best Possible Solution

Add a provider-local OpenRouter wrapper that detects Anthropic model IDs on verified OpenRouter `openai-completions` routes and strips trailing non-tool assistant prefill payloads when proxy reasoning is enabled, reusing or extending the shared Anthropic sanitizer with focused tests and a user-facing changelog entry.

## Reproduction Assessment

Yes. The issue gives a concrete OpenRouter configuration and repeated 400 logs, and current-main source inspection shows the OpenRouter path lacks the sanitizer already used by direct Anthropic and Cloudflare AI Gateway routes.

## Solution Assessment

Yes. A narrow OpenRouter-owned stream-wrapper or replay-policy extension is the best fit because it fixes Anthropic-routed OpenRouter requests without changing every `openai-completions` provider.

## Repair Work Prompt

Fix the OpenRouter Anthropic assistant-prefill regression tracked by this issue. Current main strips trailing assistant prefill payloads for direct Anthropic and Cloudflare AI Gateway, but not for Anthropic model IDs routed through OpenRouter's `openai-completions` path with proxy reasoning enabled. Implement this in the OpenRouter/provider-stream boundary, not as a broad mutation for every `openai-completions` provider. Prefer reusing or extending `createAnthropicThinkingPrefillPayloadWrapper` / `stripTrailingAnthropicAssistantPrefillWhenThinking` so OpenRouter Anthropic requests strip trailing non-tool assistant messages when `reasoning` or equivalent OpenAI-compatible thinking is enabled. Preserve assistant tool-call turns. Add focused coverage for OpenRouter Anthropic prefill stripping and any shared helper behavior needed for proxy reasoning payloads. Add a user-facing `CHANGELOG.md` fix entry with reporter attribution. Do not change unrelated provider replay policies, OpenRouter auth, catalog, pricing, or generic OpenAI request shaping.
