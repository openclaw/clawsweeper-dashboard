## Repair Work Prompt

Fix the OpenRouter Anthropic assistant-prefill regression tracked by https://github.com/openclaw/openclaw/issues/75395. Current main applies Anthropic prefill stripping for direct Anthropic and Cloudflare Anthropic routes, but not for OpenRouter `openai-completions` requests to Anthropic model IDs. Implement a provider-local OpenRouter wrapper or narrowly shared helper that strips trailing non-tool assistant prefill turns only for verified OpenRouter routes targeting Anthropic model IDs when reasoning/thinking is enabled. Reuse `stripTrailingAnthropicAssistantPrefillWhenThinking` / `createAnthropicThinkingPrefillPayloadWrapper` where possible, but account for OpenRouter's OpenAI-compatible `reasoning` / `reasoning_effort` payload shape rather than native Anthropic `thinking`. Preserve assistant tool-call tails, do not apply this globally to all `openai-completions` providers, and do not change dependency pins or unrelated provider behavior. Add focused regression tests for OpenRouter Anthropic prefill stripping and shared predicate behavior, update `CHANGELOG.md`, and update OpenRouter/transcript-hygiene docs if the new behavior is documented.
