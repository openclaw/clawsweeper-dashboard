## Summary

Keep open as a reviewable bug-fix PR: source review found the current-main stale-provider route bug and no blocking patch findings, but the PR still needs normal maintainer/CI handling because it changes compatibility-sensitive provider, auth, and session routing behavior.

## What This Changes

The PR adds catalog-aware validation to gateway session model resolution and wires `chat.send` to ignore stale persisted provider/model routes that are no longer visible in current config or catalog state.

## Best Possible Solution

Land the runtime routing fix after current CI and maintainer acceptance of the catalog-unavailable compatibility fallback, while leaving broader doctor/reset and fallback-policy work to the related issues.

## Reproduction Assessment

Yes. Current main is source-reproducible because `resolveSessionModelRef` returns persisted override/runtime provider state before checking current config or catalog visibility, and `chat.send` uses that result for provider auth and dispatch.

## Solution Assessment

Yes, with a compatibility caveat. The resolver plus `chat.send` catalog-visibility gate is the narrow runtime layer for the reported route bug; the only open product choice is whether merge-mode catalog load failure should preserve or reject non-config persisted refs.
