## Summary

This PR still needs changes before merge: the new catalog-load predicate misses provider-qualified model-only persisted session fields, so one stale deleted-provider shape can still route without catalog validation.

## What This Changes

The PR adds catalog-aware validation to gateway session model resolution, wires `chat.send` to pass model catalog visibility for risky persisted session routes, and adds gateway/session regression tests.

## Best Possible Solution

Make the catalog-load predicate derive the same persisted candidate refs that `resolveSessionModelRef` can return, then add resolver and `chat.send` regressions for model-only stale-provider entries while preserving the no-catalog compatibility cases.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a session entry with only provider-qualified `modelOverride` or only provider-qualified `model` makes the new predicate return false while the resolver can still return that provider without catalog visibility.

## Solution Assessment

No. The patch is a good direction, but it is not yet the best fix because the predicate and resolver are not symmetric for all persisted session ref shapes.
