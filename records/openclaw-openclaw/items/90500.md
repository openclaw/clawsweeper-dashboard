## Summary

Keep this PR open, but it is not merge-ready: the patch adds useful resolver-level validation, yet the actual agent-run path can still re-read stale persisted provider/model state, and required catalog validation still fails open.

## Best Possible Solution

Use one canonical catalog-validated provider/model decision for both gateway metadata and the actual agent run, define explicit required catalog-load failure behavior, and prove the deleted-provider `chat.send` path with redacted live output or logs.
