## Summary

Keep open: the patch still has a P1 correctness blocker because the catalog-aware model fallback is computed in `chat.send` metadata, but the downstream auto-reply model-selection path can still reselect the stale persisted provider/model that actually executes the turn. ## What This Changes

The PR adds catalog-aware gateway session model resolution for removed persisted providers, wires it into `chat.send`, and adds resolver plus gateway regression tests.

## Best Possible Solution
