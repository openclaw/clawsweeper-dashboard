## Summary

Keep this PR open: it targets the right gateway session resolver and includes useful regression coverage, but the new provider-row guard treats default merge-mode `models.providers` overlays as an exhaustive allowlist, which can drop valid existing session selections back to the default provider.

## What This Changes

This PR changes gateway session model resolution to ignore persisted override/runtime provider routes when their provider is absent from `models.providers`, and adds resolver regression tests.

## Best Possible Solution

Keep the fix in `resolveSessionModelRef`, but validate persisted refs through the existing model visibility/catalog semantics, especially `models.mode: "replace"` versus default merge, so removed custom providers are rejected without dropping valid provider-plugin selections.

## Reproduction Assessment

Yes. Current main is source-reproducible because `resolveSessionModelRef` returns persisted override/runtime provider state before checking current config, and chat send consumes that resolved provider for routing.

## Solution Assessment

No. The resolver is the right layer, but using any `models.providers` row as the invalidation boundary is too broad for OpenClaw's documented default merge-mode provider catalog behavior.
