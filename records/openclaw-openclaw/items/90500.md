## Summary

Keep this PR open: the main stale-provider route fix is useful, but the updated head still leaves provider-qualified model-only persisted refs outside the catalog validation predicate, so part of the deleted-provider billing/auth routing bug remains source-reproducible.

## What This Changes

The branch adds catalog-aware validation to gateway session model resolution, wires `chat.send` to load catalog visibility when persisted session routes may need validation, and adds resolver/chat regression tests.

## Best Possible Solution

Make the catalog-load predicate derive the same candidate refs that `resolveSessionModelRef` may accept, add model-only stale-provider regressions, and keep the compatibility-preserving no-catalog cases covered.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: a session entry with only a provider-qualified `modelOverride` or only a provider-qualified `model` makes the new predicate return false while the resolver can still return that provider without catalog visibility.

## Solution Assessment

No; the patch is a good direction but not yet the best fix because the predicate and resolver are not symmetric for all persisted session ref shapes.
