## Summary

## What This Changes

The PR adds catalog-aware validation to gateway session model resolution so stale persisted provider/model routes can fall back to the current configured default when the provider is no longer visible.

## Best Possible Solution

Keep the gateway resolver as the owner, narrow catalog validation to actual non-default persisted route cases, preserve the deleted-provider and merge-mode tests, and have maintainers explicitly accept the runtime fallback policy or track doctor/reset visibility separately.

## Reproduction Assessment

## Solution Assessment

No, not as-is. The resolver boundary is the right place for this mitigation, but the current helper is broader than necessary and should avoid loading the model catalog for sessions with no persisted route to validate.
