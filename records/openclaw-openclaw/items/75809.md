## Summary

Keep open. The cold-start healthcheck bug is valid, but this PR is incomplete because the bundled Compose service defines a separate 20s healthcheck grace period that remains shorter than the documented startup envelope.

## What This Changes

The PR lengthens the root Dockerfile HEALTHCHECK start period from 15s to 600s and adds an Unreleased changelog fix entry.

## Best Possible Solution

Land one narrow Docker operator fix that aligns the image and bundled Compose healthcheck grace periods to the cold-start envelope, preserves the existing probe command and retry behavior, and adds static drift coverage.

## Reproduction Assessment

Yes. The linked issue gives a concrete `docker run` plus `docker ps` unhealthy-state check, and current main statically confirms the short Dockerfile and Compose grace periods.

## Solution Assessment

No. Raising only the Dockerfile value is directionally correct, but it leaves Compose users on an independent 20s grace period and adds no guard against future drift.
