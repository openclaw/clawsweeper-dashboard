## Summary

Keep open: the Docker cold-start healthcheck bug is valid and this PR's image-level change is useful, but the patch is incomplete because the bundled Compose service still has a separate 20s healthcheck grace period that can override the Dockerfile value.

## What This Changes

The PR raises the root Dockerfile OCI healthcheck start period from 15 seconds to 600 seconds and adds an Unreleased changelog fix entry.

## Best Possible Solution

Land one narrow Docker operator fix that aligns Dockerfile and Compose healthcheck start periods to the documented cold-start envelope, preserves the `/healthz` probe and existing interval/timeout/retry behavior unless intentionally changed, and adds static drift coverage.

## Reproduction Assessment

Yes. The linked issue provides a concrete `docker run` plus `docker ps` check, and current main/PR static inspection confirms the short image and Compose grace periods that drive the premature unhealthy state.

## Solution Assessment

No. Raising the Dockerfile value is directionally correct, but it is not the best complete fix while bundled Compose keeps an overriding 20s service-level `start_period`.
