## Summary

Keep open: the PR fixes the image-level Dockerfile healthcheck grace period, but it leaves the bundled Compose healthcheck at a shorter service-level start period that can still override the image value for Compose users. The blocker is narrow and mechanical: align Compose and add static guard coverage before merge.

## What This Changes

The PR changes the root Dockerfile HEALTHCHECK start period from 15s to 600s and adds an Unreleased changelog entry for the Docker cold-start healthcheck behavior.

## Best Possible Solution

Land one narrow Docker operator fix that applies the same issue-backed cold-start grace period to the image HEALTHCHECK and the bundled Compose healthcheck, keeps the /healthz probe and existing interval/timeout/retry behavior, and adds a static guard so the two values do not drift again.
