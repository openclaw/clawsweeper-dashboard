## What This Changes

The issue requests clearer CLI output, help text, and docs so `openclaw mcp list` is not mistaken for a combined listing of OpenClaw `mcp.servers` and mcporter `config/mcporter.json` registries.

## Best Possible Solution

Review and land PR #73503, or an equivalent focused patch, so the human CLI output/help and public MCP docs explicitly label OpenClaw-managed `mcp.servers` and point mcporter users to `mcporter list` for `config/mcporter.json`, then let the issue close after merge.
