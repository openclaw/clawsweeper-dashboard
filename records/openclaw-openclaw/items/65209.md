## Summary

Keep open. Current main still has the reported UX gap: `openclaw mcp list` reads only OpenClaw `mcp.servers`, but the human CLI label/help remain generic, the MCP docs do not call out the mcporter registry distinction, and the focused follow-up PRs are closed unmerged.

## What This Changes

Clarify that `openclaw mcp list` lists only OpenClaw-managed `mcp.servers` entries, not mcporter servers from `config/mcporter.json`.

## Best Possible Solution

Ship a focused CLI help/output, docs, test, and changelog clarification while preserving the existing registry behavior and machine-readable JSON output.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: configure OpenClaw `mcp.servers` separately from `config/mcporter.json`, then run human `openclaw mcp list`; current code prints generic MCP server wording while reading only `sourceConfig.mcp?.servers`.

## Solution Assessment

Yes. The narrowest maintainable fix is wording plus documentation and a regression test; adding mcporter discovery or an all-sources mode would be broader product work beyond this UX mismatch.
